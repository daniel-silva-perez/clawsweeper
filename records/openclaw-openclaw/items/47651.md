## Summary

Keep open. Current main has shipped the narrower Gemini Embedding 2 image/audio indexing path for files under memorySearch.extraPaths, but the requested PDF/video modalities and explicit user-sent-media ingestion workflow are still missing from the public memory contract.

## What This Changes

The issue requests native Gemini Embedding 2 indexing for remembered images, video, audio, PDFs, and other inbound media in the memory search pipeline.

## Best Possible Solution

Keep the shipped image/audio extraPaths foundation, then split the remaining work into a public PDF/video modality contract and a separate safe ingestion path for attachments users explicitly ask the agent to remember.
