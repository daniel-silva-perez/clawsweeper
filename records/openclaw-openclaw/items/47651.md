## Summary

Keep open. Current main has a narrower Gemini Embedding 2 path for opt-in image and audio files under `memorySearch.extraPaths`, but it still lacks PDF/video modalities and the user-sent-media ingestion workflow described in the issue.

## What This Changes

The issue requests indexing non-Markdown images, video, audio, and PDFs, including user-sent media marked worth remembering, through Gemini Embedding 2 multimodal embeddings in the memory search pipeline.

## Best Possible Solution

Keep the shipped image/audio `extraPaths` foundation, then split the remaining work into a PDF/video multimodal memory contract and a separate explicit-remember attachment ingestion path with clear storage, deletion, consent, and provider-upload behavior.
