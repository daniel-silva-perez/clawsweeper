## Summary

Keep open. Current main shipped a useful subset: Gemini Embedding 2 can index image and audio files from configured extra paths, with provider and fallback gating. The broader request still has unimplemented scope around PDF/video modalities and a safe workflow for user-sent media that should be remembered, and the open #44734 only tracks a narrower audio-format regression.

## What This Changes

Request to index raw images, audio, video, and PDFs in memory search with Gemini Embedding 2, including media sent to an agent and marked worth remembering.

## Best Possible Solution

Keep the shipped opt-in image/audio extraPaths implementation as the foundation, then split or explicitly define the remaining product scope: PDF/video modalities in the memory indexing contract, and a separate safe persisted-media ingestion path for attachments users ask the agent to remember. Leave #44734 as the narrower OGG/Opus audio-format tracker.
