## Summary

Keep open. Current main still implements only the narrower Gemini Embedding 2 image/audio indexing path for files discovered through `memorySearch.extraPaths`; the requested PDF/video modalities and a user-sent-media remember workflow remain outside the public memory contract.

## What This Changes

Request support for Gemini Embedding 2 multimodal memory indexing of non-Markdown media, including images, video, audio, PDFs, and user-sent media marked worth remembering.

## Best Possible Solution

Keep the shipped image/audio `extraPaths` foundation, then split the remaining work into a defined PDF/video multimodal memory contract and a separate safe ingestion design for attachments users explicitly ask the agent to remember.

## Reproduction Assessment

Not applicable as a bug reproduction. There is a high-confidence current-main verification path by inspecting the modality classifier, schema, docs, and tests, all of which show image/audio support only and no PDF/video or remembered inbound-media indexing path.

## Solution Assessment

No, the current shipped subset is not a complete solution for this request. The maintainable path is an explicit public contract for new media modalities and retained attachments rather than silently indexing every raw inbound file.
