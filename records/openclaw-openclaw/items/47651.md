## Summary

Keep open. Current main has shipped a useful Gemini Embedding 2 subset for opt-in image and audio files under `memorySearch.extraPaths`, but the requested PDF/video modalities and user-sent-media ingestion path are still missing from the public memory contract.

## What This Changes

Support Gemini Embedding 2 multimodal memory indexing for non-Markdown media, including images, audio, video, PDFs, and media sent to an agent and marked worth remembering.

## Best Possible Solution

Keep the shipped image/audio extraPaths foundation, then split the remaining work into a public PDF/video modality contract and a separate safe persisted-media ingestion path for attachments users explicitly ask the agent to remember.
