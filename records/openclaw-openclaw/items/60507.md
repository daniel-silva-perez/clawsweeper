## Summary

Keep open. Current main does not implement this PR’s destructive-command catalog, and there is no merged replacement. The existing exec approval system matches executable selectors and enforces allowlist segment policy, but it does not include a curated HIL-required catalog for commands like `kubectl delete`, `terraform destroy`, or `rm`. The PR also has unresolved security-review blockers: its documented example and tests use full command-prefix matching rather than the production allowlist semantics.

## Best Possible Solution

Keep this PR open for maintainer review or author revision. The best path is to rework it around production exec parsing and approval semantics, for example a real destructive-command predicate over analyzed argv/segments with tests against the live allowlist/evaluation path, or to split it into advisory docs if maintainers do not want core enforcement. Related denylist-mode work such as #71097 can inform the design, but current main has not implemented this PR’s intended change.
