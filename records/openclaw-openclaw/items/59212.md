## Summary

Keep open. Current main still contains the clipping conditions described by the report: `.usage-left-card` is still grouped under `overflow: clip`, `.daily-chart-bars` still uses `overflow-x: auto`, and the tooltip is still absolutely positioned above each `.daily-bar-wrapper`. The linked fix PR was closed unmerged, and no current-main code or test evidence shows this has been resolved.

## Best Possible Solution

Keep this issue open and fix the Usage tab in `ui/src/styles/usage.css`: remove `.usage-left-card` from the clipped shared rule, then either make `.daily-chart-bars` allow vertical tooltip visibility or move the tooltip to fixed/portal-style positioning if horizontal scrolling must be preserved. Add a focused browser/UI regression for tall daily bars so the tooltip remains readable.
