## Summary

Keep this issue open. Current main has a Memory Palace/report surface and a wiki content preview path, but selecting a report/page card still toggles inline details rather than opening the actual report content. The only explicit content-opening path is the nested `Open wiki page` button, and current tests cover source-page preview behavior but not a mobile report-card tap opening report content or avoiding the Command Overview regression.

## Best Possible Solution

Keep the issue open and add a focused Control UI regression for the mobile-width Dreaming/Memory Palace report list. The product fix should either make tapping a report row/card open the `wiki.get` content preview, or intentionally separate expand-vs-open behavior with clear controls that work on mobile and desktop and cannot fall back to the Command Overview surface.
