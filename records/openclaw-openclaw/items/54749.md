## Summary

Keep this issue open. Current main has adjacent Control UI work for Memory Wiki reports and wiki-page previews, but the reported behavior is not clearly fixed: report cards in the Memory Palace list still use the card click to toggle details, while opening the actual page content is a separate `Open wiki page` button. I did not find a current regression test covering the mobile report-list click path described by the issue.

## Best Possible Solution

Keep the issue open and add a focused Control UI regression around the mobile-width Dreaming/Memory Wiki report list. The likely product fix is to make selecting a report row/card open the `wiki.get` content preview, or clearly separate expand-vs-open behavior in a way that works on mobile and desktop.
