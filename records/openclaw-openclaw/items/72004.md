## Summary

Keep this PR open. Current main still has the stream/finalization ordering the PR targets: active-run `final` and `aborted` events append assistant messages to `state.chatMessages` before clearing `state.chatStream`, while the Control UI render path can still render both committed history and the live stream. The PR is a focused three-file patch, linked bug #71992 remains open, prior equivalent PR #34164 is closed unmerged, and the diff has no security-sensitive surface changes.

## Best Possible Solution

Review and land this PR, or an equivalent patch, as the focused fix for #71992. The implementation should snapshot streamed fallback text, clear `chatStream`/`chatRunId`/`chatStreamStartedAt` before mutating `chatMessages` in the active-run `final` and `aborted` branches, keep the ordering regression test, and keep the changelog attribution.
