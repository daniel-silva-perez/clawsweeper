Keep open: current main still lacks the credential-placement guidance, and the PR's docs-only change matches the current schema and runtime credential resolution path. I found no blocking functional or security issue in the proposed patch, so this should proceed through normal maintainer review.

## What This Changes

Adds a Provider credentials section to `docs/nodes/media-understanding.md` explaining that `tools.media` model entries do not read `apiKey`, listing auth profiles, provider environment variables, and `models.providers.<providerId>`, with a custom-provider config example.

## Best Possible Solution

Land this focused docs clarification, or an equivalent maintainer-edited version, after normal docs validation. A runtime change to accept direct `apiKey` inside `tools.media.*.models[]` should remain a separate product/API decision if maintainers want that behavior later.

## Reproduction Assessment

## Solution Assessment

Yes. A focused documentation update is the narrowest maintainable solution for the reported setup failure because current runtime behavior already centralizes provider credentials outside `tools.media` model entries.
