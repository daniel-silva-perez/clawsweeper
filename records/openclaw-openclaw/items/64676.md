## Summary

Keep open. Current main still reaches per-agent `models.json` preparation during gateway startup and the merge path still regenerates matching provider model arrays from config/catalog data while preserving only selected `apiKey` and `baseUrl` fields. The docs describe this generated/merged behavior, but the issue asks whether direct user pruning of per-agent model lists should persist, so this needs maintainer product/API judgment rather than cleanup closure.

## What This Changes

Preserve user-edited per-agent `models.json` model lists across gateway restarts instead of restoring default or cached provider models.

## Best Possible Solution

Maintainers should define the per-agent `models.json` contract. If direct pruning is supported, preserve deletions with an explicit tombstone or replace mechanism and add a gateway-restart regression. If the file is generated registry state, align CLI, doctor/setup flows, and public docs around persistent pruning through `models.providers`, `agents.defaults.models`, or an explicit replace workflow.
