## Summary

Keep open. Current main still reaches `models.json` preparation during gateway startup and can rewrite the active agent file when the generated plan differs. In merge mode, matching provider entries are regenerated from config/catalog data while only selected `apiKey` and `baseUrl` fields are preserved, and the current docs describe that generated behavior rather than guaranteeing manual model-list pruning. The newer model visibility fix does not change this persistence path.

## What This Changes

The issue asks for user-edited per-agent `models.json` model lists to persist across gateway restarts instead of being restored from generated/default provider catalog state.

## Best Possible Solution

Maintainers should decide the intended contract for per-agent `models.json`. If direct pruning is supported, preserve deletions through an explicit tombstone or replace mechanism and add a gateway-restart regression. If it remains generated state, align CLI, doctor/setup flows, and docs around persistent pruning through `models.providers`, `agents.defaults.models`, or an explicit replace flow.
