## Summary

Keep open. Current main still prepares and may rewrite the active agent `models.json` during gateway startup, and the merge path still refreshes matching provider model arrays from config/catalog data while preserving only selected secret/base URL fields. The docs now describe that current precedence, but the issue asks whether user-pruned per-agent model lists should persist, so this needs maintainer product/API judgment rather than an autoclose.

## What This Changes

The issue asks for gateway restarts to preserve manually edited per-agent `models.json` provider model lists instead of restoring default or catalog-backed entries.

## Best Possible Solution

Maintainers should decide the intended contract for direct edits to per-agent `models.json`. If pruned model arrays are supported, add a gateway-startup regression around `ensureOpenClawModelsJson` and preserve deletions through a tombstone or explicit replace mechanism. If `models.json` is generated registry state, update user-facing guidance, doctor/config help, and setup flows so persistent pruning goes through `models.providers` or `agents.defaults.models` with the intended `--merge`/`--replace` path.
