## Summary

Keep open. Current main has memory embedding provider lookup improvements and docs that describe custom provider ids, but the reported Bailian/DashScope shape is still not proved end to end: `api: "openai-completions"` does not map to a registered memory embedding adapter, and the OpenAI memory adapter still resolves endpoint/auth through the hardcoded `openai` provider path.

## Best Possible Solution

Keep this issue open and fix or document the provider contract explicitly. The best code path is a targeted regression test for `agents.defaults.memorySearch.provider: "bailian-embedding"` with `models.providers.bailian-embedding.api: "openai-completions"`, then either alias `openai-completions` to the OpenAI-compatible memory adapter while preserving the custom provider id for endpoint/auth, or document the supported `memorySearch.provider: "openai"` plus `memorySearch.remote.baseUrl/apiKey` configuration and emit a clearer diagnostic for unsupported custom-provider shapes.
