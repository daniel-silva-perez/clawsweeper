## Summary

Keep open. Current main has partial custom memory provider lookup, but the reported Bailian/DashScope OpenAI-compatible path is still not implemented end to end: `api: "openai-completions"` is not a registered memory embedding adapter, and the OpenAI memory adapter still resolves endpoint/auth through the hardcoded `openai` provider instead of preserving a custom provider id such as `bailian-embedding`.

## Best Possible Solution

Keep this issue open and add a focused regression for `agents.defaults.memorySearch.provider: "bailian-embedding"` with `models.providers.bailian-embedding.api: "openai-completions"`, `baseUrl`, `apiKey`, and `models: [{ id: "text-embedding-v3" }]`. Then either alias `openai-completions` to the OpenAI-compatible memory embedding adapter while preserving the requested provider id for endpoint/auth/model-prefix handling, or document the supported `memorySearch.remote.baseUrl/apiKey` workaround and emit a clear unsupported-adapter diagnostic.
