## Summary

Keep open. Current main still matches the source-level duplicate prompt path: main-session cron text is queued as a `cron:` system event, heartbeat reads the same queued event, and `delivery.mode: "none"` only changes relayability, not reminder-prompt construction.

## What This Changes

The issue requests that main-session cron `systemEvent` jobs with `delivery.mode: "none"` inject only the raw system event without adding a scheduled-reminder heartbeat wrapper.

## Best Possible Solution

Land a focused cron/heartbeat fix that marks main-session-injected cron `systemEvent` entries as already transcript-visible, or otherwise excludes only that tagged class from cron reminder prompt construction, with regression coverage preserving generic cron reminders and delivery semantics.
