## Summary

Current main still matches the duplicate prompt path, and the issue is already paired with open targeted fix PR #44973. It should remain open until that PR, or an equivalent narrow replacement, lands.

## What This Changes

The issue requests that main-session cron systemEvent jobs with delivery.mode none inject only the intended system event and not also add the scheduled-reminder heartbeat wrapper.

## Best Possible Solution

Land PR #44973 or an equivalent narrow fix that marks main-session cron systemEvent injections and skips only those already-injected events during cron reminder prompt construction, while preserving isolated cron reminder behavior and outbound delivery semantics.

## Reproduction Assessment

Yes. The issue includes a concrete cron job configuration, and current-main source inspection shows the same cron-tagged systemEvent can be both rendered as a queued system event and wrapped into the heartbeat cron prompt when delivery is disabled.

## Solution Assessment

Yes. The tag-and-skip direction discussed on the issue is the narrowest maintainable fix because it deduplicates prompt construction for main-session injected cron events without changing delivery.mode semantics or broad outbound delivery behavior.
