## Summary

Current main still matches the reported duplicate prompt path, and the issue is paired with the open targeted fix PR #44973. This should remain open until that PR, or an equivalent narrow replacement, lands.

## What This Changes

The issue asks OpenClaw to stop main-session cron systemEvent jobs with `delivery.mode: "none"` from adding a redundant scheduled-reminder heartbeat wrapper in addition to the raw system event.

## Best Possible Solution

Land PR #44973 or an equivalent narrow fix that marks main-session cron systemEvent injections and skips only those already-injected events during cron reminder prompt construction, while preserving isolated cron reminder behavior and outbound delivery semantics.

## Reproduction Assessment

Yes. The issue provides a concrete job configuration, and current-main source inspection shows that the same cron-tagged systemEvent can be both rendered as a queued system event and wrapped into the heartbeat cron prompt when delivery is disabled.

## Solution Assessment

Yes. The tag-and-skip direction discussed on the issue is the narrowest maintainable fix because it deduplicates prompt construction for main-session injected cron events without adding broad outbound dedupe or changing `delivery.mode` semantics.
