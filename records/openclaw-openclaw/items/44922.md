Current main still matches the reported cron/heartbeat duplication path, and the issue has a focused open fix PR (#44973) that tags main-session cron system events and skips the redundant reminder wrapper. This should stay open until that PR is reviewed, rebased, replaced, or merged.

## What This Changes

The issue asks for main-session cron `systemEvent` jobs with `delivery.mode: "none"` to avoid adding a duplicate scheduled-reminder wrapper to the heartbeat prompt.

## Best Possible Solution

Land a focused fix that marks main-session cron system events as already injected and excludes only those tagged events from cron reminder prompt construction, with regression coverage proving the raw system event remains visible while the duplicate wrapper is absent.
