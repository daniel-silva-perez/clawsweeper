## Summary

Keep open. The branch addresses a real linked Homebrew auto-update gap, but the proposed Homebrew skip path has a blocking notification-state bug that should be fixed on the contributor PR before merge.

## What This Changes

The PR adds Homebrew package-root detection, skips the gateway auto-update child for Homebrew-managed installs with a `brew upgrade openclaw` log hint, adds focused infra tests, and updates the changelog.

## Best Possible Solution

Keep the PR focused on explicit Homebrew unsupported-auto-apply feedback, but update the Homebrew skip branch to persist `lastNotifiedVersion` and `lastNotifiedTag` whenever it logs the skip hint, with regression coverage for `update.checkOnStart: false`.

## Reproduction Assessment

Yes. Current main already has a test proving auto-update runs with `update.checkOnStart: false`; combining that mode with the PR's Homebrew branch shows the hint logs and writes state without recording the version/tag as notified.

## Solution Assessment

No as proposed. The skip-and-hint approach is a narrow maintainable fix for Homebrew installs, but the PR needs to persist notification state inside the new Homebrew branch before it is mergeable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Persist the Homebrew skip notification:** `src/infra/update-startup.ts:440`
  - body: When `update.checkOnStart` is false, the normal startup-hint branch is skipped, so this new Homebrew branch can log the skip hint and then write `nextState` without setting `lastNotifiedVersion` or `lastNotifiedTag`. That makes the claimed one-shot hint repeat on later eligible auto-update checks for the same version/tag; set those fields whenever this branch logs the Homebrew hint and cover the disabled-startup-hint case.
  - confidence: 0.88
