## Summary

Keep this PR open. The Homebrew skip-and-hint direction is a plausible narrow fix for the linked Homebrew auto-update report, but the proposed branch currently fails its own one-shot hint contract when startup update hints are disabled.

## What This Changes

The PR adds Homebrew install path detection, skips gateway auto-update for Homebrew-managed package roots with a `brew upgrade openclaw` log hint, adds infra tests, and updates the changelog.

## Best Possible Solution

Keep the PR focused on Homebrew-aware auto-update feedback, but persist the skip notification state inside the new Homebrew branch whenever the skip hint is logged and add regression coverage for disabled startup hints.

## Reproduction Assessment

Yes. Current main has a test proving auto-update still runs when `update.checkOnStart` is false; applying the PR's new Homebrew branch to that mode shows it can log the skip hint and write state without recording the version/tag as notified.

## Solution Assessment

No as proposed. The skip-and-hint design is maintainable if that is the intended Homebrew contract, but the branch needs to persist notification state before it is safe to merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Persist the Homebrew skip notification:** `src/infra/update-startup.ts:439-440`
  - body: When `update.checkOnStart` is false, the normal startup-hint branch is skipped, so this new Homebrew branch can log the skip hint and then write `nextState` without setting `lastNotifiedVersion` or `lastNotifiedTag`. That makes the claimed one-shot hint repeat on later eligible auto-update checks for the same version/tag; set those fields whenever this branch logs the Homebrew hint and cover the disabled-startup-hint case.
