## Summary

Keep open. The Homebrew skip-and-hint direction is narrow, but the PR branch still has a blocking notification-state bug: with `update.checkOnStart: false`, it can log the Homebrew skip hint without persisting the `(version, tag)` as notified, so the hint repeats for the same update. Current main does not already implement this Homebrew auto-update skip, and related issue #74754 remains paired with this PR until the branch is amended or closed.

## What This Changes

The PR adds Homebrew install path detection, skips gateway auto-update for Homebrew-managed package roots with a `brew upgrade openclaw` log hint, adds infra tests, and updates the changelog.

## Best Possible Solution

Keep the PR focused on Homebrew-aware auto-update feedback, but persist the Homebrew skip notification state whenever that branch logs and add regression coverage for `update.checkOnStart: false`. CLI-side Homebrew update parity can remain a separate follow-up if maintainers want it.

## Reproduction Assessment

Yes. Current main has a test proving auto-update still runs when `checkOnStart` is false, and the PR diff shows the new Homebrew early-return path would log from stale `state.lastNotified*` without updating `nextState.lastNotified*`.

## Solution Assessment

No as proposed. The skip-and-hint design is a maintainable fix for Homebrew-managed installs, but the branch must persist notification state before it satisfies the PR's one-shot hint contract.
