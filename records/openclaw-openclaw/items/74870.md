## Summary

Keep this PR open: the branch is useful for the Homebrew auto-update problem, but it still has a blocking state-persistence bug in the new skip path. The PR head has not changed since the existing ClawSweeper review, and current main does not already implement the requested Homebrew-specific skip/hint behavior.

## What This Changes

The PR adds Homebrew package-root detection, skips gateway auto-update for Homebrew-managed installs with a `brew upgrade openclaw` log hint, adds infra tests, and adds a changelog entry.

## Best Possible Solution

Keep the PR focused on Homebrew-aware auto-update feedback, but persist the skip notification state inside the Homebrew branch whenever the skip hint is logged and add regression coverage for `update.checkOnStart: false`.

## Reproduction Assessment

Yes. Current main already has a test proving auto-update still runs when `update.checkOnStart` is false; with the PR's new Homebrew branch in that mode, the skip hint can be logged without recording the version/tag as notified.

## Solution Assessment

No as proposed. The skip-and-hint direction is maintainable for Homebrew installs, but the implementation needs to persist notification state before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Persist the Homebrew skip notification:** `src/infra/update-startup.ts:439-440`
  - body: When `update.checkOnStart` is false, the normal startup-hint branch is skipped, so this new Homebrew branch can log the skip hint and then write `nextState` without setting `lastNotifiedVersion` or `lastNotifiedTag`. That makes the claimed one-shot hint repeat on later eligible auto-update checks for the same version/tag; set those fields whenever this branch logs the Homebrew hint and cover the disabled-startup-hint case.
  - confidence: 0.88
