## Summary

Keep this PR open. The Homebrew auto-update diagnosis is valid and the branch is pointed at the right updater path, but the proposed patch still has blocking correctness issues: it can repeat the new skip hint when startup update hints are disabled, and its opt-path detector can misclassify supported custom-prefix npm installs under `/opt/openclaw` as Homebrew-managed.

## What This Changes

The branch adds Homebrew install detection, skips gateway auto-update for detected Homebrew package roots with a `brew upgrade openclaw` log hint, adds infra tests, and updates the changelog.

## Best Possible Solution

Keep the Homebrew-aware skip-and-hint behavior, but persist the skip notification fields whenever the new branch logs the hint, add disabled-`checkOnStart` regression coverage, and constrain Homebrew detection to actual Cellar/opt Homebrew layouts without catching supported npm prefixes.

## Reproduction Assessment

Yes. A source-level reproduction is clear from current main and the PR diff: enable auto-update with `update.checkOnStart: false` on a Homebrew-shaped package root, and the branch logs the hint without writing `lastNotifiedVersion`/`lastNotifiedTag`; a second focused check is the documented `/opt/openclaw` npm prefix, which the new opt regex would match as Homebrew.

## Solution Assessment

No as proposed. Skipping the npm auto-update path for real Homebrew installs is a maintainable direction, but the branch needs narrower detection and notification-state persistence before it is the safest fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Persist the Homebrew skip notification:** `src/infra/update-startup.ts:439-440`
  - body: When `update.checkOnStart` is false, the normal startup-hint branch is skipped, so this new Homebrew branch can log the skip hint and then write `nextState` without setting `lastNotifiedVersion` or `lastNotifiedTag`. That makes the claimed one-shot hint repeat on later eligible auto-update checks for the same version/tag; set those fields whenever this branch logs the Homebrew hint and cover the disabled-startup-hint case.
  - confidence: 0.88
- **[P2] Constrain opt-path detection to Homebrew layouts:** `src/infra/homebrew-install.ts:21`
  - body: `OPT_OPENCLAW_RE` matches any `/opt/openclaw/...` path, but OpenClaw supports npm installs with `--prefix /opt/openclaw`, whose package root is `/opt/openclaw/lib/node_modules/openclaw`. This branch would treat that normal npm prefix as Homebrew and skip its auto-update with a brew hint; require a real Homebrew prefix/Cellar corroboration or reject normal npm global roots.
  - confidence: 0.83
