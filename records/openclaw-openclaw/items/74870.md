## Summary

Keep this PR open. The Homebrew skip-and-hint direction matches the linked Homebrew auto-update bug, but the current patch does not persist the Homebrew notification state when startup hints are disabled, so its stated one-shot hint contract can regress into repeated logging.

## What This Changes

The PR adds Homebrew install detection, skips the Gateway's npm-based auto-update child for Homebrew-managed package roots, logs a `brew upgrade openclaw` hint, adds infra tests, and updates the changelog.

## Best Possible Solution

Keep the PR focused on Homebrew-aware auto-update feedback, but update the Homebrew skip branch to persist `lastNotifiedVersion` and `lastNotifiedTag` whenever it logs the skip hint, with regression coverage for `update.checkOnStart: false`. CLI-side Homebrew parity can remain a separate follow-up.

## Reproduction Assessment

Yes. Current main has a test proving auto-update still runs when `checkOnStart` is false, and the PR diff shows the Homebrew branch logs based on stale `state.lastNotified*` without persisting those fields into `nextState`.

## Solution Assessment

No as proposed. Skipping the doomed npm auto-update and giving a `brew upgrade openclaw` hint is a narrow maintainable solution for Homebrew installs, but the implementation must persist the Homebrew notification state before merge.

## Review Findings
