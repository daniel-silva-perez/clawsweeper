## Summary

Keep open. Current main has a WhatsApp video payload path, but it still reports success from the Baileys `sendMessage` result/key and does not verify that outbound video media actually becomes visible to the recipient. The existing tests cover payload construction, not the reported provider-visible delivery failure, and the related closed listener-registry issues are a different failure mode from this issue’s “success message id but no visible video” report.

## Best Possible Solution

Keep this issue open and investigate in the WhatsApp plugin. The best fix is to reproduce with a minimal MP4 against a live linked WhatsApp session, trace Baileys media upload/send acknowledgements, and change `extensions/whatsapp` so video sends either produce a visible playable message or surface a concrete failure instead of treating the returned message key alone as success. Add focused regression coverage around the chosen error/diagnostic behavior and retain the existing payload-shape tests.
