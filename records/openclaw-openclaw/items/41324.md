## Summary

Current main still does not implement this PR's intended Feishu Bitable requester auto-grant behavior. The Bitable create path lacks a requester permission grant and `grant_to_requester` schema wiring, while Feishu doc creation already has the analogous requester grant behavior. This PR remains a focused implementation candidate, but the open review concerns are still actionable.

## Best Possible Solution

Keep this PR open as the focused Feishu plugin implementation candidate. The best path is to rebase it on current main, add Bitable requester-grant tests mirroring the existing doc tests, validate the Feishu permission API response before reporting success, and handle or explicitly skip `user_id` fallback senders before maintainer review/merge.
