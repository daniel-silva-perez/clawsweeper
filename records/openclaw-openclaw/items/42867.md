## Summary

Keep open. Current main has fixed the heartbeat-aware daily/idle freshness problem, but it still lacks a public turn-count TTL setting, schema support, and any reuse-time turn-count check. The overlapping #45390 is open, broader, and from the same author, so it is not a merged or maintainer-designated canonical replacement.

## What This Changes

The issue requests a configurable session reset policy that rotates a conversation after a maximum number of real turns and prevents heartbeat maintenance traffic from keeping abandoned sessions fresh.

## Best Possible Solution

Define a core session TTL contract, then add a schema-validated and documented max-turn reset policy that counts real conversation turns, excludes heartbeat/cron/exec maintenance events, triggers rollover before session reuse, and preserves existing daily and idle reset behavior with focused regression coverage.
