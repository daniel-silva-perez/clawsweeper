## Summary

Keep this issue open. Current main has the heartbeat-aware freshness fix, but the requested turn-count TTL/max-turn session rollover is still missing from the core session reset config, schema, freshness check, reuse path, docs, and tests. The overlapping same-author #45390 is open and broader, but it is not a maintainer-designated canonical replacement.

## Best Possible Solution

Keep this open as a core session lifecycle feature unless maintainers explicitly consolidate it into #45390. A complete implementation should add a schema-validated and documented max-turn reset policy, define which user/assistant turns count versus heartbeat/system events, enforce the check before session reuse/rollover, and add regression coverage across daily, idle, heartbeat, and max-turn interactions.
