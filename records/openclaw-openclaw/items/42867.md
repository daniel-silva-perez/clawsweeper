## Summary

Keep open. Current main fixes the heartbeat-aware freshness half, but the requested turn-count TTL remains absent from the session reset type, strict schema, docs, and reuse path; #45390 overlaps the remaining TTL work but is still open and broader rather than a shipped canonical replacement.

## What This Changes

The issue requests a configurable session turn-count TTL plus heartbeat-aware staleness so maintenance traffic cannot keep oversized sessions alive indefinitely.

## Best Possible Solution

Define one core session TTL contract, then add schema-validated and documented max-turn rotation that counts real conversation turns, excludes maintenance/system-event traffic, rolls before session reuse, and preserves the existing daily/idle freshness behavior.
