## Summary

Current main still keeps automatic config-write backups in a fixed adjacent five-file `.bak` ring. There is no config/schema/UI surface for backup directory, retention count, or trigger policy, and the related implementation PRs were closed unmerged.

## Best Possible Solution

Keep this issue open until maintainers either implement configurable automatic config backup policy or explicitly reject that product direction. A complete fix likely belongs in core config ownership: add typed config/schema/help/docs/UI support for backup directory, retention count, and trigger policy; thread it through `src/config/backup-rotation.ts`, `src/config/io.ts`, `src/config/mutate.ts`, and recovery/audit paths; preserve current defaults; and update rotation/recovery tests. The `openclaw backup` archive command and cron workaround can stay documented alternatives, but they do not replace configurable automatic `.json.bak` rotation.
