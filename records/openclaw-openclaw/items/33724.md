## Summary

Keep this issue open. Current main still hard-codes automatic config-write backups to an adjacent five-file `.bak` ring, with no config/schema/UI surface for backup directory, retention count, or trigger policy. The related implementation PRs #33896 and #34478 were closed unmerged, and `openclaw backup` plus cron are separate workflows rather than configuration for automatic `.json.bak` rotation.

## Best Possible Solution

Keep this issue open until maintainers either implement configurable automatic config backup behavior or explicitly reject that product direction. A complete implementation likely belongs in core config ownership: add typed config/schema/help/docs/UI support for backup directory, retention, and trigger policy; thread the policy through `src/config/backup-rotation.ts`, `src/config/io.ts`, `src/config/mutate.ts`, and recovery/audit paths; preserve current defaults; and update rotation/recovery tests. The cron workaround and `openclaw backup` archive command can remain documented alternatives, but they do not replace configurable automatic config-write rotation.
