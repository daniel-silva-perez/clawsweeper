## Summary

Keep this PR open for normal maintainer review. Current `main` still emits Codex item-event `meta` from the Codex projector but drops it before channel reply options, and Telegram still falls back to the generic item title before the structured tool name, so the PR is not obsolete or superseded.

## What This Changes

The PR forwards Codex item-event `meta` through reply options and updates Telegram streaming progress to display structured tool `name` plus `meta`, with a regression test for `read from README.md`.

## Best Possible Solution

Keep the change narrow: forward `meta` at the generic auto-reply callback boundary, let Telegram own the channel-specific display fallback, and land with the focused Telegram dispatch regression test after maintainer review.

## Reproduction Assessment

Yes. A Codex item event shaped like `{ kind: "tool", title: "Tool", name: "read", meta: "from README.md" }` is enough to trace the current-main failure: `meta` is dropped before Telegram, and Telegram would otherwise choose the generic `Tool` title.

## Solution Assessment

Yes. The proposed approach is the narrowest maintainable fix because it preserves existing `progressText` and `summary` precedence while using structured `name`/`meta` only for tool events before falling back to generic titles.
