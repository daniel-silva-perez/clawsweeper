## Summary

Keep this PR open. Current main still loses Codex item-event `meta` before Telegram rendering and therefore can show the generic `Tool` title instead of actionable structured progress; the PR is a narrow implementation of the requested Telegram behavior and has no blocking review finding from the read-only diff review.

## What This Changes

The PR adds optional item-event `meta` forwarding through auto-reply callbacks and formats Telegram tool progress from structured `name` plus `meta` before falling back to generic item titles.

## Best Possible Solution

Land the narrow callback-and-Telegram formatting change after maintainer review and targeted Telegram validation; keep broader preview parity for Slack, Discord, or Matrix as separate follow-up work if maintainers want it.

## Reproduction Assessment

Yes. A current-main Codex item event shaped like `kind: "tool"`, `title: "Tool"`, `name: "read"`, and `meta: "from README.md"` reproduces the failure path because `meta` is dropped before Telegram and the Telegram fallback then selects `title`.

## Solution Assessment

Yes. The proposed fix is the narrowest maintainable path for the reported Telegram behavior: it adds an optional callback field, forwards already-emitted structured data, and changes only Telegram's fallback after `progressText` and `summary` remain preferred.
