## Summary

Keep this PR open. Current main has adjacent gateway transport-error cleanup, but it still does not implement the PR's two central fixes: the real GatewayClient constructor still converts explicit deviceIdentity:null to undefined, and callGateway still does not preserve GatewayClient.onConnectError in final close errors. The PR also still needs revision around auth semantics because its broad loopback/no-token identity omission can suppress cached device-token auth for paired local sessions.

## Best Possible Solution

Keep this PR open for revision or replacement with a narrow updated patch: preserve explicit deviceIdentity:null in the real GatewayClient constructor, capture GatewayClient.onConnectError inside callGateway and include it in final close transport errors, and retain current main's backend/shared-token loopback carveout so paired local device-token sessions keep working. Add focused regression tests for constructor null preservation, callGateway connect-error propagation, and paired loopback stored-token behavior.
