## Summary

Keep this PR open. Current main still does not implement the PR's two central changes, and the PR branch still needs maintainer revision because its broader loopback identity omission can regress paired local sessions that rely on cached device-token auth.

## What This Changes

The PR changes GatewayClient null handling, callGateway close-error formatting, and gateway/TUI tests to preserve explicit device identity omission and include early connect errors in RPC failures.

## Best Possible Solution

Keep this PR open for revision or replacement with a narrow patch: preserve explicit deviceIdentity:null in the real GatewayClient constructor, capture GatewayClient.onConnectError inside callGateway and include it in final close transport errors, and retain current main's backend/shared-token loopback carveout so paired local device-token sessions keep working. Add focused regression tests for constructor null preservation, callGateway connect-error propagation, and paired loopback stored-token behavior.
