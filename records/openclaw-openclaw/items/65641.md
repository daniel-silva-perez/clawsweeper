## Summary

Keep this PR open. Current main has adjacent gateway-auth cleanup, but it still does not implement the PR’s two central diagnostics changes: `callGateway` does not thread `GatewayClient.onConnectError` into final close errors, and `GatewayClient` still normalizes explicit `deviceIdentity: null` to `undefined`. The PR itself also needs revision because its loopback/no-token identity omission can disable stored device-token auth for paired local sessions.

## Best Possible Solution

Keep this PR open for revision. The best fix is a narrow update that preserves explicit `deviceIdentity: null` in the real `GatewayClient` constructor, captures `GatewayClient.onConnectError` in `callGateway` and includes it in the final close error, and keeps current main’s narrower backend/shared-token loopback carveout so paired device-token local sessions continue to work.
