## Summary

Keep open. Current main still does not implement this PR’s intended gateway fallback: both gateway helper paths keep the top-level agent `name` tied to `agents.list[].name`, while `identity.name` remains nested. The related UI dropdown behavior has a separate fallback path, but this PR’s API/CLI/status consistency change is not already shipped.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is either to land a refreshed version that applies `configuredName ?? identity.name` in both `src/gateway/agent-list.ts` and `src/gateway/session-utils.ts` with the proposed regression tests, or to explicitly document/decide that top-level gateway `name` remains config-only and consumers should use nested `identity.name` for display labels.
