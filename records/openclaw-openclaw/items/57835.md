## Summary

Keep open. Current main does not implement this PR's intended gateway fallback: both gateway helper paths still set the top-level agent `name` only from `agents.list[].name`, while `identity.name` remains nested. The related UI path now falls back to nested `identity.name`, so the original dropdown issue may be separately addressed, but this PR's API/CLI/status consistency change is not already on main.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is either to land a refreshed version that applies `configuredName ?? identity.name` in both `src/gateway/agent-list.ts` and `src/gateway/session-utils.ts` with the proposed regression tests, or for a maintainer to explicitly decide that top-level gateway `name` should remain config-only because current UI consumers should use nested `identity.name`.
