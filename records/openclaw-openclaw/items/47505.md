## Summary

Keep open. Current main still lacks several central Control UI accessibility changes requested here, while the PR remains useful but broad, stale against the current UI, and needs maintainer review plus manual screen-reader validation before merge.

This PR updates Control UI CSS and Lit components with screen-reader labels and semantics, keyboard navigation affordances, raw config editor constraints, chat and command menu ARIA states, sessions sort announcements, and a Shift+Escape chat-focus shortcut.

## Best Possible Solution

Ship focused, rebased Control UI accessibility patches for the app shell, config form/raw editor, chat controls, command surfaces, and sessions table, with DOM-level accessibility assertions where practical and a manual Safari/VoiceOver pass. Keep the separate streaming live-region regression tracked in #65538 unless maintainers intentionally expand this PR's scope.
