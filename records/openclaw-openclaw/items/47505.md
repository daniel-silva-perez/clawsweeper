## Summary

Keep open. Current main has absorbed some related accessibility work, especially slash-command and command-palette listbox wiring, but it still lacks several central surfaces from this PR, including the skip link/main landmark, raw-editor VoiceOver mitigation, generated config-form labeling, sessions aria-sort, chat footer focus visibility, and Shift+Escape focus shortcut. The branch remains useful but needs maintainer review, rebase/split judgment, and manual VoiceOver validation before merge.

## What This Changes

The PR updates Control UI TypeScript and CSS with screen-reader labels and semantics, keyboard/focus affordances, reduced-motion handling, raw config editor constraints, chat and command menu ARIA states, sessions sort announcements, and a Shift+Escape chat-focus shortcut.

## Best Possible Solution

Ship focused, rebased Control UI accessibility patches for the app shell, config form/raw editor, chat controls/message navigation, command surfaces, and sessions table while preserving newer main improvements. Add DOM-level accessibility assertions where practical, keep the streaming live-region regression tracked separately in #65538 unless maintainers intentionally expand scope, and run a manual Safari/VoiceOver acceptance pass before landing.
