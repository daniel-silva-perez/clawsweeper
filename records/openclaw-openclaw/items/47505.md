## Summary

Keep open. Current main still lacks several central Control UI accessibility changes requested by this PR, especially skip navigation, semantic page headings, raw-config editor labeling/height constraints, generated config-form label wiring, keyboard/touch access for chat group footer buttons, and sessions table sort announcements. Some pieces have been independently superseded on main, so this should be rebased, split, or replaced rather than closed as obsolete.

## What This Changes

The PR updates Control UI TypeScript and CSS with screen-reader labels and semantics, keyboard/focus affordances, reduced-motion handling, raw config editor constraints, chat and command menu ARIA states, sessions sort announcements, and a Shift+Escape chat-focus shortcut.

## Best Possible Solution

Ship focused, rebased Control UI accessibility patches for the app shell, config form/raw editor, chat controls/message navigation, command surfaces, and sessions table while preserving newer main improvements. Add DOM-level accessibility assertions where practical, keep the separate streaming live-region regression tracked in #65538 unless maintainers intentionally expand scope, and do a manual Safari/VoiceOver acceptance pass before landing.

Not applicable as a normal bug reproduction for most of this feature PR. Static inspection gives high-confidence proof that current main still lacks several requested accessibility surfaces, while the VoiceOver raw-editor hang itself still needs manual Safari/VoiceOver verification.

## Solution Assessment

Unclear. The accessibility direction is appropriate and earlier concrete review bugs were followed up, but the branch is broad and stale against current main; the safer maintainable path is a rebase or split replacement that preserves newer UI accessibility work already landed.
