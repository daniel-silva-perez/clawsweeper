## Summary

Keep open. Current main still lacks the artifact RPCs, so the PR is relevant to the SDK-facing artifact work, but this revision is not merge-ready because the get/download params schema rejects valid requests, the collector misses current transcript media shapes, and protocol generated outputs are not refreshed.

## What This Changes

The branch adds read-scoped Gateway artifact RPC schemas, method discovery, handlers over session transcript media blocks, and tests for artifact list/get/download behavior and typed errors.

## Best Possible Solution

Revise this PR into a complete additive artifact contract: use validator schemas that accept real locator params, support the transcript media shapes OpenClaw already writes, restrict or explicitly reject unsupported URL/local-path sources, refresh generated protocol outputs and docs, wire the SDK artifact namespace if #74706 is meant to be closed, and then run the focused Gateway/SDK/protocol checks.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Define get/download params as one closed schema:** `src/gateway/protocol/schema/artifacts.ts:11-19`
  - body: `Type.Intersect` emits an `allOf` in typebox 1.1.34. Because both operands here set `additionalProperties: false`, Ajv rejects any real `artifacts.get` or `artifacts.download` request: the query schema rejects `artifactId`, and the id schema rejects `sessionKey`/`runId`/`taskId`. Build these params as one closed object or make the intersected pieces composition-safe.
  - confidence: 0.96
- **[P2] Handle transcript image_url media blocks:** `src/gateway/server-methods/artifacts.ts:139-144`
  - body: Current WebChat/media code can produce transcript display blocks like `{ type: "input_image", image_url: "data:..." }`, but `isArtifactBlock` ignores `image_url` and `resolveBlockDownload` never reads it. Those generated images will be missing from `artifacts.list`, which undercuts the artifact discovery contract this PR is adding.
  - confidence: 0.9
- **[P2] Regenerate protocol client models:** `src/gateway/protocol/schema/protocol-schemas.ts:345-351`
  - body: Adding these entries changes the protocol schema set, but the generated Swift `GatewayModels.swift` outputs are not included. The repo's `protocol:check` regenerates tracked protocol client models from `ProtocolSchemas`, so native clients and the protocol gate will be stale until the generated files are refreshed.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The diff adds a read-scoped artifact download surface and needs stricter source filtering before merge.

Concerns:

- **[medium] Validate artifact URL download sources:** `src/gateway/server-methods/artifacts.ts:114`
  - body: `resolveBlockDownload` classifies any non-data `url`, `openUrl`, or `source.url` string as URL-downloadable. Because this handler reads raw transcript blocks, a local path or unsupported scheme stored in a transcript could be surfaced to `operator.read` clients despite the PR's stated boundary that unsupported local-only sources return typed unsupported errors.
  - confidence: 0.82
