## Summary

Keep open. Current main still lacks the requested artifact RPC surface, and this active contributor PR is a relevant implementation for the still-open Gateway/App SDK request, but the patch needs protocol validation, transcript media coverage, generated-model, and download-source-policy fixes before merge.

## What This Changes

The PR adds Gateway `artifacts.list`, `artifacts.get`, and `artifacts.download` protocol schemas, read-scope discovery, transcript-backed handlers, and handler tests.

## Best Possible Solution

Revise this PR into a complete additive artifact contract: closed validator schemas that accept real locator params, support for current transcript media shapes, conservative URL/local-source filtering, regenerated protocol outputs, SDK wiring or an explicit follow-up for #74706, and focused Gateway/SDK/protocol validation.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a real `artifacts.get` or `artifacts.download` request containing both `artifactId` and a query key fails the intersected closed schemas, and current `input_image` transcript blocks are not collected by the PR handler.

## Solution Assessment

No. The direction is useful, but the current patch is not the best mergeable solution until it fixes the schema composition bug, includes existing transcript media shapes, refreshes generated clients, and tightens artifact download source policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Define get and download params as one closed schema:** `src/gateway/protocol/schema/artifacts.ts:13-20`
  - body: `Type.Intersect` emits `allOf` in TypeBox 1.1.34. Because both operands set `additionalProperties: false`, Ajv rejects valid requests containing both `artifactId` and `sessionKey`/`runId`/`taskId`, so `artifacts.get` and `artifacts.download` cannot be called successfully.
  - confidence: 0.96
- **[P2] Handle transcript image_url media blocks:** `src/gateway/server-methods/artifacts.ts:155-160`
  - body: Current WebChat media can write transcript blocks like `{ type: "input_image", image_url: "data:..." }`, but the PR detector ignores `image_url` and the resolver never reads it. Those generated images are omitted from `artifacts.list`, breaking artifact discovery for an existing media shape.
  - confidence: 0.9
- **[P2] Regenerate protocol client models:** `src/gateway/protocol/schema/protocol-schemas.ts:345-351`
  - body: Adding artifact schemas to `ProtocolSchemas` changes the generated protocol contract, but the tracked Swift `GatewayModels.swift` outputs are not updated. `pnpm protocol:check` regenerates those files, so native clients and the protocol gate remain stale.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The diff adds a read-scoped artifact download surface and needs stricter source filtering before merge.

Concerns:

- **[medium] Validate artifact URL download sources:** `src/gateway/server-methods/artifacts.ts:129`
