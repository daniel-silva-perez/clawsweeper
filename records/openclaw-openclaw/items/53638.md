## Summary

Keep open. Current main has the shared `channels.modelByChannel` config surface for channel/group-style conversations, but the requested per-DM behavior is still not wired because the resolver and reply call site do not pass direct-peer identifiers into model override matching.

## What This Changes

The issue requests persistent model override configuration for channel, group, and direct-message conversations.

## Best Possible Solution

Finish the generic `channels.modelByChannel` implementation for direct conversations by deriving normalized DM candidate keys from finalized context, persisted session origin, native provider identifiers, direct delivery targets, and the current session key; document the accepted DM key forms without adding provider-specific nested `model` fields.
