## Summary

Keep open. Current main has the shared `channels.modelByChannel` surface for channel/group pins, but the per-DM part of the request is still incomplete because ordinary direct conversations do not feed direct-peer identifiers into the shared resolver.

## What This Changes

Add persistent config-level model overrides for channel, group, and direct-message conversations, with per-target precedence over channel-wide and global defaults.

## Best Possible Solution

Finish `channels.modelByChannel` for direct conversations in the shared resolver: add normalized DM candidate keys from finalized context, persisted session origin, native ids, and session keys; preserve existing channel/group precedence; document the accepted DM key forms; and add Slack, WhatsApp, Discord, and resolver regression coverage without adding provider-specific `channels.<provider>.*.model` fields.
