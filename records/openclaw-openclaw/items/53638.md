## Summary

Keep open. Current main partially satisfies the request through the shared `channels.modelByChannel` config for channel/group pins, but ordinary direct messages still lack usable resolver candidates because the resolver and reply call site do not pass direct-peer identifiers.

## What This Changes

Add persistent config-based model overrides for specific channels, groups, and DMs so target-specific model selection can fall back to channel-wide and global defaults.

## Best Possible Solution

Keep this open and finish the generic `channels.modelByChannel` direct-conversation path. Extend the resolver inputs and reply call site to include normalized direct-peer candidates from finalized context, persisted session origin, native direct/channel IDs, and current session key; document supported DM key forms and add Slack/WhatsApp/Discord regression coverage while preserving current channel/group precedence.

Do not add provider-specific `channels.<provider>.model` or `channels.<provider>.dms.*.model` fields unless maintainers explicitly ask; keep the implementation generic and compatible with third-party channels.
