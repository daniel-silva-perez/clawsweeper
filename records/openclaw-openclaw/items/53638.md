## Summary

Keep open. Current main has the shared `channels.modelByChannel` config surface and covers channel/group-style conversations, including the Discord channel use case raised in discussion, but ordinary direct messages still have no stable DM candidate keys wired into model override resolution.

## What This Changes

The issue requests persistent per-channel, per-group, and per-DM model selection in config so conversations can use different models without relying on ephemeral runtime overrides.

## Best Possible Solution

Finish the generic `channels.modelByChannel` resolver for direct conversations by deriving normalized DM candidate keys from finalized context, persisted session origin/delivery data, native provider ids, and current direct session keys while preserving existing session override and channel/group precedence. Document accepted DM key forms instead of adding provider-specific nested `model` fields.

## Repair Work Prompt

Fix the remaining per-DM part of https://github.com/openclaw/openclaw/issues/53638. Current main already supports channel/group model overrides through `channels.modelByChannel`; do not add provider-specific `channels.<provider>.*.model` fields. Extend the generic channel model override path so direct conversations can match stable keys derived from finalized context/session data: current direct session key, persisted session origin/delivery target, `OriginatingTo`, `To`, `NativeChannelId`, `NativeDirectUserId`, sender/direct peer ids where appropriate, and provider-owned conversation binding helpers if needed. Preserve precedence: explicit session `/model` or stored session override wins, then per-target channel/DM key, then wildcard/channel-wide default, then global default. Preserve existing group/thread/channel behavior and parent fallback semantics. Document the accepted DM key forms in `docs/gateway/config-channels.md`, including Slack, WhatsApp, and Discord examples. Add focused tests for the resolver and reply path covering Slack DM user/native-channel ids, WhatsApp phone/direct session keys, Discord direct ids, wildcard provider default for DMs, and no regression for channel/group/thread matches. Add a user-facing changelog fix entry if behavior changes, crediting @gandalf-at-lerian and @artemp84. Do not touch unrelated provider model selection, provider auth, or plugin auto-enable behavior.
