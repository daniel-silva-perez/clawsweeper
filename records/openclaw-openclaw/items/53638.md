## Summary

Keep open. Current main partially satisfies the request with the shared `channels.modelByChannel` config for channel/group pins, but the direct-message part remains incomplete because the resolver and reply call site do not use the direct-peer identifiers that inbound contexts already carry.

## What This Changes

The issue requests persistent config-level model overrides for specific chat targets, including Slack channels, WhatsApp groups, WhatsApp DMs, optional channel-wide defaults, and similar channel/group/DM scopes.

## Best Possible Solution

Finish the generic `channels.modelByChannel` direct-conversation path without adding provider-specific model fields: pass normalized direct-peer candidates from finalized context, session origin, native IDs, and session keys into the shared resolver, preserve current channel/group precedence, document DM key forms, and add Slack/WhatsApp/Discord regression coverage.
