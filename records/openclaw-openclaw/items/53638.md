## Summary

Keep open. Current main implements the channel/group portion through the shared `channels.modelByChannel` config, including the Discord channel use case raised in discussion, but the requested per-DM behavior is still missing because the resolver and reply call site do not pass stable direct-peer identifiers.

## What This Changes

Add persistent config-based model overrides for specific channels, groups, and DMs so those conversations can use pinned models without relying on ephemeral `/model` session overrides.

## Best Possible Solution

Finish the generic `channels.modelByChannel` implementation for direct conversations by deriving normalized DM candidate keys from finalized context, persisted session origin, native provider IDs, direct delivery targets, and the current session key while preserving explicit session `/model` precedence and existing channel/group/thread behavior.
