## Summary

Keep this issue open. Current main has a useful shared `channels.modelByChannel` mechanism for per-channel/per-group model routing, and that likely covers the Discord-channel follow-up mentioned in the comment, but it does not yet fully satisfy the original per-DM request for WhatsApp/Slack direct conversations. The runtime model resolver is wired from group/channel metadata and parent session keys, while direct peer identifiers from the inbound context/session origin are not used as model override candidates.

## Best Possible Solution

Keep the issue open and finish the existing generic `channels.modelByChannel` implementation so it can match direct conversation identifiers as well as channel/group IDs. The narrow path is to extend `resolveChannelModelOverride` and its call site to include direct peer candidates from session key and session origin/context (`From`, `To`, `OriginatingTo`, `NativeDirectUserId`, Slack `user:<id>`, WhatsApp normalized phone/JID), document the supported key forms and wildcard channel default, and add focused tests for Slack and WhatsApp DMs plus existing group/channel precedence.

## Clownfish Work Prompt

Fix the remaining per-DM gap for the existing `channels.modelByChannel` model override feature. Current main already supports channel/group IDs through `src/channels/model-overrides.ts` and applies the override in `src/auto-reply/reply/get-reply.ts`; keep that generic config surface rather than adding provider-specific `channels.whatsapp.dms.*.model` or `channels.slack.channels.*.model` fields unless absolutely necessary. Extend the resolver/call site so direct conversations can match stable direct candidates from the active session and origin context, including Slack user-style keys (`user:<id>` and/or raw user id where appropriate), Slack native DM channel id when present, WhatsApp normalized phone/JID candidates, and direct session-key candidates. Preserve precedence: heartbeat override and explicit stored session/user `/model` override win over configured channel/direct defaults; configured per-target beats wildcard channel default; fallback remains `agents.defaults.model.primary`. Add focused unit tests for Slack DM, WhatsApp DM, existing Slack channel/group behavior, wildcard channel default, and user session override precedence. Update `docs/gateway/config-channels.md` with direct/DM key examples and the resolution order. Add a changelog entry only if repo policy considers this user-facing config behavior enough for the active release notes; otherwise leave changelog untouched. Do not change provider auth/model catalogs, session reset semantics, or plugin-specific config schemas.
