## Summary

Keep open. Current main covers the channel/group part through the shared `channels.modelByChannel` config, including the Discord channel use case raised in discussion, but the per-DM part is still not wired because the shared resolver receives only group/thread-style inputs and not ordinary direct-peer identifiers.

## What This Changes

Request persistent config-based model overrides for channel, group, and DM conversations, with target-specific config taking precedence over the global default model.

## Best Possible Solution

Finish `channels.modelByChannel` for direct conversations in the shared channel resolver: derive normalized DM candidate keys from finalized context, persisted origin/session data, native provider ids, and direct session keys while preserving existing session override and channel/group precedence. Document the accepted DM key forms and cover Slack, WhatsApp, Discord, and generic resolver behavior in tests without adding provider-specific `model` fields.

## Repair Work Prompt

Fix per-DM model override support for `channels.modelByChannel` without adding provider-specific `channels.<provider>.*.model` fields. Current main resolves channel/group/thread model pins, but ordinary DMs do not match because `resolveChannelModelOverride` only receives group/thread inputs. Add direct-message candidate support using normalized values from finalized context and persisted session data, including `OriginatingTo`, `To`, `From`, native direct/channel ids where available, direct session keys, and session origin ids. Preserve existing precedence: explicit session model override wins, then direct target match, then channel/group/thread parent match, then wildcard, then global default. Document accepted DM key forms in `docs/gateway/config-channels.md`. Add focused resolver coverage plus Slack, WhatsApp, Discord, and get-reply regression tests. Do not change provider-specific config schemas, model aliases, session reset semantics, or unrelated routing behavior. Changelog entry is appropriate because this fixes a user-facing config feature.
