## Summary

Keep open. Current main has the shared `channels.modelByChannel` config path for channel and group IDs, including the Discord channel use case raised in discussion, but the per-DM part of the requested behavior is still missing: the resolver and reply call site only receive group/thread-style identifiers, not stable direct-peer identifiers.

## What This Changes

The issue requests persistent config-based model overrides for specific Slack/WhatsApp channels, groups, and DMs, with target-specific overrides taking precedence over the global default.

## Best Possible Solution

Finish the generic `channels.modelByChannel` implementation for direct conversations by deriving normalized DM candidate keys from finalized context, persisted session origin, provider-native IDs, and direct session keys, while preserving explicit session `/model` precedence and existing channel/group behavior. Document the supported DM key forms and cover Slack, WhatsApp, Discord, and the shared resolver in focused tests.
