## What This Changes

The issue asks OpenClaw to avoid misleading startup warnings when intentional sibling skill references are seen from workspace skill docs.

## Best Possible Solution

Keep the issue open and make a focused skills-loader change only after confirming the exact sibling-reference shape. The desired end state is no misleading startup noise for intentional local sibling references, while real escaped skill roots remain rejected and visible through an appropriate warning, doctor check, or clearly documented debug path.
