## Summary

Keep this issue open. Current main contains adjacent approval-flow fixes and regression coverage, and the latest reporter repro no longer shows the original “approved but exec never resumes” failure. However, a maintainer explicitly kept this open pending a fresh current-main or next-release repro, and the latest concrete reporter test was on 2026.4.24 rather than current main. The remaining `allow-always` repro used strict inline eval, where current docs say inline eval should keep prompting.

## Best Possible Solution

Keep this issue open for one focused verification pass on current main or the next shipped release: test `allow-once`, non-inline `allow-always` durability, and strict inline-eval prompting separately on the affected surface. If only strict inline eval re-prompts, close this as implemented/expected and point to the exec docs; if WebChat policy still denies before approval, keep that scoped to #72858; if approval resolves but the waiting exec still does not resume, add a targeted regression test for that exact surface path.
