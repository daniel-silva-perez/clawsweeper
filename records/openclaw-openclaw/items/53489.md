## Summary

Keep this PR open. Current main has mitigations for the originally named default Slack/Discord/Telegram/Feishu paths, but the core change proposed by this PR is not implemented: parent-session forking still uses `createBranchedSession()`, and current callers can still route thread sessions through `ParentSessionKey`. This needs maintainer review/update rather than conservative cleanup closure.

## Best Possible Solution

Keep this PR open, but require an update before merge: decide the contract for opt-in parent transcript inheritance, then either replace `createBranchedSession()` with a safe clean/full-copy implementation or gate it per channel. Add a focused runtime regression test for `forkSessionFromParentRuntime()` transcript contents and audit callers that still set `ParentSessionKey`, especially Mattermost plus Slack/Discord opt-in paths.
