## Summary

Keep this PR open. Current main still trims non-streaming reply chunks in the gateway transcript/display paths and the `/btw` side-result recombination path, so indentation-sensitive content split across payload boundaries can still be flattened. The merged directive-tag fix in #55960 is present, but it is a narrower parser/display fix and does not replace this gateway recombination work.

## Best Possible Solution

Keep the PR open as a focused bug-fix candidate. Rebase it onto current main, introduce a shared gateway combiner for non-streaming final display/transcript text and `/btw` side-result text, preserve authored chunk indentation, trim only outer CRLF/CR/LF boundaries, skip whitespace-only chunks, and keep regression tests for multi-payload YAML/list indentation, `/btw`, single-newline preservation, and CRLF boundaries.
