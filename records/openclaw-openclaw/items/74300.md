## Summary

Keep this open. Current main still has adjacent version metadata and a future-config guard, but the machine-migration path does not require source/target version alignment and `openclaw doctor` has no migration-version preflight or target-version option.

## What This Changes

The issue requests migration guidance and doctor diagnostics that detect source/target OpenClaw version skew before copied state is accepted during a blue/green machine cutover.

## Best Possible Solution

Add a read-only migration-version preflight in doctor and update the migration guide so machine moves require same-version alignment before copying state; keep automatic schema rewrites and startup-time migrations out of the first fix.

## Reproduction Assessment

Yes for the actionable gap: current docs and CLI wiring can be inspected directly and show no version-alignment step or doctor option. The production auth/plugin failures are incident-specific, but the missing preflight is reproducible from source and docs.

## Solution Assessment

Yes, a conservative doctor warning/error plus docs update is the narrowest maintainable fix. It should reuse existing version parsing and available state metadata rather than adding broad automatic migrations.
