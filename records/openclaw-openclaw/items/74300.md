## Summary

Keep this open. Current main still has adjacent version metadata and future-config protections, but it does not implement a migration/cutover preflight that compares source state version against the target binary before copied state is accepted or documented as safe.

## What This Changes

Add migration guidance and a doctor-side preflight for OpenClaw source/target version skew during machine-to-machine state migration.

## Best Possible Solution

Add a conservative migration preflight that surfaces source-state/runtime version versus current target binary version before cutover, document same-version alignment as required before copying state, and reuse existing metadata where available without adding startup-time rewrites or broad automatic schema migrations.
