## Summary

Keep this issue open. Current main still does not expose or forward Telegram Bot API `entities` from the shared `message` tool to the Telegram plugin send path, so custom emoji via `custom_emoji` entities remains a real missing Telegram plugin capability.

## Best Possible Solution

Keep this open and implement it inside the bundled Telegram plugin. Add a Telegram-owned message-tool schema contribution for outbound entities, validate/narrow the Telegram `MessageEntity` payload, thread it through `extensions/telegram/src/action-runtime.ts` into `TelegramSendOpts`, and have `extensions/telegram/src/send.ts` send `entities` or `caption_entities` only on compatible text/caption payloads. Add regression tests for the tool schema, runtime forwarding, and Bot API params for a `custom_emoji` entity.
