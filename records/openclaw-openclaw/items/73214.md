## Summary

Keep this PR open for maintainer review rather than closing it as implemented. Current main already has a documented `models.pricing.enabled` opt-out for offline/restricted networks, but it still uses the 60s pricing fetch timeout; the PR’s shorter timeout/env-control work is therefore not fully implemented on main, and the submitted patch leaves timeout-warning tests stale.

## What This Changes

The PR changes the Gateway model-pricing cache to use a 15s default fetch timeout with a clamped `OPENCLAW_PRICING_FETCH_TIMEOUT_MS` override and adds `OPENCLAW_DISABLE_PRICING_REFRESH=1` to skip background pricing refresh.

## Best Possible Solution

Keep `models.pricing.enabled` as the primary offline/restricted-network control. If maintainers still want a shorter default timeout or env override, rebase the PR onto current main, fit it into the existing pricing config contract, update the timeout-warning tests and docs, and decide how invalid env values should be handled.

## Reproduction Assessment

Yes. A static reproduction is enough: the PR changes the default timeout to 15s while the unchanged pricing-cache test still expects `timeout 60s` warning strings for both remote sources.

## Solution Assessment

No. The current patch is not the best mergeable form because it duplicates an already shipped config opt-out and leaves tests stale; a rebased patch that updates tests/docs and aligns with the config surface would be safer.
