## Summary

Keep open. The PR is not mergeable as-is because it changes the default pricing timeout to 15s while leaving the existing timeout-warning test expecting 60s, and current main has already moved the offline opt-out surface to `models.pricing.enabled`; the remaining env-var timeout behavior needs maintainer decision and a rebase.

## What This Changes

The PR changes the gateway model-pricing cache to use a 15s default fetch timeout with a clamped `OPENCLAW_PRICING_FETCH_TIMEOUT_MS` override and adds `OPENCLAW_DISABLE_PRICING_REFRESH=1` to skip background pricing refresh.

## Best Possible Solution

Keep the shipped `models.pricing.enabled` opt-out as the primary offline/restricted-network control. If maintainers still want a shorter default timeout or env override, rebase this work onto current main, integrate it with the existing config seam, update the timeout-warning tests and docs, and decide whether invalid env values should warn or be rejected strictly.

## Reproduction Assessment

Yes. A static reproduction is enough: the PR head changes the timeout default to 15s, while the unchanged PR-head test still expects `timeout 60s` warning strings for both pricing sources.

## Solution Assessment

No. The current patch is not the best mergeable form because current main already implements a documented config opt-out and the PR leaves tests stale; the safer path is a rebased patch that updates tests/docs and fits the existing pricing config contract.
