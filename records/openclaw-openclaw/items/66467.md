## Summary

Keep this issue open. Current main still uses the same ACPX and ACP SDK boundary that rejects a `session/update` `usage_update` payload with `used: null` before OpenClaw's lenient ACP runtime projection can handle it. The existing review comment remains consistent with current main.

## Best Possible Solution

Fix the ACPX/OpenClaw ACP runtime boundary by upgrading or pinning acpx and @agentclientprotocol/sdk to compatible behavior, or by normalizing/dropping nullable `usage_update` fields before the SDK validates `session/update`. Add regression coverage for `sessionUpdate: "usage_update"` with `used: null` and numeric `size` through the ACPX runtime path used by Discord-bound sessions.
