## Summary

Keep open. Current main still does not enforce the configured Feishu webhookPath before processing webhook requests, and the provided GitHub context shows an open, unmerged implementation candidate with closing syntax for this issue. The issue should remain open until that PR or an equivalent Feishu-owned fix lands.

## Best Possible Solution

Keep this issue open. The best path is maintainer review and landing of PR #54904, or an equivalent Feishu-owned patch in extensions/feishu/src/monitor.transport.ts that normalizes the configured webhookPath, parses the incoming request pathname, rejects mismatched paths with 404 before body parsing/signature validation/dispatch, preserves query-string acceptance on the configured path, and adds focused regression coverage for wrong-path rejection.
