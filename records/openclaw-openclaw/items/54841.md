## Summary

Keep open. Current main still uses the configured Feishu webhookPath for logging, status, and rate-limit keys but does not compare the incoming request URL/pathname before body parsing, signature validation, challenge handling, or event dispatch. PR #54904 remains open and unmerged with closing syntax for this issue, so the issue should stay open until that PR or an equivalent Feishu-owned fix lands.

## What This Changes

Request that Feishu webhook mode reject requests whose URL path does not match channels.feishu.webhookPath before body parsing, signature validation, or event dispatch.

## Best Possible Solution

Review and land PR #54904, or an equivalent Feishu plugin patch, that normalizes the configured webhookPath, parses the incoming request pathname, rejects mismatches with 404 before body parsing/signature validation/dispatch, preserves query-string acceptance on the configured path, and adds focused regression coverage.
