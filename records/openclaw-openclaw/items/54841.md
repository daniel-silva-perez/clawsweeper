## Summary

Keep open. Current main still does not enforce the configured Feishu webhookPath before body parsing, signature validation, challenge handling, or dispatch, and PR #54904 remains open and unmerged with the targeted fix for this issue.

## What This Changes

The issue requests that Feishu webhook mode reject requests sent to any path other than the configured webhookPath before parsing, signature validation, or dispatch.

## Best Possible Solution

Review and land PR #54904, or an equivalent Feishu-owned patch, that normalizes the configured webhookPath, compares the incoming request pathname before body parsing/signature validation/dispatch, returns 404 for mismatches, preserves query-string acceptance on the configured path, and adds focused regression coverage.
