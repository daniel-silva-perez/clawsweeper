## Summary

Keep open. Current main still resolves `commands.ownerAllowFrom` by walking the raw array during command authorization and then uses array membership checks, and the provided GitHub context shows open, unmerged PR #73440 as the active implementation candidate for this issue.

## Best Possible Solution

Keep this issue open and review #73440, or an equivalent fix, against the current authorization path. The best code direction is to compile or memoize `ownerAllowFrom` per stable config array/provider/account/plugin key, preserve channel-scoped normalization behavior, and add focused regression coverage for cache hits, provider isolation, and config-reload behavior. After the PR merges, this issue can close through the normal GitHub closing flow.
