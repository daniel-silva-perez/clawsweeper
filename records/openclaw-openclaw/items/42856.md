## Summary

Keep this issue open. Current main still explains the reported Telegram-facing warning: the agent mutation classifier treats `gateway` action `config.schema.lookup` as mutating, the tool-error payload path still emits warning messages for mutating failures, and the gateway still returns `config schema path not found` for missing lookup paths. The supplied GitHub context also shows open implementation PRs #42898 and #42962 with closing syntax, so this issue should remain open until one is merged or replaced.

## Best Possible Solution

Keep this issue open and review or replace the linked implementation PRs. The likely fix belongs in `src/agents/tool-mutation.ts`: classify compound gateway/cron/canvas actions by their leaf read-only verb, or otherwise explicitly treat `config.schema.lookup` as non-mutating, with regression coverage for the mutation classifier and warning payload behavior for `config schema path not found`.
