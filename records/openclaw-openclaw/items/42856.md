## Summary

Keep open. Current main still routes missing `config.schema.lookup` lookups through the user-visible warning path: the gateway marks the method read-only and returns the reported structured error, but the agent mutation classifier compares the full dotted action against leaf read-only verbs and treats it as mutating. Open PRs #42898 and #42962 still reference this work, so closing before one canonical fix lands would be premature.

## What This Changes

Request that missing-path errors from `config.schema.lookup` return silently to the agent without producing user-facing channel notifications.

## Best Possible Solution

Land one canonical fix that makes agent warning policy method-aware for compound gateway read methods, preferably by classifying `gateway` actions such as `config.schema.lookup` from their read-only method semantics before payload rendering. Add regression coverage proving lookup misses stay silent while genuine mutating gateway failures still warn.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: call the `gateway` tool with `action: "config.schema.lookup"` and a missing path, then the handler returns `config schema path not found` while the current mutation classifier marks the full dotted action as mutating and the payload layer emits a warning if unacknowledged.

## Solution Assessment

No fix is on main yet. The classifier-level direction in #42898 is the narrowest maintainable solution because it aligns warning behavior with gateway read/write semantics; a payload-only suppression like #42962 is a fallback and depends on `mutatingAction` already being correct.
