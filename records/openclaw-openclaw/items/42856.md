## Summary

Keep open. Current main still classifies the read-only compound gateway action `config.schema.lookup` as a mutating tool call, so a missing schema path can still flow into the user-facing warning renderer; the issue is also already paired with open implementation PRs, so it should not be closed before one lands.

## What This Changes

The issue requests suppressing user-facing channel notifications when read-only `config.schema.lookup` calls miss a non-existent config schema path.

## Best Possible Solution

Land one canonical fix, preferably by making gateway action classification method-aware for compound read-only actions such as `config.schema.lookup`, with regression coverage proving lookup misses do not render channel warnings while real mutating gateway failures still do.

## Reproduction Assessment

Yes. The static reproduction path is high-confidence: a missing `config.schema.lookup` path returns `config schema path not found`, the full compound action is classified as mutating, and mutating failures are rendered as warning payloads; only the live Telegram rendering was not rerun.

## Solution Assessment

No fix is on main yet. The narrowest maintainable solution appears to be classifier-level handling for compound read-only gateway methods, with payload-specific suppression only as a fallback if maintainers want to limit scope to this exact error string.
