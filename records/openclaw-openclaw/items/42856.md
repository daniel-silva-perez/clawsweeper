## Summary

Keep open. Current main still classifies the compound read-only gateway action as mutating, so a missing `config.schema.lookup` path can still flow into the user-visible warning payload path; open PRs #42898 and #42962 already target the behavior, so this should stay open until one canonical fix lands.

## What This Changes

Suppress user-facing channel notifications for missing `config.schema.lookup` paths while preserving the structured gateway error response for the agent.

## Best Possible Solution

Land one canonical fix that makes agent warning policy respect compound gateway method semantics, preferably by classifying `gateway` actions from their leaf read/write behavior before payload rendering. Add regression coverage proving missing `config.schema.lookup` paths stay silent while real mutating gateway failures still warn.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: call the `gateway` tool with `action: "config.schema.lookup"` and a missing path, which returns the structured `config schema path not found` error while current main still marks the tool call as mutating and eligible for a channel warning.

## Solution Assessment

No fix is on main yet. The classifier-level direction in #42898 is the narrowest maintainable solution; a payload-only suppression should only be used if it also ensures read-only compound gateway actions are classified correctly.
