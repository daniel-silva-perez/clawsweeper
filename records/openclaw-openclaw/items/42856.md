## Summary

Keep open. Current main still classifies the compound gateway action `config.schema.lookup` as mutating, which leaves lookup misses on the warning-rendering path, and open PRs #42898 and #42962 already target this exact behavior with closing syntax.

## What This Changes

The issue requests that nonexistent `config.schema.lookup` paths return the structured lookup error without producing user-facing channel notifications.

## Best Possible Solution

Treat the gateway action classifier as method-aware so compound read-only methods, especially `config.schema.lookup`, are non-mutating, then add regression coverage proving a missing lookup path does not render a channel warning while real mutating gateway failures still do.
