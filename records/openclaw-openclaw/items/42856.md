Keep open. Current main still appears to classify the reported read-only compound gateway lookup as mutating, which feeds the user-facing failed-tool warning path; two open PRs already explicitly target this issue, so it should remain open until one lands or is replaced.

## What This Changes

The issue requests suppressing user-facing channel notifications when read-only `config.schema.lookup` misses return the structured `config schema path not found` gateway error.

## Best Possible Solution

Resolve this by landing or replacing the linked implementation candidate with a classifier fix that treats compound read-only gateway leaf verbs such as `lookup` as non-mutating, plus regression coverage for `config.schema.lookup` path misses and the warning payload path.
