## Summary

Keep open. Current main still leaves `config.schema.lookup` lookup misses on the user-visible warning path: the agent mutation classifier compares the full compound gateway action against leaf read-only verbs, while the gateway declares the method read-scoped and returns the reported `config schema path not found` error. Open PRs #42898 and #42962 still target this behavior, so closing before one canonical fix lands would be premature.

## What This Changes

Suppress user-facing channel warnings when read-only `config.schema.lookup` misses a non-existent path and returns a structured path-not-found error.

## Best Possible Solution

Land one canonical fix, preferably by making tool mutation classification method-aware for compound gateway read methods such as `config.schema.lookup`, with regression coverage proving lookup misses stay silent while genuine mutating gateway failures still render warnings.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: call the `gateway` tool with `action: "config.schema.lookup"` and a missing path, then the current classifier records it as mutating, the handler returns `config schema path not found`, and payload rendering emits the warning for unacknowledged mutating failures.

## Solution Assessment

No fix is on main yet. The classifier-level approach in #42898 is the narrowest maintainable direction because it aligns agent warning policy with gateway read/write scopes; a payload-only suppression is a narrower fallback and must not depend on a misclassified `mutatingAction`.
