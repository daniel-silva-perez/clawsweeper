## Summary

Keep open. Current main still leaves `config.schema.lookup` on the user-visible warning path because the agents mutation classifier checks the full compound action against leaf read-only verbs, while the gateway method itself is read-scoped and returns the reported path-not-found error. The issue is also paired with open implementation PRs, so it should stay open until maintainers land or reject one canonical fix.

## What This Changes

The issue requests that read-only `config.schema.lookup` path-miss errors return silently to the agent instead of producing user-facing channel warnings.

## Best Possible Solution

Land one canonical fix, preferably by making gateway action classification method-aware for compound read-only methods such as `config.schema.lookup`, with regression coverage proving lookup misses do not render channel warnings while real mutating gateway failures still do.

## Reproduction Assessment

Yes. A static reproduction path is high-confidence: a missing `config.schema.lookup` path returns the reported structured error, current main classifies the compound gateway action as mutating, and mutating tool errors render warning payloads; only the live Telegram delivery was not rerun.

## Solution Assessment

No fix is on main yet. The classifier-level approach is the narrowest maintainable solution because `config.schema.lookup` is already declared read-scoped; payload-specific suppression is a narrower fallback if maintainers want to limit the behavior change to this exact error.
