## Summary

Keep open. Current `main` still has the reported warning path: `config.schema.lookup` is a read-scoped gateway method, but the agent mutation classifier still checks the full compound action string against a leaf-action allowlist and therefore marks lookup misses as mutating failures.

## What This Changes

The issue requests that nonexistent `config.schema.lookup` paths return the structured lookup error without producing user-facing channel notifications.

## Best Possible Solution

Land one canonical fix that makes gateway action classification method-aware for compound read-only methods such as `config.schema.lookup`, then add regression coverage proving lookup path misses do not render channel warnings while real mutating gateway failures still do.
