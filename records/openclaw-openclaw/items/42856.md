## Summary

Keep this issue open. Current main still classifies `gateway` action `config.schema.lookup` as mutating, so a missing-path gateway error can still flow into the user-facing `⚠️ ... failed` warning path. Open PRs #42898 and #42962 are implementation candidates with closing syntax, so the issue should stay open until one is reviewed/merged or replaced.

## Best Possible Solution

Keep the issue open and review or replace the linked PRs. The preferred fix belongs in `src/agents/tool-mutation.ts`: classify `gateway`/`cron`/`canvas` compound actions by their leaf read-only verb and include `lookup`, with regression coverage in `src/agents/tool-mutation.test.ts` plus payload coverage proving `config.schema.lookup` path-miss errors do not produce user-facing warnings.
