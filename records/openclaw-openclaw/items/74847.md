## Summary

Keep open. Current main still lacks the SDK-facing task ledger RPC surface, and this PR is a relevant implementation for the open task-ledger request, but it should not merge as-is because the new RPCs are not advertised through Gateway method discovery and the protocol schema additions are not reflected in generated native protocol models.

## What This Changes

The PR adds `tasks.list`, `tasks.get`, and `tasks.cancel` Gateway handlers, protocol schemas and validators, operator scopes, SDK task methods and types, docs, and regression tests for the task ledger surface.

## Best Possible Solution

Land a polished task ledger RPC contract after adding the methods to Gateway discovery, refreshing generated native protocol models, and confirming the operator read/write metadata boundary; then the linked task-ledger issue can close through the merged implementation.

## Reproduction Assessment

Yes for the reported gap: current main has `oc.tasks.list/get/cancel` hard-coded to unsupported errors and no registered task RPC handlers. The merge-blocking PR issues are also reproducible by inspecting the PR head's missing `server-methods-list.ts` entries and generated Swift model symbols.

## Solution Assessment

No, not yet. Exposing the existing durable task registry through additive Gateway RPCs is the right direction, but this implementation must keep feature discovery and generated protocol clients in sync before it is the maintainable fix.
