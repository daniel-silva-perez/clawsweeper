## Summary

Keep open. Current main still has unsupported `oc.tasks.list/get/cancel` behavior and no `tasks.*` Gateway method registration, while this PR is a relevant active implementation for the member-authored task-ledger API request. The branch still needs normal maintainer review because it touches the Gateway protocol, SDK contracts, generated native models, and overlaps another open implementation.

## What This Changes

The PR adds Gateway `tasks.list`, `tasks.get`, and `tasks.cancel` RPC handlers, schemas, scopes, SDK task methods and types, SDK docs, generated Swift models, and regression coverage over the existing task registry.

## Best Possible Solution

Land one polished additive Gateway task-ledger contract after maintainer review confirms the metadata boundary, operator scopes, status mapping, cancel result semantics, generated native model shape, docs, and regression coverage; then let #74707 close through the merged implementation and retire the duplicate branch.

## Reproduction Assessment

Yes. Current main can be reproduced by static inspection and existing tests: the SDK task helpers throw unsupported errors, docs list them as unsupported, and Gateway discovery/scopes do not include `tasks.*`.

## Solution Assessment

Yes overall, with a small native-client caveat. Exposing the existing durable task registry through additive Gateway RPCs and SDK methods is the narrow maintainable direction, but the Swift cancel result should preserve `TaskSummary?` before native clients rely on typed responses.

## Review Findings

Overall correctness: patch is correct
