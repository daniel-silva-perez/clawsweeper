## Summary

Keep open. Current main still has the SDK task helpers hard-coded to unsupported errors and has no Gateway task RPC discovery/handler/scopes, so this PR is not obsolete; it is an active implementation for the still-open SDK task-ledger request and needs normal maintainer API/security/native-client review rather than cleanup closure.

## What This Changes

The PR adds SDK-facing `tasks.list`, `tasks.get`, and `tasks.cancel` Gateway task-ledger RPCs with protocol schemas, operator scopes, SDK types/methods, generated Swift models, docs, and regression tests.

## Best Possible Solution

Land a polished additive task-ledger RPC contract after maintainer review confirms the operator metadata boundary, native generated-model shape, and regression coverage; then let the linked task-ledger issue close through the merged implementation.

## Reproduction Assessment

Yes. Current main can be reproduced by static inspection and existing tests: `oc.tasks.list/get/cancel` call `unsupportedGatewayApi`, the SDK test asserts those unsupported errors, docs list the helpers as unsupported, and Gateway discovery/handler/scopes do not include `tasks.*`.

## Solution Assessment

Yes overall, with a small native-client caveat. Exposing the existing durable task registry through additive Gateway RPCs and SDK methods is the narrow maintainable direction; the Swift `TasksCancelResult.task` output should preserve `TaskSummary?` before merge.
