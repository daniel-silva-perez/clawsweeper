## Summary

Keep open. Current main has adjacent managed download capture for ordinary Playwright download events, but the direct `/navigate` Content-Disposition attachment path still returns only URL metadata and rethrows non-retryable `page.goto()` errors, so the reported CDP download/navigation failure is not implemented or superseded.

## What This Changes

The issue asks the browser plugin to preserve downloads and return successful `download.triggered` metadata when `/navigate` over a Playwright CDP connection hits a URL that starts a file download.

## Best Possible Solution

Implement the missing browser-plugin path for download-triggered navigation: treat Playwright's download-starting navigation abort as a successful download trigger, persist the file through the existing guarded download handling where possible, return structured download metadata from `/navigate`, and keep the existing navigation policy checks intact.
