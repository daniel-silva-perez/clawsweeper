## Summary

Keep open. Current main partially improves ordinary unmanaged download capture, but the direct browser `/navigate` path still treats Playwright's download-start navigation abort as a failure, returns only URL metadata, and has no CDP-level or Content-Disposition navigation regression coverage.

## What This Changes

The issue requests CDP browser navigation-download handling so attachment URLs save the file and return structured success metadata instead of surfacing a `Download is starting` failure.

## Best Possible Solution

Implement browser-plugin navigation download handling around guarded navigation: observe or arm a download before `page.goto`, save the captured file through the guarded downloads directory, return structured download metadata from `/navigate`, and add a focused Content-Disposition attachment regression for the CDP path while preserving existing `/download` waiter behavior.
