## Summary

Keep open. Current main has adjacent managed/click download capture, but direct browser `/navigate` still has a URL-only result contract, rethrows non-policy `page.goto()` errors, and has no CDP download protocol hook or Content-Disposition regression coverage. The linked fix PR #48132 is closed unmerged, while #64558 covered adjacent click-triggered/managed-profile download behavior rather than this direct navigation path.

## What This Changes

The issue asks for CDP-mode browser downloads triggered by navigation to be saved and reported as a successful download instead of failing with Playwright's download-starting navigation abort.

## Best Possible Solution

Implement the missing browser-plugin navigation download path: arm or observe a download around guarded navigation, treat the download-starting abort as success only when a real download is captured, persist the file through the existing guarded downloads directory, return structured download metadata from `/navigate`, and preserve the current navigation policy checks.
