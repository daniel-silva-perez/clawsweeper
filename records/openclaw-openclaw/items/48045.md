## Summary

Current main still lacks the direct `/navigate` attachment-download behavior requested here. The browser plugin now has adjacent managed download capture for ordinary Playwright download events, but `navigateViaPlaywright` still returns only `{ url }` and rethrows non-retryable `page.goto()` errors, so the reported `Download is starting` path cannot return `download.triggered` metadata yet.

## What This Changes

The issue requests CDP-backed browser navigation to attachment/download URLs to return structured success metadata and persist the downloaded file instead of surfacing Playwright's `Download is starting` navigation abort or losing the file.

## Best Possible Solution

Implement the missing browser-plugin path for download-triggered navigation: classify Playwright's download navigation abort as a successful download trigger, persist the file through the existing guarded downloads directory behavior, return structured download metadata from `/navigate`, and keep the existing pre-navigation, redirect-chain, and final URL SSRF protections intact.
