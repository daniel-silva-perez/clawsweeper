## Summary

Keep open. Current main includes adjacent click-triggered download capture, but browser `/navigate` is still a URL-only Playwright path that rethrows non-policy `page.goto()` errors and has no CDP download protocol hook or Content-Disposition navigation regression. The direct fix PR #48132 is closed unmerged, while #64558 shipped adjacent managed/click download behavior rather than this direct navigation path.

## What This Changes

The issue requests browser `/navigate` download handling for CDP-connected Chrome so attachment responses save to the downloads directory and return structured success metadata instead of surfacing `Download is starting`.

## Best Possible Solution

Add a browser-plugin navigation-download path that arms or observes a download around guarded navigation, treats download-starting aborts as success only when a real download is captured, persists through the existing guarded downloads directory, returns structured download metadata from `/navigate`, and adds a Content-Disposition attachment regression for raw CDP navigation.
