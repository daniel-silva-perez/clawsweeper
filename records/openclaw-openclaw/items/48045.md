## Summary

Keep open. Current main has adjacent managed download capture, but the reported direct browser `/navigate` to a `Content-Disposition: attachment` URL still falls through the Playwright navigation error path and cannot return `download.triggered` metadata.

## What This Changes

The issue asks for CDP-mode browser downloads and download-triggering navigations to be handled as successful saved downloads instead of discarded data or a Playwright "Download is starting" failure.

## Best Possible Solution

Keep this issue open and implement a browser-plugin fix for download-triggered navigation: detect Playwright's download navigation abort, return a structured success marker from `/navigate`, keep CDP/managed download persistence in the guarded downloads directory, preserve pre-navigation, redirect, and final-state SSRF checks, and add regression coverage for allowed attachment downloads plus blocked redirect-to-attachment cases.
