## Summary

Keep open. Current main still fixes adjacent browser download paths, but not the reported direct `/navigate` to `Content-Disposition: attachment` case: `/navigate` still calls `page.goto()` through `gotoPageWithNavigationGuard`, rethrows non-retryable Playwright navigation aborts, and returns only `{ url }` without `download.triggered` metadata.

## Best Possible Solution

Keep this issue open and implement a browser-plugin fix for download-triggered navigation: detect Playwright's download navigation abort, return a structured success marker from `/navigate`, preserve all existing navigation SSRF checks across redirects and final state, and add regression coverage for allowed attachment downloads and blocked redirect-to-attachment cases.
