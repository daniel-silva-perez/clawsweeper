## Summary

Keep open. Current main still treats browser `/navigate` as a URL-only Playwright navigation and bubbles non-policy `page.goto()` errors, while the shipped adjacent download work only normalizes Playwright download events for click/waiter paths. The linked fix PR #48132 is closed unmerged, and #64558 addressed a nearby managed/click download seam rather than direct Content-Disposition attachment navigation.

## What This Changes

The issue requests CDP browser navigation downloads to be saved and reported as successful structured download metadata instead of failing with `Download is starting` or silently discarding the file.

## Best Possible Solution

Implement the missing browser-plugin navigation download path: observe or arm downloads around guarded navigation, treat the download-starting abort as success only when a real download is captured, persist through the existing guarded downloads directory, return structured download metadata from `/navigate`, and keep current SSRF/navigation-policy and output-root protections intact.
