## Summary

Keep open. Current main partially improves browser download persistence, but it does not implement the reported navigate-to-download contract: `/navigate` still flows through Playwright `page.goto()`, returns only a URL-shaped result on success, and has no handling for Playwright's `Download is starting` navigation abort. The related implementation PR #48132 was closed unmerged.

## Best Possible Solution

Keep this issue open and implement the remaining browser-plugin behavior: catch the Playwright download-start navigation abort in the navigate path, coordinate it with the download event or CDP download events, persist the file under the guarded downloads path, return a structured success such as `download.triggered` with saved path metadata, and add a regression test for a Content-Disposition attachment URL over a CDP-connected browser. #48132 can be used as prior context, but it was closed unmerged and should be reviewed or replaced.
