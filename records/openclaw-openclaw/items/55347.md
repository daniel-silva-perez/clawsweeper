## Summary

This should remain open. Current main has related reliability pieces, including channel health restarts, last-known-good config recovery, restart sentinels, supervisor restart limits, and doctor repair commands, but it still does not implement the requested native gateway self-heal workflow with configurable gateway process monitoring, capped process restart attempts, restart-time doctor/autofix summaries, dead-man alerts, and channel recovery briefs. The related open PRs are useful narrower implementation threads, not shipped completion of this umbrella request.

## What This Changes

The issue requests a native gateway self-healing system with configurable process health monitoring, restart validation/autofix, capped recovery attempts, dead-man alerts, and channel recovery briefs.

## Best Possible Solution

Keep this issue open as the umbrella for native gateway recovery until maintainers land or split a complete design covering process supervision, bounded restart behavior, restart-time diagnostics/repair policy, and user-visible recovery notification. Review PR #64684 as a diagnostic slice and PR #46502 as the broader rescue-watchdog implementation candidate, without treating either open PR as completion before merge.
