## Summary

Keep open: current main still has the reported Telegram group-history media gap. The no-mention group path can receive downloaded media, but pending history stores only text fields, replay maps history back to text-only entries, and the related fix PRs are closed unmerged.

## What This Changes

The issue asks Telegram group messages with media sent without mentioning the bot to preserve usable media references in pending group history so a later mention can inspect that media.

## Best Possible Solution

Add a backward-compatible media-bearing pending-history contract, retain bounded local media references for skipped Telegram group messages, merge those references into later MediaPaths/MediaTypes during mention replay, and cover the image-without-mention followed by @mention flow with focused Telegram and shared-history tests.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: in a requireMention Telegram group, a media message without a mention records pending history as text only, and a later mention builds MediaPaths from current/replied-to media rather than history entries.

## Solution Assessment

Yes for the requested fix direction. The narrow maintainable solution is an additive mediaPaths/mediaTypes history extension plus bounded retention and replay merging; current main is not a solution because it preserves only placeholder text.
