## Summary

Keep open: current main still has the reported Telegram group-history media gap. The no-mention group path receives downloaded media, but pending history stores only text fields, later replay builds MediaPaths from current/replied-to media only, and the related fix PRs are closed unmerged.

## What This Changes

The issue asks Telegram group history to retain media paths/types for skipped no-mention media messages so a later @mention can access those files.

## Best Possible Solution

Add a backward-compatible media-bearing pending-history contract, retain bounded local media references for skipped Telegram group messages, merge those references into later MediaPaths/MediaTypes during mention replay, and cover the image-without-mention followed by @mention flow with focused Telegram and shared-history tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: in a requireMention Telegram group, a media message without a mention records pending history as text only, and a later mention builds MediaPaths from current/replied-to media rather than history entries.

## Solution Assessment

Yes for the requested repair direction. The narrow maintainable fix is an additive mediaPaths/mediaTypes history extension plus bounded retention/replay merging; current main is not a solution because it preserves only placeholder text.
