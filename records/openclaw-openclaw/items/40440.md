## Summary

Current main still matches the reported Telegram failure. Skipped mention-gated group messages receive downloaded media in `allMedia`, but pending history stores only sender/body/timestamp/messageId, and later Telegram context construction only forwards current-turn plus reply media. The related fix attempts #40470 and #40490 are closed unmerged, and I found no shipped source, docs, tests, changelog, or release evidence that history media replay has been implemented.

## Best Possible Solution

Keep this issue open and implement the focused Telegram/shared-history fix: extend reply-history and `MsgContext.InboundHistory` with optional media references, persist or refresh media for skipped mention-gated Telegram group messages under a clear lifetime policy, merge replayed history media into later Telegram `MediaPaths`/`MediaTypes`, update prompt metadata if useful, and add a regression test for image-without-mention followed by an @mention asking about that image.

## Risks / Open Questions

- No live Telegram group reproduction was run during this read-only review; the keep-open decision is based on the current source path directly matching the reported failure mode.

- Extending history media support touches plugin-SDK-visible `HistoryEntry` and `MsgContext.InboundHistory`, so it should be additive, documented, and covered by focused contract plus Telegram regression tests.
