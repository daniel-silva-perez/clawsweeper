## Summary

Current main still matches the reported bug: Telegram no-mention group messages are recorded as text-only pending history, and later mention-triggered context only attaches current-turn and reply media. The two related fix PRs are closed unmerged, so there is no implemented or canonical replacement yet.

## What This Changes

Telegram group history should preserve media paths/types for media messages skipped by mention gating so a later @mention can access the earlier images instead of only seeing placeholder text.

## Best Possible Solution

Add an additive media-bearing pending-history contract, have the Telegram plugin attach bounded retained media references for mention-gated skipped group messages, merge those references into later `MediaPaths`/`MediaTypes`, and cover the image-without-mention followed by @mention replay flow with focused Telegram and shared-history tests.
