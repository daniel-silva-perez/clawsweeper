## Summary

Keep open: current main still has the reported Telegram group-history media gap. Telegram media is downloaded into a local path before mention gating, but no-mention group messages are recorded as text-only pending history, and later mention replay only builds MediaPaths from the current message plus reply media. The two related fix PRs, #40470 and #40490, are closed unmerged, and the latest release tag still shows the same text-only history shape.

## What This Changes

The issue asks for Telegram no-mention group media messages to preserve media references in pending chat history so a later @mention can give the agent access to those files.

## Best Possible Solution

Add a backward-compatible media-bearing pending-history contract, have the Telegram plugin retain bounded media references for skipped mention-gated group messages, merge those references into later MediaPaths/MediaTypes during mention replay, and cover the image-without-mention followed by @mention flow with focused Telegram and shared-history tests.

## Reproduction Assessment

Yes. The high-confidence reproduction path is the source path itself: send media in a requireMention Telegram group without mentioning the bot, observe the skipped message recorded as text-only history, then mention the bot and see replay context built without any history media paths.

## Solution Assessment

Yes for a future fix: the narrow maintainable direction is an additive mediaPaths/mediaTypes history extension plus bounded media retention and replay merging. Current main is not a solution because it only preserves the placeholder text.

## Repair Work Prompt

Fix the Telegram group-history media gap tracked by https://github.com/openclaw/openclaw/issues/40440. Observable bug: in a Telegram group with requireMention enabled, media sent without @mentioning the bot is downloaded by resolveMedia, but the skipped pending-history entry stores only the placeholder/body text. When the bot is later mentioned, replay context includes the placeholder but no MediaPaths/MediaTypes for the prior image.

Expected boundary:
- Add backward-compatible optional media references to the shared pending-history contract, likely mediaPaths/mediaTypes on HistoryEntry and the corresponding InboundHistory typing/metadata surface.
- In the active Telegram plugin under extensions/telegram/src, attach media refs when recordPendingHistoryEntryIfEnabled records a skipped no-mention group message.
- Do not rely on the normal 2-minute inbound media TTL for this flow. Retain or copy media through a bounded, path-safe mechanism that preserves current media-store safety and does not create unbounded group media retention.
- On later @mention replay, merge history media into the context payload so MediaPath/MediaPaths/MediaType/MediaTypes include the relevant retained history files while preserving current-message and reply-media behavior.
- Add focused regression tests for image/media without mention followed by @mention, plus shared history/inbound metadata coverage if the public shape changes.
- Add a user-facing changelog fix entry if behavior changes ship; credit the reporter and useful prior PR context, for example Thanks @xsmyhgz, @mvanhorn, and @maweibin if their proposals inform the implementation.
- Do not move Telegram-owned behavior into generic core except for the additive shared history contract needed by plugin consumers; do not change Telegram group policy or mention-gating semantics.

Related refs: #40470 and #40490 are useful closed-unmerged source work but should not be treated as landed.
