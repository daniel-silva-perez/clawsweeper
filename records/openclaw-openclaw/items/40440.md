## Summary

Keep open: current main still has the reported Telegram group-history media gap. Skipped requireMention group messages record pending history as text only, while later mention replay builds MediaPaths from only the current message and reply media; the two related fix PRs are closed unmerged.

## What This Changes

Preserve downloaded Telegram media references for requireMention group messages stored as pending history so a later @mention can include that media in agent context.

## Best Possible Solution

Add a backward-compatible media-bearing pending-history contract, retain bounded local media references for skipped Telegram group messages, merge those references into later MediaPaths/MediaTypes during mention replay, and cover the image-without-mention followed by @mention flow with focused Telegram and shared-history tests.

## Reproduction Assessment

Yes. The high-confidence reproduction path is the source path itself: in a requireMention Telegram group, a media message without a mention is stored as pending history, then a later mention builds replay context without any history media paths.

## Solution Assessment

Yes for a future fix direction. The narrow maintainable solution is an additive mediaPaths/mediaTypes history extension plus bounded retention and replay merging; current main is not a solution because it preserves only the placeholder text.
