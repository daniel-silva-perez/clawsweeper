## Summary

Current main still has the reported Telegram group-history media gap: skipped no-mention group messages can retain only text history, while later mention replay builds media context from the current message and reply media only. The two related fix PRs in the provided GitHub context are closed unmerged, so there is no merged implementation to close against.

## What This Changes

The issue requests preserving Telegram media file references from no-mention group history entries so a later @mention can inspect earlier images.

## Best Possible Solution

Add an additive media-bearing pending-history contract, have the Telegram plugin retain bounded media references for skipped mention-gated group messages, merge those references into later MediaPaths/MediaTypes during mention replay, and cover the image-without-mention followed by @mention flow with focused Telegram and shared-history tests.
