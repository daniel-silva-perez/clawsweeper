## Summary

Current main still appears to have the reported Telegram group-history media gap: media is downloaded before mention gating, but skipped no-mention group messages are recorded as text-only history and later mention replay only builds media context from the current message plus reply media. The related fix PRs #40470 and #40490 are both closed unmerged, so there is no merged implementation to close against.

## What This Changes

The issue requests preserving Telegram media file references from no-mention group history entries so a later @mention can inspect earlier images.

## Best Possible Solution

Add a backward-compatible media-bearing pending-history contract, have the Telegram plugin retain bounded media references for skipped mention-gated group messages, merge those references into later MediaPaths/MediaTypes during mention replay, and cover the image-without-mention followed by @mention flow with focused Telegram and shared-history tests.
