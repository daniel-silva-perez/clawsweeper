## Summary

Current main still matches the reported Telegram failure. Media is downloaded into Telegram `allMedia`, but the mention-gated group skip path records only text history, the shared reply-history and inbound-history shapes still have no media fields, and later Telegram context construction only forwards current-turn plus reply media. The linked fix PRs #40470 and #40490 are closed unmerged, so there is no shipped implementation to close this against.

## Best Possible Solution

Keep this issue open. Implement a focused Telegram/shared-history fix: add optional media references to reply history and inbound history, persist or refresh media for skipped mention-gated Telegram group messages under a documented lifetime policy, merge replayed history media into later Telegram `MediaPaths`/`MediaTypes`, and add a regression test for image-without-mention followed by an @mention asking about that image.

## Risks / Open Questions

- No live Telegram group reproduction was run during this read-only review; the keep-open decision is based on current source paths directly matching the reported failure mode.
- The eventual fix needs a clear retention and privacy policy for group-history media so files survive long enough for replay without persisting longer than intended.
- Because `HistoryEntry` is exported through the plugin SDK and `MsgContext.InboundHistory` is shared prompt context, the fix should be additive and covered by focused contract plus Telegram regression tests.
