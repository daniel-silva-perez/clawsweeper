## Summary

Keep this open. Current main still has the Feishu monitor loading the full message/card handlers and parser statically from `bot.ts`/`card-action.ts`, while the parser is still implemented inside `bot.ts`; the open, unmerged PR #64820 explicitly targets this issue with closing syntax and proposes the parser extraction/lazy import fix, so the issue should stay open until that PR lands or is replaced.

## What This Changes

The issue reports a Feishu group-chat mention crash in multi-agent setups where mentioning a bot throws `ReferenceError: Cannot access 'utils_1' before initialization` and prevents the mentioned bot from responding.

## Best Possible Solution

Review, rebase if needed, and validate PR #64820 or an equivalent narrow Feishu plugin change that extracts the parser into a lightweight module, lazy-loads the heavy message/card handlers from `monitor.account.ts`, preserves the existing parser export for compatibility, and adds focused module-isolation plus group-mention coverage before closing this issue as implemented.
