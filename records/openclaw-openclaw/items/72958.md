## What This Changes

The PR changes context-window guard thresholds from fixed 16k/32k cutoffs to 4k/8k safety floors with 10%/20% derived thresholds, removes fixed runner overrides, and updates focused tests, local-model docs, and the changelog.

## Best Possible Solution

Review and land this PR, or an equivalent focused patch, if maintainers agree with the floor-plus-reference-window semantics. The implementation should keep the guard logic centralized, keep runtime overrides out of the embedded runner, and preserve the updated docs/tests/changelog coverage for #42999.

## Reproduction Assessment

## Solution Assessment

Yes, with one design point for maintainer confirmation. Centralizing dynamic threshold resolution in evaluateContextWindowGuard and removing fixed runner overrides is the narrow maintainable fix; maintainers should explicitly accept the floor-only behavior for uncapped model windows.
