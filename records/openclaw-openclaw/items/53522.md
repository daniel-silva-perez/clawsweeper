## Summary

Keep this PR open. Current main already has the web_fetch maxResponseBytes schema fix and the basic dual-stack loopback listener suppression, but it still lacks the Synology Chat HEAD probe and 200 JSON ACK behavior that this PR carries. The remaining status `gatewayReachable` refinement differs from main and needs maintainer judgment rather than cleanup closure.

## What This Changes

The PR branch modifies Synology Chat webhook handling/tests to accept HEAD probes and return 200 JSON ACKs, and changes status-all diagnosis/tests so dual-stack loopback listener suppression only applies when the gateway is reachable; its originally described web_fetch schema work is already present on main.

## Best Possible Solution

Keep this PR open for maintainer review, but narrow the decision: leave the shipped web_fetch schema fix alone, land the Synology HEAD/200 ACK behavior either via focused PR #53441 or this PR, and separately decide whether the status `gatewayReachable` refinement is the desired diagnosis behavior.
