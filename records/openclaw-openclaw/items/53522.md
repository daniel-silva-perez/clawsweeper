## Summary

Keep this PR open for maintainer review. Current main already has the web_fetch schema fix and base dual-stack listener suppression, but it still lacks the Synology Chat HEAD/200 JSON ACK behavior, and this PR also changes the status diagnostic semantics beyond the shipped fix. The overlapping focused Synology PR makes this a land/split/supersede decision rather than a conservative cleanup close.

## What This Changes

The PR branch changes Synology Chat webhook handling and tests to accept HEAD probes and return 200 JSON success ACKs with Content-Length, extends the Synology test response helper, and makes status-all dual-stack loopback suppression conditional on `gatewayReachable`.

## Best Possible Solution

Keep the shipped web_fetch schema and base dual-stack status fixes as the baseline. Land the Synology HEAD/200 JSON ACK behavior through focused PR #53441 or an equivalent narrow patch with Content-Length coverage, then decide the `gatewayReachable` status refinement separately.
