## Summary

Keep open. Current main already contains the web_fetch schema fix and the base dual-stack status suppression, but it still rejects Synology Chat HEAD probes and still ACKs accepted Synology webhook POSTs with 204, so this PR is not obsolete; because it mixes shipped overlap with remaining Synology work and a separate status semantics refinement, it needs maintainer review or splitting rather than auto-close.

## What This Changes

The PR changes Synology Chat webhook handling/tests to accept HEAD probes and return 200 JSON success ACKs with Content-Length, updates Synology test helpers/expectations, and makes status-all dual-stack loopback suppression conditional on `gatewayReachable`.

## Best Possible Solution

Keep the shipped web_fetch schema and base dual-stack status behavior as the baseline. Land the Synology HEAD/200 JSON ACK fix through a focused, rebased PR with Content-Length coverage, then evaluate the `gatewayReachable` status refinement separately if maintainers want unreachable gateways to keep warning despite same-PID loopback listeners.

## Reproduction Assessment

Yes. ## Solution Assessment

No for merging the whole PR as-is. The Synology change is a plausible narrow fix and this PR's version includes stronger ACK header coverage than the related focused diff I fetched, but the combined branch also carries already-shipped work and a separate status semantics change, so the maintainable path is split/rebase or maintainer supersedence.
