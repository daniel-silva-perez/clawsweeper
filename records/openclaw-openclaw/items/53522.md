## Summary

Keep open. Current main has already shipped the web_fetch schema fix and a broad dual-stack status suppression, but it still rejects Synology Chat HEAD probes and still ACKs successful Synology webhook POSTs with 204/no body. The remaining branch changes are useful but overlap focused PR #53441 and include a separate status-diagnosis behavior delta that needs maintainer judgment.

## What This Changes

This PR changes Synology Chat webhook handling/tests to accept HEAD probes and return 200 JSON ACKs with Content-Length, extends the test response helper, and changes status-all diagnosis/tests so dual-stack loopback suppression requires `gatewayReachable`.

## Best Possible Solution

Keep the shipped web_fetch schema and existing dual-stack status fix, then land the Synology HEAD/200 JSON ACK behavior through the focused Synology PR or an equivalent narrow patch. If maintainers want the additional `gatewayReachable` status refinement, split or review it separately with tests that define unreachable-gateway semantics.
