## Summary

Keep open for maintainer review. Current main already covers the web_fetch schema and base dual-stack status fixes, but it still rejects Synology HEAD probes and still ACKs successful Synology webhook POSTs with 204; a focused open PR #53441 covers that main remaining Synology bug, while this PR also changes status warning semantics and should be reviewed or split rather than auto-closed.

## What This Changes

This PR changes Synology Chat webhook handling and tests to accept HEAD probes and return 200 JSON ACKs with Content-Length, updates Synology response test helpers, and changes status-all dual-stack loopback diagnosis to require gatewayReachable before suppressing port warnings.

## Best Possible Solution

Keep the shipped web_fetch schema and base dual-stack suppression as the baseline. Land the Synology HEAD/200 JSON ACK behavior through focused PR #53441 or an equivalent narrow patch with Content-Length coverage, then handle the gatewayReachable status-warning policy separately if maintainers want that stricter diagnosis.
