## Summary

Keep this PR open. Current main still renders the login gate for every disconnected Control UI state, so the proposed slow-reconnect/dashboard-shell behavior is not implemented. The PR is focused on UI/auth-gate classification, the bot review follow-ups were addressed in the provided discussion, and there is no canonical merged duplicate in the supplied related context.

## Best Possible Solution

Keep the PR open for normal maintainer review. The best path is to land this Control UI-side distinction, or an equivalent implementation, using structured gateway error detail codes first and narrow message fallbacks only when needed, plus a focused regression test for disconnected-with-valid-config versus true login-required failures and the mobile Connecting indicator.
