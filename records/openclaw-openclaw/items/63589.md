## Summary

Current main still does not include participant identity or quoted reply context in the WhatsApp gateway inbound log. The data is computed and stored on the queued inbound message, and the agent-facing message can format reply context, but `inboundLogger.info()` still emits only the basic log payload. The related fix PR #63615 was closed unmerged, so this issue remains valid and is a good narrow fix-PR candidate.

## Best Possible Solution

Implement the narrow WhatsApp plugin fix in `extensions/whatsapp/src/inbound/monitor.ts`: add the already-computed participant identity and reply-context fields to the `inboundLogger.info()` object, then extend the monitor-inbox behavior test to assert those fields for a group quoted reply. Keep mention gating, queueing, delivery, and agent-facing formatting unchanged.
