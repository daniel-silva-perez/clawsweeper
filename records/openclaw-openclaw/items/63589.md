## Summary

Current main still omits WhatsApp participant identity and quoted-reply context from the gateway inbound log payload. The requested data is already computed in the same inbound monitor and preserved for queued/agent-facing messages, and the linked implementation PR #63615 is closed unmerged, so this remains a narrow valid issue.

## Best Possible Solution

Keep this issue open and land a narrow WhatsApp plugin change in extensions/whatsapp/src/inbound/monitor.ts: add the already-computed sender identity and reply-context fields to the existing inboundLogger.info payload, then extend the inbound logger test coverage with a group quoted-reply case. Do not change mention gating, access control, queue semantics, plugin hook privacy defaults, delivery behavior, or agent-facing formatting.
