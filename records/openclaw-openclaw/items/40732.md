## Summary

Keep open: current main still uses the raw textarea, so the feature is not implemented there. The PR is useful but should not merge as-is because it needs a rebase that preserves current raw-secret reveal behavior, updates the full generated locale surface, and fixes a JSON5 path edge case.

## What This Changes

The branch replaces the raw config textarea with a CodeMirror Lit component that adds cursor status, JSON5 path breadcrumbs, selection status, raw-editor styling, locale strings, dependencies, and browser coverage.

## Best Possible Solution

Land one canonical, rebased raw config editor that keeps the existing reveal-before-mount behavior for sensitive raw values, uses the current generated locale pipeline for every shipped locale, supports valid JSON5 path syntax, and then supersedes the overlapping raw-editor threads after merge.

## Reproduction Assessment

Yes. The raw-secret regression is reproducible by comparing current main's no-textarea-before-reveal render path and test with the PR hunk that mounts props.raw directly; the escaped-key issue is reproducible with a valid JSON5 key such as { \u0066oo: 1 }.

## Solution Assessment

No. The feature direction is reasonable, but this branch is not the best mergeable solution until it is rebased around current main's raw reveal gate, i18n generator, JSON5 edge cases, and the overlapping raw-editor PR/issue set.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve the reveal gate before mounting raw config:** `ui/src/ui/views/config.ts:801-804`
  - body: Current main intentionally keeps sensitive raw config text out of the DOM until the reveal button is clicked, and the browser test asserts no textarea is mounted before reveal. This hunk binds props.raw directly into the new editor, so a rebase would expose secrets before explicit reveal unless the existing redaction wrapper remains around the editor.
  - confidence: 0.9
- **[P2] Parse escaped JSON5 identifier keys:** `ui/src/ui/components/config-raw-editor-path.ts:39-40`
  - body: JSON5 object keys may be IdentifierName values, including Unicode-escaped spellings such as \u0066oo. Since the resolver rejects a leading backslash as a token start, breadcrumbs can skip or mislabel valid keys; add parsing and coverage for escaped identifier starts.
  - confidence: 0.86
- **[P2] Regenerate every shipped locale bundle:** `ui/src/i18n/locales/en.ts:127-134`
  - body: Current main ships 18 lazy non-English Control UI locales and tests that every locale structurally matches English. Adding config.rawEditor.* to English and only six older locale files leaves the newer generated bundles missing keys, so the rebased i18n check will fail until the generator updates all bundles and metadata.
  - confidence: 0.84
