## Summary

Keep this PR open: current main still uses a textarea for raw config editing, so the feature is not implemented there, but the PR is merge-conflicted and needs fixes before it is a safe candidate to land. The main blocker is that the branch does not preserve the current raw-config sensitive-value reveal gate, and the JSON5 breadcrumb parser still misses escaped identifier keys.

## What This Changes

The PR replaces the raw JSON5 config textarea with a CodeMirror 6 Lit editor, line/column and path breadcrumb status UI, raw editor styling, locale strings, CodeMirror dependencies, and browser tests.

## Best Possible Solution

Rebase one canonical raw config editor implementation onto current main, keep the sensitive raw reveal/redaction and raw save/change semantics intact, fix JSON5 breadcrumb parsing for escaped identifier keys, route strings through the current UI i18n sync pipeline, and land focused browser coverage for the final raw editor behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve raw secret redaction around the editor:** `ui/src/ui/views/config.ts:800-803`
  - body: Current main hides raw config text when sensitive values are present until the user explicitly reveals it. This branch renders `config-raw-editor` with `props.raw` directly in raw mode, so rebasing/merging as-is would put tokens back on screen and in the DOM and would fail the existing sensitive raw config browser coverage. Keep the reveal gate around the editor or make the editor read-only/hidden until reveal.
  - confidence: 0.88
- **[P2] Handle escaped JSON5 identifier keys:** `ui/src/ui/components/config-raw-editor-path.ts:39-40`
  - body: `isValueTokenStart` rejects `\`, so valid JSON5 object keys that begin with Unicode escapes, such as `\u0066oo`, are skipped or displayed as the wrong segment. Treat identifier escapes as key-token starts before computing status paths and breadcrumb jump targets.
  - confidence: 0.84
