## Summary

Keep this PR open, but it is not mergeable as-is. Current main still uses the raw textarea behind an explicit reveal gate, and this branch predates that security-sensitive behavior plus the current generated-locale policy; the feature direction is useful but needs a maintainer-chosen canonical implementation across the overlapping raw-editor work.

## What This Changes

The PR branch replaces the raw config textarea with a CodeMirror-based Lit editor that adds cursor status, JSON5 path breadcrumbs, raw-editor styling, localized status labels, dependencies, and browser coverage.

## Best Possible Solution

Land one rebased canonical raw config editor that preserves the existing reveal-before-mount behavior for sensitive raw values, follows the generated locale pipeline for every shipped locale, supports valid JSON5 path syntax, and then supersedes or closes the overlapping raw-editor threads after merge.

## Reproduction Assessment

Yes. The raw-secret regression is reproducible by comparing current main's no-textarea-before-reveal render path and test with the PR hunk that binds props.raw directly into config-raw-editor; the escaped-key issue is reproducible from the helper token-start logic with a valid JSON5 key such as { \u0066oo: 1 }.

## Solution Assessment

No. The feature direction is reasonable, but this branch is not the best mergeable solution until it is rebased around current main's raw reveal gate, generated i18n workflow, JSON5 escaped-identifier handling, and the overlapping raw-editor work.
