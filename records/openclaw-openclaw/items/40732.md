Keep this PR open, but it should not merge as-is. Current main still lacks the enhanced raw editor, so the contribution is not obsolete, but the branch is stale against current raw secret redaction, JSON5 path parsing needs one more valid-key case, and the locale changes need to follow the expanded generated i18n surface.

## What This Changes

The branch adds a CodeMirror-based raw config editor with cursor status, JSON5 path breadcrumbs, styling, locale strings, dependency updates, and browser coverage.

## Best Possible Solution

Land one rebased canonical raw config editor implementation that preserves current redaction and raw round-trip semantics, fixes JSON5 path parsing for escaped identifiers, updates locales through the i18n generator, and resolves the overlap with #40009 and #56401.

  - body: Current main withholds sensitive raw config text from the UI and DOM until the reveal toggle is pressed.   - confidence: 0.86

  - body: Current main ships 18 Control UI locales and has a test that every locale matches English structurally. Adding `config.rawEditor.*` only to English and the old six-locale set leaves the newer locale files without those keys, so the rebased i18n test will fail until the locale generator updates all bundles and metadata.
  - confidence: 0.84
