## Summary

Keep open: current main has not implemented the CodeMirror raw editor, but this PR should not merge as-is because it is dirty against main, overlaps another open raw-editor PR, and the proposed raw-editor integration bypasses current sensitive-value reveal/redaction behavior. The path resolver also still misses a valid JSON5 escaped-identifier key case.

## What This Changes

The branch adds a CodeMirror-based raw JSON5 config editor component, cursor path/status logic, styling, locale strings, CodeMirror dependencies, and browser tests.

## Best Possible Solution

Land one canonical raw config editor implementation on current main that preserves raw availability, save/reset semantics, raw pending-change redaction, and reveal-to-edit behavior; fix JSON5 path parsing for escaped identifier keys; regenerate locale output through the current UI i18n pipeline; then close or supersede the overlapping raw-editor items consistently.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve raw secret redaction around the editor:** `ui/src/ui/views/config.ts:800-803`
  - body: Current main hides raw config text when sensitive values exist until the user explicitly reveals it. This branch mounts `config-raw-editor` with `props.raw` directly in raw mode, so rebasing this hunk as-is would render secrets in the visible editor and DOM before reveal and break the existing raw redaction browser coverage. Keep the current blurred/reveal gate around the new editor.
  - confidence: 0.88
- **[P2] Handle escaped JSON5 identifier keys:** `ui/src/ui/components/config-raw-editor-path.ts:39-40`
  - body: JSON5 object member names may be ECMAScript `IdentifierName` tokens, including Unicode escape starts such as `\u0066oo`. Because `isValueTokenStart` rejects `\`, the resolver skips or mislabels those valid keys, so breadcrumb display and jump targets become wrong for valid JSON5 configs.
  - confidence: 0.84
