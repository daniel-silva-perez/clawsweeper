## Summary

Keep open. The PR is useful source work, but current main still has the textarea-based raw config editor and the branch is dirty against current main; rebasing it as-is would conflict with newer raw-config secret reveal/redaction behavior and the expanded generated locale surface.

## What This Changes

The PR replaces the raw JSON5 config textarea with a CodeMirror-based Lit editor, adds cursor status and breadcrumb path navigation, updates raw editor styling, adds CodeMirror dependencies, adds six locale entries, and adds browser tests.

## Best Possible Solution

Land one rebased canonical raw config editor that keeps the current raw secret reveal gate, updates all generated Control UI locale bundles and metadata, supports valid JSON5 path syntax, and then closes or supersedes the overlapping raw-editor threads after merge.

## Reproduction Assessment

Yes. The raw-secret regression is reproducible by comparing current main's no-mount-before-reveal test and render gate with the PR hunk that passes props.raw directly into config-raw-editor; the escaped-key issue is reproducible with a valid JSON5 key such as { \u0066oo: 1 }.

## Solution Assessment

No. The feature direction is reasonable, but this branch is not the best mergeable solution until it is rebased around current main's reveal gate, generated locale pipeline, JSON5 edge cases, and the overlapping raw-editor work.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve the reveal gate before mounting raw config:** `ui/src/ui/views/config.ts:801`
  - body: Current main intentionally keeps sensitive raw config text out of the DOM until the reveal button is clicked, and the browser test asserts no textarea is mounted before reveal. This hunk binds props.raw directly into the new editor, so a rebase would expose secrets before explicit reveal unless the existing redaction wrapper remains around the editor.
  - confidence: 0.9
- **[P2] Parse escaped JSON5 identifier keys:** `ui/src/ui/components/config-raw-editor-path.ts:39-40`
  - body: JSON5 object keys may be IdentifierName values, including Unicode-escaped spellings such as \u0066oo. Since the resolver rejects a leading backslash as a token start, breadcrumbs can skip or mislabel valid keys; add parsing and coverage for escaped identifier starts.
  - confidence: 0.86
- **[P2] Regenerate every shipped locale bundle:** `ui/src/i18n/locales/en.ts:127-134`
  - body: Current main ships 18 lazy non-English Control UI locales and tests that every locale structurally matches English. Adding config.rawEditor.* to English and only six older locale files leaves the newer generated bundles missing keys, so the rebased i18n check will fail until the generator updates all bundles and metadata.
  - confidence: 0.84
