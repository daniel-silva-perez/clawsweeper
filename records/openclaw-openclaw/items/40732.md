## Summary

Keep this PR open: current main has not implemented the CodeMirror/status-bar raw editor, and the branch contains useful UI work, but it is not mergeable as-is against current main because it would need to preserve the raw-config reveal gate, handle a valid JSON5 escaped-key case, and regenerate the full current locale surface before review can proceed.

## What This Changes

The PR replaces the raw config textarea with a CodeMirror-based Lit editor, adds JSON5 cursor breadcrumb/status parsing and raw-editor styling, adds browser coverage, adds six locale entries, and introduces CodeMirror dependencies.

## Best Possible Solution

Land one rebased canonical raw config editor that keeps the existing reveal/redaction semantics, uses generated i18n output for every registered Control UI locale, supports valid JSON5 path syntax, and then closes or supersedes the overlapping raw-editor threads after merge.

Yes. The raw-secret regression is statically reproducible by comparing current main's no-mount-before-reveal test with the PR hunk that passes props.raw into config-raw-editor, and the escaped-key gap can be checked with a valid JSON5 key such as { \u0066oo: 1 }.

## Solution Assessment

No. The direction is useful, but the current proposed branch is not the best mergeable solution until it is rebased around current main's reveal gate, full generated locale pipeline, and the overlapping raw-editor work.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve the reveal gate before mounting raw config:** `ui/src/ui/views/config.ts:801-804`
  - body: Current main intentionally avoids mounting raw config text while sensitive values are hidden, and tests assert the secret is absent before reveal. This hunk passes props.raw directly into config-raw-editor, so a rebase would put secret text in the UI/DOM before the user reveals it; keep the existing redaction callout/reveal wrapper around the new editor.
  - confidence: 0.9
- **[P2] Parse escaped JSON5 identifier keys:** `ui/src/ui/components/config-raw-editor-path.ts:39-40`
  - body: JSON5 object keys may be ECMAScript IdentifierName values, including Unicode-escaped spellings such as \u0066oo. Since the resolver rejects backslash as a token start, the breadcrumb path can skip or mislabel valid keys; add parsing and coverage for escaped identifier starts.
  - confidence: 0.86
- **[P2] Regenerate every locale bundle for the new keys:** `ui/src/i18n/locales/en.ts:127-134`
  - body: Current main ships 18 lazy Control UI locales and tests that every locale matches English structurally. Adding config.rawEditor.* to English and only the older six-locale set leaves the newer locale files and generated metadata missing keys, so the rebased i18n check will fail until the generator updates all bundles.
  - confidence: 0.84
