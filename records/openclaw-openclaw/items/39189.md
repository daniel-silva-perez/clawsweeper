## Summary

Keep this PR open. Current main still lacks the exact Anthropic `specified API usage limits` / `regain access` billing matcher, but the PR as written also has unresolved functional regressions: its billing matcher is broad enough to catch ordinary usage-window limits, and its unconditional text dedupe can suppress legitimate same-text replies sent to a different target.

## Best Possible Solution

Keep this PR open, but rework or split it. The failover fix should add a narrow Anthropic billing matcher and regression tests for the exact `specified API usage limits` plus `regain access` wording without shadowing periodic usage-window limits. The reply-dedupe fix should use identity-aware target matching, likely through the existing channel suppression hook or target normalization for iMessage/BlueBubbles aliases, while preserving same-text cross-target delivery tests.
