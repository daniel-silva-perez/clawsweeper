## Summary

Keep this issue open. Current main still has plain apt-get update/install calls in the sandbox-common build path, the documented setup script still builds Dockerfile.sandbox-common, and the closed same-author PR #52724 was not merged. The remaining work is a focused Dockerfile hardening change plus a small source-level guard test.

## Best Possible Solution

Keep the issue open and land a focused Dockerfile hardening patch. Add bounded apt retry handling to Dockerfile.sandbox-common and likely the cold-start Dockerfile.sandbox path, decide deliberately whether --fix-missing belongs in a reproducible image build, and add a narrow source-level assertion in src/docker-build-cache.test.ts or an adjacent Dockerfile test so the retry contract does not regress.
