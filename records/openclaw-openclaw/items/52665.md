## Summary

Keep this issue open. Current main still does not implement the requested sandbox-common apt retry/fix-missing hardening: the user-facing setup path builds Dockerfile.sandbox-common, whose apt operations are still plain apt-get calls, and the cold-start base image path also lacks apt retry handling. The related same-author PR #52724 was an attempted fix, but this checkout has not picked up the requested behavior.

## Best Possible Solution

Keep the issue open and land a focused Dockerfile hardening change. Apply retry handling to apt-get update and install in Dockerfile.sandbox-common, consider whether --fix-missing is appropriate, and cover the cold-start Dockerfile.sandbox path as well. Add a narrow Dockerfile/source test alongside src/docker-build-cache.test.ts or src/dockerfile.test.ts so the retry contract does not regress.
