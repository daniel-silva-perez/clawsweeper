## Summary

Keep open. Current main still does not implement this PR's intended fix: the Gateway chat history sanitizer and Control UI message extraction paths strip runtime context, inbound metadata, envelopes, and message IDs from user messages, but they do not strip `<relevant-memories>` blocks from user-role text. The PR remains useful, though it should address the review concern about unconditional `trimStart()` and be coordinated with overlapping PR #51288.

## Best Possible Solution

Keep this PR open or consolidate it into the canonical fix. The best implementation should strip `<relevant-memories>` from user-role display/history text in both `src/gateway/chat-sanitize.ts` and `ui/src/ui/chat/message-extract.ts`, preserve user-authored leading whitespace unless a memory block was actually removed, and add regression tests for string, content-array, and bare-text shapes.
