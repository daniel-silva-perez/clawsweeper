## Summary

Keep this issue open. Current main still lacks a Feishu outbound minimum-interval/rate-limit setting, and the main outbound reply, comment, message-tool, media, and streaming-card paths do not route through a shared configurable per-account pacing layer. The related #13615 is broader external API throttling work and does not supersede this Feishu-specific channel request.

## Best Possible Solution

Keep this issue open and implement Feishu-owned outbound pacing in the bundled Feishu plugin. Add documented top-level and per-account minimum-interval/rate-limit config, expose it through schema/manifest/generated metadata/docs, route reply chunks, comment replies, message-tool text/card/media sends, pairing/bot notifications, fallbacks, and streaming-card create/update/close paths through one per-account pacing helper, and cover the behavior with fake-timer tests. If #13615 later provides a generic limiter contract, use it underneath, but close this only once Feishu exposes and applies the setting.
