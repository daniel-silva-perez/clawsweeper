## Summary

Keep open. Current main still lacks a Feishu outbound minimum-interval or pacing setting, and the burst-prone reply, comment reply, message-tool, media, and streaming-card send paths are still not covered by a shared per-account outbound pacer. The related open trackers cover broader cross-provider rate limiting and reactive Feishu retry/backoff, but they do not supersede this proactive Feishu message-spacing request.

## What This Changes

Request a configurable Feishu message rate-limiting/minimum-interval setting, ideally also affecting how long LLM responses are segmented into outbound messages.

## Best Possible Solution

Add a Feishu-owned per-account outbound pacing layer, resolved from top-level and per-account config, and apply it consistently around visible outbound sends that can burst API calls. Keep broad cross-provider limit policy in #13615 and reactive rate-limit retry/backoff in #70879 unless maintainers intentionally expand the scope.
