## Summary

Keep open. Current main still lacks a Feishu-owned outbound minimum-interval or pacing configuration, and the reply, comment reply, message-tool, media, card, and streaming-card paths still have burst-prone sends without a shared per-account pacing gate. The related broader trackers cover cross-provider rate limiting and reactive retry/backoff, but they do not supersede this proactive Feishu message-spacing request.

## What This Changes

The issue requests configurable Feishu message rate limiting, including a minimum interval between outbound messages and better behavior for segmented LLM responses.

## Best Possible Solution

Add a Feishu-owned per-account outbound pacing layer, resolved from top-level and per-account config, and apply it consistently around burst-prone visible outbound sends. Keep broad cross-provider rate-limit policy in #13615 and reactive Feishu retry/backoff in #70879 unless maintainers intentionally expand scope.

## Repair Work Prompt

Implement Feishu outbound message pacing for the Feishu/Lark plugin. The observable bug is that segmented LLM replies and other burst-prone Feishu sends can call Feishu APIs back-to-back, triggering server-side flow control. Add a Feishu-owned configurable minimum interval/pacing setting, resolved from both top-level `channels.feishu` and `channels.feishu.accounts.<id>` config. Apply it consistently to visible outbound message delivery paths that can burst: normal reply chunks, comment reply chunks, message-tool/outbound text, card sends, media sends/fallbacks, and streaming-card final/closeout calls where appropriate. Prefer a small local helper/pacer in the Feishu plugin; do not introduce broad cross-provider rate-limit architecture or fold in retry-after-error behavior from #13615 or #70879 unless maintainers explicitly expand scope. Update `extensions/feishu/openclaw.plugin.json`, `docs/channels/feishu.md`, tests, and add a user-facing `CHANGELOG.md` entry crediting `Thanks @immars`. Keep core extension-agnostic and stay inside the plugin boundary.
