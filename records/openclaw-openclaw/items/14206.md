## Summary

Keep this issue open. Current main at ab5c8025c9d0a31873d311ae3e73f3664b0a1185 still lacks a Feishu outbound minimum-interval/rate-limit config field and the Feishu reply, comment, message-tool, card/media, fallback, and streaming-card paths do not share a configurable per-account pacing helper. The related #13615 remains a broader external API throttling tracker and does not supersede this Feishu-specific channel request.

## Best Possible Solution

Keep this issue open and implement Feishu-owned outbound pacing in the bundled Feishu plugin. Add a documented top-level and per-account minimum outbound interval setting, expose it through schema/manifest/docs, and route reply chunks, comment replies, message-tool text/card/media sends, fallbacks, and streaming-card create/update/close calls through one per-account pacing helper. If #13615 later adds a generic limiter contract, Feishu can adopt it underneath, but this issue should close only once the Feishu setting exists and is applied.
