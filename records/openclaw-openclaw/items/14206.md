## Summary

Keep open. Current main still lacks a Feishu outbound minimum-interval or pacing setting, and the reply, comment, message-tool, media, card, and streaming-card send paths can still call Feishu APIs without a shared per-account pacing gate. Related #13615 and #70879 are useful sibling trackers, but they do not supersede this narrower proactive Feishu pacing request.

## What This Changes

Add configurable Feishu outbound message pacing so chunked or otherwise bursty OpenClaw replies do not trigger Feishu API rate limits.

## Best Possible Solution

Add a Feishu-owned per-account outbound pacing layer, resolved from top-level and per-account config, and apply it consistently around burst-prone visible outbound sends while leaving broad cross-provider limit policy and retry/backoff semantics to their existing trackers.

## Repair Work Prompt

Implement proactive Feishu outbound message pacing for this issue. Add a Feishu-owned configurable minimum interval, preferably a clear `outboundMinIntervalMs`-style setting unless existing repo naming suggests better, accepted at both `channels.feishu` and `channels.feishu.accounts.<id>` through the config schema, types, manifest metadata, and docs. Resolve it through existing merged account config and default to disabled/0 unless a maintainer asks for a nonzero default. Add a small per-account queue/pacer in the Feishu plugin and apply it to visible outbound API calls that can burst: chunked normal replies, comment-thread replies, `feishuOutbound` message-tool text/card/media/fallback sends, `sendMessageFeishu`/`sendCardFeishu` direct create/reply calls, `sendImageFeishu`/`sendFileFeishu`, and streaming card start/close/update only where it protects Feishu calls without breaking existing streaming coalescing. Keep broad cross-provider rate limiting in #13615 and retry/backoff for rate-limit errors in #70879 out of scope. Add focused tests for schema/default/account inheritance and for consecutive chunk/media/comment sends waiting on the per-account pacer. Update `docs/channels/feishu.md`, `extensions/feishu/openclaw.plugin.json`, and add a single-line user-facing `CHANGELOG.md` entry with `Thanks @immars`. Do not add a new core rate-limit API or dependency unless maintainers explicitly request it.
