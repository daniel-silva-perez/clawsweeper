## Summary

Keep open. Current main still lacks a Feishu outbound minimum-interval or pacing configuration, and the burst-prone reply, comment reply, message-tool, media, card, and streaming-card paths still call Feishu send APIs directly or only use unrelated streaming-update/typing backoff behavior. The related open trackers cover broader cross-provider rate limiting and reactive Feishu retry/backoff, but they do not supersede this proactive Feishu message-spacing request.

## What This Changes

Add configurable Feishu message rate limiting so users can set a minimum interval between outbound messages, ideally also influencing how LLM responses are split into Feishu messages.

## Best Possible Solution

Add a Feishu-owned per-account outbound pacing layer, resolved from top-level and per-account config, and apply it consistently around visible outbound sends that can burst Feishu API calls. Keep broad cross-provider rate-limit policy in #13615 and reactive Feishu retry/backoff in #70879 unless maintainers intentionally expand the scope.
