## Summary

Keep this issue open. Current main at 9bb1e59447a5fdf2c0164672df6c748188b26442 still has no Feishu outbound minimum-interval/rate-limit config, and the visible Feishu send paths still call message/card/media/comment/streaming APIs directly without a shared configurable per-account pacing helper. The broader #13615 core rate-limit tracker and #70879 Feishu retry issue are related but do not supersede this narrower proactive Feishu outbound pacing request.

## Best Possible Solution

Implement Feishu-owned outbound pacing in the bundled Feishu plugin. Add a documented top-level and per-account minimum outbound interval setting, expose it through schema, manifest, and docs, and route normal replies, comment replies, message-tool text/card/media sends, fallbacks, pairing notifications, and streaming-card create/update/close calls through one per-account pacing helper. Keep #13615 as the broader core limiter design tracker and #70879 as the separate retry/backoff follow-up unless a PR intentionally covers those scopes too.
