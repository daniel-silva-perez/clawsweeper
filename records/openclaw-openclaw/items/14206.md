## Summary

Keep this issue open. Current main at 1b25dcf57a9fca5d00cf275046e20a2e3ea3730c still lacks a documented Feishu outbound minimum-interval/rate-limit config field, and normal reply chunks, comment replies, message-tool sends, card/media sends, fallbacks, pairing notifications, and streaming-card operations are not routed through a shared configurable per-account pacing helper. The broader #13615 tracker and the related #70879 retry issue do not supersede this Feishu-specific outbound pacing request.

## Best Possible Solution

Keep this open and implement Feishu-owned outbound pacing in the bundled Feishu plugin. Add a documented top-level and per-account minimum outbound interval setting, expose it through schema, manifest, and docs, and route normal replies, comment replies, message-tool text/card/media sends, fallback sends, pairing notifications, and streaming-card create/update/close calls through one per-account pacing helper. If #13615 later adds a generic limiter contract, Feishu can adopt it underneath, but this issue should close only once the Feishu setting exists and is applied.
