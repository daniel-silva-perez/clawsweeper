## Summary

Keep open. Current main still lacks the Moonshot/Kimi outgoing payload repair for replayed assistant tool-call messages missing top-level `reasoning_content`, and open PR #74846 remains the active implementation path for this exact bug.

## What This Changes

Restore or synthesize Moonshot/Kimi `reasoning_content` on replayed assistant tool-call messages after long-session LCM compaction or cross-model fallback.

## Best Possible Solution

Land a narrow Moonshot-only outgoing payload repair at the stream-wrapper boundary that preserves existing `reasoning_content`, adds an empty placeholder only for thinking-enabled assistant tool-call replay messages, and covers enabled, disabled, and preserve-existing cases with focused tests.

## Solution Assessment
