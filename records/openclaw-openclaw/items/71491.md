## Summary

Keep open. Current main still does not add or restore Moonshot/Kimi top-level `reasoning_content` on replayed assistant tool-call messages after compaction or cross-model fallback, and an open contributor PR (#74846) already targets the narrow provider-side repair. The issue is current, concrete, and distinct from the already-merged Kimi tool-call ID fix in #70030.

## What This Changes

The issue reports that Moonshot/Kimi K2.6 thinking conversations can fail with 400 errors after LCM compaction or fallback because replayed assistant tool-call messages lack top-level `reasoning_content`.

## Best Possible Solution

Review and land PR #74846 or an equivalent narrow Moonshot-only outgoing payload repair that preserves existing `reasoning_content`, adds a placeholder only when Moonshot thinking is enabled for replayed assistant tool-call messages, and leaves generic OpenAI-compatible replay behavior unchanged.

## Reproduction Assessment

Yes. The issue body and comments give concrete long-session and fallback reproduction paths, and current-main inspection shows the Moonshot payload path still lacks the reported `reasoning_content` repair.

## Solution Assessment

Yes. The best current path is a Moonshot stream-wrapper payload repair after context assembly, mirroring the existing DeepSeek V4 compatibility pattern while avoiding broad transcript-policy changes.
