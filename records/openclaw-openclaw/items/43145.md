## Summary

Keep this PR open. Current main still does not implement the central run-trace timeline surface: there is no `agent.timeline` Gateway method, no timeline schema/validator, no generated Swift model, and no retained `AgentRunTrace` store. Some adjacent behavior from the PR, especially synchronous block-reply delivery and fenced-block chunking coverage, is already present on main, but that does not replace the proposed timeline feature. #68204 is a useful open design thread for a broader unified run trace schema, but it is not a merged implementation and does not fully supersede this PR by itself.

## Best Possible Solution

Keep this PR open for maintainer judgment, but require a split and rebase. Align the trace design with #68204, then land a narrow trace-only PR with a canonical schema, object-level authorization, redaction, retention/span caps, generated client updates, docs, and focused tests. Drop or split unrelated cron, sandbox, planner, roadmap, generated-client, and drive-by test changes; omit already-shipped block-reply/chunker behavior from the refreshed branch.
