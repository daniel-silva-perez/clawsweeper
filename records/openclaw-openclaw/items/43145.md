## Summary

Keep this PR open. Current main still does not implement the PR’s central retained run-trace timeline surface: no `agent.timeline` Gateway method, no protocol schema/validator, and no `AgentRunTrace` timeline store are present. Some adjacent work exists, especially trajectory bundles and current block-reply/chunker behavior, but that is not the same short-lived `runId`-keyed Gateway timeline. The PR should not merge as-is because the branch is broad and still has security-sensitive review concerns around timeline authorization/retention plus unrelated sandbox, cron, planner, docs, and generated-client changes.

## Best Possible Solution

Keep this PR open for maintainer judgment, but require a split and rebase. Align the trace design with #68204, then land a narrow trace PR with canonical schema, object-level authorization, redaction, retention/span caps, generated client updates, docs, and focused tests. Drop or split the unrelated cron, sandbox, planner, roadmap, generated-client, and drive-by test changes; the already-shipped adjacent block-reply/chunker behavior can be omitted from a refreshed trace-only branch.
