## Summary

Keep this PR open. Current main still does not implement the PR’s central retained run-trace timeline surface: there is no `agent.timeline` Gateway method, no protocol schema/validator, no generated Swift model, and no `AgentRunTrace` timeline store. Some adjacent behavior has shipped, especially trajectory bundles and block-reply/chunker handling, but those do not replace the proposed short-lived `runId`-keyed Gateway timeline. The PR should not merge as-is because the branch is broad and has unresolved security-sensitive review concerns.

## Best Possible Solution

Keep this PR open for maintainer judgment, but require a split and rebase. Align the trace design with #68204, then land a narrow trace PR with canonical schema, object-level authorization, redaction, retention/span caps, generated client updates, docs, and focused tests. Drop or split the unrelated cron, sandbox, planner, roadmap, generated-client, and drive-by test changes; omit already-shipped adjacent block-reply/chunker behavior from a refreshed trace-only branch.
