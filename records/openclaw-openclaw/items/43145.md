## Summary

Keep this PR open. Current main still does not implement the central retained run-trace timeline surface: no `agent.timeline` Gateway method, no protocol schema or validator, no generated Swift model, and no `AgentRunTrace` timeline store. Some adjacent streaming/chunker behavior is already on main, and #68204 is a related open architecture issue, but neither replaces this PR’s proposed runId-keyed timeline implementation.

## Best Possible Solution

Keep this PR open for maintainer judgment, but require a split and rebase. Align the trace design with #68204, then land a narrow trace PR with a canonical schema, object-level authorization, redaction, retention/span caps, generated client updates, docs, and focused tests. Drop or split unrelated cron, sandbox, planner, roadmap, generated-client, and drive-by test changes; omit already-shipped adjacent block-reply/chunker behavior from a refreshed trace-only branch.
