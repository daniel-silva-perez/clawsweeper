## Summary

Keep this issue open. Current main has descendant subagent announce gating and task/status visibility, but the remaining report is broader: subagent task completion still follows the child agent-run outcome, while child-owned background exec/process work continues through a separate process-registry lifecycle.

## What This Changes

The issue reports that subagents can appear completed in parent/session status before delegated background exec or other long-running work owned by the child session is actually finished.

## Best Possible Solution

Define the lifecycle contract explicitly, then implement one coherent path: either keep subagent completion agent-run authoritative and surface child-owned background processes as separate active work in `/tasks` and `session_status`, or defer subagent finalization through a generic owned-process liveness seam with timeout, cancel, cleanup, and regression coverage.
