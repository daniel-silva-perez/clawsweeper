## Summary

Keep open. Current main has improved adjacent subagent announce and task-status paths, but the remaining lifecycle gap still exists: subagent task completion is finalized from the child agent-run result while child-owned background exec sessions are represented in a separate process registry and can outlive the run result.

## What This Changes

Ensure subagent completion/status stays non-terminal until the delegated user-relevant work, including background process work started by the child, is actually complete or clearly surfaced as separate active work.

## Best Possible Solution

Define the lifecycle contract explicitly, then implement one coherent path: either keep subagent completion agent-run authoritative while surfacing child-owned background processes as separate active work in `/tasks` and `session_status`, or defer subagent finalization through a generic owned-process liveness seam with bounded timeout, cancellation, cleanup, and regression coverage.
