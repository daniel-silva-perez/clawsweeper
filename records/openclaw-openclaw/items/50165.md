## Summary

Keep this open. Current main still finalizes subagent task state from the child agent-run outcome while child-owned background exec/process sessions are tracked separately, so the core lifecycle ambiguity described in the discussion remains valid.

## What This Changes

The issue asks OpenClaw to avoid reporting subagents as completed while delegated child work, especially background exec/process work, is still active or not clearly surfaced.

## Best Possible Solution

Define the lifecycle contract explicitly, then implement one coherent path: either keep agent-run completion authoritative and surface child-owned background processes as separate active work in /tasks and session_status, or defer subagent finalization through a generic owned-process liveness seam with bounded timeout, cancellation, cleanup, and regression coverage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not yet from attached logs or tests; the code supports a focused harness where a subagent starts a background exec/process, returns, and the task finalizes before the scoped process exits.

## Solution Assessment

Is this the best way to solve the issue? Unclear because no patch is proposed and the right fix depends on maintainer lifecycle policy; the safest direction is to choose one contract before adding code.
