## Summary

Keep this open. Current `main` still finalizes subagent task state from the child agent-run outcome, while yielded/backgrounded exec processes are tracked in a separate process registry and are not part of that completion gate. Recent subagent lifecycle work improves `sessions_yield`, descendant subagent announce timing, and cleanup ordering, but it does not resolve the generic background-process liveness gap described in the issue discussion.

## What This Changes

Subagents should remain visibly in progress until delegated child work and the user-relevant output pipeline are actually complete.

## Best Possible Solution

Define the lifecycle contract explicitly, then implement one coherent path: either keep agent-run completion authoritative and expose child-owned background processes as separate active work in `/tasks` and `session_status`, or defer subagent finalization through a generic owned-process liveness seam with timeout, cancellation, cleanup, and regression coverage.
