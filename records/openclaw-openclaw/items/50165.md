## Summary

Keep this open. Current main still treats subagent task completion as the child agent-run terminal outcome, while child-owned background exec/process sessions are tracked through a separate scoped process registry and can outlive the agent turn described in the report.

## What This Changes

The issue requests that subagents remain visibly in progress until delegated user-relevant work, including long-running child-side background work, is actually complete.

## Best Possible Solution

Define the lifecycle contract explicitly, then implement one coherent path: either keep subagent completion agent-run authoritative and surface child-owned background processes as separate active work in `/tasks` and `session_status`, or defer subagent finalization through a generic owned-process liveness seam with timeout, cancellation, cleanup, and regression coverage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Unclear: the report has no focused logs, but current code supports a narrow harness where a subagent starts a background exec/process, returns successfully, and the subagent task finalizes before the scoped process exits.

## Solution Assessment

Is this the best way to solve the issue? Unclear because no patch is proposed and the right fix depends on maintainer lifecycle policy; the safest direction is to choose the contract before adding code.
