## Summary

Keep this open. Current main has substantial subagent announce and task/status work, but the remaining report is still valid as a lifecycle-contract gap: subagent completion is finalized from the child agent-run result, while child-owned background exec/process sessions are tracked through a separate process registry and can outlive the agent turn.

## What This Changes

The issue asks OpenClaw to keep delegated subagents visibly in progress until their user-relevant work, including long-running background work, is actually complete.

## Best Possible Solution

Define the lifecycle contract explicitly, then implement one coherent path: either keep subagent completion agent-run authoritative and surface child-owned background processes as separate active work in `/tasks` and `session_status`, or defer subagent finalization through a generic owned-process liveness seam with timeout, cancel, cleanup, and regression coverage.
