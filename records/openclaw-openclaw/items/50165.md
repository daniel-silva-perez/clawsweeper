## Summary

Keep open. Current main has strong fixes for nested subagent descendants and announce delivery, but this issue also reports subagent completion being shown while background exec/file work is still running. The current lifecycle still finalizes the subagent task from the agent-run outcome, while background exec is tracked as a separate process session and exit wake, not as a blocker for subagent completion/status.

## Best Possible Solution

Keep this issue open and implement a lifecycle bridge between subagent runs/tasks and child-session background work. At minimum, subagent completion/status should not finalize from the agent run alone when the child session has active first-class `exec`/`process` sessions that the subagent started for the delegated task; the announce path should wait for those sessions to settle, time out, or explicitly be detached/cancelled, then deliver the final user-relevant result.

## Risks / Open Questions

- A fix should avoid treating every process session as a hard blocker forever; it needs explicit ownership, timeout, and cancellation semantics for subagent-owned background work.
- Shell-level backgrounding with a trailing `&` may remain untrackable unless OpenClaw detects or discourages it separately from first-class `exec background=true`/`yieldMs` sessions.
