## Summary

Keep open. The report still describes a real lifecycle-contract gap on current main: subagent task completion is finalized from the child agent-run result, while backgrounded exec/process sessions remain in a separate scoped registry and can outlive that agent turn. The Keep My Claw comment is a mitigation for recovery, not a fix for OpenClaw reporting completion too early.

## What This Changes

The issue asks OpenClaw to keep subagent status in-progress until delegated child work, including long-running background exec/process work, is actually finished.

## Best Possible Solution

Define the lifecycle contract explicitly, then implement one coherent path: either keep agent-run completion authoritative and surface child-owned background processes as separate active work in /tasks and session_status, or defer subagent finalization through a generic owned-process liveness seam with bounded timeout, cancellation, cleanup, and regression coverage.
