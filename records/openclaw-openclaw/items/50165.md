## Summary

Keep this issue open. Current main still has the already-fixed nested-subagent announce gating, but the remaining reported case is broader: a child subagent can background shell/process work, the subagent run can finalize from the agent-run outcome, and the exec completion path is handled separately rather than blocking subagent task completion.

## What This Changes

Request that subagent lifecycle/status reporting remain non-terminal until delegated user-relevant work, including long-running background work, is actually complete.

## Best Possible Solution

Define the lifecycle contract explicitly: either subagent completion remains agent-run authoritative and status surfaces must clearly show active child-session background processes as separate work, or subagent finalization must defer on owned background exec/process sessions through a generic liveness seam with timeout/cancel semantics and regression coverage.
