## Summary

Keep open. Current main has fixed several adjacent subagent announce and task-visibility paths, but it still treats the child agent-run outcome as authoritative for subagent task completion while shell background exec sessions live in a separate process registry and can outlive the child run.

## What This Changes

The issue asks OpenClaw to keep delegated subagents in a running or in-progress state until underlying long-running work is actually complete.

## Best Possible Solution

Define the lifecycle contract explicitly, then implement one coherent path: either keep subagent completion agent-run authoritative and surface child-owned background processes as separate active work in `/tasks` and `session_status`, or defer subagent finalization through a generic owned-process liveness seam with timeout, cancel, cleanup, and regression coverage.
