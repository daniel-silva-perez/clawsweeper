## Summary

Keep this PR open. Current main still does not implement the central behavior: rotating the reply session after definitive all-model timeout exhaustion. The PR is a plausible implementation candidate, but it is stale against current runner code and should be tightened before merge because the current diff still relies on broad timeout matching instead of structured all-attempt timeout proof.

## Best Possible Solution

Keep this PR open as a focused implementation candidate. The best path is to rebase it onto current main and implement timeout session rotation at the auto-reply runner/fallback boundary using structured `FallbackSummaryError.attempts` proof that every attempted model failed for `reason: "timeout"`, while preserving current compaction recovery, role-order recovery, transient HTTP retry, heartbeat safeguards, and normal failure text. Add tests for thrown all-timeout summaries, embedded `meta.error`, timeout `isError` payloads, mixed-cause fallback exhaustion, heartbeat turns, and transient HTTP gateway timeouts.
