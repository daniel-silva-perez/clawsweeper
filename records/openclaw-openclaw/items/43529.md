## Summary

Keep open. Current main still lacks timeout-driven session rotation, so the PR targets a real remaining bug, but the proposed patch is stale and still has blocking correctness gaps around mixed fallback failures, payload-only timeout results, and required-callsite updates.

## What This Changes

The PR adds timeout-specific session reset handling to run-reply fallback execution, wires a reset callback from `runReplyAgent`, returns timeout retry text, and adds focused run-reply tests.

## Best Possible Solution

Land a rebased implementation that rotates sessions only after definitive all-model timeout exhaustion on user turns, handles thrown, `meta.error`, and payload-only timeout surfaces, and preserves existing heartbeat, transient HTTP retry, billing, rate-limit, auth, compaction, role-ordering, and external-chat behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Require all fallback attempts to time out before resetting:** `src/auto-reply/reply/agent-runner-execution.ts:590-595`
  - body: `isTimeoutError(err)` runs on the aggregate fallback error text. A mixed fallback chain can contain one timed-out attempt plus auth, billing, rate-limit, or another failure, so this can rotate and discard the active transcript even though all models did not time out. Use `FallbackSummaryError.attempts` and require every relevant attempt to carry timeout evidence before resetting.
  - confidence: 0.91
- **[P2] Handle timeout payloads before treating the run as successful:** `src/auto-reply/reply/agent-runner-execution.ts:478-480`
  - body: The embedded runner can return a timeout as an `isError` payload with no `meta.error`. This branch only checks `runResult.meta?.error`, then the loop breaks as a successful run, so that timeout surface keeps the same bloated session and leaves the retry loop unresolved.
  - confidence: 0.88
- **[P2] Update direct callers for the required reset callback:** `src/auto-reply/reply/agent-runner-execution.ts:91-94`
  - body: Adding `resetSessionAfterTimeout` as a required parameter means existing direct `runAgentTurnWithFallback` test callers no longer satisfy the parameter object. Update the shared helper and explicit call sites with a stub callback after rebasing, or the test typecheck lane will fail.
  - confidence: 0.84
