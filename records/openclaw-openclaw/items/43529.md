## Summary

Keep this PR open, but do not merge the current patch as-is. Current main still lacks timeout-triggered reply-session rotation, and the PR is aimed at a real auto-reply death-spiral bug, but the branch still resets on mixed-cause fallback failures and misses the embedded timeout payload surface that current main can produce.

## What This Changes

The PR adds a timeout reset callback through `runReplyAgent` and `runAgentTurnWithFallback`, returns timeout-specific retry text, and adds focused tests for thrown timeout errors, `meta.error` timeout errors, reset-unavailable fallback text, and heartbeat no-reset behavior.

## Best Possible Solution

Implement timeout session rotation on current main using structured evidence: require definitive all-model timeout exhaustion from `FallbackSummaryError.attempts` or an equivalent embedded timeout classification, cover timeout `isError` payloads without `meta.error`, preserve heartbeat no-reset and transient HTTP retry behavior, and keep billing/rate-limit/provider-auth failures on their existing user-facing paths.
