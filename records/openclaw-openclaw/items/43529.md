## Summary

Keep this PR open. Current main still does not implement the central behavior: rotating the reply session after definitive all-model timeout exhaustion. Main has related timeout/fallback plumbing, but runReplyAgent only wires compaction and role-ordering session resets, and timeout payload/error paths still fall through without resetting the session.

## Best Possible Solution

Keep this PR open as a focused implementation candidate, but require a rebase and tighter fix. The best path is in the auto-reply runner/fallback boundary: detect definitive all-model timeout exhaustion from structured FallbackSummaryError attempts and timeout isError/meta/error surfaces, preserve existing high-context timeout compaction and transient HTTP retry behavior, skip heartbeat turns, rotate through resetReplyRunSession only after that proof, and port coverage into the current agent-runner-execution/runReplyAgent test split.
