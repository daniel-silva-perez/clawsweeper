## Summary

Keep open. Current main still lacks timeout-triggered session rotation, and the linked bug was closed by the stale bot rather than a merged fix. The PR is useful source work, but the patch is not merge-ready because it resets on mixed fallback failures, misses the payload-only timeout surface, and does not update current direct runner test callers.

## What This Changes

The PR adds timeout-specific session reset handling to run-reply fallback execution, wires a reset callback from `runReplyAgent`, returns retry-oriented timeout text, and adds run-reply coverage for timeout reset paths.

## Best Possible Solution

Implement timeout session rotation on current main using structured all-model timeout evidence, cover both `meta.error` and timeout `isError` payloads before the run loop treats the turn as successful, and preserve existing heartbeat, transient HTTP retry, billing, rate-limit, auth, compaction, role-ordering, and external-chat behavior.

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Require all attempts to time out before resetting:** `src/auto-reply/reply/agent-runner-execution.ts:590-595`
  - body: `isTimeoutError(err)` runs on the aggregate fallback error text. A fallback summary can contain one timed-out attempt plus auth, billing, rate-limit, or other failures, so this can rotate and discard the active transcript for a non-timeout exhaustion. Use `FallbackSummaryError.attempts` and require every relevant attempt to have timeout evidence before resetting.
  - confidence: 0.91

  - body: The embedded runner can return a timeout as an `isError` payload with no `meta.error`. This branch only checks `runResult.meta?.error`, then the loop breaks as a successful run, so that timeout surface keeps the same bloated session and leaves the retry loop unresolved.
  - confidence: 0.88

  - body: Adding `resetSessionAfterTimeout` as a required parameter means existing direct `runAgentTurnWithFallback` test callers no longer satisfy the parameter object. Update the shared helper and explicit call sites with a stub callback or the test typecheck lane will fail after rebase.
