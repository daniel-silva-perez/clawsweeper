## Summary

Keep open. Current main has timeout-triggered compaction as a partial mitigation, but it does not implement this PR’s requested all-model timeout session reset; the PR is useful source work, yet it still has blocking correctness gaps around mixed fallback failures, payload-only timeout results, and direct caller updates.

## What This Changes

The PR adds timeout-specific session reset handling to run-reply fallback execution, wires a reset callback from `runReplyAgent`, returns timeout retry copy, and adds focused run-reply tests.

## Best Possible Solution

Land a rebased implementation that rotates the run-reply session only after definitive all-model timeout exhaustion on a user turn, handles thrown, `meta.error`, and payload-only timeout surfaces, and preserves existing compaction, role-ordering, transient HTTP retry, billing, rate-limit, auth, heartbeat, and external-chat behavior.

## Reproduction Assessment

Yes. The remaining patch bugs are reproducible with focused unit cases: a `FallbackSummaryError` containing mixed timeout and non-timeout attempts should not reset, and an embedded result with only an `isError` timeout payload should reset rather than break as success.

## Solution Assessment

No, not as written. The direction is maintainable, but the safest fix should use `FallbackSummaryError.attempts` for all-attempt timeout proof and inspect timeout payloads before accepting a run as successful.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Require all fallback attempts to time out before resetting:** `src/auto-reply/reply/agent-runner-execution.ts:590-595`
  - body: `isTimeoutError(err)` is applied to the aggregate fallback error text. A mixed fallback chain can contain one timed-out attempt plus auth, billing, rate-limit, or another failure, so this can rotate and discard the active transcript even though all models did not time out. Use `FallbackSummaryError.attempts` and require every relevant attempt to carry timeout evidence before resetting.
  - confidence: 0.91
- **[P2] Handle timeout payloads before accepting the run:** `src/auto-reply/reply/agent-runner-execution.ts:478-480`
  - body: The embedded runner can return a timeout as an `isError` payload with no `meta.error`. This branch only checks `runResult.meta?.error`, then the loop breaks as a successful run, so that timeout surface keeps the same bloated session and leaves the retry loop unresolved.
  - confidence: 0.88
- **[P2] Update direct callers for the new reset callback:** `src/auto-reply/reply/agent-runner-execution.ts:91-94`
  - body: Adding `resetSessionAfterTimeout` as a required parameter means existing direct `runAgentTurnWithFallback` test callers no longer satisfy the parameter object. Update the shared helper and explicit call sites with a stub callback after rebasing, or the test typecheck lane will fail.
  - confidence: 0.84
