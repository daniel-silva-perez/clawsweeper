## Summary

Keep open. Current main still resolves plugin text transforms through the indirect active-runtime helper, and that helper only tries stable `./runtime.js`/`./runtime.ts` module names before returning null. The documented registration contract and CLI/provider prompt application paths are still present, and the related fix PR #65331 remains open rather than merged.

## What This Changes

Fix plugin-registered text transforms so they resolve from the active plugin registry in the main gateway process and are applied to CLI and provider prompts.

## Best Possible Solution

Land #65331 or an equivalent narrow change that makes the bundled main process read the active plugin registry directly while preserving the existing external-plugin fallback, then add targeted regression coverage for CLI and provider prompt transform application.
