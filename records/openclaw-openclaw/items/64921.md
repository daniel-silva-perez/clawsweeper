## Summary

Keep open. Current main still routes plugin text transform resolution through the createRequire-based active runtime helper, so the main-process-safe direct active registry lookup requested here has not landed. The plugin registration/docs contract and prompt application paths are present, and the related fix PR #65331 remains open and unmerged.

## What This Changes

Plugin-registered textTransforms via api.registerTextTransforms should be applied to CLI/provider system prompts and agent prompts in the main gateway process instead of resolving to an empty runtime transform set.

## Best Possible Solution

Update the current runtime text transform resolver to use the active plugin registry directly in the bundled main process while preserving the existing fallback for external plugin contexts, then add focused regression coverage that proves a registered transform reaches CLI and provider prompt application.
