## Summary

Keep this PR open. Current main supports global `session.dmScope` and newer route-binding `bindings[].session.dmScope`, but it still does not accept or apply the requested per-agent `agents.list[].dmScope` default. The PR remains a focused implementation candidate, but it needs a rebase around the binding-level override surface with explicit precedence.

## Best Possible Solution

Keep this PR open and request a focused rebase/rework. The best implementation would add `AgentConfig.dmScope`, strict zod validation, schema/help/label/generated docs updates, and selected-agent lookup in route resolution while preserving current binding-level overrides with precedence `binding > agent > global`. Add regression tests for per-agent override, global fallback, binding interaction, group behavior, and route-cache correctness.
