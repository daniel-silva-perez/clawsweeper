## Summary

Keep this PR open. Current main supports global `session.dmScope` and route-level `bindings[].session.dmScope`, but it still does not accept or apply the requested `agents.list[].dmScope` per-agent default. The PR remains a focused implementation candidate, with a rebase needed around the newer binding-level override surface.

## Best Possible Solution

Keep this PR open and request a focused rebase. The best implementation should add `AgentConfig.dmScope`, strict zod validation, labels/help/generated schema and docs updates, and selected-agent lookup in `resolveAgentRoute` while preserving `matched.binding.session?.dmScope ?? agent.dmScope ?? cfg.session?.dmScope ?? "main"`. Add regression tests for per-agent override, global fallback, binding-vs-agent precedence, group behavior, and route-cache invalidation.

## Risks / Open Questions

- A naive rebase could regress the newer `bindings[].session.dmScope` behavior; precedence should be locked as binding > agent > global.
- Per-agent `dmScope: "main"` can intentionally collapse multiple DMs into one shared agent session, so docs/help should call out the privacy and isolation tradeoff.
- The current PR test only covers the old per-agent override/fallback path; current main also needs tests for binding interaction, group behavior, and route-cache correctness.
