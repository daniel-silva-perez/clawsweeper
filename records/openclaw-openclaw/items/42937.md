## Summary

Keep this PR open. Current main still supports global `session.dmScope` and route-binding `bindings[].session.dmScope`, but it does not accept or apply the requested `agents.list[].dmScope` per-agent default. The PR remains a plausible core routing/config change and should be rebased against the current binding-level override surface with an explicit precedence rule.

## Best Possible Solution

Keep this PR open and request a focused rebase/rework. The best implementation would add `AgentConfig.dmScope`, strict zod validation, generated schema/help/label/public docs updates, and selected-agent lookup in route resolution while preserving binding-level overrides with explicit precedence: binding > agent > global. Add tests for per-agent override, global fallback, binding interaction, group behavior, and route-cache correctness.

## Risks / Open Questions

- The PR predates current `bindings[].session.dmScope`; a rebase needs an explicit precedence rule, likely binding override > agent override > global default.
- Per-agent `dmScope: "main"` can intentionally collapse DMs for one agent and reduce isolation, so user-facing docs/help and regression tests should call out that security/privacy tradeoff.
- A complete implementation should update generated schema/config metadata, not only handwritten zod/help/label files.
