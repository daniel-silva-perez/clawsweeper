## Summary

Keep this PR open. Current main still supports global `session.dmScope` and route-binding `bindings[].session.dmScope`, but it does not accept or apply the requested per-agent `agents.list[].dmScope` default. The PR is not implemented on main and remains a focused implementation candidate, though it needs a rebase around the newer binding-level override surface.

## Best Possible Solution

Keep this PR open and request a focused rebase. The best implementation should add `AgentConfig.dmScope`, strict zod validation, labels/help/generated schema/doc updates, and selected-agent lookup in route resolution while preserving current binding-level overrides with precedence `binding > agent > global`. Add regression tests for per-agent override, global fallback, binding interaction, group behavior, and route-cache correctness.
