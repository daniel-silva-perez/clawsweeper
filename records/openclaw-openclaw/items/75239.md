## Summary

Keep open. The report targets the latest release that already contains the earlier WebChat duplicate-message mitigations, and current main has no newer relevant Control UI change proving this post-final duplicate path is fixed. Source still allows a `session.message` arriving after a renderable `chat.final` to trigger `chat.history`, while the regression tests cover the opposite ordering more directly than the reported final/session-message/sessions.changed race.

## What This Changes

The issue reports that Control UI WebChat still renders every assistant reply twice on v2026.4.27 after earlier duplicate-message fixes.

## Best Possible Solution

Control UI WebChat should make terminal chat-event, `session.message`, and `sessions.changed` reconciliation idempotent for normal assistant finals, while preserving required history refreshes for silent finals, tool output, errors, aborts, and genuinely external transcript updates.
