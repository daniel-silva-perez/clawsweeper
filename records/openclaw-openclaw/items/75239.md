Keep open. The report targets a release that already contained the earlier WebChat duplicate-message mitigations, and current main does not show a newer Control UI reconciliation change proving this post-final event-ordering path is fixed.

## What This Changes

The issue asks OpenClaw to fix a Control UI WebChat regression where assistant replies still render as duplicate bubbles in 2026.4.27 after the earlier duplicate-message fixes.

## Best Possible Solution

Make Control UI WebChat reconciliation idempotent across chat.final, session.message, and sessions.changed ordering so a normal assistant final cannot produce a second visible bubble, while preserving required history refreshes for silent finals, tool output, errors, aborts, and external transcript updates.

## Solution Assessment
