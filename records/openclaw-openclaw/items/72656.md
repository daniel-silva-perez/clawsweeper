## Summary

Keep open. Current main has a shipped WhatsApp-internal transport watchdog, but it still does not publish `lastTransportActivityAt` through the WhatsApp channel status snapshot, and the generic gateway health policy skips stale-socket detection without that field. This PR remains a plausible implementation path, but it overlaps current main and needs maintainer review/rebase rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase it onto current main, preserve the narrow missing status-snapshot propagation if maintainers want WhatsApp to participate in generic `stale-socket` health detection, avoid duplicating the shipped internal watchdog work from v2026.4.26, and validate against the linked #67986 behavior plus existing WhatsApp watchdog tests.
