## Summary

Keep open. The PR targets a real redaction bug linked from #42982 and fixes the earlier Greptile test-quality note, but the branch is still narrower than current main's redaction surface: config-derived custom patterns can still replace built-in secret patterns in log tailing and persisted transcript redaction.

## What This Changes

The PR adds a default-plus-custom redaction pattern helper, uses it in config-driven `redactSensitiveText` and `redactToolDetail` paths, and adds tests for additive config behavior plus explicit caller-supplied pattern behavior.

## Best Possible Solution

Define one config-derived redaction contract and put the default-plus-custom merge in the shared resolver/helper used by console/file/OTLP logs, log tailing, tool details, and persisted transcript text. Keep explicit caller-supplied pattern semantics only where intentionally documented, then update docs and regression tests for every affected sink.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

  - body: The patch merges defaults only in the no-options `redactSensitiveText()` path and `redactToolDetail()`.
