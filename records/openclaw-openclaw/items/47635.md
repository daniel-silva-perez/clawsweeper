## Summary

Keep this PR open. Current main still does not implement the PR’s intended proactive session archive sweep: gateway startup only cleans stale session lock files, maintenance timers only cover tick/health/dedupe/media cleanup, and archived transcript cleanup remains tied to session-store maintenance/write paths. The PR is a focused implementation candidate for #44820, but it still needs maintainer review because the discussion identifies a remaining custom `session.store` coverage gap.

## Best Possible Solution

Keep the PR open for maintainer review or contributor updates. The best path is to land an equivalent proactive cleanup that runs at startup and on a timer, reuses the all-agent session-store discovery path so custom/templated `session.store` roots are covered, cleans `.deleted`/`.reset` archives according to retention, removes stale orphan `writeTextAtomic` `.tmp` files, bounds `.bak` artifacts, and includes focused regression tests. Close it only after equivalent behavior ships on main or a canonical replacement PR is chosen.
