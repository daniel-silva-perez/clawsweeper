## Summary

Keep open. Current main still parses `OPENCLAW_RAW_STREAM=0` and unset as disabled, but the LaunchAgent-managed disable path described by the reporter is still not resolved: raw-stream writes depend on the effective Gateway process environment, while the normal macOS LaunchAgent restart path still succeeds via `launchctl kickstart -k` without refreshing generated service-env wrapper metadata. The docs also still describe how to enable raw-stream logging, not supported disable values or LaunchAgent/service-env precedence.

## Best Possible Solution

Keep this open for a narrow macOS service-env regression fix and docs update: document accepted truthy and disabled values for `OPENCLAW_RAW_STREAM`; clarify precedence among `launchctl` inherited env, state-dir `.env`/config durable service env, and generated LaunchAgent env wrappers; and make `openclaw gateway restart`, doctor, or install refresh stale raw-stream service metadata or print an explicit repair/reinstall instruction when refresh is required.
