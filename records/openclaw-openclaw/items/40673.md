## Summary

Keep open. Current main still has correct raw-stream boolean parsing, but it does not resolve the LaunchAgent-managed disable path described here: raw-stream writes are gated by the effective Gateway process environment, while normal LaunchAgent restart still uses kickstart and does not refresh generated service-env metadata. The public docs still only describe enabling raw-stream logging, not disable values or LaunchAgent/service-env precedence. Related #18740/#18753 were about enabling raw-stream flags and do not cover this disable/restart behavior.

## Best Possible Solution

Keep this issue open for a narrow code/docs regression pass: document accepted truthy and disabled values for OPENCLAW_RAW_STREAM; explain precedence between launchctl inherited env, state-dir .env/config durable service env, and generated LaunchAgent service-env wrappers; and make gateway restart, doctor, or install refresh stale raw-stream service metadata or print a clear reinstall/repair instruction when that is required.
