## Summary

Keep open. Current main still parses unset and `OPENCLAW_RAW_STREAM=0` as disabled, but the reported LaunchAgent-managed disable path is not resolved: raw-stream writes depend on the effective Gateway process environment, LaunchAgent restarts still use the normal `kickstart -k` path without refreshing service-env wrapper metadata, and the docs still only document enabling raw-stream logging.

## Best Possible Solution

Keep this open for a narrow macOS service-env fix and docs update: document accepted truthy and disabled values for `OPENCLAW_RAW_STREAM`; clarify precedence among `launchctl` inherited env, state-dir `.env` / config durable service env, and generated LaunchAgent env wrappers; make `openclaw gateway restart`, doctor, or install refresh stale raw-stream service metadata or print an explicit repair/reinstall instruction; and add a regression test for the LaunchAgent disable path.

## Risks / Open Questions

- Raw-stream logs can contain prompts, assistant output, tool output, and other private debugging data, so an ineffective disable path is a privacy and operator-trust risk.
- There is still no focused regression test for disabling raw-stream through LaunchAgent service-env metadata followed by a managed restart.
- Docs ambiguity can leave operators believing `launchctl unsetenv` or `OPENCLAW_RAW_STREAM=0` is sufficient when a durable service env source or wrapper still enables logging.
