## Summary

Keep open. Current main has the generic raw-stream truthy parser and write guard, so `OPENCLAW_RAW_STREAM=0` disables logging only if that is the effective value inside the running Gateway process. The LaunchAgent-managed stale wrapper/durable env path and the raw-stream disable docs gap are still unresolved on current main.

## Best Possible Solution

Keep this issue open. The best fix is a narrow docs plus regression pass: document exact enabled and disabled values for `OPENCLAW_RAW_STREAM`, explain LaunchAgent effective-env precedence including launchctl env, durable `.env`/config env, and generated service-env wrappers, add tests for disabled values and LaunchAgent restart/install refresh behavior, and update status/doctor/restart output if stale wrapper state can still make operators think raw-stream logging is off when the process still sees it as enabled.
