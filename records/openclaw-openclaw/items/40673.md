## Summary

Keep open. Current main still has the raw-stream parser and write guard, but the LaunchAgent-managed disable path and documentation gap remain unresolved. A normal LaunchAgent restart still kickstarts the loaded job without rebuilding the generated service-env wrapper, and the public docs still describe only enabling raw-stream logging.

## Best Possible Solution

Keep this issue open. The best fix is a narrow code, docs, and regression pass: document exact truthy and disabled values for OPENCLAW_RAW_STREAM; explain LaunchAgent precedence between launchctl inherited env, state-dir .env/config durable service env, and generated service-env wrappers; and add LaunchAgent coverage showing that unset or 0 plus openclaw gateway restart either stops raw-stream appends by refreshing/removing stale service metadata or surfaces a clear repair instruction such as reinstall/doctor when stale wrapper state is still active.

## Risks / Open Questions

- Raw-stream logs may contain prompts, assistant output, and tool output, so an operator-visible disable path that still appends is a privacy and trust risk.
- The exact launchd effective-environment behavior still needs a macOS regression test, but current source is enough to show that normal restart does not clear stale generated service-env metadata.
- Documentation currently leaves operators without a clear disable path when launchctl env, state-dir .env, and generated service-env wrappers disagree.
