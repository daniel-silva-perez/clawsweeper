## Summary

Keep open. Current main still has the process-level raw-stream guard and falsey parser, but the LaunchAgent-managed disable path and docs gap remain unresolved. Normal LaunchAgent restart still kickstarts the existing job without refreshing service-env wrapper metadata, and the public docs still describe enabling raw-stream logging without documenting disabled values or LaunchAgent precedence.

## Best Possible Solution

Keep this issue open. The best fix is a narrow docs plus regression pass: document exact enabled and disabled values for `OPENCLAW_RAW_STREAM`, explain LaunchAgent effective-env precedence across `launchctl`, durable state `.env`, and generated service-env wrappers, and add coverage that verifies `launchctl unsetenv` or `OPENCLAW_RAW_STREAM=0` plus `openclaw gateway restart` cannot leave `raw-stream.jsonl` appending from stale service metadata. If stale wrapper state is the mechanism, restart/install/status/doctor should refresh it or surface it clearly.
