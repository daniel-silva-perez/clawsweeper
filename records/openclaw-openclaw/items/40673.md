## Summary

Keep open. Current main handles `OPENCLAW_RAW_STREAM=0` correctly only when that is the effective value inside the running gateway process, but the LaunchAgent-managed disable path and docs gap reported here are still not resolved. The raw-stream docs still only describe enabling, and the normal LaunchAgent restart path still kickstarts the loaded job without rewriting the generated service-env wrapper.

## Best Possible Solution

Keep this issue open. The best fix is a narrow docs plus regression pass: document exact enabled and disabled values for `OPENCLAW_RAW_STREAM`, explain effective env precedence for LaunchAgent-managed gateways, add tests for `appendRawStream` disabled values and LaunchAgent install/restart service-env behavior, and, if live macOS verification confirms stale wrapper or durable env confusion, update status/repair/restart output so operators can identify and clear the source.
