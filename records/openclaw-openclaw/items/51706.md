## Summary

Keep open. Current main already shipped the narrow runtime-model display fix, but this PR’s latest branch is not safely redundant because it also tries to preserve fallback visibility and that remaining behavior is not correctly handled by the branch. The patch is stale against the current status module split and contains a P2 bug that forces fallback sessions back to the global configured default instead of the stored selected fallback model.

## What This Changes

This PR changes `session_status` and status rendering tests so persisted runtime session models are shown when no explicit session override exists, with added fallback visibility handling when runtime and selected models differ.

## Best Possible Solution

Land a focused current-main replacement that carries separate selected-model and display-runtime identities through the `src/status` path, preserves unknown-provider display without inventing provider refs, and keeps active fallback visibility correct for explicit overrides plus channel/group-selected fallback sessions.
