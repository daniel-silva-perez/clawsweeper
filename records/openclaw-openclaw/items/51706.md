## Summary

Keep this PR open. Current main shipped the original runtime-model display fix via #55425, but it still feeds the persisted runtime model through the selected-model path in `session_status`. That can suppress active fallback output when no explicit session override exists. The PR is also stale against the current `src/status` split and needs a narrower rework rather than closure as already implemented.

## Best Possible Solution

Keep this PR open or replace it with a focused follow-up. Preserve #55425 for ordinary runtime-model display, add a display-only runtime model/auth path through `src/status/status-text.types.ts`, `src/status/status-text.ts`, and `src/status/status-message.ts`, keep selected-model semantics sourced from explicit overrides, channel/group selection, or persisted `fallbackNoticeSelectedModel`, and add regression coverage for runtime-only display, unknown-provider preservation, active fallback visibility, and channel/group-selected fallback sessions.
