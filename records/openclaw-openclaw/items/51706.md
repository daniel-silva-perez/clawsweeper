## Summary

Keep open. The original runtime-model symptom has already shipped on main via #55425, but this PR now also attempts fallback-visibility/display-runtime behavior, and the proposed patch is stale against the current `src/status` split and still mishandles channel/group fallback sessions by substituting the global default instead of the stored selected fallback ref.

## What This Changes

The PR updates `session_status` and status rendering tests so a persisted runtime session model can appear in the status card when no explicit session override exists, with an added path intended to keep active fallback details visible.

## Best Possible Solution

Land a focused current-main replacement that keeps ordinary runtime model display, adds a clear display-only runtime identity path where needed, preserves selected-model semantics for fallback matching, and resolves fallback selected refs from persisted fallback notice state rather than the global default.
