## Summary

Keep this PR open. Current main shipped the original runtime-model display fix through #55425, but it has not implemented the PR’s later display-only fallback preservation idea: `session_status` still feeds the persisted runtime model through the single selected-model path, while fallback rendering depends on selected-vs-active model mismatch. The branch is also stale against the current `src/status` module split and its latest fallback-preservation approach is still called out in review as wrong for channel/group-selected sessions.

## What This Changes

This PR changes `session_status` and status rendering tests so the status card uses a persisted runtime session model when no explicit session override exists, then adds a follow-up attempt to keep active fallback details visible when runtime and selected models differ.

## Best Possible Solution

Land a focused current-main replacement that keeps ordinary runtime model display, separates display-only runtime identity from selected-model semantics in the `src/status` APIs, preserves unknown-provider display without fabricating provider refs, and keeps active fallback visibility correct for explicit overrides plus channel/group-selected fallback sessions.
