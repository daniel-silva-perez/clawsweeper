## Summary

Keep this issue open. The current main branch does not contain a hardcoded `wangtao` path, but the useful root-cause from the discussion still appears valid: default onboarding/setup paths are resolved to an absolute home path and then persisted into config, so a migrated or synced config can still expose a host-specific `/Users/<name>/.openclaw/workspace` value. The related #56810 PR directly targeted this but is closed and unmerged in the provided GitHub context.

## Best Possible Solution

Land a focused fix like the closed #56810 proposal: when setup/onboard/configure use the canonical default workspace, persist a portable `~/.openclaw/workspace` value, while continuing to resolve to an absolute path at runtime and preserving explicit custom absolute workspace choices. Add regression coverage for classic onboarding, non-interactive onboarding, setup, and configure workspace persistence.
