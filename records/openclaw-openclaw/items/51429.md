## Summary

Keep this open. Current main does not appear to contain a hardcoded `wangtao` path, but the useful root cause from the thread remains actionable: default workspace values are resolved to host-absolute paths and then persisted into config by onboarding/setup/configure paths, which can make migrated or synced configs expose another host's `/Users/<name>/.openclaw/workspace`. The related #56810 PR targeted this exact issue but is closed and unmerged.

## Best Possible Solution

Keep the issue open and land a focused fix like #56810: when onboarding/setup/configure/migration flows are persisting the canonical default workspace, save a portable `~/.openclaw/workspace` value or omit the default, while continuing to resolve to absolute paths at runtime and preserving explicit custom absolute workspace values. Add regression coverage for classic onboarding, non-interactive onboarding, setup, configure, and profile-aware defaults.
