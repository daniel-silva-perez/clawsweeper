## Summary

Keep this PR open. Current main still lacks the Telegram direct `/status` fast path and the suppressed-source fallback contract that this PR adds, so the PR remains a live implementation candidate rather than conservative cleanup. The existing ClawSweeper and Greptile comments line up with the current source: the main review work is functional/privacy review, not closing as obsolete.

## Best Possible Solution

Keep this PR open and review/land it, or land an equivalent Telegram plugin plus core reply-dispatch fix. The fix should route native Telegram `/status` through the direct status helper, keep `/fast status` on the generic route, preserve topic reply threading, pass group `sourceReplyDeliveryMode` for normal and native skill turns, and add tested suppressed-final/block/progress fallbacks without bypassing sendPolicy-denied suppression.
