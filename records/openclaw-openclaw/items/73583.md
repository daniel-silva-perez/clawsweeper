## Summary

Keep this PR open. A prior ClawSweeper comment already identified the same gap, and rechecking current main at 373e7fc242338aa3643285de2bb2de293585a402 still shows Telegram native `/status` routed through the normal buffered reply dispatcher rather than the direct status helper proposed by this PR.

## Best Possible Solution

Review and land this PR or an equivalent Telegram-plugin fix. The implementation should add a `/status`-specific branch in `extensions/telegram/src/bot-native-commands.ts` that resolves the status reply with `resolveDirectStatusReplyForSession` and sends it through `deliverReplies` directly, while preserving the existing auth/session/threading semantics and adding regression coverage that proves `/status` bypasses `dispatchReplyWithBufferedBlockDispatcher`.
