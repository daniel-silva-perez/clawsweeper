## Summary

Keep this PR open. Current main still lacks the two Telegram behavior changes this PR is trying to add: native Telegram `/status` still goes through the buffered reply dispatcher, and Telegram group/native command dispatch still does not pass the resolved group `sourceReplyDeliveryMode` into reply options. The supplied PR discussion and bot reviews support this as an active implementation candidate, not obsolete cleanup.

## Best Possible Solution

Review and land this PR or an equivalent Telegram plugin fix. The implementation should special-case native Telegram `/status` after existing auth/session resolution, use `resolveDirectStatusReplyForSession`, deliver directly while preserving topic threading and reply-to behavior, keep `/fast status` on the generic native-command route, and pass the resolved Telegram group `sourceReplyDeliveryMode` into both normal message turns and native skill-command turns with regression coverage.
