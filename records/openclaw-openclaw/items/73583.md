## Summary

Keep this PR open. Current main still lacks the PR's central Telegram fixes: native Telegram `/status` still goes through the buffered reply dispatcher, Telegram reply paths do not pass the resolved group source delivery mode or a suppressed-source fallback, and the core reply option type has no fallback callback for suppressed group finals. The PR is a focused implementation candidate with reviewer feedback to resolve, not conservative cleanup material.

## Best Possible Solution

Review and land this PR or an equivalent Telegram plugin fix after resolving the reviewer note. The implementation should route native Telegram `/status` through `resolveDirectStatusReplyForSession`, preserve topic/reply threading, keep `/fast status` on the generic native-command route, pass resolved `sourceReplyDeliveryMode` for regular and native Telegram turns, and add tested suppressed-final/block/progress fallbacks that do not weaken sendPolicy-denied delivery.
