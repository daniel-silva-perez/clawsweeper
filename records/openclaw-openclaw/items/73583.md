## Summary

Keep this PR open. Current main has the later #73672 native-command visibility fix, but it still does not implement this PR's Telegram direct `/status` fast path or the suppressed-source fallback contract for regular Telegram group/topic turns.

## Best Possible Solution

Keep the PR open and review/land it or an equivalent smaller patch. Rebase on current main, drop the native-command visibility portion already covered by `195f704c74a2`/#73672, keep the Telegram direct `/status` fast path and suppressed-source fallback only if maintainers accept the visibility tradeoff, and address the `senderIsOwner` review point before merge.

## Risks / Open Questions

- The proposed suppressed-source fallback changes the group privacy boundary by making some final, block-streamed, or verbose progress output visible in Telegram after `message_tool_only` suppression; maintainers should explicitly accept that product/security tradeoff and verify `sendPolicy: deny` remains a hard stop.
- The Greptile review concern remains relevant to the PR head: hardcoding `senderIsOwner: false` in the proposed Telegram direct status path has low current impact, but could under-report owner-only status details if the shared status contract grows.
- The PR overlaps with #73672, which already landed native-command visibility, so the best path is a rebase/narrowing to avoid duplicating or drifting from current main semantics.
