## Summary

Keep this PR open. Current main still lacks the direct Telegram `/status` fast path and the suppressed-source fallback hook that this PR adds, so the PR is a live implementation candidate rather than cleanup. The final PR diff is scoped to Telegram runtime/tests and core reply dispatch/types/tests; the main review work is functional/privacy review, not supply-chain cleanup.

## Best Possible Solution

Review and land this PR, or an equivalent Telegram plugin/core reply-dispatch fix, after resolving the owner-status reviewer note and confirming the fallback privacy behavior. The fix should keep `/status` on the direct status helper, preserve Telegram topic reply threading, keep `/fast status` on the generic native-command route, and add tested suppressed-final/block/progress fallback delivery without bypassing sendPolicy-denied suppression.
