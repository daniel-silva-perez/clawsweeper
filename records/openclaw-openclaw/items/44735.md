## Summary

Keep this PR open. Current main still auto-selects the bundled Feishu channel unless an external owner declares `preferOver` or Feishu is explicitly disabled, so the PR’s enabled-alternative scenario is not implemented. The proposed PR also still needs changes because its current guard can treat `plugins.entries.openclaw-lark.enabled=true` as active even when a restrictive `plugins.allow` would make that plugin ineffective.

## Best Possible Solution

Keep this PR open for maintainer review. Either adapt it to the current split auto-enable files with an effective-owner check that honors `plugins.enabled`, allow, deny, and explicit disable, or make the replacement explicit through the external plugin manifest with `channelConfigs.feishu.preferOver: ["feishu"]` and test that path. The fix should include regression coverage for the `openclaw-lark` Feishu scenario and a restrictive `plugins.allow` case.
