## Summary

Keep this PR open. Current main has the manual `openclaw config schema` exporter, but it does not implement the PR’s intended startup/update behavior: no local `openclaw_schema.json` maintenance module, no CLI startup hook, no config-write hook that defaults `$schema`, and no `update.maintainConfigJsonSchema` config key. The PR is also paired with the same author’s still-open feature issue #55235, so it should remain available for review rather than being closed independently.

## Best Possible Solution

Keep the PR open and review it as the implementation candidate for #55235. The best path is to rebase it on current main, preserve `$include` and authored config structure when defaulting `$schema`, keep help/version and read-only command paths side-effect-free, retain the camelCase `update.maintainConfigJsonSchema` opt-out, and land it with focused config-write/startup tests plus docs.
