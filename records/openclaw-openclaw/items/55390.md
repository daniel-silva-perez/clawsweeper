## Summary

Keep open. Current main still only exposes the config schema through `openclaw config schema`, so the PR is not obsolete, but the proposed startup backfill still has a blocking config-preservation bug for include-authored configs and should not merge as-is.

## What This Changes

The PR adds best-effort local `openclaw_schema.json` generation, `$schema` backfilling on CLI startup/config writes, an `update.maintainConfigJsonSchema` opt-out, docs, and targeted tests.

## Best Possible Solution

Rebase the feature onto current main and make `$schema` backfill use an authored-root-aware config mutation path, or skip root-file mutation when the authored config uses `$include`; keep local schema file writes best-effort, preserve custom `$schema` values, and add include-owned regression coverage before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve include-owned configs during schema backfill:** `src/config/local-json-schema.ts:98`
  - body: `maintainLocalConfigJsonSchemaArtifacts` writes `nextConfig` derived from `snapshot.config` during normal CLI startup when `$schema` is missing. On this branch, `writeConfigFile` persists by applying the patch to `snapshot.resolved`, so a root `$include` can be replaced with flattened resolved content when a read-style command only backfills `$schema`. Use an authored-root-aware mutation path or skip root backfill for include-owned roots.
  - confidence: 0.87
