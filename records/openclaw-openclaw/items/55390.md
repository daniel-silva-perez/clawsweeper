## Summary

Keep open. Current main still only provides manual `openclaw config schema` export and does not maintain a local schema artifact, so the PR is not obsolete. The branch should not merge as-is because its startup `$schema` backfill writes from the resolved config snapshot and can flatten `$include`-owned authored config during normal CLI runs.

## What This Changes

The PR adds best-effort local `openclaw_schema.json` maintenance during CLI startup/config writes, defaults missing `$schema` to that local file, exposes `update.maintainConfigJsonSchema`, updates config/update docs, and adds tests for schema export and maintenance behavior.

## Best Possible Solution

Rebase the feature onto current main and implement `$schema` backfill through an authored-config-aware mutation path, or skip root-file mutation when the root config delegates through `$include`; keep local schema file writes best-effort, preserve custom `$schema` values, and add targeted coverage for invalid configs, opt-out, no-config, JSON output, and include-owned configs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P1] Preserve include directives during schema backfill:** `src/config/local-json-schema.ts:98`
  - body: This startup path calls `writeConfigFile` with `nextConfig` derived from `snapshot.config`, which is the resolved/runtime-shaped config. For root configs that use `$include`, a normal CLI run that only needs to add `$schema` can persist flattened included content into `openclaw.json` and drop the user's authored include structure. Use an authored-config-aware mutation path or skip root `$schema` mutation when the root file delegates config through `$include`.
  - confidence: 0.86
