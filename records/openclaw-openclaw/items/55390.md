## Summary

Keep open. Current main only exposes `openclaw config schema` on demand; it does not maintain a local `openclaw_schema.json`, add `update.maintainConfigJsonSchema`, or backfill `$schema` on CLI startup. The PR is still relevant to the linked feature request, but it is not merge-ready because its startup maintenance path can rewrite include-based configs through the resolved snapshot path.

## What This Changes

The PR adds local `openclaw_schema.json` maintenance beside `openclaw.json`, defaults missing `$schema` values to that file, adds the `update.maintainConfigJsonSchema` schema/docs surface, and covers the new CLI startup/write behavior with tests.

## Best Possible Solution

Rebase the feature onto current main and implement the `$schema` backfill through an include-preserving config mutation path, or skip root-file mutation when the authored config uses `$include`; keep the local schema file write best-effort, preserve custom `$schema` values, and add targeted coverage for invalid configs, opt-out, no-config, JSON output, and include-owned configs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P1] Preserve include-owned configs during schema backfill:** `src/config/local-json-schema.ts:98`
  - body: This startup path writes a config derived from `snapshot.config`, which is the post-include/runtime-shaped snapshot. On this branch `writeConfigFile` persists by applying the patch to the resolved config, so an `openclaw.json` with a root `$include` can be replaced by the flattened resolved content when a normal CLI run only needs to add `$schema`. Use an authored-config-aware mutation path or skip the backfill for include-owned roots.
  - confidence: 0.86
