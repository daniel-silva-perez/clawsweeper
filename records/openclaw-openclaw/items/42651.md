## Summary

Keep open. Current main has memory search, short-term promotion, dreaming ingestion, and adjacent LanceDB/wiki memory plugins; the discussion also points to Engram Core as a useful external implementation. It still does not provide the requested default MVP operator surface: `remember`/`recall`/`review`/`edit`/`forget`, structured recall match explanations, or first-class review/edit/forget lifecycle semantics. The related #42650 remains open for lifecycle flows, so this should stay as product/API follow-up rather than be closed as implemented or delegated entirely to ClawHub.

## Best Possible Solution

Keep this open and narrow the v1 memory operator contract. A maintainer should decide whether the MVP belongs in default `memory-core`, a bundled memory plugin/skill, or a ClawHub direction. If it stays in the default memory path, implement `remember`/`recall`/`review`/`edit`/`forget` around provenance-preserving writes, structured per-result match explanations, and non-destructive lifecycle states coordinated with #42650. Engram Core is useful prior art for transcript ingestion and search, but it should inform rather than replace the OpenClaw memory-core contract decision.
