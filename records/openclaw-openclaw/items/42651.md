## Summary

Keep open. Current main still has substantial memory infrastructure, but it does not provide the requested default MVP operator surface: first-class remember/recall/review/edit/forget commands, structured recall match explanations, or default review/edit/forget lifecycle handling. Adjacent LanceDB and memory-wiki plugin surfaces are useful context, and Engram Core is a relevant external implementation, but they do not close the default memory-core product/API question.

## Best Possible Solution

Keep this open and narrow the v1 memory operator contract. A maintainer should decide whether the MVP belongs in default memory-core commands, a bundled memory plugin/skill, or an explicit ClawHub direction. If it stays in the default memory path, implement remember/recall/review/edit/forget around provenance-preserving writes, structured per-result match explanations, and review/edit/forget lifecycle states coordinated with #42650.
