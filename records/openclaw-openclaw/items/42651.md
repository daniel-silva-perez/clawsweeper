## Summary

Keep this issue open. Current main has adjacent memory work, including memory-core search/promotion/dreaming ingestion, memory-wiki, and optional LanceDB memory tools, but the requested MVP operator surface is still not present as a default remember/recall/review/edit/forget CLI or skill/tool lifecycle with provenance-preserving writes, first-class review/edit/forget, and structured recall match explanations. The linked Engram Core project is useful prior art for external transcript ingestion and search, but it does not replace the OpenClaw memory-core/product contract decision, and #42650 remains open for the dependent lifecycle semantics.

## Best Possible Solution

Keep this open and narrow the v1 memory operator contract. A maintainer should decide whether the default MVP belongs in memory-core, a bundled memory plugin/skill, or a plugin API extension, then implement remember/recall/review/edit/forget around provenance-preserving writes, structured per-result match explanations, and non-destructive review/edit/forget states coordinated with #42650. Engram Core and memory-lancedb are useful prior art, but they should inform the contract rather than replace it.
