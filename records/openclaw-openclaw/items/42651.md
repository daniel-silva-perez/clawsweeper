## Summary

Keep this issue open. Current main has substantial adjacent memory work, including memory search, promotion, dreaming ingestion, memory-wiki, and optional LanceDB memory tools, but it still does not provide the requested default MVP operator surface: remember/recall/review/edit/forget commands, first-class review/edit/forget lifecycle, or structured recall match explanations. The related #42650 is still open and tracks core lifecycle semantics that this issue depends on.

## Best Possible Solution

Keep this open and narrow the v1 memory operator contract. A maintainer should decide whether the MVP belongs in default memory-core, a bundled memory plugin/skill, or a ClawHub direction. If it stays in the default memory path, implement remember/recall/review/edit/forget around provenance-preserving writes, structured per-result match explanations, and non-destructive lifecycle states coordinated with #42650. Engram Core is useful prior art for transcript ingestion and search, but it should inform rather than replace the OpenClaw memory-core contract decision.
