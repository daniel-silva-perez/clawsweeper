## Summary

Keep open. Current main has substantial adjacent memory work, but the default memory operator surface still does not provide the requested remember/recall/review/edit/forget command or skill lifecycle, and the remaining work depends on unresolved write/review semantics tracked in related memory MVP issues.

## What This Changes

Request to expose memory through ingestion helpers and small operator-facing remember, recall, review, edit, and forget command or skill flows with provenance and match explanations.

## Best Possible Solution

Keep this open while maintainers narrow the v1 memory operator contract. The durable end state should define one default path for remember/recall/review/edit/forget, map it to provenance-preserving writes and retrieval, include structured match explanations, and coordinate non-destructive review/edit/forget semantics with the open write-path and lifecycle issues.

## Reproduction Assessment

Yes for the missing-surface claim: static inspection of current main shows the default CLI and tool registrations do not include the requested verbs, while docs and tests cover adjacent search, promotion, dreaming, backfill, and optional plugin behavior. No live runtime reproduction is needed for this feature/MVP gap.

## Solution Assessment

No current implementation is not yet the best solution because adjacent memory features do not define the requested first-class operator lifecycle. A safer path is maintainer contract design first, then a focused implementation in the chosen default memory surface with tests for provenance, explanations, and non-destructive lifecycle transitions.
