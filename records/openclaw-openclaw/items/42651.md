## Summary

Keep open. Current main has substantial memory infrastructure, but it still does not implement the requested default MVP operator surface: first-class remember/recall/review/edit/forget commands, review/edit/forget lifecycle handling, or structured per-result why-it-matched output. The Engram Core comment and bundled memory-lancedb/memory-wiki work are useful adjacent paths, but they do not close the default OpenClaw memory-core request. Same-author #42650 remains open for overlapping review/edit/forget semantics.

## Best Possible Solution

Keep this open and narrow the v1 memory operator contract. A maintainer should decide whether the MVP lands as default memory-core subcommands, a bundled skill/plugin, or an explicit ClawHub/plugin direction. If it stays in the default memory path, implement remember/recall/review/edit/forget around provenance-preserving writes, structured per-result match explanations, and review/edit/forget lifecycle states coordinated with #42650.
