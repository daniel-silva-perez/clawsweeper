## Summary

Keep open. Current main has useful session-management pieces, including a Sessions view for listing, labeling, deleting, and navigating to stored sessions, plus a backend `sessions.create` RPC. The specific WebChat/Control UI request is not complete because the Chat plus button still sends `/new`, which resets the current session instead of creating a preserved, switchable new dashboard session.

## Best Possible Solution

Keep this issue open. The clean path is to make the Control UI plus button create a new dashboard session through `sessions.create`, switch Chat to the returned key, preserve the previous session in `sessions.list`, and keep `/new`/`sessions.reset` as explicit reset behavior. Add focused tests around `ui/src/ui/app-render.ts`, chat session switching, `ui/src/ui/controllers/sessions.ts`, and the existing `sessions.create` backend contract; then update the Control UI/WebChat docs for the new workflow.
