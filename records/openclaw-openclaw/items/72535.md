## Summary

Keep open. Current main has not absorbed this PR's hint text, and the branch is useful but too narrow to fully resolve the linked downgrade report because the new warning is emitted only after config validation succeeds. Downgrade configs with unknown keys or incompatible value shapes still take invalid-config paths before that warning, although existing doctor code supports the unknown-key repair path.

## What This Changes

The PR changes `warnIfConfigFromFuture` in `src/config/io.ts` to append downgrade guidance about `openclaw doctor --fix` and manual correction for changed value formats.

## Best Possible Solution

Surface future-version remediation from the invalid-config paths as well as the valid-config warning, using `sourceConfig.meta.lastTouchedVersion` before validation blocks. Add focused tests for unknown-key and changed-value-shape downgrade cases, and keep #72527 open until the invalid-config user path gives clear doctor/manual guidance or the downgrade repair itself is implemented.

## Reproduction Assessment

Yes for the review finding: the source shows both config load and snapshot reads return or throw on `!validated.ok` before `warnIfConfigFromFuture` runs. A live cross-version downgrade was not run because this review is read-only, but the control-flow reproduction is high confidence.

## Solution Assessment

No for the linked downgrade bug as written. The added string is a safe narrow improvement for valid future-written configs, but the maintainable fix should attach the same actionable guidance to invalid-config reporting or expand doctor downgrade repair before using closing syntax for #72527.
