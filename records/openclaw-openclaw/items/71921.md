## Summary

Keep open. Current main still preserves stale explicit Bedrock model rows that omit `input`, and the image capability gate still treats missing `input` as no vision support. The linked implementation PR #72404 remains open and specifically tracks this issue, so the issue should stay open until that PR or a replacement lands.

## What This Changes

Mark Bedrock Claude Opus/Sonnet 4.x model metadata as image-capable, including stale persisted/static catalogs, so image attachments are not rejected as text-only.

## Best Possible Solution

Review and land PR #72404, or replace it with a provider-owned fix that repairs missing Bedrock `input` capability metadata for stale explicit/static rows after discovery succeeds while preserving intentional explicit text-only opt-outs and covering foundation, regional, and global Claude 4.x ids.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is an explicit `models.providers.amazon-bedrock.models` row for a Claude 4.x id with no `input`: current main preserves that row unchanged, then `modelSupportsImages()` returns false for missing `input`.

## Solution Assessment

No for the original registry-only framing. Updating a few generated registry entries would not repair already-persisted stale static catalogs; the maintainable fix should refresh, inherit, or safely default missing capability metadata in the merge/provider path.
