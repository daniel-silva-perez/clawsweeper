## Summary

Current main still preserves the reported stale Bedrock catalog failure path. Bedrock discovery can produce `input: ["text", "image"]`, but existing non-empty static `amazon-bedrock` model lists are kept, explicit-only rows without a matching discovered id are returned unchanged, and the embedded image gate treats missing `input` as no vision. The supplied GitHub context also shows open PR #72404 referencing this issue with closing syntax, so this issue should remain open until that PR or a replacement lands.

## Best Possible Solution

Keep this issue open while #72404 is reviewed, or replace it with a provider-owned fix that refreshes stale explicit/static `amazon-bedrock` model rows from successful Bedrock discovery. The fix should include regression coverage for Bedrock Claude Opus/Sonnet 4.x explicit rows missing `input` or stuck at `["text"]`, including regional and global inference profile id variants.
