## Summary

Current main still has the stale Bedrock catalog failure mode: Bedrock discovery can produce `input: ["text", "image"]`, but existing non-empty explicit/static `amazon-bedrock` model lists are preserved, and explicit-only rows without `input` remain text-only. There is also an open unmerged PR, #72404, linked with closing syntax for this issue, so this issue should remain open until that implementation is reviewed, merged, or replaced.

## Best Possible Solution

Keep this issue open while #72404 is reviewed, or replace it with a provider-owned fix that refreshes stale explicit/static `amazon-bedrock` model rows from successful Bedrock discovery. The fix should include regression coverage for existing Bedrock Claude explicit rows missing `input` or stuck at `["text"]`, including regional/global inference profile id variants. Close only after the linked PR or a replacement lands on main and ships or has clear main-only fix provenance.
