## Summary

Keep open. Current main still leaves the reported stale Bedrock catalog path intact: explicit/static Bedrock model rows without `input` are preserved, and the image gate treats missing `input` as text-only. The issue is also paired with open PR #72404, which references this issue as the implementation candidate, so it should stay open until that PR or a replacement lands.

## What This Changes

Mark or refresh Bedrock Claude Opus/Sonnet 4.x model capability metadata so image-capable Bedrock Claude models are not treated as text-only.

## Best Possible Solution

Review and land PR #72404 or a provider-owned replacement that refreshes or inherits missing Bedrock `input` capability metadata for stale explicit/static rows after discovery succeeds, while preserving intentional explicit text-only opt-outs and covering foundation, regional, and global Claude ids.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a non-empty `models.providers.amazon-bedrock.models` list containing a Claude 4.x row with no `input`: current merge code preserves that row and `modelSupportsImages()` returns false for missing `input`.

## Solution Assessment

No for the original registry-only framing. Updating specific registry entries would not repair already-persisted static catalogs; the narrow maintainable fix should address stale explicit catalog metadata or replace it when discovery succeeds.
