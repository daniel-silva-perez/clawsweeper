## Summary

Keep open. Current main still lacks a Feishu `streamingSearchFallback` setting and still closes streamed interactive cards without sending a searchable final text copy. The PR is a small Feishu-owned implementation aligned with the linked bug report and has no clear blocking correctness or security issue from review, but it remains an open contributor PR needing maintainer review and validation rather than cleanup closure.

## What This Changes

The PR adds an optional `channels.feishu.streamingSearchFallback` setting, documents the visible duplicate-message tradeoff, wires the option through Feishu config schema and plugin metadata, and sends the final streamed response as chunked plain text after closing the streaming card when enabled.

## Best Possible Solution

Merge the Feishu-owned opt-in searchable fallback after maintainer review and targeted validation, keeping the default disabled and retaining the docs warning that enabling it creates a visible duplicate final text message.

## Reproduction Assessment

Yes. The linked issue gives a concrete live path: enable Feishu streaming, receive a streamed interactive-card reply with a distinctive keyword, then search Feishu conversation content and compare it with plain-text delivery.

## Solution Assessment

Yes. The PR's opt-in Feishu-channel setting is the narrowest maintainable solution because it avoids core changes, preserves current streaming defaults, and lets operators explicitly trade duplicate final messages for searchability.
