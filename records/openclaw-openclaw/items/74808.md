## Summary

Keep open. The PR addresses a real Feishu search gap linked from #74767 and current main still lacks an indexed final-text fallback, but the proposed plain-text fallback reuses card-formatted mention text in one path, which would produce incorrect visible fallback content when mention forwarding is involved.

## What This Changes

The PR adds an opt-in Feishu `streamingSearchFallback` config setting, sends a final streamed reply as a Feishu plain-text message when enabled, and documents the search-indexing tradeoff.

## Best Possible Solution

Merge a Feishu-owned opt-in searchable fallback after fixing the mention-formatting edge case, preserving the default `false` setting, covering normal streaming and mention-forwarded fallback behavior in tests, and keeping the docs clear about the visible duplicate-message tradeoff.
