## Summary

Keep open. Current main still lacks a Feishu searchable streaming fallback, and this PR is useful toward the linked bug, but it is not merge-ready because the new Feishu config field is not reflected in the generated bundled channel config metadata that repository checks and schema surfaces use.

## What This Changes

The PR adds an opt-in Feishu `streamingSearchFallback` setting, documents the duplicate-message tradeoff, and sends the final streamed reply through the existing Feishu text-send path after a streaming card closes.

## Best Possible Solution

Update the PR to include the regenerated bundled channel config metadata for the new Feishu setting, then proceed with Feishu maintainer review and targeted validation while keeping the fallback opt-in and documented as a visible duplicate final message.

## Reproduction Assessment

Yes. The linked issue provides a high-confidence live path: enable Feishu streaming, receive a streamed interactive-card reply with a distinctive keyword, then search Feishu conversation content and compare it with non-streaming text delivery.

## Solution Assessment

No, not as submitted. The Feishu opt-in fallback is the right narrow direction, but the config-surface change is incomplete until the generated bundled channel metadata is committed.
