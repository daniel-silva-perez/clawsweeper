## Summary

Keep this PR open. Current main does not implement the PR's intended Feishu legacy document read path: `feishu_doc` still reads only through the docx SDK APIs, the read schema and bundled skill docs still point users at `/docx/` tokens, and the proposed `old-doc` files/tests are absent. The PR remains a focused Feishu plugin implementation candidate, with review feedback already partly addressed by the author and one remaining old-doc failure-mode concern worth maintainer review before landing.

## Best Possible Solution

Keep the PR open for Feishu maintainer review. The likely best path is to finish reviewing the PR's legacy-doc read implementation in `extensions/feishu/src/old-doc.ts` and `extensions/feishu/src/docx.ts`, address or explicitly accept the remaining `/content` best-effort concern, confirm the required Feishu scopes/docs, then land it as an extension-owned feature if targeted Feishu tests pass.
