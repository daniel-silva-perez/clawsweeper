## Summary

Keep this PR open. Current main still reads Feishu documents only through the docx SDK path and does not include the proposed legacy `/docs/:token` old-document read implementation. The PR is a focused Feishu plugin implementation candidate, but it still needs maintainer review around old-doc failure handling, required scopes/docs, and the expanded agent-readable document surface.

## Best Possible Solution

Keep the PR open for Feishu maintainer review. The best path is to finish reviewing the legacy `doc` read implementation in `extensions/feishu/src/old-doc.ts` and `extensions/feishu/src/docx.ts`, fix or consciously accept the old-doc `/content` failure-mode concern, confirm required Feishu scopes and skill/docs updates, run targeted Feishu tests, then land it as an extension-owned feature if accepted.
