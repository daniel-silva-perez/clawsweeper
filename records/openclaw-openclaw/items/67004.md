## Summary

Keep this PR open. Current main still lacks the legacy Feishu old-document read path, and the PR is a focused extension-owned implementation candidate, but the current patch still has a concrete fallback bug and needs Feishu maintainer review before merge.

## What This Changes

The PR adds Feishu `/docs/:token` legacy document read support by detecting old-doc metadata, reading old documents through doc v2 REST endpoints, and routing upgraded documents to the existing docx SDK path.

## Best Possible Solution

Land an extension-owned legacy Feishu read path only after old-doc content failures propagate clearly, richer content parsing is best-effort when raw text is available, required Feishu scopes and skill/docs are updated, and targeted Feishu tests cover both success and failure paths.

## Reproduction Assessment

Yes. A high-confidence reproduction path is to extend the PR's mocked `old-doc.test.ts` harness so metadata returns `is_upgraded: false` and either `/content` or `/raw_content` fails; the current PR code catches that read failure in `docx.ts` and falls through to docx with the legacy token.

## Solution Assessment

No. The extension-owned direction is appropriate, but the current patch is not the best final fix because it still catches downstream old-doc reads as metadata fallback and makes rich content a hard dependency for returning raw text.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep old-doc reads out of the meta fallback:** `extensions/feishu/src/docx.ts:876`
  - body: `readOldDoc` still runs inside the `try` that catches metadata lookup failures. If `/raw_content` or `/content` fails after metadata has identified a legacy doc, the error is swallowed and the code falls through to the docx path with the old `/docs` token, masking the real failure and usually failing with the wrong API. Move the old-doc read after the metadata catch so only `fetchOldDocMeta` is treated as fallback-eligible.
  - confidence: 0.91
- **[P2] Make rich content best-effort for raw old-doc reads:** `extensions/feishu/src/old-doc.ts:155-158`
  - body: The returned document text comes from `/raw_content`, but the implementation waits for `/content` in the same `Promise.all`. A transient or permission failure from the richer endpoint rejects the whole read even when plain text was already available, which under the current caller then falls through to an invalid docx read. Fetch or parse rich content best-effort and still return raw text when possible.
  - confidence: 0.78
