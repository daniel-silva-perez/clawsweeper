## Summary

Keep open. Current main has generic model fallback, normal workspace prompt injection, fallback retry context preservation, and fallback observability, but it still does not implement a native `fallbackRunbook` config field, fallback-only system-prompt injection, or startup/doctor validation for configured fallback model refs.

## What This Changes

Request a native fallbackRunbook model config field that injects fallback-only guardrails/context, with related fallback model validation and active-fallback visibility.

## Best Possible Solution

Add a typed, documented fallback-only prompt field to default and per-agent model config, inject it only when the active candidate is a fallback while preserving normal workspace context, and add non-blocking doctor/models diagnostics for fallback refs that are missing from known provider catalogs.
