## Summary

Keep open. Current main still lacks a native `fallbackRunbook` config key, fallback-only system-prompt injection, and startup/doctor validation for configured fallback model refs; the newer fallback observability and normal bootstrap paths only cover part of the reported outage behavior.

## What This Changes

The issue requests a fallback-only runbook field in agent model config plus diagnostics for invalid fallback model IDs.

## Best Possible Solution

Add a typed and documented fallback-only prompt field for default and per-agent model config, inject it only when the active candidate is a fallback while preserving existing bootstrap context, and add non-blocking diagnostics for configured fallback refs that are missing from known provider catalogs.
