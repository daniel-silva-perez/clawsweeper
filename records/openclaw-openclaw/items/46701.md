## Summary

Keep open. Current main still drops ordinary Telegram group messages from senders rejected by groupAllowFrom/groupPolicy without sending the requested onboarding reply, and the visible authorization replies that do exist are for DM pairing or native command flows rather than this group-message path.

## What This Changes

Request a Telegram group auto-reply for senders rejected by groupAllowFrom/groupPolicy, with admin-contact instructions and the sender's Telegram user ID.

## Best Possible Solution

Add a Telegram-owned, explicit opt-in unauthorized-group-sender notice that is topic-aware, deduplicated or rate-limited, uses existing Telegram send/error helpers, safely includes the sender ID, documents the privacy tradeoff, and tests ordinary group messages separately from DM pairing and native command auth replies.
