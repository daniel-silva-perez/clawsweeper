## Summary

Keep open. Current main still silently drops ordinary Telegram group messages from senders rejected by groupAllowFrom/groupPolicy, and the existing visible authorization replies are limited to DM pairing or native command flows rather than the requested group auto-reply with the sender's Telegram user ID.

## What This Changes

The issue requests a Telegram group auto-reply when a sender is blocked by groupAllowFrom or groupPolicy, including instructions to contact an admin with the sender's Telegram user ID.

## Best Possible Solution

Add a Telegram-owned, explicit opt-in unauthorized-group-sender notice that is topic-aware, deduplicated or rate-limited, uses existing Telegram send/error helpers, safely includes the sender ID, documents the new config and privacy tradeoff, and tests ordinary group messages separately from DM pairing and native command auth replies.
