## Summary

Keep this open. Current main still drops ordinary Telegram group messages from senders rejected by groupAllowFrom/groupPolicy, and the existing visible auth replies are limited to native commands or DM pairing rather than the requested group auto-reply with admin-contact instructions and the sender's Telegram user ID.

## What This Changes

Add a Telegram group auto-reply when groupAllowFrom/groupPolicy blocks a sender, instructing them to contact a group admin and including their Telegram user ID.

## Best Possible Solution

Keep this open for maintainer product/security judgment. A good implementation would likely add an explicit opt-in Telegram setting for unauthorized group replies, rate-limit or de-duplicate replies, send thread-aware replies through existing Telegram API/error helpers, include the sender ID safely, document the behavior, and test ordinary group messages separately from DM pairing and native command auth replies.
