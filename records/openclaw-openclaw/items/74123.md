## Summary

Current main still has the sandbox media resolver gap reported here: sandboxed image inputs call the shared resolver with `media://inbound/...`, and that resolver attempts direct bridge path resolution before the inbound fallback can map the claim-check URI to `media/inbound/<id>`. An open linked fix PR, #74186, uses closing syntax for this issue, so this issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land #74186, or an equivalent narrow fix, so `resolveSandboxedBridgeMediaPath` recognizes canonical `media://inbound/<id>` before direct bridge resolution and maps it to the sandbox-visible `media/inbound/<id>` file when present. Add focused resolver and sandboxed image-tool regression coverage, and keep behavior aligned with the related non-sandbox work in #63497.
