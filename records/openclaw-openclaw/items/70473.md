## Summary

Keep this PR open. Current main still has the narrower overflow token extractor and only forwards `currentTokenCount` when a provider error exposes one of the existing parsed formats. The PR's central change, deriving counts from MiniMax/Kimi-style formats and forcing a minimally over-budget count for confirmed but unparseable overflow text, is not implemented on main. Related PR #70479 addresses the later session-preservation failure and does not supersede this overflow-budget extraction work.

## Best Possible Solution

Keep the PR open for review, refresh it against current main if needed, and land the narrow overflow-budget extraction/fallback behavior with the proposed regression tests and docs if CI passes. Treat #70479 as the separate session-preservation follow-up.
