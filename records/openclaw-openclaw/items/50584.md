## Summary

Keep open. Current main only implements part of the surrounding problem: it writes local-time slugged session-memory files and startup context now reads recent slugged memory artifacts, but it does not implement this PR's opt-in LLM synthesis mode or canonical daily-file append. The PR remains a live implementation candidate rather than something already implemented on main.

## Best Possible Solution

Keep this PR open for maintainer review. If the product direction is to keep synthesis in core, rebase/review the current branch and land the opt-in `synthesis` path plus canonical append with focused tests and docs. If core should not own synthesis, split the remaining request into the open memory-preservation discussion (#40418) and close this PR only after a maintainer explicitly chooses that direction.
