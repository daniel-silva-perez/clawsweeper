## Summary

Keep open. Current main only partially overlaps this PR: the timezone filename bug has shipped separately, and startup context now reads date-prefixed slugged memory files, but main still lacks the PR's opt-in session synthesis mode and canonical daily-file append behavior. This PR remains an active implementation candidate after maintainer repair on the branch, so it should be reviewed/landed, narrowed, or explicitly superseded by a maintainer rather than closed by cleanup.

## Best Possible Solution

Keep this PR open for maintainer review. The clean path is to decide whether canonical daily append is still wanted now that startup reads slugged files, then land or narrow the PR around the remaining useful behavior: opt-in session-memory synthesis with tests, clear docs, safe append semantics, and preserved contributor credit.
