## Summary

Keep open. Current main still routes plugin TypeScript source entries through the synchronous Jiti loader, and Windows remains explicitly excluded from the native Jiti/native-require fast paths. Recent fixes reduce some compiled bundled-JS overhead, but they do not implement the async TS-aware native-import path described in the report.

## Best Possible Solution

Keep the issue open and design a Windows-safe loader improvement for TS source plugin entries or their real production call sites. The fix should preserve plugin SDK aliasing, avoid the Windows raw-path native-import crash covered by #71749, and add targeted regression coverage or profiling for CLI registration/status, gateway bootstrap, and other named plugin-loading paths.
