## Summary

Keep this PR open. Current main still does not implement `channels.signal.configPath`, and the contributor has explicitly invited maintainer takeover. The idea remains valid for Signal users, but the submitted branch is stale against the current bundled Signal plugin layout and should be ported rather than merged as-is.

## Best Possible Solution

Keep this PR open for maintainer takeover or a replacement patch. Port the option to the current Signal plugin by adding optional `channels.signal.configPath` to the config type/schema and UI/help metadata, include it in configured-account detection, propagate it from monitor startup into `spawnSignalDaemon`, emit `--config <path>` only for trimmed nonblank values, update the public Signal docs, and add focused top-level plus account-scoped tests.
