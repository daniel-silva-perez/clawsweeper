## Summary

Keep this PR open. Current main still does not implement `channels.signal.configPath`, while the PR discussion treats the feature as valid but needing maintainer takeover because the branch targets the old Signal layout. The intended `signal-cli --config <dir>` behavior is supported upstream, but OpenClaw’s current Signal plugin type, strict schema, daemon args, monitor autostart path, tests, UI hints, and docs still omit it.

## Best Possible Solution

Keep this PR open for maintainer takeover or a replacement patch. Port optional `channels.signal.configPath` into the current Signal plugin by adding it to the config type, strict schema, help/label/UI metadata, docs, and configured-account detection; propagate it through monitor autostart into `spawnSignalDaemon`; emit `--config <path>` only for trimmed nonblank values; and add focused tests for top-level, account-scoped, unset, and blank values. If accepted, add a user-facing changelog entry crediting `Thanks @NorseGaud`.
