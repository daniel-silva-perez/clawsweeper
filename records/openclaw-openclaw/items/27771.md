## Summary

Keep this PR open. Current main still does not implement `channels.signal.configPath`, and the submitted change remains useful as a feature candidate, but it needs maintainer takeover because the PR targets the removed `src/signal/*` layout while Signal now lives under `extensions/signal/*`.

## Best Possible Solution

Keep this PR open for maintainer takeover or a replacement patch. Port the option to the current Signal plugin by adding optional `channels.signal.configPath` to the config type/schema and UI/help metadata, include it in configured-account detection, propagate it from monitor to daemon startup, emit `--config <path>` only for nonblank values, update the public Signal docs, and add focused tests for top-level and account-scoped config behavior.
