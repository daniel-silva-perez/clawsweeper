## Summary

Keep this PR open. Current main still does not implement `channels.signal.configPath`: the Signal config type, strict config schema, daemon argv builder, monitor autostart path, UI hints, tests, and public docs still omit the field. The PR’s requested behavior is valid for signal-cli, but the submitted branch targets the old `src/signal` layout and needs a maintainer port into the current `extensions/signal` plugin structure.

## Best Possible Solution

Keep this PR open for maintainer takeover or a replacement patch. Port optional `channels.signal.configPath` and account-scoped `channels.signal.accounts.<id>.configPath` into the current Signal plugin by updating the config type, strict schema, UI/help metadata, docs, configured-account detection, monitor autostart propagation, and daemon argv builder. Add focused tests for top-level, per-account, unset, and blank values. If accepted, add a user-facing changelog entry crediting `Thanks @NorseGaud`.
