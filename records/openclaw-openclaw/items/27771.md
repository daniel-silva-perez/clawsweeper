## Summary

Keep this PR open. Current main still does not implement `channels.signal.configPath`, and the submitted patch remains a plausible feature candidate, but it is stale against the current Signal plugin layout and needs maintainer takeover or a replacement patch under `extensions/signal`.

## Best Possible Solution

Keep this PR open for maintainer takeover or a replacement patch. Port the feature to the current bundled Signal plugin by adding optional `channels.signal.configPath` to the Signal config type and strict schema, plugin UI/help metadata, configured-account detection, docs, and monitor-to-daemon propagation; add focused tests proving `--config <path>` is emitted only for nonblank values and that the option works at the supported account scopes.
