## Summary

Keep this PR open. Current main still does not implement `channels.signal.configPath`, and the PR remains a useful feature candidate, but it needs maintainer takeover or a replacement patch because the submitted runtime diff targets the removed `src/signal/*` layout instead of the current bundled Signal plugin under `extensions/signal`.

## Best Possible Solution

Keep this PR open for maintainer takeover or a replacement patch. Port the feature to the current bundled Signal plugin by adding optional `channels.signal.configPath` to the Signal config type, strict schema, plugin UI/help metadata, configured-account detection, and Signal docs; propagate it through `monitorSignalProvider` into `spawnSignalDaemon`; and add focused tests proving `--config <path>` is emitted only for nonblank values, including account-level config behavior if supported.

## Risks / Open Questions

- The submitted diff likely cannot merge as-is because it edits removed `src/signal/*` files rather than the current `extensions/signal/*` plugin implementation.

- Current main has no tests for `channels.signal.configPath`, so a replacement patch needs focused schema, account-resolution, daemon-argv, monitor propagation, and docs coverage.
