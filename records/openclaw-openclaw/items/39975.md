## Summary

Keep this PR open. Current main still builds `message_reference` inside `sendWebhookMessageDiscord` and includes it in the Execute Webhook JSON body whenever `replyTo` is present. The bound-thread webhook path still forwards `replyToId` into that helper, so the PR's core fix remains relevant, although the branch needs to be refreshed from the old `src/discord/send.outbound.ts` path to `extensions/discord/src/send.outbound.ts`.

## Best Possible Solution

Refresh or replace this PR with the same narrow behavior change against `extensions/discord/src/send.outbound.ts`: remove `message_reference` only from `sendWebhookMessageDiscord`, preserve bot-token/channel reply behavior in `sendMessageDiscord`, media, components, voice, and draft-stream paths, and add a regression test proving a webhook send with `replyTo` does not serialize `message_reference`.
