## Summary

Keep open. Current main and latest release still serialize `message_reference` in Discord Execute Webhook bodies when bound-thread persona delivery passes `replyTo`, while Discord's webhook endpoint docs do not list that field. The PR's core fix is still useful, but the branch targets the old `src/discord` path and needs a current-path refresh plus a regression test.

## Best Possible Solution

Refresh or replace this PR with the same narrow behavior change against `extensions/discord/src/send.outbound.ts`: remove `messageReference` only from `sendWebhookMessageDiscord`'s Execute Webhook request body, preserve `sendMessageDiscord` bot-token reply behavior, and add a focused webhook request-body regression test for `replyTo` that asserts `message_reference` is not serialized.
