## Summary

Keep this PR open. Current main already covers the daemon gateway-token persistence and config-backed SecretRef install behavior, but it still does not implement the PR's ambient provider `*_FILE` environment-variable convention. Closing as implemented would drop a substantive provider-secret capability; merging the branch as-is would also overlap newer gateway-token code and carry unresolved review findings.

## Best Possible Solution

Keep this PR open as useful but stale work. Rebase or rewrite it against current main, drop the now-obsolete daemon gateway-token persistence edits, and focus on the still-missing provider `*_FILE` env-var behavior if maintainers want that convention. The provider implementation should integrate with `src/agents/model-auth-env.ts`, `src/agents/live-auth-keys.ts`, and `src/infra/provider-usage.auth.ts`, preserve direct env precedence, avoid treating `*_FILE` names as prefixed API keys, preserve separators for multi-key files, surface unreadable-file misconfiguration clearly, and add focused tests for direct-env precedence, missing/unreadable files, single-key vars, list vars, prefixed vars, and provider usage auth.
