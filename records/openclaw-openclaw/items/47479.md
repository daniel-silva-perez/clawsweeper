## Summary

Keep open. The PR targets a valid Docker setup problem, but the branch is stale against current `main`: the maintained Docker setup logic now lives in `scripts/docker/setup.sh`, while the PR modifies the old root setup surface, so it should not merge as-is.

## What This Changes

The PR branch modifies root `docker-setup.sh` to parse `.env` defaults, quote dotenv rewrites, pass `OPENCLAW_INSTALL_BROWSER` into Docker builds, and ignore `docker-compose.override.yml`.

## Best Possible Solution

Move the useful dotenv-default loading, sourceable quote-preserving `.env` writes, caller-environment precedence, and `OPENCLAW_INSTALL_BROWSER` build-arg wiring into `scripts/docker/setup.sh`; keep root `docker-setup.sh` as a wrapper and add focused e2e coverage in `src/docker-setup.e2e.test.ts`.

## Reproduction Assessment

Yes. Static current-main inspection gives a high-confidence path: put `OPENCLAW_DOCKER_APT_PACKAGES="jq ripgrep"` or `OPENCLAW_INSTALL_BROWSER=1` in root `.env` and run the active setup path; those values are not generally loaded, and managed whitespace values are still rewritten bare.

## Solution Assessment

No. The PR solves the right user problem in useful source form, but it is no longer the narrow maintainable fix because the active Docker setup implementation has moved to `scripts/docker/setup.sh`.
