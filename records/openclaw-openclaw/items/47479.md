## Summary

Keep open. Current main still has the underlying Docker setup dotenv gap, but this PR targets `docker-setup.sh`, which is now only a shim, so it would not fix the active setup path.

## What This Changes

The PR adds manual project `.env` loading and quote-preserving `.env` writes to Docker setup, ignores `docker-compose.override.yml`, and passes `OPENCLAW_INSTALL_BROWSER` as a Docker build argument.

## Best Possible Solution

Port the useful behavior into `scripts/docker/setup.sh`: non-executing dotenv loading with caller-env precedence, sourceable writes for managed `.env` keys, active `OPENCLAW_INSTALL_BROWSER` build-arg propagation, and focused Docker setup e2e coverage.
