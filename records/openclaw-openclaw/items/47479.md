## Summary

Keep open. The PR is aimed at a real Docker setup bug, but it no longer patches the active implementation on current main: the root script is now a wrapper, while the active setup script still lacks general root `.env` loading, still rewrites managed whitespace values without quotes, and still does not pass the browser-install build arg.

## What This Changes

The PR changes root `docker-setup.sh` to parse root `.env` defaults, quote managed `.env` rewrites, pass `OPENCLAW_INSTALL_BROWSER` into Docker builds, and ignore `docker-compose.override.yml`.

## Best Possible Solution

Retarget the useful dotenv-default loading, quote-preserving `.env` writes, caller-environment precedence, and `OPENCLAW_INSTALL_BROWSER` build-arg wiring into `scripts/docker/setup.sh`; keep root `docker-setup.sh` as a wrapper and drop the `.gitignore` change because current main already ignores `docker-compose.override.yml`.

Yes. ## Solution Assessment

No. The PR solves the right user problem in useful source form, but it is not the narrow maintainable fix anymore because the active Docker setup implementation moved to `scripts/docker/setup.sh`.
