## Summary

Keep open. Current main still has the Docker setup .env handling gap, and this PR's patch is now aimed at the old root script shape rather than the active setup implementation under `scripts/docker/setup.sh`. The central idea remains useful, but it needs to be ported with focused regression coverage before merge.

## What This Changes

The PR adds a manual project `.env` loader to the Docker setup shell script, writes whitespace/quote-containing `.env` values with quotes, ignores `docker-compose.override.yml`, and passes `OPENCLAW_INSTALL_BROWSER` as a Docker build argument.

## Best Possible Solution

Port the useful parts into `scripts/docker/setup.sh`: a non-executing dotenv loader that preserves caller-env precedence, sourceable writes for managed `.env` keys, active build-arg propagation for `OPENCLAW_INSTALL_BROWSER`, and targeted e2e tests around the Docker setup sandbox.
