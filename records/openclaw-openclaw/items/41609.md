## Summary

Keep open. Current main still does not implement direct inbound Feishu interactive-card parsing for `im.message.receive_v1` events. The fetched/read and thread-history path has partial `interactive` parsing, including post fallback handling, but the live bot event parser still has no `interactive` or `interactive_card` branch, so the reported inbound card content gap remains.

## Best Possible Solution

Keep this issue open. The best path is to land or replace the focused Feishu inbound parsing work: share the interactive-card parser with `parseMessageContent()`, handle both `interactive` and `interactive_card` direct event types, preserve useful raw card JSON where appropriate, ensure readable content reaches `RawBody` and agent context, add direct inbound `parseFeishuMessageEvent()` tests, and update the Feishu docs once receive support is real.
