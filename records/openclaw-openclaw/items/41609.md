## Summary

Keep open. Current main still has the Feishu direct inbound message path wired through `parseMessageContent()`, and that parser still has no `interactive` or `interactive_card` branch. Existing interactive-card parsing on main is limited to fetched/read/thread message parsing in `send.ts`, so it does not solve the live `im.message.receive_v1` issue reported here.

## Best Possible Solution

Keep this issue open and land a focused Feishu plugin fix. Share or move the existing interactive-card parser into the live inbound path so `parseMessageContent()` and merged/forwarded submessage formatting handle direct `interactive` and `interactive_card` message types, preserve useful card/action JSON where appropriate, surface readable content through `RawBody` and agent context, add direct inbound regression tests, and update Feishu docs once receive-side card parsing is real.
