## Summary

Keep open. Current main still does not parse Feishu live inbound interactive-card message content: `im.message.receive_v1` events flow through `parseFeishuMessageEvent()` and `parseMessageContent()`, and that parser still has no `interactive` or `interactive_card` branch. Recent related work improved fetched/quoted card parsing, but the merged maintainer PR explicitly says it does not close the broader inbound work, and the open follow-up PR is also scoped to quoted/fetched parser gaps.

## Best Possible Solution

Keep this issue open and land a focused Feishu plugin fix. Share or move the interactive-card parser so `parseMessageContent()` or `parseFeishuMessageEvent()` handles direct inbound `interactive` and `interactive_card` message types, preserve useful card/action JSON where appropriate, surface readable content through `RawBody` and agent context, add direct inbound regression tests, and update Feishu receive docs when support is real. Review #73203 as a useful parser-coverage follow-up, but do not treat it as sufficient unless it also fixes the live inbound event path.
