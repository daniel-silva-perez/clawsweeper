## Summary

Keep open. Current main still does not parse direct inbound Feishu interactive-card message events: live `im.message.receive_v1` events flow through `parseMessageContent()`, which has no `interactive` or `interactive_card` branch. The existing interactive-card parser is still scoped to fetched/read/thread message parsing in `send.ts`, and the related merged/open work improves quoted or fetched card parsing rather than the live inbound event path reported here.

## Best Possible Solution

Keep this issue open as the canonical direct inbound Feishu interactive-card parsing gap. A focused fix should share or move the existing parser into the live inbound path, handle `interactive` and `interactive_card` in `parseMessageContent()` and merged/forwarded submessage formatting, preserve useful raw card/action payload data where the Feishu context supports it, add direct inbound regression tests, and update Feishu docs after receive-side card parsing is real.
