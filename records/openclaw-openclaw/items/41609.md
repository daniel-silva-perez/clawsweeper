## Summary

Keep open. Current main still does not parse direct inbound Feishu interactive-card message events: live `im.message.receive_v1` events flow through `parseMessageContent()`, which has no `interactive` or `interactive_card` branch. Existing interactive-card parsing remains scoped to fetched/read/thread message parsing in `send.ts`, and card action callbacks still ignore non-`value` selection/form fields.

## Best Possible Solution

Keep this issue open as the canonical direct inbound Feishu interactive-card parsing gap. A focused fix should share or move the existing parser into the live inbound path, handle `interactive` and `interactive_card` in `parseMessageContent()` and merged/forwarded submessage formatting, fetch or preserve raw card payload data when Feishu only supplies a placeholder, coordinate with #42908 for `option`/`options`/`form_value` action callbacks, add direct inbound regression tests, and update Feishu docs once receive-side card parsing is real.
