## Summary

Keep open. Current main still has the reported Feishu direct inbound interactive-card gap: live `im.message.receive_v1` events are parsed through `parseMessageContent()`, and that parser still has no `interactive` or `interactive_card` branch. Existing card parsing on main is limited to fetched/read/thread message paths, so it does not prove this issue is fixed.

## Best Possible Solution

Keep this issue open and land a focused Feishu plugin fix: share or move the interactive-card parser so `parseMessageContent()` handles direct inbound `interactive` and `interactive_card` message types, preserve useful card/action JSON where appropriate, ensure readable content reaches `RawBody` and agent context, add direct `parseFeishuMessageEvent()` regression tests, and update Feishu receive docs once support is real. Related parser PRs can be reused for extraction details, but they should not be treated as closing this direct inbound issue unless they wire the live event path too.
