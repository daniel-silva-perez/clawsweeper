## Summary

Keep open. Current main still does not parse Feishu interactive-card content on the direct inbound `im.message.receive_v1` path: live events flow through `parseFeishuMessageEvent()` and `parseMessageContent()`, and that parser has no `interactive` or `interactive_card` branch. The fetched-message/thread-history path has partial `interactive` parsing, and card action callbacks exist, but neither proves the reported inbound message-content gap is fixed.

## Best Possible Solution

Keep this issue open and land a focused Feishu fix in the owner plugin: share or move the interactive-card parser so `parseMessageContent()` handles direct inbound `interactive` and `interactive_card` message types, preserve useful raw card/action JSON where appropriate, ensure readable card content reaches `RawBody` and agent context, add direct `parseFeishuMessageEvent()` regression tests, and update the Feishu receive docs once the support is real. Related fetched-message/card-parser follow-ups such as #60380, #60383, and #73203 can inform the parser details, but they do not replace direct inbound event coverage.
