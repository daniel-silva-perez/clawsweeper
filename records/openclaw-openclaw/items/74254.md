## What This Changes

The branch adds a Dockerfile guard requiring exactly one GnuPG `pub` primary key in Docker's apt key file before fingerprint comparison and `gpg --dearmor`, plus a focused Dockerfile ordering test.

## Best Possible Solution

Add the Unreleased Fixes changelog entry, keep the Dockerfile guard and focused ordering test, then merge this PR or an equivalent narrow hardening change so #74234 closes through the merged fix.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: a key file whose first `fpr` matches the pinned fingerprint but also contains an extra primary key would pass current main's `awk ... exit` comparison and then be fully dearmored into the apt keyring.

## Solution Assessment

Yes, aside from the missing changelog. Counting primary `pub` records before reading the first fingerprint is a narrow maintainable fix because GnuPG's colon format distinguishes primary public keys from subkeys, and the test locks the ordering before `gpg --dearmor`.
