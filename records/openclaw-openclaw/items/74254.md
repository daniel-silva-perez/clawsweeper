## Summary

Keep open: current main still has the first-fingerprint-only Docker apt key flow, while the PR is a narrow hardening change for the linked Docker GPG validation bug. Because it changes a package-repository trust path, it should get maintainer/security-aware review and merge validation rather than being auto-closed or routed to a replacement repair job.

## What This Changes

This PR adds a Dockerfile guard that requires exactly one GnuPG `pub` record in the downloaded Docker apt key before fingerprint comparison and `gpg --dearmor`, plus a Dockerfile structure test for that ordering.

## Best Possible Solution

The Docker CLI install path should reject zero or multiple primary Docker apt keys before fingerprint comparison and before `gpg --dearmor`, keep legitimate subkeys accepted, and carry focused ordering coverage plus official-key and multi-primary-key validation proof.
