## Summary

Keep open. Current main still lacks provider-qualified alias resolution in the shared model-selection path, and the linked closing PR is open and unmerged, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that Control UI/TUI mid-session model switching can persist or pass display aliases instead of canonical provider model IDs.

## Best Possible Solution

Review and land PR #75198 or an equivalent shared model-selection fix that resolves `provider/Alias` to the configured canonical provider/model before `sessions.patch` persists the session override.

## Reproduction Assessment

Yes. A focused source-level reproduction is a configured alias such as `agents.defaults.models["lmstudio-moe/qwen3.6-35b-a3b"].alias = "MoE"` with a session patch value of `lmstudio-moe/MoE`; current main parses that as provider `lmstudio-moe` and model `MoE` instead of resolving the alias suffix.

## Solution Assessment

Yes, the linked PR direction is the narrow maintainable fix: resolve aliases in the shared model-selection boundary and prove `sessions.patch` persists the canonical model. The issue should stay open until that implementation is merged.
