  - body: The diff adds error rendering to the old inline status command, but current `main` routes `/status` through `buildStatusText`; after rebasing, the user-facing fix needs to live in `src/status/status-text.ts` or chat `/status` will keep hiding provider errors and timeouts.
  - confidence: 0.86
- **[P1] Preserve existing workspaces without an env override:** `scripts/sync-runtime-config.mjs:120-124`
  - body: This startup sync rewrites `agents.defaults.workspace` to `${stateDir}/workspace` whenever `OPENCLAW_WORKSPACE_DIR` is unset. Existing deployments with a custom workspace in `openclaw.json` would silently move agent context/session paths after restart; only set this when the env override is explicit or the field is missing.
  - confidence: 0.89
- **[P1] Let the gateway start when Tailscale bootstrap fails:** `scripts/fly-entrypoint.sh:90`
  - body: Because the entrypoint runs with `set -e`, a non-zero `tailscale up` exits before `exec "$@"`. A transient Tailnet or auth-key failure would stop the whole gateway instead of only disabling Tailscale exposure, so this step should retry or degrade non-fatally.
  - confidence: 0.88
- **[P1] Drop privileges before starting OpenClaw:** `Dockerfile:262-263`
  - body: The branch removes the previous `USER node` hardening and the new entrypoint ultimately execs the gateway as root. If root is required for bootstrapping, the image or script should drop to the node user before starting OpenClaw so the runtime app does not run with root privileges.
  - confidence: 0.87
- **[P3] Use the shared probe handler for /health:** `src/gateway/server-http.ts:771-774`
  - body: `/health` is already registered as a live gateway probe and current `main` handles it before config loading. This new exact-URL shortcut returns a different shape and leaves the shared probe map with a dead duplicate path.
  - confidence: 0.8

## Security Review

Status: needs_attention

Summary: Security review needs attention because the diff introduces container privilege and supply-chain risks unrelated to the `/status` UX fix.

Concerns:

- **[high] Gateway would run as root in the Docker image:** `Dockerfile:262`
  - body: The PR removes the existing non-root runtime hardening and adds an entrypoint that ultimately execs the gateway as root, broadening the impact of any runtime compromise.
  - confidence: 0.87
- **[medium] Root-executed installers are not pinned:** `Dockerfile:131`
  - body: The Dockerfile executes remote Bun and Tailscale install scripts without version pins, so each build can execute different upstream code as root and produce a less auditable image.
  - confidence: 0.84
- **[medium] Persistent root operator config increases blast radius:** `scripts/fly-entrypoint.sh:38`
  - body: The entrypoint symlinks root-scoped Codex, GitHub CLI, Git, and SSH configuration into the Fly volume; combined with running the gateway as root, a runtime compromise could persist and reuse operator credentials across restarts.
  - confidence: 0.78
