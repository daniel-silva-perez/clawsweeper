## Summary

Keep this PR open, but it should not be merged as-is. The stated `/status` UX fix is still not implemented on current main: chat status still drops provider usage entries with `error` and also hides thrown usage-fetch failures. The PR contains a plausible small fix, but it is based on an older status file and bundles broad unrelated Fly/Docker/workflow/plugin-SDK changes with security-sensitive regressions noted in review comments.

## Best Possible Solution

Keep the PR open only as a useful signal, not as a merge candidate. The best maintainer path is a focused rework that updates `src/status/status-text.ts` to surface `usageEntry.error` and caught usage-summary failures, adds a targeted regression test for chat `/status`, and leaves the Fly/Docker/workflow/plugin-SDK changes for separate owner-reviewed PRs if still wanted.
