## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Preserve existing workspace when no override is set:** `scripts/sync-runtime-config.mjs:120-124`
  - body: This rewrites agents.defaults.workspace to `${stateDir}/workspace` on every startup when OPENCLAW_WORKSPACE_DIR is unset. Existing deployments with a custom workspace in openclaw.json would silently move agent context, memory, and session paths after restart; only set this when the env override is explicit or the field is missing.
  - confidence: 0.89
- **[P1] Keep gateway startup independent of Tailscale auth:** `scripts/fly-entrypoint.sh:90`
  - body: Because the entrypoint runs with `set -e`, a non-zero `tailscale up` exits before `exec "$@"`. A transient Tailnet or auth-key failure would stop the whole gateway instead of only disabling Tailscale exposure, so this step should retry or degrade non-fatally.
  - confidence: 0.88
- **[P1] Drop privileges before launching the gateway:** `Dockerfile:262-263`
  - body: The branch removes the previous `USER node` hardening and the new entrypoint execs the gateway as root. If root is required for bootstrapping, the script or image should drop to the node user before starting OpenClaw so the runtime container does not run the app with root privileges.
  - confidence: 0.84
- **[P3] Use the existing probe path for /health:** `src/gateway/server-http.ts:771-774`
  - body: /health is already registered in `GATEWAY_PROBE_STATUS_BY_PATH`, but this early exact-URL handler returns a different response shape and bypasses the shared probe behavior. Move the config-independent handling into the existing probe path or remove the duplicate map entry.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The diff introduces concrete container and supply-chain risks unrelated to the /status UX change.

Concerns:

- **[high] Gateway would run as root in the Docker image:** `Dockerfile:262`
  - body: The PR removes the existing non-root runtime hardening and adds an entrypoint that ultimately execs the gateway as root. That broadens impact from any runtime compromise and should be fixed by restoring USER node or explicitly dropping privileges after root-only bootstrap steps.
  - confidence: 0.86
- **[medium] Root-executed installers are not pinned:** `Dockerfile:131`
  - body: The Dockerfile executes remote install scripts for Bun and Tailscale without version pins. Each build can execute different upstream code as root, which makes release images harder to audit and reproduce.
  - confidence: 0.83
- **[medium] Persistent root SSH/GitHub/Codex config increases blast radius:** `scripts/fly-entrypoint.sh:35`
  - body: The entrypoint symlinks /root/.ssh, /root/.config/gh, and /root/.codex into the Fly volume. Combined with running the gateway as root, a runtime compromise can persist and reuse operator credentials across restarts.
  - confidence: 0.78
