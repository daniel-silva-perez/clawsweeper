## Summary

Keep this PR open. Current main still does not classify the raw MiniMax 529 text `High traffic detected. For a more stable experience, upgrade to our Plus plan` as overloaded; only explicit HTTP 529/status-prefixed paths are covered. The PR is functionally relevant, but it needs a regression test and likely maintainer judgment on whether the fix belongs in the MiniMax provider hook rather than the shared core matcher.

## Best Possible Solution

Keep this PR open for maintainer review. The best landing path is either a tested MiniMax-owned `classifyFailoverReason` hook in `extensions/minimax/provider-registration.ts`, or an explicitly accepted shared matcher change. In either case, add a focused regression test proving the exact MiniMax high-traffic message classifies as `overloaded` and not `billing`.
