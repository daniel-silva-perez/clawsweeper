## Summary

Keep open. Current main still only recognizes Slack direct `<@botUserId>` mentions, `app_mention` event signal, configured regex mention patterns, and implicit thread mentions. There is no `<!subteam^...>` or `usergroups.users.list` handling in the Slack adapter, config/schema, tests, setup manifest, or docs. The linked implementation PR #73828 is closed unmerged after maintainer discussion preferred an API-backed Slack user-group membership path instead of the config-only allowlist.

## Best Possible Solution

Keep this issue open and implement the maintainer-preferred Slack API path: detect `<!subteam^S...>` tokens, resolve membership with Slack `usergroups.users.list` after direct mention matching fails, cache results with conservative invalidation, treat membership containing the routed bot user id as an explicit mention, update Slack setup/docs for `usergroups:read`, and add focused Slack prepare tests for matching, non-members, missing-scope fallback, and requireMention gating.
