## Summary

Keep open. Current main still only recognizes Slack direct `<@botUserId>` mentions, explicit `app_mention` events, configured mention regexes, and implicit thread mentions; there is no current `<!subteam^...>`/`subteamMentions` path. The linked open PR #73828 is the active implementation candidate and uses closing syntax, so this issue should remain open until that PR lands or maintainers choose a different fix.

## Best Possible Solution

Keep this issue open and review #73828 as the implementation candidate. A complete fix should add Slack subteam token extraction and per-agent/global matching in the Slack adapter, update the strict config schema and generated/docs surfaces for any new `groupChat.subteamMentions` field, and include focused Slack mention-gating tests. If maintainers prefer automatic Slack API expansion later, track that as a separate enhancement after the config-only path is decided.
