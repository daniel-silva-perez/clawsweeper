## Summary

Keep this issue open. Current main still has the write-side contamination path: per-channel-peer direct messages resolve to scoped session keys, but several inbound direct-message paths still pass the agent main session key to last-route updates, and the central recorder still persists delivery route fields to that caller-supplied target. The supplied maintainer comments also explicitly separate this issue from the read-side mitigation in #38418, and the two linked implementation PRs for this issue are closed unmerged.

## Best Possible Solution

Keep this issue open and finish the write-side fix. The likely maintainer path is to make all direct-message inbound writers use resolveInboundLastRouteSessionKey or centralize that decision in recordInboundSession, so lastRoutePolicy=session writes delivery metadata to the scoped inbound session while lastRoutePolicy=main continues updating the main session. Add regression coverage for iMessage plus representative Slack, Signal, LINE, Matrix, Mattermost, and Discord component paths, and preserve the existing Telegram/WhatsApp guarded behavior.
