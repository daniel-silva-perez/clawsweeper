## Summary

Current main still has the documentation drift this PR targets, and the PR remains useful but incomplete. The source JSDoc and FIELD_HELP text still advertise 25000/60000 while the runtime DEFAULT_TIMING is 10000/30000; the checked-in generated schema also still embeds the stale values, so merging only the two source documentation edits would leave schema/UI metadata consumers with incorrect help text.

## Best Possible Solution

Keep this PR open and either ask the contributor to update it or apply the equivalent docs-only fix across src/config/types.messages.ts, src/config/schema.help.ts, and the regenerated src/config/schema.base.generated.ts output. No runtime change is needed. The maintainer validation path should include the config schema generation/check path, and config docs baseline checking if that surface changes.
