## Summary

Current main still has the docs drift this PR targets: the source JSDoc/help strings and generated config schema still advertise 25000/60000 while runtime defaults are 10000/30000. The PR is useful, but it changes only the two source documentation files and still needs the generated schema refreshed before it is complete.

## Best Possible Solution

Keep this PR open and ask for, or apply, the equivalent docs-only fix across src/config/types.messages.ts, src/config/schema.help.ts, and the regenerated src/config/schema.base.generated.ts output. No runtime behavior change is needed; after updating, validate with the existing config schema check path.
