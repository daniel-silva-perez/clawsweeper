## Summary

Keep open. Current main still exposes the shared `volcengine-plan` and `byteplus-plan` coding catalog entries as text-only, and the current runtime source is the plugin manifest catalog rather than the old `src/agents/volc-models.shared.ts` path used by this PR. The PR remains a plausible focused bug fix, but it needs a rebase to the manifest-backed catalogs and current tests before it can be closed or merged. Security review found no supply-chain-sensitive surface in the provided diff; the blocker is functional staleness, not suspicious code.

## Best Possible Solution

Keep this PR open but require a current-main rebase. The best fix is to update the five shared coding model rows to `input: ["text", "image"]` in both `extensions/volcengine/openclaw.plugin.json` and `extensions/byteplus/openclaw.plugin.json`, update provider docs/changelog if the behavior ships, and add current colocated regression coverage proving `buildDoubaoCodingProvider()`, `buildBytePlusCodingProvider()`, and the implicit provider surface expose image input.
