## Summary

Keep this PR open: the underlying Volcengine/BytePlus coding-plan image-capability bug is still present on current main, but this branch is stale and incomplete. Current runtime catalogs are manifest-backed, both plan catalogs still mark the relevant models as text-only, and the PR does not update the BytePlus runtime catalog while also changing the preview model contrary to its stated scope.

## What This Changes

The PR updates Volc coding model capability metadata and adds regression assertions so shared coding-plan models advertise image input for Volcengine and BytePlus.

## Best Possible Solution

Rebase or replace the PR against current main, update the manifest-backed Volcengine and BytePlus plan catalogs only for verified image-capable coding models, keep the preview model text-only unless separately verified, update user-facing provider docs if capabilities change, and add extension-level regression tests for both provider catalogs.
