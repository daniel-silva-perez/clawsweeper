## Summary

Keep open. Current main still has the hook-agnostic CLI session-id selection that this PR targets, while the PR is narrow, has focused regression coverage, and has no concrete functional or security finding from this read-only review.

## What This Changes

The PR adds hook-aware CLI session-id selection in `src/agents/cli-output.ts` and regression tests so Claude CLI `SessionStart` hook IDs do not replace the real resumable conversation ID.

## Best Possible Solution

Land or adapt the shared `nextCliSessionId` helper so all CLI output parsers prefer real non-hook session IDs while preserving hook IDs only as a last-resort fallback, with the three focused regression tests kept with the parser tests.

## Reproduction Assessment

Yes. The deterministic reproduction is a JSON/JSONL stream where Claude CLI emits `hook_started`/`hook_response` with a transient `session_id` before `init`, plus a defensive reversed-order case where a hook arrives after `init`; current main's parser code selects the wrong ID in those paths.

## Solution Assessment

Yes. The proposed shared helper is the narrowest maintainable shape I found because it fixes the known first-wins bug and the reviewer-identified last-wins variants at the three existing parser sites without changing persistence or retry policy.
