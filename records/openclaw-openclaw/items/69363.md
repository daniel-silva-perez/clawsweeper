## Summary

Keep this PR open. Current main still lacks hook-aware CLI session-id selection: `parseCliJsonl` can persist the first transient Claude hook id, and `parseCliJson` plus the streaming parser can still be clobbered by later hook records. The PR is narrow, has focused tests and reviewer context, and the security pass found no supply-chain-sensitive file changes.

Keep this PR open and review, land, or adapt its shared `nextCliSessionId(parsed, current, backend)` approach in `src/agents/cli-output.ts`, with focused `parseCliJson`, `parseCliJsonl`, and `createCliJsonlStreamingParser` regression tests. Because this PR is already the implementation candidate for the still-present bug, the best path is maintainer review/merge rather than a duplicate fix PR.
