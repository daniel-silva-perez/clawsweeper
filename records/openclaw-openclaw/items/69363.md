## Summary

Keep this PR open. Current main still lacks hook-aware CLI session-id selection: parseCliJsonl can persist the first transient Claude hook id, while parseCliJson and the streaming parser can still be clobbered by a later hook id. The PR is a narrow, reviewed root-cause fix with focused tests and no apparent supply-chain or CI-sensitive changes.

## Best Possible Solution

Keep this PR open and land or adapt its shared nextCliSessionId(parsed, current, backend) approach in src/agents/cli-output.ts, with the focused parseCliJson, parseCliJsonl, and createCliJsonlStreamingParser regression tests. Review the existing PR rather than queueing a duplicate implementation while it remains open.
