## Summary

Keep this PR open. Current `main` still has the parser behavior the PR targets: `parseCliJsonl` keeps the first `session_id`, while `parseCliJson` and the streaming parser still allow later records to overwrite the stored session id. The PR discussion also shows a focused review cycle that broadened the fix to all three parser sites.

## Best Possible Solution

Keep this PR open and land or adapt its shared `nextCliSessionId(parsed, current, backend)` approach in `src/agents/cli-output.ts`, with focused regression tests for `parseCliJson`, `parseCliJsonl`, and `createCliJsonlStreamingParser`. Review the existing PR rather than queueing a separate duplicate fix while it remains open.
