## Summary

Keep open. Current main has improved the Telegram album path plumbing, but the issue’s remaining observable problems are still present by contract: shared media understanding still processes only the first matching image by default, and Telegram still populates MediaUrls with the same local paths as MediaPaths, which renders the duplicated `path | path` media header described in the report.

## Best Possible Solution

Keep this issue open and fix it in the Telegram/shared media path. A good fix would either make Telegram albums opt into all-image understanding or add a generic media-group policy that processes every image in an album, while also avoiding duplicated local path/url rendering when the two values are identical. Add a regression test that sends a 3-image Telegram album, verifies all 3 paths remain in the media header once each, and verifies image understanding produces `[Image 1/3]`, `[Image 2/3]`, and `[Image 3/3]` descriptions when configured.
