## Summary

Keep open. Current main preserves all Telegram album media paths, but the reported behavior is not fully resolved: Telegram still mirrors local paths into URL fields, the prompt formatter still renders duplicated `path | path` text, and image understanding still defaults to processing only the first image unless configured otherwise.

## What This Changes

The issue requests that Telegram messages with multiple images include every image path once in the media header and run image understanding for every image.

## Best Possible Solution

Make Telegram album behavior explicit: preserve all `MediaPaths`, suppress duplicate URL rendering when `MediaUrls[i]` equals `MediaPaths[i]`, and either opt Telegram image albums into all-image understanding with regression coverage or document/configure `tools.media.image.attachments.mode = "all"` as the supported path.

## Reproduction Assessment

Yes. A high-confidence reproduction path exists by source inspection: a Telegram album can produce multiple `MediaPaths`, mirrors those paths into `MediaUrls`, the prompt note renders both values, and media understanding selects only the first image by default.

## Solution Assessment

Unclear. Header de-duplication is a narrow maintainable fix, but making all Telegram album images transcribe by default would intentionally diverge from the shared documented first-attachment policy and needs maintainer judgment.
