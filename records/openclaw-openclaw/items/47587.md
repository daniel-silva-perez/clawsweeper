## Summary

Keep open. Current main preserves all downloaded Telegram album media paths and no longer strips image attachments after image descriptions, but it still does not satisfy the full report: image understanding defaults to selecting only the first image, and Telegram still mirrors local paths into MediaUrls, so the prompt formatter can render duplicate `path | path` media header text.

## Best Possible Solution

Keep this issue open and make an explicit maintainer decision for Telegram album defaults. A complete fix would keep preserving all Telegram `MediaPaths`, avoid rendering a duplicate URL segment when `MediaUrls[i]` equals `MediaPaths[i]`, and add focused regression coverage showing either all album images are described by default for Telegram or the documented/configured `tools.media.image.attachments.mode = "all"` path is the supported behavior.
