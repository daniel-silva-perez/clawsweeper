## Summary

Keep open. Current main fixes part of the original Telegram album path propagation problem, but not the full reported behavior: media groups now preserve all downloaded image paths, while shared media understanding still selects only the first image by default and Telegram still sets MediaUrls to the same local paths used in MediaPaths, which can render duplicate `path | path` header text.

## Best Possible Solution

Keep this issue open and decide the intended default for Telegram albums explicitly. A complete fix would preserve all Telegram `MediaPaths`, avoid rendering a duplicate URL segment when `MediaUrls[i]` equals `MediaPaths[i]`, and add a focused regression showing either all album images are described by default for Telegram or the documented/configured `tools.media.image.attachments.mode = "all"` path is the supported behavior.
