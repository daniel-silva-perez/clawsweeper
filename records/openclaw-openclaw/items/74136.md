## Summary

Keep this PR open. Current main still uses strict equality for `ANNOUNCE_SKIP`, so the reported multi-line final-token case is not implemented; the PR is a narrow canonical patch for the open bug and has supporting related-thread context.

## What This Changes

The PR broadens `isAnnounceSkip()` to accept `ANNOUNCE_SKIP` as the trimmed standalone final line of multi-line output and adds focused unit plus announce-flow regression tests.

## Best Possible Solution

Land this PR or an equivalent narrow patch that accepts `ANNOUNCE_SKIP` only as the trimmed standalone final line for announce suppression, keeps `REPLY_SKIP` strict, and aligns docs/changelog if maintainers want this to be a documented user-facing contract.

## Reproduction Assessment

Yes. The source-level reproduction is `isAnnounceSkip("DM summary block\nANNOUNCE_SKIP")` against current main: the helper's strict trimmed equality returns false, and the announce call sites then treat the text as deliverable output.

## Solution Assessment

Yes. The proposed helper change is the narrowest maintainable fix for the observed leak because it changes only announce suppression semantics, preserves exact-token behavior, and leaves reply suppression unchanged.
