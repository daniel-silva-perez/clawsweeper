## Summary

Keep open. Current main still lacks a DeepSeek provider thinking profile, so DeepSeek V4 Flash/Pro can shape `xhigh` or `max` once accepted, but the shared profile/validation path still does not advertise or accept `max` for the native DeepSeek provider.

## What This Changes

Request DeepSeek V4 Flash and Pro to expose `xhigh` and `max` thinking levels, mapping them to DeepSeek max reasoning effort while keeping `deepseek-reasoner` out of scope.

## Best Possible Solution

Add a DeepSeek-owned `resolveThinkingProfile` for `deepseek-v4-flash` and `deepseek-v4-pro` that exposes `off`, base levels, `xhigh`, and `max`, while leaving `deepseek-reasoner` on the base profile. Land that with focused provider/profile tests and any small docs update needed to keep the public thinking-level docs accurate.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: resolve/list thinking levels for `deepseek/deepseek-v4-pro` on current main and `max` will be absent because the provider has no profile hook and the fallback resolver never adds `max`.

## Solution Assessment

Yes. A provider-owned `resolveThinkingProfile` is the narrowest maintainable fix because the SDK/docs already define that extension point and the DeepSeek V4 transport path already maps `xhigh` and `max` to DeepSeek's max effort.
