## Summary

Keep open. Current main has DeepSeek V4 request shaping that can map a forced `xhigh` or `max` thinking level to DeepSeek `reasoning_effort: "max"`, but the DeepSeek provider does not expose a provider thinking profile, so the session/UI/directive layer still advertises only the base thinking ladder and rejects `xhigh`/`max` for DeepSeek models.

## Best Possible Solution

Add a DeepSeek-owned `resolveThinkingProfile` in `extensions/deepseek/index.ts` for `deepseek-v4-flash` and `deepseek-v4-pro` that includes `off`, base levels, `xhigh`, and `max`; add focused plugin/session or directive tests proving those levels are advertised and that `deepseek-reasoner` remains excluded.
