## Summary

Keep open. Current main still has DeepSeek V4 transport mapping for `xhigh`/`max` to DeepSeek `reasoning_effort: "max"`, but the DeepSeek bundled provider does not expose a provider thinking profile or legacy `supportsXHighThinking` hook, so UI/session/directive surfaces remain profile-limited and can reject those levels for DeepSeek.

## Best Possible Solution

Add a DeepSeek-owned `resolveThinkingProfile` in `extensions/deepseek/index.ts` for `deepseek-v4-flash` and `deepseek-v4-pro` that includes `off`, the base levels, `xhigh`, and `max`, while keeping `deepseek-reasoner` excluded. Add focused DeepSeek/provider-profile or directive/session tests proving those levels are advertised and accepted only for the V4 reasoning-effort models.
