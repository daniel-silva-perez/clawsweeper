## Summary

Current main still does not expose `xhigh` or `max` in the DeepSeek provider thinking profile. DeepSeek V4 transport can already map `xhigh`/`max` to `reasoning_effort: "max"`, but UI/session/directive validation is profile-driven and the DeepSeek plugin does not provide `resolveThinkingProfile` or `supportsXHighThinking`. The related open PR #73008 remains the likely implementation path rather than a reason to close this issue.

## Best Possible Solution

Keep this issue open and review/land a DeepSeek-owned fix, likely via #73008: add `resolveThinkingProfile` in `extensions/deepseek/index.ts` for `deepseek-v4-flash` and `deepseek-v4-pro` with `off`, base levels, `xhigh`, and `max`, keep `deepseek-reasoner` excluded, and add focused tests proving session/directive/profile exposure plus the existing transport mapping.
