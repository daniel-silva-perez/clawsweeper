## Summary

Keep this PR open. Current main still enumerates raw iMessage account IDs for gateway startup, while this PR adds fallback-default filtering only to `listEnabledIMessageAccounts()`, which is not the startup enumeration surface. The linked same-author bug #65141 remains open, and #65383 is a separate open implementation candidate rather than a merged superseding fix.

## What This Changes

The PR adds iMessage account-list filtering and tests to skip a fallback-only `default` account when named accounts exist while preserving `default` when it has distinct runtime fields.

## Best Possible Solution

Land one iMessage-owned fix that changes the account list actually used for gateway monitor startup, dedupes only fallback-only `default` watchers covered by an enabled named account on the same local backend, preserves truly distinct default runtimes, uses normalized account lookup, and adds focused regression tests for the reported config plus disabled-placeholder and distinct-runtime cases.
