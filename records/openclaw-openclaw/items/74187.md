## Summary

Keep this PR open. Current main still lacks the proposed Feishu direct `/new` quick action, 3s startup/probe timeout clamp, and WSClient timeout injection. The latest provided PR diff is Feishu-only after the bot-review feedback, but the WS timeout part likely needs correction because the Feishu SDK’s WebSocket connect-config request passes an explicit 15000ms timeout and the existing wrapper preserves explicit request timeouts.

## What This Changes

The PR changes Feishu Quick Actions `New session` from approval-card metadata to direct `/new` quick-command dispatch, lowers/default-clamps Feishu probe startup timeouts, injects a timeout-aware HTTP instance into Feishu WSClient construction, and adds focused Feishu tests.

## Best Possible Solution

Keep this PR open for maintainer review and contributor follow-up. If maintainers accept the direct `/new` UX and 3s startup/status policy, fix the WS timeout injection so the SDK `request({ timeout: 15000 })` path is actually clamped or overridden, add a regression test for that exact `request` shape, keep the probe timeout clamp documented, and coordinate with #63257, #67466, and #73998 before landing.
