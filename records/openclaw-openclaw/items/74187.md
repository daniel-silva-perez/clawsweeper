## Summary

Keep this PR open. Current main still lacks the Feishu-only behavior proposed here: `New session` still routes through the approval-card flow, startup/status probes still use longer unbounded caller timeouts, and the Feishu WS client still does not inject a timeout-bound `httpInstance` for token/connect-config HTTP calls. The latest PR diff is focused on Feishu files and no longer contains the earlier `chat.history` change, but the 3s timeout clamp is a public behavior change that needs maintainer review and coordination with related Feishu startup/reconnect work.

## What This Changes

The PR changes Feishu quick-action launcher routing for `New session`, shortens Feishu startup/probe timeouts, injects a short timeout-bound `httpInstance` into Feishu WSClient token requests, and adds targeted Feishu tests for those paths.

## Best Possible Solution

Keep this PR open for maintainer review. If maintainers accept the direct `/new` quick-action UX and the 3s probe/WS timeout policy, land the focused Feishu patch with the timeout clamp documented as intentional, coordinate it with #63257, #67466, and #73998, and run the targeted Feishu tests plus the changed gate in Testbox before merge.
