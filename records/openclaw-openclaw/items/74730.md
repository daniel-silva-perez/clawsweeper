## Summary

Keep open: the reported bug appears valid and the subagent forwarding change targets the right seam, but this PR is not mergeable as-is because it also carries unrelated broken gateway HTTP RPC wiring and lacks the focused seam regression test that should lock the model-forwarding behavior.

## What This Changes

This PR forwards the resolved `sessions_spawn` model as `provider`/`model` params to the gateway `agent` call, while also adding unrelated gateway HTTP RPC imports/stage wiring and a whitespace-only change.

## Best Possible Solution

Keep the useful model-forwarding fix, drop the unrelated gateway HTTP RPC changes from this PR, and add a focused regression test that mocks `callSubagentGateway` and asserts `splitModelRef(resolvedModel)` is forwarded as `provider`/`model` for explicit `sessions_spawn.model` overrides.
