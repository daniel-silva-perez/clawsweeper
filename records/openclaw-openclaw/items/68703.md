## Summary

Keep open. Current main still has the reported Discord guild-admin authorization gap, and the linked open fix PR uses closing syntax for this issue, so the right maintainer cleanup outcome is to keep the issue open until that PR or a replacement lands.

## What This Changes

The issue requests trusted requester identity and per-requester Discord guild permission checks before Discord guild-admin message actions execute bot-privileged mutations.

## Best Possible Solution

Land a narrow Discord-owned authorization fix, either by updating PR #68716 or replacing it, that marks guild-admin tool actions as trusted-requester-only, forwards the trusted Discord requester id, checks the matching Discord permission before bot-credentialed mutations, and explicitly preserves or redesigns trusted local CLI/manual operator behavior with tests.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists on current main: the dispatcher only enforces a plugin trusted-requester hook, Discord does not provide that hook for guild-admin actions, and the guild-admin runtime performs privileged mutations after feature gates only.

## Solution Assessment

Yes for the requested direction, but the open implementation needs maintainer policy review. The maintainable fix belongs in the Discord plugin boundary with permission checks analogous to moderation, while avoiding an accidental regression for trusted CLI/manual operator flows.
