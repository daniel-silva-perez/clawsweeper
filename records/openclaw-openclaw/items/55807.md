## Summary

Keep this issue open. Current main has narrowed the default ACP stream path so thread-context/thread-bound subagent requester sessions do not implicitly start a parent relay, and it also suppresses commentary-phase assistant deltas. However, the exact parent-stream relay still emits `:progress`, `:stall`, and `:resumed` system events whenever parent streaming is active, and open PR #55896 is still an unmerged implementation candidate with closing syntax for this issue.

## Best Possible Solution

Keep the issue open while maintainers review PR #55896 against current main. The best path is either to land or replace it with a narrow filter that suppresses visible Discord-thread parent relay `:progress`, `:stall`, and `:resumed` notices while preserving start/done/error/timeout and non-Discord behavior, or to explicitly close the PR as obsolete if the current stream-plan mitigation is proven to cover all reported paths.
