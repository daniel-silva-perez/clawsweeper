## Summary

Keep this issue open. Current main reduces the default chatter path by avoiding implicit parent streaming from thread-context or thread-bound requester sessions, and it suppresses commentary-phase assistant deltas, but it still does not implement the requested Discord-thread-specific suppression for active ACP parent relays. Open PR #55896 is also still open, unmerged, and explicitly proposes to close this issue, so policy says the issue should remain open while that implementation candidate is reviewed.

## Best Possible Solution
