## Summary

Keep this issue open. Current main has a mitigation that avoids implicit parent relays for thread-context or thread-bound subagent requester sessions, and it suppresses commentary-phase assistant deltas. However, the parent-stream relay still enqueues `:progress`, `:stall`, and `:resumed` system events whenever parent streaming is active, and open PR #55896 is still an unmerged implementation candidate with closing syntax for this issue.

## Best Possible Solution

Keep the issue open while maintainers review PR #55896 against current main. The best product/code path is either to land or replace it with a narrow filter that suppresses visible Discord-thread parent relay `:progress`, `:stall`, and `:resumed` notices while preserving start/done/error/timeout and non-Discord behavior, or to explicitly close the PR as obsolete if the current stream-plan mitigation is proven to cover both default and explicit parent-stream paths.
