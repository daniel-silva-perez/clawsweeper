## Summary

Keep this open. Current main has model pricing metadata, cost display, transcript-backed usage summaries, provider quota surfaces, and billing/rate-limit cooldowns, but it still lacks the requested pre-dispatch rolling spend ceiling for fallback chains. The discussion adds useful external evidence and workarounds, especially vibe-billing's request-level proxy and KinthAI's gateway middleware, but those do not satisfy a native OpenClaw guard.

## What This Changes

The issue requests a per-hour and per-day model-spend ceiling that can pause, reject, degrade, and notify before expensive fallback cascades run away.

## Best Possible Solution

Add a gateway/model-dispatch budget seam that is backward-compatible and documented: config for hourly/daily/per-agent and aggregate limits, pre-attempt estimated-cost checks before fallback candidates run, post-response reconciliation from actual usage, pause/reject/degrade/alert actions, status/doctor visibility, and regression tests for expensive rate-limit fallback chains.

## Reproduction Assessment

Not applicable as a feature request; the absence is source-verifiable rather than dependent on a flaky runtime reproduction. A focused check is to inspect config validation and `runWithModelFallback`, where current main has no accepted budget field or pre-attempt cost gate.

## Solution Assessment

No; current main's cost reporting and cooldown behavior are not the best solution because they act after usage is recorded or after provider errors. The maintainable fix should gate model dispatch before the expensive fallback request is sent, coordinated with the broader gateway budget design in #42475.
