## Summary

Keep this issue open. Current main has cost visibility, pricing metadata, diagnostics, Prometheus spend counters, and reactive billing/rate-limit failover cooldowns, but it still lacks the requested native request-level rolling spend ceiling for fallback chains. The open related budget issue #42475 overlaps and should inform the design, but it is broader and does not cleanly replace this issue's per-hour failover guard request.

## Best Possible Solution

Keep this issue open and coordinate it with #42475. The best implementation path is a native or explicitly supported gateway/model-dispatch budget seam with documented hourly/daily limits, per-agent and aggregate accounting, pre-attempt estimated-cost checks, post-response reconciliation, pause/reject/degrade/alert behavior, owner notification, status/doctor visibility, and regression coverage for expensive fallback chains.
