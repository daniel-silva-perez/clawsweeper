## Summary

Keep this issue open. Current main has pricing metadata, cost display, transcript-backed cost summaries, provider usage/quota surfaces, and billing/rate-limit failover cooldowns, but it still does not implement the requested request-level rolling spend ceiling for model fallback chains. The related #42475 is relevant design input, but it is broader and open, so it should not replace this failover-specific request without maintainer consolidation.

## Best Possible Solution

Keep this issue open and coordinate it with #42475. The best path is a native or explicitly supported gateway/model-dispatch budget seam with documented hourly, daily, per-agent, aggregate, and possibly per-session limits; pre-attempt estimated-cost checks; post-response reconciliation; pause/reject/degrade/alert behavior; owner notification; status/doctor visibility; and regression coverage for expensive fallback chains.
