## Summary

Keep this issue open. Current main has cost/pricing metadata, `/status` and `/usage cost` visibility, provider usage windows, and billing/rate-limit failover cooldowns, but it still lacks the requested request-level rolling spend ceiling for failover chains. Related #42475 is relevant but broader and still open, so it should not replace this narrower per-hour/failover guard without maintainer consolidation.

## Best Possible Solution

Keep this issue open and coordinate it with #42475. The best path is a native or explicitly supported gateway/model-dispatch budget seam with documented hourly, daily, per-agent, aggregate, and per-session limits; pre-attempt estimated-cost checks; post-response reconciliation; pause/reject/degrade/alert behavior; owner notification; status/doctor visibility; and regression coverage for expensive fallback chains.
