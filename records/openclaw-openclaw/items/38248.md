## Summary

Keep this issue open. Current main still provides cost visibility, pricing metadata, usage summaries, and reactive billing/quota failover cooldowns, but it does not implement a native request-level rolling spend ceiling for expensive fallback chains. The later related budget issue #42475 overlaps and should inform the design, but it is broader/different enough that it is not a clean canonical replacement for this issue’s per-hour failover guard request.

## Best Possible Solution

Keep this issue open and coordinate it with #42475. The best implementation path is a native or explicitly supported blocking budget seam in the model/fallback dispatch path, with documented config for hourly/daily limits, per-agent and aggregate accounting, pre-attempt estimated-cost checks, post-response reconciliation, pause/reject/degrade/alert behavior, owner notification, status/doctor visibility, and regression coverage for expensive fallback chains.
