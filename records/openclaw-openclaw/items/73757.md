## Summary

Keep this PR open. Current main still documents per-call `host=node` from `tools.exec.host=auto` as allowed while the runtime and tests reject it when a sandbox runtime is active, and this docs-only PR is a narrow alignment with that existing behavior.

## What This Changes

The PR updates `docs/tools/exec.md`, `docs/tools/exec-approvals.md`, and `docs/nodes/index.md` to state that `tools.exec.host=auto` rejects per-call `host=node` and `host=gateway` overrides while a sandbox runtime is active.

## Best Possible Solution

Land this docs-only PR, or apply equivalent wording to the same three docs pages, so the public docs match the already-tested exec routing contract; after that, the linked open bug #61009 can close as implemented.

## Reproduction Assessment

Yes. The mismatch is reproducible by comparing current docs lines with `resolveExecTarget({ configuredTarget: "auto", requestedTarget: "node", sandboxAvailable: true })`, which the current regression test expects to throw.

## Solution Assessment

Yes. The PR is the narrowest maintainable fix because the runtime behavior and regression coverage already exist; only the three stale docs paragraphs need to change.
