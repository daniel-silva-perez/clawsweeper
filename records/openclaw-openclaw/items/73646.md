## Summary

Keep this issue open. Current main still has the explicit `openclaw crestodian` no-TTY path log the error and return normally, while the wrapper only turns thrown errors into exit 1. An open implementation candidate, #73776, is linked with closing intent and is not merged, so the right cleanup path is to review/land or replace that PR rather than close the issue now.

## Best Possible Solution

Keep this issue open and review, rebase, land, or replace #73776. The concrete fix should make the explicit `runCrestodian()` no-TTY branch throw or otherwise propagate exit 1 through the CLI runtime, add regression coverage for `openclaw crestodian < /dev/null`, and preserve the `--message` path for headless automation.
