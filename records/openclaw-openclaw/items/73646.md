## Summary

Keep this issue open. Current main still has the reported explicit `openclaw crestodian` non-TTY path: `runCrestodian()` logs the TTY error and returns normally, while the command wrapper only exits non-zero when the action throws. There is now an open closing PR, #73928, so the right cleanup is to review/land that PR or an equivalent fix, then close through the merge.

## Best Possible Solution

Keep this issue open and review/land #73928 or an equivalent focused fix. The implementation should make the explicit `runCrestodian()` non-TTY branch fail with exit code 1, add regression coverage for stdin and stdout non-TTY cases, and preserve `--message`, `--json`, and the interactive TUI path.
