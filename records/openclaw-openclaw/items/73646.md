## Summary

Keep this issue open. Current main still has the explicit `openclaw crestodian` no-TTY branch log an error and return normally, so `runCommandWithRuntime` has no thrown error to convert into a non-zero exit. The linked implementation candidate #73776 is still the right path to review or replace before closing this issue.

## Best Possible Solution

Keep this issue open and review, rebase, land, or replace #73776. The concrete fix should make the explicit `runCrestodian()` no-TTY branch throw or set exit 1 through the CLI runtime, add regression coverage for `openclaw crestodian < /dev/null`, and keep `--message` working for headless automation.
