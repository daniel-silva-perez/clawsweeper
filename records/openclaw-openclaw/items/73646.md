## Summary

Keep this issue open. Current main still has the explicit `openclaw crestodian` no-TTY path print an error and return normally, so the command wrapper has no thrown error to convert into a non-zero exit. The linked implementation candidate #73776 is open and uses closing syntax, so the issue should remain open until that PR is reviewed and merged or replaced.

## Best Possible Solution

Keep the issue open and review/land or replace #73776. The best code path is to make the explicit `runCrestodian()` no-TTY branch throw or otherwise set exit 1 through the runtime, add regression coverage for `openclaw crestodian < /dev/null`, and preserve `openclaw crestodian --message ...` for automation.
