## Summary

Keep this issue open. Current main still has the explicit `openclaw crestodian` no-TTY path log an error and return normally, so the command wrapper has no exception to convert into exit 1. There is also an open unmerged implementation candidate, PR #73654, which references this issue with closing syntax; the best path is to review and land that PR or an equivalent fix before closing.

## Best Possible Solution

Keep the issue open and review PR #73654, or land an equivalent targeted fix on main: make the explicit `runCrestodian()` no-TTY path throw or otherwise set exit 1, add a regression test for explicit `openclaw crestodian` no-TTY semantics, and preserve the working non-interactive `--message` path.
