## Summary

Keep this issue open. Current main at 3de5476f5145046c0e597b3c76691b0beb8ffaab still has the explicit `openclaw crestodian` no-TTY branch log the error and return normally, so the command wrapper has no thrown error to turn into exit 1. There is also an open closing PR, #73654, so the right path is to review/land or close that PR rather than close the issue early.

## Best Possible Solution

Keep the issue open and review PR #73654 or land an equivalent fix on main: make the explicit `runCrestodian()` no-TTY path throw or otherwise set exit 1, add a regression test for `openclaw crestodian < /dev/null` semantics, and preserve the working `--message` one-shot automation path.
