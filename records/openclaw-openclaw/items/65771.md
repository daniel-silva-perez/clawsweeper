## Summary

Keep this PR open. Current main still does not implement the PR's central behavior: cron failure alerts omit the classified cause, run-log entries and the public cron.runs schema omit errorReason, CLI JSON output is still pass-through, and generated Swift protocol models have no errorReason field. The provided PR remains a focused implementation candidate, with follow-up commits addressing earlier enum and Swift initializer review concerns, but it should be rebased and revalidated against current main's expanded FailoverReason set before landing.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase it onto current main, preserve the additive errorReason run-log protocol field, cause-first failure alert display, run-log backfill/normalization, Swift initializer compatibility, and regression tests, then validate the final protocol and macOS/client surfaces before deciding whether to land or request CLI JSON compatibility changes.
