This PR adds `openclaw config patch` for JSON5 config-shaped patches from file/stdin, recursive object patching, null deletes, `--replace-path`, shared config mutation validation/dry-run handling, nested SecretRef checks, related CLI/channel/install docs, and small AGENTS/test/channel-loader maintenance edits.

Land a maintainer-reviewed version that keeps the `config patch` naming aligned with the gateway partial-update contract, preserves nested empty-object patch values, retains the stdin size/TTY and unused `--replace-path` safeguards, and passes the targeted config CLI tests plus docs and changed-gate validation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve nested empty-object patch values:** `src/cli/config-cli.ts:1027-1031`
  - body: Because `visit()` recurses into every plain object and then returns, a nested empty object never becomes an operation. A valid patch such as adding `agents.defaults.models["openai/gpt-5.4"]: {}` is silently ignored or makes an otherwise empty patch fail, even though empty model entries are a supported config shape. Please emit a merge/set operation for empty object values at non-root paths.
  - confidence: 0.91
- **[P3] Advertise the registered patch subcommand:** `src/cli/config-cli.ts:1797`
  - body: The top-level `openclaw config --help` summary says `get/set/apply/...`, but this branch registers `config patch` and the docs use `config patch`. Please change the summary to `patch` so users are not pointed at an unregistered subcommand.
  - confidence: 0.97
