## Summary

Keep open for maintainer review. This is an active, unmerged config write-surface PR with the protected `maintainer` label; current `main` does not already provide `openclaw config apply`, and the proposed patch still has concrete correctness and low-severity security issues around `--replace-path` and stdin handling.

## What This Changes

The PR adds an `openclaw config apply` CLI for JSON5 patch files/stdin with recursive patching, null deletes, `--replace-path`, shared validation/dry-run handling, tests, Slack/Discord/exe.dev/config docs, and a small AGENTS.md final-answer policy note.

## Best Possible Solution

Land only after maintainers decide the CLI/RPC naming contract, validate or reject every requested `--replace-path`, add bounded and TTY-safe stdin handling, keep the existing validation/SecretRef/atomic-write safeguards intact, and align the setup docs with the final semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reject unused replace paths:** `src/cli/config-cli.ts:1021-1025`
  - body: After the patch traversal finishes, none of the requested `--replace-path` values are checked against the paths that were actually matched. A typo therefore silently falls back to recursive patching and can leave old entries in a subtree the caller intended to replace exactly.
  - confidence: 0.9
- **[P3] Guard and bound stdin input:** `src/cli/config-cli.ts:915-921`
  - body: `readStdinText` awaits stdin until EOF and concatenates every chunk into one string without checking `process.stdin.isTTY` or enforcing a size limit. Running `--stdin` interactively hangs, and piping a very large patch can exhaust memory before JSON5 parsing reports an error.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The diff does not touch dependencies, workflows, lockfiles, or package publishing, but it introduces unbounded stdin buffering on a config mutation command.

Concerns:

- **[low] Unbounded stdin buffering:** `src/cli/config-cli.ts:915`
  - body: `config apply --stdin` reads all input into one string with no byte cap, so an accidental or hostile large stdin stream can drive excessive memory use and crash the local CLI process.
  - confidence: 0.9
