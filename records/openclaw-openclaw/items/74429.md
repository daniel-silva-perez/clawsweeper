## Summary

Keep open. Current main does not already provide this CLI capability, and the PR is an active config write-surface change with concrete review findings around `--replace-path` correctness and stdin guardrails before it should merge.

## What This Changes

The branch adds `openclaw config apply` for JSON5 config patches from file/stdin, recursive object patching, null deletes, explicit `--replace-path` replacement, shared validation/dry-run handling, tests, and config/channel/install docs.

## Best Possible Solution

Land a reviewed version only if maintainers want this CLI surface, with final naming aligned to the RPC contract, unmatched `--replace-path` values rejected before writes, bounded and TTY-safe stdin handling, and the existing validation, dry-run, SecretRef, unsupported-surface, and atomic-write safeguards preserved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Reject unused replace paths:** `src/cli/config-cli.ts:1021-1025`
  - body: `--replace-path` only affects traversal when the exact patch path is visited. If a caller mistypes the path, the patch still applies as recursive leaf updates and stale sibling keys remain, so an explicit replacement silently becomes a merge. Track consumed replace paths and fail before writing when any requested path was not matched.
  - confidence: 0.92
- **[P3] Add stdin guardrails:** `src/cli/config-cli.ts:915-920`
  - body: `readStdinText` waits forever on an interactive terminal and concatenates an unbounded stream into memory. Since the docs recommend `--stdin` for remote setup, add an `isTTY` error and a byte cap before parsing.
  - confidence: 0.84
