## Summary

Keep open. Current main at bdfb408ce6eeb724a638c106104dbbcae50b612c still does not implement the requested alias/per-model context profile. The repository has partial, adjacent controls: global bootstrap injection presets, per-agent skill allowlists, provider/model `compat.supportsTools`, and `localModelLean`, but there is no selected-model or alias-scoped surface that independently controls workspace file injection, skills prompt visibility, and tool prompt/tool schema exposure for `/model` switching.

## Best Possible Solution

Keep this open and implement a backward-compatible selected-model or alias-scoped context profile. The likely core path is to extend `AgentModelEntryConfig`, zod/generated schemas, docs, and config UI; pass the resolved model entry/alias into bootstrap file filtering, skills prompt assembly, and effective tool visibility; update `/context` and model/status reporting; and add focused tests for default behavior, strict config validation, workspace file allowlists, skills suppression, and tool suppression. `compat.supportsTools` should remain capability fallback metadata, not the whole user-controlled prompt policy.

- A future implementation needs a clear precedence model between `contextProfile.tools`, `compat.supportsTools`, global/per-agent tool policy, `toolsAllow`, `disableTools`, and `agents.defaults.experimental.localModelLean`.
