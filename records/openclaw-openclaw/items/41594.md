## Summary

Current main still does not implement alias/per-model context profiles for workspace file injection, skills prompt visibility, and tool exposure. The repo has adjacent global/per-agent/runtime controls, but the requested `/model`-scoped profile surface remains missing.

## Best Possible Solution

Keep this open and implement a backward-compatible selected-model or alias-scoped context profile. The likely path is to extend AgentModelEntryConfig, zod/generated schemas, docs, and config UI; pass the resolved model entry or alias into bootstrap file filtering, skills prompt assembly, and effective tool/schema exposure; and add focused tests for defaults, strict validation, workspace-file allowlists, skills suppression, and tool suppression. Treat compat.supportsTools as capability metadata, not the entire user-controlled prompt policy.
