## Summary

Keep this issue open. Current main still has no per-model or alias-scoped `contextProfile` for workspace bootstrap injection, skills prompt visibility, or tool exposure. Existing knobs are global, per-agent, provider-compat, or per-run policy controls, and they do not cover the requested opt-in alias profile surface.

## Best Possible Solution

Keep this issue open. Implement a backward-compatible selected-model or alias-scoped `contextProfile` on the model-entry/config surface, or fold it into the broader model-profile design if #59168 lands first. The change should thread the resolved selected model entry through bootstrap file filtering, skills prompt assembly, tool/client-tool schema exposure, and system prompt rendering, then update zod/generated schemas, public docs, Control UI, and focused tests for defaults, validation, `/model` switching, workspace allowlists, skills suppression, tool suppression, and precedence with existing policy controls.
