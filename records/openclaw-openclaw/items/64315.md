## Summary

Keep open. Current main still has static truncation and ordinary vertical scrolling, but no optional overflow-scrolling or marquee-style status component for constrained Control UI or TUI status surfaces; the related open topbar model/token visibility request is complementary status-content work, not a replacement for overflow behavior.

## What This Changes

Add optional scrolling or marquee-style display for long status information in constrained OpenClaw UI surfaces, with accessibility safeguards such as reduced-motion handling and pause on hover or focus.

## Best Possible Solution

Define a small accessible overflow-status text component or TUI helper, then opt it into a few high-value status surfaces after maintainer UI scoping. Related model/token visibility work such as #13670 can supply useful content, but the overflow behavior should remain a separate, reusable UI capability.
