## Summary

Keep open. Current main still has static ellipsis, wrapping, native selects, and ordinary vertical/manual scrolling, but no optional overflow-scrolling or marquee-style status component for constrained Control UI or TUI status surfaces; the related open topbar model/token request #13670 is complementary status-content work, not a replacement for overflow behavior.

## What This Changes

Add optional horizontal overflow scrolling for long status or info text in constrained OpenClaw UI surfaces such as Control UI/WebChat status areas, TUI header/footer/status lines, plugin status text, and compact banners.

## Best Possible Solution

Define a small accessible overflow-status text component or TUI helper that renders statically when content fits, exposes full text accessibly, respects reduced motion, and pauses on hover/focus or offers manual scrolling. After maintainer UI scoping, opt it into one or two high-value status surfaces; keep #13670 as separate content work that could later use the reusable overflow behavior.
