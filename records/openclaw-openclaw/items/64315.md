## Summary

Keep open. Current main still does not provide an optional overflow-scrolling or marquee-style status display for constrained Control UI or TUI status surfaces, and the related open topbar model/token request is complementary content work rather than a replacement for the overflow behavior.

## What This Changes

The issue requests optional scrolling or marquee-style display for long status information in constrained OpenClaw UI surfaces such as Control UI status bars, TUI footer/header areas, and plugin-provided status text.

## Best Possible Solution

Define a small accessible overflow-status text component or TUI helper, then opt it into one or two high-value status surfaces after maintainer UI scoping. Related model/token visibility work such as #13670 can provide useful content, but the overflow behavior should remain a separate reusable capability.

## Reproduction Assessment

Not applicable as a feature request rather than a reported failure. The current-main verification path is source inspection of the Control UI topbar, compact text CSS, chat controls, and TUI footer, all of which lack optional horizontal overflow scrolling.

## Solution Assessment

Unclear for implementation until maintainers choose surfaces and motion policy. A reusable opt-in helper with reduced-motion and pause behavior is the maintainable direction; an automated broad marquee rollout across UI surfaces would not be the best first fix.
