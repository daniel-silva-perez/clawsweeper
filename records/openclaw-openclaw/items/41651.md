## Summary

Keep open. Current main still realpath-checks each skill source independently, warns and skips symlink escapes before duplicate-by-name merging, and the tests/docs still describe and assert strict resolved-realpath containment. The issue also has an open implementation candidate with closing syntax in the provided GitHub context, so it should stay open until that PR or an equivalent focused fix is resolved.

## Best Possible Solution

Keep this issue open. Review and either land or close the open implementation candidates, especially haroldfabla2-hue/openclaw#2 and the referenced #45912, or implement an equivalent focused change in `src/agents/skills/workspace.ts` that preserves realpath containment while avoiding repeated warn-level noise for trusted duplicate symlink cases. Add regression coverage for a personal `.agents/skills` symlink pointing at a bundled or otherwise trusted configured skill root before closing.
