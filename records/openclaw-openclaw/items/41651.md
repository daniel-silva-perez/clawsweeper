## Summary

Keep open. Current main still performs the realpath containment check per skill source and emits a warn-level skills log before skill-name merging, so a personal `.agents/skills` symlink to an already-bundled skill can still produce the reported warning noise. The GitHub context also includes an open unmerged implementation candidate with closing syntax, so this issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open. Review haroldfabla2-hue/openclaw#2 or land an equivalent focused change in `src/agents/skills/workspace.ts` that preserves the realpath containment guard while suppressing or downgrading warnings when a personal `.agents/skills` symlink resolves outside its root but the same skill is already available from a trusted bundled or managed source. Add a regression test for that duplicate trusted-source symlink case before closing.
