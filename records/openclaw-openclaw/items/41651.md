## Summary

Keep open. Current main still performs per-source realpath containment before duplicate-by-name merging, still emits a warn-level escaped-symlink message, and docs/tests still assert strict resolved-realpath containment. The provided GitHub context also shows unresolved implementation candidates, including an open PR with closing syntax, so this should stay open until a focused fix lands or maintainers reject the behavior change.

## Best Possible Solution

Keep this issue open. Review and either land or close #45912 and the linked haroldfabla2-hue/openclaw#2 candidate, or implement an equivalent focused change in src/agents/skills/workspace.ts that preserves strict containment for untrusted escapes while avoiding repeated warn-level noise for duplicate trusted symlink cases. Add regression coverage for a personal .agents/skills symlink pointing at a bundled or configured trusted skill root, and update the public skills docs if the containment contract changes.
