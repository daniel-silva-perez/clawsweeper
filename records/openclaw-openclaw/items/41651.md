## Summary

Keep open. Current main still warns and skips symlinked skill paths when their realpath escapes the configured root, and the duplicate-by-name merge happens only after every source has already been scanned. The issue also has an open unmerged implementation candidate with closing syntax, plus a related open upstream PR, so this should remain open until a focused fix lands or maintainers reject that direction.

## Best Possible Solution

Keep this issue open. Review the open implementation candidates, especially openclaw/openclaw#45912 and haroldfabla2-hue/openclaw#2, or land an equivalent focused change in `src/agents/skills/workspace.ts` that preserves realpath containment while suppressing or downgrading the warning for trusted duplicate symlink cases. Add regression coverage for a personal `.agents/skills` symlink pointing at a bundled or otherwise trusted configured skill root before closing.
