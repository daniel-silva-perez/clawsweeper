## Summary

Keep open. Current main and latest release still concatenate the raw `identity.emoji` value into Feishu structured-card header titles in reply/streaming and outbound card paths, and the card builders serialize the supplied title verbatim. The adjacent `cardHeader` toggle PR is useful context, but it does not implement a sanitizer for descriptive non-emoji identity text.

## What This Changes

Prevent Feishu structured-card headers from displaying descriptive non-emoji text from an agent `identity.emoji` value when card-format messages are sent.

## Best Possible Solution

Add a Feishu-local shared card-header title resolver used by reply, streaming, presentation, and direct outbound card paths. It should preserve explicit presentation titles and valid identity names, include `identity.emoji` only when it is an emoji-only sequence, drop descriptive prose from identity-derived headers, and add focused Feishu regression tests.
