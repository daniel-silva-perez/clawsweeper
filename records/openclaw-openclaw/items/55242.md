## Summary

Keep open. Current main still builds Feishu card headers from the raw `identity.emoji` value plus the agent name, so descriptive non-emoji text in that field can still be rendered in long/card-message headers.

## What This Changes

The issue requests Feishu card-message headers stop displaying descriptive non-emoji text from agent `identity.emoji`, showing only the agent name or a pure emoji prefix.

## Best Possible Solution

Add a Feishu-local card-header title resolver used by reply, streaming, presentation, and direct outbound card paths. It should avoid rendering descriptive text from `identity.emoji`, preserve explicit presentation titles and any accepted `cardHeader` config semantics, keep global identity parsing unchanged, and cover descriptive CJK text plus pure-emoji identity cases in focused Feishu tests.
