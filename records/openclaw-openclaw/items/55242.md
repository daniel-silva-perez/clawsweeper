## Summary

Keep open. Current main still builds Feishu card headers from the raw `identity.emoji` value plus the agent name, so descriptive non-emoji text in that field can still be rendered in card headers.

## Best Possible Solution

Keep the issue open and fix it in the Feishu plugin. Centralize Feishu card-header title resolution, avoid rendering descriptive text from `identity.emoji` as a header prefix, preserve reasonable pure-emoji behavior, and add regression tests for reply streaming, static structured replies, and outbound card sends.
