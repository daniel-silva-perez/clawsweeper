## Summary

Keep this PR open. Current main does not implement the proposed native Plugin SDK MCP registration seam: there is no `api.registerMcpServer(...)` on `OpenClawPluginApi`, no plugin registry storage for plugin-owned MCP servers, no manifest `contracts.mcpServers`, and embedded Pi MCP config still materializes only merged bundle/user MCP config. The PR remains a meaningful implementation candidate for related #57355, but it adds a new managed stdio process execution surface and should go through maintainer/security review rather than conservative auto-cleanup.

## Best Possible Solution

Keep the PR open for maintainer review or rebase against current main. If accepted, the best implementation is an additive public SDK seam that validates stdio-only plugin MCP registrations, stores them in the plugin registry, honors current plugin activation/workspace state, merges them into embedded MCP config below user/operator config precedence, exposes manifest/status metadata, and updates docs plus contract/runtime tests. Related #57355 should remain the canonical feature request until an implementation lands.
