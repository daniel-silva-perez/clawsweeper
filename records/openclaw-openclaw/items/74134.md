## What This Changes

The PR adds a bundled file-transfer plugin, four file/directory node-host commands and agent tools, a plugin node-invoke policy SDK seam, media-store hardening, docs/generated baselines, dependencies, and tests for binary file operations on paired nodes.

## Best Possible Solution

Keep the PR open, fix canonical-path authorization so resolved paths and dir-fetch entries require a real allowlist match or a fresh canonical-path approval/deny decision, add regression coverage for `ask: "always"` with `followSymlinks=true`, then rerun the targeted file-transfer, gateway, SDK export/API, type, lint, and changed-surface gates for maintainer review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Require canonical policy to match the allowlist:** `extensions/file-transfer/src/shared/node-invoke-policy.ts:343-344`
  - body: The canonical-path checks continue whenever `evaluateFilePolicy()` returns `ok`, but `ask: "always"` returns `ok` before matching `allowReadPaths` or `allowWritePaths`. With `followSymlinks=true`, a symlinked requested path can be approved while the resolved target outside the allowlist proceeds without a distinct canonical-path approval. Require `reason === "matched-allow"` for canonical checks, or prompt/deny the canonical target before continuing; apply the same rule to write preflight, dir-fetch entry checks, and final postflight checks.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: Security review needs attention because canonical symlink targets can still bypass allowlist matching under `ask: "always"`.

Concerns:

- **[high] Canonical symlink targets can bypass file policy:** `extensions/file-transfer/src/shared/node-invoke-policy.ts:343`
  - body: The PR re-checks canonical paths after node preflight/postflight, but accepts any `ok` policy result. Since `ask: "always"` returns `ok` before allowlist matching, an approved symlink path can resolve to a target outside `allowReadPaths` or `allowWritePaths` without a separate canonical approval.
