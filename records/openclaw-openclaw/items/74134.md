## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Require canonical paths to match policy:** `extensions/file-transfer/src/shared/node-invoke-policy.ts:343-344`
  - body: The canonical-path checks treat any `ok` result from `evaluateFilePolicy()` as authorized. With `ask: "always"`, that function returns `ok` before matching `allowReadPaths` or `allowWritePaths`, so a symlinked requested path can be approved while the resolved target outside the allowlist proceeds without a second canonical-path approval. Require `reason === "matched-allow"` for canonical paths, or prompt/deny the canonical path before continuing; apply the same rule to dir-fetch entry checks and postflight canonical checks.
  - confidence: 0.88
