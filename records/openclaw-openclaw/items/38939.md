## Summary

Keep this PR open. Current main at 43da08979035503401a21a06a55e66920411ad67 still lacks Feishu dispatch-time arbitration across sibling bot accounts, so the duplicate-reply behavior remains plausible. The PR is not superseded by #39853 or #39001, but it should not be merged unchanged because the provided review context identifies a hardcoded coordinator account and substring implicit-mention fallback risks.

## Best Possible Solution

Keep this PR open, but require a corrected implementation or a replacement PR. The fix should arbitrate before dispatch using structured mentioned bot open_ids from Feishu text/post events, derive the coordinator from the configured default Feishu account or resolved route rather than a literal "main" account id, avoid substring-only implicit mention matching, preserve account-scoped dedupe for ordinary duplicates, and add focused regression tests for specialist-only, coordinator-plus-specialist, and multi-specialist group mentions. #39853 can complement this by exposing metadata to agents, but it is not a substitute for pre-dispatch arbitration; #39001 is separate merge_forward work.
