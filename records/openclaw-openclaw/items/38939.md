## Summary

Keep this PR open. Current main still lacks Feishu dispatch-time arbitration across sibling bot accounts, and the submitted PR is not superseded by the adjacent mention-metadata or merge-forward PRs. It should not merge unchanged because the patch hardcodes `main` as coordinator and adds an untested substring text-mention fallback.

## Best Possible Solution

Keep this PR open and request changes or a maintainer-approved replacement. The corrected fix should arbitrate before dispatch using structured mentioned bot IDs from Feishu text/post events, derive the coordinator from configured Feishu routing/default account rather than a literal `main`, avoid substring-only implicit matching, preserve account-scoped dedupe for ordinary duplicates, and add regression tests for specialist-only, coordinator-plus-specialist, and multi-specialist group mentions.

## Risks / Open Questions

- Current main still allows sibling Feishu bot accounts to process the same ordinary group event when single-agent dispatch is used, so the duplicate-reply problem remains unresolved.
- The submitted PR can drop messages in deployments whose primary/coordinator Feishu account is not literally `main`, because the arbitration coordinator is hardcoded.
- The submitted PR adds substring-based implicit mention matching without focused tests, so plain text such as `@maintenance` or `@mainbot` could unexpectedly trigger `@main` fallback behavior.
- No obvious supply-chain or CI-permission regression is present in the PR diff; the security-relevant concerns are availability/routing correctness and unintended dispatch.
