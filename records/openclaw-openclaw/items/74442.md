## Summary

Keep open. Current main still has the hidden-run lifecycle session-key gap, and this is an active implementation PR for a real gateway persistence bug; the remaining decision is maintainer review of the lifecycle identity visibility boundary plus normal CI, not conservative cleanup closure.

## What This Changes

The PR changes `emitAgentEvent()` so hidden `lifecycle` events retain `sessionKey`, adds regression coverage for hidden lifecycle versus hidden assistant events, and adds a changelog entry.

## Best Possible Solution

Land or adapt a narrow lifecycle identity fix that lets gateway persistence write terminal session state for hidden runs while keeping hidden assistant, tool, and generic error streams redacted; if the privacy boundary is too broad, carry the lifecycle session key through an internal persistence path instead of broad client broadcasts.
