## Summary

Keep open. Current main still lacks the useful tools-vs-CLI clarification, but this PR is not correct as-is because it documents invalid `openclaw sessions list` usage and repeats the built-in-tools intro sentence.

## What This Changes

This PR adds a `docs/tools/index.md` callout explaining that tool identifiers such as `sessions_list` and `agents_list` are not shell commands, with terminal examples for sessions and agents commands.

## Best Possible Solution

Land a narrow docs update in `docs/tools/index.md` that explains runtime tool identifiers are not standalone shell commands, uses `openclaw sessions` and `openclaw agents list` as the shell examples, points readers to `openclaw --help` and per-command help, and keeps the existing built-in-tools table intro only once.

## Reproduction Assessment

Yes. Compare the PR's new shell examples to current main: `docs/cli/sessions.md` and `register.status-health-sessions.ts` show `openclaw sessions` is the listing command, while no `sessions list` child is registered.

## Solution Assessment

No. The requested clarification is a good docs direction, but this exact patch is not the best way to solve it until the sessions example is corrected and the duplicate intro sentence is removed.
