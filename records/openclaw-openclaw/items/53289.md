## Summary

Keep open. Current main still does not include the PR’s central tools-vs-CLI clarification, and the proposed branch is not ready as-is because it documents invalid `openclaw sessions list` usage while duplicating the built-in-tools intro sentence. The change is docs-only and has no observed security-sensitive surface.

## Best Possible Solution

Revise this PR or apply an equivalent docs update in `docs/tools/index.md`: add a concise callout that `sessions_list`, `sessions_spawn`, and `agents_list` are agent/runtime tool identifiers, point shell users to `openclaw --help` and the CLI reference, use `openclaw sessions` plus `openclaw agents list` as examples, and remove the duplicate built-in-tools sentence before merge.
