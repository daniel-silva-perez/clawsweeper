## Summary

Keep open. Current main still does not include the PR’s central docs clarification that `sessions_list` and `agents_list` are runtime tool identifiers rather than shell commands. The PR is useful but needs revision before merge because its proposed `openclaw sessions list` example does not match the current CLI surface.

## Best Possible Solution

Revise the PR or apply an equivalent docs update in `docs/tools/index.md`: add a concise callout that tool IDs such as `sessions_list` and `agents_list` are agent/runtime tool names, point shell users to `openclaw --help` and `/cli`, use `openclaw sessions` plus `openclaw agents list` as the terminal examples, and remove the duplicate built-in-tools sentence before merge.
