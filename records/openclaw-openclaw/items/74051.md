## Summary

Keep open. Current main still has the linked memory-dreaming failures this PR is trying to fix, but the latest PR head still has an actionable relocation edge case that can promote unrelated daily-note lines when a moved snippet is embedded several lines away from the stale range.

## What This Changes

The PR changes memory-core light dreaming ordering, short-term promotion rank/apply gates, live-note relocation, docs, and regression tests so daily-only signals can affect scoring without satisfying the recall-count gate.

## Best Possible Solution

Keep the recall-first ordering and recall-or-grounded gate contract if maintainers agree with that product behavior, but tighten relocation so containment matches promote only the tight live range or extracted matching line before distance tie-breaks. Then keep the docs/tests aligned and land after targeted memory-core tests plus the changed-lane gate.

## Reproduction Assessment

Yes. A focused relocation case reproduces by inspection: stale candidate start line 1, target snippet embedded on line 5, and a 20-line scan first records lines 1-5 as a quality-2 distance-0 match; the real one-line containing range has distance 4 and cannot replace it under the current condition.

## Solution Assessment

No. The rank/apply gate direction is narrow and maintainable, but the relocation fix is not yet the safest version because it preserves proximity before finding the actual containing subrange for moved embedded snippets.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Tighten moved containment relocation before promoting snippets:** `extensions/memory-core/src/short-term-promotion.ts:1477-1478`
  - body: The quality-2 branch only lets a smaller containing span replace the current match when its start is within one line of the best distance. With the new 20-line scan, a stale candidate whose original start is line 1 and whose snippet moved into an embedded line 5 first matches the whole line1-5 window with distance 0; the real one-line containing range has distance 4 and cannot replace it. That promotes intervening daily-note lines into MEMORY.md, reintroducing the adjacent-line privacy bug this PR is trying to avoid. Please choose or extract the tight containing range for containment matches before applying the distance tie-breaker.
