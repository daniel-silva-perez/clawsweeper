## Summary

Keep open. Current main still has the reported memory-core behavior, while the latest PR head appears to address the linked bug and prior relocation privacy feedback; the remaining action is maintainer review and merge judgment, not cleanup closure or a replacement PR.

## What This Changes

The PR changes memory-core light dreaming ordering, short-term promotion rank/apply gates, relocation matching, regression tests, and memory/dreaming docs so daily-ingestion-only signals can still score candidates without satisfying the recall-count gate.

## Best Possible Solution

Land this PR, or an equivalent focused patch, if maintainers agree that minRecallCount should mean recall-or-grounded evidence while daily-ingestion signals remain available for scoring; the linked issue should close through the existing closing reference after merge.

## Reproduction Assessment

Yes. Current main source and tests reproduce the mechanism statically: daily/session ingestion refreshes daily signals, Light sorts by lastRecalledAt before recallCount, and rank/apply both gate on aggregate signalCount including dailyCount.

## Solution Assessment

Yes, pending maintainer agreement on the behavior contract. The PR changes the two gate sites and the Light ordering directly, keeps daily evidence in scoring, and adds targeted relocation safeguards instead of discarding daily ingestion entirely.
