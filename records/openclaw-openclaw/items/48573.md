## Summary

Keep open. Current main still has only partial cleanup: native subagents have stale-run recovery, and ACP task maintenance can mark tasks lost only when the child session metadata disappears or can close terminal parent-owned ACP sessions. The reported active zombie class remains possible when a persisted ACP child session row still exists but the owning runtime/process is gone, because maintenance treats that row as authoritative backing state.

## Best Possible Solution

Keep this issue open until a merged fix adds process-aware or lease/heartbeat-based parent-child reconciliation for ACP and native subagent runs. The fix should cover parent abort, timeout, gateway crash/restart, cascade cancellation, and stale session metadata cleanup, with regression tests proving that still-present but orphaned ACP session rows no longer keep tasks alive indefinitely. PR #68718 may be the broader implementation path if it passes review; otherwise a narrower fix belongs in the existing ACP/task maintenance surfaces.
