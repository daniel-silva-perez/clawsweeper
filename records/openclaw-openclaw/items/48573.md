## Summary

Keep open. Current main has useful partial hardening for native subagent stale-run cleanup, restart recovery, task cancellation, and terminal ACP session cleanup, but it does not fully solve the reported parent-termination/zombie-state class. ACP task maintenance still treats persisted child session metadata as backing state, so it does not reconcile OS/runtime process death when the session row remains. The earlier focused PR #48583 was closed unmerged, and the broader durable queue/cascade-cancel work in #68718 is still open and not present on main.

## Best Possible Solution

Keep this issue open until a merged fix adds process-aware or lease/heartbeat-based parent-child reconciliation for both native subagents and ACP sessions, with regression coverage for parent abort, timeout, gateway crash/restart, cascade cancellation, and stale session metadata cleanup. #68718 may be the canonical implementation path if it passes review; otherwise a narrower fix should land in the existing subagent/task maintenance surfaces.
