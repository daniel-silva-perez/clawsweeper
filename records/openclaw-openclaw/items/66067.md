## Summary

Keep this PR open. Current main still newline-joins TUI thinking fragments, the streaming and restored-history paths consume that unnormalized formatter output, and the existing tests still lock the old newline behavior. The PR remains a focused implementation candidate for the TUI whitespace bug linked from #66008, with the remaining reviewer feedback around CRLF/bare-CR handling still worth addressing before merge.

## Best Possible Solution

Keep this PR open as the implementation candidate for the TUI show-thinking whitespace bug.
