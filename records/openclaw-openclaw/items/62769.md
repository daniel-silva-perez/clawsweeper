## Summary

Current main still does not support top-level configured ACP bindings for Telegram DMs. The Telegram configured-binding adapter is explicitly topic/group-oriented, docs describe Telegram persistent ACP bindings as forum-topic only, and current DM ACP coverage is for runtime `/acp spawn ... --thread here` bindings rather than config-driven `bindings[]`.

## Best Possible Solution

Keep this open and implement it in the bundled Telegram plugin’s configured-binding adapter: accept `match.peer.kind: "direct"` positive peer ids, match inbound DM conversation ids from `resolveTelegramDirectPeerId()`, add regression tests for top-level Telegram DM ACP `bindings[]`, and update the Telegram and ACP Agents docs to distinguish configured DM bindings from runtime `/acp spawn` bindings.

## Evidence
