## Summary

Keep open. Current main still does not provide configurable provider-level LLM retry, and the PR is a plausible active implementation, but the latest branch still has blocking correctness gaps around config reload cleanup, existing request timeout plumbing, and generated public config schema drift.

## What This Changes

The PR adds opt-in `models.providers.<name>.retry` config, provider retry runner registration, provider retry policy/tests, abort-aware retry support, OpenAI-compatible transport wrapping, and config help/label/type/schema updates.

## Best Possible Solution

Land one canonical provider-retry implementation that is opt-in, survives config reloads including removal of all providers, preserves existing provider request timeout behavior, updates generated config/schema/docs surfaces, and includes focused regression tests for retry policy, provider registry lifecycle, OpenAI-compatible transports, and config schema generation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Clear retry runners before zero-provider skips:** `src/agents/models-config.providers.normalize.ts:35`
  - body: `clearProviderRetryRunners()` is only called from `normalizeProviders`, but `planOpenClawModelsJson` returns `skip` for zero resolved providers before calling that function. Removing all providers can leave stale module-scoped retry runners active until restart, so clear the registry on the empty-provider planning path too.
  - confidence: 0.9
- **[P2] Preserve OpenAI SDK request options:** `src/agents/openai-transport-stream.ts:714`
  - body: Current main passes `buildOpenAISdkRequestOptions(model, options?.signal)` to these SDK calls so provider `timeoutSeconds` reaches each request. The wrappers here pass only `signal`, which would drop per-request timeouts when rebased; use the existing helper inside each wrapped stream factory.
  - confidence: 0.86
- **[P2] Regenerate the public config schema:** `src/config/zod-schema.core.ts:342`
  - body: Adding `models.providers.*.retry` changes the public config surface, but the generated base schema on the branch has no retry entries and `buildConfigSchema()` serves that generated payload. Regenerate and commit the base config schema and config-doc baseline so schema consumers see the new field.
  - confidence: 0.84
