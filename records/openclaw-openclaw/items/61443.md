## Summary

Current main does not implement the PR's intended configurable `models.providers.<id>.retry` behavior. Main only has shared/channel retry helpers plus provider SDK retry-after caps/failover behavior, while model-provider config types, schema/help metadata, and OpenAI-compatible transport call sites do not expose or consume provider-level retry config. The PR remains potentially useful, but it needs normal review against the related open retry PR and the unresolved review concerns before it can be merged or closed.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to consolidate the LLM retry design with #38640 and either land a provider-level retry implementation that updates `ModelProviderConfig`, strict zod schema/help metadata, transport wiring, docs, and regression tests, or close the weaker retry PR after maintainers choose the canonical approach. Before merge, address the remaining review points around abort-aware backoff, canonical provider IDs, and `Headers`-based `Retry-After` parsing.
