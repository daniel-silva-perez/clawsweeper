## Summary

Keep open. Current main does not already implement configurable provider-level retry, and this PR is a useful active implementation, but it is not merge-ready because it still misses the empty-provider registry cleanup path, drops current OpenAI SDK request timeout options, and omits generated public schema updates. The overlapping open #38640 also means maintainers should choose one canonical retry design before landing either branch.

## What This Changes

The PR adds opt-in `models.providers.<name>.retry` config, provider retry registry/policy code and tests, abort-aware retry support, OpenAI-compatible transport wrapping, and config help/label/type/schema updates.

## Best Possible Solution

Land one canonical provider-retry implementation that is opt-in, handles config reloads including removal of all providers, preserves existing provider timeout/request-option behavior, updates generated config and docs surfaces, and includes focused regression coverage for registry lifecycle, retry policy, OpenAI-compatible transports, and schema generation.

  - body: `clearProviderRetryRunners()` only runs inside `normalizeProviders`, but the planning path returns `skip` for an empty provider map before calling that function.   - confidence: 0.86
