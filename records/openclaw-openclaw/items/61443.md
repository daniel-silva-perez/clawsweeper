## Summary

Keep open. Current main still lacks configurable provider-level retry, and this PR is a live implementation path, but it is not merge-ready: the branch misses an empty-provider cleanup path, is behind current main's OpenAI SDK timeout plumbing, and does not update generated config schema artifacts for the new public config surface. The overlapping open #38640 should be reconciled by maintainers rather than auto-closing either implementation.

## What This Changes

The branch adds opt-in `models.providers.<name>.retry` config, a provider retry runner registry and policy, abort-aware retry support, OpenAI-compatible transport wrapping, config help/labels/types/schema, and focused retry tests.

## Best Possible Solution

Update this implementation or a single canonical replacement so provider retry is opt-in, scoped correctly through config reloads, preserves existing timeout/request-option behavior, updates generated config surfaces and docs, and has targeted regression coverage before #38640 is resolved.

- **[P2] Clear retry runners before zero-provider skips:** `src/agents/models-config.providers.normalize.ts:35`
  - body: `clearProviderRetryRunners()` runs only inside `normalizeProviders`, but `planOpenClawModelsJson` returns `skip` when provider resolution is empty before calling that function. If a reload removes all providers, the old module-scoped runners remain active until restart. Move the clear into the planning path before the empty-provider return or otherwise clear the registry on that skip path.
  - confidence: 0.9
- **[P2] Preserve OpenAI SDK request options:** `src/agents/openai-transport-stream.ts:714`
  - body: Current main passes `buildOpenAISdkRequestOptions(model, options?.signal)` to these SDK calls so provider `timeoutSeconds` reaches each request. The PR branch wraps stream creation but passes only `signal`, which would drop the per-request timeout when this is rebased. Use the existing helper inside each `createStream` wrapper.
  - confidence: 0.86
- **[P2] Regenerate the public config schema:** `src/config/zod-schema.core.ts:342`
  - body: Adding `models.providers.*.retry` changes the public config surface, but the generated base schema on the branch has no retry entries and `buildConfigSchema()` serves that generated payload. Regenerate and commit the base config schema and config-doc baseline so schema consumers and checks see the new field.
  - confidence: 0.84
