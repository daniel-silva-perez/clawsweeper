## Summary

Keep this PR open. Current main does not implement the PR's proposed suppression rule: mutating-tool warnings are still shown when the assistant merely produced a successful-looking reply, and tests explicitly preserve that behavior. The actual same-action recovery path already clears stale mutating failures when the retry succeeds, while broader warning-delivery policy remains tracked by related maintainer work.

## Best Possible Solution

Keep this PR open for maintainer review or contributor revision. A safe path should distinguish proven recovery from unresolved mutating failure: keep warnings visible for success-looking or ambiguous replies, suppress only when the failed action is explicitly acknowledged or the same mutating action later succeeded and cleared `lastToolError`, and leave the broader configurable delivery-policy decision to #39406.
