## Summary

Keep open. The optimization is still valid, but current main has moved the active builtin memory read path into the memory-host SDK package and still reads the whole file before slicing; the PR branch edits obsolete `src/memory` files and is currently not mergeable. A narrow replacement or branch refresh is the right path.

## What This Changes

The PR branch changes the old `src/memory` read path to stream `from`/`lines` reads with `createReadStream` and `readline`, and adds a test asserting windowed reads do not call `fs.readFile`.

## Best Possible Solution

Apply the streaming partial-read optimization to the active memory-host SDK read path, keep QMD’s existing streaming behavior unchanged, add builtin regression coverage for avoiding `fs.readFile` and missing-file-during-stream behavior, then close or supersede the source PR after the replacement is available and credited.
