## Summary

Keep open. Current main still full-reads builtin memory files before slicing line windows, while QMD already has a separate streaming path. The PR remains a useful narrow performance fix, but it is stale against the old `src/memory` layout and needs to be refreshed into the current memory-host SDK surface before merge.

## What This Changes

The PR branch modifies the pre-SDK `src/memory` read path to stream `from`/`lines` reads line-by-line, stop after the requested window, and adds a test asserting windowed reads do not call `fs.readFile`.

## Best Possible Solution

Refresh this PR into the current memory-host SDK read path: stream explicit `from`/`lines` builtin reads without `fs.readFile`, keep full/default excerpt behavior and metadata unchanged, preserve QMD’s existing streaming path, and add focused builtin tests for no-full-read and missing-file-during-stream cases.
