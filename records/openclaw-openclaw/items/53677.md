## Summary

Keep open. The performance idea is still valid, but the PR branch edits obsolete `src/memory` files while current `main` routes builtin `memory_get` reads through the memory-host SDK, which still reads the full file before slicing. A branch refresh or narrow replacement PR is needed.

## What This Changes

The PR changes the old memory read path to stream explicit `from`/`lines` requests line by line and adds a test asserting windowed reads do not call `fs.readFile`.

## Best Possible Solution

Apply the streaming partial-read optimization to `packages/memory-host-sdk/src/host/read-file.ts`, reuse the shared slice-result helper so existing response metadata stays aligned, leave QMD production behavior unchanged, add builtin regression coverage for no full-file read and stream-time missing-file behavior, then supersede this PR with credit once the replacement is available.
