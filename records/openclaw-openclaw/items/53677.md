## Summary

Keep open. Current main does not implement the PR’s intended optimization: the active builtin memory read path still reads the whole markdown file before slicing, while the PR branch edits obsolete pre-SDK files. The proposed idea remains useful and security review did not find unexplained supply-chain or permission-sensitive changes, but it needs to be refreshed into the current memory-host SDK path before merge.

## What This Changes

The PR branch changes the old `src/memory` read path to stream `from`/`lines` reads with `createReadStream` and `readline`, and adds a test spy asserting windowed reads do not call `fs.readFile`.

## Best Possible Solution

Refresh the optimization into the active memory-host SDK read path and its source mirror, keep QMD’s existing streaming path intact, and add builtin tests for no-full-read windowed access plus missing-file-during-stream behavior. If the original branch cannot be updated cleanly, create a narrow replacement PR that credits this contribution.
