## Summary

Keep this issue open. Current main still lacks the requested structured sessions_send action/outcome and explicit announce routing override contract; the A2A handoff flow remains driven by exact REPLY_SKIP / ANNOUNCE_SKIP assistant text. ThreadId propagation is partially covered, but the maintainer discussion explicitly keeps this as acceptance criteria for broader A2A handoff work.

## Best Possible Solution

Keep this issue open as acceptance criteria for first-party A2A handoff work, coordinated with #63789, #27445, and #44309. The best implementation is a backwards-compatible structured sessions_send outcome/action contract covering skip/no-reply/announce suppression, explicit routing overrides, legacy token compatibility, and regression tests for token chatter plus threaded announce routing.
