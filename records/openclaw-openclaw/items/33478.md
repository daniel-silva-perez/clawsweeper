## Summary

Keep this issue open. Current main still lacks a structured sessions_send action/outcome or explicit announce routing override, while the legacy A2A lifecycle still depends on exact REPLY_SKIP / ANNOUNCE_SKIP assistant text. ThreadId inheritance for announce delivery is partly fixed and tested, but the maintainer discussion explicitly says this should remain acceptance criteria for broader A2A handoff work.

## Best Possible Solution

Keep this issue open as acceptance criteria for first-party A2A handoff work. The best implementation is a backwards-compatible structured sessions_send outcome/action contract covering skip/no-reply/announce suppression, explicit announce routing overrides, legacy token compatibility, and regression tests for token chatter plus threaded announce routing, coordinated with #63789, #27445, and #44309.
