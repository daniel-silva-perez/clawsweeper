## Summary

Keep this issue open. Current main still lacks the requested structured `sessions_send` action/outcome and routing override contract, while A2A control flow still depends on exact `REPLY_SKIP` / `ANNOUNCE_SKIP` assistant text. Thread announce routing is partially repaired and tested, but the maintainer discussion explicitly keeps this as acceptance criteria for broader A2A handoff work.

## Best Possible Solution

Keep this issue open as acceptance criteria for first-party A2A handoff work, coordinated with #63789, #27445, and #44309. The best implementation is a backwards-compatible structured `sessions_send` outcome/action contract covering skip/no-reply/announce suppression, explicit routing overrides, legacy token compatibility, and regression tests for token chatter plus threaded announce routing.
