## Summary

Keep this PR open. Current main still does not implement the proposed `publicMode` config surface, public-facing prompt/runtime redaction, skills snapshot suppression, or `/v1` owner-trust header behavior. The PR remains a relevant core gateway/agent trust-boundary implementation candidate, but it needs maintainer review and rebase against the newer compat HTTP operator-scope contract.

## Best Possible Solution

Keep the PR open. The best path is to rebase it on current main and either integrate with the current compat HTTP operator-scope model or justify a separate owner-header contract, then finish the core implementation: add documented per-agent and default `publicMode` config/schema support, resolve mode from the effective routed session agent including default-only configs, redact runtime/workspace/docs/sandbox paths in prompts, suppress skills snapshots for public-mode agents, and cover both streaming and non-streaming `/v1/chat/completions` and `/v1/responses` paths with targeted tests.
