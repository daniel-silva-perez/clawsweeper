## Summary

Keep open. Current main still has stronger Discord gateway lifecycle handling and post-event inbound queueing, but I did not find a Discord-owned READY/reconnect catch-up path that fetches recent channel history and replays messages missed before MESSAGE_CREATE reaches the listener. The prior ClawSweeper comment remains accurate against current main 955b4df0939f90b4b6de5e7e6f3149dd125e1e50.

## Best Possible Solution

Keep this open and implement the fix inside the Discord plugin. The likely path is a Discord-owned READY/reconnect catch-up pass: maintain a per-account/channel last-seen message cursor, fetch recent messages via Discord REST after reconnect/READY, dedupe with the existing replay key shape across live and replayed events, and feed eligible messages through createDiscordMessageHandler/preflight so current authorization and routing behavior is preserved.
