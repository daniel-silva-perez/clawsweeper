## Summary

Keep this PR open. Current main does not implement the PR’s intended three-layer stuck-session defense: default transcript policy still disables synthetic tool results for unowned/default providers, tool definitions still call tools without a per-call timeout wrapper, config has no `agents.defaults.toolCallTimeoutSeconds`, and the reply runner still waits indefinitely on `Promise.allSettled(pendingToolTasks)`. The related #53889 bug remains open, while #64733 was closed unmerged and is explicitly superseded by this broader PR.

## Best Possible Solution

Keep this PR open for review. The best path is to validate or revise the proposed defense-in-depth implementation against current main, especially provider-specific synthetic result compatibility, per-tool timeout/abort semantics, config docs/schema metadata, and the pending-tool drain guard, then land it or split it into smaller reviewed PRs if maintainers prefer.
