## Risks / Open Questions

- Changing `allowSyntheticToolResults` to true for every provider could affect provider-owned and custom-provider replay semantics; this needs explicit maintainer/provider compatibility review.
- A default 60s per-tool-call timeout is a broad runtime behavior change and could abort legitimate long-running tools unless docs, config metadata, and opt-out semantics are aligned.
- The PR is partly stale because current main already owns the pending-tool-task drain layer; the branch should remove or rebase the superseded Layer 3 pieces before review.
