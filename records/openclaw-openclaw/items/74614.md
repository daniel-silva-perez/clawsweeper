## Summary

Keep open. Current main still contains the reported early return in the gateway chat lifecycle finalization path, current tests do not cover the `peek()` succeeds / `shift()` misses case, and the linked fix PR #74678 is closed without being merged.

## What This Changes

A terminal lifecycle event should emit a final chat event via the already-resolved session/run fallback even when the chat run registry entry disappears between `peek()` and `shift()`.

## Best Possible Solution

Make the gateway lifecycle finalizer fall back to the already-resolved `sessionKey` and `eventRunId` when `registry.shift(evt.runId)` misses, then cover that exact branch with a focused gateway regression test and leave broader timeout/no-terminal-event symptoms to their own tracking unless separately proven fixed.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to enter `finalizeLifecycleEvent` with `chatLink` truthy and force `chatRunState.registry.shift(evt.runId)` to return `undefined`; current main then returns before emitting the terminal `chat` event.

## Solution Assessment

Yes. The narrow gateway fallback is the best current direction because it fixes the missing terminal event at the producer, instead of adding another UI/TUI workaround after the final event has already been suppressed.
