## Summary

Keep open. Current main has substantial partial support for config-injected and forward-compatible models, including runtime fallback resolution, catalog/list visibility, provider hooks, and tests. It still does not implement the RFC's central product invariant: one shared resolved-model descriptor consumed consistently by status, list, runtime, capability gates, picker/auto-reply consumers, availability, and docs.

## Best Possible Solution

Keep this issue open as the umbrella design tracker until maintainers land or explicitly scope a shared resolved-model descriptor. The likely implementation should unify registry entries, models.providers entries, provider dynamic/forward-compat hooks, transport, capabilities, provenance, and availability, then migrate models status, models list, runtime execution, picker/auto-reply consumers, thinking gates, and docs to consume that one result.
