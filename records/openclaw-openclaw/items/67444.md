## Summary

Keep this PR open. Current main still suppresses ANNOUNCE_SKIP before the completion-delivery call, so the PR is not obsolete. The submitted change also still needs maintainer review because its fallthrough path can forward the literal ANNOUNCE_SKIP control token as completion content unless the implementation is adjusted during rebase.

## Best Possible Solution

Keep this PR open for maintainer review, but require a follow-up patch before merge. Rebase onto current main, preserve NO_REPLY suppression, allow ANNOUNCE_SKIP to skip only the soft announce step when completion delivery is required, clear or replace the control token before building findings, add positive coverage for the actual completion-delivery call, and update docs if maintainers accept the new semantics.
