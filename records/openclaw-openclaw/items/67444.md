## Summary

Keep this PR open. Current main still treats an ANNOUNCE_SKIP subagent reply as a terminal suppression before completion delivery, and both tests and docs still encode that behavior. The PR is therefore not obsolete, but the submitted implementation still needs maintainer review because the fallthrough can forward ANNOUNCE_SKIP as completion content unless the control token is cleared or replaced.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase it onto current main, preserve NO_REPLY suppression, allow ANNOUNCE_SKIP to skip only the soft announce step when completion delivery is required, clear or replace the control token before building findings, add positive delivery coverage, and update docs if maintainers accept the new semantics.
