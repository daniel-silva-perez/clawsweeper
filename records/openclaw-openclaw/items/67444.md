## Summary

Keep this PR open because current main still has the reported behavior, but the submitted patch is not merge-ready: its ANNOUNCE_SKIP fallthrough can pass the control token into completion-delivery content, and the change still needs maintainer alignment with the documented suppression semantics.

## What This Changes

The PR changes the subagent announce flow so ANNOUNCE_SKIP no longer stops inter-session completion delivery, with related announce tests updated.

## Best Possible Solution

Accept the behavior only if maintainers want ANNOUNCE_SKIP to skip the soft announce step but not deterministic REPLY_TO_SESSION completion delivery; then rebase, preserve NO_REPLY suppression, clear or replace ANNOUNCE_SKIP before building findings, add positive delivery-call coverage, and update the subagent docs to describe the narrower semantics.
