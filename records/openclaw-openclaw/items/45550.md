## Summary

Keep open. Current main and the latest release still carry the pre-GA Anthropic 1M beta-header behavior in code, tests, and docs; open PR #45613 appears to contain the intended fix but remains open/unmerged, so this issue should stay open until maintainers land, refresh, or replace that PR.

## What This Changes

Migrate Anthropic 1M context handling from beta to GA by stopping the retired beta-header/OAuth skip behavior while preserving `context1m` as an explicit opt-in and updating docs/tests.

## Best Possible Solution

Keep `context1m` as the explicit context-window sizing opt-in, remove any emission of the retired `context-1m-2025-08-07` beta including legacy user config, preserve OAuth-required Anthropic betas, and update the Anthropic provider/token-use docs plus focused wrapper and extra-params coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: configure `context1m: true` for an eligible Anthropic Opus/Sonnet model, and the active stream wrapper still emits the retired beta for API-key auth while stripping it for `sk-ant-oat` tokens. The existing tests and docs also assert that old request-building behavior.

## Solution Assessment

Yes. The narrow Phase 1 direction is the best maintainable fix: remove the stale beta-header/OAuth special case, keep `context1m` for explicit sizing/cost control, and leave default-enabling 1M context as a separate product decision.
