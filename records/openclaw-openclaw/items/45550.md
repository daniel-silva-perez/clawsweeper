## Summary

Keep this issue open. Current main still implements the pre-GA Anthropic 1M beta-header path in code, tests, and docs, while PR #45613 remains the open closing implementation candidate rather than a merged fix.

## What This Changes

The issue requests migrating Anthropic 1M context support from beta-header behavior to GA behavior while keeping `context1m` as an explicit opt-in.

## Best Possible Solution

OpenClaw should stop emitting or preserving the retired `context-1m-2025-08-07` beta for GA Claude 4.6 models, remove the obsolete OAuth skip warning for that beta while preserving other required OAuth betas, keep `context1m` as the explicit context-window and cost-control knob, and update the Anthropic provider/token-use docs plus focused wrapper tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: configure `context1m: true` for `anthropic/claude-opus-4-6` or `anthropic/claude-sonnet-4-6`, and the active wrapper still emits the retired beta for API-key auth while stripping it for `sk-ant-oat` credentials.

## Solution Assessment

Yes. The Phase 1 direction is the narrowest maintainable fix: remove stale header/OAuth special-casing and docs, keep explicit `context1m` sizing for cost control, and leave default-enabling broader 1M behavior as a separate product decision.
