## Summary

Keep open. Current main still has the pre-GA Anthropic 1M behavior in code, tests, and docs, and an open implementation PR already references this issue, so it should remain open until maintainers land or replace a canonical fix.

## What This Changes

Migrate Anthropic 1M context support from beta to GA by stopping stale beta-header injection and OAuth skip handling while preserving the explicit `context1m` sizing opt-in and updating docs/tests.

## Best Possible Solution

Land or replace the canonical implementation PR so the bundled Anthropic plugin stops sending the stale 1M beta header and stops treating OAuth tokens as incompatible with 1M, while leaving `context1m` as the explicit context-window sizing opt-in and updating the Anthropic provider and token-use docs plus focused wrapper/context tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: configure `context1m: true` for `anthropic/claude-opus-4-6` or `anthropic/claude-sonnet-4-6`, and the stream wrapper still builds the old beta header; with an `sk-ant-oat` token it strips that beta and warns. No live Anthropic call is needed to reproduce OpenClaw's request-building behavior.

## Solution Assessment

Yes. The narrow Phase 1 direction is the best maintainable fix: remove the stale GA beta-header/OAuth special case, preserve the explicit opt-in for context sizing, and leave any default-enable/cost-policy change for a separate maintainer decision.
