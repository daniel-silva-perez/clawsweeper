## Summary

Keep open. Current main still has the pre-GA Anthropic 1M beta-header behavior in code, docs, and tests, while open PR #45613 remains the active unmerged implementation candidate with closing syntax for this issue.

## What This Changes

The issue requests removing Anthropic's retired 1M context beta-header and OAuth skip behavior while keeping `context1m` as a backward-compatible explicit opt-in.

## Best Possible Solution

Review, refresh, and land PR #45613 or a narrow replacement that removes emission of the retired `context-1m-2025-08-07` beta for GA 4.6 models, removes the obsolete OAuth skip warning for that beta, preserves required OAuth betas, keeps `context1m` as an explicit cost-control/context-window opt-in, and updates Anthropic provider/token-use/troubleshooting docs plus focused wrapper tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: configure `context1m: true` for `anthropic/claude-opus-4-6` or `anthropic/claude-sonnet-4-6`, and the active wrapper still emits the retired beta for API-key auth while stripping it for `sk-ant-oat` credentials.

## Solution Assessment

Yes. The Phase 1 direction is the narrowest maintainable fix: remove stale header/OAuth special-casing and docs, keep explicit `context1m` sizing for cost control, and leave default-enabling broader 1M behavior as a separate product decision.
