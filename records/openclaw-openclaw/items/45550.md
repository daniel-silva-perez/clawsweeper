## Summary

Keep open. Current main still implements the exact pre-GA behavior this issue asks to remove: `context1m: true` adds `context-1m-2025-08-07`, OAuth tokens strip that beta and warn, tests assert both behaviors, and the Anthropic provider docs still call 1M context beta-gated. There are also open implementation PRs with closing syntax, including the same-author PR #45592 and the broader #45613, so this issue should stay open until the chosen PR lands or maintainers close the PRs as superseded.

## Best Possible Solution

Keep this issue open and review the linked implementation PRs. The best code path is to land a focused Anthropic plugin change that keeps `context1m` as the cost-aware sizing opt-in, strips or ignores `context-1m-2025-08-07` from both implicit and legacy configured betas, preserves OAuth-required Anthropic betas, and updates the Anthropic provider docs/tests/changelog. After that PR merges, this issue can close as implemented.
