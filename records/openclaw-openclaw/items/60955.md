## Summary

Keep open. Current main still does not cover the reported Telegram/Kimi fenced-shell failure mode. The shipped strict-agentic/planning-only machinery is scoped to OpenAI GPT-5-family and Gemini recovery paths, while the issue was reported on Kimi Coding with a fenced shell block. The current block-reply delivery path strips reasoning/internal tool markers and de-duplicates messaging-tool replies, but it does not require real tool lifecycle activity before emitting execution-looking assistant text.

## Best Possible Solution

Keep this issue open and revive or reimplement the #61132 direction on current main: add a channel-agnostic guard in the embedded assistant reply path that detects strong first-person execution intent plus runnable shell fences when the same turn has no real tool lifecycle, emits a truthful blocked/error reply instead, and adds regression coverage for blocked no-tool and allowed tool-backed Telegram-style replies.
