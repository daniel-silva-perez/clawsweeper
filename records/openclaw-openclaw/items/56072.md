## Summary

Keep open. Current main still creates a fresh session for daily/idle stale rollover, archives the prior transcript, and only then fires plugin lifecycle hooks. It does not run the existing compaction memory-flush silent turn before archiving, and the requested reset controls (`onReset`, `archive-only`, `off`/`disabled`) are still absent.

## Best Possible Solution

Keep this issue open. The best product path is to add a core stale-rollover preservation step that runs the existing `agents.defaults.compaction.memoryFlush` silent turn, or an explicitly accepted reset-time equivalent, before archiving the old transcript; add documented reset policy controls such as `onReset: "flush" | "archive-only"` and an `off`/manual-only mode if approved; reconcile the internal-hook vs plugin-lifecycle behavior; and cover daily, idle, and manual reset with focused tests plus config/docs updates.
