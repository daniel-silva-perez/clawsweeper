## Summary

Keep open. Current main still performs daily/idle stale rollover by creating a new session, archiving the previous transcript, and firing plugin lifecycle hooks; it does not run the compaction memory-flush silent turn before archive, and the requested reset controls are still absent.

## Best Possible Solution

Keep this issue open. The best product path is a core stale-rollover path that, before archiving the old transcript, either runs the existing `agents.defaults.compaction.memoryFlush` silent turn or an explicitly accepted reset-time equivalent; add reset policy controls such as `onReset: "flush" | "archive-only"` and an `off`/manual-only mode if maintainers approve; bridge or document internal vs plugin hook behavior consistently; and cover daily, idle, and manual reset with focused tests plus schema/docs updates.
