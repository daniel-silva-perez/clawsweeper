This is a fresh concrete 2026.4.25 report, and current main still has the reported CLI/RPC shape: `openclaw nodes list` calls `node.pair.list` first and then `node.list` through the shared 10s Gateway call timeout. Current main has adjacent probe/readiness and node last-seen improvements, but I found no source, test, release, or related-item proof that the intermittent loopback `nodes list` timeout plus immediate `gateway probe` connect-timeout window is fixed.

## Best Possible Solution

Keep this issue open and reproduce or instrument it on current main or the next tagged build.
