## What This Changes

The PR changes command poll backoff to store a -1 reset sentinel after new output, clamps that sentinel in suggestion reads, and updates the colocated unit test expectation.

## Best Possible Solution

Land the narrow backoff-state fix after targeted validation, ideally with explicit coverage that an output poll followed by a quiet poll yields 5000ms and that suggestion reads after output do not jump to the max tier.

## Reproduction Assessment

Yes. Against current main, a direct state sequence of recordCommandPoll(state, id, true) followed by recordCommandPoll(state, id, false) reproduces the skip because count is reset to 0 and then incremented to 1.

## Solution Assessment

Yes. The latest PR branch is the narrow maintainable fix: use a -1 internal sentinel so the next no-output poll lands on index 0, while clamping suggestion reads so the sentinel does not leak into the schedule lookup.
