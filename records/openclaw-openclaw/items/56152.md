## Summary

Keep open. Current main has not implemented the PR's variant-aware `/tmp` versus `/private/tmp` local media root behavior, but the PR patch still has unresolved security review findings on the filesystem containment boundary. It should be revised rather than closed as cleanup.

## Best Possible Solution

Revise the PR rather than close it: keep canonical media-path enforcement when `fs.realpath(mediaPath)` succeeds, use the lexical media path only when realpath fails, allow configured roots to match needed lexical/canonical aliases, preserve the workspace-sibling guard against canonical variants, and add regression tests for `/tmp`↔`/private/tmp`, symlink escape outside the allowed root, and workspace-sibling symlink cases.

## Risks / Open Questions

- The proposed lexical-plus-realpath cross-product can permit a path lexically under an allowed local media root while its canonical target is outside that root.
- The proposed `resolved` value changes the workspace-sibling restriction from canonical-path behavior to lexical-path behavior.
- Current main may still falsely reject some valid macOS `/tmp` and `/private/tmp` local media paths, so the user-facing bug likely still needs a safer fix.
