## Summary

Keep this PR open. Current main still does not implement the requested `gateway.nodeMaxOldSpaceMb` config or generated service heap setting, and the related Gateway OOM report remains relevant, but this patch is not merge-ready because it preserves arbitrary ambient `NODE_OPTIONS`, misses Node's underscore heap flag spelling, and targets the old onboarding wizard file instead of the current setup surface.

## What This Changes

This PR adds an optional `gateway.nodeMaxOldSpaceMb` config field, schema/help labels, gateway and node service `NODE_OPTIONS=--max-old-space-size=...` injection, a setup prompt, and focused service-env tests.

## Best Possible Solution

Ship an equivalent refreshed implementation that adds a documented `gateway.nodeMaxOldSpaceMb` config contract, generates a deterministic heap-only startup setting for gateway and node services, ports the prompt to the current setup flow with blank-to-auto behavior, recognizes both accepted Node heap flag spellings, and covers the OOM path with focused service-env/config/setup tests.

## Reproduction Assessment

Yes. The related OOM behavior has concrete reproduction steps in #52092, and the PR regression can be reproduced statically from the diff: `resolveNodeOptions({ NODE_OPTIONS: "--require /tmp/evil.js" }, 4096)` would preserve the startup hook while adding the heap flag.

## Solution Assessment

No. Raising the service heap limit is a plausible narrow fix for the reported OOM class, but this implementation should synthesize a heap-only value instead of preserving ambient `NODE_OPTIONS`, and it must be rebased onto the current setup and node-service install paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Generate a heap-only NODE_OPTIONS value:** `src/daemon/service-env.ts:340`
  - body: `resolveNodeOptions` preserves the caller's existing `NODE_OPTIONS` or appends to it, and `buildCommonServiceEnvironment` writes that into the durable service environment. That bypasses the current install hardening that strips `NODE_OPTIONS`; an install run with `NODE_OPTIONS="--require /tmp/evil.js"` would persist that startup hook. Generate only the heap flag, or use an equivalent non-ambient startup mechanism.
  - confidence: 0.91
- **[P2] Respect underscore heap flags before appending:** `src/daemon/service-env.ts:274`
  - body: Node accepts `--max_old_space_size` as well as `--max-old-space-size`, but the helper only checks for the hyphen spelling. A user who already pinned heap with the underscore flag would get a second heap flag appended, and the later generated value can silently override their limit.
  - confidence: 0.84
- **[P2] Port the prompt to the current setup wizard:** `src/wizard/onboarding.gateway-config.ts:307`
  - body: The PR adds the heap prompt to `onboarding.gateway-config.ts`, but current main no longer has that file; the active flow is `src/wizard/setup.gateway-config.ts`. Without rebasing this prompt into the current setup code, users will not see or clear the new config through setup.
  - confidence: 0.86
