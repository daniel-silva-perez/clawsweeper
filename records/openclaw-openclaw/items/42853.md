## Summary

Keep this PR open. Current main still does not implement the proposed `gateway.nodeMaxOldSpaceMb` config, install-time heap detection, or supervised-service `NODE_OPTIONS=--max-old-space-size=...` injection. The PR remains relevant to the open Gateway OOM report in #52092, but it needs a refresh for current setup code and the repo's `NODE_OPTIONS` hardening before it is merge-ready.

## Best Possible Solution

Keep this PR open for maintainer review or an equivalent replacement. The best path is to add a documented `gateway.nodeMaxOldSpaceMb` config contract, generate a deterministic heap-only startup setting for gateway and node services without preserving arbitrary ambient `NODE_OPTIONS`, update the current setup wizard path, recognize both Node heap-flag spellings, and add focused tests for service env, schema/help/docs, setup blank-to-auto behavior, hostile `NODE_OPTIONS`, and the related cron/OOM path in #52092.

## Risks / Open Questions

- Merging the PR as-is could reintroduce ambient `NODE_OPTIONS` startup control, including `--require`, into durable service environments despite the current host-env security policy.
- The PR targets the old onboarding file; current main uses `src/wizard/setup.gateway-config.ts`, so the setup prompt work would not apply cleanly without rebasing.
- The heap flag detection needs to recognize both `--max-old-space-size` and `--max_old_space_size` before adding a generated flag.
- The 85% of total RAM default still needs maintainer product judgment for WSL, containers, shared hosts, and memory-constrained systems.
