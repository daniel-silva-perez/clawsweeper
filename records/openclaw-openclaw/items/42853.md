## Summary

Keep this PR open. Current main still does not implement the proposed `gateway.nodeMaxOldSpaceMb` config, install-time Node heap detection, or supervised service `NODE_OPTIONS=--max-old-space-size=...` injection. The idea remains relevant to the open Gateway OOM report in #52092, but the PR needs a rebase and security redesign before it is merge-ready.

## Best Possible Solution

Keep this PR open for maintainer review or an equivalent replacement. The best path is to add a documented `gateway.nodeMaxOldSpaceMb` config contract, generate deterministic heap-only startup settings for gateway and node services without preserving arbitrary ambient `NODE_OPTIONS`, update the current setup flow, handle both Node heap-flag spellings, and add focused tests for service env, schema/help/docs, setup blank-to-auto behavior, hostile `NODE_OPTIONS`, and the related cron/OOM path in #52092.

## Risks / Open Questions

- Merging the PR as-is could reintroduce durable ambient `NODE_OPTIONS` startup control, including flags such as `--require`, despite the current host-env security policy.
- The PR targets the old onboarding file and would need to be rebased onto the current setup flow in `src/wizard/setup.gateway-config.ts`.
- Heap-flag detection should recognize both `--max-old-space-size` and Node’s accepted underscore spelling `--max_old_space_size` before appending a generated flag.
- The 85% of total RAM default still needs maintainer product judgment for WSL, containers, shared hosts, and memory-constrained systems.
