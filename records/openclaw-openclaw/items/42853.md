## Summary

Keep this PR open. Current main still does not implement the proposed `gateway.nodeMaxOldSpaceMb` config, service `NODE_OPTIONS` heap injection, node-service threading, or setup prompt. The PR is a plausible implementation path for the open Gateway OOM report in #52092, but it needs a refresh against current setup code and the repository's `NODE_OPTIONS` hardening before merge.

## Best Possible Solution

Keep this PR open for maintainer review or an equivalent replacement. The best path is to add a documented `gateway.nodeMaxOldSpaceMb` config contract, generate a deterministic heap-only startup setting for gateway and node services without preserving arbitrary ambient `NODE_OPTIONS`, recognize both Node heap-flag spellings, update the current setup wizard path, and add focused tests for service env, schema/help/docs, wizard blank-to-auto behavior, hostile `NODE_OPTIONS`, and the related cron/OOM path in #52092.
