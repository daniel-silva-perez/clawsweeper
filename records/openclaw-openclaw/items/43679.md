## Summary

Keep this PR open. Current main still does not implement the PR's shared suspicious-orchestration leakage detector: the normalizer handles empty, silent, heartbeat, mixed NO_REPLY, and sanitizer paths, but route-marker/tool-JSON text such as `assistant to=functions.exec` plus `{"command":...}` is still outside the shared outbound guard. The PR discussion is useful and shows the author addressed several false-positive and bypass review comments, but the branch needs rebase/adaptation to current main before maintainer review.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase it onto current main, adapt the guard to the current `hasContent`-based `normalizeReplyPayload()` flow, and land the narrow shared detector plus regression tests without weakening existing NO_REPLY, media/channelData, heartbeat, direct-message rewrite, sanitizer, and explanatory-prose behavior.
