## Summary

Keep open. Current main still does not implement the shared suspicious-reply guard, and reply delivery still flows through the normalizer, so the intended protection is not obsolete. The PR adds a hasSuspiciousReplyLeakage detector, wires it into auto-reply normalization with a new suspicious skip reason, and adds normalizer/dispatcher tests for route-marker/tool-JSON leakage, media preservation, custom silent tokens, and false-positive cases.

## Best Possible Solution
