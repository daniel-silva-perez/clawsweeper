## Summary

Keep this PR open. Current main does not implement the PR's intended `api.runtime.followup.enqueueFollowupTurn()` surface and does not include the bundled `gateway-restart` plugin. Main has adjacent primitives, including durable next-turn injections, system events/heartbeat, subagent runtime helpers, and an internal durable restart-continuation queue, but none exposes a plugin runtime API for injecting a user-role prompt into an arbitrary existing session and waking that session. The PR still needs maintainer design/security review and a rebase against the newer durable restart/session-delivery machinery.

## Best Possible Solution

Keep this PR open for maintainer review, not automatic cleanup. The best path is to decide whether OpenClaw should expose a generic plugin runtime followup API, and if yes, port it onto current main's durable session-delivery/restart-continuation infrastructure with strict session-context preservation. The bundled `gateway-restart` plugin should either be split from the core API or aligned with the existing owner-only gateway restart policy, restart sentinel, dependency metadata, docs, and focused tests.

## Risks / Open Questions

- The PR adds a plugin tool that can run configured shell commands and terminate the gateway process; it must be owner-only, respect `commands.restart`, and have clear operator opt-in semantics before merge.
- The proposed runtime API is built on the in-memory followup queue, while current main now has a durable session-delivery queue for restart continuations; merging without rebasing the design could lose callbacks across crashes/restarts or duplicate newer infrastructure.
- Reconstructing an existing session turn is context-sensitive: review comments already flagged workspace, account, group policy, streaming break, elevated execution, and transcript-path preservation risks.
- The gateway-restart plugin writes a single marker and deletes it after enqueue; review discussion flagged durability/concurrency concerns that should be resolved against the current restart-sentinel/session-delivery model.
- The PR is large and stale against current main, with unrelated qa-lab/a2ui/lockfile churn in the branch history; it should be split or rebased before normal review.
