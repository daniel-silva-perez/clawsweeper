## Summary

Keep this PR open. Current main still lacks the proposed `runtime.followup.enqueueFollowupTurn()` API and the bundled `gateway-restart` plugin, while the PR crosses plugin runtime, session-delivery, restart, and shell-execution boundaries that need maintainer design/security review rather than cleanup closure.

## What This Changes

The PR branch adds a plugin runtime followup enqueue API backed by session reconstruction/cold drain support, a disabled-by-default `gateway-restart` plugin, related tests, and small lockfile/QA test artifact updates.

## Best Possible Solution

Keep this PR open for maintainer design and security review. Decide whether OpenClaw should expose a generic plugin runtime followup API or document/extend the existing Gateway-backed runtime primitives; if the API is accepted, port it onto current session-delivery/restart-continuation machinery with strict session context preservation. The `gateway-restart` plugin should be split or reviewed separately and aligned with the existing owner-only gateway restart policy, restart-disable setting, durable marker handling, dependency metadata, docs, and focused tests.
