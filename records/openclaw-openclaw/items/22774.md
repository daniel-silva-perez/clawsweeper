## Summary

Keep this issue open. Current main still has trace/provenance metadata and browser-local sender labels, but it does not provide a durable human/main-agent/sub-agent sender identity contract through chat.send, sessions.send/sessions.steer, subagent completion delivery, transcript history/live events, and Control UI rendering. Related #52498 and #58110 are still open but track narrower or broader UX slices, and the two PRs that explicitly referenced this issue are closed unmerged.

## Best Possible Solution

Keep this issue open as the focused tracker for an additive sender identity contract. The best path is to define durable per-message sender metadata with sender id, sender kind, display name, source session/agent, and renderable avatar/color metadata; persist it in transcripts; expose it through chat.history/live events; carry it through chat.send, sessions.send/sessions.steer, and subagent announce delivery; and update Control UI grouping/avatar rendering. Coordinate with #52498 for sessions_send display and #58110 for the broader multi-agent chat UX, but do not treat either as a replacement.
