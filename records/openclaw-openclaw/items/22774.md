## Summary

Keep this issue open. Current main still lacks a durable sender identity contract for human/main-agent/sub-agent attribution across `chat.send`, `sessions.send`/`sessions.steer`, inter-session provenance, persisted transcripts, `chat.history`, and Control UI rendering. Existing support is partial: browser-local user identity, top-level `senderLabel` for user groups, and trace-style `InputProvenance`, none of which carries stable sender id/kind/display/avatar data through the full multi-agent flow.

## Best Possible Solution

Keep this issue open as the focused tracker for durable multi-agent sender attribution. The best path is an additive sender identity contract carrying sender id, sender kind, display name, source session/agent, and renderable UI identity through `chat.send`, `sessions.send`, `sessions.steer`, sub-agent announce delivery, transcript persistence, `chat.history`, live chat events, and Control UI rendering. Coordinate with #58110 for broader multi-agent chat UX and #52498 for session_send-specific display needs, but do not treat either as a replacement.
