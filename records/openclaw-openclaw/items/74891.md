## What This Changes

The PR records which skills were selected in the onboarding dependency multiselect and skips primary environment-variable prompts for unselected bin-backed skills, with regression tests for local `openai-whisper` versus `openai-whisper-api`.

## Best Possible Solution

Land this PR or an equivalent narrow change after maintainer review, keeping the fix in the shared onboarding skill prompt flow and leaving the skill manifests unchanged unless maintainers want separate UX copy follow-up.

## Reproduction Assessment

Yes. A high-confidence mocked reproduction is to return missing `openai-whisper` plus missing env-gated `openai-whisper-api` or `sag` from `buildWorkspaceSkillStatus`, select only the local Whisper install, and observe that current main can still reach `Set OPENAI_API_KEY` or `Set ELEVENLABS_API_KEY` for skipped skills.

## Solution Assessment

Yes. The proposed selection guard is the narrowest maintainable fix I found: it preserves env-only prompts and prompts for skills the user chose to install, while suppressing credentials for bin-backed skills the user skipped.
