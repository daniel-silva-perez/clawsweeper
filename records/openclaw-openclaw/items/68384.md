## Summary

Keep open. Current main still accepts the full trimmed non-JSON stdout for a single `jsonOnly: false` exec SecretRef, so this PR is not obsolete; the branch is narrow and appears functionally correct, but it changes security-sensitive secrets parsing and should get maintainer/secops review before merge.

## What This Changes

This PR changes `src/secrets/resolve.ts` so single-id `jsonOnly: false` exec SecretRef stdout must be valid JSON or exactly one non-empty plain-text line, with a regression test in `src/secrets/resolve.test.ts`.

## Best Possible Solution

Land a resolver-boundary fix only after secops agrees with the fail-closed contract, then document that `jsonOnly: false` plaintext exec output must be one clean non-empty line or use JSON, and keep #64821 open until the same-line corruption question is resolved or separately tracked.
