## Summary

Keep this PR open. Current main and the latest release still use the same command palette background for hover and keyboard-active rows, so the user-visible problem remains, but the proposed CSS needs a small selector fix so the active state remains deliberate when it overlaps pointer hover.

## What This Changes

The PR changes `ui/src/styles/components.css` to split command palette hover and active item backgrounds into separate `color-mix` rules for default and light theme modes.

## Best Possible Solution

Land a one-file CSS update that keeps hover and keyboard-active command palette rows visually distinct and makes the active or active-hover state intentional across default dark and light theme modes, without changing command palette logic or unrelated UI surfaces.

## Reproduction Assessment

Yes. Current main is reproducible from source: hover and active are grouped under one background rule, and the renderer can produce hover and active states during mixed pointer and keyboard use. The proposed patch's remaining issue is reproduced by a command palette item that is both `:hover` and `.cmd-palette__item--active`.

## Solution Assessment

No, not quite. The PR targets the right narrow file, but the selector specificity should be adjusted with an active-hover rule or matched specificity so the selected-state styling is not overridden by hover when both states apply.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Make the active style win when hovered:** `ui/src/styles/components.css:3536-3541`
  - body: The new `.cmd-palette__item:hover` selector is more specific than `.cmd-palette__item--active`, and the light-theme hover selector is also more specific than the light active rule. When a row is both keyboard-active and under the pointer, it will render the hover background instead of the selected-state background, so the active color can still disappear in the overlap case this change is meant to clarify.
  - confidence: 0.84
