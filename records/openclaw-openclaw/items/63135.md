Keep this issue open. Current main still has related no-stall safeguards, but they do not cover the reported Discord plus minimax-portal/MiniMax-M2.7 visible acknowledgement-with-no-actions path. The reporter also narrowed the symptom to Discord while Web UI completes, so this remains a plausible channel/runtime execution stall rather than an already-fixed generic task-ledger issue.

## Best Possible Solution

Keep this open as a Discord-channel execution stall. Maintainers should ask for a redacted log slice or trajectory bundle around one failed Discord request, then trace processDiscordMessage -> dispatchInboundMessage -> runWithModelFallback -> runEmbeddedPiAgent with minimax-portal/MiniMax-M2.7.
