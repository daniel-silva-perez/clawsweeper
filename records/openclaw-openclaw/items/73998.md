## Summary

Keep this PR open. Current main has the earlier Feishu startup-failure retry and heartbeat defaults from #72411, but it does not implement this PR's remaining SDK lifecycle/backoff reconciliation: no Feishu-owned lifecycle hooks are passed to `Lark.WSClient`, no guarded WebSocket endpoint `httpInstance` is present, and the monitor still waits only for abort after `start()` instead of reacting to SDK reconnect exhaustion.

## Best Possible Solution

Keep this PR open for maintainer review, resolve the PingTimeout review question, consolidate the overlapping #73945 duplicate/replacement thread, and land this or an equivalent narrow Feishu-owned patch that wires SDK lifecycle callbacks into monitor recovery, guards invalid WS endpoint ClientConfig values, avoids stacking monitor backoff during SDK reconnects, and preserves focused Feishu tests plus changelog credit.
