## Summary

Current main still only partially addresses the report: chat-visible tool warnings append raw failure details only under the verbose/timed-out-cron-exec detail gate, and same-action retry success clears `lastToolError` without emitting a user-visible retry-success confirmation. The remaining change is security-sensitive because expanding chat-visible diagnostics can leak paths, command output, provider text, or secrets, so this should stay open for maintainer policy rather than conservative cleanup.

## What This Changes

The issue requests chat-visible tool failure warnings that include a failure reason, plus a confirmation when a retry of the same failed action later succeeds.

## Best Possible Solution

Define a safe chat-visible diagnostics policy with redacted or allowlisted reason summaries by tool and channel, then add retry-success confirmation only when a previously surfaced failure is resolved by a matching successful action. Coordinate the richer diagnostics direction with #60946, the external-channel privacy boundary from #54757, and the narrow edit-mismatch approach in #67276.
