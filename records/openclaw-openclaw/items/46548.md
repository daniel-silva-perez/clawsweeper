## Summary

Keep open. Current main still only partially addresses the report: ordinary non-verbose chat-visible tool warnings can omit failure reasons, and successful same-action retries clear internal error state without emitting a user-visible retry-success confirmation. The remaining work needs maintainer policy because expanding chat-visible tool diagnostics can expose sensitive local or provider data.

## What This Changes

The issue asks for chat-visible tool failure messages to include a useful failure reason and for successful retries to be confirmed after a previously surfaced failure.

## Best Possible Solution

Define a safe chat-visible diagnostics policy with redacted or allowlisted reason summaries by tool and channel, then add retry-success confirmation only when a previously surfaced failure is resolved by a matching successful action. Keep this aligned with #54757 for external-channel privacy, #60946 for richer diagnostics, and the narrow edit-mismatch direction in #67276.
