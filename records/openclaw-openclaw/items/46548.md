## Summary

Keep open: current main still only partially addresses the requested behavior. Tool failure reasons remain hidden from ordinary non-verbose chat-visible mutating-tool warnings, and matching retry success clears the stored failure state without emitting a user-visible retry-success confirmation. The remaining change needs maintainer policy because related open privacy reports show that raw tool error details can be sensitive when delivered to external channels.

## What This Changes

The issue asks for chat-surfaced tool failure messages to include the failure reason and to confirm when a later retry succeeds.

## Best Possible Solution

Define a safe chat-visible diagnostics policy for tool failures, including redacted or allowlisted failure reasons by tool and channel, then add retry-success confirmation only when a previously surfaced failure is resolved by a matching successful action. Coordinate the policy with #54757 and the richer diagnostics direction in #60946.
