## Summary

Keep open. Current main still leaves the central user problem unresolved: ordinary non-verbose tool warnings omit the failure reason, and successful same-action retries clear internal error state without a chat-visible retry-success confirmation. The remaining fix needs maintainer policy for redaction and channel visibility before implementation.

## What This Changes

The issue asks OpenClaw to show concise failure reasons in chat-visible tool error warnings and confirm when a failed tool action later succeeds after retry.

## Best Possible Solution

Define a safe chat-visible diagnostics policy for tool failures, including redacted or allowlisted reason summaries by tool and channel, then emit retry-success confirmation only when a previously surfaced failure is resolved by the same successful mutating action.
