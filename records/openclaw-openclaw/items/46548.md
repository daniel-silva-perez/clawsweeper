## Summary

Current main still only partially addresses the request: non-verbose chat-visible tool warnings can still omit ordinary failure reasons, and a successful same-action retry clears internal state without emitting a user-visible retry-success confirmation. The issue should stay open because the remaining fix needs maintainer policy for safe, channel-aware diagnostics.

## What This Changes

Request that chat-visible tool failure messages include a useful failure reason and that a later successful retry produce visible confirmation.

## Best Possible Solution

Define a safe chat-visible diagnostics policy for tool failures, including redacted or allowlisted reasons by tool and channel, then add retry-success confirmation only when a previously surfaced failure is resolved by a matching successful action. Keep this coordinated with the external-channel privacy work in #54757, richer model-facing diagnostics in #60946, and the narrower edit-mismatch direction in #67276.
