## Summary

Keep open: current main still only partially addresses the request. Non-verbose chat-visible tool warnings generally omit the stored failure reason, and successful same-action retries clear the unresolved failure state without emitting a retry-success confirmation. The closed unmerged implementation PR #46592 did not cover retry confirmation, and open #54757 makes the failure-reason part a privacy/product policy decision rather than a safe automatic cleanup.

## What This Changes

Request chat-surfaced tool failure warnings to include a safe failure reason and to confirm when a retried tool call later succeeds.

## Best Possible Solution

Define a safe chat-visible diagnostics policy for tool failures, including redaction or allowlisting of failure reasons by tool/channel, then add retry-success confirmation semantics only for failures that were previously surfaced and were later resolved by a matching successful action.
