## Summary

Keep open. Current main still only partially addresses the requested behavior: ordinary non-verbose chat-visible tool warnings still omit failure reasons, while same-action retry success clears internal error state without emitting the requested user-visible retry-success confirmation. The remaining policy is tied to open privacy work for external channels, so this needs maintainer judgment before implementation.

## What This Changes

The issue asks for chat-surfaced tool failure messages to include a failure reason and to confirm when a later retry succeeds.

## Best Possible Solution

Define a safe chat-visible diagnostics policy for tool failures, including redacted or allowlisted reasons by tool and channel, then add retry-success confirmation only when a previously surfaced failure is resolved by a matching successful action. Coordinate that with #54757 and keep richer model-facing diagnostics aligned with #60946.
