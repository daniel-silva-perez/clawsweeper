## Summary

Keep this PR open. Current main still has the underlying MiniMax VLM SSRF/key-and-image egress risk, but this stale one-file patch should not merge as-is because it forces strict guarded fetch mode instead of the current proxy-aware provider pattern, drops the existing timeout behavior, and lacks MiniMax-specific SSRF regression coverage.

## What This Changes

The PR replaces MiniMax VLM's raw image-understanding fetch with `fetchWithSsrFGuard(withStrictGuardedFetchMode(...))` and releases the guarded dispatcher after response processing.

## Best Possible Solution

Revise or replace the PR with a current-main fix that routes MiniMax VLM through the shared provider guarded HTTP helper or an equivalent proxy-aware guarded path, includes a MiniMax audit context, preserves timeout and dispatcher cleanup behavior, and adds regression tests for blocked private, link-local, and cloud-metadata hosts through `apiHost`, `MINIMAX_API_HOST`, and `modelBaseUrl`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve proxy-compatible MiniMax egress:** `src/agents/minimax-vlm.ts:80`
  - body: The patch calls `withStrictGuardedFetchMode` directly. Current MiniMax VLM activates env-proxy support before raw fetch, and the shared provider helper auto-selects `trusted_env_proxy` when HTTP(S)_PROXY applies; this change can break proxy-only deployments instead of using the configured provider egress path.
  - confidence: 0.9
- **[P2] Preserve the MiniMax VLM request timeout:** `src/agents/minimax-vlm.ts:80-96`
  - body: Current main accepts `timeoutMs` and passes `AbortSignal.timeout(timeoutMs)` to the MiniMax request. The guarded call in this patch does not pass a timeout or signal, so slow or hung VLM requests would lose the existing timeout guard.
  - confidence: 0.88
- **[P3] Add a MiniMax audit context:** `src/agents/minimax-vlm.ts:80-81`
  - body: `fetchWithSsrFGuard` logs blocked requests under the generic `url-fetch` context when no audit label is supplied. This provider-specific security path should pass a label such as `minimax-vlm` so blocked-host audit logs are attributable.
  - confidence: 0.82
