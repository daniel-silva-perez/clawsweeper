## Summary

Keep open. Current main still has the two broad prompt sources that can steer a Discord autoThread turn toward a manual parent-channel message send while automatic final delivery targets the created thread; the linked open PRs are the right place to consolidate the remaining fix.

## What This Changes

The issue reports that Discord `autoThread: true` channel mentions can produce both a manual parent-channel `message.send` reply and a normal final assistant reply in the created thread.

## Best Possible Solution

Review and consolidate one canonical prompt/skill change that is aware of the current visible-reply policy: avoid duplicate same-conversation `message.send` calls when automatic final delivery is active, while preserving message-tool guidance for visible output in default tool-only Discord guild/channel turns.

## Reproduction Assessment

Yes. The original 2026.4.26 path is documented with concrete Discord `autoThread: true` steps, and on current main the remaining risk can still be reproduced under `messages.groupChat.visibleReplies: "automatic"` because final delivery targets the created thread while the broad prompt sources can steer a separate manual send.

## Solution Assessment

Yes as a direction, but not as a blanket static instruction. The maintainable fix is prompt/skill clarification that distinguishes automatic final delivery from message-tool-only visible delivery, leaving autoThread routing itself unchanged.
