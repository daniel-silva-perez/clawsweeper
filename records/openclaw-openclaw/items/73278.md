## Summary

Keep open. Current main still contains the broad Codex runtime and Discord skill guidance that can steer a normal Discord channel reply through `message.send`, while autoThread final-response routing still targets the created thread; the linked open PRs are the right place to consolidate the remaining fix.

## What This Changes

Discord auto-threaded channel mentions should avoid producing both a parent-channel `message.send` reply and the normal final response in the created thread.

## Best Possible Solution

Review and consolidate one canonical prompt/skill change that tells agents to use final assistant output for the normal current conversation when automatic final delivery is active, while preserving `message.send` guidance for explicit sideband sends and for current message-tool-only visible output policy.

## Reproduction Assessment

Yes. The reported 2026.4.26 Discord `autoThread: true` path is concrete, and current main still has the same prompt/routing condition under `messages.groupChat.visibleReplies: "automatic"`: final delivery targets the created thread while the broad prompt text can steer an extra manual send.

## Solution Assessment

Yes as a direction. A narrowly scoped Codex/Discord prompt clarification is preferable to changing Discord autoThread delivery mechanics, because current tests show thread final delivery is already routed to the created thread.
