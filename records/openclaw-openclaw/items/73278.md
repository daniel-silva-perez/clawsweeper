## What This Changes

The issue asks for Discord `autoThread` channel mentions to avoid producing both a parent-channel message-tool reply and a final assistant reply in the created thread.

## Best Possible Solution

Review and land one canonical narrow fix that clarifies normal current-conversation replies versus explicit Discord messaging actions, preferably covering both the Codex runtime guidance and the bundled Discord skill with a focused regression check. The Discord autoThread routing logic should stay unchanged unless live verification shows delivery suppression or target selection is also wrong.
