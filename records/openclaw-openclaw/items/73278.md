## Summary

Keep open. Current main still contains both prompt sources that can steer a Discord autoThread turn toward a manual messaging-tool reply, while the Discord autoThread delivery path already targets the created thread for the normal final response. The issue also has three open, unmerged PRs with closing references, so it should stay open until maintainers review and land one canonical fix.

## What This Changes

Fix Discord autoThread mention handling so normal conversational replies are delivered once through final-response routing to the created thread, without an extra parent-channel `message.send`.

## Best Possible Solution

Land one canonical narrow fix that clarifies normal current-conversation replies versus explicit messaging actions in the Codex runtime guidance and the bundled Discord skill, with focused regression coverage for the generated developer instructions. The Discord autoThread delivery code should remain unchanged unless live verification shows a separate routing defect.
