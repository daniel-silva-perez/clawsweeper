## Summary

Keep open. Current main still contains the broad Codex runtime and bundled Discord skill prompt guidance that can steer a Discord turn toward a manual `message.send`, while the autoThread final-response path still routes automatic delivery to the created thread; three open closing PRs already target the remaining prompt fix, so this issue should stay open until one lands or maintainers choose a canonical replacement.

## What This Changes

The issue reports that Discord `autoThread: true` channel mentions can produce a manual parent-channel `message.send` reply plus a normal final response in the created thread.

## Best Possible Solution

Review and consolidate one canonical prompt/skill change that separates normal current-conversation replies from explicit Discord messaging actions, while leaving the existing autoThread delivery mechanics and visible-reply policy intact.

## Reproduction Assessment

Yes, with current main under the opt-in legacy path `messages.groupChat.visibleReplies: "automatic"`: autoThread final delivery targets the created thread, while the broad prompt sources can still steer an agent to send a separate parent-channel `message.send`. The exact default 2026.4.26 reproduction is changed on current main because guild/channel replies now default to `message_tool_only`.

## Solution Assessment

Yes, clarifying the Codex runtime guidance and/or Discord skill is the narrowest maintainable direction because the thread creation and final-response routing code already targets the created thread. The maintainer decision is which open PR provides the right wording and coverage.
