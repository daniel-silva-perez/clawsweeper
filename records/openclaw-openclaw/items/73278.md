## Summary

Keep open. Current main still has the broad Codex developer instruction and bundled Discord skill wording that can steer a Discord turn toward `message.send`, while the autoThread delivery path already routes normal final replies to the created thread; three open, unmerged PRs explicitly target this issue, so this should stay open for maintainer consolidation rather than cleanup closure.

## What This Changes

The issue requests clarified Discord/Codex reply guidance so Discord `autoThread: true` mentions produce only the normal final reply in the created thread, not an extra `message.send` reply in the parent channel.

## Best Possible Solution

Review and land one canonical narrow fix that clarifies normal current-conversation replies versus explicit Discord messaging actions, preferably covering both `extensions/codex/src/app-server/thread-lifecycle.ts` and `skills/discord/SKILL.md` with a focused regression check. The Discord autoThread delivery logic should remain unchanged unless live proof shows routing itself is wrong.
