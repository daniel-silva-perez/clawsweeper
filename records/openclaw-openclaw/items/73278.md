## Summary

Keep open. Current main still contains the two broad prompt sources that can steer a Discord autoThread turn toward an extra manual `message.send`, and the issue is already paired with open closing PRs that should be reviewed rather than closing the issue early.

## What This Changes

The issue reports that Discord `autoThread: true` mentions can produce both a parent-channel `message.send` reply and a normal final response in the created thread.

## Best Possible Solution

Land or consolidate one canonical prompt/skill fix that tells agents to use the final assistant response for normal current-conversation replies, while keeping the `message` tool for explicit sideband Discord actions and leaving the existing autoThread delivery mechanics intact.

## Reproduction Assessment

Yes. The original 2026.4.26 report gives a concrete live Discord path, and current main still has the prompt inputs plus an automatic opt-in path that can reproduce the duplicate shape. The default current-main behavior differs because group/channel final replies now default to `message_tool_only`.

## Solution Assessment

Yes, the prompt/skill clarification direction is the narrowest maintainable fix. The autoThread delivery plan already targets the created thread, so changing routing mechanics would be broader than necessary.
