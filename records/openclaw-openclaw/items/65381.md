## Summary

Keep this PR open. Current main still underestimates `/status` cost because it collects cacheRead/cacheWrite and the shared cost helper can price them, but `buildStatusMessage` only gates and sends input/output tokens to `estimateUsageCost`; the PR is a narrow, relevant fix with focused coverage and no blocking review findings.
