## Summary

Keep open. Current main still does not expose `--profile-id` or an equivalent operator-chosen OAuth profile alias on `openclaw models auth login`; the only current `--profile-id` CLI surface is `paste-token`. The shipped OpenAI Codex identity-derived naming fix addresses the related overwrite bug, but it is narrower than this explicit alias request. PR #49315 remains the active implementation candidate.

## Best Possible Solution

Keep this issue open as the tracker for explicit OAuth profile aliases. Review and land PR #49315 or equivalent work that adds `--profile-id <id>` to `models auth login`, threads it through login options and auth persistence, validates that the requested id belongs to the selected provider, handles collisions and multi-profile auth results safely, and updates CLI docs plus parser/persistence tests.
