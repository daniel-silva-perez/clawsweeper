## Summary

Keep this PR open. It is member-authored and carries the protected `maintainer` label, so this workflow should not auto-close it. Current main also still appears to have the gateway startup-context behavior the PR is trying to harden: spawned sessions can resolve an inherited workspace and use it for startup-context memory before agent dispatch, with no sandbox access gate in that preload path.

Have a maintainer review and either land this focused gateway hardening PR, with the regression test and any small review cleanups, or close it manually after deciding that gateway startup-context reads are intentionally outside the sandbox workspace-access boundary.
