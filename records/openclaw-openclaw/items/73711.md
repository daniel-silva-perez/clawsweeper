## Summary

Keep this PR open. Current main still has the pre-existing iOS chat composer attachment UI: the only PhotosPicker and picker onChange loader are in the toolbar branch that is hidden when the standard composer is focused, and the attachment row remains a capsule filename strip without a persistent thumbnail add-more tile. The PR is open, draft, unmerged, and active; the author addressed bot feedback in the PR head but explicitly moved it back to draft pending rebase and real-device validation. The related #73710 is separate image-processing work, not a superseding UI implementation.

Keep this PR open as the focused implementation candidate. The best maintainer path is to have the author rebase onto current main, verify focused composer add-via-plus-tile, remove, and send behavior on a real iPhone or simulator, attach screenshots, and then review the SwiftUI layout polish and whether the duplicated picker onChange should remain on the tile or move to an always-present parent. Keep #73710 separate because it owns image processing and EXIF stripping, not this composer UI affordance.
