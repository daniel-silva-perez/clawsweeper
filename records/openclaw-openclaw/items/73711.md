## Summary

Keep this PR open. Current main still has the old chat composer attachment UI: the iOS PhotosPicker only lives in the toolbar that is hidden when the focused standard composer compacts, and the existing attachment strip is still a filename/capsule row with no persistent trailing add-more tile. The PR is open, draft, unmerged, and remains the active implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open as the focused implementation candidate. Maintainers should let the author rebase onto current main, rebuild the iOS app, validate focused composer add-via-`+` behavior on device or simulator, and then review whether the mirrored `.onChange` handler should remain on the tile or move to an always-present parent. Keep #73710 separate because it covers image processing rather than this composer UI affordance.
