## Summary

Keep this PR open. Current main still has the old iOS chat composer attachment flow: the only PhotosPicker and picker onChange handler live in the toolbar that is hidden while the focused standard composer is compacted, and the attachment strip remains a capsule filename row with no persistent add-more tile. The PR is live, draft, unmerged, narrowly scoped to ChatComposer.swift, and the author explicitly moved it back to draft pending rebase and real-device validation rather than abandoning it.

## Best Possible Solution

Keep this PR open as the focused implementation candidate. The best path is to rebase it onto current main, rebuild the iOS app, verify focused composer add-via-plus-tile, remove, and send flows on a real iPhone or simulator, attach screenshots, and have maintainers decide the layout polish and whether the duplicated .onChange belongs on the tile or an always-present parent. Keep #73710 separate because it owns the resize/EXIF processing pipeline, not this composer UI affordance.
