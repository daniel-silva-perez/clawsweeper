## Summary

Keep this PR open. Current main still has the old iOS chat composer attachment UI: the only `PhotosPicker` and picker `.onChange` handler live in the toolbar that is hidden for the focused standard composer, and the attachment row is still a capsule filename strip with no persistent trailing add-more tile. The PR is active, draft, and unmerged; the discussion shows review feedback was addressed in the PR head, but the author explicitly moved it back to draft pending rebase and real-device validation.

## Best Possible Solution

Keep the PR open as the focused implementation candidate. The best path is for the author to rebase onto current main, rebuild the iOS app, verify focused composer add-via-`+` behavior and remove/send flow on a real device or simulator, attach screenshots, and then have maintainers review the layout polish and whether the duplicated `.onChange` should stay on the tile or move to an always-present parent. Keep #73710 separate because it owns the image processing/security pipeline, not this composer UI affordance.
