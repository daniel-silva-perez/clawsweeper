## Summary

Keep this PR open and do not merge the latest head yet. Current main still lacks the requested thumbnail/add-more attachment UI, but the PR head `fc916584d936` still duplicates `pickerItems` observers and can process one PhotosPicker selection twice before the async loader clears state.

## What This Changes

The branch modifies `ChatComposer.swift` to replace the capsule attachment strip with thumbnail tiles, remove buttons, and a trailing add-more PhotosPicker/file-picker tile.

## Best Possible Solution

Keep this PR as the canonical UI candidate, but update the implementation to observe `pickerItems` exactly once from a stable composer-level owner with an empty-selection guard, then validate the toolbar picker, focused add-more picker, multi-select, remove, and send flows on a real iPhone.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on latest head `fc916584d936`: when the toolbar picker and add-more picker are both mounted, both observe the same `pickerItems` change while `loadPhotosPickerItems` clears state only after awaited processing, so one selection can append twice.

## Solution Assessment

No. The product direction is appropriate, but the current PR head is not the best implementation; a single stable observer plus an empty reset guard is the narrower and more maintainable fix, and the user-facing change also needs the required changelog entry.
