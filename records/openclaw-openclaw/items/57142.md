## Summary

Keep open: current main does not provide configurable custom-term suggestions for WebChat input. The only matching UI surfaces are fixed welcome chips and slash-command completion, and current plugin UI descriptors do not appear to expose a composer-autocomplete hook, so this needs maintainer scoping rather than cleanup closure.

## What This Changes

The issue requests a user-configurable autocomplete or suggestion list for project-specific terms in the WebChat chat input, especially for typing and voice/dictation workflows.

## Best Possible Solution

Define a first-class, typed custom-term suggestion contract for WebChat, project it to the relevant UI clients, render accessible inline suggestions in the chat composer, and update docs/config help plus changelog once the intended browser/native scope is chosen.

## Reproduction Assessment

Yes for the feature gap: inspecting current main shows no WebChat suggestion config key and no custom-term autocomplete path in the browser or native composer surfaces. This is a feature request rather than a runtime bug reproduction.

## Solution Assessment

Unclear because there is no proposed patch. A core WebChat/config-backed implementation appears more appropriate than ClawHub-only work today because current plugin UI descriptors do not reach chat-composer autocomplete.
