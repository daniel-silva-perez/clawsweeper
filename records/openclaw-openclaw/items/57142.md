## Summary

Current main does not implement configurable custom-term autocomplete for WebChat/Control UI chat input, and the issue is recent, non-duplicate, and still actionable. It should stay open because the remaining decision is the product/API shape, not lack of evidence.

## What This Changes

The issue requests user-configurable autocomplete suggestions for project-specific terms in the webchat chat input, especially to help typing and dictation.

## Best Possible Solution

Keep this open until Control UI/WebChat owners choose a single custom-term suggestion model, then implement it in the relevant chat composers with schema/docs updates if the terms are gateway-configured.

## Reproduction Assessment

Yes. For feature absence, inspecting the current composer implementations and gateway WebChat config is a high-confidence reproduction path, and those surfaces do not expose configurable term suggestions.

## Solution Assessment

Unclear. Autocomplete is a reasonable direction, but the issue intentionally leaves storage and scope open; maintainers should decide the smallest durable surface before implementation.
