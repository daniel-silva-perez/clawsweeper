## Summary

Keep this PR open. Current main still does not add configured agents without stored sessions to the Control UI chat session dropdown, and this PR targets the removed `resolveSessionOptions()` helper rather than the current `resolveSessionOptionGroups()` path. Open PR #48671 is the current-code implementation candidate, but it is still open, so the best maintainer path is to consolidate or refresh rather than close as implemented.

## Best Possible Solution

Refresh this PR against current main or consolidate it into #48671. The current-code fix should update `ui/src/ui/chat/session-controls.ts` so `resolveSessionOptionGroups()` merges configured agents after stored rows and the active key, dedupes normalized keys, uses `state.agentsList.mainKey`, preserves configured agent names, respects global session scope, and adds regression tests for sessionless configured agents and global-scope behavior.
