## Summary

Keep this PR open. Current main still does not synthesize configured agents without stored sessions into the Control UI webchat dropdown, and this branch targets the old `resolveSessionOptions()` helper that no longer exists on the active path. Open PR #48671 is the better current-code implementation candidate, but it is still open, so maintainer consolidation or refresh is the right path rather than cleanup closure.

## Best Possible Solution

Refresh this PR against current main or consolidate it with #48671. The current-code fix should update `resolveSessionOptionGroups()` to merge `state.agentsList.agents` after stored rows and the active key, dedupe normalized session keys, preserve configured agent names, respect `agents.list.scope` and custom `mainKey`, and add regression tests for sessionless configured agents plus global-scope behavior.
