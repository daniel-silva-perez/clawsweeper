## Summary

Keep this PR open. Current main still does not synthesize dropdown entries for configured agents that have no stored session rows, and this branch targets the removed `resolveSessionOptions()` helper rather than the current `resolveSessionOptionGroups()` implementation. Open PR #48671 appears to be the current-code implementation candidate, but it is still open, so this should be refreshed or consolidated rather than closed as implemented.

## Best Possible Solution

Refresh this PR against current main or consolidate it into #48671. The current-code fix should update `ui/src/ui/chat/session-controls.ts` so `resolveSessionOptionGroups()` merges configured agents after stored rows and the active key, dedupes normalized keys, uses `state.agentsList.mainKey`, preserves configured agent names, respects `agents.list.scope === "global"`, and adds regression tests for sessionless configured agents plus global-scope behavior.
