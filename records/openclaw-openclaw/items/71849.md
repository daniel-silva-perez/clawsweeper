## Summary

Current main at f2405c830bd925bd24f91b6bac5d417c6d36e5ba still exposes the full `openclaw_agent_consult` realtime tool for Voice Call, but it does not implement the requested low-latency `realtime.fastContext` memory/session lookup path. The maintainer discussion already confirmed this as a real gap, and the adjacent open PR #72237 tunes full-consult behavior rather than adding fast context retrieval.

## Best Possible Solution

Keep this issue open and implement a default-off Voice Call `realtime.fastContext` path in the Voice Call plugin. The fix should add config/schema/manifest/docs, accumulate and normalize partial user transcript chunks for context-trigger detection, call the public active-memory search manager under a short hard deadline with `sources` and `maxResults`, inject either bounded snippets or an explicit miss into the realtime session, reset so multiple context questions in one call can trigger, and keep full `openclaw_agent_consult` as fallback for deeper work. Treat #72237 as adjacent consult tuning, not the canonical fix for this issue.
