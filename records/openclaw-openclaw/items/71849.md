## Summary

Keep open. Current main still lacks the requested Voice Call realtime fast-context path, and the maintainer discussion explicitly says this remains a real gap: remembered-context turns still depend on the full consult tool, partial transcript chunks are ignored before call-state processing, and the strict config/manifest/docs expose no fast-context setting.

## What This Changes

The issue requests an opt-in Voice Call realtime fast-context path that retrieves memory/session context under a tight deadline before falling back to full agent consult.

## Best Possible Solution

Implement a default-off, plugin-owned realtime.fastContext feature in Voice Call that accumulates and normalizes partial user transcript chunks, detects context-seeking turns, performs a hard-deadline memory/session search through public memory-host seams, injects snippets or an explicit miss into the realtime session, resets for follow-up questions, and preserves full openclaw_agent_consult as fallback.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: current main has no fastContext config/docs/manifest surface, registers only the full consult handler for realtime remembered-context turns, and discards non-final transcript chunks before call-state processing.

## Solution Assessment

Yes. The narrow maintainable direction is an opt-in Voice Call plugin path using existing memory-host search surfaces; moving this into core or replacing the full consult tool would be broader than the issue requires.

## Repair Work Prompt

Implement a default-off Voice Call realtime fast-context path for https://github.com/openclaw/openclaw/issues/71849. Add `realtime.fastContext` to the Voice Call config schema, manifest metadata, and docs with fields such as `enabled`, `timeoutMs`, `sources`, `maxResults`, and `fallbackToConsult`. Keep existing behavior unchanged when disabled. In `extensions/voice-call/src/webhook/realtime-handler.ts`, accumulate and normalize partial user transcript chunks enough to detect common context-seeking utterances, allow multiple lookups per call, and reset the gate after an injected result or miss. In the Voice Call realtime runtime, perform a bounded memory/session search through public memory-host SDK surfaces before the full `openclaw_agent_consult` path, inject top snippets or an explicit empty/miss result into the realtime session under the configured deadline, and keep full consult available as fallback for deeper work. Do not move owner-specific policy into core, do not replace `openclaw_agent_consult`, do not add bundled plugin IDs to core, and do not change unrelated provider/runtime dependency behavior. Update focused tests and Voice Call docs; add a changelog entry only if the change is user-facing per repo policy.
