## Summary

Keep open. Current main still routes Voice Call realtime memory/context questions through the full consult tool path and lacks the requested low-latency memory/session context resolver, while maintainer discussion explicitly confirmed this as a real current-main gap.

## What This Changes

Add an opt-in fast memory/session context path for Voice Call realtime calls so remembered-context questions can be answered within a short deadline before falling back to full agent consult.

## Best Possible Solution

Implement a default-off, plugin-owned `realtime.fastContext` feature in Voice Call that accumulates and normalizes partial user transcript chunks, detects context-seeking turns, performs a bounded memory/session search through the public memory-host seam, injects snippets or an explicit miss into the realtime session, resets for follow-up questions, and keeps `openclaw_agent_consult` as fallback.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: on current main, enabling Voice Call realtime with `realtime.toolPolicy` not `none` exposes only the full consult handler for remembered-context questions, while non-final transcript chunks are discarded before call-state processing.

## Solution Assessment

Yes. The best fix is a narrow, opt-in Voice Call plugin path because the latency policy is voice-call-specific, the existing memory-host search seam appears sufficient for lookup, and full consult should remain available for deeper fallback work.
