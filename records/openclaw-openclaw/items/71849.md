## Summary

Keep open. Current main still has the full Voice Call realtime consult tool path, but it does not have the requested low-latency `realtime.fastContext` memory/session lookup path. The maintainer discussion already confirmed this as a real gap, and the adjacent open PR #72237 tunes full-consult behavior rather than implementing fast context retrieval.

## What This Changes

Add a default-off fast memory/session context path for realtime Voice Call so live calls can answer remembered-context questions under a short deadline before falling back to full `openclaw_agent_consult`.

## Best Possible Solution

Implement a default-off Voice Call `realtime.fastContext` path in the Voice Call plugin: config/schema/manifest/docs, partial-transcript accumulation and normalization, a bounded active-memory search using the public memory-host-search seam, explicit snippet or miss injection, reset for multiple context questions in one call, and full `openclaw_agent_consult` fallback for deeper work.

## Reproduction Assessment

Yes. A static reproduction path is high-confidence on current main: enable Voice Call realtime with `realtime.toolPolicy` not `none`, then memory-context questions can only use the full consult handler while non-final transcript chunks are ignored; live Twilio/Gemini proof would validate latency but is not needed to confirm the requested capability is absent.

## Solution Assessment

Yes, the requested default-off fast-context path is the narrowest maintainable direction. It keeps owner-specific behavior in the Voice Call plugin, uses an existing public memory search seam, and preserves full consult as fallback instead of changing core realtime semantics.
