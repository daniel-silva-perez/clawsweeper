## Summary

Keep open. Current main still has the Voice Call realtime consult path, but not the requested low-latency fast memory/context resolver. The code exposes `openclaw_agent_consult` through `realtime.toolPolicy`, runs the full embedded consult with `responseTimeoutMs`, records only final realtime transcripts into call state, and has no `realtime.fastContext` config, docs, or tests. Related PR #72237 is useful adjacent work for consult fast-mode/thinking overrides, but it does not implement the fast memory/context hot path described here.

## Best Possible Solution

Keep this issue open and implement a default-off Voice Call `realtime.fastContext` path. It should accumulate/normalize partial user transcripts, detect context-seeking utterances, run a direct memory/session-context lookup through the public memory search SDK under a short hard deadline, inject a bounded snippet or explicit miss into the realtime session, reset so multiple context questions per call work, and keep full `openclaw_agent_consult` as fallback for deeper work. Update the plugin manifest/config schema, tests, and https://docs.openclaw.ai/plugins/voice-call.

## Clownfish Work Prompt

Implement the Voice Call realtime fast memory/context path requested by this issue. Keep the change plugin-owned and default off.

Expected scope:
- Add `realtime.fastContext` config to the Voice Call plugin schema, manifest, config labels, normalization/tests, and docs. Suggested fields: `enabled`, `timeoutMs` default around 800 ms, `sources` such as `memory`/`sessions`, `maxResults`, and `fallbackToConsult`.
- In `extensions/voice-call/src/webhook/realtime-handler.ts`, accumulate and normalize partial user transcript chunks enough to detect common remembered-context questions. Do not depend only on final transcript events, and reset the gate so multiple context questions in one call can trigger lookup.
- In the Voice Call realtime path, use public SDK surfaces such as `openclaw/plugin-sdk/memory-host-search` / `MemorySearchManager.search` for direct bounded lookup. Do not import core internals from the plugin.
- Inject bounded snippets, or an explicit empty/miss result on timeout, into the realtime session using the existing realtime bridge/session APIs where possible. Keep `openclaw_agent_consult` available as fallback for deeper work and preserve existing behavior when `fastContext.enabled` is false.
- Add focused unit coverage for config parsing, partial transcript accumulation/normalization, deadline/miss behavior, multiple context questions per call, and fallback-to-consult behavior. Update https://docs.openclaw.ai/plugins/voice-call with the latency tradeoff.
- Do not change unrelated providers, package versions, release metadata, or broad memory search reliability tracked by #71784.
