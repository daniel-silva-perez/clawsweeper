## Summary

Keep open. Current main supports isolated agents and an agent-grouped chat session selector, but it does not implement team/project parent nodes, configurable agent team/group metadata, independent team management, or workspace-derived hierarchical grouping. Related #49828 covers a sidebar agent-list UX and #52683 covers broader shared-context project rooms; neither fully supersedes this request.

## Best Possible Solution

Keep this open for maintainer product design. The best path is to decide whether the request is a lightweight presentation feature, such as optional agents.list[].group metadata plus grouped Control UI session/agent views, or part of a broader project-room/team model. If accepted, update the agent config contract, agents.list protocol/schema, Control UI session and agents views, docs, and focused UI/backend tests together.
