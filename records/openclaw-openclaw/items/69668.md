## Summary

Keep open. Current main supports isolated agents and an agent-grouped chat session selector, but it still does not provide team/project parent nodes, team/group metadata on agents, workspace-derived hierarchy, or independent team management in the Control UI. Related #49828, #30861, and #52683 overlap nearby multi-agent UX and project-room space, but they do not supersede this specific hierarchy request.

## Best Possible Solution

Keep this open for maintainer product design. Decide whether the request should be a lightweight presentation feature, such as optional `agents.list[].group` or project metadata plus grouped Control UI session/agent views, or part of a broader project-room/team model. If accepted, update the agent config contract, `agents.list` protocol/schema, Control UI chat/sessions/agents surfaces, docs, and focused UI/backend tests together.
