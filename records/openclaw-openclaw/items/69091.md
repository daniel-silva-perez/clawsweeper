## Summary

Keep this PR open. Current main still has the three hardcoded generic transport-stream throws that the PR targets, the existing tests do not cover the errorMessage propagation branch, and the live PR diff is narrowly scoped with no apparent security-sensitive or supply-chain changes. The main review question is whether the Gemini/Google repro also needs a companion fix in the Google plugin path.

## Best Possible Solution

Keep this PR open and review or land an equivalent focused patch: propagate `output.errorMessage || "An unknown error occurred"` through the shared helper and both OpenAI Responses finalization branches, keep the fallback behavior, add the shared-helper regression tests, and decide whether Google/Gemini finishReason mapping needs a companion errorMessage fix.
