## Summary

Keep this PR open. Current main still has the hardcoded generic transport-stream throws that the PR targets, and the nearby catch paths still overwrite `output.errorMessage` with that generic thrown string. The PR is not implemented or superseded on main, and the supplied diff is narrowly scoped with no apparent supply-chain or secret-handling changes.

## Best Possible Solution

Keep this PR open and review or land an equivalent focused patch: replace the three generic transport-stream throws with `output.errorMessage || "An unknown error occurred"`, keep the fallback behavior, and add the shared-helper regression tests. During review, verify whether the Gemini/provider-owned path from the repro needs a companion fix or a separate follow-up.
