## Summary

Keep open. Current main still has the reported CLI gap: `openclaw message send --media` exposes channel account selection, but it does not expose or thread an agent selector, so media-root resolution remains tied to the default agent rather than a non-default agent workspace. The related media-root PRs remain open and do not provide a merged current-main fix for this CLI path.

## What This Changes

The issue requests that `openclaw message send --media` use an agent-scoped workspace/media allowlist instead of only default roots when sending local media files.

## Best Possible Solution

Add a documented `openclaw message ... --agent <id>` selector, validate it against configured agents, thread it through the message CLI helper and `messageCommand` into `runMessageAction`, and use the selected agent workspace for local media roots. Keep account selection separate unless account-to-agent inference is unique, deterministic, documented, and tested.
