## Summary

Keep open. Current main still has the reported CLI gap: `openclaw message send --media` accepts `--account`, but has no agent selector, and `messageCommand` still passes the configured default agent into the outbound runner. Downstream media-root code is already agent-scoped, so the missing piece is choosing the intended agent/workspace from the CLI; adjacent media-root PRs are open and do not provide a merged current-main CLI fix.

## What This Changes

Request that CLI `message send --media` resolve local media access against the selected or configured agent workspace instead of only the default media roots/default agent workspace.

## Best Possible Solution

Add an explicit `openclaw message ... --agent <id>` selector, validate it against configured agents, thread it through the message CLI helper and `messageCommand` into `runMessageAction`, and use the selected agent workspace for media roots while keeping `--account` as channel account selection. Account-to-agent inference should only be added if it is unique, deterministic, tested, and documented.
