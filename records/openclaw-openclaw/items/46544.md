## Summary

Keep open. Current main still leaves CLI media sends scoped to the configured default agent: the CLI exposes `--account` but no `--agent`, then `messageCommand` always supplies `resolveDefaultAgentId(cfg)` to the outbound runner. The downstream media-root code is already agent-scoped, so the remaining gap is selecting the intended agent/workspace for CLI sends; related media-root PRs remain open and do not provide a merged current-main CLI fix.

## What This Changes

The issue requests that `openclaw message send --media` allow media from the relevant agent workspace instead of only the default agent/default media roots.

## Best Possible Solution

Add an explicit `openclaw message ... --agent <id>` selector, validate it against configured agents, thread it through the message CLI helper and `messageCommand` into `runMessageAction`, and use that selected agent workspace for media roots. Keep `--account` as channel account selection; only add account-to-agent inference if the binding is unique, deterministic, documented, and tested.
