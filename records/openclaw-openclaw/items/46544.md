## Summary

Current main still leaves the reported CLI gap open. The message CLI can select a channel account, but it has no agent selector, and the command still passes the default agent into media-root resolution; adjacent media-root PRs are open and do not provide a merged CLI fix.

## What This Changes

The issue requests that CLI `openclaw message send --media` use agent-scoped or configured media roots so files from custom per-agent workspaces can be sent without copying them into default media roots.

## Best Possible Solution

Preserve least-privilege media access by adding an explicit message CLI `--agent` option, threading it through the CLI helper and `messageCommand` into `runMessageAction`, and using that selected agent workspace for `--media`; account-to-agent inference should only be added when the account binding is unique and documented.
