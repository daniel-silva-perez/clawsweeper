## Summary

Keep open. Current main still has the reported CLI gap: `openclaw message send --media` can select a channel account, but it does not expose or thread an agent selector, so media-root resolution remains tied to the configured default agent rather than the workspace for another bound account. The adjacent media-root PRs are still open and do not provide a merged current-main CLI fix.

## What This Changes

The issue requests that CLI `message send --media` allow local files from the relevant agent workspace, either by selecting an agent/account-scoped workspace or by adding a configurable media allowlist.

## Best Possible Solution

Add an explicit `openclaw message ... --agent <id>` selector, validate it against configured agents, thread it through the CLI helper and `messageCommand` into `runMessageAction`, and use that selected agent workspace for media roots. Account-to-agent inference should only be added later if the binding is unique, deterministic, and documented; current main should not include every configured workspace by default.
