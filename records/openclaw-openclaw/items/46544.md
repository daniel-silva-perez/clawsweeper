## Summary

Keep open. Current main still lacks a way for `openclaw message send --media` to select a non-default agent workspace: the CLI exposes account selection, but `messageCommand` still passes the configured default agent into the outbound runner, while downstream media-root logic is already scoped to whichever agent id it receives. The issue is concrete and recently active, so the stale label is not enough to close it.

## What This Changes

Request that CLI `message send --media` resolve local media roots from the intended agent workspace instead of only the default agent workspace for custom multi-agent workspaces.

## Best Possible Solution

Add a documented, validated `openclaw message ... --agent <id>` selector, thread it through the CLI helper and `messageCommand` into `runMessageAction`, and keep `--account` as channel-account selection. If account-to-agent inference is added later, make it unique, deterministic, and tested; do not include every configured workspace by default.
