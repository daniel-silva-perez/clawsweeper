## Summary

Keep open. Current main still has the reported CLI gap: `openclaw message send --media` can select a channel account, but it cannot select the agent whose workspace should scope local media access, while the downstream media code already honors whichever agent id it receives.

## What This Changes

Allow `openclaw message send --media` to send local files from a non-default agent workspace instead of being limited to the default agent/media roots.

## Best Possible Solution

Add a documented and validated `openclaw message ... --agent <id>` selector, thread it through the CLI helper and `messageCommand` into `runMessageAction`, and keep `--account` scoped to channel account selection. Do not include every configured workspace by default; account-to-agent inference should only be added if it is unique, deterministic, documented, and tested.
