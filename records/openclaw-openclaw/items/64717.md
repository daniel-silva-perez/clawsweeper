## Summary

Keep open. The PR addresses a real Telegram bug that is still present on current `main`, but the proposed test currently omits the required runtime `cfg`, so the branch needs an in-place amendment before normal maintainer review or merge.

## What This Changes

This PR updates Telegram send chunk planning to use the configured text chunk limit clamped to 4096 and adds a send-path regression for the 4096 configured-limit case.

## Best Possible Solution

Amend this PR in place: keep the Telegram-owned resolver/clamp implementation, pass the configured object as `cfg` in the new regression, optionally add one lower configured-limit case, then run the focused Telegram send test and extension changed gates before landing.
