## Summary

Keep open: current main still ignores the configured Telegram `textChunkLimit` in the direct send chunk planner, while the config surface and resolver already exist. The supplied PR remains a narrow Telegram-owned fix and does not show unrelated security-sensitive changes.

## What This Changes

This PR updates Telegram HTML/plain-text send chunk planning to use the configured text chunk limit clamped to 4096 and adds focused regression coverage for the 4096 case.

## Best Possible Solution

Keep the fix in the Telegram plugin: resolve `channels.telegram.textChunkLimit` with the existing plugin SDK resolver, clamp the effective send limit to Telegram's 4096 maximum, use that value consistently throughout `buildChunkedTextPlan`, and keep focused send-path regression coverage for configured limits.
