## Summary

Keep open. Current main still lacks per-agent scoping for the bundled memory-lancedb plugin, and this draft PR remains a plausible implementation candidate. It should not be closed as implemented or superseded because the main branch still uses one shared LanceDB table and one shared MemoryDB instance, while the PR has unresolved correctness and security-sensitive review concerns before it can land.

## Best Possible Solution

Keep this PR open as an implementation candidate. The maintainer path is to rebase it onto current main, implement the scoping inside extensions/memory-lancedb, use ctx.agentId consistently from tool factories and hook contexts, update the current before_prompt_build path, resolve LanceDB filter ordering and agentId validation semantics, log migration failures, and add regression coverage for agent A/B isolation, legacy shared memories, scoped duplicate detection, scoped forget/delete, and default unscoped behavior. Then either land it or explicitly supersede it with a canonical replacement PR.
