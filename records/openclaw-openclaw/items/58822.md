## Summary

Keep open. Current main still implements the precedence reported as the regression in both native subagent model selection and isolated cron model selection, and tests still assert that the agent's own model wins over `agents.defaults.subagents.model`. The issue is also paired with open implementation PR #58823 using closing syntax, with overlapping open PR #72877, so it should remain open until a fix is reviewed, merged, or superseded.

## Best Possible Solution

Review and land or supersede the linked implementation work. The fix should make the precedence explicit as `agentConfig.subagents.model` -> `agents.defaults.subagents.model` -> `agentConfig.model`, apply the same rule in isolated cron, and update the current tests that assert agent-model priority.
