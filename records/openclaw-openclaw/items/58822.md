## Summary

Keep open. Current main still prefers the parent or target agent's own model over `agents.defaults.subagents.model` in both native subagent selection and isolated cron selection, while docs and the reporter's confirmed regression support the opposite precedence; open PR #58823 also uses closing syntax, so this should stay open until a fix is merged or superseded.

## What This Changes

The issue requests changing subagent model selection precedence to `agentConfig.subagents.model` first, `agents.defaults.subagents.model` second, and the agent's own model only as a fallback.

## Best Possible Solution

Align the native resolver and isolated cron resolver with the documented precedence, update the tests that currently lock in agent-model priority, and then review whether #72877's runtime session-persistence change should be included with or separated from the precedence fix.

## Reproduction Assessment

Yes. The provided YAML config maps directly onto the current fallback chain, and source inspection shows `agentConfig.model` wins before `agents.defaults.subagents.model`; existing tests also encode that result.

## Solution Assessment

Yes, the requested precedence is the narrow maintainable fix for this report because it matches the docs and preserves explicit per-agent subagent and explicit spawn overrides. The safer landing path is to review/land or supersede #58823 while accounting for the broader overlap in #72877.
