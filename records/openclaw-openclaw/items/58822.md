## Summary

Keep open. Current main still prefers the parent/target agent's own model over `agents.defaults.subagents.model` in both native subagent and isolated cron model selection, and existing tests assert that opposite precedence. The issue is also paired with open implementation PR #58823 using closing syntax, so it should remain open until the fix is reviewed and merged or superseded.

## Best Possible Solution

Review and land or supersede the linked implementation PR. The code change should make the precedence explicit as `agentConfig.subagents.model` -> `agents.defaults.subagents.model` -> `agentConfig.model`, apply the same rule in isolated cron, and update the current regression tests that assert agent-model priority.
