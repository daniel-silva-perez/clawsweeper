## Summary

Keep this PR open. Current main still requests and renders a 200-message initial Control UI chat window, so the proposed 80-message performance guard is not implemented. The PR remains plausible and narrowly scoped, but it is stale against current main because active rendering now lives in `ui/src/ui/chat/build-chat-items.ts`, while the PR diff still updates the older view-local render path.

## Best Possible Solution

Keep this PR open and ask for a current-main refresh. The best implementation is to define one shared initial chat history window constant near the existing chat history limit code, use it from `ui/src/ui/controllers/chat.ts` and `ui/src/ui/chat/build-chat-items.ts`, decide whether `ui/src/ui/chat/input-history.ts` should intentionally share the same value, then update `ui/src/ui/controllers/chat.test.ts` and add direct coverage in `ui/src/ui/chat/build-chat-items.test.ts`. No Gateway protocol, server, dependency, or security-sensitive change appears necessary.
