## Summary

Keep this PR open. Current main still uses a 200-message initial Control UI `chat.history` request and a 200-message active render slice, so the intended 80-message hydration/render guard is not implemented. The PR remains directionally useful, but it is stale against current main because active render-window logic now lives in `ui/src/ui/chat/build-chat-items.ts`, while the PR changes the older view-local path. Security pass found no CI, dependency, lockfile, install script, publishing, secrets, or other code-execution surface in the provided PR file list.

## Best Possible Solution

Keep this PR open and ask for a current-main refresh. The best implementation is to introduce one shared initial chat history window constant near the existing UI history-limit code, use it from `ui/src/ui/controllers/chat.ts` and `ui/src/ui/chat/build-chat-items.ts`, decide whether `ui/src/ui/chat/input-history.ts` should share that value, then update `ui/src/ui/controllers/chat.test.ts` and add direct coverage in `ui/src/ui/chat/build-chat-items.test.ts`. No Gateway protocol, server, dependency, CI, or security-sensitive change appears necessary.
