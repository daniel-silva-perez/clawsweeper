## Summary

Keep open. Current main still lacks the requested Slack DM assistant-root thread preservation, and the proposed patch is close but still misses an existing normalized-DM contract, so this PR should not be treated as implemented or ready to merge yet.

## What This Changes

The PR updates Slack thread-context resolution and tests to preserve `MessageThreadId` for Slack Agents & Assistants DM root messages, and also adds `context_window_exceeded` matching to an agents overflow-compaction test harness.

## Best Possible Solution

Implement the Slack fix using the normalized direct-message classification already available in the message pipeline, preserve `MessageThreadId` only for assistant-DM self-thread roots, leave non-DM self-thread roots unset, and cover raw `im`, D-prefix wrong/missing `channel_type`, non-DM roots, and concurrent assistant-DM roots in focused Slack tests.
