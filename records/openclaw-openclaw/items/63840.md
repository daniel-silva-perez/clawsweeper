## Summary

Keep open. Current main still lacks the requested Slack Agents & Assistants DM-root thread preservation, but this PR should not merge as-is because the branch keys the new override on raw `channel_type` and misses D-prefixed Slack DMs that the current pipeline already normalizes as direct messages.

## What This Changes

The PR changes Slack thread-context resolution and tests to set `MessageThreadId` for DM assistant thread-root messages across non-`all` reply modes, and also adds `context_window_exceeded` matching to an agents overflow-compaction test harness.

## Best Possible Solution

Use the normalized Slack direct-message classification already available in the message pipeline, or pass an equivalent normalized boolean into `resolveSlackThreadContext`, so only assistant-DM self-thread roots preserve `MessageThreadId`. Keep non-DM self-thread roots unset, add coverage for raw `im`, D-prefix wrong/missing `channel_type`, and non-DM roots, and keep the agents overflow-compaction harness change out of this Slack PR.
