## Summary

Keep open. Current main still lacks the intended Slack Agents & Assistants DM-root thread preservation for non-`all` reply modes, so the PR is not obsolete, but the branch still needs a tighter Slack-only revision because it keys the new behavior on raw Slack `channel_type` and includes an unrelated agents test-harness edit.

## What This Changes

The PR changes Slack thread-context resolution and tests to set `MessageThreadId` for DM assistant thread-root messages regardless of reply mode, and also adds `context_window_exceeded` matching to an agents overflow-compaction test harness.

## Best Possible Solution

Implement the Slack fix using the normalized direct-message classification already available in the message pipeline, preserve `MessageThreadId` only for assistant-DM self-thread roots, keep non-DM self-thread roots unset, and cover raw `im`, D-prefix wrong/missing `channel_type`, non-DM roots, and concurrent assistant-DM roots in focused Slack tests.
