## Summary

Keep open. Current main still omits the `dependencies` field for an empty bundled runtime-deps install manifest, while this contributor PR is a narrow implementation with no blocking review findings. The remaining action is maintainer validation and merge judgment for the linked npm v11 crash report, not cleanup closure or an automated replacement PR.

## What This Changes

The PR changes bundled runtime-deps install manifests to always include a `dependencies` object and adds an empty-spec regression test.

## Best Possible Solution

Merge this narrow manifest-helper and regression-test change after maintainer validation, then let the linked bug close through the merged PR. The fix should stay limited to the generated npm execution manifest and avoid changing package-manager selection, install flags, or bundled-plugin policy.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Unclear for the full Gateway crash loop because this read-only review did not run npm v11 or Gateway startup. Yes for the manifest-shape edge case: current main statically omits `dependencies` when `ensureNpmInstallExecutionManifest` is called with an empty spec list.

## Solution Assessment

Is this the best way to solve the issue? Yes. Always emitting `dependencies: {}` is the smallest maintainable change for the generated manifest contract and preserves existing sorted non-empty dependency output without changing install execution behavior.
