## Summary

Keep this PR open. The reported memory_get symlink escape is still credible on current main, but the branch is stale against the current package-owned memory-host-sdk layout, so the hardening needs to be ported before it can safely merge.

## What This Changes

The PR adds realpath-based containment checks for builtin and QMD memory_get reads, exports the helper, and adds regression tests for directory-symlink escapes and transient missing-file races.

## Best Possible Solution

Rebase or replace the PR by moving the realpath containment helper, builtin read changes, and exports into packages/memory-host-sdk plus the current plugin SDK facade, keep src bridges thin, preserve the QMD read-path hardening, and add the directory-symlink regression tests against the current package-backed runtime.

## Reproduction Assessment

Yes. The provided steps are high-confidence and match current source: create an outside Markdown file, add a directory symlink under memory/, an extra path, or a QMD collection, then call memory_get; current main checks lexical paths and final lstat but not the canonical target root.

## Solution Assessment

No, not as currently shaped for main. Canonical realpath containment is the right primitive, but the patch needs to target the package-owned runtime/export surface and should clarify the intended workspace-internal symlink boundary before merge.
