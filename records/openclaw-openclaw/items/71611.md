## Summary

This PR should stay open but cannot merge as submitted: the intended Windows retry/fallback direction matches the linked memory-core EBUSY bug, but the patch introduces invalid TypeScript and drops SQLite sidecar and backup uniqueness behavior.

## What This Changes

This PR adds a memory-core rename retry/copy fallback helper and routes atomic-reindex file moves through it.

## Best Possible Solution

Revise this PR or replace it with a narrow memory-core patch that ports the cron-store retry/fallback behavior without changing suffix handling or backup uniqueness, adds focused tests for retry/fallback and sidecars, and leaves broader unsafe-reindex behavior to separate follow-up work.

## Reproduction Assessment

Yes. The PR defects reproduce by static inspection of the patch as invalid TypeScript, and the underlying product bug has concrete Windows reproduction steps and logs in #64187.

## Solution Assessment

No as submitted. Retrying EBUSY and falling back for Windows rename failures is a reasonable narrow direction, but this patch must preserve the original template literals, sidecar handling, UUID backup name, and add regression tests before it is mergeable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.97

Full review comments:

- **[P1] Restore sidecar path template literals:** `extensions/memory-core/src/memory/manager-atomic-reindex.ts:31-32`
  - body: The new `const source = ${sourceBase};` and `const target = ${targetBase};` lines are invalid TypeScript. They also drop the `suffix` value, so WAL and SHM sidecars would no longer move with the base SQLite DB if this were repaired incorrectly.
  - confidence: 0.99
- **[P1] Keep removing WAL and SHM sidecars:** `extensions/memory-core/src/memory/manager-atomic-reindex.ts:45`
  - body: `fs.rm(${basePath}, ...)` is invalid TypeScript and no longer includes the `suffix` for each sidecar. Restore the template literal so cleanup removes the base DB, WAL, and SHM files together.
  - confidence: 0.99
- **[P1] Preserve unique backup paths:** `extensions/memory-core/src/memory/manager-atomic-reindex.ts:49`
  - body: The new backup path expression is invalid TypeScript and removes `randomUUID()`. Without a unique backup filename, overlapping swaps could collide on the same backup path and break rollback safety.
  - confidence: 0.98
