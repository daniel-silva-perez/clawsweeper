## Summary

Keep this PR open. The underlying Windows EBUSY bug is still present on current main, and the PR targets the right memory atomic-reindex file, but the submitted patch is not mergeable because it introduces invalid TypeScript and breaks SQLite sidecar and backup path handling.

## What This Changes

This PR adds a memory-core `renameWithRetry` helper for EBUSY retries and EPERM/EEXIST copy fallback, then routes memory atomic-reindex file moves through it.

## Best Possible Solution

Revise this PR or replace it with a narrow maintainer patch that ports the cron-store retry/fallback behavior into memory atomic reindex, preserves sidecar suffixes and unique backup names, adds focused regression tests, and leaves broader unsafe-reindex behavior to separate issues.

## Reproduction Assessment

Yes. The PR defects reproduce by inspecting/parsing the changed file, and the underlying product bug has a concrete Windows reproduction and logs in #64187; I did not run a live Windows repro in this read-only review.

## Solution Assessment

No as submitted. The implementation direction is sound, but the current diff must restore the original template literals, suffix handling, and UUID backup path, then add tests for retry/fallback behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.97

Full review comments:

- **[P1] Restore sidecar path template literals:** `extensions/memory-core/src/memory/manager-atomic-reindex.ts:31-32`
  - body: The new `const source = ${sourceBase};` and `const target = ${targetBase};` lines are invalid TypeScript, so the file will not compile. They also drop the `suffix` value, so the WAL and SHM files would no longer move with the base SQLite DB if this were repaired incorrectly.
  - confidence: 0.99

  - body: `fs.rm(${basePath}, ...)` is invalid TypeScript and no longer includes the `suffix` for each sidecar. Restore the template literal so cleanup removes the base DB, WAL, and SHM files together.
  - confidence: 0.99
- **[P1] Preserve unique backup paths:** `extensions/memory-core/src/memory/manager-atomic-reindex.ts:49`
  - body: The new backup path expression is invalid TypeScript and removes `randomUUID()`. Without a unique backup filename, overlapping swaps could collide on the same backup path and break rollback safety.
  - confidence: 0.98
