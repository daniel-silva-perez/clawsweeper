## Summary

Keep this PR open. Current main still has the schema/runtime mismatch the PR targets: `message.send` advertises base64 `buffer` attachments, but `send` is excluded from attachment hydration and the runner/delivery paths only treat URL/path media as sendable attachments. The open related bug #53641 and overlapping PR #63578 are implementation context, not proof that this PR has been superseded or shipped.

## Best Possible Solution

Keep this PR open and review it with #63578 and #53641. The best fix is in the shared outbound send path: hydrate `send` media hints including `buffer`, materialize base64 through the existing media store with channel/account limits, pass the resulting media path through plugin and core delivery, and add focused regression tests for buffer-only and buffer-with-message sends.
