## Summary

Keep this PR open. Current main still has the schema/runtime mismatch the PR is trying to fix: `message.send` advertises `buffer`, but the shared attachment hydration gate skips `send`, and the send runner only routes URL/path-style media into downstream delivery. The related #63578 and #53641 are still open, so this PR remains useful implementation evidence rather than something already shipped or safely closeable.

## Best Possible Solution

Keep this PR open and review it together with #63578 and #53641. The likely product fix belongs in the shared outbound send path: hydrate `send` media hints including `buffer`, materialize base64 through the existing media store with channel/account limits, pass the resulting media path through plugin and core delivery, and add focused regression tests for buffer-only and buffer-with-message sends.
