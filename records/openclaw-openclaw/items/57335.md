## Summary

Keep this PR open. Current main still advertises `buffer` for `message.send`, but the runtime path still excludes `send` from attachment hydration and delivery only consumes URL/path-style media. The open related bug #53641 and overlapping PR #63578 are implementation context, not proof that this PR has been shipped or cleanly superseded.

## Best Possible Solution

Keep this PR open and review it together with #63578 and #53641. The best fix is one canonical outbound send implementation that hydrates `send` media hints including `buffer`, safely materializes base64 through the existing media store with channel/account limits, forwards the resulting media path through plugin and core delivery, and adds focused regression tests for buffer-only and buffer-with-message sends.
