## Summary

Keep this PR open. Current main still gates memory-wiki search on the full normalized query string, so the multi-term token-overlap behavior proposed here has not shipped. The latest PR diff is focused on the affected memory-wiki query path and tests, and it appears to address the earlier review concerns around proportional matching and digest fallback scanning.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or an equivalent memory-wiki change that replaces full-string-only candidate gates with normalized token-overlap matching, preserves exact phrase priority, keeps digest fallback/body scan behavior covered, and ships the focused regression tests. The earlier closed PR #65569 is superseded by this updated implementation rather than a reason to close it.
