## Summary

Keep open. Current main still resolves Docker sandbox `pidsLimit` only from explicit agent/global config, and `docker create` only emits `--pids-limit` when the resolved value is already positive. The related open PR #44098 contains closing syntax and is a plausible implementation path, so this issue should remain open until that PR or an equivalent fix merges.

## Best Possible Solution

Keep this issue open and review/land #44098 or an equivalent focused security fix. The fix should add a maintainer-approved default PID limit in `resolveSandboxDockerConfig`, preserve explicit agent/global overrides, add regression coverage for default and override behavior, confirm `buildSandboxCreateArgs` emits the resolved default, and update the public sandbox/config docs if the runtime default changes.
