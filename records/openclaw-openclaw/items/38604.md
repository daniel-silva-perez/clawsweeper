## Summary

Keep open. Current main still resolves Docker sandbox `pidsLimit` only from agent/global config, and `buildSandboxCreateArgs` only emits `--pids-limit` when that resolved value is a positive number. The issue is also paired with open PR #44098, so the maintainer path is to review/land that PR or an equivalent security-owned fix before closing. The WASM runtime comment links useful longer-term direction, but it does not replace fixing the Docker sandbox default.

## Best Possible Solution

Keep this issue open and review/land #44098 or an equivalent focused security fix. The fix should add a maintainer-approved default in `resolveSandboxDockerConfig`, preserve explicit agent/global overrides, prove `buildSandboxCreateArgs` emits the resolved default, update config/security docs if runtime behavior changes, and ship through the normal release path before closing as implemented.
