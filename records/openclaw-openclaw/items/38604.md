## Summary

Keep open. Current main still leaves Docker sandbox `pidsLimit` unset unless explicitly configured, and `docker create` only emits `--pids-limit` when the resolved value is positive. The issue is paired with open PR #44098 using closing syntax, so the right maintainer path is to review/land that PR or an equivalent focused security fix before closing.

## Best Possible Solution

Keep this issue open and review/land #44098 or an equivalent focused security fix. The fix should add a maintainer-approved default in `resolveSandboxDockerConfig`, preserve explicit agent/global overrides, prove `buildSandboxCreateArgs` emits the resolved default, add regression coverage for default and override behavior, and update the public sandbox/config/security docs if runtime behavior changes.
