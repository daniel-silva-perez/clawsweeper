## Summary

Keep open. Current main still resolves Docker sandbox pidsLimit only from agent/global config and only emits Docker --pids-limit when that resolved value is a positive number. The issue is also paired with open PR #44098, which explicitly intends to fix this same issue, so it should stay open until that PR or an equivalent maintainer-approved security fix lands and ships.

## Best Possible Solution

Keep this issue open and review/land PR #44098 or an equivalent focused security-owned fix. The implementation should add a maintainer-approved default in resolveSandboxDockerConfig, preserve explicit global and per-agent overrides, prove buildSandboxCreateArgs emits the resolved default, update public sandbox/config/security docs if runtime behavior changes, and close this issue only after the fix merges and ships.
