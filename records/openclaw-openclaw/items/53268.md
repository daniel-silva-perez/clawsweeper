## Summary

Keep this issue open. Current main still has no configurable URL or hostname deny list for `web_fetch`; the config types, strict runtime schema, generated schema/help, docs, runtime network-guard wiring, and SSRF tests only cover existing private/internal host blocking plus allowlist/opt-in SSRF controls.

## Best Possible Solution

Keep this issue open. If maintainers want the feature, implement a typed and documented `tools.web.fetch.denyHosts`-style option or a shared network-policy deny field; validate exact and `*.domain` patterns case-insensitively; enforce it before cache lookup, DNS, direct fetch, redirects, and provider fallback; return concise blocked-fetch errors; update generated schema/help/docs; and add focused config plus runtime SSRF tests.
