## Summary

Keep this PR open. Current main only shipped the narrower zero-valued poll-default fix, while the broader non-zero poll-param and Discord empty-modal behaviors remain unresolved; the PR itself is stale against the current Discord/plugin send architecture and should not be merged as-is.

## What This Changes

The PR changes message send handling to reject explicit poll questions, strip non-question poll fields and empty component skeletons from send params, and updates poll/send regression tests.

## Best Possible Solution

Use this PR as source material, but adapt the implementation to current main: keep the poll-send policy explicit in the shared send path, handle empty Discord modal/component skeletons inside the Discord parser/runtime, and add focused regressions for the linked poll and component failures before closing the related reports.
