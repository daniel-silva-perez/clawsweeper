## Summary

Keep this PR open. Current main has only shipped the narrower zero/default poll-param fix, while the broader non-question poll-noise behavior remains a product decision and the Discord empty-modal cleanup now belongs at the Discord plugin boundary rather than the older core component path. This PR is still useful source work, but it needs a current-main rebase/adaptation and coordination with overlapping open work before maintainers can merge or close it. https://github.com/openclaw/openclaw/pull/46455

## What This Changes

The PR changes message send handling to reject explicit `pollQuestion` on `send`, strip non-question poll fields and empty component skeletons from send params, and updates poll/send regression tests.

## Best Possible Solution

Use this PR as source material, but rebase or reimplement it on current main: handle poll-noise sanitization in the shared message send path while preserving a corrective error for explicit `pollQuestion`, and handle empty Discord modal/component skeletons in the Discord plugin parser/runtime. Add focused regression tests for #42820 and #43015, then close the linked issues only after the chosen implementation lands.
