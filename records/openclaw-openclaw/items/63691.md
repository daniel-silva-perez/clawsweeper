## Summary

Keep open. Current main has useful opt-in transcript compaction and diagnostics, but the default gateway/channel run path can still synchronously read and parse a large active session JSONL before opening the run, matching the concrete 89 MB transcript freeze described in the issue discussion.

## Best Possible Solution

Keep this issue open and fix the default embedded gateway run path. Add a cheap pre-open stat guard before repairSessionFileIfNeeded and SessionManager.open; for oversized active transcripts, either rotate/compact with bounded yielding or worker isolation, or fail fast with an actionable recovery message while keeping Gateway HTTP/RPC/channel handling responsive. Add regression coverage for an oversized JSONL and document the default recovery path alongside the existing opt-in byte threshold.
