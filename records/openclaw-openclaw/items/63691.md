## Summary

Keep open. Current main has opt-in active-transcript compaction and diagnostics, but the default embedded Gateway run path still synchronously repairs and opens the active JSONL before any default size guard, so the 89 MB transcript freeze described in the discussion remains plausible for Gateway and channel users.

## What This Changes

The issue requests a more stable update path after Gateway hangs and WeChat/Feishu channel unresponsiveness, including stronger regression testing, known-issue changelog notes, gradual rollout, and rollback support.

## Best Possible Solution

Add bounded default recovery in the embedded Gateway run path before session repair/open: use a cheap stat guard for oversized active transcripts, keep HTTP/RPC/channel handling responsive, and either rotate/compact through a bounded path or fail fast with an actionable recovery message. Keep existing beta/rollback docs, and document the default recovery behavior alongside the opt-in compaction threshold.

## Repair Work Prompt

Fix the Gateway/channel freeze reported in https://github.com/openclaw/openclaw/issues/63691. The concrete failure is a very large active session JSONL causing the embedded Gateway run path to whole-file read/parse before SessionManager.open, monopolizing the Node event loop and freezing channel handlers. Add a bounded default pre-open guard before repairSessionFileIfNeeded and SessionManager.open in the embedded Pi run path. Prefer a conservative stat-based guard that keeps HTTP/RPC/channel handling responsive and either rotates/compacts through an existing bounded transcript-rotation path or fails fast with an actionable recovery message; do not raw-split transcripts, silently delete user history, change WeChat/Feishu plugin behavior, or upgrade Pi/runtime dependencies. Preserve the existing opt-in maxActiveTranscriptBytes semantics unless intentionally adding a separate default emergency guard. Add focused regression coverage for an oversized active JSONL and update docs/reference/session-management-compaction.md plus a single-line changelog fix entry crediting Thanks @deciia and @HoroTW.
