## Summary

Keep open. Current main still lacks any injected build-time commit source and continues to let live git metadata win before build-info, so the PR targets a real remaining behavior gap; however the PR is not merge-ready as-is because this user-visible fix lacks the required changelog entry and its current checks include failures.

## What This Changes

The PR adds a tsdown build-time `__OPENCLAW_GIT_COMMIT__` define, reads it in `resolveCommitHash` before live git, and adds unit coverage for injected commit priority, env override, fallback, and caching.

## Best Possible Solution

Land this PR or an equivalent narrow implementation after adding the required changelog entry, rebasing onto current main, and getting current checks green. The desired end state is a stable build identity priority of explicit env override, baked build-time commit, live git for source/dev runs, then build-info/package fallbacks.

## Reproduction Assessment

Yes. The issue body gives a concrete build, move-HEAD, rerun-`--version` repro, and current main source confirms the failure mode because root `--version` calls `resolveCommitHash` while the resolver reads live git before build-info.

## Solution Assessment

The implementation approach is the right narrow fix: bake a sanitized commit into node build outputs and consult it after explicit env overrides but before live git. This PR is not merge-ready as-is until it gets the required changelog entry and current-main validation.
