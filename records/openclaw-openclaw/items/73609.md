## Summary

Keep open. Current main still has the ACP cleanup shape described in the report: terminal ACP task flows can be marked complete while persistent ACP session metadata remains open/resumable, runtime re-ensure can resume persistent identities, and parent-owned persistent ACP sessions are still classified as interactive rather than background/quarantined. The provided context also correctly separates this from closed #73202, which covered live npx adapter resolution rather than stale session lifecycle and Telegram routing.

## Best Possible Solution

Keep this issue open and fix it in the ACP owner path. When a parent-owned ACP child task or mirrored task flow reaches a terminal state, reconcile the task/flow with ACP session metadata, close or mark orphaned persistent ACPX records non-resumable when no active consumer remains, unbind or quarantine stale channel bindings, and add doctor/tasks-audit repair plus regression coverage for Telegram requester delivery not being hijacked by a stale ACPX session.
