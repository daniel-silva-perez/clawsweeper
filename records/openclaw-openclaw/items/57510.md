## Summary

Keep this PR open: current main does not implement a `pre_route` hook, and the PR is a meaningful implementation candidate, but it is not ready to merge because the latest head still ignores `routeOverride.agentId` on the ACP dispatch path and the branch is dirty against current main.

## What This Changes

The PR adds a blocking `pre_route` plugin hook for same-turn route/session overrides before inbound dispatch, with SDK docs and tests for duplicate handling, timeout fail-open behavior, native target sessions, and agent override forwarding.

## Best Possible Solution

Rebase the feature onto current main, add the hook to the current `hook-types.ts`/runner/docs surfaces, and make route/session/account/agent overrides apply consistently before every dispatch branch, including ACP, with focused tests for both ACP and non-ACP turns.

## Reproduction Assessment

Yes. A source-level reproduction is clear: register `pre_route` to return a session key that does not encode the desired agent plus `agentId: "codex"`, then use an ACP-enabled turn; the PR branch calls ACP dispatch before the later `replyResolver` receives `agentIdOverride`.

## Solution Assessment

No, not as currently proposed. The hook direction is plausible, but the branch is stale and the implementation should either thread the agent override through ACP resolution/dispatch as well or narrow the public contract so plugin authors cannot request an override that one dispatch path ignores.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Thread agent overrides into ACP dispatch:** `src/auto-reply/reply/dispatch-from-config.ts:819`
  - body: At this line the override is only passed to the non-ACP reply resolver, but `tryDispatchAcpReply` runs earlier with only `sessionKey`. For ACP-enabled sessions whose key does not embed the requested agent, ACP can resolve and run under the session/default agent before this option is used, so `pre_route` can still dispatch to the wrong workspace/model. Pass the resolved agent override into ACP resolution/dispatch too, or remove `agentId` from the public result until all dispatch paths honor it.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: No supply-chain or dependency issue was found, but the incomplete agent override creates a cross-agent routing/policy concern for ACP turns.

  - body: The public hook result lets a plugin select an `agentId`, but ACP dispatch still derives the agent from session metadata/defaults/session key before the override reaches non-ACP reply execution. That can route a turn through a different agent workspace or policy than the hook selected.
