## Summary

Current main still has the reported behavior path: unset Telegram `reactionLevel` resolves to `minimal`, Telegram publishes that as prompt `reactionGuidance`, and the shared system-prompt builder emits the exact `Reactions are enabled for Telegram in MINIMAL mode.` text. This is not stale and not implemented away; it needs maintainer/product judgment because the current docs also describe `minimal` as the default while a TypeScript comment still says `ack` is the default.

## Best Possible Solution

Keep this issue open for maintainer direction on Telegram reaction default semantics. The clean path is likely to make prompt-affecting reaction guidance explicit and stable: either add a separate documented `channels.telegram.reactionGuidance` config with generated schema/metadata/docs/tests, or define an upgrade/migration/doctor path that pins existing installs so unset `reactionLevel` does not suddenly change live prompts.
