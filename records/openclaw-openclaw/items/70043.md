## Summary

Current main still has the reported behavior path: unset Telegram `reactionLevel` resolves to `minimal`, Telegram exposes that as prompt `reactionGuidance`, and the shared system prompt emits the exact `Reactions are enabled for Telegram in MINIMAL mode.` text. This is not implemented away, stale, or fully superseded by #9835; it still needs maintainer direction because the runtime default, docs, TypeScript comments, and existing-session stability are not aligned.

## Best Possible Solution

Keep this issue open for maintainer direction on Telegram reaction default semantics. The best path is to make prompt-affecting reaction guidance explicit and stable: either introduce a documented `reactionGuidance` config/default contract that composes with `reactionLevel`, or add an upgrade/migration/doctor path that pins existing unset installs so live prompts do not appear to switch modes without user intent. Update Telegram runtime code, tests, generated config metadata, TypeScript comments, and public Telegram docs together.
