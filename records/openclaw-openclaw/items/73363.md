## Summary

Keep open. Current main still does not implement this PR's intended fix: the embedded runner creates reasoning payloads internally but drops `isReasoning` at the returned payload boundary, and the cron isolated-agent helper pickers still skip only `isError`. The linked bug report and prior PR discussion identify the same remaining Matrix/Feishu cron announce leak path, so this PR remains an active implementation candidate, not cleanup material.

## Best Possible Solution

Keep this PR open and review it as the focused implementation candidate for the #73186 Matrix/Feishu cron announce reasoning leak. The likely path is to land this PR or a maintainer-equivalent patch after targeted validation, then decide whether the broader open #40534 should be closed as superseded or mined for any remaining deliverable-payload/error-recovery coverage.
