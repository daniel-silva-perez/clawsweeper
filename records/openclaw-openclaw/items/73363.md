## Summary

Keep open. Current main still has the reported cron announce reasoning-leak path, and this PR is a focused implementation candidate, but the latest diff is not ready to treat as conservative cleanup or clean merge evidence because it intentionally preserves reasoning-only fallback delivery while the linked report and competing PR expect cron announce to avoid user-visible reasoning blocks.

## What This Changes

The PR carries `isReasoning` from embedded-runner payload construction into returned payloads, filters reasoning payloads from cron announce summary/output/delivery primary selection, adds regression tests, and adds a changelog entry.

## Best Possible Solution

Land one canonical cron announce fix that propagates `isReasoning` and excludes reasoning payloads from summary, output, direct delivery, first-text fallback, and error-recovery success checks unless maintainers explicitly decide that reasoning-only cron results should remain user-visible.

## Reproduction Assessment

Yes. A static reproduction is enough: current main can create a reasoning payload in `buildEmbeddedRunPayloads()`, drops the marker before returning it, and cron helper selection then treats reasoning text like ordinary successful text.

## Solution Assessment

No as-is. The marker propagation and primary-pass picker filtering are the right direction, but the reasoning-only fallback and related error-recovery path need to be reconciled with the linked bug and #40534 before this is the narrowest maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Avoid delivering reasoning-only cron payloads:** `src/cron/isolated-agent/helpers.ts:229-236`
  - body: The new filter removes `isReasoning` from the primary deliverable set, but when every payload is reasoning the code falls back through `pickLastDeliverablePayload()` and returns that reasoning text; the added tests codify this. Feishu has no downstream `isReasoning` guard and direct cron delivery sends `deliveryPayloads`, so a reasoning-only cron run can still leak thinking content into the announce message. Please make reasoning payloads non-deliverable in the fallback path too, or get an explicit maintainer decision that this fallback is intended.
  - confidence: 0.82
