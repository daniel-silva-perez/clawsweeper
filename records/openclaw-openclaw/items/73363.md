## Summary

Keep open. Current main still lacks the PR's central fix: `buildEmbeddedRunPayloads()` creates reasoning reply items but drops `isReasoning` from the returned payload, and the cron isolated-agent summary/output helpers still skip only `isError`. The related bug remains open in the provided context, and the broader #40534 PR is also open rather than shipped, so this PR is still a plausible implementation candidate.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this focused fix or a maintainer-equivalent patch that preserves `isReasoning` from `buildEmbeddedRunPayloads()` and makes cron announce selection prefer non-reasoning user-visible text, while explicitly deciding whether the broader #40534 behavior should be folded in or handled separately for #73186.
