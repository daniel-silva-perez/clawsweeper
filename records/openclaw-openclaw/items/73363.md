## Summary

Keep open. Current main still lacks the PR's central fix: `buildEmbeddedRunPayloads()` creates reasoning reply items but does not carry `item.isReasoning` into the returned payload, and the cron isolated-agent summary/delivery helpers still filter only `isError`, not reasoning payloads. The linked bug remains open, and the broader related #40534 PR is also open rather than merged, so this PR is still a plausible focused implementation candidate.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this focused fix or a maintainer-equivalent patch that propagates `isReasoning` from `buildEmbeddedRunPayloads()` and makes cron announce selection prefer non-reasoning user-visible text, while deciding whether the broader #40534 behavior should be folded in or handled as a separate follow-up for #73186.
