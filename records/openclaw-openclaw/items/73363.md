## Summary

Keep open. Current main still has the core bug this PR targets: `buildEmbeddedRunPayloads()` creates a reasoning payload internally but drops `isReasoning` in the returned payload map, and the cron isolated-agent summary/text helpers still only skip `isError`. The PR is not implemented on main and remains a plausible fix candidate for #73186, but maintainers should compare it with the broader open #40534 because current delivery-payload selection also does not filter `isReasoning`.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this focused fix only after confirming the direct announce delivery path cannot still send reasoning payloads, or to fold its tested `isReasoning` propagation into the broader #40534 approach. The final patch should carry `isReasoning` through `buildEmbeddedRunPayloads()`, filter reasoning from cron summary/output and deliverable-payload selection, and add regression tests for propagation plus Matrix/Feishu-style cron announce selection.
