## Summary

Current main still does not implement direct video upload to LLM chat models. Control UI still filters out `video/*` and common video extensions before upload, Gateway only has native image blocks plus generic managed-media staging for non-images, and the related open draft PR #74082 is partial groundwork that explicitly leaves provider-native video delivery/design open.

## Best Possible Solution

Keep this issue open as the product/design tracker for first-class direct video chat attachments to LLMs. Maintainers should review #74082 for the UI/Gateway fallback groundwork, then decide the remaining native-provider contract: accepted formats and size limits, preview/transcript metadata, managed-media fallback, provider/model video capability routing, provider upload/offload policy, and docs/tests across Control UI, Gateway, catalog metadata, and provider dispatch.
