## Summary

Keep this issue open. Current main has adjacent progress: Gateway `chat.send` now accepts non-image attachments as staged media paths, and media-understanding can summarize video when configured. But the requested direct MP4/WebM/MOV chat upload workflow is still not implemented because the Control UI explicitly filters video files and the agent dispatch path still exposes native prompt attachments only as images, not as a video-capable content model delivered directly to provider chat APIs.

## Best Possible Solution

Keep this issue open until OpenClaw has first-class direct video chat attachments: allow and preview MP4/WebM/MOV in the Control UI, define a video-capable chat attachment/content contract, persist transcript metadata, route by provider/model video capability with size/offload policy, preserve media-understanding fallback where appropriate, and cover UI, Gateway parsing, agent dispatch, media-understanding/native-provider behavior, and docs with focused tests.
