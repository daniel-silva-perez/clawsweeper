## Summary

Keep this issue open. Current main still lacks the requested declarative plugin-injected context tag contract: manifests do not declare injected tags, prompt hook result types still carry only raw text injection fields, and downstream sanitizer/ingestion paths still use local hardcoded knowledge for known tags. This is a public plugin manifest/SDK and privacy/sanitization design issue, so it needs maintainer API review rather than autoclose, ClawHub routing, or a narrow automated fix PR.

## Best Possible Solution

Keep this issue open for maintainer API review. The likely path is an additive, documented manifest/SDK contract for plugin-owned injected context blocks, an aggregated registry/getter built from plugin manifests, and staged adoption by display/history, assistant-visible text, compaction, session export, and memory/dreaming ingestion paths with tests for user-authored lookalikes and fenced-code preservation. Coordinate with #54373, but do not treat it as a duplicate.
