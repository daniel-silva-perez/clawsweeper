## Summary

Keep open. Current main has prompt-mutating hook surfaces and several point sanitizers for known metadata/memory blocks, but it does not have the requested declarative plugin manifest registry or core getter for plugin-injected context tags. This is a public plugin-manifest/SDK contract with downstream sanitization and security implications, so it needs maintainer API review rather than auto-close or ClawHub routing.

## Best Possible Solution

Keep this issue open for maintainer API review. The likely best path is an additive, documented manifest/SDK contract for plugin-owned injected context blocks, a central registry/getter built from plugin manifests, and careful adoption by display/history, assistant-visible text, compaction, session export, and memory/dreaming ingestion paths with tests for user-authored lookalikes and fenced-code preservation. Coordinate with #54373 rather than closing this as a duplicate, because provenance and downstream stripping solve adjacent but different problems.
