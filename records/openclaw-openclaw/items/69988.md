## Summary

Keep this PR open. Current main still exposes compaction only at `agents.defaults.compaction`; it does not accept or resolve `agents.list[].compaction`, and the Pi runtime compaction paths still read defaults-only config. The provided PR discussion shows active review follow-up, duplicate cleanup, and production context, so the best cleanup action is maintainer review of this focused implementation rather than closing it.

## Best Possible Solution

Keep this PR open as the current focused implementation path. Maintainers should review and land or request changes for an agent-aware config/schema/resolution update that accepts `agents.list[].compaction`, deep-merges it over `agents.defaults.compaction` where appropriate, threads the effective compaction config through Pi settings, safeguard runtime setup, direct/manual compaction, and run attempts, then updates generated schema/tests/docs.
