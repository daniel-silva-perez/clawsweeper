## Summary

Keep this PR open. Current main still contains the WhatsApp mention-detection bug that #49317 reports, and this PR is not safely obsolete. The PR patch is relevant but appears broader than its stated contract: it moves identity-overlap matching ahead of self-chat handling for all conversations, which would also make explicit `isSelfChat: true` and true self-chat mention cases return true. The related contributor PR #73453 is still open and shows a narrower direction: confine only the implicit allowFrom-derived self-chat suppression to non-group conversations while preserving explicit self-chat overrides.

## Best Possible Solution

Keep the PR open, but request repair before merge. The safest product/code direction is the narrower #73453-style change: only prevent the implicit allowFrom-derived self-chat heuristic from suppressing real group @mentions, while preserving direct self-chat suppression and explicit `mentionCfg.isSelfChat` overrides. Add regression coverage for the reporter's group LID/E.164 case, a true direct self-chat case, and explicit override behavior, then run the WhatsApp changed gate.
