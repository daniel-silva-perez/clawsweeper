## Summary

Keep this PR open. Current main still uses static 128k/8192 limits for LiteLLM and Copilot Proxy onboarding, and no current model-info discovery path is present; the proposed fix is useful but stale against the current plugin-owned layout and still has one correctness bug around partial LiteLLM capability responses.

## What This Changes

The PR adds LiteLLM-compatible `/model/info` probing during LiteLLM and Copilot Proxy onboarding so discovered `max_input_tokens` and `max_output_tokens` can replace the hardcoded 128k/8192 defaults with fallback behavior.

## Best Possible Solution

Land a refreshed implementation in the provider-owned plugin surfaces that probes the configured LiteLLM-compatible model-info endpoint, preserves existing saved limits on probe failure or incomplete data, updates Copilot Proxy model definitions per discovered model ID, and adds focused tests for discovered limits, fallback, authenticated probes, trailing `/v1`, and re-auth preservation.
