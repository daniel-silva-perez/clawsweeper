## Summary

Current main still does not implement this PR's intended fix. Gateway startup still loads internal hooks once from `params.defaultWorkspaceDir`, the hook loader still discovers one workspace hooks directory per call, and `listAgentWorkspaceDirs` is not wired into startup hook loading. The PR remains a plausible focused fix candidate, but it is stale against the current startup path and needs reconciliation with the current explicit opt-in/no-override workspace hook policy plus a security-sensitive review of broader trusted local code loading.

## Best Possible Solution

Keep this PR open as a fix candidate. The best path is a rebase or replacement patch that wires multi-workspace hook loading into `src/gateway/server-startup-post-attach.ts`, uses `listAgentWorkspaceDirs`, loads shared hooks once, loads explicitly enabled workspace-local hooks once per configured workspace, scopes local handlers to the matching event/session/agent workspace, isolates bad workspace failures, preserves the no-override contract, and adds focused Gateway startup regression coverage plus security review for broader startup code execution.
