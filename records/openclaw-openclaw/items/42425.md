## Summary

Current main still does not implement this PR's intended fix. Gateway startup loads internal hooks once from the default workspace, the hook loader still discovers one workspace hooks directory per call, and the existing multi-agent workspace enumerator is not connected to startup hook loading. The PR remains a plausible fix candidate, but it is stale against the current startup file and needs reconciliation with the current explicit opt-in/no-override workspace hook policy plus a security-sensitive review of broader trusted local code loading.

## Best Possible Solution

Keep this PR open as a fix candidate. The best path is a rebase or replacement patch against `src/gateway/server-startup-post-attach.ts` that uses `listAgentWorkspaceDirs`, loads shared hooks once, loads explicitly enabled workspace-local hooks once per configured workspace, scopes local handlers to the matching event/session/agent workspace, isolates bad workspace failures, preserves the current no-override policy, and adds focused Gateway startup regression coverage plus security review for broader startup hook execution.
