## Summary

Keep open because current main still only decays top-level dated memory files, while nested dated paths are treated as evergreen; the linked fix PR is open and unmerged, so this issue should remain the tracker until that work lands or is replaced.

## What This Changes

Request temporal decay to recognize dated markdown files under `memory/` subdirectories while preserving evergreen handling for non-dated memory files.

## Best Possible Solution

Keep this issue open and review the focused linked PR; the desired end state is a small memory-core change that parses dated markdown files below `memory/` subdirectories, leaves `MEMORY.md` and non-dated topic files evergreen, and lands focused regression tests for nested dated paths.
