## Summary

Keep open. Current main still only applies temporal decay to top-level `memory/YYYY-MM-DD.md` files, while nested dated memory paths are still treated as evergreen. #68155 is open and unmerged as the implementation candidate for this issue.

## Best Possible Solution

Keep this issue open and review #68155. If that PR lands, close this as implemented by the merged commit; if it does not land, update `extensions/memory-core/src/memory/temporal-decay.ts` so temporal decay recognizes dated markdown files under `memory/` subdirectories while preserving evergreen non-dated memory files, and add focused regression tests in `extensions/memory-core/src/memory/temporal-decay.test.ts`.
