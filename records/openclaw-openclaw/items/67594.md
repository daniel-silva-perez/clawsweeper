## Summary

Keep open. Current main still treats nested dated `memory/` files as evergreen because the dated-memory matcher only recognizes `memory/YYYY-MM-DD.md`, and the linked implementation candidate #68155 remains open and unmerged.

## Best Possible Solution

Keep this issue open and review #68155. If that PR lands, close this as implemented by the merged commit; otherwise update `extensions/memory-core/src/memory/temporal-decay.ts` so temporal decay recognizes dated Markdown files under `memory/` subdirectories while preserving evergreen non-dated memory files, with focused regression coverage in `extensions/memory-core/src/memory/temporal-decay.test.ts`.
