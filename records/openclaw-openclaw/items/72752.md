## Summary

Keep this PR open. Current main still builds Telegram model button fallback labels from the provider-local model id alone, so an OpenRouter ref like `openrouter/openai/gpt-5.4-mini` still displays `openai/gpt-5.4-mini` when no `modelNames` display name exists. The PR is a narrow implementation candidate for #67792, with focused tests and no apparent security-sensitive changes; maintainer review should decide whether the long-label truncation concern belongs in this PR or a follow-up.

## Best Possible Solution

Keep this PR open as the implementation candidate for #67792. The best path is to review and land the narrow fallback-label/test fix if maintainers accept the current truncation boundary, or ask for a small follow-up in this PR that preserves the provider segment for long nested labels before merging.
