Keep this PR open. Current main still uses the provider-local model id as the Telegram fallback label, so the reported OpenRouter label bug remains present on main; the PR is a narrow implementation candidate with focused tests, while the long-label truncation concern remains a maintainer review call.

## What This Changes

The PR updates the Telegram model keyboard fallback label to include the outer provider when the provider-local model id contains a slash, and adds two regression tests for nested OpenRouter ids.
