## Summary

Keep open. The MiniMax usage-host bug is still present on current main, and this PR is not safe to merge as-is because the branch routes the usage request from the `minimax` usage context/env rather than the credential source; CN `minimax-portal` OAuth can still be sent to the wrong host. The work also overlaps open #65059, so maintainers should consolidate or request a revised MiniMax-owned resolver.

## What This Changes

This PR adds a `baseUrl` option to `fetchMinimaxUsage`, passes a provider/env-derived host from MiniMax provider registration, and expands the MiniMax usage unit test to assert request URL and method.

## Best Possible Solution

Land one MiniMax-owned, region-aware usage endpoint resolver that preserves `GET /v1/api/openplatform/coding_plan/remains`, derives the host from the actual auth/config context for `minimax`, `minimax-portal`, and `MINIMAX_API_HOST`, constrains or clearly documents custom-origin token routing, updates MiniMax docs, and adds regression coverage for global/CN API-key and OAuth setups.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Route MiniMax usage with the token's region:** `extensions/minimax/provider-registration.ts:257-264`
  - body: `resolveUsageAuth` can return a `minimax-portal` OAuth token, but this new `baseUrl` lookup reads `models.providers[ctx.provider]` while `ctx.provider` is the usage provider `minimax`. CN OAuth onboarding stores the host under `models.providers.minimax-portal.baseUrl`, so those users fall through to the global/env fallback and usage polling can send the portal token to the wrong region.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff changes the MiniMax usage bearer-token destination and needs token-source-aware trusted-host routing before merge.

Concerns:

- **[medium] Constrain MiniMax usage token routing:** `extensions/minimax/provider-registration.ts:257`
  - body: The PR derives the usage request origin from `models.providers[ctx.provider].baseUrl` or `MINIMAX_API_HOST` after usage auth may have returned a `minimax-portal` OAuth token. That can send the portal usage token to the API-provider host or another configured origin instead of the matching MiniMax region host.
  - confidence: 0.78
