## Summary

Keep open. Current main still hardcodes the MiniMax usage request to the CN host, so the PR addresses a real remaining MiniMax usage-routing problem, but the proposed patch is not mergeable as-is because it can route a `minimax-portal` OAuth token using the `minimax` provider config/env/global fallback instead of the token's region. The overlapping #65059 should be consolidated with a region-aware fix.

## What This Changes

The branch adds an optional `baseUrl` to `fetchMinimaxUsage`, derives the usage request origin from MiniMax config or `MINIMAX_API_HOST`, switches the default host to `api.minimax.io`, and expands the MiniMax usage test to assert URL and method.

## Best Possible Solution

Land one MiniMax-owned usage endpoint resolver that preserves `GET /v1/api/openplatform/coding_plan/remains`, derives the host from the actual credential source for `minimax`, `minimax-cn`, and `minimax-portal`, constrains or clearly documents custom host routing, updates MiniMax docs, and adds provider-level regression coverage for global API-key, CN API-key, and OAuth setups.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Route MiniMax usage with the token region:** `extensions/minimax/provider-registration.ts:257-264`
  - body: `resolveUsageAuth` can return a `minimax-portal` OAuth token, but this new lookup reads `models.providers[ctx.provider]` while `ctx.provider` is the usage provider `minimax`. CN OAuth onboarding stores its host under `models.providers.minimax-portal.baseUrl`, so those users fall through to the global/env fallback and usage polling can send the portal token to the wrong region.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff changes the MiniMax usage bearer-token destination and needs credential-source-aware trusted-host routing before merge.

Concerns:

- **[medium] Constrain MiniMax usage token routing:** `extensions/minimax/provider-registration.ts:257`
  - body: The PR can route a `minimax-portal` OAuth bearer token using `models.providers.minimax.baseUrl`, `MINIMAX_API_HOST`, or the global default instead of the matching portal region. Endpoint selection should be tied to the credential source or constrained to documented MiniMax hosts.
  - confidence: 0.78
