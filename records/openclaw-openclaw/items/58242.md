## Summary

Keep open. Current main still propagates Slack `already_reacted` from the shared Slack reaction helper and the agent `react` action path. This PR remains a focused helper-level fix with regression coverage, and recent related discussion points toward helper-level handling rather than making this obsolete.

## Best Possible Solution

Keep this PR open for maintainer review and consolidate the Slack duplicate-reaction work around a helper-level fix in `extensions/slack/src/actions.ts`. The best implementation should treat only Slack `already_reacted` from `reactSlackMessage()` as idempotent success, keep unrelated Slack errors propagating, preserve the existing agent `{ ok: true, added: emoji }` payload, and include focused helper-level regression tests. Coordinate or close the narrower #71843 and broader #50881 only after a canonical helper-level patch is chosen.
