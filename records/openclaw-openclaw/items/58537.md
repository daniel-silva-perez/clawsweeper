## Summary

Keep open. Current main still routes bare Feishu `oc_` outbound targets through the Feishu session-route helper as `direct`, while Feishu send-target handling already treats the same ID shape as a chat. The related #52342 PR is open and focused on inbound OpenChat classification, so it does not make this outbound/session-key report obsolete.

## What This Changes

Request to make Feishu OpenChat `oc_` chats use one group session key for outbound and inbound delivery instead of split direct/group keys.

## Best Possible Solution

Keep the issue open and fix the Feishu plugin-owned route normalization first: bare `oc_` targets should produce the same group/chat session route as explicit `chat:oc_...` targets, with regression coverage and a changelog entry. Legacy duplicate session repair should be handled separately through doctor/fix style cleanup if it remains necessary, not as a startup core migration.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists in current main: a bare `oc_...` outbound target reaches `resolveFeishuOutboundSessionRoute` and is not marked as group unless it has an explicit `chat:`/`group:` prefix; I did not run tests or live Feishu because the review is read-only.

## Solution Assessment

Partially. Applying the `oc_` chat-id correction in the Feishu outbound route helper is the narrowest maintainable fix, while generic session-key deduplication and restart reconciliation are broader cleanup mechanisms that should not be the first-line fix for this owner-specific routing bug.
