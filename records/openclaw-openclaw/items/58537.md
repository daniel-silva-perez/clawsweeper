## Summary

Keep open. Current main still has a Feishu outbound session-route mismatch for bare `oc_...` targets: the send layer treats `oc_` as a Feishu `chat_id`, while the outbound mirror/session route records the same bare target as a direct session. The adjacent open PR #52342 covers inbound OpenChat/topic-group classification, but it is not merged and does not fix this outbound route by itself.

## Best Possible Solution

Keep the issue open and fix the Feishu plugin boundary deliberately. The likely patch belongs in `extensions/feishu/src/session-route.ts` plus focused tests: align `resolveFeishuOutboundSessionRoute` with `resolveReceiveIdType`/actual send behavior for `oc_...`, preserve explicit `dm:` and `group:` handling, and add regression coverage for bare `oc_`, `chat:oc_`, `group:oc_`, and `dm:` targets. Separately review #52342 or an equivalent inbound classifier change so inbound OpenChat/topic-group detection and outbound session mirroring share one canonical Feishu chat classification.

## Risks / Open Questions

- Closing this now would hide a still-present mismatch between Feishu delivery target classification and outbound session metadata for bare `oc_...` targets.
