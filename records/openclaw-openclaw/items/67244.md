## Summary

Keep open. Current main still does not fully fix either reported source-level behavior: ACP backend lookup remains process-local unless the acpx plugin service has started in that process, and the explicit agent JSON path still returns the raw embedded run result without a sessions_yield descendant-final reconciliation or a completed liveness state.

## Best Possible Solution

Keep this issue open and patch source. Add a policy-aware way for explicit embedded ACP runs to ensure the acpx backend is registered in the current process before ACP backend lookup. Separately reconcile explicit agent --json results after sessions_yield by draining descendant ACP/subagent completion, refreshing final assistant text, clearing replayInvalid only after proven terminal completion, and documenting/testing the terminal liveness contract.
