## Summary

Keep open. Current main still appears to have the two reported source-level gaps: the JSON agent command skips plugin preload unless `--local` or non-JSON mode is used, ACP backend lookup still depends on a process-local registry entry, and the Gateway agent response path forwards the raw run result instead of reconciling descendant completion after `sessions_yield`. The newer `sessions_yield` paused-state work for #73413 is adjacent but does not satisfy this issue's explicit ACP JSON completion-state requirement.

## What This Changes

The issue requests source fixes for explicit `openclaw agent --json` ACP runs so embedded fallback can see/register the ACPX backend and final JSON reflects completed descendant work after `sessions_yield`.

## Best Possible Solution

Add a policy-aware ACPX backend registration or activation seam for explicit embedded ACP runs, then reconcile explicit agent JSON after `sessions_yield` by waiting for descendant completion and updating final text, replay, and liveness metadata only when terminal completion is proven.
