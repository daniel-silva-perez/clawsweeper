## Summary

Keep open. Current main has partial follow-up work for yielded subagent state, but it still does not satisfy this issue’s two source-level requests: explicit `agent --json` does not reconcile descendant ACP completion into a completed final JSON state, and explicit CLI ACP backend visibility still depends on process-local backend registration via plugin service startup.

## Best Possible Solution

Keep this issue open. Patch source so explicit embedded ACP runs can resolve or register the ACPX backend in the current process without bypassing plugin policy, then add explicit `agent --json` reconciliation after `sessions_yield`: drain descendant ACP/subagent completion, refresh final assistant text/output, and only clear `replayInvalid` or mark `completed` when terminal completion is proven. Cover both embedded CLI and gateway JSON paths with focused regression tests.
