## Repair Work Prompt

Fix openclaw/openclaw issue #67244. Patch source only; do not edit generated dist files. The observable bug is that explicit `openclaw agent --json` ACP runs can fail in an embedded/fallback process because the ACPX backend is only registered in processes that start plugin services, and final JSON can keep stale `replayInvalid: true` / `livenessState: "working"` after `sessions_yield` descendant work completes. Implement a policy-aware backend registration path for explicit embedded ACP runs that honors `plugins.allow`, `plugins.deny`, disabled plugin entries, `acp.enabled`, `acp.backend`, and `OPENCLAW_SKIP_ACPX_RUNTIME`; do not blindly start every plugin service from the CLI. Separately, reconcile explicit agent JSON results after `sessions_yield`: when metadata indicates yielded/working/replay-invalid descendant work, drain descendant ACP/subagent completion through an existing or shared helper, refresh final assistant visible/raw text and payloads when available, mark liveness completed and clear replayInvalid only after terminal completion is proven, and handle the no-final-text terminal case without inventing assistant text. Add regression coverage for explicit `agent --json`, ACP runtime/session metadata, `sessions_yield`, descendant completion, final JSON metadata cleanup, timeout/abort/nonterminal paths, and policy-blocked ACPX registration. Add a user-facing changelog fix entry with `Thanks @sazora`.
