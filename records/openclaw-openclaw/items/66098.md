## Summary

Keep this PR open. Current main partially covers the thinking-label side through session rows/defaults, but it still does not expose or consume an effective reasoning default for Web UI chat rendering. The PR remains a plausible implementation candidate, but it needs rework against the newer session-row thinking-default path and the stale-cache concerns already raised in review.

## Best Possible Solution

Keep this PR open for maintainer review and rework. The best path is a backward-compatible effective reasoning contract for Web UI chat state, preferably aligned with the current session-row/default architecture, while preserving the shipped thinking-default label behavior. Add focused gateway/UI tests for fresh sessions with reasoning defaults and for session/model updates so cached effective state cannot drift. Leave the broader config.patch/defaults work tracked by #64830 unless maintainers choose to expand this PR.
