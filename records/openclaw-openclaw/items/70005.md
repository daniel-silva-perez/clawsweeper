## Summary

Current main partially mitigates the reported dreaming failure mode by filtering heartbeat/cron/system transcript noise and excluding chat-scaffolding tags such as `assistant` from REM reflections. The core request remains valid: REM/dreaming material selection still has frequency-shaped paths and no semantic-importance pre-filter or documented selection strategy knob.

## Best Possible Solution

Keep this open and fix it in memory-core with a conservative internal semantic-density/material-importance filter before REM narrative input. Preserve the existing heartbeat/cron/scaffolding filters, prefer snippets with decisions/preferences/conclusions/unusual events over role labels and repeated routine chatter, and only add a public `dreaming.promptStyle` or strategy config if maintainers explicitly want that product surface.
