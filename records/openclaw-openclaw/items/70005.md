## Summary

Keep open. Current main has partial noise mitigation for dreaming, including transcript hygiene and a REM tag blacklist for role/scaffold words, but it still does not implement the requested semantic-importance or material-density selection layer. REM reflections are still built from concept-tag frequency/counts, and Dream Diary REM inputs are derived from those reflections or the first recent entries rather than a semantic relevance filter.

## Best Possible Solution

Keep this open and fix it inside memory-core with an internal semantic-importance/material-density filter before REM reflections and Dream Diary REM narrative inputs. Preserve existing transcript hygiene and role/scaffolding tag filters, prefer snippets with decisions, preferences, conclusions, or unusual events over repeated routine chatter, and avoid adding a public selection config unless maintainers choose that product surface.
