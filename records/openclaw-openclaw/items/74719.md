## Summary

Keep open. The PR targets a real runtime-hardening gap in reply normalization, but the current patch is not merge-ready: it would reject supported attachment-only chat sends, its reply normalizer hunk is stale against current main, and the chat.send undefined-message path is already schema-validated before the sanitizer runs.

## What This Changes

The PR adds runtime guards around undefined message/text trimming in chat send, reply payload normalization, chat abort command parsing, and TUI stream assembly.

## Best Possible Solution

Adapt the useful runtime narrowing into current main by deriving a safe optional string for reply payload text once and using it through the silent-token, heartbeat, prefix, and sanitization paths, while preserving media-only/channelData-only replies and the existing attachment-only chat.send contract.
