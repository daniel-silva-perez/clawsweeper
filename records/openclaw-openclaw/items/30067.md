## Summary

Current main still lacks the requested proactive first-run/setup heads-up about temporary provider API rate-limit pressure before the TUI/Web chat handoff. The existing code shows setup completion, Control UI/token guidance, hatch selection, web-search notes, and reactive provider error handling, but no proactive warning in the onboarding or Crestodian startup paths.

## Best Possible Solution

Keep this issue open and implement a small onboarding UX change: add a concise first-run provider-rate-limit heads-up in the classic finalizeSetupWizard path before the TUI/Web/Later handoff, and mirror equivalent guidance in Crestodian only if that path can lead directly into normal agent chat. Cover the copy with focused tests and do not change provider runtime, auth, fallback, or daemon behavior.
