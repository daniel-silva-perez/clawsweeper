## Summary

Keep open. Current main still does not implement the PR’s intended Feishu behavior: it resolves media only from the triggering message, builds the agent media payload before quoted-message fetch, and fetched quoted messages retain only parsed text/placeholders rather than raw media keys. The PR remains a focused Feishu plugin fix candidate for the workflow also described in #22957, #29501, #41675, and #47509.

## Best Possible Solution

Keep this PR open or replace it with an equivalent focused Feishu plugin fix. The implementation should preserve raw parent-message media content or structured media keys, resolve quoted media only after group-context visibility checks pass, build the final media payload after all media collection, account for the current audio-preflight ordering, and add quoted-media regression coverage.
