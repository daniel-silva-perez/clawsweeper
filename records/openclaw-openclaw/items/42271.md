## Summary

Keep this PR open. Current main still only resolves media from the triggering Feishu message, fetches the quoted parent later as parsed text, and does not retain the raw parent `body.content` needed to extract `image_key`/`file_key`. The PR discussion describes a real Feishu group-chat workflow, Greptile reviewed the approach, and the author addressed the duplicated media-type-list comment; related Feishu quoted-media attempts/reports remain closed/unmerged rather than shipped fixes.

## Best Possible Solution

Keep this PR open or replace it with an equivalent updated Feishu plugin patch. The fix should preserve raw parent-message media content or structured media keys, resolve quoted media only after the fetched parent passes the current group-context visibility check, build the final media payload after all media collection, account for current audio-preflight ordering, and add quoted-media regression coverage.
