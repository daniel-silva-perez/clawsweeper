## Summary

Keep this PR open. Current main still resolves Feishu media only from the triggering message, builds the agent media payload before quoted-message fetch, and fetches quoted/replied messages only as parsed text. The PR remains a focused Feishu plugin fix candidate for a real workflow also described in #22957, #29501, #41675, and #47509; none of those related items is merged or implemented on current main.

## Best Possible Solution

Keep this PR open or replace it with an equivalent focused Feishu plugin fix. The implementation should preserve raw parent-message media content or structured media keys, apply quoted-context visibility checks before media resolution, resolve quoted media with the parent message id, build the final agent media payload after all media collection, and add quoted-media regression coverage.
