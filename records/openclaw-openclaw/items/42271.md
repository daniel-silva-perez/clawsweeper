## Summary

Keep this PR open. Current main still does not implement the intended Feishu behavior: direct inbound media is resolved, but media attached to a quoted/replied parent message is fetched only as parsed text/placeholders and cannot reach the agent media payload.

## Best Possible Solution

Keep this PR open or replace it with an equivalent focused Feishu plugin fix. The fix should preserve raw parent-message media content or structured media keys, resolve quoted media only after group-context visibility checks pass, build the final media payload after all media collection, account for current audio-preflight ordering, and add quoted-media regression coverage.
