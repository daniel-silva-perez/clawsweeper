## Summary

Keep this PR open. Current main still resolves Feishu media only from the triggering message, builds the agent media payload before the quoted-message fetch, and fetched parent messages still expose parsed placeholders without raw media keys. The PR addresses a real bundled Feishu plugin gap, with useful live verification and related prior attempts, but it needs rebasing and regression coverage before it is landable.

## Best Possible Solution

Keep this PR open or replace it with an equivalent updated Feishu plugin patch. The fix should preserve raw fetched parent-message content or structured media keys, resolve quoted media only after group-context visibility checks pass, build the final media payload after all current and quoted media collection, account for current audio-preflight ordering, and add regression coverage for quoted image/file media plus blocked quoted-context senders.
