## Summary

Keep this open. Current main still has a concrete MSTeams path that can turn a clean channel route into a doubly thread-qualified session key: the Teams handler mutates the route object returned by shared routing, the route cache can retain that mutated miss result, and the thread helper appends to whatever base key it receives.

## What This Changes

The issue reports that Microsoft Teams channel-thread handling can produce a malformed session key with two `:thread:` suffixes after an older thread-qualified session is selected and a new thread id is appended.

## Best Possible Solution

Teams thread routing should derive each thread-qualified session key from a clean immutable base route, avoid mutating cached route objects, strip any pre-existing generic thread suffix before appending the current Teams thread id, and include a regression test proving two different thread messages in the same normalized channel produce exactly one current `:thread:<id>` suffix.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/66771. Current main can produce `base:thread:old:thread:new` for Microsoft Teams channel threads because the Teams message handler mutates `route.sessionKey` returned by `resolveAgentRoute`, `resolveAgentRoute` stores and returns the same route object on cache misses, and `resolveThreadSessionKeys` appends to an already thread-qualified base. Keep the fix narrow: derive Teams thread session keys from a clean base route, avoid mutating cached route objects in the Teams message and feedback/invoke paths, and strip any existing generic `:thread:` suffix before appending the current Teams thread id. Use a generic route-cache hardening change only if it stays backward-compatible and broadly safe. Add regression coverage for two Teams channel-thread messages in the same normalized channel: first old thread, then new thread, asserting the second `SessionKey`/recorded inbound session is exactly `base:thread:new` and contains only one `:thread:` suffix. Also cover the helper with an already-threaded base. Coordinate with #58615/#59294 but do not absorb unrelated outbound targeting, attachment, OAuth, or Microsoft tracker work. Add a single-line changelog fix entry credited `Thanks @hay7a4jhgqjw4`.
