## Summary

Keep open. Current main does not implement `channels.whatsapp.denyFrom`, and the related #69926 thread explicitly treats this PR's global denylist idea as complementary rather than a replacement for its per-group allowlist request. The PR still contains potentially useful WhatsApp access-control work, but it needs cleanup before it is a merge candidate: the branch includes unrelated deployment/config artifacts and unresolved functional/security-review concerns from bot review comments.

## Best Possible Solution

Keep the PR open as a possible implementation candidate, but require a clean revision before maintainer review: drop the private deployment/config files and unrelated ops commits, keep the change scoped to WhatsApp config/schema/runtime/tests/docs, verify the dedicated deny matcher is threaded through all shared access-control helpers, and add regression coverage for `dmPolicy: "open"`, owner self-chat, pairing, group behavior, and command-gate authorization.
