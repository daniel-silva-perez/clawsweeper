## Summary

Keep this PR open. Current main still joins inbound metadata and user content with blank lines in both the normal text path and the captionless-media fallback, and there is no `---openclaw:user-msg---` separator or marker-stripping implementation in source or tests. The PR is a plausible implementation candidate, but the supplied latest diff still has unresolved prompt/sanitizer fixture and marker-stripping concerns from review.

## Best Possible Solution

Keep this PR open for maintainer review and rework. The best path is to centralize metadata/body prompt composition in a shared helper used by normal text, captionless-media fallback, raw-body tests, and prompt-composition fixtures; strip only the injected separator associated with removed OpenClaw metadata; preserve user-authored marker text; then run the changed OpenClaw gates for the auto-reply, gateway sanitizer, UI/session sanitization, and prompt fixture surfaces.
