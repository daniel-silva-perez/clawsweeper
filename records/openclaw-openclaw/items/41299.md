## Summary

Keep this PR open. Current main still joins inbound metadata and the user body with blank lines, has no shipped `---openclaw:user-msg---` marker, and has no sanitizer path for removing that marker. The PR remains a focused bug-fix candidate, but it needs rebase and maintainer review because later review comments identify prompt-composition and sanitizer edge cases that are not resolved on main.

## Best Possible Solution

Keep this PR open. The best path is to rebase it onto current main, add the unique separator in `runPreparedReply` for both normal text and media-only prompt bodies, update prompt-composition helpers/tests to match runtime prompt construction, make sanitizer removal narrowly tied to actual stripped OpenClaw metadata, cover all generated inbound metadata block labels, and run the changed OpenClaw gates before merge.
