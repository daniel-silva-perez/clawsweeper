## Summary

Keep this PR open. Current main still does not implement the intended separator behavior: prompt construction joins inbound metadata and user content with blank lines, the media-only fallback does the same, and the metadata sanitizer has no `---openclaw:user-msg---` removal path. The PR remains a focused bug-fix candidate, but its latest review context still needs maintainer attention before merge.

## Best Possible Solution

Keep this PR open for maintainer review and rework. The best path is to rebase onto current main, introduce a shared prompt-body helper for metadata/body separation, cover both normal text and media-only fallback, update sanitizer logic to remove only the synthetic marker after actual OpenClaw metadata removal, audit all generated inbound metadata labels, align prompt-composition tests, and then run the changed OpenClaw gates.
