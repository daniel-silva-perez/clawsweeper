Keep this PR open. Current main still builds inbound metadata and user text with only blank-line separation, and no `---openclaw:user-msg---` implementation exists on main. The PR remains a relevant bug-fix candidate, but the latest diff still needs maintainer review/rework around separator stripping scope, media-only prompt parity, and prompt-composition test coverage.

## Best Possible Solution

Keep this PR open for maintainer review and rework. The best path is to rebase onto current main, introduce one shared metadata/body separator helper used by both normal text and media-only prompt construction, strip only the injected separator associated with removed OpenClaw metadata, preserve user-authored marker text, update prompt-composition/raw-body tests to match runtime behavior, and then run the changed OpenClaw gates outside this read-only review.
