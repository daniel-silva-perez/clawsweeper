## Summary

Keep this PR open. Current main still joins OpenClaw inbound metadata and the actual user body with blank lines, so the intended fix is not implemented on main. The PR remains a plausible implementation candidate, but the supplied latest diff still needs maintainer review/rework around captionless media, prompt-composition fixtures, and exact sanitizer stripping scope before it should land.

## Best Possible Solution

Keep this PR open for maintainer review and rework. The best path is to introduce a shared prompt-body composition helper used by normal text, captionless media fallback, raw-body tests, and prompt-composition fixtures; strip only the injected separator associated with removed OpenClaw metadata; preserve user-authored marker text; then run the changed OpenClaw gates after the patch is updated.
