## Summary

Keep open. Current main still drops image blocks from ACP tool call result content, while this PR is a focused implementation candidate for that missing mapper behavior. The PR is not obsolete; it should be finished by tightening the image-field validation for whitespace-only values before merge.

## Best Possible Solution

Keep this PR open. The best path is to finish the ACP mapper change on this PR or an equivalent maintainer patch: forward valid image blocks from `extractToolCallContent()` for both partial and final tool updates, reject missing, empty, and whitespace-only `data`/`mimeType`, preserve existing text and fallback behavior, and add focused mapper tests for text-only, image-only, mixed, malformed, empty, whitespace-only, and string fallback cases.
