## Summary

Keep this PR open. Current main still contains the two skill-command resolution behaviors described in the PR: `resolveReplyDirectives` only loads workspace skill commands when configured model aliases exist, and `handleInlineActions` treats an empty upstream `skillCommands` array as already-loaded data. That means the reported `/skill ...` and custom workspace slash-skill cases can still miss command transformation when no model aliases are configured. The supplied discussion adds useful reproduction coverage for direct tool-dispatch workspace skills, so the best path is to review and land this PR or an equivalent fix with the regression test, while addressing the minor duplicate-scan concern raised in review.

## Best Possible Solution

Keep the PR open and have a maintainer review/land it or an equivalent patch. The fix should decouple skill-command discovery from model-alias configuration, preserve a clear loaded-vs-not-loaded contract to avoid unnecessary duplicate scans, and add a focused regression test for `/skill` and/or direct tool-dispatch workspace skill commands when no model aliases are configured.
