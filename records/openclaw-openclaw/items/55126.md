## Summary

Current main still accepts and carries `memorySearch.outputDimensionality` through the generic memory-search config path, but the bundled OpenAI embedding provider does not map it to OpenAI's `dimensions` request field for inline or batch embeddings. The supplied GitHub context also shows an open, unmerged implementation candidate (#55507) with closing syntax for this issue, which is an explicit reason to keep the issue open until that PR or a replacement lands.

## Best Possible Solution

Keep this issue open. Review and update #55507, or replace it with a current-main patch that maps `memorySearch.outputDimensionality` to OpenAI `dimensions` for supported OpenAI embedding models, covers both inline and batch embedding requests, includes the dimension in cache/reindex identity, and adds regression tests for inline bodies, batch JSONL bodies, and cache-key behavior.
