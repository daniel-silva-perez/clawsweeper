## Summary

Current main still drops the configured memory embedding dimension before the OpenAI provider builds inline and batch embedding requests. The issue is also paired with an open, unmerged closing PR, so it should remain open until that PR or a replacement lands.

## What This Changes

Request that `memorySearch.outputDimensionality` be forwarded to OpenAI embedding requests as `dimensions` so reduced-size text-embedding-3 vectors work.

## Best Possible Solution

Keep the issue open and land an updated #55507 or replacement patch that threads `memorySearch.outputDimensionality` into the OpenAI embedding client, sends OpenAI `dimensions` for inline and batch `/v1/embeddings` requests when supported, includes the dimension in provider cache/reindex identity, and adds regression coverage for inline bodies, batch JSONL bodies, and cache-key behavior.
