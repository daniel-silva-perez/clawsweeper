## Summary

Current main still accepts and resolves `memorySearch.outputDimensionality`, but the OpenAI memory embedding path does not carry that value into the OpenAI client, inline `/embeddings` request bodies, batch JSONL request bodies, or provider cache identity. The issue should stay open because the bug remains present and an open, unmerged closing PR already exists for the same work.

## What This Changes

The issue requests plumbing `memorySearch.outputDimensionality` into OpenAI memory embedding requests as the OpenAI `dimensions` parameter so reduced-dimension Matryoshka embeddings work.

## Best Possible Solution

Land an updated #55507 or replacement patch that threads `memorySearch.outputDimensionality` into the OpenAI embedding client, sends OpenAI `dimensions` for both inline and batch `/v1/embeddings` calls when configured, includes the dimension in provider cache/reindex identity, and adds regression coverage for those paths.
