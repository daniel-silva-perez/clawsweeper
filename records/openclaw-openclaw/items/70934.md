## Summary

Keep open. Current main still does not provide the requested value-first first-run onboarding: classic CLI onboarding and macOS onboarding still configure setup/bootstrap before a runnable example task, while Crestodian and Control UI suggestions only partially overlap with the requested flow.

## What This Changes

Request to delay non-essential first-run personalization until after a new user can pick and run a useful example task with sensible defaults.

## Best Possible Solution

Define a maintainer-approved first-run sequence that performs only the minimum model/session setup, presents a small set of useful bootstrap templates or example tasks, runs one task immediately, and then offers optional identity, persona, channel, search, skill, and style customization with docs and focused onboarding tests.

## Reproduction Assessment

Not applicable as a feature request. The current-main verification path is code/docs inspection of CLI, macOS, Control UI, and Crestodian onboarding order, which shows the requested value-first flow is still absent.

## Solution Assessment

Unclear. The requested direction is plausible, but the best implementation needs maintainer UX/product sequencing because it changes when setup, auth, bootstrap/persona, and optional configuration happen across multiple first-run surfaces.
