## Summary

Keep open. Current main still puts first-run setup and bootstrap/persona work ahead of a first useful task in the classic CLI and macOS flows, and Crestodian only partially overlaps as an opt-in modern preview rather than the default onboarding contract.

## What This Changes

Request to reorder first-run onboarding so users can run a useful example or template before optional agent identity, reply style, symbols, and persona customization.

## Best Possible Solution

Define a maintainer-approved first-run contract that performs only mandatory model/session setup, offers a small set of useful starter tasks or templates, runs one immediately, and moves identity, persona, channel, style, search, skill, and other customization into optional follow-up steps with matching CLI, macOS, Control UI, docs, and tests.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection, which shows current CLI and macOS first-run paths still put setup/bootstrap work before the first ordinary useful task.

## Solution Assessment

Unclear. The requested direction is plausible, but the best implementation needs maintainer UX judgment about required auth/setup prerequisites and whether Crestodian, classic CLI, macOS, and Control UI should converge or keep surface-specific variants.
