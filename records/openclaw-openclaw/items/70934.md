## Summary

Keep open. Current main still has a setup-first onboarding flow: CLI QuickStart delays details but still runs workspace, auth/model, gateway, channels, search, skills, and bootstrap setup before the first chat/value moment; macOS similarly places the setup wizard and permissions before the onboarding chat. This is a cross-surface product sequencing change, not a conservative cleanup close.

## What This Changes

Request to reorder first-run onboarding so new users can run a useful template or task before optional identity, persona, reply-style, symbol, channel, and other customization steps.

## Best Possible Solution

Define one maintainer-approved first-run sequence that preserves required security/auth prerequisites, presents a small set of useful starter tasks or templates as soon as a runnable model/session exists, and moves identity/persona/channel/style customization into optional follow-up across CLI, macOS, and docs.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection of the current first-run order, which still shows setup before the first useful task/chat moment.

## Solution Assessment

Unclear. The requested direction is reasonable, but the best implementation needs product and UX judgment about which prerequisites are mandatory before value can be shown and how to keep CLI, macOS, and docs aligned.
