## Summary

Keep open. Current main still does not provide a value-first onboarding default: the classic CLI and macOS first-run paths put setup, auth, bootstrap, and persona work before the first ordinary useful task, while Crestodian is still an opt-in modern preview rather than the default onboarding contract.

## What This Changes

Delay non-essential first-run personalization until after a new user can run a useful starter task or template with sensible defaults.

## Best Possible Solution

Define a maintainer-approved first-run contract that keeps only mandatory model/session/security prerequisites before the first result, offers a small set of useful starter tasks or templates, runs one immediately, and moves identity, persona, channel, style, search, skill, and other customization into optional follow-up steps with matching CLI, macOS, Control UI, docs, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection, which shows the current CLI and macOS first-run paths still sequence setup and bootstrap/persona work before the first ordinary useful task.

## Solution Assessment

Unclear. The requested direction is plausible, but the best implementation needs maintainer UX and security judgment about required auth/setup prerequisites and whether Crestodian, classic CLI onboarding, macOS onboarding, and Control UI should converge or keep surface-specific variants.
