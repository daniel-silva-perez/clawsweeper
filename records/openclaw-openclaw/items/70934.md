## Summary

Keep open. Current main still does not implement a value-first first-run onboarding contract: the recommended classic CLI onboarding path collects setup/auth/model/gateway/channel/search/skills/hooks before the first normal task, macOS local onboarding places chat after connection, wizard, and permissions pages, and the macOS kickoff still asks the user to run the bootstrap/SOUL persona ritual before choosing chat channels. Crestodian improves configless startup, but it is documented as the bare-root helper and `openclaw onboard --modern` preview rather than replacing the recommended `openclaw onboard` flow.

## What This Changes

The issue requests a first-run onboarding flow that shows starter tasks or a quick useful result before optional agent/persona/channel customization.

## Best Possible Solution

Define a maintainer-approved first-run contract that keeps only mandatory model/session/security prerequisites before the first result, offers a small set of useful starter tasks or templates, runs one immediately, and moves identity, persona, channel, style, search, skill, and other customization into optional follow-up steps with matching CLI, macOS, Control UI, docs, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection, which shows current CLI and macOS first-run paths still put setup/bootstrap/persona work before the first ordinary useful task.

## Solution Assessment

Unclear. The value-first direction is plausible, but the best implementation needs maintainer UX and security judgment about which auth, model, gateway, permissions, and trust prompts must remain before any first task.
