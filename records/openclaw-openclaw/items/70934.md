## Summary

Keep open. Current main still lacks the requested value-first onboarding flow: classic CLI and macOS first-run paths configure auth, workspace, gateway, channels/permissions, and bootstrap before the first chat/value moment, while Crestodian only partially overlaps as an opt-in preview path.

## What This Changes

Request to delay first-run personalization and setup choices until after users can pick or run a useful bootstrap/example task and see OpenClaw produce value.

## Best Possible Solution

Define a maintainer-approved first-run flow that does the minimum required model/session setup, presents a small set of useful example tasks or templates, runs one task immediately, and then offers identity, persona, channel, style, search, skill, and other customization as optional follow-up across CLI and macOS with focused docs/tests.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection of CLI, macOS, Getting Started, and Crestodian onboarding order, which still shows setup before the first useful task/chat moment.

## Solution Assessment

Unclear. The requested direction is plausible, but the best implementation needs maintainer UX/product judgment about minimum auth prerequisites, default versus preview flows, and how to share behavior across CLI, macOS, Control UI, and Crestodian without duplicating onboarding logic.
