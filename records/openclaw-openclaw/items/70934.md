## Summary

Keep open. Current main still has a real cross-surface onboarding sequencing gap: classic CLI and macOS onboarding put setup, auth/model, workspace/bootstrap, channels/search/skills, and persona-oriented first-run ritual work before a first ordinary useful task. Crestodian overlaps as an opt-in modern preview, but it is not the default onboarding contract requested here.

## What This Changes

Request to reorder first-run onboarding so new users can run a useful example or template before optional agent identity, reply style, symbols, and persona customization.

## Best Possible Solution

Define a maintainer-approved first-run contract that performs only mandatory setup, offers a small set of useful starter tasks/templates, runs one immediately, and moves identity, persona, channel, style, search, skill, and other customization into optional follow-up steps with matching CLI, macOS, Control UI, docs, and tests.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection, which shows current CLI and macOS first-run paths still put setup and bootstrap/persona work before the first ordinary useful task.

## Solution Assessment

Unclear. The value-first direction is plausible, but the best implementation needs maintainer UX judgment about required prerequisites and whether Crestodian, classic CLI, macOS, and Control UI should converge or keep surface-specific variants.
