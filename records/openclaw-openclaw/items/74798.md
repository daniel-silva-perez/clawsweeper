## Summary

Current `main` and the shipped release line already cover this leak path: gateway reset/delete cleanup retires the session MCP runtime, runtime disposal closes MCP transports/clients, and stdio MCP transports tear down the child process tree. The report matches an older 2026.4.9 behavior rather than the current implementation.

## What This Changes

The issue requests gateway session-end cleanup for session-scoped MCP runtimes so `vibe-trading-mcp` stdio child processes do not accumulate under `openclaw-gateway`.

## Best Possible Solution

Keep the shipped cleanup design. A new report would only be needed if someone can reproduce surviving MCP children on `v2026.4.22` or newer after session reset/delete or after the configured `mcp.sessionIdleTtlMs` backstop.
