## Summary

Keep open. Current main still sends streaming Control UI assistant markdown through the same Lit unsafeHTML markdown path, and the linked implementation candidate #72983 remains open and unmerged with review feedback to address before this issue should close.

## What This Changes

The issue requests a Control UI streaming renderer that keeps completed markdown stable instead of replacing the full unsafeHTML-rendered subtree on each streamed update.

## Best Possible Solution

Keep this issue open and use #72983, or an equivalent focused Control UI patch, to add a streaming-safe render path for assistant markdown while preserving sanitized markdown rendering after the stream completes and adding grouped-render regression coverage.

## Reproduction Assessment

Yes. The issue gives concrete manual steps with a long streaming markdown response, and current-main code inspection confirms active stream text reaches renderGroupedMessage with isStreaming while still rendering markdown through unsafeHTML.

## Solution Assessment

Unclear. A paragraph-stable renderer is the best UX, but the open PR's plain-text streaming fallback is a smaller maintainable fix if its whitespace issue and missing regression coverage are addressed.
