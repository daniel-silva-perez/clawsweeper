## Summary

Keep open. Current main still sends active Control UI assistant streaming text through `renderGroupedMessage` with `isStreaming: true`, but the assistant and tool markdown branches still render changing markdown strings through `unsafeHTML(toSanitizedMarkdownHtml(markdown))`. The linked closing PR #72983 is open and unmerged, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests a streaming-safe Control UI markdown render path so completed paragraphs and code fences do not flicker as assistant content streams in.

## Best Possible Solution

Land PR #72983 after addressing its review feedback, or land an equivalent focused Control UI patch that avoids full markdown DOM replacement while `isStreaming` is true, preserves sanitized markdown rendering after completion, and adds grouped-render regression coverage for multi-paragraph and fenced markdown streaming.

## Reproduction Assessment

Yes. The issue gives concrete manual steps with a long streaming markdown response, and current-main code inspection confirms that active stream text reaches `renderGroupedMessage` with `isStreaming: true` while still using the markdown `unsafeHTML` path.

## Solution Assessment

Unclear. A paragraph-stable renderer is the best UX, but the open PR's plain-text streaming fallback is a smaller maintainable fix if its whitespace issue and missing regression coverage are addressed.
