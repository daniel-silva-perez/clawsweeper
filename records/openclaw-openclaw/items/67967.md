## Summary

Keep open. Current main still routes MiniMax through the generic hybrid Anthropic/OpenAI replay hooks, which keep strict tool-call ID sanitization on the Anthropic-compatible path. The PR remains a focused MiniMax-owned fix, and the security pass found only provider/test changes with no CI, dependency, lockfile, script, publishing, or secret-handling impact.

## What This Changes

The PR adds a MiniMax replay-policy wrapper that disables tool-call ID sanitization for `anthropic-messages` while preserving strict sanitization for OpenAI-compatible MiniMax paths, with a matching regression-test expectation change.

## Best Possible Solution

Land this PR or an equivalent MiniMax-owned replay-policy hook so MiniMax Anthropic-compatible requests echo provider-native tool-call IDs verbatim, while OpenAI-compatible MiniMax requests keep strict ID sanitization. A shared factory option can remain a later cleanup if additional providers need the same knob.
