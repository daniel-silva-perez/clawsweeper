## Summary

Keep open. This is a recent, concrete Linux heartbeat reliability report with a follow-up from another user after the 2026.3.28 timer re-arm fix. Current main has several relevant scheduler hardening changes and tests, but I do not see source, changelog, or related-item evidence that the specific long-gap/silent-stop behavior reported here has been definitively fixed or superseded.

## Best Possible Solution

Keep this issue open and treat it as the canonical Linux long-run heartbeat reliability report unless a narrower merged fix is identified. The best next maintainer path is to request a diagnostics export and heartbeat log slice from an affected current build, then add targeted coverage or instrumentation around last/next heartbeat state, requests-in-flight/session-lane retries, event-loop liveness, and gateway restart/reload paths before closing.
