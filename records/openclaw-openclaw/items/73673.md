## Summary

Keep open. Current main and the latest release still generate the macOS Gateway LaunchAgent with unconditional `KeepAlive=true` and a one-second throttle, tests/audit/docs still pin that policy, and the linked closing PR is open rather than a landed fix.

## What This Changes

The issue requests a debuggable macOS launchd failure behavior so gateway startup crashes do not enter immediate restart loops under the generated LaunchAgent.

## Best Possible Solution

Keep this issue open and land a focused macOS launchd early-failure strategy, either by completing PR #73849 or replacing it with a coordinated plist, audit, run-loop, tests, docs, and changelog change that preserves intentional supervisor relaunches.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: generate or install the LaunchAgent, trigger an invalid gateway startup configuration, and launchd will supervise a process whose generated plist still has unconditional `KeepAlive=true` with a one-second throttle.

## Solution Assessment

No. Current main has not solved the issue, and `SuccessfulExit=false` alone is not sufficient because current launchd-supervised restart/update paths still rely on clean exit relaunch semantics.
