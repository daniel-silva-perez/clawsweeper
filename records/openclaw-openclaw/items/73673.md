## Summary

Keep open. Current main and the latest release still generate the macOS LaunchAgent with unconditional `KeepAlive=true` and `ThrottleInterval=1`; tests, service audit, and docs continue to pin that behavior. Open PR #73849 is paired with this issue using closing syntax but remains open/unmerged, so this issue should stay open until that path is reviewed, narrowed, replaced, or landed.

## What This Changes

The issue requests a debuggable macOS launchd behavior for gateway startup failures instead of immediate repeated restarts that obscure the root cause.

## Best Possible Solution

Keep this issue open and land a focused macOS launchd early-failure strategy that preserves intentional `openclaw gateway restart` and update relaunches while adding a debuggable stop, backoff, or repeated-failure path for startup crashes, with plist generation, audit checks, tests, and docs updated together.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: install or generate the LaunchAgent, trigger an invalid gateway startup configuration, and launchd will see a nonzero exit while the generated plist still has unconditional `KeepAlive=true` with a one-second throttle.

## Solution Assessment

No. Current main has not solved the issue, and a plist-only switch to `SuccessfulExit=false` is not sufficient because launchd would still restart nonzero startup failures while risking current clean-exit supervised restart and update flows.
