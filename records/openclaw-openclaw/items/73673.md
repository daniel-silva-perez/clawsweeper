## Summary

Keep this issue open. Current main still matches the reported launchd restart-loop behavior: the generated Gateway LaunchAgent uses `RunAtLoad=true`, `KeepAlive=true`, and a one-second throttle, tests preserve that policy, doctor recommends restoring missing `KeepAlive`, and the stop path disables relaunch only for explicit operator stops. The related launchd reports cover narrower lock-contention or config-restart cases, not a general debug/backoff path for arbitrary startup failures.

## Best Possible Solution

Keep the issue open and design a targeted macOS launchd crash-loop/debug path. The likely implementation belongs around `src/daemon/launchd-plist.ts`, `src/daemon/launchd.ts`, restart handoff, and service audit/doctor guidance, with tests in `src/daemon/launchd.test.ts` and docs updates for macOS Gateway lifecycle. A good fix would preserve fast explicit restarts while adding configurable or automatic backoff, stop-after-N early failures, or clear debug-mode guidance for repeated startup crashes.
