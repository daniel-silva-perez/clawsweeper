## Summary

Keep open. Current main still generates a macOS Gateway LaunchAgent with `KeepAlive=true` and a one-second throttle, the tests explicitly preserve that policy, and the docs describe crash relaunch as intended launchd behavior. The linked open PR uses closing syntax but is unmerged, so this issue should remain open while maintainers review that PR or a replacement fix.

## What This Changes

The issue requests a macOS launchd supervision change so Gateway startup crashes do not create rapid restart loops that obscure debugging.

## Best Possible Solution

Keep this issue open and resolve it through PR #73849 or a replacement that adds a targeted macOS launchd crash-loop/debug policy, with tests in `src/daemon/launchd.test.ts`, audit handling in `src/daemon/service-audit.ts`, and docs updates for the macOS Gateway lifecycle while preserving explicit restart reliability.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: install the current LaunchAgent, trigger a startup failure, and launchd sees `KeepAlive=true` with `ThrottleInterval=1`, which current tests intentionally preserve.

## Solution Assessment

No current-main solution exists yet. The open PR is the right review venue, but its `SuccessfulExit=false` approach should be checked against non-zero startup crashes because the issue is about repeated crash relaunches, not only clean exits.
