## Summary

Keep open. Current main and the latest v2026.4.29 release still generate the macOS Gateway LaunchAgent with unconditional KeepAlive=true and a one-second throttle, while tests, audit logic, and docs still describe that policy. The linked closing PR is open, so this issue should stay open until that path is reviewed, replaced, or merged.

## What This Changes

Request to prevent macOS launchd-supervised Gateway startup failures from entering immediate restart loops so users can inspect and debug the failure.

## Best Possible Solution

Keep this issue open and land a focused macOS launchd early-failure strategy, either by completing PR #73849 or replacing it with a coordinated plist, audit, restart-loop, tests, docs, and changelog change that prevents startup crash loops without breaking intentional supervisor relaunches.
