## Summary

Keep this issue open. Current main still preserves the reported macOS launchd crash-loop behavior: the generated Gateway LaunchAgent uses `KeepAlive=true` with a one-second throttle, docs describe crash relaunch as intended launchd behavior, and the existing manual stop path disables relaunch only for explicit operator stops. The related launchd reports in the supplied context cover narrower lock-contention or config-restart races, not the broader debugging problem for arbitrary startup failures.

## Best Possible Solution

Keep the issue open and design a targeted macOS launchd crash-loop/debug path. The likely fix belongs in the LaunchAgent/runtime supervision path, with tests in `src/daemon/launchd.test.ts`, doctor/service-audit guidance for repeated early failures, and docs updates for the macOS Gateway lifecycle. Preserve fast explicit restarts while adding configurable or automatic backoff/stop-after-threshold behavior for repeated startup crashes.
