## Summary

Keep this PR open. Current main has part of the local assistant override behavior in identity/bootstrap state, but the chat avatar refresh/render path still lets a fetched `chatAvatarUrl` blob take precedence over a browser-local assistant data-image override. The PR is a focused, recent implementation candidate split from closed unmerged #71858, not obsolete cleanup.

## Best Possible Solution

Review and land this focused UI fix after validating the targeted avatar tests, or ask for any remaining maintainer-requested adjustment. Once merged, the shipped implementation can close the underlying avatar override race; until then the PR still contains useful unmerged behavior.
