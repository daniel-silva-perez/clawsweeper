## Summary

Keep this issue open. Current main still ignores `OPENCLAW_WORKSPACE_DIR` in the default agent workspace resolver, then passes that fallback through run workspace resolution and into the system prompt. The related fix PR #66831 was closed unmerged, so there is no shipped implementation to close against.

## Best Possible Solution

Keep this issue open and reimplement the closed PR #66831 behavior on current main: make `resolveDefaultAgentWorkspaceDir` prefer a non-empty `OPENCLAW_WORKSPACE_DIR` before `OPENCLAW_PROFILE`/home fallback logic, preserve explicit per-agent and `agents.defaults.workspace` precedence, and add regression coverage for the resolver, default-agent fallback, run workspace fallback, and system-prompt path display.
