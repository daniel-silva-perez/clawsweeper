## Summary

Current `main` still preserves the reported CLI allowlist misclassification path, and the issue has explicit maintainer signal making it the canonical tracker for command names being confused with plugin IDs under restrictive `plugins.allow`. This should stay open and is a good narrow ClawSweeper repair candidate.

## What This Changes

The issue requests that built-in and synthetic CLI command names such as `help`, `tool`, `run`, and `subagents` stop being misreported as plugin IDs when `plugins.allow` is restrictive.

## Best Possible Solution

Implement the maintainer-suggested command-classification layer before the plugin allowlist fallback: framework/core help and known built-in command surfaces bypass plugin allowlist guidance, manifest command aliases point to the owning plugin ID, runtime slash aliases explain chat-only or related CLI usage, and unknown/stale non-plugin names fall through to normal Commander unknown-command handling.

## Repair Work Prompt

Implement a narrow CLI policy fix for openclaw/openclaw#64732. Current `main` still routes unresolved primary command tokens through `resolveMissingPluginCommandMessageFromPolicy`, which can tell users to add non-plugin names such as `help`, `tool`, `run`, or `subagents` to `plugins.allow`. Add an explicit command-classification layer before the plugin allowlist fallback: framework/core help and known built-in/core/subcli surfaces must not produce plugin allowlist guidance; manifest command aliases must continue to point to the owning plugin ID; runtime slash aliases must explain chat-only or related CLI usage; unknown/stale non-plugin tokens should fall through to normal Commander unknown-command handling instead of suggesting synthetic plugin IDs. Preserve the plugin-ID-only config contract; do not change `plugins.allow` to accept command names. Treat #63907, #65997, and #67901 as duplicates covered by the same fix and keep #65319 as adjacent unless the same manifest-alias path is directly fixed. Add focused regression coverage for restrictive allowlists covering `openclaw help`, non-plugin `tool`, and representative built-in/synthetic roots such as `run` or `subagents`, while preserving existing behavior for real plugin IDs like `browser`, manifest CLI aliases like `wiki`, and runtime slash aliases like `dreaming`. Add a user-facing changelog Fixes entry with appropriate reporter attribution unless maintainers decide the release note should be grouped with a broader CLI allowlist fix.
