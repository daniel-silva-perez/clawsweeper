## Summary

Keep open. This is an active stacked docs PR, and against current main it would publish stable plugin docs for SDK methods that are not exposed yet; the PR body also says the page should land only after the open implementation follow-ups. The author has addressed several inline docs comments on the latest head, but the stack-order/API availability blocker remains.

## What This Changes

This PR adds a host-hook examples and recipes page, links it into the plugin docs navigation, and adds zh-CN glossary labels for the new page.

## Best Possible Solution

Keep the docs split, but merge it only after the dependent workflow SDK follow-ups have landed and the examples match the generated public SDK contracts; until then, keep future API material out of stable navigation or mark it as proposed design material.

## Reproduction Assessment

Not applicable as a runtime reproduction. The high-confidence static check is to compare the documented methods in `docs/plugins/host-hooks-examples.md` against `OpenClawPluginApi` and `buildPluginApi` on current main.

## Solution Assessment

No, not as a standalone merge against current main. Splitting the large recipe page is maintainable, but stable docs should not publish examples for unmerged SDK methods.
