## Summary

Keep open. This is an active stacked docs PR, and current main still lacks the workflow SDK methods that the new stable docs page publishes; the PR body and related PR status both say it should land only after #73384 and #74483.

## What This Changes

Adds a public host-hook examples and recipes documentation page, links it into plugin docs navigation, and adds zh-CN glossary labels for the new page.

## Best Possible Solution

Keep the docs split, but merge it only after the workflow SDK follow-ups land and the examples match the generated/public SDK contracts; until then, keep future API material out of stable navigation or label it as proposed design material.

## Reproduction Assessment

Yes for the docs-contract mismatch. Compare the PR head page at docs/plugins/host-hooks-examples.md:89 with current main's OpenClawPluginApi and buildPluginApi surfaces; the documented workflow methods are absent.

## Solution Assessment

No, not as a standalone merge against current main. Splitting the recipe page is maintainable, but stable docs should not publish examples for unmerged SDK methods; the safer path is stack-order review or removing/marking future API references until the implementation lands.
