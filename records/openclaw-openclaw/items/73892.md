## Summary

Keep this PR open. Current main still drops the `process` tool AbortSignal and the `poll` loop waits until process exit or the capped deadline; the PR is a focused implementation candidate for the open abort-signal gap discussed in #50996 and #65223, and it follows the smaller follow-up path invited when #72625 was closed.

## Best Possible Solution

Keep this PR open as a focused implementation candidate. The likely maintainer path is to make `process.poll` observe the run AbortSignal, return promptly with the existing running/partial-output semantics without killing the background child, add the regression test, and leave the broader framework-level tool-abort design in #65223.
