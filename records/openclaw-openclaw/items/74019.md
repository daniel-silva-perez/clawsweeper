## Summary

Current main still matches the reported bug path: `llm-task` starts an embedded JSON/model run with `disableTools: true` while passing through caller config, and the embedded runner still sends config-derived allowlist sources into the empty-tool guard. There is also an open linked implementation PR, #74139, that targets this issue with GitHub closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land PR #74139, or an equivalent narrow fix, so intentionally disabled embedded model runs ignore config-derived allowlist sources for the empty-tool guard while preserving fail-closed behavior for explicit runtime tool requests. Add focused regression coverage for the `llm-task`/Lobster path and a changelog fix entry crediting @technicallysecure.
