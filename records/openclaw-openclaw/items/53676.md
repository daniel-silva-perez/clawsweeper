## Summary

Keep open for maintainer review. Current main only covers the diagnostics part through the merged Google Chat auth logging work; it still requires email/email_verified matching in the app-url verifier and still lacks this PR's event.user.email fallback when message.sender exists without an email. The auth relaxation also touches the webhook trust boundary and is not proven by the PR's mocked tests against the real google-auth-library issuer behavior.

## What This Changes

The PR changes Google Chat webhook auth to accept iss-based app-url/add-on JWT payloads without email claims, adds auth rejection diagnostics/tests, and falls back sender email extraction to event.user.email.

## Best Possible Solution

Land a rebased or replacement Google Chat change only after confirming the real Chat app-url/add-on token shapes and verifier API. It should preserve current account-specific diagnostics, keep principal binding strict, and add focused tests for any accepted issuer/email claim shape plus the sender email fallback.
