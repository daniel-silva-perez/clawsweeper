## Summary

Keep open. Current main partially supersedes the diagnostics portion through merged Google Chat auth rejection logging, but this PR is not implemented on main and its issuer-only auth path is not proven against google-auth-library's real issuer gate. The branch also has a concrete multi-target logging regression, so it needs maintainer review and rebase rather than cleanup closure.

## What This Changes

The PR changes Google Chat webhook auth to accept issuer/email variants when email claims may be absent, adds auth rejection diagnostics and tests, and falls back sender email extraction to `event.user.email`.

## Best Possible Solution

A rebased or replacement Google Chat fix should preserve current per-target diagnostics, validate real Google Chat token claim shapes against google-auth-library behavior, keep add-on `sub`/appPrincipal binding strict, and add focused tests for any accepted claim variants plus the sender-email fallback.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Verify issuer-only tokens through the real issuer gate:** `extensions/googlechat/src/auth.ts:128-132`
  - body: The new issuer-only acceptance checks run after `verifyIdToken`, but google-auth-library 10.6.2 rejects `payload.iss` values outside `accounts.google.com`, `https://accounts.google.com`, or the configured universe domain before returning a ticket. Tokens with `iss: chat@system.gserviceaccount.com` or the add-on service account never reach this branch, so the mocked tests do not prove the intended production fix.
  - confidence: 0.88
- **[P2] Log failures through the checked target:** `extensions/googlechat/src/monitor-webhook.ts:108`
  - body: `logAuthFailure` always selects `targets[0]` as the logger even though the collected rejection reasons may belong to later accounts on the same webhook path. That routes diagnostics through the wrong account runtime sink in multi-account deployments and weakens the per-account troubleshooting this change is meant to add.
  - confidence: 0.83
