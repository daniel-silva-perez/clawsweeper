## Summary

Keep open for maintainer review. Current main supersedes the PR's diagnostics through merged Google Chat auth rejection logging, but it does not include the issuer-only auth matching or the sender-email fallback. The auth relaxation touches a webhook trust boundary, and the PR's mocked issuer-only tests are not enough to prove compatibility with the real google-auth-library verifier or Google Chat token contract.

## What This Changes

The PR changes Google Chat app-url/add-on webhook verification to accept issuer/email variants without email claims, adds auth rejection diagnostics/tests, and falls back sender email extraction to event.user.email.

## Best Possible Solution

A rebased or replacement Google Chat fix should preserve current account-specific diagnostics, confirm the real app-url/add-on token shapes, keep add-on sub principal binding strict, and add focused tests for any accepted issuer/email claim shape plus the sender-email fallback.
