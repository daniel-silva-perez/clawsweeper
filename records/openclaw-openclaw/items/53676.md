## Summary

Keep open. Current main includes the later Google Chat webhook auth rejection logging and appPrincipal warnings, but it still does not implement this PR's issuer-claim matching for app-url tokens without email, add-on matching via iss, or sender email fallback. The remaining auth relaxation changes the webhook trust boundary and needs maintainer/security review rather than cleanup closure.

## What This Changes

The PR changes Google Chat auth verification to accept iss-based app-url and add-on JWTs when email is absent, adds auth rejection diagnostics/tests, and falls back sender email extraction to event.user.email.

## Best Possible Solution

Keep this PR open for maintainer review, but do not merge it as-is. The best path is to rebase or replace it with a narrow Google Chat auth/sender fallback change that preserves current main diagnostics, verifies the real google-auth-library issuer contract for app-url/add-on tokens, and adds focused tests for the accepted token shape plus event.user.email fallback. If maintainers decide the shipped numeric appPrincipal behavior fully resolves the auth reports, close the auth portion and ask for a separate small sender fallback PR if still wanted.
