## Summary

Keep open. Current main already includes the Google Chat webhook auth rejection diagnostics and appPrincipal warnings from later work, but it does not include this PR's issuer-claim matching for app-url tokens without email, add-on issuer matching via iss, or the sender email fallback. The remaining auth change touches the webhook trust boundary and the PR tests mock the verifier payload rather than proving google-auth-library will accept those issuers, so this needs maintainer/security review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review, but do not merge it as-is. The best path is to rebase or replace it with a narrow Google Chat auth/sender fallback change that preserves the current main per-account diagnostics from #71145, proves the real google-auth-library issuer contract for app-url/add-on tokens, and adds focused tests for the accepted token shape plus the event.user.email fallback. If maintainers decide the already-shipped numeric appPrincipal behavior fully resolves the auth reports, close this PR with that explanation and ask for a separate small sender fallback PR if still wanted.
