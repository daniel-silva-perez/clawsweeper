## Summary

Keep this issue open. Current main still lacks a Telegram config-level default outbound topic/thread binding per agent: topic config remains inbound-routing oriented, schema validation is strict, and outbound thread resolution only uses explicit thread targets or current Telegram context auto-threading.

## Best Possible Solution

Keep this issue open. If maintainers want the feature, implement it in the bundled Telegram plugin/config surface with typed topic-level outbound-default metadata, zod schema and generated config metadata/help/docs updates, and focused tests. Runtime precedence should be explicit threadId or :topic: target first, current conversation/tool context next, then configured per-agent outbound default, with deterministic ambiguity handling for multiple defaults.
