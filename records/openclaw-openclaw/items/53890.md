## Summary

Keep this issue open. Current main still has inbound Telegram topic-to-agent routing and explicit outbound topic targeting, but it does not provide a config-level per-agent default outbound topic/thread binding. The existing channel/account `defaultTo` and per-agent heartbeat `to` settings can target a Telegram `:topic:NNN`, but they are not the requested per-agent default for proactive outbound sends from the Telegram topic config.

## Best Possible Solution

Keep this issue open. If maintainers want the feature, implement it in the bundled Telegram plugin/config surface with typed topic-level outbound-default metadata, zod schema and generated config metadata/help/docs updates, and focused tests for send and poll actions. The runtime should preserve explicit `threadId` and `:topic:` targets, preserve current-context auto-threading, then fall back to a configured per-agent outbound default with deterministic ambiguity handling.
