## Summary

Keep this PR open. Current main still does not strip fenced code blocks in the shared `stripMarkdown` helper, and BlueBubbles sends the helper's output directly, so the reported plain-text channel regression is still plausible. The PR is focused and not security-sensitive, but its regex needs the inline triple-backtick review concern resolved before merge or before folding the work into #56283.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to fix the inline triple-backtick span edge case, keep the fenced-block regression coverage, and then either land this focused `stripMarkdown` fix or deliberately fold the same behavior and tests into #56283.
