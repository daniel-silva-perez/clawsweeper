## Summary

Keep open. Current main at f2405c830bd925bd24f91b6bac5d417c6d36e5ba still does not implement per-job activeUntil/expiresAt/onExpire semantics for recurring cron jobs. The cron types, protocol schema, scheduler lifecycle, CLI add/edit surfaces, agent cron tool, Control UI, docs, and generated Swift protocol model still only expose one-shot cleanup through deleteAfterRun, while recurring every/cron jobs continue by recomputing nextRunAtMs. Related #71712 is open and explicitly depends on recurring expiration semantics, so this needs maintainer API/product design rather than cleanup closure.

## Best Possible Solution

Keep this open and decide the additive public cron lifecycle contract first: canonical field name, timestamp parsing/timezone behavior, onExpire default and allowed values, audit/run-log event shape, and interaction with #71712. Then implement it consistently across cron types/schema/normalization, scheduler expiration handling, store persistence, CLI add/edit/list/show, agent cron tool, Control UI/macOS protocol models, tests, and public docs.
