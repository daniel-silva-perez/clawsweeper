Keep open. This is a fresh Windows regression report against v2026.4.26 that explicitly says the earlier c85065e restart-probe fix is present, yet the attached logs still show slow startup and loopback WebSocket policy closes/handshake timeouts. Current main does contain the c850 policy-close allowlist and tests, so the remaining report should not be closed as already implemented without a Windows repro of the newer failure mode.

## Best Possible Solution

Keep this issue open and reproduce on Windows 11 with v2026.4.26 and current main, collecting `openclaw gateway restart --json`, `openclaw gateway status --deep`, `gateway-restart.log`, and a startup trace. If confirmed, fix the Windows restart health/schtasks/startup timing path in `src/cli/daemon-cli/*` or `src/daemon/schtasks.ts`; if the third-party plugin is the cause, document the plugin-side manifest/runtime issue separately rather than closing this as the c850 policy-close fix.
