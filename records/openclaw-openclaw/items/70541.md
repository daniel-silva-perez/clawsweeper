## Summary

Keep this issue open. Current main still has the inconsistent post-compaction caller: the auto-compaction system-event hook reads AGENTS.md from process.cwd(), while the adjacent preflight compaction refresh path uses the queued run's workspaceDir. The report is narrow, reproducible from source, and suitable for one focused fix PR with a regression test.

## Best Possible Solution

Fix the auto-compaction system-event hook in src/auto-reply/reply/agent-runner.ts to call readPostCompactionContext with followupRun.run.workspaceDir, preserving the existing agentId and enqueueSystemEvent behavior. Add a focused regression test where process.cwd() differs from the queued run workspace and the workspace AGENTS.md content is the one injected.
