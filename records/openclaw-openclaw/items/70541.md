## Summary

Keep this issue open. Current main still has the reported wrong-root post-compaction caller: the auto-compaction system-event hook reads post-compaction context from `process.cwd()` while the adjacent preflight compaction path uses the queued run workspace, so the reported workspace mismatch remains valid and narrow enough for a focused fix PR.

## What This Changes

The issue asks for post-compaction context injection after auto-compaction to read `AGENTS.md` from the queued run's configured workspace instead of the runner process cwd.

## Best Possible Solution

Use `followupRun.run.workspaceDir` for the auto-compaction system-event hook in `src/auto-reply/reply/agent-runner.ts`, preserve the existing trusted enqueue and agent-id handling, and add a regression test proving the hook reads the queued workspace when cwd differs.

## Reproduction Assessment

Yes. A source-level reproduction is clear: run the auto-compaction completion path with a `sessionKey`, `autoCompactionCount > 0`, and `process.cwd()` different from `followupRun.run.workspaceDir`; current main will read the cwd `AGENTS.md` because that value is passed into `readPostCompactionContext`.

## Solution Assessment

Yes. The narrowest maintainable fix is to pass the already-required `followupRun.run.workspaceDir` to `readPostCompactionContext`; no new config, API, or fallback behavior is needed for the normal queued-run path.
