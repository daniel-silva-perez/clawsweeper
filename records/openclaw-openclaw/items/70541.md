## Summary

Keep this issue open. Current main still has the reported wrong-root post-compaction caller, and the surrounding source confirms the correct workspace root is already available on the queued run.

## What This Changes

Post-compaction context injection should read `AGENTS.md` from the queued run `workspaceDir` instead of the runner process cwd after auto-compaction.

## Best Possible Solution

Keep the issue open until `runReplyAgent` uses the queued run workspace for post-compaction context refresh and has a regression test that fails when cwd and `workspaceDir` differ.

## Reproduction Assessment

Yes. A source-level reproduction is clear: run the `runReplyAgent` auto-compaction completion path with `sessionKey`, `autoCompactionCount > 0`, and cwd different from `followupRun.run.workspaceDir`; current main passes cwd to `readPostCompactionContext`.

## Solution Assessment

Yes. The narrowest maintainable fix is to pass `followupRun.run.workspaceDir` to `readPostCompactionContext` while preserving the existing `cfg`, agent-id resolution, and trusted enqueue behavior.
