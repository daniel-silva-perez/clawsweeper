## Summary

Keep this issue open. Current main still has the reported mismatch: the auto-compaction system-event hook reads post-compaction context from `process.cwd()`, while the adjacent preflight compaction path reads from the queued run's required `workspaceDir`. The report is source-confirmed, still present in the latest release tag, and narrow enough for one focused regression-fix PR.

## Best Possible Solution

Change the auto-compaction system-event hook in `src/auto-reply/reply/agent-runner.ts` to call `readPostCompactionContext(followupRun.run.workspaceDir, ...)`, preserving the existing agent-id resolution, enqueue behavior, and best-effort error handling. Add a focused regression test where `process.cwd()` differs from the queued run workspace and the injected system event comes from the workspace `AGENTS.md`.
