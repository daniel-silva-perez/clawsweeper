## Summary

Keep this PR open. Current main still accepts the reported `gateway.auth.mode="none"` plus `gateway.tailscale.mode="serve"` configuration, and the same-author linked security bug #50630 remains open. The PR is still a focused gateway-auth hardening candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open as the implementation candidate for #50630. Rebase it onto current main, add a targeted runtime rejection for Serve plus explicit `auth.mode="none"`, add or adjust tests so that combination is rejected, and cover install/doctor/service-repair validation or normalization so the invalid config is caught before writing a managed service. Preserve allowed coverage for local loopback `none`, Serve with real auth or verified Tailscale identity as intended, and Funnel's password-only rule.
