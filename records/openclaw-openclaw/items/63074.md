## Summary

Keep this PR open. Current main still short-circuits Windows drive-letter bind sources as `non_absolute` before checking blocked credential roots, so the PR’s intended security classification fix is not implemented. The PR also still deserves maintainer review because its latest diff includes unrelated Mattermost policy and npm postpublish verifier changes alongside the sandbox security change.

## Best Possible Solution

Keep the PR open, but ask for a focused sandbox-only patch or a clearly split follow-up. The best implementation should reorder `getBlockedBindReason` so dangerous Windows credential sources are checked before the non-absolute fallback, include raw Windows home roots such as `USERPROFILE` where needed, add regression coverage for Windows credential binds and the audit severity path, and coordinate with the broader Windows bind-mount work in #42174 rather than closing this PR as obsolete.
