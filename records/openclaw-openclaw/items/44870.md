## Summary

Keep open. Current main still has the reported custom-provider onboarding gap: OpenClaw can run manually configured OpenAI Responses providers, but custom onboarding only exposes OpenAI Chat Completions or Anthropic Messages compatibility and verifies ordinary OpenAI-compatible endpoints through `/chat/completions`.

## What This Changes

The issue asks OpenClaw custom-provider onboarding to verify and configure an OpenAI-compatible endpoint whose model catalog is reachable but whose onboarding verification returns HTTP 503.

## Best Possible Solution

Add first-class custom-provider onboarding support for OpenAI Responses endpoints: interactive choice or detection, non-interactive flag parsing, verification against `/responses`, persistence as `models.providers.<id>.api: "openai-responses"`, and matching docs/tests while preserving the existing Chat Completions default.
