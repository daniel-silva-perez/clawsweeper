## Summary

Keep open for maintainer review: current main already shipped the SearXNG part, but Tavily generic freshness is still absent, and the PR branch is mixed-scope with blocking correctness and security-audit regressions plus unrelated provider-normalization changes.

## What This Changes

The PR branch adds a bundled SearXNG web-search plugin, adds Tavily generic `web_search` freshness support, and changes docs, config schemas, onboarding, generated plugin metadata, security audit heuristics, tests, and Pi embedded provider normalization.

## Best Possible Solution

Keep the shipped SearXNG implementation from #57317. If Tavily freshness is still desired, land it as a narrow Tavily-plugin change against current main: accept OpenClaw's current freshness values, map them to Tavily request behavior, include the value in cache keys, update focused Tavily/web-search tests and docs, and avoid adding legacy config paths or unrelated model-provider normalization changes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Keep custom OpenRouter endpoints on their configured API:** `src/agents/pi-embedded-runner/model.provider-normalization.ts:25-38`
  - body: The new OpenRouter branch rewrites every `openrouter` model using `openai-completions` to `openai-responses`, including custom or proxied `baseUrl` deployments. Those proxies may only implement Chat Completions, so this unrelated change can break working configurations that explicitly chose that API surface.
  - confidence: 0.86
- **[P2] Map the current freshness values for Tavily:** `extensions/tavily/src/tavily-search-provider.ts:33-47`
  - body: The Tavily mapper handles only `pd`/`pw`/`pm`/`py`, but current `web_search` docs and normalization also use `day`/`week`/`month`/`year`. A caller passing `freshness: "week"` would be accepted by the tool surface but silently send no Tavily recency filter.
  - confidence: 0.8
- **[P2] Do not accept Tavily keys on an unread config path:** `src/config/zod-schema.agent-runtime.ts:319-325`
  - body: Adding `tools.web.search.tavily.apiKey` to the strict runtime schema makes that config validate, but Tavily credential resolution still reads only `plugins.entries.tavily.config.webSearch.apiKey` or `TAVILY_API_KEY`. Users can follow the accepted config and still get missing-key errors at runtime.
  - confidence: 0.82
- **[P2] Count keyless SearXNG search in exposure audits:** `src/security/audit-extra.sync.ts:335-339`
  - body: SearXNG is intentionally usable with only a base URL, but this audit heuristic treats web search as configured only when an API key or key env var exists. Active keyless SearXNG search can therefore be underreported in small-model exposure auditing.
  - confidence: 0.78
- **[P2] Avoid defaulting SearXNG to blocked localhost:** `extensions/searxng/src/config.ts:5`
  - body: The provider defaults to `http://localhost:8080` while private-network access defaults to false and is enforced by the network guard. A default or onboarding-style SearXNG selection can therefore point at localhost and reject its own request until a separate flag is set.
  - confidence: 0.74
