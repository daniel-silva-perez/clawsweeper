## Summary

Keep open: current main already shipped the SearXNG provider portion, but it still lacks the PR’s Tavily generic `web_search` freshness support. The branch also mixes that still-useful Tavily change with already-shipped SearXNG work and unrelated Pi embedded runner/provider normalization changes, so it needs maintainer direction rather than cleanup closure.

## What This Changes

The PR branch adds a bundled SearXNG web-search plugin with docs/config/onboarding wiring, adds Tavily generic `web_search` freshness-to-days support, updates generated/config/audit surfaces, and changes Pi embedded runner/provider normalization code.

## Best Possible Solution

Keep the shipped SearXNG implementation on main, then land a narrow Tavily-only change in the Tavily plugin that adds generic `web_search` freshness support, maps supported recency values to Tavily `days`, includes that value in the request body and cache key, and updates focused tests/docs without touching unrelated provider runtime code.
