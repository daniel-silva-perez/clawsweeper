## Summary

Keep open for maintainer review: current main already shipped the SearXNG half of the proposal, but the Tavily generic `web_search` freshness support is still missing. The PR is not a clean merge candidate because it also carries superseded SearXNG code and unrelated provider-runtime changes in a security-sensitive model execution path.

## What This Changes

The PR branch adds a bundled SearXNG web-search plugin with docs/config/onboarding wiring, adds Tavily generic `web_search` freshness-to-days support, updates generated config/audit surfaces, and changes Pi embedded runner/provider normalization code.

## Best Possible Solution

Keep the shipped SearXNG implementation from #57317, and land Tavily freshness as a narrow Tavily-plugin change that adds a generic `web_search` freshness field, maps supported recency values to Tavily `days`, includes the value in request/cache behavior, and updates only focused Tavily/web-search tests and docs.
