## Summary

Keep open: current main covers the SearXNG half, but the Tavily generic `web_search` freshness behavior requested by this PR is still absent, so current main does not fully implement the PR’s intended change. The branch also is not a clean merge vehicle because it combines superseded SearXNG work with unrelated model-runtime and security-sensitive surface changes.

## What This Changes

The PR branch adds a bundled SearXNG web-search plugin with docs/config/onboarding wiring, adds Tavily `web_search` freshness-to-days support, updates generated/config/audit surfaces, and also changes Pi embedded runner/provider normalization code.

## Best Possible Solution

Keep the shipped SearXNG implementation on main, close or supersede the SearXNG portion of this branch, and land a narrow Tavily-only change that adds generic `web_search` freshness support in the Tavily plugin with mapping to Tavily `days`, cache-key/body coverage, focused tests, and docs.
