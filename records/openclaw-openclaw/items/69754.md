## Summary

Keep open. Current main supports normal Feishu interactive card payloads, but Feishu document/comment targets still degrade approval-style interactive payloads to plain text without explicit surface guidance, and the current text fallback can reduce command/value buttons to labels. The related Feishu card support work fixed a separate normal-card path, not this fallback-copy problem.

## Best Possible Solution

Land a narrow Feishu fallback UX fix: when approval-style interactive buttons degrade to text on Feishu document/comment surfaces, preserve exact `/approve ...` command values and add explicit copy explaining that interactive buttons are unavailable in that Feishu surface, whether same-chat text approval is valid, and when Web UI or terminal UI is the fallback. Update focused Feishu/approval docs and regression tests without changing approval authorization semantics.
