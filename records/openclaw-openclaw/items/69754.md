## Summary

Current main partially improves the situation: normal Feishu outbound payloads now render shared interactive buttons as native Feishu cards, and #58298 was closed after that related card-path work landed. This issue is still valid because current main also has an explicit Feishu degraded-text path for document/comment targets that strips interactive metadata and sends plain fallback text without explaining why buttons are unavailable or where to complete approval; the Feishu docs also do not describe approval fallback behavior.

## Best Possible Solution

Keep this issue open and land a narrow Feishu approval fallback UX fix: when approval buttons are degraded to text on Feishu, especially document/comment targets, preserve the exact `/approve ...` action path and add explicit copy that interactive buttons are unavailable for the current Feishu surface/context, whether same-chat text approval is still valid, and when the operator should use Web UI or terminal UI instead. Update focused Feishu/approval docs and tests without changing approval authorization semantics.
