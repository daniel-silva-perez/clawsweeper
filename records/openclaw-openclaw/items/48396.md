## Summary

Keep open. Current main still does not implement last-message previews in the Control UI Sessions list, and this PR is not safe to close as obsolete. The PR is paired with the same-author feature request #47120 and overlaps open PR #47243, but neither implementation has landed; the submitted patch also renders `subject` metadata instead of the backend `lastMessagePreview` field and adds an unrelated scratch document.

## Best Possible Solution

Keep the PR open only if it is revised into an end-to-end Control UI implementation: remove `SESSION_LIST_IMPROVEMENT.md`, add `lastMessagePreview` to the UI row type and session event merge surface, request preview data in a bounded way, render normalized timestamp-plus-preview text from `row.lastMessagePreview`, and add focused controller/view coverage. Coordinate with #47120 and #47243 so only one corrected implementation lands.
