## Summary

Keep this PR open. Current main still does not implement Control UI last-message previews, and this PR is not obsolete or safely closeable as implemented. The backend and TUI already use the dedicated `lastMessagePreview` contract, but Control UI still does not request, type, event-merge, or render that field. The submitted PR also uses `subject` instead of transcript preview content and includes an unrelated scratch document, while the same-author feature request #47120 and overlapping PR #47243 remain open.

## Best Possible Solution

Keep this PR open for maintainer or author follow-up. The best path is to land one corrected Control UI implementation: remove the scratch document, add `lastMessagePreview` to the UI row/event surface, request previews through a bounded `sessions.list` or visible-row preview flow, render normalized timestamp-plus-preview text from `row.lastMessagePreview`, add focused controller/view coverage, and coordinate with #47120 and #47243 so only one implementation ships.
