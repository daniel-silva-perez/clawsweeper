## Summary

Keep this PR open. Current main still does not implement Control UI last-message previews, and the submitted PR is not ready to close as obsolete or implemented: it renders `subject` metadata instead of the backend `lastMessagePreview` field and includes an unrelated scratch Markdown file. The same-author feature request #47120 is still open, and overlapping PR #47243 is also open, so maintainers should coordinate one corrected implementation rather than auto-close this PR.

## Best Possible Solution

Keep this PR open for maintainer or author follow-up. The best path is to land one corrected Control UI implementation: remove the scratch document, add `lastMessagePreview` to the UI row type and session event merge surface if needed, request previews through a bounded `sessions.list` or visible-row preview flow, render normalized timestamp-plus-preview text from `row.lastMessagePreview`, add focused controller/view coverage, and coordinate with #47120 and #47243 so only one implementation ships.
