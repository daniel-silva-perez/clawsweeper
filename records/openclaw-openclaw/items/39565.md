## Summary

Keep this PR open. Current main still has the pre-PR formatter behavior: copy-sensitive checks mostly run against the raw token, `*` is not stripped as edge punctuation, dotted IDs without underscores are still chunked, and the regression tests added by this PR are not present. The PR remains a focused implementation candidate for open bug #39505, and its later commits address the review-raised guardrails for punctuation-only and prose-like hyphenated tokens.

## Best Possible Solution

Keep this PR open for maintainer review, or land a rebased/cherry-picked equivalent. The concrete fix belongs in `src/tui/tui-formatters.ts`: classify the punctuation-stripped candidate, include `*` in edge punctuation, preserve dotted IDs and digit-bearing hyphenated IDs, keep punctuation-only and prose-like long tokens chunkable, and retain the focused regression tests in `src/tui/tui-formatters.test.ts`. Consider dropping the unrelated `src/cli/daemon-cli/lifecycle.test.ts` formatting-only hunk before landing.
