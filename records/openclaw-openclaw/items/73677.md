## Summary

Keep this open. Current main still does not preserve Feishu/Lark custom sticker labels on the inbound agent text path: text messages return only the parsed text field, standalone sticker messages become a generic media placeholder, and the existing emotion renderer is limited to rich-text post content. The adjacent Feishu reaction work is related channel context but does not cover this inbound sticker-label gap.

## What This Changes

Request Feishu/Lark inbound custom stickers such as [OK], [完成], and [调皮] be preserved for agents as Unicode emoji or text labels instead of being dropped.

## Best Possible Solution

Fix this in the bundled Feishu plugin by preserving a stable sticker label or text fallback from inbound payloads before considering Unicode mappings. Add focused parser and inbound tests for plain text with sticker metadata, standalone sticker messages, rich-text emotion elements, Chinese bracket labels, and fallback behavior when no label is available.

## Reproduction Assessment

Unclear for the exact live Feishu mixed-message reproduction because the report lacks raw event JSON and there were no local Feishu credentials for live replay. Source-level reproduction is clear: current main returns only parsed.text for text messages and renders standalone sticker messages as generic media placeholders.

## Solution Assessment

Yes, a Feishu plugin parser fix is the right direction; this should not require a core channel API change. The safer path is to preserve labels or text fallbacks first, then add Unicode conversion only for verified mappings.
