## Summary

Keep this open. Current main still does not preserve Feishu/Lark custom sticker labels in the agent-visible inbound body; text messages use only the Feishu `text` field and standalone sticker messages are reduced to a generic media placeholder. The related Feishu emoji/reaction reports cover outbound reaction normalization, not inbound custom sticker content.

## What This Changes

Request that the Feishu/Lark channel preserve inbound custom sticker context by converting bracket-style stickers such as `[OK]`, `[完成]`, and `[调皮]` to Unicode emoji or agent-visible text.

## Best Possible Solution

Implement this in the Feishu plugin inbound parser by preserving verified sticker labels or stable text fallbacks from raw event payloads. Unicode conversion should be limited to confirmed sticker names; unknown labels should remain visible as bracketed text rather than disappearing.

## Reproduction Assessment

Unclear for a high-confidence live reproduction because the issue does not include raw Feishu webhook JSON. Source-level verification is clear: current main uses only `parsed.text` for text messages and `<media:sticker>` for standalone sticker messages.

## Solution Assessment

Yes, the right maintainable direction is a Feishu plugin parser fix, not a core channel API change. The safer alternative to a broad Unicode table is first preserving Feishu-provided text labels, then adding verified mappings only where the payload contract supports them.
