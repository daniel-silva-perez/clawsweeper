## Summary

Keep this open. Current main still does not preserve Feishu/Lark custom sticker labels on the inbound agent path: text messages return only `parsed.text`, standalone `sticker` messages become a generic media placeholder, and the existing emotion renderer only applies to rich-text `post` payloads. The issue is Feishu-plugin-owned and not covered by the adjacent Feishu reaction/custom-emoji reports.

## What This Changes

Preserve Feishu/Lark custom sticker context in inbound agent messages by converting verified sticker labels to Unicode emoji or stable text fallbacks.

## Best Possible Solution

Fix this in `extensions/feishu` by preserving sticker labels or stable text fallbacks from inbound payloads, using Unicode mappings only for verified sticker names. Add focused parser/inbound tests for mixed text plus sticker metadata, standalone `sticker` messages, rich-text `emotion` elements, Chinese bracket labels, and unknown-label fallback behavior.

## Reproduction Assessment

Unclear for the exact live Feishu mixed text/sticker event because the report does not include raw event JSON. Source-level reproduction is clear: current `main` returns only `parsed.text` for text messages and `<media:sticker>` for standalone sticker messages.

## Solution Assessment

Yes, a Feishu plugin parser fix is the right direction and should not require a core channel API change. The safest path is label/text preservation first, then verified Unicode mapping where Feishu payload fields make that reliable.
