## Summary

Keep this issue open. Current main still does not preserve Feishu custom bracket-style sticker labels on the inbound agent path: text messages are reduced to `parsed.text`, standalone stickers become a generic media placeholder, and the only emotion renderer is limited to rich-text `post` elements. The prior ClawSweeper comment and the provided related items support the same conclusion: this is not covered by the Feishu reaction API work or the Telegram custom emoji issue.

## Best Possible Solution

Keep this open and fix it in the bundled Feishu plugin. Capture the actual mixed text/custom-sticker payload, then extend `extensions/feishu/src/bot-content.ts` and focused Feishu tests to preserve a bracket/text representation or map known sticker labels to Unicode only where the mapping is safe. Cover plain text with sticker metadata, standalone `sticker` messages, rich-text `emotion` elements, and Chinese bracket labels.
