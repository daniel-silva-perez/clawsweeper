## Summary

Keep this issue open. Current main still does not preserve Feishu/Lark custom sticker labels on the inbound agent path: text messages return only `parsed.text`, standalone `sticker` messages become a generic media placeholder, and the existing `emotion` renderer only applies to rich-text `post` content. The prior ClawSweeper keep-open review remains valid; there were no relevant changes to the Feishu parsing/docs files between that reviewed SHA and current main.

## Best Possible Solution

Keep this issue open and fix it in the bundled Feishu plugin. Extend inbound Feishu parsing to preserve custom sticker labels or safe text fallbacks for mixed text/sticker payloads and standalone `sticker` messages, then add focused tests in `extensions/feishu/src/bot-content.ts` / Feishu inbound tests for bracket labels, Chinese sticker names, `post` emotion elements, and generic media fallback behavior.
