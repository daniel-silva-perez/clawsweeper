## Summary

Keep this open. Current main still does not preserve Feishu/Lark custom sticker labels in the agent-visible inbound message body, and the related Feishu emoji/reaction items cover outbound reaction normalization rather than inbound sticker content.

## What This Changes

The issue requests preserving Feishu/Lark custom stickers in inbound agent-visible text, either as Unicode emoji equivalents or as bracketed text labels.

## Best Possible Solution

Implement this in the Feishu plugin inbound parser by preserving verified sticker labels or stable text fallbacks from raw event payloads. Add focused parser and dispatch tests for mixed text plus sticker metadata, standalone sticker messages, rich-text emotion elements, Chinese bracket labels, and unknown-label fallback behavior.

## Reproduction Assessment

Unclear for the exact live Feishu event because the report does not include raw payload JSON. Source-level reproduction is clear: current main returns only `parsed.text` for text messages and `<media:sticker>` for standalone sticker messages.

## Solution Assessment

Yes, a Feishu plugin parser fix is the right maintainable direction and should not require a core channel API change. The safest path is label/text preservation first, with Unicode mapping only for verified sticker names.
