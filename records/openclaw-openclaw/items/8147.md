## Summary

Keep this issue open. Current main still hardcodes the Control UI/WebChat composer to send on plain Enter, has no persisted send-shortcut preference, and the WebChat config/docs surface only exposes chat history truncation. There is also an open unmerged implementation candidate, #73390, that explicitly targets this issue, so the right cleanup path is to review/land or close that PR rather than close the issue early.

## Best Possible Solution

Keep this issue open as the canonical tracker. Review #73390 for product fit, UI persistence choice, tests for plain Enter, Shift+Enter, IME composition, Cmd/Ctrl+Enter, and any needed docs/config updates; after it lands, close this issue as implemented and reconcile the related open duplicate #19859.
