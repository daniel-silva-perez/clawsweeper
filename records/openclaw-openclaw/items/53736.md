Keep open. ## What This Changes

The PR adds global and per-agent `tools.exec.denyPatterns` config/schema/generated artifacts, merges and forwards those patterns into exec tool defaults and chat-bash, rejects matching commands in `createExecTool`, and adds deny-pattern tests.

## Best Possible Solution

Maintain this as an open implementation candidate until a maintainer decides the product contract: either land a rebased, documented soft guardrail for steering coding CLIs toward session spawning, or fold the idea into the broader exec denylist security design tracked by #71097 with explicit matching semantics and config-boundary tests.
