## Summary

Keep open. The report remains actionable and is already paired with open PR #74060; current main has account-aware Feishu resolution and gateway health probing, but the Feishu setup/status status-line probe still uses top-level credentials, so this should wait for maintainer review of the linked fix.

## What This Changes

Make Feishu health/status output recognize credentials stored under `channels.feishu.accounts.*.appId/appSecret` when multi-account Feishu config is used.

## Best Possible Solution

Review and land PR #74060, or make an equivalent Feishu-plugin-owned change that routes setup/status probing through the selected resolved Feishu account and keeps focused regression coverage for account-scoped credentials.

## Reproduction Assessment

Yes for the narrowed setup/status defect: an account-only Feishu config can be marked configured while `resolveStatusLines` still tries to probe only top-level credentials. The exact live health table was not exercised because no real Feishu credentials were used in this read-only review.

## Solution Assessment

Yes, the linked PR direction is the narrowest maintainable solution: reuse Feishu's existing account resolver in the Feishu setup/status surface instead of adding core-specific Feishu behavior or duplicating credential lookup logic.
