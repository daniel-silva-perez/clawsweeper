## Summary

Keep this PR open. Current main at 1b25dcf57a9fca5d00cf275046e20a2e3ea3730c still does not forward config env.vars into sandbox exec environments, so the PR is not obsolete or implemented. The proposed direct forwarding still needs maintainer/security review because it changes sandbox secret exposure and does not preserve applyConfigEnvVars runtime guards for existing env precedence or unresolved placeholders.

## Best Possible Solution

Keep this PR open and revise it into a guarded sandbox env propagation patch: derive an effective config env map that mirrors applyConfigEnvVars semantics, layer it below sandbox/backend env and per-call params, add regression tests for forwarding, precedence, blocked keys, existing env precedence, and unresolved placeholders, and update public sandbox/env docs if maintainers decide config env.vars is part of the sandbox exec contract.
