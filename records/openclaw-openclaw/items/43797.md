Keep open. Current main still removes the sandbox runtime and registry entry without deleting the corresponding Docker sandbox workspace directory, and open PR #43835 explicitly targets this issue with closing syntax, so this issue should remain open until that PR lands or is superseded.

The issue asks for sandbox prune to clean up expired non-shared sandbox workspace directories as well as Docker containers.

## Best Possible Solution
