## Summary

Keep this open. Current main still removes the sandbox runtime and registry entry without deleting the corresponding non-shared Docker sandbox workspace directory, and open PR #43835 explicitly targets this issue with closing syntax, so the issue should remain open until that PR lands or is superseded.

## What This Changes

The issue requests that sandbox pruning remove expired sandbox workspace directories as well as Docker containers to prevent disk leaks and stale workspace state.

## Best Possible Solution

Land PR #43835 or an equivalent sandbox-owned fix that removes only safe non-shared Docker sandbox workspace directories after runtime removal, preserves shared-scope and `rw` agent workspaces, and adds focused regression coverage for prune and recreate behavior.
