## Summary

Keep this open. Current main still removes expired sandbox runtimes and registry entries without deleting the corresponding non-shared local sandbox workspace directory, and open PR #43835 remains an unmerged implementation candidate that explicitly targets this issue.

## What This Changes

Sandbox prune should delete expired non-shared sandbox workspace directories as well as removing sandbox runtimes and registry entries.

## Best Possible Solution

Review, rebase, and land #43835 or supersede it with an equivalent sandbox-owned fix that removes only safe non-shared workspace directories after runtime removal, reuses the registry scope key, covers manual recreate/removal if intended, and adds focused regression coverage for session, agent, and shared scope behavior.
