## Summary

Keep this issue open. Current main still prunes sandbox runtimes and registry entries without deleting the local Docker sandbox workspace directory, and #43835 remains the open implementation candidate linked with GitHub closing syntax for this issue.

## Best Possible Solution

Keep this issue open and review #43835 as the current implementation candidate. The preferred fix belongs in the sandbox owner code: derive non-shared workspace paths from the resolved `workspaceRoot` plus registry `sessionKey`, remove them with safe recursive cleanup after runtime removal, preserve shared and still-in-use workspaces, and add focused regression tests for session, agent, shared scope, and manual recreate behavior if recreate is intended to refresh local workspaces.
