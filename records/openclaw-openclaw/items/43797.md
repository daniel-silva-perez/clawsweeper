Current main still prunes sandbox runtimes and registry rows without deleting the corresponding local Docker sandbox workspace directory, and an open unmerged PR already explicitly targets this issue with closing syntax, so the issue should remain open until that PR is landed or superseded.

The issue requests that sandbox pruning clean up expired non-shared workspace directories under the sandbox workspace root in addition to removing runtimes and registry entries.
