## Summary

Keep open. Current main still lacks the requested model failover and terminal failure hooks, and the PR is a plausible implementation direction, but the current diff exposes terminal provider failure text through a new ungated plugin hook and needs maintainer/security handling before merge.

## What This Changes

The PR adds typed `model_failover` and `model_failure_terminal` plugin hooks, emits them from embedded failover and terminal before-reply failure paths, and adds focused hook plumbing tests.

## Best Possible Solution

Keep the narrow hook API if maintainers accept the product direction, but make the terminal failure payload match the existing plugin data-access contract, document the new hooks and access requirements, add the required changelog entry, and then validate the focused plugin and agent fallback tests before merge.

## Reproduction Assessment

Yes. The review can reproduce the gap and the patch issue by searching current main for the hook names, inspecting the PR diff's terminal emission, and tracing current fallback summary construction plus conversation-hook gating.

## Solution Assessment

No, not as currently written. The hook direction is maintainable, but the terminal hook should be sanitized or gated like other conversation/error-observation hooks before becoming a public plugin contract.
