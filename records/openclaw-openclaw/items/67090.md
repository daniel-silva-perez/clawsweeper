## Summary

Keep this PR open for maintainer review. Current main still implements and documents post-compaction AGENTS.md extraction as an H2/H3 contract, while this PR broadens that contract to H1-H4 without updating the public config/docs/schema surfaces and without answering the maintainer's concrete H1/H4 rationale question.

## What This Changes

The PR widens `extractSections` heading detection from H2/H3 to H1-H4 and adds regression tests for H1/H4 extraction, H1 section boundaries, and nested headings.

## Best Possible Solution

End with a maintainer-approved heading-level contract where parser behavior, regression tests, config comments, generated help/schema text, public docs, and any needed changelog entry agree. If H1 support is accepted, narrow the implementation to the intended levels and update those surfaces together; otherwise retain the documented H2/H3 behavior and close manually as not planned.

## Reproduction Assessment

Yes for the narrow parser behavior: current main's regex excludes H1, so an AGENTS.md with `# Session Startup` is not extracted by this path. The unresolved part is not code reproduction, but whether H1 and H4 should be supported by the public contract.

## Solution Assessment

No as currently proposed. Widening the code to H1-H4 alone is broader than the reported H1 case and leaves docs/config help inconsistent; the safer path is a maintainer-approved set of heading levels with all public contract surfaces updated together.
