## Summary

Keep open. Current main has the general plugin HTTP dispatcher and stale route-registry safeguards, and the related closed #45888 context shows that older generic failure was considered fixed. But this issue is a fresh, concrete Docker/workspace-plugin report against v2026.4.26, and I did not find a newer route-dispatch fix on main after that release. The right cleanup path is reproduction against current main/Docker, not auto-closing.

## Best Possible Solution

Reproduce this against current main in Docker with a minimal workspace plugin that registers `/plugins/webhooks/gmail-push`. If it reproduces, fix the Gateway startup/runtime wiring so workspace `api.registerHttpRoute(...)` registrations land in the same HTTP route registry used by the dispatcher, including any deferred plugin reload path, and add an integration regression that checks both `plugins inspect` route count and an actual HTTP POST handler invocation.
