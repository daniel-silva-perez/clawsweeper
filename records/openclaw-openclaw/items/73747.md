## Summary

Keep open. This is a fresh, concrete Docker/workspace-plugin report against v2026.4.26, and current main still does not prove the reported inspect-time route registration versus request-time 404 mismatch is fixed. The documented API promises Gateway HTTP dispatch, but the current startup/runtime wiring still has plausible registry split paths and the existing coverage is unit-level rather than the reporter's Docker workspace-plugin POST path.

## Best Possible Solution

Reproduce on current main in Docker with a minimal trusted config-loaded workspace plugin under the default agent workspace that registers `/plugins/webhooks/gmail-push`; verify both `openclaw plugins inspect` route count and an actual POST handler invocation. If it reproduces, fix the Gateway startup/runtime HTTP-route registry wiring so the dispatcher resolves the same registry that served startup/config-loaded plugins, then add an integration regression for the workspace-plugin POST path. If it does not reproduce, document the exact container restart, config load path, auth/path checks, and live Gateway verification steps needed to diagnose stale inspect-vs-runtime state.
