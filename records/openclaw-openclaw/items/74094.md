## Summary

Keep open. Current main has task status snapshots, progress/terminal summaries, CLI tables, /tasks, /status, session_status, plugin task views, and notification formatting, but it still lacks the requested shared compressed operational-summary contract with current state, last successful checkpoint, current blocker, and recommended next action. The prior ClawSweeper keep-open comment remains consistent with current main.

## What This Changes

Add a deterministic shared task operational-summary builder with current state, last successful checkpoint, current blocker, and recommended next action, reused across CLI, task views, and auto-reply/status rendering.

## Best Possible Solution

Implement a shared deterministic task operational-summary builder in the task subsystem, expose it additively where task views need it, and reuse it across CLI list/show, /tasks, /status/session_status task lines, plugin task-domain views, and task notification rendering. Keep raw task fields intact, document the operator-visible output change, and cover active, blocked, failed, timed-out, cancelled, lost, and succeeded cases.
