## Summary

Keep open. Current main has task records, status snapshots, CLI summary columns, /tasks and /status task lines, session_status task context, plugin task views, and notification strings, but it still does not provide the requested shared compressed operational-summary contract with current state, last good checkpoint, current blocker, and recommended next action.

## What This Changes

The issue requests a shared compact task operational-summary builder reused across task CLI output, task summary views, auto-reply status rendering, and related task surfaces.

## Best Possible Solution

Add an additive task operational-summary helper in the task subsystem that returns structured fields plus a concise deterministic display string, then reuse it from CLI task list/show, /tasks, /status, session_status, plugin task views, and notification rendering while preserving existing raw task fields.

## Reproduction Assessment

Not applicable as a feature request rather than a runtime bug. The current-main source check is high confidence: the requested shared summary fields and builder are absent, and the relevant surfaces still format task state independently.

## Solution Assessment

Yes, the requested shared builder is the narrowest maintainable direction. It should live in the task subsystem, expose an additive contract, and be reused by existing renderers instead of replacing the task registry or plugin raw-field model.
