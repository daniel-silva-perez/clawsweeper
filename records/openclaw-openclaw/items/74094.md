## Summary

Keep open. Current main still has raw task status/progress/terminal fields plus several surface-specific renderers, but it does not provide the requested shared compressed operational-summary contract with current state, last good checkpoint, blocker, and recommended next action.

## What This Changes

Add a shared compressed task operational-summary builder/contract used consistently by task CLI output, task summary/status views, and auto-reply status rendering.

## Best Possible Solution

Add an additive TaskOperationalSummary contract in the task subsystem, derive it deterministically from structured task state, expose it through task-domain views, and make CLI/chat/status/session_status renderers consume it while preserving existing raw fields for compatibility.

Not applicable as a runtime reproduction because this is an enhancement request. Source inspection gives a high-confidence verification path: current main has raw task fields and per-surface formatting, not the requested shared operational-summary layer.

## Solution Assessment

Yes. A shared additive summary builder is the narrowest maintainable direction; per-surface wording tweaks would leave the inconsistency the issue is trying to remove.
