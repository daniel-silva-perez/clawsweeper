## Summary

Keep open. Current main still has raw task fields, cleanup-aware snapshots, and several per-surface task formatters, but it does not provide the requested shared compressed operational-summary contract with current state, last good checkpoint, blocker, and next action. The prior ClawSweeper comment reached the same conclusion, and the current source/docs still match that assessment.

## What This Changes

This issue requests a shared deterministic task operational-summary layer reused across task CLI output, task views, auto-reply/status rendering, and notifications.

## Best Possible Solution

Add an additive TaskOperationalSummary contract in the task subsystem, derive it deterministically from structured task state, expose it through task-domain views, and make CLI/chat/status/session_status/notification renderers consume it while retaining the existing raw fields.

## Reproduction Assessment

Not applicable as a runtime reproduction; this is an enhancement request. There is a high-confidence verification path by source/docs inspection: the current task model and renderers expose raw summaries and independent formatting, not the requested shared summary contract.

## Solution Assessment

Yes, the requested shared builder is the right maintainable direction if it stays additive and deterministic. A narrower per-surface wording tweak would leave the same inconsistency the issue is trying to remove.
