## Summary

Keep open. Current main has task records, cleanup-aware snapshots, CLI summary columns, /tasks and /status task lines, session_status task context, Task Flow blocked summaries, and notification strings, but it still does not provide the requested shared compressed operational-summary contract with current state, last good checkpoint, current blocker, and recommended next action.

## What This Changes

Add a shared compressed operational summary for tasks that captures current state, last successful checkpoint, current blocker, and recommended next action across CLI, task views, and auto-reply/status rendering.

## Best Possible Solution

Implement a shared deterministic task operational-summary builder in the task subsystem, then reuse it across CLI list/show, /tasks, /status and session_status task lines, task-domain/plugin views, and task notification rendering while keeping raw task fields intact and updating docs/changelog for the visible output change.
