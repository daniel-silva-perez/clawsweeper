## Summary

Keep open. Current main has adjacent task status, progress, blocked-flow, CLI summary, /tasks, and session_status rendering, but it does not implement the requested shared compressed operational summary contract with current state, last successful checkpoint, current blocker, and recommended next action. The existing surfaces still hand-roll different summary strings from status/progress/error fields.

## Best Possible Solution

Keep this open and implement a shared deterministic task operational-summary builder in the task subsystem, then have CLI list/show, /tasks auto-reply, session_status task lines, task-domain views, and task notification rendering consume it. The builder should infer or expose current state, last good checkpoint when known, blocker when present, and next action when known without replacing existing raw JSON fields.
