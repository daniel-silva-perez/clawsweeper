## Summary

Keep open. Current main still has the old `resolveWorkdir` behavior: non-sandbox exec checks the raw `workdir`, does not expand `~`, and falls back with a warning instead of erroring. This PR is still useful, but it needs maintainer/author follow-up because its final diff also removes unrelated changelog entries and is stale against later exec/shared-file refactors.

## Best Possible Solution

Keep this PR open and ask for a rebase/cleanup: preserve all unrelated changelog history, keep the focused `resolveWorkdir` fix that expands `~` against OS home rather than `OPENCLAW_HOME`, throw on missing or non-directory workdirs, and land targeted unit coverage for valid absolute paths, `~`, `~/subpath`, missing paths, and file-not-directory paths.
