## Summary

Keep this PR open. Current main has authenticated per-session history APIs, but it does not implement the PR's central user-facing change: a Sessions-page History shortcut or standalone archived-history viewer. The PR still needs maintainer/author revision because its proposed raw transcript viewer is security-sensitive and should align with the existing authenticated `chat.history` / session-history surfaces.

## Best Possible Solution

Keep the PR open for maintainer/author revision. The best path is to build the Sessions-page navigation on top of the existing authenticated session-history APIs, keep basePath and UI i18n behavior consistent, add focused UI/gateway coverage, and avoid a standalone unauthenticated `/history.js` raw transcript dump. If maintainers do not want a global archive viewer in core, close manually after saying that product decision explicitly.
