## Summary

Keep open. Current main does not implement the proposed Sessions shortcut or standalone viewer, and the PR still carries material endpoint-design and security questions around serving transcript data from the Control UI HTTP surface. The work is useful, but it needs maintainer/security review before merge rather than conservative cleanup closure.

## What This Changes

The PR adds a Sessions header History link and subtitle, standalone Control UI `/history` and `/history.js` transcript viewer routes with focused tests, and a `.vs/` gitignore entry.

## Best Possible Solution

Land a reworked history navigation design that keeps the Sessions shortcut basePath-safe, reuses the existing authenticated `/sessions/<key>/history` API or adds an equally scoped endpoint, respects configured agent/session-store behavior, handles archived and rotating transcript artifacts, and has focused gateway/UI tests for auth, basePath, and transcript-loading boundaries.
