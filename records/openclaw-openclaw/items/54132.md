Keep this PR open for maintainer review or rework. ## What This Changes

The PR adds a Sessions-page History link/subtitle, standalone `/history` and `/history.js` Control UI routes for archived session logs, focused UI/gateway tests, and a `.vs/` gitignore entry.

## Best Possible Solution

Keep this PR open for maintainer review or rework. The product direction is reasonable, but the landed design should add a basePath-safe Sessions History shortcut backed by the existing authenticated session-history API or a similarly scoped API, preserve configured agent/store behavior, bound or lazy-load transcript reads before loading contents, handle archived/rotated/deleted transcripts, and include focused gateway/UI tests.
