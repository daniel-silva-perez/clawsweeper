## Summary

Keep this PR open. Current main has not implemented the intended Feishu `ackReaction` customization: Feishu typing reactions still use the hardcoded `Typing` emoji, and the Feishu config schema/docs do not expose a channel or per-account `ackReaction` field. The PR remains a plausible focused feature candidate, but review discussion also flags behavior that still needs maintainer review before merge, especially preserving empty-string disable semantics.

## Best Possible Solution

Keep the PR open and revise it in the Feishu plugin: add account/channel `ackReaction` config, resolve it through the shared priority chain with a Feishu `Typing` channel default, preserve empty-string disable semantics, validate or safely fall back for unsupported Feishu emoji types, and align docs plus plugin/config metadata before landing.
