## Summary

Keep open. Current main still has the reported Memory Dreaming behavior, and open PR #74051 is the active implementation candidate with closing syntax, so this issue should remain open until that PR or a focused replacement lands.

## What This Changes

The issue reports that Memory Dreaming Light sorting, recall-gate eligibility, scoring, and rehydration let daily/session signals hide real recall data and block valid promotions.

## Best Possible Solution

Review and land #74051, or a focused replacement, so real recall signals drive Light ordering and recall-gate eligibility, daily/session ingestion can still inform scoring without satisfying recall gates by itself, rehydration handles shifted snippets, and tests/docs define the corrected contract.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the mechanism, and the existing promotion test explicitly shows daily-only signals with `recallCount: 0` clearing the default promotion gate.

## Solution Assessment

No for the original sort-only snippet. The maintainable fix should separate recall-gate semantics from daily/session scoring, preserve grounded evidence, widen rehydration, and update memory-core tests/docs together.
