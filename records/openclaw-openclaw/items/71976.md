## Summary

Keep open. Current main still contains the reported Memory Dreaming behavior, and open PR #74051 is linked with closing syntax for this issue, so the right cleanup path is to review or land that PR rather than close the issue early.

## What This Changes

The issue asks memory-core to stop daily/session-ingested signals from hiding real recall data in Light dreaming and deep promotion, and to widen live snippet rehydration for shifted daily-note ranges.

## Best Possible Solution

Review and land #74051, or a focused replacement, so real recall signals drive default Light selection and recall-gate eligibility, daily/session ingestion remains useful for scoring without satisfying recall gates by itself, rehydration handles shifted snippets, and memory-core tests/docs define the corrected signal contract.

## Reproduction Assessment

Yes. Current-main source verifies the reported mechanism: daily/session ingestion refreshes lastRecalledAt, Light sorts by that timestamp before recallCount, promotion gates on combined signalCount, and relocation still uses the narrow span.

## Solution Assessment

No for a sort-only fix. The reporter's follow-up correctly shows the maintainable fix spans Light ordering, recall-gate semantics, scoring expectations, rehydration, and matching tests/docs.
