## Summary

Keep open. Current main still contains the reported Memory Dreaming behavior, and open PR #74051 references this issue with closing syntax, so the maintainer-safe path is to review or land that PR before closing this issue.

## What This Changes

The issue reports that Memory Dreaming lets daily/session signals dominate Light ordering and promotion gates, burying real recall data and making rehydration too fragile.

## Best Possible Solution

Review and land #74051, or a focused replacement, so real recall signals drive default Light selection and recall-gate eligibility, daily/session ingestion remains useful without satisfying recall gates by itself, rehydration handles shifted snippets, and memory-core tests/docs define the corrected signal contract.

## Reproduction Assessment

Yes. Current-main source gives a high-confidence reproduction path: daily/session ingestion refreshes lastRecalledAt, Light sorts by that timestamp before recallCount, promotion gates on combined signalCount, and relocation still uses the narrow span.

## Solution Assessment

No for a sort-only fix. The reporter's follow-up and current source show the maintainable fix spans Light ordering, recall-gate semantics, scoring expectations, rehydration, and matching tests/docs.
