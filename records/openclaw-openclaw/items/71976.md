## Summary

Keep open. Current main still matches the reported memory-core behavior: Light dreaming refreshes daily/session signals before selecting candidates, sorts by refreshed recency before real recall count, and deep promotion still lets combined daily/grounded/recall signal totals satisfy the default recall gate. The issue is also paired with open, unmerged PR #74051 using closing syntax, so it should wait for that PR or a maintainer-approved replacement.

## What This Changes

The issue asks Memory Dreaming to prioritize real recall signals over daily/session ingestion in light selection and promotion gates, and to widen snippet rehydration for moved broad daily-note ranges.

## Best Possible Solution

Land or replace a focused memory-core fix that makes real recall data drive Light selection and default promotion eligibility, keeps daily/session ingestion from satisfying recall-count gates by itself while preserving intended scoring inputs, widens robust live-note rehydration, and updates the tests/docs that define the signal contract.
