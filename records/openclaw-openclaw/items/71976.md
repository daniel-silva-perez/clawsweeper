## Summary

Keep open. Current main still contains the reported Memory Dreaming behavior, and open PR #74051 is linked with closing syntax for this issue, so the right cleanup path is to review or land that PR rather than close the issue early.

## What This Changes

The issue asks Memory Dreaming to prioritize real recall signals over daily/session recency and to avoid letting daily-only signals hide or satisfy recall-based promotion data.

## Best Possible Solution

Review and land #74051, or a focused replacement, so real recall signals drive Light ordering and recall-gate eligibility, daily/session ingestion remains useful without satisfying recall gates by itself, rehydration handles shifted snippets, and memory-core tests/docs define the corrected contract.

## Reproduction Assessment

Yes. Current-main source inspection reproduces the mechanism: daily/session ingestion refreshes lastRecalledAt, Light sorts by that timestamp before recallCount, and promotion ranking/apply filters use combined signalCount for minRecallCount.

## Solution Assessment

No for the original sort-only fix. The reporter's follow-up points to the maintainable boundary: separate recall-gate semantics from daily/session scoring, keep grounded evidence intentional, widen rehydration, and cover the contract in tests/docs.
