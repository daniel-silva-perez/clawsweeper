## Summary

Keep this issue open. Current main still has the reported Light dreaming recency-first ordering, still lets daily signals contribute to the default recall-count gate, and still uses the narrow rehydration span. The issue is also paired with open, unmerged PR #74051, which explicitly targets these memory-core fixes and should be reviewed or replaced before this issue closes.

## What This Changes

The issue requests that Memory Dreaming prioritize real recall signals over daily/session ingestion signals in Light dreaming ranking, promotion recall gates, scoring, and short-term rehydration so valid recall candidates are not hidden or filtered out.

## Best Possible Solution

The desired end state is a focused memory-core change that makes real recall signals drive Light dreaming selection and promotion eligibility, prevents daily/session ingestion from satisfying recall gates by itself, widens robust snippet rehydration, and updates docs/tests around the signal contract.
