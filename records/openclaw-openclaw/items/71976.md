## Summary

Keep open. Current main still shows the reported Memory Dreaming behavior, and open PR #74051 is the active implementation candidate linked to this issue, so this should remain open until that PR or a focused replacement lands.

## What This Changes

The issue asks Memory Dreaming to prioritize real recall signals over daily/session ingestion noise in Light sorting and promotion gates, and to make rehydration more tolerant of shifted snippets.

## Best Possible Solution

Review and land #74051, or a focused replacement, so real recalls drive Light ordering and recall-gate eligibility, daily/session signals can still support scoring without satisfying recall gates alone, rehydration handles shifted snippets, and tests/docs define the corrected Memory Dreaming contract.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the mechanism: daily/session ingestion refreshes lastRecalledAt, Light sorts by that timestamp before recallCount, and deep promotion gates use combined signalCount rather than real recallCount.

## Solution Assessment

No for the original sort-only fix. The better solution is a memory-core fix that separates recall-gate semantics from daily/session scoring, preserves intentional grounded evidence, widens or otherwise hardens rehydration, and covers the behavior in focused tests/docs.
