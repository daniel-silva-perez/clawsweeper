## Summary

Current main at 68561a8c still forwards ordinary Telegram inline button callback_query data to the agent as the raw callback payload, while the public Telegram docs promise the text shape `callback_data: <value>`. PR #54962 remains open and unmerged as the implementation candidate, so this issue should stay open until that fix or an equivalent change lands.

## Best Possible Solution

Keep this issue open. Review and land PR #54962, or apply an equivalent Telegram plugin change that forwards ordinary generic callback_query fallback text as `callback_data: ${data}` while preserving native `tgcmd:` callbacks and the existing approval, plugin, pagination, and model callback paths. Add a focused regression test for the documented prefix before closing.
