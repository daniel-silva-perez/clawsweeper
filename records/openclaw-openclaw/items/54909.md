## Summary

Keep open. Current main still forwards generic Telegram callback_query fallback data to the agent as the raw callback payload, while the Telegram docs promise the `callback_data: <value>` text form. The linked PR #54962 is still an open, unmerged implementation candidate for this issue, so this should not be closed before that fix or an equivalent change lands.

## What This Changes

The issue requests that Telegram inline button callback_query payloads be routed to the agent as documented callback_data text so configured tools can act on the button press.

## Best Possible Solution

Review and land PR #54962, or apply an equivalent targeted change in `extensions/telegram/src/bot-handlers.runtime.ts` that forwards only generic Telegram callback fallback text as `callback_data: ${data}` while preserving internal callback handlers. Add a regression test for generic callback prefixing, then close this issue after the fix merges.
