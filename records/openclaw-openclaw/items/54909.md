## Summary

Keep open. Current main still routes generic Telegram `callback_query` fallback payloads to the agent as raw callback text, while the Telegram docs promise `callback_data: <value>`. The provided GitHub context also shows open PR #54962 uses closing syntax for this issue and remains unmerged, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that Telegram inline button `callback_query` payloads be routed to the agent in the documented `callback_data: <value>` form so prompt instructions can trigger an HTTP tool call instead of a fabricated confirmation.

## Best Possible Solution

Review and land PR #54962, or apply the same targeted change in `extensions/telegram/src/bot-handlers.runtime.ts` so only generic callback fallback text is forwarded as `callback_data: ${data}` while internal callback handlers keep their current behavior. Add a regression test for the generic callback prefix and close this issue after the fix merges.
