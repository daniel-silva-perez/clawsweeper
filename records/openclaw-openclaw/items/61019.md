## Summary

Keep this issue open. Current main still has manual per-session reset paths, but `openclaw models set` only updates the configured default model and does not migrate existing session rows or active runtime model state. Existing session display/resolution paths can still prefer persisted runtime or override fields before falling back to the new default, so the reported active-session inheritance request is not fully implemented.

## Best Possible Solution

Keep the issue open and implement a deliberate migration or refresh path for sessions after the global default changes. A safe direction is to let `openclaw models set` offer or trigger a scoped refresh for sessions that only have stale runtime model identity, while preserving explicit user-selected per-session overrides unless the user opts to reset them.
