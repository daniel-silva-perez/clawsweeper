## What This Changes

The issue requests updating the bundled Notion skill docs or adding a helper so Notion API examples work in strict exec allowlist setups without shell-wrapped curl patterns.

## Best Possible Solution

Update the bundled Notion skill so its default runnable examples avoid shell command substitution, `$` expansion, semicolon chaining, and backslash continuations under strict exec policies. A small stdlib helper script invoked as a normal `python3 <script>` file is likely cleaner than relaxing core allowlist behavior.
