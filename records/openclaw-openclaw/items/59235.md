## Summary

Keep this issue open. Current main still documents the bundled Notion skill with `NOTION_KEY=$(cat ...)` plus `curl` examples that rely on shell expansion, while the macOS exec-approval docs say shell control/expansion syntax is treated as an allowlist miss. The two linked fix PRs, #59254 and #59870, were closed unmerged, so there is no shipped or canonical merged fix to close against.

## Best Possible Solution

Keep the issue open and update `skills/notion/SKILL.md` to use allowlist-friendly Notion examples: either direct executable invocations without shell expansion/chaining, a small helper script pattern using an already allowlisted binary, or an explicit note warning that shell variable setup and `$NOTION_KEY` expansion can miss under strict allowlist/off-ask policies. The closed PRs #59254 and #59870 are useful references for the intended docs direction.
