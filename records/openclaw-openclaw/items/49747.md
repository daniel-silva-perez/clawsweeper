## Summary

Keep this issue open. Current main still has the central parser gap in `resolveGroupSessionKey()`: it treats only the first `From` segment as the provider, so an agent-prefixed Slack route can still fall back to `ctx.Provider` such as `webchat`. The restart-sentinel fallback path also still merges base-session routing into partial thread metadata, and the provided timeline shows open PR #49750 as the active implementation candidate rather than a reason to close early.

## Best Possible Solution

Keep this issue open and review/land #49750 or an equivalent maintainer fix. The durable fix should make `resolveGroupSessionKey()` use the shared agent-aware session-conversation parser, preserve explicit A2A target delivery context over session/base fallback, limit base fallback to missing fields on the same route, and add focused regression coverage for agent-prefixed Slack thread metadata plus announce/restart routing.
