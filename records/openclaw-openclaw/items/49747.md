## Summary

Current main still has the reported session-key parsing gap: `resolveGroupSessionKey()` treats the first segment of `ctx.From` as the possible transport, so an agent-prefixed key such as `agent:builder:slack:channel:...` can fall back to `ctx.Provider` instead of extracting `slack`. The restart/announce path has partial-route fallback logic that can still merge a base session route when the current session lacks a routable `channel` + `to`. The issue also has an open related implementation candidate, #49750, so this issue should stay open until that PR is reviewed and merged or rejected.

## Best Possible Solution

Keep this issue open and review #49750 as the likely fix path. The implementation should make shared session-key parsing extract the real transport from agent-prefixed keys, preserve explicit target delivery context for A2A runs, and add focused regression coverage for `agent:<id>:slack:channel:<channelId>:thread:<threadTs>` metadata and announce routing.
