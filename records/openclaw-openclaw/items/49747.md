## Summary

Keep this open. Current main still has the central parser gap: `resolveGroupSessionKey()` only treats the first `From` segment as a possible provider, so `agent:builder:slack:channel:...` can still fall back to `ctx.Provider` such as `webchat`. The related restart/announce fallback path also still merges base-session routing when the current route is incomplete, and the provided timeline shows open PR #49750 as the implementation candidate.

## Best Possible Solution

Keep this issue open and review #49750 as the likely fix path. The fix should teach the shared session-key parser to extract the real provider from `agent:<id>:<provider>:...`, preserve explicit A2A target delivery context over base-session fallback, and add focused regression coverage for Slack thread metadata plus announce routing.
