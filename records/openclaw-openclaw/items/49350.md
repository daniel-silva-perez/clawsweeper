## Summary

Keep this issue open. Current main still omits `agents.list[].identity.name` from the Google Chat typing display-name fallback, still sends reply/action/attached-result text to Google Chat without a channel-owned Markdown strip/convert step, and the open PR linked from the issue remains an unmerged implementation candidate for only the typing half.

## Best Possible Solution

Keep this issue open. Review and either land or close PR #49361 for the typing fallback, then add a Google Chat-owned Markdown normalization step before every `sendGoogleChatMessage`/`updateGoogleChatMessage` text path used for replies, message actions, and attached results, with focused regression tests and any needed metadata/docs adjustment.
