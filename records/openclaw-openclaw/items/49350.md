## Summary

Keep this issue open. Current main still lacks the Google Chat typing fallback to `agent.identity.name`, and the Google Chat send paths still pass markdown-formatted text through without a channel-owned strip/convert step. The provided GitHub context also shows open PR #49361 explicitly references this issue with closing syntax, so the issue should stay open until that PR or an equivalent complete fix lands.

## Best Possible Solution

Keep the issue open. Review/land or close #49361 for the typing-indicator identity fallback, then implement the Google Chat markdown normalization in `extensions/googlechat` send paths with focused regression tests before closing this issue.
