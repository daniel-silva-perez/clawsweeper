## Summary

Keep open. Current main and the latest release still leave both reported Google Chat behaviors unresolved, and open PR #49361 remains an unmerged implementation candidate for only the typing-display-name half.

## What This Changes

The issue requests Google Chat outbound text to strip or convert Markdown and the typing indicator to use the configured agent identity name instead of falling back to OpenClaw.

## Best Possible Solution

Review and either land or replace PR #49361 for the typing fallback, then add a Google Chat-owned Markdown normalization step across reply delivery, message actions, media captions, and direct outbound adapter sends with focused regression coverage.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: with only agents.list[].identity.name configured, resolveBotDisplayName falls through to OpenClaw; with Markdown text, reply and action paths pass chunks or content directly into Google Chat text fields.

## Solution Assessment

Yes for the direction, not yet for current main. The narrow maintainable fix belongs inside the Google Chat plugin delivery boundary, adding identity.name to the fallback chain and normalizing outbound Google Chat text without changing core channel behavior.
