## Summary

Keep this issue open. Current main still does not use `agents.list[].identity.name` in the Google Chat typing display-name fallback, and the Google Chat reply/action send paths still forward markdown-formatted text without a channel-owned strip/convert step. Live PR #49361 remains open and unmerged with closing syntax for the typing fallback, while the markdown PR #49406 is closed and unmerged.

## Best Possible Solution

Keep this issue open. Review and either land or close PR #49361 for the typing-indicator fallback, then add a Google Chat-owned markdown normalization step across reply delivery, message actions, and attached-results outbound delivery with focused regression tests before closing.
