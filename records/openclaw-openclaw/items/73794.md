## Summary

Keep this PR open. Current main still contains the `hasExplicitPluginConfig(params.cfg?.plugins)` guard, and `hasExplicitPluginConfig` returns true for any non-empty `plugins.entries`, so the reported installed-plugin case still bypasses active-registry reuse for non-memory/non-speech capability lookups. The PR is a narrow two-file fix with a targeted regression test and no workflow, dependency, lockfile, release, package metadata, or secrets surface changes.

## Best Possible Solution

Keep this PR open for maintainer review. If maintainers agree that loader cache-key compatibility already protects real config changes, land this PR or an equivalent patch with the explicit `plugins.entries` regression test, then let #73793 close through the merge.
