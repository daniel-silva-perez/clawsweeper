## Summary

Keep open. Current main still does not expose `sessionId` or `sessionKey` on `ProviderCreateStreamFnContext`, and the provider stream registration paths still drop the session identity that the embedded attempt, compaction, and BTW paths already have. The PR remains paired with open issue #73488 and concrete downstream glueclaw issue/PR context describing transcript bleed. Security pass found no CI, dependency, lockfile, install, release, or downloaded-code changes in the PR diff; the main review risk is documenting the trusted provider-plugin boundary for exposing routing identifiers.

## Best Possible Solution

Keep this PR open for normal maintainer review. If accepted, land the additive optional `sessionId` and `sessionKey` fields on `ProviderCreateStreamFnContext`, thread them through `registerProviderStreamForModel` from embedded attempt, compaction, and BTW paths, leave one-off simple/image completion paths undefined where appropriate, add focused passthrough tests, and document the provider-plugin trust boundary before closing #73488.
