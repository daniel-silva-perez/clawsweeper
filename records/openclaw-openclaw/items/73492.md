## Summary

Keep open. Current main still does not expose `sessionId` or `sessionKey` to provider `createStreamFn` plugins, and the PR is a focused additive API change tied to the still-open #73488 plus downstream glueclaw transcript-bleed context.

## Best Possible Solution

Keep this PR open for normal maintainer review. If accepted, land the additive optional `sessionId` and `sessionKey` fields on `ProviderCreateStreamFnContext`, pass them through `registerProviderStreamForModel` from the embedded attempt, compaction, and BTW paths, leave one-off image/simple-completion paths undefined, add narrow passthrough tests, and document the `sessionKey` trust boundary before closing #73488.
