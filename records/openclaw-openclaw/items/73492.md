## Summary

Keep this PR open. Current main still does not expose `sessionId` or `sessionKey` to provider `createStreamFn` plugins, and the same-author linked issue #73488 describes a concrete concurrent-conversation transcript-bleed failure mode. This PR is a focused implementation candidate rather than obsolete or already shipped.

## Best Possible Solution

Keep this PR open for normal maintainer review. If accepted, land the additive optional SDK fields with focused regression coverage for the attempt, compaction, and BTW paths, and document the sensitivity/preferred use of `sessionKey`; then #73488 can close after merge.
