## Summary

Keep open. Current main at 230f7122dd5cd8a37a55e671254bd9be8278f6e8 still lacks sessionId/sessionKey on ProviderCreateStreamFnContext and still drops session identity in provider stream registration, even though the embedded attempt, compaction, and BTW paths already have the relevant values. The PR is a focused additive core plugin API change paired with open #73488 and external glueclaw repro context; it is not implemented, not superseded, and not a ClawHub-only item because plugins cannot obtain these fields without a core API seam.

## Best Possible Solution

Keep this PR open for maintainer review. If accepted, add optional sessionId/sessionKey to ProviderCreateStreamFnContext, thread them through registerProviderStreamForModel from embedded attempt, compaction, and BTW, leave one-off image/simple-completion paths undefined or explicitly documented, add focused passthrough tests, and document the provider-plugin trust boundary before closing #73488.
