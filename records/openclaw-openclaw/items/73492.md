## Summary

Current main still does not expose `sessionId` or `sessionKey` on `ProviderCreateStreamFnContext`, and the embedded attempt, compaction, and BTW provider stream registration paths still drop session identity before provider `createStreamFn` plugins can see it. The PR is a focused additive API/plumbing change paired with open #73488 and concrete downstream glueclaw transcript-bleed context; the remaining action is maintainer API, test, and trust-boundary review, not cleanup closure.

Keep this PR open for normal maintainer review.
