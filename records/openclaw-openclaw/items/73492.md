## Summary

Keep open. Current main still does not expose `sessionId` or `sessionKey` on `ProviderCreateStreamFnContext`, and the embedded attempt, compaction, and BTW provider-stream registration paths still drop session identity that is already in scope. The PR is also paired with open issue #73488 and linked downstream glueclaw issue/PR context describing transcript bleed, so it should stay open for maintainer review rather than be auto-closed.

## Best Possible Solution

Keep this PR open for maintainer review. If accepted, land the additive optional `sessionId` and `sessionKey` fields on `ProviderCreateStreamFnContext`, thread them through `registerProviderStreamForModel` from embedded attempt, compaction, and BTW paths, leave one-off image/simple completion paths undefined where appropriate, add focused passthrough tests, and document the trusted provider-plugin boundary before closing #73488.
