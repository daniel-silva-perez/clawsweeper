## Summary

Keep this PR open. Current main still does not expose `sessionId` or `sessionKey` on `ProviderCreateStreamFnContext`, and the embedded attempt, compaction, and BTW provider-stream paths still call `registerProviderStreamForModel` without session identity. The linked #73488 remains open, this PR uses closing syntax for it, and the downstream glueclaw issue/PR context describes a concrete transcript-bleed failure that the current core API still cannot solve.

## Best Possible Solution

Keep this PR open for normal maintainer review. If accepted, land the additive optional `sessionId` and `sessionKey` fields on `ProviderCreateStreamFnContext`, thread them through `registerProviderStreamForModel` from embedded attempt, compaction, and BTW paths, leave one-off image/simple-completion paths undefined where appropriate, add focused passthrough tests, and document the provider-plugin trust boundary before closing #73488.
