Keep this PR open. Current main has overlapping generic memory embedding provider fallback behavior, but it does not include this PR's explicit memory CLI provider-registration bootstrap or CLI-specific regression coverage, and the PR is part of an active same-author stacked memory series.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to decide whether the memory CLI should explicitly seed built-in embedding providers before manager creation, or whether the current manifest/capability fallback is the intended architecture and only CLI regression coverage is useful. If the hook is kept, validate the guard semantics and run the targeted CLI test or CI, then coordinate with the stacked memory PRs.
