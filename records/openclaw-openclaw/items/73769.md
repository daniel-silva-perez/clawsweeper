## Summary

Keep this PR open. Current main has a generic memory embedding provider fallback through plugin capability discovery and the memory-core manifest declares the local provider, but main does not include this PR‚Äôs explicit memory CLI provider-registration bootstrap or its CLI-specific regression coverage. The PR is also the root of an active same-author stacked memory series, so it should get maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to decide whether the memory CLI should explicitly register built-in embedding providers before manager creation, or whether the current manifest/capability fallback is the intended architecture and only CLI regression coverage is useful. Coordinate that decision with the open stacked memory PRs #73770 through #73773.
