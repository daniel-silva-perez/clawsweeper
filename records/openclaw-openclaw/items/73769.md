## Summary

Keep this PR open. Current main has a generic cold-registry memory embedding provider fallback and the bundled memory-core manifest declares the local provider, but main does not contain this PR's memory CLI bootstrap hook or its CLI-specific regression coverage. The PR is also the root of an active same-author memory stack, so it should get maintainer review rather than conservative cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to either land a narrowed version that adds explicit memory CLI provider initialization plus regression coverage, or ask the author to rebase/drop the initialization if maintainers decide current capability-backed provider resolution is sufficient and only the test is useful. Coordinate that decision with the open stacked memory PRs.
