## Summary

Keep open. Current main still matches the reported Matrix/MAS cross-signing failure path, and the same-author implementation PR #74509 is open, so closing this issue before that PR lands or is superseded would be premature.

## What This Changes

The issue requests Matrix cross-signing bootstrap support for MAS-fronted Synapse PAT accounts by honoring MSC3967 first-publish behavior, parsing UIA challenges, threading UIA session data, and surfacing MAS reset URLs instead of the password fallback message.

## Best Possible Solution

Land or supersede #74509 with a Matrix plugin-owned fix that parses Matrix UIA 401 challenges, preserves the UIA session, surfaces a typed MAS reset URL for operator approval, keeps MSC3967 first-publish uploads available, and updates focused crypto-bootstrap tests plus Matrix operator docs.

## Reproduction Assessment

Yes. The code-level reproduction is high-confidence: make the Matrix SDK auth callback reject no-auth and dummy UIA with no configured password, and current tests assert the same misleading password fallback error; the issue also provides a concrete MAS 401 response shape from a live deployment.

## Solution Assessment

Yes at the design level, but it is not implemented on main yet. Handling this in the Matrix plugin through the existing cross-signing auth callback and homeserver-capability probe is the narrowest maintainable path; the open PR should be reviewed, landed, or superseded.
