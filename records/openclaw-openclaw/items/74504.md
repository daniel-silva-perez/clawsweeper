## Summary

Keep open: the report matches current `main` behavior. The Matrix bootstrap UIA callback still retries only no-auth, dummy, and password auth, does not inspect the 401 UIA body or propagate `session`, and the reset fallback can still retry with `setupNewCrossSigning: true` after an initial key-mismatch failure. Current docs and tests also encode the password-fallback-only behavior, while upstream Synapse now exposes MAS cross-signing reset via OAuth/custom UIA stages, so this needs a Matrix plugin fix rather than cleanup closure.

## What This Changes

The issue requests MAS-aware Matrix cross-signing bootstrap support for PAT-authenticated bots, including MSC3967 first-publish handling, structured MAS reset URL surfacing, and removal of the misleading password-only UIA guidance.

## Best Possible Solution

Implement this in the Matrix plugin boundary: add typed parsing for Matrix UIA challenge data, preserve and resend UIA `session`, detect `m.oauth`/`org.matrix.cross_signing_reset` and surface a structured operator-facing reset URL, keep MSC3967 first-publish uploads unblocked, and cover the flow with focused crypto-bootstrap tests plus Matrix docs for MAS operators.
