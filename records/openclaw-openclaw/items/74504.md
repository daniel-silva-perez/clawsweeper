## Summary

Keep open. Current main still matches the reported Matrix UIA fallback behavior and does not include MAS `org.matrix.cross_signing_reset`, `auth_metadata`, typed reset-url, or session-threading support; the same-author implementation PR #74509 remains open, so this issue should stay open until that PR lands or is superseded.

## What This Changes

The issue requests Matrix cross-signing bootstrap support for MAS-fronted Synapse, including MSC3967 first-publish behavior, UIA session handling, and surfacing the MAS cross-signing reset URL.

## Best Possible Solution

Review and land or supersede #74509 with a Matrix plugin-owned fix that parses UIA 401 challenges, preserves the UIA session, surfaces a typed MAS reset URL for operator approval, keeps MSC3967 first-publish uploads available, and updates focused crypto-bootstrap tests plus Matrix operator docs.

## Reproduction Assessment

Yes. The code-level reproduction is high-confidence: invoke the current Matrix SDK auth callback with no configured password while no-auth and dummy UIA fail, and current tests assert the same misleading password fallback error; the report also provides a concrete live MAS 401 response shape.

## Solution Assessment

Yes at the design level, but it is not implemented on main yet. Handling this in the Matrix plugin's cross-signing UIA callback and homeserver-capability probe is the narrowest maintainable path; the active PR is the candidate to review, land, or replace.
