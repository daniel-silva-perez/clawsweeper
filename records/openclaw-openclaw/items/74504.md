## Summary

Current main still follows the reported Matrix UIA fallback path and lacks MAS `org.matrix.cross_signing_reset`, `auth_metadata`, MSC3967-specific, or typed reset-url handling; the same-author implementation PR #74509 is open, so closing this before that PR lands or is superseded would be premature.

## What This Changes

The issue requests a Matrix plugin fix so MAS-fronted Synapse/PAT bot cross-signing bootstrap honors MSC3967 first-publish, preserves UIA session handling, and surfaces the MAS cross-signing reset URL instead of suggesting password UIA.

## Best Possible Solution

Land or supersede #74509 with a Matrix plugin-owned fix that parses Matrix UIA 401 challenges, preserves the UIA session, surfaces a typed MAS reset URL for operator approval, keeps MSC3967 first-publish uploads available, and updates focused crypto-bootstrap tests plus Matrix operator docs.
