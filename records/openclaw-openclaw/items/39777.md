## Summary

Keep open. Current main still has the reported Control UI feedback gap: the Debug snapshot refresh returns with no user-facing state when disconnected, and the Snapshots card still only changes the button label while loading. The linked implementation PR #39806 remains open and unmerged, so the issue should stay open until that PR is reviewed, landed, or replaced.

## What This Changes

Add visible loading/progress feedback in the Control UI Debug Snapshots card and show a clear disconnected-gateway message when refresh cannot run.

## Best Possible Solution

Resolve this through the existing Control UI path: pass connection/refresh-disabled state into the Debug view, render an inline loading state while debugLoading is true, render a disconnected callout when refresh cannot run, and add focused view/controller tests for those states.

## Reproduction Assessment

Yes. The current source gives a high-confidence static reproduction path: loadDebug exits on a missing/disconnected client, and renderDebug has no inline loading or disconnected branch in the Snapshots card.
