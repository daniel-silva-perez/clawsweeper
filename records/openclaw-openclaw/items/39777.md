## Summary

Keep this issue open. Current main still lacks an inline loading/progress state in the Debug Snapshots card, and the refresh controller still returns without user-facing feedback when no client/connection is available. PR #39806 remains an open, unmerged implementation candidate tied directly to this report, so the issue should stay open until that PR is reviewed/landed or replaced.

## Best Possible Solution

Keep the issue open. Review, refresh, or replace PR #39806; the maintainable fix belongs in the Control UI Debug surface by passing connection/disabled state into renderDebug(), showing an inline loading/progress state while debugLoading is true, showing a clear disconnected callout when refresh cannot run, and adding focused controller/view tests for those states.
