## Summary

Keep this issue open. Current main covers threaded Google Chat replies for delivered messages, but it still does not implement the second request: optional receipt of all non-mention messages in spaces via user OAuth, Workspace Events subscriptions, or an equivalent delivery mode. The open related PR #68967 is scoped to thread/session behavior, and #9764 remains the adjacent Google Chat user-OAuth foundation tracker.

## Best Possible Solution

Keep this issue open, or split it after #68967 resolves the threading half. The remaining best path is Google Chat plugin-owned: design an optional user-OAuth or Workspace Events subscription delivery mode for non-mention space messages, keep the existing service-account webhook path fail-closed, extend config/schema/setup/docs with secret-aware credentials and delivery-mode behavior, and coordinate with #9764 for shared OAuth foundations.
