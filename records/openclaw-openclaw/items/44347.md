## Summary

Keep this issue open. Current main appears to cover the threaded reply half for delivered Google Chat messages, but it still does not implement optional receipt of all non-mention messages in spaces via user OAuth, chat.messages scope, or an equivalent delivery mode. The related open PR #68967 is scoped to thread/session behavior, and related issue #9764 remains the better adjacent tracker for the Google Chat user-OAuth credential foundation.

## Best Possible Solution

Keep this issue open, or split it after #68967 resolves the threading half. The remaining best path is Google Chat plugin-owned: design an optional user-OAuth or verified event-subscription mode for non-mention space messages, keep the existing service-account webhook path fail-closed, extend config/schema/setup/docs with secret-aware credentials and delivery-mode behavior, and add focused tests for routing, mention gating, redaction, scope/token selection, and backward compatibility. Coordinate with #9764 because the OAuth credential foundation likely overlaps.
