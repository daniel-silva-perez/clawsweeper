## Summary

Keep this issue open. Current main still omits `--exclude-labels` from the Gmail watcher serve command, the strict Gmail config surface still has no `excludeLabels` field, and there are already open PRs with closing syntax for this report, so the maintainer path is to choose and land one canonical fix rather than close the issue early.

## What This Changes

Restore Gmail watcher excluded-label handling so `gog gmail watch serve` excludes spam, trash, drafts, and sent mail by default, ideally with typed configuration.

## Best Possible Solution

Land one canonical fix that restores the default excluded labels to `SPAM,TRASH,DRAFT,SENT`, wires a typed `hooks.gmail.excludeLabels` override through schema, runtime config, generated config help/labels, CLI/docs if exposed, and adds regression coverage for default, custom, and intentionally empty exclusion behavior.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to resolve the default Gmail hook config and inspect `buildGogWatchServeArgs` or `buildGogWatchServeLogArgs`; current main cannot emit `--exclude-labels` because neither the runtime config nor the arg builder contains that field.

## Solution Assessment

Yes. A configurable `excludeLabels` default is the narrowest maintainable fix because it restores the old default while giving operators an explicit override; a one-line hardcoded flag would address only the default behavior and leave the config gap reported here.
