## Summary

Keep this issue open. Current main still omits `--exclude-labels` from the Gmail watcher serve args, the strict Gmail config schema and exported config type still have no `excludeLabels` field, and open PRs with closing syntax are already tracking candidate fixes, so maintainers should choose and land one canonical fix rather than close this issue early.

## What This Changes

Restore Gmail watcher excluded-label handling so sent mail, drafts, spam, and trash are excluded from webhook processing by default, preferably with a typed `hooks.gmail.excludeLabels` override.

## Best Possible Solution

Land one canonical fix that restores the default excluded labels to `SPAM,TRASH,DRAFT,SENT`, adds a typed `hooks.gmail.excludeLabels` override through config schema/types/runtime resolution, updates generated config help/docs as needed, and adds focused regression coverage for default, custom, and intentionally empty exclusion behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to resolve the default Gmail hook config and inspect `buildGogWatchServeArgs` or `buildGogWatchServeLogArgs`; current main cannot emit `--exclude-labels`, and the existing test expectation confirms that omission.

## Solution Assessment

Yes. The best fix is the configurable default proposed by the fuller open PR path because it restores the old default and closes the reported config gap; a hardcoded one-line flag would only restore default behavior.
