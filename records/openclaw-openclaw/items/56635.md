## Summary

Keep this issue open. Current main still omits `--exclude-labels` from the Gmail watcher serve command, the strict Gmail config schema has no `excludeLabels` field, and multiple open PRs already reference this issue with closing syntax, so the right maintainer path is to review and land one canonical fix rather than close the report early.

## What This Changes

The issue reports that Gmail watcher startup no longer passes `--exclude-labels SPAM,TRASH,DRAFT,SENT` to `gog gmail watch serve`, causing sent and draft Gmail events to reach OpenClaw.

## Best Possible Solution

Land one canonical fix that restores Gmail watcher excluded-label handling, preferably by adding a typed/configured `excludeLabels` field defaulting to `SPAM,TRASH,DRAFT,SENT`, wiring it through runtime config and docs/schema metadata, and adding regression tests for default, custom, and empty exclusion behavior.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to resolve the default Gmail hook config and inspect `buildGogWatchServeArgs`: current main cannot emit `--exclude-labels` because neither the runtime config nor the arg builder contains that field.

## Solution Assessment

Yes, a configurable fix like the one described in #56720 appears to be the narrowest maintainable direction because it restores the previous default while giving operators an explicit override; the issue should remain open until a canonical PR is merged.
