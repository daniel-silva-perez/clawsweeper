## Summary

Keep this PR open. Current main still relies on CSS-only hiding for WebChat code-block copy labels, so the intended `CopyCopied!` fallback fix is not implemented. The PR remains useful, but it needs a current-main refresh because main now has separate fenced and indented code-block renderers, while the submitted patch only covers the older single renderer path and introduces a broader DOMPurify `hidden` allowlist change that deserves maintainer review.

## Best Possible Solution

Keep the PR open and ask for a current-main update that applies the hidden fallback to both fenced and indented code-block copy buttons, toggles `HTMLElement.hidden` symmetrically with the `copied` class in `renderChat`, and adds regression coverage for default label visibility. Prefer setting `hidden` only on generated `.code-block-copy__done` elements, or document maintainer acceptance if the global DOMPurify allowlist is retained.
