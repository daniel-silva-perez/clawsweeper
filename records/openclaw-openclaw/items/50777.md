## Summary

Keep this PR open. Current main and the latest release still render Control UI code-block copy labels with CSS-only hiding, so the user-facing bug is not fixed, but the proposed patch needs a current-main refresh because there are now separate fenced and indented code-block renderer paths.

## What This Changes

The PR adds `hidden` to the code-block copy button's `Copied!` span, toggles that attribute in the chat copy handler, allowlists `hidden` in DOMPurify, and adds a markdown regression test.

## Best Possible Solution

Refresh or replace the branch with a current-main fix that gives both fenced and indented code-block copy buttons a non-CSS initial hidden state, keeps the click-state toggle symmetric, and adds focused regression coverage for both renderer paths.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: current main emits both copy labels for fenced and indented code blocks while relying on CSS to hide `Copied!`, so delaying or omitting CSS exposes `CopyCopied!`.

## Solution Assessment

No. The proposed direction is right, but the submitted patch is not the best current-main solution because it only covers the older/fenced renderer shape and uses a broader sanitizer allowlist than necessary.
