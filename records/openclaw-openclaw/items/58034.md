## Summary

Keep this PR open. Current main has shared trusted env-proxy guard machinery and the #59007 DNS-before-proxy ordering fix, but it still does not implement this PR's central user-facing change: a default-off `tools.web.fetch.useTrustedEnvProxy` config field wired into built-in `web_fetch`. The discussion also contains useful live proxy-only sandbox verification from mjamiv and smallmj, and the author correctly distinguishes this work from #59007 and the RFC2544 fake-IP path: those adjacent fixes do not provide the explicit `web_fetch` opt-in, docs/schema surface, or full trusted-proxy hardening proposed here.

## Best Possible Solution

Keep this PR open as the implementation candidate for proxy-only built-in `web_fetch` support. The best path is maintainer review of the latest branch, preserving strict default behavior while landing a default-off `tools.web.fetch.useTrustedEnvProxy` config field from schema/docs through `createWebFetchTool` into `fetchWithWebToolsNetworkGuard`, with hostname-policy checks before trusted proxy dispatch and tests for default strict mode, matching proxy env vars, no matching proxy, `NO_PROXY` fallback, and dangerous hostnames.
