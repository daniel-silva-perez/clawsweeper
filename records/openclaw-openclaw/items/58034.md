## Summary

Keep this PR open. Current main still has the lower-level trusted-env-proxy machinery, but it does not implement the PR's central user-facing change: a default-off `tools.web.fetch.useTrustedEnvProxy` config surface wired into ordinary `web_fetch`. Main also still lacks the PR's trusted-env-proxy hostname-policy precheck in the env-proxy branch, so the branch is not superseded by #59007 or later cleanup.

## Best Possible Solution

Keep this PR open as the canonical review thread for proxy-only `web_fetch` support. The best path is to review the latest head, then land a default-off `tools.web.fetch.useTrustedEnvProxy` setting wired from schema/docs through `createWebFetchTool` into `fetchWithWebToolsNetworkGuard`, preserve protocol-aware proxy and `NO_PROXY` fallback behavior aligned with Undici, keep hostname policy checks before env-proxy dispatch, and retain focused tests for strict default behavior, explicit trusted proxy mode, no matching proxy, ALL_PROXY-only behavior, and NO_PROXY fallback.
