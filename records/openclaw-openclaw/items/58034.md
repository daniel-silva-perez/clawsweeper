## Summary

Keep this PR open. Current main has some shared trusted-env-proxy machinery and protocol-aware proxy matching, but it still does not implement the PR's central user-facing change: a default-off `tools.web.fetch.useTrustedEnvProxy` config surface wired into ordinary `web_fetch`. Main also still lacks the PR's trusted-env-proxy pre-dispatch hostname-policy hardening in `fetchWithSsrFGuard`. Related #59007 merged the narrower DNS-before-proxy ordering fix, while #63565 and #71056 point back here as the remaining canonical implementation thread.

## Best Possible Solution

Keep this PR open as the canonical review thread for proxy-only `web_fetch` support. The best path is to land a default-off `tools.web.fetch.useTrustedEnvProxy` config field wired from schema/docs through `createWebFetchTool` into `fetchWithWebToolsNetworkGuard`, preserve protocol-aware proxy and `NO_PROXY` fallback behavior aligned with Undici, keep hostname policy checks before proxy dispatch, and retain focused regression tests for strict default behavior, opt-in trusted proxy mode, no matching proxy, and `NO_PROXY` fallback.
