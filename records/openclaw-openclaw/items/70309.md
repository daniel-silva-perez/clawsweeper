## Summary

Keep open. Current main still has MCP configuration editing and session-time tool discovery, but no dedicated Control UI panel or Gateway diagnostics API for per-server connected/error state, tool names, manual refresh/reconnect, or surfaced redacted error details.

## What This Changes

The issue requests a Control UI MCP connectivity panel that shows per-server status, available tools, reconnect/refresh actions, and useful connection errors for configured `mcp.servers`.

## Best Possible Solution

Keep this open and design an additive Gateway operator diagnostics surface for MCP, backed by the MCP config/runtime loader, returning each configured server name, transport summary, connected/error state, tool names/count, and redacted last error, with a scoped refresh action. Then render it under Control UI Infrastructure > MCP and document the operator behavior.

## Reproduction Assessment

Not applicable as a feature request. The current-main check is high-confidence: the UI only exposes MCP configuration/counts, docs state config commands do not connect to servers, and no Gateway MCP health or reconnect method exists.

## Solution Assessment

Yes, the maintainable direction is an additive Gateway protocol/server-method plus Control UI panel. Reusing `tools.catalog` or `tools.effective` alone is insufficient because those methods cannot represent configured-but-failed MCP servers or provide a refresh/reconnect action.
