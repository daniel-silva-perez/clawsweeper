## Summary

Keep this PR open. Current main and the latest release still lack dynamic module-load scanner rules, but the PR’s regex-only approach has security-sensitive bypasses and false positives that should block merge until the scanner classifies the full require()/import() argument and has focused tests.

## What This Changes

The branch modifies src/security/skill-scanner.ts by adding critical dynamic-require and dynamic-import LINE_RULES that flag calls whose first argument does not start with a single or double quote.

## Best Possible Solution

Replace the first-character regex rules with scanner logic that classifies the full require()/import() argument, then add targeted tests for variables, concatenation, template literals with and without interpolation, comments around the callee and argument, multiline literals, and ordinary static literals before secops review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Classify the whole module specifier:** `src/security/skill-scanner.ts:178-185`
  - body: The new regexes only inspect the first non-whitespace character after `(`, so `require("./" + name)` and `import("./" + name)` look static and avoid a critical finding even though the resolved module target is runtime-controlled.
  - confidence: 0.92
- **[P1] Handle comments before the call paren:** `src/security/skill-scanner.ts:178-185`
  - body: JavaScript permits comments between the callee and `(`, but these patterns only allow whitespace there. Calls like `require/*x*/(userInput)` bypass the new critical rule while still dynamically loading a module.
  - confidence: 0.9
- **[P2] Skip leading comments before literal specifiers:** `src/security/skill-scanner.ts:185`
  - body: A static call such as `import(/* webpackChunkName: "x" */ "./mod")` starts with `/` after the paren, so the new rule reports a critical dynamic import even though the module specifier is a literal after the comment.
  - confidence: 0.86
- **[P2] Treat constant template literals as static literals:** `src/security/skill-scanner.ts:178-185`
  - body: Because the lookahead excludes only single and double quotes, constant backtick specifiers like `import(`./fixed.js`)` or `require(`./fixed.js`)` are reported as critical even when they contain no interpolation, creating install-blocking false positives.
  - confidence: 0.88
