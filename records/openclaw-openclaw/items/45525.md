## Summary

Keep this PR open. Current main still lacks the requested dynamic require/import detection, but the proposed first-character regex rules are not merge-ready because they leave dynamic-load bypasses and add install-blocking false positives on a security scanner path.

## What This Changes

The PR adds two critical `LINE_RULES` in `src/security/skill-scanner.ts` to flag `require()` and `import()` calls whose first argument does not start with a single or double quote.

## Best Possible Solution

Replace the first-character regex rules with argument-aware scanner logic that classifies the complete `require()` or `import()` specifier, then add focused tests for variables, concatenation, template literals with and without interpolation, comments around the callee and argument, multiline literals, and ordinary static literals before merging.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Classify the whole module specifier:** `src/security/skill-scanner.ts:182-189`
  - body: The new regexes only check the first non-whitespace character after `(`. Calls like `require("./" + name)` and `import("./" + name)` start with a quote, so they skip the critical rule even though the resolved module target is runtime-controlled.
  - confidence: 0.93
- **[P1] Handle comments before the call paren:** `src/security/skill-scanner.ts:182-189`
  - body: JavaScript permits comments between the callee and `(`, but these patterns only allow whitespace there. `require/*x*/(userInput)` and the matching `import/*x*/(userInput)` form bypass the new critical rule while still dynamically loading modules.
  - confidence: 0.9
- **[P2] Skip comments before literal import specifiers:** `src/security/skill-scanner.ts:189`
  - body: A static import such as `import(/* webpackChunkName: "x" */ "./mod")` starts with `/` after the paren, so the new `dynamic-import` rule reports a critical dynamic load even though the module specifier is a literal after the comment.
  - confidence: 0.87
- **[P2] Treat constant template literals as static:** `src/security/skill-scanner.ts:182-189`
  - body: Because the lookahead excludes only single and double quotes, constant backtick specifiers like `import(`./fixed.js`)` or `require(`./fixed.js`)` are reported as critical even when they contain no interpolation, creating install-blocking false positives.
  - confidence: 0.88
