## Summary

Current main still does not implement the PR's intended dynamic require()/import() detection, and the PR remains a focused security-hardening candidate rather than obsolete cleanup. It should stay open, but the regex-only implementation needs tests and fixes for the bypasses and false positives already identified in review comments.

## Best Possible Solution

Keep this PR open as a valid security-hardening candidate, but require a more robust implementation before merge. The best path is to add explicit tests in src/security/skill-scanner.test.ts for variable specifiers, concatenated paths, dynamic and static template literals, inline comments between callee and paren, static literal imports, and multiline/commented import forms, then update src/security/skill-scanner.ts with detection that distinguishes static literals from runtime-resolved specifiers without relying only on the first post-paren character.
