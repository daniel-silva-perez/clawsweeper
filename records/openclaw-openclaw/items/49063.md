## Summary

Keep this PR open. Current main still does not implement the intended Telegram native `channel_post` slash-command path: grammY can invoke `bot.command` for channel posts, but OpenClaw's native built-in and plugin command handlers still read only `ctx.message` and return when it is absent. The auth path is also still group/DM-shaped, and the PR is paired with open auth-semantics issue #49062 plus security-sensitive review comments about command authorization behavior.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to refresh it against current main, make Telegram native command handlers use the correct `ctx.msg`/channel-post message source, preserve `commands.allowFrom` and explicit chat allowlist semantics, resolve or document #49062's sender-scoped channel-post policy, handle inline callback/session routing for channel commands, and keep regression coverage for allowed, denied, `useAccessGroups`, `commands.allowFrom`, `sender_chat`, and `channel_post` cases.
