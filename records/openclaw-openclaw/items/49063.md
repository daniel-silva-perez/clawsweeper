## Summary

Keep this PR open. Current main still does not implement the PR’s intended native Telegram channel-post command behavior: grammY native command handlers are registered before the generic channel_post handler, but the native handlers read only ctx.message and return when it is absent. The PR also changes a command authorization surface and remains paired with open follow-up #49062, so it needs maintainer review rather than cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to refresh it against current main, use grammY ctx.msg for native command message access, preserve explicit chat allowlist and commands.allowFrom semantics, define or document the #49062 sender-scoped channel_post policy, handle sender_chat and interactive callback/session routing, and keep regression tests for allowed, denied, useAccessGroups, commands.allowFrom, and channel_post authorization cases.
