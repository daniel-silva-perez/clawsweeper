## Summary

Keep open. Current main still does not implement the PR's central behavior: Telegram native command handlers read only ctx.message, so channel_post updates do not reach built-in or plugin native command auth/execution. The same-author follow-up issue #49062 is also still open, and the PR discussion contains unresolved authorization/security semantics that should be reviewed before merge.

## Best Possible Solution

Keep this PR open for maintainer review. Refresh it against current main, make built-in and plugin Telegram native command handlers consume the correct channel_post message source, align authorization with explicit chat allow, commands.allowFrom, sender_chat, groupPolicy, and per-chat allowlist semantics, resolve or document #49062, and add regression tests for allowed and denied channel_post commands plus callback/session follow-ups.
