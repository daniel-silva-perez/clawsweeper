## Summary

Keep this PR open. Current main at f256eeba431b5f59e6085b7c11c9c0a309e408b7 still does not implement the PR's intended Telegram native channel_post command path: built-in and plugin native command handlers still read only ctx.message, while channel_post support exists only in the later generic inbound handler. The PR is also paired with open auth-semantics issue #49062 and changes a command execution authorization surface, so it needs maintainer review rather than cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to refresh it against current main, make native Telegram handlers use the correct message source for channel_post, preserve commands.allowFrom and explicit chat allowlist semantics, resolve or document #49062's sender-scoped channel_post policy, handle interactive callback/session routing, and keep regression coverage for allowed, denied, useAccessGroups, commands.allowFrom, sender_chat, and channel_post cases.
