## Summary

Keep this PR open. Current main still does not implement the intended Telegram native channel_post command path, and the PR remains a focused implementation candidate with unresolved authorization/security semantics to review before merge.

## Best Possible Solution

Keep this PR open for maintainer review. Refresh it against current main, make built-in and plugin Telegram native command handlers use the correct channel-post message source, align channel-post authorization with explicit chat allow, commands.allowFrom, sender_chat, groupPolicy, and per-chat allowlist semantics, resolve or document #49062, and add regression coverage for allowed and denied channel_post commands plus callback/session follow-ups.
