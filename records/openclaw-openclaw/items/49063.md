## Summary

Keep open. Current main still does not implement the PR's central behavior: Telegram native command handlers read only `ctx.message`, so `channel_post` updates still do not reach built-in or plugin native command authorization/execution. The same-author follow-up #49062 is also still open, and the PR discussion has unresolved authorization/callback semantics that should be reviewed before merge.

## Best Possible Solution

Keep this PR open for maintainer review. Refresh it against current main, make built-in and plugin Telegram native command handlers consume the correct `channel_post` message source (`ctx.msg`/`ctx.channelPost`), define authorization semantics across explicit chat allow, `commands.allowFrom`, `sender_chat`, `groupPolicy`, and per-chat `allowFrom`, resolve or document #49062, and add regression tests for allowed and denied channel_post commands plus callback/session follow-ups.
