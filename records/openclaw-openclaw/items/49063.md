## Summary

Keep this PR open. Current main still drops Telegram native `channel_post` slash commands before auth/execution because built-in and plugin `bot.command` handlers read only `ctx.message`, while grammY 1.42 detects commands in channel posts and exposes the shared message via `ctx.msg`. The PR also has an open paired auth-semantics issue and unresolved security-sensitive command-authorization review concerns, so it is not safe maintainer cleanup to close it.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to refresh it against current main, make Telegram native built-in and plugin command handlers use the correct `ctx.msg`/channel-post message source, preserve `commands.allowFrom`, explicit chat allowlist, per-channel policy, and `sender_chat` semantics, resolve or document #49062, and add regression coverage for allowed and denied `channel_post` commands plus callback/session follow-ups.
