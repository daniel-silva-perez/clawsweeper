## Summary

Keep this PR open. Current main at 230f7122dd5cd8a37a55e671254bd9be8278f6e8 still does not implement IRCv3 draft/multiline support: IRC outbound text is still flattened, there is no CAP/BATCH negotiation or IRCv3 tag parsing, and tests still assert the old 350-character IRC outbound limit. The PR is a focused implementation candidate for an existing bundled IRC plugin capability, but it needs rebase/review and protocol-length fixes before merge.

## Best Possible Solution

Keep the PR open as the implementation candidate. Rebase it onto current main, integrate the outbound changes through the current IRC outbound-base/client split, add focused tests for CAP negotiation, outgoing BATCH delivery, incoming batch reconstruction, fallback chunking, and sanitizer behavior, then resolve the remaining IRC line-length concerns before merge.
