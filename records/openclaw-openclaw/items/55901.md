## Summary

Keep this PR open. Current main at 6d323ee73640afdf02ed9fa3aded13da23359b98 still does not implement IRCv3 draft/multiline support: outgoing IRC text is flattened and sent as ordinary PRIVMSG chunks, the parser has no IRCv3 tags, and there is no CAP/BATCH negotiation or incoming batch reconstruction. The PR is still a focused candidate for a bundled IRC plugin feature, but it needs rebase, tests, and protocol-length review before merge.

## Best Possible Solution

Keep the PR open as the implementation candidate. Rebase it onto the current IRC outbound-base/client structure, wire draft/multiline through the current adapter design, preserve newline behavior only where safe for message delivery, split long BATCH payload lines to IRC-safe byte lengths, and add focused tests for CAP negotiation, no-CAP fallback, outgoing BATCH delivery, incoming batch reconstruction, blank lines, sanitizer/NickServ behavior, and the 350-character fallback path before merge.
