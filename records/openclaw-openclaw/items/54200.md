## Summary

Keep this open. Current main still ignores a legacy-named `$HOME/.openclaw/moltbot.json` beside a missing or skeletal canonical config, and the existing #54201-related startup wording only improves the symptom rather than detecting or repairing the reported migration gap.

## Best Possible Solution

Keep this open for a targeted config recovery fix. Add startup and/or `openclaw doctor` detection for `$HOME/.openclaw/moltbot.json` when canonical `openclaw.json` is missing, empty, or skeletal; offer a safe doctor repair path when feasible; and update the public migration/troubleshooting docs while preserving the intentional removal of broad legacy runtime config probing.
