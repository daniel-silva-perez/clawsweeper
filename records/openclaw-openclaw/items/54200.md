## Summary

Keep this issue open. Current main still does not discover, warn about, or repair a legacy-named `$HOME/.openclaw/moltbot.json` beside a missing or skeletal canonical config; the newer startup and backup-recovery work mitigates some clobber cases but does not address the reported sibling legacy-config layout.

## Best Possible Solution

Keep this open for a narrow config recovery change: detect `$HOME/.openclaw/moltbot.json` when canonical `openclaw.json` is missing, empty, or skeletal; surface a high-signal startup/doctor warning; provide a safe `openclaw doctor --fix` repair path when feasible; and update the public migration/troubleshooting docs while preserving the intentional removal of broad legacy runtime config probing.
