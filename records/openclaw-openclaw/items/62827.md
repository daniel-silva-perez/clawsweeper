## Summary

Keep open. Current `main` still does not recognize `/activate mention`: the parser only accepts `/activation`, and the command registry/detection path only advertises `/activation`, so a parser-only change may not be enough for group command routing. The issue is also paired with an open PR using closing syntax, so it should remain open until that PR is reviewed, adjusted if needed, and merged or closed.

## What This Changes

Request support for `/activate mention` and `/activate always` as aliases for the existing WhatsApp group `/activation` command.

## Best Possible Solution

Review the open paired PR or an equivalent narrow fix that adds `/activate` to the activation command detection/normalization path, keeps `/activation` working, covers `/activate` parser and control-command detection tests, and updates user-facing docs only if the alias is intended to be documented.

## Reproduction Assessment

Yes. Current `main` has a direct source-level reproduction: the parser regex and command registry both accept `/activation` but not `/activate`, and a read-only regex check confirmed `/activate mention` does not match.

## Solution Assessment

Partly. Supporting an alias is the right narrow direction, but a parser-only patch is probably incomplete unless the command registry/detection path also recognizes `/activate` before WhatsApp group mention gating decides whether to wake the command handler.
