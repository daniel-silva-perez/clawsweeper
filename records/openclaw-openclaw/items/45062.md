## Summary

Keep this PR open. The performance goal is still not fully implemented on current main, and the branch contains useful follow-up work, but the PR head is not safe to merge as-is because it is stale against the current cache-boundary and group-metadata hardening and moves group context to the final system-prompt position.

## What This Changes

This PR reorders `src/agents/system-prompt.ts` for better prefix-cache reuse, splits dynamic runtime fields into a tail line, moves tool/skills/model/context sections, sanitizes context file headers, and updates system-prompt tests.

## Best Possible Solution

The maintainable end state is a rebased prompt-cache ordering that preserves current main's boundary-aware transport, keeps channel/user-sourced metadata in untrusted user context, and carries forward the narrow context-header sanitization and message-hint gating with focused prompt/cache-boundary tests.

## Reproduction Assessment

Not applicable as a performance/refactor PR rather than a user bug. The cache claims can be reproduced by rendering two prompts with changed runtime/context inputs and measuring the first differing byte; source inspection confirms the main-vs-PR structural differences.

## Solution Assessment

No, not as-is. The volatility-ordering direction is plausible, but the branch must be rebased over current main and redesigned so untrusted group metadata is not the final system-prompt content.
