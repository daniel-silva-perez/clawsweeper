## Summary

Keep this PR open. Current main still does not implement the PR’s central CI behavior: `detect-secrets` is configured with a tracked baseline, but the active `security-scm-fast` job only runs `detect-private-key` and `zizmor`. The branch also needs a security-conscious rebase because its workflow addition does not use the trusted pre-commit config pattern now present on main, and some side changes are stale or already absorbed.

## Best Possible Solution

Keep the PR open for maintainer follow-up, but do not merge it as-is. Rebase onto current main, add `detect-secrets` to `security-scm-fast` using `pre-commit run --config "${PRE_COMMIT_CONFIG_PATH:-.pre-commit-config.yaml}" --all-files detect-secrets`, update `docs/gateway/security/index.md` and `docs/ci.md` to match the actual job, and drop the stale zh-CN symlink plus already-absorbed cron cleanup changes.
