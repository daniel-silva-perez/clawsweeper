## Summary

Keep this PR open. Current main still has the core CI gap: `detect-secrets` is configured with a tracked baseline, but `security-scm-fast` does not invoke it, and the gateway security docs still describe stale CI behavior. The PR should be rebased and narrowed because some side changes are stale or already absorbed, and the CI addition should follow the current trusted pre-commit config pattern.

## Best Possible Solution

Keep the PR open for maintainer follow-up, but do not merge it as-is. Rebase onto current main, add `detect-secrets` to `security-scm-fast` with `pre-commit run --config "${PRE_COMMIT_CONFIG_PATH:-.pre-commit-config.yaml}" --all-files detect-secrets`, update `docs/gateway/security/index.md` and `docs/ci.md` to match the actual job, and drop the stale zh-CN symlink plus already-absorbed cron cleanup changes.
