## Summary

Current main still has the reported Telegram gap. The Telegram block reply dispatcher does not pass `humanDelay`, while the core dispatcher only delays block replies when that option is provided. The issue also has open implementation candidate PR #69022, so this should stay open until that PR is reviewed, merged, or closed.

## Best Possible Solution

Keep this issue open. Review and either land or close PR #69022; the intended fix is narrow: expose/import `resolveHumanDelayConfig` through the Telegram runtime seam, pass `humanDelay: resolveHumanDelayConfig(cfg, route.agentId)` into Telegram `dispatcherOptions`, and keep a focused regression test proving the dispatcher receives the configured delay.
