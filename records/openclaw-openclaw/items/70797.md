## Summary

Keep open. Current main still has the reported leak class: bare OpenAI-style `tool_calls` JSON is detected only diagnostically, while the assistant-visible sanitizer and Telegram/direct partial-preview path can still treat it as normal text. The two related fix attempts are closed unmerged, so there is no canonical implemented fix to close against.

## What This Changes

Prevent bare OpenAI-style `tool_calls` JSON emitted during background exec/process flows from appearing in Telegram/direct user-visible replies.

## Best Possible Solution

Add a shared, streaming-aware guard for bare OpenAI-style `tool_calls` JSON at the assistant-visible boundary, reuse it for final text and partial preview delivery, and preserve ordinary JSON plus fenced/inline code examples.

## Reproduction Assessment

Yes for the leak class: the reported bare JSON shape can be traced through current source because the sanitizer returns it unchanged and the partial-preview path can pass the resulting text to `onPartialReply`. The live Telegram/tinyrouter timing remains sketched rather than fully minimized.

## Solution Assessment

Yes: the narrow maintainable fix is to sanitize or withhold internal tool-call payload text before user-visible delivery, not to add provider-specific workarounds or attempt to execute downgraded text as a tool call.
