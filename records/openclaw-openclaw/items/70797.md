## Summary

Current main still appears to allow the reported bare OpenAI-style `tool_calls` JSON to reach user-visible delivery paths. The shared sanitizer exits unless bracketed tool markers or historical-context markers are present, the preview streaming path builds and emits `cleanedText` without a raw `tool_calls` JSON guard, and the current diagnostic path only logs tool-shaped assistant text. The two directly related fix PRs, #71954 and #72011, are closed unmerged, so this should stay open for a focused core fix.

## Best Possible Solution

Keep this open and land a narrow core fix that sanitizes or withholds bare OpenAI-style `tool_calls` JSON before assistant-visible final text and partial preview delivery reach channels. The fix should preserve ordinary JSON and fenced/inline code examples, handle incomplete streamed fragments safely, and add focused regression tests for the shared sanitizer plus the direct/Telegram preview path.
