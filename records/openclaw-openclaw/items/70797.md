## Summary

Current main still appears vulnerable: the shared downgraded-tool-call sanitizer exits before handling a bare OpenAI-style `{"tool_calls":[...]}` payload, and the streaming preview path can pass cleaned reply text to `onPartialReply` without a streaming-aware raw tool-call JSON guard. The two related fix PRs were closed unmerged, so this should remain open.

## Best Possible Solution

Keep this open and land a narrow core fix that sanitizes or withholds bare OpenAI-style `tool_calls` JSON before assistant-visible final text and preview partials reach channel delivery. The fix should preserve ordinary JSON and code examples, cover incomplete streamed fragments, and add regression tests for the Telegram/direct preview path plus shared final visible-text cleanup.
