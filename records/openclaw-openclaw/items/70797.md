## Summary

Keep open. Current main still has the reported leak class: bare `{"tool_calls":[...]}` text is not stripped by the shared assistant-visible sanitizer, and Telegram partial-preview delivery can still forward that text to draft previews. The two directly linked repair PRs are closed unmerged, so there is no active canonical fix for this issue.

## What This Changes

The issue requests preventing raw internal `tool_calls` JSON from appearing in user-visible Telegram/direct replies during background `exec` followed by `process poll` flows.

## Best Possible Solution

Add a shared, streaming-aware assistant-visible guard for bare OpenAI-style `tool_calls` JSON, use it before both final visible delivery and partial preview callbacks, and cover complete, incomplete, inline, fenced, and ordinary JSON cases with focused regressions.

## Reproduction Assessment

Yes. The exact live race is not minimized, but the payload shape has a high-confidence source reproduction through the current sanitizer early exit and `onPartialReply` preview path.

## Solution Assessment

Yes. A narrow sanitizer/preview guard is the best maintainable fix; provider-specific tinyrouter workarounds or trying to execute downgraded text as a tool call would be broader and less reliable.
