## Summary

Keep open. Current main still does not provide the reported Feishu text-command fast path: Feishu receive events enter the inbound debouncer before dispatch, same-key non-debounced work can wait behind an active chain, and Feishu sequential routing still leaves `/status` and `/new` on the normal per-chat lane. The issue is also paired with open PR #43512, which explicitly targets this regression, so maintainers should review, update, land, replace, or close that implementation candidate before closing the issue.

## What This Changes

Feishu text commands such as `/stop`, `/new`, and `/status` should bypass active-run queueing and execute immediately during Feishu agent runs.

## Best Possible Solution

Keep this issue open until maintainers land or replace #43512 with a current-main Feishu fix that detects command-only text before same-key inbound debounce serialization, gives `/stop`, `/new`, and `/status` a true immediate control path, preserves Feishu authorization and mention-gating semantics, and adds regression coverage for commands sent during an active same-chat run.
