## Summary

Current main still does not satisfy the reported Feishu text-command fast path: Feishu messages always enter the inbound debouncer before dispatch, only abort text gets a dedicated Feishu control lane, and current tests still expect `/status` to use the normal per-chat lane. The issue should also stay open because open PR #43512 explicitly targets this issue with closing syntax, so maintainers should review, update, land, replace, or close that implementation candidate first.

## What This Changes

Restore Feishu plain-text command handling so `/stop`, `/new`, and `/status` bypass active same-chat agent-run queues and execute immediately.

## Best Possible Solution

Keep the issue open until maintainers land or replace #43512 with a current-main Feishu fix that detects command-only messages before same-key inbound debounce serialization, gives `/stop`, `/new`, and `/status` a true control fast path, preserves Feishu authorization and mention-gating behavior, and adds regression coverage for commands sent during an active same-chat run.
