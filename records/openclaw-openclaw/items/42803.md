## Summary

Keep open. Current main still routes Feishu text receive events through inbound debounce before command dispatch, so command-only messages can wait behind an active same-key chain. The issue is also paired with open PR #43512 using closing syntax, so it should stay open until that PR is reviewed, replaced, landed, or closed.

## What This Changes

Feishu text commands such as `/stop`, `/new`, and `/status` should bypass Feishu queueing and execute immediately during an active agent run.

## Best Possible Solution

Review and land or replace #43512 with a current-main Feishu fix that detects command-only text before same-key inbound debounce serialization, or otherwise routes `/stop`, `/new`, and `/status` onto an immediate control path while preserving auth, mention handling, dedupe, and normal-message serialization. Add focused regression coverage for commands sent during an active same-chat run.
