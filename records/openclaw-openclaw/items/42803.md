Keep open. Current main still does not provide the reported Feishu text-command fast path before same-key inbound debouncing, and an open PR explicitly targets this regression with closing syntax, so the issue should stay open until that implementation is reviewed, landed, replaced, or closed.

Restore Feishu plain-text control commands such as `/stop`, `/new`, and `/status` so they execute immediately during an active Feishu agent run instead of waiting behind the active chat queue.
