## Summary

Keep open. Current main still matches the reported failure path: the DeepSeek provider accepts V4 thinking at the stream/payload layer but does not expose a provider thinking profile, so `/think xhigh` and `/think max` are rejected before request construction. The open PR #73008 is the correct review point because it references this issue with closing syntax and proposes the focused provider-local fix.

## What This Changes

The issue requests a DeepSeek provider `resolveThinkingProfile` for `deepseek-v4-flash` and `deepseek-v4-pro` so `/think xhigh` and `/think max` are accepted and mapped by the existing V4 stream wrapper.

## Best Possible Solution

Review and land PR #73008, or an equivalent DeepSeek-local patch, so V4 Flash/Pro expose `off|minimal|low|medium|high|xhigh|max` through the provider profile while non-V4 DeepSeek models remain on their existing profile.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current main has no DeepSeek provider `resolveThinkingProfile`, and the resolver therefore returns the base level list before the stream wrapper can map `xhigh` or `max`.

## Solution Assessment

Yes. A provider-local `resolveThinkingProfile` with focused regression coverage is the narrowest maintainable fix; changing the core resolver is unnecessary for this provider-specific model policy.
