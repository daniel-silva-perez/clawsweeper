## Summary

Current main still matches the reported ACP configured-binding collision path: Discord parent-channel fallback materializes the parent conversation identity, the ACP configured session key hashes only that conversation id, and an open same-author PR (#64322) is already paired with this issue. This should not be closed by cleanup before that PR is reviewed, landed, replaced, or closed.

## Best Possible Solution

Keep this issue open and review, land, replace, or close #64322. The fix should preserve Discord parent-channel binding inheritance while materializing inherited thread ACP bindings with distinct persistent session keys, keep direct parent-channel and direct thread bindings working, and add regression coverage proving two Discord thread ids under one configured parent resolve to different ACP target session keys.
