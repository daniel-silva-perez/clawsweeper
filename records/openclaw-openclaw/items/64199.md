## Summary

Current main still has the reported configured ACP session-key collision path for inherited Discord thread routing, and the paired open fix PR #64322 remains unmerged. This should stay open until that PR is reviewed, landed, replaced, or closed.

## What This Changes

The issue reports that persistent ACP configured bindings for Discord parent-channel routes key sessions by the parent channel instead of the active thread, causing sibling threads to share one Claude Code process.

## Best Possible Solution

Keep this open until #64322 or an equivalent fix lands. The end state should give inherited Discord parent-channel ACP bindings distinct persistent configured ACP session keys per runtime thread, preserve direct parent-channel and direct thread bindings, and add regression coverage showing two thread ids under one parent produce different targetSessionKey values.
