## Summary

Current main still has the reported collision path: inherited Discord thread matches return the parent channel as the configured ACP conversation, and the persistent ACP key is computed from that conversation id only. The paired fix PR #64322 is still open and unmerged, so this issue should stay open for maintainer review.

## What This Changes

Fix configured Discord ACP bindings so persistent ACP session keys are scoped per thread instead of shared across sibling threads under the same parent channel.

## Best Possible Solution

Preserve parent-channel route inheritance while materializing inherited Discord thread ACP bindings with the live thread conversation id, keep direct parent-channel and direct thread bindings working, and add regression coverage proving two thread ids under one parent produce different configured ACP targetSessionKey values.
