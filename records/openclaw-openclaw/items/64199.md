## Summary

Current main still matches the reported configured ACP session-key collision for inherited Discord thread routing, and the paired same-author fix PR #64322 remains open/unmerged. This should not be closed until that PR or an equivalent fix lands.

## What This Changes

The issue reports that persistent ACP configured bindings for Discord parent-channel routes use the parent channel id in the session key, causing sibling Discord threads to share one Claude Code process.

## Best Possible Solution

Keep this open until #64322 or an equivalent fix lands. The end state should preserve parent-channel route inheritance while materializing inherited Discord thread ACP bindings with thread-scoped persistent session keys, plus regression coverage proving two thread ids under one parent produce different targetSessionKey values.
