## Summary

Keep open. Current main still lacks this PR's central diagnostics: `/status` has no bootstrap file health section, edit mismatch recovery does not mention bootstrap truncation, and the injected bootstrap warning does not include edit-failure guidance. The PR is draft and needs rebase/test work, but it is not obsolete or superseded by current main.

## What This Changes

The PR branch adds edit-failure guidance to bootstrap truncation warnings, adds workspace bootstrap file health output to status rendering, and appends a large-file truncation hint to edit mismatch errors.

## Best Possible Solution

Adapt the idea to current main by rendering any status addition from `src/status/status-message.ts`, deriving health thresholds from `SessionSystemPromptReport` and resolved config values, adding focused regression coverage for the three user-visible surfaces, and keeping `/context list` as the detailed per-file breakdown if maintainers prefer `/status` to stay concise.
