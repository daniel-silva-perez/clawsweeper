## Summary

Keep this draft PR open: the underlying diagnostics gap still exists on current main, but the branch is stale against the current status rendering path and hardcodes an obsolete bootstrap limit, so it needs maintainer review and a rebase rather than closure.

## What This Changes

The PR adds edit-failure guidance to bootstrap truncation warnings, adds workspace bootstrap file health output to `/status`, and appends a large-file truncation note to edit mismatch errors.

## Best Possible Solution

Keep the PR as useful source work, but adapt it to current main by rendering any status addition from `src/status/status-message.ts`, deriving thresholds from `SessionSystemPromptReport` and resolved config values, and adding focused coverage for bootstrap warning text, status output, and edit mismatch diagnostics.

## Reproduction Assessment

Yes. A workspace bootstrap file over the current 12,000-character default reproduces the remaining visibility gap: `/context` can report truncation, while current `/status` does not show per-file health and edit mismatch errors only include a current-content snippet.

## Solution Assessment

No. The requested direction is reasonable, but this branch is not the best current implementation because it edits the old status builder location and hardcodes 20,000 chars instead of using the current report/config-backed bootstrap limits.
