## Summary

Keep this PR open: current main does not yet implement the requested bootstrap truncation diagnostics, and the branch is a useful but draft/stale implementation that needs maintainer review before it can land. The remaining work is not a close reason because the intended behavior is still missing from main, and the linked #45415 issue remains open as the broader user-visible size-warning request.

## What This Changes

The PR branch adds edit-failure guidance to bootstrap truncation warnings, adds workspace bootstrap file health output to `/status`, and appends a large-file truncation hint to edit mismatch errors.

## Best Possible Solution

Rework this on current main by adding edit-failure recovery guidance to the shared bootstrap warning path, implementing any `/status` summary in `src/status/status-message.ts` from `SessionSystemPromptReport` plus resolved config limits, and adding focused regression coverage for bootstrap warning text, edit mismatch diagnostics, and whichever status/context visibility maintainers choose.
