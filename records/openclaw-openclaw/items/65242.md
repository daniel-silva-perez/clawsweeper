## Summary

Keep this PR open for maintainer review. Current main still lacks the proposed completion-delivery gate and ACP child-session silent-wake behavior, while the PR branch has blocking completion ownership, cleanup, shadow-mode, and Discord thread-binding findings plus overlap with open PR #64416.

## What This Changes

The PR adds an env-flagged completion-delivery gate across task-registry, subagent announce, and farewell paths, adds ACP silent-wake policy/tests, and changes ACP requester-origin plus Discord child-thread binding behavior.

## Best Possible Solution

Land a revised, narrower completion ownership design that coordinates task-registry banner/session fallback, subagent announce, and farewell delivery after actual delivery outcomes are known, preserves cleanup/retry behavior, keeps shadow mode observational, and splits or separately validates Discord/requester-origin routing changes while reconciling #64416.

## Reproduction Assessment

Yes. The current-main behavior is provable from source and tests that still deliver ACP child completions as visible task banners, and the branch regressions are directly traceable from the supplied PR diff and review comments. I did not run tests because this was a read-only checkout review.

## Solution Assessment

No. The PR is not the best fix as-is because it claims ownership before actual delivery, can bypass cleanup, changes behavior in shadow mode, and bundles unrelated Discord/requester-origin routing changes into the completion gate rollout.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Claim announce completion only after successful delivery:** `src/agents/subagent-announce.ts:231-235`
  - body: The new gate claim is taken before `runSubagentAnnounceFlow` has waited for output or delivered anything. This function can still return `false` later for deferred or failed announce paths, but the retained claim lets fallback paths treat the run as already delivered, leaving the requester with no completion message.
  - confidence: 0.9
- **[P1] Keep the dedupe short-circuit inside cleanup flow:** `src/agents/subagent-announce.ts:236-238`
  - body: When another path already claimed the completion, this `return true` exits before the function enters its `try`/`finally`. That skips cleanup such as label patching and `sessions.delete`, while callers interpret the result as a successful announce and do not retry.
  - confidence: 0.88
- **[P2] Keep shadow mode observational for farewells:** `src/agents/subagent-registry-completion.ts:82-83`
  - body: `isCompletionClaimed` returns true in shadow mode because claims are stored, so this branch can flip `sendFarewell` to false even though shadow rollout is supposed to log would-be blocks without changing user-visible behavior.
  - confidence: 0.86
- **[P1] Preserve parent-channel lookup for child bindings:** `extensions/discord/src/monitor/thread-bindings.manager.ts:641`
  - body: For child placement, `conversationId` can be the current thread when `parentConversationId` is absent. Assigning it directly to `channelId` makes `bindTarget` try to create a child thread under a thread, which Discord rejects; keep the existing thread-to-parent resolution fallback.
  - confidence: 0.84
