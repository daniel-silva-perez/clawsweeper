## Summary

Keep this PR open. Current main still lacks the proposed completion-delivery gate and still has ACP task paths that can deliver visible `Background task done` output, while this unmerged PR has unresolved P1 review findings and overlaps with open PR #64416 rather than being superseded by a merged fix.

## What This Changes

The PR adds an env-flagged `CompletionDeliveryGate`, wires it into task-registry, announce, and farewell completion paths, adds ACP silent-wake policy/tests, and also changes ACP requester-origin and Discord child-thread binding behavior.

## Best Possible Solution

Land a revised, narrower completion ownership design that coordinates task-registry banner/session fallback, subagent announce, and farewell delivery without claiming ownership before actual delivery, preserves cleanup/retry behavior, keeps shadow mode observational, and splits or separately validates Discord/requester-origin routing changes while reconciling the overlap with #64416.
