## Summary

Keep open. Current main has strong in-memory tool_use/tool_result repair before provider replay, and it now has a pre-load session-file repair hook, but that on-disk repair does not run the pairing repair or rewrite orphaned/displaced toolResult entries. That is the durable restart self-healing this issue is specifically asking for.

## Best Possible Solution

Keep this issue open for a focused core fix. Extend the pre-SessionManager.open repair path, or add a locked load-repair pass, to apply the existing tool_use/tool_result pairing repair to persisted message entries and rewrite the session file with a backup when orphaned, duplicated, displaced, or missing tool results are found. Add session-file-repair regression coverage for those cases and preserve the current aborted/error-turn safety policy.
