## Summary

Current main has newer incomplete-turn defenses, but they do not fully implement the reported Anthropic-compatible tool_result case. The code can retry or surface an explicit error for some empty replies, and it has a zero-token Claude-specific path, but it still does not synthesize a successful user-visible reply from a successful toolResult after an empty final assistant message. The related #62128 thread is ACP/session-new specific and closed, so it does not supersede this Anthropic Messages/tool-result report.

## Best Possible Solution

Keep this issue open and add a targeted regression around a custom provider using api=anthropic-messages: assistant tool_use, successful non-error toolResult with text/details, then final assistant content: [] and stopReason: stop with positive usage. The fix should either preserve a real final assistant reply from the provider path or synthesize a safe user-visible result from the already-persisted successful toolResult without rerunning mutating tools.
