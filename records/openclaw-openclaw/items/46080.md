## Summary

Keep this open. Current main has generic empty-turn safety, but it still does not implement the reported successful custom `anthropic-messages` tool_result follow-up: a final `stop` assistant turn with empty content and nonzero usage is not retried for custom Anthropic-compatible providers, and the terminal path does not synthesize a successful user-visible reply from successful tool results.

## Best Possible Solution

Keep this issue open and add a targeted embedded-runner fix for `anthropic-messages` tool-result follow-ups. Replay-safe successful tool calls that end with `stop` and empty assistant content should get one visible-answer continuation; replay-unsafe side-effectful successful tool runs should not be blindly retried and should receive a sanitized user-visible fallback instead of a silent turn or generic error. Add regression coverage for both read-like and mutating successful tool-result cases.
