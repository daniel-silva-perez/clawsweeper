## Summary

Keep this PR open. Current main still does not apply configured-provider inference to selected bare `modelOverride` values in `resolveSessionModelRef()`, but the PR patch is not merge-ready because its local normalizer can regress explicit wrapper-provider routing and misses some non-fresh session/default-provider cases already called out in review comments.

## What This Changes

The PR adds a `resolveSessionModelRef()` normalization helper and a new gateway regression test file so bare canonical model overrides can infer a unique configured provider.

## Best Possible Solution

Move the selected-session bare-model inference into the shared model-selection resolver path, preserving explicit `providerOverride`, wrapper providers with slash-containing model ids, stored runtime semantics, ambiguous configured matches, and non-default-provider setups. Keep focused regression coverage in `src/gateway/session-utils.test.ts` or a colocated gateway resolver test.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: apply the PR normalizer to a session with `providerOverride: "openrouter"` and `modelOverride: "anthropic/claude-haiku-4.5"`, or to a session with a non-OpenAI default provider and bare `modelOverride: "gpt-5.4"`; the changed branches return the wrong provider.

## Solution Assessment

No. The PR's goal is good, but a local post-processing normalizer is not the best fix because it duplicates and partially bypasses existing model-selection semantics; extending the shared selected-model resolver is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Preserve explicit providers before parsing slash models:** `src/gateway/session-utils.ts:1049-1054`
  - body: `normalizedModel.includes("/")` is handled before checking `entry.providerOverride`, so a wrapper selection such as `providerOverride: "openrouter"` with `modelOverride: "anthropic/claude-haiku-4.5"` is reparsed as provider `anthropic`. That regresses the current wrapper-provider contract and routes the session through the wrong provider.
  - confidence: 0.91
- **[P2] Do not let stored runtime providers suppress inference:** `src/gateway/session-utils.ts:1065-1078`
  - body: When an existing session has `entry.modelProvider`, this branch returns that runtime provider before the inferred configured provider can be used. Any session that has already run once can therefore keep routing a bare selected override through stale/default runtime metadata instead of the unique configured provider this PR is trying to select.
  - confidence: 0.86
- **[P2] Infer from configured models for any fallback provider:** `src/gateway/session-utils.ts:1077-1081`
  - body: The new inference only applies when `params.ref.provider` is empty or `DEFAULT_PROVIDER`. If an agent default is `anthropic` and `gpt-5.4` uniquely belongs to `openai-codex` in config, the resolver leaves the session on `anthropic/gpt-5.4`, so the feature fails for non-OpenAI defaults.
  - confidence: 0.84
