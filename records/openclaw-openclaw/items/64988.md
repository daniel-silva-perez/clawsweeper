## Summary

Keep this PR open. Current main still leaves the selected-session bare override path routed through the default provider, so the underlying feature is not fully obsolete, but the proposed patch introduces provider-precedence regressions that should block merge until revised. The diff is limited to gateway model-selection code and a colocated test, so I did not find a security or supply-chain concern.

## What This Changes

The PR adds a selected-session model normalization step and gateway regression tests so bare canonical model overrides can infer a unique configured provider.

## Best Possible Solution

Integrate the feature through the shared model-selection resolver so bare selected-session overrides infer a unique configured provider, while preserving explicit provider overrides, wrapper providers with slash-containing model IDs, existing-session behavior, and non-default-provider setups. Keep the focused gateway regression tests and align them with the current `resolveAllowedModelRef` contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve providerOverride before parsing slash model overrides:** `src/gateway/session-utils.ts:1047-1054`
  - body: `normalizeResolvedSessionModelRef` reparses any model containing `/` before it checks `entry.providerOverride`. A selected wrapper provider such as `openrouter` with `modelOverride: "anthropic/claude-haiku-4.5"` would now route as provider `anthropic`, contradicting the existing wrapper-provider preservation contract.
  - confidence: 0.94
- **[P2] Infer configured providers for non-default provider sessions:** `src/gateway/session-utils.ts:1083-1088`
  - body: The inference branch only runs when `params.ref.provider` is empty or equal to `DEFAULT_PROVIDER`. If the agent default provider is `anthropic` and the bare override uniquely maps to `openai-codex`, the resolver still returns `anthropic/gpt-5.4`, so the new feature fails outside OpenAI-default setups.
  - confidence: 0.88
- **[P2] Do not let stored runtime providers suppress a new bare override:** `src/gateway/session-utils.ts:1073-1078`
  - body: When `entry.modelProvider` is set, the new `runtimeProvider` branch returns that provider before the inferred provider can be used. Existing sessions commonly have stored runtime metadata, so a new bare override can remain pinned to the stale runtime provider instead of the unique configured provider.
  - confidence: 0.82
