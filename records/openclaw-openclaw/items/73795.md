## Summary

Keep open. Current main still lacks a supported suppression path for `streaming.preview.toolProgress` on WhatsApp and LINE, while the shared setting exists for preview-streaming channels and there is no global `agents.defaults.streaming.preview.toolProgress` fallback. The Slack part of the original report appears different on current main because Slack does expose the setting, and open PR #73054 is related to Slack defaults rather than the remaining cross-channel config gap.

## Best Possible Solution

Keep this issue open. The conservative fix is to add a supported suppression path for channels that do not use preview streaming, either by extending the shared channel config contract so WhatsApp/LINE can preserve `streaming.preview.toolProgress: false` without requiring `streaming.mode`, or by adding `agents.defaults.streaming.preview.toolProgress` with clear per-channel override precedence. Update strict schemas, generated metadata/docs, and add regression tests for config round-tripping plus outbound tool/progress behavior.
