## Summary

Keep open. Current main still does not provide a supported `channels.whatsapp.streaming.preview.toolProgress` or LINE equivalent, and there is no global `agents.defaults.streaming.preview.toolProgress` fallback. Slack, Telegram, Discord, and Matrix have explicit streaming preview schemas/docs, but WhatsApp and LINE schemas remain strict and omit `streaming`, so the reported cross-channel config gap is still real rather than implemented or duplicated by the related reports.

## Best Possible Solution

Add a supported suppression path for tool/progress previews on all relevant channel types. The conservative implementation would extend the shared channel config contract so non-preview or buffered channels can preserve `streaming.preview.toolProgress: false` without requiring `streaming.mode`, update bundled WhatsApp and LINE schemas/generated metadata/docs, and add regression coverage for config validation/write round-tripping plus outbound behavior. If maintainers prefer a global switch, implement `agents.defaults.streaming.preview.toolProgress` with clear per-channel override precedence.
