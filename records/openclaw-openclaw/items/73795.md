## Summary

Keep open. Current main still has the reported cross-channel config gap: `streaming.preview.toolProgress` exists as a shared preview-streaming concept, but WhatsApp and LINE still do not expose a supported `streaming` config object, and there is no global `agents.defaults.streaming.preview.toolProgress` fallback. The earlier ClawSweeper comment remains accurate against current main.

## Best Possible Solution

Keep this issue open and add a supported suppression path for tool/progress previews on all relevant channels. The conservative fix is to extend the shared channel config contract so non-preview or buffered channels can preserve `streaming.preview.toolProgress: false` without requiring `streaming.mode`, update WhatsApp/LINE schemas and generated metadata/docs, and add regression coverage for config round-tripping plus outbound tool/progress behavior. If maintainers prefer a global switch, add `agents.defaults.streaming.preview.toolProgress` with explicit per-channel override precedence.

## Risks / Open Questions

- Potential privacy/security exposure remains plausible because affected chat channels still lack a supported way to suppress tool/progress narration through config.
- The fix crosses schemas, generated metadata, docs, and runtime delivery behavior; it should stay backward-compatible for third-party plugins and avoid enabling preview streaming on channels that intentionally buffer or send final-only replies.
- Slack appears to have the shared preview schema on current main, so the remaining issue should be scoped carefully to unsupported channels such as WhatsApp and LINE rather than treating every named channel identically.
