## Summary

Keep this PR open. Current main at 68561a8c still does not implement the PR's intended Feishu routing behavior: thread-id-only group messages still force thread replies even when `replyInThread` is disabled, while the overlapping open PR #39507 proposes different semantics. The PR remains potentially useful, but needs maintainer product judgment and a current-main refresh.

## Best Possible Solution

Keep this PR open for maintainer review or consolidate it only after deciding the intended Feishu `root_id`/`thread_id` semantics. A current-main solution should update `extensions/feishu/src/bot-content.ts` and `extensions/feishu/src/reply-dispatcher.ts` together, decide whether `root_id` bypasses disabled config, scope any root-id forcing to the intended group surfaces, preserve or intentionally change p2p quote behavior, and add regression coverage for thread-id-only disabled groups, root-id groups, topic sender routing, and p2p quote replies.
