## Summary

Keep this PR open. Current main does not implement the proposed `neverReply` config or equivalent no-LLM group suppression across channels, and the existing alternatives are materially narrower: `sendPolicy: deny` suppresses outbound delivery but still runs the agent, silent ingest is only present on Telegram/Signal mention-skip paths, and `inbound_claim` is still not broadcast for unbound conversations. The discussion also contains useful evidence for a plugin direction, including flowolforg's linked `https://github.com/flowolforg/openclaw-listen-only`, but current main still lacks the generic broadcast hook gap that would make that a complete replacement for this PR.

## Best Possible Solution

Keep this PR open for maintainer product and architecture review. The best path is either to narrow and review the core `neverReply` implementation with strong cross-channel tests, or first land a generic plugin/API seam that lets the linked listen-only plugin cover the use case externally; after such a replacement ships on main, this PR could be closed as superseded or ClawHub-scoped.
