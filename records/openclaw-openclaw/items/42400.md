## Summary

Keep this PR open. Current main still lacks the proposed `neverReply` config and no equivalent no-LLM/no-outbound group observation path exists. The external listen-only plugin idea is useful, but current main still does not broadcast `inbound_claim` to unbound plugin conversations, and this PR also carries security-sensitive plugin loader/path-trust changes that need maintainer review or splitting.

## Best Possible Solution

Keep this PR open for maintainer architecture and security review. The best path is either a focused core `neverReply` implementation with schema/docs/tests and consistent cross-channel pending-history behavior, or an explicit maintainer decision to add the missing plugin hook/API seam and move listen-only behavior into a plugin such as the linked external implementation. If this PR continues, split the plugin-loader/path-trust changes from the channel behavior.
