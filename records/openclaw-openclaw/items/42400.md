## Summary

Keep this PR open. Current main still does not ship the requested `neverReply`/passive group observation config, and the current plugin hook surface is relevant but does not clearly supersede this first-class cross-channel config proposal without maintainer product/architecture judgment. The PR also needs a normal security review because the provided diff includes unrelated plugin-loading, installer, and trash-root changes alongside the channel feature.

## Best Possible Solution

Keep this PR open for maintainer architecture and security review. The best path is either a focused core implementation of passive group observation with schemas/docs/tests across the supported channels, or an explicit maintainer decision to supersede it with a documented ClawHub/plugin implementation such as the linked listen-only plugin. If this PR continues, rebase it onto current main and split away unrelated installer, plugin-loader, and trash-policy changes first.
