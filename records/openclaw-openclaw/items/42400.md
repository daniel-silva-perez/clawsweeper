## Summary

Keep this PR open. Current main still does not implement `neverReply` or an equivalent passive group-observation mode that records group context while guaranteeing no agent/model run and no outbound reply. The linked plugin direction is relevant, but main still does not broadcast unbound `inbound_claim` hooks, so it does not yet supersede this PR.

## Best Possible Solution

Keep this PR open for maintainer product and architecture review. The best path is either to finish a generic, well-tested passive group observation design across the affected channel owners, or first land the missing plugin hook/API behavior needed by the linked listen-only plugin and then explicitly supersede this PR with that plugin direction. The shipped design should clearly separate pending context accumulation, LLM invocation, automatic final-reply suppression, explicit message-tool sends, and transport-level outbound blocking.
