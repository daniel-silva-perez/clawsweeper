## Summary

Keep this PR open. Current main still does not implement the requested `neverReply` passive group-observation config, and the shipped alternatives only partially overlap: mention-gated skips can store pending context, and `visibleReplies` suppresses normal final room output, but it still runs the agent and can still use the message tool. The discussion also surfaced a useful external listen-only plugin path, but current main still does not broadcast `inbound_claim` hooks for ordinary core-owned conversations, so that is not a shipped replacement for the PR’s stated account/channel/default config behavior.

## Best Possible Solution

Keep this PR open for maintainer architecture and security review. The best path is either a focused core implementation of passive group observation with schemas, docs, and tests across supported channels, or an explicit maintainer decision to supersede it with a documented plugin/ClawHub approach such as the linked listen-only plugin. If this branch continues, split out the unrelated plugin-loader, doctor, and installer-adjacent changes first.
