## Summary

Keep open. Current main still lacks the proposed `neverReply` config/resolver and has no equivalent no-LLM/no-outbound group observation mode. Existing `visibleReplies`/`sendPolicy` paths suppress delivery but still run the agent, and the plugin-hook alternative discussed in the PR is not a complete replacement because current dispatch still does not broadcast `inbound_claim` for unbound conversations. The latest provided PR file list no longer shows the earlier plugin loader/path-trust mix-in, but the change remains broad and safety-sensitive enough for maintainer architecture/security review rather than conservative cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer architecture and security review. The best path is either a focused core `neverReply` implementation with schema/docs/tests and consistent per-channel pending-history behavior, or an explicit maintainer decision to add the missing plugin hook/API seam and move listen-only behavior into a plugin such as `https://github.com/flowolforg/openclaw-listen-only`.
