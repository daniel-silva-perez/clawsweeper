## Summary

Kept this PR open. Current `main` still does not implement the PR's requested `neverReply`/passive group observation config, and the existing plugin/hook alternative is not a completed replacement because the inbound-claim broadcast gap is still visible in current dispatch tests/source. The PR also remains an active implementation candidate with a recent force-pushed head, but its diff includes security-sensitive plugin-loader/test-environment changes outside the stated channel feature, so it needs maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer architecture and security review. The best path is either a focused core implementation of passive group observation with schemas, docs, and tests across supported group-capable channels, or an explicit maintainer decision to supersede it with a documented plugin/ClawHub path after the missing inbound-claim broadcast seam lands. If the PR continues, split or remove the unrelated plugin-loader, doctor, and bundled-root test changes so the channel behavior can be reviewed on its own.
