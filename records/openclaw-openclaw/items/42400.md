## Summary

Keep this PR open. Current main does not implement the proposed `neverReply` config/resolver or an equivalent zero-LLM, zero-reply passive group observation mode. Main has related pieces, such as mention-gated pending history and default group final-reply suppression through `message_tool`, but those still allow mentioned group turns to reach the agent and explicit message-tool sends. The plugin direction raised in the discussion is useful, especially the linked `openclaw-listen-only` approach, but current main still does not broadcast `inbound_claim` to unbound plugins, so that is not a complete replacement for this PR today.

## Best Possible Solution

Keep the PR open for maintainer product judgment. The clean path is either to finish a generic, well-tested suppression/ingest design across the affected channel owners, or first land the missing plugin API needed for a ClawHub listen-only plugin and then supersede this PR with that plugin direction. In either path, the shipped behavior must clearly separate pending context accumulation, LLM invocation, and transport-level outbound blocking.
