## Summary

Keep this PR open. Current main still does not implement `neverReply` or an equivalent passive group observation mode that records group context while guaranteeing no LLM invocation and no outbound reply. Main has adjacent pieces, including mention-skip pending history, group visible-reply suppression, `sendPolicy` delivery suppression, and plugin inbound-claim hooks, but those do not fully satisfy the PR’s stated behavior. The linked `openclaw-listen-only` plugin direction is useful, but current dispatch still does not broadcast generic `inbound_claim` hooks for unbound conversations, so it does not clearly supersede this channel-level implementation.

## Best Possible Solution

Keep the PR open for maintainer product and architecture review. The best path is either to finish a generic, well-tested passive group observation design across the affected channel owners, or first land the missing plugin API/broadcast behavior needed by the linked listen-only plugin and then explicitly supersede this PR with that plugin direction. In either approach, the shipped behavior should clearly separate pending context accumulation, LLM invocation, automatic final-reply suppression, explicit message-tool sends, and transport-level outbound blocking.
