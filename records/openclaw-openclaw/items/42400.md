## Summary

Keep this PR open. Current main still does not implement the proposed `neverReply` config/resolver or an equivalent zero-LLM passive group observation mode across group-capable channels. Main has adjacent mention-skip pending history and default group final-reply suppression, and the linked `openclaw-listen-only` plugin direction is useful, but the current dispatch path still does not broadcast unbound `inbound_claim` handling in a way that would fully supersede this PR.

## Best Possible Solution

Keep the PR open for maintainer product and architecture judgment. The best path is either to finish a generic, well-tested suppression/ingest design across the affected channel owners, or first land the missing plugin API for a ClawHub listen-only plugin and then explicitly supersede this PR with that plugin direction. In either path, the shipped behavior should clearly separate pending context accumulation, LLM invocation, normal final-reply suppression, explicit message-tool sends, and transport-level outbound blocking.
