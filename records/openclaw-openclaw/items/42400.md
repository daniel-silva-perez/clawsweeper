## Summary

Keep open. Current main still does not implement the proposed `neverReply` config/resolver or an equivalent no-LLM/no-outbound group observation mode. The PR remains a plausible implementation path, but it is broad, channel-spanning, and safety-sensitive, so it needs maintainer architecture/security review rather than conservative cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer architecture and security review. The best path is either a focused core `neverReply` implementation with schema/docs/tests and consistent per-channel pending-history semantics, or an explicit maintainer decision to add the missing plugin hook/API seam and move listen-only behavior into a plugin such as `https://github.com/flowolforg/openclaw-listen-only`.
