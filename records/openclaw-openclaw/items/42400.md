## Summary

Keep this PR open. Current main still does not implement the requested `neverReply` / passive group observation config, and the existing plugin hook surface does not clearly supersede it because normal core-owned conversations still do not broadcast `inbound_claim`. The linked listen-only plugin idea from the discussion is useful context, but maintainers still need to choose a focused core config path or explicitly bless/extend the plugin path.

## Best Possible Solution

Keep this PR open for maintainer architecture and security review. The best path is either a focused core implementation of passive group observation with schemas, docs, and tests across the supported channels, or an explicit maintainer decision to supersede it with a documented plugin/ClawHub path such as the linked listen-only plugin after the necessary hook surface is available. If this PR continues, rebase onto current main and split unrelated installer, plugin-loader, and trash-policy changes first.
