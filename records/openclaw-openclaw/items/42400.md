Keep this PR open. Current main still does not implement the proposed `neverReply` passive group-observation config, and the related plugin/listen-only path is not a completed replacement because current dispatch only runs `inbound_claim` for plugin-bound conversations. ## Best Possible Solution

Keep this PR open for maintainer architecture and security review. If the PR continues, split the unrelated plugin-loader and bundled-root trust changes from the channel behavior.
