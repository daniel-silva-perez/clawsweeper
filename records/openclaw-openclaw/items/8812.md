## Summary

Keep open. Current main still renders URL-bearing tool output as escaped/plain text in the Control UI tool-card paths and as plain SwiftUI Text in native WebChat. The shared markdown renderer linkifies normal chat text, but tool-card output bypasses that renderer. Closed PR #8880 remains useful prior art, but it was not merged or shipped.

## Best Possible Solution

Keep this issue open. Implement a shared safe URL-aware renderer for Control UI tool output that preserves JSON/plain-text formatting, escapes arbitrary text before link creation, supports only safe schemes, handles trailing punctuation/whitespace, and is covered by focused tests across expanded output, raw details, collapsed previews, inline short cards, grouped tool-message paths, and sidebar/raw views. If native WebChat remains in scope, add equivalent SwiftUI URL rendering with AttributedString or Link handling rather than plain Text. Treat PR #8880 as useful prior art, not shipped behavior.

## Risks / Open Questions

- A naive Control UI fix that uses HTML injection without escaping first would turn arbitrary tool output into an XSS surface; the renderer should escape text and only produce safe http/https anchors.
- The Control UI has several separate output paths: expanded blocks, raw details, collapsed previews, short inline cards, grouped tool messages, and sidebar/raw views. Fixing only one path would leave the user-visible issue partially unresolved.
