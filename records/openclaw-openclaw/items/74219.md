## Summary

Keep this PR open. The linked Feishu regression is real and current main still lacks the Feishu SDK facade, but this PR needs maintainer reconciliation before merge because it restores helper facade files while only publishing the base `feishu` subpath and it does not align the public SDK docs/API/test surface.

## What This Changes

The PR adds Feishu plugin SDK compatibility facade source files, registers the base `feishu` SDK entrypoint/export, and re-exports legacy Feishu helpers for older `@openclaw/feishu` packages.

## Best Possible Solution

Land one maintainer-approved Feishu compatibility PR that restores `openclaw/plugin-sdk/feishu` for older published Feishu packages, explicitly decides whether the two helper subpaths remain public, and keeps the entrypoint registry, package exports, API baseline, docs, and regression tests aligned.

## Reproduction Assessment

Yes. Current main lacks the Feishu SDK package export/entrypoint, and the published `@openclaw/feishu@2026.3.13` tarball imports `openclaw/plugin-sdk/feishu` and calls `createScopedPairingAccess`, matching the linked runtime log.

## Solution Assessment

No, not as-is. The compatibility direction is plausible, but this PR partially restores the old Feishu facade surface and needs maintainer reconciliation with #74223 plus SDK docs/API/test alignment before it is the safest fix.
