## Summary

Keep open. Current main still lacks the Feishu SDK facade needed by the linked regression, so the PR is not obsolete; however, this branch needs maintainer reconciliation before merge because it restores a broad compatibility surface, omits docs/API-baseline/regression-test updates, and creates helper facade files that are not wired as public subpath exports.

## What This Changes

This PR adds Feishu plugin-SDK compatibility facade source files, wires only `openclaw/plugin-sdk/feishu` into the package export map and generated SDK entrypoint list, and classifies Feishu as a supported bundled facade.

## Best Possible Solution

Land one narrow compatibility fix that restores the legacy `openclaw/plugin-sdk/feishu` import required by published Feishu packages, aligns package exports, entrypoint registry, API baseline, docs, and subpath regression coverage, and explicitly decides whether the setup/conversation helper facades are public compatibility subpaths or should be removed from the patch.
