## Summary

Keep this issue open. Current main still treats Synology Chat inbound webhooks as text-first/text-only: the payload type, parser, delivery object, inbound context, docs, and tests do not carry images, CSVs, PDFs, filenames, MIME types, or media paths into the agent turn. Outbound URL-based file delivery exists, but it does not satisfy the requested inbound attachment support.

## Best Possible Solution

Keep the issue open and implement inbound attachment handling in the bundled Synology Chat plugin. Confirm Synology Chat's attachment webhook/API shape, extend the typed payload/parser, safely download or stage accepted images, CSVs, and PDFs through existing media-safe helpers, expose MediaPath(s), MediaUrl(s), MediaType(s), and filename metadata in the inbound context, update https://docs.openclaw.ai/channels/synology-chat, and add focused webhook plus inbound-context regression tests.
