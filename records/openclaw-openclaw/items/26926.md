## Summary

Keep this issue open. Current main at 1b25dcf57a9fca5d00cf275046e20a2e3ea3730c still handles Synology Chat inbound webhooks as text-only: payload parsing requires text, delivery and inbound context omit media metadata, docs only describe outbound URL-based file delivery, and existing tests cover text delivery rather than inbound attachments. The prior Codex review reached the same conclusion, and the later current-main Synology changes are DM/access-policy maintenance rather than attachment support.

## Best Possible Solution

Keep this issue open. The best path is to implement inbound attachment handling inside the bundled Synology Chat plugin after confirming Synology Chat's attachment webhook/API shape: extend the typed payload/parser, safely download or stage accepted images, CSVs, and PDFs through existing media-safe helpers, expose MediaPath(s), MediaUrl(s), MediaType(s), and filename metadata in the inbound context, update the public Synology Chat docs, and add focused webhook plus inbound-context regression tests. Avoid moving Synology-specific behavior into core unless a missing generic plugin seam is proven.
