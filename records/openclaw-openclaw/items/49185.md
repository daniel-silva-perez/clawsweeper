## Summary

Keep open. Current main has adjacent failover behavior and a Stainless SDK Retry-After wait cap, but it still does not expose the requested configurable per-provider or per-candidate model API retry budget. The model provider config type and strict schema have no retry-budget field, and the OpenAI/Azure SDK client options only plumb timeout/fetch behavior, not maxRetries.

## Best Possible Solution

Keep the issue open and implement a typed model-provider retry budget, likely under models.providers.<id>.request.retry or a dedicated models.providers.<id>.retry field. Plumb it through OpenAI/Azure client construction and provider-owned transports that retry internally, document how it interacts with https://docs.openclaw.ai/concepts/retry and https://docs.openclaw.ai/concepts/model-failover, and add schema plus transport/fallback tests proving low maxRetries advances fallback quickly on 429/529-style failures.
