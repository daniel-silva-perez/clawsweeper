## Summary

Keep open. Current main has partial transparency and recovery support for compaction, including opt-in notices, checkpoint listing/branch/restore, plugin hooks, and context-engine seams, but it still does not provide the requested preview-before-apply or require-confirmation modes for automatic compaction/trimming/message dropping. The remaining work is broad enough that it should be handled as product/design work rather than conservative cleanup.

## What This Changes

Request to add user-visible notifications, preview, confirmation, rollback/re-expansion, configuration, and developer hooks for automatic context compaction, trimming, and message dropping.

## Best Possible Solution

Define a maintainer-approved product shape that preserves current automatic recovery by default, keeps existing notifyUser/checkpoint/context-engine pieces, and adds any new preview/approval controls only where the channel or Control UI can safely pause, display the proposed summary, and resume or branch without blocking emergency overflow recovery.

## Reproduction Assessment

Not applicable as a bug report. There is a high-confidence inspection path for the feature gap: current config/code/docs show notices, checkpoints, hooks, and context engines, but no preview-before-apply or require-confirmation mode for automatic compaction.

## Solution Assessment

Unclear. The requested direction is useful, but a blanket confirmation gate is not obviously the safest implementation; a narrower design around opt-in notices, Control UI preview/restore, and non-blocking pre-compaction warnings would be more maintainable.
