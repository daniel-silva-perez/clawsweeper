## Summary

Keep open. Current main has the server-side ACP stdout protection, but this PR is not fully obsolete: its stated CLI startup guard before command registration is still absent, and the review discussion shows real scoping concerns around preserving `openclaw acp client` stdout behavior.

## Best Possible Solution

Keep this PR open for maintainer review or supersede it with a corrected main change that covers ACP bridge stdout hygiene from CLI startup through `serveAcpGateway`, explicitly excludes `openclaw acp client`, and decides whether the standalone ACP server entrypoint also needs console capture. If maintainers accept the current server-only fix as sufficient, close the PR manually with that narrower rationale.

## Risks / Open Questions

- Closing this PR as implemented would drop the still-unmerged CLI startup-routing portion unless maintainers explicitly decide the server-only fix is the intended final scope.
