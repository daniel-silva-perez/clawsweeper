## Summary

Keep open. Current main implements the server-side ACP stdout protection from #49060 and records it in v2026.4.26, but this PR is broader than that shipped fix: its CLI startup guard before command registration is still absent from main, and the `openclaw acp client` stdout behavior still needs careful scoping.

## Best Possible Solution

Keep this PR open for maintainer review or supersede it with a focused main change that routes ACP bridge logs to stderr before CLI command-registration side effects, explicitly excludes the real `openclaw acp client` debug mode, and decides whether the standalone ACP server entrypoint also needs console capture. If maintainers accept the shipped server-only fix as sufficient, close this PR manually with that narrower rationale.
