## Summary

Keep open. Current main shipped the server-side ACP stdout protection, but this PR still contains unmerged CLI startup work: routing ACP bridge console logs to stderr before command/subcommand registration. Current main does not have an equivalent ACP-specific guard in `runCli()`, and the server-only release note does not cover that broader PR scope.

## Best Possible Solution

Keep this PR open for maintainer review, or supersede it with a focused current-main patch that routes ACP bridge logs to stderr immediately after console capture and before `buildProgram()`/command registration, excludes the real `openclaw acp client` mode including option-value edge cases, and decides whether direct `src/acp/server.ts` execution should enable console capture. Add regression tests for bridge flags, client flags, option-value false positives, and registration ordering.
