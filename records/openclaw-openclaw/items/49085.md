## Summary

Keep open. Current main implements the server-side ACP stdout protection from the related bug, but it does not implement this PR's broader CLI startup guard in `runCli()`. The unmerged part still matters because console capture is enabled before `buildProgram()` and command/subcommand registration, while no ACP-specific `routeLogsToStderr()` call exists in `src/cli/run-main.ts`.

## Best Possible Solution

Keep this PR open for maintainer review, or supersede it with a focused current-main patch that routes ACP bridge logs to stderr immediately after console capture and before `buildProgram()`/command registration, excludes real `openclaw acp client` invocations including option-value edge cases, and decides whether the standalone ACP server entrypoint should install console capture. Add regression tests for bridge flags, client flags, option-value false positives, and registration ordering.
