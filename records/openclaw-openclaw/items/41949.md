## Summary

Keep open. Current main still has useful partial mitigations, but it does not implement the requested browser-specific, task-aware context budgeting. Default AI snapshots can still be 40,000 characters, efficient snapshots are opt-in or config-defaulted only when no explicit format is supplied, and the browser tool still returns wrapped snapshot text directly as a tool result in model context rather than keeping full page state out-of-context with fetch-on-demand detail retrieval.

## Best Possible Solution

Implement a browser-owned context-budget policy: default agent-facing snapshots to compact/actionable output where appropriate, apply budget limits consistently across AI, role, Chrome MCP, and efficient paths, adapt detail level under context pressure, keep full page state outside the main prompt by default, expose selector/ref/detail fetch-on-demand, and add regression coverage for browser tool snapshot actions, resolveSnapshotPlan, /snapshot routing, Chrome MCP snapshots, and repeated browser-tool loops.
