## Summary

Keep open. Current main has useful partial mitigations, including generic live tool-result truncation and opt-in efficient browser snapshots, but it still does not implement the browser-specific, task-aware context budgeting requested here. Default AI snapshots remain broad, efficient mode is conditional, role snapshot paths do not consistently enforce the efficient character budget, and browser snapshots are still returned as normal text tool results that enter the session context.

## Best Possible Solution

Keep this open and implement a browser-owned context-budget policy: default agent-facing snapshots to compact/actionable output where appropriate, apply max-character/detail budgets consistently across AI, role, Playwright, CDP, and Chrome MCP paths, adapt detail under context pressure, keep full page state outside the main prompt by default, expose selector/ref/detail fetch-on-demand, and add regression coverage for executeSnapshotAction, resolveSnapshotPlan, /snapshot routing, Chrome MCP snapshots, role snapshots, and repeated browser-tool loops.

## Risks / Open Questions

- The issue still lacks a concrete URL, transcript, or token trace, so implementation should start with a current reproduction or telemetry capture.
- Changing snapshot defaults could affect debugging, ref fidelity, and workflows that intentionally need full AI/ARIA inspection output; explicit full-detail modes should remain available.
- Generic tool-result truncation can hide the worst overflow symptom while still leaving low-signal browser text in context, so the browser fix should complement rather than replace the existing guardrails.
