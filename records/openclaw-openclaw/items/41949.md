## Summary

Keep open. Current main has useful partial mitigations, but it still does not implement the browser-specific, task-aware context budgeting requested here. Explicit AI snapshots can still use a broad 40,000-character path, efficient snapshots are opt-in/config-defaulted rather than universal, and the browser output path does not adapt to remaining model context or keep full page state out of the main prompt by default.

## Best Possible Solution

Implement a browser-owned context-budget policy: default agent-facing snapshots to compact/actionable output where appropriate, apply budget limits consistently across AI, role, Chrome MCP, and efficient paths, tighten detail under context pressure, support selector/ref/detail fetch-on-demand so full page state stays outside the main prompt, and add regression coverage for browser-tool.actions, resolveSnapshotPlan, /snapshot routing, Chrome MCP snapshots, and repeated browser tool loops.
