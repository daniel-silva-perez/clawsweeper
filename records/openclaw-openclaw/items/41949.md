## Summary

Keep open. Current main still has partial mitigations, but it does not implement the browser-specific, context-budget-aware behavior requested here: normal AI snapshots can still contribute large model-visible text, efficient mode is optional/conditional, and not all snapshot paths consistently apply the compact budget.

## What This Changes

The issue requests browser interaction snapshots that send only task-relevant, budget-aware page state into model context instead of repeatedly carrying large raw page content.

## Best Possible Solution

Implement a browser-owned context budget policy that defaults agent-facing snapshots to compact/actionable output where appropriate, applies max-character/detail limits consistently across AI, role, CDP, Chrome MCP, and efficient paths, adapts under context pressure, and keeps full page state available through explicit detail fetches rather than normal tool-result text.

## Reproduction Assessment

No high-confidence end-to-end reproduction is available because the issue does not provide a specific URL, transcript, token trace, or model/session config. Source and tests do give a high-confidence static path showing large browser snapshot text can still enter tool context.

## Solution Assessment

Unclear because there is no proposed patch to assess. The best maintainable direction is a browser-specific budget policy layered onto existing snapshot modes, not only generic session pruning after the tool result has already been produced.
