## Summary

Keep open. Current main still has useful partial mitigations, but it does not implement the browser-specific, task-aware context budgeting requested here. Default AI snapshots are still broad, efficient mode remains optional/conditional, managed role snapshot paths do not consistently apply the efficient max-character budget, and browser snapshot text is still returned as normal tool-result text that enters model context.

## Best Possible Solution

Implement a browser-owned context-budget policy: default agent-facing snapshots to compact/actionable output where appropriate, apply max-character/detail budgets consistently across AI, role, CDP, Chrome MCP, and efficient paths, adapt detail under context pressure, keep full page state outside the main prompt by default, expose selector/ref/detail fetch-on-demand, and add regression coverage for browser snapshot tool calls, resolveSnapshotPlan, /snapshot routing, Chrome MCP snapshots, and repeated browser-tool loops.
