## Summary

Keep open. Current main still has partial browser snapshot mitigations, but it does not implement the requested browser-specific, task-aware context budgeting. Explicit AI snapshots can still take the broad 40,000-character path, efficient snapshots remain opt-in or config-defaulted, and the tool returns the wrapped snapshot text directly into agent context rather than keeping full page state out of the prompt with fetch-on-demand detail retrieval.

## Best Possible Solution

Implement a browser-owned context-budget policy: default agent-facing snapshots to compact/actionable output where appropriate, apply budget limits consistently across AI, role, Chrome MCP, and efficient paths, adapt detail level under context pressure, keep full page state outside the main prompt by default, expose selector/ref/detail fetch-on-demand, and add regression coverage for browser tool snapshot actions, resolveSnapshotPlan, /snapshot routing, Chrome MCP snapshots, and repeated browser-tool loops.
