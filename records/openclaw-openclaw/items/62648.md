## Summary

Keep this PR open. Current main still does not implement the PR’s intended plugin/config schema-validator behavior: `additionalProperties` AJV errors pass through AJV’s generic message without appending the offending property name. A separate gateway protocol formatter already handles `params.additionalProperty`, but that is a different validation path and does not cover plugin config errors like the linked report. The provided PR diff is narrow and security-neutral, so the best path is to rebase/review a focused version rather than close it.

## Best Possible Solution

Keep this PR open and review it as the likely narrow fix: rebase onto current main, preserve the existing terminal sanitization path for rendered error text, and add the included regression test for `additionalProperties` so plugin and config validation messages name the unexpected property.
