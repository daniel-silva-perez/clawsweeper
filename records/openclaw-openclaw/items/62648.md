Keep this PR open. Current main still lacks the requested `additionalProperties` property-name formatting in the plugin schema validator, while the submitted diff is narrow, matches AJV's documented `params.additionalProperty` contract, and adds the missing unit coverage. ## What This Changes

The PR updates plugin schema validation to append the AJV `additionalProperty` name to `additionalProperties` error messages and adds a schema-validator unit test for an unexpected property.

## Best Possible Solution

Land or adapt this narrow schema-validator formatter/test change once CI is green, keeping the fix limited to AJV `additionalProperties` error formatting. If maintainers want consistency, use the gateway protocol wording while preserving the same `params.additionalProperty` extraction and regression coverage.

## Solution Assessment

Yes. The proposed approach is the narrowest maintainable fix for this PR's target: it reads AJV's documented singular `additionalProperty` param, guards non-string and empty values, leaves validation semantics unchanged, and adds focused coverage.
