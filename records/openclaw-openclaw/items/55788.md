## Summary

Keep this PR open. The reported bug is real on current main, but the branch is stale against the current manifest-based media defaults and still has a concrete endpoint-routing bug in its Codex audio wrapper.

## What This Changes

The PR adds `openai-codex` audio capability and a transcription handler, adds an old default audio model entry, and extends the OpenAI plugin test assertion.

## Best Possible Solution

Refresh the branch against current main, add Codex audio consistently to the runtime provider and plugin manifest metadata/default model/auto-priority paths, preserve explicit non-ChatGPT audio endpoint overrides, and update focused tests plus docs if the public support matrix changes.
