## Summary

Keep this PR open. Current main still does not implement the intended OpenAI Codex OAuth audio media-understanding support: the codex media provider is image-only, has no `transcribeAudio` handler, and auto audio selection derives from provider audio capability/handler metadata. The PR is plausible but needs refresh/review against current main because the defaults path changed and the PR head has an unresolved base URL override concern.

## Best Possible Solution

Keep the PR open and review or refresh it against current main. The implementation should add `audio` capability, an audio default model, audio auto-priority/metadata, and a `transcribeAudio` handler for `openai-codex`; add regression coverage for the handler and auto audio selection; and handle codex transcription base URLs so the ChatGPT backend URL is not used for `/audio/transcriptions` while explicit operator overrides remain respected.
