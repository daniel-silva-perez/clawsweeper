## Summary

Keep open. Current main still lacks OpenAI Codex audio transcription, but this PR is not merge-ready because current main now resolves media defaults and auto selection from provider and manifest metadata, and the proposed Codex audio wrapper would discard configured audio endpoints.

## What This Changes

The PR adds audio capability and a `transcribeAudio` wrapper for the OpenAI Codex media-understanding provider, adds an `openai-codex` default audio model entry, and updates an OpenAI plugin test assertion.

## Best Possible Solution

Refresh the branch against current main, add Codex audio consistently to the runtime provider and plugin manifest metadata/default model/auto-priority paths, preserve explicit OpenAI-compatible audio endpoint overrides while stripping only known ChatGPT backend URLs, and update focused tests plus docs if the public support matrix changes.

## Reproduction Assessment

Yes. Current-main code inspection gives a high-confidence reproduction path: configure `openai-codex` for media audio and the runner cannot select or execute it because the provider has no audio capability metadata or `transcribeAudio` handler.

## Solution Assessment

No. The PR fixes part of the runtime gap, but the current branch is no longer the best solution because it bypasses explicit endpoint policy and updates an obsolete defaults table instead of current provider and manifest metadata.

## Review Findings

  - confidence: 0.9

  - confidence: 0.86
