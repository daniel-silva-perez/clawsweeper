## Summary

Keep open. Current main still lacks the requested group-to-main routing surface, and the open patch is a plausible starting point but has blocking coverage gaps in outbound routing, Telegram topic overrides, and privacy documentation.

## What This Changes

The PR adds a per-binding group/channel session-scope option to config and routing, and also bundles an IRC active-client registry for proactive sends.

## Best Possible Solution

Ship this as a first-class binding session-scope feature on current main: choose the canonical schema shape, propagate it through inbound and outbound session derivation, preserve it across Telegram topic overrides, update generated config help and public docs, add focused regression tests, and split or rebase the IRC active-client work after #45662 is resolved.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: current main rejects/does not expose `groupScope`, group/channel peers derive isolated non-main keys, and the PR diff can be checked against outbound and Telegram recomputation paths that do not carry the new scope. No tests were run because this review was explicitly read-only.

## Solution Assessment

No. The opt-in per-binding direction is plausible, but this patch is not yet the best implementation because it misses supported session derivation paths, lacks privacy-facing documentation, and bundles unrelated IRC work.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Propagate groupScope into outbound session derivation:** `src/routing/resolve-route.ts:673`
  - body: When a binding sets `groupScope: "main"`, inbound routing can collapse group/channel traffic onto the main session, but outbound helpers still derive group targets from only global `dmScope`. Sends that omit an explicit session key can therefore fork the same configured conversation back into a per-channel session instead of the main session.
  - confidence: 0.89
- **[P2] Preserve groupScope in Telegram topic overrides:** `src/routing/resolve-route.ts:673`
  - body: Telegram forum topics with `topicAgentId` recompute `route.sessionKey` using `buildAgentSessionKey()` and only global `dmScope`. A topic matching `groupScope: "main"` would be overwritten back to a per-topic group session, so that supported channel path would not honor the new option.
  - confidence: 0.84
- **[P3] Fall back when active IRC sends fail:** `extensions/irc/src/send.ts:72-74`
  - body: When an active monitor client is found, the new IRC path calls `active.sendPrivmsg()` and returns. If the monitor connection drops after `isReady()` but before the write, proactive sends fail instead of trying the transient fallback preserved for missing active clients.
  - confidence: 0.72
