## Summary

Keep open. The requested route-binding group/session scope is not implemented on current main, and the PR is still a useful implementation candidate, but the proposed patch is incomplete across outbound routing, Telegram topic overrides, and privacy-facing config/docs guidance before it should merge.

## What This Changes

The PR adds a top-level route-binding `groupScope` option for matched group/channel peers to use an agent main session and also adds IRC active-client reuse for proactive sends.

## Best Possible Solution

Refresh this as a first-class route-binding session-scope change on current main, ideally after splitting or landing the IRC proactive-send fix separately. The accepted implementation should choose the canonical config shape, update TypeScript/Zod/generated schema/help/docs, propagate the scope through inbound routing, outbound session derivation, Telegram topic overrides, and focused tests, and clearly document the privacy consequence of merging a selected group/channel into main.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: current main has no `groupScope` symbol, strict route-binding schema would reject the PR's example key, and non-direct peers still resolve to per-channel/per-group session keys.

## Solution Assessment

No. The proposed direction is plausible and opt-in, but this patch is not yet the best maintainable solution because it only threads the scope through inbound routing and leaves outbound sends, Telegram topic overrides, docs/help metadata, and the separate IRC work unresolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Propagate groupScope into outbound session derivation:** `src/routing/resolve-route.ts:673`
  - body: When a binding sets `groupScope: "main"`, inbound routing can collapse group/channel traffic onto the main session, but outbound helpers still derive group targets from only global `dmScope`. Sends that omit an explicit session key can fork the same conversation back into a per-channel session instead of the configured main session.
  - confidence: 0.89
- **[P2] Preserve groupScope in Telegram topic overrides:** `src/routing/resolve-route.ts:673`
  - body: Telegram forum topics with `topicAgentId` recompute `route.sessionKey` using `buildAgentSessionKey()` and only `dmScope`. A topic that matches `groupScope: "main"` will be overwritten back to a per-topic group session, so that supported channel path will not honor the new option.
  - confidence: 0.84
- **[P3] Fall back when the active IRC send fails:** `extensions/irc/src/send.ts:72-74`
  - body: When an active monitor client is found, the new IRC path calls `active.sendPrivmsg()` and returns. If the monitor connection drops after `isReady()` but before the write, proactive sends fail instead of trying the transient fallback preserved for missing active clients.
  - confidence: 0.72
