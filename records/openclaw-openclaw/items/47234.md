## Summary

Keep open. Current main still does not expose `groupScope`, and the proposed patch is a real implementation candidate, but it is incomplete against current routing behavior: it only handles inbound route resolution while outbound session derivation and Telegram topic overrides can still create per-channel sessions, and the bundled IRC send change overlaps the separate open #45662 work.

## What This Changes

The PR adds a top-level route-binding `groupScope` field to config/types/schema, threads it through inbound route session-key construction, and adds IRC active-client reuse for proactive sends.

## Best Possible Solution

Refresh this as a first-class route-binding session-scope change on current main, preferably after splitting or landing the IRC proactive-send work separately. The merged version should choose the canonical binding config shape, update TypeScript/Zod/generated schema/help/docs, preserve the scope across inbound routing, outbound target/session derivation, Telegram topic overrides, and add focused tests plus clear privacy guidance.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: current main has no `groupScope`, strict route-binding schema would reject the PR example key, and non-direct peers still resolve to per-channel or per-group session keys; the proposed patch can be checked against outbound fallback and Telegram topic code paths without live IRC/Telegram credentials.

## Solution Assessment

No. The opt-in product direction is plausible, but this patch is not yet the best maintainable solution because it is incomplete across outbound and Telegram routing, lacks user-facing privacy/schema documentation, and bundles separate IRC proactive-send work that is already tracked by #45662.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Propagate groupScope into outbound session derivation:** `src/routing/resolve-route.ts:673`
  - body: When a binding sets `groupScope: "main"`, inbound routing can collapse group/channel traffic onto the main session, but outbound helpers still derive group targets from only global `dmScope`. Sends that omit an explicit session key can therefore fork the same configured conversation back into a per-channel session instead of the main session.
  - confidence: 0.89
- **[P2] Preserve groupScope in Telegram topic overrides:** `src/routing/resolve-route.ts:673`
  - body: Telegram forum topics with `topicAgentId` recompute `route.sessionKey` using `buildAgentSessionKey()` and only global `dmScope`. A topic matching `groupScope: "main"` would be overwritten back to a per-topic group session, so that supported channel path would not honor the new option.
  - confidence: 0.84
- **[P3] Fall back when active IRC sends fail:** `extensions/irc/src/send.ts:72-74`
  - body: When an active monitor client is found, the new IRC path calls `active.sendPrivmsg()` and returns. If the monitor connection drops after `isReady()` but before the write, proactive sends fail instead of trying the transient fallback preserved for missing active clients.
  - confidence: 0.72
