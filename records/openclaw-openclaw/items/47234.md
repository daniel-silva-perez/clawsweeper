## Summary

Keep open. Current main still does not implement `groupScope`, and the PR remains a useful implementation candidate, but the proposed patch is incomplete because inbound group/channel routing can collapse to main while outbound/proactive session derivation and Telegram topic overrides still derive per-channel/per-topic session keys. The branch also bundles the separate IRC active-client work from #45662, which makes conservative review and landing harder.

## What This Changes

This PR adds a top-level route-binding `groupScope` option to route matched group/channel peers to the agent main session and bundles an IRC active-client registry for proactive sends.

## Best Possible Solution

Refresh the feature as a first-class route-binding session-scope change after splitting or landing #45662 separately. The implementation should align the public config shape, strict Zod schema, generated schema/help metadata, docs, inbound route resolution, outbound session derivation, Telegram topic overrides, and focused tests, with an explicit privacy note for group/channel messages merged into main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Propagate groupScope into outbound sends:** `src/routing/resolve-route.ts:673`
  - body: The patch only threads `groupScope` through inbound `resolveAgentRoute()`. Outbound paths such as `gateway send` and tool-driven sends still use `resolveOutboundSessionRoute()` -> `buildOutboundBaseSessionKey()`, which passes only global `dmScope`; a channel configured to merge inbound traffic into main will still mirror/proactively send into `agent:<agent>:<channel>:...`, splitting the same conversation across sessions.
  - confidence: 0.89
- **[P2] Preserve groupScope in Telegram topic overrides:** `src/routing/resolve-route.ts:673`
  - body: Telegram forum topics with `topicAgentId` recompute `sessionKey` in `extensions/telegram/src/conversation-route.ts` using only `dmScope`. After this change, a topic that matches `groupScope: "main"` will have the main-session route overwritten back to a per-topic group session, so that supported channel path does not honor the new option.
  - confidence: 0.84
- **[P3] Fall back when the active IRC send fails:** `extensions/irc/src/send.ts:72-74`
  - body: When an active monitor client is found, the new IRC path calls `active.sendPrivmsg()` and returns without a transient fallback. If the monitor connection drops after `isReady()` but before the write, proactive sends now fail instead of trying the fresh transient path this branch otherwise preserves for missing active clients.
  - confidence: 0.72
