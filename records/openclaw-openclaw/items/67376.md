## Summary

Keep this PR open. Current main has shipped important partial MCP loopback hardening for owner vs non-owner tool visibility, but it still does not implement this PR's central per-backend token-bound scope model: session key, account id, and message channel still come from mutable request headers and are still exported into backend env/config.

## Best Possible Solution

Keep this PR open for maintainer security review, or supersede it with a narrower PR that preserves the shipped owner/non-owner bearer mitigation while adding a per-backend token registry for session key, account id, message channel/provider, and owner status; stops emitting scope headers/env vars; invalidates scoped tool-cache entries on revocation; adds negative spoofing tests for all scope fields; fixes token lifecycle races; and removes or splits the unrelated plugin loader change.
