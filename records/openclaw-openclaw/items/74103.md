Current main still has the behavior this PR targets: ACP background task progress is capped at 240 characters, ordinary successful completions return no terminalSummary, and the task completion formatter only uses terminalSummary for the delivered “Background task done” payload. The PR is a focused one-file implementation candidate for #74070, not obsolete, but it should get maintainer review and ideally regression coverage before landing.
