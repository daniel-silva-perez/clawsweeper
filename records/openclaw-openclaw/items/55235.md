## Summary

Keep open. Current main supports manually printing the generated `openclaw.json` schema and accepts a root `$schema` key, but it does not generate `openclaw_schema.json`, set `$schema` automatically, or expose `update.maintain_config_json_schema`. The issue also has an open same-author implementation PR with closing syntax, so the issue should stay open until that PR is reviewed and either merged or closed.

## Best Possible Solution

Keep this issue open and use PR #55390, or its current successor, as the implementation path. The best fix belongs in the CLI/config write or update/bootstrap flow: generate `openclaw_schema.json` beside the active config, stamp the schema with the OpenClaw version, preserve a user-specified non-empty `$schema`, add `update.maintain_config_json_schema` defaulting true, and cover update/bootstrap/config-write behavior with focused tests and docs.
