## Summary

Keep open. Current main has the manual `openclaw config schema` command and preserves an existing root `$schema`, but it does not generate or refresh `openclaw_schema.json` during CLI bootstrap/update, does not default a missing `$schema`, and does not expose the requested `update.maintain_config_json_schema` setting. The provided context also shows an open unmerged PR with closing syntax for this issue, so the issue should remain open until that PR is reviewed, landed, replaced, or closed.

## What This Changes

Request automatic local generation and updating of `openclaw_schema.json`, defaulting missing `openclaw.json#$schema` to that file, adding an opt-out update setting, and documenting/testing the behavior.

## Best Possible Solution

Review, land, or replace PR #55390 with a narrow implementation that maintains the local schema artifact through the existing config/update paths, preserves user-authored schema settings, adds the requested opt-out to the config contract, and updates the CLI docs and focused tests.
