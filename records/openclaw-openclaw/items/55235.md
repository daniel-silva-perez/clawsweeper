## Summary

Keep open. Current main still provides only manual schema printing and preserves existing root `$schema` values; it does not create or refresh `openclaw_schema.json`, default a missing schema reference, or expose the requested update opt-out. The issue is also paired with open PR #55390, so it should remain open until that implementation candidate is reviewed, landed, replaced, or closed.

## What This Changes

Request automatic local maintenance of `openclaw_schema.json` during OpenClaw bootstrap/update, including default `$schema` wiring, schema version metadata, an `update.maintain_config_json_schema` opt-out, docs, and tests.

## Best Possible Solution

Implement a narrow, idempotent schema-maintenance path that writes `openclaw_schema.json` beside the active config, adds schema-version metadata and `update.maintain_config_json_schema`, preserves custom schema URIs, and documents/tests the defaulting, opt-out, rerun, and version-refresh behavior.
