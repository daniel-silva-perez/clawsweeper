## Summary

Keep this issue open. Current main still renders the macOS Talk Mode overlay as the generic Circle/RadialGradient orb, and the macOS Talk Mode config/state path only carries voice settings plus `ui.seamColor`, not assistant identity or avatar data. The gateway already exposes resolved assistant identity/avatar metadata through `agent.identity.get`, so this looks like a narrow macOS app integration gap rather than a duplicate of the closed Control UI identity issue #22199.

## Best Possible Solution

Keep this issue open and implement a narrow macOS app change: resolve the active assistant identity/avatar for Talk Mode using the existing gateway identity/avatar contract, render a circular cropped avatar inside the Talk overlay when available, preserve the existing rings/pulse/thinking arcs, and fall back to the current orb whenever identity/avatar resolution fails. The closed #22199 Control UI work should remain separate because it does not update this macOS Talk Mode surface.
