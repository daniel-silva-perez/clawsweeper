## Summary

Keep this issue open. Current main still renders the macOS Talk Mode overlay through the generic Circle/RadialGradient TalkOrbView and the Talk config path only carries voice settings plus ui.seamColor; the existing assistant identity/avatar resolver and avatar route exist, but they are not wired into the macOS Talk overlay. The closed #22199 work is distinct Control UI identity work, as the reporter clarified.

## Best Possible Solution

Keep this issue open and implement a narrow macOS app change: fetch the active assistant identity/avatar for Talk Mode via the existing agent.identity.get and /avatar/<agentId>?meta=1 contracts, render a circular cropped avatar inside the existing Talk overlay when an image avatar is resolvable, keep the current rings/pulse/thinking arcs and click/drag/pause behavior, and fall back to the current orb on missing identity, unsupported avatar, auth failure, or image-load failure. Leave #22199 closed as the separate Control UI identity work.
