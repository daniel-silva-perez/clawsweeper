## Summary

Keep open. Current main still formats the per-response `/usage` footer from turn token/cache/cost/session data only; the existing `/status` path already loads provider quota windows, and the GitHub Copilot plugin already exposes the requested Premium/Chat windows. The direct implementation PR is closed unmerged, while the broader open footer PR overlaps nearby plumbing but does not currently claim this provider-quota footer request.

## What This Changes

Add GitHub Copilot Premium/Chat quota percentages to the per-response `/usage` footer when `/usage tokens` or `/usage full` is enabled.

## Best Possible Solution

Implement a generic reply-footer enhancement that appends a compact formatUsageWindowSummary result for the active provider’s usage windows when `/usage tokens` or `/usage full` is enabled, while keeping provider-specific auth and fetching inside plugins. Add reply-runner regression coverage, update token-use docs, and include a user-facing changelog entry credited to the reporter.
