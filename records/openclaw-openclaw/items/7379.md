## Summary

Keep this issue open. Current main still formats the per-response `/usage` footer from turn token/cache/cost data only; provider quota windows are still loaded and formatted on the `/status` path. The GitHub Copilot plugin already provides Premium/Chat windows, and PR #43098 directly attempted this work but is closed unmerged, so there is no implemented-on-main close path.

## Best Possible Solution

Keep this issue open and implement a focused reply-footer enhancement: when `/usage tokens` or `/usage full` is enabled and the active provider has provider usage windows, append a generic formatted quota summary such as `Premium 86% left · Chat 100% left` to the existing per-response footer. Keep Copilot-specific fetching in the plugin, avoid blocking final replies on quota fetch failures, and add regression coverage for the footer path.
