## Summary

Keep open. Current main still lacks the requested bridge between provider quota windows and the per-response `/usage` footer: reply formatting only uses token/cache/cost/session data, while `/status` and the GitHub Copilot plugin already expose the Premium/Chat usage-window data needed for this feature.

## What This Changes

The issue requests showing GitHub Copilot Premium/Chat remaining quota percentages in the per-response `/usage` footer.

## Best Possible Solution

Add a generic reply-footer integration that appends `formatUsageWindowSummary` output when `/usage tokens` or `/usage full` is enabled and the active provider exposes usage windows, while keeping provider-specific fetching and auth inside plugins. Update token-use docs, add a user-facing changelog entry, and cover the reply-runner footer path with regression tests.

Yes. A high-confidence code-level reproduction is to enable `responseUsage` in the reply path with mocked turn usage and mocked active provider usage windows; current main appends only token/cache/cost/session text and never calls the provider usage loader for the footer.
