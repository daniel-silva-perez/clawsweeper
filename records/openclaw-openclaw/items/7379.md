## Summary

Keep this issue open. Current main still does not add provider quota windows to the per-response `/usage` footer: the reply footer only formats token/cache/cost/session data, while `/status` already loads generic provider usage windows and the GitHub Copilot plugin already maps Premium/Chat quota data. The linked implementation PR #43098 is closed unmerged, so there is no merged canonical fix to close against.

## Best Possible Solution

Keep the issue open and implement a focused reply-footer enhancement: when `/usage tokens` or `/usage full` is enabled and the active provider has generic provider usage windows, append a compact quota summary such as `Premium 86% left · Chat 100% left` to the existing footer without blocking final reply delivery. Keep provider-specific fetching in plugins, update `docs/reference/token-use.md` and changelog if the behavior changes user-facing docs, and add regression coverage for the footer path.
