## Summary

Keep open. Current main still does not connect provider quota windows to the per-response `/usage` footer: the reply path formats only token/cache/cost/session data, while `/status` already loads provider usage windows and the GitHub Copilot plugin already exposes the requested Premium/Chat percentages. The direct implementation PR #43098 is closed unmerged, and open PR #69618 overlaps nearby footer plumbing but does not specifically claim or implement this provider-quota footer request.

## What This Changes

Add the active provider's quota-window summary, especially GitHub Copilot Premium/Chat percentages, to the per-response `/usage tokens` or `/usage full` footer.

## Best Possible Solution

Implement a generic reply-footer enhancement that appends `formatUsageWindowSummary` for the active provider's usage windows when `/usage tokens` or `/usage full` is enabled, while keeping provider-specific fetch/auth behavior inside plugins. Add reply-runner regression coverage, update token-use docs, and include a user-facing changelog entry.
