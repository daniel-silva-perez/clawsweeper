## Summary

Keep this issue open. Current main still only appends token/cache/cost/session data to the per-response `/usage` footer; provider quota windows are available through the status/provider-usage path and the GitHub Copilot plugin, but they are not wired into the reply footer. The linked implementation PR #43098 is closed unmerged, so there is no canonical merged fix to close against.

## Best Possible Solution

Keep the issue open and implement a focused reply-footer enhancement: when `/usage tokens` or `/usage full` is enabled and the active provider has generic provider usage windows, append a compact quota summary such as `Premium 86% left · Chat 100% left` to the existing footer without blocking final reply delivery. Keep provider-specific fetching in plugins, update `docs/reference/token-use.md` and changelog if required, and add regression coverage for the footer path.
