## Summary

Keep open. Current main still does not bridge active provider quota windows into the per-response `/usage` footer: the reply path formats token/cache/cost/session data only, while `/status` already loads provider usage windows and the GitHub Copilot plugin already exposes the requested Premium/Chat data. The direct implementation PR #43098 is closed unmerged, and open #69618 overlaps footer plumbing but does not claim this provider-quota footer behavior.

## What This Changes

Request to add GitHub Copilot Premium/Chat quota percentages to the per-response `/usage` footer when `/usage tokens` or `/usage full` is enabled.

## Best Possible Solution

Implement a generic reply-footer enhancement that appends `formatUsageWindowSummary` for the active provider’s usage windows when `/usage tokens` or `/usage full` is enabled, while keeping provider-specific auth and fetch behavior inside plugins. Cover the reply-runner/footer path with regression tests, update token-use docs, and add a user-facing changelog entry.
