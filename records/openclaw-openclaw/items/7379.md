## Summary

Keep this issue open. Current main still formats the per-response `/usage` footer from turn token/cache/cost data only; provider quota windows are fetched and formatted on status/reporting paths, and the Copilot plugin already supplies Premium/Chat windows. PR #43098 directly attempted the requested footer wiring but is closed unmerged, so the remaining work is a focused implementation rather than cleanup closure.

## Best Possible Solution

Implement a focused reply-footer enhancement: when `/usage tokens` or `/usage full` is enabled and the active provider has generic provider usage windows, append a compact formatted quota summary such as `Premium 86% left · Chat 100% left` to the existing footer. Keep provider-specific fetching in plugins, avoid blocking final replies on usage failures, and add regression coverage for the footer path.
