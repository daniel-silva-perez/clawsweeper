## Summary

Current main still does not add provider quota windows to the per-response `/usage` footer. The reply path formats only turn token/cache/cost/session data, while `/status` already has generic provider-usage plumbing and the GitHub Copilot plugin already exposes Premium/Chat windows. The direct implementation PR #43098 is closed unmerged, and open #69618 overlaps footer plumbing but does not implement or claim this quota-window request.

## What This Changes

Add GitHub Copilot Premium/Chat provider quota percentages to the per-response `/usage` footer when available.

## Best Possible Solution

Implement a generic reply-footer enhancement: when `/usage tokens` or `/usage full` is enabled and the active provider has provider usage windows, append a compact `formatUsageWindowSummary` result to the existing footer while keeping provider-specific fetching inside plugins. Update token-use docs, add a user-facing changelog entry, and cover the reply-runner/footer path with regression tests.
