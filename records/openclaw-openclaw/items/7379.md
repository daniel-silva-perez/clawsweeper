## Summary

Keep open. Current main still does not append provider quota windows to the per-response `/usage` footer: the reply path only composes token/cache/cost/session usage text, while `/status` already loads provider usage windows and the GitHub Copilot plugin exposes the requested Premium/Chat data. The direct implementation PR #43098 is closed unmerged, and open PR #69618 overlaps footer plumbing but does not specifically close this quota-window request.

## What This Changes

The issue requests adding GitHub Copilot Premium/Chat quota percentages to the per-response `/usage` footer when `/usage tokens` or `/usage full` is enabled.

## Best Possible Solution

Implement a generic reply-footer enhancement that appends `formatUsageWindowSummary` for the active provider's usage windows when `/usage tokens` or `/usage full` is enabled, while keeping provider-specific fetching inside plugins. Add regression coverage for the reply-runner footer path, update token-use docs, and include a user-facing changelog entry.
