## Summary

Keep open. Current main still lacks the requested bridge between provider quota windows and the per-response `/usage` footer: the reply path formats token/cache/cost/session data only, while `/status` and the GitHub Copilot plugin already have the generic Premium/Chat usage-window plumbing. The direct implementation PR #43098 is closed unmerged, and open #69618 overlaps footer plumbing but does not implement this quota-window request.

## What This Changes

Add GitHub Copilot Premium/Chat provider quota percentages to the per-response `/usage` footer when available.

## Best Possible Solution

Implement a generic reply-footer enhancement: when `/usage tokens` or `/usage full` is enabled and the active provider has usage windows, append a compact `formatUsageWindowSummary` result to the existing footer while keeping provider-specific fetching inside plugins. Update token-use docs and changelog, and add regression coverage for the reply-runner/footer path.
