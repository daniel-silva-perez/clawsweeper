## Summary

Keep open. Current main still does not append provider quota windows to the per-response `/usage` footer: the reply path formats turn token/cache/cost/session data only, while `/status` and the GitHub Copilot plugin already expose the requested Premium/Chat usage-window plumbing. The direct implementation PR #43098 is closed unmerged, and open #69618 overlaps footer plumbing but does not claim or implement this quota-window request.

## What This Changes

The issue requests adding GitHub Copilot Premium/Chat provider quota percentages to the per-response `/usage` footer when available, similar to `/status`.

## Best Possible Solution

Implement a generic reply-footer enhancement that, when `/usage tokens` or `/usage full` is enabled and the active provider has usage windows, appends a compact `formatUsageWindowSummary` result to the existing footer while keeping provider-specific fetching inside plugins. Update the token-use docs and changelog, and add regression coverage for the reply-runner/footer path.
