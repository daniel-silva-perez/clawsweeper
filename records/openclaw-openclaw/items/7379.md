## Summary

Keep open. Current main still has provider quota-window plumbing for `/status` and GitHub Copilot, but the per-response `/usage` footer only formats turn token/cache/cost/session data and never loads or appends Premium/Chat usage windows. The directly linked implementation PR #43098 is closed unmerged, while open #69618 overlaps footer plumbing but does not claim this specific quota-window request.

## What This Changes

Add GitHub Copilot Premium/Chat quota percentage information to the per-response `/usage` footer when `/usage tokens` or `/usage full` is enabled.

## Best Possible Solution

Add a generic reply-footer integration that appends the existing provider usage-window summary for the active provider when response usage is enabled, while preserving plugin-owned quota fetching, OAuth/API-key auth rules, and existing token/cost/session footer behavior. Document the new surface and add focused regression coverage for the reply-runner footer path.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to enable `responseUsage` in the reply path with mocked turn usage and mocked active provider usage windows; current main only appends token/cache/cost/session text and never calls `loadProviderUsageSummary` for the footer.

## Solution Assessment

Yes. The narrow maintainable solution is to reuse the existing generic provider usage loader and `formatUsageWindowSummary` in the reply footer composition path, not to add GitHub Copilot-specific quota fetching inside core reply code.
