## Summary

Keep open. Current main still has the requested exposure-matrix gap, so this PR is not obsolete, but the PR head has a concrete false-positive bug around legacy `dm.policy` precedence that should block merge until fixed.

## What This Changes

The PR modifies `src/security/audit-extra.sync.ts` so the existing open-group elevated/runtime exposure checks also consider channel/account `dmPolicy="open"` and legacy `dm.policy="open"` paths while preserving the existing `open_groups` check IDs.

## Best Possible Solution

Update the DM-policy collector to mirror effective policy resolution, then add focused tests proving modern `dmPolicy` wins over legacy `dm.policy` while still catching legacy-only and account-level open DM policies.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Honor dmPolicy before legacy dm.policy:** `src/security/audit-extra.sync.ts:351-352`
  - body: The new legacy scan reports `dm.policy="open"` even when the same config section explicitly sets a safer `dmPolicy`. Discord and Slack resolve these as `dmPolicy ?? dm.policy`, so a migrated config with `dmPolicy: "allowlist"` plus stale `dm.policy: "open"` would get a false CRITICAL exposure finding. Gate the legacy branch on the modern key being absent, including the account-level branch.
  - confidence: 0.9
