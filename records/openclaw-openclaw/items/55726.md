## Summary

Keep open. Current main still misses the requested dmPolicy exposure-matrix coverage, so this PR is not obsolete, but the current PR head has a real false-positive bug around legacy `dm.policy` precedence that should be fixed before merge.

## What This Changes

The PR modifies `src/security/audit-extra.sync.ts` so existing open-group exposure checks also consider channel/account `dmPolicy="open"` and legacy `dm.policy="open"` paths while preserving the existing `open_groups` check IDs.

## Best Possible Solution

Keep the PR open, update the dmPolicy collector to mirror effective policy resolution precedence, and add focused tests in the current audit test files for modern dmPolicy, legacy fallback, account-level policy, and the stale-legacy-overridden-by-modern-key negative case.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Honor dmPolicy before legacy dm.policy:** `src/security/audit-extra.sync.ts:351-352`
  - body: The new legacy scan reports `dm.policy="open"` even when the same config section explicitly sets a safer `dmPolicy`. Discord and Slack resolve these as `dmPolicy ?? dm.policy`, so a migrated config like `dmPolicy: "allowlist"` plus stale `dm.policy: "open"` would now get a false CRITICAL exposure finding. Gate the legacy branch on `section.dmPolicy === undefined`, and do the same for account-level policy.
  - confidence: 0.9
