## Summary

Keep open. Current main still does not include dmPolicy in collectExposureMatrixFindings(), so the PR’s intended security-audit behavior is not implemented on main. The PR is also not cleanly closeable as obsolete because the linked issue #55612 remains open and the supplied review discussion identified a real false-positive risk in the PR implementation that should be fixed before landing.

## Best Possible Solution

Keep this PR open as a plausible implementation candidate, but require a revision before landing: rebase onto current main, make dmPolicy exposure detection mirror the effective policy precedence used by provider schemas, add regression tests for modern dmPolicy, legacy dm.policy, and the modern-over-legacy override case, then land it or supersede it with a focused PR that closes #55612.
