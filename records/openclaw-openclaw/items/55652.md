## Summary

Keep this PR open. Current main still does not implement Discord cross-instance channel ownership claims, and the Discord message preflight, autocomplete, native slash-command, and provider startup paths still rely on local guild/channel allowlist state. The PR remains a plausible implementation candidate, but its shared claims-file design needs maintainer review before merge.

## Best Possible Solution

Keep this PR open for maintainer review or replacement. The best path is a Discord-plugin-owned fix under extensions/discord/src/monitor that performs robust per-instance and per-bot ownership checks before local allowlist denial in message preflight, autocomplete, and native slash-command dispatch, with regression coverage and live validation for same-token multi-instance routing plus the authorization edge cases already raised in review.
