## Summary

Current main still does not expose an OpenAPI/Swagger schema for the Gateway HTTP API. The Gateway has explicit HTTP route handling and a separate WebSocket JSON Schema generator, but I found no OpenAPI document, generator dependency, committed spec, or `/openapi.json`-style runtime discovery route for programmatic HTTP client generation.

## Best Possible Solution

Keep this issue open. Maintainers should first define the supported public Gateway HTTP API boundary, then add either a synchronized committed/generated OpenAPI artifact or a served schema endpoint such as `/openapi.json`, with docs and a drift check/test. The existing WebSocket protocol JSON Schema should remain separate unless maintainers explicitly decide to model part of that surface in OpenAPI.
