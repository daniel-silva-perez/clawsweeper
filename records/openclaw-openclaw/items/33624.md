## Summary

Keep open. Current main still has explicit Gateway HTTP route handlers and docs for `/v1/models`, `/v1/embeddings`, `/v1/chat/completions`, `/v1/responses`, `/tools/invoke`, session, plugin, media, and Control UI paths, but no OpenAPI/Swagger schema artifact, generator dependency, or `/openapi.json`/Swagger discovery route for client generation.

## Best Possible Solution

Keep this issue open for maintainer API-boundary design. The best path is to define the supported public Gateway HTTP surface, then add either a committed/generated OpenAPI artifact or a served discovery endpoint such as `/openapi.json`, with docs and a drift check/test. The existing WebSocket protocol JSON Schema should stay separate unless maintainers explicitly decide to model that surface in OpenAPI.
