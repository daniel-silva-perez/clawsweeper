## Summary

Keep open. Current main still does not expose an OpenAPI/Swagger schema artifact, generator, Swagger UI, or `/openapi.json` discovery route for the Gateway HTTP APIs. The repository has concrete HTTP endpoints and adjacent WebSocket/config schema machinery, but the requested HTTP OpenAPI contract remains product/API-boundary work rather than cleanup that can be closed.

## Best Possible Solution

Keep this issue open for maintainer API-boundary design. Define which Gateway HTTP routes are public and stable, then add a generated or committed OpenAPI artifact, docs, and a drift check; optionally serve the artifact at an authenticated discovery endpoint such as `/openapi.json`. Keep the WebSocket protocol JSON Schema and config schema surfaces separate unless maintainers explicitly choose to model them in OpenAPI.
