Keep open. Current main still does not implement the Discord adapter guard proposed by this PR, and the linked Discord bug remains open with this PR as the active implementation candidate. The PR also appears paired with the open contributor issue it closes, so it should stay open until the implementation lands or maintainers choose a different fix.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or land an equivalent Discord adapter fix. The best implementation is to omit ThreadStarterBody when the effective Discord thread session already exists while preserving first-turn starter context, with the regression test proving existing thread sessions omit the starter and new sessions still receive it. After merge, the linked Discord bug can close through the PR's closing reference.
