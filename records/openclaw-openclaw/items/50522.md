## Summary

Keep this PR open. Current main still has the Discord adapter pass `ThreadStarterBody` through on thread messages without checking whether the effective session already exists, and the linked Discord bug remains open. The branch is a focused implementation candidate, recently maintained and marked merge-ready in the provided discussion, rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open and review or land the narrow Discord adapter guard, with the included regression test proving existing thread sessions omit `ThreadStarterBody` while new thread sessions still receive first-turn starter context. After merge, the linked Discord bug can close through the PR’s closing reference.
