## Summary

Current main still leaves the reported Big Sur installer UX unresolved. The runtime intentionally requires Node >=22.14, but the macOS installer path still detects only generic macOS and attempts Homebrew `node@24` without a macOS 11 preflight, a verified Node 22 fallback, or public install-doc guidance for Big Sur users.

## What This Changes

The issue asks for macOS Big Sur installation to either be supported or fail early with clear installer/docs guidance instead of reaching the Node/Homebrew failure path.

## Best Possible Solution

Keep the Node >=22.14 runtime requirement, then make the installer and public install docs explicit for macOS 11: either fail early with a friendly unsupported Big Sur message and alternatives, or use a verified Node 22 path if upstream support is confirmed.
