## Summary

Current main still has the macOS Big Sur installer/docs UX gap. OpenClaw intentionally requires Node >=22.14, while `scripts/install.sh` still detects Darwin only as generic macOS, installs Homebrew Node 24, and lacks a Big Sur or older-macOS preflight, Node 22 fallback, or public install-doc warning.

## Best Possible Solution

Keep this issue open and narrow it to installer/docs UX. Preserve the Node >=22.14 runtime requirement, add an early macOS product-version check or tested fallback before `install.sh` tries Homebrew `node@24`, update the public install/Node/installer docs to say whether Big Sur or older macOS is unsupported or needs a specific path, and add focused installer coverage for the macOS version decision.
