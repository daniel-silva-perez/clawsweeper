## Summary

Current main has not resolved the macOS Big Sur installer UX gap. OpenClaw still intentionally requires Node >=22.14, while `install.sh` still tries Homebrew `node@24` on macOS without a macOS version preflight, fallback to Node 22, or install-doc warning for Big Sur/older macOS.

## Best Possible Solution

Keep this issue open and narrow it to installer/docs UX. Preserve the runtime requirement at Node >=22.14, add an early macOS product-version check or Node 22 fallback before `install.sh` tries Homebrew `node@24`, and update the public install docs to explain the tested macOS/Node path for Big Sur or clearly mark it unsupported. Add focused installer coverage for the macOS version decision if the shell code is changed.
