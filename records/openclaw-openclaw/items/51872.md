## Risks / Open Questions

- PR #73168 currently appears Android-focused, while the issue asks for Android, iOS, QR/bootstrap fields, fixed headers/cookies, and OIDC-style browser auth; maintainers should decide whether to narrow scope or track the remaining surfaces separately.
- Transport proxy credentials are secrets on the HTTP upgrade path. Any implementation needs review for storage, logging, header scoping, redirect behavior, canvas/media subrequests, and accidental mixing with OpenClaw application-level token/password auth.
- The linked PR body also mentions disabling TLS fingerprint checks; that is security-sensitive and should be reviewed separately from bearer/basic proxy-header support.
