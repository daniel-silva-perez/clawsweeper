## Summary

Keep open. Current main has loop-detection machinery and clearer parameter-validation error text, but the guard is still disabled by default and generic repeated calls remain warning-only until the global no-progress breaker when explicitly enabled, so the reported default Feishu/Telegram message-spam loop is not proven fixed.

## What This Changes

The issue asks OpenClaw to stop agents from repeatedly retrying failed tool calls and repeatedly sending near-duplicate progress messages when a deterministic tool error occurs.

## Best Possible Solution

Land a focused core runtime guard that handles deterministic tool validation failures as non-retryable within a run, caps repeated identical tool+args+error outcomes, and prevents repeated progress-message delivery for the same failing action while preserving configurable thresholds for legitimate polling.
