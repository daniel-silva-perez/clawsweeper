## Summary

Keep open. Current main still does not implement this PR's intended Google Chat typing-indicator fallback: `resolveBotDisplayName` falls from account config name to top-level `agent.name` and then `OpenClaw`, while `agents.list[].identity.name` is still a supported identity field and the normal target of `openclaw agents set-identity`. The PR is narrow and still useful; the linked markdown-rendering concern in #49350 should remain separate.

## Best Possible Solution

Review and land the narrow Google Chat plugin fallback change in `extensions/googlechat/src/monitor.ts`, ideally with a small regression test covering `typingIndicator: "message"` where the routed agent has `identity.name` but no top-level `name`. Track the separate Google Chat markdown/plain-text rendering issue independently.
