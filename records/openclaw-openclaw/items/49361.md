## Summary

Current main still does not implement this PR's intended Google Chat typing-indicator fallback. `resolveBotDisplayName` uses account config name, then top-level `agent.name`, then hardcoded `OpenClaw`, while `agents.list[].identity.name` remains a supported identity field and the normal `set-identity` storage path. The PR is narrow and still useful, though it should not be treated as the complete fix for the broader linked Google Chat markdown issue.

## Best Possible Solution

Review and land this narrow Google Chat plugin fallback change, or an equivalent main-branch patch, in `extensions/googlechat/src/monitor.ts`. Add a focused regression test for `typingIndicator: "message"` where the routed agent has `identity.name` but no top-level `name`; keep the separate Google Chat markdown/plain-text rendering work tracked independently.
