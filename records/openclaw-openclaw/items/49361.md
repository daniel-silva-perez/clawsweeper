## Summary

Keep open. Current main still does not implement this PR's intended Google Chat typing-indicator fallback: `resolveBotDisplayName` falls from account config name to top-level `agent.name` and then hardcoded `OpenClaw`, while `agents.list[].identity.name` remains a supported agent identity field. The PR is narrow and still useful; the linked issue's markdown-rendering concern should remain separate.

## Best Possible Solution

Review and land the narrow Google Chat plugin fallback change, ideally with a small regression test covering `typingIndicator: "message"` where the routed agent has `identity.name` but no top-level `name`. Track the separate Google Chat markdown/plain-text rendering issue independently.
