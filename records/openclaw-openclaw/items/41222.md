## Summary

Current main still does not fully implement this PR's intended outbound indentation preservation. The directive/code-block side has partially landed, but `splitMediaFromOutput` still rewrites multiline body text after media extraction by trimming/collapsing whitespace, and `parseInlineDirectives` still normalizes no-tag text instead of preserving it byte-for-byte. This PR is also the open implementation candidate for the paired bug #41215, so it should stay open for maintainer follow-up rather than being closed as obsolete.

## Best Possible Solution

Keep this PR open. Refresh or selectively cherry-pick the narrow remaining behavior: preserve meaningful multiline body indentation through `splitMediaFromOutput` after `MEDIA:` and audio tag stripping, decide explicitly whether no-tag `parseInlineDirectives` input must be byte-preserved, add focused regressions for media-extraction and outbound JSON payload text, and resolve the existing directive-edge review comments before merge.
