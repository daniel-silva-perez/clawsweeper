## Summary

Current main still does not fully implement this PR's intended outbound indentation preservation. The directive/code-block side has adjacent fixes, but `splitMediaFromOutput` still collapses retained multiline body whitespace after `MEDIA:` extraction, and `parseInlineDirectives` still normalizes no-tag text instead of preserving it byte-for-byte. This PR is also the open implementation candidate for paired bug #41215, so it should stay open for maintainer follow-up.

## Best Possible Solution

Keep this PR open. Refresh or selectively cherry-pick the narrow remaining behavior: preserve meaningful multiline indentation through `splitMediaFromOutput` after `MEDIA:` and audio tag stripping, decide explicitly whether no-tag `parseInlineDirectives` input must be byte-preserved, add focused regressions for media extraction and outbound JSON payload text, and resolve the existing directive-edge review comments before merge.
