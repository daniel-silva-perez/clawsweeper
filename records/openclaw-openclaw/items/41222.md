## Summary

Keep this PR open. Current main overlaps with the directive-tag side of the change, especially code-block indentation preservation, but it still does not implement the PR's full outbound whitespace-preservation behavior: the media parser continues to collapse indentation after MEDIA extraction, and the no-inline-tag path still intentionally normalizes prose rather than returning text exactly unchanged.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to refresh or cherry-pick the still-missing pieces narrowly: preserve meaningful indentation in `splitMediaFromOutput` after `MEDIA:`/audio tag stripping, add the PR's media-extraction and outbound JSON regression cases, and explicitly decide whether no-directive text should be returned exactly unchanged or continue through prose normalization.
