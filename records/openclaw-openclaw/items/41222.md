## Summary

Keep this PR open. Current main has later directive-tag code-block indentation fixes, but it does not fully implement this PR’s stated outbound whitespace preservation work, especially the `MEDIA:` extraction path and the no-inline-tag fast path behavior described in the PR body.

## Best Possible Solution

Keep the PR open for maintainer review. Either refresh this PR against current main and decide the remaining whitespace contract explicitly, or port the narrow missing pieces: preserve meaningful indentation in `splitMediaFromOutput` after stripping `MEDIA:`/audio tags, add regression coverage for indented text remaining after media extraction, and decide whether no-directive text should be returned exactly unchanged or intentionally normalized.
