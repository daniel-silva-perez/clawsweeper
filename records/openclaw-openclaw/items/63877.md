## Summary

Keep open. Current main still rejects the reported self-hosted Firecrawl configuration: `resolveEndpoint()` requires HTTPS and limits hosts to `api.firecrawl.dev`, and the tests/docs still assert that lock-down. There are open implementation PRs, including #63941 with closing syntax and #74013 with an explicit `allowSelfHosted` approach, so this issue should stay open until a fix merges.

## Best Possible Solution

Keep this issue open and review/land one focused Firecrawl plugin fix, preferably the explicit opt-in shape from #74013 or an equivalent maintainer-approved variant. The fix should add a documented self-hosted/private endpoint config, preserve official cloud HTTPS defaults, keep URL credential/query/hash sanitization, update `openclaw.plugin.json`, docs, and Firecrawl tests, then close this issue after merge/release.
