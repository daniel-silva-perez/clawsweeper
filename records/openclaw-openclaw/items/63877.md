## Summary

Keep open. Current main still rejects the reported local/self-hosted Firecrawl baseUrl forms, the latest release contains the same restriction, and multiple open PRs already target this issue, including #63941 with an explicit implementation link. This should be resolved by reviewing or landing one canonical Firecrawl plugin fix, not by conservative cleanup.

## What This Changes

The issue requests allowing the Firecrawl web search/fetch configuration to use a local or self-hosted Firecrawl baseUrl over HTTP or a custom HTTPS host.

## Best Possible Solution

Keep this issue open and review or land one canonical Firecrawl plugin fix. The best fix should preserve the strict hosted default, add an explicit operator opt-in such as allowSelfHosted, allow custom HTTPS endpoints only under that reviewed policy, allow cleartext HTTP only for verified private/loopback/internal targets via the existing SSRF helper, route trusted private endpoints through the trusted web-tools guard, preserve credential/query/hash sanitization, and update Firecrawl tests, manifest schema/UI hints, docs, and changelog as appropriate.
