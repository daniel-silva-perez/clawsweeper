## Summary

Keep open. Current main still rejects the reported local/self-hosted Firecrawl configuration: Firecrawl endpoint resolution requires HTTPS and only allows api.firecrawl.dev, while both firecrawl_search and firecrawl_scrape still use that resolver. The latest release tag has the same restriction, and related implementation PRs remain open/unmerged, so this issue should stay open for a canonical Firecrawl plugin fix.

## Best Possible Solution

Keep this issue open and review or land one canonical Firecrawl plugin fix. The fix should preserve the strict hosted default, require explicit self-hosted/private endpoint opt-in, allow public custom hosts only over HTTPS, allow cleartext HTTP only for verified private or loopback targets, keep credential/query/hash sanitization, route private endpoints through the appropriate trusted web-tools guard, and update Firecrawl tests, manifest schema, and docs.
