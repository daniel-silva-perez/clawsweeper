## Summary

Keep this PR open. Current main still does not implement the PR's central behavior: the Chat New session control still sends `/new` through the current-session reset path, while the newer `sessions.create` backend primitive exists but is not wired into the Control UI create-and-switch flow. Some adjacent pieces are present, including explicit option selection and backend recency ordering, but the intended independent persisted chat-session workflow remains missing.

## Best Possible Solution

Keep this PR open as a plausible implementation candidate, but update it for current main: wire the Chat New session button to `sessions.create`, switch immediately through the existing session-switch helper, preserve or explicitly test current recency ordering, keep the Lit selector binding stable, and add targeted Control UI tests for creation, switching, duplicate labels, and navigation-return synchronization.
