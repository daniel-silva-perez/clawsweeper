## Summary

Keep this PR open. Current main still does not implement the PR's central create-and-switch behavior: the Chat New session button still sends `/new` through the reset-style chat path instead of creating a distinct persisted session and switching to it. Some adjacent pieces exist, including `sessions.create`, explicit option selection, and backend recency ordering, but the intended Control UI workflow remains missing.

## Best Possible Solution

Keep this PR open as a plausible implementation candidate, but update it for current main: wire the Chat New session button to `sessions.create`, switch immediately through the existing session-switch path, preserve or explicitly test current recency ordering, keep the selector binding stable, and add focused Control UI tests for creation, switching, duplicate labels, and navigation-return synchronization.
