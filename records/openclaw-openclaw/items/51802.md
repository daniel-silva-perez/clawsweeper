## Summary

Keep open. Current main still does not implement this PR's central behavior: the Chat New session button still routes through the `/new` current-session reset path instead of creating and switching to a separate persisted chat session. Main now has a better backend primitive, `sessions.create`, and normal `sessions.list` output is already sorted by `updatedAt`, but the Control UI is not wired to use that create-and-switch flow.

## Best Possible Solution

Keep this PR open as a plausible implementation candidate, but update it for current main: wire the Chat New session button to `sessions.create`, switch immediately via the existing session-switch helper, preserve the current `sessions.list` recency behavior or add focused ordering tests where useful, keep the Lit selector binding stable, and add targeted Control UI tests for creation, switching, duplicate labels, and navigation-return synchronization.
