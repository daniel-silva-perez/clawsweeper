## Summary

Keep this PR open. Current main still does not expose the proposed injected `api.runtime.llm.complete()` Plugin SDK surface, does not export the proposed `PluginLlmComplete*` types, and does not inject an LLM helper into context-engine runtime contexts. The PR remains an implementation candidate for the still-open SDK/API request #45865, but it adds a new trusted model-inference capability and needs maintainer API/security review rather than conservative cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review or replacement. The best path is an additive plugin SDK LLM completion seam that routes through OpenClaw's normal model/auth/provider runtime path, binds clearly to the active agent or an explicit agent, documents permission/cost/observability semantics in `docs/plugins/sdk-runtime.md`, exports stable SDK types, and adds focused runtime/API tests. Treat #45865 as the canonical product/API discussion if maintainers choose a different shape.

## Risks / Open Questions

- The PR creates a new path for plugins and context engines to spend model tokens and use configured model credentials, so maintainers need explicit permission, cost-attribution, observability, and rate-limit semantics before merging.
- The proposed omitted-`agentId` fallback can route completions through the default agent rather than the active session agent in multi-agent setups unless the final API binds the caller explicitly or requires explicit agent binding.
- The proposed public SDK surface still needs docs, API/export baseline coverage, and focused tests for default and non-default agent model/auth-profile resolution.
