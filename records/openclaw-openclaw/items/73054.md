## Summary

Keep open. Current main still has the reported Slack partial-preview default behavior, and the proposed patch is a narrow, supported fix with focused tests, corroborating discussion, and no blocking review findings.

## What This Changes

The PR changes Slack preview streaming so `partial` mode defaults `streaming.preview.toolProgress` off, while `block` and `progress` keep it on and explicit config still overrides the mode default.

## Best Possible Solution

Land a rebased version of this Slack-owned fix that keeps the default decision in the Slack dispatch layer, continues to use the shared helper for explicit override semantics, updates the Slack docs/UI metadata, and keeps the focused mode-default regression tests.

## Reproduction Assessment

Yes. On current main, Slack preview-only `partial` drafts can call the shared helper with its unset default of true, which allows `pushPreviewToolProgress` to edit the draft with `Working…` lines before answer text; the thread also has a user confirming the same regression.

## Solution Assessment

Yes. The PR uses the narrowest maintainable fix: Slack supplies a mode-derived default while preserving explicit `streaming.preview.toolProgress` overrides and leaving progress-oriented `block`/`progress` behavior intact; the safer next step is rebasing to remove unrelated generated drift.
