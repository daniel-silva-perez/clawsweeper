## Summary

Keep open. Current main still has the Slack partial-preview behavior this PR is trying to change: draft previews inherit the shared unset `toolProgress` default of `true`, so the PR remains useful and appears narrowly correct, but it is currently dirty against main and needs normal maintainer review/rebase before merge.

## What This Changes

The PR changes Slack preview streaming so `partial` mode defaults `streaming.preview.toolProgress` off, while `block` and `progress` keep it on, and updates Slack docs, UI hints, generated config metadata, and focused streaming tests.

## Best Possible Solution

Land a rebased version of this Slack-owned fix that keeps the mode-specific default decision in the Slack dispatch layer, preserves the shared helper’s explicit override semantics, updates Slack docs/UI copy, and retains the focused mode-default tests.

## Reproduction Assessment

Yes. On current main, the Slack draft-preview path can reproduce this with `streaming.mode: "partial"`, draft preview active, and `streaming.preview.toolProgress` unset, because dispatch uses the shared helper’s default `true` and then routes tool/progress events into `pushPreviewToolProgress`.

## Solution Assessment

Yes. The PR is the narrowest maintainable solution: Slack supplies a mode-derived default while leaving explicit `streaming.preview.toolProgress` values and progress-oriented `block`/`progress` modes intact; the safer next step is to rebase so unrelated generated drift is resolved against current main.
