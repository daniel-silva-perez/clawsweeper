## Summary

Keep open. Current main has not implemented the PR's central web_fetch proxy behavior: strict web_fetch still goes through the strict guarded-fetch path without setting env-proxy dispatcher policy, and the current regression test explicitly asserts that HTTP_PROXY does not produce an EnvHttpProxyAgent. The PR also has an unresolved review concern about redirect protocol selection and bundles cron delivery changes that already appear present on main, so it should be rebased/split rather than closed as obsolete.

## Best Possible Solution

Keep this PR open for maintainer/contributor follow-up. The best path is a focused web_fetch proxy fix that wires protocol-aware env proxy selection into the strict web_fetch path while preserving explicit dispatcherPolicy overrides, DNS pinning, NO_PROXY behavior, and per-redirect protocol checks; the already-present cron changes should be dropped from the rebased PR or handled separately.
