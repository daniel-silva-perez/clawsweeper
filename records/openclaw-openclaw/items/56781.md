## Summary

Keep open. Current main still has a real feature gap: core compaction accepts and resolves only one compaction model, and its compaction path does not enter the normal model fallback runner. The lossless-claw portion is also not resolved in this repository because that `summaryModel` surface is owned by the external plugin, although current OpenClaw plugin config can carry plugin-owned settings.

## What This Changes

Request to add fallback model chains for OpenClaw compaction and lossless-claw LCM summarization so transient 429, 500, or timeout failures try secondary models.

## Best Possible Solution

Add a first-class, documented core compaction fallback contract that reuses normal model failover semantics for transient errors, with tests proving the compaction runner tries secondary models. Handle lossless-claw summary fallback in the external plugin or through a generic plugin SDK helper if that plugin lacks access to the needed model-fallback primitive.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: on current main, configuring `agents.defaults.compaction.fallbacks` would be rejected or ignored by the strict schema, and the compaction runner resolves one model target without calling `runWithModelFallback`.

## Solution Assessment

Unclear. The requested direction is sound for core compaction, but the exact config shape and the external lossless-claw `summaryModel` ownership need maintainer/API judgment before this should be turned into an automated fix.
