## Summary

Keep open. Current main still has the reported feature gap: core compaction accepts and resolves a single compaction model, and the direct compaction path does not use the shared model fallback runner. The Lossless Claw summary model portion is also not solved in this repository because that is plugin-owned behavior, though OpenClaw already has a generic plugin config surface that can carry plugin settings.

## What This Changes

Request fallback model chains for core compaction and Lossless Claw summary-model selection so transient model failures can try secondary models before compaction or summarization gives up.

## Best Possible Solution

Add a documented core compaction fallback contract that reuses the existing transient-error model failover semantics, with tests proving compaction tries secondary candidates. Keep Lossless Claw summary fallback in the owning plugin unless the plugin needs a generic OpenClaw SDK/runtime helper, in which case add that helper as a plugin-agnostic API.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: `agents.defaults.compaction.fallbacks` is not in the strict schema/type, and the compaction runner resolves one model target without entering `runWithModelFallback`.

## Solution Assessment

Unclear. The requested direction is sound for core compaction, but the exact config shape and the external Lossless Claw summary-model ownership should be settled by a maintainer/plugin owner before an automated repair attempts the change.
