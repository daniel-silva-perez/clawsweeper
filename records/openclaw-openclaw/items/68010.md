## Summary

Keep this PR open. Current main still eagerly resolves the compatibility provider env-var export at import time, and OpenRouter's auth env var is manifest-owned, so the PR targets a real unresolved bug. The branch is not cleanup-closable because its regression test is currently weak and its provider env-var hardening changes touch env secret resolution policy that needs maintainer security review.

## What This Changes

The PR aliases `PROVIDER_ENV_API_KEY_CANDIDATES` to the lazy provider auth env-var proxy, adds a compat lazy-loading regression test, and adds provider env-var candidate hardening in `src/secrets/provider-env-vars.ts`.

## Best Possible Solution

Keep the PR open as the implementation candidate for the unresolved lazy-loading bug, but land only after the compat export delegates to the lazy proxy with a real post-import regression test and the provider env-var security hardening is either split out or explicitly approved with targeted coverage.
