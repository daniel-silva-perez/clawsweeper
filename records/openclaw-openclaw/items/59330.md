## Summary

Current main still has the reported Raw-mode failure path: safe-bin normalization preserves own `undefined` keys, raw redaction still compares restored raw against the materialized runtime config, and the Control UI still disables Raw mode when the snapshot returns `raw: null`. The linked fix PR #59336 is still open and unmerged, so this issue should stay open for maintainer review of that PR or a replacement.

## What This Changes

The issue asks to restore Control UI Raw mode by fixing config snapshot raw redaction round-trip checks that fail against materialized runtime config and `undefined` safe-bin keys.

## Best Possible Solution

Keep this issue open and route it through #59336 or a maintainer-owned replacement. The fix should safely omit empty safe-bin fields without allowing inherited polluted values, compare restored raw text against the authored/source snapshot or a deliberately normalized JSON-serializable source view instead of `snapshot.config`, address substring-based raw redaction corruption, and add focused regression tests for safe-bin empty normalization, source-vs-materialized snapshot differences, and short sensitive values such as `file`.
