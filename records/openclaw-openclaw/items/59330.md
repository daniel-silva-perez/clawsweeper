## Summary

Keep this issue open. Current main and the latest release still contain the reported Raw-mode failure path: snapshots are materialized before redaction, raw round-trip fallback still compares restored raw text against the materialized `snapshot.config`, empty safe-bin normalization still leaves own properties with `undefined` values, and the Control UI still disables Raw mode when redaction returns `raw: null`. The linked implementation PR #59336 remains open and unmerged, so this issue should stay open until that PR or a maintainer replacement lands.

## Best Possible Solution

Keep this issue open and use #59336, or a maintainer-owned replacement, as the implementation path. The fix should delete empty safe-bin keys instead of assigning `undefined`, compare raw restoration against the authored/source snapshot or a deliberately normalized JSON-serializable view rather than the materialized runtime config, and add focused regression coverage for safe-bin normalization, materialized snapshot differences, and the comment-reported substring redaction corruption.
