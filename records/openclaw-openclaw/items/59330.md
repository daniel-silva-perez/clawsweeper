## Summary

Keep this issue open. Current main still contains the reported failure path: safe-bin normalization writes undefined-valued keys into the materialized snapshot, raw redaction still round-trip compares against the materialized config instead of the authored/source config, and Control UI still disables Raw mode whenever redaction returns `raw: null`. The provided GitHub context also shows open implementation PR #59336 with closing syntax for this issue, so the issue should stay open until that PR, or a replacement, lands.

## Best Possible Solution

Keep this issue open and review #59336, or a maintainer-owned replacement PR, as the implementation path. The durable fix should delete empty safe-bin keys instead of assigning `undefined`, compare raw restoration against the authored/source config or otherwise normalize materialization-only diffs safely, avoid global substring replacement corrupting raw keys, and add regression coverage for safe-bin, materialized-config, and substring-redaction fallback cases.
