## Summary

Keep open. Current main still contains the reported false-positive Raw-mode fallback path, and the open implementation PR that closes this issue is unmerged, with a recent user report confirming the latest release is still affected.

## What This Changes

The issue asks OpenClaw to restore Control UI Raw mode by avoiding undefined safe-bin keys and comparing raw redaction round-trips against the authored/source config shape instead of the materialized runtime config.

## Best Possible Solution

Land #59336, #1824, or a maintainer-owned replacement that omits empty safe-bin fields, compares the raw redaction round-trip against a JSON-serializable authored/source config shape, preserves fallback for ambiguous redaction collisions, and adds focused regression tests for the reported cases.
