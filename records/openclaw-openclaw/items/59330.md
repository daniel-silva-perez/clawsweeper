## Summary

Keep open. Current main and the latest release still contain the reported false-positive Raw-mode fallback path, and an open PR with closing syntax is already paired with this issue rather than merged.

## What This Changes

Fix Control UI Config Raw mode being disabled when runtime materialization and undefined safe-bin properties make the raw redaction round-trip comparison fail.

## Best Possible Solution

Land #59336, #1824, or a maintainer-owned replacement that deletes empty safe-bin fields, compares the raw redaction round trip against a JSON-serializable authored/source config shape, preserves fallback for ambiguous redaction collisions, and adds focused regression coverage for the reported configs.

## Reproduction Assessment

Yes. The current code path can be reproduced by producing a snapshot whose runtime config has undefined own safe-bin fields or source/runtime materialization drift, causing `redactConfigSnapshot` to set `raw` to `null`; Control UI then forces Form mode from that null raw snapshot.

## Solution Assessment

Unclear until the open patch is reviewed. The proposed direction is plausible, but the final fix must preserve the redaction safety gate for ambiguous replacements while narrowing the equality target to the authored/source shape.
