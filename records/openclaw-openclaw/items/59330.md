## Summary

Keep this issue open. Current main still has the reported Raw-mode failure path: safe-bin normalization leaves own properties with `undefined`, raw redaction still round-trips against the materialized snapshot config, and the Control UI still disables Raw mode when redaction returns `raw: null`. The linked implementation PR #59336 remains open and unmerged, so the best next step is maintainer review or replacement of that PR rather than closing this issue.

## Best Possible Solution

Keep this issue open and route it through #59336 or a maintainer-owned replacement. The fix should delete empty safe-bin keys instead of assigning `undefined`, compare raw restoration against the authored/source snapshot or a deliberately JSON-serializable normalized view instead of the materialized runtime config, and add focused regression coverage for safe-bin empty normalization, source-vs-materialized snapshot differences, and the comment-reported substring redaction corruption.
