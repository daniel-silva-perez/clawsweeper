## Summary

Keep this PR open. Current main still has the exact no-callback gap the PR targets: WS `thinking` agent events are only emitted after `streamReasoning` is enabled, and `streamReasoning` still requires both `reasoningMode: "stream"` and an `onReasoningStream` callback. The PR is not implemented on main, not clearly superseded by a canonical maintainer-selected item, and its latest diff still needs review for hidden-output suppression before it should land.

## Best Possible Solution

Keep this PR open as a valid implementation candidate. The best path is to decouple WS `thinking` event emission from the messaging `onReasoningStream` callback, while preserving `silentExpected` and deterministic-approval suppression, avoiding the tagged-stream O(n^2) hot path, and adding focused tests for Gateway `chat.send` clients with no callback plus hidden-run non-leak cases. Maintainers should also decide whether this PR or #47613 is the canonical vehicle.
