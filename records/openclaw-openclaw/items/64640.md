## Summary

Keep open. Current main has already superseded the webhooks half through synchronous plugin registration and request-time SecretRef resolution, but the memory-search half is still not fully implemented because provider, fallback, and multimodal lookup stay on a registered-only helper instead of the cfg-aware runtime capability resolver.

## What This Changes

The PR branch forwards active memory-search config into embedding provider lookup, changes the webhooks plugin to register inline-secret routes synchronously, and adds regression tests for both paths.

## Best Possible Solution

Rebase and narrow the branch to the memory-search fix only: use the cfg-aware memory embedding provider runtime resolver for primary, fallback, and multimodal validation lookup, keep the existing #67941 webhooks design, and add a focused regression proving cfg is forwarded into runtime provider resolution without broad plugin activation.
