## Summary

Keep open. Current main still only partially satisfies the request: agent scoping, JSON output, and recent-session filtering exist, but the requested duration-style older/newer filters, size-threshold filter, and byte-size metadata needed for size queries are still absent from the CLI, gateway protocol, and agent tool surface.

## What This Changes

The issue requests richer `openclaw sessions` metadata queries for agent, older/newer age thresholds, size thresholds, JSON output, and combined automation-friendly filters.

## Best Possible Solution

Define the size metric first, then add additive session-list query parameters across the CLI, gateway `sessions.list` protocol, and `sessions_list` tool, carrying matching row metadata into JSON output and docs with focused coverage for duration parsing, gateway filtering, tool forwarding, and CLI examples.
