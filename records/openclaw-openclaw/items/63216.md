## Summary

Keep open. Current main still preserves the reported recovery shape: the reset copy can still fall back to the generic `reserveTokensFloor` hint, embedded context-overflow recovery still uses bounded compaction retries followed by a session reset path, retry attempts still rebuild prompt/bootstrap context, and tool-result truncation can still report `empty session`. Existing changes mitigate related small-context and tool-result cases, but they do not close the whole report.

## What This Changes

The issue asks OpenClaw to investigate and improve repeated context-overflow hard resets on the same session key when reserve-token headroom is already high, especially misleading reset guidance, retry-time bootstrap reinjection, and empty-session truncation diagnostics.

## Best Possible Solution

Define a maintainer-approved overflow-recovery contract that separates model-window misconfiguration from reserve-floor tuning, makes reset guidance budget-aware, and gives retry/post-compaction paths explicit rules for when bootstrap context is re-used, suppressed, reduced, or annotated.
