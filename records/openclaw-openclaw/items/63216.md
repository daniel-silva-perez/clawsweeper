## Summary

Keep open. Current main has partial mitigations for small context-window reserve floors and aggregate tool-result truncation, but it still preserves the reported remaining behavior: the hard-reset text can still default to a reserveTokensFloor suggestion, overflow recovery still retries compaction up to three times, and post-compaction paths still re-inject AGENTS-derived context unless explicitly disabled. There is no open PR closing this issue, and the related reinjection-loop issue #66126 is still open rather than a merged fix.

## Best Possible Solution

Keep this issue open. The best path is a maintainer-scoped fix that makes the hard-reset guidance aware of the effective reserve floor, configured contextWindow/contextTokens, and already-high reserve settings, then deliberately decides whether retry/post-compaction paths should suppress, shrink, dedupe, or mark bootstrap/context reinjection. Keep #66126 as the narrower retry reinjection loop and #55366 as the hard-reset state-loss UX follow-up unless maintainers intentionally merge those scopes.
