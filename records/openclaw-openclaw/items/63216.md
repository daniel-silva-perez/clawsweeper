## Summary

Current main still matches the remaining reported failure shape. Small-context reserve-floor and aggregate tool-result mitigations exist, but the reset hint can still fall back to reserveTokensFloor guidance, overflow recovery still performs bounded prompt retries, tool-result truncation can still report an empty session, and post-compaction AGENTS.md reinjection remains default behavior. The narrower retry-reinjection report #66126 is still open, so this should not be closed as implemented.

## What This Changes

Investigate repeated context-overflow hard resets where high compaction reserve settings, retry compaction, tool-result truncation, and bootstrap context reinjection combine to keep the same session key failing.

## Best Possible Solution

Define a maintainer-approved context-overflow recovery contract that distinguishes effective model budget problems from already-high reserve settings, makes reset guidance budget-aware, and gives retry/post-compaction paths an explicit policy for suppressing, shrinking, deduping, or marking bootstrap reinjection. Keep #66126 focused on retry reinjection and #55366 focused on hard-reset state recovery unless maintainers intentionally merge those scopes.
