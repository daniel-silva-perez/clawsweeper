## What This Changes

The issue asks OpenClaw to apply the plugin allowlist/startup scope during bundled plugin discovery and manifest loading so skipped bundled plugins are not read, validated, or retained at boot.

## Best Possible Solution

Add a conservative cold-start plugin scope before bundled discovery and installed-index manifest loading, thread it through discovery/manifest-registry APIs, and skip out-of-scope bundled directories before package or plugin manifest reads while preserving explicit channel, provider, slot, and config-activation semantics.

## Reproduction Assessment

Yes for the control-flow bug: current main shows gateway startup building the metadata snapshot and manifest registry before `onlyPluginIds` limits runtime loading. I did not reproduce the exact 500 MB heap number in this read-only pass.

## Solution Assessment

Yes, discovery-time scoping is the right direction, but the narrow maintainable fix should derive a conservative startup/plugin scope instead of blindly using `plugins.allow`; explicit channel/provider/slot activation still needs to pull in required bundled plugins.
