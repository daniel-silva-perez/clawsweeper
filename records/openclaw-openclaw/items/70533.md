## Summary

Keep open. Current main still discovers the bundled plugin root and materializes the manifest registry for the full candidate set before applying `onlyPluginIds` or `plugins.allow` scoping; the related #48644 closure addressed an older startup/module-import path, not this discovery-time manifest heap report.

## Best Possible Solution

Keep this issue open and implement scoped cold discovery: derive the effective plugin/startup scope before bundled manifest materialization, pass that scope into discovery/manifest-registry loading, skip known out-of-scope bundled directories before `package.json`/`openclaw.plugin.json`/schema validation where safe, and add regression coverage proving skipped bundled plugin manifests are not read while existing channel/provider/slot activation semantics still work.
