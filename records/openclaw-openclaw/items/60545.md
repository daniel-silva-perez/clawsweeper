## Summary

Keep open. Current main partially covers the PR’s intended behavior for fallback retry prompt preservation and machine-clean `openclaw agent --json`, but it does not fully implement the explicit session-id rollover cleanup or the one-shot sandbox browser-bridge teardown. The PR also has unresolved review-confirmed compile blockers and an availability risk in the sandbox teardown approach, so it should not be closed as implemented or merged unmodified.

## Best Possible Solution

Keep the PR open for maintainer follow-up, but do not merge it as-is. The best path is a rebase that drops or adapts the parts already covered on main, adds a focused explicit-session rollover fix that clears all session-scoped continuity fields, and implements sandbox browser-bridge teardown with ownership/refcounting or one-shot/private-scope gating plus regression tests for overlapping shared-scope runs and TypeScript coverage.

## Risks / Open Questions

- The PR is not merge-ready as submitted: review comments identify TypeScript compile failures from the wrong `./sandbox/context.js` type import and required `SandboxContext.scopeKey` values missing from existing constructors/fixtures.
- The proposed unconditional `disposeSandboxContext(sandbox)` can stop a shared or agent-scoped browser bridge while another run is still using it; teardown needs ownership, refcounting, or one-shot/private-scope gating.
- The session rollover cleanup should clear the complete CLI continuity set, including `cliSessionBindings` and `claudeCliSessionId`, not only `cliSessionIds`.
- Closing this PR as implemented would lose still-relevant work around explicit session-id rollover and sandbox browser-bridge lifecycle cleanup.
