## Summary

Keep open: the gateway reasoning bug still appears valid on current main, but this PR is not mergeable as-is. The branch mixes a focused WebSocket reasoning fix with unrelated scheduled-reset/session-memory work and introduces concrete correctness and security regressions that need maintainer triage or a focused replacement.

## What This Changes

The PR forwards stored session reasoning into embedded Pi runs, relaxes embedded reasoning stream callback gating, and adds scheduled-reset/session-memory metadata handling with tests.

## Best Possible Solution

Land a focused gateway reasoning fix in the owner modules: pass stored `sessionEntry.reasoningLevel` into `runEmbeddedPiAgent()`, allow callback-less gateway `thinking` agent events while preserving `canShowReasoning` and silent-reply gates, and add targeted gateway plus embedded-subscribe coverage. Keep scheduled-reset/session-memory behavior separate from this PR.

## Reproduction Assessment

Yes. A high-confidence source reproduction is: set a WebSocket session to `reasoningLevel: "stream"` via `sessions.patch`, then send through `chat.send`; current main neither forwards stored session reasoning into `runEmbeddedPiAgent()` nor provides `onReasoningStream` from the gateway chat path.

## Solution Assessment

No. The proposed branch addresses the right user-visible gap but is not the best fix because it copies a large command implementation, mixes unrelated session-memory work, and weakens existing trust/visibility gates; a small owner-module patch is safer.
