## Summary

Keep open. The reported Gateway reasoning bug still appears valid on current main, but this PR branch is not safe to merge because it mixes the focused reasoning fix with stale copied agent-command code and unrelated scheduled-reset/session-memory changes that introduce blocking correctness and security regressions.

## What This Changes

This PR forwards stored session reasoning into embedded agent runs, decouples reasoning stream emission from the channel callback, and adds scheduled-reset/session-memory metadata handling with tests.

## Best Possible Solution

Land a focused Gateway reasoning fix in the owner modules: pass stored `sessionEntry.reasoningLevel` into `runEmbeddedPiAgent()`, allow callback-less Gateway `thinking` agent events, preserve `canShowReasoning` and `silentExpected` gates, and keep scheduled-reset/session-memory changes in a separate reviewed change.
