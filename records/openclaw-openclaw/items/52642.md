## Summary

Keep open for maintainer review. Current main still has the gateway no-callback reasoning-stream gap and still does not forward `reasoningLevel` through the agent-command embedded path, but this PR should not merge as-is because it mixes the gateway fix with unrelated session-memory/reset-hook work and replaces the current command barrel with a large copied implementation that risks current ingress trust-boundary behavior.

## What This Changes

The PR decouples embedded Pi reasoning stream emission from `onReasoningStream`, attempts to forward session `reasoningLevel` into embedded agent runs, and also adds unrelated scheduled-reset/session-memory metadata handling while replacing `src/commands/agent.ts` with a copied agent-command implementation.

## Best Possible Solution

Keep this PR open for maintainer handling, but do not merge the branch as-is. The best path is a focused gateway reasoning patch that preserves current `silentExpected`, `canShowReasoning`, and ingress trust guards, emits `stream: "thinking"` gateway agent events when `reasoningMode: "stream"` even without a channel callback, forwards `sessionEntry.reasoningLevel` into embedded agent-command runs where needed, and adds focused regression tests. Keep the unrelated session-memory/reset-hook work separate from this PR.
