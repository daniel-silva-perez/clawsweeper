## Summary

Keep open for maintainer review. Current main still has the reported gateway reasoning gaps, but this PR is not merge-ready because it is based on a stale copied command implementation, mixes unrelated session-memory work, and introduces compile/runtime and ingress trust-boundary regressions.

## What This Changes

This PR changes embedded Pi reasoning stream gating, replaces the `src/commands/agent.ts` barrel with a copied agent command implementation that forwards session reasoning, and adds scheduled-reset/session-memory metadata handling with tests.

## Best Possible Solution

Land a focused gateway reasoning fix in the owner modules: pass stored `reasoningLevel` from the command attempt path into `runEmbeddedPiAgent()`, allow callback-less gateway `emitAgentEvent({ stream: "thinking" })` while preserving privacy/silent gates, add targeted gateway and embedded-subscribe tests, and keep the session-memory work split into a separate reviewed change.

  - body: The copied command implementation seeds `provider` and `model` only from defaults and stored session overrides, and never applies `opts.provider` or `opts.model`. Authorized gateway and local one-shot override runs will silently use the wrong model.
  - confidence: 0.91

  - confidence: 0.9

  - body: Setting `streamReasoning` to `reasoningMode === "stream"` alone drops the current guard that reasoning can be shown. A callback-less WebSocket fix should still honor the thinking/reasoning visibility and silent-reply gates before broadcasting `thinking` events.
  - confidence: 0.82

  - body: The new scheduled-reset branch fires reset hooks before checking `command.isAuthorizedSender`. In shared or group sessions, the first unauthorized post-expiry message could trigger user-visible or side-effecting `command:new` hook automation.
