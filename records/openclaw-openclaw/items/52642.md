## Summary

Keep open. Current main still does not satisfy the gateway WebSocket reasoning behavior described by the PR, but this branch is not mergeable as-is: it is dirty against current main, mixes the gateway fix with scheduled-reset/session-memory changes, and introduces compile/runtime and trust-boundary regressions already visible in the diff and review comments.

## What This Changes

This PR changes embedded Pi reasoning stream gating, replaces the `src/commands/agent.ts` re-export with a copied agent command implementation that forwards stored session reasoning, and adds scheduled-reset/session-memory metadata handling with tests.

## Best Possible Solution

Land a focused gateway reasoning fix in the owner modules: pass stored `sessionEntry.reasoningLevel` into `runEmbeddedPiAgent()`, allow callback-less gateway `thinking` agent events while preserving `canShowReasoning` and silent-reply gates, add targeted gateway/embedded-subscribe coverage, and keep scheduled-reset/session-memory behavior in a separate reviewed change.

Full review comments:

- **[P1] Import reset hooks before the scheduled fallback:** `src/auto-reply/reply/get-reply.ts:341`
  - body: When `scheduledResetTriggered` is true, this branch calls `emitResetCommandHooks()` before importing it; the only import remains in the later manual reset path, so the first scheduled reset would throw instead of flushing hooks.

  - confidence: 0.95
- **[P1] Preserve one-off provider and model overrides:** `src/commands/agent.ts:936-942`
  - body: The copied command implementation seeds provider/model from defaults and stored session overrides but never applies authorized `opts.provider` or `opts.model`, so gateway `agent.create` and local one-shot override runs silently use the wrong model.
  - confidence: 0.91
- **[P2] Require allowModelOverride on ingress:** `src/commands/agent.ts:1255`
  - body: `agentCommandFromIngress()` no longer requires an explicit `allowModelOverride` boolean, weakening the runtime guard that keeps network-facing model overrides opt-in for untyped callers.
  - confidence: 0.9
- **[P2] Keep reasoning broadcasts behind visibility gates:** `src/agents/pi-embedded-subscribe.ts:52`
  - body: Setting `streamReasoning` to `reasoningMode === "stream"` alone drops the established `canShowReasoning` gate; a WebSocket fix still needs to honor reasoning visibility before broadcasting thinking text.
  - confidence: 0.82
- **[P2] Authorize scheduled reset hooks before firing them:** `src/auto-reply/reply/get-reply.ts:340-350`
  - body: The scheduled-reset branch fires reset hooks before checking `command.isAuthorizedSender`, so an unauthorized post-expiry message in a shared session can trigger user-visible or side-effecting reset automation.
  - confidence: 0.84
- **[P2] Pass storePath to fallback transcript lookup:** `src/commands/agent.ts:1074`
  - body: The no-`sessionKey` fallback call to `resolveSessionTranscriptFile()` omits `storePath`, so custom session stores can resolve transcripts from the default location instead of the configured store root.
  - confidence: 0.82
- **[P2] Strip envelope hints before writing memory text:** `src/hooks/bundled/session-memory/handler.ts:85`
  - body: The new memory cleanup removes leading metadata blocks but can still leave OpenClaw envelope hints such as `[message_id: ...]` in stored memory text; the session-memory path should use the same envelope cleanup applied by gateway chat sanitization.
  - confidence: 0.78
