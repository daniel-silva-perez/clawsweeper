## Summary

Keep open for maintainer review: current main and the latest release still have the gateway reasoning-stream gap, but this PR is not merge-ready because it combines that fix with unrelated session-memory reset work and rewrites a security-sensitive command ingress surface.

## What This Changes

This PR decouples embedded Pi reasoning stream emission from channel-specific callbacks, forwards stored session reasoning into embedded agent runs, and adds daily-reset/session-memory metadata handling with tests.

## Best Possible Solution

Land a focused gateway reasoning fix in the existing owner modules: pass stored `reasoningLevel` from the command attempt path into `runEmbeddedPiAgent()`, allow callback-less gateway `emitAgentEvent({ stream: "thinking" })` while preserving `silentExpected` and `canShowReasoning`, keep the command barrel and ingress guards intact, and split the #43524 session-memory work into a separate reviewed change.
