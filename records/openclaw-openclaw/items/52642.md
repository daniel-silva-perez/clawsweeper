## Summary

Keep open: the Gateway reasoning bug is still present on current main, but this PR branch is not mergeable as-is because it mixes the focused Gateway fix with unrelated session-memory/reset work and introduces concrete correctness and security regressions.

## What This Changes

This PR forwards stored session reasoning into embedded agent runs, decouples reasoning stream emission from the channel callback, and adds scheduled-reset/session-memory metadata handling with tests.

## Best Possible Solution

Land a focused Gateway reasoning fix in the owner modules: forward stored `reasoningLevel` from the command attempt path into `runEmbeddedPiAgent()`, let callback-less Gateway runs broadcast `stream: "thinking"` events while preserving `canShowReasoning` and `silentExpected` gates, add targeted Gateway/subscriber tests, and keep scheduled-reset/session-memory work in a separate reviewed change.
