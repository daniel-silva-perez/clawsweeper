## Summary

Keep open for maintainer review. Current main still has the callback-gated Gateway reasoning-stream gap, but this PR is not mergeable as-is because it mixes unrelated session-memory work with the Gateway fix and introduces clear regressions in module loading, per-run model overrides, reasoning visibility gates, and ingress authorization checks.

## What This Changes

This PR decouples embedded Pi reasoning stream emission from `onReasoningStream`, forwards stored session reasoning through a copied agent-command implementation, and adds scheduled-reset/session-memory metadata handling with tests.

## Best Possible Solution

Land a focused Gateway reasoning fix in the owner modules: pass stored `reasoningLevel` into `runEmbeddedPiAgent()` where the agent command path needs it, allow callback-less Gateway `emitAgentEvent({ stream: "thinking" })` while preserving `silentExpected` and `canShowReasoning`, add targeted Gateway/subscriber coverage, and keep the session-memory reset work separate.
