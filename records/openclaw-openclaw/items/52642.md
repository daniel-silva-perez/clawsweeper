## Summary

Keep this PR open for maintainer handling. Current main still has the gateway no-callback reasoning-stream gap, so the intended fix is not implemented on main. The branch should not merge as-is because it mixes the gateway fix with unrelated session-memory/reset-hook work and replaces the current agent-command barrel with a copied implementation that drops current trust-boundary checks.

## Best Possible Solution

Keep this PR open for maintainer action, but do not merge the branch as-is. The best path is a focused gateway reasoning patch that preserves current `silentExpected` and `canShowReasoning` guards, emits `stream: "thinking"` gateway agent events when `reasoningMode: "stream"` even without a channel callback, keeps the optional channel callback for Telegram-style live edits, and adds a regression test for no-`onReasoningStream` gateway emission. Split the session-memory/reset-hook work into its own review path, and consider #47613 as the narrower related fix candidate after rebasing against current main.
