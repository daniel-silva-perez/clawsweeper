## Summary

Keep open for maintainer review: current main still has the Gateway WebSocket reasoning-stream gap, but this branch is not mergeable as-is because it mixes unrelated session-memory/reset work with the Gateway fix and introduces blocking correctness and security regressions.

## What This Changes

This PR decouples embedded Pi reasoning stream emission from `onReasoningStream`, forwards stored session reasoning through a copied agent-command implementation, and adds scheduled-reset/session-memory metadata handling with tests.

## Best Possible Solution

Land a focused Gateway reasoning fix in the owner modules: pass stored `reasoningLevel` into `runEmbeddedPiAgent()`, allow callback-less Gateway `emitAgentEvent({ stream: "thinking" })` while preserving `silentExpected` and visibility gates, add targeted Gateway/subscriber coverage, and keep session-memory reset work separate.
