## Summary

Keep this PR open. Current main still extracts user/assistant text from session-memory transcripts unchanged and writes it directly into memory markdown, so the intended token-sanitization fix is not implemented on main. The PR is recent, focused, and security-hardening; the remaining maintainer decision is whether to land this narrow chat-template-token fix or supersede it with one of the broader open session-memory sanitization PRs.
