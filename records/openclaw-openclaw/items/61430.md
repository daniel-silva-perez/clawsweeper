## Summary

Keep this issue open. Current main still lacks the requested optional exec `description`/`intent` field in the model-facing exec schema, runtime params, exec approval payload, public docs, and approval UI. The related #61562 was closed as a duplicate of this older tracker, so it is not a superseding canonical item.

## Best Possible Solution

Keep the issue open. The likely implementation is an additive optional `description` or `intent` field on the exec tool schema, threaded through exec runtime params, hook-visible params, exec approval request payload/schema, approval UI/audit display, docs, and focused regression tests. It should be documented as informational metadata, not a substitute for command-based safety checks.
