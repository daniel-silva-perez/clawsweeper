## Summary

Keep this issue open: current main has a related mitigation for skipped memory inspection and `status --all`, but it does not yet implement the requested `status --deep` live-gateway memory-provider health path, and an open PR still explicitly proposes to close this issue.

## Best Possible Solution

Keep this issue open and resolve PR #60777 and PR #59685 first. If the intended product behavior is that `status --deep` should trust the live gateway memory provider, land a focused version with regression coverage for human and JSON deep status. If maintainers decide the current `not checked` / `--all` behavior is intentional, close or supersede those PRs and then close this issue with that explicit shipped evidence.
