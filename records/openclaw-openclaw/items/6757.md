## Summary

Keep open. Current main still has manual `/compact`, admin/control-plane `sessions.compact`, automatic/runtime compaction, and plugin/context-engine compaction hooks, but it does not expose the requested in-session agent-triggered self-compaction action through `session_status`, the agent-facing `gateway` tool, or a dedicated `openclaw compact --session ...` command. The latest provided maintainer review also explicitly kept this open, and PR #43401 remains useful prior art but was closed unmerged.

## Best Possible Solution

Keep this issue open until OpenClaw implements a first-class in-session self-compaction action, preferably `session_status(action="compact")` or an equivalent agent tool. The implementation should use direct compaction when already inside a run, preserve session visibility and permission semantics, mark the action as mutating, update compaction counters/token snapshots, and add focused tests for `session_status`, mutation tracking, and active-run behavior. PR #43401 is useful prior art but is not shipped.
