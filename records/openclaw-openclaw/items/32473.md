## Summary

Keep open. Current main still intentionally rejects remote non-localhost plain-HTTP Control UI sessions without device identity, and the browser client still skips device identity generation when `crypto.subtle` is unavailable. The item is also paired with open PR #56830, which uses closing syntax for this issue but only changes messaging, so this should stay open for maintainer review rather than be auto-closed.
