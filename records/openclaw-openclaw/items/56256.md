## Summary

Keep open. Current main has the existing SHA-256 `computeSkillFingerprint` helper, but it does not record skill hashes in `.clawhub/lock.json`, expose `verifySkillIntegrity` / `verifyAllSkillIntegrity`, wire `skills.integrity.*` findings into `openclaw security audit`, or perform load-time integrity checks. The live PR is still open and unmerged, and it remains a plausible security-hardening contribution rather than cleanup-close material, though it is not merge-ready as-is.

## Best Possible Solution

Keep the PR open for maintainer security review. A mergeable path would be a revised core implementation that records install-time hashes for ClawHub workspace skills, defines a compatible lockfile schema, verifies hashes through the intended audit/load surface, handles unreadable or malformed skill directories explicitly, decides the correct hidden-file metadata exclusion policy, documents the resulting `skills.integrity.*` checks, and adds focused tests for install, audit mismatch, missing-hash, and error cases.
