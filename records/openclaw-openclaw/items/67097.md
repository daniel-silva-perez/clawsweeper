## Summary

Keep this issue open. Current main still generates `models.json` by resolving plugin-owned implicit provider catalogs before mode-specific handling, and the Codex and Arcee paths named in the report still exist. Related #66957 covers the narrower `models.mode="replace"` ordering bug, but this issue also needs a maintainer product decision about whether merge/default generated `models.json` should include bundled provider catalogs without explicit onboarding/config intent.

## Best Possible Solution

Keep this issue open for maintainer judgment on the generated `models.json` contract. A concrete path is to short-circuit `models.mode === "replace"` before implicit discovery, keep #66957 as the narrow replace-mode bug if desired, and separately decide whether merge/default generated manifests should document provider-owned bundled catalogs as intentional or gate Codex/Arcee catalog hooks on explicit activation, onboarding, auth, or config intent.
