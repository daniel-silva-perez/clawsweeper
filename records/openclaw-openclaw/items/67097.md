## Summary

Keep open. Current main still generates `models.json` through implicit plugin provider discovery before mode-specific handling, and Codex still contributes a fallback catalog with synthetic auth. Related #66957 covers the narrower `models.mode: "replace"` ordering bug, but this issue still needs a maintainer decision about the default/merge generated-manifest contract.

## What This Changes

The issue asks for generated `models.json` to reflect only providers introduced through onboarding or explicit config, instead of receiving bundled Codex/Arcee provider catalog entries through plugin-owned discovery paths.

## Best Possible Solution

Define the generated `models.json` contract explicitly: keep #66957 focused on short-circuiting implicit discovery for replace mode, then either document provider-owned bundled catalogs as intentional in merge/default mode or gate Codex/Arcee catalog contribution on explicit activation, onboarding, auth, or config intent.
