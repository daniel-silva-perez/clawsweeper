## Summary

Keep open. Current main still lets Telegram persisted activation override an explicit per-topic requireMention setting in the inbound auto-reply path, the public Telegram docs say topic entries override group settings, and open PR #49980 is an unmerged implementation candidate with GitHub closing syntax for this issue.

## Best Possible Solution

Review and either land PR #49980 or replace it with a superseding Telegram plugin patch. The fix should make explicit `topicConfig.requireMention` outrank persisted activation in the inbound auto-reply path, preserve activation fallback when no topic override exists, and add focused regression coverage for topic `requireMention: false` over group `true`, over `groupActivation: "mention"`, and for the no-activation/account-scoped topic ID path reported in the discussion.
