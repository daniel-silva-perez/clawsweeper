## Summary

Keep open. Current main still evaluates Telegram inbound persisted activation before `topicConfig.requireMention`, the docs promise per-topic override inheritance, and open PR #49980 is an unmerged implementation candidate that uses closing syntax for this issue.

## Best Possible Solution

Review and either land PR #49980 or replace it with a superseding Telegram plugin patch. The fix should make explicit `topicConfig.requireMention` outrank persisted activation in the inbound auto-reply path, keep activation as fallback when no topic override exists, and add focused regression coverage for topic `requireMention: false` over group `true`, over `groupActivation: "mention"`, and for the no-activation reproduction path involving topic IDs/account-scoped config.
