## Summary

Keep open. Current main has the Telegram topic config surface and some topic-over-group precedence coverage, but the inbound reply path still evaluates persisted group activation before `topicConfig.requireMention`. More importantly, open PR #49980 explicitly uses closing syntax for this issue and is an unmerged implementation candidate, so this issue should stay open until that PR or a superseding fix is reviewed and resolved.

## Best Possible Solution

Review PR #49980 or a superseding Telegram plugin patch. The likely fix is to make explicit `topicConfig.requireMention` outrank persisted activation in the inbound auto-reply path while preserving activation fallback when no topic override is configured, with focused regression tests for topic `requireMention: false` over group `true` and over persisted `groupActivation: "mention"`. Close this issue only after that PR is merged or maintainers decide to close the implementation path.
