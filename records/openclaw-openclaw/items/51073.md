## Summary

Keep open. Current main has adjacent hardening for tool-shaped assistant text, provider `tool_choice` defaults, trajectory export, and tool-error surfacing, but it still does not implement the core requested guarantee: blocking or warning when an agent claims Google/property/source research without a matching real web_search/browser/tool-backed source trail. The supplied discussion and related items make this still actionable, not stale: #45049 and #49876 remain open for adjacent tool/fabrication failures, while closed #68051 does not supersede this source-verification reproduction.

## Best Possible Solution

Keep this issue open as a core trust/integrity bug until maintainers decide the product boundary. The likely path is to coordinate with #45049 on strict real-tool enforcement, keep #49876 in view for fail-closed delivery behavior, and add a default or configurable source/claim verification layer that checks final claims about searches/sources against actual web_search/browser/tool-call telemetry. Optional policy customization can still live in plugins or ClawHub, but the remaining question is whether OpenClaw core should provide a default guard for unsupported source claims.
