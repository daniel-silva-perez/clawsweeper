## Summary

Keep open. Current main has adjacent hardening for tool-shaped text and proxy tool-choice defaults, but I did not find a built-in source/claim verification gate that would prevent or flag the reported failure: an agent claiming Google/property research without a matching real search/browser/tool-backed source trail. The related open #45049 covers part of simulated tool-call enforcement, and closed #68051 moved optional claim-verification toward ClawHub/plugin scope, but this issue still carries distinct source-verification and self-report integrity evidence.

## Best Possible Solution

Keep this issue open as a core trust/integrity bug until maintainers decide the product boundary. The likely path is to coordinate strict real-tool enforcement with #45049, preserve this issue as the property/source-verification reproduction, and use the plugin/ClawHub path from #68051 only for optional policy layers that do not need default core behavior.
