## Summary

Keep this PR open. Current main still does not implement the proposed VS Code debugging workflow, and the PR remains useful, but the advertised clean rebuild launch has a concrete generated-asset regression that should be fixed before merge.

## What This Changes

The PR adds VS Code Gateway launch/tasks files, an opt-in `OUTPUT_SOURCE_MAPS` tsdown source-map toggle, a `clean:dist` package script, and a VS Code section in the debugging guide.

## Best Possible Solution

Land the DX improvement after the automated launch path uses a supported source-map build flow that preserves or recreates required generated assets, with docs that clearly distinguish one-click debugging from manual `OUTPUT_SOURCE_MAPS=1 pnpm build` workflows.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the introduced regression: use the PR's `Rebuild and Debug Gateway` path, then run `/export-session`; source inspection shows the launch deletes `dist/`, while `run-node` does not recreate `dist/export-html` before the command reads those templates.

## Solution Assessment

Is this the best way to solve the issue? No. The opt-in source-map toggle is a reasonable part of the solution, but deleting all of `dist/` before a narrower `run-node` rebuild is not the narrowest maintainable launch path; an asset-preserving clean, full debug build profile, or purpose-built source-map build task would be safer.
