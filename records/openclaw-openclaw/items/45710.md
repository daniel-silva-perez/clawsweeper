## Summary

Keep this PR open. Current main still does not implement the proposed VS Code debugging path: only `.vscode/settings.json` and `.vscode/extensions.json` are checked in, `tsdown.config.ts` has no source-map toggle, `package.json` has no `clean:dist` script, and `docs/help/debugging.md` has no VS Code launch section. The PR is plausible DX work, but the proposed one-click clean/rebuild flow still needs maintainer review because it deletes `dist/` and then relies on `scripts/run-node.mjs`, which rebuilds a narrower runtime artifact set than `pnpm build`.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a narrow DX change that enables source-map debugging without changing default builds, but the one-click VS Code launch should use a supported build path or a dedicated source-map build profile that preserves generated runtime assets. Documentation should clearly separate the automatic launch path from manual `OUTPUT_SOURCE_MAPS=1` rebuilds and should name any unsupported debug-session surfaces.
