## Summary

Keep this PR open. Current main still lacks the proposed VS Code debugging path, and the PR branch is useful DX work, but the checked-in one-click flow still has a concrete generated-asset regression because it deletes `dist/` and then relies on the narrower `run-node` build path.

## What This Changes

The PR adds VS Code launch/tasks configuration, an opt-in `OUTPUT_SOURCE_MAPS` tsdown sourcemap toggle, a `clean:dist` script, and VS Code debugging documentation.

## Best Possible Solution

Land a narrow DX improvement only after the VS Code launch flow either uses a supported source-map build profile that preserves generated assets or avoids wiping `dist/`; the docs should clearly distinguish the automatic launch path from manual `OUTPUT_SOURCE_MAPS=1` builds and name unsupported surfaces if any remain.

## Reproduction Assessment

Yes for the introduced regression: start from the PR’s clean `dist/` launch path and then use `/export-session`; source inspection shows `run-node` will not recreate `dist/export-html`, while the command reads those templates synchronously. I did not run VS Code because this review was read-only, but the file-path dependency is deterministic.

## Solution Assessment

No. The opt-in sourcemap toggle is a plausible part of the solution, but the current one-click `clean:dist` plus `run-node` flow is not the best maintainable path because it discards generated artifacts that the narrower runtime rebuild does not restore.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve export templates in clean debug launches:** `.vscode/launch.json:8-10`
  - body: The new prelaunch task deletes all of `dist/`, then the launch runs through `pnpm openclaw gateway run`, whose `run-node` rebuild only runs tsdown plus runtime-postbuild. It does not run the full-build `copy-export-html-templates` step, so a gateway started by this advertised flow will throw when `/export-session` reads its template files from `dist/export-html`. Use a supported build profile that preserves those assets, or avoid deleting the whole `dist/` tree before launch.
  - confidence: 0.88
