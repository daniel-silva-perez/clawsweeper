## Summary

Keep this PR open. Current main still drops the direct `anthropic` Claude-name fallback in `resolveGatewayModelSupportsImages`, so the proposed fix remains relevant, but the PR is not merge-ready because the user-facing fix needs a `CHANGELOG.md` entry and maintainer review of the provider-specific gateway heuristic.

## What This Changes

The PR extends the existing Claude-name image-support fallback from `claude-cli` to direct `anthropic` models and adds stale-catalog and missing-catalog regression tests.

## Best Possible Solution

Request the required changelog entry, then either land this focused fallback plus its two regression tests or replace it with a provider-owned image-capability seam that preserves direct Anthropic Claude image attachments when catalog metadata is stale or absent.

## Reproduction Assessment

Yes. A focused resolver check with `provider: "anthropic"`, `model: "claude-opus-4-7"`, and either an empty catalog or a text-only catalog entry follows current main to `false`, which then makes chat attachment parsing treat images as unsupported.

## Solution Assessment

Unclear as a final architecture, but the functional patch is narrow and targets the observed bug. It is not merge-ready as-is because it lacks the required changelog entry, and maintainers may prefer a provider-owned capability seam over adding another provider id to the gateway resolver.
