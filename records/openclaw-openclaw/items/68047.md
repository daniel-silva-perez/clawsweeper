## Summary

Keep this PR open. Current main still applies the Claude-name image-support fallback only to `claude-cli`, while direct `anthropic` remains a documented/default Anthropic path for Opus 4.7. The PR is a narrow implementation candidate with focused tests, and the security pass found no unrelated workflow, dependency, secret, build, or release-script changes.

## What This Changes

The PR extends `resolveGatewayModelSupportsImages` so direct `anthropic` Claude model names share the existing `claude-cli` image-support fallback, and adds stale-catalog and missing-catalog regression tests.

## Best Possible Solution

Review this focused PR against current main and either land the direct `anthropic` Claude-name fallback with its two regression tests, or replace it with a provider-owned capability resolver that preserves images for direct Anthropic Claude models when catalog metadata is stale or absent.
