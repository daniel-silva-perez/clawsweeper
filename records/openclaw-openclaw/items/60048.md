## Summary

Keep open. Current main still lacks the proposed memory deep-dive page and docs navigation entry, so the PR's intended docs change is not implemented on main. The PR remains useful, but it needs revision before merge because its daily-log injection wording conflicts with current docs, runtime behavior, and tests around startup context for bare /new and /reset.

## Best Possible Solution

Keep this PR open as the canonical docs follow-up. Revise the new guide to say daily memory is on-demand on ordinary turns, while recent daily files can be runtime-preloaded for bare /new and /reset via agents.defaults.startupContext; then re-check current memory defaults and proceed through normal docs review.
