## Summary

Keep open. Current main still does not implement systemd sd_notify readiness: generated systemd units have no Type=notify/NotifyAccess, gateway readiness only updates OpenClaw internal state/logs, and direct systemctl restart still returns based on systemctl completion rather than a READY=1 notification.

## Best Possible Solution

Keep this open for a focused Linux daemon/gateway change: add a small systemd notify helper that detects NOTIFY_SOCKET and sends READY=1 only after the existing gateway ready boundary, update generated and documented Linux units to Type=notify with NotifyAccess=main, add unit/startup tests, and coordinate optional WatchdogSec/WATCHDOG=1 work with #11973.
