## Summary

Keep this PR open. Current main still sends Feishu text posts through a raw `md` element, so the reported underscore-URL rendering bug is not implemented on main; the PR is a focused implementation candidate, but it overlaps with the broader open Feishu URL formatter PR and needs maintainer scope selection rather than cleanup closure.

## What This Changes

This PR changes Feishu outbound post payload construction so single bare HTTP(S) URL messages are emitted as explicit rich-text link elements, and adds regression tests for underscore-containing URLs while preserving normal Markdown text.

## Best Possible Solution

Land one Feishu plugin-local URL preservation fix, then close #41860 and explicitly supersede the overlapping Feishu URL PRs. If the target is only the reported CLI URL-only send case, this PR's explicit link-element approach is sufficient after normal rebase and CI review; if embedded bare URLs inside longer Markdown messages are in scope, maintainers should repair and review #68514 as the canonical formatter change.
