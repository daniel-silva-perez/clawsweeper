## Summary

Keep this PR open. Current main still sends Feishu text posts through the raw Markdown-only payload path, so the linked bug is not implemented on main; the PR is a focused plugin-local fix but needs maintainer selection against the broader open Feishu URL approach before merge.

## What This Changes

The PR changes the Feishu post payload builder so a trimmed single bare HTTP(S) URL is emitted as an explicit `a` link element instead of an `md` element, and adds regression tests for underscore URLs and normal Markdown text.

## Best Possible Solution

Land one Feishu plugin-local URL preservation fix, close #41860, and explicitly supersede the overlapping Feishu URL PRs. This PR is the narrow fit for the reported URL-only CLI send case; #68514 or an adapted version is the better end state if maintainers want embedded bare URLs inside longer Markdown messages covered too.

## Reproduction Assessment

Yes for the source-level payload cause: current main routes Feishu text sends through `buildFeishuPostMessagePayload()`, which emits only an `md` element for the reporter's bare URL. The live Feishu rendering step is documented in #41860 with a screenshot, but was not re-run during this read-only review.
