## Summary

Keep open. Current main still sends Feishu text posts through a single Markdown `md` element, so the linked underscore-URL rendering bug is not implemented on main, and this PR remains a focused plugin-local fix with appropriate regression coverage.

## What This Changes

This PR changes Feishu text-post payload construction so single bare HTTP(S) URL messages are sent as explicit `a` link elements and adds regression tests while preserving normal Markdown payloads.

## Best Possible Solution

Land one plugin-local Feishu URL preservation fix, then close #41860 and explicitly close or supersede the overlapping PRs. If maintainers only want the reported URL-only case, this minimal `a` element approach is sufficient; if embedded bare URLs are in scope, choose the broader formatter path from #68514 after its review feedback is resolved.
