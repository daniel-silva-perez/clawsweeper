## Summary

Keep open for maintainer review, but the PR is not merge-ready as-is. Current main already dedupes manifest registry `plugins[]` rows, while this branch adds broader warning-policy changes and a loader lookup change that appears to conflict with existing duplicate-precedence expectations and with the newer maintainer-labeled replacement PR #72409.

## What This Changes

The PR changes plugin manifest duplicate handling to suppress bundled-vs-global/workspace duplicate warnings, adds `recordsByRootDir` to the manifest registry for loader lookups, updates related tests, and adds changelog entries.

## Best Possible Solution

Keep the manifest registry dedupe already present on main, then have maintainers choose one duplicate-warning policy. If bundled shadow warnings should be quieter, rebase a narrow patch on current main that preserves installed-plugin provenance semantics, updates loader/runtime status expectations deliberately, and coordinates with #72409 rather than mixing unrelated provenance-warning changes.

## Reproduction Assessment

Yes. The duplicate-root behavior is covered by `src/plugins/manifest-registry.test.ts` and runtime precedence scenarios in `src/plugins/loader.test.ts`; the review did not run them, but the current tests and PR diff give a clear reproduction path.

## Solution Assessment

No, not as-is. The PR combines a registry ABI change, warning suppression, runtime loader behavior, and unrelated provenance-warning changes; a safer path is a rebased, policy-aligned patch after #72409 is reviewed.

  - confidence: 0.8
