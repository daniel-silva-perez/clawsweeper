## Summary

Close as implemented: current main and released v2026.4.26 stage npm global updates into a temporary prefix and then swap the verified package plus bin shim into the real global prefix, which addresses the reported container EEXIST at /usr/local/bin/openclaw. The earlier --force proposal in PR #65147 and the cross-referenced fork PR are superseded by the shipped staged install path.

## What This Changes

A Docker/npm global update from OpenClaw 2026.4.10 failed with npm EEXIST at /usr/local/bin/openclaw, and the report asks for container package-manager updates to replace the old OpenClaw binary automatically.

## Best Possible Solution

Keep the staged npm global update flow shipped in v2026.4.26. It is narrower than adding npm --force globally, preserves verification and rollback behavior, and leaves any remaining container guidance to the documented updater/reinstaller recovery path.
