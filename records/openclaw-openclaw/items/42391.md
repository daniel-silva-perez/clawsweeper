## Summary

Keep this issue open. Current main still has the requested hook surface, but it does not expose structured progress metrics on `before_tool_call`, `after_tool_call`, or `agent_end`. The related historical items implemented adjacent hook/lifecycle observability, not the numeric progress contract requested here.

## Best Possible Solution

Keep this open. The best path is an additive plugin-facing progress observation contract, either optional `metrics` on the requested hook events or a dedicated progress hook. Emit metrics only when OpenClaw has a defensible step model, document absent/unknown semantics, and add tests for monotonicity, success completion, failure/cancellation behavior, and compatibility with existing plugins.

## Risks / Open Questions

- A numeric progress contract can mislead users unless unknown totals, changing plans, retries, subagents, failures, cancellations, and terminal success semantics are specified.
- This is a public plugin SDK/API change, so it needs typed contracts, docs, and focused runtime/contract tests rather than an ad hoc plugin-only workaround.
