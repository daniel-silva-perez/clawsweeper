## Summary

Keep open. Current main still does not expose structured progress metrics on the requested plugin hook events, and the prior ClawSweeper keep-open analysis remains accurate. The hook types, public plugin API, runtime emitters, tests, and docs still describe textual/timing observation rather than `metrics.stepIndex`, `stepTotal`, `percent`, or `etaSec`.

## Best Possible Solution

Keep this open. The conservative implementation path is to add an additive progress observation contract, either optional `metrics` on the relevant hook events or a dedicated progress hook, emit values only when OpenClaw has a defensible step model, document absent/unknown semantics in the public plugin docs, and add contract/runtime tests for monotonicity, success completion, failure/cancellation, and compatibility with existing plugins.

## Evidence

- **Hook event contracts lack metrics:** `PluginHookAgentEndEvent`, `PluginHookBeforeToolCallEvent`, and `PluginHookAfterToolCallEvent` expose run/message/success/tool/result/error/duration fields, but no `metrics`, `stepIndex`, `stepTotal`, `percent`, or `etaSec` fields.
  - file: [src/plugins/hook-types.ts:272](https://github.com/openclaw/openclaw/blob/bdfb408ce6eeb724a638c106104dbbcae50b612c/src/plugins/hook-types.ts#L272)
  - sha: [bdfb408ce6ee](https://github.com/openclaw/openclaw/commit/bdfb408ce6eeb724a638c106104dbbcae50b612c)
- **Public plugin API uses those hook contracts:** Plugin authors register handlers through `api.on<K extends PluginHookName>(..., handler: PluginHookHandlerMap[K])`, so the missing fields are absent from the public typed plugin API.
  - file: [src/plugins/types.ts:2412](https://github.com/openclaw/openclaw/blob/bdfb408ce6eeb724a638c106104dbbcae50b612c/src/plugins/types.ts#L2412)

## Risks / Open Questions

- A numeric progress contract could mislead users unless unknown totals, changing plans, retries, subagents, failures, and cancellations have explicit semantics.
- This is a public plugin SDK/API surface change, so it needs shared types, docs, and focused runtime/contract tests for compatibility and terminal success/failure behavior.
