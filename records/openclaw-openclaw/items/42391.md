## Summary

Keep this issue open. Current main still exposes the relevant hook events, but the plugin-facing contracts and emitters do not provide structured progress metrics such as stepIndex, stepTotal, percent, or etaSec on before_tool_call, after_tool_call, or agent_end. The related items cover adjacent hook/lifecycle observability, not the numeric progress contract requested here.

## Best Possible Solution

Keep this open for additive plugin-facing progress observation work. The best path is either optional metrics on the requested hook events or a dedicated progress hook, emitted only when OpenClaw has a defensible step model, with typed contracts, public docs, and focused tests for monotonicity, success completion, failure/cancellation behavior, and compatibility with existing plugins.
