## Summary

Keep open. Current main still does not implement the PR's intended Signal group-id allowlist behavior: the Signal handler extracts an inbound groupId but does not pass it into resolveSignalAccessState, and groupAllowFrom is still evaluated through a sender-only Signal phone/UUID comparator. The PR remains a plausible fix candidate, but it needs maintainer review for access-control semantics and the Greptile-noted callback scoping issue before landing.

## Best Possible Solution

Keep this PR open for Signal maintainer review. The best path is a Signal-local fix that passes the inbound groupId into access-policy resolution, compares group ids only for the groupAllowFrom/group path, preserves sender phone/UUID matching only where intentionally supported, adds regression tests for group-id allowlists and DM non-matching, and updates docs/changelog wording so groupAllowFrom semantics are explicit. If #53308's requireMention portion remains unresolved, avoid closing the whole issue from this patch alone.

## Risks / Open Questions

- The proposed diff changes group access-control semantics. If groupAllowFrom remains documented or understood as a sender allowlist, matching the group id directly can broaden triggering from specific senders to anyone in an allowed group, subject to mention gating. Maintainers should align docs/config semantics with the accepted behavior.
- Greptile's review points out that the PR's isSenderOrGroupAllowed callback captures groupId and is reused while computing dmAccess, so a group id in allowFrom could mark dmAccess as allowed for a group-message event. The current handler only consumes dmAccess under the non-group branch, so this does not look like a practical bypass, but the callback should be scoped to the group allowlist path to avoid semantic drift.
- The linked issue #53308 also reports requireMention default behavior. This PR addresses the groupAllowFrom matching portion, so the closing reference should be adjusted or the remaining requireMention behavior should be verified before merge.
