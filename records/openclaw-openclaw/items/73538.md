## Summary

Keep this PR open. Current main still routes Signal group allowlist checks through `isSignalSenderAllowed` without passing the inbound Signal `groupId` into `resolveSignalAccessState`, so the PR’s intended group-id allowlist behavior is not implemented on main. The linked bug report remains open, and this PR is still a plausible Signal-local implementation candidate rather than conservative cleanup material.

## Best Possible Solution

Keep this PR open for Signal maintainer review. The best path is to land or revise a Signal-local fix that compares `groupAllowFrom` against the inbound `groupId` only for the group access path, preserves intended sender allowlist compatibility, adds regression tests for group-id allowlists and DM non-matching, and narrows the linked issue/closing wording if the `requireMention` portion remains outside the patch.
