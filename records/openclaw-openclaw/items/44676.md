## Summary

Keep this PR open. Current main still does not implement the PR's central fail-closed behavior for unrecoverable malformed Kimi tool-call JSON: the helper only replays cached repaired arguments, preserves preexisting streamed arguments when reevaluation fails, and has no cleared-index/final-result clearing path at `toolcall_end`. The PR also conflicts with shipped preservation behavior from #54491, so this needs maintainer policy/security judgment and a rebase onto the current split helper file rather than conservative cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to decide the Kimi policy explicitly: whether unrecoverable malformed final JSON should always clear all dispatch/final snapshots, or whether the shipped #54491 preservation behavior should remain. If fail-closed is desired, rebase the implementation into `src/agents/pi-embedded-runner/run/attempt.tool-call-argument-repair.ts` and add focused coverage for event `toolCall`, `partial`, `message`, and `stream.result()` final snapshots without regressing valid JSON, prefixed repair, cached-repair invalidation, and openai-completions transport coverage.
