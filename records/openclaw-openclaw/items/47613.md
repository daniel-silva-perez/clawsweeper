## Summary

Keep open. Current main still has the callback gate that this PR is trying to remove, so the proposed behavior is not implemented yet; however, the change overlaps other open reasoning-stream PRs and needs maintainer review before landing because it changes live reasoning visibility for Gateway clients.

## What This Changes

The PR changes `src/agents/pi-embedded-subscribe.ts` so `streamReasoning` and `stream: "thinking"` agent-event emission no longer require `onReasoningStream`, while making the callback invocation optional.

## Best Possible Solution

Land one canonical fix that decouples Gateway `stream: "thinking"` emission from channel-specific callbacks while preserving explicit stream mode, thinking-enabled, and silent-turn gates; add a no-callback Gateway/WebSocket regression test and update the reasoning visibility docs if Gateway streaming becomes supported.
