## Summary

Keep open. Current main still gates reasoning-stream emission on `onReasoningStream`, while Gateway chat dispatch does not provide that callback, so this PR is not obsolete. It should get maintainer review because overlapping open PRs cover the same Gateway reasoning gap and the accepted fix needs to preserve current thinking/silent gates and update the public contract if Gateway streaming is supported.

## What This Changes

The PR changes `src/agents/pi-embedded-subscribe.ts` so `reasoningMode: "stream"` can broadcast `stream: "thinking"` agent events without requiring `onReasoningStream`, while still calling that callback when present.

## Best Possible Solution

Land one canonical Gateway reasoning-stream implementation that decouples `emitAgentEvent` from channel-specific callbacks, preserves explicit stream-mode, thinking-enabled, and silent-turn gates, adds no-callback WebSocket regression coverage, and updates `docs/tools/thinking.md` if Gateway streaming becomes supported.
