## What This Changes

The PR changes `src/agents/pi-embedded-subscribe.ts` so `reasoningMode: "stream"` can broadcast `stream: "thinking"` agent events without requiring `onReasoningStream`, while still invoking that callback when present.

## Best Possible Solution

Land one canonical Gateway reasoning-stream implementation that decouples `emitAgentEvent` from channel-specific callbacks, preserves explicit stream mode, thinking visibility, and silent-turn gates, adds no-callback Gateway/WebSocket regression coverage, and updates the reasoning docs if Gateway streaming becomes supported.

Yes. A high-confidence source reproduction is to set a Gateway session to `reasoningLevel: "stream"` and send through `chat.send`: the Gateway path does not pass `onReasoningStream`, and current main returns before emitting `stream: "thinking"`.

## Solution Assessment

No. The PR direction is useful, but this exact patch is not the best merge path against current main because it would need a careful rebase preserving `canShowReasoning` and focused Gateway coverage; maintainers should pick one canonical overlapping PR or request a narrow replacement.
