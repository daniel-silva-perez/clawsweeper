## Summary

Keep open. Current main still requires an `onReasoningStream` callback before it enters the reasoning stream path or emits `stream: "thinking"`, while the Gateway chat path does not provide that callback. The PR addresses a real remaining gap, but it overlaps open PRs #52642 and #61469 and needs maintainer choice on the live-reasoning Gateway visibility boundary.

## What This Changes

The PR changes `src/agents/pi-embedded-subscribe.ts` so stream-mode reasoning no longer depends on `onReasoningStream`, emits `stream: "thinking"` agent events for Gateway clients, and calls the channel callback only when present.

## Best Possible Solution

Land one canonical Gateway reasoning-stream implementation that decouples `emitAgentEvent` from channel-specific callbacks, keeps explicit stream-mode, thinking-enabled, and silent-turn gates, adds no-callback WebSocket regression coverage, and updates the reasoning docs if Gateway streaming becomes supported.
