## Summary

Keep open for maintainer review. Current main still has the no-callback Gateway reasoning-stream gap, and this PR is a focused implementation candidate, but it overlaps other open PRs and needs a rebase that preserves current reasoning-visibility gates before it can be considered merge-ready.

## What This Changes

The PR changes `src/agents/pi-embedded-subscribe.ts` so `reasoningMode: "stream"` can broadcast `stream: "thinking"` agent events without requiring `onReasoningStream`, while still calling that callback when present.

## Best Possible Solution

Land one canonical Gateway reasoning-stream implementation that decouples `emitAgentEvent` from channel-specific callbacks, preserves explicit stream-mode, thinking-enabled, and silent-turn gates, adds no-callback WebSocket regression coverage, and updates `docs/tools/thinking.md` if Gateway streaming is now supported.
