## Summary

Keep open. Current main still gates reasoning streaming on `params.onReasoningStream`, so Gateway/WebSocket paths without that channel callback still cannot emit `stream: "thinking"` events. The PR is relevant, but it overlaps open PRs #52642 and #61469 and needs maintainer dedupe, rebase, and regression coverage before merge.

## What This Changes

The PR modifies `src/agents/pi-embedded-subscribe.ts` so `streamReasoning` depends on `reasoningMode === "stream"`, emits `stream: "thinking"` agent events without requiring `onReasoningStream`, and calls the callback only when provided.

## Best Possible Solution

Keep this PR available for maintainer review or use it as the focused basis for the fix. The best implementation is to decouple `emitAgentEvent({ stream: "thinking" })` from channel-specific `onReasoningStream`, preserve `canShowReasoning` and `silentExpected` gating, call `onReasoningStream` only when provided, add a no-callback Gateway/WebSocket regression test, and dedupe with #52642 and #61469 before landing.
