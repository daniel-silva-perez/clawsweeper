## Summary

Keep open. Current main still has the callback-gated reasoning stream path this PR targets, so it is not obsolete; however, the patch is not merge-ready because rebasing its `streamReasoning` change against current main would drop the newer `canShowReasoning` visibility gate.

## What This Changes

The PR changes `src/agents/pi-embedded-subscribe.ts` so `reasoningMode: "stream"` can broadcast `stream: "thinking"` agent events without requiring `onReasoningStream`, while still calling that callback when present.

## Best Possible Solution

Land one canonical Gateway reasoning-stream implementation that decouples `emitAgentEvent` from channel-specific callbacks, preserves explicit stream-mode, thinking-enabled, and silent-turn gates, adds no-callback WebSocket regression coverage, and updates `docs/tools/thinking.md` if Gateway streaming is supported.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a Gateway `chat.send` run for a session with `reasoningLevel: "stream"`: the Gateway path does not pass `onReasoningStream`, and current main returns before emitting `stream: "thinking"`.

## Solution Assessment

No. The direction is useful, but this exact patch is not the best fix against current main because it removes the callback requirement by making `streamReasoning` depend only on stream mode; the safer fix removes only the callback dependency while preserving `canShowReasoning` and `silentExpected` behavior.
