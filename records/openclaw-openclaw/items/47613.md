## Summary

Keep this PR open. Current main still has the no-callback reasoning-stream gap for Gateway/WebSocket clients: `streamReasoning` is still gated on `params.onReasoningStream`, and `emitReasoningStream()` still returns before `emitAgentEvent({ stream: "thinking" })` when that callback is absent. The PR is relevant, but it needs rebase/test coverage and dedupe with the overlapping open PRs #52642 and #61469.

## Best Possible Solution

Keep this PR available for maintainer review or use it as the focused basis for the fix. The best implementation is to decouple `emitAgentEvent({ stream: "thinking" })` from channel-specific `onReasoningStream`, keep `canShowReasoning` and `silentExpected` protections, call `onReasoningStream` only when provided, and add a regression test proving the Gateway/WebSocket no-callback path receives thinking events. Dedupe this with #52642 and #61469 before landing.
