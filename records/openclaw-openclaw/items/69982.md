## Summary

Keep this PR open, but do not merge it as-is. Current main still has the delete-confirm document listener leak, and the PR fixes the Cancel/Delete paths, but it leaves the existing-popover toggle path removing the popover without unregistering the captured document click listener.

## What This Changes

The PR changes `ui/src/ui/chat/grouped-render.ts` to add a shared delete-confirm popover dismissal helper that removes the popover and unregisters the document click listener for Cancel and Delete dismissals.

## Best Possible Solution

Update this PR or land an equivalent narrow patch that routes every delete-confirm dismissal path through one cleanup routine, including Cancel, Delete, outside click, and clicking the delete button again while the popover is open. Add a focused regression test that spies on `document.addEventListener` and `document.removeEventListener`; no docs or changelog entry is needed for this internal UI leak fix.

## Reproduction Assessment

Yes. Current main has a direct code-backed reproduction by opening the delete confirmation and dismissing it with Cancel/Delete, and the PR still leaves a reproduction by opening it, waiting for the `requestAnimationFrame` listener registration, then clicking the same delete button again to close it.

## Solution Assessment

No, not as written. The shared helper is the right direction, but it needs to cover the existing-popover toggle branch too; otherwise the PR only partially fixes the listener leak it targets.
