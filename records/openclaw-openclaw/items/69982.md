## Summary

Keep this PR open. Current main still has the document click listener leak in the delete-confirm popover: Cancel and Delete remove the popover but do not unregister the captured document listener. This PR is narrow, changes only the affected Control UI file, and the security pass found no CI, dependency, lockfile, script, package-publishing, secret, or artifact-download changes.

## Best Possible Solution

Keep this PR open for maintainer review and land it or an equivalent current-main patch. The fix belongs in `ui/src/ui/chat/grouped-render.ts`: route every delete-confirm dismissal path through one helper that removes both the popover and the captured document click listener. A small regression test in `ui/src/ui/chat/grouped-render.test.ts` that spies on document listener add/remove for Cancel and Delete would make the fix sturdier; no docs or changelog entry is necessary for this internal UI leak fix.
