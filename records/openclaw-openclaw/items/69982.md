## Summary

Keep open. Current main still has the listener leak: Cancel and Delete remove the popover without removing the captured document click listener, while the PR's one-file patch addresses exactly that path. The patch is narrow, CI was green on the PR SHA, and the security pass found no workflow, dependency, lockfile, script, or package-publishing changes.

## Best Possible Solution

Keep this PR open and have a maintainer review, refresh, and land it or an equivalent current-main patch. The fix belongs in `ui/src/ui/chat/grouped-render.ts`: route every delete-confirm dismissal path through one helper that removes both the popover and the captured document listener, ideally with a focused grouped-render regression test.
