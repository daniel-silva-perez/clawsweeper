## Summary

Keep open. Current main still implements the Feishu broadcast-only `@all`/`@_all` suppression that the report challenges, and the behavior is covered by source, tests, and docs rather than being obsolete or already fixed. The related open #74817 overlaps, but it is broader and mixes in a separate multi-bot echo-loop request, so this narrower channel-policy dispute should stay available for maintainer judgment.

## What This Changes

The issue asks OpenClaw to stop dropping Feishu broadcast `@all` mentions at the channel/API layer and preserve them for prompt-level or configurable handling.

## Best Possible Solution

Represent Feishu broadcast mentions as a distinct channel fact that can be included in agent context without automatically satisfying the bot-mention gate, then let maintainers choose prompt or config semantics and update Feishu tests/docs together.

## Reproduction Assessment

Yes. A focused fixture with group text `@_all` or mention metadata `{ key: "@_all", id.open_id: "all" }` produces `mentionedBot=false`; with group `requireMention=true`, the handler returns before dispatch.

## Solution Assessment

Unclear. The requested revert is not obviously the safest solution because it could restore the prior broadcast false positive; a narrower design would preserve broadcast context separately from the boolean direct-bot-mention gate.
