## Summary

Keep open. Current main still intentionally suppresses Feishu broadcast-only `@all`/`@_all` as a bot mention and the issue is asking maintainers to reconsider that channel policy, not reporting behavior that is already implemented. The related open #74817 overlaps, but it is broader and mixes in multi-bot echo-loop behavior, so this narrower policy dispute remains useful.

## What This Changes

The issue asks OpenClaw to preserve Feishu broadcast `@all`/`@_all` mentions for prompt-level or configurable handling instead of hard-coded channel suppression.

## Best Possible Solution

Represent Feishu broadcast mentions as a distinct channel fact that can be included in agent context without automatically satisfying the direct bot-mention gate, then let maintainers choose prompt or config semantics and update Feishu tests and docs together.

## Reproduction Assessment

Yes. A focused fixture with group text `@_all` or mention metadata `{ key: "@_all", id.open_id: "all" }` produces `mentionedBot=false`; with group `requireMention=true`, the handler returns before dispatch.

## Solution Assessment

Unclear. The requested revert is not obviously the safest solution because it can restore the prior broadcast false positive; preserving broadcast context separately from the boolean direct-bot-mention gate is the narrower maintainable direction.
