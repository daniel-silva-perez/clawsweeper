## Summary

Keep this PR open. Current main still silently disables an explicit `pty: true` request whenever exec resolves to the sandbox path, and I found no current warning branch, regression test, docs update, merged duplicate, or release evidence that makes this PR obsolete. The linked #38601 remains the same unresolved exec-tool UX bug.

## Best Possible Solution

Keep this PR open or land an equivalent narrow exec fix: push a warning onto the existing exec `warnings` path when `params.pty === true && sandbox`, ensure it appears in both running and foreground result text, add regression coverage for sandbox and non-sandbox cases, and optionally clarify the public exec docs at `docs.openclaw.ai/tools/exec`.
