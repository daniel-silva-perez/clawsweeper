## Summary

Keep this PR open. Current main still suppresses an explicit `pty: true` request whenever exec resolves to the sandbox path, and there is still no sandbox-PTY warning branch, docs clarification, merged duplicate, or shipped implementation that makes the PR obsolete. The linked bug remains open and the related duplicate PR was closed unmerged.

## Best Possible Solution

Keep this PR open or land an equivalent narrow fix in `src/agents/bash-tools.exec.ts`: push a warning when `params.pty === true && sandbox`, ensure it flows through both running and foreground exec results, add focused regression coverage for sandbox and non-sandbox PTY cases, and consider a short docs note on the public exec page.
