## Summary

Current main still does not implement the requested per-agent daily/monthly cost budget enforcement at the gateway/model-dispatch boundary. The issue remains actionable, has recent production-context comments, and the related #38248 tracks a narrower per-hour/failover ceiling rather than the full per-agent budget/status/doctor scope.

## Best Possible Solution

Keep this issue open as the broader core/gateway budget-enforcement request. The best path is to design a documented budget config under agent defaults and per-agent entries, add durable per-agent spend accounting or reservation before every model attempt including fallback/subagent paths, define warn/block/pause or degradation semantics, expose status and doctor output, and cover gateway admission, fallback, concurrency, missing pricing metadata, and reset-window behavior in tests. If maintainers decide this should be externalized, first add a blocking pre-model-call plugin/API hook with budget status projection so a plugin can enforce the same boundary reliably.
