## Summary

Keep this issue open. Current main still has cost visibility, provider pricing metadata, usage summaries, and some plugin/proxy workaround paths, but it does not define per-agent daily/monthly spend budgets or enforce a pre-model-call budget gate at the Gateway across chat.send, cron/heartbeat, fallback attempts, and subagents. The prior discussion is useful: ATXP, nornr.com, and vibe-billing show external approaches, but they do not make the OpenClaw core request obsolete.

## Best Possible Solution

Keep this open as a core/gateway guardrail request. The best path is to add a documented agents.defaults and agents.list[] budget config, durable per-agent spend accounting or reservation before each model attempt/fallback, explicit warn/block/pause behavior, status and doctor projection, and regression coverage across chat.send, cron, heartbeat, subagents, provider fallback chains, concurrent runs, and missing pricing metadata. If maintainers want plugin-first, first add or document a blocking pre-model-call budget API plus session/status projection so an external plugin can enforce the same boundary reliably.
