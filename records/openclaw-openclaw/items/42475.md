## Summary

Current main still does not implement per-agent daily/monthly cost-budget enforcement at the gateway/model dispatch boundary. The issue remains concrete and actionable, with recent production-context comments, and related #38248 is narrower rather than a canonical replacement.

## Best Possible Solution

Keep this issue open as the broader core/gateway budget-enforcement request. The best path is to design a documented budget contract under agent defaults and per-agent entries, add durable pre-dispatch spend accounting or pessimistic reservation across gateway admission, model fallback, and subagent/delegation paths, define warn/block/pause or degradation semantics, surface status and doctor output, and cover missing pricing metadata, concurrency, reset windows, and structured budget-exhausted errors in tests. If maintainers choose a plugin direction instead, first add a blocking pre-model-call plugin/API hook that can enforce the same boundary reliably.
