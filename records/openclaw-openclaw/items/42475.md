## Summary

Keep open. Current main still provides usage and cost visibility, but not per-agent daily/monthly budget config or a gateway admission check before model and fallback dispatch. The external approaches discussed in comments are useful prior art, and #38248 is related, but it is narrower and still open.

## Best Possible Solution

Keep this open as a core/gateway guardrail request. The strongest path is a documented agents.defaults and agents.list[] budget config, durable per-agent spend accounting or pessimistic reservations before every model attempt and fallback, explicit warn/block/pause or degradation semantics, status and doctor projection, and regression coverage across chat.send, auto-reply, heartbeat, cron, subagents, fallback chains, concurrency, and missing pricing metadata. If maintainers choose plugin-first, add a blocking pre-model-call budget API plus status/session projection first.

## Risks / Open Questions

- Runaway model spend remains possible because the gateway/runtime can accept and dispatch model attempts without a cost budget decision.
- A plugin-only fix would need a new blocking pre-model-call or reservation API; current hooks can observe or route but do not consistently enforce spend before provider calls.
- #38248 may be a consolidation target, but it is narrower (per-hour/failover ceiling) and open, so closing this broader daily/monthly per-agent budget request would lose scope.
