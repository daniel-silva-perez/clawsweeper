## Summary

Keep this issue open. Current main still provides usage/cost visibility and external proxy-style spend-limit options, but it does not define per-agent daily/monthly cost budget config or enforce a pre-model-call spend gate at the Gateway/runtime boundary. The related comments are useful prior art, and #38248 is related, but it is a narrower open per-hour/failover ceiling request rather than a complete duplicate of this broader per-agent budget proposal.

## Best Possible Solution

Keep this open as a core/gateway guardrail request. The best implementation path is a documented agents.defaults and agents.list[] budget config, durable per-agent spend accounting or reservation before every model attempt and fallback, explicit warn/block/pause semantics, status and doctor projection, and regression coverage across chat.send, auto-reply/heartbeat, cron, subagents, fallback chains, concurrency, and missing pricing metadata. If maintainers choose plugin-first, first add a blocking pre-model-call budget API plus status/session projection so an external plugin can enforce the same boundary reliably.
