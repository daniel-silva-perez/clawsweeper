## Summary

Keep open. Current main still lacks the PR's historical replay suppression: embedded subscribe initializes no preexisting-message or replay-window state, subscribes directly, and the dispatcher/assistant handlers do not filter preexisting assistant messages. The PR is a small source/test-only implementation candidate, but it needs maintainer review of the replay contract before merge because the proposed guard depends on how initial replay is delivered.

## What This Changes

The PR snapshots existing embedded session messages, suppresses initial replay and preexisting assistant-message delivery, and adds a regression test that only fresh assistant replies are emitted after subscription.

## Best Possible Solution

Land a narrow embedded subscribe fix after maintainer review confirms the replay semantics: snapshot `session.messages`, suppress historical replay/preexisting assistant delivery, preserve fresh assistant replies, and keep focused regression coverage.
