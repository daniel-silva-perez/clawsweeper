## Summary

Keep open. Current main still lacks the PR's initial-replay/preexisting-message suppression and the matching regression test, so this PR is not obsolete. It remains a narrow implementation candidate, but maintainer review should confirm the replay contract because the current pinned `@mariozechner/pi-coding-agent` `AgentSession.subscribe()` implementation only registers listeners and does not itself replay existing messages.

## What This Changes

The PR adds embedded subscribe state to snapshot existing session messages, suppresses events during initial replay and for preexisting assistant messages, and adds a regression test that historical assistant replies are not emitted again before a fresh reply.

## Best Possible Solution

Keep this PR open as the current implementation candidate, but require maintainer review of the replay contract before merge. If accepted, rebase onto current main, remove or justify the redundant inner replay guard, preserve the `preexistingMessages` protection for known message objects that resurface after subscribe, and run the targeted regression plus the changed agent lane.
