## Summary

Current main still lacks a Control UI per-assistant-message print action. Assistant message actions are still limited to expand-to-canvas and copy-as-markdown, the adjacent helper only writes to the clipboard, and targeted searches found no Control UI `window.print`, single-message print document, or print CSS path. The request is narrow, valid, and a good focused fix-PR candidate.

## Best Possible Solution

Implement this as a Control UI-only chat feature: add a print button beside the existing assistant message expand/copy actions, render only the selected assistant message through the existing sanitized markdown path into an isolated print document or print-only view, call the browser print dialog, update action styling for three controls, add focused UI tests, and add a user-facing changelog entry crediting @Hag-Fish.
