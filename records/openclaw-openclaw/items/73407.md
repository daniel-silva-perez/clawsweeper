## Summary

Keep this open. Current main still ships the WhatsApp plugin path from `@openclaw/whatsapp` to Baileys and the pinned GPL-3.0 libsignal package, and the latest release tag still contains the same lockfile path; this is a real supply-chain/license posture question, not conservative cleanup.

## What This Changes

The issue reports that the WhatsApp plugin brings in a GPL-3.0 libsignal component through its dependency stack and asks maintainers to address the license-compliance risk.

## Best Possible Solution

Maintain an explicit license posture for `@openclaw/whatsapp`: replace or remove the GPL-3.0 dependency path, isolate/disable the affected distribution route, or document an approved distribution model with accompanying dependency and release metadata changes.
