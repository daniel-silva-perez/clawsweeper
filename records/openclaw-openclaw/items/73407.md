## Summary

Keep this open: current main and the latest release still contain the WhatsApp plugin dependency path from Baileys to the pinned GPL-3.0 libsignal package, and resolving that is a maintainer/legal distribution decision rather than conservative cleanup.

## What This Changes

The issue asks OpenClaw to address a reported GPL-3.0 libsignal dependency risk introduced through the WhatsApp plugin dependency chain.

## Best Possible Solution

Keep this open until maintainers choose a license posture for the WhatsApp plugin, then apply the corresponding narrow change: replace or isolate the GPL-3.0 dependency path, disable/remove the affected distribution path, or document an approved distribution model.

Yes. Static inspection of current main and `v2026.4.29` package metadata, lockfile resolution, upstream pinned package metadata, and the WhatsApp runtime Baileys call path verifies the issue without a live WhatsApp account.
