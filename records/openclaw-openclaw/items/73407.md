## Summary

Current main still has the reported dependency path: the supported WhatsApp plugin depends on Baileys, Baileys depends on the WhiskeySockets libsignal-node fork, and the exact pinned fork commit declares GPL-3.0. This is a concrete licensing/compliance question on a maintained plugin surface, so it should remain open for maintainer review rather than being swept closed.

## Best Possible Solution

Keep this issue open and resolve it through an explicit maintainer decision for the WhatsApp plugin: replace or isolate the GPL-3.0 libsignal dependency, document and approve the license posture for `@openclaw/whatsapp`, or temporarily disable/remove the published WhatsApp plugin path until compliance is clear.
