## Summary

Keep open. Current main still publishes an install-on-demand WhatsApp plugin that depends on Baileys, and the lockfile resolves Baileys' libsignal dependency to the whiskeysockets/libsignal-node fork. Upstream metadata for the exact pinned fork commit declares GPL-3.0 and ships a GPLv3 LICENSE, so the reported dependency-license concern is concrete and current rather than stale or non-reproducible.

## Best Possible Solution

Keep this issue open and resolve it through an explicit maintainer decision for the WhatsApp plugin: either replace or isolate the GPL-3.0 libsignal dependency with a license-compatible alternative, document and approve the license posture for `@openclaw/whatsapp`, or temporarily remove/disable the published WhatsApp plugin path until the compliance position is clear.
