## Summary

Current main still carries the reported WhatsApp dependency chain: the supported `@openclaw/whatsapp` plugin depends on Baileys, the lockfile resolves Baileys to the WhiskeySockets libsignal-node fork, and that exact upstream fork commit declares GPL-3.0. This remains an actionable licensing/compliance question for a documented, published plugin surface, so it should not be auto-closed.

## Best Possible Solution

Keep this issue open for an explicit maintainer compliance decision on the WhatsApp plugin: replace or isolate the GPL-3.0 libsignal dependency, document and approve the license posture for `@openclaw/whatsapp`, or temporarily disable/remove the published WhatsApp plugin path until the dependency posture is clear.
