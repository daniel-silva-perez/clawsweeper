Keep this open: current main and the latest release still contain the WhatsApp plugin dependency path from Baileys to the pinned GPL-3.0 libsignal package, and resolving that is a maintainer/legal distribution decision rather than conservative cleanup.

## What This Changes

The issue asks OpenClaw to address a reported GPL-3.0 libsignal dependency risk introduced through the WhatsApp plugin dependency chain.

## Best Possible Solution

Keep the issue open until maintainers choose and apply a license posture for the WhatsApp plugin: replace or isolate the GPL-3.0 libsignal dependency, disable/remove the affected distribution path, or explicitly document an approved distribution model.
