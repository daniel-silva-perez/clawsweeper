## Summary

Keep open. Current main still ships the WhatsApp plugin as an install-on-demand, published plugin, and that plugin still depends on Baileys. The current lockfile resolves Baileys' `libsignal` dependency to the WhiskeySockets `libsignal-node` GitHub tarball at commit `1c30d7d7e76a3b0aa120b04dc6a26f5a12dccf67`; upstream package metadata for that exact commit declares `GPL-3.0` and its LICENSE is GPLv3. The prior ClawSweeper comment correctly identified this dependency path, and the current main SHA has not removed, replaced, disabled, or documented approval for it.

## Best Possible Solution

Keep this issue open for an explicit WhatsApp plugin compliance decision: replace the Baileys/libsignal path with a license-compatible alternative, isolate or disable the published install path until resolved, or document a maintainer-approved license posture for `@openclaw/whatsapp`.
