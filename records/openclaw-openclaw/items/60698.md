## Summary

Keep open. Current main still does not implement per-agent Bedrock requestMetadata, and the PR is relevant to the linked cost-attribution request, but this branch should not merge as-is because it can send invalid Bedrock metadata values and it regresses the existing provider-wrapper precedence contract.

## What This Changes

The PR adds automatic Bedrock requestMetadata injection for OpenClaw agent id/name in the embedded Pi extra-params wrapper, and also adds an unrelated Feishu outbound media MIME-sniffing fallback with tests.

## Best Possible Solution

Land a focused Bedrock implementation that keeps provider-wrapper precedence unchanged, sanitizes or skips requestMetadata keys/values according to the AWS ConverseStream contract, preferably sends a stable low-sensitivity agent id by default, and adds targeted unit coverage around zero-extra-param Bedrock calls plus invalid agent names. The Feishu MIME fallback should be split to its own PR if still desired.

## Reproduction Assessment

Yes. The review findings are reproducible by inspection: configure a Bedrock agent with a name like `Support 🤖` or `O'Reilly` and this PR sends that raw value into requestMetadata despite AWS’s documented value pattern; configure `openaiWsWarmup: false` and the PR’s wrapper reordering lets provider defaults override the explicit extra param.

## Solution Assessment

No. The general extra-params path is a plausible implementation point, but this patch needs validation/sanitization and must preserve the existing provider/default precedence contract; a narrow Bedrock-specific metadata merge after provider wrapping, with tests, would be safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Sanitize Bedrock metadata before sending it:** `src/agents/pi-embedded-runner/extra-params.ts:284-286`
  - body: `agents.list[].id` and `agents.list[].name` are currently plain config strings, but Bedrock requestMetadata values only accept a limited character set. Passing raw names here means values like `Support 🤖` or `O'Reilly` can make every Bedrock stream call for that agent fail with a ValidationException; normalize or skip invalid metadata before adding it to the payload.
  - confidence: 0.9
- **[P2] Preserve extra-param precedence over provider defaults:** `src/agents/pi-embedded-runner/extra-params.ts:507`
  - body: Moving `applyPrePluginStreamWrappers` before `wrapProviderStreamFn` reverses the existing wrapper order. Provider wrappers can now pass their defaults as `options` into the inner extra-params wrapper, and because that wrapper spreads `...options` last, explicit config such as `openaiWsWarmup: false` can be overwritten by a provider default.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The diff introduces a low-severity data-disclosure concern by automatically sending raw agent identity strings to AWS Bedrock request metadata.

Concerns:

- **[low] Avoid raw agent names in Bedrock metadata:** `src/agents/pi-embedded-runner/extra-params.ts:286`
  - body: The patch sends `agents.list[].name` to AWS requestMetadata on every Bedrock call. AWS documents this field as invocation-log filter metadata, and OpenClaw does not currently constrain agent names to non-sensitive values, so private project/customer labels could be exposed in AWS logs and cost tooling without an opt-in.
  - confidence: 0.78
