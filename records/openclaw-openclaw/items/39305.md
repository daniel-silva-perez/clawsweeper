## Summary

Keep this issue open. Current main still has wall-clock subagent run timeouts plus manual kill/steer controls, but it does not implement activity-based last-tool-call tracking, a nudge stage, automatic stall kill escalation, config fields, or `sessions_spawn` overrides. The related implementation PR #43017 was closed unmerged, while #39127 and #39141 remain broader open activity/watchdog design work rather than shipped subagent recovery.

## Best Possible Solution

Keep this issue open until maintainers either implement activity-based subagent stall recovery or explicitly consolidate it into a canonical session activity/watchdog design. A good implementation would add strict config and `sessions_spawn` override fields, track child tool/run activity, inject auditable nudges into child sessions, terminate through the existing subagent kill/cancellation path after the kill threshold, document the behavior, and cover config/runtime/docs with focused tests.
