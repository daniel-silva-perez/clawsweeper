## Summary

Keep this issue open. Current main has the related beforeDeliver/message_sending hook plumbing from the later merged work, but it does not implement the reported universal, fail-closed structured-output validation gate. Several current paths still rely on optional hooks, caller-supplied beforeDeliver behavior, or permissive structured payload preservation.

## Best Possible Solution

Add a shared structured ReplyPayload validation seam before native dispatcher delivery and align deliverOutboundPayloads/message.action/cron/ACP paths with it. The implementation should validate only structured operator-facing fields, fail closed on invalid structured payloads, preserve simple replies, and include regression tests for the dispatcher, outbound delivery, Telegram paths that supply their own beforeDeliver, message.action, and background delivery paths.
