## Summary

Keep open. Current main shipped a related hook-layer improvement, but it still does not provide the universal structured-output validation gate described here: the dispatcher seam is optional, `message_sending` is text-focused and hook-dependent, and some direct/native paths still reach delivery without a canonical structured payload validator.

## What This Changes

The issue requests a universal fail-closed pre-send validation gate for structured operator-facing payloads at the native reply-dispatch to delivery handoff.

## Best Possible Solution

Define a shared, channel-agnostic structured reply payload validation helper and run it before every native dispatcher delivery, composing it with the existing `message_sending` hook seam. Keep channel-specific schema decisions in channel/plugin-owned normalizers, but fail closed for malformed shared `presentation` or `interactive` payloads and add focused dispatcher/direct-path regression tests.

## Reproduction Assessment

Yes. A source-level reproduction is clear: create a dispatcher without `beforeDeliver`, enqueue a payload with structured fields such as `presentation` or `channelData`, and the current handoff can call `options.deliver` without a canonical validation step.

## Solution Assessment

Unclear. The dispatcher handoff is the right leverage point, but the best fix needs maintainer agreement on which structured fields are universal enough to validate fail-closed and how plugin-owned `channelData` should be treated.
