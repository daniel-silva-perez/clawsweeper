## Summary

Keep this PR open. Current main still does not implement the PR's central memory isolation change: builtin memory index tables, query filters, write/delete paths, FTS rows, and QMD mcporter daemon state are not scoped by an agent namespace. The PR is also an active stacked implementation candidate with unresolved review feedback, so it is not obsolete cleanup.

## Best Possible Solution

Keep this PR open. Review or resolve #73769 and #73770 first, then rebase/narrow this PR, fix the package-copy migration bug and any package-boundary drift, and validate that builtin shared `store.path` databases store/query/delete by agent namespace consistently across keyword, vector, FTS, file-state, status, and QMD mcporter paths.
