## Summary

Keep open. The memory-isolation direction is valid and current main still lacks agent_id scoping in the built-in memory index, but this stacked PR is not mergeable as-is because the patch introduces migration, reindex, concurrency, and large-index regressions in security-sensitive memory behavior.

## What This Changes

The PR branch adds built-in memory CLI provider registration, memory_search agent-scope/provenance handling, agent_id scoping for SQLite memory index schema/search/write paths, QMD mcporter state scoping, and related regression tests.

## Best Possible Solution

Rebase the stacked memory series onto current main, keep packages/memory-host-sdk as the schema source of truth, implement agent_id isolation without discarding other agents in shared stores, preserve existing large-index streaming/yielding and non-batch concurrency contracts, then land the series with focused shared-store migration/search/reindex coverage.

## Reproduction Assessment

Yes. Current-main source inspection shows the unscoped schema/filter behavior, and the PR patch has deterministic failure paths: Set.length on a Set, existing tests for streaming fallback and non-batch concurrency, and a shared-store force reindex path that swaps in a temp DB containing only the current agent.

## Solution Assessment

No, not as-is. The owner-area direction is right, but the patch needs a rebase and targeted fixes before it is the narrowest maintainable solution for shared memory isolation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Preserve other agents during safe reindex:** `extensions/memory-core/src/memory/manager-sync-ops.ts:1176`
  - body: With agent_id-scoped rows, safe reindex now builds a fresh temp database and atomically swaps it over the shared store, but the build only seeds cache and indexes the current agent. A forced/full reindex for one agent on a shared store.path will drop the other agents' files, chunks, FTS, and vector rows; copy non-current agent rows or switch to an in-place agent-scoped rebuild.
  - confidence: 0.93
- **[P1] Use Set.size for the FTS rebuild check:** `packages/memory-host-sdk/src/host/memory-schema.ts:46`
  - body: readColumns returns a Set, so existingFtsColumns.length is undefined and this condition never rebuilds old FTS tables that lack agent_id. Existing package-backed databases keep the old virtual table, and later inserts that include agent_id will fail after upgrade.
  - confidence: 0.98
- **[P2] Restore non-batch embedding concurrency behavior:** `extensions/memory-core/src/memory/manager-embedding-ops.ts:94-95`
  - body: This change drops remote.nonBatchConcurrency and the Ollama default of 1, so inline indexing falls back to the generic default or batch concurrency even when batch mode is off. The existing manager-embedding-timeout tests cover both contracts and would fail with this implementation.
  - confidence: 0.95
- **[P2] Keep fallback vector scoring bounded:** `extensions/memory-core/src/memory/manager-search.ts:254`
  - body: The fallback vector path now loads every matching chunk with .all(), scores all rows, sorts the full array, and then slices. Current main streams with iterate and keeps only the top K results, so this regresses large indexes by materializing all chunks and embeddings in memory.
  - confidence: 0.91
- **[P2] Preserve yielding during cache seeding:** `extensions/memory-core/src/memory/manager-sync-ops.ts:319`
  - body: Changing cache seeding to .all() plus a synchronous loop removes the existing setImmediate yield during large rebuilds. Large embedding caches can block the event loop long enough to delay gateway health probes and other async work.
  - confidence: 0.88
- **[P2] Point the shared-DB test at real agent workspaces:** `extensions/memory-core/src/memory/index.test.ts:259`
  - body: The new test writes fixtures under workspaceDir/agent-a and workspaceDir/agent-b, but createCfg marks each tested agent as the default agent, so resolveAgentWorkspaceDir resolves both managers to workspaceDir instead of those subdirectories. The test will not index the files it writes; give each agent an explicit workspace or write under the resolved workspace.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: Security review needs attention because the patch changes memory isolation and has concrete migration and shared-store reindex flaws before the isolation guarantee can be trusted.

Concerns:

- **[high] Shared-store reindex can discard other agents:** `extensions/memory-core/src/memory/manager-sync-ops.ts:1176`
  - body: The PR introduces agent-scoped rows but safe reindex still swaps the whole SQLite database with a temp database built only for the current agent, which can delete other agents' indexed memory from a shared store.path.
  - confidence: 0.9
- **[medium] Existing FTS tables may not receive agent scoping:** `packages/memory-host-sdk/src/host/memory-schema.ts:46`
  - body: The package schema migration checks Set.length instead of Set.size, so existing FTS tables without agent_id are not rebuilt and the security-sensitive isolation migration can fail on upgrade.
  - confidence: 0.96
