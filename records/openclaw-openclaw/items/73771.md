## Summary

Keep this PR open. Current main still does not enforce an agent namespace inside the builtin memory index tables, source filters, write/delete paths, FTS rows, or QMD mcporter daemon state. The PR is also an active stacked implementation candidate, with #73769 and #73770 preceding it and #73772/#73773 depending on it, so it is not obsolete or superseded by current main.

## Best Possible Solution

Keep the PR open. The best path is to review the vector-memory stack in order, land or close #73769 and #73770 first, then rebase/narrow this PR and validate the shared-DB namespace behavior with focused memory-core tests plus the appropriate build/check gate. The implementation should preserve default per-agent stores while making explicitly shared `store.path` databases safe by storing and querying an agent namespace consistently.
