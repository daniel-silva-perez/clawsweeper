## Summary

Keep open. Current main still has the older Discord and Telegram native-command overflow warnings, so the PR is not obsolete, but the proposed Telegram truncation warning drops fallback guidance for plugin/custom command overflow and the Discord hunk is stale after current main moved that logic into a helper file.

## What This Changes

This PR changes Discord and Telegram native-command overflow and retry log messages to recommend `commands.nativeSkills` or provider-specific overrides, and updates one Telegram test assertion.

## Best Possible Solution

Rebase the PR onto current main, update the moved Discord command resolver plus the two Telegram warning paths, and keep the final copy layered: suggest `commands.nativeSkills` for per-skill overflow, reserve `commands.native: false` for disabling all native commands, and retain plugin/custom reduction guidance where the capped list can include plugin or custom commands.

Yes. A code-level reproduction is clear: drive Discord command resolution above 100 commands, or register more than 100 Telegram commands; current tests already cover Telegram custom-command truncation, and the PR's changed copy can be reviewed against that combined native/plugin/custom command list.

## Solution Assessment

No, not as currently proposed. The copy-only approach is the right narrow direction for #49273, but this branch should first restore the plugin/custom fallback guidance and rebase the Discord change onto the current helper file.
