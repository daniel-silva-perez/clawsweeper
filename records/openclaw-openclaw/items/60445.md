## Summary

Keep open. Current main still does not implement the PR's central change: exec-approval-unavailable reply payloads are text-only, while the delivery gate only allowlists `channelData.execApproval` pending payloads. The PR remains a plausible focused fix, though it needs review/rebase against newer source-reply delivery behavior on main.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is to rebase it onto current main, add `channelData.execApprovalUnavailable: { reason }` to every `buildExecApprovalUnavailableReplyPayload` return path, update delivery/progress-suppression gates to preserve that marker where pending approvals are preserved, and add focused tests for unavailable payloads in the current source-reply delivery model.
