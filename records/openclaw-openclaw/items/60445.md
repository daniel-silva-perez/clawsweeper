## Summary

Keep this PR open. Current main still has the exact gap the PR is trying to fix: exec-approval-unavailable replies are built as text-only payloads, while the delivery gate only allowlists `channelData.execApproval`, so unavailable approval prompts can still be suppressed in message-tool-only/group-native delivery paths. The PR is focused, and the provided review thread shows the only bot concern was test-performance style and the author amended the branch to address it.

## Best Possible Solution

Keep the PR open and review or refresh it against current main. The likely correct path is to add a typed `channelData.execApprovalUnavailable` marker to every `buildExecApprovalUnavailableReplyPayload` return path, teach the dispatch tool-delivery filters to preserve that marker alongside `execApproval`, and add focused tests in the existing exec-approval helper and dispatch coverage without reintroducing per-test module reload overhead.
