## Summary

Current main still matches the reported bundled-plugin mismatch: Active Memory's blocking recall sub-agent is explicitly prompted and allowlisted for `memory_search`/`memory_get`, while the bundled `memory-lancedb` memory-slot plugin registers `memory_recall`/`memory_store`/`memory_forget`. The prior ClawSweeper comment reached the same conclusion, and the touched surface is unchanged on current main. This remains an actionable OpenClaw bundled-plugin integration bug, not a stale, duplicate, or ClawHub item.

## Best Possible Solution

Keep open. The best fix is to align the bundled memory plugin boundary: Active Memory should either discover and use the selected memory slot's typed recall tool/capability, or Memory LanceDB should expose the same common recall contract expected by Active Memory. Add a regression test covering Active Memory with `plugins.slots.memory = "memory-lancedb"`, and update the Active Memory and Memory LanceDB docs so they describe one supported integration path.

## Risks / Open Questions

- Closing this now would leave a current bundled-plugin incompatibility for users who enable Active Memory while selecting `memory-lancedb` as the memory slot.
- The fix needs a maintainer decision on the intended common memory contract: adapt Active Memory to the selected memory backend's recall capability, or make `memory-lancedb` expose the shared `memory_search`/`memory_get` surface if that is the slot API.
