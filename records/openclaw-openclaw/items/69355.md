## Risks / Open Questions

- The PR adds a normal `openclaw doctor` network request to the npm registry without a bounded timeout, so a blackholed registry path could stall a non-essential doctor diagnostic.
- The PR's stale-cache fallback can revive expired registry data after rejecting stale cache earlier, which could emit false `version-unreleased` warnings when current registry data is unavailable.
- The PR reads both `package.json.version` and `dist/build-info.json.version` but does not compare them, so partial or interrupted installs can be evaluated against the wrong version.
- The PR does not touch workflows, lockfiles, package manager sources, or third-party action refs; the main security-sensitive surface is the added network/cache behavior in the doctor command.
