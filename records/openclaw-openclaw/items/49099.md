## Repair Work Prompt

Fix inbound WhatsApp GIF/media metadata handling for this issue. In the WhatsApp plugin, detect media-only `videoMessage.gifPlayback` messages and classify their placeholder as GIF rather than regular video. Extract `message.videoMessage.contextInfo.externalAdReply` and, where appropriate, image/video `externalAdReply` title/body/sourceUrl metadata, but do not inline provider-fetched strings into pseudo-XML placeholder attributes. Carry the metadata through explicit `untrustedStructuredContext` entries such as a WhatsApp media metadata object so prompt rendering emits fenced untrusted JSON. Keep the change plugin-owned unless a genuinely generic already-existing seam must be reused. Add focused tests for extractor classification, monitor enrichment, dispatch forwarding, and prompt rendering. Update `docs/channels/whatsapp.md` and `CHANGELOG.md` if the agent-visible prompt surface changes. Do not change dependency versions, core channel ownership, or unrelated media-download behavior. Related refs: this issue and closed PR #49103.
