## Summary

Keep open. Current main still normalizes inbound WhatsApp video messages, including GIF-playback videos, to the generic `<media:video>` placeholder and has no WhatsApp inbound extraction for `contextInfo.externalAdReply` title/body/source URL. The related implementation PR #49103 is closed and unmerged in the provided GitHub context, and no landed history for `externalAdReply` appears in the current checkout.

## Best Possible Solution

Keep the issue open. The best fix is a focused WhatsApp plugin change in `extensions/whatsapp/src/inbound/extract.ts` and its inbound tests that detects `videoMessage.gifPlayback`, extracts safe GIF/media metadata from `contextInfo.externalAdReply`, and treats that metadata as untrusted while preserving useful placeholders for the agent.
