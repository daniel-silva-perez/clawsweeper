## Summary

Keep open. Current main still emits fixed WhatsApp media placeholders, does not inspect `videoMessage.gifPlayback` or `contextInfo.externalAdReply`, and only forwards contact context through the existing untrusted structured metadata path. The closed companion PR #49103 confirms the intended narrow fix did not land, while the pinned Baileys version exposes the requested fields.

## What This Changes

The issue asks WhatsApp inbound handling to distinguish picker GIFs from regular videos and preserve `externalAdReply` title, body, and source URL metadata for agent context.

## Best Possible Solution

Implement the fix inside the WhatsApp plugin: classify `videoMessage.gifPlayback` media-only inbound messages as GIFs and carry `externalAdReply` title/body/source URL through explicit untrusted media metadata, with focused unit and dispatch coverage plus user-facing docs/changelog if the prompt surface changes.
