## Summary

Keep this PR open. Current main still has the Copilot Claude replay gap the PR targets: Copilot Claude enables `dropThinkingBlocks`, but both session-history replay and tool-continuation replay still call the generic helper that preserves latest assistant thinking blocks. The PR remains a plausible implementation candidate, though it needs maintainer review around provider normalization and the bundled ACP/guard-script changes.

## Best Possible Solution

Keep this PR open. The best maintainer path is to refresh it against current main, model the Copilot behavior as an explicit replay sanitizer strategy or normalized provider-policy result, apply it consistently to session-history replay, tool-continuation replay, and context-pruning estimates, update the Copilot regression tests to drop latest plain thinking blocks while preserving non-Copilot Anthropic behavior, and split or clearly justify the ACP/guard-script allowlist changes before review.
