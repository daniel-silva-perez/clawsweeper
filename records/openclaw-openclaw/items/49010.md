## Summary

Keep this issue open. Current main does not implement opt-in MCPS/message-level signing for stdio MCP traffic, and the existing plugin/bundle MCP surfaces package MCP server definitions rather than exposing a clear public hook to decorate OpenClaw's core stdio JSON-RPC transport. The reporter's linked `razashariff/mcps-openclaw` package is useful prior art, but it is not shipped in this repo and the request is a security-sensitive core/API design question rather than stale cleanup.

## Best Possible Solution

Keep the issue open for maintainer security/API design. A good implementation would either add a generic, opt-in MCP transport signing/decorator seam with documented config, tests for replay/stripping/forgery, and dependency review, or explicitly document why this should remain an external wrapper and what plugin/API surface external packages should use.
