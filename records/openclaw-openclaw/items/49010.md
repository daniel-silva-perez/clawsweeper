## Summary

Keep this issue open. Current main still sends stdio MCP JSON-RPC messages unsigned, has no supported MCPS/signing config surface, and the plugin/bundle MCP path contributes server definitions rather than a public transport decorator hook. The reporter's linked mcps-openclaw package is useful prior art, but this remains a security-sensitive core/API design decision.

## Best Possible Solution

Keep the issue open for maintainer security/API design. A good path is either a generic opt-in MCP stdio transport signing/decorator seam with documented config and tests for replay, stripping, forgery, and downgrade behavior, or an explicit decision that this should remain external plus a documented plugin/API surface external packages can use safely.
