## Summary

Keep this PR open. Current main at f256eeba431b5f59e6085b7c11c9c0a309e408b7 still lacks Feishu missed-message recovery after gateway sleep/reconnect, and this PR is a concrete attempt at that Feishu plugin behavior. It is not mergeable as-is: the proposed module is not wired into the monitor lifecycle, imports a helper that does not exist on main, and the review discussion identifies runtime/API and message-routing problems that need author or maintainer rework rather than cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer judgment, but require a replacement implementation before merge: implement Feishu-owned recovery inside the existing monitor lifecycle, use createFeishuClient with the established im.chat.list and im.message.list patterns, preserve shared chatHistories and sequential/dedupe behavior, keep mention gating structured and policy-aware, document required permissions/config, and add targeted recovery tests plus live API validation where feasible.
