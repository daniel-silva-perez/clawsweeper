## Summary

Keep this PR open. Current main does not implement Feishu missed-message recovery after sleep/reconnect, and this PR is a concrete attempt at the still-missing Feishu plugin behavior, but the proposed one-file implementation is not mergeable as-is and needs maintainer rework rather than automated cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer judgment, but require a replacement implementation before merge: add a Feishu-owned recovery lifecycle hook in `monitorSingleAccount`, use the existing `createFeishuClient`/chat/message API patterns, preserve shared `chatHistories`, route recovered messages through the same sequential/dedupe path as live events, keep mention gating structured and policy-aware, document required permissions/config, and add focused startup/reconnect recovery tests plus live API proof where possible.
