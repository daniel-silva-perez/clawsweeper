## Summary

Keep this PR open. Current main still does not implement Feishu missed-message recovery after sleep/reconnect, and the PR is a focused attempt at the linked Feishu plugin behavior, but the proposed patch needs substantial rework before it is mergeable.

## Best Possible Solution

Keep the PR open for maintainer/author rework. The best fix is a Feishu-owned recovery path inside the existing monitor lifecycle that uses createFeishuClient with im.chat.list and im.message.list, threads recovered events through the shared chatHistories, dedupe, and sequential handling, uses structured mention/policy checks, documents required Feishu scopes/config, and adds focused tests plus live API proof where feasible.

- A correct implementation still needs targeted tests and preferably live Feishu API validation for chat/message listing permissions and payload shape.
