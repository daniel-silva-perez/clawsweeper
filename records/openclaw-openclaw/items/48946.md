## Summary

Keep this PR open. Current main still does not implement Feishu missed-message recovery after gateway sleep/reconnect, and this PR is a focused attempt at the linked #28218 behavior. It is not obsolete, but it needs substantial rework before it is mergeable.

## Best Possible Solution

Keep the PR open for maintainer/author rework. The best implementation is a Feishu-owned recovery path inside the existing monitor lifecycle: use `createFeishuClient` with `im.chat.list` and `im.message.list`, thread recovered events through the shared `chatHistories`, dedupe and sequential queue, use structured mention/policy checks, document required Feishu scopes/config, and add focused tests plus live Feishu API proof where feasible.
