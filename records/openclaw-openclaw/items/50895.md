## Summary

Keep this PR open. Current main still does not implement Brazilian WhatsApp ninth-digit equivalence, and the PR targets a real linked WhatsApp delivery/access problem from #20187. The work needs a rebase/port to the current bundled plugin layout plus fixes for the later reaction and session-routing review concerns before it can merge.

## Best Possible Solution

Keep the PR open and ask for a rebase/port onto current main. The revised implementation should add a narrow WhatsApp BR direct-target equivalence helper in the bundled plugin surface, use it for allowlist/command/self-chat/heartbeat authorization, preserve the original explicit chat target for reactions, quote lookup, and outbound session routing, and add focused regression tests in the current extensions/whatsapp paths.
