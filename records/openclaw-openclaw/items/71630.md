## Summary

Keep this open. Current main still lacks the requested always-visible composer runtime chip; the nearby Web UI surfaces are action buttons, Talk-only text, token/compaction/fallback indicators, and reconnect cleanup that silently clears orphaned run state rather than showing an Interrupted/Done/In progress lifecycle.

## What This Changes

Request to add a compact, accessible In progress/Interrupted/Done runtime status indicator beside the Web UI composer action controls.

## Best Possible Solution

Add a narrow Control UI composer status chip driven by explicit local chat lifecycle state now, fail closed to Interrupted when reconnect or lost-terminal-event continuity is uncertain, and later swap its source to the gateway activity API if #39127 lands.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Static verification against current main is high-confidence because the composer render path, run controls, reconnect handling, docs, and gateway method list do not expose the requested status indicator.

## Solution Assessment

Yes, a narrow Web UI implementation is the best first step if it uses explicit lifecycle state and avoids transcript messages. The broader gateway-owned activity API should remain tracked separately in #39127 rather than being bundled into this UI chip.
