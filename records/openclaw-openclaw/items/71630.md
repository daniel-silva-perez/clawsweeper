## Summary

Current main still lacks the requested compact Control UI composer runtime status indicator. The issue remains actionable and distinct from broader session activity API work in #39127 and channel status-surface work in #52640.

## What This Changes

Add a compact, accessible Web UI composer status chip near the attachment/Talk controls showing at least In progress, Interrupted, and Done for the current session turn.

## Best Possible Solution

Keep this open and implement a narrow Control UI composer status chip that uses existing chat lifecycle signals now, fails closed to Interrupted after uncertain reconnects, avoids transcript messages, and can later consume the gateway-owned per-session activity API from #39127 when that lands.

## Solution Assessment

Yes for a first repair if scoped to the composer UI and existing lifecycle signals, with explicit fail-closed reconnect handling. A full authoritative solution should not hard-code text heuristics and should be designed to swap to #39127's session activity API later.
