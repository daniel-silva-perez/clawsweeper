## Summary

Keep this open. Current main has local chat lifecycle state and composer action controls, but it still does not expose the requested always-visible composer status chip, and the related gateway activity API work remains a separate open dependency rather than an implementation of this narrower Control UI request.

Add a compact runtime status indicator beside the Web UI composer actions showing In progress, Interrupted, and Done without adding transcript messages.

Ship a focused Control UI composer chip now, driven by explicit local chat lifecycle and reconnect signals, with stable desktop/mobile layout and accessibility; leave the cross-client authoritative activity model to #39127.
