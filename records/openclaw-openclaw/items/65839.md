## Summary

Keep this open. Current main still exposes the reported generic context-overflow hint with the hard-coded `20000 or higher` recommendation, and the open unmerged PR #73561 is explicitly paired to this issue as the active implementation candidate.

## What This Changes

The issue asks for context-limit recovery guidance to stop recommending a hard-coded `reserveTokensFloor` value of `20000` when larger model context windows need higher practical reserves.

## Best Possible Solution

Keep this open and review #73561 or an equivalent focused patch that centralizes context-aware overflow guidance, derives the suggested reserve from the active model/session context window, preserves the heartbeat-specific diagnostic, and updates the focused recovery-text tests plus compaction/config docs.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: call `buildContextOverflowRecoveryText()` without heartbeat-model evidence and current main returns the generic hint containing `reserveTokensFloor` and `20000 or higher`; the existing focused tests also assert the generic fallback remains present.

## Solution Assessment

Yes, the best direction is a shared context-aware recovery-hint helper in the existing overflow reset path. A blind global increase would be weaker because `applyPiCompactionSettingsFromConfig()` already caps reserve floors for small context windows, so the user-facing advice should reflect the active context budget.
