## Summary

Current main still exposes the reported generic context-overflow hint with a hard-coded `reserveTokensFloor` value of `20000 or higher`. Open PR #73561 explicitly targets this issue with context-window-aware guidance and has not merged, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Request to replace or improve the context-overflow recovery hint that recommends `agents.defaults.compaction.reserveTokensFloor` of `20000 or higher`, because that value can be too low for larger model context windows.

## Best Possible Solution

Keep this issue open and review #73561. The preferred fix is a shared, tested recovery hint that derives practical reserve guidance from the active model/context window, preserves small-context safety, covers both reset paths, and updates public compaction/config guidance if the user-facing behavior changes.
