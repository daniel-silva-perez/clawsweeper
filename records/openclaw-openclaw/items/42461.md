## Summary

Keep this issue open. Current main still builds the default TUI Gateway URL from env/config/default port resolution and does not consult a running Gateway's active runtime port. The linked implementation PR #73338 is open with closing linkage, so the issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review, land, or supersede PR #73338 with a small trusted fix: persist or otherwise expose the active local Gateway port, make default TUI loopback resolution consult it after explicit URL/env/remote precedence, and cover 48789 plus stale-lock and override cases in tests.
