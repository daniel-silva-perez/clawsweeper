## Summary

Keep this issue open. Current main still builds the default TUI Gateway URL from config/env/default port resolution, while `openclaw gateway --port` is a runtime-only override. The issue also has open PR #73338 with closing syntax, so cleanup policy says the issue should stay open until that PR or an equivalent fix merges.

## Best Possible Solution

Keep this issue open and review, land, or supersede #73338 with a small trusted runtime-port fix: persist or otherwise expose the active local Gateway port, make default TUI loopback resolution consult it only after explicit URL/env/remote config precedence, and add regression tests for an active Gateway on 48789 plus stale-lock and override cases.
