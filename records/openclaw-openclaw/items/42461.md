## Summary

Keep this issue open. Current main still builds the default TUI Gateway URL from env/config/default port resolution, not from a running Gateway's active runtime port. The provided GitHub context also shows PR #73338 is open, unmerged, and uses closing linkage for this issue, so the right maintainer path is to review/land or supersede that PR rather than close the issue now.

## Best Possible Solution

Keep this issue open and review, land, or supersede PR #73338 with a focused fix that exposes the active local Gateway port to TUI default resolution, ideally by storing trusted runtime port metadata in the Gateway lock or another current runtime state seam, plus regression coverage for `48789`, stale/non-Gateway lock handling, and explicit URL/remote precedence.
