## Summary

Keep this issue open. Current main still resolves the default TUI Gateway URL from config/env/default port resolution, while `openclaw gateway --port 48789` is only applied in the Gateway process path. The related PR #73338 is still an implementation candidate with closing linkage, so policy says this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review, land, or supersede PR #73338 with a small trusted runtime-port fix: persist or otherwise expose the active local Gateway port, make default TUI loopback resolution consult it only after explicit URL/env/remote config precedence, and add regression tests for an active Gateway on 48789 plus stale-lock and override cases.
