## Summary

Keep open. Current main still matches the reported v2 chat composer usability problem: the visible composer is a single bordered surface, but only the textarea portion is editable/click-focusable, and the lower toolbar remains compact icon-only controls. A related placeholder regression appears fixed separately, but this issue’s central click-target and button usability complaints remain.

## Best Possible Solution

Keep this issue open and fix it in the Control UI composer: either make the whole bordered composer focus the textarea or visually separate the toolbar from the input field, increase or label the primary controls where practical, and add a focused UI regression test for composer hit targets and accessible/visible control affordances.
