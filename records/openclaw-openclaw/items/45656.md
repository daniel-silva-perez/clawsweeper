## Summary

Keep open. Current main still matches the reported Control UI regression: the bordered composer contains a short textarea plus a separate lower toolbar, empty composer chrome has no focus path to the textarea, and the controls remain small icon-only buttons.

## What This Changes

The issue requests a Control UI v2 chat composer usability fix for the dead lower click area, space-wasting layout, and tiny tooltip-dependent controls compared with v1.

## Best Possible Solution

Fix the Control UI composer so the entire visible text-entry area, including empty non-control chrome, focuses the textarea; increase practical control hit targets and discoverability while preserving Send/Queue/Stop, attachment, Talk, paste, and Shift+Enter behavior with focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence reproduction path is to open the Control UI chat composer and click inside the bordered lower composer area outside the actual textarea and buttons; current source shows no wrapper click handler to focus the textarea, so text entry is not activated.

## Solution Assessment

No current fix is present. The narrow maintainable solution is a composer interaction and sizing repair with tests, while a full rollback to the v1 button layout should remain a separate maintainer product decision.
