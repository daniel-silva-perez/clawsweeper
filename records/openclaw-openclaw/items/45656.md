## Summary

Keep open. Current main still has the Control UI composer shaped as one bordered area containing a short textarea plus a separate lower toolbar, and the controls remain small icon-only buttons. That preserves the main usability regression reported here, so this is neither stale nor implemented-on-main cleanup.

## What This Changes

The issue requests restoring or improving the v2 Control UI chat composer usability, especially the click target, space use, and action-button discoverability compared with v1.

## Best Possible Solution

Fix the Control UI composer directly so the visible text-entry area and interactive focus target match, composer controls have practical hit sizes and discoverable labels or equivalent affordances, and existing Send/Queue/Stop, attachment, Talk, and Shift+Enter behavior remains intact with focused regression coverage.

## Reproduction Assessment

Yes. The current source gives a high-confidence reproduction path: open the Control UI chat composer and click empty lower composer chrome outside an actual button; because only the textarea handles text input and the wrapper has no focus handler, that click will not focus text entry.

## Solution Assessment

No current fix is present. The maintainable path is a narrow Control UI composer repair plus tests, not a full rollback of the v2 dashboard layout unless maintainers explicitly want that broader redesign.
