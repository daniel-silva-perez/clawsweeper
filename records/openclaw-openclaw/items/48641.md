## Summary

Keep this issue open. Current main still resolves Discord account config through the generic account merge helper, where account-scoped `allowFrom` arrays override top-level `allowFrom`; the DM preflight path then authorizes DMs against that resolved per-account list. The related fix PR #49365 was closed unmerged, and current tests/docs preserve the override semantics, so this remains a real Discord access-control behavior decision rather than cleanup-close material.

## Best Possible Solution

Keep the issue open for a maintainer-owned Discord access-control decision. Define whether top-level `channels.discord.allowFrom` is a shared allowlist, a default inherited only when account `allowFrom` is unset, or a list that unions with every account. Then implement that chosen behavior in `extensions/discord/src/accounts.ts` or the DM preflight access path, add regression coverage for default and named accounts, and update `docs/channels/discord.md` plus changelog because the security impact is user-facing.
