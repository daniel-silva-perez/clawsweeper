## Summary

Keep this issue open. Current main still resolves Discord per-account config through the generic account merge path, where account-scoped `allowFrom` arrays override top-level `allowFrom`, and the Discord DM preflight path uses that resolved per-account list for allowlist decisions. The related fix PR #49365 was closed unmerged, and current docs/tests preserve override semantics, so the reported multi-account DM allowlist problem remains a real product/security design question rather than cleanup-close material.

## Best Possible Solution

Keep the issue open for a maintainer-owned Discord access-control decision. Define whether top-level `channels.discord.allowFrom` is a shared allowlist, a default inherited only when account `allowFrom` is unset, or a list that unions with every account. Then implement that chosen behavior in `extensions/discord/src/accounts.ts` or the preflight access path, add regression coverage for default and named accounts, and update `docs/channels/discord.md` plus changelog so operators understand the security impact.
