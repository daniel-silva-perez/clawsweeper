## Summary

Keep this issue open. The reported Discord DM allowlist/multi-account path is still a live product/design question: the proposed fix PR was closed unmerged, current main still resolves Discord account config through generic spread-style account overrides, and tests/docs preserve account-scoped allowFrom override semantics rather than merging global and account lists.

## Best Possible Solution

Keep the issue open for a maintainer-owned Discord access-control design decision. The best fix is to define the intended global-vs-account allowFrom semantics for multi-account Discord DMs, implement that in the Discord account resolution/preflight path, add regression tests for global-only, account-only, merged/deduped or override behavior as chosen, and update docs/changelog so users understand whether account allowlists narrow or extend the global list.
