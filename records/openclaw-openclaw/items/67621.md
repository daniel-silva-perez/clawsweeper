## Summary

Keep open: current main still applies the all-caps dollar-token scanner to raw Python/Node source content without string, docstring, comment, or help-text awareness. The reported direct `python3 script.py` shape remains in the preflight path outside the documented gateway YOLO skip, and the related implementation PRs were closed unmerged.

## What This Changes

The issue reports that exec script preflight rejects Python files when `$VAR`-style text such as `$SIM` appears inside benign string literals, docstrings, or help text.

## Best Possible Solution

Make `validateScriptFileForShellBleed` lexical-context aware for Python and Node, or narrow the detector to real execution/shell-leak contexts, then add focused regression tests for `$SIM` in docstrings/help text and benign dollar-formatted strings while retaining the existing negative `$DM_JSON` and YOLO-skip coverage.
