## Summary

Current main still scans raw Python/Node source files for all-caps `$VAR` tokens without string/comment awareness, so the reported `$SIM` string-literal false positive remains actionable outside the documented YOLO skip path. The linked fix PRs were closed unmerged, and current tests still cover rejection plus the YOLO exemption rather than benign `$SIM` docstrings/help text.

## Best Possible Solution

Keep this issue open. Implement the fix in `src/agents/bash-tools.exec.ts` by making script-file preflight lexical-context aware for Python and Node, or by narrowing detection to real execution/shell-leak contexts. Add focused regression coverage in `src/agents/bash-tools.exec.script-preflight.test.ts` for `$SIM` in docstrings/help text and benign dollar-formatted output, while retaining negative tests for raw `$DM_JSON` code paths. The closed unmerged PRs #67644 and #67924 are useful references but not canonical shipped fixes.
