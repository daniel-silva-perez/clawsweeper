## Summary

Keep this issue open. Current main already has plaintext SKILL.md loading, skill config, watcher refresh, and persisted session skill snapshots, but it does not implement the requested signed/encrypted commercial wrapper or settle the trusted-source security policy; the provided GitHub context also shows open PR #74035 with closing syntax, so this issue should remain open until that PR or an equivalent maintainer-approved design lands.

## What This Changes

The issue requests persistent skill configuration plus commercial support for verified, signed, or encrypted Markdown skill wrapping and less restrictive security-review handling.

## Best Possible Solution

Keep the existing plaintext skill persistence path while maintainers decide a concrete trust model: either review and land a narrow persistence/trusted-source design from #74035, or split signed/encrypted commercial skill distribution into a ClawHub/private-bundle or plugin/API design with explicit provenance, scanning, and operator controls.
