## Summary

Keep this issue open. Current main already has plaintext skill discovery, gating, watcher refresh, and persisted session skill snapshots, but it does not implement the requested signed/encrypted commercial Markdown wrapper or settle the commercial security-review policy; the linked open PR #74035 also uses closing syntax, so the issue should remain open until that PR merges or maintainers choose a different direction.

## What This Changes

The issue requests persistent commercial skill/.md configuration plus an official verified, signed, or encrypted Markdown wrapper path and a less restrictive commercial security-review model.

## Best Possible Solution

Keep the existing plaintext skill persistence behavior, then have maintainers either review and land a narrowly acceptable trusted-source/persistence change or split the signed/encrypted commercial wrapper into an explicit ClawHub/plugin/API security design with provenance, scanning, operator controls, and documentation.

Not applicable as a feature and product/security policy request. Source inspection gives a high-confidence current-main check for existing plaintext skill persistence and the absence of signed/encrypted wrapper support, but there is no concrete runtime bug reproduction for skills being forgotten.

Unclear. The open PR’s `persist` and `trustedSources` approach may address one slice of the report, but the requested commercial wrapper and security-review exception are broader trust-model decisions and may need a ClawHub/plugin design instead of a core bypass.
