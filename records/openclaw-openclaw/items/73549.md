## Summary

Keep this issue open. Current main already has persistent plaintext skill loading, per-skill config, ClawHub skill install/update docs, and session skill snapshots, but the requested paid verified/signed or encrypted Markdown wrapper plus commercial/security-review policy change is not implemented and needs maintainer product/security judgment.

## Best Possible Solution

A maintainer should split or answer the request in two tracks: keep the existing plaintext skill persistence/config path as the current supported behavior, and separately decide whether commercial IP protection belongs in ClawHub private packages, official publisher verification, or a new core/plugin API for signed/encrypted skill bundles. If the reporter only needs stronger persistence, ask for a concrete repro against the current snapshot/watch/config behavior; if they need encrypted paid skills, open a design discussion with the loader format, trust model, and review policy.
