## Summary

Keep open. Current main already has plaintext `SKILL.md` discovery, skill config, watcher refresh, ClawHub skill distribution, and persisted session skill snapshots, but it does not implement the requested verified/signed or encrypted Markdown wrapper and has not settled the commercial security-review policy. Open PR #74035 also uses closing syntax for this issue, so the issue should remain open until that PR is reviewed/merged or maintainers choose a different product/security direction.

## What This Changes

The issue requests persistent commercial skill/.md support, including verified/signed or encrypted Markdown wrapping and less restrictive commercial security-review handling.

## Best Possible Solution

Keep the current plaintext skill persistence path, then have maintainers either review and land a narrowly safe version of PR #74035 or define a signed/private ClawHub skill-bundle model with clear provenance, scanning, audit logging, and operator controls.

## Reproduction Assessment

Not applicable as a feature and product/security policy request. Current-main inspection gives a high-confidence check of existing plaintext skill persistence, but the issue does not include concrete steps for reproducing skills being forgotten or ignored.

## Solution Assessment

Unclear. Existing main solves plain skill persistence, while a paid encrypted wrapper or trusted security-scan bypass is broader than a small config flag and needs maintainer trust-model review before it can be considered the best maintainable solution.
