## Summary

Keep this issue open. Current main already supports plaintext `SKILL.md` discovery, skill config, ClawHub install/update metadata, watcher-triggered refresh, and persisted session skill snapshots, but it does not implement the requested paid verified/signed or encrypted Markdown wrapper and does not settle the commercial security-review policy question. The linked open PR with closing syntax is an implementation candidate, so the issue should remain open until that PR merges or maintainers choose a different product/security direction.

## What This Changes

The issue requests persistent skill/Markdown configuration support for commercial use, including official verified/signed or encrypted `.md` wrapping and less restrictive commercial security-review handling.

## Best Possible Solution

Maintain the existing plaintext skill persistence path while maintainers split the remaining request into a concrete design: either review and land a narrow trusted-source/persistence PR, or define a signed/encrypted skill-bundle model through ClawHub/private packages or a new loader API with clear provenance, scanning, and operator controls.
