## Summary

Keep this PR open. Current main still does not implement configurable user-facing timestamps in the terminal TUI chat log, and the linked feature request remains open. The PR is a plausible implementation candidate, but the submitted approach still needs revision before it can land.

## Best Possible Solution

Keep this PR open as an implementation candidate for the linked TUI timestamp feature. A solid revision should add a typed/schema-backed and documented timestamp setting or settings toggle, use existing theme formatters such as `theme.dim`, preserve assistant prefixes through streaming and final updates, use persisted message timestamps during history replay, and add focused ChatLog/history coverage.
