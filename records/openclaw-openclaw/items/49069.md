## Summary

Keep this PR open. Current main still does not implement configurable user-facing timestamps in the TUI chat log, and the linked feature request #48708 remains open. The supplied PR is a focused implementation candidate, but it is not merge-ready: review discussion identified unsupported config/schema usage, `theme.muted` usage that does not exist on the current theme object, assistant streaming/final update paths that would drop prefixes, and history replay that still ignores persisted message timestamps for display. Security review found no supply-chain or secrets-sensitive changes in the supplied diff; the risk is functional/config correctness.

## Best Possible Solution

Keep this PR open as an implementation candidate for #48708. A solid revision should add a typed/schema-backed and documented TUI timestamp setting or settings toggle, use existing theme formatters such as `theme.dim`, preserve assistant prefixes through streaming and final updates, use persisted message timestamps during history replay, and add focused ChatLog/history coverage.
