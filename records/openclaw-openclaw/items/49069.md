## Summary

Keep this PR open. Current main still does not implement configurable timestamps in the TUI chat log: ChatLog renders raw system/user/assistant text, runTui constructs ChatLog without a timestamp option, history replay ignores message timestamps for display, and the strict config surface has no `tui.showTimestamps` or equivalent TUI timestamp key. The PR remains a plausible implementation candidate for the open linked feature request, but the provided diff is not merge-ready because review comments identify real functional blockers.

## Best Possible Solution

Keep the PR open as an implementation candidate for the still-open TUI timestamp feature. The best path is a revised patch that adds a schema/type/docs-backed timestamp setting, threads timestamp metadata into ChatLog for system/user/assistant/history paths, preserves assistant prefixes through streaming and finalize updates, uses an existing theme formatter such as `theme.dim`, and adds focused ChatLog plus TUI history replay tests.

## Risks / Open Questions
