## Summary

Keep this PR open. Current main still does not implement configurable user-facing timestamps in the terminal TUI chat log, and the linked timestamp feature request remains open. The PR is a plausible implementation candidate, but the submitted diff is not merge-ready because the proposed config key is not schema-backed, assistant/history timestamp handling is incomplete, and the diff calls a theme formatter that current main does not expose.

## Best Possible Solution

Keep this PR open as an implementation candidate for the TUI timestamp feature. A mergeable revision should add a typed/schema-backed and documented setting or a settings toggle, use existing theme formatters such as theme.dim, preserve assistant prefixes through streaming/final updates, use persisted transcript timestamps during history replay, and add focused ChatLog/history coverage.
