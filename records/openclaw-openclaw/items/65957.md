## Summary

Keep open. Current main still does not implement this PR's central prompt-selection behavior: heartbeat delivery target `none` resolves to no outbound delivery, and `runHeartbeatOnce` still turns that into `canRelayToUser: false` for exec/cron prompts. The route-poisoning concern is partly superseded by #66073, but the direct webchat relay prompt exception remains absent from main and the refreshed PR is still a plausible implementation candidate.

## Best Possible Solution

Keep this PR open as an active implementation candidate. Review the refreshed diff against current main, keep it narrowly scoped to direct internal webchat `heartbeat.target: "none"` relay-prompt behavior, rely on #66073 for system-event route preservation, and require targeted heartbeat/session tests plus the normal changed gate before merge.
