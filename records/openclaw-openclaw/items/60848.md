## Summary

Keep this PR open. Current main still does not implement the PR's central Feishu fix: topic sessions without `root_id` still do not persist `message_id` as `MessageThreadId`, and several outbound `threadId` fallback paths still omit or miscompute `replyInThread`. Related #67069 overlaps only the inbound metadata area and remains open; it does not supersede the outbound fixes.

## Best Possible Solution

Keep this PR open as a plausible Feishu bug-fix candidate. Rebase it onto current main, reconcile the narrower #67069 inbound metadata overlap, keep the `message_id` fallback guarded to topic sessions with `replyInThread` enabled and no `root_id`, propagate `replyInThread` consistently through Feishu outbound text, markdown-card, structured-card, payload-media, direct media, and local-image paths, then run targeted Feishu extension tests plus `pnpm check:changed` before merge or request changes.
