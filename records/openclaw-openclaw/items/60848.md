## Summary

Keep open. The branch addresses a real Feishu threading gap, but it is still incomplete against current main because the native payload path can use the same threadId fallback without consistently passing the Feishu thread-reply flag, and #67069 overlaps the inbound thread metadata semantics.

## What This Changes

The PR updates the Feishu plugin to persist a no-root topic message_id as MessageThreadId and to propagate replyInThread through threadId fallback text, card, media, and local-image sends with focused unit tests.

## Best Possible Solution

Update the branch on current main, keep the guarded no-root message_id fallback, compute one trimmed thread-reply decision for every Feishu outbound surface, apply it to text, markdown/structured/native card, payload media, regular media, and local-image sends, and add regression coverage for native payload threadId fallback while coordinating inbound metadata with #67069.
