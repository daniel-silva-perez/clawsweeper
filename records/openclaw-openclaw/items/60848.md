## Summary

Keep this PR open. The branch addresses useful Feishu thread anchoring work and the author already fixed earlier media/local-image review comments, but current main has a newer native payload send path that still does not carry the same in-thread reply flag, so the patch is incomplete against the current Feishu surface.

## What This Changes

The PR updates the Feishu plugin to persist `message_id` as `MessageThreadId` for no-root topic-thread sessions and to propagate `replyInThread` through outbound threadId fallback text, card, media, and local-image paths with focused tests.

## Best Possible Solution

Rebase/update the PR on current main, keep the guarded no-root `message_id` fallback, compute one trimmed `replyInThread` decision for all Feishu outbound surfaces, apply it to text, markdown-card, structured-card, native payload, media, and local-image sends, and add regression coverage for native payload threadId fallback.
