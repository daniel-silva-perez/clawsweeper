## Summary

Keep this PR open. Current main still intentionally limits the Gateway control-plane direct path to literal loopback IP URLs, while this PR proposes changing the Windows/WSL localhost proxy behavior. The PR is recent, contributor-authored, not protected by maintainer labels, and its current head appears to address the earlier bot-reported P1 defects, but the behavior is not implemented on main and needs maintainer review because it changes a documented network-proxy security boundary.

## Best Possible Solution

Review this as an active Gateway/proxy bug-fix PR. Maintainers should decide whether localhost should join the control-plane bypass for the Windows WSL inherited-proxy case. If yes, keep the client and proxy guard aligned on isLoopbackHost, preserve proxy env snapshot/restore semantics, add regressions for ws://localhost and env restoration when OpenClaw proxy is inactive, and update the network proxy docs. If no, close the PR with the documented workaround of using ws://127.0.0.1:18789 or ws://[::1]:18789 for local direct Gateway connections.
