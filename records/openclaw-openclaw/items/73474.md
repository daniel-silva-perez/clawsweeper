## Summary

Keep this PR open. Current main still excludes `localhost` from the Gateway direct control-plane proxy bypass, while this PR is an active contributor bug-fix candidate that changes that documented security boundary for the Windows/WSL inherited-proxy case. It is not implemented on main, not merged, and not superseded by another canonical item in the provided context.

## Best Possible Solution

Review this as an active Gateway/proxy bug-fix PR. If maintainers accept `localhost` as safe for the Gateway control-plane direct bypass, align the client and proxy guard on `isLoopbackHost`, preserve proxy environment snapshot/restore semantics, add regressions for `ws://localhost` and env restoration, and update the public network proxy docs. If maintainers want to preserve the existing security contract, close the PR with the documented workaround of using `ws://127.0.0.1:18789` or `ws://[::1]:18789` for local direct Gateway connections.
