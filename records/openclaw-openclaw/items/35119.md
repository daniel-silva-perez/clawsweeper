## Summary

Keep this issue open. Current main already handles the original numeric HTTP 500 and JSON `server_error` forms, but the later reported raw OpenAI Codex wording beginning `The server had an error processing your request...` is still not present in the transient server-error patterns or tests. The assistant failover path still depends on a classifier/failover signal or timeout before it advances to model fallback, so the remaining reported shape is not proven fixed.

## Best Possible Solution

Keep this issue open. Add a focused regression for an OpenAI Codex assistant turn with `stopReason: "error"`, raw text beginning `The server had an error processing your request...`, and configured model fallbacks, then classify that wording as a transient server failure so the assistant path throws a `FailoverError` and the outer model fallback loop reaches the next candidate.
