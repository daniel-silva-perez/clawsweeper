## Summary

Keep this issue open. Current main handles the original numeric HTTP 500 and JSON `server_error` cases, but the later concrete OpenAI Codex raw text `The server had an error processing your request...` is still not in the server-error classifier, and the assistant fallback policy still does not advance to model fallbacks without a classified failover signal, failover failure, or timeout.

## Best Possible Solution

Keep the issue open. Add a focused regression for an OpenAI Codex assistant turn with `stopReason: "error"`, raw text beginning `The server had an error processing your request...`, and configured model fallbacks, then classify that wording as a transient server failure so the assistant path reaches the fallback model without overbroadly treating unrelated generic error text as failover-worthy.
