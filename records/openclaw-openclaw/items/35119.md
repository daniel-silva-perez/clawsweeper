## Summary

Keep this issue open. Current main fixes the original numeric HTTP 500 and JSON `server_error` cases, but the later reported raw OpenAI Codex wording beginning `The server had an error processing your request...` is still absent from the transient server-error classifier, and the assistant failover path still needs a classified failover signal, timeout, or failover failure before it advances to model fallback.

## Best Possible Solution

Keep the issue open. Add a focused regression for an OpenAI Codex assistant turn with `stopReason: "error"`, raw text beginning `The server had an error processing your request...`, and configured model fallbacks, then classify that wording as a transient server failure so the assistant path throws a `FailoverError` and the outer model fallback loop reaches the next candidate.
