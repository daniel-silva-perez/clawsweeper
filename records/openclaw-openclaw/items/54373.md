## Summary

Keep open: current main still renders injected project/bootstrap context as flat path/content text without source, injection-time, or volatility metadata, and the same-author Phase 1 PR #54830 remains open and unmerged with closing syntax for this issue.

## What This Changes

Request to add source, injection-time, and volatility provenance metadata to injected context segments so agents can distinguish stale bootstrap or memory context from fresh tool/read output.

## Best Possible Solution

Keep this issue open while maintainers review PR #54830 as the immediate Phase 1 candidate. If accepted, land a focused implementation for embedded context metadata, bootstrap assembly, prompt rendering, and tests, then track memory freshness, stronger prompt rules, and orchestration refresh as narrower follow-ups or align them with adjacent provenance/tag-registry RFCs.
