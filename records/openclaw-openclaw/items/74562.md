## Summary

Keep open. Current `main` still has the focused live-suite rerun issue this PR targets: `live_suite_filter` is passed as a free string, broad job guards can start a matrix where every suite step is skipped by exact-equality checks, and `live-e2e` reruns still launch the Docker release-path job when a focused native live filter is present. The PR is a narrow workflow/test fix, and static review found no definite correctness or security blocker.

## What This Changes

This PR adds reusable-workflow validation for non-empty `live_suite_filter`, skips package and Docker release-path work for focused live-suite reruns, and adds workflow regression assertions.

## Best Possible Solution

Land a narrow workflow fix that makes non-empty `live_suite_filter` fail fast unless it selects a runnable suite for the chosen profile, and ensure focused native live-suite reruns do not spend release capacity on unrelated Docker release-path validation.
