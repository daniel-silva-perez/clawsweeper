## Summary

Keep open. This is a fresh Windows/Node 24 report against v2026.4.26 and 2026.4.23, and the reported release already contains the obvious HTTP/1-only mitigations for Telegram and guarded model fetch. Current main does not add clear proof that the remaining LLM timeout path is fixed, so closing as implemented would be premature.

## Best Possible Solution

Keep this open as an active Windows/provider transport regression. A maintainer should capture the reporter's resolved provider/model metadata for openai/claude-opus-4-7, confirm whether the failing call uses createBoundaryAwareStreamFnForModel and buildGuardedModelFetch, then reproduce on Windows Node 24 with Telegram polling plus a real LLM call. If reproduced, fix the remaining bypass or add an explicit provider transport family/dispatcher override with focused regression coverage.
