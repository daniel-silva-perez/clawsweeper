## Summary

Keep open. Current main still unsubscribes from the Pi session listener immediately after aborting compaction, so this PR’s intended event-loss fix is not implemented or superseded. The PR is directionally useful, but the provided review discussion and current call sites show it needs rework before it is merge-ready.

## Best Possible Solution

Keep this PR open for a focused rework: preserve the existing synchronous unsubscribe contract or introduce an explicit awaited cleanup promise across all teardown callers; register the cleanup waiter without a check/register race; resolve only after the final non-retry compaction end; clear timeout handles; and add targeted tests using current `compaction_*` event names with fake timers for timeout behavior.
