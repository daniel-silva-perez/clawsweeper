## Summary

Keep open. Current main still does not implement a `MEDIA:/path#doc` delivery hint or a WhatsApp force-document path for image media. The existing force-document plumbing remains effectively Telegram-specific, while WhatsApp direct-send and auto-reply paths still classify `image/*` as image and send image payloads. Related PR #42044 is useful prior art, but it was closed unmerged.

## Best Possible Solution

Keep this issue open. The best fix is to add a shared outbound media delivery hint, either by parsing a reserved `#doc` suffix before media resolution or by carrying structured `forceDocument`/`asDocument` metadata, then wire it through WhatsApp direct sends and auto-reply delivery so image MIME can be preserved while Baileys receives a document payload. Add parser/loader tests, WhatsApp direct-send tests, auto-reply tests, and docs updates; use #42044 as prior art rather than a close reason.
