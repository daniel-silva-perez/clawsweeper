## Summary

Keep open. Current main still has the Mattermost preview-streaming config mismatch: docs advertise `channels.mattermost.streaming` including `off`, but the Mattermost schema/types/generated metadata do not expose that key, and inbound replies still create/update the draft preview stream without resolving a streaming mode. The open closing PR #73252 is the active implementation path, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for Mattermost draft-preview streaming to be configurable instead of hardcoded on/off across versions.

## Best Possible Solution

Land PR #73252 or an equivalent Mattermost-local fix that accepts canonical `streaming.mode` config at channel and account scope, defaults to today’s preview behavior, disables draft-preview callbacks and in-place finalization when mode is `off`, and updates tests/docs/generated config metadata so the public contract matches runtime behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main’s strict Mattermost schema and generated config metadata lack `streaming`, while the inbound monitor unconditionally creates `createMattermostDraftStream` and routes partial/tool/reasoning updates into it.

## Solution Assessment

Yes. Reusing the existing channel streaming contract is the narrowest maintainable fix; a Mattermost-only `draftPreview` alias would add another public config shape instead of honoring the documented `streaming.mode` path.
