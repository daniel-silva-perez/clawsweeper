## Summary

Keep this issue open. Current main still has the reported Mattermost draft-preview streaming mismatch: docs advertise configurable `channels.mattermost.streaming` including `off`, but the Mattermost schema/types/generated metadata do not expose that key and the inbound runtime still creates and updates the draft preview path without a mode gate. Open PR #73252 is the active closing implementation path, so the issue should not be closed before that PR or an equivalent fix lands.

## What This Changes

The issue requests a Mattermost configuration path that lets users opt out of draft-preview streaming instead of having preview edits hardcoded by version.

## Best Possible Solution

Land PR #73252 or an equivalent Mattermost-local fix that accepts canonical `streaming.mode` config at channel and account scope, preserves the current preview default when omitted, disables draft-preview callbacks/finalization when mode is `off`, and updates tests plus generated config metadata so docs, schema, and runtime match.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main's strict Mattermost schema and generated metadata omit `streaming`, while the inbound monitor unconditionally creates `createMattermostDraftStream` and routes partial/tool/reasoning updates into it.

## Solution Assessment

Yes. Reusing the existing `streaming.mode` channel contract is the narrowest maintainable fix; a Mattermost-only `draftPreview` boolean would add a second public config shape instead of honoring the documented streaming mode path.
