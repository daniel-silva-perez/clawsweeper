## Summary

Keep open. Current main still has the Mattermost draft-preview config mismatch: the docs advertise `channels.mattermost.streaming` and `off`, but the strict schema/generated metadata do not expose `streaming`, and the inbound runtime still creates and updates the draft preview without checking a streaming mode. Open PR #73252 is the active closing implementation path, so this should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests a configurable Mattermost draft-preview streaming opt-out and alignment between Mattermost schema, docs, generated metadata, and runtime behavior.

## Best Possible Solution

Land PR #73252 or an equivalent Mattermost-local fix that accepts canonical `streaming.mode` at channel and account scope, preserves the current preview default when omitted, disables draft-preview callbacks and in-place finalization when mode is `off`, and updates tests/docs/generated config metadata together.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: the strict Mattermost schema rejects the documented `channels.mattermost.streaming` key while the monitor unconditionally creates and updates the draft preview stream.

## Solution Assessment

Yes. Reusing the existing channel streaming contract is the narrowest maintainable direction; adding a Mattermost-only `draftPreview` boolean would create a second public config shape instead of honoring the documented mode path.
