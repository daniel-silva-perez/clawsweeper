## Summary

Keep open. Current main still has the reported Mattermost mismatch: the schema does not accept a `streaming` config, the inbound runtime still creates and updates the draft-preview stream unconditionally, and the docs now say `streaming: "off"` disables preview streaming. The linked open PR #73252 uses closing syntax and is the active implementation path, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for Mattermost draft-preview streaming to be configurable so users can opt out of placeholder-post and in-place update behavior instead of having it hardcoded.

## Best Possible Solution

Land PR #73252 or an equivalent Mattermost-local fix that accepts the canonical `streaming.mode` config at channel and account scope, defaults to today’s preview behavior, disables draft-preview callbacks and in-place finalization when mode is `off`, and updates tests/docs so the public config contract matches runtime behavior.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: current main’s strict Mattermost schema has no `streaming` key, while the monitor unconditionally creates `createMattermostDraftStream` and wires partial/tool/reasoning updates into it.

## Solution Assessment

Yes, the linked PR direction is the narrow maintainable path because it reuses the existing channel streaming config shape instead of adding a one-off Mattermost-only toggle. Maintainers should still review the branch details before merge because it changes schema, runtime delivery, and regression tests.
