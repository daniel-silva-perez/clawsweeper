## Summary

Keep open. Current main still has the Mattermost preview-streaming config mismatch: docs advertise `channels.mattermost.streaming` with `off`, but the Mattermost schema/types/generated metadata do not expose that key and the inbound runtime still creates and updates the draft preview stream without a mode gate. Open PR #73252 is the active closing implementation path, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests a configurable opt-out for Mattermost draft-preview streaming because current behavior is hardcoded and mismatched with the documented channel streaming config.

## Best Possible Solution

Land PR #73252 or an equivalent Mattermost-local fix that accepts canonical `streaming.mode` at channel and account scope, preserves today’s preview default when omitted, disables draft-preview callbacks and in-place finalization when mode is `off`, and updates tests/docs/generated config metadata so the public contract matches runtime behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main’s strict Mattermost schema and generated metadata lack `streaming`, while the inbound monitor unconditionally creates `createMattermostDraftStream` and routes partial/tool/reasoning updates into it.

## Solution Assessment

Yes. Reusing the existing channel streaming contract is the narrowest maintainable direction; a Mattermost-only `draftPreview` boolean would add a second public config shape instead of honoring the documented `streaming.mode` path.
