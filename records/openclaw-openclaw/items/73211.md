## Summary

Keep open. Current main still has the reported Mattermost preview-streaming contract mismatch: docs describe configurable `channels.mattermost.streaming`, but the Mattermost schema/types/generated metadata do not expose that setting and the inbound runtime still creates and updates the draft preview unconditionally. The linked open PR #73252 uses closing syntax for this issue and is the active implementation path, so the issue should remain open until that or an equivalent fix lands.

## What This Changes

The issue asks for Mattermost draft-preview streaming to be configurable instead of hardcoded, including an opt-out path that matches the documented streaming contract.

## Best Possible Solution

Land PR #73252 or an equivalent Mattermost-local fix that accepts the canonical `streaming.mode` config at channel and account scope, defaults to today’s preview behavior, disables draft-preview callbacks and in-place finalization when mode is `off`, and updates tests/docs/generated config metadata so the public contract matches runtime behavior.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: current main’s strict Mattermost schema and generated metadata do not accept `streaming`, while the monitor unconditionally creates the draft stream and routes partial/reasoning/tool updates into it.

## Solution Assessment

Yes. The linked PR direction is the best current path because it reuses the existing channel streaming contract instead of adding a Mattermost-only `draftPreview` alias; maintainers still need to review, rebase, and land or replace it.
