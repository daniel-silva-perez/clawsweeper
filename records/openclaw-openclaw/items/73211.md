## Summary

Keep open. Current main still matches the reported Mattermost bug: the strict Mattermost config schema does not accept `streaming`, the inbound monitor still creates and wires the draft preview path without resolving a preview mode, and the docs advertise `streaming: "off"` as disabling preview streaming. The linked open PR #73252 is the active implementation path and should be reviewed or rebased before this issue closes.

## What This Changes

Allow Mattermost users to opt out of draft-preview streaming through the documented streaming configuration instead of always creating and updating a preview post.

## Best Possible Solution

Land PR #73252 or an equivalent Mattermost-local fix that accepts canonical `streaming.mode` at channel and account scope, keeps default preview behavior, disables draft-preview callbacks and in-place finalization when mode is `off`, and adds regression coverage so docs, schema, and runtime behavior agree.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main rejects the documented `channels.mattermost.streaming` config while the inbound monitor still unconditionally creates and updates the draft-preview stream.

## Solution Assessment

Yes. The best path is to reuse the existing channel streaming config contract, especially `streaming.mode`, rather than adding a Mattermost-only `draftPreview` toggle.
