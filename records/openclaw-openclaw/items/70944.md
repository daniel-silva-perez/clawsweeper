## Summary

Keep open. Current main still does not implement the PR's central behavior: Discord sender identity is not resolved through `session.identityLinks`, Discord inbound/native/component contexts do not carry a trusted sender principal, inbound trusted metadata does not render `sender_principal`, and the canonical identity-link helper is still private rather than exported through `openclaw/plugin-sdk/routing`. The PR is a plausible focused implementation candidate for maintainer review, not obsolete cleanup.

## Best Possible Solution

Keep this PR open for maintainer review. If accepted, the best path is a scoped Discord/inbound-metadata change: resolve canonical principals from `session.identityLinks` using trusted Discord IDs, prefer `pkInfo.sender` for PluralKit when available, thread the value through Discord preflight/process/component/native-command contexts, render `sender_principal` in trusted inbound metadata, and deliberately decide whether the helper belongs in the public plugin SDK or should stay local.
