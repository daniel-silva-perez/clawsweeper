## Summary

Keep open. Current main still does not implement the PR's central behavior: Discord sender identity does not resolve `session.identityLinks`, Discord inbound/native command contexts do not carry `TrustedSenderPrincipal`, the trusted inbound metadata prompt does not render `sender_principal`, and the proposed routing helper is not exported through the plugin SDK. The related #57709 is closed and unmerged in the provided context, so it does not supersede this PR.

## Best Possible Solution

Keep this PR open for normal maintainer review. If accepted, the best implementation path is a scoped Discord/inbound-metadata change: resolve canonical principals from `session.identityLinks` using trusted Discord user IDs, prefer `pkInfo.sender` for PluralKit-proxied messages, thread the value through preflight/process/native-command contexts as `TrustedSenderPrincipal`, render `sender_principal` in trusted inbound meta, and either intentionally promote the routing helper through the plugin SDK with contract updates or keep the helper local if the public seam is not needed.
