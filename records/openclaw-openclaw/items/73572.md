## Summary

Keep open. Current main still has the Signal attachment bug: `getAttachment` responses are capped at a hardcoded 1 MiB before `mediaMaxMb` is enforced. This PR is the active implementation candidate for #73564, and the same contributor-authored issue remains open with closing syntax from the PR.

## Best Possible Solution

Keep the PR open for maintainer review and landing as the implementation candidate for #73564. A small follow-up test around the monitor/getAttachment threading would strengthen the regression coverage before merge; after merge, GitHub can close the linked issue as implemented.
