## Summary

Keep open. Current main still has the Signal attachment cap bug this PR targets: Signal RPC responses are capped at a hardcoded 1 MiB before decoded attachment size is checked against the effective `mediaMaxMb`. This PR is the active implementation candidate for the paired open bug report #73564, and current main does not yet implement the proposed `maxResponseBytes` threading or media-derived attachment cap.

## Best Possible Solution

Keep this PR open for maintainer review and land it, or an equivalent patch, after verifying mergeability and focused Signal tests. The implementation should thread `maxResponseBytes` through the Signal RPC client, derive the `getAttachment` cap from the effective `mediaMaxMb` with base64 and JSON headroom, preserve the 1 MiB default for other RPC calls, and add client plus monitor-level regression coverage. After merge, #73564 can close as implemented.
