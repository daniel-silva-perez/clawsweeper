## Summary

Current main still does not provide a data-preserving migration for legacy `plugins.entries.memory-core.config.dreaming.storage: "inline" | "separate" | "both"`. The manifest still requires an object, AJV plugin config validation still rejects the string before runtime resolver fallback can help, and `doctor --fix` can only quarantine the invalid memory-core config by disabling the plugin and deleting its config payload. This is a valid narrow upgrade regression and should stay open for a focused compatibility repair.

## Best Possible Solution

Keep this issue open and add a narrow memory-core doctor compatibility migration that converts legacy `plugins.entries.memory-core.config.dreaming.storage` strings (`inline`, `separate`, `both`) to `{ mode: <value>, separateReports: false }` before invalid-plugin quarantine runs. Keep the manifest schema strict, leave invalid strings invalid, add focused regression tests, and include a changelog fix entry crediting the reporter.
