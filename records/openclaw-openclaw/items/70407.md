## Summary

Current main partially mitigates the original total CLI outage by letting some recovery/status paths proceed with invalid config, but it still does not migrate the legacy `plugins.entries.memory-core.config.dreaming.storage: "both"` shape. The strict memory-core schema still requires an object, and current `doctor --fix` behavior would quarantine the invalid plugin config rather than preserve the user’s storage mode.

## Best Possible Solution

Keep this issue open and add a narrow memory-core compatibility migration that converts legacy `dreaming.storage` string values (`inline`, `separate`, `both`) into the supported object shape while preserving the configured mode. The fix should run through the doctor/setup migration path, keep the manifest schema strict, and add focused regression coverage plus a changelog fix entry.
