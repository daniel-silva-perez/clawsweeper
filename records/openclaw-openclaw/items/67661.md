## Summary

Keep open. Current main still has the reported call-site gap: both incomplete-turn auth-profile failure writes omit the active model id, while model-scoped cooldown persistence depends on that id. The linked implementation PR #67662 is still open and unmerged in the provided GitHub context, so closing this issue now would preempt the active fix candidate.

## What This Changes

The issue asks for incomplete-turn auth-profile failure writes in runEmbeddedPiAgent to include the active modelId so same-session model-scoped cooldowns do not retry an exhausted model.

## Best Possible Solution

Review and either land or close PR #67662. The desired end state is a narrow runner fix that passes the active modelId into both incomplete-turn failure writes and keeps focused regression coverage for reasoning-only exhaustion and general incomplete-turn cooldown persistence.
