## Summary

Keep open: current main and the latest release still materialize Discord parent-channel ACP matches with the parent conversation ID, so sibling Discord threads under one configured parent can still share the same configured ACP session key. The PR is a valid implementation candidate, with prior review discussion already narrowing the fix to Discord while preserving Feishu-style canonical remaps, but it needs maintainer rebase/review against the current split Discord conversation helper and coordination with overlapping ACP thread-binding work.

## What This Changes

The PR changes Discord configured ACP parent-channel matching to use the runtime thread ID for session-key materialization, adds regression coverage for distinct sibling thread keys and reverse-lookup limits, and adjusts configured ACP reset metadata handling plus small channel lint/test cleanups.

## Best Possible Solution

Land this PR after porting the Discord matcher change to current main, or land an equivalent narrow patch that makes parent-channel Discord matches materialize the live thread ID while preserving channel-canonical match remaps for Feishu and other plugins, with regression coverage for sibling Discord thread keys and configured-format reset metadata clearing.
