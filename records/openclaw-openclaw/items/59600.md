## Summary

Keep open. Current main has improved the supported exec allowlist path, but the issue's exact `tools.exec.safeBins` compound-command reproduction is not implemented: safeBins still match the executable name only and require a safe-bin profile. Docs now clarify that safeBins are a narrow stdin-only fast path, not a generic command trust list, so closing this as implemented would be misleading without an explicit maintainer product/security decision.

## Best Possible Solution

Keep the issue open for maintainer judgment. If the current boundary is intended, a maintainer should close it with a clear docs-backed explanation that compound CLI trust belongs in exec approvals allowlist entries, not `tools.exec.safeBins`. If the original request is still desired, implement it as a carefully reviewed explicit allowlist feature rather than broadening the stdin-only safeBins fast path.
