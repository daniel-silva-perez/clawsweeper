## Summary

Current main still preserves the reported memory-core behavior: default dreaming promotion requires repeated signal and query/day diversity, and tests explicitly assert that a one-off high-scoring recall does not reach MEMORY.md under default thresholds. The newer comment reinforces that this is a real bundled memory-quality gap rather than an obsolete report.

## Best Possible Solution

Keep this open for memory-core maintainer/product design. The best implementation is a tested durable lane for high-priority single-event candidates, such as explicit remember-directive metadata, a manual priority flag, critical-feedback classification, or a session-end sweep with narrow bypass semantics for signal and diversity gates. Update memory-core tests and https://docs.openclaw.ai/concepts/dreaming once the behavior is defined.
