## Summary

Current main still has the reported memory-core behavior. Deep dreaming promotion is thresholded on repeated signal and query/day diversity, the automatic dreaming flow passes those gates through ranking and apply, and an existing test explicitly asserts that a one-off high-scoring recall does not reach MEMORY.md under default thresholds. The newer KinthAI discussion reinforces this as a real product-quality gap rather than an obsolete report.

## Best Possible Solution

Keep this open for memory-core maintainer/product design. The best path is a tested narrow lane for high-importance single-event candidates, such as explicit remember-directive metadata, a manual priority flag, critical-feedback/emotional-importance classification, or a session-end sweep. It should update memory-core ranking/apply tests and the public Dreaming/Memory CLI docs, while preserving the default noise-control gates for ordinary candidates.
