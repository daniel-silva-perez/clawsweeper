## Summary

Keep this open. Current main still uses frequency and diversity gates for deep memory promotion, and the bundled regression test explicitly preserves the behavior that a one-off high-scoring recall does not reach `MEMORY.md` under defaults. The KinthAI follow-up and adjacent open memory-quality issues make this a live memory-core product gap, not an obsolete report.

## What This Changes

The issue asks memory-core to support durable promotion of critically important single-occurrence events that do not meet repeated-recall promotion thresholds.

## Best Possible Solution

Define a narrow memory-core lane for high-importance singleton candidates while preserving the default repeated-signal gates for ordinary material. The accepted design should include metadata or classification that explains why the candidate bypassed frequency/diversity gates, focused ranking/apply tests for both accepted one-off and rejected noisy singleton cases, and docs updates for https://docs.openclaw.ai/concepts/dreaming and https://docs.openclaw.ai/cli/memory.

## Reproduction Assessment

Yes. Current source and `extensions/memory-core/src/dreaming.test.ts:1969` give a high-confidence reproduction path: record a single high-scoring short-term recall, run deep dreaming with default thresholds, and `MEMORY.md` remains empty.

## Solution Assessment

Unclear. The issue lists plausible directions rather than a patch; a simple threshold change is not the best fix because it would weaken memory quality, while a durable priority/directive/classifier/review lane needs maintainer product agreement.
