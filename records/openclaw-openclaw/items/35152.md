## Summary

Keep this PR open. Current main has adjacent installer-time duplicate npm global install warnings and doctor-side gateway service/split-brain safeguards, but it still does not implement the PR's intended on-demand `openclaw doctor` duplicate binary/install-root inventory with versions and remediation.

## Best Possible Solution

Refresh or supersede this PR with a current doctor health implementation that safely inventories duplicate OpenClaw binaries/package roots, deduplicates by realpath, reports versions and relevant service metadata, respects the existing `doctor --deep` boundary for system service scans, uses safe process APIs such as `execFileSync`, derives npm package/bin locations correctly, and adds focused regression tests.
