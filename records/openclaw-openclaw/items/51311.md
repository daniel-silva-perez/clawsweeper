## Summary

Keep open. Current main still has the unguarded `WebSocketTaskBox.sendPing()` checked-continuation path, and the keepalive loop still calls it. The PR is a narrow, plausible fix for a live crash path, but it is not implemented on main and the related open PRs are not clearly canonical or superseding yet.

## Best Possible Solution

Keep this PR open as a plausible focused fix. Add the explicit `import os`, add a narrow duplicate-callback regression test if feasible, compare it with #41653 and #51371, then land one canonical `GatewayChannel.swift` guard and close the other duplicate PRs after merge or maintainer selection.
