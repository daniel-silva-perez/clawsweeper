## Summary

Keep this PR open. Current main still has the unguarded `WebSocketTaskBox.sendPing()` checked-continuation wrapper, and the keepalive loop still calls it, so the crash path described by the PR is not implemented on main. The PR is narrow and plausibly correct, but it should address the explicit `import os` concern, add focused duplicate-callback coverage if feasible, and be reconciled with the other open PRs for the same WebSocket ping double-resume bug.

## Best Possible Solution

Keep this PR open as a plausible focused fix.
