## Summary

Keep this PR open. Current main still has the unguarded `WebSocketTaskBox.sendPing()` checked-continuation wrapper, and the keepalive loop still calls that wrapper, so the crash path described by the PR is not implemented on main. The PR is narrow and plausibly useful, but it should resolve the explicit `import os` concern, add focused duplicate-callback coverage if feasible, and be reconciled with the other open PRs for the same WebSocket ping double-resume bug.

## Best Possible Solution

Keep this PR open as a plausible focused fix. Add the explicit `import os`, add a narrow duplicate-callback regression test if feasible, compare it with #41653 and #51371, then land one canonical `GatewayChannel.swift` guard and close the remaining duplicate PRs after merge or explicit maintainer selection.
