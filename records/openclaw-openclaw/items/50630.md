## Summary

Keep open. Current main still permits the reported `gateway.auth.mode="none"` plus `gateway.tailscale.mode="serve"` startup path, the auth path still treats mode `none` as credential-free, and open PR #50631 uses closing syntax for this issue, so the issue should remain open until that PR or an equivalent hardening change is resolved.

## What This Changes

The issue asks OpenClaw to reject or harden `gateway.auth.mode="none"` when `gateway.tailscale.mode="serve"` exposes the loopback gateway through Tailscale Serve.

## Best Possible Solution

Review and either land PR #50631 or replace it with an equivalent narrow gateway hardening PR that rejects explicit `gateway.auth.mode="none"` with Tailscale Serve, preserves trusted loopback-only no-auth usage, preserves Serve with real auth or verified Tailscale identity where intended, and updates the runtime-config tests plus public gateway docs if the startup contract changes.
