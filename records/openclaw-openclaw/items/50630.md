## Summary

Keep open. Current main still permits the reported `gateway.auth.mode="none"` plus `gateway.tailscale.mode="serve"` startup path, the HTTP/auth path still treats mode `none` as credential-free, and open PR #50631 uses closing syntax for this issue. Per policy, this issue should stay open until that PR or an equivalent gateway hardening change is merged or intentionally closed.

## What This Changes

The issue requests rejecting or otherwise hardening `gateway.auth.mode="none"` when `gateway.tailscale.mode="serve"` exposes the gateway through Tailscale Serve.

## Best Possible Solution

Keep this issue open and route it through review of PR #50631 or an equivalent narrow gateway hardening PR. The fix should reject explicit `gateway.auth.mode="none"` when `gateway.tailscale.mode="serve"` is active, preserve trusted loopback-only `mode: none`, preserve Serve with real auth or verified Tailscale identity where intended, and update `src/gateway/server-runtime-config.test.ts` plus gateway docs if the startup contract changes.
