## Summary

Keep open. Current main still accepts the reported Gateway startup shape, `auth.mode="none"` still short-circuits authentication before Tailscale identity-header auth can run, and linked PR #50631 remains open with closing syntax.

## What This Changes

The issue asks OpenClaw to reject or otherwise harden `gateway.auth.mode="none"` when `gateway.tailscale.mode="serve"` exposes the Gateway over a Tailnet.

## Best Possible Solution

Harden Gateway startup so explicit no-auth mode cannot be combined with Tailscale Serve remote exposure unless maintainers intentionally define a verified identity-bearing contract for that mode, while preserving trusted loopback-only no-auth use and updating runtime-config tests plus public Gateway/security docs.
