## Summary

Keep open. Current main still accepts the reported Tailscale Serve plus no-auth startup shape, `mode: "none"` still authorizes before Tailscale identity-header verification, and open PR #50631 remains an unmerged implementation candidate linked with closing syntax.

## What This Changes

The issue reports that `gateway.auth.mode: "none"` combined with `gateway.tailscale.mode: "serve"` exposes the Gateway to Tailnet Serve callers without requiring token, password, or verified identity auth.

## Best Possible Solution

Harden Gateway startup so explicit no-auth mode cannot be combined with Tailscale Serve remote exposure unless maintainers intentionally define a separate verified identity-bearing contract, while preserving trusted loopback-only no-auth use and updating runtime-config tests plus public gateway/security docs for the chosen contract.
