## Summary

Keep open. The provided Apr 29 reporter update and prior ClawSweeper comments still match current main: startup validation rejects Funnel without password but still has no Serve plus no-auth rejection, and an open unmerged PR already references this issue with closing syntax.

## What This Changes

The issue asks OpenClaw to reject or harden `gateway.auth.mode="none"` with `gateway.tailscale.mode="serve"` so Tailnet Serve exposure cannot bypass Gateway authentication.

## Best Possible Solution

Harden Gateway startup so explicit `mode: "none"` cannot be combined with Tailscale Serve remote exposure unless maintainers intentionally define a verified identity-bearing contract, while preserving trusted loopback-only no-auth use and updating runtime-config tests plus public gateway/security docs for the final contract.
