## Summary

Keep this issue open. Current main still keeps the reported Feishu metadata shape in agent-facing text, and the Feishu outbound path can send model-produced text without stripping that legacy inline form, so the cleanup has not clearly landed.

## What This Changes

The issue requests that Feishu quoted/replied messages stop exposing internal metadata such as `message_id`, `sender_id`, and raw `ou_...` IDs in user-visible message content.

## Best Possible Solution

Move Feishu message and sender identifiers out of inline body text and rely on the existing structured inbound metadata fields, with regression coverage proving quoted/replied Feishu turns retain tool context while visible reply text does not contain `[message_id: ...]` or raw `ou_...:` labels.

## Reproduction Assessment

Yes. A focused unit reproduction can use `buildFeishuAgentBody` or the current Feishu handler with a quoted message; existing tests already assert the problematic `[message_id: ...]\nou-...: ...` `BodyForAgent` shape.

## Solution Assessment

No. Current main is not the best end state because it keeps volatile Feishu identifiers in inline prompt body text even though structured context fields already exist; the maintainable fix is a Feishu-owned cleanup plus targeted sanitizer coverage, not a broad core migration.
