## Summary

Keep open. Current main still builds Feishu agent-facing message text with the same inline `[message_id: ...]` and sender-id-style prefix described in this issue, including quoted-message context, and I did not find a current Feishu fix or config option that makes those legacy prompt tags safely invisible if echoed into a user-visible reply.

## Best Possible Solution

Fix this in the Feishu plugin by moving message id, sender id, and quoted-message metadata out of inline natural-language prompt text and into the existing structured inbound metadata/context fields, or add a narrowly scoped output sanitizer for the legacy Feishu preface before any visible reply is sent. Add regression coverage for a Feishu quoted reply where the model echoes `[message_id: ...]` / `ou_xxx:` so the final delivered text and native quote preview stay clean while agent routing still has `MessageSid`, `SenderId`, `ReplyToId`, and `ReplyToBody`.
