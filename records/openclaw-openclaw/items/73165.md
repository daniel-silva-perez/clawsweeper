## Summary

Keep this PR open. Current main still does not implement the proposed MCP readiness gate for advertised skills, so the PR is not obsolete or implemented-on-main. The patch remains a live implementation candidate, but it needs maintainer review because the readiness logic still appears to allow a skill-level denial to fall through to a server-level grant, and the PR also carries unrelated drift outside the stated readiness gate.

## Best Possible Solution

Continue review rather than close. If maintainers want this core safety behavior, keep the readiness work focused on skills/MCP prompt exposure, fix skill-record precedence, decide and document stale-readiness policy, keep the smoke script opt-in, and split or rebase unrelated package-boundary, qa-channel, cache-trace, and gateway changes before merge.
