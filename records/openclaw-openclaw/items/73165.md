## Summary

Keep open. Current main still does not implement the MCP skill metadata/readiness gate, so this PR is not obsolete, but the PR head is not safe to merge as-is because an explicit skill-level readiness denial can be overridden by a server-level grant and the branch mixes unrelated surfaces that need maintainer review.

## What This Changes

The PR adds MCP metadata parsing and an `mcp-readiness.json` prompt gate for MCP-backed skills, plus a dry-run mcporter smoke script and several qa-channel, gateway, package-boundary, cache-trace, and test adjustments.

## Best Possible Solution

Keep review active and narrow the branch to the skills/MCP prompt-exposure contract: make skill-level readiness records decisive over server-level records, define the freshness policy, add regression coverage for denial precedence and owner scoping, keep the smoke script opt-in, and rebase or split unrelated gateway/qa/package-boundary changes.

## Reproduction Assessment

Yes. A high-confidence static reproduction is a readiness file with `skills["mcp-memory"] = { declared: true, callable: false }` and `servers["memory"] = { declared: true, callable: true }`; the PR code rejects the skill record, then falls through to the server record and returns true.

## Solution Assessment

No. The proposed direction is plausible, but this PR is not the best mergeable shape until explicit skill-level denies win, stale-readiness behavior is decided, regression tests cover the gate, and unrelated changes are separated or rebased away.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Honor explicit skill readiness denials:** `src/agents/skills/workspace.ts:174-180`
  - body: When a readiness file contains a skill-specific record that is not callable, the code falls through to the server-level record and can return true. That means `skills["mcp-memory"].callable = false` or an owner mismatch is ignored if `servers["memory"].callable = true`, defeating the prompt-exposure gate this patch adds. Treat an existing skill record as decisive before checking the server record.
  - confidence: 0.92
