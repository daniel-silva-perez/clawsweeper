## Summary

Keep this PR open. Current main still has the Feishu-specific documentation gap for `match.accountId`, and the PR is not implemented or superseded. The proposed docs need a wording revision, though: Feishu `defaultAccount` is documented as outbound/default API selection, while inbound routing uses the concrete runtime account id and treats omitted binding `accountId` as the literal `default` routing bucket.

## Best Possible Solution

Keep the PR open and revise the Feishu docs patch. Add `match.accountId` to the Feishu routing fields, explain that omitted means the literal default routing scope/account id (`default`) rather than all accounts, explicit ids such as `main` match that named Feishu account, and `accountId: "*"` matches any configured Feishu account. Also distinguish this from `channels.feishu.defaultAccount`, which controls outbound/default account selection. Coordinate wording with #49769, but keep a Feishu-specific example on `docs/channels/feishu.md`.
