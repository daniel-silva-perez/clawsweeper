## Summary

Keep this PR open. Current main still has the Feishu-specific documentation gap for `match.accountId`, and the PR is not implemented or superseded. The proposed docs direction is useful, but the current wording should be revised so `default account` cannot be confused with `channels.feishu.defaultAccount`, which the Feishu page documents as outbound-only selection.

## Best Possible Solution

Keep this PR open and revise the Feishu docs patch. Add `match.accountId` to the Feishu routing fields, explain that omitted `match.accountId` maps to the default routing bucket rather than all accounts, explicit ids match concrete Feishu runtime account ids, and `accountId: "*"` matches any configured Feishu account. Also distinguish this from `channels.feishu.defaultAccount`, which controls outbound/default API account selection. Coordinate terminology with #49769 while keeping a Feishu-specific example in `docs/channels/feishu.md`.
