## Summary

Keep this PR open. Current main still does not contain the Feishu-specific `match.accountId` routing clarification or wildcard example, and the PR is not implemented or superseded. The proposed docs-only direction is useful, but the wording should be revised so users do not confuse the routing default bucket with `channels.feishu.defaultAccount`, which the Feishu page describes as outbound API selection.

## Best Possible Solution

Keep this PR open and revise the Feishu docs patch. Add `match.accountId` to the Feishu routing fields, explain omitted `match.accountId` as the default routing bucket rather than all accounts, document explicit ids and `accountId: "*"`, and distinguish that from `channels.feishu.defaultAccount` for outbound/default API account selection. Coordinate wording with #49769 while preserving a Feishu-specific example in `docs/channels/feishu.md`.
