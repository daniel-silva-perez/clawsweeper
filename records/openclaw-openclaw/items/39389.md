## Summary

Keep this PR open. Current main still lacks the Feishu-specific `match.accountId` routing clarification and wildcard example, and the proposed docs-only change is not implemented or superseded. The patch needs wording changes before merge so users do not confuse inbound binding routing with `channels.feishu.defaultAccount`, which the Feishu page documents as outbound API selection.

## Best Possible Solution

Keep this PR open and revise the Feishu docs patch. Add `match.accountId` to the Feishu routing fields, explain omitted `match.accountId` as the default routing bucket rather than all accounts, document explicit ids and `accountId: "*"`, and clearly distinguish that from `channels.feishu.defaultAccount` for outbound/default API account selection. Coordinate wording with #49769 while preserving a Feishu-specific example in `docs/channels/feishu.md`.
