## Summary

Keep this issue open. The current bundled Twitch plugin still matches the reported lifecycle bug: `gateway.startAccount` awaits `monitorTwitchProvider`, but that monitor resolves immediately after registering the IRC message handler and returns `{ stop }`. The gateway supervisor still treats a resolved `startAccount` as a clean channel exit and schedules auto-restart, which is the exact loop reported here and in the related LINE/Nextcloud/Google Chat issues.

## Best Possible Solution

Fix the bundled Twitch plugin by wrapping `monitorTwitchProvider` in `runStoppablePassiveMonitor` or `runPassiveAccountLifecycle`, then add a Twitch lifecycle test using `startAccountAndTrackLifecycle`/`expectStopPendingUntilAbort` so `startAccount` remains pending until abort and calls the returned `stop` once. The gateway restart behavior should remain as-is unless maintainers intentionally change the broader channel contract.
