## Summary

Keep this PR open. Current main still sends Feishu card-worthy outbound text through the card helpers without a 230099/11310 table-limit fallback, while this PR is an active focused implementation with no definite blocking correctness finding from the static review.

## What This Changes

The PR adds a Feishu CardKit table-limit error parser, wraps outbound markdown and structured card sends with plain-text fallback for 230099/11310 failures, preserves thread routing on those fallback paths, and adds focused tests.

## Best Possible Solution

Land a narrow Feishu outbound fallback that catches the known CardKit table-limit rejection, resends the same content through the existing plain-text path with thread metadata preserved, and keep reply-dispatcher or streaming-card fallback changes in the dedicated related work.

## Reproduction Assessment

Yes. Current main can be inspected through sendOutboundText and feishuOutbound.sendText: both route card-worthy outbound text directly to Feishu card send helpers, and related issue #43690 provides the concrete 230099/11310 table-limit error shape for messages with many markdown tables.

## Solution Assessment

Yes for this PR's stated outbound scope. The proposed approach reuses the existing sendMessageFeishu fallback, preserves non-table-limit errors by rethrowing, and avoids bundling broader Feishu delivery changes into this PR.
