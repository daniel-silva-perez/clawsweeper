## Summary

Keep open. Current main still only forwards Feishu `MessageThreadId` for topic-session scopes, while this PR asks for broader `root_id` forwarding into agent-visible `topic_id` metadata. ## What This Changes

The PR adds Feishu `ctx.rootId` and `ctx.parentId` to the inbound context payload as `MessageThreadId` and `ReplyToId`.
