## Summary

Keep open. Current main still has the reported Discord voice gap: voice transcript turns resolve Discord channel config for authorization, but the successful voice path never carries the resolved per-channel `systemPrompt` into the agent run, while Discord text/native ingress already carries that value through `GroupSystemPrompt` into `extraSystemPrompt`.

## What This Changes

Discord voice transcript turns should honor per-channel `systemPrompt` overrides the same way Discord text and native-command ingress do.

## Best Possible Solution

Carry the trusted resolved Discord channel config through the voice authorization path, derive the same trimmed group prompt used by text/native ingress, and pass it as `extraSystemPrompt` to `agentCommandFromIngress` for authorized guild voice transcript runs while keeping STT transcript content untrusted.
