## Summary

Keep open. Current main still has the reported Discord realtime voice gap: voice transcript turns resolve Discord channel config for authorization but do not carry the prompt-bearing config into the agent run, while normal Discord text/native ingress already builds `GroupSystemPrompt` and the shared reply runner forwards it as `extraSystemPrompt`.

## What This Changes

Apply Discord per-channel `systemPrompt` overrides to realtime voice transcript agent runs, matching normal Discord text ingress behavior.

## Best Possible Solution

Carry the trusted resolved Discord channel config or derived group prompt through the voice ingress path and pass the same trimmed per-channel prompt as `extraSystemPrompt` to the agent run, while keeping STT transcript text, speaker labels, and channel metadata in user/untrusted context rather than trusted system prompt content.
