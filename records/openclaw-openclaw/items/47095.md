## Summary

Keep this issue open. Current main still routes Discord voice transcript turns directly through `agentCommandFromIngress` without resolving or forwarding the per-channel Discord `systemPrompt`, while the normal Discord text path does build `GroupSystemPrompt` and the shared reply runner forwards it as `extraSystemPrompt`. The prior ClawSweeper comment remains accurate on current main; the related Slack item was separate, and #53562/#53741 are adjacent session-channel transcript-routing work rather than this prompt-injection bug.

## Best Possible Solution

Fix this in the Discord plugin voice path: resolve the same guild/channel config used by text ingress or voice access, reuse `buildDiscordGroupSystemPrompt`/`buildDiscordInboundAccessContext`, and pass the resulting prompt to the voice `agentCommandFromIngress` call as `extraSystemPrompt`. Add a focused voice manager regression test for `channels.discord.guilds.<guild>.channels.<voiceChannel>.systemPrompt` and keep #53562/#53741 as separate text-channel transcript routing work.
