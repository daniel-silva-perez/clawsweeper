## Summary

Keep open. Current main still has the reported Discord voice gap: voice transcript runs resolve Discord channel config for authorization, but the successful voice path does not carry the per-channel `systemPrompt` into `agentCommandFromIngress`, while Discord text/native ingress already routes the same config prompt through `GroupSystemPrompt` and then into `extraSystemPrompt`.

## What This Changes

Apply Discord per-channel `systemPrompt` overrides to Discord voice transcript agent runs the same way Discord text ingress does.

## Best Possible Solution

Carry the trusted resolved Discord channel config through successful voice authorization, reuse `buildDiscordGroupSystemPrompt` for guild voice transcript runs, and pass the resulting prompt as `extraSystemPrompt` to `agentCommandFromIngress` while keeping STT transcript text and channel metadata as untrusted user/context content.

## Reproduction Assessment

Yes. The source-level reproduction path is high confidence: configure a Discord guild voice channel with `systemPrompt`, process a voice segment, and inspect the `agentCommandFromIngress` options; current main has no path that can include that prompt.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing Discord channel config resolution and prompt helper in the Discord voice plugin, rather than changing the shared agent runner or adding a core special case.
