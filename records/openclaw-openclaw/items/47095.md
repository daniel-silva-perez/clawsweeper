## Summary

Keep this issue open. Current main still processes Discord voice transcript turns through the voice manager’s direct `agentCommandFromIngress` path without resolving the per-channel Discord `systemPrompt` or passing any `extraSystemPrompt`, while the Discord text ingress path still builds `GroupSystemPrompt` and the shared reply runner forwards it into the agent run.

## What This Changes

Discord voice transcript turns should apply the same per-channel systemPrompt override that Discord text ingress applies.

## Best Possible Solution

Fix this in the Discord plugin voice path by resolving the same guild/channel config used by Discord text ingress or voice access, deriving the configured group system prompt with the existing Discord helper, and passing it to the voice `agentCommandFromIngress` call as `extraSystemPrompt`. Add a focused voice manager regression test and keep the session-channel routing work in #53562/#53741 separate.
