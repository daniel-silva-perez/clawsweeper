## Summary

Keep open. Current main still has the reported Discord voice gap: realtime transcript turns call the agent directly without carrying the per-channel Discord `systemPrompt`, while the text/native Discord ingress path already resolves that config into `GroupSystemPrompt` and the shared reply runner forwards it as `extraSystemPrompt`.

## What This Changes

The issue requests that Discord voice realtime transcript agent turns apply the configured per-channel `systemPrompt` override, matching Discord text message ingress.

## Best Possible Solution

Update the Discord plugin voice ingress so authorized guild voice transcript runs resolve the same trusted channel config as text/native ingress, build the per-channel prompt with the existing Discord helper, and pass it as `extraSystemPrompt` to the agent while keeping STT transcript and channel metadata out of trusted system prompt text.
