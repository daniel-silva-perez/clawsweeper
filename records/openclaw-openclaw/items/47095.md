## Summary

Keep open. Current main still has the reported Discord realtime voice gap: the voice transcript path authorizes against Discord channel config but never carries the per-channel systemPrompt into the agent run, while Discord text and native-command ingress already build and forward that prompt.

## What This Changes

Discord voice transcript turns should apply the same per-channel systemPrompt override that Discord text/native ingress applies.

## Best Possible Solution

Update the Discord voice ingress path to reuse the trusted resolved channel config, derive the same trimmed group system prompt used by text/native ingress, and pass it as extraSystemPrompt to the voice agent run while keeping STT transcript text and channel metadata in untrusted user/message context.
