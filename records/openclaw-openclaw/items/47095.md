## Summary

Keep open. Current main still has the reported Discord voice gap: voice transcript runs resolve Discord channel config for authorization, but the successful voice path never carries the per-channel `systemPrompt` into the agent run, while the text/native Discord path already does.

## What This Changes

Discord voice transcript turns should apply the same per-channel Discord `systemPrompt` override that Discord text-channel ingress applies.

## Best Possible Solution

Update Discord voice ingress so successful guild voice transcript runs reuse the trusted resolved channel config, build the same per-channel prompt used by text/native ingress, and pass it as `extraSystemPrompt` to the agent while keeping STT transcript text and channel metadata out of trusted system prompt content.
