## Summary

Keep open. Current main still sends Discord voice transcript turns through `agentCommandFromIngress` without `extraSystemPrompt`, while Discord text/native ingress resolves `channelConfig.systemPrompt` into `GroupSystemPrompt` and the shared reply runner forwards that into the agent prompt.

## What This Changes

Discord voice transcript turns should honor the same per-channel Discord `systemPrompt` override that text-channel ingress applies.

## Best Possible Solution

Fix this in the Discord plugin voice path by reusing the existing Discord channel-config and group-prompt helpers, forwarding the trusted per-channel prompt as `extraSystemPrompt` on voice transcript runs, and adding focused voice-manager coverage without folding in the separate text-session routing feature.
