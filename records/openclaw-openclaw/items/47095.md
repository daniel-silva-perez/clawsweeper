## Summary

Keep open. Current main still sends Discord voice STT turns directly through `agentCommandFromIngress` without the Discord text-path inbound context that computes `GroupSystemPrompt` from `channels.discord.guilds.<guild>.channels.<channel>.systemPrompt`. The related Slack prompt issue does not supersede this Discord voice-specific gap.

## Best Possible Solution

Fix this in the Discord voice owner module by resolving the channel config for the voice session channel, reusing the existing Discord group-system-prompt helper or inbound-context builder, and passing the resulting prompt into the voice agent run as `extraSystemPrompt` or via the normal Discord inbound/reply pipeline. Add a Discord voice manager regression test that configures `channels.discord.guilds.<guild>.channels.<voiceChannel>.systemPrompt` and asserts the agent run receives it.
