## Summary

Keep open. Current main still has the reported Discord voice ingress gap: realtime voice transcript runs authorize against Discord channel config but never pass the resolved per-channel `systemPrompt` into the agent run, while Discord text ingress already turns that config into `GroupSystemPrompt` and the shared reply runner forwards it as `extraSystemPrompt`.

## What This Changes

The issue requests that Discord realtime voice transcript turns apply the same per-channel `systemPrompt` override that Discord text-channel ingress applies.

## Best Possible Solution

Carry the trusted resolved Discord channel config through the voice ingress authorization path, derive the same trimmed group system prompt used by text/native ingress, and pass it as `extraSystemPrompt` to `agentCommandFromIngress` for guild voice transcript runs while keeping STT transcript text and channel metadata in untrusted message context.
