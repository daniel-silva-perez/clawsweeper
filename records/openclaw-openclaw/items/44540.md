## Summary

Keep open. Current main still applies the same weighted BM25 + vector merge to all builtin memory results, and multimodal image/audio chunks are indexed as short label text without a persisted modality signal that hybrid ranking can use. The existing tests cover multimodal indexing and literal `image`/`audio` searches, but not the reported natural-language multimodal recall failure under default hybrid weights.

## Best Possible Solution

Fix this in the builtin memory engine by carrying reliable image/audio modality metadata from multimodal indexing into stored chunks and search results, then make hybrid merging avoid BM25 dampening for non-text vector candidates while preserving existing weighted BM25 + vector behavior for text chunks. Add a regression test with mixed Markdown plus image/audio candidates showing a natural-language visual/audio query surfaces multimodal results under default hybrid weights.

## Clownfish Work Prompt

Fix the builtin memory-search ranking bug reported in https://github.com/openclaw/openclaw/issues/44540. Current hybrid scoring penalizes multimodal image/audio chunks because they carry little label text and get `textScore: 0`, so default 0.7/0.3 BM25+vector ranking can bury good image/audio vector matches. Implement this in the builtin memory engine, not QMD/LanceDB: carry a reliable image/audio modality signal from multimodal indexing into persisted chunks and search results, with a backward-compatible schema/migration/reindex strategy as needed; update `mergeHybridResults` so non-text vector candidates are not dampened by the BM25 weight while preserving existing text-on-text hybrid ranking, minScore fallback, temporal decay, and MMR behavior. Avoid path-only modality inference except as a clearly documented migration fallback. Add a regression test with mixed Markdown plus image/audio candidates and deterministic embeddings showing a natural-language visual/audio query surfaces the multimodal result under default 0.7/0.3 hybrid weights. Update docs/config help only if the public behavior or schema contract changes. Add a user-facing changelog fix entry if repo policy requires one, crediting @markclawbot; do not change provider defaults, fallback policy, or unrelated memory backends.
