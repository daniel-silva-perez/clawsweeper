## Summary

Keep this issue open. Current main still does not prove the reported custom/local `copilot-api` Anthropic Messages path is fixed: the malformed tool-call argument repair is gated to Kimi on `anthropic-messages` and all `openai-completions`, while custom Anthropic-compatible provider ids remain outside that gate.

## What This Changes

The issue reports that long, tool-heavy webchat sessions using a custom `copilot-api` Anthropic Messages proxy can drop `write`/`exec` tool-call arguments, especially large `content` or `command` fields, leaving validation to see `{}` or partial args.

## Best Possible Solution

Keep this open until a maintainer can capture or replay a custom/local `anthropic-messages` stream with large fragmented `input_json_delta` tool arguments, then add a provider-safe repair or explicit parse/size diagnostic before empty arguments reach tool validation.
