## Summary

Keep open. Current main still does not prove the reported custom `copilot-api` `anthropic-messages` long-session argument-loss path is fixed: the repair wrapper exists, but the runner only enables it for normalized Kimi Anthropic Messages or OpenAI Completions, while custom Anthropic-compatible providers continue through the direct `input_json_delta` parse path. The related Kimi reports are closed and a custom-endpoint beta-header mitigation exists, but neither supersedes this report or #55224's broader long-session parameter-loss cluster.

## What This Changes

The issue asks OpenClaw to reliably preserve `write` and `exec` tool arguments in long, tool-heavy conversations using a custom Anthropic-compatible provider, especially for large CJK content.

## Best Possible Solution

Add or replay a custom/local `anthropic-messages` stream fixture with large fragmented `write` and `exec` `input_json_delta` arguments, then implement provider-safe repair or explicit diagnostics before empty arguments reach tool validation while preserving the custom-proxy beta-header suppression.
