## Summary

Keep this issue open. Current main has a useful new mitigation for custom Anthropic-compatible proxies by suppressing implicit Anthropic beta headers, but it still does not prove the reported long-session `copilot-api` argument-loss path is fixed: malformed argument repair is still gated away from arbitrary `anthropic-messages` provider IDs, and there is no custom large fragmented `write`/`exec` stream fixture covering this report.

## What This Changes

The issue reports intermittent loss of required `write` and `exec` tool arguments in long Anthropic-compatible webchat sessions, especially with large CJK content via a local `copilot-api` proxy.

## Best Possible Solution

Capture or replay a custom/local `anthropic-messages` stream with large fragmented `write` and `exec` `input_json_delta` arguments, then add a provider-safe repair or explicit parse/size diagnostic before empty arguments reach tool validation, while preserving the new custom-proxy beta-header suppression.
