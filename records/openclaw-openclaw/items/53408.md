## Summary

Keep open. Current main still leaves the reported custom Anthropic-compatible long-session argument loss unproven and likely uncovered: malformed argument repair is gated away from custom `anthropic-messages` providers, while the Anthropic transport still parses partial tool JSON directly into `{}` on parse failure.

## What This Changes

The issue asks OpenClaw to reliably preserve `write`, `exec`, `edit`, and related tool arguments in long, tool-heavy sessions, especially through Anthropic-compatible custom/proxy providers.

## Best Possible Solution

Keep this as the canonical reliability bug until a maintainer captures or synthesizes a custom Anthropic-compatible stream fixture with large fragmented `write`, `exec`, and preferably `sessions_spawn` arguments, then adds provider-safe repair or explicit diagnostics for malformed live tool-argument assembly.

## Reproduction Assessment

No. The issue and #55224 give credible long-session reproduction descriptions, but current main does not include a deterministic fixture or live replay for the reported custom Anthropic-compatible stream path.

## Solution Assessment

Unclear. The current scoped repair is appropriate for the known Kimi and OpenAI-compatible regressions, but it is not a complete solution for this issue; the safer next step is a fixture-backed Anthropic-compatible repair or diagnostic path rather than a blind provider-wide toggle.
