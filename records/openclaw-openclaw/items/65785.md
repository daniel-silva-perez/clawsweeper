## Summary

Keep open. Current main still does not implement this PR’s intended MiniMax web_search OAuth-token path: the runtime key resolver only reads the configured web-search key, `MINIMAX_CODE_PLAN_KEY`, `MINIMAX_CODING_API_KEY`, and `MINIMAX_API_KEY`, while provider metadata, tests, and web-search docs still omit `MINIMAX_OAUTH_TOKEN`. The PR remains a plausible focused fix for linked open bug #65768, though it needs rebasing onto the current runtime split and should account for the review note about `MINIMAX_API_KEY` metadata consistency.

## Best Possible Solution

Keep the PR open and have maintainers review or adapt it onto current main: add `MINIMAX_OAUTH_TOKEN` to the MiniMax web_search runtime credential resolution, error text, provider metadata/public contract metadata, and regression tests, while resolving the existing `MINIMAX_API_KEY` metadata inconsistency noted in review. Once that lands, linked bug #65768 can close through the normal PR closing flow.
