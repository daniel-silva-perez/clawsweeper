## Summary

Keep this PR open. Current main still does not implement the proposed doctor check for volatile Linux state-directory filesystems: doctor only wires macOS cloud-sync and Linux SD/eMMC storage warnings, and there is no volatile-filesystem detector, formatter, test file, or docs entry on main. The PR discussion is useful and the author addressed the two review concerns by removing overlay from the volatile set and fixing the mountInfo-unavailable test, so the remaining path is maintainer review/rebase rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to refresh it against current main, preserve the later scoped behavior of detecting true volatile filesystems such as tmpfs and ramfs without overlay false positives, add focused tests for mount matching and unavailable mountinfo, update the public doctor docs if the behavior lands, and run the changed doctor checks in the normal gate.
