## Summary

Keep open. Current main already covers the native `streamTo` tolerance, but it still lacks the PR's bare `agentId` send behavior, and the submitted branch introduces blocking cross-session routing and channel-specific prompt leakage issues. Existing review comments in the discussion also point at the same risky surfaces, so this should be revised rather than closed as implemented.

## What This Changes

The PR adds bare `agentId` resolution for `sessions_send`, changes native `sessions_spawn` handling of ACP-only `streamTo`, adds Feishu group orchestration prompt guidance, and updates related agent tool tests.

## Best Possible Solution

Revise or replace the branch so bare `agentId` targeting is explicit and preserves label-miss failure semantics, keeps A2A and visibility checks intact, leaves tool metadata channel-neutral, and gates Feishu-specific orchestration guidance only on Feishu runtime-channel prompt context.

## Reproduction Assessment

Yes. The misrouting risk is reproducible by inspection with a focused unit scenario: call `sessions_send` with a missing `label` plus an `agentId`, and the PR path can resolve the newest visible session for that agent instead of returning the label lookup error.

## Solution Assessment

No. Ignoring native `streamTo` is already solved on current main, and the remaining branch should be narrowed to explicit `agentId` sends plus channel-gated Feishu prompt guidance rather than changing failed-label targeting semantics.
