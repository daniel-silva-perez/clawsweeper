## Summary

Keep open. Current main does not implement the central bare-agentId `sessions_send` behavior, and the PR branch still has blocking routing regressions called out by both the provided review comments and source inspection. The `streamTo` subagent-runtime part is already handled on main, but the remaining session targeting and prompt changes need repair before this PR can merge or be replaced.

## What This Changes

This PR changes `sessions_send` to resolve an `agentId` to the most recent visible session, makes `sessions_spawn` ignore ACP-only `streamTo` for native subagents, and adds Feishu-oriented group orchestration prompt guidance and tests.

## Best Possible Solution

Keep the useful `agentId` targeting idea, but implement it as an explicit bare-agentId path only: label lookups should fail closed on label misses, A2A enforcement should preserve current main-session semantics or be deliberately documented, and Feishu-specific prompt text should be gated to Feishu runtime context. The already-shipped `streamTo` tolerance should be left as-is on main.

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Keep label misses from falling back to agentId:** `src/agents/tools/sessions-send-tool.ts:243-246`
  - body: When `sessions.resolve` misses a label, this branch falls back to `requestedAgentId` or even a valid-looking label string and sends to the newest visible session for that agent. A typoed label can deliver the message to an unrelated conversation instead of failing, which is a confidentiality regression.
  - confidence: 0.93
- **[P1] Preserve the main-session A2A bypass:** `src/agents/tools/sessions-send-tool.ts:124-125`
  - body: The old check only enforced A2A when `requesterAgentId` was present; main/orchestrator session keys can resolve to null and used to send by label without enabling `tools.agentToAgent`. This refactor treats null as a different agent and forbids those sends when A2A is disabled.
  - confidence: 0.9
- **[P2] Scope Feishu prompt text to Feishu sessions:** `src/agents/system-prompt.ts:253-255`
  - body: This summary is emitted for every non-ACP `sessions_spawn` prompt, so Discord, Telegram, and other channels see Feishu-specific orchestration advice despite the separate gated Feishu block below. Gate this summary on `runtimeChannel === "feishu"` or keep it generic.
  - confidence: 0.87
