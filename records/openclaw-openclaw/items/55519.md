## Summary

Keep open. The PR still contains useful remaining work around bare `agentId` targeting and Feishu group orchestration guidance, but this revision is not merge-ready: the label miss fallback can route a message to the newest visible agent session instead of failing, and Feishu-specific guidance is added to generic tool metadata shown outside Feishu. Current main has already shipped the `streamTo` subagent tolerance portion, so the branch also needs a rebase/narrowing.

## What This Changes

The PR adds bare `agentId`-based `sessions_send` resolution to the latest visible session, ignores ACP-only `streamTo` for native subagent spawns, and adds Feishu-specific long-running group orchestration prompt guidance and tests.

## Best Possible Solution

Keep the useful intent, but revise or replace the patch so `sessions_send` supports an explicit bare `agentId` target without converting label misses into agent-targeted sends, preserve the already-shipped native-subagent `streamTo` tolerance from main, and keep Feishu guidance only in channel-gated prompt text/tests rather than generic tool descriptions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Require explicit agent targeting after label misses:** `src/agents/tools/sessions-send-tool.ts:242-253`
  - body: This fallback turns a failed label lookup into an agent lookup whenever the label itself is a valid agent id, or when `label` and `agentId` were provided together. A typoed or missing label should fail; otherwise `sessions_send` can deliver the message to the newest visible session for that agent instead of the requested conversation.
  - confidence: 0.92
- **[P2] Gate Feishu-specific summary text by channel:** `src/agents/system-prompt.ts:253-255`
  - body: This non-ACP `sessions_spawn` summary is shown for every runtime channel, so Discord, Telegram, and other non-Feishu sessions would see Feishu-specific orchestration advice. Keep this summary generic and put the Feishu wording only in the existing `runtimeChannel === "feishu"` block.
  - confidence: 0.91
- **[P2] Remove Feishu text from the generic spawn description:** `src/agents/tools/sessions-spawn-tool.ts:83-84`
  - body: `createSessionsSpawnTool` has no runtime-channel input, so this description is attached to the tool for all deployments. Adding ordinary-Feishu-group guidance here pollutes non-Feishu tool metadata; leave the tool description channel-neutral and rely on channel-gated prompt guidance.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The PR introduces a concrete message misrouting/data exposure risk in `sessions_send` label fallback handling.

Concerns:

- **[high] Label fallback can send to the wrong visible session:** `src/agents/tools/sessions-send-tool.ts:242`
  - body: When a requested label cannot be resolved, the PR can fall back to the newest session for an agent id derived from the label or provided alongside it. That can deliver a message to a different conversation than the caller targeted, which is a cross-session disclosure risk.
  - confidence: 0.9
