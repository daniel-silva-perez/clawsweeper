## Summary

Keep open. Current main already contains the `sessions_spawn` ACP-only-field tolerance, but it still does not implement the bare-`agentId` `sessions_send` targeting or the Feishu group orchestration guidance, so the PR is not obsolete. The current patch should not merge as-is because its label-miss fallback can send to the newest visible session for an agent instead of failing the requested label lookup, and it also leaks Feishu-specific guidance into generic non-Feishu prompt/tool metadata.

## What This Changes

The PR adds `agentId`-based `sessions_send` targeting, changes `sessions_spawn` to ignore ACP-only `streamTo` for native subagent runtime, and adds Feishu group orchestration prompt guidance with tests.

## Best Possible Solution

Keep the useful intent, but revise or replace the patch so bare `agentId` targeting is explicit-only, label misses fail instead of falling through to newest-agent-session lookup, the already-landed `streamTo` tolerance is preserved, and Feishu guidance stays behind `runtimeChannel === "feishu"` prompt logic rather than generic tool metadata.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Require explicit agent targeting after label misses:** `src/agents/tools/sessions-send-tool.ts:242-253`
  - body: This fallback turns a failed label lookup into an agent lookup whenever the label itself is a valid agent id, or when `label` and `agentId` were provided together. A typoed or missing label should fail; otherwise `sessions_send` can deliver the message to the newest visible session for that agent instead of the requested conversation.
  - confidence: 0.92
- **[P2] Gate Feishu-specific summary text by channel:** `src/agents/system-prompt.ts:253-255`
  - body: This non-ACP `sessions_spawn` summary is shown for every runtime channel, so Discord, Telegram, and other non-Feishu sessions would see Feishu-specific orchestration advice. Keep this summary generic and put the Feishu wording only in channel-gated prompt guidance.
  - confidence: 0.91
- **[P2] Keep sessions_spawn description channel-neutral:** `src/agents/tools/sessions-spawn-tool.ts:83-84`
  - body: `createSessionsSpawnTool` has no runtime-channel input, so this description is attached to the tool for all deployments. Adding ordinary-Feishu-group guidance here pollutes non-Feishu tool metadata; rely on channel-gated prompt guidance instead.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The PR introduces a concrete cross-session message misrouting and data exposure concern in `sessions_send`.

Concerns:

- **[high] Label fallback can send to the wrong visible session:** `src/agents/tools/sessions-send-tool.ts:242`
  - body: When a requested label cannot be resolved, the PR can fall back to the newest session for an agent id derived from the label or supplied alongside it. That can deliver a message to a different conversation than the caller targeted, which is a cross-session disclosure risk.
  - confidence: 0.9
