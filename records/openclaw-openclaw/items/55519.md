## Summary

Keep open. Current main only implements one part of this PR: ACP-only `streamTo`/`resumeSessionId` are ignored for native subagent spawns. The central `sessions_send` behavior from the PR is still missing because bare `agentId` is not a valid target on main, and the Feishu-specific orchestration prompt guidance is also absent. The PR also has unresolved review findings around possible label-miss misrouting and A2A gating regressions, so it should not be closed as implemented or merged as-is.

## Best Possible Solution

Keep this PR open for maintainer review or revision. A safe path would rebase onto current main, drop the already-shipped `streamTo` fallback changes unless still needed, implement explicit bare-`agentId` targeting without label-miss fallback, preserve existing A2A semantics for main/orchestrator sessions, gate Feishu prompt guidance on `runtimeInfo.channel === "feishu"`, and add targeted tests for each behavior.

## Risks / Open Questions

- Merging the PR as submitted could misroute a label-targeted `sessions_send` after a label typo or missing label by falling back to the newest visible session for the requested agent.
- The PR discussion reports an A2A policy regression for main/orchestrator sessions when `agentToAgent.enabled` is false; that needs a focused regression test before merge.
- The PR also appears to put Feishu-specific prompt text in generic non-ACP tool guidance, which would leak channel-specific advice into other deployments.
