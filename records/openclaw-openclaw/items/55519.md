## Summary

Keep open. Current main still lacks the PR's central bare-agentId `sessions_send` capability, but the branch has blocking correctness and security problems around label-miss fallback, and its Feishu guidance leaks into generic prompt/tool metadata. The `streamTo` tolerance part has already been implemented independently on main, so the remaining useful work should be revised rather than merged as-is.

## What This Changes

The PR adds `sessions_send` agentId-based target resolution, changes native `sessions_spawn` to ignore ACP-only `streamTo`, and adds Feishu group orchestration prompt guidance with tests.

## Best Possible Solution

Revise or replace the branch so `sessions_send` supports only explicit bare `agentId` targeting, preserves failed-label lookup semantics, keeps A2A and visibility checks intact, leaves generic tool metadata channel-neutral, and puts Feishu-specific guidance only behind Feishu runtime-channel prompt logic while retaining the already-main `streamTo` tolerance.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Require explicit agent targeting after label misses:** `src/agents/tools/sessions-send-tool.ts:242-253`
  - body: When label resolution fails, this fallback can use either the provided `agentId` or a label that happens to be a valid agent id and send to that agent's newest visible session. A typoed or missing label should fail instead of delivering the message to a different conversation.
  - confidence: 0.92
- **[P2] Gate Feishu-specific summary text by channel:** `src/agents/system-prompt.ts:253-255`
  - body: This non-ACP `sessions_spawn` summary is emitted for every runtime channel, so Discord, Telegram, and other sessions would see Feishu-specific orchestration advice. Keep this summary generic and put Feishu wording only in channel-gated prompt text.
  - confidence: 0.91
- **[P2] Remove Feishu text from generic spawn metadata:** `src/agents/tools/sessions-spawn-tool.ts:83-84`
  - body: `createSessionsSpawnTool` has no runtime-channel input, so this description applies globally. Feishu group guidance in tool metadata pollutes non-Feishu deployments; keep the tool description channel-neutral.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The PR introduces a concrete cross-session misrouting/data exposure risk in `sessions_send` label fallback handling.

Concerns:

- **[high] Label fallback can send to the wrong visible session:** `src/agents/tools/sessions-send-tool.ts:242`
  - body: When a requested label cannot be resolved, the PR may fall back to the newest visible session for an agent id derived from the label or supplied alongside it. That can deliver message content to a different conversation than the caller targeted.
  - confidence: 0.9
