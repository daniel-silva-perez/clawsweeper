## Summary

Keep open. Current main already has the underlying attachment path and Control UI image/file upload via file picker, drag/drop, and pasted images, but it does not implement the requested screenshot/scissors button, screen-region capture flow, or browser/native screen capture API in the chat composer. The related items in the supplied context are adjacent Control UI button/attachment issues, not a canonical tracker for this screenshot capture feature.

## Best Possible Solution

Implement this as a focused Control UI chat composer enhancement: add a screenshot/scissors button beside the existing paperclip, capture a screen/window/region through an approved browser or native bridge path, convert the captured image to the existing ChatAttachment payload shape, and reuse the current preview/remove/chat.send.attachments flow. Add focused UI tests for button availability, captured-image preview/removal, and attachment serialization; update the public Control UI docs once the workflow exists.
