## Summary

Keep open. Current main has the Control UI attachment pipeline for file picker, drag/drop, pasted images, previews, removal, and chat.send attachment serialization, but it still does not provide the requested screenshot/scissors button or screen-region capture flow in the chat composer. The related items in the supplied context are adjacent Control UI button/file issues, not a canonical tracker for this specific screenshot capture feature.

## Best Possible Solution

Keep this open and implement it as a focused Control UI chat composer enhancement: add a screenshot/scissors button beside the existing paperclip, capture a screen/window/region through an approved browser or native bridge path, convert the captured image into the existing ChatAttachment payload store, reuse the current preview/remove/chat.send.attachments flow, add focused UI/controller tests, and update the public Control UI docs when the workflow exists.

## Risks / Open Questions

- Browser screen capture APIs and native screenshot bridges have different permission, platform, and region-selection behavior; the implementation should choose an explicit fallback path instead of assuming one API covers all targets.
- Screen capture can expose sensitive pixels, so the UI should keep preview/removal clear and reuse existing attachment size/type validation before sending.
- The existing upload/paste attachment path is already good groundwork; adding a separate screenshot payload pipeline would duplicate preview, removal, and chat.send serialization behavior.
