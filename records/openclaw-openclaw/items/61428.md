## Summary

Keep this PR open. Current main still does not implement mapping-level `sessionTarget` for webhook hook mappings: config/schema reject or omit the field, mapping resolution does not carry it, and gateway hook agent dispatch still hardcodes isolated cron runs. The PR is not obsolete, but it needs maintainer review and likely fixes for the policy/security comments before merge.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate. The best path is to update the PR so mapping-level `sessionTarget` is accepted by config/schema/generated metadata, threaded through mapping transforms and dispatch, included in replay dedupe, and validated against session policy using the effective target session key before execution. Add focused tests for default isolated behavior, `sessionTarget: "main"` with a non-default `agentId`, prefix-policy handling, and config schema acceptance.

## Risks / Open Questions

- Merging the PR as shown could misapply or bypass `hooks.allowedSessionKeyPrefixes` because `sessionTarget: "main"` changes the effective execution key after request-handler validation.
- The PR adds a public config key, so generated config metadata/help/labels, docs, and focused tests should be updated with the runtime/schema changes.
- Replay dedupe scope should include `sessionTarget` if otherwise identical mapped hook requests can dispatch to different session targets.
