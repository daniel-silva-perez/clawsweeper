## Summary

Keep open. The user problem is still real on current main: per-agent model values override the default, but the current config-set path only emits generic update messages. The PR is useful, but it is based on an older inline config-set handler and needs to be adapted to the current `runConfigSet` and batch-operation flow before it can merge.

## What This Changes

The PR adds a CLI warning and regression tests for `openclaw config set` when changing `agents.defaults.model` while agents have explicit `agents.list[].model` overrides.

## Best Possible Solution

Implement the warning in current `runConfigSet` after a successful non-dry-run `replaceConfigFile`, match parsed `setPath` segments for `agents.defaults.model` and `agents.defaults.model.primary` across single and batch operations, scan the resulting config for string and object agent model overrides, and add focused tests for single-path, bracket notation, batch, dry-run, unrelated paths, and no-overrides cases.
