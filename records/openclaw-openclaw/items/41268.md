## Summary

Current main still does not implement this PR's intended CLI warning. `runConfigSet` writes the updated config and only prints the generic restart hint; there is no post-write scan for `agents.list[].model` overrides. The underlying user confusion remains real because per-agent model config still takes precedence over `agents.defaults.model`, and both tests/docs preserve that behavior. The PR appears narrow and security-low-risk, but it needs rebase/adaptation to the current `runConfigSet` and batch-mode plumbing before it can land.

## Best Possible Solution

Keep this PR open and adapt it onto current main. The implementation should hook into `runConfigSet` after a successful non-dry-run write, detect any operation whose parsed path is `agents.defaults.model` or `agents.defaults.model.primary` including bracket notation and batch entries, scan the resulting config for explicit `agents.list[].model` values in both string and object forms, and add focused CLI tests for object-style overrides, string-style overrides, bracket notation, no overrides, unrelated paths, and batch mode. No core model-resolution or schema change is needed.
