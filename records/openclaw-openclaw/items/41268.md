## Summary

Keep open. The requested warning still describes a real CLI confusion point, but the PR is stale against current main: `config set` now writes through `runConfigSet` and `runConfigOperations`, while this patch adds the warning to the older inline set action. Current main also has no equivalent per-agent override warning, so the right outcome is to port the idea into the active shared runner rather than close or merge as-is.

## What This Changes

The PR adds a CLI warning after `openclaw config set agents.defaults.model*` when configured agents have explicit per-agent model overrides, plus regression tests for object-style models, string-style models, unrelated paths, and bracket notation.

## Best Possible Solution

Keep the warning behavior, but implement it in `runConfigOperations` after a successful non-dry-run `set` write, using parsed `ConfigSetOperation` paths and the final config snapshot so value mode, bracket notation, and batch mode behave consistently without changing model resolution, schema, or gateway restart semantics.

## Reproduction Assessment

Yes. Source inspection on current main is a high-confidence reproduction path: `config set` delegates to `runConfigSet` and `runConfigOperations`, which write and print only the generic restart message while explicit agent models still override defaults.

## Solution Assessment

No. The proposed user-facing warning is a good narrow fix, but this patch is no longer the best implementation because it targets an inactive inline action instead of the current shared config operation runner.
