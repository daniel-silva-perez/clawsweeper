## Summary

Keep open. The warning addresses a real CLI confusion, but this PR is based on an older config-set implementation and places the warning after an inline `writeConfigFile` path that current `main` no longer uses for `openclaw config set`. Current `main` routes set operations through `runConfigSet` and `runConfigOperations`, where the warning is still absent.

## What This Changes

This PR adds a CLI warning after setting `agents.defaults.model` paths when configured agents have explicit model overrides, plus focused config CLI tests for object-style, string-style, bracket-notation, and no-warning cases.

## Best Possible Solution

Keep the warning behavior, but implement it once in the shared non-dry-run set operation path after a successful `replaceConfigFile`, using parsed operation paths and the final config snapshot so value, bracket notation, builder, and batch flows stay consistent without changing model resolution, schema, or gateway restart semantics.

## Reproduction Assessment

Yes. Source inspection reproduces the gap: current `config set` flows through `runConfigSet` into `runConfigOperations`, writes through `replaceConfigFile`, and emits only the generic restart message while per-agent models still take precedence over defaults.

## Solution Assessment

No. The requested warning is the right narrow solution, but this PR is not currently the best implementation because the warning is attached to an inactive older write path instead of the active shared set runner.
