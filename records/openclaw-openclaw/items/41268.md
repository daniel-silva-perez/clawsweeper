## Summary

Keep open. Current main still lacks the PR’s intended `openclaw config set agents.defaults.model.primary ...` warning, while per-agent `agents.list[].model` overrides still take precedence over defaults. The PR remains a focused CLI/DX fix, and the discussion shows earlier bot review feedback was addressed, but it needs to be rebased/adapted to the current `runConfigSet` operations and batch-mode flow before it can land.

## Best Possible Solution

Keep this PR open and rebase/adapt it onto current main. Add a small helper near `runConfigSet` that detects successful non-dry-run operations whose parsed `setPath` is `agents.defaults.model` or `agents.defaults.model.primary`, including bracket notation and batch mode, then scans the resulting config for explicit `agents.list[].model` values in both string and object forms and prints a focused warning. Add targeted tests for object-style overrides, string-style overrides, bracket notation, no overrides, unrelated paths, dry-run behavior, and batch mode. No schema, runtime model-resolution, or gateway restart change is needed for this PR.
