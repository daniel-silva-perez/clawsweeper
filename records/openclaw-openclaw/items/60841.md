## Summary

Keep open. Current main still builds the core tool list through normal config policy before applying `toolsAllow`, so a restrictive `tools.allow: ["group:openclaw"]` can still remove `exec` before the embedded cron run post-filter sees it. There is also an open same-author implementation candidate, #60842, with closing syntax for this issue, so repository policy says the issue should remain open until that PR is reviewed/landed or closed.

## Best Possible Solution

Keep this issue open and review #60842, or replace it with a maintainer patch that forwards runtime `toolsAllow` into core tool construction/policy merging as a narrow force-allow for known core tools. The fix should preserve explicit deny and runtime restriction precedence, then add focused regression coverage in `src/agents/pi-tools-agent-config.test.ts` and `src/agents/pi-embedded-runner/run/attempt*.test.ts` for `tools.allow: ["group:openclaw"]` with `toolsAllow: ["exec", "read", "write"]`.
