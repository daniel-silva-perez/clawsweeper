## Summary

Keep this issue open. Current main still applies `toolsAllow` only after `createOpenClawCodingTools()` has already run the normal config allowlist pipeline, so a restrictive `tools.allow: ["group:openclaw"]` can still remove `exec`/`read`/`write` before the embedded cron run gets to post-filter. There is also an open same-author PR, #60842, with closing syntax for this issue, so the right maintainer path is to review/land or close that implementation candidate rather than closing this issue early.

## Best Possible Solution

Keep this issue open and review #60842, or replace it with a maintainer patch that forwards runtime `toolsAllow` into the core tool construction/policy merge as a narrow force-allow for known core tools. The implementation should preserve deny precedence and sandbox/subagent/provider/runtime restrictions, then add focused regression tests in `src/agents/pi-tools-agent-config.test.ts` and `src/agents/pi-embedded-runner/run/attempt*.test.ts` for `tools.allow: ["group:openclaw"]` plus `toolsAllow: ["exec", "read", "write"]`.
