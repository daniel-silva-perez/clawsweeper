## Summary

Keep this PR open. Current main still has the Overview input behavior the PR targets, so the PR is not obsolete or implemented. The linked bug remains open, and this PR is a focused implementation candidate, but its exact one-line fix needs maintainer review because preserving a bearer token across arbitrary gateway URL edits can send a selected-gateway token to a different endpoint.

## Best Possible Solution

Review or rework the Control UI fix. A safer path is to stop losing a token during harmless editing while preserving the per-gateway credential boundary: use a draft URL field or confirmation before applying a different endpoint, keep tokens only for the same normalized gateway, and carry a token to a new gateway only when it was explicitly supplied or confirmed for that gateway. Add browser regression coverage for typed-token editing, same-gateway edits, and different-gateway token scoping.
