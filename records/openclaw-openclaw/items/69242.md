## Summary

Current main has partial exec failure classification and the Linux OOM victim-selection work from #70419, but the reported user problem is not fully resolved. A foreground SIGKILL still collapses to generic signal text, and the tool result does not consistently expose the signal/termination metadata needed to distinguish timeout, OpenClaw cancellation, OS/container kill, or another supervisor cause. #70419 explicitly scoped this issue as separate SIGKILL observability work, and #72240 remains open for the same diagnostic class on macOS.

## Best Possible Solution

Keep this issue open and implement a narrow exec/supervisor observability fix: consistently carry `exitSignal`, `exitReason`/termination reason, `failureKind`, `timedOut`, duration, and captured output tail into foreground exec results and background `process` poll/log/list results. The user-facing text should distinguish OpenClaw timeouts/manual cancellation from externally signaled SIGKILLs, and should avoid claiming OOM unless OpenClaw has concrete evidence. Leave broader resource limits or broad-search guardrails for a follow-up after better telemetry or reproduction data exists.
