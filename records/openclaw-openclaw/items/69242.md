## Summary

Current main still leaves the Linux SIGKILL observability problem open. The supervisor and exec runtime preserve signal, timeout, and failure-kind data internally, and current process list/log paths now expose some exitSignal fields, but foreground exec results and process poll details still omit key structured cause metadata needed to distinguish timeout, manual cancellation, external SIGKILL, and other failures.

## Best Possible Solution

Keep this open and land a narrow generic exec/process observability fix: surface exitSignal, failureKind, exitReason or terminationReason, timedOut/noOutputTimedOut, duration/runtime, and captured tail consistently in foreground exec results and background process list/poll/log results. Reuse the existing supervisor/runtime fields, cover related #72240 where the same metadata applies, and leave resource limits or broad find/grep guardrails for separate product work.
