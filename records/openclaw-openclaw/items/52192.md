## Summary

Keep open. Current main still has the Feishu sendMedia duplicate-text path: a non-empty text value is sent before mediaUrl is checked, then the no-media fallback sends the same text again. The PR is narrow and security-benign, but it should get maintainer review and preferably a focused regression test before landing.

## What This Changes

The PR changes the Feishu outbound sendMedia handler so the pre-media companion text send only runs when mediaUrl is present.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land the minimal mediaUrl guard in extensions/feishu/src/outbound.ts and add a focused regression test in extensions/feishu/src/outbound.test.ts for sendMedia with text but no mediaUrl, leaving unrelated Feishu upload-failure behavior for a separate follow-up if needed.
