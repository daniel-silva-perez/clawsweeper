## Summary

Keep open. Current main still has the Feishu sendMedia duplicate-text path: non-empty text is sent before mediaUrl is checked, and the no-media fallback sends the same text again. The PR is narrow and security-benign, but should get maintainer review and preferably a focused regression test before landing.

## What This Changes

The PR changes the Feishu outbound sendMedia handler so the pre-media companion text send only runs when mediaUrl is present, with a nearby comment update.

## Best Possible Solution

Land the minimal mediaUrl guard in the Feishu outbound sendMedia handler and add a focused regression test for text plus no mediaUrl, while leaving unrelated Feishu upload-failure behavior for a separate follow-up if needed.
