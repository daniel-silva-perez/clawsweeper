## What This Changes

The PR changes the Feishu outbound `sendMedia` handler so companion text is sent before media only when `mediaUrl` is present, and updates the adjacent comment.

## Best Possible Solution

Land the minimal `mediaUrl` guard in the Feishu outbound `sendMedia` handler, preferably with a focused regression test proving text plus no `mediaUrl` sends exactly one Feishu text message; keep broader media-upload failure fallback behavior as separate follow-up work if maintainers want to change it.

## Reproduction Assessment

Yes. A direct `feishuOutbound.sendMedia` call with non-empty `text` and no `mediaUrl` reaches the current companion text send and then the no-media fallback, which is visible from source inspection without needing a live Feishu account.

## Solution Assessment

Yes. The proposed guard is the narrowest maintainable fix for this duplicate path because it preserves text-plus-media and media-only behavior while letting the existing text-only fallback handle the no-media case once.
