## Summary

Keep this issue open. Current main has bundled Qwen Coding Plan support and a Qwen-specific `enable_thinking` stream wrapper, but the requested `qwen3.5-plus`, `glm-5`, and `kimi-k2.5` Coding Plan path is still not enabled because those catalog entries remain `reasoning: false`, and the wrapper only runs for reasoning-enabled models.

## Best Possible Solution

Keep this issue open and implement the remaining work inside the bundled `qwen` provider: add Qwen-owned thinking metadata/profile coverage for the Coding Plan models that support it, map OpenClaw thinking controls to DashScope `enable_thinking` for those models, only add `thinking_budget` where Alibaba documents it, update Qwen docs, and add regression tests proving catalog metadata, payload construction, and UI/gateway thinking policy behavior for `qwen3.5-plus`, `glm-5`, and `kimi-k2.5`.
