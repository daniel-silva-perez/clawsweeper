## Summary

Current main still does not implement the PR’s Windows skill-path normalization. The prompt path flow compacts home-relative skill paths to `~/...` but leaves the suffix unchanged, then emits that value directly into `<location>`. The PR remains a focused implementation candidate for the closed linked bug, though it may need rebase/adjustment around the newer `compactHomePath()` helper on main.

## Best Possible Solution

Keep this PR open. Review/rebase it against current main and land either this change or an equivalent small fix in `src/agents/skills/workspace.ts` that normalizes prompt-facing Windows skill paths after home compaction, with a regression test in `src/agents/skills.compact-skill-paths.test.ts`.
