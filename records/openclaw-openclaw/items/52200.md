## Summary

Current main still does not implement the Windows skill-path normalization this PR is trying to add. The prompt-facing skill path flow compacts home paths to `~/...` but does not normalize remaining Windows backslashes before emitting `<location>`, so the PR remains a focused implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open. Rebase or adapt it against current main and land an equivalent small fix in `src/agents/skills/workspace.ts` that normalizes prompt-facing Windows skill paths after home compaction, with the regression coverage in `src/agents/skills.compact-skill-paths.test.ts`.
