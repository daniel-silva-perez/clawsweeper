## Summary

Keep this PR open. Current main still compacts skill paths with `~/` but does not normalize remaining Windows backslashes before rendering prompt-facing `<location>` values, so the PR's intended Windows fix is not implemented on main.

## Best Possible Solution

Keep this PR open and land an equivalent small fix on current main: normalize prompt-facing Windows skill paths after home compaction, preferably at the current `compactHomePath()` or `compactSkillPaths()` boundary, and include the regression assertion that `<location>` contains no backslashes.
