## Summary

Keep this PR open. Current main still compacts home-prefixed skill paths without normalizing remaining Windows backslashes, while both full and compact skill prompt renderers emit the compacted `filePath` unchanged; the PR remains a focused fix for the linked Windows bug, though it needs adaptation to the current `compactHomePath()` helper shape.

## What This Changes

The PR normalizes Windows backslashes in prompt-facing compacted skill paths and adds a regression assertion that rendered skill `<location>` values do not contain backslashes.

## Best Possible Solution

Normalize prompt-facing Windows skill paths once at the skills path compaction boundary so both full and compact available-skills prompt formats receive slash-consistent locations, while preserving existing home-prefix and realpath-prefix behavior on current main.

## Reproduction Assessment

Yes. The code path is clear without a live Windows run: on Windows, `path.sep` is `\`, `compactHomePath()` slices after the home prefix and prepends `~/`, and the renderers emit the resulting `filePath` unchanged into `<location>`.

## Solution Assessment

Yes in direction, with one adaptation needed. The narrowest maintainable fix is to normalize after compaction on current main, preferably inside `compactHomePath()` or immediately after `compactSkillPaths()` maps paths, plus a regression test for rendered `<location>` values.
