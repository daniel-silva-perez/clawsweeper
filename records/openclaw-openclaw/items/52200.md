## Summary

Keep this PR open. Current main still compacts home-prefixed skill paths to `~/` without normalizing remaining Windows backslashes, and the prompt renderers emit those compacted paths directly. The PR is a focused implementation candidate for the linked Windows bug #52022, with the bot-requested test assertion fix already addressed and no security-sensitive file changes in the provided diff.

## Best Possible Solution

Keep this PR open and land an equivalent small fix on current main: normalize prompt-facing Windows skill paths after home compaction, preferably at the current `compactHomePath()` or `compactSkillPaths()` boundary, and include a regression assertion that rendered `<location>` values contain no backslashes on Windows-style paths.
