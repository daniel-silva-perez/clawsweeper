## Summary

Keep this issue open. Current main still routes WebChat image attachments through Gateway media handling that can reach in-process sharp/libvips image operations on Linux/Node, and I did not find source, tests, docs, changelog, or related-report evidence that the reported libvips SIGILL crash has been fixed or made obsolete. A prior ClawSweeper comment also kept it open after tracing the same path, and the newer main changes still preserve a native image-processing route rather than isolating it from the gateway process.

## Best Possible Solution

Keep this issue open. A maintainer should reproduce or collect a core/backtrace on current main or the latest shipped release in a Linux npm-global Node 22 setup, identify whether sharp's bundled libvips, system libvips selection, CPU feature dispatch, or OpenClaw's image optimization path is responsible, then ship a known-safe fallback or isolation strategy plus a WebChat image-flow regression smoke.
