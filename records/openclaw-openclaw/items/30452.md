## Summary

Keep this issue open. Current main now implements the top-level `agents.defaults.compaction.model` override and the requested `agents.defaults.compaction.memoryFlush.model` override, but it still does not implement `memoryFlush.resetAfterFlush` or any flush-then-reset policy. The issue discussion already called out the model-selection work as partial, and the remaining reset mode is a real core session-lifecycle feature rather than a stale or duplicate request.

## Best Possible Solution

Keep the issue open, but narrow the remaining scope to `agents.defaults.compaction.memoryFlush.resetAfterFlush`. The model override pieces are now implemented; the best next product/code path is a maintainer-reviewed design for resetting only after a successful memory flush, using existing session reset/rotation machinery while preserving queue/session metadata and adding focused regression coverage for transcript handling, active runs, subagents/ACP, and channel behavior.
