## Summary

Current main has implemented the model-override parts of the request, but the remaining flush-then-reset mode is still absent. The issue should stay open and be narrowed to `agents.defaults.compaction.memoryFlush.resetAfterFlush` or an equivalent maintainer-approved reset policy.

## Best Possible Solution

Keep this issue open, but narrow the remaining scope to a maintainer-reviewed `memoryFlush.resetAfterFlush` design. The best path is to add an explicit config/schema/docs surface, only reset after a verified successful memory flush, reuse existing session reset or successor-transcript machinery, skip redundant compaction when configured, and cover queue/session metadata, transcript replay, active runs, subagents/ACP, and channel behavior with focused tests.
