## Summary

Current main still does not recover an active Control UI/WebChat run across a WebSocket reconnect. The reconnect hello path clears `chatRunId` and stream state, the Gateway hello snapshot still lacks an active-run recovery contract, and open PR #41067 remains the relevant implementation candidate rather than a merged fix.

## Best Possible Solution

Keep this issue open as the canonical Control UI/WebChat active-run reconnect recovery report. Review and refresh PR #41067 or land an equivalent focused fix that adds an additive active chat-run recovery contract in the Gateway hello/snapshot or a dedicated reconnect RPC, preserves/reconciles `chatRunId` and busy/stream state across reconnect, avoids clearing active state until the run is known terminal, and adds regression coverage for WebSocket close/reconnect during an active `chat.send`. Keep #45952 separate for outbound message queue/ACK replay.
