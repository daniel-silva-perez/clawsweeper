## Summary

Keep open. Current main still lacks the requested actual-runtime-model transport surface on the Gateway OpenAI-compatible HTTP endpoints: the handlers pass the request-side agent target/backend override into the run and echo the request model in response bodies/SSE chunks, while the actual selected model callback remains below the HTTP layer. The prior maintainer review also kept this open because streaming headers flush before late fallback selection can be represented as a normal response header.

## What This Changes

Request adding an `X-Actual-Model` HTTP response header that exposes the runtime model that actually processed each Gateway HTTP agent response.

## Best Possible Solution

Define and document a Gateway actual-model metadata contract for non-streaming and streaming OpenAI-compatible HTTP responses, then plumb selected provider/model from runtime selection into that contract with focused handler tests.
