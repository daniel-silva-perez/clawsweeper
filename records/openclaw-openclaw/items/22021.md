## Summary

Keep open. Current main still lacks the requested actual-runtime-model transport surface on the Gateway OpenAI-compatible HTTP endpoints; the handlers echo the request-side agent target model while the selected-model callback remains below the HTTP layer.

## What This Changes

The issue requests an `X-Actual-Model` HTTP response header, or equivalent transport-layer surface, showing the runtime model that actually handled each Gateway request.

## Best Possible Solution

Define and document a Gateway actual-model metadata contract across `/v1/chat/completions` and `/v1/responses`, including non-streaming and streaming behavior, then wire selected provider/model from `onModelSelected` or agent metadata with focused tests.
