## Summary

Keep this PR open. Current main still does not implement the proposed exported Windows argument validator, null-byte/control-character checks, or PowerShell-pattern guard. The PR also is not safe to merge as written: its command-execution diff rejects normal Windows path separators, slash switches, equals-style flags, and would echo full argument values in validation errors.

## Best Possible Solution

Keep this PR open for maintainer review or replace it with a narrower Windows command-hardening PR. The fix should stay scoped to real cmd.exe wrapper hazards, keep path separators, scoped package specs, slash switches, and equals flags working, avoid applying PowerShell policy to non-PowerShell execution, redact argument values in validation errors, and add focused Windows regression tests for .cmd/.bat paths, npm fallback, equals flags, scoped package specs, and null/control-character handling.

## Risks / Open Questions
