## Summary

Keep this PR open. Current main still does not implement the proposed exported Windows argument validator, null-byte/control-character checks, or PowerShell-pattern guard. The PR also should not merge as-is: its diff broadens the cmd.exe blacklist to reject normal Windows path separators, slash switches, equals flags, and echoes full argument values in errors on a command-execution surface.

## Best Possible Solution

Keep the PR open for rework or replace it with a narrower canonical Windows command-hardening PR. The fix should stay scoped to cmd.exe wrapper hazards, keep path separators, scoped package specs, slash switches, and equals flags working, avoid applying PowerShell policy to non-PowerShell execution, redact argument values in validation errors, and add focused Windows regression tests for .cmd/.bat paths, npm fallback, equals flags, scoped package specs, and null/control-character handling.
