## Summary

Keep open. Current main still returns raw or 600-character-truncated text from the final `formatAssistantErrorText` fallback, and that formatter output still reaches embedded reply and lifecycle error surfaces. This PR remains a relevant implementation candidate, but it needs a small security-focused refresh before merge.

## Best Possible Solution

Keep this PR open and ask for a focused refresh against current main: preserve the generic user-safe catch-all for truly unrecognized assistant errors, keep explicit actionable branches before it, redact raw error text before truncating any operator preview, pass the preview through `sanitizeForLog`, and retain regression tests for orphaned call-id style errors and long unrecognized strings tied to #11038 and #16948.
