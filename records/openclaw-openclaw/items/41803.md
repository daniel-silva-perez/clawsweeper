## Summary

Current main still returns raw or truncated unrecognized assistant error strings from `formatAssistantErrorText`, and that formatter still feeds user-visible reply/lifecycle paths. This PR remains relevant, but should be refreshed before merge because the PR logging preview still needs the security hardening raised in review.

## Best Possible Solution

Refresh this PR against current main: preserve known actionable auth, model, session, image, billing, rate-limit, transport, replay, and schema branches; make only the truly unrecognized catch-all user-safe; redact the full raw error before truncating any operator preview; pass that preview through `sanitizeForLog`; and keep regression tests for orphaned call-id errors, long unknown strings, and preserved actionable branches.
