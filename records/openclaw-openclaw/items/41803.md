## Summary

Keep this PR open. Current main still returns raw or truncated text from the final `formatAssistantErrorText` fallback, and that formatter output still feeds user-facing embedded reply and lifecycle error surfaces. The PR remains relevant, but it should be refreshed before merge because its proposed warning preview still needs stronger log-safe handling.

## Best Possible Solution

Keep this PR open and refresh it against current main: keep the explicit actionable branches for known auth, model, session, image, billing, rate-limit, transport, and replay errors; make only the truly unrecognized catch-all user-safe; redact before truncating any operator preview; pass the preview through `sanitizeForLog`; and retain regression tests for orphaned call-id errors, long unknown strings, and the known actionable branches referenced in review.
