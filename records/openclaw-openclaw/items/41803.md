## Summary

Keep open. Current main still returns raw or truncated unrecognized assistant error text from the final `formatAssistantErrorText` fallback, and that formatter output still feeds user-facing reply and lifecycle error surfaces. The PR remains relevant, but should be refreshed before merge because the proposed log preview still redacts after truncation and strips only CR/LF instead of using the repo's log sanitizer.

## Best Possible Solution

Keep this PR open and refresh it against current main: preserve the generic user-safe catch-all for truly unrecognized assistant errors, keep explicit actionable branches for known auth/model/session/image failures before it, redact raw text before truncating any operator preview, pass the preview through `sanitizeForLog`, and retain regression tests for orphaned call-id style errors and long unrecognized strings tied to #11038 and #16948.
