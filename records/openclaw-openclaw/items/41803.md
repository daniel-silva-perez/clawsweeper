## Summary

Keep this PR open. Current main still returns raw or truncated text from the final `formatAssistantErrorText` fallback, and that output still feeds user-facing reply and lifecycle surfaces. The PR remains relevant, but should be refreshed before merge to handle the warning-preview security issues raised in review.

## Best Possible Solution

Refresh this PR against current main: keep the known actionable branches for auth, model, session, image, billing, rate-limit, transport, replay, and schema errors; make only the truly unrecognized catch-all user-safe; redact the full raw error before truncating any operator preview; pass that preview through `sanitizeForLog`; and retain regression tests for orphaned call-id errors, long unknown strings, and the preserved actionable branches.
