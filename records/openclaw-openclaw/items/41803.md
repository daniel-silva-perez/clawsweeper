## Summary

Keep open. Current main still returns raw or truncated unrecognized assistant error text from `formatAssistantErrorText`, and that formatter feeds user-visible reply/lifecycle paths. The PR remains relevant, but should be refreshed before merge because its operator warning preview still needs the redaction-before-truncation and log-sanitization hardening raised in review.

## Best Possible Solution

Refresh this PR against current main: preserve all known actionable formatter branches, make only the true unrecognized catch-all return a generic user-safe message, log only a redacted-then-truncated and `sanitizeForLog`-cleaned operator preview, and add regression tests for orphaned call-id errors, long unknown strings, preserved actionable branches, and sanitized/redacted preview logging.
