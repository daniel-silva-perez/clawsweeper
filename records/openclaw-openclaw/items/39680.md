## Summary

Current main still matches the reported false-positive behavior: the sanitizer warns on any 80+ character value in the base64 character class, `sanitizeEnvVars` does not pass env-var key context into value validation, and tests still expect a warning for non-credential key `SAFE_TEXT`. The related fix PRs are closed unmerged, and no open PR found by title/body search closes this issue.

## Best Possible Solution

Keep this issue open. Tighten `src/agents/sandbox/sanitize-env-vars.ts` by making the base64 warning structurally aware and/or env-key aware, then add focused regression tests for legitimate long alphanumeric, hex-like, and JWT-segment-like values while preserving explicit secret-name blocking, null-byte blocking, max-length behavior, and suspicious credential warnings.
