## Summary

Keep open. Current main still has a plausible hydration-to-save path in the macOS General settings Tailscale section, and I did not find an implementation or regression test proving that opening General is now read-only when no user setting changed.

## Best Possible Solution

Fix this in the macOS app by separating hydration from user edits in `TailscaleIntegrationSection`: suppress saves while loading, compare the normalized desired config with the current config before writing, and add a regression test using `ConfigStore` overrides that opens/loads the section and asserts no save and no `gateway.bind`, `gateway.auth`, `meta.lastTouchedAt`, or wizard metadata mutation without a user action.
