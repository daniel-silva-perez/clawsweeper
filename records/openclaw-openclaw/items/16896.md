## Summary

Keep open: current main still lacks a user-selected Dashboard/WebChat reply workflow. The only reply UI currently present renders inbound assistant directive metadata, while message actions remain copy/expand only and the composer/send path has no selected quote state.

## What This Changes

Request adds a right-click context menu in Dashboard WebChat so users can reply to a selected message by quoting or referencing it and focusing the composer.

## Best Possible Solution

Implement the reply UX in the Control UI owner area: accessible message-level Reply action, right-click menu, selected quote state, focus management, dismissible composer preview, and quoted Markdown on send as the first compatible behavior.
