## Summary

Keep this issue open. Current main still does not implement a Dashboard/WebChat user reply workflow: there is no message context-menu handler, no visible/keyboard Reply action, no selected reply target in chat/composer state, and `chat.send` still sends only message/delivery/idempotency/attachments fields. The existing reply pill is assistant directive metadata rendering, not a user-selected quote action. Related #29732 is already closed back to this issue as the canonical tracker.

## Best Possible Solution

Implement a focused Dashboard/WebChat reply UX in the Control UI owner area: add a visible accessible message-level Reply action plus right-click context menu, store the selected quoted message in composer state, focus the textarea, render a dismissible quote preview, and send quoted Markdown for the first pass. Keep protocol-level persisted reply metadata out of scope unless maintainers explicitly choose that larger design. Keep #29732 closed as duplicate context and use this issue as the canonical tracker.
