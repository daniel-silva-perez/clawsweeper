## Summary

Current main still does not implement the requested Dashboard/WebChat reply workflow. The chat UI has no right-click/context-menu handler, no user-facing Reply action, no selected quote target in composer/controller state, and no chat.send reply/quote field. The existing reply pill only renders assistant-supplied directive metadata, and related #29732 is already closed back to this issue as the canonical tracker.

## Best Possible Solution

Implement a focused Dashboard/WebChat reply UX in the WebChat owner area: add an accessible message-level Reply action plus right-click context menu, store the selected quoted message in composer state, focus the input, render a dismissible quote preview, and send either quoted Markdown for a UI-only first pass or additive reply metadata if maintainers decide persisted references are required. Keep #29732 closed as duplicate context and use this issue as the canonical tracker.
