## Summary

Keep this issue open. Current main still lacks a Dashboard/WebChat user-selected reply workflow: there is no message context menu, no reply callback/state in the chat view, no composer quote preview, and the `chat.send` UI/protocol path still sends only message/delivery/idempotency/attachment-style fields. The existing reply pill is assistant directive metadata rendering, not a user action. The provided discussion also shows #29732 was closed back to this issue as the canonical tracker.

## What This Changes

Add a Dashboard/WebChat message reply or quote workflow with a right-click Reply option, accessible message-level Reply action, focused composer, quote preview, and quoted send behavior.

## Best Possible Solution

Implement a focused Control UI/WebChat reply UX in the UI owner area: add an accessible message-level Reply action plus right-click context menu, store the selected quoted message in composer state, focus the textarea, render a dismissible quote preview, and send quoted Markdown for the first pass. Keep protocol-level persisted reply metadata out of scope unless maintainers explicitly request it, and keep #29732 as duplicate context.
