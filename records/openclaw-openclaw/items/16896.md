## Summary

Keep open. Current main still lacks the requested Dashboard/WebChat user-selected reply workflow: no context-menu handler, no message-level Reply action, no composer-selected quote state or preview, and the existing reply pill only renders inbound assistant directive metadata. The closed related #29732 points back to this request as the canonical UI tracker, while open PR #62262 is gateway/channel reply-threading work and does not implement the Control UI UX.

## What This Changes

Add a Dashboard/Control UI WebChat right-click context-menu Reply action that quotes or references a selected message and focuses the composer.

## Best Possible Solution

Implement this in the Control UI chat owner area: add accessible message-level Reply controls plus a right-click menu, keep composer-local selected-message quote state, focus the textarea, show a dismissible quote preview, and prepend a safe Markdown blockquote on send. Leave gateway protocol/storage reply metadata and channel-native threading for a separate design if maintainers want it.
