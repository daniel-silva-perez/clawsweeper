## Summary

Keep open. Current main still does not provide the requested Dashboard/WebChat right-click or message-level reply workflow: the chat view has no context-menu handler or reply callback, rendered message actions remain copy/expand-only, the composer has no selected quote preview state, and `chat.send` still submits plain text plus attachments. The closed WebChat quote issue points back here as the canonical UI tracker, while the open reply-threading PR in the provided context is adjacent gateway/channel work and does not implement this Control UI UX.

## What This Changes

Request to add a Dashboard/WebChat message reply workflow where right-clicking a message offers Reply, quotes the selected message, and focuses the composer.

## Best Possible Solution

Implement this in the Control UI chat owner area: add accessible message-level Reply controls plus a right-click menu, track a composer-local selected message, focus the textarea, show a dismissible quote preview, and prepend a safe Markdown blockquote when sending. Keep gateway protocol, storage migrations, and channel-native threading as separate product work unless maintainers promote that broader design.
