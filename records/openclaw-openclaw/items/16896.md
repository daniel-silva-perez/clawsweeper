## Summary

Current main still lacks the requested Dashboard/WebChat right-click or message-level Reply workflow. The UI can display assistant-supplied reply pills from inline directive metadata, but there is no selected reply target, context-menu/right-click handler, composer quote state, message-level Reply action, or WebChat `chat.send` reply field. Related #29732 is already closed back to this issue, so this should remain the canonical tracker.

## Best Possible Solution

Keep this issue open as the canonical Dashboard/WebChat reply UX tracker. A focused fix should add an accessible message-level Reply action and right-click context menu, store the selected quoted message in composer state, focus the input, render the selected quote clearly, and either send plain quoted text or introduce an additive protocol metadata field if maintainers want persisted reply references.
