## Summary

Keep open. Current main still lacks the requested Dashboard/WebChat user reply workflow: no message context-menu handler, no Reply action in the chat bubble actions, no selected reply target in composer state, and the existing reply pill only renders inbound assistant directive metadata. Closed #29732 points back here as the canonical UI tracker, while open #62262 is broader gateway/channel reply threading and does not implement this Control UI feature.

## What This Changes

The issue requests a Dashboard/WebChat right-click message Reply action that quotes or references the selected message and focuses the composer.

## Best Possible Solution

Implement the first pass in the Control UI chat owner area: accessible message-level Reply controls plus a right-click menu, selected-message quote state in the composer, textarea focus, a dismissible quote preview, and safe Markdown quote prepending on send. Keep native channel threading, storage migrations, and gateway protocol reply metadata separate unless maintainers promote that broader design.
