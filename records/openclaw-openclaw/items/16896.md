## Summary

Keep open. Current main still lacks a user-selected Dashboard/WebChat reply workflow: there is no right-click context-menu handler, no Reply action in the message toolbar, no composer-side selected quote state, and the existing reply pill only displays inbound assistant reply directives. The closed related #29732 points back here as the canonical tracker, while open #62262 is a gateway/channel reply-threading proposal and does not implement this Control UI feature.

## What This Changes

The issue requests a Dashboard/WebChat message Reply action, including right-click context menu support, that quotes or references a selected message in the composer.

## Best Possible Solution

Implement this in the Control UI chat owner area: add accessible message-level Reply controls plus a right-click menu, track the selected message in composer state, focus the textarea, show a dismissible quote preview, and prepend a safe Markdown quote block when sending. Keep storage migrations, native channel threading, and gateway protocol reply metadata out of the first pass unless maintainers explicitly broaden the design.
