## Summary

Keep this issue open. Current main still lacks a Dashboard/WebChat user reply workflow: no right-click/context-menu handler, no message-level Reply action, no selected quote state in the composer, and no WebChat `chat.send` reply field. Existing reply pills only render assistant-supplied inline directive metadata. Related #29732 is closed back to this issue, so this remains the canonical tracker.

## Best Possible Solution

Keep this issue open and implement a focused Dashboard/WebChat reply UX: add an accessible message-level Reply action plus right-click context menu, store the selected quoted message in composer state, focus the input, render a clear selected quote, and send either plain quoted Markdown or an additive protocol metadata field if maintainers want persisted reply references. Update focused UI tests, docs if behavior is documented, and the user-facing changelog.
