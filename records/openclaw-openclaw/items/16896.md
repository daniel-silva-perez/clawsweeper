## Summary

Keep open. Current main still lacks the requested Dashboard/WebChat right-click or message-level reply workflow, and the existing reply UI only displays inbound assistant directive metadata rather than letting a user select a message to quote from the composer.

## What This Changes

Add a right-click context menu and reply/quote workflow to Dashboard/WebChat messages so users can reference a specific prior message from the composer.

## Best Possible Solution

Implement the feature in the Control UI chat owner area: add accessible message-level Reply controls plus a right-click menu, track a composer-local selected message, focus the textarea, show a dismissible quote preview, and prepend a safe Markdown blockquote when sending. Keep gateway protocol, storage migrations, and channel-native threading separate unless maintainers explicitly promote that broader design.

Yes. A high-confidence source-level reproduction path exists: inspect current main's Control UI chat renderer and send path and confirm there is no context-menu handler, reply action callback, selected quote preview state, or reply field in chat.send.

## Solution Assessment
