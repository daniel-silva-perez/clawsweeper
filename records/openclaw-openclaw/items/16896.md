## Summary

Keep open. Current main still lacks the requested Dashboard/WebChat right-click reply workflow: there is no context-menu handler, message-level Reply action, composer-selected quote state, or quote preview, and the existing reply pill only renders inbound assistant directive metadata. The related #29732 is already closed back to this request as the canonical UI tracker, while open PR #62262 is gateway/channel reply-threading work and does not implement the Control UI UX.

## What This Changes

Add a right-click context menu and Reply action to Dashboard/WebChat messages so the selected message is quoted or referenced in the composer before sending.

## Best Possible Solution

Implement the feature in the Control UI chat owner area: add accessible message-level Reply controls plus a right-click menu, keep composer-local selected-message quote state, focus the textarea, show a dismissible quote preview, and prepend a safe Markdown blockquote on send. Leave gateway protocol/storage reply metadata and channel-native threading to a separate explicit design if maintainers want it.
