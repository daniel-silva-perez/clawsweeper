## Summary

Keep open. Current main still lacks the requested Dashboard/WebChat right-click or message-level reply workflow: the chat surface has no context-menu handler, no Reply action in the rendered message actions, no selected quote state in the composer, and the send path still submits only plain message text plus attachments. The related closed WebChat quote issue points back here as the canonical UI tracker, while open PR #62262 is adjacent gateway/channel reply-threading work and does not implement this Control UI UX.

## What This Changes

Add a right-click context menu and Reply action in Dashboard/WebChat so users can quote or reference a specific prior message from the composer.

## Best Possible Solution

Implement this in the Control UI chat owner area: add accessible message-level Reply controls plus a right-click menu, keep composer-local selected-message quote state, focus the textarea, show a dismissible quote preview, and prepend a safe Markdown blockquote on send. Leave gateway protocol/storage reply metadata and channel-native threading for a separate design if maintainers want that broader model.
