## Summary

Keep open: current main still lacks the requested user-selected Dashboard/WebChat reply workflow. The relevant chat surface has no context-menu handler, no Reply action in the message toolbar, no selected quote state in the composer, and the existing reply pill only renders inbound assistant directive metadata. The closed related #29732 points back here as the canonical tracker, and a search found no open PR with closing syntax for this issue.

## What This Changes

Request adds a right-click Reply option in Dashboard WebChat that quotes or references a selected message and focuses the composer.

## Best Possible Solution

Implement the feature in the Control UI chat owner area: add accessible message-level Reply controls plus a right-click menu, track the selected message in composer state, focus the textarea, show a dismissible quote preview, and prepend a safe quoted-Markdown block on send. Keep persisted reply metadata, storage migrations, and channel threading separate unless maintainers promote that broader design.
