## Summary

Keep this open. Current main still ships the requested installer responsibilities through one 2,770-line `scripts/install.sh`, with no `scripts/install/` helper tree, and the docs/tests/smoke workflow still assume the single public installer surface. The report is valid, but the change is a broad public installer refactor that needs maintainer agreement on module boundaries and compatibility before implementation.

## What This Changes

Request to split `scripts/install.sh` into a smaller main entrypoint plus sourced shell helper modules for UI, download, platform, npm/git install, and build-tools logic.

## Best Possible Solution

Keep the public `scripts/install.sh` and `https://openclaw.ai/install.sh` entrypoints stable while extracting narrow sourced helpers behind that wrapper, then update installer docs plus unit and Docker smoke coverage around the new module layout.
