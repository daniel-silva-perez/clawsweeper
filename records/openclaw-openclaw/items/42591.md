## Summary

Keep this open. Current main still has the requested installer responsibilities in one public `scripts/install.sh` entrypoint, with no `scripts/install/` module tree, and the docs/tests/smoke workflow still assume the single-script installer surface. The earlier ClawSweeper review comment reached the same keep-open conclusion, and the current checkout still supports that result.

## What This Changes

Request to split the large Bash installer into sourced modules for UI, downloading, platform detection, build tools, npm/git install logic, gum support, and the main flow.

## Best Possible Solution

Preserve `scripts/install.sh` and `https://openclaw.ai/install.sh` as stable public entrypoints while extracting narrow sourced helpers behind that wrapper, then update installer docs plus unit and Docker smoke coverage around the new module boundaries.
