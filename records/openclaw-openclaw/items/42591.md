## Summary

Keep open. Current main still has the monolithic Bash installer structure described by the report, and the requested split would affect a public curl-piped installer plus release and smoke-test assumptions, so this needs maintainer architecture/release review rather than conservative cleanup closure.

## What This Changes

The issue requests splitting `scripts/install.sh` into smaller installer modules for UI, download, platform detection, npm/git install logic, build-tool setup, and the main entrypoint.

## Best Possible Solution

Preserve `scripts/install.sh` and `https://openclaw.ai/install.sh` as stable public entrypoints, then land a maintainer-approved modularization or build-time bundling plan that updates installer docs, unit probes, Docker smoke coverage, and release validation together.

## Reproduction Assessment

Not applicable as a runtime bug; this is a maintainability refactor request. The source-inspection path is high confidence: current main has no `scripts/install/` module tree and still keeps the requested responsibilities in `scripts/install.sh`.

## Solution Assessment
