## Summary

Current main still has the requested installer responsibilities concentrated in one large public Bash installer, with tests, docs, and CI smoke coverage built around `scripts/install.sh` and `https://openclaw.ai/install.sh`. The request is valid, but it is broad public installer work that needs maintainer direction before a safe split.

## What This Changes

The issue requests splitting `scripts/install.sh` into modules for UI, download, platform detection, npm install logic, git install logic, build-tool setup, and gum integration.

## Best Possible Solution

Keep `scripts/install.sh` and `https://openclaw.ai/install.sh` stable as the public wrapper, then extract narrowly scoped sourced helpers only after maintainers approve the layout, delivery model, compatibility plan, docs updates, targeted tests, Docker smoke coverage, and release validation.
