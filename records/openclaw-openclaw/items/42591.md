## Summary

Keep open because current main still has the monolithic Bash installer shape described in the report, and the requested module split is not present in source, docs, or installer tests.

## What This Changes

The issue requests splitting the public Bash installer into smaller modules for UI, download, platform detection, npm, git, and build-tool responsibilities.

## Best Possible Solution

Keep `scripts/install.sh` and `https://openclaw.ai/install.sh` as stable public entrypoints, then land a maintainer-approved modularization that updates installer docs, unit probes, Docker installer smoke coverage, and release-check expectations together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a maintainability refactor request. Source inspection gives a high-confidence verification path: current main has no `scripts/install/` module tree and still keeps the installer responsibilities in `scripts/install.sh`.

## Solution Assessment

Unclear whether the exact proposed file tree is the best final shape. The maintainable solution is a reviewed installer architecture that preserves the public entrypoint and validates the curl-piped, npm, git, and release-smoke paths.
