## Summary

Keep open: current main still lacks the requested modular installer tree, and the public installer remains a single documented, tested, and CI-smoked Bash surface. The refactor request is valid, but it is broad public installer work that needs maintainer architecture, security, and release-validation decisions before implementation.

## What This Changes

The issue requests splitting `scripts/install.sh` into a smaller main entrypoint plus focused helper modules for UI, downloads, platform detection, npm/git install logic, build tools, and gum handling.

## Best Possible Solution

Preserve `scripts/install.sh` and `https://openclaw.ai/install.sh` as the stable public entrypoint while maintainers decide an approved helper-module architecture, then land the split with matching docs, unit probes, Docker installer smoke coverage, and release-check updates.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a maintainability refactor request. The current layout is high-confidence to verify by source inspection: no `scripts/install/` tree exists and the public Bash installer remains a 2,773-line single file.

## Solution Assessment

Unclear whether the exact proposed module tree is the best final shape. The direction is reasonable, but the safer path is a maintainer-approved installer architecture and delivery model before changing the public shell execution path.
