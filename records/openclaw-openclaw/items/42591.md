## Summary

Keep this issue open. Current main still has the requested installer responsibilities concentrated in `scripts/install.sh`, the proposed `scripts/install/` module tree is absent, and docs/tests still target the single public installer entrypoint. The prior ClawSweeper comment matches the current checkout after a fresh shell check against main SHA `17811480dae20f65b526454d75e971ae8424ec10`.

## Best Possible Solution

Keep this issue open for a conservative installer maintainability refactor. Preserve `scripts/install.sh` as the stable public entrypoint, extract narrow sourced modules for UI/logging, download/checksum helpers, gum bootstrap, platform/package-manager detection, build tools, npm install/repair, git install, and main argument flow, then update `docs/install/installer.md` and targeted installer tests/smokes around the new layout.
