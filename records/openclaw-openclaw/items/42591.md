Keep open. Current main still has the monolithic Bash installer shape described in the report, with no requested `scripts/install/` module tree, and the refactor would affect the public curl-piped installer, sourceable tests, Docker smoke, and release validation paths.

## What This Changes

The issue requests splitting `scripts/install.sh` into smaller installer modules for UI, download, platform detection, npm/git install, and build-tool setup to improve maintainability and testability.

## Best Possible Solution

Preserve `scripts/install.sh` and `https://openclaw.ai/install.sh` as stable public entrypoints, then land a maintainer-approved modularization or build-time bundling plan that updates installer docs, unit probes, Docker smoke coverage, and release validation together.
