## Summary

Keep open: current main still has no modular installer tree, and the single public Bash installer remains the documented, tested, and CI-smoked surface. The refactor request is valid, but it is broad public installer work that needs maintainer design and security/release review before implementation.

## What This Changes

The issue requests splitting the large `scripts/install.sh` installer into smaller sourced shell modules for UI, download, platform, npm, git, and build-tool responsibilities.

## Best Possible Solution

Keep the issue open until maintainers decide the installer architecture, then preserve `scripts/install.sh` and `https://openclaw.ai/install.sh` as the stable public entrypoint while extracting approved helper modules with matching docs, tests, Docker smoke coverage, and release validation.
