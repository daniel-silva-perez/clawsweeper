## Summary

Keep open. Current main still lacks the target-derived outbound tool context this PR is trying to add, and the PR head is not merge-ready because the new context is not carried through every delivery path needed for the reported multi-channel approval failure.

## What This Changes

The PR adds `toolContext` plumbing for forwarded exec-approval notices, derives current channel identity from the resolved forward target, and adds regression coverage for request and resolved deliveries.

## Best Possible Solution

Land an adapted fix on current main that makes target-derived channel identity part of the typed outbound delivery and queue contracts, reaches real channel adapter contexts, and is covered for immediate forwarded approval sends, resolved notices, and recovery replay.
