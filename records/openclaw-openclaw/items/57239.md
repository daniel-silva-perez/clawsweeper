## Summary

Keep this PR open. Current main still serializes active-session replies through the followup queue and does not contain the proposed fast-lane override or short-message heuristics. Main does have `/btw` for explicit tool-less side questions, which overlaps with part of the UX problem, but it is not the same automatic immediate-response behavior this PR proposes for #56880. The PR is still a plausible implementation candidate, but it needs maintainer review and fixes for the concurrency/session-isolation concerns raised in review.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is either to revise this branch into a safe, tested implementation of #56880 or to close it manually in favor of a narrower maintainer-approved design, likely building on the existing `/btw` side-result model or a carefully isolated fast-lane API with explicit queue, session-state, and tool-safety guarantees.
