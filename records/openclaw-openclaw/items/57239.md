## Summary

Keep this PR open. Current main still serializes active-session replies through the followup queue and does not implement the PR's automatic fast-lane override for ordinary short messages. The shipped `/btw` path overlaps with quick side questions, but it is explicit, tool-less, and ephemeral, so it does not supersede the proposed behavior. The linked feature request #56880 is still open, and the PR discussion contains unresolved queue/session-safety concerns rather than evidence that the work is obsolete.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to revise it into a safe, tested implementation of #56880 with explicit queue ordering, session isolation, no-tool or tool-safety guarantees, and integration coverage around busy sessions, or have a maintainer close it manually in favor of a narrower design built on the existing `/btw` side-result model.
