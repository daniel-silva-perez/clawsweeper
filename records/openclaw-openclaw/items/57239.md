## Summary

Keep this PR open. Current main still serializes active-session replies through the followup queue and does not implement the proposed automatic fast-lane override for short messages. Main has `/btw` for explicit ephemeral side questions, but that is not the same behavior as ordinary quick messages bypassing the queue. The linked feature request #56880 remains open, and the PR discussion shows unresolved concurrency/session-isolation concerns rather than obsolete work.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to either revise this branch into a safe, tested implementation of #56880 with explicit queue/session/tool-safety guarantees, or have a maintainer close it manually in favor of a narrower design, possibly building on the existing `/btw` side-result model.
