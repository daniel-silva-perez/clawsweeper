## Summary

Keep open. Current main has partial follow-up for the doctor migrations, Slack socket handling, and private-WS messaging, but one of the central requested stability changes is still intentionally not implemented: subagent completion/direct announce calls still default to 120s, and the regression test asserts that default. The issue also contains two concrete gateway memory/Slack stale-socket reports, so it should not be swept closed as implemented without a narrower verified fix or split follow-up.

## Best Possible Solution

Keep this issue open until maintainers either reduce/rework the subagent announce timeout default and verify the gateway memory/Slack stale-socket reports, or split the remaining symptoms into narrower canonical issues. The shipped doctor and Slack hardening should be preserved, but they are not enough to close the whole report.
