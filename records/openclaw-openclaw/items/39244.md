## Summary

Current main still does not implement the PR’s requested `tools.exec.nsSandbox` / bubblewrap layer for safeBins-approved gateway exec. Main has safe-bin argv/path hardening and full sandbox backends, but no safeBins-only Linux namespace filesystem isolation, no config/schema surface for `nsSandbox`, and no bwrap wrapping in the gateway approval or runtime spawn paths. This PR remains a plausible but security-sensitive implementation candidate rather than an auto-close item.

## Best Possible Solution

Keep this PR open for maintainer/security judgment. The best path is a narrowed, rebased optional Linux bwrap layer for safeBins-approved gateway exec that updates config types, strict schemas, schema help/labels/generated docs, per-agent defaults, gateway allowlist and approval follow-up execution, runtime spawning, user docs, and focused regression tests. Remove unrelated package/UI/test churn unless separately justified. If maintainers decide core bwrap support is not wanted, close only with an explicit product/security rationale and a documented alternative using current sandbox backends.
