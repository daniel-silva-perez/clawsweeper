Keep this PR open. Current main does not implement the proposed `tools.exec.nsSandbox` / bubblewrap namespace sandbox for safeBins-approved gateway exec, and the PR remains a plausible but security-sensitive implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open for maintainer/security follow-up. The best path is a narrowed, rebased optional Linux bwrap layer for safeBins-approved gateway exec that wires config types, strict schemas, schema help/generated docs, per-agent defaults, gateway allowlist and approval follow-up execution, runtime spawning, public docs, and focused regression tests. Split or drop unrelated package/UI/browser-test changes unless separately justified. If maintainers decide core bwrap support is not wanted, close only with an explicit product/security rationale and a documented alternative using the existing sandbox backends.
