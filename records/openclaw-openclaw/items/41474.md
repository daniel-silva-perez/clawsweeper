## Summary

Keep open. Current main still matches the unresolved macOS companion behavior: signed builds enable Sparkle updates by default, one preference still drives both automatic checks and downloads, and the macOS Swift config reader audits update-channel-only clobbers but returns the clobbered config instead of restoring the prior good config. The reporter also already confirmed the suggested recovery workaround did not help.

## Best Possible Solution

Keep this open for the macOS companion owner area. Split persisted updater preferences so automatic checks, automatic downloads, install-on-quit/restart behavior, and beta-channel opt-in are explicit; route companion config reads/writes through the Gateway/core config I/O recovery path when possible or port equivalent last-known-good restore behavior into `OpenClawConfigFile`; add signed-build updater preference coverage and an update-channel-only clobber regression test that proves the companion returns/restores the good config.
