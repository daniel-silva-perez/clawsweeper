## Summary

Keep open. Current main still leaves the macOS companion update/config-loss path unresolved: signed builds default Sparkle update checks/downloads on, use one preference for checks and downloads, and the macOS config reader audits update-channel-only clobbers but returns the clobbered config instead of restoring a known-good state. The reporter also confirmed the suggested recovery workaround did not help.

## Best Possible Solution

Keep this open for the macOS companion owner area. Split Sparkle preferences for checking, downloading, install-on-restart/quit behavior, and beta-channel opt-in; default risky automatic apply/download behavior conservatively; and reuse or port Gateway last-known-good recovery so `OpenClawConfigFile` restores a good config instead of returning an audited clobber. Add/update a regression test proving the companion returns/restores the good config after an update-channel-only clobber.
