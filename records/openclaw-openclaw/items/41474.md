## Summary

Keep open. Current main still matches the unresolved macOS companion behavior: signed builds default the Sparkle updater on, one default-on preference drives both automatic checks and automatic downloads, and the Swift config reader detects update-channel-only clobbers but returns the clobbered config instead of restoring the previous config. The reporter also confirmed the suggested recovery workaround did not help.

## Best Possible Solution

Keep this open for the macOS companion owner area. Split persisted updater settings so automatic checks, downloads, install-on-quit/restart behavior, and beta-channel opt-in are explicit; route companion config reads/writes through the core config I/O recovery path or port equivalent last-known-good restore behavior into OpenClawConfigFile; add signed-build updater and update-channel-only clobber regression coverage.
