## Summary

Keep open. Current main still leaves the macOS companion path unresolved: signed builds create a Sparkle updater at launch, default automatic updates to enabled when no preference exists, use one preference for both checks and downloads, and the Swift config reader audits update-channel-only clobbers but returns the clobbered config instead of restoring the last-known-good state. The reporter also confirmed the suggested recovery workaround did not help.

## Best Possible Solution

Keep this open for the macOS companion owner area. Split Sparkle updater preferences for automatic checks, automatic downloads, install-on-restart/quit behavior, and beta-channel opt-in; default risky apply/download behavior conservatively; reuse the Gateway/core last-known-good config recovery path from the companion where possible or port equivalent restore behavior into `OpenClawConfigFile`; then replace the current clobber-audit-only test with a regression test proving the companion returns/restores the good config.
