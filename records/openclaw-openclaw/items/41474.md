## Summary

Keep open. Current main still has the reported macOS companion risk: signed Sparkle builds default automatic checks/downloads on through one setting, and the macOS config reader only audits/preserves an update-channel-only clobber instead of restoring the known-good config. The issue discussion also shows the suggested recovery workaround did not help the reporter.

## Best Possible Solution

Keep this open for the macOS companion owner area. Split Sparkle preferences for checking, downloading, install-on-restart/quit behavior, and beta-channel opt-in; default risky automatic apply/download behavior conservatively; and port or reuse the Gateway last-known-good recovery path so `OpenClawConfigFile.loadDict()` restores/returns a good config after an update-channel-only clobber. Add a macOS regression test that expects the recovered config, not the clobbered root.
