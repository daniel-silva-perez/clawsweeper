## Summary

Keep open. Current main has meaningful partial fixes for session transcript noise, but the central request is still not complete: daily/session ingestion still assigns fixed source scores, session dedupe is still tied to timestamp-or-line position, and existing tests only prove draining one large transcript rather than fair rotation across many large session files.

## What This Changes

The issue requests better memory-core dreaming corpus filtering, content-aware scoring, recency weighting, and fair session-corpus scan rotation so useful session material can promote beyond light sleep.

## Best Possible Solution

Keep the existing heartbeat/cron/sentinel filtering, then finish memory-core ingestion with deterministic content-level dedupe, a bounded source/content/recency scoring helper before `recordShortTermRecalls`, and regression coverage that proves multi-file session corpus rotation under tight sweep budgets.
