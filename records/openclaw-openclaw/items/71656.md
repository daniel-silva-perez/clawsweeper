## Summary

Keep open. Current main has useful partial denoising and checkpointing for memory-core dreaming, but the central request remains incomplete: ingestion scores are still fixed, repeated content can still be treated as distinct when timestamp or line changes, and the session scan path still lacks a strong fair-rotation guard for many files under a tight cap.

## What This Changes

The issue requests corpus denoising/deduplication, content-aware weighted scoring, recency bias, and fair scan rotation for memory-core dreaming promotion.

## Best Possible Solution

Finish this inside memory-core and the session export layer: keep the existing cron/heartbeat/sentinel filtering, add deterministic content-level dedupe for repeated session snippets, compute bounded content/source/recency ingestion scores before recall recording, and add regression coverage showing many transcript files all make progress under a tight sweep cap.

## Reproduction Assessment

Yes for the remaining source-level defects: current main directly shows fixed ingestion scores, position-scoped dedupe, and only partial scan-budget coverage. The exact April 9 production stall is unclear without the reporter's local phase-signals/session state.

## Solution Assessment

Yes, a narrow memory-core/session export repair is the best direction. It should avoid core plugin API changes, avoid changing Dream Diary artifact behavior, and coordinate with adjacent memory dreaming reports before altering promotion-gate semantics.
