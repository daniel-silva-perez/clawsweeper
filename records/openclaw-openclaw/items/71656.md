## Summary

Keep open. Current main has meaningful partial denoising and checkpointing for memory-core dreaming, but the central request is still incomplete: session and daily ingestion still use fixed source scores, session corpus dedupe is timestamp-or-line scoped rather than content scoped, and the multi-file session scan fairness path is not covered strongly enough to close the reported stall.

## What This Changes

Request memory-core dreaming to pre-filter/deduplicate noisy session corpus entries, score corpus items by content/source quality and recency, and ensure session corpus scans continue under growing scan budgets.

## Best Possible Solution

Keep the existing heartbeat/cron/sentinel filtering, then finish the owner-local memory-core path with deterministic content-level dedupe, a bounded source/content/recency scoring helper before `recordShortTermRecalls`, and regression coverage proving many session files make progress under a tight sweep budget.

## Solution Assessment
