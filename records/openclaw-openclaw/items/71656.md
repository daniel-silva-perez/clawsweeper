## Summary

Keep open. Current main has useful partial denoising and checkpointing for memory-core dreaming, but the core request remains incomplete: ingestion scores are still fixed, repeated content can still be tracked as distinct when timestamp or line changes, and the tight-budget multi-file scan path still lacks strong fairness coverage.

## What This Changes

Request memory-core dreaming to pre-filter and deduplicate noisy session corpus entries, weight daily/session ingestion by content quality and recency, and keep session corpus scans progressing as the corpus grows.

## Best Possible Solution

Finish this in memory-core and the session export layer: keep the existing heartbeat/cron/sentinel filtering, add deterministic content-level dedupe for repeated session snippets, score daily/session ingestion with a bounded content/source/recency helper before recall recording, and add regression coverage showing many transcript files all make progress under a tight sweep cap.

## Reproduction Assessment

Yes for the remaining code-level defects: current main directly shows fixed ingestion scores, position-scoped session dedupe, and only partial scan-budget coverage. The exact production April 9 stall is not high-confidence reproducible without the reporter's local dreaming state.

## Solution Assessment

Yes, a narrow memory-core/session export fix is the best direction. It should avoid core plugin API changes, avoid Dream Diary behavior changes, and coordinate with adjacent PRs before finalizing scoring or promotion-gate semantics.
