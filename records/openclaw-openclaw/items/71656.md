## Summary

Keep open. Current main has partial fixes for session-corpus noise and cursoring, but the central request is still incomplete: daily/session ingestion still uses fixed source scores, repeated identical session content can be treated as new when timestamp or line differs, and there is still no many-file fairness guard for tight session scan budgets.

## What This Changes

The issue requests memory-core dreaming improvements for noisy session corpus ingestion, content-aware scoring, recency/rotation-aware scan budgeting, and avoiding session corpus starvation.

## Best Possible Solution

Keep the existing heartbeat/cron/sentinel filtering, then finish the memory-core owner-local path with deterministic content-level dedupe, a bounded source/content/recency scoring helper before `recordShortTermRecalls`, and regression coverage that proves many session files make progress under tight sweep budgets.
