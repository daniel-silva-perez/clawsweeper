## Summary

Keep open. Current main has useful partial denoising for session transcript export, but the broader request remains incomplete: ingestion scores are still fixed, repeated content can still become distinct when timestamp or line changes, and scan-budget fairness is only partially covered. The related #68876 discussion is useful context for one corpus-noise path, but it does not finish this issue.

## What This Changes

Improve memory-core dreaming by filtering and deduplicating noisy session corpus entries, weighting ingestion scoring by content/source/recency, and rotating session scans so all corpus sources keep making progress.

## Best Possible Solution

Finish this in memory-core and the session export layer: preserve the current heartbeat/cron/sentinel filtering, add deterministic content-level dedupe for repeated session snippets, score daily/session ingestion with a bounded content/source/recency helper before recall recording, and add tight-budget multi-file scan fairness coverage.

## Reproduction Assessment

Yes for the remaining code-level defects: current main directly shows fixed ingestion scores, position-scoped session dedupe, and missing many-file fairness coverage. The exact production April 9 stall is unclear without the reporter's local dreaming state.

## Solution Assessment

Yes, a narrow memory-core/session-files repair is the best direction. It should avoid core plugin API changes and keep existing weighted deep ranking while improving the input signals and scan progress feeding it.
