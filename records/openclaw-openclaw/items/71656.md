## Summary

Keep open. Current main has partial noise filtering for session transcript ingestion, but the central request is still incomplete: daily and session ingestion still feed fixed source scores into promotion state, session dedupe is still scoped by timestamp-or-line position, and coverage still proves one large transcript drains rather than fair rotation across many large session files.

## What This Changes

The issue asks memory-core dreaming to pre-filter and dedupe noisy session corpus content, score entries by content quality and recency rather than fixed source constants, and rotate scan budget fairly across corpus sources.

## Best Possible Solution

Finish memory-core ingestion with deterministic content-level dedupe, a bounded source/content/recency scoring helper before `recordShortTermRecalls`, and regression coverage that proves fair multi-file session corpus progress under tight sweep budgets while preserving the existing heartbeat, cron, sentinel, and sensitive-content filters.

## Reproduction Assessment

Yes for the source-level defects: current main directly shows fixed source scores and timestamp-or-line-scoped session dedupe, and the searched tests lack a multi-file fairness case. The exact production stall remains source-inferred because this read-only review did not replay the reporter's private corpus.

## Solution Assessment

Yes, the best path is a focused memory-core fix in ingestion/session export and promotion tests. It should avoid unrelated memory search, Dream Diary narrative generation, or core plugin API changes.
