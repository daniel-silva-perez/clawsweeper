## Summary

Keep this PR open. Current main still lacks the proposed Bonjour per-instance mDNS service-name config: the bundled bonjour plugin still derives the advertised service instance from `ctx.machineDisplayName`, the manifest exposes no `instanceName` config key, and the docs only document hostname/disabling knobs. The PR remains a focused implementation candidate for #54467, with no security-sensitive changes evident in the current PR diff.

## Best Possible Solution

Review and land this PR or an equivalent bonjour-plugin-owned implementation: add `plugins.entries.bonjour.config.instanceName` to the bonjour manifest schema, read it through `api.pluginConfig` in `extensions/bonjour/index.ts`, preserve the existing `(OpenClaw)` suffix behavior, and keep the focused plugin-entry tests, docs, and changelog coverage.
