## Summary

Keep this PR open. Current main still lacks the proposed Bonjour per-instance mDNS service-name config: the bundled bonjour plugin always derives `instanceName` from `ctx.machineDisplayName`, the manifest exposes no `instanceName` config property, and the public docs still only document hostname/disabling knobs. The PR remains a focused implementation candidate for the related multi-gateway Bonjour collision report, with no security-sensitive changes evident from the provided diff.

## Best Possible Solution

Review and land this PR or an equivalent bonjour-plugin-owned implementation: add `plugins.entries.bonjour.config.instanceName` to the bonjour manifest schema, read it through `api.pluginConfig` in `extensions/bonjour/index.ts`, route it through the existing `(OpenClaw)` suffix semantics, and keep the tests, docs, and changelog coverage. Closing the PR now would leave the per-instance mDNS service-name gap unresolved on current main.
