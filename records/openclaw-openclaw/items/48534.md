## Summary

Keep this issue open. Current main has partial hook timeout infrastructure, but it still does not apply a default/global timeout to legacy `before_agent_start` handlers. A plugin that registers `before_agent_start` without `opts.timeoutMs` can still block both the model-resolution and prompt-build paths indefinitely.

## Best Possible Solution

Add a fail-open default timeout for legacy `before_agent_start`, or a documented global plugin hook timeout policy that covers it while preserving per-registration `timeoutMs` overrides. The timeout should log plugin id, hook name, elapsed budget, and run id when available, then continue the run. Add regression tests for hung `before_agent_start` handlers on both model-resolution and prompt-build paths, and align config schema/docs if a global setting is introduced.
