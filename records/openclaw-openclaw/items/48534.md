## Summary

Keep this issue open. Current main has timeout infrastructure for typed hooks, but the reported core guardrail is still missing for legacy `before_agent_start`: handlers without an explicit registration `timeoutMs` can still be awaited indefinitely on the agent model-resolution and prompt-build paths. The related OpenViking plugin issue was closed and the OpenClaw PR #48741 attempted the right direction, but that PR is closed unmerged and current main does not expose a global `plugins.hookTimeoutMs` style config.

## Best Possible Solution

Add a bounded default timeout for modifying lifecycle hooks that includes legacy `before_agent_start`, preferably with a documented global/default policy and per-registration override. The fix should preserve fail-open behavior for prompt/model hooks, log plugin id, hook name, elapsed budget, and run id when available, and add regression tests proving a hung `before_agent_start` handler does not prevent later hooks or the agent run from continuing.
