## Summary

Current main still does not implement the requested generic model rate limiter. OpenClaw has adjacent reactive handling for 429/Retry-After, auth-profile cooldown/rotation, model fallback, and provider usage/status snapshots, but there is no documented config/schema or shared request-path scheduler for per-model tokens-per-interval or requests-per-interval budgets. The commenter’s OpenAI `x-ratelimit-*` suggestion also remains unimplemented as scheduling behavior.

## Best Possible Solution

Keep this issue open for a core model request-path design: add documented per-provider/per-model, and likely per-auth-profile, request and token budget schema with interval windows; enforce it before shared model calls; reconcile actual usage after responses/streams; optionally ingest provider headers such as OpenAI `x-ratelimit-*`; and add focused tests/docs covering scheduler, fallback, retry, streaming, and auth-profile interactions.
