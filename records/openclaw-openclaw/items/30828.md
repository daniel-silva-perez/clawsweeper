## Summary

Current main still does not implement manually configured model request/token budgets. The code has adjacent reactive handling for provider 429s, Retry-After, auth-profile cooldowns, key/profile rotation, and model fallback, but there is no config/schema/docs surface or shared request-path scheduler for tokens per interval or requests per interval. The OpenAI x-ratelimit header suggestion in the issue discussion remains observation-only in docs/live header probes, not enforced scheduling behavior.

## Best Possible Solution

Keep this issue open for a core model request-path design: add documented per-provider/per-model, and likely per-auth-profile, request and token budget schema with interval windows; enforce preflight scheduling before shared model calls; reconcile actual usage after responses and streams; optionally ingest provider headers such as OpenAI x-ratelimit-*; and add focused tests/docs covering scheduler, fallback, retry, streaming, and auth-profile interactions.
