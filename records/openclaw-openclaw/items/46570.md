## Summary

Current main still does not fix the reported builtin CLI behavior. Memory and session chunks are indexed with separate sources, but `openclaw memory search` still has no corpus/source option and the builtin hybrid path can return strict session-heavy results before relaxed keyword-backed memory-file hits are considered.

## Best Possible Solution

Keep this issue open and fix the builtin memory-core mixed-source search path. Add a regression where an exact phrase present only in `MEMORY.md` or `memory/*.md` appears in default mixed `sources: ["memory", "sessions"]` builtin hybrid search even when session transcript chunks also produce strict hits, then adjust ranking/result selection so keyword-backed memory hits are not dropped solely because session results meet strict scoring. A CLI `--corpus memory|sessions|all` can be added, but default mixed search should still surface relevant memory-file hits.
