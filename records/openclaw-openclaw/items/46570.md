## Summary

Keep this open. Current main has source-aware memory/session indexing, but the builtin hybrid search path still only relaxes keyword-only results when there are no strict hits at all. With session transcript vector hits above the default threshold, exact memory-file keyword hits can still be filtered out, which matches the reported local-provider mixed memory/session failure mode. The QMD backend has a source-diversification fix and tests, but the reported builtin/local path does not show equivalent coverage.

## Best Possible Solution

Keep the issue open and fix the builtin `memory-core` hybrid search path. The likely maintainer path is to add deterministic regression coverage for mixed `memory` + `sessions` sources under the builtin local/vector+FTS manager, then preserve exact memory-file keyword hits through result limiting, either with source diversification like QMD or a keyword-retention quota that still respects explicit `sources` filters and `maxResults`.

## Evidence

- **Issue context:** The report is for OpenClaw 2026.3.11 with provider `local`, hybrid mode, memory files indexed with chunks, and searches returning only session transcript results despite exact memory-file content. Related #47730 is explicitly different: memory files were found there but snippets pointed at the start of files.
- **Builtin indexing is source-aware:** Builtin indexing writes every chunk to `chunks` and `chunks_fts` with a `source` field, so current main does distinguish `memory` from `sessions` while indexing.
