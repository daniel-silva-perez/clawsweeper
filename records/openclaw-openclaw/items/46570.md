## Summary

Keep open. Current main still has the reported mixed-source hybrid search failure shape: indexed memory files and session transcripts share the same default search path, but strict hybrid session hits can return before relaxed keyword-backed memory hits are considered.

## What This Changes

The issue reports that `openclaw memory search` returns session transcript matches while omitting exact matches from indexed `MEMORY.md` or `memory/*.md` files in a mixed memory-plus-sessions setup.

## Best Possible Solution

Fix the selection/ranking in `MemoryIndexManager.search` so default all-source hybrid search still surfaces relevant keyword-backed memory-file hits when session chunks also produce strict hits. Add regression coverage for mixed `memory` plus `sessions` sources; a CLI corpus filter can be additive, but should not be the only fix.

## Reproduction Assessment

Yes. The issue gives a concrete CLI reproduction, and the same behavior can be made deterministic in a unit test by seeding an exact phrase in a memory file plus session chunks that produce strict hits, then asserting a memory-source hit is returned.

## Solution Assessment

Yes. A narrow manager-level hybrid selection fix with manager and CLI-equivalent regression coverage is the best path; a CLI-only source filter would be a workaround and would leave default mixed search behavior broken.
