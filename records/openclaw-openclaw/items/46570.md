## Summary

Keep open. Current main still has the reported mixed-source hybrid search failure shape: memory files and session transcripts can both be indexed, but the builtin CLI search uses the default mixed manager search, and the manager returns strict merged hits before considering relaxed keyword-backed memory hits. That means relevant exact memory-file matches can still be crowded out when session transcript vector hits pass the strict score threshold.

## What This Changes

The issue requests that `openclaw memory search` return indexed `MEMORY.md` and `memory/*.md` matches instead of returning only session transcript results in mixed-source hybrid mode.

## Best Possible Solution

Fix the builtin memory-core search path so default mixed-source hybrid search preserves relevant memory-file keyword hits even when session transcript vector hits also satisfy strict scoring. A CLI `--corpus memory|sessions|all` option would be useful as an additional operator control, but the default mixed search should not require that workaround.

## Reproduction Assessment

Yes. The issue gives concrete CLI steps, and the current source shows a focused test path: enable builtin hybrid search with memory plus sessions, add an exact phrase only in a memory file, add session chunks that produce strict vector hits, then assert the manager and CLI-equivalent path still return a memory-source hit.

## Solution Assessment

Yes. A narrow memory-core manager fix with regression coverage is the best path; a CLI-only source filter would help diagnosis but would not solve the default mixed-source ranking bug reported here.
