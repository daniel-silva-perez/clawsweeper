## Summary

Current main still does not provide high-confidence proof that the reported builtin CLI path is fixed. Memory files are indexed as `source: "memory"`, and tool calls can request `corpus=memory`, but `openclaw memory search` still calls `manager.search` without a source/corpus filter. In the builtin hybrid path, keyword-only memory hits are relaxed only when there are no strict merged hits at all, so strict session-vector hits can still crowd out exact memory-file matches.

## Best Possible Solution

Keep this issue open and add a focused builtin memory-core regression for mixed `sources: ["memory", "sessions"]` hybrid search: an exact phrase present only in `MEMORY.md` or `memory/*.md` should appear in `MemoryIndexManager.search` and the CLI-equivalent default search even when session transcript chunks also produce strict vector hits. Then adjust builtin ranking/result selection so keyword-backed memory hits are not dropped solely because session hits satisfy strict scoring. A CLI `--corpus memory|sessions|all` can be considered, but the default mixed search should still surface relevant memory-file hits.
