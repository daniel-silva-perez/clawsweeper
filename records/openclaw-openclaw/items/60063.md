## Summary

Keep open. Current main has part of the replacement protocol on the agent producer side, but the Gateway chat consumer still ignores `data.replace` and applies the normal 150ms throttle to all assistant deltas. The PR is not obsolete, but it needs rework because the submitted Gateway replacement path can blank the chat buffer for empty/media-only replace payloads.

## Best Possible Solution

Keep this PR open and ask for a narrowed Gateway patch: consume explicit assistant `replace` events in `server-chat`, bypass the delta throttle only for non-empty textual replacements or otherwise safe replacements, preserve existing text for empty/media-only replace payloads, and add regression coverage for same-window replacement plus empty replacement behavior. Keep the existing agent-side replacement producer behavior on main.
