## Summary

Keep open. Current main still stores cumulative WebChat stream snapshots as committed segments, and this PR only fixes that commit path; the live preview still renders cumulative text unchanged after prior segments, so the branch is not complete before merge and overlaps broader open fixes.

## What This Changes

The PR changes Control UI tool-stream segment commits to store only the newly streamed suffix after the total length of prior committed stream segments.

## Best Possible Solution

Land one focused WebChat streaming fix that treats cumulative assistant snapshots consistently across committed pre-tool segments and the live preview, preferably with a shared helper or explicit committed-offset state plus regression tests for multi text/tool cycles.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Deduplicate the live stream preview too:** `ui/src/ui/app-tool-stream.ts:444-451`
  - body: This slices text only when committing `chatStreamSegments`. After a tool starts, the next chat delta still assigns the gateway's full cumulative snapshot to `chatStream`, and `buildChatItems` renders that live stream unchanged after the committed segment. The pre-tool text can still appear again in the live bubble until another tool event or final/history reload, so the live preview needs the same committed-offset dedupe.
  - confidence: 0.86
