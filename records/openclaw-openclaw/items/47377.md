## Summary

Keep open. The PR addresses part of the cumulative streaming duplication, but the reviewed path still leaves cumulative live preview text rendered after already committed stream segments, and overlapping open PRs are tracking the more complete Web UI streaming fix.

## What This Changes

The branch changes `ui/src/ui/app-tool-stream.ts` so a stream segment committed before a tool card stores only the suffix after the total length of earlier committed segments.

## Best Possible Solution

Land one focused Control UI streaming fix that treats cumulative assistant snapshots consistently for both committed pre-tool segments and the live preview, preferably with a shared helper or explicit committed-offset state plus regression tests for multiple text/tool cycles.

## Reproduction Assessment

Yes. The source path is enough to reproduce: the gateway sends cumulative `mergedText`, Control UI stores it as `chatStream`, and the renderer appends that live stream unchanged after prior stream segments.

## Solution Assessment

No. Segment suffix slicing is necessary, but this PR is incomplete because the live preview needs the same committed-offset dedupe and focused regression coverage.
