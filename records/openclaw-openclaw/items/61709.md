## Summary

Keep open. Current main has per-agent auth order overrides, but I did not find a host-wide or atomic command that switches a provider profile across every agent-scoped auth store; the related open #8190 covers session auth-cache reset, not this auth-store migration request.

## What This Changes

Request a CLI command to atomically switch a provider auth profile across all agent-scoped auth stores and optionally remove stale profiles.

## Best Possible Solution

Add an owner-reviewed auth management path that enumerates configured and safely discoverable agent directories, validates the requested provider/profile per target before writing, applies existing per-store locks with dry-run and JSON reporting, defines partial-failure or rollback semantics, and documents verification through `models status --agent` and `--probe`.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the missing product surface: current CLI registration and implementation only support per-agent auth order overrides, and targeted search found no host-wide auth migration command. The exact local broken-profile case remains anecdotal without the reporter's stores.

## Solution Assessment

Is this the best way to solve the issue? Unclear. A global auth switch command is plausible, but directly editing `lastGood` or removing profiles across agents needs maintainer/secops agreement on validation, dry-run behavior, rollback, and live cache interaction before implementation.
