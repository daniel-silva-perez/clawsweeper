## Summary

Keep open. Current main supports per-agent auth order overrides, but I did not find a host-wide or atomic command that updates every agent-scoped auth store; the related open #8190 covers session auth cache reset, not this auth-store migration request.

## What This Changes

The issue requests a CLI command to atomically switch a provider's active auth profile across every agent-scoped auth store on the host.

## Best Possible Solution

Add an owner-reviewed auth management path that enumerates configured and discoverable agent dirs, validates the requested provider/profile for each target, writes per-agent auth-state order using existing file locks with dry-run/JSON reporting, defines clear partial-failure or rollback behavior, and documents verification with `models status --agent` and `--probe`.
