## Summary

Keep this issue open. Current main has useful adjacent tooling: per-agent `openclaw models auth order get|set|clear`, per-agent auth stores, status/probe diagnostics, and locked auth-state writes. It still does not provide the requested host-wide atomic migration or set-active command across all agent stores, and `lastGood` is explicitly not the active-profile selection mechanism, so the proposed workflow remains unsolved.

## Best Possible Solution

Keep this issue open and implement an additive core CLI flow under the existing `openclaw models auth` namespace, for example `models auth set-active --provider <id> --profile <id> --all-agents` or `models auth migrate --provider <id> --from <id> --to <id>`. It should enumerate configured agents via the existing agent-scope helpers, resolve each `agentDir`, validate the target profile per store, take the existing auth-store locks, write explicit per-agent `auth-state.json` order overrides, optionally remove stale profiles with dry-run/reporting, and coordinate with #8190 if active sessions need refresh after the switch.

## Risks / Open Questions

- An implementation should probably set explicit `order` in each agent's `auth-state.json`, not `lastGood`, because current routing deliberately does not prioritize `lastGood`.
- True all-file atomicity across multiple agent stores needs a careful locking and rollback/summary plan; failure must report exactly which stores changed and which did not.
- Any `--remove-stale` mode must avoid logging credential material and should require conservative confirmation or dry-run output.
