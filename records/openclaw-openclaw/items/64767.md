## Summary

Keep open. Current main still contains the reported unbounded JSONL transcript read/split paths in gateway session utilities, and active gateway/history/session callers still reach those paths before response caps or projections can reduce the data. The related direct fix PR #64858 is closed unmerged, so there is no implementation on main to close against.

## What This Changes

The issue reports that very large session JSONL transcripts can block the gateway event loop when transcript readers materialize the whole file and run regex line splitting.

## Best Possible Solution

Land a focused gateway transcript-reader fix that replaces full-file string materialization with bounded, decoder-safe line or entry iteration for message and usage readers, while preserving current branch traversal, compaction markers, sequence metadata, malformed-line tolerance, cross-agent transcript resolution, and usage aggregation semantics.

## Repair Work Prompt

Fix the bloated session JSONL gateway hang reported in this issue, using PR #64858 only as context. On current main, `src/gateway/session-utils.fs.ts` still uses full-file `fs.readFileSync(..., "utf-8")` plus regex splitting in `readSessionMessages` and `readLatestSessionUsageFromTranscript`; callers such as `chat.history`, session events, sessions history, send-followup sequencing, and managed image attachment indexing reach those helpers before output caps. Implement a bounded, decoder-safe transcript line/entry iterator that avoids materializing the whole JSONL as one string and avoids regex line splitting over the whole file. Use `StringDecoder` or equivalent handling so multi-byte UTF-8 across chunk boundaries is preserved. Do not rely on `SessionManager.open(filePath).getBranch()` for already-large tree transcripts unless you first prove that dependency path is bounded; pinned `@mariozechner/pi-coding-agent` 0.70.5 currently reads and splits the full file in its session manager loader. Preserve public function signatures, cross-agent `sessionFile` resolution, malformed-line tolerance, CRLF handling, `__openclaw.seq` metadata, compaction marker synthesis, active-branch selection for `id`/`parentId` tree transcripts, and usage aggregation/model/cost semantics. Keep scope to transcript reader safety; do not add session maintenance size caps, doctor warnings, or new product policy in the same PR. Add a user-facing changelog Fixes entry credited with `Thanks @ppronobis`. Validate with targeted gateway tests and leave broad `pnpm check:changed` for Testbox before handoff.
