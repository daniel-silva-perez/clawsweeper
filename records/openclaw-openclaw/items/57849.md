## Summary

Keep open. Current main still misses the PR's central behavior for agent_end and several before_compaction paths, while the PR is a real fix direction backed by the linked provenance report and Honcho attribution bug. The current patch should not merge as-is because it broadens raw transcript delivery to an ungated compaction hook and its new reader is not branch-aware.

## What This Changes

The PR adds a shared JSONL transcript-message reader and wires agent_end, before_compaction, and before_reset hook payloads to use transcript messages so provenance and custom fields reach plugins.

## Best Possible Solution

Refresh this PR into a narrow hook-payload fix that reuses or extracts a branch-aware transcript reader, preserves provenance/custom fields for agent_end and all before_compaction call sites, keeps the existing before_reset archive fallback, and either gates raw before_compaction/before_reset message delivery through the conversation-access policy or documents why that exposure is acceptable.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: provenance is applied on the persistence path, but current main still feeds agent_end from messagesSnapshot and leaves direct compaction paths without transcript-derived messages; the linked issue gives a concrete sessions_send multi-agent check.

## Solution Assessment

No, not as written. The direction is the right one, but the current patch needs a rebase plus branch-aware transcript handling and an explicit conversation-access decision before it is the narrowest maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Require conversation access for compaction messages:** `src/agents/pi-embedded-runner/compaction-hooks.ts:208`
  - body: This adds raw transcript messages to before_compaction, but that hook is not in the current CONVERSATION_HOOK_NAMES allowlist. A non-bundled plugin can register before_compaction without allowConversationAccess and receive conversation history/custom fields through this path, bypassing the documented conversation-content gate.
  - confidence: 0.86
- **[P2] Read only the active transcript branch:** `src/hooks/session-transcript-messages.ts:23-24`
  - body: The helper pushes every message record in the JSONL file. Tree-style transcripts can contain inactive branch/fork records; existing session history readers detect those entries and use SessionManager.getBranch(), so hook payloads from this helper can include messages outside the active conversation branch.
  - confidence: 0.79
