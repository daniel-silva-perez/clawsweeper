## Summary

Keep this PR open. Current main still does not implement the PR’s central behavior for `agent_end` and several `before_compaction` hook paths: provenance is persisted to transcripts, but those hook payloads still come from in-memory messages or metrics-only events. `before_reset` is now largely covered on main by its own transcript parser, so that part of the PR is partly redundant rather than obsolete.

## Best Possible Solution

Keep this PR open and ask the author to refresh it against current main. The best path is a narrow implementation that reuses or extracts the current transcript JSONL parser, preserves provenance/custom fields for `agent_end` and all `before_compaction` call sites, leaves the already-improved `before_reset` behavior compatible, adds focused hook payload tests, removes unrelated formatting churn, and explicitly decides whether transcript-backed `before_compaction.messages` requires the same conversation-access gate as `agent_end`.

## Risks / Open Questions

- Security review: the PR changes runtime hook payload content rather than supply-chain files, but adding transcript-derived `messages` to `before_compaction` may expose raw conversation history through a hook that is not currently conversation-access gated.
- The `agent_end` transcript read is fire-and-forget and should be reviewed for run-scoped payload correctness, especially if a later turn can append to the same transcript before the read resolves.
- The PR includes unrelated formatting churn in `src/media/store.ts`; that should be removed or separated before merge.
