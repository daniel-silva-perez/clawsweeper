## Summary

Keep this PR open. Current main still does not implement the PR's central behavior for the agent_end and before_compaction hook surfaces: agent_end is still fed from the in-memory snapshot, and multiple compaction paths still omit transcript-derived messages. before_reset already has transcript-reading behavior on main, so the PR is partly redundant there, but not obsolete overall. The provided discussion also links the concrete Honcho attribution bug and the closed provenance issue, which supports that this is a real plugin-author need rather than cleanup churn.

## Best Possible Solution

Keep this PR open or ask the author to refresh it against current main. The best path is a narrow implementation that shares the existing transcript JSONL parser, preserves provenance/custom fields for agent_end and all before_compaction call sites, keeps before_reset behavior compatible, adds focused tests for provenance in those hook payloads, and explicitly reviews whether before_compaction/before_reset message payloads need conversation-access gating before shipping.
