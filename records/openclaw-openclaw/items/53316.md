## Summary

Keep open. The PR addresses a real iMessage routing/privacy bug, but current main moved the live chat-target parser into a shared SDK-backed helper, so the branch’s old iMessage helper edit is stale and would not fix the current runtime path without being ported.

## What This Changes

The PR adds decimal-only validation before parsing iMessage `chat_id` values and adds regression tests for rejecting hex `chat_id` and allowlist false matches.

## Best Possible Solution

Land a narrow shared-parser fix in `src/channels/plugins/chat-target-prefixes.ts` that requires decimal-only `chat_id` values before parsing, throws a neutral invalid-`chat_id` error for send targets, silently rejects invalid allowlist `chat_id` entries, covers iMessage plus BlueBubbles/shared-helper regressions, and adds a user-facing changelog fix with reporter/contributor credit.

## Review Findings

  - body: On current main, `extensions/imessage/src/target-parsing-helpers.ts` is only a re-export and the live iMessage and BlueBubbles paths call `src/channels/plugins/chat-target-prefixes.ts`. Use a neutral decimal-only error and optionally add the hex hint only for hex-looking input.
  - confidence: 0.83
