## Summary

Keep open. The underlying `parseInt` truncation bug is real and still present on current `main` and the latest release, but this PR is not merge-ready because it applies the guard to an iMessage helper body that is now a re-export while live parsing runs through the shared chat-target parser used by iMessage and BlueBubbles.

## What This Changes

The PR adds decimal-only guards for iMessage `chat_id` parsing and iMessage regression tests for hex target rejection plus allowlist false-match prevention.

## Best Possible Solution

Land a narrow shared-parser fix in `src/channels/plugins/chat-target-prefixes.ts` that requires decimal-only `chat_id` values before parsing, throws a neutral invalid-`chat_id` error for send targets, silently rejects invalid allowlist `chat_id` entries, preserves `chat_guid` and `chat_identifier` behavior, adds shared/iMessage/BlueBubbles regression coverage, and includes a user-facing changelog fix with reporter and contributor credit.
