## Summary

Keep open. Current main and the latest tagged release still parse explicit chat_id values with parseInt in the shared chat-target helper, and the PR branch now targets an old iMessage helper body that is only a re-export on main. The fix remains useful, but it needs to be ported to the shared parser with iMessage and BlueBubbles/shared coverage before merge.

## What This Changes

The PR branch adds /^\d+$/ guards to the old iMessage target parsing helper and adds iMessage tests for hex chat_id rejection and allowlist false-match regression.

## Best Possible Solution

Land a narrow shared-parser fix in src/channels/plugins/chat-target-prefixes.ts that requires decimal-only chat_id values before parsing, rejects invalid send targets with a neutral invalid-chat_id error, silently rejects invalid allowlist chat_id entries, adds iMessage plus BlueBubbles or shared-helper regression coverage for the misrouting and allowlist cases, and records a user-facing changelog fix with proper contributor/reporter credit.
