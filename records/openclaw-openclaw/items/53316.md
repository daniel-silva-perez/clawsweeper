## Summary

Keep open. The reported routing bug is still present on current main in the shared chat target parser, while this PR changes the old iMessage helper body that has since become a re-export, so the branch is useful source work but not mergeable as the live fix.

## What This Changes

The PR adds decimal-only guards before parsing iMessage chat_id values in the old iMessage helper and adds iMessage regression tests for hex chat_id rejection plus allowlist false-match prevention.

## Best Possible Solution

Land a narrow shared-parser fix in src/channels/plugins/chat-target-prefixes.ts that requires decimal-only chat_id values before parsing, throws a neutral invalid-chat_id error for send targets, prevents invalid allowlist chat_id entries from matching numeric chat IDs, preserves chat_guid/chat_identifier behavior, and credits the reporter/contributor in the changelog.
