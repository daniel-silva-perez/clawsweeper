## Summary

Keep open. Current main has adjacent public subagent hooks (`subagent_spawning`, `subagent_delivery_target`, `subagent_spawned`, `subagent_ended`) and internal delivery-result tracking, but it still does not expose the requested post-subagent-completion/pre-announcement plugin hook that can inspect child output/history, observe delivery result, or enrich/suppress the parent announcement. The issue discussion also explicitly notes that #35080 and #47433 fixed adjacent delivery symptoms without covering this extension point, and a recent maintainer review comment kept the item open for the same remaining gap.

## Best Possible Solution

Keep this open until OpenClaw either adds a public typed plugin hook at the post-subagent-completion/pre-announcement point or folds this exact scope into a canonical lifecycle/delivery design. The contract should expose bounded child/session/run metadata, approved completion output or child-history access, timing/usage where available, delivery status/error visibility where appropriate, and explicit return semantics for announcement enrichment, suppression, or filtering, with public docs and ordering regression tests.

## Risks / Open Questions

- Adding this hook is a public plugin API change with privacy and trust-boundary implications if child transcript/history, completion output, usage, or delivery errors are exposed to plugins.
- Announcement enrichment/suppression needs explicit ordering, failure, idempotency, and permission semantics so plugins cannot silently drop important completion delivery without an intentional contract.
- There is overlap with broader lifecycle/delivery design work such as #34938 and #66638, so a narrow hook should not lock in a contract that conflicts with the larger direction.
