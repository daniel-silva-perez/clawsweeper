## Summary

Keep this issue open. Current main still has subagent spawning, delivery-target, spawned, and ended hooks, plus generic agent/model hooks, but it does not provide the requested post-subagent-completion/pre-announcement plugin hook that can inspect child output/history, observe delivery status, or modify/suppress the completion announcement. The issue discussion also calls out that adjacent delivery fixes improved symptoms without covering this extension point.

## Best Possible Solution

Keep this open until OpenClaw adds a public typed plugin hook at the post-subagent-completion/pre-announcement point, or explicitly folds this exact scope into a canonical lifecycle/delivery design. The contract should expose bounded child/session/run metadata, completion output or approved child-history access, timing/usage where available, delivery status/error visibility where appropriate, and return semantics for announcement enrichment, suppression, or filtering, with public docs and ordering regression tests.

## Risks / Open Questions

- Adding this hook would expose child completion output or history to plugins; the contract needs explicit redaction, permission, and cross-session privacy rules.
- Return semantics that enrich, suppress, or filter completion announcements touch delivery idempotency, retries, channel routing, and existing `subagent_delivery_target` ordering.
- Related lifecycle/delivery work such as #66638 may overlap, so the design should avoid creating a second incompatible lifecycle notification path.
