## Summary

Keep open. Current main still has adjacent subagent spawning, delivery-target, spawned, and ended hooks, plus message/delivery hooks, but it does not expose the requested post-subagent-completion/pre-announcement plugin hook that can inspect child output/history, observe delivery result, or enrich/suppress the completion announcement. The issue discussion explicitly notes that delivery fixes like #35080 improved symptoms without covering this extension point, and a maintainer review comment already reached the same keep-open conclusion.

## Best Possible Solution

Keep this open until OpenClaw either adds a public typed plugin hook at the post-subagent-completion/pre-announcement point or folds this exact scope into a canonical lifecycle/delivery design. The contract should expose bounded child/session/run metadata, approved completion output or child-history access, timing/usage where available, delivery status/error visibility where appropriate, and explicit return semantics for announcement enrichment, suppression, or filtering, with public docs and ordering regression tests.
