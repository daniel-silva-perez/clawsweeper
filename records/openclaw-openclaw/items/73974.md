## Summary

Keep open. Current main still lacks the macOS OpenClaw.app LaunchAgent ownership guard that this PR targets, and the PR is tied to the still-open #67631 report with closing syntax. The PR is not obsolete, but it should not merge as-is because review identified a real `OPENCLAW_PROFILE=mac` label-collision path that can make a CLI-only profile LaunchAgent look app-owned.

## Best Possible Solution

Keep this PR open and revise it in place. The fix should disambiguate real OpenClaw.app ownership from a CLI `OPENCLAW_PROFILE=mac` LaunchAgent, likely by checking plist `ProgramArguments` or another app-specific marker before skipping CLI repair. Preserve the intended default-profile skip, add explicit regression tests for the `mac` profile collision and unrelated profiles, keep the #67631/#42573 attribution, and run `pnpm check:changed` before maintainer review.
