## Summary

Keep this PR open. Current main still lacks the coordinated compaction/reset invariant fix: zero-token compaction results are still filtered out in core accounting paths, reset hooks still omit provider context, subscribe-time compaction hooks still omit provider plumbing, and the shared provider resolver/regression tests from the PR are absent. The PR is also the active same-author implementation candidate for the open paired issue #69269, with maintainer comments preserving this branch as canonical.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or replace it with an equivalent canonical fix that adds shared hook provider resolution, preserves explicit/internal provider identities, carries stable `messageProvider` through compaction/reset hooks, and refreshes token/cost accounting for zero-token compaction outcomes with regression coverage. Close #69269 and the token-accounting duplicates only after the chosen fix merges.
