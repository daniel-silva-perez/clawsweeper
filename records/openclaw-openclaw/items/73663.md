## Summary

Keep open. Current main still lacks the Amazon Bedrock Opus 4.7 temperature omission and still does not classify the reported Bedrock ValidationException with nested Anthropic invalid_request_error as a failover-worthy schema rejection. The related Bedrock PR #73689 is an implementation candidate, but it is not merged in the inspected main tree.

## Best Possible Solution

Keep this issue open. Review and land #73689 or an equivalent provider-owned fix that strips `inferenceConfig.temperature` for `amazon-bedrock` Opus 4.7 model IDs across the direct and application-inference-profile payload paths, with regression tests. Separately, add Bedrock failover classification for `ValidationException` messages containing nested Anthropic `invalid_request_error` such as deprecated temperature, so the error surfaces clearly and advances the configured fallback chain.

## Risks / Open Questions

- Closing now would leave Bedrock Opus 4.7 sessions vulnerable to the same rejected `temperature` request shape when temperature is configured.

- The temperature omission should be scoped to the Bedrock Opus 4.7 family, or represented in provider/model capability metadata, so Opus 4.6 and earlier behavior is preserved.
