## Summary

Keep this issue open. Current main still has generic directory/name-resolution plumbing, but the bundled WhatsApp plugin only exposes config-backed allowlist peers and does not subscribe to or persist Baileys contact-list events for saved phone contacts. The related PRs in the provided context are closed unmerged and do not supersede the requested WhatsApp contact directory behavior.

## Best Possible Solution

Implement this in the bundled WhatsApp plugin: subscribe to Baileys contacts.upsert/contacts.update where the active socket is managed, persist account-scoped contacts locally behind an explicit privacy/config switch, expose them through directory.listPeers as ChannelDirectoryEntry rows with stable id/name/handle/raw metadata, and route outbound name resolution through unique directory matches while preserving allowFrom/outbound policy checks. Add focused tests for ingestion, query filtering, ambiguity, account isolation, persistence/logout cleanup, and docs for https://docs.openclaw.ai/cli/directory and https://docs.openclaw.ai/channels/whatsapp.
