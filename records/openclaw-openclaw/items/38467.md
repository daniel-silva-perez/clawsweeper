## Summary

Keep this issue open. Current main still does not persist Baileys contact-list events, expose WhatsApp phone contacts through `openclaw directory peers list --channel whatsapp`, or resolve outbound WhatsApp targets by saved contact name. The existing plugin/API surfaces are sufficient for this to be implemented in the bundled WhatsApp plugin, but the implementation is not present. The earlier ClawSweeper comment and PR #33924 context remain accurate: that PR was closed unmerged and addressed outbound allowlist policy, not contact ingestion.

## Best Possible Solution

Implement this in the bundled WhatsApp plugin: subscribe to Baileys contact-list events, persist account-scoped contacts locally behind an explicit privacy/config decision, expose those contacts through the WhatsApp directory with stable id/name/handle/raw metadata, wire outbound message-tool resolution through unique contact-name matches while preserving allowlist checks, and add tests/docs for ingestion, query filtering, ambiguity, account isolation, persistence cleanup, and privacy behavior.
