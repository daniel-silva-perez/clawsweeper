## Summary

Keep this PR open for author or maintainer revision. The requested diagnostic improvement is still not implemented on current main, but the proposed patch has two concrete merge blockers: it leaves dead code in the validator and gives `edit` a payload example that current main rejects.

## What This Changes

The PR appends hard-coded read/write/edit parameter examples to missing-parameter validation errors and threads the tool name through `assertRequiredParams`.

## Best Possible Solution

Revise this PR or replace it with a focused update that matches current `REQUIRED_PARAM_GROUPS`, uses a valid `edit({"path":..., "edits":[...]})` example, removes dead code, and adds regression tests for empty-object and missing-field validation messages.

## Reproduction Assessment

Yes for the validation-message surface: current main can be checked through `assertRequiredParams({}, REQUIRED_PARAM_GROUPS.read, "read")` or the existing empty-record test, which shows no example is emitted. Live Kimi self-correction speed is not verified by this PR.

## Solution Assessment

No, not as currently written. The direction is narrow and plausible, but the patch should be rebased to the current canonical tool schema and covered by tests before merge.
