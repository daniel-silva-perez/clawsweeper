## Summary

Keep open. Current main has partially grouped CI execution, but it still delegates to many individual boundary/check scripts rather than the requested unified, config-driven architecture-check framework. The provided related items are title-search noise and do not supersede this tooling refactor request.

## Best Possible Solution

Keep this issue open as the canonical cleanup request unless maintainers choose to split it. A good implementation would build on the existing shared helpers in `scripts/lib/guard-inventory-utils.mjs`, `scripts/lib/ts-guard-utils.mjs`, and `scripts/lib/callsite-guard.mjs`, migrate a meaningful first batch of checks into a documented unified runner/config, update CI/package scripts, and leave compatibility wrappers or a clear deprecation path for old guard entrypoints.
