## Summary

Keep this issue open. Current main still rejects a root-level `paperclip` field on Gateway `agent` params: the schema remains closed, the server validates before dispatch, and the shipped Paperclip-related change only relaxed `wake` payloads. The related open PR #69139 is still the implementation candidate rather than proof that the bug is fixed.

## Best Possible Solution

Keep this issue open as the canonical tracker. Maintainers should review or replace #69139 with an additive Gateway protocol fix, likely accepting documented opaque metadata on `agent` params or coordinating Paperclip to send its context through a supported field, then add regression coverage and generated/client updates as needed.
