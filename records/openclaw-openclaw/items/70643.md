## Summary

Keep this issue open. Current main still lacks an advisory lock around OpenClaw-owned config read/merge/write operations, and the related lock PR was closed unmerged. The code has temp-file rename and validation/recovery safeguards, but those do not linearize concurrent config mutations.

## Best Possible Solution

Keep open and land a focused core config fix that serializes OpenClaw-owned `openclaw.json` mutations with the existing re-entrant file-lock helper across the full read/merge/write sequence, with regression coverage for concurrent writers and no broad config refactor.
