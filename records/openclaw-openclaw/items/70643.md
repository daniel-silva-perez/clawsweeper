## Summary

Keep this issue open. Current main still has OpenClaw-owned config writes protected by validation, temp-file write, rename/copy fallback, backups, audit, and last-known-good recovery, but I did not find a lock spanning the read/merge/mutate/write cycle for openclaw.json. The related lock PR is closed unmerged, and the closed auto-restore issue #71540 addresses a different plugin allowlist recovery path.

## Best Possible Solution

Keep this issue open and land a focused core config fix that serializes OpenClaw-owned openclaw.json mutations across snapshot read, merge/mutate, validation, persistence, audit, and runtime refresh. Reuse the existing file-lock helper, handle the include write-through path deliberately, and add a regression test for concurrent config writers without changing config schema or recovery semantics.
