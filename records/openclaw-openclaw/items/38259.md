## Summary

Keep this issue open. Current main still parses dotenv files with `dotenv.parse` and assigns accepted entries directly to `process.env`, so `.env` values such as `${OPENAI_API_KEY}` or `$OPENAI_API_KEY` remain literal. The linked PR #38254 is useful prior art but was closed unmerged, and its review thread highlights expansion edge cases that a future fix must handle.

## Best Possible Solution

Keep this issue open and implement dotenv-level expansion in `src/infra/dotenv.ts` after key normalization and workspace/global filtering but before assignment to `process.env`. Support `${VAR}` and `$VAR` using process env plus earlier accepted dotenv entries, preserve existing precedence and conflict behavior, define missing/cyclic/escaped-dollar semantics, and add focused regression coverage in `src/infra/dotenv.test.ts`. Update `docs/help/environment.md` and add a changelog entry if the behavior change is user-facing.
