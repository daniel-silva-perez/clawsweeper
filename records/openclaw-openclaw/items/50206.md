## Summary

Keep this PR open. Current main still does not implement opt-in per-agent outbound LLM request headers, and this PR remains a plausible implementation candidate for the linked feature request rather than obsolete work. The review concerns are actionable update items: resolve provider config through normalized provider IDs and document the trusted-proxy privacy implications of sending agent IDs upstream.

## Best Possible Solution

Keep the PR open and request a focused update: resolve provider config through the existing normalized-provider lookup pattern, keep the option explicitly opt-in, add regression coverage for enabled/disabled/missing-agent and alias-normalized provider IDs, and document the privacy/trusted-proxy implication. If maintainers prefer this outside core, first extend the provider/plugin hook context to expose agentId so a plugin can implement the header cleanly.

- Closing this PR now would drop the only in-repo implementation candidate shown for the linked per-agent LLM request-header feature while current main still lacks the behavior.
