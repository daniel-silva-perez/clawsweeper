## Summary

Keep this issue open. The report is recent, detailed, and still maps to current core/plugin-runtime surfaces on main: raw timed subprocess results remain plugin-visible, the safer normalized command helper is opt-in, and plugin CLI command loading can still broaden when command ownership metadata is missing. The exact recipes/kitchen behavior is external to this checkout, but the generic OpenClaw surfaces are not proven fixed.

## Best Possible Solution

Keep the issue open for maintainer investigation with the reported external plugins installed on macOS. Profile `openclaw recipes list` and `recipes workflows worker-tick`, verify the external plugins declare `activation.onCommands` and lightweight CLI metadata, and harden plugin-facing subprocess use so timeout or unparseable JSON cannot be cached as success. Add launchd jitter or deployment guidance only after the baseline single-command startup cost is understood.
