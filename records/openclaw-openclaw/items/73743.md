## Summary

Keep open. Current main still contains the same plugin CLI startup and plugin-facing subprocess timeout surfaces implicated by the report, while the exact recipes/Kitchen implementation is external to this checkout and has not been proven fixed.

## What This Changes

The issue reports slow macOS `openclaw` subprocess startup that compounds under launchd workflow fan-out and can poison an external Kitchen subprocess cache when timed-out partial output is treated as success.

## Best Possible Solution

Profile the external recipes/Kitchen setup on macOS, then make the narrowest fix that reduces plugin CLI startup for the owned command path and prevents timed-out partial subprocess output from being cached as success. Launchd jitter or deployment guidance can follow as mitigation once the single-call startup cost and timeout contract are understood.
