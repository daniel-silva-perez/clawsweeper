## Summary

Keep open. Current main still has the same relevant core shape as the earlier ClawSweeper review: useful plugin CLI laziness and timeout metadata exist, but they do not prove the reported macOS recipes CLI startup/fan-out regression is fixed, and the named recipes and ClawKitchen implementations are not present in this repository.

## What This Changes

The issue requests investigation and fixes for slow external plugin CLI startup under macOS launchd fan-out, plus safer subprocess timeout handling so partial output cannot poison ClawKitchen's recipe cache.

## Best Possible Solution

Treat this as an active performance/reliability investigation: profile `openclaw recipes list` and `recipes workflows worker-tick` on macOS under idle and fan-out conditions, move recipes CLI paths onto descriptor-backed or gateway-backed lightweight execution where feasible, and make timeout results impossible for plugin callers to cache as successful partial output.

Yes. The issue provides a concrete macOS launchd reproduction path with idle and fan-out `time openclaw recipes list` comparisons plus cache observations; current source inspection confirms the relevant plugin CLI and subprocess timeout surfaces still exist, though this checkout cannot remeasure macOS timing.

## Solution Assessment

Unclear as a single fix. The suggested investigation areas are directionally sound, but the maintainable solution likely splits between core plugin CLI startup/timeout semantics and the external recipes/ClawKitchen cache handling.
