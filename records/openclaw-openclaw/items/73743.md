## Summary

Keep open. The report is recent, detailed, and still maps to active core/plugin-runtime surfaces on current main: the recipes/kitchen implementation is not bundled in this checkout, plugin-facing `runtime.system.runCommandWithTimeout` still exposes the raw timeout result shape, and plugin CLI startup still depends on manifest command/activation metadata for narrowing. A newer CLI startup fix after the prior bot review only skips plugin bootstrap for JSON `openclaw agent` gateway calls, not for external `openclaw recipes ...` commands or the reported ClawKitchen cache poisoning path.

## Best Possible Solution

Keep this issue open. Maintainers should profile `openclaw recipes list` and `recipes workflows worker-tick` on macOS with the reported external plugin set, verify the recipes plugin declares command ownership/activation and lightweight CLI metadata, and harden the plugin-facing subprocess API or documented helper path so timeouts and invalid JSON cannot be cached as successful output. Treat launchd jitter as a secondary mitigation after single-command startup cost and timeout semantics are understood.
