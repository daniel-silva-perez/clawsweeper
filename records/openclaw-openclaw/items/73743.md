## Summary

Keep open. Current main has useful timeout metadata and plugin CLI laziness infrastructure, but it does not prove the reported macOS `recipes` CLI startup/fan-out regression is fixed; the named recipes and ClawKitchen implementations are absent from this repository, and unknown plugin CLI commands still go through plugin command registration/loading paths before execution.

## What This Changes

Investigate and reduce slow external plugin CLI startup and launchd fan-out on macOS, and prevent subprocess timeouts from poisoning ClawKitchen's recipe cache.

## Best Possible Solution

Treat this as an active performance/reliability investigation: profile `openclaw recipes list` and `recipes workflows worker-tick` on macOS under idle and fan-out conditions, move the recipes CLI paths onto descriptor-backed or gateway-backed lightweight execution where feasible, and make timeout results impossible for plugin callers to cache as successful partial output.

## Reproduction Assessment

Yes. The report gives a concrete macOS launchd reproduction path with idle and fan-out `time openclaw recipes list` comparisons plus cache observations; source inspection confirms the relevant plugin CLI and subprocess timeout surfaces still exist, although this checkout cannot remeasure macOS timing.

## Solution Assessment

Unclear as a single fix. The requested investigation areas are directionally sound, but the maintainable solution likely splits between core plugin CLI startup/timeout semantics and external recipes/ClawKitchen cache handling.
