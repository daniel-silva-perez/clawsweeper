## Summary

Keep open. The report is recent, detailed, and still maps to current main: the exact recipes/Kitchen plugins are not bundled here, while OpenClaw still has the plugin CLI startup and raw subprocess timeout surfaces implicated by the failure.

## What This Changes

The issue reports a macOS launchd fan-out regression where external `recipes` CLI commands take about 25 seconds cold, compound under parallel worker ticks, and can poison Kitchen subprocess caching after timeouts.

## Best Possible Solution

Reproduce on macOS with the external recipes/Kitchen plugins installed, profile the external `recipes` command startup path, ensure the plugin declares narrow command ownership or activation metadata, and harden plugin-facing subprocess timeout/JSON handling so partial output is never cached as success. Launchd jitter should be treated as a mitigation after the single-call startup cost and timeout semantics are understood.

## Solution Assessment
