## Summary

Keep this PR open for maintainer review. Current main still lacks the realtime consult override path for both Voice Call and Control UI Talk, and the PR is a focused implementation candidate with positive bot review plus reported production verification, though it has a small docs-defaults issue and still needs an appropriate changed gate before merge.

## What This Changes

The PR adds optional realtime consult thinking-level and fast-mode overrides to Voice Call and Control UI Talk config/schema/docs, threads them into consult agent runs and one-shot chat.send options, and adds focused tests.

## Best Possible Solution

Land or adapt this PR after resolving the docs-default wording, confirming one-shot fast-mode policy, and getting a green relevant changed gate. Keep #71849 open for the separate fast memory/context retrieval design.

## Reproduction Assessment

Yes. For the missing capability, source inspection on current main is enough: the strict schemas, Talk session result, chat.send schema, Voice Call runtime, and shared consult runtime do not carry consultThinkingLevel or consultFastMode today.

## Solution Assessment

Yes for this narrower override feature: additive config plus one-shot runtime handoff is a maintainable path that preserves existing unset behavior. It should not be treated as the full solution for the broader live-call fast context issue tracked separately.
