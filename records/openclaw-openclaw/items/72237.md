## Summary

Keep this PR open, but do not merge it as-is. Current main still lacks the realtime consult thinking-level and fast-mode override path, and the PR remains the active implementation candidate with useful Greptile review and production-test context, but the branch currently has two concrete merge blockers: Talk consult thinking overrides would still persist through the existing `/think` directive path, and the config docs baseline hash update was removed after adding new config surface.

## What This Changes

The PR adds optional Voice Call realtime and Control UI Talk consult thinking/fast-mode config fields, schema/docs/protocol/UI handoff, runtime propagation, and focused tests for realtime `openclaw_agent_consult` overrides.

## Best Possible Solution

Revise and land or adapt this PR so Voice Call and Control UI Talk realtime consults can pass nonpersistent thinking-level and fast-mode overrides into the underlying agent run, keep unset behavior unchanged, update generated/config-doc baselines consistently, and leave #71849 as the separate fast memory/context retrieval follow-up.

## Reproduction Assessment

Yes for the review findings by static path: a configured Talk consult sends `thinking` through `chat.send`, current `chat.send` converts that to `/think`, and directive persistence writes `sessionEntry.thinkingLevel`. The config-baseline issue is also traceable from the PR patch and the `config:docs:check` hash comparison code, although tests were not run because this review was read-only.

## Solution Assessment

No, not as currently proposed: the Voice Call side is a narrow additive fit, but the Talk side should use a truly one-shot override path rather than the existing persistent `thinking` command carrier. The branch also needs the config baseline hash regenerated after the schema/help additions.
