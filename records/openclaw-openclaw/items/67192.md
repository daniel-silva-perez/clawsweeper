## Summary

Keep open. Current main still treats any truthy streamed `delta.content` as text without a runtime string guard and only parses fallback reasoning fields when they are strings, so the reported Mistral array/object reasoning stream case is not fixed. The linked implementation PR #67203 is still open and references this issue with closing syntax, so this should remain the canonical report until that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw to normalize Mistral reasoning stream blocks for `/think` so non-string `reasoning_content` or content chunks become standard thinking/text deltas instead of `[object Object]` channel output.

## Best Possible Solution

Land or replace #67203 with a focused fix in `src/agents/openai-transport-stream.ts` that accepts only string `delta.content` as text, normalizes Mistral reasoning/content block arrays or objects into thinking/text deltas, and adds regression tests for those stream chunk shapes.
