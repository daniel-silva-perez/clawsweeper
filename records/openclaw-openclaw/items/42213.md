## Summary

Current main still matches the reported failure mode. Control UI chat busy state is latched by `chatRunId`, normal follow-up messages enqueue while that latch is set, queued sends do not flush while busy, and terminal chat events are still the normal path that clears the latch. I found no Control UI `agent.wait` watchdog/recovery path for a stale active chat run. The same-author implementation candidate #42258 remains open and unmerged with closing syntax for this issue, so the maintainer path is to review, land, or close that PR rather than closing this issue now.

## Best Possible Solution

Keep this issue open and review #42258 as the implementation candidate. Land that PR or an equivalent Control UI recovery that probes `agent.wait` for the active `chatRunId` after inactivity, clears busy state only on terminal wait results, and includes regression coverage for stale-finished recovery, still-active timeout behavior, watchdog reset/cancellation, and queued-message flushing.
