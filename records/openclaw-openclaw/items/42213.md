## Summary

Keep this issue open. Current main still uses `chatRunId` as the Control UI busy latch, normal follow-up messages still queue while that latch is set, and I did not find a Control UI `agent.wait` watchdog/recovery path. The same-author implementation candidate #42258 is open and unmerged, so this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review #42258 as the implementation candidate. Land that PR or an equivalent Control UI recovery that probes `agent.wait` for the active `chatRunId` after inactivity, clears busy state only on terminal wait results, and includes regression coverage for stale-finished recovery, still-active timeout behavior, watchdog reset/cancellation, and queued-message flushing.
