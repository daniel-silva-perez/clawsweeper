## Summary

Keep this issue open. Current main still has the reported stale-busy failure mode: Control UI busy state is latched by `chatRunId`, normal follow-up messages queue while that latch is set, queue flushing exits while busy, and the UI still has no `agent.wait` watchdog/recovery path for a missed terminal chat event. The same-author implementation candidate #42258 is open and unmerged with closing syntax for this issue, so the maintainer path is to review and land or close that PR.

## Best Possible Solution

Keep this issue open and review #42258 as the implementation candidate. Land that PR or an equivalent Control UI recovery that probes `agent.wait` for the active `chatRunId` after inactivity, clears busy state only on terminal wait results, and includes regression coverage for stale-finished recovery, still-active timeout behavior, watchdog reset/cancellation, and queued-message flushing.
