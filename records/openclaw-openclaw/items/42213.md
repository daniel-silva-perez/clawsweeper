## Summary

Keep this issue open. Current main still lacks a Control UI `agent.wait` watchdog/recovery path for a missed terminal chat event: `chatRunId` remains the busy latch, normal follow-ups enqueue while that latch is set, and queue flushing returns while busy. The supplied GitHub context also shows same-author PR #42258 is still open and unmerged with closing syntax for this issue, so the maintainer path is to review/land or close that PR rather than closing this issue now.

## Best Possible Solution

Keep this issue open and review #42258, or land an equivalent Control UI fix that probes `agent.wait` for the active `chatRunId` after inactivity, clears busy state only on terminal wait results, leaves genuinely active runs untouched, and adds regression coverage for stale-finished recovery, still-active timeout behavior, watchdog cancellation/reset, and queued-message flushing.
