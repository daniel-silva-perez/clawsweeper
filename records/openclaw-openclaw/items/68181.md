## Summary

Keep this issue open. Current main still matches the reported macOS screen.snapshot behavior, and the same author has open unmerged PR #68186 using closing syntax as the implementation candidate, so this issue should close only after that PR or an equivalent fix lands.

## What This Changes

The issue requests that macOS screen.snapshot reject malformed params and prevent oversized base64 JSON responses before they exceed the gateway payload budget.

## Best Possible Solution

Review and land #68186, or apply an equivalent macOS runtime fix that separates omitted params from malformed params, returns INVALID_REQUEST before capture on malformed JSON, enforces the gateway payload budget before response emission, and adds focused MacNodeRuntimeTests coverage.

## Reproduction Assessment

Yes. The issue has a high-confidence source-level reproduction: call MacNodeRuntime.handleInvoke with malformed screen.snapshot params or with an injected large snapshot result; current main still contains the exact fallback and unguarded base64 emission paths.

## Solution Assessment

Yes, the paired PR direction is the narrowest maintainable fix path: keep valid in-bounds snapshot behavior unchanged, add fail-closed request validation and payload guards in the macOS runtime, and cover the behavior with targeted Swift tests.
