## Summary

Current main still does not implement the PR’s intended Discord log enrichment. The active Discord no-mention guild-drop log only includes `channelId` and `reason`, while `params.accountId` and `params.data.guild_id` are already available nearby. The PR remains useful but is path-stale because Discord moved from `src/discord/...` to `extensions/discord/...`. Security review found no supply-chain-sensitive changes in the provided one-file, two-line logging diff.

## Best Possible Solution

Keep this PR open as a valid but path-stale logging fix, or reapply the same additive change on current main in `extensions/discord/src/monitor/message-handler.preflight.ts` by adding `accountId: params.accountId` and `guildId: params.data.guild_id` to the no-mention `logger.info` payload, ideally with a focused test assertion for that structured log output.
