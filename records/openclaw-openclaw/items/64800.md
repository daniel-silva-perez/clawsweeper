## Summary

Keep open. The underlying bug is still present on current main, and the PR contains the right narrow idea plus the previously requested filtering/dedup safeguards, but the active implementation moved from `provider-runtime.ts` to `provider-hook-runtime.ts` after the PR branch was cut, so it should not merge as-is.

## What This Changes

The PR changes provider stream wrapping to compose the primary provider wrapper with matching `hookAliases` provider wrappers and adds a regression test for chaining while skipping unrelated providers.

## Best Possible Solution

Implement wrapper composition in `src/plugins/provider-hook-runtime.ts`: apply the primary matched provider first, then other matching hook providers, dedupe by stable provider identity, and keep regression coverage in `src/plugins/provider-runtime.test.ts`.
