## Summary

Keep this PR open. Current main still calls only one provider `wrapStreamFn`, so the reported hookAlias composition bug remains valid; the submitted patch is useful source work but needs to be ported from the older `provider-runtime.ts` implementation into the current `provider-hook-runtime.ts` surface.

## What This Changes

The PR changes provider stream wrapping to compose the primary provider wrapper with matching hookAlias provider wrappers and adds a regression test for primary-plus-hookAlias chaining while skipping unrelated providers.

## Best Possible Solution

Port the wrapper composition into `src/plugins/provider-hook-runtime.ts`, using the primary matched provider first and then other matching hook providers, with stable dedupe and explicit regression coverage for the chaining and filtering behavior.
