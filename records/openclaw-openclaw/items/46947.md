## Summary

Keep open. Current main and the latest stable release still leave sidebar markdown images with the loose `max-width: 100%` / `max-height: 420px` rule, while this PR is a narrow CSS-only implementation that matches the existing main-chat cap and has no discrete patch defect. The remaining action is maintainer UI confirmation, fresh current CI, and merge judgment, especially because the linked regression remains open.

## What This Changes

This PR changes `ui/src/styles/chat/sidebar.css` so sidebar markdown inline images use `max-width: min(100%, 420px)` and `max-height: 320px` instead of filling the sidebar up to 420px high.

## Best Possible Solution

Land this sidebar CSS cap, or an equivalent shared markdown image sizing rule, after a quick visual check with image-heavy markdown in the Control UI and fresh CI; then let the linked regression close only if maintainers agree the sidebar path covers the remaining observable bug.

## Reproduction Assessment

Yes for the sidebar path: render a base64 markdown image inside a wide `.sidebar-markdown` container on current main, and the image can fill the sidebar up to the current 420px height rule. The broader extended-session main-chat behavior from the linked bug is only partially reproduced by source inspection, not by a live browser run in this read-only pass.

## Solution Assessment

Yes. The proposed CSS-only cap is the narrowest maintainable fix for the known sidebar inconsistency because it aligns sidebar markdown images with the already-capped main chat rule without changing rendering or sanitizer behavior.
