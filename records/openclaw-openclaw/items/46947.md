## Summary

Keep open. Current main still leaves sidebar markdown images capped only by the sidebar width and a 420px height, while this PR applies the same bounded image sizing already used in the main chat area; no cleanup close reason applies because the linked regression remains open and the patch has not landed.

## What This Changes

This PR updates `ui/src/styles/chat/sidebar.css` so sidebar markdown images use `max-width: min(100%, 420px)` and `max-height: 320px`, matching the main chat image sizing rule.

## Best Possible Solution

Land this CSS cap or an equivalent shared markdown image sizing rule after maintainer UI confirmation and fresh validation, then allow the linked regression to close through the PR if the sidebar surface fully covers it.

## Reproduction Assessment

Yes. The source path is clear: the sidebar renders sanitized markdown inside `.sidebar-markdown`, base64 markdown images become `.markdown-inline-image`, and current main lets those images fill the sidebar width up to 420px high.

## Solution Assessment

Yes. The proposed CSS-only cap is the narrowest maintainable fix for the observed sidebar inconsistency; deduplicating the chat/sidebar image rule could be a later cleanup but is not required for this fix.
