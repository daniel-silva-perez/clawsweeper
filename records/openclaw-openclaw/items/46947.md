## Summary

Keep open. Current main still has the loose sidebar markdown image sizing, while this PR is a narrow CSS-only implementation that matches the existing main chat image cap and is paired with an open regression report.

## What This Changes

This PR updates `ui/src/styles/chat/sidebar.css` so `.sidebar-markdown .markdown-inline-image` uses `max-width: min(100%, 420px)` and `max-height: 320px`.

## Best Possible Solution

Land this CSS cap or an equivalent shared markdown image sizing rule after maintainer UI confirmation and validation, then let the paired regression close through the merged PR if the sidebar surface fully covers it.

## Reproduction Assessment

Yes. The source path is clear: base64 markdown images become `.markdown-inline-image`, the sidebar renders them inside `.sidebar-markdown`, and current main lets them fill the sidebar width up to 420px high.

## Solution Assessment

Yes. The proposed CSS-only cap is the narrowest maintainable fix for the observed sidebar inconsistency; deduplicating the chat/sidebar image rule could be a later cleanup but is not required for this fix.
