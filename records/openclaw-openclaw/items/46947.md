## Summary

Keep open. Current main and the latest release still have the looser sidebar markdown image sizing, while the main chat markdown image rule already uses the cap proposed by this PR. The patch is a narrow CSS-only change that maps to the sidebar renderer and has no blocking review finding; it needs maintainer review and fresh validation rather than closure.

## What This Changes

This PR changes `.sidebar-markdown .markdown-inline-image` in `ui/src/styles/chat/sidebar.css` from full-width/420px sizing to a 420px width cap and 320px height cap.

## Best Possible Solution

Land this CSS cap, or an equivalent shared markdown image sizing rule, after confirming the linked regression reproduces on the sidebar markdown path; then #45846 can close through the PR's linked fix path.

## Reproduction Assessment

Yes. A source-level/manual reproduction path is clear: render a data URI markdown image in the markdown sidebar on a wide sidebar, where current main allows `100%` width and `420px` height while the main chat surface is capped at `420px` / `320px`.

## Solution Assessment

Yes. The proposed CSS change is the narrowest maintainable fix for the observed sidebar inconsistency; a shared markdown image sizing rule could reduce duplication later, but it is not required to solve this PR's target behavior.
