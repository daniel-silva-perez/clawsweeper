Keep open. Current main still lacks the sidebar image cap and the linked regression issue remains open; the PR is a narrow CSS-only implementation with no blocking review finding, so it needs maintainer review and fresh validation rather than cleanup closure.

## What This Changes

The PR changes `.sidebar-markdown .markdown-inline-image` in `ui/src/styles/chat/sidebar.css` to cap width at `min(100%, 420px)` and height at `320px`.

## Best Possible Solution

The desired end state is a constrained sidebar markdown image rule, or an equivalent shared markdown image sizing rule if maintainers prefer deduplication, verified against the linked sidebar/webchat regression path.

## Reproduction Assessment

Yes. A high-confidence source-level/manual path exists: render a base64 data URI markdown image in the markdown sidebar on a wide sidebar; current main emits `markdown-inline-image` and the sidebar CSS allows `100%` width / `420px` height while chat text is capped.

## Solution Assessment

Yes. The proposed two-declaration CSS change is the narrowest maintainable fix for the observed sidebar inconsistency; a shared rule could reduce duplication later but is not required for this behavior.
