## Summary

Keep this PR open. Current main does not disable the pi-coding-agent auto-retry path, and the linked regression remains open. The dependency source supports the proposed direction because pi-coding-agent defaults agent-level retry on and removes the error assistant message before calling `agent.continue()`, while OpenClaw already has its own outer retry and failover loop. The branch still needs maintainer review before landing because the proposed setter may persist global Pi settings in the file-backed trusted-settings path.

## What This Changes

The PR disables pi-coding-agent retry in `createPreparedEmbeddedPiSettingsManager` and adds a Vitest assertion that the prepared manager reports retry disabled.

## Best Possible Solution

Land a narrow reviewed fix that disables pi-coding-agent auto-retry for OpenClaw embedded sessions without unexpectedly changing user/global Pi settings, while preserving OpenClaw's existing failover, auth rotation, thinking fallback, and empty-error retry behavior.
