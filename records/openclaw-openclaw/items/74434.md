## Summary

Keep this PR open. Current main still lacks the proposed SDK retry disablement, and the branch targets a plausible linked replay-loop regression, but the implementation needs revision because `setRetryEnabled(false)` can write through to file-backed Pi settings on trusted project-settings paths.

## What This Changes

The PR disables pi-coding-agent retry in `createPreparedEmbeddedPiSettingsManager` and adds a Vitest assertion that the prepared manager reports retry disabled.

## Best Possible Solution

Apply a session-scoped Pi retry override for OpenClaw embedded sessions without writing to file-backed Pi global settings, keep OpenClaw's outer retry/failover behavior intact, and add coverage for both default in-memory settings and the trusted project-settings path.

## Review Findings

Full review comments:

- **[P2] Avoid persisting the retry override to Pi settings:** `src/agents/pi-project-settings.ts:58`
  - body: `createEmbeddedPiSettingsManager()` can return a file-backed manager when project settings policy is `trusted` and no bundle settings are loaded. Upstream `setRetryEnabled(false)` writes `retry.enabled` into global settings, so this can permanently disable Pi auto-retry for that agent instead of applying a session-scoped OpenClaw override. Please avoid saving this override to file-backed settings and cover the trusted policy path.
