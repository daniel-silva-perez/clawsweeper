## Summary

Keep this PR open. The proposed change targets a plausible nested retry loop and current main does not disable pi-coding-agent auto-retry for embedded sessions, but the patch uses an upstream persistent settings setter, so it can permanently change Pi retry settings instead of applying a session-only OpenClaw runtime override. The overlapping #74503 work also needs maintainer choice before this narrow PR should land.

## What This Changes

The PR disables pi-coding-agent SDK auto-retry in `createPreparedEmbeddedPiSettingsManager` and adds a Vitest assertion that the prepared settings manager reports retry disabled.

## Best Possible Solution

Land a narrow embedded-session override that disables pi-coding-agent agent-level auto-retry without writing global or project Pi settings, applies at the effective runtime point even after resource reload, and adds coverage for the trusted file-backed path plus the existing embedded retry surface.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Keep retry disablement session-scoped:** `src/agents/pi-project-settings.ts:57`
  - body: Calling `setRetryEnabled(false)` here is not a pure runtime override. `createEmbeddedPiSettingsManager` can return a file-backed manager when project settings policy is `trusted` and no plugin settings are loaded, and upstream `setRetryEnabled()` writes `retry.enabled` into global settings and saves it. An embedded run can therefore permanently disable Pi retry for the agent instead of only disabling the inner retry for this OpenClaw session.
  - confidence: 0.88
