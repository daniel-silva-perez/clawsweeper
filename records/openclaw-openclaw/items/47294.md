Keep open. Current main still routes Feishu document writes through Feishu's Markdown conversion API without any equation-aware preservation path, and the issue provides a concrete formula that maps directly to the bundled Feishu document tool surface.

## What This Changes

Preserve LaTeX backslashes inside Feishu document `$$...$$` and `<equation>` formula content when creating, writing, appending, or inserting document content.

## Best Possible Solution

Implement this inside the Feishu plugin by preserving raw formula bodies before Markdown conversion and emitting Feishu's supported equation representation for block formulas and `<equation>` tags, with regression coverage for the reported formula and no core Markdown contract change unless another plugin needs the same seam.
