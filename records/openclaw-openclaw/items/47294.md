## Summary

Keep open. Current main still routes Feishu document write, append, and insert content through Feishu's Markdown conversion endpoint without an equation-aware preservation path, and the prior ClawSweeper comment correctly kept this as an actionable bundled-plugin bug.

## What This Changes

The issue requests preserving LaTeX backslashes inside Feishu document `$$...$$` formula blocks and `<equation>...</equation>` tags when writing document content.

## Best Possible Solution

Implement the fix inside the Feishu plugin by detecting raw formula blocks before Markdown conversion and emitting a supported equation representation that preserves LaTeX verbatim, with regression tests for the reported formula and a documented fallback only if Feishu's API cannot create the desired equation block.

## Reproduction Assessment

Yes. The issue includes a concrete complex formula, and current main has a deterministic source path that sends that Markdown to Feishu's converter without equation-specific preservation; live Feishu credentials would complete rendered-output confirmation.

## Solution Assessment

Yes. A Feishu-plugin-local repair is the narrow maintainable path because the failure is in the bundled Feishu document write surface, not in core Markdown parsing or cross-plugin behavior.
