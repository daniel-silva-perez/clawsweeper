## Summary

Keep this issue open. The report is recent, concrete, and maps to the bundled Feishu document plugin path on current main. Current code still sends write/append/insert content through Feishu's markdown convert API without a formula-aware preservation layer, and I found no current source or test coverage for LaTeX/equation backslash handling.

## Best Possible Solution

Keep this open as a Feishu plugin bug. The likely fix belongs in `extensions/feishu/src/docx.ts`: preserve `$$...$$` and `<equation>` contents before markdown conversion or construct Feishu equation blocks directly, then add focused tests with the reported complex formula to prove backslashes survive write/append/insert paths.
