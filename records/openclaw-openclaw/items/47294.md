## Summary

Keep open. The reported Feishu document math-loss path still exists on current main: `feishu_doc` write/append/insert sends the original Markdown through Feishu `document.convert` with `content_type: "markdown"`, and there is still no equation/LaTeX-specific preservation path or regression test for `$$...$$` or `<equation>` content. The earlier ClawSweeper comment is still consistent with current main.

## Best Possible Solution

Implement a Feishu-plugin-local fix that preserves raw LaTeX inside `$$...$$` blocks and `<equation>...</equation>` tags for `feishu_doc` write/append/insert, preferably using the official Feishu equation block contract or a verified escaping strategy. Keep the change out of core Markdown behavior, add regression tests for the complex formula from the report, and document any remaining Feishu API limitation if full rendering is not possible.
