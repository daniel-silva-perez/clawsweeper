## Risks / Open Questions

- The Memoria status API contract needs maintainer verification before adding prompt-time probes; the proposed fallback can report a misleading memory count when stats are unavailable.
- Unbounded startup fetches can delay embedded runs and command/system-prompt generation on slow or blackholed endpoints.
- The file-memory report needs to align with documented bootstrap behavior for `MEMORY.md`, lowercase `memory.md`, and `memory/*.md` daily notes.
- The Feishu outbound delivery logging is unrelated to the memory feature and records chat, reply, and message identifiers, so it should be split or explicitly reviewed as a channel logging/privacy change.
- The product boundary is still unsettled: automatic core prompt injection may be the wrong surface if memory-core status or a documented plugin seam should own memory availability reporting.
