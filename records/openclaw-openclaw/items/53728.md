Keep this PR open. ## What This Changes

The PR adds a new agent memory-source report that probes Memoria and file memory for prompt workspace notes in embedded and command/system-prompt flows, while also changing Feishu reply dispatch logging and tests to record outbound send results.

## Best Possible Solution

Land a focused, maintainer-approved memory-source visibility design that uses a documented memory status contract or generic prompt-note seam, bounds all startup probes, represents unknown counts honestly, matches documented MEMORY.md and daily-note semantics, and handles the Feishu delivery logging as a separate reviewed change if it is still desired.
