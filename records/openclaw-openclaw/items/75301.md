## Summary

Keep open. Current main has partial cleanup paths for bundled runtime-deps and session artifacts, but it still lacks the requested cross-cache `openclaw caches` inventory/prune surface, and the remaining deletion boundary spans enough local state that it needs maintainer product and safety design before implementation.

## What This Changes

Add a user-facing cache inventory and safe prune command for cache-like directories under the OpenClaw state directory, including plugin runtime deps, browser/tools downloads, memory artifacts, and orphan transcripts.

## Best Possible Solution

Design a core-owned cache inventory and dry-run-first prune surface with explicit cache-vs-data classifications, a deletion allowlist, current-version and lock protection for runtime deps, fresh/live transcript safeguards, and opt-in handling for expensive or user-adjacent caches such as memory, browser, and tools.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection provide high-confidence verification that current main lacks the requested cross-cache command while only partial cleanup surfaces exist.

## Solution Assessment

Unclear whether the final UI should be a top-level command or doctor integration. The safest direction is the requested class-based dry-run inventory plus narrow pruning rules; broad automatic deletion should wait for explicit maintainer guardrails.
