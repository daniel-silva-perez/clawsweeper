## Summary

Keep this issue open. Current main still has session labels, backend `sessions.create`, and a WebChat selector for existing sessions, but the visible WebChat New Session button still sends `/new`, which resets the current session instead of creating and switching to an isolated labeled dashboard session. The active implementation candidate is open PR #73725; the earlier related PRs #52042 and #52746 are closed unmerged, and #51793 was already closed as duplicate/superseded in favor of this issue.

## Best Possible Solution

Keep this issue open as the canonical tracker. Review, fix if needed, and land or close #73725 for the narrow WebChat New Chat -> `sessions.create` create/switch flow. After that lands, maintainers should decide whether to split the remaining agent-controlled create/relabel API and broader rename/delete/sidebar management into narrower follow-ups.
