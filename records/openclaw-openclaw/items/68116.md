## Summary

Keep this PR open. Current main still lacks the requested LanceDB index maintenance commands, and the PR is directionally useful, but it is not merge-ready because it adds user-facing CLI behavior without docs, has no targeted tests for the new LanceDB calls, and is substantially behind current main after adjacent memory-lancedb CLI work landed.

## What This Changes

The PR adds `MemoryDB.reindex`, `MemoryDB.dropIndexes`, `ltm reindex --timeout`, and `ltm drop-index` to the memory-lancedb plugin, wiring LanceDB index creation waiting and index dropping into the plugin CLI.

## Best Possible Solution

Land a rebased plugin-owned PR that adds the maintenance commands, reuses strict shared option parsing, documents the new CLI surface on the memory-lancedb docs page, and adds mocked tests for the LanceDB index calls before release.

## Reproduction Assessment

Not applicable as a feature PR rather than a bug report. Current-main source inspection is a high-confidence verification that `reindex` and `drop-index` are absent, and the PR diff shows the intended additions.

## Solution Assessment

No, not yet. The owner-module location and LanceDB APIs are appropriate, but the current PR should be rebased onto current main, documented, tested, and tightened around timeout parsing before it is the best maintainable solution.
