## Summary

Keep this PR open. Current main still does not provide `MemoryDB.reindex`, `MemoryDB.dropIndexes`, `ltm reindex`, or `ltm drop-index`, so the PR is not obsolete; however the branch still needs maintainer review because the new timeout option lacks positive-integer validation, the maintenance commands are undocumented and untested, and open PR #67952 edits the same `ltm` CLI area.

## What This Changes

The PR adds LanceDB memory maintenance methods and CLI commands for `ltm reindex --timeout` and `ltm drop-index`, wiring `waitTimeoutSeconds` into LanceDB index creation and dropping existing table indexes in sorted order.

## Best Possible Solution

Keep this PR open and request a focused revision: validate `--timeout` as a positive integer before calling LanceDB, add mocked tests for `reindex`, `dropIndexes`, empty-table behavior and `waitTimeoutSeconds` wiring, document `ltm reindex` and `ltm drop-index` on the memory-lancedb docs page, coordinate shared CLI parsing with #67952, then route the destructive maintenance command for maintainer review.
