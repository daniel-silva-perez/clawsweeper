## Summary

Keep this PR open. Current main still lacks the requested memory-lancedb reindex/drop-index commands, and the PR uses the right plugin-owned surface, but it is not merge-ready because the new user-facing maintenance commands are undocumented and the timeout parser still accepts partial numeric input.

## What This Changes

The PR adds `MemoryDB.reindex`, `MemoryDB.dropIndexes`, `openclaw ltm reindex --timeout`, and `openclaw ltm drop-index` to the `memory-lancedb` plugin using LanceDB index creation, listing, and drop APIs.

## Best Possible Solution

Land a memory-lancedb-owned update after rebase that documents `ltm reindex` and `ltm drop-index`, keeps the LanceDB API calls inside the plugin, uses strict timeout parsing, and adds focused mocked tests for the new index maintenance paths.

## Reproduction Assessment

Not applicable as a feature PR rather than a bug report. Source inspection gives a high-confidence check: current main lacks the commands, while the PR head adds them and exposes the remaining docs/parser gaps.

## Solution Assessment

No, not yet. The plugin-local implementation and LanceDB APIs are appropriate, but the maintainable solution should document the new maintenance commands, reuse strict option parsing, and add focused tests before merge.

- **[P2] Document the new ltm maintenance commands:** `extensions/memory-lancedb/index.ts:830-852`
  - body: The patch registers `ltm reindex` and `ltm drop-index`, but the memory-lancedb docs still list only `ltm list`, `ltm search`, and `ltm stats`.
