## Summary

Keep this PR open. Current main still lacks the memory-lancedb reindex/drop-index maintenance surface, and the PR is the active implementation candidate, but it still needs focused maintainer review because the timeout flag is unvalidated, the destructive index-drop command is undocumented and untested, and a related open CLI PR by the same author should be coordinated rather than closed around it.

## What This Changes

This PR adds memory-lancedb `MemoryDB.reindex` and `MemoryDB.dropIndexes` helpers plus `ltm reindex --timeout` and `ltm drop-index` CLI commands backed by LanceDB index APIs.

## Best Possible Solution

Revise this PR in place: validate `--timeout` as a positive integer before calling LanceDB, add focused mocked tests for reindex/drop-index behavior, document the new commands on the memory-lancedb docs page, and get maintainer approval for the drop-index UX before merge.
