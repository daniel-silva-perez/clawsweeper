## Summary

Keep this PR open. Current main still does not implement the requested LanceDB index maintenance commands, and the live PR adds the intended surface, but it is not ready for maintainer cleanup/merge because timeout input is still unvalidated, no tests or docs were added for the new user-facing commands, an unrelated workspace settings change is included, and the same file overlaps with the open #67952 memory-lancedb CLI PR.

## What This Changes

The PR adds `MemoryDB.reindex`, `MemoryDB.dropIndexes`, `ltm reindex --timeout`, and `ltm drop-index` to the bundled `memory-lancedb` plugin, and also removes `makefile.configureOnOpen` from `.vscode/settings.json`.

## Best Possible Solution

Keep this PR open and request a focused revision: remove the unrelated workspace settings change, validate `--timeout` as a positive integer before calling LanceDB, add mocked tests for `reindex`, `dropIndexes`, empty-table behavior, timeout parsing, and `waitTimeoutSeconds` wiring, document the two new `ltm` maintenance commands on the memory-lancedb docs page, and coordinate the shared CLI parsing/conflict with #67952 before normal maintainer review.
