## Summary

Keep open. Current main still does not enforce the configured Feishu webhook path, and the PR remains a focused implementation candidate for the linked Feishu ingress bug. The provided latest PR patch is Feishu-owned, adds targeted regression coverage, and does not show unrelated CI, dependency, publishing, or supply-chain changes.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or replace it with an equivalent Feishu-owned fix in `extensions/feishu/src/monitor.transport.ts` that normalizes configured and request webhook paths via the plugin SDK helper, rejects malformed/non-origin request targets before body parsing/signature validation/dispatch, returns 404 for mismatched paths, preserves query-string acceptance on the configured path, and keeps the bypass regression tests.
