## Summary

Keep this PR open. Current main still starts the Feishu webhook server with the configured `webhookPath` only as a logged/status/rate-limit value, then processes every request that passes generic guards without comparing the request path before body parsing, signature validation, and dispatch. The PR remains a focused Feishu-owned implementation candidate for the linked ingress bug and does not appear to include unrelated security-sensitive or supply-chain changes.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or replace it with an equivalent Feishu-owned fix in `extensions/feishu/src/monitor.transport.ts` that normalizes the configured and request paths with the plugin SDK helper, rejects malformed/non-origin request targets before body parsing/signature validation/dispatch, preserves query-string acceptance on the configured path, and keeps regression tests for wrong-path and bypass cases. The linked #54841 should close only after this or an equivalent fix lands on main.
