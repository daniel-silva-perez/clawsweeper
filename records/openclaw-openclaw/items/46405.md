## Summary

Keep this PR open for maintainer review: current main still drops MiniMax XML tool calls, so the agent-side problem is real, but the proposed parser has blocking correctness bugs and the gateway half is stale against main’s newer config-write-flow shape plus the active browser reload tracker.

## What This Changes

The PR adds MiniMax M2.5 XML tool-call parsing/promotion, changes config.patch restart scheduling and gateway tool wording, and adds agent/gateway regression tests.

## Best Possible Solution

Replace or rework this into a focused MiniMax parser change on current main that promotes XML invocations through the existing toolUse execution path, parses XML attributes independent of order, and decodes standard XML entities. Leave browser-profile reload classification to the active #43803 path unless maintainers intentionally scope a fresh gateway change against the current config-write-flow design.

## Reproduction Assessment

Yes. The MiniMax report has concrete configuration steps, and the PR findings reproduce by feeding the parser valid XML like <invoke id="x" name="Bash"> or encoded parameter text such as &amp;; current main’s tests and sanitizer source also show XML is stripped rather than executed.

## Solution Assessment

No. The PR addresses a real agent bug, but it is not the best merge shape because it combines unrelated agent and gateway changes, contains parser correctness bugs, and overlaps gateway code that has since been refactored on main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Parse XML attributes in any order:** `src/agents/pi-embedded-utils.ts:41`
  - body: parseMinimaxToolCallXml only matches <invoke> and <parameter> when name is the first attribute. Valid XML such as <invoke id="x" name="Bash"> is skipped, after which the sanitizer strips the XML and silently drops the intended tool call.
  - confidence: 0.9
- **[P2] Decode XML entities before building tool input:** `src/agents/pi-embedded-utils.ts:48-51`
  - body: Parameter text is copied directly from the XML capture, so encoded values like &amp;, &lt;, or &quot; reach tools unchanged. Decode standard XML entities before assigning toolUse.input values so commands and file contents match what the model encoded.
  - confidence: 0.88
