## Summary

Keep open. Current main still strips MiniMax XML invocations instead of promoting them to toolUse blocks, so this PR is not obsolete; however the PR is not merge-ready because the MiniMax parser mishandles valid XML attribute ordering and XML entities, while the gateway restart half has been superseded by current config-write flow work.

## What This Changes

The PR adds MiniMax M2.5 XML tool-call parsing/promotion and changes gateway config.patch restart scheduling, prompt text, tool descriptions, and regression tests so hot-reloadable config patches skip direct SIGUSR1 restarts.

## Best Possible Solution

Replace or rework this into a focused MiniMax parser change on current main that promotes XML invocations through the existing toolUse execution path with attribute-order-independent parsing and standard XML entity decoding, while leaving the gateway restart behavior to the current shared config-write implementation unless a fresh gateway regression is reproduced.

## Reproduction Assessment

Yes for the MiniMax side: the linked report gives concrete model/config steps, and current main source plus tests show MiniMax XML is stripped rather than promoted. The gateway side was not live-reproduced here, but current main source and tests show the direct SIGUSR1 scheduling path has already been refactored.

## Solution Assessment

No. The current PR is useful source work but not the best merge shape because it combines a still-needed MiniMax fix with gateway code that current main has already moved, and the MiniMax parser has correctness bugs around XML attributes and entities.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Parse XML attributes independent of order:** `src/agents/pi-embedded-utils.ts:35`
  - body: `parseMinimaxToolCallXml` only matches `<invoke>` when `name` is the first attribute, so valid XML like `<invoke id="x" name="Bash">...` is skipped. The later sanitizer then strips the XML, silently dropping the intended tool call; parse attributes in any order for invoke and parameter tags.
  - confidence: 0.9
- **[P2] Decode XML entities before passing tool inputs:** `src/agents/pi-embedded-utils.ts:48-51`
  - body: Parameter text is copied directly from XML captures, so valid encoded values such as `&amp;`, `&lt;`, or `&quot;` reach tools unchanged. That changes commands and file contents from what the model encoded; decode standard XML entities before building `toolUse.input`.
  - confidence: 0.88
