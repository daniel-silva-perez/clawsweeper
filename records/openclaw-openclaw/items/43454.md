## Summary

Keep this issue open. Current main has adjacent Plugin SDK lifecycle hooks, context-engine subagent callbacks, and internal Gateway shutdown hooks, but it still does not expose the requested workspace/internal HOOK.md events for subagent completion, turn completion, or tool-call thresholds. The fresh discussion adds a concrete external memory-layer use case and an author workaround, so this needs maintainer API/product judgment rather than conservative cleanup.

## Best Possible Solution

Keep this open for maintainer API judgment. The likely path is a narrow, documented bridge from the existing Plugin SDK subagent_ended flow into an internal/workspace subagent:ended event with tests, plus a separate design decision for agent:turnComplete and any tool-call count or threshold semantics. If maintainers decide lifecycle automation should stay Plugin SDK-only, close later with a clear product decision and docs pointing users to Plugin hooks and context-engine lifecycle APIs.
