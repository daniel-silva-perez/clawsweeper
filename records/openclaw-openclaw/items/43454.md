## Summary

Keep this issue open. Current main still has adjacent Plugin SDK lifecycle hooks for subagent completion, tool calls, and agent end, and internal/workspace hooks now include some Gateway lifecycle events, but the requested workspace/internal `HOOK.md` events for `subagent:ended`, `agent:turnComplete`, and `agent:toolCallThreshold` are not implemented. The issue also has fresh user discussion and an external memory-layer example, so this needs maintainer API/product judgment rather than cleanup.

## Best Possible Solution

Keep this open for maintainer API judgment. The likely implementation path is a narrow, documented internal-hook bridge for `subagent:ended` with regression tests, plus an explicit design decision for `agent:turnComplete` and any tool-count or threshold semantics. If maintainers decide lifecycle automation should stay Plugin SDK-only, close later with a clear product decision and docs pointing users to Plugin hooks and context-engine lifecycle APIs.
