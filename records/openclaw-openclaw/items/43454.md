## Summary

Keep this issue open. Current main has adjacent Plugin SDK lifecycle hooks and now has some internal Gateway shutdown lifecycle events, but it still does not implement the requested workspace/internal `HOOK.md` events for `subagent:ended`, `agent:turnComplete`, or `agent:toolCallThreshold`. The fresh issue discussion also shows an external memory-layer use case and an author workaround, so this needs maintainer API/product judgment rather than cleanup.

## Best Possible Solution

Keep this open for maintainer API judgment. The likely path is a narrow, documented internal-hook bridge for `subagent:ended` with tests, plus an explicit design decision for `agent:turnComplete` and any tool-count or threshold semantics. If maintainers decide lifecycle automation should stay Plugin SDK-only, close later with a clear product decision and docs pointing users to Plugin hooks and context-engine lifecycle APIs.
