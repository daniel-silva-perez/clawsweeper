## Summary

Keep this PR open. Current main still does not implement the intended bare `group:<numeric>` Telegram delivery normalization in the active Telegram plugin, and the PR itself targets obsolete `src/telegram/*` paths rather than `extensions/telegram/src/*`.

## Best Possible Solution

Keep this PR open until it is reworked or replaced against the current plugin layout. The durable fix should live in `extensions/telegram/src/targets.ts`, normalize bare `group:<numeric>` before `parseTelegramTarget` handles topic and generic colon forms, preserve explicit `telegram:group:` and topic behavior, and add focused coverage for normalization, parsing, chat-type resolution, send-time delivery, and exec approval routing.
