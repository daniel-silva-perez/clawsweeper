## Summary

Current main still does not implement this PR's intended bare `group:<numeric>` Telegram delivery normalization in the active Telegram plugin. The PR also targets the old `src/telegram` layout and remains functionally incomplete for the shared parser path that send and approval routing use.

## Best Possible Solution

Keep this PR open until it is reworked or replaced by a current-layout fix. The durable implementation should live in `extensions/telegram/src/targets.ts`, normalize bare `group:<numeric>` before topic and generic colon parsing in `parseTelegramTarget`, preserve explicit `telegram:group:` and topic behavior, and add focused coverage for strip/normalize/parse/chat-type behavior plus send-time delivery and exec approval routing. Because related PR #51688 is closed/merged but current main and `v2026.4.26` still show the unfixed behavior, it is not enough to close this PR as implemented or superseded yet.
