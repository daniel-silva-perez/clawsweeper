## Summary

Current main still does not implement this PR's intended bare `group:<numeric>` Telegram delivery normalization in the active Telegram plugin. The active code is under `extensions/telegram/src`, and it still preserves bare `group:` prefixes before parsing and delivery resolution; the open PR also edits obsolete `src/telegram/*` paths rather than the current plugin files.

## Best Possible Solution

Keep this PR open for rework or replacement against `extensions/telegram/src`. The durable fix should normalize bare `group:<numeric>` before parser topic/generic-colon handling, preserve explicit `telegram:group:` and topic behavior, and add focused coverage for normalization, parsing, send-time delivery, messaging target normalization, and exec approval routing.
