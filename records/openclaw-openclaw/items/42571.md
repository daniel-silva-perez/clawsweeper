## Summary

Current main still does not implement the intended bare `group:<numeric>` Telegram delivery normalization in the active Telegram plugin. The existing discussion correctly identified the parser gap, and this PR also targets obsolete `src/telegram/*` paths instead of the current `extensions/telegram/src/*` implementation.

## Best Possible Solution

Keep this PR open until it is reworked or replaced against the current plugin layout. The durable fix should live in `extensions/telegram/src/targets.ts`, normalize bare `group:<numeric>` before parser topic/generic-colon handling, preserve explicit `telegram:group:` and topic behavior, and add focused coverage for normalization, parsing, chat-type resolution, send-time delivery, and exec approval routing.
