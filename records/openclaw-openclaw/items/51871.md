## Summary

Keep this issue open. The discussion identified a concrete malformed persisted cron row shape, and current main still has Control UI and cron service paths that dereference `schedule.kind` or `payload.kind` without quarantining invalid `jobs.json` rows before they reach the dashboard.

## Best Possible Solution

Keep open. Harden persisted cron job loading in `src/cron/service/store.ts`/`src/cron/normalize.ts` so jobs missing required `schedule` or valid `payload.kind` are either safely inferred, quarantined with diagnostics, or surfaced as invalid without crashing. Then add defensive UI rendering in `ui/src/ui/controllers/cron.ts` and `ui/src/ui/views/cron.ts`, plus regression tests for `cron.list`, scheduler load, and Control UI rendering with malformed `jobs.json` rows.
