## Risks / Open Questions

- Defaulting emoji filtering to enabled changes spoken output for users who intentionally want emoji names or symbols spoken; the fix should document the default and support `filterEmoji: false` as an opt-out.
- Emoji filtering is subtle for variation selectors, skin-tone modifiers, flags, keycaps, and ZWJ sequences; reuse or centralize the existing Unicode-property approach instead of scattering unrelated regexes.
- A correct fix likely touches schema/generated config metadata, docs, runtime, tool/RPC behavior, and tests; generated/help drift should be checked explicitly.
