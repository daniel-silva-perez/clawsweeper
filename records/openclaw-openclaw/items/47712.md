## Summary

Keep open. Current main still lacks `messages.tts.filterEmoji` in the strict TTS config schema, public TTS type surface, docs, and the tool/RPC/auto-TTS runtime paths that send text to speech providers.

## What This Changes

Add a `messages.tts.filterEmoji` option so TTS can filter emoji from spoken output before speech generation, with a documented default and opt-out.

## Best Possible Solution

Keep this open and implement a narrow TTS config/runtime change: add `messages.tts.filterEmoji?: boolean` to schema, types, generated config docs/help, and public docs; default it as the maintainers choose, preferably with the requested filter-on behavior plus opt-out; apply it only to text sent to speech providers and spoken-text/tool metadata, while preserving visible chat text.
