## Summary

Current main does not implement the requested `messages.tts.filterEmoji` behavior. The schema/types/docs do not expose that key, the core TTS tool and auto-TTS pipeline pass text through without a generic emoji filter, and the only emoji stripping I found is provider/channel-specific.

## Best Possible Solution

Keep this issue open and implement a shared TTS text-sanitization option: add `filterEmoji?: boolean` to `TtsConfig`, `TtsConfigSchema`, generated config metadata/docs, and apply it in the shared speech-core text preparation path before provider synthesis for the agent tool, auto-TTS, `/tts audio`, gateway RPC, and telephony paths. Add focused tests with emoji-only text, mixed emoji/punctuation, and a provider-spy assertion that the provider receives filtered text when enabled.
