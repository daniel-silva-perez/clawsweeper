## Summary

Keep open. Current main still does not define, document, validate, or apply `messages.tts.filterEmoji`; existing emoji stripping is limited to provider- or channel-local paths and does not satisfy the requested shared TTS behavior.

## What This Changes

Add a `messages.tts.filterEmoji` option so TTS synthesis can skip emoji, preferably by default, before generating speech.

## Best Possible Solution

Add a shared TTS text-preparation path with a documented `messages.tts.filterEmoji?: boolean` default, apply it before synthesis across auto-TTS, the agent tool, and Gateway conversion, and keep visible chat reply text unchanged.
