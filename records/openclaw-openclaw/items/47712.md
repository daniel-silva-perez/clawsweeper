## Summary

Keep open. Current main still does not accept or honor `messages.tts.filterEmoji`; the request remains a narrow TTS config/runtime/docs gap rather than something already implemented or stale.

## What This Changes

The issue requests native `messages.tts.filterEmoji` support, defaulting to filtering emoji before text is sent to speech generation.

## Best Possible Solution

Add a shared TTS text-preparation helper controlled by `messages.tts.filterEmoji?: boolean`, document the default, apply it consistently to auto-TTS, gateway conversion, and the agent `tts` tool, and keep provider-local sanitizers either removed or layered behind the shared behavior without changing visible replies.
