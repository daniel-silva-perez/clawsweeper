## Summary

Keep open. Current main still does not define or honor `messages.tts.filterEmoji`; the shared config contract, generated schema, docs, automatic reply TTS path, agent `tts` tool, and Gateway `tts.convert` path all lack config-controlled emoji filtering. Provider-local emoji stripping exists in Local CLI and Discord voice paths, but that does not satisfy the requested shared TTS behavior.

## What This Changes

Add a `messages.tts.filterEmoji` option so text-to-speech can skip emoji before speech synthesis.

## Best Possible Solution

Add `messages.tts.filterEmoji?: boolean` to the public config contract with a documented default, implement one shared TTS text-preparation helper that strips markdown and optionally emoji, and route automatic reply TTS, the agent `tts` tool, and Gateway `tts.convert` through that helper while preserving visible chat reply text and provider selection behavior.
