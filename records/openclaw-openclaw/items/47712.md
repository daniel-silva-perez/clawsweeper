## Summary

Keep open. Current main still does not accept or honor `messages.tts.filterEmoji`; the shared TTS config contract, docs, auto-TTS path, agent `tts` tool, and Gateway `tts.convert` path all lack config-controlled emoji filtering. Existing Local CLI and Discord voice emoji stripping is path-local and does not satisfy the requested core TTS behavior.

## What This Changes

Add a `messages.tts.filterEmoji` option so TTS skips emoji before synthesis, defaulting to emoji filtering as requested.

## Best Possible Solution

Expose and document `messages.tts.filterEmoji?: boolean`, default it consistently, and route all synthesis entry points through one shared spoken-text preparation helper while preserving visible chat text and transcript content.
