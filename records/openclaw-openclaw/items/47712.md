## Summary

Keep open. Current main still does not accept or honor `messages.tts.filterEmoji`; the shared TTS config, generated schema, docs, auto-TTS path, agent tool, and `tts.convert` path all lack config-controlled emoji filtering. Provider-local sanitizers exist for Local CLI and Discord voice, but they do not solve the requested core TTS behavior.

## What This Changes

Add `messages.tts.filterEmoji` support so text-to-speech can skip emoji, preferably by default, before speech synthesis.

## Best Possible Solution

Add a shared TTS text-preparation helper controlled by `messages.tts.filterEmoji?: boolean`, document its default and override behavior, apply it consistently before synthesis in auto-TTS, the agent `tts` tool, and `tts.convert`, and keep provider-local sanitizers either removed or layered behind the shared behavior without changing visible replies.
