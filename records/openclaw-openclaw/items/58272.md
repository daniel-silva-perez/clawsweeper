## Best Possible Solution

Keep this issue open. The next maintainer step should be a targeted regression/reproduction around read-tool path arguments containing mixed CJK+ASCII directory and filename segments, ideally capturing the raw provider tool-call payload. If the provider emits a mutated path, improve provider-specific tool-call guidance/repair or return a clearer diagnostic; if OpenClaw mutates it after receipt, fix the responsible parser/path layer and add coverage for both directory and filename cases.
