## Summary

Keep this PR open. Current main still only has a partial Feishu duration sink: `uploadFileFeishu` accepts and forwards `duration` when supplied, but the normal `sendMediaFeishu` outbound path does not compute or pass duration for Opus/MP4 media. The PR also remains non-landable as-is based on the maintainer review and the security/guardrail-sensitive changes in its diff.

## Best Possible Solution

Keep this PR open for maintainer follow-up, but require a rework or replacement patch: preserve `resolveFeishuOutboundMediaKind` and Feishu download metadata extraction, compute duration from normalized/temp media inputs using the existing safe `runFfprobe`/`execFile` helper, pass duration into `uploadFileFeishu` for proven Opus/MP4 uploads, and add focused Feishu tests proving duration propagation while MIME fallback, metadata extraction, and MP3 generic-file fallback still work. Coordinate the fix with #53798 as the canonical user-facing bug.
