## Summary

Keep this PR open. Current main still does not automatically derive or pass Feishu audio/video duration on the normal media send paths; it only forwards a `duration` value if a caller already supplies one. The PR targets a real Feishu behavior gap also tracked by #53798, but the current PR discussion and diff show it still needs a smaller, safer rework before it is landable.

## Best Possible Solution

Keep this PR open for maintainer follow-up or replacement. The best fix is a narrow Feishu-only patch that preserves `extractFeishuDownloadMetadata` and `resolveFeishuOutboundMediaKind`, computes duration after media normalization using the shared `runFfprobe`/`execFile`-based helper, passes duration only for proven Feishu Opus/MP4 uploads, and adds focused tests for duration propagation plus existing MIME fallback, metadata, native Opus/Ogg, MP4, and MP3 fallback behavior. Coordinate the final fix with #53798.
