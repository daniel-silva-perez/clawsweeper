## Summary

Keep this PR open. Current main still only has a Feishu duration sink in `uploadFileFeishu`; the normal `sendMediaFeishu` message/reply path routes Opus/MP4 but does not compute or pass a duration. The PR is also not safe to close as implemented or merge as-is because the provided diff/discussion show security-sensitive and regression-prone changes, and #53798 remains the better canonical user-facing bug to fix.

## Best Possible Solution

Keep this PR open for maintainer follow-up, but require a rework or replacement patch: preserve `extractFeishuDownloadMetadata` and `resolveFeishuOutboundMediaKind`, compute duration after media normalization using the shared `runFfprobe`/`execFile`-based helper, pass duration only for proven Feishu Opus/MP4 uploads, and add focused tests for duration propagation plus the existing MIME fallback, metadata extraction, native Opus/Ogg, MP4, and MP3 fallback behaviors. Coordinate the final fix with #53798.
