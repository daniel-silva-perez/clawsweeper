## Summary

Keep open. Current main still lets failed tool/toolResult text become subagent completion fallback output, and the childCompletionFindings path can bypass ANNOUNCE_SKIP/NO_REPLY cleanup. The direct fix PR #39367 is closed unmerged; #59649 fixed adjacent internal-context surfaces but did not touch this extractor/announce branch; #74113 is open but addresses an adjacent post-compaction retry path, not the failed-tool filtering gap.

## Best Possible Solution

Keep this issue open and land a focused agents fix. Filter failed tool/toolResult messages before they can populate subagent fallback output, and make the childCompletionFindings branch honor the same ANNOUNCE_SKIP/NO_REPLY and reply-classification rules as the normal reply branch. Add regression coverage for failed-tool filtering, successful tool fallback preservation, and child aggregate skip/silent behavior; coordinate with #74073/#74113 only for overlap.

## Repair Work Prompt

Fix the subagent completion output leak described by openclaw/openclaw#39032. Work in the agents subagent announce path only. In `src/agents/subagent-announce-output.ts`, add a narrow helper that identifies failed `tool`/`toolResult` messages before they populate `latestRawText`; cover top-level `is_error`, top-level `isError`, and structured `details.status` error-like statuses such as error, failed, timeout, denied, cancelled, invalid, or forbidden. Preserve successful tool/toolResult fallback output. In `src/agents/subagent-announce.ts`, make the `childCompletionFindings` path honor the same `ANNOUNCE_SKIP`/`NO_REPLY` and `stripAndClassifyReply` cleanup semantics as the normal reply path, or sanitize/suppress the aggregate before it becomes `task_completion.result`. Preserve deterministic child result ordering and successful child completion synthesis. Add regression tests proving failed tool results are skipped, assistant output still wins, successful tool-only fallback still works, only-failed-tool output does not leak, and child completion aggregates cannot bypass skip/silent output decisions. Check related #39367, #59649, #74073, and #74113 for context/overlap, but do not broaden this PR into a compaction-race redesign unless required for the specific leak. Add a user-facing changelog entry under the active version if repo policy still expects one for this bug fix; credit @cryptopoly, @mvanhorn if reusing the prior approach, and @dcapclaw for the childFindings repro context as appropriate. Do not change provider behavior, channel routing, or unrelated task/status sanitizers.
