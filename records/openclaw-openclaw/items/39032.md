## Summary

Keep open. Current main still allows failed tool/toolResult content to become subagent completion fallback output, and the childCompletionFindings path can bypass the ANNOUNCE_SKIP/NO_REPLY cleanup that normal replies receive. The direct fix PR #39367 is closed unmerged, and adjacent hardening in #59649 did not cover this extractor/announce branch.

## Best Possible Solution

Keep this issue open and land a focused agents fix. Filter failed tool/toolResult messages before they can populate subagent fallback output, and make the childCompletionFindings branch honor the same ANNOUNCE_SKIP/NO_REPLY and reply-classification rules as the normal reply branch. Add regression coverage for failed-tool filtering, successful tool fallback preservation, and child aggregate skip/silent behavior.
