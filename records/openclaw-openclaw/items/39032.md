## Summary

Keep open. Current main still allows failed tool/toolResult text to become subagent completion fallback output: the extractor copies tool result text without checking error markers, stores it as latestRawText, and the announce path can forward that as requester-visible task_completion result. The prior direct fix PR #39367 was closed unmerged, and the adjacent merged hardening in #59649 did not change this extractor path.

## Best Possible Solution

Keep this issue open and land a focused fix in src/agents/subagent-announce-output.ts: ignore failed tool/toolResult messages before they populate latestRawText, reuse or mirror existing tool-result error/status semantics, and add regression tests proving failed tool results are skipped while successful tool-only fallback still works.
