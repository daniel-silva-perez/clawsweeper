## Summary

Keep open. Current main has cron run delivery diagnostics and generic message-sent hooks, but it does not emit the requested structured `cron.delivery.sent` log line and does not carry `cronJobId`/`cronRunId` through each outbound delivery attempt with the platform message ID. The related #69054 is a companion lint/config-validation request, not a canonical replacement for this forensic logging work.

## Best Possible Solution

Implement this in the core cron outbound path: thread `{ cronJobId, cronRunId, agentId }` plus resolved `{ accountId, channel, to }` into `deliverOutboundPayloads` or a cron-owned wrapper, emit exactly one structured privacy-safe log record on success and error, include the returned platform message ID fields, and add focused tests for success, thrown errors, best-effort partial failures, chunked sends, and Telegram/Discord result mapping.
