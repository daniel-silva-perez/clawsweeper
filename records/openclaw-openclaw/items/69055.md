## Summary

Keep open. Current main has cron run delivery diagnostics and generic outbound message hooks, but it still does not emit the requested privacy-safe structured `cron.delivery.sent` log line with cron job/run attribution plus platform message IDs for every outbound cron delivery attempt. The related #69054 is a companion config-lint request, not a substitute for this forensic logging work.

## Best Possible Solution

Implement this in the core isolated cron outbound path: thread `{ cronJobId, cronRunId, agentId }` plus resolved `{ accountId, channel, to }` into the outbound delivery attempt, emit exactly one privacy-safe structured `cron.delivery.sent` record on success and error, include platform message ID fields from channel results, and add focused tests for success, thrown errors, best-effort partial failures, chunked sends, and Telegram/Discord result mapping.
