## Summary

Current main still does not satisfy the request. OpenRouter's own API exposes the routed model and provider usage fields, but OpenClaw still stores the selected `openrouter/auto` model, normalizes only token counters, computes local estimated cost, and formats response usage footers without a resolved-model label or provider-reported cost. Prior PR #43026 was closed unmerged, while #9016 and #68066 remain related open cost-tracking work rather than replacements for the resolved-model request.

## Best Possible Solution

Keep this issue open. The best implementation path is to capture OpenRouter's response `model` and provider-reported `usage.cost` / `cost_details` in the OpenAI-compatible transport, preserve selected-versus-resolved model metadata in assistant/session records, and thread that metadata through response footers, usage/session/status surfaces, transcripts, and targeted `openrouter/auto` regression tests. Treat #9016 and #68066 as related cost work, and #43026 plus the ClawRouter comment as useful prior art rather than replacements.
