## Repair Work Prompt

Fix the Slack directory CLI gap from this issue. Current main routes `openclaw directory peers/groups list` through config-backed `plugin.directory.listPeers/listGroups` only, while Slack registers token-backed `listPeersLive/listGroupsLive` separately; `directory self --channel slack` returns null because Slack does not override the default self resolver. Implement a live-aware generic directory CLI path that prefers live list methods when present and preserves config-backed fallback. Add Slack-owned self identity support using the existing `ChannelDirectoryEntry` shape, likely by resolving the effective read token and Slack auth/profile data without exposing raw token or full profile details. Keep the fix scoped to the CLI/directory and Slack directory surfaces; do not change unrelated Slack auth/write semantics or plugin SDK contracts unless a narrow additive seam is required. Add focused regressions for live preference, fallback, Slack peers/groups, and Slack self. Add a user-facing changelog entry with reporter credit if runtime behavior changes.
