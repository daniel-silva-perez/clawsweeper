## Repair Work Prompt

Fix the Slack directory CLI gap reported in https://github.com/openclaw/openclaw/issues/50776. Current `openclaw directory peers/groups list --channel slack` uses config-backed `listPeers`/`listGroups` and ignores Slack `listPeersLive`/`listGroupsLive`; `openclaw directory self --channel slack` returns the default null self resolver. Implement a generic live-aware directory CLI path that prefers live methods when present and preserves config-backed fallback. Add Slack-owned current-account identity support that returns the existing `ChannelDirectoryEntry` shape for the resolved current account. Keep the change scoped to directory CLI and Slack directory behavior; do not change Slack token write policy, `userTokenReadOnly` semantics, unrelated Slack scopes, plugin loading, or outbound target resolution except as required for shared helper reuse. Add focused tests for live preference, config fallback, Slack peers/groups wiring, and Slack self identity. Add a single-line user-facing changelog fix entry under the active version with required `Thanks @pjaillon` attribution if runtime behavior changes.
