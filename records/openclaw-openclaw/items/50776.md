## Summary

Keep open. Current main still routes `openclaw directory peers/groups list --channel slack` through Slack's config-backed directory methods, not the Slack live directory methods that call `users.list`/`conversations.list` with the configured user token. Slack `directory self` also still falls back to the generic null resolver because the Slack adapter does not provide a self resolver. The reported empty peers/groups and null self results can therefore still happen on a valid Slack connection when no Slack allowlist/channel config is present.

## Best Possible Solution

Keep this issue open and fix the Slack directory surface by making the CLI use live directory methods when a channel provides them, or by adding an explicit live-directory mode, then implement Slack `directory.self` using the resolved read token and add regression coverage for Slack peers, groups, and self with a configured user token and empty config-backed allowlists.
