## Summary

Keep open. Current main still leaves the reported Slack directory gap: the CLI peers/groups paths call only config-backed directory methods, Slack's user-token-backed live directory readers are exposed separately, and Slack directory self still falls through to the default null resolver. The report has concrete reproduction commands and token evidence, and the remaining fix is narrow enough for one focused PR.

## What This Changes

Slack directory CLI commands should return Slack self, peers, and groups when a valid Slack user token is configured instead of returning null or empty arrays.

## Best Possible Solution

Keep this issue open and land a narrow fix: make `openclaw directory peers/groups list` prefer live directory methods when a channel provides them while retaining config fallback, add a Slack `self` resolver backed by resolved Slack auth/read-token metadata, and add targeted CLI plus Slack regression tests and a user-facing changelog entry.
