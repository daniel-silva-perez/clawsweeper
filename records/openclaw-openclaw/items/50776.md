## Summary

Keep open. Current main still has the reported Slack directory gap: the public directory CLI calls only the config-backed list methods, Slack exposes token-backed live list methods separately, and Slack does not override the default null self resolver. The report is concrete, recent, non-maintainer-authored, and narrow enough for a focused repair PR.

## What This Changes

Slack directory CLI commands should return live Slack self, peer, and group directory data when valid Slack tokens are configured instead of empty config-backed results.

## Best Possible Solution

Add a live-aware generic directory CLI path that prefers live list methods when present and preserves config-backed fallback, then add Slack-owned self identity support plus focused regressions for token-backed peers, groups, and current-account identity.
