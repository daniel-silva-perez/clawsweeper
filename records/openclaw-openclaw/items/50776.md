## Summary

Keep open. Current main still leaves the reported Slack directory CLI gap: `peers list` and `groups list` call only config-backed directory readers, Slack live user-token-backed readers remain exposed separately as `listPeersLive`/`listGroupsLive`, and Slack `directory self` still falls through to the default `null` resolver. The report has concrete reproduction commands and token evidence, is not old enough for stale-insufficient-info, and the remaining fix is narrow.

## Best Possible Solution

Keep this issue open and fix the Slack directory surface in one narrow PR: make the directory CLI prefer live directory methods when a channel provides them while retaining config fallback, add a Slack `self` resolver backed by the resolved Slack read token/auth metadata, and add targeted regression tests for Slack peers, groups, and self plus a user-facing changelog entry.
