## Summary

Keep open. Current main still routes the public Slack directory list commands through config-backed methods while Slack's token-backed live readers are registered separately, and Slack still falls back to the default null self resolver. The merged #28103 token-sourcing work is adjacent but does not connect these CLI commands to live Slack directory reads or current-account identity.

## What This Changes

Slack directory CLI commands should return live Slack users, channels, and current account identity when valid Slack tokens are configured instead of empty arrays or null.

## Best Possible Solution

Add a live-aware generic directory CLI path that prefers live list methods when present and preserves config-backed fallback, then add Slack-owned current-account identity support with focused regressions for peers, groups, and self.

## Reproduction Assessment

Yes. The issue gives exact CLI commands, and source inspection against current main shows those commands still call config-backed list methods and the default null self resolver rather than Slack's token-backed live readers.

## Solution Assessment

Yes. The narrow maintainable fix is a generic CLI live-reader preference plus Slack-owned self resolution; moving Slack-specific IDs or token policy into core would be broader than needed.
