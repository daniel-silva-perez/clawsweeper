## Summary

Keep open. Current main still matches the reported Slack directory failure path: the public directory CLI calls only config-backed peer/group methods, Slack's token-backed live directory readers are exposed separately, and Slack `self` falls through to the generic null resolver. The report is concrete, has no protected-label or maintainer-author block, and the remaining fix is narrow enough for a focused ClawSweeper repair PR.

## What This Changes

Slack directory CLI commands should return the connected Slack workspace identity, peers, and groups when valid Slack tokens are configured instead of returning `null` or empty arrays.

## Best Possible Solution

Implement the missing runtime path in core CLI generically, then add Slack-owned `self` support and Slack regressions so config-backed directories keep working while token-backed Slack workspaces return live users, channels, and current-account identity.
