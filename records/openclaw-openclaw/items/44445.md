## Summary

Keep open. Current main still stores raw user transcript text in session-memory output without applying the existing inbound metadata, envelope, or message-id cleanup. The PR addresses a valid bug, but it is not ready to close or merge as-is because it also adds scheduled reset command-hook dispatch; the daily/idle reset hook problem is now tracked separately by open PR #61675.

## Best Possible Solution

Keep this PR open for maintainer follow-up, but ask for a narrower rebase: implement session-memory-specific user text sanitization in src/hooks/bundled/session-memory/transcript.ts using the existing inbound metadata, envelope, and message-id helpers while preserving user-authored content. Add focused regression tests for Feishu metadata blocks, timestamp-prefixed transcripts, message-id hint lines, slash-prefixed real content, and user-authored message-id-looking lines. Leave daily/idle reset hook emission to the active #61675 path or another dedicated lifecycle-hook fix.
