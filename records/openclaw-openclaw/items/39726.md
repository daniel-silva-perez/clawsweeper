## Summary

This PR should stay open for maintainer review. Current main has not implemented the proposed chat-context default for quiet successful background exec completions: `notifyOnExitEmptySuccess` is still explicit-only, while the PR remains a focused implementation candidate for the narrower chat-surface part of #18237.

## Best Possible Solution

Review and either land or explicitly decline this PR. If accepted, keep the repaired narrow behavior from the branch: default empty-success completion notifications only for real reply-capable chat delivery contexts with a target, preserve explicit `notifyOnExitEmptySuccess: false`, keep generic/non-chat provider labels silent, and retain the regression tests plus changelog credit.
