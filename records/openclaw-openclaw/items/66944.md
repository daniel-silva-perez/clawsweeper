## Summary

Keep open. Current main still lacks native plugin-contributed Control UI pages: navigation and content rendering are built around fixed tab IDs, the bootstrap config carries no plugin page/import-map metadata, the manifest type has no `ui.pages` declaration, and the existing host-hook descriptor surface is narrower than full plugin-owned pages. The related #71736 RFC and #74483 draft PR are active adjacent work, but they do not currently implement or fully supersede this request.

## What This Changes

The issue requests a plugin UI extension system that lets plugins declare native Control UI pages or tabs, loaded as Custom Elements inside the main Control UI shell.

## Best Possible Solution

Keep this as active design work for a versioned Control UI plugin contribution API. The best end state should decide whether OpenClaw supports full plugin-owned Custom Element pages, host-owned descriptor slots, or both; then document manifest or SDK declarations, asset serving, navigation grouping, gateway/client access, auth scopes, unload/disable semantics, CSP/trust boundaries, and contract/UI coverage before external plugin authors depend on it.

## Reproduction Assessment

Reproduction is not applicable because this is a feature/API request, not a bug report. Current-main source inspection gives high-confidence absence proof: the manifest, bootstrap config, navigation shell, and UI renderer have no native plugin page path.

## Solution Assessment

Unclear. The full Custom Element page loader proposed here is not yet proven to be the best first implementation because the related #71736 design intentionally explores safer host-owned UI contribution descriptors; maintainers should choose the security and API shape before code work starts.

## Security Review

Status: needs_attention

Summary: The requested feature is security-sensitive because it would place plugin-owned browser code or UI descriptors inside the authenticated operator Control UI.

Concerns:

- **[high] Define the Control UI plugin code trust boundary:**
