## Summary

Current main still does not strip JSON Schema `if`/`then`/`else` conditionals in the Gemini tool-schema cleaner, so the reported Gemini/cron schema-rejection path remains valid. An open PR (#70497) already carries the narrow fix and regression test, so the issue should stay open until that PR is reviewed/landed or replaced.

## Best Possible Solution

Keep this issue open and review/land #70497, or apply the equivalent small patch: add `if`, `then`, and `else` to `GEMINI_UNSUPPORTED_SCHEMA_KEYWORDS` and add a focused `cleanSchemaForGemini` regression test that preserves normal properties while stripping conditionals.
