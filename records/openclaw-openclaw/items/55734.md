## Summary

Keep this open. Current main still has the browser plugin selecting CDP page candidates without excluding Chrome internal UI URLs, and the directly related fix PR was closed unmerged, so the reported CDP navigation bug remains actionable.

## What This Changes

The issue requests that CDP browser tab selection ignore Chrome internal `chrome://` and `chrome-untrusted://` targets so navigation happens in real web-content tabs.

## Best Possible Solution

Add one shared browser-plugin predicate for selectable web-content CDP targets, apply it consistently to raw CDP tab listing, Playwright-backed listing, implicit/default selection, explicit target resolution, and managed tab cleanup, and add regression tests covering omnibox popup, tab search, and the fallback to opening/selecting a real `about:blank` page.
