## Summary

Keep this issue open. Current main still has no generic filter for Chrome internal CDP targets: raw `/json/list` entries are preserved when they have any target id, Playwright-backed tab listing returns every resolved page target, and default target selection only prefers `type === "page"`, which still includes `chrome://omnibox-popup.top-chrome/` in the reported reproduction. The linked fix PR #55740 was closed unmerged, so it should be treated as useful prior context rather than proof that the bug landed.

## What This Changes

Filter Chrome internal CDP targets such as `chrome://omnibox-popup.top-chrome/`, `chrome://tab-search.top-chrome/`, and `chrome-untrusted://` out of browser tab selection so navigation lands in real content tabs.

## Best Possible Solution

Add one shared browser-plugin predicate for selectable web-content CDP targets, apply it consistently to raw CDP tab listing, Playwright-backed remote listing, default target selection, and tab-count/cleanup paths, and add regression tests using the issue's `chrome://*.top-chrome` examples plus a fallback that opens or selects `about:blank` when only internal Chrome UI targets are present.
