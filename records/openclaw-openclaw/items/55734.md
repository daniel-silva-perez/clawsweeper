## Summary

Keep open. Current main still leaves the reported Chrome internal UI targets eligible for implicit/default CDP tab selection, and `/navigate` uses that selected target. The related fix PR #55740 is useful context, but it was closed unmerged, so it does not resolve this issue.

## Best Possible Solution

Add a shared browser-target eligibility predicate in the browser plugin that excludes Chrome internal UI URLs such as `chrome://`, `chrome-untrusted://`, and top-chrome surfaces from implicit/default navigation target selection. If no eligible content tab exists, open a real `about:blank` tab. Add focused regression tests for both raw `/json/list` CDP fallback and Playwright-backed remote selection using the Omnibox Popup and Tab Search fixture from the report.

- The fix should be careful about whether internal targets disappear from all tab listings or only from implicit/default navigation target selection; explicit diagnostics may still need clear behavior.
