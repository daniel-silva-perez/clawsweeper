## Summary

Keep this PR open. Current main still has the session_status-specific bare-model provider inference gap: the tool resolves bare model input with the current session provider before allowlist validation, while the shared model-selection resolver and sessions.patch path already perform allowlist-aware provider inference. The PR remains functionally useful, though the best maintainer version should reuse the shared resolver and clean up the optional-chaining nit.

## Best Possible Solution

Keep this PR open or replace it with an equivalent maintainer patch that makes session_status(model=...) use the shared allowlist-aware resolver, then land the bare cross-provider regression test and clean up the optional-chaining nit.
