## Summary

Keep open. Current main still has the session_status-specific gap this PR targets: bare model overrides are resolved with the current provider before allowlist validation, while the shared session model resolver already has allowlist-based provider inference.

## Best Possible Solution

Keep this PR open or replace it with a maintainer patch that makes session_status(model=...) use the same allowlist-aware provider inference as sessions.patch, ideally through resolveAllowedModelRef. Add the bare cross-provider regression test from this PR and address the small optional-chaining review nit before landing.
