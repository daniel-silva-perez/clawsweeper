## Summary

Current main still has the session_status-specific provider inference gap this PR targets. The shared sessions.patch resolver now infers a provider from the configured allowlist for bare model IDs, but session_status(model=...) still resolves bare input with the current session provider before checking the allowlist, so the PR remains useful or should be replaced by an equivalent maintainer patch.

## Best Possible Solution

Keep this PR open or replace it with a maintainer patch that routes session_status(model=...) through the shared allowlist-aware resolver, then add the PR's bare cross-provider regression test and address the small optional-chaining review nit before landing.
