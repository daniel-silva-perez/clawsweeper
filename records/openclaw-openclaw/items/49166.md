## Summary

Current main still has the session_status-specific provider inference gap this PR targets. The shared sessions.patch resolver can infer a unique provider for bare model IDs from the configured allowlist, but session_status(model=...) still resolves bare input with the current session provider before allowlist validation, so this PR or an equivalent maintainer patch remains useful.

## Best Possible Solution

Keep this PR open or replace it with an equivalent maintainer patch that makes session_status(model=...) use the shared allowlist-aware resolver, then land the bare cross-provider regression test and address the optional-chaining nit.
