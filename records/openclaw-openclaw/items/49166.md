## Summary

Keep open. Current main still has the `session_status(model=...)` gap this PR targets: the tool resolves bare model overrides with the current session provider before allowlist validation, while the shared session/model resolver used elsewhere already has allowlist-based provider inference.

## Best Possible Solution

Keep this PR open or replace it with a maintainer patch that makes `session_status(model=...)` use the same allowlist-aware provider inference as `sessions.patch`, ideally through the shared resolver rather than a parallel implementation. Add the PR's openai-codex to bare deepseek-chat regression test in `src/agents/openclaw-tools.session-status.test.ts`, and address the optional-chaining review nit.
