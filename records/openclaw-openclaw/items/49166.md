## Summary

Keep this PR open. Current main still resolves session_status bare model overrides with the current session provider before allowlist validation, while the shared model-selection resolver and sessions.patch path already infer a unique configured provider for bare refs. The PR remains useful, but the best maintainer version should reuse the shared resolver and keep the regression test.

## Best Possible Solution

Keep this PR open or replace it with a maintainer patch that makes session_status(model=...) use the shared allowlist-aware resolveAllowedModelRef path, then land the cross-provider bare-model regression test and remove the optional-chaining nit.
