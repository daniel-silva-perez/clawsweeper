## Summary

Current main still does not restore the cross-machine Chrome extension relay workflow. The issue remains concrete, confirmed by multiple users, and materially distinct from the broader open feature request because it documents the managed-hosting regression and the ClawHosters relay implementation that needs either a hardened core path or a public plugin seam.

## What This Changes

The issue requests restoring or replacing the removed Chrome extension browser relay so a remote OpenClaw instance can control a user's local Chrome across machines without requiring a local node host, CLI, or remote CDP setup.

## Best Possible Solution

Keep this issue open until maintainers choose a deliberate path: either an opt-in hardened extension relay in the browser plugin with schema/docs/tests/security review, or an additive public browser profile-driver/transport seam that lets the ClawHosters relay or a ClawHub plugin integrate without relying on private internals.
