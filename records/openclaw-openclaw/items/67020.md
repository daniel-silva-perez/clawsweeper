## Summary

Keep open. Current main still has the reported Slack monitor gap: Slack accepts and documents `dmHistoryLimit`, but the active Slack monitor only derives and carries `historyLimit`, and the message preparation/dispatch paths still gate pending-history and inbound-history handling on the roomish path that excludes 1:1 DMs. The linked fix PR #67054 was closed unmerged, while #64427 remains broader cross-session Slack history hydration rather than a clean replacement for this narrower monitor-wiring bug.

## What This Changes

The issue requests wiring Slack `dmHistoryLimit` into the Slack channel monitor so non-threaded 1:1 DM replies can include prior DM context instead of dropping inbound history.

## Best Possible Solution

Land a focused Slack plugin fix that derives an effective DM history limit from the resolved Slack account config, carries it through or computes it inside the Slack monitor path, records pending 1:1 DM history when enabled, uses that effective limit for prompt body history, structured `InboundHistory`, and cleanup, and adds focused regression coverage plus a user-facing changelog entry. Keep Slack API cross-session history hydration and cross-channel parity as separate follow-up scope.
