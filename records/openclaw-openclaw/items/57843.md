## Summary

Keep this PR open. Current main still has the pre-PR outbound delivery contract: `deliverOutboundPayloads()` returns `OutboundDeliveryResult[]`, hook cancellation still collapses to an empty result array, non-bestEffort mid-stream failures still rethrow the original error without `sentBeforeError`, and gateway/cron callers still use array length as the status signal. The PR’s intended change is not implemented on main, and the related delivery-status PRs remain open/dependent rather than superseding it.

## Best Possible Solution

Keep the PR open and either land this implementation after maintainer review or implement the same core contract on main: expose delivery outcome metadata from `src/infra/outbound/deliver.ts`, preserve partial-send metadata for non-bestEffort failures, update all result-consuming callers, and keep the gateway/cron tests covering hook cancellation and partial-send retry behavior. #53961 and #57755 can then consume the new outcome instead of treating hook cancellation as failure.
