## Summary

Keep this PR open. Current main still has the pre-PR outbound delivery contract: `deliverOutboundPayloads()` returns `OutboundDeliveryResult[]`, hook cancellations collapse to empty results, non-bestEffort mid-stream failures rethrow without partial-send metadata, and gateway/cron callers still use array length as the delivery-status signal. The PR is active, focused, and not implemented or superseded on main.

## Best Possible Solution

Keep this PR open for maintainer review and either land it or implement the same contract on main: expose delivery outcome metadata from `src/infra/outbound/deliver.ts`, preserve partial-send metadata for non-bestEffort failures, update all result-consuming callers, and keep tests for hook cancellation, gateway status, cron retry behavior, and partial-send replay. #53961 and #57755 can then consume the new outcome instead of treating hook cancellation as failure.
