## Summary

Keep open. Current main still does not implement this PR's intended proxy-fetch behavior: both proxy wrappers still pass caller bodies directly into `undici.fetch` while only adding a dispatcher. A separate runtime FormData normalizer exists, but it is not wired into `makeProxyFetch()` or `resolveProxyFetchFromEnv()`, and the affected media path still passes an env-proxy fetch function together with a global `FormData` upload body. The linked same-author bug #48554 remains open, and overlapping PR #66243 is also open, so this PR is still a live implementation candidate rather than cleanup-closeable.

## Best Possible Solution

Keep this PR open for maintainer review, or supersede it with a current-main patch. The best implementation is to normalize global FormData in both `makeProxyFetch()` and `resolveProxyFetchFromEnv()`, preferably sharing the existing runtime normalizer, clearing stale multipart headers, adding targeted proxy-wrapper tests for conversion and passthrough, and coordinating the result with #48554 and #66243.
