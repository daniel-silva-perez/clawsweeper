## Summary

Keep open. Current main still does not implement this PR's intended proxy-fetch behavior: `makeProxyFetch()` and `resolveProxyFetchFromEnv()` still forward caller `init.body` directly to `undici.fetch` while only adding a dispatcher. The separate runtime FormData normalizer exists, but the affected explicit proxy-fetch path can bypass it when a non-ambient fetch implementation is supplied. The linked same-author bug #48554 is still open in the supplied GitHub context, and overlapping PR #66243 is also open rather than merged or maintainer-designated as the canonical replacement.

## Best Possible Solution

Keep this PR open for maintainer review, or explicitly supersede it with a merged/current-main patch coordinated with #48554 and #66243. The best implementation belongs in `src/infra/net/proxy-fetch.ts`: share or align with the runtime FormData normalizer, apply it to both proxy wrappers, clear stale multipart headers, preserve filenames deliberately, and add targeted proxy-wrapper regression tests. After that lands, close the linked bug and any superseded PRs with the merged commit as canonical evidence.
