## Summary

Keep this PR open. Current main still does not implement the intended proxy-fetch FormData normalization: both proxy wrappers spread `init` directly into `undici.fetch`, while the existing FormData normalizer only runs through the separate runtime dispatcher path. The same-author linked bug #48554 is open, maintainer discussion there explicitly left the issue open, and related PR #66243 is also open rather than merged or designated as the replacement.

## Best Possible Solution

Keep this PR open for maintainer review, or explicitly supersede it only after a merged/current-main patch coordinated with #48554 and #66243. The best implementation belongs in `src/infra/net/proxy-fetch.ts`: share or align with the runtime FormData normalizer, apply it to both `makeProxyFetch()` and `resolveProxyFetchFromEnv()`, preserve filenames deliberately, clear stale multipart framing headers when conversion occurs, and add targeted proxy-wrapper regression tests. Then close the linked bug and any superseded PRs with the merged commit as canonical evidence.
