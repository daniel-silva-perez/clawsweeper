## Summary

Keep open. Current main still does not implement this PR's intended proxy-fetch behavior: `makeProxyFetch()` and `resolveProxyFetchFromEnv()` still forward `init.body` directly to `undici.fetch` while only overriding the dispatcher. A separate runtime FormData normalizer exists, but the proxy wrapper does not call it, and the affected media path still builds global `FormData` and passes a proxy fetch when proxy env vars are set. The linked same-author bug report #48554 remains open, and broader overlapping PR #66243 is also open, so this PR is still a live implementation candidate rather than cleanup-closeable.

## Best Possible Solution

Keep this PR open for maintainer review or supersede it with a current-main patch. The best implementation is to normalize FormData in both `makeProxyFetch()` and `resolveProxyFetchFromEnv()`, preferably by sharing or aligning with the existing runtime FormData normalizer, clearing stale multipart headers, adding targeted proxy-wrapper tests for global FormData conversion and Undici FormData passthrough, and coordinating the result with #48554 and #66243.
