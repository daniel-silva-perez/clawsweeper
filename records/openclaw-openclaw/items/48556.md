## Summary

Keep this PR open. Current main still does not implement the PR's intended proxy-fetch FormData normalization: both proxy wrappers pass `init.body` directly into `undici.fetch`, while the existing FormData normalizer is limited to the separate runtime dispatcher path. The linked same-author bug #48554 is still open, and related PR #66243 is also open rather than merged or maintainer-designated as the replacement.

## Best Possible Solution

Keep this PR open for maintainer review, or explicitly supersede it with a merged/current-main patch coordinated with #48554 and #66243. The best implementation belongs in `src/infra/net/proxy-fetch.ts`: share or align with the runtime FormData normalizer, apply it to both `makeProxyFetch()` and `resolveProxyFetchFromEnv()`, clear stale multipart headers, preserve filenames intentionally, and add targeted proxy-wrapper regression tests. After that lands, close the linked bug and any superseded PRs with the merged commit as canonical evidence.
