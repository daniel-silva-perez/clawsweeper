## Summary

Keep open. Current main has explicit Claude CLI routing through `claude-cli/*` model refs or `agentRuntime.id: "claude-cli"`, but the requested `cliBackends`-only contract is still not implemented for PI/subagent dispatch. Because changing that would alter transport and billing behavior for canonical Anthropic models, this needs maintainer product judgment rather than cleanup closure.

## What This Changes

Request PI/subagent dispatch to honor configured Claude CLI backends so eligible Anthropic OAuth calls use the same CLI transport as chat instead of direct API billing.

## Best Possible Solution

Define one explicit contract and make it consistent: either add doctor/docs warnings for Claude CLI configs that set only `cliBackends` without `agentRuntime.id`, or implement a deliberate opt-in runtime policy that routes eligible Anthropic PI/subagent/model paths through the configured Claude CLI backend with focused regression tests.

## Reproduction Assessment

Yes for the OpenClaw routing behavior. With `agents.defaults.model.primary` set to `anthropic/...`, `agents.defaults.cliBackends.claude-cli` configured, and no `agentRuntime.id`, the inspected source keeps the execution provider as `anthropic` and falls through to embedded PI; live billing proof is not required to reproduce that branch.

## Solution Assessment

Unclear. Inferring CLI transport from `cliBackends` alone may surprise existing configs because current docs use `agentRuntime.id` as the execution selector; a safer fix may be an explicit opt-in or a doctor warning, depending on maintainer intent.
