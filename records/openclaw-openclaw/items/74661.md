## Summary

Keep open. Current main still has the Feishu empty-text path: `{"text":""}` parses to an empty `ctx.content`, media resolution returns an empty list for text messages, and the handler proceeds toward session recording and dispatch without a no-text/no-media guard. The PR is a narrow Feishu-plugin fix for the paired bug and I did not find a blocking correctness issue in the proposed patch.

## What This Changes

The PR adds a Feishu message-handler guard that skips empty text with no resolved media, plus a regression test and changelog entry for the Feishu blank-turn bug.

## Best Possible Solution

Review and land the narrow Feishu-plugin guard with its regression test and changelog entry, keeping the behavior owned by the Feishu channel rather than moving provider-specific empty inbound-message policy into core.
