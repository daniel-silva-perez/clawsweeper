## Summary

Keep open. Current main still has adjacent loop detection, channel/group tool allow/deny policy, and explicit `sessions_spawn`, but it does not implement the requested channel-scoped max-per-turn tool-call budget or the `warn`/`block`/`auto-delegate` overflow behavior; the latest related maintainer-facing discussion also kept this open for design follow-up.

## What This Changes

Add per-channel tool-call budgets that enforce per-turn limits and support warn, block, or auto-delegate overflow behavior for channel-specific resource constraints.

## Best Possible Solution

Define a generic scoped tool-call budget model that composes with existing global, agent, provider, channel/group, sandbox, and subagent policies. Ship deterministic warn/block enforcement first in the trusted before-tool path, then design auto-delegation separately around `sessions_spawn`, child-run tracking, sandbox/tool inheritance, and completion handoff.
