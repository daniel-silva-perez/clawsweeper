## Summary

Keep open. Current main still has adjacent loop detection, channel/group allow-deny tool policy, plugin before-tool hooks, and explicit subagent spawning, but it does not implement a channel-scoped max-per-turn tool-call budget or the requested warn/block/auto-delegate overflow behavior. The related discussion also kept this open for design follow-up; #9912 is broader/global and still open, while #14373 is closed but not a complete substitute.

## What This Changes

Request a per-channel tool-call budget that can warn, block, or auto-delegate excess tool work to a subagent.

## Best Possible Solution

Define a generic scoped tool-call budget model that resolves alongside the existing effective tool-policy pipeline. Implement deterministic warn and block behavior first in the trusted before-tool-call path with run-scoped counters, then design auto-delegation separately around sessions_spawn with explicit sandbox/tool inheritance, target-agent policy, child-run tracking, and completion handoff.
