## Summary

Keep open. Current main still has only adjacent loop detection, channel/group allow-deny tool policy, plugin before-tool hooks, and explicit `sessions_spawn`; it does not implement a channel-scoped max-per-turn tool-call budget or the requested `warn`, `block`, and `auto-delegate` overflow behavior. Recent review comments and adjacent open delegation work also treat this as unresolved design work rather than cleanup closure.

## What This Changes

Request a gateway-enforced per-channel tool-call budget with max-per-turn limits and overflow actions for warn, block, and auto-delegate.

## Best Possible Solution

Define a generic scoped tool-call budget model that resolves alongside effective tool policy. Land deterministic run-scoped `warn` and `block` enforcement in the trusted before-tool path first, then design `auto-delegate` separately around `sessions_spawn` authority inheritance, child-run tracking, and completion handoff.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The high-confidence current-main check is source, schema, docs, and test inspection showing that the requested config keys and enforcement paths are absent.

## Solution Assessment

Unclear. A per-channel budget is a plausible direction, but bundling warning, blocking, and policy-triggered subagent creation into one first change is not the safest maintainable path; deterministic budget enforcement should precede automatic delegation.
