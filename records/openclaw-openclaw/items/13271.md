## Summary

Keep open. Current main has adjacent loop detection, per-channel/group tool allow/deny policy, plugin before-tool hooks, and explicit subagent spawning, but it still does not provide a channel-scoped max-per-turn tool-call budget or the requested warn/block/auto-delegate overflow behavior. The existing comments also keep this open for maintainer design rather than treating #9912 or the closed #14373 as a complete substitute.

## What This Changes

Request a per-channel tool-call budget configuration that limits tool calls per turn and supports warn, block, or auto-delegate overflow behavior for channels such as Discord.

## Best Possible Solution

Define a generic scoped tool-call budget model that resolves alongside the existing effective tool-policy pipeline. Ship deterministic warn and block behavior first in the trusted before-tool-call path, then design auto-delegation separately around sessions_spawn with explicit sandbox/tool inheritance, target-agent policy, child-run tracking, and completion handoff.
