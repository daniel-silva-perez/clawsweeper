## Summary

Keep open. Current main has related pieces, including explicit `sessions_spawn`, loop detection, plugin before-tool hooks, and channel/group tool allow/deny policy, but it still does not implement the requested per-channel max-per-turn tool budget, `onExceed` warn/block/auto-delegate policy, or automatic delegation after a budget is exceeded.

## Best Possible Solution

Keep this open for maintainer design. The best path is to align with #9912 first, add a generic scoped tool-budget policy with explicit global/agent/channel/group precedence, implement deterministic warn/block semantics in the trusted before-tool-call path, and then design auto-delegation separately around the existing `sessions_spawn`/subagent registry and completion-delivery model.
