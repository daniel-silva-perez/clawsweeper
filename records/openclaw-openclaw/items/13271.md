## Summary

Keep open. Current main still has adjacent loop detection, channel/group allow/deny policy, plugin tool hooks, and explicit `sessions_spawn`, but it does not implement a channel-scoped max-per-turn tool budget or the requested `warn`, `block`, and `auto-delegate` overflow actions. The recent maintainer and ClawSweeper comments also treated this as the canonical design follow-up, not cleanup closure.

## What This Changes

Request a configurable per-channel tool-call budget that can warn, block, or auto-delegate once a channel-specific per-turn limit is exceeded.

## Best Possible Solution

Define a generic scoped tool-call budget model that resolves alongside effective tool policy. Ship deterministic run-scoped `warn` and `block` behavior in the trusted before-tool path first, then design `auto-delegate` as a separate extension of `sessions_spawn` with explicit authority inheritance, child-run tracking, and completion handoff semantics.
