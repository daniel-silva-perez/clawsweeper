## Summary

Keep open. Current main still has adjacent loop detection, channel/group tool allow-deny policy, plugin before-tool hooks, and explicit `sessions_spawn`, but it does not implement a channel-scoped max-per-turn tool-call budget or the requested `warn`, `block`, and `auto-delegate` overflow behaviors. The recent discussion also treats this as unresolved maintainer design work, not stale or already implemented.

## What This Changes

Request a gateway-enforced per-channel tool-call budget with warn, block, and auto-delegate overflow behavior for channel-specific resource limits such as Discord.

## Best Possible Solution

Define a generic scoped tool-call budget resolver that composes with effective tool policy. Ship deterministic run-scoped `warn` and `block` enforcement in the trusted before-tool path first, then design `auto-delegate` separately around `sessions_spawn` authority inheritance, child-run tracking, sandbox/tool inheritance, and completion handoff.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The high-confidence current-main check is source, schema, docs, and test-surface inspection showing the requested config keys and enforcement paths are absent.

## Solution Assessment

Unclear. A per-channel budget is plausible, but combining budget enforcement and policy-triggered subagent creation into one first change is not the safest maintainable path; deterministic budget enforcement should come before automatic delegation.
