## Summary

Keep open. Current main still has adjacent loop detection, channel/group tool allow/deny policy, and explicit `sessions_spawn`, but it does not implement a channel-scoped max-per-turn tool-call budget or the requested `warn`/`block`/`auto-delegate` overflow behavior; the existing maintainer-facing review also kept this open for design follow-up.

## What This Changes

Request channel-scoped per-turn tool-call limits with warn, block, or auto-delegate overflow behavior for multi-channel agents.

## Best Possible Solution

Define a generic scoped tool-call budget model that composes with the existing tool-policy pipeline. Ship deterministic `warn` and `block` enforcement first in the trusted before-tool-call path, then design `auto-delegate` separately around `sessions_spawn`, child-run tracking, sandbox/tool inheritance, and completion handoff.
