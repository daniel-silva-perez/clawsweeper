## Summary

Keep open. Current main still has adjacent loop detection, channel/group allow-deny tool policy, plugin before-tool hooks, and explicit `sessions_spawn`, but it does not implement channel-scoped max-per-turn tool-call budgets or the requested `warn`, `block`, and `auto-delegate` overflow behavior. The existing discussion also treats this as unresolved maintainer design work rather than a stale or already-implemented request.

## What This Changes

Request a per-channel tool-call budget config with max-per-turn limits and overflow modes that warn, block, or auto-delegate remaining work to a subagent.

## Best Possible Solution

Define a generic scoped tool-call budget resolver that composes with effective tool policy. Ship deterministic run-scoped `warn` and `block` enforcement in the trusted before-tool path first, then design `auto-delegate` separately around `sessions_spawn` authority inheritance, child-run tracking, and completion handoff.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The high-confidence current-main check is source, schema, docs, and test-surface inspection showing that the requested config keys and enforcement paths are absent.

## Solution Assessment

Unclear. A per-channel budget is a plausible direction, but combining budget enforcement and policy-triggered subagent creation into one first change is not the safest maintainable path; deterministic budget enforcement should come before automatic delegation.
