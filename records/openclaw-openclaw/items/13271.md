## Summary

Keep open. Current main still has adjacent loop detection, channel/group tool allow/deny policy, plugin before-tool hooks, and explicit `sessions_spawn`, but it does not implement a channel-scoped max-per-turn tool-call budget or the requested `warn`, `block`, and `auto-delegate` overflow behaviors. The recent issue discussion also treats this as unresolved maintainer design work.

## What This Changes

Request a per-channel tool-call budget that can warn, block, or automatically delegate overflow work to a sub-agent after a configured per-turn limit.

## Best Possible Solution

Define a generic scoped tool-call budget resolver that composes with existing effective tool policy. Ship deterministic run-scoped `warn` and `block` enforcement in the trusted before-tool path first, then design `auto-delegate` separately around `sessions_spawn` authority inheritance, child-run tracking, sandbox/tool inheritance, and completion handoff.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The high-confidence current-main check is source, schema, docs, and test-surface inspection showing the requested config keys and enforcement path are absent.

## Solution Assessment

Unclear. Per-channel budgeting is plausible, but combining budget enforcement and automatic subagent creation in the first change is not the safest maintainable path; deterministic budget enforcement should come before auto-delegation.
