## Summary

Keep open. Current main has related mechanisms, including repetitive tool-loop detection, trusted/plugin before-tool hooks, channel/group tool allow/deny policy, and explicit `sessions_spawn`, but it still does not implement the requested per-channel max-per-turn tool budget, `onExceed` warn/block/auto-delegate policy, or automatic delegation after a budget is exceeded.

## What This Changes

Add per-channel tool-call budgets with max-per-turn limits and warn/block/auto-delegate overflow behavior, including automatic sub-agent delegation after a channel budget is exceeded.

## Best Possible Solution

Keep this open for maintainer design. Align with #9912 first, define a generic scoped tool-call budget policy with clear global/agent/channel/group precedence, implement deterministic warn/block behavior in the trusted before-tool-call path, and design auto-delegation separately around `sessions_spawn`, subagent registry, sandbox/tool policy, and completion delivery.
