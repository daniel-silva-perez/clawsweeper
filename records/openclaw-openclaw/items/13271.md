## Summary

Keep open. Current main still has adjacent tool-loop detection, effective tool allow/deny policy, plugin before-tool hooks, and explicit `sessions_spawn`, but it does not implement channel-scoped per-turn tool-call budgets or the requested `warn`/`block`/`auto-delegate` overflow behavior. The recent maintainer-facing review context also kept this open for design follow-up, and the remaining work is not fully covered by the open global budget request or the closed inline-delegation request.

## What This Changes

The issue requests per-channel tool-call limits with configurable warn, block, or automatic subagent delegation when a channel exceeds its budget.

## Best Possible Solution

Define a generic scoped tool-call budget model that resolves alongside the existing effective tool-policy pipeline. Ship deterministic warn/block behavior first in the trusted before-tool-call path with run-scoped counters, then design auto-delegation as a separate, explicit extension of `sessions_spawn` authority and completion handoff.
