## Summary

Keep open. Current main has adjacent loop detection, channel/group tool allow/deny policy, plugin before-tool hooks, and explicit `sessions_spawn`, but it still does not implement a channel-scoped max-per-turn tool-call budget or the requested `warn`, `block`, and `auto-delegate` overflow actions. Recent maintainer/ClawSweeper discussion also treated this as an open design follow-up rather than cleanup closure.

## What This Changes

The issue requests a per-channel max-per-turn tool-call budget with `warn`, `block`, and `auto-delegate` overflow handling for multi-channel agents.

## Best Possible Solution

Define a generic scoped tool-call budget model that resolves alongside effective tool policy. Ship deterministic run-scoped `warn` and `block` enforcement in the trusted before-tool path first, then design `auto-delegate` separately around `sessions_spawn` authority inheritance, child-run tracking, and completion handoff.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The high-confidence current-main check is source/config verification that the requested config keys and enforcement paths are absent.

## Solution Assessment

Unclear. A per-channel budget is a plausible direction, but bundling `warn`, `block`, and automatic delegation into one change is not the safest first implementation; deterministic budget enforcement should land before policy-triggered subagent creation.
