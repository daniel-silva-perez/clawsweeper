## Summary

Keep open. Current main has adjacent mechanisms for repetitive tool-loop detection, channel/group tool allow/deny policy, trusted/plugin before-tool hooks, and explicit `sessions_spawn`, but it still lacks the requested channel-scoped max-per-turn tool-call budget, `onExceed` warn/block/auto-delegate behavior, and automatic delegation after overflow. The existing maintainer-facing discussion also explicitly kept this open for design follow-up.

## What This Changes

Request a gateway-enforced per-channel tool-call budget with configurable warn, block, or auto-delegate behavior when a turn exceeds the channel limit.

## Best Possible Solution

Define one generic scoped tool-call budget model that composes with existing global, agent, provider, channel/group, sandbox, and subagent tool policy. Ship deterministic warn/block behavior first in the trusted before-tool path, then separately design auto-delegation around `sessions_spawn`, child-run registry, sandbox/tool inheritance, and completion handoff.
