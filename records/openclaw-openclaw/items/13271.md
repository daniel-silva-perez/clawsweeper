## Summary

Keep open. Current `main` still has adjacent loop detection, channel/group tool allow/deny policy, plugin `before_tool_call` hooks, and explicit `sessions_spawn`, but it does not implement a channel-scoped max-per-turn tool-call budget or the requested `warn`/`block`/`auto-delegate` overflow behavior. The existing discussion already kept this open for maintainer design follow-up, and the related open global limit request does not supersede the channel-scoped feature.

## What This Changes

Request a per-channel tool-call budget configuration that can warn, block, or auto-delegate work to a sub-agent after a configured per-turn limit is exceeded.

## Best Possible Solution

Define a generic scoped tool-call budget model that composes with the existing tool-policy pipeline. Land deterministic `warn` and `block` enforcement first in the trusted before-tool-call path, then design `auto-delegate` separately around `sessions_spawn`, child-run tracking, sandbox/tool inheritance, and completion handoff.
