## Summary

Keep open. Current main still lacks channel-scoped max-per-turn tool-call budgets and the requested `warn`, `block`, and `auto-delegate` overflow behavior; the implementation has adjacent loop detection, tool allow/deny policy, and explicit subagent spawning only. The related discussion also treats this issue as the canonical design follow-up rather than cleanup closure.

## What This Changes

Request a per-channel tool-call budget that can warn, block, or automatically delegate to a sub-agent after a per-turn limit is exceeded.

## Best Possible Solution

Define a generic scoped tool-call budget model that resolves alongside effective tool policy. Ship deterministic run-scoped `warn` and `block` behavior first in the trusted before-tool path, then design `auto-delegate` as a separate extension of `sessions_spawn` with explicit authority inheritance, child-run tracking, and completion handoff semantics.
