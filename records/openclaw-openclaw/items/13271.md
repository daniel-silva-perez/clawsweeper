## Summary

Keep open. Current main has explicit `sessions_spawn`, per-agent/global repetitive tool-loop detection, and channel/group tool allow/deny policy, but it still does not provide a per-channel max-per-turn tool-call budget, `onExceed` policies, or automatic delegation after a budget is exceeded.

## Best Possible Solution

Keep this open for maintainer design. A good path is to align with #9912 first, add a generic scoped tool-budget policy with clear global/agent/channel/group precedence, implement deterministic warn/block semantics, then separately design auto-delegation around existing `sessions_spawn`/subagent limits and result delivery.

## Risks / Open Questions

- Config precedence needs product design because the request overlaps global #9912, per-agent `tools.loopDetection`, channel/group tool policy, and subagent policy.
- `warn` and `block` behavior must preserve provider transcript validity and tool-result replay semantics across embedded runner and channel-routed sessions.
- Automatic delegation needs explicit remaining-task construction, child limits, timeout/concurrency controls, and a reliable result-delivery path back to the original channel.
- A ClawHub-only plugin would not fully cover this as requested because the current core hook path does not expose a first-class per-channel budget policy or built-in overflow delegation seam.
