## Summary

Keep open. Current main has hardened the raw Telegram HTML path, including escaping raw model HTML and retrying plain text on parse errors, but it still intentionally hides text after an unclosed reasoning tag when visible text already exists. The issue also has an open same-author implementation candidate, #49996, with closing syntax, so this should remain open until maintainers review/land or replace that fix.

## Best Possible Solution

Keep this issue open and review #49996, or replace it with a narrower maintainer-owned patch that defines the strict-mode policy for unclosed reasoning tags, updates `stripReasoningTagsFromText` and Telegram reasoning split coverage, and separately tracks the requested non-2xx Telegram send logging improvement if maintainers want that defense-in-depth change.
