## Summary

Keep open. Current main has hardened raw Telegram HTML rendering and retry behavior, but the remaining reported unclosed-reasoning-tag tail drop is still the documented and tested strict-mode behavior. The issue also has an open same-author implementation candidate, #49996, with closing syntax, so it should not be closed before that PR is reviewed, landed, or replaced.

## Best Possible Solution

Keep this issue open and review #49996, or replace it with a maintainer-owned patch that explicitly defines the strict-mode policy for unclosed reasoning tags, updates `stripReasoningTagsFromText` and Telegram reasoning-split coverage, and keeps the public thinking docs aligned. The non-2xx Telegram send logging improvement raised in discussion should be split into a narrower follow-up if maintainers want that defense-in-depth change.
