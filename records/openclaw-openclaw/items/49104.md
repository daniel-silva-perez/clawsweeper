## Summary

Keep open. Current main escapes raw Telegram HTML and retries plain text on HTML parse failures, but the remaining reported tail-drop path is still present by policy: strict reasoning stripping hides text after an unclosed `<think>` when visible text already exists. There is also an open same-author implementation candidate, #49996, with closing syntax for this issue, so this should stay open until that PR is reviewed, landed, or replaced.

## Best Possible Solution

Keep this issue open and review #49996, or replace it with a maintainer-owned patch that explicitly defines the strict-mode policy for unclosed reasoning tags, updates `stripReasoningTagsFromText` plus Telegram reasoning split coverage, and keeps `docs/tools/thinking.md` aligned. The non-2xx Telegram send logging improvement raised in discussion should be split into a narrower follow-up if maintainers want that defense-in-depth change.
