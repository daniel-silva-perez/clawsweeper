## Summary

Keep this issue open. Current main still documents and displays a Nextcloud Talk bot install command without `--feature response`, while the send path still explains HTTP 401 only as a bot-secret problem. Upstream Nextcloud Talk stable32 requires `FEATURE_RESPONSE` for bot message sends, and the two related fix PRs were closed unmerged, so there is no implemented or shipped fix to close against.

## What This Changes

Request that Nextcloud Talk onboarding/docs and 401 send errors document that bots need `--feature response` to post replies.

## Best Possible Solution

Update the Nextcloud Talk docs and setup wizard to register bots with `--feature webhook --feature response --feature reaction`, and make the 401 send error mention both bot-secret mismatch and missing `--feature response`, with focused regression tests for the setup text and diagnostic.
