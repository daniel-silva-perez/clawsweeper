## Summary

Keep this issue open. Current main still documents and displays a Nextcloud Talk bot install command without `--feature response`, while the outbound send path still maps HTTP 401 only to a bot-secret problem. Upstream Nextcloud Talk stable32 requires the response feature for bot message sends, both related fix PRs are closed unmerged, and no open PR currently references this issue as an implementation candidate.

## What This Changes

Request to document the required Nextcloud Talk `--feature response` bot flag and improve the misleading HTTP 401 send diagnostic.

## Best Possible Solution

Update the Nextcloud Talk docs and setup wizard to register bots with `--feature webhook --feature response --feature reaction`, and make the 401 send error mention both bot-secret mismatch and missing `--feature response`, with focused regression tests for setup text and the diagnostic.
