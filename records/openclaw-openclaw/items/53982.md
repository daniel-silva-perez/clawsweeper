## Summary

Keep this issue open. Current main still documents and displays a Nextcloud Talk bot install command without `--feature response`, and the outbound send path still maps HTTP 401 only to a bot-secret diagnostic. The related fix PRs in the provided context were closed unmerged, so there is no shipped implementation to close against.

## Best Possible Solution

Land a narrow Nextcloud Talk plugin/docs fix: update `docs/channels/nextcloud-talk.md` and `extensions/nextcloud-talk/src/setup-surface.ts` to show `--feature webhook --feature response --feature reaction`, update `extensions/nextcloud-talk/src/send.ts` so 401 errors mention both bot-secret mismatch and missing response feature, and add focused tests for the setup text or 401 diagnostic.
