## Summary

Current main does not implement this PR’s intended behavior. `sessions_send` still starts the A2A announce/send flow, assistant-visible suppression is still phase-only, and the sanitizer has no self-instruction prelude stripping. The PR is useful but needs review/rework because review comments identify route-persistence and sanitizer risks in the proposed patch.

## Best Possible Solution

Keep this PR open for maintainer review or a revised patch. The best fix should keep `sessions_send` session-local when appropriate, avoid starting the announce/send flow unless explicitly intended, suppress current-turn external delivery without overwriting persisted external routes, and use structured tool-use/phase signals before adding any broad sanitizer heuristic. Address the existing P1/P2 review comments and cover both route preservation and no-announcement behavior with targeted tests.
