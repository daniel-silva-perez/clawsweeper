## Summary

Keep open. Current main still does not accept `agentId` under `channels.discord.guilds.<guild>.channels.<channel>`, and the implemented guidance/repair paths only cover top-level `bindings[]`, channel-local ACP binding guidance, and a separate Discord `allow` alias migration. The related Telegram topic `agentId` fix does not resolve this Discord config surface.

## Best Possible Solution

Keep this issue open and fix the Discord config UX deliberately: either support `channels.discord.guilds.<guild>.channels.<channel>.agentId` as a Discord-owned shorthand that routes through the same binding semantics, or add a targeted validation/doctor rule that detects it and points to or rewrites into a top-level `bindings[]` entry with `match.channel: "discord"` and `peer.kind: "channel"`. Add regression coverage for raw config validation and doctor/fix behavior, plus a small docs note if the field remains unsupported.
