## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve synthetic final payload before retrying:** `src/auto-reply/reply/dispatch-acp-replay.ts:352-364`
  - body: If `recordProjectorPreparedSyntheticFinal()` throws after `resolveSyntheticFinalPayload()` builds the final TTS payload, the retry starts with a fresh coordinator and skips previously checkpointed block effects, so `resolveSyntheticFinalPayload()` returns null and final TTS is never delivered. Persist or otherwise reconstruct the payload before retries can skip the block state, and add a regression for the prepare-write failure path.
  - confidence: 0.83
