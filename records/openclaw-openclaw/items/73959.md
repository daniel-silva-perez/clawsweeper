## Summary

Keep this PR open. Current main still lacks the requested Bedrock override surface and the linked bug remains open, but the proposed patch has a blocking correctness defect: regex overrides are not tested against stripped inference-profile IDs, so one of the new scenarios falls back to the default context window.

## What This Changes

The PR adds an Amazon Bedrock `modelContextWindowOverrides` discovery config with regex-based per-model context-window fallbacks, cache-key handling, tests, manifest metadata, docs, and an exported config type update.

## Best Possible Solution

Land one canonical Bedrock context-window fix. If this PR is the chosen direction, make regex overrides use the same full, stripped, and version-normalized candidates as the built-in lookup, keep deterministic cache keys, and align manifest/docs/tests around that behavior.

## Reproduction Assessment

Yes. The PR's own new inference-profile test is a high-confidence reproduction path: with no base ARN, `us.example...` plus `^example\.` cannot match because the implementation tests only the prefixed `modelId`.

## Solution Assessment

No, not as written. The configurable override approach is plausible, but the implementation misses the stripped-ID path it documents and tests; maintainers should also choose whether this or #73382 is the preferred policy.

## Review Findings

Overall correctness: patch is incorrect
