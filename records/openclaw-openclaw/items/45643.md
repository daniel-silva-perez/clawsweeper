## Summary

Keep this PR open. Current main has adjacent Brave llm-context validation/docs and broader restart-deferral work, but it still does not implement several central PR changes: the macOS wizard challenge timeout is still 0.75s, persisted Control UI nav width is still capped at 400px, skill-creator still omits the direct quick_validate.py audit workflow, there is no automatic pre-reset memory-flush hook, and browser restart deferral is not wired to active browser profile work.

## Best Possible Solution

Keep this PR open for maintainer/contributor action, but split or rebase it into focused PRs: one for the macOS wizard timeout, one for Control UI nav width/storage bounds if that UX is still desired, one for skill-creator quick_validate documentation, and a separately reviewed reset-memory hook that addresses the existing review comments. Browser restart deferral should be redesigned against the current extensions/browser plugin service or a generic task/work-count seam instead of the obsolete src/browser/server.ts shape.
