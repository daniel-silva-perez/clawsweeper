## Summary

Keep this PR open. Current main has some adjacent work, especially Brave llm-context validation/docs and broader restart deferral counters, but it still does not implement several central PR changes: the macOS wizard challenge timeout remains 0.75s, persisted Control UI nav width remains capped at 400px, the skill-creator guide still omits the standalone quick_validate.py audit workflow, there is no automatic pre-reset memory-flush hook, and browser active profile state is not wired into restart deferral.

## Best Possible Solution

Keep this PR open for maintainer/contributor follow-up, but split or rebase it into focused changes: one for the macOS wizard timeout, one for Control UI nav width/storage bounds if that UX is still desired, one for skill-creator quick_validate documentation, and a separately reviewed reset-memory hook that addresses the existing review comments. Browser restart deferral should be redesigned against the current extensions/browser plugin service or a generic active-work/task seam rather than the obsolete src/browser/server.ts shape.
