## Summary

Keep open. Current main has some adjacent work, especially Brave llm-context validation/docs and generic restart deferral, but it does not definitely implement this PR’s intended bundle. The macOS wizard timeout is still 0.75s, Control UI nav width is still capped at 400px in persisted settings, skill-creator still does not document the direct quick_validate.py audit workflow, and I found no automatic pre-reset memory-flush hook or browser active-work count wired into gateway restart deferral.

## Best Possible Solution

Keep this PR open for maintainer/contributor action. The best path is to split or rebase it into narrower PRs: one for the macOS timeout, one for Control UI nav width/storage bounds, one for skill-creator quick_validate docs, and a separately reviewed reset-memory hook that addresses the existing review comments. Browser restart deferral should be revisited against the current browser plugin architecture instead of applying the old src/browser/server.ts shape.
