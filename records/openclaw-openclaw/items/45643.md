## Summary

Keep open. Current main has adjacent Brave llm-context validation/docs and an opt-in bundled session-memory hook, but it still does not implement several central PR changes: the macOS connect challenge timeout remains 0.75s, Control UI nav width is fixed/capped below the requested 600px behavior, skill-creator docs still do not expose quick_validate.py as a standalone audit workflow, there is no automatic pre-reset agentic memory flush hook, and active browser profile state is not wired into restart deferral.

## Best Possible Solution

Keep this PR open for maintainer or contributor follow-up, but split or rebase it into focused changes. The macOS timeout, Control UI nav behavior, and skill-creator docs can be small separate patches. The reset-memory behavior should be redesigned against the current hook/memory infrastructure and address the existing review comments. Browser restart deferral should integrate with the current extensions/browser plugin or a generic active-work/task seam rather than resurrecting obsolete core browser paths.
