Current main still lacks the requested `/total_usage` command, flexible range parsing, Telegram quick-select UI, and command-path model breakdown, so the branch is not obsolete; however, the proposed patch still has concrete correctness and performance regressions in shared usage aggregation/discovery that should block merge until addressed.

This PR adds a `/total_usage` chat/native command with hour/day/all ranges, Telegram quick-select buttons, cross-session usage/cost model aggregation, usage summary type changes, docs, and focused tests.

## Best Possible Solution

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Partition model totals by provider and model:** `src/infra/session-cost-usage.ts:423-424`
  - body: The new aggregate key is only `entry.model`, while the bucket stores a single provider from the first matching entry. If two providers emit the same model id, `/total_usage by-model` will merge their tokens/costs and label the row with whichever provider was seen first; use the existing provider-plus-model key shape used by session usage aggregation.
  - confidence: 0.9
- **[P2] Restore transcript prefiltering for range summaries:** `src/infra/session-cost-usage.ts:399-403`
  - body: Removing the `mtimeMs < sinceTime` prefilter makes narrow queries like `/total_usage 5h` enumerate and scan every historical transcript before `onEntry` drops old records. Long-lived installations can turn a small range command into a full-history I/O scan, so keep the lower-bound file filter for bounded ranges.
  - confidence: 0.84
- **[P2] Keep discovery filtering bounded by the requested start:** `src/infra/session-cost-usage.ts:514-516`
  - body: `discoverAllSessions()` is used by `sessions.usage` before the result is sorted, limited, and per-session summaries are loaded. Dropping the `startMs` mtime filter makes date-scoped list requests inspect old transcript files that cannot contribute usage, which can regress dashboard/API latency on large histories.
  - confidence: 0.78
- **[P3] Guard model percentages when total tokens are zero:** `src/auto-reply/reply/commands-session.ts:515`
  - body: The model distribution formatter divides by `summary.totals.totalTokens` without the guard used for the cache rate. A zero-token summary with model entries will render `NaN%`; return `0.0` when the denominator is zero.
  - confidence: 0.82
