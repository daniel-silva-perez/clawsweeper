## Summary

Keep open. Current main still does not provide `/total_usage`, so the PR is not obsolete, but the branch still has correctness and validation blockers around provider-aware model aggregation, bounded transcript filtering, discovery filtering, and typed fixtures.

## What This Changes

The PR adds a `/total_usage` chat command with hour/day/all range parsing, Telegram quick-select buttons, per-model aggregate usage output, usage summary type changes, focused tests, and slash-command documentation.

## Best Possible Solution

Revise this PR in place so `/total_usage` keeps provider-plus-model buckets, preserves bounded transcript prefiltering in both summary loading and discovery, updates all `CostUsageSummary` fixtures, and lands with the focused usage/command tests plus the normal changed gate.

## Reproduction Assessment

Yes. The PR diff itself reproduces the blockers: same-named models from different providers collide under the new model-only key, bounded range queries lose the current `mtimeMs` prefilters, and typed fixtures that still omit `models` conflict with the newly required summary field.

## Solution Assessment

No. The requested command is useful and belongs in this area, but the current branch is not the narrowest maintainable solution until it preserves the existing provider-aware aggregation and bounded filtering contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Partition model totals by provider and model:** `src/infra/session-cost-usage.ts:426-431`
  - body: The new aggregate key is only `entry.model`, while each bucket stores a single provider from the first matching entry. If two providers emit the same model id, `/total_usage by-model` will merge their tokens/costs and label the row with whichever provider was seen first; use the provider-plus-model key shape already used by `sessions.usage`.
  - confidence: 0.9
- **[P2] Restore transcript prefiltering for range summaries:** `src/infra/session-cost-usage.ts:399-402`
  - body: Removing the `mtimeMs < sinceTime` prefilter makes bounded queries such as `/total_usage 5h` enumerate and scan every historical transcript before `onEntry` drops old records. Long-lived installations can turn a small range command into a full-history I/O scan, so keep the lower-bound file filter for bounded ranges.
  - confidence: 0.84
- **[P2] Keep discovery filtering bounded by the requested start:** `src/infra/session-cost-usage.ts:497-500`
  - body: `discoverAllSessions()` no longer honors `params.startMs`, but `sessions.usage` still sorts and slices discovered sessions before loading per-session summaries. Out-of-range transcript files can therefore consume the limit and push in-range sessions out of the response/totals; restore the start-bound filter during discovery.
  - confidence: 0.78
- **[P2] Update CostUsageSummary fixtures:** `src/infra/session-cost-usage.types.ts:68`
  - body: Adding `models` as a required `CostUsageSummary` field changes existing typed fixtures. `commands-session-usage.test.ts` still returns summary objects without `models`, so the test-type lane needs those fixtures updated or the field made backward-compatible.
  - confidence: 0.78
