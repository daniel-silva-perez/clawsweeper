## Summary

Keep open. Current main still sends proactive WhatsApp phone targets as PN JIDs, while this PR is a focused implementation of the LID forward-mapping fix for the linked ghost-chat delivery bug; the only concrete merge blocker I found is the missing required changelog entry for a user-facing fix.

## What This Changes

The branch adds an authDir-aware WhatsApp outbound JID resolver for Baileys LID forward mapping files, threads authDir into createWebSendApi, and adds unit/integration coverage for mapped, unmapped, and no-authDir sends.

## Best Possible Solution

Land the focused WhatsApp plugin fix at the active send boundary, keep unmapped or no-authDir accounts on the existing PN fallback, add the required changelog entry, and track the separate re-link reverse-mapping problem independently if it still reproduces.

Yes. ## Solution Assessment

Yes, functionally. Resolving PN targets to LID in createWebSendApi using the listener authDir is the narrowest maintainable fix for text/media, polls, and composing presence, with legacy fallback preserved; the patch still needs the required changelog entry before merge.
