## Summary

Keep this PR open. The docs gap is still real on current main: source and schema bound `commands.bashForegroundMs` to 0..30000 ms, but the user-facing slash-command docs and generated help text still omit the maximum. The submitted PR is not merge-ready as-is because it adds a 2693-line block to `docs/gateway/configuration-reference.md`, while that page now explicitly sends command-catalog detail to the Slash Commands page.

## Best Possible Solution

Ask for a narrow docs update instead of merging the current diff. The best fix is to update `docs/tools/slash-commands.md` and `src/config/schema.help.ts` so `commands.bashForegroundMs` is documented as an integer from 0 to 30000 ms, default 2000, with `0` backgrounding immediately; regenerate/check the config docs baseline as needed. A concise note in `docs/gateway/config-channels.md` is reasonable, but avoid duplicating command catalog content in `docs/gateway/configuration-reference.md`.
