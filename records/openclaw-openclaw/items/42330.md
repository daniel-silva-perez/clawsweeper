## Summary

Keep this PR open. Current main covers the duplicate-webhook problem through the shared inbound dedupe path, but it does not implement the requested Google Chat reply-to default change and still drops ADDED_TO_SPACE before message processing. The PR also has unresolved security and behavior review findings, so the best path is a focused rewrite or split into narrower follow-ups rather than closing as implemented or merging as-is.

## Best Possible Solution

Keep the PR open for maintainer judgment, but do not merge it as-is. Preserve the shared inbound dedupe already on main, split any desired Google Chat reply-to default change into a small adapter/config/docs/test PR, and reconcile space-type/lifecycle payload handling with #58993 and #65058. Drop or rewrite the manual auth, logging, mention, media, and local dedupe changes unless they pass focused security and regression review.
