## Summary

Keep this PR open for maintainer follow-up. Current main partially covers the deduplication goal with a stronger shared inbound dedupe path and preserves local media handling, but it does not implement the PR's requested Google Chat reply-to default change or message/lifecycle handling. The PR also still carries unresolved security and behavior risks from its auth, webhook, mention, and media changes.

## Best Possible Solution

Keep this PR open for maintainer judgment, but do not merge it as-is. Preserve the shared inbound dedupe already on main, handle Google Chat reply-to default changes in a small targeted PR with tests and config/docs metadata updates, and reconcile lifecycle/space detection work with focused follow-ups #58993 and #65058. Any custom auth/webhook/media changes should be rewritten or dropped unless they pass a separate security review.
