## Summary

Keep this PR open. Current main already covers the deduplication part with a stronger shared inbound dedupe path, but it does not implement the PR’s Google Chat reply-to default change or add-on lifecycle handling, and the PR still has unresolved security and behavior risks from its auth/webhook/media changes.

## Best Possible Solution

Keep this PR open for maintainer judgment, but do not merge it as-is. Preserve the shared inbound dedupe already on main, review #58993 and #65058 as focused Google Chat follow-ups, and handle any replyToMode default change in a small PR with targeted tests plus config/docs metadata updates. Rewrite or drop the PR’s custom auth/webhook/media changes unless they pass a separate security review.
