## Summary

Keep this PR open. Current main implements the duplicate-webhook part through the shared inbound dedupe path, but it still does not implement the Google Chat default reply-to mode change, still ignores ADDED_TO_SPACE in the event processor, and still lacks the narrower Google Chat spaceType/singleUserBotDm handling covered by #58993. The PR also has unresolved security and regression review findings, so it should not be closed as implemented or merged as-is.

## Best Possible Solution

Keep the PR open for maintainer judgment, but do not merge it as-is. Preserve the shared inbound dedupe already on main, split any desired Google Chat reply-to default change into a focused adapter/config/docs/test PR, and reconcile payload lifecycle and space-type handling with #65058 and #58993. Drop or rewrite the manual auth, debug logging, mention, media, and local dedupe changes unless they pass focused security and regression review.
