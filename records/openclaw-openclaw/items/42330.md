## Summary

Keep this PR open. Current main covers the duplicate-webhook problem through the shared inbound dedupe path, but it does not implement the PR's stated Google Chat reply-to default change or end-to-end ADDED_TO_SPACE/lifecycle handling. The PR also has unresolved security and regression findings, so it should not be closed as implemented or merged as-is.

## Best Possible Solution

Keep this PR open for maintainer judgment, but do not merge it as-is. Preserve the shared inbound dedupe already on main, split any desired Google Chat reply-to default change into a focused adapter/config/docs/test PR, and handle lifecycle payload variants through the narrower #65058 path. The PR's auth, logging, mention, media, and local dedupe changes should be rewritten or dropped unless they pass focused security and regression review.
