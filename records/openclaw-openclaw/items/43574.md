## Summary

Keep this PR open. The reported Discord attachment bug still exists on current main, but the branch is not merge-ready because it misses the current `mediaReadFile` propagation path and puts coverage in a shared test file that no longer exists on main.

## What This Changes

The PR updates Discord `thread-reply` attachment parsing to use `media`, `path`, then `filePath`, forwards `mediaLocalRoots`, and adds alias and passthrough tests.

## Best Possible Solution

Land a current-main Discord extension fix that normalizes `thread-reply` media aliases with `media > path > filePath` precedence, propagates both `mediaLocalRoots` and `mediaReadFile` through the Discord action adapter/parser/runtime options, and covers the behavior in the active Discord extension tests.

## Reproduction Assessment

Yes. The linked bug gives concrete steps, and current main has a high-confidence static reproduction: `thread-reply` reads only `media`, while `send` reads `media`, `path`, and `filePath`.

## Solution Assessment

No, not as currently proposed. The alias fallback is the right narrow direction, but the PR needs a current-main repair because it omits `mediaReadFile` propagation and places tests in a removed shared test file.
