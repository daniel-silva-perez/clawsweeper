## Summary

Keep open. The Discord bug remains valid on current main, and this PR is pointed at the right narrow fix, but the branch is not merge-ready because it does not carry the existing `mediaReadFile` callback through the thread-reply path and its tests target a shared action test file that no longer exists on current main.

## What This Changes

The PR changes Discord `thread-reply` attachment parsing to accept `media`, then `path`, then `filePath`, forwards `mediaLocalRoots`, and adds alias precedence tests under the old shared channel action test file.

## Best Possible Solution

Land a current-main Discord extension fix that normalizes `thread-reply` attachment aliases with `media > path > filePath` precedence, carries both `mediaLocalRoots` and `mediaReadFile` through the adapter and guild-admin parser into the existing runtime, and moves coverage into the active Discord extension test suites.

## Reproduction Assessment

Yes. A static reproduction is high-confidence: call Discord `message(action="thread-reply")` with `path` or `filePath`; current main's parser reads only `media`, so the attachment is dropped before the runtime sends the message.

## Solution Assessment

No. The alias fallback is the right narrow direction, but this PR is not the best merge as proposed because it only forwards `mediaLocalRoots`, not `mediaReadFile`, and its tests are placed in a file removed from the active current-main Discord suite.
