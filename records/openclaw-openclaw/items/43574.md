## Summary

Keep this PR open. Current main still has the Discord thread-reply alias gap, and the paired bug remains open; the proposed direction is correct, but the branch needs a current-main update because the patch only forwards `mediaLocalRoots` while the current Discord send/runtime path also carries `mediaReadFile` for agent-scoped media reads.

This PR updates Discord `thread-reply` handling to read `media`, `path`, and `filePath` aliases, forwards media roots to the Discord action runtime, and adds tests for alias precedence and media passthrough.

## Best Possible Solution
