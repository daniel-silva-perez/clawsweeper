## Summary

Keep this PR open. Current main still ignores `path` and `filePath` for Discord `thread-reply`, while the sibling Discord `send` action already supports those aliases; the PR is also paired with open same-author bug #43570, so it remains an implementation candidate rather than cleanup.

## Best Possible Solution

Keep this PR open and update it against current main. The best fix is narrow: in `extensions/discord/src/actions/handle-action.guild-admin.ts`, make `thread-reply` read `media ?? path ?? filePath`, forward the same media access options expected by the Discord send/runtime path, and add current-location Discord tests for `filePath`, `path`, precedence, and media option passthrough before landing it with #43570.
