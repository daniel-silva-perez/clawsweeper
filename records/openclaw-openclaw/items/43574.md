## Summary

Keep open. Current main still drops `path` and `filePath` for Discord `thread-reply`, so the PR targets a valid attachment bug, but the branch is not merge-ready because it is stale against the current extension test layout and does not propagate the current `mediaReadFile` callback needed for host-mediated file reads.

## What This Changes

The PR branch changes Discord `thread-reply` attachment parsing to read `media`, then `path`, then `filePath`, forwards `mediaLocalRoots` into the runtime call, and adds alias precedence tests under the old shared action test path.

## Best Possible Solution

Land a current-main Discord extension fix that normalizes `thread-reply` media aliases with the same precedence as `send`, propagates both `mediaLocalRoots` and `mediaReadFile` through the Discord action adapter/parser/runtime options, and covers alias precedence plus callback propagation in the current extension test suite.

## Reproduction Assessment

Yes. The linked bug gives concrete steps, and current main source has a high-confidence static reproduction: `thread-reply` reads only `media`, while `send` reads `media`, `path`, and `filePath`.

## Solution Assessment

No, not as currently proposed. The alias fallback direction is the right narrow fix, but the PR needs a current-main repair because it omits `mediaReadFile` propagation and places tests in a removed shared test file.
