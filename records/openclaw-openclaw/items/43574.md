## Summary

Keep this PR open. Current main still has the Discord thread-reply attachment gap: the guild-admin handler only reads `media`, while the sibling Discord send path already accepts `media`, `path`, and `filePath`. The PR is also paired with the same-author open bug #43570, so it remains a live implementation candidate rather than cleanup.

## Best Possible Solution

Keep this PR open and adapt it to current main. The narrow fix belongs in `extensions/discord/src/actions/handle-action.guild-admin.ts`: make `thread-reply` read `media ?? path ?? filePath`, pass the same media access options expected by the runtime, and add current-location Discord tests for `filePath`, `path`, precedence, and media access passthrough before landing it with #43570.
