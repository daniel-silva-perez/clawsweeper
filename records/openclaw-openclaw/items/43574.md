## Summary

Keep this PR open. The reported Discord thread-reply attachment bug is still present on current main, and the PR is useful but not merge-ready because it misses part of the current media access contract and targets an obsolete test path.

## What This Changes

The PR updates Discord `thread-reply` attachment handling to accept `media`, `path`, and `filePath`, passes `mediaLocalRoots` into the Discord action runtime, and adds alias precedence tests.

## Best Possible Solution

Land a current-main Discord extension fix that normalizes `media ?? path ?? filePath` for `thread-reply`, forwards both `mediaLocalRoots` and `mediaReadFile` into the action runtime, covers alias precedence plus agent-scoped media passthrough in the current extension tests, and lets #43570 close through the merged fix.
