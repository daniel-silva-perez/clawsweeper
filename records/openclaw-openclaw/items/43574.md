## Summary

Keep this PR open. Current main still has the reported Discord `thread-reply` attachment alias gap, but the proposed patch is incomplete against the current media-read contract because it forwards only `mediaLocalRoots` and not `mediaReadFile`; it also needs to be adapted to the current extension-owned test files before merge.

## What This Changes

This PR updates Discord `thread-reply` handling to read `media`, `path`, and `filePath` attachment aliases, passes `mediaLocalRoots` into the Discord action runtime, and adds tests for alias precedence and media passthrough.

## Best Possible Solution

Land a current-main Discord-owned fix that normalizes `media ?? path ?? filePath` for `thread-reply`, forwards the full media access path including `mediaReadFile`, covers alias precedence plus agent-scoped media passthrough in current extension tests, and lets #43570 close through the merged fix.
