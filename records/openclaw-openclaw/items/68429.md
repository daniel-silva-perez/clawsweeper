## Summary

Keep open. The issue is paired with open PR #68433 using closing syntax, and current main still shows the relevant path as outbound session metadata/mirroring plus Telegram callback-to-message dispatch, without an equivalent prewarm step before the first callback turn.

## What This Changes

The issue asks for Telegram inline-button callback turns from `openclaw message send --buttons` to wait for or reuse a fully bootstrapped session before the first model response.

## Best Possible Solution

Review and land #68433, or an equivalent narrow fix, so outbound mirrored sessions are warmed with the same bootstrap/skills context expected by the first Telegram callback-driven turn and covered by a regression test for outbound send followed by immediate callback.

## Reproduction Assessment

Yes. The issue provides concrete Telegram `message send` plus immediate inline-button steps, and current source confirms callback payloads are routed as inbound text while outbound mirrored session creation does not prewarm bootstrap context.

## Solution Assessment

Unclear until #68433 is reviewed, but the direction is appropriate: fix the outbound mirrored-session/bootstrap boundary rather than adding Telegram-only callback delays or changing callback payload semantics.
