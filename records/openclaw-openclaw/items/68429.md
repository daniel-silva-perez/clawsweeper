## Summary

Keep this open. Current main still records outbound session metadata before Telegram delivery and mirrors the sent assistant message, but I found no current-main prewarm of the mirrored outbound session's bootstrap/skills context before the first callback-driven turn. The linked PR #68433 is open, unmerged, and explicitly targets this issue, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Ensure Telegram inline-button callbacks on freshly created outbound/mirrored sessions wait for or reuse full session bootstrap context before the first agent turn.

## Best Possible Solution

Review and land PR #68433 or an equivalent focused fix that prewarms the mirrored outbound session after session metadata is recorded and before a first callback can dispatch, with a regression test covering proactive outbound send followed by the first callback turn.

## Reproduction Assessment

Yes. The issue includes concrete Telegram `message send --buttons` steps and a sanitized transcript; current-main inspection also traces the callback from Telegram fallback into the normal first-turn dispatch path without an outbound prewarm step.

## Solution Assessment

Unclear until the linked PR is fully reviewed, but the proposed direction is the right narrow boundary: prewarm or reuse the existing session skills/bootstrap snapshot for mirrored outbound sessions rather than changing Telegram callback parsing itself.
