## Summary

Keep open. Current main still lacks the PR's full Telegram forum-topic announce fix: slash targets are not parsed by the Telegram target parser, isolated cron target resolution has no pre-bootstrap topic extractor, explicit target parsing is loaded-plugin-only, and delivery-specific errors are not propagated through the run outcome. The PR head also has unresolved review feedback for bare `chatId:topicId` pre-bootstrap parsing.

## Best Possible Solution

Keep this PR open for maintainer review, revision, or replacement. The best path is a focused fix that parses all supported Telegram forum-topic target forms before isolated cron plugin bootstrap, adds slash support to the Telegram target parser, deliberately keeps or rejects the cfg-backed plugin fallback, propagates delivery-specific errors into `lastDeliveryError`, and adds regression coverage for slash, `:topic:`, and bare `chatId:topicId` announce delivery paths.
