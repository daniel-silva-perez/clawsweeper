## Summary

Keep open. Current main still does not implement the PR's intended slash-format Telegram forum-topic parsing, pre-bootstrap target extraction, cfg passthrough for explicit target parsing, or delivery-specific error propagation; the PR head also has unresolved review feedback for the bare chatId:topicId shorthand.

## Best Possible Solution

Keep this PR open for maintainer review, rebase, or replacement. The best fix should parse all documented Telegram forum-topic formats before isolated cron plugin bootstrap, add slash support to the Telegram target parser, deliberately handle any cfg-backed plugin fallback, propagate delivery-specific errors into lastDeliveryError, address the existing review comments, and keep regression coverage for final assistant-visible Telegram topic delivery.
