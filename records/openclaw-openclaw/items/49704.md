## Summary

Keep this PR open. Current main has only the narrower Telegram `:topic:` cron path and explicit `delivery.threadId` support; it still lacks the PR’s slash-format Telegram target parsing, pre-bootstrap extraction for all forum-topic forms, cfg-backed explicit-target parsing fallback, and delivery-specific error propagation. The PR head also has unresolved review feedback for bare `chatId:topicId` pre-bootstrap parsing, so it is not ready to close as implemented or merge as-is.

## Best Possible Solution

Keep this PR open for maintainer review, revision, or replacement. The best path is a focused fix that parses all supported Telegram forum-topic target forms before isolated cron plugin bootstrap, adds slash support to the Telegram target parser, either intentionally implements or rejects the cfg-backed plugin fallback, propagates delivery-specific errors into `lastDeliveryError`, and adds regression coverage for slash, `:topic:`, and bare `chatId:topicId` announce delivery paths.
