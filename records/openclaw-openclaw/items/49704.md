## Summary

Keep this PR open. Current main only has partial Telegram forum-topic cron support: `:topic:` targets and explicit `delivery.threadId` are covered, but the PR’s slash-format Telegram parser path, pre-bootstrap extraction for all topic forms, cfg-backed explicit-target parsing fallback, and delivery-specific error propagation are still not fully implemented on main.

## Best Possible Solution

Keep this PR open for maintainer review, revision, or replacement. The best path is a focused fix that parses all supported Telegram forum-topic target forms before isolated cron plugin bootstrap, adds slash support to the Telegram parser, intentionally handles any cfg-backed plugin fallback, propagates delivery-specific errors into `lastDeliveryError`, and adds regression coverage for slash, `:topic:`, and bare `chatId:topicId` announce delivery paths.
