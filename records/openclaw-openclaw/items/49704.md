## Summary

Keep open. Current main has partial Telegram forum-topic cron support for `:topic:` targets, but it still does not implement this PR's intended slash-format parsing, pre-bootstrap extraction for all documented topic forms, cfg-backed explicit-target parsing, or delivery-specific error propagation. The PR head also has unresolved review feedback for the bare `chatId:topicId` shorthand case.

## Best Possible Solution

Keep this PR open for maintainer review, revision, or replacement. The best fix should parse all documented Telegram forum-topic target formats before isolated cron plugin bootstrap, add slash support to the Telegram target parser, deliberately handle any cfg-backed plugin fallback, propagate delivery-specific errors into `lastDeliveryError`, address the existing shorthand review comments, and keep regression coverage for final assistant-visible Telegram topic delivery.
