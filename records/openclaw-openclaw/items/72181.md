## Summary

Keep this issue open. Current main still lacks the requested BlueBubbles trainer/supervised owner-approval mode, the request targets an existing bundled BlueBubbles plugin surface, and the feature changes user-facing safety semantics that need BlueBubbles owner/product review rather than automated cleanup.

## Best Possible Solution

Keep the issue open for BlueBubbles owner/product review. A good implementation would stay under extensions/bluebubbles, add schema-backed trainerMode, trainerNotifyNumber, optional agentTag, owner command parsing, pending-message TTL/expiry handling, docs, and tests proving drafts are intercepted before outbound DM delivery and only approved/custom replies can reach the original sender in supervised mode.
