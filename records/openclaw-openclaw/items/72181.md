## Summary

Keep open. Current main still lacks the requested BlueBubbles trainer/supervised reply mode: there is no public config/docs surface for the proposed fields, and the BlueBubbles DM delivery path still sends the agent's final reply directly to the resolved sender/chat. The feature is fresh and actionable, but owner-gated private-message delivery, correction logging, and AI send tagging need BlueBubbles maintainer/product/privacy review before automation should implement it.

## What This Changes

Request adding optional BlueBubbles trainer modes that run the agent to draft replies, forward drafts to an owner for iMessage approval or correction, and prevent delivery to the original sender unless the owner authorizes it.

## Best Possible Solution

Keep this BlueBubbles-owned unless implementation proves a reusable channel approval seam is needed: add schema-backed trainerMode/trainerNotifyNumber/agentTag, intercept DM final delivery before sendMessageBlueBubbles, store pending drafts with TTL, parse owner approve/correct commands, deliver only approved supervised replies, log corrections with explicit privacy semantics, and update BlueBubbles docs plus focused tests.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence current-main verification path. Source and docs searches show the requested config/API is absent, and the DM reply path still sends assistant chunks directly to the resolved BlueBubbles target.

## Solution Assessment

Unclear until BlueBubbles owner/product review. The proposed plugin-local boundary is the right starting point, but the exact owner command grammar, correction logging, AI tagging, and privacy defaults should be decided before a fix PR is queued.
