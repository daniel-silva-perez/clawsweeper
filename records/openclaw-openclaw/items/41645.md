## Summary

Keep open. Current main has substantial session/token/cost usage reporting, daily totals, and Dashboard drilldowns, but it still does not implement the requested per-user input/output/total token aggregation, user filter/reporting surface, weekly/monthly user rollups, or quota/billing management. The prior ClawSweeper comment reached the same conclusion, and I found no newer implementation on the checked-out main SHA.

## Best Possible Solution

Keep this issue open and scope it as a core usage-accounting enhancement. Define a normalized user/principal key from SessionOrigin metadata, extend sessions.usage/shared types with byUser plus daily/weekly/monthly rollups, add Dashboard filters/export for users, and document privacy/trust-model semantics. If quota enforcement is included, add access-control and cross-channel identity-normalization tests.
