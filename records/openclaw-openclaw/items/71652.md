## Summary

Keep open. Current main still lacks the proposed local `tsgo` pressure refusal and temp redirection, and this PR targets the active wrapper path, but the patch should be revised before landing because its pressure check still probes `/tmp` after redirecting the actual temp environment elsewhere.

## What This Changes

The PR adds a local `tsgo` preflight host-pressure guard, an explicit force override, temp-dir redirection into `.artifacts`, and focused run-tsgo tests.

## Best Possible Solution

Land the host-pressure guard in the existing script runtime seam, but make the temp-space check inspect the filesystem backing the effective heavy-check temp directory, preserve sparse-guard and lock behavior, and keep focused tests for refusal, force bypass, and temp routing.

## Reproduction Assessment

Yes. The diff gives a high-confidence reproduction path: configure or run on a host where `/tmp` and `.artifacts/tmp/local-heavy-checks` are different mounts, then the PR's `df -Pk /tmp` check can disagree with the filesystem that `tsgo` will actually write to.

## Solution Assessment

No. The PR is aimed at the right wrapper/runtime seam, but the temp-pressure implementation is not the narrowest correct fix until it resolves and checks the effective temp directory instead of hard-coding `/tmp`.
