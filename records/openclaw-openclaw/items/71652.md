## Summary

Keep open. Current main still lacks the proposed local tsgo host-pressure guard, so the PR is not obsolete, but the branch is not ready to merge because it redirects heavy-check temp writes to the worktree artifact directory while the new temp-pressure probe still checks `/tmp`. That can produce false refusals or false passes on split-mount hosts, matching the prior Codex and Greptile review discussion.

## What This Changes

The PR adds a local `tsgo` heavy-check pressure guard, redirects heavy-check temp files into `.artifacts/tmp/local-heavy-checks`, adds an `OPENCLAW_HEAVY_CHECK_FORCE` bypass, and extends focused run-tsgo tests.

## Best Possible Solution

Land the guard in the existing script wrapper/runtime seam after the temp-space check resolves the effective heavy-check temp directory and tests cover split-mount behavior; if `/tmp` is still desired as a separate host-health signal, make that explicit so it cannot replace validation of the real temp target.

## Reproduction Assessment

Yes. On a host where `/tmp` and the worktree artifact directory are different filesystems, make one low on space and the other healthy; after the PR sets `TMPDIR` to `.artifacts/tmp/local-heavy-checks`, `df -Pk /tmp` can disagree with the filesystem `tsgo` will actually use.

## Solution Assessment

No. The wrapper/runtime seam is the right place, but the proposed patch is not the narrowest correct fix until it checks the effective temp filesystem instead of hard-coding `/tmp`.
