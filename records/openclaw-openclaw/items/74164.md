## Summary

Keep this issue open. Current main still sends gateway-executed plugin message actions through an explicit 10,000 ms timeout, the `openclaw message send --media` CLI path does not expose or pass a longer timeout, and the linked closing PR #74169 is still an open, unmerged implementation candidate rather than a shipped fix.

## What This Changes

The issue reports `openclaw message send --channel wecom --media` timing out after 10,000 ms when sending an approximately 10 MB file via the external WeCom plugin.

## Best Possible Solution

Review, revise, land, or replace PR #74169 with a targeted core fix for gateway-executed plugin message sends. The end state should let large media sends complete without changing unrelated gateway RPC budgets unnecessarily, with focused regression coverage for the CLI/media plugin dispatch path and docs or changelog only if the user-facing CLI contract changes.
