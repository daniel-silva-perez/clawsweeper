## Summary

Keep this issue open. Current main still routes gateway-executed plugin message sends through an explicit 10,000 ms timeout, the `openclaw message send --media` CLI path has no timeout option, and the linked fix PR #74169 remains open rather than merged.

## What This Changes

The issue asks OpenClaw to avoid timing out after 10000 ms when sending a roughly 10 MB file via the WeCom channel with `openclaw message send --media`.

## Best Possible Solution

Keep this issue open and review, revise, land, or replace PR #74169 with a generic core fix for gateway-executed plugin message sends. The fix should address the explicit message-action timeout boundary, add focused regression coverage for CLI/media gateway plugin sends, and update docs/changelog only if the user-facing CLI behavior changes.
