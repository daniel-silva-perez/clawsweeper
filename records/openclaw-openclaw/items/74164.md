## Summary

Keep this issue open. Current main still routes CLI-driven gateway-executed plugin message sends through an explicit 10,000 ms gateway RPC timeout, and the `openclaw message send --media` CLI path does not expose or pass a longer timeout. PR #74169 is open and links this issue for closure, so the right maintainer action is to review, revise, land, or replace that PR rather than closing this issue early.

## Best Possible Solution

Keep this issue open and review PR #74169. The best fix is a generic OpenClaw core change that lets `openclaw message send`, especially media sends executed by gateway-run channel plugins, use a longer or configurable gateway RPC timeout without adding WeCom-specific code. Add focused regression tests for CLI message options and gateway-executed plugin sends, plus docs/changelog if the CLI behavior changes.
