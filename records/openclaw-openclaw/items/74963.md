## Summary

Keep open. Current main still hashes the packaged bundled runtime-deps stage key from the lexical `path.resolve(packageRoot)`, and the provided discussion shows open PR #75048 with closing syntax for this exact hash-divergence bug, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports divergent bundled plugin runtime-deps hashes causing packaged bundled channel `dist/*.js` ENOENT failures in multi-instance or equivalent-package-root deployments.

## Best Possible Solution

Review and land #75048, or an equivalent narrow replacement, so packaged bundled runtime-deps canonicalize `packageRoot` before hashing the stage key and include regression coverage for equivalent package-root views. After that, evaluate any remaining disabled/unconfigured bundled-channel import noise as a separate focused follow-up.

## Reproduction Assessment

Yes. A focused reproduction is to expose the same packaged OpenClaw root through real and symlink/junction-style lexical paths and compare the resolved external runtime-deps install roots; current main still has the lexical hash seam that can split those roots.

## Solution Assessment

Yes. Canonicalizing `packageRoot` before hashing the runtime-deps stage key is the narrowest maintainable fix because it converges the shared stage directory without changing plugin activation policy or broadening bundled channel loading.
