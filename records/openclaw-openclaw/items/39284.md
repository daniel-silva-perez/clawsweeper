## Summary

Keep this PR open. Current main has automatic workspace bootstrap injection and `sessions_spawn(lightContext)`, but it still does not expose or forward the PR's explicit `contextFiles` or `inheritContext` API. The PR also still needs maintainer product judgment because its proposed requester-workspace semantics conflict with current cross-agent target-workspace behavior and its diff has security/resource-bound concerns.

## Best Possible Solution

Keep this PR open for maintainer product judgment. If explicit spawn-time file selection is still desired, rework it on top of current bootstrap plumbing with documented requester-vs-target workspace semantics, bounded `contextFiles` schema, prompt-size limits, same-agent and cross-agent regression tests, docs updates, and removal or separation of the unrelated plugin logger change. If automatic bootstrap injection plus `lightContext` is the intended product direction, maintainers should close it manually with that rationale.
