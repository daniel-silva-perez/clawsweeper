## Summary

Keep open: current main does not contain the proposed split, and the PR head is not mergeable as-is because it has module/type failures and drops current-main dreaming behavior and test exports. The existing bot comments are supported by source, CI, and current-main evidence.

## What This Changes

The PR extracts shared dreaming types/helpers and session transcript ingestion from extensions/memory-core/src/dreaming-phases.ts into two new modules while leaving dreaming-phases.ts as the dreaming phase orchestration layer.

## Best Possible Solution

Land a current-main refactor only after it preserves exported SDK subpaths, the legacy registration/test hooks, live recall filtering, detached narrative handling, and dreaming model overrides, with the memory-core targeted lane and changed gate proving no behavior drift.

## Reproduction Assessment

Yes. Static inspection of the PR head reproduces the module/type failures, exact-head CI already failed in type/test/check lanes, and current-main tests show the removed exports are still required.

## Solution Assessment

No. Splitting the large file is a reasonable maintainability direction, but this branch is not the best current solution because it breaks compile contracts and misses current-main fixes; a rebased or replacement split against current main is safer.
