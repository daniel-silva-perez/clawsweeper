## Summary

Current main still has the reported documentation gap. The Media Understanding page documents provider model entries with provider/model/profile fields, but does not explain that media provider credentials resolve through auth profiles, environment variables, or models.providers.<provider>.apiKey, and runtime/schema code still does not support inline apiKey on tools.media model entries. Related PR #51089 was the right shape of docs fix, but it is closed and unmerged.

## Best Possible Solution

Keep this issue open and land a small docs-only patch to docs/nodes/media-understanding.md. The patch should state near Config overview or Model entries that provider media entries do not accept inline apiKey, credentials resolve through auth profiles, provider environment variables, or models.providers.<provider>.apiKey, and include a short models.providers example or link to the public auth/config docs.
