## Summary

Current main still has the reported documentation gap. The Media Understanding page shows provider entries with `provider`, `model`, `profile`, and `preferredProfile`, but it does not state that API keys resolve through auth profiles, environment variables, or `models.providers.<provider>.apiKey`, and it does not say inline `apiKey` is unsupported in `tools.media.*.models[]`. The runtime and config types confirm that behavior. Related PR #51089 proposed the exact docs clarification and used closing syntax for this issue, but it is closed and unmerged, so the issue should remain open.

## Best Possible Solution

Keep the issue open and land a small docs-only patch to `docs/nodes/media-understanding.md`: add a credential-source note near Config overview or Model entries stating that provider media entries do not accept inline `apiKey`, that credentials resolve through auth profiles, provider env vars, or `models.providers.<provider>.apiKey`, and include a short `models.providers` example or link to the public configuration/auth docs.
