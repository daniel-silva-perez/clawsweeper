## Summary

Keep open. Current main still has only partial google-vertex support: metadata, model-id normalization, and ADC discovery exist, but the runtime still routes google-vertex through the Google AI Studio provider and Google Generative AI transport instead of a native Vertex AI URL/auth path. The supplied timeline also shows open PR #60860 uses closing syntax for this issue, so the issue should remain open until that PR or an equivalent maintainer fix lands.

## What This Changes

The issue requests restoring google-vertex as a distinct Vertex AI transport and auth path while still sharing Google Gemini model-id normalization.

## Best Possible Solution

Land PR #60860 or a maintainer-owned equivalent that registers a native google-vertex runtime path, keeps ADC/project/location handling separate from Google AI Studio API-key auth, shares only model-id normalization, and adds focused regression coverage for Vertex URL construction and auth headers.
