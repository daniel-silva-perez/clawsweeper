## Summary

Keep open. Current main still has only partial google-vertex metadata/model normalization; the chat transport remains routed through the Google AI Studio provider path and does not build Vertex project/location URLs or exchange ADC credentials for a Vertex bearer token. The supplied GitHub context also shows PR #60860 is open, unmerged, and explicitly closes this issue, so repository policy says the issue should remain open until that PR or a maintainer-owned equivalent lands.

## Best Possible Solution

Keep this issue open while PR #60860, or a maintainer-owned equivalent, is reviewed. The best fix belongs in the bundled Google plugin: register a native google-vertex provider/runtime path, keep ADC/project/location resolution separate from Google AI Studio API-key auth, construct Vertex project/location URLs, share only safe model-id normalization, and add focused regression tests for request URL and auth header behavior.
