## Summary

Keep open. Current main still has partial `google-vertex` support: manifest metadata, model-id normalization, endpoint attribution, and ADC discovery exist, but runtime routing still goes through the Google AI Studio provider and Google Generative AI transport rather than a native Vertex AI URL/auth path. The issue is also explicitly paired with open PR #60860, so it should remain open until that PR or an equivalent maintainer fix lands.

## What This Changes

The issue asks for `google-vertex` to keep Vertex AI transport and ADC/service-account auth while sharing Google Gemini model-id normalization.

## Best Possible Solution

Land PR #60860 or a maintainer-owned equivalent that registers a native `google-vertex` runtime provider, keeps Vertex endpoint/project/location handling and ADC Bearer auth separate from Google AI Studio API-key auth, shares only Gemini model-id normalization, and adds focused regression coverage for Vertex URL construction and auth headers.
