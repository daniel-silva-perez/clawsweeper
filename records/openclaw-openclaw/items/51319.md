## Summary

Keep open. Current main still lacks Darwin zombie detection in the shared PID liveness helper, and this remains a focused implementation candidate, but the latest head is not merge-ready because the new macOS regression test mocks the wrong `child_process` binding and CI confirms the test fails.

## What This Changes

This PR extends the shared PID liveness helper with a Darwin `ps -o state=` zombie check and adds a macOS zombie regression test.

## Best Possible Solution

Land a focused version of this change after fixing the test mock so it exercises the actual production `spawnSync` binding. Keep the fix in the shared PID helper so session locks, gateway locks, and plugin SDK file locks inherit the same Darwin zombie semantics.

## Reproduction Assessment

Yes for the PR defect: the PR head has a check-run annotation showing the macOS zombie test fails with `expected true to be false`, and the Node ESM probe explains why the spy does not intercept the named import. For the underlying product bug, current-main source inspection shows the missing Darwin path, but this review did not create a live macOS zombie process.

## Solution Assessment

No, not as submitted. The implementation boundary is the right one and the production logic is directionally sound, but the regression test must mock the same `child_process` binding production uses before this is the best mergeable fix.
