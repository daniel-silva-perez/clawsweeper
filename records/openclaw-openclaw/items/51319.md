## Summary

Keep this PR open. Current main still lacks Darwin zombie detection in the shared PID liveness helper, and that helper is still used by session-lock, gateway startup cleanup, gateway-lock, and plugin SDK stale-lock paths. The PR is not obsolete or implemented on main; it remains the active focused implementation candidate, with a small security review item around adding a subprocess call and a test-mocking detail maintainers should verify before merge.

## Best Possible Solution

Keep this PR open as the active implementation candidate. Maintainers should review the cleaned-up two-file diff, make sure the Darwin zombie test genuinely mocks the production `spawnSync` call, run `pnpm test src/shared/pid-alive.test.ts` plus the appropriate changed gate, and merge if the subprocess/env handling and CI are acceptable.
