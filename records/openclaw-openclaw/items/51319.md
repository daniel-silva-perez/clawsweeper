## Summary

Keep this PR open. Current main still has Linux-only zombie detection in the shared PID liveness helper, while session-lock cleanup, gateway startup session cleanup, gateway locks, and plugin SDK file locks depend on that helper. The latest PR head remains a focused implementation candidate and appears to address the prior review feedback by switching to argument-array `spawnSync`, checking Darwin state strings by first character, removing committed logs, and adding a mocked Darwin regression test.

## Best Possible Solution

Keep this PR open as the active implementation candidate. Maintainers should review the latest two-file diff, ensure the Darwin zombie regression test truly exercises the production `spawnSync` call, run `pnpm test src/shared/pid-alive.test.ts` plus the appropriate changed gate, and merge if the subprocess behavior is acceptable.
