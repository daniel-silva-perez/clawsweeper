## Summary

Keep this PR open as a useful fix candidate. Current main still has no shared macOS zombie detection in `isPidAlive()`, so the PR’s intended behavior is not implemented on main. Gateway lock recovery is partly handled by newer port and argv checks, but shared session/plugin lock callers still depend on the Linux-only PID liveness helper. The PR also needs cleanup before merge because its diff includes unrelated runtime log artifacts and the macOS test does not exercise zombie parsing.

## Best Possible Solution

Keep the PR open as a focused fix candidate, but require a cleaned-up revision: remove the committed log files, keep the shell-safe `spawnSync("ps", ["-o", "state=", "-p", String(pid)])` style darwin check, treat any `ps` state beginning with `Z` as zombie, and add mocked cross-platform regression coverage in `src/shared/pid-alive.test.ts`. Gateway-lock-specific behavior should remain aligned with the current port and argv recovery already on main.
