## Summary

Keep this issue open. Current main has the related #48930 browser-bundle hardening for the `fs.constants.W_OK` crash, but the served Control UI bootstrap still relies on side-effect Lit decorator registration and the current Chromium browser tests do not exercise the real served `ui/index.html` path or assert root custom-element registration.

## Best Possible Solution

Keep this issue open until maintainers add or run a served-Control-UI Chromium regression that loads the real built/served shell, captures module/CSP/resource/service-worker errors, asserts `customElements.get("openclaw-app")`, and verifies Chat renders after backend RPC success. If #48930 is believed to be the fix, link that targeted regression or reporter confirmation here before closing.
