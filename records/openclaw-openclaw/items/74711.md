## Summary

Keep open. Current main still has `sqlite-vec` as a root runtime dependency but does not mirror it into `openclaw.bundle.mirroredRootRuntimeDependencies`; the memory loader dynamically imports `sqlite-vec`, and the bundled runtime-deps planner only includes root package dependencies that are explicitly listed in that mirror metadata. The PR is a narrow package metadata fix for the concrete #74692 failure, and I found no blocking correctness issue in the patch.

## What This Changes

The PR adds `sqlite-vec` to the root `openclaw.bundle.mirroredRootRuntimeDependencies` list in `package.json` so bundled plugin runtime dependency snapshots include it.

## Best Possible Solution

Land the narrow metadata fix after validation, and consider a follow-up guard that covers `packages/memory-host-sdk` bare runtime imports so future package-level mirror misses are caught before release.
