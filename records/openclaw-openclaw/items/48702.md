## Summary

Keep this PR open. Current main still does not implement the PR's intended /status clarification: the status renderer still suppresses reasoning when it is off and still prints raw values such as `on` instead of user-facing visibility labels such as `visible`. The PR is stale against the newer status-rendering file layout, but the focused UX change remains useful, and the linked review discussion correctly flags normalization of persisted/resolved reasoning values as the main implementation detail to preserve during a rebase.

## Best Possible Solution

Keep the PR open and ask for a rebase onto current main. Move the display change into `src/status/status-message.ts`, always render a reasoning visibility line, map canonical values to `hidden`/`visible`/`stream`, normalize persisted and resolved reasoning inputs safely, and update the current status and system-prompt tests around the new file layout.
