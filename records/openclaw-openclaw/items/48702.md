## Summary

Keep this PR open. Current main still does not implement the intended `/status` clarification: reasoning is omitted when it is `off`, enabled reasoning still renders raw values like `on`, and the system prompt still describes `/status` as showing reasoning only when enabled. The PR is stale against the newer `src/status/status-message.ts` renderer, but the focused UX change remains useful.

## Best Possible Solution

Keep the PR open and ask for a rebase onto current main. Move the display change into `src/status/status-message.ts`, always render a reasoning visibility line, map canonical values to `hidden`/`visible`/`stream`, normalize resolved and persisted reasoning inputs safely, and update the current status and system-prompt tests around the new file layout.
