## Summary

Keep open. Current main still does not implement the intended `/status` clarification: the off state is omitted and enabled reasoning is displayed as raw internal values, while this PR also needs adaptation to the current status renderer before it can be merge-ready.

## What This Changes

The PR updates `/status` display logic, prompt copy, and tests so reasoning visibility is always shown as `hidden`, `visible`, or `stream` while keeping `Think: <level>` as the separate effort setting.

## Best Possible Solution

Implement the display change in `src/status/status-message.ts`: normalize canonical and legacy reasoning inputs, always render a reasoning visibility label, keep thinking effort unchanged, and update the current status and system-prompt tests around the shared renderer.

## Reproduction Assessment

Yes. A source-level check against current main is enough: `buildStatusMessage` omits `Reasoning` for `off` and prints raw `on`, and the current tests still encode that behavior.

## Solution Assessment

No. The product direction is narrow and useful, but this branch is not the best current implementation because it edits the pre-refactor renderer and leaves `resolvedReasoning` normalization incomplete.
