## Summary

Keep this PR open. Current main still uses unset-sensitive Docker Compose bind sources for both config and workspace mounts, and live `docker compose config` without those env vars still fails with an empty-source volume spec. The PR is a focused active implementation candidate, not obsolete or implemented on main.

## Best Possible Solution

Keep this PR open and review it as the active Docker Compose fix. The best path is a narrow compose change that defaults host bind sources without propagating host paths into container runtime config, plus verification of `docker compose config` with env unset and with explicit overrides, and aligned docs/changelog coverage.
