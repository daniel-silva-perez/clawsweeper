## Summary

Keep open. Current main only partially addresses the report: fallback retry with history now prepends a retry marker to the original prompt instead of replacing it entirely, but that marker is still treated as the visible embedded prompt. The embedded runner already has a `transcriptPrompt` mechanism for separating runtime context from transcript-visible user text, but the fallback retry path does not pass it, so the user-visible transcript can still contain non-user-authored fallback text.

## Best Possible Solution

Keep this issue open and fix the fallback retry path by preserving the original user body as the transcript-visible prompt, while moving retry/prelude context into `transcriptPrompt` separation, hidden runtime context, a lifecycle event, or UI fallback metadata. Update `attempt-execution.test.ts` to expect the original body for fallback retries with history and add an embedded-run regression test proving the persisted `role: user` turn does not include fallback marker text.
