## Summary

Keep open. Current main still sends fallback retry marker text through the transcript-visible embedded prompt path, so the original user body is no longer preserved exactly in persisted visible history when fallback retries have existing history or a prior-context prelude.

## What This Changes

The issue asks OpenClaw to preserve the original user prompt as the visible transcript turn on model fallback retries instead of recording synthetic retry text as user-authored content.

## Best Possible Solution

Preserve the original user body as the transcript-visible prompt for fallback retries, while moving retry markers and any harvested prior-session prelude into hidden runtime context, lifecycle metadata, or another non-user-authored channel. Add focused unit coverage plus an embedded-run regression proving persisted visible `role: user` content excludes fallback marker text.

## Reproduction Assessment

Yes. The current helper and tests deterministically reproduce the bad transformation for `isFallbackRetry: true` with session history or a prelude, and the embedded call path passes that transformed value as the submitted visible prompt without `transcriptPrompt` separation.

## Solution Assessment

No, not exactly as proposed. Always returning the body is too blunt because it can remove useful fallback context; the maintainable fix is to keep that context available to the model but separate it from the user-authored transcript body.
