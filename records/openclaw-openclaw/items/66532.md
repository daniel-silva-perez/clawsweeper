## Summary

Keep open. Current main still has the embedded-run abort/wait/active/queue helpers internally, and it has adjacent Gateway `chat.abort` / `sessions.abort` support for agent RPC runs, but the requested in-process plugin runtime contract is still missing from `api.runtime.agent`. The CoClaw plugin context remains a valid example of why plugin authors need a stable runtime surface instead of a global-state side door.

## Best Possible Solution

Keep this issue open and implement the additive plugin runtime contract: re-export the embedded-run abort/wait/isActive/queue helper family from `src/plugins/runtime/runtime-embedded-pi.runtime.ts`, lazy-bind them in `createRuntimeAgent()`, declare them on `PluginRuntimeCore["agent"]`, and update runtime shape tests, plugin runtime mocks, public plugin runtime docs, and any bundled-plugin boundary inventories. Treat the shipped Gateway abort path and agent-harness-only exports as related surfaces, not substitutes for the requested in-process plugin API.
