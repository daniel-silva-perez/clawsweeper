## Summary

Keep this PR open. Current main still has the exact gap described by the PR: `handleCodeBlockCopy` is delegated only from `.chat-thread`, while `.chat-sidebar` is a sibling container that renders markdown content containing `.code-block-copy` buttons. The PR's one-line change remains a plausible focused fix, and current main does not yet implement it.

## Best Possible Solution

Review and land the focused UI fix if it still applies, ideally with a small regression test that renders sidebar markdown with a code block and verifies the clipboard handler runs from the sidebar container. Current main should not close this PR as implemented because the sidebar binding is still missing.
