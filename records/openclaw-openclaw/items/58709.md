## Summary

Keep open. Current main still has the sidebar copy-button gap: markdown code blocks render copy buttons, but the sidebar container does not delegate clicks to the existing copy handler. The PR is a narrow, correct-looking fix and should go through normal maintainer review rather than cleanup closure.

## What This Changes

This PR adds the existing code-block copy click handler to the WebChat markdown sidebar container in `ui/src/ui/views/chat.ts`.

## Best Possible Solution

Land the minimal event-delegation fix, preferably with a focused UI/DOM regression check that clicks a sidebar markdown code-block copy button and verifies the clipboard write path.

## Reproduction Assessment

Yes. Render a markdown sidebar containing a fenced code block on current main: the markup includes `.code-block-copy`, but clicks inside `.chat-sidebar` do not reach `handleCodeBlockCopy` because the handler is only attached to `.chat-thread`.

## Solution Assessment

Yes. Reusing the existing guarded click delegate on the sibling sidebar container is the narrowest maintainable fix; moving the handler to a broader common parent would touch more behavior without a clear benefit.
