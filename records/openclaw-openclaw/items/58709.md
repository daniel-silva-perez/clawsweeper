## Summary

Keep this PR open. Current main still has the reported bug: sidebar markdown can emit code-block copy buttons, but the delegated copy handler is only attached to the sibling chat thread. The PR is a narrow, correct fix and needs maintainer review plus targeted validation rather than ClawSweeper replacement work.

## What This Changes

This PR adds the existing delegated code-block copy click handler to the chat sidebar container in `ui/src/ui/views/chat.ts` so sidebar markdown code-block copy buttons can write to the clipboard.

## Best Possible Solution

Land this PR or an equivalent one-line sidebar event binding, ideally with a small regression test or focused UI proof; no broader markdown/sidebar redesign is needed.

## Reproduction Assessment

Yes. Render sidebar markdown containing a fenced code block on current main and click its copy button: the button is emitted by the shared markdown renderer, but the only delegated copy listener is on the sibling `.chat-thread` container.

## Solution Assessment

Yes. Reusing the existing delegated handler on `.chat-sidebar` is the narrowest maintainable fix because the handler already ignores unrelated clicks via `.closest(".code-block-copy")`.
