## Summary

Current main still has the brittle respawn behavior described here: detached gateway respawn builds child args from `process.execArgv` plus `process.argv.slice(1)`, and both normal restart and `update.run` update restart paths use it. The related fixing PR was closed unmerged, so this remains actionable in the repo.

## Best Possible Solution

Keep this issue open and implement a focused respawn planner in `src/infra/process-respawn.ts`: resolve the OpenClaw package root, rewrite pnpm versioned package realpaths to a stable `openclaw.mjs` or existing source wrapper when appropriate, preserve dev/source entrypoints, and add targeted tests covering both `restartGatewayProcessWithFreshPid()` and `respawnGatewayProcessForUpdate()`. The closed PR #52319 is useful prior art but should be reviewed for scope before reviving or replacing it.
