## Summary

Current main still has the eager image/video/music provider-listing behavior targeted by this PR: the shared resolver accepts a `CapabilityProvider[]`, and the three model-config callers still compute provider arrays before the explicit model-config short-circuit can return. The related #73793 cache fix is present on current main and likely reduces the original reload cost, but it does not implement this lazy call-site cleanup; the PR remains a narrow source/test change for linked #74096 and should stay open for maintainer review.

## What This Changes

This PR changes media-generation tool model-config resolution so image, video, and music provider lists are passed as lazy thunks and only evaluated when explicit model config is incomplete, with regression tests for both paths.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or land an equivalent narrow patch. The best implementation remains to make `resolveCapabilityModelConfigForTool` accept a provider thunk, evaluate it once only after explicit model config fails, update the image/video/music callers, and keep regression coverage for the complete-explicit-config and incomplete-config paths; maintainers should also consider adjusting the performance wording now that #73793 has landed.
