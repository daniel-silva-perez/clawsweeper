## Summary

Keep this PR open. Current main still passes eagerly evaluated image/video/music provider arrays into `resolveCapabilityModelConfigForTool`, so the lazy provider-listing cleanup proposed here is not implemented on main. The broader #73793 cache path appears to have changed on main, but it does not remove the eager call-site work targeted by this PR.

## What This Changes

This PR changes `resolveCapabilityModelConfigForTool` to accept a provider-list thunk, updates the image/video/music generation tool config resolvers to pass thunks, and adds regression tests for the explicit-modelConfig short-circuit and incomplete-config fall-through paths.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or land an equivalent patch, because current main still computes image/video/music provider lists before the complete explicit model-config short-circuit. Before merge, rebase onto current main, run the targeted media tool tests, consider a changelog fix entry for the user-facing hosted-gateway latency improvement, and run `pnpm build` if maintainers treat this as a lazy-loading/plugin-runtime boundary change.
