## Summary

Keep open. Current main still passes eager provider arrays into the media tool model-config helper, so the PR's intended lazy-provider-listing change is not implemented on main. The diff is narrow and consistent with the linked performance bug, and the security pass did not show unrelated workflow, dependency, lockfile, script, secret, or publishing changes.

## What This Changes

This PR changes the shared media tool model-config resolver to accept a provider-list thunk, updates the image/video/music generation config resolvers to pass thunks, and adds regression tests for the explicit-config short-circuit and incomplete-config fall-through paths.

## Best Possible Solution
