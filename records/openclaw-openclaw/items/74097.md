## Summary

Keep open. Current main still eagerly evaluates the image, video, and music provider lists before the complete explicit model-config short-circuit, so this PR addresses a real remaining hosted-gateway latency path. The code change is narrow and appears correct; the only concrete follow-up I found is the required changelog entry for a user-facing performance fix.

## What This Changes

The PR changes `resolveCapabilityModelConfigForTool` to accept a lazy provider-list thunk, updates the image/video/music model-config resolvers to pass thunks, and adds regression tests for the explicit-config short-circuit and incomplete-config fall-through paths.

## Best Possible Solution

Land this lazy provider-listing change, or an equivalent patch, with the added regression tests and a single-line `CHANGELOG.md` entry under Unreleased Fixes crediting the contributor; keep the paired bug report open until the PR merges.

## Reproduction Assessment

Yes. The current-main call path is enough to reproduce the eager evaluation: the three callers compute `listRuntime*GenerationProviders` before `resolveCapabilityModelConfigForTool` can return for a complete explicit `modelConfig`, and #74096 provides production timing logs for the hosted-gateway impact.

## Solution Assessment

Yes. Deferring the provider list behind a thunk is the narrowest maintainable fix for this specific waste path because it avoids the lookup entirely when the helper already has a complete explicit model config, while preserving the existing incomplete-config candidate resolution behavior.
