## Summary

Keep this issue open. Current main still ships `skills/xurl/SKILL.md` with the active X write surface and secret-safety guidance, but it does not include the requested confirm-before-post/write workflow, redacted audit-log instructions, or more concrete rate-limit/backoff guidance. This is not a ClawHub cleanup candidate because it is safety hardening for an already-bundled skill, parallel to the related `wacli` safety issue that was kept open.

## Best Possible Solution

Keep the issue open and make a focused bundled-skill safety update in `skills/xurl/SKILL.md`: add a clear Safety section that classifies read-only vs write actions, requires user confirmation showing the exact action/content/target before every X write, adds redacted audit-log instructions for write results, and gives practical 429/Retry-After/backoff guidance. If maintainers want stronger guarantees than instructions provide, track a separate plugin/runtime enforcement design using `before_tool_call` approval hooks.
