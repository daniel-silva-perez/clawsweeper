## Summary

Keep this issue open. Current main still ships the bundled xurl skill with public write actions, but its SKILL.md remains focused on secret handling and command examples; it does not classify read versus write actions, require exact-content confirmation before writes, add redacted xurl write audit logging, or provide xurl-specific Retry-After/backoff guidance. The prior ClawSweeper comment and related context do not supersede this xurl-specific bundled-skill safety request.

## Best Possible Solution

Keep this issue open and make a focused safety update to `skills/xurl/SKILL.md`: classify read-only versus write actions, require explicit user confirmation showing exact content/action/target before every X write, add redacted audit-log instructions for write results, and document practical 429/Retry-After/backoff behavior. If maintainers want true enforcement, track or implement an xurl write classifier through the existing before_tool_call or exec approval policy surface as a separate hardening step.
