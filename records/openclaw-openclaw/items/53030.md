## Summary

Keep this PR open. Current main still does not implement the central change: the docs i18n glossary checker remains zh-CN-only and the ja-JP glossary is still missing representative entries from the PR. The PR is useful source work, but it should not merge as-is because the proposed checker can weaken validation by succeeding when no glossary files are detected and by returning before validating malformed glossary JSON or explicit language inputs in some docs-change cases.

## What This Changes

The PR expands `docs/.i18n/glossary.ja-JP.json` and refactors `scripts/check-docs-i18n-glossary.mjs` to auto-detect glossary languages, add `--lang`, and dynamically exclude translated docs directories from the English docs scan.

## Best Possible Solution

Refresh the branch or replace it with a narrow follow-up that keeps the ja-JP glossary additions, makes the docs glossary checker validate all selected locale inputs before any empty-term exit, fails hard when no glossary files are detected, preserves zh-CN behavior, and adds focused regression tests for auto-detection, explicit `--lang`, invalid JSON, zero glossaries, body-only docs edits, and existing zh-CN coverage.
