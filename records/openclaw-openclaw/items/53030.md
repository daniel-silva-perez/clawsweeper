## Summary

Keep this PR open. Current main still has the zh-CN-only docs glossary checker and the ja-JP glossary still lacks the proposed parity terms, so the PR is not obsolete. The patch is not merge-ready because the refactored checker can return before validating selected glossary files and can silently disable the gate when no glossary files are detected; the security pass found no unrelated workflow, dependency, lockfile, package, or secret-handling changes.

## What This Changes

The PR expands `docs/.i18n/glossary.ja-JP.json` and refactors `scripts/check-docs-i18n-glossary.mjs` to auto-detect glossary languages, add `--lang`, and dynamically exclude translated docs directories from the English docs scan.

## Best Possible Solution

Land a refreshed narrow docs-i18n update that keeps the ja-JP glossary additions, makes multi-language glossary validation fail closed, preserves existing zh-CN behavior, and adds focused regression tests for auto-detection, explicit locale selection, invalid JSON, zero glossary files, body-only docs edits, and the existing docs gate wiring.

  - body: This early return runs before any selected glossary is loaded. For changed docs that add no new title or short link-label terms, malformed glossary JSON or an explicit `--lang` typo now exits successfully instead of failing the docs gate as the previous checker did after docs changed.

  - confidence: 0.88
