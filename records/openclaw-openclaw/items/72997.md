## Summary

Keep open. Current main has adjacent automatic channel restart behavior for config hot reload, secret reload, and health-monitor recovery, but it still does not expose the requested manual `openclaw channels reload <provider>` command or `channels.reload` Gateway RPC. The existing `channels.start` RPC is not a reload for an already-running channel because it only calls the start path and the channel manager skips accounts with active tasks.

## What This Changes

The issue requests a targeted operator command or Gateway RPC to disconnect and reconnect one named channel plugin without restarting the gateway or interrupting other channels/sessions.

## Best Possible Solution

Add an admin-scoped `channels.reload` Gateway method and an `openclaw channels reload --channel <provider> [--account <id>]` CLI wrapper that reuses the existing channel manager stop/start lifecycle, avoids restarting the gateway process, returns structured status, and updates protocol schemas, method scopes, CLI/Gateway docs, and focused tests. If maintainers intentionally do not want a manual reload surface, document the supported alternatives instead.
