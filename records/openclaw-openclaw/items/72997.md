## Summary

Keep open. Current main still lacks the requested manual channel reload CLI/RPC surface; existing automatic config, secrets, and health-monitor restart paths are useful context but do not give operators an explicit targeted reconnect command.

## What This Changes

Request a targeted command or Gateway action to disconnect and reconnect one named channel plugin without restarting the full Gateway or interrupting unrelated sessions.

## Best Possible Solution

Add an admin-scoped `channels.reload` Gateway method and `openclaw channels reload --channel <provider> [--account <id>]` CLI wrapper that reuses the existing stop/start lifecycle, preserves unrelated channels and sessions, returns structured status, and updates protocol schemas, method scopes, docs, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The current-main gap is high-confidence from source and docs inspection of the CLI registration, Gateway method list, admin scopes, protocol schema, existing start/stop behavior, and automatic reload handlers.

## Solution Assessment

Unclear pending maintainer API direction. A targeted reload is a maintainable direction, but the final solution should align with the account-scoped lifecycle semantics being discussed in #43938 before code is written.
