## Summary

Keep open. Current main still has automatic per-channel stop/start paths for config hot reload, secrets reload, and health monitoring, but it does not expose the requested manual `openclaw channels reload <provider>` command or a `channels.reload`/`channel.reload` Gateway method. `channels.start` is not equivalent because it does not stop an already-running channel account first.

## What This Changes

The issue requests a manual per-channel reload command or Gateway action that reconnects one named channel without restarting the whole gateway or interrupting other channels/sessions.

## Best Possible Solution

Add an admin-scoped `channels.reload` Gateway method plus an `openclaw channels reload --channel <provider> [--account <id>]` CLI wrapper that reuses the existing stop/start lifecycle, preserves unrelated channels and sessions, returns structured status, and updates protocol schemas, method scopes, CLI/Gateway docs, and focused tests.
