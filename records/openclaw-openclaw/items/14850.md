## Summary

Current main still lacks a route-binding-level session target that can make a Discord guild/channel binding share an agent main or alias session with Telegram DMs. The only route binding session override is `session.dmScope`, the schema is strict, and non-direct peers still build isolated channel/group session keys. The related open identity hook PR is DM-focused and does not supersede the requested non-DM binding override.

## Best Possible Solution

Keep this issue open for maintainer product/security design. The best path is an explicit opt-in route binding session target, such as `bindings[].session.target`, `sessionKey`, or a well-defined alias/scope, with validation limited to canonical agent session keys or approved aliases. Preserve default group/channel isolation, thread the target through route resolution and session-key construction, update generated config schema/help and public docs, and add regression tests proving a configured Discord channel can intentionally share an agent main/alias session while unconfigured channels remain isolated.
