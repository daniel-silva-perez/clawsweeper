## Summary

Keep this issue open. Current main still has only a DM-scope override on route bindings, while Discord guild/channel peers continue to resolve to isolated channel session keys and the generated schema rejects any binding-level session key, alias, scope, or target field. The related open identity hook work is DM-focused and does not supersede this non-DM binding override request.

## What This Changes

The issue requests a route-binding-level session override so a configured non-DM channel, such as a Discord guild channel, can intentionally share an agent session with Telegram or other direct-message sessions.

## Best Possible Solution

Keep this open for maintainer product/security design. The likely path is an explicit opt-in route binding session target, validated to canonical agent session keys or approved aliases, threaded through route resolution and session-key construction while preserving default group/channel isolation. Update schema/help/docs and add regression tests proving a configured Discord channel can share an agent main/alias session while unconfigured channels remain isolated.

## Risks / Open Questions

- Changing route/session-key construction also affects lastRoute, delivery, reset, status, and generated config schema/help surfaces.
