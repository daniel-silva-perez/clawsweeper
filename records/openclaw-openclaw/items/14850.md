## Summary

Keep open. Current main still only supports binding-level `dmScope` for direct-message peers; Discord guild/channel peers continue to resolve to isolated channel session keys. The requested cross-surface session unification is valid, but it crosses the current shared-channel/private-session boundary and needs maintainer product and privacy review before implementation.

## What This Changes

Add an optional binding-level session target or alias so a configured Discord guild/channel binding can share an agent's main or otherwise canonical session with that agent's direct-message surfaces.

## Best Possible Solution

Add an explicit opt-in route-binding session target contract in config schema/types and route resolution, validate it to safe canonical agent session targets, preserve default group/channel isolation, and update generated schema/help/docs plus regression coverage for configured channel-to-main sharing and unchanged default behavior.
