## Summary

Keep open. Current main still lacks the requested binding-level session target for unifying a Discord guild/channel route with an agent main or DM session; the implemented binding override is only `session.dmScope`, and non-direct peers still resolve to isolated channel/group keys. The existing maintainer review also kept this open because the remaining work crosses product and security boundaries.

## What This Changes

The issue requests an optional binding-level session key, alias, or scope override so matched guild/channel conversations can intentionally share an agent's main cross-channel session.

## Best Possible Solution

Add an explicit opt-in route-binding session target or alias contract in the config schema/types and route resolver, validate it to safe canonical agent session targets, preserve default group/channel isolation, and update generated schema/help/docs plus regression coverage for configured channel-to-main sharing and unchanged unconfigured channel behavior.
