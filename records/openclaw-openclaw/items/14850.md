## Summary

Keep open. Current main still lacks a binding-level session target, alias, or scope that can route a Discord guild/channel binding into an agent main or direct-message session; the existing binding override is limited to DM scoping, and current tests/docs intentionally preserve isolated channel sessions.

## What This Changes

The issue requests an opt-in binding-level session key, alias, or scope override so a bound guild/channel conversation can intentionally share an agent’s main or direct-message session across channels.

## Best Possible Solution

Add an explicit opt-in route-binding session target or alias contract in the config schema/types and route resolver, validate it to safe canonical agent session targets, preserve default group/channel isolation, and update generated schema/help/docs plus regression coverage for configured channel-to-main sharing and unchanged unconfigured channel behavior.
