## Summary

Keep open. Current `main` still has only a DM-scoping override on route bindings; it does not expose a binding-level session target, alias, or main-session override that can intentionally route a Discord guild/channel binding into the same session used by Telegram DMs. The previous maintainer and ClawSweeper comments remain consistent with the current source, generated schema, tests, and docs.

## What This Changes

The issue requests an optional binding-level session key, alias, or scope so routed guild/channel traffic can share an agent session with direct-message traffic across Telegram and Discord.

## Best Possible Solution

Add an explicit opt-in route-binding session target or alias contract in config schema/types and the route resolver, validate it to safe canonical agent session targets, keep unconfigured group/channel sessions isolated by default, and update generated schema/help/docs plus regression coverage for configured channel-to-main sharing and unchanged default behavior.
