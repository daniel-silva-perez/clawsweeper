## Summary

Keep open. Current main still lacks a binding-level session target or alias for regular route bindings; the implemented binding session option is limited to `dmScope`, and current code, tests, and docs still preserve isolated session keys for Discord guild/channel peers. The request is valid, but changing shared-channel conversations to target a private/main session needs maintainer product and privacy review before implementation.

## What This Changes

Request an optional binding-level session key, alias, or scope override so a Discord guild channel route binding can share an agent's main or DM session with Telegram/Discord direct surfaces.

## Best Possible Solution

Add an explicit opt-in route-binding session target or alias contract in the config schema/types and route resolver, validate it to safe canonical agent session targets, preserve default group/channel isolation, and update generated schema/help/docs plus regression coverage for configured channel-to-main sharing and unchanged unconfigured channel behavior.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for current behavior verification: a Discord channel peer route still resolves to an isolated `agent:<agentId>:discord:channel:<id>` key, and the current schema rejects any binding session field other than `dmScope`.

## Solution Assessment

Is this the best way to solve the issue? Unclear as proposed; a raw `sessionKey` on bindings may be too permissive, so the safer direction is a typed, validated, explicit opt-in target or alias contract with default channel isolation preserved.
