## Summary

Current main still lacks the requested binding-level session target for regular agent route bindings. The only route-binding session override is `dmScope`, it applies only to direct-message peers, and current tests/docs continue to preserve isolated session keys for Discord guild channels and other non-DM conversations.

## What This Changes

The issue requests an opt-in `bindings[]` session key, alias, or scope override so a non-DM channel binding can share an agent session with direct-message surfaces.

## Best Possible Solution

Add an explicit opt-in route-binding session target or alias contract in the config schema/types and route resolver, validated to safe canonical agent session targets, while preserving default group/channel isolation. Update generated schema/help/docs and regression coverage for configured channel-to-main sharing plus unchanged unconfigured channel behavior.
