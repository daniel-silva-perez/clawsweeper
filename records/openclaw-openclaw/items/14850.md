## Summary

Keep open. Current main still only supports `bindings[].session.dmScope` for direct-message scoping, while non-DM peers such as Discord guild channels continue to resolve to isolated channel session keys. The related merged DM-scope override and the open identity hook work do not provide a binding-level session key, alias, or target for channel peers.

## What This Changes

Request an optional route-binding session key, alias, or scope override so a bound Discord guild channel can share the same agent session as the agent's DMs across Telegram and Discord.

## Best Possible Solution

Add an explicit opt-in route-binding session target or alias contract in the config schema/types and route resolver, validated to safe canonical agent session targets, while preserving default group/channel isolation. Update generated schema/help/docs and add regression coverage proving configured channel-to-main sharing plus unchanged behavior for unconfigured channels.
