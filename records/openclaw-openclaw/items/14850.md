## Summary

Keep open. Current main still has only binding-level DM scoping, while Discord guild/channel peers continue to resolve to isolated channel session keys. The requested cross-channel session unification is valid, but it intentionally crosses the current shared-channel/private-session boundary and needs maintainer product and privacy review before implementation.

## What This Changes

Request adds an optional binding-level session key, alias, or scope so a Discord guild/channel binding can share an agent’s main or DM session with another surface such as Telegram DM.

## Best Possible Solution

Add an explicit opt-in route-binding session target contract in config schema/types and route resolution, validate it to safe canonical agent session targets, preserve default group/channel isolation, and update generated schema/help/docs plus regression coverage for configured channel-to-main sharing and unchanged default behavior.
