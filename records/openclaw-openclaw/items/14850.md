## Summary

Keep open. Current main still only supports binding-level DM scoping, and non-direct Discord guild/channel peers continue to resolve to isolated channel session keys; the requested cross-channel session unification is valid but needs maintainer product and privacy review before implementation.

## What This Changes

Add a binding-level session target, alias, or scope override so configured Discord guild/channel route bindings can share an agent's main/DM session across channels such as Telegram and Discord.

## Best Possible Solution

Add an explicit opt-in route-binding session target contract in config schema/types and resolve-route, validate it to safe canonical agent session targets, keep unconfigured group/channel sessions isolated by default, and update generated schema/help/docs plus regression coverage for configured channel-to-main sharing and unchanged default isolation.
