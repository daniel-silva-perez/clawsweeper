## Summary

Keep this issue open. Current main has substantial adjacent memory work, including memory_search/memory_get guidance, startup daily-memory preload for bare /new and /reset, compaction memory flush, and opt-in Active Memory, but it still does not close the reported default reliability gap for project/task/focus recall across realistic explicit-session conversations. There is still no documented canonical PROJECTS.md/TASKS.md open-task contract, Active Memory remains agent/config/session gated, the startup preload is scoped to bare reset flows, and OpenAI prompt cache keys are still forwarded from the session id without provider-length normalization.

## Best Possible Solution

Keep this issue open and split the remaining product work into concrete owner-owned pieces: add a repo-contained memory/task recall gate; decide and document whether OpenClaw supports a canonical project/task/focus workspace-state contract; make default recall prefer current canonical state over stale semantic hits where such state exists; ensure any OpenClaw-owned task surface defaults "what is left" prompts to open tasks only; and validate or normalize provider prompt-cache/session keys before sending them to OpenAI transports.
