## Summary

Keep this issue open. Current main has useful adjacent memory improvements, but it still does not provide a default, deterministic project/task/focus recall contract for explicit `openclaw agent --session-id` workflows, and the long session-id prompt-cache-key path still appears unresolved.

## Best Possible Solution

Keep this issue open and turn it into concrete maintainer-owned work: add a repo-contained memory/task recall gate, decide and document whether OpenClaw has a canonical project/task/focus workspace-state contract, make default recall prefer current canonical state over stale semantic hits where such state exists, define open-task retrieval semantics for any OpenClaw-owned task surface, and validate or normalize provider prompt-cache/session keys before OpenAI transport.
