## Summary

Keep open. Current main still has separate controls for backend selection, semantic index settings, QMD fallback, and compaction-time Markdown flushes; it does not provide the unified strict memory-mode contract requested here.

## What This Changes

The issue requests a stricter, team-consistent memory management contract, proposed as `memory.type` modes for builtin-only, qmd-only, and mixed behavior with workspace-scoped storage.

## Best Possible Solution

Define one coherent memory-mode contract covering durable writes, recall backend, QMD fallback policy, disabled memory, workspace and agent storage scope, migration for existing Markdown and SQLite artifacts, and user-facing config/docs guidance.
