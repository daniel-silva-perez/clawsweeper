## Summary

Keep open. Current main has clearer memory docs and QMD/builtin controls, but it still does not implement the requested strict single memory-mode contract. Builtin SQLite search, QMD indexing, daily Markdown flushes, QMD fallback, active memory, and SQLite store scoping remain separate controls, so the team-inconsistency problem can still occur by configuration or environment.

## What This Changes

The issue asks for a strict, documented memory management mode that makes builtin SQLite, QMD daily Markdown, mixed behavior, disabled memory, and workspace scoping behave consistently across users.

## Best Possible Solution

Treat this as a focused maintainer design task for `memory-core` plus config/schema/docs. The end state should define one coherent memory-mode contract or a documented successor that covers durable writes, recall/index backend, fallback behavior, workspace and agent storage scope, migration for existing Markdown and SQLite artifacts, and operator guidance.
