## Summary

Keep open. Current main still has separate controls for the global memory backend, builtin SQLite index storage, memory-search enablement/provider settings, QMD indexing/fallback behavior, and compaction-time Markdown flushes; it does not provide the strict builtin-only/qmd-only/mixed/disabled memory-mode contract requested here.

## What This Changes

The issue requests a stricter, unified memory-management contract so OpenClaw instances do not diverge between builtin SQLite, daily Markdown/QMD, mixed, or disabled behavior.

## Best Possible Solution

Define one coherent memory-mode contract or documented successor covering durable writes, recall backend, QMD fallback policy, disabled memory, workspace and agent storage scope, migration for existing Markdown and SQLite artifacts, and user-facing config/docs guidance.
