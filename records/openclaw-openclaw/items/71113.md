## Summary

Keep open. Current main still lacks the proposed AGENTS.md size guard, so the PR is distinct work, but the patch has a blocking implementation bug: its path normalizer only splits on the host separator, which leaves Windows-style backslash paths unchanged on Linux and makes the new exemption test fail. The security pass found no concrete supply-chain or secrets concern.

## What This Changes

The PR adds a TypeScript AGENTS.md line-count guard with exemptions, wires it into package.json and the check-additional boundary shard, and adds Vitest coverage for offenders, exemptions, sorting, and the default cap.

## Best Possible Solution

Normalize both forward slashes and backslashes before comparing exemption paths, keep the focused guard/tests if maintainers accept the policy, then validate the script and boundary integration before merge.

## Reproduction Assessment

Yes for the review finding. On Linux, path.sep is /, so the proposed normalizer leaves docs\reference\templates\AGENTS.md unchanged and the added Windows-style exemption assertion would fail.

## Solution Assessment

No as proposed. The guard belongs in the expected script/check-additional surface, but the normalizer should handle both separators, for example with a slash/backslash regex or explicit backslash replacement before exemption comparison.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:
