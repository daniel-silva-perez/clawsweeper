## Summary

Keep open. Current main already handles the workspace-file half through shared Project Context/bootstrap injection, but Slack still lacks an account/global `systemPrompt` field in the typed config, strict schema, runtime prompt resolver, message/slash paths, and docs. The prior maintainer and ClawSweeper comments remain accurate after rechecking current main.

## What This Changes

The issue requests Slack parity by adding a top-level/account-level `channels.slack.systemPrompt` that applies across Slack conversations unless overridden, and by ensuring workspace bootstrap files are injected for Slack sessions.

## Best Possible Solution

Keep workspace-file injection on the shared Project Context path, and add a documented Slack account/global prompt contract that validates in strict config, inherits through account resolution, stacks predictably with per-channel overrides, and reaches both normal message and native slash-command inbound contexts.

## Reproduction Assessment

Not applicable as a traditional runtime bug, but there is a high-confidence code-inspection reproduction path: current main has no typed/schema/runtime path for `channels.slack.systemPrompt`, and Slack prompt context reads only per-channel `systemPrompt`.

## Solution Assessment

Yes. A narrow Slack config/schema/runtime/docs/test change is the best direction; adding a Slack-specific workspace bootstrap injector would duplicate the generic Project Context mechanism that already handles AGENTS.md, SOUL.md, MEMORY.md, and related files.
