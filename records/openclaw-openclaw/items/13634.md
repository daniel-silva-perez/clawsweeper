## Summary

Keep open. Current main covers the workspace-file part through shared Project Context/bootstrap injection, but the requested Slack account/global `systemPrompt` surface is still absent from typed config, strict schema, Slack runtime prompt resolution, and docs; the related Slack file-access PR is about media hydration, not this prompt surface.

## What This Changes

The issue requests a Slack account/global `systemPrompt` that applies across Slack conversations, plus workspace file injection parity with Telegram.

## Best Possible Solution

Add a documented Slack account/global prompt field in the Slack config contract, validate it in the strict schema, carry it through account resolution, and feed it into Slack message and native slash command prompt context for DMs, MPIMs, public/private channels, and threads while leaving workspace-file bootstrap injection as the shared Project Context mechanism.

## Reproduction Assessment

Not applicable as a runtime bug report, but the missing feature is reproducible by code inspection: `channels.slack.systemPrompt` has no Slack account/schema field, and Slack prompt context currently reads only per-channel `systemPrompt`.

## Solution Assessment

Yes, a narrow Slack config/runtime/docs change is the best path; changing workspace bootstrap injection or adding a new core file-injection mechanism would solve the already-implemented half and risk disturbing shared prompt behavior.
