## Summary

Current main still does not implement the remaining requested Slack account/global `channels.slack.systemPrompt` surface. Shared workspace bootstrap / Project Context injection now covers the AGENTS.md/SOUL.md/MEMORY.md-style part, but Slack still only accepts `systemPrompt` on nested per-channel entries and applies it only when a room channel resolves a `channelConfig`; DMs and MPIM/group DMs do not get an account-level prompt fallback.

## Best Possible Solution

Keep this issue open and implement the remaining account-level Slack prompt surface. Add `channels.slack.systemPrompt` and account override support to Slack config types/schema/docs/config surfaces, then plumb it through Slack message and native slash-command paths with precedence: exact channel entry, wildcard channel entry, then account/global prompt. Apply it consistently for public channels, private channels, DMs, and MPIM/group DMs. Do not add Slack-specific AGENTS.md/SOUL.md injection; shared Project Context already handles workspace bootstrap files.
