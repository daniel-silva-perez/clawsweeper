## Summary

Keep open. Current main already handles the workspace-file part through shared Project Context/bootstrap injection, but Slack still lacks the requested account/global `systemPrompt` contract in typed config, strict schema, runtime prompt resolution, and docs.

## What This Changes

Request Slack workspace-file context parity and a Slack account/global `systemPrompt` so operators do not have to repeat the same prompt per Slack room.

## Best Possible Solution

Keep workspace-file loading on the shared Project Context path, and add a documented Slack account/global prompt field that validates in strict config, inherits through base-to-account resolution, stacks predictably with per-channel overrides, and reaches Slack rooms, private channels, DMs, MPIM/group DMs, and native slash commands.

## Reproduction Assessment

Yes. Configure `channels.slack.systemPrompt` against current main: the strict Slack schema has no such account/global key, and runtime prompt assembly only reads per-channel `systemPrompt`.

## Solution Assessment

Partly. The account/global prompt inheritance request is the narrow maintainable fix; adding a separate Slack-specific workspace-file injector would duplicate the generic Project Context mechanism that already covers workspace files.
