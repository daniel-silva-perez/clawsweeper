## Summary

Keep open. Current main covers the workspace-file side through shared Project Context/bootstrap injection, but Slack still does not expose or apply the requested account/global `systemPrompt`; the typed config, strict schema, docs, normal-message path, and native slash-command path only show per-channel prompt handling.

## What This Changes

The issue requests Slack parity with Telegram by adding a global Slack `systemPrompt` and Slack workspace file injection behavior.

## Best Possible Solution

Add a documented Slack account/global prompt field that inherits through base-to-account config, applies consistently across Slack DMs, MPIM/group DMs, public/private channels, and native slash commands, and composes predictably with per-channel prompts while leaving shared Project Context as the workspace-file mechanism.
