## Summary

Current main covers the workspace-file half through shared Project Context, but the remaining Slack account/global `systemPrompt` surface is still missing. Slack only validates and resolves `systemPrompt` on per-channel entries or the `channels['*']` wildcard, and DMs/MPIMs do not get that channel config path.

## Best Possible Solution

Keep this issue open and implement the remaining Slack account/global prompt surface only. Add `channels.slack.systemPrompt` plus per-account override support, apply it consistently to Slack message and native slash-command paths for channels, private channels, DMs, and MPIM/group DMs, and keep shared Project Context as the workspace-file solution instead of adding Slack-specific workspace injection.
