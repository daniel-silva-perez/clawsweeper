## Summary

Keep open. Current main already covers the workspace-file part through the shared Project Context/bootstrap prompt path, but the requested Slack account/global `systemPrompt` surface is still missing from the typed config, strict schema, prompt resolver, runtime message/slash paths, and docs.

## What This Changes

Request to add Slack-wide `channels.slack.systemPrompt` plus Slack workspace-file context injection so Slack has prompt/context parity with Telegram-style channel behavior.

## Best Possible Solution

Add a documented Slack account/global prompt field to the config contract, validate it in strict schema, carry it through account resolution, and feed it consistently into Slack DMs, MPIMs, public/private channels, threads, and native slash commands while keeping workspace-file injection on the shared Project Context path.

## Reproduction Assessment

Not applicable as a runtime bug report, but the missing feature is reproducible by code inspection: current main has no strict-schema or runtime path for `channels.slack.systemPrompt`, and Slack prompt context reads only per-channel `systemPrompt`.

## Solution Assessment

Yes, a narrow Slack config/runtime/docs/test change is the best path. Adding another workspace-file injection mechanism would duplicate the shared Project Context behavior that already handles AGENTS.md, SOUL.md, MEMORY.md, and related files.
