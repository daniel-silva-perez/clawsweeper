## Summary

Current main covers the workspace-file half through shared Project Context/bootstrap files, but the Slack account/global `systemPrompt` surface requested here is still missing. Slack still exposes `systemPrompt` only on `channels.slack.channels.<id>` entries and the `*` wildcard path, and the message/slash-command paths only resolve that per-channel config for room channels, leaving DMs and MPIM/group DMs without a global Slack prompt fallback.

## Best Possible Solution

Keep this issue open and implement only the remaining Slack account/global prompt surface. Add `channels.slack.systemPrompt` plus per-account override support, apply it consistently to Slack message and native slash-command paths for channels, private channels, DMs, and MPIM/group DMs, and keep shared Project Context as the workspace-file solution instead of adding Slack-specific workspace injection.
