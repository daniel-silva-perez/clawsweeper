## Summary

Current main still leaves the remaining requested Slack account/global `channels.slack.systemPrompt` surface unimplemented. The workspace bootstrap-file part is now covered by shared Project Context injection, but Slack still only accepts and applies `systemPrompt` on nested per-channel entries or wildcard channel entries, and DMs/MPIMs have no account-level prompt fallback.

## Best Possible Solution

Keep this issue open and implement the remaining account-level Slack prompt surface. Add `channels.slack.systemPrompt` to Slack config types, strict schema, generated config metadata/UI hints, docs, and runtime prompt plumbing. Preserve precedence as exact channel entry, wildcard channel entry, then account/global prompt, and apply it consistently to Slack message and native slash-command paths across public channels, private channels, DMs, and MPIM/group DMs. Do not add Slack-specific AGENTS.md/SOUL.md injection; the shared Project Context bootstrap path already handles that part.
