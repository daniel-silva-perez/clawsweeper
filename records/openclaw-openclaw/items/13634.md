## Repair Work Prompt

Implement a narrow fix for this issue: add Slack account/global `systemPrompt` support without adding a separate Slack-specific workspace-file injector. Current main already injects workspace files through shared Project Context. Add `systemPrompt?: string` to the Slack account/global config surface and strict schema, include it in Slack account surface/resolution as needed, and compose it predictably with per-channel and `*` wildcard prompts. Apply the composed prompt consistently for Slack normal messages and native slash commands across DMs, MPIM/group DMs, public/private channels, and room channels. Keep Project Context stable above the prompt-cache boundary; the Slack account/global prompt should remain channel/session guidance, not workspace bootstrap content. Update Slack and gateway config docs, and add a user-facing changelog entry with `Thanks @visshehi`. Do not change Slack media/file ingestion behavior or try to resolve related #62792 in this PR.
