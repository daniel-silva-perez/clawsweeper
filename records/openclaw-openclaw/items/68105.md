## Summary

Keep open. Current main still confines RTL RLI/PDI isolation to the TUI formatter, while the shared assistant-visible/user-facing sanitizers, gateway live chat projection, and outbound delivery path still pass Hebrew/Arabic text through without a shared bidi-isolation stage. The related #21936 work proves the TUI-only fix, #32852 is a Telegram preview-specific prior report, and no open PR search found a current implementation paired with this issue.

## What This Changes

The issue requests shared RTL bidi isolation for gateway and outbound replies so Hebrew/Arabic punctuation renders correctly on non-TUI channels.

## Best Possible Solution

Add a shared RTL isolation helper under src/shared/text, reuse it from the TUI formatter, and apply it at the user-visible outbound boundary before channel chunking/sending; include focused tests for the shared helper, TUI reuse, outbound delivery, and any gateway live projection path that should be covered.
