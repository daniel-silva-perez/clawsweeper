## Summary

Keep this open. Current main still has a source-reachable path where `channels.modelByChannel` selects the runtime model for `/status`, but a stale positive `sessionEntry.contextTokens` remains preferred over the active channel model's context window when the selected and active labels match.

## What This Changes

The issue asks for `/status` to show the active `modelByChannel` model's context window instead of a stale persisted session `contextTokens` value from a previous model.

## Best Possible Solution

Teach status/session reconciliation to treat a channel-selected model as authoritative for the displayed context window when there is no explicit session model override, while preserving `/model` precedence and fallback runtime snapshot behavior.
