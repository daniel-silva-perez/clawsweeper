## Summary

Keep open. Current main has partial context-token hardening for fallback and explicit session model override paths, but the reported `channels.modelByChannel` `/status` path is still source-reachable: a stale positive `sessionEntry.contextTokens` can remain authoritative over the channel-selected model's computed context window.

## Best Possible Solution

Keep this issue open and fix `/status` context-token reconciliation for channel-selected models. A narrow fix should detect when `channels.modelByChannel` selected the active model without an explicit session `/model` override, then ignore, clear, or recompute stale `sessionEntry.contextTokens` before status formatting. Add a regression test with a stale 1M persisted session context, a channel override to a smaller MiniMax-style context window, and `/status` showing the active model's configured window.
