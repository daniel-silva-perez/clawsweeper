## Summary

Keep open. Current main still preserves the reported failure path: `--profile` sets `OPENCLAW_PROFILE` without clearing `OPENCLAW_LAUNCHD_LABEL`, and launchd label resolution still prefers `OPENCLAW_LAUNCHD_LABEL` before deriving the label from `OPENCLAW_PROFILE`. PR #66000 is an open, unmerged implementation candidate that explicitly links this issue for closure, so this issue should remain open until that PR is reviewed and landed or rejected.

## Best Possible Solution

Keep this issue open and review PR #66000 as the active implementation candidate. The best fix is to make an explicit `--profile` prevent a stale inherited `OPENCLAW_LAUNCHD_LABEL` from overriding the selected profile’s launchd plist, while preserving any intentional override behavior maintainers want, and to add focused regression tests for conflicting, matching, empty, and absent label cases.
