## Summary

Current main still preserves the reported failure path: explicit `--profile` updates `OPENCLAW_PROFILE` but leaves an inherited `OPENCLAW_LAUNCHD_LABEL` intact, and the macOS launchd resolver still lets that inherited label win. PR #66000 is an open, unmerged closing implementation candidate, so this issue should remain open until that PR is reviewed and landed or rejected.

## What This Changes

Explicit `--profile` gateway commands should stop a stale inherited `OPENCLAW_LAUNCHD_LABEL` from making launchd status resolve the wrong profile plist.

## Best Possible Solution

Keep this issue open and review PR #66000 as the active implementation candidate. The narrow fix belongs in CLI profile environment handling and focused regression tests, preventing a conflicting inherited launchd label from overriding an explicit profile while preserving any intentional same-profile/custom-label behavior maintainers want.
