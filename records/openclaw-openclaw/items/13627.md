## Summary

Keep open. Current main still lacks a configurable Slack outbound minimum-delay throttle: Slack sends are serialized per target/thread, but there is no timestamp-spacing delay in the normal send path, multipart chunk path, slash response_url follow-up path, config schema/types, docs, or tests. The related PR #26732 is useful prior art but was closed unmerged.

## Best Possible Solution

Implement a focused Slack-owned outbound minimum-delay throttle in the bundled Slack plugin. Add a typed, strict-schema-validated, documented per-account/top-level setting with a default around one second, apply it to normal channel sends, DMs, thread replies, multipart chunks, and in-scope slash response_url follow-ups, and keep the fix out of Telegram/Discord/core unless a generic seam is clearly needed later.
