## Summary

Keep open. Current main serializes Slack sends per account/token/recipient/thread, but it still does not provide the requested configurable minimum-delay throttle for Slack timestamp ordering; multi-part Slack post loops and slash-command follow-ups can still emit consecutive messages without any spacing, and the related broad outbound throttle PR was closed unmerged.

## What This Changes

Add a configurable Slack outbound send throttle/minimum inter-message delay so rapid Slack replies do not display out of order when Slack groups messages at roughly one-second timestamp resolution.

## Best Possible Solution

Implement a Slack-owned throttle setting with typed config, strict schema/UI hints, docs, and focused tests, then apply it consistently after the existing per-target serialization across normal posts, DMs, thread replies, block/text chunks, media-caption follow-ups, and slash response_url follow-ups.
