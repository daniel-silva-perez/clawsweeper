## Summary

Current main still only serializes Slack sends; it does not provide the requested configurable minimum-delay throttle for Slack timestamp ordering. The prior broad PR that referenced this work is closed unmerged, and there is no open replacement PR found for this issue.

## Best Possible Solution

Keep this issue open and implement a focused Slack-owned outbound minimum-delay throttle in the bundled Slack plugin. Add a typed strict-schema setting, document it, expose UI/config hints, and apply the spacing to normal channel sends, DMs, thread replies, block sends, multipart chunks, media-caption follow-ups, and slash response_url follow-ups without adding generic Telegram/Discord/core behavior unless a shared seam proves necessary.
