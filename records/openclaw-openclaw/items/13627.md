## Summary

Keep open. Current main serializes Slack sends per account/token/recipient/thread, but it still does not implement the requested configurable minimum delay for Slack timestamp ordering; multipart post paths and slash response_url follow-ups can still emit consecutive Slack messages without spacing, and the linked broad throttle PR was closed unmerged.

## What This Changes

Add a configurable Slack outbound send throttle so rapid sequential Slack channel messages, thread replies, and DMs receive distinct timestamps and display in order.

## Best Possible Solution

Implement a Slack-owned account/global delivery setting with typed config, schema/UI hints, docs, and focused tests, then apply it consistently after the existing per-target serialization across normal posts, DMs, thread replies, block/text chunks, media-caption follow-ups, and slash response_url follow-ups.
