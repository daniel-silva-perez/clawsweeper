## Summary

Keep open. Current main serializes Slack sends per account/token/recipient/thread, but it still lacks the requested configurable minimum-delay throttle for Slack timestamp ordering; multipart post paths and slash-command follow-ups can still emit consecutive Slack messages without spacing, and the related broad throttle PR is closed unmerged.

## What This Changes

The issue requests a configurable Slack outbound send throttle so rapid consecutive Slack messages receive distinct timestamps and preserve display ordering.

## Best Possible Solution

Implement a Slack plugin/account-level delivery setting with typed config, schema/UI hints, docs, and focused tests, then apply it consistently after the existing per-target serialization across normal posts, DMs, thread replies, block/text chunks, media-caption follow-ups, and slash response_url follow-ups.
