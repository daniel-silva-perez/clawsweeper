## Summary

Keep open. Current main still serializes Slack sends but does not implement a configurable outbound minimum-delay throttle for Slack timestamp ordering. The normal send path, multipart chunk path, slash response_url follow-up path, config types/schema, UI hints, docs, and tests still lack a Slack send-spacing setting. The related broad PR #26732 is closed unmerged, so there is no merged implementation to treat as superseding this issue.

## Best Possible Solution

Implement a focused Slack-owned outbound minimum-delay throttle in the bundled Slack plugin. Add a typed, strict-schema-validated, documented per-account/top-level setting, default around one second unless maintainers choose another value, and apply it to normal channel sends, DMs, thread replies, multipart chunks, block/payload sends, and in-scope slash response_url follow-ups. Keep this out of Telegram/Discord/core unless implementation proves a generic channel seam is needed.
