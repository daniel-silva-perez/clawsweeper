## Summary

Keep open. Current main serializes Slack sends per account/token/recipient/thread, but it still does not implement the requested configurable minimum-delay throttle for Slack timestamp ordering, and the only linked implementation PR was closed unmerged.

## What This Changes

Add a Slack-specific configurable minimum delay between sequential outbound messages so rapid Slack replies get distinct timestamps and preserve display order.

## Best Possible Solution

Implement a focused Slack-owned throttle setting with typed config, schema/UI hints, docs, and tests, then apply it after the existing per-target serialization across normal posts, DMs, thread replies, block/text chunks, media-caption follow-ups, and slash response_url follow-ups.
