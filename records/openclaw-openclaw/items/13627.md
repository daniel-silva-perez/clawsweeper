## Summary

Keep open. Current main still serializes Slack sends per account/token/recipient/thread, but it does not implement the requested configurable minimum-delay throttle for Slack timestamp ordering, and the related broad outbound throttle PR is closed unmerged.

## What This Changes

Add a configurable Slack-specific outbound message throttle so rapid sequential Slack channel messages, thread replies, and DMs receive distinct timestamps and display in order.

## Best Possible Solution

Implement a focused Slack-owned throttle setting with typed config, strict schema/UI hints, docs, and tests, then apply it consistently after the existing per-target serialization across normal posts, DMs, thread replies, block/text chunks, media-caption follow-ups, and slash response_url follow-ups.
