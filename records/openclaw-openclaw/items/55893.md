## Summary

Keep this PR open. Current main still does not classify `deactivated_workspace` or `workspace deactivated` as `auth_permanent`, and the PR’s regression coverage is not present. The related workspace-selection issue #56693 is broader and does not supersede this narrow failover-classification fix.

## Best Possible Solution

Keep this PR open and land an equivalent narrow fix after maintainer review. Prefer adding `deactivated[_ ]workspace` or `workspace deactivated` classification in the OpenAI Codex provider-owned failover hook with regression coverage through `resolveFailoverReasonFromError` or the provider dispatch path; use the PR’s generic matcher approach only if maintainers want this marker treated as shared auth-permanent behavior.
