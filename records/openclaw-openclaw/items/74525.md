## Summary

Keep this PR open. Current main still drops the target agent auth directory before `/models` visibility filtering, so the regression is real, but the proposed patch leaves `buildModelsProviderData` opt-in for `agentDir`; existing bundled callers that pass only `agentId` would still read the default auth store.

## What This Changes

The PR threads `agentDir` and `env` through the `/models` reply path into model catalog visibility filtering and adds a regression test for a worker-agent auth store.

## Best Possible Solution

Complete the fix at the shared provider-data boundary so any caller with an `agentId` uses that agent's auth store by default, preserve explicit `agentDir` overrides, keep the handler regression coverage, and then run the changed gate before merge.
