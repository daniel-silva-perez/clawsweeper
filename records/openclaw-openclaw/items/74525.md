## Summary

Keep this PR open for maintainer handling. Current main still has the reported `/models` auth-scope gap, but this PR only fixes the direct command path and leaves shared `buildModelsProviderData(cfg, agentId)` callers using the default auth store; the open replacement #74564 carries the shared-boundary shape, so maintainers should choose one canonical branch.

## What This Changes

The PR threads `agentDir` and `env` through the direct `/models` reply path into provider visibility filtering, adds a worker-agent auth regression test, and moves the related changelog entry.

## Best Possible Solution
