## Summary

Keep this PR open. Current main still scopes Control UI token and session-selection storage by exact parsed gateway host, so the intended localhost/127/[::1] loopback sharing is not implemented. The PR targets the right storage surface and the author addressed the earlier IPv6/dead-code review, but the current diff still has a security/correctness issue around `127.*` hostnames and its tests are stale against the current `UiSettings` shape.

## Best Possible Solution

Keep the PR open and request a focused revision in `ui/src/ui/storage.ts`: normalize only `localhost` plus actual numeric IPv4/IPv6 loopback literals, preserve protocol/port/path boundaries, add positive localhost/127/[::1] same-port sharing tests, add negative DNS-hostname tests, update fixtures for the current `UiSettings` fields, then run the targeted Control UI storage check before landing.

## Risks / Open Questions

- Loopback-alias token fragmentation remains unresolved on current main until a corrected implementation lands.
- Landing the PR as-is could share a browser session token between a real loopback gateway and non-loopback DNS hostnames whose names start with `127.`.
- The PR tests need rebasing for the current `UiSettings` contract and should add negative coverage for hostnames such as `127.example.com` and `127.0.0.1.nip.io`.
