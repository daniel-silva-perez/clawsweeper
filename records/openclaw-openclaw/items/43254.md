## Summary

Keep this PR open. Current main still scopes Control UI token and session-selection storage by the exact parsed gateway URL, so the localhost/127.0.0.1/[::1] loopback sharing this PR proposes has not been implemented. The PR targets the right storage surface, but the current patch needs a focused security/correctness revision for 127.* DNS hostnames and a rebase to the current UiSettings shape before it is merge-ready.

## Best Possible Solution

Keep the PR open and request a focused revision in ui/src/ui/storage.ts: normalize only localhost plus actual numeric IPv4 loopback literals and bracketed IPv6 loopback, preserve protocol/port/path boundaries, add positive localhost/127/[::1] same-port sharing tests, add different-port and DNS-hostname negative tests, update fixtures for the current UiSettings fields, then run the targeted Control UI storage test before maintainer review/merge.
