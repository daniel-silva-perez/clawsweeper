## Summary

Keep this PR open. Current main still scopes Control UI session tokens by the exact normalized gateway URL, so the loopback-alias UX problem is not implemented on main. The PR targets the right storage surface and the author addressed the initial IPv6/dead-code review, but the current diff still has an unresolved security/correctness issue because `host.startsWith("127.")` also matches DNS hostnames like `127.example.com` or `127.0.0.1.nip.io`. The branch also needs rebasing against the current `UiSettings` shape before it can be landed.

## Best Possible Solution

Keep this PR open and request a focused revision: normalize only `localhost` and actual numeric loopback IP literals, preserve protocol/port/path isolation, add negative tests for DNS names such as `127.example.com` and `127.0.0.1.nip.io`, update the tests for the current `UiSettings` fields, then run the appropriate targeted UI changed gate before landing.
