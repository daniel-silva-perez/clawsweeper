## Summary

Keep this PR open, but do not merge it as-is. Current main still lacks the requested `devices list` metadata display, and the PR branch has blocking issues: the added pending-device test has no matching pending-table implementation, and the paired-table rewrite drops existing terminal-output sanitization for device-controlled fields.

## What This Changes

The PR adds client/OS/last-active formatting helpers and paired-device table columns for `openclaw devices list`, adds CLI tests for pending and paired metadata output, and updates the `devices remove` description.

## Best Possible Solution

Revise the PR so both pending and paired device tables expose the requested device ID/name/client/OS/last-active details where data exists, preserve `sanitizeForLog` on all device-controlled terminal cells, and keep #23916 as the broader tracking issue for future device-list detail or rename workflows.
