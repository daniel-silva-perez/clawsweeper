## Summary

Keep open. Current main still lacks the requested richer device-list display, but this PR is not mergeable as-is: the added pending-device test is not matched by a production pending-table change, and the paired-table change drops existing terminal sanitization for device-controlled metadata.

## What This Changes

The branch adds optional device metadata types/helpers, changes paired `devices list` output to show Device ID/Name/Client/OS/IP/Last active, adds CLI tests for pending and paired metadata, and updates `devices remove` help text.

## Best Possible Solution

Revise the PR so pending and paired device tables both expose Device ID, Name, Client, OS, and activity data where available, preserve `--json`, sanitize every device-controlled terminal cell, add focused regression assertions, and update `docs/cli/devices.md` if the human-readable output changes.
