## Summary

Keep this PR open. Current main still has the Feishu HTTP shutdown behavior the PR targets: server.close() is called without awaiting the close callback, state maps are cleared immediately, and the public stop wrapper remains synchronous. The linked bug #48183 remains open, and this PR is a focused implementation candidate rather than obsolete or already implemented.

## Best Possible Solution

Keep this PR open as the focused Feishu implementation candidate for #48183. The best path is to review the final diff against the current plugin lifecycle, settle the public stop contract, add targeted cleanup/restart coverage if feasible, and then land this PR or a maintainer-owned equivalent.
