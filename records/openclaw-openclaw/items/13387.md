## Summary

Keep open. Current main still lacks a replay-context pass that collapses adjacent assistant silent replies; the embedded runner sanitizes, validates, heartbeat-filters, limits, and repairs tool-pairing without pruning consecutive `NO_REPLY` or silent-envelope assistant turns. The prior implementation PR is closed and unmerged, and current docs still describe transcript hygiene without this behavior, so the requested fix remains actionable.

## What This Changes

Request built-in transcript/context hygiene that prunes consecutive assistant `NO_REPLY` messages, keeping at most one adjacent silent reply to avoid silence cascades.

## Best Possible Solution

Add documented in-memory replay-history hygiene in the embedded runner that collapses only adjacent assistant silent replies, keeps one by default, reuses `isSilentReplyPayloadText`, handles string/text-block/strict JSON silent envelopes, and leaves persisted transcripts unchanged.

## Reproduction Assessment

Yes. A replay transcript containing adjacent assistant messages whose content is `NO_REPLY` or a strict silent-reply envelope will pass through the current embedded-runner source path unchanged unless it is a recognized heartbeat prompt/ack pair; source inspection gives a high-confidence reproduction path without a live model run.

## Solution Assessment

Yes. The narrow maintainable path is a replay-context pruning helper wired into the existing embedded-runner hygiene flow with focused tests; the closed prior PR is useful context, but a replacement should avoid a broad new config surface unless maintainers explicitly want one.
