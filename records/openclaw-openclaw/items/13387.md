## What This Changes

Request to collapse consecutive assistant `NO_REPLY` or silent-envelope turns from the model replay transcript while keeping isolated silent replies intact.

## Best Possible Solution

Add documented in-memory replay-history hygiene in the embedded runner that collapses only adjacent assistant silent replies, keeps one by default, reuses the shared silent-reply payload detector, handles legacy string/text-block/strict JSON silent envelopes, and leaves stored JSONL unchanged.

Yes. ## Solution Assessment

Yes. The narrow maintainable path is a replay-context pruning helper wired into the existing embedded-runner hygiene flow with focused tests and docs; the closed prior PR is useful prior art, but a replacement should avoid adding a broad config surface unless maintainers explicitly want one.
