## Summary

Keep open: current main still does not collapse consecutive assistant silent replies before replay, and the linked implementation PR is closed/unmerged. The affected embedded-runner replay path, shared silent-token detector, docs, and tests make this a narrow valid follow-up rather than a cleanup close.

## What This Changes

The issue requests pruning adjacent assistant NO_REPLY transcript turns before replay so repeated silent replies do not bias future model responses.

## Best Possible Solution

Add an in-memory replay-history pruning step in the embedded runner that collapses only adjacent assistant silent replies, keeps one by default, reuses the shared silent-token detection contract, handles legacy string/text-block/JSON-envelope and known empty silent-stop shapes, and documents the transcript-hygiene behavior without rewriting stored transcripts.
