## Summary

Keep open. Current main still sends replay context through sanitization, validation, heartbeat filtering, history limiting, and tool-use/result repair, but has no pass that collapses consecutive `NO_REPLY` or other silent assistant turns. The linked implementation PR is closed and unmerged, and the latest maintainer comment already routes this as a narrow embedded-runner follow-up.

## What This Changes

Request to auto-prune consecutive `NO_REPLY` assistant messages from replay transcripts while preserving isolated intentional silent replies.

## Best Possible Solution

Add a focused in-memory replay-context pruning step in the embedded runner history pipeline that collapses only consecutive assistant silent replies, keeps one by default, uses the shared silent-token detection helpers, covers legacy text/string/JSON-envelope and empty silent-stop shapes, and documents the transcript-hygiene behavior.
