## Summary

Keep open. Current main still sends replay history through sanitization, validation, heartbeat filtering, history limiting, and tool-pairing repair without any pass that collapses adjacent assistant `NO_REPLY` or silent-envelope turns. The linked implementation PR is closed and unmerged, and the docs still describe transcript hygiene without this behavior, so the remaining work is a narrow valid fix rather than cleanup-close material.

## What This Changes

Request to prune adjacent assistant silent replies from model replay context while preserving isolated `NO_REPLY` turns as legitimate silence decisions.

## Best Possible Solution

Add documented in-memory replay-history hygiene in the embedded runner that collapses only adjacent assistant silent replies, keeps one by default, reuses the shared silent-token detector, and leaves persisted transcripts unchanged.

## Reproduction Assessment

Yes. A replay transcript containing adjacent assistant messages whose content is `NO_REPLY` or a strict silent-reply envelope will pass through the current source path unchanged unless it is part of a heartbeat prompt/ack pair; the source inspection gives a high-confidence reproduction path without needing a live model run.

## Solution Assessment

Yes. The narrow maintainable fix is a replay-context pruning helper wired into the existing embedded-runner hygiene path, using `isSilentReplyPayloadText` and focused tests; the older PR is useful source context but should be adjusted rather than copied wholesale.
