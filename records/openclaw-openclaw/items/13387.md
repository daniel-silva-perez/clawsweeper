## Summary

Current main still does not implement consecutive NO_REPLY/silent-reply pruning in the replay context. The embedded runner sanitizes, validates, heartbeat-filters, history-limits, and repairs tool-use/result pairing, but there is no stage that collapses consecutive assistant silent turns. The linked implementation PR #13388 is closed and unmerged, so this remains a valid, narrow follow-up.

## Best Possible Solution

Keep this issue open and implement a focused in-memory replay-context prune in the embedded runner history pipeline. Collapse only consecutive assistant silent replies, keep at most one by default, reuse shared silent-token detection, handle legacy string/text-block/JSON-envelope and clean empty-content silent-stop shapes, preserve tool-use/tool-result pairing, and add focused replay-context tests. Treat closed PR #13388 as prior art, not merged work.
