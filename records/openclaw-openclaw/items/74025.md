## Summary

Keep open. Current main still provides Lobster run/resume and managed Task Flow support through the optional agent tool path, plus Task Flow inspect/cancel surfaces, but it does not provide the requested manual channel slash command; the remaining work needs maintainer design because it is a security-sensitive command surface.

## What This Changes

The issue requests a channel slash command for manually executing Lobster or Task Flow workflows with optional JSON arguments.

## Best Possible Solution

Keep this open for maintainer design. If approved, the narrowest end state is a Lobster-owned plugin command using the existing run/resume and managed Task Flow runtime, with docs and focused tests for JSON args, flow tracking, authorization, approval handling, and safe error reporting; core should only gain generic SDK seams if that implementation exposes a real gap.

## Reproduction Assessment

Not applicable as a runtime bug, but there is a high-confidence verification path. Source and docs inspection on current main show no manual run slash command: /tasks rejects arguments, Lobster registers only an optional tool, and the slash-command docs list no /lobster or /taskflow command.

## Solution Assessment

Unclear until maintainers settle the security and UX contract. A bundled Lobster plugin command via api.registerCommand looks narrower than adding core-specific workflow execution, but the command should not be automated before that contract is approved.
