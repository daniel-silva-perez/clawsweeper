## Summary

Keep open. Current main still has Lobster as an optional agent tool and Task Flow inspection/cancel surfaces, but no explicit channel slash command that manually runs or resumes a Lobster/Task Flow workflow.

## What This Changes

Add a channel slash command to manually execute Task Flow or Lobster workflows with optional JSON arguments.

## Best Possible Solution

Keep the issue open for maintainer design. If approved, the likely direction is a Lobster-owned `api.registerCommand()` implementation that reuses the existing runner and managed TaskFlow runtime, with docs and focused tests for JSON args, run/resume, approval handling, authorization, and safe error messages.

## Reproduction Assessment

Not applicable as a runtime bug, but there is a high-confidence verification path: source and docs inspection show no `/lobster`, `/taskflow`, or flow-run command on current main, while `/tasks` is status-only.

## Solution Assessment

Unclear until maintainers settle the command contract. A plugin-owned command wrapping the existing Lobster tool/runtime looks narrower than adding core-specific workflow execution, but the current issue should stay open for that design decision.
