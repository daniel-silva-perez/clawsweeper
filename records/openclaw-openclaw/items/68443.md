- The PR's threadId fallback should be fixed before merge; Feishu thread IDs should not be sent as chat IDs to container_id_type="chat".

- There is overlap with open #41306, so maintainers should choose one canonical Feishu chat-history direction rather than landing divergent implementations.
- Adding a new shared message action name touches the plugin contract surface; reviewers should confirm list is the right generic action rather than a Feishu-specific tool action.
