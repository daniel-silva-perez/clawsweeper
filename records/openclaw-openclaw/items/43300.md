## Summary

Keep open. Current main has a documented trusted-proxy local password fallback and explicit same-host proxy opt-in, but the originally reported headerless internal self-connection shape still fails without a configured password, and an open unmerged implementation candidate still references this issue with closing syntax.

## Best Possible Solution

Keep this issue open while maintainers decide whether the current password fallback is sufficient, review or close the open loopbackUser implementation candidate, and fold any longer-term internal service identity work into #69066 if that becomes the canonical direction.
