## Summary

Keep open. Current main has partial mitigation for trusted-proxy local clients, but it does not fully resolve the reported headerless internal self-connection shape. It still requires either proxy identity headers with explicit loopback trust or a configured gateway password. The provided issue context also shows an open, unmerged PR that uses GitHub closing syntax for this issue, so the issue should remain open until that implementation candidate is merged, closed, or explicitly superseded.

## Best Possible Solution

Keep this issue open while maintainers review, merge, close, or explicitly supersede BingqingLyu/openclaw#2181, and decide whether #69066 should own the longer-term internal service-identity design. If maintainers choose the current password fallback as the final resolution, close only after the open implementation candidate is resolved or marked superseded.
