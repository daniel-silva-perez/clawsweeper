## Summary

Keep this issue open. Current main still defaults Telegram DMs to pairing, still sends pairing challenges to unknown DM senders, and setup/docs/tests still model manual numeric allowFrom entry rather than token-verified owner ID capture during onboarding. The issue comment about requiring a verification token remains relevant because blindly capturing the first inbound Telegram sender would create a race.

## Best Possible Solution

Keep this issue open and implement it as Telegram-owned onboarding work: generate a short verification token, ask the intended owner to send that exact token to the bot, capture the matching numeric Telegram user ID, write dmPolicy="allowlist" plus allowFrom for that account, and update schema/runtime/docs/tests while preserving compatibility for existing explicit pairing-mode configs.

- Existing pairing-mode installations need compatibility handling so an improved fresh-setup default does not unexpectedly remove access for current users.
