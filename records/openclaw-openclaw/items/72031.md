## Summary

Keep open. Current main at 7ddd815e469e5f5a9407d67cd8dc306eff771b84 still unconditionally requires an API key in the media-understanding image and audio/video auth paths, while the Bedrock docs and chat/simple-completion path support aws-sdk auth without a static key. The linked PR #72092 is open and unmerged, so this issue should remain open until that fix lands or is superseded.

## Best Possible Solution

Keep this issue open and review/land or supersede #72092. The narrow fix belongs in media-understanding auth resolution: allow an empty apiKey only when ResolvedProviderAuth.mode is "aws-sdk", skip storing empty runtime keys, pass apiKey: "" through to Bedrock-backed calls, and add regression coverage for amazon-bedrock aws-sdk image plus audio/video media-understanding paths.
