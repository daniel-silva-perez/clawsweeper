## Summary

Keep open. Current main still routes Bedrock aws-sdk media auth through unconditional API-key requirements, and the linked closing PR #72092 is open rather than merged.

## What This Changes

The issue requests that Bedrock media-understanding paths allow AWS SDK credential-chain auth without a static apiKey, matching the chat/completion path.

## Best Possible Solution

Land or supersede #72092 with the narrow media auth fix: allow missing static key material only for aws-sdk mode, avoid storing empty runtime keys, preserve missing-key errors for every other auth mode, and make sure image plus audio/video execution paths can pass through the SDK-chain case without dropping the empty-key sentinel.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: resolve auth for amazon-bedrock to { mode: "aws-sdk" } with no apiKey, then the media paths reach requireApiKey and throw before the Bedrock provider can invoke the AWS SDK credential chain.

## Solution Assessment

Yes. Mirroring the chat path's missing-key allowance inside the media-specific auth resolution and execution path is the narrow maintainable fix; weakening requireApiKey globally would be broader than needed.
