## Summary

Keep open. Current main still routes Bedrock `aws-sdk` media auth through unconditional API-key requirements, and the linked closing PR #72092 remains open rather than merged.

## What This Changes

The issue asks media-understanding image and related audio/video auth paths to support Bedrock `aws-sdk` credentials without requiring a static API key.

## Best Possible Solution

Land or supersede #72092 with the same narrow behavior: allow empty API-key material only for `aws-sdk` media auth, avoid storing empty runtime keys, preserve missing-key errors for other modes, and keep regression coverage for image plus provider execution paths.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: `auth.mode === "aws-sdk"` with no `apiKey` reaches `requireApiKey` in the media paths and throws before the Bedrock provider can use the AWS SDK credential chain.

## Solution Assessment

Yes. Mirroring the chat path’s missing-key allowance in the media-specific auth resolvers is the narrow maintainable fix; weakening `requireApiKey` globally would be broader than needed.
