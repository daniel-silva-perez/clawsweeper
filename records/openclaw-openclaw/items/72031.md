## Summary

Keep open. Current main still rejects Bedrock `aws-sdk` media-understanding calls before the provider can use the AWS SDK credential chain, and the linked closing PR is still open rather than merged.

## What This Changes

Allow Bedrock media-understanding image and audio/video paths using `auth: "aws-sdk"` to proceed without an explicit API key, matching the chat/completion path.

## Best Possible Solution

Review, land, or supersede #72092. The desired end state is for the image runtime, MiniMax fallback, and audio/video auth path to pass an empty API key only for `aws-sdk`, avoid storing empty runtime keys, preserve missing-key errors for other auth modes, and add regression coverage for Bedrock media calls.

## Reproduction Assessment

Yes. A source-level reproduction is enough: an auth result with `mode: "aws-sdk"` and no `apiKey` reaches `requireApiKey` in the media paths and throws before the Bedrock provider runs.

## Solution Assessment

Yes. Mirroring the chat path’s missing-key allowance in the media-specific auth resolvers is the narrow maintainable fix; weakening `requireApiKey` globally would be broader than needed.
