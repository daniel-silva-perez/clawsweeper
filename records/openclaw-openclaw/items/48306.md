## Summary

Keep this issue open. Current main has a healthy latest appcast entry and the latest release has macOS assets, but the release path still does not enforce asset/appcast consistency: the public macOS workflow is validation-only, appcast generation can still publish a derived GitHub URL without reachability proof, current main lacks the proposed verifier/CODEOWNERS protection, and the related verifier PR remains open and unmerged.

## Best Possible Solution

Keep this issue open as the canonical macOS updater/release consistency bug until release-owner work lands and ships. The best fix is a hardened version of PR #66213 or an equivalent stronger guard that verifies stable releases have the macOS .zip, .dmg, and .dSYM.zip assets, verifies the appcast zip URL is reachable before appcast.xml advances, protects appcast ownership, and optionally improves the macOS updater failure message or fallback behavior.
