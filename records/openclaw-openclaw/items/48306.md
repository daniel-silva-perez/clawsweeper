## Summary

Keep this issue open. Current main and the latest release have a healthy appcast entry and reachable macOS assets for v2026.4.26, but current main still does not enforce the release/appcast consistency invariant that this issue reports. The public macOS workflow is validation-only, appcast generation still publishes a derived GitHub release URL without release-asset reachability proof, there is no verifier workflow or appcast CODEOWNERS protection on main, and the directly related verifier PR #66213 is still open and unmerged.

## Best Possible Solution

Keep this issue open as the canonical macOS updater/release consistency bug until release-owner work lands and ships. The best path is to review and either land PR #66213 or replace it with a stronger guard that verifies stable releases have the .zip, .dmg, and .dSYM.zip assets, verifies the appcast enclosure URL before appcast.xml advances, protects appcast ownership, and optionally improves the macOS updater failure/fallback behavior for missing artifacts.
