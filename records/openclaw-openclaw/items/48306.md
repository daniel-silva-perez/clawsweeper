## Summary

Keep this issue open. Current main has a fresh appcast entry and the latest release currently has macOS assets, but the underlying release-pipeline gap is still present: the public macOS workflow is validation-only, appcast generation does not verify the GitHub asset exists, and the related asset-verifier PR is still open and unmerged.

## Best Possible Solution

Keep this issue open as the canonical bug until the macOS release path enforces asset/appcast consistency. The best fix is release-owner work that either lands a hardened version of PR #66213 or an equivalent stronger guard: verify stable GitHub releases have the macOS .zip/.dmg/.dSYM.zip, ensure appcast.xml advances only after the zip exists and is reachable, protect appcast ownership, and optionally improve the app’s updater failure message or fallback behavior. After that merges and ships, close this issue as implemented with the fixing commit/release evidence.
