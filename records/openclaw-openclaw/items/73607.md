## Summary

Keep this issue open. The reported fields are still a documented and schema-supported config contract, and current main does not show a targeted fix or regression coverage proving that gateway/runtime write-back preserves `agents.defaults.params` and `agents.defaults.models[...].params` across the reported reload/persist path.

## Best Possible Solution

Add a focused regression around the gateway/config write-back path: start with source config containing both `agents.defaults.params` and `agents.defaults.models["openai/gpt-5.4"].params`, simulate the runtime snapshot plus an `agents.list` or bindings write via `replaceConfigFile`, and assert the persisted source config retains both params objects. If that fails, fix the lossy projection/mutator path and keep the existing extra-params runtime behavior unchanged.
