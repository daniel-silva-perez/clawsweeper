## Summary

Keep this PR open. Current main still ignores `auth.order` on the implicit GitHub Copilot discovery path, and the PR is not safe to close as implemented because its patch targets the old monolithic provider file while current main routes Copilot discovery through the GitHub Copilot plugin catalog.

## Best Possible Solution

Keep the PR open and ask for a rebase/rework against current main. The fix should live in `extensions/github-copilot/auth.ts` and the plugin catalog path, replace first-profile selection with order-aware Copilot token resolution, preserve environment-token precedence and explicit provider config behavior, preserve plugin disablement, and add focused regression coverage for `auth.order`, unresolved ordered tokenRefs, explicit empty order/disablement, and stale-order handling.
