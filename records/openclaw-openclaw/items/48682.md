## Summary

Keep this PR open. Current main still does not implement the intended fix: implicit GitHub Copilot discovery still flows through the plugin-owned resolver that selects the first stored Copilot profile rather than the configured auth order. The PR remains a useful implementation candidate for #46031, but it is stale against the current plugin-owned discovery path and needs rework before merge.

## Best Possible Solution

Keep this PR open as an implementation candidate for #46031, but request a rebase/rework against current main. The fix should live in `extensions/github-copilot/auth.ts` and the GitHub Copilot catalog path, use the existing order-aware provider-auth SDK seam, preserve env-token precedence plus explicit provider config and plugin discovery disablement, and add focused regression coverage for ordered profiles, unresolved ordered tokenRefs, stale order handling, and discovery disablement.
