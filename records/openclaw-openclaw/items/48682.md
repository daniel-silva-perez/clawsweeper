## Summary

Keep this PR open. Current main still does not implement the intended behavior: implicit GitHub Copilot discovery is plugin-owned, and the current plugin resolver still selects the first stored Copilot profile rather than honoring `auth.order`. The PR remains a useful implementation candidate for #46031, but its patch is stale against current main and needs rework in the GitHub Copilot plugin path.

## Best Possible Solution

Keep this PR open as an implementation candidate, but ask for a rebase/rework against current main. The fix should live in `extensions/github-copilot/auth.ts` and the plugin catalog path, use the existing order-aware provider-auth SDK seam, preserve env-token precedence plus explicit provider config and plugin discovery disablement, and add focused regression coverage for ordered profiles, unresolved ordered tokenRefs, stale order handling, and discovery disablement.
