## Summary

Keep open. Current main has partial effective tool visibility through `tools.effective` and the Control UI, but it still does not implement strict declared-vs-effective inheritance or a general clipping/removal trace with per-layer reasons.

## Best Possible Solution

Keep this open as the umbrella for tool capability determinism. The best path is a maintainer-approved design for a generic policy-resolution trace that records declared inputs, effective tools, removed tools, and removal layer/reason; exposes it through `tools.effective`, `/tools` or debug/status surfaces, Control UI, and docs; and only then adds strict inheritance with explicit fail-closed behavior for runtime/security layers that cannot honor parity.
