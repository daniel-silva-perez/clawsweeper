## Summary

Keep this PR open. Current main still drains formatted system events only from the active session key, so the linked #42919 behavior is not implemented on main; however, the PR branch still needs rework because its fallback drains the config-default main key instead of the active agent's main key, and the intended scope for non-DM sessions remains a maintainer decision.

## What This Changes

This PR changes formatted system-event draining to also drain the config-resolved main session queue for non-main sessions and adds a `system-events` regression test for per-channel-peer DMs.

## Best Possible Solution

Keep this PR open for focused maintainer rework. The receiver-side merge belongs in `src/auto-reply/reply/session-system-events.ts`, but the fallback key should be derived from `resolveAgentIdFromSessionKey(params.sessionKey)` plus `resolveAgentMainSessionKey(...)`, with tests for per-channel-peer delivery, multi-agent isolation, and whatever non-main session scope maintainers decide is intentional.
