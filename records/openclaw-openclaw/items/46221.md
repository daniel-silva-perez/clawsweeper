## Summary

Keep open. The underlying delivery gap is still present on current main, but this PR is not merge-ready because the fallback key is resolved from the config default agent and the fallback drains every non-main session shape, creating correctness and security concerns.

## What This Changes

The branch changes formatted system-event draining to merge the resolved main-session queue into non-main session drains, sorts merged events by timestamp, and adds a per-channel-peer Telegram DM regression test.

## Best Possible Solution

Port the receiver-side fix onto current main using the active session's agent id for the fallback main key, then either restrict fallback delivery to approved direct-message scopes or document and test the broader non-main-session behavior explicitly.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Scope fallback drain to the active agent:** `src/auto-reply/reply/session-system-events.ts:82`
  - body: `resolveMainSessionKey(params.cfg)` picks the configured default/first agent, not the agent in the active session key. A turn in `agent:ops:telegram:direct:...` can drain `agent:main:main` while leaving `agent:ops:main` queued, crossing agent boundaries or missing the intended event. Derive the fallback from `params.sessionKey`'s agent id.
  - confidence: 0.91
- **[P2] Constrain the main-key merge scope:** `src/auto-reply/reply/session-system-events.ts:83-87`
  - body: The `mainSessionKey !== params.sessionKey` guard also fires for group, channel, and thread sessions. That lets an unrelated non-DM turn consume and clear main-session wake context; restrict the fallback to approved direct-message scopes or add explicit maintainer-approved semantics and tests for the broader behavior.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The diff can route untrusted or agent-specific system-event context into the wrong agent/session until the fallback key and scope are constrained.

Concerns:

- **[medium] Cross-agent system-event drain:** `src/auto-reply/reply/session-system-events.ts:82`
  - body: Using the config-default main key means a non-default agent turn can drain another agent's main-session queue. System events can include untrusted hook text, so this crosses agent isolation boundaries and can inject context into the wrong prompt.
  - confidence: 0.86
- **[low] Over-broad non-main session drain:** `src/auto-reply/reply/session-system-events.ts:83`
  - body: The fallback applies to every non-main session, including groups, channels, and threads. That can move main-session wake context into unrelated conversations and clear it before the intended session sees it.
  - confidence: 0.74
