## Summary

Keep open. Current main still drains formatted system events only from the active session key while the reported enqueue paths still write to a main-session key, so the PR targets a real gap; however, the proposed fallback drains the config-default agent's main key and applies to every non-main session shape, which leaves correctness and security concerns before merge.

## What This Changes

This PR changes formatted system-event draining to merge the configured main-session queue into non-main session drains, sorts merged events by timestamp, and adds a per-channel-peer Telegram DM regression test.

## Best Possible Solution

Port the receiver-side fix onto current main using the active session's agent id for the fallback main key, then either limit fallback delivery to approved direct-message scopes or explicitly document and test the broader non-main-session semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Scope fallback drain to the active agent:** `src/auto-reply/reply/session-system-events.ts:82`
  - body: `resolveMainSessionKey(params.cfg)` selects the configured default or first agent, not the agent in the active session key. A turn in `agent:ops:telegram:direct:...` can drain `agent:main:main` while leaving `agent:ops:main` queued, crossing agent boundaries or missing the intended event. Derive the fallback from `params.sessionKey`'s agent id before merging queues.
  - confidence: 0.91
- **[P2] Limit main-key fallback to approved session scopes:** `src/auto-reply/reply/session-system-events.ts:83-87`
  - body: The `mainSessionKey !== params.sessionKey` guard also matches group, channel, thread, and other custom non-main sessions. That can consume and clear main-session wake context in an unrelated conversation. Restrict the fallback to the intended direct-message isolation cases, or add explicit maintainer-approved semantics and tests for the broader behavior.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The diff can route untrusted or agent-specific system-event context into the wrong agent/session until the fallback key and scope are constrained.

Concerns:

- **[medium] Cross-agent system-event drain:** `src/auto-reply/reply/session-system-events.ts:82`
  - body: Using the config-default main key means a non-default agent turn can drain another agent's main-session queue. System events can include untrusted hook text, so this crosses agent isolation boundaries and can inject context into the wrong prompt.
  - confidence: 0.86
- **[low] Over-broad non-main session drain:** `src/auto-reply/reply/session-system-events.ts:83`
  - body: The fallback applies to every non-main session, including groups, channels, and threads. That can move main-session wake context into unrelated conversations and clear it before the intended session sees it.
  - confidence: 0.74
