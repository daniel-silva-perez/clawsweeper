## Summary

Keep open. Current main still drains only the active session queue while system-event, wake-hook, and main-session cron paths enqueue under canonical main-session keys. The PR targets a real bug, but the provided diff has unresolved maintainer questions around active-agent isolation and whether all non-main sessions should consume main-session events.

## What This Changes

The PR changes formatted system-event draining to also drain a config-resolved main-session queue for non-main sessions and adds a regression test for per-channel-peer DM delivery.

## Best Possible Solution

Port the receiver-side fix onto current main, derive the fallback main key from the active session's agent id via `resolveAgentIdFromSessionKey(params.sessionKey)` plus `resolveAgentMainSessionKey(...)`, and add regression coverage for per-channel-peer delivery, multi-agent isolation, and the maintainer-approved non-main session scopes.
