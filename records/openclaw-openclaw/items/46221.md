## Summary

Keep open. Current main still drains formatted system events only from the active session key, while the CLI system event, wake hook, and main-session cron paths enqueue to the agent main session key. The PR addresses a real bug, but it needs maintainer review for fallback scope and active-agent isolation before merge.

## What This Changes

The PR changes formatted system-event draining to merge a main-session event queue into non-main session drains and adds a per-channel-peer DM regression test.

## Best Possible Solution

Port the receiver-side fix onto current main, derive the fallback main key from the active session's agent id, and add regression coverage for per-channel-peer delivery, multi-agent isolation, and whichever non-main session scopes maintainers explicitly approve.
