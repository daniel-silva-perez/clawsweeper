## Summary

Keep this PR open. Current `main` still does not implement the proposed per-message `message.processing.*` or skill lifecycle `skill.execution.*` diagnostics/OTel surface, and this active PR remains an implementation candidate that needs maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to decide whether OpenClaw wants first-class per-message and skill lifecycle telemetry in the core diagnostics contract, then either land a focused implementation with contract/exporter tests, public docs, and explicit privacy/cardinality guidance, or close it after that product/API decision.
