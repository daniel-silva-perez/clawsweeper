Keep the PR open while maintainers settle the event naming/phase model and OTLP privacy/cardinality policy, then land a narrowed implementation with contract tests, runner emission tests, diagnostics-otel exporter tests, stability-recorder coverage, and public OTel docs aligned to the final contract.

Full review comments:

- **[P1] Fix the skill attrs type before typechecking:** `extensions/diagnostics-otel/src/service.ts:2226-2228`
  - body: `skillExecutionBaseAttrs` is typed as possibly returning boolean values, but this `attrs` object narrows the spread to `Record<string, string | number>`. TypeScript can reject the assignment; the same pattern appears in `recordSkillExecutionError`, so the branch should narrow the helper return type or widen both attrs objects before merge.
  - confidence: 0.87
- **[P1] Keep session identifiers out of exported run attrs:** `extensions/diagnostics-otel/src/service.ts:1166-1170`
  - body: This adds raw `sessionKey` and `sessionId` to the shared run attribute helper used by exported spans. The current public OTel contract says spans never include session keys, so these identifiers need to be removed, hashed, or handled by a reviewed privacy contract before being sent to external collectors.
  - confidence: 0.9
- **[P1] Remove unbounded IDs from metric attributes:** `extensions/diagnostics-otel/src/service.ts:1407-1412`
  - body: `messageProcessingBaseAttrs` is reused for the new counter and duration histogram, so every run, message, chat, and session value becomes an OTel metric label. That creates unbounded time-series cardinality in Prometheus-style backends; keep those IDs on spans/log correlation only or replace them with bounded metric attributes.
  - confidence: 0.86
- **[P2] Wire the new OTel guide into docs metadata:** `docs/diagnostics/otel-skill-observability.md:1`
  - body: The new diagnostics page starts directly with an H1 and the PR does not update the Mintlify sidebar, so the user-facing guide lacks the normal docs front matter and navigation entry. Add front matter/read_when metadata and place it in `docs/docs.json` or fold the content into the existing OpenTelemetry page.
  - confidence: 0.75

## Security Review

Status: needs_attention

Summary: Security review needs attention because the PR exports raw session identifiers and unbounded per-message identifiers through external OTLP telemetry paths.

Concerns:

- **[medium] Raw session identifiers exported to OTLP:** `extensions/diagnostics-otel/src/service.ts:1166`
  - body: The PR adds `openclaw.session_key` and `openclaw.session_id` to a shared span attribute helper even though current public OTel docs say spans never include session keys. External collectors may retain those identifiers outside the OpenClaw trust boundary.
