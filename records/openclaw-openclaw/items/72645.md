Keep this PR open. Current main still does not implement the proposed per-message `message.processing.*` and skill lifecycle `skill.execution.*` diagnostics/OTel surface, and the PR is an active implementation candidate that needs maintainer API, privacy, and scope review rather than conservative cleanup.

## Best Possible Solution
