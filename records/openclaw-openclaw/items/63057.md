## Summary

Keep this PR open. Current main still has the targeted Anthropic legacy thinking-budget fallback, and the resolver can still pass an explicit zero budget into payload construction; the PR is a useful narrow fix direction, but it needs regression tests and should probably guard every Anthropic-invalid budget below the documented 1,024-token minimum, not only zero.

## What This Changes

The PR changes `src/agents/anthropic-transport-stream.ts` so legacy Anthropic thinking budget payloads use a nullish fallback and skip emitting `thinking` when the resolved budget is zero.

## Best Possible Solution

Land a narrow Anthropic transport fix that only emits legacy `thinking: { type: "enabled", budget_tokens }` when the resolved budget is valid for Anthropic, preserves the undefined-budget `1024` fallback, omits thinking when no valid budget can be formed, and covers those paths with targeted transport tests.
