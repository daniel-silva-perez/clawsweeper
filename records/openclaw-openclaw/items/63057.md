## Summary

Keep this PR open. Current main still has the exact Anthropic legacy `budget_tokens: options.thinkingBudgetTokens || 1024` fallback targeted by the PR, and the resolver can still produce an explicit zero thinking budget before payload construction. The PR remains a plausible narrow implementation candidate, but it still needs focused regression coverage and maintainer judgment on whether the guard should cover all invalid legacy budgets, not only zero.

## Best Possible Solution

Keep this PR open as an implementation candidate. Add targeted tests in `src/agents/anthropic-transport-stream.test.ts` for the explicit-zero path and the undefined-budget default, then decide whether the legacy Anthropic branch should omit thinking for any invalid budget rather than only for zero.
