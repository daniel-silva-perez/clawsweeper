## Summary

Keep this PR open. Current main still has the old Anthropic legacy thinking fallback, and the PR is aimed at a real bug, but the proposed `budgetTokens > 0` guard is not sufficient for Anthropic's documented minimum-budget contract and should not merge as-is.

## What This Changes

The PR changes `src/agents/anthropic-transport-stream.ts` so the legacy Anthropic thinking payload uses a nullish `thinkingBudgetTokens` fallback and omits `thinking` when the resolved budget is zero.

## Best Possible Solution

Land a narrow Anthropic transport fix that emits legacy `thinking: { type: "enabled", budget_tokens }` only when the resolved budget satisfies Anthropic's validity rules, preserves adaptive-thinking behavior, and adds colocated tests for zero, default, valid, and sub-minimum legacy budget paths.
