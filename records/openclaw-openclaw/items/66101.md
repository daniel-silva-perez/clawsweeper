## Summary

Keep this PR open. Current main still lacks the proposed main-specific host-level OpenClaw self-inspection exception, and this PR remains a focused but security-sensitive implementation candidate with unresolved bypasses in the proposed boundary.

## Best Possible Solution

Keep this PR open for maintainer review or author revision. The best implementation path is to decide whether main should receive a narrow unsandboxed full-trust self-inspection exception, then implement it with parser/segment-based command detection, cross-platform OpenClaw path handling, strict read-only argv allowlists, rejection of all mutating flags and mixed shell/pipeline forms, and focused regression tests for the identified bypasses.
