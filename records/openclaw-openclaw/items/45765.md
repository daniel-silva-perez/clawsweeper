## Summary

Current main still has the reported OPENCLAW_HOME nesting behavior. `resolveStateDir()` builds the default state directory by joining the effective home with `.openclaw`, and `resolveConfigDir()` separately does the same; with `OPENCLAW_HOME` already pointing at `~/.openclaw`, both paths can resolve under `~/.openclaw/.openclaw`. The issue is valid and already has open related PR #45784, so it should stay open until that PR lands or is replaced by a complete fix.

## Best Possible Solution

Keep this issue open and review/land #45784 only if it fully fixes both `resolveStateDir()` and `resolveConfigDir()` without changing the documented parent-home contract. If #45784 is incomplete, replace it with a focused PR that adds regression tests for `OPENCLAW_HOME=~/.openclaw`, tilde expansion, legacy state dirname behavior, `OPENCLAW_STATE_DIR` precedence, and the negative parent-home case.
