## Summary

Current main still has the reported path behavior: both state and config path resolution append `.openclaw` underneath `OPENCLAW_HOME`, and tests currently codify that convention. The linked open PR #45784 is an implementation candidate, while the earlier related PRs were closed unmerged, so this issue should stay open until a fix lands or maintainers explicitly decide that the documented `OPENCLAW_HOME` semantics should remain parent-home-only.

## Best Possible Solution

Keep this issue open. Maintainers should either review and land a targeted fix such as #45784, or author a fresh fix that updates `src/config/paths.ts` and `src/utils.ts` together, adds regression tests for `OPENCLAW_HOME=~/.openclaw`, preserves `OPENCLAW_STATE_DIR` and `OPENCLAW_CONFIG_PATH` precedence, and explicitly decides whether non-state-dir values such as `/srv/app` continue to resolve to `/srv/app/.openclaw`. If maintainers decide current parent-home semantics are intentional, update the docs and close with that product decision rather than as implemented.
