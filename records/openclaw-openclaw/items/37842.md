## Summary

Keep open. Current main still has source-only inter-session provenance and pairwise A2A ping-pong limiting; it does not carry a visited-agent chain that would block triangular or polygonal `sessions_send` cascades. The linked implementation PRs are closed unmerged, and #51832 is a different gateway error-cascade mechanism rather than a canonical replacement.

## What This Changes

Add graph-aware `sessions_send` loop detection so multi-agent chains such as A->B->C->A are detected and blocked instead of bypassing pairwise `maxPingPongTurns`.

## Best Possible Solution

Implement a narrow core sessions_send guard: add an optional normalized visited-agent chain to inter-session provenance and the gateway schema, thread it through tool construction and nested A2A calls, block or warn before delivering to a target already in the chain, and add regression tests for self-send, A->B->C->A, legacy keys, schema acceptance, and existing pairwise `maxPingPongTurns`.

## Reproduction Assessment

Yes. The issue provides a concrete 3-agent A->B->C->A reproduction path, and current-main source inspection shows there is no chain state that could stop that path; this read-only review did not run a live multi-agent deployment.

## Solution Assessment

Yes, the proposed provenance-chain direction is the narrowest maintainable fix, but it must include the gateway schema update and canonical agent-ID normalization that the earlier unmerged PR missed.

## Repair Work Prompt

Implement a narrow OpenClaw core fix for graph-aware `sessions_send` cycle detection for issue #37842. Current main tracks only `sourceSessionKey`/`sourceChannel`/`sourceTool` plus pairwise `maxPingPongTurns`, so triangular A->B->C->A chains bypass the guard. Add an optional normalized visited-agent chain to `InputProvenance` (name can be `chainAgentIds` or a clearly equivalent additive field), update the gateway `InputProvenanceSchema` because it currently has `additionalProperties: false`, and thread incoming provenance from `runEmbeddedAttempt` params through `createOpenClawCodingTools` -> `createOpenClawTools` -> `createSessionsSendTool`. When `sessions_send` resolves the target, normalize/sanitize requester and target agent IDs with existing routing helpers, append the requester to the incoming chain, and reject or warn before delivery if the target already appears in the chain. Preserve existing direct A<->B `maxPingPongTurns`, A2A announce behavior, and legacy/non-agent-prefixed key behavior unless a regression test proves a required narrow adjustment. Add focused tests for self-send, A->B->C->A, schema acceptance, propagation through tool construction, and no false positives for legacy keys. Use PR #37945 as source context but address its known gaps: missing gateway schema update and raw agent-ID comparison. Do not implement broader A2A duplicate-message, Slack error-cascade, human queue-priority, or `maxPingPongTurns` ceiling changes in this PR. Add a user-facing changelog fix entry with `Thanks @sprfrkr` and credit source PR context where appropriate.
