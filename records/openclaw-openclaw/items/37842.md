## Summary

Keep this issue open. Current main still lacks graph-aware sessions_send cycle detection: inter-session provenance carries source-only metadata, the gateway schema has no trace/chain field, sessions_send creates fresh provenance for each hop, and the A2A guard still bounds only a requester/target pair. The related PR #37945 proposed the right shape with chainAgentIds but was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep this issue open for a core sessions_send fix. Add an explicit trace or chain field to InputProvenance and the gateway schema, thread it through createOpenClawTools/createSessionsSendTool, runAgentStep, and A2A follow-up runs, reject or warn when the target agent/session already appears in the chain, and add regression coverage for A->B->C->A while preserving maxPingPongTurns for direct A<->B exchanges.
