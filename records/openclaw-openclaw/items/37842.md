## Summary

Keep this issue open. Current main still lacks graph-aware sessions_send cycle detection: the related PR #37945 proposed chainAgentIds but closed unmerged, InputProvenance remains source-only, the gateway schema rejects unknown provenance fields, and sessions_send/A2A still create fresh pairwise provenance rather than carrying a visited-agent chain.

## Best Possible Solution

Keep this issue open for a core sessions_send fix. Add an explicit chain or trace field to InputProvenance and the gateway schema, thread it through createOpenClawTools/createSessionsSendTool, runAgentStep, and A2A follow-up runs, reject or warn when the target agent/session already appears in the chain, and add regression coverage for A->B->C->A while preserving existing maxPingPongTurns behavior for direct A<->B exchanges.
