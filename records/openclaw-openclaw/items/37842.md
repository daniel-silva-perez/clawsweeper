## Summary

Keep open. Current main still implements only source-only inter-session provenance and pairwise requester/target A2A ping-pong limiting; it does not carry a trace/visited-agent chain or block A->B->C->A sessions_send cycles. The linked fix PR #37945 proposed the right chainAgentIds shape, but it was closed unmerged, and the related #51832 is a different gateway/channel error-cascade mechanism rather than a canonical replacement.

## Best Possible Solution

Keep this issue open for a core sessions_send fix. Add an additive chain/trace field to InputProvenance and the gateway schema, thread it through createSessionsSendTool, runAgentStep, and A2A follow-up runs, reject or warn when the target agent/session already appears in the chain, and add regression tests for triangular/polygonal cascades while preserving existing pairwise maxPingPongTurns semantics.
