## Summary

Keep this PR open for maintainer action. The reported Android gateway TLS failure still matches current `main`, but the PR branch is stale and would regress the newer non-trust-all TLS probe that current `main` added for an Android CodeQL security finding.

## What This Changes

The PR updates Android gateway TLS code to use `X509ExtendedTrustManager` overloads and removes the `openclaw.local`/`ts.net` domain-config entries from the Android network security config.

## Best Possible Solution

Preserve the current-main non-trust-all fingerprint probe, then apply the extended trust-manager overload fix to both the main gateway trust manager and the probe path. Remove the domain-config entries only if Android maintainers confirm the base-config behavior is the intended cleartext policy.
