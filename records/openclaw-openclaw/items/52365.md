## Summary

Keep this PR open. Current main has adjacent cron timeout work, including deferred execution timeout arming, but it still does not implement this PR's core proactive behavior: passing an outer cron deadline into isolated fallback execution and stopping a new fallback candidate before it starts when the remaining cron budget is too low.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a scoped version that threads a cron deadline into isolated execution and adds a tested pre-attempt fallback budget gate, or supersede it only with a newer canonical PR that covers the same proactive cron-budget behavior. Before landing, resolve or explicitly split the timeout-policy DoS concerns and the existing review comments about stop-error propagation and guard sizing.
