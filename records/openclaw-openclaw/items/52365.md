## Summary

Keep open. Current main still lacks the PR's proactive cron fallback budget guard, so the branch is addressing a real remaining runtime behavior, but the patch should not merge as-is because the new guard is based on the outer cron job timeout rather than the actual per-attempt agent timeout.

## What This Changes

The PR adds an optional pre-attempt stop hook to model fallback, threads cron deadline metadata into isolated cron execution, skips fallback attempts when little cron budget remains, and adds deadline/budget regression tests.

## Best Possible Solution

Land a rebased, narrow fix that keeps the generic pre-attempt stop seam, computes the skip threshold from the real prepared per-attempt execution budget, preserves current deferred cron timeout cleanup, and coordinates the final abort/fallback contract with #42482 and #62682.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Base the fallback budget on the attempt timeout:** `src/cron/isolated-agent/run.ts:986`
