## Summary

Keep open. Current main still does not implement the proposed pre-attempt cron fallback budget guard, and the PR branch is useful but review-blocked because the guard is computed from cron’s safety timeout instead of the actual agent attempt timeout and still needs reconciliation with current deferred timeout semantics.

## What This Changes

The PR adds a model-fallback pre-attempt stop hook, threads cron deadline metadata into isolated cron execution, skips low-budget fallback candidates, and adds cron/model fallback regression tests.

## Best Possible Solution

Land a rebased, narrow fix that keeps the generic pre-attempt stop seam, derives the fallback skip threshold from the prepared per-attempt execution budget, starts any propagated deadline when isolated execution actually starts, and preserves explicit stop errors when no prior candidate produced a real failure.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Base the fallback budget on the attempt timeout:** `src/cron/isolated-agent/run.ts:988-992`
  - body: `resolveCronJobTimeoutMs(params.job)` falls back to the 60-minute cron safety ceiling when `payload.timeoutSeconds` is unset, but the actual model attempt uses `prepared.context.timeoutMs` from `resolveAgentTimeoutMs`. Jobs configured with a 5-10s `agents.defaults.timeoutSeconds` can still skip useful fallbacks whenever under 30s remain, even though another attempt would fit the real budget.
  - confidence: 0.88
- **[P2] Start the deadline when isolated execution starts:** `src/cron/service/timer.ts:89-93`
  - body: Current main deliberately starts isolated cron timeouts from `onExecutionStarted`, so queue wait does not consume execution budget. Computing `deadlineAtMs` before `executeJobCore` means lane wait can drain the propagated fallback budget and make the new guard skip fallbacks before the job has had real execution time.
  - confidence: 0.84
- **[P2] Preserve stop errors without prior failures:** `src/agents/model-fallback.ts:915-916`
  - body: When `beforeAttempt` stops after cooldown/auth skips but before any candidate has failed, `lastError` is still undefined. This branch breaks and later throws a synthesized fallback summary, dropping the hook's explicit stop message that the cron budget path is supposed to surface.
  - confidence: 0.78
