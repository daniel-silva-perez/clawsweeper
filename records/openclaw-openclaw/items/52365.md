## Summary

Keep open. Current main still lacks the proactive pre-attempt fallback budget gate and cron deadline propagation this PR is trying to add, but the proposed patch should not merge as-is because its budget threshold can suppress valid fallbacks for jobs using agent-default timeouts.

## What This Changes

The PR adds a pre-attempt stop hook to model fallback, threads cron deadline information into isolated cron execution, and adds tests for skipping fallback attempts when little cron budget remains.

## Best Possible Solution

Land a rebased, narrow fix that keeps current deferred isolated-timeout cleanup semantics, adds only the needed generic pre-attempt stop seam, computes the skip threshold from the correct execution budget, and coordinates the terminal-abort contract with #42482 and #62682.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Base the fallback budget on the attempt timeout:** `src/cron/isolated-agent/run.ts:986`

With a 5-10s agent default, this guard still requires 30s of remaining outer budget and skips fallbacks that could have completed within the real attempt timeout.
