## Summary

Keep this PR open. Current main has the related deferred cron execution timeout, but it does not implement this PR’s core behavior: propagating an outer cron deadline into isolated cron fallback execution or stopping new fallback candidates before they start when the remaining cron budget is too low. The related open PRs are complementary/narrower, not a clear supersession of this proactive budget gate.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to either land a scoped version that threads the cron deadline and adds a tested pre-attempt fallback budget gate, after resolving the noted review/security concerns, or deliberately supersede it with a newer canonical PR only if that PR covers this same proactive cron-budget behavior.
