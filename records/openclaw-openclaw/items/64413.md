## Summary

Keep this issue open. Current main still exposes cron and task/message delivery outcomes as coarse status enums plus raw error text, so the requested normalized delivery failure reasons are not implemented. The adjacent open issues #49190 and #74095 cover related status/taxonomy work, but neither fully supersedes this delivery-specific cross-surface request.

## What This Changes

Request to add structured, normalized delivery failure reason codes across cron runs, messaging/background delivery, UI status surfaces, and JSON output.

## Best Possible Solution

Define a shared additive delivery outcome taxonomy and expose it consistently across cron run state/history, task delivery records, Gateway protocol schemas, plugin hook payloads, Control UI filters/cards, CLI JSON/text output, and docs while preserving existing status, deliveryStatus, delivered, deliveryError, and task deliveryStatus fields for compatibility.
