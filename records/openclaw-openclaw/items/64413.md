## Summary

Keep this issue open. Current main still does not expose a shared normalized delivery failure reason model across cron and messaging/background task delivery; the observable surfaces remain coarse status enums plus raw error text. The prior ClawSweeper keep-open comment remains consistent with current main at 17811480dae20f65b526454d75e971ae8424ec10.

## Best Possible Solution

Keep this issue open. The best path is an additive shared delivery outcome model across cron job state, cron run logs, background task records, Gateway protocol schemas, Control UI filters/cards, CLI JSON/text output, and docs. Preserve existing compatibility fields, add stable delivery reason codes such as not_configured, channel_unavailable, skipped, attempted_failed, and rate_limited, and cover config/routing failures, outbound adapter failures, provider/rate-limit failures, best-effort skips, fallback delivery, and UI/API filtering in focused tests.
