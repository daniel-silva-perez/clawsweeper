## Summary

Keep open for maintainer review. Current main and the latest release still eager-warm the canonical TUI command, while the branch is a narrow, CI-backed implementation; the only real product boundary left is whether the `chat` and `terminal` aliases should share the same skip behavior.

## What This Changes

The PR adds `tui` to the eager context-window warmup skip list and adds a regression assertion for `shouldEagerWarmContextWindowCache(["node", "openclaw", "tui"])`.

## Best Possible Solution

Land the narrow skip-list and regression-test change if maintainers want canonical `openclaw tui` to avoid eager model/plugin warmup, then either document that alias behavior is intentionally different or follow up with explicit alias coverage.

## Reproduction Assessment

Yes for the gate-level behavior: current main registers `openclaw tui` as a primary command, omits it from the skip set, and runs warmup when the gate is true. A live terminal latency reproduction is not included in this read-only review.

## Solution Assessment
