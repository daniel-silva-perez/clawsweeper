## Summary

This should stay open: current `main` and the latest release still eager-warm for canonical `openclaw tui`, while the branch is a narrow covered fix with no concrete correctness or security finding. The remaining maintainer question is whether the `chat` and `terminal` aliases should receive the same behavior.

## What This Changes

The PR adds `tui` to the eager context-window warmup skip list and adds a regression assertion that `shouldEagerWarmContextWindowCache(["node", "openclaw", "tui"])` returns false.

## Best Possible Solution

Land the narrow skip-list and regression-test change if maintainers want canonical `openclaw tui` startup to avoid eager context/model metadata work, then handle any alias behavior as an explicit follow-up if desired.

## Reproduction Assessment

Yes for the gate-level behavior: current main parses `openclaw tui` as a real primary CLI command and does not include `tui` in the skip set, so import-time eager warmup remains enabled. The user-visible PTY lag itself was not reproduced in this read-only review.

## Solution Assessment

Yes for the canonical command: adding `tui` to the existing skip list is the narrowest maintainable fix and preserves on-demand context lookup. It is unclear whether the same PR should also cover `chat` and `terminal`, which is safer as an explicit maintainer decision.
