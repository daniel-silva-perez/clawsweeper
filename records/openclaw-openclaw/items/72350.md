## Summary

Keep this PR open: current main and `v2026.4.27` still omit `tui` from the eager context-window warmup skip list, while the PR directly adds that narrow skip and a regression assertion. The related #75104 was closed unmerged, so it does not supersede this PR.

## What This Changes

The PR adds `tui` to the eager context-window warmup skip list and adds a regression assertion for `shouldEagerWarmContextWindowCache(["node", "openclaw", "tui"])`.

## Best Possible Solution

Keep the branch under maintainer review and land the narrow skip-list/test change if the intended boundary is canonical `openclaw tui`; decide separately whether the `chat` and `terminal` aliases should receive the same startup behavior.

## Reproduction Assessment

## Solution Assessment

Yes for the canonical command: adding `tui` to the existing skip list is the narrowest maintainable fix and the added assertion covers the regression. It is unclear whether the same patch should also include `chat` and `terminal`, which is a separate maintainer/product call.
