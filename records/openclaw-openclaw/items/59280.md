## Summary

Keep this PR open. Current main still only exposes `openclaw backup create` and `openclaw backup verify`; there is no first-party `openclaw backup restore` command, implementation file, or docs entry. The open related feature request #41111 tracks the same missing core backup-lifecycle capability, and the author's linked OpenClaw Snapshot skill is a useful workaround, not evidence that main already implements this PR.

## Best Possible Solution

Keep this PR open as a candidate implementation for #41111. The best path is maintainer review of the first-party restore product decision, then addressing the remaining archive safety review comments, adding user-facing docs/changelog coverage, and validating with focused restore tests plus the normal changed gate. Close only after a core restore implementation lands or maintainers explicitly decide backup restore should remain external to core.
