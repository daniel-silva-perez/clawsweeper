## Summary

Keep open. Current main still supports `compaction` and `contextPruning` only under `agents.defaults`; `agents.list[]` remains a strict per-agent schema without those keys, the effective agent resolver does not merge them, and the Pi compaction/pruning runtime reads defaults-only settings. The related per-agent override PR is useful prior art, but it is not shipped on current main.

## Best Possible Solution

Keep this issue open. The best fix is to add typed and zod-validated `agents.list[].compaction` and `agents.list[].contextPruning`, merge them over `agents.defaults` in the effective agent/runtime config, thread the effective values into Pi compaction and cache-ttl pruning paths, and add config/runtime tests plus docs for the per-agent precedence.
