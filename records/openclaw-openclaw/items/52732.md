## Summary

Current main still does not support per-agent `compaction` or `contextPruning` in `agents.list[]`. The settings are accepted only under `agents.defaults`, `AgentEntrySchema` is still strict without those keys, `resolveAgentConfig()` does not merge them, and the Pi compaction/pruning runtime reads defaults-only config. The related PR #52751 remains useful prior art but is not shipped on main.

## Best Possible Solution

Keep this issue open. The best fix is to add typed and strict-schema `agents.list[].compaction` and `agents.list[].contextPruning`, merge each agent's values over `agents.defaults`, thread the effective values through Pi compaction settings, safeguard/runtime compaction, cache-ttl pruning, and any memory-flush paths, then add config/resolver/runtime regression tests plus docs for precedence. Reviewing, revising, or superseding PR #52751 is the likely path.
