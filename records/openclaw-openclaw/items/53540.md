## Summary

Keep open. Current main still does not prove a complete fix for the reported OpenRouter to Anthropic large tool-parameter path: the default embedded-runner idle watchdog is still chunk-based, OpenRouter still resolves through the OpenAI-compatible transport, and native Anthropic fine-grained tool streaming is only present on the direct Anthropic transport path.

## What This Changes

The issue asks OpenClaw to avoid failing embedded-runner LLM calls with `Network connection lost` when a model takes longer than the current timeout window to generate large tool-call parameters.

## Best Possible Solution

Keep this as the canonical tracker for large OpenRouter/Anthropic tool-parameter generation failures. The durable path should add a synthetic OpenAI-compatible regression that withholds tool-call argument chunks, then implement the chosen provider/runtime policy: supported provider-side tool-argument streaming, a safe timeout policy for silent tool-argument generation, or a tool-loop strategy that avoids requiring one huge `Write` payload.
