## Summary

Keep open. Current main still leaves the reported default OpenRouter-to-Anthropic large tool-parameter path unresolved: OpenRouter resolves through the OpenAI-compatible transport, native Anthropic fine-grained tool streaming is only on the direct Anthropic path, and the embedded runner still has a chunk-based idle watchdog that can fire while a transport withholds tool-call argument chunks.

## What This Changes

The issue asks OpenClaw to handle LLM calls that spend longer than the underlying timeout generating large tool-call parameters, especially OpenRouter-routed Anthropic tool calls such as a large `Write` payload.

## Best Possible Solution

Keep this as the canonical tracker and choose the provider/runtime policy before implementation: add a synthetic OpenAI-compatible regression that withholds tool-call argument chunks, then either enable supported provider-side tool-argument streaming, adjust the safe idle/request timeout policy, or add a durable large-tool-payload strategy that avoids one huge generated argument.
