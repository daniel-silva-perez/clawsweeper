## Summary

Keep open. Current main still does not prove a complete fix for the updated OpenRouter/Anthropic large tool-parameter failure: OpenRouter resolves through the OpenAI-compatible transport, native Anthropic fine-grained tool streaming is gated to direct Anthropic routes, and the embedded runner still has a chunk-idle watchdog that can abort silent streams by default.

## What This Changes

The issue requests that embedded agent model calls survive or clearly handle large tool-call parameter generation when a provider stays silent long enough to exceed the request or idle timeout.

## Best Possible Solution

Keep this as the canonical tracker for OpenRouter/Anthropic large tool-parameter generation failures. The end state should include a synthetic OpenAI-compatible regression that withholds tool-call argument chunks long enough to exercise the idle/request-timeout path, plus the chosen provider/runtime fix for safe timeout behavior or supported tool-argument streaming.
