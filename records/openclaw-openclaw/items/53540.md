## Summary

Keep this issue open. Current main has partial timeout plumbing that may mitigate explicit no-timeout subagent runs, but OpenRouter Anthropic models still route through the OpenAI-compatible transport, OpenRouter does not enable fine-grained Anthropic tool-argument streaming, and the default embedded-runner idle watchdog can still fire when an OpenAI-compatible stream withholds chunks while large tool parameters are generated.

## Best Possible Solution

Keep this issue open as the canonical tracker for large OpenRouter/Anthropic tool-parameter generation failures. Add a synthetic OpenAI-compatible streaming regression where no chunks are emitted while a large tool-call argument is generated, then fix the implicated path by extending/disabling the silent idle policy for this case, enabling supported tool-argument streaming for OpenRouter Anthropic routes if available, or adding a runner/tool-loop strategy that avoids single huge Write-style arguments.
