## Summary

Keep open. Current main still does not prove a complete fix for the updated large tool-call-parameter failure: OpenRouter routes through the OpenAI-compatible transport, Anthropic fine-grained tool streaming is limited to direct Anthropic routes, and the embedded-runner idle watchdog remains chunk-based unless a provider timeout is explicitly configured.

## What This Changes

The issue requests that embedded LLM calls survive slow generation of large tool-call parameters, especially OpenRouter to Anthropic `Write` calls that currently fail with `Network connection lost.`

## Best Possible Solution

Keep this as the canonical tracker for OpenRouter/Anthropic large tool-parameter generation failures. The durable fix should add a synthetic OpenAI-compatible regression that withholds tool-call argument chunks, then implement the chosen policy: supported tool-argument streaming, a safe timeout model for silent generation, or a runtime/tool-loop strategy that avoids one huge tool payload.
