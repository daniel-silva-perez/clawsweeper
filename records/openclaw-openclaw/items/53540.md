## Summary

Keep open. Current main still lacks source, test, or docs proof for the updated OpenRouter-to-Anthropic large tool-parameter failure: OpenRouter remains on the OpenAI-compatible stream path, direct Anthropic fine-grained tool streaming does not apply to that route, and the embedded runner still uses a chunk-idle watchdog that can abort a silent stream by default.

## What This Changes

The issue asks OpenClaw embedded runs to survive or clearly handle LLM calls that spend longer than the underlying timeout generating large tool-call parameters, especially OpenRouter-to-Anthropic `Write` calls with large `content` arguments.

## Best Possible Solution

Keep this as the canonical tracker for OpenRouter/Anthropic large tool-parameter generation failures. The end state should add a synthetic OpenAI-compatible streaming regression that withholds tool-call argument chunks long enough to exercise the watchdog path, then implement the selected provider/runtime policy for safe timeout behavior or supported tool-argument streaming.

## Reproduction Assessment

Yes: the issue gives a precise OpenRouter/Anthropic Opus subagent `Write` repro that should hold the OpenAI-compatible stream silent while large tool arguments are generated, and current source still has a 120s chunk-idle watchdog. I did not run the provider live in this read-only pass.

## Solution Assessment

No current-main solution is proven complete. Existing provider timeout plumbing and direct Anthropic fine-grained tool streaming may help some routes, but they do not cover the default OpenRouter OpenAI-compatible path without a maintainer policy decision.
