## Summary

Keep open. Current main has useful timeout plumbing, but it still does not prove a complete fix for the updated OpenRouter-to-Anthropic large tool-parameter failure: OpenRouter resolves through the OpenAI-compatible transport, direct Anthropic fine-grained tool streaming is gated away from that route, and the default direct-agent path still has a 120-second chunk-idle watchdog unless a provider request timeout or explicit no-timeout run changes the policy.

## What This Changes

The issue requests that embedded LLM calls survive slow generation of large tool-call parameter payloads instead of failing with a generic network-loss error when argument deltas are delayed or withheld.

## Best Possible Solution

Keep this as the canonical tracker for OpenRouter/Anthropic large tool-parameter generation failures. The durable fix should add a synthetic OpenAI-compatible regression where tool-call argument chunks are withheld long enough to exercise the idle/request timeout path, then implement the chosen runtime policy for silent tool-argument generation or supported provider-side argument streaming.
