## Summary

Keep open. Current main still resolves wake-triggered heartbeats through an originating forced session before checking explicit `heartbeat.session`, and Telegram private DMs still default to `agent:main:main` in tests and docs. The linked open PR #56092 uses closing syntax for this issue, so the issue should remain open while that implementation candidate is reviewed or closed.

## What This Changes

The issue requests stricter isolation so Telegram DMs and system-event-triggered heartbeats cannot pollute `agent:main:main` or each other’s configured sessions.

## Best Possible Solution

End state should make configured heartbeat sessions win for system-event-triggered wakes, preserve or migrate pending system events without cross-session pollution, and separately settle Telegram DM default behavior by either documenting intentional main-session sharing or changing the Telegram plugin to isolate DMs with matching tests and docs.
