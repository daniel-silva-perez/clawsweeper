## Summary

Keep open. Current main still has the two relevant unresolved behaviors: wake-triggered heartbeats can carry an originating session that wins before explicit `heartbeat.session`, and default Telegram private DMs are still tested and documented as sharing `agent:main:main`. The linked open PR #56092 uses closing syntax for this issue, so the issue should remain open while that implementation candidate is reviewed rather than being closed ahead of merge.

## What This Changes

The issue asks for Telegram DMs and heartbeat-triggered work to remain isolated so ordinary direct-chat traffic cannot pollute `agent:main:main` or a configured heartbeat session.

## Best Possible Solution

End state should make configured heartbeat sessions win for system-event-triggered wakes, preserve or migrate pending system events without cross-session pollution, and settle Telegram DM default behavior by either documenting intentional main-session sharing or changing the Telegram plugin to isolate DMs with matching tests and docs.
