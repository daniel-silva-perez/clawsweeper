## Summary

Keep open. Current main still resolves a non-subagent wake-provided session before an explicit heartbeat session, so the confirmed exec/system-event heartbeat pollution path remains present, and Telegram private DMs still default to the agent main session by documented/tested policy. The open PR #56092 uses closing syntax for this issue, so maintainers should review, land, or supersede that implementation instead of closing the report early.

## What This Changes

The report asks OpenClaw to prevent Telegram DMs and heartbeat/system-event runs from sharing or polluting `agent:main:main`, especially by isolating Telegram DM sessions and honoring dedicated heartbeat session configuration.

## Best Possible Solution

Review, land, or supersede #56092 for system-event-triggered heartbeat routing so configured heartbeat sessions win over originating wake sessions, then separately settle whether Telegram DMs should keep the documented main-session default or move to isolated direct-session keys with aligned tests and docs.

## Reproduction Assessment

Yes for the confirmed exec-completion subcase: `notifyOnExit` enqueues a system event on the originating session, the wake path forwards that session key, and current heartbeat resolution accepts it before `heartbeat.session`. The original Telegram-only pollution path is still unclear without a fresh minimal live repro.

## Solution Assessment

Unclear until #56092 is reviewed. Its direction matches the confirmed root cause, but the durable fix should preserve pending system events safely and treat the Telegram DM default as a separate code/docs/product decision.
