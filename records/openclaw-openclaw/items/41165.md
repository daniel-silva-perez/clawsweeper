## Summary

Keep open. Current main still lets an originating wake session resolve before an explicit `heartbeat.session`, while Telegram private DMs still default to `agent:main:main`; open PR #56092 uses closing syntax for this issue, so the issue should remain open until that implementation candidate is reviewed and either merged or closed.

## What This Changes

Telegram DMs and system-event-triggered heartbeats should stay in the intended isolated or configured sessions instead of polluting `agent:main:main` or an active DM session.

## Best Possible Solution

The end state should make configured heartbeat sessions win for system-event-triggered wakes, preserve or migrate pending system events without cross-session pollution, and explicitly settle Telegram DM default behavior through either documented intentional main-session sharing or a Telegram plugin isolation change with matching tests and docs.
