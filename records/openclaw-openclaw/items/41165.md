## Summary

Keep open. Current main still documents and tests default direct-message routing to the main session, and the heartbeat runner still lets a wake/exec-originating forced session win before `heartbeat.session`. The provided discussion also has #56092 open and unmerged with closing syntax for this issue, so the issue should remain open while that implementation candidate is reviewed or explicitly closed.

## Best Possible Solution

Keep this issue open while maintainers review #56092 and either land it or close it with explicit rationale. Separately decide the Telegram product behavior: either keep and document default DM-to-main routing as intentional with `session.dmScope`/`isolatedSession` guidance, or implement Telegram DM isolation in the Telegram plugin with matching routing tests and docs.
