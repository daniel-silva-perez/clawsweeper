## Summary

Keep open. Current main still has the reported session-routing risk: system-event heartbeat wakes can carry an originating session that is resolved before an explicit `heartbeat.session`, and Telegram private DMs with the default runtime `dmScope` still target `agent:main:main`. The open implementation PR #56092 uses closing syntax for this report, so the issue should wait for maintainer review, merge, or supersession rather than being closed early.

## What This Changes

The report asks for Telegram direct messages and heartbeat/system-event runs to stay in separate session namespaces so direct-message traffic cannot pollute `agent:main:main` or heartbeat history.

## Best Possible Solution

Configured heartbeat sessions should win for system-event-triggered wakes, pending system events should be handled without cross-session pollution, and the Telegram DM default should be settled explicitly with aligned Telegram tests and public channel-routing docs.
