## Summary

Keep open. Current main still has the two relevant unresolved paths: wake-triggered heartbeats can use an originating session before an explicit heartbeat session, and Telegram direct messages still default to `agent:main:main` unless configuration isolates DMs. The provided context also shows open PR #56092 uses GitHub closing syntax for this issue, so the issue should stay open until that implementation candidate is reviewed and either merged or closed.

## What This Changes

Telegram direct messages and system-event-triggered heartbeat wakes should stay isolated from the agent main or heartbeat session so user DM traffic cannot pollute heartbeat context.

## Best Possible Solution

Make configured heartbeat sessions take precedence over originating wake sessions, safely migrate or re-home pending system events without cross-session pollution, and settle Telegram DM default behavior through either an explicit docs-backed product decision or a Telegram routing change with matching tests.
