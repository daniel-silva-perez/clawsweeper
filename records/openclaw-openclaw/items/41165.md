## Summary

Keep open. Current main still has the relevant unresolved paths: heartbeat session resolution accepts a non-subagent forced session before checking `heartbeat.session`, exec/wake paths continue to pass the originating session into that branch, and Telegram private DMs without an isolated `dmScope` are still documented and tested as `agent:main:main`. The linked open PR #56092 uses closing syntax and should be reviewed before this issue is closed.

## What This Changes

The issue requests stricter isolation so Telegram DMs and system-event-triggered heartbeats cannot pollute `agent:main:main` or each other's configured sessions.

## Best Possible Solution

Land or supersede #56092 for configured heartbeat session precedence and safe system-event handling, then settle the Telegram DM default behavior explicitly with aligned Telegram tests and public channel-routing docs.
