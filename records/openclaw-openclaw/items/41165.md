## Summary

Keep open. Current main still has the two relevant unsolved behaviors: exec/system-event heartbeat wakes can carry an originating session that is resolved before an explicit heartbeat session, and Telegram private DMs still default to the agent main session unless dmScope or account fallback isolates them. The related open PR #56092 uses closing syntax for this issue and is the right implementation candidate to review rather than closing the issue ahead of merge.

## What This Changes

The issue requests stricter isolation so Telegram DMs do not pollute `agent:main:main` and system-event-triggered heartbeats honor a dedicated heartbeat session instead of the originating chat session.

## Best Possible Solution

End state should make configured heartbeat sessions win for system-event-triggered wakes, preserve or migrate pending system events without cross-session pollution, and separately settle Telegram DM default behavior by either documenting intentional main-session sharing or changing the Telegram plugin to isolate DMs with matching tests and docs.
