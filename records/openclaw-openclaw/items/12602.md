## Summary

Keep open. Current main already supports authored Slack Block Kit payloads, shared presentation blocks, message-tool `blocks`, and opt-in interactive controls, but the ordinary Slack markdown reply path still renders through text/mrkdwn chunks and `markdown.tables: "block"` is still coerced to `code`. The issue remains a valid umbrella for the unresolved markdown-table/native rich-output path, with #57717 and #57708 carrying the active follow-up scope.

## What This Changes

Request richer Slack agent responses by allowing Block Kit payloads, structured sections, tables, buttons, selects, context blocks, and safe fallbacks instead of plain markdown-only replies.

## Best Possible Solution

Keep this as the Slack rich-output umbrella until maintainers either land native markdown-table Block Kit delivery or explicitly split that work to the narrower active follow-ups; preserve the existing raw-block, presentation, and interactive-directive paths with tests and public docs for any user-visible send-path change.
