## Summary

Keep open. Current main supports authored Slack Block Kit payloads and opt-in interactive reply controls, but the broader Slack rich-output umbrella is still incomplete because the ordinary markdown reply path still renders text chunks and still coerces `markdown.tables: "block"` back to code rendering. The supplied discussion also identifies #57717 and #57708 as active follow-ups rather than cleanup closure.

## What This Changes

Request to let Slack agents send richer Block Kit responses, including structured sections, tables, context, buttons, selects, and safe text fallback.

## Best Possible Solution

Keep this as the Slack richer-output umbrella until the native markdown-table Block Kit send path lands or is replaced, then continue the broader rich_text conversion discussion in #57708 with safe text/code fallbacks and public Slack docs plus changelog updates for user-visible behavior.
