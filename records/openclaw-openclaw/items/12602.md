## Summary

Current main still does not fully implement the remaining Slack Block Kit umbrella. Raw `channelData.slack.blocks`, shared presentation blocks, and interactive buttons/selects are implemented, but normal Slack markdown replies still go through mrkdwn text chunking, and `markdown.tables: "block"` is still coerced back to `code`. The latest issue discussion also explicitly keeps this issue open until the table/rich Markdown path lands or is superseded.

## Best Possible Solution

Keep this issue open for now. The best path is to review, rebase, and either land or replace #57717 for the Slack markdown-table send path, while keeping #57708 as the focused rich_text design follow-up if maintainers want broader Markdown-to-Block-Kit conversion. The implementation should stay in the bundled Slack plugin, consume `markdownToIRWithMeta` table metadata when `markdown.tables: "block"` is configured, add Slack outbound/slash/media fallback coverage, document the behavior on the public Slack docs page, and avoid duplicating the already-shipped raw block, presentation, and interactive directive support.
