## Summary

Keep open. Current main covers authored Slack blocks, shared presentation blocks, and opt-in interactive reply controls, but the ordinary Slack markdown reply path still sends text chunks and still coerces `markdown.tables: "block"` back to code rendering. The supplied issue discussion and latest maintainer review also identify #57717 and #57708 as remaining follow-ups rather than cleanup closure.

## What This Changes

Allow Slack agents to send richer Block Kit responses, including structured blocks, visual hierarchy, buttons/selects, context metadata, and better table rendering instead of plain text only.

## Best Possible Solution

Keep this open as the Slack richer-output umbrella. Land or replace #57717 for markdown-table Block Kit delivery, continue #57708 for broader Markdown-to-Slack rich_text conversion, preserve safe text/code fallbacks, and update the public Slack docs and changelog when user-visible behavior changes.
