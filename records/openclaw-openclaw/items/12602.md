## Summary

Keep open. Current main implements explicit Slack Block Kit delivery and opt-in interactive reply controls, but the broader rich Slack output request is still incomplete because ordinary markdown replies remain mrkdwn text chunks and `markdown.tables: "block"` is still downgraded to code rendering; #57717 and #57708 are active follow-ups for the remaining table/rich-text scope.

## What This Changes

The issue requests richer Slack agent-message output via Block Kit, including structured blocks, buttons/selects, context blocks, and native table-style rendering instead of plain markdown text.

## Best Possible Solution

Keep this as the Slack rich-output umbrella until the native markdown-table Block Kit send path lands or maintainers explicitly split that scope out, while leaving broader rich_text conversion design in #57708 and preserving the shipped raw-block and interactive-directive support.
