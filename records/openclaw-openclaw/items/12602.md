## Summary

Keep open. Current main implements direct authored Slack blocks, message-tool blocks, shared presentation blocks, and opt-in interactive controls, but the ordinary markdown reply path still sends text/mrkdwn chunks and still coerces `markdown.tables: "block"` back to code rendering. The supplied discussion identifies #57717 and #57708 as active follow-ups for the remaining table/rich-output scope, so closing this umbrella now would be premature.

## What This Changes

The issue requests richer Slack agent output through Block Kit-style sections, buttons, selects, context, and table rendering rather than only plain mrkdwn text.

## Best Possible Solution

Keep this as the Slack rich-output umbrella until native markdown-table Block Kit delivery lands or maintainers explicitly split that scope to the active follow-ups; preserve the shipped raw-block, presentation, and interactive-directive paths with docs and changelog coverage for any user-visible send-path change.
