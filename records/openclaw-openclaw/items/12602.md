## Summary

Keep open. Current main already supports authored Slack Block Kit payloads and opt-in interactive reply controls, but the umbrella is not fully implemented because normal Slack markdown replies still render through mrkdwn text chunks and `markdown.tables: "block"` is explicitly coerced back to code rendering. The supplied discussion also points to #57717 and #57708 as active follow-ups rather than evidence that this can be closed.

## What This Changes

Request to let Slack agents send richer Block Kit messages, including structured sections, fields, dividers, buttons, selects, context blocks, and better table-style output.

## Best Possible Solution

Keep this as the Slack rich-output umbrella until the native markdown-table Block Kit send path lands or maintainers explicitly split that scope out, while leaving broader rich_text conversion design in #57708 and preserving the shipped raw-block and interactive-directive support.
