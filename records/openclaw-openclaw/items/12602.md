## Summary

Current main still only partially satisfies the Slack Block Kit umbrella. Authored `channelData.slack.blocks`, shared presentation blocks, and opt-in interactive buttons/selects are present, but ordinary Slack markdown replies still render through mrkdwn text chunking, and `markdown.tables: "block"` is still coerced back to `code`. The latest discussion and local ClawSweeper records also keep the remaining table/rich Markdown work open under #57717 and #57708.

## Best Possible Solution

Keep this issue open as the umbrella until maintainers either land or replace #57717 for Slack markdown-table Block Kit delivery, and keep #57708 as the focused design follow-up for broader Markdown-to-Slack-rich_text conversion. The implementation should stay in the bundled Slack plugin, consume `markdownToIRWithMeta` table metadata when `markdown.tables: "block"` is enabled, preserve safe text/code fallback behavior, add Slack outbound/slash/media coverage, and document the public Slack behavior.
