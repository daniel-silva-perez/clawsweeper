## Summary

Keep open. Current main supports explicit Slack Block Kit payloads and opt-in interactive controls, but the ordinary markdown reply path still does not send native Slack table blocks and the broader automatic rich Slack rendering work remains split across active follow-ups #57717 and #57708.

## What This Changes

Request Slack agent messages support richer Block Kit output, including structured sections, fields, context, buttons/selects, and table/rich rendering instead of plain text-only replies.

## Best Possible Solution

Keep this as the Slack rich-output umbrella until native markdown-table Block Kit delivery lands or maintainers explicitly split that scope out; then leave the broader automatic `rich_text` conversion design to #57708 while preserving the shipped raw-block and interactive-directive paths.
