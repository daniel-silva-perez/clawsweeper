## Summary

Keep open: current main implements several important Slack Block Kit slices, including authored raw blocks, message-tool blocks, shared presentation rendering, and interactive controls, but the ordinary Slack markdown reply path still does not render markdown tables as native Slack Block Kit table blocks. The supplied discussion also points to #57717 and #57708 as active follow-ups, so closing this umbrella now would hide remaining user-visible work.

## What This Changes

The issue requests richer Slack agent-message output using Slack Block Kit structures such as sections, fields, dividers, buttons, select menus, context blocks, and table-like layouts.

## Best Possible Solution

Keep this as the Slack rich-output umbrella until maintainers land native markdown-table Block Kit delivery or explicitly split that remaining work to #57717 and #57708. Any send-path change should preserve the existing raw-block, presentation, and interactive-directive behavior with focused Slack tests and public docs updates.

## Reproduction Assessment

Yes for the remaining observable gap: current main resolves Slack markdown.tables = block to code, and ordinary Slack replies without explicit blocks flow through markdownToSlackMrkdwnChunks. That source path is enough to reproduce why markdown tables still do not become native Slack table blocks on the default reply path.

## Solution Assessment

No, closing this as implemented is not the best current action. Raw Block Kit and interactive slices are implemented, but the table/rich-output use case remains open and is better resolved by reviewing the active follow-ups rather than starting a duplicate repair.
