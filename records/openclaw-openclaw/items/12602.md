## Summary

Keep open. Current main implements important Slack Block Kit slices, including explicit raw blocks, shared presentation rendering, and interactive controls, but the remaining umbrella gap is still present: ordinary Slack markdown replies still use text/mrkdwn chunking and `markdown.tables: "block"` is still coerced back to `code`. The discussion also identifies #57717 and #57708 as active follow-ups, so closing this now would hide remaining user-visible Slack rich-output work.

## What This Changes

The issue requests richer Slack agent messages using Block Kit payloads, structured sections, buttons/selects, context blocks, and native table-style rendering instead of plain text or markdown-only replies.

## Best Possible Solution

Keep this as the Slack rich-output umbrella until maintainers land or explicitly replace the native markdown-table Block Kit send path, then move the broader markdown-to-rich_text conversion design into #57708. The implementation should stay in the Slack plugin send/rendering layer, preserve explicit raw blocks and shared presentation behavior, keep conservative text/code fallbacks, and update public Slack or presentation docs for user-visible behavior.

## Reproduction Assessment

Yes. For the remaining gap, source and tests show `channels.slack.markdown.tables: "block"` is still resolved to `code`, and the ordinary Slack reply path posts mrkdwn text chunks when no explicit blocks are supplied.

## Solution Assessment

No, closing this as implemented is not the best current action. Raw Block Kit and shared presentation support are implemented, but the remaining native table/rich-output work is better resolved by reviewing #57717 and scoping #57708 rather than starting a duplicate automated repair.
