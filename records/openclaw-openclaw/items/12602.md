## Summary

Keep open. Current main already supports explicit Slack Block Kit payloads, message-tool blocks, presentation blocks, and opt-in interactive controls, but the ordinary markdown reply path still renders text chunks and still coerces `markdown.tables: "block"` back to code, so the table/rich-output portion discussed in the issue remains unresolved.

## What This Changes

Request richer Slack agent messages using Block Kit structures such as sections, context, tables, buttons, selects, and fallback text instead of plain markdown-only responses.

## Best Possible Solution

Keep this as the Slack rich-output umbrella until native markdown-table Block Kit delivery lands or maintainers explicitly split that scope out; leave broader automatic `rich_text` conversion to #57708 while preserving the shipped raw-block and interactive-directive paths.
