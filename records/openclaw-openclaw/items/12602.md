## Summary

Current main partially satisfies the original Slack Block Kit request, but not enough to close the umbrella. Explicit `channelData.slack.blocks`, Slack message-tool `blocks`, presentation blocks, and opt-in interactive replies are present. The ordinary Slack markdown reply path still renders text chunks through mrkdwn, `markdown.tables: "block"` is still coerced to `code`, and the remaining table/rich-text work is already represented by open follow-ups #57717 and #57708 plus the latest maintainer review keeping this issue open.

## Best Possible Solution

Keep this issue open as the Slack richer-output umbrella until maintainers land or replace #57717 for markdown-table Block Kit delivery. Broader automatic Markdown-to-Slack-rich_text conversion should stay in #57708. The implementation should remain in the bundled Slack plugin, consume shared markdown table metadata only when `markdown.tables: "block"` is enabled, preserve safe text/code fallbacks, cover outbound/slash/media/preview paths, and document the public Slack behavior.
