## Summary

Keep open. Current main still lacks the requested pre-write path guard and exact audit remediation command, but this PR is not merge-ready because the new schema walker can reject valid nested SecretRef paths used by the current generated config schema.

## What This Changes

The PR adds `openclaw config set` path validation, workspaceOnly path suggestions, an exact workspaceOnly security-audit remediation command, and related CLI/config/security/test-environment coverage.

## Best Possible Solution

Keep the useful direction, rebase onto current main, and make the validator traverse the runtime schema with full recursive combinator support, preferably by reusing the existing schema lookup traversal semantics so path validation matches `openclaw config schema`.

## Reproduction Assessment

Yes. Static source proof is enough: against the PR logic, `validateConfigPath(["models","providers","openai","apiKey","provider"], schemaInfo)` reaches an `anyOf` member that wraps a nested `oneOf`, then rejects it before normal whole-config validation could accept an existing SecretRef object edit.

## Solution Assessment

No. The PR targets the right narrow user problem, but the current validator is not the best implementation because it duplicates only a partial JSON Schema traversal; recursive combinator expansion, and possibly reuse of the schema lookup path, is the safer fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Traverse nested schema combinators:** `src/config/validate-config-path.ts:40-53`
  - body: `expandSchemaVariants` only includes direct `anyOf`/`oneOf`/`allOf` members. Current generated config schema represents SecretRef fields as `anyOf` with a nested `oneOf` of object shapes, so valid updates like `models.providers.openai.apiKey.provider` can be rejected as an unknown config path before normal config validation accepts them. Recursively flatten combinator wrappers, or reuse the existing schema lookup traversal, before rejecting the path.
