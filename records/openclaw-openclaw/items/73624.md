## Summary

Keep open. Current main still has the reported prompt-composition bug: the override branch returns before the system-prompt builders that append `extraSystemPrompt`, and the directly related implementation PR was closed unmerged.

## What This Changes

The issue asks for agent-level `systemPromptOverride` to remain active while still appending subagent task instructions carried through `params.extraSystemPrompt`.

## Best Possible Solution

Add a shared prompt-override composition helper, use it in the embedded attempt, compaction retry, and CLI prompt assembly paths, and add focused regression coverage proving override-only, no-override, and override-plus-extra behavior.

## Reproduction Assessment

Yes. A static reproduction path exists on current main: configure `agents.defaults.systemPromptOverride` or per-agent `systemPromptOverride`, spawn a native subagent so the task is carried in `extraSystemPrompt`, and observe the override-first branches return before the builders append that extra prompt.

## Solution Assessment

Yes. The narrow helper approach is the best maintainable fix because it preserves existing no-override and override-only behavior while appending trimmed runtime extra prompt text only when both an override and extra prompt are present.
