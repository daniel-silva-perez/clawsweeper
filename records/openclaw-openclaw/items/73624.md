## Summary

Keep open. Current main still has the reported prompt-composition failure: native subagent spawn puts the delegated task into extraSystemPrompt, but the embedded/CLI prompt paths short-circuit to systemPromptOverride before the builder path that appends that extra prompt. The linked open PR #73637 uses closing syntax and is an implementation candidate, so this issue should remain open until that PR, or an equivalent fix, is merged.

## Best Possible Solution

Review and land PR #73637, or an equivalent source change, so systemPromptOverride is preserved while runtime extraSystemPrompt is appended for subagent/task contexts across embedded run, compaction, and CLI prompt paths. Add regression coverage that asserts a subagent spawned under an agent with systemPromptOverride receives both the override text and the generated Your Role task block.
