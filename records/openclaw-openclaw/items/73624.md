## Summary

Keep open. Current main still has the reported prompt-composition bug in the embedded attempt, compaction retry, and CLI prompt assembly paths, while the directly linked implementation PR was closed unmerged.

## What This Changes

The issue asks for subagent runs to preserve agent-level `systemPromptOverride` while still appending task-specific `extraSystemPrompt` containing the delegated `Your Role` block.

## Best Possible Solution

Add a shared prompt-override composition helper and use it in all three current prompt assembly paths so overrides remain intact while runtime `extraSystemPrompt` is appended. Keep the visible initial-message behavior separate unless maintainers decide to address #73656 at the same time.

## Solution Assessment

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/73624. Current main drops subagent task instructions when an agent has `systemPromptOverride` because `src/agents/pi-embedded-runner/run/attempt.ts`, `src/agents/pi-embedded-runner/compact.ts`, and `src/agents/cli-runner/prepare.ts` use `resolveSystemPromptOverride(...) ?? build...({ extraSystemPrompt })`. Add a small shared helper next to `resolveSystemPromptOverride` that returns undefined when there is no override, returns the trimmed override when there is no nonblank `extraSystemPrompt`, and returns `<override>\n\n<extraSystemPrompt>` when both are present. Use that helper in all three prompt assembly paths while preserving the existing builder fallback for no-override agents. Add focused regression coverage in `src/agents/system-prompt-override.test.ts` and at least one caller-level test proving a configured `systemPromptOverride` plus runtime `extraSystemPrompt` produces a prompt containing both the override and the delegated `## Your Role` block. Do not change the visible initial subagent message behavior; #73656 tracks that separately. Add a user-facing changelog fix entry if the branch changes runtime behavior, crediting the reporter and any reused closed-PR source work per repo changelog policy.
