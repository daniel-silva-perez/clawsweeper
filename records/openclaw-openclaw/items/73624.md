Keep open. Current main still has the override-first prompt assembly shape that drops `extraSystemPrompt` whenever `systemPromptOverride` resolves, and the directly related fix PR was closed unmerged, so the reported subagent task-loss path remains valid and actionable.

## What This Changes

The issue requests preserving task-specific subagent instructions from `params.extraSystemPrompt` when an agent-level `systemPromptOverride` is configured.

## Best Possible Solution

Compose nonblank runtime `extraSystemPrompt` additions with resolved prompt overrides through a shared helper and apply it consistently across embedded attempt, compaction retry, and CLI runner prompt assembly, while leaving visible first-message task duplication to the separate #73656 discussion.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/73624. Current main drops subagent task instructions when `systemPromptOverride` resolves because `params.extraSystemPrompt` is only passed into fallback system prompt builders. Add a small shared helper next to `resolveSystemPromptOverride` that returns undefined when no override exists, returns the trimmed override when there is no nonblank extra prompt, and returns `<override>\n\n<extraSystemPrompt>` when both are present. Use it in `src/agents/pi-embedded-runner/run/attempt.ts`, `src/agents/pi-embedded-runner/compact.ts`, and `src/agents/cli-runner/prepare.ts`. Add regression tests in `src/agents/system-prompt-override.test.ts` and update the nearest CLI/embedded prompt helper tests so override-backed runs include both the override and the dynamic task/context block. Do not change the visible subagent bootstrap message behavior from #73656, do not duplicate the task into the first user message, and do not change plugin/channel boundaries. Add a `CHANGELOG.md` fix entry because this is user-facing runtime behavior; credit `@ramitrkar-hash` and, if reusing the prior PR shape, `@hclsys`. Clarify `docs/gateway/config-agents.md` only if the public `systemPromptOverride` semantics need to mention runtime-delivered subagent/task context still being appended.
