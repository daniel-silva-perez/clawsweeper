## Summary

Keep open. Current main still has the override-first prompt assembly shape in the embedded attempt, compaction retry, and CLI runner paths, so a configured `systemPromptOverride` bypasses the builder branch that appends the subagent task carried in `extraSystemPrompt`; the directly related fix PR #73637 is closed unmerged, and #73656 tracks a separate visible-bootstrap fallback request.

## What This Changes

Preserve task-specific subagent instructions from `params.extraSystemPrompt` when an agent-level `systemPromptOverride` is configured.

## Best Possible Solution

Add a shared, tested prompt composition helper near `resolveSystemPromptOverride` that preserves existing no-override behavior, returns the trimmed override alone when no runtime extra prompt exists, and appends trimmed `extraSystemPrompt` with a blank-line separator when both are present. Apply it consistently to the embedded attempt, compaction retry, and CLI runner prompt assembly paths, with docs or changelog updates only for the public behavior being clarified.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: configure an agent with `systemPromptOverride`, spawn a native subagent, and the child task flows through `extraSystemPrompt` while the override branch returns before the builders append that runtime prompt block.

## Solution Assessment

Yes. The issue’s proposed composition helper is the narrowest maintainable fix because it preserves agent-level overrides and only changes the override branch to retain runtime prompt additions; changing the visible bootstrap message is a separate fallback design tracked by #73656.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/73624. Current main passes native subagent task instructions through `params.extraSystemPrompt`, but `systemPromptOverride` short-circuits the prompt builders in `src/agents/pi-embedded-runner/run/attempt.ts`, `src/agents/pi-embedded-runner/compact.ts`, and `src/agents/cli-runner/prepare.ts`. Add a small helper next to `resolveSystemPromptOverride` that composes a trimmed override with a trimmed runtime `extraSystemPrompt`: return `undefined` when no override exists so callers keep using the existing builders, return the override alone when extra is missing or blank, and return `override + "\n\n" + extra` when both exist. Use the helper at all three call sites. Add focused tests in `src/agents/system-prompt-override.test.ts` and, where practical, a runner-level assertion that override-backed prompts still include dynamic extra prompt text. Do not address #73656 by moving the full delegated task into `buildSubagentInitialUserMessage`; that is separate. Update `docs/gateway/config-agents.md` only if clarifying the public semantics, and add a `CHANGELOG.md` fix entry with appropriate contributor attribution. Avoid unrelated prompt refactors.
