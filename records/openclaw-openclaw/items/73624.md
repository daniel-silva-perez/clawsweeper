## Summary

Keep open. Current main still has the reported prompt-composition bug: subagent task instructions are passed through extraSystemPrompt, but all three prompt assembly paths short-circuit to systemPromptOverride before the builder that appends that runtime text can run. The linked implementation PR #73637 was closed unmerged after prompt-authority/security concerns, so this should not be cleaned up as implemented or stale.

## What This Changes

The issue asks OpenClaw to preserve agent-level systemPromptOverride while still appending task-specific extraSystemPrompt for subagent runs.

## Best Possible Solution

Keep the issue open until maintainers choose the prompt contract. The eventual fix should preserve the subagent task across embedded attempt, compaction retry, and CLI runner paths, add override-plus-extra regression coverage, and update the public systemPromptOverride docs if runtime extra prompt text is intentionally no longer treated as fully replaced.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: configure a non-empty systemPromptOverride, spawn a native subagent so the task enters child params.extraSystemPrompt, and the override-first ?? branches return before the builders append that extra prompt.

## Solution Assessment

Unclear. The reporter's local concat patch and PR #73637 fix the functional drop, but the closed PR discussion shows raw system-message concatenation has prompt-authority tradeoffs; the safer path needs maintainer choice between a documented combined-prompt helper and a broader separate-message contract.
