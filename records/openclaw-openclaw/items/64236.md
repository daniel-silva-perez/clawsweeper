## Summary

Keep this issue open. Current main still lacks a per-model `reasoningDefault` slot under `agents.defaults.models[...]`; reasoning visibility is only first-class on `agents.list[]` plus a catalog-derived on/off fallback, so the requested per-model `stream` or `off` defaults are not implemented.

## Best Possible Solution

Keep this issue open and implement a first-class per-model reasoning visibility default on the model entry config, likely `AgentModelEntryConfig` / `agents.defaults.models[modelRef].reasoningDefault`, with Zod schema, generated schema/help, docs, config validation, status/reporting, and runtime precedence. A reasonable precedence is directive, session override, per-agent default, selected-model default, then catalog capability fallback. Review #71817 separately as partial channel work, not as completion of this request.
