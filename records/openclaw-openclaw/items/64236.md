## Summary

Keep this issue open. Current main has adjacent per-agent and global `reasoningDefault` support, but the requested per-model `reasoningDefault` under `agents.defaults.models[...]` is still absent from the model-entry type, strict config schema, generated schema, runtime resolution, and user docs. The related open PR #71817 is useful context for Telegram/Feishu per-agent fallback behavior, but it does not implement this core per-model config surface.

## What This Changes

Allow `reasoningDefault` to be configured per model under `agents.defaults.models[...]` so reasoning visibility can default differently by selected model.

## Best Possible Solution

Add a first-class, additive `reasoningDefault: "off" | "on" | "stream"` field to model entries, validate and generate schema/help metadata for it, resolve it for the selected model after directive/session/per-agent defaults and before global/catalog fallback, and update docs/tests for both explicit `off` and `stream` cases.
