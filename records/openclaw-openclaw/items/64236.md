## Summary

Keep this issue open. Current main now has a global `agents.defaults.reasoningDefault`, but it still does not support the requested per-model `reasoningDefault` inside `agents.defaults.models[...]`; the model-specific fallback only derives `on` or `off` from catalog `reasoning` metadata and cannot express `stream` or model-by-model visibility overrides.

## What This Changes

Add per-model reasoning visibility defaults to model pool entries so different selected models can default to `on`, `off`, or `stream` without per-session commands.

## Best Possible Solution

Add an additive, documented `reasoningDefault: "off" | "on" | "stream"` field to `AgentModelEntryConfig` and the generated config schema, then resolve it from the selected model after directive/session/per-agent defaults and before the global/catalog fallback. Keep the catalog `reasoning: true` fallback as a compatibility default, and update docs/tests so `stream` is explicitly covered where the channel supports it.
