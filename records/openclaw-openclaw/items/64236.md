## Summary

Keep open. Current main partially addresses the report by supporting global and per-agent reasoning defaults, but the requested per-model `reasoningDefault` under `agents.defaults.models[...]` is still absent from the config type, strict schema, generated schema, runtime resolution path, and user docs.

## What This Changes

The issue requests per-model `reasoningDefault` configuration so different configured models can default reasoning visibility to `off`, `on`, or `stream` without per-session commands.

## Best Possible Solution

Add an additive, documented `reasoningDefault: "off" | "on" | "stream"` field to configured model entries, validate and generate schema metadata for it, resolve it for the selected model in the runtime default order, and keep catalog `reasoning: true` as a compatibility fallback.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence verification path: the proposed key is absent from the model-entry type and rejected by the strict config schema, and runtime resolution never reads it.

## Solution Assessment

Yes. A narrow config/schema/runtime/docs/test change is the best fit; the channel-specific PR #71817 is useful context but is not a replacement for a core per-model config surface.
