## Summary

Keep this issue open. Current main now supports a global `agents.defaults.reasoningDefault` and per-agent `agents.list[].reasoningDefault`, but the requested per-model `reasoningDefault` under `agents.defaults.models[...]` is still absent from the config type/schema, generated schema, runtime fallback resolution, and docs. The related open PR #71817 is useful channel-specific context, but it only targets Telegram/Feishu per-agent fallback behavior.

## What This Changes

Requests model-entry-level `reasoningDefault` overrides so different configured models can default to `off`, `on`, or `stream` without per-session commands.

## Best Possible Solution

Add a first-class, additive `reasoningDefault: "off" | "on" | "stream"` field to model entries, validate and generate schema/help metadata for it, resolve it for the selected model before catalog fallback, and document/test precedence for explicit `off` and `stream` cases.
