## Summary

Keep this PR open. Current main still creates Discord gateway WebSockets without a `handshakeTimeout`, and the pinned Carbon 0.16.0 path does not supply one upstream. The PR's user problem remains valid, but its old one-file patch shape is stale against the newer OpenClaw Discord gateway wrapper and should be rebased so it preserves the injected WebSocket constructor, proxy agent, debug capture, transport-liveness events, and `binaryType` behavior.

## Best Possible Solution

Keep the PR open, but rebase or replace the patch for current main: add a plugin-owned 30s `handshakeTimeout` to the existing WebSocket constructor options object, widen `DiscordGatewayWebSocketCtor` and test mocks to accept `handshakeTimeout?: number`, preserve the proxy `agent`, injected constructor, capture hooks, transport-liveness events, and `binaryType` assignment, then update focused direct and proxy WebSocket tests to assert the timeout.
