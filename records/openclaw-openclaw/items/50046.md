## Summary

Keep this PR open. Current main still constructs Discord gateway WebSockets without a `handshakeTimeout`, while the pinned `ws` contract supports that option and Carbon 0.16.0 still constructs `new WebSocket(url)` without options. The PR is functionally relevant, but its patch needs a rebase onto the newer Discord gateway socket wrapper so it preserves the injected constructor, proxy agent, transport capture, and binaryType handling.

## Best Possible Solution

Keep the PR open and rebase it onto current main. Add a Discord-plugin-owned 30s handshake timeout to the existing WebSocketCtor options object, widen the local constructor option type to include `handshakeTimeout?: number`, preserve the proxy `agent`, injected testing constructor, transport capture, and binaryType handling, then update focused tests for both direct and proxy WebSocket construction. No core API change appears necessary.
