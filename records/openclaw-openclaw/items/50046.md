## Summary

Keep this PR open. Current main still creates Discord gateway WebSockets without a `handshakeTimeout`, while the pinned Carbon/ws contract supports the option and Carbon 0.16.0 does not provide it upstream. The PR remains relevant, but the patch needs to be rebased onto the newer Discord gateway wrapper so it preserves the injected WebSocket constructor, proxy agent, debug capture, transport-liveness event handling, and `binaryType` behavior.

## Best Possible Solution

Keep the PR open and rebase or replace it with a current-main Discord-plugin patch: add a plugin-owned 30s handshake timeout to the existing `WebSocketCtor` options object, widen `DiscordGatewayWebSocketCtor` and test mocks to include `handshakeTimeout?: number`, preserve the proxy `agent`, injected constructor, capture hooks, transport-liveness events, and `binaryType` assignment, then update focused direct and proxy WebSocket tests to assert the timeout.
