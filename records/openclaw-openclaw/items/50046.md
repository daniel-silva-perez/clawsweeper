## Summary

Keep this PR open. Current main still does not implement the Discord gateway WebSocket handshake timeout: SafeGatewayPlugin creates the socket with only an optional proxy agent, and existing tests still encode the no-timeout behavior. The PR remains functionally relevant, but it needs a rebase onto the newer gateway instrumentation and focused tests.

## Best Possible Solution

Keep the PR open and rebase it onto current main by adding a Discord-plugin-owned handshake timeout to the existing WebSocketCtor options object. Preserve the proxy agent, injected testing constructor, transport capture, and binaryType handling, and add focused regression assertions for both direct and proxy paths. No core API change appears necessary.
