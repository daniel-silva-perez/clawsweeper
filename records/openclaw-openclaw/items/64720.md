## Summary

Keep this PR open. Current main still does not apply external-content wrapping to shared channel message bodies or plain thread-context prompt text, so the hardening is not obsolete. The current PR head also still needs revision before merge because its trust decision uses route metadata and an incomplete internal-provider allowlist, which can misclassify heartbeat/cron automation prompts.

## What This Changes

This PR adds external-content wrapping for channel message bodies and thread-context bodies in auto-reply prompt assembly, extends external-content source labels, and adds inbound-text unit tests for normalization, sanitization, provider classification, and channel-body wrapping.

## Best Possible Solution

Revise the PR so external wrapping is derived from the actual ingress/source provider, not reply-routing metadata; keep channel body and thread-context wrapping for real external channel input; preserve raw queue summaries; explicitly treat heartbeat, cron-event, exec-event, node, CLI, and other internal sources as trusted; and add regression coverage for channel bodies, thread context, routed heartbeat delivery, and internal-only scheduler prompts.
