## Summary

Keep open. Current main has a canonical assistant-visible sanitizer that strips memory scaffolding, but the outbound streaming delivery paths still do not consistently use it; Telegram partial previews and text_end/block replies can still carry assistant text that only passed through narrower runtime-context cleanup.

## What This Changes

The issue requests stripping `<relevant-memories>` blocks from assistant responses before delivery through outbound messaging channels such as Telegram, Discord, Slack, and iMessage.

## Best Possible Solution

Use the existing code-fence-aware assistant-visible sanitizer on every assistant-origin outbound delivery path, including partial previews and block replies, and add focused regressions for Telegram/default streaming plus shared block delivery.
