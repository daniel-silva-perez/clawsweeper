Keep this PR open, but do not merge it as-is. The patch removes only the manifest suppression, while current main's docs, resolver path, and tests treat `openai-codex/gpt-5.4-mini` as unsupported on the PI Codex OAuth route; the PR's positive evidence is for the separate Codex app-server route.

This PR deletes the `openai-codex/gpt-5.4-mini` `modelCatalog.suppressions` entry from the OpenAI plugin manifest without updating provider routing, tests, docs, or changelog.

## Best Possible Solution

## Solution Assessment

No. Removing the manifest entry alone is not the best fix because it contradicts current route contracts and re-enables stale rows without route-specific live proof; the safer path is maintainer live verification followed by synchronized code, test, docs, and changelog updates if the route is truly supported.
