## Summary

Keep open for maintainer review: current main intentionally suppresses `openai-codex/gpt-5.4-mini` on the PI Codex OAuth route, and this PR removes only the manifest block without route-specific live proof or synced tests/docs. ## What This Changes

The PR removes the OpenAI plugin manifest suppression entry for `openai-codex/gpt-5.4-mini`, re-enabling that model on the Codex OAuth provider route.

## Best Possible Solution

Keep the suppression unless a maintainer can live-verify `openai-codex/gpt-5.4-mini` through the PI `openai-codex-responses` route on a ChatGPT-backed Codex OAuth account; if verified, update the manifest, resolver/provider tests, docs, and changelog together so the route contract stays consistent.
