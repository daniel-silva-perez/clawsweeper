## Summary

Keep open. Current main still blocks the reported Android onboarding state: the app can show `Connected (operator offline)` when the node session is connected and the operator session is not, but onboarding only exposes `Finish` when both sessions are connected. The open PR #61269 is a focused implementation candidate and uses closing syntax for this issue, so this should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review #61269 or land an equivalent Android app fix. The likely product path is to change `apps/android/app/src/main/java/ai/openclaw/app/ui/OnboardingFlow.kt` so a healthy node-connected/operator-offline state can complete onboarding, update the final summary copy/status accordingly, and revise `apps/android/app/src/test/java/ai/openclaw/app/ui/OnboardingFlowLogicTest.kt` to allow that state while still blocking genuinely disconnected or no-node states.
