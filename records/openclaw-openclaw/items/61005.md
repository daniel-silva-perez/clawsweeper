## Summary

Keep open. Current main still blocks Android onboarding completion in the reported `Connected (operator offline)` state, and open PR #61269 is a focused closing-syntax implementation candidate that has not merged.

## Best Possible Solution

Keep this issue open and review or replace PR #61269. The likely fix belongs in `apps/android/app/src/main/java/ai/openclaw/app/ui/OnboardingFlow.kt`: allow onboarding completion when the node session is connected even if the operator session is offline, keep truly disconnected/no-node states blocked, and update `apps/android/app/src/test/java/ai/openclaw/app/ui/OnboardingFlowLogicTest.kt` to lock in that contract.
