## Summary

Keep open. Current main still has the Nostr setup-status lazy-import problem, and this PR is a focused implementation that removes the problematic delegated status path without changing the runtime channel behavior; it needs normal maintainer review and validation rather than closure or a replacement repair job.

## What This Changes

This PR replaces the Nostr setup-only delegated wizard proxy with a local lightweight setup-status wizard, adds a regression test that fails if `setup-surface.js` loads during status checks, and adds a changelog fix entry.

## Best Possible Solution

Land this PR or an equivalent owner-local patch that keeps Nostr status discovery entirely inside `extensions/nostr/src/channel.setup.ts`, reserves `setup-surface.js` for actual setup/runtime work, and keeps the regression test proving status checks do not import the full setup surface.

## Reproduction Assessment

Yes. Current main has a high-confidence source reproduction path: setup status collection calls Nostr wizard status, the delegated proxy loads `setup-surface.js`, and that surface reaches `nostr-tools` through `nostr-key-utils`. The PR's new test reproduces the boundary by making `setup-surface.js` throw during status resolution.

## Solution Assessment

Yes. The PR is the narrowest maintainable fix because it changes only the Nostr setup-only status surface instead of weakening the shared delegated proxy or changing packaged dependency behavior for runtime Nostr.
