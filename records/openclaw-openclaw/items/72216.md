## Summary

Keep open. Current main still has the Nostr setup-status import chain described by the linked bug, while this PR is a narrow implementation that keeps status resolution on the lightweight setup surface and adds a regression test for that boundary.

## What This Changes

The PR replaces Nostr's setup-only delegated wizard proxy with a local lightweight setup-status wizard, adds a regression test that fails if `setup-surface.js` is imported during status checks, and adds a changelog entry for #72182.

## Best Possible Solution

Land this PR or an equivalent owner-local patch: keep Nostr setup status resolution inside `extensions/nostr/src/channel.setup.ts`, reserve the full `setup-surface.js` for real setup/runtime work, and keep the regression test proving status checks do not import the full surface.

## Reproduction Assessment

Yes. The current-main source path is high-confidence: setup status collection calls the Nostr wizard status, the delegated proxy loads `setup-surface.js`, and that surface reaches `nostr-tools` through `nostr-key-utils`.

## Solution Assessment

Yes. This is the narrowest maintainable fix because it changes only the Nostr setup-only surface and avoids weakening the shared delegated proxy behavior used by other plugins.
