## Summary

Keep open. Current main still routes Nostr setup-only status through the delegated wizard proxy, which can import the full Nostr setup surface and its `nostr-tools` dependency during status discovery. The PR is a focused owner-local fix with a regression test and no blocking review or security findings.

## What This Changes

The PR replaces Nostr's setup-only delegated wizard proxy with a local lightweight status wizard, adds a regression test that makes `setup-surface.js` throw if loaded during status checks, and adds a changelog fix entry.

## Best Possible Solution

Land this PR or an equivalent owner-local patch: keep Nostr setup status resolution inside `extensions/nostr/src/channel.setup.ts`, reserve the full `setup-surface.js` for actual setup/runtime work, and retain the regression test proving status checks do not import the full surface.
