## Summary

Keep open. Current main does not prove the reported abort-garbage path is fixed, and there is an open unmerged PR with explicit closing syntax for this issue. The current source, docs, and tests still describe and assert retention of nonblank aborted assistant partials rather than a discard gate for stale aborted output.

## Best Possible Solution

Keep this issue open while maintainers review, rebase, land, or close #48283. The best fix is an abort-path gate in the embedded agent/session delivery path that prevents stale `stopReason: "aborted"` assistant content from becoming a channel payload or transcript reply, with regression coverage for rapid same-session messages and no upstream provider request.
