## Summary

Keep this issue open. Current main still has code, tests, and docs that preserve and can surface aborted assistant partials, and #48283 remains an open implementation candidate with explicit closing syntax for this issue.

## Best Possible Solution

Keep this issue open while maintainers review, rebase, land, or close #48283. The best fix is a narrow abort-path gate in the embedded agent/session delivery flow that prevents stale stopReason="aborted" assistant content from becoming a channel payload or ordinary transcript reply, while preserving any deliberately supported partial-retention UI behavior with clear metadata and tests.
