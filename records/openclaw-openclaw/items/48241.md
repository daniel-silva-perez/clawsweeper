## Summary

Keep this issue open. Current main still preserves and can surface aborted assistant partials, and #48283 is an open, unmerged implementation candidate with explicit closing syntax for this issue. The existing source, tests, and docs do not yet prove the reported stale/garbage abort path is fixed.

## Best Possible Solution

Keep this issue open while maintainers review, rebase, land, or close #48283. The best fix is a narrowly tested abort-path gate in the embedded agent/session delivery flow that prevents stale stopReason "aborted" assistant content from becoming a channel payload or transcript reply, while preserving any intentionally supported partial-retention behavior.
