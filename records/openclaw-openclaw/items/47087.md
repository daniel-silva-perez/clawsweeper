## Summary

Keep open: current main still lacks `sessions_spawn` input provenance, and this draft PR remains the focused implementation for the open paired tracker. The diff addresses a valid gap, but it should be refreshed and validated rather than closed or replaced.

## What This Changes

The PR adds an `inputProvenance` payload with `sourceTool: "sessions_spawn"` to native subagent child `agent` calls and adds a workspace regression test for that payload.

## Best Possible Solution

Native `sessions_spawn` should consistently mark the initial child task as inter-session input before it enters the gateway and runner, using the requester session key and caller channel context while leaving existing delivery behavior unchanged and covering both routed and channel-only spawn contexts.
