## Summary

Keep open. Current main still uses synchronous filesystem APIs in the user-facing export-session command, so this PR's intended async migration is not implemented on main. The PR is focused, but the proposed patch still needs the already-reviewed cleanup for access-before-mkdir and should update the colocated tests from sync fs mocks to promise-based fs behavior.

## Best Possible Solution

Keep this PR open and ask for a small revision: rebase if needed, convert the export-session command to promise-based fs APIs, load templates with Promise.all, call recursive mkdir directly without an access preflight, remove the redundant return await, and update src/auto-reply/reply/commands-export-session.test.ts to mock and assert the async filesystem path.
