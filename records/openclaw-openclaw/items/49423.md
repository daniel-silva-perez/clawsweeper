## Summary

Keep open. Current main still uses synchronous filesystem APIs in the export-session command, so the PR’s intended async migration is not implemented on main. The PR remains focused and potentially useful, but the proposed patch needs revision for the already-reviewed access-before-mkdir TOCTOU pattern and should update the colocated tests from sync fs mocks to promise-based fs behavior.

## Best Possible Solution

Keep this PR open and ask for a small revision: convert `src/auto-reply/reply/commands-export-session.ts` to promise-based fs APIs, load the export templates with `Promise.all`, call `fs.promises.mkdir(outputDir, { recursive: true })` directly without an `access` preflight, remove the redundant `return await`, and update `src/auto-reply/reply/commands-export-session.test.ts` to mock/assert the async filesystem path.
