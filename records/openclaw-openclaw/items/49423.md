## Summary

Keep this PR open. Current main still uses synchronous filesystem APIs in the user-facing export-session command, and the colocated tests still mock the sync fs surface. The PR is a focused implementation candidate, but the existing review notes still need to be addressed before merge, especially the unnecessary access-before-recursive-mkdir pattern.

## Best Possible Solution

Keep this PR open for a small revision: convert the command to promise-based fs APIs, load templates with `Promise.all`, call recursive `mkdir` directly without a prior `access` check, remove redundant `return await`, and update the colocated tests to exercise the async fs path before normal maintainer review.
