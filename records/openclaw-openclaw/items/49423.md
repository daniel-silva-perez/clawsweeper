## Summary

Keep this PR open. Current main still uses synchronous filesystem APIs in the user-facing export-session command, so the PR's intended async cleanup is not implemented. The PR remains focused, but existing review comments identify changes needed before merge: remove the redundant access-before-recursive-mkdir pattern, remove redundant return await, and update tests that still mock/assert sync fs behavior.

## Best Possible Solution

Keep this PR open for a small revision: convert the command to promise-based fs APIs, load templates with Promise.all, call recursive mkdir directly without a prior access check, remove redundant return await, and update the colocated tests to exercise async fs behavior before normal maintainer review.
