## Summary

Keep this PR open. Current main still uses synchronous filesystem APIs in the user-facing `export-session` command, and the PR is open, unmerged, and dirty. The patch is focused, but it still needs the existing review cleanup for the `access`-before-`mkdir` TOCTOU pattern and test updates for the async fs path.

## Best Possible Solution

Keep this PR open and ask for a small revision: rebase, convert `export-session` to promise-based fs APIs, load templates with `Promise.all`, call `fs.promises.mkdir(outputDir, { recursive: true })` directly, remove the redundant `return await`, and update the colocated tests to exercise the async filesystem path.
