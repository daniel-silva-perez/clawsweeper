## Summary

Keep this PR open. Current main still uses synchronous filesystem APIs in the user-facing export-session command, and the colocated tests still mock/assert sync fs behavior, so the PR's intended async cleanup is not implemented on main. The PR is scoped to the right file, but the existing review comments still need a small revision before merge.

## Best Possible Solution

Keep this PR open for a small revision: convert the command to promise-based fs APIs, load templates with `Promise.all`, call recursive `mkdir` directly without a prior `access` check, remove redundant `return await`, and update the colocated tests to exercise the async fs path before normal maintainer review.
