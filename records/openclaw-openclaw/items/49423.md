## Summary

Keep open. Current main still uses synchronous filesystem APIs in the user-facing export-session command, so this PR's intended async migration is not implemented on main. The PR is focused, but it still needs the existing review cleanup for the access-before-mkdir pattern and test updates for the async fs path.

## Best Possible Solution

Keep this PR open and ask for a small revision: rebase, convert the export-session implementation to promise-based fs APIs, load templates with Promise.all, call recursive mkdir directly without an access preflight, remove the redundant return await, and update the colocated tests to exercise the async filesystem path.
