## Summary

Current main still does not implement source-based or wrapper-aware filtering for memory-lancedb autoCapture. The requested config key is absent from the typed parser and manifest schema, and the agent_end path still sends extracted user text directly through shouldCapture before embedding/storing memories.

## Best Possible Solution

Keep this open. Implement a backward-compatible memory-lancedb option such as plugins.entries.memory-lancedb.config.captureFilter.excludeWrapped, expose it in the typed config parser, manifest schema, UI hints, and docs, then apply a robust source/wrapper-aware pre-filter before shouldCapture in the agent_end auto-capture path. Add focused regression tests proving wrapped EXTERNAL_UNTRUSTED_CONTENT with trigger words is skipped while normal direct preference capture still works. Coordinate with the broader provenance discussions in #48519 and #29442, but this narrow memory-lancedb filter remains independently actionable.
