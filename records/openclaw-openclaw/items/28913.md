## Summary

Current main still does not implement the requested source- or wrapper-aware autoCapture filter. The bundled memory-lancedb plugin accepts only the existing config keys, the manifest/docs do not expose captureFilter.excludeWrapped, and the agent_end auto-capture path still sends extracted user text through shouldCapture without checking EXTERNAL_UNTRUSTED_CONTENT boundaries first.

## Best Possible Solution

Keep this issue open and implement a backward-compatible memory-lancedb `captureFilter.excludeWrapped` option. The fix should update the plugin config parser, manifest schema/UI hints, public docs, and agent_end auto-capture path, with focused tests proving EXTERNAL_UNTRUSTED_CONTENT-wrapped trigger text is skipped while ordinary direct user preference capture still works. If robust marker detection needs shared logic, expose or reuse a small plugin-SDK security helper instead of deep-importing core internals.
