## Summary

Current main still lacks source- or wrapper-aware filtering for memory-lancedb autoCapture. The requested captureFilter/excludeWrapped config is absent from the parser, manifest schema, UI hints, docs, and agent_end auto-capture path, while EXTERNAL_UNTRUSTED_CONTENT boundaries still exist elsewhere in core.

## Best Possible Solution

Keep this open. Implement a backward-compatible memory-lancedb captureFilter.excludeWrapped option in the plugin-owned config parser, manifest schema/UI hints, docs, and agent_end auto-capture path. Add focused tests proving wrapped EXTERNAL_UNTRUSTED_CONTENT with trigger words is skipped while ordinary direct preference capture still works, and leave broader provenance/privilege-separation work to #29442 and #48519.

## Risks / Open Questions

- A naive string check could drift from the external-content marker format or miss spoofing/sanitization edge cases; the filter should reuse or mirror the current marker semantics deliberately.
- The fix needs schema, parser, manifest/UI hints, docs, and regression tests to stay aligned; adding only runtime logic would leave the option undocumented or invalid.
- This narrow filter should not be presented as a complete provenance/privilege-separation solution; #29442 and #48519 still track broader architecture questions.
