## Summary

Keep open. Current main still preserves single soft newlines through Telegram Markdown rendering and outbound delivery, while Telegram config, docs, and UI hints do not expose a collapse-newlines option. The request is valid and narrow, but related newline-preservation reports mean the fix should stay Telegram-owned and conservative.

## What This Changes

Request to collapse single mid-paragraph newlines before Telegram delivery while preserving intentional paragraph breaks, with a possible per-channel config option.

## Best Possible Solution

Add a Telegram-owned, Markdown-aware soft-line-break normalization path, preferably behind an explicit config option, that collapses prose/list hard wraps while preserving blank lines, fenced code, blockquotes, list boundaries, escaping, chunk limits, and plain-text fallback behavior.

## Reproduction Assessment

Yes. A focused static reproduction is `markdownToTelegramHtml("Line one\nLine two")`: current source maps the Markdown softbreak to a literal newline and Telegram delivery paths send it without normalization.

## Solution Assessment

Yes, if solved as a conservative Telegram-specific option. A shared Markdown IR or global chunking change would be broader and would risk regressing newline preservation tracked by related reports.
