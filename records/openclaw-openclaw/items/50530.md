## What This Changes

The issue asks Telegram outbound delivery to collapse soft single newlines inside prose or list paragraphs while preserving intentional breaks such as blank lines, code blocks, blockquotes, and list structure.

## Best Possible Solution

Add a Telegram-owned, Markdown-aware soft-line-break normalization path, probably guarded by an explicit config option, that collapses prose/list hard wraps while preserving blank lines, fenced code, blockquotes, list boundaries, HTML escaping, and plain-text fallback behavior.
