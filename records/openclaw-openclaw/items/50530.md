## Summary

Keep open. Current main still preserves single soft newlines through Telegram Markdown rendering and both reply/direct outbound delivery paths, while Telegram config and docs do not expose a collapse-newlines option. The request is valid and narrow, but related newline-preservation reports mean the repair should be Telegram-owned and conservative rather than a shared Markdown or global chunking change.

## What This Changes

Collapse single mid-paragraph soft line breaks in Telegram delivery while preserving intentional paragraph breaks, lists, blockquotes, and code blocks, possibly behind a Telegram config option.

## Best Possible Solution

Add a Telegram-owned, Markdown-aware soft-line-break normalization option that can collapse prose/list hard wraps while preserving blank lines, fenced code, blockquotes, list boundaries, escaping, chunk limits, and plain-text fallback behavior. Keep shared Markdown IR and global chunking semantics unchanged.

## Reproduction Assessment

Yes. A focused static reproduction is `markdownToTelegramHtml("Line one\nLine two")`: current source maps the Markdown softbreak to a literal newline and sends that through Telegram delivery without normalization.

## Solution Assessment

Yes, if implemented as a conservative Telegram-specific option. A shared Markdown IR or global chunking change would be broader and would risk regressing the newline-preservation behavior tracked by related reports.
