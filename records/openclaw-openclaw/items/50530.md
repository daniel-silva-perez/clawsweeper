## Summary

Keep open. Current main still preserves single newlines through the Telegram Markdown-to-HTML delivery path, and the Telegram config/docs surface does not expose a soft-line-break collapse option. The prior keep-open ClawSweeper comments remain accurate against the current checkout.

## What This Changes

The issue requests a Telegram outbound formatting option or behavior to collapse soft single newlines inside prose/list paragraphs while preserving intentional paragraph breaks.

## Best Possible Solution

Add a Telegram-owned, Markdown-aware soft-line-break normalization path, preferably behind an explicit per-channel config option, that collapses prose/list hard wraps while preserving blank lines, fenced code, blockquotes, list boundaries, HTML escaping, and plain-text fallback behavior.
