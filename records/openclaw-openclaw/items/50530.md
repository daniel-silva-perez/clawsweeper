## Summary

Keep open. Current main still preserves single newlines through the Telegram Markdown-to-HTML delivery path, and Telegram config/docs expose chunking and formatting controls but no option that collapses soft mid-paragraph line breaks. The related newline-preservation reports make a default/global collapse risky, but an opt-in Telegram-owned fix is narrow and actionable.

## What This Changes

Add Telegram delivery behavior that collapses single mid-paragraph newlines before sending while preserving intentional double newlines and Markdown structure.

## Best Possible Solution

Add a Telegram-owned, Markdown-aware soft-line-break normalization path, preferably behind an explicit `channels.telegram.collapseNewlines`-style config field, that collapses prose/list hard wraps while preserving blank lines, fenced code, blockquotes, list boundaries, HTML escaping, plain-text fallback behavior, and existing shared chunking semantics.
