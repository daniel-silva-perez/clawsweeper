## Summary

Keep open. Current main still preserves single newlines through the Telegram Markdown-to-HTML delivery path, and Telegram config/docs expose chunking and formatting controls but no option that collapses soft mid-paragraph line breaks. The related newline-preservation reports make a default/global collapse risky, but an opt-in Telegram-owned fix is narrow and actionable.

## What This Changes

The issue requests Telegram outbound formatting that collapses soft mid-paragraph line breaks while preserving intentional paragraph breaks, possibly behind a per-channel option.

## Best Possible Solution

Add a Telegram-owned soft-line-break normalization path, preferably behind an explicit `channels.telegram.collapseNewlines`-style config field, that collapses prose/list hard wraps while preserving intentional Markdown structure and existing shared Markdown/chunking semantics.
