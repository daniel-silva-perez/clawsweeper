## Summary

Keep open. Current main still preserves soft/mid-paragraph newlines on the Telegram send path, and the existing Telegram config/docs only expose chunking and Markdown table controls, not newline-collapsing behavior.

## Best Possible Solution

Keep this issue open and implement the behavior in the bundled Telegram plugin, likely as a Markdown/IR-aware formatter option that collapses soft line wraps in prose/list text while preserving intentional blank lines, code blocks, blockquotes, hard breaks, and HTML-mode caller markup. Add focused tests in `extensions/telegram/src/format.test.ts` or adjacent send-path tests, then document any new `channels.telegram` option if it is configurable.

## Risks / Open Questions

- A naive newline collapse could damage code blocks, blockquotes, Markdown tables, hard breaks, or caller-owned `textMode: "html"` payloads.
- The existing `chunkMode="newline"` wording may be confused with this request, but it only changes chunk boundaries and does not rewrite message text.
