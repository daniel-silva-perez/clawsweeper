## Summary

Keep open. Current main still preserves single newlines through the Telegram Markdown-to-HTML reply path, and Telegram config/docs expose chunking and formatting controls but no collapse-newlines option. The related closed newline-preservation reports make a global/default collapse risky, but an opt-in Telegram-owned formatter path is narrow and actionable.

## What This Changes

The issue requests that Telegram delivery collapse soft single newlines inside prose/list paragraphs while preserving intentional breaks and structured formatting.

## Best Possible Solution

Add a Telegram-owned soft-line-break normalization pass, preferably behind an explicit `channels.telegram.collapseNewlines`-style option that defaults conservatively, and cover formatter, delivery, config schema/UI hints, docs, and user-facing changelog behavior without changing shared Markdown IR or global chunking semantics.
