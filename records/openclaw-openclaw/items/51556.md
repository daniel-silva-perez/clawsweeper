## Summary

Keep this PR open. The underlying stat-error inconsistency still exists on current main, but the proposed patch is stale against the current helper split and introduces a sync-only read-file behavior that does not match the async contract.

## What This Changes

The PR changes two catch branches in the old sync pairing allowlist reader to rethrow non-ENOENT stat and readFile errors instead of treating the allowlist as missing.

## Best Possible Solution

Refresh the work into the current shared allowlist file helper, add focused tests for non-ENOENT stat failures, and make read-file failures either fall back consistently in both sync and async paths or change the shared read contract deliberately with broader coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep sync read failures aligned with async reads:** `src/pairing/pairing-store.ts:429`

That preserves a parity bug unless both contracts change together.
