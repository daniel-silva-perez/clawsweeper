## Summary

Keep open. Current main still has the shared sync allow-from reader returning an empty allowlist for non-ENOENT stat/read errors, so this PR's intended behavior is not implemented. The patch is also stale against the current helper layout, and the readFileSync half needs contract review because the async read helper still falls back on non-ENOENT read failures.

## Best Possible Solution

Keep this PR open and refresh it against `src/pairing/allow-from-store-file.ts`, coordinating with related PR #63324. Add targeted regression tests for non-ENOENT `stat` failures, then make an explicit contract decision for read-file failures: either align both async and sync paths to throw, or narrow the fix to stat failures and keep read errors fallback-compatible.
