## Summary

Keep open. Current main still renders the chat delete confirmation popover as an absolutely positioned element above the delete button with no viewport flip or clamp logic, and the latest release tag has the same placement. The prior related fix PRs are closed unmerged, while the only open related PR in the supplied context addresses listener cleanup rather than this overflow behavior.

## What This Changes

The issue asks for the chat delete confirmation dialog to stay within the visible page bounds when deleting a message, with a separate request for broader Chinese localization.

## Best Possible Solution

Add viewport-aware placement for the Control UI chat delete confirmation: measure the trigger and popover, choose above or below based on available space, clamp horizontally within the viewport, preserve skip-confirm/cancel/delete/outside-click behavior, and cover those edge cases in focused grouped-render tests. Keep the broader Chinese localization request separate from this bug fix.
