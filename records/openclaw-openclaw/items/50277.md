## Summary

Keep open. Current main still absolutely positions the chat delete confirmation popover above the trigger without viewport measurement or flip/clamp logic, latest release v2026.4.26 contains the same placement, and the related overflow-fix PRs are closed unmerged. The open related PR only addresses listener cleanup, so this remains a narrow Control UI bug fix candidate.

## What This Changes

The issue asks for the chat delete confirmation popover to stay within the visible page so the Delete button remains reachable when deleting messages near the top or edge of the chat view; the localization suggestion is separate.

## Best Possible Solution

Keep this open and implement a narrow Control UI fix: position the chat delete confirmation relative to the viewport, clamp it horizontally and vertically, flip below the trigger when there is not enough room above, preserve delete/cancel/outside-click and skip-confirm behavior, clean up dismissal listeners on every path, and add focused grouped-render tests. Treat the localization request as a separate i18n/product item.
