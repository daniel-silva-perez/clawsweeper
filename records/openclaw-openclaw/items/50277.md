## Repair Work Prompt

Fix the Control UI chat delete confirmation overflow reported in https://github.com/openclaw/openclaw/issues/50277. Current main renders `.chat-delete-confirm` above the delete button using CSS in `ui/src/styles/chat/grouped.css` and creates it in `ui/src/ui/chat/grouped-render.ts` without measuring viewport bounds. Add viewport-aware placement for this popover: measure the trigger and popover after insertion, choose above or below based on available space, clamp horizontally within the viewport, and preserve the existing skip-confirm, cancel, delete, and outside-click behavior. Add focused coverage in `ui/src/ui/chat/grouped-render.test.ts` for top-edge overflow, bottom-edge placement, and narrow-width horizontal clamping. Treat #50316 as useful prior implementation context, but implement against current main. Keep the broader Chinese localization suggestion out of scope and do not hand-edit generated locale bundles. Coordinate with #69982 if it lands first because it touches the same render helper for listener cleanup. Changelog is likely not needed unless maintainers want user-facing UI bugfix notes for this release cycle.
