## Summary

Keep open. Current main still has the reported Control UI overflow path: the delete confirmation popover is absolutely positioned above the trigger with no viewport-aware flip or clamp logic, and the latest release tag contains the same relevant implementation.

## What This Changes

The issue reports that the chat delete confirmation popover can render outside the page boundary, preventing the user from clicking the delete button.

## Best Possible Solution

Add viewport-aware placement for the Control UI chat delete confirmation: measure the trigger and popover, choose above or below based on available space, clamp horizontally within the viewport, preserve skip-confirm/cancel/delete/outside-click behavior, and cover edge cases in focused grouped-render tests. Keep the broader Chinese localization suggestion separate.

Fix the Control UI chat delete confirmation overflow reported in https://github.com/openclaw/openclaw/issues/50277. The popover currently lives in `ui/src/ui/chat/grouped-render.ts` and `ui/src/styles/chat/grouped.css` and is anchored above the delete button with no viewport-aware placement. Add a narrow placement fix that measures the trigger/popover, flips above or below based on available vertical space, clamps horizontally within the viewport, and preserves skip-confirm localStorage, cancel, delete, and outside-click behavior.
