## Summary

Keep open: current main still has the reported Control UI delete-confirmation placement problem, and the latest release tag has the same relevant implementation. The prior fix PRs for this overflow were closed unmerged, while the remaining open related PR covers listener cleanup rather than viewport-aware placement.

## What This Changes

The issue reports that the chat delete confirmation popover can extend outside the visible chat page, making the delete action unreachable.

## Best Possible Solution

Add viewport-aware placement for the Control UI chat delete confirmation: measure the trigger and popover after insertion, choose above or below based on available space, clamp horizontally within the viewport, preserve skip-confirm/cancel/delete/outside-click behavior, and cover these cases in focused grouped-render tests. Keep localization work separate from this bug fix.

## Reproduction Assessment

Yes. The report's steps map directly to current main: clicking a chat delete button creates an absolutely positioned popover above the trigger with no viewport measurement or flip/clamp logic, which is enough to reproduce the edge-placement failure in a top-edge or clipped chat-thread scenario.

## Solution Assessment

Yes. A narrow viewport-aware placement fix in the grouped chat delete popover is the best path; the reporter's workaround of removing absolute positioning may avoid the overflow but would put the confirmation back into document flow and risk shifting chat layout.
