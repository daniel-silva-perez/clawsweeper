## Summary

Keep open. The reported overflow is still present on current main and in the latest release: the delete confirmation is absolutely positioned above the trigger, the render path has no viewport flip or clamp logic, and current tests only cover left/right anchoring. The related overflow-fix PRs are closed unmerged; the open related PR addresses listener cleanup, not placement.

## What This Changes

The issue asks for the chat delete confirmation popover to stay within the page bounds when deleting a chat message, with a separate broader request for Chinese localization.

## Best Possible Solution

Implement viewport-aware placement in Control UI chat: measure the trigger and popover, flip above or below based on available space, clamp horizontally and vertically, preserve delete/cancel/outside-click and skip-confirm behavior, clean up dismissal listeners on every path, and add focused grouped-render tests. Treat broader Chinese localization as a separate i18n/product item.
