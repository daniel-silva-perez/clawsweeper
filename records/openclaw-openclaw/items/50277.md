## Repair Work Prompt

Fix the Control UI chat delete confirmation overflow reported in https://github.com/openclaw/openclaw/issues/50277. Current main creates `.chat-delete-confirm` inside `.chat-delete-wrap` and CSS anchors it above the delete button with `bottom: calc(100% + 6px)`, so when the trigger is near the top edge the confirmation can leave the viewport and the Delete button becomes unreachable. Implement viewport-aware placement in the grouped chat delete path: measure the trigger and popover after insertion, place above or below depending on available space, clamp horizontally within the viewport, and preserve existing skip-confirm, Cancel, Delete, and outside-click behavior. Treat #50316 as useful prior context, but write a clean current-main patch. Also be aware that #69982 touches the same helper for listener cleanup; do not regress listener cleanup if it has landed. Likely files: `ui/src/ui/chat/grouped-render.ts`, `ui/src/styles/chat/grouped.css`, and `ui/src/ui/chat/grouped-render.test.ts`. Add focused tests for top-edge flip/bottom placement and horizontal clamping; avoid broad Control UI rewrites and keep the broader Chinese localization suggestion out of scope. Changelog is probably not needed for this narrow UI bug unless maintainers request one. Validate with `pnpm test ui/src/ui/chat/grouped-render.test.ts`, `pnpm exec oxfmt --check --threads=1 ui/src/ui/chat/grouped-render.ts ui/src/styles/chat/grouped.css ui/src/ui/chat/grouped-render.test.ts`, a browser/DOM geometry smoke for top-edge, bottom-edge, and narrow-width placement, and `pnpm check:changed` via Testbox before handoff.
