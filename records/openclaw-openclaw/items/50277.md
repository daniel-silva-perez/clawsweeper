## Summary

Keep open. Current main and the latest release still anchor the chat delete confirmation popover above the trigger with absolute positioning and no viewport flip or clamp logic, so the reported edge overflow remains plausible from source. The existing focused test only checks left/right side classes, the prior overflow-fix PRs are closed unmerged, and the only open related PR is about listener cleanup rather than placement.

## What This Changes

The issue requests fixing the Control UI chat delete confirmation popover so it stays within the visible page bounds when deleting a message or conversation near an edge.

## Best Possible Solution

Add viewport-aware placement for the Control UI chat delete confirmation: measure the trigger and popover, choose above or below based on available space, clamp within the viewport, preserve skip-confirm/cancel/delete/outside-click behavior, and cover the edge cases in focused grouped-render tests. Keep the broader Chinese localization request as a separate i18n/product discussion.
