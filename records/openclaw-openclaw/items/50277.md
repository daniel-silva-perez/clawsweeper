## Summary

Keep this issue open. Current main still renders the chat delete confirmation as an absolutely positioned popover above the delete button, which matches the reported failure mode near the top of the chat viewport. The related fix PRs for this issue are all closed and unmerged, and the latest release tag still contains the same implementation.

## Best Possible Solution

Keep the issue open and implement the focused UI fix in the chat renderer/styles: position the delete confirmation with viewport-aware coordinates, flip it below the trigger when there is not enough space above, clamp horizontally, and add regression coverage for near-edge placement in the grouped chat tests. Treat the broader localization suggestion as separate follow-up work if maintainers want to pursue it.
