## Summary

Current main still does not implement the requested iMessage rich text spans, text-size attributes, underline/strikethrough, or Shake/Nod/Explode/Ripple/Bloom/Jitter-style text animations. The legacy iMessage path remains plain `imsg` text delivery, while BlueBubbles has only a narrower classic `effectId` send path and strips Markdown on normal text sends; the prior maintainer-side comment in the issue is still accurate.

## What This Changes

Request support for rich iMessage formatting and message/text effects instead of sending only plain text through the iMessage channel.

## Best Possible Solution

Define a typed iMessage/BlueBubbles outbound capability model for supported style spans, text sizes, underline/strikethrough, and effect/animation enums; implement only the upstream-backed subset per transport, and document any legacy `imsg` limitations in the public channel docs.
