## Summary

Keep this issue open. Current main still does not implement iMessage rich text spans, text sizing, or the requested newer text animations. Legacy iMessage still sends a plain `text` JSON-RPC payload through `imsg`; the outbound adapter is still plain-text chunking; BlueBubbles has a narrower `effectId` action path but strips Markdown before sending and does not carry rich style spans or text-size attributes. The existing maintainer-side Codex review in the discussion already kept this open, and current main still matches that assessment.

## Best Possible Solution

Keep this issue open for maintainer product/API direction. The best path is a typed iMessage/BlueBubbles rich-text and text-effect send surface covering supported style spans, text size, and supported effect/animation enums, wired through outbound delivery and message actions with tests and public docs. Legacy `imsg` should either gain upstream-supported capability handling or explicitly document these attributes as unsupported.
