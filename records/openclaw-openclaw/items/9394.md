## Summary

Keep this issue open. Current main still does not provide iMessage rich text spans, text-size attributes, underline/strikethrough, or the requested Shake/Nod/Explode/Ripple/Bloom/Jitter-style animations. Legacy iMessage still sends a plain `text` JSON-RPC payload through `imsg`, while BlueBubbles has a narrower `effectId` path for classic message effects but strips Markdown on normal text sends.

## What This Changes

Request support for rich text styling, text size variations, and native iMessage effects/animations in the iMessage channel instead of plain-text-only sends.

## Best Possible Solution

Keep this open for maintainer product/API direction. The best implementation would define a typed iMessage/BlueBubbles send surface for supported style spans, underline/strikethrough, text size, and supported effect/animation enums, wire it through outbound delivery and message actions, add focused tests/docs, and explicitly document unsupported legacy `imsg` behavior if upstream cannot send those attributes.
