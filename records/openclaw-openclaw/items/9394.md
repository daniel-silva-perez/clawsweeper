## Summary

Keep this issue open. Current main still does not implement rich iMessage style spans, text-size attributes, underline/strikethrough, or the requested Shake/Nod/Explode/Ripple/Bloom/Jitter text animations; the legacy imsg path remains plain-text delivery, while BlueBubbles only exposes a narrower effectId action and strips Markdown on normal sends. The existing maintainer-side review in the discussion is still accurate against the current main SHA.

## What This Changes

The issue requests rich iMessage outbound formatting, text-size attributes, underline/strikethrough, and message/text animation effects beyond plain text delivery.

## Best Possible Solution

Define a typed outbound capability model for iMessage/BlueBubbles formatting and effects, implement only the upstream-backed subset per transport, preserve plain-text fallback behavior where rich formatting is unsupported, and document legacy imsg limitations on the public channel docs.
