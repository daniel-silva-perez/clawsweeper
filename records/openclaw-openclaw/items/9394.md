## Summary

Keep this issue open. Current main still lacks outbound rich iMessage style spans, text-size attributes, underline/strikethrough handling, and the requested Shake/Nod/Explode/Ripple/Bloom/Jitter animation support. Legacy iMessage remains a plain imsg text send path, while BlueBubbles only exposes a narrower effectId action and strips Markdown during normal sends; the issue discussion also already keeps this open for maintainer product/API follow-up.

## What This Changes

Request iMessage outbound support for rich text styling, text-size changes, and Apple message effects such as Shake, Nod, Explode, Ripple, Bloom, and Jitter.

## Best Possible Solution

Define a typed outbound capability model for iMessage and BlueBubbles formatting/effects, implement only the upstream-backed subset per transport, preserve plain-text fallback where rich formatting is unsupported, and document legacy imsg limitations in the public channel docs.
