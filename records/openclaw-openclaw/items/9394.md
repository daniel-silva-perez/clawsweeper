## Summary

Keep this issue open. Current main still lacks outbound rich text/style spans, text-size attributes, underline/strikethrough handling, and the requested newer iMessage text animations; the recent maintainer-side keep-open comments remain accurate against the current tree.

## What This Changes

The issue requests outbound iMessage support for rich text formatting, text-size variations, and newer message effects such as Shake, Nod, Explode, Ripple, Bloom, and Jitter.

## Best Possible Solution

Define a typed outbound capability model for iMessage transports, implement only the upstream-backed rich formatting and effect subsets per transport, preserve deterministic plain-text fallback where unsupported, and document legacy imsg limitations alongside BlueBubbles capabilities.

## Reproduction Assessment

Yes for the observable current gap: source and focused tests show legacy imsg sends plain text/file/reply params, while BlueBubbles strips Markdown and only carries an optional effectId string. A live transport smoke would still be needed before choosing exact rich-attribute serialization.

## Solution Assessment

Unclear as an implementation question because there is no proposed patch and one-off imsg flags would be brittle. The maintainable direction is a capability-gated transport model with provider-specific fallbacks and docs.
