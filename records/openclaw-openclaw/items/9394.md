## Summary

Keep this issue open. Current main still treats legacy iMessage as plain text/media/reply delivery, iMessage is still excluded from the shared Markdown IR renderers, and BlueBubbles only has narrow classic effectId support while stripping Markdown before send. The requested rich style spans, text sizing, and Shake/Nod/Explode/Ripple/Bloom/Jitter text animations remain unimplemented and need maintainer product/API direction.

## Best Possible Solution

Keep this issue open for maintainer product/API direction. The likely path is a typed iMessage/BlueBubbles rich-text and text-effect send surface covering supported style spans, text size, and effect/animation enums, wired through outbound delivery and message actions with tests and public docs. Legacy imsg should either gain upstream-supported capability handling or document these attributes as unsupported.
