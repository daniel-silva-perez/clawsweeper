## Summary

Keep this issue open. Current main still lacks outbound rich iMessage style spans, text-size attributes, underline/strikethrough, and the requested Shake/Nod/Explode/Ripple/Bloom/Jitter text animations; legacy iMessage remains a plain `imsg` text send path, while BlueBubbles strips Markdown on normal sends and only exposes a narrower `effectId` action for classic effects.

## What This Changes

The issue requests rich text formatting, text size attributes, and iMessage text effects for outbound iMessage messages.

## Best Possible Solution

Define a typed outbound capability model for iMessage and BlueBubbles formatting/effects, implement only the upstream-backed subset per transport, preserve plain-text fallback behavior where rich formatting is unsupported, and document legacy `imsg` limitations in the public channel docs.
