## Summary

Keep this issue open. Current main still does not implement iMessage rich text spans, text sizing, underline/strike/native style attributes, or the requested newer text animations. Legacy iMessage still sends only a plain `text` JSON-RPC payload through `imsg`; the outbound adapter is wired as plain text; BlueBubbles has a narrow `effectId` path for classic effects, but normal sends strip Markdown and do not carry rich style spans or text-size attributes. The prior maintainer-side Codex review in the discussion remains accurate on current main, and I did not find an open PR that specifically implements this issue.

## Best Possible Solution

Keep this issue open for maintainer product/API direction. The best path is a typed iMessage/BlueBubbles rich-text and text-effect send surface covering supported style spans, underline/strikethrough, text size, and supported effect/animation enums, wired through outbound delivery and message actions with tests and public docs. Legacy `imsg` should either gain upstream-supported capability handling or explicitly document these attributes as unsupported; coordinate with #24943 so plain-text fallback behavior remains clear.
