## Summary

Keep this issue open. Current main still lacks outbound rich iMessage style spans, text-size attributes, underline/strikethrough handling, and the requested Shake/Nod/Explode/Ripple/Bloom/Jitter animation support; legacy iMessage remains a plain imsg text send path, and BlueBubbles only exposes a narrower effectId action while stripping Markdown on normal sends. The existing issue discussion also already keeps this open for maintainer product/API follow-up.

## What This Changes

Request support for iMessage outbound rich text formatting, text-size variation, and Messages-style text animations/effects.

## Best Possible Solution

Define a typed outbound capability model for iMessage and BlueBubbles formatting/effects, implement only the upstream-backed subset per transport, preserve plain-text fallback where rich formatting is unsupported, and document legacy imsg limitations in the public channel docs.
