## Summary

Keep this PR open. Current main still does not implement `<notts>...</notts>` visual-only TTS behavior: the active speech runtime only understands bracketed `[[tts...]]` directives, and this PR is not merge-ready because it targets stale core TTS files while bundling unrelated, security-sensitive gateway authorization changes.

## Best Possible Solution

Keep this PR open for focused maintainer or author rework, or replace it with a smaller PR. The best implementation should target `extensions/speech-core/src/tts.ts`, define whether `<notts>` belongs in the directive contract in `src/tts/directives.ts`, add focused runtime/contract tests for visible-vs-spoken output and tagged-mode behavior, update `docs/tools/tts.md`, and split the unrelated gateway/macOS/media/script changes into separate PRs or drop them.
