## Summary

Keep this PR open. Current main still does not implement `<notts>...</notts>` visual-only TTS behavior, and the submitted branch is not merge-ready because it targets stale core TTS files and bundles unrelated, security-sensitive gateway authorization changes with the TTS feature.

## Best Possible Solution

Keep this PR open for maintainer or author rework, or replace it with a focused PR. The best implementation should target `extensions/speech-core/src/tts.ts`, update `src/tts/directives.ts` only if the syntax becomes part of the directive contract, add focused contract/runtime tests for visible-vs-spoken output and tagged-mode behavior, and update `docs/tools/tts.md`. Split or drop the unrelated gateway authorization, macOS speech, media conversion, MIME, agent-media, and helper-script changes.
