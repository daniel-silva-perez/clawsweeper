## Summary

Keep this PR open. Current main still lacks `<notts>...</notts>` visual-only TTS behavior, but the branch is not merge-ready: it is dirty against current main, targets stale in-core TTS files instead of the active speech-core runtime, and mixes the TTS feature with unrelated gateway authorization changes that need separate security review.

## What This Changes

The PR adds HTML-style `<notts>/<tts>` TTS preprocessing docs/tests and a helper script, while also changing gateway node-role authorization, macOS speech recognition, media conversion, MIME handling, and agent audio delivery plumbing.

## Best Possible Solution

Keep the useful visual-only TTS idea, but land it as a focused change against `extensions/speech-core/src/tts.ts` and the directive/contract surfaces current main actually uses. Add focused tests for visible-vs-spoken output, tagged-mode triggering, and disable-flag behavior; update `docs/tools/tts.md`; split or drop the unrelated gateway, macOS, media, MIME, agent-media, and helper-script changes.

- **[P1] Remove the node-role privilege expansion:** `src/gateway/method-scopes.ts:32-47`
  - body: This TTS PR makes node-role clients authorized for chat, config, session, status, talk, and TTS RPC methods. That broadens gateway authorization for paired node/mobile tokens and is unrelated to visual-only TTS, so it should be split into a dedicated gateway-auth change with tests and owner review.
  - confidence: 0.91
- **[P2] Move `<notts>` handling into speech-core:** `src/tts/tts.ts:39-40`
  - body: Current main's `src/tts/tts.ts` is only a lazy facade; outbound TTS is implemented in `extensions/speech-core/src/tts.ts`. This branch still changes the older in-core runtime, so after rebasing it will not update the active path that handles replies.
  - confidence: 0.86
- **[P2] Mirror `<notts>` filtering in the helper script:** `scripts/tts-filter.sh:24-40`
  - body: The added script only strips or unwraps `<tts>`. A message containing `<notts>...</notts>` would print raw tags in text output and leave visual-only content in `VOICE_CLEAN`, so that content would still be spoken whenever the helper is used.
  - confidence: 0.92
- **[P2] Treat `<notts>` as tagged-mode TTS intent:** `src/tts/tts.ts:871-873`
  - body: Tagged auto mode only checks `<tts>` in addition to bracket directives. A reply using only `<notts>` to hide a table from speech returns before synthesis, even though the remaining text is the intended spoken content.
  - confidence: 0.86
- **[P2] Honor `processTtsTags` when triggering tagged mode:** `src/tts/tts.ts:871-873`
  - body: The added regex forces tagged-mode audio for literal `<tts>` even when `messages.tts.processTtsTags` is false. In that configuration the tags are supposed to be treated as ordinary text, so the gate should respect the resolved setting.
  - confidence: 0.79
