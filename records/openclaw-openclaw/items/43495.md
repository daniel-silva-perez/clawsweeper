Current main still lacks the central `<notts>` visual-only TTS behavior in the active speech-core runtime, while the PR is broad, stale against the pluginized TTS path, incomplete in its added shell filter/tagged-mode behavior, and includes an unrelated gateway authorization expansion.

## What This Changes

## Best Possible Solution

Implement `<notts>` as a narrow current-main change in `extensions/speech-core`, with focused runtime/docs/tests for visible/spoken splitting, tagged auto-mode behavior, and disabled tag processing; split gateway authorization, macOS speech, media conversion, MIME, and tool-result audio work into separately reviewed changes.
