## Summary

Keep open for maintainer review. The requested `<notts>` behavior is still absent from current `main`, but this PR is not mergeable as-is because it targets the stale pre-`speech-core` TTS runtime and bundles a security-sensitive gateway role-policy change with the TTS feature.

## What This Changes

The branch adds visual-only TTS tag preprocessing, docs, tests, and a shell filter, while also changing `audioAsVoice` propagation, gateway node-role authorization, macOS speech recognition setup, whisper-cli transcoding, and audio MIME mapping.

## Best Possible Solution

Split and rebase the useful TTS feature into `extensions/speech-core/src/tts.ts` with focused speech-core tests and `docs/tools/tts.md` updates, drop side changes already present on main, and handle any node-role UI access expansion as a separate security-reviewed gateway design.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Rebase preprocessing onto speech-core:** `src/tts/tts.ts:42`
  - body: Current main routes TTS through the `speech-core` runtime via the plugin-sdk facade. Adding preprocessing to the old `src/tts` runtime surface means the active auto-TTS path will not process `<notts>` after rebase.
  - confidence: 0.9
- **[P1] Keep node role from bypassing operator scopes:** `src/gateway/role-policy.ts:21-23`
  - body: The PR allows node-role clients to call additional UI/read/write methods, but gateway authorization returns before operator-scope checks for role `node`. That grants paired node clients scope-free access to methods such as `chat.send` and `config.get`.
  - confidence: 0.88
- **[P2] Handle notts in the shell filter:** `scripts/tts-filter.sh:22-44`
  - body: The added shell filter only strips or unwraps `<tts>` tags. A `<notts>...</notts>` block would show raw tags in text output and be spoken in voice output, contradicting the visual-only contract.
  - confidence: 0.92
- **[P2] Respect disabled tag processing in tagged mode:** `src/tts/tts.ts:869-874`
  - body: In tagged auto mode, tag-based triggering must honor the resolved `processTtsTags` setting. Otherwise literal `<tts>` markup can force audio generation even when tag processing is disabled.
  - confidence: 0.84
