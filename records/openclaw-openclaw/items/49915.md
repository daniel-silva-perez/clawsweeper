## Summary

Keep open. Current main already handles the Feishu parsing/mention-stripping side of the report, but the shared command authorization path still denies Feishu group slash commands when `commands.useAccessGroups=true` and no command allowlist is configured. PR #49958 is still open and explicitly references this issue as its closing target, so the issue should remain open until maintainers review and merge or close that implementation candidate.

## Best Possible Solution

Keep this issue open while maintainers review PR #49958 or an equivalent fix. The likely path is to decide the intended semantics for `commands.useAccessGroups=true` when no command allowlist is configured, update `src/channels/command-gating.ts`, and add/adjust Feishu regression coverage for group slash commands without `allowFrom`.
