## Summary

Current main preserves Feishu slash text and strips bot mentions for command probing, but the group command authorization path still denies commands when `commands.useAccessGroups=true` and no command allowlist is configured. An open, unmerged PR already references this issue with closing syntax, so the issue should stay open until maintainers review and merge or close that implementation candidate.

## Best Possible Solution

Keep this issue open while maintainers review PR #49958 or its forked counterpart. The concrete fix path is to decide the intended access-group semantics for unconfigured command allowlists, update `src/channels/command-gating.ts`, and add or adjust Feishu regression coverage for group slash commands without `allowFrom`.
