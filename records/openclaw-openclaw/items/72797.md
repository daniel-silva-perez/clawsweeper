## Summary

Keep open. The request is plausible, but current main does not already implement it and the proposed patch still opens a host-exec environment injection path after reserved-key sanitization, so it needs maintainer design and security review before merge.

## What This Changes

The PR adds a typed `resolve_exec_env` plugin hook, exposes it through the hook runner, calls it from `exec` environment construction, and adds focused hook tests.

## Best Possible Solution

Revise the hook into an additive, documented exec-environment seam that preserves the existing host env security policy, defines whether any plugin can set reserved keys or only trusted/bundled code can, carries the necessary sender/account/thread context, and has focused coverage for gateway, node, and sandbox execution. If maintainers decide env injection is too broad, use a narrower tool-execution metadata or identity hook that also addresses #73343.

## Reproduction Assessment

Not applicable as a feature PR, but there is a high-confidence inspection path: current main lacks the hook, and the PR patch visibly merges hook output after host env sanitization. No runtime reproduction is needed to see that the proposed diff can bypass the existing reserved-key policy.

## Solution Assessment

No, not as-is. The hook direction is plausible, but the safest maintainable solution must filter or gate reserved env keys and align the API with the related sender/chat identity work before exposing it as public SDK surface.

## Security Review

Status: needs_attention

Summary: The PR introduces a security-sensitive plugin env injection path that can bypass current host exec reserved-key filtering.

Concerns:

- **[medium] Hook env can bypass host exec reserved-key filtering:** `src/agents/bash-tools.exec.ts:1627`
  - body: Because plugin-provided env values are assigned after host env sanitization, the proposed seam can reintroduce keys the current policy rejects for host execution, including `PATH`, loader-style variables, proxy/TLS overrides, and secret-bearing env names. That changes the documented exec security boundary and should be gated or filtered before merge.
