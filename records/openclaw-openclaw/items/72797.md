## Summary

Keep this PR open. Current main at ab5c8025c9d0a31873d311ae3e73f3664b0a1185 still does not expose or run a `resolve_exec_env` plugin hook, and the proposed change remains security-sensitive because it would let plugin-provided environment values merge into exec after current host/sandbox env construction.

## Best Possible Solution

Keep this PR open for maintainer review. If maintainers accept the seam, land it as a typed plugin SDK hook with explicit gateway/sandbox/node semantics, channel/account/thread context, SDK/docs/API export coverage, focused tests, and a clear reserved-key or trusted-plugin policy. If env injection is too broad, redirect the work toward a narrower exec/tool hook design rather than closing it as obsolete.
