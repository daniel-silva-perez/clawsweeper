## Summary

Keep open. Current main at 43da08979035503401a21a06a55e66920411ad67 still does not expose `resolve_exec_env`, has no `runResolveExecEnv` runner, and sends exec environments into gateway/node/sandbox execution without plugin env injection. The PR remains plausible plugin SDK work, but it touches a security-sensitive exec env path and still needs maintainer policy review for reserved env keys and host/sandbox/node semantics.

## Best Possible Solution

Keep this PR open for maintainer review. If maintainers accept the seam, land it as a typed plugin SDK hook with explicit host/sandbox/node semantics, channel/account/thread context, docs and API export coverage, focused tests, and a clear reserved-key or trusted-plugin policy. If env injection is too broad for the core SDK, redirect the PR toward a narrower exec/tool hook design rather than closing it as obsolete.
