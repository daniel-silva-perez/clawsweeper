## Summary

Keep this PR open. Current main still pads code-rendered Markdown tables with `cell.text.length`, so CJK and grapheme display width is not implemented in the shared renderer used by Telegram and Discord table conversion. The draft PR remains a relevant implementation candidate for the linked table-alignment issue. Security pass: the provided diff only touches the Markdown renderer and its tests, with no CI, dependency, lockfile, install, release, or secret-handling changes.

## Best Possible Solution

Keep the PR open or replace it with a current branch that updates the shared `renderTableAsCode` width calculation to use display-width-aware cell measurement, adds CJK and grapheme regression tests, and verifies the Telegram and Discord send paths continue to use that shared renderer. That would resolve the linked CJK table-alignment bug while keeping the fix scoped to Markdown formatting.
