## Summary

Keep open. Current main still computes code-mode Markdown table widths with `String.length`, and Telegram/Discord routes still depend on that shared table renderer, so the reported CJK alignment problem is not implemented on main. The PR is a focused draft with relevant CJK/emoji/symbol tests and I did not find a definite blocking correctness issue in the proposed diff, but it still needs normal maintainer handling and validation before merge.

## What This Changes

The PR adds display-width-aware padding for Markdown code-mode table rendering in `src/markdown/ir.ts` and extends `src/markdown/ir.table-code.test.ts` with CJK, emoji, and symbol-width cases.

## Best Possible Solution

Land a refreshed version of this narrow fix that measures code-mode table padding by display width at the shared Markdown IR boundary, keeps Telegram and Discord using the existing table-mode flow, and covers CJK plus emoji edge cases with targeted Markdown table tests.
