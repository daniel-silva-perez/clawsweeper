## Summary

Keep this PR open. Current main still has the browser-reachable slash-command path importing `commands-registry.shared.ts`, which still statically imports `thinking.js` and therefore reaches provider-thinking runtime code. The session chat-type portion of the PR is now obsolete because main solved that path differently, but the central thinking-level import-boundary fix is still real and useful if narrowed.

## Best Possible Solution

Keep the PR open, but ask for a rebase/narrowing: retain only the `commands-registry.shared.ts` / `commands-registry.data.ts` dependency-injection change using `thinking.shared.ts` as the shared fallback, add a focused import-boundary regression proving `ui/src/ui/chat/slash-commands.ts` does not pull `src/auto-reply/thinking.js` or provider runtime, and drop the stale `send-policy.ts`, `session-chat-type.ts`, and `session-key-utils.ts` session-chat-type changes.
