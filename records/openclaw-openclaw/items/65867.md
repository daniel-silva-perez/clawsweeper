## Summary

Keep open: current main still has the narrow final-tag matchers in the sanitizer and streaming subscriber, while the issue has open implementation PRs with closing syntax targeting this exact leak. Closing before one of those fixes lands would be premature.

## What This Changes

The issue reports Gemini/Gemma responses leaking internal `<final>` and `<final/>` wrapper tags into Control UI or delivered messages after upgrading to OpenClaw 2026.4.11.

## Best Possible Solution

Keep this issue open while maintainers review the linked final-tag PRs, converge on one implementation that aligns sanitizer, streaming, message-end/history behavior, and regression tests with the shared reasoning-tag matcher, then close after the merged fix ships.

## Reproduction Assessment

Yes. A focused source-level reproduction is available by passing self-closing or attributed final tags such as `<final/>Hello` or `<final id="1">Hello</final>` through the current sanitizer/subscriber patterns; those variants are not fully matched while the shared reasoning helper would match them.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse or align with the broader shared final-tag pattern across the sanitizer and streaming subscriber, with regression tests for self-closing and attributed variants plus a check that Control UI history replay stays clean.
