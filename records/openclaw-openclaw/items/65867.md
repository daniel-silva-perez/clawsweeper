## Summary

Keep open: current main and the latest shipped tag still use narrower final-tag patterns in two user-facing stripping paths, and an open PR with closing syntax targets this exact gap, so cleanup closure would be premature.

## What This Changes

The issue reports Gemini/Gemma responses leaking internal `<final>` wrapper tags into Control UI or delivered messages after upgrading to OpenClaw 2026.4.11.

## Best Possible Solution

Keep this issue open while maintainers review the linked final-tag PRs, converge on one implementation that aligns sanitizer, streaming, message-end/history, and tests with the shared reasoning-tag matcher, then close after the merged fix ships.

## Reproduction Assessment

Yes. A focused source-level reproduction is available: pass self-closing or attributed final tags such as `<final/>Hello` or `<final id="1">Hello</final>` through the current sanitizer/subscriber patterns and they are not fully matched, while the shared reasoning helper pattern would match them.

## Solution Assessment

Yes, the narrow maintainable direction is to reuse or align with the broader shared final-tag pattern across the sanitizer and streaming subscriber, with regression tests for self-closing and attributed variants. The open PRs should be reviewed together so persistence/history behavior is not left behind.
