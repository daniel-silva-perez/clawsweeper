## Summary

Keep open. Current main has not implemented the requested behavior, and the open PR still needs revision because its token writeback path can turn authored env-referenced Twitch tokens into literal config secrets.

## What This Changes

The PR adds Twitch refreshed OAuth token writeback with tests, wires the Twitch refresh callback to that helper, and makes terminal branches in an OpenProse fast-loop example exclusive.

## Best Possible Solution

Preserve or skip authored env-reference token fields during Twitch refresh writeback, add regression coverage for base-level, `accounts.default`, and named-account `${...}` accessToken configs, then rebase and keep the OpenProse exclusivity change if it still applies cleanly.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve env-referenced Twitch tokens before writeback:** `extensions/twitch/src/token-writeback.ts:105-106`
  - body: This reads the env-substituted config snapshot and then changes `accessToken`. If the user authored `channels.twitch.accessToken: "${OPENCLAW_TWITCH_ACCESS_TOKEN}"`, `writeConfigFile` can only restore the env ref when the incoming value still matches the env value, so the refreshed token is written as a literal secret. Preserve or skip authored env-reference token fields before writing.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The PR introduces a concrete secret-handling regression for Twitch configs that store access tokens as env references.

Concerns:

- **[medium] Env token refs can become literal OAuth tokens:** `extensions/twitch/src/token-writeback.ts:105`
  - body: The writeback helper mutates a resolved config snapshot before calling `writeConfigFile`; once the refreshed token differs from the env-resolved value, the existing env-ref restoration path will not restore the original `${...}` reference and can persist the OAuth token in plaintext config.
  - confidence: 0.9
