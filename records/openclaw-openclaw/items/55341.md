## Summary

Keep open. Current main still lacks both the Twitch refresh-token persistence and the OpenProse fast-loop exclusivity change, so this PR is not obsolete, but the proposed token writeback has a blocking secret-handling regression for env-referenced Twitch token fields.

## What This Changes

The PR adds a Twitch refreshed-token writeback helper with tests, wires Twitch refresh callbacks to call it, and changes an OpenProse fast-loop example so terminal output branches are mutually exclusive.

## Best Possible Solution

Implement Twitch refresh persistence inside the Twitch plugin while preserving authored env references and SecretRef-style token fields; only write literal refreshed tokens back to literal config fields, add regression coverage for base-level, accounts.default, and named-account env-ref cases, and keep the OpenProse else-branch fix if it still applies cleanly.

## Reproduction Assessment

Yes. A high-confidence reproduction path is a Twitch config whose authored accessToken or refreshToken is an env reference, then a RefreshingAuthProvider onRefresh event: the PR treats the resolved value as tokenSource config, mutates the resolved snapshot, and writeConfigFile leaves changed env-ref paths as literals.

## Solution Assessment

No. The requested behavior is useful, but this implementation is not the safest maintainable fix until it distinguishes literal config-backed tokens from authored env/secret references before writeback.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve env-referenced Twitch tokens before writeback:** `extensions/twitch/src/token-writeback.ts:105-108`
