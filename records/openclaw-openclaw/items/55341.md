## Summary

Keep open. The PR addresses a real gap in current main: Twitch refresh callbacks currently only log refreshed tokens and do not persist them, and the OpenProse example still has fall-through after terminal outputs. The branch is useful, but it should not merge as-is because the new writeback path can turn an authored `${...}` Twitch token env reference into a literal refreshed OAuth token in the config file.

## What This Changes

The PR adds Twitch refreshed OAuth token writeback for config-backed accounts with tests, and changes an OpenProse fast-loop example so terminal output branches are exclusive.

## Best Possible Solution

Keep the PR open and revise the Twitch writeback so it persists only literal config-backed token fields, preserves or skips authored env-reference token fields, and adds regression coverage for base-level, accounts.default, and named-account `${...}` accessToken configs. The OpenProse exclusivity change can stay if it still applies cleanly after rebase.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Preserve env-referenced Twitch tokens before writeback:** `extensions/twitch/src/token-writeback.ts:105-108`
  - body: This helper writes the resolved config snapshot after changing `accessToken`. If the user authored `channels.twitch.accessToken: "${OPENCLAW_TWITCH_ACCESS_TOKEN}"`, config loading resolves that to the secret and `writeConfigFile` will not restore the env ref once the refreshed token differs on a changed path. That silently persists a literal OAuth token and breaks env-managed token setups; detect the source token field from the authored/source config and skip or preserve env-ref-backed holders.
  - confidence: 0.89

## Security Review

Status: needs_attention

Summary: The PR introduces a concrete secret-handling regression for Twitch configs that store accessToken as an env reference.

Concerns:

- **[medium] Env token refs can become literal OAuth tokens:** `extensions/twitch/src/token-writeback.ts:105`
  - body: `readConfigFileSnapshotForWrite()` exposes resolved `snapshot.config`, and this PR mutates that value before `writeConfigFile()`. Because OpenClaw only restores env refs when the incoming value still matches the env-resolved value or the path is unchanged, a refreshed access token would be written as a literal secret over an authored `${...}` reference.
  - confidence: 0.9
