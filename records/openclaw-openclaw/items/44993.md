## Repair Work Prompt

Fix the stale `Current time:` heartbeat/main-session cron timestamp bug from this issue. Current `src/agents/current-time.ts` returns the original text whenever `base.includes("Current time:")`, which preserves stale timestamps between heartbeat runs. Update `appendCronStyleCurrentTimeLine` so empty/whitespace-only input still returns the trimmed base, but any existing `Current time:` line is replaced with the fresh `resolveCronStyleNow(cfg, nowMs).timeLine`; when no line exists, append the fresh line as today. Keep this generic in `src/agents/current-time.ts`; do not add extension/plugin-specific logic or change unrelated cron scheduling behavior. Add focused Vitest coverage, likely `src/agents/current-time.test.ts`, for appending when absent, replacing an older timestamp with a newer one, preserving surrounding text, avoiding duplicate lines over repeated calls, and keeping empty/whitespace inputs unchanged. Prior closed unmerged PRs #45071 and #45188 proposed the same direction; use them as context, not as something already landed. Validation: run `pnpm test src/agents/current-time.test.ts`; if a heartbeat-runner regression test is added, run that targeted heartbeat test too, then use the normal changed gate/Testbox path before handoff. Changelog likely not required for this tiny regression test/fix unless maintainers want user-facing bugfix notes.
