## Summary

Keep open. Current main still preserves a stale line-bound `Current time:` value in the shared helper, heartbeat still routes each run through that helper, and an open linked implementation PR now explicitly closes this issue, so this should stay open until that PR is reviewed and merged or closed.

## What This Changes

Heartbeat and cron prompts should refresh the injected `Current time:` timestamp on every run instead of reusing a timestamp from an earlier prompt.

## Best Possible Solution

Review and land PR #75025 if its implementation and CI are acceptable; otherwise close that PR and apply the same narrow helper-level idempotent refresh with focused regression tests for append, refresh, and duplicate-collapse behavior. This issue should close only after the active fix is merged.

## Reproduction Assessment

Yes. Pass text containing an older line-bound `Current time:` value plus a newer `nowMs` into `appendCronStyleCurrentTimeLine`; current main returns the old text unchanged, and heartbeat still routes run prompts through that helper.

## Solution Assessment

Yes. The narrow maintainable direction is to make the shared helper refresh or collapse existing line-bound `Current time:` markers while preserving empty-input behavior; fixing individual heartbeat or cron callers would leave shared consumers divergent.
