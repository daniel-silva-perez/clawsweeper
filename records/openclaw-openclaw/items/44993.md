## Summary

Keep open. Current main still has the shared current-time helper return an existing `Current time:` prompt unchanged, the heartbeat path still relies on that helper for freshness, and both linked fix PRs are closed unmerged while the latest release tag still carries the same guard.

## What This Changes

Refresh heartbeat and main-session cron `Current time:` prompt lines on each run instead of preserving an older timestamp.

## Best Possible Solution

Make the shared current-time append helper idempotent: keep empty input behavior, append one fresh line when absent, refresh or collapse stale OpenClaw-owned `Current time:` lines when present, and add focused regression tests for the helper plus heartbeat/reset/compaction callers.

## Reproduction Assessment

Yes. A deterministic code-level reproduction is to pass text containing an older `Current time:` line plus a newer `nowMs` into `appendCronStyleCurrentTimeLine`; current main returns the old text unchanged, and heartbeat passes each run's fresh `startedAt` through that helper.

## Solution Assessment

Yes. The narrow maintainable fix is in the shared helper, with caller-level tests proving heartbeat and adjacent reset/compaction paths receive a fresh timestamp without changing isolated cron's direct fresh-line construction.
