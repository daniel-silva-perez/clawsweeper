## Summary

Keep open. Current main still preserves any prompt containing `Current time:` unchanged in the shared helper, so heartbeat and main-session cron prompts can keep a stale helper-emitted timestamp even though each run computes a fresh `startedAt`; the latest release tag still has the same guard and the two linked fix PRs are closed unmerged.

## What This Changes

Heartbeat and main-session cron prompt construction should refresh OpenClaw-injected `Current time:` lines on every run instead of preserving an old timestamp.

## Best Possible Solution

Make the shared current-time helper idempotent for OpenClaw-owned timestamp lines: append one fresh line when absent, refresh or collapse stale helper-emitted `Current time:` lines when present, preserve unrelated prompt content, and add focused regression tests for repeated heartbeat/main-session cron prompt construction plus adjacent reset/compaction users.
