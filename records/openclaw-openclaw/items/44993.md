## Summary

Keep open. Current main still preserves an existing `Current time:` line in the shared heartbeat/cron helper instead of refreshing it, and the latest release tag contains the same guard. The two directly related fix PRs are closed unmerged, so there is still narrow follow-up work here.

## What This Changes

Heartbeat and cron prompt construction should refresh the injected `Current time:` line on each run instead of carrying a stale timestamp forward.

## Best Possible Solution

Make the shared current-time helper idempotent for OpenClaw-owned timestamp lines: preserve empty-input behavior, append when no timestamp exists, refresh a stale helper-emitted line when one exists, and cover repeated heartbeat/main-session cron prompt construction with focused regression tests.
