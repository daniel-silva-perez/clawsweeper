## Summary

Keep open. Current main still preserves an existing `Current time:` line unchanged, while heartbeat and main-session cron prompts still pass each run's fresh timestamp through that helper, so the reported stale timestamp path remains valid. The two linked fix PRs are closed unmerged, and the latest release tag still carries the same guard.

## What This Changes

Heartbeat and main-session cron prompts should refresh their `Current time:` timestamp on every run instead of preserving a timestamp from earlier prompt text.

## Best Possible Solution

Make the shared current-time append helper idempotent: preserve empty-input behavior, append one fresh OpenClaw `Current time:` line when absent, refresh or collapse existing line-bound `Current time:` markers when present, and cover the helper plus heartbeat/reset/plugin caller paths with focused regression tests.

## Reproduction Assessment

Yes. A deterministic reproduction is to pass text containing an older `Current time:` line plus a newer `nowMs` into `appendCronStyleCurrentTimeLine`; current main returns the old text unchanged, and heartbeat feeds each run's fresh `startedAt` through that helper.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the shared helper rather than in each caller, because heartbeat, session reset, and SDK/plugin callers all rely on the same append contract and would otherwise keep divergent freshness behavior.
