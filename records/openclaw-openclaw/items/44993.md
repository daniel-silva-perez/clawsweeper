## Summary

Keep open. Current main still preserves an existing `Current time:` line in the shared heartbeat/cron helper instead of refreshing it, and the latest release tag contains the same guard. The two directly related fix PRs were closed unmerged, so there is still narrow follow-up work here.

## What This Changes

The issue requests that heartbeat and cron prompt injection refresh the `Current time:` timestamp on each run instead of preserving an older timestamp line.

## Best Possible Solution

Make the shared current-time helper idempotent for OpenClaw-owned timestamp lines: keep empty-input behavior, append when no timestamp exists, refresh a stale helper-emitted line when one exists, and add focused regression coverage for repeated heartbeat/main-session cron prompt construction.

In `src/agents/current-time.ts`, update `appendCronStyleCurrentTimeLine` so empty or whitespace-only input still returns the trimmed base, text with no timestamp still appends the fresh `resolveCronStyleNow(cfg, nowMs).timeLine`, and an existing OpenClaw-owned `Current time:` timestamp line is refreshed instead of preserving the stale value. Be careful not to rewrite arbitrary user-authored reminder text that merely starts with `Current time:` unless the chosen contract intentionally treats that line as OpenClaw-owned; capture the boundary in tests. Add focused Vitest coverage, likely `src/agents/current-time.test.ts`, for appending when absent, refreshing an older timestamp with a newer one, preserving surrounding text, avoiding duplicate lines after repeated calls, and keeping empty/whitespace inputs unchanged. Add or update a heartbeat-runner regression only if needed to prove the main-session heartbeat path. Treat closed PRs #45071 and #45188 as context, not landed code.
