## Repair Work Prompt

Fix the stale heartbeat/cron `Current time:` regression tracked by https://github.com/openclaw/openclaw/issues/44993. On current main, `src/agents/current-time.ts` returns any prompt containing `Current time:` unchanged, so heartbeat/main-session cron prompts can preserve an old timestamp even though `runHeartbeatOnce` passes a fresh `startedAt` at `src/infra/heartbeat-runner.ts:1206`. Make `appendCronStyleCurrentTimeLine` idempotent for OpenClaw-owned timestamp lines: keep empty/whitespace behavior, append exactly one fresh line when none exists, and refresh or collapse stale helper-emitted `Current time:` lines on repeated calls. Use #45071 and #45188 as useful prior proposals, but implement cleanly against current main. Add focused regression coverage in a new `src/agents/current-time.test.ts` plus any narrow heartbeat/session-reset/post-compaction coverage needed to prove repeated construction refreshes without duplication.
