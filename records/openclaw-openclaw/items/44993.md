## Summary

Keep open. Current main still has the stale `Current time:` helper behavior, and a new open PR (#75025) now explicitly closes this issue, so the issue should remain open until that PR is reviewed and merged or closed.

## What This Changes

Heartbeat and cron prompt timestamps should refresh on every run instead of preserving an older `Current time:` line.

## Best Possible Solution

Review and land PR #75025 if its implementation and CI are acceptable; otherwise close that PR and apply the same narrow helper-level fix with focused regression tests. The issue should close only after the active fix is merged.

## Reproduction Assessment

Yes. Pass text containing an older line-bound `Current time:` value plus a newer `nowMs` into `appendCronStyleCurrentTimeLine`; current main returns the old text unchanged, and heartbeat still routes run prompts through that helper.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the shared current-time helper because heartbeat, session reset, and plugin/runtime callers share that contract; fixing individual callers would leave divergent behavior.
