## Summary

Keep open. Current main still omits both elapsed duration and the requester-approved absolute start time from the shared status footer, even though the session start timestamp is already stored/recoverable. The related timing issue remains open and does not supersede this narrower footer request.

## What This Changes

Add the current session elapsed duration, or an accepted absolute session start timestamp, to the `🧵 Session:` line in the `/status` and `session_status` status footer.

## Best Possible Solution

Implement this in the shared status renderer so both chat `/status` and the `session_status` tool show the same session start visibility. Resolve `sessionEntry.sessionStartedAt` first, fall back through `resolveSessionLifecycleTimestamps()`, then render a `started <timestamp>` or `duration <compact duration>` segment before the existing `updated ...` text, with focused coverage for the footer output.
