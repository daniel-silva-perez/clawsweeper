## Summary

Keep open. Current main still uses browser-visible `token=` assistant-media URLs, and this PR implements the requested scoped-ticket direction for #70830, but the proposed UI caches short-lived tickets indefinitely, which would break media reloads after the ticket TTL.

## What This Changes

The PR adds short-lived HMAC-scoped Gateway assistant-media tickets from authenticated metadata checks and updates Control UI chat rendering/tests to use Authorization headers for probes and `mediaTicket=` URLs for rendered media.

## Best Possible Solution

Land the scoped-ticket architecture after the UI tracks `mediaTicketExpiresAt` and refreshes metadata before cached tickets expire, preserving native media rendering without reintroducing long-lived credentials in URLs.

## Reproduction Assessment

Yes. On the PR branch, fetch assistant-media metadata once, wait past the 5-minute ticket TTL, then re-render or click/load the same local attachment; the UI cache will reuse the stale `mediaTicket` and the Gateway verifier will reject it.

## Solution Assessment

Not yet. HMAC-scoped media tickets are the narrow maintainable direction requested by #70830, but the implementation needs expiry-aware UI caching before it is safe to merge.
