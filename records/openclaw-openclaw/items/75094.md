## What This Changes

The PR adds short-lived HMAC-scoped assistant-media tickets in the Gateway, updates Control UI rendering to probe metadata with bearer auth and render `mediaTicket=` URLs, adds Gateway/UI tests, and records the user-facing fix in the changelog.

## Best Possible Solution

Land this PR after maintainer acceptance of the auth-boundary transition; the linked follow-up can then close through the merge, with any removal of legacy query-token compatibility tracked separately if desired.

## Reproduction Assessment

Yes. Current main reproduces the issue by rendering local assistant media as `assistant-media?...&token=...`, and the Gateway route still accepts query-token auth for that surface.

## Solution Assessment

Yes, with a maintainer-review caveat. Short-lived source-scoped HMAC tickets minted after authenticated metadata and local-root checks are a narrow fit for native browser media rendering without exposing long-lived credentials in rendered URLs.
