## Summary

Keep open. Current main still uses token-bearing assistant-media URLs, and this PR is the active implementation of the scoped-ticket direction from #70830, but it is not merge-ready until the required changelog entry is added and the failed CI checks are resolved or proven unrelated.

## What This Changes

The PR adds gateway-minted short-lived HMAC assistant-media tickets, accepts them for assistant media fetches, and updates Control UI chat rendering/tests to fetch metadata with Authorization headers and render `mediaTicket=` URLs.

## Best Possible Solution

Add the missing single-line Unreleased changelog entry with contributor attribution, get CI green, then land the scoped-ticket architecture if maintainers accept retaining legacy query-token compatibility for older callers.

## Reproduction Assessment

Yes. Current main can be verified by rendering a local assistant attachment with `assistantAttachmentAuthToken`, which produces an assistant-media URL containing `token=`; the PR head changes the same path to an authenticated metadata probe plus `mediaTicket=` render URL.

## Solution Assessment

Yes, with merge-readiness fixes. Gateway-signed, source-scoped, short-lived HMAC tickets are the narrowest maintainable solution for native browser media rendering without putting long-lived credentials in rendered URLs.
