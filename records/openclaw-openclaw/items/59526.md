## Summary

Keep this issue open. Current main supports manual per-session auth-profile selection with `/model <model>@<profileId>`, but the `/models` picker/menu path still exposes provider/model choices only; the channel adapter and Telegram callback flow do not carry or set an account/profile selection.

## What This Changes

Add account/profile switching to the `/models` buttons or menu for GPT/OpenAI and Claude/Anthropic-backed models, including visible current-account state.

## Best Possible Solution

Extend the existing model picker flow to reuse session `authProfileOverride` semantics: list available provider auth profiles for the selected provider/model, show the current profile in the picker/header, and set model plus profile together without introducing a new raw model-ref convention.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: `/model openai/gpt-4o@<profileId>` is parsed and tested, while `/models` button data and Telegram callbacks only carry provider/model and never write `authProfileOverride`.

## Solution Assessment

Yes, the requested direction is the maintainable fix if implemented as an additive account-aware picker over the existing auth-profile system. A safer alternative to avoid is encoding new account semantics into model ids instead of reusing profile overrides.
