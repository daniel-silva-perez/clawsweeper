## Summary

Keep this issue open. Current main supports manual `/model provider/model@profile` auth-profile overrides and shows the active provider auth label in parts of `/models`, but the interactive `/models` pickers still only select provider/model (and Discord runtime), not provider account/auth profile. That leaves the central requested workflow unfinished.

## Best Possible Solution

Keep the issue open and implement account/profile selection as an additive `/models` picker path. Reuse the existing auth-profile store/order and `applyModelOverrideToSessionEntry({ profileOverride })` plumbing, add current-account display to Telegram and Discord model views, and add focused tests for GPT/OpenAI and Claude/Anthropic profile-backed selections.
