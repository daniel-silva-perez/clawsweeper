## Summary

Keep this draft PR open as a possible implementation thread, but it is not merge-ready. Current main still lacks the proposed dedicated Activity tab and activity event stream, while the PR introduces blocking privacy/storage and UI filtering defects that need maintainer design review before the branch can be considered.

## What This Changes

The PR adds gateway activity events for agent lifecycle, thinking, tools, and subagents, plus a new Control UI Activity tab with an execution tree, timeline, metrics, filters, and persisted client-side state.

## Best Possible Solution

Rework this into a narrow, additive activity contract plus a Control UI panel that is ephemeral and redacted by default, preserves tree structure under filters, handles subagent failure paths, updates English/i18n through the supported pipeline, and aligns scope with #12831 and #54577 before merge.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature PR review. There is a high-confidence current-main comparison path: source and docs show no Activity tab, no /activity route, and no dedicated activity event stream on main.

## Solution Assessment

No, the current patch is not the best way to solve it as-is. The direction is plausible, but the safer solution is to keep sensitive activity data ephemeral by default, preserve ancestry for filtered execution trees, and land the protocol/UI work behind a reviewed privacy and UX design.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Keep activity events ephemeral by default:** `ui/src/ui/activity/activity-controller.ts:247`
  - body: The controller schedules a localStorage save for every activity event. Since the gateway events include summarized tool args/results and subagent results, opening the Activity tab would persist sensitive execution data despite the PR's stated ephemeral default. Gate persistence behind an opt-in or store only redacted/minimal state.
  - confidence: 0.93
- **[P2] Preserve ancestors in filtered trees:** `ui/src/ui/activity/activity-tree.ts:348-349`
  - body: `filterTree` drops run nodes when the kind filter excludes `run`, but the view renders the execution tree only from `rootNodes`. Selecting a tool-only or subagent-only filter can leave the tree panel blank even though matching nodes exist. Keep ancestors needed for display or build filtered roots from matching descendants.
  - confidence: 0.9
- **[P2] Close failed subagent starts:** `src/agents/subagent-spawn.ts:721`
  - body: `subagent.start` is emitted before the child gateway run request, but the catch path returns an error without emitting a terminal activity event. A failed spawn would leave a running subagent node in the Activity panel until reload/prune. Emit an error/end event on failure or move start emission after successful registration.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The PR introduces a concrete privacy/supply-chain-adjacent data handling concern by persisting activity metadata containing tool inputs, tool outputs, and subagent results in browser localStorage by default.

Concerns:

- **[medium] Activity metadata is persisted by default:** `ui/src/ui/activity/activity-controller.ts:247`
  - body: The proposed activity events attach summarized tool args/results and subagent result previews, and the controller saves the serialized activity tree to localStorage after handling events. That changes ephemeral execution telemetry into durable browser data and can retain sensitive commands, file paths, output snippets, or child-agent results without an opt-in.
  - confidence: 0.92
