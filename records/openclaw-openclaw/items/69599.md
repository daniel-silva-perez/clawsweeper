## Summary

Current main still does not make Control UI New Session, typed /new, or typed /reset converge on the backend fresh session creation path. The UI sends /new or /reset through chat.send, and gateway tests explicitly preserve reset-in-place behavior for chat.send. The backend already has sessions.create for independent dashboard sessions, so this is a valid narrow Control UI behavior gap rather than implemented, stale, or a duplicate of #47561.

## Best Possible Solution

Keep this issue open and implement a Control UI-owned fresh-session path: New Session and Control UI typed /new or /reset should call sessions.create for the active agent, switch the UI to the returned dashboard session key, refresh session/history state, and leave non-UI channel/gateway slash reset semantics unchanged unless maintainers explicitly choose a broader product change.
