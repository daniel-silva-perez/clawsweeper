## Summary

Keep open. The issue is an Android node roadmap item where several referenced parts are already merged, but current main still does not implement the remaining requested device.control, ADB, or dynamic program execution capability. The remaining work is real Android node/app surface work, not a stale duplicate or fully implemented request.

## Best Possible Solution

Keep this issue open or split it into focused Android-node follow-ups. Treat #44073, #48299, and #53752 as completed subitems, then implement the remaining capabilities in the Android app/node owner surface with protocol constants, registry/dispatcher handlers, gateway command policy, permission UX, tests, and docs. Dynamic execution should not land without a reviewed sandbox/approval model.
