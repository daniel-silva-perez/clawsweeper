## Summary

Keep this issue open. Current main has the slash-command `/acp spawn --bind here` path for current-conversation ACP binding, but the agent-facing `sessions_spawn` tool still exposes only `thread: true` for ACP binding. On Discord, that path resolves to child-thread placement, so it does not provide the requested programmatic equivalent for binding an ACP session to the current Discord thread in place.

## Best Possible Solution

Keep the issue open and implement an additive agent-facing ACP bind option, such as `bindTo: "current"`, in `sessions_spawn`. It should be plumbed through the tool schema, `SpawnAcpParams`, and ACP thread-binding preparation so Discord can reuse the existing current-conversation binding semantics without creating a child thread or requiring the `spawnAcpSessions` child-thread flag. Add focused coverage for Discord `sessions_spawn({ runtime: "acp", mode: "session", bindTo: "current" })` from an existing thread, plus docs alongside the existing `/acp spawn --bind here` section.
