## Summary

Keep open. Current main still has the Mattermost streaming contract mismatch: the docs advertise `channels.mattermost.streaming` modes, while the Mattermost strict schema/type/metadata omit that key and the inbound runtime still wires draft-preview streaming without resolving an `off | partial | block | progress` mode. Open PR #73252 is the active narrow fix candidate, so this should not be closed or queued as a duplicate repair job before that work lands or is replaced.

## What This Changes

The issue asks Mattermost to honor documented `channels.mattermost.streaming` modes and fix noisy, notification-unfriendly draft-preview finalization behavior.

## Best Possible Solution

Keep this as the Mattermost streaming-contract tracker until PR #73252 or an equivalent Mattermost-local fix lands. The desired end state is schema/type/metadata support for root and account `streaming`, preview-mode resolution through the shared streaming helper, legacy `blockStreaming` compatibility, an `off` path that bypasses draft-preview callbacks/finalization, and focused schema/runtime/docs coverage.

## Reproduction Assessment

Yes for the documented config mismatch: current main’s docs advertise `channels.mattermost.streaming`, but the strict Mattermost schema/type/metadata omit it and the inbound monitor wires draft previews unconditionally. The notification symptom has reporter evidence, but I did not live-test Mattermost client notifications.

## Solution Assessment

No current-main solution exists. The narrow maintainable path is a Mattermost-local implementation of documented `streaming.mode` behavior, with PR #73252 as the visible candidate; final notification behavior can be evaluated after the mode contract is real.
