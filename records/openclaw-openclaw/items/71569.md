## Summary

Keep open. Current main still has the core Mattermost streaming mismatch: docs advertise `channels.mattermost.streaming` modes, but the Mattermost schema, exported config type, generated metadata, and inbound runtime do not implement mode-aware preview behavior. Open PR #73252 is a relevant fix candidate, but it is unmerged and should be reviewed before this issue closes.

## What This Changes

The issue requests that Mattermost implement the documented `channels.mattermost.streaming` modes and improve draft-preview/final notification behavior.

## Best Possible Solution

Keep this as a Mattermost streaming-contract tracker until PR #73252 or an equivalent fix lands. The end state should add `streaming` schema/types/generated metadata, preserve compatibility for legacy block fields, resolve `off | partial | block | progress` in the Mattermost inbound runtime, and cover final-only/progress behavior with focused tests.
