## Summary

Keep this PR open. Current main still has the Feishu skill documentation gap: the skill doc documents obsolete `owner_open_id` and `full_access` guidance, while the current schema/runtime accept `grant_to_requester` and do not accept or consume `content` on `create`. The PR is not ready to close as implemented or land as-is because the review discussion identified real accuracy issues in the proposed wording and example.

## Best Possible Solution

Keep the PR open and ask for a small docs revision: state that `create` accepts title/folder/grant options only and content must be added with `write` or `append`; replace `full_access` with `edit`; qualify requester grants as available only with a trusted Feishu requester identity; and use `result.details.document_id` in the workflow example. If the separate create-with-content work in #48377 lands first, update this docs change to match that final behavior.
