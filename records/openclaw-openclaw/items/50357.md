## Summary

Keep this PR open. Current main still has the Feishu skill documentation gap, but this PR is not ready to close as implemented or merge as-is because its proposed wording conflicts with the current schema, runtime return shape, requester-grant gating, and permission level.

## Best Possible Solution

Keep the PR open and ask for a focused docs revision: state that current `create` accepts title/folder/grant options only and content must be added with `write` or `append`; replace `owner_open_id`/`full_access` with `grant_to_requester`/`edit`; qualify grants as available only when a trusted Feishu requester identity exists; and use `result.details.document_id` in the workflow example. If #48377 lands first, update the docs to match the shipped create-with-content behavior instead.
