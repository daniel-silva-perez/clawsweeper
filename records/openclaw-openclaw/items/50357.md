## Summary

Keep this PR open. Current main still has the Feishu skill documentation gap, but the PR is not safe to close as implemented or merge as-is: its proposed wording conflicts with the current create schema, runtime permission behavior, trusted-requester gating, and returned result shape. The related open PR #48377 may change the final docs direction if create-with-content lands first.

## Best Possible Solution

Keep the PR open and ask for a focused docs revision: say current create accepts title, folder_token, and grant_to_requester only; explain that content must be added with write or append unless #48377 lands; replace owner_open_id/full_access with trusted Feishu requester context/edit permission; qualify that requester grants require a trusted Feishu-channel requester identity; and use result.details.document_id in the workflow example.
