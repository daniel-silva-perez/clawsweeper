## Summary

Keep this PR open. Current main still has the Feishu skill documentation gap the PR is trying to address: the `create` schema has no `content` parameter, but `extensions/feishu/skills/feishu-doc/SKILL.md` still documents obsolete `owner_open_id`/`full_access` guidance and does not explain the create-then-write workflow. The PR is not ready to close or land as-is because the review comments identify real accuracy problems in the proposed docs: requester access is `edit`, requester grants only work with a trusted Feishu requester identity, and the result payload is wrapped under `details`.

## Best Possible Solution

Keep the PR open and ask for a small revision: document that `create` only accepts title/folder/grant options and requires a follow-up `write` or `append` for content; replace `full_access` with `edit`; qualify requester grants as available only when the tool call has a trusted Feishu requester identity; and use `result.details.document_id` in the example. After that, it can be treated as a focused docs fix unless the separate create-with-content work lands first.
