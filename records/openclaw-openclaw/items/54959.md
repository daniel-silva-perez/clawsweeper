## Summary

Keep open. Current main still lacks an OpenShell Platforms deployment page, and the PR addresses a real docs gap for a bundled OpenShell surface, but the latest patch still publishes unsupported security-sensitive proxy guidance and duplicate glossary entries, so it should stay under maintainer review rather than be auto-closed or merged as-is.

## What This Changes

The PR adds a new OpenShell sandbox deployment guide, registers it in the Platforms navigation, and adds OpenShell-related zh-CN glossary entries.

## Best Possible Solution

Publish a maintainer-reviewed OpenShell deployment guide, either as a dedicated Platforms page or folded into the canonical OpenShell docs, that uses the supported proxy.enabled / OPENCLAW_PROXY_URL lifecycle, reflects current Slack/WebSocket proxy behavior, preserves strict SSRF guard boundaries, and scopes advanced supervisor replacement guidance with clear trust and rollback warnings. The glossary additions should be deduplicated before merge.

## Security Review

Status: needs_attention

Summary: The diff is docs-only, but it introduces security-sensitive operator guidance for process-wide proxy hooks and supervisor binary replacement that should be reviewed before publication.

Concerns:

- **[medium] Custom preload and module monkey-patch guidance:** `docs/platforms/openshell.md:328`
  - body: The guide instructs operators to use NODE_OPTIONS to preload JavaScript that installs a global proxy dispatcher and monkey-patches Module._load for ws. That is process-wide code-execution and egress-routing guidance, and it conflicts with OpenClaw's supported proxy lifecycle.
  - confidence: 0.86
- **[low] Supervisor binary replacement instructions:** `docs/platforms/openshell.md:744`
  - body: The advanced section documents replacing the OpenShell supervisor binary inside the cluster container with docker cp and kill -9. That may be useful for upstream testing, but it normalizes swapping a trusted runtime binary outside a release process without enough trust, source, or rollback framing.
  - confidence: 0.78
