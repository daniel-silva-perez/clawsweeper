## Summary

Keep this PR open. Current main still does not include an OpenShell platform deployment page or Platforms navigation entry, and the existing OpenShell docs cover using OpenShell as a sandbox backend from a host Gateway, not operating OpenClaw inside an OpenShell sandbox. The PR is docs-only from a repository diff perspective, but the operator guidance is security-sensitive and still needs maintainer docs/security review rather than conservative cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer docs/security review. The best path is to reconcile the new platform guide with docs/gateway/openshell.md and current proxy/fetch-guard behavior, remove duplicate glossary entries, then either merge it as a dedicated Platforms deployment guide or fold the durable operator guidance into the canonical OpenShell docs.
