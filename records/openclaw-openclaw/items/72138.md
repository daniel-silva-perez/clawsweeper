## Summary

Keep open. Current main still does not emit plugin `message_sent` or internal `message:sent` hooks from Feishu normal conversation replies, while this PR remains the focused implementation for that Feishu regression and is not superseded by shipped code.

## Best Possible Solution

Keep this PR open and review/land it, or land an equivalent Feishu-owned patch. The implementation belongs in `extensions/feishu/src/reply-dispatcher.ts` and `extensions/feishu/src/bot.ts`: emit canonical plugin `message_sent` and internal `message:sent` for normal replies, pass routed session/group mirror metadata, include streaming-card message identity, avoid false failure hooks when only companion media fails, and keep focused Feishu regression tests. Because this is a user-visible Feishu hook fix, landing should also consider the normal changelog credit policy.
