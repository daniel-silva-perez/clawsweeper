## Summary

Keep open. Current main still lacks Feishu normal-reply `message_sent` and internal `message:sent` emission, and this PR is the focused implementation for the linked Feishu hook regression. It is not ready to merge as-is because the PR branch is behind current main and drops the newer late-streaming-final suppression behavior in the same dispatcher path.

## What This Changes

The PR adds sent-hook emission for Feishu normal replies, passes session/group hook metadata from Feishu bot dispatch call sites, captures streaming-card message IDs, and adds Feishu dispatcher regression tests.

## Best Possible Solution

Land this PR or an equivalent Feishu-owned patch after preserving current main's late-final streaming suppression, keeping sent-hook emission for text, card, media, streaming close, and failure paths, adding any required changelog credit, and getting the focused Feishu tests plus changed gate green.

## Reproduction Assessment

Yes. The linked bug has a clear source-level reproduction: configure a `message:sent` hook, send a normal Feishu bot conversation message, and current main's normal reply dispatcher sends directly without any sent-hook emitter or session metadata. The PR regression also has a code-level path covered by current main's late-final test: close a streaming card, then deliver a distinct late final with media.

## Solution Assessment

Not as currently proposed. The Feishu-owned dispatcher boundary is the right place because normal Feishu replies bypass shared outbound delivery, but the patch should be rebased or adjusted to preserve current main's `streamingClosedForReply` late-final guard while adding the new hook emission.
