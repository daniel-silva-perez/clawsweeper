## Summary

Keep open. Current main still calls bundled MCP tools without passing MCP `RequestOptions.timeout`, while the bundled SDK defaults requests to 60,000 ms. The configured `connectionTimeoutMs` is still only used for the initial MCP connection path. Open PR #73536 is an unmerged implementation candidate with closing syntax for this issue, so the issue should remain open until that PR, or an equivalent fix, lands.

## Best Possible Solution

Keep this issue open and review PR #73536 as the canonical implementation candidate. The likely code fix belongs in `src/agents/pi-bundle-mcp-runtime.ts`, carrying a resolved per-server request timeout into `BundleMcpSession` and passing it as the third argument to `client.callTool()`, with a regression test that captures the MCP SDK request options. If maintainers prefer separate connection and call timeouts, add a documented config field and schema/type/docs coverage instead of reusing `connectionTimeoutMs`.
