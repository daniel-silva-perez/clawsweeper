## Summary

Keep open. Current main still has the unsafe mismatch: doctor classifies on-disk agent directories by comparing them only to configured/default agent ids, so `agents/main` is treated as orphaned once `main` is not in `agents.list`, while current model/auth paths and docs still preserve `main` as a compatibility/default auth store.

## What This Changes

The issue requests that `openclaw doctor` stop presenting `~/.openclaw/agents/main` as safely removable stale data when current commands or compatibility auth/session paths may still use it.

## Best Possible Solution

Keep this open and make a narrow diagnostics fix: `doctor` should not present `agents/main` as removable stale data while current runtime/auth paths still use it, and the `models status` default-agent contract should be aligned with docs or the docs should be corrected. Add regression coverage for a non-main default agent plus an existing `agents/main/agent` directory, while preserving warnings for truly unknown agent dirs.
