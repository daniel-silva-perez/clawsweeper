## Summary

Keep open. Current main still has the reported doctor/models-status mismatch, and the linked fix PR is open and unmerged, so this issue should stay available for normal maintainer review and landing.

## What This Changes

Prevent `openclaw doctor` from treating `agents/main` as removable orphan state when a non-`main` default agent is configured but the compatibility store is still live.

## Best Possible Solution

Review and land #74438, or an equivalent narrow fix, so `models status` and `models status --probe` resolve the configured default agent when no override is present, while doctor exempts only the live compatibility `main` store and continues warning on real orphan directories.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is available: configure a non-`main` default agent, leave `agents/main/agent` on disk, and current main will classify `main` as orphaned while omitted `models status` still resolves through the legacy `main` compatibility path.

## Solution Assessment

Yes. The best path is a narrow command/path-resolution fix like #74438 rather than closing the issue early or broadly suppressing all orphan-agent warnings.
