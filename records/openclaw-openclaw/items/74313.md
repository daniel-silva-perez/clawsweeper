## Summary

Keep open. Current `main` still has the reported doctor/models-status mismatch, and #74438 is an open, unmerged closing PR for the same bug, so this issue should stay available until that PR or an equivalent fix lands. 

## What This Changes

The issue reports that `openclaw doctor` can label `agents/main` as an orphaned directory even while commands such as `models status` still use it as live auth/session state when the configured default agent is non-`main`.

## Best Possible Solution

Land #74438 or an equivalent narrow patch so `models status`, `models status --probe`, and doctor share the documented precedence: explicit `--agent`, then compatibility env override, then configured default agent, while doctor exempts only the live compatibility `main` store and still warns on real orphan directories.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a non-`main` default agent, leave `agents/main/agent` on disk, and current `main` classifies `main` as orphaned while omitted `models status` still resolves its auth/model store through the legacy `main` path.

## Solution Assessment

Yes. The best route is to review and land #74438, or a maintainer-equivalent narrow fix, rather than closing early or blanket-suppressing orphan-agent warnings.
