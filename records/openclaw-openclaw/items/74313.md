## Summary

Keep open. Current main still contains the reported mismatch: doctor classifies an unconfigured `main` agent directory as orphaned, while omitted-agent `models status` and probe paths still fall back to the compatibility `main` agent dir. The provided GitHub context also shows #74438 is an open, unmerged closing PR for the same behavior, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for `openclaw doctor` to stop presenting the live compatibility `agents/main` auth/session store as removable orphaned state when the configured default agent is not `main`, or to downgrade the warning with safer cleanup guidance.

## Best Possible Solution

Land #74438 or an equivalent narrow fix that makes omitted-agent `models status` and probe resolution use the configured default agent, exempts only the live compatibility `main` store from doctor cleanup warnings, and adds regression coverage for non-`main` defaults plus remaining real orphan directories.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a non-`main` default agent, leave `agents/main/agent` on disk, and current main will classify `main` as orphaned while omitted `models status` and probe paths still resolve auth state through the compatibility `main` path.

## Solution Assessment

Yes. The best fix shape is narrow path-resolution alignment plus a targeted doctor exemption, not closing the issue early or broadly suppressing all orphan-agent diagnostics.
