## Summary

Keep open. Current main still has the reported mismatch: doctor classifies nested on-disk agent dirs only against the configured default plus `agents.list`, while omitted-agent `models status` still resolves its auth store through the legacy `agents/main/agent` path. The provided context shows #74438 is open, unmerged, and uses closing syntax for this issue, so the issue should remain open for normal PR review.

## What This Changes

The issue asks OpenClaw to stop presenting the live `agents/main` compatibility/default auth and session store as removable orphaned state when a non-`main` agent is configured as default.

## Best Possible Solution

Review and land #74438, or an equivalent narrow fix, so omitted-agent `models status` and probe resolution use the configured default agent while doctor exempts only the live compatibility `agents/main` store and still reports true orphan directories.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: configure a non-`main` default agent, keep `agents/main/agent` on disk, and `listOrphanAgentDirs` will classify `main` as unconfigured while omitted-agent `models status` still reads through `resolveOpenClawAgentDir()`.

## Solution Assessment

Yes. The best fix direction is the linked PR or equivalent combined command-path repair; suppressing every `main` orphan warning alone would reduce the immediate data-loss risk but leave the documented default-agent behavior and status/probe resolution partly inconsistent.
