## Summary

Keep open. Current main still has the reported doctor/models-status mismatch, and #74438 is already the open, unmerged closing PR for the same behavior, so this issue should remain available for normal maintainer review and landing.

## What This Changes

The issue asks doctor to stop presenting a live compatibility `agents/main` store as removable orphaned state when the configured default agent is not `main`.

## Best Possible Solution

Land #74438 or an equivalent narrow fix that makes omitted-agent models status/probe resolve the configured default agent while doctor exempts only the live compatibility `main` store and continues reporting real orphan directories.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is available: configure a non-`main` default agent, leave `agents/main/agent` on disk, and current main will classify `main` as orphaned while omitted `models status` still resolves the auth store through the compatibility `main` path.

## Solution Assessment

Yes. The best fix shape is narrow command/path-resolution alignment plus a targeted doctor exemption, not closing the issue early or broadly suppressing all orphan-agent warnings.
