## Summary

Keep open. Current main still has the reported mismatch: doctor treats on-disk agent directories as reachable only when they match the configured default id or an `agents.list` entry, while omitted-agent `models status` still resolves its auth/model store through the legacy `agents/main/agent` path. The linked implementation PR #74438 is open and unmerged in the provided context, so this issue should stay open until maintainers land or close that PR.

## What This Changes

The report asks OpenClaw to stop presenting `agents/main` as safe orphan cleanup when a non-main default agent is configured but current commands still use the main compatibility/default store.

## Best Possible Solution

Review and land #74438, or an equivalent narrow fix, so omitted-agent `models status` and probe resolution match the documented default-agent contract while doctor exempts only the live compatibility `agents/main` store and continues warning on true orphan directories.
