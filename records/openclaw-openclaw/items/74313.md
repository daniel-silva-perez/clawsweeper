Keep open. Current main still has the reported doctor/models-status mismatch, and #74438 is an open, unmerged PR with closing syntax for the same behavior, so the issue should remain available until that PR is reviewed and landed or replaced.

## What This Changes

The issue requests that `openclaw doctor` stop presenting a retained `agents/main` compatibility store as removable orphaned state when a non-`main` agent is configured as default.

## Best Possible Solution

Land #74438 or an equivalent narrow fix that makes omitted `models status` and `models status --probe` resolve the configured default agent while doctor exempts only the live compatibility `main` store and continues reporting real orphan directories.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is available: configure a non-`main` default agent, leave `agents/main/agent` on disk, and current main will classify `main` as orphaned while omitted `models status` still resolves auth state through the compatibility `main` path.
