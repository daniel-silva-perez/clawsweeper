## Summary

Keep open. Current main still has the reported mismatch: doctor flags on-disk agent dirs by comparing only against the resolved default id and configured `agents.list` ids, while no-`--agent` model status still resolves the auth store through the legacy `agents/main/agent` path. The linked implementation PR #74438 is open and unmerged, so the issue should stay open until maintainers land or close that PR.

## What This Changes

Fix `openclaw doctor` so a retained `agents/main` compatibility/default store is not presented as a removable orphan when a non-main agent is configured as default.

## Best Possible Solution

Land or close #74438 as the canonical fix path: align `models status` and probe default-agent resolution with the documented contract, and make doctor exempt only the live compatibility `agents/main` store while preserving warnings for real orphan agent dirs.
