## Summary

Keep open. Current main still has the reported mismatch: doctor treats `agents/main` as an orphan whenever `main` is absent from `agents.list`, while current model/auth resolution and docs still keep `main` as a compatibility/default store. The discussion comments from @hclsys and @martingarramon correctly trace the same `listOrphanAgentDirs` path and outline narrow fix shapes, so this is an actionable bug rather than a cleanup close.

## What This Changes

The issue asks for `openclaw doctor` to stop presenting `~/.openclaw/agents/main` as removable orphaned state when a non-main configured default agent is in use but current commands or compatibility paths still use `agents/main`.

## Best Possible Solution

Keep the issue open and land a narrow diagnostics fix: doctor should not classify the documented `agents/main` compatibility store as removable stale state while current auth/model paths still use it, and regression tests should preserve warnings for truly unknown or unreachable agent directories.
