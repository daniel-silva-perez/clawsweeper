## Summary

Keep open. Current main still has the reported mismatch: doctor can mark `agents/main` orphaned when a non-`main` default agent is configured, while `models status` without `--agent` still reads the legacy `agents/main/agent` path. The linked #74438 PR is the right open fix vehicle and is still unmerged in the provided context, so this issue should remain open for maintainer review/landing of that PR.

## What This Changes

The issue reports that `openclaw doctor` may classify `~/.openclaw/agents/main` as an orphaned agent directory when a non-`main` configured default agent exists, even though other commands and docs still treat `agents/main` as live compatibility state.

## Best Possible Solution

Review and land #74438, or an equivalent narrow fix, so omitted `models status` and `models status --probe` resolve the configured default agent while preserving explicit `--agent` and env override precedence, and doctor exempts only the live compatibility `main` store while still warning on true orphan dirs.

## Reproduction Assessment

Yes. The source path gives a high-confidence reproduction: configure default agent `alfred`, leave `agents/main/agent` on disk, and current `listOrphanAgentDirs` marks `main` orphaned while no-`--agent` `models status` resolves through `resolveOpenClawAgentDir()` to `agents/main/agent`.

## Solution Assessment

Yes, the best current path is the linked narrow PR or equivalent command-seam fix, not closing this issue early. It addresses both sides of the mismatch and can be validated with targeted doctor/model-status tests.
