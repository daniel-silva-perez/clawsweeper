## Summary

Keep open. Current main still has the reported doctor/models-status mismatch, and the provided GitHub context shows #74438 is an open, unmerged closing PR for the same behavior, so this issue should remain available for normal maintainer review and landing.

## What This Changes

`openclaw doctor` should avoid presenting `agents/main` as safely removable orphan state when that same install can still use it as a live compatibility/default auth or session store.

## Best Possible Solution

Review and land #74438, or an equivalent narrow fix, so omitted-agent `models status` and `models status --probe` resolve the configured default agent while doctor exempts only the live compatibility `main` store and continues warning on real orphan directories.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a non-`main` default agent, leave `agents/main/agent` on disk, and current main will classify `main` as orphaned while omitted `models status` still resolves through the compatibility `main` path.

## Solution Assessment

Yes. The best current path is to resolve the command/path mismatch and add regression coverage, as #74438 proposes, rather than closing the issue early or suppressing all orphan-agent warnings.
