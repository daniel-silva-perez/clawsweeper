## Summary

Keep open. Current main still contains the reported doctor/models-status mismatch, and the provided GitHub context shows #74438 is an open, unmerged PR with closing syntax for the same behavior, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks `openclaw doctor` to stop presenting `~/.openclaw/agents/main` as a removable orphan when a non-`main` configured default agent coexists with live compatibility auth/session state.

## Best Possible Solution

Land #74438 or an equivalent narrow fix that makes omitted `models status` and `models status --probe` resolve the configured default agent while doctor exempts only the live compatibility `main` store and continues reporting real orphan directories.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a non-`main` default agent, leave `agents/main/agent` on disk, and current main will classify `main` as orphaned while omitted `models status` still resolves auth state through the compatibility `main` path.

## Solution Assessment

Yes. The linked fix direction appears to be the narrow maintainable path: align status/probe agent-scope resolution with the documented configured-default contract and keep doctor’s orphan warning focused on truly unreachable stores.
