## Summary

Keep this PR open. Current main has shipped better runtime diagnostics for invalid thread-bound `sessions_spawn` calls, but it still does not fully implement the PR’s requested prompt, tool-description/schema, and public-doc fallback guidance for unsupported requester channels.

## Best Possible Solution

Keep this PR open and ask for a refresh against current main. Preserve the shipped runtime diagnostics, drop obsolete runtime-copy hunks, then add focused fallback guidance to `buildAgentSystemPrompt`, `buildSubagentSystemPrompt`, `describeSessionsSpawnTool` / `createSessionsSpawnToolSchema`, and the public subagent/ACP docs, with prompt/tool-copy tests for unsupported-channel fallback wording.
