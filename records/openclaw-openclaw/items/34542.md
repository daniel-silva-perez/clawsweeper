## Summary

Keep this PR open. Current main has shipped stronger runtime diagnostics for invalid thread-bound `sessions_spawn` combinations, but it still does not fully implement the PR's intended prompt, live tool description/schema, and fallback docs guidance for unsupported requester channels.

## Best Possible Solution

Keep this PR open and ask for a refresh against current main: preserve the shipped runtime diagnostics, drop obsolete error-copy hunks, and add focused fallback guidance to `buildAgentSystemPrompt`, `buildSubagentSystemPrompt`, `describeSessionsSpawnTool` / `createSessionsSpawnToolSchema`, and the public subagent/ACP docs, with targeted tests for unsupported-channel fallback wording.
