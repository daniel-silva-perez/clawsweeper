## Summary

Keep this PR open. Current main has stronger runtime diagnostics for invalid thread-bound `sessions_spawn` combinations, but it still lacks the PR's requested prompt, live tool description/schema, and public docs fallback guidance that tells agents to use `mode: "run"` on unsupported requester channels.

## Best Possible Solution

Keep this PR open and ask for a refresh against current main: preserve the shipped runtime diagnostics, drop obsolete error-copy hunks, add focused fallback guidance to `buildAgentSystemPrompt`, `buildSubagentSystemPrompt`, `describeSessionsSpawnTool` / `createSessionsSpawnToolSchema`, and the public subagent/ACP docs, then add targeted tests for unsupported-channel fallback wording.
