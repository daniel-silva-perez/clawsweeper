## Summary

Keep this PR open. Current main has shipped stronger runtime diagnostics for invalid thread-bound `sessions_spawn` combinations, but it still does not fully cover the PR's stated prompt, live tool description/schema, and docs fallback guidance for unsupported requester channels. The PR discussion includes concrete repeated error examples, and the prior ClawSweeper review identified the same remaining gap; current main still matches that gap.

## Best Possible Solution

Keep this PR open and ask for a refresh against current main. Preserve the shipped runtime diagnostics, drop obsolete runtime-copy hunks, and add focused fallback guidance to `buildAgentSystemPrompt`, `buildSubagentSystemPrompt`, `describeSessionsSpawnTool` / `createSessionsSpawnToolSchema`, and the public subagent/ACP docs, with tests for unsupported-channel fallback wording.
