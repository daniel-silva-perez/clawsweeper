## Summary

Current main has shipped newer runtime error diagnostics for unsupported thread-bound `sessions_spawn` cases, but it has not implemented the PR’s prompt, tool-description/schema, and public docs guidance. This PR should not be closed as implemented; it needs a refresh against current main because several original hunks are now stale while the core guidance gap remains.

## Best Possible Solution

Keep this PR open and ask for a refresh against current main. Preserve the shipped runtime diagnostics, drop obsolete hunks such as the old `docs/tools/index.md` sessions section and older error-copy changes, then add focused fallback guidance to `buildAgentSystemPrompt`, `buildSubagentSystemPrompt`, `describeSessionsSpawnTool`/`createSessionsSpawnToolSchema`, and the public subagent/ACP docs. Add prompt/tool-copy tests that lock in unsupported-channel fallback wording.
