## Summary

Keep this PR open. Current main still does not contain the proposed dedicated `openclaw system event` recipes page, Automation nav entry, or CLI docs cross-link. The PR remains useful docs work for the discoverability gap described in #39298 and reinforced by #48869, but it needs revision before merge because the proposed webhook recipe is unsafe for untrusted input and the common-mistakes table contains misleading claims about `openclaw agent` and direct channel sends.

## Best Possible Solution

Keep this PR open and revise it before merge: add the dedicated recipes page and nav/cross-links, make webhook examples safe for untrusted input, explain that writing files alone does not wake the agent, recommend `openclaw system event --mode now` for external scripts, and correct the `openclaw agent` and direct-channel anti-pattern text against current behavior.
