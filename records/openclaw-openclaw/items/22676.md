## Summary

Current main still has the reported Signal daemon restart race. The Signal daemon handle remains synchronous, monitor shutdown does not await signal-cli exit, Signal config changes are still routed through channel reload, and an open PR (#71863) already targets this issue with an async stop/wait/SIGKILL fix. Per policy, this issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land PR #71863, or an equivalent focused fix, before closing. The fix should make Signal daemon stop asynchronous, wait for signal-cli exit after SIGTERM, escalate to SIGKILL on timeout, and await that shutdown through monitor/channel reload teardown before starting a replacement Signal channel, with regression coverage for daemon stop wait/escalation and abort-driven monitor shutdown.
