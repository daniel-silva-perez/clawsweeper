## Summary

Current main still has the Signal daemon restart race described here: the daemon handle stop contract is synchronous, the monitor calls it during abort/finally without awaiting child exit, Signal config changes are still hot-reloaded through channel stop/start, and an open paired PR (#71863) already targets this issue. Per policy, this issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land PR #71863, or an equivalent focused fix. The fix should make Signal daemon stop asynchronous, wait for signal-cli exit after SIGTERM, escalate to SIGKILL on timeout, and await that shutdown through monitor/channel reload teardown before starting a replacement Signal channel, with regression coverage for daemon stop wait/escalation and abort-driven monitor shutdown.
