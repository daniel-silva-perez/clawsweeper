## Summary

Current main still has the reported Signal daemon restart race: `SignalDaemonHandle.stop` remains synchronous, monitor teardown does not await `signal-cli` exit, and Signal config changes still hot-reload through stop/start. PR #71863 is open, unmerged, and directly targets this issue, so this issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land PR #71863, or an equivalent focused Signal-owned fix. The fix should make daemon `stop()` asynchronous and idempotent, send SIGTERM, wait for `signal-cli` exit, escalate to SIGKILL after a bounded timeout, and await that shutdown through monitor/channel reload teardown before starting a replacement Signal channel, with regression coverage for daemon stop wait/escalation and abort-driven monitor shutdown.
