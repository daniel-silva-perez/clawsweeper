## Summary

Current main still has the reported Signal daemon restart race: Signal daemon stop is synchronous, monitor teardown does not await child exit, and Signal config changes still hot-reload through stop/start. PR #71863 is open, unmerged, and directly references this issue with closing syntax, so the issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land PR #71863, or an equivalent focused fix. The fix should make Signal daemon stop asynchronous, wait for `signal-cli` exit after SIGTERM, escalate to SIGKILL on timeout, and await that shutdown through monitor/channel reload teardown before starting a replacement Signal channel, with regression coverage for daemon stop wait/escalation and abort-driven monitor shutdown.
