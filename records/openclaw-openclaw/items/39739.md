## Summary

Keep open. Current main still only accepts the global `*` wildcard or an exact normalized origin for `gateway.controlUi.allowedOrigins`; it does not support `chrome-extension://*`, and the related open PR is intentionally narrower.

## Best Possible Solution

Keep this issue open for explicit gateway product/security judgment. If accepted, implement a narrowly scoped `chrome-extension://*` matcher in `src/gateway/origin-check.ts`, likely alongside exact non-standard origin normalization, while preserving literal `Origin: null` rejection, existing HTTP(S) exact matches, host-header fallback, and loopback behavior. Add focused unit and WebSocket handshake tests for exact extension origins, `chrome-extension://*`, non-extension negatives, malformed origins, and docs/config security guidance. Coordinate with #38290, but keep this as the wildcard tracking item unless a maintainer-approved replacement supersedes it.
