## Summary

Keep this issue open. Current main still builds ClawDock Docker Compose commands from `docker-compose.yml` plus optional `docker-compose.extra.yml` only, so `docker-compose.override.yml` remains unsupported by `clawdock-start` and related helper commands.

## Best Possible Solution

Keep this open and fix `scripts/clawdock/clawdock-helpers.sh` so `_clawdock_compose()` appends `docker-compose.override.yml` when present, after confirming intended precedence relative to `docker-compose.extra.yml`. Update the public ClawDock/Docker docs and add focused coverage or a shell-level assertion for the effective Compose file list. Treat the comments' effective-running-config visibility idea as a follow-up enhancement, not a blocker for this narrow bug.
