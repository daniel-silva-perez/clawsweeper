## Summary

Keep open. Current main still builds ClawDock Compose invocations from `docker-compose.yml` plus optional `docker-compose.extra.yml` only; there is no current code, docs, or test coverage for `docker-compose.override.yml`. The related context does not supersede this: #59523 is about `docker-compose.sandbox.yml`, and #50029 only referenced this issue as local-model setup context.

## Best Possible Solution

Keep this issue open and fix it in `scripts/clawdock/clawdock-helpers.sh` by appending `docker-compose.override.yml` when present, with docs updated on the public ClawDock/Docker install pages and focused coverage or a scripted assertion for the generated Compose argument order.
