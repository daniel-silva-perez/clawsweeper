## Summary

Keep this issue open. Current main still routes ClawDock through explicit Docker Compose `-f` arguments for `docker-compose.yml` and optional `docker-compose.extra.yml` only, so a user-created `docker-compose.override.yml` is still not loaded by `clawdock-start`, `clawdock-stop`, `clawdock-update`, or other helper calls that use `_clawdock_compose()`.

## Best Possible Solution

Keep this open and fix `scripts/clawdock/clawdock-helpers.sh` so `_clawdock_compose()` includes `docker-compose.override.yml` when present, with documented precedence relative to `docker-compose.extra.yml`. Update `scripts/clawdock/README.md`, `docs/install/docker.md`, and ideally `docs/install/clawdock.md`; add a focused shell/helper assertion for the generated Docker Compose file order. Treat the comments' request for effective-running-config visibility as a useful follow-up, not a blocker for the narrow bug.
