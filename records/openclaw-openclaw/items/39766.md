## Summary

Keep this PR open. Current main at ef58307f843eed1a7832e9c23c519591f1b7cdbd still requires `models.providers.<id>.models` in the Zod schema, generated schema, and exported config type, while the OpenAI embedding path would use `models.providers.openai.baseUrl` if validation allowed the credential/baseUrl-only provider entry. The linked bug #39589 remains open, and related attempts #46980 and #71636 were closed unmerged rather than superseding this PR.

## Best Possible Solution

Keep this PR open, or land an equivalent fix, after aligning the source Zod schema, generated config schema, and exported `ModelProviderConfig` type so input can omit `models` while parsed/runtime code still has a safe array contract. Add focused regression coverage for config parsing plus OpenAI embedding `models.providers.openai.baseUrl` fallback behavior, stub DNS/SSRF lookups in tests, and explicitly review the existing remote embedding SSRF policy before merge.
