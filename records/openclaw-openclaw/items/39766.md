## Summary

Keep this PR open. Current main still does not implement the intended config-schema change: `models.providers.<id>` entries still require `models`, the generated schema and exported type still reflect that requirement, and validation fails before the embedding runtime can use a credential/baseUrl-only provider entry. The linked bug #39589 remains open, and #73577 is related diagnostic evidence rather than a replacement fix.

## Best Possible Solution

Keep this PR or the canonical bug #39589 open until main accepts `models.providers.<id>` entries without a `models` array, preserves a safe parsed/runtime array contract, regenerates the generated config schema, aligns the exported `ModelProviderConfig` type, and adds focused regression coverage for config parsing plus OpenAI embedding `baseUrl` fallback behavior. During review, address the DNS-dependent test and evaluate the existing SSRF/baseUrl policy for newly reachable credential-only provider entries.
