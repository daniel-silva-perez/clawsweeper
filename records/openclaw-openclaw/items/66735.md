## Summary

Keep this PR open. Current main still performs Linux/systemd restarts synchronously from restartSystemdService and lacks the self-cgroup handoff plus marker-based SIGTERM restart semantics that this PR adds. The PR is a focused implementation candidate, with useful author-reported live user-systemd smokes and a later ProjectClownfish repair on the branch.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate. The best path is maintainer review and landing after current CI plus a Linux user-systemd smoke confirm self-restarts return scheduled while external restarts remain synchronous/completed, with particular attention to the new command execution and marker-handling path.
