## Summary

Keep open. Current main has a shared CLI progress helper and several slow-path wrappers, but the broad loading-state request is still only partially implemented; representative ClawHub skills and plugin install/update paths still use ordinary log lines or tracing rather than a consistent progress wrapper.

## What This Changes

Add loading/progress indicators to slow OpenClaw CLI commands so users can tell commands are still progressing.

## Best Possible Solution

Keep this as a scoped CLI UX tracking item: choose the slow command families and delay/TTY semantics, then apply the existing progress helper consistently at those command boundaries while preserving JSON/plain output and adding focused tests/docs.
