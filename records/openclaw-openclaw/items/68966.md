## Summary

Keep open. Current main has WSL2/Windows Chrome remote-CDP guidance, browser command timeout hardening, and clearer CDP failure paths, but this recent report still lacks a direct shell reproduction or gateway/browser logs. The provided transcript looks like agent progress text, so SIGKILL could be the agent exec watchdog killing a stalled command rather than the browser driver itself crashing.

## Best Possible Solution

Keep this issue open and ask for a direct shell reproduction on current main or the latest release: openclaw browser status --json, openclaw browser doctor --deep, the redacted browser config/profile, gateway logs around the failure, and whether the target is Windows host Chrome or Linux Chrome inside WSL. If the target is Windows Chrome, route the user through the documented remote-CDP profile path; if logs show the OpenClaw exec watchdog killed a stalled browser command, improve browser CLI/gateway diagnostics or timeout handling for that path.
