## Summary

Current main still exposes the reported footgun in the workspace AGENTS template: it tells agents to track checks in `memory/heartbeat-state.json` with raw Unix epoch values, without guidance to obtain those values via a tool. OpenClaw now has a safer `HEARTBEAT.md` `tasks:` mode backed by session `heartbeatTaskState`, but the template that seeds user workspaces still points at manual JSON timestamps, so this issue remains actionable.

## Best Possible Solution

Keep this issue open and fix the core workspace guidance. The best path is to update `docs/reference/templates/AGENTS.md` to stop encouraging manual Unix epoch edits, preferably pointing users at `HEARTBEAT.md` `tasks:` blocks and `heartbeatTaskState`; if raw `heartbeat-state.json` remains documented, add an explicit instruction to obtain timestamps with a tool such as `date +%s` and consider adding Unix epoch seconds to the heartbeat current-time line or a defensive clamp for future timestamps.
