## Summary

Current main still leaves the reported heartbeat timestamp footgun reachable. The default AGENTS template continues to recommend `memory/heartbeat-state.json` with raw Unix epoch values and no instruction to obtain them via a tool, while the injected current-time line does not include epoch seconds. Safer `HEARTBEAT.md` task mode exists, but the template does not steer users away from manual timestamp state.

## Best Possible Solution

Keep this open and update core workspace guidance. Prefer changing `docs/reference/templates/AGENTS.md` to steer interval checks toward `HEARTBEAT.md` `tasks:` blocks and session-backed `heartbeatTaskState`. If `memory/heartbeat-state.json` remains documented, add explicit guidance that agents must obtain Unix timestamps through a tool such as `date +%s` and must not calculate epochs manually; optionally add epoch seconds to heartbeat current-time injection or clamp future stored timestamps for defense in depth.
