## Summary

Current main still leaves the reported heartbeat timestamp footgun reachable: the workspace AGENTS template recommends manual raw epoch values in memory/heartbeat-state.json without telling agents to obtain them via a tool, while the heartbeat current-time injection does not provide Unix epoch seconds. The safer HEARTBEAT.md tasks mode exists, but the default workspace guidance still points at the risky manual JSON pattern.

## Best Possible Solution

Keep this issue open and update the core workspace guidance. Prefer changing docs/reference/templates/AGENTS.md to recommend HEARTBEAT.md tasks blocks and session-backed heartbeatTaskState for interval checks. If memory/heartbeat-state.json remains documented, add explicit guidance that agents must obtain timestamps via a tool such as date +%s and should never calculate Unix epochs manually; consider a small runtime mitigation such as epoch injection or future-timestamp clamping if maintainers want defense in depth.
