## Summary

Keep this PR open. Current main still omits the turnSource* route fields when before_tool_call plugin approvals call plugin.approval.request, so the linked #74003 bug remains real. The proposed PR is a focused implementation candidate, but its current approach forwards routing from tool-call params rather than trusted runtime context, and the author has already acknowledged that spoofing concern and asked for maintainer direction.

## Best Possible Solution

Keep this PR open for maintainer/security review. The best fix is to thread trusted turn-source metadata from the gateway/embedded run context into HookContext or an equivalent trusted agent-side structure, have requestPluginToolApproval forward only that trusted data to plugin.approval.request, and cover both routed and route-less callers with focused regression tests. The current baseParams approach should not merge without that trust-boundary fix.
