## Summary

Keep this PR open. Current main still omits turnSource* routing fields when before_tool_call plugin approvals call plugin.approval.request, so the linked #74003 behavior remains valid. The PR is a focused implementation attempt, but its current patch forwards routing fields from model/tool-call params, and both the security bot and the PR author identify that as a spoofable trust-boundary issue requiring maintainer/security direction before merge.

## Best Possible Solution

Keep this PR open for maintainer/security review. The best fix is to thread trusted turn-source metadata from the gateway/embedded run context into HookContext or an equivalent trusted agent-side structure, have requestPluginToolApproval forward only that trusted data to plugin.approval.request, and add focused regression coverage for both routed and route-less plugin approval callers. The baseParams implementation should not merge without that trust-boundary fix.
