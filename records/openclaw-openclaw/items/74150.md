## Summary

Keep open. Current main still omits the turnSource* route fields from before_tool_call plugin approval requests, so the linked #74003 bug remains real; however, this PR reads routing from model/tool-call params, which creates a trust-boundary concern that needs maintainer/security direction before merge.

## What This Changes

The PR forwards turnSourceChannel, turnSourceTo, turnSourceAccountId, and turnSourceThreadId from before-tool-call params to plugin.approval.request, adds two regression tests, and adds a changelog entry for #74003.

## Best Possible Solution

Keep this PR open for maintainer/security review. The best fix is to thread trusted turn-source metadata from the gateway/embedded run context into HookContext or an equivalent trusted agent-side structure, forward only that trusted data to plugin.approval.request, and cover routed plus route-less callers with focused regression tests. The current baseParams approach should not merge without that trust-boundary fix.
