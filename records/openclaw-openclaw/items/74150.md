## Summary

Keep this PR open. Current main still has the reported agent-side plugin approval routing gap, but the proposed implementation takes route metadata from tool-call params, and the PR discussion plus source inspection show that boundary is model/plugin-influenced rather than trusted runtime context.

## What This Changes

The PR forwards turnSourceChannel, turnSourceTo, turnSourceAccountId, and turnSourceThreadId from before-tool-call params into plugin.approval.request, adds two regression tests, and adds a changelog entry for the linked bug.

## Best Possible Solution

Thread trusted turn-source metadata from the gateway or embedded run context into HookContext or an equivalent trusted agent-side structure, forward only that trusted data to plugin.approval.request, and keep focused coverage for routed and route-less approval callers.
