## Summary

Keep this PR open. Current main still calls `plugin.approval.request` without `turnSourceChannel`, `turnSourceTo`, `turnSourceAccountId`, or `turnSourceThreadId`, while the gateway protocol and handler already accept and route those fields. The PR is a plausible focused fix for #74003, but the implementation should be reviewed so the approval route comes from trusted runtime context rather than spoofable tool params.

## Best Possible Solution

Keep this PR open for maintainer review. The best fix is to route `before_tool_call` plugin approvals with trusted origin metadata from the agent runtime, add focused regression coverage for routed and route-less callers, and validate the live Telegram/Discord/Slack-style approval path before merge.
