## Summary

Keep this PR open. Current main still omits the routing fields needed for plugin approvals, so the linked bug is real, but the PR forwards those fields from model/tool-call params and the discussion already identified that as an approval-routing spoofing risk. Maintainer/security review should choose the trusted metadata path before merge.

## What This Changes

The PR forwards turnSourceChannel, turnSourceTo, turnSourceAccountId, and turnSourceThreadId from before-tool-call params into plugin.approval.request, adds regression coverage, and adds a changelog entry for #74003.

## Best Possible Solution

Thread trusted turn-source metadata from the gateway/message-channel run context into HookContext or an equivalent trusted agent-side structure, forward only those trusted fields to plugin.approval.request, preserve route-less caller behavior, and cover routed plus route-less plugin approvals with focused regression tests before landing or replacing this PR.
