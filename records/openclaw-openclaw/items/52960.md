## Summary

Keep this issue open. Current main has Control UI chat streaming, live tool cards, queued follow-ups, Stop/abort controls, and narrow compaction/fallback status toasts, but it still does not expose the requested compact active-run progress model with phase, latest action, last activity, elapsed time, subagent/tool waiting state, or reconnect-during-run visibility. The provided discussion also correctly separates merged PR #31192 as delivery-queue retry work, not this UI feature.

## Best Possible Solution

Keep this issue open and implement a Control UI active-run progress model fed by gateway chat, agent lifecycle, tool, subagent, delivery, and reconnect signals. Render a compact strip near the thread or composer with current phase, latest meaningful action, last activity, elapsed time, waiting target/subagent summary, and reconnect count; add focused UI/controller tests and update Control UI docs.
