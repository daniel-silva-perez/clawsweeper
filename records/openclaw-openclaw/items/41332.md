## Summary

Keep this PR open but do not merge as-is: current main does not implement the gateway-specific exit path, but the PR removes a documented/tested transient cooldown fallback path and adds a public cron config key without regenerating the schema surface.

## What This Changes

The PR adds a gateway-specific `process.reallyExit` runtime, changes model fallback cooldown probing, introduces `cron.startupStaggerMs`, and updates related tests plus fake-secret allowlist comments.

## Best Possible Solution

Refresh or split the work before landing: keep the gateway shutdown fix focused with Node/macOS shutdown proof, preserve the documented transient cooldown sibling-probe contract unless maintainers intentionally redesign it, and route cron restart controls through the public config/schema/docs work already tracked by #42039.
