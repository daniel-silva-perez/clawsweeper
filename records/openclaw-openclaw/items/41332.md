## Summary

Keep open: this is a concrete implementation PR, but the current patch should not merge as-is. The PR still has blocking P2 correctness concerns around the gateway exit dependency contract, the current model-fallback cooldown policy, and generated public config schema coverage, and its cron config direction overlaps the still-open #42039 follow-up.

## What This Changes

The PR adds a gateway-specific runtime exit path, tightens model-fallback cooldown probing, adds `cron.startupStaggerMs` for startup missed-job spacing, updates related tests, and allowlists fake test-token strings in one agent test.

## Best Possible Solution

Refresh or split the work into maintainable pieces: prove a real no-native-atexit gateway shutdown strategy on the target macOS/GGML stack, preserve the intentional transient same-provider cooldown recovery path unless maintainers explicitly redesign fallback policy, and handle cron startup catch-up configuration through the generated schema/docs pipeline in the same direction as #42039.

Yes for the PR review findings: source and patch inspection against current main reproduces the cooldown regression and generated-schema gap, and Node v22 source inspection reproduces the `reallyExit` API concern. The original GGML Metal shutdown crash itself still lacks a high-confidence live reproduction path in the PR record.

## Solution Assessment

No. The PR bundles three behavioral changes, and the current implementations are not the narrowest maintainable fixes because the runtime exit path is dependency-unproven, fallback policy conflicts with current tested behavior, and cron config needs the generated public schema/docs path or consolidation with #42039.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Prove the gateway exit path skips native atexit:** `src/runtime.ts:61`
  - body: `createGatewayRuntime().exit` relies on `process.reallyExit` while documenting that it skips native/C++ at-exit handlers. Node v22 source shows `ReallyExit` invokes `RunAtExit(env)` before exiting, so this does not establish that the proposed path avoids the reported GGML native cleanup crash.
  - confidence: 0.86
- **[P2] Preserve transient same-provider cooldown probes:** `src/agents/model-fallback.ts:445-451`
  - body: Changing active-cooldown attempts to `params.isPrimary && shouldProbe` skips same-provider fallback siblings during transient `rate_limit`, `overloaded`, and `timeout` cooldowns. Current main intentionally allows those probes and has regression tests asserting `allowTransientCooldownProbe: true` for that path.
  - confidence: 0.9
- **[P2] Regenerate the public config schema:** `src/config/zod-schema.ts:512`
  - body: Adding `startupStaggerMs` only to the zod/type surface leaves the generated base config schema unchanged, but `openclaw config schema` and `config.schema.lookup` are served from that generated schema path. The new public key will not be exposed consistently without regenerating schema/docs or folding this into #42039.
  - confidence: 0.84
