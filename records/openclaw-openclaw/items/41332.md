## Summary

Keep this PR open, but do not merge it as-is. Current main still lacks the gateway shutdown change and the proposed cron startup config, so the PR is not obsolete; however, the patch conflicts with the current documented model-fallback cooldown policy and leaves the new cron config surface incomplete.

## What This Changes

The PR adds a gateway-specific `process.reallyExit` runtime, changes model fallback cooldown probing, introduces `cron.startupStaggerMs`, and updates related tests and fake-secret allowlist comments.

## Best Possible Solution

Split or refresh the PR before landing: keep the gateway shutdown work as a focused runtime patch with Node and Metal shutdown proof, preserve the current transient cooldown probe contract unless maintainers deliberately redesign it with docs/tests, and handle cron restart catch-up as complete public config work aligned with #42039.
