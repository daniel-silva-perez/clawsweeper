## Summary

Keep open: current main does not contain the gateway runtime change, and the PR still has blocking correctness issues relative to current main and Node source. The key blockers are the unproven `process.reallyExit` shutdown strategy, removal of transient same-provider cooldown recovery that current tests intentionally protect, and an incomplete public config schema/docs path for the cron key.

## What This Changes

The PR adds a gateway-specific runtime exit path, tightens model-fallback cooldown probing, adds `cron.startupStaggerMs` for startup catch-up spacing, and updates related tests plus fake-secret allowlist comments.

## Best Possible Solution

Refresh or split the PR into focused changes: prove a real no-atexit gateway shutdown strategy on the target macOS/GGML stack, preserve transient same-provider cooldown probes unless maintainers explicitly redesign fallback policy, and consolidate cron catch-up public configuration with #42039 including generated schema, docs, and tests.

## Reproduction Assessment

Yes for the PR findings: source inspection against current main, the PR patch, and Node v22 source reproduces the cooldown regression, generated-schema gap, and `reallyExit` API concern. The original GGML Metal crash itself does not yet have a high-confidence live reproduction path in the PR record.

## Solution Assessment

No. The current PR bundles three behavior changes and the proposed implementations are not the narrowest maintainable fixes: the runtime exit path is dependency-unproven, the fallback policy conflicts with current tests, and cron public config needs the generated schema/docs pipeline or consolidation with #42039.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Prove the gateway exit path skips native atexit:** `src/runtime.ts:61`
  - body: `createGatewayRuntime().exit` relies on `process.reallyExit` while documenting that it skips native/C++ exit handlers. Node v22 source shows `ReallyExit` invokes `RunAtExit(env)` before exiting, so this does not prove it avoids the reported GGML native cleanup crash path.
  - confidence: 0.86
- **[P2] Preserve transient same-provider cooldown probes:** `src/agents/model-fallback.ts:445-451`
  - body: Changing active-cooldown attempts to `params.isPrimary && shouldProbe` skips same-provider fallback siblings during transient `rate_limit`, `overloaded`, and `timeout` cooldowns. Current main intentionally allows those probes and has regression tests asserting `allowTransientCooldownProbe: true`.
  - confidence: 0.9
- **[P2] Regenerate the public config schema:** `src/config/zod-schema.ts:512`
  - body: Adding `startupStaggerMs` only to zod/types leaves the generated base config schema unchanged, but `openclaw config schema` and `config.schema.lookup` are served from that generated schema path. The new public key will not be exposed consistently without regenerated schema/docs or consolidation with #42039.
  - confidence: 0.84
