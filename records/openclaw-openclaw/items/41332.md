## Summary

Keep open. Current main still does not implement the PR's three central changes, so the work is not obsolete, but the patch conflicts with the current documented fallback cooldown contract and leaves the new cron config surface incomplete.

## What This Changes

The PR adds a gateway-specific `process.reallyExit` runtime, removes active-cooldown model fallback probes, adds `cron.startupStaggerMs`, and updates related tests/mocks plus incidental fake-secret allowlist comments.

## Best Possible Solution

Update or split the PR before landing: keep the gateway shutdown change as a focused runtime patch with Node/Metal proof, preserve or explicitly redesign transient cooldown probing with docs/tests, and handle cron restart catch-up as a complete public config change reconciled with #42039.
