## Summary

Keep open. Current main still does not implement the PR's three central changes: the gateway run path still exits through defaultRuntime/process.exit, model fallback still has cooldown probe paths this PR removes, and cron has no public startupStaggerMs config or immediate startup catch-up sleep. The PR discussion shows useful review feedback was addressed, but the branch is now stale against newer cooldown policy and tests, so it remains a maintainer review/rebase candidate rather than conservative cleanup closure.

## Best Possible Solution

Keep this PR open, but require a rebase and likely split review before landing. The gateway shutdown fix should be proven on the Node gateway path with clear cleanup-order evidence and Metal shutdown verification; the cooldown changes should be reconciled with current timeout/unknown/model-scoped transient probe behavior and tests; the cron startup staggering should add config, schema, docs/generated help, and restart catch-up coverage; and the incidental secret allowlist change should be removed or justified separately.
