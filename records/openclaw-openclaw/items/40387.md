## Summary

Keep this PR open. Current main still has no EasyRunner guide or navigation entry, and the PR is a focused docs-only contribution in an existing hosting docs area. It is not safe to close as implemented or obsolete, but it still needs maintainer docs/security review before merge because the proposed guide treats `gateway.controlUi.dangerouslyDisableDeviceAuth=true` as normal reverse-proxy setup and still has operational gaps around update rebuilds and current Control UI origin guidance.

## Best Possible Solution

Keep the PR open and request a rebased docs revision. The best path is to move or redirect the EasyRunner guide into the current Install > Hosting surface, align Podman/Caddy setup with current container bind and `gateway.controlUi.allowedOrigins` behavior, remove normal-use `dangerouslyDisableDeviceAuth` guidance, add a rebuild step before redeploying updates, and have maintainers review EasyRunner-specific reverse-proxy/trusted-proxy assumptions before merge.
