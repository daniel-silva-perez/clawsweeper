## Summary

Keep this PR open. Current main still has no EasyRunner deployment guide or navigation entry, and provider-specific hosting docs are an established OpenClaw docs surface. The PR is docs-only from a repository supply-chain perspective, but the proposed operator guidance remains security-sensitive and needs maintainer docs/security review before merge.

## Best Possible Solution

Keep the PR open and ask for a revised, rebased docs contribution. The best path is to place or link EasyRunner under the current Install > Hosting surface, align the Caddy/Podman example with current gateway bind/auth/allowedOrigins requirements, keep secrets in env or secret storage, remove normal-use dangerouslyDisableDeviceAuth guidance, and have maintainers review the EasyRunner-specific deployment flow for real operational accuracy before merge.
