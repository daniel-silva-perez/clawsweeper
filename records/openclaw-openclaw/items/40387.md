## Summary

Keep this PR open. Current main still has no EasyRunner deployment guide or docs navigation entry, and provider-specific hosting docs remain an established OpenClaw docs surface. The PR is docs-only from a supply-chain standpoint and later commits did address some early review feedback, but the proposed guide still needs rebase plus maintainer docs/security review before merge.

## Best Possible Solution

Keep the PR open and request a rebased docs revision. The best path is to place or link EasyRunner under the current Install > Hosting surface, align the Podman/Caddy flow with current Docker/Podman setup, gateway bind, auth, and `allowedOrigins` behavior, remove normal-use `dangerouslyDisableDeviceAuth` guidance, preserve the improved secret-handling and image-build notes, and have maintainers review the EasyRunner-specific operational steps before merge.
