## Summary

Keep this PR open. Current main still has no EasyRunner deployment guide or navigation entry, and hosting/provider deployment docs are an established OpenClaw docs surface. The contribution is docs-only from a repo security/supply-chain perspective and it has addressed some early review feedback, but the proposed operator guidance still needs maintainer docs/security review before merge rather than conservative cleanup closure.

## Best Possible Solution

Keep the PR open and ask for a revised, rebased docs contribution. The best path is to move or link EasyRunner under the current Install > Hosting surface, align the Caddy/Podman commands with the gateway bind and allowedOrigins contracts, keep secrets in env/secret storage, avoid normal-use dangerouslyDisableDeviceAuth guidance, and have maintainers review the EasyRunner-specific deployment flow for security and real operational accuracy.
