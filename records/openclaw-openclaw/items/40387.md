## Summary

Keep this PR open. Current main still has no EasyRunner deployment guide or navigation entry, and the PR is a focused docs contribution in an established hosting/deployment docs surface. It is not safe to close as implemented or obsolete, but it still needs maintainer docs/security review because the proposed guide normalizes `gateway.controlUi.dangerouslyDisableDeviceAuth=true` for routine reverse-proxy access and is out of date with current hosting navigation and Control UI origin guidance.

## Best Possible Solution

Keep the PR open and request a rebased docs revision. The best path is to place or redirect EasyRunner under the current Install > Hosting surface, align Podman/Caddy instructions with current Docker/container bind and `gateway.controlUi.allowedOrigins` behavior, remove normal-use `dangerouslyDisableDeviceAuth` guidance, add the missing rebuild step in the update flow, and have maintainers review the EasyRunner-specific operational assumptions before merge.
