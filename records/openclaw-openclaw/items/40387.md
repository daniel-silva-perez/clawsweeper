## Summary

Keep this PR open. Current main still does not include an EasyRunner deployment guide or navigation entry, and hosting/provider deployment docs are an established docs surface. The PR is not obsolete, but it still needs a rebase plus maintainer docs/security review because the proposed guide normalizes `gateway.controlUi.dangerouslyDisableDeviceAuth=true` for ordinary reverse-proxy use and does not align the public domain flow with current Control UI allowed-origin requirements.

## Best Possible Solution

Keep the PR open and request a rebased docs revision. The best path is to place or link EasyRunner under the current Install > Hosting surface, align the Podman/Caddy flow with current container bind and `gateway.controlUi.allowedOrigins` behavior, remove normal-use `dangerouslyDisableDeviceAuth` guidance, keep the improved image-build and secret-handling notes, and have maintainers review the EasyRunner-specific operational steps before merge.
