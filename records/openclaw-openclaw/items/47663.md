## Summary

Keep open: the reported reminder-note false positive still exists on current main, and this PR is a relevant implementation, but the branch is not merge-ready because the new public config key is not added to the typed config contract and generated/help metadata. It also overlaps an open alternate UX fix in #39330, so maintainers should choose the desired product/API direction before landing either approach.

## What This Changes

This PR adds a `messages.unscheduledReminderNote` boolean gate around the unscheduled-reminder note append path, adds the key to the Zod session schema and field labels, and adds reminder-guard unit tests.

## Best Possible Solution

Settle the reminder-guard UX, then land one complete fix: either a fully documented/generated `messages.unscheduledReminderNote` config with runtime wiring, public types, schema/help metadata, docs/changelog, and integration coverage, or the competing delivery-stripping approach that prevents this internal note from appearing in user-visible replies.
