This PR adds a `messages.unscheduledReminderNote` boolean opt-out around the unscheduled-reminder note, adds the key to the Zod schema and field labels, and adds reminder-guard unit tests.

## Best Possible Solution

Land one coherent reminder-guard UX implementation: either a fully typed, documented, generated, and tested `messages.unscheduledReminderNote` config option, or the competing delivery-strip/system-annotation approach that prevents this internal note from appearing as agent-authored user text.

## Solution Assessment

Unclear. A default-on config opt-out can address the reported pain, but it is not yet clearly the best maintainable UX while #39330 proposes stripping the internal note from delivery entirely and this branch has not completed the public config contract.
