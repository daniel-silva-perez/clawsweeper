## Summary

Keep open. The reported reminder-note UX problem still exists on current main and this branch is a plausible small fix, but it is not merge-ready because it reads a new public config key without completing the typed config contract or regenerated config metadata. Maintainers also need to choose between this opt-out config approach and overlapping reminder-guard PRs such as #39330 and #52571.

## What This Changes

This PR adds a `messages.unscheduledReminderNote` boolean gate around the unscheduled-reminder note append path, adds the key to the session schema and labels, and adds reminder-guard unit tests.

## Best Possible Solution

Settle the reminder-guard UX direction, then land one complete implementation: either a fully typed, documented, generated `messages.unscheduledReminderNote` config with runtime wiring and focused coverage, or the delivery-stripping/system-annotation approach that prevents this internal note from appearing in user-visible replies.
