## What This Changes

The issue reports that a clean Ubuntu Kylin 20.04 npm install should let the Web Control UI WebSocket connect instead of failing with `token_mismatch` and `device_token_mismatch` after pairing.

## Best Possible Solution

Reproduce the Control UI connection on a Linux systemd install and compare the dashboard URL token, effective gateway service environment, `gateway.auth.token`, stored browser device token, and pairing record. If drift is confirmed, fix the narrow token-source or token-repair gap without weakening gateway auth, and split provider chat timeout behavior into a separate follow-up if it remains after WebSocket auth succeeds.
