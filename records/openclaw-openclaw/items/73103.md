## Summary

Current main already contains a broader merged implementation for the same web_fetch legacy charset problem, with source, regression tests, changelog/release-note evidence, and release provenance. This PR's intended behavior is therefore redundant with the canonical fix that merged in #73513 and shipped in v2026.4.29-beta.1.

## What This Changes

The PR changes web_fetch response reading to accumulate raw bytes, detect charsets from Content-Type or bounded HTML meta tags, decode with TextDecoder, and add ISO-8859-1 regression coverage.

## Best Possible Solution

Close this PR and keep the merged #73513 implementation on main; any future charset changes should build on the current attribute-parsing and BOM/XML/HTML sniffing code in src/agents/tools/web-shared.ts.

## Reproduction Assessment

## Solution Assessment
