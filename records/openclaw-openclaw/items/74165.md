## Summary

Keep this open. Current main still has the centralized embedded-run abort path and the separate abort triggers, but it does not implement the requested `OPENCLAW_LOG_ABORT_SOURCES` gate, `[abort-source]` marker, source classifier, stack-frame capture, or tests. The related implementation PR #74110 is closed unmerged, so it remains useful prior art rather than shipped behavior.

## What This Changes

Add an opt-in `OPENCLAW_LOG_ABORT_SOURCES` diagnostic that logs one classified `[abort-source]` line with stack frames whenever embedded `abortRun()` is entered.

## Best Possible Solution

Keep this open for maintainer product and operability review. If accepted, adapt #74110 as a narrow core change: add a helper under `src/agents/pi-embedded-runner/run`, call it once from `abortRun()`, classify external signal, LLM idle timeout, compaction timeout, run timer, and explicit cancel, test default-off/env-gated behavior, and avoid runtime behavior changes beyond the opt-in diagnostic line.
