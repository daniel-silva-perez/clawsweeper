## Summary

Keep open. Current main still has the central embedded-run abort path and the separate abort triggers described in the issue, but it does not implement `OPENCLAW_LOG_ABORT_SOURCES`, `[abort-source]` logging, source classification, or stack-frame capture. The linked implementation PR #74110 is closed unmerged, so it remains useful prior art rather than shipped behavior.

## Best Possible Solution

Keep this open for maintainer product and operability review. If accepted, revive or adapt #74110 as a narrow core change: add a helper under `src/agents/pi-embedded-runner/run`, call it once from `abortRun()`, classify external signal, LLM idle timeout, compaction timeout, run timer, and explicit cancel, test env gating/default-off behavior, and avoid runtime behavior changes beyond the opt-in diagnostic line.
