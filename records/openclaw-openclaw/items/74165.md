## Summary

Keep this open. Current main still lacks the requested opt-in abort-source diagnostic, while the centralized embedded-run abort path and the separate abort triggers described in the report are still present. The related implementation PR is closed unmerged, so it remains prior art rather than shipped behavior.

## What This Changes

Add opt-in `OPENCLAW_LOG_ABORT_SOURCES` structured logging for embedded run aborts, classifying abort source and capturing caller stack frames without changing default runtime behavior.

## Best Possible Solution

Keep the issue open for maintainer review. If accepted, implement the diagnostic as a narrow core runner helper called once from `abortRun()`, cover env gating and source classification with focused helper tests, and leave normal production logs unchanged by default.

## Reproduction Assessment

Not applicable as a normal bug reproduction; this is a feature request. The missing behavior is high-confidence to verify by exact symbol search and inspection of the current `abortRun()` and trigger paths on main.

## Solution Assessment

Unclear until maintainer acceptance. The proposed single central helper is likely the narrowest maintainable implementation if the core diagnostic is approved, but the env knob and stack logging policy should be decided first.
