## Summary

Keep this open. Current main still has the centralized embedded-run abort path and separate abort triggers, but it does not have the requested `OPENCLAW_LOG_ABORT_SOURCES` gate, `[abort-source]` marker, source classifier, stack-frame capture, or tests. The related implementation PR #74110 is closed unmerged, so it remains useful prior art rather than shipped behavior.

## Best Possible Solution

Keep this open for maintainer product and operability review. If accepted, revive or adapt #74110 as a narrow core change: add an abort-source helper under `src/agents/pi-embedded-runner/run`, call it once from `abortRun()`, classify external signal, LLM idle timeout, compaction timeout, run timer, and explicit cancel, test env gating and default-off behavior, and avoid any runtime behavior change beyond the opt-in diagnostic line.
