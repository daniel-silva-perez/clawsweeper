## Summary

Keep this PR open. Current main still drops Feishu card-action `form_value`, so the intended behavior is not implemented yet. The PR is a relevant Feishu-scoped bug-fix candidate, but it appears incomplete because the live parser still reconstructs `action` with only `value` and `tag`, so `form_value` would not reach the decoder/handler on real `card.action.trigger` callbacks. It should be reviewed alongside the broader open Feishu callback-field work in #42754, #42908, #43202, and #43953.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to either fold it into the broader Feishu callback-field fix already tracked by #42754/#42908/#43202/#43953, or revise it narrowly so the live parser preserves `action.form_value`, the decoder/handler/fallback consume it consistently, tests cover real `card.action.trigger` parser pass-through, and the command/text encoding policy is explicit.

## Risks / Open Questions

- The live Feishu parser path is not updated in the PR context, so the new handler/decoder fields may never see `form_value` from real callbacks.
- Appending raw `key=value` pairs to command text can be ambiguous for whitespace, arrays/objects, and command-parser-sensitive characters; a structured JSON payload or explicit escaping policy may be safer.
- Related open PRs #42908 and #43953 cover overlapping Feishu callback fields, so maintainers should avoid landing multiple partial fixes that encode the same data differently.
