## Summary

Keep this PR open. Current main still drops Feishu card-action `form_value`/selector sibling fields before command dispatch, so the intended behavior is not implemented on main. The latest PR context shows the branch now addresses the earlier parser pass-through gap, but it still needs maintainer review because it overlaps #42754, #42908, #43202, and #43953 and changes how user-controlled card fields are encoded into synthetic command text.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to choose one canonical Feishu card callback-field fix across #42754, #42908, #43202, #43953, and this PR: preserve missing fields in `parseFeishuCardActionEventPayload`, expose them through the decoder/handler consistently, decide whether command text or structured JSON is the supported payload format, and add parser pass-through plus handler tests for real `card.action.trigger` payloads.
