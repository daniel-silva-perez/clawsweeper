## Summary

Keep this PR open. Current main still does not implement the intended fix: `tools.subagents.tools.alsoAllow` is accepted and used to remove built-in subagent denies, but `alsoAllow`-only plugin tool entries are dropped before the plugin tool allowlist is built. The current PR remains useful and now appears to address the earlier review comments by preserving `alsoAllow` in both subagent policy constructors, the shared policy type, plugin-group expansion, and focused collector tests.

## Best Possible Solution

Review and land this PR, or supersede it with an equivalent maintainer patch. The fix should preserve `alsoAllow` through subagent policy construction, include it in plugin tool allowlist collection and plugin-group expansion, keep deny precedence intact, and ideally add native and Gateway regression coverage for an optional plugin tool allowed only by `tools.subagents.tools.alsoAllow`.
