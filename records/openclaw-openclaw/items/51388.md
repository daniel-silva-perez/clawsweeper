## Summary

Keep this PR open. Current main still does not implement the intended fix: subagent `alsoAllow` is accepted in config and used only to remove default deny entries, then dropped before `pluginToolAllowlist` is built. Optional plugin tools listed only in `tools.subagents.tools.alsoAllow` can still be omitted from subagent native and gateway tool resolution.

## Best Possible Solution

Review and land this PR, or supersede it with an equivalent maintainer patch. The fix should preserve `alsoAllow` through both subagent policy constructors, include it in `collectExplicitAllowlist()`, preserve it during plugin-group expansion, and add focused tests for an optional plugin tool allowed only by `tools.subagents.tools.alsoAllow` on native and gateway resolution paths.
