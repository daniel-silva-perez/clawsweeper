## Summary

Keep this PR open. Current main still drops `tools.subagents.tools.alsoAllow` before optional plugin tool resolution: subagent policy constructors use `alsoAllow` only to remove default denies, then return no `alsoAllow` field, and `collectExplicitAllowlist()` only reads `allow`. The PR remains a focused candidate fix and the latest provided diff addresses the earlier bot review concerns by preserving `alsoAllow`, expanding it through plugin-group handling, and adding collector coverage.

## Best Possible Solution

Review and land this PR, or supersede it with an equivalent maintainer patch. The fix should preserve `alsoAllow` through subagent policy construction, include it in plugin tool allowlist collection and plugin-group expansion, keep deny precedence intact, and ideally add native and Gateway regression coverage for an optional plugin tool allowed only by `tools.subagents.tools.alsoAllow`.
