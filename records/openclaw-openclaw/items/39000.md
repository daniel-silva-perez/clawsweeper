## Summary

Current main still lacks the requested plugin-side gateway method caller. Plugins can register gateway RPC methods, and core has a private dispatcher used by gateway-bound runtime helpers, but neither `OpenClawPluginApi` nor `PluginRuntime` exposes `api.callGatewayMethod()` or `api.runtime.gateway.dispatchMethod()`. The related implementation candidate, PR #46485, is open and unmerged in the supplied GitHub context, so this issue should stay open for Plugin SDK/runtime design review.

## Best Possible Solution

Keep this issue open and handle it through Plugin SDK/runtime design. The best path is to review, revise, or replace PR #46485 with a documented, tested, constrained request/response seam, such as `api.runtime.gateway.dispatchMethod` with explicit caller identity and scope semantics, or a narrower typed capability/service API that satisfies inter-plugin reads without exposing the whole gateway RPC bus.
