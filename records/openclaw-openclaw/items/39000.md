## Summary

Keep this open. The request is still valid: current main supports plugin registration of Gateway RPC methods, but it does not expose a plugin-facing `api.callGatewayMethod()` or `api.runtime.gateway.dispatchMethod` caller. The linked metabolism/stability use case remains a real Plugin SDK/runtime design gap, and the related implementation candidate #46485 is still open, not merged.

## Best Possible Solution

Keep this issue open for Plugin SDK/runtime design. The best path is to review and either land or replace #46485 with a documented, tested plugin runtime request/response seam, possibly `api.runtime.gateway.dispatchMethod` or a narrower typed capability/service API, with explicit tests for plugin isolation, caller identity/scopes, plugin-registered handler dispatch, errors, lifecycle timing, recursion, and docs/API baseline coverage.
