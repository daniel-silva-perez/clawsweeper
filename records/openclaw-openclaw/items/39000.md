## Summary

Keep this open. Current main still has plugin registration of Gateway RPC methods and external dispatch, but no plugin-facing `api.callGatewayMethod()`, `api.runtime.gateway.dispatchMethod`, or equivalent generic in-process request/response API. The related open PR #46485 targets the same gap as `runtime.gateway.dispatchMethod`, but it is not merged and needs maintainer API/security review before this issue can be considered solved.

## Best Possible Solution

Keep this issue open for Plugin SDK/runtime design. The best path is to review and either land or replace #46485 with a documented, tested plugin runtime request/response seam, possibly `api.runtime.gateway.dispatchMethod` or a narrower typed capability/service API, with tests for plugin isolation, caller identity/scopes, plugin-registered handler dispatch, errors, lifecycle timing, recursion, and docs/API baseline coverage.
