## Summary

Current main still does not expose `api.callGatewayMethod()` or `api.runtime.gateway.dispatchMethod` to plugins. Plugins can register gateway methods, and core has a private internal dispatcher used by specific runtime helpers, but the requested inter-plugin request/response API remains missing; related PR #46485 is open and unmerged.

## Best Possible Solution

Keep this issue open and route it through Plugin SDK/runtime design. The best path is to review, revise, or replace #46485 with a documented and tested plugin request/response seam, such as a constrained `api.runtime.gateway.dispatchMethod` with explicit caller identity and scope semantics, or a narrower typed capability/service API that satisfies inter-plugin reads without exposing the whole gateway RPC bus.
