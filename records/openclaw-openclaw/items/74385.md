## What This Changes

The issue reports that Terminal Hatch during `openclaw onboard` appears stuck or very slow before the embedded TUI returns, and should show loading/progress while initialization runs.

## Best Possible Solution

Review and land PR #75120 if it preserves existing send, wait, and stream states; otherwise merge an equivalent narrow TUI fix that marks post-connect refresh/history work as busy and adds focused regression coverage. Any remaining measured slowness in `refreshAgents()` or `loadHistory()` should be tracked separately.
