## Summary

Current main does not implement the PR’s central behavior: health and status session summaries still list store entries directly, so duplicate rows sharing the same sessionId can still appear. The PR remains a plausible implementation candidate, especially after the author addressed the stale first-seen dedupe review point, but it should get maintainer review because it also carries unrelated overflow-guidance behavior.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The best path is to review the freshest-entry dedupe in src/commands/health.ts and src/commands/status.summary.ts with the added regression tests, then split or explicitly approve the unrelated overflow-guidance changes before landing.
