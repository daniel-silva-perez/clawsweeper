## Summary

Keep open: current main still runs service-marker stale PID cleanup before any liveness probe, so the reported gateway-kill bug remains. The submitted patch is not merge-ready because it makes that safety decision through a PATH-resolved `curl` call, hard-codes the probe host, and does not make the healthy-existing-gateway path a clean no-op.

## What This Changes

The PR adds a `curl`-based HTTP `/health` probe in `src/cli/gateway-cli/run.ts` before service-marker stale PID cleanup and skips cleanup when that probe appears healthy.

## Best Possible Solution

Implement the health gate in the current service-marker startup path using an in-process probe against the resolved gateway probe host. A healthy existing gateway should return successfully without cleanup or bind attempts, while stale cleanup should run only after the probe fails, with focused coverage for healthy, unhealthy, and custom-bind cases.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Return after detecting a healthy existing gateway:** `src/cli/gateway-cli/run.ts:271-275`
  - body: In the healthy branch, the patch only logs that cleanup was skipped and then continues into normal gateway startup. That can still contend with the already-running gateway instead of making service-mode startup a clean success path.
  - confidence: 0.88
- **[P2] Probe the resolved gateway host:** `src/cli/gateway-cli/run.ts:234-238`
  - body: The new check always probes `127.0.0.1`, but gateway startup can bind through custom, tailnet, or other resolved hosts. A healthy gateway that is not reachable on loopback can still be treated as stale and killed.
  - confidence: 0.87
- **[P2] Remove the external curl dependency:** `src/cli/gateway-cli/run.ts:234-238`
  - body: If `curl` is missing or shadowed in `PATH`, the probe resolves false and the code proceeds to stale cleanup, so the correctness of the safety gate depends on an external executable even though Node can perform the HTTP probe in-process.
  - confidence: 0.91
