## Summary

Keep this PR open. Current main still calls service-mode stale gateway cleanup before any health probe, so the central bug is not fixed; however, this PR is not merge-ready because its healthy path falls through to startup, its probe is hard-coded to loopback, and it adds a PATH-resolved curl subprocess.

## What This Changes

The PR adds a service-mode HTTP health check in `src/cli/gateway-cli/run.ts` before `cleanStaleGatewayProcessesSync` and skips stale-PID cleanup when `GET /health` appears healthy.

## Best Possible Solution

Implement the health gate in the gateway startup path using OpenClaw’s resolved bind/probe host and built-in Node HTTP/fetch support. A healthy existing gateway should make service-mode startup return successfully; stale cleanup should run only after the probe fails, with focused regression tests for healthy, unhealthy, and custom-bind cases.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Return when the existing gateway is healthy:** `src/cli/gateway-cli/run.ts:271-275`
  - body: In the healthy branch, the PR only logs and then falls through to the normal gateway startup path. The existing healthy process still owns the port, so `runGatewayLoop` will hit the same lock/bind conflict instead of making service-mode startup a no-op.
  - confidence: 0.92
- **[P2] Probe the resolved bind host before cleanup:** `src/cli/gateway-cli/run.ts:235-238`
  - body: The new gate always checks `127.0.0.1`, but gateway startup can use a custom or non-loopback bind host. A healthy gateway that does not expose loopback will be treated as unhealthy and the stale-PID cleanup will still kill it.
  - confidence: 0.88
- **[P2] Remove the external curl dependency:** `src/cli/gateway-cli/run.ts:234-238`
  - body: If `curl` is unavailable or shadowed in `PATH`, the probe resolves false and the code proceeds to stale cleanup, so correctness now depends on an external executable even though Node can probe HTTP directly.
  - confidence: 0.9
