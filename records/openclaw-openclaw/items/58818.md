## Summary

Keep this issue open. Current main has adjacent continuity mechanisms, but it still lacks the requested configurable guarantee that the last N raw user/assistant messages survive both compaction and daily/idle/manual session reset as model context; the Apr 30 discussion points to related open reset/compaction work rather than an already-shipped fix.

## What This Changes

Request a configurable session raw-message tail guarantee that preserves at least the last N user/assistant messages across compaction and carries them into successor sessions after reset.

## Best Possible Solution

Define one schema-backed, generic session-continuity contract for raw-tail preservation, with a budget cap, an explicit clean-reset opt-out, docs, and tests covering manual reset, daily/idle rollover, compaction, and context-engine assembly.

## Reproduction Assessment

Yes for the current feature gap: source inspection shows reset creates a new session and archives the old transcript, startup context only loads memory files, and compaction has token-based tail behavior rather than a raw-message-count floor. A focused future regression test should assert that no previous raw transcript tail appears in a fresh reset context until this feature is implemented.

## Solution Assessment

Unclear. The requested direction is plausible, but the exact config shape, default N, clean-reset override, and plugin contract need maintainer design; the safer solution is an additive generic session/context contract rather than extending the session-memory hook alone.
