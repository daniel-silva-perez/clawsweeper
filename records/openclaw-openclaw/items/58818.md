## Summary

Keep this issue open. Current main has adjacent mechanisms for compaction tails, transcript archiving, reset hooks, and the session-memory hook, but it does not implement the requested user-facing guarantee that the last N raw messages survive both compaction and daily/idle/manual session reset as model context.

## Best Possible Solution

Keep this issue open and treat it as core session/context work. The best implementation would add typed and schema-backed session config for a bounded raw-message tail, enforce it in built-in compaction, define the context-engine contract, and seed daily/idle/manual reset successor sessions from the previous transcript while preserving an explicit clean-reset option. Add focused tests for manual `/new`/`/reset`, daily/idle rollover, transcript rotation, and context-engine assembly.

## Risks / Open Questions

- A future implementation needs an explicit clean-reset escape hatch because carrying raw messages across `/new` or `/reset` can violate a user's expectation of a fresh context.
- Context-engine plugins can already implement custom recall, but without a core contract they cannot prove the repository-wide guarantee requested here.
