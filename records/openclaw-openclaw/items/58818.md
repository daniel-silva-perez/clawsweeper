## Summary

Keep this issue open. Current main has adjacent continuity features, but it still lacks the requested configurable guarantee that the last N raw user/assistant messages survive both compaction and session reset; the Apr 30 discussion correctly points to related open reset/compaction work rather than an already-shipped fix.

## What This Changes

Request a configurable session-level minimum raw-message tail that compaction preserves and reset/startup carries forward into new agent context.

## Best Possible Solution

Define one generic session-continuity contract that covers reset seeding and compaction preservation, with schema/docs/tests, a clear cap when the model budget is too small, and an explicit opt-out for clean resets; then thread that contract through the default Pi runtime and context-engine APIs.

## Reproduction Assessment

Yes for the current gap: the reset path source and tests show default daily/idle expiry creates a new sessionId, while startupContext only loads memory files. A focused future test should assert that no previous raw transcript tail appears in a fresh reset context until this feature is implemented.

## Solution Assessment

Unclear. The requested behavior is plausible, but the exact config surface and privacy semantics should be settled before implementation; the safest direction is an additive generic contract rather than extending the session-memory hook alone.
