## Summary

Keep this open. Current main has adjacent memory and reset-continuity pieces, including daily-memory startup context and a small raw-tail replay helper for silent runtime rotations, but it still lacks the requested user-facing contract that a configurable last-N raw message tail survives normal compaction plus daily, idle, and manual session reset.

## What This Changes

The issue requests a configurable session continuity guarantee that preserves the last N raw user/assistant messages across compaction and reset boundaries.

## Best Possible Solution

Define an additive, generic session-continuity contract that covers reset seeding and compaction preservation, with config/schema/docs/tests, a budget cap when the model context is too small, and an explicit opt-out for clean resets; then thread it through the default runtime and context-engine API.

## Reproduction Assessment

Yes. A focused source-level repro can create an expired or explicitly reset session with a prior JSONL transcript, then verify the first prompt/new transcript is built from the new session plus memory-file startup context, not from the previous raw user/assistant tail.

## Solution Assessment

Unclear. The requested direction is plausible, but the current partial replay helper is too narrow and non-configurable; the maintainable solution should be a reviewed core contract rather than extending only session-memory or only the runtime failure reset helper.
