## Summary

Current main has absorbed most of the cron `delivery.threadId` plumbing, including the public gateway schema, but this PR is not fully obsolete. The PR head still contains a useful failure-destination deduplication fix that current main does not implement: `isSameDeliveryTarget` still compares only mode/channel/to/accountId and ignores a primary `delivery.threadId`, so topic-routed Telegram cron jobs can still suppress a same-chat failure destination.

## Best Possible Solution

Keep this PR open for maintainer review or a focused rebase. Preserve the already-landed cron `delivery.threadId` plumbing and gateway schema behavior on main, then port only the remaining failure-destination deduplication fix with a regression test proving that a primary same-chat Telegram topic target does not suppress a distinct failure destination.
