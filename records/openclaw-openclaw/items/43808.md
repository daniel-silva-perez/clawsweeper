## Summary

Keep open. Current main already implements the main cron `delivery.threadId` path for Telegram announce delivery, including schema/docs/tests, but it still lacks the PR head's remaining failure-destination dedupe guard: same-chat failure alerts are still considered identical to a topic-routed primary announce target because `isSameDeliveryTarget` ignores `threadId`.

## What This Changes

The PR branch adds cron `delivery.threadId` typing, normalization, gateway schema round-trip, isolated announce forwarding, Telegram forum-topic regression coverage, and a failure-destination dedupe regression test.

## Best Possible Solution

Keep the already-landed cron threadId support and add the missing failure-destination comparison behavior plus focused regression coverage so Telegram topic-routed cron jobs do not suppress valid failure notifications.
