## Summary

Keep open. The linked bug is plausible and current main still falls back from empty heartbeat config to the default cadence, but this PR changes only the summary helper while scheduler selection, system-prompt/bootstrap gating, and ACP spawn policy still use separate heartbeat enable/cadence logic. The branch also creates a new `{}`-means-disabled contract that is not yet documented as the public disable mechanism.

## What This Changes

The PR modifies `src/infra/heartbeat-summary.ts` to add an empty-object heartbeat helper and treat `heartbeat: {}` as disabled in summary/enabled/interval helpers.

## Best Possible Solution

If `{}` is accepted as disabled, the desired end state is a shared heartbeat disable/cadence predicate used by summary, scheduler, system-prompt/bootstrap filtering, ACP spawn policy, and tests, with docs updated to describe the new contract. If maintainers prefer the existing contract, the branch should be revised or closed in favor of documenting `0m` clearly on #64293.
