## Summary

Keep this PR open. Current main still has the reported empty-heartbeat behavior, and this PR is relevant to the linked bug, but the patch only changes the summary helper while other heartbeat policy paths would remain inconsistent.

## What This Changes

The PR modifies `src/infra/heartbeat-summary.ts` to add an empty-object heartbeat guard and have the summary/enabled/interval helpers treat `{}` as disabled.

## Best Possible Solution

If maintainers want `heartbeat: {}` to disable heartbeats, introduce one shared empty-heartbeat predicate and apply it consistently across summary, scheduler selection, runtime cadence checks, system-prompt/bootstrap gating, and ACP spawn policy, with regression tests and docs. If `0m` remains the only supported disable contract, close or revise this branch and document that outcome on the linked bug.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep empty-heartbeat policy shared with the scheduler:** `src/infra/heartbeat-summary.ts:36`
  - body: This makes the summary predicate ignore `heartbeat: {}`, but `resolveHeartbeatAgents` in the runner still treats any truthy per-agent heartbeat block as explicit and then merges defaults. With defaults plus `agents.list[].heartbeat: {}`, the scheduler can still run that agent at the default cadence while status/policy reports a different enabled set. Move the predicate into a shared helper and use it in runner selection too.
  - confidence: 0.88
- **[P2] Apply the disable rule to prompt and bootstrap gating:** `src/infra/heartbeat-summary.ts:64-66`
  - body: The new empty-object disable rule only reaches the summary/interval helpers. `shouldIncludeHeartbeatGuidanceForSystemPrompt` still falls back to `DEFAULT_HEARTBEAT_EVERY`, and bootstrap filtering uses that gate for `HEARTBEAT.md`, so a config meant to disable heartbeats can still inject heartbeat instructions/context on normal runs. Extend the shared rule to those paths and cover it with tests.
  - confidence: 0.82
