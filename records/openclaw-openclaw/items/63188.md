## Summary

Keep open. Current main preserves provider `stopReason: "length"` metadata, but native `mode: "run"` subagent completion still treats an `agent.wait` status of `ok` as a successful subagent outcome and has no configurable `onLengthTruncation` / `maxContinuations` / hard-fail policy. The issue has fresh production and local-model reproductions plus related #63210/#63689 context, so it is actionable rather than stale or already implemented.

## Best Possible Solution

Keep this open and implement a narrow first PR for native `mode:"run"` subagents: when the child terminal assistant has `stopReason:"length"`, the parent/orchestrator should receive an error/partial signal instead of `{status:"ok"}`. The lowest-risk first step is an opt-in fail policy with config/schema/docs/tests; continue-mode should wait until replay safety and side-effect semantics are specified.
