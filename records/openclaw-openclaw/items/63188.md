## Summary

Keep open. Current main has adjacent recovery for empty, reasoning-only, and zero-output error turns, but it does not implement the requested configurable stopReason="length" handling for mode:"run" subagent sessions. The issue discussion also adds a second provider-class reproduction and explicitly relates the same root cause to #63210 and #63689, so this is still actionable rather than stale.

## Best Possible Solution

Keep this open and have the agents/subagents maintainer decide the product boundary. A good first implementation path is a narrow fail-mode signal for stopReason="length" in native mode:"run" subagent runs, with tests proving the parent/orchestrator receives a failed run instead of a successful partial result. Continue-mode should follow only after replay-safety and side-effect semantics are explicit.
