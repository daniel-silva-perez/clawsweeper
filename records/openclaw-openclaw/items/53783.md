## Summary

Keep open. Current main still has agent-scoped Telegram group session keys and generic cross-agent session-tool guards, and I did not find a source, doc, or test change that proves two agents in the same Telegram group can list and send to each other symmetrically without the configured cross-agent policy. The related #59933 remains distinct because it covers same-agent group listing rather than this cross-agent one-way visibility/send path.

## Best Possible Solution

Keep open. Reproduce with two configured agents in the same Telegram group under explicit `tools.sessions.visibility="all"`, enabled and allowlisted `tools.agentToAgent`, and a known sandbox clamp setting. If it still reproduces, add regression coverage proving both agents list both `agent:<agent>:telegram:group:<id>` keys and can `sessions_send` both directions, then fix the gateway/session-store/visibility path. If maintainers intend same-group cross-agent access to remain opt-in, improve the error diagnostics/docs so missing policy does not look like a one-way missing session.
