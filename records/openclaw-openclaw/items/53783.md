## Summary

Keep open. Current main still uses agent-scoped Telegram group session keys plus generic session-tool visibility guards, and I found no same-Telegram-group symmetry rule or regression coverage proving two agents in the same Telegram group can list and send to each other both directions. The related #59933 remains distinct because it covers same-agent group listing, not this cross-agent one-way visibility/send failure.

## Best Possible Solution

Keep open. Reproduce with two configured agents in the same Telegram group under explicit `tools.sessions.visibility="all"` and enabled, allowlisted `tools.agentToAgent`. If it still reproduces, add regression coverage proving both agents list both `agent:<agent>:telegram:group:<id>` keys and can `sessions_send` both directions, then fix the gateway/session-store/visibility path. Track #59933 separately because it is same-agent group listing, not this cross-agent mismatch.
