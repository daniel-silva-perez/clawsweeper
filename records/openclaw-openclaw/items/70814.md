## Summary

Keep this PR open. Current main already supports numeric Telegram chat IDs when they are passed correctly as `delivery.to`, but it does not implement this PR's intended compatibility path for a numeric target stored in `delivery.channel`. The proposed inference also needs maintainer review for allowFrom/explicit-target semantics.

## Best Possible Solution

Keep the PR open and review it as a focused cron-delivery compatibility fix. The best path is a narrowed implementation that either repairs legacy misplaced `delivery.channel` values in doctor/migration or performs runtime recovery while preserving intended allowFrom/implicit-delivery semantics, with tests for single-channel recovery, multi-channel ambiguity, and allowFrom behavior.
