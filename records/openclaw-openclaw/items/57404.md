## Summary

Keep this issue open. Current main still has no passive WebSocket agent lifecycle usage event: lifecycle docs and emitters cover start/end/error, usage is persisted and exposed through RPC/CLI/status or OpenAI-compatible SSE surfaces, and the related implementation PR #51317 is closed unmerged.

## Best Possible Solution

Keep this issue open and either revive/supersede PR #51317 or land a fresh additive protocol change: emit one normalized post-run lifecycle usage event after usage is known, include provider/model, accumulated usage, lastCallUsage, duration, and run/session identity, then update Gateway protocol docs plus focused tests for WebSocket broadcast behavior and each run path.
