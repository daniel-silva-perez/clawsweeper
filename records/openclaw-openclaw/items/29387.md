## Summary

Keep this open. Current main still loads bootstrap context from `workspaceDir` only and lacks an `agentDir` diagnostic, while the linked implementation PR is open and unmerged, so the issue should remain the canonical tracker until that PR is reviewed, landed, or superseded.

## What This Changes

The issue requests either per-agent bootstrap-file loading from `agentDir` or a clear warning when bootstrap-named files under `agentDir` are ignored.

## Best Possible Solution

Keep this issue open as the canonical tracker and review PR #72871 as the narrow Option B diagnostic path. If maintainers prefer Option A, define a tested merge/override contract for workspace and `agentDir` bootstrap files before implementation.

## Reproduction Assessment

Yes. The source path is high-confidence: bootstrap files are resolved from `workspaceDir` only, and the relevant runtime call sites do not provide `agentDir`; an `agentDir`-only `SOUL.md` or `AGENTS.md` will therefore not be injected or warned about on current main.

## Solution Assessment

Unclear for Option A because per-agent merge/override semantics need product agreement. Option B is the narrowest maintainable fix for the confirmed silent-failure bug: thread `agentDir` through the resolver call sites, warn on misplaced standard bootstrap files, and cover it with focused tests and docs.
