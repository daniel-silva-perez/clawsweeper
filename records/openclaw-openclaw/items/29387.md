## Summary

Keep this issue open. Current main still resolves bootstrap context from the workspace only, has no agentDir parameter or diagnostic in the bootstrap resolver, and the live GitHub metadata shows open PR #72871 as an unmerged implementation candidate for the warning-based fix. Earlier linked attempts #29450, #29545, and #52916 are closed unmerged, so they do not prove implementation on main.

## Best Possible Solution

Keep this issue open as the canonical tracker. Review and either land or supersede PR #72871 with a maintained fix that emits a clear warning for recognized bootstrap filenames found under agentDir but not injected, or deliberately define a tested agentDir/workspace merge or override contract across CLI, embedded runner, compaction, and command prompt paths. Update the public Agent workspace and Multi-agent docs to make the workspace-vs-agentDir bootstrap boundary explicit.
