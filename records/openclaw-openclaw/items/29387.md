## Summary

Keep this issue open. Current main still builds bootstrap context from the workspace path only, with no `agentDir` input, warning, merge, or override path; open PR #72871 is the active unmerged implementation candidate and should be reviewed or superseded before the issue closes.

## Best Possible Solution

Keep this issue open as the canonical tracker. Review, land, or supersede PR #72871 with a maintained fix that either emits a clear warning when recognized bootstrap files are present in `agentDir` but not loaded, or deliberately defines and tests an `agentDir`/workspace merge or override contract across CLI, embedded runner, and compaction bootstrap call paths.
