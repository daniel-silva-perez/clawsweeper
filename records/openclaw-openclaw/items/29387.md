## Summary

Keep this issue open. Current main still resolves and injects bootstrap context from `workspaceDir` only, has no `agentDir` parameter or misplaced-file diagnostic in the resolver or runtime call sites, and live PR #72871 remains open/unmerged with closing syntax for this issue. Earlier linked fix attempts are closed unmerged, so they do not prove implementation on main.

## Best Possible Solution

Keep this issue open as the canonical tracker. Review and either land or supersede PR #72871 with a maintained Option B warning, or deliberately define a tested Option A agentDir/workspace merge or override contract. The fix should thread `agentDir` through all bootstrap-context call sites, add focused regression tests, and update public docs for the workspace-vs-agentDir boundary.
