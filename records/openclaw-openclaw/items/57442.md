## Summary

Keep this issue open. Current main still does not show elapsed/session duration in the shared `/status` or `session_status` card, and the per-response usage footer still has no response-duration field. The related #68226 and #49762 remain useful narrower tracks, but they are open/unmerged and do not make this broader timing-observability request obsolete.

## Best Possible Solution

Keep this issue open as the broad timing-observability tracker. Implement session elapsed duration in the shared status renderer so both `/status` and `session_status` benefit, likely using `sessionStartedAt` with `resolveSessionLifecycleTimestamps()` fallback. Coordinate the per-response footer portion with #49762 or a replacement PR that preserves existing `/usage` mode semantics. Add focused coverage for the shared status footer and the response usage footer, plus docs if the user-visible footer changes.
