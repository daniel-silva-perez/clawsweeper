## Summary

Keep this issue open. Current main still does not show elapsed/session duration in the shared `/status` or `session_status` card, and it still does not show per-response duration in the user-visible usage footer. #68226 is a narrower open tracker for session start/duration in the status footer, while #49762 is an open, unmerged PR for response duration in the usage footer and still has review concerns, so neither is enough to close this broader report.

## Best Possible Solution

Keep this issue open as the broad timing-observability tracker. Implement session start/duration in the shared status renderer so both `/status` and `session_status` benefit, likely using `sessionStartedAt` with `resolveSessionLifecycleTimestamps()` fallback, and coordinate the per-response footer portion with #49762 or a replacement PR that preserves existing `/usage` formatting and mode semantics. Add focused tests for the shared status footer and usage footer, plus docs if the user-visible footer changes.
