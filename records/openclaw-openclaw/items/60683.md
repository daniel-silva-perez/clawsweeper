## Summary

Keep this PR open, but it is not ready to merge as-is. Current main already covers `local.contextSize`, while the PR still contains useful remaining work for `gpu`, `gpuLayers`, and `flashAttention`; however, the branch needs to account for provider-key reindexing, regenerated base config schema output, and current docs/config behavior before it can safely land.

## What This Changes

The PR adds `memorySearch.local` GPU backend, GPU layer, context-size, and flash-attention options for the local `node-llama-cpp` embedding provider, wires them through config/provider creation, and documents/tests the settings.

## Best Possible Solution

Refresh the branch on current main and keep the useful local llama controls, but make the normalized local runtime-affecting settings part of the local provider `cacheKeyData`, regenerate the base config schema and docs baselines as required, preserve current `contextSize` semantics, and keep localized docs changes out of this repo.

## Reproduction Assessment

Not applicable as a user bug report, but the review findings have high-confidence static reproductions: the PR adds runtime-affecting local options while the current provider key path only hashes provider/model, and the generated base schema file is not in the PR's changed files despite new runtime schema keys.

## Solution Assessment

No, not as currently written. Exposing these knobs is a reasonable direction, but the patch needs the reindex/cache-key and generated-schema follow-through before it is the narrow maintainable fix.
