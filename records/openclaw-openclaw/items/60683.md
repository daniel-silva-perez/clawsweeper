## Summary

Keep this PR open. Current main only implements the narrower `memorySearch.local.contextSize` support from later work; it still does not expose or pass through the PR's `gpu`, `gpuLayers`, or `flashAttention` local llama settings. The review comments about provider-key/reindex behavior and generated schema coverage also remain relevant to any refreshed version of this PR.

## Best Possible Solution

Keep this PR open and refresh it against current main: preserve the existing `contextSize` behavior, add `gpu`, `gpuLayers`, and `flashAttention` through the config types, strict schema, generated schema/help/docs, memory config merge, embedding provider option types, and node-llama calls; include the settings in the memory provider key/reindex trigger and add focused tests for schema acceptance, default/override merging, node-llama passthrough, and reindex/cache-key changes.
