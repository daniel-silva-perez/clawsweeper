## Summary

Keep this PR open. Current main still lacks `includedWorkDirs`, so the capability is not obsolete, but the latest PR head still has blocking correctness and security issues around filesystem-root allowlists, inconsistent image/PDF tool propagation, whitespace-only edit targets, and private QA loader behavior.

## What This Changes

The PR adds `agents.list[].includedWorkDirs` config/schema support and threads extra allowed roots through agent scope, workspace path inference, path guards, `apply_patch`, PI file tools, image/PDF media handling, tests, and some CI/QA workflow maintenance.

## Best Possible Solution

Land the capability after narrowing the branch to the agent workspace feature or clearly separating unrelated QA/workflow edits, rejecting filesystem roots for included dirs, deriving included roots consistently in every tool factory, preserving existing edit semantics, and adding focused regression tests plus user-facing config docs/changelog as needed.

## Reproduction Assessment

Yes. The blocking root-allowlist issue has a high-confidence code reproduction: configure `includedWorkDirs` with `/` or a drive root while `tools.fs.workspaceOnly` is enabled, then any absolute path can match that root through `resolvePathWithinRoots`. The image/PDF and edit regressions are also directly traceable from current call sites and adapter code.

## Solution Assessment

No. The proposed direction is maintainable, but this PR is not the best current implementation until it preserves workspace containment, resolves included roots inside `createOpenClawTools` when config/session context is available, keeps whitespace-only edits valid, and leaves the private QA loader's explicit gate intact.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Reject filesystem-root included work dirs:** `src/agents/agent-scope.ts:191-197`
  - body: `resolveAgentIncludedWorkDirs` accepts entries that resolve to `/` or a drive root and forwards them into the allowed-root list. In `tools.fs.workspaceOnly` mode, every absolute path is then inside an allowed root, so read/write/edit/apply_patch containment can be disabled by a config mistake. Reject filesystem roots before adding them to the allowlist.
  - confidence: 0.9
- **[P2] Derive included roots for image and PDF tools:** `src/agents/openclaw-tools.ts:148`
  - body: `createOpenClawTools` forwards `includedWorkDirs` only when callers pass the new option explicitly. Gateway and inline-command call sites already pass `config` plus the session/agent context but not this option, so configured included roots still will not work for image/PDF tools outside the coding-tools path. Resolve the agent's included roots in this factory when the option is omitted.
  - confidence: 0.86
- **[P2] Preserve whitespace-only edit targets:** `src/agents/pi-tools.read.ts:665-668`
  - body: The canonical edit adapter drops replacements whose `oldText` is whitespace-only. That silently discards valid edits targeting indentation or blank-line blocks, and if all edits are filtered it falls back to the legacy tool with the incompatible `{ path, edits }` payload. Accept whitespace-only strings as long as `oldText` is a string.
  - confidence: 0.88
- **[P2] Keep the private QA loader behind its flag:** `src/cli/program/private-qa-cli.ts:82-86`
  - body: When the source-checkout resolver returns null, the new no-params path still tries runtime QA imports. If a built QA module is present, the private QA CLI can load even when `OPENCLAW_ENABLE_PRIVATE_QA_CLI` is disabled, making command availability depend on local layout rather than the explicit gate.
  - confidence: 0.82
- **[P2] Only fall back when the candidate module is missing:** `src/cli/program/private-qa-cli.ts:87-90`
  - body: The runtime import loop treats every `ERR_MODULE_NOT_FOUND` as a missing candidate, but Node uses the same code when a dependency inside `qa-lab` is missing. That can mask packaging errors and load another candidate instead of failing on the broken module. Restrict fallback to errors for the candidate specifier itself.
  - confidence: 0.78
