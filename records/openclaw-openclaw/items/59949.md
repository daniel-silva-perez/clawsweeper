## Summary

Keep open. Current main does not have `includedWorkDirs` support, and the PR is a relevant implementation attempt, but the branch is not merge-ready because it changes filesystem access boundaries and still has concrete correctness/security issues in the proposed patch.

## What This Changes

The branch adds an `agents.list[].includedWorkDirs` config/schema field and threads extra allowed roots through agent scope, workspace path inference, path guards, `apply_patch`, PI read/write/edit tools, image/PDF media path handling, tests, and a few workflow/QA maintenance edits.

## Best Possible Solution

Land this capability only after the branch rejects filesystem-root included directories, derives included roots consistently for all agent tool entrypoints, preserves existing edit semantics, keeps private QA loading gated as intended, and adds focused regression coverage plus docs/changelog for the new config field.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Reject filesystem-root included work dirs:** `src/agents/agent-scope.ts:191-197`
  - body: `includedWorkDirs` entries are resolved and accepted without the root-directory guard used by workspace normalization. If an agent is configured with `/` or a drive root, `resolvePathWithinRoots` treats every absolute path as inside an allowed root, so `tools.fs.workspaceOnly` no longer contains read/write/edit/apply_patch for that agent. Reject filesystem roots before adding them to the allowlist.
  - confidence: 0.9
- **[P2] Derive included roots for image and PDF tools:** `src/agents/openclaw-tools.ts:148`
  - body: `createOpenClawTools` forwards `includedWorkDirs` only when the caller passes the new option explicitly. Gateway and inline-command call sites already pass `config` plus the session/agent id but not this option, so configured included roots still will not work for image/PDF tools outside the `createOpenClawCodingTools` path. Resolve the agent's included roots in this factory when the option is omitted.
  - confidence: 0.86
- **[P2] Preserve whitespace-only edit targets:** `src/agents/pi-tools.read.ts:665-668`
  - body: The canonical edit adapter drops replacements whose `oldText.trim().length === 0`. That silently discards valid edits that target indentation or blank-line blocks, and when all edits are filtered it falls through to the legacy tool with an incompatible `{ path, edits }` payload. Accept whitespace-only strings as long as `oldText` is a string.
  - confidence: 0.88
- **[P2] Keep the private QA loader behind its flag:** `src/cli/program/private-qa-cli.ts:82-86`
  - body: When the env flag is disabled, `resolvePrivateQaSourceModuleSpecifier` returns null, but the new no-params path still tries runtime QA imports and can load the private QA CLI if a built module is present. That bypasses the helper's explicit `OPENCLAW_ENABLE_PRIVATE_QA_CLI` gate and makes behavior depend on local layout.
  - confidence: 0.82
- **[P2] Only fall back when the candidate module is missing:** `src/cli/program/private-qa-cli.ts:87-90`
  - body: The runtime import loop treats every `ERR_MODULE_NOT_FOUND` as a missing candidate, but Node uses the same code when a dependency inside `qa-lab` is missing. In that case this can mask the actual packaging error and load another candidate instead of failing on the broken module. Restrict fallback to errors for the candidate specifier itself.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The PR expands agent filesystem access by configuration, and the current branch accepts filesystem roots as included work dirs, which can defeat workspace-only containment.

Concerns:

- **[high] Filesystem-root allowlist defeats workspace-only:** `src/agents/agent-scope.ts:191`
  - body: `resolveAgentIncludedWorkDirs` accepts `/` or a drive root and forwards it into the allowed-root boundary list. Because every absolute path is within that root, a single misconfigured included work dir can make workspace-only file tools effectively unrestricted for that agent.
  - confidence: 0.9
