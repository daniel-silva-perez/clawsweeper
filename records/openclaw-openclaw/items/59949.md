## Summary

Keep this PR open. Current main does not implement the requested `agents.list[].includedWorkDirs` capability: the config types/schema lack the field, path normalization does not treat it as a path list, agent workspace resolution still uses only the primary workspace, and the path-guarded tools are still rooted to a single workspace. The PR is a coherent repo-scope feature, but it needs maintainer review and security cleanup before it can land.

## Best Possible Solution

Keep the PR open and review it as a real feature candidate. The best path is a cleaned, focused implementation that adds `agents.list[].includedWorkDirs` across config types, zod/generated schema/help, path normalization, agent-scope resolution, multi-root path policy, read/write/edit/apply_patch/media tools, docs, and targeted security tests, while splitting or justifying unrelated workflow/private-QA changes and addressing the unresolved boundary-review comments.
