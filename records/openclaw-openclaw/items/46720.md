## Summary

Keep open. The PR is useful directionally, but it is stale against current main and still has concrete docs correctness gaps: non-mention group mode needs permission guidance, localized docs should not be edited in this repo, and the live setup fallback moved to `setup-surface.ts`.

## What This Changes

The PR replaces Feishu/Lark permission batch-import guidance in English and Chinese docs with a smaller tenant-scope JSON and changes the old Feishu onboarding helper to point at the channel docs instead of listing scopes.

## Best Possible Solution

Refresh or replace the PR against current main: keep scan-to-create as the primary English docs path, update only the English manual fallback plus `extensions/feishu/src/setup-surface.ts` with verified permission guidance, explicitly cover all-group-message permissions for `requireMention: false`, and leave localized docs generation to the docs pipeline.
