## Summary

Keep this PR open. Current main still lacks outbound Slack `reply_broadcast` support, and the PR is a useful narrow implementation attempt, but the proposed patch is not ready because it documents invalid message-tool parameters, accepts the flag for upload/media paths where it is not honored, and wires schema discovery through a stale surface relative to current main.

## What This Changes

The PR adds a Slack `replyBroadcast` parameter across outbound send plumbing, action dispatch, message-tool discovery, docs, and action-runtime test expectations for threaded Slack replies.

## Best Possible Solution

Keep this PR as the narrow outbound implementation candidate, but update it to the current Slack message-tool schema contribution API, document valid `message` tool parameters, forward `replyBroadcast` only for explicit threaded `chat.postMessage` sends, and cover schema, dispatch, send, chunking, and upload/media edge cases with focused Slack tests while #63904 continues tracking broader mirroring policy.
