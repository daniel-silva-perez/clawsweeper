## Summary

Keep open: current main still lacks outbound Slack `replyBroadcast`/`reply_broadcast` support, so the PR is not obsolete. The branch is also dirty against current main and still has blocking correctness issues in docs, upload/media behavior, and schema discovery wiring, so it needs maintainer review rather than cleanup closure.

## What This Changes

The PR adds a Slack `replyBroadcast` message-tool parameter and threads it through docs, action runtime, the action adapter, message dispatch, send payload construction, and tests so threaded Slack replies can request `reply_broadcast`.

## Best Possible Solution

Keep this PR as the narrow outbound implementation candidate, but update it to the current Slack message-tool discovery API, document valid message-tool params, honor or reject the flag for media uploads, and leave broader inbound mirroring policy in #63904.
