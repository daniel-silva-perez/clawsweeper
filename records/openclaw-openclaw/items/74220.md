## Summary

Keep this PR open. Current main already has the status fast-path behavior, but it still has the agents-side hang risk this PR targets: bare `openclaw agents` still falls through the Commander parent action, and text `agents list` still uses the plugin-preloading text-only startup policy. The provided PR discussion also shows maintainer content approval followed by a changelog-conflict rebase request, so this should proceed as the active implementation rather than be closed or replaced.

## What This Changes

This PR adds a bare `agents` route-first catalog entry, changes `agents list` startup policy from text-only plugin preload to never load plugins, expands agents routing/startup/list regression tests, and adds a changelog entry for the CLI text hang fix.

## Best Possible Solution

Ship the narrow agents startup/routing fix so `openclaw agents` and text `openclaw agents list` stay on read-only metadata paths without plugin runtime preload, while preserving the already-implemented status split where plain status is fast and `--deep`/`--all` keep live channel checks.
