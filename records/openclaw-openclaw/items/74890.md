## Summary

Keep open. Current `main` still has the macOS direct/local config-write gap: the Swift writer replaces `openclaw.json` before treating `gateway-mode-removed` as suspicious, and `ConfigStore.save` can fall back to that writer when the gateway `config.set` path fails. The browser Control UI and TypeScript gateway writer are guarded, so the remaining actionable area is the macOS Swift writer/fallback path.

## What This Changes

The issue asks for Dashboard/Control UI/macOS app config writes to preserve or reject removal of required config envelope fields such as `meta` and `gateway.mode`.

## Best Possible Solution

Direct/local macOS saves should share the gateway writer’s pre-commit safety contract: merge against a fresh config or reject/preserve required envelope fields before replacing `openclaw.json`, with focused Swift regression coverage for `OpenClawConfigFile.saveDict` and the `ConfigStore.save` local fallback.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to start with an existing config containing `gateway.mode: local`, then call `OpenClawConfigFile.saveDict` with a root whose `gateway` object lacks `mode`; current `main` writes that payload before only auditing `gateway-mode-removed`.

## Solution Assessment

Yes. The requested fix boundary is the narrowest maintainable path: centralize the Swift direct-write guard and fallback behavior, while leaving the browser Control UI `config.set` base-hash path and TypeScript gateway protections intact.
