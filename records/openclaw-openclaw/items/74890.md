## Summary

Keep this issue open. Current main still has a macOS direct/local config write path that can replace an existing config with a root missing `gateway.mode`; the stronger gateway RPC and TypeScript writer protections do not cover that fallback path.

## What This Changes

The issue requests that Dashboard/Control UI/macOS app config writes preserve required root envelope fields such as `meta` and `gateway.mode` instead of producing partial snapshots that trigger last-good restore churn.

## Best Possible Solution

The macOS app should share the same pre-commit safety contract as the gateway writer: direct/local saves should merge against a fresh config and reject or preserve required envelope fields before replacing `openclaw.json`, while leaving the browser Control UI `config.set` base-hash path intact.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to start with an existing config containing `gateway.mode: local`, then call `OpenClawConfigFile.saveDict` with a partial root whose `gateway` object lacks `mode`; current main writes the payload and only records `gateway-mode-removed` afterward.

## Solution Assessment

Yes. A narrow macOS writer guard is the right fix; the safer form is a shared check in `OpenClawConfigFile.saveDict` and/or the `ConfigStore` local fallback rather than caller-by-caller patches.
