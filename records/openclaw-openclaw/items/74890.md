## Summary

Keep this open. Current main still has a macOS direct/local config-save path that writes the supplied root before classifying `gateway-mode-removed` as suspicious, so the reported semantic clobber is still possible outside the guarded browser Control UI and TypeScript gateway writer paths.

## What This Changes

The issue asks for Dashboard/Control UI/macOS app config writes to preserve or reject removal of required envelope fields such as `meta` and `gateway.mode` instead of relying on auto-restore after a bad snapshot is committed.

## Best Possible Solution

Direct/local macOS saves should share the gateway writer’s pre-commit safety contract: merge against a fresh full config or reject/preserve required envelope fields before replacing `openclaw.json`, with focused Swift coverage for `OpenClawConfigFile.saveDict` and the `ConfigStore.save` local fallback.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to start with a config containing `gateway.mode: local`, then call `OpenClawConfigFile.saveDict` or the `ConfigStore.save` fallback with a root whose `gateway` object lacks `mode`; current main writes the payload and only records `gateway-mode-removed` afterward.

## Solution Assessment

Yes. The requested fix boundary is the narrow maintainable path: add a shared Swift guard or merge-preserve step in the direct writer/fallback while keeping the existing browser `config.set` base-hash path and TypeScript destructive-write rejection intact.
