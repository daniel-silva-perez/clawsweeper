## Summary

Keep this issue open. Current main still has the reported macOS direct-write gap: the Swift config writer can replace the config before treating `gateway-mode-removed` as suspicious, and `ConfigStore.save` can fall back from gateway validation to that direct writer after any gateway save failure. The browser Control UI and TypeScript gateway write paths are already guarded, so the remaining work is a narrow macOS writer/fallback repair.

## What This Changes

The issue asks for Dashboard/Control UI/macOS config write paths to preserve required root envelope fields such as `meta` and `gateway.mode` and reject partial snapshots before replacing the live config.

## Best Possible Solution

Direct/local macOS saves should share the gateway writer's pre-commit safety contract: merge against a fresh config or reject/preserve required envelope fields before replacing `openclaw.json`, with focused Swift regression coverage for `OpenClawConfigFile.saveDict` and the `ConfigStore.save` local fallback.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to start with an existing config containing `gateway.mode: local`, then call `OpenClawConfigFile.saveDict` with a root whose `gateway` object lacks `mode`; current main writes that payload before only auditing `gateway-mode-removed`.

## Solution Assessment

Yes. The narrow maintainable solution is to fix the macOS direct writer and local fallback behavior while leaving the already-guarded browser Control UI `config.set` base-hash path and TypeScript gateway rejection path intact.

## Repair Work Prompt

Fix the macOS/OpenClaw app config-loss bug where direct/local config writes can remove an existing `gateway.mode` or other required envelope data and trigger auto-restore churn. Keep the work scoped to the macOS-owned config mutation path unless a very small generic seam is strictly necessary. Make `OpenClawConfigFile.saveDict` preserve required envelope fields from a fresh on-disk config or reject before replacement when a write would remove existing `gateway.mode` or otherwise clobber mandatory root data. Ensure `ConfigStore.save` cannot bypass gateway `config.set` validation by falling back to a stale or partial direct write after gateway save failure. Audit direct `OpenClawConfigFile.saveDict` callers so the central writer protects app settings such as browser, workspace, session store, Tailscale, and AppState sync. Keep the browser Control UI and TypeScript gateway contracts intact; do not weaken `baseHash`, SecretRef resolution, or gateway-side destructive-write rejection. Add Swift regression tests showing direct saves and the local `ConfigStore.save` fallback preserve or reject when an existing `gateway.mode: local` would be stripped, while intentional local/remote mode changes still work. Add a single-line user-facing changelog fix entry if behavior changes, crediting `Thanks @ctbritt`.
