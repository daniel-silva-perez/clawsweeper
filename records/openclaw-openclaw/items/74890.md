## Summary

Keep this issue open. Current main still allows the macOS direct config writer and local-mode fallback to replace the live config with a root that drops `gateway.mode`; the TypeScript gateway writer and browser Control UI path are already guarded, so the remaining actionable bug is narrow and macOS-owned.

## What This Changes

The issue asks OpenClaw to make Dashboard/macOS config writes preserve mandatory config envelope fields such as `meta` and `gateway.mode` instead of writing partial snapshots that trigger auto-restore churn.

## Best Possible Solution

Direct/local macOS saves should share the gateway writer’s pre-commit safety contract: preserve or reject loss of required envelope fields before replacing `openclaw.json`, with focused Swift regression coverage for both the direct writer and the `ConfigStore.save` local fallback.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to seed a config containing `gateway.mode: local`, then call the Swift direct writer or local fallback with a root whose `gateway` object lacks `mode`; current main writes that payload and only records `gateway-mode-removed` afterward.

## Solution Assessment

Yes for the remaining fix boundary. The narrowest maintainable solution is a macOS direct-writer/fallback change that preserves or rejects destructive envelope loss before disk replacement, while leaving the already guarded browser Control UI and TypeScript gateway writer paths intact.
