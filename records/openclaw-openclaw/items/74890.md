## Summary

Keep open. Current main still has a macOS direct config-write path that can commit a config missing `gateway.mode` before classifying the write as suspicious, and `ConfigStore.save` can fall back to that path when the gateway `config.set` path fails. The browser Control UI and TypeScript gateway writer have base-hash/destructive-write protections, so the remaining gap is the macOS local fallback/shared Swift writer.

## What This Changes

The request is to make Dashboard/Control UI/macOS app config writes preserve or reject removal of required config envelope fields such as `meta` and `gateway.mode`.

## Best Possible Solution

Direct/local macOS saves should share the gateway writer’s pre-commit safety contract: merge against a fresh config or reject/preserve required envelope fields before replacing `openclaw.json`, with focused Swift regression coverage for `OpenClawConfigFile.saveDict` and the `ConfigStore.save` local fallback.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to start with a config containing `gateway.mode: local`, then call `OpenClawConfigFile.saveDict` with a root whose `gateway` object lacks `mode`; current main writes that payload before only auditing `gateway-mode-removed`.

## Solution Assessment

Yes. The requested fix boundary is the narrowest maintainable path: centralize the Swift direct-write guard and fallback behavior, while leaving the browser Control UI `config.set` base-hash path intact.
