## Summary

Keep this issue open. Current main still has a macOS direct/local config-write path that can commit a root config missing `gateway.mode`; the browser Control UI and TypeScript gateway writer protections do not cover that fallback path.

## What This Changes

The issue asks OpenClaw to prevent Dashboard/macOS config writes from stripping mandatory root envelope fields such as `meta` and `gateway.mode` before replacing `openclaw.json`.

## Best Possible Solution

Direct/local macOS saves should share the gateway writer’s pre-commit safety contract: merge against a fresh config or reject/preserve required envelope fields before replacing `openclaw.json`, with focused Swift regression coverage for `OpenClawConfigFile.saveDict` and the `ConfigStore.save` local fallback.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to start with an existing config containing `gateway.mode: local`, then call `OpenClawConfigFile.saveDict` with a root whose `gateway` object lacks `mode`; current main writes the payload and only records `gateway-mode-removed` afterward.

## Solution Assessment

Yes. The requested direction is the right fix boundary; the safest implementation is a shared guard in `OpenClawConfigFile.saveDict` and/or the `ConfigStore` local fallback rather than caller-by-caller patches, while leaving the browser Control UI `config.set` base-hash path intact.
