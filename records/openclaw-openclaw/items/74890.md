## Summary

Keep this issue open. Current main still has the macOS direct/local config-write gap: the Swift writer can replace `openclaw.json` before treating `gateway-mode-removed` as suspicious, and `ConfigStore.save` can fall back to that writer when the gateway `config.set` path fails. The browser Control UI and TypeScript gateway writer are already guarded, so the remaining actionable work is in the macOS Swift writer and fallback path.

## What This Changes

Dashboard/macOS config writers should preserve required config envelope fields such as `meta` and `gateway.mode`, or reject partial root writes before replacing `openclaw.json`.

## Best Possible Solution

Direct/local macOS saves should share the gateway writer's pre-commit safety contract: merge against a fresh config or reject/preserve required envelope fields before replacing `openclaw.json`, with focused Swift regression coverage for `OpenClawConfigFile.saveDict` and the `ConfigStore.save` local fallback.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to start with an existing config containing `gateway.mode: local`, then call `OpenClawConfigFile.saveDict` with a root whose `gateway` object lacks `mode`; current main writes that payload before only auditing `gateway-mode-removed`.

## Solution Assessment

Yes. The issue's suggested repair boundary is the narrowest maintainable path: fix the macOS direct writer/fallback behavior while leaving the already guarded browser Control UI `config.set` base-hash path and TypeScript gateway rejection path intact.
