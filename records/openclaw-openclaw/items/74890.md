## Summary

Keep this open. Current main still has an actionable macOS gap: the Swift direct config writer can commit a root that removed `gateway.mode` before it only audits `gateway-mode-removed`, and `ConfigStore.save` can still fall back to that direct writer after a gateway `config.set` failure. The browser Control UI and TypeScript gateway writer are already guarded, so the remaining fix should stay in the macOS direct/local save path.

## What This Changes

The issue asks OpenClaw to prevent dashboard/macOS app config writes from stripping required config envelope fields such as `meta` or `gateway.mode` and causing last-good restore churn.

## Best Possible Solution

Direct/local macOS saves should share the gateway writer's pre-commit safety contract: merge against a fresh config or reject/preserve required envelope fields before replacing `openclaw.json`, with focused Swift regression coverage for `OpenClawConfigFile.saveDict` and the `ConfigStore.save` local fallback.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to start with an existing config containing `gateway.mode: local`, then call `OpenClawConfigFile.saveDict` through the direct writer or local fallback with a root whose `gateway` object lacks `mode`; current main writes that payload before only auditing `gateway-mode-removed`.

## Solution Assessment

Yes. The best fix is a narrow macOS direct-writer/fallback change that preserves or rejects destructive envelope loss before disk replacement, while leaving the already guarded browser Control UI and TypeScript gateway writer paths intact.
