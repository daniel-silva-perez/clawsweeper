## Summary

Keep open. Current main still leaves the reported macOS direct/local config-write gap: the Swift writer commits a supplied root before treating `gateway-mode-removed` as suspicious, and `ConfigStore.save` can fall back to that writer when gateway `config.set` fails. The browser Control UI and TypeScript gateway writer are already guarded, so the remaining actionable repair is in the macOS Swift write/fallback path.

## What This Changes

The issue asks for Dashboard/Control UI and macOS app config writes to preserve mandatory root envelope fields such as `meta` and `gateway.mode`, and to reject partial root writes before replacing `openclaw.json`.

## Best Possible Solution

Direct/local macOS saves should share the gateway writer's pre-commit safety contract: merge against a fresh full config or reject/preserve required envelope fields before replacing `openclaw.json`, with focused Swift regression coverage for both `OpenClawConfigFile.saveDict` and the `ConfigStore.save` local fallback.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to start with an existing config containing `gateway.mode: local`, then call `OpenClawConfigFile.saveDict` or the `ConfigStore.save` fallback with a root whose `gateway` object lacks `mode`; current main writes that payload before only auditing `gateway-mode-removed`.

## Solution Assessment

Yes. The narrowest maintainable fix is to harden the Swift direct-write/fallback path while leaving the already guarded browser Control UI `baseHash` flow and TypeScript gateway write rejection behavior intact.
