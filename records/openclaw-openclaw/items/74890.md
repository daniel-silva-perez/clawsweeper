## Summary

Keep open. Current main still has the macOS Swift direct/local config-write gap: the writer replaces `openclaw.json` before treating `gateway-mode-removed` as suspicious, and the local `ConfigStore.save` fallback can bypass the guarded gateway write path. The browser Control UI and TypeScript gateway writer are already guarded, so the remaining actionable scope is narrow and valid.

## What This Changes

The issue asks for Dashboard/Control UI/macOS app config writes to preserve required root envelope fields such as `meta` and `gateway.mode`, and to reject invalid partial snapshots before replacing the live config.

## Best Possible Solution

Direct/local macOS saves should share the gateway writer’s pre-commit safety contract: preserve required envelope fields by merging against a fresh config or reject the write before replacement, and cover both `OpenClawConfigFile.saveDict` and the `ConfigStore.save` local fallback with focused Swift regression tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to start with a config containing `gateway.mode: local`, then route a root whose `gateway` object lacks `mode` through `OpenClawConfigFile.saveDict`; current main writes the payload before only auditing `gateway-mode-removed`.

## Solution Assessment

Yes. The best direction is a narrow Swift writer/fallback repair that matches the existing TypeScript gateway safety contract; broader gateway recovery or runtime restart work would not address the demonstrated writer that strips envelope fields.
