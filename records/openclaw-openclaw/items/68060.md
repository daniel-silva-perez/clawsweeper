## Summary

Keep open. Current main still does not implement the proposed smart memory extraction path, and the PR remains a scoped memory-lancedb feature rather than obsolete cleanup. The author addressed some early bot feedback, but the remaining review context points to real manifest, capture-safety, prompt-injection, and observability issues that should be resolved before merge.

## What This Changes

The PR adds an opt-in `smartExtraction` config path to the bundled `memory-lancedb` plugin, a new LLM smart-extractor module with parser tests, and an auto-capture branch that can decompose conversations into atomic memories through a chat-completion call.

## Best Possible Solution

Keep the PR open and require focused rework in the memory-lancedb plugin before merge: align the manifest, UI hints, docs, and runtime config; preserve the existing capture safety gate before any LLM call; treat stored memories as untrusted prompt data; make extraction failures observable or fall back safely; and add targeted config, parser, hook, and safety tests for the opt-in behavior.
