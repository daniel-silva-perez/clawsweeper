## Summary

Keep open. Current main still does not implement the requested runtime-default model sentinel, and this PR is a plausible implementation path, but the proposed patch has merge-blocking consistency issues around subagent session metadata and cron fallback classification, plus missing user-facing release/docs follow-through.

## What This Changes

The PR adds a shared `@default` model sentinel and routes cron payload/session/subagent overrides plus persisted model refs through it, with new unit and integration coverage for cron and subagent model selection.

## Best Possible Solution

Land a revised version only if maintainers accept the `@default` product behavior, with sentinel semantics applied consistently to cron selection, cron fallback policy, subagent session metadata, persisted override resolution, docs, changelog, and focused regression tests; then use the related runtime-default cron model issue as the canonical follow-up to close once merged.

## Reproduction Assessment

Yes. The PR defects are reproducible by calling the subagent planning path with `modelOverride: "@default"` and expecting `modelOverrideSource: "auto"`, and by comparing `resolveCronFallbacksOverride` for omitted `payload.model` versus `payload.model: "@default"` after the PR's selection change.

## Solution Assessment

No, not as currently patched. The sentinel helper is a reasonable narrow approach for the requested behavior, but the patch needs the metadata predicate, cron fallback policy, docs, and changelog tightened before it is the best maintainable solution.
