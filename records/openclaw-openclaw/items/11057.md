## Summary

Keep this issue open. Current main still does not provide native 1M Opus 4.6 support for Claude Max/browser-session auth: Anthropic 1M remains gated by `params.context1m`, OAuth/setup-token-style `sk-ant-oat` credentials still strip the 1M beta header, and the regression tests/docs preserve that boundary. The custom-provider workaround in the discussion is useful context, but it does not resolve the built-in provider request.

## Best Possible Solution

Keep this issue open until maintainers either implement native Claude Max/browser-session 1M support for Opus 4.6 in the built-in Anthropic provider, document an explicit API-key/custom-provider-only product boundary, or link a narrower canonical follow-up. The likely implementation should align Anthropic beta/header behavior, setup-token or Claude CLI auth semantics, context-window reporting/model listing, provider docs, and regression tests; coordinate with #66766/#66862 if that no-flag 4.6 work lands first.
