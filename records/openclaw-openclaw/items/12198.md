## Summary

Keep this open. Current main has partial safety scanners and path guards, but it still does not provide the shared prompt-bearing Markdown validation contract requested here. Workspace bootstrap files and SKILL.md content can still be loaded as raw Markdown, native ClawHub skill install copies downloaded skill trees after archive/root checks, and the existing built-in install scanner skips Markdown files.

## Best Possible Solution

Keep this open until maintainers define a shared prompt-bearing Markdown validation contract. The best path is to make rule/severity/override behavior explicit, reuse or align with the Skill Workshop scanner where appropriate, and apply the contract consistently to workspace bootstrap ingestion, extra bootstrap files, model-visible SKILL.md discovery/read paths, native ClawHub skill install/update, and ClawHub publish-side coordination, with focused tests and docs.
