## Summary

Keep this open. Current main still has path guards, size caps, code-focused install scanning, and the optional Skill Workshop scanner, but it does not implement a shared prompt-bearing Markdown validation contract for workspace bootstrap files, model-visible SKILL.md reads, or native ClawHub skill installs.

## What This Changes

The issue requests Markdown validation infrastructure for prompt-bearing files such as skills and workspace bootstrap Markdown, including prompt-injection rules like disallowing HTML comments and possible ClawHub publish/install enforcement.

## Best Possible Solution

Keep this open until maintainers define a shared prompt-bearing Markdown validation contract. The best path is to make rule, severity, override, and tagging behavior explicit; reuse or align with the Skill Workshop scanner where appropriate; and apply the contract consistently to workspace bootstrap ingestion, extra bootstrap files, model-visible SKILL.md discovery/read paths, native ClawHub skill install/update, and ClawHub publish-side coordination, with focused tests and docs.
