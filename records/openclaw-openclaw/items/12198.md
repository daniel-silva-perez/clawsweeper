## Summary

Keep open. Current main still lacks a shared prompt-bearing Markdown validation contract: workspace bootstrap files, extra bootstrap files, model-readable SKILL.md files, and native ClawHub skill installs can still flow into prompt-visible context after path, size, symlink, and frontmatter checks. Recent maintainer and ClawSweeper comments reached the same keep-open conclusion, and the current source still matches that gap.

## What This Changes

Request to add validation infrastructure for Markdown files used as system-prompt or skill context, especially to catch prompt-injection patterns such as HTML comments before local or ClawHub skill content becomes model-visible.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract and apply it consistently before bootstrap files, extra bootstrap files, normal SKILL.md reads, native ClawHub skill install/update, and ClawHub publish-side scan/tag behavior expose content to the model, with documented severities, overrides, warning/quarantine behavior, and focused tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to put an HTML comment or other candidate prompt-injection marker in AGENTS.md, BOOTSTRAP.md, or a SKILL.md file; current loaders return the raw content and the prompt/skill flow exposes it without a shared Markdown validator.

## Solution Assessment

Unclear. The requested direction is appropriate, but a one-off HTML-comment ban would be under-scoped; maintainers should first settle the shared contract, severity model, overrides, and ClawHub publish/install behavior.
