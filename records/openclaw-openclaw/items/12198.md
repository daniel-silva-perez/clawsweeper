## Summary

Keep open. Current main still has path, size, symlink, archive-root, and code-safety checks, plus an optional Skill Workshop scanner, but it still lacks a shared validation contract for prompt-bearing Markdown before workspace bootstrap files, extra bootstrap files, model-readable SKILL.md content, or native ClawHub skill installs become model-visible.

## What This Changes

The issue requests Markdown validation or linting for prompt-bearing files such as skill definitions and workspace bootstrap Markdown to reduce prompt-injection and security risks.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract and apply it consistently before workspace bootstrap files, extra bootstrap files, normal SKILL.md reads, native ClawHub skill install/update, and ClawHub publish-side scan/tag behavior expose content to the model. The contract should document severities, overrides, warning or quarantine behavior, ClawHub tagging, and focused regression tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to place an HTML comment or prompt-injection marker in AGENTS.md, an extra bootstrap file, a local SKILL.md, or a ClawHub-installed skill; current loaders read or install that Markdown without a shared validator before it can become model-visible.

## Solution Assessment

Unclear. The requested direction is valid, but an HTML-comment-only linter is under-scoped; the maintainable fix is a reusable policy contract with explicit enforcement and override behavior across all prompt-bearing Markdown paths.
