## Summary

Keep this open. Current main still has partial scanners and safety guards, but not the shared prompt-bearing Markdown validation contract requested here. Workspace bootstrap files and extra bootstrap files are read and injected with boundary/size checks only; SKILL.md discovery validates path/frontmatter but not Markdown policy; Skill Workshop has a narrow generated-skill scanner; and native ClawHub skill install/update still copies the downloaded skill tree after archive/root checks without invoking a reusable Markdown validator.

## Best Possible Solution

Keep this open until maintainers define and implement a shared prompt-bearing Markdown validation contract. The best path is to align or reuse the Skill Workshop scanner where appropriate, make rule/severity/override behavior explicit, and apply the contract consistently to workspace bootstrap ingestion, extra bootstrap files, model-visible SKILL.md discovery/read paths, native ClawHub skill install/update, and ClawHub publish-side coordination, with focused tests and docs for the chosen policy.
