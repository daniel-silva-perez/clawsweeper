## Summary

Keep this open. Current main still has boundary, size, symlink, code-scan, ClawHub install, and Skill Workshop protections, but it does not provide a shared validation contract for prompt-bearing Markdown across workspace bootstrap files, extra bootstrap files, SKILL.md reads, and native ClawHub skill install/update paths.

## What This Changes

The issue requests Markdown validation for files automatically exposed to prompts, especially skills and workspace bootstrap files, to detect prompt-injection or security patterns such as HTML comments.

## Best Possible Solution

Define a shared prompt-bearing Markdown validator module and apply it consistently to workspace bootstrap ingestion, extra bootstrap files, normal SKILL.md discovery/read paths, native ClawHub skill install/update, and ClawHub publish-side scan/tag policy, with documented severity, override, warning, and recovery behavior.
