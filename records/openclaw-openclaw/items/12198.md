## Summary

Keep open. Current main still lacks a shared prompt-bearing Markdown validation contract across workspace bootstrap files, extra bootstrap files, model-readable SKILL.md paths, and native ClawHub skill installs, matching the recent maintainer and ClawSweeper comments that kept this issue open.

## What This Changes

The issue requests mandatory or configurable validation for Markdown files that can become prompt-visible context, with HTML comments suggested as an initial prompt-injection rule.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract and apply it consistently before workspace bootstrap files, extra bootstrap files, normal SKILL.md reads, native ClawHub skill install/update, and ClawHub publish-side scan/tag behavior expose content to the model, with documented severities, overrides, warnings/quarantine, and focused tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to place an HTML comment or other candidate prompt-injection marker in AGENTS.md, BOOTSTRAP.md, an extra bootstrap file, or SKILL.md; current loaders return that Markdown content and prompt assembly or skill-read guidance exposes it without a shared validator.

## Solution Assessment

Unclear. The requested shared validation direction is appropriate, but the initial HTML-comment-only rule would be under-scoped; maintainers should first settle the reusable policy contract and ClawHub behavior.
