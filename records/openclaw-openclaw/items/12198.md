## Summary

Keep open. Current main still has path, size, symlink, ClawHub archive, and code-scanner protections, but it does not define or enforce a shared validation policy for Markdown that becomes model-visible through workspace bootstrap files or SKILL.md content; the recent maintainer/Codex comments reached the same conclusion and the inspected code still matches that gap.

## What This Changes

Request a mandatory or configurable validator for model-visible Markdown prompt files such as workspace bootstrap files and SKILL.md, including ClawHub publish/install behavior and optional read/write enforcement.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract and apply it consistently before workspace bootstrap files, extra bootstrap files, normal SKILL.md reads, native ClawHub skill install/update, and ClawHub publish-side scan/tag behavior expose content to the model, with documented severity, overrides, quarantine/warning behavior, and tests.
