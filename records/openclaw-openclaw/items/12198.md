## Summary

Keep open. Current main still has path, size, symlink, archive-root, code-safety, and optional Skill Workshop scanning protections, but it does not define one shared prompt-bearing Markdown validation contract for bootstrap files, extra bootstrap files, model-readable SKILL.md content, or native ClawHub skill installs. Recent maintainer and ClawSweeper comments reached the same conclusion, and the current source still supports that gap.

## What This Changes

The issue requests a Markdown validation/linting contract for model-visible prompt files, especially skill definitions and bootstrap Markdown, to reduce prompt-injection and security risks.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract and apply it consistently before workspace bootstrap files, extra bootstrap files, normal SKILL.md reads, native ClawHub skill install/update, and ClawHub publish-side scan/tag behavior expose content to the model. The contract should document severities, overrides, warnings or quarantine, ClawHub tagging, and focused regression tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to place an HTML comment or prompt-injection marker in AGENTS.md, an extra bootstrap file, a local SKILL.md, or a ClawHub-installed skill; current loaders read or install that Markdown without a shared validator before it can become model-visible.

## Solution Assessment

Unclear. The requested direction is valid, but an HTML-comment-only linter would be under-scoped; the maintainable fix is a reusable policy contract with explicit enforcement and override behavior across all prompt-bearing Markdown paths.
