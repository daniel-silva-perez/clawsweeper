## Summary

Keep this open. Current `main` still has boundary checks, size limits, code-oriented install scanning, ClawHub install path hardening, and the optional Skill Workshop scanner, but it does not have a shared prompt-bearing Markdown validation contract for workspace bootstrap files, extra bootstrap files, model-visible `SKILL.md` reads, or native ClawHub skill installs. The April 26 maintainer/Codex comment reached the same conclusion, and current `main` has not closed that gap.

## What This Changes

Request a Markdown validation layer for prompt-bearing files, especially skills, ClawHub skill publishing/install, and workspace bootstrap Markdown, to block or tag high-risk prompt-injection patterns such as HTML comments.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract, then apply it consistently to workspace bootstrap ingestion, extra bootstrap files, normal `SKILL.md` discovery/read paths, native ClawHub skill install/update, and ClawHub publish-side scan/tag behavior. Align with Skill Workshop and external-content warning patterns where appropriate, but make the public severity, override, warning/tagging, docs, and tests explicit.
