## Summary

Keep this open. Current main still has boundary checks, size caps, code-focused install scanning, and the optional Skill Workshop scanner, but it does not provide a shared prompt-bearing Markdown validation contract for bootstrap files, model-visible SKILL.md reads, or native ClawHub skill installs. The recent maintainer/Codex comment reached the same conclusion, and current main has not closed that gap.

## What This Changes

Request to add validation for Markdown files that are appended to prompts, especially SKILL.md and workspace bootstrap files, so prompt-injection patterns such as HTML comments can be blocked, tagged, or overridden consistently.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract, then apply it consistently to workspace bootstrap ingestion, extra bootstrap files, SKILL.md discovery/read paths, native ClawHub skill install/update, and ClawHub publish-side scan/tag behavior. Reuse or align with Skill Workshop’s rules where appropriate, but make the public severity, override, warning/tagging, tests, and docs explicit.
