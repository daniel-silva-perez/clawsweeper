## Summary

Keep open. Current main still has path, size, symlink, frontmatter, ClawHub archive, and code-scanner protections, but it does not define or enforce a shared validation policy for Markdown that becomes model-visible through workspace bootstrap files or SKILL.md content; the recent maintainer/Codex discussion reached the same conclusion and current source still matches that gap.

## What This Changes

The issue requests validation for Markdown files that are automatically included in system prompts, especially to catch prompt-injection patterns such as HTML comments before skills or bootstrap files become model-visible.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract and apply it consistently before workspace bootstrap files, extra bootstrap files, normal SKILL.md reads, native ClawHub skill install/update, and ClawHub publish-side scan/tag behavior expose content to the model, with documented severity, overrides, quarantine or warning behavior, and tests.
