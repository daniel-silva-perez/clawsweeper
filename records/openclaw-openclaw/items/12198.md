## Summary

Keep open. Current main still has path, size, symlink, code-scan, ClawHub install, and optional Skill Workshop protections, but it does not provide a shared validation policy for prompt-bearing Markdown before workspace bootstrap files or SKILL.md content become model-visible.

## What This Changes

Request a Markdown validation/linting layer for prompt files, especially skill definitions and other Markdown appended to system prompts, to flag or block prompt-injection patterns such as HTML comments and to support ClawHub policy.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract, then apply it consistently to workspace bootstrap ingestion, extra bootstrap files, normal SKILL.md discovery/read paths, native ClawHub skill install/update, and ClawHub publish-side scan/tag behavior, with explicit docs and tests for severity, overrides, warnings, quarantine/tagging, and recovery.
