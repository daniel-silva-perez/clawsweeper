## Summary

Keep this open. Current main still has partial safety controls, but no shared prompt-bearing Markdown validation contract across workspace bootstrap files, extra bootstrap files, local SKILL.md reads, and native ClawHub skill installs. The existing Skill Workshop scanner is scoped to generated workspace skill proposals, and the docs explicitly distinguish native ClawHub skill install from the scanner-backed installer-metadata path.

## Best Possible Solution

Keep this issue open until maintainers define a shared prompt-bearing Markdown validation contract. The implementation should likely reuse or align with the existing Skill Workshop scanner, then apply the contract consistently to workspace bootstrap ingestion, extra bootstrap files, model-visible SKILL.md reads/discovery, native ClawHub skill install/update handling, and ClawHub publish-side coordination, with focused tests and docs for severity/override behavior.
