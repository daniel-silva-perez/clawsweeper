## Summary

Keep open. Current main still has path, size, symlink, frontmatter, ClawHub archive, and code-scanner protections, plus the optional Skill Workshop scanner, but it does not provide a shared prompt-bearing Markdown validation contract across workspace bootstrap files, extra bootstrap files, model-visible SKILL.md reads, and native ClawHub skill install/update paths. Recent maintainer and ClawSweeper comments reached the same conclusion, and the current source still matches that gap.

## What This Changes

The issue requests Markdown validation for files that become prompt material, especially skills and bootstrap/workspace Markdown, to detect or block prompt-injection and security patterns such as HTML comments and to support ClawHub publish/install policy.

## Best Possible Solution

Define one shared prompt-bearing Markdown validation contract and apply it consistently before workspace bootstrap files, extra bootstrap files, normal SKILL.md reads, native ClawHub skill install/update, and ClawHub publish-side scan/tag behavior expose content to the model, with documented severities, overrides, warnings or quarantine behavior, and focused tests.
