## Summary

Keep this PR open. Current main already implements the safer MiniMax-style raw JSON `api_error` fallback behavior from the merged #49611 path, but it still does not implement this PR's explicit HTTP 520 status handling. The best path is a small rebase/rework that adds 520 to the transient/status classifiers and tests while preserving main's stricter `api_error` transient-signal guard.

## Best Possible Solution

Rebase or replace this PR with a narrow patch that preserves current main's transient-signal `api_error` classifier and adds HTTP 520 support in `TRANSIENT_HTTP_ERROR_CODES`, `classifyFailoverClassificationFromHttpStatus`, and focused tests for both raw leading-status and structured-status paths.
