## Summary

Keep this issue open. Current main still has no per-server bundle-MCP startup cooldown/backoff for rate-limited streamable HTTP or SSE server initialization; startup failures are logged and disposed, while the session runtime manager remains session-scoped. Related LLM cooldown work and the open broader MCP retry PR do not make this implemented on main.

## What This Changes

Add per-server exponential backoff, jitter, and attempt caps for rate-limited MCP bundle server initialization failures.

## Best Possible Solution

Keep this open and add a small shared bundle-MCP startup cooldown layer in `src/agents/pi-bundle-mcp-runtime.ts`: classify transient rate-limit initialization failures for streamable HTTP/SSE servers, record per-server attempts across session runtimes with jittered capped backoff and an hourly attempt cap, skip only cooling servers while keeping other MCP servers available, and reset state on successful initialization. Cover the runtime with targeted tests and update docs/changelog if the behavior becomes user-facing.
