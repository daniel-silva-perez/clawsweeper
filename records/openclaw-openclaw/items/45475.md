## Summary

Keep open. Current main still has no `channels.slack.ignoreSubtypes` or equivalent Slack config surface, and Slack `message_changed` / `message_deleted` events still resolve to subtype handlers that enqueue system events with a session key. The related `thread_broadcast` work is fixed on main, but it does not supersede this edit/delete suppression request.

## Best Possible Solution

Keep this issue open and implement a narrow opt-in Slack config key such as `channels.slack.ignoreSubtypes` with account inheritance, schema/type/docs coverage, and event-layer filtering before edit/delete subtype system-event enqueue. Preserve current defaults and the fixed `thread_broadcast` behavior.
