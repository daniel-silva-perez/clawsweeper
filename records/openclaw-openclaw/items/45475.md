## Summary

Keep open. Current main still maps Slack `message_changed` and `message_deleted` events to system events that can wake sessions, and there is still no `channels.slack.ignoreSubtypes` config surface in the Slack types, schema, docs, or runtime.

## Best Possible Solution

Keep this issue open and implement a narrow Slack-owned config feature: add a typed and documented `ignoreSubtypes` allowlist for `message_changed` and `message_deleted`, validate it in the Slack config schema, resolve it through Slack account/channel config, and short-circuit in `extensions/slack/src/monitor/events/messages.ts` after subtype resolution but before `authorizeAndResolveSlackSystemEventContext(...)` and `enqueueSystemEvent(...)`. Add regression tests proving ignored subtypes do not enqueue system events or call the message handler, while non-ignored subtype behavior and Assistant DM rehydration remain intact.
