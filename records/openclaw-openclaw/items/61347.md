## Summary

Keep open. Current main has useful partial controls for bootstrap context cost, including continuation-skip, total/per-file truncation, and heartbeat lightweight context, but it does not implement the requested per-file conditional injection, frontmatter/config-driven inject rules, lazy TOC mode, or budget-aware summarization. The open related #13968 tracks only the HEARTBEAT.md subset, so this broader request is not fully superseded.

## What This Changes

Request configurable selective, lazy, and budget-aware workspace bootstrap context injection to reduce per-turn token overhead.

## Best Possible Solution

Keep this open as a design/feature umbrella, but split implementation into narrower accepted contracts: land or resolve the HEARTBEAT.md on-demand behavior through #13968, then decide whether broader per-file policies belong in core bootstrap config, plugin hooks, or the context-engine interface with docs and focused regression tests.

## Reproduction Assessment

Not applicable as a feature request. The missing behavior is verifiable from current main by inspecting the fixed bootstrap loader, coarse run filters, and truncation-only budget path rather than by running a user-facing reproduction.

## Solution Assessment

Unclear. The existing continuation-skip, heartbeat.lightContext, and character budgets are good partial mitigations, but they are not the full selective/lazy/summarizing contract requested; a narrower HEARTBEAT.md fix plus a separate design for lazy bootstrap files would be safer than one broad patch.
