## Summary

Keep open. Current main has meaningful partial fixes for workspace-context token pressure, including global continuation/never injection modes, bootstrap character caps, heartbeat-only lightweight context, and isolated heartbeat sessions. It still does not implement the broader requested selective/lazy workspace-file injection: the core resolver still loads a fixed bootstrap file set as full file content, there is no per-file frontmatter/config policy for AGENTS/SOUL/USER/MEMORY/etc., no TOC-only mode, and no token-budget-aware summarization beyond truncation.

## Best Possible Solution

Keep this as the product/API tracker for selective workspace bootstrap injection. A good implementation would extend the bootstrap resolver and documented config/SDK surface with per-file injection policies, optional TOC/lazy mode, and budget behavior that can summarize or defer files instead of only truncating them, while retaining the existing heartbeat `lightContext`, `includeSystemPromptSection`, `contextInjection`, and bootstrap cap controls.
