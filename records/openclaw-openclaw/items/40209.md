## Summary

Keep this PR open. Current main still stores system-event queues under exact session-key casing, while forced heartbeat session resolution canonicalizes agent-scoped session keys to lowercase before peeking and consuming queued events. The linked bug report was closed as stale, but the source path remains current and this PR is still a focused implementation candidate.

## Best Possible Solution

Review and land this focused system-event session-key normalization fix, or an equivalent patch that normalizes the queue key used by enqueue, peek, drain, consume, and has-events. Include regression coverage for mixed-case channel enqueue/peek/drain and cross-case dedupe; narrow the normalization if non-agent system-event keys must remain case-sensitive.
