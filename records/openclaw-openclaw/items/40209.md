## Summary

Keep this PR open. Current main at 16906780fd2c481308ec4b80f96eb3b93d0add46 still stores system-event queues by exact session-key casing, while heartbeat session resolution canonicalizes agent-scoped session keys to lowercase before peeking and consuming queued events. The PR remains a focused implementation candidate for the plugin-facing mismatch described in linked #34338 and in the prior bot reviews.

## Best Possible Solution

Review and land a focused system-event session-key normalization fix with regression coverage for mixed-case channel enqueue/peek/drain and cross-case dedupe. If case-sensitive non-agent queue keys are an intended contract, scope normalization to recognized agent-scoped session keys instead of lowercasing every key.
