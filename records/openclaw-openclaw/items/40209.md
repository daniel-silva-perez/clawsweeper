## Summary

Keep this PR open. Current main still preserves exact session-key casing in the system-events queue while heartbeat forced-session resolution canonicalizes agent-scoped keys to lowercase before peeking and consuming queued events. The PR remains a focused implementation candidate for the linked heartbeat/channel-session bug, with no obvious security-sensitive surface in the provided diff.

## Best Possible Solution

Review and land this focused system-event session-key normalization fix, or an equivalent patch that normalizes the queue key used by enqueue, peek, drain, consume, and has-events. Include regression coverage for mixed-case channel enqueue/peek/drain and cross-case dedupe; narrow the normalization if non-agent system-event keys must remain case-sensitive.
