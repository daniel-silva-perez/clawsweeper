## Summary

Keep this PR open. Current main still stores system events under the caller's exact session-key casing, while the heartbeat forced-session path canonicalizes agent session keys to lowercase before peeking and consuming queued events. The PR remains a focused implementation candidate for the linked channel-session heartbeat bug, and its two-file diff does not introduce supply-chain or CI/security-sensitive changes.

## Best Possible Solution

Review and land this PR or an equivalent patch that normalizes the queue key consistently for enqueue, peek, drain, consume, and has-events. If non-agent system-event keys must remain case-sensitive, scope the normalization to canonical agent session keys and keep the mixed-case channel and cross-case dedupe regression coverage.
