## Summary

Keep this PR open. Current main still preserves caller casing for system-event session keys, while the heartbeat forced-session path canonicalizes agent session keys to lowercase before peeking queued events. The PR remains a focused implementation candidate for the linked channel-session heartbeat bug, and its supplied diff does not introduce CI, dependency, publishing, or other supply-chain/security-sensitive changes.

## Best Possible Solution

Review and land this PR or an equivalent patch that normalizes system-event queue keys consistently for enqueue, peek, drain, consume, and has-events. Keep the mixed-case channel lookup and cross-case dedupe regression coverage; if non-agent system-event keys must remain case-sensitive, normalize only canonical agent-scoped keys.
