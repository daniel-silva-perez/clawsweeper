## Summary

Keep this PR open. Current main still stores system events under the caller's exact session-key casing, while the heartbeat forced-session path canonicalizes agent session keys to lowercase before peeking/consuming queued events. The PR remains a focused implementation candidate for the linked channel-session heartbeat bug, and the supplied two-file diff does not show security-sensitive or supply-chain changes.

## Best Possible Solution

Review and land this focused system-event session-key normalization fix, or an equivalent patch that normalizes the queue key used by enqueue, peek, drain, consume, and has-events. Keep the regression coverage for mixed-case channel enqueue/peek/drain and cross-case dedupe, and narrow the normalization only if non-agent system-event keys must remain case-sensitive.
