## Summary

Keep this PR open. Current main at 891c7d9f1c1f5e0f52935fadfeacf695b0ef1eb8 still stores system-event queues under exact session-key casing, while heartbeat forced-session resolution canonicalizes agent session keys to lowercase before peeking/consuming queued events. The PR remains a focused implementation candidate for the linked channel-session heartbeat miss, even though linked #34338 was closed as stale.

## Best Possible Solution

Review and land a focused system-event session-key normalization fix with regression coverage for mixed-case channel enqueue/peek/drain and cross-case dedupe. If non-agent system-event keys are meant to remain case-sensitive, scope normalization to parseable agent-scoped session keys instead of lowercasing all queue keys.
