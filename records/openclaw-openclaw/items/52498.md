## Summary

Keep this issue open. Current main has machine-oriented inter-session provenance and sender-side tool detail for sessions_send, but it still does not provide the requested human-readable, filterable sender/recipient identity at the message/session history level.

## Best Possible Solution

Keep open and implement this in the core session/provenance path: add an additive display-safe source identity such as sourceSessionLabel/sourceAgentId/sourceDisplayName, resolve sourceSessionKey to labels for history/UI where possible, preserve sender-side target metadata, and add focused tests for receiving-session display plus sender-session tool/result display.
