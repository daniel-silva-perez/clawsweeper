## Summary

Keep open. Current main still does not capture OpenAI-compatible `response.model`/`chunk.model` from routing proxies, does not persist separate `resolvedModel`/`resolvedProvider` fields, and does not expose that proxy-resolved backend identity through `session_status`. Existing selected-vs-active fallback display is adjacent, but it covers OpenClaw-managed fallback state rather than LiteLLM/OpenRouter backend routing metadata.

## Best Possible Solution

Keep this open and implement the missing OpenAI-compatible transport/session pipeline: capture `response.model` and streamed `chunk.model` when present, thread requested and resolved identities through assistant output and `agentMeta`, persist requested-vs-resolved state on sessions, expose it in `GatewaySessionRow`, `session_status`, and status output, and add focused LiteLLM/OpenRouter-style tests proving silent proxy routing is visible in-band.
