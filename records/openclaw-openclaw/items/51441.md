## Summary

Keep open. Current main has adjacent selected-vs-active fallback display for OpenClaw-managed model fallback, but it still does not capture the backend model returned by OpenAI-compatible routing proxies such as LiteLLM/OpenRouter, nor does it expose separate resolvedModel/resolvedProvider/fallbackUsed fields in session rows or session_status. Ryce’s comment usefully frames this as session state integrity, not only UI observability, and the KeepMyClaw workaround/plugin context does not replace the missing core transport/session metadata path.

## Best Possible Solution

Keep this issue open and implement it in the OpenAI-compatible transport/session pipeline: capture response.model or chunk.model when present, thread it through assistant output and agentMeta as resolved model metadata without losing the requested alias, persist requested-vs-resolved state on sessions, expose it in GatewaySessionRow/session_status/status output, and add targeted LiteLLM/OpenRouter-style tests that prove silent proxy routing is visible in-band.
