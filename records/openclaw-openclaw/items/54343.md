## Summary

Keep open for maintainer review. Current main has not added the requested internal `operator.admin` gate to `/stop`, but this PR is not safe to merge as-is because it replaces the existing sender authorization gate with an internal-only scope helper and the proposed admin requirement conflicts with the current gateway abort scope model.

## What This Changes

The PR changes the auto-reply `/stop` handler to use `requireGatewayClientScopeForInternalChannel` with `operator.admin` instead of the existing `rejectUnauthorizedCommand` sender gate.

## Best Possible Solution

If maintainers want this hardening, keep `rejectUnauthorizedCommand(params, "/stop")`, add a correctly imported internal `operator.admin` scope check immediately after it, and cover internal `operator.write`, internal `operator.admin`, and external unauthorized sender behavior in focused tests.
