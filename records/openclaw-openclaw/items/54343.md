## Summary

Keep this PR open. Current main and the latest release still route `/stop` through the sender authorization gate only, so the PR’s intended internal `operator.admin` hardening is not implemented. The proposed branch is also not merge-ready because the diff replaces the existing sender gate with an internal-only scope helper, which would weaken external channel authorization, and the supplied latest diff still does not show the required helper import.

## What This Changes

The PR changes the auto-reply `/stop` command handler to use an internal gateway `operator.admin` scope gate instead of the existing sender authorization gate.

## Best Possible Solution

Keep this PR open for maintainer review and revision. If maintainers want the hardening, preserve `rejectUnauthorizedCommand(params, "/stop")`, import and add `requireGatewayClientScopeForInternalChannel` immediately after it for internal `operator.admin` enforcement, add focused `/stop` authorization tests, and explicitly confirm the intended policy relationship with `chat.abort` and `sessions.abort` remaining under `operator.write`.
