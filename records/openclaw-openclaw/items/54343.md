## Summary

Keep open for maintainer review. Current main has not implemented the requested internal `operator.admin` gate for `/stop`, but the PR branch is not safe to merge as-is because it replaces the existing sender authorization gate with an internal-only scope helper and leaves the `/stop` authorization policy unresolved against existing `operator.write` abort APIs.

## What This Changes

The PR changes the auto-reply `/stop` command handler to use `requireGatewayClientScopeForInternalChannel` with `operator.admin` instead of the existing `rejectUnauthorizedCommand` sender gate.

## Best Possible Solution

If maintainers want this hardening, keep the existing `rejectUnauthorizedCommand(params, "/stop")`, add a correctly imported `requireGatewayClientScopeForInternalChannel` check immediately after it for internal `operator.admin` enforcement, and add focused tests proving internal `operator.write` is blocked, internal `operator.admin` is allowed, and external unauthorized senders remain blocked.
