## Summary

Keep open. Current main and the latest release still route `/stop` through `rejectUnauthorizedCommand` only, so the PR’s intended internal `operator.admin` gate is not implemented. The PR remains plausible security hardening, but the submitted diff is not merge-ready because it replaces the external sender gate with an internal-only helper and still does not add the needed helper import.

## Best Possible Solution

Keep the PR open for maintainer review and revision. If maintainers want this hardening, preserve the existing `rejectUnauthorizedCommand(params, "/stop")` check, import and add `requireGatewayClientScopeForInternalChannel` immediately after it for internal-channel `operator.admin` enforcement, and add focused tests for low-scope internal rejection, admin success, and external sender authorization behavior. Also confirm the intended policy relationship with `chat.abort` and `sessions.abort` remaining under `operator.write`.
