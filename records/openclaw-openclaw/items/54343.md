## Summary

Keep this PR open for maintainer review. Current main still uses only the existing sender authorization gate for `/stop`, so the intended hardening is not implemented, but the submitted diff has blocking correctness and security issues before it can merge.

## What This Changes

The PR changes the auto-reply `/stop` handler in `src/auto-reply/reply/commands-session-abort.ts` to call `requireGatewayClientScopeForInternalChannel` with `operator.admin` instead of the existing `rejectUnauthorizedCommand` gate.

## Best Possible Solution

If maintainers want this hardening, revise the branch to preserve `rejectUnauthorizedCommand(params, "/stop")`, import and add `requireGatewayClientScopeForInternalChannel` immediately after it for internal-channel `operator.admin` enforcement, and add focused tests for internal `operator.write` blocked, internal `operator.admin` allowed, and external unauthorized senders still blocked.
