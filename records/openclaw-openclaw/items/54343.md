## Summary

Keep this PR open, but do not merge it as-is. Current main does not implement the proposed internal `operator.admin` hardening for `/stop`; however, the patch still has blocking correctness and security issues because it references an unimported helper and replaces the existing sender authorization gate with an internal-channel-only scope check.

## What This Changes

The PR changes the auto-reply `/stop` handler in `src/auto-reply/reply/commands-session-abort.ts` to call `requireGatewayClientScopeForInternalChannel` with `operator.admin` instead of the existing `rejectUnauthorizedCommand` sender gate.

## Best Possible Solution

If maintainers want this hardening, keep the existing `rejectUnauthorizedCommand(params, "/stop")` gate, import and add `requireGatewayClientScopeForInternalChannel` immediately after it for internal-channel `operator.admin` enforcement, and add focused tests for internal scope behavior plus external sender authorization.
