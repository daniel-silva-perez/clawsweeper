## Summary

Keep this PR open, but it is not merge-ready. Current main has not implemented the requested `/stop` internal admin gate, and the PR branch still has blocking correctness and security defects: it calls an unimported helper and replaces the existing sender authorization gate with an internal-only scope helper.

## What This Changes

The PR changes the auto-reply `/stop` handler in `src/auto-reply/reply/commands-session-abort.ts` to require `operator.admin` through `requireGatewayClientScopeForInternalChannel` instead of using the existing `rejectUnauthorizedCommand` sender gate.

## Best Possible Solution

If maintainers want this hardening, preserve `rejectUnauthorizedCommand(params, "/stop")`, import and add `requireGatewayClientScopeForInternalChannel` immediately afterward for internal-channel `operator.admin` enforcement, and add focused tests for internal `operator.write` blocked, internal `operator.admin` allowed, and external unauthorized senders still blocked.

## Reproduction Assessment

Yes for the PR defects: static inspection shows the helper is called without an import, and the helper contract shows non-internal channels bypass the new check. The underlying policy claim is only partially reproducible because current gateway abort RPC methods still use `operator.write`.

## Solution Assessment

No. Replacing the existing sender gate is not the best fix; the maintainable approach is an additive internal scope gate after the existing authorization check, subject to maintainer confirmation of the `/stop` admin policy.
