## Summary

Keep this PR open. Current main does not add an internal `operator.admin` gate to `/stop`, so the intended hardening is not obsolete, but the branch is not safe to merge because it calls an unimported helper and replaces the existing sender authorization gate with an internal-channel-only scope helper.

## What This Changes

The PR changes the auto-reply `/stop` handler in `src/auto-reply/reply/commands-session-abort.ts` to call `requireGatewayClientScopeForInternalChannel` with `operator.admin` instead of the existing `rejectUnauthorizedCommand` gate.

## Best Possible Solution

If maintainers want this hardening, preserve `rejectUnauthorizedCommand(params, "/stop")`, import and add `requireGatewayClientScopeForInternalChannel` immediately after it for internal-channel `operator.admin` enforcement, and add focused regression tests for internal `operator.write` blocked, internal `operator.admin` allowed, and external unauthorized senders still blocked.
