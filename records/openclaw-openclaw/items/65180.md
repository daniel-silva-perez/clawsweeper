## Summary

Keep open. Current main already supersedes the local model-run session-reuse half with a lean one-shot completion path, but the profile-local transcript fallback change is still not present: fresh fallback paths still call the default agent transcript resolver before persistence. The branch is also draft/outdated and its canary docs need maintainer judgment, so this should be reviewed or selectively cherry-picked rather than auto-closed.

## What This Changes

The PR changes local `infer model run --local` to use explicit ephemeral sessions through the older `agentCommand` path, changes fresh transcript fallback paths to use the sessions directory derived from `storePath`, adds focused tests, and adds canary baseline docs.

## Best Possible Solution

Keep the current lean local model-run implementation on main, then land a narrow transcript fallback change that resolves fresh fallback files through the sessions directory derived from `storePath`, with focused tests for profile-local stores and thread/topic paths. If the canary docs are kept, anonymize the node-specific details or move them out of published docs.
