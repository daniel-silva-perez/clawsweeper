## Summary

Keep open. Current main already makes local model runs stateless through the newer lean completion path, but the profile-local transcript fallback half of this PR is still not implemented. The PR is draft and partly outdated, with normal landing issues in changelog/docs, so it needs maintainer review or a selective rebase rather than cleanup closure.

## What This Changes

The PR changes local `infer model run --local` to use explicit ephemeral sessions through the older `agentCommand` path, changes fresh transcript fallback resolution to use the sessions directory derived from `storePath`, adds focused tests, and adds canary baseline docs.

## Best Possible Solution

Keep the current lean local model-run implementation, then land a narrow transcript fallback change that resolves fresh fallback files inside the sessions directory derived from `storePath`, with focused tests for profile-local stores and topic/thread paths. If the canary baseline docs are retained, make them generic and add the required changelog entry before merge.

## Reproduction Assessment

Yes. The remaining gap can be reproduced by tracing `resolveSessionTranscriptFile` with a profile-local `storePath` and a session entry without `sessionFile`; current main creates the fallback via `resolveSessionTranscriptPath`, which resolves against the default agent state root before persistence.

## Solution Assessment

No, not as a whole. The transcript fallback change is the right narrow direction, but the CLI half is obsolete on current main and the docs/changelog pieces need maintainer cleanup before this branch is mergeable.
