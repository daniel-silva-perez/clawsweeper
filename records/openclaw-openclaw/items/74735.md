## Summary

Keep this PR open for maintainer review. Current main does not implement session-scoped `bootstrap-extra-files`, but the proposed patch should not merge as-is because it removes the basename gate for every configured glob and can make existing broad glob configs inject arbitrary workspace files into model context.

## What This Changes

The PR adds exact session-key `sessions` entries to the bundled `bootstrap-extra-files` hook, allows extra bootstrap files with custom basenames, deduplicates/order-stabilizes extra file loading, and updates docs and tests.

## Best Possible Solution

Land one coordinated bootstrap-extra-files contract that supports exact session-scoped explicit paths and custom context files while preserving a safe migration path for legacy glob configs, with docs, tests, and changelog credit aligned with #39120 and #73917.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Keep legacy globs from injecting arbitrary files:** `src/agents/workspace.ts:738-740`
  - body: Removing the basename allowlist for all resolved patterns changes existing broad configs from loading only AGENTS/TOOLS-style files to reading every matched workspace file into bootstrap context. Workspace containment and size checks still allow in-workspace files, so this can expose unrelated files to the model unless arbitrary basenames are explicitly opted in or limited to exact custom paths/session entries.
