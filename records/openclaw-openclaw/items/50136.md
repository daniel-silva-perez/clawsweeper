## Summary

Keep this PR open. Current main still has the Windows cleanup behavior the PR is trying to harden: the restart path enables unknown-listener fallback on Windows, folds unknown listener PIDs into `staleGatewayPids`, and then terminates that same list. Main also still classifies any listener text containing `openclaw` as a gateway. The PR is not merge-ready as submitted because review comments identify warning-array and test-isolation issues, but it is not obsolete.

## Best Possible Solution

Do not auto-close. Maintainers should either ask the contributor to rebase and address the review findings, or replace it with a focused maintainer patch that separates diagnostic unknown listener PIDs from verified actionable gateway PIDs, tightens listener classification with parsed argv/executable checks, keeps the existing runtime-lag probe success behavior, and adds targeted Windows restart-health and stale-cleanup tests.
