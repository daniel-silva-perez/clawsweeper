## Summary

Keep this PR open. Current main still has the Windows restart cleanup behavior the PR targets: the daemon restart path enables unknown-listener fallback on Windows, unknown listener PIDs can still be merged into `staleGatewayPids`, and restart/update paths still terminate that combined list. Current main also still classifies any listener text containing `openclaw` as a gateway. The PR is not merge-ready as submitted, but it is not obsolete or implemented on main.

## Best Possible Solution

Do not auto-close. Maintainers should either ask the contributor to rebase and address the review findings, or replace it with a focused maintainer patch that separates diagnostic unknown-listener PIDs from verified actionable gateway PIDs, tightens listener classification using parsed argv/executable checks, preserves the existing runtime-lag probe success behavior, and adds targeted Windows restart-health and stale-cleanup tests. If #66627 is meant to be the canonical thread despite being closed unmerged, confirm and communicate that relationship before superseding this PR.
