## Summary

Keep this PR open. Current main only covers part of the proposal: it already treats a busy port plus successful local probe as healthy when runtime metadata lags, but it still has the central Windows cleanup risk the PR targets. The restart path still enables unknown-listener fallback on Windows, merges unknown listener PIDs into `staleGatewayPids`, then terminates those PIDs. Port listener classification also still treats any listener text containing `openclaw` as a gateway. The PR is not merge-ready as submitted because review comments found warning-array and test-isolation problems, but current main has not made the PR obsolete.

## Best Possible Solution

Do not auto-close. Maintainers should either ask the contributor to rebase and address the review findings, or replace it with a focused maintainer patch that separates diagnostic unknown listener PIDs from verified actionable gateway PIDs, tightens listener classification with parsed argv/executable checks, keeps the existing runtime-lag probe success behavior, and adds targeted Windows restart-health and stale-cleanup tests.
