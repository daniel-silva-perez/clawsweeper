## Summary

Keep this issue open. Current main partially addresses the field report with clearer exec-host docs/schema help, background-task/process cleanup, MCP cleanup notes, and the newer Gateway stability diagnostics, but the central requests are not fully complete: invalid `tools.exec.host` enum values still do not get nearest-value guidance, fanout observability is still split across several surfaces, and the deterministic local maintenance-runner pattern is not documented.

## What This Changes

The issue requests improved guidance for the `tools.exec.host` config enum, better operator visibility into runtime/process/MCP/session fanout during memory spikes, documentation for model-free recurring maintenance via local runners, and consideration of inline dashboard log review UX.

## Best Possible Solution

Split this field report into focused maintainer-triaged follow-ups: add a small invalid-enum hint for `tools.exec.host` if that UX is desired, design one operator status surface that joins task/subagent/session/process/MCP/runtime-load signals, document deterministic local runners for repetitive maintenance that does not need model judgment, and handle inline dashboard log review as a separate Control UI product decision.
