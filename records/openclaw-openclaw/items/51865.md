## Summary

Keep open. Current `main` still appears to report the documented high-risk node commands as unknown when they appear only in `gateway.nodes.denyCommands`: the audit builds its known-command set from `resolveNodeCommandAllowlist`, while the node policy intentionally excludes commands such as `camera.snap`, `screen.record`, `contacts.add`, `calendar.add`, and `sms.send` unless `allowCommands` opts them in. The related PRs in the provided context were closed unmerged, and current source does not contain the described fix of adding `DEFAULT_DANGEROUS_NODE_COMMANDS` to the audit known-command set.

## Best Possible Solution

Keep the issue open and make the audit distinguish known command IDs from currently allowed command IDs. A minimal fix should add the documented dangerous node command IDs, preferably from `DEFAULT_DANGEROUS_NODE_COMMANDS` or a shared complete command registry, to `listKnownNodeCommands`, then add a regression test covering the issue's `denyCommands` example so those entries no longer produce `gateway.nodes.deny_commands_ineffective` unknown-command output.
