## Summary

Current main still has the reported security-audit false positive. `collectNodeDenyCommandPatternFindings` derives known command names from `listKnownNodeCommands`, and that helper still builds the set from `resolveNodeCommandAllowlist` only; the documented dangerous node commands remain excluded from default allowlists unless `allowCommands` opts them in. The related fix PRs in the provided thread context were closed unmerged, and current tests do not cover deny-only dangerous commands as valid known IDs.

## Best Possible Solution

Keep this issue open. The best fix is to separate known command IDs from currently allowed command IDs: add `DEFAULT_DANGEROUS_NODE_COMMANDS` or a shared complete node-command registry to the audit known-command set, then add a regression test in `src/security/audit-node-command-findings.test.ts` covering deny-only `camera.snap`, `camera.clip`, `screen.record`, `calendar.add`, `contacts.add`, and `sms.send` so they do not produce `gateway.nodes.deny_commands_ineffective` unknown-command output.
