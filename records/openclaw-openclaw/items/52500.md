## Summary

Keep open. Current main at 955b4df0939f90b4b6de5e7e6f3149dd125e1e50 still maps numeric Zod ceiling/floor failures to the native message without adding the explicit maximum/minimum hint requested here, and open PR #73220 already targets this issue. Per cleanup policy, the issue should remain open until that PR is reviewed and merged or closed.

## Best Possible Solution

Keep this issue open and review PR #73220 against current `src/config/validation.ts`. If the PR is sound, land it with regression coverage for `too_big` and `too_small` numeric bounds. If it is not acceptable, implement the same generic bound-hinting in the central mapper and keep tests in `src/config/validation.allowed-values.test.ts` or a focused sibling test file.
