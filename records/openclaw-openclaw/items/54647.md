## Summary

Keep this PR open. Current main still resolves ACP session rows through the generic session model path, so it can ignore `entry.acp.runtimeOptions.model` and serialize a configured default model for ACP rows with no explicit ACP runtime model. The PR targets the right gateway bug, but it needs current-main rework so it preserves newer explicit `modelProvider` behavior before merge.

## What This Changes

The PR changes gateway session model resolution and tests so ACP session list rows read `entry.acp.runtimeOptions.model` and avoid default-model fallback when no ACP runtime model is known.

## Best Possible Solution

Update the gateway session-row model resolution on current main so ACP rows use explicit ACP runtime metadata when present, leave model fields absent when the ACP runtime model is unknown, and retain current generic-session provider normalization with focused regression coverage.
