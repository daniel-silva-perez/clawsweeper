## Summary

Keep this PR open. Current main still reports ACP session rows through the generic session model resolver, so it can ignore `entry.acp.runtimeOptions.model` and fall back to the configured default model; the PR targets the right bug but is now dirty against main and needs to preserve newer explicit `modelProvider` behavior before it can land.

## What This Changes

The PR updates gateway session model resolution and regression tests so ACP session rows read `entry.acp.runtimeOptions.model` and omit default-model fallback when no ACP runtime model is known.

## Best Possible Solution

Update the gateway session-row model resolution on current main so ACP rows use explicit ACP runtime metadata when present, leave model fields absent when the ACP runtime model is unknown, and retain the current generic-session provider normalization behavior with focused regression coverage.
