## Summary

Keep this PR open. Current main still routes `sessions.list` through the generic session model resolver, which ignores `entry.acp.runtimeOptions.model` and falls back to configured/default model identity. The PR targets a real remaining ACP reporting gap, though the proposed patch needs review because a bot review flagged a provider-preservation bug in the diff.

## Best Possible Solution

Keep the PR open, rebase it onto current main, preserve explicit `modelProvider` when resolving slash-prefixed runtime models, and add focused regression tests for ACP rows with `entry.acp.runtimeOptions.model` and ACP rows with no explicit ACP model. After that, run the targeted gateway session-utils test lane and the normal changed gate.
