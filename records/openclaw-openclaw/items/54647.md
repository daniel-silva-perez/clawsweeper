## Summary

Keep this PR open, but it needs changes before merge. Current main still has the ACP session model-reporting bug, while the PR introduces a provider-preservation regression for stored wrapper providers with slash-prefixed model ids.

## What This Changes

The PR updates gateway session model resolution and tests so ACP session rows read persisted ACP runtimeOptions.model and omit generic default-model fallback when no explicit ACP model is known.

## Best Possible Solution

Update the current gateway session-row resolver so ACP rows use explicit ACP runtime metadata when present, leave modelProvider and model absent when the ACP runtime model is unknown, and preserve existing generic-session provider handling, transcript fallback, live subagent model handling, selected-model display, and display normalization with focused regressions.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call listSessionsFromStore with an ACP session key and entry.acp.runtimeOptions.model; current main routes through the generic resolver and never reads that ACP metadata. A second check is an ACP row with no generic or ACP model metadata, which currently falls back through DEFAULT_MODEL.

## Solution Assessment

No. The PR targets the right narrow surface, but it is not the best implementation because it re-parses prefixed model ids even when an explicit provider is already stored; the safer fix keeps explicit providers authoritative while adding the ACP-only no-default path.
