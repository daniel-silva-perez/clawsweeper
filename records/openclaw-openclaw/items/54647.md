## Summary

Keep this PR open. Current main still ignores `entry.acp.runtimeOptions.model` when building `sessions.list` rows and still falls back to the configured/default model for ACP rows with no explicit ACP runtime model. The PR targets the right bug from linked #54640, but it needs to be updated over current main's explicit `modelProvider` preservation behavior before it is safe to merge.

## What This Changes

The PR updates gateway session model resolution and regression tests so ACP session rows read `entry.acp.runtimeOptions.model` and omit default-model fallback when no ACP runtime model is known.

## Best Possible Solution

Keep this PR open and update it onto current main.
