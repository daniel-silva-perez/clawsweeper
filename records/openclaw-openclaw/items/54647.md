## Summary

Keep this PR open, but do not merge it as-is. The linked gateway bug is still present on current main, and the PR is directionally useful, but its resolver refactor introduces a blocking provider-preservation regression for explicit wrapper providers such as OpenRouter.

## What This Changes

The PR changes gateway session model resolution and tests so ACP session rows read `entry.acp.runtimeOptions.model` and omit default-model fallback when no explicit ACP runtime model is known.

## Best Possible Solution

Update the gateway session-row resolver so ACP rows use explicit ACP runtime metadata when present, leave model fields absent when the ACP runtime model is unknown, and preserve existing generic-session provider handling, transcript fallback, selected-model display, and display normalization with focused regressions.

## Reproduction Assessment

Yes: a high-confidence source-level reproduction is to call `listSessionsFromStore` with an ACP session key and `entry.acp.runtimeOptions.model`; current main routes through the generic resolver and never reads that ACP metadata. A second reproduction is an ACP row with no generic or ACP model metadata, which falls back to `DEFAULT_MODEL` through `buildGatewaySessionRow`.

## Solution Assessment

No: the proposed direction is the right narrow place to fix the bug, but this PR is not yet the best implementation because it re-parses prefixed model ids even when an explicit provider is already stored. The safer fix is to keep explicit providers authoritative while adding the ACP-only no-default path.
