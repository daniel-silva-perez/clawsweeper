## Summary

Keep this PR open. Current main still does not read ACP runtime model metadata for session rows, but the proposed branch has a blocking provider-preservation regression in its new model identity helper, so it should not merge as-is.

## What This Changes

The PR updates gateway session model resolution and tests so ACP session rows read `entry.acp.runtimeOptions.model` and omit default-model fallback when no explicit ACP runtime model is known.

## Best Possible Solution

Update the current gateway session-row resolver so ACP rows use explicit ACP runtime metadata when present, leave `modelProvider` and `model` absent when the ACP runtime model is unknown, and preserve existing generic-session provider handling, transcript fallback, selected-model display, and display normalization with focused regressions.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `listSessionsFromStore` with an ACP session key and `entry.acp.runtimeOptions.model`; current main routes through the generic resolver and never reads that ACP metadata. A second reproduction is an ACP row with no generic or ACP model metadata, which falls back to `DEFAULT_MODEL` through `buildGatewaySessionRow`.

## Solution Assessment

No. The PR targets the right narrow surface, but it is not the best implementation because it re-parses prefixed model ids even when an explicit provider is already stored; the safer fix keeps explicit providers authoritative while adding the ACP-only no-default path.
