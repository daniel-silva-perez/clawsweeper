## Summary

Keep open. Current main still serializes ACP rows through the generic model path, so the underlying bug remains, but the proposed patch should not merge as-is because it would regress explicit wrapper providers for vendor-prefixed model ids; the GitHub API also reports the PR branch as dirty.

## What This Changes

The branch updates gateway session model resolution and colocated tests so ACP session rows read `entry.acp.runtimeOptions.model` and omit the default-model fallback when no ACP runtime model is recorded.

## Best Possible Solution

Update the current gateway session-row resolver so ACP rows use explicit ACP runtime metadata when present, leave `modelProvider` and `model` absent when the ACP runtime model is unknown, and preserve existing generic-session provider handling, transcript fallback, selected-model display, and display normalization with focused regressions.
