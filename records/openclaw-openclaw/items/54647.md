## Summary

Keep open. The ACP sessions-list bug is still present on current main, but this PR should not merge as-is because the new resolver reparses prefixed model strings even when an explicit provider is already stored, which can rewrite wrapper providers such as `openrouter` to the vendor prefix provider.

## What This Changes

The PR changes gateway session model resolution and tests so ACP session rows read `entry.acp.runtimeOptions.model` and omit the default-model fallback when the ACP runtime model is unknown.

## Best Possible Solution

Update the current gateway session-row resolver so ACP rows use explicit ACP runtime metadata when present, leave `modelProvider` and `model` absent when the ACP runtime model is unknown, and preserve existing generic-session provider handling, transcript fallback, selected-model display, and display normalization with focused regressions.
