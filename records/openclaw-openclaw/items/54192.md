## Repair Work Prompt

Fix openclaw/openclaw#54192. Current main can retry CLI fallback candidates with an existing reusable CLI backend session and still send the prompt again, which can duplicate user turns after rate-limit or timeout failures. Implement a CLI-owned fallback retry repair in the attempt/CLI runner path that prevents resubmitting the same prompt into an already-mutated resumed backend session while preserving the original task. Avoid a simplistic prompt-skip unless the backend transcript confirms the first attempt persisted the user turn; prefer a fresh/reseeded CLI session or backend-confirmed transcript repair. Keep the embedded runner repair behavior unchanged, and do not broaden provider-specific policy into core beyond the generic CLI seam needed here. Add focused regression coverage for timeout and rate-limit fallback retries with an existing cliSessionId. No changelog entry is required unless the final change is user-facing enough to describe as a bug fix; if added, follow the active changelog attribution rules.
