## Summary

Keep open. Current main and the latest release still append the workspace HEARTBEAT.md hint to configured heartbeat prompts, so the linked regression is valid; this PR is a focused implementation candidate but needs maintainer review and a rebase against the newer heartbeat due-commitment prompt flow before merge.

## What This Changes

The branch adds an `isUserCustomPrompt` guard to heartbeat workspace-hint injection and adds a focused unit test for custom, default, unrelated, and already-hinted prompts.

## Best Possible Solution

Land a rebased version that preserves user-authored heartbeat prompts verbatim, keeps workspace-path guidance for built-in and internally generated heartbeat prompts, and covers the real runner path for both default and configured prompt cases.

## Reproduction Assessment

Yes. Set a non-empty `agents.defaults.heartbeat.prompt` or per-agent `heartbeat.prompt` containing `HEARTBEAT.md`; current main resolves that configured string and then appends the workspace path hint, contradicting the docs' verbatim prompt contract.

## Solution Assessment

Yes, the intended guard is the narrow maintainable fix for the regression. The implementation should be rebased onto the current heartbeat runner so the guard also covers the due-commitment append path, with integration-style regression coverage if feasible.
