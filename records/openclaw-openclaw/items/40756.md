## Summary

Keep open: current main and the latest release still mutate heartbeat prompts that mention `HEARTBEAT.md` even though the docs say custom heartbeat prompts are sent verbatim, and the linked regression remains open. The PR targets a real current product gap.

## What This Changes

The PR exports the heartbeat workspace-hint helper, adds an `isUserCustomPrompt` guard so configured heartbeat prompts are not modified, and adds focused unit coverage for custom, default, unrelated, and already-hinted prompts.

## Best Possible Solution

Preserve user-authored heartbeat prompts verbatim while retaining workspace-path guidance for built-in default and internally generated heartbeat prompts, then land this PR or an equivalent refreshed patch that also covers the current-main due-commitment path.
