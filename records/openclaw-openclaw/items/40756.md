## Summary

Keep this PR open. Current main still appends the HEARTBEAT.md workspace-path hint based only on prompt text, including supported custom heartbeat prompts, and the linked regression #40255 remains open. The PR remains a focused implementation candidate, with the main maintainer decision being the prompt-safety tradeoff raised in review comments.

## Best Possible Solution

Keep this PR or #40255 open. The best path is a small heartbeat-runner fix that distinguishes non-empty user-configured heartbeat prompts from built-in/generated heartbeat prompts, preserves the workspace-path hint and idempotency for default/internal prompts, adds a regression test for custom prompts mentioning HEARTBEAT.md, and explicitly resolves the prompt-safety tradeoff before merge.
