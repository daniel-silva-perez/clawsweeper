## Summary

Keep open: current main and the latest release still append the heartbeat workspace-path hint to any prompt mentioning HEARTBEAT.md, while the docs say custom heartbeat prompts are sent verbatim. The provided PR diff directly addresses the live regression linked from #40255 and has no blocking review finding from the available diff; it needs maintainer review and mergeability refresh rather than cleanup closure.

## What This Changes

The PR exports the heartbeat hint helper for testing, adds an `isUserCustomPrompt` guard so user-configured heartbeat prompts are not mutated, derives that guard from configured heartbeat prompt values when no exec/cron event is active, and adds focused unit coverage.

## Best Possible Solution

Land this narrow fix, or replace it with an equivalent maintainer-owned patch, so user-authored heartbeat prompts remain verbatim while default and internally generated heartbeat prompts keep workspace-path guidance. Treat stronger hint de-duplication as a separate follow-up unless maintainers want to expand the scope deliberately.
