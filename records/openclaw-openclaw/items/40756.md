## Summary

Keep open. Current main and the latest release still append the workspace HEARTBEAT.md hint whenever the resolved heartbeat prompt mentions HEARTBEAT.md, while the docs say custom heartbeat prompts are sent verbatim. The PR targets that remaining regression with a small guard and focused tests; I did not find a blocking functional or security issue in the current diff, so this needs normal maintainer review and merge-or-replace handling.

## What This Changes

The PR exports the heartbeat hint helper for testing, adds an `isUserCustomPrompt` guard derived from non-empty configured heartbeat prompts when no exec/cron event is active, and adds a focused custom-prompt unit test.

## Best Possible Solution

Preserve user-authored heartbeat prompts verbatim while retaining workspace-path guidance for default and internally generated heartbeat prompts; handle stronger hint de-duplication or non-overridable path-safety policy as an explicit follow-up rather than silently mutating custom prompt text.
