## Summary

Keep open: current main and the latest release still append the workspace path hint to any heartbeat prompt that mentions HEARTBEAT.md, while the docs say custom heartbeat bodies are sent verbatim. This PR remains a valid implementation candidate for #40255 and needs maintainer review rather than cleanup closure.

## What This Changes

This PR changes heartbeat prompt resolution so non-empty user-configured heartbeat prompts skip the workspace-path hint while default/internal prompts keep it, and adds focused unit coverage for the custom-prompt guard and idempotency.

## Best Possible Solution

Review and either land this PR or a narrow replacement that preserves user-configured heartbeat prompts while retaining default/internal prompt hinting and idempotency. The maintainer decision should explicitly cover whether custom prompts need a separate non-overridable or tool-layer HEARTBEAT.md path guard before closing #40255.
