## Summary

Keep open. Current main and v2026.4.26 still append the HEARTBEAT.md workspace-path hint after resolving the configured/default heartbeat prompt, while the docs say custom heartbeat prompts are sent verbatim. This PR matches the open regression in #40255, and the related duplicate PRs were closed without a merged fix, so the remaining action is maintainer review rather than cleanup closure.

## What This Changes

The PR changes heartbeat prompt resolution so non-empty user-configured heartbeat prompts skip workspace-path hint injection while default/internal heartbeat prompts keep the hint path, and adds focused unit coverage for that behavior.

## Best Possible Solution

Preserve user-authored heartbeat prompts verbatim while retaining workspace-path guidance for default and internally generated heartbeat prompts, with any stronger hint de-duplication or path-safety enforcement handled explicitly rather than silently mutating custom prompt text.
