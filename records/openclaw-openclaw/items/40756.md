## Summary

Keep this PR open: current main and the latest release still append the workspace HEARTBEAT.md hint to configured heartbeat prompts, while the docs say custom heartbeat prompt bodies are sent verbatim. The PR targets the right regression and has no blocking review finding, but it needs maintainer review and likely a rebase so the guard covers the current heartbeat runner shape.

## What This Changes

The PR exports the heartbeat workspace-hint helper for tests, adds an `isUserCustomPrompt` guard so configured heartbeat prompts skip hint injection, and adds focused unit coverage for custom, default, unrelated, and already-hinted prompts.

## Best Possible Solution

Land a rebased version that preserves user-authored heartbeat prompts verbatim, keeps workspace-path guidance for built-in and internally generated heartbeat prompts, and adds regression coverage through the runner path so future heartbeat prompt changes cannot reintroduce the mutation.

## Reproduction Assessment

Yes. Set a non-empty `agents.defaults.heartbeat.prompt` or per-agent `heartbeat.prompt` containing `HEARTBEAT.md`; current main resolves that configured string and then appends the workspace path hint despite the verbatim prompt contract.

## Solution Assessment

Yes, the PR direction is the narrow maintainable fix for the regression. The implementation should be rebased onto current main so the guard also covers the due-commitment append path, with runner-level regression coverage if feasible.
