## Summary

Keep open. Current main still has the reported heartbeat behavior: it appends the workspace-path hint based only on prompt text and does so after resolving the heartbeat prompt, even though the docs describe custom heartbeat prompts as verbatim. This PR is a plausible fix for the open regression tracked in #40255, but it needs maintainer review rather than cleanup closure.

## What This Changes

The PR changes heartbeat prompt resolution so non-empty user-configured heartbeat prompts skip workspace-path hint injection, keeps internally generated prompts on the existing hint path, exports the helper for testing, and adds focused unit coverage for custom prompt preservation, default hinting, and de-duplication.

## Best Possible Solution

Land this PR or a narrow replacement that preserves user-configured heartbeat prompts verbatim while retaining workspace-path guidance for default/internal heartbeat prompts, with an explicit maintainer decision on whether path-confusion protection for custom prompts needs a separate enforcement point.
