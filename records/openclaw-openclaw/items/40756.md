## Summary

Keep this PR open. Current main still appends the HEARTBEAT.md workspace-path hint based only on prompt text, including supported user-configured heartbeat prompts, while the linked regression #40255 remains open. The PR is still a focused implementation candidate, but maintainers should resolve the prompt-safety tradeoff raised in review before merge.

## Best Possible Solution

Keep this PR or #40255 open. The best path is a small heartbeat-runner fix that distinguishes non-empty user-configured heartbeat prompts from built-in/generated heartbeat prompts, preserves workspace-path hinting and idempotency for default/internal prompts, adds a regression test for custom prompts mentioning HEARTBEAT.md, and explicitly resolves the path-disambiguation safety question before merge.
