## Summary

Keep open for maintainer review. Current main and the latest release still append the heartbeat workspace-path hint to configured prompts despite the docs saying custom heartbeat prompt bodies are sent verbatim; this PR targets the right regression and has supportive review context, but the branch is currently dirty against main and needs a rebase/mergeability refresh before it can land.

## What This Changes

The PR exports `appendHeartbeatWorkspacePathHint`, adds an `isUserCustomPrompt` guard derived from non-empty configured heartbeat prompts outside exec/cron event handling, and adds focused unit coverage for custom, default, unrelated, and no-duplicate prompt behavior.

## Best Possible Solution

Land a rebased version that preserves user-authored heartbeat prompts verbatim while keeping workspace-path guidance for built-in, exec, cron, and other internally generated heartbeat prompts, with regression coverage through the heartbeat runner path.

## Reproduction Assessment

Yes. Configure a non-empty `agents.defaults.heartbeat.prompt` or per-agent `heartbeat.prompt` containing `HEARTBEAT.md`; current main resolves that configured prompt and still appends the workspace path hint in `resolveHeartbeatRunPrompt()`.

## Solution Assessment

Yes, the PR direction is the narrow maintainable fix for the reported regression. The landed version should be rebased onto current main and pass the guard through the newer due-commitment append path as well as the ordinary default prompt path.
