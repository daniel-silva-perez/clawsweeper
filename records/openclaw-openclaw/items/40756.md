## Summary

Keep open: current main and the latest release still append the workspace HEARTBEAT.md hint to resolved heartbeat prompts without distinguishing user-configured prompts, while the docs say custom heartbeat prompt bodies are sent verbatim. The PR targets the right regression and should get maintainer review plus a current-main rebase/validation rather than cleanup closure.

## What This Changes

The PR exports the heartbeat workspace-hint helper for testing, adds an `isUserCustomPrompt` guard so configured heartbeat prompts skip hint injection, and adds a focused helper unit test.

## Best Possible Solution

Land a rebased version that preserves user-authored heartbeat prompts verbatim, keeps workspace-path guidance for built-in and internally generated heartbeat prompts, and adds regression coverage through the heartbeat runner path so future prompt changes cannot reintroduce mutation.

## Reproduction Assessment

Yes. Configure a non-empty `agents.defaults.heartbeat.prompt` or per-agent `heartbeat.prompt` containing `HEARTBEAT.md`; current main resolves that configured prompt and still appends the workspace path hint in `resolveHeartbeatRunPrompt()`.

## Solution Assessment

Yes, the PR direction is the narrow maintainable fix for the regression. The landed version should be rebased onto current main and pass the custom-prompt guard through the due-commitment append path as well as the ordinary default prompt path.
