## Summary

Keep this issue open. Current main still treats Google Chat `appPrincipal` and `channels.googlechat.accounts` as optional in the config schema/types/generated metadata, while the app-url add-on auth path still rejects missing principal bindings and can return 401. The related fix PR #51817 was closed unmerged, so it is not a reason to close this issue.

## Best Possible Solution

Keep the issue open and implement the schema/docs contract deliberately: require or validate `appPrincipal` when Google Chat uses `audienceType: "app-url"` add-on delivery, update `GoogleChatAccountConfig`, generated config metadata/docs, and add focused config validation tests. Preserve the current project-number behavior unless maintainers intentionally decide that all Google Chat configs must provide `appPrincipal`.
