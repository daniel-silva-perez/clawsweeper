## Summary

Current main still has the reported Google Chat config/docs mismatch. `appPrincipal` and `channels.googlechat.accounts` remain optional in schema/types/generated metadata, while app-url add-on authentication still rejects missing principal bindings. The related fix PR #51817 was closed unmerged, so it does not supersede this issue.

## Best Possible Solution

Keep this issue open and make the config contract match runtime behavior deliberately: add schema/config validation or generated metadata that requires `appPrincipal` when Google Chat uses `audienceType: "app-url"` add-on delivery, update `GoogleChatAccountConfig` and the public Google Chat docs together, and preserve the project-number path unless maintainers intentionally choose a global requirement.
