## Summary

Keep this PR open. Current main still renders and patches channel config forms at the channel root, so it has not implemented the intended account-scoped form resolution. The PR is a relevant fix candidate for a real account-path UI problem, but the review discussion identifies correctness blockers that should be addressed before merge.

## Best Possible Solution

Keep this PR open as a candidate, but require a corrected implementation before merge: pass the preferred/default account id into the config section, resolve account-scoped values when present, fall back both schema and value handling to the root channel config for root-backed channels, keep patch paths aligned with the object being displayed, and add regression tests for account-scoped and root-fallback cases.
