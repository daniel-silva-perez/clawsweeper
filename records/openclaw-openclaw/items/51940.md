## Summary

Keep this PR open. Current main still renders channel config forms against the channel root only, while account-scoped channel config is a documented and schema-backed product surface. The PR is still a relevant fix candidate for #51335, but the proposed diff has substantive correctness blockers around schema fallback and patch-path alignment before it should be merged.

## Best Possible Solution

Keep this PR open as a focused fix candidate, but require a revised implementation: pass the preferred/default account id into channel config rendering, resolve account-scoped values when present, fall back both schema and value handling to the root channel config for root-backed channels, align the patch path with the object actually being displayed, update extras/status consumers if they should reflect account-scoped fields, and add regression tests for account-scoped plus root-fallback cases.
