## Summary

Keep open. Current main does not implement write-time stripping for persisted tool-result image blocks, so the PR is not obsolete, but the proposed empty-string persistence is not safe as written because replay sanitization turns those images into text placeholders and loses recent image context that current main deliberately preserves for follow-up turns.

## What This Changes

The PR adds a guardSessionManager persistence transform that replaces tool-result image block data with an empty string and adds a colocated test file that copies the stripping helper logic.

## Best Possible Solution

A maintainable fix should store references to durable media and hydrate them for provider replay, or at minimum preserve recent image payloads while pruning only older or replaceable images, with tests that exercise guardSessionManager and replay sanitization together.

## Reproduction Assessment

Yes. The source path is enough to reproduce: persist a toolResult image with the PR transform, reload or sanitize session history, and the empty image block becomes a text placeholder instead of usable image content.

## Solution Assessment

No. Emptying every tool-result image at persistence time is narrower than #1210's media-reference approach but removes the recent replay window that current main intentionally keeps for follow-up turns.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve replayable image context instead of emptying it:** `src/agents/session-tool-result-guard-wrapper.ts:37`
  - body: This writes every persisted tool-result image block back with `data: ""`. On the next session load, `sanitizeContentBlocksImages` converts that empty image into a text placeholder, so follow-up turns lose the screenshot/image even though current main keeps recent image blocks for three completed turns before pruning. Please store a media reference and hydrate it, or otherwise keep recent replayable image data instead of blanking it at write time.
  - confidence: 0.88

  - body: The new tests define their own `stripBase64ImagesFromToolResult` rather than importing production code or driving `guardSessionManager`. If the real transform is removed, reordered, or not wired into persistence, this file still passes, so it does not guard the behavior the PR changes.
  - confidence: 0.9
