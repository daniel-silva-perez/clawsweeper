## Summary

Keep this PR open. Current main still lacks the PR’s central behavior: outbound reply payload helpers accept and resolve only `mediaUrls` / `mediaUrl`, while the proposed loose `media` alias is only present in the PR context and in separate message-action surfaces.

## Best Possible Solution

Keep the PR open for maintainer review or rebase. If OpenClaw wants this compatibility behavior, land a refreshed patch in `src/plugin-sdk/reply-payload.ts` that consistently accepts `media` in normalization, media resolution, content detection, sendable-part extraction, and `sendPayloadWithChunkedTextAndMedia`, with tests for array/string aliases, blank alias entries, and empty `mediaUrl` fallback.
