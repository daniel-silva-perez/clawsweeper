## Summary

Keep this PR open. Current main still does not implement the PR’s central behavior: the public reply-payload SDK normalizer and helper functions only accept `mediaUrls` / `mediaUrl`, while the proposed loose `media` alias support remains only in the PR patch context.

## Best Possible Solution

Keep the PR open for maintainer review or rebase. If OpenClaw wants this compatibility behavior, land a refreshed patch in `src/plugin-sdk/reply-payload.ts` that consistently accepts `media` in normalization, media resolution, content detection, sendable-part extraction, and chunked media sending, with regression tests for string/array aliases, blank alias entries, and empty `mediaUrl` fallback.
