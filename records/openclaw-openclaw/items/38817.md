## Summary

Keep this issue open. Current main still strips unsupported tool-schema keywords only when resolved model compat declares `unsupportedToolSchemaKeywords`; the bundled Volcengine/Ark/Kimi catalog rows do not declare that compat and the Volcengine provider does not register a schema or resolved-model compat hook. The two related fix PRs are closed and unmerged, so the user-visible failure still needs a narrow provider-owned fix.

## What This Changes

Volcengine/Ark-hosted Kimi models should strip unsupported JSON Schema validation keywords such as `minLength`, `maxLength`, and `minItems` from tool parameter schemas so provider validation does not reject tool calls.

## Best Possible Solution

Implement provider-owned compatibility for bundled Volcengine/Ark/Kimi models. Prefer row-level `compat.unsupportedToolSchemaKeywords` in the Volcengine manifest catalog because the manifest provider builder already preserves model compat; use a Volcengine-owned `normalizeResolvedModel`, `contributeResolvedModelCompat`, or `normalizeToolSchemas` hook only if manifest/model compat is insufficient. Add regression coverage proving affected Volcengine/Ark/Kimi rows strip unsupported keywords while unrelated providers keep validation keywords, and consider the sibling BytePlus Ark/Kimi rows in the same pass.
