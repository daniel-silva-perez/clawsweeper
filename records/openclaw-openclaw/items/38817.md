## What This Changes

The issue requests support for Volcengine-hosted Kimi models whose tool schemas are rejected when they contain JSON Schema validation keywords such as `minLength`.

## Best Possible Solution

Add provider-owned `compat.unsupportedToolSchemaKeywords` metadata to the affected Volcengine/Ark/Kimi catalog rows, include BytePlus only if the same backend behavior applies, and keep core normalization generic and driven by resolved model compat metadata.
