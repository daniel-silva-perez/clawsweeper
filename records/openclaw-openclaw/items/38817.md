## Summary

Keep this open. Current main still depends on resolved-model `compat.unsupportedToolSchemaKeywords` to strip schema validation keywords, but the bundled Volcengine/Ark/Kimi rows do not declare that compat and the Volcengine provider does not add an equivalent hook. The two PRs that directly targeted this report are closed and unmerged; the newer open Kimi compaction PR is related but tracks #44549 and is not a direct Volcengine/Ark fix.

## What This Changes

The issue reports Volcengine-hosted Kimi models rejecting tool schemas containing unsupported validation keywords such as `minLength`, and requests provider-owned stripping for those models.

## Best Possible Solution

Add provider-owned compat metadata for affected Volcengine/Ark/Kimi catalog rows, and likely the sibling BytePlus Ark/Kimi rows if they share the same backend behavior, so the existing generic tool-schema cleanup strips unsupported validation keywords without hardcoding Volcengine or Kimi detection in core.
