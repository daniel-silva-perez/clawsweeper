## Summary

Keep open. Current main still strips unsupported tool-schema keywords only when the resolved model declares `compat.unsupportedToolSchemaKeywords`, while the Volcengine and BytePlus Ark/Kimi catalog rows still lack that compat metadata. The two direct fix PRs, #38883 and #46887, are closed unmerged, and #44888 tracks a different Kimi schema-compaction failure.

## What This Changes

The issue requests support for Volcengine/Ark-hosted Kimi models that reject validation keywords such as `minLength`, `maxLength`, and `minItems` in tool parameter schemas.

## Best Possible Solution

Add provider-owned `compat.unsupportedToolSchemaKeywords` metadata to the affected Volcengine/Ark/Kimi catalog rows, and include sibling BytePlus rows if they share the same backend behavior, while preserving the existing generic compat-driven normalization path in core.
