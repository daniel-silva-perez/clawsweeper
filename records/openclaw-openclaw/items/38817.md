## Summary

Keep this issue open. Current main still has the generic mechanism for stripping unsupported tool-schema keywords, but Volcengine/Ark/Kimi models do not contribute `unsupportedToolSchemaKeywords` through model compat or a provider tool-schema hook. The related fix PRs #38883 and #46887 were closed unmerged, so there is no merged or shipped fix to close against.

## Best Possible Solution

Implement provider-owned compatibility for bundled Volcengine/Ark/Kimi models. Prefer adding `unsupportedToolSchemaKeywords` through the Volcengine provider's resolved-model compat path or model catalog rows, or a provider `normalizeToolSchemas` hook if that better matches the transport family. Add focused regression tests for `volcengine`, `volcengine-plan`/Ark, and Kimi model IDs, plus a negative case proving unrelated providers keep validation keywords.
