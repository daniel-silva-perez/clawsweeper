## Summary

Keep open. Current main still depends on generic `modelCompat.unsupportedToolSchemaKeywords` for stripping unsupported tool-schema keywords, but the affected Volcengine Ark/Kimi catalog rows do not declare that compat metadata. The direct fix PRs are closed unmerged, and the open Kimi compaction PR tracks a different failure mode.

## What This Changes

Volcengine-hosted Kimi models should avoid sending unsupported JSON Schema validation keywords such as `minLength` in tool parameter schemas.

## Best Possible Solution

Add provider-owned `compat.unsupportedToolSchemaKeywords` metadata to the affected Volcengine Ark/Kimi catalog rows, include sibling BytePlus rows only if the same backend behavior is confirmed, and keep core normalization provider-agnostic with focused regression coverage.
