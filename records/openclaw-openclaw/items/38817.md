## Summary

Keep open. Current main still strips unsupported tool-schema keywords only when the resolved model declares `compat.unsupportedToolSchemaKeywords`; the bundled Volcengine/Ark/Kimi catalog rows do not declare that compat, and the two direct fix PRs for this report were closed without merge.

## What This Changes

The issue requests support for Volcengine/Ark-hosted Kimi models that reject tool parameter schemas containing validation keywords such as `minLength`, `maxLength`, and `minItems`.

## Best Possible Solution

Add provider-owned compat metadata for affected Volcengine/Ark/Kimi catalog rows, and likely sibling BytePlus Ark/Kimi rows if they share the same backend behavior, so the existing generic normalization strips unsupported validation keywords without hardcoding Volcengine or Kimi detection in core.
