## Summary

Keep this issue open. Current main still strips unsupported tool-schema keywords only when resolved model compat declares them, and Volcengine/Ark/Kimi currently does not declare `unsupportedToolSchemaKeywords` in its manifest catalog or via a provider hook. The two linked fix PRs are closed and unmerged, and no open PR currently appears to cover this issue.

## Best Possible Solution

Implement provider-owned compatibility for bundled Volcengine/Ark/Kimi models. Prefer adding `compat.unsupportedToolSchemaKeywords` through the Volcengine manifest catalog rows or the Volcengine provider's resolved-model compat path, and use a provider `normalizeToolSchemas` hook only if that better matches the transport family. Add focused regression coverage for `volcengine`, `volcengine-plan`/Ark, and Kimi model IDs, plus a negative case proving unrelated providers keep validation keywords.
