## Summary

Keep open. Current main still does not implement the requested ISO 8601 heartbeat/current-time contract: the shared cron-style helper still emits a prose `formatUserTime(...)` value plus a `YYYY-MM-DD HH:MM UTC` suffix, and heartbeat still passes its run `startedAt` into that same `Current time:` line without adding a separate trigger timestamp. The linked implementation PR #55957 is closed and unmerged, while #63181 is adjacent temporal-grounding work rather than a replacement for this formatting request.

## Best Possible Solution

Implement the timestamp contract in the shared cron-style current-time helper and heartbeat caller: emit a machine-readable ISO 8601 local timestamp with offset for `Current time`, decide whether heartbeat should add a distinct `Heartbeat triggered at` value from the queued/run trigger time, and update heartbeat, cron, reset, post-compaction, memory flush, and timestamp-detection tests in one coordinated change. Treat #63181 as adjacent temporal-grounding work, not a replacement for this formatting request.
