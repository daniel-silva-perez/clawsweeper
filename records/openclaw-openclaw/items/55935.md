## Summary

Keep open. Current main still builds the shared cron/heartbeat `Current time:` line from the locale-style `formatUserTime(...)` output plus a `YYYY-MM-DD HH:MM UTC` suffix, not an ISO 8601 local timestamp with offset. Heartbeat still passes the run `startedAt` value into that line and there is no separate `Heartbeat triggered at` field. The related implementation PR #55957 was closed unmerged, and the adjacent open issue #63181 covers a different temporal-grounding prompt-label problem rather than superseding this request.

## Best Possible Solution

Keep this issue open and implement the timestamp contract in the shared cron-style current-time helper. Emit an ISO 8601 local timestamp with offset for `Current time`, decide whether heartbeat should add a distinct `Heartbeat triggered at` value from `startedAt`, and update heartbeat, cron, reset, post-compaction, memory flush, and timestamp-detection tests in one coordinated change. Treat #63181 as adjacent temporal-grounding work, not a replacement for this formatting request.
