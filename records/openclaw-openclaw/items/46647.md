## Summary

Keep this issue open. Current main still has the original prompt mismatch: generic message-tool guidance treats inline buttons as enabled only when the runtime capability set contains `inlineButtons`, while Slack object-form config enables the actual directive pipeline through `interactiveReplies`. The later streaming Block Kit preservation concern appears addressed, but that does not fix the contradictory prompt guidance.

## Best Possible Solution

Keep this open and land a narrow prompt fix. Either make the generic inline-button affordance treat Slack `interactiveReplies` as enabled, or suppress the generic `inlineButtons` disabled line for Slack and rely on Slack-specific `messageToolHints`. Add regression coverage for Slack object-form `{ interactiveReplies: true }` so the prompt does not suggest `slack.capabilities.inlineButtons`.
