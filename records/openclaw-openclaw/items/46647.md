## Summary

Keep this issue open. Current main has made real progress on Slack interactive replies, especially the Slack-specific prompt hints and streaming/block delivery path, but the original prompt bug is still visible: the generic message-tool guidance still keys inline-button support only off `inlineButtons`, while Slack object-form config uses `interactiveReplies`.

## Best Possible Solution

Keep this open and land a narrow prompt fix: either treat Slack `interactiveReplies` as enabling the relevant prompt affordance, or suppress the generic `inlineButtons` disabled line for Slack and rely on the Slack-specific `messageToolHints`. Add a regression test that builds the system prompt with `channels.slack.capabilities: { interactiveReplies: true }` and asserts it does not suggest `slack.capabilities.inlineButtons`.

## Evidence

There is no Slack `interactiveReplies` branch here.
