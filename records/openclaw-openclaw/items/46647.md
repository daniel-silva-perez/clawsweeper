## Summary

Keep this issue open. Current main still has the original prompt mismatch: the generic message-tool prompt decides inline-button support only from the `inlineButtons` runtime capability, while Slack object-form config enables the real directive pipeline through `interactiveReplies`. Slack-specific hints and the streaming/block preservation path have improved, but they do not remove the contradictory generic disabled guidance.

## Best Possible Solution

Keep this open and land a narrow prompt fix. Either treat Slack `interactiveReplies` as enabling the relevant prompt affordance, or suppress the generic `inlineButtons` disabled line for Slack and rely on Slack-specific `messageToolHints`. Add regression coverage that builds the system prompt for Slack with object-form `{ interactiveReplies: true }` and asserts it does not suggest `slack.capabilities.inlineButtons`.
