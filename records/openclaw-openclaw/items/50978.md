## Summary

Keep this PR open. Current main still does not implement the intended force-enable behavior: `skills.entries.<key>.enabled: true` still falls through to runtime eligibility, so unmet `requires.bins`/OS/env/config gates can exclude the skill. The PR remains a focused implementation candidate for the already-linked bug report #32752.

## Best Possible Solution

Keep the PR open for maintainer review. If `enabled: true` is meant to force-include a skill, land this PR or an equivalent patch that updates `shouldIncludeSkill`, keeps `skills.allowBundled` precedence, keeps skill status/listing behavior consistent, and updates the skills docs. If runtime eligibility should remain authoritative, close the PR with that explicit product decision.
