## Summary

Keep this PR open. The checked-out main has useful overlapping shutdown hardening for lifecycle hooks, runtime disposal, WebSocket close, HTTP close, and channel stop grace, but it still does not implement the PR's central change: structured per-subsystem shutdown warning collection with a caller-visible `ShutdownResult`. The PR is also active again after a maintainer repair push and merge-ready labeling, so conservative cleanup should not close it.

## Best Possible Solution

Keep this PR open for maintainer review and likely rebase. The best path is to land the contributor branch if maintainers want `ShutdownResult` as the gateway close contract, while preserving current main's lifecycle/runtime/WebSocket/HTTP hardening, reconciling overlap with #56258, and adding focused tests for per-subsystem warning aggregation, HTTP timeout warnings, duplicate warning names, and sensitive-log safety.
