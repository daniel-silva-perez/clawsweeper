## Summary

Keep open. Current main has overlapping gateway shutdown hardening for lifecycle hooks, WebSocket close, HTTP close, MCP/LSP runtime disposal, and channel stop waits, but it still does not implement this PR’s central per-subsystem warning collection or structured ShutdownResult observability. The PR also needs a rebase against newer shutdown code before maintainers can evaluate it cleanly.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase it onto current main, preserve the newer shutdown hardening already shipped, decide whether ShutdownResult belongs in the gateway close contract, and add focused tests for per-subsystem warning collection and any timeout behavior that remains after reconciliation. Related open PR #56258 covers per-subsystem shutdown timeouts, but it does not fully supersede this PR’s warning/result observability work.
