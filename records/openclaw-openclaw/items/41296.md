## Summary

Keep this PR open. Current main has adjacent gateway shutdown hardening, including lifecycle hook timeouts plus WebSocket and HTTP close grace/force handling, but it still does not implement the PR’s central structured shutdown-warning contract: no `ShutdownResult`, no shared `shutdownStep`, no returned warning aggregation from the close handler, and no final clean/with-warnings summary. The PR is also active after a maintainer repair, the author offered follow-up help, and the PR head remains a distinct implementation rather than obsolete main-branch code.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase it onto current main, preserve the newer lifecycle/WebSocket/HTTP shutdown hardening, decide and implement whether `ShutdownResult` should propagate through `GatewayServer.close`, reconcile overlap with #56258, keep the focused warning/timeout tests, and review shutdown warning text for sensitive detail exposure before merge.
