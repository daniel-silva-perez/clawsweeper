## Summary

Keep open. Current main still routes `openclaw backup create` into a single `node-tar` archive operation that will reject when a source file disappears during the walk, while session maintenance can still rename deleted transcript files. The open closing PR #67472 and broader related PR #72251 remain unmerged, so the issue should wait for maintainer PR selection or a merged replacement fix.

## What This Changes

The issue asks for `openclaw backup create` to skip session files that disappear during concurrent session cleanup instead of aborting the whole backup with ENOENT.

## Best Possible Solution

Land one coherent backup archiver fix in `src/infra/backup-create.ts` that treats expected per-entry deleted-file ENOENT as non-fatal, reports skipped files clearly, and adds a regression test proving backup creation succeeds when a planned session transcript is removed or renamed during archive creation.
