## Summary

Keep open. Current main still has the reported live-backup race surface, and an open PR with closing syntax already attempts the narrow ENOENT fix while a broader live-backup race PR is also still open. This issue should not be closed before one of those implementations lands or is deliberately superseded.

## What This Changes

The issue asks for `openclaw backup create` to skip session transcript files that are deleted by concurrent session cleanup instead of aborting the backup with ENOENT.

## Best Possible Solution

Land one coherent backup archiver fix that handles files disappearing during archive creation, preferably with a regression test that proves backup creation succeeds when a planned session transcript is removed or renamed during the archive walk. If the broader live-backup design is chosen, it should explicitly cover this deleted-session-file case before this issue is closed.
