## Summary

Keep this PR open. The checked-out main branch still does not pass trusted sender context into channel-owned agent tool factories, and the PR remains a narrow additive implementation candidate for an extension-facing SDK gap. The linked issue #58806 is closed, but this PR uses closing syntax and has not merged, so it should not be closed as implemented yet.

## Best Possible Solution

Review and land this PR, or an equivalent additive SDK/core change, with a focused regression test proving createOpenClawCodingTools forwards senderId and senderIsOwner into a channel agentTools factory. Preserve the existing trusted requester semantics used by channel message-action discovery.
