## Summary

Current main still lacks the sender-aware ChannelAgentToolFactory wiring this PR proposes. The PR remains a narrow, plausible public plugin SDK improvement rather than obsolete work.

## Best Possible Solution

Keep this PR open and review/land it, or an equivalent core SDK change, with a narrow regression test. The implementation should extend ChannelAgentToolFactory and listChannelAgentTools params with optional requesterSenderId and senderIsOwner, thread options.senderId/options.senderIsOwner from createOpenClawCodingTools, and document the trusted server-injected nature of those fields.
