## Summary

Current main still lacks the additive sender-context seam this PR is trying to add. The PR remains a narrow, plausible implementation candidate for the linked channel-plugin use case rather than something that should be closed as already implemented or obsolete.

## Best Possible Solution

Keep the implementation additive and document the trusted server-injected semantics consistently with the existing message-action discovery context.
