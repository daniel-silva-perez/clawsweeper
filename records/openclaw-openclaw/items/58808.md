## Summary

Keep this PR open. Current main still does not pass requester sender context into channel-owned agent tool factories, and the live PR is a narrow additive implementation candidate for that public plugin SDK gap. The security review found no unrelated CI, dependency, lockfile, script, secret, generated-code, or artifact-execution changes.

## Best Possible Solution

Review and land this PR, or an equivalent additive SDK/core change, with a focused regression test in the agent tool composition area. Keep the new fields optional, document them as trusted server-injected context, and preserve compatibility for existing factories that only use cfg.
