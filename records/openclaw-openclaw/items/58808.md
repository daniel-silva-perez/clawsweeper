## Summary

Keep this PR open. Current main still does not pass requester sender context into channel-owned agent tool factories, while the PR remains a narrow additive implementation candidate for a real public plugin SDK gap. The diff is small and security-relevant in a controlled way: it forwards trusted server-side context to plugin code, touches no dependency/CI/release paths, and matches adjacent message-action discovery semantics.

## Best Possible Solution

Review and land this PR, or an equivalent additive SDK/core change, with a focused regression test in the agent tool composition area. Keep the new fields optional, document them as trusted server-injected context, and preserve the existing public plugin compatibility for factories that only use cfg.
