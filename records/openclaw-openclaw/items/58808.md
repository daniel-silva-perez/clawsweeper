## Summary

Keep this PR open. Current main still does not pass requester sender context into channel-owned agent tool factories, and the PR remains a narrow additive implementation candidate for a real plugin SDK gap.

Keep the trusted sender semantics aligned with the existing message-action discovery and execution context.
