## Summary

Keep this PR open. Current main still does not implement the requested sender-aware `ChannelAgentToolFactory` context, and the PR remains a narrow additive SDK/core change with a concrete plugin-author use case. The diff does not show supply-chain or CI/security-sensitive drift, but it still needs maintainer review and preferably a focused regression test before landing.

## Best Possible Solution

Review and land this PR, or an equivalent additive SDK/core change, with a focused regression test that registers a channel `agentTools` factory and proves `createOpenClawCodingTools({ senderId, senderIsOwner })` forwards those values. Keep the fields optional and documented as trusted server-injected context, matching the existing message-action discovery seam.
