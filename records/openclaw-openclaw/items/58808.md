## Summary

Current main still has the gap this PR targets: channel agent-tool factories receive only config, while the adjacent core tool path receives trusted sender context. The PR is a narrow additive SDK/core wiring change and is not obsolete or implemented on main.

## Best Possible Solution

Review and land this PR, or an equivalent core SDK change, with a narrow regression test. The implementation should extend `ChannelAgentToolFactory` and `listChannelAgentTools` params with optional `requesterSenderId` and `senderIsOwner`, thread `options.senderId` and `options.senderIsOwner` from `createOpenClawCodingTools`, and document that these are trusted server-injected context for channel-owned agent tools.
