## Summary

Keep open. The inline Anthropic provider defaulting bug still exists on current main, but this PR should not merge as-is because the branch is stale and includes an unrelated Telegram retry change that can replay non-idempotent sends.

## What This Changes

The PR applies provider-level API defaulting in inline provider model construction, adds regression tests, exports the config default helper, and also broadens Telegram send retry classification.

## Best Possible Solution

Land a model-only patch against current main that applies provider-policy/default API normalization before inline provider models are built, preserves explicit model/provider API precedence, adds focused inline-provider regression tests, and leaves Telegram retry semantics unchanged.

## Reproduction Assessment

Yes. A source-level reproduction is an Anthropic inline provider with models and no explicit api: current buildInlineProviderModels leaves api undefined, and resolveExplicitModelWithRegistry then skips the inline match because inlineMatch?.api is false.

## Solution Assessment

No, not as this PR branch. The model fix direction is narrow and maintainable, but the patch needs to be ported to the current split inline-provider file and the unrelated Telegram retry change should be dropped.

## Review Findings
