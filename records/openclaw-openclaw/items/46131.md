## Summary

Keep this PR open as useful source work, but do not land it as-is. Current main still misses the inline Anthropic provider API default on the inline model path, while the PR is stale against the current file split and also includes an unrelated Telegram retry change that would regress non-idempotent send safety.

## What This Changes

The PR exports provider API defaulting, applies it to inline provider model construction with regression tests, and also broadens Telegram retry classification for 429 and grammY failed-after errors.

## Best Possible Solution
