## Summary

Keep open. The model bug is still present on current main, so this PR is not obsolete, but the proposed branch should not merge as-is because it carries an unrelated Telegram retry change that weakens the non-idempotent send retry boundary.

## What This Changes

The PR exports provider API defaulting, applies it while building inline provider models, adds model regression tests, and also changes Telegram retry classification for 429 and grammY failed-after errors.

## Best Possible Solution

Land a model-only update against current main that applies provider-policy defaulting before inline provider models are built, adds focused regression coverage for implicit Anthropic provider defaults, and leaves Telegram retry semantics to the existing safe-send and rate-limit boundaries.
