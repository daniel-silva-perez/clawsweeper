## Summary

Keep this PR open. Current main still lacks both core duplicate-user-message protections proposed here: inbound dedupe still returns no key without MessageSid, and chat.send user transcript updates still omit idempotencyKey. The linked regression issue remains open in the provided GitHub context, and this PR is an active unmerged implementation candidate, though it needs rebase and maintainer review before merge.

## Best Possible Solution

Keep this PR open as a plausible implementation candidate. The best path is to rebase it onto current main, adapt the inbound fallback to the current channelRouteDedupeKey shape, add focused tests for missing/unstable MessageSid and user transcript idempotency, resolve the content-hash review concern, and either split or explicitly coordinate the overlapping Control UI render-race portion with #72004.
