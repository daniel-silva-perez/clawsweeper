## Summary

Keep open. Current main still has only the Control UI assistant-message copy action labeled “Copy as markdown”; it writes the markdown string directly to the clipboard and there is no Markdown/plain-text selector, user preference, or test/doc coverage showing the requested plain-text copy option has shipped.

## Best Possible Solution

Keep this issue open. The best path is a focused Control UI change around ui/src/ui/chat/copy-as-markdown.ts and ui/src/ui/chat/grouped-render.ts: add a compact Markdown vs plain-text copy menu or a persisted copy-format preference, choose or adapt a markdown-to-plain-text conversion helper with clear behavior for code blocks/lists/links, and add targeted clipboard/UI tests proving both formats write the expected text.
