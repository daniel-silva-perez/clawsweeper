## Summary

Keep this PR open. The proposed render-time ID capture addresses a real Control UI race for exec approvals, but the branch re-derives the approval kind from the mutable live queue, so plugin approvals still fail in the same stale-render window after the entry is pruned.

## What This Changes

The PR updates Control UI approval buttons to pass a render-time approval ID into `handleExecApprovalDecision` and widens the shared view-state handler signature for that optional ID.

## Best Possible Solution

Keep the render-time capture approach, but capture stable routing metadata too, either the approval kind or the exact resolver method, and add focused UI coverage for both exec and plugin approval button clicks after queue mutation.
