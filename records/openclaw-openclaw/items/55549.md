## Summary

Keep open. Current main does not implement the PR’s intended fix: the Control UI approval buttons still call the decision handler without a captured approval id, and the handler still re-reads `execApprovalQueue[0]` at click time. The PR is a plausible implementation candidate for #55251, but its submitted diff also needs the existing review feedback addressed for plugin approvals.

## Best Possible Solution

Keep this PR open as the implementation candidate, but update it before merge to pass a stable render-time approval identity that includes both id and kind/method, preserve the current backend validation behavior, and add a focused Control UI regression test that clicks a rendered approval after the queue has been cleared. After that, review and land it or replace it with an equivalent main-branch fix.
