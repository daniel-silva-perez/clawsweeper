## Summary

Keep open. Current main and the latest release still only parse the legacy `IDEProvisioningTeams` preference path, while this PR adds useful support for `IDEProvisioningTeamByIdentifier`; #22773 is adjacent but did not ship this parser. The remaining merge blocker is the PR test gap: the changed smoke test still uses a fake Python executable, so it does not exercise the new inline plist parser.

## What This Changes

The PR updates `scripts/ios-team-id.sh` to read Xcode team IDs from both `IDEProvisioningTeamByIdentifier` and `IDEProvisioningTeams`, and adjusts the iOS team-id test fixtures and expectations.

## Best Possible Solution

Land the focused signing-helper support after the regression test runs the real inline Python parser against mocked `defaults export` plist data containing `IDEProvisioningTeamByIdentifier`, while preserving the existing `plutil`, managed-profile, legacy-defaults, explicit-env, and optional keychain fallback behavior.
