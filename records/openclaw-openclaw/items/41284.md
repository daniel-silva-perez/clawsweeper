## Summary

Keep this PR open. Current main still lacks `IDEProvisioningTeamByIdentifier` parsing, and the related merged #22773 only added provisioning-profile fallback and guidance for a different failure path. The PR is a focused script/test change, but its regression test still uses a fake Python executable and does not exercise the new inline parser.

## What This Changes

The PR updates `scripts/ios-team-id.sh` to parse Xcode preference data from `defaults export` for both `IDEProvisioningTeamByIdentifier` and `IDEProvisioningTeams`, preserves the `plutil` fallback, and adjusts the `ios-team-id` test fixture.

## Best Possible Solution

Keep the PR open and land the focused signing-helper support after the regression test executes the real inline Python parser against mocked `defaults export` plist data containing `IDEProvisioningTeamByIdentifier`, while preserving the existing `plutil`, managed-profile, legacy-defaults, and explicit-env fallback behavior.
