## Summary

Keep open. Current main still lacks the requested identifier-key support, and the PR is a useful focused fix, but the latest head still has a false-positive regression test for the new parser path before it is merge-ready.

## What This Changes

This PR updates `scripts/ios-team-id.sh` to parse Xcode team data from `defaults export` for both `IDEProvisioningTeamByIdentifier` and `IDEProvisioningTeams`, and adjusts the iOS team-id test fixture and expected selection.

## Best Possible Solution

Land the focused signing-helper support after the identifier-key regression case runs the actual inline Python parser against mocked `defaults` and `plutil` tools, while preserving explicit env overrides, existing `IDEProvisioningTeams`, managed-profile, legacy defaults, and optional keychain fallback behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists with an Xcode preferences plist that only has `IDEProvisioningTeamByIdentifier`: current main only extracts `IDEProvisioningTeams`, and the consumer warning path matches the PR body.

## Solution Assessment

No, not as proposed yet. Parsing exported Xcode preferences while retaining the `plutil` fallback is the right narrow direction, but the changed smoke test should execute the real parser before the patch is merge-ready.
