## Summary

Keep this PR open: current main still lacks `IDEProvisioningTeamByIdentifier` support, and the branch is a useful focused fix for an active iOS signing path. The remaining blocker is test quality: the changed smoke test still uses a fake Python executable that bypasses the new inline parser, so the core parser path can regress while the new test remains green.

## What This Changes

The PR changes `scripts/ios-team-id.sh` to parse exported Xcode preferences for both identifier-key and legacy team layouts, and updates `test/scripts/ios-team-id.test.ts` fixtures and expectations.

## Best Possible Solution

Land the focused signing-helper support after the regression test runs the real inline Python parser against mocked `defaults export` plist data containing `IDEProvisioningTeamByIdentifier`, while preserving the existing `plutil`, managed-profile, legacy-defaults, explicit-env, and optional keychain fallback behavior.
