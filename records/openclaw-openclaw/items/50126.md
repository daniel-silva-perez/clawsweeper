## Summary

Keep open. Current main has substantial hook coverage in the shared outbound delivery path and Telegram direct replies, but the umbrella problem is still current: Slack and Feishu normal reply dispatchers still send directly without the shared deliverer or hook emission, and plugin-handled message actions still return before the core send path that emits hooks.

## Best Possible Solution

Keep this open as the umbrella until outbound delivery has one documented hook-firing seam, or until every normal reply path is individually converted. A conservative fix would route Slack and Feishu reply dispatch through `deliverOutboundPayloads` or an equivalent shared observer, wrap plugin-handled message actions with `message_sending`/`message_sent`, and add focused regression tests for shared delivery plus representative channel-local paths including webchat/TUI.
