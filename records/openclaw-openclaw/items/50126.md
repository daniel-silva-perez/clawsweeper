## Summary

Current main fixes the shared outbound path and normal Telegram reply path, but the umbrella report remains valid: webchat/chat final delivery still broadcasts directly without `message_sending`, `message_sent`, or internal `message:sent`, and plugin-handled message actions can still return before the shared outbound hook layer. The narrower open reports #70928 and #65792 cover important slices, while the closed unmerged Telegram PR #50934 only overlaps part of the Telegram streaming/internal-hook behavior.

## What This Changes

Request consistent internal `message:sent` and plugin `message_sending`/`message_sent` hook coverage across outbound dispatch paths including Telegram replies, webchat/TUI delivery, RPC respond paths, and plugin-handled message actions.

## Best Possible Solution

Keep this issue open as the maintainer-scoped umbrella until OpenClaw either adds one generic post/pre-delivery observer seam that every outbound surface calls exactly once, or deliberately splits and lands the remaining narrow fixes for webchat/TUI and plugin-handled message actions, then closes the umbrella with those shipped references.
