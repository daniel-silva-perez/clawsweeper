## Summary

Keep open. Current main fixes the shared outbound delivery path and normal Telegram reply hook emission, but the broader consistency problem remains: webchat final delivery still broadcasts directly without outbound hook calls, and plugin-handled message actions can return before the shared outbound delivery hook layer.

## What This Changes

The issue requests consistent outbound lifecycle hook coverage across all dispatch paths, including internal `message:sent` and plugin `message_sending`/`message_sent` events.

## Best Possible Solution

Define one documented outbound lifecycle observer seam, or explicitly land the narrower fixes, so every outbound delivery path invokes `message_sending` before delivery and `message_sent`/`message:sent` after success or failure exactly once, with regression tests for shared outbound, Telegram, webchat/TUI, and plugin-handled send paths.
