## Summary

Current main fixed the originally confirmed Telegram reply gap and the shared outbound delivery path, but the broader hook-coverage inconsistency remains on WebChat/Gateway chat finals, embedded TUI finals, and plugin-handled message actions. Related open issues #65792 and #70928 confirm narrower remaining surfaces, while this issue still works as the umbrella tracker.

## What This Changes

The issue requests consistent internal `message:sent` and plugin `message_sent` / `message_sending` hook coverage across all outbound dispatch paths, especially Telegram replies, WebChat/TUI finals, and plugin-handled message actions.

## Best Possible Solution

Keep this as the umbrella tracker until maintainers either define one documented outbound lifecycle observer seam or split the remaining WebChat, Gateway-backed TUI, embedded TUI, and plugin-handled action gaps into canonical narrower fixes with explicit exactly-once, cancellation, and transcript-write semantics.
