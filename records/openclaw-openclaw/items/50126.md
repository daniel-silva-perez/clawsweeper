## Summary

Keep open. Current main has fixed important slices of the report, especially shared outbound delivery and Telegram reply delivery, but the umbrella hook-coverage problem still reproduces in source: WebChat final delivery, Gateway chat finalization, local embedded TUI final delivery, and plugin-handled message actions still bypass the same outbound `message_sending` / `message_sent` / internal `message:sent` hook path.

## What This Changes

The issue asks for consistent outbound `message:sent` and plugin `message_sent` hook emission across the shared outbound dispatcher, Telegram replies, WebChat/TUI final delivery, RPC responses, and plugin-handled message actions.

## Best Possible Solution

Define and document one outbound lifecycle observer seam, or split the umbrella into canonical narrower issues, so external channel replies, WebChat, Gateway-backed TUI, embedded TUI, and plugin-handled sends invoke the applicable pre-send and post-send hooks exactly once with stable session/run context and focused regression tests.
