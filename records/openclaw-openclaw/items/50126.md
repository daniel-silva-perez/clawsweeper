## Summary

Keep open. Current main fixes important parts of the report, especially shared outbound delivery and normal Telegram replies, but the umbrella problem is still real: webchat/chat final delivery still broadcasts directly without `message_sent` / `message:sent`, and plugin-handled message actions can return before the shared outbound hook layer. Related #70928 and #65792 track narrower webchat/TUI slices, while #50934 was closed unmerged and only overlaps the Telegram streaming slice.

## Best Possible Solution

Keep this issue open for maintainer-scoped outbound lifecycle cleanup. The best path is to either add a generic post-delivery observer seam that every outbound surface calls exactly once, or split and land narrower fixes for #70928/#65792 plus a plugin-handled message-action coverage issue, then close this umbrella only after remaining paths are fixed or deliberately scoped out.
