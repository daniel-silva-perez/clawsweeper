## Summary

Keep open. Current main has fixed important shared outbound and Telegram reply hook paths, but direct WebChat/Gateway/TUI final-event delivery and plugin-handled message actions still bypass the same outbound hook runner, so the umbrella coverage problem remains valid.

## What This Changes

The issue asks OpenClaw to make outbound lifecycle hooks fire consistently across all outbound message dispatch paths, including internal `message:sent` and plugin `message_sent`/`message_sending` coverage.

## Best Possible Solution

Define and document one outbound lifecycle observer seam, or split the umbrella into canonical narrower issues, so external channel replies, WebChat, Gateway-backed TUI, embedded TUI, and plugin-handled sends invoke the applicable pre-send and post-send hooks exactly once with stable session/run context and focused regression coverage.
