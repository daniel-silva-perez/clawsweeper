## Summary

Keep open. Current main has fixed the shared outbound path and the Telegram reply cases, but WebChat, Gateway-backed TUI, local embedded TUI, and plugin-handled message actions still have paths that bypass the same outbound hook coverage, so the umbrella inconsistency remains valid.

## What This Changes

The issue asks for consistent internal `message:sent` and plugin `message_sent` hook emission across outbound dispatch paths, especially Telegram replies, conversation-stream logging, RPC replies, and plugin/tool send paths.

## Best Possible Solution

Define and document one outbound lifecycle observer seam, or split the remaining work into canonical narrower fixes, so external channel replies, WebChat, Gateway-backed TUI, embedded TUI, and plugin-handled sends invoke the applicable pre-send and post-send hooks exactly once with stable session/run context and focused regression tests.
