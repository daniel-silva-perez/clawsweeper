## Summary

Keep open. Current main wires the shared outbound delivery path and normal Telegram replies through outbound hooks, but the umbrella inconsistency remains: webchat/TUI final delivery still emits directly without hook runner calls, and plugin-handled message actions can return before the shared outbound delivery layer.

## What This Changes

Request consistent `message:sent`, `message_sent`, and related outbound hook coverage across Telegram, shared outbound delivery, plugin-handled sends, and RPC/webchat delivery paths.

## Best Possible Solution

Define and document one outbound lifecycle observer seam, or intentionally split this into narrower fixes, so webchat/TUI, shared outbound delivery, Telegram, and plugin-handled sends invoke pre-send and post-send hooks exactly once with consistent context.
