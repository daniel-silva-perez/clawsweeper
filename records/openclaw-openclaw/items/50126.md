## Summary

Keep open. Current main has fixed important shared outbound and Telegram direct-reply slices, but the WebChat/Gateway final event path, local embedded TUI final event path, and plugin-handled message action path still bypass the same outbound message hook runner, so the umbrella hook-coverage problem remains valid.

## What This Changes

The issue asks OpenClaw to make outbound `message:sent`, `message_sent`, and `message_sending` hook coverage consistent across all outbound dispatch paths.

## Best Possible Solution

Define and document one outbound lifecycle observer seam, or split the umbrella into canonical narrower issues, so external channel replies, WebChat, Gateway-backed TUI, embedded TUI, and plugin-handled sends invoke the applicable pre-send and post-send hooks exactly once with stable session/run context and focused regression tests.
