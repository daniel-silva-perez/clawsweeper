## Summary

Current main has zh-CN locale loading and some generated Chinese strings, but the reported inner Control UI tab surfaces are still only partially localized. The Agents, Skills, Channels, Sessions, Nodes, and Debug lazy-loaded views still contain many hardcoded English labels and messages, so this issue remains actionable rather than implemented or obsolete.

## Best Possible Solution

Keep this issue open. The best fix is to move the remaining user-facing strings in the affected Control UI lazy views and related controllers into `ui/src/i18n/locales/en.ts`, replace hardcoded render text with `t(...)`, regenerate the generated locale bundles through the Control UI i18n pipeline, and add focused render or i18n coverage for representative zh-CN strings in Agents, Skills, Channels, Sessions, Nodes, and Debug.
