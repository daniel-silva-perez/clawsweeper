## Summary

Keep this issue open. Current main has zh-CN locale loading and translated tab labels, but the reported inner Control UI lazy views still render many hardcoded English strings in Agents, Skills, Sessions, Nodes, and Debug. The affected paths are unchanged from the prior ClawSweeper review SHA to current main, so the earlier keep-open conclusion still holds.

## Best Possible Solution

Keep this issue open. The best fix is to move remaining user-facing strings in the Agents, Skills, Channels, Sessions, Nodes, Debug lazy views and related panels/controllers into ui/src/i18n/locales/en.ts, replace direct render text with t(...), regenerate non-English locale bundles through the Control UI i18n pipeline rather than hand-editing zh-CN, and add representative zh-CN render or i18n coverage for the affected tabs.
