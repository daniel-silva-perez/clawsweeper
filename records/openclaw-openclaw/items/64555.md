## Summary

Keep this issue open. Current `main` still has the WhatsApp default web auth directory exported as an eagerly resolved value, and the account discovery/default/legacy auth paths still resolve the OAuth root from ambient process env instead of an explicit profile env or state-dir seam. The target-repo fix PR #64563 is closed unmerged, and the external fork candidate in the provided context has not landed in `openclaw/openclaw`.

## What This Changes

The issue requests fixing WhatsApp credential path resolution so `--profile` / `OPENCLAW_STATE_DIR` keeps Baileys auth state inside the active profile instead of the default `~/.openclaw` credentials directory.

## Best Possible Solution

Land or recreate a focused WhatsApp fix that lazily resolves the default web auth dir, threads explicit env/state-dir resolution through account discovery/default/legacy auth-dir helpers, and adds a regression test proving `OPENCLAW_STATE_DIR` profile isolation for the default Baileys auth path. Doctor/docs guidance for `OPENCLAW_OAUTH_DIR` can follow after the core credential isolation path is corrected.
