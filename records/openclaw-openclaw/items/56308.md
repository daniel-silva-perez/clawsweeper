## Summary

Keep this issue open. Current main has partial session-identification primitives in the Gateway and Control UI, but the requested user-facing workflow is not implemented: the CLI session list still centers the technical key, there is no `/title` slash command, and automatic session summaries are not a shipped behavior in this repo.

## What This Changes

The issue requests automatic session titles or summaries, a manual `/title` command, and title display in `openclaw sessions` so past sessions are easier to identify.

## Best Possible Solution

Keep this open until maintainers decide the scope. A maintainable direction would reuse the existing session `label`/`displayName`/`derivedTitle` fields for CLI and manual-title workflows first, then handle automatic LLM-generated summaries as a separate, documented policy if accepted.

## Reproduction Assessment

Not applicable as a feature request, but the absence is source-verifiable. The CLI table and row model do not expose title metadata, and the command registry/docs do not define `/title`.

## Solution Assessment

No, current main is not yet the best complete solution for the request. The existing Gateway/Control UI primitives are useful groundwork, but the requested CLI/manual-title/auto-summary behavior still needs a scoped implementation or split follow-up design.
