## Summary

Keep open. Current main fixes the original stale `sessionFile` carryover, but the later issue discussion describes a remaining normal isolated-heartbeat rotation leak, and the current heartbeat runner still overwrites the isolated session entry without archiving the prior transcript for that same key.

## Best Possible Solution

Keep this issue open and implement the narrow heartbeat-runner fix described by the later comments and closed PR #64808: capture the prior `cronSession.store[isolatedSessionKey]` entry before overwriting it, archive that prior `(sessionId, sessionFile)` with `archiveRemovedSessionTranscripts({ reason: "reset", restrictToStoreDir: true })` after the new entry is saved, and add a regression test that two isolated heartbeat runs rotate to a fresh transcript while the prior primary transcript is renamed to `.reset.<timestamp>`. Keep #70680-style trajectory classification as separate doctor hygiene.
