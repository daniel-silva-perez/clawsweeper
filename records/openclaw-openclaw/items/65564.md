## Summary

Keep open. Current `main` fixes the original stale `sessionFile` carryover, but the later discussion and current source show the remaining same-key isolated heartbeat rotation path still overwrites the store entry without archiving the previous primary transcript.

## What This Changes

The issue asks OpenClaw to ensure `heartbeat.isolatedSession: true` uses fresh transcript history across runs and does not leave or reuse prior heartbeat transcript files.

## Best Possible Solution

Review and land #73809 or an equivalent narrow patch that captures the prior isolated heartbeat entry before overwrite, archives its transcript with `reason: "reset"` and `restrictToStoreDir: true`, and keeps trajectory/doctor classification work in separate tracking.

## Reproduction Assessment

Yes. The issue gives a concrete two-run `heartbeat.isolatedSession: true` reproduction, and current source confirms the same-key rotation path still overwrites the prior isolated entry without adding it to the archival set.

## Solution Assessment

Yes, the narrow archival approach in the later discussion and #73809 is the best fit: preserve the already-fixed fresh transcript behavior, add only missing lifecycle cleanup for the previous same-key transcript, and validate with a focused heartbeat runner regression test.
