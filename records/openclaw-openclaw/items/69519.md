## Summary

Keep this PR open. Current main still renders the Sessions Limit control as a plain text input, stores the filter as a string, and only coerces it later in the controller before sending `sessions.list`. The PR is an unmerged implementation candidate, but the provided review context shows it still needs rework around type consistency, empty input handling, and unrelated CSS formatting churn before it should land.

## Best Possible Solution

Keep the PR open and request a focused revision: either consistently migrate the Sessions limit filter to `number | undefined` end to end, or keep string UI state for editability and parse only at the controller/request boundary. The final patch should use `type="number"`, `min="1"`, and `step="1"`, preserve a blank field while editing, never send non-integer limits to `sessions.list`, add targeted view/controller coverage, and drop unrelated CSS-only formatting churn unless a repo formatter requires it.
