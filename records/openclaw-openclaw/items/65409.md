The branch adds report MCP config plumbing for CLI bundle MCP prep, materializes bundle MCP tool metadata in `prepareCliRunContext`, passes those tools into `buildSystemPromptReport`, and extends bundle MCP tests to assert report tool entries and schema chars.

## Best Possible Solution

Land a rebased version of this PR or an equivalent narrow change that derives the CLI report tool list from the same merged bundle MCP configuration the backend receives, disposes any temporary report runtime promptly, and covers bundle entries plus schema chars in regression tests.
