## What This Changes

The PR extends durable cron recovery to selected terminal `lost` cron task rows, clears stale terminal errors when recovery has no error, and adds regression tests for recovery and explicit error clearing.

## Best Possible Solution

Review and land this narrow task-maintenance fix after adding the required Unreleased changelog entry and passing focused task-registry validation; then use the merged PR to resolve the terminal lost cron false-positive reports while leaving broader maintenance cleanup to separate follow-up work.

## Reproduction Assessment

Yes. A high-confidence unit repro is a `runtime: "cron"`, `status: "lost"`, `error: "backing session missing"` task whose `runId` matches a durable `finished`/`ok` cron run log; current main skips it because durable cron recovery requires an active task.

## Solution Assessment

Yes, with the changelog and validation caveats. Reusing the existing durable cron history path and limiting terminal recovery to empty or backing-session-missing lost cron rows is narrower and safer than pruning or rewriting all lost records.
