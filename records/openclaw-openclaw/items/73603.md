## Summary

Keep open. Current main has related in-process observability surfaces, plugin hooks, and per-session trajectory sidecars, but it does not implement the requested default-off `events.emit.plan` contract, `/var/log/openclaw-events.jsonl` sink, plan IDs, or terminal `closes_plan` semantics. This also is not a clean ClawHub close: a plugin can observe some agent/tool/subagent events, but the stable plan/session correlation contract requested here appears to need core schema/API design first.

## Best Possible Solution

Keep this issue open for maintainer design. The best path is an additive observability contract: stable plan/run IDs, `update_plan` open/update events, terminal close/abort events, subagent spawn/yield/ended correlation, and either a default-off core JSONL writer or a documented plugin API capable of producing that writer safely, with docs and focused tests around schema, redaction, and disabled-by-default behavior.

## Risks / Open Questions

- There are overlapping observability surfaces (`api.runtime.events.onAgentEvent`, tool hooks, subagent hooks, and trajectory sidecars), so a fix should avoid creating a second inconsistent schema without deciding whether this is a core JSONL sink or a documented plugin-backed runtime service.
- A plan/session event stream can expose private prompt/tool/session metadata; implementation should define redaction, opt-in config, rotation/path ownership, and schema stability before shipping.
