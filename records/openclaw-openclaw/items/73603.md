## Summary

Keep open. Current main still has in-process plan, tool, lifecycle, task, plugin-hook, and SDK event surfaces, but it does not provide the requested default-off `events.emit.plan` config, canonical `/var/log/openclaw-events.jsonl` writer, stable plan identifiers, `plan_ref` correlation, or terminal `closes_plan` semantics. This is not a conservative ClawHub close because the useful external auditor depends on a core schema/API, redaction, and sink-ownership decision first.

## What This Changes

Request to add a default-off plan/session lifecycle audit stream that writes correlated plan, spawn, yield, and close events to `/var/log/openclaw-events.jsonl`.

## Best Possible Solution

Design an additive observability contract for stable run/plan identifiers, plan update/open records, subagent spawn/yield/end correlation, and terminal close/abort records. The implementation should be disabled by default, document schema and redaction guarantees, include config help, and add focused tests for disabled behavior and lifecycle completeness.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: the requested config keys and JSONL sink are absent, while existing plan and lifecycle events remain separate in-process/run-level surfaces without stable plan close correlation.

## Solution Assessment

Unclear. The audit problem is valid, but the proposed direct `/var/log/openclaw-events.jsonl` writer should be decided by maintainers against plugin hooks, SDK event subscriptions, task registry behavior, schema stability, and redaction requirements before implementation starts.
