## Summary

Keep open. Current main still does not provide the requested default-off durable plan/session audit stream: `update_plan` only returns an in-turn tool result, plan events are update-only in-process notifications, and subagent/session lifecycle is tracked by run/task/session keys rather than a stable plan-id plus terminal close/abort contract. The request is valid, but it needs maintainer product/API/security design before implementation.

## What This Changes

Requests a default-off config option to emit plan and subagent/session lifecycle records with stable plan IDs and close/abort correlation into `/var/log/openclaw-events.jsonl`.

## Best Possible Solution

Define an additive observability contract with stable plan/run identifiers, plan update/open records, subagent spawn/yield/end correlation, and exactly one terminal close/abort record. Keep it disabled by default, document schema and redaction guarantees, wire config/help/docs, and add focused tests for disabled behavior plus lifecycle completeness.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: the requested config keys, durable JSONL writer, stable plan identifiers, and terminal plan close correlation are absent, while current lifecycle surfaces are in-process or task/run scoped.

## Solution Assessment

Unclear. A default-off durable JSONL stream is plausible, but maintainers should first choose the schema, redaction/access model, sink ownership, and core-versus-plugin boundary before an implementation PR is safe.
