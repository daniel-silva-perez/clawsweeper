## Summary

Keep open. Current main still does not provide the requested default-off durable plan/session audit stream: plan events are in-process update-only events, `update_plan` does not create a stable plan identity, and subagent/session lifecycle tracking is run/task scoped rather than a terminal plan correlation contract. The request is valid and well-evidenced, but it needs maintainer product/API/security design before implementation.

## What This Changes

The issue requests a default-off config flag that emits plan and session lifecycle records to a canonical JSONL audit stream for task-completion monitoring.

## Best Possible Solution

Define an additive observability contract with stable plan/run identifiers, plan update/open records, subagent spawn/yield/end correlation, and exactly one terminal close/abort record. Keep it disabled by default, document schema and redaction guarantees, wire config/help/docs, and add focused tests for disabled behavior plus lifecycle completeness.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: the requested config keys, durable JSONL writer, stable plan identifiers, and terminal plan close correlation are absent, while current lifecycle surfaces are in-process or task/run scoped.

## Solution Assessment

Unclear. The requested default-off durable JSONL stream is plausible, but maintainers should first choose the schema, redaction/access model, sink ownership, and core-versus-plugin boundary before an implementation PR is safe.
