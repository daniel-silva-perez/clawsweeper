## Summary

Keep open. Current main still has adjacent observability surfaces, but it does not implement the requested default-off plan/session event contract, canonical JSONL sink, stable plan identifiers, spawn/yield correlation, or terminal close/abort semantics. This is not a ClawHub close because plugins can observe existing events, but the missing plan/session lifecycle schema appears to need core or plugin-SDK design first.

## What This Changes

Request a default-off `events.emit.plan` option that writes stable plan/session lifecycle JSONL events, including plan IDs and terminal close/abort correlation, to `/var/log/openclaw-events.jsonl`.

## Best Possible Solution

Keep this issue open for maintainer observability design. The end state should be an additive, documented contract for stable plan/run IDs, update/open events, subagent spawn/yield/end correlation, exactly-one terminal close/abort/timeout semantics, redaction rules, and focused schema tests; the durable writer should then use that contract without hardcoding brittle internals.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: current main has no requested config key or sink, `update_plan` returns only an update result, plan events are in-process update payloads, and targeted search finds no `plan_id`/`plan_ref`/`closes_plan` contract.

## Solution Assessment

Unclear. The stable plan/session lifecycle contract is the right direction, but the exact `/var/log` JSONL writer and config shape need maintainer design; a configurable writer or plugin-backed sink over a core schema may be safer.
