## Summary

Keep open. Current main has related in-process agent lifecycle, tool, plan-update, task, plugin-event, and trajectory surfaces, but it still lacks the requested default-off `events.emit.plan` contract, canonical `/var/log/openclaw-events.jsonl` writer, stable plan IDs, `plan_ref` correlation, and terminal `closes_plan` semantics. This is also not a clean ClawHub close because the requested audit stream needs a core schema/API decision before a plugin can reliably implement it.

## What This Changes

Request a default-off configuration option that emits plan and session lifecycle events to a canonical JSONL audit stream with plan/session correlation and terminal close or abort records.

## Best Possible Solution

Design an additive observability contract that exposes stable run/plan identifiers, `update_plan` open/update records, terminal close/abort records, and subagent spawn/yield/end correlation through either a default-off core JSONL sink or a documented plugin API capable of producing that sink safely. The implementation should include schema docs, redaction rules, config help, and focused tests for disabled-by-default behavior and lifecycle completeness.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: the requested config keys and JSONL sink are absent, while `update_plan`, agent lifecycle, tool events, and trajectory capture remain separate in-process or per-session surfaces without plan close correlation.

## Solution Assessment

Unclear. The requested stable audit contract is the right problem to solve, but the exact `/var/log/openclaw-events.jsonl` core-writer shape should be decided by maintainers against the existing plugin event and trajectory systems before a narrow implementation PR starts.
