## Summary

Keep open. Current main still lacks the requested durable plan/session lifecycle audit contract for `/var/log/openclaw-events.jsonl`; the existing plan, tool, lifecycle, task, plugin-event, and trajectory surfaces are adjacent but do not provide stable plan IDs, `plan_ref` correlation, or terminal close/abort semantics. This is not a ClawHub cleanup because a plugin cannot reliably implement the requested audit stream without a core schema/API decision first.

## What This Changes

The issue requests a default-off `events.emit.plan` option that writes plan/session lifecycle records to `/var/log/openclaw-events.jsonl` for external task-completion auditing.

## Best Possible Solution

Design an additive observability contract that exposes stable plan/run identifiers, plan open/update events, terminal close/abort events, and subagent spawn/yield/end correlation through either a default-off gateway JSONL sink or a documented plugin API capable of producing that sink safely. The finished change should include schema documentation, config help, redaction rules, and focused tests proving disabled-by-default behavior and lifecycle completeness.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: the requested config keys and canonical JSONL sink are absent, while current plan and session lifecycle data remains split across in-process agent events, tool events, task records, and trajectory sidecars without stable plan close correlation.

## Solution Assessment

Unclear. The requested audit stream addresses a real observability gap, but the exact `/var/log/openclaw-events.jsonl` core-writer shape should be decided by maintainers against existing plugin events, task registry, and trajectory capture before a narrow implementation starts.
