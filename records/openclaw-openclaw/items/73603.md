## Summary

Keep open. Current main still lacks the requested default-off plan/session audit contract: there is no `events.emit.plan` config, no `/var/log/openclaw-events.jsonl` writer, no stable plan identifiers, and no terminal `closes_plan` correlation. The request is valid, but it needs maintainer product/API/security decisions before implementation.

## What This Changes

Request to add a default-off config flag that emits correlated plan, subagent spawn/yield, and plan close events to `/var/log/openclaw-events.jsonl`.

## Best Possible Solution

Define an additive observability contract for stable plan/run identifiers, plan update/open records, subagent spawn/yield/end correlation, and terminal close/abort records. Keep it disabled by default, document schema and redaction guarantees, add config help, and cover disabled behavior plus lifecycle completeness with focused tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: the requested config keys and JSONL sink are absent, while current plan and lifecycle events remain separate in-process/run-level surfaces without stable plan close correlation.

## Solution Assessment

Unclear. The direct `/var/log/openclaw-events.jsonl` proposal may be viable, but maintainers should decide it against plugin hooks, SDK subscriptions, task registry behavior, schema stability, and redaction before a fix is queued.
