## Summary

Keep open. Current main has related observability surfaces, including the `update_plan` tool, in-process agent event streams, plugin agent-event subscriptions, subagent lifecycle hooks, background task records, and trajectory sidecars, but it still does not implement the requested default-off `events.emit.plan` contract or a canonical `/var/log/openclaw-events.jsonl` plan/session lifecycle stream with stable plan IDs and terminal close/abort semantics. This is not a clean ClawHub close because the missing stable plan/session correlation contract appears to require core/API design before a plugin can produce the requested audit stream safely.

## Best Possible Solution

Keep this issue open for maintainer design. The best path is an additive core or plugin-SDK observability contract: stable plan/run IDs, `update_plan` open/update events, subagent spawn/yield/end correlation, exactly-one terminal close/abort/timeout semantics, redaction rules, docs, and focused tests. The JSONL writer can be default-off or plugin-backed, but the stable lifecycle schema needs to come from the core agent/session contract first.
