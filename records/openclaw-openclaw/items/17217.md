## Summary

Keep this issue open. Current main documents session boundaries and exposes reset/compact controls, but the requested wall-clock TTL pruning for normal conversation history and a manual prune-context API are still missing. The existing cache-ttl path remains an in-memory tool-result prompt-cache optimization, not a session-history expiry mechanism.

## Best Possible Solution

Keep the issue open for maintainer product work. Decide whether contextPruning.ttl should expire or summarize normal conversation history, or remain a tool-result prompt-cache optimization. If full TTL pruning is desired, add an explicit gateway/protocol prune surface and implement age-based history removal or summarization in the Pi/context-engine path with focused tests and docs. If current behavior is intentional, update user-facing docs to make that limitation explicit and point users to /compact, /new, and /reset as the supported controls.
