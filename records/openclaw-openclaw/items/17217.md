## Summary

Keep open. Current main documents session boundaries and exposes reset/compact controls, but the requested wall-clock TTL pruning of normal conversation history and a manual prune-context API are still not implemented. The existing `contextPruning.mode: "cache-ttl"` path remains a prompt-cache/tool-result pruning optimization that runs during context assembly, not a session-history expiry mechanism.

## Best Possible Solution

Keep this issue open for maintainer product work. Decide whether `contextPruning.ttl` should expire/summarize normal conversation history or remain a tool-result prompt-cache optimization. If full TTL pruning is desired, add an explicit gateway/protocol prune surface and implement age-based history removal or summarization in the Pi/context-engine path with tests and docs; if current behavior is intentional, update docs to make the limitation explicit and point users to `/compact`, `/new`, and `/reset` as supported controls.
