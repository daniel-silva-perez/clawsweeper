## Summary

Keep open. Current main documents session lifecycle and exposes reset/compact controls, but `contextPruning.mode: "cache-ttl"` is still a tool-result prompt-cache optimization that runs during context assembly, not wall-clock expiry of normal conversation history, and there is still no manual prune-context gateway/API surface.

## Best Possible Solution

Keep this open for maintainer product work. Decide whether `contextPruning.ttl` should remain a prompt-cache/tool-result optimization or become normal conversation-history expiry. If full TTL pruning is desired, add an explicit gateway/protocol prune surface plus Pi/context-engine implementation and tests; if current behavior is intentional, update docs to say TTL does not expire normal conversation history and point users to `/compact`, `/new`, and `/reset` as supported controls.
