## Summary

Keep open. Current main still does not implement dynamic low default temperatures for tool-enabled embedded-runner turns, and the PR head still needs maintainer rebase/split work because it suppresses the new behavior on reasoning-visibility settings, misses compaction runs, and carries unrelated secrets and lockfile changes.

## What This Changes

The PR adds dynamic low temperature defaults for tool-enabled embedded-runner turns, forwards tool names through normal attempts, expands extra-param tests, and also modifies file-secret allowInsecurePath handling, lockfile importers, and the changelog.

## Best Possible Solution

Land the agent-temperature behavior as a focused agent-only change that preserves explicit user parameters and plain-chat omission, gates only on active provider generation thinking constraints, passes tool-name context through both normal attempts and compaction, and drops the unrelated secrets, lockfile, and stale changelog hunks.
