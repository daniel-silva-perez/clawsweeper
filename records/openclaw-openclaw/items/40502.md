## Summary

Keep open. Current main has not implemented the tool-temperature heuristic, so this PR is not obsolete, but it is dirty against main and not mergeable as-is because the final diff suppresses the new default when reasoning visibility is enabled and still carries stale, unrelated security-sensitive changes.

## What This Changes

The branch adds dynamic default temperatures for embedded tool-enabled runs, passes allowed tool names into extra-params handling, expands related tests, and also carries stale secrets, lockfile, and changelog changes.

## Best Possible Solution

Rebase onto current main, drop stale secrets/lockfile/changelog changes, and implement the temperature default only from actual tool availability plus true thinking/extended-reasoning activation signals while preserving plain-chat and explicit-temperature behavior.

## Reproduction Assessment

Yes. The review path is high-confidence from the PR diff and docs: with tools available, `reasoningLevel: "on"` makes the new helper skip dynamic temperature even though `/reasoning` is only a visibility setting.

## Solution Assessment

No, not as-is. The target behavior belongs in the embedded extra-params/attempt surface, but the current branch needs a rebase, narrower scope, and a corrected thinking-vs-visibility gate before it is the maintainable fix.
