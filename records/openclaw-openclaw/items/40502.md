## Summary

Keep open. Current main still lacks the requested dynamic default temperature for tool-enabled turns, and this PR is useful but not mergeable as-is because the branch suppresses its main behavior when reasoning visibility is enabled, misses the compaction tool path, and still carries unrelated stale security/package/changelog hunks.

## What This Changes

The PR adds embedded-runner dynamic default temperatures based on available tool names, expands related tests, and also includes file-secret allowInsecurePath, lockfile importer, and changelog changes.

## Best Possible Solution

Land a focused agent-only change that preserves explicit user temperatures and plain-chat omission, opts out only for actual provider generation-thinking constraints, applies the same tool-name heuristic to normal and compaction runs, and drops the stale secrets, lockfile, and unrelated changelog changes.
