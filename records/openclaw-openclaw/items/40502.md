## Summary

Keep open. Current main still does not implement dynamic low-temperature defaults for tool-enabled embedded-runner calls, so the PR is not obsolete, but the branch is not merge-ready because it suppresses the new default on reasoning-visibility settings, misses compaction behavior, and carries unrelated stale security-sensitive secrets and lockfile hunks.

## What This Changes

The PR adds dynamic default temperatures for embedded-runner tool calls, focused tests and a changelog entry, while also changing file-secret allowInsecurePath handling and pnpm lockfile importers.

## Best Possible Solution

Land a focused agent-only rebase that preserves explicit user temperatures and plain-chat omission, skips injected temperatures only when actual provider generation thinking is active, passes the tool-name heuristic through both normal attempts and compaction, and drops the stale secrets, lockfile, and changelog carry-over hunks.

Yes. ## Solution Assessment

No. The requested direction is reasonable, but this branch is not the narrowest maintainable solution until it is rebased onto current main, the reasoning-vs-thinking gate is corrected, compaction is wired, and unrelated security/package churn is split out.
