## Summary

Keep open. Current main still lacks the requested tool-run temperature default, so the PR is not obsolete, but the proposed branch is not ready to merge because the agent heuristic still has behavioral gaps and the branch carries unrelated security-sensitive changes.

## What This Changes

The PR adds low default embedded-runner temperatures for tool-enabled and shell-like turns, expands runner tests, and also includes secrets config/resolver, lockfile, and changelog changes.

## Best Possible Solution

Land the agent-temperature idea as a focused agent-only change that preserves explicit user parameters and plain-chat omission, gates only on active generation thinking constraints, passes tool-name context through both normal attempts and compaction, and drops unrelated secrets, lockfile, and stale changelog hunks.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Gate on active thinking, not reasoning visibility:** `src/agents/pi-embedded-runner/extra-params.ts:338-340`
  - body: `reasoningLevel` controls whether reasoning text is shown, not whether provider thinking is active. With this check, tool runs using `/reasoning on` skip the new low-temperature default even when generation thinking is off, disabling the main behavior change for those sessions.
  - confidence: 0.9
- **[P2] Apply tool-temperature defaults to compaction runs:** `src/agents/pi-embedded-runner/run/attempt.ts:1966`
  - body: This only forwards tool names through normal attempts. Compaction already computes `allowedToolNames` and creates a tool-capable session, but extra params are still applied without that override, so post-compaction tool planning stays on provider-default temperature.
  - confidence: 0.86
