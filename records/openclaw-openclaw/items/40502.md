## Summary

Keep open. Current main still does not implement dynamic low-temperature defaults for tool-enabled embedded-runner turns, so the PR addresses real unshipped behavior, but the branch should not merge as-is because it gates on reasoning visibility, misses the compaction path, and carries unrelated security-sensitive secrets plus stale lockfile changes.

## What This Changes

The PR adds dynamic low default temperatures for tool-enabled embedded-runner turns, forwards tool names through normal attempts, expands extra-param tests, and also includes unrelated secrets config, lockfile, and changelog changes.

## Best Possible Solution

Land the agent-temperature behavior as a focused agent-only change that preserves explicit user parameters and plain-chat omission, gates only on active provider generation thinking constraints, passes tool-name context through normal attempts and compaction, and drops the unrelated secrets, lockfile, and stale changelog hunks.
