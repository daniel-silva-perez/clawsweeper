## Summary

Keep this PR open. Current main still does not implement the requested lower default temperature for tool-enabled or shell-like embedded agent runs, so the PR is not obsolete. The best maintainer path is a focused rebase/split rather than closure, because the submitted diff also bundles unrelated secrets/config, lockfile, changelog, and test-loosening changes, and the review discussion identifies real provider-thinking, cache-retention, and compaction edge cases that still need design/test coverage.

## Best Possible Solution

Keep this PR open, but require a focused rebase before maintainer review. The narrow fix should limit the diff to embedded runner temperature defaulting; pass tool-name context through a typed non-stream-param seam for normal attempts and compaction; preserve explicit configured temperatures; respect Anthropic and other provider thinking/reasoning constraints; avoid cache-retention side effects; drop unrelated secrets/config, lockfile, changelog, and test-loosening hunks; and add focused tests for plain chat, generic tools, shell-like tools, explicit temperature, thinking-enabled cases, cache retention, and compaction.
