## Summary

Keep this PR open. Current main still does not implement low default temperatures for embedded tool runs, so the PR is not obsolete, but the branch is not merge-ready: the latest diff still suppresses the intended default based on reasoning visibility, misses compaction tool runs, and mixes the agent change with unrelated secret-path and lockfile/package-surface changes.

## What This Changes

The PR lowers default embedded-agent temperatures for tool-enabled and shell-like runs while preserving provider defaults for plain chat turns.

## Best Possible Solution

Keep the useful idea, but require a focused rebase that limits the diff to embedded-runner temperature behavior, passes typed tool-name context through both normal attempts and compaction, preserves explicit temperature and cache-retention semantics, respects provider thinking/reasoning constraints, and drops unrelated secrets, lockfile, and stale changelog hunks.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Stop treating reasoning visibility as active thinking:** `src/agents/pi-embedded-runner/extra-params.ts:339-340`
  - body: `reasoningLevel` controls whether reasoning text is shown (`on`/`stream`), not whether the model has active thinking enabled. With this branch, any tool run that has reasoning visibility enabled skips the new low-temperature default and falls back to the provider default, so the main behavior change is disabled for those sessions. Gate on generation thinking/reasoning params instead of `/reasoning` visibility.
  - confidence: 0.9
- **[P2] Wire the tool-temperature heuristic into compaction runs:** `src/agents/pi-embedded-runner/run/attempt.ts:1966`
  - body: This only passes `availableToolNames` through the normal attempt path. Compaction already computes `allowedToolNames` and creates a tool-capable session, but it still calls `applyExtraParamsToAgent` without an override, so tool planning after compaction keeps provider-default temperature instead of the new default.
  - confidence: 0.86
