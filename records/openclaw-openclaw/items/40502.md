Keep this PR open. Current main still does not implement the requested dynamic low default temperature for tool-enabled or shell-like embedded agent runs. The PR remains a plausible implementation candidate, but the latest provided diff still needs maintainer review and likely a focused rebase/split because it includes unrelated secrets/config and lockfile/package-surface changes plus previously discussed provider-thinking and compaction edge cases.

Keep this PR open, but require a focused rebase before maintainer review: limit the diff to the embedded runner temperature heuristic, drop unrelated secrets/config/lockfile/test-loosening hunks, pass tool-name context through the normal attempt and compaction extra-params/runtime-plan paths, preserve explicit configured temperatures, respect provider thinking/reasoning constraints, avoid unintended cache-retention changes, and add focused tests for plain chat, generic tools, shell-like tools, explicit temperature, thinking-enabled cases, and compaction.
