## Summary

Keep open. Current main still exposes the bundled `coding-agent` skill, and its Claude Code examples still launch `claude --permission-mode bypassPermissions --print` directly without unsetting `CLAUDECODE`, `CLAUDE_CODE_SESSION`, or `CLAUDE_CODE_ENTRYPOINT`. The existing Claude CLI backend hardening is scoped to OpenClaw-managed `claude-cli` backend execution and does not cover the raw `bash background:true` path this skill tells agents to use.

## Best Possible Solution

Keep this issue open and patch the bundled `skills/coding-agent/SKILL.md` Claude Code examples to unset `CLAUDECODE`, `CLAUDE_CODE_SESSION`, and `CLAUDE_CODE_ENTRYPOINT` before launching `claude`. Add a small bundled-skill guidance regression test, likely alongside `src/agents/skills.env-path-guidance.test.ts`, and separately consider whether the host exec sanitizer should drop Claude nesting variables for broader protection.
