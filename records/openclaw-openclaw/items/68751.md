## Summary

Keep open. Current main still has the reported session-memory persistence behavior: `/reset` and `/new` both save prior user/assistant transcript text into `memory/YYYY-MM-DD-*.md` under `## Conversation Summary` without untrusted-data markers or a neutral summary. There is newer startup-context framing for bare `/new`/`/reset`, but that is only one injection path; `memory/*.md` is still indexed/read by memory tools, and the saved file content remains transcript-shaped.

## Best Possible Solution

Keep this issue open and fix it in the bundled `session-memory` hook: stop persisting on `/reset` or make `/reset` explicitly forget, and for `/new` either wrap persisted conversation content in clear untrusted historical markers before it reaches any memory file/recall path or replace raw turns with a neutral generated summary. Add regression coverage for both the saved file shape and the `/reset` behavior, while coordinating any malformed-token stripping with the #69943/#69961 sanitizer work.
