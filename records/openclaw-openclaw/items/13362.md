## Summary

Keep open. Current main still lacks a first-class `openclaw doctor --dry-run`: the doctor command does not register the flag, the typed options cannot carry it, docs omit it, and several doctor repair paths can still write config, install dependencies, rewrite cron state, or repair services. The related global dry-run issue and open `--check-config` PR are adjacent scopes, not superseding fixes for this request.

## What This Changes

Add an `openclaw doctor --dry-run` mode that runs doctor validation and shows the repair plan without prompting for or applying config, dependency, cron, or service mutations.

## Best Possible Solution

Add a native doctor dry-run contract that registers and types the flag, propagates it through repair-mode, prompter, config, and health contribution flows, renders the same recommendations without prompts or mutations, defines non-zero exit behavior when fixable issues are found, and updates the public doctor docs and changelog.

Implement https://github.com/openclaw/openclaw/issues/13362 as a focused native `openclaw doctor --dry-run` feature. Add and type the CLI flag, thread it through `DoctorOptions`, repair-mode/prompter/config-flow/health contribution paths, and ensure dry-run renders the same repair recommendations without prompts or writes. Suppress all doctor side effects during dry-run, including `replaceConfigFile`, bundled plugin runtime dependency installation, cron store normalization writes, gateway daemon install/start/restart, gateway service config rewrites, legacy service cleanup, and any gateway token persistence. Define and test exit behavior: dry-run should not mutate state and should return a non-zero result when actionable issues are found. Keep #41418 global agent/tool dry-run and PR #47302 `--check-config` separate; do not implement a global tool dry-run or merge check-config behavior into this PR. Update `docs/cli/doctor.md` and add a user-facing changelog entry with proper attribution. Do not change unrelated doctor migrations or dependency versions.
