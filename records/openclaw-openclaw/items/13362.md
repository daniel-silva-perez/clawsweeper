## Summary

Keep open. Current main still lacks a native `openclaw doctor --dry-run` surface: the doctor CLI does not register the flag, `DoctorOptions` has no dry-run state, the doctor docs omit it, and the existing preview/confirmation path can still write config when accepted. Related #41418 and #47302 cover different dry-run/check-config scopes rather than superseding this request.

## Best Possible Solution

Keep this open and define a native `openclaw doctor --dry-run` contract. The implementation should add a doctor option and docs, propagate dry-run through `DoctorOptions`/prompter/config flow, collect the same validation and repair recommendations without prompts or writes, block all doctor repair side effects, report a clear change plan or diff, and define exit-code behavior. Coordinate wording with `openclaw config set --dry-run`, while keeping #41418 as the separate global agent-tool dry-run scope and #47302 as the separate deep `--check-config` validation scope.
