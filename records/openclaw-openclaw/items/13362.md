## Summary

Keep open. Current main at f2405c830bd925bd24f91b6bac5d417c6d36e5ba still lacks a native `openclaw doctor --dry-run` surface: the doctor CLI does not register the flag, `DoctorOptions` has no dry-run state, the config finalization path can still mark repairs for writing, and the write-health contribution persists accepted repairs. Related #41418 and #47302 cover different dry-run/check-config scopes rather than superseding this request.

## Best Possible Solution

Implement a native `openclaw doctor --dry-run` contract on this issue: register and type the option, propagate it through the prompter/config/health flow, collect the same validation and repair recommendations without prompts or writes, produce a clear plan or diff, define exit-code behavior, document it at https://docs.openclaw.ai/cli/doctor, and keep #41418 and #47302 as separate scopes.

It should show a clear change plan or diff, return a non-zero exit code when actionable issues are found if that matches the existing doctor command contract, and leave existing `openclaw doctor`, `--fix`, `--repair`, `--yes`, and `--non-interactive` behavior unchanged outside dry-run.
