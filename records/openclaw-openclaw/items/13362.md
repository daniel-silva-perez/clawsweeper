## Summary

Keep open. Current main still has no first-class `openclaw doctor --dry-run` contract: the doctor command does not register the flag, `DoctorOptions` cannot carry it, docs omit it, and doctor still has write/install/service-repair paths that would need explicit dry-run suppression. The related global agent dry-run issue and `--check-config` PR cover different scopes.

## What This Changes

Add `openclaw doctor --dry-run` so users can preview doctor validation and repair recommendations without writing config or performing other doctor side effects.

## Best Possible Solution

Add a native doctor dry-run mode that registers and types the flag, propagates it through doctor prompter/config/health flows, renders the same repair plan without prompts or writes, suppresses non-config side effects, defines exit behavior, and documents the command on the public doctor page.

## Repair Work Prompt

Implement `openclaw doctor --dry-run` for this issue. Add a first-class dry-run option to the doctor CLI, type it in `DoctorOptions`, and propagate it through the doctor prompter/config/health contribution flows. In dry-run mode, doctor should perform validation and render the same repair recommendations/plan, but must not prompt for changes, write config, install bundled plugin runtime deps, rewrite cron state, install/bootstrap/restart gateway services, or perform other doctor side effects. Define and test the exit-code behavior for issues found versus clean runs. Keep #41418 global agent dry-run and #47302 `--check-config` as separate scopes. Update https://docs.openclaw.ai/cli/doctor source docs and add a user-facing changelog entry crediting @Riya-chandra. Do not broaden this into a global tool dry-run feature or a deep config-validation feature.
