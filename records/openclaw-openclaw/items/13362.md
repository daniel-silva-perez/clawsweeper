## Summary

Keep open. Current main still does not implement a first-class `openclaw doctor --dry-run`: the command registration and typed doctor options have no dry-run flag, public doctor docs omit it, and the repair/write paths still include config writes plus other mutable repairs that would need an explicit dry-run contract. The adjacent global dry-run issue and open `--check-config` PR cover different layers and do not supersede this request.

## What This Changes

Request adding `openclaw doctor --dry-run` so doctor validation and fix recommendations can be previewed without writing config or performing repair side effects.

## Best Possible Solution

Add a native doctor dry-run contract that registers and types the flag, propagates it through repair mode, prompting, config, and health contribution flows, renders the same recommendations without prompts or mutations, returns a defined non-zero status when fixable issues are found, and updates the public doctor docs and changelog.

## Reproduction Assessment

Not applicable as a feature request, but the missing behavior is directly verifiable: `openclaw doctor --dry-run` is not registered in the CLI, cannot be represented by `DoctorOptions`, and is absent from the docs on current main.

## Solution Assessment

Yes, the requested native doctor dry-run is the narrowest maintainable solution for this issue because it targets doctor’s own mutation paths; the global agent dry-run and `--check-config` work are useful adjacent features but do not prevent doctor repairs from writing or installing during a preview.
