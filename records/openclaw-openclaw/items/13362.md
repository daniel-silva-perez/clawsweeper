## Summary

Keep open. Current main still does not implement a native `openclaw doctor --dry-run` path: the doctor command has no flag, the typed doctor options cannot carry dry-run state, docs omit the command, and several doctor repair paths still have direct write/install/service side effects. The related global agent dry-run request and open `--check-config` PR cover different scopes and do not supersede this request.

## What This Changes

The issue requests a native `openclaw doctor --dry-run` mode that validates and previews doctor repairs without writing config or performing other repair side effects.

## Best Possible Solution

Add a first-class doctor dry-run contract that registers and types the flag, propagates it through the prompter/config/health contribution flows, renders the repair plan without prompts or writes, suppresses non-config side effects, defines exit-code behavior, and documents the command on https://docs.openclaw.ai/cli/doctor.
