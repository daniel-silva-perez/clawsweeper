## Summary

Keep open. Current main still does not implement a native `openclaw doctor --dry-run` contract: the doctor CLI does not register the flag, `DoctorOptions` has no dry-run state, the config finalization path can still mark repairs for writing, and the write-health contribution still persists accepted/detected config changes.

## What This Changes

The issue requests a native `openclaw doctor --dry-run` mode that validates and previews doctor repair recommendations without writing configuration changes.

## Best Possible Solution

Implement a focused native `openclaw doctor --dry-run` contract: register and type the option, propagate it through prompter/config/health flows, collect the same validation and repair recommendations without prompts or writes, show a clear plan/diff, define non-zero exit behavior when issues are found, document it at `https://docs.openclaw.ai/cli/doctor`, and keep #41418 and #47302 as separate scopes.
