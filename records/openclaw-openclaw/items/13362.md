## Summary

Keep this issue open. Current main has safer config write-back protections, including env-var reference restoration, and normal `doctor` can preview recommended config repairs before prompting. But the requested `openclaw doctor --dry-run` surface is still missing: the doctor CLI has no `--dry-run` option, `DoctorOptions` has no dry-run state, the docs omit it, and the existing interactive preview can still write when accepted. Related #41418 explicitly treats this as a separate doctor/config dry-run scope, and #47302 tracks a different `--check-config` validation mode rather than this write-free doctor repair preview contract.

## Best Possible Solution

Keep this issue open and define a native `openclaw doctor --dry-run` contract. The best implementation should add a doctor option and docs, run validation and recommendation collection without prompts or writes, block all doctor repair/write side effects, optionally show a structured change plan/diff, and exit non-zero when actionable issues or would-change repairs are found. Coordinate wording with `openclaw config set --dry-run`, keep #41418 as the separate global agent-tool dry-run scope, and keep #47302 as the separate deep `--check-config` validation scope.
