## Summary

Keep open. Current main still lacks a first-class `openclaw doctor --dry-run` mode: the doctor command does not register the flag, the typed options cannot carry it, docs omit it, and several doctor repair paths can still write config, install staged plugin dependencies, rewrite cron state, or repair services.

## What This Changes

Request to add a `openclaw doctor --dry-run` mode that previews validation and repair changes without writing to disk, then exits non-zero when issues are found.

## Best Possible Solution

Add a native doctor dry-run contract that registers and types the flag, propagates it through the doctor prompter/config/health flows, renders the same repair plan without prompts or mutations, suppresses non-config repair side effects, defines exit semantics, and updates the public doctor docs.
