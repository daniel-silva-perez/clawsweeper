## Summary

Keep open. Current main still does not provide a first-class `openclaw doctor --dry-run` mode: the doctor command does not register the flag, the typed doctor options cannot propagate it, the docs omit it, and repair paths still have config, dependency-install, cron, and service side effects. The related global dry-run issue and open `--check-config` PR are adjacent scopes, not superseding fixes for this doctor safety request.

## What This Changes

The issue requests a native `openclaw doctor --dry-run` mode that previews doctor validation and repairs without writing config or performing other repair side effects.

## Best Possible Solution

Add a native doctor dry-run contract: register and type the flag, propagate it through the prompter, repair mode, config, and health contribution flows, render the same repair plan without prompts or mutations, suppress dependency install, cron, gateway service, daemon, and config writes, define exit behavior, and update the public doctor docs plus changelog.
