## Summary

Keep open. Current main still lacks a native `openclaw doctor --dry-run` contract: the doctor command does not register the flag, the typed options do not carry it, the config flow can still prompt/apply repairs, and the health contribution can still persist config changes.

## What This Changes

The issue requests a native `openclaw doctor --dry-run` mode that validates and previews doctor repair recommendations without writing configuration changes.

## Best Possible Solution

Add a focused native doctor dry-run mode that registers and types the option, propagates it through prompter/config/health flows, collects the same repair plan without prompts or writes, reports the planned changes clearly, defines non-zero behavior when issues are found, and documents the command on the public doctor CLI page while keeping #41418 and #47302 separate.
