## Summary

Keep open. Current main still does not implement a native doctor dry-run path: the CLI has no `doctor --dry-run` option, the typed doctor options cannot carry dry-run state, docs omit the command, and multiple doctor repair paths can still write config, save cron state, install plugin runtime dependencies, or repair services. The related global agent dry-run issue and the open `--check-config` PR cover different scopes and do not supersede this request.

## What This Changes

Request to add `openclaw doctor --dry-run` so users can preview validation and repair changes without writing config or triggering doctor side effects.

## Best Possible Solution

Add a first-class doctor dry-run contract that registers and types the flag, propagates it through the prompter/config/health contribution flows, renders the same repair plan without prompts or writes, suppresses non-config side effects, defines exit-code behavior, and documents the command on https://docs.openclaw.ai/cli/doctor.
