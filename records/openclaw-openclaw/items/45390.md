## Summary

Keep this issue open. Current main has improved daily/idle reset behavior and prevents heartbeat/cron/exec system events from keeping stale sessions fresh, but it still does not implement the requested generic session TTL surface: there is no session max-turns policy, no general max-age duration field like `sessionMaxAge`, and the bundled `session-memory` hook is still scoped to explicit `/new` and `/reset` commands rather than automatic rotation. The related #42867 is useful and overlapping for turn-count TTL, but it is narrower than this issue's max-age plus safe pre-rotation persistence request.

## Best Possible Solution

Keep this issue open or explicitly consolidate it into a maintainer-approved session-lifecycle umbrella. The concrete implementation path is a generic session reset policy extension, likely under `session.reset`, with max-age and max-turn evaluation based on persisted lifecycle timestamps and counted conversational turns, plus automatic rotation logging and safe persistence behavior through session lifecycle hooks or bundled `session-memory` support. Coordinate the max-turn portion with #42867 and the automatic session-memory behavior with #42677/#50933.
