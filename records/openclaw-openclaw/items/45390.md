## Summary

Keep this issue open. Current main has daily and idle reset freshness, heartbeat-aware timestamps, heartbeat isolated sessions, session lifecycle hooks, and pre-compaction memory flush support, but it still does not expose the requested normal-session max-age or max-turn TTL policy. The remaining work overlaps with open same-author #42867 and open automatic session-memory reset PRs, so it needs maintainer consolidation rather than automatic closure.

## Best Possible Solution

Keep open for maintainer consolidation. The best path is to choose whether #42867 becomes the canonical turn-count slice or this issue remains the umbrella, then design a normal-session reset policy with explicit max age and max conversation turns, persisted turn accounting, clear system-event semantics, lifecycle reasons/logging, and coordinated memory persistence through existing lifecycle hooks or the open automatic session-memory reset work. Update config types/schema/docs and add focused reset-policy plus `initSessionState` regression tests.
