## Summary

Keep this open. Current main has daily/idle session freshness, heartbeat-aware lifecycle timestamps, isolated heartbeat sessions, and lifecycle hooks, but it still lacks the requested configurable normal-session max-age and max-turn rotation policy.

## What This Changes

The issue requests configurable session maximum age and maximum turn limits that automatically rotate long-lived agent sessions while preserving memory safely.

## Best Possible Solution

Define one generic session reset contract that supports explicit max age and max conversation turns, persists the needed accounting, excludes or deliberately handles system-event traffic, emits clear lifecycle reasons/logs, and coordinates pre-rotation memory persistence through the existing hook/lifecycle system.

## Reproduction Assessment

Not applicable as a feature request. We do have high-confidence verification that the requested knobs are absent by inspecting the current config types, strict schema, runtime freshness gate, docs, and targeted source search.

## Solution Assessment

Unclear until maintainer product/API review. The narrow maintainable direction is a generic session reset policy, but the public config shape and turn-count semantics should be consolidated with #42867 and the automatic session-memory PRs before implementation.
