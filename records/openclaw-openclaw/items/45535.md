## Summary

Keep this PR open. Current main still has the mixed zero/positive token alias gap the PR targets, and the PR is stale against the active CLI usage parser. The work is still useful, but it needs a refreshed patch and regression coverage before it can land.

## Best Possible Solution

Refresh this PR or replace it with a current-main patch that updates `src/agents/usage.ts` and `src/agents/cli-output.ts`: choose the first positive alias before falling back to finite zero values, add `prompt_tokens` / `promptTokens` and `completion_tokens` / `completionTokens` CLI aliases, preserve the zero-only JSONL guard, and add regression tests for the mixed zero/positive alias cases.
