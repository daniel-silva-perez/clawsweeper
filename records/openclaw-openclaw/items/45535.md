## Summary

Keep this PR open. Current main still has the mixed-alias token normalization gap this PR targets, and the current CLI usage parser still lacks `prompt_tokens` / `completion_tokens` aliases. The submitted patch is also stale against the current parser location and needs the existing zero-only JSONL overwrite concern addressed before it can be merged safely.

## Best Possible Solution

Refresh this PR or replace it with a current-main patch that updates `src/agents/usage.ts` and `src/agents/cli-output.ts`: select the first positive alias before falling back to finite zero values, add `prompt_tokens` / `promptTokens` and `completion_tokens` / `completionTokens` CLI aliases, preserve `undefined` for all-zero usage events, and add regression tests for mixed zero/positive aliases plus the JSONL zero-only guard.
