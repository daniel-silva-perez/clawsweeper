## Summary

Keep this PR open. Current main still has the specific mixed-alias token normalization gap the PR targets, and the active CLI usage parser still lacks `prompt_tokens` / `completion_tokens` aliases. The PR is also stale against the current parser location and needs the existing all-zero JSONL overwrite concern handled before it can be merged.

## Best Possible Solution

Refresh this PR or replace it with a current-main patch that updates `src/agents/usage.ts` and `src/agents/cli-output.ts`: choose the first positive alias before falling back to finite zero values, add `prompt_tokens` / `promptTokens` and `completion_tokens` / `completionTokens` CLI aliases, keep all-zero usage events as `undefined`, and add regression tests for mixed zero/positive aliases plus the JSONL zero-only guard.
