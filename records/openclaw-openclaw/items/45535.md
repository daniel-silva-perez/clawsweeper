## Summary

Keep this PR open. Current main still has the mixed zero/positive usage-alias gap the PR is trying to fix, and the submitted patch is stale against the active CLI usage parser. The linked Bailian/OpenAI-compatible usage reports are closed, but this PR's narrower behavior is not implemented on main yet.

## Best Possible Solution

Refresh this PR or replace it with a current-main patch that updates `src/agents/usage.ts` and `src/agents/cli-output.ts`: choose the first positive alias before falling back to finite zero values, add CLI support for `prompt_tokens` / `promptTokens` and `completion_tokens` / `completionTokens`, preserve the zero-only JSONL guard, and add regression tests for mixed zero/positive alias payloads.
