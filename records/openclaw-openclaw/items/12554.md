## Summary

Keep open. Current main still has lower-level supervised exec/process primitives and subagent-run kill plumbing, but it does not expose the requested session-wide operator surface to list and terminate all active exec children for an arbitrary stored session. The prior discussion already identified the same gap, and inspection of origin/main at 12a82aa78855f73377dc4233d3875a6576ba69c3 still matches that conclusion.

## What This Changes

Request a session-level command or Gateway API that lists running exec processes spawned by a session and terminates those processes as a group.

## Best Possible Solution

Add an explicit additive Gateway protocol and CLI operator that resolves the canonical session key, lists active supervised exec records for that session scope, terminates matching active runs through supervisor cancellation with process-tree fallback where appropriate, returns per-run status, and documents authorization/isolation rules without silently changing default sessions.abort behavior.
