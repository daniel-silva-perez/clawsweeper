## Summary

Current main still does not implement the requested session-wide operator surface to list and terminate all exec/process runs associated with an OpenClaw session. There are useful per-process controls and active chat-run abort wiring, but `sessions.abort` has no kill/list option, the Gateway has no `sessions.kill` or `sessions.exec-list` RPC, the CLI/docs expose no equivalent, and backgrounded exec runs are intentionally not killed by tool-run abort signals.

## Best Possible Solution

Keep this issue open for maintainer design of an additive Gateway/RPC/CLI operator surface: resolve the canonical session key, list managed exec/process runs for that session, terminate matching runs through supervisor cancellation plus process-tree fallback, return per-run status, and document the contract. The design should preserve current default `sessions.abort` behavior while adding explicit kill/list semantics with clear authorization, session isolation, sandbox/node-host, background exec, and subagent behavior.
