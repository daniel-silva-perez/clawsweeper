## Summary

Keep open. Current main has session-scoped process records, individual `process kill` support, active chat-run aborts, and subagent-kill plumbing, but it still lacks the requested operator surface to list and terminate every active exec child for an arbitrary stored session. Related #39960 appears to cover active stop/abort behavior, while open #38721 is adjacent shutdown cleanup work and does not supersede this broader session-wide process-control request.

## What This Changes

The issue requests a `sessions_kill` or equivalent CLI/Gateway surface that can list and terminate all active `exec` child processes associated with a session.

## Best Possible Solution

Add an explicit additive operator contract that resolves the canonical session key, lists active supervised exec records for that session scope, terminates matching active runs through supervisor cancellation with process-tree fallback where appropriate, returns per-run status, and documents authorization/isolation behavior without silently changing default `sessions.abort` semantics.
