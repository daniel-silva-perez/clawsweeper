## Summary

Keep open. Current main has supervised exec runs, per-session/process scoping, process-tree kill fallbacks, and active-run abort plumbing, but it still lacks the requested operator-facing Gateway/RPC/CLI surface to list and terminate all running exec children for an arbitrary stored session. The related #39960 cleanup appears to cover active stop/abort behavior, while adjacent #38721 remains open and does not supersede this broader session operator request.

## What This Changes

The issue requests a session-level operator command/API to list and terminate all running exec child processes associated with a session.

## Best Possible Solution

Add an explicit, additive operator contract that resolves the canonical session key, lists active supervised exec records for that session scope, terminates matching active runs through supervisor cancellation with process-tree fallback where appropriate, returns per-run status, and documents the authorization and isolation behavior without silently changing default sessions.abort semantics.
