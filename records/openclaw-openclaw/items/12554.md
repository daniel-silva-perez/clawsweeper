## Summary

Keep open. Current main has per-session process scoping, process-tool cancellation, and active run abort support, but it still does not provide the requested operator-facing Gateway/RPC/CLI surface to list and terminate all running exec child processes for an arbitrary stored session. The prior maintainer and ClawSweeper comments reached the same conclusion, and the current checkout still matches that state.

## What This Changes

Request an operator command or abort option that can list and terminate all active exec or system.run child processes associated with a session.

## Best Possible Solution

Add an explicit additive Gateway protocol and CLI surface that resolves a canonical session key, lists active supervised exec records for that session scope, terminates matching active runs through supervisor cancellation with process-tree fallback where appropriate, returns per-run status, and documents authorization and isolation rules without silently changing default sessions.abort behavior.
