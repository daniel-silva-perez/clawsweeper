## Summary

Keep open. Current main has scoped process supervision and same-session `process` tool controls, but it still lacks the requested operator-facing Gateway/RPC/CLI surface to list and terminate active exec child processes for an arbitrary stored session.

## What This Changes

The issue requests a session-level command/API to list and terminate all active exec child processes spawned by a session.

## Best Possible Solution

Add an explicit, additive Gateway protocol and CLI surface that resolves a canonical session key, lists active supervised exec/process records for that session scope, terminates matching active runs through supervisor cancellation with process-tree fallback where appropriate, returns per-run status, and documents authorization and isolation rules without silently changing default `sessions.abort` behavior.

## Reproduction Assessment

Yes for the missing operator surface: current-main source and docs inspection shows no session exec-list/kill command or abort flag, while tests assert background exec survives a tool abort. A live zombie-process scenario was not run because this review was read-only and the source-level absence is enough to keep the feature open.

## Solution Assessment

Unclear as a ready implementation: the requested list/kill capability is the right end state, but overloading `sessions.abort` would blur authorization and behavior. The safer direction is a new additive operator API with explicit scopes and per-run results.
