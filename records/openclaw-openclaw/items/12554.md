## Summary

Keep open. Current main still lacks a session-wide operator surface to list and terminate all exec/process runs owned by a session; existing controls are per-process, active-run abort, task/subagent cancellation, or the subagent-only HTTP kill route.

## What This Changes

The issue requests a `sessions_kill` or equivalent CLI/Gateway operator action that lists and terminates all child exec processes spawned by a session.

## Best Possible Solution

Keep the issue open for maintainer design of an additive Gateway/RPC/CLI operator surface: resolve the canonical session key, list managed exec/process runs for that exact session, terminate matching runs through process-supervisor cancellation plus process-tree fallback, return per-run status, and document the authorization/isolation contract without changing default `sessions.abort` behavior.
