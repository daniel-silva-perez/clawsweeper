## Summary

Keep open. Current main has the lower-level process-supervisor and scoped process-tool pieces, but it still lacks the requested Gateway/RPC/CLI operator surface to list and terminate all exec child processes for an arbitrary stored session. Existing `sessions.abort` only aborts the active chat run, and the HTTP session kill route is subagent-run control rather than session-wide exec cleanup.

## What This Changes

Add a session-level operator command or abort enhancement that can list and terminate all running exec child processes spawned by a session.

## Best Possible Solution

Design an additive Gateway protocol and CLI surface that resolves the canonical session key, lists managed exec/process records for that session scope, terminates matching active runs through supervisor cancellation with process-tree fallback where appropriate, returns per-run status, and documents the authorization and isolation rules without silently changing default `sessions.abort` behavior.
