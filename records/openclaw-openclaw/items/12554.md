## Summary

Keep open. Current main has useful building blocks: exec runs are scoped by session key, the process tool can list/kill individual background exec sessions, the process supervisor can cancel a scope internally, and sub-agent kill routes exist. It still does not expose the requested operator surface to list all running exec processes for an OpenClaw session and terminate that session's exec runs as one unit.

## Best Possible Solution

Keep this issue open for maintainer design of an additive Gateway/RPC/CLI operator surface: resolve the canonical session key, list managed exec/process runs for that session scope, terminate matching runs through supervisor cancellation plus process-tree fallback, return per-run status, document the contract, and cover same-session kill, unrelated-session isolation, background exec, HTTP/RPC/CLI validation, and sub-agent cancellation interactions.
