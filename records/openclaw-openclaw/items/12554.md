## Summary

Keep open. Current main has lower-level supervised process primitives, per-agent `process` tool management, active chat-run aborts, and subagent kill plumbing, but it still does not expose the requested session-wide operator surface to list and terminate all active exec children for an arbitrary stored session.

## What This Changes

Request an operator command/API such as `sessions_kill` or `sessions exec-list` that lists and terminates all still-running exec child processes associated with a session.

## Best Possible Solution

Add an explicit additive Gateway protocol and CLI operator that resolves the canonical session key, lists active supervised exec records for that session scope, terminates matching active runs through supervisor cancellation with process-tree fallback where appropriate, returns per-run status, and documents authorization/isolation rules without silently changing default `sessions.abort` behavior.
