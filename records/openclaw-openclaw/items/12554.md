## Summary

Keep open. Current main has per-session process scoping, individual background `process` list/kill controls, and subagent kill routes, but it still lacks the requested operator surface to list all running exec processes for an OpenClaw session and terminate them as one unit.

## Best Possible Solution

Keep this issue open and design an additive Gateway/RPC/CLI operator surface: resolve the canonical session key, list managed exec runs for that session scope, terminate matching runs through process-supervisor cancellation plus process-tree fallback, return per-process status, document the contract, and cover same-session kill, unrelated-session isolation, background exec, HTTP/RPC/CLI validation, and subagent cancellation interactions.
