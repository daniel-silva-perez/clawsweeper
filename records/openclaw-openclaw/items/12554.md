## Summary

Keep open. Current main has lower-level process scoping and cancellation primitives, and active chat abort can stop the current run, but there is still no session-wide Gateway/RPC/CLI operator that lists and terminates all active exec child processes for an arbitrary stored session. Prior maintainer and ClawSweeper comments reached the same conclusion, and the current checkout still matches it.

## What This Changes

Add a session-wide operator to list and terminate all active exec child processes spawned by a stored session or sub-agent session.

## Best Possible Solution

Add an explicit additive Gateway protocol and CLI surface that resolves a canonical session key, lists active supervised exec records for that session scope, terminates matching active runs through supervisor cancellation with process-tree fallback where appropriate, returns per-run status, and documents authorization and isolation rules without silently changing default sessions.abort behavior.
