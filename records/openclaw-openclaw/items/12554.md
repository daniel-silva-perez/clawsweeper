## Summary

Keep open. Current main still does not provide the requested session-wide operator surface to list and terminate all active exec child processes for an arbitrary stored session. The lower-level scoped process supervisor and per-session process tool exist, and subagent kill now controls subagent runs, but `sessions.abort` and the HTTP session kill route do not enumerate or cancel exec/process records for the target session. The remaining design crosses process visibility and termination boundaries, so it needs maintainer product/security review before an automated repair PR is safe.

## What This Changes

Request an operator-facing `sessions_kill` or equivalent session abort enhancement that lists and terminates all active `exec`/`system.run` child processes associated with a session.

## Best Possible Solution

Add an explicit additive Gateway protocol and CLI operator that resolves the canonical session key, lists active supervised exec records for that session scope, terminates matching active runs through supervisor cancellation with process-tree fallback where appropriate, returns per-run status, and documents authorization/isolation rules without silently changing default `sessions.abort` behavior.
