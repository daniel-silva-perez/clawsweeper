## Summary

Keep this PR open. Current main still has the bug the PR targets: Android onboarding only enables Finish when both the operator session and node session are connected, while the linked bug and PR are specifically about the valid state where the node is connected and the operator is offline. The PR is a focused candidate fix for open issue #61005, with a small cleanup noted by bot review.

## Best Possible Solution

Keep the PR open and review it as the implementation candidate for #61005. The maintainer path is to apply the node-only finish gate, update the unit test as shown, remove or justify the now-unused operator `isConnected` subscription, then run the targeted Android unit test and land if CI is clean.
