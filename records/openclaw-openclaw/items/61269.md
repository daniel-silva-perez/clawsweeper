## Summary

Keep this PR open. Current main still has the linked Android onboarding bug, and this PR is a focused implementation candidate with closing syntax for the open linked issue; the functional direction is correct, with one small cleanup finding before landing.

## What This Changes

This PR changes Android onboarding and its logic test so finishing the flow depends on the Android node connection instead of also requiring the operator session to be connected.

## Best Possible Solution

Land this PR or an equivalent narrow Android fix after removing the unused subscription and validating the focused Android unit test, then let the linked bug close from the merged implementation.

## Reproduction Assessment

Yes. Current main has a high-confidence pure-code reproduction: `NodeRuntime` can produce `Connected (operator offline)` with the node connected, while `canFinishOnboarding(isConnected=false, isNodeConnected=true)` returns false and keeps Finish unavailable.

## Solution Assessment

Yes, with a small cleanup. The root cause is in the Android UI finish gate, and using authenticated node connectivity as the completion signal is narrower and more maintainable than changing gateway pairing or operator reconnect behavior.
