## Summary

Current main still has the TUI event-ordering bug this PR targets: gateway agent/tool events can carry the right session key, but `handleAgentEvent` still rejects unknown run IDs before adopting same-session early events. The PR remains a focused implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open. The best path is to review and land this approach, or an equivalent TUI fix, that adds `sessionKey` to `AgentEvent`, adopts same-session early agent events before the known-run filter, preserves the existing BTW and optimistic-message bookkeeping, and adds focused TUI handler regression tests before merge.
