## Summary

Keep open. Current main still drops the Telegram `audioAsVoice` hint in the current plugin payload adapter, so the intended behavior remains useful, but this PR is not merge-ready because it patches removed legacy Telegram files and its generic delivery change can route text-only `audioAsVoice` payloads around normal text chunking.

## What This Changes

The PR adds `audioAsVoice` propagation to the legacy Telegram outbound adapter, widens generic outbound delivery routing for `audioAsVoice`, and adds regression tests for those paths.

## Best Possible Solution

Rebase or replace this PR with a current-layout fix in the Telegram plugin: map `payload.audioAsVoice === true` to Telegram `asVoice: true` in `extensions/telegram/src/outbound-adapter.ts`, keep text-only payloads on the normal chunking path, and add focused regression coverage in the current Telegram adapter and generic delivery tests.
