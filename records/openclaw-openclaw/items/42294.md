## Summary

Keep this PR open. Current main has partially moved the generic delivery behavior forward, but the Telegram plugin payload adapter still does not translate `payload.audioAsVoice === true` into the `asVoice: true` option needed by `sendMessageTelegram`, so the PR’s intended Telegram voice-note fix is not implemented on main.

## Best Possible Solution

Keep this PR open and rebase the focused fix into the current Telegram plugin layout. The likely implementation is to thread `payload.audioAsVoice === true` from `sendTelegramPayloadMessages` into `sendMessageTelegram` as `asVoice: true` for media sends, add a focused regression in `extensions/telegram/src/outbound-adapter.test.ts`, and handle the text-only routing concern while preserving the generic delivery coverage already on main.
