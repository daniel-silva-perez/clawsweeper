## Summary

Keep this PR open. Current main has generic audioAsVoice propagation, but the Telegram plugin payload adapter still does not map payload.audioAsVoice === true to Telegram asVoice: true, so the intended outbound native voice-note fix is not implemented on main.

## Best Possible Solution

Keep this PR open and ask for a focused rebase onto the current Telegram plugin layout. The best fix is to thread payload.audioAsVoice === true from sendTelegramPayloadMessages into sendMessageTelegram as asVoice: true for media sends, add a regression in extensions/telegram/src/outbound-adapter.test.ts, and adjust the generic routing so text-only audioAsVoice payloads keep normal chunking behavior.
