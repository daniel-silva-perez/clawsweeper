## Summary

Keep this PR open. Current `main` preserves `audioAsVoice` through some generic outbound paths, but the current Telegram plugin payload adapter still does not map `payload.audioAsVoice === true` to Telegram `asVoice: true`, so the intended native voice-note behavior is not implemented for this payload path. The branch is stale against the plugin layout and has a real text-only routing regression in the legacy generic-delivery change, but the narrow Telegram adapter fix remains valid.

## What This Changes

The PR adds `audioAsVoice` propagation through the legacy Telegram outbound adapter and generic outbound delivery routing, with regression tests for Telegram payload sends and outbound delivery.

## Best Possible Solution

Land a focused current-layout fix in the Telegram plugin: pass `asVoice: true` from `payload.audioAsVoice === true` into Telegram media payload sends, add a regression in `extensions/telegram/src/outbound-adapter.test.ts`, and keep text-only delivery on the normal chunking path.
