## Summary

Keep open. The requested Telegram voice-note behavior is still not fully implemented on current main, and this PR is a useful but stale attempt: it targets removed legacy Telegram files and its delivery routing hunk would bypass text chunking for text-only `audioAsVoice` payloads.

## What This Changes

The PR threads `audioAsVoice` into a legacy Telegram outbound adapter, broadens generic outbound `sendPayload` routing for `audioAsVoice`, and adds regression tests for the legacy Telegram and delivery paths.

## Best Possible Solution

Replace or rebase the PR onto the current Telegram plugin layout: convert `audioAsVoice` to Telegram `asVoice` only for actual audio media sends, keep text-only flagged payloads on the chunked text path, and add focused regressions for the current adapter and delivery routing.
