## Summary

Keep this issue open. Current main still lacks a same-product cross-platform `openclaw migrate export` / archive `migrate import` workflow: the existing `migrate` command is a plugin-provider import path for other agent systems, `backup` only exposes create/verify, and machine-to-machine migration docs still describe manual tar/copy/doctor steps. The issue is also paired with open PR #53520, which uses closing syntax for this request, so the tracker should remain open until that PR is reviewed, landed, or closed with maintainer rationale.

Keep this issue as the canonical tracker.
