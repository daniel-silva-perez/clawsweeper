## Summary

Keep open. The report is a concrete npm global downgrade failure, and current main still uses the same staged npm install plus exact-version verification path that can produce the reported `global install verify` failure. Post-release changes are adjacent service-stop/post-core handling, not a package-helper change proving 2026.4.26-to-2026.4.25 downgrades now succeed.

## What This Changes

The issue requests that `openclaw update --tag 2026.4.25` successfully downgrade an npm global install from 2026.4.26 to 2026.4.25 instead of leaving the installed version at 2026.4.26 with `global install verify`.

## Best Possible Solution

Fix the npm global downgrade path so an explicit older version installs into the staged prefix, verifies as that exact version, swaps into the live global root, and leaves post-update doctor/plugin/restart behavior intact. Add focused regression coverage for a 2026.4.26-to-2026.4.25 npm global downgrade while preserving staged install safety and verifier strictness.
