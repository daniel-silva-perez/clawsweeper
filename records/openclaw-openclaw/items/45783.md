## Summary

Keep this PR open. Current main at c0ec58f4b6906ad06fd4c6f265d43be053b2d193 still does not implement the PR's central LLM-boundary privacy pipeline: no native privacy module, no privacy config/schema, no encrypted mapping store, and no bidirectional outbound replacement/inbound restoration wrapper around the main provider calls. Existing behavior is log/transcript redaction plus static plugin text transforms. Related context like ShellWard for Chinese PII is useful, but it does not replace the broader native LLM-boundary contract proposed here.

## Best Possible Solution

Keep this PR open for maintainer architecture and security review, or supersede it with a smaller maintainer-approved design. The best path is a first-class LLM-boundary privacy contract covering config/schema/docs, explicit fail-open versus fail-closed behavior, mapping persistence semantics, custom-rule validation, and coverage for the embedded runner, cache traces, BTW, TTS, payload logging, provider stream/result shapes, and OpenAI-style text blocks.
