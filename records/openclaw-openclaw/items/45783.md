## Summary

Keep this PR open. Current main still does not implement the central feature: native LLM-boundary privacy detection, replacement, encrypted mapping persistence, bidirectional restoration, custom rule loading, or a `privacy` config section. Main only has log/transcript redaction plus static plugin text transforms, and this PR is security-sensitive rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open for maintainer architecture and security review, or supersede it with a smaller maintainer-approved core design. The best path is to define a first-class LLM-boundary privacy contract with config/schema/docs, explicit fail-open versus fail-closed behavior, mapping persistence semantics, custom-rule validation, and coverage for embedded runner, cache traces, BTW, TTS, logs, provider stream/result shapes, and OpenAI-style text blocks.
