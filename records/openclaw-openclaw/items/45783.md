## Summary

Keep this PR open. Current main still does not implement the PR's central feature: a native LLM-boundary privacy detector/replacer with encrypted mapping persistence, bidirectional restore, `privacy` config, custom JSON5 rules, and coverage across normal agent, BTW, TTS, logs, and provider stream/result shapes. The PR is also security-sensitive and still needs maintainer architecture/security review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer architecture and security review, or supersede it with a smaller maintainer-approved core design. The best path is to define a first-class LLM-boundary privacy contract with config/schema/docs, explicit fail-open versus fail-closed behavior, mapping persistence semantics, custom-rule validation, and tests for embedded runner, provider stream/result shapes, cache traces, BTW, TTS, logs, and OpenAI-style text blocks. If maintainers decide this belongs outside core, first add a plugin API that supports dynamic per-session outbound replacement and inbound restoration across provider calls.
