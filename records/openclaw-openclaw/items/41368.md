## Summary

Keep open. Current main still lacks user-facing Feishu Bitable delete tools and this PR covers more than the narrower #40697, but the branch is stale and dirty against current main with blocking rebase regressions before it can merge.

## What This Changes

The PR modifies `extensions/feishu/src/bitable.ts` to add Feishu Bitable single-record deletion, batch-record deletion, and field deletion helpers, schemas, and tool registrations.

## Best Possible Solution

Land a Feishu-owned implementation after rebase that preserves current main's `typebox` and runtime-api imports, placeholder-row cleanup, error formatting, and account routing, then adds the three delete tools with consistent token lookup guidance, irreversible-action wording, and API-confirmed responses.

## Solution Assessment

  - body: The branch still imports `Type` from `@sinclair/typebox`, but current main's Feishu package only depends on `typebox` and this file now imports `OpenClawPluginApi` from the local runtime API.   - confidence: 0.9

  - body: Current main treats rows whose fields contain only default-empty values as placeholders; the PR head falls back to deleting only rows with no fields. Rebase the delete-tool changes without dropping that cleanup fix, or `feishu_bitable_create_app` will again leave Feishu's default rows behind.
  - confidence: 0.88
- **[P3] Restore token lookup hints on delete schemas:** `extensions/feishu/src/bitable.ts:589-613`
  - body: The new delete schemas use bare `app_token` and `table_id` descriptions, unlike the existing Bitable tools that point the model to `feishu_bitable_get_meta` and the URL table parameter. For destructive tools, losing that guidance increases the chance of targeting the wrong table.
  - confidence: 0.83
- **[P3] Return API-confirmed batch delete records:** `extensions/feishu/src/bitable.ts:474-476`
  - body: `batchDeleteRecords` reports only the input count and drops `res.data?.records`. The SDK returns per-record delete results, so callers lose API-confirmed IDs and weaker confirmation than the create/update helpers provide.
  - confidence: 0.78
