## Summary

Keep open. Current main still lacks user-facing Feishu Bitable delete tools, and the related open single-record work does not cover this PR's broader batch-record and field-delete scope. The patch is a plausible Feishu-owned implementation, but destructive tool wording, overlap with #40697, response details, and focused validation still need normal maintainer review before merge.

## What This Changes

This PR adds Feishu Bitable record deletion, batch record deletion, and field deletion helpers, TypeBox schemas, and tool registrations in `extensions/feishu/src/bitable.ts`.

## Best Possible Solution

Merge a Feishu-owned implementation after rebase that exposes the needed delete tools with consistent token lookup hints, explicit destructive-action wording, SDK-confirmed batch response details, and focused tool registration/schema coverage; consolidate with #40697 if maintainers prefer one canonical PR.

## Review Findings

Overall correctness: patch is correct

Overall confidence: 0.82

Full review comments:

- **[P3] Restore the token lookup hints on delete schemas:** `extensions/feishu/src/bitable.ts:588-610`
  - body: The three new delete schemas use bare `"Bitable app token"` and `"Table ID"` descriptions, unlike the existing Bitable schemas that point the model to `feishu_bitable_get_meta` and the URL `?table=` value. These tools delete data, so dropping the established lookup guidance makes it easier for the model to operate on the wrong table.
  - confidence: 0.83
- **[P3] Return the batch delete response records:** `extensions/feishu/src/bitable.ts:473`
  - body: `batchDeleteRecords` reports `count: recordIds.length`, which is only the input size, and drops `res.data?.records`. The SDK returns per-record delete results, so callers lose the API-confirmed IDs and get a less reliable confirmation than the create/update helpers provide.
  - confidence: 0.74
