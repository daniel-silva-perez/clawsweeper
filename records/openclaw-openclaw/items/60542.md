## What This Changes

The issue requests recovery for persisted main-session rows that are terminal but older than their transcript, because that divergence can wedge new input across Telegram and the Control UI.

## Best Possible Solution

Add a focused session-store recovery path that detects a main-session terminal registry row older than its transcript or lifecycle tail and safely rolls forward to a state that accepts new input, with regression coverage that preserves the existing lifecycle-persistence and restart-recovery behavior.
