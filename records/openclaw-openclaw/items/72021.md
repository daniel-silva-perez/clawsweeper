## Summary

Keep open. Current main still has the reported mixed-signal gate, and #74114 is an open, unmerged PR with closing syntax for this issue, so the issue should remain open until maintainers review, land, or reject that PR.

## What This Changes

The issue asks for short-term memory promotion to stop treating daily/session/grounded signals as real user recalls for the `minRecallCount` gate and to revisit scoring so recall-only candidates remain reachable.

## Best Possible Solution

Use #74114 as the active implementation thread: decide whether `minRecallCount` means real user recalls only, then either land a recall-only gate with matching tests/docs/diagnostics and any needed score calibration, or close the PR with a documented product decision that aggregate signals remain threshold evidence.

## Reproduction Assessment

Yes for the mixed-signal gate: current source and existing tests show a `recallCount: 0`, `dailyCount: 3` candidate can clear the gate because `minRecallCount` is checked against aggregate `signalCount`. The exact reporter score-calibration numbers would need their store data or a new fixture to reproduce precisely.

## Solution Assessment

Unclear. The open PR is the narrow active fix for the gate, but it changes currently tested daily/grounded semantics and preserves aggregate score denominators, so maintainers should decide the product semantics and update tests/docs before merging.
