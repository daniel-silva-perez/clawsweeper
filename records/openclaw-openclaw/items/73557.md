## Summary

Keep this PR open. Current main still calls `resolveImplicitProviders` unconditionally before `models.mode` is read, while the documented config semantics say `replace` uses only configured providers. The linked bug #66957 remains open and this PR is a focused implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open and review it as the likely fix for #66957. The best product/code path is a small guard in `src/agents/models-config.plan.ts` that skips implicit provider discovery only when `cfg.models?.mode === "replace"`, plus regression coverage for replace, merge, and default mode behavior.
