## Summary

Keep open for maintainer review. Current main still performs implicit provider discovery before `models.mode` can suppress it, while the documented config contract says replace mode should use only configured providers. The proposed patch is focused and I did not find a blocking correctness or security issue, but merge readiness still needs normal maintainer review and validation.

## What This Changes

The PR guards model provider planning so `models.mode: "replace"` skips implicit provider discovery and adds a unit test covering replace, merge, and default behavior.

## Best Possible Solution

Land a narrow guard before implicit discovery when replace mode is active, preserve merge/default behavior and existing normalization/secret enforcement after explicit providers are selected, and handle pricing-bootstrap skipping in the separate startup follow-up if maintainers agree.
