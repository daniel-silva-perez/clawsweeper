## Summary

Keep this PR open. Current main still calls implicit provider discovery before any `models.mode` handling, while the config help says `replace` should use only configured providers; the PR is a focused implementation with matching regression coverage, so the remaining action is maintainer review and validation rather than cleanup closure.

## What This Changes

The PR changes agent model-provider planning to skip implicit provider discovery when `models.mode` is `replace` and adds a unit test file covering replace, merge, and default behavior.

## Best Possible Solution

Land a narrow guard before implicit provider discovery when `models.mode` is `replace`, keep merge/default behavior unchanged, preserve provider normalization and source-managed secret enforcement for explicit providers, and track pricing-bootstrap behavior separately in #74020.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: configure `models.mode` as `replace`, call the planning path with an injected `resolveImplicitProviders`, and it is invoked before mode handling; #66957 also includes concrete config context and startup-delay logs.

## Solution Assessment

Yes. The PR's guard at the resolver callsite is the narrowest maintainable fix for this bug because it prevents the expensive work before it starts while leaving merge/default discovery and provider merging semantics intact.
