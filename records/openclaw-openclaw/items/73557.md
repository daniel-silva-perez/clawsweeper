## Summary

Keep this PR open. Current main still calls implicit provider discovery before it reads `models.mode`, while the documented config semantics say `replace` uses only configured providers. The linked bug #66957 remains open, and this PR is a focused implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open and review it as the likely fix for #66957. The best code path is a small guard in `src/agents/models-config.plan.ts` that skips `resolveImplicitProviders` only when `cfg.models?.mode === "replace"`, with regression coverage for replace, merge, and default mode behavior.
