## Summary

Keep open. Current main still runs implicit provider discovery before it reads `models.mode`, while the documented `replace` contract says only configured providers should be used. This PR is still a focused, relevant implementation candidate for #66957; I did not find a concrete code-review bug in the patch, but the latest head CI needs maintainer follow-up before merge.

## What This Changes

The PR changes `src/agents/models-config.plan.ts` to skip `resolveImplicitProviders` when `cfg.models.mode` is `replace` and adds a colocated unit test covering replace, merge, and default mode behavior.

## Best Possible Solution

Land a narrow fix that branches before implicit discovery when `models.mode` is `replace`, keeps merge/default behavior unchanged, and preserves provider normalization plus source-managed secret enforcement after explicit providers are selected.
