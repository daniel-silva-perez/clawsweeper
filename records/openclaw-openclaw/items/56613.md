## Summary

Keep open. Current main implements the per-agent TTS half through `agents.list[].tts`, but the central Talk/Voice tab session-switching request is still not implemented on Android: the Voice tab has no session picker and both manual mic and Talk Mode still route through the resolved main session key. The issue discussion also includes a maintainer comment explicitly keeping this open for the remaining Talk/Voice UI session-selection work.

## Best Possible Solution

Keep this issue open for the remaining Talk/Voice session-selection work. A good fix would add selected Voice/Talk agent or session state, expose a picker or reuse the existing Chat selector pattern, route Android `MicCaptureManager` and `TalkModeManager` through that selected key instead of only `resolveMainSessionKey()`, consider parity for iOS/macOS Talk surfaces, add focused routing/UI tests, and keep `agents.list[].tts` as the canonical per-agent voice configuration instead of adding `talk.voiceId`.
