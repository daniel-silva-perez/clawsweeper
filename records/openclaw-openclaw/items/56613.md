## Summary

Keep open. Current main implements the per-agent TTS half through `agents.list[].tts`, and iOS now has a settings-level agent picker that feeds Chat/Talk, but the Android Voice tab still has no agent/session selector and its manual mic and Talk Mode routing still use the resolved main session key. The maintainer discussion explicitly preserved this issue for the remaining Talk/Voice session-selection work.

## Best Possible Solution

Keep this issue open for the remaining Talk/Voice session-selection work. A good fix would add Voice/Talk selected agent or session state on Android, expose a picker or reuse the Chat selector pattern, route both manual mic and Talk Mode through that selected key instead of only `resolveMainSessionKey()`, add focused routing/UI tests, and keep `agents.list[].tts` as the canonical per-agent voice configuration. For iOS/macOS, maintainers should decide whether the existing settings-level iOS picker and macOS active-WebChat-session routing are sufficient or whether the Voice/Talk surfaces need their own explicit picker.
