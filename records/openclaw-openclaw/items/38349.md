## Summary

Keep open. Current main still only supports mention-based or always-on group activation, and the requested contextual peeking/engaged LLM decision mode, contextualActivation config, decision logs, and decision-log CLI are not present. The linked implementation PR #38350 is closed unmerged and the older #1721 is closed, so there is no shipped or active canonical implementation to close against.

## Best Possible Solution

Keep this open as the canonical product/API discussion. The best path is either a first-class contextual activation feature spanning config schemas, session/protocol state, Telegram and WhatsApp pre-dispatch gating, observability/CLI/docs/tests, or a typed plugin/channel activation seam that lets an external ClawHub plugin make a peeking/engaged decision before bundled runtimes drop unmentioned group messages.
