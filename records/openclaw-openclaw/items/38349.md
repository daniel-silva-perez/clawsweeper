## Summary

Keep open. Current main still does not implement contextual peeking/engaged group activation: the shipped activation contract is still mention/always only, Telegram and WhatsApp group gates still skip unmentioned messages before any LLM decision path, config/protocol schemas have no contextualActivation surface, and the linked implementation PR is closed unmerged.

## Best Possible Solution

Keep this open as the canonical product/API discussion. The best path is either a first-class contextual activation feature spanning config schemas, session/protocol state, Telegram and WhatsApp pre-dispatch gating, observability/CLI/docs/tests, or a typed pre-gating plugin/channel activation seam that lets an external plugin make the decision before bundled runtimes drop unmentioned group messages.
