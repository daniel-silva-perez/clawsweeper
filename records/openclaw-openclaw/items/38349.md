## Summary

Keep open. Current main still exposes group activation as `mention` or `always` plus `requireMention` gating; it does not implement the requested contextual peeking/engaged decision model, `contextualActivation` config, decision-log storage, or `openclaw gateway decisions`/decision-log viewer. The linked implementation PR #38350 is closed unmerged, and #1721 is closed rather than an active canonical replacement.

## Best Possible Solution

Keep this issue open as the canonical product/API discussion. The maintainer path is either a first-class contextual activation feature across config schemas, session/protocol state, Telegram and WhatsApp pre-dispatch gating, observability/CLI/docs/tests, or a typed plugin/channel activation seam that lets external plugins make a peeking/engaged decision before bundled runtimes drop unmentioned group messages.
