## Summary

Keep open. Current main still does not implement contextual peeking/engaged group activation: the shipped contract remains mention/always, Telegram and WhatsApp still drop unmentioned group messages before any LLM decision path, config/protocol schemas have no contextualActivation surface, and the linked implementation PR is closed unmerged.

## Best Possible Solution

Keep this open as the canonical product/API discussion. The best path is either a first-class opt-in contextual activation feature across config schemas, Gateway session/protocol state, Telegram and WhatsApp pre-dispatch gating, observability/CLI/docs/tests, and privacy-safe defaults, or a typed pre-gating plugin/channel activation seam that lets an external plugin decide before bundled runtimes drop unmentioned group messages.
