## Summary

Keep this PR open. Current main still classifies dotted read-only gateway actions such as `config.schema.lookup`, `config.get`, and `agents.files.list` as mutating because `isMutatingToolCall` only checks the full action string against flat read-only verbs. The PR remains a plausible narrow fix for the linked warning-noise bug, and the related payload-specific PR #42962 does not supersede the classifier gap.

## Best Possible Solution

Keep this PR open for maintainer review, or land an equivalent fix in `src/agents/tool-mutation.ts` that classifies known read-only dotted gateway actions as non-mutating while keeping unsupported cron/canvas actions fail-closed. Add the regression tests from this PR, then close #42856 once the classifier behavior ships.
