## Summary

Keep this PR open. Current main still has the classifier bug the PR targets: dotted read-only gateway actions cannot match the flat read-only action set, while gateway scope/docs/tool surfaces still treat `config.schema.lookup` and related methods as read-only. The related #42962 is a narrower warning-suppression attempt and does not replace this classifier fix.

## Best Possible Solution

Keep this PR open for maintainer review, or land an equivalent fix in `src/agents/tool-mutation.ts` that classifies known read-only dotted gateway actions as non-mutating while keeping unsupported cron/canvas actions fail-closed. Add the regression tests from this PR, coordinate with #42962 if needed, then close #42856 after the classifier behavior ships.
