## Summary

Keep this PR open. Current main still classifies dotted read-only gateway actions such as `config.schema.lookup`, `config.get`, and `agents.files.list` as mutating, while the gateway scope table treats those methods as read-only and the user-visible warning path still depends on that mutation bit. The PR remains a plausible focused fix for #42856; the related #42962 payload suppression is narrower and does not replace the classifier fix.

## Best Possible Solution

Keep this PR open for maintainer review, or land an equivalent fix in `src/agents/tool-mutation.ts` that classifies known read-only dotted gateway actions as non-mutating while keeping unsupported cron/canvas actions fail-closed. Add the regression tests from this PR, then close #42856 once the classifier behavior ships.
