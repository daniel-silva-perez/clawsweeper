## Summary

Keep this PR open. Current main still classifies dotted read-only gateway actions such as `config.schema.lookup` as mutating, while the gateway scope model, agent tool surface, docs, and tests still treat that action as a read-only inspection path. The related #42962 is a narrower warning-suppression PR and does not supersede this classifier fix.

## Best Possible Solution

Keep this PR open for maintainer review, or land an equivalent fix in `src/agents/tool-mutation.ts` that classifies known read-only dotted gateway actions as non-mutating while keeping unsupported cron/canvas actions fail-closed. Add the regression tests from this PR, coordinate with #42962 if warning suppression still needs a narrower follow-up, then close #42856 after the classifier behavior ships.
