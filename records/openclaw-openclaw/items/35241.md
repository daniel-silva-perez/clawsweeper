## Summary

Keep this PR open. Current main still does not expose `openclaw agent --session-key`, while lower-level session resolution, embedded agent execution, Gateway schema, and Gateway server handling already support `sessionKey`. The PR is still relevant, but should be refreshed with consistent validation, docs, and focused CLI tests before merge.

## Best Possible Solution

Keep this PR open. The best path is a small refreshed patch that registers `openclaw agent --session-key <key>`, adds `sessionKey` to `AgentCliOpts`, treats it as a valid selector, passes it through Gateway and embedded fallback paths, mirrors Gateway validation for malformed and mismatched agent session keys, and updates CLI docs/help plus focused tests. Handle the related `--no-fallback` request as a separate follow-up.
