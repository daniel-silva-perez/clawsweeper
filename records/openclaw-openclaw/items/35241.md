## Summary

Keep this PR open. Current main still lacks `openclaw agent --session-key`, while the embedded agent command, session resolver, and Gateway protocol/server already support `sessionKey`; the PR addresses a real CLI surface gap and has downstream demand in the discussion. It should be refreshed and tightened before merge rather than closed as obsolete.

## Best Possible Solution

Keep the PR open. The best path is a small refreshed CLI change that registers `openclaw agent --session-key <key>`, adds `sessionKey` to `AgentCliOpts`, treats it as a valid selector, passes it through gateway and embedded fallback paths, mirrors malformed-key and `--agent` conflict validation consistently, and updates CLI help/docs plus focused tests. Handle `--no-fallback` separately.
