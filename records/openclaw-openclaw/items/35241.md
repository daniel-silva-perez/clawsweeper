## Summary

Keep this PR open. Current main still does not expose `openclaw agent --session-key`, while the lower-level embedded command, session resolver, Gateway protocol, and Gateway server already understand `sessionKey`. The patch is still relevant, has concrete downstream demand in the discussion, and should be refreshed with validation/docs/tests rather than closed as obsolete.

## Best Possible Solution

Keep this PR open. The best path is a small refreshed patch that registers `openclaw agent --session-key <key>`, adds `sessionKey` to `AgentCliOpts`, treats it as a valid selector, passes it through gateway and embedded fallback paths, mirrors malformed-key and `--agent` conflict validation consistently, and updates CLI docs/help plus focused tests. Handle `--no-fallback` as a separate follow-up.
