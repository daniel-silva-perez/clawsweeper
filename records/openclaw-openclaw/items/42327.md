## Summary

Keep this PR open. Current main still registers `openclaw cron runs` as `--id`-only, the public cron docs still document only the flag form, and current cron CLI tests do not cover the positional alias this PR adds. The PR remains a focused implementation candidate for the linked CLI UX bug, with small review cleanup and docs follow-through still needed.

## Best Possible Solution
