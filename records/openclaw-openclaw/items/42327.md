## Summary

Keep this PR open. Current main still registers `openclaw cron runs` as a `--id`-only command, the docs still show only `openclaw cron runs --id <job-id>`, and there is no current CLI regression coverage for the proposed positional alias. The PR remains a focused implementation candidate for the linked CLI UX bug in #42300.

## Best Possible Solution

Keep this PR open as the focused CLI fix candidate for #42300.
