## Summary

Keep open. Current main still implements `openclaw cron runs` as a flag-only command requiring `--id`, while this PR is the focused implementation candidate for the positional job-id UX reported in #42300. The diff is narrow, but it still needs review cleanup, no-id regression coverage, docs updates, and normal validation before merge.

## Best Possible Solution

Keep this PR open as the focused fix candidate. Rebase it on current main, remove the dead returns after `command.error()`, add a no-id regression test, update the cron docs to show both positional and `--id` forms, then run the targeted cron CLI test and the appropriate changed gate before merge.
