## Summary

Keep this PR open. Current main still requires `openclaw cron runs --id <id>` and does not accept a positional job id, while this PR remains a focused implementation candidate for the linked cron CLI UX bug #42300.

## Best Possible Solution

Keep this PR open as the focused CLI fix candidate. Rebase it on current main, address the two existing review comments, update the public cron docs to document both positional and `--id` forms, then run the targeted cron CLI test and the appropriate changed gate before merge.
