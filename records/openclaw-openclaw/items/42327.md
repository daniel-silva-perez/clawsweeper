## Summary

Keep this PR open. Current main still registers `openclaw cron runs` as a flag-only command requiring `--id`, while this PR is a focused implementation candidate for the linked cron CLI UX bug. The remaining work is review cleanup, missing no-id coverage, and docs updates rather than closure.

## Best Possible Solution

Keep this PR open as the focused fix candidate. Rebase it on current main, remove the dead returns after `command.error()`, add the missing `cron runs` no-id test, update the public cron docs for both positional and `--id` forms, then run the targeted cron CLI test plus the appropriate changed gate before merge.
