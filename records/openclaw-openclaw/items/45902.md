## Summary

Keep this PR open. Current main has generic parent-session model fallback when a parent already has an override, but it still does not implement the PR's central Telegram behavior: setting `/model` inside a Telegram topic/thread does not persist a same-chat parent future-thread default, and new Telegram threads are not seeded from such a default. The PR also remains broad and has unresolved functional and security-sensitive review concerns.

## Best Possible Solution

Keep this PR open, but ask for a scoped revision or replacement focused on Telegram model inheritance. The implementation should persist a same-chat parent future-thread model default from directive-only, inline, and Telegram picker `/model` paths; seed only genuinely new Telegram forum-topic and DM-topic sessions from that default; avoid cross-chat leakage for main-scoped DM keys; handle `direct_messages_topic` payloads if needed; and cover concurrency, parent-key, and pre-created metadata edge cases with focused tests. Split memory, gateway, recovery-script, and live E2E tooling into separate maintainer-reviewed PRs.
