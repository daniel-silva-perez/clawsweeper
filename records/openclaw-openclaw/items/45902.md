## Summary

Keep this PR open. Current main still does not implement the central Telegram behavior: there is no parent-chat future-thread model default in the session schema, the `/model` persistence paths only update the active session, and new Telegram thread/topic sessions are not seeded from any parent default. The PR also remains broad and has unresolved functional and security-sensitive review concerns.

## Best Possible Solution

Keep this PR open, but ask for a scoped revision or replacement focused on Telegram model inheritance. The runtime change should persist a same-chat parent future-thread model default from all `/model` paths, seed only genuinely new Telegram forum-topic and DM-topic sessions from that default, prevent cross-chat leakage for main-scoped DM keys, cover `direct_messages_topic` payloads if needed, and add focused tests for concurrency, parent-key, and metadata pre-create edge cases. Split memory, gateway, recovery-script, and live E2E tooling into separate maintainer-reviewed PRs.
