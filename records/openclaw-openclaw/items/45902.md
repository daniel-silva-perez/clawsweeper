## Summary

Keep this PR open. Current main still does not implement the central Telegram behavior: setting `/model` inside a Telegram topic/thread does not store a same-chat future-thread default, and new Telegram threads are not seeded from such a default. The PR contains relevant work, but the discussion and diff surface still show unresolved correctness concerns plus security-sensitive, unrelated script/gateway/memory changes that need a scoped maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open but ask for a scoped revision or replacement focused on Telegram model inheritance. The implementation should persist a same-chat parent future-thread default from both directive-only and inline Telegram `/model` paths, seed only genuinely new Telegram topic/DM-thread sessions from that default, prevent cross-chat leakage for main-scoped DM keys, handle Telegram `direct_messages_topic` payloads if needed for the feature, and cover the edge cases with focused tests. Split memory, gateway, recovery-script, and optional live E2E tooling into separate maintainer-reviewed PRs.
