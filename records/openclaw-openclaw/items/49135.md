## Summary

Keep this PR open. Current main still lacks Telegram's `streaming.blockStreamingCoalesceDefaults`, while the shared block-streaming resolver still reads provider defaults from `plugin.streaming.blockStreamingCoalesceDefaults`. The PR remains a focused implementation candidate for the Telegram newline-splitting regression, and its second commit addressed the review concern about adding `"streaming"` to the `Pick` return type. The single-file diff is security-neutral.

## Best Possible Solution

Keep this PR open, rebase it onto current main, preserve the `streaming` return type addition, add `blockStreamingCoalesceDefaults: { minChars: 1500, idleMs: 1000 }` through `createTelegramPluginBase`, and add a targeted regression test before landing. The linked regression issue being closed is not enough to close this PR because current main still lacks the implementation.
