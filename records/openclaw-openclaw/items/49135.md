## Summary

Keep this PR open. Current main still enables Telegram block streaming but does not define `streaming.blockStreamingCoalesceDefaults` for the Telegram plugin, while the shared block-streaming resolver still reads plugin-level defaults before falling back to the global 800-character minimum. The PR remains a focused implementation candidate for the linked Telegram newline/message-splitting regression, and the later PR commit addresses the earlier review concern by adding `"streaming"` to the returned `Pick` type.

## Best Possible Solution

Keep this PR open, rebase it onto current main, preserve the `"streaming"` return type addition, add `blockStreamingCoalesceDefaults: { minChars: 1500, idleMs: 1000 }` through `createTelegramPluginBase`, and add a targeted regression assertion before maintainer review or landing.
