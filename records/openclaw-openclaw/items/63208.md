Keep open. Current main still only reads direct Markdown children of each memory-wiki page-group directory, so nested source pages are skipped by compile, local wiki query/get fallback, and status counts. ## What This Changes

The issue requests recursive memory-wiki Markdown discovery so page-group subdirectories can be compiled and searched without flattening existing vault structures.

## Best Possible Solution

Add an extension-owned recursive Markdown page collector for memory-wiki page groups, use it consistently in compile, local query/get fallback, and status, document nested page-group support, and cover nested page paths with focused regression tests.
