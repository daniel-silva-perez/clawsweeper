## Summary

Keep open. Current main still has flat-only memory-wiki Markdown discovery in compile, local search/get, and status, so nested pages under directories such as sources/OpenClaw配置文件指南/AGENTS.md are still skipped.

## What This Changes

The issue requests recursive memory-wiki handling for Markdown pages in nested wiki page-group directories while preserving the user's directory organization and page identity.

## Best Possible Solution

Keep this open and implement a shared deterministic recursive Markdown page collector inside the memory-wiki plugin, then use it consistently for compile, local query/get, and status counting. Add nested-source regression tests, document recursive page-group support in the memory-wiki and wiki CLI docs, and add a changelog fix entry crediting @arlen8411.
