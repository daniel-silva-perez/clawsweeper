## Summary

Keep open: current main still suppresses Discord ack/status reactions for guild/channel source turns whenever the source reply mode is `message_tool_only`. The reported `ackReactionScope: "all"` shape is covered by a current test that expects no Discord reaction call, and the channel-resolution failure path does not by itself remove ID-keyed configured guild/channel entries. The remaining question is product intent: whether configured ack/status reactions should be exempt from the v2026.4.27 tool-only visible-output privacy default.

## What This Changes

The issue reports that Discord automatic ack reactions no longer fire in guild channels on v2026.4.27 when `ackReaction` is configured for all messages and channel resolution is failing.

## Best Possible Solution

Define the Discord/group-channel contract explicitly. If configured ack reactions are allowed in tool-only rooms, exempt the Discord ack/status reaction gate narrowly and update the matching tests/docs; if not, document `messages.groupChat.visibleReplies: "automatic"` as the supported visible-reaction workaround and add clearer suppression logging.

## Reproduction Assessment

Yes. A high-confidence source/test reproduction exists: a guild/channel route with `ackReactionScope: "all"` and `ackReaction: "👀"` currently resolves to `message_tool_only` and asserts that Discord reaction calls are not made.

## Solution Assessment

Unclear. Exempting `ackReactionScope: "all"` from tool-only suppression would solve this report, but it may not be the best fix unless maintainers confirm that ack/status reactions should bypass the shipped privacy contract.
