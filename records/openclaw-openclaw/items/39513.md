## Summary

Keep this PR open. Current main still has the cron-event behavior the PR documents: main-session/tagged cron events bypass HEARTBEAT.md file-gate parsing and use the specialized cron prompt, while the current cron docs and AGENTS.md template still do not explain that HEARTBEAT.md is not automatically read for those cron events or show the workaround. The PR discussion supports the docs change as accurate, with style/template wording feedback remaining.

## Best Possible Solution

Keep this PR open or replace it with an equivalent docs-only patch. The maintainer path is to document that main-session cron system events use a self-contained cron-event prompt and do not include the default HEARTBEAT.md instruction, add a clear workaround in docs/automation/cron-jobs.md, and make docs/reference/templates/AGENTS.md contain direct agent-facing cron handling guidance. Any runtime behavior change should be tracked separately with prompt-selection and HEARTBEAT.md file-gate tests.
