## Summary

Keep open. Current `main` still has only the 200-message Control UI history cap, so this PR is not obsolete; the branch also incorporates the earlier review feedback about tool-result `content` and hidden tool messages. However, the repaired diff adds an unbounded recursive raw-content walk in the UI history-trimming path, so it still needs maintainer/author follow-up before merge.

## What This Changes

This PR adds a Control UI chat-history character budget alongside the existing message-count cap, updates the history notice/counting behavior, adds focused `buildChatItems` tests, and records the fix in the changelog.

## Best Possible Solution

Land a bounded UI-local history windowing fix that counts the payloads the Control UI can render, consistently excludes hidden tool messages from both budget and notice counts, short-circuits estimation once the budget is exceeded, and keeps the gateway `chat.history` contract unchanged.

## Reproduction Assessment

Yes. The original issue has a credible reproduction path using long Control UI sessions with large tool-result-heavy histories, and the PR adds deterministic `buildChatItems` unit scenarios for count caps, character caps, tool-result content, and hidden tool messages.

## Solution Assessment

Not yet. The overall direction is the right narrow layer for immediate relief, but the raw-content estimator should be made depth/node/budget bounded before this is the safest maintainable fix.

## Review Findings

- **[P2] Bound the raw content estimator:** `ui/src/ui/chat/build-chat-items.ts:159-171`
