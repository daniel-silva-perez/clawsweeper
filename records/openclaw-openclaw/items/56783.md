## Summary

Keep open. The checked-out main still renders Control UI chat history with only the 200-message count cap, while this open PR remains the active implementation candidate for adding an aggregate character-budget window. The PR also has recent maintainer branch repair and unresolved availability/security review findings, so the best maintainer action is to review and harden the PR rather than close it as obsolete.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate. The best path is to rebase or refresh it as needed, resolve or explicitly accept the availability findings, and land a bounded, cycle-safe Control UI history-budget helper with tests for the count cap, aggregate character budget, tool_result content including nested content, hidden tool results, malformed entries, and accurate visible/hidden notices.
