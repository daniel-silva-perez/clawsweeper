## Summary

Keep this PR open. Current main still does not contain xCloud hosting docs, but the proposed official guide still has concrete docs correctness issues and under-specifies the hosted Control UI security/auth path, so it needs maintainer or provider validation before merge.

## What This Changes

Adds an xCloud hosting guide under `docs/install`, registers xCloud navigation and redirects in `docs/docs.json`, and adds an xCloud card to the VPS provider picker.

## Best Possible Solution

Publish an xCloud hosting guide only after a maintainer or xCloud/provider owner verifies the managed deployment against current OpenClaw Gateway and Control UI security contracts, then revise the page to document the exact auth, origin, device-pairing, remote-access, Telegram allowlist, and provider-credential behavior.
