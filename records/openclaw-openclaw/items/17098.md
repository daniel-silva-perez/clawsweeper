## Summary

Keep this PR open. Current main still lacks xCloud hosting docs, navigation, and VPS-picker coverage, so it is not obsolete or implemented. The proposed docs-only addition is real, but it still needs maintainer/provider validation and revision because the guide contains inaccurate provider wording and under-documents the security contract for a hosted Control UI flow.

## What This Changes

Adds an xCloud hosting guide under docs/install, registers xCloud docs navigation and redirects, and adds an xCloud card to the VPS provider page.

## Best Possible Solution

Publish an xCloud hosting guide only after a maintainer or xCloud/provider owner verifies the managed deployment against current OpenClaw Gateway and Control UI security contracts, then revise the guide to document the exact auth/origin/device-pairing behavior and accurate model-provider terminology.
