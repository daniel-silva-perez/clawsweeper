## Summary

Keep this PR open. Current main still does not include xCloud hosting docs, navigation, or VPS picker coverage, but the proposed docs are not merge-ready because they still contain inaccurate provider wording and do not document the hosted Control UI auth/origin/device-pairing path required by current Gateway behavior.

## What This Changes

Adds an xCloud hosting guide under docs/install, registers xCloud docs navigation and redirects, and adds an xCloud card to the VPS provider page.

## Best Possible Solution

Publish an xCloud hosting guide only after a maintainer or provider owner verifies the managed deployment against current Gateway and Control UI security contracts, then revise the guide to document the exact auth/origin/device-pairing behavior and accurate model-provider terminology.
