## Summary

Keep this PR open. Current main still has no xCloud hosting page, navigation entry, redirect, or VPS picker card, so the contribution is not implemented on main; however the PR head still needs revision and maintainer/provider validation before merge because it misstates the provider model and under-specifies a hosted Control UI auth path.

## What This Changes

Adds an xCloud hosting guide under `docs/install`, registers xCloud docs navigation and redirects in `docs/docs.json`, and adds an xCloud card to the VPS provider picker.

## Best Possible Solution

Publish the xCloud guide only after a maintainer or xCloud/provider owner verifies the managed deployment against current OpenClaw Gateway and Control UI security contracts, then revise the page to document the exact auth, origin, device-pairing, remote-access, Telegram allowlist, and provider-credential behavior.
