Keep this PR open. Current main still has no xCloud hosting guide, VPS card, navigation entry, or redirects, so the contribution is not obsolete; however the PR is not ready as-is because the guide still has incorrect provider wording and under-specifies the hosted Control UI auth/security path for an official third-party deployment guide.

## What This Changes

## Best Possible Solution

Publish an xCloud guide only after a maintainer or xCloud/provider owner verifies the managed deployment against current OpenClaw Gateway and Control UI security contracts, then revise the page to document the exact auth, origin, device-pairing, remote-access, Telegram allowlist, and provider-credential behavior.

Not applicable as a docs feature PR rather than a runtime bug. The high-confidence verification path is source/docs inspection: current main lacks xCloud coverage, while the PR head adds the page and exposes reviewable wording/security gaps.

## Solution Assessment
