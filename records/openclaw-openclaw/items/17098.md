## Summary

Keep open. Current main still does not contain the proposed xCloud hosting docs, VPS card, navigation entry, or redirects, so the PR is not obsolete. The PR also should not be merged as-is because the proposed guide still has inaccurate model-provider wording and an under-specified hosted Control UI auth story for a third-party managed deployment.

## What This Changes

Adds an xCloud hosting guide under `docs/install`, registers xCloud docs navigation and redirects, and adds an xCloud card to the VPS provider page.

## Best Possible Solution

Publish an xCloud hosting guide only after a maintainer or provider owner verifies the managed deployment against current OpenClaw Gateway and Control UI security contracts, then revise the guide to document the exact auth/origin/device-pairing behavior and accurate model-provider terminology.
