## Summary

Keep this PR open. Current main still lacks xCloud install docs, a VPS picker entry, docs navigation, or redirects, so the contribution is not implemented or obsolete. The PR is also not ready for cleanup-close because the proposed guide still contains a concrete provider wording error and needs maintainer/provider validation for the hosted Control UI, Gateway auth/origin, device-pairing, and Telegram allowlist behavior before it becomes official OpenClaw deployment guidance.

## What This Changes

Adds a new `docs/install/xcloud.mdx` hosting guide, an xCloud card on the VPS page, and docs navigation plus redirects for `/xcloud` and `/platforms/xcloud`.

## Best Possible Solution

Publish an xCloud hosting guide only after the provider flow is verified against current OpenClaw Gateway and Control UI security contracts, with the guide documenting the exact hosted auth/origin/device-pairing behavior and using accurate provider terminology; otherwise keep the service-specific instructions outside core docs until they are accurate.
