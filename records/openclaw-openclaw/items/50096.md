## Summary

Keep this issue open. Current main has a much stronger memory system than the original report describes, including durable Markdown memory, memory_search/memory_get, builtin/QMD backends, opt-in session transcript indexing, Active Memory, Dreaming promotion, memory-wiki, and plugin/context-engine seams. The umbrella still contains actionable first-party gaps: memoryFlush is still a reply/compaction-time path rather than a gateway shutdown/session-stop reliability contract, and MEMORY.md injection controls are broad bootstrap controls rather than a MEMORY.md-specific full/core-only/recall-only policy.

## Best Possible Solution

Keep this open unless maintainers replace it with explicit canonical follow-ups: one for graceful gateway-shutdown/session-stop memory flush semantics, one for MEMORY.md-specific full/core-only/recall-only injection configuration or documentation, and one product/docs note that recommends the shipped memory-core, Active Memory, Dreaming, QMD, memory-wiki, and plugin/ClawHub paths for larger memory architectures while acknowledging external UAML/Engram-style engines.
