## Summary

Keep this issue open. Current main now covers much of the long-term memory surface through memory-core search, opt-in session transcript indexing, Active Memory, scheduled Dreaming promotion, memory-wiki, QMD, and plugin-owned memory seams. However, the issue also asks for core behavior that is still not fully satisfied: memoryFlush is implemented/documented as a pre-compaction/turn-time path rather than a graceful gateway-shutdown flush, and current context injection controls are broad bootstrap controls rather than a MEMORY.md-specific full/recall-only mode.

## Best Possible Solution

Keep this issue open or replace it with explicit canonical follow-ups: one for graceful shutdown/session-stop memory persistence semantics, one for MEMORY.md-specific full/recall-only injection configuration or docs, and one product/docs note that points operators to the shipped memory-core + Dreaming + Active Memory + QMD + memory-wiki path while keeping UAML/Engram-style external memory engines in the plugin/ClawHub lane.
