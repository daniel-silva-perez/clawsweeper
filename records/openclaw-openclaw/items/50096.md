## Summary

Keep this issue open. Current main has a substantial memory stack, including markdown memory files, memory_search/memory_get, builtin/QMD memory indexing, opt-in session transcript search, Active Memory, Dreaming promotion, memory-wiki, and plugin memory seams. The central umbrella still includes unresolved first-party behavior: memoryFlush remains a pre-compaction/turn-time path rather than a graceful gateway-shutdown flush, and MEMORY.md injection controls are broad bootstrap controls rather than a MEMORY.md-specific full/core-only/recall-only policy.

## Best Possible Solution

Keep this open or replace it with explicit canonical follow-ups: one for graceful gateway-shutdown/session-stop memory flush semantics, one for MEMORY.md-specific full/core-only/recall-only injection configuration or documentation, and one docs/product note that recommends the shipped memory-core, Active Memory, Dreaming, QMD, memory-wiki, and plugin/ClawHub paths for larger memory architectures while acknowledging UAML/Engram-style external engines.
