## Summary

Keep open. Current main already has the general token-saving skill behavior from related #6691: a bounded `<available_skills>` metadata catalog and one selected `SKILL.md` read on demand. This issue asks for a more specific skill dependency graph: dependency parsing, depth-limited dynamic expansion, `openclaw skills graph --dot`, and a `skills.maxDepth`-style config knob. I did not find those graph/dependency surfaces in current source, docs, CLI commands, config types, or tests.

## Best Possible Solution

Keep this issue open for maintainer product/design review. The best direction is likely an explicit skill dependency contract, then a graph builder that respects existing skill eligibility/allowlists and can power `openclaw skills graph --dot`. Runtime dependency expansion should be designed around the current bounded `<available_skills>` catalog rather than freeform parsing alone, and the config name should avoid confusion with `agents.defaults.subagents.maxSpawnDepth`.
