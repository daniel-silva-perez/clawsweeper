## Summary

Keep open. Current main has progressive skill disclosure, a compact/truncated flat skills catalog, and skill prompt-size caps, but it still does not implement the requested skill dependency graph, dependency-depth expansion, `openclaw skills graph --dot`, or `skills.maxDepth`-style configuration. Related open issues #52861 and #73248 overlap on token overhead but do not subsume this dependency-graph proposal.

## What This Changes

Request to add dependency-aware skill loading, graph visualization, and depth configuration so OpenClaw can load only the needed skill dependency chain.

## Best Possible Solution

Define a backward-compatible skill dependency metadata contract, then add a graph builder that respects existing skill eligibility, allowlists, visibility, prompt budgets, and security boundaries; only after that should CLI DOT output, depth configuration, tests, and docs be added.

Not applicable: this is a feature request rather than a bug. The high-confidence check is source and docs inspection on current main, which shows the requested graph, CLI, and config surfaces are absent.
