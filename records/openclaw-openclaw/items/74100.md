## Summary

Keep open. Current main has the related token-saving behavior from #6691: a bounded `<available_skills>` metadata catalog and a single selected `SKILL.md` read on demand. It does not implement the requested dependency graph, dependency-depth expansion, `openclaw skills graph --dot`, or a `skills.maxDepth`-style config surface, so closing would conflate a partial mitigation with the feature requested here.

## What This Changes

Request a Skill Graph feature that parses inter-skill dependencies, expands dependency layers on demand, exposes DOT graph output, and adds a depth config knob to reduce skill-token overhead.

## Best Possible Solution

Keep this open for maintainer design. The durable path is an explicit backward-compatible skill dependency contract, a graph builder that respects existing skill eligibility and allowlists, optional DOT output for inspection, and runtime expansion semantics that work with the current bounded `<available_skills>` catalog without confusing skill dependency depth with `agents.defaults.subagents.maxSpawnDepth`.
