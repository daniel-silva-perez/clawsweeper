## Summary

Keep open. Current main has progressive skill disclosure, prompt-size caps, and sub-agent spawn-depth controls, but it does not implement the requested skill dependency graph, depth-based skill expansion, DOT graph output, or `skills.maxDepth`-style config; related open issues #52861 and #73248 overlap on token overhead but do not fully cover this dependency-graph request.

## What This Changes

The issue requests a dependency-aware Skill Graph that lazily expands skill dependency layers, emits DOT graph output, and adds configurable skill expansion depth.

## Best Possible Solution

Define a backward-compatible skill dependency contract first, then add a graph builder that respects existing eligibility, allowlists, prompt budgets, and skill exposure rules, with optional DOT output and docs that distinguish skill dependency depth from sub-agent spawn depth.
