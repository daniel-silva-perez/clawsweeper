## Summary

Keep open. Current main has progressive skill disclosure and prompt-size caps, but it does not implement the requested skill dependency graph, depth-based expansion, `openclaw skills graph --dot`, or `skills.maxDepth`-style configuration; related open issues #52861 and #73248 overlap on token overhead but do not subsume the dependency-graph request.

## What This Changes

The issue requests a skill dependency graph that parses inter-skill references, lazily expands dependency layers at runtime, exposes DOT graph output, and adds configurable dependency depth to reduce token use.

## Best Possible Solution

Define a backward-compatible skill dependency contract first, then add a graph builder that respects existing eligibility, allowlists, prompt budgets, and skill exposure rules, with optional DOT output and docs for how graph depth differs from sub-agent spawn depth.
