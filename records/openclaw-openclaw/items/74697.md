## Summary

Keep open for normal PR review. Current main still has the SDK wait-status gap: Gateway wait-only deadlines and terminal runtime timeouts can both use `status: "timeout"`, while the SDK currently only returns `timed_out` when `stopReason` or `aborted` is present. The proposed patch is narrow and consistent with the Gateway distinction; I found no blocking correctness or security issue in the diff.

## What This Changes

This PR updates SDK wait-status normalization, adds a focused SDK regression test, and adds a changelog fix entry.

## Best Possible Solution

Land this PR or an equivalent narrow fix after validation passes, preserving the documented behavior that bare wait deadlines remain `accepted` while actual runtime timeouts are reported as `timed_out`.
