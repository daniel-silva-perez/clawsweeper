## Summary

Keep open. Current main still rejects the requested persistent `mediaLocalRoots` config keys for Telegram and agent defaults; the only merged controls are Telegram's separate `trustedLocalFileRoots`, managed/agent-scoped media roots, and broad filesystem-policy driven root expansion. The related shared `tools.fs.roots` direction is still an open, security-sensitive implementation candidate rather than shipped behavior.

## What This Changes

The issue requests a validated configuration surface for outbound local-media roots so Telegram can send local files outside OpenClaw-managed directories.

## Best Possible Solution

Ship a shared, documented, schema-backed operator allowlist, likely `tools.fs.roots` or an equivalent filesystem/media policy surface, that feeds local media access across channels with config types, generated schema/help, docs, and regression tests aligned.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current strict Zod schemas omit `mediaLocalRoots` at both reported config locations while Telegram outbound can only consume roots supplied by core.

## Solution Assessment

No for a Telegram-only schema exception as the final direction. The maintainable fix should be a shared filesystem/media root policy reviewed across host FS tools, media loading, outbound delivery, and channel adapters.
