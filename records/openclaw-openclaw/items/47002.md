## Summary

Keep open. Current main still has the reported config/schema gap: Telegram outbound runtime accepts forwarded mediaLocalRoots, but strict validation still does not accept mediaLocalRoots under channels.telegram, agents.defaults, per-agent entries, or top-level media. The related CLI-media issue is addressed by source-based parent-root expansion, but that does not provide the persistent config allowlist requested here.

## Best Possible Solution

Keep this open and implement a schema-backed, documented outbound media allowlist if persistent roots are still desired. The likely direction is a generic config surface merged by src/media/local-roots.ts, with validation for absolute/narrow roots and regression coverage proving the chosen key validates and reaches Telegram outbound sends. If maintainers decide source-based expansion is the intended replacement, update docs and stale type/API references explicitly instead of closing this report as fixed.
