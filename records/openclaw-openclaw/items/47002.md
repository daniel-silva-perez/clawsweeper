## Summary

Current main still does not implement the requested schema-backed outbound media root allowlist. Telegram config now has `trustedLocalFileRoots`, but that is scoped to self-hosted Telegram Bot API absolute `file_path` reads, not outbound local media sends. `agents.defaults.mediaLocalRoots` and `channels.telegram.mediaLocalRoots` remain invalid, while the runtime only has built-in roots, agent workspace roots, and fs-policy-based widening for concrete media sources. Related PR #52951 may supersede the design, but it is still open and broad/security-sensitive.

## Best Possible Solution

Keep this open and route it through the shared media/FS policy decision. The best implementation is a documented, schema-backed operator allowlist, likely the `tools.fs.roots` direction from #52951 or an equivalent shared surface, that feeds outbound media access across channels with config types, generated schema/help, docs, and regression tests aligned. If #52951 lands and covers Telegram outbound media from configured roots, this can close as implemented by that merged PR.
