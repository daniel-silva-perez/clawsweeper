## Summary

Keep open. Current main still rejects the requested persistent `mediaLocalRoots` config keys for Telegram and agent defaults; Telegram runtime paths can consume roots when core supplies them, but the shipped schema/config surface only has `trustedLocalFileRoots`, managed media roots, and dynamic filesystem-policy root expansion. The overlapping shared `tools.fs.roots` solution remains open in PR #52951 and needs maintainer product/security review before this issue can be resolved.

## What This Changes

The issue asks for a schema-backed way to configure additional local media roots for Telegram outbound media through `channels.telegram`, `agents.defaults`, or a shared media/filesystem policy.

## Best Possible Solution

Ship a shared, documented, schema-backed operator allowlist, such as `tools.fs.roots` or an equivalent filesystem/media policy surface, that feeds local media access across channels with config types, generated schema/help, docs, and regression tests aligned.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: the strict current schemas omit `mediaLocalRoots` at both reported config locations while Telegram outbound code still accepts supplied roots.

## Solution Assessment

No, not as a Telegram-only schema exception. The maintainable solution should be a shared filesystem/media root policy reviewed across host FS tools, local media loading, outbound delivery, and channel adapters.
