## Summary

Current main still does not provide the requested schema-backed outbound media root allowlist. The exact `mediaLocalRoots` type/schema mismatch is partly stale because Telegram config now exposes `trustedLocalFileRoots` instead, but that setting is for self-hosted Telegram Bot API absolute `file_path` reads, not outbound local media sends. Runtime can widen roots for concrete media sources when filesystem policy allows it, and `tools.fs.workspaceOnly=false` covers some host-local sends, but there is still no granular validated allowlist such as `agents.defaults.mediaLocalRoots`, `channels.telegram.mediaLocalRoots`, or merged `tools.fs.roots` on main. Open PR #52951 may become the canonical shared solution, but it is still open and security-sensitive.

## Best Possible Solution

Keep this open and route it through the shared media/FS policy decision. The best implementation is a documented, schema-backed operator allowlist, likely the `tools.fs.roots` direction from #52951 or an equivalent shared surface, that feeds outbound media access across channels with config types, generated schema/help, docs, and regression tests aligned. If #52951 lands and covers Telegram outbound media from configured roots, this can close as implemented by that merged PR.
