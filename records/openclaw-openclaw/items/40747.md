## Summary

Keep this PR open, but it still needs changes before merge. Current main does not support the requested prepend/root-index insert shape: `feishu_doc` still requires `after_block_id` and computes insertion from that anchor. The PR direction is valid and dependency-backed, but the new `index` parameter is under-validated as `Type.Number()`, allowing negative and fractional positions that should be rejected at tool-schema validation time.

## What This Changes

The PR makes Feishu `feishu_doc` insert accept an optional `after_block_id`, adds an optional root-child `index`, and forwards that index into `insertDoc` when no anchor block is provided.

## Best Possible Solution

Land the Feishu plugin feature after constraining `index` to a non-negative integer, preserving the existing anchored insert behavior, adding focused regression coverage for root-index and batched insert paths, updating bundled Feishu doc skill guidance, and adding a user-facing changelog entry crediting the contributor.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Constrain insert index to non-negative integers:** `extensions/feishu/src/doc-schema.ts:43-48`
  - body: `index` is a 0-based child position, but `Type.Number()` accepts fractional and negative values, so invalid inputs bypass tool validation and fail later in Feishu. Use `Type.Integer({ minimum: 0 })`, matching the existing `upload_image` positional schema.
