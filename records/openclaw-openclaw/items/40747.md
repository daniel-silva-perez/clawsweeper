## Summary

Keep this PR open. Current main still has the reported Feishu doc insert gap, and the PR is a focused plugin-side implementation, but the proposed schema accepts invalid positional values and the change still needs regression coverage and skill-doc updates before merge.

## What This Changes

This PR makes `feishu_doc` insert's `after_block_id` optional, adds an optional root-child `index` parameter, and forwards that index into `insertDoc` when no anchor block is supplied.

## Best Possible Solution

Land this feature in the Feishu plugin with `after_block_id` optional only for `action: "insert"`, `index` validated as a non-negative integer, root insertion defaulting to `index ?? 0`, preserved after-block behavior, batched insertion coverage, and Feishu doc skill guidance updated to replace the stale specific-location workaround.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Constrain insert index to non-negative integers:** `extensions/feishu/src/doc-schema.ts:43-48`
