## Summary

Keep this PR open. The Feishu insert-by-index feature is still missing from current main and the PR is focused on the right plugin boundary, but the proposed schema accepts invalid positional values and the branch lacks regression tests and skill-doc updates needed before merge.

## What This Changes

This PR makes the Feishu `feishu_doc` insert action accept an optional root-child `index` by making `after_block_id` optional and forwarding the index into `insertDoc` when no anchor block is supplied.

## Best Possible Solution

Land this feature in the Feishu plugin with `after_block_id` optional only for `action: "insert"`, `index` validated as a non-negative integer, root insertion defaulting to `index ?? 0`, preserved after-block behavior, batched insertion coverage, and Feishu doc skill guidance updated to replace the stale specific-location workaround.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Constrain insert index to non-negative integers:** `extensions/feishu/src/doc-schema.ts:43-48`
  - body: `index` is a 0-based child position, but `Type.Number()` lets callers pass negative or fractional values that bypass tool validation and fail later in the Feishu API. Use `Type.Integer({ minimum: 0 })`, matching the existing `upload_image` positional schema.
