## Summary

Keep this PR open. The requested Feishu insert-by-index feature is still missing on current main and the proposed approach matches the SDK capability, but the branch is not merge-ready because the new `index` parameter is under-validated and the feature still needs focused tests and skill-doc updates.

## What This Changes

The PR changes the Feishu `feishu_doc` insert action to make `after_block_id` optional and forward an optional root-child `index` into `insertDoc` when no anchor block is supplied.

## Best Possible Solution

Land the feature inside the Feishu plugin with `after_block_id` optional only for `action: "insert"`, `index` validated as a non-negative integer, root insertion defaulting to `index ?? 0`, preserved after-block behavior, coverage for normal and batched insert paths, updated Feishu doc skill guidance, and a user-facing changelog entry crediting the contributor.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Constrain insert index to non-negative integers:** `extensions/feishu/src/doc-schema.ts:43-48`
  - body: `index` is a 0-based child position, but `Type.Number()` accepts negative and fractional values, so invalid inputs bypass tool validation and fail later in the Feishu API. Use `Type.Integer({ minimum: 0 })`, matching the existing `upload_image` index schema.
  - confidence: 0.93
