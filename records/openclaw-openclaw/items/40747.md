## Summary

Keep this PR open. It addresses a real current-main Feishu gap, but the proposed schema exposes the new child-position `index` as an unconstrained number, so fractional and negative values would pass OpenClaw tool validation and fail later at the Feishu API boundary.

## What This Changes

The PR makes Feishu `feishu_doc` insert accept an optional `after_block_id`, adds a root-child `index` parameter, and forwards that index to Descendant insertion when no anchor block is provided.

## Best Possible Solution

Land the Feishu plugin feature after constraining `index` to a non-negative integer, preserving anchored insert behavior, adding focused runtime/schema coverage for root-position inserts, updating the bundled Feishu document skill guidance, and adding a changelog entry that credits the contributor.

## Reproduction Assessment

Yes. Static current-main verification is enough: the insert schema requires `after_block_id`, and the runtime always fetches an anchor block before computing the insertion index, so a no-anchor prepend request has no supported path.

## Solution Assessment

No as proposed. Exposing the existing Feishu Descendant root insertion path through the Feishu plugin is the right boundary, but the schema should reject fractional and negative indexes and the landing change should include tests/docs/changelog.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Constrain insert index to non-negative integers:** `extensions/feishu/src/doc-schema.ts:43-48`
