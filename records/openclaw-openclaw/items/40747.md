## Summary

Keep this PR open. Current main still lacks the requested Feishu `feishu_doc` no-anchor root-index insert path, and the PR targets the right Feishu plugin boundary, but the patch is not merge-ready because its new `index` parameter accepts negative and fractional numbers before they reach the Feishu API.

## What This Changes

The PR makes Feishu `feishu_doc` insert accept an optional `after_block_id`, adds an optional root-child `index`, and forwards that index into `insertDoc` when no anchor block is supplied.

## Best Possible Solution

Land the Feishu plugin feature after constraining `index` to a non-negative integer, preserving existing anchored insert behavior, adding focused regression coverage for single and batched root-index inserts, updating the Feishu document skill guidance, and adding a changelog entry that credits the contributor.

## Reproduction Assessment

Yes. Current main can be checked directly: the `insert` schema requires `after_block_id`, and `insertDoc` fetches that anchor before it computes the insertion position, so a no-anchor prepend call has no supported path.

## Solution Assessment

No as proposed. Exposing the Descendant API `index` through the Feishu plugin is the right narrow boundary, but the PR should validate `index` with `Type.Integer({ minimum: 0 })` and add tests/docs/changelog before merge.
