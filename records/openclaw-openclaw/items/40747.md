## Summary

Keep this PR open. Current main still requires `after_block_id` for `feishu_doc` insert and does not expose root-child `index` insertion; the PR remains a focused Feishu plugin implementation candidate, but needs the reviewed schema validation fix plus tests/docs before merge.

## Best Possible Solution

Keep this PR open, or land an equivalent maintainer patch in `extensions/feishu`: make `after_block_id` optional only for `action: "insert"`, add `index: Type.Optional(Type.Integer({ minimum: 0 }))`, default missing `after_block_id` to root insertion at `index ?? 0`, preserve the existing after-block branch, forward the index through normal and batched Descendant insertion paths, and add targeted Feishu doc tool tests plus skill docs for the new mode.

## Risks / Open Questions

- The submitted PR should not merge as-is because `index: Type.Number()` would allow negative and fractional values through tool validation; use `Type.Integer({ minimum: 0 })`.
- The feature needs focused regression coverage for index-only root insertion, preserved `after_block_id` behavior, and the batched insertion path.
- The Feishu document skill docs should be updated so users know the insert-by-index mode exists and stale specific-position guidance is corrected.
