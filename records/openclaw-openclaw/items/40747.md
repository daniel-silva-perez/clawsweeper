## Summary

Keep this PR open. Current main still lacks no-anchor Feishu document insertion, and the PR is a useful localized feature, but it should not merge as-is because the new root-child `index` accepts invalid negative and fractional values before Feishu rejects them downstream; tests, skill docs, and changelog follow-through are still needed.

## What This Changes

The PR makes Feishu `feishu_doc` insert accept an optional `after_block_id`, adds an optional root-child `index`, and forwards that index into `insertDoc` when no anchor block is supplied.

## Best Possible Solution

Land the Feishu plugin feature after constraining `index` to a non-negative integer, preserving existing anchored insert behavior, adding focused regression coverage for single and batched root-index inserts, updating the Feishu doc skill guidance, and adding a user-facing changelog entry crediting the contributor.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Constrain insert index to non-negative integers:** `extensions/feishu/src/doc-schema.ts:43-48`
