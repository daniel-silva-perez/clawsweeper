## Summary

Keep this PR open. Current main still lacks no-anchor Feishu document insertion, but the patch needs a schema fix before merge because it accepts fractional and negative root indexes instead of rejecting them at tool validation; it also needs focused tests, docs, and changelog follow-through.

## What This Changes

The PR makes Feishu `feishu_doc` insert accept an optional `after_block_id`, adds an optional root-child `index`, and forwards that index into `insertDoc` when no anchor block is supplied.

## Best Possible Solution

Land the Feishu plugin feature after constraining `index` to a non-negative integer, preserving existing anchored insert behavior, adding focused regression coverage for single and batched root-index inserts, updating the bundled Feishu document skill guidance, and adding a changelog entry that credits the contributor.

## Reproduction Assessment

Yes. Current main can be checked directly: the `insert` schema requires `after_block_id`, and `insertDoc` fetches that anchor before it computes the insertion position, so a no-anchor prepend call has no supported path.

## Solution Assessment

No as proposed. Exposing Feishu Descendant API root insertion through the Feishu plugin is the right narrow boundary, but the schema should use `Type.Integer({ minimum: 0 })` and the branch should include tests/docs/changelog before merge.
