## Summary

Keep this PR open. Current main still resolves google-vertex ADC to the non-secret marker `gcp-vertex-credentials`, then stores that marker as runtime auth in both embedded run and compaction when no provider runtime auth hook returns a real credential. The Google transport still turns raw non-JSON credentials into `x-goog-api-key`, so the PR’s intended bug is not implemented on main. The PR also remains stale against the refactored run auth controller and still has out-of-scope compact-flow Copilot auth review concerns.

## Best Possible Solution

Keep this PR open as a useful bug-fix candidate, but require a rebase and a current-main fix. The best path is to address the active `gcp-vertex-credentials` marker in both `src/agents/pi-embedded-runner/run/auth-controller.ts` and `src/agents/pi-embedded-runner/compact.ts`, or add a google-vertex runtime auth hook that produces real request/Bearer auth. Remove or separately justify the Copilot compact change, then add regression tests for ADC marker suppression and real google-vertex API-key preservation.
