## Summary

Keep open. Current main has refactored the embedded run auth path, but it still does not implement this PR’s intended google-vertex ADC fix: ADC resolves to the current non-secret marker `gcp-vertex-credentials`, that marker is still stored as runtime auth when no plugin exchange returns a real token, and the Google transport serializes raw non-JSON credentials as `x-goog-api-key`. The PR is also stale against current main and still needs the review concerns addressed before it would be safe to land.

## Best Possible Solution

Keep this PR open only as a useful bug-fix candidate. The best path is to rebase it onto current main, address the current `gcp-vertex-credentials` marker path in both `run/auth-controller.ts` and `compact.ts` or add a google-vertex runtime auth hook that produces real Bearer/request auth, remove or separately justify the github-copilot compact-flow change, and add focused regression tests proving ADC markers are not stored as API keys while real google-vertex API keys still are.
