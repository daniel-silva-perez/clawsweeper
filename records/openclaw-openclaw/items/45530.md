## Summary

Keep open. Current main still lacks persistent TTS voice switching, and this branch is stale against the speech-core/plugin-provider refactor, so it would miss the active parser, runtime facade, speech-core synthesis paths, and Discord voice path even though earlier review feedback was addressed on the branch.

## What This Changes

The PR adds provider-keyed TTS voice preferences plus a `persist=true` directive, with reads, best-effort writes, and tests wired through the older core TTS, telephony, and Discord voice paths.

## Best Possible Solution

Port the feature to the current provider-aware TTS architecture: add a typed persistence signal to `src/tts/directives.ts` and `TtsDirectiveParseResult`, add provider-keyed voice prefs and runtime facade APIs in speech-core/SDK, resolve the active provider’s voice override through provider-owned parsing data, write prefs only after successful synthesis as best-effort, cover telephony and Discord voice, and document the durable directive behavior.
