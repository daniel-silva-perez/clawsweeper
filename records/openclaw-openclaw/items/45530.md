## Summary

Keep this PR open. Current main still does not implement persistent TTS voice switching via `persist=true`, and the branch is now stale against the speech-core/plugin-provider refactor: the active directive parser, prefs model, runtime facade, speech-core synthesis paths, and Discord voice path have no persisted-voice signal or API. The earlier review feedback was addressed in the PR discussion, but the feature needs to be ported to the current architecture before it can be considered landable.

## What This Changes

The PR adds provider-keyed persisted TTS voice preferences and a `persist=true` TTS directive, wiring reads and best-effort writes through the old core TTS, Discord voice, telephony, and tests.

## Best Possible Solution

Port the feature to the current provider-aware TTS architecture: add a typed persistence signal to `src/tts/directives.ts` and `TtsDirectiveParseResult`, add provider-keyed voice prefs and runtime facade APIs in speech-core/SDK, resolve the active provider’s voice override through provider-owned parsing data, write prefs only after successful synthesis as best-effort, cover telephony and Discord voice, and document the durable directive behavior.
