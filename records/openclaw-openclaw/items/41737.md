## Summary

Keep open. Current main still lacks the requested vLLM endpoint-management flow, and the PR branch is not merge-ready because its new vLLM auth handler is unreachable from production dispatch and the branch includes unrelated security-policy, build-script, secrets-baseline, agents, and memory-test scope.

## What This Changes

The PR adds a managed vLLM setup flow with multi-endpoint keys, model discovery, stale default cleanup, docs/tests, and additional changes to A2UI bundling, zizmor/secrets metadata, agents, and memory tests.

## Best Possible Solution

Preserve the vLLM endpoint-management direction, rebase it onto the current plugin-provider setup, wire the vLLM-specific flow through a production provider-owned setup seam, keep stale model cleanup covered by targeted tests, update the public docs, and split A2UI, zizmor, secrets-baseline, agents, and memory-test changes into separately reviewed work.
