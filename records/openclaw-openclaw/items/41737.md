## Summary

Keep this PR open. Current main still provides vLLM through the bundled provider plugin's generic single-endpoint self-hosted setup flow, not the endpoint lifecycle manager, multi-endpoint `vllm-2` style selection, or stale vLLM default/fallback cleanup proposed here. The PR is still potentially useful, but it needs a rebase/rework against the current provider-plugin architecture and a scope/security cleanup pass before maintainer review.

## Best Possible Solution

Keep this PR open for contributor/maintainer rework. The best path is to rebase onto current main, implement vLLM endpoint add/update/delete, multi-model discovery/selection, managed secondary provider keys, and stale default/fallback pruning through the current bundled vLLM provider/plugin setup path, add focused tests and docs, and drop or split unrelated `.secrets.baseline`, memory test, A2UI script, and `zizmor.yml` changes.
