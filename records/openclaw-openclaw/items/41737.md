## Summary

Keep open. The vLLM endpoint-management feature is still not present on current main, but this PR is not merge-ready: the branch is unmergeable, targets older core command setup paths instead of the current plugin-owned provider setup surface, leaves the new vLLM apply handler unreachable, and mixes unrelated build/security/test changes into the provider feature.

## What This Changes

The PR branch adds a core vLLM endpoint-management setup flow with multi-endpoint provider keys, model discovery and multi-select, stale vLLM default cleanup, docs/tests, and unrelated A2UI bundling, zizmor/secrets, agents, and memory-test changes.

## Best Possible Solution

Preserve the vLLM endpoint-management direction, but rework it as a focused plugin-owned change under the current vLLM provider and provider-setup SDK surfaces, with stale model cleanup covered by targeted tests and public docs; split A2UI, zizmor, secrets-baseline, agents, and memory-test changes into separately reviewed work.
