## Summary

Keep open. Current main still drains queued runtime system events, formats them as `System:` lines, and prepends them into the command and follow-up user prompt paths; the linked closing PR #72761 is open and unmerged, so this should wait for maintainer review/merge or an equivalent fix.

## What This Changes

The issue requests that background task updates, deferred maintenance notices, and heartbeat/system prompts stop being injected as user-role prompt text and instead use system/runtime context or another non-user mechanism.

## Best Possible Solution

Review and land #72761 or an equivalent narrow core change that routes drained runtime-generated system events through runtime system context, keeps them out of `commandBody`, `followupRun.prompt`, and transcript prompt text, and preserves the existing untrusted-event sender downgrade.

## Reproduction Assessment

Yes. The read-only reproduction path is the existing mocked `drainFormattedSystemEvents` tests, which currently assert the event block appears in `commandBody` and `followupRun.prompt` instead of runtime system context.

## Solution Assessment

Yes for the queued runtime system-event bug: moving drained events into `extraSystemPrompt`/runtime context is the narrow maintainable fix. Heartbeat prompt semantics should remain a separate maintainer-scoped decision unless the linked PR intentionally expands scope.
