## Summary

Keep open. Current main still routes drained runtime system events into user prompt surfaces, and the linked closing PR #72761 is open and unmerged, so this should wait for maintainer review/merge or an equivalent fix rather than cleanup closure.

## What This Changes

The issue requests that background task updates, deferred maintenance notifications, and heartbeat/system notices stop being delivered to the model as user-role prompt text.

## Best Possible Solution

Land #72761 or an equivalent narrow change that routes drained runtime-generated system events through runtime system context, keeps them out of commandBody, followupRun.prompt, and transcript prompt text, and preserves the existing untrusted-event ownership downgrade unless maintainers intentionally revise heartbeat semantics.

## Reproduction Assessment

Yes. The source and tests provide a high-confidence read-only reproduction path: mocked drainFormattedSystemEvents output is asserted to appear in commandBody and followupRun.prompt rather than runtime system context.

## Solution Assessment

Yes for the queued runtime-system-event bug. Moving drained events out of buildReplyPromptBodies and into extraSystemPrompt/runtime context is the narrow maintainable fix; heartbeat prompt semantics need explicit maintainer scope if they are meant to change too.
