## Summary

Keep open. Current main still routes drained runtime system events into user prompt text, and the open closing PR #72761 is unmerged, so this should wait for maintainer review/merge of that PR or an equivalent fix.

## What This Changes

The issue asks for background task and maintenance system updates to be delivered as system/runtime context instead of polluting user-role prompt text.

## Best Possible Solution

Land #72761 or an equivalent narrow change that moves drained runtime-generated system events into runtime system context, keeps them out of commandBody, queued prompts, and transcript prompts, and preserves the existing untrusted-event ownership downgrade.

## Reproduction Assessment

Yes. Current main has a focused source/test reproduction path: mock drainFormattedSystemEvents during runPreparedReply and observe the event text in commandBody and followupRun.prompt instead of extraSystemPrompt.

## Solution Assessment

Yes for the queued runtime-system-event bug. Moving drained events out of buildReplyPromptBodies and into extraSystemPrompt/runtime context is the narrow maintainable fix; heartbeat prompt behavior should stay as documented unless maintainers choose a broader product change.
