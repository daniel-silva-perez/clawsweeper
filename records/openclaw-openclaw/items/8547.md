## Summary

Keep open. Current main still does not provide a user-configurable Control UI/WebChat font-size or font-scale setting in browser-local UI settings, Appearance/Quick Settings, gateway `ui` config, public docs, or CSS variables. The related #3027 is closed and does not supersede this remaining accessibility/customization request; the existing maintainer-side Codex review comment also kept this open for the same core UI work.

## Best Possible Solution

Implement a bounded Control UI text-size or font-scale setting in `UiSettings`, expose it in Appearance and Quick Settings, apply it via CSS custom properties across chat and surrounding UI, and add storage/rendering tests plus public docs. Extend the gateway `ui` schema/types/docs only if maintainers want server-side font sizing as a public config contract.
