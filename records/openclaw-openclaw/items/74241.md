## Summary

Current main has partial timezone support, but it does not fully satisfy this issue. Gateway `agent` and `chat.send` turns get a timestamp from `agents.defaults.userTimezone`, and channel envelopes can use that default when configured, but transport timestamps still default to host-local time and `agents.list[]` has no per-agent/profile timezone field. A Docker-host/user timezone split can therefore still drift unless the gateway is globally configured, and the requested per-profile/user-local runtime context remains missing.

## Best Possible Solution

Keep this issue open and implement a narrow timezone-resolution seam: allow per-agent/profile timezone and time-format overrides that fall back to `agents.defaults`, use that resolved value for gateway `agent`/`chat.send` timestamp injection and system prompt timezone context, and update envelope/user-timezone docs and tests without changing cron/heartbeat behavior unexpectedly.

## Repair Work Prompt

Fix the remaining timezone desync gap for this issue. Current main already has `injectTimestamp` and `agents.defaults.userTimezone`, but it lacks per-agent/profile timezone support and several call sites resolve only defaults.

Expected boundary:
- Add a per-agent timezone/time-format config surface under `agents.list[]` if it fits existing config patterns, falling back to `agents.defaults.userTimezone` and `agents.defaults.timeFormat`.
- Prefer a small shared resolver, likely near `src/agents/agent-scope-config.ts` or `src/agents/date-time.ts`, that accepts `cfg` plus `agentId` and returns the effective timezone/time format.
- Use the effective timezone for gateway `agent` and `chat.send` timestamp injection. Preserve the existing rule that dynamic date/time does not go into the cached system prompt.
- Use the effective timezone for system prompt timezone context where an `agentId` is already available.
- Consider envelope/user timestamp resolution for routed channel turns, but do not silently break the documented host-local default unless tests/docs explicitly justify the change.
- Update config types, zod/schema/generated docs as required by repo conventions, and user-facing docs at `docs/date-time.md`, `docs/concepts/timezone.md`, and/or `docs/gateway/config-agents.md`.
- Add focused tests for default fallback, per-agent override, invalid timezone fallback, and no double-stamping/cron timestamp regression.
- Do not add dynamic current clock text back into `src/agents/system-prompt.ts`; keep prompt cache stability.

Validation to run: targeted Vitest for timestamp/prompt/envelope/config tests, targeted oxfmt check for touched files, and `pnpm check:changed` via Testbox before handoff if config/generated/docs surfaces changed. Changelog likely not needed unless maintainers treat this as user-facing config behavior; if added, follow the active-version changelog attribution rules.
