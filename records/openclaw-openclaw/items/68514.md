## Summary

Keep open. Current main still sends Feishu post messages as a single `md` tag with the raw message text, so the underscore URL boundary bug is not implemented on main; this PR is useful but has a blocking markdown masking bug before it should merge.

## What This Changes

The branch adds a Feishu-local URL wrapping helper, wires it into `buildFeishuPostMessagePayload`, and adds unit plus send-path coverage for bare http(s) URLs with underscores in post `md` messages.

## Best Possible Solution

Land one Feishu-local fix that preserves full clickable bare URLs in post messages while robustly leaving existing Markdown links, image links, autolinks, inline code, and fenced code untouched; then close the duplicate Feishu URL PRs and the shared bug report from that canonical merge.

- **[P2] Mask nested-bracket markdown links before wrapping URLs:** `extensions/feishu/src/markdown-urls.ts:8`
