## Summary

Keep open. The PR addresses a real Feishu URL rendering bug that is still present on current main, but the proposed markdown-link masking has a concrete parser defect that mutates existing links with nested brackets, so the patch should not merge as-is.

## What This Changes

The branch adds a Feishu-local helper that wraps bare http(s) URLs before building post markdown payloads, wires it into `buildFeishuPostMessagePayload`, and adds helper plus send-path tests for underscore-containing URLs.

## Best Possible Solution

Land one Feishu-local implementation that keeps full bare URLs clickable in post messages while robustly preserving existing Markdown links, image links, angle autolinks, inline code, and fenced code; then close the duplicate Feishu URL PRs and the shared bug report from that canonical merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Mask nested-bracket markdown links before wrapping URLs:** `extensions/feishu/src/markdown-urls.ts:8`
  - body: The new `MARKDOWN_LINK_RE` only accepts link text without `]`, so an existing link like `[foo [bar]](https://example.com/a_b)` is not protected before the bare-URL pass. The helper then rewrites the URL inside the already-linked destination, mutating content it promises to leave untouched and potentially breaking Feishu rendering.
  - confidence: 0.91
