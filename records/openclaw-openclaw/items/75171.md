## Summary

Keep open. Current main still matches the reported SDK compatibility regression: the focused channel reply SDK exports `createReplyPrefixContext`, but the deprecated compat/root SDK surface used by legacy external plugins does not, so the reported Feishu dispatch `TypeError` remains actionable.

## What This Changes

The issue reports a Feishu external plugin dispatch regression where a legacy root `openclaw/plugin-sdk` import does not provide `createReplyPrefixContext` after updating OpenClaw.

## Best Possible Solution

Restore the missing channel reply helper exports on the deprecated compat/root SDK surface for the active migration window, keep `openclaw/plugin-sdk/channel-reply-pipeline` as the canonical import, and add contract coverage so future SDK migration edits cannot silently drop those legacy exports.

## Reproduction Assessment

Yes. The issue gives deterministic Feishu dispatch steps, and current-main source inspection reproduces the underlying exported-shape failure: the helper exists on the focused subpath but not on the compat-backed root SDK path used by legacy external plugin imports.

## Solution Assessment

Yes. A narrow SDK compatibility repair is the best solution; changing bundled Feishu would not address external plugins still allowed to use deprecated root/compat imports during the documented migration window.
