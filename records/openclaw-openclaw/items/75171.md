## What This Changes

The issue reports a Windows Feishu plugin dispatch regression after updating to OpenClaw 2026.4.27 where legacy `openclaw/plugin-sdk` consumers cannot call `createReplyPrefixContext`.

## Best Possible Solution

Restore the missing channel reply helper family on the deprecated compat/root SDK surface for the active migration window, keep `openclaw/plugin-sdk/channel-reply-pipeline` as the canonical focused import, and add contract coverage so future SDK migration work cannot drop those legacy exports silently.

## Reproduction Assessment

Yes. The issue gives deterministic Feishu dispatch steps, and current main reproduces the underlying failure condition by inspection: legacy root imports resolve through `root-alias.cjs` to `compat.ts`, where `createReplyPrefixContext` is absent while the focused subpath exports it.

## Solution Assessment

Yes. A narrow SDK compatibility repair is the best path; changing only Feishu would not help external plugins still allowed to use deprecated root/compat imports during the migration window.
