## Summary

Keep open. Current main still treats the reported iMessage chat.db permission denial as an ordinary probe/RPC failure, while this PR is a narrow plugin-layer fix with focused coverage and no blocking correctness or security finding from this review.

## What This Changes

The PR adds iMessage RPC permission-denied text detection in the client and probe paths, fails pending RPC calls on fatal startup denial, terminates the child process, and adds a probe regression test for chat.db access denial.

## Best Possible Solution

Review and land this PR or an equivalent narrow iMessage plugin fix, while keeping #43440 open for the separate process-context or upstream imsg architecture work.

## Solution Assessment

Yes for the bug this PR targets. The narrowest maintainable fix is in the iMessage RPC client/probe layer because the monitor already has a generic fatal-probe path and core does not need iMessage-specific permission policy.
