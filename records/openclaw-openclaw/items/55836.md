## Summary

Keep open. Current main still has no opt-in vi/vim modal input mode: the TUI uses a thin CustomEditor wrapper around the upstream editor, Escape is bound to dismiss/abort behavior, and there is no config/docs surface for `tui.viMode`, `tui.inputMode`, or `OPENCLAW_TUI_INPUT_MODE`. The related implementation PR was closed unmerged, while #22770 tracks a different external-editor compose flow.

## What This Changes

Add an opt-in vi/vim keybinding mode for the TUI input editor so users can use modal editing inside the TUI composer.

## Best Possible Solution

Implement a default-off vi/vim mode in the TUI editor layer, wire it through the supported config schema and docs, and keep #22770 separate as the external-editor compose feature.
