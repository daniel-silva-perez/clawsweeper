## Summary

Keep this PR open. Current main still does not implement the PR's intended compatibility path for runtime legacy `{ ok, to }` results from `messaging.normalizeTarget`; those values are treated as non-strings and the normalized target is lost. The PR addresses a real gap, but its current diff still needs maintainer cleanup around unrelated secret-baseline churn and a broad OAuth test mock.

## Best Possible Solution

Keep the PR open and rework it against current main. The best path is a small runtime-only helper in `src/infra/outbound/target-normalization.ts` that treats the normalizer result as `unknown`, extracts a trimmed nonblank string from legacy `{ ok, to }` values, and leaves malformed objects on the existing fallback/validation path. Add focused coverage in `src/infra/outbound/target-normalization.test.ts` plus one resolver or send-runner regression, then remove the unrelated `.secrets.baseline` churn and broad OAuth mock before maintainer review.
