## Summary

Keep this PR open. Current main still lacks the intended compatibility path for legacy runtime `{ ok, to }` results from `messaging.normalizeTarget`; the normalizer path remains string-only and drops non-string results before target resolution/send dispatch. The PR addresses a real gap, but it needs rework against current main because the runner tests have since been split and the provided diff includes unrelated `.secrets.baseline` churn plus a broad hoisted OAuth test mock.

## Best Possible Solution

Keep the PR open and rework it against current main. The best implementation is a small runtime-only helper in `src/infra/outbound/target-normalization.ts` that treats the normalizer result as `unknown`, extracts a trimmed nonblank `to` from legacy `{ ok, to }` objects, and otherwise preserves existing fallback/validation behavior. Add focused coverage in `src/infra/outbound/target-normalization.test.ts` plus one resolver or current split send-runner regression, then remove the unrelated `.secrets.baseline` churn and broad OAuth mock before maintainer review.
