## Summary

Keep this PR open. Current main still does not tolerate legacy runtime `{ ok, to }` results from `messaging.normalizeTarget`; the normalized target path remains typed and implemented as string-only, so the PRâ€™s core compatibility fix is not obsolete. The PR should be reworked before merge because current main has split the runner tests, the diff still includes unrelated `.secrets.baseline` churn, and the OAuth test mock is broader than the outbound regression needs.

## Best Possible Solution

Keep the PR open and ask for a rebase/rework against current main. The best fix is a small runtime-only helper in `src/infra/outbound/target-normalization.ts` that treats the normalizer result as `unknown`, extracts only a trimmed nonblank string `to` from legacy `{ ok, to }` objects, and otherwise preserves existing fallback/validation behavior. Add focused coverage in `src/infra/outbound/target-normalization.test.ts` plus one current split send/dispatch regression, and remove the unrelated `.secrets.baseline` changes and broad OAuth mock before maintainer review.
