## Summary

Keep this PR open. Current main still does not implement the PR’s legacy normalizer compatibility: object-shaped runtime returns from `messaging.normalizeTarget` are not unwrapped, so the embedded `to` value is still lost. The PR is not obsolete, but it needs rework against current main and should drop unrelated security-baseline/test-workaround churn before maintainer review.

## Best Possible Solution

Keep the PR open and ask for a rebase/rework against current main. The best patch is a small runtime-only helper in `src/infra/outbound/target-normalization.ts` that treats the normalizer candidate as `unknown`, extracts only a trimmed nonblank string `to` from legacy `{ ok, to }` objects, and otherwise preserves existing fallback/validation behavior. Add focused coverage in `src/infra/outbound/target-normalization.test.ts` plus one current split send/dispatch regression, and remove the unrelated `.secrets.baseline` changes and broad OAuth mock before maintainer review.
