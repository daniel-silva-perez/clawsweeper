## Summary

Current main still lacks the requested first-class `openclaw sessions rename` or `openclaw sessions set-subject` command. The Gateway has lower-level `sessions.patch` label plumbing, but the local `openclaw sessions` CLI registration, table rows, JSON output, docs, and tests still do not expose a user-facing rename/set-label flow or show labels in the sessions list.

## Best Possible Solution

Keep this issue open. Implement a focused `openclaw sessions rename <session-key> <label>` or `set-label` command that calls the existing Gateway `sessions.patch` label path, then teach `openclaw sessions` text and JSON output to include an effective human label while preserving channel-owned `subject` and `displayName` semantics. Add targeted command-registration/session-table tests and update https://docs.openclaw.ai/cli/sessions.
