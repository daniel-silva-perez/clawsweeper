## Summary

Keep open. Current main still has the Telegram answer-lane preview gate described in the report, and the docs plus doctor warning surfaces do not yet explain that native quote replies suppress the stream used for tool-progress previews. The issue is also already paired with open closing PRs, so the maintainer path is to review and land the preferred PR rather than close this issue before merge.

## What This Changes

The issue requests documentation and/or an `openclaw doctor` warning when Telegram `replyToMode` prevents `streaming.preview.toolProgress` preview updates from rendering.

## Best Possible Solution

Ship a Telegram-scoped clarification that documents the quote-reply/tool-progress preview trade-off and preferably adds an `openclaw doctor` warning for affected configs, while leaving the intentional runtime quote-reply gate unchanged unless maintainers later choose a larger separate-message fallback design.

## Reproduction Assessment

Yes. A high-confidence reproduction path exists: configure Telegram preview streaming with `replyToMode` enabled and a quotable inbound message, trigger tool use, and the current source creates no answer-lane stream for tool-progress updates; the issue discussion confirms this on v2026.4.27.

## Solution Assessment

Yes. Docs plus a doctor warning is the narrowest maintainable fix for the reported doc/runtime mismatch; a persistent separate-message fallback would be a broader product change and should be treated as follow-up work.
