## Summary

Keep open. Current main still has the Telegram quote-reply gate that prevents the answer preview stream from being created, and the docs plus doctor warnings still do not explain the `replyToMode`/tool-progress exclusion. The issue also has two open closing PRs, so it should stay open until maintainers choose and land or close those candidate fixes.

## What This Changes

The issue asks OpenClaw to document, warn about, or otherwise address Telegram tool-progress preview being silently unavailable when `channels.telegram.replyToMode` enables native quote replies.

## Best Possible Solution

Land the preferred Telegram-scoped clarification, ideally documenting the quote-reply/tool-progress trade-off and adding a doctor warning for affected configs; leave separate persistent-message fallback behavior for a later product decision if maintainers want broader runtime work.

## Reproduction Assessment

Yes. Configure Telegram preview streaming with `streaming.preview.toolProgress` enabled and `replyToMode` set to a quote-reply mode, then trigger tool use on a quotable inbound message; current source shows no answer-lane stream is created, and the issue discussion confirms this on released builds.

## Solution Assessment

Yes. Keeping the issue open for the paired PRs is the right path; a docs plus doctor-warning fix is the narrow maintainable answer to the silent doc/runtime mismatch, while runtime fallback delivery would be broader behavior work.
