## Summary

Keep open. Current main still has the Telegram quote-reply gate that prevents an answer-lane preview stream, tool-progress previews still depend on that stream, and the docs/doctor surfaces do not yet explain the trade-off. The issue is also already paired with open closing PRs #73495 and #73508, so it should remain open until maintainers choose and land the preferred fix.

## What This Changes

The issue requests a Telegram-scoped docs, doctor, or runtime clarification for the case where `replyToMode != "off"` disables tool-progress preview updates.

## Best Possible Solution

Ship a Telegram-scoped clarification that documents the quote-reply/tool-progress preview trade-off and preferably adds a doctor warning for affected configs, while leaving the intentional quote-reply runtime gate unchanged unless maintainers later choose a broader separate-message fallback design.

## Reproduction Assessment

Yes. Configure Telegram preview streaming with `replyToMode` enabled and a quotable inbound message, then trigger tool use; the current source creates no answer-lane stream for tool-progress updates, and the discussion confirms the behavior on v2026.4.27.

## Solution Assessment

Yes. Docs plus a doctor warning is the narrowest maintainable fix for the reported doc/runtime mismatch; delivering tool-progress through separate persistent messages would be a broader product change.
