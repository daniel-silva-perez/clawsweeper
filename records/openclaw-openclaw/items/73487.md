## Summary

Keep open. Current main still has the Telegram quote-reply gate that prevents the answer preview stream from being created, while the Telegram and streaming docs plus doctor warning path still do not explain that `replyToMode` exclusion; open closing PRs already exist for maintainer review.

## What This Changes

The issue asks OpenClaw to document, warn about, or otherwise handle Telegram tool-progress preview being suppressed when `channels.telegram.replyToMode` enables native quote replies.

## Best Possible Solution

Land the preferred Telegram-scoped clarification, ideally documenting the quote-reply/tool-progress trade-off and adding a doctor warning for affected configs, while leaving any persistent-message fallback as a separate product decision.

## Reproduction Assessment

Yes. Configure Telegram preview streaming with `streaming.preview.toolProgress` enabled and `replyToMode` set to a quote-reply mode, then trigger tool use on a quotable inbound message; current source and tests show no answer-lane stream is created, and the issue discussion confirms this on released builds.

## Solution Assessment

Yes. A docs plus doctor-warning fix is the narrowest maintainable solution for the reported silent doc/runtime mismatch; changing runtime delivery to send separate persistent progress messages would be broader behavior work.
