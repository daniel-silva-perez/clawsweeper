## Summary

Keep open. Current main still preserves the Telegram quote-reply gate that prevents the answer preview stream from being created, and the docs plus doctor warnings still do not surface the `replyToMode` exclusion; open closing PRs #73495 and #73508 should be reviewed before this issue is closed.

## What This Changes

The issue asks OpenClaw to document, warn about, or otherwise fix the Telegram configuration case where native quote replies disable tool-progress preview updates despite the docs describing them as default-on when preview streaming is active.

## Best Possible Solution

Land the preferred Telegram-scoped clarification, ideally documenting the quote-reply/tool-progress trade-off and adding a doctor warning for affected configs; leave separate persistent-message fallback behavior for a later product decision if maintainers want broader runtime work.

## Reproduction Assessment

Yes. Configure Telegram preview streaming with `streaming.preview.toolProgress` enabled and `replyToMode` set to a quote-reply mode, then trigger tool use on a quotable inbound message; current source shows no answer-lane stream is created, and the issue discussion confirms this on released builds.

## Solution Assessment

Yes. Keeping this issue open for the paired PRs is the right path; a docs plus doctor-warning fix is the narrowest maintainable answer to the silent doc/runtime mismatch, while runtime fallback delivery would be broader behavior work.
