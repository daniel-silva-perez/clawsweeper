## Summary

Keep open. Current main still has the Telegram answer-lane stream gate described by the report, and neither the Telegram/streaming docs nor the Telegram doctor warnings explain the `replyToMode` trade-off. The issue is also paired with open closing PRs, so it should stay open until maintainers choose and land the preferred fix.

## What This Changes

The issue reports that Telegram tool-progress preview updates are silently unavailable when native reply threading is enabled because `replyToMode` prevents the answer-lane preview stream while docs and doctor output do not disclose that condition.

## Best Possible Solution

Land one canonical Telegram-scoped fix that documents the native quote-reply/tool-progress preview trade-off and preferably adds an `openclaw doctor` warning, while leaving the intentional runtime quote-reply gate unchanged unless maintainers choose a broader separate-message fallback design later.

## Reproduction Assessment

Yes. Configure Telegram preview streaming with `replyToMode: "first"` or `"all"`, trigger a tool-using turn with quotable inbound context, and current source creates no answer-lane stream for tool-progress updates; current tests and the v2026.4.27 confirmation comment support that path.

## Solution Assessment

Yes. A docs clarification plus a Telegram doctor warning is the narrowest maintainable fix for the reported silent doc/runtime mismatch; a persistent separate-message fallback would be a larger product change.
