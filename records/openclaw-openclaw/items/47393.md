## Summary

Keep this PR open. Current main still lacks the requested Discord route-owner suppression and bot-identity fallback behavior, but the proposed branch is not mergeable as-is because it introduces blocking routing bugs, violates the plugin boundary, and is stale against current routing types and provider startup structure.

## What This Changes

The PR adds Discord route-owner resolution, explicit bot mention and bot-name owner bypass handling, `botUserName` threading, an application-id fallback for `botUserId`, and focused Discord monitor tests.

## Best Possible Solution

Refresh or replace the branch on current main with a Discord-plugin-owned implementation that ranks only active Discord accounts, suppresses non-owner guild/channel traffic by default, preserves explicit Discord bot mention bypass, avoids owner blocking for DMs, uses plugin SDK exports or local Discord extension APIs, and includes regression tests for owner precedence, explicit mention bypass, stale/disabled accounts, DMs, wildcard bindings, and identity fallback.

## Reproduction Assessment

Yes. The PR body gives a concrete multi-account Discord reproduction, current main code confirms there is no cross-account owner check, and the PR-introduced regressions are reproducible from code with direct-peer or stale-binding configurations.

## Solution Assessment

No. The product direction is useful, but the current patch is not the best mergeable solution because it blocks valid DMs, can route ownership to inactive accounts, bypasses the extension boundary, and misses current-main wildcard route matches.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Exclude direct messages from owner blocking:** `extensions/discord/src/monitor/message-handler.preflight.ts:437-464`
  - body: The owner guard runs whenever there is no thread or configured binding, including DMs, but `bypassOwnerBecauseMentioned` is always false for DMs. If another account has a more specific direct-peer binding, this bot can silently drop a DM even though that other bot will not receive this DM event.
  - confidence: 0.91
- **[P2] Ignore inactive accounts when choosing the owner:** `extensions/discord/src/monitor/route-owner.ts:36-39`
  - body: `listDiscordCandidateAccountIds` includes account IDs that appear only in bindings. A removed, disabled, or duplicate-token account can win ownership and make active accounts suppress channel messages for a bot that is not running.
  - confidence: 0.88
- **[P2] Import routing through the plugin SDK:** `extensions/discord/src/monitor/route-owner.ts:1-7`
  - body: This new production module imports config and routing internals through `../../../../src/**`. Bundled Discord code should use `openclaw/plugin-sdk/*` or local extension barrels, so this bypasses the plugin boundary expected for bundled and third-party plugins.
  - confidence: 0.86
- **[P2] Add wildcard route priority:** `extensions/discord/src/monitor/route-owner.ts:10-18`
  - body: Current main's `ResolvedAgentRoute["matchedBy"]` includes `binding.peer.wildcard`, but the new priority table omits it. After rebasing, wildcard owner matches will either fail the type contract or get an undefined priority.
  - confidence: 0.84
