## Summary

Keep open. Current main still does not implement the requested cross-account Discord owner suppression or application-id identity fallback, but this PR is not mergeable as-is: the branch is stale against the current Discord plugin layout and introduces concrete routing regressions already visible from the diff.

## What This Changes

The PR adds Discord route-owner selection across accounts, explicit bot/user-name mention owner-bypass handling, `botUserName` plumbing, an application-id fallback for `botUserId`, and focused Discord monitor tests.

## Best Possible Solution

Refresh or replace the branch with a Discord-plugin-owned implementation on current main that ranks only active Discord accounts, suppresses non-owner guild/channel traffic by default, preserves explicit Discord bot mention bypass, avoids owner blocking for DMs, uses plugin SDK or local Discord extension exports, and includes regression coverage for owner precedence, explicit mentions, inactive/stale accounts, wildcard peer bindings, direct messages, and identity fallback.

## Reproduction Assessment

Yes. The PR body gives a concrete multi-account Discord reproduction, current main confirms the missing cross-account owner check, and the PR-introduced regressions follow directly from the added owner-check and candidate-account code paths.

## Solution Assessment

No. The requested direction is useful, but this patch is not the best mergeable solution because it blocks valid DM cases, can route ownership to inactive accounts, bypasses the extension boundary, and misses the current-main wildcard route tier.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Exclude direct messages from owner blocking:** `extensions/discord/src/monitor/message-handler.preflight.ts:437-464`
  - body: The owner guard runs whenever there is no thread or configured binding, including DMs, but `bypassOwnerBecauseMentioned` is always false for DMs. If another account has a more specific direct-peer binding, this bot can silently drop a DM even though the other bot will not receive this DM event.
  - confidence: 0.91
- **[P2] Ignore inactive accounts when choosing the owner:** `extensions/discord/src/monitor/route-owner.ts:36-39`
  - body: `resolveDiscordRouteOwner` considers account IDs that appear only in bindings. A removed, disabled, or duplicate-token account can win ownership and make active accounts suppress channel messages for a bot that is not running.
  - confidence: 0.88
- **[P2] Import routing through the plugin SDK:** `extensions/discord/src/monitor/route-owner.ts:1-7`
  - body: This new production module imports config and routing internals through `../../../../src/**`. Bundled Discord code should use `openclaw/plugin-sdk/*` or local extension barrels, so this bypasses the plugin boundary expected for bundled and third-party plugins.
  - confidence: 0.86
- **[P2] Add wildcard route priority:** `extensions/discord/src/monitor/route-owner.ts:10-18`
  - body: Current main's `ResolvedAgentRoute["matchedBy"]` includes `binding.peer.wildcard`, but the new priority table omits it. After rebasing, wildcard owner matches will either fail the type contract or get an undefined priority.
  - confidence: 0.84
