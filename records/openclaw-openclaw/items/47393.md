## Summary

Keep this PR open. Current main still lacks the proposed cross-account Discord owner suppression and identity-fetch fallback, but the branch is not merge-ready because the patch can drop direct messages, can route ownership to inactive/stale accounts, bypasses the bundled-plugin boundary, and is stale against current main routing/provider contracts.

## What This Changes

The PR adds a Discord route-owner resolver with tests, augments message preflight with explicit bot mention/name owner bypass handling, threads `botUserName`, and preserves `botUserId` from the application id when Discord identity lookup fails.

## Best Possible Solution

Refresh or replace the branch on current main with a Discord-plugin-owned implementation that compares only active Discord accounts, suppresses non-owner guild/channel traffic by default, preserves explicit bot mention bypass, handles reply-target precedence if maintainers fold in the related routing issues, and keeps all production imports behind plugin SDK or local Discord extension APIs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Exclude direct messages from owner blocking:** `extensions/discord/src/monitor/message-handler.preflight.ts:437`
  - body: The new owner guard runs whenever there is no thread/configured binding, including DMs, but the bypass path is forced off for DMs. If another account has a more specific direct-peer binding, this bot can drop a DM even though the other bot will not receive that DM event.
  - confidence: 0.91
- **[P2] Ignore inactive accounts when choosing the owner:** `extensions/discord/src/monitor/route-owner.ts:36-39`
  - body: `listDiscordCandidateAccountIds` includes account IDs that appear only in bindings. A removed, disabled, or duplicate-token account can win ownership and make active accounts suppress channel messages for a bot that is not running.
  - confidence: 0.88
- **[P2] Use plugin SDK exports from Discord production code:** `extensions/discord/src/monitor/route-owner.ts:1-7`
  - body: This new production module imports config and routing internals via `../../../../src/**`. Bundled Discord code should use `openclaw/plugin-sdk/*` or local extension barrels, so this bypasses the plugin boundary expected for bundled and third-party plugins.
  - confidence: 0.86
- **[P2] Add the wildcard route priority:** `extensions/discord/src/monitor/route-owner.ts:10-18`
  - body: Current main's `ResolvedAgentRoute["matchedBy"]` includes `binding.peer.wildcard`, but the new priority table omits it. After rebasing, wildcard owner matches will either fail the type contract or get an undefined priority.
  - confidence: 0.84
