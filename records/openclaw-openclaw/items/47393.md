## Summary

Keep this PR open. Current main still lacks the proposed Discord cross-account owner guard and identity-fetch fallback, but the PR should not merge as-is because the proposed owner guard can silently drop DMs, considers stale/non-running accounts as owners, and needs refresh to the current plugin SDK boundary.

## What This Changes

The PR adds Discord cross-account route-owner suppression with explicit mention bypass, bot-name mention fallback, tests for ownership routing, and an application-id fallback for bot identity lookup failures.

## Best Possible Solution

Refresh or replace the PR on current main, keep Discord production code behind plugin SDK/local extension APIs, implement guild-channel owner resolution only across active runnable Discord accounts, preserve explicit mention bypass and the application-id identity fallback, skip or redesign DM owner blocking, and add focused regression tests for owner precedence, explicit alternate-bot mentions, stale accounts, identity-fetch failure, and any scoped reply-target behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Exclude direct messages from owner blocking:** `extensions/discord/src/monitor/message-handler.preflight.ts:444`
  - body: The new owner guard runs whenever there is no thread/configured binding, including DMs, but `bypassOwnerBecauseMentioned` is always false for DMs. If another account has a more specific direct-peer binding for the same sender, this account drops the DM even though the other bot will not receive that DM channel event.
  - confidence: 0.91
- **[P2] Ignore unavailable accounts when resolving the owner:** `extensions/discord/src/monitor/route-owner.ts:36`
  - body: `listDiscordCandidateAccountIds` includes account IDs found only in bindings. A stale binding for a removed or disabled Discord account can therefore win ownership and cause the active account to suppress messages for an owner that is not running.
  - confidence: 0.88
- **[P2] Use plugin SDK exports from Discord production code:** `extensions/discord/src/monitor/route-owner.ts:1-7`
  - body: The new Discord production module imports config and routing internals via `../../../../src/**`, which violates the bundled plugin boundary and bypasses the public SDK/local extension APIs used on current main. Refresh this code to use `openclaw/plugin-sdk/*` and Discord-local exports before merge.
  - confidence: 0.84
