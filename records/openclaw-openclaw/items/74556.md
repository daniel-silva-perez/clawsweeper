## What This Changes

The PR isolates local Codex stdio app-server `CODEX_HOME` and `HOME` per agent, adds Codex migration/doctor/docs coverage for native Codex assets, forwards Codex auth profiles into harness runs, and adds Slack/Signal untrusted system-event coverage.

## Best Possible Solution

Fix the test type issue, get the PR head validation green, then have a maintainer review and land the stdio-only Codex isolation plus deliberate migration path if the Codex state boundary is accepted.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: local stdio Codex app-server launches inherit the gateway process environment because OpenClaw does not set `CODEX_HOME` or `HOME` before the transport copies `process.env`.

## Solution Assessment

Mostly yes, but not merge-ready as submitted. The product direction is the narrow maintainable fix for OpenClaw-owned stdio launches while leaving WebSocket ownership external and keeping executable native Codex state manual-review-only, but the typecheck failure must be resolved first.
