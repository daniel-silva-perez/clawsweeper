## Summary

Keep open. Current main has the already-landed generic failover handling for connection/internal-server errors, but it does not yet include the Arcee/Trinity compatibility guard that the issue asks for. A maintainer-authored open PR, #73388, is the canonical implementation candidate and directly changes `arcee/trinity-large-thinking` to `compat.supportsTools: false` with catalog tests for both direct Arcee and OpenRouter paths.

## Best Possible Solution

Keep this issue open and use #73388 as the current implementation candidate. The best fix is to land that PR or an equivalent Arcee-owned catalog/compat change, preserving the generic failover handling already covered by #62848 while making Trinity main-session request shaping match the stable text-only path.
