## Summary

Current main still does not implement a scroll-aware TUI transcript viewport or TUI auto-scroll preference. ChatLog remains an append/prune container mounted directly in the TUI root; streaming updates mutate the assistant component and request renders without checking a user scroll anchor; history reloads rebuild the log; and settings/docs do not expose paging, wheel scroll, auto-follow, or jump-to-latest behavior. Related PR #51128 is open and unmerged, so this issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and route the fix through the TUI transcript path. The best near-term path is to review, rebase, and land #51128 if it covers the reported scroll anchor/auto-follow behavior on current main; otherwise implement the equivalent in ChatLog/TUI input handling so user-scrolled transcripts are preserved, auto-follow only happens when already at latest, and a clear jump-to-latest or paging behavior is available. Add focused TUI component/root tests and update docs/web/tui.md; keep Control UI/WebChat scope separate under #7648.
