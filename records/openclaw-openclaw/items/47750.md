## Summary

Keep open. Current main supports OpenAI Codex browser OAuth and a device-code pairing flow, plus an interactive/manual redirect paste fallback, but it does not expose the requested non-interactive callback URL or env-driven Codex OAuth setup path. The available non-interactive onboarding examples cover API-key/env-ref flows, not Codex OAuth callback completion.

## Best Possible Solution

Keep this feature request open and implement a supported OpenAI Codex OAuth callback completion path. A conservative direction is a generic provider-auth CLI seam such as `openclaw models auth login --provider openai-codex --callback-url <url>` or a documented env equivalent, wired through the OpenAI provider auth flow, with tests in the OpenAI provider and models auth CLI plus docs in `docs/providers/openai.md` and `docs/start/wizard-cli-automation.md`.
