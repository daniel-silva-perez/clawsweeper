## Summary

Keep this PR open. Current main has the related #41606 mitigation, but it still does not implement the PR's explicit chokidar depth hardening, and the memory side needs to be rebased to the current memory-core plugin layout.

## Best Possible Solution

Keep the PR open as a focused hardening candidate, or replace it with a rebased patch against src/agents/skills/refresh.ts, extensions/memory-core/src/memory/manager-sync-ops.ts, and likely extensions/memory-core/src/memory/qmd-manager.ts. Maintainers should choose the intended depth and symlink policy, preserve or document recursive extraPaths semantics, and add regression tests that assert the chokidar options.
