## Summary

Keep this PR open. Current main shipped a root-and-filter mitigation for the linked FD-exhaustion report, but it still has not implemented this PR's explicit chokidar `depth` hardening. The active memory watcher has moved into the bundled `memory-core` plugin, and QMD has a similar uncapped watcher surface that a complete fix should consider.

## Best Possible Solution

Keep this PR open as a focused hardening candidate, or replace it with a rebased patch against `src/agents/skills/refresh.ts` and `extensions/memory-core/src/memory/manager-sync-ops.ts`. Maintainers should choose the intended depth, symlink, and `extraPaths` contract, decide whether QMD needs the same policy, update public memory docs if recursive watch behavior changes, and add regression tests asserting the chokidar options.
