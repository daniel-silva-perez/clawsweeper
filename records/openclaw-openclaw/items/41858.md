## Summary

Keep open. Current main shipped a related root/filter mitigation for the FD-exhaustion report, but it still does not implement explicit chokidar depth hardening, the PR's memory hunk targets a removed core path, and the active QMD memory watcher is not covered.

## Best Possible Solution

Keep this PR open as a focused hardening candidate, or replace it with a rebased patch against src/agents/skills/refresh.ts, extensions/memory-core/src/memory/manager-sync-ops.ts, and extensions/memory-core/src/memory/qmd-manager.ts. Maintainers should choose the intended depth and symlink policy, preserve or explicitly document recursive extraPaths semantics, and add regression tests that assert the chokidar options.
