## Summary

Keep this PR open. Current main shipped a related mitigation for the linked FD-exhaustion bug by watching concrete roots with filters, but it still does not implement this PR's explicit chokidar depth hardening. The memory implementation also moved into the bundled memory-core plugin, so the PR is stale as-is but the hardening question remains actionable.

## Best Possible Solution

Keep this PR open as a focused hardening candidate, or replace it with a rebased patch that targets src/agents/skills/refresh.ts, extensions/memory-core/src/memory/manager-sync-ops.ts, and likely extensions/memory-core/src/memory/qmd-manager.ts. Maintainers should choose the intended depth and symlink contract, update docs if recursive memory extraPaths semantics change, and add regression tests that assert the chokidar options.
