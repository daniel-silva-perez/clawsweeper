## Summary

Keep this PR open. Current main shipped a related mitigation for the linked FD-exhaustion report, but it still does not implement the PR's explicit chokidar `depth` hardening, and the memory-side code moved from the PR's old `src/memory` target into the bundled `memory-core` plugin.

## Best Possible Solution

Keep this PR open as a focused hardening candidate, or replace it with a rebased patch that targets `src/agents/skills/refresh.ts`, `extensions/memory-core/src/memory/manager-sync-ops.ts`, and likely `extensions/memory-core/src/memory/qmd-manager.ts`. Maintainers should choose the intended depth and symlink contract, update memory docs if recursive extraPaths semantics change, and add regression tests that assert the chokidar options.
