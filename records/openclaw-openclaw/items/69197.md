## Summary

Keep this PR open. Current main and the latest tagged release still lack an active-poll guard or equivalent in the background exec notifyOnExit path, so the PR is not obsolete. The PR remains a plausible focused fix for the ghost completion notification race, but it needs maintainer review because the proposed boolean `pollActive` guard can be racy with overlapping same-session polls and the change touches agent exec/process runtime behavior.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The best path is maintainer review plus an amendment or explicit justification for the active-poll design, preferably using an in-flight poll counter/token unless same-session polls are proven serialized above the process-tool layer. The final patch should preserve normal `notifyOnExit` behavior, suppress only active-poll ghost completions, cover overlapping polls directly, and keep the diff confined to the agent exec/process runtime and tests.
