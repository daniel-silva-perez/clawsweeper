## Summary

Keep this PR open. Current main and latest release v2026.4.26 still lack an active-poll guard or equivalent in the background exec notifyOnExit path, so the PR is not obsolete; it remains a plausible focused fix but needs maintainer review because the boolean pollActive design can be racy with overlapping same-session polls.

## What This Changes

The PR adds ProcessSession.pollActive, suppresses maybeNotifyOnExit while process poll is active, sets/clears the flag around poll handling, marks exits as notified when poll observes completion, and adds regression tests for ghost notification suppression.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The best path is maintainer review plus either an in-flight poll counter/token or a clearly documented serialization invariant above the process-tool layer, with tests proving normal notifyOnExit still works and active process poll suppresses only the ghost completion event.
