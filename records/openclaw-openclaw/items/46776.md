## Summary

Keep open. Current main still does not implement the two requested Feishu behaviors: broadcast-only `@all` is intentionally not treated as a bot mention, and direct-message and group conversations use distinct session keys. The adjacent Feishu broadcast support is opt-in and observer-style, so it does not make every Feishu agent visibly receive and reply to an `@all` group message.

## Best Possible Solution

Keep this issue open and split it into two maintainer decisions: whether Feishu should add an explicit opt-in `@all` or multi-mention fan-out mode where every targeted bot/agent receives and replies once, and whether Feishu should support opt-in DM/group session linking or instead document that cross-surface continuity belongs in memory/session tools rather than shared transcripts. Coordinate the multi-bot delivery portion with #43563.

## Risks / Open Questions

- Closing now would discard a still-unimplemented product request: Feishu `@all` fan-out to every targeted agent/bot is intentionally not equivalent to a bot mention on current main.
- Naively sharing DM and group transcripts could leak private DM context into groups, or group context into direct messages, so this needs explicit product/security policy before implementation.
- Naively making every `@all`-targeted agent visibly reply could duplicate command execution and flood Feishu groups unless it is explicit, opt-in, and deduped.
