## Summary

Keep open. Current main still follows the reported behavior: Feishu broadcast-only `@all` is not a bot mention, broadcast observers are processed silently rather than visibly replying, and direct versus group conversations still use separate session keys. The related #43563 covers part of the multi-agent broadcast problem but not the direct/group context-continuity request.

## What This Changes

The issue reports that Feishu group `@all` or multi-agent messages do not reach or visibly wake every OpenClaw agent, and that the same agent does not share context between direct and group conversations.

## Best Possible Solution

Define an opt-in Feishu multi-agent fan-out model with loop control and visible-reply rules, and separately define an explicit continuity model for same-agent direct/group context, likely through session linking or memory/session tooling with clear privacy boundaries.

## Reproduction Assessment

Yes. There is a high-confidence source-level reproduction path: broadcast-only Feishu `@all` parses as not mentioning the bot, mention-gated group handling returns before dispatch, and direct/group routing builds distinct session keys.

## Solution Assessment

Unclear. The current implementation is deliberate isolation and does not satisfy the requested workflow, but the narrowest maintainable fix depends on maintainer choices around visible fan-out, loop control, and privacy.
