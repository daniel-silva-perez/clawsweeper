## Summary

Keep open: current main still does not implement the reported Feishu `@all` fan-out behavior or DM/group transcript continuity for the same agent; both remaining parts need maintainer product and safety decisions.

## Best Possible Solution

Keep this issue open and split it into two maintainer decisions: whether Feishu should add an explicit opt-in `@all` or multi-mention fan-out mode where every targeted bot/agent receives and replies once, and whether Feishu should support opt-in DM/group session linking or instead document that cross-surface continuity belongs in memory/session tools rather than shared transcripts. Coordinate the multi-bot delivery portion with #43563.
