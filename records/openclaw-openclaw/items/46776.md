## Summary

Keep open. Current main still preserves the behavior reported here: Feishu broadcast-only `@all` is explicitly not a bot mention, configured broadcast dispatch only gives the active agent a real Feishu reply path while observer agents are silent, and direct/group conversations continue to use separate routing/session keys. The related Feishu broadcast issue covers only the multi-bot delivery half, not the requested direct/group continuity behavior.

## What This Changes

The issue asks Feishu group `@all` messages to reliably reach every OpenClaw agent in a multi-agent group and asks the same agent to share conversational context between direct and group Feishu sessions.

## Best Possible Solution

Split this into two maintainer-owned decisions: define an opt-in Feishu multi-agent fan-out mode for `@all` or multi-mention workflows where each targeted agent can receive and, when intended, visibly reply once; separately document or implement an opt-in direct/group continuity model, likely through session-linking or memory/session tools rather than automatic transcript sharing. Coordinate the broadcast half with #43563.

## Reproduction Assessment

Yes. There is a high-confidence source-level reproduction path: broadcast-only Feishu `@all` parses as not mentioning the bot, mention-gated group handling returns before dispatch, and direct/group routing builds distinct session keys.

## Solution Assessment

Unclear. The current implementation is deliberate and documented, but it does not satisfy the requested workflow; the narrowest maintainable fix needs maintainer product judgment around visible fan-out semantics, privacy, and loop control.
