## Summary

Keep open. Current main still does not satisfy the reported workflow: Feishu broadcast-only @all is intentionally not a bot mention, mention-gated groups return before dispatch, broadcast observer agents process silently rather than visibly replying, and direct/group conversations remain separate sessions by design.

## What This Changes

The issue asks for Feishu multi-agent group @all messages to reach every OpenClaw agent and for the same agent to share conversation context between direct chats and group sessions.

## Best Possible Solution

Define an opt-in Feishu multi-agent fan-out model with loop control and visible-reply rules, and separately define an explicit same-agent direct/group continuity model through session linking or memory/session tooling with clear privacy boundaries.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: broadcast-only @all yields mentionedBot=false, mention-gated group handling returns before dispatch, and direct/group routing builds distinct session keys.

## Solution Assessment

Unclear. The current implementation is consistent with existing isolation rules but does not solve the requested workflow; the narrowest maintainable fix depends on maintainer choices around visible fan-out, loop prevention, and privacy.
