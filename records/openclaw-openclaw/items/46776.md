## Summary

Keep open. Current main has related Feishu work, but it does not implement the two behaviors requested here: broadcast-only Feishu @all is now explicitly not treated as a bot mention, and Feishu group sessions remain separate from direct-message sessions. The configured broadcast path is adjacent, but it is opt-in, runs only after normal reply eligibility, and intentionally has one active visible reply path.

## Best Possible Solution

Keep this issue open for maintainer product judgment. A good resolution should split the two asks: decide whether Feishu should support an explicit opt-in @all or multi-mention fan-out mode where every targeted bot/agent replies once, and decide whether Feishu should add an opt-in DM/group session-linking feature or instead document that cross-session continuity belongs in memory/session tools rather than shared transcripts. If the multi-bot portion is narrowed, coordinate with open Feishu cross-bot broadcast work such as #43563 rather than closing this issue prematurely.
