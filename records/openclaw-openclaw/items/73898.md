## Summary

Keep open. Current main generates TTS audio attachments when Auto-TTS is enabled, and the Control UI renders those attachments with a normal manual audio control. I found no browser-local Control UI setting, autoplay/playback state, or blocked-autoplay prompt for assistant TTS attachments. The request is narrow and actionable as a focused Control UI enhancement.

## Best Possible Solution

Add a browser-local Control UI setting, default off, for assistant TTS/audio attachment auto-play. Persist it with the existing `UiSettings`, expose a clear toggle in the Control UI, pass the setting into chat rendering, auto-play only newly arrived assistant audio attachments after the user's opt-in gesture, show a small enable-playback prompt if the browser blocks playback, and improve the audio card loading state without changing core TTS or channel behavior.
