## Summary

Keep this PR open. Current main still does not implement configurable fixed-delay retries for recurring cron jobs before the next natural scheduled run, and this PR remains an implementation candidate for open feature request #49740 rather than obsolete work.

## Best Possible Solution

Keep this PR open as an implementation candidate for #49740, or supersede it with a focused maintainer patch that adds per-job recurring retry policy end to end: cron types/state, create/update validation, scheduler behavior, Gateway schema and generated clients, CLI add/edit flags, docs, and regression tests. Resolve the remaining review questions before landing.
