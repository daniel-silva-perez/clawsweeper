## Summary

Keep this PR open. Current main still does not implement configurable fixed-delay retries for recurring cron jobs before the next natural scheduled run, and the PR is an active implementation candidate for open feature request #49740 with unresolved semantics and protocol/security-adjacent review points.

## Best Possible Solution

Keep this PR open as an implementation candidate for #49740, or supersede it with a focused maintainer-owned patch that adds per-job recurring retry policy end to end: cron types/state, create/update validation, scheduler behavior, gateway schema and generated clients, CLI add/edit flags, docs, and regression tests. Before landing, settle one-shot semantics, half-configured retry policies, retry-state clearing/recompute on edits, and cadence preservation after success or exhaustion.
