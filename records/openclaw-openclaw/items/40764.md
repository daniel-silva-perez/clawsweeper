## Summary

Keep this PR open. Current main still lacks an Android Talk language override, so the contribution addresses a real missing feature, but the proposed patch is not merge-ready because changing the local language setting can force the existing gateway refresh path into a false offline failure state and the branch needs normal Android maintainer review/rebase.

## What This Changes

The PR adds an Android language override preference and Settings dropdown, then threads the selected locale through recognizer intents, system TTS, gateway connect options, device info, and related tests.

## Best Possible Solution

Land an Android-owned language override that keeps auto as the default, accepts valid BCP 47 locale tags, applies the selected locale consistently through Android recognizer/TTS/gateway/device-info paths, avoids false offline reconnect failures, documents the Android Talk setting, and includes focused tests for locale propagation plus offline setting changes.

## Reproduction Assessment

Yes. The PR-branch regression is reproducible by inspection: selecting a new language calls setLanguageOverride(), which unconditionally calls refreshGatewayConnection(), and the current refresh path writes Failed: no cached gateway endpoint when no endpoint is cached.

## Solution Assessment

No. The feature direction is reasonable, but this patch should persist the desired locale and update pending connection options without forcing the failed refresh path while offline; it should reconnect only when an active or cached endpoint exists and be rebased onto current Android Talk code.
