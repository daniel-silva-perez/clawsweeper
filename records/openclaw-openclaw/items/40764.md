## Summary

Keep this PR open. Current main still does not implement an Android Talk Mode language override, so the contribution is not obsolete, but the patch is not merge-ready: it no longer applies cleanly to current main and the latest diff still has a user-visible offline refresh bug when changing the setting before any gateway endpoint is cached.

## What This Changes

The PR adds an Android Settings language override persisted in preferences and threads the selected locale into STT intents, system TTS, gateway connect metadata, and device.info locale reporting.

## Best Possible Solution

Adapt the feature onto the current Android voice and gateway runtime, preserve Android-owned settings/prefs/UI boundaries, avoid reconnect failure when no cached endpoint exists, and add focused Android coverage plus docs for the supported override behavior.

## Review Findings
