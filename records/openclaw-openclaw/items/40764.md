## Summary

Keep open. Current main still does not provide an Android Talk language override, so this PR is not obsolete, but the branch is not merge-ready because the setting update can surface a false gateway failure while offline and the Android Talk/TTS code has moved since the branch was cut.

## What This Changes

The PR adds an Android settings language override for Talk/voice mode, persists it as a preference, and threads the selected locale through recognizer intents, local TTS, gateway connect metadata, device.info, and related tests.

## Best Possible Solution

Rebase the Android-owned language override onto current main, preserve auto/system language as the default, apply the selected locale through the current recognizer, local TTS, gateway connect, and device.info paths, avoid false reconnect failures when no cached endpoint exists, and add focused Android tests plus Talk docs for the supported behavior.

Full review comments:

- **[P2] Avoid offline failure when updating language:** `apps/android/app/src/main/java/ai/openclaw/app/NodeRuntime.kt:640`
