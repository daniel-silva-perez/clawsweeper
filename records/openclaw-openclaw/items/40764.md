## Summary

Keep this PR open. Current main still lacks the Android Talk-mode language override, so the contribution is not obsolete, but the branch is stale against current Android voice/gateway code and has a concrete user-visible offline-status bug before it is merge-ready.

## What This Changes

This PR adds an Android language override setting and threads the selected locale through Android Settings, preferences, STT/TTS handling, gateway connect metadata, and device.info locale reporting.

## Best Possible Solution

Rebase and adapt the feature onto the current Android voice/gateway runtime, keep the override scoped to Android app-owned settings and locale metadata, avoid false failure states when no reconnect target exists, and add focused Android unit/Robolectric coverage plus Android maintainer review for the language list.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Avoid false failure when changing language offline:** `apps/android/app/src/main/java/ai/openclaw/app/NodeRuntime.kt:637`
  - body: `setLanguageOverride()` calls `refreshGatewayConnection()` unconditionally. On the current runtime, `refreshGatewayConnection()` sets the visible status to `Failed: no cached gateway endpoint` when no endpoint has ever been connected, so selecting a language before pairing or after clearing state can surface a false connection failure. Update cached connect options only when possible, or skip the reconnect path when there is no cached endpoint.
  - confidence: 0.87
