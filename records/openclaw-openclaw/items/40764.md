## Summary

Keep this PR open. Current main still lacks an Android Talk language override, and the PR addresses a real user gap with useful Pixel 6 verification from the author, but the branch is not merge-ready because changing the local language setting can incorrectly surface a gateway failure while offline.

## What This Changes

The PR adds a persisted Android language override setting and threads that locale through Settings, SecurePrefs, voice recognition, system TTS, gateway connect metadata, device info, and related tests.

## Best Possible Solution

Land an Android-owned language override that keeps auto as the default, accepts valid BCP 47 locale tags, applies the selected locale through Android recognizer/TTS/gateway/device-info paths, avoids false offline reconnect failures, documents the Android Talk setting, and includes focused tests for locale propagation and offline setting changes.

## Reproduction Assessment

Yes. The PR-branch regression is reproducible by inspection: selecting a new language calls setLanguageOverride(), which unconditionally calls refreshGatewayConnection(), and current refreshGatewayConnection() writes Failed: no cached gateway endpoint when connectedEndpoint is null.

## Solution Assessment

No. The feature direction is reasonable, but this patch should persist the desired locale and update reconnect options without forcing the failed refresh path while offline; it should reconnect only when an active or cached endpoint exists and be rebased onto current Android Talk code.
