## Summary

Keep this PR open. Current main still lacks an Android Talk language override, and the PR is a real implementation attempt, but it is not merge-ready because the branch is stale and the proposed setting change can surface a false gateway failure while offline.

## What This Changes

The PR adds an Android `languageOverride` preference and settings dropdown, then threads the selected locale through voice recognition, local TTS, gateway connect metadata, device info, and related tests.

## Best Possible Solution

Rebase the Android language override onto the current voice stack, keep `auto` as the default, apply the selected locale through current recognizer/TTS/gateway/device-info paths, avoid false offline reconnect errors, and document the Android Talk setting with focused locale propagation tests.

## Reproduction Assessment

Yes. The failure is reproducible by inspection: on the PR branch, selecting a different language calls `setLanguageOverride()` and then `refreshGatewayConnection()`; when no endpoint is cached, the current refresh path writes `Failed: no cached gateway endpoint`.

## Solution Assessment

No. The feature direction is reasonable, but the patch should not refresh through a user-visible failure path when offline; it should persist desired options and reconnect only when there is a cached or active endpoint, then be adapted to current main’s Talk/TTS implementation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Avoid offline failure when updating language:** `apps/android/app/src/main/java/ai/openclaw/app/NodeRuntime.kt:640`
  - body: `setLanguageOverride()` persists the setting and then calls `refreshGatewayConnection()` unconditionally. When the app has no cached endpoint, such as before first connect or after disconnect, that refresh path sets the visible status to `Failed: no cached gateway endpoint`, so changing a local language setting can look like a connection failure even though no reconnect is possible.
