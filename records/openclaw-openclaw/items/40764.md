## Summary

Keep this PR open. Current main still lacks an Android Talk Mode language override, so the feature is not obsolete, but the branch is not merge-ready because the latest PR diff can show a false gateway failure when changing the setting offline and it needs rebase/adaptation to current Android Talk/TTS code plus focused docs/tests.

## What This Changes

The PR adds an Android settings language override for Talk Mode, persists it, and threads the selected locale through recognizer intents, local TTS, gateway connect metadata, device.info, and Android tests.

## Best Possible Solution

Rebase the Android-owned language override onto current main, keep auto/system-language behavior as the default, avoid reconnect/status changes when no cached endpoint exists, apply the override through the current recognizer and TTS paths, and add focused Android tests plus Talk docs for the supported behavior.
