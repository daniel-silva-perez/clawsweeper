## Summary

Keep this PR open. Current main still lacks an Android language override for Talk/STT, so the contribution is not obsolete, but the branch is not merge-ready because the latest patch can show a false gateway failure when the user changes language before pairing or after disconnecting, and the patch is stale against current main.

## What This Changes

The PR adds an Android talk-mode language override setting persisted in preferences and threads the selected locale into STT intents, local system TTS, gateway connect metadata, device.info locale reporting, and device-handler tests.

## Best Possible Solution

Adapt the feature onto the current Android runtime, keep the language setting Android-owned, avoid reconnect/status changes when no cached endpoint exists, and add focused Android coverage plus Talk docs for the supported override behavior.
