## Summary

Keep this PR open. Current main at 891c7d9f1c1f5e0f52935fadfeacf695b0ef1eb8 still does not implement a TTL cache for provider usage summaries or the proposed `usage.cacheTtlMs` config, while status surfaces still call `loadProviderUsageSummary` directly. The PR remains a useful implementation candidate for the linked Anthropic usage 429 problem, but it needs rebase/rework against the current loader contract before merge.

## Best Possible Solution

Keep this PR open as a useful implementation candidate, but require a rebase or replacement patch against current main. The best fix should add the TTL cache in or near `src/infra/provider-usage.load.ts`, preserve the current config/env/workspace/auth contracts, wire any accepted `usage.cacheTtlMs` setting through types/schema/help/labels/generated metadata, and add regression tests for cache hit, TTL expiry, disabled cache, provider-list boundaries, workspace/config/env boundaries, explicit-auth bypass, and non-caching of transient failures.
