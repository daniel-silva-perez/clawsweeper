## Summary

Keep this PR open. Current main at 16906780 still does not implement a TTL cache for `loadProviderUsageSummary` or the proposed `usage.cacheTtlMs` config, and the active status/gateway usage paths still call the provider usage loader directly. The PR remains a plausible implementation candidate for the Anthropic usage 429 problem from #45332, but it needs rebase/rework against the current loader contract before merge rather than being closed as already implemented.

## Best Possible Solution

Keep this PR open as a useful implementation candidate, but require a rebase or replacement patch against current main. The best fix should add the TTL cache in or near `src/infra/provider-usage.load.ts`, preserve the current injected config/env/workspace/auth contracts, wire any accepted `usage.cacheTtlMs` setting through types/schema/help/labels/generated metadata, and add regression tests for cache hit, TTL expiry, disabled cache, provider-list boundaries, workspace/config/env boundaries, explicit-auth bypass, and non-caching of transient failures.

## Risks / Open Questions

- Merging the PR as-is may regress the current provider-usage loader contract because current main supports injected `config`, `env`, `workspaceDir`, and `skipPluginAuthWithoutCredentialSource` options that the PR patch predates.
- A provider-usage cache needs careful key boundaries for provider list, agent/workspace/config/env, and explicit auth, and it should avoid caching transient provider failures such as 429s or timeouts.
