## Summary

Keep this PR open. It addresses a real current-main reset-hook gap for non-slash Discord reset triggers, but the current PR head still has a blocking regression: explicit custom transcript filenames inside the default sessions directory can be rewritten to a generated session-id filename during gateway reset.

## What This Changes

This PR changes custom reset-trigger handling so non-slash Discord reset triggers emit reset hooks, adjusts gateway session transcript path rotation on reset, and adds bang-prefix reset-trigger tests.

## Best Possible Solution

Land a rebased fix that emits reset hooks for all matched custom resetTriggers and rotates generated/default transcripts cleanly while preserving explicitly pinned sessionFile values, including custom filenames inside the default sessions directory.

## Reproduction Assessment

Yes. Current main can be traced directly from resetTriggers matching in initSessionState to the slash-only fallback in get-reply.ts, and the PR regression is covered by the existing custom-owned-child-transcript reset test shape.

## Solution Assessment

No. The hook-emission change is the right narrow direction, but the session reset path should distinguish generated stale transcript paths from explicitly pinned custom transcript filenames instead of classifying only outside-directory paths as custom.
