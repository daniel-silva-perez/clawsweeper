## Summary

Keep this PR open, but do not merge it as-is. Current main still has the custom reset-trigger hook gap, and the PR addresses the right surface, but the latest branch can regress explicitly pinned transcript files that live in the default session-store directory.

## What This Changes

This PR changes custom reset-trigger handling so non-slash Discord reset triggers emit reset hooks, adjusts gateway session transcript path rotation on reset, and adds bang-prefix reset-trigger tests.

## Best Possible Solution

Land a rebased fix that emits reset hooks for matched custom resetTriggers and clears generated/default transcript history while preserving explicitly pinned `sessionFile` values, including custom filenames inside the default sessions directory, with coverage for Discord bang triggers and gateway reset transcript path cases.

## Reproduction Assessment

Yes. Current main can be traced directly: `initSessionState` can match configured `!new` or `!reset` triggers and set `resetTriggered`, while `get-reply.ts` still skips hook emission unless the normalized command is slash-prefixed `/new` or `/reset`. The PR's remaining defect is reproducible from the existing in-directory custom `sessionFile` reset test shape.

## Solution Assessment

No. The hook-emission change is the right narrow direction, but the session reset path should distinguish generated transcript paths from explicitly pinned `sessionFile` values instead of treating only outside-directory paths as custom.
