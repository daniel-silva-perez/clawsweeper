## Summary

Keep open: current main still has the custom reset-trigger hook gap this PR targets, but the PR is not mergeable as-is because the session reset path would still relocate explicitly pinned transcript files when the custom filename lives in the default sessions directory.

## What This Changes

This PR changes custom reset-trigger handling so non-slash Discord reset triggers emit reset hooks, adjusts gateway session transcript path rotation on reset, and adds bang-prefix reset-trigger tests.

## Best Possible Solution

Land a rebased fix that keeps custom reset triggers emitting reset hooks and clears generated/default transcript history without relocating explicitly pinned sessionFile values, with coverage for Discord bang triggers plus default, outside-directory, and in-directory custom transcript cases.

## Reproduction Assessment

Yes. The linked behavior is provable from current main because configured resetTriggers can set resetTriggered for !new or !reset while get-reply.ts still skips hook emission unless the normalized command is /new or /reset; the PR's remaining defect is covered by the existing in-directory custom sessionFile reset test shape.

## Solution Assessment

No. The hook-emission part is the right narrow direction, but the session-reset path should distinguish generated transcript paths from explicitly pinned sessionFile values instead of classifying custom paths only by directory.
