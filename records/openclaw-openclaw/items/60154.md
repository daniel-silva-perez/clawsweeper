## Summary

Keep this issue open. Current main still has no Feishu account-level inbound directory setting, and the active Feishu inbound media path saves every downloaded image/file/audio/video/post attachment through the shared `media/inbound` bucket. The broader related #19330 remains useful context for generic per-agent media isolation, but it does not fully supersede this Feishu multi-account/account-boundary request.

## Best Possible Solution

Keep this open and implement a scoped inbound media design. A narrow version would add validated `channels.feishu.accounts.<id>.inboundDir`, pass the resolved account storage target into Feishu media saving, and cover it with Feishu schema/docs/tests. A stronger generic version would add account/agent-scoped inbound buckets in the core media store and update reference resolution plus sandbox staging so bundled channels can isolate media without ad hoc per-channel paths.

## Risks / Open Questions

- A Feishu-only `inboundDir` could overlap with the broader per-agent media-isolation design in #19330; maintainers should decide whether to solve this as Feishu account-scoped storage, generic scoped inbound buckets, or both.
- Any implementation needs careful path validation, directory permissions, cleanup behavior, `media://inbound` reference semantics, and sandbox/mediaLocalRoots interaction so it does not trade one isolation issue for another.
