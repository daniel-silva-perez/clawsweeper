## Summary

Keep this issue open. Current main still has no Feishu account-level inbound directory or account-scoped inbound media bucket, and Feishu inbound downloads still save through the shared global `media/inbound` path. The broader #19330 is relevant, but it does not fully replace this Feishu multi-account/account-boundary request.

## Best Possible Solution

Keep this issue open and implement scoped inbound media storage. The narrow path is a validated `channels.feishu.accounts.<id>.inboundDir` wired into Feishu media saving with schema, docs, and tests. The stronger path is a generic account/agent-scoped inbound media store that bundled channels can use, with Feishu covered by account-scoped tests and #19330 used as the broader design tracker.

## Risks / Open Questions

- A Feishu-only `inboundDir` could conflict with the broader media-isolation design in #19330 if both are implemented independently.
- Any fix needs careful path validation, directory permissions, cleanup behavior, `media://inbound` reference semantics, and sandbox/mediaLocalRoots interaction so isolation is not bypassed.
