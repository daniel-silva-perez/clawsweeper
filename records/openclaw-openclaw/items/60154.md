## Summary

Keep open. Current main still saves Feishu inbound media for all accounts through the shared media bucket, and neither the strict Feishu account schema nor the public Feishu docs expose an account-level inbound directory. The broader #19330 tracker is relevant but does not fully supersede this Feishu account-boundary request.

## What This Changes

Add an optional `channels.feishu.accounts.<id>.inboundDir` so each Feishu account can save downloaded inbound media into an isolated directory instead of the shared inbound bucket.

## Best Possible Solution

Implement a scoped inbound media design that isolates Feishu accounts while staying compatible with the shared media pipeline. A conservative path is validated account-scoped inbound buckets under OpenClaw's media root, with Feishu schema/docs/tests updated; a broader path should align with #19330 so channels can share generic account/agent-scoped media storage semantics.

## Reproduction Assessment

Yes. A code-level reproduction is high-confidence: call the Feishu media path for two configured accounts and both flows pass `"inbound"` into `saveMediaBuffer`, while `channels.feishu.accounts.<id>.inboundDir` is not accepted by the strict schema.

## Solution Assessment

Unclear. The requested `inboundDir` would address the Feishu account isolation symptom, but arbitrary absolute inbound directories are security-sensitive; a validated account-scoped bucket design, possibly shared with the generic media-isolation work, is likely safer.
