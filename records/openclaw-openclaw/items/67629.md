## Summary

Keep open. Current main still has cron job identifiers and running-state data in the backend, Gateway protocol, and CLI list output, but the visible Control UI cron cards and agent-specific cron panel still do not render each job's stable ID or derive the displayed status from state.runningAtMs. The prior ClawSweeper review identified the same UI gap, and current source inspection shows it remains.

## Best Possible Solution

Implement a focused Control UI cron visibility enhancement: show each job's stable id on the main cron job cards and the agent cron panel, derive the displayed active status from job.state.runningAtMs before falling back to disabled/last-run/idle state, add or reuse English i18n labels, and add UI tests covering running, idle, disabled, and identifier display.
