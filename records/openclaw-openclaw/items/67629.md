## Summary

Keep open. Current main has the underlying cron identifier and running-state fields in the model, Gateway protocol, UI types, runtime, tests, and CLI list output, but the visible Control UI cron job cards still do not render each job ID or derive the displayed job status from `state.runningAtMs`. The agent-specific cron panel has the same gap via the shared presenter.

## Best Possible Solution

Implement a focused Control UI cron visibility enhancement: render each job's stable ID on the main cron card and agent cron panel, derive an active status from `job.state?.runningAtMs` before falling back to disabled/last-run state, add English i18n labels, and add UI tests covering running, idle, disabled, and identifier display.
