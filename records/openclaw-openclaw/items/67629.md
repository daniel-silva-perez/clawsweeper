## Summary

Keep open. Current main already carries stable cron job IDs and `state.runningAtMs` through the model, protocol, runtime, and CLI list output, but the visible Control UI cron job cards and agent cron panel still omit the job ID and derive their status from the last completed run rather than the active running marker.

## What This Changes

The issue requests that each timed cron task visibly show whether it is running and include an identifier.

## Best Possible Solution

Add a focused presentation-layer enhancement that shows each cron job's stable ID and treats `state.runningAtMs` as the active status before falling back to disabled, last-run, next-run, or idle state. The backend and protocol should stay unchanged unless a UI integration test exposes a missing event/refresh path.
