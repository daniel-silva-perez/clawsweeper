## Summary

Keep open: current main still classifies `browser.profiles.*` under the browser plugin's broad `browser` restart rule, so `config.patch` can still schedule direct SIGUSR1 in hot mode and the config watcher can still queue a restart in hybrid mode. The focused related PR is closed unmerged and the narrower browser-rule issue is closed, leaving this issue as the best open tracker.

## What This Changes

`config.patch` should avoid SIGUSR1/full gateway restart when changed paths such as `browser.profiles.*` can be handled by hot reload.

## Best Possible Solution

Add granular browser reload metadata or a specificity-aware reload planner so `browser.profiles.*` and `browser.defaultProfile` avoid full gateway restarts where the browser runtime can refresh them, while broader browser runtime settings still require restart. Cover both the `config.patch` direct restart path and the file-watcher reload path, with docs/changelog updates if user-facing behavior changes.

## Repair Work Prompt

Fix the remaining `config.patch`/config-reload restart bug for browser profile changes. Current `main` still registers the browser plugin as `restartPrefixes: ["browser"]`, so `browser.profiles.*` and likely `browser.defaultProfile` match a broad restart rule; in hot mode `config.patch` can still schedule direct SIGUSR1, and in hybrid mode the watcher can still queue a full restart. Implement a narrowly scoped solution using plugin-owned granular reload metadata or a backwards-compatible specificity-aware planner so browser profile/default-profile changes that the browser runtime can refresh do not require a full gateway restart. Preserve restart-required behavior for broader browser runtime/security/control-server settings. Do not broaden agent-facing config write permissions, do not weaken config validation/restart guards, and do not add core hardcoded bundled-plugin special cases if plugin metadata or generic rule specificity can express the behavior. Update/extend tests around reload planning, `config.patch` restart scheduling, watcher restart behavior, and browser profile hot refresh.
