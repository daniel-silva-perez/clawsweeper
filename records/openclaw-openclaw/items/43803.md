## Repair Work Prompt

Fix the remaining browser-profile config reload bug tracked by https://github.com/openclaw/openclaw/issues/43803. Current main still treats `browser.profiles.*` as restart-required because `extensions/browser/plugin-registration.ts` exports `restartPrefixes: ["browser"]` and `src/gateway/config-reload-plan.ts` matches the broad restart prefix before any narrower hot/no-op plugin prefixes. Implement a narrow fix so profile CRUD, `browser.profiles.*`, and likely `browser.defaultProfile` do not trigger `restartGateway` or direct `config.patch` SIGUSR1 when the browser runtime can refresh them. Preserve restart-required behavior for browser plugin activation/registration and security-sensitive/runtime settings such as `browser.enabled`, executable path policy, SSRF policy, and other settings that truly require a full restart. Do not rely on simply adding `hotPrefixes`/`noopPrefixes` under the existing broad `browser` restart prefix unless the planner becomes specificity-aware or the browser restart metadata is split so narrower paths win. Use closed PR #45624 only as source context; create a fresh narrow implementation rather than reviving the broad mixed work. Add targeted coverage in the gateway reload planner, `config.patch` direct-restart behavior, and browser profile refresh tests. Update `docs/tools/browser.md` if the restart boundary is user-visible. Add a changelog fix entry in the active version if repo policy expects one for this user-facing behavior, crediting the reporter and any source PR author used. Do not change unrelated config safety guards, protected config patch paths, package metadata, dependency pins, or release files beyond the normal changelog/docs/test surface.
