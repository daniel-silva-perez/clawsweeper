## Summary

Keep open. Current main still lacks the active-run input and avatar activity rendering needed for the pre-stream assistant waiting state, while the PR remains the focused implementation candidate and has no blocking correctness finding from this read-only pass.

## What This Changes

The PR adds an optional active-run flag to web chat item building, renders a small assistant avatar activity badge for pre-stream reading indicators, adds focused chat rendering tests/CSS, and adjusts two bundled plugin tests.

## Best Possible Solution

Keep this PR open, review the focused Control UI behavior, rebase if needed, and land it if the targeted chat tests plus the changed gate pass; the implementation should stay limited to web chat item building/rendering and avoid broader runtime or plugin API changes.
