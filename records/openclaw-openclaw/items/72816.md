## Summary

Keep open. Current main does not implement the PR’s intended pre-stream assistant activity state: chat item building only emits the reading indicator when `stream !== null` and empty, `renderChat` does not pass run activity into `buildChatItems`, and main has no `chat-avatar-activity` helper/styles. The PR is a focused UI change with tests, and the provided discussion shows the Greptile concern was addressed by limiting the badge to the pre-stream reading-indicator state.

## Best Possible Solution

Keep this PR open as the active implementation candidate. Maintainers should review the focused UI behavior and land it if CI and visual expectations are acceptable; otherwise request targeted changes to the pending assistant activity indicator rather than closing it as already implemented.
