## Summary

Keep open. Current main still does not implement this PR’s central behavior: pre-stream assistant activity is not passed into chat item building, `buildChatItems` only emits the reading indicator when `stream !== null`, and the renderer/styles still lack the proposed assistant avatar activity badge. The PR remains the active implementation candidate, with the Greptile concern addressed in the provided discussion by limiting the badge to the pre-stream reading-indicator state.

## Best Possible Solution

Keep this PR open as the active implementation candidate. Maintainers should rebase it onto current main, keep the focused Control UI behavior for active runs before stream text exists, verify the Greptile concern remains fixed so the avatar badge is pre-stream only, and then land it if targeted UI tests, CI, and a quick visual check pass.
