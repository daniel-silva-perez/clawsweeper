## Summary

Keep this PR open. Current main still ignores `filePath` and `path` in the Mattermost message-tool send path, so the PR is useful, but the proposed ternary is not safe to merge because blank `media` values prevent the new aliases from being considered.

## What This Changes

The PR updates Mattermost message-tool `handleAction` send handling to derive `mediaUrl` from `params.media`, then `params.filePath`, then `params.path`.

## Best Possible Solution

Land a corrected Mattermost alias fix: normalize each candidate independently, preserve the chosen precedence, add focused handleAction tests for both aliases and blank-media fallthrough, and keep the broader #67784 work coordinated with #52120 and #58439.

## Reproduction Assessment

Yes. A focused unit test can call Mattermost `handleAction` with `action: "send"`, `media: "   "`, and a valid `filePath`; the PR head would pass `mediaUrl: undefined`, while current main also ignores `filePath` and `path` entirely.

## Solution Assessment

No, not as currently patched. The direction is right, but the implementation should use normalized optional-string fallback semantics for each media candidate and add regression tests before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Skip blank media before falling back to aliases:** `extensions/mattermost/src/channel.ts:216-218`
