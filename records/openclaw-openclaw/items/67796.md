## Summary

Keep open. Current main still resolves Mattermost message-tool attachments only from `params.media`, while the shared send schema and neighboring channel handlers support `filePath` and `path`; the PR addresses a real Mattermost parity gap. The branch should not merge as-is because its fallback short-circuits on blank `media` and it lacks focused regression tests for the new aliases.

## What This Changes

The PR branch updates Mattermost message-tool `handleAction` to derive `mediaUrl` from `params.media`, then `params.filePath`, then `params.path` when sending a message.

## Best Possible Solution

Land a corrected version of the Mattermost alias fix: normalize each media candidate independently, preserve the intended precedence, add focused handleAction tests for both aliases and blank-media fallthrough, and keep the broader #67784 threadId/mediaLocalRoots work coordinated with #52120 and #58439.
