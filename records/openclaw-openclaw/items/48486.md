## Summary

Current main still leaves the reported web_fetch sanitizer gap open. The shared sanitizer strips exact hidden, transparent, and zero-size inline-style patterns, but it does not normalize CSS colors, compare explicit foreground/background colors, or reject tiny nonzero font sizes; because this is prompt-injection hardening with false-positive tradeoffs, it should stay open for maintainer/security policy review.

## What This Changes

The issue asks OpenClaw to harden the web_fetch visibility sanitizer against near-white text, tiny nonzero fonts, and matching foreground/background colors.

## Best Possible Solution

Add conservative CSS declaration handling in the shared sanitizer: normalize common explicit opaque colors, remove elements whose explicit foreground and background colors normalize to the same value, and apply a maintainer-approved tiny-font threshold. Cover both the basic and Readability paths with targeted tests, and add a changelog entry if maintainers classify this as user-facing security hardening.
