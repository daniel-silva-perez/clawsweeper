## Summary

Keep open. Current main still has the reported web_fetch visibility sanitizer gap: it strips exact hidden CSS and transparent/zero-alpha text, but it does not normalize colors, compare foreground/background colors, or reject tiny nonzero font sizes. The issue is stale-labeled but still actionable and security-sensitive.

## What This Changes

The issue asks to harden web_fetch HTML visibility sanitization against near-white or same-color text and tiny nonzero font sizes that can hide prompt-injection content from human review.

## Best Possible Solution

Add conservative CSS declaration handling in the shared sanitizer: normalize common explicit opaque color values, drop elements whose explicit foreground and background colors normalize to the same value, and apply a maintainer-approved tiny-font threshold. Cover the basic and Readability paths with targeted tests for same-color text, selected near-white/default-white cases, and tiny nonzero font sizes; add a changelog entry if maintainers treat this as user-facing security hardening.
