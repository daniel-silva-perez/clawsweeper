## Summary

Keep open. Current main still leaves the reported `web_fetch` visibility sanitizer gap in place, and the issue is security-sensitive enough to need maintainer policy judgment rather than stale cleanup.

## What This Changes

The issue requests `web_fetch` sanitizer hardening for near-white or same-color CSS text and tiny nonzero font sizes that are difficult for humans to see but still reach extracted agent context.

## Best Possible Solution

Add conservative CSS declaration handling in the shared sanitizer: normalize common explicit opaque color forms, remove elements whose explicit foreground and background colors normalize to the same value, and apply a maintainer-approved tiny-font policy with coverage for both basic and Readability extraction paths.
