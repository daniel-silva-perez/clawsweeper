## Summary

Current main still appears to have the reported `web_fetch` visibility sanitizer gap. The shared sanitizer removes exact hidden/transparent/zero-size patterns, but it does not normalize opaque CSS colors, compare explicit foreground/background colors, or reject tiny nonzero font sizes; because this is prompt-injection hardening with false-positive tradeoffs, it should remain open for maintainer/security policy review.

## What This Changes

The issue requests broader `web_fetch` HTML sanitization for near-invisible CSS, including near-white text, tiny nonzero font sizes, and matching foreground/background colors.

## Best Possible Solution

Add conservative CSS declaration handling in the shared sanitizer: normalize common explicit opaque colors, remove elements whose explicit foreground and background colors normalize to the same value, and apply a maintainer-approved tiny-font threshold. Cover both basic and Readability extraction paths with targeted tests, and add a changelog entry if maintainers classify this as user-facing security hardening.
