## Summary

Keep open. Current main still does not cover the issue's central near-invisible CSS cases: `color:white`/near-white colors, tiny non-zero font sizes, or same foreground/background color. The shared sanitizer is used by both the Readability plugin path and the basic HTML fallback, so this remains on the active `web_fetch` extraction surface rather than being obsolete or out of repo scope.

## Best Possible Solution

Keep this issue open and implement the sanitizer hardening in `src/agents/tools/web-fetch-visibility.ts`: add a small CSS color normalization helper for named/hex/rgb(a)/hsl(a) values, detect same foreground/background colors, handle near-white-on-default-white text if that policy is desired, and add conservative tiny-font detection. Add focused tests in `src/agents/tools/web-fetch-visibility.test.ts` for the examples from the report and, if useful, one Readability-path test proving the shared sanitizer protects normal `web_fetch` extraction.
