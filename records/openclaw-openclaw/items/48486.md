## Summary

Keep open. Current main still has the reported web_fetch visibility sanitizer gap: it strips exact hidden CSS patterns such as `font-size:0`, transparent text, and zero-alpha colors, but it does not normalize colors, compare explicit foreground/background colors, or remove tiny nonzero font sizes. The issue discussion also points to normalized same foreground/background color handling as the right fix direction.

## Best Possible Solution

Keep this open and land a focused sanitizer hardening change in `src/agents/tools/web-fetch-visibility.ts`: parse inline style declarations conservatively, normalize common opaque CSS color forms, drop text when explicit foreground and background colors normalize to the same value, and add a carefully chosen tiny-font threshold. Add targeted tests in `src/agents/tools/web-fetch-visibility.test.ts` for same-fg-bg, near-white/default-white cases if maintainers choose that policy, and tiny nonzero fonts; include a changelog entry if maintainers treat this as user-facing security hardening.
