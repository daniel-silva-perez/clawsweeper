## Summary

Keep open. Current main still has the reported `web_fetch` visibility sanitizer gap: the shared sanitizer removes exact hidden/transparent/zero-size patterns, but it does not normalize opaque CSS colors, compare explicit foreground/background colors, or reject tiny nonzero font sizes. Because this is prompt-injection hardening with real false-positive tradeoffs, it should stay with maintainer/security review rather than be auto-closed or queued blindly.

## What This Changes

The issue requests hardening `web_fetch` HTML sanitization so near-white text, tiny nonzero fonts, and same foreground/background colors are not passed into extracted content.

## Best Possible Solution

Add conservative CSS declaration handling in the shared sanitizer: normalize common explicit opaque colors, remove elements whose explicit foreground and background colors normalize to the same value, and apply a maintainer-approved tiny-font threshold. Cover both basic and Readability extraction paths with targeted tests, and add a changelog entry if maintainers classify this as user-facing security hardening.
