## Summary

Keep open. Current main still has the reported web_fetch visibility sanitizer gap: the shared sanitizer strips exact hidden/transparent/zero-size patterns, but it does not normalize colors, compare explicit foreground/background colors, or reject tiny nonzero font sizes. Because this is prompt-injection hardening with policy choices around false positives, it should stay with maintainer/security review rather than be swept closed.

## What This Changes

Harden web_fetch HTML sanitization to remove near-invisible CSS content, including near-white or same-color text and tiny nonzero fonts.

## Best Possible Solution

Add conservative CSS declaration handling in the shared sanitizer: normalize common explicit opaque colors, remove elements whose explicit foreground and background colors normalize to the same value, and apply a maintainer-approved tiny-font threshold. Cover both the basic and Readability paths with targeted tests; add a changelog entry if maintainers classify this as user-facing security hardening.
