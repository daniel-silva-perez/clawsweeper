## Summary

Keep open. Current main still has the reported web_fetch visibility sanitizer gap: it removes exact hidden CSS patterns, transparent or zero-alpha text, and known hidden elements, but it does not normalize colors, compare explicit foreground/background colors, or drop tiny nonzero font sizes. The existing discussion correctly points toward a policy-sensitive same-foreground/background and tiny-font hardening pass rather than closure.

## What This Changes

The issue requests hardening the web_fetch HTML visibility sanitizer against near-invisible CSS such as near-white text, tiny nonzero fonts, and matching foreground/background colors.

## Best Possible Solution

Keep this open and land a focused sanitizer hardening change in src/agents/tools/web-fetch-visibility.ts: parse inline style declarations conservatively, normalize common opaque CSS color forms, remove content when explicit foreground and background colors normalize to the same value, and add a carefully chosen tiny-font threshold. Add targeted tests in src/agents/tools/web-fetch-visibility.test.ts for same-fg-bg, selected near-white/default-white policy cases, and tiny nonzero fonts; include a changelog entry if maintainers treat this as user-facing security hardening.
