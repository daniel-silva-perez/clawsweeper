## Summary

Keep open. Current main still has the reported web_fetch visibility sanitizer gap: inline styles are checked with fixed hidden-value regexes, but there is no color normalization, foreground/background comparison, or tiny nonzero font threshold. Ryce's issue comment usefully frames the likely direction as normalized same-color handling plus a careful threshold decision, which makes this security-sensitive hardening rather than stale cleanup.

## What This Changes

The issue requests hardening the web_fetch HTML visibility sanitizer so near-white text, tiny nonzero font sizes, and same foreground/background color content are removed before extraction.

## Best Possible Solution

Add conservative CSS declaration handling in the shared sanitizer: normalize common explicit opaque colors, remove elements whose explicit foreground and background colors normalize to the same value, and apply a maintainer-approved tiny-font threshold. Cover both the basic and Readability extraction paths with targeted tests, and add a changelog entry if maintainers classify this as user-facing security hardening.
