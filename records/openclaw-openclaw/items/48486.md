## Summary

Keep open. Current main still has the reported `web_fetch` visibility sanitizer gap: the shared sanitizer removes exact hidden, transparent, and zero-size patterns, but it does not normalize opaque CSS colors, compare explicit foreground/background colors, or reject tiny nonzero font sizes. Because this is prompt-injection hardening with false-positive tradeoffs, it needs maintainer/security review rather than cleanup closure.

## What This Changes

The issue requests broader `web_fetch` HTML sanitization for near-invisible CSS patterns such as near-white text, tiny nonzero fonts, and identical foreground/background colors.

## Best Possible Solution

Add conservative CSS declaration handling in the shared sanitizer: normalize common explicit opaque colors, remove elements whose explicit foreground and background colors normalize to the same value, and apply a maintainer-approved tiny-font threshold. Cover both the basic and Readability paths with targeted tests, and add a changelog entry if maintainers classify this as user-facing security hardening.
