## Summary

Keep open. Current main still cannot express an independent memorySearch fallback endpoint or model: fallback is a string-only config field, the resolver produces one shared remote block, and both fallback creation paths reuse that shared configuration. The related implementation PR was closed unmerged and superseded by this narrower issue, so there is still concrete repo work to do.

## What This Changes

Add support for configuring memorySearch fallback embedding providers with independent provider, model, and remote settings.

## Best Possible Solution

Add a backward-compatible fallback object form for memorySearch and thread fallback-specific provider, model, remote, headers, input-type, dimensionality, and local options through validation, resolved config, provider creation, docs/help/generated schema, and focused regression tests while preserving the current string shorthand.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/43760 with a narrow, backward-compatible memorySearch fallback config change. Preserve the existing `fallback: "openai"` string shorthand, and add an object form such as `fallback: { provider, model?, remote?, inputType?, queryInputType?, documentInputType?, outputDimensionality?, local? }` so the fallback embedding provider can use a different endpoint, API key, headers, model, and provider-specific embedding options than the primary. Thread this through `MemorySearchSchema`, public config types, generated schema/help/labels/hints/redaction where needed, `resolveMemorySearchConfig`, `ResolvedMemorySearchConfig`, `resolveMemoryPrimaryProviderRequest`, `resolveMemoryFallbackProviderRequest`, immediate provider-create fallback, and runtime fallback activation. Add focused regression tests proving a Gemini primary can use default Google credentials/base URL while an OpenAI-compatible fallback uses its own SiliconFlow-style `remote.baseUrl`, `remote.apiKey`, and model. Do not add startup migrations, do not change main model fallback behavior, and do not move provider-owned auth/defaults into core. Update the public memory config docs and add a user-facing changelog entry with `Thanks @jackiedepp`.
