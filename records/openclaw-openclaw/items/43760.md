## Summary

Keep open: current main still has the reported limitation. `memorySearch.fallback` remains a provider string, the resolver still builds one shared top-level `remote` block, and both fallback creation paths reuse that shared configuration; the related implementation PR was closed unmerged rather than shipped.

## What This Changes

Support a memory search fallback configuration with independent provider, model, and remote endpoint settings for fallback embedding providers.

## Best Possible Solution

Add a backward-compatible fallback object form to the memory search config contract and thread fallback-specific provider, model, remote, headers, input-type, dimensionality, and local options through validation, resolved config, immediate provider creation, runtime fallback activation, docs/help/generated schema, and focused regression tests.
