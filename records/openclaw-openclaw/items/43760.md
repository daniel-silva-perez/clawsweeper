## Summary

Keep open: current main still has the reported config limitation. `memorySearch.fallback` is still a provider string, the resolver still builds one shared `remote` block, and both fallback creation paths reuse that shared provider request. The prior implementation PR was closed unmerged and explicitly left this issue as the canonical narrow tracker.

## What This Changes

The issue requests separate fallback-specific `memorySearch` provider, model, and remote endpoint configuration so a Gemini primary can use an OpenAI-compatible fallback endpoint without sharing the same `remote` block.

## Best Possible Solution

Add a backward-compatible fallback object form to the memory search config contract and thread fallback-specific provider, model, remote, headers, input-type, and dimensionality through schema validation, resolved config, immediate provider creation, runtime fallback activation, docs/help/generated schema, and focused regression tests.

## Repair Work Prompt

Implement a narrow fix for this issue on current main: support a backward-compatible object form for `agents.*.memorySearch.fallback`, such as `{ provider, model?, remote?, inputType?, queryInputType?, documentInputType?, outputDimensionality? }`, while preserving the existing string form exactly. Thread the normalized fallback config through `MemorySearchSchema`, `MemorySearchConfig`, `ResolvedMemorySearchConfig`, `resolveMemorySearchConfig`, immediate `createEmbeddingProvider` fallback creation, and runtime `resolveMemoryFallbackProviderRequest` / fallback activation. The fallback object should allow a Gemini primary to use env/default Google auth and an OpenAI-compatible fallback with its own `remote.baseUrl` and `remote.apiKey`. Preserve custom provider-id lookup through `models.providers.<id>.api`. Update schema labels/help/generated config schema, docs/reference/memory-config.md, and changelog if user-facing change policy requires it. Add focused regression tests for string compatibility, fallback object validation, fallback remote/model override, Gemini primary plus OpenAI-compatible fallback request construction, and runtime fallback activation. Do not carry forward unrelated SSRF, compaction, or binary file changes from PR #44878.
