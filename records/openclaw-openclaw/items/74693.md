## Summary

Current main already fixes the reported packaged fallback crash. The read-only channel setup fallback now detects bundled `dist/` importers, resolves the plugin loader from the installed OpenClaw package's `dist/plugins/*` entries, and has regression coverage for the exact path escape described in the report; the fix is on main only and is not included in `v2026.4.27`.

## What This Changes

The issue reports that `openclaw status` crashes in the read-only channel setup fallback when a packaged build resolves the plugin runtime loader from `dist/read-only-*.js` to a path outside the installed `openclaw` package.

## Best Possible Solution

Keep the main-branch packaged loader fix and let the next OpenClaw release ship it; the Tencent plugin metadata follow-up can proceed separately without keeping this core crash report open.
