## Summary

Close as implemented on main. Current main fixes the packaged read-only channel setup fallback loader so packaged `dist/read-only-*.js` chunks resolve the plugin loader inside the OpenClaw package, and the fix is covered by a regression test and changelog entry. The fix is not in the latest published `v2026.4.27` release yet, so it is current-main-only until the next release ships.

## What This Changes

`openclaw status` should not crash when read-only channel setup fallback loads an external channel plugin from a packaged OpenClaw install.

## Best Possible Solution

Keep the current-main packaged-loader fix, ship it in the next OpenClaw release, and leave the linked Tencent WeChat plugin `channelConfigs` metadata work as a separate external plugin follow-up rather than relying on that plugin change to prevent a core CLI crash.

## Reproduction Assessment

Yes. The issue has multiple independent 2026.4.27 reports with the same stack, and the v2026.4.27 source shows the exact `../../plugins/loader.js` candidate that escapes a packaged `dist/read-only-*.js` location; current main's regression test models that packaged importer path.

## Solution Assessment

Yes. The current-main fix is the narrow maintainable core solution: make the fallback resolve a stable packaged dist entry and keep source-mode candidates for development, instead of depending on third-party plugin metadata or hashed chunk discovery.
