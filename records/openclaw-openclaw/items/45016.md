## Summary

Keep open. Current main has provider/model-driven replay sanitization and a helper for last-assistant thinking retention, but it still does not expose the requested operator-facing `last-only` / `strip` / `keep` history-retention setting; choosing that override would affect deliberate Claude 4-family prompt-cache and replay behavior.

## What This Changes

The issue requests a configurable thinking-history retention policy that can keep only the latest assistant thinking blocks, strip all thinking blocks, or preserve current behavior.

## Best Possible Solution

Add a documented provider-aware transcript-history retention override that keeps current provider-safe defaults unless explicitly configured, supports `last-only`, `strip`, and `keep`, and covers Anthropic, Bedrock, custom Anthropic-compatible providers, and prompt-cache behavior with focused tests.
