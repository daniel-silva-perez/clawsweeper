## Summary

Keep open. Current main still handles thinking replay through provider/model-derived transcript policy, but it does not provide the requested operator-facing `last-only` / `strip` / `keep` history-retention setting; choosing that default and config scope would change intentional Claude prompt-cache and replay behavior.

## What This Changes

The issue requests a configurable thinking-history retention policy that can keep only the latest assistant thinking blocks, strip all thinking blocks, or preserve current behavior.

## Best Possible Solution

Add a documented provider-aware transcript-history retention setting that keeps current provider-safe defaults unless explicitly overridden, supports `last-only`, `strip`, and `keep`, and covers Anthropic, Bedrock, and custom-provider replay paths with focused tests.
