## Summary

Keep open. Current main has provider-policy replay sanitizers and a last-assistant-preserving `dropThinkingBlocks()` helper, but it still does not provide the requested user-facing `last-only` / `strip` / `keep` history policy. The remaining decision is product/config policy because current Claude replay logic intentionally preserves thinking blocks for newer Claude-family models rather than applying broad token-saving stripping.

## What This Changes

Add a configurable transcript-history policy that strips assistant `thinking` and `redacted_thinking` blocks from replay history, especially older assistant turns, to reduce token usage.

## Best Possible Solution

Keep this open until maintainers decide whether OpenClaw should expose an explicit transcript-history retention policy. If accepted, implement it as a documented provider-aware config surface that preserves current safe defaults, allows deliberate operator overrides, and adds replay-policy tests for `last-only`, `strip`, and `keep`.
