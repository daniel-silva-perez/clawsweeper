## Summary

Keep open. Current main has provider-policy replay sanitizers and a last-assistant-preserving `dropThinkingBlocks()` helper, but it still does not provide the requested operator-facing `last-only` / `strip` / `keep` history policy. The remaining decision is product/configuration scope because current Claude-family policy deliberately preserves thinking blocks for newer models to protect prompt-cache behavior, while older/problematic replay cases are handled by provider policy.

## What This Changes

Request a configurable thinking-block retention policy for assistant history replay so operators can keep, strip, or keep only the latest assistant thinking blocks to reduce token usage.

## Best Possible Solution

If maintainers accept the feature, add a documented provider-aware transcript-history retention setting that preserves current safe defaults, permits explicit operator overrides, and covers `last-only`, `strip`, and `keep` with replay-policy tests across Anthropic/Bedrock/custom-provider paths.
