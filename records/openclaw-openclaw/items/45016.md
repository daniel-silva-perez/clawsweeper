## Summary

Keep open. Current main has provider/model-driven replay sanitization and a last-assistant thinking-block helper, but it still does not provide the requested operator-facing `last-only` / `strip` / `keep` retention setting; changing defaults also needs maintainer policy because newer Claude replay intentionally preserves thinking blocks for prompt-cache behavior.

## What This Changes

Request a configurable transcript-history policy that strips assistant `thinking` and `redacted_thinking` blocks in `last-only`, `strip`, or `keep` modes to reduce replay token usage.

## Best Possible Solution

Add a documented provider-aware transcript-history retention override that preserves current provider-safe defaults unless explicitly configured, supports `last-only`, `strip`, and `keep`, and includes focused coverage for Anthropic, Bedrock, custom Anthropic-compatible transports, and prompt-cache behavior.

## Reproduction Assessment

Not applicable as a runtime bug reproduction: this is a feature/config request. Current-main verification is high-confidence because source, config schema, docs, and tests show the requested retention override is absent while provider-gated stripping already exists.

## Solution Assessment

Unclear. An explicit opt-in override is plausible, but changing the default to `last-only` is not yet proven best because current Claude policy deliberately preserves newer thinking blocks for prompt-cache semantics.
