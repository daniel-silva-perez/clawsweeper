## Summary

Keep open. Current main has an internal last-assistant-preserving sanitizer and narrower provider replay fixes, but it does not implement the requested user-facing history policy or broad token-savings behavior. For newer Claude models, current policy intentionally preserves prior thinking blocks for prompt-cache compatibility, so this remains a product/config decision rather than an already-implemented cleanup.

## What This Changes

Add a configurable policy for assistant `thinking`/`redacted_thinking` history, with modes such as preserving, stripping all, or stripping all except the latest assistant message.

## Best Possible Solution

Keep this open until maintainers decide whether OpenClaw should expose an explicit agent or gateway transcript-history policy. If accepted, the design should preserve provider-safe defaults, allow deliberate operator overrides, document prompt-cache and tool-continuation tradeoffs, and add targeted replay-policy/config tests.
