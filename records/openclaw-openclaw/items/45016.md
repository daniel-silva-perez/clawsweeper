## Summary

Keep open. Current main has provider-gated last-assistant-only thinking stripping, but there is no public `thinkingHistory`-style config, and modern Claude policy deliberately preserves historical thinking blocks because prior stripping caused prompt-cache regressions.

## What This Changes

Add a configurable policy for handling assistant `thinking` and `redacted_thinking` blocks in replay history, with modes such as last-only, strip, and keep.

## Best Possible Solution

Keep the issue open and design an explicit, documented replay-thinking-history policy that defaults to provider-safe current behavior, allows opt-in stripping only where provider/cache semantics permit it, and updates config schema, docs, replay-policy tests, and context-pruning estimates together.

## Reproduction Assessment

Not applicable as a feature request. Current-main inspection does verify the missing capability: there is no public config knob for the requested modes, and replay stripping is controlled only by provider/model transcript policy.

## Solution Assessment

Unclear. The requested last-only default is not universally safe because current code intentionally preserves thinking for modern Claude cache behavior, so the best path is a maintainer-approved opt-in policy rather than a blanket default change.
