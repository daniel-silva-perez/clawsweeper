## Summary

Keep open. Current main has provider/model-driven replay sanitization and a helper that can strip thinking blocks from non-latest assistant turns, but it still lacks the requested operator-facing `last-only` / `strip` / `keep` retention setting; the remaining decision is product/API policy because newer Claude replay intentionally preserves thinking blocks for prompt-cache behavior.

## What This Changes

Request a configurable gateway or per-agent setting that controls assistant thinking-block retention before API replay to reduce token usage and related replay failures.

## Best Possible Solution

Add a documented, provider-aware transcript-history retention override that preserves current provider-safe defaults unless explicitly configured, supports `last-only`, `strip`, and `keep`, and includes focused coverage for Anthropic, Bedrock, custom Anthropic-compatible transports, and prompt-cache behavior.

## Reproduction Assessment

Not applicable as a runtime bug reproduction: this is a feature/config request. The current-main verification is high-confidence because exact source/docs searches and replay-policy implementation checks show the requested retention override is not present.

## Solution Assessment

Unclear. An explicit opt-in override is plausible, but changing the default to `last-only` is not yet proven best because current Claude policy deliberately preserves newer thinking blocks for prompt-cache semantics.
