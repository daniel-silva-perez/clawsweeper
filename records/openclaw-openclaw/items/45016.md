## Summary

Keep open. Current main already has provider-policy replay sanitizers and a last-assistant-preserving helper, but it still does not expose the requested operator policy for `last-only`, `strip`, or `keep`. The remaining work is a product/config decision because current Claude-family replay policy deliberately preserves thinking blocks for newer models instead of applying broad token-saving stripping.

## What This Changes

Request a configurable transcript-history policy that strips `thinking` and `redacted_thinking` blocks from assistant history before API replay to reduce token usage and replay errors.

## Best Possible Solution

Keep this open until maintainers decide whether OpenClaw should expose an explicit transcript-history retention policy. If accepted, implement it as a documented provider-aware config surface that preserves current safe defaults, supports deliberate operator overrides, and adds replay-policy tests for `last-only`, `strip`, and `keep`.
