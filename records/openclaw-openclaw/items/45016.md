## Summary

Keep open. Current main has provider/model-driven replay sanitization and a helper that strips thinking blocks from non-latest assistant turns, but it still lacks the requested operator-facing `last-only` / `strip` / `keep` retention setting. The remaining decision is not just plumbing because modern Claude replay intentionally preserves thinking blocks for prompt-cache behavior.

## What This Changes

Request adding a config-controlled policy for whether assistant `thinking` and `redacted_thinking` blocks are kept, stripped, or retained only on the latest assistant turn before provider replay.

## Best Possible Solution

Add a documented, provider-aware transcript-history retention override that preserves current provider-safe defaults unless explicitly configured, supports `last-only`, `strip`, and `keep`, and includes focused coverage for Anthropic, Bedrock, custom Anthropic-compatible transports, and prompt-cache behavior.

## Reproduction Assessment

Not applicable as a runtime bug reproduction: this is a feature/config request. The current-main check is high-confidence because exact source/docs searches and the replay-policy implementation show the requested retention override is not present.

## Solution Assessment

Unclear. An explicit override is plausible, but making `last-only` the default is not yet proven best because current provider policy deliberately preserves modern Claude thinking blocks for prompt-cache semantics; the safer direction is an explicit opt-in override that leaves existing defaults intact.
