## Summary

Keep open. Current main partially covers the mechanics by stripping non-latest thinking blocks when a provider replay policy opts into `dropThinkingBlocks`, but the requested user-visible `thinkingHistory`/last-only/strip/keep configuration does not exist, and newer Claude models are intentionally exempted from dropping prior thinking blocks for prompt-cache stability.

## What This Changes

The issue requests a configurable policy for stripping `thinking` and `redacted_thinking` blocks from assistant history, with a `last-only` default proposed for token savings.

## Best Possible Solution

If maintainers want this feature, add a documented opt-in retention policy that layers on top of provider replay policy instead of silently changing provider-safe defaults. It should cover config schema/help/docs plus tests for legacy Claude, Claude 4.5+, Bedrock/Anthropic-compatible replay, and normal tool-continuation paths.

## Reproduction Assessment

Yes for the implementation gap: source inspection shows no `thinkingHistory` key, and both replay-history and run-loop stripping are gated by `policy.dropThinkingBlocks`. A live provider token test is not necessary to show that the requested config surface and default behavior are not present.

## Solution Assessment

Unclear. The requested `last-only` default is not obviously the best global default because current main deliberately preserves prior thinking for newer Claude models to protect prompt-cache prefix matching; a maintainer-reviewed opt-in or per-model policy is the safer direction.
