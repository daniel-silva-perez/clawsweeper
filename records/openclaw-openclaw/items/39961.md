## Summary

Keep this PR open. Current main still lacks a streaming text-repetition guard or textRepetitionGuard config surface, while the PR remains unique unmerged work for a real core agent streaming gap. It still needs maintainer engineering review because the guard would sit in the assistant streaming/final-flush lifecycle and automated review raised unresolved abort/flush and false-positive concerns.

## Best Possible Solution

Keep this PR open for maintainer engineering review. The best path is to land a robust core streaming text-repetition guard, or an explicit maintainer decision not to add one, with config/schema/docs/tests covering false positives, abort persistence, queued events, chunked and non-chunked replies, final flush, lifecycle fallback flushing, and session-history sanitization or rollback behavior where needed.
