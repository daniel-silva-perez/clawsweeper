## Summary

Keep open. Current main has partial internal `argPattern` plumbing, but it does not implement the requested cross-platform argument-level allowlist for the reported macOS/Linux-style `node script args` use case. The matcher explicitly ignores `argPattern` outside Windows, and the public docs, CLI, and UI still present manual allowlists as executable path or bare command-name patterns only.

## Best Possible Solution

Keep this issue open and implement a cross-platform, documented argument-level allowlist contract for host exec approvals. Maintainers should decide whether to expose the existing `argPattern` name or add the proposed `argsPattern`, define glob vs regex semantics, enforce it on macOS/Linux as well as Windows, and update CLI/UI/docs/tests around the interpreter-script case such as `/opt/homebrew/bin/node /Users/me/.openclaw/mute_user.mjs * *`.

## Risks / Open Questions

- The existing internal `argPattern` field is Windows-only and regex-based, so expanding it cross-platform needs a deliberate compatibility and security design rather than treating the current field as already shipped for this use case.
- Changing allowlist semantics affects host exec security; implementation should cover shell tokenization, interpreter/script paths, durable allow-always entries, CLI/UI editing, docs, and regression tests before this can close.
