## Summary

Keep this PR open. Current main contains a narrower Microsoft Teams proactive-send fix for tenant/AAD forwarding, but it does not implement this PR’s central change: preserving the stored conversation payload through the adapter/send path. The PR also appears security-contained to the Teams plugin runtime/tests, so the useful next step is normal maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase or apply a scoped version that passes the stored Teams conversation payload through `continueConversation`/`createSendContext`, adds the adapter and send-helper regression tests described in the PR, and addresses the bot’s minor dead-code review note before merge. If maintainers decide the existing tenant/AAD forwarding is the intended boundary, close with that explicit product rationale rather than as already implemented.
