## Summary

Keep this PR open. Current main still consumes the model-supplied exec `security` argument and combines it with configured policy via `minSecurity`, so the reported `tools.exec.security="full"` plus model `security:"allowlist"` denial remains reproducible from source. The PR is a plausible fix direction, but its current head still needs cleanup before landing.

## Best Possible Solution

Keep this PR open and review it as a focused exec policy fix. The best landing shape is a minimal change in `src/agents/bash-tools.exec.ts` that makes configured/session exec security authoritative for model tool calls, preserves the explicit operator-granted elevated-full override, removes unused `minSecurity`/`normalizeExecSecurity` usage from that path and the unused `maxSecurity` helper/tests from the PR, adds regressions for configured `full` with model `allowlist`, configured `allowlist` with model `deny`, and configured `deny` with model `full`, and updates or clarifies the exec tool schema/docs if model-supplied `security` becomes a compatibility no-op.
