## Summary

Current main still has the reported zero cache pricing for both bundled Doubao/Volcengine and BytePlus model catalogs. Those defaults flow into the registered provider models, and cost estimation multiplies cached-token usage by `cacheRead` and `cacheWrite`, so zeros still undercount cached-token cost. PR #55913 remains open and unmerged with GitHub closing syntax for this issue, so this should stay open until that PR lands or is replaced.

## Best Possible Solution

Keep this issue open and review, update, or replace PR #55913 as the implementation path. The fix should update `extensions/volcengine/models.ts` and `extensions/byteplus/models.ts` with vendor-backed nonzero `cacheRead` and `cacheWrite` pricing in USD per 1M tokens, and add focused regression tests for both bundled provider plugins.
