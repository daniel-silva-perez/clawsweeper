## Summary

Current main still carries zero cache-read and cache-write pricing for the bundled Volcengine/Doubao and BytePlus catalogs, and the runtime cost path preserves and uses those fields directly. Because open PR #55913 already references this issue as its implementation target, the issue should remain open until that PR lands, is updated, or is replaced.

## What This Changes

Fill vendor-backed nonzero context-caching cacheRead/cacheWrite prices in the bundled Doubao/Volcengine and BytePlus model catalogs so cached-token usage is costed correctly.

## Best Possible Solution

Review, retarget, or replace PR #55913 as the implementation path. The merged fix should update the bundled Volcengine and BytePlus plugin model catalogs with vendor-backed USD-per-million-token cache prices and add focused regression coverage so cache pricing cannot silently return to zero.
