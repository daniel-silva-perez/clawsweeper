## Summary

Keep open. Current main still has zero cache-read and cache-write prices in the bundled Volcengine/Doubao and BytePlus catalogs, and the runtime cost estimator multiplies cached-token counts by those exact fields. Open PR #55913 already references this issue as its implementation target, so a separate cleanup close or repair job would be premature.

## What This Changes

The issue asks for vendor-backed nonzero cacheRead and cacheWrite pricing in the bundled Doubao/Volcengine and BytePlus model catalogs so cached-token usage is not reported as free.

## Best Possible Solution

Review, update, or replace PR #55913 as the implementation path. The merged fix should update the bundled Volcengine and BytePlus manifests plus compatibility constants with current USD-per-million-token cache prices, and add focused regression coverage that prevents these provider catalog values from returning to zero.
