## Summary

Keep open. Current `origin/main` still lacks Chat Completions client-tool request parsing, `clientTools` handoff, and OpenAI-compatible `tool_calls` output, while PR #66278 is an open unmerged implementation candidate that explicitly closes this issue.

## What This Changes

The issue requests client-provided function tools on `/v1/chat/completions`, including `tools`, `tool_choice`, assistant `tool_calls`, and `role:"tool"` follow-up handling for clients such as Home Assistant Extended OpenAI Conversation.

## Best Possible Solution

Review and land PR #66278 or an equivalent focused gateway change that reuses the existing `clientTools` runtime seam, adds Chat Completions request parsing and tool-call response mapping, and covers non-stream, streaming, `tool_choice`, and tool-result follow-up behavior in `src/gateway/openai-http.test.ts`.
