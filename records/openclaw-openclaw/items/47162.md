## Summary

Keep open. Current main still has the reported Python/shell scanner blind spot, and this PR is useful source work, but the proposed patch is not merge-ready because the new coverage still misses extensionless shell entrypoints and variable-based Python command execution patterns in a security-sensitive scanner.

## What This Changes

The PR expands `src/security/skill-scanner.ts` and its tests to scan `.py`, `.sh`, and `.bash` files with new Python and shell detection rules plus line-rule extension guards.

## Best Possible Solution

Revise this PR, or replace it with a focused reviewed scanner-hardening patch, so Python and shell coverage includes extension-based files plus extensionless shebang or declared entrypoints, and Python command-exec matching catches non-literal `os.*` payloads without reintroducing noisy subprocess false positives.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: current main excludes `.py`, `.sh`, and `.bash`, while the PR head still skips extensionless files through `path.extname()` and does not match `cmd = ...; os.system(cmd)`.

## Solution Assessment

No. The direction is right, but this is not the best complete fix until it handles extensionless shell entrypoints and non-literal Python command payloads with regression tests for both bypasses and false-positive boundaries.

## Review Findings

Full review comments:

- **[P2] Scan extensionless shell entrypoints:** `src/security/skill-scanner.ts:50`
  - body: Adding `.sh` and `.bash` only expands extension-based coverage. The scanner still uses `path.extname()` in the directory walk and forced include path handling, so common shebang entrypoints like `bin/install` can contain `curl | bash` and remain unscanned.
  - confidence: 0.93

  - body: The new `os.system`/`os.popen`/`os.exec*` arm only matches quoted first arguments. A script can build `cmd` separately and call `os.system(cmd)`, which is the same dangerous behavior but avoids the new critical finding.
  - confidence: 0.88
