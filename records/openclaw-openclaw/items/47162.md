## Summary

Keep open. Current main still scans only JavaScript/TypeScript-family files, while this PR usefully expands the scanner toward Python and shell coverage but leaves concrete bypasses in the new security surface. The author addressed earlier review feedback, but the latest branch still skips extensionless shell entrypoints and misses variable-based Python command execution.

## What This Changes

The PR adds `.py`, `.sh`, and `.bash` to the skill scanner, adds Python and shell detection rules, and extends `src/security/skill-scanner.test.ts` with positive and negative coverage for those rules.

## Best Possible Solution

Revise this PR, or replace it with a focused scanner-hardening patch, so Python and shell coverage includes extension-based files plus extensionless shebang or declared entrypoints, and Python command-exec matching catches non-literal `os.*` payloads without reintroducing noisy subprocess false positives.

## Reproduction Assessment

Yes. Source and diff inspection give a high-confidence reproduction path: current main excludes `.py`, `.sh`, and `.bash`, while the PR head still skips extensionless files through `path.extname()` and does not match `cmd = ...; os.system(cmd)`.

## Solution Assessment

No. The direction is right, but this is not the best complete fix until it handles extensionless shell entrypoints and non-literal Python command payloads with regression tests for both bypasses and false-positive boundaries.
