## Summary

Keep this PR open. The prior review discussion correctly identified the headline `web_fetch` change as useful, but current `main` still rethrows no-fallback network failures unchanged, and the branch remains bundled with unrelated Feishu, Telegram, config/model, tool-result, and package override changes that need maintainer rework before landing.

## Best Possible Solution

Keep this PR open for maintainer rework or replacement. The best path is a narrow `web_fetch`-only patch that wraps direct network-level failures only after provider fallback misses, extracts `error.cause.message` or `error.cause.code`, preserves the original error as `cause`, includes safe requested-URL context, and adds a regression test for the no-fallback network-error case. Split or drop the unrelated Feishu, Telegram, config/schema, model/table, image-helper, tool-result, and package override changes.
