## Summary

Keep open. Current main still leaves the reported parent/child workspace split undetected: the legacy workspace detector returns an empty list, the status warning depends on that list, and the relevant docs do not explain this Docker-to-host migration layout. The latest reporter comment confirms the desired fix is narrow doctor/status detection plus docs, not implicit secret migration.

## What This Changes

Request doctor/status detection and documentation for Docker-to-host migrations that leave an active `workspace-local/local` workspace split from parent-held `workspace-local` operational files.

## Best Possible Solution

Add a narrow doctor/docs fix: detect an active `workspace-local/local` child when the parent contains relevant operational markers, report the active workspace and parent-held files clearly, and document canonical Docker-to-host workspace/config expectations without changing startup migration or SecretRef semantics.
