## Summary

Keep open. Current main still leaves the reported parent/child workspace layout silent: the detector always returns no legacy directories, the doctor/status note only fires from that list, and the docs do not explain this Docker-to-host split. The latest reporter comment narrows the requested fix to read-only doctor/status detection plus documentation, not automatic secret migration.

## What This Changes

Request detection, warning, optional repair guidance, and documentation for Docker-to-host migrations where an active `workspace-local/local` child and parent-held operational files remain split.

## Best Possible Solution

Add read-only doctor/status detection that recognizes an active `workspace-local/local` child with relevant parent-held operational markers, reports both the active workspace and parent files clearly, and update the workspace/Docker docs with canonical post-migration expectations while preserving intentional per-agent child workspaces.
