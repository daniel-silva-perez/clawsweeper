## Summary

Keep open. Current main still does not detect or document the reported Docker-to-host `workspace-local` / `workspace-local/local` split: the doctor workspace detector returns an empty legacy-dir list unconditionally, the status note only warns when that list is non-empty, and the relevant docs only describe generic extra workspaces such as `~/openclaw` or Docker mount persistence.

## Best Possible Solution

Keep this open and implement a narrow doctor/docs fix: detect an active `workspace-local/local` child when the parent `workspace-local` contains operational markers such as `secrets/` or workspace bootstrap/state files, report both paths clearly, and document the canonical Docker-to-host workspace and secret/config locations. Avoid startup-time migration and avoid moving/copying/deleting secrets without explicit maintainer-approved `doctor --fix` behavior.
