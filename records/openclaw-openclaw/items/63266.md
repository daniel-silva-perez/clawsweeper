## Summary

Keep open. Current main does not yet detect, warn about, repair, or document the reported `workspace-local` / `workspace-local/local` split. The existing doctor workspace hook has the right shape for this issue, but `detectLegacyWorkspaceDirs()` currently returns an empty list unconditionally, so doctor/status cannot surface the migrated split layout.

## Best Possible Solution

Keep this open and implement a narrow doctor/docs fix: detect the `workspace-local` parent plus active `workspace-local/local` child layout when parent-level operational files such as `secrets/`, workspace bootstrap files, or plugin/workspace state exist; report the active workspace and parent-held files clearly; document Docker-to-host canonical locations. Any automatic move/copy of secrets should be opt-in through doctor repair only after maintainer review.
