## Summary

Keep open. Current main still does not detect the reported Docker-to-host `workspace-local` / `workspace-local/local` split: the doctor detector always returns an empty legacy list, the status note only warns when that list is non-empty, and the workspace/Docker docs still cover only generic extra workspaces and Docker persistence rather than this parent/child migration layout. The reporter also confirmed the narrow doctor/docs direction in the provided discussion.

## What This Changes

Request doctor/status detection and documentation for Docker-to-host migrations that leave active `workspace-local/local` files split from parent-held `workspace-local` operational files.

## Best Possible Solution

Keep this issue open and implement a narrow doctor/docs fix: detect an active `workspace-local/local` child when the parent `workspace-local` contains relevant operational markers, show a clear doctor/status warning that names the active workspace and parent-held files, and document canonical Docker-to-host workspace/config expectations. Avoid startup-time migration and avoid moving/copying/deleting secrets except through a separately reviewed opt-in repair path.
