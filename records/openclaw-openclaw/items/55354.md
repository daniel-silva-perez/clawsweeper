## Summary

Keep open. Current main still has the reported TUI status-line display-name bug: a direct main session with a sticky `channel: "telegram"` can still get a group-style `telegram:g-agent-main-main` display name before the row builder falls back to the TUI/WebChat origin label.

## Best Possible Solution

Keep this issue open. The narrow fix belongs in gateway session row/search display-name derivation: only call `buildGroupDisplayName()` when the entry or parsed key is actually a group/channel session, then add a regression test for a direct Telegram-backed main/TUI session that keeps the `openclaw-tui` origin label instead of a `telegram:g-...` label.
