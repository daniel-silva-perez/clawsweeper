## Summary

Keep open. Current main still has the reported TUI footer display-name bug path: a direct session entry with a sticky `channel` can still be formatted through the group display-name builder before the row falls back to the TUI/WebChat origin label.

## Best Possible Solution

Keep this issue open. The best fix is to guard `buildGroupDisplayName()` in both `buildGatewaySessionRow()` and `resolveSessionListSearchDisplayName()` so it only runs for parsed group/channel keys or entries whose `chatType` is `group` or `channel`, then add a regression test for a direct main/TUI session with `channel: "telegram"`, `origin.label: "openclaw-tui"`, and `chatType: "direct"` that preserves the TUI label instead of producing `telegram:g-...`.
