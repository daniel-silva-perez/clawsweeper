## Summary

Keep this PR open. Current main still has the two behaviors the PR targets: ACP dispatch skips URL-only image attachments before the media cache can fetch them, and Synology Chat invalid-token throttling still locks one attempt after the configured budget rather than exactly at exhaustion. The PR remains a focused implementation candidate, with the review-thread zero-budget concern addressed on the PR branch rather than already shipped on main.

## Best Possible Solution

Keep the PR open, rebase or refresh it against current main, and review it as a focused bug-fix candidate. The best path is to land both fixes with the targeted regression tests from the PR branch, or split ACP attachment forwarding and Synology invalid-token throttling if maintainers want separate review tracks.
