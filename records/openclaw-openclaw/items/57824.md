## Summary

Keep this PR open. Current main still has both targeted behaviors, and the PR remains a focused implementation candidate with the earlier zero-budget review concern addressed on the branch rather than shipped on main.

## What This Changes

The PR removes ACP's local-path-only image attachment gate so URL-only images can reach the media cache, and changes Synology Chat invalid-token throttling to lock at the configured threshold with regression tests including zero-rate-limit coverage.

## Best Possible Solution

Refresh or rebase the PR if needed, then land the two narrow fixes with the focused ACP and Synology regression tests, or split them only if maintainers want separate review tracks for core ACP dispatch and the Synology Chat plugin.

## Reproduction Assessment

Yes. Current main can be reproduced by a static path check: MediaUrl-only image attachments are normalized but skipped by the ACP path guard, and a Synology limit of 5 reaches 429 only after the strict > comparison sees a sixth invalid attempt.

## Solution Assessment

Yes. The PR takes the narrow maintainable route: remove only the redundant ACP path gate while preserving URL stripping for local paths, and change the Synology boundary to >= with a minimum invalid-token budget clamp for zero-rate configurations.
