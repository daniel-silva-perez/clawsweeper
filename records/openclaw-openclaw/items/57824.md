## Summary

Keep this PR open. Current main at 230f7122dd5cd8a37a55e671254bd9be8278f6e8 still has both behaviors the PR targets: ACP URL-only image attachments are skipped before the media cache can fetch them, and Synology Chat invalid-token throttling still locks one attempt after the configured budget is exhausted. The PR remains a focused implementation candidate, and the review-thread zero-budget concern was addressed on the PR branch rather than already shipped on main.

## Best Possible Solution

Keep the PR open and review it as a focused bug-fix candidate against current main. The best path is to rebase or refresh it, land both fixes with the targeted regression tests, or split ACP attachment forwarding and Synology invalid-token throttling if maintainers want separate review tracks.
