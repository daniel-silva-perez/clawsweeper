## Summary

Keep this PR open. Current main still only has unified sendPayload coverage for Discord and Slack; Google Chat, Mattermost, Microsoft Teams, and Synology Chat still expose sendText/sendMedia without a sendPayload implementation, and the attached-results plugin builder does not synthesize one. The PR remains relevant, but it needs maintainer review and a rebase because the Discord/Slack portions now overlap current main and prior review comments raised payload semantics that should be reconciled with current helper patterns.

## Best Possible Solution

Keep this PR open for maintainer follow-up. Rebase it on current main, drop or reconcile the already-shipped Discord and Slack portions, implement sendPayload for Google Chat, Mattermost, Microsoft Teams, and Synology Chat using the current shared payload helpers where they fit, and add focused adapter/contract tests for text-only, media-only, multi-media, channelData/presentation payloads, reply/thread propagation, chunking, and no-op behavior.
