## Summary

Keep this PR open. Current main has newer sendPayload coverage for Discord and Slack, but Google Chat, Mattermost, MS Teams, and Synology Chat still do not expose sendPayload. Since sendPayload remains optional and delivery only takes the unified payload path when an adapter provides it, the PR’s core adapter-parity work is not fully implemented on main.

## Best Possible Solution

Keep this PR open for maintainer action, but require a rebase and narrowing before merge. Drop or reconcile the already-shipped Discord and Slack pieces, implement sendPayload for Google Chat, Mattermost, MS Teams, and Synology Chat using current shared payload helpers, and add outbound payload contract tests covering text-only, media-only, mixed media, empty/channelData-only behavior, reply-to fanout, chunking, and fallback delivery behavior.
