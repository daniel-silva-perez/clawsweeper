## Summary

Keep this PR open. Current main partially supersedes it with Discord and Slack sendPayload implementations, but Google Chat, Mattermost, Microsoft Teams, and Synology Chat still do not expose the unified outbound payload entry point this PR was meant to add.

## Best Possible Solution

Keep this PR open for maintainer follow-up. Rebase it on current main, drop or reconcile the already-shipped Discord and Slack work, implement sendPayload for Google Chat, Mattermost, Microsoft Teams, and Synology Chat using the current plugin SDK payload helpers where appropriate, and add focused adapter/contract tests for text-only, media-only, multi-media, channelData/presentation or interactive payloads, reply-to fanout, chunking, and fallback behavior.
