## Summary

Keep this PR open. Current main at 891c7d9f1c1f5e0f52935fadfeacf695b0ef1eb8 still does not fully implement the PR’s intended batch-b sendPayload coverage: Discord and Slack have shipped current-main sendPayload implementations, but Google Chat, Mattermost, Microsoft Teams, and Synology Chat still lack sendPayload on their outbound adapters. The PR is partly obsolete and needs rebase/narrowing, but it is not safe to close as implemented.

## Best Possible Solution

Keep this PR open for maintainer follow-up. The best path is to rebase on current main, drop or reconcile the already-shipped Discord and Slack pieces, implement current-pattern sendPayload support for Google Chat, Mattermost, Microsoft Teams, and Synology Chat through the existing plugin SDK payload helpers where appropriate, and add focused adapter/contract tests for text-only, media-only, multi-media, channelData-only, reply-to fanout, chunking/formatting, and fallback delivery behavior.
