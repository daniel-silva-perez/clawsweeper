## Summary

Keep open. Current main still has only global memory-core dreaming configuration: the strict agent schema and AgentConfig type do not expose `agents.list[].dreaming`, the workspace resolver still includes every configured agent, and the runtime still processes the full resolved workspace list. The same author also has open PR #67420 with closing syntax for this issue, so the maintainer path is to review, land, or close that PR rather than close this issue early.

## What This Changes

Request per-agent memory-core dreaming controls, including per-agent enable/disable and optional custom schedules, so operators can avoid running every agent workspace during the same global sweep.

## Best Possible Solution

Keep this issue open while #67420 is reviewed. The desired end state is a documented, schema-backed `agents.list[].dreaming` contract at least for `enabled`, wired through config types/generated schema and `resolveMemoryDreamingWorkspaces`, with resolver/runtime/docs tests. If per-agent schedules stay in scope, memory-core should explicitly model multiple scoped cron jobs or the schedule request should be split into a narrower follow-up.
