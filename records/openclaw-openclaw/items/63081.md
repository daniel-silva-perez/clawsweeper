## Summary

Current main already implements this PR's intended fix. Pairing state JSON is now read as unknown, coerced through a shared record guard that rejects arrays and other non-object values, and both device and node pairing load paths use it. Regression tests cover array-shaped device and node pairing state files, and the changelog records the fix for #63035 in 2026.4.27 with credit to @sar618.

## Best Possible Solution

Close this PR as implemented on main. Keep the shipped shared coercePairingStateRecord implementation and the device/node regression tests; no further action is needed on this PR unless maintainers want to revisit the broader state-file validation policy separately.
