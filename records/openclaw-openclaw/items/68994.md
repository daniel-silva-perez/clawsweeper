## Summary

Keep open. Current main still has the old session resolver behavior: structural cross-root session paths are returned as stored instead of rebased, and the current regression test explicitly expects the stale stored path when the state root changes. Docker setup has newer permission hardening, but it does not include this PR's JSON path normalization. The PR is still potentially useful, but it needs maintainer review because the Docker setup rewrite is security-sensitive and the PR discussion includes an unresolved Node fallback concern.

## Best Possible Solution

Keep this PR open for maintainer review or author revision. The best path is to land a focused core session-path rebase with regression coverage, then either drop the Docker JSON rewrite or make it narrow, idempotent, and tested against Docker sandbox host-path requirements.
