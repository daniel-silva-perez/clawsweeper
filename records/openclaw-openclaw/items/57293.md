## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve inline environment override precedence:** `src/daemon/systemd.ts:463`
  - body: This filters inline service environment values solely by key presence in `.env`. The install environment is built with `.env` first and later config/auth/service values overriding it, so a stale `.env` key can cause the fresh inline value to be removed and the service to start with the stale `EnvironmentFile` value instead. Keep differing inline values, or generate the env file only from values that match the install plan.
  - confidence: 0.91
- **[P2] Use the same state-dir source for filtering:** `src/daemon/systemd.ts:454`
  - body: The PR derives `dotEnvPath` with `resolveGatewayStateDir(env)`, but the service environment being filtered is built from config state-dir resolution, which has legacy fallback behavior. In legacy/default migration cases these paths can diverge, so `.env` values can still be written inline or the wrong file can be referenced. Use the same resolved state dir/input as the install-plan environment.
  - confidence: 0.82
- **[P2] Do not reject quoted EnvironmentFile paths:** `src/daemon/systemd-unit.ts:54-56`
  - body: After calling `systemdEscapeArg`, paths with spaces can be rendered safely, but this check throws before escaping. A supported `OPENCLAW_STATE_DIR` containing whitespace would make systemd service staging fail even though the renderer already has a quoting path for unit values.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The PR touches service secret handling and can unintentionally make stale `.env` secrets override fresher inline config/auth values.

Concerns:

- **[medium] Stale `.env` secrets can override fresher service values:** `src/daemon/systemd.ts:463`
  - body: Filtering by key only removes newer inline values whenever a matching key exists in `.env`; because systemd loads `EnvironmentFile=` before later inline values would normally override it, this can start the gateway with stale credentials or tokens.
  - confidence: 0.91
- **[medium] State-dir mismatch can leave secrets inline:** `src/daemon/systemd.ts:454`
  - body: The PR's `.env` discovery path does not mirror the config state-dir resolver used to build durable service env vars, so legacy/default state-dir installs can miss the real `.env` source and continue writing managed secret values inline.
  - confidence: 0.82
