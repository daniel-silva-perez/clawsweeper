## Summary

Current main still has the reported token-gate failure, and the provided GitHub context shows open PR #50694 explicitly closes this issue but has not merged. The issue should stay open until maintainers land that PR or a replacement fix.

## What This Changes

The issue reports that `reserveTokensFloor` values equal to the context window can make the memory-flush threshold non-positive, preventing token-triggered memory flush from running.

## Best Possible Solution

Keep this issue open and review or land PR #50694, or replace it with a maintainer-preferred fix that keeps the memory-flush threshold positive when `reserveTokensFloor` or `softThresholdTokens` are misconfigured relative to `contextWindowTokens`, while adding regression coverage for equal and greater-than-context reserve values.
