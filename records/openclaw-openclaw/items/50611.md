## Summary

Keep open. Current main still disables the token-based memory-flush and preflight-compaction gates when the computed threshold is non-positive, and the issue is already paired with open unmerged PR #50694, so this should remain open until that PR or a replacement lands.

## What This Changes

The issue asks OpenClaw to handle `reserveTokensFloor >= contextWindow` so memory flush and compaction still trigger instead of silently filling the full context window.

## Best Possible Solution

Keep this issue open until PR #50694, or a maintainer-preferred replacement, guarantees a positive token-based flush/preflight threshold for misconfigured reserve values and adds focused regression coverage for reserve values equal to and greater than the context window.
