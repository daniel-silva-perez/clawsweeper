## Summary

Keep open. Current main still treats a non-positive token threshold as a disabled memory-flush/preflight-compaction gate, and the issue is already paired with open PR #50694, which explicitly targets this bug and has not merged.

## What This Changes

The issue requests preventing memory flush and preflight compaction from being disabled when `reserveTokensFloor` is equal to or greater than the model context window.

## Best Possible Solution

Keep this issue open until PR #50694, or a maintainer-preferred replacement, lands a positive token threshold for both memory flush and preflight compaction with regression coverage for reserve values equal to and greater than the context window.
