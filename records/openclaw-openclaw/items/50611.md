## Summary

Keep open. Current main still disables the token-based memory-flush and preflight-compaction paths when `contextWindowTokens - reserveTokensFloor - softThresholdTokens` is non-positive, and an open unmerged PR already targets the same bug.

## What This Changes

The issue reports that memory flush and compaction do not trigger correctly when `reserveTokensFloor` is configured equal to the model context window.

## Best Possible Solution

Review and land PR #50694, or replace it with a maintainer-preferred fix that guarantees a positive token-based memory-flush and preflight-compaction threshold when reserve settings exceed the model context budget, with regression coverage for equal and greater-than-context reserve values.
