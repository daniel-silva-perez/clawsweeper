## Summary

Keep open. Current main still disables the token-based memory-flush and preflight-compaction gates when the computed threshold is non-positive, and an open unmerged PR explicitly targets the same issue, so this should remain open until that PR or a replacement lands.

## What This Changes

The issue asks OpenClaw to keep memory flushing active when `reserveTokensFloor` is equal to the model context window instead of letting that configuration disable compaction triggering.

## Best Possible Solution

Keep this issue open and land PR #50694 or a maintainer-preferred replacement that guarantees a positive token-based flush/preflight threshold for misconfigured reserve values, with focused regression coverage for reserve values equal to and greater than the context window.
