Current main still has the reported token-gate failure, and open PR #50694 explicitly closes this issue but has not merged. The issue should stay open until maintainers land that PR or a replacement fix.

## What This Changes

Fix memory flush behavior when `reserveTokensFloor` is equal to or greater than the context window so compaction/flush gating still triggers safely.

## Best Possible Solution

Keep this issue open and review/land PR #50694, or replace it with a maintainer-preferred fix that clamps or validates `reserveTokensFloor` and `softThresholdTokens` relative to `contextWindowTokens`, preserves sane defaults, and adds regression coverage for reserve values equal to or greater than the context window.
