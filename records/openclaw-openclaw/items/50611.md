Keep open. Current main still has the reported token-gate failure, and an open unmerged PR explicitly targets this issue, so this should remain open until maintainers land that PR or a replacement fix.

## What This Changes

The issue asks OpenClaw to keep memory-flush triggering functional when `reserveTokensFloor` is configured equal to or greater than the resolved context window.

## Best Possible Solution

Keep this issue open and review or land PR #50694, or replace it with a maintainer-preferred fix that guarantees a positive memory-flush threshold when reserve and soft-threshold settings are misconfigured relative to the context window, with regression coverage for equal and greater-than-context reserve values.
