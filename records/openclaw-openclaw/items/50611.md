## Summary

Current main still has the reported edge case: the memory-flush gate treats a non-positive threshold as no gate state, so `reserveTokensFloor >= contextWindowTokens` prevents the token-based flush path from firing. There is also an open implementation PR in the main repository that uses closing syntax for this issue, so the issue should remain open until that PR is reviewed, landed, or closed.

## Best Possible Solution

Review PR #50694 as the canonical implementation candidate. If it is acceptable, land it or an equivalent patch that clamps/validates the memory-flush threshold and adds regression tests for `reserveTokensFloor >= contextWindowTokens`; if it is not acceptable, close that PR with rationale and keep this issue open until the edge case is fixed.
