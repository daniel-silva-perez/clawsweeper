## Summary

Keep open. Current main still disables token-based memory flush and preflight compaction when the computed threshold is non-positive, and an open unmerged PR explicitly targets this issue, so the issue should remain open until maintainers land that fix or a replacement.

## What This Changes

The issue reports that memory flush never triggers when `reserveTokensFloor` is configured equal to the context window, because the computed flush threshold becomes zero or negative.

## Best Possible Solution

Keep this issue open and review or land PR #50694, or replace it with a maintainer-preferred fix that guarantees a positive token-based flush/preflight threshold when reserve and soft-threshold settings are misconfigured relative to the context window, with focused regression coverage for equal and greater-than-context reserve values.
