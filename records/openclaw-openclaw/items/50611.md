## Summary

Keep open. Current main still disables the token-based memory-flush and preflight-compaction paths when the computed threshold is non-positive, while an open unmerged PR explicitly targets this same issue, so this should remain open until that PR or a replacement lands.

## What This Changes

The issue asks for memory flush and preflight compaction to handle `reserveTokensFloor` values equal to or greater than the context window instead of silently disabling token-based compaction.

## Best Possible Solution

Keep this issue open and land PR #50694 or a maintainer-preferred replacement that guarantees a positive token-based flush and preflight-compaction threshold when reserve settings are misconfigured relative to the context window, with focused regression coverage for equal and greater-than-context reserve values.
