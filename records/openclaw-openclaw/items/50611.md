## Summary

Current main still has the reported token-gate failure: the memory flush threshold is clamped to 0 and then treated as disabled when reserve/headroom consume the context window. An open implementation PR, #50694, explicitly closes this issue and is unmerged, so this issue should remain open until maintainers land or replace that fix.

## Best Possible Solution

Keep this issue open and review/land PR #50694, or replace it with a maintainer-preferred fix. The fix should clamp or validate reserveTokensFloor and softThresholdTokens relative to contextWindowTokens, preserve sane defaults, and add focused regression coverage for reserveTokensFloor equal to or greater than the context window.
