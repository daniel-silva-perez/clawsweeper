## Summary

Keep this issue open. Current main has automatic Discord typing infrastructure for inbound replies and heartbeat delivery, but the shared `message` tool still has no `typing` or `typing-indicator` action, Discord does not advertise that action, and the dispatcher would reject it. The related lifecycle bugs appear to be cleanup/keepalive fixes, not the requested agent-callable message-tool surface.

## Best Possible Solution

Keep this issue open until maintainers either add a shared `typing` or `typing-indicator` message-tool action, or explicitly decide that automatic inbound/heartbeat typing is the intended product boundary. A concrete fix should update the shared action registry and target mode, expose the action through Discord discovery, route it to `sendTypingDiscord`, define lifecycle semantics, add Discord regression tests, and document the public message-tool behavior.
