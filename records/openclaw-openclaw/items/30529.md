## Summary

Keep this issue open. Current main already has automatic Discord typing plumbing for inbound/component/heartbeat paths, but it still does not expose a `typing` or `typing-indicator` action through the shared `message` tool or Discord message-action dispatcher. The prior ClawSweeper keep-open conclusion remains accurate against current main `955b4df0939f90b4b6de5e7e6f3149dd125e1e50`.

## Best Possible Solution

Keep this open until maintainers either add a shared `typing` or `typing-indicator` message-tool action, or explicitly document that automatic inbound/component/heartbeat typing is the intended OpenClaw boundary. A complete implementation likely belongs in the shared action registry/type, message-tool schema and docs, CLI action normalization/target mode, Discord discovery and dispatch, and lifecycle tests around Discord's temporary typing endpoint.
