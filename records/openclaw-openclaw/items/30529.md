## Summary

Keep this issue open. Current main has automatic Discord typing infrastructure for inbound replies and heartbeat targets, but it still does not expose the requested agent-callable `typing` or `typing-indicator` action through the shared `message` tool. The related discussion points to lifecycle bugs and cross-provider design concerns, not to a canonical duplicate or shipped manual message-tool action.

## Best Possible Solution

Keep the issue open until maintainers either add a shared `typing` or `typing-indicator` message-tool action, or explicitly decide automatic typing is sufficient. A concrete implementation should add the shared action name and schema fields, advertise it from Discord `describeMessageTool`, route it through `handleDiscordMessageAction` to `sendTypingDiscord` with account and target resolution, define lifecycle semantics, add Discord regression coverage, and update the public message/typing docs. If cross-provider parity is desired, design the action so Telegram, Slack, Discord, and similar plugins can opt in through the channel plugin contract.
