## Summary

Keep this issue open. Current main has automatic Discord typing indicators for inbound reply and heartbeat/component paths, but it still does not expose an agent-callable `typing` or `typing-indicator` action through the shared `message` tool. The previous ClawSweeper keep-open review remains accurate against current main e583db63c602d8d21b4ebdcd119061571a2e40a9.

## Best Possible Solution

Keep the issue open until maintainers either add a shared `typing` or `typing-indicator` message-tool action, or explicitly decide that automatic inbound reply and heartbeat typing are the intended OpenClaw boundary. A complete implementation would update the shared action registry/type, message-tool schema, CLI normalization/target mode, Discord discovery, Discord action dispatch, lifecycle semantics, regression tests, and public docs.
