## Summary

Keep open. Current main still switches and acknowledges `/model` without a switch-time comparison between existing session context and the target model budget or rate-limit risk; the existing guards run later in the agent path, and related #58957 is broader and still open rather than a merged superseding fix.

## What This Changes

The issue asks for `/model` switches and fallback paths to estimate current session context against the target model's limits and rate-limit characteristics, then warn or offer compaction before switching.

## Best Possible Solution

Add a shared model-switch context-risk warning used by WebChat and messaging directives: resolve the target model's effective `contextTokens` and output reserve, compare them with the freshest available session prompt usage, and recommend `/compact` or `/new` when the session is near or above a conservative threshold. Treat automatic compaction and account-tier TPM prediction as follow-up policy decisions.
