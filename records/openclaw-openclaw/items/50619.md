## Summary

Keep this open. Current main still acknowledges `/model` switches without comparing the current session's prompt/context pressure against the newly selected model's effective budget; the existing safeguards run later in the agent path and handle overflow, not the requested switch-time warning for context or account-rate-limit risk.

## What This Changes

The issue requests that `/model` estimate the current session context and warn before switching to a target model whose context or rate-limit characteristics may make the carried-over session unsafe.

## Best Possible Solution

Add a shared model-switch context-risk warning used by WebChat/TUI and messaging directives: resolve the target model's effective `contextTokens`/`maxTokens`, compare it with the freshest available current prompt/session usage, and append a clear recommendation to run `/compact` or `/new` when the session is near or above a conservative threshold. Keep auto-compaction before switching as a separate maintainer product decision.
