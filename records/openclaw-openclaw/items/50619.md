## Summary

Keep open. Current main still applies `/model` switches and acknowledges them before comparing the existing session context with the target model's effective budget or the reported large-request TPM/rate-limit risk; the runtime context guards run later, and related #58957 is broader and still open rather than a merged superseding fix.

## What This Changes

The issue requests a `/model` switch-time check that estimates current session context against the target model's limits and warns or offers compaction before risky switches.

## Best Possible Solution

Add a shared model-switch context-risk warning used by WebChat and messaging directives: resolve the target model's effective `contextTokens` and output reserve, compare that with the freshest available session prompt usage, and recommend `/compact` or `/new` when the session is near or above a conservative threshold. Treat automatic compaction and provider/account-specific TPM prediction as explicit follow-up decisions.
