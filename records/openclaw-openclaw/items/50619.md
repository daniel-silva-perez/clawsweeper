## Summary

Keep open. Current main still changes the session model immediately and acknowledges the switch without estimating the existing session context against the target model budget or warning about the reported large-request TPM/rate-limit case. The existing runtime guards and WebChat context notice help later or elsewhere, but they do not satisfy the requested switch-time warning; related #58957 overlaps but is broader and open, not a clear superseding canonical item.

## What This Changes

The issue requests a `/model` switch-time check that estimates current session context, compares it with target model limits and rate-limit characteristics, and warns or offers compaction before switching.

## Best Possible Solution

Add a shared model-switch context-risk warning used by WebChat and messaging directives: resolve the target model's effective contextTokens/maxTokens, compare it with the freshest available session prompt usage, and recommend `/compact` or `/new` when the session is near or above a conservative threshold. Treat automatic compaction and provider/account-specific TPM predictions as explicit follow-up decisions.
