## Summary

Keep open. Current main still applies model switches immediately and acknowledges them without a switch-time comparison between existing session size and the target model budget or rate-limit risk. Existing context overflow handling runs later in the prompt path, and the WebChat context notice is a general current-session pressure indicator, not the requested `/model` warning.

## What This Changes

The issue requests that `/model` estimate current session context, compare it with the target model's effective limits and rate-limit characteristics, and warn or offer compaction before switching when the carried-over context is risky.

## Best Possible Solution

Add a shared model-switch context-risk check used by WebChat and messaging directives. It should resolve the target model's effective contextTokens/contextWindow/maxTokens, compare them with the freshest available prompt/session usage, and recommend `/compact` or `/new` before the switch is treated as safe; automatic compaction and account-specific TPM prediction should remain explicit follow-up decisions.
