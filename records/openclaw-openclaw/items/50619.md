## Summary

Keep this issue open. Current main has lower-level context-window enforcement for model switches, but it still does not do the requested user-visible `/model` pre-switch context/rate-limit risk check. Both WebChat and messaging command paths persist the model and acknowledge it without comparing fresh session usage against the target model budget, and the reported ~200k-token OpenAI Codex case can sit below the current 272k runtime cap while still being risky for account TPM pressure.

## Best Possible Solution

Keep this open and narrow the implementation to a shared model-switch context-risk warning. Resolve the target model's effective `contextTokens`/`maxTokens`, compare it with fresh current session prompt usage when available, and append a clear warning to `/model` acknowledgements across WebChat/TUI and messaging channels when the session is near or above a conservative target threshold. Recommend `/compact` or `/new`; auto-compaction before switching should remain a separate maintainer product decision. Coordinate with #58957 for broader silent model-switch recovery gaps while leaving already-closed smaller-window enforcement work closed.
