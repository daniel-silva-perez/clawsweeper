## Summary

Keep open. Current main still records and acknowledges `/model` changes without estimating the current session's prompt/context pressure against the target model's effective budget; the existing context safeguards run later in the agent path and do not provide the requested switch-time warning or account-rate-limit guidance.

## What This Changes

Warn users during `/model` switches when the current session context is too large or risky for the target model, with guidance to compact or start a new session.

## Best Possible Solution

Add a shared model-switch context-risk warning used by WebChat and messaging directives: resolve the target model's effective `contextTokens`/`maxTokens`, compare it with the freshest available session prompt usage, and append a clear `/compact` or `/new` recommendation when the session is near or above a conservative threshold. Keep automatic compaction and provider/account-specific TPM predictions as explicit follow-up decisions.
