## Summary

Keep open. Current main has useful post-switch protections: model switches clear stale cached context budgets, the runner resolves the selected model's effective context cap, and oversized prompts can be compacted before provider submission. But the requested pre-switch `/model` behavior is still missing: `/model` and `sessions.patch` apply the override without estimating the current session context, comparing it to the target model, warning the user, or offering `/compact`/`/new`. The reported TPM/rate-limit case is also still outside the context-overflow guard.

## Best Possible Solution

Keep this open, or explicitly merge it into a maintainer-chosen canonical model-switch context issue. The best implementation path is to add a `/model`/`sessions.patch` preflight that resolves the target model's effective runtime context cap, estimates the current session prompt/context size, warns or blocks above a safe threshold, and offers `/compact` or `/new`. Provider-specific TPM/quota checks should be conservative and only use reliable provider-owned metadata or visible usage windows.
