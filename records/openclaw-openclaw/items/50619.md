## Summary

Keep open. Current main still switches and acknowledges the requested model before any user-visible comparison between the existing session context and the target model budget; later runtime guards and general context notices do not satisfy the requested switch-time warning.

## What This Changes

The issue requests a `/model` switch preflight that estimates current session context, compares it with the target model's effective limits and rate-limit risk, and warns or offers compaction before switching.

## Best Possible Solution

Add a shared model-switch context-risk warning used by WebChat and messaging directives: resolve the target model's effective contextTokens/maxTokens, compare it with the freshest available session prompt usage, and recommend `/compact` or `/new` when the session is near or above a conservative threshold. Treat automatic compaction and account-specific TPM prediction as explicit follow-up decisions.
