## Summary

Keep open. Current main still stores and acknowledges `/model` changes without estimating the existing session context against the selected model's effective budget or warning about the reported large-request TPM/rate-limit case. Existing context safeguards run later in the agent path, and related #58957 is broader and open rather than a merged or clearly superseding fix.

## What This Changes

Add a `/model` switch preflight that estimates current session context and warns or offers compaction before switching to a target model with a tighter effective context or rate-limit budget.

## Best Possible Solution

Add a shared model-switch context-risk warning used by WebChat and messaging directives: resolve the target model's effective `contextTokens` and output reserve, compare that with the freshest available session prompt usage, and recommend `/compact` or `/new` when the session is near or above a conservative threshold. Treat automatic compaction and provider/account-specific TPM prediction as explicit follow-up decisions.
