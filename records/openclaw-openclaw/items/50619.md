## Summary

Keep open. Current main still applies `/model` changes and acknowledges them without checking the existing session context against the target model's effective budget or the reported large-request rate-limit risk; existing guards run later in the agent prompt path, and related #58957 is broader and still open rather than a merged superseding fix.

## What This Changes

The issue requests a `/model` switch-time check that estimates existing session context and warns or guides users before moving to a target model whose effective context or rate-limit budget is risky.

## Best Possible Solution

Add a shared model-switch context-risk warning used by WebChat and messaging directives. It should resolve the target model's effective `contextTokens`/`contextWindow`/`maxTokens`, compare it with the freshest prompt or transcript usage available for the current session, and recommend `/compact` or `/new` before treating the switch as safe; account-tier TPM prediction and automatic compaction should be follow-up policy choices.

## Reproduction Assessment

Yes for the observable missing warning: create or inspect a session with large carried-over token usage, invoke `/model <provider/model>` through WebChat or directive-only chat, and current main will persist and acknowledge the switch without a context-risk comparison. The exact HTTP 429 case depends on provider account tier and quota state.

## Solution Assessment

No. Current main's later pre-prompt compaction and context overflow messaging are useful safeguards, but they are not the requested switch-time warning; the safer first fix is a shared warning path before adding automatic compaction or quota prediction.
