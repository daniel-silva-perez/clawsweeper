## Summary

Keep open. Current main at f256eeba431b5f59e6085b7c11c9c0a309e408b7 still has manual/daily/idle resets, one-bit abort tracking, abort cutoff handling, and targeted context/compaction/role-ordering recovery, but it does not have a configurable gateway/session circuit breaker that counts consecutive aborts and auto-rotates the session after a threshold. Related open breaker reports and PR #42933 show this work remains unresolved rather than safely closeable.

## Best Possible Solution

Keep this open, or have a maintainer explicitly consolidate it with a canonical session-level circuit-breaker tracker. The likely core implementation should persist per-session consecutive abort state, expose a documented configurable threshold, reset the counter on successful runs or explicit session reset, call the existing session reset path when the breaker trips, and emit a clear log/session event so operators know why a fresh session was created. PR #42933 may be a useful starting point, but it needs maintainer review before it can close this issue.
