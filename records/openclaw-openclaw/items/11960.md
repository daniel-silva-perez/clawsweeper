## Summary

Keep open. Current main still captures old session state for automatic daily/idle rollover but only emits typed session_start/session_end lifecycle hooks; it does not route stale rollovers through the reset persistence path used by manual /new, /reset, and gateway reset handling. The hook-config schema mismatch called out in the issue is fixed, and open PR #71792 explicitly closes this issue while #61675 overlaps, so this should stay open for maintainer review of those implementation candidates.

## What This Changes

The issue requests a session reset hook surface so session-memory and custom hooks can persist context before automatic daily or idle resets clear a session.

## Best Possible Solution

Land #71792, #61675, or a maintainer-approved replacement that makes automatic daily and idle stale rollovers invoke the same effective pre-reset persistence surface as manual resets, passes the previous session entry/transcript to session-memory, preserves typed lifecycle hooks, and adds daily plus idle regression coverage.
