## Summary

Keep open. Current main still has the primary reset-persistence gap: automatic daily and idle rollovers retain the previous session entry and emit typed lifecycle hooks, but they do not invoke the manual reset hook path that powers session-memory persistence. Because open PR #71792 explicitly closes this issue and #61675 overlaps, maintainers should review those candidates or choose a replacement before closing.

## What This Changes

The issue requests a session reset or before-reset hook surface for automatic daily/idle/manual resets so session-memory and custom hooks can persist context, plus a schema fix for per-hook config keys such as session-memory messages.

## Best Possible Solution

Land #71792, #61675, or a maintainer-approved replacement that makes automatic daily and idle stale rollovers invoke the same effective pre-reset persistence surface as manual resets, passes the previous session entry/transcript to session-memory, preserves typed lifecycle hooks, and adds daily plus idle regression coverage.
