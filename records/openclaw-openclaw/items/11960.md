## Summary

Keep open. Current main still fixes only the secondary hook-config schema mismatch; the primary daily/idle auto-rollover path still rolls sessions through lifecycle hooks without emitting the internal command reset hook or plugin before_reset surface that session-memory depends on. Open PR #71792 explicitly closes this issue, with broader overlapping PR #61675 also open, so the issue should stay open until one implementation lands or maintainers choose a replacement.

## What This Changes

Request a session reset hook surface, or equivalent reset hook dispatch, so session-memory and custom hooks can persist context before daily, idle, manual, and command resets, plus allow hook-specific config such as session-memory.messages.

## Best Possible Solution

Keep this issue open and review/land #71792, #61675, or a maintainer-approved replacement. The fix should make automatic daily and idle stale rollovers invoke the same effective reset persistence surface as manual resets, pass previousSessionEntry to session-memory, fire before_reset if that remains the intended plugin contract, preserve typed session_end/session_start behavior, and add daily plus idle regression coverage.
