## Summary

Keep open. Current main still covers only the secondary hook-config schema mismatch; the primary daily/idle auto-rollover path archives the old transcript and emits typed session lifecycle hooks, but it does not emit the internal command reset hook or plugin `before_reset` surface that `session-memory` depends on. Open PR #71792 explicitly closes this issue, with broader overlapping PR #61675 also open, so this should stay open until maintainers land or reject an implementation.

## What This Changes

The issue requests a `session:reset` or `session:before-reset` hook event for automatic daily/idle resets and hook-specific config schema support for `session-memory.messages`.

## Best Possible Solution

Review and land #71792, #61675, or a maintainer-approved replacement that makes automatic daily and idle stale rollovers invoke the same effective reset persistence surface as manual resets, passes the previous session entry to `session-memory`, preserves typed `session_end`/`session_start` behavior, and adds daily plus idle regression coverage.
