## Summary

Keep open. Current main fixes the secondary hook-config schema mismatch, but the primary auto daily/idle reset path still does not emit the internal command reset hook or plugin before_reset hook that session-memory relies on. Open PR #71792 is already an implementation candidate for this issue, with #61675 as a broader related candidate, so this should stay open until one lands or maintainers choose a replacement.

## Best Possible Solution

Keep this issue open and review/land or close #71792, #61675, or a maintainer-approved replacement. The fix should make automatic daily and idle stale rollovers invoke the same effective reset persistence surface as manual resets, pass previousSessionEntry to session-memory, fire before_reset if that remains the intended plugin contract, preserve typed session_end/session_start behavior, and add daily plus idle regression coverage.
