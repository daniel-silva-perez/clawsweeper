## Summary

Keep open. Current main still does not route automatic daily or idle stale rollovers through the reset persistence path used by session-memory; the secondary hook-config schema mismatch is fixed, and open implementation PRs already cover the remaining primary gap.

## What This Changes

The issue requests a reset hook for automatic daily/idle session resets, plus hook config schema support for session-memory options such as messages.

## Best Possible Solution

A maintainer-approved patch should make automatic daily and idle rollovers invoke the same effective pre-reset persistence surface as manual resets, pass previous session entry/transcript data and reset reason to session-memory and custom hooks, preserve typed lifecycle hooks, and add daily plus idle regression coverage.

## Reproduction Assessment

Yes. Configure a stale daily or idle session and send the next non-system message: current main creates a new session with resetTriggered false, archives the previous transcript, emits only lifecycle hooks, and leaves session-memory gated to command:new/command:reset.

## Solution Assessment

Unclear. The requested direction is valid, but the best implementation should be selected from PR #71792, PR #61675, or a maintainer-approved replacement so the repo does not gain competing reset hook semantics.
