## Summary

Keep open. Current main still fixes only the secondary hook-config schema mismatch; the primary daily/idle auto-reset path does not emit the internal reset hook that `session-memory` listens to. Open PR #71792 carries closing syntax for this issue, so the issue should remain open until that PR, #61675, or a replacement merges.

## Best Possible Solution

Keep this issue open until #71792, #61675, or a replacement lands a narrow fix that fires the existing reset hook behavior, or an equivalent documented session reset surface, during automatic daily and idle stale rollover. The fix should pass `previousSessionEntry` to session-memory, preserve typed `session_end`/`session_start`, run `before_reset` if that remains the intended plugin contract, avoid duplicate manual-reset hook emission, and add daily plus idle regression coverage. The secondary schema mismatch is already fixed on current main.
