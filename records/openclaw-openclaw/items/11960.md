## Summary

Keep open. Current main still does not route automatic daily or idle stale rollovers through the reset persistence hook surface used by manual resets, so the primary memory-persistence request remains unresolved. The secondary hook-config schema mismatch is fixed, and open PR #71792 explicitly closes this issue while #61675 overlaps, so the right next step is maintainer review of the existing implementation candidates rather than closing or queueing a new repair.

## What This Changes

Request a `session:reset`/`session:before-reset` hook or equivalent reset persistence event so automatic daily and idle session resets can save previous session memory before the old context is discarded, plus allow hook-specific config such as `messages`.

## Best Possible Solution

Land #71792, #61675, or a maintainer-approved replacement that wires automatic daily and idle rollovers into the same effective pre-reset persistence surface as manual resets, supplies previous transcript data to session-memory and custom hooks, preserves typed lifecycle hooks, and adds daily plus idle regression coverage.
