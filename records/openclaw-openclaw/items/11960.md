## Summary

Keep open. Current main still does not make automatic daily/idle stale rollovers invoke the reset persistence surface used by manual `/new`, `/reset`, and gateway reset handling; it only emits typed lifecycle hooks. The secondary per-hook config schema mismatch is fixed, and open PR #71792 explicitly closes this issue while #61675 overlaps, so the remaining action is maintainer review of those implementation candidates.

## What This Changes

Request to add a `session:reset`/`session:before-reset` hook path for automatic daily and idle session rollovers so `session-memory` and custom hooks can persist previous session context, plus allowing hook-specific config keys such as `messages`.

## Best Possible Solution

Land #71792, #61675, or a maintainer-approved replacement that makes automatic daily and idle stale rollovers invoke the same effective pre-reset persistence surface as manual resets, passes the previous session entry/transcript to session-memory, preserves typed lifecycle hooks, and adds daily plus idle regression coverage.
