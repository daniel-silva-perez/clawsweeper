## Summary

Keep open. Current main still leaves the primary automatic reset persistence gap: daily and idle rollovers have previous-session data available and emit typed lifecycle hooks, but they do not run the internal command reset hook or plugin before_reset path that session-memory uses. The secondary hook-config schema mismatch is fixed, and open PR #71792 plus overlapping PR #61675 make this maintainer-review work rather than a new autonomous repair candidate.

## What This Changes

Request a session reset hook/event so automatic daily and idle resets can persist memory before clearing context, plus support hook-specific config such as session-memory messages.

## Best Possible Solution

Land #71792, #61675, or a maintainer-approved replacement that wires automatic daily and idle rollovers into the same effective pre-reset persistence surface as manual resets, supplies previous transcript data to session-memory/custom hooks, preserves typed lifecycle hooks, and adds daily plus idle regression coverage.
