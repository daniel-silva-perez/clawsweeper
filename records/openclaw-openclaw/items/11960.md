## Summary

Keep open. Current main still resolves only the secondary hook-config schema mismatch; the primary daily/idle auto-rollover path captures and archives the old session and fires typed `session_end`/`session_start` plugin hooks, but it still does not emit the internal command reset hook or `before_reset` path that `session-memory` listens to. Open PR #71792 carries closing syntax for this issue, and #61675 is a larger related implementation candidate, so the issue should remain open until one lands or maintainers choose a replacement.

## Best Possible Solution

Keep this issue open until #71792, #61675, or a replacement lands a narrow fix that fires the existing internal reset hook behavior, or an equivalent documented reset surface, during automatic daily and idle stale rollover. The fix should pass `previousSessionEntry` to session-memory, preserve typed `session_end`/`session_start`, run `before_reset` if that remains the intended plugin contract, avoid duplicate manual-reset hook emission, and add daily plus idle regression coverage. The secondary hook-config schema mismatch should be left as-is because current main already allows per-hook keys like `messages`.
