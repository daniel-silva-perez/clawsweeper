## Summary

Keep open. Current main still does not make automatic daily/idle rollovers invoke the same reset persistence surface as manual `/new` and `/reset`; the secondary hook-config schema mismatch is fixed, but the primary memory-persistence hook gap remains. Open PR #71792 explicitly closes this issue and #61675 overlaps, so maintainers should land, replace, or reject one of those implementations before this issue closes.

## What This Changes

The issue asks for automatic daily, idle, and manual session resets to expose a reset hook surface so session-memory and custom hooks can persist context before clearing the prior session.

## Best Possible Solution

Land #71792, #61675, or a maintainer-approved replacement that makes automatic daily and idle stale rollovers invoke the same effective pre-reset persistence surface as manual resets, passes the previous session entry/transcript to session-memory, preserves typed session lifecycle hooks, and adds daily plus idle regression coverage.
