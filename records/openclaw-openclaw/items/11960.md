## Summary

Keep open. The primary reset-persistence gap is still present in the inspected main snapshot: automatic daily/idle rollover captures the previous session and emits typed lifecycle hooks, but it does not invoke the reset hook path used by manual resets. The secondary hook config schema mismatch is already fixed, and open PR #71792 explicitly closes this issue while #61675 overlaps, so maintainers should review those implementation candidates rather than close this issue now.

## What This Changes

The issue requests a hook event for automatic daily and idle session resets so session-memory and custom hooks can persist context before rollover, plus schema support for hook-specific config keys.

## Best Possible Solution

Land #71792, #61675, or a maintainer-approved replacement that makes automatic daily and idle stale rollovers invoke the same effective pre-reset persistence surface as manual resets, passes the previous session entry/transcript to session-memory, preserves typed lifecycle hooks, and adds daily plus idle regression coverage.
