## Summary

Keep this PR open. Current main still has the Copilot API-token cache collision in both the active GitHub Copilot plugin token helper and the legacy agent helper, and this PR remains a focused implementation candidate for the still-open paired bug #43658.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #43658. Maintainers should review and land this patch or an equivalent fix that binds cached Copilot API tokens to the GitHub credential/profile in both active token helper paths, with regression coverage for cross-profile cache misses and legacy unowned cache entries.

## Risks / Open Questions

- The PR persists a stable 16-hex SHA-256-derived fingerprint of the GitHub PAT in `credentials/github-copilot.token.json`; it is not the raw token, but maintainers should accept that correlation/privacy tradeoff or ask for a different cache-key design.
- The PR still uses a single shared cache file, so alternating healthy profiles will evict each other and refetch tokens. That is an efficiency follow-up, not a reason to close the correctness fix.
- No local tests were run because this review was required to keep the checkout read-only; the decision is based on source inspection, the supplied PR context, and the live diff text.
