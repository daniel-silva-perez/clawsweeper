## Summary

Keep this PR open. Current main still reuses a valid cached GitHub Copilot API token without binding it to the incoming GitHub credential, and this PR remains a focused implementation candidate for the same-author paired bug #43658. The security-sensitive part of the diff is limited to persisting a short SHA-256-derived credential fingerprint; that design needs maintainer acceptance, but I did not see unrelated CI, dependency, package, or workflow changes in the provided PR file list.

## Best Possible Solution

Keep this PR open for maintainer review. Land it, or an equivalent fix, if the credential-fingerprint cache ownership design is acceptable; otherwise request a revised per-profile cache-key design while keeping #43658 open until the active plugin and legacy agent helper both stop reusing Copilot API tokens across GitHub profiles.
