## Risks / Open Questions

- The PR cannot land as-is because the edited `src/providers` path is not present on current main.
- Persisting a stable credential-derived fingerprint is not the raw token, but maintainers should decide whether that cache ownership marker is acceptable or whether a per-profile cache key/design is preferable.
