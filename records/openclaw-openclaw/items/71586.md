## Summary

Keep open. Current main still uses the native SwiftUI WebChat scroll path with programmatic bottom-pinning and no visible scrollbar-drag guard, and neither the latest release comparison nor the related closed Control UI scroll PRs show a native macOS fix for this reported hang.

## What This Changes

The issue requests that the macOS companion app chat remain responsive when a user drags the chat scrollbar thumb rapidly through history.

## Best Possible Solution

Keep the issue open and fix the shared native macOS chat scroll behavior so user-driven scrollbar-thumb interaction is never fought by automatic bottom-follow updates, while preserving auto-follow for new user sends and active streaming when the user is actually at the bottom.

## Reproduction Assessment

Yes for triage: the report gives exact macOS companion steps and current main still uses the same native SwiftUI scroll architecture. It was not independently live-reproduced here, so landing should include manual macOS scrollbar-thumb verification.
