## Summary

Keep open. The report is recent and specific, and current main still routes the macOS companion chat through the same native SwiftUI message list with bound `scrollPosition` and animated bottom-pinning; I found no code, test, docs, release, or related-item evidence that scrollbar-thumb dragging has been fixed or superseded.

## What This Changes

The issue reports that the macOS companion app WebChat freezes when a user rapidly drags the chat history scrollbar thumb.

## Best Possible Solution

Fix the native macOS WebChat scroll behavior so programmatic bottom-pinning does not fight user-driven scrollbar interaction. The end state should keep the chat responsive during rapid scrollbar-thumb dragging while preserving sensible auto-follow behavior for newly sent messages and streaming assistant output.

## Reproduction Assessment

Yes for triage: the issue gives concrete steps against v2026.4.22 on macOS 26.4.1, and current main still uses the implicated native SwiftUI scroll architecture. It was not independently live-reproduced in this Linux read-only checkout.

## Solution Assessment

Not applicable as a patch review because there is no proposed fix in this item. The narrowest maintainable solution likely belongs in `OpenClawChatView`, with a macOS-aware guard for user scroll interaction or an `NSScrollView`-backed replacement if SwiftUI `scrollPosition` remains unstable.
