## Summary

Keep open. The report is recent, specific, and not protected from normal triage, but current main still routes the macOS companion chat through the same native SwiftUI `ScrollView`/`LazyVStack` with bound `scrollPosition` and repeated programmatic bottom-pinning; I found no merged fix, release note, test, or linked PR that resolves scrollbar-thumb dragging hangs.

## What This Changes

The issue reports that the macOS companion app chat window hangs when the scrollbar thumb is dragged rapidly through chat history.

## Best Possible Solution

Fix the native macOS `OpenClawChatView` scroll behavior so programmatic bottom-pinning does not fight user-driven scrollbar interaction, while preserving bottom-follow behavior for sends, newly appended user messages, pending runs, and streaming assistant output.

## Reproduction Assessment

Yes for triage: the issue gives concrete macOS companion steps against v2026.4.22 on macOS 26.4.1, and current main still uses the same implicated native SwiftUI scroll architecture. This review did not independently live-reproduce the hang outside macOS.

## Solution Assessment

Not applicable as a patch review because there is no proposed fix in this item. The narrowest maintainable direction likely belongs in `OpenClawChatView`, with a macOS-aware guard for active user scroll interaction or an `NSScrollView`-backed replacement if SwiftUI `scrollPosition` remains unstable.
