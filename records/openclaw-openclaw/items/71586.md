## Summary

Keep open. The report is recent and specific, and current main still uses the same native SwiftUI chat scroll path with bound `scrollPosition` plus repeated programmatic bottom-pinning; I found no merged fix, test, release note, or linked PR that resolves scrollbar-thumb dragging hangs.

## What This Changes

The issue reports that the macOS companion chat UI freezes when the user drags the scrollbar thumb rapidly through chat history.

## Best Possible Solution

Fix the native macOS `OpenClawChatView` scroll behavior so bottom-pinning does not fight user-driven scrollbar interaction, while preserving bottom-follow behavior for sends, newly appended user messages, pending runs, and streaming assistant output.

## Reproduction Assessment

Yes for triage: the issue provides concrete macOS companion steps against v2026.4.22 on macOS 26.4.1, and current main still uses the implicated native SwiftUI scroll architecture. This review did not independently live-reproduce the hang on macOS.

## Solution Assessment

Not applicable as a patch review because there is no proposed fix in this item. The narrowest maintainable direction likely belongs in `OpenClawChatView`, with a macOS-aware guard for active user scroll interaction or an `NSScrollView`-backed replacement if SwiftUI `scrollPosition` remains unstable.
