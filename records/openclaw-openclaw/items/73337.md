## Summary

Keep open. This is a fresh, concrete macOS Canvas/A2UI regression report, and current main still contains the auto-navigation and push-result behavior that can plausibly leave the WKWebView blank after a successful A2UI push. The older related reports #7143 and #22292 are closed as stale, so they are useful context but not a canonical active tracker.

## Best Possible Solution

Keep this issue open and fix it in the macOS Canvas/A2UI owner path. Start by making the auto-navigation guard skip reloads when the current target already equals the last A2UI target, then add a macOS WKWebView regression or manual smoke that pushes minimal v0.8 JSONL and verifies visible content or snapshot pixels rather than only bridge success. If the blank render remains after the reload guard, instrument `ensureA2UIHost`, bundle loading, the host `connectedCallback`, `openclawA2UI.applyMessages`, and WKWebView console errors to isolate mount versus message application versus rendering.
