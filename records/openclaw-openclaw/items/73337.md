## Summary

Keep open. The report is fresh, has a concrete reproduction, and current main still contains macOS Canvas/A2UI behavior that can plausibly produce the reported blank-render regression. The older related reports were closed as stale, so they should not supersede this fresh reproduction.

## Best Possible Solution

Keep this issue open and fix it in the macOS Canvas/A2UI owner path. Start by changing the auto-navigation guard so an already-loaded A2UI URL is not reloaded on gateway snapshots or eval/show reuse, then add a macOS regression test or manual smoke that pushes minimal v0.8 JSONL and verifies the WebView renders a visible A2UI surface rather than only returning `{ ok: true }`. If the render failure persists after the reload guard, instrument `ensureA2UIHost`, `applyMessages`, and the WKWebView console path to distinguish bundle load, host upgrade, message application, and surface render failures.
