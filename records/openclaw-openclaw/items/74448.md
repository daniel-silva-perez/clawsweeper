## Summary

Keep open. Current main still has a plausible premature-completion path: subagent wait finalization treats any non-pending, non-yield terminal wait as completion, then freezes output from the child session immediately. I found adjacent fixes for sessions_yield and announce retry behavior, but not a compaction/resume-chain contract that keeps the original subagent task pending across internal compaction.

## What This Changes

Make subagent completion and `agent.wait` semantics survive auto-compaction so parents receive the child’s final handoff after actual task end instead of mid-task frozen output.

## Best Possible Solution

Add a compaction-aware lifecycle contract so an internal compaction/resume does not satisfy the parent-visible subagent completion contract. The final implementation should freeze and announce exactly once, after the resumed child run actually reaches terminal state, while preserving the existing sessions_yield pause behavior.

## Reproduction Assessment

Yes. The issue includes a high-confidence reproduction pattern with three compaction incidents and timestamps tying the parent’s frozen result to pre-final child history; current source inspection shows the matching wait/freeze path still exists.

## Solution Assessment

Yes, the preferred direction is to keep `agent.wait` or the subagent registry pending across compaction/resume rather than only extending the 1.5s output capture retry. A late freeze workaround could reduce symptoms, but the cleaner contract is that compaction remains internal and does not produce task completion.
