## Summary

Keep open. Current main still has the same observable contract gap: a terminal `agent.wait` result for the child run immediately drives subagent completion bookkeeping and freezes output from the child session history, while the inspected compaction path is separate lifecycle state and I found no current code or regression test that transfers the wait across an internal compaction/resume boundary or suppresses that premature completion.

## What This Changes

The issue reports that subagent completion can be announced when session compaction ends the original run, before the resumed worker has actually finished and produced its final handoff.

## Best Possible Solution

Add an explicit compaction-aware subagent lifecycle contract so internal compaction/resume keeps completion pending, preserves accumulated timing/usage, and freezes the child handoff exactly once after the resumed turn reaches its real terminal state.
