## Summary

Keep open. Current main still treats a terminal non-yielding `agent.wait` result as subagent completion and immediately freezes the child session's current visible assistant text; I did not find compaction/resume-chain handling that would keep the parent wait pending until the resumed run's real final handoff. The shipped #73413 sessions_yield fix is adjacent and useful, but it does not cover the reported compaction-as-terminal-success path.

## What This Changes

The issue asks OpenClaw to keep subagent completion pending across internal session compaction so the parent receives the final child handoff rather than a mid-task frozen result.

## Best Possible Solution

Add an explicit compaction-aware lifecycle contract: a subagent run that compacts and resumes should remain non-terminal to its parent until the resumed turn actually ends, then freeze and announce exactly one final child handoff.

## Reproduction Assessment

Yes. The issue gives three same-day long-running subagent reproductions with compaction timestamps, premature parent completion timestamps, and later final child messages; source inspection on current main matches the reported failure mode, though I did not run the live repro in this read-only pass.

## Solution Assessment

Yes, Option A from the issue is the narrowest maintainable direction: make `agent.wait`/subagent lifecycle follow the compaction resume chain rather than compensating by guessing during result freezing. A freeze-time deferral can be a useful guard, but it should not be the primary contract.
