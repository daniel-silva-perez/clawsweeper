## Summary

Keep this issue open. Current main still routes WhatsApp group mention detection through `extractMentionedJids()`, which only reads a narrow set of `contextInfo.mentionedJid` arrays and still returns undefined when those arrays are absent. The reported bug is specifically a native WhatsApp MD group mention where mention metadata appears to exist but `mentionedJids` is null, and the current tests do not cover that payload shape.

## Best Possible Solution

Keep the issue open and fix it in the WhatsApp plugin. Add a regression fixture for the real WhatsApp MD group @-mention payload, broaden `extractMentionedJids()` or the current-message context extraction path to read the actual native mention metadata without leaking quoted-message mentions, then verify group gating sets `wasMentioned=true` while preserving the separate self-chat/LID safeguards tracked by #49317/#73453.
