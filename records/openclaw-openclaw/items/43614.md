## Summary

Keep open. Current main and the latest stable release still emit the misleading zero-pending bootstrap wording, while the linked bug is supported by source and docs. The PR is a narrow, correct implementation candidate; the remaining action is normal maintainer review and merge or direct application.

## What This Changes

This PR changes the zero-pending `openclaw status` agents summary from `no bootstrap files` to `no workspaces bootstrapping` and updates status command regression coverage.

## Best Possible Solution

Land this PR, or apply the same focused main-branch change, so `openclaw status` reports zero pending `BOOTSTRAP.md` work as no active bootstrapping while preserving regression coverage for mature workspaces.

## Reproduction Assessment

Yes. The current-main renderer deterministically emits `no bootstrap files` when `bootstrapPendingCount` is zero, and that count is computed solely from `BOOTSTRAP.md` existence in each workspace.

## Solution Assessment

Yes. The PR is the narrowest maintainable fix: it changes the human-facing label to match the existing signal and adds regression coverage without changing bootstrap detection, workspace injection, or status scanning behavior.
