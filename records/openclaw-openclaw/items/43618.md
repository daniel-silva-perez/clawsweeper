## Summary

Keep this PR open. Current main still does not implement the proposed queue diagnostics snapshot, `diagnostics.queue` Gateway RPC, or `openclaw gateway diagnostics queue` CLI. The related `queue_status` issue remains adjacent and agent-facing, not a shipped replacement for the operator-facing Gateway diagnostics surface proposed here.

## Best Possible Solution

Keep this PR open for maintainer review or an updated successor. The best path is a small core-owned, read-only queue snapshot seam in `src/process/command-queue.ts`, exposed through `diagnostics.queue` and `openclaw gateway diagnostics queue`, scoped to `operator.read`, documented in the Gateway and queue docs, and covered by focused queue, Gateway handler, CLI wiring, and authorization tests. Keep #9797 separate for the agent-facing `queue_status` tool discussion.
