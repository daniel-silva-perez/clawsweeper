## Summary

Keep this PR open. Current main still lacks the proposed `diagnostics.queue` RPC and `openclaw gateway diagnostics queue` command, so the feature is not obsolete; however, the branch is stale against current Gateway diagnostics and queue-runtime code and needs maintainer rebase/review before it can merge.

## What This Changes

The PR adds an age-aware queue diagnostics snapshot, a read-only `diagnostics.queue` Gateway RPC, a `gateway diagnostics queue` CLI command, docs, and focused tests.

## Best Possible Solution

Rebase the feature onto current main as a core-owned operator diagnostics surface: add `diagnostics.queue` beside `diagnostics.stability`, add `queue` under the existing `gateway diagnostics` CLI namespace, preserve the current global queue runtime and stuck-session recovery behavior, document the RPC/CLI contract, and keep agent-facing `queue_status` work in #9797 separate.
