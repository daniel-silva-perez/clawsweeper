## Summary

Keep open. Current main still lacks the proposed `diagnostics.queue` RPC and `openclaw gateway diagnostics queue` CLI surface, so this PR is not obsolete. The branch needs a rebase because current main already has `diagnostics.stability`, a `gateway diagnostics export` namespace, and singleton-backed queue state that this patch must preserve.

## What This Changes

The PR adds an age-aware command queue diagnostics snapshot, a read-only Gateway `diagnostics.queue` RPC, `openclaw gateway diagnostics queue` CLI output, docs, and focused tests.

## Best Possible Solution

Land a rebased core operator diagnostics feature that adds `diagnostics.queue` alongside the existing stability diagnostics, registers `queue` under the current `gateway diagnostics` CLI namespace, extends the singleton-backed command queue with age data, documents the stable JSON output, and leaves the agent-facing `queue_status` tool to #9797.
