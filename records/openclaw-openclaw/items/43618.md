## Summary

Keep open because current main still lacks the operator queue diagnostics surface, and the PR branch is useful but stale against current Gateway diagnostics and queue internals. Closing would lose a real implementation candidate rather than cleaning up obsolete work.

## What This Changes

The PR adds an age-aware command queue diagnostics snapshot, a read-only Gateway `diagnostics.queue` RPC, `openclaw gateway diagnostics queue` CLI output, docs, scope registration, config-guard allowance, and focused tests.

## Best Possible Solution

Land a rebased core operator diagnostics feature that adds `diagnostics.queue` beside `diagnostics.stability`, registers `queue` under the existing `gateway diagnostics` CLI namespace, extends the singleton command queue state with age data, documents the stable JSON output, and keeps the agent-facing `queue_status` tool tracked separately in #9797.
