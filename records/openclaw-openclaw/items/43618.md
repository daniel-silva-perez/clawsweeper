## Summary

Keep open. Current main still does not implement `diagnostics.queue`, so the PR is not obsolete, but the branch is stale and merge-dirty against current Gateway diagnostics and queue runtime work. It needs a rebase that preserves existing stability/export diagnostics and layers queue age tracking onto the current shared queue state before normal maintainer review.

## What This Changes

The PR adds an age-aware command queue diagnostics snapshot, a read-only `diagnostics.queue` Gateway RPC, an `openclaw gateway diagnostics queue` CLI command, docs, and focused tests.

## Best Possible Solution

Rebase this as a core-owned operator diagnostics feature: add `diagnostics.queue` alongside `diagnostics.stability`, register `queue` under the existing `gateway diagnostics` CLI namespace, extend the current singleton-backed command queue snapshot with age data, document the RPC/CLI contract, and keep agent-facing `queue_status` work in #9797 separate.
