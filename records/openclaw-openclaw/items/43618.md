## Summary

Keep open. Current main still lacks the proposed queue diagnostics RPC and CLI command, so this PR is not obsolete; however, the branch is stale against main and would need a careful rebase to preserve newer stability diagnostics, diagnostics export CLI wiring, and singleton-backed queue state.

## What This Changes

The PR adds a queue diagnostics snapshot, a read-only `diagnostics.queue` Gateway RPC, `openclaw gateway diagnostics queue` CLI output, docs, scope/config-guard wiring, and focused tests.

Land a rebased operator diagnostics feature that adds `diagnostics.queue` beside `diagnostics.stability`, registers `queue` under the existing `gateway diagnostics` command tree, extends the singleton command-queue state with age data, documents the stable JSON output, and leaves the agent-facing `queue_status` tool tracked separately in #9797.
