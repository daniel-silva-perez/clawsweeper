## Summary

Keep this issue open. Current main still has the OpenAI-compatible streaming parser gap that can coerce structured Mistral `delta.content` blocks into `[object Object]`, and the issue is explicitly paired with open PR #75339, so it should close only after that PR or an equivalent parser-side fix lands.

## What This Changes

Mistral thinking-mode responses should not leak `[object Object]` into agent replies, session transcript files, or memory.

## Best Possible Solution

Land PR #75339 or an equivalent shared OpenAI-compatible stream parser fix that converts typed `thinking` blocks into hidden thinking deltas, visible `text` blocks into assistant text, and drops unknown objects without string coercion; then close this issue after the merge.
