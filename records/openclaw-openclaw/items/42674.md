## Summary

Keep open. Current main still does not implement Telegram `message.send` inline `buffer`/`filename`/`contentType` delivery, so this pull request is not obsolete. It remains a useful but stale implementation candidate for the open Telegram buffer-send bug tracked by #41779; #63578 overlaps shared hydration only and does not complete the active Telegram transport path.

## Best Possible Solution

Keep this PR open as a stale but useful implementation candidate for #41779. A mergeable version should retarget `extensions/telegram/src/action-runtime.ts`, `extensions/telegram/src/channel-actions.ts`, and `extensions/telegram/src/send.ts`; coordinate shared `send` hydration with #63578; add focused regression coverage for buffer-only sends and data-URL normalization; and enforce a pre-decode base64 length limit before allocating a `Buffer`.
