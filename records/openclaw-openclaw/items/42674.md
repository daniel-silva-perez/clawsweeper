## Summary

Keep this PR open. Current main still does not implement Telegram `message.send` inline buffer/filename/contentType delivery, and the PR remains a useful but stale implementation candidate for #41779. It needs to be refreshed for the pluginized Telegram paths and hardened before merge.

## Best Possible Solution

Keep this PR open as a stale but useful implementation candidate for #41779. A mergeable version should retarget `extensions/telegram/src/action-runtime.ts`, `extensions/telegram/src/channel-actions.ts`, and `extensions/telegram/src/send.ts`, update shared send hydration in `src/infra/outbound` in coordination with #63578, add focused regression coverage for buffer-only sends and data-URL normalization, and enforce an encoded-length limit before decoding inline base64.
