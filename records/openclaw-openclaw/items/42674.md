Keep this PR open. Current main still does not implement Telegram message.send inline buffer/filename/contentType delivery, and the PR is still a useful but stale implementation candidate for the open Telegram buffer-send bug tracked by #41779.

## Best Possible Solution

A mergeable version should retarget extensions/telegram/src/action-runtime.ts, extensions/telegram/src/channel-actions.ts, and extensions/telegram/src/send.ts; update shared send hydration in src/infra/outbound consistently with #63578; add focused regression coverage for buffer-only sends and data-URL normalization; and enforce a pre-decode base64 length limit before allocating a Buffer.
