## Summary

Keep this PR open. Current main and the latest release still use ambient HTTP(S) proxy env for Feishu WebSocket, the PR is paired with open bug #65799 via closing syntax, and the branch still needs the reported test env cleanup plus maintainer agreement on changing Feishu WebSocket proxying from ambient default to explicit opt-in.

## What This Changes

The PR gates Feishu WebSocket proxy-agent creation behind `OPENCLAW_FEISHU_WS_USE_PROXY=1` and updates Feishu client tests to cover default proxy bypass and explicit proxy opt-in.

## Best Possible Solution

Keep this PR and #65799 open. Update the PR so `OPENCLAW_FEISHU_WS_USE_PROXY` is captured, deleted before each test, and restored in `afterEach`; then have a Feishu/channel maintainer decide whether explicit opt-in is the desired default. If accepted, land it with targeted Feishu tests plus a small docs/changelog note for the new opt-in; if rejected, close the PR while leaving #65799 to track a different mitigation or diagnostic improvement.
