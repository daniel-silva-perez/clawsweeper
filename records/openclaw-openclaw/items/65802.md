## Summary

Keep this PR open. Current main and the latest release still make Feishu WebSocket inherit ambient proxy env through `resolveAmbientNodeProxyAgent()`, while the PR is open, unmerged, and paired with open bug #65799. The patch is focused, but it still needs the reviewed `OPENCLAW_FEISHU_WS_USE_PROXY` test cleanup and maintainer agreement on changing Feishu WebSocket proxying from ambient default to explicit opt-in.

## Best Possible Solution

Keep this PR and #65799 open. Update the PR so `OPENCLAW_FEISHU_WS_USE_PROXY` is captured, deleted before each test, and restored in cleanup, then have a Feishu/channel maintainer decide whether explicit opt-in is the desired default. If accepted, land it with targeted Feishu tests plus a small docs/changelog note for the new opt-in; if rejected, close the PR while leaving #65799 to track a different mitigation or diagnostic improvement.

## Risks / Open Questions

- The PR patch sets `OPENCLAW_FEISHU_WS_USE_PROXY=1` in tests without restoring or deleting it in the existing `afterEach`, so it can leak state across `--isolate=false` Vitest runs.
- Changing Feishu WebSocket proxying from ambient default to explicit opt-in can break deployments that rely on inherited proxy env for Feishu long connections.
- The same behavior change may weaken operator-managed WebSocket egress routing unless the new opt-in is documented and intentionally accepted for Feishu.
