## Summary

Keep open. Current main still does not implement the PR's central `google-vertex` runtime provider, ADC-to-Bearer exchange, or Vertex project/location request routing. Main has partial metadata/auth-marker handling and the Gemini thought-signature replay behavior, but `google-vertex` is still effectively handled through the generic Google provider path. The linked open issues #58775 and #60736 remain valid implementation targets, and this PR is still an active candidate rather than cleanup-closeable.

## Best Possible Solution

Keep this PR open for maintainer review, or replace it with a maintainer-owned implementation in the Google plugin. The implementation should give `google-vertex` exact provider ownership, remove the generic Google alias collision, resolve project/location/baseUrl deterministically, mint supported ADC credentials into Bearer auth with clear unsupported-credential errors, build the Vertex `/v1/projects/{project}/locations/{location}/publishers/google/models/...` URL, preserve the already-shipped Gemini thought-signature replay behavior, and add focused auth/routing tests plus user-facing docs. After that lands, let #58775 and #60736 close through the implementation.
