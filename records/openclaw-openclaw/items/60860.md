## Summary

Keep open. Current main still has only partial `google-vertex` metadata/auth-marker support; it does not implement the PR’s dedicated Vertex runtime provider, ADC-to-Bearer exchange, or Vertex project/location URL routing. The PR is also paired with open related work, including same-author #60736 and regression issue #58775, so it remains an implementation candidate rather than cleanup-close material.

## Best Possible Solution

Keep this PR open for maintainer review, or supersede it with a maintainer-owned Google plugin implementation that gives `google-vertex` exact provider ownership, deterministic project/location/baseUrl resolution, ADC-to-Bearer exchange with clear unsupported-credential errors, Vertex `/v1/projects/{project}/locations/{location}/publishers/google/models/...` routing, focused tests, and docs for the supported setup path. After that lands, close #60736 and #58775 through the implementation.
