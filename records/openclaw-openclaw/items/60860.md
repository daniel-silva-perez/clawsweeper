## Summary

Keep open. Current main has a partial Google Vertex path for authorized_user ADC, but it does not implement this PR's separate google-vertex provider ownership or service-account ADC support; the latest PR branch still has merge-blocking auth/proxy and credential-cache defects.

## What This Changes

The PR registers a separate google-vertex provider in the Google plugin, adds Vertex region/catalog/ADC helpers and tests, removes the legacy Google AI Studio alias, and routes Vertex requests through ADC-derived Bearer auth and Vertex project/location URLs.

## Best Possible Solution

Land or replace this with a maintainer-approved Google plugin implementation that gives google-vertex exact provider ownership, aligns selection-time and runtime ADC semantics, supports authorized_user and service_account credentials, preserves the injected fetch/proxy seam, keys token caching by credential identity, and adds focused routing/auth coverage plus user-facing setup docs.

## Reproduction Assessment

Yes for the PR findings. The branch source shows resolveGoogleVertexAdcToken() is called without the stream fetch option, and resolveGoogleVertexAdcToken returns a cached token before reading current ADC/env state.

## Solution Assessment

No, not as-is. The dedicated provider direction is likely the maintainable product boundary, but the branch should preserve fetch injection and credential-keyed token caching before it is safe to merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Pass the stream fetch into ADC refresh:** `extensions/google/transport-stream.ts:287`
  - body: The branch calls resolveGoogleVertexAdcToken() without passing the stream option's fetch implementation. Current main threads options.fetch into the OAuth token exchange, so proxy-aware callers and tests that supply a custom fetch now have Vertex ADC refresh fall back to global fetch instead.
  - confidence: 0.88
- **[P2] Key the ADC token cache by credential source:** `extensions/google/vertex-adc.ts:151-152`
  - body: resolveGoogleVertexAdcToken returns any unexpired cached token before reading the current ADC path, credential identity, or project env. In a long-running gateway, switching GOOGLE_APPLICATION_CREDENTIALS or credentials can reuse a token minted for the previous account until expiry.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The PR touches ADC token minting and has a concrete auth-isolation issue in the token cache.

Concerns:

- **[medium] Unkeyed ADC token cache can reuse prior credentials:** `extensions/google/vertex-adc.ts:151`
  - body: The new cache returns an unexpired token without checking the current credential path, refresh token, service account identity, or project env. A gateway that rotates or switches Google credentials can send Vertex requests using a previous account's token.
  - confidence: 0.84
