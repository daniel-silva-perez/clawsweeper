## Summary

Keep open. Current main still has only partial google-vertex metadata/auth-marker handling; it does not implement the PR's central dedicated Vertex runtime provider, ADC-to-Bearer exchange, or Vertex project/location URL routing. The PR is also an active implementation candidate for related open work, including #60736 and #58775, so conservative cleanup should not close it.

## Best Possible Solution

Keep this PR open for maintainer review, or replace it with a maintainer-owned Google plugin implementation that gives `google-vertex` exact provider ownership, removes the generic Google alias collision, resolves project/location/baseUrl deterministically, exchanges supported ADC credentials into Bearer auth with clear unsupported-credential errors, builds the Vertex `/v1/projects/{project}/locations/{location}/publishers/google/models/...` URL, preserves the already-shipped Gemini thought-signature replay behavior, and adds focused Google Vertex auth/routing tests plus user-facing docs. After that lands, let the linked open issues close through the implementation.
