## Summary

Keep this PR open. Current main still does not expose `after_tools_resolved` or an equivalent plugin-facing API for discovering the final per-attempt tool list and schemas. The linked feature issue points back to this PR, and the latest maintainer comment explicitly says to keep this PR open for follow-up.

## Best Possible Solution

Keep this PR open as the active implementation/design path. A maintainer-quality resolution should either land an additive resolved-tools hook with SDK baseline, docs, focused tests, and read-only schema handling, or explicitly supersede it with a pull-based resolved-tools API that safely exposes final per-attempt schemas.

## Risks / Open Questions

- The proposed hook would expose the full post-policy tool list and provider-facing schemas, including tools never called, to loaded plugins; maintainers should confirm this fits the trusted-plugin boundary.
- The PR body says `parameters` are passed by reference. If handlers receive schema objects before the provider/session is done with them, the implementation should clone, freeze, or otherwise prevent accidental mutation.
- Fire-and-forget per-attempt emissions can repeat on retries; the landed design should document retry/dedupe expectations or include a stable fingerprint.
