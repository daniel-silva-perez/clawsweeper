## Summary

Keep this PR open. Current main still does not implement `after_tools_resolved` or an equivalent plugin-facing API for discovering the final per-attempt tool list and schemas. The latest maintainer comment in the provided discussion also explicitly asks to keep this PR open for follow-up.

## Best Possible Solution

Keep this PR open as the active implementation/design path. A maintainer-quality resolution is to review and either land an additive resolved-tools hook with SDK baseline, docs, and focused tests, or explicitly supersede it with a pull-based `getResolvedTools` API that safely exposes final per-attempt schemas.

## Risks / Open Questions

- The proposed hook would expose full post-policy tool names and provider-facing parameter schemas to loaded plugins, including tools never called in a session; maintainers should confirm this fits the trusted-plugin boundary.
- The PR body says `parameters` are passed by reference. If hook handlers can mutate schema objects before provider/session use, that could become an integrity or behavior risk unless the implementation clones, freezes, or clearly constrains timing.
