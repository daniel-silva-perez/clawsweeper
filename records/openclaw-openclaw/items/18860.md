## Risks / Open Questions

- The proposed hook would expose the full post-policy tool list and provider-facing schemas, including tools never called, to loaded plugins; maintainers should confirm that fits the trusted-plugin boundary.
- The PR body and diff describe schema `parameters` as passed by reference. If handlers receive live provider-facing schema objects, accidental mutation could affect later session/provider behavior unless cloned, frozen, or clearly treated as read-only.
- Fire-and-forget per-attempt emissions can repeat on retries; the final design should document retry/dedupe expectations or provide a stable fingerprint.
- No supply-chain or CI expansion is visible in the provided PR file list; the security-sensitive part is API/data exposure rather than new third-party code execution.
