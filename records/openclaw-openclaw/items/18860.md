## Summary

Keep this PR open. Current main still lacks the requested `after_tools_resolved` hook and lacks an equivalent plugin-facing API for resolved tool/schema discovery. The newer operator-facing `tools.catalog` and `tools.effective` surfaces are useful inventory/status endpoints, but they do not expose JSON Schema parameters and are not a per-attempt plugin hook. The related feature request #21330 is closed, but the provided discussion identifies this PR as the active implementation path. Security review found no CI, dependency, lockfile, package-script, or third-party execution changes in the PR file list; the remaining concerns are API-boundary semantics.

## Best Possible Solution

Keep this PR open as the active implementation/design path for plugin-facing resolved tool/schema discovery. A maintainer-quality path is to finish review and either land the additive `after_tools_resolved` hook with docs, SDK baseline, and focused tests, or explicitly supersede it with a chosen pull-based resolved-tools API that exposes final per-attempt schemas safely.
