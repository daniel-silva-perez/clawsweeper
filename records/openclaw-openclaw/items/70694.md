## Summary

Keep open. Current main explains the very low visible first-turn input-token symptom for ordinary native spawns because the delegated task is intentionally single-sourced through `extraSystemPrompt`, but there is still a real unresolved path where `systemPromptOverride` can discard that delegated task. The narrower #73624 tracker remains open, and the related direct-fix PR was closed unmerged, so this broader report should stay available for maintainer consolidation rather than be swept as implemented.

## What This Changes

The issue reports that long native `sessions_spawn` subagent tasks can appear truncated or lost, producing very low child input-token counts and irrelevant subagent output.

## Best Possible Solution

Keep this report open until maintainers consolidate it with #73624 and decide the prompt-composition contract. The end state should prove that delegated `Your Role` task instructions reach subagents under ordinary embedded attempts, compaction retries, and CLI runner preparation, while preserving the #72019 single-source token behavior and documenting any changed override semantics.

## Reproduction Assessment

Unclear for the reporter's exact environment because the issue lacks trajectory/config evidence for the seven failed spawns. There is a high-confidence code-level reproduction for the narrower path: configure `systemPromptOverride`, spawn a native subagent, and inspect that the override branch returns before `extraSystemPrompt` is appended.

## Solution Assessment

No current implementation is the best solution yet. The narrow helper approach from the closed PR is plausible functionally, but maintainers need to decide whether runtime task instructions should compose with an override as system text or be delivered through a safer separate task segment/user-role contract.
