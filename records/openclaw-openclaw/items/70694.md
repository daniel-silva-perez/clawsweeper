## Summary

Keep this issue open. Current main intentionally moved the delegated task out of the first user turn and into the subagent system prompt, so the reported `in 8` token counter is not enough by itself to prove task loss. However, current main still has a related live failure path where `systemPromptOverride` can bypass the builder that appends subagent `extraSystemPrompt`, and the narrower open tracker #73624 covers that unresolved behavior. The reporter did not provide enough config/trajectory detail to prove this issue is exactly the same case, so it should stay open for maintainer triage rather than be closed as implemented or duplicate.

## Best Possible Solution

Keep this issue open while maintainers confirm whether the affected agent used `systemPromptOverride`. If yes, consolidate with #73624 and fix the prompt-composition path so overrides are combined with subagent `extraSystemPrompt` in embedded, compaction, and CLI runners. If not, ask for a raw trajectory/config excerpt showing the spawned runtime, context mode, `lightContext`, first child request, and token accounting so the broader delivery failure can be reproduced.
