## Summary

Keep open. Current `main` explains the reported 8 visible input tokens for normal native spawns because the task is now intentionally carried in `extraSystemPrompt`, but it still has a plausible unresolved task-loss path when `systemPromptOverride` is configured. The narrower tracker #73624 remains open, and the attempted PR #73637 was closed unmerged after prompt-authority/security concerns, so this needs maintainer design or consolidation rather than cleanup closure.

## What This Changes

The issue reports that long `sessions_spawn` native subagent task prompts can be lost or truncated, producing very low child input-token counts and irrelevant subagent output.

## Best Possible Solution

Define a safe prompt-composition contract for runtime subagent task instructions under `systemPromptOverride`, then implement it consistently in embedded attempts, compaction retries, and CLI runner preparation. The end state should prove that override text and delegated task instructions both reach the child without weakening operator-defined prompt authority, with docs updated if override semantics change.

## Reproduction Assessment

Unclear for the reporter's exact environment because the issue lacks config/trajectory proof. There is a high-confidence code-level reproduction for the narrower #73624 path: configure a target agent with `systemPromptOverride`, spawn a native subagent, and inspect the child compiled system prompt for the missing `## Your Role` task block.

## Solution Assessment

No current implementation is the best solution yet. The direct append approach from PR #73637 was plausible functionally but raised prompt-injection/authority concerns; a safer fix likely needs an explicit runtime task segment or separate lower-priority/user-role delivery contract across providers.
