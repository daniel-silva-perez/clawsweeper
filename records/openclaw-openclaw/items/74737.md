The branch changes Discord config accessor resolution to read allowFrom/defaultTo from normalized merged account config without resolving the token, and adds a SecretRef regression test for those accessors.

## Best Possible Solution

Land a narrow fix that keeps Discord read-only config accessors on merged config data for allowFrom/defaultTo and reserves token SecretRef resolution for runtime paths that actually need a bot token.
