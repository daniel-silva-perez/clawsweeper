## Summary

Keep open. Current main now covers the reporter’s concrete `multimedia.nt.qq.com.cn` example through the default `*.qq.com.cn` QQ Bot media allowlist, but it still does not provide or wire a supported `channels.qqbot.mediaAllowHosts` setting. The remaining request is valid because QQ Bot media downloads still use a fixed SSRF policy, while the comparable Microsoft Teams channel has a configurable media host allowlist.

## What This Changes

Add a QQ Bot `channels.qqbot.mediaAllowHosts` configuration option that merges operator-specified hostnames with the default QQ Bot media SSRF allowlist.

## Best Possible Solution

Implement this inside the bundled QQ Bot plugin by adding `mediaAllowHosts` to the QQ Bot account/channel config type, Zod schema, plugin manifest schema, generated metadata, and docs; normalize configured host patterns through the existing SSRF policy semantics; merge them with the conservative QQ/Tencent defaults; and thread the resulting policy into inbound attachment downloads and outbound URL fallback downloads while preserving HTTPS-only and private-network protections.
