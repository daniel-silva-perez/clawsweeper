## Summary

Keep this PR open. Current main still normalizes cron webhook URLs with string trimming, URL parsing, and http(s) protocol validation only, so private/internal literal hosts can still be saved in cron delivery config. Runtime delivery is still protected by the SSRF-guarded fetch layer, but the PR's pre-persistence defense-in-depth change is not implemented on main. The review feedback in the PR was also addressed by the author with extra edge-case tests, and the supplied diff is limited to the relevant cron normalizer and tests.

## Best Possible Solution

Review and land this focused PR, or reimplement the same fix on main: call `isBlockedHostnameOrIp(parsed.hostname)` inside `normalizeHttpWebhookUrl`, add direct normalizer tests for reserved hostnames, private/special-use IPs, `0.0.0.0`, and IPv4-mapped IPv6 literals, and keep `fetchWithSsrFGuard` as the DNS-aware runtime guard.
