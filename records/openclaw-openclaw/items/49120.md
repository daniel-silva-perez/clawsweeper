## Summary

Keep this PR open. Current main still normalizes cron webhook URLs with only trimming, URL parsing, and http(s) protocol validation, so private/internal literal hosts can still be persisted in cron delivery config. Runtime dispatch remains protected by `fetchWithSsrFGuard`, but that does not implement the PR's pre-persistence hardening goal.

## Best Possible Solution

Review and land this focused PR, or reimplement the same fix on main: call `isBlockedHostnameOrIp(parsed.hostname)` inside `normalizeHttpWebhookUrl`, add direct normalizer tests for reserved hostnames, private/special-use IPs, `0.0.0.0`, and IPv4-mapped IPv6 literals, and keep `fetchWithSsrFGuard` as the DNS-aware runtime guard.
