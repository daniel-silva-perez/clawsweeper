## Summary

Keep this PR open. Current main still lacks the missing-MIME image fallback and still serializes attachments from the data URL MIME rather than the resolved attachment MIME. The live PR is the active implementation candidate for the linked Control UI drag/drop bug and has already received maintainer repair/validation context; it should not be closed before merge.

## Best Possible Solution

Keep this PR open for maintainer review and landing. Rebase or apply the focused fix onto current main by adding a missing-MIME resolver in ui/src/ui/chat/attachment-support.ts, using it for selected/dropped image files in ui/src/ui/views/chat.ts, preserving the resolved MIME through payload-store/send serialization in ui/src/ui/controllers/chat.ts, and keeping the regression tests for extension fallback plus data URL serialization. Then run the targeted Control UI tests and the changed gate before merge.
