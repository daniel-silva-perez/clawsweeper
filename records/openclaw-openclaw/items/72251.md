## Summary

Keep this draft PR open. Current main still performs a single `tar.c(...)` write for backup creation and lacks the PR's volatile-file filter, EOF retry/backoff loop, enriched final EOF diagnostics, logger plumbing, and `skippedVolatileCount` result/summary/JSON field. The PR is a focused implementation attempt for the live-backup EOF race from #72249, and the author addressed the bot review findings, but it needs maintainer review and rebase work before it can safely land.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase it onto current main, preserve installed-plugin `node_modules` exclusion, keep `--json` stdout pure, decide and document the default volatile-skip semantics, retain focused retry/filter tests, and coordinate explicitly with #67417/#67472 and #67990 rather than treating the broader backup design as complete.
