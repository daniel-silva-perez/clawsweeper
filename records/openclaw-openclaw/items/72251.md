## Summary

Keep this PR open. Current main still lacks the volatile backup filter, tar EOF retry/backoff, enriched final EOF diagnostics, and skipped-volatile reporting proposed here; the active branch is a focused implementation for a real backup reliability issue, but it still needs maintainer review for backup-content semantics, JSON-output cleanliness, docs/changelog coverage, and coordination with the related ENOENT and broader exclusion-design work.
