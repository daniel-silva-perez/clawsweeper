## Summary

Keep open. Current main does not implement the EOF-race mitigation, and this PR is still useful, but the current diff misses OpenClaw's actual per-agent session transcript path and cron run-log extension while also sending non-fatal backup notices to stdout in JSON mode.

## What This Changes

The PR adds a volatile backup-path filter, tar EOF retry/backoff with cleanup and enriched errors, `skippedVolatileCount` reporting, command logger plumbing, and focused backup retry/filter tests.

## Best Possible Solution

Revise this PR so the volatile filter covers OpenClaw's actual `agents/<agentId>/sessions/*.jsonl` and `cron/runs/*.jsonl` paths, keep JSON output machine-readable, add docs/changelog coverage for the new backup semantics, then merge it as the narrow EOF-race fix while leaving the broader ENOENT/exclusion-design work to its linked issues.

## Reproduction Assessment

Yes. The PR-introduced path bug is reproducible by calling `isVolatileBackupPath` for `{stateDir}/agents/main/sessions/live.jsonl` or `{stateDir}/cron/runs/job.jsonl`; the JSON regression is reproducible when `openclaw backup create --json` emits a retry or volatile-skip notice before the JSON payload.

## Solution Assessment

No, not as currently written. Retry-plus-filter is a plausible narrow fix for the tar EOF race, but the filter needs to match current storage contracts and the command must suppress or reroute non-fatal notices in JSON mode before this is the best mergeable solution.
