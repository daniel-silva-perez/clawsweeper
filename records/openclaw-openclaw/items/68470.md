## Summary

Keep open. Current main still does not normalize the MiniMax cache-hit field described in the report into the shared usage contract, while timeout compaction still relies on additive prompt-token math; the linked implementation candidate remains open and should be reviewed or replaced before this issue closes.

## What This Changes

The issue requests fixing MiniMax prompt-cache usage normalization so cached prompt tokens are not counted twice in context usage and compaction decisions.

## Best Possible Solution

Keep this issue open until #68750 or an equivalent narrow fix lands; the fix should normalize MiniMax cache-hit usage at the shared usage boundary and add focused regression coverage for the MiniMax payload shape.

## Reproduction Assessment

Yes for a focused source-level reproduction: pass a MiniMax-style prompt usage shape through `normalizeUsage` and then `derivePromptTokens`, and verify cached prompt tokens are de-duplicated before the timeout-compaction ratio is computed. A live MiniMax run was not needed for the keep-open decision because current code and tests still lack the reported field handling.

## Solution Assessment

Yes, the narrowest maintainable solution is to fix the shared usage normalization contract in `src/agents/usage.ts` and cover it in `src/agents/usage.test.ts`, rather than changing compaction thresholds or disabling memory flush behavior.
