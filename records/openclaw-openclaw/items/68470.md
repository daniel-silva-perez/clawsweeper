## Summary

Keep open. Current main still does not recognize MiniMax's `prompt_cache_hit_tokens` usage field, the timeout-compaction path still depends on derived prompt usage, and #68750 is an open implementation candidate that links this issue with closing syntax.

## Best Possible Solution

Keep this issue open while maintainers review #68750 or land an equivalent fix. The likely fix belongs in `src/agents/usage.ts`: add MiniMax `prompt_cache_hit_tokens` to the usage shape, cache-read extraction, OpenAI-style prompt-total de-duplication, and focused regression coverage in `src/agents/usage.test.ts`; after that PR or an equivalent patch merges, GitHub can close this issue as implemented.
