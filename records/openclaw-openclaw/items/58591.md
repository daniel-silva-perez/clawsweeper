## Summary

Keep this PR open. Current main still does not pass agent-scoped `mediaLocalRoots` through the Feishu reply attachment path, while the PR is a narrow, well-targeted implementation with focused test coverage and no concrete security or supply-chain concern found.

## What This Changes

The PR imports the plugin SDK media-root helper in the Feishu reply dispatcher, forwards agent-scoped `mediaLocalRoots` into reply attachment sends, and adds a unit test for that forwarding.

## Best Possible Solution

Land this PR or an equivalent canonical Feishu fix that computes agent-scoped media roots in the reply dispatcher, forwards them to `sendMediaFeishu`, keeps the local-file sandbox scoped to the active agent workspace/default media roots, and then close or supersede the overlapping Feishu media-root PRs.

## Reproduction Assessment

Yes. A focused seam reproduction is to create a Feishu reply dispatcher on current main, deliver a local `mediaUrl` under an agent workspace, and observe that `sendMediaFeishu` receives no `mediaLocalRoots`; the author also reported an end-to-end `LocalMediaAccessError` for `MEDIA:/...` reply attachments before the patch.

## Solution Assessment

Yes. Using the existing plugin SDK `getAgentScopedMediaLocalRoots(cfg, agentId)` helper in `sendMediaReplies` is the narrowest maintainable fix for workspace-scoped reply attachments and avoids introducing a new Feishu-only config path.
