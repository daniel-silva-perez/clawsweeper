## Summary

Keep this PR open. Current main still rejects Telegram `dmPolicy="open"` unless the effective `allowFrom` already includes `"*"`, and validation returns before runtime materialization. Main has an `openclaw doctor --fix` repair path, but not the requested startup/pre-validation auto-heal. Since current main also tightened open-DM allowlist semantics, auto-inserting a wildcard at startup remains a security/product decision rather than an obsolete PR.

## Best Possible Solution

Keep this PR open for maintainer security/product judgment against current main. If startup recovery is still wanted, rework it narrowly around current `open` semantics: only mutate the intended top-level channel or account, never widen sibling inherited allowlists, avoid order-dependent warning state, cover missing/empty/non-wildcard/inherited account cases, and update docs/changelog. If maintainers prefer the current explicit contract, close this PR intentionally and keep `openclaw doctor --fix` plus setup/config writes as the supported repair path.
