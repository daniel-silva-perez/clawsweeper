## Summary

Keep this issue open. Current main supports thinking effort and reasoning visibility, including `/reasoning on|off|stream`, but I found no shipped config, session field, UI control, or docs surface for choosing the language of visible reasoning output. The requested `agents.defaults.reasoningLanguage`-style capability remains unimplemented.

## Best Possible Solution

Keep this open for product/design implementation. A good fix would add a documented language preference, likely under agent defaults plus optional per-agent/session override, wire it through config schema/types, Control UI and session APIs, and the agent runtime prompt/provider request path, with tests proving final response language remains independent from visible reasoning language.

## Risks / Open Questions

- Provider-native reasoning summaries and hidden reasoning fields may not all obey a prompt-level language instruction, so a fix likely needs provider-aware behavior and clear docs.
- Reasoning visibility is security-sensitive; any implementation should preserve existing hidden-by-default behavior and avoid broadening reasoning exposure in group or channel surfaces.
