## Summary

Keep this PR open. Current main still renders the Slack app manifest JSON inside the Slack setup intro note, so the copy/paste bug remains, but this PR's raw console.log/prepare approach needs revision because it bypasses wizard-session delivery and does not preserve the env-token shortcut behavior called out in review.

## Best Possible Solution

Keep this PR open, but request a revised implementation. The fix should make the Slack manifest copyable outside the framed clack note while preserving wizard/session delivery and env-shortcut suppression, likely by extending the setup wizard/prompter surface with a generic copyable/plain output step rather than printing directly to stdout.
