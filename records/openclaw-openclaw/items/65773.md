## Summary

Keep this PR open. Current main still embeds the Slack app manifest JSON in the Slack setup intro note, so the underlying copy/paste bug remains. The PR is plausible but should be revised before merge because its raw console.log prepare hook bypasses wizard-session delivery and can change env-shortcut behavior.

## Best Possible Solution

Keep this PR open, but request a revised implementation. The fix should make the Slack manifest copyable outside the framed clack note while preserving wizard/session delivery and env-shortcut suppression, likely through a generic wizard/prompter plain or copyable output step rather than a direct console.log in Slack prepare.
