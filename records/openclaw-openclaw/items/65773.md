## Summary

Keep this PR open. Current main still embeds the Slack app manifest JSON in the Slack setup intro note, so the underlying copy/paste bug remains. The PR is useful directionally, but as written its raw `console.log` prepare hook can bypass wizard-session delivery, can change env-shortcut behavior, and appears to need a rebase onto the current `setup-shared.ts` layout.

## What This Changes

The PR branch exports the Slack manifest builder, removes the manifest body from Slack setup `introNote.lines`, and adds a Slack setup `prepare` hook that prints the raw manifest JSON.

## Best Possible Solution

Keep this PR open and request a revised implementation. The fix should emit a copyable/plain Slack manifest through the wizard/prompter contract so both CLI and gateway/UI setup clients receive it, preserve env-shortcut suppression, add focused regression coverage, and rebase onto the current `extensions/slack/src/setup-shared.ts` layout.
