## Summary

Keep this PR open. Current main still lacks the requested per-model probe flag, and the branch implements the right core direction, but it still has a small typed-output gap and should be refreshed against recent model-status auth-store work before merge.

## What This Changes

The branch adds status-level `--all` plumbing for `openclaw models status --probe`, expands probe targets across all configured candidate models, warns when `--all` is used without `--probe`, and adds route/parser plus probe-target tests.

## Best Possible Solution

Land a refreshed version that preserves the new per-model probe behavior, exposes `probeAll` in the exported summary type, updates the public `models status` docs/help surface, and rebases cleanly over current model-status auth-store changes while keeping default `--probe` behavior unchanged.
