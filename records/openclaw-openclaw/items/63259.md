## Summary

Keep this PR open. Current main still has no `openclaw models status --probe --all` implementation and continues to select one representative model per provider, so this remains the active implementation path for open feature #63145; the branch should fix the remaining exported type-contract issue before merge.

## What This Changes

The PR adds `--all` plumbing for `openclaw models status --probe`, expands auth probe targets to every configured candidate model, warns when `--all` is used without `--probe`, and adds focused parser/probe-target tests.

## Best Possible Solution

Land a refreshed version of this PR after aligning the exported probe summary type and public docs with the new flag, while preserving the existing one-model-per-provider behavior when `--all` is omitted.
