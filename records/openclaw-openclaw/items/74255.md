## Summary

Keep open. Current main still lacks the proposed prompt-authority contract and still lets the embedded runner pass pre-assembly context-engine history into the hard preemptive overflow check after assembled messages are applied. This remains a useful implementation PR for the open context-engine overflow bug, but it needs maintainer review of the plugin-facing contract and follow-up on failing head CI before merge.

## What This Changes

The PR adds optional `AssembleResult.promptAuthority`, defaults assembled context as the precheck authority in `runEmbeddedAttempt`, preserves pre-assembly overflow gating only for explicit opt-in engines, and adds/updates embedded runner regression tests.

## Best Possible Solution

Keep the PR open for maintainer review. If the prompt-authority contract is accepted, land the narrow host-side fix with the regression tests, document the plugin-facing default and opt-in behavior, then close #74233 after merge while leaving the already-closed #74247 superseded by this branch.
