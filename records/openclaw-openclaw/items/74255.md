## Summary

Keep open. Current main still has the reported raw-history precheck behavior, and this PR is a narrow implementation candidate rather than something already implemented or superseded. It needs maintainer review of the plugin-facing `AssembleResult` contract, a rebase because the PR is currently dirty against main, and CI follow-up before merge.

## What This Changes

The PR adds an optional `AssembleResult.promptAuthority` field, defaults embedded-runner overflow prechecks to assembled context, preserves raw preassembly gating only for explicit opt-in engines, and adds regressions for both behaviors.

## Best Possible Solution

Keep this PR open, rebase it onto current main, document the new prompt-authority default and opt-in in the context-engine docs/API surface, rerun the targeted runner tests plus plugin SDK strict smoke, and land it if maintainers accept the contract. After merge, close #74233 and leave #74247 superseded by this branch.

## Reproduction Assessment

Yes. Current-main source inspection gives a high-confidence reproduction path: a context engine can return a small assembled prompt while the runner still supplies the larger preassembly snapshot to the precheck, whose helper chooses the larger estimate; the PR’s new tests model that exact huge-history/small-assembled prompt shape.

## Solution Assessment

Mostly yes. Defaulting to assembled prompt authority while preserving explicit `preassembly_may_overflow` is narrower than removing the raw-history safeguard entirely, but the plugin-facing contract should be documented and maintainer-approved before merge.
