## Summary

Keep this open. Current main still uses a fixed 150 ms gateway chat-delta throttle, the requested resolver/env var is absent, and the linked implementation PR is closed unmerged. This is a valid narrow feature request, but it changes an operator-facing runtime knob, so it needs maintainer product/API review before an automated fix lane should implement it.

## What This Changes

Request to make the gateway chat-delta broadcast throttle configurable through `OPENCLAW_CHAT_DELTA_THROTTLE_MS` while preserving the 150 ms default and honoring `0` as throttle-disabled.

## Best Possible Solution

If maintainers accept the surface, add a small gateway-owned throttle resolver with the 150 ms default, apply it only to the shared chat-delta broadcast path, update the fixed-window comments/tests, and document or log the knob only to the extent OpenClaw wants to support it as an operator control.
