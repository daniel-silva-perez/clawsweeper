## Summary

Keep this issue open. Current main still hardcodes the gateway chat-delta broadcast throttle at 150 ms, has tests/comments that assume that fixed window, and has no `OPENCLAW_CHAT_DELTA_THROTTLE_MS` resolver or docs/config surface. The linked implementation PR #74152 was closed unmerged after the reporter filed this issue first, while #38066 is Telegram-specific and does not cover the shared gateway WebSocket chat broadcast path.

## Best Possible Solution

Keep this issue open for maintainer product/API review. If accepted, implement a focused gateway change that preserves the 150 ms default, centralizes throttle resolution, validates a non-negative `OPENCLAW_CHAT_DELTA_THROTTLE_MS` override if env is the approved surface, updates the current comments/tests, and adds docs only if maintainers want this treated as a supported operator knob.
