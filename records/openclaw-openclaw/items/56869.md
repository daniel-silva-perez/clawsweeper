## Summary

Keep open. The contribution is useful, but the current patch is not mergeable as written because the proposed `NO_PROXY` examples include CIDR ranges that current OpenClaw and undici env-proxy matching do not support, so users would get misleading Docker proxy behavior.

## What This Changes

The PR adds commented Docker Compose proxy environment examples and a Docker install docs section for configuring outbound proxy variables in container deployments.

## Best Possible Solution

Revise or replace this PR with Docker proxy guidance that matches current main: remove unsupported CIDR `NO_PROXY` examples, describe only supported bypass patterns if raw env vars are shown, acknowledge the bundled Compose file already maps `host.docker.internal`, and link or summarize managed proxy configuration for broader runtime routing.

## Reproduction Assessment

Yes. A high-confidence reproduction path exists by comparing the proposed `NO_PROXY` value with current OpenClaw `matchesNoProxy` behavior and undici v8.1.0 `EnvHttpProxyAgent` source; no live Docker proxy smoke is required to prove the CIDR mismatch.

## Solution Assessment

No as written. The narrow maintainable fix is to keep the Docker docs addition but make the examples accurate against current proxy semantics and current Compose defaults, preferably cross-linking the managed network proxy docs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Remove unsupported CIDR entries from NO_PROXY examples:** `docker-compose.yml:17`
  - body: The added `NO_PROXY` value includes CIDR ranges, but OpenClaw's env-proxy path follows undici matching semantics: entries are exact hosts or domain suffixes with optional ports, not IP ranges. Users copying this example will expect private-network IPs to bypass the proxy, but those CIDR entries are never matched and traffic can still be sent through the proxy. The same value is repeated in the Docker install docs and should be fixed there too.
  - confidence: 0.94
- **[P3] Align the extra_hosts note with current Compose defaults:** `docs/install/docker.md:153-156`
  - body: Current main already includes `extra_hosts: ["host.docker.internal:host-gateway"]` for the gateway service. The new docs tell Linux users to add that block unconditionally, which can lead readers on current Compose to add a duplicate service key instead of recognizing the bundled file already handles it.
  - confidence: 0.82
