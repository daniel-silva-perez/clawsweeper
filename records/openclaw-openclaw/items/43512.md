## Summary

Current main still does not implement this PR's intended broad Feishu control-command fast path: abort-like text and `/btw` have separate lanes, but `/status`, `/new`, `/reset`, `/help`, `/model`, and other command-only messages still flow through the debouncer and then the normal per-chat sequentializer. The linked Feishu regression remains valid, but the branch is stale against the refactored Feishu handler split and has blocking ordering/authorization concerns in the proposed patch.

## What This Changes

The PR adds a Feishu receive-handler branch that detects command-only text before debouncing, dispatches those commands directly, drops pending debounce buffers for destructive commands, and extends the shared inbound debouncer with a `dropKey` API.

## Best Possible Solution

Keep this PR open as a useful implementation candidate for #42803, but rework it onto the current `createFeishuMessageReceiveHandler` and `getFeishuSequentialKey` design. The end state should define exactly which Feishu command classes bypass the base lane, preserve or explicitly document ordering for buffered text, avoid unauthorized destructive side effects, and add focused Feishu tests for `/stop`, `/new`, `/reset`, `/status`, and read-only command behavior during active runs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Clear queued chat work before destructive commands:** `extensions/feishu/src/monitor.account.ts:441`
  - body: This direct-dispatch path drops only the debounce buffer before running `/stop`, `/new`, or `/reset`. Any normal Feishu message that has already flushed into the per-chat queue still runs after the destructive command, so stale text can restart work immediately after a stop/reset and defeat the command's intent.
  - confidence: 0.84
- **[P2] Gate destructive drops on command authorization:** `extensions/feishu/src/monitor.account.ts:438`
  - body: `dropKey` runs from text classification before `handleFeishuMessage` computes `CommandAuthorized`. In a group where the sender can chat but cannot run destructive slash commands, an unauthorized `/new`, `/reset`, or `/stop` can still discard that sender's pending debounced text.
  - confidence: 0.8
