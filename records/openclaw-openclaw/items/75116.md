## Summary

Keep open. Current main still lacks the requested per-server caller-context injection, and the linked open PR #75118 is the active implementation candidate that should be reviewed before this issue is resolved.

## What This Changes

Add a default-off `mcp.servers.<name>.injectCallerContext` option for remote bundled MCP servers so OpenClaw can merge caller identity/session headers without overwriting user headers.

## Best Possible Solution

Review PR #75118 for the trust model and implementation details, then land a core bundle-MCP/config change if accepted: remote-only, per-server opt-in, default off, non-overwriting header merge, stripped downstream control flag, placeholder env support for opted-in layers, and matching schema/docs/tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure a remote `mcp.servers.<name>.url` with the proposed flag and inspect the generated bundle MCP config; current main has no explicit schema/type support, no merge-time header injection, and loopback-only placeholder env population.

## Solution Assessment

Yes, pending maintainer approval. A per-server default-off flag is the narrowest maintainable direction for this trust boundary; a global flag would create broader accidental caller-identity disclosure risk.
