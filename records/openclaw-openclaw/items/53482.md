## Summary

Keep this issue open. Current main still lacks a gateway startup/runtime-auth warning for legacy `CLAWDBOT_*` or `MOLTBOT_*` environment variables, and PR #53667 remains the open implementation candidate linked to this issue.

## Best Possible Solution

Keep this issue open and review, land, or replace PR #53667. The best fix belongs in the gateway startup/runtime-auth and doctor diagnostics path: detect `CLAWDBOT_*` and `MOLTBOT_*` keys early, warn loudly with accurate current-name or setup guidance, improve misleading auth/channel errors, and add focused regression tests while preserving the intentional removal of legacy fallback support.
