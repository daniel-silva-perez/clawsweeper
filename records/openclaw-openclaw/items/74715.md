## Summary

Keep open. Current main still has the virtual Notifications section wired into the Communication render path, but the generated Communication navigation metadata still omits `__notifications__`, so the user-facing tab remains unreachable without this PR or the overlapping open #73894 work.

## What This Changes

The PR adds a `__notifications__` Communication navigation entry with a bell icon and a browser regression test for schema-less virtual Notifications tabs.

## Best Possible Solution

Keep the virtual Notifications section represented in the Communication settings navigation with focused browser coverage; either merge this narrow patch or carry the same behavior through #73894, without introducing schema migration or broader notification-product changes here.

## Reproduction Assessment

Yes. On current main, render Communication settings with `includeSections` containing `__notifications__`, `includeVirtualSections: true`, and a schema without that key; because Communication category metadata omits the virtual key, the generated top tabs cannot include Notifications.

## Solution Assessment

Yes. Adding `__notifications__` to `SECTION_CATEGORIES` with a matching icon is the narrowest maintainable fix for reachability, and the added browser test covers the schema-less virtual-section case; the remaining question is only PR coordination with #73894.
