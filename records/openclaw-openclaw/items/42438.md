## Summary

Keep this open. Current main still only covers the silent-block half through allowlist mode; it does not add a blocked-DM owner notification path, a webhook policy for blocked attempts, or a configurable way to suppress live pairing replies while still recording or notifying the owner. The related #64081 remains adjacent task-notification work, not channel DM access control.

## What This Changes

Request an opt-in silent/notify policy or config for unauthorized DMs so strangers receive no reply while the owner can learn about blocked contact attempts.

## Best Possible Solution

Design this as an additive channel access-control feature: keep allowlist as the silent block policy, add an opt-in blocked-DM owner notification event/config with metadata-minimal payloads, and treat pairing-reply suppression as a separate explicit option if maintainers want that behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and test inspection give a high-confidence current-behavior check: allowlist blocks without an outgoing message, while live pairing-mode WhatsApp DMs still send a pairing reply and no blocked-attempt notification path exists.

## Solution Assessment

No proposed patch is under review. The requested direction is valid, but the maintainable solution should likely be an additive notification/suppression config rather than unreviewed new dmPolicy values across all channels.
