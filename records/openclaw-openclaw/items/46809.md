## Summary

Keep open. Current main still has only the generic Agents > Files editor for bootstrap files and root MEMORY.md; it has no dedicated Memory panel, no /memory Control UI route, and no editable daily memory file surface. The related memory-view PR #73773 is open and unmerged, so it is useful context but not shipped behavior.

## What This Changes

Add a dedicated Control UI memory panel that displays and edits USER.md, MEMORY.md, and today's memory/YYYY-MM-DD.md for the selected agent.

## Best Possible Solution

Expose USER.md, root MEMORY.md, and today's memory/YYYY-MM-DD.md through a scoped Gateway API and Control UI memory surface with strict path/date validation, UI coverage, gateway path-safety tests, docs, and a user-facing changelog entry; #73773 is the current candidate to review or narrow.
