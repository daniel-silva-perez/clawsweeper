## Summary

Keep open. Current main still only partially overlaps through the generic Agents > Files editor for bootstrap files and root MEMORY.md; it has no dedicated Control UI Memory panel, no /memory route, and no editable daily memory note surface. The related #73773 implementation is useful context, but it remains open and unmerged, so it is not shipped behavior.

## What This Changes

Add a dedicated Control UI memory panel that displays and edits USER.md, MEMORY.md, and today's memory/YYYY-MM-DD.md file.

## Best Possible Solution

Add a scoped Gateway memory-files API and Control UI Memory surface for USER.md, root MEMORY.md, and today's memory/YYYY-MM-DD.md, with strict workspace-bound path/date validation, save behavior, UI coverage, gateway safety tests, docs, and a user-facing changelog entry.

## Solution Assessment

No; the issue's UI-only framing is not the safest complete solution. The maintainable path is a narrow scoped Gateway API plus the Control UI view, rather than reusing the generic bootstrap-file editor for arbitrary daily paths.
