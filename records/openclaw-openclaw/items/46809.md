## Summary

Keep open. Current main still has only the generic Agents > Files editor for bootstrap/core workspace files plus root `MEMORY.md`; it does not provide a dedicated Memory panel or an editable surface for today's `memory/YYYY-MM-DD.md`. A related PR (#73773) proposes a broader memory management view, but it remains open/unmerged in the provided context and is not present in this checkout.

## What This Changes

Add a dedicated Control UI memory panel that displays and edits `MEMORY.md`, `USER.md`, and today's daily `memory/YYYY-MM-DD.md` file.

## Best Possible Solution

Keep this issue open. The best path is maintainer review of #73773 versus a narrower Agents > Files extension; either implementation should expose `USER.md`, root `MEMORY.md`, and today's `memory/YYYY-MM-DD.md` for the selected agent through a tightly validated Gateway API, with Control UI tests, gateway path-safety tests, docs, and a user-facing changelog entry.
