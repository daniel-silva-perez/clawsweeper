## Summary

Keep open. Current main still only has a generic Agents > Files editor for root workspace files; it does not provide the requested dedicated Memory panel or a read/write surface for today's `memory/YYYY-MM-DD.md`. A related open PR (#73773) adds a broader `/memory` view, but it is not merged and needs maintainer review before it can be treated as the canonical path.

## Best Possible Solution

Keep this issue open. The best path is for a maintainer to decide whether #73773 is the intended memory-management direction or whether this should be a narrower extension of Agents > Files. A focused implementation would expose `USER.md`, root `MEMORY.md`, and today's `memory/YYYY-MM-DD.md` for the selected agent through a tightly validated gateway API, with Control UI tests, gateway path-safety tests, docs, and a user-facing changelog entry.
