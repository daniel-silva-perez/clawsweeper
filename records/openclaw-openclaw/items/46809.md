## Summary

Keep open. Current main only partially overlaps through the generic Agents > Files editor for bootstrap files and root MEMORY.md; it still lacks a dedicated Control UI Memory panel, a /memory route, and editable daily memory note support. The related #73773 implementation remains open and unmerged, so it is useful context but not shipped behavior.

## What This Changes

Add a dedicated Control UI memory panel that can display and edit USER.md, MEMORY.md, and the current daily memory note under memory/YYYY-MM-DD.md.

## Best Possible Solution

Add a scoped Gateway memory-files API plus a Control UI Memory surface for USER.md, root MEMORY.md, and today's memory/YYYY-MM-DD.md, with strict workspace-bound path/date validation, save behavior, UI coverage, gateway safety tests, docs, and a user-facing changelog entry.

Not applicable as a bug reproduction; this is a feature request. The missing surface is high-confidence from source inspection: current main has no /memory route or Memory panel, and agents.files does not expose daily memory notes.

## Solution Assessment

No; the issue's UI-only framing is not the safest complete solution. The maintainable direction is a narrow scoped Gateway API plus the Control UI view, rather than reusing the generic bootstrap-file editor for arbitrary daily paths.
