## Summary

Keep open. This issue is now the canonical tracker for the macOS gateway/LaunchAgent exec reachability failure: the maintainer discussion says current main has adjacent exec policy/doc changes but no proven fix, and a newer LAN reachability report was closed as a duplicate of this issue. Current main still routes gateway exec through the upper env/policy/allowlist layer before `runExecProcess`, and I found no source, docs, tests, or changelog marker proving the SSH/LAN `No route to host` symptom is fixed.

## Best Possible Solution

If reproduced, fix the shared gateway exec/service environment or LaunchAgent network-context path and add the narrowest regression coverage around that boundary.
