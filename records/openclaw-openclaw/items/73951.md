## Summary

Keep this issue open. Current main still allows Feishu inbound `thread_id`/`root_id` metadata to override `replyInThread: "disabled"`; the reporter's compiled-file/string-truthiness diagnosis is not the current TypeScript root cause, but the observable disabled-thread behavior is still real. Open PR #39507 already targets this behavior and should be reviewed/rebased rather than queueing a separate automated fix.

## Best Possible Solution

Keep this issue open and review/rebase PR #39507 or an equivalent maintainer patch. If `replyInThread: "disabled"` is intended as a hard off switch, update `resolveFeishuGroupSession` and the dispatcher path so inbound `thread_id`/`root_id` cannot force `reply_in_thread: true` in non-topic groups, while preserving explicit `"enabled"` and topic-session behavior. Update the current regression test that expects disabled config to force thread replies.
