## Summary

Keep this issue open. Current `origin/main` still lets Feishu inbound thread metadata override `replyInThread: "disabled"`: the resolver turns any `thread_id`/`root_id` into `threadReply`, the dispatcher converts that into an effective thread reply, and the send helper then emits `reply_in_thread: true`. The reporter's exact compiled-file/string-truthiness diagnosis does not match the current TypeScript boundary, but the observable bug is still real and is already overlapping with open Feishu PRs #39507 and #40147.

## Best Possible Solution

Keep the issue open and consolidate it with Feishu PRs #39507 and #40147. Maintainers should decide the intended semantics for `replyInThread: "disabled"`; if disabled is a hard off switch, update `extensions/feishu/src/bot-content.ts` and `extensions/feishu/src/reply-dispatcher.ts`, preserve explicit enabled/topic-session behavior where intended, and update the current regression tests that expect disabled config to force thread replies.
