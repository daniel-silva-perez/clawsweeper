## Summary

Keep open. Current main still allows Feishu inbound `thread_id` or `root_id` metadata to override `replyInThread: "disabled"`, so the observable bug remains even though the reporter's compiled-file truthiness diagnosis is not the current TypeScript boundary. Open PR #39507 appears to target this behavior but is still unmerged, and closed PR #40147 did not land.

## What This Changes

The issue asks Feishu to honor `replyInThread: "disabled"` by not sending thread replies or `reply_in_thread: true` for quote/thread-originated group messages.

## Best Possible Solution

Review and land #39507 or an equivalent narrow Feishu fix that makes `replyInThread: "disabled"` suppress thread mode at the resolver/dispatcher boundary, while preserving explicitly enabled topic-thread behavior and updating the current regression tests to match the intended contract.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists in `extensions/feishu/src/bot.test.ts`: a group config with `replyInThread: "disabled"` plus an inbound `thread_id` currently expects `replyInThread: true`, and the source path shows that value becomes `reply_in_thread: true` in the Feishu API payload.

## Solution Assessment

No for current main; the behavior still contradicts the disabled-mode contract. The reporter's literal compiled-file fix is not the best current patch because TypeScript now passes a boolean into send helpers, so the maintainable fix belongs in `bot-content.ts` and/or `reply-dispatcher.ts` with matching tests.
