## Summary

Keep this issue open. Current main still lacks a supported cron job tags/categories contract across persistence, Gateway validation/filtering, CLI, agent tool schema, Control UI filtering, and public cron docs. The prior ClawSweeper comment reached the same conclusion, and a fresh source pass on current main still shows only name/description/agent text filtering plus schedule/status UI filters, not freeform job tags.

## What This Changes

Add optional freeform cron job tags/categories that persist with jobs and are exposed for listing, filtering, UI organization, and later bulk actions.

## Best Possible Solution

Ship an additive core cron tags contract: `tags?: string[]` on persisted jobs, create/update/list Gateway schemas with tag filtering, CLI add/edit/list/show support, Control UI create/edit/list/filter controls, agent cron tool support, tests, and public docs. Keep bulk tag actions as a follow-up after the base contract is reviewed.
