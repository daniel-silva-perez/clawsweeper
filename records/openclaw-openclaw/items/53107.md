## What This Changes

The issue requests optional freeform cron job tags/categories that persist, appear in list and UI surfaces, and support filtering or later bulk operations.

Add an additive core cron tags contract: normalized optional `tags?: string[]` on persisted jobs, Gateway add/update/list schemas and filtering, CLI list/show/add/edit support, Control UI create/edit/list/filter controls, agent cron tool support, and public cron docs.
