## Summary

Keep this issue open. Current main still lacks a supported cron tags/categories contract across the persisted job type, Gateway protocol, server filtering, CLI, agent tool, Control UI, and docs. The recent prior ClawSweeper keep-open comment remains accurate against current main, and the related reports are adjacent cron/history/UI/hook items rather than a canonical replacement for this tags/filtering request.

## Best Possible Solution

Keep this issue open. The best implementation path is an additive core cron contract: add normalized optional tags?: string[] to the cron job model and persisted store, accept and validate tags in Gateway add/update/list schemas, implement tag filtering in cron.list/listPage, expose tags in CLI list/show/add/edit plus Control UI create/edit/list/filter controls, update the agent cron tool schema and public cron docs, and add focused regression tests. Tag-based bulk enable/disable can follow as a narrower feature after the foundational tags contract ships.
