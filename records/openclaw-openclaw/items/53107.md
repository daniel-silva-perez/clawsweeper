## Summary

Keep this issue open. Current main still does not provide cron job tags/categories across the persisted cron model, Gateway protocol, server filtering, CLI, Control UI, agent cron tool, or docs. The prior ClawSweeper keep-open comment remains accurate, and the related cron/history/UI/hook items are adjacent rather than a canonical replacement for this specific tags/filtering request.

## Best Possible Solution

Keep this issue open. The best implementation path is an additive core cron contract: add normalized optional tags?: string[] to the persisted cron job type and store, accept and validate tags in Gateway add/update/list schemas, implement tag filtering in cron.list/listPage, expose tags in CLI list/show/add/edit plus Control UI create/edit/list/filter controls, update the agent cron tool schema and public cron docs, and add focused regression tests. Tag-based bulk enable/disable can follow as a narrower feature after the foundational tags contract lands.
