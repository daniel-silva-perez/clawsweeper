## Summary

Keep open. Current main still has an early defer once a Discord native command object exists, but plugin-owned native command specs are still sourced from the runtime plugin command registry, and manifest command aliases such as Codex do not by themselves create Discord handlers during startup. The prior ClawSweeper comment kept this open on the same lifecycle gap, and the central Discord/plugin command files have not materially changed since that review.

## What This Changes

Fix Discord plugin slash commands so plugin-registered native commands acknowledge within Discord's 3-second interaction deadline instead of failing with Unknown interaction or already-acknowledged errors.

## Best Possible Solution

Define one Discord-native readiness contract for plugin-owned slash commands: either predeploy manifest-declared commands with a lightweight immediate-defer dispatcher that safely activates the owner plugin, or only deploy commands whose handlers are loaded before gateway readiness. Keep /vc voice command validation and timeout work separate from generic plugin command dispatch.

## Reproduction Assessment

Unclear: the report has concrete logs and plausible steps, and current source still shows the lifecycle gap, but this review did not run a live Discord reproduction. A high-confidence repro should enable commands.native with a startup-lazy plugin alias such as /codex and capture interaction received-to-defer plus plugin registration timing.

## Solution Assessment

Unclear because there is no proposed patch. The suggested blanket eager loading is probably not the best maintainable fix; an explicit Discord-visible plugin command readiness/activation contract is safer.
