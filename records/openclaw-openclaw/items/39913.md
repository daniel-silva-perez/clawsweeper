## Summary

Keep this issue open. Current main still implements `messages.responsePrefix` for normal routed replies, but the message-tool `send` path continues to pass the parsed raw message through plugin dispatch, core `sendMessage`, and gateway/direct delivery without resolving or applying the effective prefix. The prior ClawSweeper keep-open review remains accurate against current main `69fb7455c6223682a45dd8b8fb720bc3231bfa43`.

## Best Possible Solution

Keep this issue open. Implement prefixing once at the shared message-tool outbound `send` boundary before plugin/core/gateway dispatch, resolving the effective prefix with agent, channel, and account context and using the same idempotent/template semantics as direct replies. Add focused regression coverage for `runMessageAction`/`handleSendAction`, plugin-dispatch send, and core/gateway send delivery, including media-only and already-prefixed cases.
