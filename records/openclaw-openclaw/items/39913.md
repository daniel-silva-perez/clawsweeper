## Summary

Current main still has the reported gap. Normal routed replies resolve and apply `messages.responsePrefix`, but the message-tool `send` path parses and forwards the raw message through gateway plugin dispatch, plugin fallback, and core `sendMessage` without resolving or applying the effective prefix. A prior related PR, #9103, attempted similar message-tool prefixing but was closed unmerged, so it does not make this issue obsolete.

## Best Possible Solution

Keep this issue open. Implement prefix application once at the shared message-tool outbound `send` boundary before gateway plugin dispatch, plugin fallback, and core `sendMessage`, resolving the same effective `messages.responsePrefix` cascade used by replies with agent, channel, and account context. Add focused regression coverage for `runMessageAction`/`handleSendAction`, plugin dispatch, core fallback delivery, media-only sends, and already-prefixed text.
