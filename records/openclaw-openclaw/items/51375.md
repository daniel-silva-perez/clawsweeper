## Summary

Keep this PR open. Current main still lacks the Feishu reply-dispatcher `mediaLocalRoots` plumbing this PR adds, while `sendMediaFeishu` and `loadWebMedia` already require scoped roots for local file reads. The PR is a focused implementation candidate; overlapping Feishu media-root PRs exist, but none is merged or clearly canonical yet.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land one focused Feishu reply-path fix that adds `mediaLocalRoots` to `CreateFeishuReplyDispatcherParams`, computes roots with `getAgentScopedMediaLocalRoots` at both Feishu bot dispatcher call sites, forwards them into `sendMediaFeishu`, and adds a targeted regression test for the dispatcher plumbing. After that merges, close or supersede the overlapping Feishu mediaLocalRoots PRs against the chosen canonical fix.
