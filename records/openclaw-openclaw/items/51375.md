## Summary

Keep this PR open. Current main still lacks the Feishu reply-dispatcher mediaLocalRoots plumbing this PR is trying to add, while the existing Feishu media sender and core local-media guard require scoped local roots for agent-generated local files. The provided review context and related open Feishu PRs corroborate the same gap, but there is not a clearly merged or canonical superseding fix yet.

## Best Possible Solution

Keep this PR open for maintainer review, or supersede it with one canonical Feishu reply-path fix. The chosen patch should add mediaLocalRoots to CreateFeishuReplyDispatcherParams, compute roots with getAgentScopedMediaLocalRoots at both Feishu bot dispatcher call sites, forward them into sendMediaFeishu, and add a focused reply-dispatcher regression test for the forwarding.

## Risks / Open Questions

- The change expands Feishu reply-path local file-read capability, so the implementation should stay limited to getAgentScopedMediaLocalRoots and should not introduce account-level arbitrary roots or localRoots: "any" without a separate security review.
- Multiple overlapping open Feishu mediaLocalRoots PRs exist (#39520, #49082, and the locally reported #58591), so maintainers should choose one canonical patch to avoid partial or divergent fixes.
- This PR is functionally relevant but should add or be paired with a focused reply-dispatcher regression test asserting mediaLocalRoots is forwarded to sendMediaFeishu.
