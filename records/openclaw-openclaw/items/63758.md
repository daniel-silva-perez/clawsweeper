## Summary

Current main still matches the reported cron producer gap. The task ledger contract supports progress summaries, and status surfaces render them for active tasks, but scheduled, startup catch-up, and manual cron paths still create running cron task rows without `progressSummary` and finalize them only with terminal summaries. No cron code imports or calls the progress writer.

## Best Possible Solution

Keep this issue open. The cron service should seed a coarse `progressSummary` when scheduled, startup catch-up, and manual cron task rows are created; update it through `recordTaskRunProgressByRunId` at meaningful execution phases; preserve cron's silent notification default; and add focused regression coverage proving active cron work exposes useful detail in `/tasks`, `/status`, and/or `session_status` before `terminalSummary` is written.
