## Summary

Current main still matches the reported cron producer gap. The task ledger and status surfaces support and display `progressSummary`, but scheduled cron, startup catch-up cron, and manual cron runs still create running task records without a progress summary and only finalize them with terminal summaries. The managed memory/dreaming cron path feeds the same cron service, so the observed managed-cron case is still covered by this issue.

## Best Possible Solution

Keep this issue open. The cron service should seed a coarse `progressSummary` when scheduled, startup catch-up, and manual cron task rows are created; update it through `recordTaskRunProgressByRunId` at meaningful phases; preserve cron's silent notification default; and add focused regression coverage for active cron detail in `/tasks`, `/status`, and/or `session_status` before `terminalSummary` is written.
