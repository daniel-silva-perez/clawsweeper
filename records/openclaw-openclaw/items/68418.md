## Summary

Keep open. Current main still has the array-coercion failure path, the linked Mistral bug remains open, and this PR is a plausible targeted fix, but it needs review changes before merge: the changelog entry misses required contributor attribution and the new recognizer can still skip same-chunk reasoning/tool calls when recognized typed blocks contain only empty strings.

## What This Changes

The PR adds OpenAI-completions stream handling for array-shaped typed `delta.content` blocks, adds regression tests for Mistral-style thinking/text chunks, and documents the user-facing fix in the changelog.

## Best Possible Solution

Land a rebased parser fix that unpacks typed content arrays in the current OpenAI-completions stream loop, preserves same-chunk reasoning/tool-call processing whenever no non-empty typed delta is emitted, keeps the existing string fast path, and retains focused regression coverage plus a correctly attributed changelog entry.

## Reproduction Assessment

Yes. The linked issue gives a concrete Mistral Small thinking scenario, and the PR adds a mocked stream with the same typed `delta.content` shape; current main's direct text append path explains the `[object Object]` output without needing live credentials.

## Solution Assessment

Unclear as submitted. Parsing typed content blocks in the transport is the right owner boundary, but the implementation should avoid treating empty supported typed blocks as consumed content and should be rebased onto the current parser before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Fall through when typed blocks produce no deltas:** `src/agents/openai-transport-stream.ts:1272`
  - body: `sawSupportedBlock` is set even when a supported `text` or `thinking` block contains only an empty string, so the caller treats the array as recognized and `continue`s without processing `reasoning_*` fields or `tool_calls` in the same chunk. Mark the content as consumed only after at least one non-empty delta is emitted, or otherwise fall through when `deltas` is empty.
  - confidence: 0.82

  - body: This user-facing fix adds a changelog entry, but OpenClaw changelog policy requires contributor-authored `fix` entries to include a `Thanks @...` attribution. Add `Thanks @briandevans.` to the new bullet before merge.
  - confidence: 0.95
