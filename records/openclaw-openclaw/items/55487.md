## Summary

Keep open. The PR is still relevant because current main only partially overlaps the gateway/CLI crash-hardening request and lacks the raw upgrade-socket fallback, but the branch needs revision before merge because its uncaught-exception suppression can run before fatal/config-style policy and suppress wrapped fatal failures.

## What This Changes

The PR adds strict transient network/TLS suppression to both uncaughtException handlers, attaches a logged raw upgrade-socket error listener, and expands classifier unit coverage.

## Best Possible Solution

Preserve the crash-hardening goal, but implement it through the current uncaught-exception policy seam with explicit fatal/config precedence, narrow TLS/undici/socket signatures, and regression coverage for both CLI entrypoints plus the upgrade-socket path.

## Reproduction Assessment

Yes for the PR finding: the diff allows a synthetic fatal wrapper such as ERR_WORKER_UNCAUGHT_EXCEPTION with a nested ECONNRESET/TLS cause to satisfy the strict transient classifier before fatal handling. The original gateway crash remains intermittent, but #36585 and #35257 provide concrete load/TLS failure context.

## Solution Assessment

No as-is. The raw socket listener and narrower transient direction are useful, but the safest solution is to adapt the branch to current main's uncaught-exception policy with fatal/config precedence before any transient suppression.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve fatal precedence before suppressing uncaught errors:** `src/cli/run-main.ts:164-169`
  - body: This handler returns for `isStrictTransientNetworkError(error)` before any fatal/config-style classification. Because the strict classifier walks nested causes, a fatal wrapper such as `ERR_WORKER_UNCAUGHT_EXCEPTION` with a transient network cause can be logged as non-fatal and leave the CLI or gateway running after a fatal exception. Keep fatal checks ahead of transient suppression or make the strict check ignore fatal wrappers.
  - confidence: 0.86
