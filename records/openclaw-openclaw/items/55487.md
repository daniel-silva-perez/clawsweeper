## Summary

Keep open. Current main only treats benign uncaught stream errors such as EPIPE/EIO as non-fatal; it does not implement this PR's broader uncaughtException hardening for transient network/TLS failures, and the gateway upgrade handler still lacks the proposed raw-socket error listener. The PR is not obsolete, but it still needs maintainer review and likely revision because this is a global exception-handling safety boundary.

## Best Possible Solution

Keep the PR open for maintainer review or replace it with a current-main patch. The best fix should share the uncaughtException policy across both entrypoints, preserve fatal/config error precedence, use a narrow network/TLS allowlist that is proven against the linked crash shape, add the gateway upgrade socket error listener with logging if still needed, and include regression coverage for both safe suppression and fatal-error fallthrough.
