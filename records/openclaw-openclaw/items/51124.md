## Summary

Keep open. Current main still does not implement this PR's intended Telegram session selector cleanup, and the PR remains useful but needs a rebase plus fixes for the existing review comments before it can land.

## Best Possible Solution

Keep this PR open or replace it with an updated patch in ui/src/ui/chat/session-controls.ts. The best implementation should normalize only known Telegram transport noise, consume origin.label when it is the only human conversation label, produce focused DM/slash labels, dedupe only the main and agent:main:main alias pair when both rows exist, preserve free-form labels with colons, and add regression tests for Telegram DM, slash, group/topic labels, and the bare-main fallback.

## Risks / Open Questions

- Merging the PR as-is could mis-select the active session when only the bare main row exists but the select value is forced to agent:main:main.
- Broad prefix stripping could corrupt legitimate user-authored labels containing colons, such as project:alpha.
- Label-based main dedupe could hide a real contact, room, or renamed session whose visible label is main instead of deduping only the main/agent:main:main alias pair.
- The implementation target has moved to ui/src/ui/chat/session-controls.ts, so the patch against ui/src/ui/app-render.helpers.ts needs a rebase.
