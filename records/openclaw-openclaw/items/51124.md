## Summary

Keep open. Current main still does not implement this PR's intended Telegram session selector cleanup: the Control UI resolver only uses `label` and `displayName`, does not consume `origin.label`, does not strip Telegram transport noise, and does not add DM/slash-specific labels. The PR is focused, but the existing review comments identify functional issues to fix before merge, and the implementation now needs to be rebased onto `ui/src/ui/chat/session-controls.ts`.

## Best Possible Solution

Keep this PR open or replace it with an updated patch in `ui/src/ui/chat/session-controls.ts`. The best implementation would normalize only known Telegram transport noise, use gateway `origin.label` when it is the only human conversation label, produce focused DM/slash labels, dedupe only the `main` and `agent:main:main` alias pair when both exist, preserve free-form labels with colons, and add regression tests for Telegram DM, slash, group/topic labels, and the bare-main fallback case.

## Risks / Open Questions

- Merging the PR as-is could mis-select the active session when only the bare `main` row is present.
- Broad prefix stripping could corrupt legitimate user labels containing colons, such as `project:alpha`.
- Label-based `main` dedupe could hide a real session or room named `main`; dedupe should target only the alias pair.
- The touched implementation has moved from `ui/src/ui/app-render.helpers.ts` to `ui/src/ui/chat/session-controls.ts`, so the patch needs a rebase/update.
