## Summary

Keep open. Current main still does not implement this PR's intended Telegram session-label cleanup: the Control UI selector resolves labels from `label`/`displayName` only, does not consume `origin.label`, does not strip Telegram transport prefixes or `id:` suffixes, and does not add the PR's targeted Telegram DM/slash label behavior. The PR is focused and likely useful, but the review comments identify correctness issues that should be fixed before merge, and the current code location has moved to `ui/src/ui/chat/session-controls.ts`.

## Best Possible Solution

Keep this PR open or replace it with an updated patch in `ui/src/ui/chat/session-controls.ts`. The best path is to implement narrow Telegram label normalization using the gateway's existing `origin.label` data, add the missing UI row type coverage for `origin`, dedupe only the `main`/`agent:main:main` alias pair when both are present, preserve user-authored labels with colons, and add focused regression tests for Telegram DM, slash, group/topic display names, and the main-alias fallback case. Related ClawSweeper context for #68850 suggests the broader synthetic-label cleanup is still unsolved on main, so this should not be closed as implemented.
