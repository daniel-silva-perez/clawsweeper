## Summary

Keep this PR open. Current main covers the embedded-run abort portion through registered abort controllers and abort-signal propagation, but it still does not implement this PR's stated session queue and sub-agent cleanup behavior, and the PR's current direct cleanup calls have unresolved run/session scoping and owner-isolation concerns.

## Best Possible Solution

Revise or replace the PR with a narrower implementation that keeps the current abort-signal plumbing for embedded runs, adds queue and sub-agent cleanup only when the abort is truly session-wide and authorized, and introduces a run-scoped queue/sub-agent mapping if runId cancellation needs cleanup without clearing unrelated session work.

## Risks / Open Questions

- The PR as written may convert chat.abort(runId) from targeted run cancellation into a session-wide stop by aborting the embedded session and clearing session queues.
- The sessionKey path may break owner isolation when multiple clients share a session key because cleanup is keyed by session rather than the authorized run ids returned from the abort helper.
- Current main still lacks a complete regression path for queue and sub-agent cleanup that preserves runId targeting and multi-owner authorization boundaries.
