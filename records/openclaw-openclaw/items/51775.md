## Summary

Keep this PR open. Current main has signal-backed cancellation for embedded runs, but it does not implement the PR's stated session queue and sub-agent cleanup behavior, and the PR's direct cleanup approach still has unresolved run/session scoping and owner-isolation concerns.

## Best Possible Solution

Revise or replace the PR with a narrower implementation: keep the current abort-signal plumbing for embedded runs, add queue and sub-agent cleanup only when the abort is truly session-wide and authorized, and introduce run-scoped queue/sub-agent ownership if `runId` cancellation needs cleanup without clearing unrelated session work.

## Risks / Open Questions

- The PR as written can convert `chat.abort(runId)` from targeted run cancellation into session-wide cleanup by aborting the embedded session and clearing queues keyed only by session.
- The sessionKey path can undermine owner isolation when multiple clients share a session key because cleanup is keyed by session rather than by the authorized run ids returned from the abort helper.
- Current main still lacks a safe implementation and regression coverage for queue/sub-agent cleanup that preserves runId targeting and multi-owner authorization boundaries.
