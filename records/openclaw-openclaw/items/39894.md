Keep this PR open. Current main still parses outbound reply text before stripping echoed inbound metadata, so the intended bug fix is not implemented. The PR contains the right narrow idea, but it is draft, stale against current main, and mixes the outbound fix with unrelated security-sensitive CI/test/macOS changes that should be split or separately reviewed.

## Best Possible Solution

Keep this PR or #39847 open until a focused fix lands on current main: import stripInboundMetadata in src/infra/outbound/payloads.ts, strip payload.text before parseReplyDirectives() inside createOutboundPayloadPlanEntry(), and add the metadata-plus-MEDIA regression to src/infra/outbound/payloads.test.ts. Rebase the PR and split or drop the unrelated CI/macOS/test-runner changes before maintainer review.
