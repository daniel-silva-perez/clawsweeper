## Summary

Keep open. An open, unmerged PR (#59208) explicitly targets this issue, and current main still appears not to thread the session-selected OAuth profile into provider usage resolution.

## Best Possible Solution

Keep this issue open and review #59208, or replace it with an equivalent focused patch. The fix should thread the active session auth profile from the status surface into loadProviderUsageSummary and resolveProviderAuths, validate that the preferred profile belongs to the requested usage provider, preserve existing provider-level fallback behavior, and add regression coverage for multiple openai-codex OAuth profiles plus stale or mismatched overrides.
