## Summary

Keep open: the PR targets a real ACPX Codex duplicate `-c` crash path and is not obsolete on current `main`, but the proposed key-skip behavior can silently discard explicit runtime model/thinking overrides when the configured command already has that key. That conflicts with the documented `sessions_spawn.model` override contract, so the patch should be revised before merge.

## What This Changes

The PR adds a `commandHasConfigKey` helper to avoid appending duplicate Codex ACP `-c model` and `-c model_reasoning_effort` command overrides, plus ACPX runtime tests for duplicate and partial-overlap cases.

## Best Possible Solution

Keep the fix extension-local, but normalize the Codex ACP command so duplicate `-c` keys are removed or replaced while explicit runtime `model`/`thinking` overrides keep the documented precedence. Add tests for same-value duplicates, different-value conflicts, partial overlap, quoted args, wrapper commands, and the no-runtime-override case, plus the required changelog entry.

## Reproduction Assessment

Yes. The linked issue gives concrete operator steps, and the PR's own partial-overlap test statically reproduces the risky behavior by showing a runtime `model: "gpt-5.5"` request being dropped when the configured command already has `-c model=gpt-5.4`.

## Solution Assessment

No. Skipping already-present keys fixes identical-value duplication, but it is not the best solution because it silently lets configured command defaults override explicit runtime values; replacing existing same-key `-c` entries or failing clearly on conflict would preserve the public contract more safely.
