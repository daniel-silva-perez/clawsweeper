## Summary

Keep this PR open. Current main still appends Codex ACP runtime model/reasoning `-c` overrides without checking whether the configured command already contains those keys, and the linked ACPX Codex crash report remains open with a narrow proposed fix.

## What This Changes

This PR updates ACPX Codex command override handling to skip already-configured `-c` keys and adds focused regression tests for full and partial duplicate model/reasoning overrides.

## Best Possible Solution

Review and, if accepted, merge the narrow ACPX runtime guard with focused regression coverage, then validate the ACPX runtime tests and a configured-command Codex ACP smoke when credentials are available. No core API, dependency, or docs change appears necessary for this PR as written.
