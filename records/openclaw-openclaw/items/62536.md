## Summary

Keep open. Current main still sends subagent announce replies through the normal delivery logging path, so the original gateway log-noise/privacy bug is not implemented on main; this PR is useful and directionally correct, but it is dirty against current main and still misses the empty-payload announce path.

## What This Changes

This PR routes subagent announce delivery output and announce warning/error logs through an `agents/announce` subsystem logger, with metadata at info, bodies at debug, and a focused delivery unit test.

## Best Possible Solution

Land a rebased, narrow logging fix that detects announce runs before any local delivery logging, sends all announce non-json delivery diagnostics through `agents/announce`, keeps normal agent/json/nested delivery behavior unchanged, and preserves focused unit coverage for full-body and empty-payload announce cases.

## Reproduction Assessment

Yes. Current main can be traced from `subagent_announce` provenance in the announce delivery call into `deliverAgentCommandResult()`, where formatted reply text still reaches `runtime.log()` and console capture files it at info; the linked bug also includes concrete gateway log examples and the PR author supplied after-fix E2E notes.

## Solution Assessment

No as written. The subsystem logger approach is the right narrow direction, but the PR should be rebased onto current main and should route the empty-payload announce case through the same logger before it is merge-ready.
