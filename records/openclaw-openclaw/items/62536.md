## Summary

Keep open. Current main does not implement this PR's intended logging change: subagent announce calls still mark `sourceTool: "subagent_announce"`, but `deliverAgentCommandResult()` ignores that provenance and still routes visible payload text through `runtime.log()`, which reaches console capture and file logs at `info`. The supplied PR context is a focused bug fix with useful validation, not obsolete work.

## Best Possible Solution

Keep this PR open for maintainer review, or recreate the same narrow fix on current main. The implementation should detect `inputProvenance.sourceTool === "subagent_announce"` before both empty-payload and normal payload logging, route default-level announce logs to stable metadata through `createSubsystemLogger("agents/announce")`, keep full bodies debug-only, move announce retry/error logs off `defaultRuntime`, and retain focused regression coverage.
