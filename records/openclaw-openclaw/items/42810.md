## Summary

Keep this PR open. Current main still does not implement the proposed ACP control-plane audit logging system, while the PR remains a plausible implementation candidate that needs maintainer review, rebase, and security/reliability hardening before merge.

## Best Possible Solution

Keep this PR open for maintainer follow-up. The best path is a rebased, narrower ACP-owned audit implementation: add a best-effort `AcpSessionManager` audit seam, wire a real production sink under the normal OpenClaw state/log directory with restrictive permissions, enforce bounded failure behavior, make retention/query semantics correct, cover session init/close/cancel/run/runtime-control events with focused tests, and split unrelated config/schema drift into separate work.
