## Summary

Keep open. Current main still lacks ACP control-plane audit logging, but the proposed branch is not merge-ready for a security/compliance feature: normal production construction still resolves to a null logger, file logging has durability and bounded-memory issues, and the branch is stale against recent ACP lifecycle changes.

## What This Changes

The PR adds a new ACP control-plane audit module with file/null loggers, audit types/utilities/tests, and AcpSessionManager audit calls for session initialization and close paths.

## Best Possible Solution

Keep the work open, but require a rebased and narrower ACP-owned audit implementation that uses an OpenClaw state/log directory with restrictive permissions, makes audit emission best-effort, bounds failure behavior, defines retention/query semantics precisely, and covers session init/close/cancel/run/runtime-control events with focused tests.

## Reproduction Assessment

Not applicable as a feature PR rather than a bug report. The review does have a high-confidence verification path: current-main source inspection shows no ACP audit module or audit dependency seam, while PR diff inspection shows the proposed implementation remains incomplete.

## Solution Assessment

No. The requested direction is plausible, but this branch is not the best mergeable form because the normal runtime still uses a null sink and the file logger is not robust enough for a security audit trail; a smaller rebased implementation with a real production sink and hardening is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Replace the default audit logger with a real sink:** `src/acp/control-plane/manager.types.ts:146`
  - body: The production singleton still constructs AcpSessionManager with DEFAULT_DEPS, and DEFAULT_DEPS now binds auditLogger to createNullAuditLogger(). That means normal ACP initialize/close paths call the audit seam but persist nothing, so the advertised audit trail is absent unless a test or custom caller injects a logger.
  - confidence: 0.96
- **[P2] Make audit emission best-effort:** `src/acp/control-plane/manager.core.ts:347`
  - body: initializeSession and closeSession await auditLogger.log() in finally blocks. If the audit sink rejects, that rejection overrides a successful lifecycle operation or masks the original failure, so an unhealthy audit sink can break ACP session management.
  - confidence: 0.88
- **[P2] Enforce a hard cap after flush failures:** `src/acp/control-plane/audit/audit-logger.file.ts:244`
  - body: flush() requeues failed batches with unshift(), while log() keeps pushing new entries and only schedules another asynchronous flush after maxBufferSize. A sustained disk or permission failure can grow memory without bound instead of honoring the advertised buffer limit.
  - confidence: 0.9
- **[P2] Store audit logs with private state-directory permissions:** `src/acp/control-plane/audit/audit.types.ts:163`
  - body: The default file sink uses a relative .audit directory, and appendFile() does not request restrictive file permissions. Audit entries can include actor/session details, so enabled logging should write under the OpenClaw state/log directory with 0700 directories and 0600 files like the existing config audit helper.
  - confidence: 0.84
