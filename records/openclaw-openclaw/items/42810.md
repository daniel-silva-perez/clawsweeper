## Summary

Keep this PR open. Current main still does not implement the proposed ACP control-plane audit logging system: there is no ACP audit module, no audit logger dependency or production sink, and ACP session lifecycle/control paths do not emit durable audit records. The PR remains potentially useful, but it needs a narrower rebase and security/reliability hardening before it is mergeable.

## Best Possible Solution

Keep this PR open, but require a rebased and narrower ACP audit implementation before merge. The best path is a best-effort AcpSessionManager audit seam with a real production sink under the normal OpenClaw state/log directory, restrictive file permissions, bounded buffering/failure behavior, correct retention/query semantics, focused coverage for session init/close/cancel/run/runtime-control events, and unrelated config/schema changes split out.
