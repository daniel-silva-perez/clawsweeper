## Summary

Close as implemented on current main and shipped in v2026.4.27. The custom-provider onboarding path now infers image-capable model IDs, prompts for unknown models, supports explicit non-interactive overrides, and has tests/docs covering the behavior that the issue requested.

## What This Changes

The issue asks non-Azure custom-provider onboarding to stop saving all models as text-only and to support image/vision metadata through inference, a prompt, or an override.

## Best Possible Solution

Keep the shipped v2026.4.27 implementation. Users with configs generated before the fix should rerun current onboarding, use `--custom-image-input`, or set `models.providers.*.models.*.input` manually rather than adding a broad startup migration.

## Reproduction Assessment

Yes. The issue's reproduction path maps to `applyCustomApiConfig` and the custom-provider wizard tests; current main verifies that known non-Azure vision IDs now produce `input: ["text", "image"]` and unknown IDs prompt for image support.

## Solution Assessment

Yes. The current implementation is narrower and safer than marking every custom model image-capable: it infers known vision/text families, prompts on unknown interactive setup, and gives non-interactive users explicit opt-in/opt-out flags.
