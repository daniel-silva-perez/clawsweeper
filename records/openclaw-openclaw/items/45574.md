## Summary

Current main still has the reported Slack `download-file` schema gap. Slack advertises `download-file` and both the dispatch and runtime paths require `fileId`, but the shared `message` tool schema still does not expose a `fileId` parameter. PR #45626 is already open as the focused fix candidate and references this issue with closing syntax, so this issue should stay open until that PR or an equivalent fix merges.

## Best Possible Solution

Keep this issue open and review or land PR #45626, or an equivalent narrow fix, that exposes `fileId` as an optional message-tool parameter and adds regression coverage for the shared schema plus Slack `download-file` dispatch/runtime. Close as implemented only after the fix reaches main.
