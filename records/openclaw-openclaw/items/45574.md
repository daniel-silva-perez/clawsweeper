## Summary

Current main still has the reported Slack `download-file` schema gap. Slack advertises and documents `download-file`, and Slack dispatch/runtime still require `fileId`, but the shared `message` tool schema does not define `fileId` and Slack discovery contributes no schema field to add it. Open PR #45626 is an unmerged implementation candidate with closing syntax for this issue, so the issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land PR #45626 or an equivalent focused fix on current main. The best fix is to expose an optional schema-visible `fileId` parameter for Slack `download-file`, either in the shared message-tool schema or as a Slack `describeMessageTool()` schema contribution, with a regression assertion in the message-tool schema path plus the existing Slack dispatch/runtime coverage. PR #62792 addresses related Slack media hydration, but it does not replace this schema pass-through fix.
