## Summary

Keep open. Current main still does not contain a durable path-preserving locale switcher, and the prior standalone-script attempts were closed because the docs site does not prove docs-root JavaScript is loaded or synchronized with the generated locale source of truth.

## What This Changes

The issue asks for the Mintlify docs language switcher to preserve the current page path when switching between English and localized docs instead of navigating to the target locale homepage.

## Best Possible Solution

Keep this as the canonical tracker until maintainers choose the boundary: either get Mintlify to preserve page paths upstream, or wire a repo-owned workaround through a supported docs publish/config mechanism that derives locale mappings from GENERATED_LOCALES and is verified in a Mintlify preview or live deployment.

## Reproduction Assessment

Yes. The issue gives concrete live-docs steps, the earlier #34021 report corroborates the same behavior, and current source/live asset checks show no deployed path-preserving handler; final fix validation still needs a browser switch from /install/docker to /zh-CN/install/docker.

## Solution Assessment

No for the previously attempted standalone root JavaScript file. The narrower maintainable solution is a verified Mintlify-supported hook or publish-pipeline integration, or an upstream Mintlify platform fix.
