## Summary

Keep open. Current main still does not contain a durable path-preserving docs locale switcher or a verified Mintlify custom-script deployment hook, and the prior standalone-script PRs were closed unmerged because that deployment path was not proven for this repo.

## What This Changes

The issue asks for the docs language switcher to preserve the current page path when switching locales instead of navigating to the target locale homepage.

## Best Possible Solution

Keep this as the canonical tracker until maintainers choose the boundary. If OpenClaw owns a workaround, it should use a Mintlify-supported deployment hook, derive locale mapping from `GENERATED_LOCALES` or generated publish config, and be verified in a Mintlify preview or live docs deployment; otherwise the durable end state is an upstream Mintlify fix for path-preserving language switching.
