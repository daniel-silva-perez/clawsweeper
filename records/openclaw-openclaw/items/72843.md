## Summary

Keep open. The PR addresses a real current-main gateway warmup bug and fixes the initial generic index.js false positive, but the revised OpenClaw-ancestor check still misses documented Docker and cloud daemon launches that run node dist/index.js gateway from /app.

## What This Changes

The PR changes context-window eager-warmup gating to recognize OpenClaw index.js/index.mjs entrypoints, allow gateway commands to warm the cache, and add argv-shape regression tests.

## Best Possible Solution

Broaden the entrypoint recognition so real OpenClaw dist/index.js gateway daemons are covered across npm/global installs, dev checkouts, and container/cloud /app layouts, while retaining an explicit negative guard for arbitrary plugin-sdk consumers named index.js.

## Reproduction Assessment

Yes. Against the PR head logic, shouldEagerWarmContextWindowCache(["node", "/app/dist/index.js", "gateway"]) or ["node", "dist/index.js", "gateway"] would return false because neither path has an exact openclaw ancestor, matching documented daemon launch shapes.

## Solution Assessment

No, not as-is. Removing gateway from the skip list is the right direction, but the exact-ancestor index.js gate is too narrow; the safer fix should use a real OpenClaw dist entry signal that also covers /app and relative dist launches.
