## Summary

Keep this PR open for maintainer/contributor follow-up. Current main has broader tool execution diagnostic events and the linked umbrella bug is closed, but main still does not implement two narrow pieces this PR proposes: warn-level structured logging for embedded tool handler failures and skipped block-reply payload context. The PR also has review-identified defects that should be fixed before merge, especially the over-broad Control UI visibility bypass.

## Best Possible Solution

Keep this PR open only as a narrow follow-up candidate: serialize skipped block-reply payload context or intentionally extend logVerbose, scope any visibility bypass to the exact terminal/error events intended, and add focused regression tests. If maintainers consider the broader #40082 work sufficient, they should close this manually as superseded after that product judgment.

- As written, the PR's Control UI visibility change can expose all hidden-run tool stream events by attaching sessionKey when event.stream === "tool", contrary to the current hidden-run contract and tests.
