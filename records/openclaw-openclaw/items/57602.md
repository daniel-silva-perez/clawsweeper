## Summary

Keep this PR open for maintainer/contributor follow-up. Current main now implements the Control UI visibility piece more narrowly and has broader tool-execution diagnostics, but the PR still proposes two narrow changes that are not on main and the submitted diff has concrete blocking defects.

## What This Changes

The PR raises embedded tool handler failures from debug to warn, adds skipped block-reply payload context to timeout logging, and makes selected agent events bypass Control UI visibility filtering.

## Best Possible Solution

Keep the shipped lifecycle-only Control UI visibility behavior, revise this PR only for the two remaining narrow diagnostics if maintainers want them, and add focused tests for block-reply timeout log content plus hidden-run tool-event suppression.

## Reproduction Assessment

Yes for the PR defects: current main shows logVerbose only accepts one string argument, and current tests encode that hidden non-lifecycle events omit sessionKey. Applying this diff would make skippedPayload logging inert and violate the hidden-run visibility contract.

## Solution Assessment

No. The PR is not the best solution as written because one logging change has no runtime effect and the visibility change is broader than the current main contract; a safer fix would serialize skippedPayload and keep visibility bypasses scoped to lifecycle terminal events only.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Serialize the skipped payload before logging it:** `src/auto-reply/reply/block-reply-pipeline.ts:141-144`
  - body: `logVerbose` only accepts `message: string`, so the added metadata object is ignored and the advertised `skippedPayload` context never reaches logs. Serialize the payload key into the message or intentionally extend `logVerbose`.
  - confidence: 0.96
- **[P2] Narrow the hidden-run visibility bypass:** `src/infra/agent-events.ts:71-74`
  - body: The new `event.stream === "tool"` check treats every tool event as critical. Current main protects hidden non-lifecycle runs by omitting `sessionKey`, so this would expose hidden tool start/update traffic and break that suppression contract.
  - confidence: 0.92
