## Summary

Keep this PR open. Current main still lacks sender-aware non-owner exec deny behavior, and the PR touches command-execution security semantics with an unresolved partial-defaults gap plus overlap with same-author open PR #64719.

## What This Changes

The PR adds `senderIsOwner` to auto-reply exec override resolution, passes the command owner flag at the directive call site, and expands unit coverage for non-owner deny defaults and ignored inline/session exec overrides.

## Best Possible Solution

Revise the PR so non-owner auto-reply ignores user-controlled inline/session exec fields, fails closed with `security=deny` and `ask=always` when no fully explicit safe operator policy exists, and coordinates the final tool-access boundary with #64719 before merge.
