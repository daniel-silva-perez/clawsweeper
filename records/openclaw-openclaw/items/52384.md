## Summary

Keep open. Current main has strong adjacent mitigations for model failover, auth-profile cooldowns, and restart/orphan recovery, but it still does not implement this issue's requested circuit breaker for auto-resume injection after consecutive model errors. The resume paths still inject synthetic agent messages when restart/orphan recovery runs, while the model fallback logic is scoped to a single run/candidate chain and auth cooldown state rather than persisted session-level suppression of further resume prompts after N model failures.

## Best Possible Solution

Keep this issue open and route the fix through the agent/session recovery owner path: add persisted per-session model-error accounting around automatic resume attempts, suppress or back off further synthetic resume injection after the chosen threshold, reset the breaker on the first successful model response, and cover both main-session and subagent restart/orphan recovery with focused regression tests. Coordinate with the open session-level circuit-breaker work in #42933/#45389 so there is one coherent breaker state model rather than separate ad hoc counters.
