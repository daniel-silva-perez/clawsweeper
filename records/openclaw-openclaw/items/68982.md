## Summary

Keep open. The latest PR head is still a UI-only renderer/build change, while current main's live gateway and Control UI controller path still broadcasts and stores chat stream payloads as text-only, so structured reasoning-only deltas cannot reach the new renderer path. The patch also widens reasoning visibility by falling back from `/reasoning` state to `/think` effort, which needs maintainer handling before merge.

## What This Changes

The PR updates Control UI chat item building and grouped streaming rendering to accept structured stream message payloads, pass reasoning visibility into streaming groups, and add grouped-render regression tests.

## Best Possible Solution

Land one consolidated end-to-end implementation that carries structured thinking/text payloads from agent events through gateway chat delta/final payloads, Control UI state, item building, and streaming rendering, while preserving `/reasoning` visibility semantics and the reading indicator when reasoning is hidden.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: current main emits/stores live chat deltas as text-only strings, so a reasoning-only live delta cannot reach the PR's new object-stream renderer path.

## Solution Assessment

No. Renderer-side support alone is not the best fix because the source live payload is still collapsed before rendering; the safer path is end-to-end structured stream plumbing coordinated with the existing broader PR #59086.

## Review Findings

Full review comments:

- **[P1] Plumb structured payloads before rendering:** `ui/src/ui/chat/build-chat-items.ts:15`
  - body: `BuildChatItemsProps.stream` now accepts object payloads, but the real live path still cannot supply one: gateway chat delta/final payloads are text-only, `ChatState.chatStream` remains `string | null`, and `handleChatEvent` stores `extractText(payload.message)`. Reasoning-only deltas are still dropped before this branch can run, so live reasoning streaming remains unfixed.
  - confidence: 0.94
- **[P2] Preserve the reading indicator when reasoning is hidden:** `ui/src/ui/chat/build-chat-items.ts:296-301`
  - body: The object-stream path emits a stream item for any structured message, even when it has no visible text. With reasoning hidden, `renderGroupedMessage` suppresses the thinking block and returns no bubble, so the previous reading indicator is replaced by an empty assistant row until answer text arrives.
  - confidence: 0.88
- **[P2] Keep thinking effort out of reasoning visibility:** `ui/src/ui/views/chat.ts:197`
  - body: `resolveStreamingReasoningLevel` falls back from a missing session reasoning level to `props.thinkingLevel`, but `/think` controls model effort while `/reasoning` controls whether reasoning is shown and falls back to `off`. A session with thinking enabled but no reasoning visibility override can show reasoning unexpectedly.
  - confidence: 0.9
