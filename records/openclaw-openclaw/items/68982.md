## Summary

Keep open. The latest PR is still a renderer-side Control UI patch, while current main still projects gateway chat deltas/finals and Control UI live stream state as text-only, so structured reasoning payloads cannot reach the new object-stream rendering path in real live chat. The patch also needs maintainer judgment because it overlaps the broader open #59086 work and introduces a reasoning-visibility regression risk.

## What This Changes

The PR changes four Control UI files to accept structured stream message payloads, pass reasoning visibility into streaming rendering, render reasoning-only assistant content, and add grouped-render regression tests.

## Best Possible Solution

Land one consolidated implementation that preserves structured assistant reasoning/text payloads from gateway chat events through controller/app stream state, item building, and rendering, while keeping `/reasoning` visibility separate from `/think` effort and preserving the reading indicator for hidden reasoning-only streams.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists by tracing a live reasoning event through current main: gateway chat deltas/finals are emitted as text-only assistant messages, and `handleChatEvent` stores only `extractText(payload.message)` in `chatStream`, so the PR's object-stream branch is unreachable in real live chat.

## Solution Assessment

No. This PR is not the best complete fix yet because renderer support alone cannot stream reasoning; the gateway chat event projection and Control UI state pipeline need to carry structured payloads end to end, and display must continue to honor `/reasoning` rather than `/think`.

## Review Findings

Full review comments:

- **[P1] Plumb structured stream payloads through chat state:** `ui/src/ui/chat/build-chat-items.ts:15`
  - body: `BuildChatItemsProps.stream` now accepts objects, but the rest of the real live path still cannot populate that object: current gateway `chat` deltas/finals are text-only and `handleChatEvent` stores `extractText(payload.message)` into a `string | null` stream. Reasoning-only deltas are still dropped before this branch can run, so the PR does not fix live reasoning streaming.
  - confidence: 0.94
- **[P2] Keep hidden reasoning streams on the reading indicator:** `ui/src/ui/chat/build-chat-items.ts:297-298`
  - body: The object-stream path emits a stream item whenever a message object exists, even if it has no visible text. When `showReasoning` is false, `renderGroupedMessage` suppresses the thinking block, so the old reading indicator is replaced by an empty assistant row until answer text arrives.
  - confidence: 0.88
- **[P2] Keep thinking effort out of reasoning visibility:** `ui/src/ui/views/chat.ts:198`
  - body: `resolveStreamingReasoningLevel` falls back from a missing session `reasoningLevel` to `props.thinkingLevel`, but `/think` controls model effort while `/reasoning` controls whether thinking is displayed and falls back to off. This can show reasoning blocks for sessions that never enabled reasoning visibility.
  - confidence: 0.9
