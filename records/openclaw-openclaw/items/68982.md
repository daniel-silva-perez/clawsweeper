## Summary

Keep open. Current main still does not carry live reasoning through the gateway-to-Control-UI chat stream, and the latest PR shape remains incomplete because structured payload support is added at the renderer/item layer without the live state and gateway plumbing needed to feed it.

## What This Changes

The PR changes Control UI chat item building and grouped streaming rendering to accept optional structured stream payloads, render reasoning-only assistant content, pass reasoning visibility into streaming groups, and add grouped-render tests.

## Best Possible Solution

Land one consolidated end-to-end implementation that carries structured assistant thinking/text payloads from agent events through gateway chat delta/final payloads, Control UI state, item building, and streaming rendering, while keeping /reasoning visibility independent from /think effort and preserving the reading indicator when reasoning is hidden.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: current main emits live chat state as text-only, and handleChatEvent stores only extractText(payload.message), so a reasoning-only live delta cannot reach the new object-stream renderer branch.

## Solution Assessment

No. Renderer-side support alone is not the best fix because the source payload is still collapsed before rendering; the safer path is end-to-end structured payload plumbing, likely coordinated with open PR #59086, while preserving the documented reasoning visibility fallback.

  - body: `BuildChatItemsProps.stream` now accepts object payloads, but the real live path still cannot supply one: gateway chat delta/final payloads are text-only and `handleChatEvent` stores `extractText(payload.message)` into string `chatStream` state. Reasoning-only deltas are still dropped before this renderer branch can run, so the advertised live reasoning stream remains unfixed.
  - confidence: 0.94

  - body: The object-stream branch emits a stream item whenever a message object exists, even if it has no visible text. With `showReasoning` false, `renderGroupedMessage` suppresses the thinking block and returns nothing, replacing the previous reading indicator with an empty assistant group until answer text arrives.
  - confidence: 0.88
- **[P2] Keep thinking effort out of reasoning visibility:** `ui/src/ui/views/chat.ts:197-199`
  - body: `resolveStreamingReasoningLevel` falls back from a missing session reasoning level to `props.thinkingLevel`, but `/think` controls model effort while `/reasoning` controls whether reasoning is shown and falls back to `off`. A session with thinking enabled but reasoning visibility unset can show reasoning blocks unexpectedly.
  - confidence: 0.9
