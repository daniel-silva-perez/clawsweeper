## Summary

Keep open. The latest branch only changes Control UI item/rendering code, while current main still projects live chat deltas/finals and controller stream state as text-only, so reasoning-only live events cannot reach the new object stream path. The new fallback from reasoning visibility to thinking effort also conflicts with the documented `/reasoning` default of `off`.

## What This Changes

The PR changes four Control UI files to accept object-valued stream messages, preserve message payloads on stream items, render structured streaming messages with optional reasoning, and add grouped-render regression tests.

## Best Possible Solution

Land one consolidated implementation that carries structured assistant reasoning/text payloads from gateway chat events through controller/app state, stream segments, item building, and rendering, while preserving `/reasoning` visibility semantics and keeping hidden reasoning-only streams on the normal reading indicator.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists by tracing a `stream: "thinking"` event through current main: it exists as an agent event, but gateway `chat` events and Control UI state still reduce live messages to visible text before the PR's object-stream branch can run.

## Solution Assessment

No. The current PR is not the best complete fix because renderer-side object support is not enough; the gateway and UI state pipeline must preserve structured payloads end to end, and reasoning display must remain controlled by `/reasoning`, not `/think`.

  - body: The object-stream path emits a stream item whenever a message object exists, even when it has no visible text. With `showReasoning` false, `renderGroupedMessage` suppresses the thinking block and returns no bubble, so the old reading indicator is replaced by an empty assistant row until answer text arrives.
  - confidence: 0.88
- **[P2] Keep thinking effort out of reasoning visibility:** `ui/src/ui/views/chat.ts:198-199`
  - body: `resolveStreamingReasoningLevel` falls back from a missing session `reasoningLevel` to `props.thinkingLevel`, but `/think` controls model effort while `/reasoning` controls whether thinking is displayed and falls back to off. This can show reasoning blocks for sessions that never enabled reasoning visibility.
  - confidence: 0.9
