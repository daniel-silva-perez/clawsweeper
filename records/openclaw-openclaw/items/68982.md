## Summary

Keep open. The PR is useful but not merge-ready: the current head is UI-only and leaves the structured reasoning payload path unreachable from the live gateway/controller pipeline, while also weakening reasoning visibility semantics.

## What This Changes

The PR changes four Control UI files to accept object-valued stream messages, pass optional stream message payloads into streaming rendering, render reasoning-only bubbles when reasoning is visible, and add grouped-render regression tests.

## Best Possible Solution

Land one consolidated implementation that preserves structured assistant reasoning/text payloads from gateway chat events through Control UI controller state, app state, item building, and rendering, while keeping `/reasoning` visibility independent from `/think` effort.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists by tracing a live `stream: "thinking"` agent event: current main broadcasts it as an agent event, but the chat delta path only handles assistant text and the UI controller stores extracted text, so a reasoning-only live delta cannot reach the new renderer.

## Solution Assessment

No. Renderer-side object support alone is not the best fix; the maintainable solution needs end-to-end structured payload plumbing and must preserve the documented `/reasoning` default-off visibility semantics.

  - body: `BuildChatItemsProps.stream` now accepts object messages, but the real live path still cannot supply one: current app/view state is string-only, `handleChatEvent` stores `extractText(payload.message)`, and gateway chat events emit text blocks. Reasoning-only deltas are still dropped before this branch can render.
  - confidence: 0.95
- **[P2] Keep hidden reasoning streams on the reading indicator:** `ui/src/ui/chat/build-chat-items.ts:299-300`
  - body: `hasStreamMessage` treats any object stream as renderable even when extracted text is empty. With reasoning hidden, `renderGroupedMessage` suppresses the thinking block, so the previous reading indicator is replaced by an empty assistant group until answer text arrives.
  - confidence: 0.88
- **[P2] Keep thinking effort out of reasoning visibility:** `ui/src/ui/views/chat.ts:199`
  - body: `resolveStreamingReasoningLevel` falls back to `thinkingLevel` when the session has no reasoning level, but `/think` controls model effort while `/reasoning` controls display and falls back to off. This can show reasoning for sessions that only enabled thinking effort.
  - confidence: 0.91
