## Summary

Keep open. The latest PR still does not solve live reasoning streaming end to end: the structured render path it adds is unreachable from current main's text-only gateway/controller stream state, and it also risks changing reasoning visibility semantics by falling back from `/reasoning` to `/think`. Because this is an active contributor PR with overlap against open PR #59086, the right next step is maintainer/author consolidation or revision, not cleanup closure.

## What This Changes

The PR widens Control UI stream item rendering to accept structured assistant messages, passes reasoning visibility into streaming bubbles, renders reasoning-only assistant bubbles, and adds grouped-render/build-chat-items regression coverage.

## Best Possible Solution

Land one consolidated implementation that carries structured assistant stream payloads from gateway chat events through controller/app state, stream segments, item building, and rendering, while preserving `/reasoning` visibility semantics and keeping the reading indicator for hidden reasoning-only streams.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists by tracing an assistant `thinking` event through current main: `emitChatDelta` creates a text-only message, `handleChatEvent` calls `extractText`, and `chatStream`/`ChatProps.stream` are string-only, so the PR's object stream branch cannot be reached.

## Solution Assessment

No. The renderer changes are a useful slice, but the maintainable fix needs end-to-end structured payload plumbing and visibility-aware stream item building rather than only widening renderer inputs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Plumb structured payloads before widening render items:** `ui/src/ui/chat/build-chat-items.ts:15`
  - body: `BuildChatItemsProps.stream` now accepts objects, but real live deltas still cannot pass objects here: current main emits text-only chat delta messages, `handleChatEvent` stores `extractText(payload.message)` in `chatStream`, and app/render props are still `string | null`. Reasoning-only deltas are dropped before this branch can run, so the PR does not fix the live-stream path it targets.
  - confidence: 0.94
- **[P2] Preserve the indicator when reasoning is hidden:** `ui/src/ui/chat/build-chat-items.ts:300`
  - body: The object-stream path emits a stream item for any structured object, even when it has no visible text. With `showReasoning` false, the grouped message renders no bubble content, so the reading indicator is replaced by an empty assistant row until text arrives.
  - confidence: 0.88
- **[P2] Keep thinking effort separate from reasoning visibility:** `ui/src/ui/views/chat.ts:194`
  - body: `/think` controls model effort, while `/reasoning` controls whether thinking is displayed and falls back to off. Falling back from an absent `reasoningLevel` to `props.thinkingLevel` can show live reasoning for sessions that should inherit hidden reasoning.
  - confidence: 0.84
