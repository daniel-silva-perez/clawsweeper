## Summary

Keep open. The latest branch is still a UI-only rendering change, while the live chat pipeline on current main emits and stores chat stream payloads as text-only, so the intended live reasoning behavior is not fixed end to end. The patch also introduces concrete reasoning-visibility regressions that should be resolved or consolidated with the broader overlapping #59086 work before merge.

## What This Changes

The PR updates Control UI stream item/rendering code to accept structured assistant stream messages, pass a reasoning visibility flag into streaming renders, and add grouped-render regression tests.

## Best Possible Solution

Land one coherent implementation that carries structured assistant reasoning/text payloads from gateway chat events through controller/app stream state, item building, and rendering, while keeping `/reasoning` as the sole visibility control and falling back to the reading indicator when hidden reasoning has no visible text.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists by tracing a `stream: "thinking"` event through current main: gateway chat broadcasts are text-only and `handleChatEvent` stores only extracted text, so reasoning-only live deltas are dropped before the PR's object-stream rendering branch can run.

## Solution Assessment

No. The proposed renderer-side object support is not the best complete fix because the gateway and Control UI state pipeline still need structured payload plumbing, and reasoning display must remain governed by `/reasoning`, not `/think`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Plumb structured stream payloads through chat state:** `ui/src/ui/chat/build-chat-items.ts:15`
  - body: `BuildChatItemsProps.stream` now accepts object payloads, but real live deltas still cannot populate this path: gateway chat deltas/finals are text-only and `handleChatEvent` stores `extractText(payload.message)` into a `string | null` stream. Reasoning-only deltas are still dropped before this branch runs, so the PR does not fix live reasoning streaming.
  - confidence: 0.94
- **[P2] Keep hidden reasoning streams on the reading indicator:** `ui/src/ui/chat/build-chat-items.ts:299-300`
  - body: The object-stream path emits a stream item whenever a message object exists. For thinking-only payloads with `showReasoning` false, `renderGroupedMessage` suppresses the reasoning and returns no bubble, so the existing reading indicator is replaced by an empty assistant row until answer text arrives.
  - confidence: 0.88
- **[P2] Keep thinking effort out of reasoning visibility:** `ui/src/ui/views/chat.ts:198-199`
  - body: `resolveStreamingReasoningLevel` falls back to `props.thinkingLevel`, but `/think` controls model effort while `/reasoning` controls whether thinking is displayed and falls back to off. This can show reasoning for sessions that enabled thinking effort but never enabled reasoning visibility.
  - confidence: 0.9
