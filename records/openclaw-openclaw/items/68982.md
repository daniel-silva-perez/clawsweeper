## Summary

Keep open. The latest PR branch is useful renderer-side work, but it is still not mergeable because the real live chat path remains text-only before it reaches this new object-stream renderer, and the diff can also hide the reading indicator or widen reasoning visibility contrary to the documented `/reasoning` default. The overlap with open PR #59086 should be resolved by maintainer/author judgment rather than closing this PR as implemented or superseded.

## What This Changes

This PR changes Control UI chat item construction and grouped streaming rendering to carry optional structured stream message payloads, render reasoning-only assistant content, pass reasoning visibility into streaming groups, and add grouped-render regression tests.

## Best Possible Solution

Land one consolidated end-to-end implementation that carries structured assistant thinking/text payloads from agent events through gateway chat delta/final payloads, Control UI state, item building, and streaming rendering, while keeping `/reasoning` visibility independent from `/think` effort and preserving the reading indicator when reasoning is hidden.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: current main emits text-only chat deltas/finals and `handleChatEvent` stores only `extractText(payload.message)` in a string `chatStream`, so a thinking-only live delta cannot reach the new object renderer branch. The hidden-reasoning UI regression follows from the PR emitting stream items for object messages even when `showReasoning` suppresses all visible content.

## Solution Assessment

No. Renderer-side support alone is not the best fix because the source payload is still collapsed before rendering; the safer path is end-to-end structured payload plumbing, coordinated with #59086 or explicitly split from it, while preserving the documented reasoning visibility fallback.
