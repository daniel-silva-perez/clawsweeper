## Summary

Keep open. The latest branch still leaves the live reasoning stream incomplete because the gateway chat projection and Control UI stream state on current main are text-only, so reasoning-only deltas cannot reach the new renderer path. The patch also risks showing reasoning when only thinking effort is enabled.

## What This Changes

This PR changes Control UI chat item building and grouped streaming rendering to carry optional structured stream payloads, render reasoning-only assistant content, pass reasoning visibility into streaming groups, and add grouped-render tests.

## Best Possible Solution

Land one consolidated end-to-end implementation that carries structured assistant thinking/text payloads from agent events through gateway chat delta/final payloads, Control UI state, item building, and streaming rendering, while keeping /reasoning visibility independent from /think effort and preserving the reading indicator when reasoning is hidden.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: current main emits thinking as agent events, but gateway chat events are text-only and handleChatEvent stores only extractText(payload.message) in string chatStream, so a reasoning-only live delta cannot reach the new object-stream renderer branch.

## Solution Assessment

No. Renderer-side support alone is not the best fix because the source payload is still collapsed before rendering; the safer path is end-to-end structured payload plumbing, likely coordinated with open PR #59086, while preserving the documented reasoning visibility fallback.

Full review comments:

- **[P1] Plumb structured payloads through the live stream state:** `ui/src/ui/chat/build-chat-items.ts:15`
  - body: BuildChatItemsProps.stream now accepts object payloads, but the real live path still cannot supply one: gateway chat delta/final payloads are text-only and handleChatEvent writes extractText(payload.message) into string chatStream state. Reasoning-only deltas are still dropped before this renderer branch can run, so the advertised live reasoning stream remains unfixed.
  - confidence: 0.94
- **[P2] Preserve the indicator when reasoning is hidden:** `ui/src/ui/chat/build-chat-items.ts:299-300`
  - body: The object-stream branch emits a stream item whenever a message object exists, even if it has no visible text. With showReasoning false, renderGroupedMessage suppresses the thinking block and returns nothing, replacing the previous reading indicator with an empty assistant group until answer text arrives.
  - confidence: 0.88
- **[P2] Keep thinking effort out of reasoning visibility:** `ui/src/ui/views/chat.ts:197-199`
  - body: resolveStreamingReasoningLevel falls back from a missing session reasoningLevel to props.thinkingLevel, but /think controls model effort while /reasoning controls whether reasoning is shown and falls back to off. A session with thinking enabled but reasoning visibility unset can show reasoning blocks unexpectedly.
  - confidence: 0.9
