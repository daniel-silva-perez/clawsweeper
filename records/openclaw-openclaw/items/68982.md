## Summary

Keep open. The PR targets a real live-reasoning gap, but the latest head is still a UI-only renderer change while current main emits chat deltas/finals as text-only payloads and the Control UI controller stores only extracted text, so reasoning-only live events still do not reach the new object-rendering branch. The related open #59086 remains relevant because the durable fix needs an end-to-end gateway/controller/UI contract, not just renderer support.

## What This Changes

The PR widens Control UI stream item/rendering code to accept structured assistant stream messages, passes reasoning visibility into streaming groups, renders reasoning-only assistant bubbles, and adds grouped-render regression tests.

## Best Possible Solution

Land one consolidated implementation that carries structured assistant reasoning/text payloads from gateway chat events through controller/app state, stream segments, item building, and rendering, while preserving `/reasoning` visibility semantics and keeping hidden reasoning-only streams on the normal reading indicator.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists by tracing a `stream: "thinking"` event through current main: it is emitted as an agent event, but gateway chat deltas/finals are text-only and the Control UI chat controller stores only extracted text.

## Solution Assessment

No. The current PR is not the best complete fix because renderer-side object support cannot render live reasoning unless gateway and controller stream state also preserve structured assistant payloads; it also weakens the documented separation between thinking effort and reasoning visibility.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Plumb structured stream payloads through chat state:** `ui/src/ui/chat/build-chat-items.ts:15`
  - body: `BuildChatItemsProps.stream` now accepts objects, but real live deltas still cannot pass objects here: current main emits text-only chat delta/final payloads, `handleChatEvent` stores `extractText(payload.message)` in `chatStream`, and app/render props are still `string | null`. Reasoning-only deltas are dropped before this branch can run, so the PR does not fix the live-stream path it targets.
  - confidence: 0.94
- **[P2] Keep hidden reasoning streams on the reading indicator:** `ui/src/ui/chat/build-chat-items.ts:296`
  - body: The object-stream path emits a stream item for any structured object, even when it has no visible text. With `showReasoning` false, `renderGroupedMessage` suppresses the reasoning and returns no bubble content, so the reading indicator is replaced by an empty assistant row until text arrives.
  - confidence: 0.88
- **[P2] Keep thinking effort out of reasoning visibility:** `ui/src/ui/views/chat.ts:194`
  - body: `/think` controls model effort, while `/reasoning` controls whether thinking is displayed and falls back to off. Falling back from an absent session reasoning level to `props.thinkingLevel` can show live reasoning for sessions that should inherit hidden reasoning.
  - confidence: 0.86
