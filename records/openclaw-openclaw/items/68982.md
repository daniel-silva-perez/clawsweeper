## Summary

Keep this PR open. Current main still does not implement the PR's intended live chat reasoning path: the gateway chat delta/final payloads remain text-only, Control UI chat stream state remains string-only, and the streaming renderer still constructs text-only messages with reasoning disabled. The PR is an active implementation candidate, but its provided latest diff is UI-only while the stated behavior needs end-to-end gateway-to-UI structured payload plumbing.

## Best Possible Solution

Keep this PR open for maintainer review or consolidation with #59086. The best fix is an end-to-end implementation that carries structured assistant stream payloads, including thinking and text blocks, from gateway chat events through Control UI state into rendering; respects `showThinking` and session `reasoningLevel`; preserves reading indicators when reasoning is hidden; and adds focused gateway plus UI regression coverage before landing.
