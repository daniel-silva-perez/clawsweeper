## Summary

Keep open. The latest PR diff is not safe to merge as-is: it adds Control UI rendering support for structured stream payloads, but current main still emits and stores live chat deltas/finals as text-only, so the new object-stream path cannot carry reasoning in real live chat. The diff also risks widening reasoning visibility by falling back from `/reasoning` state to `/think` effort.

## What This Changes

This PR changes Control UI chat item building and grouped streaming rendering to accept structured stream message payloads, pass reasoning visibility into streaming render calls, render reasoning-only assistant content, and add grouped-render regression coverage.

## Best Possible Solution

Land one consolidated end-to-end implementation that carries structured assistant reasoning/text payloads from gateway chat events through controller/app stream state, item building, and rendering, while keeping `/reasoning` visibility independent from `/think` effort and preserving the reading indicator when reasoning is hidden.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists by tracing a live reasoning event through current main: gateway chat deltas/finals are emitted as text-only assistant messages, and handleChatEvent stores only extractText(payload.message) into string chatStream state, so the PR's object-stream branch is unreachable in real live chat.

## Solution Assessment

No. Renderer-side support alone is not the best fix because the gateway event projection and Control UI state pipeline still need to preserve structured payloads end to end, and display logic must continue to honor `/reasoning` rather than `/think`.
