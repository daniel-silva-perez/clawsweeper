## Summary

Keep open. Current main still has the live Control UI reasoning-stream gap, and the latest PR head is not ready because its UI-only diff does not carry structured reasoning deltas through the gateway/controller stream state and can widen reasoning visibility beyond the documented `/reasoning` setting.

## What This Changes

The PR branch changes Control UI stream item construction and rendering to accept structured assistant stream messages, forwards reasoning visibility into streaming groups, renders reasoning-only assistant bubbles, and adds grouped-render regression tests.

## Best Possible Solution

Land one consolidated implementation that carries structured assistant stream payloads from gateway chat events through controller/app state, stream segments, item building, and rendering, while preserving `/reasoning` visibility semantics and keeping the reading indicator for hidden reasoning-only streams.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists by tracing a `stream: "thinking"` event through current main: the gateway emits chat deltas only for assistant text, the Control UI controller stores only `extractText(payload.message)` in a string `chatStream`, and the renderer receives only text.

## Solution Assessment

No. Renderer-side object support is not sufficient; the maintainable fix needs end-to-end structured payload plumbing and must keep `/think` effort separate from `/reasoning` display visibility.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Plumb structured stream payloads through chat state:** `ui/src/ui/chat/build-chat-items.ts:15`
  - body: `BuildChatItemsProps.stream` now accepts objects, but real live deltas still cannot pass objects here: current main emits text-only chat delta messages, `handleChatEvent` stores `extractText(payload.message)` in `chatStream`, and app/render props are still `string | null`. Reasoning-only deltas are dropped before this branch can run, so the PR does not fix the live-stream path it targets.
  - confidence: 0.94
- **[P2] Keep hidden reasoning streams on the reading indicator:** `ui/src/ui/chat/build-chat-items.ts:296`
  - body: The object-stream path emits a stream item for any structured object, even when it has no visible text. With `showReasoning` false, `renderGroupedMessage` suppresses the reasoning and returns no bubble content, so the reading indicator is replaced by an empty assistant row until text arrives.
  - confidence: 0.88
- **[P2] Keep thinking effort out of reasoning visibility:** `ui/src/ui/views/chat.ts:194`
  - body: `/think` controls model effort, while `/reasoning` controls whether thinking is displayed and falls back to off. Falling back from an absent session reasoning level to `props.thinkingLevel` can show live reasoning for sessions that should inherit hidden reasoning.
  - confidence: 0.86
