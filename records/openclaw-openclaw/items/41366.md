## Summary

Keep this issue open. Current main has related memory, active-memory recall, standing orders, mention gating, deterministic single-agent routing, and configured broadcast groups, but it still does not provide the requested governed durable natural-language rule layer or channel-neutral explicit multi-mention semantics where every explicitly mentioned bot replies once and unmentioned bots stay silent.

## Best Possible Solution

Keep this open for maintainer design. Define a governed durable rule or typed memory/rule layer with provenance, opt-in promotion, precedence, supersession, rollback, and clear interaction with `AGENTS.md`, `SOUL.md`, `MEMORY.md`, dreaming, and active memory. Separately define a channel/plugin SDK contract for explicit mention target sets so each explicitly mentioned bot or agent can produce one visible reply while unmentioned bots stay silent. Treat flowolforg/openclaw-listen-only and #42400 as useful partial listen-only/context-isolation work, not the canonical fix.

## Risks / Open Questions

- The issue combines two design surfaces: durable conversational rule promotion and group-chat multi-mention routing semantics. They may need separate product/API tracks.
- Durably promoting natural-language instructions from group chats has prompt-injection and governance risk unless provenance, operator intent, supersession, and rollback are explicit.
- A plugin can help with listen-only/context isolation, but the requested channel-neutral mention target set appears to require core/plugin SDK contract work.
- Open PR #42400 should be reviewed on its own merits for listen-only behavior, but it should not close this broader issue before merge or without covering the remaining rule and multi-mention requirements.
