## Summary

Keep this issue open. Current main has related pieces: durable Markdown memory, dreaming promotion into MEMORY.md, active-memory recall, standing orders, boolean mention gating, deterministic one-agent routing, and configured broadcast groups. It still does not implement the requested governed natural-language rule layer with precedence/supersession, nor a channel-neutral explicit multi-mention target-set contract where every explicitly mentioned bot replies once while unmentioned bots stay silent.

## Best Possible Solution

Keep this open for maintainer design. Define a governed durable rule or typed memory/rule layer with provenance, opt-in promotion, precedence, supersession, rollback, and clear interaction with AGENTS.md, SOUL.md, MEMORY.md, dreaming, and active memory. Separately define a channel/plugin contract for explicit mention target sets so each explicitly mentioned bot can produce one visible reply while unmentioned bots stay silent. Treat flowolforg/openclaw-listen-only and #42400 as useful partial context-isolation/listen-only work, not replacements for the broader product/API design.

## Risks / Open Questions

- The issue combines two product surfaces that probably need separate design tracks: durable conversational rule promotion and explicit multi-mention routing semantics.
- Durably promoting natural-language instructions from group chats has prompt-injection and governance risk, especially when observed chatter is not trusted operator intent.
- A plugin-only workaround can address listen-only/context isolation, as flowolforg/openclaw-listen-only suggests, but the requested channel-neutral target-set and rule-precedence semantics appear to need core/plugin SDK design.
- Open PR #42400 may still be worth reviewing for listen-only behavior, but it should not be treated as the canonical fix for this issue before merge.
