## Summary

Keep this issue open. Current main has durable Markdown memory, dreaming promotion, Active Memory recall, manual standing orders, mention gating, and configured broadcast groups, but it still does not provide the requested governed durable natural-language rule layer or a channel-neutral explicit multi-mention target-set contract where every explicitly mentioned bot replies once and unmentioned bots stay silent. The linked openclaw-listen-only plugin and open #42400 remain useful partial listen-only/context-isolation work, not superseding implementations.

## Best Possible Solution

Keep this open for maintainer design. Split the work into two concrete tracks: a governed durable rule or typed memory/rule layer with provenance, opt-in promotion, precedence, supersession, rollback, and documented interaction with AGENTS.md, SOUL.md, MEMORY.md, dreaming, Memory Wiki, and Active Memory; and a channel/plugin SDK contract for explicit mention target sets so channels can route one visible reply from each explicitly mentioned bot/agent while keeping unmentioned agents silent by default. Treat flowolforg/openclaw-listen-only and #42400 as partial listen-only/context-isolation work, not the canonical fix.

## Risks / Open Questions

- Promoting natural-language instructions from group chats into durable rules needs explicit operator intent, provenance, precedence, supersession, rollback, and prompt-injection safeguards.
- Multi-bot reply fan-out needs a shared channel/plugin SDK contract to avoid duplicate replies, cross-account races, and accidental responses from unmentioned agents.
- The report combines two product surfaces that may need separate design tracks: durable conversational rule promotion and explicit group-chat mention target routing.
