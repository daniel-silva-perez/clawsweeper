## Summary

Keep this issue open. Current main has related pieces: Markdown memory, dreaming promotion, Memory Wiki, active-memory recall, mention gating, Feishu mention normalization, and configured broadcast groups. It still does not implement the requested governed durable natural-language rule layer or channel-neutral multi-mention semantics where every explicitly mentioned bot replies once while unmentioned bots stay silent. The flowolforg/openclaw-listen-only comment and open #42400 are useful partial context-isolation/listen-only work, but they do not supersede the broader product/API request.

## Best Possible Solution

Keep this open for maintainer design. Define a governed durable rule or typed memory/rule layer with provenance, opt-in promotion, precedence, supersession, rollback, and documented interaction with AGENTS.md, SOUL.md, MEMORY.md, dreaming, Memory Wiki, and active memory. Separately define a channel/plugin SDK contract for explicit mention target sets so channels can route one visible reply from each explicitly mentioned bot/agent while unmentioned bots stay silent by default. Treat flowolforg/openclaw-listen-only and #42400 as useful partial listen-only/context-isolation work, not the canonical fix.

## Risks / Open Questions

- The issue combines two design surfaces that may need separate tracks: durable conversational rule promotion and explicit group-chat mention target routing.
- Durably promoting natural-language instructions from group chats has prompt-injection and governance risk unless operator intent, provenance, precedence, supersession, and rollback are explicit.
- The flowolforg/openclaw-listen-only approach and #42400 can reduce observed group-chat instruction bleed, but they are partial mitigations rather than the requested core/plugin SDK contract.
