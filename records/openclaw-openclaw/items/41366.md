## Summary

Current main has several adjacent pieces: durable Markdown memory, dreaming/short-term promotion, Active Memory recall, shared boolean mention gating, Feishu mention normalization, and Feishu/WhatsApp broadcast-style multi-agent routing. Those do not satisfy the central request for a governed durable natural-language rule layer with precedence/supersession against AGENTS.md/SOUL.md, nor a channel-neutral explicit multi-mention target-set contract where every mentioned bot replies once and unmentioned bots stay silent. The linked openclaw-listen-only plugin and open #42400 are useful partial listen-only/context-isolation work, but they do not supersede this issue.

## Best Possible Solution

Keep this open for maintainer design and split it into two concrete tracks: a governed durable rule or typed memory/rule layer with opt-in promotion, provenance, precedence, supersession, rollback, and documented interaction with AGENTS.md, SOUL.md, MEMORY.md, dreaming, Memory Wiki, and Active Memory; and a channel/plugin SDK contract for explicit mention target sets so channels can route one visible reply from each explicitly mentioned bot or agent while keeping unmentioned agents silent by default. Treat flowolforg/openclaw-listen-only and #42400 as useful partial listen-only/context-isolation work, not the canonical fix.
