## Summary

Keep this issue open. Current main has related memory and group-chat pieces, but it still does not provide the requested governed durable natural-language rule layer or a channel-neutral multi-mention target-set contract where every explicitly mentioned bot replies once and unmentioned bots stay silent. The flowolforg/openclaw-listen-only comment and open #42400 are useful partial listen-only/context-isolation work, not a superseding implementation.

## Best Possible Solution

Keep this open for maintainer design. Define a governed durable rule or typed memory/rule layer with provenance, opt-in promotion, precedence, supersession, rollback, and documented interaction with AGENTS.md, SOUL.md, MEMORY.md, dreaming, Memory Wiki, and active memory. Separately define a channel/plugin SDK contract for explicit mention target sets so channels can route one visible reply from each explicitly mentioned bot/agent while unmentioned bots stay silent by default. Treat flowolforg/openclaw-listen-only and #42400 as useful partial listen-only/context-isolation work, not the canonical fix.

## Risks / Open Questions

- The report combines two product surfaces that may need separate design tracks: durable conversational rule promotion and explicit group-chat mention target routing.
- Durably promoting natural-language instructions from group chats has prompt-injection and governance risk unless operator intent, provenance, precedence, supersession, and rollback are explicit.
- Multi-bot reply fan-out needs a channel/plugin SDK contract that avoids duplicate replies, cross-account races, and accidental replies from unmentioned agents.
