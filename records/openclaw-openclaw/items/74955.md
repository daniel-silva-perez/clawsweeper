## Summary

Keep open. Current main still lacks the tagged-TTS hook on the message-tool send path, so the PR addresses a real outbound/TTS parity bug, but the proposed hook drops the agent/account TTS resolution context when it calls the runtime helper and should be fixed before merge.

## What This Changes

The PR adds a lazy tagged-TTS runtime hook to `handleSendAction`, a focused outbound regression test, and a changelog entry for message-tool, CLI, and gateway send parity.

## Best Possible Solution

Update the PR so the new outbound hook passes `agentId` and `accountId` into `maybeApplyTtsToPayload`, add a regression case for an agent or account TTS override on message-tool sends, then land the narrow hook after normal branch validation while keeping #73483 as the separate shared `asVoice` pass-through track.

## Reproduction Assessment

Yes. A high-confidence code reproduction exists: configure tagged TTS, send `[[tts:text]]...[[/tts:text]]` through the message-tool or CLI send path, and current main reaches outbound delivery without invoking the TTS runtime hook.

## Solution Assessment

No as currently proposed. Hooking `handleSendAction` behind `shouldAttemptTtsPayload` is the right narrow design, but the runtime call must receive the same agent/account context used by the gate so documented TTS overrides behave like auto-replies.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Pass TTS context into the runtime hook:** `src/infra/outbound/message-action-runner.ts:597-601`
  - body: The new gate checks `shouldAttemptTtsPayload` with `agentId` and `accountId`, but the following `maybeApplyTtsToPayload` call omits both fields. With a documented `agents.list[].tts` or account-level TTS override, the gate can enable the hook while the runtime resolves only the global/channel config, so synthesis can use the wrong provider/voice or skip. Pass `agentId` and `accountId: accountId ?? undefined` into the runtime call and cover one override case.
  - confidence: 0.88
