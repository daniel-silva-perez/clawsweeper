## Summary

Keep open. Current main still has no configurable session-end agent turn: reset and rollover paths emit void lifecycle hooks and archive or replace the prior session, while the existing silent model/tool turn remains scoped to pre-compaction memory flush. The reporter's compaction-path comparison is valid, but the feature needs maintainer design around opt-in behavior, timeouts, recursion, delivery suppression, and tool/sandbox policy.

## What This Changes

Add a configurable session-end cleanup turn that lets the agent run with conversation history and tool access before `/new`, idle, daily, or compaction-related session teardown destroys or archives the old context.

## Best Possible Solution

Design a generic, opt-in core lifecycle feature for a bounded silent final turn before manual reset and idle/daily rollover archive the previous transcript. It should reuse the pre-compaction memory-flush machinery where appropriate, but add explicit config schema, docs, and tests for ordering, failure handling, transcript preservation, no recursion, delivery suppression, and tool/sandbox behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by following `performGatewaySessionReset` for Gateway resets and `initSessionState` for stale daily/idle rollover; both paths replace/archive the old session and only emit hooks, with no call to the agent runner.

## Solution Assessment

Unclear because there is no proposed patch. Reusing the compaction memory-flush runner is plausible, but the best maintainable solution needs explicit maintainer decisions before implementation rather than a narrow automatic fix.
