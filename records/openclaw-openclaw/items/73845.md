## Summary

Keep this open. Current main still lacks a configurable session-end agent turn: reset and rollover paths archive/rotate sessions and emit void lifecycle hooks, while the existing silent agent turn remains scoped to pre-compaction memory flush. The related #58818 is complementary raw-context carry-forward work, not a superseding tracker for this cleanup-turn feature.

## What This Changes

The issue requests an opt-in session-end agent turn that can run cleanup with conversation history and tool access before manual reset, idle timeout, or daily rollover archives the session.

## Best Possible Solution

Add a generic, opt-in core lifecycle feature for a bounded silent final turn before manual reset and idle/daily rollover archive the outgoing transcript. It should reuse the proven compaction memory-flush runner pieces where appropriate, but add first-class config schema, docs, and tests for ordering, timeout, no recursion, delivery suppression, transcript preservation, and tool/sandbox behavior.

## Reproduction Assessment

Not applicable as a defect reproduction; this is a feature request. The absence is high-confidence from current-main config/schema checks plus both reset paths showing only void lifecycle hooks and no agent-runner call.

## Solution Assessment

Unclear pending maintainer design. Reusing the compaction memory-flush pattern is a plausible implementation direction, but the proposed `sessions.onEnd.agentTurn` needs a precise config and safety contract before it is clearly the best shape.

## Security Review

Status: needs_attention

Summary: The issue requests a new silent, tool-enabled agent execution point during session teardown, so the design needs explicit security review before implementation.

Concerns:

- **[medium] Bound teardown tool execution:**
  - body: A final cleanup turn with tool access could run during reset/rollover when users expect teardown, so any implementation should be opt-in and bounded by timeout, cancellation, sandbox, and tool-allow policy.
  - confidence: 0.86
- **[medium] Prevent hidden side effects behind silent delivery:**
  - body: The compaction precedent suppresses user delivery, but a reset-time cleanup turn can still mutate files or external services; delivery suppression should not hide approvals, audit logs, or failures for side-effecting tools.
  - confidence: 0.8
