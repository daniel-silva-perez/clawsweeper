## Summary

Keep this open. Current main still does not provide a configurable session-end agent turn: session reset and idle/daily rollover emit void lifecycle hooks, while the existing silent agentic memory flush remains scoped to pre-compaction. The reporter's follow-up correctly identifies reusable precedent in the compaction path, but implementing it for reset/rollover needs maintainer design around opt-in config, timeouts, recursion, delivery suppression, and tool/sandbox policy.

## Best Possible Solution

Keep this issue open for maintainer design. The best implementation path is a generic, opt-in core lifecycle feature for a bounded silent final turn before manual reset and idle/daily rollover archive the old transcript, likely reusing the existing compaction memory-flush runner pieces while adding config schema/docs and tests for ordering, timeout, no recursion, delivery suppression, transcript preservation, and tool/sandbox behavior.
