## Summary

Close as duplicate/superseded by the still-open canonical reset-memory design thread. Current main still lacks the requested session-end agent turn, but open #45608 already tracks the same core behavior: run the existing silent agentic memory-flush pattern before `/new`, `/reset`, daily reset, and rollover destroy/archive the outgoing session, with the same unresolved design questions around blocking, timeouts, config, and safety.

## What This Changes

The issue requests a configurable session-end agent turn with full conversation history and tool access before manual reset, idle/daily rollover, or session archival destroys the outgoing session.

## Best Possible Solution

Keep one canonical open design thread on #45608 and fold this issue's configurable `sessions.onEnd.agentTurn` framing, idle-rollover coverage, and tool-access concerns into that discussion before implementing a bounded opt-in final-turn feature.

## Reproduction Assessment

Not applicable as a defect reproduction; this is a feature request. The source-level gap is high-confidence: current main has void lifecycle hooks and a compaction-only silent memory flush, with no reset/rollover agent-runner call.
