## Summary

Keep this PR open. Current main still does not resolve iMessage inbound reply/quote context from chat.db or GUID metadata when imsg omits resolved reply_to_* fields. The PR is a useful implementation candidate, but it targets the old src/imessage path and needs to be ported/refined for the active bundled plugin under extensions/imessage.

## Best Possible Solution

Keep this PR open as a useful implementation candidate, or supersede it with a maintainer PR that ports the idea into extensions/imessage/src/monitor. The final fix should enrich missing inbound reply context before inboundDebouncer.enqueue, map it into the existing ReplyToId/ReplyToBody/ReplyToSender flow, use the reliable origin GUID source, skip local database reads for remote SSH-wrapper sessions, use extension-safe public SDK seams, and add regression coverage for GUID-only replies plus permission/schema/locking failures.
