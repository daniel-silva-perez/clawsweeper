## Summary

Keep this PR open. Current main still does not resolve iMessage inbound reply/quote context from chat.db or GUID metadata when imsg omits resolved `reply_to_*` fields. The PR remains a useful implementation candidate, but it is stale against the active bundled plugin path and should be ported/refined before merge.

## Best Possible Solution

Keep this PR open as a useful implementation candidate, or supersede it with a maintainer PR that ports the idea into `extensions/imessage/src/monitor`. The final fix should enrich missing inbound reply context before `inboundDebouncer.enqueue`, map it into the existing `ReplyToId`/`ReplyToBody`/`ReplyToSender` flow, use the reliable origin GUID source, skip local database reads for remote SSH-wrapper sessions, and add regression coverage for GUID-only replies plus permission/schema/locking failures.
