Keep open. Current main does not implement the PR’s intended first-class `list` filesystem tool: there is no `list` registration, parameter group, implementation file, or tool assembly path. The PR still contains useful work, but the submitted patch has unresolved functional and security review findings, so it should be revised/reviewed rather than closed.

## Best Possible Solution

Keep the PR open for revision or maintainer review. The best path is to implement the first-class `list` tool on main with catalog/profile/group wiring, required-parameter validation, sandbox bridge support, deterministic sorting/truncation, safe symlink metadata handling, and focused tests/docs for filesystem policy behavior.

## Risks / Open Questions

- Closing this PR would drop a focused implementation candidate while current main still lacks the requested first-class directory listing tool.
- The PR patch should not merge as-is: sandboxed sessions need a `SandboxFsBridge`-backed listing path, not direct host `fs.readdir`/`fs.stat` use.
- The PR needs deterministic sorted-then-sliced truncation and clearer symlink metadata semantics, preferably using `lstat` or an explicit safe symlink policy.
- The PR needs tests for host and sandbox behavior, `tools.fs.workspaceOnly`, `group:fs` allow/deny behavior, missing/non-directory paths, >1000 entries, and symlinks; docs should be updated if the tool ships.
