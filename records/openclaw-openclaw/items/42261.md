## Summary

Current main still matches the reported regression: live reasoning production is coupled to the optional `onReasoningStream` callback, while WebChat/Control UI `chat.send` does not provide that callback. The related fix PRs were closed unmerged, so this issue still tracks real remaining work.

## Best Possible Solution

Keep this issue open. Fix `src/agents/pi-embedded-subscribe.ts` so stream reasoning state depends on `reasoningMode === "stream"` plus the existing visibility/silent gates rather than callback presence, emit `stream: "thinking"` before optionally invoking `params.onReasoningStream`, add a callback-absent regression test proving agent-event/WebSocket clients receive thinking deltas, and update the `/reasoning stream` docs and Control UI handling if visible WebChat rendering is intended.
