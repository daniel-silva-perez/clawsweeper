## Summary

Current main still matches the reported regression: live reasoning is enabled only when `onReasoningStream` exists, and `emitReasoningStream()` still returns before `emitAgentEvent({ stream: "thinking" })` for callback-less WebChat/Control UI runs. The related fix PRs in the provided context are closed unmerged, so there is no canonical merged fix to close against.

## Best Possible Solution

Keep this issue open. Fix `src/agents/pi-embedded-subscribe.ts` so streaming reasoning state depends on `reasoningMode === "stream"` plus existing visibility/silent gates rather than callback presence, emit `stream: "thinking"` before optionally invoking `params.onReasoningStream`, add a callback-absent regression test proving agent-event/WebSocket clients receive thinking deltas, and then decide whether Control UI should render those deltas directly.
