## Summary

Keep this issue open. Current main still matches the reported producer-side bug: live reasoning is only enabled when an `onReasoningStream` callback exists, while WebChat/Control UI `chat.send` does not provide that callback, so callback-less WebSocket clients cannot receive `stream: "thinking"` events.

## Best Possible Solution

Keep this issue open. Fix `src/agents/pi-embedded-subscribe.ts` so streaming reasoning state depends on `reasoningMode === "stream"` plus the existing visibility/silent gates, emit `stream: "thinking"` before optionally invoking `params.onReasoningStream`, and add callback-absent regression coverage proving WebSocket/agent-event clients receive thinking deltas.
