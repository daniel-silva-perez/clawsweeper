## Summary

Keep this issue open. Current main still matches the reported producer-side regression: WebSocket-capable agent events exist, but live reasoning is only produced when a surface-specific `onReasoningStream` callback is present, and WebChat/Control UI `chat.send` does not provide that callback.

## Best Possible Solution

Keep this issue open. Fix `src/agents/pi-embedded-subscribe.ts` so streaming reasoning state depends on `reasoningMode === "stream"` plus the existing visibility/silent gates, emit `stream: "thinking"` before optionally invoking `params.onReasoningStream`, and add a callback-absent regression test proving WebSocket/agent-event clients receive thinking deltas.
