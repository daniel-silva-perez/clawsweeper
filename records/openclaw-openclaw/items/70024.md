## Summary

Keep open. Current main still matches the reported bug: the stop-timeout branch marks the account running, returns before deleting the task/abort entries, and the start path still skips when a stale task key exists. The issue is also paired with open PR #70056, which explicitly targets this fix, so the right cleanup is to review/land or close that PR first.

## Best Possible Solution

Keep this issue open and review PR #70056, or land an equivalent focused fix that marks stop timeouts as not running, records lastStopAt, removes the stale task/abort entries, and flips the existing regression test to prove a restart can run after the timeout.
