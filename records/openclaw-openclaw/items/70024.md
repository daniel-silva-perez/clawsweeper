## Summary

Current main still matches the reported channel lifecycle bug: the stop-timeout branch marks the account running and returns before task/abort cleanup, while later starts still skip when a stale task entry exists. PR #70056 is an open, unmerged closing PR for this issue, so the issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land PR #70056, or land an equivalent focused fix in src/gateway/server-channels.ts plus src/gateway/server-channels.test.ts that clears stale task/abort entries on stop timeout, marks the account running:false, records lastStopAt, and proves a restart can run after the timeout while preserving lifecycle race safety.
