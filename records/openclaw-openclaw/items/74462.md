## Summary

Keep open. Current main still rejects the requested durable config keys, and the source/docs/release search does not show a matching `contextSafety` runtime or the reported rollover/handoff commands. This is a real contract question, but not safe to close or auto-fix as a schema-only change.

## What This Changes

The issue requests durable `contextSafety` configuration support, preferably under `session.contextSafety`, with matching schema, TypeScript types, metadata/docs, and a decision on a top-level legacy alias.

## Best Possible Solution

Decide the product contract first. If context safety is intended, implement it end to end with schema, exported config types, generated schema/help metadata, docs, validation tests, and runtime tests proving the durable path is consumed; if not, steer users to the supported `agents.defaults.compaction` controls and remove or explain any package artifact drift.

## Reproduction Assessment

Yes for the validation failure: current-main schema and validation code show `session.contextSafety` and top-level `contextSafety` are rejected. The runtime-support half is unclear because the reported runtime symbol and slash commands are absent from current source and the inspected release tag.

## Solution Assessment

Unclear. A schema-only change is not the best fix unless maintainers confirm or add the runtime contract; the safer solution is an end-to-end config/runtime/docs decision rather than exposing unsupported keys.
