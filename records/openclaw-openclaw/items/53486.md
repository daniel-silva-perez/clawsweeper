## Summary

Keep open. Current main still only routes Feishu proactive sends to cards when structured `presentation`, `interactive`, or `channelData.feishu.card` data is present; a Feishu card JSON string in the plain `message` field remains on the text path, and the fresh 2026-04-29 reproduction matches that code path.

## What This Changes

Restore Feishu proactive `message(action=send)` behavior so serialized Feishu interactive-card JSON in `message` renders as a native interactive card instead of plain text.

## Best Possible Solution

Add Feishu-local compatibility detection for valid native interactive-card JSON strings on `send` and `thread-reply`, support the known Feishu card shapes, sanitize through the existing native-card safety path, route valid cards to `sendCardFeishu`, and keep malformed card JSON or ordinary JSON on the text path. Keep shared message-tool parsing provider-agnostic unless a generic plugin seam becomes necessary.
