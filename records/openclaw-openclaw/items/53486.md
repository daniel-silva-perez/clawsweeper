## Summary

Keep this issue open. Current main still parses only structured `presentation`, `delivery`, and `interactive` parameters before message dispatch, and the Feishu send handler only converts normalized `presentation` into a card; a valid native card JSON string in the plain `message` parameter remains on the text path. The merged native-card payload work in #72667 is adjacent but does not cover this compatibility path, and the 2026-04-29 reporter confirmation keeps the regression current.

## What This Changes

Feishu should render valid interactive-card JSON supplied through `message(action=send)` as an interactive card instead of sending the JSON as plain text.

## Best Possible Solution

Add Feishu-local compatibility detection for valid native interactive-card JSON strings in `send` and `thread-reply`, sanitize through the existing native-card rules, route valid cards to `sendCardFeishu`, and leave malformed or ordinary JSON on the text path. Keep the shared message schema provider-agnostic unless a generic plugin seam is clearly needed.
