## Summary

Keep this issue open. Current main still sends a Feishu `send` or `thread-reply` plain `message` string through the Feishu text path unless the caller provides structured `presentation`, `interactive`, or explicit `channelData.feishu.card` data; the adjacent native-card payload work does not cover serialized card JSON in `message`, and the issue has a fresh reproduction comment for 2026.4.25.

## What This Changes

The issue requests restoring Feishu `message(action=send)` compatibility so valid card JSON passed in the plain `message` parameter is delivered as an interactive card instead of text.

## Best Possible Solution

Add a Feishu-owned compatibility parser for valid native interactive-card JSON strings on `send` and `thread-reply`, sanitize through the existing native-card sanitizer, route valid cards to `sendCardFeishu`, and leave malformed card JSON or unrelated JSON on the normal text path. Keep Feishu-specific parsing inside the Feishu plugin unless a generic plugin seam becomes necessary.
