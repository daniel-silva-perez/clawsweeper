## Summary

Keep open. Current main has real iMessage echo-loop hardening, but the reported duplicate/loop problem is not proven fixed: the echo cache still uses a 4s exact-text fallback without U+FFFD/control-prefix normalization, tests still codify duplicate delivery after TTL expiry, and related open follow-up work remains active.

## Best Possible Solution

Keep this issue open until maintainers land or reject the remaining iMessage echo fixes. The likely path is to review the normalization/test work in #62191 and #63666, add regression coverage for delayed and U+FFFD/control-prefixed reflected inbound messages, verify the iMessage monitor behavior on current main, then close only after the canonical fix is merged and shipped.
