## Summary

Keep this issue open. Current main still advertises ACP session listing but exposes the old `unstable_listSessions` handler, while the pinned `@agentclientprotocol/sdk` 0.20.0 dispatcher requires `listSessions` for `session/list`. The prior fix PR is closed unmerged, and the latest release tag still contains the mismatch.

## Best Possible Solution

Keep the issue open and patch `src/acp/translator.ts` so `AcpGatewayAgent` exposes `listSessions` for the pinned ACP SDK. If older SDK support is still required, retain `unstable_listSessions` as an alias that delegates to the same implementation. Add a regression test through the ACP SDK dispatcher or `AgentSideConnection` path for `session/list`, and keep the public ACP docs claim aligned with the tested behavior.
