## Summary

Keep this issue open. Current main still advertises ACP session listing but the ACP agent class only implements the old `unstable_listSessions` method, while the pinned `@agentclientprotocol/sdk` 0.20.0 dispatcher calls `agent.listSessions` for `session/list`. The prior direct-fix PR is closed unmerged, and the latest release tag still contains the same mismatch.

## Best Possible Solution

Keep this issue open and patch `src/acp/translator.ts` so `AcpGatewayAgent` exposes `listSessions` for the pinned ACP SDK. If older SDK compatibility is still intended, keep `unstable_listSessions` as a delegating alias. Add a regression test through the ACP SDK dispatcher or `AgentSideConnection` path for `session/list`, and keep `docs/cli/acp.md` aligned with the tested behavior.
