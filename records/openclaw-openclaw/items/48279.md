## Summary

Keep open. Current main still advertises ACP session listing but implements the old `unstable_listSessions` handler name, while the pinned ACP SDK 0.20.0 dispatcher requires `listSessions` for `session/list`. The related fix PR was closed unmerged, and the latest release still has the same mismatch.

## Best Possible Solution

Patch `src/acp/translator.ts` so `AcpGatewayAgent` exposes `listSessions` for the pinned ACP SDK, optionally retaining `unstable_listSessions` as a compatibility alias only if older SDK support is still required. Add a regression test that exercises `session/list` through the ACP SDK dispatcher or otherwise proves the current handler name is callable, then keep the existing ACP docs claim aligned with the tested behavior.
