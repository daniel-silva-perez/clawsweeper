## Summary

Keep open. Current main still lacks scoped mention-pattern policy support, so this PR is the active implementation candidate, but the latest diff still has a provider schema mismatch that makes several newly wired provider-scoped policies unreachable through validated config.

## What This Changes

The PR adds shared mention-pattern policy config and runtime helpers, exports them through channel SDK/runtime surfaces, wires configured regex mention gating across bundled channel handlers, and updates docs, generated schema metadata, and tests.

## Best Possible Solution

Keep the PR open and align the support boundary: either add MentionPatternsPolicySchema to every channel/account schema whose runtime now calls the policy resolver, or narrow the runtime and docs to only the providers that intentionally support provider-scoped policies, then validate the shared config and channel test lanes before merge.
