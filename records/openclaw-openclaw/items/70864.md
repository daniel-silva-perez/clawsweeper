## Summary

Keep open. Current main does not include scoped mention-pattern policy, and the latest PR branch is still an active broad implementation with a concrete account-scope schema/runtime mismatch in Slack that should be fixed before merge.

## What This Changes

The branch adds shared mention-pattern policy config/resolution, exposes it through channel SDK/runtime surfaces, wires configured pattern allow/deny gating across bundled channel handlers, and updates generated config metadata, docs, and tests.

## Best Possible Solution

Before merge, make the config contract explicit and consistent: either support account-scoped mentionPatterns by passing each channel’s resolved account policy into resolveMentionPatternsEnabled, with tests, or keep the feature provider-level only by removing account-scoped schema/metadata/docs exposure.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Honor account mention policy in Slack gating:** `extensions/slack/src/monitor/message-handler/prepare.ts:314-319`
  - body: `SlackAccountSchema` now accepts `channels.slack.accounts.<id>.mentionPatterns`, but this new call only lets the resolver read the top-level `channels.slack.mentionPatterns`. For named Slack accounts, account-scoped allow/deny policy is silently ignored, so configured policy can accept or drop group messages incorrectly. Pass the merged account policy, for example from `account.config.mentionPatterns`, or stop exposing the account-scoped key.
