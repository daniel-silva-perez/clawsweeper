## Summary

Keep open. Current main still leaves heartbeat reply payloads unsanitized, so the reported heartbeat leak remains valid, but this PR should not merge as-is because it mixes the narrow heartbeat fix with Telegram message hook plumbing that can fire before the streamed preview is finalized.

## What This Changes

The PR adds heartbeat-mode stripping of bracket-format `[TOOL_CALL]` and `[TOOL_RESULT]` blocks in `buildReplyPayloads` and also wires Telegram draft-stream `message:sent` hook emission with a dispatch test.

## Best Possible Solution

Land a focused heartbeat sanitizer in the auto-reply payload path with regression coverage for bracket-format heartbeat replies, and split any Telegram streaming hook changes into a separate PR reviewed against the existing #50917 finalization behavior.
