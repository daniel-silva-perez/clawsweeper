## Summary

Keep this PR open for maintainer review: the heartbeat leak is valid on current main, but the branch mixes that narrow fix with Telegram streaming hook timing changes that conflict with current preview-finalization semantics.

## What This Changes

The PR adds heartbeat-mode stripping of `[TOOL_CALL]`/`[TOOL_RESULT]` bracket blocks in `buildReplyPayloads` and wires Telegram draft-stream `message:sent` hook emission with test coverage.

## Best Possible Solution

Land a narrow heartbeat sanitizer in the auto-reply payload path with regression coverage for bracket-format heartbeat replies, and move any Telegram streaming hook behavior changes into a separate review against the existing #50917 finalization semantics.

## Reproduction Assessment

Yes. A source-level reproduction is clear: call `buildReplyPayloads` with `isHeartbeat: true` and a text payload containing `[TOOL_CALL]...[/TOOL_CALL]`; current main preserves the payload text on that branch.

## Solution Assessment

No. The heartbeat sanitizer is the right narrow direction, but the current PR is not the best patch because it also emits Telegram `message:sent` during preview send time instead of after finalization.
