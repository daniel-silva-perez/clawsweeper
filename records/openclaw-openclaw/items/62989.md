## Summary

Keep this PR open. Current main has partial overlap from the strict-agentic/update_plan work and the 120s LLM idle timeout, but it still does not implement the PR's central dispatch-level plan-stall fail-close or memory session-delta fetch-failure retry/backoff. The PR also still contains unresolved functional review risks and mixes several runtime/memory/session-state concerns, so the best path is a rebase and split rather than closure as obsolete.

## Best Possible Solution

Keep this PR open, request a rebase onto current main, and split it into smaller reviewed patches. The useful remaining slices are dispatch-level fail-closed handling for plan/control progress stalls, memory session-delta fetch-failure retry/backoff that preserves pending work, and memory search fallback that preserves existing hybrid/vector semantics. Treat shipped strict-agentic behavior from #64241 and the 120s idle-timeout default as partial overlap, not a replacement for the remaining dispatch and memory fixes.
