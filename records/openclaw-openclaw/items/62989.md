## What This Changes

The PR adds dispatch plan-stall abort/reply handling, memory session-delta fetch-failure retry and search degradation, subagent orphan retention, diagnostic stale-session reconciliation, command-lane fail-close state updates, LLM timeout/preemptive-compaction tweaks, and tests.

The subagent, diagnostics, command-lane, timeout, and compaction changes should land only if separately justified against current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Refresh the plan-stall timer on each plan update:** `src/auto-reply/reply/dispatch-from-config.ts:332-333`
  - body: The guard returns while `controlPlaneStallTimer` is already set, so a run that keeps emitting plan updates for more than 45 seconds before concrete output is aborted even though progress is still active. Refresh or rearm the deadline on each update.
  - confidence: 0.91
- **[P2] Clean up forwarded abort listeners on early exits:** `src/auto-reply/reply/dispatch-from-config.ts:315-317`
  - body: The forwarded abort listener is installed before plugin-bound early-return paths, but cleanup only runs in the later resolver `finally`. Handled, declined, or errored plugin-bound dispatches can leak listeners on long-lived upstream abort signals.
  - confidence: 0.87
- **[P2] Enforce retry backoff when a debounce timer exists:** `extensions/memory-core/src/memory/manager-sync-ops.ts:490-492`
  - body: The fetch-failure retry path returns whenever `sessionWatchTimer` exists, but active transcript traffic keeps that normal 5s debounce timer populated. During an outage this skips the intended 60s backoff and continues retrying at debounce cadence.
  - confidence: 0.88
- **[P2] Fall back to keyword results when embedding lookup fails:** `extensions/memory-core/src/memory/manager.ts:393-400`
  - body: When embedding fails, the patch leaves `queryVec` empty but the vector-first path still returns vector-only results when hybrid/FTS combination checks are false. That makes transient embedding outages silently produce empty memory searches instead of keyword fallback or a visible error.
  - confidence: 0.84
- **[P2] Preserve completed cleanup state for orphaned runs:** `src/agents/subagent-registry-helpers.ts:232-234`
  - body: Clearing `cleanupCompletedAt` reopens runs that were already announced and cleaned up on a prior boot. They can be retried and announced again as fresh orphan errors, overwriting terminal state and duplicating parent-facing failure messages.
  - confidence: 0.85
- **[P2] Limit stale reconciliation to tracked run sources:** `src/logging/diagnostic.ts:402-409`
  - body: The heartbeat flips any `processing` session to `idle` when it is missing from the embedded/reply registries, but some dispatch paths do not register there. Active plugin inbound turns can be reclassified as idle mid-flight, suppressing stuck-session diagnostics and skewing state telemetry.
  - confidence: 0.86
- **[P2] Resolve the true session before fail-closing main lane:** `src/process/command-queue.ts:71-72`
  - body: Mapping the shared `main` lane to `agent:main:main` marks the wrong session idle when a failover error belongs to another run using that lane. The real session remains stuck while diagnostics point at the main session.
  - confidence: 0.86
