## Summary

Keep this PR open. Current main still uses the numbered `.bak`/`.bak.1`-through-`.bak.4` config-backup ring, the latest release has the same behavior, the linked issue #39923 is still open, and parallel PR #39982 is another unmerged implementation candidate. The PR is focused and not supply-chain sensitive, but migration of legacy backups and timestamp collision behavior still need maintainer resolution.

## Best Possible Solution

Keep this PR open for maintainer review alongside #39923 and #39982. The best implementation path is one consolidated datetime-backup design in `src/config/backup-rotation.ts` with collision-safe destinations, bounded retention, owner-only permission hardening, and migration behavior that preserves or safely converts legacy `.bak.N` recovery points until equivalent datetime retention exists; update `src/config/config.backup-rotation.test.ts` and add docs/changelog only if the behavior ships.
