## Risks / Open Questions

- Merging this PR as-is may delete existing legacy `.bak.N` rollback points on the first config write after upgrade, reducing recovery history before the datetime ring is populated.
- The PR's timestamp destination remains collision-sensitive: same-millisecond writes can overwrite or lose one backup slot depending on platform semantics.
- There is another open implementation candidate for the same linked issue, so maintainers should choose or consolidate one approach before landing.
