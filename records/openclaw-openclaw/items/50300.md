## Summary

Keep this PR open. Current main still does not implement the proposed local-offset transcript timestamp behavior: transcript headers and multiple SessionManager append/rewrite paths still use UTC `toISOString()` output or direct `SessionManager.open(...)`, while the PR remains a focused implementation candidate for the linked timestamp bug.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a narrow transcript timestamp fix that centralizes explicit local-offset ISO formatting for OpenClaw-owned transcript headers and SessionManager-managed entries, covers branch/compaction/gateway-injection paths, adds regression tests for preserving the instant with an explicit offset, and handles malformed existing timestamps without persistent session open/write failures.

## Risks / Open Questions

- The PR approach wraps private `@mariozechner/pi-coding-agent` internals (`_appendEntry`, `_rewriteFile`, `flushed`, `fileEntries`), so maintainers should decide whether this wrapper is acceptable or whether OpenClaw needs an upstream/public SessionManager seam.
- The security review comment identifies a low-severity availability risk: the proposed formatter throws on malformed existing timestamp values, which could make a corrupted transcript fail repeatedly until repaired. A final fix should make normalization best-effort or otherwise non-fatal.
