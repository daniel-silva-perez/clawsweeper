## Summary

Keep this PR open. Current main still lacks the transcript-local timestamp helper/wrapper proposed by this PR, and multiple session transcript creation and SessionManager append/rewrite paths still persist UTC-style `toISOString()` timestamps. The linked bug #50201 is closed, but the observable fix is not present on main, so this PR remains a relevant implementation candidate rather than cleanup-close material.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a narrow transcript timestamp fix on main that centralizes explicit local-offset ISO formatting for manual headers and SessionManager-managed entries, covers branch/compaction/gateway-injection paths, adds regression tests for preserving the instant with an explicit offset, and handles malformed existing timestamps without making session open/write paths fail persistently. Maintainers should also decide whether wrapping private SessionManager internals is acceptable or whether OpenClaw needs a durable upstream/public API seam.
