## Summary

Keep this PR open. Current main still writes session transcript headers and SessionManager-managed entries through UTC-style `toISOString()` paths, and the transcript-local timestamp helper/wrapper proposed by this PR is not present on main. The PR remains a focused implementation candidate, but it needs maintainer follow-up on the private SessionManager hook strategy and malformed timestamp handling raised in review.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a narrow transcript timestamp fix on current main that centralizes explicit local-offset ISO formatting, covers manual transcript headers plus SessionManager append/rewrite/branch/compaction/gateway-injection paths, adds regression tests for offset formatting preserving the instant, and makes normalization non-fatal for malformed existing transcript timestamps. Maintainers should also decide whether the private SessionManager wrapper is acceptable or whether OpenClaw should add or request a more durable public dependency seam.
