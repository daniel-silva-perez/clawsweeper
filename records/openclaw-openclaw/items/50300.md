## Summary

Keep this PR open. Current main still does not implement the PR’s intended local-offset transcript timestamp behavior: OpenClaw-owned transcript headers still use `new Date().toISOString()`, several transcript append/branch/rewrite paths still call `SessionManager` directly, and the pinned `@mariozechner/pi-coding-agent` 0.70.5 source still emits UTC `...Z` timestamps internally. The PR remains a focused implementation candidate, but it still needs maintainer review for the private-API wrapper and malformed-timestamp availability concern raised in the discussion.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a narrow transcript timestamp fix that centralizes explicit local-offset ISO formatting for OpenClaw-owned transcript headers and SessionManager-managed entries, covers header, append, branch, compaction, gateway injection, and truncation paths, adds regression tests for preserving the instant with an explicit offset, and makes malformed existing timestamp normalization best-effort/non-fatal.
