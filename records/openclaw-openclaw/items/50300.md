## Summary

Keep this PR open. Current main still writes session transcript headers and SessionManager-managed entries through UTC-style `toISOString()` paths, and the PR’s local-offset formatter/wrapper is not present on main. The PR remains a focused implementation candidate, with maintainer-review work still needed around the private SessionManager hook strategy and malformed timestamp handling raised in review.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a narrow transcript timestamp fix on current main that centralizes local-offset ISO formatting, covers manual transcript headers plus SessionManager append/rewrite/branch/compaction/gateway-injection paths, adds regression tests for explicit local offsets preserving the same instant, and makes normalization best-effort for malformed existing JSONL timestamps.

## Risks / Open Questions

- The PR hooks private `@mariozechner/pi-coding-agent` SessionManager internals (`_appendEntry`, `_rewriteFile`, `fileEntries`, and `flushed`); maintainers should verify that contract or prefer an upstream/public API seam before merge.
- The Aisle review reports a low availability concern in the PR implementation: malformed persisted timestamp values can throw during normalization, potentially making a corrupted session repeatedly fail until repaired.
- The linked bug report #50201 is closed, but current main still does not implement the PR’s intended transcript timestamp behavior, so closing this PR as obsolete would drop a still-relevant focused fix path.
