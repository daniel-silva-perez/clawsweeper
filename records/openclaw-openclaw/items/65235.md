## Summary

Keep this issue open. Current main suppresses raw validation details and default exec-like warnings in some paths, but write/message-style parameter validation failures still become mutating-tool warning payloads on normal chat surfaces, and the related open PR #68070 only addresses retryability metadata rather than final reply suppression.

## What This Changes

Requests that tool parameter validation failures be handled internally by the agent runtime and not delivered as raw or synthetic warnings to Feishu/DM chat surfaces.

## Best Possible Solution

Add a typed validation-error classification from the parameter validation wrapper through the adapter, tool-result extraction, `ToolErrorSummary`, and final payload policy. Validation-only failures should remain visible to the model/runtime and logs, but normal chat replies should suppress synthetic mutating-tool warnings unless verbose diagnostics are enabled; real side-effect failures should continue to warn.
