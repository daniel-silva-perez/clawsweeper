## Summary

Keep this issue open. Current main suppresses raw validation details and now suppresses default exec-like warnings, but write/message-style required-parameter failures still flow through the mutating-tool warning branch and produce a user-facing generic failure payload. The related open PR #68070 addresses retryability metadata for validation errors, not chat-surface suppression.

## What This Changes

Suppress tool-parameter validation failures from normal user chat surfaces while still returning/logging them internally for the agent runtime.

## Best Possible Solution

Add typed validation-error metadata through the tool validation wrapper, adapter, tool-result summary, and final payload policy. Validation-only failures from write/edit/message-like tools should be logged and returned to the model/runtime, but normal chat replies should suppress the synthetic mutating-tool warning unless verbose diagnostics are enabled; real side-effect failures should continue to produce warnings.
