## Summary

Keep open. Current main still only partially addresses the report: transcript/tool image dimensions and OpenAI-compatible chat-completion image budgets are configurable, but there is no shared `media.image` contract and fixed image byte/input-pixel caps remain in sanitizer, generic media, gateway attachment, sandbox staging, and ACP paths.

## What This Changes

The issue requests configurable image byte, input-pixel, dimension, and per-request limits across OpenClaw media sanitization layers.

## Best Possible Solution

Define a backwards-compatible, documented `media.image` limit contract with global defaults and scoped overrides, preserve existing aliases such as `agents.defaults.imageMaxDimensionPx`, and thread the resolved limits through agent sanitization, media image-ops, generic media caps, media-store/sandbox staging, gateway attachment parsing, ACP image reads, and OpenAI-compatible HTTP paths.
