## Summary

Keep open. Current main partially addresses the report with `agents.defaults.imageMaxDimensionPx` and configurable OpenAI-compatible chat-completion image budgets, but it still lacks the requested shared `media.image` limit contract and several sanitizer, generic media, gateway, sandbox, and ACP caps remain fixed.

## What This Changes

Expose configurable image byte, input-pixel, dimension, and total-request limits across image sanitization, media, gateway, sandbox, ACP, and OpenAI-compatible paths.

## Best Possible Solution

Define a backwards-compatible, documented `media.image` contract with global defaults and scoped overrides, preserve existing aliases such as `agents.defaults.imageMaxDimensionPx` where needed, then thread the resolved limits through agent sanitization, media image-ops, generic media caps, media-store/sandbox staging, gateway attachment parsing, ACP image reads, and OpenAI-compatible HTTP paths.

## Risks / Open Questions

- A cross-layer image limit contract changes public config, generated schema/help, resource ceilings, and multiple ingestion paths, so the exact config shape needs maintainer design before implementation.
- Raising image byte and pixel limits can increase memory, CPU, provider-payload, and disk exposure unless defaults, maximums, and per-scope bounds are chosen deliberately.
- Open related slice work (#40950, #73710, #61041) overlaps parts of the problem but does not define the shared contract, so changes should be coordinated to avoid divergent aliases.
