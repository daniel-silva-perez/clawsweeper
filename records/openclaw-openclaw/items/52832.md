## Summary

Keep open. ## What This Changes

The PR rebuilds the iOS home bottom toolbar as floating Liquid Glass-style status/action capsules, switches the toolbar from a bottom safe-area inset to an overlay, extends ScreenTab into the bottom safe area, starts the app through RootView, and adds a DeepSeek provider contract registry import/entry.

## Best Possible Solution

Rebase and narrow the branch to an iOS-only bottom-bar redesign that uses supported iOS 26 glass surfaces, preserves bottom content reachability and compact-layout behavior, removes the DeepSeek registry change, and coordinates with #40874 only if maintainers want one broader Liquid Glass direction.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Restore content avoidance for the floating toolbar:** `apps/ios/Sources/RootCanvas.swift:478-479`
  - body: Changing the toolbar from a bottom safe-area inset to a plain overlay removes the reserved vertical space for the control. Since ScreenWebView disables UIKit inset adjustment and the canvas scaffold only pads for the OS safe area, bottom canvas content can render underneath the new toolbar and become hidden or hard to tap.
  - confidence: 0.88
- **[P2] Let the status label compress in compact widths:** `apps/ios/Sources/HomeToolbar.swift:90`
  - body: The added fixedSize(horizontal: true) makes the status title non-compressible while the trailing action cluster still consumes fixed width. With a longer state like Connecting plus an activity icon, compact widths or larger Dynamic Type can overflow and clip the action buttons that need to remain reachable.
  - confidence: 0.82
