## Summary

Keep open: current main still has the preexisting iOS toolbar strip, while this active PR remains unmerged, dirty against main, and has concrete layout concerns plus an unrelated provider-registry hunk that needs maintainer review before merge.

## What This Changes

The PR rebuilds the iOS home bottom toolbar as Liquid Glass-style floating status/action capsules, moves it from a bottom safe-area inset to an overlay, switches app startup to RootView, expands ScreenTab into the bottom safe area, and adds a DeepSeek provider contract registry entry.

## Best Possible Solution

After rebase and cleanup, land a narrow iOS-only bottom-bar redesign that uses supported iOS 26 glass surfaces while preserving bottom content reachability and compact-layout behavior; keep DeepSeek/provider registry work out of this branch and coordinate with #40874 if maintainers want one Liquid Glass direction.

- **[P2] Restore content avoidance for the floating toolbar:** `apps/ios/Sources/RootCanvas.swift:479`
  - body: Changing the toolbar from a bottom safe-area inset to a plain overlay removes the reserved vertical space for the control.
