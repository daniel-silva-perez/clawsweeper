## Summary

Keep open. Current main has already solved the original doctor false-positive with a simpler unconditional trajectory exclusion, but this PR still proposes distinct context-aware behavior to preserve `.trajectory`-suffixed session IDs, so it is not obsolete. The patch should not land as-is because no-context helper calls now classify `.trajectory.jsonl` as primary/usage-counted again, and the branch still mixes the session fix with unrelated node exec changes plus failing CI.

## What This Changes

The PR changes session artifact classification and scanner call sites so `.trajectory.jsonl` is treated as a sidecar only when a matching primary transcript exists, adds doctor/session tests, updates memory-host/session usage scans, and also changes node exec explicit-workdir handling.

## Best Possible Solution

Land a narrowed session-artifact fix that makes directory context explicit, keeps sidecars out of no-context/public helper behavior or exposes a separate disambiguating API, covers doctor/usage/disk-budget/memory-host behavior, and splits node exec workdir changes into a separate PR if they are still needed.
