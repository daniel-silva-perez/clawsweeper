## Summary

Keep this PR open. Current main still only archives stale isolated heartbeat keys, while same-key isolated-session rotation can overwrite the prior store entry without archiving its transcript; the PR is a narrow fix for the linked live bug, with one low-priority policy finding for the missing changelog entry.

## What This Changes

The PR captures the previous isolated heartbeat session before overwrite, archives its transcript with `reason: "reset"`, and adds a regression test for the same-key rotation case.

## Best Possible Solution

Land this narrow heartbeat-owned fix after adding the required changelog entry and running the targeted heartbeat test plus the normal changed gate; once merged, the linked bug can close as implemented.

## Reproduction Assessment

Yes. Seed the isolated heartbeat store with an existing `<base>:heartbeat` entry and a matching `<previousSessionId>.jsonl`, then run `runHeartbeatOnce`; current main overwrites the store entry without a same-key archive path, while the PR's regression test exercises that exact path.

## Solution Assessment

Yes, with one policy cleanup. Capturing `previousSessionId` before the overwrite and using the existing transcript archive helper with `reason: "reset"` is the narrowest maintainable fix because it reuses the same live-reference protection already used for deleted stale keys.
