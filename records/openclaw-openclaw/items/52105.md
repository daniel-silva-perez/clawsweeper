## Summary

Keep this issue open. Current main still has the Control UI local slash-command dispatch gap: the typed command is cleared before dispatch, the direct dispatch path is not wrapped in a catch, and a missing browser client still returns silently. The `/think` executor itself catches its normal `sessions.list` / `models.list` failures, but the browser dispatch/injection path identified in the issue discussion is still not fixed. The linked fix PR #52113 proposed the right direction but is closed unmerged.

## Best Possible Solution

Keep the issue open and implement the dispatch-layer fix: make `dispatchSlashCommand` return an inline system message when the Control UI is not connected, wrap unexpected local slash-command failures so they are injected into chat instead of escaping after input clearing, and add `app-chat` regression tests for `/think` or another local command when the client rejects or is unavailable. The closed PR #52113 is a useful reference, but the fix needs to land on main with coverage.
