## Summary

Keep this issue open. Current main still supports `/think` as a Control UI local slash command, but the direct Control UI dispatch path can still clear the input before a silent no-client return or an uncaught dispatch failure. The `/think` executor catches its normal RPC failures, so the remaining actionable gap is in `ui/src/ui/app-chat.ts`, matching the issue discussion and the closed, unmerged #52113 direction.

## Best Possible Solution

Keep the issue open and land a focused Control UI dispatch fix: in `ui/src/ui/app-chat.ts`, inject a visible system message when a local slash command cannot run because the browser client is unavailable, wrap unexpected `executeSlashCommand`/dispatch failures so they render inline instead of escaping after input clearing, and add `app-chat` regression tests for missing client and rejected dispatch using `/think` or another local command. #52113 is a useful reference, but the fix still needs to land on main with coverage.
