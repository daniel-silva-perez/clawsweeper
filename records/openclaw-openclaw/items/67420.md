## Summary

Keep this PR open. Current main still lacks per-agent dreaming control, and this PR is paired with the same-author open feature issue #67413. The latest PR revision usefully addresses the earlier schema/test review gap, but it still needs maintainer review for all-disabled fallback behavior and scope relative to per-agent schedules.

## What This Changes

This PR adds `agents.list[].dreaming.enabled` to the agent config type, strict Zod schema, generated base schema, docs, and memory dreaming workspace resolution, with a unit test for mixed enabled/disabled agents.

## Best Possible Solution

Keep the PR open and either update it or land a successor that adds per-agent dreaming config across the exported type, strict schema, generated schema/help/docs, and workspace resolver; preserves default-agent fallback only when no agents are configured; skips explicitly disabled agents without reintroducing fallback work; and adds tests for mixed enabled/disabled, all-disabled, and config validation cases.
