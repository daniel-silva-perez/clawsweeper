## Summary

Keep open. Current main only partially satisfies the requested private-network support: SearXNG web_search and the shared SSRF guard have private-network policy support, but web_fetch and Gateway OpenResponses input_file/input_image URL fetches still expose only strict or narrow exceptions.

## What This Changes

The issue requests configurable private/local network access for web tools and gateway URL fetch paths while preserving default SSRF protections.

## Best Possible Solution

Keep this as the broader tracker until the default-off web_fetch opt-in is reviewed and a separate, documented Gateway OpenResponses URL-input policy is designed with tests for redirects, DNS resolution, allowlists, and cache isolation.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: add the requested key to web_fetch config and the strict schema has no field for it, while the runtime builds no private-network policy; Gateway input_file/input_image URL fetches hardcode allowPrivateNetwork: false.

## Solution Assessment

Unclear as proposed. A default-off, explicitly named policy switch is the maintainable direction, but automatically enabling private-network access in local gateway mode should wait for maintainer security/product approval.
