## Summary

Keep this issue open. Current main has partial support for trusted SearXNG web_search endpoints and model-provider private-network request transport, but the requested core surfaces are still incomplete: built-in web_fetch has no allowPrivateNetwork config/policy path, and Gateway OpenResponses input_file/input_image URL fetches force private-network denial.

## What This Changes

The issue requests a default-off private/local network access option for web_fetch, web_search/image tooling, and Gateway URL fetches, with local gateway deployments able to opt in or auto-enable it.

## Best Possible Solution

Add a narrow, documented, default-off private-network policy for built-in web_fetch and Gateway OpenResponses URL inputs, propagate it only through intended media/image fetch paths, keep SearXNG/provider-request support as partial precedent, and cover redirects, DNS, cache partitioning, and schema/docs behavior before enabling any local-mode default.
