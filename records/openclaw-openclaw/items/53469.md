## Summary

Keep this open. Current main has useful partial support for trusted web_search provider endpoints, including SearXNG, and model-provider request transport, but the broader request is still missing for generic web_fetch and Gateway OpenResponses URL file/image inputs.

## What This Changes

Request to add private/local network access opt-ins for web tools and Gateway URL fetching, with backward-compatible defaults and possible local-gateway behavior.

## Best Possible Solution

Add a narrow, default-off private-network policy for web_fetch and Gateway OpenResponses URL inputs, propagate it to remote media/image references that already inherit web_fetch SSRF policy, document the security model, and keep SearXNG/provider-request support as partial precedent rather than treating it as full resolution.
