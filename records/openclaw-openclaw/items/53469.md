## Summary

Keep open. Current main only partially satisfies the request: SearXNG web_search and model-provider request transport have private-network support, but web_fetch still lacks a broad allowPrivateNetwork config/runtime path and Gateway OpenResponses input_file/input_image URL fetches still force private-network denial. The remaining work changes SSRF-protected user-input URL surfaces, so it needs maintainer security/product review before implementation.

## What This Changes

The issue requests opt-in private-network access for web_fetch, web_search, image-related URL fetches, and Gateway URL fetches, with possible local-gateway auto-enable behavior.

## Best Possible Solution

Define a narrow, documented, default-off private-network policy for tools.web.fetch.ssrfPolicy and Gateway OpenResponses files/images URL fetches, propagate it only into the intended fetchWithSsrFGuard calls with cache-key isolation and redirect/DNS coverage, and decide separately whether any local-mode default should exist.
