## Summary

Keep this PR open. Current main has adjacent persistent-session recovery for normal ACP turns and part of the no-stable-identity hardening, but it still lacks the PR's bounded `/acp model` / `setSessionConfigOption()` retry and the narrowed `ACP_TURN_FAILED` stale-binding cleanup path. The PR remains a useful implementation candidate, though it needs normal maintainer review or a rebase/port before merge.

## What This Changes

This PR hardens persistent ACP `--bind here` recovery by treating absent resume state as retryable, adding a fresh-handle retry around `setSessionConfigOption()`, expanding stale-binding cleanup for narrowed `ACP_TURN_FAILED` resume failures, and adding manager/dispatch tests.

## Best Possible Solution

Keep this PR open for maintainer review or port its still-missing behavior onto current main: add a bounded fresh-handle retry around `setSessionConfigOption()`, add the narrow `ACP_TURN_FAILED` stale-binding predicate for persistent-session resume failures, preserve metadata-write failure signaling with `failOnError`, and keep regression tests that distinguish stale session 404s from unrelated resource 404s.
