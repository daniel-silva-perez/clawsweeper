## Summary

Keep this PR open. Current main has provider-discovery scoping for explicit startup provider ids, but it still does not derive non-live discovery scope from configured agent model refs; the PR is useful source work, but it needs maintainer coordination with the newer startup-discovery changes and the existing fallback test gap before merge.

## What This Changes

The PR adds configured model/provider hint collection to scope non-live provider discovery to owning plugin ids and adds regression tests for scoped and broad fallback behavior.

## Best Possible Solution

Adapt one canonical implementation on top of current main: combine explicit startup provider ids with configured provider/model refs, resolve owners through the existing metadata snapshot and model-owner seams, keep conservative fallback behavior when ownership cannot be proven, and cover defaults, fallbacks, subagent models, named model entries, and broad fallback cases in targeted tests.

## Reproduction Assessment

Not applicable as a performance/behavior PR rather than a crash report. The source-level reproduction is high-confidence: on current main, a non-live config with only agents.defaults.model does not influence onlyPluginIds because resolveProviderDiscoveryFilter returns undefined without providerDiscoveryProviderIds.

## Solution Assessment

Unclear that this exact patch is the best final form. The direction is right, but the patch should be rebased into the current startup-discovery implementation, coordinated with #63552 and #73557, and tightened around unresolved-owner fallback behavior before merge.
