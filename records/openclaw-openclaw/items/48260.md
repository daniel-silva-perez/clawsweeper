## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Re-probe cached legacy gateways before blocking Local mode:** `ui/src/ui/controllers/usage.ts:425-426`
  - body: When the persisted compatibility cache says a gateway rejected date-interpretation params, this new branch throws in Local mode before sending another request. A gateway upgraded at the same URL will therefore keep failing until the browser storage entry is cleared, even though it may now support the fields.
  - confidence: 0.89
- **[P2] Preserve open-ended usage date ranges:** `src/gateway/server-methods/usage.ts:431-432`
  - body: This turns any request with only startDate or only endDate into INVALID_REQUEST. Those params are still independently optional in the protocol schema, and current main falls back to the days/default window when one bound is missing, so older clients can start failing after this deploy.
  - confidence: 0.86
- **[P2] Cache the time-zone date formatter in session scans:** `src/infra/session-cost-usage.ts:182-187`
  - body: formatDateKeyInTimeZone creates a new Intl.DateTimeFormat for every transcript entry visited by the usage scanners. Large histories can allocate thousands of formatters per Usage page load; cache one per time zone as the Gateway helper already does.
  - confidence: 0.8
