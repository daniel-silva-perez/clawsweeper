## Summary

Keep this PR open. Current main still lacks the Active Time overview card, but the latest proposed branch is not merge-ready because it can mix clipped active duration with full-session throughput totals and it omits the generated Control UI i18n metadata expected by the repo workflow.

## What This Changes

This PR adds an Active Time card to the Control UI Usage Overview, clips duration stats for selected day/hour filters, adds usage metrics/render tests, and updates locale strings for the new labels.

## Best Possible Solution

Land the Active Time card as a narrow Control UI change using the existing duration stats, keep throughput numerators and denominators on the same filtered scope, and regenerate Control UI i18n outputs through the repository pipeline.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the feature gap, and the blocking PR issues are reproducible from the selected-hour totals path plus the Control UI i18n source-hash/metadata contract.

## Solution Assessment

No. The feature direction is reasonable, but this diff needs metric-scope alignment and generated i18n output before it is the narrowest maintainable solution.
