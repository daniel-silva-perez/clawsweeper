## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Re-probe cached legacy gateways before blocking Local mode:** `ui/src/ui/controllers/usage.ts:425-426`
  - body: When the persisted compatibility cache says date-interpretation params are unsupported, this branch throws in Local mode before sending another request. A gateway upgraded at the same URL can therefore keep failing until browser storage is cleared, even though it may now support the fields.
  - confidence: 0.89
- **[P2] Preserve open-ended usage date ranges:** `src/gateway/server-methods/usage.ts:431-432`
  - body: This turns any request with only startDate or only endDate into INVALID_REQUEST. Those fields are still independently optional in the public schema, and current main falls back to the days/default window when one bound is missing, so older clients can start failing after this deploy.
  - confidence: 0.86
- **[P2] Cache the time-zone formatter in session scans:** `src/infra/session-cost-usage.ts:182-187`
  - body: formatDateKeyInTimeZone creates a new Intl.DateTimeFormat for every transcript entry visited by the usage scanners. Large histories can allocate thousands of formatters per Usage page load; cache one per time zone as the Gateway helper already does.
  - confidence: 0.8
- **[P2] Localize the Active Time overview copy:** `ui/src/ui/views/usage-render-overview.ts:513-517`
  - body: The new Active Time title, hint, and sublabel are hardcoded in the renderer while current Usage Overview copy uses i18n keys. Add English keys and regenerate locale outputs so non-English UI builds can translate the card.
  - confidence: 0.84
