## Summary

Keep this PR open. Current main still does not show a separate Active Time summary card, so the contribution is not obsolete, but the current branch is too broad and has blocking correctness issues around legacy gateway compatibility, open-ended date ranges, and session-scan formatter allocation.

## What This Changes

The branch adds an Active Time card to the Control UI Usage Overview and expands usage date interpretation with IANA time-zone support across the web UI, Gateway usage APIs, session-cost bucketing, protocol schema, tests, and Swift protocol models.

## Best Possible Solution

Land a narrow, localized Active Time overview card using the existing duration stats with focused UI and metrics coverage, and split the IANA time-zone protocol, Gateway, session-cost, and Swift client changes into a separate compatibility-reviewed PR after fixing the identified regressions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Re-probe cached legacy gateways before blocking Local mode:** `ui/src/ui/controllers/usage.ts:423-424`
  - body: When the persisted compatibility cache says a gateway rejected date-interpretation params, this branch throws in Local mode before sending another request. A gateway upgraded at the same URL will keep failing until browser storage is cleared, even though it may now support the fields.
  - confidence: 0.89
- **[P2] Preserve open-ended usage date ranges:** `src/gateway/server-methods/usage.ts:433-435`
  - body: This turns any request with only startDate or only endDate into INVALID_REQUEST. Those params remain independently optional in the protocol schema, and current main falls back to the days/default window when one bound is missing, so older clients can start failing after this deploy.
  - confidence: 0.86
- **[P2] Cache the time-zone date formatter in session scans:** `src/infra/session-cost-usage.ts:184-190`
  - body: formatDateKeyInTimeZone creates a new Intl.DateTimeFormat for each day-key lookup, and the helper is called inside per-entry transcript scans. Large histories can allocate thousands of formatters per Usage page load; cache one per time zone as the Gateway helper already does.
  - confidence: 0.8
- **[P2] Localize the Active Time overview copy:** `ui/src/ui/views/usage-render-overview.ts:513-517`
  - body: The new Active Time title and sublabel are hardcoded in the renderer, while current Usage Overview copy is routed through i18n keys. Add the English keys and generated locale updates so non-English UI builds can translate this card.
  - confidence: 0.84
