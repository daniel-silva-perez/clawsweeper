## Summary

Keep this PR open. The Active Time feature is still absent from current main, but the branch is not merge-ready because it now combines that UI card with broader Gateway protocol, session-cost, Control UI, and Swift client changes that have unresolved compatibility and performance regressions.

## What This Changes

This PR adds an Active Time card to Usage Overview and expands usage date interpretation with IANA time-zone support across the Control UI, Gateway usage handlers, session-cost bucketing, protocol schema, tests, and Swift protocol models.

## Best Possible Solution

Land a narrow, localized Active Time card that reuses the existing duration stats with focused UI and metrics coverage, and move the IANA time-zone protocol/Gateway/session-cost/Swift client work into a separate compatibility-reviewed PR.

## Reproduction Assessment

Not applicable as a bug reproduction for the requested feature; current main inspection is enough to show the card is missing. The blocking PR regressions have high-confidence code-level repro paths: cached legacy gateway state, one-bound date params, and large transcript scans through the new time-zone formatter.

## Solution Assessment

No. The Active Time card direction is reasonable, but this PR is not the best way to solve it because it couples the UI display change to broader protocol/client date-interpretation behavior that needs a separate compatibility design.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Re-probe cached legacy gateways before blocking Local mode:** `ui/src/ui/controllers/usage.ts:425-426`
  - body: When the compatibility cache says date-interpretation params are unsupported, this branch throws in Local mode before sending another request. A gateway upgraded at the same URL can remain broken until browser storage is cleared, even though it may now support the fields.
  - confidence: 0.89
- **[P2] Preserve open-ended usage date ranges:** `src/gateway/server-methods/usage.ts:431-432`
  - body: This rejects requests with only startDate or only endDate. Those fields are still independently optional in the public schema, and current main falls back to the days/default window, so older clients can start failing after this deploy.
  - confidence: 0.86
- **[P2] Cache the time-zone formatter in session scans:** `src/infra/session-cost-usage.ts:182-187`
  - body: formatDateKeyInTimeZone creates a new Intl.DateTimeFormat for each transcript entry visited by the usage scanners. Large histories can allocate thousands of formatters per Usage page load; cache one per time zone like the Gateway helper does.
  - confidence: 0.8
- **[P2] Localize the Active Time overview copy:** `ui/src/ui/views/usage-render-overview.ts:513-517`
  - body: The new Active Time title, hint, and sublabel are hardcoded in the renderer while current Usage Overview copy uses i18n keys. Add English keys and regenerate locale outputs so non-English UI builds can translate the card.
  - confidence: 0.84
