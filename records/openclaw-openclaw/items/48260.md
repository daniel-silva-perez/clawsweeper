## Summary

Keep open. Current main still lacks the Active Time overview card, and the latest PR branch still has a blocking correctness issue: it clips active duration for day/hour filters while leaving throughput totals on the whole-session scope, so filtered throughput can be inflated.

## What This Changes

This PR adds an Active Time card to the Control UI Usage Overview, clips duration stats for selected day/hour filters, adds focused usage metrics/render tests, and syncs Control UI locale strings plus i18n metadata.

## Best Possible Solution

Land the Active Time card as a narrow Control UI improvement after making every derived metric use the same filtered scope: either compute scoped token/cost totals for day/hour filters or suppress throughput where only clipped duration is available.

## Reproduction Assessment

Yes. Source inspection gives a concrete reproduction: select one hour within a multi-hour session and the PR clips active duration while still using whole-session token/cost totals for throughput.

## Solution Assessment

No. The feature direction is reasonable, but the current implementation is not the best merge shape until throughput numerator and denominator are aligned under selected day/hour filters.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep throughput aligned with filtered active time:** `ui/src/ui/views/usage.ts:325-329`
