## Summary

Keep open. Current main still lacks a separate Active Time summary card, so the PR addresses a real UI gap, but the latest diff is not merge-ready because it can mix full-session throughput totals with clipped active duration and it omits required generated Control UI i18n metadata.

## What This Changes

The PR adds an Active Time card to the Control UI Usage Overview, adds focused usage metrics/rendering tests, clips duration stats to selected day/hour filters, and adds localized copy across Control UI locale bundles.

## Best Possible Solution

Land the Active Time card as a narrow Control UI improvement, but first keep throughput numerators and duration denominators on the same filtered scope and commit the generated i18n outputs expected by the Control UI localization pipeline.

## Reproduction Assessment

Yes. Current-main source inspection shows the feature gap, and the blocking PR issues have code-level reproduction paths: a partially overlapping filtered session exercises the full-totals/clipped-duration mismatch, and added English source keys without updated ui/src/i18n/.i18n outputs exercise the i18n drift check.

## Solution Assessment

No. The feature direction is maintainable, but this implementation should be revised to align filtered throughput math and regenerate i18n metadata before it is the best merge path.
