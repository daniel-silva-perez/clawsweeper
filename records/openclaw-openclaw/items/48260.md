## Summary

Keep this PR open. Current main still lacks a separate Active Time overview card, so the PR is not obsolete, but the branch should not merge as-is because it combines the UI feature with Gateway protocol, session-cost, and Swift client date-interpretation changes that introduce compatibility and performance regressions already called out in prior review comments.

## What This Changes

The PR adds an Active Time summary card to Usage Overview, adds usage metrics tests, and expands usage date interpretation with IANA time-zone support across the Control UI, Gateway usage APIs, session-cost bucketing, protocol schema, and Swift protocol models.

## Best Possible Solution

Land a narrow, localized Active Time overview card using the existing duration stats with focused UI/metrics coverage, and split the IANA time-zone protocol, Gateway, session-cost, and Swift client changes into a separate compatibility-reviewed PR.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Re-probe cached legacy gateways before blocking Local mode:** `ui/src/ui/controllers/usage.ts:423-424`
  - body: When the persisted compatibility cache says a gateway rejected date-interpretation params, this branch throws in Local mode before sending another request. A gateway upgraded at the same URL will keep failing until browser storage is cleared, even though it may now support the fields.
  - confidence: 0.89
- **[P2] Preserve open-ended usage date ranges:** `src/gateway/server-methods/usage.ts:433-435`
  - body: This turns any request with only startDate or only endDate into INVALID_REQUEST. Those params remain independently optional in the protocol schema, and current main falls back to the days/default window when one bound is missing, so older clients can start failing after this deploy.
  - confidence: 0.86
- **[P2] Cache the time-zone formatter during session scans:** `src/infra/session-cost-usage.ts:184-190`
  - body: formatDateKeyInTimeZone creates a new Intl.DateTimeFormat for every transcript entry visited by the usage scanners. Large histories can allocate thousands of formatters per Usage page load; cache one per time zone as the Gateway helper already does.
  - confidence: 0.8
- **[P2] Localize the Active Time overview copy:** `ui/src/ui/views/usage-render-overview.ts:513-517`
  - body: The new Active Time title, hint, and sublabel are hardcoded in the renderer, while current Usage Overview copy is routed through i18n keys. Add the English keys and generated locale updates so non-English UI builds can translate this card.
  - confidence: 0.84
