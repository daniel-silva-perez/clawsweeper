## Summary

Current main still gives Feishu WebSocket an ambient proxy-derived agent, and the existing regression tests still expect proxy inheritance when HTTP(S) proxy env is set. The same contributor has an open, unmerged implementation PR with closing syntax, so this issue should stay open until #65802 is reviewed and either merged or closed.

## Best Possible Solution

Keep this issue open and review #65802 as the implementation candidate. The likely fix belongs in `extensions/feishu/src/client.ts`: bypass Feishu WebSocket ambient proxy inheritance by default, add a deliberate opt-in if maintainers want proxy support, and update `extensions/feishu/src/client.test.ts`; docs should mention any new opt-in env if it becomes a supported user-facing escape hatch.
