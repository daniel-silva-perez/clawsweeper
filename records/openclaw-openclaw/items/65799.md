## Summary

Current main still passes an ambient proxy-derived agent into Feishu WebSocket clients when HTTP(S) proxy env is configured, and the existing regression tests still assert that behavior. The issue should remain open because #65802 is an open, unmerged PR with closing syntax that directly implements the requested Feishu-plugin-local fix.

## Best Possible Solution

Keep this issue open and review #65802 as the focused implementation candidate. The fix should remain inside the Feishu plugin boundary: bypass ambient proxy inheritance for Feishu WebSocket by default, provide an explicit opt-in only if maintainers want to support proxied Feishu WS deployments, update `extensions/feishu/src/client.test.ts`, and document any supported opt-in env if it ships.
