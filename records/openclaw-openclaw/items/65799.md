## Summary

Current main still routes Feishu WebSocket construction through `resolveAmbientNodeProxyAgent()` and the regression tests still expect proxy env to create a WS agent. This issue should stay open because open PR #65802 has closing syntax and directly implements the requested Feishu-local change, but it has not merged.

## What This Changes

Make Feishu WebSocket bypass ambient proxy environment variables by default, with an explicit opt-in if proxied Feishu WS is supported.

## Best Possible Solution

Keep this issue open and review #65802 as the focused implementation candidate. The fix should stay inside the Feishu plugin boundary, update `extensions/feishu/src/client.ts` and `extensions/feishu/src/client.test.ts`, and document any supported `OPENCLAW_FEISHU_WS_USE_PROXY` opt-in if it ships, while checking it against the public network-proxy contract.
