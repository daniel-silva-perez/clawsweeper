## Summary

Keep this PR open. The requested default-local behavior is plausible and still not implemented on current main, but this branch removes a released and documented `openclaw logs --local-time` flag while adding `--utc`, so it needs maintainer coordination with the overlapping implementation before merge.

## What This Changes

This PR changes `openclaw logs` timestamp option parsing so human-readable output defaults to local time by replacing `--local-time`/`localTime` with `--utc` and deriving display mode from `!opts.utc`.

## Best Possible Solution

Land one coordinated implementation that makes human-readable `openclaw logs` default to local time, keeps `--local-time` accepted as a compatibility alias or follows an explicit deprecation path, exposes a clear UTC opt-out, updates docs and tests, then resolves the overlapping PR and linked issue.

## Reproduction Assessment

Yes. Current main shows the existing behavior in source and tests: without `--local-time` or a valid `TZ`, `localTime` stays false, and the test suite still exercises the released `--local-time` parser option.

## Solution Assessment

No. Defaulting to local time is a reasonable direction, but removing the released `--local-time` flag is not the narrowest maintainable fix; preserving it while adding a UTC opt-out is safer.
