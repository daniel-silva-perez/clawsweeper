## Summary

Keep open. Current main and the latest release still use the old UTC-by-default behavior unless `--local-time` is passed or `TZ` is valid, so this PR remains relevant to #46748. The branch is not merge-ready as-is because it removes the documented/released `--local-time` flag and does not update the matching tests or docs.

## What This Changes

This PR changes `openclaw logs` timestamp display to default to local time, adds a `--utc` opt-out, removes the `--local-time` option/type field, and drops the direct `isValidTimeZone` use from `src/cli/logs-cli.ts`.

## Best Possible Solution

Revise this PR or a replacement implementation so human-readable `openclaw logs` defaults to local time, `--utc` opts back into UTC, the released `--local-time` flag remains compatible or has an explicit maintainer-approved deprecation path, and `src/cli/logs-cli.test.ts`, `docs/cli/logs.md`, and `docs/logging.md` describe the new contract before closing #46748.
