## Summary

Keep this PR open. Current main still does not implement default-local `openclaw logs` output, but this branch is not ready to merge because it removes the released and documented `--local-time` flag while another open PR proposes a different compatibility contract for the same linked issue.

## What This Changes

This PR changes `openclaw logs` timestamp option parsing so human-readable output defaults to local time by replacing `--local-time` with `--utc` and deriving display mode from `!opts.utc` in `src/cli/logs-cli.ts`.

## Best Possible Solution

Land one coordinated implementation that makes human-readable `openclaw logs` default to local time, keeps `--local-time` accepted as a compatibility alias or follows an explicit deprecation path, exposes a clear UTC opt-out, updates docs and tests, then resolves the overlapping PR and linked issue.

## Reproduction Assessment

Yes. Current main provides a high-confidence source/test reproduction: `localTime` is false unless `--local-time` or a valid `TZ` is supplied, and `formatLogTimestamp` still returns the UTC ISO string by default.

## Solution Assessment

No. Defaulting logs to local time is a reasonable direction, but removing the released `--local-time` flag is not the narrowest maintainable fix; preserving it while adding an explicit UTC opt-out is safer.
