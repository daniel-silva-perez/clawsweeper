## Summary

Keep this PR open. The requested default-local-time behavior is still relevant on current main, but this branch would break the already shipped `--local-time` CLI contract and overlaps with another open PR that chooses a different UTC opt-out shape, so it needs maintainer/author follow-up before merge.

## What This Changes

The branch changes `src/cli/logs-cli.ts` so human-readable `openclaw logs` output defaults to local time, adds `--utc`, and removes the existing `--local-time` option.

## Best Possible Solution

Land one coordinated implementation that defaults human-readable `openclaw logs` output to local time, keeps the shipped `--local-time` flag accepted as a compatibility alias/no-op or follows an explicit maintainer-approved deprecation path, exposes a clear UTC opt-out, and updates CLI tests plus `docs/cli/logs.md` and `docs/logging.md` before closing #46748.

## Reproduction Assessment

Yes. The compatibility regression is directly reproducible from the diff: current main registers, documents, and tests `openclaw logs --local-time`, while this branch deletes that option, so existing scripts would hit an unknown-option path.

## Solution Assessment

No. Defaulting to local time is a reasonable fix direction, but removing the released `--local-time` flag is not the narrowest maintainable solution; preserving it as an alias/no-op or using an explicit deprecation/conflict path is safer.
