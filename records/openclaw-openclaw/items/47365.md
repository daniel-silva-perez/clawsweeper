## Summary

Keep this PR open, but it should not merge as-is. The default-local behavior is a reasonable fix direction for #46748, but the current branch removes the shipped and documented `--local-time` flag while an overlapping open PR (#46782) proposes a different compatibility contract.

## What This Changes

This PR changes `src/cli/logs-cli.ts` so human-readable `openclaw logs` output defaults to local time by deriving `localTime` from `!opts.utc`, removes `--local-time`, and adds `--utc` for UTC output.

## Best Possible Solution

Land one coordinated implementation that makes human-readable `openclaw logs` default to local time, keeps `--local-time` accepted as a compatibility alias/no-op or follows an explicit deprecation path, exposes one clear UTC opt-out, updates docs and tests, then resolves the overlapping PR and closes #46748.

## Reproduction Assessment

Yes. Current main shows the existing behavior in source and tests: without `--local-time` or a valid `TZ`, `localTime` stays false, while this branch removes the currently accepted `--local-time` parser option.

## Solution Assessment

No. Defaulting to local time is a reasonable fix direction, but removing the released `--local-time` flag is not the narrowest maintainable solution; preserving it while adding a UTC opt-out is safer.
