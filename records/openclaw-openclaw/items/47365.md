## Summary

Keep open. The underlying `openclaw logs` default-local-time request is still not implemented on current main, but this PR is not merge-ready because it removes a released `--local-time` CLI flag and conflicts with the shipped docs/tests and overlapping PR #46782.

## What This Changes

The PR changes `openclaw logs` timestamp handling in `src/cli/logs-cli.ts` so human-readable output defaults to local time and `--utc` opts back into UTC.

## Best Possible Solution

Land one coordinated implementation that defaults human-readable `openclaw logs` output to local time, keeps the shipped `--local-time` flag accepted as a compatibility alias or follows an explicit deprecation decision, adds a clear UTC opt-out, and updates the CLI tests plus `docs/cli/logs.md` and `docs/logging.md` before closing #46748.
