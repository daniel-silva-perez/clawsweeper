## Summary

Keep open: the user problem is still present on current main, and this PR fixes it in a direction that would break a shipped CLI flag. The related open implementation #46782 proposes a different opt-out contract, so the remaining decision is the compatibility shape, not cleanup closure.

## What This Changes

The PR changes `src/cli/logs-cli.ts` so human-readable `openclaw logs` output defaults to local time, adds `--utc`, and removes the existing `--local-time` option/type field.

## Best Possible Solution

Land one coordinated implementation that defaults human-readable `openclaw logs` output to local time, keeps the shipped `--local-time` flag accepted as a compatibility alias/no-op or follows an explicit maintainer-approved deprecation path, exposes a clear UTC opt-out, and updates CLI tests plus `docs/cli/logs.md` and `docs/logging.md` before closing #46748.

## Reproduction Assessment

Yes. The PR regression is reproducible from the diff against current main: `openclaw logs --local-time` is documented, tested, and shipped today, but this branch removes the option registration so existing scripts would hit an unknown-option path.

## Solution Assessment

No. Defaulting to local time is a reasonable fix direction, but removing the released `--local-time` flag is not the narrowest maintainable solution; preserving it as an alias/no-op or using an explicit deprecation/conflict path is safer.
