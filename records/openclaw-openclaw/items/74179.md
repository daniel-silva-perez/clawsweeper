## Summary

Current main and v2026.4.26 already solve the observable Feishu cross-app failure this PR targets: subagent completion bookkeeping stores the parent/requester origin, and bound completion routing rejects child bindings whose channel account differs from the requester account. The PR's proposed parent-first reorder is not needed for that bug and the branch is not safe as-is because it deletes the `router` declaration while still using `router`.

## Best Possible Solution

Close this PR as already handled by the shipped mainline fix. Keep the current split-route behavior: completion bookkeeping preserves the parent/requester delivery account, cross-account child bindings cannot take over announce delivery, and same-account thread completions can still route to the child thread. Do not merge this branch unless it is rewritten with tests and a maintainer intentionally changes that routing contract.
