## Summary

Keep this PR open. Current main still lacks the PR's central feature: no `gateway.tailscale.controlUrl` schema/type, no `--tailscale-control-url` CLI override, no runtime validation or daemon ControlURL comparison, no Headscale/custom-control-server docs, and no focused test coverage. The linked feature issue is closed, but this PR is unmerged and remains the active implementation candidate rather than work already shipped or superseded.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to finish this PR or an equivalent core Gateway implementation with `gateway.tailscale.controlUrl` in schema/types/generated config metadata, an http/https-only CLI flag, runtime validation that handles official vs custom control servers, daemon ControlURL comparison in the supported tailnet/off mode, redacted errors/logs, public Tailscale/Headscale docs, and focused tests. If maintainers decide OpenClaw should not first-class this behavior, close it explicitly as a product decision rather than as implemented.
