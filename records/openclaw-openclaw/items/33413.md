## Summary

Current main still does not implement Slack tool-level assistant thread status updates. Slack dispatch still sets the assistant thread status to the static "is typing..." at reply start and clears it at stop; tool-start events are available and already feed Slack reactions/draft-preview progress, but they do not call `setSlackThreadStatus` with per-tool labels. The direct implementation PR #33420 is closed unmerged, and related open work #42555/#73054/#67129 does not supersede this specific request.

## Best Possible Solution

Keep this issue open and implement the narrow Slack-owned wiring: add a Slack-local tool-name-to-status-label helper, update `ctx.setSlackThreadStatus` from Slack dispatch on tool-start when an assistant thread target exists, preserve the existing start and clear behavior, avoid update-phase spam, and add focused Slack coverage for the status call sequence. Document the behavior if it becomes a supported Slack UX surface; keep #42555/#67129 as separate configurable-status discussions unless maintainers choose to merge the designs.

Preserve the existing static start status and final clear behavior, and do not change Slack draft-preview semantics, status reactions, message delivery, or native streaming behavior.
