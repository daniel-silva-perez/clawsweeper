## Summary

Current main still supports Discord native reply references via `replyToMode`, but not the requested opt-in visible `<@originalAuthorId>` injection. The Discord config schema and docs have no `autoMentionReply` field, the send paths only attach `message_reference`, and the delivery/plugin hook surfaces carry `replyToId` rather than the original author ID needed to synthesize a mention. The existing Ryce comment is directionally useful: this belongs in the Discord plugin formatting/delivery layer, but it is not implemented yet.

## Best Possible Solution

Keep this issue open. The best fix is an opt-in Discord plugin change: add a typed/schema/docs config field such as `channels.discord.autoMentionReply`, resolve the original author from inbound reply context or by safely fetching the referenced Discord message, prepend `<@authorId>` only to the first visible outbound guild reply when absent, and add focused tests across reply modes, DMs, existing mentions, chunking, streaming, media/components, and webhook paths. If maintainers want this outside the bundled plugin, first add a plugin API/hook surface that exposes enough reply-author context safely.
