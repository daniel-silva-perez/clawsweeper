## Risks / Open Questions

- The PR changes shared `parseFenceSpans` behavior used by markdown code spans, auto-reply chunking, canvas shortcode extraction, and embedded block chunking, so the stricter closing-fence rule needs maintainer review across those consumers.
- The PR diff makes `splitMediaFromOutput` case-sensitive for `MEDIA:` parsing, while current main still buffers streaming directive tails case-insensitively in `src/auto-reply/reply/streaming-directives.ts`; that alignment should be settled before merge.
- The indented-code heuristic is intentionally approximate and may need explicit list, blockquote, and container-context tests before maintainers are comfortable landing it.
- The security-sensitive surface is functional rather than supply-chain: media extraction controls hidden attachment delivery and text stripping, so false-positive and text-loss regressions need focused tests even though the diff does not add third-party execution or dependency risk.
