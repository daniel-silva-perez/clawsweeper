## Summary

Keep this PR open. Current main still has the reported indented-code false-positive, but the proposed patch is not merge-ready because it changes MEDIA directive case handling without aligning block-reply parsing, streaming buffering, and existing lowercase coverage.

## What This Changes

The PR updates media and Markdown fence parsing to skip `MEDIA:` extraction in indented code blocks, preserve retained code indentation, reject invalid fence closers with trailing text, make MEDIA token matching case-sensitive, and add parser tests.

## Best Possible Solution

Land a focused parser fix that preserves code-block `MEDIA:` examples without silently changing existing directive compatibility, or intentionally changes case semantics across every MEDIA surface with matching tests and documentation. If the fence-closing tightening stays, validate all shared fence consumers as part of the same review.
