## Summary

Keep open. Current main still has the reported indented-code MEDIA extraction gap, but the latest PR head is not merge-ready because it makes MEDIA parsing case-sensitive while current block-reply and streaming surfaces still treat lowercase `media:` as directive-shaped input, and it still has an audio-tag cleanup path that can strip a retained tab-indented code line.

## What This Changes

The PR updates media and Markdown fence parsing to skip `MEDIA:` extraction from indented code blocks, preserve retained code indentation, tighten fence closing validation, remove case-insensitive MEDIA token matching, and add parser regression tests.

## Best Possible Solution

Land a focused version that keeps the indented-code protection and code-indentation preservation, but first makes MEDIA case semantics consistent across parser, block-reply, streaming buffering, docs, and tests, and validates shared `parseFenceSpans` consumers if the fence change remains in scope.
