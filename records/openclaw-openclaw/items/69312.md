## Summary

Keep this PR open. Current main still has the reported indented-code MEDIA extraction problem, so the work is not obsolete; however, the current patch still changes lowercase `media:` handling without updating the block-reply and streaming surfaces that currently treat lowercase directives as valid.

## What This Changes

The PR changes media and fence parsing to skip `MEDIA:` directives in indented code blocks, preserve retained code indentation, tighten fence closing validation, make MEDIA matching case-sensitive, and add parser regression tests.

## Best Possible Solution

Land a focused version after case semantics are made consistent across `splitMediaFromOutput`, block-reply parsing, streaming directive buffering, and tests. Keep the indented-code protection and code-indentation preservation, and validate the shared fence parser consumers if the fence change remains part of the patch.
