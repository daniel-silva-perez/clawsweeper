## Summary

Keep this PR open. Current main still has the reported indented-code MEDIA extraction path, and this PR is still the active implementation candidate rather than cleanup material. The linked issue being closed does not make the unmerged PR obsolete because main does not contain the intended behavior.

## Best Possible Solution

Keep the PR open and have a maintainer review or land a corrected version that adds indented-code protection in src/media/parse.ts, preserves code indentation during cleanedText normalization, adds the media/fence regression tests, and deliberately aligns MEDIA case handling across splitMediaFromOutput(), streaming-directives buffering, and reply delivery behavior.
