## Summary

Keep this PR open. Current main still lacks indented-code protection for `MEDIA:` extraction, so the contribution is not obsolete, but the branch also changes lowercase directive semantics and shared fence parsing behavior that need maintainer review before merge.

## What This Changes

The PR updates media and markdown fence parsing plus tests to skip `MEDIA:` extraction from indented code blocks, preserve retained code indentation during cleanup, make `MEDIA:` matching case-sensitive, and reject fenced-code closers with trailing text.

## Best Possible Solution

Land a focused parser fix that preserves `MEDIA:` lines in indented code blocks, keeps retained code indentation intact after real media extraction, and explicitly documents or tests the chosen case-sensitivity contract. If the shared fence parser change remains, validate all current consumers before merging and add a user-facing changelog fix entry.
