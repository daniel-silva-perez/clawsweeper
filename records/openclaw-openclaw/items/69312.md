## Summary

Keep open. The underlying bug is real on current main, but this PR is not ready to merge because it changes tested lowercase media directive behavior and the exact PR head already fails a relevant auto-reply check; there is also a remaining tab-indentation edge case after audio tag stripping.

## What This Changes

The PR updates media and fence parsing plus tests to skip MEDIA extraction from indented code blocks, preserve retained code indentation, reject malformed fence closers with trailing text, and make MEDIA token matching case-sensitive.

## Best Possible Solution

Land a focused version after keeping lowercase MEDIA semantics consistent across `splitMediaFromOutput`, block-reply normalization, streaming buffering, and tests, then reuse the retained-code-line state after audio tag stripping and validate the shared fence-parser consumers touched by the stricter close rule.
