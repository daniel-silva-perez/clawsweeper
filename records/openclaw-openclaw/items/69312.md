## Summary

Keep this PR open. Current main still extracts `MEDIA:` from 4-space/tab-indented code lines because `splitMediaFromOutput` only skips fenced spans before trimming indentation and matching `MEDIA:`. The PR remains a plausible implementation candidate, but it needs maintainer review because it also changes `MEDIA:` case semantics while current streaming/directive tests still treat lowercase `media:` as valid, and it changes the shared fence parser used by several chunk/render consumers.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a narrow indented-code fix for `splitMediaFromOutput` with focused regressions, make an explicit product decision on lowercase `media:` compatibility and align parse/delivery/streaming behavior, validate the shared `parseFenceSpans` consumers, and add a user-facing changelog fix entry if the bug fix lands.
