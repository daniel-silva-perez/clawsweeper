## Summary

Keep this issue open. Current main still lacks per-Telegram-group or per-topic session reset configuration: the config types and strict Zod schemas do not accept the proposed nested `session.reset` shape, and the runtime reset resolver still receives only global, type, and channel-level policy inputs. The linked implementation PR #8134 is closed unmerged, and #20888 is a broader closed lifecycle request rather than a canonical tracker for this exact feature.

## Best Possible Solution

Keep this issue open. Implement an additive, account-aware reset override for Telegram group/topic config, or introduce a generic conversation-scoped reset resolver if maintainers want the behavior across plugins. Update Telegram config types, strict Zod schema, generated labels/help, public docs, and focused precedence tests covering topic/group over channel over type/global, including multi-account Telegram cases.
