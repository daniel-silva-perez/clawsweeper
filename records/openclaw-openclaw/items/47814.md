## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Reject collapsed empty segments in named-DM parsing:** `src/sessions/session-key-utils.ts:184-185`
  - body: `parseNamedDmSessionKey` delegates to `parseAgentSessionKey`, which drops empty `:` segments before the named-DM shape check. A malformed key like `agent:main:dm-named:123::work` can therefore parse as a valid `{ peerId: "123", name: "work" }`; validate the raw suffix or reject empty segments first.
  - confidence: 0.92
- **[P2] Fail closed for malformed stored active names:** `src/gateway/session-utils.ts:866-870`
  - body: `activeNamedSession` is persisted store data, but `getActiveNamedSessionKey` passes it directly to `buildNamedDmSessionKey`. A manually edited or stale row containing `:` can throw during active-session resolution despite the helper's `string | null` contract; validate the stored name and return `null` or repair it.
  - confidence: 0.9
- **[P2] Prioritize dm-named before token chat-type checks:** `src/sessions/session-key-utils.ts:50`
  - body: Adding `dm-named` to the direct-token branch after `group` and `channel` still misclassifies named DM keys whose user-chosen name is `group` or `channel`. Put the named-DM shape check before arbitrary token scans so direct-message sessions cannot inherit group/channel policy rules.
  - confidence: 0.91
- **[P2] Keep the changelog entry in the active section only:** `CHANGELOG.md:109`
  - body: The branch repeats the named-DM changelog bullet in historical release sections. That rewrites old release history and can confuse release-note tooling; keep one credited entry in `Unreleased` or the active release section for this change.
  - confidence: 0.96
- **[P3] Fix the helper nullability docs:** `docs/concepts/session.md:79`
  - body: The docs describe null/undefined-safe helper behavior too broadly, but `buildNamedDmSessionKey` requires a params object and throws for invalid inputs. Narrow the docs to the parser/predicate helpers or change the implementation and types to match.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff does not add dependencies, CI, secrets, or package execution paths, but named-DM key classification can misapply send-policy rules for user-controlled names.

Concerns:

- **[medium] Named-DM names can collide with group/channel policy tokens:** `src/sessions/session-key-utils.ts:50`
  - body: Because the PR classifies `dm-named` only after `group` and `channel` token checks, a direct-message session named `group` or `channel` can be treated as that chat type when send policy falls back to key-derived classification. That can apply the wrong outbound allow/deny policy to a DM session.
  - confidence: 0.86
