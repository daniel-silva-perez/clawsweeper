## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reject empty segments before parsing named-DM keys:** `src/sessions/session-key-utils.ts:184-185`
  - body: `parseNamedDmSessionKey` delegates to `parseAgentSessionKey`, which drops empty `:` segments before this length check. A malformed key like `agent:main:dm-named:123::work` can be accepted as `{ peerId: "123", name: "work" }`; validate the raw suffix or reject empty segments before returning a parsed named-DM key.
  - confidence: 0.92
- **[P2] Fail closed for malformed persisted active names:** `src/gateway/session-utils.ts:862-870`
  - body: `activeNamedSession` is persisted session-store data, but this helper passes the stored value directly into `buildNamedDmSessionKey`. If an existing or manually edited store row contains `:`, active-session lookup throws despite the `string | null` contract; validate the stored name and return `null` or repair it before building the key.
  - confidence: 0.9
- **[P2] Classify dm-named keys before user name tokens:** `src/sessions/session-key-utils.ts:50`
  - body: Adding `dm-named` to the direct-token branch after `group` and `channel` still misclassifies valid named DM keys whose user-chosen name is `group` or `channel`. Since send policy can derive chat type from the key, recognize the named-DM shape before scanning arbitrary name tokens.
  - confidence: 0.91
- **[P2] Keep the named-DM changelog entry in Unreleased:** `CHANGELOG.md:109`
  - body: The branch repeats the named-DM changelog bullet in a historical release section. That rewrites old release history and can confuse release-note tooling; keep a single credited entry in the active changelog section for this change.
  - confidence: 0.96
- **[P3] Fix the helper nullability docs:** `docs/concepts/session.md:79`
  - body: The docs say all three helpers accept `null` or `undefined`, but `buildNamedDmSessionKey` requires a params object and will throw if called that way. Narrow this sentence to the parse/predicate helpers or change the implementation and types to match.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff does not add dependencies, CI, secrets, or package execution paths, but the named-DM chat-type change can misapply send-policy rules for user-controlled names.

Concerns:

- **[medium] Named-DM names can hit group/channel send policies:** `src/sessions/session-key-utils.ts:50`
  - body: Because `dm-named` is only added after the `group` and `channel` token checks, a DM session named `group` or `channel` can be classified as that chat type when policy falls back to key-derived chat type, applying the wrong outbound send-policy rules to a direct-message session.
  - confidence: 0.86
