## Summary

Keep open. Current main still lacks the named-DM key helpers, `activeNamedSession` store field, gateway helpers, and docs this PR proposes, so the work is not obsolete or implemented; however, the proposed patch has concrete blocking correctness issues already surfaced in prior review comments and still needs maintainer product/design review before any merge or replacement branch.

## What This Changes

This PR adds named-DM session key utilities and tests, an optional `activeNamedSession` field on `SessionEntry`, gateway helpers for active named sessions, session documentation, and changelog entries without wiring routing behavior.

## Best Possible Solution

Keep this work open for maintainer review or a narrow rebased replacement. If named-DM session switching is still wanted, land it with strict raw key parsing, fail-closed persisted-state handling, explicit `dm-named` direct classification before arbitrary token checks, focused malformed-key and policy-collision tests, contributor docs that distinguish primitives from routed behavior, and one credited changelog entry in the active section only.

## Reproduction Assessment

Yes. The code-level paths are deterministic: `agent:main:dm-named:123::work` is accepted after empty-segment filtering, `activeNamedSession: "foo:bar"` throws during lookup, and `agent:main:dm-named:123:channel` can be classified as `channel` when send policy falls back to key-derived chat type.

## Solution Assessment

No. The infrastructure direction may be reasonable, but this patch is not the best mergeable solution until the parser validates raw segments, stored active names fail closed, named-DM chat type takes precedence over name tokens, changelog history is cleaned up, and the branch is rebased onto current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reject empty named-DM key segments:** `src/sessions/session-key-utils.ts:180-185`
  - body: `parseNamedDmSessionKey` relies on `parseAgentSessionKey`, which drops empty `:` segments before this code validates the named-DM shape. A malformed key like `agent:main:dm-named:123::work` can therefore parse as peer `123` and name `work`; validate the raw segment count or explicitly reject empty segments before returning a parsed key.
  - confidence: 0.92
- **[P2] Fail closed for malformed active session names:** `src/gateway/session-utils.ts:866-870`
  - body: `activeNamedSession` is persisted store data, but `getActiveNamedSessionKey` passes the stored name directly into `buildNamedDmSessionKey`. If an existing or manually edited row contains `:`, active-session resolution throws instead of returning `null`; validate the stored value and ignore or repair malformed names before building the key.
  - confidence: 0.9
- **[P2] Classify dm-named before group and channel tokens:** `src/sessions/session-key-utils.ts:50`
  - body: Adding `dm-named` to the direct branch after the `group` and `channel` checks still misclassifies valid named DMs whose user-chosen name is `group` or `channel`. Since send policy can derive chat type from the session key, those direct-message sessions can pick up group/channel policy rules unless `dm-named` is recognized before scanning arbitrary name tokens.
  - confidence: 0.91
- **[P2] Keep the changelog entry in Unreleased only:** `CHANGELOG.md:109`
  - body: The same named-DM changelog bullet is repeated in a historical release section. That rewrites old release history and can confuse release-note tooling; keep a single credited entry under `Unreleased` for this unshipped change.
  - confidence: 0.96
- **[P3] Correct the helper nullability docs:** `docs/concepts/session.md:79`
  - body: The docs say all three helpers accept `null` or `undefined`, but `buildNamedDmSessionKey` requires a params object and immediately reads `params.agentId`. Either narrow the sentence to the parser/predicate helpers or change the implementation and types to match the documented contract.
  - confidence: 0.86
