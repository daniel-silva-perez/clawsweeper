## Summary

Keep open. Current main still uses shared main-session heartbeat runs unless `isolatedSession: true` is explicitly set, and the docs/tests still describe unset as the shared-session/default-false path. The PR is not obsolete, but it should be updated against current main’s newer isolated-session event-drain behavior before maintainers consider landing it.

## Best Possible Solution

Keep this PR open as a valid implementation candidate, but update it on current main. The best path is a narrow default change from opt-in to opt-out isolation, preserving current base-session event consumption and isolated delivery-routing safeguards, with tests for unset default isolation, explicit `isolatedSession: false`, main-session abort recovery, and cron/wake event consumption under the new default.
