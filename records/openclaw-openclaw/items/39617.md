## Summary

Keep this PR open. Current main still does not fully implement the requested fix: Telegram native commands now reload runtime config and have regression coverage, but Slack slash commands and Discord native commands still resolve routes with provider-startup config snapshots while `resolveAgentRoute` derives DM session keys from `session.dmScope`. The provided PR is also stale against the current pluginized paths, so the remaining work needs a rebase rather than closure.

## Best Possible Solution

Keep this PR open or replace it with a fresh PR linked to #39605. Rebase the remaining fix onto `extensions/discord/src/monitor/native-command.ts`, `extensions/discord/src/monitor/native-command-route.ts`, and `extensions/slack/src/monitor/slash.ts`; load current runtime config narrowly before native/slash route resolution and configured binding readiness; leave Telegram's current fresh-runtime-config path intact; and add Slack/Discord regression tests that change `session.dmScope` or bindings after provider startup without recreating the provider.
