## Summary

Keep this PR open. Current main still has the reset/queue bug the PR targets: reset-triggered turns compute `effectiveResetTriggered`, but active-run queue policy still ignores it, so `steer`, `followup`, `collect`, and `steer-backlog` can enqueue the reset turn behind an active run. The PR is a focused implementation candidate for #74093, with no supply-chain-sensitive diff, but it needs maintainer review and likely broader reset-run coverage before merge.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this focused core reset-queue fix or a narrow equivalent, with regression coverage proving reset-triggered turns bypass active-run followup/steer queues, clear the session lane, abort the active run, and preserve existing bare `/new`/`/reset` acknowledgements plus normal non-reset queue modes. The later #74093 session-rotation/hang note should be verified before deciding whether it needs a follow-up issue or additional tests in this PR.
