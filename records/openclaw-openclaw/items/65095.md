## Summary

Keep this open. Current main still enforces strict gateway protocol matching and the macOS app still treats older gateway versions as incompatible; the docs and tests describe that strict behavior rather than a grace-period, compatibility flag, or offline mode. The linked #65081 covered broader release cadence and is closed, but it does not supersede this narrower gateway/client version-skew request.

## What This Changes

Request gateway/client version-skew tolerance, a compatibility mode, a stable/LTS channel, or documented offline deployment so remote or offline users are not forced to update immediately after a release.

## Best Possible Solution

Define a maintainer-approved gateway compatibility policy: supported app/gateway and client/server skew ranges, explicit protocol negotiation semantics, docs for offline/remote deployment, and regression tests for old/new client handshakes. Any bypass should be narrow enough that incompatible wire contracts still fail loudly.
