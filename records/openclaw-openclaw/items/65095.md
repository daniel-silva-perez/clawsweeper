## Summary

Keep this open. Current main still enforces strict gateway protocol matching and macOS app/gateway version compatibility, while docs and tests document that strict behavior rather than a grace period, compatibility flag, or offline older-client path. The related #65081 covered broader release cadence and is closed, but it does not supersede this narrower gateway/client version-skew request.

## What This Changes

The issue requests a grace period, compatibility mode, stable/LTS channel, or offline deployment guidance so older clients can continue using a gateway after releases.

## Best Possible Solution

Define a maintainer-approved compatibility policy for app/gateway and client/server skew: supported version ranges, protocol negotiation semantics, docs for remote/offline deployment, and regression tests for old/new handshakes. Any bypass should be narrow enough that incompatible wire contracts, auth scopes, and schema changes still fail loudly.
