## Summary

Keep this PR open. Current main still has the reported boundary problem: hook/memory context can be prepended into the prompt, and prompt-image loading still scans that submitted prompt for `[media attached: ...]` references without excluding `<relevant-memories>` blocks. The linked Telegram issue being closed does not make this memory-context image path implemented on main.

## Best Possible Solution

Keep the PR open and either update it or replace it with an equivalent main-branch fix that prevents hook/memory-injected context from contributing prompt-local images while preserving real current-turn image loading. Add focused regression coverage for a `before_prompt_build` `<relevant-memories>` block containing `[media attached: ...]`, plus a real current prompt image outside the block, and cover the CLI prompt-ref path or share the sanitizer boundary there too.
