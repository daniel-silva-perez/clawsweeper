## Summary

Keep this PR open. Current main still scans the final prompt text for `[media attached: ...]` markers after plugin `prependContext` has been prepended, and `memory-lancedb` still injects `<relevant-memories>` blocks via `before_prompt_build`. The newer history-image pruning path only sanitizes older replayed session messages before hooks run; it does not sanitize hook-injected memory context before prompt image detection. The PR is still directionally useful, but the existing review comments about the case-sensitive fast path and vacuous test assertion should be fixed before landing.

## Best Possible Solution

Keep this PR open and land an updated version, or an equivalent main-branch fix, that excludes injected memory/hook context from prompt-local image detection while preserving normal current-turn image loading. Add regression coverage that exercises `before_prompt_build` memory context containing a `[media attached: ...]` marker plus a real current prompt image outside that context, and address the existing review comments before merge.
