## Summary

Keep this PR open. Current main still scans the effective prompt for image references after plugin hook context can be prepended, and the memory-lancedb auto-recall hook still emits `<relevant-memories>` blocks as `prependContext`. The linked Telegram issue being closed does not make this PR obsolete because this PR targets a separate memory-context image injection path that remains plausible on main.

## Best Possible Solution

Keep this PR open and either land an updated version or replace it with an equivalent main-branch fix that prevents injected hook/memory context from contributing prompt-local images while preserving real current-turn image loading. Add focused regression coverage for a `before_prompt_build` memory block containing a `[media attached: ...]` marker plus a real current prompt image outside the block, and address the existing review comments before merge.
