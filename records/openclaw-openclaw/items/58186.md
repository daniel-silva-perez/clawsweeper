## Summary

Keep this PR open. Current main does not implement the intended Gmail hook `systemPrompt` support: the built-in Gmail preset still only defines `sessionKey` and `messageTemplate`, hook mapping/config schemas have no `hooks.mappings[].systemPrompt`, and the hook-to-cron runner path does not carry a system prompt into `extraSystemPrompt`. The PR discussion also captured a real security constraint: any implementation must keep `systemPrompt` literal-only so webhook payload fields cannot enter the system prompt channel.

## Best Possible Solution

Keep the PR open as the implementation candidate, but require a current-main rebase that ports literal-only `hooks.mappings[].systemPrompt` through the hook mapping, config schema/generated metadata, cron payload, and isolated runner `extraSystemPrompt` path; trim unrelated generated drift; then run the targeted gateway hook tests plus the changed gate before landing.
