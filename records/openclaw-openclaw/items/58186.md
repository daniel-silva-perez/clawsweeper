## Summary

Keep this PR open. Current main still lacks Gmail hook `systemPrompt` support, and the PR remains the canonical implementation path, but it needs a current-main rebase/port because Gateway hook dispatch and cron isolated execution have moved since the branch was written.

## What This Changes

The PR adds optional literal `hooks.mappings[].systemPrompt` support, a default Gmail preset system prompt, config/docs metadata, and tests intended to carry the value from hook mapping through isolated cron agent runs.

## Best Possible Solution

Land a rebased version that adds a literal-only hook mapping `systemPrompt` field end to end through config validation, generated schema metadata, hook mapping, the current hook request handler, cron payload types, the current isolated cron executor, docs, and focused tests, while preserving external-content wrapping for dynamic Gmail payload text.
