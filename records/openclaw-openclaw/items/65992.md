## Summary

Keep this PR open. Current main still derives memory-wiki local-file source identity from the slugified title alone and writes directly to the matching source markdown path, so same-slug source ingests can still overwrite each other. The PR remains a focused implementation candidate for the linked memory-wiki collision bug.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or refine a memory-wiki-owned fix that preserves readable source page names, adds a stable collision-resistant suffix for new local-file source pages, reuses legacy source pages where possible, and keeps regression coverage focused on source-page collision behavior.
