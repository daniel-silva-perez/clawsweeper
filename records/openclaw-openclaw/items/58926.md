## Summary

Keep open. Current main still emits bare not-found errors for `config get` and `config unset`, while this PR adds a relevant suggestion helper and wires it into those paths. The earlier review feedback on broad prefixes, case handling, same-depth tail matching, and template-key leakage appears addressed; the remaining concern is a non-blocking scope gap around plugin/channel config metadata.

## What This Changes

The branch adds a config-path suggestion helper with unit tests and prints `Did you mean` hints from `openclaw config get` and `openclaw config unset` when a path is not found.

## Best Possible Solution

The best end state is to land a config-path suggestion feature after maintainers decide whether suggestions should be core-only or use the same schema/hint inventory that already covers bundled channel and plugin config paths.

## Reproduction Assessment

Not applicable as a standalone bug report. The PR behavior has a clear before/after check: current main's `runConfigGet` and `runConfigUnset` only emit the bare not-found error, while the PR adds hints for close static config paths such as `gateway.prot`.

## Solution Assessment

Unclear. Wiring suggestions directly into `get` and `unset` is the narrow right integration point, but `FIELD_HELP` is not the complete current config-path inventory because plugin and channel metadata are merged elsewhere; using that shared inventory would be the more complete solution.

## Review Findings

Overall correctness: patch is correct

Overall confidence: 0.82

Full review comments:

- **[P3] Include plugin config paths in suggestions:** `src/config/suggest-path.ts:73-75`
  - body: The new candidate list is built only from `FIELD_HELP`, but current config metadata is also supplied by plugin/channel schemas and hints. For example, Telegram streaming paths are generated under bundled channel metadata, so mistyped channel/plugin paths will not receive a hint even though they are documented config paths. Consider seeding candidates from the same schema/hint inventory, or explicitly scope this feature to core config paths.
