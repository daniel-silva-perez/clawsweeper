## Summary

Keep open. Current main has already absorbed the Control UI configured-state fallback, but it still does not implement the Slack auth-shaped unhandled-rejection behavior that is the central gateway fix in this PR. The existing main test coverage still expects a non-transient Slack request error to exit, and the proposed core-level Slack regex would also need maintainer review because plugin-owned unhandled-rejection handlers already exist.

## Best Possible Solution

Keep this PR open for maintainer review or author rework. The best path is to rebase on current main, drop the duplicate Control UI/changelog/test changes, address the Greptile comments, and implement the remaining Slack auth floating-rejection fix through the Slack plugin using registerUnhandledRejectionHandler or through a generic plugin-owned classification seam, with a focused regression test in the unhandled-rejection or Slack monitor test area.
