## Summary

Keep open. Current main still has the built-in microsoft-foundry provider constrained to OpenAI-compatible Foundry APIs, and the related guardrail PR #72446 is open and unmerged with a closing reference to this issue.

## Best Possible Solution

Keep this issue open while maintainers review #72446. The best short-term fix is to land that guardrail or an equivalent microsoft-foundry change that filters/warns on Anthropic deployments; a longer-term enhancement can add native Anthropic Foundry routing with `/anthropic/v1/messages` if maintainers want Option A.
