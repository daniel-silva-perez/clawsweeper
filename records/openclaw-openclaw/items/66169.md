## Summary

Keep this PR open. Current main still does not implement the attributed-prefix normalization the PR is trying to add: iMessage echo-cache text keys only normalize CRLF and trim whitespace, so reflected text like `\x93b\x02Cached reply` does not match the clean outbound cache entry. The supplied PR remains a focused implementation candidate, and the discussion shows a maintainer pushed a narrow repair to the branch for review.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate. The best path is maintainer review and merge, or an equivalent narrow patch, preserving the echo-cache and self-chat regression tests and validating with the targeted iMessage monitor tests plus the changed gate.
