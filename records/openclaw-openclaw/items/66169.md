## Summary

Current main still does not implement the iMessage attributed-prefix echo-cache normalization. `normalizeEchoTextKey()` only normalizes CRLF and trims, so a reflected body like `\x93b\x02Cached reply` still misses the clean outbound text key; this PR remains the focused implementation candidate, with maintainer follow-up and targeted tests in the branch context.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate. The best path is maintainer review and merge, or an equivalent narrow patch, preserving the echo-cache and self-chat regression tests and validating with the targeted iMessage monitor tests plus the changed gate.
