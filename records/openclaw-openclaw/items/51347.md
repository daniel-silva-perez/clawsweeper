## Summary

Keep open. Current main has generic outbound local-media root widening that can make some user-supplied local paths work, but it does not close this Feishu bug: the direct Feishu adapter still catches media upload failures and returns a successful text fallback, and Feishu still has no supported/documented `channels.feishu.mediaLocalRoots` config surface.

## Best Possible Solution

Keep this issue open and implement a focused Feishu outbound-media contract. Either stage CLI local paths into managed media before Feishu direct send, or make `LocalMediaAccessError` and other required media upload failures fail the send with a clear non-zero CLI result. If `channels.feishu.mediaLocalRoots` is intended, add it to Feishu schema, generated metadata, docs, merge semantics, and tests; otherwise document the supported roots/policy and avoid implying that unsupported config is honored.
