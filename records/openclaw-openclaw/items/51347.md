## Summary

Keep open. Current main still lacks a supported Feishu `mediaLocalRoots` config surface and still converts Feishu media-send failures into a successful text fallback, although generic media-root widening may mitigate the exact `/tmp` reproduction under default filesystem policy. The related PRs in the provided context are closed and unmerged, so there is no shipped canonical fix.

## Best Possible Solution

Keep this issue open and implement a focused Feishu outbound media contract. Required media upload/access failures should either fail the CLI send with a clear non-zero result or report partial delivery explicitly. If `channels.feishu.mediaLocalRoots` is intended, add it to Feishu schema, generated metadata, docs, merge semantics, and regression tests; otherwise document the supported core-managed roots and remove the implication that the unsupported Feishu key is honored.

## Risks / Open Questions

- The exact `/tmp` reproduction may be policy-dependent on current main because generic media-root widening can append concrete parent directories when host filesystem reads are allowed.
- Blindly honoring a per-channel `mediaLocalRoots` key could widen host file-read access; any fix needs explicit schema, docs, merge behavior, and security semantics.
- Maintainers still need to choose whether Feishu media upload/access failure should fail the whole send or produce an explicit partial-delivery result instead of a success summary.
