## Summary

Keep open. Current main still does not support or read `channels.feishu.mediaLocalRoots`, and Feishu media-send failures are still caught and converted into a successful text/link fallback. Generic media-root widening may reduce the exact `/tmp` reproduction in some policy modes, but it does not resolve the unsupported config key or the misleading success path.

## Best Possible Solution

Keep this issue open and implement a focused Feishu outbound media contract. Either add a supported `channels.feishu.mediaLocalRoots` surface with schema, generated metadata, docs, merge semantics, and tests, or document that Feishu CLI sends only use core-managed roots. In either case, required media upload/access failures should fail the CLI send with a clear non-zero result or report partial delivery explicitly instead of formatting the fallback as a successful media send.

## Risks / Open Questions

- Supporting per-channel media roots would widen host file-read scope, so a fix needs explicit schema, docs, merge order, and security semantics.
- Maintainers still need to choose whether Feishu media access/upload failure should fail the whole send or return an explicit partial-delivery result instead of a success summary.
- The exact `/tmp` reproduction can vary with current filesystem root-expansion policy, so regression coverage should include restricted policy behavior as well as CLI output formatting.
