## Summary

Keep this issue open. Current main partially mitigates local path handling through generic outbound media-root widening, but the Feishu direct media adapter still catches media upload/access failures and returns a successful text fallback, and Feishu still has no supported `channels.feishu.mediaLocalRoots` config surface. The discussion correctly identified two separable bugs; neither related closed PR is merged on current main.

## Best Possible Solution

Keep this issue open and implement a focused Feishu outbound-media contract. Required media upload/access failures should either fail the CLI send with a clear non-zero result, or the CLI should explicitly report partial delivery instead of success. If `channels.feishu.mediaLocalRoots` is intended, add it to Feishu schema, generated metadata, docs, merge semantics, and tests; otherwise document the supported roots and remove the implication that that config is honored.
