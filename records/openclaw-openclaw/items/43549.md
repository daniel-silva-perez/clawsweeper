## Summary

Keep open. Current main still only reports Telegram runtime/config status, while persisted Telegram JSON readers collapse corrupt state to null/empty state or verbose logs; the Keep My Claw link in the discussion is useful backup coverage, but it does not provide the requested status/doctor diagnosis or recovery UX.

## What This Changes

The issue asks Telegram status/doctor to detect malformed, oversized, or unreadable persisted Telegram JSON state and guide a safe backup/reset recovery flow.

## Best Possible Solution

Add a Telegram-owned read-only state diagnostic that feeds channel status and doctor output. It should report known Telegram state artifact paths, readability, size, JSON parse/schema health, likely impact, and backup-before-reset guidance, while deferring destructive reset or startup quarantine to an explicit follow-up decision.
