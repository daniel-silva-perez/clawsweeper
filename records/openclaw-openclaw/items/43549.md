## Summary

Keep open. Current main still surfaces Telegram runtime, credential, webhook/polling, and group-membership warnings, but it does not inspect Telegram persisted JSON state for size, readability, parse/schema health, or give a safe backup/reset recovery path. The Keep My Claw comment is useful backup context, but it was explicitly framed as a safety net rather than a fix for Telegram status/recovery UX.

## What This Changes

The issue requests Telegram status diagnostics and safe recovery guidance for oversized, malformed, or otherwise unusable persisted Telegram session JSON state.

## Best Possible Solution

Add a Telegram-owned persisted-state diagnostic that feeds channel status and, if appropriate, doctor warnings. It should identify account-scoped Telegram state artifacts, report path/readability/size/parse/schema health and likely impact, and recommend a backup-before-reset recovery path; destructive reset or auto-quarantine should remain separate maintainer-reviewed work.
