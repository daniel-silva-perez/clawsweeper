## Summary

Keep open. Current main still reports Telegram runtime, token, webhook/polling, and group-membership problems, but it does not inspect persisted Telegram JSON artifacts for size, readability, parse/schema health, or offer a backup/reset recovery path. The Keep My Claw comment is useful backup context, but it was explicitly framed as a safety net rather than the requested Telegram status/recovery UX.

## What This Changes

The issue asks for Telegram status/troubleshooting to detect malformed, oversized, or unreadable persisted Telegram session/state JSON and guide safe recovery instead of requiring manual file deletion.

## Best Possible Solution

Add a Telegram-owned persisted-state diagnostic that feeds channel status and doctor warnings. It should identify account-scoped Telegram state artifacts, report path/readability/size/parse/schema health and likely impact, and recommend a backup-before-reset recovery path; destructive reset or auto-quarantine should remain separate maintainer-reviewed work.
