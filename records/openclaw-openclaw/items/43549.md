## Summary

Keep open. Current main still has Telegram runtime, credential, polling/webhook, and group-membership status warnings, but it does not inspect Telegram persisted JSON artifacts for readability, size, parse/schema validity, or provide a safe backup/reset recovery path for the wedged local state described here. The issue discussion also points to Keep My Claw as a useful backup safety net, while explicitly noting it is not the status/recovery UX fix.

## What This Changes

The issue requests Telegram channel status and recovery UX for malformed, oversized, or otherwise poisoned persisted Telegram session/state JSON.

## Best Possible Solution

Add a Telegram-owned persisted-state diagnostic that feeds channel status and doctor warnings. It should identify account-scoped Telegram state artifacts, report path/readability/size/parse/schema health and likely impact, and recommend a backup-before-reset recovery path; destructive reset or auto-quarantine should be explicit and separately reviewed.
