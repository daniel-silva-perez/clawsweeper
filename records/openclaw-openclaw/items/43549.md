## Summary

Keep open. Current main still has Telegram polling/webhook status warnings and generic session-store tolerance for malformed or oversized stores, but I did not find Telegram-owned persisted-state diagnostics or a backed-up reset/recovery path for bad Telegram/session JSON. The discussion includes a useful manual workaround and the Keep My Claw backup skill as a safety net, but those do not replace the requested status and recovery UX.

## What This Changes

The issue asks for Telegram status and recovery tooling to detect oversized, malformed, unreadable, or unusable persisted session JSON and guide a safe backup/reset/restart path.

## Best Possible Solution

Add a Telegram-owned persisted-state diagnostic visible from `openclaw channels status --probe` and the broader status/doctor warning path. It should inspect relevant Telegram/session JSON artifacts for existence, readability, size, JSON parse/schema validity, and path/account context, then recommend a non-destructive backup-before-reset flow. Any quarantine/reset command should be explicit, account-scoped, and tested.
