## Summary

Keep this PR open. Current main still stores workspace setup state in the dot-prefixed workspace subdirectory, so the intended compatibility fix is not implemented on main; the PR branch is useful but still has unresolved review findings around filename collision, best-effort legacy reads, and legacy-state promotion.

## What This Changes

The PR branch changes `src/agents/workspace.ts` to write workspace setup state at `<workspace>/workspace-state.json`, fallback-read legacy `<workspace>/.openclaw/workspace-state.json`, and stop creating the dot-prefixed state directory.

## Best Possible Solution

Revise this PR around the same goal: keep setup state out of dot-prefixed workspace entries, use a collision-resistant non-dot state filename or a foreign-file guard, make legacy probing best-effort, migrate legacy state forward after a successful fallback read, and add focused workspace tests. Treat the default workspace nesting reported in #44783 as a separate maintainer decision if broader path simplification is desired.
