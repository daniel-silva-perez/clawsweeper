## Summary

Keep this PR open. Current main and the latest release still store workspace setup state under the dot-prefixed workspace subdirectory, so the proposed compatibility fix is not obsolete, but the branch still needs maintainer review for migration behavior, filename collision safety, and the narrower scope relative to #44783.

## What This Changes

The PR branch changes `src/agents/workspace.ts` to write workspace setup state at `<workspace>/workspace-state.json`, read legacy `<workspace>/.openclaw/workspace-state.json`, and stop creating the state subdirectory.

## Best Possible Solution

Revise the PR around the same product goal: keep state out of dot-prefixed workspace entries, use a collision-resistant non-dot state filename or foreign-file guard, treat legacy probing as best-effort, migrate legacy state forward after a successful fallback read, and add focused workspace tests. Handle the broader default workspace nesting from #44783 separately if maintainers want that change.
