## Summary

Keep this PR open. The underlying request is still valid because current main and the latest release still store setup state under a dot-prefixed workspace subdirectory, but this branch has blocking correctness and data-integrity issues that should be addressed before merge.

## What This Changes

The PR changes `src/agents/workspace.ts` to write workspace setup state at `<workspace>/workspace-state.json`, fallback-read legacy `<workspace>/.openclaw/workspace-state.json`, and stop creating the dot-prefixed state directory.

## Best Possible Solution

Revise this PR around the same goal: keep setup state out of dot-prefixed workspace entries, use a collision-resistant non-dot filename or detect foreign files before writing, make legacy path probes best-effort, promote successfully loaded legacy state to the new location, and add focused tests for first-run, legacy migration, collision handling, and dot-path I/O failures.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Make legacy state probes best-effort:** `src/agents/workspace.ts:269`
  - body: `readWorkspaceSetupStateForDir` probes `.openclaw/workspace-state.json` whenever the new root file has no markers, but `readWorkspaceSetupState()` rethrows non-ENOENT errors. On dot-restricted filesystems, first-run or status checks can fail before the new root state is written; catch and ignore legacy-probe I/O failures.
  - confidence: 0.91
- **[P2] Avoid overwriting foreign workspace-state files:** `src/agents/workspace.ts:207`
  - body: Writing OpenClaw state to `<workspace>/workspace-state.json` can clobber an existing project file with the same generic name. If that file parses without OpenClaw markers, setup treats it as empty state and later overwrites it, so use a safer non-dot name or detect foreign files before writing.
  - confidence: 0.88
- **[P2] Promote fallback state to the new location:** `src/agents/workspace.ts:401`
  - body: When `ensureAgentWorkspace` loads state from the legacy path, `stateDirty` remains false, so the later write never creates the new root state file. Existing workspaces keep depending on `.openclaw/workspace-state.json`; persist fallback-loaded state to complete the migration.
  - confidence: 0.86
- **[P2] Update workspace state path tests:** `src/agents/workspace.ts:207`
  - body: The branch changes writes to `<workspace>/workspace-state.json`, but the workspace test helper still reads `<workspace>/.openclaw/workspace-state.json`. Those tests will fail and will not cover the new path or legacy fallback behavior until updated.
  - confidence: 0.83
