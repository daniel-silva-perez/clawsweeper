## Summary

Keep open. Current main still does not enforce the hooks.enabled/hooks.token dependency at config validation time, so the PR is not obsolete; however the proposed branch introduces a blocking UX regression by recommending an unsupported `openclaw config.patch` command. The branch should be corrected and covered by focused tests before merge.

## What This Changes

The PR adds hooks.enabled/hooks.token cross-field validation in config validation and replaces the gateway startup error with a more actionable token-setting hint.

## Best Possible Solution

Land a corrected version that rejects `hooks.enabled: true` with a missing or blank `hooks.token` during config write validation, keeps the runtime guard as a safety net, uses only supported CLI/RPC remediation text, and adds focused validation/runtime tests.
