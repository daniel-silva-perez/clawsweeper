## Summary

Current main has adjacent memory-core introspection, but not this requested feature. Scheduled deep reports still write aggregate ranked/promoted counts only, verbose candidate detail still goes to logger output rather than a workspace artifact, and the CLI has `memory promote-explain` for one selected candidate but no aggregate `memory dreaming doctor` gate diagnostic.

## Best Possible Solution

Keep this open and implement a shared memory-core deep-candidate evaluation surface that returns scores, components, threshold pass/fail state, and rejection reasons for all bounded considered candidates. Use it to extend the scheduled deep report, optionally write verbose JSONL when `verboseLogging` is enabled, and add an aggregate CLI diagnostic for gate reachability with docs and focused tests.

## Clownfish Work Prompt

Implement the #70072 memory-core dreaming introspection feature on current main. Add a shared deep-candidate evaluation/explanation helper, preferably in `extensions/memory-core/src/short-term-promotion.ts` or a small sibling module, that evaluates live short-term recall entries without dropping rejected entries and returns rank/order, score, components, signal counts, unique query/context diversity, threshold values, pass/fail state, promoted/apply status where known, and concise rejection reasons. Use this helper in the scheduled deep dreaming run so `memory/dreaming/deep/YYYY-MM-DD.md` keeps the current aggregate lines and also includes a bounded ranked-candidates table plus gate thresholds for the run. When `verboseLogging: true`, write a structured JSONL artifact under `memory/.dreams/deep/` with one evaluated candidate per line and component/gate details; keep daemon logger output compatible. Add an aggregate CLI diagnostic matching the intent of `openclaw memory dreaming doctor` that reports store span, gate reachability counts, threshold ceilings, last deep-run summary when available, and JSON output. Reuse or factor existing `memory promote-explain` behavior so CLI/report/JSONL logic does not diverge. Do not change promotion thresholds or wildcard/random promotion policy in this PR. Keep artifacts bounded enough to avoid huge reports, and preserve existing redaction/workspace-memory boundaries. Update `docs/cli/memory.md`, `docs/concepts/dreaming.md`, and add a user-facing changelog entry crediting `@iris-nvdh` (and `@nicolevanderhoeven` if appropriate).
