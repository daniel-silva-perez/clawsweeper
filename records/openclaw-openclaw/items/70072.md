## Summary

Current main still does not implement the requested deep-dreaming introspection. Deep reports still contain only aggregate ranked/promoted counts, verbose scoring remains logger-only, rejected candidates are filtered out before callers can explain them, and the CLI has only single-candidate `memory promote-explain`, not an aggregate `memory dreaming doctor` gate diagnostic. The issue is a distinct additive follow-up to #68882, not a duplicate of that closed calibration bug.

## Best Possible Solution

Keep this open and implement a shared memory-core deep-candidate evaluation surface that returns bounded considered candidates with score components, threshold pass/fail state, promotion status, and rejection reasons. Use it to extend scheduled deep reports, optionally write verbose JSONL when `verboseLogging` is enabled, and add an aggregate CLI diagnostic for gate reachability, with docs and focused tests.
