## Summary

Keep this PR open. Current main still has the skill-creator validation gap the PR targets: blank or whitespace-only `name` and `description` values are stripped and then skipped by truthy guards in `validate_skill()`, while the current regression tests do not cover those cases. The PR is narrow, the author is still active, and the related PRs that mentioned this work are closed unmerged rather than shipped canonical fixes.

## Best Possible Solution

Review and land this PR, or apply an equivalent maintainer patch scoped to `skills/skill-creator/scripts/quick_validate.py` and `skills/skill-creator/scripts/test_quick_validate.py`. The fix should reject empty or whitespace-only `name` and `description` immediately after trimming, with regression coverage for all four empty/whitespace cases.

## Evidence
