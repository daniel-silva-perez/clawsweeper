## Summary

Keep this PR open. Current main still has the exact skill-creator validation gap: `validate_skill()` strips `name` and `description`, then skips validation for blank values behind truthy guards, and the current regression tests do not cover empty or whitespace-only metadata. The PR remains a narrow, relevant bug fix with no obvious security or supply-chain concern in the two-file diff.

## Best Possible Solution

Review and land this PR, or apply the same narrow fix on main: reject empty or whitespace-only `name` and `description` immediately after trimming in `skills/skill-creator/scripts/quick_validate.py`, with regression tests for empty and whitespace-only values in `skills/skill-creator/scripts/test_quick_validate.py`.
