## Summary

Keep this PR open. Current main still trims `name` and `description` but only validates them inside truthy guards, so empty or whitespace-only metadata can still pass the skill-creator validator. The proposed regression tests are still absent, and the related hardening PRs in the provided context were closed unmerged rather than shipped.

## Best Possible Solution

Keep this PR open and review or land it, or apply an equivalent maintainer patch. The fix should stay scoped to `skills/skill-creator/scripts/quick_validate.py` and its tests: after trimming, reject empty or whitespace-only `name` and `description`, and add regression coverage for both fields.
