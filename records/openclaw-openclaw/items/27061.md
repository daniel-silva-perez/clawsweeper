## Summary

Keep open. Current main still strips `name` and `description` and then skips validation under truthy guards, so quoted empty or whitespace-only SKILL.md metadata can still pass; the PR is a narrow validator-and-test fix that matches the documented skill metadata contract.

## What This Changes

The PR updates `skills/skill-creator/scripts/quick_validate.py` to reject trimmed-empty required skill metadata and adds four regression tests in `skills/skill-creator/scripts/test_quick_validate.py`.

## Best Possible Solution

Land this PR or apply an equivalent maintainer patch scoped to the quick validator and its focused tests, keeping valid skill metadata behavior unchanged while rejecting blank required metadata.

Yes. A SKILL.md with `name: ""`, `name: '   '`, `description: ""`, or `description: '   '` is enough to trace the current-main failure path through the strip-plus-truthy guards; the PR adds direct fixtures for those cases.

## Solution Assessment

Yes. The PR is the narrowest maintainable fix for the reported bug: it rejects trimmed-empty required fields immediately after type checking and adds focused regression coverage without changing unrelated packaging or loader behavior.
