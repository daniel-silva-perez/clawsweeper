## Summary

Keep this PR open. Current main still has the validator gap it targets: blank or whitespace-only `name` and `description` values are stripped, skipped by truthy guards, and can reach the success return. The PR is narrow, adds matching regression coverage, and the related duplicate attempts/reports I checked are closed unmerged or stale rather than a shipped canonical fix.

## Best Possible Solution

Review and land this PR, or apply an equivalent maintainer patch scoped to `skills/skill-creator/scripts/quick_validate.py` and `skills/skill-creator/scripts/test_quick_validate.py`. The fix should reject empty or whitespace-only `name` and `description` immediately after trimming, while keeping regression coverage for all four empty/whitespace cases.
