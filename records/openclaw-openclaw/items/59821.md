## Summary

Keep open. Current main still treats an explicit empty `skills.allowBundled` array as unrestricted, so this PR’s intended bug fix is not implemented yet.

## Best Possible Solution

Keep this PR open and review or land an equivalent current-main patch that preserves explicit empty arrays through `normalizeAllowlist()`, distinguishes `undefined` from `[]` in `isBundledSkillAllowed()`, and adds focused regression coverage for bundled skills being blocked while workspace skills remain available.
