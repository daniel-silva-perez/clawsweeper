## Summary

Keep this PR open. Current main still does not implement the PR's intended startup/provider-registry reload reduction: gateway prewarm does not pass workspace or entries-only discovery through models.json, models-config has no `providerDiscoveryEntriesOnly` fingerprint/planning option, and capability-provider fallback registry loads remain unscoped by `onlyPluginIds`. The PR is also an active implementation candidate for open linked reports #73835 and #73729, and hclsys confirmed in the discussion that it is complementary to open sister PR #73847 rather than a duplicate.

## Best Possible Solution

Keep this PR open for normal maintainer review. The best path is to review and land or request focused changes on the scoped startup/provider-registry reductions, coordinate changelog and cache semantics with #73847, and then let GitHub close the linked issues that this PR genuinely fixes after merge.
