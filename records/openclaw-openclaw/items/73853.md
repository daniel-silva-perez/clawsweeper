## Summary

Keep this PR open. Current main does not already implement the proposed startup/provider-registry reload reduction: the gateway prewarm path still omits workspace and entries-only discovery threading, models.json planning/fingerprinting has no `providerDiscoveryEntriesOnly` option, and capability provider fallback loads still call the runtime registry without `onlyPluginIds`. The PR is also paired with open linked issue #73835 from the same author and coordinates with open sister PR #73847, so this is an active implementation candidate rather than cleanup fodder.

## Best Possible Solution

Keep the PR open for normal maintainer review. The best path is to review the proposed scoped startup/provider changes, coordinate with #73847 so the web-provider and capability/startup fixes do not duplicate changelog or cache semantics, run the changed plugin/gateway/agents validation, and then merge or request focused changes. After merge, GitHub can close the linked issues that the PR genuinely fixes.
