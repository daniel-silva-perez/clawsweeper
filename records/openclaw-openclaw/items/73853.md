## Summary

Keep open. Current main does not implement the PR’s intended startup/provider-registry reductions: gateway startup still prewarms models without passing workspace-scoped entry-only provider discovery through models.json, and capability-provider fallback loads still call the runtime registry without `onlyPluginIds`. The PR is also paired with open same-lane reports, including #73835 and #73729, while #73847 covers a related but separate web-provider cache path.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to review it alongside #73847, confirm the focused performance behavior with the listed startup/provider tests and changed gate, then either land the scoped fix for #73835/#73729 or request targeted changes if the cache/threading shape needs adjustment.
