## Summary

Keep this issue open. Current main partially addresses Telegram sender context by exposing sender name, id, and username to the agent context, but it does not expose the requested Telegram `msg.from.is_bot` / bot-vs-user classification field or a richer `msg.from` metadata object. The request is a concrete bundled Telegram plugin enhancement, not a duplicate or a ClawHub-only item.

## Best Possible Solution

Add a narrow Telegram-owned inbound context field such as `SenderIsBot` or a bounded structured Telegram sender metadata block, populate it from `msg.from.is_bot`, document it in the Telegram channel docs, and add focused Telegram inbound-context tests proving bot and non-bot sender messages expose the expected metadata to agents.
