## Summary

Keep this open. Current main partially exposes Telegram sender identity to the agent context, but the requested `msg.from.is_bot` / bot-vs-user classification is still not exported as a context field or documented capability.

## Best Possible Solution

Add a narrow Telegram-owned context field such as `SenderIsBot` or a bounded structured sender metadata block populated from `msg.from.is_bot`, document it in the Telegram channel docs, and add focused tests for bot and non-bot inbound messages reaching the agent context.
