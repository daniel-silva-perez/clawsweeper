## Summary

Keep open. Current main still has the reported Copilot token-cache bug: both active token helpers use one shared `credentials/github-copilot.token.json` cache and return any usable cached Copilot API token before checking which GitHub token/profile is currently selected. The issue is also paired with open same-author PR #43659 using closing syntax, so the right maintainer path is to review, land, or close that PR rather than close this issue early.

## Best Possible Solution

Keep this issue open and resolve it through #43659 or an equivalent maintainer-reviewed patch. The fix should bind cached Copilot API tokens to the selected GitHub token/profile in both active helper paths, treat legacy unbound cache entries deliberately as misses or safe refreshes, and add focused regression tests for cross-profile cache bypass.
