Keep open. Current main still returns several hook 4xx responses without a hook journal warning, including the exact invalid-JSON path and mapped-hook validation rejects described in the report. The direct fix PR #69267 was closed unmerged, so it remains useful prior art but not an implemented fix.

Implement a small hook rejection helper in `src/gateway/server/hooks-request-handler.ts` that logs one sanitized warn line before every non-2xx hook response, with status, stable reason code, normalized path, content type, safe body metadata, and active trace/request context. Add focused tests for invalid JSON, mapping validation, direct wake/agent validation, auth/path/method rejects, and policy rejects. Keep response compatibility unless intentionally improving the ambiguous mapping error with tests and changelog credit.
