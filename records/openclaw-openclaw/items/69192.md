## Summary

Keep open. Current main still returns multiple hook gateway 4xx responses without a hook warning/journal line, the mapped-hook error remains ambiguous, and the direct prior fix PR #69267 is closed unmerged, so the report remains valid and narrow enough for a focused repair PR.

## What This Changes

Request that every non-2xx hook gateway rejection emit a sanitized warning/journal entry with a stable internal reason code and useful request metadata.

## Best Possible Solution

Keep this open and implement a small centralized hook rejection logging helper in `src/gateway/server/hooks-request-handler.ts`. Route every non-2xx hook response through it, emit one sanitized warn line with a stable internal reason code and safe request metadata, preserve response compatibility unless deliberately improving ambiguous mapped-hook errors, and add focused tests for direct and mapped 4xx branches.
