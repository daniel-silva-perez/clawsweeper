## Summary

Keep open: current main still has the reported sanitizer gap. Final tags are stripped as plain text, duplicate collapse still only handles exact adjacent blank-line blocks, outbound replies still pass through this sanitizer, and the related same-line duplicate PRs are closed unmerged while adjacent open work does not specifically supersede this issue.

## What This Changes

The issue requests stronger `sanitizeUserFacingText()` duplicate cleanup for same-line, single-newline, partial, and `</final>` tag-boundary repeats that survive the current paragraph-only duplicate collapse.

## Best Possible Solution

Add a conservative sanitizer pass for the concrete glued `</final>` repeat and explicitly supported adjacent same-line or single-newline exact duplicate forms, with regression tests proving ordinary repeated wording, normal single-newline formatting, trailing whitespace, and code-block indentation are preserved.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: after `</final>` replacement the reported example becomes one glued block, and the current duplicate collapse returns single-block text unchanged because it only splits on blank lines.

## Solution Assessment

Yes. A narrow sanitizer repair is the best maintainable direction; broad partial/near-duplicate or whole-response coherence heuristics should be avoided unless tightly constrained to observed leak patterns and covered by tests.
