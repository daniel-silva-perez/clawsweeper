## Summary

Current main still has the reported sanitizer gap. `sanitizeUserFacingText()` strips `<final>` tags textually and only collapses exact adjacent duplicate blocks separated by blank lines, so same-line repeats, single-newline repeats, partial-prefix repeats, and boundary-aware `</final>` cleanup remain unimplemented.

## Best Possible Solution

Keep this issue open and implement a narrow sanitizer fix in `src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts`. The best path is to handle the concrete `</final>` same-line repeat and adjacent single-newline duplicate shapes conservatively, with regression tests proving normal repeated wording, ordinary single-newline formatting, trailing whitespace behavior, and code-block indentation are preserved. Coordinate with #41851 if it lands first.
