## Summary

Keep open. Current main still has the reported sanitizer gap: final tags are stripped as plain text, duplicate collapse only handles adjacent blank-line blocks, and outbound replies still flow through that sanitizer. The closed same-line duplicate PRs were not merged, and the currently open adjacent PR targets code-block indentation rather than tag-boundary or same-line duplicate removal.

## What This Changes

The issue requests stronger user-facing output sanitization for same-line duplicates, partial duplicates, and duplicates exposed after `<final>` tag stripping.

## Best Possible Solution

Add a conservative sanitizer pass for the concrete glued final-tag repeat and any deliberately supported adjacent single-newline duplicate form, with regression tests that preserve ordinary repeated phrasing, normal single-newline formatting, trailing whitespace, and code-block indentation.

Implement a conservative pass in `src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts` that handles the concrete tag-boundary/same-line duplicate without broad fuzzy removal. Add focused tests in `src/agents/pi-embedded-helpers.sanitizeuserfacingtext.test.ts` for the real-world glued `</final>` shape, same-line duplicate text, any supported single-newline duplicate form, and negative cases preserving ordinary repeated wording, normal single-newline formatting, trailing whitespace, and code-block indentation.
