## Summary

Keep open: current main still has the reported sanitizer gap. Final tags are stripped as plain text, duplicate collapse still only handles exact adjacent blank-line blocks, outbound replies still pass through this sanitizer, and the prior same-line duplicate PRs are closed unmerged rather than shipped.

## What This Changes

The issue asks for `sanitizeUserFacingText()` to catch same-line, partial, and tag-boundary duplicate output left behind after final-tag stripping.

## Best Possible Solution

Add a conservative sanitizer pass for the concrete glued `</final>` repeat and any intentionally supported adjacent same-line or single-newline duplicate form, with regression tests that preserve ordinary repeated wording, normal single-newline formatting, trailing whitespace, and code-block indentation.
