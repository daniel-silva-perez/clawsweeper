## What This Changes

The issue requests stronger output sanitization for same-line, single-newline, partial duplicate, and final-tag-boundary duplicate assistant output.

## Best Possible Solution

Add a conservative sanitizer pass for the concrete glued `</final>` repeat and explicitly supported adjacent same-line or single-newline duplicate forms, with regression tests proving ordinary repeated prose, normal single-newline formatting, trailing whitespace, and code-block indentation are preserved.

## Reproduction Assessment

Yes. Source inspection shows the reported example becomes a single glued block after `</final>` replacement, and the current duplicate collapse returns single-block text unchanged because it only splits on blank lines.

## Solution Assessment

Yes. A narrow sanitizer repair is the best maintainable direction; broad near-duplicate or whole-response coherence heuristics should be avoided unless they are tightly scoped to observed leak patterns and covered by tests.
