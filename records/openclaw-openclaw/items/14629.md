## Summary

Current main still has the reported sanitizer gap. `sanitizeUserFacingText()` strips `<final>` tags textually and then only collapses exact adjacent duplicate blocks separated by blank lines, so same-line repeats, single-newline repeats, partial-prefix repeats, and `</final>` boundary-aware cleanup remain unimplemented. The maintainer comment already identified the same gap, #16813 and #19113 attempted part of it but were closed unmerged, and #41851 is adjacent indentation work rather than a fix for this issue.

## Best Possible Solution

Keep this issue open and implement a narrow sanitizer fix in `src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts`. The best path is to handle the concrete `</final>` same-line repeat and single-newline adjacent-repeat shapes conservatively, with regression tests in `src/agents/pi-embedded-helpers.sanitizeuserfacingtext.test.ts` proving normal repeated wording, ordinary single-newline formatting, and code-block indentation are preserved.
