## Summary

Current main still has the reported sanitizer gap: final tags are stripped as plain text, duplicate collapse is limited to exact adjacent blank-line-separated blocks, and outbound replies still route through this sanitizer. The prior same-line duplicate PRs are closed unmerged, and the open adjacent indentation PR does not address this behavior.

## What This Changes

Improve `sanitizeUserFacingText()` so duplicate assistant text created by final-tag stripping, same-line repeats, and carefully bounded partial repeats is removed before user-facing delivery.

## Best Possible Solution

Add a conservative sanitizer pass for the concrete glued `</final>` repeat and any deliberately supported adjacent single-newline duplicate form, with tests proving ordinary repeated wording, normal single-newline formatting, trailing whitespace, and code-block indentation are preserved.
