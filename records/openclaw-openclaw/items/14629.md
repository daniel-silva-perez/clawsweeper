## Summary

Keep open: current main still has the reported sanitizer gap. Final tags are stripped as plain text, duplicate collapse only handles exact adjacent paragraph blocks separated by blank lines, outbound replies still use this sanitizer, and the latest release has the same behavior. The related same-line duplicate PRs are closed unmerged; the open adjacent PRs do not currently close this issue.

## What This Changes

Improve `sanitizeUserFacingText()` so same-line, single-newline, partial, or tag-boundary duplicate assistant output is removed conservatively without damaging normal formatting.

## Best Possible Solution

Add a conservative sanitizer pass for the concrete glued `</final>` repeat and any deliberately supported adjacent same-line or single-newline duplicate form, with regression tests proving ordinary repeated wording, normal single-newline formatting, trailing whitespace, and code-block indentation are preserved.
