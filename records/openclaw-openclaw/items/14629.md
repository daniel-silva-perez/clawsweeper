## Summary

Current main still has the reported sanitizer gap: final tags are stripped textually, duplicate collapse only handles exact adjacent blank-line-separated blocks, and the outbound reply path still uses this sanitizer. Prior closed PRs #16813 and #19113 attempted a same-line duplicate fix but were not merged, and open PR #41851 is adjacent indentation work rather than a fix for this issue.

## Best Possible Solution

Keep this issue open and implement a narrow sanitizer fix in `src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts`, with focused regression coverage in `src/agents/pi-embedded-helpers.sanitizeuserfacingtext.test.ts`. The best fix should handle the concrete `</final>` glued same-line repeat and any deliberately supported adjacent single-newline duplicate shape while preserving normal repeated wording, ordinary single-newline formatting, trailing whitespace behavior, and code-block indentation.
