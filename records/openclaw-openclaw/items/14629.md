## Best Possible Solution

Add a conservative sanitizer pass for the concrete glued `</final>` repeat and any explicitly supported adjacent same-line or single-newline duplicate form, with regression tests proving ordinary repeated phrasing, normal single-newline formatting, trailing whitespace, and code-block indentation are preserved.

## Repair Work Prompt

Fix the remaining sanitizer gap from https://github.com/openclaw/openclaw/issues/14629. Current `sanitizeUserFacingText()` strips final tags as text and only collapses exact duplicate paragraph blocks split by blank lines, so glued outputs like `Greeting.</final>Greeting.Follow-up` can still reach users. Implement a conservative cleanup in the sanitizer path, not a broad model-output rewrite: handle the concrete glued final-tag repeat and any deliberately supported adjacent same-line or single-newline duplicate form, while preserving intentional repeated wording, ordinary single-newline formatting, trailing whitespace where current tests preserve it, and code-block indentation. Before patching, check overlap with #41851, #66225, and draft #68986; do not fold in their broader scope. Add focused regression tests in `src/agents/pi-embedded-helpers.sanitizeuserfacingtext.test.ts`. Update changelog only if maintainers expect a user-facing sanitizer fix entry; if adding one, keep it single-line and credit a real reporter/source author per repo rules. Validate with the targeted sanitizer test, targeted oxfmt check, and changed gate.
