## Summary

Keep open because current main still has the reported sanitizer gap: final tags are removed with plain regex replacement, duplicate collapsing still only handles exact adjacent blank-line-separated blocks, and outbound replies still pass through this sanitizer. The related same-line duplicate PRs are closed unmerged, while the open adjacent PRs target different or broader visible-output work.

## What This Changes

Improve `sanitizeUserFacingText()` so it catches glued same-line and conservative partial duplicates left after final-tag stripping without damaging normal reply formatting.

## Best Possible Solution

Add a conservative sanitizer pass for the concrete glued `</final>` repeat and explicitly supported adjacent same-line or single-newline duplicate forms, with regression tests proving ordinary repeated prose, normal single-newline formatting, trailing whitespace, and code-block indentation are preserved.

## Repair Work Prompt

Fix the sanitizer gap from https://github.com/openclaw/openclaw/issues/14629. Current `sanitizeUserFacingText()` strips final tags textually and `collapseConsecutiveDuplicateBlocks()` only collapses exact duplicate paragraph blocks split by blank lines, so glued output like `A</final>A B` can still deliver duplicated visible text. Implement a conservative sanitizer repair in `src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts` and tests in `src/agents/pi-embedded-helpers.sanitizeuserfacingtext.test.ts`. Scope the fix to concrete adjacent/glued/tag-boundary duplicate patterns; do not add a broad fuzzy partial-duplicate heuristic that can remove intentional repeated wording. Preserve normal single-newline formatting, trailing whitespace behavior, code-block indentation, and ordinary repeated prose. Before editing, inspect overlap with #41851, #66225, and draft #68986. No changelog is required unless the maintainer wants a user-facing fix note. Validate with `pnpm test src/agents/pi-embedded-helpers.sanitizeuserfacingtext.test.ts`, targeted `oxfmt --check`, and `pnpm check:changed` in Testbox if promoted by a maintainer.
