## Summary

Keep this issue open. Current main still watches only the active root config path, while include resolution happens later during snapshot reads. The existing helper for collecting recursive include paths is not wired into the gateway config reloader, and the related implementation PR was closed unmerged.

## Best Possible Solution

Keep this issue open and implement the narrow gateway-side fix: after startup and after each successful snapshot read, collect recursive `$include` paths from the source-authored config, add newly referenced files to the chokidar watcher, unwatch removed paths, and add focused tests for include edit, include removal, nested include, and scanner failure behavior. The closed #59632 is useful prior art but should be re-reviewed against current reload and recovery code before landing.
