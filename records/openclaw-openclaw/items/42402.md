## Summary

Keep this PR open. Current main still lacks the runtime type guard in the `env.vars` collection path, so a truthy non-string value from malformed JSON can still reach `containsEnvVarReference()` and throw before schema validation. The PR remains a focused fix candidate, though it needs a rebase to the current config file layout and a maintainer call on silent skip versus diagnostics.

## Best Possible Solution

Keep this PR open as the focused implementation candidate. Rebase it onto current main, apply `typeof value !== "string" || !value.trim()` in `collectConfigEnvVarsByTarget` for `env.vars`, add mixed-type regression coverage in `src/config/config.env-vars.test.ts`, decide the diagnostic behavior for malformed values, then run the targeted config tests and the appropriate changed gate.
