## Summary

Keep this PR open. Current main still lacks the runtime type guard in the `env.vars` collection path, so truthy non-string JSON values can still reach `containsEnvVarReference()` before schema validation and throw. The submitted patch is still a focused fix candidate, but it needs rebasing to the current config file layout and a maintainer decision on silent skip versus diagnostics.

## Best Possible Solution

Keep this PR open as the focused implementation candidate. Rebase it onto current main, add `typeof value !== "string" || !value.trim()` handling in the `envConfig.vars` loop in `src/config/config-env-vars.ts`, add mixed-type regression coverage in `src/config/config.env-vars.test.ts`, decide the diagnostic behavior for malformed values, then run the targeted config tests and the appropriate changed gate.
