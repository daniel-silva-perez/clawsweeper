## Summary

Keep this PR open. Current main still lacks the runtime type guard for `env.vars`, and the PR's focused guard/test/changelog matches the current file layout. The crash path remains visible in source: parsed JSON can provide a truthy non-string `env.vars` value, the collector stores it, `applyConfigEnvVars()` passes it to `containsEnvVarReference()`, and that function still calls `.includes()` on its argument.

## Best Possible Solution

Keep this PR open as the focused implementation candidate. Preserve the string-only schema, add the runtime guard in `collectConfigEnvVarsByTarget` before normalization/reference checks, keep the mixed-type regression test in `src/config/config.env-vars.test.ts`, decide warning versus silent skip behavior, then run the targeted config tests and changed gate.
