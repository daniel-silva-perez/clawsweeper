## Summary

Keep this PR open. Current main still lacks the proposed runtime type guard in the `env.vars` collection path, so a truthy non-string value parsed from JSON can still flow into `containsEnvVarReference()` and throw on `.includes()`. The linked bug is closed and the broader related PR #42377 is closed unmerged, so this PR remains the focused implementation candidate, though it needs a rebase to the current config file layout.

## Best Possible Solution

Keep this PR open as the focused bug-fix candidate. Rebase it onto current main, apply `typeof value !== "string" || !value.trim()` in `collectConfigEnvVarsByTarget` for `env.vars`, add mixed-type regression coverage in `src/config/config.env-vars.test.ts`, decide the diagnostic behavior for malformed values, then run the targeted config tests and the appropriate changed gate.
