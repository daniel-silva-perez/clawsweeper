## Summary

Keep this PR open. Current main still lacks the runtime type guard in the `env.vars` collection path, so a truthy non-string value parsed from JSON can still flow into `containsEnvVarReference()` before config schema validation. The proposed fix remains a plausible focused bug fix, but it needs to be rebased onto the current config file layout and needs maintainer judgment on silent skip versus diagnostics.

## Best Possible Solution

Keep this PR open as the focused implementation candidate. Rebase it onto current main, apply a runtime guard such as `typeof value !== "string" || !value.trim()` in `collectConfigEnvVarsByTarget` for `env.vars`, add mixed-type regression coverage in `src/config/config.env-vars.test.ts`, decide diagnostic behavior for malformed values, then run the targeted config tests and the appropriate changed gate.
