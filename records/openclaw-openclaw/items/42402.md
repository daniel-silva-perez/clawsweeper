## Summary

Keep this PR open. Current main at 16906780fd2c481308ec4b80f96eb3b93d0add46 still lacks the proposed runtime type guard in the `env.vars` collection path, so truthy non-string values can still reach `containsEnvVarReference()` and throw on `.includes()`. The linked bug is closed and the earlier broader PR #42377 is closed unmerged, so this PR remains the focused implementation candidate, although it needs a rebase to the renamed config files.

## Best Possible Solution

Keep this PR open as a focused bug-fix candidate. Rebase the `typeof value !== "string" || !value.trim()` guard into `collectConfigEnvVarsByTarget` in `src/config/config-env-vars.ts`, add mixed-type `env.vars` regression coverage in `src/config/config.env-vars.test.ts`, preserve the string-valued public config contract, decide on diagnostics for skipped malformed values, then run targeted config tests and the appropriate changed gate.
