## Summary

Keep open. Current main still enforces Discord slash-command routing and authorization after an interaction is submitted, but it does not implement per-guild-channel command picker visibility for multi-bot Discord setups. Commands are still built from a full provider command catalog per account, and the config/schema surface has no Discord application-command permission reconciliation or per-channel visibility setting.

## Best Possible Solution

Keep this open as a Discord plugin feature request. Maintainers should decide whether OpenClaw should own Discord application-command permission reconciliation, including the OAuth/Bearer-token flow and limits. If not, update Discord and slash-command docs with explicit supported workarounds such as per-account `commands.native=false`, text commands, or server isolation.
