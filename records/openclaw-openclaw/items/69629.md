## Summary

Keep open. Current main protects Discord slash-command execution and routing, but it still does not implement the requested per-channel Discord command-palette visibility for multi-bot setups. The Discord docs explicitly acknowledge that commands may remain visible in the Discord UI for unauthorized users, and the current deploy/config paths do not expose or reconcile per-channel application-command visibility.

## Best Possible Solution

Keep this open as a bundled Discord plugin feature request. The best path is to verify Discord's current application-command permission model, then add a documented Discord-owned config/reconciliation path that can hide native commands per allowed guild channel for each account where the platform supports it. If Discord cannot reliably support per-channel command hiding from a bot token, update the public Discord/slash-command docs with explicit workaround guidance for per-account channels.discord.commands.native=false, text commands, or server isolation.
