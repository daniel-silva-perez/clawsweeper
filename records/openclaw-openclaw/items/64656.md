## Summary

Keep this PR open. Current main has overlapping reset cleanup and bounded overflow recovery, but it still lacks the PR's dedicated failover state model and the /new-vs-/reset LCM conversation identity split; the open P1/P2 review comments also need resolution before this can be considered obsolete or merge-ready.

## What This Changes

This PR changes session fallback persistence to a `SessionEntry.failoverState`, adds `memoryConversationKey` for LCM/context-engine identity, and extends embedded-runner overflow recovery with structured diagnostics and a circuit breaker.

## Best Possible Solution

Rebase and narrow the branch into a maintainer-reviewed implementation: decide whether OpenClaw should move from the current `modelOverrideSource: "auto"` policy to a separate `failoverState`, implement the LCM identity split consistently across all context-engine lifecycle calls while preserving legacy `/reset` continuity, and keep only overflow diagnostics or breaker behavior that improves on main without exposing sensitive session details.
