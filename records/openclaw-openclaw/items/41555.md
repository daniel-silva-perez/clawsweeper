## Summary

Keep open. Current main has nearby reset hardening for active runs and auto fallback pins, but the reported recovery expectation is still not implemented: bare `/new` and `sessions.reset` intentionally preserve explicit model selection plus thinking/reasoning session overrides, so a user-selected Opus/Kimi thinking-loop state can still require changing model or thinking mode.

## Best Possible Solution

Keep this open for a deliberate stuck-run recovery design in core session/run controls. The likely product path is an explicit recovery/reset option that aborts active work and can clear selected model, thinking, and reasoning overrides back to defaults, while preserving current `/new` semantics unless maintainers intentionally change them. If current behavior is intended, document the recovery path clearly, such as `/think off`, `/new <model>`, or switching away from the affected model before resetting.
