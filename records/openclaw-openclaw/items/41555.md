## Summary

Keep open. Current main still treats `/new`, `/reset`, and gateway `sessions.reset` as transcript/session resets that preserve user-selected model and thinking/reasoning state. Adjacent fixes reduce some thinking-related failures, but they do not implement the reported recovery behavior where bare `/new` clears a stuck Opus/Kimi thinking state without a model or thinking-mode change.

## Best Possible Solution

Keep this open for an explicit stuck-run recovery design. A conservative path would add a separate recovery/reset option that aborts active work and can clear selected model plus thinking/reasoning overrides back to defaults, while preserving current `/new` semantics unless maintainers intentionally change them. If current behavior is intended, document the recovery path clearly, such as `/think off`, switching models, or a reset command that includes model/thinking cleanup.
