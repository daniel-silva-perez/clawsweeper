## Summary

Keep this issue open. Current main still treats `/new`, `/reset`, and gateway `sessions.reset` as fresh transcript/session resets that preserve user-selected model and thinking/reasoning state. Adjacent auto-fallback reset fixes are present, but they do not implement the reported stuck-model recovery behavior where a bare `/new` clears Opus/thinking state back to a safe default.

## Best Possible Solution

Keep this open for an explicit stuck-run recovery design in core session/run controls. A conservative implementation would add a separate recovery/reset option that aborts active work and can clear selected model plus thinking/reasoning overrides back to defaults, while preserving current `/new` semantics unless maintainers intentionally change them. If the current behavior is intended, document the recovery path clearly, such as `/think off`, switching models, or a reset command that includes model/thinking cleanup.
