## Summary

Keep this issue open. Current main has useful reset cleanup, including aborting active runs and clearing auto fallback model pins, but bare `/new` and `sessions.reset` still intentionally preserve user-selected model plus session thinking/reasoning overrides. That means the reported Opus/Kimi reasoning-loop recovery problem is not implemented unless the user also changes model or thinking state.

## Best Possible Solution

Keep this open for a deliberate stuck-run recovery design in core session/run controls. The likely fix is an explicit recovery/reset option that aborts active work and can clear selected model, thinking, and reasoning overrides back to defaults, while preserving existing `/new` semantics unless maintainers intentionally change them. If current behavior is intended, document the recovery path clearly, such as `/think off`, `/new <model>`, or switching away from the affected model before resetting.
