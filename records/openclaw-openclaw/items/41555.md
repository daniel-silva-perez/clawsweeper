## Summary

Keep this issue open. Current main still preserves explicit user-selected model overrides and session thinking/reasoning overrides across bare `/new`, `/reset`, and gateway `sessions.reset`, so the reported expectation that `/new` alone should recover from a stuck Opus/thinking configuration is not implemented. Recent fixes cover adjacent active-run cleanup and auto-fallback pins, but not clearing user-selected model or thinking state on reset.

## Best Possible Solution

Keep this open for a deliberate stuck-run recovery design in core session/run controls. The likely path is an explicit recovery/reset option that aborts active work and can clear selected model, thinking, and reasoning overrides back to defaults, while preserving current `/new` semantics unless maintainers intentionally change them. If current behavior is intended, document the recovery path clearly, such as `/think off`, `/new <model>`, or switching away from the affected model before resetting.
