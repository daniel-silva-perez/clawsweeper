## Summary

Keep this issue open. Current main still treats bare `/new`, `/reset`, and gateway `sessions.reset` as fresh transcript/session resets, not as a stuck-model recovery that clears explicit model selection or thinking/reasoning overrides. Adjacent fixes clear auto-fallback pins, but the reported Opus/thinking recovery expectation remains unimplemented.

## Best Possible Solution

Keep this open for an explicit stuck-run recovery design in core session/run controls. The conservative path is a separate recovery/reset option that aborts active work and can clear selected model, thinking, and reasoning overrides back to defaults, while preserving current `/new` semantics unless maintainers intentionally change them. If current behavior is intended, document the recovery path clearly, such as `/think off`, switching models, or a reset command that includes model/thinking cleanup.
