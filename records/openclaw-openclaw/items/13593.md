## Summary

Keep open. Current main has useful cron diagnostic foundations, including run status fields, consecutive error/skip counters, failure alerts, durable JSONL run history, and generic gateway probes, but it still lacks the central requested observability surfaces: per-attempt runtime logs for all scheduled/manual outcomes, cron-aware health/readiness reporting, and `openclaw cron list` warning annotations. The overlapping diagnostics PR #44363 is still open, and #46501 was closed unmerged, so the remaining work is not implemented or superseded.

## What This Changes

The issue requests cron execution logging, skipped/failure warnings, cron-aware health reporting, and CLI status warnings when cron jobs fail, skip, or become overdue.

## Best Possible Solution

Land a coordinated cron-observability change that records bounded structured logs for scheduled, startup catch-up, and manual run outcomes; warns on skipped and non-throwing error results; exposes cron health through the maintainer-chosen probe surface; and annotates `openclaw cron list` for overdue jobs and repeated failures/skips while preserving existing run history, counters, and failure-alert behavior.

## Reproduction Assessment

Yes for the missing observability surfaces. Current-main source inspection shows non-throwing scheduled/manual outcomes can finish through state updates, events, and run history without per-attempt logger calls; `/health/cron` is absent; and `printCronList` has no warning annotation path.

## Solution Assessment

Unclear as an autonomous fix, but the requested direction is sound. The narrow implementation should be coordinated with #44363 and the existing generic probe/readiness contract before adding a cron-specific health surface.
