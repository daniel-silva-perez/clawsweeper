## Summary

Keep open. Current main has useful cron diagnostic foundations, including status fields, consecutive error/skip counters, failure alerts, durable JSONL run history, and generic gateway probes, but it still lacks the central requested observability surfaces: per-attempt runtime logs for all scheduled/manual outcomes, cron-aware health/readiness reporting, and warning annotations in `openclaw cron list`. The overlapping diagnostics PR #44363 is still open, and #46501 was closed unmerged, so the remaining work is not implemented or cleanly superseded.

## What This Changes

The issue requests cron execution observability through structured per-attempt logs, cron health reporting, and CLI warnings for failed, skipped, or overdue jobs.

## Best Possible Solution

Land a coordinated cron-observability change that records bounded structured logs for scheduled, startup catch-up, and manual run outcomes; warns on skipped and non-throwing error results; exposes cron health through the maintainer-chosen probe surface; and annotates `openclaw cron list` for overdue jobs and repeated failures/skips while preserving existing run history, counters, and failure-alert behavior.

## Reproduction Assessment

Yes for the missing observability surfaces. Read-only source inspection on current main shows non-throwing scheduled/manual outcomes can finish through state updates, events, and run history without per-attempt logger calls; `/health/cron` is absent; and `printCronList` has no warning annotation path.

## Solution Assessment

Unclear as an autonomous fix, but the requested direction is sound. The narrow implementation should be coordinated with #44363 and the existing gateway probe/readiness contract before adding a cron-specific health surface.
