## Summary

Keep open. Current main has useful cron diagnostic foundations, including persisted run status, consecutive error/skip counters, failure alerts, durable JSONL run history, and generic gateway probes, but it still does not cover the central request: per-attempt runtime logging for all scheduled/manual outcomes, cron-aware health/readiness reporting, and `openclaw cron list` warning annotations. Related PR #44363 is still open and overlapping, while #46501 was closed unmerged rather than shipped.

## What This Changes

The issue requests cron execution observability: per-attempt logs and warnings, cron health reporting, and `openclaw cron list` warnings when scheduled jobs fail, skip, or fall behind.

## Best Possible Solution

Land a coordinated cron-observability change that records bounded structured logs for scheduled, startup catch-up, and manual run outcomes; warns on skipped and non-throwing error results; exposes cron health through the chosen probe surface; and annotates `openclaw cron list` for overdue jobs and repeated failures/skips while preserving the existing run history, counters, and failure-alert behavior.
