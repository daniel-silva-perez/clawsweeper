## Summary

Keep open. Current main has partial cron diagnostics, including state counters, failure alerts, durable run-history JSONL, and generic liveness/readiness probes, but it still lacks the requested per-attempt runtime logs, cron-aware health/readiness surface, and warning annotations in `openclaw cron list`; the overlapping diagnostics PR #44363 remains open and #46501 was closed unmerged.

## What This Changes

The issue requests cron observability for execution failures: structured logs for every execution attempt, warnings for skipped or failed runs, a cron health endpoint, and CLI list warnings for unhealthy jobs.

## Best Possible Solution

Land a coordinated cron-observability change that records bounded structured logs for scheduled, startup catch-up, and manual outcomes; warns on skipped and non-throwing error results; exposes cron health through the maintainer-chosen probe surface; and annotates `openclaw cron list` for overdue jobs and repeated failures or skips while preserving existing counters, run history, and failure-alert behavior.

## Reproduction Assessment

Yes for the missing observability surfaces. Read-only source inspection on current main shows non-throwing scheduled/manual outcomes can complete through state updates, events, and run history without per-attempt logger calls; `/health/cron` is absent; and `printCronList` has no warning annotation path.

## Solution Assessment

Unclear as an autonomous fix, but the requested direction is sound. The narrow implementation should be coordinated with #44363 and the existing gateway probe/readiness contract before adding a cron-specific health surface.
