## Summary

Keep open. Current main has useful cron diagnostic foundations, including persisted run status, consecutive error/skip counters, failure alerts, durable JSONL run history, and generic gateway probes, but it still lacks the central requested cron observability: per-attempt runtime logs for all scheduled/manual outcomes, cron-aware health/readiness reporting, and `openclaw cron list` warning annotations. Related PR #44363 remains open and overlapping, while #46501 was closed unmerged rather than shipped.

## What This Changes

The issue requests cron execution observability: structured logs for each execution attempt, warnings for skipped/failure outcomes, cron-aware health reporting, and CLI list warnings for unhealthy jobs.

## Best Possible Solution

Land a coordinated cron-observability change that records bounded structured logs for scheduled, startup catch-up, and manual run outcomes; warns on skipped and non-throwing error results; exposes cron health through the maintainer-chosen probe surface; and annotates `openclaw cron list` for overdue jobs and repeated failures/skips while preserving the existing run history, counters, and failure-alert behavior.
