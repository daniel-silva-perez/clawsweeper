## Summary

Keep this issue open. Current main has partial cron diagnostics, including counters, failure alerts, durable JSONL run history, and generic liveness/readiness probes, but it still lacks the central requested observability: per-attempt runtime logs for all scheduled/manual outcomes, cron-aware health/readiness reporting, and `openclaw cron list` warnings for overdue or repeatedly failing/skipped jobs. The overlapping diagnostics PR #44363 is still open, and #46501 was closed unmerged.

## What This Changes

The issue requests cron execution observability: per-attempt logging, skipped/failure warnings, cron-aware health reporting, and `openclaw cron list` warnings for unhealthy jobs.

## Best Possible Solution

Land a coordinated cron-observability change that records bounded structured logs for scheduled, startup catch-up, and manual run outcomes; warns on skipped and non-throwing error results; exposes cron health through the chosen probe surface; and annotates `openclaw cron list` for overdue jobs and repeated failures/skips while preserving existing run history, counters, and failure-alert behavior.
