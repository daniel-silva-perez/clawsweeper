## Summary

Keep this issue open. Current main has cron state tracking, run history, failure-alert counters, and generic Gateway health/readiness probes, but it still does not implement the central requested observability: structured log output for every cron attempt, warning logs for skipped/non-throwing failures, cron-aware health/readiness, or `openclaw cron list` annotations for overdue/repeatedly failing jobs. The directly relevant PR #46501 is closed unmerged, and open PR #44363 overlaps cron diagnostics enough that maintainers should coordinate scope before starting duplicate automated work.

## Best Possible Solution

Keep this issue open and coordinate with #44363 before implementation. The best fix is a focused cron-observability change: log scheduled, startup-catch-up, and manual run attempts with status, timing, job id/name, and bounded errors; warn for skipped and non-throwing error outcomes; add cron health either as `/health/cron` or as an explicit component in the existing readiness/health model; and annotate `openclaw cron list` for overdue jobs and repeated failures/skips. #46501 is useful prior art but should not be treated as shipped.
