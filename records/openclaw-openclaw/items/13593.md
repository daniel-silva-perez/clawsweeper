## Summary

Keep open. Current main has partial cron diagnostics, including persisted run outcome state, consecutive error/skip counters, failure alerts, durable JSONL run history, and generic gateway probes, but it still lacks the central requested per-attempt runtime logging, cron-aware health/readiness reporting, and `openclaw cron list` warning annotations. The overlapping diagnostics PR #44363 is still open, and #46501 was closed unmerged rather than shipped.

## What This Changes

The issue requests cron observability for execution failures, including per-attempt logging, skipped/failure warnings, cron health checks, and CLI warnings for overdue or repeatedly unhealthy jobs.

## Best Possible Solution

Land a coordinated cron-observability change that records bounded structured logs for scheduled, startup catch-up, and manual run outcomes; warns on skipped and non-throwing error results; exposes cron health through the chosen probe surface; and annotates `openclaw cron list` for overdue jobs and repeated failures/skips while preserving the existing run history, counters, and failure-alert behavior.
