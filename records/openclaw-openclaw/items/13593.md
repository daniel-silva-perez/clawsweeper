## Summary

Keep this issue open. Current main has cron state counters, failure alerts, durable JSONL run history, and generic liveness/readiness probes, but the central observability request is still incomplete: scheduled/manual outcomes are not logged for every success, skipped, or non-throwing failure path; the HTTP probes are not cron-aware; and `openclaw cron list` does not annotate overdue jobs or repeated failures/skips. The related diagnostics PR #44363 is still open, while #46501 was closed unmerged.

## What This Changes

Request cron execution observability: log every scheduled/manual attempt and skipped/failed outcome, expose cron health, and flag unhealthy or overdue jobs in `openclaw cron list`.

## Best Possible Solution

Land a coordinated cron-observability change that records bounded structured logs for scheduled, startup catch-up, and manual run outcomes; warns on skipped and non-throwing error results; exposes cron health through the chosen probe surface; and annotates `openclaw cron list` for overdue jobs and repeated failures/skips while preserving existing run-history and failure-alert behavior.
