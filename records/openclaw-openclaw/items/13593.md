## Summary

Keep this issue open. Current main has useful partial cron diagnostics, including persisted outcome state, consecutive error/skip counters, failure alerts, durable JSONL run history, and generic gateway probes, but it still lacks the central requested surfaces: per-attempt runtime logging for every scheduled/manual success, skip, and non-throwing failure; cron-aware health/readiness reporting; and warning annotations in `openclaw cron list`. The overlapping diagnostics PR #44363 is still open, while #46501 was closed unmerged and should not be treated as shipped.

## What This Changes

The issue requests cron execution observability through structured execution logs and skip warnings, a cron health endpoint or probe integration, and `openclaw cron list` warnings for overdue or repeatedly failing/skipped jobs.

## Best Possible Solution

Land a coordinated cron-observability change that records bounded structured logs for scheduled, startup catch-up, and manual run outcomes; warns on skipped and non-throwing error results; exposes cron health through the chosen probe surface; and annotates `openclaw cron list` for overdue jobs and repeated failures/skips while preserving the existing run history, counters, and failure-alert behavior.
