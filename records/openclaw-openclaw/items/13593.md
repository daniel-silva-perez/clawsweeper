## Summary

Keep open. Current main has useful cron diagnostic foundations, including persisted run status, consecutive error/skip counters, failure alerts, durable run-history JSONL, and generic gateway probes, but it still lacks the central requested cron observability: per-attempt runtime logs for all scheduled/manual outcomes, cron-aware health/readiness reporting, and `openclaw cron list` warning annotations. The overlapping diagnostics PR #44363 is still open, while #46501 was closed unmerged.

## What This Changes

The issue requests structured logging and warnings for cron execution failures, including per-attempt execution logs, cron health reporting, and CLI warnings for unhealthy scheduled jobs.

## Best Possible Solution

Land a coordinated cron-observability change that records bounded structured logs for scheduled, startup catch-up, and manual run outcomes; warns on skipped and non-throwing error results; exposes cron health through the maintainer-chosen probe surface; and annotates `openclaw cron list` for overdue jobs and repeated failures/skips while preserving existing run history, counters, and failure-alert behavior.
