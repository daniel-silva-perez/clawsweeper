## Summary

Keep open. Current main still lacks the gateway-layer outbound secret sanitization/reference-handle behavior, and this PR remains a relevant but incomplete security implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open for maintainer security review or a revised patch. The stronger direction is to implement sanitization at the final outbound send boundary after hooks, rendering, and normalization; cover text, presentation, interactive, and channelData string surfaces; build named handles from the canonical secret target registry plus active runtime/auth-profile resolution metadata; align any messages.sanitizeSecrets setting across schema, exported types, docs, and generated config help; and add regression tests for hook rewrites and structured payload delivery.
