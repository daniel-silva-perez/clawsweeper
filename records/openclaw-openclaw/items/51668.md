## Summary

Keep open. Current main still does not implement this PR's intended gateway-layer outbound secret sanitization, and the PR remains a plausible but incomplete security implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open for maintainer security review or request a revised patch. The stronger product/code direction is to implement sanitization at the final outbound send boundary after hooks/rendering/normalization, cover text, presentation, interactive, and channelData string surfaces, build named handles from the canonical secret target registry plus active runtime/auth-profile resolution metadata, add or remove the `messages.sanitizeSecrets` config contract consistently across schema/types/docs, and add regression tests for hook rewrites plus structured payload delivery.
