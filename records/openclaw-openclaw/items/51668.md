## Summary

Keep this PR open. Current main still does not implement gateway-layer outbound secret sanitization with reference handles, and the PR remains a relevant security implementation candidate rather than obsolete work. The earlier review context is useful: the author fixed the initial compile/tokenizer findings, but later review comments and current-main source still show unresolved coverage and config-contract gaps.

## Best Possible Solution

Keep this PR open for maintainer security review or a revised patch. The stronger product direction is to implement sanitization at the final outbound send boundary after hooks, rendering, and normalization; cover text, presentation, interactive, and channelData string surfaces; derive named handles from the canonical secret target registry plus resolved runtime/auth-profile metadata; add any messages.sanitizeSecrets setting to schema, exported types, docs, and generated config help; and add regression tests for hook rewrites and structured payload delivery.
