## Summary

Keep this PR open. Current main still has the old read-only behavior, so the PR is not obsolete, but the proposed patch is incomplete because it does not update the SSH remote sandbox FS bridge and leaves public reference docs inconsistent with the changed memory-flush behavior.

## What This Changes

The PR changes sandbox mount permissions, local FS bridge write checks, config-hash recreation, memory flush gating, docs, and tests so `workspaceAccess: "none"` treats the isolated sandbox workspace as writable.

## Best Possible Solution

Land a corrected sandbox-permission patch that applies the same isolated-writable `none` semantics across Docker mounts, local FS bridge, SSH remote FS bridge, config-hash recreation, memory flush, and all public docs, with regression tests and at least targeted runtime proof for Docker plus the remote shell bridge.
