## Summary

Keep open. Current main still sends native subagent completion text into the parent-session internal event without a dedicated subagent-result cap, and the closest implementation remains open rather than merged.

## What This Changes

Add a bounded default or configuration knob so oversized subagent completion results are truncated before they are injected into a parent session prompt.

## Best Possible Solution

Review, focus, and land PR #73320 or an equivalent narrow patch that bounds task-completion internal event results before prompt injection, with tests for both runtime-context and plain-prompt formatting. If maintainers want a user-facing knob, it should align with the existing agent context-limits configuration model rather than adding a separate session-level key without schema/docs follow-through.

## Reproduction Assessment

Yes. A source-level reproduction is clear: a long child assistant reply or child completion findings value flows into internalEvents[0].result and then into formatTaskCompletionEvent without a current-main size cap.

## Solution Assessment

Yes in direction, unclear on the exact proposed key. A generic internal task-completion result cap is the narrow maintainable fix; a public knob should probably live under agents.defaults/list[].contextLimits if maintainers decide configurability is needed.
