## Summary

Keep open. Current main still passes native subagent completion text into the parent-session internal event without a dedicated subagent-result cap, and the closest fix is an open implementation PR that has not landed yet.

## What This Changes

The issue requests a configurable cap for subagent completion results before they are injected into the parent session prompt.

## Best Possible Solution

Review, fix, and land PR #73320 or an equivalent narrow patch that bounds task-completion internal event results before prompt injection, removes unrelated provider-runtime changes, and updates docs/schema only if a public config knob is added.

## Reproduction Assessment

Yes. A source-level reproduction is clear: a long child assistant reply flows through `findings` into `internalEvents[0].result`, then `formatTaskCompletionEvent` embeds it into the parent prompt without a current-main size cap.

## Solution Assessment

Yes in principle. A generic internal task-completion result cap is a narrow maintainable fix for this issue, but the exact `session.maxSubagentResultChars` key is probably not the best fit for the existing agent-scoped context-limit model.
