## Summary

Keep open. Current main still forwards the full captured subagent completion text into the requester-session internal event and prompt path with no subagent-result cap; existing tool-result and context-limit guardrails do not cover this surface, and the adjacent #6662 and #41312 cleanups address different problems.

## What This Changes

The issue requests a configurable maximum character budget for subagent completion results before they are injected back into the parent session.

## Best Possible Solution

Add a focused subagent completion-result budget before `internalEvents[].result` is rendered or forwarded, using the existing head/tail truncation helper and the current per-agent context-limit pattern. Document the default and override paths, and cover direct, queued, nested, and fallback announce paths with targeted regression tests.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence on current main: a long captured child reply flows unchanged into `internalEvents[0].result`, then the internal event formatter injects it into the requester prompt between child-result markers.

## Solution Assessment

Yes for the requested behavior, but not for the literal proposed `session.maxSubagentResultChars` shape. The narrower maintainable solution is to add an equivalent budget under the existing `agents.defaults.contextLimits` / `agents.list[].contextLimits` surface unless maintainers explicitly prefer a subagent-owned config key.
