## Summary

Keep open. Current main still has the reported backup staging path, and an open unmerged PR with closing syntax is already the focused implementation candidate, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Fix `openclaw backup create` so backup staging cannot place `manifest.json` inside an archived state tree when macOS service `TMPDIR` points under `~/.openclaw/tmp`.

## Best Possible Solution

Review and land PR #75031, or an equivalent focused change, so backup staging uses a trusted temp root outside every archived source and adds a regression that asserts exactly one root `manifest.json` when service `TMPDIR` points under the state directory.

## Reproduction Assessment

Yes. The static current-main path is high-confidence: macOS service env sets `TMPDIR` under state, backup creation writes `manifest.json` under `os.tmpdir()`, and the tar input includes both that manifest and the recursive state source. I did not create a live archive because this review was required to keep the checkout read-only.

## Solution Assessment

Yes in direction. Replacing the ambient `os.tmpdir()` dependency with `resolvePreferredOpenClawTmpDir()` or a backup-specific trusted temp root outside included sources is the narrow maintainable fix, and the open PR appears to target that exact boundary.
