## Summary

Keep open. Current main still stages backup metadata under ambient `os.tmpdir()`, while macOS service startup can set `TMPDIR` inside the OpenClaw state tree; the archive path still includes both the explicit manifest and the recursive state source. Open PR #75031 is the active unmerged implementation candidate with closing syntax, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The report asks `openclaw backup create` to stage its temporary manifest outside the backed-up OpenClaw state tree so LaunchAgent `TMPDIR=~/.openclaw/tmp` does not produce duplicate `manifest.json` entries.

## Best Possible Solution

Review and land PR #75031, or an equivalent focused change, so backup staging uses a trusted temp root outside every included backup source and a regression asserts exactly one root manifest when service `TMPDIR` points under the state directory.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists on current main: macOS service env places `TMPDIR` under state, backup creation writes `manifest.json` under `os.tmpdir()`, and tar archives both the explicit manifest and the recursive state source; I did not create a live archive because this review had to keep the checkout read-only.

## Solution Assessment

Yes in direction. Using `resolvePreferredOpenClawTmpDir()` or a backup-specific trusted staging root outside included assets is the narrow maintainable fix, provided the final patch includes the duplicate-manifest regression.
