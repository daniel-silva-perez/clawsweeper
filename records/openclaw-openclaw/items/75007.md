## Summary

Keep open. Current main still stages the backup manifest under ambient `os.tmpdir()` while macOS service environments set `TMPDIR` under the OpenClaw state directory, and the tar invocation still archives both the explicit manifest and the state source tree. The linked PR #75031 is open and unmerged with closing syntax for this issue, so this should remain open until that PR or an equivalent focused fix lands.

## What This Changes

Change `openclaw backup create` so its temporary manifest/staging directory stays outside the backed-up state tree when `TMPDIR` points at `~/.openclaw/tmp`, preventing duplicate root `manifest.json` archive entries.

## Best Possible Solution

Review and land PR #75031, or replace it with the same narrow change: use `resolvePreferredOpenClawTmpDir()` or a backup-specific safe temp root outside every included source tree, remove the ambient `os.tmpdir()` dependency for manifest staging, and add a regression asserting exactly one root manifest when service `TMPDIR` is inside state.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists from current source: macOS services set `TMPDIR` to state `tmp`, backup creation writes `manifest.json` under `os.tmpdir()`, and tar archives both that explicit manifest and the state tree. I did not create a live archive because this review was required to keep the checkout read-only.

## Solution Assessment

Yes in direction. Moving backup staging to `resolvePreferredOpenClawTmpDir()` or an equivalent safe temp root is the narrow maintainable fix, and the linked PR appears to target that exact boundary with regression coverage.
