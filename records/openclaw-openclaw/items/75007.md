## Summary

Keep open. Current main still stages the backup manifest under `os.tmpdir()` while macOS service setup deliberately points `TMPDIR` inside the OpenClaw state directory, and an open PR with closing syntax is already the implementation candidate.

## What This Changes

The issue reports that `openclaw backup create` can archive `manifest.json` twice when `TMPDIR` is set inside the OpenClaw state directory on macOS.

## Best Possible Solution

Keep this issue open until PR #75031 or an equivalent focused fix lands. The final implementation should stage backup metadata outside every included source tree and include a regression proving `TMPDIR` inside state produces exactly one root manifest entry.

## Reproduction Assessment

Yes. Current main gives a high-confidence static reproduction path: macOS service setup places `TMPDIR` under state, backup creation stages `manifest.json` under `os.tmpdir()`, and tar archives both that explicit file and the state source tree.

## Solution Assessment

Yes in direction. Using `resolvePreferredOpenClawTmpDir()` or a backup-specific guarded staging root outside all included sources is the narrow maintainable fix, and PR #75031 appears to be the current implementation candidate.
