## Summary

Keep open. Current main still stages backup metadata under ambient `os.tmpdir()` while macOS service startup sets `TMPDIR` inside the OpenClaw state directory, and backup creation still archives both the explicit manifest and the state source tree. PR #75031 is open with closing syntax for this issue, so the right next step is to review or land that PR rather than close this issue early.

## What This Changes

The issue reports duplicate root `manifest.json` entries in `openclaw backup create` archives when macOS service `TMPDIR` points inside `~/.openclaw/tmp`.

## Best Possible Solution

Review and land PR #75031, or an equivalent focused fix, so backup staging uses an OpenClaw-managed temp root outside every archived source tree and a regression proves a state-dir `TMPDIR` archive contains exactly one root manifest.

## Reproduction Assessment

Yes. The source-level path is high confidence: macOS services set `TMPDIR` under state, backup creation writes `manifest.json` under `os.tmpdir()`, and tar archives both that explicit manifest and the state tree. I did not create a live archive because this was a read-only review.

## Solution Assessment

Yes in direction. Using `resolvePreferredOpenClawTmpDir()` or a backup-specific safe temp root outside archived assets is the narrow maintainable fix, with a regression test for `TMPDIR` inside state.
