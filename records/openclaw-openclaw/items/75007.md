## Summary

Keep open. Current main still stages backup metadata under ambient `os.tmpdir()` while macOS service startup sets `TMPDIR` under the OpenClaw state directory, and the backup tar call still archives both the explicit manifest path and the state source tree. The linked PR #75031 is the active closing implementation candidate, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

`openclaw backup create` should stage its temporary manifest outside the archived OpenClaw state tree when macOS service `TMPDIR` points at `~/.openclaw/tmp`, so archives contain only one root `manifest.json`.

## Best Possible Solution

Review and land PR #75031, or an equivalent focused change, so backup creation uses a staging directory outside every included source tree and includes a regression proving a state-dir `TMPDIR` archive has exactly one root manifest.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists from current source: macOS service `TMPDIR` resolves under state, backup creation writes `manifest.json` under ambient `os.tmpdir()`, and tar archives both that explicit manifest and the state tree.

## Solution Assessment

Yes in direction. The narrow maintainable fix is to use `resolvePreferredOpenClawTmpDir()` or a backup-specific safe staging helper outside all archived sources, with a focused regression test for `TMPDIR` inside state.
