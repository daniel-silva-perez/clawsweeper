## Summary

Keep open. Current main still stages backup metadata under ambient `os.tmpdir()` while macOS service startup can point `TMPDIR` inside the OpenClaw state tree, so the reported duplicate-manifest path remains valid. The issue also has open PR #75031 with closing syntax, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that `openclaw backup create` can archive two identical root `manifest.json` entries when `TMPDIR` points under `~/.openclaw/tmp`.

## Best Possible Solution

Review and land PR #75031 or an equivalent focused change that creates backup staging directories outside every included backup source, removes the ambient `os.tmpdir()` dependency from manifest staging, and adds a regression asserting exactly one root manifest when service `TMPDIR` points under state.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists from current main: macOS service env sets `TMPDIR` under state, backup creation writes `manifest.json` under `os.tmpdir()`, and tar recursively archives the state source plus the explicit manifest. I did not create a live archive because this review was required to keep the checkout read-only.

## Solution Assessment

Yes in direction. Using `resolvePreferredOpenClawTmpDir()` or a backup-specific trusted temp root outside all archived sources is the narrow maintainable fix, and it should be paired with a regression for `TMPDIR` inside the state directory.
