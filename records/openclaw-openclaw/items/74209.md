## Summary

Keep open. Current main has mitigations for Bonjour failures and bundled runtime-dependency staging, but it still keeps Bonjour in the default gateway startup path and still treats default-enabled bundled plugins as active unless config explicitly restricts them. The remaining request is broader than the open narrow Bonjour watchdog PR and needs maintainer default-policy judgment.

## What This Changes

The issue asks OpenClaw to avoid letting default-enabled bundled plugins, especially Bonjour, block or destabilize gateway startup after upgrades on constrained Linux hosts.

## Best Possible Solution

Define and implement the intended upgrade/startup policy: either make optional bundled startup services such as Bonjour opt-in or provide a supported minimal/safe plugin profile for upgrades, while ensuring default-enabled plugins cannot delay gateway readiness or stage unused dependencies on constrained hosts. Keep #74242 as a separate narrow Bonjour watchdog mitigation if maintainers agree with that fix.
