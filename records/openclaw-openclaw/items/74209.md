## Summary

Keep open. Current main contains meaningful mitigations for the Bonjour churn loop and discovery-startup timeout, but it does not resolve the broader reported problem: Bonjour is still a default-enabled startup plugin, default-enabled bundled plugins still participate in runtime-dependency selection, and a maintainer comment explicitly kept this issue open after landing the narrow Bonjour fix.

## What This Changes

The issue requests an upgrade/default-plugin policy where optional bundled plugins, especially Bonjour, cannot make gateway startup or normal operation unusable and where operators can recover with a minimal configured plugin set.

## Best Possible Solution

Keep the issue open until maintainers choose and implement the desired default policy: either make Bonjour and optional bundled services opt-in/minimal after upgrade, or keep them default-on while proving they cannot delay readiness, restart the gateway, or stage unused dependencies on constrained hosts. The end state should include code, tests, and operator recovery docs for the selected policy.
