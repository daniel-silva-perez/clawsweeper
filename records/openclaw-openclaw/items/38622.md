## Summary

Current main still appears to reproduce the reported behavior: fixed workspace bootstrap files are read through the workspace-boundary reader, outside-workspace symlink targets are rejected by the boundary resolver, and failed reads become `[MISSING]` bootstrap records. The issue should stay open because there are open implementation PRs for this exact behavior, including a maintainer-labeled path, and the fix has not landed on main.

## Best Possible Solution

Keep this issue open until a canonical implementation PR lands. The best fix is a narrow `src/agents/workspace.ts` fallback for explicit fixed workspace bootstrap filenames that follows user-created symlinks to readable regular files while preserving the existing size budget, hardlink rejection, dangling-symlink missing behavior, non-regular rejection, cache correctness, and unchanged extra/glob bootstrap boundaries. Review and repair/land #73401 or #73988, or close those PRs with a clear replacement path before closing this issue.
