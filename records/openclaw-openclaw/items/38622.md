## Summary

Current main still matches the report: fixed workspace bootstrap files are read via the workspace boundary reader with no symlink fallback, outside-root symlink targets are rejected by boundary resolution, and those read failures become `[MISSING] Expected at: ...` Project Context entries. Keep this issue open because #73401 and #73988 are open implementation PRs for this behavior, and a fork PR still references this issue with closing syntax.

## Best Possible Solution

Keep this issue open until a canonical implementation PR lands. The best fix is a narrow src/agents/workspace.ts fallback for explicit fixed workspace bootstrap filenames that follows user-created symlinks to readable regular files while preserving size limits, cache identity, hardlink/non-regular/dangling protections, and unchanged extra/glob bootstrap behavior; review and land #73401 or #73988, or close the overlapping PRs with a clear replacement path.
