## Summary

Keep this issue open. Current main still routes fixed workspace bootstrap files through the workspace-boundary reader, converts boundary failures into missing bootstrap records, and has no targeted fallback for an explicit in-workspace symlink whose target resolves outside the workspace. There are also open implementation PRs, including one that explicitly closes this issue, so the right cleanup path is to review/land or close the canonical PR rather than closing the report early.

## Best Possible Solution

Keep this issue open until a canonical implementation PR lands. The best fix is a narrow `src/agents/workspace.ts` fallback for explicit fixed workspace bootstrap filenames that follows user-created symlinks to readable regular files while preserving the existing `MAX_WORKSPACE_BOOTSTRAP_FILE_BYTES`, hardlink rejection, dangling-symlink missing behavior, non-regular rejection, cache correctness, and unchanged extra/glob bootstrap boundary behavior. Review #73401 or #73988 as the likely current canonical paths, and close #38650 if it is superseded.
