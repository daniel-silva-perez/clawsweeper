## Summary

Current main still matches the reported bug: fixed workspace bootstrap files are read through the workspace boundary reader with no symlink fallback, outside-root symlink targets are rejected by boundary resolution, and failed reads render as `[MISSING] Expected at: ...`. Keep this issue open because related implementation PRs are still open, including #73401, #73988, and an open fork PR with closing syntax.

## Best Possible Solution

Keep this issue open until a canonical implementation PR lands. The best fix is a narrow `src/agents/workspace.ts` fallback for explicit fixed workspace bootstrap filenames that follows user-created symlinks to readable regular files while preserving size limits, cache identity, hardlink/non-regular/dangling protections, and unchanged extra/glob bootstrap behavior; review and land #73401 or #73988, or close the overlapping PRs with a clear replacement path.
