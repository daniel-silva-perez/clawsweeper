## Summary

Keep this issue open. Current main has the broader timezone-only system prompt and cache-boundary work from the related prompt-caching issues, but the narrower report still matches source: exact wall-clock `Current time:` text is generated and injected into heartbeat, isolated cron, and post-compaction prompt content, and tests explicitly lock in that behavior.

## Best Possible Solution

Keep this open for maintainer design review. The best fix is likely provider-aware prompt/payload composition: preserve stable timezone metadata, keep exact wall-clock time available when the turn truly needs it, and avoid placing volatile timestamp text inside cache-tagged stable blocks or cache checkpoints. Add regression coverage for heartbeat, isolated cron, post-compaction, and Anthropic-family payload cache markers.
