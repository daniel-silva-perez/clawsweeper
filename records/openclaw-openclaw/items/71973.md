## Summary

Keep open. Current main still matches the report: the stable system prompt only carries timezone metadata, but exact wall-clock `Current time:` text is still generated and injected into heartbeat, isolated cron, post-compaction, and session-reset prompt paths. The issue is valid, but the fix needs prompt/cache owner design rather than automatic cleanup because exact time is currently used for startup and automation grounding.

## What This Changes

Request moving volatile exact current-time injection out of cache-sensitive prompt prefixes for heartbeat, cron, post-compaction, and related automation turns.

## Best Possible Solution

Make a provider-aware prompt composition change that keeps stable timezone metadata in the system prompt, preserves exact wall-clock time only where a turn truly needs it, and prevents volatile timestamps from landing in cache-tagged stable blocks or provider cache checkpoints. Add regression coverage for heartbeat, isolated cron, post-compaction context refresh, session reset, and Anthropic/OpenAI-family payload shaping.

## Reproduction Assessment

Yes. A read-only source check on current main follows the reported paths and finds `Current time:` injected into heartbeat, isolated cron, post-compaction, and reset prompts, with existing tests asserting those lines.

## Solution Assessment

Unclear for implementation details. The requested direction is maintainable, but removing or moving the line mechanically could break daily-memory startup, cron external-content wrappers, or provider cache-control behavior, so prompt/cache owners should set the boundary first.
