## Summary

Keep this open. Current main has the broader timezone-only system-prompt/cache-boundary work, but the narrower volatile wall-clock report still matches source: exact `Current time:` text is generated and injected into heartbeat, isolated cron, post-compaction, and session-reset prompts, and Anthropic-family payload shaping can still tag the trailing user message with cache control.

## What This Changes

The issue requests moving exact wall-clock `Current time:` injection for heartbeat, cron, and post-compaction/reset prompts out of cache-sensitive prompt prefixes while retaining stable timezone metadata.

## Best Possible Solution

Make a provider-aware prompt composition change that keeps stable timezone metadata in the system prompt, preserves exact wall-clock time only where a turn truly needs it, and prevents that volatile timestamp from being placed in cache-tagged stable blocks or cache checkpoints. Add regression coverage for heartbeat, isolated cron, post-compaction context refresh, session reset, and Anthropic/OpenAI-family cache-boundary payload shaping.

## Reproduction Assessment

Yes. A read-only source check on current main follows the exact paths from the report and finds `Current time:` injected into heartbeat, isolated cron, post-compaction, and reset prompts, with existing tests asserting those lines.

## Solution Assessment

Unclear for implementation details. The requested direction is maintainable, but the exact fix should be reviewed by prompt/cache owners because removing or moving the line blindly could break daily-memory startup, cron external-content wrappers, or provider cache-control behavior.
