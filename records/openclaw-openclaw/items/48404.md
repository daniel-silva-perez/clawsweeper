## Summary

Keep open. The report is recent, concrete, and platform-specific: OpenClaw v2026.3.13 segfaults on ImmortalWrt/OpenWrt x86_64 for both the launcher and direct `dist/entry.js --help`. Current main has startup/help and compile-cache mitigations, but they do not prove the packaged OpenWrt segfault is fixed, and Linux/Node remains a supported surface.

## Best Possible Solution

Keep the issue open and ask for a retest on the latest release (`v2026.4.26`) or current main on the same ImmortalWrt/OpenWrt x86_64 host. Useful next evidence: kernel `dmesg` segfault lines, Node build/source details, `strace`/gdb/core or Node report if available, and differential runs for `NODE_DISABLE_COMPILE_CACHE=1 openclaw --help`, `NODE_DISABLE_COMPILE_CACHE=1 node dist/entry.js --help`, `OPENCLAW_NO_RESPAWN=1 openclaw --help`, `openclaw status --json`, and a normal gateway startup. If it still crashes, the fix likely belongs in the launcher/entry compile-cache/bootstrap path or in packaging/native-dependency pruning plus an OpenWrt-specific diagnostic doc/smoke check.
