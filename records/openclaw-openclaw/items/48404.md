## Summary

Keep open. The issue is recent, concrete, and platform-specific: OpenClaw v2026.3.13 segfaulted on ImmortalWrt/OpenWrt x86_64 for both the launcher and direct `dist/entry.js --help`. Current main has CLI help/startup mitigations, but source inspection does not prove the direct packaged OpenWrt crash is fixed, and Linux/Node remains a supported surface.

## Best Possible Solution

Keep the issue open and ask for a retest on the latest release or current main on the same ImmortalWrt/OpenWrt x86_64 host. Useful next evidence: `dmesg` segfault lines, Node build/source details, `strace`/gdb/core or Node report if available, and differential runs with `NODE_DISABLE_COMPILE_CACHE=1`, `OPENCLAW_NO_RESPAWN=1`, direct `node dist/entry.js --help`, `openclaw status --json`, and normal gateway startup. If it still crashes, the fix likely belongs in the launcher/entry compile-cache/bootstrap path or in package/native-dependency pruning plus an OpenWrt-specific diagnostic doc or smoke check.
