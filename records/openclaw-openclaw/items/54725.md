## Summary

Keep this PR open. Current main still does not expose session identity to provider extra-parameter hooks, and the requested direction fits OpenClaw's plugin API boundary, but the branch needs maintainer API review and rebase work before it is ready because the public type change currently outpaces the runtime contexts it affects.

## What This Changes

The PR adds optional `sessionKey` and `sessionId` fields to the provider extra-params context and threads them from embedded attempts into provider extra-param preparation.

## Best Possible Solution

Expose request/session identity through the provider hook contract in one consistent current-main path: add the typed fields, pass them through runtime-plan resolution, embedded attempts, and compaction where session context exists, and cover `prepareExtraParams`, transport params, and stream-wrapper contexts with focused contract tests or a deliberately narrower type split.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.78

Full review comments:

- **[P2] Pass session fields to every context that now advertises them:** `src/plugins/types.ts:476-481`
  - body: Adding `sessionKey` and `sessionId` to `ProviderPrepareExtraParamsContext` also exposes them on `ProviderExtraParamsForTransportContext` and `ProviderWrapStreamFnContext` through the existing derived types, but this patch only populates the `prepareExtraParams` object. A provider that reads the newly typed fields in transport or stream-wrapper hooks would get `undefined` at runtime, so either pass the fields into those contexts too or split the types so only the prepared-extra-params hook advertises them.
