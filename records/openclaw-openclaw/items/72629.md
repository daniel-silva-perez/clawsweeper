## Summary

Keep open. This is current architecture/product design work with concrete cost observations in the discussion, and current main has useful sub-agent isolation, fork, context-engine, compaction, usage, and concurrency primitives but no settled OpenClaw policy for tiered conversational context delivery, shared decision summaries, saliency routing, or pessimistic multi-agent budget reservation.

## What This Changes

The issue asks OpenClaw maintainers to decide how lower-cost conversational multi-agent coordination should preserve useful cross-agent context without replaying every prior agent message to every agent each turn.

## Best Possible Solution

Keep the design thread open until maintainers choose the boundary: define a generic coordination/cost-governance primitive, extend context-engine or sub-agent hooks so plugins can implement tiered delivery and budget reservation, or document that these policies belong in scenario-specific harnesses rather than OpenClaw core.

No, not as a high-confidence repo-local reproduction. The issue gives a plausible manual setup and later cost observations, and current source/docs confirm the structural risk, but there are no committed scripts, token traces, or deterministic test inputs to replay.
