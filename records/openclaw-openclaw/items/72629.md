## Summary

Keep open. Current main has lower-level sub-agent, session-history, usage, and context-engine primitives, but it does not resolve the product/design question raised here: how OpenClaw should preserve useful cross-agent conversational context while bounding token growth and budget risk at larger multi-agent scales.

## What This Changes

The issue requests design input on token-cost scaling for conversational multi-agent coordination, especially full-transcript broadcast across agents and rounds via session tools.

## Best Possible Solution

Keep the design discussion open until maintainers decide the boundary. A good end state would document whether OpenClaw should provide a generic coordination/cost-governance primitive, extend context-engine or sub-agent hooks, or explicitly leave tiered conversational policies to external plugins and harnesses.

## Reproduction Assessment

No high-confidence repo-local reproduction exists. The issue gives a plausible manual setup and later cost observations, but it does not include runnable scripts or token traces; source and docs do confirm that repeated full-history coordination would grow context and cost structurally.

## Solution Assessment

Not applicable as a patch because no implementation is proposed. The current primitives are useful but not a complete solution to the design question; the next step is maintainer architecture/product judgment, not a narrow code change.
