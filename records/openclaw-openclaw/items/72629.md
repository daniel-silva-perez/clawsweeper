## Summary

Keep open. The discussion is active design input with concrete cost observations, and current main has useful sub-agent isolation, context, compaction, usage, and fan-out guard primitives, but no settled OpenClaw policy for lower-cost conversational broadcast coordination, saliency routing, shared decision summaries, or pessimistic multi-agent budget reservation.

## What This Changes

The issue asks OpenClaw maintainers to decide how lower-cost conversational multi-agent coordination should preserve useful cross-agent context without replaying every prior agent message to every agent each turn.

## Best Possible Solution

Keep the design thread open until maintainers choose the boundary: a generic coordination/cost-governance primitive, context-engine or sub-agent plugin hooks for tiered delivery and budget reservation, or explicit documentation that these policies belong in scenario-specific orchestrators.
