## Summary

Keep open. This is a current design issue with concrete user and community cost observations, and current main has relevant sub-agent isolation, forking, context-engine, compaction, usage, and concurrency primitives but no decided product boundary or built-in policy for tiered conversational context delivery, shared decision summaries, saliency routing, or pessimistic multi-agent budget reservation.

## What This Changes

The issue asks OpenClaw maintainers to decide how lower-cost conversational multi-agent coordination should preserve useful cross-agent context without replaying every prior agent message to every agent each turn.

## Best Possible Solution

Keep the design thread open until maintainers choose the boundary: either define a generic coordination/cost-governance primitive, extend context-engine/sub-agent hooks so plugins can implement tiered delivery and budget reservation, or document that these policies belong in scenario-specific harnesses rather than OpenClaw core.
