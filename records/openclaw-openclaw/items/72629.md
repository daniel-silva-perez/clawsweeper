## Summary

Keep open. Current main has relevant sub-agent, session-history, usage, compaction, and context-engine primitives, but it does not answer the product/design question here: how OpenClaw should preserve useful cross-agent conversational context while bounding token growth and budget risk at larger multi-agent scales.

## What This Changes

The issue asks for design input on token-cost scaling for conversational multi-agent coordination where agents repeatedly ingest shared conversation history.

## Best Possible Solution

Keep the design discussion open until maintainers choose the boundary. A good end state would document whether OpenClaw should provide a generic coordination/cost-governance primitive, extend context-engine or sub-agent hooks for tiered delivery and budget reservation, or explicitly leave these policies to plugins and scenario harnesses.

No, not as a high-confidence repo-local reproduction. The issue gives a plausible manual setup and later cost observations, but not runnable scripts or token traces; current source and docs confirm the structural risk when an orchestrator repeatedly replays full history.
