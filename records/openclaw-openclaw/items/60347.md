## Summary

Keep open. Current main still does not implement a `memorySearch.local.gpu` selector for local embeddings. The PR addresses a real gap and includes useful author verification on an Orange Pi 5 Plus with Vulkan, but it is not obsolete or already implemented on main; it needs a rebase and end-to-end threading through the current memory-core/plugin config path before maintainers can close or merge it.

## Best Possible Solution

Keep this PR open as a useful implementation candidate, but require an end-to-end rebase: add `gpu?: "auto" | "vulkan" | "cuda" | "metal" | false` to the public memory config type, strict runtime schema, generated schema labels/help/docs, resolved memory config type and merge path, plugin/provider create option surfaces, local embedding options, and `node-llama-cpp` wrapper types; pass it to `getLlama` while preserving explicit `false`; add validation/resolver/pass-through tests for string and `false`; document CPU-only and backend-specific behavior. Keep #44202 as the broader stability/guidance tracker unless this PR intentionally expands to cover that work.
