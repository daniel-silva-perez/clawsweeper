## Summary

Keep open. Current main contains partial latency mitigations and diagnostics, but the issue is fresh, corroborated, and still broader than any one source change: Feishu streaming-card start/backoff, embedded-run startup, plugin loading, model resolution, and Active Memory setup remain on the reported reply path without end-to-end timing proof that the 4.26 regression is fixed.

## What This Changes

The issue reports a 4.22/4.23 to 4.26 upgrade regression where Feishu reactions and replies became noticeably slower, with commenters also reporting first-message, embedded-run, plugin-loader, model-catalog, and Active Memory latency.

## Best Possible Solution

Keep this as the broad upgrade-regression tracker until maintainers measure the same redacted Feishu/channel configuration on 4.22 or 4.23, 4.26, latest release, and current main. Then split or close it only after reply/reaction latency is back within the pre-regression budget, or after the remaining work is fully covered by narrower canonical issues such as #67000 and #68576.
