## Summary

Keep open. Current main has several partial performance mitigations after v2026.4.26, but it does not prove the reported Feishu reaction/reply and first-message regression is fixed. The current code still has Feishu streaming-start backoff/wait paths, default sender-name and typing API calls, active-memory still runs a fresh blocking embedded subagent per eligible turn, and related plugin-loader / embedded-run latency trackers remain open.

## Best Possible Solution

Keep this issue open as the broad Feishu/plugin-runtime latency regression cluster. Ask reporters or maintainers to retest current main or the next release with redacted per-phase timings for inbound receipt, debounce/queue, sender-name lookup, typing reaction, streaming-card start/update/fallback, active-memory hook start/end, provider/model discovery, embedded-run setup, first model token, and final delivery. If one layer is confirmed, split or link a narrow canonical fix; otherwise keep the blocker open until the 4.22/4.23 to 4.26 regression is measurably resolved.
