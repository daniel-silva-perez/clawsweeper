## Summary

Keep open. The current tree contains several latency mitigations and diagnostics, but this fresh Feishu/Lark upgrade regression is still broader than any one landed change and has no end-to-end timing proof that replies/reactions are back to the 4.22/4.23 budget on current main or the latest release.

## What This Changes

The issue reports a severe 4.22/4.23 to 4.26 latency regression for Feishu replies and reactions, with discussion later pointing to streaming-card startup/backoff plus embedded-run plugin/model setup overhead.

## Best Possible Solution

Keep this as the broad upgrade-regression tracker until maintainers measure the same redacted Feishu/Lark configuration on 4.22 or 4.23, 4.26, v2026.4.29, and current main. Then split out focused repairs only for the measured dominant bottleneck, while keeping narrower existing items such as #67000, #66792, #68576, and #42079 linked for their specific remaining surfaces.

Yes.
