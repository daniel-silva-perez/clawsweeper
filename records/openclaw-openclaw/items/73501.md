## Summary

Keep open. The report is fresh, corroborated by multiple users, and current main shows mitigation and diagnostic work after v2026.4.26 but not a complete, shipped fix for the broad reply/reaction latency regression.

## What This Changes

The issue reports a 4.22/4.23 to 4.26 regression where Feishu reactions/replies and other chat surfaces respond seconds to tens of seconds slower with the same configuration.

## Best Possible Solution

Keep this as the broad upgrade-regression tracker until maintainers can compare v2026.4.22/v2026.4.23, v2026.4.26, and current main with redacted per-phase timings. Use the new embedded-run startup logs plus Feishu streaming/reaction timings to split narrow fixes only after a layer is proven, then close this when the user-visible 4.26 latency regression is measurably resolved or fully covered by narrower canonical issues.
