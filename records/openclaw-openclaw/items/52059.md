## Summary

Keep this PR open. Current main still does not implement `gateway.auth.scopes`, still clears self-declared scopes for device-less shared-token/password WebSocket sessions, and still has tests/docs describing that behavior. The PR remains a plausible gateway auth feature, but the current PR head is stale against main’s `auth-resolve.ts` split and its live diff still uses `sharedAuthOk` as the regrant guard, which would include trusted-proxy auth on current main.

## Best Possible Solution

Keep this PR open for maintainer or contributor follow-up. The best path is a rebase onto current `auth-resolve.ts`, adding `gateway.auth.scopes` to config types/schema/help/labels, restoring configured scopes only for device-less connections whose actual auth method is token or password, and adding focused gateway auth tests for configured scopes, default cleared scopes, and trusted-proxy/Tailscale non-elevation.
