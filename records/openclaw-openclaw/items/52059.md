## Summary

Keep this PR open. Current main still does not implement `gateway.auth.scopes`: the config type and strict schema do not accept the field, resolved gateway auth does not carry configured scopes, and the WebSocket handshake still clears device-less shared-auth scopes without a configured regrant. The PR remains a plausible gateway-auth feature, but it needs a rebase onto the current `auth-resolve.ts` / `ws-connection` structure plus a security-tight token/password-only gate and regression coverage.

## Best Possible Solution

Keep this PR open for contributor or maintainer follow-up. The best path is to rebase onto current main, add `gateway.auth.scopes` through the config type/schema/generated help/docs surface, carry it through `auth-resolve.ts`, restore configured scopes only for device-less sessions whose actual auth method is token or password, and add focused gateway tests for configured regrant, default cleared scopes, trusted-proxy non-elevation, and Tailscale non-elevation.
