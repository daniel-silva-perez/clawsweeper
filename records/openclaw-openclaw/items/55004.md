## Summary

Keep this PR open because the reported dashboard preflight bug is still present on current main, but the PR head is not safe to merge as-is. The branch adds the intended local reachability gate, yet it is stale against later dashboard token-redaction and TLS-link fixes and would regress both if merged directly.

## What This Changes

The PR adds a local-mode dashboard reachability preflight before clipboard/browser side effects, prints config and daemon remediation when unreachable, adds dashboard command tests, and exposes daemon loaded state in status summaries.

## Best Possible Solution

Reapply the local dashboard reachability preflight on current main while preserving current token redaction, SecretRef handling, TLS-aware link resolution, --no-open behavior, and existing custom/tailnet bind behavior, then land it with focused dashboard regression coverage and a changelog fix entry if user-visible behavior changes.

## Reproduction Assessment

Yes. The original bug is reproducible from source and the linked report: current main reaches clipboard/browser side effects without a local reachability check, while the PR regressions are reproducible by token-auto-auth and TLS-enabled dashboard configurations against the PR head.

## Solution Assessment

No. The intended preflight is the right narrow direction, but this PR head is not the best current solution because it predates later token-redaction and TLS work; a clean rebase or replacement patch on current main is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep bearer tokens out of dashboard logs:** `src/commands/dashboard.ts:149`
  - body: This branch logs `dashboardUrl`, which includes `#token=...` when token auto-auth is enabled. Current main logs only the non-tokenized base URL and has regression coverage that logged dashboard lines never contain the bearer token, so merging this would re-expose dashboard auth tokens.
  - confidence: 0.91
- **[P1] Stop passing tokens into SSH hints:** `src/commands/dashboard.ts:187-191`
  - body: When browser open fails, this branch passes the bearer token into `formatControlUiSshHint`; that helper includes an authenticated URL in its returned string, and `dashboardCommand` logs that hint. Current main deliberately calls the helper without token material.
  - confidence: 0.89
- **[P2] Preserve TLS-aware dashboard links:** `src/commands/dashboard.ts:136-141`
  - body: This branch calls `resolveControlUiLinks` without `tlsEnabled`, so `gateway.tls.enabled: true` still produces `http://`/`ws://` dashboard links and probes the wrong scheme. Current main passes TLS state and the docs require `https://`/`wss://`.
  - confidence: 0.87
