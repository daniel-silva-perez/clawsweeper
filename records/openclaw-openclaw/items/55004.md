Full review comments:

- **[P1] Keep bearer tokens out of dashboard logs:** `src/commands/dashboard.ts:149`
  - body: This branch logs `dashboardUrl`, which includes `#token=...` when token auto-auth is enabled. Current main logs only `links.httpUrl` and has a regression test ensuring logs never contain the bearer token or `#token=`, so merging this branch would re-expose dashboard auth tokens.
  - confidence: 0.91
- **[P1] Stop passing tokens into SSH hints:** `src/commands/dashboard.ts:190`
  - body: When browser open fails, this branch passes the bearer token into `formatControlUiSshHint`; that helper includes an authenticated URL in the returned string, and `dashboardCommand` logs the hint. Current main deliberately calls the helper without a token and tests the SSH path for token redaction.
  - confidence: 0.89
- **[P2] Preserve TLS-aware dashboard links:** `src/commands/dashboard.ts:136-141`
  - body: This branch calls `resolveControlUiLinks` without `tlsEnabled`, so `gateway.tls.enabled: true` still produces `http://`/`ws://` dashboard links and probes the wrong scheme. Current main passes TLS state, and docs/tests require `https://`/`wss://`.
  - confidence: 0.87
