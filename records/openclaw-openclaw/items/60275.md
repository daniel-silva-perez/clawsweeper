## Summary

Current main does not implement native per-session heartbeat entries. Heartbeat is still modeled as one merged heartbeat config per enabled agent, with an optional single session override and ad hoc targeted wake/run overrides. Cron already has persistent session-target scheduling and run history, but the docs and implementation keep that distinct from heartbeat, and the requested first-class heartbeat entry/control-plane model remains unimplemented.

## Best Possible Solution

Keep this issue open as a core automation design request. A complete fix would add native durable heartbeat entries per agent/session, explicit concurrency and skip/queue semantics, persisted enablement/run state, operator RPC/CLI/UI inspection and mutation, and docs that clearly separate heartbeat-entry scheduling from session-local gating and cron.
