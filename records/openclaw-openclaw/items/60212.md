## Summary

Keep this PR open. Current main still only retries likely interim acknowledgement text, not empty sanitized final output, and it does not have the PR’s CLI/embedded no-tools repair path or fail-loud empty-output handling. The PR also has an unresolved P1 review concern about CLI session reuse that should be addressed before landing.

## Best Possible Solution

Keep the PR open and revise it: implement the empty-output repair in the cron execution path, make the repair pass tool-free for embedded and CLI-backed runs without breaking CLI session reuse, convert unrecovered empty deliverables into an explicit cron error, and keep focused regression tests for repair success, repair failure, and bundled-MCP disablement.
