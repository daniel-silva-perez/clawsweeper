## Summary

Keep this PR open. Current main still catches Discord `fetchUser("@me")` failures and continues with an undefined bot identity, while the submitted patch is a stale, incomplete implementation candidate for the still-open security/regression tracker #42219.

## Best Possible Solution

Keep this PR open as a useful but stale implementation candidate. The best fix is a current-main rewrite in `extensions/discord/src/monitor/provider.startup.ts` that fails Discord startup when `fetchUser("@me")` rejects or returns no usable bot id, with focused regression tests for both cases and any defense-in-depth mention-gate cleanup tracked with #42219.
