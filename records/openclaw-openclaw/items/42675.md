## Summary

Keep this PR open. Current main still lets Discord startup continue when the bot identity lookup throws, and the submitted patch targets the old `src/discord` layout rather than the active Discord plugin code under `extensions/discord`.

## Best Possible Solution

Keep this PR open as an implementation candidate, or supersede it with a current-layout PR. The maintainer fix should port fail-fast behavior into `extensions/discord/src/monitor/provider.startup.ts`, explicitly reject a resolved bot identity without a usable `id`, add focused regression tests for rejected and malformed identity lookups, and then resolve #42219 through the merged fix.
