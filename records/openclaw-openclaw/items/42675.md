## Summary

Keep this PR open. Current main still fails open when Discord bot identity lookup rejects or returns no usable id, and the submitted one-line patch is a stale, incomplete implementation candidate for the still-open security/regression tracker #42219.

## Best Possible Solution

Keep this PR open as a useful but stale implementation candidate. The best fix is a current-main update in `extensions/discord/src/monitor/provider.startup.ts` that aborts Discord startup when `fetchUser("@me")` rejects or yields no valid bot id, plus focused regression tests and any mention-gate defense-in-depth tracked with #42219.
