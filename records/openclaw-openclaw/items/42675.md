## Summary

Keep this PR open. Current main still fails open when Discord bot identity lookup rejects or returns no usable id, and the linked security/regression tracker #42219 remains open. The contribution is directionally useful, but it is stale against the current plugin layout and incomplete because it only handles thrown `fetchUser("@me")` failures, not empty or invalid identity responses.

## Best Possible Solution

Keep this PR open as a useful but stale implementation candidate. The best maintainer path is to port the fail-fast behavior to `extensions/discord/src/monitor/provider.startup.ts`, validate that the returned bot identity has a usable id, add focused regression tests for rejected and invalid lookup results, and then resolve #42219 through the merged fix or close this PR as superseded by a newer implementation.
