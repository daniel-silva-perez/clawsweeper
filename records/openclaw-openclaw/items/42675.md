## Summary

Keep this PR open. Current main still allows Discord startup to continue with an undefined bot identity, and the linked security/regression tracker #42219 remains open. The PR is directionally useful, but it targets the old `src/discord` layout and needs a current-layout port plus validation that a resolved bot identity has a usable `id`.

## Best Possible Solution

Keep this PR open as an implementation candidate. The maintainer path is to port the fail-fast behavior into `extensions/discord/src/monitor/provider.startup.ts`, explicitly reject a resolved bot identity without a usable `id`, add focused regression tests for rejected and malformed identity lookups, and then resolve #42219 through the merged fix or close this PR as superseded by a newer complete implementation.
