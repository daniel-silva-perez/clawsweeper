## Summary

Keep open. Current main still reproduces the reported plugin hook context behavior, and PR #75249 is already linked with closing syntax, so this issue should remain open until that PR lands or is replaced.

## What This Changes

The issue reports that `api.registerHook()` shallow-clones hook context while injecting `pluginConfig`, causing plugin-registered `agent:bootstrap` mutations to `event.context.bootstrapFiles` to be discarded before prompt context is built.

## Best Possible Solution

Keep the issue open while PR #75249 is reviewed; the desired end state is a backwards-compatible wrapper change that preserves original hook context identity for mutable hook fields while temporarily exposing and restoring `pluginConfig`, with regression coverage for plugin-registered `agent:bootstrap` mutations.

## Reproduction Assessment

Yes. Register a plugin hook through `api.registerHook('agent:bootstrap')`, assign a sentinel array to `event.context.bootstrapFiles`, then resolve bootstrap context; current main's wrapper sends the assignment to a cloned context while the bootstrap consumer reads the original event.

## Solution Assessment

Yes. The narrowest maintainable fix is in the plugin hook wrapper, not a special case in bootstrap resolution or a breaking immutable-hook redesign; the issue should remain open until the linked fix PR or an equivalent replacement merges.
