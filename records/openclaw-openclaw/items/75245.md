## Summary

Keep open. Current main still has the plugin hook wrapper behavior described in the report, and an open fix PR already references this issue with closing syntax, so the issue should remain open until that PR lands or is replaced.

## What This Changes

The issue requests preserving mutable `event.context` changes made by plugin-registered hooks, especially `agent:bootstrap` updates to `context.bootstrapFiles`.

## Best Possible Solution

Review and land PR #75249, or an equivalent narrow change that preserves the original hook context while temporarily adding and restoring `pluginConfig`, with regression coverage for plugin-registered `agent:bootstrap` mutation and pluginConfig cleanup.

## Reproduction Assessment

Yes. A source-level reproduction is clear: register a plugin hook through `api.registerHook('agent:bootstrap')`, assign a sentinel array to `event.context.bootstrapFiles`, then resolve bootstrap context and observe current main still reads the unmodified original event context.

## Solution Assessment

Yes. Keeping this issue open while the linked fix PR is reviewed is the right workflow; the narrow code fix should live in the plugin hook wrapper, not in an `agent:bootstrap` special case or a larger immutable hook API redesign.
