## Summary

Keep open. Current main still emits the reported `ready for user delivery` wording in the subagent completion handoff path, tests still assert that wording, and the requested `available for review` plus external-final-state verification guard is not present. The latest tagged release `v2026.4.26` contains the same source/test wording, and related cleanup reports describe local compiled-bundle patches rather than a merged source fix.

## Best Possible Solution

Keep this issue open as the canonical source-level bug. The best fix is a focused PR that updates `src/agents/subagent-announce.ts` so top-level subagent completion handoff wording says the child result is available for review and explicitly tells the requester agent to verify whether the child produced only text/analysis or actually applied and verified the external final state before claiming done/ready/complete. Update the colocated formatting tests, the subagents docs, and a user-facing changelog fix entry credited to the reporter.
