## What This Changes

The issue requests changing subagent completion handoff wording from “ready for user delivery” to review-oriented wording with an explicit instruction to verify whether external state was actually applied before claiming completion.

## Best Possible Solution

Keep this issue open as the canonical source-level bug. The maintainable fix belongs in `src/agents/subagent-announce.ts`: top-level subagent completion handoff should describe the child output as available for review and instruct the requester to verify whether the output is only text/analysis or whether the external target was actually applied and verified before claiming done/ready/complete; tests, public subagents docs, and a credited changelog entry should move with it.
