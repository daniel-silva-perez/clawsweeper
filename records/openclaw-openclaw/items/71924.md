## Summary

Current main still does not implement this PR’s intended GitHub Copilot catalog behavior. The bundled Copilot provider still publishes an empty catalog, does not include plain `gpt-5.5` in the Copilot xhigh thinking list, and still resolves unknown plain `gpt-5.5` through the synthetic 128k/8192 fallback. The PR remains a focused implementation candidate for the linked GPT-5.5 `/models` capability issue rather than cleanup-close material.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate. The best path is to land this PR or an equivalent provider-owned patch that fetches GitHub Copilot `/models`, maps capability fields into model metadata, includes plain `gpt-5.5` in Copilot xhigh thinking profiles, and keeps focused coverage in the Copilot plugin tests.
