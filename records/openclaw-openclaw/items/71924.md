## Summary

Current main still does not implement this PR's intended GitHub Copilot catalog behavior. The bundled Copilot provider returns an empty model catalog, does not include gpt-5.5 in the xhigh thinking profile list, and still relies on conservative synthetic fallback metadata for unknown Copilot models. The PR is a focused implementation candidate for the linked GPT-5.5 catalog problem, so conservative cleanup should leave it open for maintainer review rather than close it.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate. The best path is to land this PR or an equivalent provider-owned patch that fetches GitHub Copilot /models during catalog discovery, maps capability fields into model metadata, adds gpt-5.5 xhigh thinking support, and keeps focused coverage in extensions/github-copilot/index.test.ts plus the existing Copilot model tests.
