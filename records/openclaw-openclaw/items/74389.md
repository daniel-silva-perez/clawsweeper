## Summary

Keep open. This PR is aimed at a valid Control UI/WebChat live-rendering bug, and its terminal cleanup direction is useful, but the current patch still leaves the multi-tool cumulative duplicate case unresolved because it trims only the live stream tail after previously committed stream segments are already rendered unchanged.

## What This Changes

The PR updates Control UI chat controller/rendering to clear live tool stream artifacts on terminal chat events, trim committed stream prefixes from the live stream tail, and add regression tests for cleanup and one-segment stream trimming.

## Best Possible Solution

Normalize live Control UI stream text at the point where cumulative provider output becomes displayed stream cards, so each narration card contains only the fresh delta across multiple tool cycles, then keep the terminal-event artifact cleanup while preserving `toolStreamOrder` until gateway history reload decisions run.

## Reproduction Assessment

Yes. #72341 gives a concrete multi-tool live-turn reproduction, and the source path confirms it: each new tool event can commit the whole current cumulative `chatStream` into `chatStreamSegments`, which the renderer then displays unchanged.

## Solution Assessment

No. The terminal cleanup is a useful part of the fix, but the proposed renderer change is not sufficient for the reported multi-tool duplicate sequence because it trims only the final live tail, not cumulative text already committed into earlier stream segment cards.
