## Summary

Keep open. Current main does not implement the PR's exact terminal-event cleanup inside `handleChatEvent`, and the linked report has fresh Control UI repros consistent with stale live render state. The PR is useful, but it needs maintainer review because clearing tool-stream state inside `handleChatEvent` may run before the gateway wrapper checks whether a final-with-tools history reload is needed.

## What This Changes

The PR adds terminal-event calls to reset live Control UI tool-stream state in `handleChatEvent` and adds a chat controller regression test for final payloads after tool streaming.

## Best Possible Solution

Land a Control UI fix that clears stale live tool/text artifacts on terminal owned chat events while preserving the final-with-tools history reload contract, likely by snapshotting `hadToolEvents` before any reset or moving the reset to a point that does not erase the reload signal.
