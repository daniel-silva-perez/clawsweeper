## Summary

Keep open. Current main still lacks the requested per-channel or broader tool-output visibility policy. The global `messages.suppressToolErrors` setting still does not suppress mutating tool failures such as the reported `message send` case, and the channel streaming controls hide progress chatter rather than error payloads. Related PR #51091 and broader issue #39406 are open, but neither is shipped current-main behavior.

## Best Possible Solution

Keep this issue open for a core runtime/config design. Add a documented tool-output visibility policy with a global default and per-channel overrides, apply it before chat-surface delivery while keeping tool failures available to the agent, preserve safe defaults for unresolved mutating actions, and add focused tests for `message` and `exec` failures across Telegram, Discord, and Slack streaming paths. Coordinate with #39406 and #51091 rather than treating either open item as shipped behavior.
