## Summary

Keep open. Current main still lacks the requested per-channel or broader tool-output visibility policy. The existing global `messages.suppressToolErrors` setting is still bypassed for mutating tool failures, including the issue's `message send` example, and the channel streaming knobs suppress progress chatter rather than error payloads.

## Best Possible Solution

Keep the issue open for a core runtime/config design: add a documented tool-output visibility policy with global defaults and per-channel overrides, apply it before chat-surface delivery while keeping tool errors available to the agent, update schema/docs, and add focused tests for mutating `message` and `exec` failures across Telegram/Discord/Slack streaming paths. Coordinate with #51091 rather than treating that narrower open PR as a complete replacement.
