## Summary

Keep open. Current main still does not implement derived `routeTo` model refs or per-entry `agents.defaults.models[...]` `contextTokens`, and the PR diff still has concrete correctness issues around dynamic-provider preparation and context-budget semantics that need follow-up before merge.

## What This Changes

This PR adds `routeTo` and `contextTokens` to `agents.defaults.models`, routes runtime model resolution through the configured upstream ref, and adds schema and unit-test coverage for derived model refs.

## Best Possible Solution

Keep this PR open as an implementation candidate, but update it so routed-model resolution is computed once and applied consistently to sync resolution, async dynamic-provider preparation, runtime execution, context budgeting, schema/help, and picker/list behavior. The final contract should explicitly define params inheritance and include focused tests for same-provider routing, cross-provider dynamic routing, cycles, aliases, and context budgets.

## Reproduction Assessment

Yes. The PR findings have high-confidence static repro paths: configure a derived ref that `routeTo`s a dynamic-only provider model to hit the unprepared target provider path, and configure upstream `params.context1m` with a derived ref that omits `params`/`contextTokens` to observe the context-budget fallthrough.

## Solution Assessment

No, not as-is. A generic `routeTo` field is a reasonable direction, but the implementation should route before both async preparation and final resolution, and it should either inherit upstream params intentionally or document and test that params are not inherited.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Prepare routed providers before dynamic resolution:** `src/agents/pi-embedded-runner/model.ts:616-626`
  - body: `resolveModelWithRegistry` rewrites `provider/modelId` via `routeTo`, but `resolveModelAsync` still prepares dynamic models using the original provider/model before it calls into that resolver. If a derived ref routes across providers to a model that is only available after the target provider's prepare hook runs, OpenClaw prepares the wrong provider and can still return `Unknown model`. Compute the routed ref before the async prepare step and use the same effective provider/model for prepare and resolution.
  - confidence: 0.91
- **[P2] Define context params inheritance for routed refs:** `src/agents/context.ts:402-410`
  - body: The new context path reads `params` from the original derived ref, then checks the routed upstream ref for Anthropic 1M support. A derived ref that omits `params` and relies on the upstream entry's `params.context1m` will fall through to the smaller/default budget even though execution routes to the 1M-capable upstream model. Either fall back to routed-ref params or document that params are not inherited and add coverage for that case.
  - confidence: 0.86
