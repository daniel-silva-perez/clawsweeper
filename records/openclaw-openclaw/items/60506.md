## Summary

Keep open. Current main does not implement the PR's derived `routeTo` model refs or per-entry `agents.defaults.models[...]` `contextTokens`; it only has aliases/params/streaming on that config surface and separate `models.providers.*.models[].contextTokens` support. The PR is still a plausible feature path, but it needs maintainer review and updates for the unresolved review comments before it could land.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is to update it against current main with a documented generic routed-model contract: add `routeTo` and per-entry `contextTokens` to the config type, Zod schema, generated schema/help as needed; route both sync and async model resolution through the effective provider/model; define or document whether params inherit from the upstream entry; and add focused tests for picker/list visibility, runtime execution, context budgeting, cycles, and dynamic-provider preparation.
