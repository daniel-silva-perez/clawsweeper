## Summary

Keep open. The PR targets a real unresolved tool-authorization bug on current main, but it carries the protected `maintainer` label and the proposed patch has a concrete CI-backed regression in the media-root contract plus a migration-warning false positive that should be fixed before merge.

## What This Changes

The PR removes implicit exec/process and filesystem profile grants from configured `tools.exec`/`tools.fs` sections, adds a startup migration warning, updates focused policy tests, and adds a changelog entry.

## Best Possible Solution

Land a maintainer-reviewed security fix that separates tool configuration from tool grants, aligns the media-root tests/contract with the intended explicit filesystem access rule, narrows migration warnings to configurations that actually lose access, and then lets #47487 close through the merged fix.

## Reproduction Assessment

Yes. The original issue has a high-confidence source-level reproduction through `resolveEffectiveToolPolicy()` on current main, and the PR regression has a high-confidence CI reproduction from the failing `src/media/local-roots.test.ts` annotation on the PR head.

## Solution Assessment

No, not yet. Removing implicit grants is the right direction, but the patch needs to resolve the media-root contract failure and avoid warning for profiles that already grant the detected tools before it is the best maintainable fix.
