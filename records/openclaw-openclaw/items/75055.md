## Summary

Keep open. The PR is a relevant fix for the #47487 tool-profile bypass, but it has a protected `maintainer` label and is not merge-ready: the PR head already has a failed media-root CI shard, and the new migration warning appears broader than the actual restrictive-profile migration case.

## What This Changes

The PR branch stops `tools.exec` and `tools.fs` config sections from being merged into profile-stage tool allowances, adds a startup migration warning, updates policy tests, and adds a changelog entry.

## Best Possible Solution

Land a maintainer-reviewed security fix that decouples tool configuration from tool grants, updates the media-root contract/tests to require explicit filesystem access, narrows migration warnings to configurations that actually lose access, and then let #47487 close through the merged fix.

## Solution Assessment
