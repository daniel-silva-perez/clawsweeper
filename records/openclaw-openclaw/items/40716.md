## Summary

Current main partially addresses the model replay/context side by filtering transcript-only OpenClaw assistant artifacts before provider replay, but it still does not implement this PR's visible/API consumer-path filtering for `chat.history` or `sessions.get`. The PR discussion also has maintainer signal keeping this as the canonical Track B assistant-message-shape candidate under #69208, so it should remain open for maintainer review or replacement by an equivalent narrow rebased patch.

## Best Possible Solution

Keep this PR open as the canonical Track B candidate under #69208, or replace it with an equivalent narrow rebased patch. The best implementation is a shared transcript-only OpenClaw assistant-artifact predicate applied before pagination and byte budgeting in visible/API consumer paths such as `chat.history` and `sessions.get`, while preserving raw JSONL audit entries and adding focused regression tests.
