## Summary

Keep this issue open. Current main fixed the concrete pre-compaction memory-flush leak by making that embedded run runtime-only, but it did not implement the requested distinct internal-system UI treatment. Control UI still renders `system` role groups through the generic `other` bubble path, and TUI system entries are dim text helpers rather than a structured/badged block for persisted internal-system transcript rows.

## Best Possible Solution

Keep this issue open as the UI/provenance follow-up. The best fix is to carry a structural internal-system marker such as `role: "system"` or `provenance.kind: "internal_system"` through `chat.history`, render it in Control UI with a dedicated system class/header/badge and optional collapse behavior, and teach the TUI history/live paths to display the same rows as system blocks instead of user bubbles or generic dim text. Memory flush can remain runtime-only unless a future diagnostic path intentionally opts into visible system display.
