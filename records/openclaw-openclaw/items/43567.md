## Summary

Keep this issue open. Current main still has the reported memory-flush prompt path as a normal embedded prompt with no empty transcript prompt or internal-system provenance, and the Gateway/UI history paths do not convert that persisted user-role row into a distinct runtime/system block. The provided discussion also identifies related open items and PR #70348, but that PR is unmerged and only filters some chat history artifacts rather than implementing the requested distinct system presentation.

## Best Possible Solution

Keep this issue open and implement a structural runtime/system transcript contract for memory flush and similar internal turns. The smallest robust path is to pass `transcriptPrompt: ""` when the flush prompt should not be persisted as user-visible chat, or attach `provenance.kind: "internal_system"` with a stable source when it should remain visible. Then update `chat.history` projection, Control UI rendering, and TUI history loading to show those rows as muted/badged system blocks, with focused tests for memory-flush projection plus both UI surfaces.
