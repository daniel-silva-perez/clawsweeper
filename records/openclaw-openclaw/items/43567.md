## Summary

Keep open. Current main fixed the concrete pre-compaction memory-flush leak, but it has not implemented the requested distinct system/runtime UI treatment; Control UI still sends `system` role rows through the generic `other` bubble path, Gateway projection does not promote `internal_system` provenance into a display row, and TUI history does not replay persisted `role: "system"` messages as system blocks.

## What This Changes

The issue requests showing internal runtime/system messages in a clearly distinct system-originated UI treatment instead of ordinary chat bubbles.

## Best Possible Solution

Define the display contract for intentionally visible runtime/system messages, carry a structural marker such as `role: "system"` or `provenance.kind: "internal_system"` through `chat.history`, render those rows with a dedicated Control UI system treatment, and make TUI history/live display use the same class while keeping runtime-only turns hidden.
