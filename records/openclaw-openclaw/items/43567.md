## Summary

Keep open. Current main has fixed the concrete pre-compaction memory-flush leak by making that run runtime-only, but it has not implemented the requested distinct UI treatment for intentionally visible internal/runtime/system messages; Control UI still sends `system` through the generic `other` bubble path, and the shared projection does not classify `internal_system` provenance into a system display row.

## What This Changes

Render internal runtime/system messages as clearly distinct system-originated UI blocks instead of ordinary user or assistant chat bubbles.

## Best Possible Solution

Keep this as the UI/provenance follow-up: carry an explicit structural marker such as `role: "system"` or `provenance.kind: "internal_system"` through `chat.history` only for intentionally visible runtime messages, render those rows in Control UI with a dedicated system treatment, and teach TUI history/live display to show the same class distinctly while leaving hidden runtime-only turns hidden.
