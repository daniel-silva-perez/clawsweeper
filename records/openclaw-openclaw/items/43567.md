## Summary

Keep open. Current main fixes the narrow pre-compaction memory-flush transcript leak, but it still does not provide a dedicated system/runtime display contract across Gateway projection, Control UI rendering/styles, and TUI history replay; open #70348 also overlaps the hide-versus-render decision.

## What This Changes

Render intentionally visible internal system/runtime messages in a distinct UI treatment instead of making them look like ordinary user or assistant chat traffic.

## Best Possible Solution

Define a structural display contract for intentionally visible runtime/system messages, carry that marker through chat.history/session.message, render it with a dedicated Control UI system treatment, and make TUI live/history display use the same system path while keeping runtime-only turns hidden.
