## Summary

Keep open. Current main fixed the narrow pre-compaction memory-flush leak into the user transcript, but it still does not implement the requested cross-surface system/runtime display contract: Gateway projection does not promote internal provenance into a distinct visible role, Control UI maps system-like rows into the generic other treatment, and TUI history replay has no dedicated system-message replay path for chat history rows.

## What This Changes

Render internal system/runtime messages in a visually distinct UI treatment rather than making them look like user or assistant chat bubbles.

## Best Possible Solution

Define a structural display contract for intentionally visible runtime/system messages, carry that marker through chat.history/session.message, render it with a dedicated Control UI system treatment, and make TUI live/history display use the same system path while continuing to suppress runtime-only artifacts that should not be user-visible.
