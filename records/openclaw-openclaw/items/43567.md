## Summary

Keep open. Current main fixes the specific pre-compaction memory-flush prompt leak, but it still does not implement the broader requested system-message display contract: gateway projection preserves remaining visible system/provenance rows without a distinct display role, Control UI renders `system` rows through generic `other` styling, and TUI history replay ignores ordinary `role: "system"` rows instead of sending them through its system notice path.

## What This Changes

The issue asks OpenClaw to render intentionally visible internal/runtime/system-originated messages in a distinct system treatment instead of making them look like normal chat bubbles.

## Best Possible Solution

Define a structural display contract for intentionally visible runtime/system messages, carry it through `chat.history` and session message projection, render it with a dedicated Control UI system treatment, and make TUI live/history display use the same system notice path while continuing to suppress runtime-only artifacts that should not be user-visible.

## Reproduction Assessment

Yes. A source-level reproduction is available on current main: pass or trace a visible `role: "system"`/`internal_system` row through projection and rendering, and it is not mapped to a dedicated system display block; the original pre-compaction flush prompt itself is no longer the right reproduction because that narrow leak is fixed.

## Solution Assessment

Unclear for an automated fix lane. Distinct rendering is the right direction for intentionally visible system/runtime messages, but maintainers should first decide which artifacts should be hidden entirely versus visibly labeled, especially with #70348 still open.
