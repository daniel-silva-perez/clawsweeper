## Summary

Keep open. Current main fixes the narrow pre-compaction memory-flush transcript leak, but it still does not implement the requested cross-surface contract for intentionally visible runtime/system messages: gateway projection does not promote internal provenance into a system display role, Control UI still falls through to generic styling for system-like rows, and TUI history replay only renders command-marked rows as system notices.

## What This Changes

Request to render internal system/runtime messages in a visually distinct Control UI/TUI chat treatment instead of normal user or assistant chat bubbles.

## Best Possible Solution

Define a structural display contract for intentionally visible runtime/system messages, carry that marker through `chat.history` and session message projection, render it with a dedicated Control UI system treatment, and make TUI live/history display use the same system path while continuing to suppress runtime-only artifacts that should not be user-visible.

## Reproduction Assessment

Yes for the remaining behavior: a source-level projection/UI replay check shows no dedicated path for `internal_system` provenance or normal `role: "system"` history rows. The original pre-compaction memory-flush example is no longer a high-confidence reproduction because current main now leaves `transcriptPrompt` empty for that embedded run.

## Solution Assessment

Unclear as a direct implementation lane: distinct rendering is a reasonable solution for intentionally visible system/runtime messages, but related filtering work in #70348 means maintainers should first decide which artifacts should be hidden and which should be visibly labeled.
