## Summary

Keep open. Current main still lacks a manifest-level install-time config patch contract, the linked implementation PR is closed unmerged, and the remaining work is a product/security design decision for third-party persistent config mutation.

## What This Changes

Request to let `openclaw.plugin.json` declare an install-time `configPatch` that `openclaw plugins install` applies and validates so plugins can set required config without manual edits.

## Best Possible Solution

Define a documented, auditable install-time plugin config intent contract that works across service, tool, provider, hook, and channel setup. It should validate before commit, show an exact diff with a skip path, define conflict and array behavior, record provenance for uninstall guidance, and cover CLI/chat/ClawHub/npm/local install routes with tests and docs.

## Reproduction Assessment

Not applicable as a feature request, not a bug reproduction. There is a high-confidence current-main verification path: inspect the manifest type/loader, install result, persistence path, install callers, provider-auth prior art, and docs to confirm manifest-level `configPatch` is absent.

## Solution Assessment

No: the raw automatic deep-merge proposal is not yet the best maintainable solution without guardrails. A constrained config-intent contract with validation, diff display, skip behavior, allowed-path/trust rules, array semantics, and provenance is safer.
