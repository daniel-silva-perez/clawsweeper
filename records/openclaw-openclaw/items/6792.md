## Summary

Keep open. Current main still lacks a manifest-level install-time config patch contract, and the linked implementation PR was closed unmerged; the remaining work is valid but needs explicit product/security design before code changes.

## What This Changes

The issue requests an `openclaw.plugin.json` `configPatch` field that `openclaw plugins install` would merge into user config automatically during plugin installation.

## Best Possible Solution

Define a documented, auditable install-time plugin config intent contract that covers service, tool, provider, hook, and channel setup while validating before commit, showing an exact diff with a skip path, defining allowed paths and conflict/array behavior, and recording provenance for uninstall guidance across all install routes.

## Reproduction Assessment

Not applicable as a feature request rather than a bug reproduction. There is a high-confidence current-main verification path: inspect the manifest type/loader, install result, install persistence, install callers, provider-auth prior art, and docs to confirm manifest-level `configPatch` is absent.

## Solution Assessment

No: the raw automatic deep-merge proposal is not yet the best maintainable solution without guardrails. A constrained config-intent contract with validation, diff display, skip behavior, trust rules, array semantics, and provenance is safer.
