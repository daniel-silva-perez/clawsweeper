## Summary

Keep open. Current main still lacks a manifest-level install-time config patch contract: manifests do not expose or normalize `configPatch`, install results cannot carry it, and plugin install persistence only updates plugin enablement/install metadata. The prior provider-auth `configPatch` path is scoped to interactive provider setup, and related PR #43421 was closed unmerged, so the reported install friction remains unresolved.

## What This Changes

The issue requests an `openclaw.plugin.json` `configPatch` field that `openclaw plugins install` can validate, merge into user config, and report during plugin installation.

## Best Possible Solution

Define a documented, auditable install-time plugin config intent contract that works across service, tool, provider, hook, and channel setup. It should validate before commit, show the exact diff with a skip path, define conflict and array behavior, record provenance for uninstall guidance, and cover all install routes with regression tests.
