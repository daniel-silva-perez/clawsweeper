## Summary

Keep open. Current main still lacks a manifest-level install-time config patch contract: plugin manifests do not expose `configPatch`, install results cannot carry it, and CLI persistence only enables/records plugin installs. The provider-auth `configPatch` prior art remains scoped to auth/setup flows, while related PR #43421 was closed unmerged and the latest maintainer review kept the product/security contract open.

## What This Changes

Allow plugin manifests to declare an install-time `configPatch` that `openclaw plugins install` validates, merges into user config, and reports to the user.

## Best Possible Solution

Define a documented, auditable install-time plugin config intent contract that works across service, tool, provider, hook, and channel setup without accepting arbitrary silent config writes. It should validate before commit, show the exact diff with a skip path, define conflict and array behavior, record provenance for uninstall guidance, and cover every install route with regression tests.
