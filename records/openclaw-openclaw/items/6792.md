## Summary

Keep open. Current main still lacks a manifest-level install-time config patch contract: manifests do not parse or expose `configPatch`, install results cannot carry it, and the install persistence path only updates plugin state and install records. The existing provider-auth `configPatch` prior art is still scoped to auth/setup flows, and the related implementation PR #43421 was closed unmerged while the remaining product/security contract is unresolved.

## What This Changes

The issue requests a plugin manifest `configPatch` field that `openclaw plugins install` would validate, deep-merge into user config, and report during installation.

## Best Possible Solution

Define a documented, auditable install-time plugin config intent contract that can cover service, tool, provider, and hook setup without accepting arbitrary silent config writes. It should validate before commit, show a diff with a skip path, define conflict and array behavior, record enough provenance for uninstall guidance, and cover all install routes with regression tests.
