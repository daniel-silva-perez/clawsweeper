## Summary

Keep open. Current main still lacks a manifest-level install-time config mutation contract: native manifests do not expose or normalize `configPatch`, install results cannot carry it, and plugin install persistence only updates enablement, slot selection, install records, and registry refresh. The linked implementation PR #43421 was closed unmerged, and the remaining question is a maintainer-level product/security contract for third-party config writes.

## What This Changes

The issue requests a manifest-level `configPatch` field in `openclaw.plugin.json` so `openclaw plugins install` can automatically apply validated plugin-required config changes.

## Best Possible Solution

Define a documented, auditable install-time plugin config intent contract that works across service, tool, provider, hook, and channel setup without accepting silent arbitrary config writes. The contract should validate before commit, show the exact diff with a skip path, define conflict and array behavior, record provenance for uninstall guidance, and cover all install routes with regression tests and docs.
