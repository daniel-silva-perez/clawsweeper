## Summary

Keep open. Current main still does not implement manifest-level install-time config mutation: manifests do not expose or normalize `configPatch`, install results cannot carry it, install persistence only changes plugin enablement/install records/slot state, and public docs still tell users to configure plugin config manually after install. The provider-auth `configPatch` helper is real prior art, but it remains scoped to provider setup/auth flows; the linked implementation PR #43421 is closed and unmerged while the product/security contract is still unresolved.

## What This Changes

Add a manifest-level install-time config patch or intent mechanism so plugin installs can automatically apply validated configuration changes instead of requiring manual `openclaw.json` edits.

## Best Possible Solution

Define a documented, auditable install-time plugin config intent contract that works across service, tool, provider, hook, and channel setup without accepting arbitrary silent config writes. It should validate before commit, show the exact diff with a skip path, define conflict and array behavior, record provenance for uninstall guidance, and cover all install routes with regression tests and docs.
