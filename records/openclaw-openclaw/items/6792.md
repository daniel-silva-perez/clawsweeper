## Summary

Keep open. Current main still only supports `configPatch` as a provider-auth result; native plugin manifests and the install persistence path do not parse, carry, validate, diff, or apply a manifest-level install-time config patch. The related implementation PR #43421 was closed unmerged, and the remaining work needs maintainer design because it would let third-party plugin manifests mutate user config.

## What This Changes

Add a manifest-level `configPatch` or equivalent install-time config intent so plugin installs can configure required tools, providers, services, hooks, or endpoints without manual `openclaw.json` edits.

## Best Possible Solution

Design a typed, auditable install-time plugin config intent API rather than silently accepting raw arbitrary patches. It should define allowed config surfaces, conflict and array semantics, validation-before-write, visible diff plus skip/no-patch behavior, uninstall/manual-revert messaging, public docs, and regression coverage across npm, local/link, archive, bundled, ClawHub, and chat-native install flows.
