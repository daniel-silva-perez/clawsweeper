## Summary

Keep open. Current main still has no manifest-level install-time config patch contract: manifests do not expose `configPatch`, install results cannot carry it, and install persistence only enables/records plugin installs. The related implementation PR was closed unmerged, and the remaining work is a product/security contract rather than a narrow missing code path.

## What This Changes

The issue requests an `openclaw.plugin.json#configPatch` field that `openclaw plugins install` would apply to user config automatically, with deep merge behavior, validation, and visible install output.

## Best Possible Solution

Define a documented, auditable install-time plugin config intent contract that works across service, tool, provider, hook, and channel setup. It should validate before commit, show the exact diff with a skip path, define conflict and array behavior, record provenance for uninstall guidance, and cover all install routes with regression tests.
