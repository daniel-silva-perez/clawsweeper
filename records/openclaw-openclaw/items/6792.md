## Summary

Current main still does not implement manifest-level install-time `configPatch`; the existing `configPatch` support is provider-auth scoped, and plugin install paths only enable/record plugins. The issue should stay open for maintainer design because the requested API would let third-party plugin manifests mutate user config and needs typed, auditable semantics before implementation.

## What This Changes

The issue requests a manifest-level install-time `configPatch` API so plugin installs can apply required OpenClaw config automatically.

## Best Possible Solution

Keep this issue open for maintainer design of a typed, auditable install-time plugin config intent API. The design should define allowed config surfaces, conflict and array semantics, validation before write, a visible diff plus skip path, uninstall/manual-revert messaging, docs, and regression coverage across npm, local/link, archive, bundled, ClawHub, and chat-native install flows without clobbering unrelated config.
