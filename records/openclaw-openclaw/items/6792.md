## Summary

Keep open. Current main still lacks a manifest-level install-time config patch path: the manifest contract does not expose or normalize `configPatch`, install results cannot carry it, and install persistence only updates plugin enable/install state. The existing `configPatch` prior art remains provider-auth scoped, while related PR #43421 was closed unmerged, so the reported plugin install friction remains unresolved.

## What This Changes

The issue requests a manifest-level `configPatch` contract so plugin install can apply required OpenClaw config changes automatically.

## Best Possible Solution

Define a documented, auditable install-time plugin config intent contract that works across service, tool, provider, hook, and channel setup. It should validate before commit, show the exact diff with a skip path, define conflict and array behavior, record provenance for uninstall guidance, and cover all install routes with regression tests.
