## Summary

Keep this issue open. Current main still does not expose a delivery post-condition or retry path for recovered/fallback agent runs: `deliverAgentCommandResult()` resolves delivery, awaits `deliverOutboundPayloads()`, discards its result array, and returns only payload/meta. The directly related PR #53961 remains open and draft, so it is an implementation candidate rather than a reason to close this issue.

## Best Possible Solution

Keep this issue open as the canonical bug for recovered/fallback agent-run delivery drops across Discord and Telegram. The likely path is to land or revise #53961 for a structured delivery-status contract, then add a narrowly scoped post-condition/retry at the `agentCommandFromIngress()`/`deliverAgentCommandResult()` boundary that re-resolves current session delivery context when delivery was requested but not attempted or not completed, with regression coverage for overload recovery and model-fallback races.
