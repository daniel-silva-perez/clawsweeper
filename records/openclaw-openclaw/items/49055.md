## Summary

Keep this issue open. Current main still has no delivery post-condition or retry path for recovered agent runs: the gateway resolves delivery context before the run, `deliverAgentCommandResult()` discards the lower-level delivery results, and the related delivery-status PR is still open as a draft rather than merged.

## Best Possible Solution

Keep the issue open and use it as the canonical bug report for recovered-run delivery drops across Discord/Telegram. The likely implementation path is to land or revise #53961 for structured delivery status, then add a scoped retry/post-condition at the `agentCommandFromIngress()`/`deliverAgentCommandResult()` boundary that re-resolves current session delivery context when delivery was requested but not attempted or not completed, with regression coverage for overloaded recovery and model-fallback races.
