## Summary

Keep this issue open. Current main still lacks a delivery post-condition/status/retry path for recovered or fallback agent runs: the agent delivery helper awaits outbound delivery but returns only payload/meta, and the Gateway marks the run succeeded from agent completion rather than delivery completion. The related PR #53961 remains open/draft in the provided context and its head SHA is not present on current main.

## Best Possible Solution

Keep this issue open as the canonical bug for recovered/fallback agent-run delivery drops across Discord and Telegram. The best path is to land or revise #53961, or an equivalent structured delivery-status contract, then add a narrowly scoped post-condition/retry at the agentCommandFromIngress()/deliverAgentCommandResult() boundary that re-resolves fresh session delivery context when delivery was requested but not attempted or did not complete. Add regression coverage for overloaded_error recovery and model-fallback races, including the duplicate-send edge cases around empty outbound results.
