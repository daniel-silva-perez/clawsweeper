## Summary

Keep open. The requested memory-index exclusion feature is still absent from current main, and the contributor PR remains useful source work, but it is not merge-ready because it targets pre-refactor memory paths and includes an unrelated DNS/network helper.

## What This Changes

The PR adds a `memorySearch.excludeDirs` config field with default directory basenames, threads it into recursive memory file discovery/status/tests, and also adds an unrelated root `ipv4-fix.cjs` helper.

## Best Possible Solution

Land a focused current-main implementation that adds additive, normalized memory-indexing exclusions across config schema/types/help/generated metadata, builtin discovery, sync/watch/status/reindex behavior, docs, and tests; keep Telegram/network behavior out of the change and explicitly scope any QMD follow-up.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Remove the DNS override from this PR:** `ipv4-fix.cjs:3-4`
  - body: `ipv4-fix.cjs` changes process-wide DNS/address-family behavior for any process that loads it, but this PR is about memory indexing. Keep Telegram or network tuning in the existing scoped channel configuration paths instead of shipping an unrelated root helper.
  - confidence: 0.9
- **[P2] Move the walker changes to the current memory host:** `src/memory/internal.ts:56`
  - body: The PR wires exclusion logic through the old `src/memory/internal.ts` path, while current main's active builtin discovery is `packages/memory-host-sdk/src/host/internal.ts` and sync runs through `extensions/memory-core`. As written, the patch is stale and would miss or conflict with the production path.
  - confidence: 0.88
- **[P2] Skip excluded extraPaths roots before walking:** `src/memory/internal.ts:122`
  - body: The exclusion check only runs for child entries inside `walkDir`, so an `extraPaths` entry whose basename is itself excluded, such as `node_modules` or `.venv`, is still walked and indexed. Check the normalized extra path root before calling `walkDir`.
  - confidence: 0.86
- **[P2] Regenerate config metadata for excludeDirs:** `src/config/zod-schema.agent-runtime.ts:553`
  - body: Adding a strict config field without updating generated config metadata leaves schema consumers and generated config docs out of sync. A current-main implementation should regenerate the config artifacts after adding `excludeDirs`.
  - confidence: 0.78
- **[P3] Normalize user-provided excludeDirs:** `src/agents/memory-search.ts:221-225`
  - body: User-provided `excludeDirs` entries are merged raw, unlike `extraPaths`. Values with surrounding spaces or blanks will not match directory basenames, so trim and drop empty entries before adding them to the resolved set.
  - confidence: 0.74
