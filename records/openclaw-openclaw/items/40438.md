## Summary

Keep this PR open, but do not merge it as-is. The requested memory exclusion feature is still useful and not implemented on current main, but the branch is stale against the active memory-host/memory-core paths and includes an unrelated process-wide DNS helper that needs removal before any maintainer could consider the memory change.

## What This Changes

The PR adds a `memorySearch.excludeDirs` config field with old `src/memory` walker/status/test wiring and also adds a root `ipv4-fix.cjs` DNS/address-family helper.

## Best Possible Solution

Land a focused current-main memory implementation that normalizes additive directory exclusions, threads them through config schema/types/help/generated metadata, active memory-host discovery, memory-core sync/watch/status/reindex behavior, docs, and tests, while leaving Telegram/network behavior out of scope.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure `memorySearch.extraPaths` to point at a project directory containing `node_modules` or `.venv` with Markdown files, and current `listMemoryFiles` will recursively walk it because there is no configurable basename exclusion input; I did not run tests because this review is read-only.

## Solution Assessment

No. The feature direction is reasonable, but this PR is not the best implementation because it targets old paths, misses current-main reindex/generated metadata concerns, and includes an unrelated DNS helper; a rebased focused patch or replacement PR is safer.

  - body: `ipv4-fix.cjs` changes process-wide DNS/address-family behavior for any process that loads it, but this PR is about memory indexing. Remove this helper and keep Telegram or network tuning in the scoped channel/runtime configuration paths.
  - confidence: 0.9
- **[P2] Move the walker changes to the active memory host:** `src/memory/internal.ts:56`
  - body: The branch wires exclusions through the old `src/memory/internal.ts` path, while current main's builtin discovery is hosted in `packages/memory-host-sdk/src/host/internal.ts` and production sync calls it from `extensions/memory-core`.   - confidence: 0.88
- **[P2] Skip excluded extraPath roots before walking:** `src/memory/internal.ts:122`
  - body: The exclusion check only runs for child entries inside `walkDir`, so an `extraPaths` entry whose basename is itself excluded, such as `node_modules` or `.venv`, is still walked and indexed. Check the normalized extra-path root before calling `walkDir`.
  - confidence: 0.86
- **[P2] Include exclusions in the reindex scope:** `src/agents/memory-search.ts:328`
  - body: Adding `excludeDirs` changes which files belong in the index, but the branch does not include that input in the current memory reindex scope. Existing junk rows can remain until some unrelated full reindex occurs, so the normalized exclusions should be part of the scope hash and covered by a reindex-state test.
  - confidence: 0.8
- **[P2] Regenerate config metadata for the new field:** `src/config/zod-schema.agent-runtime.ts:553`
  - body: The PR adds a strict config field but does not update generated config metadata, leaving schema consumers and generated docs out of sync. Regenerate and check the config docs artifacts when adding `excludeDirs`.
  - confidence: 0.78
- **[P3] Normalize user-provided excludeDirs:** `src/agents/memory-search.ts:221-225`
  - body: User-provided entries are merged raw, unlike `extraPaths`. Values with surrounding spaces or blanks will not match directory basenames, so trim and drop empty entries before adding them to the resolved set.
  - confidence: 0.74
