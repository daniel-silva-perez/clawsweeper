## Summary

Keep this PR open for maintainer handling, but do not merge the branch as-is. The original directory-exclusion request remains useful and current main still lacks a configurable `memorySearch.excludeDirs`, while this branch is stale against the active memory host, misses current reindex/generated-metadata paths, and includes an unrelated DNS helper.

## What This Changes

The PR adds a `memorySearch.excludeDirs` config field wired through the old memory walker/status/test paths, and also adds a root `ipv4-fix.cjs` DNS/address-family helper.

## Best Possible Solution

Land a focused current-main memory implementation that normalizes additive directory exclusions, threads them through schema/types/help/generated metadata, active builtin discovery, sync/watch/status/reindex behavior, docs, and tests, while leaving Telegram/network behavior out of scope.

  - body: The exclusion check only runs for child entries inside `walkDir`, so an `extraPaths` entry whose basename is itself excluded, such as `node_modules` or `.venv`, is still walked and indexed. Check the normalized extra-path root before calling `walkDir`.
  - confidence: 0.86

  - confidence: 0.8
- **[P2] Regenerate config metadata for the new field:** `src/config/zod-schema.agent-runtime.ts:553`
