## Summary

Keep open. The feature direction is useful and current main still lacks configurable memory indexing directory exclusions, but this PR is not merge-ready because it targets removed memory files, misses the active memory-host and reindex/config metadata paths, and adds an unrelated process-wide DNS helper.

## What This Changes

The branch adds a `memorySearch.excludeDirs` config field through old memory config, walker, status, and tests, and also adds a root `ipv4-fix.cjs` DNS/address-family helper.

## Best Possible Solution

Land a focused current-main implementation that normalizes additive default/user directory exclusions, threads them through schema/types/help/generated metadata, active memory-host discovery, memory-core sync/status/reindex behavior, docs, and focused tests, while leaving Telegram/network behavior out of scope.

Yes. Source inspection gives a high-confidence path: configure `memorySearch.extraPaths` to a project directory containing `node_modules` or `.venv` with Markdown files, and current builtin discovery has no configurable basename exclusion before recursive walking.

## Solution Assessment

No. The feature direction is reasonable, but this branch is not the best implementation because it targets removed memory files, misses active memory-host/reindex/generated metadata paths, and mixes in an unrelated DNS helper.
