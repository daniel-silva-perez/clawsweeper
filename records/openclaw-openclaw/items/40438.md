## Summary

Keep this PR open, but do not merge it as-is. Current main still lacks a configurable `memorySearch.excludeDirs` surface, so the requested feature is not implemented; however the branch is stale against the active memory-host/memory-core paths and includes an unrelated DNS helper that should not land with memory indexing work.

## What This Changes

The branch threads a `memorySearch.excludeDirs` option through config/help/status, an older builtin memory walker and tests, and adds a root `ipv4-fix.cjs` DNS/address-family helper.

## Best Possible Solution

Land a focused current-main implementation that normalizes additive default/user directory exclusions, threads them through schema/types/help/generated metadata, active memory-host discovery, memory-core sync/status/reindex behavior, docs, and focused tests, while leaving Telegram/network behavior out of scope.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure builtin `memorySearch.extraPaths` to a project directory containing `node_modules` or `.venv` with Markdown files, and current active discovery has no configurable basename exclusion before recursive walking.

## Solution Assessment

No. The feature direction is reasonable, but this branch is not the best implementation because it targets removed memory files, misses active memory-host/reindex/generated metadata paths, does not skip excluded extraPath roots, and mixes in an unrelated DNS helper.

## Review Findings

Overall confidence: 0.91

Full review comments:

- **[P2] Remove the DNS override from this PR:** `ipv4-fix.cjs:3-4`
  - body: `ipv4-fix.cjs` changes process-wide Node DNS/address-family behavior for any process that loads it, but memory indexing does not need network stack changes. Remove this helper and keep any channel/network tuning in scoped runtime code.
  - confidence: 0.9
- **[P2] Move the walker changes to the active memory host:** `src/memory/internal.ts:56`
  - body: The patch wires exclusions through `src/memory/internal.ts`, but current main moved builtin discovery to `packages/memory-host-sdk/src/host/internal.ts` and production sync calls that package from `extensions/memory-core`. As written, the branch is stale and would not fix the active path.
  - confidence: 0.88
- **[P2] Skip excluded extraPath roots before walking:** `src/memory/internal.ts:122`
  - body: The exclusion check only runs for child entries inside `walkDir`, so an `extraPaths` entry whose basename is itself excluded, such as `node_modules` or `.venv`, is still walked and indexed. Check the normalized root basename before recursing.
  - confidence: 0.86
- **[P2] Include excludeDirs in the reindex scope:** `src/agents/memory-search.ts:328`
  - body: Adding `excludeDirs` changes which files belong in the index, but the branch does not include that input in the persisted reindex scope. Existing junk rows can remain until an unrelated full reindex occurs.
  - confidence: 0.8
- **[P2] Regenerate config metadata for the new field:** `src/config/zod-schema.agent-runtime.ts:553`
  - body: The PR adds a strict config field but does not update generated config metadata, leaving schema consumers and generated docs out of sync with the new public setting.
  - confidence: 0.78
- **[P3] Normalize user-provided excludeDirs:** `src/agents/memory-search.ts:221-225`
  - body: User-provided entries are merged raw, unlike `extraPaths`. Values with surrounding spaces or blanks will not match directory basenames, so trim and drop empty entries before resolving the set.
  - confidence: 0.74
