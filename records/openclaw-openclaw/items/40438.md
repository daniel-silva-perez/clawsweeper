## Summary

Keep open for maintainer follow-up. The underlying memory-index exclusion request is still useful and current main has not implemented it, but this PR is not merge-ready because it targets stale memory paths, misses current generated/config surfaces, and includes an unrelated DNS helper.

## What This Changes

The PR adds a `memorySearch.excludeDirs` config field, threads it into memory file discovery/status/tests, and adds an unrelated root `ipv4-fix.cjs` helper.

## Best Possible Solution

Land a focused current-main implementation that adds additive, normalized memory-indexing exclusions across config schema/types/help/generated metadata, builtin discovery, sync/watch/status/reindex behavior, docs, and tests; remove the DNS helper and explicitly decide whether QMD needs matching exclusion semantics or a separate follow-up tied to #54463.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Remove the DNS override from this PR:** `ipv4-fix.cjs:1-4`
  - body: `ipv4-fix.cjs` changes process-wide DNS/address-family behavior if loaded, but this PR is about memory indexing. Keep Telegram/network tuning in the existing scoped Telegram config path instead of shipping an unrelated root helper with this feature.
  - confidence: 0.9
- **[P2] Rebase onto the current memory host paths:** `src/memory/internal.ts:56`
  - body: The PR wires exclusion logic through `src/memory/internal.ts`, but current main's active builtin discovery is `packages/memory-host-sdk/src/host/internal.ts` and sync runs through `extensions/memory-core`. As written, the feature would miss the current production path or conflict with the refactor.
  - confidence: 0.88
- **[P2] Skip excluded extraPaths roots before walking:** `src/memory/internal.ts:122`
  - body: The exclusion check only runs for children inside `walkDir`, so an `extraPaths` entry whose basename is itself excluded, such as `node_modules` or `.venv`, is still walked and indexed. Check the normalized extra path root before calling `walkDir`.
  - confidence: 0.86
- **[P2] Update generated config metadata for the new field:** `src/config/zod-schema.agent-runtime.ts:553`
  - body: Adding a strict config field without updating generated config metadata/docs leaves schema consumers and the config docs generation check out of sync. Regenerate the config artifacts after adding `excludeDirs`.
  - confidence: 0.78
- **[P3] Normalize user excludeDirs entries:** `src/agents/memory-search.ts:218-225`
  - body: User-provided `excludeDirs` are merged without trimming or dropping blank strings, unlike `extraPaths`. A value like `" tmp "` will not match a `tmp` directory, so normalize entries before adding them to the resolved set.
  - confidence: 0.74

## Security Review

Status: needs_attention

Summary: The memory exclusion feature is not inherently security-sensitive, but the PR introduces an unrelated process-wide DNS helper.
