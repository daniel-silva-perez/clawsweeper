## Summary

Keep this PR open. Current main still does not implement configurable memory-index directory exclusions, so the PR is not obsolete. It remains a useful implementation candidate, but it needs a focused rebase onto the current memory-host-sdk/memory-core layout and the unrelated process-wide IPv4/DNS change should be removed before maintainer review.

## Best Possible Solution

Keep this PR open as a useful implementation candidate, but require a focused rebase onto current main: remove ipv4-fix.cjs, add an additive memorySearch.excludeDirs or broader exclude-pattern contract across schema/types/help/docs/generated metadata, thread it through resolveMemorySearchConfig, packages/memory-host-sdk file discovery, and extensions/memory-core sync/watch/status/reindex paths, skip excluded roots as well as descendants, and either implement equivalent QMD semantics or document QMD scope with tests.
