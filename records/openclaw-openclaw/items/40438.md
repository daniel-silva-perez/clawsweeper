## Summary

Keep this PR open because current main still lacks configurable memory-index directory exclusions, so the contribution is not obsolete. The branch is not merge-ready as-is because it targets the older memory implementation layout and includes an unrelated process-wide DNS/IPv4 helper.

## What This Changes

The PR adds a `memorySearch.excludeDirs` config field with defaults, threads it through the older memory indexing/status files, adds tests/help/labels, and adds a root `ipv4-fix.cjs` script.

## Best Possible Solution

Land a focused version on current main that adds an additive, documented memory indexing exclusion contract across schema/types/help/generated metadata, builtin discovery, sync/watch/status/reindex behavior, and tests; drop the IPv4 helper; and either map the behavior cleanly for QMD or document and test that QMD has separate scanner semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Remove the DNS override from this PR:** `ipv4-fix.cjs:1-4`
  - body: `ipv4-fix.cjs` changes process-wide DNS/address-family behavior if loaded, but the PR is about memory indexing. Keeping this unrelated root script in the branch creates an ops-sensitive side effect path that should not be reviewed or shipped with this feature.
  - confidence: 0.9
- **[P2] Skip excluded extraPaths roots before walking:** `src/memory/internal.ts:122`
  - body: The exclusion check only runs for children inside `walkDir`, so an `extraPaths` entry whose basename is itself excluded, such as `node_modules` or `.venv`, is still walked and indexed. Check the normalized extra path root before calling `walkDir`.
  - confidence: 0.88
- **[P3] Normalize user excludeDirs entries:** `src/agents/memory-search.ts:218-225`
  - body: User-provided `excludeDirs` are merged without trimming or dropping blank strings, unlike `extraPaths`. A config value such as `" tmp "` will not match a `tmp` directory, so normalize entries before adding them to the resolved set.
  - confidence: 0.76
