## Summary

Keep open. Current main still does not implement configurable memory exclude directories, so the feature is useful source work, but this PR is not merge-ready because it targets older memory paths, misses current generated/config surfaces, and includes an unrelated root IPv4 helper.

## What This Changes

The PR adds a memorySearch excludeDirs option with default directory-name filtering, threads it through memory file discovery/status/config metadata, adds tests, and also adds an unrelated root ipv4-fix.cjs file.

## Best Possible Solution

Land the feature as a current-main implementation in the memory host SDK and config contract: normalize and merge default/custom exclude dir names, prune excluded root extraPaths and nested directories before indexing, update generated schema/docs/test baselines, add focused tests for builtin discovery, and either implement or explicitly document QMD behavior.

## Reproduction Assessment

Yes. A focused source-level reproduction is clear: configure memorySearch.extraPaths to a directory containing node_modules/*.md on current main and listMemoryFiles will walk that directory because it has no excludeDirs input or basename pruning.

## Solution Assessment

No. The feature direction is reasonable, but this PR is not the best merge path as-is; it should be rebased onto current memory-host-sdk surfaces, remove the unrelated root script, handle excluded extraPath roots, and regenerate config artifacts.
