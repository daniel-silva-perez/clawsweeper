## Summary

Keep this PR open. The requested memory indexing exclusion feature is still missing on current main, so the PR is not obsolete, but the branch is not merge-ready because it carries an unrelated DNS helper, misses an excluded-root case, and now targets older memory files after the memory host split.

## What This Changes

The PR adds a `memorySearch.excludeDirs` config field with default directory basenames, threads it through the older builtin memory indexing and status paths, adds schema/help/label/test coverage, and adds a root `ipv4-fix.cjs` helper.

## Best Possible Solution

Land a focused current-main implementation that adds an additive, documented memory indexing exclusion contract across schema/types/help/generated metadata, builtin discovery, sync/watch/status/reindex behavior, docs, and tests; drop the IPv4 helper; and either explicitly scope QMD separately or add matching QMD-safe exclusion semantics with coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Remove the DNS override from this PR:** `ipv4-fix.cjs:1-4`
  - body: `ipv4-fix.cjs` changes process-wide DNS/address-family behavior if loaded, but the PR is about memory indexing. Keeping this unrelated root script in the branch creates an ops-sensitive side effect path that should not ship with this feature.
  - confidence: 0.9
- **[P2] Skip excluded extraPaths roots before walking:** `src/memory/internal.ts:122`
  - body: The exclusion check only runs for children inside `walkDir`, so an `extraPaths` entry whose basename is itself excluded, such as `node_modules` or `.venv`, is still walked and indexed. Check the normalized extra path root before calling `walkDir`.
  - confidence: 0.88
- **[P3] Normalize user excludeDirs entries:** `src/agents/memory-search.ts:218-225`
  - body: User-provided `excludeDirs` are merged without trimming or dropping blank strings, unlike `extraPaths`. A config value such as `" tmp "` will not match a `tmp` directory, so normalize entries before adding them to the resolved set.
  - confidence: 0.76

## Security Review

Status: needs_attention

Summary: The memory exclusion change is not inherently security-sensitive, but the PR introduces an unrelated process-wide DNS/IPv4 helper that should not be merged with it.

  - body: `ipv4-fix.cjs` disables Node auto address-family selection and forces IPv4-first DNS resolution for any process that loads it. That network behavior change is unrelated to memory indexing and should be removed from this PR.
