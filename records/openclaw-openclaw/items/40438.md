## Summary

Keep this PR open. Current main still lacks a memory indexing directory-exclusion contract, so the contribution is not obsolete, but the branch is not merge-ready because it targets older memory paths, misses an excluded-root case, and includes an unrelated DNS/IPv4 helper.

## What This Changes

The PR adds a `memorySearch.excludeDirs` config field across schema/types/help/status and older memory indexing paths, adds tests, and also adds a root `ipv4-fix.cjs` helper.

## Best Possible Solution

Land a focused current-main implementation that defines additive, documented memory indexing exclusions across config schema/types/help/generated metadata, builtin discovery, sync/watch/status/reindex behavior, docs, and tests; drop the IPv4 helper and make the QMD scope explicit or add matching QMD-safe coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Remove the DNS override from this PR:** `ipv4-fix.cjs:1-4`
  - body: `ipv4-fix.cjs` changes process-wide DNS/address-family behavior if loaded, but the PR is about memory indexing. Keeping this unrelated root script creates an ops-sensitive side effect path that should not ship with this feature.
  - confidence: 0.9
- **[P2] Skip excluded extraPaths roots before walking:** `src/memory/internal.ts:122`
  - body: The exclusion check only runs for children inside `walkDir`, so an `extraPaths` entry whose basename is itself excluded, such as `node_modules` or `.venv`, is still walked and indexed. Check the normalized extra path root before calling `walkDir`.
  - confidence: 0.88
- **[P3] Normalize user excludeDirs entries:** `src/agents/memory-search.ts:218-225`
  - body: User-provided `excludeDirs` are merged without trimming or dropping blank strings, unlike `extraPaths`. A config value such as `" tmp "` will not match a `tmp` directory, so normalize entries before adding them to the resolved set.
  - confidence: 0.76

## Security Review
