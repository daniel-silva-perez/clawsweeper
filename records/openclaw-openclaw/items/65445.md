## Summary

Keep open. Current main still only partially addresses session cruft: age-based maintenance cleans `.deleted` and `.reset` archives, optional disk-budget cleanup can remove archive artifacts under pressure, and doctor can interactively archive orphan primary transcripts, but there is still no requested garbage-collection/recycle config and `.jsonl.bak.*` is intentionally preserved by an integration test.

## What This Changes

The issue requests session-maintenance garbage collection for orphaned `.jsonl.reset.*`, `.jsonl.deleted.*`, `.jsonl.bak.*`, `.bak`, and `sessions_saved.json` artifacts, with a configurable recycle path and retention window.

## Best Possible Solution

Define one explicit session-artifact retention policy that reconciles archive preservation with disk cleanup, then implement it through `session.maintenance` types, schema/help, docs, `src/config/sessions/*`, and the gateway archive helper. Coverage should include `.jsonl.deleted.*`, `.jsonl.reset.*`, `.jsonl.bak.*`, optional legacy store backups or `sessions_saved.json` if intentionally supported, and the interaction with disk-budget cleanup.

## Reproduction Assessment

Yes. A high-confidence code/test reproduction exists: current tests create an old `.jsonl.bak.*` artifact and assert it remains after session maintenance, while source only performs age cleanup for `.deleted` and `.reset` archives.

## Solution Assessment

Unclear as proposed. The requested recycle-and-retention model may be useful, but the best fix needs maintainer policy first because current code already has partial hard-delete paths and related #56609 argues session archives should not be treated as disposable.
