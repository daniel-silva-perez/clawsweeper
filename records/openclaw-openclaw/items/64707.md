## Summary

Keep this issue open. Current main still has the shared synthetic-default account path on Feishu status/summary, the latest release comparison does not show a relevant fix, and same-author PR #64712 remains the active implementation candidate rather than a merged resolution.

## What This Changes

The issue requests that Feishu gateway restarts stop creating or surfacing an unconfigured implicit `default` account unless the user explicitly configured that account.

## Best Possible Solution

Review and land #64712, or a replacement, that distinguishes explicitly configured account ids from synthetic fallback ids in the channel status/summary path, preserves diagnostics for genuinely configured but unresolved accounts, and adds Feishu-focused multi-account restart/status regression coverage.
