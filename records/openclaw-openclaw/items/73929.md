## Summary

Keep this PR open. Current main still treats any non-empty replay-history array as prompt content, so blank replay entries can bypass the empty prompt/history/images skip path. The linked bug #73926 is still open, and the PR is a focused implementation candidate rather than something current main already implements.

## Best Possible Solution

Review and land a focused fix that changes `hasPromptSubmissionContent` from replay-array presence to model-visible replay content, with regression coverage for blank string/object/array replay entries and preservation of structured non-text replay such as images. Keep the changelog attribution for #73926 if the PR lands.
