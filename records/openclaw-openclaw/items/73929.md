## Summary

Keep this PR open. Current main still counts any non-empty replay-history array as prompt content, so blank replay entries can bypass the empty prompt/history/images skip guard. The PR is a focused implementation candidate for the linked open bug #73926, and the diff is limited to the helper, its tests, and a changelog entry.

## Best Possible Solution

Review and land this PR or an equivalent focused fix that changes `hasPromptSubmissionContent` from replay-array presence to model-visible replay content, keeps structured non-text replay such as images, and adds the blank replay regression tests plus the changelog attribution for #73926.
