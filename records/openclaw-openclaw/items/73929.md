## Summary

Keep this PR open. Current main still uses a length-only replay-history check in `hasPromptSubmissionContent`, so blank replay entries can keep an otherwise empty `/new` or `/reset` submission alive. The PR is a focused implementation candidate for the linked bug #73926, with a small diff limited to the prompt-content helper, colocated regression tests, and changelog.

## Best Possible Solution

Review and land a focused fix that changes the embedded-run submission guard from replay-array presence to model-visible replay content, with regression coverage for blank string/object/array replay entries and preservation of structured non-text replay such as images. Keep the changelog attribution for #73926 if this PR lands, and adjust the type-only block predicate only if maintainers want stricter accepted block shapes.
