## Summary

Keep open for normal maintainer review. Current main still advertises Bonjour in WSL2 when no explicit disable override is set, while this PR makes a narrow extension-owned change at the existing auto-disable boundary, adds focused tests and docs, and has no blocking review findings.

## What This Changes

The PR imports `isWSL2Sync` into the Bonjour advertiser, auto-disables mDNS inside WSL2 unless explicitly forced on, adds mocked unit coverage, and updates the gateway Bonjour/discovery docs.

## Best Possible Solution

Merge the extension-owned WSL2 auto-disable behavior after maintainer review, while leaving probing-loop recovery and broader bundled-plugin default policy to their existing related tracks.

## Reproduction Assessment

Yes. On current main, a WSL2 environment without `OPENCLAW_DISABLE_BONJOUR` still passes through `isDisabledByEnv()` and reaches `createService`; the PR's mocked WSL2 tests are the focused reproduction and regression path.

## Solution Assessment

Yes. The PR uses the existing public runtime-env helper at the Bonjour extension's existing disable boundary, preserves explicit opt-in semantics, and avoids moving WSL2 policy into core.
