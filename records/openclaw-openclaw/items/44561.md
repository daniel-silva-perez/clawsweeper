## Summary

Current main still does not implement this PR's intended gateway display-name stabilization. `buildGatewaySessionRow` can still derive a row `displayName` from stale `origin.label`, `listSessionsFromStore` and session-change snapshots both use that shared projection path, and the UI prefers the row label/displayName before its `Main Session` fallback. The PR remains useful but needs rebasing/adapting to the current row projection path.

## Best Possible Solution

Keep this PR open as a focused implementation candidate. Rebase it onto current main, move the main-session display guard into `buildGatewaySessionRow` or a shared row-display helper, preserve explicit `entry.displayName`, use `resolveAgentMainSessionKey` and `resolveMainSessionKey`, and add regression coverage for default agent main, configured main keys, global main, and stale `origin.label: "heartbeat"`.
