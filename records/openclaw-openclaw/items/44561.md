## Summary

Keep this PR open. Current main still does not implement the intended gateway-side display-name stabilization: gateway session rows can still derive `displayName` from stale `origin.label`, list responses and event snapshots both use that row projection path, and the Control UI prefers row `label`/`displayName` before its `Main Session` key fallback. The PR remains a focused implementation candidate, but its old `listSessionsFromStore` patch shape needs to move to `buildGatewaySessionRow` or a shared helper on current main.

## Best Possible Solution

Keep this PR open as a focused implementation candidate. Rebase/adapt the fix onto current main by moving the main-session display guard into `buildGatewaySessionRow` or a small shared row-display helper, preserving explicit `entry.displayName`, recognizing `resolveAgentMainSessionKey` and `resolveMainSessionKey`, and adding regression coverage for default agent main, configured main keys, global main, stale `origin.label: "heartbeat"`, list responses, and session-change snapshots.
