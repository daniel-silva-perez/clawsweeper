## Summary

Keep open. Current main still does not fully resolve the reported multi-agent orchestration failure cluster, and there is an open same-author implementation PR for the config-write race. OAuth refresh races and some session/run serialization paths are partly mitigated on main, but concurrent `openclaw agents add` still uses a stale-base conflict model rather than a serialized re-read/merge transaction, and the detached/subagent completion concerns remain active in linked open work.

## Best Possible Solution

Keep this issue open. Review and either land or close #44997 for serialized config mutation, with regression coverage proving concurrent `openclaw agents add` calls all persist. Then track the remaining orchestration pieces explicitly: run/session queue semantics, local lock/failover boundaries, provider auth noise boundaries, CLI-owned child-process cancellation and visibility, and subagent completion recovery in coordination with #44925 and #45200.
