## Summary

Keep open. Current main has several mitigations, including OAuth refresh serialization and task ledger visibility, but it still does not resolve the end-to-end multi-agent orchestration instability. The provided GitHub context also shows an open same-author PR, #44997, using closing syntax for this issue while explicitly calling the fix partial, so the issue should stay open until that PR is reviewed and the remaining session-lock, fallback, cancellation, and detached-work gaps are handled.

## Best Possible Solution

Keep this issue open. First review and either land or close #44997 for serialized config mutation. Then split or link the remaining work into focused maintainer-trackable items for per-agent/session run serialization semantics, local session-lock errors aborting model fallback, runtime-backed cancellation for CLI-owned work, operator visibility for active detached work, and subagent completion recovery in coordination with #44925 and #45200.

## Evidence
