## Summary

Keep open. Current main has mitigations for parts of the report, including per-session run queueing, stale/dead session-lock recovery, and OAuth refresh serialization, but the core concurrent `agents add` path is still not a serialized read-modify-write operation and the end-to-end orchestration report remains broader than the shipped fixes. The provided discussion also shows an open same-author PR with closing syntax for this issue, explicitly marked partial, so the issue should not be closed before that PR is reviewed, landed, or closed.

## Best Possible Solution

Keep this issue open. Review and either land or close #44997 for serialized config mutation, then split the remaining orchestration work into narrower maintainer-trackable items covering per-agent/session run serialization semantics, local session-lock errors aborting inappropriate provider fallback, runtime-backed cancellation for CLI-owned child work, operator visibility for orphaned/detached work, and subagent completion recovery in coordination with #44925 and #45200.
