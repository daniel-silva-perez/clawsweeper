## Summary

Current main still has the old 32-character TUI long-token chunking behavior and does not implement this PR's 128-character threshold or dot/paren/bracket code-like token preservation. The PR should stay open for normal review rather than being closed as implemented or superseded.

## Best Possible Solution

Keep this PR open for normal maintainer review. The best path is to preserve genuinely copy-sensitive code-like fragments without weakening the narrow-terminal guard, add regression coverage for both preserved code-like fragments and overflow protection, and coordinate with #39565 if the narrower dotted-token fix becomes the canonical formatter repair.
