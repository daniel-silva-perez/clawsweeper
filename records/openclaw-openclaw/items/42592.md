## Summary

Current main still has partial installer npm diagnostics and broad retry behavior, but it does not implement the requested structured npm error classification, retryable-only retry policy, code-specific remediation hints, Windows parity, or explicit Docker config-read failure handling. This remains actionable in the OpenClaw installer/Docker setup surfaces rather than something to close as stale, duplicate, ClawHub, or implemented.

## Best Possible Solution

Keep this issue open and implement a focused installer reliability change in the installer-owned surfaces: add tested npm error-code extraction/classification, retry only known transient npm/network failures with a bounded budget, print actionable remediation for terminal npm errors, make Docker gateway.controlUi.allowedOrigins config-read failures explicit while preserving first-run defaults, and mirror reasonable behavior in scripts/install.ps1.
