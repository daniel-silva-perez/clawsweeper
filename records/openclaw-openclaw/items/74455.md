## Summary

Keep open. Current main still evaluates raw configured model strings in the security audit collectors, so an alias such as `gpt-prev` can be classified before it is resolved through `agents.defaults.models`. The existing model alias resolver is available in the agents model-selection layer, and the requested fix is narrow enough for a focused repair PR.

## What This Changes

The issue asks for `openclaw security audit --deep` to resolve configured model aliases through `agents.defaults.models` before applying model-tier hygiene checks, while preserving the original config source path in findings.

## Best Possible Solution

Add a shared audit model collection/resolution helper that builds the model alias index once, resolves every collected model ref through `resolveModelRefFromString()`, evaluates hygiene and parameter-size checks on the canonical provider/model id, and keeps the original source path for reporting. Cover alias-backed default and fallback models in focused security audit tests.

## Repair Work Prompt

Fix the security audit model alias false positive reported in openclaw/openclaw#74455. In `openclaw security audit --deep`, model hygiene and small-model checks must resolve configured aliases from `agents.defaults.models` before tier/parameter classification. Use the existing `buildModelAliasIndex()` and `resolveModelRefFromString()` APIs from the agents model-selection layer; do not invent a second alias parser. Preserve the original source path in finding details, such as `agents.defaults.model.fallbacks`, while evaluating the resolved canonical `provider/model` id. Handle both collectors currently duplicated in `src/security/audit-extra.sync.ts` and `src/security/audit-extra.summary.ts`; prefer a small shared helper if that avoids drift without breaking lazy/runtime boundaries. Add focused tests showing an alias like `gpt-prev` resolving to `openai-codex/gpt-5.4` does not emit `models.weak_tier`, and add a small-model alias test so `models.small_params` still fires when an alias resolves to a small model. Do not change runtime model-selection semantics, provider defaults, config migrations, plugin APIs, or unrelated audit checks. Add a single-line changelog fix entry with `Thanks @GitHoubi`.
