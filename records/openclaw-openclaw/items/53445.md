## Summary

Keep open. Current main still renders the Sessions Updated column as a relative timestamp and has no full date-time formatter, while the linked feature request remains open. The PR is useful but not merge-ready because its missing-timestamp path would drop the existing localized fallback in non-English Control UI locales.

## What This Changes

The PR adds a `formatFullDateTime` helper with tests and changes the Control UI Sessions Updated cell to show a local `YYYY-MM-DD HH:mm` timestamp with the relative time in the tooltip.

## Best Possible Solution

Rebase the contributor implementation onto current main, keep the change limited to a reusable UI date-time formatter plus the Sessions Updated cell, preserve the current localized missing-value fallback, retain relative time as the tooltip, and validate the focused formatter/Sessions tests plus a quick Control UI browser spot-check.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve the localized missing timestamp fallback:** `ui/src/ui/views/sessions.ts:432`
  - body: Current main renders missing Updated values through `t("common.na")`, and locale bundles translate that value. This PR calls `formatFullDateTime(row.updatedAt)` with the helper's default `"n/a"`, so rebasing/merging would regress non-English sessions to an English fallback whenever `updatedAt` is missing. Pass `t("common.na")` into the formatter or otherwise keep this path localized.
