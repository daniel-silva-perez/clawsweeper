## Summary

Keep open. Current main still lacks the requested full timestamp, and this open contributor PR remains the active implementation candidate, but it is currently dirty against main and would regress the localized missing timestamp fallback if rebased without adjustment.

## What This Changes

The PR branch adds a `formatFullDateTime` Control UI formatter with tests and changes the Sessions Updated cell to show local `YYYY-MM-DD HH:mm` text with the relative timestamp in the title tooltip.

## Best Possible Solution

Rebase the contributor branch onto current main, keep the change limited to a reusable UI date-time formatter plus the Sessions Updated cell, pass the localized `t("common.na")` fallback for missing timestamps, retain the relative-time tooltip, and verify with focused formatter/Sessions tests plus a quick Control UI browser check.

## Reproduction Assessment

Yes. Source inspection reproduces the current behavior: current main renders the Sessions Updated column through `formatRelativeTimestamp`, while the PR patch changes that call site to a full date formatter without the localized missing-value fallback.

## Solution Assessment

No, not as-is. The implementation direction is narrow and appropriate, but the best maintainable solution needs a rebase and a one-line localization fix before merge.

## Review Findings
