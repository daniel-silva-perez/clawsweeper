## What This Changes

The PR adds a Control UI full date-time formatter with tests and changes the Sessions Updated cell to show local `YYYY-MM-DD HH:mm` with the relative time preserved in the title tooltip.

## Best Possible Solution

Rebase the contributor branch onto current main, keep the feature limited to a reusable UI date-time formatter plus the Sessions Updated cell, pass the localized `t("common.na")` fallback for missing timestamps, retain the relative-time tooltip, and verify with focused formatter/Sessions tests plus a quick Control UI browser spot-check.

Yes. Current main can be verified from source: the Sessions renderer still uses `formatRelativeTimestamp(row.updatedAt)`, and the PR diff would route missing timestamps through `formatFullDateTime(row.updatedAt)` without the localized fallback.
