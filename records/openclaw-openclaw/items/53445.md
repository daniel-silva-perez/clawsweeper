## Summary

Keep open. Current main still renders the Sessions Updated column as a relative timestamp, so the linked feature request is not implemented yet, but the proposed patch should not merge as-is because it would replace the localized missing-value fallback with English `n/a`.

## What This Changes

The branch adds a `formatFullDateTime` UI utility with tests and changes the Control UI Sessions Updated column to show `YYYY-MM-DD HH:mm` with the relative timestamp in the cell title.

## Best Possible Solution

Rebase the contributor branch onto current main, keep the formatter and Sessions cell changes narrow, pass `t("common.na")` or an equivalent localized fallback through the full-date formatting path, retain the relative timestamp as the tooltip, and verify the focused formatter/Sessions tests plus one Control UI browser check.
