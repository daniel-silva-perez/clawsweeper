## Summary

Keep this PR open. Current main still renders the Sessions Updated column as relative time only, so the requested UI improvement is not implemented, but the proposed patch has a concrete localization regression for missing timestamps and needs normal maintainer review before merge.

## What This Changes

The PR adds a `formatFullDateTime` Control UI formatter with tests and changes the Sessions Updated cell to show `YYYY-MM-DD HH:mm` while retaining relative time in a tooltip.

## Best Possible Solution

Rebase the contributor branch onto current main, keep the implementation limited to a reusable UI date-time formatter plus the Sessions Updated cell, pass the localized `t("common.na")` fallback through the new formatting path, preserve the relative timestamp as a tooltip, and verify with focused formatter/Sessions tests plus a quick Control UI browser check.
