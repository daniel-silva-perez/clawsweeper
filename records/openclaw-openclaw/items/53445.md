## Summary

Keep open. Current main still renders the Sessions Updated column as a relative timestamp and has no full date-time formatter, so the PR addresses a real open UI request, but the patch should not merge as-is because it drops the localized missing-value fallback and is currently dirty against main.

## What This Changes

The PR adds a `formatFullDateTime` UI helper with tests and changes the Control UI Sessions Updated cell to show local `YYYY-MM-DD HH:mm` with the relative time preserved in a tooltip.

## Best Possible Solution

Rebase the contributor branch onto current main, keep the feature limited to a reusable UI date-time formatter plus the Sessions Updated cell, pass the existing localized `t("common.na")` fallback for missing timestamps, retain the relative timestamp in the tooltip, and add or adjust focused tests for the formatter and Sessions rendering path.
