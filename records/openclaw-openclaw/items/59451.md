## Summary

Keep this issue open. Current main still requires `node:sqlite` to construct the builtin memory index before `memory_search` can search `MEMORY.md` or `memory/*.md`; when that manager cannot open, the tool returns an unavailable payload rather than running a direct file/grep keyword fallback. The related #69199 work is scoped to the narrower error-message issue (#59457), not the fallback requested here.

## Best Possible Solution

Implement the fallback in the memory-core plugin path: when builtin SQLite manager creation fails specifically because `node:sqlite` is unavailable, scan the configured memory files (`MEMORY.md`, `memory/*.md`, and any intentionally supported memory paths if maintainers choose) with a small deterministic keyword scorer, return results marked as fallback keyword search, and add focused tests for the missing-sqlite path. Keep #69199 separate as the narrower error-message fix for #59457.
