## Summary

Keep this PR open. Current main does not already provide `before_assemble` or `after_assemble`, so it is not obsolete, but the current PR head still has blocking contract and security issues: transcript-bearing hooks bypass the existing prompt/conversation policy gates, the sequential message-transform contract does not actually feed accumulated messages to later handlers, and the observer hook does not receive the final prompt input it claims to observe. The force-push removed the earlier AICM source files, but left a stale lockfile importer behind.

## What This Changes

The PR adds `before_assemble` and `after_assemble` typed plugin hooks, wires them into the Pi embedded runner, re-exports their event/result types, and updates `pnpm-lock.yaml` with an `extensions/aicm` importer.

## Best Possible Solution

Define the assembly hook contract as a narrow, guarded plugin API: classify transcript observation and mutation under the existing policy gates or a new explicit gate, document ordering and payload authority, add SDK/docs/contract tests, pass transformed state through sequential handlers, expose the actual final prompt payload for observation, and remove unrelated lockfile drift.

## Reproduction Assessment

Yes. Static review is enough to reproduce the blocking defects: the PR diff adds the new hook names without policy classification, the current runner utility passes the same event to every modifying handler, and the `after_assemble` call passes only `activeSession.messages` while the model call uses separate prompt/system/image inputs.

## Solution Assessment

No. A guarded assembly hook may be a reasonable core direction, but this patch is not the best way to solve it because it opens raw transcript mutation/observation without the current safety gates and does not accurately implement the stated sequential and final-context semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Gate assembly hooks through hook policies:** `src/plugins/hook-types.ts:106-107`
  - body: The new hooks expose or mutate full conversation messages, but the diff only adds them to the generic hook-name list. Because they are not classified with the prompt-injection and conversation-access hooks, a non-bundled plugin can register raw transcript hooks without the current `allowPromptInjection` and `allowConversationAccess` gates.
  - confidence: 0.93
- **[P2] Pass accumulated messages to each assembly handler:** `src/plugins/hooks.ts:1321-1324`
  - body: `runBeforeAssemble` promises chained transformations, but `runModifyingHook` calls every handler with the original event. A later plugin therefore sees the unmodified message list and can undo an earlier plugin's deletion or reorder when it returns its own `messages`.
  - confidence: 0.9
- **[P2] Expose the actual assembled prompt to after_assemble:** `src/agents/pi-embedded-runner/run/attempt.ts:2914`
  - body: The `after_assemble` payload passes only `activeSession.messages`, while the request sent to the model also includes `promptSubmission.prompt`, system/runtime context, and optional images. Hook users will not observe the final context the PR says is about to be sent to the LLM.
  - confidence: 0.84
- **[P3] Remove the stale AICM lockfile importer:** `pnpm-lock.yaml:234`
  - body: The force-pushed PR no longer contains an `extensions/aicm` package, but the lockfile still adds `extensions/aicm: {}` as a workspace importer. That leaves package metadata inconsistent with the workspace package set and should be removed from this PR.
  - confidence: 0.8
