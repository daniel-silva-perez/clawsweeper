## Summary

Keep open. Current main does not implement the requested assembly hooks or AICM plugin, so this PR is still live work, but the proposed branch is not merge-ready because it introduces ungated transcript-bearing hook access and a bundled transcript editor with active-session scoping problems.

## What This Changes

The PR adds `before_assemble` and `after_assemble` plugin hooks, wires them into the Pi embedded runner, adds a bundled AICM transcript-management plugin, and updates plugin package metadata plus the lockfile.

## Best Possible Solution

Split the work into a narrow, documented assembly-hook contract with explicit policy classification, ordering semantics, SDK/config/docs updates, and contract tests. Review AICM separately as an external or maintainer-owned plugin that uses scoped session/transcript APIs instead of direct global transcript discovery and mutation.

## Reproduction Assessment

Yes. Static review is enough for the blocking PR defects: the branch diff shows transcript-bearing hook names omitted from the policy sets, and the AICM source shows global transcript discovery plus shared `current` deletion state.

## Solution Assessment

No. A guarded assembly hook may be a reasonable core direction, but this patch is not the narrowest maintainable solution because it bundles the public API, runner integration, and a transcript-editing plugin before the policy and scoping model is settled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Gate assembly hooks through hook policies:** `src/plugins/hook-types.ts:106-107`
  - body: The new `before_assemble` and `after_assemble` hooks expose or mutate full conversation messages, but they are only added to `PluginHookName`. Because they are not also classified in the prompt-injection and conversation-access hook sets, non-bundled plugins can register raw transcript hooks without the current `allowPromptInjection` and `allowConversationAccess` gates.
  - confidence: 0.93
- **[P1] Scope AICM transcript access to the active session:** `extensions/aicm/index.ts:69`
  - body: AICM discovers transcripts from a fixed `agents/main/sessions` location, stores deletion state under a shared `current` key, and rewrites transcript files from agent-callable tools. In multi-agent or non-default installs, those tools can read or mutate a session other than the one invoking the tool.
  - confidence: 0.94
- **[P2] Pass accumulated messages into before_assemble handlers:** `src/plugins/hooks.ts:1321-1328`
  - body: `runBeforeAssemble` promises sequential message transformations, but `runModifyingHook` calls every handler with the original event. A later plugin can therefore return messages based on the unmodified list and undo an earlier plugin's deletion or reorder.
  - confidence: 0.91
- **[P3] Declare the AICM package as ESM:** `extensions/aicm/package.json:13`
  - body: The package exposes `dist/index.js` but does not set `type: module`, unlike the bundled plugin package convention. Since the TypeScript output uses ESM syntax, direct package entry loading can parse the built `.js` as CommonJS.
  - confidence: 0.79
- **[P3] Remove the stale broken source file:** `extensions/aicm/index.ts.broken:1`
  - body: The PR adds `index.ts.broken` with an older implementation that imports missing local modules. Even if TypeScript excludes it, keeping a broken source artifact in the bundled plugin tree makes package and review behavior unreliable.
  - confidence: 0.82
