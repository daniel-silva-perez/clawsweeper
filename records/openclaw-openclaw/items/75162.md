## Summary

Keep this PR open, but it is not merge-ready. Current main does not already implement the requested `before_assemble`/`after_assemble` surface, while static review of the PR head shows blocking security and correctness defects in the new public hook API and bundled AICM transcript plugin.

## What This Changes

The PR adds `before_assemble` and `after_assemble` plugin hook types/runners, wires them into the embedded runner, adds a bundled `extensions/aicm` transcript-management plugin, updates a plugin contract test, and adds package lock metadata.

## Best Possible Solution

Split the work into a narrow assembly-hook contract first: define policy classification, ordering, data authority, SDK/docs updates, and focused contract/runner tests. Treat AICM separately as a ClawHub plugin or an explicitly owned bundled plugin that uses active-session APIs and scoped persistent state instead of filesystem-wide transcript discovery.

## Reproduction Assessment

Yes. Static review is a high-confidence reproduction path for the PR defects: the PR head shows the unclassified hook names, the fixed transcript path and shared `current` state key, and the hook runner behavior that passes the original event to every modifying handler.

## Solution Assessment

No. A guarded assembly hook may be a reasonable direction, but this patch is not the best way to solve it because it bundles public API expansion with a transcript editor while missing policy gates, session scoping, docs, changelog, and focused tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Gate assembly hooks through hook policies:** `src/plugins/hook-types.ts:106-107`
  - body: The new hooks receive or mutate full conversation messages, but they are only added to the generic hook-name list. Because they are not also classified in the prompt-injection and conversation-access hook sets, non-bundled plugins can register raw transcript hooks without the existing safety gates.
  - confidence: 0.93
- **[P1] Scope AICM transcript access to the active session:** `extensions/aicm/index.ts:69`
  - body: AICM discovers transcripts from a fixed sessions location, chooses recent files, and stores deletion state under a shared `current` key. In multi-agent or non-default installs, agent-callable tools can read or mutate a session other than the one invoking the tool.
  - confidence: 0.94
- **[P2] Pass accumulated messages into each before_assemble handler:** `src/plugins/hooks.ts:1321-1328`
  - body: `runBeforeAssemble` promises sequential transformations, but `runModifyingHook` calls every handler with the original event. A later plugin can return messages based on the unmodified list and undo an earlier plugin's deletion or reorder.
  - confidence: 0.91
- **[P3] Declare the AICM package as ESM:** `extensions/aicm/package.json:13`
  - body: The package exposes `dist/index.js` but does not set `type: module`, unlike the bundled plugin package convention. Since the TypeScript output uses ESM syntax, direct package entry loading can parse the built file as CommonJS.
  - confidence: 0.8
- **[P3] Remove the stale broken source file:** `extensions/aicm/index.ts.broken:1-6`
  - body: The PR adds `index.ts.broken` with an older implementation that imports missing local modules. Even if TypeScript excludes it, keeping a broken source artifact in the bundled plugin tree makes package and review behavior unreliable.
  - confidence: 0.82
- **[P3] Add the required changelog entry:** `src/plugins/hook-types.ts:106-107`
  - body: This is a user-facing `feat` change that adds a public plugin API and a bundled plugin, but the PR does not update `CHANGELOG.md`. Add a single-line entry under the active version with appropriate contributor credit.
  - confidence: 0.88
