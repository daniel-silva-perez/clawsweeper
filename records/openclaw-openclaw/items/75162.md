## Summary

Keep this PR open for maintainer review, but do not merge it as-is. Current main does not implement the requested assembly hooks or AICM plugin, and the PR introduces security-sensitive transcript access plus an incomplete public hook policy contract.

## What This Changes

The PR adds before_assemble and after_assemble plugin hooks, wires them into the Pi embedded runner, adds a bundled AICM transcript-management plugin, and updates package metadata plus the lockfile.

## Best Possible Solution

Split the work into a narrow, documented assembly-hook contract with explicit policy classification, ordering semantics, SDK/docs updates, and contract tests. Review AICM separately as an external or maintainer-owned plugin that uses scoped session/transcript APIs instead of direct transcript discovery and mutation.

## Reproduction Assessment

Yes. Static review is enough for the blocking PR defects: the diff adds transcript-bearing hook names without adding them to the policy sets, and AICM's transcript path/session selection is visible in the added source.

## Solution Assessment

No. A guarded context-assembly hook may be a reasonable core direction, but this patch is not the narrowest maintainable solution because it bundles API design, runner behavior, and a security-sensitive transcript editor together.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Gate assembly hooks through hook policies:** `src/plugins/hook-types.ts:106-107`
  - body: The new before_assemble and after_assemble hooks receive or mutate full conversation messages, but the PR only adds them to PluginHookName. Because they are not also classified in the prompt-injection and conversation-access hook sets used by registry.ts, a non-bundled plugin can register raw transcript hooks without the current allowConversationAccess and allowPromptInjection gates.
  - confidence: 0.93
- **[P1] Scope AICM transcript access to the active session:** `extensions/aicm/index.ts:69`
  - body: AICM discovers transcripts from a fixed agents/main/sessions location, chooses recent JSONL files, stores deletion state under a shared current key, and rewrites transcript files from agent-callable tools. In multi-agent or non-default installs, those tools can read or mutate a session other than the one invoking the tool.
  - confidence: 0.94
- **[P2] Pass accumulated messages into each before_assemble handler:** `src/plugins/hooks.ts:1321-1328`
  - body: runBeforeAssemble promises sequential message transformations, but runModifyingHook calls every handler with the original event. A later plugin can therefore return messages based on the unmodified list and undo an earlier plugin's deletion or reorder.
  - confidence: 0.91
- **[P3] Declare the AICM package as ESM:** `extensions/aicm/package.json:13`
  - body: The package exposes dist/index.js but does not set type: module, unlike the bundled plugin package convention. Since the TypeScript output uses ESM syntax, direct package entry loading can parse the built .js as CommonJS.
  - confidence: 0.79
- **[P3] Remove the stale broken source file:** `extensions/aicm/index.ts.broken:1`
  - body: The PR adds index.ts.broken with an older implementation that imports missing local modules. Even if TypeScript excludes it, keeping a broken source artifact in the bundled plugin tree makes package and review behavior unreliable.
  - confidence: 0.82
