## Summary

Keep open. Current main does not implement the requested assemble hooks, but the PR is not merge-ready because it combines a public plugin lifecycle API with a bundled transcript-editing plugin that has a concrete session-scope security flaw and hook chaining semantics that do not match the stated design.

## What This Changes

The PR adds `before_assemble` and `after_assemble` plugin hook contracts and runner methods, wires them into the Pi embedded prompt path, and adds a bundled AICM plugin with package and lockfile entries.

## Best Possible Solution

Split the work: first design a narrow, typed, documented context-assembly hook with contract tests and clear ordering semantics, then review AICM separately as an external plugin or maintainer-owned bundled plugin that uses scoped session APIs instead of filesystem guessing.

## Reproduction Assessment

Yes. Static review gives a high-confidence path: current main lacks the requested hooks, while PR head shows the unsafe hard-coded session path and the hook runner invokes each modifying handler with the original event.

## Solution Assessment

No. A generic assembly hook may be a reasonable product direction, but this PR combines API design, Pi-only wiring, and a bundled transcript editor; a smaller core hook contract plus separate AICM security review is the safer path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Scope AICM file access to the active session:** `extensions/aicm/index.ts:29`
  - body: AICM discovers transcripts from a hard-coded `/workspace/projects/agents/main/sessions` directory and stores deletion state under a shared `current` key. Because its tools are agent-callable, a multi-agent or non-default install can read or rewrite the newest main session instead of the session invoking the tool. Use request-scoped session context and OpenClaw transcript APIs before adding this bundled plugin.
  - confidence: 0.93
- **[P2] Feed before_assemble results into the next handler:** `src/plugins/hooks.ts:1326-1328`
  - body: `runBeforeAssemble` promises sequential transformations, but `runModifyingHook` still calls every handler with the original event and only merges the latest returned `messages`. A later plugin can therefore reintroduce messages removed by an earlier plugin; pass the accumulated message list into the next handler or use a hook-specific chain runner.
  - confidence: 0.9
- **[P3] Declare the AICM package as ESM:** `extensions/aicm/package.json:13`
  - body: The package compiles ES modules to `dist/index.js` and exposes that file as `main`, but the package lacks `"type": "module"`. Existing bundled plugin packages and the plugin tutorial use ESM packages, so direct package entry loading can parse the built `.js` as CommonJS.
  - confidence: 0.75
- **[P3] Drop the broken source artifact:** `extensions/aicm/index.ts.broken:1`
  - body: The PR adds `index.ts.broken` with an older uncompiled implementation. It is not part of the extension tsconfig, but it would still live in the bundled plugin source/package tree and make review and packaging less reliable; remove the artifact before merging.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff introduces a security-sensitive bundled plugin that can read and rewrite transcript files selected from a hard-coded global path instead of the invoking session.

Concerns:

- **[high] Global transcript read/write path:** `extensions/aicm/index.ts:29`
  - body: AICM registers agent-callable tools but discovers transcripts from `/workspace/projects/agents/main/sessions`, selects the newest JSONL file, and stores soft deletes under a shared `current` key. That can expose or mutate another agent/session's history and bypass OpenClaw's normal session ownership model.
