## Summary

Keep open. Current main does not implement the requested assemble hooks or AICM plugin, and the PR head still has blocking API/security/correctness problems: the new raw transcript hooks bypass the existing hook policy gates, AICM reads and rewrites a hard-coded global transcript path, and `before_assemble` does not pass earlier transformations to later handlers. This needs maintainer product/security review and author revision, not conservative cleanup closure.

## What This Changes

The PR adds `before_assemble` and `after_assemble` plugin hook contracts and Pi embedded-runner wiring, plus a bundled AICM plugin with package and lockfile entries.

## Best Possible Solution

Split the work into a narrow, documented, typed context-assembly hook contract with policy classification, contract tests, and clear ordering semantics; review AICM separately as an external or maintainer-owned plugin that uses scoped session APIs instead of direct global transcript file discovery and mutation.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: at PR head, add a non-bundled plugin registering `before_assemble` or `after_assemble`; the names are accepted as hooks but absent from the hook policy classification arrays, so the existing conversation-access and prompt-injection gates do not apply. The AICM transcript scope issue is also directly visible from its hard-coded session directory and write path.

## Solution Assessment

No. A generic assembly hook may be a valid direction, but this PR combines public API design, Pi-only wiring, and a bundled transcript-editing plugin with unsafe filesystem behavior; a smaller guarded hook contract plus separate AICM review is the safer maintainable path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Gate assemble hooks through hook policies:** `src/plugins/hook-types.ts:106-107`
  - body: The new `before_assemble` and `after_assemble` hooks receive or mutate full conversation messages, but the PR only adds them to `PluginHookName`. Because they are not also classified in the prompt-injection and conversation-access hook sets used by `registry.ts`, a non-bundled plugin can register raw transcript hooks without `allowConversationAccess=true`, and `allowPromptInjection=false` will not block `before_assemble`.
  - confidence: 0.93
- **[P1] Scope AICM transcript access to the active session:** `extensions/aicm/index.ts:29`
  - body: AICM discovers transcripts from `/workspace/projects/agents/main/sessions`, chooses the newest JSONL file, stores deletion state under `current`, and rewrites that file from agent-callable tools. In multi-agent or non-default installs, those tools can read or mutate a session other than the one invoking the tool; use request-scoped session context and OpenClaw transcript APIs before bundling this plugin.
  - confidence: 0.94
- **[P2] Pass accumulated messages to each before_assemble hook:** `src/plugins/hooks.ts:1326-1328`
  - body: `runBeforeAssemble` promises sequential transformations, but it delegates to `runModifyingHook`, which calls every handler with the original event. A later plugin can therefore return messages based on the unmodified list and undo an earlier plugin's deletion or reorder; this hook needs a chain runner that feeds the accumulated message list into the next handler.
  - confidence: 0.91
- **[P3] Declare the AICM package as ESM:** `extensions/aicm/package.json:13`
  - body: The package exposes `dist/index.js` but does not set `"type": "module"`, unlike existing bundled plugin packages. Since the TypeScript output uses ESM syntax, direct package entry loading can parse the built `.js` as CommonJS.
  - confidence: 0.79
- **[P3] Remove the stale broken source file:** `extensions/aicm/index.ts.broken:1`
  - body: The PR adds `index.ts.broken` with an older implementation that imports missing local modules. Even if TypeScript excludes it, keeping a broken source artifact in the bundled plugin tree makes package/review behavior unreliable and should be removed before merge.
  - confidence: 0.82
