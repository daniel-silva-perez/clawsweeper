## Summary

Keep open. Current main does not implement these assemble hooks or the AICM plugin, but the proposed patch is not merge-ready because it introduces a public hook contract and a large bundled transcript-mutation plugin with concrete correctness and security blockers.

## What This Changes

The PR adds `before_assemble` and `after_assemble` hook types and runner methods, wires them into the Pi embedded runner prompt path, and adds a bundled `aicm` plugin with transcript search/delete/restore tools, audit helpers, runtime files, and a lockfile importer entry.

## Best Possible Solution

Split the work: first design a narrow, typed, documented context-assembly hook with contract tests and clear ordering semantics, then review AICM separately as an external ClawHub/plugin package or a maintainer-owned bundled plugin that uses scoped session APIs instead of filesystem guessing.

## Reproduction Assessment

Yes. Static review gives a high-confidence reproduction path: current main has no assemble hooks, while the latest PR head still shows the unsafe hard-coded session path and the hook runner still invokes each modifying handler with the original event.

## Solution Assessment

No. A generic assembly hook may be a reasonable product direction, but this patch combines public API design, Pi-only wiring, and a security-sensitive bundled transcript editor; the safer path is a smaller core hook contract first and a separate AICM review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Scope AICM file access to the active session:** `extensions/aicm/index.ts:29`
  - body: AICM discovers transcripts from a hard-coded `/workspace/projects/agents/main/sessions` directory and stores deletion state under a shared `current` key. Because its tools are callable by the agent, a multi-agent or non-default install can read or rewrite the most recently modified main session instead of the session invoking the tool. Use request-scoped session context and OpenClaw transcript APIs before adding this bundled plugin.
  - confidence: 0.93
- **[P2] Feed before_assemble results into the next handler:** `src/plugins/hooks.ts:1326-1328`
  - body: The new `runBeforeAssemble` merge only remembers the latest returned `messages`, but `runModifyingHook` still calls every handler with the original event. If one plugin removes messages and a later plugin reorders based on the original list, the later result can reintroduce the removed messages, so the promised sequential chaining is not true.
  - confidence: 0.9
- **[P3] Declare the AICM package as ESM:** `extensions/aicm/package.json:13`
  - body: The package compiles ES2022 modules to `dist/index.js` and exposes that file as `main`, but its `package.json` does not declare `"type": "module"`. Existing bundled plugin packages and the plugin tutorial use ESM packages, so direct package entry loading can parse the built `.js` as CommonJS.
  - confidence: 0.74

## Security Review

Status: needs_attention

Summary: The diff introduces a security-sensitive bundled plugin that can read and rewrite transcript files selected from a hard-coded global path instead of the current session context.

Concerns:

- **[high] Global transcript read/write path:** `extensions/aicm/index.ts:29`
  - body: AICM registers agent-callable tools but discovers transcripts from `/workspace/projects/agents/main/sessions`, selects the newest JSONL file, and stores soft deletes under a shared `current` key. That can expose or mutate another agent/session's history and bypass OpenClaw's normal session ownership model.
