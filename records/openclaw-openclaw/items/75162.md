## Summary

Keep open. Current main does not implement the requested assembly hooks or AICM plugin, and the PR is not ready to merge because it adds a public transcript-mutation hook surface plus a bundled transcript editor without the existing hook policy gates or active-session scoping.

## What This Changes

The PR adds `before_assemble` and `after_assemble` plugin hooks, wires them into the Pi embedded runner, adds an `aicm` bundled plugin, and updates plugin contract tests plus package metadata.

## Best Possible Solution

Split this into a narrow assembly-hook contract first: classify the hooks under explicit policy gates, document ordering and data authority, add SDK/docs/contract tests, and prove the runner integration. Treat AICM separately as an external or explicitly owned plugin that uses scoped session APIs instead of filesystem-wide transcript discovery and shared state keys.

## Reproduction Assessment

Yes. Static review is enough for the blocking PR defects: the diff adds transcript-bearing hook names without policy classification, and the AICM transcript path/state selection is visible in the added source.

## Solution Assessment

No. A guarded assembly hook may be a reasonable core direction, but this patch is not the narrowest maintainable solution because it bundles API design, runner behavior, and a transcript editor while missing policy and scoping safeguards.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Gate assembly hooks through hook policies:** `src/plugins/hook-types.ts:106-107`
  - body: The new hooks receive or mutate full conversation messages, but the diff only adds them to the generic hook-name list. Because they are not also classified in the prompt-injection and conversation-access hook sets, non-bundled plugins can register raw transcript hooks without the current safety gates.
  - confidence: 0.93
- **[P1] Scope AICM transcript access to the active session:** `extensions/aicm/index.ts:69`
  - body: AICM discovers transcripts from a fixed sessions location, chooses recent files, and stores deletion state under a shared `current` key. In multi-agent or non-default installs, agent-callable tools can read or mutate a session other than the one invoking the tool.
  - confidence: 0.94
- **[P2] Pass accumulated messages into each before_assemble handler:** `src/plugins/hooks.ts:1321-1328`
  - body: `runBeforeAssemble` promises sequential transformations, but `runModifyingHook` calls every handler with the original event. A later plugin can therefore return messages based on the unmodified list and undo an earlier plugin's deletion or reorder.
  - confidence: 0.91
- **[P3] Declare the AICM package as ESM:** `extensions/aicm/package.json:13`
  - body: The package exposes `dist/index.js` but does not set `type: module`, unlike the bundled plugin package convention. Since the TypeScript output uses ESM syntax, direct package entry loading can parse the built file as CommonJS.
  - confidence: 0.79
- **[P3] Remove the stale broken source file:** `extensions/aicm/index.ts.broken:1`
  - body: The PR adds `index.ts.broken` with an older implementation that imports missing local modules. Even if TypeScript excludes it, keeping a broken source artifact in the bundled plugin tree makes package and review behavior unreliable.
  - confidence: 0.82
