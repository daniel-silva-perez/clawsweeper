## Summary

Keep open. Current main has adjacent update-safety, doctor repair, plugin compatibility, and post-update plugin sync paths, but it still does not implement the requested native pre/post-update compatibility scanner, `doctor --upgrade-scan`, risk-scored JSON/human report, smoke recommendations, or rollback recommendation hints. Related PR #54354 is still open and plugin-specific, and related issue #14526 tracks a broader safe-update/rollback workflow rather than superseding this scanner/report contract.

## Best Possible Solution

Keep this open as a core update/doctor safety enhancement. The best path is a maintainer-scoped `openclaw update` and/or `openclaw doctor --upgrade-scan --json` report contract that reuses existing config validation/migrations, doctor health contributions, plugin compatibility diagnostics, update dry-run, plugin sync/integrity drift, health/status checks, and documented rollback guidance. Coordinate with #14526 for backup/rollback policy and #54354 for plugin compatibility, but do not close this until the scanner/report contract is implemented and documented.
