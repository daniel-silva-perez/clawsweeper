## Summary

Keep open. Current main has adjacent update dry-run JSON, post-update doctor repair, plugin sync/integrity-drift reporting, backup/rollback docs, and plugin compatibility notices, but it still does not implement a native pre/post-update compatibility scanner, `doctor --upgrade-scan`, risk-scored JSON/human report, smoke recommendations, or rollback recommendation hints.

## Best Possible Solution

Keep this open as a core update/doctor safety enhancement. The best path is a maintainer-designed `openclaw update` and/or `openclaw doctor --upgrade-scan --json` report contract that reuses existing config validation/migrations, doctor health contributions, plugin compatibility diagnostics, update dry-run planning, plugin sync/integrity-drift checks, health/status probes, backup support, and documented rollback guidance. Coordinate with #14526 for backup/rollback policy and #54354 for plugin compatibility, but do not close this until the scanner/report contract is implemented and documented.
