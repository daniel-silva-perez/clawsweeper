## Summary

Keep open. Current main still does not implement a configurable Gemini web-search base URL: the Google plugin manifest exposes only `webSearch.apiKey` and `webSearch.model`, the runtime still builds requests from `DEFAULT_GOOGLE_API_BASE_URL`, and the Gemini search docs only document `apiKey` and `model`. The PR remains a plausible implementation candidate for a real provider-owned gap, though it needs rebase/review against the current split runtime/shared files and the existing bot comments about normalization and runtime type safety.

## Best Possible Solution

Keep this PR open for normal review/rebase. The best path is to implement `plugins.entries.google.config.webSearch.baseUrl` in the current Google plugin manifest and shared/runtime Gemini web-search files, normalize it with the existing Google base URL helper, include it in the cache key, add focused unit tests and docs, and coordinate with the related open Gemini base URL/provider-config work rather than closing this implementation candidate.
