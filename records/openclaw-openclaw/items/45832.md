## Summary

Keep this PR open. Current main still follows symlinked WhatsApp `creds.json` in the active pluginized implementation, while this PR targets now-removed `src/web/*` paths. The change remains useful, but it needs a current-main port into `extensions/whatsapp`.

## Best Possible Solution

Keep this PR open and ask for a current-main port. The implementation should live in `extensions/whatsapp/src/{creds-files.ts,auth-store.ts,session.ts}`: add a shared credential-file read/inspect helper that rejects symlinked `creds.json` and `creds.json.bak` before all read/status paths, harden auth directory creation with `0o700` plus best-effort chmod for existing dirs, preserve the current atomic save flow, and add focused regression tests for primary creds, backup restore, identity/status/age reads, and directory mode behavior.
