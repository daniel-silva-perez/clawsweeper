## Summary

Keep this PR open. Current main still does not implement the intended WhatsApp credential-file symlink rejection or auth-directory mode hardening, and the PR’s useful hardening now needs to be ported from stale `src/web/*` paths to the current `extensions/whatsapp` plugin code.

## Best Possible Solution

Keep this PR open and ask for a current-main port. The implementation should live in `extensions/whatsapp/src/{creds-files.ts,auth-store.ts,session.ts}`: add a shared credential-file inspection/read helper that rejects symlinked `creds.json` and `creds.json.bak` before all read/status paths, harden auth directory creation with `0o700` plus best-effort chmod for existing directories, preserve the current atomic save flow, and add focused regression tests for primary creds, backup restore, identity/status/age reads, and directory mode behavior.
