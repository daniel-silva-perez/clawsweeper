## Summary

Current main does not implement this PR's central lock-loss protection. The session lock watchdog still force-releases held locks without notifying the active holder, the public lock API has no `onForceRelease` or lock-held query, and the embedded attempt path opens a guarded `SessionManager` without any write guard for lost lock ownership. The PR remains a plausible core agents fix, but it is open, unmerged, dirty against main, and needs maintainer review/rebase rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review and rebase. The best core direction is to land this fix or an equivalent implementation in the agents/session owner boundary: notify the active holder before watchdog force-release, abort the active run and session on lock loss, reject later transcript writes after ownership loss, cover rewrite paths, and add focused regression tests for watchdog callbacks, lock-held checks, lost-lock write rejection, cleanup ordering, and active-session abort behavior. The linked keepmyclaw backup discussion is useful operational mitigation, but it is not a substitute for fixing the core transcript race.
