## Summary

Current main still does not implement this PR's central lock-loss protection. The session watchdog force-releases held locks without notifying the holder, the public lock API has no onForceRelease or lock-held query, and the embedded attempt path opens a guarded SessionManager without a lost-lock write guard. This PR remains plausible core agents/session work, but it is open, unmerged, dirty against main, and should be reviewed/rebased rather than closed.

## Best Possible Solution

Keep this PR open for maintainer review and rebase, or land an equivalent fix in the agents/session owner boundary. The core fix should notify the active holder before watchdog force-release, abort the active run and session on lock loss, reject later transcript writes after ownership loss, cover rewrite paths, and add focused regression tests for watchdog callbacks, lock-held checks, lost-lock write rejection, cleanup ordering, and active-session abort behavior. The linked keepmyclaw backup discussion is useful operational mitigation, but it is not a substitute for fixing the core transcript race.
