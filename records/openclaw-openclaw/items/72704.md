## Summary

Keep open: current main still prepends full fenced Conversation info and Sender JSON blocks to Telegram-style direct user messages, and the linked closing PR remains open and unmerged, so cleanup closure would be premature.

## What This Changes

The issue requests reducing Telegram inbound prompt noise by moving per-message metadata out of full inline JSON blocks or rendering it in a compact form before the actual user text.

## Best Possible Solution

Review and land PR #72749, or an equivalent narrow prompt-builder/config change, so direct Telegram-style chats can opt into compact per-message metadata while preserving full untrusted context for groups, replies, forwards, locations, and richer history surfaces.

## Reproduction Assessment

Yes. A direct Telegram message context with ChatType direct, OriginatingChannel telegram, OriginatingTo, MessageSid, and sender fields flows into buildInboundUserContextPrefix, which currently renders the full fenced Conversation info and Sender blocks before the user body.

## Solution Assessment

Yes. Keeping this issue open while PR #72749 is reviewed is the right workflow; the current main behavior has not implemented the requested compact mode, and closing before merge would hide remaining work.
