## Summary

Keep open. Current main still prepends full fenced Conversation info and Sender JSON metadata blocks to direct Telegram/external-channel user messages, and the linked closing PR #72749 remains the active unmerged implementation candidate, so cleanup closure would be premature.

## What This Changes

The issue requests reducing or moving Telegram inbound user-message metadata out of bulky inline JSON blocks so short user text is easier for models to process.

## Best Possible Solution

Review and land #72749, or an equivalent narrow config-backed prompt-builder change, so direct Telegram-style chats can opt into compact metadata while preserving full untrusted context for groups, replies, forwards, locations, and richer history surfaces.

## Reproduction Assessment

Yes. A Telegram direct-message context sets `ChatType: "direct"`, `OriginatingChannel: "telegram"`, message id, and sender fields; current prompt assembly then prepends the full fenced Conversation info and Sender JSON blocks ahead of the user body.

## Solution Assessment

Yes. Keeping this issue open while the paired closing PR is reviewed is the right handling; current main has not implemented the requested compact mode, and closing before merge would hide remaining work.
