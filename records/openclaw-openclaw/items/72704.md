## Summary

Keep open. Current main still prepends direct external-channel user messages with full fenced JSON Conversation info and Sender blocks, and the linked PR #72749 is open/unmerged with closing syntax for this issue, so the issue should remain open until that implementation is reviewed and landed or replaced.

## What This Changes

The issue asks OpenClaw to reduce or restructure verbose inline Telegram inbound metadata that is prepended to short user messages before model inference.

## Best Possible Solution

Review and land #72749, or an equivalent narrow prompt-builder/config change, so direct Telegram-style chats can use a compact metadata shape while preserving full untrusted context for groups, replies, forwards, locations, and history; include config schema/docs and sanitizer tests before closing this issue.

## Reproduction Assessment

Yes. A direct external-channel context with `ChatType: "direct"`, `OriginatingChannel: "telegram"`, and sender fields reaches `buildInboundUserContextPrefix`, and current source/tests show full Conversation info and Sender JSON blocks are prepended before the user body.

## Solution Assessment

Yes, keeping this issue open while the linked closing PR is reviewed is the best handling. The current-main implementation has not shipped the requested compact mode, and closing before #72749 merges would hide the remaining work.
