## Summary

Keep this issue open. Current main still leaves macOS launchd supervisor logs at fixed `gateway.log` and `gateway.err.log` paths with no OpenClaw-managed size cap or rotation, while the existing `logging.maxFileBytes` rotation applies to the structured JSONL logger surface. The prior ClawSweeper comment’s core finding remains accurate on current main.

## Best Possible Solution

Add bounded retention for macOS launchd supervisor logs, either by routing `StandardOutPath` / `StandardErrorPath` through an OpenClaw-managed rotating sink or by adding safe pre-start/restart rotation for `gateway.log` and `gateway.err.log`. Cover install and restart plist rewrite paths with targeted tests, document how supervisor-log retention differs from `logging.maxFileBytes`, and separately trace the heartbeat-timeout producer for rate-limiting or deduplication.
