## Summary

Keep this issue open. The current main branch still writes macOS launchd supervisor stderr directly to `gateway.err.log` with no OpenClaw-managed rotation, max-size cap, truncation, or heartbeat-line deduplication. The structured JSONL logger does rotate at `logging.maxFileBytes`, but that is a separate log surface from the reported supervisor stderr file.

## Best Possible Solution

Add bounded retention for macOS supervisor logs in the daemon layer: either route launchd `StandardOutPath` and `StandardErrorPath` through an OpenClaw-managed rotating sink, or perform safe pre-start/restart rotation for `gateway.log` and `gateway.err.log`. Cover install and restart plist rewrite paths with targeted tests, document that supervisor-log retention is separate from `logging.maxFileBytes`, and separately trace the heartbeat timeout producer for deduplication or rate limiting.
