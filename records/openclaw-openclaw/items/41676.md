## Summary

Keep this issue open. Current main still sends macOS launchd supervisor stderr directly to `gateway.err.log` without an OpenClaw-managed size cap, rotation, truncation, or deduplication path. The structured JSONL logger has `logging.maxFileBytes`, and current Discord code has some heartbeat reconnect hardening, but neither bounds the reported supervisor stderr file.

## Best Possible Solution

Add bounded retention for macOS supervisor logs in the daemon layer: either route launchd `StandardOutPath` and `StandardErrorPath` through an OpenClaw-managed rotating sink, or perform safe pre-start/restart rotation for `gateway.log` and `gateway.err.log`. Cover install and restart plist rewrite paths with targeted tests, document the distinction from `logging.maxFileBytes`, and separately trace the heartbeat timeout producer for deduplication or rate limiting.
