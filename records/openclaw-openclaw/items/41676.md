## Summary

Keep this issue open. Current main still has bounded rotation for the structured JSONL logger, but the reported `gateway.err.log` path is a separate macOS launchd supervisor stderr file wired directly through `StandardErrorPath`; I did not find OpenClaw-managed size capping, rotation, truncation, or heartbeat-line deduplication for that surface on current main `891c7d9f1c1f5e0f52935fadfeacf695b0ef1eb8`.

## Best Possible Solution

Add bounded retention for macOS launchd supervisor logs, either by routing `StandardOutPath` / `StandardErrorPath` through an OpenClaw-managed rotating sink or by adding safe pre-start/restart rotation for `gateway.log` and `gateway.err.log`. Cover install and restart plist rewrite paths with targeted tests, document that supervisor-log retention is separate from `logging.maxFileBytes`, and separately trace the heartbeat-timeout producer for deduplication or rate limiting.
