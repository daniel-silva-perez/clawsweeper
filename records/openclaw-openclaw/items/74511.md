## Summary

Keep open. The reported resolver gap is real on current main and this PR is narrow, but the proposed inline-row suppression call is not ready because it omits the same `baseUrl` and `config` context that the existing resolver path already passes to manifest suppression.

## What This Changes

The PR adds a suppression check before resolving configured inline model rows with `api` and adds a unit test for `openai-codex/gpt-5.4-mini`.

## Best Possible Solution

Revise this PR so the inline `api` branch uses the same context-aware suppression call as the existing resolver path, then add a regression case that would fail without `baseUrl`/`config` context while keeping catalog visibility work coordinated with #73280.

## Review Findings
