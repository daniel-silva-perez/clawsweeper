## Summary

Keep open. The PR targets a real current-main bug, but the added contextless suppression call is not a reliable unconditional-only check under the documented manifest suppression contract, so the branch needs a small revision or explicit maintainer acceptance before merge.

## What This Changes

The PR adds an inline-model suppression check for configured `openai-codex/gpt-5.4-mini` rows with `api`, adds a regression test, and updates the changelog.

## Best Possible Solution

Keep the OpenAI Codex unconditional retirement behavior, but implement it through an explicit unconditional-suppression path or corrected condition matching semantics, then cover both the stale OpenAI inline row and a conditional provider-API-only suppression case.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Use an explicit unconditional suppression check:** `src/agents/pi-embedded-runner/model.ts:596-599`
  - body: The new check treats `shouldSuppressBuiltInModel({ provider, id })` as an unconditional-only test. That is not guaranteed by the manifest contract: a suppression can be conditional only on `when.providerConfigApiIn`, and the matcher skips that condition when `config` is omitted, so valid inline `api` rows could be rejected. Please add an explicit unconditional-suppression path, or make missing config fail `providerConfigApiIn`, and cover that case.
  - confidence: 0.83
