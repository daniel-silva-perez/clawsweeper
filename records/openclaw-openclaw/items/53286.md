## Summary

Keep this issue open. Current main at 17811480dae20f65b526454d75e971ae8424ec10 still gates `/status` and per-response `/usage full` cost display on API-key auth, while Bedrock resolves to `aws-sdk` and explicit model cost config is resolvable independently of auth. The same-author fix candidate #53303 is closed unmerged, and #29100 is adjacent auto-discovery pricing work rather than a duplicate for this explicit-cost display bug.

## Best Possible Solution

Implement a narrow display-layer fix: resolve model cost config independently of auth for `/status` and the response usage footer, show estimated USD for API-key/mixed auth or any auth mode with explicit usable pricing, and add regression coverage for `amazon-bedrock` using `auth: "aws-sdk"` with nonzero configured cost. Update the public token-use and Bedrock docs to explain explicit-pricing cost display for non-API-key providers.
