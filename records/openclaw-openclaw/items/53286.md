## Summary

Keep open. Current main still resolves estimated-cost display through auth-mode gates in both the status card and per-response usage footer, so an explicit Bedrock/aws-sdk model `cost` config is skipped before pricing is read. The previously linked fix PR is closed unmerged, and the related Bedrock discovery issue tracks a broader pricing-overlay feature rather than this explicit-cost display bug.

## What This Changes

Request to show estimated USD cost in `/status` and `/usage full` for AWS Bedrock/aws-sdk and other non-API-key providers when explicit model pricing is configured.

## Best Possible Solution

Implement a narrow display-layer fix that resolves explicit model cost config before auth filtering, shows USD estimates when usable nonzero pricing exists for Bedrock aws-sdk, preserves token-only output for missing or all-zero pricing, and updates focused tests plus token-use/API-cost docs and changelog.

## Reproduction Assessment

Yes. A static path is enough: configure `amazon-bedrock` with `auth: "aws-sdk"`, nonzero `models[].cost`, and response usage; `resolveModelAuthMode` returns `aws-sdk`, after which the status and usage footer gates keep `showCost` false before pricing is resolved.

## Solution Assessment

Yes. The maintainable fix is to decouple display eligibility from auth mode while still requiring usable pricing; the broader auto-discovery pricing overlay in #29100 should remain separate.
