## Summary

Keep open. Current main still gates estimated-cost display on auth mode in both `/status` and the per-response `/usage` footer, so Bedrock providers that resolve to `aws-sdk` skip explicit model pricing even though the pricing resolver itself is auth-independent. The linked fix PR is closed unmerged, and #29100 tracks a separate auto-discovery pricing overlay rather than this explicit-cost display bug.

## What This Changes

The issue requests that explicitly configured model cost metadata be used for AWS Bedrock `aws-sdk` providers in `/status` and `/usage full` instead of showing token counts only.

## Best Possible Solution

Implement a focused display-layer fix that resolves explicit model cost config before the auth check, shows USD estimates for Bedrock `aws-sdk` when usable nonzero pricing is configured, preserves token-only output for missing or zero pricing, and adds targeted status/usage regression coverage plus user-facing docs/changelog updates.
