## Summary

Keep this issue open. Current main still gates the reported `/status` and per-response `/usage full` cost display paths on API-key auth, while Bedrock resolves to `aws-sdk` and explicit model cost config is already resolvable independently of auth. The same-author fix PR #53303 is closed unmerged, and #29100 is adjacent auto-discovery pricing work rather than a duplicate for explicit-cost display.

## Best Possible Solution

Implement a narrow display-layer fix: resolve model cost config independently of auth for `/status` and the response usage footer, show estimated USD for API-key/mixed auth or for any auth mode with explicit usable pricing, and add regression coverage for an `amazon-bedrock` provider using `auth: "aws-sdk"` with nonzero configured cost. Update the public token-use and Bedrock docs to describe explicit-pricing cost display for non-API-key providers.
