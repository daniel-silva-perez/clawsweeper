## Summary

Keep open. Current main still suppresses estimated USD display before resolving explicit model pricing on both `/status` and the per-response usage footer, while Amazon Bedrock resolves to `aws-sdk`; the related fix PR is closed unmerged after maintainer review requested a focused split, and #29100 is adjacent discovery-pricing work rather than a replacement for this display bug.

## What This Changes

The issue requests using explicit `models.providers.*.models[].cost` pricing to show estimated USD usage regardless of provider auth mode, especially for Amazon Bedrock providers configured with `auth: "aws-sdk"`.

## Best Possible Solution

Implement a narrow display-layer fix that resolves model cost config independently of auth for `/status` and the response usage footer, shows USD estimates for API-key or mixed auth and for any auth mode with explicit usable pricing, keeps missing or zero placeholder pricing token-only, adds Bedrock `aws-sdk` regression coverage, and updates the token-use and Bedrock docs.
