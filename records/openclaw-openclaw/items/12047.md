## Summary

Keep open. Current main already downloads Telegram document attachments and can append text-like/PDF content through the shared media path, but the common Office-document part of the request is still missing: Office MIME types are excluded by the default file allowlist, non-JSON/XML `application/vnd.*` attachments are skipped as binary before extraction, and the bundled document extractor only advertises PDF. The related DOCX implementation PR is still open and unmerged, so this is not implemented on main yet.

## What This Changes

The issue requests extracting text from Telegram document attachments such as PDF, TXT, DOCX, Markdown, and similar files so the agent can read their contents instead of only seeing a media placeholder.

## Best Possible Solution

Finish this through the shared document extraction pipeline rather than Telegram-specific parsing: review or adapt #57612 for DOCX, decide the allowed Office MIME/dependency/resource policy, keep binary-safety and untrusted-content wrapping intact, and add focused Telegram plus shared media/input-file tests and docs for the supported formats.
