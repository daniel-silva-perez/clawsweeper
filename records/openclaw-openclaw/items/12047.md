## Summary

Keep open. Current main downloads Telegram document attachments and can extract text/JSON/PDF through the shared media path, but the requested Office-document part is still missing: Office MIME types are excluded or skipped before extraction, the bundled document extractor only advertises PDF, and the related DOCX implementation PR remains open and unmerged.

## What This Changes

Request to extract text from Telegram document attachments such as PDF, DOCX, TXT, and Markdown files and include that content in the agent context instead of only a media placeholder.

## Best Possible Solution

Finish the capability through the shared document extraction pipeline: add bounded Office MIME and extractor support in the document-extract/shared media path, keep Telegram as a download/context handoff, preserve untrusted-content wrapping and binary-safety checks, and cover Telegram plus shared media/input-file behavior with focused tests and docs.

## Reproduction Assessment

Yes. A DOCX/XLSX/PPTX Telegram document can be followed from Telegram download into MediaPath/MediaType and then into applyMediaUnderstanding, where application/vnd.* Office MIME types are skipped before extraction; the current XLSX regression test confirms that behavior.

## Solution Assessment

No for a Telegram-specific extractor. The narrow maintainable fix is a shared document-extraction capability, likely by reviewing or adapting #57612 with explicit parser, MIME-scope, decompression, and resource-limit decisions.
