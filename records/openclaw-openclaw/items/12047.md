## Summary

Keep this issue open. Current main still matches the prior review: Telegram downloads document attachments and passes MediaPath/MediaType into the shared media pipeline, and text-like files plus PDFs can be extracted, but Office-style document support remains missing. DOCX/XLSX/PPTX MIME types are not in the default file allowlist, application/vnd Office MIME types are skipped as binary before extraction, non-PDF document extractors are not reached from the shared input-file path, and the bundled document-extract plugin is still PDF-only.

## Best Possible Solution

Keep this open and consolidate it with the active Office document extraction work. The best path is shared and channel-independent: keep Telegram forwarding attachments through MediaPath/MediaType, extend safe file/media MIME handling so selected Office document MIME types can reach document extractors, generalize non-PDF document-extractor dispatch, add or adapt a bounded Office/DOCX extractor in the document-extract plugin, and cover Telegram attachments plus shared extraction/docs with focused tests.
