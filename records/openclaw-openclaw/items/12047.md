## Summary

Keep this issue open. Current main partially satisfies Telegram document attachment handling: Telegram downloads and forwards document media into the shared media pipeline, and shared extraction handles text-like files plus PDFs. The requested Office-style document support remains missing because default input-file MIME allowlists omit DOCX/XLSX/PPTX, the shared file pipeline skips application/vnd Office MIME types as binary, and the bundled document-extract plugin only registers a PDF extractor.

## Best Possible Solution

Keep this open and solve it in the shared document extraction path, not only Telegram. The likely direction is to let selected Office document MIME types reach bounded document extractors, generalize dispatch beyond the current PDF-only branch, add or adapt a safe DOCX/Office extractor in the bundled document-extract plugin or plugin API, and cover Telegram attachments plus shared extraction behavior with focused tests and docs.
