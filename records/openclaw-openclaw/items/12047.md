## Summary

Keep open. Current main downloads Telegram document media and can extract shared text/JSON/PDF content, but standard Office documents remain intentionally excluded from extraction, and the related DOCX implementation PR is still open for maintainer review.

## What This Changes

Request text extraction for Telegram document attachments so agents can read document contents instead of receiving only a media placeholder.

## Best Possible Solution

Finish this through the shared document extraction pipeline rather than a Telegram-only path: review or adapt the open DOCX PR, add bounded Office MIME and extractor support, keep untrusted-content wrapping and binary-safety checks, and cover Telegram handoff plus shared media/input-file behavior with focused tests and docs.

## Reproduction Assessment

Yes. A Telegram DOCX/XLSX/PPTX attachment can be traced from msg.document into MediaPath/MediaTypes and then into applyMediaUnderstanding, where Office MIME types are skipped before extraction; current regression tests confirm the skip behavior.

## Solution Assessment

No for a Telegram-specific extractor. The maintainable path is shared document-extraction support with explicit parser, MIME-scope, decompression/XML, and resource-limit decisions so other input-file and media paths get the same behavior safely.
