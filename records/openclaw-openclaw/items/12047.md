## Summary

Keep open. Current main has the Telegram download and shared text/JSON/PDF extraction pieces, but the requested Office-document path is still missing: Office MIME types are excluded or skipped before extraction, the bundled document extractor only advertises PDF, and the related DOCX PR is still open and unmerged.

## What This Changes

The issue asks for Telegram document attachments such as PDF, TXT, DOCX, and Markdown to have extracted text included in agent context instead of only a media placeholder.

## Best Possible Solution

Finish this through the shared document extraction pipeline, not Telegram-specific parsing: review or adapt #57612, add bounded Office MIME/extractor support in the document-extract/shared media path, preserve binary-safety checks and untrusted-content wrapping, and cover Telegram handoff plus shared media/input-file behavior with focused tests and docs.
