## Summary

Keep open. Current main downloads Telegram document attachments and can add text-like/PDF file content through the shared media path, but it still does not satisfy the full request for common document attachments: Office MIME types are not in the default file allowlist, non-JSON/XML `application/vnd.*` attachments are skipped as binary before extraction, and the bundled document extractor only advertises PDF support. The related DOCX implementation PR remains open, so this is not implemented on main yet.

## What This Changes

The issue requests extracting text from Telegram document attachments and adding that text to the agent message context instead of exposing only a media placeholder.

## Best Possible Solution

Finish the capability through the shared document extraction path: review or adapt #57612, add bounded Office MIME/extractor support after dependency and resource-limit choices are settled, preserve binary-safety checks and untrusted-content wrapping, and cover Telegram plus shared media/input-file behavior with focused tests and docs.
