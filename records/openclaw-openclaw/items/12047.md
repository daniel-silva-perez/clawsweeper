## Summary

Keep open. Current main already handles Telegram document download and shared text/PDF extraction, but the Office-document part of the request is still not implemented: Office MIME types are not in the default file allowlist, non-JSON/XML vendor MIME attachments are skipped as binary before extraction, and the bundled document extractor only registers PDF. The related DOCX implementation candidate remains open and unmerged.

## What This Changes

Request that Telegram document attachments such as DOCX, PDF, TXT, and Markdown have extractable text added to the agent message context instead of only a media placeholder.

## Best Possible Solution

Finish this through the shared document extraction pipeline rather than Telegram-specific parsing: review or adapt #57612, decide the allowed Office MIME/dependency/resource policy, preserve binary-safety checks and untrusted-content wrapping, and cover Telegram plus shared media/input-file behavior with focused tests and docs.
