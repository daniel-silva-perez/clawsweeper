## Summary

Keep open. Current main has the Telegram download and shared text/PDF extraction pieces, but the remaining Office-document part of the request is still not implemented: default file MIME support excludes Office formats, Office vendor MIME types are skipped as binary before extraction, and the bundled document extractor only advertises PDF support. The related DOCX PR is still an open implementation candidate, not a merged fix.

## What This Changes

The issue requests Telegram document attachments such as DOCX, PDF, TXT, and Markdown files to have extracted text included in the agent context instead of only exposing a media placeholder.

## Best Possible Solution

Finish the capability through the shared document extraction pipeline rather than Telegram-specific parsing: review or adapt #57612, add bounded Office MIME/extractor support after dependency and resource-limit policy is settled, preserve binary-safety checks and untrusted-content wrapping, and cover Telegram plus shared media/input-file behavior with focused tests and docs.
