## Summary

Keep open. Current main downloads Telegram document attachments and can append shared text/PDF extraction into agent context, but the remaining Office-document part of the request is still missing: Office MIME types are not in the default file allowlist, vendor Office attachments are skipped as binary before extraction, and the bundled document extractor only registers PDF. The related DOCX implementation candidate remains open and unmerged.

## What This Changes

Extract text from Telegram document attachments such as .docx, .pdf, .txt, and .md, and include that text in the agent context instead of only a media placeholder.

## Best Possible Solution

Finish this through the shared document extraction pipeline rather than Telegram-specific parsing: review or adapt #57612, add bounded Office MIME/extractor support after dependency and resource-limit policy is settled, preserve binary-safety checks and untrusted-content wrapping, and cover Telegram plus shared media/input-file behavior with focused tests and docs.
