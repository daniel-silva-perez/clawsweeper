## Summary

Keep open. Current main handles Telegram document download and shared text/JSON/PDF extraction, but the requested Office-document text extraction remains missing because Office MIME types are still excluded or skipped before the extractor path, and the related DOCX implementation PR remains open and unmerged.

## What This Changes

Request to extract text from Telegram document attachments such as .docx, .pdf, .txt, and .md files and include that text in the agent message context.

## Best Possible Solution

Finish this in the shared document extraction pipeline instead of Telegram-specific parsing: review or adapt #57612, add bounded Office MIME/extractor support behind the existing media limits and untrusted-content wrapping, preserve the binary-safety checks, and cover Telegram handoff plus shared media/input-file behavior with focused tests and docs.
