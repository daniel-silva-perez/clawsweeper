## Summary

Keep open. Current main handles Telegram document download and shared text/JSON/PDF extraction, but standard Office documents are still intentionally skipped as binary before extraction, and the related DOCX implementation PR remains open for maintainer review.

## What This Changes

Request to extract readable text from Telegram document attachments such as DOCX, PDF, TXT, and Markdown and include that text in the agent context.

## Best Possible Solution

Finish this through the shared document extraction pipeline rather than a Telegram-only path: review or adapt #57612, add bounded Office MIME and extractor support, preserve untrusted-content wrapping and binary-safety checks, and cover Telegram handoff plus shared media/input-file behavior with focused tests and docs.

## Reproduction Assessment

Yes. A Telegram DOCX/XLSX/PPTX document can be traced from msg.document into MediaPath/MediaTypes and then into applyMediaUnderstanding, where Office MIME types are skipped before extraction; current regression tests confirm the skip behavior.

## Solution Assessment

No for a Telegram-specific extractor. The narrow maintainable solution is shared document-extraction support with explicit parser, MIME-scope, decompression, XML, and resource-limit decisions.
