## Summary

Keep open. Current main has solved the Telegram download and shared text/PDF extraction parts, but the remaining Office-document capability is still missing: default file MIME support is text/JSON/PDF only, standard Office MIME types are skipped as binary before extraction, the bundled document extractor advertises only PDF, and the related DOCX implementation PR remains open/unmerged.

## What This Changes

The issue requests extracting text from Telegram document attachments and adding that text to the agent message context instead of exposing only a media placeholder.

## Best Possible Solution

Keep the issue open and finish the capability through the shared document extraction path: review or adapt #57612, add bounded Office MIME/extractor support only after dependency and resource-limit choices are settled, preserve binary-safety checks and untrusted-content wrapping, and cover Telegram plus shared media/input-file behavior with focused tests and docs.
