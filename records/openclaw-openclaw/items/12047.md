## Summary

Keep open. Current main downloads Telegram document media and forwards it into shared media context, but standard Office documents are still excluded before extraction; shared file extraction supports text/JSON/PDF, the bundled document extractor advertises only PDF, and the provided issue context shows #57612 is still the open related DOCX implementation candidate.

## What This Changes

Request to extract text from Telegram document attachments such as DOCX, PDF, TXT, and Markdown and include that content in agent context instead of only a media placeholder.

## Best Possible Solution

Finish this through the shared document-extraction pipeline rather than a Telegram-only path: review or adapt #57612, add bounded Office MIME/extractor support, keep untrusted-content wrapping and binary-safety checks, and cover Telegram handoff plus shared media/input-file behavior with focused tests and docs.

## Reproduction Assessment

Yes. A Telegram DOCX/XLSX/PPTX attachment can be traced from msg.document into MediaPath/MediaTypes, then into applyMediaUnderstanding where Office MIME types are skipped before extraction; current tests assert that skip behavior for XLSX and legacy Word/OLE files.

## Solution Assessment

No for a Telegram-specific extractor. The maintainable solution is shared document-extraction support with explicit parser, MIME-scope, decompression/XML, and resource-limit decisions so Telegram, input_file, and other media paths behave consistently.
