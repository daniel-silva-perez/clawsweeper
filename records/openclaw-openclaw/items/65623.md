## Summary

Current main still serializes raw `snapshot.parsed` for both full and path-scoped `/config show` chat replies, while the gateway `config.get` path already uses schema-aware redaction. The latest release is still affected, and #65637 is an open, unmerged closing PR for the same bug, so this should remain open for maintainer/security review and landing of that PR or an equivalent fix.

## What This Changes

The issue requests that `/config show` chat replies redact sensitive config values, including path-scoped lookups, instead of returning plaintext secrets from the parsed config.

## Best Possible Solution

Keep this open and review/land #65637, or replace it with an equivalent maintainer-approved patch that redacts the cloned parsed config before both full-config rendering and path-specific lookup in `handleConfigCommand`. Add regression coverage for full and path-scoped `/config show` secret redaction, then close only after the fix merges and release provenance is known.
