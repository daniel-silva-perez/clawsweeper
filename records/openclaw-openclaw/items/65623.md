## Summary

Current main still serializes raw `snapshot.parsed` for both full and path-scoped `/config show` chat replies, while the gateway `config.get` path already uses schema-aware redaction. The latest release is still affected, and PR #65637 is an open, unmerged closing PR for the same bug, so this issue should remain open for maintainer/security review and landing of that PR or an equivalent fix.

## Best Possible Solution

Keep this issue open and review/land PR #65637, or replace it with an equivalent maintainer-approved patch that redacts the cloned parsed config before both full-config rendering and path-specific lookup in `handleConfigCommand`. Add regression coverage for full and path-scoped `/config show` secret redaction, then close only after the fix merges and release provenance is known.
