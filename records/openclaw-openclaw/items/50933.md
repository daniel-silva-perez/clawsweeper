## Summary

Keep open. Current main still does not implement this PR's intended automatic reset hook behavior: daily/idle rollover does not emit internal `session:idle_reset` or `session:daily_reset` events, and the bundled `session-memory` hook is still registered and implemented only for manual `/new` and `/reset` command events.

## Best Possible Solution

Keep this PR open for maintainer follow-up. The best path is a focused implementation that emits an automatic reset signal early enough for `session-memory` to read the previous transcript, updates the bundled hook metadata/handler/docs/tests, and resolves the existing review concerns before merge. If maintainers choose a newer PR such as #61675 as canonical, close this PR only after that replacement is explicitly selected or merged.
