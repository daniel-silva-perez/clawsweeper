## Summary

Keep open. Current main still does not implement this PR's intended behavior: automatic daily/idle session rollover does not emit internal session reset hook events, and the bundled session-memory hook remains registered and implemented only for manual command:new/command:reset events. The PR is still potentially useful, but it needs maintainer review or a clearly selected superseding implementation before cleanup can close it.

## Best Possible Solution

Keep this PR open for maintainer follow-up. The best path is a focused implementation that emits an automatic reset signal early enough for session-memory to read the previous transcript, updates the bundled hook metadata/handler/docs/tests, and resolves the existing review concerns before merge. If maintainers choose a newer PR as canonical, close this only after that replacement is explicitly selected or merged.
