## Summary

Keep this PR open. Current main still treats Linux version-manager paths as expected daemon minimal PATH entries when those dirs exist or are configured through NVM_DIR/FNM_DIR/VOLTA_HOME/ASDF_DATA_DIR/BUN_INSTALL, so the PR's intended hardening is not implemented on main. The PR is a focused implementation candidate for the #71578 PATH-audit contradiction, though it likely needs rebase/adaptation to the newer current-main helper shape.

## Best Possible Solution

Keep the PR open for maintainer review or land an equivalent current-main patch: Linux minimal service PATH should keep stable user bins, PNPM/NPM prefix, Nix profiles, and system dirs while excluding Linux version-manager directories; macOS version-manager behavior should remain unchanged; service-env, service-audit, and public docs should be updated together for the hardened PATH policy.
