Keep open. ## What This Changes

The PR changes Linux daemon minimal service PATH assembly and tests to exclude nvm, fnm, Volta, asdf, and Bun roots while keeping stable user-bin, npm/pnpm, and Nix paths and leaving macOS version-manager behavior in scope.

## Best Possible Solution

Rebase or replace the PR with a narrow current-main patch that changes the Linux policy at the service-env source, preserves macOS behavior and current PATH-injection hardening, updates service-env and service-audit assertions for env-configured and existing version-manager roots, and records the user-facing fix in the changelog.

Yes. Yes for the technical direction, but not as the branch currently stands.
