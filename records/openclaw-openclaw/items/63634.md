## Summary

Keep this PR open. Current main still uses the strict MCP SDK listTools path, so the linked typeless inputSchema failure is not implemented on main. The PR remains a useful implementation candidate, but its raw client.request replacement still needs rework because MCP SDK 1.29.0 listTools caches output-schema validators and task-required metadata that callTool later consults.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rework bundle MCP discovery so typeless root object schemas are accepted, missing type is backfilled, explicit non-object root schemas are skipped with a warning, and MCP SDK 1.29.0 listTools metadata semantics are preserved or reimplemented for outputSchema validation and task-required tools. Add regression coverage for typeless discovery, malformed-root skipping, and a metadata-dependent callTool path before landing.
