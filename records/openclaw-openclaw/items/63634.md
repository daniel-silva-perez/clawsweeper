## Summary

Keep this PR open. Current main still uses the strict MCP SDK `client.listTools()` discovery path, so the typeless `inputSchema` failure described by the linked bug is not implemented on main. The PR head now addresses the earlier normalization and malformed-root review comments, but its raw `client.request()` replacement still needs rework because the pinned MCP SDK 1.29.0 `listTools()` path caches output-schema validators and task-required tool metadata that `callTool()` later consults.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rework bundle MCP discovery so typeless root object schemas are accepted, missing `type: "object"` is backfilled, explicit non-object root schemas are skipped with a warning, and SDK 1.29.0 `listTools()` metadata semantics are preserved or reimplemented for output-schema validation and task-required tools. Add regression coverage for typeless discovery, malformed-root skipping, and a metadata-dependent `callTool()` path before landing.
