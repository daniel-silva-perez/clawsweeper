## Summary

Keep open. Current main still does not implement `channels.signal.linkPreview`, Signal config/schema/docs/generated metadata do not expose it, and the Signal JSON-RPC send path still sends only message, styles, account, attachments, and target params. The related #29466 item is a separate closed WebChat UI preview-card request, not a Signal channel fix. The request is valid but needs maintainer product/security review before a repair PR because signal-cli configuration sync is primary-device-only and link-preview metadata overlaps the open prompt-injection concern in #22060.

## Best Possible Solution

Keep this open for maintainer-reviewed Signal plugin work: add `channels.signal.linkPreview` to the Signal config type, strict schema, generated metadata, docs, and tests; decide top-level versus per-account semantics and default behavior; define when `updateConfiguration` should run and how linked-device failures are handled; and pass only explicit safe Signal preview fields through JSON-RPC send unless a separate metadata-fetching security design resolves the #22060 concern.
