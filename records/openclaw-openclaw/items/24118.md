## Summary

Keep open. Current main still lacks a `channels.signal.linkPreview` config contract and still does not pass Signal link-preview fields or `updateConfiguration` through the Signal JSON-RPC send path. The closed WebChat preview-card issue #29466 is a different UI surface and does not supersede this Signal channel enhancement.

## Best Possible Solution

Keep this issue open until the Signal plugin gets a maintainer-approved design for `channels.signal.linkPreview`: add the config type/schema/generated metadata/docs and tests, resolve top-level versus per-account behavior, and then wire only the safe signal-cli JSON-RPC pieces. Maintainers should decide the `updateConfiguration` lifecycle/error policy and whether OpenClaw should ever fetch preview metadata itself before an implementation PR starts.
