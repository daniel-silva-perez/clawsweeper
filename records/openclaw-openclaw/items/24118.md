## Summary

Keep open. Current main still lacks `channels.signal.linkPreview` in the Signal config type, strict schema, generated channel metadata, docs, and outbound Signal JSON-RPC send path. The linked WebChat preview-card issue is a different surface and does not supersede this Signal channel enhancement.

## Best Possible Solution

Keep this issue open until the bundled Signal plugin adds a backward-compatible `channels.signal.linkPreview` setting across config types, strict schema, generated metadata, docs, and top-level/per-account resolution, then wires a cautious signal-cli integration path with focused schema/send-path tests. Maintainers should first decide how to handle primary-device-only `updateConfiguration`, default behavior, and whether OpenClaw should ever fetch preview metadata itself.
