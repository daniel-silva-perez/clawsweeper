## Risks / Open Questions

- `signal-cli updateConfiguration --link-previews` is documented as primary-device-only, so OpenClaw needs an explicit lifecycle and error policy for normal linked-device Signal setups before wiring it broadly. 
- The requested default of `true` could be a behavior change if OpenClaw actively syncs account configuration for existing users; maintainers should decide whether the config controls only pass-through fields, account sync, or both.
- If the implementation fetches link metadata itself instead of only passing caller-provided preview fields through, it introduces SSRF, privacy, and untrusted-content concerns that need a separate security design.
