## Summary

Keep open. The updated PR now adds the missing message-tool sanitization and a boot-run echo guard, but the guard samples the boot prompt every 20 characters and can miss unaligned 80-98 character literal echoes, leaving a markerless BOOT.md disclosure path reachable.

## What This Changes

The PR wraps BOOT.md content in internal-runtime-context delimiters, strips internal runtime context from message-tool text fields, adds a per-session BOOT.md echo guard, and adds changelog plus unit coverage.

## Best Possible Solution

Keep the PR open and revise the echo guard so its substantial-literal-echo detection actually covers every outbound substring at or above the chosen threshold, preferably with a sliding-window or fingerprint approach plus regression tests for unaligned 80-98 character slices.

## Reproduction Assessment

Yes. A high-confidence mock reproduction is to register a boot prompt and send a message-tool text field containing an unaligned 80-98 character contiguous slice of that prompt; the current PR loop can miss it because it only checks every twentieth start offset plus the tail.

## Solution Assessment

No. The PR is directionally right and addresses the earlier marker-only gap, but the sampled-window algorithm is not the narrowest reliable security fix; a complete sliding-window or hashed-substring guard would be safer.
