## Summary

Keep this PR open. Current main still lacks completed-run dedupe for retransmitted terminal Control UI chat events, so the intended UI fix is not obsolete. The PR branch also should not merge as-is because it bundles unrelated provider, Discord, sandbox, ops, and gaxios runtime changes, and the existing review discussion identifies gaps in the proposed dedupe path.

## Best Possible Solution

Keep this PR open for maintainer follow-up, but revise it into a minimal UI-only patch. Add typed completed-run dedupe state to the Control UI chat state, apply it to own-run `final`/`aborted` events and other-run/sub-agent final events, preserve the set across same-session history reloads, clear it on real session or agent switches, and add focused regression tests for retransmitted terminal chat events. Move the unrelated Google Vertex, Discord, sandbox, AGENTS, and gaxios changes to separate reviewed PRs or drop them from this branch.

## Risks / Open Questions

- Current main still appears vulnerable to duplicate Control UI assistant bubbles when the same renderable `final` event is delivered again after `chatRunId` is cleared.
- The PR branch mixes a narrow UI bug fix with unrelated provider auth, Discord, sandbox/config, ops, and runtime compatibility changes, increasing review and regression risk.
- The proposed `globalThis.window` shim is security-sensitive because it changes browser-environment detection for later Node dependencies and is unrelated to the stated UI duplicate-message fix.
- The proposed dedupe logic needs revision to cover own-run terminal events, other-run/sub-agent finals, and history-reload timing without silently no-oping on untyped state.
