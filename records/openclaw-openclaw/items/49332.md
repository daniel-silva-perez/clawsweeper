## Summary

Keep this PR open. Current main still lacks a completed-run dedupe guard for renderable terminal Control UI chat events, so the PR is not obsolete; however, the branch should not merge as-is because it bundles unrelated stale runtime/Discord/config/docs changes and introduces concrete correctness and security-review issues.

## What This Changes

The PR adds Control UI runId tracking for terminal chat-event deduplication and also includes unrelated changes to Discord monitoring, sandbox tool policy, Google Vertex/gaxios auth compatibility, config types, and AGENTS.md.

## Best Possible Solution

Land a narrow UI-only fix that records processed terminal runIds for the active session, covers own-run `final`/`aborted` and other-run/sub-agent finals, preserves the set through same-session history reloads, clears it only on real session/agent changes, and adds focused regression tests for replaying the same renderable final event.

## Reproduction Assessment

Yes. A high-confidence code-path reproduction is to call `handleChatEvent` with an own-run renderable `final`, observe `chatRunId` clear, then call it again with the same runId and payload; current main appends the assistant message a second time and the gateway path does not force history reload for renderable finals.

## Solution Assessment

No. The idea of runId-based terminal-event dedupe is the right direction, but this PR is not the best solution because it clears the dedupe set on history reload, misses the other-run/sub-agent final append path, and mixes the UI fix with unrelated runtime, Discord, sandbox, and repo-instruction changes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Remove the broken core Discord copies:** `src/discord/monitor/provider.lifecycle.ts:8`
  - body: The new `src/discord/monitor` files import siblings such as `../gateway-logging.js` and `./format.js` that are not added under `src/discord` on this branch; current main keeps those modules under `extensions/discord`. This unrelated addition will break module resolution/typecheck, so it should be removed from this UI dedupe PR or moved as a complete Discord-plugin change.
  - confidence: 0.94
- **[P2] Keep processed run IDs across history reloads:** `ui/src/ui/controllers/chat.ts:89-93`
  - body: Clearing `processedChatRunIds` in `loadChatHistory()` reopens the exact reconnect/jitter case this fix targets: terminal handling can mark a run processed, trigger a same-session history reload, then a retransmitted final with the same runId arrives after the set was cleared and appends again. Preserve the completed-run set for the active session and clear it only on real session/agent switches.
  - confidence: 0.9
- **[P2] Mark other-run finals before returning:** `ui/src/ui/controllers/chat.ts:273-280`
  - body: The new dedupe guard only works for runIds that were previously inserted, but the other-run/sub-agent final branch appends `finalMessage` and returns without recording `payload.runId`. A retransmitted sub-agent final therefore passes the new guard and duplicates the same appended message.
  - confidence: 0.88
- **[P2] Avoid creating a global window shim in Node:** `src/infra/gaxios-fetch-compat.ts:191-206`
  - body: This UI fix also adds a process-wide `globalThis.window` in a Node compatibility layer. That permanently changes `typeof window` for later dependencies and can send unrelated libraries down browser-only paths; the gaxios workaround should not mutate global browser-detection state, and it does not belong in this PR.
  - confidence: 0.87
