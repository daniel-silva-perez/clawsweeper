## Summary

Keep open. Current main still only has the aggregate `cron.store`/`jobs.json` definition file plus `jobs-state.json`, and the provided discussion identifies open PR #41206 as the active implementation candidate rather than merged behavior.

## What This Changes

The issue requests an additive directory-backed cron job store so each job can live in its own JSON file for NixOS/GitOps-style declarative management.

## Best Possible Solution

Land or rework #41206, or an equivalent additive implementation, so existing `cron.store` and `jobs.json` users remain compatible while a documented directory-store mode deterministically loads per-job JSON definitions, handles lifecycle operations safely, and updates schema/help, doctor, docs, and focused cron store/Gateway tests.
