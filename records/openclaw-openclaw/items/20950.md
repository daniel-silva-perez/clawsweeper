## Summary

Keep open. Current main still routes cron through a single `cron.store` path and aggregate `jobs.json` definition file, while open PR #41206 is the active unmerged implementation candidate for the directory-backed store.

## What This Changes

Request support for an opt-in directory-backed cron job store where each job lives in its own JSON file for NixOS/GitOps declarative setups.

## Best Possible Solution

Ship an additive, documented directory-store mode that keeps existing `cron.store` and `jobs.json` users compatible, deterministically loads per-job JSON definitions, safely handles lifecycle operations and path/symlink behavior, preserves runtime state in the sidecar model, and updates schema/help, doctor, docs, and focused cron store/Gateway tests.
