## Summary

Keep open. Current main still lacks the requested directory-backed cron definition store: cron config exposes only the aggregate `store` path, the schema rejects unknown cron keys, Gateway startup passes one store path into cron, and the docs still describe `jobs.json` plus `jobs-state.json`. The issue also has an open implementation candidate in #41206, so this should remain the canonical tracker until that PR or an equivalent replacement lands.

## What This Changes

The issue requests support for managing cron job definitions as one JSON file per job in a `jobs.d`-style directory for NixOS/GitOps workflows.

## Best Possible Solution

Ship an additive, documented directory-store mode for cron definitions that keeps existing `cron.store` and `jobs.json` users compatible, loads per-job JSON deterministically, preserves the runtime-state sidecar model, and updates config schema/help, doctor behavior, docs, and focused cron store/Gateway tests.
