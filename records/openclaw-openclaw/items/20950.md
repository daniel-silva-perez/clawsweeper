## Summary

Keep this issue open. Current main still supports a single aggregate cron definition file plus a derived runtime-state sidecar, while the requested directory-backed one-file-per-job store remains unimplemented and is already represented by open PR #41206 as the active implementation candidate.

## What This Changes

Add optional directory-backed cron job definitions, one JSON file per job, for declarative NixOS/GitOps management alongside or instead of `jobs.json`.

## Best Possible Solution

Keep this as the canonical tracker and land or rework #41206, or an equivalent additive implementation, so existing `cron.store` and `jobs.json` users remain compatible while a documented directory-store mode deterministically loads per-job JSON definitions, handles lifecycle operations safely, and updates schema/help, doctor, docs, and focused cron store/Gateway tests.
