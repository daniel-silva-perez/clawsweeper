## Summary

Keep open. Current main still has only the aggregate `jobs.json` definition store plus the `jobs-state.json` runtime sidecar, while #41206 is an open, unmerged implementation candidate for the requested directory-backed cron store.

## What This Changes

Add opt-in per-job cron JSON directory support for declarative NixOS/GitOps setups while preserving existing `jobs.json` behavior.

## Best Possible Solution

Keep this as the canonical tracker and ship an additive, documented directory-store mode that keeps existing `cron.store`/`jobs.json` users compatible, deterministically loads per-job JSON definitions, safely handles lifecycle and filesystem behavior, preserves the runtime-state sidecar model, and updates schema/help, doctor, docs, and focused cron store/Gateway tests.
