## Summary

Current main still does not implement directory-backed cron job definitions. Cron definitions are still loaded and saved through one aggregate `cron.store` path, defaulting to `cron/jobs.json`, with runtime state split into `jobs-state.json`. This issue should remain open as the canonical product request while open PR #41206 remains the implementation candidate; #53581 only solved the complementary runtime-state split and #70531 was closed as a duplicate into this request.

## Best Possible Solution

Keep this issue open as the canonical product request. The best path is to review and either land #41206 or replace it with an additive implementation that preserves existing `cron.store`/`jobs.json` compatibility, adds a documented directory-store option, deterministically loads per-job JSON files, keeps runtime state compatible with the current `jobs-state.json` sidecar model, and updates config schema/help/docs/doctor plus focused cron store and gateway tests.
