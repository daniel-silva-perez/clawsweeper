## Summary

Keep open. Current main still only supports the aggregate cron definitions file plus a runtime-state sidecar, and the requested per-job directory store is not implemented in config, schema, Gateway startup, store load/save logic, or docs. The discussion already identifies open PR #41206 as the active implementation candidate, so this should remain the tracker for maintainer review rather than being closed or queued as a separate ClawSweeper fix.

## What This Changes

Add support for a cron jobs.d-style directory where each JSON file defines one cron job for declarative NixOS/GitOps management.

## Best Possible Solution

Ship an additive, documented cron directory-store mode that keeps existing cron.store and jobs.json users compatible, deterministically loads per-job JSON definitions, preserves the current runtime-state sidecar model, and updates config schema/help, doctor behavior, docs, and focused cron store/Gateway tests.
