## Summary

Keep open. Current main has addressed much of the Twitch schema/setup/status mismatch, but the reported post-connect auto-restart loop still matches the source: Twitch connects, the monitor setup returns, and the gateway treats that clean return as a channel exit and schedules restart attempts.

## Best Possible Solution

Keep this issue open and fix the Twitch gateway lifecycle. `extensions/twitch/src/plugin.ts` should keep the passive monitor task alive until `abortSignal` fires, likely by using `runStoppablePassiveMonitor` plus a scoped status sink like IRC/Nextcloud Talk. Add a regression test proving Twitch `startAccount` does not resolve after a successful monitor setup and only stops on abort; then re-check status/probe behavior for env-token default accounts.
