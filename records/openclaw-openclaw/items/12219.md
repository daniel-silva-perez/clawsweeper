## Summary

Keep open. Current main still lacks the requested skill.yaml permission manifest standard, install-time permission consent, skills audit/trust grades, signing, and attestation, while the issue discussion and latest maintainer review frame the remaining work as security-sensitive product/API design rather than conservative cleanup.

## What This Changes

Request a standardized skill.yaml permission manifest for skills, with validation, user-visible install/audit review, trust scoring, signing, and attestation support.

## Best Possible Solution

Keep this open for explicit maintainer product/security design. The end state should be a versioned skill permission manifest schema with parser/types, ClawHub and direct install/update integration, visible consent and audit/trust reporting, documented signing/attestation expectations, and a clear relationship to before_install and before_tool_call enforcement hooks.
