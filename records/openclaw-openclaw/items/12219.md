## Summary

Keep open. Current main still has SKILL.md/frontmatter skills, ClawHub scan visibility, and generic install/tool-call hooks, but it does not implement the requested standardized skill.yaml permission manifest, install-time consent review, skills audit/trust grades, signing, or attestation; the recent maintainer discussion also explicitly kept this security/product design open.

## What This Changes

The issue requests a standardized skill.yaml permission manifest for skills, with install-time review, auditing/trust scoring, signing, and attestation support.

## Best Possible Solution

Keep this open for explicit maintainer product/security design. The desired end state is a versioned skill permission manifest schema with parser/types, ClawHub and direct install/update integration, visible consent and audit/trust reporting, documented signing/attestation expectations, and a clear relationship to before_install and before_tool_call enforcement hooks.
