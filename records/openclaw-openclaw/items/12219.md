## Summary

Keep open. Current main has SKILL.md/frontmatter skills, ClawHub scan visibility, and generic before_install/before_tool_call hooks, but it still does not implement the requested standardized skill.yaml permission manifest, install-time consent review, skills audit/trust grades, signing, or attestation. The discussion also has useful external references, including openclaw-secure, shell-security-ultimate, and globalcaos/tinkerclaw, and the latest maintainer comment explicitly kept this open for product/security follow-up.

## What This Changes

The issue requests a standard skill.yaml permission manifest for skills, plus install-time permission review, audit/trust reporting, signing, and attestation support.

## Best Possible Solution

Keep this open for explicit maintainer product/security design. The end state should be a versioned skill permission manifest schema with parser/types, ClawHub and direct install/update integration, visible consent and audit/trust reporting, documented signing/attestation expectations, and a clear relationship to before_install and before_tool_call enforcement hooks.
