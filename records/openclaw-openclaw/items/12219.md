## Summary

Keep open. Current main still implements SKILL.md/frontmatter-based skills, ClawHub scan visibility, and generic install/tool-call hooks, but it does not implement the requested skill.yaml permission manifest, install-time consent, skills audit/trust grades, signing, or attestation. Recent maintainer discussion also explicitly keeps this as product/security design work rather than cleanup closure.

## What This Changes

The issue requests a standardized skill.yaml permission manifest for skills, including declared network/filesystem/shell/credential access, validation, signing/attestation, trust scoring, and install/audit/ClawHub integration.

## Best Possible Solution

Keep this open for explicit maintainer product/security design. The desired end state is a versioned skill permission manifest schema with parser/types, docs, install/update consent, skills inspect/audit reporting, ClawHub display and policy integration, documented signing/attestation expectations, and a clear relationship to before_install and before_tool_call without breaking existing SKILL.md skills.
