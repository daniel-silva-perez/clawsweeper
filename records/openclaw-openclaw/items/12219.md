## Summary

Keep open. Current main still does not implement a `skill.yaml`/`skill.yml` skill permission manifest standard, install-time permission consent, `openclaw skills audit`, signing/attestation, or trust grading; the current discussion also includes explicit maintainer guidance to keep this as product/security design work.

## Best Possible Solution

Keep this open for explicit maintainer product/security design. The likely path is a versioned skill permission manifest with validation during ClawHub and direct skill install/update flows, visible install-time permission review for `openclaw skills install`, an audit/trust report or equivalent, and integration with existing `before_install` and `before_tool_call` enforcement hooks. Use `@ellistevo/openclaw-secure`, ClawHub `shell-security-ultimate`, and `globalcaos/tinkerclaw` as references while coordinating with #10827, #12678, and #50090.
