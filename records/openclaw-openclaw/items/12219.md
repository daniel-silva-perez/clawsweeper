## Summary

Keep open. Current main still lacks a standardized `skill.yaml`/`skill.yml` permission manifest, install-time consent prompt, `openclaw skills audit`, signing/attestation, and trust grading. Existing ClawHub scans, `SKILL.md` metadata, dangerous-code install scans, and plugin hooks are useful adjacent pieces but do not implement the requested product/security contract.

## Best Possible Solution

Keep this open for explicit maintainer product/security design. The likely path is a versioned skill permission manifest with validation during ClawHub and direct skill install/update flows, visible install-time permission review for `openclaw skills install`, an audit/trust report or equivalent, and integration with existing `before_install` and `before_tool_call` enforcement hooks. Use `@ellistevo/openclaw-secure`, ClawHub `shell-security-ultimate`, and `globalcaos/tinkerclaw` as reference implementations while coordinating with #10827, #12678, and #50090.
