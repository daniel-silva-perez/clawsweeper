## Summary

Keep open. Current main still does not implement the requested standardized `skill.yaml`/`skill.yml` permission manifest, install-time permission review, `openclaw skills audit`, signing/attestation, or trust grading. The discussion also has a recent maintainer comment explicitly keeping this open for product/security follow-up, plus useful external reference implementations that should inform the design.

## Best Possible Solution

Keep this open for explicit maintainer product/security design. The best path is a versioned skill permission manifest with schema validation, ClawHub and direct-install/update integration, visible install-time permission review, an audit/trust report or equivalent, and a documented relationship to `before_install` and `before_tool_call` enforcement hooks. Coordinate with #10827, #12678, and #50090 while using @ellistevo/openclaw-secure, ClawHub shell-security-ultimate, and globalcaos/tinkerclaw as reference implementations.
