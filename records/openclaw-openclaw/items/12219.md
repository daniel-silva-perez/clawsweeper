## Summary

Keep open. Current main still uses SKILL.md/frontmatter skills, ClawHub scan visibility, and generic install/tool-call hooks, but it does not implement the requested standardized skill.yaml permission manifest, install-time consent, skills audit/trust grades, signing, or attestation; the latest maintainer discussion also kept this open for product/security follow-up.

## What This Changes

The issue proposes a standard skill.yaml permission manifest for skills, with validation, user review before install, auditing/trust scoring, signing, and attestation support.

## Best Possible Solution

Keep this open for explicit maintainer product/security design. The end state should be a versioned skill permission manifest schema with parser/types, ClawHub and direct install/update integration, visible consent and audit/trust reporting, documented signing/attestation expectations, and a clear relationship to before_install and before_tool_call enforcement hooks.
