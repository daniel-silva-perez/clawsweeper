## Summary

Keep open. Current main still exposes SKILL.md/frontmatter skills, ClawHub scan visibility, and generic plugin install/tool hooks, but it does not implement the requested standardized `skill.yaml` permission manifest, install-time consent review, `openclaw skills audit`, trust grades, signing, or attestation. The latest maintainer comment also explicitly kept this open for product/security follow-up with useful external references.

## What This Changes

Request a standardized `skill.yaml` permission manifest for skills, including permission declarations, install-time review, audit/trust scoring, signing, and attestation.

## Best Possible Solution

Keep this open for explicit maintainer product/security design. The end state should be a versioned skill permission manifest schema with parser/types, ClawHub and direct install/update integration, visible consent and audit/trust reporting, documented signing/attestation expectations, and a clear relationship to `before_install` and `before_tool_call` enforcement hooks.
