## Summary

Keep open. Current main still has SKILL.md/frontmatter-based skills, ClawHub scan visibility, gateway-backed installer scanning, and generic plugin hooks, but it does not implement the requested standardized skill.yaml permission manifest, install-time consent review, openclaw skills audit command, trust grades, signing, or attestation. The latest issue discussion also includes a maintainer comment explicitly keeping this open for product/security follow-up, with useful external references from @ellistevo/openclaw-secure, shell-security-ultimate, and globalcaos/tinkerclaw.

## What This Changes

The issue requests a standardized skill permission manifest (`skill.yaml`) with declared network/filesystem/shell/credential permissions, install-time review, audit/trust scoring, and signing/attestation support.

## Best Possible Solution

Keep this open for explicit maintainer product/security design. The best path is a versioned skill permission manifest with schema validation, ClawHub and direct-install/update integration, visible install-time permission review, an audit/trust report or equivalent, and a documented relationship to `before_install` and `before_tool_call` enforcement hooks. Coordinate with #10827, #12678, and #50090 while using @ellistevo/openclaw-secure, shell-security-ultimate, and globalcaos/tinkerclaw as reference implementations.
