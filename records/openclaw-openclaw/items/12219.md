## Summary

Keep open. Current main still documents and implements SKILL.md/frontmatter-based skills plus generic install/tool-call hooks, but it does not provide the requested skill.yaml permission manifest, install-time consent review, skills audit/trust grading, signing, or attestation. The issue discussion and related open RFCs also show this is still active product/security design work, not conservative cleanup.

## What This Changes

The issue requests a standardized skill.yaml permission manifest with declared network/filesystem/shell/credential access, validation, signing, attestation, trust scoring, install-time review, and ClawHub/audit integration.

## Best Possible Solution

Keep this open for explicit maintainer product/security design. The desired end state is a versioned skill permission manifest schema with parser/types, backward-compatible SKILL.md integration, install/update consent, skills inspect/audit reporting, ClawHub display and policy integration, documented signing/attestation expectations, and a clear relationship to before_install and before_tool_call enforcement hooks.
