## Summary

Keep open. Current main has SKILL.md/frontmatter skills, ClawHub scan visibility, and generic install/tool-call hooks, but it still does not implement the requested standardized skill.yaml permission manifest, install-time consent review, skills audit command, trust grades, signing, or attestation. The latest maintainer discussion also explicitly kept this open for product/security follow-up.

## What This Changes

Requests a versioned skill.yaml manifest standard for skills to declare network, filesystem, shell, and credential permissions, with install-time review plus optional audit, signing, trust scoring, and attestation integration.

## Best Possible Solution

Keep this open for explicit maintainer product/security design. The end state should be a versioned skill permission manifest schema with parser/types, ClawHub and direct install/update integration, visible consent and audit/trust reporting, documented signing/attestation expectations, and a clear relationship to before_install and before_tool_call enforcement hooks.
