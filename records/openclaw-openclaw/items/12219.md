## Summary

Keep open. Current main still exposes SKILL.md/frontmatter skills, ClawHub scan visibility, and generic before_install/before_tool_call hooks, but it does not implement the requested standardized skill.yaml permission manifest, install-time consent, skills audit/trust grades, signing, or attestation; the issue discussion also includes maintainer guidance to keep this open for product/security follow-up.

## What This Changes

The issue requests a standardized skill.yaml permission manifest for skills, with validation, install-time permission review, audit/trust reporting, signing, and attestation.

## Best Possible Solution

Keep this open for explicit maintainer product/security design. The end state should be a versioned skill permission manifest schema with parser/types, ClawHub and direct install/update integration, visible consent and audit/trust reporting, documented signing/attestation expectations, and a clear relationship to before_install and before_tool_call enforcement hooks.
