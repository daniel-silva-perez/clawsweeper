## Summary

Keep this issue open. Current main has related auth-profile rotation, model fallback, and a fixed overloaded backoff knob, but it does not implement the requested configurable LLM request retry policy with max attempts, retry classes, total wall-clock cap, or retry notification. The issue also has an open implementation candidate with closing syntax, so it should remain open until maintainers review/land or close that PR.

## Best Possible Solution

Keep the issue open and use it as the product/behavior tracker for configurable LLM request retry. Maintainers should review whether to land a focused version of #49800, fold it into the broader #38640 work, or design a first-class `agents.defaults.llmRetry`/global retry config that reuses the existing transient error classification without retrying auth, billing, format, or quota failures.
