## Summary

Keep this issue open. Current main has related auth-profile rotation and fixed overloaded backoff controls, plus cron retry prior art, but it still lacks the requested configurable LLM request retry policy with retry classes, per-attempt backoff array, max attempts, max wall-clock cap, or retry notification. The issue also has open implementation candidates with closing/related context, so it should not be closed before maintainers review those PRs.

## Best Possible Solution

Keep this issue open as the product tracker for configurable main LLM request retry. Maintainers should review whether to land a focused version of #49800, fold the work into #38640, or design a first-class `agents.defaults.llmRetry`/global retry policy that reuses existing transient error classification while avoiding retries for auth, billing, format, and quota/rate-limit failures unless explicitly chosen.
