## Summary

Keep this issue open. Current main has related auth-profile rotation controls, fixed overloaded backoff before profile/fallback rotation, and cron retry prior art, but it still lacks the requested configurable main LLM retry policy with retry classes, a per-attempt backoff array, max attempts, max wall-clock cap, or retry notification. The issue also has open implementation candidates with closing/related context, especially #49800, so cleanup should not close it before maintainer review.

## Best Possible Solution

Keep this issue open as the product tracker for configurable main LLM request retry. Maintainers should review whether to land a focused version of #49800, fold the work into #38640, or design a first-class `agents.defaults.llmRetry` or global retry policy that reuses the existing transient error classification while avoiding retries for auth, billing, format, and quota/rate-limit failures unless explicitly chosen.
