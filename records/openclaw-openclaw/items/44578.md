## Summary

Keep open. Current main still has the silent-success fallback this PR targets: a core `send` result without a `toolResult` is serialized as a normal JSON result, while the outbound direct-send path can produce `sendResult.result: undefined` when delivery returns no concrete result. The PR remains a focused implementation candidate, and the author recently added the dry-run and successful-messageId regression coverage requested by the prior Codex review.

## Best Possible Solution

Keep this PR open. The best path is to rebase or cherry-pick the focused message-tool implementation onto current main, preserve the missing-delivery guard plus the dry-run and successful-messageId regression tests, then run the targeted message-tool test and the normal changed gate before merge.
