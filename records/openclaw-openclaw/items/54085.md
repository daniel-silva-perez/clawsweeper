## Summary

Keep open. Current `main` does not implement the PR’s short-lived-command respawn skip, and the proposed branch shape still needs maintainer rework because the existing skip hook gates both warning suppression and startup-time `NODE_EXTRA_CA_CERTS` preparation. The prior ClawSweeper review comment already identified this same blocker and the current source/tests still support that conclusion.

## What This Changes

The PR changes CLI respawn policy so most short-lived commands skip the process respawn, keeps only selected long-running commands on the respawn path, and moves respawn policy coverage into a dedicated test file.

## Best Possible Solution

Split the respawn planner so mandatory startup-environment reexec can still run when needed, while ExperimentalWarning-only respawn can be skipped for verified short-lived commands. Keep the documented `NODE_EXTRA_CA_CERTS` contract intact and cover both command policy and startup TLS behavior in tests.
