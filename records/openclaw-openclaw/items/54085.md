## Summary

Keep open. Current main still does not implement the PR’s intended broad short-lived-command skip, and the existing PR shape needs maintainer review because the same skip gate also bypasses startup TLS reexec. This is an open draft implementation PR with a concrete design concern, not a stale, duplicate, or already-implemented cleanup case.

## What This Changes

The PR changes the CLI respawn policy so most short-lived commands skip process reexec while gateway, daemon, agent, and memory commands still respawn, and it moves respawn policy assertions into a dedicated test file.

## Best Possible Solution

Split the respawn planner so mandatory startup-environment reexec can still run when needed, while ExperimentalWarning-only suppression can be skipped for verified short-lived commands. Preserve the documented `NODE_EXTRA_CA_CERTS` startup contract and cover both command policy and startup TLS behavior in tests.
