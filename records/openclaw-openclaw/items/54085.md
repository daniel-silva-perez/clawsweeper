## Summary

Keep this draft PR open. Current main still does not implement the PR's broad short-lived-command respawn skip, and the PR remains useful but needs maintainer rework because the current respawn plan also prepares NODE_EXTRA_CA_CERTS before Node startup, not only ExperimentalWarning suppression. The prior ClawSweeper review comment identified the same blocker, and current main at 39f810911c382f92ea80b2fb406fdf7dbd405721 still has that coupling.

## What This Changes

The PR changes CLI respawn policy so only gateway, daemon, agent, and memory commands re-exec for ExperimentalWarning suppression while short-lived CLI commands skip the respawn, with tests moved into a dedicated respawn-policy test file.

## Best Possible Solution

Keep this draft PR open and rework it against current main by separating warning-only respawn suppression from mandatory startup environment reexec. The implementation should preserve NODE_EXTRA_CA_CERTS readiness, either preserve or explicitly revisit the current foreground gateway run skip policy, and add tests proving short-lived commands can skip warning-only respawn without skipping required TLS startup preparation.
