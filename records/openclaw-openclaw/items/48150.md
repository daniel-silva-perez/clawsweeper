## Summary

Keep this PR open. Current main still does not expose `email` or `enterprise_email` on Feishu live directory peers or match peer queries against those fields, and the PR remains a focused Feishu plugin implementation candidate rather than obsolete work.

## Best Possible Solution

Keep the PR open as a focused Feishu plugin implementation candidate. Rebase it onto current main, preserve the current normalization helper style, add the requested email-only query regression test, and have a maintainer review the intentional directory metadata exposure before landing.
