## Summary

Keep this issue open. Current main has only a partial session-restart mitigation that references this report; it does not resolve the reported Manifest/clawrouter port retry loop, process signal listener accumulation, or session persistence lock-hold/WS starvation path. The follow-up discussion also says the reporter has operational mitigations but no upstream source-level PR, and the related paperclip adapter retry work is downstream rather than an OpenClaw fix.

## Best Possible Solution

Keep this open as a runtime lifecycle bug. The best path is to split or stage fixes for: Manifest/clawrouter idempotent port binding or parent-socket reuse; centralized/deduplicated plugin resource cleanup for signal handlers and sockets across reloads; and a session persistence redesign that keeps fsync/durable JSON work outside latency-sensitive locks or batches it. Add focused regression coverage for listener counts across reloads, no duplicate listen attempts, bounded lock holds, and WebSocket handshake latency while session writes are active.
