## Summary

Keep this issue open. The report is recent, specific, and the current main still contains the relevant gateway/plugin lifecycle and session I/O surfaces. Main added plugin lifecycle tracing and event-loop readiness diagnostics after v2026.4.26, but those are observability improvements, not a clear fix for the Manifest port retry, plugin-owned signal/listener cleanup, or sync session/transcript I/O starvation. The reporter's link to paperclipai/paperclip#4684 is useful adapter-side mitigation context, not a superseding OpenClaw fix; the comment distinguishing #73653 as a separate schema regression also matches the code paths reviewed.

## Best Possible Solution

Keep the issue open as a gateway runtime lifecycle bug. Best path: reproduce with the Manifest/clawrouter plugin enabled and a cron-driven session, then add regression coverage that plugin reloads do not duplicate listen attempts or process listeners, old plugin resources are disposed before replacement where required, and WebSocket upgrade latency stays bounded while session store/transcript writes are active. The likely fix spans plugin cleanup semantics and moving hot session JSON/session JSONL work off synchronous event-loop paths.
