## Summary

Keep open. Current main has partial resilience for invalid plugin config, plugin load/register failures, hook-level timeouts, Active Memory recall timeouts, typing cleanup, and stuck-session diagnostics, but it still lacks the requested plugin-scoped circuit breaker, a generic hard timeout around plugin-owned tool execution, and typing-TTL-driven session recovery. Open #41899 overlaps on the circuit-breaker portion but does not cover the tool-timeout and typing-TTL recovery requests.

## Best Possible Solution

Keep this issue open unless maintainers explicitly consolidate it into #41899 and carry over the missing pieces. The best implementation is a core plugin-runtime resilience design: plugin-scoped rolling failure budgets around hook/register/tool paths, timeout and AbortSignal semantics for plugin-owned tools, persisted open/half-open/forced-open breaker state, operator status/reset controls, and an explicit product decision for whether typing TTL or stuck-session diagnostics should restart, fail, or recover the active session with user-visible outage handling.
