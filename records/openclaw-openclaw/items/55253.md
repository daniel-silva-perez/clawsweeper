## Summary

Keep open. Current main has meaningful partial resilience for invalid plugin config, selected hook timeouts, Active Memory recall timeout handling, typing cleanup, and other watchdog paths, but it still does not implement this issue's requested plugin-scoped circuit breaker, generic hard timeout for plugin-owned tool execution, or typing-TTL-driven session recovery. Open #41899 overlaps on the circuit-breaker design, but the supplied timeline/comment context does not make it the canonical replacement for this narrower incident report, and this issue also asks for tool timeout and typing TTL recovery behavior.

## Best Possible Solution

Keep this issue open unless maintainers explicitly consolidate it into #41899. The best implementation is a core plugin-runtime resilience design: plugin-scoped rolling failure budgets around hook/tool/register paths, timeout and AbortSignal semantics for plugin-owned tools, persisted breaker state with open/half-open/forced-open modes, status/reset controls, and a product decision for whether typing TTL should fail, restart, or recover the active session with user-visible outage handling.
