## Summary

Keep open. Current main still implements the reported 16k context-window minimum as a hard runtime block, not warning-only behavior; the guard, embedded runner setup path, custom-provider normalization, tests, and local-model docs all still encode the block-below-16k policy. The issue is valid, but changing it is runtime/product policy rather than conservative cleanup.

## What This Changes

Request to remove the hard 16k minimum context-window block for models and replace it with warning-only behavior.

## Best Possible Solution

Define the intended runtime policy for sub-16k models first: warning-only, explicit opt-in override, or a dynamic threshold. Once that is decided, update the guard, embedded-runner setup, custom-provider onboarding normalization, regression tests, and local-model docs together so small-model behavior is deliberate and documented.
