## Summary

Current main still enforces the requested 16k context-window minimum as a hard runtime block. The issue remains real product/runtime policy work: the guard, embedded-runner setup path, custom-provider onboarding normalization, tests, and local-model docs all still encode block-below-16k behavior.

## Best Possible Solution

Keep this issue open for maintainer product judgment. A complete implementation should replace the sub-16k hard block with warning-only behavior or a deliberate opt-in override, preserve clear low-context warnings for local and self-hosted models, stop onboarding from clamping intentionally smaller custom model windows if the policy changes, and update the guard/runtime tests plus local-model docs together.
