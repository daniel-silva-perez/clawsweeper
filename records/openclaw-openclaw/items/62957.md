## Summary

Current main still enforces the requested 16k context-window minimum as a hard runtime block. The central feature request is not implemented: sub-16k model context windows still produce warnings and then fail before the provider attempt runs, and the colocated tests continue to assert that below-16k models are blocked.

## Best Possible Solution

Keep this issue open for maintainer product judgment. The likely implementation path is to replace the sub-16k hard block with warning-only behavior or an explicit opt-in override, preserve clear low-context warnings for local/self-hosted models, ensure configured contextWindow/contextTokens values below 16k are honored intentionally, and update the guard/runtime tests plus docs together. If maintainers decide the hard floor is intentional product policy, close only after documenting that decision clearly.
