## Summary

Keep open. The linked bug is still present on current main and the PR is a relevant owner-module fix attempt, but the current branch is not merge-ready because it introduces external Poppler execution for untrusted PDFs with blocking security and correctness defects. The existing ClawSweeper comment remains applicable after the later merge-from-main commit because the fetched PR head still contains the same temp-file, pixel-budget, and unbounded `execSync` implementation.

## What This Changes

The PR adds a `pdftoppm`-based PDF page rendering fallback in `extensions/document-extract/document-extractor.ts` when `@napi-rs/canvas` cannot be loaded.

## Best Possible Solution

Keep the work in the `document-extract` plugin boundary if maintainers accept a Poppler fallback: implement private restrictive temp storage, argv-based bounded process execution, pixel-budget-faithful rendering, focused tests, PDF docs, and a changelog entry. If that external binary contract is not accepted, the safer product state is a clear degradation/error path instead of silent text-only output.

## Reproduction Assessment

Yes. The linked bug has a concrete Ubuntu setup without `@napi-rs/canvas`, and current main source shows that this path returns empty images; the PR defects are statically reproducible from the added DPI floor, `/tmp` writes, and unbounded `execSync` call in the fetched PR head.

## Solution Assessment

No. A Poppler fallback is plausible, but this implementation is not the best way because it bypasses current resource bounds and adds unsafe temp-file/process handling; a bounded `execFile` or `spawn` implementation with private temp storage is the safer alternative.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use private temp files for PDF rendering:** `extensions/document-extract/document-extractor.ts:174-178`
  - body: This writes user PDF bytes to `/tmp/openclaw-pdf-*` with default file creation behavior before invoking Poppler. On shared hosts that can expose sensitive PDFs and leaves a local race/symlink surface; create a private restrictive temp directory/file and clean it up from there.
  - confidence: 0.88
- **[P2] Honor the pixel budget in the pdftoppm path:** `extensions/document-extract/document-extractor.ts:171`
  - body: When `resolveRenderPlan()` returns `scale < 1`, this line forces the fallback to render at least 72 DPI while later subtracting only `plan.pixels`. Large pages can therefore produce images above `request.maxPixels`, defeating the existing total render budget.
  - confidence: 0.93
- **[P2] Bound the external renderer process:** `extensions/document-extract/document-extractor.ts:180-183`
  - body: The fallback invokes `pdftoppm` through synchronous `execSync` with no timeout or abort path while processing untrusted PDFs. A malformed or expensive PDF can block the Gateway event loop until Poppler exits; use argv-based process execution with a timeout and output bounds.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `extensions/document-extract/document-extractor.ts:149`
  - body: This is a user-facing PDF extraction behavior change, but the PR only changes plugin source. Add a `CHANGELOG.md` entry under the active version and credit the contributor per repo policy.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The PR introduces external PDF rendering for untrusted inputs with unsafe temp-file handling and unbounded synchronous process execution.

Concerns:

- **[high] PDF contents written to shared temp paths:** `extensions/document-extract/document-extractor.ts:174`
  - body: The fallback writes user-supplied PDF bytes to `/tmp/openclaw-pdf-*` with default write-file permissions and no private temp directory or exclusive-open contract. Sensitive PDFs can be exposed on shared Linux hosts, and global temp paths increase local race/symlink risk.
  - confidence: 0.88
- **[medium] Unbounded synchronous Poppler execution:** `extensions/document-extract/document-extractor.ts:180`
  - body: The renderer invokes `pdftoppm` via `execSync` without a timeout, abort signal, or explicit process/output limits. A hostile or malformed PDF can keep the Gateway event loop blocked while the external renderer runs.
  - confidence: 0.86
