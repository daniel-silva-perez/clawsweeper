## Summary

Keep open. The PR addresses a real gap in the PDF extraction fallback path, but the current draft is not merge-ready because it adds untrusted PDF rendering through a synchronous shell command, can still bypass the existing pixel budget, and omits the docs/changelog/test surface expected for a new optional system dependency.

## What This Changes

The PR modifies `extensions/document-extract/document-extractor.ts` to detect `pdftoppm` and render PDF pages through temporary files when `@napi-rs/canvas` cannot load.

## Best Possible Solution

If maintainers accept Poppler as an optional fallback, keep the change inside the `document-extract` plugin but make the renderer bounded and documented: use argv-based async process execution, avoid shell/PATH surprises, enforce pixel/output/time limits, add canvas-missing fallback tests, update `docs/tools/pdf.md`, and add the required `CHANGELOG.md` entry.

## Reproduction Assessment

Yes. The original bug has a high-confidence static reproduction on current main: low/no-text PDFs reach image fallback, `loadCanvasModule()` throws when `@napi-rs/canvas` is absent, and the extractor returns no images. The PR defects are visible in the latest patch at the `Math.max(72, ...)` DPI floor and `execSync()` call.

## Solution Assessment

No. A Poppler fallback is a plausible owner-module fix, but this draft is not the best implementation because it can exceed the render budget, blocks the Gateway event loop through shell execution, and lacks the dependency contract and regression coverage needed for a new external renderer path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Honor the render pixel budget:** `extensions/document-extract/document-extractor.ts:174`
  - body: When `resolveRenderPlan()` chooses `scale < 1`, this `Math.max(72, ...)` still renders at full 72 DPI in the Poppler path. Large or low-budget pages can produce more pixels than `request.maxPixels` while the loop subtracts only `plan.pixels`, so derive the Poppler output size from the render plan without the 72-DPI floor.
  - confidence: 0.94
- **[P2] Run Poppler asynchronously without a shell:** `extensions/document-extract/document-extractor.ts:187-190`
  - body: This uses `execSync()` for every rendered PDF page. A malformed or expensive PDF can block the Node event loop for up to 30 seconds per page, and the interpolated command still routes paths through shell parsing; use `execFile` or `spawn` with argv, timeout/abort handling, and output bounds instead.
  - confidence: 0.9
- **[P3] Document the optional Poppler dependency:** `extensions/document-extract/document-extractor.ts:148-153`
  - body: The fallback only works when `pdftoppm` is installed and on PATH, but the PR does not update the PDF docs that currently describe the canvas-only image-rendering path. Add user-facing install and behavior notes for the optional Poppler renderer.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `extensions/document-extract/document-extractor.ts:148`

## Security Review

Status: needs_attention

Summary: The diff introduces external rendering of untrusted PDFs and still needs safer process execution plus hard resource-bound enforcement before merge.

Concerns:

- **[medium] Synchronous shell execution of external PDF renderer:** `extensions/document-extract/document-extractor.ts:187`
  - body: The patch invokes `pdftoppm` through `execSync()` with an interpolated shell command and bare binary lookup. Even with a timeout, hostile or malformed PDFs can block the Gateway event loop per page, and argv-based execution would avoid shell parsing of temp paths.
  - confidence: 0.9
- **[medium] Pixel budget bypass amplifies untrusted PDF rendering:** `extensions/document-extract/document-extractor.ts:174`
  - body: The Poppler fallback keeps a 72-DPI floor even when the render plan chose a smaller scale to satisfy `maxPixels`, allowing oversized PNG renders and larger memory or prompt payloads for attacker-supplied PDFs.
  - confidence: 0.92
