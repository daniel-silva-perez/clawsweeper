## Summary

Keep open. Current main still does not support Feishu `ignoreFileContent` or an equivalent inbound file-content opt-out. The Feishu inbound path still downloads file/post media into `MediaPaths`, and shared media-understanding still extracts text-like attachments into the agent prompt.

## Best Possible Solution

Implement a Feishu-owned inbound attachment-content opt-out. Add documented top-level and per-account Feishu config, apply it before file/document resources enter `resolveFeishuMediaList` or before those entries enter `mediaPayload`, preserve placeholder text like `<media:document>`, and add focused Feishu plus shared media-understanding regression tests. Update Feishu schema/config metadata/help and the public Feishu docs page.
