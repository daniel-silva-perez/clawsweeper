## Summary

Keep open. Current main still does not implement Feishu `ignoreFileContent` or any equivalent inbound file-content opt-out. Feishu still downloads file/post media into `MediaPaths`, and shared media-understanding still extracts text-like attachments into the agent prompt.

## Best Possible Solution

Implement a Feishu-owned inbound attachment-content opt-out. Add documented top-level and per-account Feishu config, enforce it before file/document entries enter `resolveFeishuMediaList` or before they become `MediaPaths` for shared media-understanding, preserve placeholder text such as `<media:document>`, and add focused Feishu plus shared media-understanding regression tests. Update Feishu schema/config metadata/help and the public Feishu docs page.
