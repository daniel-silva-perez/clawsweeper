## Summary

Keep open. Current main at e476523082a6fff3a0c3048077d2f0e52413fbf2 still has no Feishu `ignoreFileContent` or equivalent inbound file-content opt-out. Feishu still resolves `file` and `post` media into saved inbound attachments, passes those paths through `MediaPaths`, and shared media-understanding can extract text-like file contents into the agent body.

## Best Possible Solution

Implement a Feishu-owned inbound attachment-content opt-out, ideally available at both `channels.feishu` and `channels.feishu.accounts.<id>`. Enforce it before file/document entries become `MediaPaths`, preserve a placeholder such as `<media:document>`, add focused Feishu dispatch and media-understanding regression tests, and update Feishu config schema/help plus https://docs.openclaw.ai/channels/feishu.
