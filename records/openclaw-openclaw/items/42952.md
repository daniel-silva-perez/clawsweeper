## Summary

Keep open. Current main still lacks any Feishu `ignoreFileContent` or equivalent inbound attachment-content opt-out, and the existing Feishu path still downloads file/post media into `MediaPaths`. Shared media-understanding can then extract text-like file attachments and append that content to the agent body, so the reported user problem remains current.

## Best Possible Solution

Implement a Feishu-owned inbound attachment-content opt-out, with both top-level and per-account config. Enforce it before file/document entries enter `resolveFeishuMediaList` or before they become `MediaPaths` for shared media-understanding, preserve a placeholder such as `<media:document>`, add focused Feishu and media-understanding regression tests, and update Feishu schema metadata/help plus the public Feishu docs page.
