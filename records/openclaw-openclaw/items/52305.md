## Summary

Keep open. Current main has partial session-scoped wake plumbing for internal exec/node-event paths, but the user-facing `openclaw system event` path still has no explicit session target and the issue is paired with open PR #50818 using closing syntax. That PR should be reviewed, updated, landed, or closed before this issue is closed.

## Best Possible Solution

Keep this issue open and use PR #50818, or a narrower replacement, as the implementation candidate. The core fix should provide an explicit session-targeted external completion path for `system event`/`wake` or a first-class async completion callback, preserve requester-session delivery context end to end, and address the remaining 1006 transport fragility if the CLI remains part of the supported callback path.
