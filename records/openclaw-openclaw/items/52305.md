## Summary

Keep open. Current main still leaves the user-facing `openclaw system event` / `wake` path without an explicit session target, while internal exec/node-event wake plumbing is only partially session-scoped. The issue is also paired with open PR #50818, whose body uses closing syntax for this issue, so it should remain open until that PR or a replacement is resolved.

## Best Possible Solution

Keep this issue open and use PR #50818, or a narrower replacement, as the implementation candidate. The core fix should provide an explicit session-targeted external completion path for `system event`/`wake` or a first-class async completion callback, preserve requester-session delivery context end to end, and add targeted tests for CLI/system-event routing plus the 1006 retry/failure behavior if the CLI remains a supported callback path.
