## Summary

Keep this PR open. Current main still does not implement the PR’s intended zero-valued provider usage behavior, and the PR’s one-file change is incomplete against the current token-usage pipeline. The existing review comments are still directionally correct: OpenClaw needs to distinguish “usage observed with zero counters” from “no usage observed,” then carry that distinction through aggregation, session persistence, and status display.

## Best Possible Solution

Keep the PR open for author or maintainer follow-up, or replace it with a complete current-main fix. The implementation should track whether usage was observed independently from whether counters are nonzero, return zero-valued totals when appropriate, mark fresh zero usage correctly in session persistence, preserve the no-usage stale fallback, and add focused regression tests through the subscription plus session-store/status path.
