## Summary

Keep this PR open. Current main still treats all-zero provider usage as missing usage across the subscription, accumulator, persistence, and status paths, so the PR’s intended user-visible fix is not implemented on main and the PR diff remains incomplete as described in the existing review comments.

## Best Possible Solution

Keep the PR open for author or maintainer follow-up, or replace it with a complete current-main fix. The implementation should track whether usage was observed independently from whether counters are nonzero, return and persist zero-valued totals when appropriate, keep truly missing-usage fallback behavior intact, and add focused regression coverage for subscription totals, runner metadata, session persistence, and status rendering.
