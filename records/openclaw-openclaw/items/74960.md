## Summary

Keep open. Current main still contains the Control UI image paste, side payload store, `chat.send`, and Gateway attachment dispatch path, but the source and existing tests do not prove or fix the reported stale/wrong-image byte mismatch in v2026.4.26/v2026.4.27; the linked WeChat context also makes this broader than the earlier paste-support closure.

## What This Changes

Report that Control UI/WebChat pasted images are delivered to the agent as a different or stale image, with similar stale-image behavior reported through the external openclaw-weixin channel.

## Best Possible Solution

Land one focused fix for the #74960/#62604 paste/media cluster: add byte-identity regression coverage from clipboard/channel input through the Control UI side store, `chat.send` serialization, Gateway parsing, and `dispatchInboundMessage`, then fix the smallest generic attachment/media boundary that fails. Keep openclaw-weixin-specific code out of core unless the regression proves a missing generic plugin/media API.
