## Summary

Keep this PR open. Current main does not implement either central behavior: `workspaceDir` is still not accepted by the gateway agent protocol, gateway dispatch still only forwards stored spawned-workspace metadata, and missing bootstrap files are still rendered as `[MISSING]` prompt entries. The PR also has unresolved review feedback that looks material: one stale test outside the PR validation set and one security-sensitive concern about accepting arbitrary workspace paths from write-scoped gateway callers.

## Best Possible Solution

Keep the PR open for revision rather than closing it as implemented. The best path is to split or tighten the two behaviors: gate any explicit gateway `workspaceDir` override behind a trusted internal/admin path or spawned-session metadata, and, if omitting missing bootstrap files is still desired, update all affected bootstrap/cache/stability tests and coordinate with the narrower missing-bootstrap work already discussed around #40210 and #73401.
