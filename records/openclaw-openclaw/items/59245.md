## Summary

Keep this issue open. Current main has useful voice-call primitives, but not the requested autonomous outbound task-call workflow; a maintainer has already made the same current-main assessment and explicitly kept the issue open.

## What This Changes

Add an outbound voice-call mode where an agent can call third parties for delegated tasks, handle IVR/hold/callback situations, escalate to the user, and share only authorized information.

## Best Possible Solution

Define and implement a documented task-call orchestration layer in the voice-call plugin on top of the existing outbound call, DTMF, realtime voice, and status primitives. The design should include a task objective schema, call state machine, IVR/hold strategy, trusted owner escalation with timeout behavior, allowed-information policy, duration/cost limits, transcript/reporting, and callback/reschedule semantics.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. There is a high-confidence current-main gap check: source, docs, tests, and targeted searches show primitives only and no autonomous outbound task-call workflow.

## Solution Assessment

Unclear for a full implementation because no patch is proposed. The narrowest maintainable direction is to layer task-call orchestration into the voice-call plugin; provider media offloading like VoIPBin can be evaluated as an adapter option, not a replacement for the product contract.
