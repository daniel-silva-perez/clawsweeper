## Summary

Keep open. Current main has not implemented this PR’s intended behavior: TaskFlow audit freshness is still based only on the flow record, so recent linked child-task activity cannot suppress a `stale_running` finding. The PR is small, directly targets the implicated code path, adds the missing regression shape, and does not appear to introduce supply-chain or security-sensitive changes.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to update or land the focused TaskFlow audit fix after deciding the active-vs-all linked task scope, with regression coverage that a managed running flow with recent linked child `lastEventAt` is not reported as `stale_running`.
