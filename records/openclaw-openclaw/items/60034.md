## Summary

Keep this issue open. Current main still writes bundled `session-memory` output to a fixed `<workspace>/memory/YYYY-MM-DD-slug.md` path, and the handler does not implement `outputPath`, `filename`, or placeholder expansion. The related implemented cleanup for memory naming/startup context does not cover this configurable writer request.

## Best Possible Solution

Keep the issue open for a focused maintainer decision. A good implementation would add a small, validated `session-memory` path/filename template contract under `hooks.internal.entries.session-memory`, reject absolute/path-traversal outputs, support documented date/slug placeholders, preserve the existing default, and update the bundled hook tests plus public docs. If maintainers prefer not to grow the bundled hook, the issue should be explicitly redirected to a custom hook/plugin path rather than closed as already implemented.
