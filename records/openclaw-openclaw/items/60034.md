## Summary

Keep this issue open. Current main still hardcodes bundled `session-memory` output to `<workspace>/memory/YYYY-MM-DD-slug.md`; the handler does not implement `outputPath`, `filename`, placeholder expansion, or empty-directory cleanup. Related startup-context work only made read-side startup memory loading tolerate slugged files, so it does not satisfy this configurable writer request.

## Best Possible Solution

Keep the issue open for a focused maintainer decision. The best product path is to add a small validated `session-memory` path/filename template contract under `hooks.internal.entries.session-memory`, preserve the current default, support documented date/slug placeholders, reject unsafe paths, add bundled hook tests for nested output and fallback behavior, and update the public hooks docs. If maintainers do not want to expand the bundled hook, they should explicitly redirect this to a custom hook/plugin approach rather than closing it as implemented.
