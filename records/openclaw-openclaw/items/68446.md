## Summary

Keep this PR open. Current main and the latest release still contain the WhatsApp `allowFrom` to `groupAllowFrom` fallback that this PR targets, and the PR is open/unmerged. The provided review discussion also shows this is security-adjacent and may still need a command-authorization follow-up before landing, so it is not a conservative cleanup close.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to revise or confirm the PR so WhatsApp group handling disables DM `allowFrom` fallback in both auto-reply group gating and command authorization, add focused regression tests for DM `allowFrom` present with `groupAllowFrom` absent under `groupPolicy: "allowlist"`, and update docs if WhatsApp no longer supports explicit `allowFrom` fallback for group sender authorization.
