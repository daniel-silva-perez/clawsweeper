## Summary

Keep this issue open. Current main has partial Telegram reaction support, but the requested `reactionGuidance: "off" | "minimal" | "full"` config, strict schema/generated metadata/docs coverage, gating of prompt guidance by `reactionNotifications`, and incoming-reaction soft-CTA prompt guidance are still missing.

## Best Possible Solution

Keep this open. If maintainers want the feature, implement it in the bundled Telegram plugin plus shared agent prompt composition: add and document `reactionGuidance` with `off|minimal|full` semantics, define how it composes with `reactionNotifications` and existing `reactionLevel`, gate incoming soft-CTA prompt guidance on enabled reaction notifications, update strict schema/generated config metadata/docs, and add focused tests for off/minimal/full plus notification gating.
