## Summary

Current main still does not implement the requested one-click Anthropic browser OAuth onboarding. Anthropic setup remains Claude CLI reuse, manual setup-token paste, or API-key auth; existing OAuth-token handling is runtime compatibility for already-provided tokens, not an onboarding/sign-in flow.

## Best Possible Solution

Keep this open for maintainer/product handling. If accepted, implement an Anthropic provider-owned browser OAuth auth method in extensions/anthropic, expose it through providerAuthChoices, use the existing ProviderAuthContext.openUrl and OAuth helper seams, persist the resulting auth profile safely, select subscription-vs-API defaults explicitly, update onboarding/configure/docs, and add focused Anthropic auth-choice/runtime regression coverage. If Anthropic cannot support a sanctioned browser OAuth flow, retitle or close around the supported Claude CLI/setup-token/API-key UX and consider a narrower token-prefix detection improvement separately.
