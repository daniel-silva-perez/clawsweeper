## Summary

Current main still does not implement one-click Anthropic browser OAuth onboarding. The bundled Anthropic provider exposes Claude CLI reuse, manual setup-token paste, and API-key auth, while docs still present Anthropic setup as API key or Claude CLI. Existing OAuth/token compatibility and provider OAuth seams are relevant, but they do not satisfy the requested browser sign-in flow or account-type auto-configuration.

## Best Possible Solution

Keep this open for maintainer/product handling. If accepted, implement an Anthropic provider-owned browser OAuth method in extensions/anthropic, expose it through providerAuthChoices, use ProviderAuthContext.openUrl plus existing OAuth helper seams, persist the resulting auth profile safely, choose subscription-vs-API defaults explicitly, and update onboarding docs/tests. If Anthropic cannot support a sanctioned browser OAuth flow, retitle or split this into supported Claude CLI/setup-token/API-key UX improvements and a separate token-prefix auto-detection task.
