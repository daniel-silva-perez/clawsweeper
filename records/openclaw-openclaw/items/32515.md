## Summary

Current main still does not implement one-click Anthropic browser OAuth onboarding. Anthropic setup remains Claude CLI reuse, manual setup-token, or API-key auth; existing OAuth handling is runtime compatibility for already-provided `sk-ant-oat` credentials, not a browser sign-in/account-detection flow. Related #48420 and #41461 cover runtime OAuth header behavior, not this onboarding feature.

## Best Possible Solution

Keep this open for maintainer/product handling. If accepted, implement an Anthropic provider-owned browser OAuth auth method in `extensions/anthropic`, expose it through `providerAuthChoices`, use the existing `ProviderAuthContext.openUrl` and OAuth helper seams, persist the resulting auth profile safely, select subscription-vs-API defaults explicitly, update onboarding/configure/docs, and add focused Anthropic auth-choice and runtime regression coverage. If Anthropic cannot support a sanctioned browser OAuth flow, maintainers should retitle or close around the supported Claude CLI/setup-token/API-key UX instead.
