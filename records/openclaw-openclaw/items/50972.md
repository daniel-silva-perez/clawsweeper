## Summary

Keep this issue open. Current main still ships `skills/gh-issues/SKILL.md` as a single oversized file at 885 lines / 34,800 bytes, and `skills/gh-issues` has no `references/` extraction. Two PRs attempted the requested split and explicitly linked this issue, but both are closed unmerged, so the reported cleanup is still actionable rather than implemented or superseded.

## Best Possible Solution

Refactor `skills/gh-issues/SKILL.md` into a concise core procedure under the stated size target, moving long phase details, API examples, and sub-agent/review prompt templates into `skills/gh-issues/references/` and scripts where appropriate. Coordinate with #54311 so the size cleanup can also help the broader bundled-skill quality work, but keep this issue open until that change lands on main.
