## Summary

Keep open. Current main still implements `POST /hooks/agent` as an async ingress that returns only a `runId`; it does not expose the requested opt-in wait/result mode or `announceToMain`-style suppression. The open PR #67433 is a direct implementation candidate, so the issue should remain open until that PR or an equivalent core change lands.

## Best Possible Solution

Keep this issue open and use PR #67433, or an equivalent maintainer-reviewed implementation, as the candidate path. The core change should add an opt-in blocking/result mode to `POST /hooks/agent`, return a narrow deterministic result, keep existing async behavior by default, document the new request/response fields, and add focused gateway hook tests covering default async mode, blocking success/error/timeout, idempotency, delivery suppression, and security policy boundaries.
