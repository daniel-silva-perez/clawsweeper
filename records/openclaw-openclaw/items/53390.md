## Summary

Keep this issue open. Current main still has no viewport-aware text snapshot contract or implementation for the browser tool: snapshot parameters and route planning do not carry current viewport/visible-only state, and the backend paths still take page/root accessibility snapshots or Chrome MCP structured snapshots rather than clipping text output to the scrolled viewport. The prior ClawSweeper keep-open comment remains consistent with current main, and the mentioned #24951 is a separate closed act-timeout report rather than a canonical fix.

## Best Possible Solution

Keep this issue open. The best fix is to define an explicit viewport-aware text snapshot contract, pass it through the browser tool schema/action and /snapshot route plan, implement consistent behavior across Playwright, raw CDP, and Chrome MCP paths, then add a long-page regression test that scrolls first and verifies the next text snapshot reflects the current viewport.
