## Summary

Keep open. Current main still has no viewport-aware text snapshot contract for the browser tool: the public schema, tool-to-route query, route plan, and backend snapshot paths all operate on the selected tab/root/full accessibility tree rather than the scrolled viewport. The linked #24951 context is a separate closed act-timeout report, and current search did not find an open PR that references this issue.

## What This Changes

Request that browser snapshot text output reflect the current visible viewport after scrolling instead of returning top-of-page or full-tree content.

## Best Possible Solution

Define and document a viewport-aware browser text snapshot contract, then implement it consistently through the tool schema, executeSnapshotAction query, /snapshot plan, Playwright, raw CDP, and Chrome MCP paths. Add a long-page regression that scrolls first and asserts the next text snapshot reflects visible content after the scroll, plus docs for any default or opt-in behavior.
