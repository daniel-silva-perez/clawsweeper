## Summary

Keep this issue open. Current main still has no viewport-aware text snapshot contract for the browser tool: the public schema, tool-to-route query, route plan, Playwright paths, raw CDP paths, and Chrome MCP path do not carry current viewport bounds or a visible-only option. The existing `labels` support is a viewport screenshot overlay, not viewport-clipped text snapshot behavior.

## Best Possible Solution

Keep this issue open. Define an explicit viewport-aware text snapshot contract, wire it through the browser tool schema/action and `/snapshot` route plan, implement consistent behavior across Playwright, raw CDP, and Chrome MCP paths, and add a long-page regression test that scrolls before snapshot and verifies the returned text reflects the visible viewport.
