## Summary

Keep open. Current main still lacks a viewport-aware text snapshot contract for the browser tool: the schema, action query, route plan, and backend snapshot implementations expose format/ref/selector/label options but no current-viewport or visible-only mode, and the snapshot paths still read page/root accessibility trees rather than filtering by the scrolled viewport.

## What This Changes

The issue requests that browser text snapshots reflect the currently visible viewport after scrolling instead of continuing to show top-of-page content.

## Best Possible Solution

Define and document a viewport-aware browser text snapshot contract, then implement it consistently through the tool schema, action query, snapshot route plan, Playwright path, CDP fallback, and Chrome MCP path. Add a long-page regression that scrolls first and asserts the next text snapshot reflects visible content after the scroll.
