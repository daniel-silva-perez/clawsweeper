## Summary

Keep open. Current main fixes the standalone non-streaming Telegram progress path, but partial preview streaming still intentionally renders visible `Working...` tool-progress edits by default, which is the central behavior reported here.

## What This Changes

Request that Telegram-bound sessions hide intermediate working/tool-call/output status messages and deliver only the final assistant reply to the chat.

## Best Possible Solution

Settle the Telegram product policy for partial streaming. If final-only or tool-progress-hidden behavior should be the default, make that change in the Telegram dispatch/config path, keep approval/media/error delivery exceptions intact, and update the Telegram tests plus public streaming docs; otherwise document the current `streaming.preview.toolProgress: false` and `streaming.mode: "off"` choices as the intended resolution.
