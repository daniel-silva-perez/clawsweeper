## Summary

Keep open. Current main still intentionally renders Telegram tool-progress preview edits by default when preview streaming is active, and the docs/tests describe that as released behavior with opt-outs rather than a completed final-only fix.

## What This Changes

Request that Telegram-bound sessions deliver only the final assistant reply while keeping intermediate tool calls, working indicators, and raw tool output invisible to the Telegram chat.

## Best Possible Solution

Maintainers should decide the Telegram privacy default for partial preview streaming. If final-only or no-tool-progress should be the default, update the Telegram preview-progress resolver/config default, keep explicit opt-in available, preserve approval/media/error delivery, and align focused tests, docs, and changelog.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by dispatching a Telegram turn with `streamMode: "partial"` and invoking `onToolStart` plus `onItemEvent`; the current focused test expects the visible `Working…` progress preview, and comments confirm live 2026.4.25 DM/group reproduction.

## Solution Assessment

Unclear. Hiding tool progress by default is technically narrow, but it changes an intentionally documented released behavior; the safer next step is maintainer product/security approval before changing the default.
