## Summary

Keep open. Current main and the latest release still make Telegram partial preview streaming show `Working...` tool-progress edits by default, while `streaming.mode: "off"` and `streaming.preview.toolProgress: false` are documented opt-outs. The related non-streaming and failed-tool-warning work does not supersede the remaining default-policy question for partial streaming.

## What This Changes

The issue requests that Telegram-bound sessions hide intermediate tool execution progress, working indicators, and raw tool/progress snippets so users see only the final assistant reply.

## Best Possible Solution

Maintainers should make an explicit Telegram privacy/security policy call. If the desired default is final-only or progress-hidden, change the Telegram dispatch/config default, preserve approval/media/error delivery, update focused Telegram tests, and revise the Telegram and streaming docs; if visible progress is intended, document the disclosure tradeoff clearly before closing as expected behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: with `streamMode: "partial"`, the current Telegram dispatch test calls tool progress callbacks and expects a visible `Working...` draft update, matching the reported Telegram behavior. I did not run a live Telegram session in this read-only review.

## Solution Assessment

Unclear. The current opt-outs are maintainable for operators who know to configure them, but they do not satisfy the requested default of only delivering the final assistant reply; flipping the default is narrow technically but needs maintainer product/security approval.
