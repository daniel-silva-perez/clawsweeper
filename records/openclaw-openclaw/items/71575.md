## Summary

Keep open. Current main still renders Telegram tool-progress preview edits by default when partial preview streaming is active; the later non-streaming/off-path suppression only covers `streaming.mode: "off"`. The remaining request is a privacy-sensitive default/product decision rather than an already-implemented fix.

## What This Changes

The issue requests that Telegram-bound sessions deliver only the final assistant reply, keeping intermediate tool calls, working indicators, and raw tool output invisible.

## Best Possible Solution

Maintainers should choose the Telegram privacy default for partial streaming. If tool progress should be hidden by default, update the Telegram preview-progress resolver or config default, keep explicit opt-in available, preserve approval/media/error routing, and align focused tests, docs, and changelog.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by dispatching a Telegram turn with `streamMode: "partial"` and invoking `onToolStart` plus `onItemEvent`; the current focused test expects the visible `Working…` preview output, and issue discussion includes a 2026.4.25 DM/group reproduction.

## Solution Assessment

Unclear. The current implementation is internally consistent with docs and opt-outs, but it does not satisfy the requested final-only-by-default behavior; flipping that default is technically narrow but needs maintainer privacy/product approval.
