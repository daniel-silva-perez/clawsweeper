## Summary

Keep open. Current main and the latest release still intentionally render Telegram tool-progress preview edits by default when partial preview streaming is active; the available opt-outs and the non-streaming suppression do not satisfy the issue's final-only default request.

## What This Changes

Telegram-bound sessions should keep tool-call progress, working indicators, and raw tool output internal so users see only the final assistant reply.

## Best Possible Solution

Maintainers should decide the Telegram privacy default for partial preview streaming. If the desired end state is final-only or no-tool-progress by default, update the Telegram preview-progress default/resolver, preserve an explicit opt-in for users who want progress previews, and align tests, docs, config hints, and changelog.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by dispatching a Telegram turn with `streamMode: "partial"` and invoking `onToolStart` plus `onItemEvent`; the focused test currently expects the visible `Working…` progress preview.

## Solution Assessment

Unclear. The technical change is narrow, but it reverses a documented released default; the safer solution is maintainer approval on the privacy/product default before changing code.
