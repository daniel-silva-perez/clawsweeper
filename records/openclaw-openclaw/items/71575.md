## Summary

Keep open. Current main still sends Telegram tool-progress preview edits by default when partial preview streaming is active, including visible `Working...` lines and tool/item progress text; the current docs describe opt-outs, but the issue's requested final-only-by-default behavior remains unresolved and privacy-sensitive.

## What This Changes

The issue requests that Telegram-bound sessions hide intermediate tool-progress previews, working indicators, and raw tool output so users receive only the final assistant reply.

## Best Possible Solution

Maintainers should choose the Telegram privacy default for partial streaming. If final-only or tool-hidden delivery should be the default, update the Telegram preview-progress resolver or config default, preserve explicit opt-in plus approval/media/error routing, and align focused tests, docs, and changelog.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by dispatching a Telegram turn with `streamMode: "partial"` and invoking `onToolStart` plus `onItemEvent`; the current test expects the visible `Working...` draft update, and the issue comments include a 2026.4.25 DM/group reproduction.

## Solution Assessment

Unclear. The current implementation is documented and has opt-outs, but it does not solve the requested final-only default; flipping the default is technically narrow but needs maintainer product/security approval because current tests and docs preserve visible progress previews.
