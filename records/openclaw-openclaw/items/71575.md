## Summary

Keep open. Current main still intentionally sends Telegram tool-progress preview edits by default under partial streaming, including visible `Working...` lines and tool/item progress text; the issue discussion adds a confirmed 2026.4.25 reproduction and the related PRs only cover narrower non-streaming or failed-tool-warning paths.

## What This Changes

The issue asks for Telegram sessions to hide intermediate tool progress, status, and raw tool output so only the final assistant reply is visible.

## Best Possible Solution

Maintainers should decide the Telegram privacy default for partial preview tool progress. If progress should be hidden by default, change the Telegram streaming default or preview-progress resolver, preserve approval/media/error delivery, update the focused Telegram tests, and revise the Telegram and streaming docs; otherwise document the disclosure tradeoff as intentional before closing.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by dispatching a Telegram turn with `streamMode: "partial"` and tool callbacks; the current test expects the visible `Working...` draft update, and a commenter confirmed the same behavior on OpenClaw 2026.4.25 in DMs and groups.

## Solution Assessment

Unclear. The current opt-outs are maintainable workarounds, but they do not satisfy final-only-by-default delivery; flipping the default is technically narrow but needs maintainer product/security approval because current tests and docs preserve visible progress previews.
