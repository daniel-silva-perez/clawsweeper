## Summary

Keep open. Current main still renders Telegram tool-progress preview edits by default when partial preview streaming is active, including visible `Working...` status and tool/item progress lines. The current opt-outs and final-only mode are documented workarounds, while the related #72482 and #73994 context covers narrower non-streaming or failed-tool-warning paths and does not supersede the remaining partial-streaming default policy.

## What This Changes

The issue requests that Telegram-bound sessions hide intermediate tool execution progress and raw tool output so only the final assistant reply is visible.

## Best Possible Solution

Maintainers should decide the Telegram privacy/security default for partial preview tool progress. If progress should be hidden by default, change the Telegram dispatch/config default, preserve approval/media/error delivery, update focused Telegram tests, and revise the public Telegram and streaming docs; otherwise document the intentional disclosure tradeoff before closing as expected behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: with `streamMode: "partial"`, the current Telegram dispatch test calls tool progress callbacks and expects a visible `Working...` draft update, matching the issue's reported Telegram behavior.

## Solution Assessment

Unclear. The current documented workaround is maintainable for operators who opt out, but it does not satisfy the requested final-only-by-default behavior; flipping the default is narrow technically but needs maintainer product/security approval because current tests and docs intentionally preserve visible progress previews.
