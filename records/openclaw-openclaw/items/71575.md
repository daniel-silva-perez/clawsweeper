## Summary

Keep open. Current main still sends visible Telegram `Working...` tool-progress preview edits by default when partial preview streaming is active; the existing opt-outs and final-only mode are documented workarounds, not an implementation of the requested final-only-by-default behavior. The related closed PRs cover narrower non-streaming or failed-tool-warning paths and do not supersede the remaining partial-streaming default policy.

## What This Changes

The issue requests that Telegram-bound sessions hide intermediate tool execution progress and raw tool output so only final assistant replies are shown.

## Best Possible Solution

Maintainers should decide the Telegram privacy/security default for partial preview tool progress. If it should be hidden by default, change the Telegram dispatch/config default, preserve approval/media/error delivery, add focused Telegram regression coverage, and update the public streaming docs; otherwise document the intentional security tradeoff clearly before closing as product behavior.
