## Summary

Keep this issue open. Current main still does not implement Slack bot-message unfurl controls: the strict Slack config schema and TypeScript config type lack `unfurlLinks`/`unfurlMedia`, the Slack `chat.postMessage` payload builder never sends `unfurl_links`/`unfurl_media`, docs/tests only cover adjacent inbound Slack unfurl handling, and prior matching PRs were closed unmerged rather than landing this feature.

## Best Possible Solution

Keep this issue open and implement it in the bundled Slack plugin. Add schema-validated and typed `unfurlLinks` and `unfurlMedia` fields at `channels.slack` and `channels.slack.accounts.<id>`, resolve them through the existing account merge path, pass them to Slack as `unfurl_links` and `unfurl_media` for all normal `chat.postMessage` text/block/follow-up sends, and update Slack docs plus focused config/send tests. Treat #48151 as useful config-expressiveness context and #34512/#34526/#34593 as prior implementation attempts, not as canonical trackers that supersede this open request.
