## Summary

Current main does not implement Slack bot-message unfurl controls. The strict Slack config schema and TypeScript config type still lack `unfurlLinks`/`unfurlMedia`, outbound `chat.postMessage` calls never pass `unfurl_links`/`unfurl_media`, docs/tests only cover adjacent Slack behavior, and the locked Slack Web API types support the requested fields. This remains an actionable bundled Slack plugin feature request, not a duplicate of the adjacent SecretRef/config expressiveness issue.

## Best Possible Solution

Keep this issue open and implement the feature in the bundled Slack plugin: add typed and schema-validated `unfurlLinks` and `unfurlMedia` config fields at both `channels.slack` and `channels.slack.accounts.<id>`, resolve them through the existing account merge path, pass them to Slack as `unfurl_links` and `unfurl_media` for `chat.postMessage` text/block/follow-up sends, and update Slack docs plus focused config/send tests. Treat #48151 as useful config-expressiveness context, not as a canonical duplicate.
