## Summary

Current main does not implement the requested Slack bot-message unfurl controls. The bundled Slack plugin schema has no `unfurlLinks`/`unfurlMedia` or `unfurl_links`/`unfurl_media` fields, the outbound Slack send path never forwards those Slack Web API arguments, and the Slack docs/tests only cover adjacent streaming, chunking, media, and inbound unfurl handling. The related #48151 discussion is useful context about config expressiveness, but it does not supersede this Slack-specific feature request.

## Best Possible Solution

Keep this issue open and implement it in the bundled Slack plugin: add typed, schema-validated top-level and per-account Slack config fields such as `unfurlLinks` and `unfurlMedia`, resolve them through account config, pass them to `chat.postMessage` as `unfurl_links` and `unfurl_media` for text/block follow-up sends, and update Slack docs plus focused send/config tests. Use the existing Telegram `linkPreview` setting as a nearby precedent while preserving Slack’s two-field API shape.
