## Summary

Keep this issue open. Current main still routes provider usage through a closed provider ID list that excludes Kimi, Kimi Coding, and Moonshot, and the bundled Kimi/Moonshot provider plugins do not implement the existing provider usage auth/snapshot hooks. The linked prior PR is useful implementation context, but it was closed unmerged and had a reported provider-aliasing problem, so it is not a shipped fix.

## What This Changes

The issue requests Kimi/Moonshot balance and usage snapshots for OpenClaw's provider usage/status surfaces, with Kimi Coding credentials and Moonshot balance fallback support.

## Best Possible Solution

Implement provider-owned usage integration for Kimi Coding and Moonshot using the existing plugin hooks, extend or generalize usage-provider discovery so those providers can be selected, keep Kimi/Moonshot/z.ai rows separate, and update usage-tracking plus Moonshot/Kimi docs. Add a /quota command only if maintainers explicitly want a new chat command beyond the existing usage/status surfaces.
