## Summary

Current main still lacks the requested native PowerShell contributor-command CI smoke. The related docs formatter has been moved behind a Node script, but Windows CI still only runs the existing Windows test task under bash, and the only pwsh step is the Defender helper. Keep open; #44293 remains a related narrower docs-command issue, and #74163 is a broad maintainer tracker, not an implementation PR for this request.

## Best Possible Solution

Add a small checks-windows matrix entry that runs selected contributor commands under shell: pwsh, starting with pnpm check:docs, while preserving the existing Windows test task and read-only permissions. Document the guarantee in docs/ci.md and add/update CI-routing tests so future workflow changes do not drop the PowerShell smoke.
