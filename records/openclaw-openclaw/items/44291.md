## Summary

Keep this issue open. Current main has made the related docs formatter more PowerShell-friendly by moving `format:docs:check` to `scripts/format-docs.mjs`, but it still does not add the requested native PowerShell CI smoke lane for contributor commands. The Windows CI matrix still contains only the existing test task, the `checks-windows` job defaults to bash, and the only `pwsh` step is a Windows Defender setup helper rather than `pnpm check:docs`, `pnpm build:strict-smoke`, or another contributor-command smoke.

## Best Possible Solution

Keep this open and add a deliberately small native PowerShell Windows CI smoke for selected contributor commands. A conservative path is a new or extended `checks-windows` matrix task that runs under `shell: pwsh`, starts with `pnpm check:docs`, documents the guarantee in `docs/ci.md`, and keeps permissions/cache/install behavior aligned with the existing Windows job without duplicating the full Windows matrix.
