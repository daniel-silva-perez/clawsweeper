## Summary

Current main still rejects the reported live Docker attestation shape, while this PR is a narrow verifier/test update that matches the live GHCR payload and upstream attestation storage contract without changing the Docker release workflow or broadening execution surfaces.

## What This Changes

The PR updates `scripts/verify-docker-attestations.mjs` and its Vitest coverage to accept Docker attestation manifests without `artifactType` when their layers are in-toto JSON.

## Best Possible Solution

Land this PR or an equivalent focused verifier update that accepts the current Buildx OCI-manifest-with-in-toto-layers shape while preserving legacy `artifactType` support, descriptor prefiltering, required SBOM/provenance predicate checks, and malformed-manifest diagnostics.
