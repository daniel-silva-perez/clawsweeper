## Summary

Keep open for maintainer review. Current main still rejects the live GHCR attestation shape, while this PR matches BuildKit's documented attestation storage model and keeps the existing descriptor prefilter plus required predicate checks. I found no actionable correctness or security blocker.

## What This Changes

The PR updates `scripts/verify-docker-attestations.mjs` and its Vitest coverage to accept Docker attestation manifests that omit `artifactType` when their layers are in-toto JSON.

## Best Possible Solution

Land this or an equivalent focused verifier update that accepts the current Buildx OCI-manifest-with-in-toto-layers shape while preserving legacy `artifactType` support, descriptor prefiltering, required SBOM/provenance predicate checks, and malformed-manifest diagnostics.
