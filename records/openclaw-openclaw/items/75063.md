## Summary

Keep this PR open for maintainer review and landing. Current main still rejects the live Buildx attestation shape, the proposed change matches the upstream BuildKit/OCI contracts, and I found no blocking correctness or security issue in the patch.

## What This Changes

The PR updates `scripts/verify-docker-attestations.mjs` to recognize attestation manifests either by the legacy Docker `artifactType` or by non-empty all-in-toto layers, and adds Vitest coverage for the current Buildx shape plus a malformed-manifest rejection case.

## Best Possible Solution

Land this PR or an equivalent focused verifier update that accepts the current Buildx OCI-manifest-with-in-toto-layers shape while preserving legacy `artifactType` support, descriptor prefiltering, required SBOM/provenance predicate checks, and malformed-manifest diagnostics.

## Reproduction Assessment

Yes. Current main reproduces with `node scripts/verify-docker-attestations.mjs --platform linux/amd64 --platform linux/arm64 ghcr.io/openclaw/openclaw:2026.4.29-beta.1`, and raw manifest inspection confirms the attestations omit `artifactType` while retaining in-toto SBOM/provenance layers.

## Solution Assessment

Yes. This is the narrowest maintainable fix I found: it updates the verifier's attestation-shape recognition at the existing boundary without changing Docker release workflow permissions, image publication, CLI parsing, or predicate requirements.
