## Summary

Keep this PR open. Current main still does not implement the proposed `web_fetch` env-proxy forwarding, but the branch changes a security-sensitive default network path that conflicts with existing strict-mode coverage and overlaps the broader explicit-opt-in design in #58034, so it needs maintainer/security policy review rather than cleanup closure.

## What This Changes

The PR adds `useEnvProxy: true` to the `web_fetch` guarded-fetch call site and adds a regression test asserting that `createWebFetchTool` forwards that flag.

## Best Possible Solution

Land a policy-consistent web_fetch proxy design: either keep strict DNS pinning as the default and add a documented explicit trusted-proxy opt-in with config/schema/docs/tests, or intentionally change the default with matching security documentation and updated regression coverage.
