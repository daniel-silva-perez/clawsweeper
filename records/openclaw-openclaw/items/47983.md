## Summary

Keep open: current main has the file input and selected-file handling, and v2026.4.26 shipped broader non-video attachment support, but the reported visible upload-button path is still fragile and untested. The toolbar button still uses a global hidden-input lookup/click and lacks focused coverage that clicking the visible button invokes the intended picker.

## What This Changes

The issue reports that the WebChat upload button is visible in the Control UI chat composer but clicking it does not open the file picker on Chrome for Windows.

## Best Possible Solution

Keep the current non-video attachment support, but make the chat composer's visible attach control reliably target its own file input through a component-scoped ref or label-backed input, add type="button" to the toolbar button, and cover the visible button-to-input click path plus existing selection/preview behavior.
