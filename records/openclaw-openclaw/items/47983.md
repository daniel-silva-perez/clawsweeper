## Summary

Keep open. Current main still has the same visible WebChat/Control UI attach button path under review: the hidden file input and post-selection attachment pipeline exist, and the merged non-image attachment work in #72299 helps after file selection, but the visible button still triggers the picker through an unscoped document-wide lookup and the tests do not exercise the reported button-to-file-picker activation path.

## What This Changes

The issue requests that clicking the WebChat/Control UI upload button open the file chooser and allow selected files to be sent as attachments.

## Best Possible Solution

Keep the existing attachment serialization and Gateway media path intact, but make the chat composer Attach file control activate its own file input through a Lit ref/createRef or label-backed input, add an explicit button type, and add regression coverage for clicking the visible control.
