## Summary

Keep open. Current main supports selected-file handling and downstream non-video attachment serialization, but the user-reported visible Attach file button path still goes through an unscoped document-wide query and has no regression test that clicks the visible button. Related attachment PRs cover downstream payload, append, size, or video behavior rather than this picker activation report.

## What This Changes

The issue reports that the WebChat/Control UI upload button in OpenClaw 2026.3.13 does nothing when clicked in Chrome on Windows instead of opening the file picker.

## Best Possible Solution

Keep the existing selected-file and Gateway attachment pipeline, but make the chat composer Attach file control activate its own file input through a component-scoped ref or label-backed input, add an explicit type="button", and cover the visible button-to-input click route in the chat view tests.
