## Summary

Keep open. Current main has the downstream attachment-selection and non-video file pipeline, but the visible Control UI/WebChat attach button still activates the hidden input through an unscoped document-wide lookup, and the tests only exercise direct hidden-input change events rather than the reported visible button path.

## What This Changes

The issue asks for the WebChat/Control UI upload button to open the file picker and let the selected file be sent as an attachment.

## Best Possible Solution

Keep the existing attachment serialization and Gateway media path intact, but make the chat composer Attach file control target its own file input through a component-scoped ref/createRef or label-backed input, add an explicit button type, and add a regression test that clicks the visible control and verifies the hidden input activation path.
