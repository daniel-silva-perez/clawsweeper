## Summary

Keep open: current main still renders the WebChat/Control UI attach button through a fragile document-wide hidden input click, and the focused tests cover selected-file handling but not the visible upload button path the reporter says is dead. The shipped non-video attachment work reduces the downstream file-handling gap, but it does not prove the paperclip button reliably opens the picker.

## What This Changes

The issue asks for the WebChat/Control UI upload button to open the file chooser and allow selected files to be sent as attachments.

## Best Possible Solution

Keep the shipped attachment support, but make the chat composer's visible attach control reliably target its own file input through a component-scoped ref or label-backed input, add type="button", and cover the visible button-to-input click path alongside the existing selection and preview behavior.
