## Summary

Keep open. Current main supports selected-file handling and non-video attachment payloads, but the visible WebChat attach button still depends on a document-wide hidden-input click path, and existing tests do not cover the button-to-picker route reported as dead.

## What This Changes

The issue asks for the WebChat/Control UI upload button to open the native file picker and send selected files as attachments on Chrome/Windows.

## Best Possible Solution

Keep the existing attachment pipeline, but make the chat composer attach control target its own file input through a component-scoped ref or label-backed input, add an explicit button type, and cover the visible button-to-input click path in the chat view tests.
