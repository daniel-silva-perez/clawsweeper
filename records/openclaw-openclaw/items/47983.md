## Summary

Keep open: current main supports attachment selection and serialization after a file is selected, but the reported paperclip-button path still uses a global hidden-input click and lacks focused coverage that clicking the visible button opens the intended picker.

## What This Changes

The issue reports that the WebChat/Control UI upload button in OpenClaw 2026.3.13 on Chrome/Windows does not open the file picker.

## Best Possible Solution

Make a narrow Control UI fix in the chat composer: replace the global `document.querySelector` picker trigger with a component-scoped input ref or label-backed input, add `type="button"` to the toolbar button, preserve current non-video attachment support, and add focused coverage that clicking the visible attach button invokes the intended hidden input while existing selection/preview behavior still passes.
