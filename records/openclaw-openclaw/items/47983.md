## Summary

Keep open because current main still has a fragile visible upload-button path: selected-file handling exists, but the Attach file button still triggers the hidden input through a document-wide selector and there is no regression test that clicks the visible button.

## What This Changes

The issue asks for the WebChat upload button to open a file picker and allow selected files to be sent as attachments.

## Best Possible Solution

Keep the existing selected-file and Gateway attachment pipeline, but make the composer Attach file control target its own input through a scoped ref or robust label-backed control, add an explicit button type, and cover the visible button-to-input click route in the chat view tests.
