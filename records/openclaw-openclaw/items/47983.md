## Summary

Keep open. Current main has WebChat/Control UI attachment handling, but the reported picker-open failure is still a distinct, unproven interaction path: the attach button still programmatically clicks a hidden file input via a global selector, and the existing tests only dispatch a change event after files are already selected.

## Best Possible Solution

Keep this issue open and fix or verify the actual picker activation path. A focused maintainer path would be to replace the global selector with a scoped `ref` or accessible `label`/input wiring, add a regression test that clicking the Attach file button invokes the intended file input, and then manually verify the Control UI/WebChat picker in Chrome on Windows or an equivalent browser smoke.
