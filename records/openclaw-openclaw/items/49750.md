## Summary

Current main still does not implement the PR's central parser fix: resolveGroupSessionKey() reads ctx.From from the first segment, so agent-prefixed Slack thread keys can still fall back to the context provider such as webchat. The restart-sentinel area has partial overlap on base-session fallback, but current main does not parse a Slack target from the session key before falling back to base routing. This PR remains a relevant implementation candidate, though its parser patch should address the unresolved review feedback about non-group agent-prefixed keys.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rework the parser to reuse or mirror parseRawSessionConversationRef() so agent:<id>:<provider>:<group|channel>:... is recognized only when an explicit group/channel kind is present, add regression coverage for agent-prefixed Slack thread keys under webchat/provider-fallback contexts, and align restart-sentinel fallback so parsed thread targets win before base-session route fill-ins.
