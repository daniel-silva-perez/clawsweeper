## Summary

Keep open. Current main still lacks the central parser fix for agent-prefixed Slack thread keys, so this PR is not obsolete. The restart-sentinel side has some overlap on base-session fallback, but it does not make the PR fully implemented, and the PR also has unresolved review feedback about safely parsing agent-prefixed keys.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rework the parser to reuse or mirror the explicit `agent:<id>:<provider>:<group|channel>:...` parsing contract, add regression coverage for `agent:<id>:slack:channel:<channelId>:thread:<threadTs>` under webchat/provider-fallback contexts, and keep the restart-sentinel fallback behavior aligned so partial session hints do not override parsed thread targets.

## Evidence

For `agent:builder:slack:channel:...`, that is `agent`, so the encoded Slack provider is not selected before falling back to the context provider.
