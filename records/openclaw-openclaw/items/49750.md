## Summary

Keep open. Current main still does not implement the central parser fix: `resolveGroupSessionKey()` still reads only the first `From` segment, so an agent-prefixed Slack thread key can still be resolved under the context provider such as `webchat`. The restart fallback work overlaps part of the PR, but the PR is still a relevant implementation candidate and also has existing review feedback that should be addressed before merge.

## Best Possible Solution

Keep this PR open as an implementation candidate, but rework the group-key parser to require an explicit `group` or `channel` kind after `agent:<id>:<provider>` or reuse the existing session-conversation parser. Add focused regression coverage for `agent:<id>:slack:channel:<channelId>:thread:<threadTs>` under webchat/provider-fallback contexts and for non-group aliases such as `agent:main:webchat`, then re-check the restart-sentinel fallback behavior so parsed thread targets and partial account hints are preserved correctly.
