## Summary

Keep this PR open. Current main still does not surface session label/displayName through the status summary, CLI status table, or TUI /status summary, and the proposed patch needs rework for the current status rendering pipeline plus the existing resolver and terminal-sanitization review concerns.

## Best Possible Solution

Rework this PR against current main: add `label` and `displayName` to `SessionStatus` and `GatewayStatusSummary`, copy them from `SessionEntry` in `status.summary.ts`, render a sanitized friendly session name in `status.command-sections.ts` and `tui-status-summary.ts`, prefer the resolvable label or show label/displayName clearly, and add focused CLI/TUI status tests including unsafe control characters. That would make this a useful implementation candidate for #51184.

## Risks / Open Questions

- The proposed diff would print `label`/`displayName` without `sanitizeTerminalText` or `sanitizeRenderableText`, so control characters, newlines, or ANSI escapes from session labels or channel/thread metadata could affect CLI/TUI output.
- The proposed `displayName ?? label ?? key` order can hide the `label` value that `sessions.resolve` can actually resolve; follow-up commands accept `key`, `sessionId`, or `label`, not `displayName`.
- The PR patch targets the older `status.command.ts` row site, while current main routes CLI session rows through `status.command-report-data.ts` and `status.command-sections.ts`, so it needs a rebase/rework before review.
