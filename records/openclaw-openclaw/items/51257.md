## Summary

Keep open. Current main still does not implement the intended feature, and this PR is a plausible implementation candidate for #51184, but it needs rework for the current status rendering path plus the unresolved resolver and terminal-sanitization concerns.

## Best Possible Solution

Rework this PR against current main: add `label` and `displayName` to the status payload/types, copy them from `SessionEntry` in `status.summary.ts`, render a sanitized friendly session name in the current CLI and TUI status paths, preserve or clearly expose the resolvable label for follow-up commands, and add targeted tests for CLI/TUI output including unsafe control characters.

## Risks / Open Questions

- The proposed `displayName ?? label ?? key` rendering can hide the resolvable `label` value because gateway resolution does not accept `displayName`.
- The proposed output path would expose session labels/display names to CLI/TUI terminal rendering without sanitizing control characters, newlines, or ANSI escape sequences.
- The PR patch is stale against current main because the CLI session table now renders through `status.command-report-data.ts` and `status.command-sections.ts`, not the old direct row construction in `status.command.ts`.
- No focused regression coverage is present for friendly session names in current status summary, CLI status rows, TUI gateway status formatting, or unsafe label/displayName text.
