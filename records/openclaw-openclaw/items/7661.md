## Summary

Keep open. Current main still lacks the requested end-to-end channel creation workflow: the shared action vocabulary includes `channel-create` and Discord has partial provider-side support, but the CLI and public message docs expose only channel info/list, and Slack still has no discovery, gate, docs, scope, or runtime handler for channel creation. The related #65065 remains open and only addresses part of the Discord CLI surface, so it does not supersede the Slack/provider policy work.

## What This Changes

The issue requests native `openclaw message channel create` and message-tool `channel-create` support for Slack, Discord, and similar messaging providers.

## Best Possible Solution

Keep the issue open until maintainers approve the channel-management authority model, then implement provider-owned channel creation: Slack should add an explicit channel-management gate, `channels:manage` setup/docs, discovery, runtime support backed by Slack channel creation APIs, and targeted tests; Discord CLI wiring can land through #65065 or a replacement while keeping provider-specific behavior inside the Discord plugin boundary.
