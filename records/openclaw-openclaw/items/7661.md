## Summary

Keep open. Current main still lacks the requested end-to-end channel creation capability: Discord has provider-side `channel-create` execution, but the CLI and docs still expose only channel info/list, and Slack has no `channel-create` discovery, dispatch, channel-management action gate, or setup-scope guidance. The related open PR #65065 remains relevant only to a partial Discord CLI slice and is not merged.

## What This Changes

Add native message-tool and CLI support for creating messaging channels, especially Slack channel creation and Discord guild channel creation.

## Best Possible Solution

Approve a provider-owned channel-management authority model, then add Slack channel creation behind an explicit action gate with setup/docs for the required Slack scope and focused discovery/runtime tests; land or supersede the Discord CLI wiring separately while keeping provider-specific behavior inside the Discord plugin boundary.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. The feature gap is high-confidence from static current-main checks: the CLI/docs lack `message channel create`, Slack does not advertise or execute `channel-create`, and Discord only covers the provider-side slice.

## Solution Assessment

Unclear until maintainer review. Provider-owned implementation behind explicit gates is the narrow maintainable direction, but the current request should not be auto-implemented before the workspace/server administration authority model is approved.
