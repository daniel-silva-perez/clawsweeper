## Summary

Keep open. Current main has partial Discord provider support for `channel-create`, but the requested end-to-end surface is still incomplete: the CLI still exposes only `message channel info|list`, Slack does not advertise or execute channel creation, and Slack setup/docs do not include a channel-management action gate or `channels:manage` scope. The prior maintainer discussion also explicitly kept this open for product/security review because agent-driven channel creation grants workspace/server administration authority.

## What This Changes

The issue requests native `openclaw message channel create` and message-tool `channel-create` support for Slack, Discord, and similar messaging providers.

## Best Possible Solution

Land provider-owned channel creation only after maintainer review of the authority model: Slack should add an explicit channel-management action gate, setup/docs for `channels:manage`, discovery/runtime support backed by Slack `conversations.create`, and focused tests; Discord CLI wiring can either land via #65065 or be superseded while keeping provider-specific behavior inside the Discord plugin boundary.
