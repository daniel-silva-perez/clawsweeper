## Summary

Current main still does not implement the clock-skew guard proposed by this PR. `isTaskDue` treats only `undefined` as immediately due and otherwise compares `nowMs - lastRunMs >= intervalMs`, so a future `lastRunMs` remains not due. The PR is still a narrow, relevant bug-fix candidate, and the provided diff only touches the heartbeat helper and focused tests.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this narrow behavior change, or an equivalent patch, by treating future heartbeat task `lastRunMs` values as due in `src/auto-reply/heartbeat.ts`, preserving focused `isTaskDue` regression coverage, then running the targeted heartbeat tests and the appropriate changed-lane gate before merge.
