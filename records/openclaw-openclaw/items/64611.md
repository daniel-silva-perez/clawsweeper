## Summary

Keep open. Current main still lacks the heartbeat task clock-skew guard, so a future `lastRunMs` can make a task look not due and skip heartbeat execution; the PR remains a narrow, relevant bug fix with focused tests and no evident security-sensitive surface change.

## What This Changes

This PR changes heartbeat task due calculation to treat a future `lastRunMs` as due and adds unit coverage for never-run, elapsed, not-elapsed, and clock-skew cases.

## Best Possible Solution

Heartbeat task due checks should treat future last-run timestamps as due, preserve normal elapsed and not-elapsed behavior, and keep the regression locked in with focused `isTaskDue` unit coverage.
