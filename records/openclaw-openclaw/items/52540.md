## Summary

Keep open. Current main still contains the bundled 1Password skill guidance reported here: it requires tmux for all `op` commands and forbids direct `op` calls, while the same skill reference explicitly documents macOS desktop app integration and `op vault list` as the normal integrated sign-in path. The latest release tag also still contains the tmux-only wording, so this is not implemented or obsolete.

## Best Possible Solution

Keep this open and update `skills/1password/SKILL.md` to make the workflow auth-method aware: direct exec for macOS desktop app integration, tmux only where it is actually appropriate for an interactive/session-token flow, and clear guidance for service account or API-token setups. Then review adjacent 1Password guidance for consistency without weakening the separate release-maintainer npm authentication safety note.
