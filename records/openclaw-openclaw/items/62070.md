## Summary

Keep this PR open. Current main still has the image sanitization gap and the lifecycle-error streaming gap that the PR targets, and linked #59913 is still the active bug report. The PR may need adaptation to current main, but it is not obsolete or already implemented.

## Best Possible Solution

Keep this PR open as a valid implementation candidate for #59913. The best path is to rebase/adapt it to current main, sanitize `existingImages` on the no-ref path in `detectAndLoadPromptImages`, add focused regression coverage for that path, and surface exactly one generic error content chunk before SSE finalization when lifecycle `error` arrives without assistant deltas.
