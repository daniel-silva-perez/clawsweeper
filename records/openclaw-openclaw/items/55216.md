## Summary

Keep open. Current main still matches the central Telegram album bug: recoverable media-group download failures are logged and skipped, then the remaining media are sent onward with no user-visible warning. Single-media failures do send a Telegram warning, so this is a real behavior gap rather than an intentional no-notification policy.

## Best Possible Solution

Implement this in the Telegram plugin by tracking failed recoverable downloads and `null` media results inside `processMediaGroup`, then send a concise user-visible Telegram warning such as “Received N of M images; K could not be fetched” while still processing the successful media. Add focused tests for private/group media groups and channel posts that assert both the partial album reaches the agent and the skipped item is reported. Treat configurable media-group flush timing as a separate batching follow-up unless maintainers want to broaden the scope.
