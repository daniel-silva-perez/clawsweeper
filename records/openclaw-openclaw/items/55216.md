## Summary

Keep open. Current main still matches the central Telegram album bug: recoverable media-group download failures are logged and skipped, then the remaining media are sent onward with no user-visible warning. The related reports are adjacent but not superseding: #40842/#40903 cover polling-offset behavior, and #39343 covers broader batching work, not skipped-item notification for Telegram albums.

## What This Changes

The issue requests that Telegram media groups notify users when one or more album images fail to download instead of silently sending fewer images to the agent.

## Best Possible Solution

Implement the behavior inside the Telegram plugin: count recoverable failures and null media results during processMediaGroup, process the successfully downloaded media as today, and send a concise Telegram warning such as “Received N of M images; K could not be fetched” for the skipped items. Keep media-group timer configurability and deeper partial-file validation as separate follow-ups unless maintainers choose to broaden the scope.
