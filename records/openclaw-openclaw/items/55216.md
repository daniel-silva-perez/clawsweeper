## Summary

Current main still has the reported Telegram album behavior: recoverable media-group download failures are logged and skipped, then the remaining media are forwarded without a user-visible warning. The adjacent gateway-level batching request is broader and does not supersede this specific notification gap.

## What This Changes

The issue requests a Telegram album warning when one or more media-group images fail to download and are skipped.

## Best Possible Solution

Implement the fix in the Telegram plugin by counting recoverable exceptions and null media results during `processMediaGroup`, preserving successful album media processing, and sending one concise Telegram warning that states how many album items were skipped.
