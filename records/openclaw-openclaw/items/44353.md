## Summary

Keep this issue open. Current main already has the broad provider-error fallback loop, and Bedrock has provider-level failover classification hooks, but the reported subagent config shape still loses `agents.defaults.subagents.model.fallbacks` before the fallback runner builds candidates. The remaining fix is subagent fallback propagation, not the generic fallback loop.

## Best Possible Solution

Keep the issue open and narrow it to subagent fallback propagation. Preserve fallbacks from `agents.defaults.subagents.model.fallbacks` and `agents.list[].subagents.model.fallbacks` for automatic native subagent runs, while keeping explicit `sessions_spawn.model` overrides strict. Add focused regression coverage proving a spawned subagent with a Bedrock-style primary attempts a configured cross-provider fallback after a provider-level error.
