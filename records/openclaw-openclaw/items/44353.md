## Summary

Keep this issue open. Current main has the broad provider-error fallback loop, and Bedrock inference-profile discovery exists, but the exact reported subagent configuration still drops `agents.defaults.subagents.model.fallbacks` before `runWithModelFallback` builds its candidate chain.

## Best Possible Solution

Keep this issue open and narrow it to subagent fallback propagation. The best fix is a generic core change that preserves fallbacks from `agents.defaults.subagents.model.fallbacks` and `agents.list[].subagents.model.fallbacks` for automatic native subagent runs, while keeping explicit `sessions_spawn.model` overrides strict. Add focused regression coverage proving a spawned subagent with a Bedrock-style primary attempts a configured cross-provider fallback after a provider-level error.
