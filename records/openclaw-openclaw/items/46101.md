## Summary

Keep open. Current main still has the documented mixed heartbeat scoping rule, and `openclaw doctor` still lacks a targeted warning for configs where `agents.defaults.heartbeat` is shadowed by explicit per-agent heartbeat blocks. The broader inheritance policy remains in #41879, but this report is a narrow diagnostics gap with a bounded fix.

## What This Changes

Add an `openclaw doctor` diagnostic when `agents.defaults.heartbeat` is configured, at least one listed agent has an explicit heartbeat block, and other listed agents lack explicit heartbeat blocks.

## Best Possible Solution

Add a narrow, non-mutating doctor config-health warning that detects mixed `agents.defaults.heartbeat` plus explicit per-agent heartbeat blocks and names listed agents without explicit heartbeat blocks as not scheduled under the current rule. Leave scheduler inheritance behavior and broader policy changes to #41879.

## Reproduction Assessment

Yes. The source and tests provide a high-confidence reproduction path: configure `agents.defaults.heartbeat`, list `main` without `heartbeat`, and list `ops` with `heartbeat`; current code reports `main` disabled and schedules only `ops`.

## Solution Assessment

Yes. A doctor warning is the narrowest maintainable fix for this report because docs and tests treat the mixed scoping rule as intentional; changing that rule belongs in the broader heartbeat inheritance tracker.
