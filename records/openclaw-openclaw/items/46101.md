## Summary

Keep open. Current main still documents and implements the reported heartbeat scoping behavior, and I did not find a doctor/startup warning for agents that lose heartbeats when another agent has an explicit heartbeat block.

## Best Possible Solution

Keep this open unless maintainers choose to make #49613 the sole canonical behavior-change thread. The concrete fix here belongs in doctor/config health: when `agents.defaults.heartbeat` exists and any `agents.list[]` entry has `heartbeat`, warn for listed agents without explicit heartbeat blocks, with coverage for the main/ops scenario from the report. If heartbeat inheritance is changed instead, update docs/tests and close this as superseded by that behavior fix.
