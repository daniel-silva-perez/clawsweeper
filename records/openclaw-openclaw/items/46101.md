Current main still has the reported heartbeat scoping behavior and I did not find a doctor/startup warning for agents that lose inherited heartbeats when another agent has an explicit heartbeat block. The broader heartbeat-policy issue #41879 remains open, but this narrower diagnostics request is still actionable on its own.

Keep this issue open and add a narrow `openclaw doctor` config-health warning: when `agents.defaults.heartbeat` is configured and at least one `agents.list[]` entry has an explicit `heartbeat` block, warn for listed agents without explicit heartbeat blocks and explain that only agents with explicit blocks will run heartbeats under the current scoping rule. Do not change heartbeat scheduling or inheritance here; leave that broader policy to #41879.
