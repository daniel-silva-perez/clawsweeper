## Summary

Keep open. Current main still has the mixed per-agent heartbeat scoping behavior, and the doctor preview warning path still does not report configs where `agents.defaults.heartbeat` is shadowed by one or more explicit `agents.list[].heartbeat` blocks. The broader inheritance policy remains related to #41879, but this report is a narrow diagnostics gap.

## What This Changes

Add an `openclaw doctor` warning when `agents.defaults.heartbeat` is configured, at least one listed agent has an explicit `heartbeat` block, and other listed agents lack one.

## Best Possible Solution

Add a non-mutating doctor config-health warning that names listed agents missing explicit heartbeat blocks when defaults exist and at least one peer has an explicit heartbeat block. Keep scheduler inheritance behavior unchanged here and leave any policy change to the broader heartbeat inheritance tracker.

## Reproduction Assessment

Yes. A high-confidence reproduction is a config with `agents.defaults.heartbeat`, `main` listed without `heartbeat`, and `ops` listed with `heartbeat`; current source and tests show `main` is disabled while `ops` runs.

## Solution Assessment

Yes. A doctor warning is the narrowest maintainable fix for this report because current docs and tests encode the mixed scoping rule; changing scheduling semantics belongs in the broader policy issue.
