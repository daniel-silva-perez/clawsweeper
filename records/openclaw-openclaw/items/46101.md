## Summary

Current main still has the reported heartbeat scoping behavior and still lacks a doctor warning for agents that lose inherited heartbeats when another agent has an explicit heartbeat block. The related broader heartbeat-policy issue #41879 remains open, but this request is a narrower, actionable diagnostics fix rather than a duplicate close candidate.

## Best Possible Solution

Keep this open and add a narrow `openclaw doctor` config-health warning: when `agents.defaults.heartbeat` is configured and at least one `agents.list[]` entry has an explicit `heartbeat` block, warn for listed agents without explicit heartbeat blocks and explain that only agents with explicit blocks will run under the current scoping rule. Do not change heartbeat scheduling; leave broader policy changes to #41879.
