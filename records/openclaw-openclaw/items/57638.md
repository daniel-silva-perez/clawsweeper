## Summary

Keep open. Current main still supports per-job cron model, fallbacks, delivery, and lightweight context controls, but it does not implement the requested fleet-wide `cron.defaults` config surface for `model`, `delivery`, or `contextTokens`. The related #9155 is a narrower open tracker for delivery defaults only, so it does not supersede this broader request.

## Best Possible Solution

Add a first-class `cron.defaults` core config surface if maintainers want this feature: update `CronConfig`, zod/generated config schema metadata, labels/help, docs, and tests; apply defaults only to `agentTurn` jobs when the corresponding explicit job field is absent; preserve explicit job fields as highest precedence; keep `systemEvent` jobs unaffected; add or define the missing `contextTokens` cron job contract if explicit per-job precedence is desired; and coordinate the delivery portion with open #9155.
