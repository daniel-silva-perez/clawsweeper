## Summary

Keep open. Current main supports per-job cron model, fallback, delivery, and lightweight-context knobs, but it does not implement the requested fleet-wide `cron.defaults` surface for `model`, `delivery`, or `contextTokens`. Related #9155 remains useful for the narrower delivery-default part, but this issue is broader and still actionable in core.

## Best Possible Solution

If maintainers want this feature, add a first-class `cron.defaults` config surface in core: update `CronConfig`, zod/generated schema metadata, docs, and tests; apply defaults only to `agentTurn` cron jobs when the corresponding per-job payload/delivery/context field is absent; preserve explicit job fields as highest precedence; keep `systemEvent` jobs unaffected; and coordinate the delivery portion with the narrower #9155 tracker.
