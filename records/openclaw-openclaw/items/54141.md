## Summary

Keep open. Current main still has the reported auth-profile audit false positive, and the PR is useful but incomplete because its `inlineRef` guard still misses dual-field profiles where an explicit `keyRef` or `tokenRef` coexists with an inline `$VAR` or `${VAR}` value.

## What This Changes

The PR adds `$VAR` env SecretRef shorthand parsing, adjusts auth-profile audit handling for inline env refs, and adds parser/audit regression tests.

## Best Possible Solution

Revise this PR so auth audit classifies the inline credential value itself before the plaintext fallback, even when an explicit sibling ref is present, while preserving `PLAINTEXT_FOUND` for real leftover credential strings and keeping config-reporting semantics coordinated with #64344.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Classify inline auth env refs before plaintext fallback:** `src/secrets/audit.ts:303-305`
  - body: `resolveSecretInputRef` returns `inlineRef: null` when `keyRef` or `tokenRef` is present, so this guard only skips plaintext for env refs without an explicit sibling ref. A profile with `keyRef` plus `key: "$OPENAI_API_KEY"` or `"${OPENAI_API_KEY}"` still reaches the plaintext fallback and reports `PLAINTEXT_FOUND`; check the inline value itself before treating it as plaintext while preserving the explicit-ref-plus-real-plaintext case.
