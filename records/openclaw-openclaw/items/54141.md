## Summary

Keep this PR open. Current main still does not implement the PR’s intended behavior: `$VAR` shorthand is not parsed as a SecretRef, and auth-profile inline refs still fall through to `PLAINTEXT_FOUND`. The linked bug #53998 is still open, and the related #64344 is also open and only partially overlaps rather than superseding this PR.

## Best Possible Solution

Keep this PR open and review it as an implementation candidate for #53998, or deliberately fold its `$VAR` shorthand plus auth-profile audit tests into a single canonical secrets-audit fix. Before landing, maintainers should preserve the latest explicit-ref plaintext guard and cover `$VAR`, `${VAR}`, and explicit-ref-plus-leftover-plaintext cases in tests.
