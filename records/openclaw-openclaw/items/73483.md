## Summary

Keep this PR open. Current main still exposes `asVoice` in the message tool schema, but the shared core send path does not read it or forward `audioAsVoice` through `handleSendAction`, `executeSendAction`, `sendMessage`, or gateway `send`. Lower outbound delivery already has `audioAsVoice` support, so the PR is a plausible focused fix for the linked #73448 rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open and review it as the implementation candidate for #73448. The best path is to land an additive pass-through for `asVoice`/`audioAsVoice` across the message tool, CLI, shared outbound runner, direct `sendMessage`, gateway `send`, protocol schema, generated Swift models, docs, and regression tests, with no unrelated infrastructure or dependency changes.
