## Summary

Keep this PR open. Current main still exposes `asVoice` in the message tool schema, but the shared send path does not read it or preserve `audioAsVoice` through the core, CLI/gateway, or generated protocol surfaces. The PR remains a focused implementation candidate for open #73448 rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate for #73448. The best path is to land an additive `asVoice`/`audioAsVoice` pass-through across the message tool, CLI, shared outbound runner, direct `sendMessage`, gateway `send`, protocol schema, generated Swift models, docs, and focused regression tests, with no unrelated infrastructure or dependency changes.
