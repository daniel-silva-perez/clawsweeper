## Summary

Keep open. Current main has a `chat.history` path that can persist generated images as managed media blocks, but the live Gateway `chat` WebSocket event path still constructs delta/final assistant messages from text buffers only. The report is fresh and plausibly describes that live push surface, so it should not be auto-closed as implemented.

## Best Possible Solution

Clarify the Gateway WebSocket media contract, then either add an additive media-bearing projection for `chat` final events in `src/gateway/server-chat.ts` or document the history-refresh contract. If implementing, add a focused regression covering an `image_generate`/MiniMax-style media reply through `chat.send`, asserting both `chat.history` and the live final `chat` event expose usable image information without leaking raw base64.
