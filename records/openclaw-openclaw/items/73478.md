## Summary

Keep open. Current main still has the Gateway WebSocket live-chat media gap: agent assistant events can carry generated-image `mediaUrls`, but the `chat` delta/final bridge in `src/gateway/server-chat.ts` projects only buffered text; the existing managed-image projection is limited to the non-agent `chat.send` completion/history path.

## What This Changes

The issue requests that Gateway WebSocket chat output include generated-image information for MiniMax image-generation replies instead of returning only assistant text.

## Best Possible Solution

Add generic media-aware projection for agent-run Gateway `chat` final events when assistant stream/final data includes generated-image media, reusing the managed outgoing image/block machinery already used by `chat.send` history and avoiding raw base64 or MiniMax-specific behavior.

## Reproduction Assessment

Yes. A focused source-level reproduction can register a chat run, emit an assistant event with `data.text` plus `data.mediaUrls` through `createAgentEventHandler`, then emit lifecycle end; current `chat` delta/final payloads will contain only text content.

## Solution Assessment

Yes. A narrow Gateway live-event fix is the best maintainable direction because the agent event stream already preserves `mediaUrls` and the Gateway already has managed media projection helpers; a docs-only answer would conflict with the documented media-attachment behavior.
