## Summary

Keep open. Current main still has a concrete Gateway WebSocket behavior gap: assistant agent events can carry media URLs, and WebChat already has a managed outgoing image path, but the live `chat` projection in `src/gateway/server-chat.ts` emits only text blocks for delta/final messages.

## What This Changes

The issue asks Gateway WebSocket `chat`/`chat.history` output to include generated image information for a MiniMax image-generation reply instead of returning only assistant text.

## Best Possible Solution

Add a generic Gateway agent-event media projection that carries assistant `mediaUrls` into WebSocket `chat` final output and persisted WebChat history through the existing managed outgoing image attachment machinery, without MiniMax-specific branching.

## Reproduction Assessment

Yes. A focused source-level reproduction can register a chat-linked run, emit an assistant event with `data.text` plus `data.mediaUrls`, then emit lifecycle end; current `src/gateway/server-chat.ts` will broadcast text-only `chat` delta/final payloads.

## Solution Assessment

Yes. The narrowest maintainable repair is to reuse or factor the existing managed outgoing media display path for agent-event chat projection; provider-specific MiniMax handling or a docs-only change would leave the Gateway contract gap intact.
