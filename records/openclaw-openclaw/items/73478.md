## Summary

Keep open. Current main still has the Gateway WebSocket live-chat media gap: agent assistant events can carry `mediaUrls`, but the `chat` delta/final bridge in `src/gateway/server-chat.ts` projects only buffered text, while the media-aware projection exists only in the non-agent `chat.send` completion path.

## What This Changes

The issue asks for Gateway WebSocket `chat` events from `chat.send` image-generation runs to include generated image information instead of text-only assistant content.

## Best Possible Solution

Add a media-aware projection for agent-run Gateway `chat` final events when assistant stream/final data includes generated-image media. Reuse the managed outgoing image URL/block path already used by `chat.send` completion/history, avoid raw base64 in broadcasts, and keep the fix generic to assistant media rather than MiniMax-specific.

## Reproduction Assessment

Yes. A focused source-level reproduction can register a chat run, emit an assistant event with `data.text` plus `data.mediaUrls` through `createAgentEventHandler`, then emit lifecycle end; current `chat` delta/final payloads will contain only text content.

## Solution Assessment

Yes. A narrow Gateway live-event fix is the best maintainable direction because the agent event stream already preserves `mediaUrls` and the Gateway already has managed media projection helpers; a docs-only answer would need an explicit maintainer decision that live events are text-only notifications.
