## Summary

Keep this issue open. Current main still matches the reported iMessage bug: blank media-only sends synthesize `<media:image>` / `<media:${kind}>`, then pass that value as outbound JSON-RPC `text` with the media file. The linked fix PRs #40341 and #46343 were closed unmerged, and the latest release tag still has the same behavior.

## Best Possible Solution

Keep this issue open. The best fix belongs in `extensions/imessage/src/send.ts`: stop synthesizing visible `<media:...>` fallback text for outbound media-only sends, preserve real captions and `replyToId`, keep the guard that rejects sends with neither text nor media, add a direct regression test asserting blank text plus media sends RPC `text: ""` with `file`, and adjust iMessage deliver/echo/self-chat tests so dedupe does not depend on a visible outbound placeholder.
