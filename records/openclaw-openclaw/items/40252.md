## Summary

Keep this issue open. Current main still synthesizes `<media:image>` / `<media:${kind}>` for blank iMessage media sends and sends that synthesized value as the JSON-RPC `text` field. The latest release tag has the same implementation, and the two related fix PRs are closed unmerged.

## Best Possible Solution

Keep this issue open. The best fix belongs in `extensions/imessage/src/send.ts`: preserve real captions, reply metadata, and the no-text/no-media guard, but stop substituting `<media:...>` into outbound media-only `text`. Add a direct regression test for blank text plus media sending RPC `text: ""` with `file`, and adjust iMessage deliver/echo/self-chat tests so dedupe does not depend on a visible outbound placeholder.
