## Summary

Current main still matches the reported iMessage bug: media-only sends synthesize `<media:image>` / `<media:${kind}>` and pass that value as the outbound JSON-RPC `text` field with the attachment. The related fix PRs #40341 and #46343 were closed unmerged, so there is no shipped implementation to close against.

## Best Possible Solution

Keep this issue open. The best fix belongs in `extensions/imessage/src/send.ts`: stop synthesizing visible `<media:...>` fallback text for outbound media-only sends, preserve user captions and `replyToId`, keep the guard that rejects sends with neither text nor media, add a direct regression test that blank text plus media sends RPC `text: ""` with `file`, and adjust deliver/echo/self-chat tests so dedupe does not depend on visible outbound placeholder text.
