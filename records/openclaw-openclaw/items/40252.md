## Summary

Keep this issue open. Current main still turns blank iMessage media sends into `<media:image>` / `<media:${kind}>`, then forwards that synthesized value as the JSON-RPC `text` field. The related fix PRs #40341 and #46343 are closed unmerged, and the latest release tag has the same send implementation.

## Best Possible Solution

Keep this issue open. The best fix belongs in `extensions/imessage/src/send.ts`: preserve real captions, keep `replyToId` and the no-text/no-media guard, but stop substituting `<media:...>` into outbound media-only `text`. Add a direct regression test for blank text plus media sending RPC `text: ""` with `file`, and update iMessage deliver/echo/self-chat tests so dedupe does not depend on a visible outbound placeholder.
