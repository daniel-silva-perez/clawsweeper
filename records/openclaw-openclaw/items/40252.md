## Summary

Keep this issue open. Current main still synthesizes `<media:image>` / `<media:${kind}>` for blank iMessage media sends, sends that synthesized value as the JSON-RPC `text` field, and the latest release tag has the same implementation. The related fix PRs were closed unmerged, so there is no implemented or canonical merged fix to close against.

## Best Possible Solution

Keep this issue open. The best fix belongs in `extensions/imessage/src/send.ts`: preserve real captions, reply metadata, markdown/directive stripping, and the no-text/no-media guard, but stop substituting `<media:...>` into outbound media-only `text`. Add a direct regression test that blank text plus media sends RPC `text: ""` with `file`, then adjust iMessage deliver/echo tests so dedupe does not rely on a visible outbound placeholder.
