## Summary

Keep open. Current main still lacks a Claude Code-discoverable Slack MCP/tool surface and does not implement the requested Slack admin/app manifest actions, while the discussion correctly frames this as a product and security design decision rather than a narrow cleanup.

## What This Changes

Request to expose Slack channel actions to Claude Code/ToolSearch and add gated Slack admin operations for channel, user, and app manifest management.

## Best Possible Solution

Settle the Claude Code exposure contract first, then implement Slack admin as a separate, explicitly opt-in action group with documented token semantics, least-privilege scope guidance, SecretRef-compatible config, owner-aware gating, and focused tests for both the MCP/tool bridge and Slack action runtime.
