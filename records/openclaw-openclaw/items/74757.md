## Summary

Keep open. Current main still has Slack channel/message actions and user-token read plumbing, but no Slack-specific Claude/MCP tool exposure or requested admin/app-manifest action group; the remaining work crosses harness architecture and high-privilege Slack token policy, so it needs maintainer direction rather than cleanup closure.

## What This Changes

The issue requests making Slack actions discoverable to Claude Code/ToolSearch and adding Slack admin actions for channels, members, user lookup, and app manifests with explicit token plumbing.

## Best Possible Solution

Define the canonical Slack exposure path first, likely by bridging channel-owned Slack actions into an explicit MCP/bundle surface or by suppressing unreachable Slack skill docs for Claude Code sessions, then add a separately gated Slack admin action group with documented token semantics and tests.
