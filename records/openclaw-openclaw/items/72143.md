## Summary

Keep open. Current main still exposes the internal resume-failure prefix for denied and other non-success direct fallbacks, and the session resume path still performs a single `agent` gateway call before falling back.

## What This Changes

The issue requests retry/backoff for exec approval follow-up session resume, silent direct fallback without the internal resume-failure prefix, and internal-only sanitized logging.

## Best Possible Solution

Land a focused agents change that removes the resume-failure prefix from every direct fallback delivery path, adds bounded retry/backoff around the session follow-up call, and records sanitized internal diagnostics only after retries are exhausted.

## Reproduction Assessment

Yes. The existing unit test at `src/agents/bash-tools.exec-approval-followup.test.ts:199` mocks a failed session resume and currently expects the denied Telegram fallback to include the internal prefix.

## Solution Assessment

Yes. The requested direction is the narrow maintainable fix for the observed bug; it should stay localized to the exec approval follow-up helper and tests rather than becoming a broader delivery-policy rewrite.
