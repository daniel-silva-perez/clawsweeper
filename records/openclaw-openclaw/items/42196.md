## Summary

Keep this issue open. Current main still treats `openclaw cron add --message ...` as an isolated agent-turn job when `--agent` is omitted, forwards `agentId: undefined`, stores the job unpinned, and resolves execution through the default-agent path. The linked implementation PR #42245 remains open and unmerged with closing syntax for this issue, so this should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review PR #42245 or land an equivalent focused fix. The implementation should either require `--agent` or warn on stderr for agent-turn `openclaw cron add` jobs that omit it, keep system-event jobs quiet, preserve explicit `--agent` and intentional session routing behavior, and add regression coverage in `src/cli/cron-cli.test.ts`.
