## Summary

Keep open. Current main still allows `openclaw cron add` agent-turn jobs without `--agent`, forwards `agentId: undefined`, stores the job unpinned, and resolves execution through the default agent path. Live PR #42245 remains open with closing syntax for this issue, so this should stay open until that PR or an equivalent fix merges.

## Best Possible Solution

Review, refresh, and land PR #42245, or land an equivalent focused change in `src/cli/cron-cli/register.cron-add.ts` that warns on stderr when an agent-turn `cron add` omits `--agent`, preserves explicit `--agent` and `--session-key` behavior, keeps system-event jobs quiet, and adds regression coverage for omitted and explicit agent cases. Close this issue only after that implementation merges or maintainers choose a different shipped behavior.
