## Summary

Keep open. Current main still allows `openclaw cron add --message ...` without `--agent`, forwards `agentId: undefined`, stores the cron job unpinned, and resolves execution through the default-agent path. PR #42245 remains open and unmerged as the linked implementation candidate for this issue.

## Best Possible Solution

Keep this issue open and review, refresh, and land PR #42245 or an equivalent focused change. The fix should warn on stderr, or intentionally require `--agent`, for agent-turn `openclaw cron add` jobs that omit `--agent`; keep system-event jobs quiet; preserve explicit `--agent` and chosen `--session-key` behavior; and add regression coverage in `src/cli/cron-cli.test.ts`. Close this issue only after the implementation merges or maintainers explicitly choose a different shipped behavior.
