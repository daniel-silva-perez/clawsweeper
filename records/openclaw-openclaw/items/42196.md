## Summary

Keep open. Current main still accepts `openclaw cron add` for agent-turn jobs without `--agent`, sends `agentId: undefined` through the CLI/protocol/service path, and resolves omitted cron agent IDs to the configured default agent at run time. The provided GitHub context also shows open PR #42245 as an unmerged implementation candidate for the warning, so this issue should not be closed before that PR or an equivalent fix lands.

## Best Possible Solution

Review and refresh PR #42245, or apply an equivalent focused change in `registerCronAddCommand`: warn when an agent-turn `cron add` omits `--agent`, keep system-event jobs quiet, write the warning to stderr, and add regression coverage for omitted `--agent`, explicit `--agent`, and `--session-key` without `--agent`. Close this issue only after that implementation merges or maintainers explicitly choose a different shipped behavior.
