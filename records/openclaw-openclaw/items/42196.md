## Summary

Keep open. Current main still allows `openclaw cron add` agent-turn jobs without `--agent`, forwards `agentId: undefined`, stores the missing agent id, and resolves the job to the configured default agent at run time. The provided GitHub context also shows open PR #42245 with closing syntax for this issue, so the issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Review and refresh PR #42245, or land an equivalent focused change in `registerCronAddCommand`: warn on stderr when an agent-turn `cron add` omits `--agent`, keep explicit `--agent` and `--session-key` behavior intact, avoid warning for system-event jobs, and add regression coverage for omitted and explicit agent cases. Close this issue only after that implementation merges or maintainers choose a different shipped behavior.
