## Summary

Keep open. Current main still allows `openclaw cron add` for agent-turn jobs without `--agent`, passes `agentId` as omitted, and the Gateway resolves that missing agent to the configured default agent. The provided GitHub context also shows open PR #42245 as an unmerged implementation candidate with closing syntax for this issue, so the issue should not be closed before that work lands or maintainers choose a different behavior.

## Best Possible Solution

Review and refresh PR #42245, or apply an equivalent focused change in `registerCronAddCommand`: warn when an agent-turn `cron add` omits `--agent`, keep system-event jobs quiet, write the warning to stderr, and add regression coverage for omitted `--agent`, explicit `--agent`, and `--session-key` without `--agent`. Close this issue only after that implementation merges or maintainers explicitly choose a different shipped behavior.
