## Summary

Keep open. Current main still lacks the PR's ReadStream cleanup in both Feishu string-path upload helpers, and the live PR remains a focused one-file candidate fix rather than obsolete work.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a cleaned-up variant that destroys only ReadStreams created from string-path inputs in `uploadImageFeishu` and `uploadFileFeishu`, preserves Buffer behavior, and adds focused rejection-path regression tests. If maintainers no longer want string-path helper support, remove that contract explicitly instead of leaving the leak-prone branch in place.
