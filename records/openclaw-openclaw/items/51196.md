## Summary

Keep open. Current main still lacks the PR's Feishu ReadStream cleanup in both string-path upload helpers, and the PR remains a focused one-file fix rather than obsolete work.

## Best Possible Solution

Keep this PR open and land a cleaned-up variant with regression tests. The best path is to clean up only ReadStreams created from string-path inputs in `uploadImageFeishu` and `uploadFileFeishu`, preserve Buffer behavior, and add rejection-path tests for both helpers.
