## Summary

Keep open. Current main still does not implement Anthropic `context_management` passthrough/native support, context-editing or compaction replay semantics, `pause_after_compaction`, or a provider-defined `memory_20250818` tool path. The existing code supports Anthropic beta header merging through `anthropicBeta`/`context1m` and OpenAI-specific `context_management` injection, but the broader Anthropic integration requested here remains missing. Related #10213 and #48558 are narrower open slices, not superseding coverage for this broader request.

## Best Possible Solution

Keep this open for maintainer/product design in the bundled Anthropic provider. A complete path should add documented Anthropic model params for beta headers and `context_management`, validated payload shaping for compaction and context editing, stream/replay support for compaction blocks and `pause_after_compaction`, coverage for both direct Anthropic transport and wrapper paths, and either a provider-owned `memory_20250818` implementation or a generic provider-defined-tool SDK seam with safe filesystem handling.

## Risks / Open Questions

- Raw `anthropicBeta` header passthrough can cover header experiments only; without request-body validation and stream/replay handling, users still cannot safely use Anthropic compaction/context-editing semantics through OpenClaw.
