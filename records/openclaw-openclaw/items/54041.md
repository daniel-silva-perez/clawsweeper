## Summary

Keep open. Current main still does not implement the requested Anthropic context-management support: configured beta headers can be appended, but OpenClaw does not shape Anthropic `context_management` request bodies, handle compaction/context-editing stream metadata, preserve compaction blocks in replay, or pass through/execute the native `memory_20250818` tool. The related open issues are narrower or overlapping, not superseding, and the closed PR was unmerged.

## What This Changes

The issue requests opt-in Anthropic Memory Tool, server-side compaction, context editing, beta-header, and provider-defined tool passthrough support for Anthropic models.

## Best Possible Solution

Keep this issue open for bundled Anthropic provider design. The desired end state is a documented opt-in Anthropic context-management surface that validates and sends `context_management.edits`, appends the right beta headers only when needed, preserves compaction blocks through streaming and replay, records context-editing metadata, supports pause/resume behavior, and deliberately handles `memory_20250818` either as a provider-owned memory tool or through a generic provider-defined-tool SDK seam.
