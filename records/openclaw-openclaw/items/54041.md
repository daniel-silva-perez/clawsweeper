## Summary

Keep open. Current main still only exposes Anthropic header/service-tier/cache-related knobs and OpenAI-specific server-side compaction; it does not implement Anthropic `context_management` payload shaping, compaction/context-editing stream semantics, `pause_after_compaction`, or a `memory_20250818` provider-defined tool path. The production +1 reinforces that the remaining request is not header-only and needs provider/product design rather than cleanup closure.

## What This Changes

Request native or passthrough support for Anthropic beta context-management features, including Memory Tool, server-side compaction, context editing, and related beta headers/parameters.

## Best Possible Solution

Keep this open for bundled Anthropic provider design. The end state should add documented Anthropic model params and beta-header handling, validated `context_management.edits` payload shaping for compaction and context editing, stream/replay support for Anthropic compaction blocks and `pause_after_compaction`, and a deliberate design for `memory_20250818` as either a provider-owned tool implementation or a generic provider-defined-tool SDK seam.
