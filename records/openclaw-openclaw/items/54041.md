## Summary

Keep open. Current main still does not implement the requested Anthropic context-management support beyond configurable beta-header append: it does not shape Anthropic `context_management.edits`, preserve/stream compaction blocks, handle Anthropic `context_management.applied_edits`, support `pause_after_compaction`, or expose/execute `memory_20250818`. The related open issues are narrower or overlapping rather than superseding this broader provider-design request.

## What This Changes

The issue requests native or passthrough support for Anthropic beta context-management features: Memory Tool, server-side compaction, context editing, and the required Anthropic beta headers.

## Best Possible Solution

Keep this open for bundled Anthropic provider design. The desired end state is a documented opt-in Anthropic context-management surface that validates and sends `context_management.edits`, appends required beta headers only when needed, preserves compaction blocks through streaming and replay, records context-editing metadata, handles `pause_after_compaction` continuation behavior, and deliberately supports `memory_20250818` either as a provider-owned memory tool or through a generic provider-defined-tool SDK seam.
