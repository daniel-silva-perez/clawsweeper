## Summary

Keep open. Current main still preserves the key failure mode for Discord-bound agents with explicit or legacy messaging-profile tool policy: `exec` is not in the messaging profile, explicit profiles are preserved, and there is still no merged Discord regression proving an inbound Discord turn can execute `exec` and persist the expected JSONL transcript.

## Best Possible Solution

Keep this open until maintainers either ship a narrow migration or doctor diagnostic for legacy/explicit messaging-profile configs that expect runtime tools, or add a targeted Discord-bound regression showing an inbound Discord turn can execute `exec` and persist the JSONL session transcript. If messaging profiles are intentionally chat-only, add a clear user-facing diagnostic pointing users to `tools.profile: "coding"`, `tools.alsoAllow`, or `tools.exec`.
