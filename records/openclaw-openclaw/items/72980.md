## Summary

Keep open. Current main does not expose `OPENCLAW_SESSION_KEY` on the exec runtime path, so the PR is addressing a real missing capability, but the proposed placement has correctness issues around ambient env overrides and node-host analysis. The PR also overlaps open PR #40423, which proposes a broader runtime marker contract, so this needs maintainer review rather than cleanup closure.

## What This Changes

The PR injects `defaults.sessionKey` as `OPENCLAW_SESSION_KEY` into exec child environments and adds gateway tests for default injection, absence, and caller override behavior.

## Best Possible Solution

Define the runtime marker contract once, then land either a narrow exec-only implementation with explicit gateway/sandbox scope, ambient-env override handling, docs/security notes, and sandbox plus gateway coverage, or fold this into #40423's broader runtime marker work.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Override ambient session markers:** `src/agents/bash-tools.exec.ts:1615-1617`
  - body: The guard checks the fully merged env, so an inherited gateway env or sandbox env value for `OPENCLAW_SESSION_KEY` prevents `defaults.sessionKey` from being written. That violates the promised runtime-guaranteed current session key; only an explicit `params.env.OPENCLAW_SESSION_KEY` should win.
  - confidence: 0.9
- **[P2] Keep session markers out of node-host analysis:** `src/agents/bash-tools.exec.ts:1615-1617`
  - body: This block runs before the `host === "node"` branch. Node execution forwards only `requestedEnv`, while approval/allowlist analysis reads the full request env, so node commands can be analyzed with `OPENCLAW_SESSION_KEY` present and then run remotely without it. Gate the injection to gateway/sandbox paths.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: No dependency or CI supply-chain change was found, but the patch deliberately exposes the session key through subprocess environment data and needs maintainer acceptance of that contract.

Concerns:

- **[medium] Session key becomes subprocess environment data:** `src/agents/bash-tools.exec.ts:1615`
  - body: The new env var makes the agent session key available to affected child processes and descendants. Env values can be echoed, inherited, logged, or captured in diagnostics, so this should be documented and intentionally accepted before merge.
  - confidence: 0.88
