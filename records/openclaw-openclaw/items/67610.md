## Summary

Current main still uses a module-local sandbox backend registry, while plugin-owned sandbox backends are registered through the public plugin SDK path and resolved later by core. The same-author implementation candidate #67623 remains open and unmerged, so this issue should stay open for maintainer review.

## Best Possible Solution

Keep this issue open and review #67623, or a maintainer-authored equivalent, to move sandbox backend registrations to a shared process-global registry that is visible across plugin/core module instances. Add a regression test that registers a backend through `openclaw/plugin-sdk/sandbox` and verifies core sandbox context resolution can require that backend.
