## Summary

Keep this issue open. Current main still keeps sandbox backend registrations in a module-local Map, while plugin backends are registered through the public plugin SDK subpath and later resolved by core via requireSandboxBackendFactory. The provided GitHub context also shows a same-author open PR (#67623) for this exact bug, so this issue should stay open until that PR or a maintainer-approved equivalent is reviewed and resolved.

## Best Possible Solution

Keep this issue open and use #67623, or a maintainer-authored replacement, to move sandbox backend registrations to a process-global registry that survives plugin/core module instance splits. The fix should include a regression test that registers a backend through the plugin SDK path and verifies core requireSandboxBackendFactory can resolve it during sandbox context creation.
