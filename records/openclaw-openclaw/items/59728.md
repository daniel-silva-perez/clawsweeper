## Summary

Keep open. Current main has not implemented the requested scripts reorganization or README index: most tracked script files still live directly under `scripts/`, `scripts/README.md` is absent, and package, CI, docs, and release references still hard-code root-level script paths. This should not be queued as an automated cleanup because the requested all-at-once move spans CI, package contents, release checks, docs, and many tests, so the taxonomy and compatibility plan need maintainer agreement first.

## What This Changes

Request to reorganize the `scripts/` tree into categorized subdirectories, add a root and per-directory README index, and update package, CI, workflow, and docs references.

## Best Possible Solution

Define a maintainer-approved taxonomy first, then land the work in phases: add a root `scripts/README.md` index, move only agreed categories with compatibility wrappers where needed, update package/workflow/docs/test references together, and validate the touched CI/package/release surfaces.
