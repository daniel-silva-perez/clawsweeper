## Summary

Keep open. Current main still routes only chat/chat.side_result/agent events in the TUI and does not subscribe the TUI backend to session.message transcript updates; the issue is also paired with open PR #45417 using closing syntax, plus open maintainer PR #73402 for the safer transcript-subscription path.

## Best Possible Solution

Keep this issue open until a fix PR lands. The best implementation path is to wire the TUI backend to sessions.messages.subscribe/unsubscribe for the active session, handle session.message transcript updates in the TUI without duplicating local optimistic rows, unsubscribe on session switches/stop, and add focused TUI/gateway tests for external Telegram-style transcript appends, active-run deferral, and session filtering. Review/land #73402 or rework #45417 toward that path, then close this issue after merge.
