## Summary

Keep open. Current main still has the gateway transcript event surface but the bundled TUI does not subscribe to or route `session.message` updates, and there are open implementation PRs paired with this live-tail bug.

## What This Changes

The issue requests real-time TUI live-tailing for a Telegram-backed `--session` view, including inbound channel messages and agent responses without manual refresh or relaunch.

## Best Possible Solution

Land a focused TUI fix that subscribes to the active session's transcript stream, unsubscribes on session switch or shutdown, handles `session.message` only for the active session, and covers duplicate prevention plus active-run/history-reload behavior with regression tests.

## Reproduction Assessment

Yes. The issue gives a concrete command, and current source inspection provides a high-confidence code-path reproduction: gateway transcript events exist, but the TUI neither subscribes to nor handles them.

## Solution Assessment

Yes. The narrow maintainable fix is to use the existing `sessions.messages.subscribe` / `session.message` transcript path rather than widening generic chat-event fanout for external channel traffic.
