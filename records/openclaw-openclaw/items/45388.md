## Summary

Keep open. This issue is still paired with open unmerged implementation PR #45417 using closing syntax, and current main does not show the TUI consuming the existing transcript event path needed for live external-session updates.

## Best Possible Solution

Review and either land or supersede PR #45417. The best implementation path is for TUI to subscribe to the active session with `sessions.messages.subscribe` on connect/session switch, handle `session.message` with dedupe/history reconciliation, and keep external-channel hidden-run fanout boundaries intact. Close this issue only after that TUI-side live-tail path is merged.
