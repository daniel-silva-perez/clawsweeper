## Summary

Keep this PR open. Current main still has uncapped `.clobbered.*` snapshot writes, so the defensive cap is useful, but the proposed async implementation does not enforce a hard per-path cap under concurrent suspicious reads because it counts siblings and writes the snapshot in separate operations.

## What This Changes

The PR adds a 32-file per-config-path cap and single per-path warning around clobbered config snapshot writes in `src/config/io.ts` and `src/config/io.observe-recovery.ts`, plus async/sync regression coverage.

## Best Possible Solution

Land a hard per-path clobber snapshot cap in both config persistence paths using a per-config-path critical section or atomic bounded slot strategy, keep one bounded warning per path, preserve audit records after snapshots are skipped, and add concurrent async regression coverage before merge.

## Reproduction Assessment

Yes. Current main can be checked source-level by observing the direct `.clobbered.*` writes, and the PR’s remaining defect can be reproduced with parallel async suspicious reads against one config path after seeding a last-good backup.

## Solution Assessment

No. Counting existing siblings before a separate async write is not the narrowest reliable fix for a hard cap; the cap check and snapshot claim need to be serialized or made atomic per config path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Serialize the async cap check with snapshot creation:** `src/config/io.observe-recovery.ts:535-542`
  - body: `persistClobberedConfigSnapshot` counts existing siblings and then performs a separate async `writeFile`. If many suspicious reads for the same config path overlap, each call can observe `existing < 32` before any write lands and then create another timestamped snapshot, so the advertised cap can be exceeded by the concurrency level. Put the count/claim/write path behind a per-config-path lock or use atomic bounded slots, and mirror the fix in the parallel `src/config/io.ts` helper.
  - confidence: 0.88
