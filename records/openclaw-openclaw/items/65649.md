## Summary

Keep this PR open. Current main still writes `.clobbered.*` snapshots without a per-path cap in both config persistence paths, and the existing hash-and-reasons dedupe only suppresses identical suspicious payloads. The PR addresses a remaining disk-exhaustion guard; the prior repeated-warning review appears addressed in the PR context, while the later count/write concurrency review still needs maintainer attention before landing.

## Best Possible Solution

Keep the PR open. The best path is to finish review of the defensive cap, address the concurrent count/write race, and land a focused implementation in both `src/config/io.ts` and `src/config/io.observe-recovery.ts` with regression tests for async and sync paths, max file count, audit preservation, and one warning per path.
