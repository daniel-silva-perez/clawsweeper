## Summary

Keep this PR open. Current main still has the durable restore failure gap: a restore failure is swallowed after `restoreAttempted` is set, with no rollback of `subagentRuns` or retry reset. The PR is not obsolete, but it needs the existing review addressed because rollback should also cover `resumedRuns` before it is safe to merge.

## Best Possible Solution

Keep the PR open as candidate bug-fix work. The best path is to rebase it, extend the rollback to restore all retry-sensitive state (`subagentRuns`, `resumedRuns`, and any scheduled resume side effects if applicable), keep `restoreAttempted` retryable after transient failure, and add focused regression coverage for a partial restore/resume throw followed by a clean retry.
