## Summary

Keep this PR open. Current main still does not implement the PR's central update-path behavior: detached `openclaw update` restart verification calls `waitForGatewayHealthyRestart` without the Windows unknown-listener stale-PID fallback, while the regular `openclaw gateway restart` path already opts into that fallback. The PR still appears useful, but the provided PR file list and review discussion show it is not merge-ready as-is because it bundles unrelated workflow, config, Matrix/channel-contract, secrets-baseline, Tlon dependency, and lockfile changes with a supply-chain-sensitive dependency-resolution change.

## Best Possible Solution

Keep the PR open, but require a focused refresh before merge: rebase onto current main, keep only the detached update restart-health implementation plus targeted regression coverage, preserve the normal health-window delay before Windows unknown-listener fallback, and split or drop the unrelated workflow, config, Matrix/channel-contract, secrets-baseline, Tlon dependency, and lockfile changes. Any dependency or CI changes should be reviewed in a separate PR with install proof.
