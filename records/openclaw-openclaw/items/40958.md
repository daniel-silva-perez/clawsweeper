## Summary

Keep this PR open. Current main still does not implement the central update-path behavior: `openclaw update` verifies detached restart health without passing the Windows unknown-listener stale-PID fallback that the shared restart-health helper supports and normal daemon restart already uses. The PR is also not merge-ready as-is because it is stale and bundles unrelated config, channel-contract, workflow, Tlon dependency, lockfile, and secrets-baseline changes with security/supply-chain-sensitive dependency resolution churn.

## Best Possible Solution

Keep the PR open, but require a focused refresh before merge: rebase onto current main, keep only the update restart-health change plus targeted regression coverage, preserve the delayed Windows fallback semantics after the normal health window, and drop or split the unrelated config, workflow, Matrix/channel-contract, Tlon dependency, lockfile, and baseline changes. Any dependency-resolution or CI changes should be reviewed in their own PR with install proof.
