## Summary

Keep open. Current main still lacks the Windows unknown-listener stale-PID fallback in the detached `openclaw update` restart verification path, but this PR is not mergeable as-is because it mixes the useful update fix with an incorrect stale-PID overwrite, unrelated config/channel/workflow churn, and a concrete Tlon dependency supply-chain/build risk.

## What This Changes

The PR adds Windows detached restart health fallback handling and regression coverage for `openclaw update`, while also changing workflow smoke checks, Matrix channel contract tests, gateway remote config validation, Tlon dependency resolution, the lockfile, and `.secrets.baseline`.

## Best Possible Solution

Land a focused current-main fix for detached Windows update restart health that mirrors the daemon restart stale-listener recovery semantics, preserves an already observed stale-PID signal, and adds targeted update CLI regression coverage without dependency, config, workflow, Matrix, lockfile, or secrets-baseline churn.
