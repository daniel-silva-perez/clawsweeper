## Summary

Keep this PR open. Current main has already covered the exec default-host PATH failure and the cron force-run deadlock through newer designs, but the browser portion is still not implemented in the current `extensions/browser` layout: status has no layered diagnostics payload, CLI status does not merge Control UI/auth/origin checks, and non-rate-limit browser HTTP errors still reflect upstream response text directly.

## Best Possible Solution

Keep this PR open, but require a split or rebase. Drop the exec and cron changes already covered on main, then port the still-missing layered browser status diagnostics and bounded HTML upstream-error summarization into `extensions/browser`. Address the two existing review comments, keep config-derived diagnostics redacted and timeout-bounded, and verify with focused browser client/server/CLI tests plus a security review for reflected error text and config exposure.
