## Summary

Keep this PR open. Current main covers the exec PATH fallback and the cron force-run deadlock shape through newer `host=auto` and `cron-nested` behavior, but it still does not implement the PR’s browser layered status diagnostics or bounded HTML upstream-error summarization in the current `extensions/browser` plugin layout.

## Best Possible Solution

Keep this PR open, but require a split or rebase. Drop the exec and cron hunks already covered on current main, then port the still-missing browser layered status diagnostics and bounded HTML upstream-error summarization into `extensions/browser`, with focused browser client/server/CLI tests and resolution of the two existing review comments.

- Merging the PR as-is would target stale root `src/browser` and `src/cli` paths instead of the current `extensions/browser` plugin layout.
