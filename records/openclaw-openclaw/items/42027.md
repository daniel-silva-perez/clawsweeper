## Summary

Keep this PR open. Current main has already solved the exec default-host and cron deadlock shapes through newer mechanisms, but the PR’s browser status diagnostics and HTML upstream-error summarization are still not implemented in the current `extensions/browser` plugin layout.

## Best Possible Solution

Keep the PR open, but require a split/rebase. Drop the exec and cron hunks already covered by current main, then port the remaining browser work into `extensions/browser`: layered status diagnostics for control-ui/profile/CDP, bounded HTML upstream-error summarization, focused server/client/CLI tests, and fixes for the two existing review comments.
