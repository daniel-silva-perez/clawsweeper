## Summary

Keep this PR open. Current main covers the exec PATH fallback shape through `host=auto` resolving to gateway without a sandbox runtime, and covers the cron force-run deadlock shape through the newer `cron-nested` lane. The PR’s browser work is still not implemented on current main in the `extensions/browser` layout: `browser status` has no layered diagnostics payload/output, and browser HTTP failures still throw raw upstream response text for non-rate-limit errors.

## Best Possible Solution

Keep this PR open, but require a split or rebase: drop the exec and cron hunks already covered on current main, then port the still-missing browser layered status diagnostics and bounded HTML upstream-error summarization into `extensions/browser` with focused client/server/CLI tests and resolution of the two existing review comments.
