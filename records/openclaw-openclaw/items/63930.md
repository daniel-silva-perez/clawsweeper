## Summary

Keep open. Current main still lacks Anthropic advisor support and still drops or ignores the Anthropic server-side stream block family described in the report, while the same reporter has an open implementation PR for this exact work. This issue should stay open until that PR lands, is replaced, or maintainers choose a different provider/transport design.

## What This Changes

Request to add Anthropic advisor beta server-side tool support, preserve or display Anthropic server-side tool result blocks, and avoid crashing on unknown Anthropic stop reasons.

## Best Possible Solution

Land #64064 or a maintainer-approved replacement that keeps the owner boundary intact: core transport should handle Anthropic server-side result/display/replay semantics generically, while the bundled Anthropic plugin owns advisor config parsing, beta-header/tool injection, model validation, docs, and focused tests. Any intentionally deferred encrypted-content replay or cost-accounting behavior should be tracked as explicit follow-up work.
