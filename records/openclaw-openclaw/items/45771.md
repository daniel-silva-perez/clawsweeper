## Summary

Keep open. Current main has normalized provider usage visibility and subagent fan-out guards, but it still lacks the requested pace-aware autonomous-agent policy: no Anthropic inference response-header capture, projected pace tiers, persisted per-profile pace state, `agents.defaults.pace` config, or prompt/throttle behavior for non-interactive agent work.

## What This Changes

Add built-in pace-aware rate-limit tracking for autonomous agents using Anthropic usage windows, projected pace tiers, status/session visibility, and optional subagent throttling.

## Best Possible Solution

Define a provider-aware pace design before implementation: usage sources and freshness, per-profile persisted windows, observed request-cost calibration, structured status/session output, agent prompt context, notifications, and explicit subagent/non-interactive enforcement rules without hard-coding an Anthropic-only policy into unrelated core paths.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source/docs inspection of current main, which shows usage snapshots and subagent controls exist but the requested pace policy surfaces do not.

## Solution Assessment

Unclear as proposed. Header capture and status visibility are narrow and useful, but utilization-only projected tiers are likely insufficient; the better solution should combine provider-reported windows with observed request cost and clear enforcement UX.
