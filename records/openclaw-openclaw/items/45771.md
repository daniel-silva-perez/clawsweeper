## Summary

Keep open. Current main has adjacent provider usage-window visibility and subagent safety limits, but it does not implement the requested pace-aware autonomous-agent policy: no Anthropic inference response-header capture, no projected pace tiers, no `agents.defaults.pace` config, and no autonomous prompt/throttle policy for subagents or non-interactive work. The recent discussion also raises a real design question about utilization-only pace calculations miscalibrating across model and context-size costs.

## What This Changes

The issue requests built-in Anthropic usage-header capture, projected 5h/7d pace tiers, status and `session_status` display, and configurable autonomous-agent throttling based on pace.

## Best Possible Solution

Define a provider-aware pace design before implementation: model usage sources and freshness, persisted per-profile windows, observed cost-per-request calibration, structured status/session_status output, agent prompt context, notifications, and explicit subagent/non-interactive enforcement rules without hard-coding Anthropic-only policy into unrelated core paths.
