## Summary

Keep this open. Current main has adjacent provider usage visibility for Anthropic OAuth usage windows and shows normalized usage in status/session_status, plus subagent spawn guardrails, but it still does not implement built-in pace-aware autonomous-agent rate limiting: no Anthropic inference response-header capture, no persisted runtime pace state, no projected pace tiers/actions, no agents.defaults.pace config, no pace prompt injection/notifications, and no proactive subagent or non-interactive throttling. The Claude Usage Tracker and claude-pace prior art in the report are useful design inputs, and related open items #13615, #38775, and #71770 are adjacent but do not fully supersede this specific request.

## Best Possible Solution

Keep this open as a pace-aware autonomous-agent design item. The best path is to consolidate the design with #13615, #38775, and #71770, then decide whether pace state and enforcement are core-owned or exposed through a provider/plugin policy API. A complete design should cover Anthropic response headers versus usage polling, per-profile persistence, provider fallback behavior, status/session context, prompt injection, notifications, and explicit subagent/non-interactive throttling rules.
