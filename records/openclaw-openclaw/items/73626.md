## Summary

Keep open. Current main still allows a persisted Discord conversation binding to override the freshly resolved route, so a channel can keep using an old agent/session key after routing or account configuration changes. The related items in the provided context cover adjacent Discord lifecycle, DM, PluralKit, and macOS service issues, but none supersede this stale Discord runtime persona/routing mismatch.

## Best Possible Solution

Keep this issue open and fix the Discord ingress/conversation-binding path. Add a freshness check or route fingerprint for Discord current-conversation bindings, compare the stored target agent/session against the current effective route when channel/defaultAccount/agent bindings change, and ignore, unbind, or clearly diagnose stale normal channel bindings while preserving intentional ACP and plugin-owned bindings. Add focused tests for a stale agent:<old>:discord:channel:<id> binding when the current route resolves to a different agent/account.
