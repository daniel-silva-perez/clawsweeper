## Summary

Keep open. Current main still has a plausible stale Discord binding path: Discord preflight resolves the current configured route, but a persisted current-conversation binding can override the route session key and agent id from its stored targetSessionKey. Existing startup reconciliation is scoped to ACP thread bindings and does not cover generic/current Discord channel bindings or route/default-account drift. The related reports in the provided context cover adjacent Discord lifecycle, DM-key, PluralKit, and macOS service issues, but none supersede this exact stale runtime persona/routing mismatch.

## Best Possible Solution

Keep this issue open and fix it in the Discord ingress/conversation-binding path. The best direction is to add a freshness check or route fingerprint for Discord current-conversation bindings, compare the stored target agent/session against the current effective route when channel/defaultAccount/agent bindings change, and ignore, unbind, or clearly diagnose stale normal channel bindings while preserving intentional ACP and plugin-owned bindings. Add focused tests for a stale agent:<old>:discord:channel:<id> binding with a current route to a different agent/account.
