## Summary

Keep this issue open. Current main still lacks Slack App Home support: the Slack plugin enables the App Home messages tab but does not subscribe to `app_home_opened`, register a Home handler, call Slack `views.publish`, expose Home-tab config/actions, or document/test the setup. Open PR #73389 is the active implementation candidate, so the right cleanup path is review or land that PR rather than closing this tracker.

## What This Changes

Add bundled Slack App Home support by listening for `app_home_opened`, publishing a default Block Kit Home view with `views.publish`, and optionally allowing agent/config customization.

## Best Possible Solution

Keep this issue open as the canonical bundled Slack App Home tracker. Review and either land or close #73389; the implementation should stay in `extensions/slack`, add `app_home_opened` to setup/docs, register a monitor handler, publish a safe default Home view via Slack `views.publish`, and include focused tests. Broader agent-customized Home-tab actions/config should stay small and reviewed or be split into a follow-up.
