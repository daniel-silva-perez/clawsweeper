## Summary

Keep open. Current main still does not implement Slack App Home support: the Slack plugin has no `app_home_opened` monitor handler, no `views.publish` call, and no generated setup/docs subscription for the event. Open maintainer-labeled PR #73389 remains the active implementation path, so this issue should stay open rather than spawning a duplicate repair lane. 

## What This Changes

Add bundled Slack App Home support by subscribing to `app_home_opened`, publishing a default/customizable Home view with Slack `views.publish`, and updating setup/docs/tests accordingly.

## Best Possible Solution

Keep this as the canonical bundled Slack App Home tracker. The desired end state is a narrow `extensions/slack` implementation that subscribes to `app_home_opened`, registers a monitor handler, publishes a safe default Home view through `views.publish`, updates setup/docs/tests/changelog, and treats broader custom Home-tab actions or config as a scoped follow-up if needed.

## Solution Assessment
