Keep open. Current main still lacks Slack App Home event subscription, monitor handling, and `views.publish` behavior, while open PR #73389 is already the active maintainer-labeled implementation candidate, so this should not be closed or duplicated by a new repair job.

## What This Changes

The issue requests Slack App Home tab support by subscribing to `app_home_opened`, publishing a Home view through Slack `views.publish`, and optionally exposing customization for Block Kit content.

## Best Possible Solution

Keep this as the canonical bundled Slack App Home tracker.
