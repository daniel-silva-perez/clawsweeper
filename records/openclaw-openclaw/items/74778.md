## What This Changes

The PR updates the Bonjour advertiser watchdog, targeted advertiser tests, and Bonjour docs so ciao `probing` and fresh name/hostname conflict renames are treated as in-progress states before recovery.

## Best Possible Solution

Land a focused Bonjour plugin fix that treats ciao negotiation and fresh conflict renames as in-progress, preserves bounded disable behavior for genuinely stuck advertisers, and leaves broader bundled-plugin/default policy work to #74209 and WSL2 environment handling to #74820.

Yes. A unit-level reproduction on current main can hold the mocked ciao service in `probing`; the watchdog does not skip that state and reaches the non-announced re-advertise branch before eventual stuck recovery.

## Solution Assessment

Yes for this PR's scope. The patch stays inside the Bonjour plugin, follows ciao's state/event contract, adds targeted regression tests and docs, and does not mix in the broader default-plugin or WSL2 policy questions.
