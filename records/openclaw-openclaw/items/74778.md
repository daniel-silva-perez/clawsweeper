## Summary

Keep open. Current main fixed the sliding-window restart cap, but it still lets a `probing` ciao service fall through to the watchdog re-advertise path and has no conflict-settle grace after ciao rename events. The PR is a narrow follow-up for that remaining Bonjour watchdog behavior, and I found no blocking correctness or security finding in the diff.

## What This Changes

The PR updates the Bonjour advertiser watchdog to defer retries during ciao probing and fresh name/hostname conflict renames, with targeted tests, Bonjour docs, and a changelog entry.

## Best Possible Solution

Land a focused Bonjour plugin fix that treats ciao negotiation and fresh conflict renames as in-progress, preserves bounded disable behavior for genuinely stuck advertisers, and keeps the broader default bundled-plugin policy tracked separately in #74209.
