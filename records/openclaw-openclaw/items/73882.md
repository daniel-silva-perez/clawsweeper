## Summary

Current main still has the reported doctor warning path. `openclaw doctor` calls `doctor.memory.status` with `probe: false`; the gateway returns a skipped cached-readiness payload when no cached embedding probe exists; the CLI then coerces that payload into a checked/not-ready probe and `noteMemorySearchHealth` warns for key-optional providers such as `ollama`. This should stay open for a focused fix rather than close as implemented or stale.

## Best Possible Solution

Keep this open and fix the doctor memory probe contract narrowly: preserve `embedding.checked === false` from `doctor.memory.status`, treat skipped/inconclusive cached readiness as non-actionable for key-optional providers such as `ollama`, and continue warning for explicit checked failures or missing remote credentials. Add a regression test for `ollama` plus the skipped `memory embedding readiness not checked` payload.

## Clownfish Work Prompt

Observable bug: with `memory.search.provider: "ollama"`, normal `openclaw doctor` calls `doctor.memory.status` with `probe: false`; if no cached embedding readiness exists, the gateway returns `embedding.checked === false` with `memory embedding readiness not checked`, but the CLI currently coerces that into a checked not-ready probe and surfaces a warning. Expected boundary: keep `doctor.memory.status` fast/cached by default; do not add a live embedding ping to normal doctor.
