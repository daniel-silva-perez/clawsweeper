## Summary

Keep open. Current main still has the unguarded WebSocket ping continuation path this PR targets, so closing would leave the crash risk unresolved. The PR is useful, but its diff also bundles an unrelated Swift CI workflow; maintainers should either trim this PR to the focused crash fix or choose a cleaner canonical PR for the same bug.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to remove or split the added Swift workflow, then land a focused GatewayChannel.swift change that gates the sendPing continuation at most once. If maintainers prefer #51311 as the cleaner canonical implementation, close the other PR only after that choice is explicit or after the canonical fix lands on main.
