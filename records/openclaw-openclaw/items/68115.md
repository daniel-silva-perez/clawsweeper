## Summary

Keep open. Current main still lacks the requested first-class setup/onboarding hook for credentialed tool-only plugins: setup descriptors and setup runtime collection are provider/CLI-backend shaped, and the generic plugin config wizard does not collect sensitive API-key fields for newly installed tool plugins. The provided related item #61551 is a closed runtime hook-runner bug and does not track or supersede this SDK/setup API gap.

## What This Changes

The issue requests an additive global setup/onboarding surface so tool-only plugins can declare setup metadata, prompt for credentials during `openclaw setup`, and persist plugin-owned config without pretending to be providers.

## Best Possible Solution

Define a backward-compatible, manifest-first tool-plugin setup contract that lets plugins declare setup metadata cheaply, register setup-time credential/config handlers, reuse generic SecretInput prompting safely, persist under plugin-owned config, and document/test the SDK surface alongside provider and channel setup APIs.
