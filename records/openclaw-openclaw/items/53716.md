## Risks / Open Questions

- Merging the PR as-is could expose a new `gateway watchdog` command that fails for packaged users unless the runtime script is included in the npm package or moved into built dist assets.
- The watchdog option parsers still need clear rejection of non-numeric interval/max-attempt values before spawning the shell script.
- The Bash script initializes `TOKEN` independently from `OPENCLAW_GATEWAY_TOKEN`, so env-token deployments can be misdiagnosed as down unless token handling is fixed and tested.
- The branch adds local shell execution, desktop notification behavior, restart commands, and token-bearing health checks; those are appropriate to review as security-sensitive runtime behavior even though no third-party dependency or workflow changes were added.
- There is adjacent open rescue-watchdog work in #46502, so maintainers should decide whether this PR should merge as a smaller CLI/script feature, be split, or be superseded by the broader rescue-watchdog design.
