## Summary

Keep this PR open. Current main has some adjacent gateway diagnostics, including persisted stability bundles for startup/restart failures, but it does not implement the PR's intended `openclaw gateway watchdog` command, packaged external watchdog script, desktop notification/restart loop, or `startup-error.txt` diagnostic file. The linked feature issue remains open, and the PR still contains useful implementation work plus unresolved review/security/packaging concerns that need maintainer review rather than cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to either land a corrected, packaged watchdog/startup-diagnostics implementation or split it into smaller PRs: one for source-backed startup diagnostics, one for a first-class packaged watchdog command with auth/env handling, validation, docs, and tests. After merge, the linked gateway failure recovery issue can be reassessed against the remaining multi-instance/dead-man-switch scope.
