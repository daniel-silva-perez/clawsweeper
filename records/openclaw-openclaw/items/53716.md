## Summary

Keep open. Current main still does not ship this PR's watchdog command, so the contribution is not obsolete, but the branch has blocking correctness issues around packaging, stale diagnostic cleanup, option validation, and child-process error handling. It also overlaps the broader rescue-watchdog work, so this needs maintainer rebase/split/supersede direction before merge.

## What This Changes

The PR adds startup-error file persistence for fatal CLI/Gateway startup failures and registers a `gateway watchdog` command that runs a Bash health-check, restart, logging, and desktop-notification script.

## Best Possible Solution

Land only a rebased, maintainer-reviewed shape that fits the current stability/fatal-hook architecture. The final implementation should package any runtime assets, validate CLI input before executing the script, handle child-process failures predictably, document the public command if it remains public, and either split or intentionally supersede the watchdog layer against the broader rescue-watchdog design.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Ship the watchdog script with the packaged CLI:** `src/cli/gateway-cli/run.ts:573`
  - body: The command resolves `scripts/openclaw-watchdog.sh` at runtime, but the npm package file list only includes selected script files and this PR does not add the new script to packaged assets. Installed users would get a `gateway watchdog` command that points at a missing file.
  - confidence: 0.94
- **[P2] Clear startup errors after startup actually succeeds:** `src/cli/gateway-cli/run.ts:463-466`
  - body: This cleanup runs after `await startLoop()`, but the Gateway loop is long-running and normally waits for stop/restart signals instead of resolving after a successful bind. A stale `startup-error.txt` can therefore survive normal recovery.
  - confidence: 0.88
- **[P2] Reject invalid watchdog numeric options before spawning:** `src/cli/gateway-cli/run.ts:582-590`
  - body: The parsers can return `NaN` for values such as `--interval foo` or `--max-attempts foo`, and that string is forwarded into shell arithmetic. The script then fails with an opaque shell error instead of a clear Commander validation failure.
  - confidence: 0.9
- **[P2] Handle watchdog spawn errors with an error listener:** `src/cli/gateway-cli/run.ts:615-623`
  - body: Wrapping `spawn("bash", ...)` in `try/catch` does not handle normal spawn failures like `ENOENT`; Node emits those on the returned child as an `error` event. Without a listener, missing Bash can become an uncaught exception instead of a controlled CLI error.
  - confidence: 0.86
