## Summary

Keep open. Current main still does not provide the proposed Gateway watchdog command, so the PR is not obsolete; however, the branch is not merge-ready because the watchdog script is not packaged for installed CLI users, startup-error cleanup is placed after a long-running Gateway loop, invalid numeric options can reach shell arithmetic, and child spawn failures are not handled correctly. The work also overlaps with the broader open rescue-watchdog design thread, so maintainer direction is needed before landing or replacing it.

## What This Changes

The PR adds startup-error file persistence for fatal CLI/Gateway startup failures and registers an `openclaw gateway watchdog` subcommand that spawns a Bash health-check/restart script with desktop notifications.

## Best Possible Solution

Land a rebased implementation only after the watchdog boundary is clear. The final shape should preserve the current stability/fatal-hook diagnostics architecture, ship any runtime assets used by the CLI, validate inputs before shell execution, document the user-facing command if it remains a public surface, and add focused CLI/infra/script coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Ship the watchdog script with the packaged CLI:** `src/cli/gateway-cli/run.ts:573`
  - body: The new command resolves `scripts/openclaw-watchdog.sh` at runtime, but the package file list does not include that script. Installed package users would get a `gateway watchdog` command that points at a missing file, so the feature is unusable outside a source checkout unless the script is packaged or moved into a shipped runtime asset.
  - confidence: 0.94
- **[P2] Move startup-error cleanup before the long-running loop:** `src/cli/gateway-cli/run.ts:463-466`
  - body: This cleanup runs only after `await startLoop()` returns, but the Gateway loop is designed to stay running until shutdown or exit. On normal successful startup this block never executes, leaving stale `startup-error.txt` content visible after the Gateway has recovered.
  - confidence: 0.88
- **[P2] Reject invalid watchdog numeric options before spawning:** `src/cli/gateway-cli/run.ts:583-590`
  - body: Both numeric parsers can return `NaN`, and that value is forwarded into shell arithmetic. The script then aborts with a shell arithmetic error instead of a clear Commander validation failure for invalid `--interval` or `--max-attempts` input.
  - confidence: 0.9
- **[P2] Handle child spawn failures with an error listener:** `src/cli/gateway-cli/run.ts:615-623`
  - body: Wrapping `spawn("bash", ...)` in try/catch does not handle normal spawn failures such as `ENOENT`; Node emits those on the returned `ChildProcess` as an `error` event. Without an `error` listener, a missing Bash runtime can become an uncaught exception instead of a controlled CLI error.
  - confidence: 0.86
