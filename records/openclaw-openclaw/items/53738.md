## Summary

Keep open. Current main already covers the image-side configured-provider behavior, but the PDF tool still uses the registry-only resolver, and this PR remains an unmerged, dirty branch with a PDF fallback correctness issue that needs maintainer choice alongside the narrower open PDF PR.

## What This Changes

The PR exports the unknown-model helper, removes the shared registry-only media resolver, switches image and PDF model lookup toward `resolveModelWithRegistry`, and adds image/PDF regression tests for configured provider behavior.

## Best Possible Solution

Land one canonical PDF execution-time resolver path that honors configured provider overrides for registry-backed, config-defined, and plugin-dynamic models while preserving native PDF handling and extracted-text fallback behavior; keep the already-implemented image path on main.

## Reproduction Assessment

Yes. The PR body and #65742 provide concrete config steps for the PDF `Unknown model` failure, and current main still has the registry-only PDF lookup that explains it. The PR finding is reproducible with a plugin-dynamic text-only model and a PDF that has both images and extractable text.

## Solution Assessment

No. The direction is valid, but this branch is stale/unmergeable and the added PDF guard is not the narrowest maintainable fix because it rejects plugin-dynamic resolved models instead of using the resolved model's capability/source.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Allow plugin-resolved PDF text models through fallback:** `src/agents/tools/pdf-tool.ts:239-240`
  - body: `resolveModelWithRegistry` can return a plugin-dynamic text-only model that is not declared in `models.providers` and is not found by `modelRegistry.find`. This guard treats that resolved model as unsupported and throws before the existing extracted-text fallback, so PDFs with extractable text plus page images can fail instead of returning a text-only summary. Use the resolved model capability/source rather than only config or registry membership.
  - confidence: 0.84
