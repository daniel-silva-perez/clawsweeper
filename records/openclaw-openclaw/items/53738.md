## Summary

Keep this PR open. Current main has adopted the config-aware model resolver for image media understanding, but the PDF tool still resolves models through the registry-only helper, so the PR’s central PDF/baseUrl problem is still present on main. Related #65746 covers a narrower custom-provider PDF failure, but it does not fully supersede the built-in provider baseUrl override case described here.

## Best Possible Solution

Keep the PR open but narrow the remaining work to the PDF path on current main: replace the registry-only PDF lookup with a config-aware resolver or a smaller equivalent that applies provider overrides, preserve existing text-extraction fallback behavior, add PDF regression coverage for built-in baseUrl override and config-defined custom providers, and reconcile with #65746 rather than closing either item prematurely.
