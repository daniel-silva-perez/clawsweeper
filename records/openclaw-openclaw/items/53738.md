## Summary

Keep open. Current main already covers the image-side config-aware resolver path, but the PDF tool still resolves execution models through the registry-only helper, so the central PDF/baseUrl problem is not fully implemented on main. The linked runtime report and open narrower PDF PR make this a maintainer triage item rather than an obsolete PR.

## What This Changes

This PR changes image and PDF media-tool model resolution to use the config-aware model resolver, exports the shared unknown-model error helper, removes the registry-only helper, and adds media-tool regression coverage around configured provider models and fallback behavior.
