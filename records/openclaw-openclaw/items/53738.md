## Summary

Keep open: current main already covers the image-side behavior, but the PDF execution path still uses the registry-only resolver, and this PR's PDF patch has a blocking fallback regression for plugin-dynamic text-only models. With #65746 still open and this branch currently unmergeable, maintainers should choose the canonical PDF resolver path before merge.

## What This Changes

The PR exports the unknown-model error helper, removes the registry-only media resolver, switches image and PDF model lookup toward `resolveModelWithRegistry`, and adds image/PDF regression tests for configured provider behavior.

## Best Possible Solution

Land one canonical PDF resolver path that honors configured provider overrides for registry-backed, config-defined, and plugin-dynamic models while preserving native PDF handling and the documented extracted-text fallback; keep the image path already present on main.

## Reproduction Assessment

Yes. The original PDF problem is reproducible from the #65742 config steps, and the PR finding is reproducible with a plugin-dynamic text-only model plus a PDF containing both images and extractable text.

## Solution Assessment

No. The direction is valid, but this branch is stale/unmergeable and the new PDF guard is not the narrowest maintainable fix because it rejects plugin-dynamic resolved models instead of using the resolved model's capability/source.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Allow plugin-resolved PDF models through text fallback:** `src/agents/tools/pdf-tool.ts:239-240`
  - body: `resolveModelWithRegistry` can return a plugin-dynamic text-only model that is not declared in `models.providers` and is not found by `modelRegistry.find`. This guard treats that resolved model as unsupported and throws before the existing extracted-text fallback, so PDFs with extractable text plus page images can fail instead of returning a text-only summary. Use the resolved model capability/source rather than only config or registry membership.
  - confidence: 0.86
