## Summary

Keep open. Current main already covers the image-side model resolution path, but the PDF execution path still resolves through the registry-only helper, and this PR is not merge-ready because its branch is unmergeable and its PDF guard can regress plugin-dynamic text-only models.

## What This Changes

The PR exports the unknown-model error helper, removes the shared registry-only media resolver, routes image and PDF model lookup through `resolveModelWithRegistry`, and adds image/PDF regression tests for configured provider and fallback behavior.

## Best Possible Solution

Preserve the current image implementation on main, then land one canonical PDF resolver path that applies configured provider overrides for registry-backed models and config-defined custom providers while preserving extraction fallback for registry, config, and plugin-dynamic text-only models.

## Reproduction Assessment

Yes. #65742 gives concrete config steps and real-runtime before/after evidence for `local-openai/gpt-5.4-mini`, and current main still has the registry-only PDF execution call that explains the `Unknown model` failure.

## Solution Assessment

No. The direction is valid, but this PR is broader than the remaining main bug and adds a PDF guard that misses plugin-dynamic text-only models; the safer path is a rebased, PDF-focused resolver change with focused fallback tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Let plugin-resolved PDF text models use fallback:** `src/agents/tools/pdf-tool.ts:239-240`
  - body: `isKnownTextOnlyModel` only accepts text-only models declared in config or found in the registry. `resolveModelWithRegistry` can also return a plugin-dynamic text-only model, and this new guard now throws before the existing text-extraction fallback runs, so PDFs with images plus extractable text can fail instead of returning a text-only summary. Use the resolved model capability to distinguish real text-only models from synthetic unresolved fallbacks.
