## Summary

Keep open: current main covers the image side, but the PDF path still uses registry-only model resolution, and this branch has a concrete PDF fallback regression plus an overlapping narrower open PR that requires maintainer choice.

## What This Changes

The PR exports the unknown-model helper, removes the shared registry-only media resolver, routes image and PDF model lookup through `resolveModelWithRegistry`, and adds image/PDF regression tests for configured-provider behavior.

## Best Possible Solution

Land one canonical PDF model-resolution path that honors configured provider overrides for registry-backed, config-defined, and plugin-dynamic models while preserving native PDF handling and the extracted-text fallback for text-only models; keep the already-shipped image implementation on main.

## Reproduction Assessment

Yes. #65742 gives concrete config steps and real-runtime evidence for the PDF `Unknown model` failure, and current main still has the registry-only PDF lookup that explains it. The PR finding is reproducible with a plugin-dynamic text-only model and a PDF that has both images and extractable text.

## Solution Assessment

No. The direction is valid, but this branch is stale/unmergeable and the added PDF guard is not the narrowest maintainable fix because it rejects plugin-dynamic resolved models instead of using the resolved model's capability/source.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Let plugin-resolved PDF text models use fallback:** `src/agents/tools/pdf-tool.ts:239-240`
  - body: `resolveModelWithRegistry` can return a plugin-dynamic text-only model that is not declared in `models.providers` and is not found by `modelRegistry.find`. This guard treats that resolved model as unsupported and throws before the existing extracted-text fallback, so PDFs with extractable text plus page images can fail instead of returning a text-only summary. Use the resolved model capability/source rather than only config or registry membership.
  - confidence: 0.84
