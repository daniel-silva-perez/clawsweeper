## Summary

Current main only implements the image/media-understanding half of this PR. The PDF tool still resolves execution models through the registry-only helper, so configured provider models and `models.providers.<provider>.baseUrl` can still be missed; the linked open #65746 is the narrower active PDF fix and needs maintainer triage before either PR is closed.

## What This Changes

This PR changes image and PDF media-tool model resolution to use `resolveModelWithRegistry` with config, exports `buildUnknownModelError`, removes the old registry-only helper, and adds regression coverage around configured provider/baseUrl and fallback behavior.

## Best Possible Solution

Keep this PR open for maintainer triage. The maintainer should decide whether #65746 is the canonical narrow PDF fix or whether this PR should be rebased and reduced so it preserves the already-implemented image path, fixes PDF execution-time model resolution through an approved config-aware helper, and adds focused PDF regression tests for configured provider models and baseUrl overrides.
