## Summary

Keep open. Current main still has a plausible provider-metadata sanitation gap: Anthropic assistant text events are treated as normal visible output, stored in `assistantTexts`, and passed to channel delivery, while the bundled Anthropic stream wrapper has no Max usage-alert filter. The issue discussion already identified the same path, and the remaining fix is narrow enough for a focused provider-owned PR.

## What This Changes

Strip Anthropic Claude Max usage/billing alert text blocks before they are persisted in transcripts or delivered as assistant replies to channels.

## Best Possible Solution

Add a narrowly scoped Anthropic-provider sanitizer before the embedded subscriber persists or delivers assistant text. Prefer an extension-owned stream/result wrapper or equivalent provider output seam that strips the exact Max usage alert block from standalone and embedded assistant text, with regression coverage for transcript text and channel block replies; avoid a broad user-configurable strip-pattern feature unless maintainers explicitly want that product surface.

## Reproduction Assessment

Yes. A high-confidence unit reproduction can emit an Anthropic assistant text block containing the reported `Claude Max Usage Alert` pattern through the embedded subscriber and assert it currently reaches `assistantTexts` and `onBlockReply`; the provided Discord/session evidence supports the same runtime path.

## Solution Assessment

Yes for a targeted provider-owned sanitizer; the current code has no implementation. The issue's configurable `messages.outputStripPatterns` idea is broader than needed for this provider-owned billing metadata leak and should not be the first fix unless maintainers choose a general policy surface.
