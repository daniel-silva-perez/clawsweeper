## Summary

Keep open. The current main pipeline still treats ordinary Anthropic assistant text as user-facing output, and I found no current sanitizer or regression coverage for the reported Claude Max usage alert block; GMTekAI's earlier comment matches the code path I inspected.

## What This Changes

The issue requests stripping Anthropic Max usage alert text blocks before they are persisted to transcripts or delivered to external channels.

## Best Possible Solution

Add a provider-owned Anthropic output sanitizer before the embedded subscriber builds `assistantTexts` and block replies, preserving normal assistant text while removing only the Claude Max usage alert metadata from OAuth/Max response streams.

## Reproduction Assessment

Yes. A high-confidence synthetic reproduction is to feed an assistant `text_delta`/`text_end` or final assistant message containing the reported alert into the embedded subscriber; the inspected code would currently emit and retain it as ordinary assistant text.

## Solution Assessment

Yes for a narrow provider-owned sanitizer. A global configurable `messages.outputStripPatterns` surface is broader than needed and would put provider billing metadata policy into user config/core instead of the Anthropic provider boundary.
