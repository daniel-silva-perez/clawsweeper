## Summary

Keep open. Current main still does not appear to strip Anthropic Max usage alert text blocks before visible assistant extraction or channel delivery. The existing issue comment matches the current code shape: reasoning/tool scaffolding is filtered, but provider-injected billing/usage text is not treated as metadata.

## Best Possible Solution

Keep the issue open. Add a shared assistant-output metadata sanitizer for the Anthropic Max usage alert pattern, apply it before `assistantTexts` and reply payload generation, and ensure persisted transcript content is stripped as well. Add regression tests for standalone alert-only assistant messages and alerts embedded in otherwise valid assistant text.

## Evidence

- **Issue discussion identifies the same pipeline gap:** The existing comment says current delivery suppresses reasoning payloads but found no equivalent provider-injected assistant text metadata filter, and recommends filtering before transcript persistence and external delivery. That remains consistent with current source inspection.
- **Shared assistant-visible sanitizer has no Anthropic Max usage alert stage:** The delivery/history sanitizer strips Minimax tool XML, model special tokens, relevant-memory tags, tool-call XML, plain-text tool-call blocks, downgraded tool text, and reasoning tags. There is no stage for `Claude Max Usage Alert`, `Session`, `Weekly`, or `Extra` billing text.
