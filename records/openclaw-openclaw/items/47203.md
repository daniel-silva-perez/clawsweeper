## Summary

Keep this issue open. Current main still has the core navigation gap described by the report: `navigate` accepts only URL/target at the route, browser tool, and CLI layers, still uses a 20s request budget, and still calls Playwright `page.goto` without an explicit wait policy such as `commit` or `domcontentloaded`. Adjacent fixes for browser action timeouts and browser-service error clarity do not close this specific `navigate` behavior.

## Best Possible Solution

Keep this issue open and fix it in the bundled browser plugin. Extend `/navigate`, `browserNavigate`, the browser tool, and the CLI to support an explicit navigation timeout and wait policy, add enough transport slack so the client/gateway timeout does not race the inner navigation timeout, improve timeout error wording for reachable service vs navigation failure, and cover the behavior in route, client, CLI, and Playwright navigation tests. The existing `browser open <url>` workaround from the discussion is useful, but it does not replace fixing `navigate`.
