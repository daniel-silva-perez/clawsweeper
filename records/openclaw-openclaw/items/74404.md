## Summary

Keep open. The report is fresh, concrete, and current main still has the same conservative recovery shape: HTTP liveness can stay green, diagnostics can warn about liveness and stuck sessions, but active embedded/reply work is not force-aborted by the stuck-session watchdog. Adjacent unreleased fixes improve client readiness, diagnostics, and plugin scan performance, but they do not prove that the reported Linux CPU saturation, WS/RPC timeouts, and hours-old heartbeat processing state are fully fixed.

## What This Changes

Investigate a regression where the Linux Gateway becomes CPU-saturated after v2026.4.23+, causing WS/RPC calls and heartbeat/cron-driven agent progress to stall while HTTP liveness probes still respond.

## Best Possible Solution
