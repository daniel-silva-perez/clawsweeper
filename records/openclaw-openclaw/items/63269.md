## Summary

Keep open. Current main still has the nominal Mattermost WebSocket `posted` parser and channel/group dispatch path, but I did not find source, test, release, or related-PR proof that the reported `chatmode: "oncall"` plus `groupPolicy: "open"` public/group @mention regression is fixed. ## What This Changes

The issue reports a Mattermost regression where public/group channel WebSocket messages are silently dropped while DMs still work, especially for @mentioned `oncall` traffic with open group policy.

## Best Possible Solution

Mattermost should reliably process allowed public, private, and group channel `posted` WebSocket frames, including @mentioned `oncall` posts with open group policy, with focused regression coverage and redacted diagnostics when a frame is filtered.
