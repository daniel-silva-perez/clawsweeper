## Summary

Keep open. Current main still gates Telegram DM topic auto-labeling on session-store `systemSent`, and the rename call remains unreachable once that flag is already true; I did not find a current regression test or shipped change that covers the reported bootstrap-state case.

## What This Changes

The issue reports that Telegram DM forum topics keep their provisional title because the `autoTopicLabel` first-turn rename path no longer runs reliably after the first user message.

## Best Possible Solution

Fix this in the Telegram plugin by making auto-topic labeling idempotent on a Telegram-owned per-topic label attempt/state instead of relying solely on core `systemSent`, while preserving the existing config precedence and fire-and-forget rename behavior.

## Reproduction Assessment

Yes. A focused unit test can create a DM-topic dispatch with `SessionKey`, `RawBody`, a final response, and a session-store entry where `systemSent: true`; current main will skip `generateTopicLabel` and `editForumTopic` because `isFirstTurnInSession` becomes false.

## Solution Assessment

No patch is proposed here. The narrow maintainable fix is Telegram-owned: decouple the rename trigger from `systemSent` or add an idempotent topic-label marker, rather than changing core session bootstrap semantics.
