## Summary

Keep open. Current main still lacks the bounded EIO bootstrap retry this branch is trying to add, so the work is not obsolete, but the branch should not merge as-is because its new force-kill path can target a reused PID and the implementation predates the current launchd lifecycle code.

## What This Changes

The PR adds LaunchAgent bootstrap retries for transient EIO, waits for previous launchd PIDs with SIGKILL escalation during install/restart, routes doctor repair through the retry helper, and expands launchd unit tests.

## Best Possible Solution

Port a shared bounded EIO-only bootstrap retry onto the current launchd lifecycle and route doctor repair, install activation, restart fallback, and detached handoff through it. Keep force-kill escalation separate unless maintainers explicitly want it and the code verifies the saved PID still belongs to the expected launchd job before killing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Verify the saved PID before sending SIGKILL:** `src/daemon/launchd.ts:440`
  - body: The branch waits up to 15 seconds and then sends SIGKILL to the saved numeric PID if kill(pid, 0) succeeds. That probe only proves some process currently has the PID; if the original LaunchAgent exits and the PID is reused, install or restart can terminate an unrelated process. Avoid SIGKILL here or verify the PID still belongs to the expected launchd job before force killing it.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: No dependency or supply-chain changes were found, but the diff introduces a process-control path that can SIGKILL a reused PID without job identity verification.

Concerns:

- **[medium] SIGKILL can target a reused PID:** `src/daemon/launchd.ts:440`
  - body: The proposed escalation sends SIGKILL to a saved PID after a delay based only on kill(pid, 0). PID reuse can satisfy that check for a different process, creating a local availability and safety risk in a service-management command.
  - confidence: 0.82
