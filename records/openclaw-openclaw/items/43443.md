## Summary

Keep open. The PR is not obsolete: current main and v2026.4.27 still use raw `launchctl bootstrap` paths with no EIO-only retry. The branch should not merge as-is because its SIGKILL escalation is based on a saved PID liveness probe and can terminate a reused PID.

## What This Changes

The PR adds EIO-only launchctl bootstrap retries, a PID wait helper that escalates to SIGKILL, applies the retry to launchd repair/install/restart paths, and expands launchd unit coverage.

## Best Possible Solution

Port a bounded shared EIO-only bootstrap retry onto the current launchd lifecycle and use it for doctor repair, install activation, restart fallback, and detached handoff bootstrap attempts. Keep force-kill escalation out unless maintainers explicitly accept it and the implementation verifies the saved PID still belongs to the expected launchd job before signaling.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Verify the saved PID before sending SIGKILL:** `src/daemon/launchd.ts:440`
  - body: The branch waits and then sends SIGKILL to a saved numeric PID if `kill(pid, 0)` succeeds. That only proves some process currently has the PID; if the original LaunchAgent exits and the PID is reused, install or restart can terminate an unrelated process. Avoid SIGKILL here or verify the PID still belongs to the expected launchd job before force killing it.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: No dependency, workflow, or supply-chain changes were found, but the diff introduces a process-control path that can SIGKILL a reused PID without job identity verification.

Concerns:

- **[medium] SIGKILL can target a reused PID:** `src/daemon/launchd.ts:440`
  - body: The proposed escalation sends SIGKILL to a saved PID after a delay based only on `kill(pid, 0)`. PID reuse can satisfy that check for a different process, creating a local availability and safety risk in a service-management command.
  - confidence: 0.82
