## Summary

Keep open. The EIO bootstrap retry remains useful because current main still raw-bootstraps several launchd paths, but this PR should not merge as-is: the branch is based on an older launchd lifecycle and introduces delayed SIGKILL by saved numeric PID without proving job identity.

## What This Changes

The PR adds EIO-only launchctl bootstrap retries, a PID wait helper with SIGKILL escalation, applies the retry to launchd repair/install/restart paths, and expands launchd unit coverage.

## Best Possible Solution

Implement a shared bounded EIO-only launchctl bootstrap retry helper on current main and use it for bootstrapLaunchAgentOrThrow, repairLaunchAgentBootstrap, and generated restart handoff bootstrap attempts. Keep force-kill escalation out unless maintainers explicitly accept it and the implementation verifies that any signaled PID still belongs to the expected launchd job.

## Reproduction Assessment

Yes for the blocking PR finding: branch inspection shows waitForPidExitWithEscalation sends SIGKILL after only kill(pid, 0), which cannot verify launchd job identity. The original EIO bug also has a concrete manual repro in the PR body, but it was not rerun during this read-only review.

## Solution Assessment

No. The retry direction is likely the right narrow fix, but this branch solves it through an older bootout/bootstrap lifecycle and adds unsafe PID-based force killing; the current-main fix should be retry-focused or prove process identity before any signal escalation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Verify the saved PID before SIGKILL:** `src/daemon/launchd.ts:440`
  - body: The branch waits, checks kill(pid, 0), then sends SIGKILL to the saved PID. That only proves some process currently has the PID; if the LaunchAgent exited and the PID was reused, install or restart can terminate an unrelated process. Avoid SIGKILL here or verify the PID still belongs to the expected launchd job before signaling.
  - confidence: 0.86
