## Summary

Current main still matches the reported memory-core failure mode: daily ingestion records heading-prefixed/list-joined snippets, while promotion rehydration relocates with whitespace-only equality/includes checks and silently returns zero when selected candidates all fail relocation. This should stay open as a focused fix candidate.

## What This Changes

The issue requests fixing memory-core promotion rehydration so daily-ingestion snippets with heading/list normalization can relocate in live daily notes instead of silently producing zero applied promotions.

## Best Possible Solution

Fix memory-core by aligning the relocation comparison with daily-ingestion snippet normalization, add a diagnostic when selected candidates all fail rehydration, and keep the existing live-note/deleted-source protection documented for `memory promote --apply`.

## Repair Work Prompt

Fix the memory-core short-term promotion rehydration bug described in this issue. Current daily ingestion stores snippets with meaningful headings converted to a `Heading: ...` prefix, list markers removed, list items joined with `; `, and text truncated/collapsed, while `relocateCandidateRange` compares only whitespace-collapsed live ranges. Update the promotion apply path so heading-prefixed/list-normalized daily-ingestion candidates can still relocate against the live daily note. Preserve the documented behavior that edited/deleted source snippets are not promoted from stale recall-store snapshots; do not replace relocation with a blind stored-candidate fallback. Add a warning or structured diagnostic when selected candidates exist but all fail rehydration, including a count and representative candidate key/path without leaking unrelated content. Add focused regression coverage for a daily-ingestion heading/list chunk that ranks in dry-run and applies successfully, plus coverage that deleted/changed source content still skips. Likely files: `extensions/memory-core/src/short-term-promotion.ts`, `extensions/memory-core/src/short-term-promotion.test.ts`, and possibly `extensions/memory-core/src/dreaming-phases.ts` / `extensions/memory-core/src/dreaming-phases.test.ts` if a shared normalizer is factored there. Add a single-line `CHANGELOG.md` Fixes entry for the active version with `Thanks @haumanto` if maintainers classify this as user-facing released behavior. Validation: run the targeted memory-core tests and formatter listed in `workValidation`; before handoff, run `pnpm check:changed` in Testbox if code changes are promoted.
