## Summary

Keep open: current main still has the reported memory-core mismatch between daily-ingestion snippets and promotion rehydration, so qualified candidates can be dropped during apply without a specific diagnostic.

## What This Changes

Fix memory-core short-term promotion so daily-ingestion snippets with heading/list normalization can be rehydrated and applied instead of silently dropping selected candidates.

## Best Possible Solution

Align memory-core's relocation comparison with the daily-ingestion snippet shape, preserving live source rehydration and deleted-source skipping. Add a regression test for heading-prefixed list chunks and a clear diagnostic when selected candidates all fail rehydration.

## Reproduction Assessment

Yes. The current source plus the issue's sample shape gives a high-confidence reproduction: daily ingestion records a heading-prefixed/list-joined snippet with a body-only line range, while relocation compares only whitespace-collapsed source windows.

## Solution Assessment

Yes, the best fix is structural normalization alignment in memory-core rather than falling back to stored candidates. That keeps the documented live-note protection while allowing daily-ingestion candidates to rehydrate.
