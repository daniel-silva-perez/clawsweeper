## Summary

Current `main` still shows the reported memory-core mismatch, so this should stay open. Daily ingestion can store heading-prefixed/list-joined snippets, while promotion rehydration still compares only whitespace-collapsed live markdown windows and silently returns no applied promotions when every selected candidate fails relocation.

## What This Changes

The issue requests fixing memory-core promotion apply so stored daily-ingestion snippets can be rehydrated from live daily notes instead of being silently dropped by incompatible normalization.

## Best Possible Solution

Memory-core should use a shared or compatible normalization path for daily-ingestion snippets and live relocation windows, keep live-source rehydration as the authority, and add an aggregate warning when all selected candidates fail rehydration.

## Repair Work Prompt

Fix the memory-core promotion rehydration normalization mismatch reported in https://github.com/openclaw/openclaw/issues/74334. Current daily ingestion stores rich snippets from `extensions/memory-core/src/dreaming-phases.ts` by stripping list markers, prefixing meaningful headings as `Heading: ...`, joining list chunks with `; `, truncating, and recording body line ranges. Current apply rehydration in `extensions/memory-core/src/short-term-promotion.ts` compares only whitespace-collapsed live markdown windows, so `memory promote --apply` can silently drop every selected daily-ingestion candidate. Implement a narrow fix that makes relocation compare against a normalization compatible with the ingestion snippet shape while preserving live-source rehydration as the authority; do not use the issue's stored-candidate fallback as the primary fix. Add a defensive aggregate warning or diagnostic when all selected candidates fail rehydration. Coordinate with open PR #74051 if it lands first because it also edits relocation span/tie-break behavior, but do not assume it fixes this normalization issue. Add regression coverage for a daily note with a meaningful heading plus bullet list whose stored candidate snippet is `Heading: item; item` and whose stored line range excludes the heading, then verify `applyShortTermPromotions` appends the live rehydrated promotion instead of returning `applied: 0`. Also preserve the deleted/stale-source skip behavior. Because this is user-facing memory behavior, add a `CHANGELOG.md` fix entry with `Thanks @haumanto`; do not add broad refactors or new config/API surface.
