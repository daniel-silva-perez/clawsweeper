## Summary

Keep open. Current main still has the reported memory-core mismatch: daily ingestion can store heading-prefixed, list-joined snippets, while promotion rehydration compares only whitespace-collapsed live markdown windows and silently returns zero applied promotions when every selected candidate fails relocation.

## What This Changes

The issue requests fixing memory-core promotion rehydration so candidates stored by daily dreaming ingestion can be relocated against live daily notes instead of silently producing `applied: 0`.

## Best Possible Solution

Memory-core should use a shared or compatible normalization path for daily-ingestion snippets and live relocation windows, keep live-source rehydration as the authority, and emit one aggregate warning when all selected candidates fail rehydration.

## Reproduction Assessment

Yes. The issue includes affected-host dry-run/apply output, and the current source-level comparison can be reproduced with a heading-prefixed/list-joined stored snippet against the raw markdown bullet window.

## Solution Assessment

Yes. Aligning the normalizers is the narrow maintainable fix; blindly falling back to stored candidates would weaken the documented stale-snippet guard and can promote stale offsets.

## Repair Work Prompt

Fix the memory-core promotion rehydration bug reported at https://github.com/openclaw/openclaw/issues/74334. Current main stores daily-ingestion snippets with heading prefixes, stripped list markers, `; ` list joins, and truncation, but `relocateCandidateRange` compares only whitespace-collapsed live markdown windows. Align the stored-snippet and live-window normalization so a heading-prefixed/list-joined daily-ingestion candidate can be relocated from the live daily note. Keep live-source rehydration authoritative; do not implement the workaround that blindly promotes the stored candidate when relocation fails. Add one aggregate warning or equivalent diagnostic when a non-empty selected set all fails rehydration. Add focused tests that exercise the daily-ingestion stored snippet shape through `applyShortTermPromotions`, plus any needed unit coverage around the shared normalizer. Coordinate with open PR #74051 if it lands first because it touches nearby relocation/gate logic. Add a user-facing `CHANGELOG.md` fix entry with `Thanks @haumanto`.
