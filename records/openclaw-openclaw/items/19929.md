## Summary

Keep open. Current main still does not implement selective shared sessions across chosen group/channel/thread chats: non-direct peers remain isolated by channel/peer key, thread suffixing remains default-on, and the strict session config/docs expose no channel grouping or thread-isolation override. The closed implementation PR was not merged, and the related open issues are design inputs rather than replacements.

## Best Possible Solution

Keep this issue open as the canonical tracker for selective shared sessions across chosen group/channel/thread chats. The best path is an additive config design with isolated defaults, explicit channel/thread membership, optional thread inheritance, reply routing back to the originating channel, grouped queue serialization, aligned schema/types/help/docs, and regression coverage across routing, Discord, Slack, and sandbox-sensitive session behavior. Treat #7524, #43286, #51805, and the closed #20078 implementation attempt as design inputs.
