## Summary

Keep this issue open. Current main still isolates Discord/Slack/group/channel peers by session key and only exposes DM-centric sharing controls (`dmScope`, `identityLinks`) plus broad/global behavior; it does not provide selective shared sessions for chosen group/channel/thread chats. The related PR #20078 was closed unmerged, and related issues #7524, #43286, and #51805 track narrower adjacent shapes rather than replacing this request.

## Best Possible Solution

Keep this open as the canonical tracker for selective shared sessions across chosen group/channel/thread chats. A good implementation should add a backward-compatible core config/routing surface, preserve ungrouped behavior, route replies to the originating channel, serialize grouped queues through the shared session key, define optional thread inheritance, and update schema/types/help/docs plus routing/thread/sandbox regression tests. #7524, #43286, and #51805 should be considered related narrower design inputs, not replacements.
