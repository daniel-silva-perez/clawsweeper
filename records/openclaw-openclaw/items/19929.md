## Summary

Current main still does not implement selective shared sessions across chosen group/channel/thread chats. The central routing path keeps non-direct peers isolated by channel/peer id, thread replies still append a thread suffix by default, and the strict session config surface has no `channelGroups`, `groupScope`, or thread-inheritance setting. The linked implementation PR #20078 was closed unmerged, while #7524, #43286, and #51805 remain narrower adjacent trackers rather than replacements for this broader request.

## Best Possible Solution

Keep this open as the canonical tracker for selective shared sessions across chosen group/channel/thread chats. A good maintainer path is to design an additive session-group config surface, preserve isolated behavior by default, route replies to the originating channel, serialize grouped messages through the shared session key, define optional thread inheritance, and update schema/types/help/docs plus routing/thread/sandbox regression tests. Treat #7524, #43286, and #51805 as related design inputs rather than replacements.
