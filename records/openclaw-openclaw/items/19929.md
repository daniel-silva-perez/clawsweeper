## Summary

Keep open. Current main still routes non-direct peers to isolated channel/group session keys, threads still suffix by default, and the strict session config/types/docs do not expose a selective channel/thread grouping surface. The already-linked implementation PR was closed unmerged, and the related open issues are useful design inputs rather than replacements.

## Best Possible Solution

Keep this issue open as the canonical tracker for selective shared sessions across chosen group/channel/thread chats. The best path is an additive config design with isolated defaults, explicit channel/thread membership, optional thread inheritance, reply routing back to the originating channel, grouped queue serialization, and aligned schema/types/help/docs plus routing/thread/sandbox regression coverage. Treat #7524, #43286, and #51805 as related design inputs rather than replacements.
