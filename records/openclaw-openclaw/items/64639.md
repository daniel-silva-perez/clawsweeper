## Summary

Keep this issue open. Current main still hardcodes the personal `~/.agents/skills` source in both discovery and watcher setup, while the public config surface only supports extra directories, watcher toggles, per-skill disables, allowlists, and prompt limits after discovery.

## What This Changes

Allow users to disable specific skill source directories, especially `agents-skills-personal`, so large personal skill trees are not scanned, watched, or injected when external skill routing is used.

## Best Possible Solution

Add a backwards-compatible skills load setting that skips disabled built-in source roots before discovery and watcher registration, preserving existing defaults and precedence when unset. Keep the broader on-demand skill-selection architecture as a separate design discussion.
