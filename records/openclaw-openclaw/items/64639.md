## Summary

Keep this open. Current main still loads and watches the personal `.agents/skills` source without a source-level config guard, and the documented config surface only offers extra directories, watcher toggles, per-skill visibility, allowlists, and prompt limits after discovery. The short-term config toggle remains a valid, narrow repair candidate; the broader on-demand skill router should stay out of scope.

## What This Changes

The issue requests a configuration option to disable built-in skill source directories, especially the personal `~/.agents/skills` source, so externally routed skills do not get discovered and injected into every session.

## Best Possible Solution

Add a backwards-compatible skills load setting that skips disabled built-in source roots before filesystem discovery and watcher registration, preserving today’s default precedence when unset. Keep the long-term on-demand skill selection/router architecture as a separate design item.
