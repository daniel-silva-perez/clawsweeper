## Summary

Keep this PR open. Current main still forwards the CLI timeout into the gateway-owned `agent` run, so the PR's central behavior is not implemented on main. The PR is also not ready to merge as submitted because the added accepted-ownership guard imports a symbol that current `src/gateway/call.ts` does not export, matching the existing review comments.

## Best Possible Solution

Keep the PR open, but require a revision before merge: remove the forwarded `params.timeout` from the gateway `agent` request, add a regression test proving the gateway params omit `timeout` while `callGateway.timeoutMs` remains the client wait budget, either drop the accepted-ownership catch path or implement/export the guard in `src/gateway/call.ts` with production-backed tests, and update `docs/cli/agent.md` if gateway-mode `--timeout` is now intentionally client-side only.
