## Summary

Keep open for maintainer/author follow-up. The PR targets a real gateway CLI timeout path that is still present on current main, but the branch is not mergeable as-is: it imports an accepted-ownership guard that the real gateway call module does not export, and its test mocks create an API that production code does not have.

## What This Changes

The PR removes the forwarded `timeout` field from gateway `agent` params, keeps the CLI wait budget in `timeoutMs`, and adds regression tests plus accepted-ownership fallback handling in `agentCliCommand`.

## Best Possible Solution

Decide the intended gateway CLI timeout contract, then implement it end-to-end: either keep `--timeout` as the gateway-owned agent timeout, or add a real client-side accepted-run wait budget with a produced/exported accepted-ownership error path, aligned tests, and docs for `openclaw agent`.

## Reproduction Assessment

Yes. A static reproduction is high-confidence: applying the PR diff imports `isGatewayAcceptedOwnershipError` from `src/gateway/call.ts`, while current main has no such export; the existing tests mask that by mocking the missing gateway call API.

## Solution Assessment

No. The current PR is not the best complete solution because it adds a consumer and mocked test for an accepted-ownership error without adding the real gateway call/client contract that would export and throw that error.
