## Summary

Keep open: current main still has the PDF registry-only lookup that this PR is trying to fix, but the proposed helper regresses existing normalized registry resolution and inline-provider alias matching. The useful PDF custom-provider fallback should be repaired before merge, with awareness of the overlapping broader media-provider PR #53738.

## What This Changes

Adds a PDF-only resolver fallback from runtime registry lookup to `models.providers` inline provider models, wires it into `runPdfPrompt`, and adds a regression test for a registry miss.

## Best Possible Solution

Land a config-aware PDF model resolver that keeps the existing normalized registry lookup first, then falls back to `models.providers` using the embedded runner's normalized provider and provider-scoped model matching semantics. Maintainers should decide whether that narrow repair lands here or is folded into the broader #53738 media-tools provider-config work.
