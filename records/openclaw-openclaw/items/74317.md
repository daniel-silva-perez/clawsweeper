## Summary

Keep open: current main does not already contain the bundled guardrails plugin, and the existing hook surface can support the proposed shape, but the latest PR head still introduces security-sensitive credential, dynamic-code, and availability issues that need maintainer/secops review before merge.

## What This Changes

The PR adds a bundled `extensions/guardrails` plugin that registers `before_dispatch` allow/block checks using blacklist, HTTP-provider, and dynamic-import connectors, plus package metadata, static asset packaging, lockfile updates, and tests.

## Best Possible Solution

Decide the ownership model first: either land this as an official bundled security plugin after SecretRef-aware credential handling, bounded initialization fallback, docs/changelog alignment, dependency review, and secops approval, or ask the contributor to publish it as an external ClawHub/native plugin using the existing hook surface.

## Reproduction Assessment

Yes for the blocking review defects: configure an HTTP provider key as a SecretRef object and it is ignored by the string-only resolver, or register an HTTP/import backend whose `init()` never settles and the first matching `before_dispatch` waits indefinitely. For the feature itself, reproduction is not applicable because current main lacks the bundled plugin.

## Solution Assessment

No, not as-is. The plugin-shaped approach uses an appropriate extension seam, but a bundled security plugin should not merge before it supports the existing secret pipeline, deterministic timeout/fallback behavior during initialization, and explicit maintainer/secops ownership review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Make provider keys SecretRef-aware:** `extensions/guardrails/openclaw.plugin.json:26-28`
  - body: The new HTTP connector accepts provider API keys from plugin config, but the manifest declares `http.apiKey` and channel override `apiKey` as plain strings with no `configContracts.secretInputs` or sensitive UI hints, and the resolver only accepts strings. Bundled guardrail provider tokens would therefore live in normal plugin config instead of the existing SecretRef resolution/redaction path.
  - confidence: 0.93
- **[P2] Bound HTTP provider initialization waits:** `extensions/guardrails/index.ts:178`
  - body: If a registered HTTP provider's `init()` never settles, the first matching `before_dispatch` event awaits `entry.initPromise` with no guardrails timeout or fallback. That can stall inbound dispatch indefinitely instead of returning the configured fail-open/fail-closed result.
  - confidence: 0.86
- **[P2] Bound import connector initialization waits:** `extensions/guardrails/index.ts:322`
  - body: The import connector waits for module load and `init(args)` before the per-check timeout wrapper exists. A bad custom checker can hang external-channel dispatch during initialization instead of honoring `timeoutMs` and `fallbackOnError`.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: Security review needs attention because the PR adds credential-bearing outbound checks and trusted dynamic code execution without fully integrating the existing SecretRef pipeline or bounding initialization waits.

Concerns:

- **[high] Plaintext provider API keys:** `extensions/guardrails/openclaw.plugin.json:26`
  - body: `http.apiKey` and per-channel override keys are declared as plain strings with no SecretRef contract or sensitive UI hint, so guardrail provider tokens would bypass runtime secret resolution and redaction.
  - confidence: 0.93
- **[medium] Trusted local code execution surface:** `extensions/guardrails/src/import-connector.ts:23`
  - body: The import connector executes operator-specified local modules through `jiti`. That may be acceptable for trusted deployments, but bundled inclusion needs explicit docs and threat-model clarity around absolute paths, hot reload, and trusted code boundaries.
  - confidence: 0.84
- **[medium] Unbounded initialization can stall inbound dispatch:** `extensions/guardrails/index.ts:178`
  - body: HTTP and import connector initialization promises are awaited from `before_dispatch` without the plugin's configured timeout, making a hung provider/module an availability issue at a high-risk ingress boundary.
  - confidence: 0.87
