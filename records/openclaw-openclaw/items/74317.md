## Summary

Keep open: current main does not include the proposed guardrails plugin, and the PR is a live implementation candidate rather than obsolete cleanup. The current head still needs maintainer/secops review because it adds a bundled security-sensitive plugin with plaintext API-key config and unbounded connector initialization paths.

## What This Changes

The PR adds a bundled `extensions/guardrails` plugin with blacklist, HTTP-provider, and dynamic-import connectors for `before_dispatch` allow/block checks, plus packaging metadata, static asset handling, and tests.

## Best Possible Solution

Decide the ownership model first: either land this as an official bundled security plugin after SecretRef-aware credential handling, bounded initialization fallback, docs/changelog alignment, dependency review, and secops approval, or ask the contributor to publish it externally through ClawHub/native plugin distribution using the existing hook surface.

## Reproduction Assessment

Yes for the blocking review defects: configure an HTTP provider API key as a SecretRef object and the string-only schema/type path rejects or drops it; configure a custom HTTP/import backend whose `init()` never settles and the `before_dispatch` path can wait indefinitely. For the feature request itself, reproduction is not applicable because current main lacks the bundled plugin.

## Solution Assessment

No, not as-is. The plugin-shaped approach uses an appropriate extension seam, but the current bundled implementation is not the narrowest maintainable solution until credential resolution, initialization timeouts, and maintainer/secops ownership are resolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Make guardrails API keys SecretRef-aware:** `extensions/guardrails/openclaw.plugin.json:26-28`
  - body: The new HTTP connector accepts provider API keys from plugin config, but the manifest declares `http.apiKey` and channel override `apiKey` as plain strings and `HttpConfig.apiKey` is a string. That forces bundled guardrail provider tokens into normal plugin config instead of the existing SecretRef/redaction path, and SecretRef objects will not validate or resolve.
  - confidence: 0.93
- **[P2] Bound HTTP provider initialization waits:** `extensions/guardrails/index.ts:178`
  - body: If a registered HTTP provider's `init()` never settles, the first matching `before_dispatch` event awaits `entry.initPromise` before any guardrails timeout or fallback is applied. Because the hook registration has no timeout budget, inbound dispatch can stall indefinitely instead of returning the configured fail-open/fail-closed result.
  - confidence: 0.87
- **[P2] Bound import connector initialization waits:** `extensions/guardrails/index.ts:322`
  - body: The import connector waits for module load and `init(args)` to finish before the per-check timeout wrapper exists. A bad custom checker can therefore hang external-channel dispatch during initialization instead of honoring `timeoutMs` and `fallbackOnError`.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: Security review needs attention because the PR adds credential-bearing outbound checks and trusted dynamic code execution without fully integrating the existing SecretRef pipeline or bounding initialization waits.

Concerns:

- **[high] Plaintext provider API keys:** `extensions/guardrails/openclaw.plugin.json:26`
  - body: `http.apiKey` and per-channel override keys are declared as plain strings and resolved as strings, so guardrail provider tokens would bypass SecretRef resolution and redaction if this lands as a bundled plugin.
  - confidence: 0.93
- **[medium] Trusted local code execution surface:** `extensions/guardrails/src/import-connector.ts:23`
  - body: The import connector executes operator-specified local modules through `jiti`. That may be acceptable for trusted deployments, but bundled inclusion needs explicit docs and threat-model clarity around absolute paths, hot reload, and trusted code boundaries.
  - confidence: 0.84
- **[medium] Unbounded initialization can stall inbound dispatch:** `extensions/guardrails/index.ts:178`
  - body: HTTP and import connector initialization promises are awaited from `before_dispatch` without the plugin's configured timeout, making a hung provider/module an availability issue at a high-risk ingress boundary.
  - confidence: 0.87
