## Summary

Keep open. Current main does not already include the bundled guardrails plugin, and the linked SDK seam work does not replace the concrete plugin implementation, but the patch is not merge-ready because it adds a security-sensitive bundled plugin with unresolved credential, dynamic-code, network, and initialization-fallback issues.

## What This Changes

The PR adds a bundled `extensions/guardrails` plugin with `before_dispatch` allow/block checks, blacklist, HTTP, and dynamic-import connectors, static asset packaging, lockfile updates, and tests.

## Best Possible Solution

Decide the ownership model first: either land this as an official bundled security plugin after SecretRef-aware config, bounded initialization fallback, docs/changelog/dependency review, and secops approval, or ask the contributor to publish it as an external ClawHub/native plugin using the existing `before_dispatch` hook surface.

## Reproduction Assessment

Yes for the blocking review defects: configure `http.apiKey` as a structured SecretRef object and it is rejected or ignored by the string-only schema/resolver, or register an HTTP/import backend whose `init()` never settles and the first matching `before_dispatch` waits indefinitely. For the feature request itself, reproduction is not applicable because current main does not ship the bundled plugin.

## Solution Assessment

No, not as-is. The plugin-shaped approach uses the right extension seam, but a bundled security plugin should not merge before it supports the existing SecretRef pipeline, deterministic timeout/fallback behavior during initialization, and explicit maintainer/secops ownership review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Make provider keys SecretRef-aware:** `extensions/guardrails/openclaw.plugin.json:26-28`
  - body: The new HTTP connector accepts provider API keys from plugin config, but the manifest declares `http.apiKey` and channel override `apiKey` as plain strings with no `configContracts.secretInputs` contract, and the resolver only accepts strings. Bundled guardrail provider tokens would therefore live in ordinary plugin config instead of the existing SecretRef resolution/redaction path.
  - confidence: 0.93
- **[P2] Apply fallback timeout while HTTP providers initialize:** `extensions/guardrails/index.ts:178`
  - body: If a registered HTTP provider's `init()` never settles, the first matching `before_dispatch` event awaits `entry.initPromise` with no guardrails timeout or fallback. That can stall inbound dispatch indefinitely instead of returning the configured fail-open or fail-closed result.
  - confidence: 0.86
- **[P2] Apply fallback timeout while import modules initialize:** `extensions/guardrails/index.ts:322`
  - body: The import connector waits for module load and `init(args)` before the per-check timeout wrapper exists. A bad custom checker can hang external-channel dispatch during initialization instead of honoring `timeoutMs` and `fallbackOnError`.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: Security review needs attention because the PR adds credential-bearing outbound checks and trusted dynamic code execution without fully integrating SecretRef handling or bounding initialization waits.

Concerns:

- **[high] Plaintext provider API keys:** `extensions/guardrails/openclaw.plugin.json:26`
  - body: `http.apiKey` and per-channel override keys are declared as plain strings with no SecretRef contract or sensitive UI hint, so guardrail provider tokens would bypass runtime secret resolution and redaction.
  - confidence: 0.93
- **[medium] Trusted local code execution surface:** `extensions/guardrails/src/import-connector.ts:23`
  - body: The import connector executes operator-specified local modules through `jiti`. That may be acceptable for trusted deployments, but bundled inclusion needs explicit docs and threat-model clarity around absolute paths, hot reload, and trusted code boundaries.
  - confidence: 0.84
- **[medium] Unbounded initialization can stall inbound dispatch:** `extensions/guardrails/index.ts:178`
  - body: HTTP and import connector initialization promises are awaited from `before_dispatch` without the plugin's configured timeout, making a hung provider/module an availability issue at a high-risk ingress boundary.
  - confidence: 0.87
