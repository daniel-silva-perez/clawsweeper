## Summary

Keep open: current main has the generic `before_dispatch` hook surface but does not contain the proposed guardrails plugin, and this branch is a large security-sensitive bundled-plugin proposal tied to open security-labeled #72741. The remaining decision is not stale cleanup; it needs maintainer/security review on whether this belongs as an official bundled plugin or as an external ClawHub plugin, plus contract/docs alignment before merge.

## What This Changes

This PR adds a bundled `extensions/guardrails` plugin that registers `before_dispatch` allow/block checks with blacklist, HTTP-provider, and dynamic-import connectors, plus package metadata, static asset packaging, lockfile updates, and boundary/release tests.

## Best Possible Solution

Maintainers should choose the ownership model first: either accept this as an official bundled security plugin after secops/product review with resolved hook semantics and docs, or ask the contributor to publish it as an external ClawHub/native plugin that uses the existing hook surface. If it lands in core, the remaining review findings, dependency review, provider behavior, docs, and packaging checks should be completed before merge.
