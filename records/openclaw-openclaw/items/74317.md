## Summary

Keep open for maintainer review. Current main already has the generic `before_dispatch` hook and external plugin distribution path needed for a guardrails plugin, but it does not include this guardrails implementation; the PR is a large, security-sensitive bundled plugin with new in-process import execution, HTTP provider calls, runtime packaging, and lockfile changes, so the remaining decision is product/security ownership rather than conservative cleanup.

## What This Changes

The PR adds a bundled `extensions/guardrails` plugin with `before_dispatch` allow/block checks, blacklist/import/HTTP connectors, default keyword assets, runtime packaging wiring, boundary allowlist coverage, tests, and lockfile dependency changes.

## Best Possible Solution

Maintain the existing generic hook/API seam in core. If maintainers want this capability bundled, land it only after security review, connector correctness fixes, docs/config guidance, and clean dependency provenance; otherwise publish the guardrails implementation as an external ClawHub/npm plugin that uses `before_dispatch` without adding another optional bundled plugin to core.
