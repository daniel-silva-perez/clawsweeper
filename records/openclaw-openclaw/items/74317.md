## Summary

Keep open. Current main has the generic `before_dispatch` hook this feature can build on, but it does not contain the bundled guardrails plugin, and the proposed patch still has blocking security and availability issues around provider credentials and connector initialization.

## What This Changes

This PR adds a bundled `extensions/guardrails` plugin with blacklist, HTTP-provider, and dynamic-import connectors, runtime asset packaging, lockfile updates, and tests for pre-dispatch allow/block checks.

## Best Possible Solution

Decide the ownership model first. If maintainers accept a bundled security plugin, require SecretRef-aware config contracts and resolution, bounded initialization fallback, explicit trusted-code documentation, dependency review, docs/changelog coverage, and secops approval; otherwise route this as an external ClawHub/native plugin using the existing hook surface.

## Reproduction Assessment

Yes for the blocking review defects: configure `http.apiKey` as a SecretRef object and the proposed schema/resolver rejects or drops it, or use an HTTP/import backend whose `init()` never settles and the first matching `before_dispatch` waits without the configured plugin timeout. For the feature itself, reproduction is not applicable because current main lacks the bundled plugin.

## Solution Assessment

No, not as-is. The plugin-shaped approach uses the right general seam, but a bundled security plugin should not merge until it honors the existing SecretRef pipeline, applies deterministic timeout/fallback semantics during initialization, and has explicit maintainer/security ownership approval.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Make guardrail API keys SecretRef-aware:** `extensions/guardrails/openclaw.plugin.json:26-28`
  - body: The HTTP provider key is declared as a plain string and the resolver only accepts strings, while runtime SecretRef materialization for plugin config depends on manifest `configContracts.secretInputs`. Operators would have to store guardrail provider tokens as normal plugin config instead of using the existing SecretRef resolution and redaction path.
  - confidence: 0.93
- **[P2] Apply timeout before awaiting HTTP init:** `extensions/guardrails/index.ts:177-178`
  - body: `makeHttpBackendFn` awaits `entry.initPromise` before any guardrails timeout or fallback is applied. Since `before_dispatch` has no default hook timeout here, a custom provider whose `init()` never resolves can stall inbound dispatch instead of returning the configured fail-open or fail-closed decision.
  - confidence: 0.87
- **[P2] Apply timeout before awaiting import init:** `extensions/guardrails/index.ts:321-322`
  - body: The import connector waits for module load and optional `init(args)` before the per-check timeout wrapper exists. A trusted local checker with a never-settling init can hang external-channel dispatch instead of honoring `timeoutMs` and `fallbackOnError`.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The diff adds credential-bearing outbound checks and trusted dynamic code execution, and still needs SecretRef-aware config plus bounded initialization behavior before merge.

Concerns:

- **[high] Plaintext provider API keys:** `extensions/guardrails/openclaw.plugin.json:26`
  - body: `http.apiKey` and per-channel override keys are plain string fields without a `configContracts.secretInputs` declaration or sensitive UI hints, so guardrail provider tokens bypass the runtime SecretRef materialization/redaction path.
  - confidence: 0.93
- **[medium] Trusted local code execution surface:** `extensions/guardrails/src/import-connector.ts:23`
  - body: The import connector executes operator-specified local modules through `jiti`. That can be acceptable for trusted deployments, but bundled inclusion needs explicit docs and threat-model clarity around absolute paths, hot reload, and trusted code boundaries.
  - confidence: 0.84
- **[medium] Unbounded initialization can stall dispatch:** `extensions/guardrails/index.ts:177`
  - body: HTTP and import connector initialization promises are awaited from `before_dispatch` without the plugin's configured timeout, making a hung provider/module an availability issue at an external-channel ingress boundary.
  - confidence: 0.87
