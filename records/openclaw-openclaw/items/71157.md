## Summary

Keep this issue open. Current main still lacks the requested WhatsApp disappearing-message configuration and does not forward `ephemeralExpiration` on the active outbound paths; the two related implementation PRs, #73080 and #73083, are closed and unmerged.

## What This Changes

Add WhatsApp channel/account disappearing-message expiration config and apply the effective value to outbound replies across text, quoted, media, voice-note companion text, poll, and tool/channel send paths.

## Best Possible Solution

Implement this in the bundled WhatsApp plugin: add optional root and account config with account override precedence, resolve only positive effective values, merge the value into Baileys send options wherever outbound WhatsApp messages are emitted, and update schema/docs/tests/changelog without adding WhatsApp-specific policy to core.

## Reproduction Assessment

Yes. A source-level check on current main reproduces the gap: the proposed config keys are absent from strict schema and resolved account state, and every active outbound Baileys call omits `ephemeralExpiration`.

## Solution Assessment

Yes. The requested plugin-owned config and Baileys send-option forwarding is the narrowest maintainable fix; a generic core channel seam is not needed unless another channel needs the same transport-specific metadata.
