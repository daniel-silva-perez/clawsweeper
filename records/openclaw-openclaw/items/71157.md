## Summary

Keep this issue open. Current main still lacks the requested WhatsApp disappearing-message expiration config and does not forward Baileys `ephemeralExpiration` from the active outbound text, media, voice-note companion text, or poll send paths; the related implementation PRs are closed and unmerged.

## What This Changes

Request to add optional WhatsApp channel/account config that applies a default disappearing-message expiration to outbound WhatsApp text, quoted, poll, media, voice-note, and tool replies.

## Best Possible Solution

Implement this inside the bundled WhatsApp plugin: add typed and schema-validated channel/account config, resolve a positive effective expiration with account override semantics, and merge it into Baileys send options across every outbound WhatsApp transport path while preserving current behavior when unset, zero, or null.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: the proposed config is absent from strict schema/type definitions, resolved account state has nowhere to carry it, and current send code never passes `ephemeralExpiration` despite the pinned Baileys type supporting that option.

## Solution Assessment

Yes. The requested direction is the narrowest maintainable solution because this is WhatsApp-owned transport metadata and can be handled in the bundled WhatsApp plugin send/options layer without a new core channel contract.

## Repair Work Prompt

Implement support for WhatsApp outbound disappearing-message expiration. Add `channels.whatsapp.disappearingMessagesSeconds` and `channels.whatsapp.accounts.<accountId>.disappearingMessagesSeconds` with account-level override semantics. Accept positive effective values for Baileys `ephemeralExpiration`; treat unset, `0`, and `null` as no OpenClaw override. Keep the fix owned by the bundled WhatsApp plugin; do not add a generic core channel contract and do not change Baileys dependency versions or patches. Update TypeScript config types, zod schema, config metadata/docs, WhatsApp docs/config examples, and `CHANGELOG.md` with appropriate `Thanks @dougvk` and source PR credit for @hclsys if reusing that work. Resolve the effective expiration alongside existing merged account config, then merge it with existing quoted-message options on every outbound path: normal text, media, voice-note companion text, quoted replies, polls, and channel/tool sendPayload routes. Use #73080 and #73083 as source context only; adapt to current main and include complete schema/docs/test coverage. If a reliable Baileys credential default is identified, add it only with dependency-backed tests; otherwise leave unset behavior to Baileys/chat policy. Preserve existing behavior when no positive config is set.
