## Summary

Keep this issue open. Current main still lacks the requested WhatsApp disappearing-message expiration config and does not forward Baileys `ephemeralExpiration` from the active outbound text, media, voice-note companion text, or poll paths; the two related implementation PRs are closed and unmerged.

## What This Changes

Request to add optional WhatsApp channel/account config that applies a default disappearing-message expiration to outbound WhatsApp text, quoted, poll, media, voice-note, and tool sends.

## Best Possible Solution

Implement this inside the bundled WhatsApp plugin: add typed and schema-validated channel/account config, resolve a positive effective expiration with account override semantics, and merge it into Baileys send options across every outbound WhatsApp transport path while preserving current behavior when unset, zero, or null.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: the proposed config is absent from strict schema/type definitions, resolved account state has nowhere to carry it, and current send code never passes `ephemeralExpiration` despite Baileys supporting the option.

## Solution Assessment

Yes. The requested direction is the narrowest maintainable solution because this is WhatsApp-owned transport metadata and can be handled in the WhatsApp plugin send/options layer without a new core channel contract.
