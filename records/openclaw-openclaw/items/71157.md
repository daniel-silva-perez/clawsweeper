## Summary

Keep this issue open. Current main still lacks a WhatsApp channel/account disappearing-message expiration config, cannot carry an effective per-account expiration through resolved account state, and does not pass Baileys `ephemeralExpiration` on the active outbound text/media/voice-note/poll paths; the related implementation PRs were closed unmerged.

## What This Changes

Request support for a WhatsApp-specific `disappearingMessagesSeconds` channel/account setting that forwards a positive effective value as Baileys `ephemeralExpiration` on outbound replies.

## Best Possible Solution

Implement the feature inside the bundled WhatsApp plugin: add optional channel/account config and schema/docs metadata, resolve a positive effective value with account override semantics, and thread it through every WhatsApp outbound transport path that calls Baileys send APIs while preserving existing behavior when unset, zero, or null.

## Reproduction Assessment

Yes. A current-main source check reproduces the absence: the strict config schema lacks the proposed keys, resolved account state has no expiration field, and outbound send code never passes `ephemeralExpiration` to Baileys.

## Solution Assessment

Yes. The requested direction is the narrowest maintainable fix because this is WhatsApp plugin-owned config and transport metadata, not a generic core channel contract; the closed source PRs demonstrate a plausible shape but still need a merged, validated implementation.
