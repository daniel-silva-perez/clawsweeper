## Summary

Keep this PR open. Current main still collapses Gemini CLI OAuth extraction failures into the same generic setup error, while this PR remains a narrow diagnostics-and-tests change against the active bundled Google plugin. The only plausible superseding path is still open #54289, which would need a maintainer security/product decision to remove or gate automatic extraction rather than improve diagnostics.

## Best Possible Solution

Keep this PR open for maintainer review. If automatic Gemini CLI credential extraction remains supported, land a narrow diagnostics fix in extensions/google/oauth.credentials.ts with regression coverage around resolveOAuthClientConfig() for missing binary, missing credential source, parse failure, and read/extraction exceptions. If #54289 removes or gates extraction, close this PR as superseded by that security/product decision.
