## Summary

Keep open. Current main still has the root stale-provider retention behavior and provider-unaware context-window lookup paths, so this PR is not obsolete; the branch also needs focused maintainer review because it changes merge-mode semantics and mixes unrelated embedded-runner/cron behavior with unresolved review comments.

## What This Changes

This PR changes generated `models.json` merge behavior, switches several session/status/runtime context-token paths toward provider-qualified resolution, adds focused regressions, and includes embedded-runner/cron terminal-error handling changes.

## Best Possible Solution

Keep this PR open, but focus it: decide and document merge-mode orphan-provider semantics, finish provider-aware context-token resolution in gateway defaults, CLI session display, and cron accounting, preserve persisted `contextTokens` only as fallback when provider/model is known, and split unrelated embedded-runner or cron terminal-error changes into separate work.
