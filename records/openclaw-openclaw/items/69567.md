## Summary

Keep this PR open. The hardening is valid, but the branch only updates `.github/workflows/ci.yml`; current main still has the same derived checkout `auth_header` pattern in both Testbox workflows without `::add-mask::`, so the stated security coverage is incomplete.

## What This Changes

The branch adds `echo "::add-mask::${auth_header}"` after 11 derived checkout `auth_header` constructions in `.github/workflows/ci.yml`.

## Best Possible Solution

Extend the log-redaction hardening so every current custom checkout block that constructs `auth_header` masks it immediately after construction and before any Git command uses it, covering `.github/workflows/ci.yml`, `.github/workflows/ci-check-testbox.yml`, and `.github/workflows/ci-build-artifacts-testbox.yml` without changing job graph, permissions, runners, action refs, or checkout semantics.
