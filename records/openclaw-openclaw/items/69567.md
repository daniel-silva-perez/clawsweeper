## Summary

Keep open. Current main still has unmasked derived Basic-auth checkout headers: 13 `auth_header` constructions across the main CI workflow and two Testbox workflows, with no `::add-mask::${auth_header}` match. The PR is useful, narrow security hardening, but it needs maintainer review/update or an equivalent replacement patch to cover the current workflow surface before merge.

## What This Changes

The PR adds `echo "::add-mask::${auth_header}"` after derived checkout auth-header construction sites in `.github/workflows/ci.yml`.

## Best Possible Solution

Keep this PR open and review it as focused CI security hardening. Update the branch, or land an equivalent maintainer-owned replacement, so every current custom checkout block that derives `auth_header` in `.github/workflows/ci.yml`, `.github/workflows/ci-check-testbox.yml`, and `.github/workflows/ci-build-artifacts-testbox.yml` immediately registers it with `::add-mask::${auth_header}` before any Git command can use or print it. Keep the patch limited to masking the derived header and validate with the workflow security audit plus normal PR CI.
