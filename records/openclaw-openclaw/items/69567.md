## Summary

Keep open. The PR is useful and narrowly improves the main CI workflow, but the same derived Basic-auth checkout header still exists in the two Testbox workflows without runner masking, so the security hardening is incomplete before merge.

## What This Changes

The PR adds `echo "::add-mask::${auth_header}"` after derived Basic-auth header construction in 11 custom checkout blocks in `.github/workflows/ci.yml`.

## Best Possible Solution

Land focused CI hardening that masks every current custom checkout `auth_header` immediately after construction across `.github/workflows/ci.yml`, `.github/workflows/ci-check-testbox.yml`, and `.github/workflows/ci-build-artifacts-testbox.yml`, while keeping the patch limited to log redaction and leaving job graph, permissions, action refs, runners, and checkout behavior unchanged.
