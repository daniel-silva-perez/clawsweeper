## Summary

Keep this PR open. Current main still builds derived Basic-auth checkout headers from `github.token` in `.github/workflows/ci.yml` without registering `auth_header` with `::add-mask::`, so the PR is not implemented on main. The PR diff remains narrowly scoped security hardening, but it is non-mergeable against current main and needs a rebase/recount before review or landing.

## Best Possible Solution

Keep this PR open as a useful security-hardening contribution. The best path is to rebase it onto current main, add `echo "::add-mask::${auth_header}"` immediately after every current `auth_header` assignment and before any Git command that can use the header, avoid unrelated workflow, permission, action-ref, dependency, or script changes, and rerun the relevant workflow checks (`actionlint`, `security-scm-fast`, and CI) on the rebased head.
