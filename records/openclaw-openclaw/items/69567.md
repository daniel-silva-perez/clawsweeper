## Summary

Keep open. The PR is useful, narrow CI security hardening, and current main still has derived Basic-auth checkout headers without runner masking. It should not be closed as cleanup, but the branch is incomplete against current main because it masks `.github/workflows/ci.yml` only while the same custom checkout pattern remains in both Testbox workflows.

## What This Changes

The PR adds `echo "::add-mask::${auth_header}"` after derived auth-header construction in 11 custom checkout blocks in `.github/workflows/ci.yml`.

## Best Possible Solution

Land focused CI hardening that masks every current custom checkout `auth_header` immediately after construction across the main CI and Testbox workflows, while keeping the patch limited to log redaction and leaving job graph, permissions, runners, and checkout behavior unchanged.
