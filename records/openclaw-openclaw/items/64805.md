## Summary

Keep open. Current main still renders Feishu card footers with agent/model/provider identity only and does not pass session usage lookup data into the dispatcher, so the requested feature is not implemented on main. The PR is useful contributor work and the author addressed the earlier P2 freshness/fallback review comments in follow-up commit 020c6544da83336f709aa4f7ca6321d780017196, but it still needs maintainer review, a rebase against current Feishu test coverage, and live Feishu validation rather than cleanup closure or an automated replacement job.

## What This Changes

The PR threads Feishu session key/store path into the reply dispatcher, formats streaming card closeout footers with fresh token/context usage from the session store, and removes older Feishu monitor comment handler tests.

## Best Possible Solution

Rebase the PR onto current main, keep the session usage read on the narrow session-store helper surface, preserve current Feishu comment-monitor coverage, add focused reply-dispatcher tests for fresh usage, stale/missing usage, and fallback identity/model/provider notes, then merge if maintainers want token/context usage in Feishu streaming card footers.
