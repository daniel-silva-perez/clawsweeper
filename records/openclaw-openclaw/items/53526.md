## Summary

Keep open. Current main still applies the implicit daily reset default to all session types, and tests/docs still describe that behavior, so this PR is not implemented on main. The proposal is useful source work for the linked session-reset problem, but it needs maintainer product/privacy review and a refresh against the current reset-policy file before merge.

## What This Changes

The PR changes implicit session reset defaults so group and thread sessions use idle reset with a 7-day window while direct/DM sessions keep the daily default, and updates reset-policy tests accordingly.

## Best Possible Solution

If maintainers accept the behavior change, implement the type-aware implicit default in `src/config/sessions/reset-policy.ts`, keep explicit base/type/channel overrides authoritative, add exact group/thread `10080` coverage including the no-config path, and update user-facing session docs and changelog.
