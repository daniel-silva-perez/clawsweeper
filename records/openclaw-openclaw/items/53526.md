## Summary

Keep open. Current main has not implemented the requested behavior, and the proposed patch is not mergeable as-is because its new group/thread default is suppressed whenever any unrelated resetByType entry exists. The behavior change also increases default context retention in shared channels, so it needs maintainer product/privacy approval before a refreshed implementation is requested.

## What This Changes

The PR changes implicit session reset defaults so group and thread sessions use idle reset with a 10080-minute window while direct/DM sessions keep daily reset, and updates reset-policy tests.

## Best Possible Solution

If maintainers accept the behavior change, implement the type-aware implicit default in src/config/sessions/reset-policy.ts, keep base/type/channel overrides authoritative, add exact 10080 coverage for no-config and unrelated-override group/thread paths, and update the public session docs plus changelog.
