## Summary

Keep open. Current main still defaults group/thread sessions to the global daily reset unless an explicit policy applies, so this PR's intended behavior is not implemented on main. The PR is a plausible fix for the user problem raised in #32109, but the provided review context also identifies a real bug in the proposed guard, so it should be revised rather than closed as implemented.

## Best Possible Solution

Keep this PR open for revision or replace it with an updated patch against current main. The fix now belongs in `src/config/sessions/reset-policy.ts`: make the implied default type-aware for `group` and `thread`, scope the explicit-policy guard to `baseReset || typeReset`, preserve direct/DM daily defaults, add exact tests for no-config group/thread defaults and unrelated `direct`/`dm` overrides, and update the public session/config docs if the behavior changes.
