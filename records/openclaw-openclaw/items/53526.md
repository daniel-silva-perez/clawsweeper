## Summary

Keep open. Current main and the latest release still use the daily implicit reset default for unconfigured group/thread sessions, while the runtime already passes direct/group/thread reset type into the policy resolver. The PR addresses remaining behavior, but it needs maintainer product/privacy review and branch refresh because the submitted implementation has a reviewed logic bug and now targets a moved implementation file.

## What This Changes

The PR changes implicit session reset defaults so group and thread sessions use idle reset with a 7-day window while direct/DM sessions keep the daily default, and updates reset-policy tests.

## Best Possible Solution

If maintainers accept the behavior change, implement the type-aware implicit default in `src/config/sessions/reset-policy.ts`, keep explicit base/type/channel overrides authoritative, add exact `10080` coverage for group/thread including the no-config and unrelated-override paths, and update the public session docs plus changelog.
