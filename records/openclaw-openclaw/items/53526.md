## Summary

Keep open. Current main and the latest release still use the daily implicit reset default, while this PR is not merge-ready because its default-mode guard is suppressed by unrelated resetByType config and the branch targets the pre-split reset implementation. The requested behavior also changes shared-channel context retention, so it needs explicit maintainer product/privacy acceptance before a refreshed fix should land.

## What This Changes

The PR changes implicit group and thread session reset defaults to idle with a 10080-minute window while keeping direct/DM sessions daily, and updates reset-policy coverage.

## Best Possible Solution

If maintainers accept the behavior change, implement the type-aware implicit default in src/config/sessions/reset-policy.ts, preserve explicit base/type/channel override precedence, scope the implied-default check to the current reset type, add exact 10080 coverage for no-config and unrelated-override group/thread paths, and update the session docs plus changelog.
