## Summary

Keep open. Current main still applies the implicit daily reset default to unresolved group/thread sessions, so the PR’s intended behavior is not implemented on main. The PR remains a plausible focused fix, but it needs maintainer review because the submitted patch has a known guard bug, targets an older file layout, and changes default context-retention behavior for shared conversations.

## What This Changes

The PR changes session reset policy defaults so group and thread sessions use idle reset with a 7-day window while direct sessions remain on daily reset.

## Best Possible Solution

Keep this PR open for maintainer review. If maintainers accept the default-behavior change, refresh it against current main by implementing the type-aware default in `src/config/sessions/reset-policy.ts`, scoping the explicit-policy guard to the current reset type, pinning group/thread `idleMinutes` to `10080` in tests, adding no-config group/thread coverage, and updating user-facing session docs/changelog.
