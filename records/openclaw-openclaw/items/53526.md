## Summary

Keep open. Current main and the latest release still use the implicit daily reset default, so this PR is not obsolete, but the submitted branch is not merge-ready: the implementation target has moved to `reset-policy.ts`, the patch has a type-override logic bug already called out in review, and changing default retention for group/thread conversations needs maintainer product/privacy judgment.

## What This Changes

The PR changes implicit session reset defaults so group and thread sessions use idle reset with a 7-day window while direct/DM sessions keep daily reset, and updates reset-policy tests.

## Best Possible Solution

If maintainers accept the behavior change, implement the type-aware implicit default in `src/config/sessions/reset-policy.ts`, keep explicit base/type/channel overrides authoritative, add exact `10080` coverage for no-config and unrelated-override group/thread paths, and update the user-facing session docs plus changelog.
