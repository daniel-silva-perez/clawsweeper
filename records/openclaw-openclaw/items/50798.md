## Summary

Keep open. Current main still lacks a supported visible coordinator-to-ACP thread send path that bypasses main Discord session creation: `sessions_send` remains an internal agent turn injection, the public send schemas do not expose `visible` or `proxyOnly`, and both the agent message path and gateway send path still write outbound session metadata when a route resolves. Related ACP/thread fixes cover bound follow-ups and visible replies, but not this exact proxy-only visible instruction gap.

## What This Changes

Request a visible coordinator-to-ACP Discord thread messaging path that posts into an ACP-bound thread without creating or updating a main Discord channel/thread session.

## Best Possible Solution

Add a narrow, explicit visible proxy-delivery contract after maintainer API review: it should post the coordinator instruction through the requested Discord account into the ACP-bound thread, bypass outbound session-entry creation only for that call, preserve ordinary session metadata writes for normal sends, and document the supported ACP coordination workflow.
