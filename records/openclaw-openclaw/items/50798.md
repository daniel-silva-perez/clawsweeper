## Summary

Keep open. Current main still lacks a supported initiating visible coordinator-to-ACP-thread delivery path with a metadata bypass; the implemented ACP binding and A2A work covers bound human follow-ups and visible follow-up/announce output, not the proxy-only instruction path described here.

## What This Changes

The issue requests a supported way for a coordinator agent to send visible messages into an ACP thread-bound Discord session without creating a main Discord channel session.

## Best Possible Solution

Add a narrow, explicit visible proxy-delivery contract after maintainer API and security review. The contract should post the coordinator-authored instruction through the selected Discord account into the ACP-bound thread, bypass main outbound session entry creation only for that authorized call, preserve normal metadata writes everywhere else, and document the supported ACP coordination workflow.
