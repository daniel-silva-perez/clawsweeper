## Summary

Keep open. Current main still lacks the requested supported path for a coordinator to post the initiating instruction visibly into an ACP-bound Discord thread while bypassing normal main-session metadata creation; existing code and docs cover bound follow-up routing and visible announce/final delivery, not proxy-only initiating delivery.

## What This Changes

The issue requests visible coordinator-to-ACP-thread messaging for ACP thread-bound Discord sessions without creating `agent:main:discord:channel:<thread_id>` session metadata.

## Best Possible Solution

Add a narrow, explicit visible proxy-delivery contract after maintainer API and security review. It should post the coordinator-authored instruction through an authorized Discord account into the ACP-bound thread, bypass only the derived main outbound session entry for that explicit call, preserve normal metadata writes elsewhere, and document the supported ACP coordination workflow.
