## Summary

Keep open. Current main still lacks a supported visible coordinator-to-ACP-thread delivery path that bypasses main Discord channel-session metadata creation; the related A2A work only surfaces follow-up/announce output after an internal send, not the initiating visible instruction path described here.

## What This Changes

Request a way for a coordinator agent to post a visible instruction into an ACP-bound Discord thread without creating or updating an `agent:main:discord:channel:<thread_id>` session entry.

## Best Possible Solution

Add a narrow, explicit visible proxy-delivery contract after maintainer API review. It should post the coordinator-authored instruction through the requested Discord account into the ACP-bound thread, bypass outbound main-session entry creation only for that authorized call, preserve normal metadata writes elsewhere, and document the supported ACP coordination workflow.
