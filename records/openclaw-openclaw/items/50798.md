## Summary

Keep open. Current main still lacks a supported coordinator-originated visible send into an ACP-bound Discord thread that bypasses main-agent Discord channel-session metadata creation; the existing A2A work covers internal injection and visible follow-up/announce delivery, not the initiating visible proxy message requested here.

## What This Changes

Request a visible coordinator-to-ACP-thread delivery path, such as `sessions_send visible`, `message proxyOnly`, or an ACP-bound-thread exclusion rule, without creating main Discord channel-session stubs.

## Best Possible Solution

Add a narrow, explicit visible proxy-delivery contract after maintainer API and security review. It should post the coordinator-authored instruction through an authorized Discord account into the ACP-bound thread, bypass only the derived main outbound session entry for that explicit call, preserve normal metadata writes elsewhere, and document the supported ACP coordination workflow.
