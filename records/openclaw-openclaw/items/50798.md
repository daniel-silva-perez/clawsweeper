## Summary

Current `main` still does not implement visible coordinator-to-ACP thread messaging without main Discord session creation. `sessions_send` remains an internal-delivery tool with `deliver: false`, the gateway `sessions.send` and `send` schemas have no `visible` or `proxyOnly` field, and both the agent `message` tool path and gateway `send` path still resolve outbound session routes and record outbound session metadata for Discord channel/thread targets. Adjacent ACP/thread fixes make bound-thread replies and inbound routing work, but they do not remove the outbound proxy-send stub side effect described here.

## Best Possible Solution

Keep this issue open and implement a narrow visible proxy-delivery path for ACP-bound Discord threads. The safest product shape appears to be an explicit `message`/gateway send option that posts through the selected Discord account while bypassing outbound session entry creation, with tests proving normal Discord outbound sends still record session metadata. A `sessions_send visible` variant would require an additive gateway protocol extension and docs, so it should be chosen deliberately if maintainers prefer that UX.
