## Summary

Keep open. Current main has useful ACP/thread-binding improvements, including routing inbound Discord bound-thread messages to the bound ACP/subagent session, and `sessions_send` now runs an A2A announce path. However, the exact requested capability is still not present: `sessions_send` has no `visible` option, the `message` tool has no `proxyOnly` option, and the generic outbound send path still records an outbound session entry for sends. That leaves the reported no-stub visible proxy delivery requirement unresolved.

## Best Possible Solution

Keep this issue open and implement a first-class proxy-visible delivery path: either `sessions_send({ visible: true })` or `message({ proxyOnly: true })` should post the coordinator-authored instruction to the bound thread using the target ACP session's binding/delivery context while skipping `ensureOutboundSessionEntry` for the coordinator/main Discord channel route. Add a regression test that sends to a Discord ACP-bound thread and asserts no `agent:main:discord:channel:<thread_id>` session entry is created, plus docs for the supported ACP thread-bound visible A2A workflow.
