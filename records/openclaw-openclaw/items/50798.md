## Summary

Keep open. Current main still supports bound-thread inbound routing and visible follow-up/announce delivery, but it does not provide a supported initiating visible coordinator-to-ACP-thread send that bypasses normal outbound main-session metadata creation.

## What This Changes

Request a visible coordinator-to-ACP worker thread delivery path, such as `sessions_send({ visible: true })`, `message({ proxyOnly: true })`, or an ACP-bound Discord thread exclusion rule, without creating main Discord channel-session stubs.

## Best Possible Solution

Add a narrow, explicit visible proxy-delivery contract after maintainer API and security review. The implementation should post the coordinator-authored instruction through the authorized Discord account into the ACP-bound thread, bypass only the derived main outbound session entry for that explicit call, keep normal metadata writes everywhere else, and document the supported ACP coordination workflow.
