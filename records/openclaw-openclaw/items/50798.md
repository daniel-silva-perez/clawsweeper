## Summary

Keep this issue open: current main still lacks a supported visible coordinator-to-ACP thread send path that bypasses main Discord session creation. The existing `sessions_send` path remains internal-only, the gateway/tool schemas have no `visible` or `proxyOnly` option, and both outbound message paths still create or update outbound session metadata when a Discord channel/thread route is resolved.

## What This Changes

Add a visible agent-to-agent/proxy delivery path for ACP-bound Discord thread sessions that posts into the bound thread without creating a main `agent:main:discord:channel:<thread_id>` session entry.

## Best Possible Solution

Keep the issue open and implement a narrow additive visible proxy-delivery path, preferably an explicit `message`/gateway `send` option that posts through the requested Discord account while bypassing outbound session-entry creation only for that call. Preserve normal session metadata writes for ordinary outbound sends, and document the ACP-bound thread coordination workflow.
