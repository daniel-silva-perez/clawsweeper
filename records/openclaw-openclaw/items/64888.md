## Summary

Keep open. Current main still collapses user aborts without payloads into the existing blocked liveness state, and several embedded-run rethrow paths can still leave terminal lifecycle metadata without the requested replay/liveness detail. The related merged runtime hardening improved adjacent fallback/lifecycle behavior, but the provided #70772 context explicitly leaves cancellation/liveness cleanup to focused follow-up work.

## What This Changes

The issue requests an embedded-run error boundary that always surfaces terminal lifecycle metadata plus a distinct cancelled liveness state for user-initiated aborts.

## Best Possible Solution

Add cancelled to the embedded liveness contract, have aborted non-timeout terminal runs surface cancelled while preserving blocked for timeouts and hard runtime errors, and ensure every embedded rethrow path emits replay/liveness terminal metadata through either the attempt lifecycle sink or the caller backstop. Cover the abort and forced-throw cases with focused runner/lifecycle tests.
