## Summary

Current main still has the reported Control UI refresh-during-streaming gap. Reconnect clears the local active run id, `chat.history` does not return active-run metadata, and the chat toolbar only shows Stop/Queue when `chatRunId` is present.

## Best Possible Solution

Keep this issue open. Add an authoritative active-run hydration path for Control UI on page load/reconnect, likely through hello/bootstrap or `chat.history`, returning the active run id plus enough buffered stream state for the visible session. Restore `chatRunId`, `chatStream`, and `chatStreamStartedAt` before rendering controls, and add regression coverage for refresh during active streaming plus terminal cleanup.
