## Summary

Current main still leaves the Control UI refresh-during-streaming regression unresolved. A refreshed/reconnected page clears local active-run state, neither hello nor chat history hydrates an active run id, and the toolbar still shows Stop/Queue only when `chatRunId` is present.

## Best Possible Solution

Keep this issue open. Add a source-of-truth active-run hydration path for Control UI on page load/reconnect, likely through hello/bootstrap or `chat.history`, returning the active run id plus enough buffered stream state for the visible session. Restore `chatRunId`, `chatStream`, and `chatStreamStartedAt` before rendering controls, and add regression coverage for refresh during active streaming plus terminal cleanup.
