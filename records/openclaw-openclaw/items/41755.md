## Summary

Current main still leaves this Control UI refresh regression unresolved. The prior ClawSweeper comment remains accurate against 891c7d9f1c1f5e0f52935fadfeacf695b0ef1eb8: a refreshed page/reconnect clears local active-run state, chat history does not hydrate an active run id or stream snapshot, and the toolbar still renders Stop/Queue only from `chatRunId`.

## Best Possible Solution

Keep this issue open. Add an authoritative active-run hydration path for Control UI on page load/reconnect, likely through hello/bootstrap or `chat.history`, returning the active run id plus enough buffered stream state for the visible session. Then restore `chatRunId`, `chatStream`, and `chatStreamStartedAt` before rendering controls, with regression coverage for refresh during active streaming and terminal cleanup.
