## Summary

Keep open. Current main still counts every top-level session file ending in `.jsonl` for `openclaw memory status`, while the session indexing path uses the usage-counted transcript helper that excludes compaction checkpoint snapshots. The reported `Indexed: 3/4 files` mismatch can still happen when a `<session>.checkpoint.<uuid>.jsonl` snapshot is present.

## Best Possible Solution

Keep this issue open and make a narrow memory-core fix: align `scanSessionFiles` with the same usage-counted session transcript corpus used by indexing, preferably by exposing/reusing the existing predicate through the memory host runtime contract. Add focused `memory status` coverage for a normal transcript plus a `.checkpoint.<uuid>.jsonl` snapshot, and add a user-facing changelog fix crediting @xinzf.
