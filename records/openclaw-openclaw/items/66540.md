## Summary

Keep open: current main still contains the reported false-default veto in the shared reply threading helper, and the linked fix PRs were closed without merging. The issue is narrow and still actionable, but the repair should account for the command path that also writes `replyToCurrent: false`.

## What This Changes

The issue requests that implicit reply threading still assign `replyToId` when a payload carries the default `replyToCurrent: false` value from directive parsing.

## Best Possible Solution

Update the shared reply-threading contract so default parser output does not suppress implicit threading, add a regression test for `replyToCurrent: false` payloads, and keep explicit opt-out behavior on a separate policy path if command replies still need it.
