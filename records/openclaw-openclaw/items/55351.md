## Summary

Keep open. Current main does not implement the Open WebUI UUID session preservation, and the PR contains useful source work, but the branch is not mergeable as-is because the session fix still has concrete edge-case regressions and the branch also carries unrelated auth/transport/runtime changes that need separate owner review.

## What This Changes

The PR preserves Open WebUI UUID session keys across gateway store resolution and Control UI URL handling, threads agent context into transcript lookup, adds regression tests, and also adds unrelated Codex proxy transport, model auth, compaction auth, and outbound routing changes.

## Best Possible Solution

Split the work: land a narrow Open WebUI UUID session fix that preserves agent-store round trips and normalizes legacy aliases correctly, then move the Codex proxy/auth/routing changes into separate owner-reviewed PRs with their own tests and security review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Keep UUID keys scoped in combined stores:** `src/gateway/session-utils.ts:640-642`
  - body: Returning bare UUIDs from session-key canonicalization also affects combined per-agent store listings. UUID entries from an `ops` store become unscoped `<uuid>` rows, so later `sessions.get`, patch, or delete resolves that key through the current default agent store and can return empty or wrong history after default-agent changes or with multiple agent stores.
  - confidence: 0.87
- **[P2] Normalize UUID aliases for the configured default agent:** `ui/src/ui/app-settings.ts:48-50`
  - body: `normalizeChatUrlSessionKey` only strips `agent:main:<uuid>`, but the legacy prefixed key was generated with whichever agent was default. Installs with a default such as `ops` will keep `agent:ops:<uuid>` in the Control UI URL/settings and continue the split identity instead of converging on the bare UUID.
  - confidence: 0.86
- **[P2] Resolve main-key aliases before preserving UUIDs:** `src/gateway/session-utils.ts:665-667`
  - body: The UUID fast path runs before the existing `session.mainKey` alias handling. If an operator configured a UUID-shaped main key, `resolveSessionStoreKey` now returns the bare UUID instead of the canonical main session key, splitting clients that use the configured main alias.
  - confidence: 0.8
- **[P2] Send the system prompt only once:** `src/agents/pi-embedded-runner/codex-proxy-transport.ts:728-729`
  - body: `buildRequestBody` sets `instructions: context.systemPrompt`, while `convertResponsesMessages` already injects the same prompt as a system/developer input message. That duplicates every system prompt in Codex proxy calls, increasing token use and potentially changing behavior for long prompts.
  - confidence: 0.88
