## Summary

Keep this PR open. Current main still does not implement the Open WebUI UUID-session preservation this PR targets: bare UUID-like session keys are still canonicalized under the default agent, the Control UI still applies `session` URL params directly, and `sessions.get` still reads transcripts without passing the resolved agentId. The PR should not be auto-closed as implemented; it needs maintainer review or a narrowed replacement because the branch also includes unrelated agent/Codex transport and auth changes with unresolved security-sensitive review comments.

## Best Possible Solution

Keep this PR open for maintainer handling, but require a narrowed branch or replacement PR. The focused fix should preserve bare Open WebUI UUID session keys in gateway store resolution, keep reciprocal bare/prefixed lookup aliases readable, normalize Control UI UUID session URLs without hard-coding only `main`, pass resolved agent context into transcript lookup, and add regression tests for those paths. The unrelated Codex transport/auth/compaction/routing work should be split out or removed before this session fix is considered for merge.
