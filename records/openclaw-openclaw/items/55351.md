## Summary

Keep open. The underlying Open WebUI UUID session split is still real on current main, but this PR is not merge-ready: the useful session fix still has canonicalization edge cases, and the branch has grown into unrelated credentialed Codex proxy/auth/routing work that needs maintainer and security review before any merge path.

## What This Changes

The PR branch preserves Open WebUI UUID session keys across gateway store resolution and Control UI URLs, threads agent context into transcript lookup, adds regression tests, and also adds Codex proxy transport, compaction-auth, model-auth, and outbound routing changes.

## Best Possible Solution

Land a narrow Open WebUI UUID session fix that preserves bare UUIDs without losing agent ownership, normalizes legacy aliases for any default agent, resolves configured main aliases first, and keeps history/live subscriptions on one canonical identity; move the Codex proxy/auth/routing changes into separate owner-reviewed PRs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve agent scope for UUID keys in combined stores:** `src/gateway/session-utils.ts:643-644`
  - body: Returning bare UUIDs from `canonicalizeSessionKeyForAgent` also runs while per-agent stores are merged for gateway listings. A UUID row from an `ops` store can be listed as an unscoped key, so later get/patch/delete resolves through the current default agent store and can miss or mutate the wrong history.
  - confidence: 0.87
- **[P2] Normalize UUID aliases for non-main default agents:** `ui/src/ui/app-settings.ts:51-52`
  - body: `normalizeChatUrlSessionKey` strips only `agent:main:<uuid>`, but the old canonicalizer used whichever agent was default. Installs that had `ops` or another default keep `agent:<id>:<uuid>` in URL/settings and can continue the split identity this PR is meant to converge.
  - confidence: 0.86
- **[P2] Resolve main-key aliases before UUID preservation:** `src/gateway/session-utils.ts:668-670`
  - body: The UUID fast path runs before the existing `session.mainKey` alias handling. If an operator configured a UUID-shaped main key, this now returns the bare UUID instead of the canonical main session key, splitting clients that use the configured main alias.
  - confidence: 0.8
- **[P2] Canonicalize UUID aliases for message subscriptions:** `src/gateway/session-utils.ts:668-670`
  - body: Bare UUID and `agent:<id>:<uuid>` forms remain different canonical keys, while session-message subscriptions and transcript fan-out use exact string keys. A legacy client subscribed on the prefixed form can miss live messages emitted under the bare UUID form.
  - confidence: 0.78
- **[P2] Send the system prompt only once:** `src/agents/pi-embedded-runner/codex-proxy-transport.ts:722-723`
  - body: `buildRequestBody` sets `instructions` from `context.systemPrompt`, while `convertResponsesMessages` also injects that prompt into the input messages. That duplicates system instructions on every Codex proxy call and can change behavior or consume context unnecessarily.
  - confidence: 0.88
