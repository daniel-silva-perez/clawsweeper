## Summary

Keep this issue open. Current main still preserves an explicit node `workdir=/Users/...`, and the headless node-host approval planner still runs the reported synchronous canonical-cwd path without a macOS firmlink/TCC-specific fix. The related shipped change only avoids forwarding a gateway default cwd when no workdir was explicitly requested.

## Best Possible Solution

Implement a targeted node-host fix in `src/node-host/invoke-system-run-plan.ts`: handle macOS `/Users` firmlink canonicalization or permission-context differences without weakening symlink/directory-drift protections, add regression coverage for explicit `/Users`-style cwd behavior, and clarify whether `OPENCLAW_NODE_EXEC_HOST=app` is the intended workaround for launchd/TCC cases.
