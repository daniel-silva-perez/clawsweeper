## Summary

Keep this PR open. Current `main` and the latest release still use the old `OPENCLAW_HOME` / `HOME` / `USERPROFILE` / `os.homedir()` fallback chain with no guarded Android/Termux `PREFIX` home fallback, while this PR remains the focused implementation candidate for the linked Termux startup crash and tilde-expansion edge case. The PR discussion shows earlier review concerns were addressed on the branch by sharing the helper with tilde expansion and rejecting generic Android chroot prefixes.

## Best Possible Solution

Review and land this focused PR, or a maintainer-equivalent implementation, preserving existing `OPENCLAW_HOME`, `HOME`, `USERPROFILE`, `OPENCLAW_STATE_DIR`, and `OPENCLAW_CONFIG_PATH` precedence while adding the guarded Termux `PREFIX` fallback for both normal home resolution and tilde expansion. Keep the regression tests and docs/changelog update, and decide whether the memory-host SDK resolver needs the same behavior in this PR or a follow-up.
