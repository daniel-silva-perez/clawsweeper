## Summary

Keep this PR open. Current main and the latest release still use the old home-directory fallback chain with no Android/Termux PREFIX-derived fallback, while this PR remains the focused implementation candidate for the linked Termux startup crash and the tilde-expansion edge case. The branch discussion also shows the earlier review concern was addressed by adding a stricter Termux guard and sharing the helper with OPENCLAW_HOME tilde expansion.

## Best Possible Solution

Review and land this focused PR, preserving the existing OPENCLAW_HOME, HOME, USERPROFILE, OPENCLAW_STATE_DIR, and OPENCLAW_CONFIG_PATH precedence while adding the guarded Termux PREFIX fallback for both normal home resolution and tilde expansion. Keep the regression tests and docs/changelog update, and consider whether the duplicate memory-host-sdk resolver needs the same behavior or an explicit follow-up.
