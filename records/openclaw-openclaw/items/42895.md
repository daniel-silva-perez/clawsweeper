## Summary

Keep this PR open. Current main still resolves OpenClaw home through OPENCLAW_HOME, HOME, USERPROFILE, and os.homedir() only, so the Android/Termux PREFIX fallback proposed here is not implemented on main. The PR remains a focused implementation candidate and its later commit addresses the earlier review gap around OPENCLAW_HOME tilde expansion.

## Best Possible Solution

Keep this PR open, refresh it against current main if needed, and review or land the focused Termux home-resolution fix with regression tests. Preserve existing OPENCLAW_HOME, HOME, USERPROFILE, OPENCLAW_STATE_DIR, and OPENCLAW_CONFIG_PATH precedence, and update the environment docs when the fallback ships.
