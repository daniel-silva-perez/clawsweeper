## Summary

Keep this PR open. The checked-out main, the newer remote main ref inspected read-only, and the latest release still use the old `OPENCLAW_HOME` / `HOME` / `USERPROFILE` / `os.homedir()` fallback chain with no guarded Android/Termux `PREFIX` fallback. The PR remains the focused implementation candidate, and the discussion shows the earlier tilde-expansion and generic-Android-chroot review concerns were addressed on the branch.

## Best Possible Solution

Review and land this focused PR, or a maintainer-equivalent implementation, preserving `OPENCLAW_HOME`, `HOME`, `USERPROFILE`, `OPENCLAW_STATE_DIR`, and `OPENCLAW_CONFIG_PATH` precedence while adding a guarded Termux `PREFIX` fallback for both normal home resolution and tilde expansion. Keep the regression tests and docs/changelog update, and either align the memory-host SDK resolver or explicitly track that as a follow-up.
