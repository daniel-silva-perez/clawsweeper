## Summary

Keep open. Current main still matches the reported mismatch: the headless node host declares and implements `system.execApprovals.get/set`, and the dedicated approvals CLI path can invoke them, but the gateway command allowlist used during node connection still omits those commands before `node.describe` and the Control UI see the node command list.

## Best Possible Solution

Keep this issue open. Fix the gateway node command policy so supported headless node hosts can advertise `system.execApprovals.get/set` by default, then add regression coverage for `resolveNodeCommandAllowlist`, node connection/describe behavior, and the Control UI exec-approvals target selector. Preserve the existing generic `node.invoke` block so edits continue to go through `exec.approvals.node.*`.
