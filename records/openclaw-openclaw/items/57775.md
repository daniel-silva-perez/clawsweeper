## Summary

Keep open. Current main still appears to have the reported mismatch: the headless node host declares and implements `system.execApprovals.get/set`, but the gateway filters node-declared commands through a platform allowlist that omits those commands before `node.list` / `node.describe` and the Control UI see them.

## Best Possible Solution

Keep this issue open and fix the gateway/node command policy so headless/macOS node hosts can advertise `system.execApprovals.get/set` by default on supported platforms, while preserving the `node.invoke` block for those commands. Add regression coverage for `resolveNodeCommandAllowlist`, the node connect/describe path, and the Control UI exec-approvals target selector.
