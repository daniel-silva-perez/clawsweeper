## Summary

Current main already solves the plugin ACP orchestration capability through the supported `openclaw/plugin-sdk/acp-runtime` SDK subpath, and the related tracking issue was closed by a maintainer as implemented with that same source-backed rationale. The exact `api.runtime.acp.*` convenience namespace from this PR is not present, but the user problem no longer requires this branch.

## What This Changes

The PR adds `api.runtime.acp.spawn()` and `api.runtime.acp.prompt()` behind `plugins.allowAcpSpawn`, updates runtime types/mocks/tests/config schema, and includes Discord channel-snowflake normalization for ACP thread binding.

## Best Possible Solution

Keep the shipped `openclaw/plugin-sdk/acp-runtime` seam as the canonical plugin-facing ACP path, and only revisit an injected `api.runtime.acp` convenience wrapper if maintainers explicitly want that ergonomic alias on top of the existing SDK contract.
