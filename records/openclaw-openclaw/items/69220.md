## Summary

Current main still combines tagged Google/Gemini reasoning-output instructions with native Gemini thinking payloads, matching the reported failure mode. The latest inspected release, v2026.4.26, still contains the same combination. Related PR #67781 is adjacent empty-payload work, but it does not supersede the root tagged-vs-native reasoning conflict.

## Best Possible Solution

Fix the bundled Google/Gemini reasoning-output contract in the Google plugin/shared provider family so API paths that send native Gemini thinking payloads resolve to native output mode and do not inject `<think>/<final>` directives. Keep Gemini replay sanitization and thought-signature handling intact, add focused provider/runtime tests, and review #67781 separately as adjacent empty-payload handling rather than a substitute for this root fix.
