## Summary

Keep open. Current main still forces Gemini/Google reasoning output to tagged `<think>/<final>` mode while the direct Google transport also sends native Gemini `thinkingConfig` with `includeThoughts: true`, matching the reported conflict. The latest `v2026.4.26` tag still contains the same behavior.

## Best Possible Solution

Keep this issue open and fix the Google/Gemini provider reasoning-output contract: native Gemini thinking should remain enabled through `thinkingConfig`, but direct Google/Gemini provider paths should not inject tagged `<think>/<final>` prompt requirements. Update the provider hook/fallback tests and add a regression proving Gemini reasoning no longer sets `reasoningTagHint` while native `thinkingConfig` is still sent.
