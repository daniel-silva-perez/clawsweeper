## Summary

Current main still has the Mattermost bug this PR targets: partial assistant text updates the active draft stream, then `onToolStart` writes the tool status into the same pending draft slot without flushing or rotating the post. The PR remains a focused implementation candidate, not implemented or superseded on main.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to preserve streamed Mattermost assistant text across tool-start events using the existing draft stream lifecycle, add a focused regression test for partial text followed by tool start, and reconcile the docs if the accepted behavior intentionally starts a separate status post at tool boundaries.
