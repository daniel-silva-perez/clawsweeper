## Summary

Keep open. Current main has important adjacent fixes for replay/legacy cron spam, especially #65138 and #70737, but the requested per-phase last-emitted cutoff or cross-cycle duplicate suppression is not present. The light phase still builds output from the fixed lookback window of recent short-term entries, passes the same capped snippets into Dream Diary generation, and the diary append path writes another timestamped entry without automatic duplicate suppression across distinct timestamps.

## Best Possible Solution

Keep this open for a maintainer decision on the exact product contract. The most likely code direction is a targeted memory-core change that records a per-phase Dream Diary input/body fingerprint or last-emitted state under `memory/.dreams/` and skips appending identical light-phase narrative material across a recent sweep window, while preserving deep promotion ranking and the existing legacy replay fixes.
