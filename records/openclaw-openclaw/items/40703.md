## Summary

Keep this PR open. Current main still does not implement the PR's fail-fast startup auth guard, but the PR is stale against the post-attach startup split and would use a narrower auth check than runtime model execution, so it should not land unchanged.

## Best Possible Solution

Keep this PR open for revision rather than closing or landing it unchanged. Rebase or rewrite the guard into `src/gateway/server-startup-post-attach.ts` before channel startup, preserve `OPENCLAW_SKIP_CHANNELS` and `OPENCLAW_SKIP_PROVIDERS`, and base the check on the same non-mutating auth contract runtime execution uses, such as `hasAvailableAuthForProvider` or a safe `resolveApiKeyForProvider` probe supplied with config and agentDir. Add focused tests for missing auth, config-backed apiKey auth, Bedrock/aws-sdk auth, synthetic local/plugin auth, skip-channel bypasses, and non-circular remediation text.
