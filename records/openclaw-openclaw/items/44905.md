## Summary

Keep open. Current main covers some related cases, including exact silent payload normalization and the merged commentary-phase fix from #61282, but the remaining Discord/internal-envelope leak family is not fully handled: mixed `NO_REPLY ... to=functions...` text and raw envelope-looking text can still pass as ordinary outbound text.

## What This Changes

Prevent Discord and shared outbound delivery from exposing internal tool-call and control traces such as `NO_REPLY`, `commentary`, `to=functions.*`, JSON tool envelopes, `REPLY_SKIP`, and `ANNOUNCE_SKIP`.

## Best Possible Solution

Add a conservative shared outbound/internal-trace sanitizer at the delivery boundary, plus a Discord send-level exact silent-token backstop where appropriate, so malformed internal control text is consumed before any user-facing channel send while preserving legitimate content and media-only replies.

## Repair Work Prompt

Fix the remaining internal trace leakage described in openclaw/openclaw#44905. Current main already handles exact `NO_REPLY` payloads and #61282 handles phase-marked commentary, so do not undo those paths and do not add Discord to a plain-text HTML sanitizer as a substitute. Add a conservative shared outbound sanitizer at or before `normalizePayloadsForChannelDelivery` that suppresses payloads made only of internal control/tool-routing traces and strips trace-only lines from mixed text. Cover at least exact `NO_REPLY`, `ANNOUNCE_SKIP`, `REPLY_SKIP`, `to=functions.*`, standalone `commentary`, `recipient_name`, `parameters`, and bare tool-call JSON/envelope shapes when they are clearly internal; avoid suppressing ordinary prose, code examples, or legitimate JSON snippets. Add a Discord send-level exact silent-token backstop only for safe cases with no media/components/embeds/presentation, if the shared boundary does not fully cover direct send calls. Keep media-only silent replies deliverable with an empty caption. Add focused tests for shared payload normalization/delivery, Discord send or outbound adapter behavior, and the session/subagent sanitizer path where appropriate. Add a user-facing changelog fix entry crediting the reporters/commenters, following the repo changelog attribution rules. Do not refactor unrelated channel delivery behavior.
