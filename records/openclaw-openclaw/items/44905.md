## Summary

Keep open. Current main suppresses some exact silent-token and commentary-phase paths, but the shared outbound and Discord send paths still do not cover the broader reported leak family: tool-routing text, envelope field names, raw tool JSON, and REPLY_SKIP/ANNOUNCE_SKIP-style control tokens.

## What This Changes

The issue requests that Discord and outbound channel delivery suppress internal silent/control tokens, commentary/tool-routing traces, envelope fields, and raw tool-call JSON before sending user-visible messages.

## Best Possible Solution

Add a conservative shared outbound/internal-trace sanitizer at the delivery boundary, with a Discord exact-control-token backstop if needed, so internal control tokens and tool-envelope text are consumed before channel sends while preserving legitimate messages, media-only replies, code fences, and channel formatting.
