## Summary

Keep open. Current main covers the commentary-phase subset and exact `NO_REPLY` media-caption normalization, but it still lacks a shared outbound/Discord boundary that suppresses raw `to=functions.*` routing text, envelope field names, generic raw tool-call JSON, and `REPLY_SKIP`/`ANNOUNCE_SKIP`-style control tokens before Discord delivery.

## What This Changes

The issue asks OpenClaw to prevent Discord and outbound delivery from exposing internal LLM tool-call traces, raw tool-envelope text, and control tokens to user-visible channels.

## Best Possible Solution

Add a conservative delivery-boundary sanitizer for internal trace/control artifacts, backed by Discord send-level suppression for bypass cases, while preserving legitimate prose, code fences, Discord-native formatting, media-only sends, and existing silent-reply policy behavior.
