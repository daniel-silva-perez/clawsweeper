## Summary

Keep this issue open. Current main still only supports a single run-level `heartbeat.model`; structured `HEARTBEAT.md` tasks have no model/escalation field and due tasks are folded into one heartbeat prompt before one model call. The related items cover the already-fixed run-level heartbeat model override, not per-task selection.

## Best Possible Solution

Keep open for maintainer product/API design. A complete implementation should add a documented optional `model` or escalation field to structured `HEARTBEAT.md` tasks, define mixed-model execution semantics, update heartbeat docs/config help, and add regression tests for cheap/default tasks, primary/escalated tasks, skipped tasks, timestamp updates, failure handling, and delivery behavior.
