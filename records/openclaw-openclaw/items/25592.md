## Summary

Keep open. Current main has important partial fixes for commentary-phase leakage and group/channel message-tool-only defaults, but direct chats still default to automatic delivery and unphased assistant text is still treated as sendable, which matches the newest Telegram/Slack reproduction reports. The context also lists open implementation PRs with closing syntax, so the issue should remain open until those candidates are reviewed, merged, or explicitly closed.

## What This Changes

The issue asks OpenClaw to prevent intermediate agent text between tool calls from being delivered to messaging channels, so only final or explicitly sent responses reach users.

## Best Possible Solution

Keep this issue as the canonical channel-leak product bug. Maintainers should choose a direct-chat-safe policy, such as explicit sourceReplyDeliveryMode/message-tool-only for DMs, configurable final-tag enforcement, or a final-only delivery guard, then add regression coverage for unphased assistant text before and between tool calls in streaming and final payload assembly.

## Reproduction Assessment

Yes. A high-confidence source/test reproduction exists: direct chats resolve to automatic delivery, and current tests still prove unphased assistant text is emitted as a visible block reply; recent issue comments add shipped-stable Telegram direct-chat repros.

## Solution Assessment

No, current main is not a complete solution. The commentary-phase and group/channel fixes are necessary partial safeguards, but the broader bug needs a maintainer-approved delivery policy for direct chats and unphased assistant text.
