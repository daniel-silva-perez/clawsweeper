## Summary

Keep this issue open. Current main has meaningful partial fixes, including commentary-phase suppression and default message-tool-only behavior for group/channel rooms, but direct chats still use automatic delivery and the embedded subscriber still buffers unphased assistant text before tool execution. The provided GitHub context also shows open PRs with closing syntax against this issue, plus a recent maintainer review and a same-day stable-release reproduction saying the broader bug remains current.

## Best Possible Solution

Keep this issue open as the canonical product bug. The maintainer path should decide and implement a direct-channel-safe policy, such as opt-in sourceReplyDeliveryMode/message-tool-only for direct chats, configurable enforceFinalTag/final-only delivery, or another explicit-send mode, then add regression coverage for unphased assistant text before and between tool calls in both block streaming and final payload assembly.
