## Summary

Keep open. Current main has important partial fixes, but the broader channel-leak problem is still live: direct chats still default to automatic delivery, unphased assistant text can still enter the block reply buffer before a tool call, and the current tests still assert that buffered pre-tool text is delivered. The provided GitHub context also has unmerged PRs using closing syntax for this issue, so those should be reviewed or closed before the issue is resolved.

## Best Possible Solution

Keep this issue open as the canonical product bug. Maintainers should review the linked open implementation PRs and choose a direct-chat-safe policy, such as explicit sourceReplyDeliveryMode/message-tool-only for DMs, configurable final-tag enforcement, or another final-only delivery guard, then add regression coverage for unphased assistant text before and between tool calls in both streaming and final reply assembly.
