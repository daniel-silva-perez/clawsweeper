## Summary

Keep open. Current main still has the reported behavior in the async exec approval completion paths: gateway approval completion takes only the last 400 characters and passes it through the whitespace-collapsing notify formatter before sending the follow-up back into the agent session. The linked fix PR #41170 matches this bug and would have addressed it, but it is closed and unmerged, so it is not a reason to close this issue.

## Best Possible Solution

Reopen/rebase #41170 or reimplement its narrow fix on current main: keep ordinary `notifyOnExit` compact summaries unchanged, but make async approval completion for gateway and node use the full captured exec output up to the existing capture cap, preserve newlines, and explicitly mark capture-cap truncation. Add regression coverage with multiline output over 400 characters through the approval follow-up path, plus a cap-hit case.

## Risks / Open Questions

- Leaving this open is the conservative path because the current implementation still matches the reporter's reproduction: approved async exec output can be silently truncated and flattened before the agent sees it.
- The likely fix increases the amount of approved exec output injected into the follow-up turn, so it should preserve the existing exec capture cap and add an explicit truncation marker when that cap is reached.
