## Summary

Keep open. Current main still sends async exec approval completion output through the compact notification formatter: gateway and node paths keep only the 400-character tail and collapse whitespace before the follow-up turn. The linked fix PR #41170 matches this root cause, but it is closed and unmerged, so it does not make the issue closeable.

## Best Possible Solution

Reopen/rebase #41170 or reimplement the same narrow fix on current main: keep ordinary background notifyOnExit compact summaries unchanged, but make async approval completion for gateway and node deliver the captured exec output with newlines preserved up to the existing capture cap, and explicitly mark capture-cap truncation. Add regression coverage for multiline output over 400 characters and a cap-hit case through the approval follow-up path.
