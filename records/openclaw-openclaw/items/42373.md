## Summary

Keep this issue open. Current main still has no `costCurrency` or equivalent pricing-currency contract; cost configuration remains numeric-only and USD-specific through schema/types, runtime formatting, persisted/session API fields, CLI/chat output, Control UI, macOS, telemetry, and docs. The earlier ClawSweeper review comment is still accurate against current main SHA `891c7d9f1c1f5e0f52935fadfeacf695b0ef1eb8`.

## Best Possible Solution

Keep this open for a core pricing-currency design. Maintainers should decide whether OpenClaw stores explicit USD-only estimates, per-provider/per-model currency metadata, global display currency, conversion, or split mixed-currency summaries, then update config schema/types/generated help, pricing lookup and aggregation, transcript/session/API fields, CLI/chat/status output, Control UI/macOS displays, telemetry/docs, and tests.
