## Summary

Current main still does not implement `costCurrency` or equivalent currency metadata. The pricing contract remains USD-shaped from config schema/types through runtime cost lookup/formatting, session/API fields, CLI/status output, Control UI/macOS formatting, diagnostics metrics, and docs. The prior ClawSweeper comment is still accurate against current main `e476523082a6fff3a0c3048077d2f0e52413fbf2`.

## Best Possible Solution

Keep this open for a core pricing-currency design. Maintainers should decide whether OpenClaw remains USD-only, stores per-provider/per-model currency metadata, supports a global display currency, supports conversion, or handles mixed-currency summaries; then update config schema/types/generated help, pricing lookup and aggregation, transcript/session/API fields, CLI/chat/status output, Control UI/macOS displays, telemetry/docs, and focused tests.
