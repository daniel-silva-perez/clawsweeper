## Summary

Current main still does not implement `costCurrency` or equivalent cost-currency metadata. The pricing schema and docs remain explicitly USD-based, the runtime cost index copies only numeric cost fields, and user-facing CLI/status/UI/macOS formatting still hardcodes dollar output.

## Best Possible Solution

Keep this open for a core pricing-currency design. Maintainers should decide whether OpenClaw remains USD-only, stores per-provider/per-model currency metadata, supports a global display currency, supports conversion, or handles mixed-currency summaries; then update config schema/types/generated help, pricing lookup and aggregation, transcript/session/API fields, CLI/chat/status output, Control UI/macOS displays, metrics/docs, and focused tests.
