## Summary

Keep this PR open. Current main still contains the two important Nostr DM delivery failures this PR tries to fix: Nostr startup settles immediately and the DM subscription passes an array-wrapped filter to `nostr-tools`. The PR is useful, but stale against current main and should not merge as-is because its lifecycle patch targets the older `channel.ts` shape and its `sendMedia` fallback conflicts with the current no-media Nostr contract.

## Best Possible Solution

Keep this PR open as a useful but stale implementation candidate. The maintainer path is a refreshed Nostr-owned patch on current main: make `startNostrGatewayAccount()` remain pending until `ctx.abortSignal` with idempotent cleanup, pass the single `Filter` object required by pinned `nostr-tools` 2.23.3 in the DM subscription path, add focused lifecycle and subscription-shape tests, and leave media unsupported unless a separate public-URL-only media design updates the channel contract, metadata, and docs. Open #53858 should remain the canonical user-facing restart-loop report until that corrected fix lands.
