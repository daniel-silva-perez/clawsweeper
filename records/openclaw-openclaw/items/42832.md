## Summary

Keep this PR open. Current main still has the two Nostr DM delivery failures the PR is trying to fix: the gateway account task resolves immediately after startup, and the DM subscription still passes an array-wrapped filter to `nostr-tools` even though pinned `nostr-tools` 2.23.3 expects a single `Filter`. The PR is stale against the current file layout and its `sendMedia` fallback conflicts with the current no-media Nostr contract, so the best maintainer path is a refreshed Nostr-owned fix rather than closing this as obsolete.

## Best Possible Solution

Refresh the PR or land an equivalent Nostr-owned patch on current main: keep `startNostrGatewayAccount()` pending until abort using `runPassiveAccountLifecycle` or equivalent idempotent cleanup, pass the single `Filter` object required by pinned `nostr-tools` 2.23.3 in the DM subscription path, and add focused lifecycle plus subscription-shape regression tests. Leave media unsupported unless a separate public-URL-only media design updates the Nostr channel metadata, docs, and tests. Keep #53858 as the canonical user-facing restart-loop report until that corrected fix lands.

## Risks / Open Questions

- Closing this PR now would discard a useful implementation candidate while current main still has the startup-lifetime and `subscribeMany()` filter-shape bugs.
