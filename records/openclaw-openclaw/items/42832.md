## Summary

Keep this PR open. Current main at e476523082a6fff3a0c3048077d2f0e52413fbf2 still has the two core Nostr DM delivery failures this PR targets: the Nostr gateway account task resolves immediately and the DM subscription still passes an array-wrapped filter to nostr-tools. The PR should be refreshed rather than closed, because its lifecycle patch targets the older channel.ts shape and its sendMedia addition now conflicts with current no-media Nostr behavior and the generic text fallback already on main.

## Best Possible Solution

Refresh this PR or land an equivalent Nostr-owned fix on current main: make startNostrGatewayAccount stay pending until abort with idempotent cleanup, pass the single Filter object required by nostr-tools 2.23.3 in the DM receive path, and add lifecycle plus subscription-shape regression tests. Drop the sendMedia fallback from this patch unless a separate media design updates Nostr metadata, docs, and tests. Keep #53858 as the canonical user-facing restart-loop report until the refreshed fix lands.

## Risks / Open Questions

- Merging this PR as-is would likely need a rebase/port because current startup ownership is in extensions/nostr/src/gateway.ts, while the PR lifecycle patch was written against the older extensions/nostr/src/channel.ts layout.
- The proposed sendMedia fallback can leak local or otherwise non-public mediaUrl strings into encrypted Nostr DM text and conflicts with the channel's declared media: false contract.
- The PR's subscription-shape test is mock-based; a refreshed patch should include a focused regression test that guards the pinned nostr-tools single-filter call shape.
