## Summary

Keep this PR open. Current main still lacks the two core Nostr DM delivery fixes that remain relevant: the gateway account task resolves immediately instead of staying alive until abort, and the DM subscription still passes an array-wrapped filter to nostr-tools. The PR needs a refresh because its media fallback is now obsolete and risky under the current text-only outbound fallback and Nostr no-media contract.

## Best Possible Solution

Refresh this PR or land an equivalent Nostr-owned fix on current main: make startNostrGatewayAccount stay pending until ctx.abortSignal with idempotent cleanup, pass a single Filter object to nostr-tools subscribeMany in the DM receive path, and add lifecycle plus subscription-shape regression tests. Drop the sendMedia fallback unless a separate media design updates Nostr capabilities, docs, and URL-safety behavior. Keep #53858 as the canonical user-facing restart-loop report until the refreshed fix lands.

## Risks / Open Questions

- Merging the PR as-is would likely be wrong on current main because the lifecycle fix targets the older channel.ts layout instead of the current gateway.ts startup owner.
- The PR's sendMedia fallback can expose local or otherwise non-public mediaUrl strings by placing them directly into Nostr DM text, and it conflicts with the plugin's declared media: false contract.
- The PR's subscription-shape coverage is mock-based. A refreshed patch should add a focused regression that locks the pinned nostr-tools 2.23.3 single-filter call shape.
