## Summary

Keep open. Current main still preserves the reported short explicit isolated cron agentTurn timeout behavior, and open PR #74893 still references this issue with closing syntax, so the issue should stay open until that implementation candidate is reviewed and merged or closed.

## What This Changes

The issue requests a warning, safer default timeout, or prominent docs for short isolated agentTurn cron jobs that omit lightContext and toolsAllow and can spend their timeout budget on cold-start bootstrap.

## Best Possible Solution

Review PR #74893 and land a narrow guardrail that explains the cold-start budget issue for affected jobs while preserving explicit timeout semantics; broader hard wall-time timeout behavior can stay with related issue #74681.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is an isolated agentTurn cron job with a short explicit timeout, no lightContext, and no toolsAllow; current main starts the cron timeout after the runner execution-start callback while later bootstrap work can still run inside that budget.

## Solution Assessment

Unclear until PR #74893 is reviewed. A targeted warning or docs fix looks safer for this issue than silently changing explicit timeout behavior, while related issue #74681 covers the broader hard wall-time enforcement question.
