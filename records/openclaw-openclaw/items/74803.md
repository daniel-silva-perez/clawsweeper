## Summary

Keep open. Current main still preserves the short explicit isolated cron `agentTurn` timeout behavior, and open PR #74893 already references this issue with closing syntax, so the issue should stay open until that implementation candidate is reviewed and merged or closed.

## What This Changes

The issue requests a warning, safer default, or prominent docs for isolated cron `agentTurn` jobs that omit `lightContext` and `toolsAllow` because cold-start bootstrap can consume short timeout budgets before useful work begins.

## Best Possible Solution

Review PR #74893 and land a narrow guardrail that warns on create/update for short explicit isolated cron `agentTurn` jobs without lightweight context or a tool allowlist, with public cron docs explaining the cold-start budget tradeoff while preserving explicit timeout semantics.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is an isolated `agentTurn` cron job with a short explicit timeout, no `lightContext`, and no `toolsAllow`; current main starts the timeout after the runner execution-start callback while later bootstrap work can still run inside that budget.

## Solution Assessment

Unclear until PR #74893 is reviewed. A targeted warning/docs fix looks safer than silently changing explicit timeout behavior, but maintainers still need to choose the threshold and user-facing surface.
