## Summary

Keep open. Current main still lacks a create/update warning or prominent docs caveat for short isolated cron `agentTurn` runs without `lightContext` or `toolsAllow`, and open PR #74893 references this issue with closing syntax, so the issue should stay open until that PR lands or is explicitly superseded.

## What This Changes

The issue requests a guardrail or documentation update for isolated cron `agentTurn` jobs with short explicit timeouts that omit lightweight context and a tool allowlist.

## Best Possible Solution

Review and land PR #74893 or an equivalent narrow fix that warns for risky isolated cron `agentTurn` jobs and updates the public cron docs, while preserving explicit timeout semantics.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to create an isolated `agentTurn` cron job with `timeoutSeconds: 60`, no `lightContext`, and no `toolsAllow`; current code arms the timeout before embedded workspace/runtime bootstrap, though this read-only review did not run a live 60-second job.

## Solution Assessment

Unclear until PR #74893 is reviewed, but the direction is sound: a targeted warning/docs guardrail is safer than silently inflating explicit user timeouts or changing broad scheduler semantics.
