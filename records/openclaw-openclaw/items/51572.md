## Summary

Keep this issue open. Current main still does not fire the bundled `session-memory` internal hook for automatic daily/idle rollover, and session maintenance pruning still deletes stale entries without any pre-prune memory hook or lifecycle dispatch. Open PRs #61675 and #71792 cover the reset half of the request, but they do not cover the explicit `pruneAfter`/maintenance-prune behavior described here.

## Best Possible Solution

Keep this open or explicitly split it: let maintainers review/land a narrow daily/idle reset fix via #61675, #71792, or a replacement, then add a prune-specific hook path for session maintenance before stale entries are deleted. The prune fix likely belongs near `src/config/sessions/store.ts` / `store-maintenance.ts`, should pass the removed `SessionEntry` and stable transcript path to either a documented lifecycle hook or a session-memory-compatible internal event, and should include regression coverage for `pruneAfter` maintenance deletion.
