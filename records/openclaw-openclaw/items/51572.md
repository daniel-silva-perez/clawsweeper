## Summary

Keep this issue open. Current main at 955b4df0939f90b4b6de5e7e6f3149dd125e1e50 still does not route automatic daily/idle rollover through the bundled session-memory internal hook or before_reset plugin hook, and session maintenance pruning still removes stale entries before any memory/lifecycle hook dispatch. Open PRs #61675 and #71792 cover daily/idle reset persistence, but neither covers the explicit pruneAfter maintenance-prune path requested here.

## Best Possible Solution

Keep this issue open or explicitly split it. Maintainers should review and either land #61675, #71792, or a replacement for the daily/idle reset half, then add a prune-specific path near src/config/sessions/store.ts / src/config/sessions/store-maintenance.ts that dispatches a documented lifecycle or session-memory-compatible hook before stale entries are deleted. That prune path should pass the removed SessionEntry plus a stable transcript path and include regression coverage for pruneAfter maintenance deletion.
