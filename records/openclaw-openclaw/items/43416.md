## Summary

Keep open. Current main still lacks the native shared OpenClawChatUI assistant-message copy affordance, so this PR is not obsolete, but the latest patch should not merge as-is because it copies raw message payload while the renderer may hide assistant thinking segments. Earlier review discussion appears to have narrowed the timer, hit-testing, iOS pasteboard, and tool-result scope concerns, leaving the raw visible-text mismatch as the concrete blocker.

## What This Changes

The PR adds a copy button to shared native assistant message bubbles in `ChatMessageViews.swift`, using hover state on macOS, platform pasteboard writes, and a short copied-state icon transition.

## Best Possible Solution

Keep the native copy affordance, but derive the clipboard payload from the same visible `AssistantTextParser` segment path used for rendering, respecting `showsAssistantTrace`, then add focused Swift coverage for trace-off and trace-on copy payloads in the shared OpenClawChatUI area.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Copy only the visible assistant text:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatMessageViews.swift:231`
  - body: `CopyMessageButton` is fed raw `primaryText`, but assistant rendering goes through `AssistantTextParser.segments(..., includeThinking: showsAssistantTrace)`. With traces hidden, a message like `<think>secret</think><final>answer</final>` displays only the final answer while this button copies the hidden trace and tags to the system clipboard, so the payload should come from the same visible segment path as the UI.
  - confidence: 0.94

## Security Review

Status: needs_attention

Summary: The patch introduces a privacy-sensitive clipboard mismatch by copying hidden assistant trace payload that may not be visible in the UI.

Concerns:

- **[medium] Hidden assistant trace can be copied:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatMessageViews.swift:231`
  - body: The new copy button writes raw `primaryText` to the system clipboard. When `showsAssistantTrace` is false, the UI can hide thinking segments, but the clipboard can still receive those hidden segments and tags.
  - confidence: 0.93
