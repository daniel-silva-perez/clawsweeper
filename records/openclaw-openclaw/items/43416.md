## Summary

Keep this PR open. Current main still lacks the native OpenClawChatUI assistant-message copy affordance, so the feature is not obsolete, but the proposed patch has a blocking privacy bug: it copies raw assistant payload text while the visible bubble renders through the trace-aware assistant text parser, so hidden thinking content and tags can be placed on the system clipboard.

## What This Changes

The PR adds a SwiftUI copy button to native assistant messages in OpenClawChatUI, using platform pasteboards, macOS hover visibility, iOS availability, and temporary copied-state feedback.

## Best Possible Solution

Keep the native copy affordance, but compute the clipboard payload from the same assistant segment path used for rendering, respecting showsAssistantTrace, and cover the helper with focused OpenClawChatUI tests for hidden-trace and visible-trace cases.

## Reproduction Assessment

Yes. A code-level reproduction is high confidence: with assistant text like `<think>secret</think><final>answer</final>` and `showsAssistantTrace == false`, current rendering displays only the response segment while the PR passes raw primary text to the copy button.

## Solution Assessment

No. The feature direction is reasonable, but the proposed implementation should copy rendered/visible assistant content rather than raw payload text; a shared visible-copy-text helper or equivalent segment join is the safer maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Copy only rendered assistant text:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatMessageViews.swift:231`
  - body: `CopyMessageButton` receives raw `primaryText`, but assistant rendering uses `AssistantTextParser.segments(..., includeThinking: showsAssistantTrace)`. With traces hidden, a message like `<think>secret</think><final>answer</final>` displays only the answer while this button copies the hidden trace and tags to the clipboard, so feed the button the same visible text the UI renders.
  - confidence: 0.94

## Security Review

Status: needs_attention

  - body: The new copy button writes raw assistant message text to the system clipboard. When `showsAssistantTrace` is false, the UI can hide thinking segments, but the clipboard can still receive those hidden segments and tags.
