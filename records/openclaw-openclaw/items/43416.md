## Summary

Keep this PR open. Current main still does not implement the requested native Swift assistant-message copy button, but the PR should not be closed as implemented or merged as-is because its copy payload is still the raw primary text rather than the visible assistant text after trace/thinking filtering.

## Best Possible Solution

Keep the PR open and request revisions. The native copy action should derive its clipboard payload from the same AssistantTextParser.segments(from:includeThinking:) path used by rendering, joined as the currently visible assistant text for the active trace setting. Keep the assistant-role guard and platform pasteboard handling, add a focused parser/helper test for copied visible text, and verify the shared native chat UI on macOS and iOS before landing.
