## Summary

Keep open. Current main still does not implement a native assistant-message copy button in the shared Swift chat view, and the PR's remaining diff copies raw `primaryText` rather than the visible `AssistantTextParser` output, so hidden trace/thinking text can be placed on the clipboard when traces are off. Earlier review concerns around timer cancellation, hit testing, iOS pasteboard support, and tool-result scoping appear addressed in the latest diff, but the raw clipboard payload is still a blocking privacy issue.

## What This Changes

The PR adds hover state and a macOS/iOS pasteboard-backed copy button with copied-state feedback to shared native assistant message bubbles in `ChatMessageViews.swift`.

## Best Possible Solution

Revise the PR so the clipboard payload is derived from the same visible `AssistantTextParser` segment path used for rendering, respecting `showsAssistantTrace`, then add focused Swift coverage for trace-on and trace-off copy payloads while keeping the change limited to the shared native chat UI.
