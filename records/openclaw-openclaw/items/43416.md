## Summary

Keep this PR open. Current main still does not implement the requested native Swift assistant-message copy button; the existing WebChat copy-as-markdown action is a separate TypeScript surface. The PR is not ready to close as implemented or merge as-is because its clipboard payload is still raw primary text, while the native renderer intentionally filters thinking/final tags before display.

## Best Possible Solution

Keep the PR open and request a revision. The native copy action should keep the assistant-role guard and platform pasteboard handling, but derive its clipboard string from the same AssistantTextParser.segments(from:includeThinking:) path used by rendering, joined as the currently visible assistant text for the active trace setting. Add a focused parser/helper test for copied visible text and verify the shared native chat UI on macOS and iOS before landing.
