## Summary

Keep open. Current main still has the unsafe Feishu empty-content boundary: the sequential lane and debounce path assume parsed content is a string, while normal ingress rejects missing content as malformed instead of safely dispatching mention-only group messages. The related OpenClaw PRs were closed unmerged, and the only open implementation reference is in an external fork.

## What This Changes

Feishu group messages that contain only an @mention or otherwise have missing content should not throw or fail to dispatch when the Feishu plugin derives sequential and debounce text.

## Best Possible Solution

Normalize Feishu inbound message text to a string at the parser or receive-handler boundary, preserve existing group mention gating and sequential lane semantics, and add regression coverage for mention-only and missing-content events.

## Reproduction Assessment

Yes for a focused source-level reproduction: construct a Feishu text event whose runtime `message.content` is absent or undefined and call the current sequential/debounce path, which still reaches unsafe string methods. Live Feishu payload proof is not present, and the normal current ingress parser may reject missing content before dispatch.

## Solution Assessment

Yes, a narrow Feishu plugin fix is the best path. Boundary normalization plus regression tests is safer than only optional chaining at one call site, and no core or cross-channel API change appears necessary.
