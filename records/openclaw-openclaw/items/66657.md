## Summary

Keep open. Current main still does not safely handle the Feishu missing-content / mention-only path: one lane still trims parsed content directly, the debounce path trims the resolver result directly, and the receive parser currently rejects missing or empty content as malformed instead of proving the reported group mention dispatch works.

## What This Changes

The issue asks OpenClaw to handle Feishu group messages that contain only an @mention or otherwise have empty/undefined content without throwing a `.trim()` TypeError or dropping the dispatch.

## Best Possible Solution

Normalize Feishu inbound message text to a string at the parser or receive-handler boundary, preserve existing group mention gating, guard sequential/debounce string operations, and add regression coverage for missing-content and mention-only group events.
