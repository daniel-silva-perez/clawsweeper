## Summary

Keep open. Current main still has an unsafe Feishu parsed-content trim path, and the current payload guard would reject a missing-content message as malformed rather than dispatching the @mention-only group message. The related target-repo PR attempts are closed/unmerged, so there is not a shipped or merged fix to close against.

## What This Changes

The issue asks OpenClaw to handle Feishu group messages that contain only an @mention or otherwise have empty/undefined content without throwing `Cannot read properties of undefined (reading 'trim')`.

## Best Possible Solution

Land a Feishu-plugin fix that makes parsed inbound message content a safe string before debounce/sequential-key/control-command handling, preserves existing group mention policy, and adds regression coverage for missing-content and empty-text @mention-only messages.
