## Summary

Keep this PR open. Current main still has the explicit-empty-tool prompt/report gap: empty effective tool lists fall into the standard tool-catalog prompt path, skills are still injected and reported from raw skills input, and provider prompt hooks still do not receive final tool availability context. The open #70790 work is payload-level overlap, not a replacement for this prompt/report fix.

## Best Possible Solution

Keep the PR open and review or split it into focused changes: explicit empty-tool semantics in `src/agents/system-prompt.ts`, rendered-prompt-based skills reporting in `src/agents/system-prompt-report.ts`, final effective tool/client-tool propagation through CLI and embedded runners, provider overlay alignment with final tool availability, and regression tests for no-tools, omitted-tool legacy behavior, hosted/client-only tools, and prompt reports. Coordinate with #70790 for provider payload stripping, but treat that as adjacent rather than a full replacement.
