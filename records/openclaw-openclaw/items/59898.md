## Summary

Keep this PR open. Current main still does not implement the explicit-empty-tool prompt/report behavior: empty tool names fall back to the standard tool catalog and tool-only guidance, skills remain included from raw skillsPrompt, and prompt reports still count raw skills input rather than the rendered prompt. The related #70790 payload-level work does not supersede this prompt-layer/report fix.

## Best Possible Solution

Keep this PR open and review or split it into focused changes: explicit empty-tool prompt semantics in `src/agents/system-prompt.ts`, rendered-prompt-based skills reporting in `src/agents/system-prompt-report.ts`, final effective tool/client-tool propagation through CLI and embedded runners, provider overlay alignment, and regression tests. Coordinate with #70790 for payload stripping, but do not close this PR unless that work also covers the prompt/report behavior.
