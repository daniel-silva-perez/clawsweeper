## Summary

Keep this PR open. Current main still has the prompt/report bug, and the branch addresses the right surface, but the latest PR patch is not ready to merge because explicit no-tool sessions can still receive skills catalog text, prompt reports still derive skills from raw input, and provider overlays can still inject tool-oriented guidance.

## What This Changes

The PR updates agent system prompt construction, prompt reporting, embedded-runner tool context, and OpenResponses client-tool validation/tests to distinguish explicit empty OpenClaw tool lists from omitted tool lists and hosted-tool-only sessions.

## Best Possible Solution

Keep the prompt-layer fix alive, but require the branch to make explicit no-tool prompts truly omit skills/tool-only guidance, derive skills reporting from the rendered prompt, propagate final effective tool and hosted-tool context into provider hooks, and coordinate separately with #70790 for payload-level empty-tools stripping.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Drop skills catalog from no-tool prompts:** `src/agents/system-prompt.ts:185-191`
  - body: The explicit no-tool path still renders the skills catalog through `catalog-only` mode. A deny-all session with a non-empty skills snapshot will still receive `<available_skills>` content even though the branch is meant to prevent skills leakage in tool-free sessions. Suppress `skillsPrompt` for the explicit-empty case, or make this mode return no skills block, and add a regression with a non-empty raw skills prompt.
  - confidence: 0.87
- **[P2] Report rendered skills instead of raw input:** `src/agents/system-prompt-report.ts:149-174`
  - body: `buildSystemPromptReport` still parses `params.skillsPrompt` and uses its raw length. Once explicit no-tool prompts suppress the skills block, `/context` will continue reporting skill entries and bytes that were not actually sent to the model. Derive the skills stats from the rendered `systemPrompt` or structured render metadata.
  - confidence: 0.82
- **[P2] Gate provider overlays for no-tool prompts:** `src/agents/system-prompt.ts:959-966`
  - body: Only `execution_bias` overrides are disabled when no first-class tools are available. Provider `interaction_style` and `stablePrefix` still render, and the OpenAI GPT-5 overlays include tool-discipline/heartbeat wording that tells the model to use tools or capabilities. Explicit no-tool sessions can still get contradictory tool guidance unless provider contributions receive effective tool context or tool-oriented provider blocks are suppressed in no-tool mode.
  - confidence: 0.78
