## Summary

Keep open. Current main appears to have improved the Discord health/status projection, but the reported gateway SecretRef false positive still exists: doctor still warns that a SecretRef-managed gateway token is unavailable without using the async SecretRef resolver or the secrets-audit resolvability result.

## Best Possible Solution

Keep this issue open. Finish the gateway SecretRef part by making doctor gateway auth health use resolveGatewayAuthToken or the same secrets-audit/snapshot resolvability result before warning, and add a regression for a resolvable gateway.auth.token SecretRef.
