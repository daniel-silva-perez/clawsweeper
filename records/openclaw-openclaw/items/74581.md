## Summary

Keep open. Current main still has the reported Telegram native-quote fallback gap: the retry path exists, but its matcher only recognizes `quote not found`, while the reported `QUOTE_TEXT_INVALID` string has no current source or regression-test coverage.

## What This Changes

The issue requests treating Telegram `QUOTE_TEXT_INVALID` quote failures like existing native-quote fallback errors so replies retry without `reply_parameters.quote`.

## Best Possible Solution

Keep the native quote feature, but make the Telegram quote-error classifier recognize both Bot API symbolic codes and human-readable quote-text-invalid variants, then pin that behavior with a focused Telegram delivery regression test and a small user-facing changelog fix entry.
