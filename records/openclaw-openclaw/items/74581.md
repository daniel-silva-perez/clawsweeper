## Summary

Keep open. Current main still has the reported Telegram native-quote fallback gap: the retry path exists, but its quote-error matcher only recognizes `quote not found`, while `QUOTE_TEXT_INVALID` and normalized `quote text invalid` have no current source or regression-test coverage. The latest shipped release has the same incomplete matcher, and prior related PR context confirms this is a known Telegram edge rather than a new product request.

## What This Changes

Make Telegram delivery treat `QUOTE_TEXT_INVALID` native-quote send failures like existing quote-parameter failures by retrying without the native quote.

## Best Possible Solution

Keep native Telegram quote excerpts, but make the quote-error classifier recognize Telegram symbolic codes and human-readable quote-text-invalid variants, then pin the fallback with a focused delivery regression test and a small user-facing changelog fix entry.
