## Summary

Keep this PR open. Current main still aggregates the latest usable descendant fallback reply from each distinct child and joins multiple child outputs, while this PR changes that user-visible behavior to return only the freshest overall child reply. The current tests still assert multi-child aggregation, and unresolved review comments identify real fan-out and interim-filtering risks in the PR branch.

## Best Possible Solution

Keep this PR open for maintainer review or revision. The best path is a focused cron fallback change that suppresses stale same-child historical replies while preserving one usable latest reply per distinct child for fan-out runs, with a narrower acknowledgement filter and tests covering same-child stale history, multi-child aggregation, silent replies, and true interim acknowledgements. If maintainers explicitly want freshest-only delivery, update the docs and aggregation tests to make that product decision clear.
