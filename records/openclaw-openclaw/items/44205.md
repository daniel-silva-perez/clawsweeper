## Summary

Keep this PR open. Current main still aggregates up to four latest descendant fallback replies, and the current regression test still asserts joined multi-child output, so main has not implemented the PR's freshest-only behavior. The PR also still needs maintainer judgment because review comments identify real fan-out and interim-filtering risks if it lands as-is.

## Best Possible Solution

Keep this PR open for maintainer review or revision. The best path is a focused cron fallback change that suppresses stale same-child historical replies while preserving one usable latest reply per distinct child for fan-out runs, with a narrower acknowledgement filter and tests covering same-child stale history, multi-child aggregation, silent replies, and true interim acknowledgements.
