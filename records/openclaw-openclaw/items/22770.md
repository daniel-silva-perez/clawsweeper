## Summary

Current main still does not implement the requested $VISUAL/$EDITOR compose flow in the TUI. The existing maintainer/Codex comment also explicitly kept this open, and the related vi/vim keybinding issue #55836 is adjacent rather than a substitute.

## Best Possible Solution

Keep this issue open. After maintainer signoff on the exact UX and security boundary, implement a core TUI composer flow that adds a deliberate shortcut or slash command, suspends the TUI, writes the current composer text to a temporary markdown prompt file, launches $VISUAL then $EDITOR without auto-submitting, reloads edited content with clear status/error handling, and documents the shortcut in the public TUI guide. Keep #55836 separate for in-place vi/vim keybindings.
