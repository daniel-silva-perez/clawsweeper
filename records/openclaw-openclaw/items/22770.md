## Summary

Current main still does not implement the requested external-editor compose flow for the TUI. The editor wrapper has no Ctrl+E or external-editor hook, runTui only uses TUI suspension for local auth, slash commands provide no edit/editor/compose substitute, and the public TUI docs do not document $VISUAL/$EDITOR handoff. Related #55836 is adjacent vi/vim in-place editing, not a shipped replacement for this issue.

## Best Possible Solution

Keep this issue open and implement the feature in the core TUI composer after maintainer signoff on the exact UX/security boundary: add a deliberate shortcut or command, suspend the TUI, write current composer text to a temporary markdown prompt file, launch $VISUAL then $EDITOR and wait, load edited content back into the composer without auto-submitting, surface clear status/error messages, document the shortcut in the public TUI guide, and keep #55836 separate for vi/vim in-place editing mode.
