## Summary

Keep this issue open. Current main still has the broad session-history guard described by the reporter, and the fallback retry prompt remains a hard-coded prefix with no supported operator hook or config override. The related PR #69109 was closed unmerged, so it is useful context but not a shipped fix.

## Best Possible Solution

Keep this issue open and split the work if needed: land an equivalent of PR #69109's latest-user-scoped `sessionFileHasContent` regression fix, then design a small supported retry-prompt override or hook in the attempt-execution/model-fallback path with config/docs/tests. The closed PR is good prior art, but current main still needs the code change.
