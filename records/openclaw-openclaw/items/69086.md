## Summary

Keep this issue open. Current main still has the broad assistant-anywhere session-history guard, still hard-codes the fallback retry marker, and the related PR #69109 was closed unmerged. The latest checked-out main at 955b4df0939f90b4b6de5e7e6f3149dd125e1e50 does not implement either requested fix.

## Best Possible Solution

Keep this issue open. Land an equivalent of #69109's latest-user-scoped `sessionFileHasContent` fix with the reporter's regression fixture, then design a small supported retry-continuation prompt override or hook in the attempt-execution/model-fallback path with config schema, docs, and targeted tests.
