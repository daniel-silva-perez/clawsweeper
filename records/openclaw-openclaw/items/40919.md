## Summary

Keep open. Current main still avoids some unnecessary session syncs with dirty-file tracking, hashes, embedding cache, and session delta thresholds, but a selected changed transcript is still rebuilt from the full JSONL file and written through the clear-and-reinsert chunk path. The related implementation PR #59577 remains open rather than merged, so this issue should stay available for that review or a replacement fix.

## What This Changes

Request to replace full delete-and-reinsert session JSONL memory indexing with append-aware incremental session sync for growing transcripts.

## Best Possible Solution

Land or replace #59577 with a current-path implementation in the memory-core plugin and memory host SDK that keeps valid existing session chunks, stores durable append/chunk metadata, falls back to a full reindex on unsafe changes, and adds regression coverage for append growth, unchanged transcripts, overlap boundaries, compaction/truncation, and recovery after failed embedding writes.
