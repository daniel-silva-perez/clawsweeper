## Summary

Keep open. Current main still mitigates some redundant session syncing with dirty-file tracking, hash checks, embedding cache, and delta thresholds, but a changed session transcript is still rebuilt from the full JSONL file and written through the file-level clear-and-reinsert chunk path. The provided context also shows #59577 is still open as the implementation candidate rather than merged.

## What This Changes

The issue requests incremental session-memory indexing for growing JSONL transcripts instead of repeatedly deleting and reinserting all session chunks.

## Best Possible Solution

Land or replace #59577 with a current-path memory-core and memory-host-sdk implementation that preserves valid existing session chunks, stores durable append or chunk metadata, falls back to a full reindex on unsafe changes, and adds regression coverage for append growth, unchanged transcripts, overlap boundaries, compaction/truncation, and recovery after failed writes.
