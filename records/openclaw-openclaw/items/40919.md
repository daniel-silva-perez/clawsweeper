## Summary

Keep open. Current main still narrows which session transcripts are selected for sync, but a selected changed transcript is still rebuilt from the full JSONL file and sent through the same clear-and-rewrite chunk path. The embedding cache and dirty-file thresholds reduce some repeated work, but they do not provide durable append-only or chunk-level incremental session indexing, and the directly related implementation PR remains open rather than merged.

## What This Changes

The issue asks for session memory sync to avoid full delete-and-reinsert indexing for growing session JSONL transcripts by using incremental append or chunk-aware indexing.

## Best Possible Solution

Keep this issue open and land or replace #59577 with a current-path implementation in the memory-core plugin and memory host SDK. The end state should preserve existing session chunks when valid, add durable append/chunk metadata, fall back to full reindex on unsafe changes, and include regression coverage for append growth, unchanged transcripts, overlap boundaries, compaction/truncation, and recovery after failed embedding writes.
