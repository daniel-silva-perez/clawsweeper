## Summary

Keep open. Current main still limits session work to dirty or targeted transcripts, but each selected transcript is still rebuilt from the full JSONL file and written through the same clear-and-rewrite indexing path. The embedding cache and session delta thresholds are useful mitigations, not durable append-only or chunk-level incremental session indexing. Related #59577 remains an open implementation candidate, while #50908 was closed unmerged.

## Best Possible Solution

Keep this issue open and review, rework, or replace #59577 with a current memory-core implementation. The fix should add durable append/chunk metadata, preserve unchanged session chunks instead of clearing all rows, reuse the embedding cache when valid, fall back to a full reindex on truncation/compaction/config/model/parser changes, and add regression coverage for append growth, unchanged transcripts, overlap boundaries, compaction, and failure recovery.
