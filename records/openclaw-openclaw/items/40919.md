## Summary

Keep open. Current main has useful mitigations for session churn, but it still rebuilds a full rendered session entry for changed transcripts and writes chunks through the file-level clear-and-reinsert path. The related incremental-session-sync implementation, #59577, is still open and unmerged, so this issue should stay open for maintainer review rather than be closed as implemented.

## What This Changes

The issue requests replacing full delete-and-reinsert session JSONL memory sync with incremental append/change-aware indexing, with optional archival or size-based sync throttling for long sessions.

## Best Possible Solution

Implement session-specific incremental indexing that preserves valid existing chunks, stores durable append or chunk metadata, falls back to full reindexing when a transcript is compacted or otherwise unsafe, and covers append growth, unchanged transcripts, overlap boundaries, truncation/compaction, and failed-write recovery in tests.
