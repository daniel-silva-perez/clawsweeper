## Summary

Keep open. Current main still writes `openclaw backup create` archives through an adjacent random `.tmp` file and only removes that file when the archive path unwinds through normal JavaScript `finally` cleanup; the backup CLI has no scoped SIGINT/SIGTERM, timeout, or abort cleanup path for interrupted runs. The issue discussion also confirms the reporter is using an external cleanup task as a workaround, not a shipped OpenClaw fix.

## What This Changes

The issue requests that `openclaw backup create` clean up large temporary `.tmp` archive files when a timed-out or interrupted backup exits before normal completion.

## Best Possible Solution

Backup creation should own the active temporary archive lifecycle: track the exact temp archive during the write window, clean it on graceful interruption or abort, preserve the existing no-overwrite final publication behavior, avoid deleting unrelated `*.tmp` files, and document unavoidable hard-kill limits.
