## Summary

Keep open. Current main still handles ordinary thrown backup failures, but it does not own cleanup for an externally timed-out or signal-interrupted `openclaw backup create`; the issue remains valid and narrow enough for a focused core backup fix.

## What This Changes

The issue requests that `openclaw backup create` remove large adjacent `.tmp` archive files when backup creation times out, is interrupted, or fails unexpectedly.

## Best Possible Solution

Backup creation should track the active temporary archive path for the duration of the archive write, remove it on graceful interruption or abort, keep the no-overwrite publication semantics, and document the unavoidable crash/SIGKILL limits.
