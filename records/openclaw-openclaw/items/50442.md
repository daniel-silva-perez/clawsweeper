## Summary

Keep open. Current main still writes `openclaw backup create` archives through an adjacent random `.tmp` file and only removes it during normal JavaScript unwinding; there is no backup-scoped SIGINT/SIGTERM or abort cleanup for externally timed-out runs. The workaround discussion is useful, but it confirms a manual mitigation rather than a fix in the backup tool.

## What This Changes

Fix `openclaw backup create` so interrupted or timed-out backup runs clean up large temporary archive files and only publish a final archive on success.

## Best Possible Solution

Backup creation should own the active temporary archive lifecycle: track the exact temp archive path, clean it up on graceful termination or abort during the archive-write window, preserve no-overwrite final publication, and document any unavoidable SIGKILL/crash limits.

Fix the open `openclaw backup create` timeout/interruption cleanup bug. Implement backup-owned cleanup for the active archive write window: track the exact temp archive path, install scoped graceful termination/abort handling only during backup creation, remove handlers after completion, preserve existing no-overwrite final publication semantics, and keep SIGKILL/crash limitations explicit.
