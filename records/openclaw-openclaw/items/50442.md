## Summary

Keep open. Current main still writes backup archives through an adjacent `.tmp` file and only removes it when the JavaScript stack reaches the `finally` block; the backup create path has no scoped SIGINT/SIGTERM or abort cleanup, so an external timeout/process termination can still strand the large temp archive described in the report.

## What This Changes

`openclaw backup create` should clean up large temporary archive files when interrupted by a timeout or external termination instead of leaving adjacent `.tmp` files behind.

## Best Possible Solution

Add backup-owned graceful interruption cleanup around the active tar write: track the exact temp archive path, install scoped SIGINT/SIGTERM and abort cleanup only during backup creation, remove handlers in `finally`, preserve the current no-overwrite publish semantics, and cover the interrupted-write path with focused regression tests.
