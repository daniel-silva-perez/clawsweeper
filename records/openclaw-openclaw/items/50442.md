## Summary

Keep open. Current main still writes backup archives through an adjacent random `.tmp` file and cleans it up only when the backup promise unwinds normally; I did not find a backup-scoped SIGINT, SIGTERM, timeout, or abort cleanup path for externally interrupted runs. The reporter's workaround confirms the operational impact remains mitigated outside the tool, not fixed in OpenClaw.

## What This Changes

The issue requests that `openclaw backup create` clean up large temporary archive files left behind when a backup times out or is interrupted.

## Best Possible Solution

Backup creation should own the active temporary archive lifecycle: track the exact temp archive path during the write window, clean it up on graceful interruption or abort, preserve existing no-overwrite final publication semantics, avoid deleting unrelated `*.tmp` files, and document unavoidable hard-kill limits.
