## Summary

Keep open. Current main still has the gap the reporter described: plugin `api.runtime.subagent.run()` is a thin Gateway `agent` dispatch and does not register the run with the subagent completion registry, so the completion announce path and `subagent_delivery_target` hook are not reached for that plugin-created child run.

## Best Possible Solution

Keep this issue open and implement an explicit plugin-runtime completion-delivery path. The likely core fix is a small public API addition that lets trusted plugin code bind a plugin-created child run to a requester session/origin and `expectsCompletionMessage`, then reuses `registerSubagentRun` so `runSubagentAnnounceFlow()` and `subagent_delivery_target` execute normally. Update plugin runtime docs and add regression coverage for a plugin hook starting a child run, returning an immediate handled reply, and later delivering the child completion through the announce path.
