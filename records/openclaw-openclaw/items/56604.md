## Summary

Keep open. Current main still does not implement per-dialed-number Voice Call routing. The plugin captures and stores the provider `to` number, but config, manifest metadata, UI hints, inbound greeting, response agent/model/system prompt, and TTS voice selection still use one global Voice Call config. The provided discussion also includes a maintainer comment keeping this open, and related #59245 is broader outbound task-call work rather than a duplicate for inbound per-number persona routing.

## Best Possible Solution

Keep this issue open and implement it in `extensions/voice-call`: add a typed and documented per-number route table to the config schema, manifest metadata, UI hints, and docs; normalize provider `to` values at call creation; store a stable route id or effective config on the call; and make inbound greeting, response agent/model/system prompt, realtime consult settings, and TTS voice selection use that effective config with global fallback. Add focused tests for route matching, fallback behavior, provider number normalization, and stable per-call routing. Keep #59245 separate because it covers broader outbound task-call behavior.
