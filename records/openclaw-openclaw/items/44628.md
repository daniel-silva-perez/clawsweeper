## Summary

Keep open. Current main still does not implement this PR's intended systemd scope selection for the managed Linux gateway lifecycle, and the PR is not merge-ready because it mixes that work with an unrelated Docker mount-validation change and has unresolved review feedback on scope-selection behavior.

## Best Possible Solution

Keep this PR open for maintainer action, but do not merge it as-is. The best path is a focused systemd lifecycle patch in src/daemon/systemd.ts with tests in src/daemon/systemd.test.ts, using #44772 as prior art but not as shipped proof. It should resolve user vs system scope for restart, stop, and runtime/status, report the selected scope where the API supports it, prefer the active/running service over enabled-only state, and leave Docker mount validation out of this PR.
