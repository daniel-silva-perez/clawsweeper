## Summary

Keep this PR open. Current main still has the exact gaps this PR is trying to fix: message-tool-only group delivery returns before voice media can be sent, OpenAI-compatible PCM TTS is returned as raw .pcm bytes, and WhatsApp audio voice-note paths still duplicate visible text. The PR is a focused implementation candidate, with a security-sensitive delivery exception that needs maintainer review rather than cleanup closure.

## Best Possible Solution

Continue reviewing this PR as the implementation candidate. The best path is to validate the trustedLocalMedia-gated voice-media exception, keep normal text and untrusted media suppressed in message_tool_only groups, remove duplicate WhatsApp text sends for voice notes, wrap declared PCM as WAV with observable mismatch handling, keep the targeted regression tests, and run the normal changed-surface gate before merge. Close only after this PR merges or a newer canonical fix supersedes it.
