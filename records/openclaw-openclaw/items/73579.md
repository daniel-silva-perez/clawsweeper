## Summary

Keep this PR open. Current main still has the behaviors the PR is trying to fix: suppressed group/channel source delivery drops block/final payloads, OpenAI-compatible PCM TTS is returned as raw `.pcm`, and WhatsApp voice-note paths still send visible text separately.

## Best Possible Solution

Continue reviewing this PR as the current implementation candidate. The best path is to harden the `trustedLocalMedia`-gated suppressed-delivery exception, keep normal text and untrusted media suppressed in `message_tool_only` rooms, wrap declared PCM into a deliverable container with bounded parsing and useful logging, update WhatsApp/TTS docs if the duplicate-text behavior changes, preserve the targeted regression tests, and close only after this PR merges or a newer canonical fix supersedes it.
