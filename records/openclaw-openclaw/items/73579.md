## Summary

Keep open. Current main still has the behaviors this PR is trying to fix: suppressed `message_tool_only` delivery returns before trusted TTS voice media can be sent, OpenAI-compatible PCM TTS still returns raw `.pcm`, and WhatsApp voice-note paths still duplicate visible text. This PR remains a focused implementation candidate with security-sensitive delivery-policy changes that need maintainer review rather than cleanup closure.

## Best Possible Solution

Continue reviewing this PR as the implementation candidate. The best path is to harden and land the `trustedLocalMedia`-gated voice-media exception, keep ordinary text and untrusted media suppressed in `message_tool_only` groups, stop duplicate WhatsApp text sends for voice notes, wrap declared PCM into a deliverable container with observable mismatch handling, and preserve the targeted regression tests. Close only after this PR merges or a newer canonical fix supersedes it.
