## Summary

Keep this PR open. Current main still has the user-visible behaviors the PR targets: suppressed group delivery drops TTS voice media, OpenAI-compatible PCM TTS is returned as raw `.pcm`, and WhatsApp voice-note sends duplicate visible text. The PR is a live implementation candidate, not obsolete cleanup.

## Best Possible Solution

Continue reviewing this PR as the implementation candidate. The best path is to land a hardened trusted-local-media exception with the included regression tests, keep untrusted and remote media suppressed, wrap valid declared PCM into a deliverable container, stop duplicate WhatsApp voice-note text sends, and update the WhatsApp/TTS docs before or with merge.
