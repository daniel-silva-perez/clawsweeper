## Summary

Keep open. Current main does not implement this PR's intended behavior: message-tool-only delivery still suppresses final/block replies without an audioAsVoice media exception, OpenAI-compatible PCM responses are returned as raw pcm rather than WAV, and the WhatsApp paths still duplicate visible text after voice-note media. The PR appears to be a focused bug-fix candidate with regression-test coverage in the provided diff, not obsolete cleanup.

## Best Possible Solution

Continue reviewing this PR as the implementation candidate. The best path is to validate the scoped audioAsVoice media exception for final and pending block/tool media, keep text suppression intact for message_tool_only groups, remove duplicate WhatsApp text sends for voice notes, and add/keep targeted regression tests plus the normal changed-surface gate before merging. Do not close this PR unless it is merged or superseded by another canonical fix.
