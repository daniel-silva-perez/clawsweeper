## Summary

Keep this issue open. Current main has the updatedAt-based Sessions panel MVP and live sessions.changed refresh plumbing, but it still lacks the requested semantic activity model: no session-level lastMeaningfulActivityAt or lastVisibleConversationAt, no visible activity reason, no sort key that excludes heartbeat/internal bookkeeping churn, and no parent sort rollup from child/sub-agent activity. The raw sub-agent label concern was split to #51030, so that closed related item does not resolve the remaining sorting/activity contract here.

## Best Possible Solution

Keep this issue open and implement a Gateway-backed session activity contract: store and expose raw event time, meaningful activity time, visible conversation time, and a concise activity reason; classify heartbeat/internal/background writes so they do not bubble sessions; roll child/sub-agent meaningful activity up to the parent row for sorting; then update sessions.list, sessions.changed, docs, and the Control UI to sort by that field and display the reason line. Treat #51030 as the already-split label/readability follow-up.
