## Summary

Keep this issue open. Current main has a partial sessions UX improvement: the Control UI defaults to Updated descending and reloads on session-change events. It still does not implement the requested semantic activity model: there is no session-level `lastMeaningfulActivityAt`, no visible reason label, no sort key that excludes heartbeat/internal churn, and no parent-session sort rollup from child activity. The sub-agent label concern was split to #51030, but the core sorting request remains actionable here.

## Best Possible Solution

Keep this issue open and implement a session activity contract in the Gateway: track raw event time, meaningful activity time, visible conversation time, and a displayable reason; classify heartbeat/internal/background writes so they do not bubble sessions; roll child/sub-agent activity up to the parent row for sorting; then update the Control UI to sort by that field and show the reason line. Treat #51030 as the separate label/readability follow-up that has already been split out.
