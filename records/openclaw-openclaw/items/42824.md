## Summary

Keep this PR open. Current main still does not implement the intended IRC default: when `channels.irc.blockStreaming` is unset, IRC passes `disableBlockStreaming: undefined`, which lets the shared reply resolver inherit `agents.defaults.blockStreamingDefault`. The PR remains a narrow implementation candidate with regression coverage, and I did not find security- or supply-chain-sensitive changes in its diff.

## Best Possible Solution

Keep this PR open as the implementation candidate. Rebase or port the IRC-owned helper onto current main so `disableBlockStreaming` is true unless `account.config.blockStreaming === true`, keep unset/false/true regression coverage in `extensions/irc/src/inbound.policy.test.ts`, then run the targeted IRC tests and changed gate before merging.
