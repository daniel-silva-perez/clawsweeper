## What This Changes

The PR changes IRC inbound reply dispatch to disable block streaming unless the IRC account explicitly sets `blockStreaming: true`, and adds policy tests for unset, false, and true account values.

## Best Possible Solution

Land or port the IRC-owned default so IRC sends final replies by default and only enables block streaming when `channels.irc.blockStreaming` is explicitly true, with the proposed regression tests preserved.
