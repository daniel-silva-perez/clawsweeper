## Summary

Keep open. Current main and the latest release still let unset IRC `blockStreaming` fall through as `disableBlockStreaming: undefined`, which can inherit the global block-streaming default and matches the reported final-reply suppression path. The PR is a focused IRC-owned fix with no definite correctness or security finding, so the next step is maintainer review and validation rather than cleanup closure.

## What This Changes

The PR adds an IRC helper that disables block streaming unless the IRC account config sets `blockStreaming: true`, wires it into inbound reply dispatch, and adds policy tests for unset, false, and true config values.

## Best Possible Solution

Merge or port the IRC-owned default so final IRC replies are preserved unless IRC block streaming is explicitly opted in, while keeping regression coverage for unset, false, and true account config values.
