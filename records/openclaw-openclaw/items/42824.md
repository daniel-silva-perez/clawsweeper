## Summary

Keep open. Current main still leaves IRC able to inherit the global block-streaming default when IRC account config omits `blockStreaming`, while this PR directly targets that path with a narrow IRC-owned default change and regression tests.

## What This Changes

This PR changes IRC reply dispatch to disable block streaming unless the IRC account explicitly sets `blockStreaming: true`, and adds policy tests for unset, false, and true account values.

## Best Possible Solution

Land or port the IRC-owned default so IRC sends final replies by default and only enables block streaming when `channels.irc.blockStreaming` is explicitly true, with the proposed unset/false/true regression coverage preserved.
