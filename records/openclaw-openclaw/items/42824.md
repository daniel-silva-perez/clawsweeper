## Summary

Keep open. Current main and the latest stable release still have the IRC unset-config fallback that can enable block streaming from the global default, while this PR remains a focused IRC-owned fix with no blocking review finding; it needs maintainer rebase or port validation before merge.

## What This Changes

The PR changes IRC inbound reply options so unset or false IRC `blockStreaming` disables block streaming while explicit true opts in, and adds policy tests for all three config states.

## Best Possible Solution

Land or port the IRC-owned default so final IRC replies are preserved unless IRC block streaming is explicitly opted in, keep regression coverage for unset, false, and true account config values, then validate the IRC test surface and changed gate before landing.

## Reproduction Assessment

Yes. On current main, set `agents.defaults.blockStreamingDefault` to `on` with `channels.irc.blockStreaming` unset: IRC passes `disableBlockStreaming: undefined`, and the shared resolver enables block streaming from the global default.

## Solution Assessment

Yes. The PR keeps the fix at the IRC ownership boundary, preserves explicit per-account opt-in with `blockStreaming: true`, and avoids changing the shared resolver or other channels.
