## Summary

Keep this issue open. Current main still lacks a TUI subscription/handler for live transcript `session.message` events, and an open maintainer PR (#73402) currently targets the exact fix with closing syntax, so the issue should remain open until that PR is reviewed, merged, or closed.

## Best Possible Solution

Keep this issue open while #73402 is reviewed. The best fix remains a focused TUI change: add active-session subscribe/unsubscribe support to the gateway TUI backend/client, subscribe after connect and session switches, handle `session.message` for the active session without disrupting active streams, ignore other sessions, clean up stale subscriptions, and add targeted TUI regression tests. If #73402 merges, close this as implemented by that merge; if it is closed unmerged, keep this issue as the canonical bug.
