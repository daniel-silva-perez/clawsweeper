## Summary

Keep open. Current main still exposes the Gateway transcript subscription API, but the bundled TUI does not subscribe to `session.message` transcript updates or route those events into the active chat view. The open maintainer PR #73402 is the active implementation path and explicitly targets this issue, so this should remain open until that PR is reviewed and merged or closed unmerged.

## Best Possible Solution

Keep this issue open while #73402 is reviewed. The best fix is the focused TUI change described there: add active-session subscribe/unsubscribe support in the Gateway TUI backend/client, subscribe after connect and session switches, route `session.message` for the active session without disrupting local streams, ignore non-active sessions, clean up stale subscriptions, and add targeted TUI regression tests. If #73402 merges, close this as implemented by that merge; if it closes unmerged, keep this issue as the canonical bug.
