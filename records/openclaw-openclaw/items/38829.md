## Summary

Keep open. Current main still lacks a bundled TUI subscription/handler for live transcript `session.message` events, while the gateway already exposes and tests the needed `sessions.messages.subscribe` API. Open maintainer PR #73402 remains open and unmerged with closing syntax for this exact issue, so this issue should stay open until that PR is reviewed, merged, or closed.

## Best Possible Solution

Keep this issue open while #73402 is reviewed. The best fix remains a focused bundled TUI change: add active-session subscribe/unsubscribe support to the gateway TUI backend/client, subscribe after connect and after session switches, handle `session.message` for the active session without disrupting active local streams, ignore other sessions, clean up stale subscriptions, and add targeted TUI regression tests. If #73402 merges, close this as implemented by that merge; if it is closed unmerged, keep this issue as the canonical bug.
