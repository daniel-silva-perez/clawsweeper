## Summary

Keep open. Current main does not contain this PR's routeReply-based direct final-output relay, but the proposed patch should not merge as-is because it can corrupt streamed whitespace and can duplicate the existing ACP task terminal delivery path; the direct child-output path also needs maintainer approval against the documented parent-completion handoff model.

## What This Changes

This PR adds routeReply-based final and error delivery from the ACP parent stream relay for non-threaded parent sessions, plus tests for direct-chat success, threaded suppression, lookup failure, and error delivery.

## Best Possible Solution

Choose one owner for ACP completion delivery. Prefer extending the existing task-completion or parent handoff path to carry the final visible child result with sanitization, thread policy, and duplicate suppression; if maintainers explicitly want a direct relay, rework it to preserve streamed whitespace, suppress the generic terminal notification for the same run, use the current relay delivery context, and cover success, error, truncation, threaded, and duplicate paths in tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve whitespace-only assistant deltas:** `src/agents/acp-spawn-parent-stream.ts:367`
  - body: The new direct-reply buffer is appended after the existing `!delta.trim()` early return, so standalone space or newline chunks are discarded before direct delivery. Current assistant event emitters can derive deltas from `cleanedText.slice(...)`, so a stream split like `"hello"`, `" "`, `"world"` can become `"helloworld"` in the user-visible final reply.
  - confidence: 0.86
- **[P2] Suppress terminal task delivery before direct sending:** `src/agents/acp-spawn-parent-stream.ts:395-400`
  - body: Current main already sends this ACP run's terminal task notification to the requester origin. Adding this `routeReply` on lifecycle end sends the raw child result while the task registry can still send the generic completion message, so non-threaded chats can receive duplicate completion messages for one child run.
  - confidence: 0.8
