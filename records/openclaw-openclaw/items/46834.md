## Summary

Keep open for maintainer review. The PR targets a real ACP parent-relay delivery gap, but current main has since grown a task-notifier/parent-streaming path around the same surface, and this branch still has blocking correctness issues in the proposed direct-reply code.

## What This Changes

The PR adds ACP parent-stream relay buffering that sends final or error text through `routeReply` for non-threaded direct-chat parent sessions, with unit coverage for direct delivery, threaded exclusion, lookup failure, and error delivery.

## Best Possible Solution

Fold the useful behavior into the current ACP parent-owned delivery model deliberately: preserve route context and de-duplication, deliver the final visible result once when product policy wants raw child output, keep threaded surfaces out of the direct path, and test whitespace-preserving deltas plus current task-notifier interactions.

## Reproduction Assessment

Yes for the PR defects: the type issue is visible from the PR head source and `routeReply` contract, and the whitespace bug reproduces by streaming `"hello"`, `" "`, `"world"` before lifecycle end. The original direct-chat report also has concrete WhatsApp ACP steps, but this review did not run a live channel smoke.

## Solution Assessment

No, not as-is. The narrow direct-reply idea may be useful, but the patch should first be rebased onto the newer task/relay code and should preserve raw assistant whitespace while keeping `cfg` and the direct target narrowed together.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Require cfg before direct replies:** `src/agents/acp-spawn-parent-stream.ts:302`
  - body: `cfg` is declared as possibly `undefined` after the best-effort `loadSessionEntry` catch, but this call passes it to `routeReply`, whose params require `cfg: OpenClawConfig`. With strict TypeScript this branch should fail to typecheck; keep the config and target in one narrowed object or return early unless both are present.
  - confidence: 0.92
- **[P2] Preserve whitespace-only deltas in direct replies:** `src/agents/acp-spawn-parent-stream.ts:363-367`
  - body: The new direct-reply buffer is appended only after the existing `!delta.trim()` return, so standalone spaces or newlines emitted as their own assistant chunks are dropped. A stream like `"hello"`, `" "`, `"world"` would be delivered as `helloworld`; append the raw delta before the progress-only trim filter or maintain a separate final-output buffer.
  - confidence: 0.9
