## Summary

Keep open: the reported gap remains meaningful on current main, but this PR is stale and the patch has build and behavior blockers. Current main now has an ACP task-notifier path that overlaps with the proposed independent direct send, so the fix needs to be adapted rather than merged as-is.

## What This Changes

The PR adds direct `routeReply` delivery from the ACP parent stream relay for non-threaded parent delivery contexts and adds tests for direct-chat completion, threaded exclusion, startup best-effort behavior, and error delivery.

## Best Possible Solution

Integrate final visible ACP child output into the current task-completion delivery model for parent-owned `streamTo: "parent"` runs, preserving exact streamed text boundaries, using the parent/requester origin once, and suppressing duplicate generic terminal notifications when the final result is delivered.

## Reproduction Assessment

Yes. The PR body and tests provide a direct-chat scenario, and static inspection of current main shows the relay still only emits parent-session system events while the task notifier direct-sends a generic terminal message. The PR-specific blockers are also reproducible by source inspection: `cfg` is optional at the `routeReply` call and whitespace-only deltas are returned before buffering.

## Solution Assessment

No, not as written. The direction is useful, but the patch should be rebased onto current main and folded into the existing task delivery path, with a required config value, whitespace-preserving buffering, and one terminal outbound message per ACP child run.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Guard cfg before routing direct replies:** `src/agents/acp-spawn-parent-stream.ts:302`
  - body: `cfg` is declared as possibly undefined after the best-effort session lookup, but `routeReply` requires an `OpenClawConfig`. This should fail the TypeScript lane; keep `cfg` in the same object as `directReplyTarget` or return before calling `routeReply` when it is missing.
  - confidence: 0.94
- **[P2] Append whitespace-only deltas before returning:** `src/agents/acp-spawn-parent-stream.ts:367`
  - body: The direct-reply buffer is appended only after `!delta.trim()` returns, so standalone space/newline chunks are dropped. Current assistant streaming can emit deltas from `cleanedText.slice(...)`, so a reply streamed as `"hello"`, `" "`, `"world"` can be delivered as `"helloworld"`.
  - confidence: 0.92
- **[P2] Fold final direct delivery into the task notifier:** `src/agents/acp-spawn-parent-stream.ts:396-400`
  - body: On current main, `streamTo: "parent"` ACP runs already create a pending ACP task with `requesterOrigin`, and the task registry sends the terminal update to that same channel. This new independent `routeReply` call would leave the generic task terminal send active, producing duplicate completion messages in direct chats.
  - confidence: 0.84
