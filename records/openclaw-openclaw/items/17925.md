## Summary

Keep this issue open. Current main implements the Gemini side as a bundled `gemini` web_search provider using Google Search grounding, but the requested ZAI/GLM native web_search path is still missing from the ZAI plugin manifest, runtime registration, contract expectations, and docs. The related ZAI/Zhipu PRs mentioned in the issue context are closed unmerged, and the remaining Gemini freshness behavior is separately tracked by #66498.

## Best Possible Solution

Keep this issue open until the bundled ZAI plugin registers a provider-owned GLM/ZAI web_search provider through the existing `webSearchProviders` contract, with runtime code, manifest metadata, docs, and registration/provider tests. The Gemini portion is already implemented; freshness semantics should stay tracked by #66498.
