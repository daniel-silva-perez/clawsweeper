## Summary

Keep open. Current main implements the Gemini half through the bundled Google/Gemini web_search provider, but the requested ZAI/GLM native search path is still absent from the ZAI plugin manifest, runtime registration, shared registration contract, docs, and tests. The related Gemini PR is merged; the prior ZAI search PRs were closed unmerged, so the remaining user-visible request is still valid.

## What This Changes

Add provider-native web_search support for ZAI/GLM and Google/Gemini so users can use provider-owned search without a separate Brave, Perplexity, or similar search key.

## Best Possible Solution

Add a bundled ZAI-owned web_search provider through the existing webSearchProviders plugin contract, with manifest metadata, public provider artifact, lazy runtime execution, ZAI credential/config reuse, guarded HTTP handling, docs, changelog, and registration/runtime tests. Keep the existing Gemini provider intact and leave Gemini freshness behavior to #66498.
