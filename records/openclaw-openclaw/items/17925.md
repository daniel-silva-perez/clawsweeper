## Summary

Keep this issue open. Current main already implements the Gemini side as a bundled `gemini` web_search provider using Google Search grounding, but the ZAI/GLM side remains missing from the ZAI plugin manifest, registration, shared registration expectations, tests, and docs. The related Zhipu/ZAI PRs named in the discussion are closed unmerged, and Gemini freshness is separately tracked by #66498.

## What This Changes

Add provider-native or provider-owned `web_search` support for ZAI/GLM and Google Gemini so users can search with existing provider credentials instead of separate third-party search keys.

## Best Possible Solution

Keep this open until the bundled ZAI plugin registers a provider-owned GLM/ZAI `web_search` provider through the existing `webSearchProviders` contract, with runtime code, manifest metadata, docs, and registration/provider tests. Preserve the current Gemini implementation and leave Gemini freshness behavior to #66498.
