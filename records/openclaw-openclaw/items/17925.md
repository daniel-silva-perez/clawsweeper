## Summary

Keep open. Current main implements the Gemini half through the bundled Google web_search provider, but the ZAI/GLM half is still absent from the ZAI plugin manifest, runtime registration, shared registration contract, tests, and docs. Related Gemini work is merged, while the ZAI-related attempts remain closed unmerged and no open replacement PR was found in the provided context or a bounded GitHub search.

## What This Changes

Request native/free provider-owned web_search support for ZAI/GLM and Google/Gemini so users can search with existing model-provider credentials instead of a separate third-party search key.

## Best Possible Solution

Add a bundled ZAI-owned web_search provider through the existing webSearchProviders plugin contract, with manifest metadata, lazy runtime execution, ZAI credential/config reuse, docs, changelog, and registration/runtime tests. Keep the existing Gemini provider intact and leave Gemini freshness behavior to #66498.

## Reproduction Assessment

Yes. The high-confidence structural reproduction is that current main has no ZAI webSearchProviders manifest entry, no registerWebSearchProvider call, and no ZAI web_search runtime, while Gemini has all three.

## Solution Assessment

Yes. A provider-plugin implementation is the narrowest maintainable solution because OpenClaw already models Gemini/Kimi/Grok search as plugin-owned web_search providers; injecting a ZAI special case into core model traffic would bypass that contract.
