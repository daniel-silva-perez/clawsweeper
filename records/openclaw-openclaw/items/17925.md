## Summary

Keep open. Current main implements the Gemini half through the bundled Google/Gemini web search provider, but the requested ZAI/GLM web_search path is still absent from the ZAI plugin manifest, runtime registration, shared registration contract expectations, docs, and tests. The related Zhipu/ZAI PRs are useful prior art but were closed unmerged; #66498 is only a narrower Gemini freshness follow-up.

## What This Changes

Request native provider-backed web_search support for ZAI/GLM and Google/Gemini so users can use provider-included search without separate Brave, Perplexity, or other search credentials.

## Best Possible Solution

Add a bundled ZAI-owned web_search provider through the existing webSearchProviders plugin contract, with manifest metadata, a public provider artifact, lazy runtime execution, ZAI credential/config reuse, guarded HTTP handling, docs, and registration/runtime tests. Keep the existing Gemini provider intact and leave Gemini freshness behavior to #66498.
