## Summary

Keep open. Current main implements the Gemini side through the bundled Google/Gemini web search provider, but the requested ZAI/GLM native web_search path is still absent from the ZAI plugin manifest, runtime registration, shared registration contract expectations, docs, and tests. The related Zhipu/ZAI implementation PRs were closed unmerged, while #66498 is only a narrower Gemini freshness follow-up.

## What This Changes

Request native provider web_search support for ZAI/GLM and Gemini so users can search with provider API keys instead of separate third-party search credentials.

## Best Possible Solution

Add a bundled ZAI-owned web_search provider through the existing webSearchProviders plugin contract, with manifest metadata, a public provider artifact, lazy runtime execution, ZAI credential/config reuse, guarded HTTP handling, docs, and registration/runtime tests. Keep the existing Gemini provider intact and leave Gemini freshness behavior to #66498.
