## Summary

Keep open. Current main satisfies the Gemini half through the bundled Google/Gemini web_search provider, but the ZAI/GLM native search path is still absent from the ZAI plugin manifest, runtime registration, shared registration contract expectations, docs, and tests. The related Gemini PR is merged, while the prior ZAI search PRs were closed unmerged, so the remaining user-visible request is still valid and narrow enough for a focused provider-plugin fix.

## What This Changes

Request native/provider-owned web_search support for ZAI/GLM and Google/Gemini so users can search with existing model-provider credentials instead of configuring a separate third-party search API key.

## Best Possible Solution

Add a bundled ZAI-owned web_search provider through the existing webSearchProviders plugin contract, with manifest metadata, a lightweight public provider artifact, lazy runtime execution, ZAI credential/config reuse, guarded HTTP handling, docs, changelog, and registration/runtime tests. Keep the existing Gemini provider intact and leave Gemini freshness behavior to #66498.
