## Summary

Keep open. Current main covers the Gemini side through the bundled Gemini web_search provider, but the ZAI/GLM side is still missing from the bundled ZAI plugin contract, registration, runtime provider, tests, and docs. The discussion already has maintainer signal to keep this open for ZAI work, while Gemini freshness is separately tracked by #66498.

## What This Changes

Request provider-native web_search support for ZAI/GLM and Google/Gemini so users can search through those model providers without a separate third-party search API key.

## Best Possible Solution

Add a bundled ZAI-owned web_search provider through the existing webSearchProviders plugin contract, with manifest metadata, runtime execution, credential/config handling, docs, and registration/provider tests. Keep the existing Gemini provider intact and leave Gemini freshness behavior to #66498.
