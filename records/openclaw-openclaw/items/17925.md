## Summary

Keep open. Current main implements the Gemini side through the bundled Google/Gemini web search provider, but the ZAI/GLM side is still missing from the bundled ZAI plugin manifest, runtime registration, shared registration contract expectations, and web search docs. The issue discussion already correctly distinguishes the landed Gemini/Kimi work from the remaining ZAI request, and the remaining work is narrow enough for a focused provider-plugin fix.

## What This Changes

Add native/provider-owned web search support for ZAI/GLM and Google/Gemini so OpenClaw can use those providers' built-in search without a separate third-party search key.

## Best Possible Solution

Add a bundled ZAI-owned web_search provider through the existing webSearchProviders plugin contract, with manifest metadata, lazy runtime execution, credential/config handling, docs, and provider registration/runtime tests. Keep the existing Gemini provider intact and leave Gemini freshness behavior to #66498.
