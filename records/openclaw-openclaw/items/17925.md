## Summary

Keep open. Current main covers the Gemini half through the bundled Google/Gemini web search provider, but the requested ZAI/GLM web_search path is still missing from the ZAI plugin manifest, runtime registration, shared contract expectations, tests, and docs. The prior Zhipu/ZAI PRs are useful implementation context but were closed unmerged, and #66498 only tracks a narrower Gemini freshness follow-up.

## What This Changes

Request to add native provider-backed web_search support for ZAI/GLM and Gemini so configured model-provider keys can power web search without separate third-party search credentials.

## Best Possible Solution

Add a bundled ZAI-owned web_search provider through the existing webSearchProviders plugin contract, with manifest metadata, a public provider artifact, lazy runtime execution, ZAI credential/config reuse, guarded HTTP handling, docs, and registration/runtime tests. Keep the existing Gemini provider intact and leave Gemini freshness behavior to #66498.
