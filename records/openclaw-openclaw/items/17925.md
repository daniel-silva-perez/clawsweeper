## Summary

Keep open. Current main implements the Gemini half through the bundled Google/Gemini web_search provider, but ZAI/GLM remains missing from the ZAI plugin manifest, runtime registration, shared registration contract, and docs. The prior ZAI implementation PRs were closed unmerged, while #66498 is a separate Gemini freshness follow-up, so there is still narrow work to do.

## What This Changes

Request provider-owned web_search support for ZAI/GLM and Google/Gemini so users can use those providers' native search without a separate third-party search key.

## Best Possible Solution

Add a bundled ZAI-owned web_search provider through the existing webSearchProviders plugin contract, with manifest metadata, a public provider artifact, lazy runtime execution, ZAI credential/config reuse, guarded HTTP handling, docs, and registration/runtime tests. Keep the existing Gemini provider intact and leave Gemini freshness behavior to #66498.
