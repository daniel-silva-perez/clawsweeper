## Summary

Keep open. Current main covers the Gemini side through the bundled Gemini web_search provider, but the ZAI/GLM side is still missing from the bundled ZAI plugin manifest, registration path, shared registration contract expectations, docs, and provider runtime.

## What This Changes

Request native provider-backed web_search support for ZAI/GLM and Google/Gemini so users can search with existing model-provider credentials instead of configuring a separate search API key.

## Best Possible Solution

Add a bundled ZAI-owned web_search provider through the existing webSearchProviders plugin contract, with manifest metadata, lazy runtime execution, credential/config handling, docs, and provider registration tests. Keep the existing Gemini provider intact and leave Gemini freshness behavior to #66498.
