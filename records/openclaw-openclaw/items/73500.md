## Summary

Keep open. Current main still has the gateway/runtime mismatch described by the linked bug: the gateway `agent` handler rejects `session.sendPolicy=deny` before checking delivery intent, while the runtime agent path only enforces that policy for explicit delivery. The PR is a focused implementation candidate and I did not find a blocking correctness or supply-chain issue in the provided diff.

## What This Changes

The PR moves the gateway `agent` handler's send-policy denial under `request.deliver === true`, adds gateway regression tests for non-delivery allow versus `deliver: true` denial, and adds a changelog fix entry.

## Best Possible Solution

Review and land this PR, or an equivalent maintainer patch, with the gateway send-policy check gated on explicit delivery and regression coverage for both non-delivery execution under deny policy and `deliver: true` rejection. After merge, the linked #73381 bug can close through the PR relationship.

## Reproduction Assessment

Yes. Current main can be reproduced by forcing `resolveSendPolicy` to return `deny` for a non-delivery gateway `agent` invocation; source inspection shows the handler rejects before delivery intent is considered, and the linked #73381 report includes concrete CLI steps from OpenClaw `2026.4.25`.

## Solution Assessment

Yes. The PR is the narrowest maintainable fix because it aligns the gateway handler with the existing runtime `agent-command.ts` contract while preserving the denial path for explicit outbound delivery.
