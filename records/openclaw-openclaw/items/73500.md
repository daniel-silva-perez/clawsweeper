## Summary

Keep this PR open. Current main still checks `resolveSendPolicy(...) === "deny"` in the gateway `agent` handler before delivery intent is computed, while the runtime `agent-command.ts` path already gates that policy on `opts.deliver === true`. The PR is a focused implementation candidate for the linked #73381 bug and includes targeted regression coverage for the deny/non-delivery split.

## Best Possible Solution

Review and land this PR, or an equivalent maintainer patch, by moving the gateway `resolveSendPolicy` deny check under `request.deliver === true` while retaining the runtime `agent-command.ts` guard. Keep regression tests for both non-delivery allow under deny policy and `deliver: true` rejection, then let #73381 close after the fix merges.
