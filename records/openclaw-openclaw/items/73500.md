## Summary

Keep this PR open. Current main still enforces `resolveSendPolicy(...) === "deny"` in the gateway `agent` handler before it computes whether the request actually asks for delivery, so the PR's intended fix is not implemented on main. The linked bug remains plausible and the PR is a focused implementation candidate with regression tests for the deny/non-delivery split.

## Best Possible Solution

Review and land this PR, or an equivalent maintainer patch, by moving the gateway `resolveSendPolicy` deny check under `request.deliver === true` while retaining the runtime `agent-command.ts` guard. Keep regression coverage for both non-delivery allow and delivery deny cases, then let the linked #73381 close after the fix merges.
