## Summary

Keep this PR open. Current main still rejects gateway `agent` requests with `sendPolicy=deny` before delivery intent is applied, while the runtime agent command path already gates that policy on explicit delivery. The PR is a focused fix for the linked #73381 regression, but GitHub currently reports the branch as dirty/unmergeable, so it needs maintainer review and likely a rebase before landing.

## What This Changes

The PR gates the gateway `agent` handler's send-policy rejection on `request.deliver === true`, makes the gateway handler test mock for `resolveSendPolicy` configurable, adds deny/non-delivery regression tests, and adds a changelog entry.

## Best Possible Solution

Review, rebase, and land this PR or an equivalent maintainer patch that moves the gateway `resolveSendPolicy` denial under explicit delivery intent while keeping the runtime `agent-command.ts` guard and regression tests for both non-delivery allow and `deliver: true` rejection.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: with a requested session whose `resolveSendPolicy` returns `deny`, a non-delivery gateway `agent` request currently returns `INVALID_REQUEST` before `wantsDelivery` is computed. The linked #73381 report also provides a concrete CLI smoke repro on OpenClaw 2026.4.25 and a matching diagnostic hotfix.

## Solution Assessment

Yes. The PR is the narrowest maintainable fix because it aligns the gateway preflight check with the existing runtime `opts.deliver === true` gate and preserves explicit delivery rejection. The safer landing path is to rebase the branch and run the targeted gateway test plus the changed gate rather than widening the change into unrelated session policy work.
