## Summary

Keep open. Current main still allows persisted plugin debug entries to flow back into later verbose or trace payloads, and the Active Memory plugin only updates its own entry when its hook runs. This PR remains a focused runtime/session-store fix with targeted regression coverage and no blocking findings from this read-only review.

## What This Changes

This PR adds turn-start cleanup for session plugin debug entries, preserve-activity metadata merging for that cleanup, verbose-refresh suppression after cleanup failure, and targeted session/reply-runner regression tests.

## Best Possible Solution

Land this as a narrow runtime/session-store bug fix after rebasing onto current main and validating the touched tests. Keep the cleanup generic in the reply runtime, preserve activity timestamps for metadata-only clears, and leave Active Memory's plugin-owned writer behavior unchanged.

## Reproduction Assessment

Yes. Seed a session with prior pluginDebugEntries, run a verbose or heartbeat reply where no plugin rewrites debug entries, and current main's refresh/payload path can still append the stale lines; the PR also simulates persisted clear failure for the verbose rehydration case.

## Solution Assessment

Yes. The proposed direction is the narrowest maintainable fix: clear turn-scoped debug metadata at reply turn start, preserve activity timestamps for that metadata-only write, and suppress disk rehydration only when cleanup fails for that turn.
