## Summary

Keep open. Current main still does not provide the browser tab's current URL to `before_tool_call` for `browser` `act`/`type` calls. The browser route now returns a policy-checked `url` in many `/act` responses, which supports the reporter's cache workaround, but that happens after execution and is too late for a domain-pin check before typing a secret.

## Best Possible Solution

Add a narrow browser-owned hook enrichment or documented query seam. Before `browser` `act`/`type` hooks run, resolve the target tab's policy-checked current URL from `targetId`/profile/target and expose it as a reserved field such as `_currentUrl`, or expose an explicit SDK/runtime helper that `before_tool_call` hooks can call. Cover it with browser-tool and hook-contract tests, while keeping existing after-call URL responses as compatible fallback behavior.
