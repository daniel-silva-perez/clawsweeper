## Summary

Keep open. Current main still lacks a trustworthy browser current-URL value at the `before_tool_call` decision point for browser `act`/`type`; the hook pipeline receives only the incoming tool params before browser execution, while the browser route can resolve and policy-filter a tab URL only after the `/act` request is executing. The reporter’s credential-vault workaround is useful and fail-secure, but it confirms the native hook-time contract gap remains.

## What This Changes

The issue requests that browser `before_tool_call` params for `act`/`type` include a trustworthy current page URL, or expose an equivalent hook-time query path for plugins.

## Best Possible Solution

Add an additive browser/plugin contract that gives pre-action browser policy hooks a policy-safe current tab URL for `act`/`type`, or an equivalent supported query seam, with docs and tests covering managed, existing-session, node-proxy, blocked/private URL, and cold-target behavior.
