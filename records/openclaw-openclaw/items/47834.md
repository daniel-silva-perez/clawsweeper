## Summary

Keep open. Current main still lacks a trustworthy browser current-page URL at the `before_tool_call` decision point for browser `act`/`type`: the generic hook runner forwards only the incoming params before the browser tool executes, while the browser route can resolve and policy-filter a tab URL only once the route is already running and returning a result. The reporter's credential-vault workaround is useful and fail-secure, but it confirms the native hook-time contract gap remains.

## What This Changes

Expose the browser's current tab URL, or an equivalent supported query seam, to plugin `before_tool_call` hooks for browser `act`/`type` actions so credential plugins can enforce domain pins before injecting text.

## Best Possible Solution

Add a narrow, additive browser/plugin contract that gives pre-action browser policy hooks a policy-safe current tab URL for `act`/`type`, or an equivalent supported query seam, with docs and tests proving managed, existing-session, node-proxy, blocked/private URL, and cold-target behavior.
