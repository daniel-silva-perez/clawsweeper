## Summary

Keep open. Current main still does not give `before_tool_call` policy hooks a browser tab URL for `act`/`type` decisions: the generic hook pipeline receives the original tool params before browser execution, while the browser route can only add a policy-safe URL to successful responses after the action runs. The reporter's credential-vault workaround is useful and fail-secure, but it confirms the remaining hook-time API gap rather than replacing it.

## What This Changes

The issue requests adding the current browser page URL to `before_tool_call` parameters for browser `act`/`type` actions, or an equivalent hook-time query seam, so plugins can enforce domain-pinned credential injection.

## Best Possible Solution

Add a narrow, additive browser/plugin contract that gives pre-action browser policy hooks a policy-safe current tab URL for `act`/`type`, or an equivalent supported query seam, with docs and tests proving blocked/private URLs are omitted and supported profiles behave consistently.
