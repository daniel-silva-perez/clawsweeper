## Summary

Keep open. Current main still does not provide browser `act`/`type` `before_tool_call` hooks with a trustworthy current tab URL at the hook decision point; the generic hook runner receives only the incoming params before the browser tool executes, while browser route code can only add a policy-checked `url` to post-action responses. The reporter's `openclaw-credential-vault` workaround is useful and fail-secure, but it confirms the native hook-time API gap remains.

## What This Changes

The issue requests a browser/plugin hook contract that exposes the current page URL, or an equivalent query seam, before browser `act`/`type` actions so plugins can enforce domain-pinned credential injection.

## Best Possible Solution

Add a narrow, additive browser/plugin contract that gives pre-action browser policy hooks a policy-safe current tab URL for `act`/`type`, or an equivalent supported query seam, with docs and tests covering managed, existing-session, node, and blocked/private URL behavior.
