## Summary

Keep open. Current main still does not give browser `act`/`type` `before_tool_call` hooks a policy-safe current tab URL at the decision point; the generic hook receives only the incoming params before the browser action executes, while current URL resolution happens inside browser routes for post-action responses. The reporter's `openclaw-credential-vault` workaround is useful and fail-secure, but it confirms the remaining native hook-time API gap.

## What This Changes

Expose the current browser page URL, or an equivalent query seam, to `before_tool_call` hooks for browser `act`/`type` actions so domain-pinned credential plugins can decide before injecting text.

## Best Possible Solution

Add a narrow, additive browser/plugin contract that provides a policy-safe current tab URL for pre-action browser policy hooks, or an equivalent supported query mechanism, with docs and tests proving blocked/private URLs are omitted and managed plus existing-session profiles behave consistently.
