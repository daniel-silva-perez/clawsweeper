## Summary

Keep this issue open. Current main still passes `before_tool_call` the incoming browser tool params only; it does not enrich `browser` `act`/`type` calls with a policy-checked current page URL before execution. The reporter's openclaw-credential-vault v1.0.0-beta.5 workaround is useful and fail-secure, but it is still an in-memory cache workaround and does not close the native hook-time URL gap. Related PR #30323 is closed unmerged and was not a shipped safe fix.

## Best Possible Solution

Keep this open for a maintainer-reviewed, narrow browser/plugin API fix: expose a policy-checked current tab URL to `before_tool_call` for browser `act`/`type` decisions, or provide a supported hook-time browser URL query seam. Keep the implementation in the browser plugin/tool-hook boundary, preserve existing param rewrite/block semantics, avoid unguarded Playwright/CDP URL reads, and test flattened and `request` act params, stale or missing targetIds, node/proxy behavior, and fail-closed credential-domain cases.
