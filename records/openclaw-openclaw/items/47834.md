## Summary

Keep this open. Current main still runs `before_tool_call` with the incoming browser tool params only; browser `act`/`type` calls are not enriched with a policy-checked current page URL before plugin hooks decide whether to rewrite, approve, or block. The credential-vault workaround is useful and fail-secure, but it remains an external cache workaround rather than a native hook-time contract.

## What This Changes

Request that browser `before_tool_call` hook params for `act`/`type` actions include the current page URL, or that hooks get a supported way to query it before credential injection.

## Best Possible Solution

Keep this open for a maintainer-reviewed, narrow browser/plugin API fix: expose a policy-checked current tab URL to `before_tool_call` for browser `act`/`type` decisions, or add a supported hook-time browser URL query seam. The fix should live at the browser tool/hook boundary, preserve existing param rewrite/block/approval semantics, respect node/proxy and existing-session profiles, and avoid unguarded Playwright/CDP URL reads.
