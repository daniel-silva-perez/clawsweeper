## Summary

Keep open. Current main still calls `hasIndexedContent()` from the memory search path and that method still performs SQLite probes against `chunks` and `chunks_fts`. The provided GitHub context shows an open, unmerged PR (#62597) with closing syntax for this issue, so the issue should remain open while that implementation candidate is reviewed.

## Best Possible Solution

Review and either land PR #62597 or replace it with an equivalent memory-core patch that caches the positive indexed-content state, invalidates or recomputes it on deletes and full resets, handles safe reindex and FTS-only writes, and adds a focused manager-level regression test for repeated search probes and cache invalidation.
