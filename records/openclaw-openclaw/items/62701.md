## Summary

Keep this PR open. Current main does not implement the PR’s intended additive hook-context fields: the public before_tool_call event type still exposes only toolName, params, runId, and toolCallId, and the agent tool-definition/abort wrappers still do not plumb extensionContext in the way this PR’s follow-up commits describe. The linked source issue is closed and #13225 is broader adjacent routing work, but neither is a shipped replacement for this PR on main.

## Best Possible Solution

Keep the PR open for maintainer review. If OpenClaw wants this narrow compliance/audit hook context, land or revise this PR with focused tests and docs; broader routing/delegation behavior should remain tracked separately in #13225 rather than treating it as a replacement for these two additive fields.
