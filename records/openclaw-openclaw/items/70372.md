## Summary

Keep this PR open. Current main still does not implement the PR's central Anthropic/Bedrock unphased-thinking fallback: Anthropic streaming emits thinking and text events without assistant phase metadata, while the channel delivery handler only switches to phase-aware visible-text extraction when `resolveAssistantMessagePhase(...)` returns a phase. Related phased-commentary fixes exist, but they do not make this PR obsolete.

## Best Possible Solution

Keep this PR open as a useful implementation candidate, rebase it onto current main, scope the derived phase to the unphased Anthropic/Bedrock path or otherwise prove it cannot affect late-phased providers, add a targeted subscription regression test for thinking events followed by unphased text, and coordinate overlap with #70344 rather than closing this as obsolete.
