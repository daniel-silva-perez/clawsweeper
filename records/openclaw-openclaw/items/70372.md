## Summary

Keep open. The PR targets a real Anthropic/Bedrock streaming leak that current main still does not cover, but the proposed synthetic phase is too broad and can regress OpenAI Responses pending-phase suppression before real phase metadata arrives.

## What This Changes

The PR adds a subscription-state flag for native thinking events and uses it to synthesize a `final_answer` delivery phase for later unphased assistant text, preserving that flag across stream-item state resets.

## Best Possible Solution

Keep the useful Anthropic/Bedrock fix direction, but implement it with an explicit provider/transport signal or a narrowly scoped unphased-thinking fallback that cannot bypass OpenAI Responses' pending-phase guard. Add focused tests for both the intended Anthropic/Bedrock suppression and the OpenAI Responses pending-phase protection.

## Reproduction Assessment

Yes for the code-level behavior. The current Anthropic transport and subscription handler show the reported unphased thinking/text path, and the proposed patch's OpenAI regression follows directly from the existing OpenAI Responses event order where `text_delta` precedes `textSignature` phase metadata.

## Solution Assessment

No. The current PR is not the best way yet because deriving `final_answer` from any prior thinking event is broader than the Anthropic/Bedrock gap; a safer fix should scope the inference to those unphased transports or attach explicit phase metadata at the transport boundary.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Keep pending OpenAI text unclassified until phase arrives:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:457`
  - body: This fallback makes `deliveryPhase` truthy after any thinking event, so OpenAI Responses `text_delta` events no longer hit the existing `isPhasePendingOpenAiResponsesTextItem` return path. OpenAI emits text deltas before encoding the real item phase at `text_end`; a commentary item after reasoning would now be treated as `final_answer` and delivered before the phase says to suppress it. Scope the fallback to the Anthropic/Bedrock unphased stream shape or keep OpenAI pending-phase handling ahead of the synthetic phase.
