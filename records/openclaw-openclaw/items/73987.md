## Summary

Keep this PR open. Current main still scopes Control UI gateway tokens by the exact parsed gateway host, so the localhost/127.x/[::1] same-machine token-sharing behavior proposed here is not implemented. The PR is a focused repair of the closed, unmerged #43254 branch, but it needs maintainer review because it changes bearer-token reuse semantics across loopback aliases and has unresolved review discussion around IPv6/test fidelity.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land this PR or an equivalent focused change after resolving the loopback-token security boundary and IPv6/test review thread, preserving protocol, port, path, and DNS-hostname isolation. #69647 should stay separate because it tracks reverse-proxy path persistence, not same-machine loopback token scope.
