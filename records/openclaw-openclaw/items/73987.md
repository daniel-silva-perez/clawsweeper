## Summary

Keep this PR open. Current main still scopes Control UI gateway tokens by the exact parsed gateway host, so the proposed loopback alias normalization is not implemented, but this branch is not merge-ready because its positive test changes `location.host` while keeping one mocked `sessionStorage`, which does not match browser origin-scoped storage. The Greptile IPv6 bracket finding did not reproduce in the local Node URL API check: `URL.hostname` returned `[::1]` for a bracketed IPv6 URL.

## What This Changes

The PR adds Control UI token-scope normalization for `localhost`, IPv4 `127/8`, and `[::1]`, adds storage tests for same-port/path sharing and isolation, and adds a changelog entry.

## Best Possible Solution

Keep the useful focused loopback-token work alive, but first decide whether the intended fix is same-origin gateway URL normalization or true dashboard-link reuse across different loopback origins. For same-origin normalization, narrow the changelog/tests and explicitly approve or reduce the `127/8` scope; for true cross-origin dashboard links, use a canonical host redirect or server-mediated handoff instead of relying on browser `sessionStorage`.

## Reproduction Assessment

Yes for the review finding: the PR's test changes the mocked host while retaining one `sessionStorage`, while the browser storage contract is origin-scoped. Current main's exact-host token scope is also directly visible in `ui/src/ui/storage.ts`.

## Solution Assessment

No. The storage-key normalization is a narrow implementation for same-origin gateway URL aliases, but it is not the best complete fix for the advertised loopback dashboard-link flow because browser storage is partitioned by origin; the safer path is either narrower messaging/tests or a canonical/server-mediated cross-origin design.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Do not model host changes with one sessionStorage:** `ui/src/ui/storage.node.test.ts:284`
  - body: This test switches `location.host` from `127.0.0.1` to `localhost` and `[::1]` while keeping the same mocked `sessionStorage`. Real browsers associate `sessionStorage` with the page origin, so those hosts get separate storage areas and a token saved before the host change is unavailable after navigating to the alias. Please either narrow this PR to same-origin `gatewayUrl` edits or add a real cross-origin handoff/canonical-host design before claiming dashboard link reuse.
  - confidence: 0.86
