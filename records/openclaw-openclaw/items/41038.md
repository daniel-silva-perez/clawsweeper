## Summary

Keep this PR open: current main still drops structured `stdout` in the Swift Chat UI formatter, and the PR is a narrow implementation with matching regression tests. It should get maintainer review and metadata cleanup rather than be closed or replaced.

## What This Changes

This PR updates the Swift Chat UI tool-result formatter to prefer trimmed structured `stdout` before generic message fields and adds two OpenClawKit formatter tests for stdout precedence.

## Best Possible Solution

Land this formatter change after maintainer review with the closing reference corrected, focused OpenClawKit formatter validation passing, and the scope kept to stdout display for structured successful payloads rather than adding stderr or broader failure-policy behavior in the same PR.

Yes. Static inspection gives a high-confidence reproduction path: a structured JSON tool result with only `stdout` returns an empty display string on current main, and one with both `stdout` and `message` returns the generic message instead of stdout.

## Solution Assessment

Yes. The proposed code path is the narrow maintainable fix because it reuses the existing trimmed string extraction helper, preserves error and nodes-summary precedence, and returns stdout before generic message fields; the redundant empty-string check is nonblocking style cleanup.
