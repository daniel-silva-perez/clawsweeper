## What This Changes

This PR updates the Swift Chat UI tool-result formatter to prefer trimmed structured `stdout` before generic message fields and adds two formatter tests covering exec-style payloads.

## Best Possible Solution

Land the formatter change after maintainer review with the closing reference corrected and focused OpenClawKit formatter validation passing, keeping the scope to stdout display for successful structured payloads rather than adding stderr or broader failure-policy behavior in this PR.

## Reproduction Assessment

Yes. A structured JSON tool result with only `stdout` returns an empty display string on current main, and one with both `stdout` and `message` returns the generic message because `renderDictionary` does not read `stdout`.

## Solution Assessment

Yes. The proposed implementation is the narrow maintainable fix: it reuses the existing trimmed string helper, preserves error and nodes-summary precedence, and returns stdout before generic message fields; the redundant empty-string check is nonblocking style cleanup.
