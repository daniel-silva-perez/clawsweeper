## Summary

Keep open. Current main still does not implement the proposed `===DONE_OK===` cron-success sentinel. The existing classifier only clears trailing structured errors when a later success payload exists or when the trailing error is the narrow `⚠️ ✉️ Message failed` presentation-warning case; real trailing tool errors after successful text remain fatal and still set isolated cron run status to `error`. The PR is plausible but needs maintainer review because a global text sentinel is a cron runtime contract decision and the branch is stale against the current classifier shape.

## Best Possible Solution

Keep this draft PR open for maintainer review. If accepted, rebase it onto current main and implement the sentinel in `src/cron/isolated-agent/helpers.ts` without weakening `runLevelError`, fatal `failureSignal`, denial-token, or `⚠️ ✉️ Message failed` presentation-warning behavior. Add focused regression coverage in `src/cron/isolated-agent.helpers.test.ts`, and decide explicitly whether the token name, scope, and payload-scanning behavior are the right runtime contract.
