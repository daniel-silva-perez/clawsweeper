## Summary

Keep open. Current main still has no `===DONE_OK===` or equivalent sentinel path. It has gained a narrower presentation-warning exception for trailing `⚠️ ✉️ Message failed`, but real trailing errors after successful text are still classified fatal and still drive isolated cron run status to `error`. The PR remains useful, but it needs maintainer review because a global success sentinel is a cron runtime contract decision and the branch is now stale against the evolved classifier.

## Best Possible Solution

Keep this draft PR open for maintainer review, then either land a rebased narrow fix or ask for a revised contract. The implementation should live in `src/cron/isolated-agent/helpers.ts`, keep `runLevelError`, fatal `failureSignal`, and denial-token paths authoritative, avoid weakening the current `⚠️ ✉️ Message failed` presentation-warning handling, and add focused regression coverage near `resolveCronPayloadOutcome` tests, preferably in `src/cron/isolated-agent.helpers.test.ts` on current main.
