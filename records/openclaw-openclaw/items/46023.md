## Summary

Keep this issue open. Current main still computes Telegram mediaMaxBytes, but the final-reply delivery path does not thread it into deliverReplies, deliverMediaReply, or loadWebMedia. The linked implementation candidate #46425 was closed unmerged, and the current source still matches the reported gap.

## Best Possible Solution

Keep the issue open and either revive/review #46425 or implement the same narrow fix on main: add mediaMaxBytes to Telegram delivery base options, deliverReplies, and deliverMediaReply; pass it as maxBytes to buildOutboundMediaLoadOptions; cover both normal final replies and native-command delivery paths with focused regression tests.
