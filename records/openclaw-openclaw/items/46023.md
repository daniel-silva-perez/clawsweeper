## Summary

Keep open. Current main still has the reported Telegram reply-media limit gap, and the issue is paired with open PR #74490 using closing syntax, so this should remain open until that PR is reviewed and merged or closed.

## What This Changes

The issue asks for Telegram final reply media delivery to honor the configured `channels.telegram.mediaMaxMb` limit by threading the computed byte cap into outbound media loading.

## Best Possible Solution

Review #74490 as the canonical implementation candidate, ensure it carries the configured media cap through normal message replies and native-command reply delivery into `loadWebMedia`, and land it with focused regression tests for configured and default cap behavior.
