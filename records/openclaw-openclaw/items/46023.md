## Summary

Keep open. Current main still has the reported Telegram final-reply media limit gap: the Telegram plugin computes the configured media cap and direct sends pass it into media loading, but the normal final reply and native command delivery paths do not thread any media cap into `deliverReplies`, `deliverMediaReply`, or `loadWebMedia`. The related implementation PR is closed unmerged, so it is useful context but not a superseding fix.

## What This Changes

Telegram final replies with `mediaUrl` or `mediaUrls` should honor the configured `channels.telegram.mediaMaxMb` cap during outbound media delivery.

## Best Possible Solution

Land a narrow Telegram plugin fix that carries the computed media cap from bot setup through normal message processing and native command delivery into `deliverReplies` and `deliverMediaReply`, then passes it as `maxBytes` to `buildOutboundMediaLoadOptions`, with regression tests proving configured limits reach `loadWebMedia` for final media replies.
