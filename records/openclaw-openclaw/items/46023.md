## Summary

Keep open. Current main still has the reported Telegram reply-media limit gap: the configured media cap is computed and documented, but the final reply and native-command delivery paths do not pass it into `deliverReplies`, `deliverMediaReply`, or `loadWebMedia`. Open PR #74490 is already paired to this issue with closing syntax, so the issue should remain open until that PR is reviewed and merged or closed.

## What This Changes

Telegram final replies with outbound media should honor `channels.telegram.mediaMaxMb` by threading the computed byte cap into the reply media loading path.

## Best Possible Solution

Review and land #74490, or an equivalent narrow Telegram plugin fix, that carries `mediaMaxBytes` from bot setup through message processing and native command delivery into `deliverReplies` and `deliverMediaReply`, then passes it as `maxBytes` to `buildOutboundMediaLoadOptions` with regression tests for configured and default cap behavior.
