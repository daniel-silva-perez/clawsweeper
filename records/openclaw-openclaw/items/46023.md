## Summary

Keep open. Current main still has the reported Telegram outbound reply-media limit gap, and open PR #74490 is already paired to this bug with closing syntax, so the issue should remain open until that implementation is reviewed and merged or closed.

## What This Changes

Telegram final assistant and native-command replies with media should honor the configured `channels.telegram.mediaMaxMb` limit in the outbound reply delivery path.

## Best Possible Solution

Review and land #74490, or an equivalent narrow Telegram plugin fix, that threads `mediaMaxBytes` from bot setup through normal message processing and native-command delivery into `deliverReplies` and `deliverMediaReply`, then passes it as `maxBytes` to `buildOutboundMediaLoadOptions` with regression coverage for configured and default cap behavior.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: configure Telegram `mediaMaxMb`, trigger a final reply with `mediaUrl`/`mediaUrls`, and current main shows the computed cap is never accepted by `deliverReplies` or passed to `loadWebMedia` on the reply-media path.

## Solution Assessment

Yes, the paired PR direction is the narrow maintainable fix: parameter-thread the existing computed cap through the Telegram-owned reply delivery path and test that `loadWebMedia` receives it. Closing this issue before that PR merges would be premature because current main still lacks the propagation.
