## Summary

Current main still has the reported Telegram reply-media limit gap, and open PR #74490 is already paired to the bug with closing syntax. The issue should stay open until that implementation, or an equivalent narrow fix, is reviewed and merged.

## What This Changes

The issue requests that Telegram final replies with outbound media honor configured `mediaMaxMb` by threading the computed byte cap through reply delivery into media loading.

## Best Possible Solution

Land #74490, or an equivalent narrow Telegram plugin fix, that carries `mediaMaxBytes` from bot setup through message processing and native command delivery into `deliverReplies` and `deliverMediaReply`, then passes it as `maxBytes` to `buildOutboundMediaLoadOptions` with regression tests for configured and default cap behavior.
