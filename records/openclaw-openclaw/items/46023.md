## Summary

Keep open. Current main still matches the reported Telegram reply-media limit gap, and the issue is now paired with open PR #74490 using closing syntax, so the issue should stay open until that PR is merged or closed. The code computes the Telegram media cap and the direct-send path uses it, but the final reply and native command reply delivery paths still do not pass a max byte cap into reply media loading.

## What This Changes

Telegram final replies with media should honor configured `channels.telegram.mediaMaxMb` by threading the computed media byte cap into outbound reply media loading.

## Best Possible Solution

Review #74490 as the canonical implementation candidate, ensure it threads the configured media cap through message and native-command reply delivery into `loadWebMedia`, and land it with focused regression tests covering configured and default cap behavior.
