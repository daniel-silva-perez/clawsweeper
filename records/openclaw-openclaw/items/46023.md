## Summary

Current main still matches the reported Telegram outbound final-reply media limit bug. The Telegram config/docs expose `mediaMaxMb`, bot setup computes `mediaMaxBytes`, and the direct send path honors it, but the final reply delivery path still does not pass that cap into `deliverReplies`, `deliverMediaReply`, or `loadWebMedia`. The related fix PR #46425 is useful prior implementation context, but it was closed unmerged, so it does not supersede this issue.

## Best Possible Solution

Keep this issue open and implement the narrow Telegram plugin fix: thread the computed `mediaMaxBytes` from bot setup through normal message processing, dispatch delivery base options, native command delivery base options, `deliverReplies`, and `deliverMediaReply`, then pass it as `maxBytes` to `buildOutboundMediaLoadOptions` while preserving current behavior when no cap is supplied. Use #46425 as prior context, but create a fresh current-main PR with focused regression tests.
