## Summary

Keep open. Current main still lacks first-class outgoing Telegram custom emoji support for messages, captions, replies/edits, and the shared message tool; the closed entity-tool request was consolidated into this tracker, and the open reaction PR covers only a narrower reaction subset.

## What This Changes

Add Telegram Bot API custom emoji support for outgoing messages, captions, replies, and shared message-tool flows.

## Best Possible Solution

Define a small public custom-emoji representation first, then implement validated Telegram entity handling through send, caption, reply, edit, delivery, and streaming paths plus the shared message tool, with docs and focused regression coverage that preserve the existing HTML fallback behavior.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: current main has no custom-emoji span or entity-array send surface, and Telegram send/edit/delivery paths still rely on HTML parse_mode without entities or caption_entities.

## Solution Assessment

Unclear. The feature direction is valid, but the best maintainable fix needs maintainer API-shape judgment; a narrow raw <tg-emoji> passthrough would not cover message-tool entities, captions, edits, chunking, or offset validation.
