## Summary

Keep this issue open. Current main still lacks first-class outgoing Telegram custom emoji support for messages/captions/replies/edits and the shared message tool, and the related items do not supersede the remaining API-shape work.

## What This Changes

Add Telegram Bot API custom emoji support for outgoing messages, captions, replies, and related message-tool flows.

## Best Possible Solution

Define the public custom-emoji representation first, then implement validated Telegram entities through send, caption, reply, edit, delivery, and streaming paths plus the shared message tool, with docs and focused regression coverage that preserve current fallback behavior.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: current main has no custom-emoji span or entity-array send surface, and Telegram send/edit/delivery paths still rely on HTML parse_mode without entities or caption_entities.

## Solution Assessment

Unclear until maintainer design is settled. The narrowest maintainable fix is likely a small public representation plus Telegram-owned validation and offset handling, not a one-off raw HTML passthrough or an unscoped entity bag.
