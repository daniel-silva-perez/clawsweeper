## Summary

Keep this issue open. Current main still does not provide first-class outgoing Telegram custom emoji support for messages/captions/replies: the shared Markdown IR has no custom-emoji span, Telegram rendering emits only the existing safe HTML subset, send/caption paths still use HTML parse mode rather than entity arrays, and the shared message tool has no custom emoji/entity input surface. The narrower #40848 was closed as a duplicate of this canonical tracker, #13905 is closed unmerged prior art, and #47562 is scoped to reactions rather than message/caption rendering.

## What This Changes

Request Telegram custom emoji support for bot messages, captions, and replies, including custom emoji identifiers and rendering according to Telegram Bot API formatting rules.

## Best Possible Solution

Define the public representation first, then implement validated Telegram custom emoji entities through the Telegram plugin send, caption, reply, edit, delivery, and streaming paths plus the shared message tool, with docs and focused regression tests that preserve current HTML fallback behavior.
