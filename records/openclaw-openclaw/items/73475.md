## Summary

Keep open. Current main partially covers the model-name side through bundled Anthropic aliases/defaults, but the direct Anthropic provider path still injects the retired 1M context beta header and still treats 1M context for Claude Opus/Sonnet 4.6 as opt-in rather than native behavior.

## What This Changes

Update OpenClaw's Anthropic provider for Claude Sonnet 4.6 API migration by using the new model ids, removing the retired 1M context beta header, refreshing aliases, and testing compatibility.

## Best Possible Solution

Treat Claude Opus 4.6, Claude Sonnet 4.6, and newer GA 1M Anthropic models as native 1M-capable without adding the retired context beta header, while preserving explicit custom beta support, auth-required OAuth headers, older-model compatibility where still documented, and aligned docs/tests/changelog.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: configure `params.context1m: true` for direct Anthropic API-key traffic on `claude-sonnet-4-6` or `claude-opus-4-6`, and the current wrapper/test path adds `context-1m-2025-08-07`.

## Solution Assessment

Yes. The best fix is a narrow Anthropic-provider and context-window update rather than a global default-context change; it should remove or suppress only the retired 1M context beta behavior for GA 1M models and update the matching tests/docs.
