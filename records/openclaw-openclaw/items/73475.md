## Summary

Keep open. Current main already has the Sonnet 4.6 model aliases, but it still treats 1M Anthropic context as an opt-in beta header path for direct API-key traffic, while current Anthropic release notes and model docs say Claude Opus 4.6 and Sonnet 4.6 support 1M context with no `context-1m-2025-08-07` beta header required. ([platform.claude.com](https://platform.claude.com/docs/en/release-notes/overview))

## What This Changes

The issue requests updating OpenClaw's Anthropic provider for Sonnet 4.6 model IDs, aliases/defaults, and removal of the retired `context-1m-2025-08-07` beta header.

## Best Possible Solution

Update the Anthropic provider so Claude Opus 4.6, Claude Sonnet 4.6, and newer 1M-native Anthropic models are treated as 1M-capable without injecting the retired `context-1m-2025-08-07` beta header, while keeping the other required Anthropic beta headers and aligning provider tests, context-token accounting, docs, and changelog.

## Reproduction Assessment

Yes. A high-confidence static reproduction is available: configure `params.context1m: true` on direct Anthropic API-key traffic for an Opus/Sonnet 4.x model and the current wrapper/test path adds `context-1m-2025-08-07`; live API verification was not run.

## Solution Assessment

Yes, the best fix is a narrow Anthropic-provider update, not a global context default change. It should remove or suppress only the retired 1M context beta for GA 1M models, update context-window resolution for those models, and preserve all unrelated Anthropic headers.
