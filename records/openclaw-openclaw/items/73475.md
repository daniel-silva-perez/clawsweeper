## Summary

Keep open. Current main already covers the Sonnet 4.6 alias/model-name side, but the Anthropic provider, tests, and docs still model 1M context as an opt-in `context-1m-2025-08-07` beta header path; Anthropic release notes say Opus 4.6 and Sonnet 4.6 now support 1M context with no beta header required. citeturn2view1

## What This Changes

The issue requests updating OpenClaw's Anthropic provider for Sonnet 4.6 model IDs, aliases/defaults, API compatibility, and removal of the retired 1M context beta header.

## Best Possible Solution

Update the Anthropic provider and context accounting so Claude Opus 4.6, Claude Sonnet 4.6, Opus 4.7, and newer native-1M Anthropic models resolve to 1,048,576 tokens without requiring `params.context1m` and without adding `context-1m-2025-08-07`, while keeping older beta-only models and unrelated Anthropic headers correctly scoped.

## Reproduction Assessment

Yes. A high-confidence static reproduction is available: configure `params.context1m: true` for direct Anthropic API-key traffic on an Opus/Sonnet 4.x model and the current wrapper/test path adds `context-1m-2025-08-07`; live API verification was not run.

## Solution Assessment

Yes. The best fix is a narrow Anthropic-provider and context-window update, not a global default context-token change; it should suppress only the retired 1M context beta for GA 1M models and preserve all unrelated provider behavior.
