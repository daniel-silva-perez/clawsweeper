## Summary

Keep this issue open. Current main has useful lower-level inter-session and agent-to-agent building blocks, but it does not implement the proposed teleconference lifecycle layer: named dial/pickup/send/wrap/list/show/transcript verbs, meeting IDs, durable pending/active/closed meeting state, visible meeting frames, or transcript artifacts. The existing maintainer comment also treats the proposal as useful future work above #63789, not as obsolete.

## Best Possible Solution

Keep the RFC open and steer it toward a narrow implementation PR: meeting id, dial/pickup/send/wrap/list/show/transcript commands, durable meeting state, visible channel frames, transcript artifacts, and focused state-transition/rendering tests. It should compose with #63789 or document a temporary bridge, while leaving external A2A interop, dashboards, autonomous dialing, and broad orchestration to separate work.
