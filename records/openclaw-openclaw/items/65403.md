## Summary

Keep open. Current main has adjacent agent CLI, delivery, subagent, and bounded `sessions_send` agent-to-agent exchange pieces, but it does not implement the requested scoped meeting lifecycle, named teleconference verbs, durable pending/active/closed meeting records, or transcript artifact. The maintainer discussion also narrowed the item toward a lifecycle layer above #63789 rather than closing it as already covered or moving it to Google Meet/browser voice.

## What This Changes

The issue requests a first-class agent teleconference primitive with dial, pickup, send, wrap, list, show, and transcript lifecycle verbs backed by durable meeting files and visible channel frames.

## Best Possible Solution

Land a small native lifecycle layer over the existing handoff/session primitives: meeting id, dial/pickup/send/wrap/list/show/transcript verbs, atomic state transitions, durable transcript output, visible channel frames, and focused tests/docs, while leaving broader collaboration phases to separate trackers.

## Reproduction Assessment

Not applicable as a feature/RFC rather than a bug report. The current-main gap is source-verifiable: the CLI and A2A/session code expose adjacent primitives but no meeting lifecycle surface.

## Solution Assessment

Yes for the maintainer-narrowed direction, not for the whole broad RFC at once. The maintainable path is a minimal lifecycle layer that composes with #63789 or documents a temporary fallback, with broader orchestration explicitly deferred.
