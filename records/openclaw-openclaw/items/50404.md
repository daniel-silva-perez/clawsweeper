## Summary

Keep open. Current main has useful Gateway session primitives, a Chat header session dropdown, and a separate Sessions management table, but it still does not provide the requested native Control UI conversation-history sidebar with search, inline rename, delete/archive actions, and ChatGPT-style switching. The New Session affordance still drives the `/new` reset path, while open PR #73725 only covers the narrower create-and-switch behavior.

## What This Changes

Request to add a Control UI left sidebar for saved chat sessions with persistent history, New Chat create/switch behavior, renaming, deletion/archive controls, search/filtering, and cross-refresh survival.

## Best Possible Solution

Keep a native Control UI conversation-management track open. Review and land or close #73725 for the immediate create-and-switch fix, then designate either this issue or #27526 as the canonical sidebar work. A complete implementation should reuse `sessions.list`, `sessions.create`, `sessions.patch`, and `sessions.delete`, expose derived titles/previews, define archive behavior before adding archive UI, and update Control UI docs once shipped.
