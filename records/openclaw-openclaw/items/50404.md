## Summary

Keep open. Current main has adjacent session-management pieces, including a Sessions tab, session selector, labels, delete, and a Gateway `sessions.create` API, but the requested Control UI workflow is not implemented: the Chat plus/New Session button still sends `/new` into the active session reset path instead of creating and switching to a persisted dashboard session. The linked member PR #73725 is an open implementation candidate for that narrow create/switch slice, not merged main behavior.

## Best Possible Solution

Keep this open. Review and either land or supersede #73725 for the narrow New Chat -> `sessions.create` -> switch-session behavior, then decide whether to extend the Chat surface with a true sidebar/dropdown that reuses `sessions.list`, `sessions.patch`, `sessions.delete`, and existing Sessions-page filtering. Archive still needs a real session-state/API design because current entries expose label/displayName but no archived-session contract.
