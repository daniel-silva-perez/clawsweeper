## Summary

Keep open. Current main has Gateway session primitives, a Chat header selector, and a separate Sessions table, but it still lacks the requested native ChatGPT-style Control UI sidebar with chronological conversation rows, active highlighting, in-chat search, inline rename/context actions, and clear archive semantics. The KeepMyClaw and Jupyter comments are useful workaround/context, while open PR #73725 only covers the narrower New Chat create-and-switch slice.

## What This Changes

Add a Control UI session sidebar for saved chat history, new-chat creation, renaming, switching, persistence, search, delete, and archive-style management.

## Best Possible Solution

Keep a native Control UI conversation-management track open. Review and land or close #73725 for the immediate New Chat create/switch behavior, then designate a canonical sidebar issue and implement the remaining sidebar/search/rename/delete/archive UX against the existing session APIs, with focused UI/controller tests and docs once the behavior ships.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The current gap is high-confidence from static inspection of current main: Chat renders a header select, the sidebar has only a Chat tab, and New Session still sends /new.

## Solution Assessment

Unclear as one bundled change. The maintainable path is to split the immediate create/switch fix from the larger sidebar/search/rename/delete/archive workflow, because backend primitives mostly exist but the UX and archive semantics still need maintainer scoping.
