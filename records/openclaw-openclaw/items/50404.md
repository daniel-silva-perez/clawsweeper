## Summary

Keep open. Current main has Gateway session primitives, a Chat header session dropdown, and a separate Sessions table, but it still does not provide the requested native Control UI conversation-history sidebar with chronological rows, active highlighting, inline rename/context actions, archive semantics, and ChatGPT-style switching. The related KeepMyClaw and Jupyter notebook comments are useful workarounds/context, and open PR #73725 covers only the narrower New Chat create-and-switch slice.

## What This Changes

The issue requests ChatGPT-style Control UI session management with a left conversation sidebar, saved switchable history, session titles/renaming, search/filtering, delete/archive actions, and persistence across refreshes/devices.

## Best Possible Solution

Keep a native Control UI conversation-management track open. Review and land or close #73725 for the immediate New Chat create/switch behavior, then designate a canonical sidebar issue and implement the remaining UX against existing sessions.list, sessions.create, sessions.patch, sessions.delete, and sessions.preview/derived-title primitives, with docs and focused Control UI tests once shipped.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The current gap is high-confidence from static inspection: Chat still renders a header select and separate Sessions table, and the New Session button still sends /new.

## Solution Assessment

Unclear as one all-in bundled change. The maintainable path is to split the immediate create/switch repair from the larger sidebar/search/rename/delete/archive UX, because the backend primitives mostly exist but archive semantics and canonical tracker choice still need maintainer scoping.
