## Summary

Keep open. Current main has Gateway session primitives, a Chat header session dropdown, and a separate Sessions table, but it still lacks the requested native Control UI conversation-history sidebar with search, inline rename/context actions, archive semantics, and ChatGPT-style switching. The New Session button still sends `/new`, and open PR #73725 covers only the narrower create-and-switch slice.

## What This Changes

Requested behavior is a native Control UI session sidebar for persisted chat history, switching, search/filtering, renaming, deletion/archive actions, and New Chat session creation.

## Best Possible Solution

Keep a native Control UI conversation-management track open. Review and land or close #73725 for the immediate create-and-switch fix, then designate either this issue or #27526 as the canonical sidebar work. A complete implementation should reuse `sessions.list`, `sessions.create`, `sessions.patch`, and `sessions.delete`, expose derived titles/previews, define archive behavior before adding archive UI, and update Control UI docs once shipped.
