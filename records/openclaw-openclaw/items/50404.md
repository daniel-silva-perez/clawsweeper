## Summary

Keep open. Current main has Gateway session primitives, a Chat session dropdown, and a separate Sessions management table, but it still does not implement the requested native Control UI conversation sidebar. The Chat New Session button still sends `/new` into the active session/reset path, and open PR #73725 only covers the narrower create-and-switch slice.

## What This Changes

Add a native Control UI session sidebar with persisted chat history, session creation/switching, rename/delete/archive management, search, and cross-refresh persistence.

## Best Possible Solution

Keep this issue open. Review and land or close #73725 for the immediate New Chat `sessions.create`/switch behavior, then decide whether this issue or #27526 is the canonical product track for a native Control UI conversation sidebar. A complete implementation should reuse `sessions.list`, `sessions.create`, `sessions.patch`, and `sessions.delete`, use derived titles/previews, define archive semantics before exposing archive UI, cover Chat/Sessions tests, and update the Control UI docs once shipped.
