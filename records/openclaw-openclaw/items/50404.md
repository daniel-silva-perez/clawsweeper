## Summary

Keep open. Current main still has only partial Control UI session management: Gateway session primitives, a chat-header session selector, a separate Sessions table, and a New session button that sends `/new` through the reset path. The requested native conversation-history sidebar with search, active highlighting, inline rename/context actions, archive semantics, and ChatGPT-style session switching is not implemented; related workarounds and open PR #73725 are useful but do not close the broader request.

## What This Changes

Add a native Control UI left sidebar for saved chat sessions, including create, switch, rename, delete/archive, search/filter, and persisted conversation history management.

## Best Possible Solution

Keep a native Control UI conversation-management track open. Review and either land or close #73725 for the immediate create-and-switch repair, then designate one canonical sidebar issue and implement the remaining UX against the existing Gateway session APIs, including derived titles/previews, search, active highlighting, inline label editing, delete behavior, a defined archive model if desired, mobile layout, and updated Control UI docs.
