## Summary

Keep open. Current main has session primitives, a Chat session dropdown, and a Sessions management table, but it still does not implement the requested native Control UI session-history sidebar and the New Session action still resets via `/new` instead of creating/switching to a persisted dashboard session.

## Best Possible Solution

Keep this issue open. Review and land or close #73725 for the immediate New Chat `sessions.create`/switch behavior, then decide whether this issue or #27526 should be the canonical product track for a native Control UI conversation sidebar. A complete implementation should reuse `sessions.list`, `sessions.create`, `sessions.patch`, `sessions.delete`, derived titles/previews, define archive semantics before showing archive UI, and update the Control UI docs once shipped.
