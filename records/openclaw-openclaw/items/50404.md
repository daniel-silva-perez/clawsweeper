## Summary

Keep open. Current main still has partial session management, but not the requested native Control UI conversation-history sidebar: Chat has a header session selector, the Sessions page is a separate table, and the New Session button still sends `/new` through the reset path. Related open PR #73725 covers only the narrower create-and-switch behavior, so the remaining sidebar/search/rename/archive UX still needs maintainer product direction.

## What This Changes

The issue requests a ChatGPT-style Control UI session sidebar for listing, creating, renaming, searching, switching, deleting or archiving, and persisting saved chat sessions.

## Best Possible Solution

Keep a focused native Control UI conversation-management track open. Review and land or close #73725 for the immediate New Chat create/switch repair, then define the sidebar UX for search, active highlighting, derived titles/previews, inline rename, delete/archive behavior, and mobile layout before implementing it against the existing Gateway session APIs and updating Control UI docs.
