## Summary

Keep open. Current main still has only partial session management for the Control UI: a chat header session selector, a separate Sessions table, and Gateway session primitives. It does not provide the requested native conversation-history sidebar workflow, and the visible New Session button still follows the `/new` reset path. The related KeepMyClaw and Jupyter workarounds are useful context, but they do not replace the native UI request; open PR #73725 covers only the narrower create-and-switch slice.

## What This Changes

Request a native Control UI session sidebar for saved chat history, New Chat persistence, session switching, search/filtering, rename/delete/archive actions, and restart-safe session persistence.

## Best Possible Solution

Keep a focused native Control UI conversation-management track open. Review and land or close #73725 for the immediate New Chat create/switch repair, then define the sidebar UX for search, active highlighting, derived titles/previews, inline rename, delete versus archive behavior, and mobile layout before implementing it against the existing Gateway session APIs and documenting the shipped behavior.
