## Summary

Keep open. Current main still has partial session-management support but not the requested native Control UI conversation-history sidebar: the Chat surface has a header select and the Sessions page has a separate management table, while the New Session affordance still sends `/new` into the current session reset path. The linked open #73725 PR is useful but covers only the narrower create-and-switch slice, and the remaining sidebar/search/rename/archive UX needs maintainer product direction.

## What This Changes

Request to add a Control UI session sidebar for saved conversation history, new-chat creation, renaming, deletion/archive actions, switching, search, and persistence.

## Best Possible Solution

Keep this as the focused native Control UI sidebar/session-history tracker. Review and land or close #73725 for the immediate New Chat create/switch behavior, then define the sidebar UX for search, rename, delete/archive, active highlighting, derived titles/previews, and mobile behavior before implementing it against the existing Gateway session APIs and updating the Control UI docs.
