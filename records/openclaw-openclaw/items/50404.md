## Summary

Keep open. Current main still has partial session management through Gateway primitives, a Chat header dropdown, and a separate Sessions table, but it does not provide the requested native conversation-history sidebar with search, active row highlighting, inline rename/context actions, archive semantics, and ChatGPT-style session switching. The KeepMyClaw and Jupyter comments are useful workaround/context, and open PR #73725 covers the narrower New Chat create/switch slice, but neither closes the broader Control UI request.

## What This Changes

Request a ChatGPT-style Control UI session sidebar for saved conversation history, new-session creation, switching, renaming, deletion/archive actions, search, and persistence.

## Best Possible Solution

Keep a native Control UI conversation-management track open. Review and land or close #73725 for the immediate New Chat create/switch fix, then designate a canonical sidebar tracker and implement the remaining UX against the existing session APIs, including derived titles/previews, search, active highlighting, inline label editing, delete behavior, a defined archive model if desired, mobile layout, and public Control UI docs once shipped.
