## Summary

Keep this PR open. Current main still lacks legacy iMessage typing-indicator wiring, and this PR is the active implementation attempt, but it needs maintainer review before merge because the branch has unresolved review defects and the upstream imsg v0.5.0 contract includes RPC typing methods that conflict with the PR's CLI-only rationale.

## What This Changes

The PR adds an iMessage typing helper that shells out to `imsg typing` and rewires the iMessage monitor from the plain reply dispatcher to the typing-aware dispatcher so inbound replies can send start/stop typing indicators.

## Best Possible Solution

Keep this PR open as the active legacy iMessage typing-indicator implementation candidate. The best path is maintainer review plus a rebase, then either use the existing long-lived imsg RPC client with `typing.start`/`typing.stop` or deliberately justify a serialized CLI fallback. The implementation should preserve `opts`/config precedence, remove the stale import, gate start/stop ordering, and add focused iMessage tests for target selection, typingMode lifecycle, graceful degradation, and concurrency.
