## Summary

Current main and v2026.4.26 already implement WebChat non-video file upload support: Control UI accepts the requested Office/PDF formats, sends them as file attachments, Gateway stores and stages them as managed media paths, and tests/docs/changelog cover the behavior. Related merged PR #72299 repaired #70947 and explicitly tracked this request.

## Best Possible Solution

Close as implemented and keep the shipped generic non-video WebChat attachment path. If a future gap appears, track it as a narrower document-extraction or format-specific analysis issue rather than reopening the upload plumbing request.
