## Summary

Keep this issue open. Current main still treats non-zero poll duration fields as poll creation intent, so a default-filled `pollDurationHours: 24` on `action: "send"` still trips the send-action poll guard. The workspace media-root portion appears partially addressed, and the shared message tool no longer appears to expose `components` in the scoped schema test, but the underlying Discord component parser still rejects all-empty modal defaults if such an object reaches it.

## Best Possible Solution

Keep this issue open and fix the message tool boundary to normalize semantic no-op defaults before validation and routing. At minimum, add regression coverage for `action: "send"` with empty `pollQuestion`, `pollDurationHours: 24`, `pollMulti: false`, and an all-empty Discord components object, then make poll intent depend on actual poll content rather than duration defaults alone. Keep the workspace media-root behavior as the partial fix for the media path portion.
