## Summary

Keep this issue open. Current main appears to have partially addressed workspace media roots, but the central GPT-5.4 default-parameter problem is still present for poll detection and Discord component parsing: `pollDurationHours: 24` is still treated as poll intent for `action: send`, and an all-empty `components.modal.fields: []` object still throws instead of being treated like an omitted components payload.

## Best Possible Solution

Keep the issue open and fix the message tool boundary to normalize semantic no-op defaults before validation/routing. At minimum, add regression tests for `action: "send"` with empty `pollQuestion`, `pollDurationHours: 24`, `pollMulti: false`, and all-empty Discord `components`, then update poll detection and `readDiscordComponentSpec` so filled model defaults behave like omitted optional parameters.

Related issue #52118 only covered zero-valued poll defaults, not the non-zero default duration or empty components case.
