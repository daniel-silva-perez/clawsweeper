## Summary

Keep open. Current main still routes builtin memory through `node:sqlite` and the sqlite-vec loadable-extension path, with fallback behavior when loading fails, but I did not find a macOS repair, alternate SQLite binding, or supported documented workaround that resolves the reported native sqlite-vec load failure.

## What This Changes

The issue requests a macOS fix or supported workaround for native sqlite-vec loading when `node:sqlite` cannot enable SQLite loadable extensions, causing degraded vector similarity search.

## Best Possible Solution

Keep this issue as the canonical macOS sqlite-vec/node:sqlite tracker until maintainers either validate and document a supported macOS Node/runtime combination where builtin sqlite-vec loads successfully, or change builtin memory to use a SQLite loading strategy that supports sqlite-vec on macOS. If native acceleration remains unsupported, document the macOS limitation and the Homebrew/QMD workaround while keeping the in-process cosine path described as degraded fallback behavior.
