## Summary

Keep open. Current main still routes builtin memory through `node:sqlite` plus the sqlite-vec loadable-extension path, then degrades when loading fails; I did not find a macOS-specific native load fix, alternate SQLite binding, or supported documented workaround. The closed unmerged #74435 is related but narrower and does not supersede this tracker.

## What This Changes

The issue requests that sqlite-vec vector search work on macOS under OpenClaw builtin memory, or that OpenClaw provide a supported workaround or documented limitation instead of degrading recall.

## Best Possible Solution

Keep this as the canonical macOS sqlite-vec/node:sqlite tracker until maintainers choose and validate a supported path: a Node/SQLite runtime that can load sqlite-vec on macOS, an alternate SQLite binding/loading strategy, or explicit docs for the macOS limitation and supported workaround while preserving degraded fallback behavior.
