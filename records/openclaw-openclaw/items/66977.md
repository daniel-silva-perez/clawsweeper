## Summary

Keep open. Current main still uses the builtin `node:sqlite` runtime plus sqlite-vec loadable-extension APIs for builtin memory, then records degraded vector state when loading fails. The related macOS report #74401 is closed and #74435 was closed unmerged, so neither supersedes this canonical macOS runtime/packaging question.

## What This Changes

Request to make sqlite-vec native vector acceleration load on macOS despite Node SQLite extension-loading limits, or provide a supported workaround/documented limitation.

## Best Possible Solution

Keep this as the canonical macOS sqlite-vec/node:sqlite tracker until maintainers choose a validated path: ship/use a macOS SQLite runtime that supports sqlite-vec extension loading, add an alternate loading strategy with clear package tradeoffs, or explicitly document the macOS limitation and supported workaround while preserving degraded fallback behavior.
