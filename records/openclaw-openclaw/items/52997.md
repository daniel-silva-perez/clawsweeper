## Summary

Keep open. Current main still intentionally isolates Telegram forum topics and DM threads into topic/thread-specific session keys, while only preserving topic/thread metadata for prompt/reply routing. There is no Telegram config schema/type/docs/test coverage for an opt-in shared session mode, and related reports cover narrower topic targeting or topic-label display rather than this shared-context behavior.

## Best Possible Solution

Keep this open for a Telegram-plugin-owned design. If maintainers accept the product behavior, add an explicit opt-in setting for forum groups and DM topics, preserve topic/thread metadata and reply delivery targets, define how shared group sessions interact with sandbox/session-tool visibility, and add regression tests plus docs for forum topics and DM threads.
