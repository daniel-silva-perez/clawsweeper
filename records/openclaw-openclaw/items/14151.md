## Summary

Keep open. Current main still handles model-emitted reasoning tags, malformed local-model tags, reasoning visibility, and final-tag enforcement, but it does not expose or thread a default-off option to force-prepend an opening `<think>` tag for streaming or non-streaming assistant output. The related implementation PR #14323 is closed unmerged and remains useful context rather than a landed fix.

## Best Possible Solution

Keep this issue open and implement a default-off, typed `forcePrependThinkTag`-style option with clear global and per-model precedence. The implementation should resolve config before the run, thread the flag through runner params into `subscribeEmbeddedPiSession`, inject exactly once before reasoning parsing for both streaming and non-streaming assistant text, guard existing/native tags, and update schema/docs/tests. Use #14323 as context, but do not treat it as landed.
