## Summary

Keep open. Current main still lacks the requested default-off synthetic `<think>` prepend option across config, runner/subscriber parameters, docs, and the message parsing path. The related implementation PR is closed unmerged and the current code only parses or strips tags that the model already emitted.

## What This Changes

The issue requests a default-off option to prepend an opening `<think>` tag, optionally per model or provider, so local reasoning models that omit native think delimiters can still be parsed consistently.

## Best Possible Solution

Add a default-off documented config contract for synthetic think-tag prepending, resolve it before embedded Pi runs, thread it through runner and subscriber params, and apply it before existing reasoning parsing with native-tag detection.
