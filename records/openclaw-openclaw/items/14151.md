## Summary

Current main still does not implement the requested default-off option to force-prepend an opening `<think>` tag. The existing code strips/parses model-emitted reasoning tags and handles malformed tags, but there is no config key, schema support, runner parameter, or subscribe-level flag for injecting a tag before parsing; related PR #14323 was closed unmerged.

## Best Possible Solution

Keep this issue open and implement a default-off, typed force-prepend think-tag option with clear global/per-model resolution, config schema/docs alignment, runner-to-subscribe threading, exactly-once injection before reasoning parsing, and guards for existing tags/native reasoning/final enforcement. PR #14323 is useful context but should not be treated as landed.

## Clownfish Work Prompt

Implement the open force-prepend think-tag feature for local reasoning models. Add a default-off typed option, preferably `forcePrependThinkTag`, with global `agents.defaults.forcePrependThinkTag` support and per-model `agents.defaults.models[modelRef].forcePrependThinkTag` override. Keep the behavior provider/model agnostic: do not hard-code GLM, MiniMax, StepFun, Ollama, or other IDs in core. Resolve per-model over global, thread the resolved boolean through auto-reply/embedded runner params into `subscribeEmbeddedPiSession`, and inject one opening `<think>` before the first assistant text delta or non-streaming assistant text before existing reasoning parsing/stripping runs. Guard against double-prepending when the assistant output already begins with a reasoning tag such as `<think>`, `<thinking>`, `<thought>`, or native phase-aware reasoning/final blocks. Preserve default behavior when the option is unset or false, and keep `enforceFinalTag` behavior intact. Update config types, Zod schema, generated/help docs as required by the repo, docs for thinking/config, and add a user-facing changelog entry thanking `@LisaMacintosh`. Use PR #14323 only as context; do not copy over its missing wiring or double-tag behavior.
