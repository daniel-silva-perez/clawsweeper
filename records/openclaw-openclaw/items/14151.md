## Summary

Keep open. Current main still lacks the requested default-off synthetic `<think>` prepend option across the typed config, strict schema, docs, runner/subscriber params, and message parsing path. The only linked implementation PR is closed unmerged, and current code only extracts or strips tags the model already emitted.

## What This Changes

The issue requests a user-configurable option to inject an opening `<think>` tag for local reasoning models before reasoning parsing, with streaming and non-streaming support and default behavior left off.

## Best Possible Solution

Add a documented, default-off config contract for synthetic think-tag prepending, resolve it before embedded Pi runs, thread it through runner and subscriber params, and apply it before existing reasoning extraction with native opening-tag detection.
