## Summary

Keep open. Current main still lacks the requested default-off force-prepend `<think>` option: there is no typed config/schema key, no runner or subscriber parameter, and the message path only parses or strips tags that are already present. The related implementation PR #14323 is closed unmerged, so there is no landed fix to close against.

## What This Changes

The issue requests a user-configurable option to prepend an opening `<think>` tag, and optionally ensure closure, for local reasoning models before reasoning parsing in streaming and non-streaming assistant output.

## Best Possible Solution

Add a default-off, documented config contract for synthetic think-tag prepending, resolve global and per-model settings before embedded Pi runs, thread the value through runner and subscriber params, and apply it once before existing reasoning parsing with native-tag detection.

## Repair Work Prompt

Implement a default-off force-prepend opening `<think>` tag option for local/tagged reasoning models. Add a typed config key, preferably `forcePrependThinkTag`, with global `agents.defaults.forcePrependThinkTag?: boolean` and per-model `agents.defaults.models[modelRef].forcePrependThinkTag?: boolean` support, then update Zod schema, generated config docs/schema, and user docs. Resolve the effective value in the auto-reply/embedded runner path before Pi execution, pass it through `RunEmbeddedPiAgentParams`, `EmbeddedRunAttemptParams`, and `SubscribeEmbeddedPiSessionParams`, and apply it in the embedded subscriber before existing reasoning extraction/stripping. The behavior must be default off, apply exactly once per assistant message, work for streaming deltas and final/non-streaming assistant text, and avoid double-prepending when the model already emits an opening `<think>` tag. Do not change default reasoning visibility, provider reasoning-output mode semantics, or `enforceFinalTag` behavior beyond the new opt-in path. Include focused tests for schema acceptance/rejection, runner param threading, streaming prepend, final-message handling, native-tag guard, and reset between assistant messages. Changelog: add a user-facing fix/change entry with reporter credit, not bot credit.
