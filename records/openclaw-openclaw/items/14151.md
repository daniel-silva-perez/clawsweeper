## Summary

Keep open. Current main still lacks a configurable force-prepend `<think>` option; the reasoning path parses or strips tags that the model already emitted, and related PR #14323 was closed unmerged with missing wiring and double-tag concerns.

Keep this issue open and implement a default-off, typed force-prepend think-tag option with clear global/per-model resolution, no hard-coded provider IDs, runner-to-subscribe threading, exactly-once injection before reasoning parsing, guards for existing tags/native reasoning/final enforcement, docs, changelog, and focused streaming/non-streaming tests. PR #14323 is useful context but should not be treated as a landed fix.
