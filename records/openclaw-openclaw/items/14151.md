## What This Changes

Requests a user-configurable option to prepend an opening `<think>` tag, optionally scoped per model/provider, so local reasoning models that omit tags can still be parsed consistently.

## Best Possible Solution

Add a documented, default-off config contract for synthetic think-tag prepending, resolve it before embedded Pi runs, pass it through runner and subscriber params, and apply it before existing reasoning extraction with an exactly-once state and native-tag guard.

## Repair Work Prompt

Implement the requested default-off synthetic `<think>` opening-tag prepend support for local reasoning models. Use this issue as the canonical request and use https://github.com/openclaw/openclaw/pull/14323 only as context, not as code to copy blindly.

Expected boundary:
- Add a documented config contract, likely global `agents.defaults.forcePrependThinkTag?: boolean` plus per-model override under `agents.defaults.models[model].forcePrependThinkTag?: boolean`, defaulting to false.
- Update config types, strict Zod schema, schema help/labels/generated docs as needed, and public docs.
- Resolve the effective value before embedded Pi runs, pass it through `RunEmbeddedPiAgentParams`, attempt params, subscription params, and subscriber state.
- In the message parsing path, synthesize one opening `<think>` before existing reasoning extraction/stripping when enabled, but do not double-prepend if the model already begins with a native opening think tag. Handle streaming first chunks and final/non-streaming text paths.
- Preserve existing `enforceFinalTag`, reasoning visibility, block streaming, reply directive parsing, and malformed-tag recovery behavior.
- Add focused tests for config acceptance/defaults, runner/subscriber wiring, streaming injection, non-streaming/final text handling, and native-tag guard.
- Add a user-facing changelog entry with `Thanks @LisaMacintosh`; do not change default behavior for users who do not opt in.

Do not introduce provider-specific defaults or bundled model IDs in core; keep the option generic and default off.
