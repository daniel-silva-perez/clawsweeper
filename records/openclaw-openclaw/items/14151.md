## Summary

Keep open. Current main still lacks the requested default-off synthetic opening `<think>` option in the typed agent config, strict schema, embedded runner parameters, subscriber parameters, and message parsing path; the directly related implementation PR is closed unmerged, so there is no landed fix to close against.

## What This Changes

The issue requests a user-configurable option to prepend an opening `<think>` tag for local reasoning models that do not reliably emit reasoning delimiters.

## Best Possible Solution

Add a documented, default-off config contract for synthetic think-tag prepending, resolve it before embedded Pi runs, pass it through runner and subscriber params, and apply it before existing reasoning extraction with exactly-once state and a native opening-tag guard.

## Reproduction Assessment

Yes for the feature gap: current main can be checked statically because the proposed config key is absent from the exported types and strict Zod schema, and there is no runner/subscriber/message-handler parameter to exercise. No live GLM or MiniMax model is needed to reproduce the missing OpenClaw control surface.

## Solution Assessment

Yes, the narrow maintainable path is a default-off option threaded through the existing agent defaults, runner, and subscriber parsing path. A prompt-template-only workaround would not satisfy the requested frontend/orchestrator boundary and would remain model-specific.
