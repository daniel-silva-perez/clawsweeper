## Summary

Keep open. Current main still lacks the requested default-off synthetic `<think>` opening-tag option in agent defaults, strict config validation, embedded runner parameters, subscriber parameters, and the streaming/final message parsing path; the directly related implementation PR was closed unmerged.

## What This Changes

Request a user-configurable, default-off option to prepend an opening `<think>` tag for local reasoning models that omit reasoning delimiters, with streaming and non-streaming support.

## Best Possible Solution

Add a documented, default-off config contract for synthetic think-tag prepending, resolve it before embedded Pi runs, pass it through runner and subscriber params, and apply it before existing reasoning extraction with exactly-once state plus a guard for native opening tags.

## Reproduction Assessment

Yes for the feature gap: current main can be checked statically because the proposed config key is absent from exported types and strict Zod schema, and no runner/subscriber/message-handler parameter exists to exercise. No live GLM, MiniMax, or step model is required to reproduce the missing OpenClaw control surface.

## Solution Assessment

Yes, the narrow maintainable solution is a default-off option threaded through agent defaults, embedded runner, and subscriber parsing. Prompt-template-only workarounds would remain model-specific and would not satisfy the requested frontend/orchestrator boundary.
