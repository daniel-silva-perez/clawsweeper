## Summary

Keep open. Current main still lacks the requested default-off synthetic opening `<think>` option in the typed agent defaults, strict config schema, embedded runner parameters, subscriber parameters, and streaming message path; the directly related implementation PR is closed unmerged.

## What This Changes

The issue requests a configurable way to prepend an opening `<think>` tag, optionally per model or provider, before OpenClaw parses local reasoning-model output.

## Best Possible Solution

Add a documented, default-off config contract for synthetic think-tag prepending, resolve it before embedded Pi runs, pass it through runner and subscriber params, and apply it before existing reasoning extraction with exactly-once state plus a native opening-tag guard.

## Reproduction Assessment

Yes for the feature gap: current main can be checked statically because the proposed config key is absent from exported types and the strict Zod schema, and no runner/subscriber/message-handler parameter exists to exercise. No live GLM, MiniMax, or step model is needed to reproduce the missing OpenClaw control surface.

## Solution Assessment

Yes, the best maintainable direction is a default-off option threaded through the existing agent defaults, runner, and subscriber parsing path. Prompt-template-only workarounds would not satisfy the requested frontend/orchestrator boundary and would remain model-specific.
