## Summary

Keep this issue open. Current main still routes Feishu setup status through `probeFeishu`, and that probe still constructs the normal `@larksuiteoapi/node-sdk` HTTP client without a quiet/custom logger option, while the SDK defaults the client logger to info and logs `client ready`. The linked same-author PR #67734 is still the open implementation candidate and uses closing syntax, so the issue should wait for that PR to be reviewed, landed, or closed.

## What This Changes

The issue asks to suppress Feishu SDK info logs during setup/status probing while preserving user-facing Feishu connection status lines.

## Best Possible Solution

Land a Feishu-owned fix that creates a quiet/custom SDK logger path for setup/status probes, avoids leaking SDK info output into the wizard, preserves `Feishu: connected as ...`, and adds focused regression coverage in Feishu client/probe/setup-surface tests.
