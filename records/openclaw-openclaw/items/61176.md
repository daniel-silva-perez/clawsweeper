## Summary

Keep open. Current main does not already contain the proposed session-sidebar feature, and this PR is still an implementation candidate, but the patch is not mergeable as-is: diff inspection and existing review comments show the preserved /new history row can point at a transcript path that reset later archives, the Ctrl/Cmd+` shortcut double-toggles, and closing the session list collapses existing tool-output sidebar state.

## What This Changes

The PR adds a Control UI chat session sidebar with context-meter, New Chat, and recent-history UI, changes sessions.reset to preserve /new history entries with previousSessionKey, copies session-key helpers into the UI, and adds a Vite build stub for server-only modules.

## Best Possible Solution

Keep the PR open for revision or maintainer replacement: implement the session sidebar on current main, preserve /new history with the archived transcript path that can actually be reopened, make the sidebar shortcut single-owner or stop propagation, decouple session-sidebar state from tool-output sidebar state, and add focused UI/gateway tests for those behaviors.

## Reproduction Assessment

Yes for the blocking PR bugs by code inspection. Pressing Ctrl/Cmd+` inside the chat path would run the chat handler and then bubble to the document handler, and a /new reset with preserveHistory stores the old row before the transcript archive rename.

## Solution Assessment

No. The requested UX direction may be useful, but this implementation is not the narrowest maintainable fix until history preservation uses the stable archived transcript path, keyboard ownership is cleaned up, sidebar state is decoupled, and the branch is rebased onto current main's helper APIs.
