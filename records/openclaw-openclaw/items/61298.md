## Summary

Keep open. Current main still lacks the PR's central Android assistant launch hardening, but the PR is not merge-ready because it is stale against the shipped Android security change that keeps `ASK_OPENCLAW` prompts draft-only instead of auto-sending them.

## What This Changes

The PR adds bounded Android assistant prompt sanitization, assistant launch fingerprint/state restoration, `SavedStateHandle`-backed launch routing state, and unit coverage for those paths.

## Best Possible Solution

Keep the PR open and update it on current main: preserve `ASK_OPENCLAW` as `autoSend = false`, apply the prompt length cap to the draft path, pass restored launch fingerprints explicitly into `handleAssistantIntent`, persist launch routing state through `SavedStateHandle`, and update tests for draft-only app actions plus restoration behavior.

## Reproduction Assessment

Yes. The issue has a high-confidence source/unit reproduction path: current main lacks the sanitizer, saved-state fingerprinting, and `SavedStateHandle` state, while the PR-provided tests exercise those intended behaviors.

## Solution Assessment

No. The hardening direction is appropriate, but this PR is not the best mergeable solution as-is because it conflicts with the shipped draft-only security behavior; the safer solution is a rebase/update that keeps external prompts as drafts while adding the lifecycle hardening.
