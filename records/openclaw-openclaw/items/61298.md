## Summary

Keep open. Current main does not implement the PR's main hardening: App Action prompts are still only trimmed, assistant launch handling is not fingerprinted/restored through saved instance state, and MainViewModel launch routing state is not SavedStateHandle-backed. The PR still contains useful work, but it needs a rebase/update because current main has since shipped the #70714 Android security change that keeps ASK_OPENCLAW prompts as draft-only instead of auto-sending them.

## Best Possible Solution

Keep the PR open as useful but require an update on current main: preserve ASK_OPENCLAW autoSend=false, apply bounded prompt sanitization to the draft path, add restored-launch fingerprinting without a transient instance-field contract, persist launch routing state through SavedStateHandle, update tests to assert draft-only app actions plus restoration behavior, then run the targeted Android assistant launch tests and Kotlin compile lane.
