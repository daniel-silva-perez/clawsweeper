## Summary

Current main still defaults the Control UI Usage range to today only. The PR remains a focused candidate fix: `usageStartDate` and `usageEndDate` are both initialized from a fresh `new Date()` on main, and the usage loader sends those state values directly to `sessions.usage` and `usage.cost`. The existing Last 7d preset is user-triggered and does not change the initial default.

## Best Possible Solution

Keep the PR open and review or land the focused `ui/src/ui/app.ts` initializer change, ideally with a small regression test or documented manual check that a fresh Control UI Usage tab defaults to today minus 6 days through today.
