## Summary

Keep this PR open. Current main still initializes the Control UI Usage start and end dates to today, while the PR is a focused and correct fix to make the default range the last seven calendar days; the only meaningful gap is missing automated coverage for that initializer.

## What This Changes

The PR changes `ui/src/ui/app.ts` so `usageStartDate` initializes to today minus six days while `usageEndDate` remains today.

## Best Possible Solution

Land the focused initializer change in the Control UI, ideally with a small regression test or explicit maintainer manual check that a fresh Usage tab defaults to today minus six days through today.

## Reproduction Assessment

Yes. On current main, reading a fresh `OpenClawApp` state shows both `usageStartDate` and `usageEndDate` initialized from today's date, and `loadUsage` sends those state values directly to the gateway.

## Solution Assessment

Yes. The PR changes the narrow owner surface that controls the default query range and matches the existing inclusive `Last 7d` preset behavior; adding coverage would reduce future regression risk but is not required to make the approach sound.
