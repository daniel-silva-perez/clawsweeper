## Summary

Keep this issue open. Current main has meaningful partial hardening for direct completion fallback, but it still lacks the durable completion spool/inbox requested here, still has terminal retry-limit/expiry cleanup, and still prunes missing-session restored/orphaned runs before any pending completion payload could survive.

## Best Possible Solution

Keep this issue as the planning/canonical lifecycle thread until OpenClaw has a persisted completion spool/inbox on SubagentRunRecord, creates the spool before cleanup retry/give-up branches, drives task delivery status from spool state, retries pending spooled completions after restart/drain, preserves pending completions through orphan reconciliation, and updates the sub-agent docs and targeted restart/orphan tests. Review #65037 as partial retry-state hardening, but do not treat it as closing the broader durable completion delivery request unless it is expanded to cover those acceptance criteria.
