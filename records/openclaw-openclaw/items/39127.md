## Summary

Keep open. Current main still lacks the requested Gateway-owned per-session activity RPCs and WebSocket statechange contract. Existing session rows and events expose adjacent lifecycle metadata, but not the requested `sessions.activity.get/list`, `runtimeActivity`, explicit attention states, stale-run handling, monotonic activity versioning, or bounded child rollups. Related PRs #39438 and #41399 are useful prior art, but both are closed and unmerged.

## Best Possible Solution

Keep this open as the canonical core Gateway API feature. The best implementation path is an additive `sessions.activity.get`, `sessions.activity.list`, and `sessions.activity.statechange` contract with exported schemas/types, read-scope authorization, run-id based activity tracking, stale-run cleanup, explicit attention transitions, bounded cross-agent child rollups, reconnect resync semantics, public protocol docs, and focused Gateway tests. Use #39438 and #41399 as prior art, not as shipped or superseding work.
