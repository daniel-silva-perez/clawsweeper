## Summary

Keep open. Current main still lacks the requested gateway-owned per-session activity RPCs and WebSocket statechange contract. Existing `sessions.list` rows, `sessions.changed` broadcasts, and channel/account `busy` fields are adjacent but do not provide the requested `runtimeActivity`, explicit `attention`, monotonic versioning, stale-run handling, or bounded child rollup API. Related PRs #39438 and #41399 are useful prior art, but both are closed and unmerged.

## Best Possible Solution

Keep this open as the canonical core Gateway API feature. The best implementation path is an additive `sessions.activity.get`, `sessions.activity.list`, and `sessions.activity.statechange` contract with exported schemas/types, read-scope authorization, run-id based activity tracking, stale-run cleanup, explicit attention transitions (`awaiting_user`, `awaiting_subagent`, approvals/blocked/paused as appropriate), bounded cross-agent child rollups, reconnect resync semantics, public protocol docs, and focused Gateway tests. Use #39438 and #41399 as prior art, not as shipped or superseding work.
