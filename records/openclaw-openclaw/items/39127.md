## Summary

Current main still does not implement the requested Gateway-owned per-session activity API or WebSocket statechange contract. The existing code has adjacent session lifecycle events and channel/account busy fields, but no `sessions.activity.get/list`, no `sessions.activity.statechange`, no activity schemas, no explicit attention state, no stale-run counter, and no bounded child rollup. Related PRs #39438 and #41399 are useful prior art, but both are closed and unmerged.

## Best Possible Solution

Keep this open as the canonical core Gateway API feature. The best implementation path is an additive `sessions.activity.get`, `sessions.activity.list`, and `sessions.activity.statechange` contract with exported schemas/types, `operator.read` authorization, run-id-set based activity tracking, stale-run cleanup, explicit attention transitions, bounded cross-agent child rollups, reconnect resync semantics, public Gateway protocol docs, and focused Gateway tests. Use #39438 and #41399 as prior art, not as shipped or superseding work.
