## Summary

Keep open. Current main still lacks a public `sessions_spawn.lane` parameter, native and ACP spawned runs still dispatch through the fixed `subagent` lane, and config/docs/tests still describe one global subagent concurrency cap. Related lane/concurrency items exist, but none appears to implement or supersede custom `sessions_spawn` lanes.

## Best Possible Solution

Keep open until OpenClaw adds an additive, backward-compatible `sessions_spawn.lane` API that defaults to `subagent`, validates custom lane names, forwards the resolved lane through both native and ACP spawn paths, supports per-lane subagent concurrency configuration and reload, preserves the existing global `agents.defaults.subagents.maxConcurrent` behavior for the default lane, and updates docs plus regression tests. Coordinate the config model with the adjacent lane/concurrency PRs before implementation.
