## Summary

Keep this issue open. Current main still does not implement multi-lane `sessions_spawn`: the tool schema has no `lane` parameter, neither native nor ACP spawn params carry a lane, both runtime paths dispatch child `agent` runs on the fixed `subagent` lane, and config/docs/tests still model sub-agent concurrency as one global `agents.defaults.subagents.maxConcurrent` scalar. The existing maintainer comment already reached the same conclusion and remains accurate on current main.

## Best Possible Solution

Keep this open until OpenClaw adds an additive `sessions_spawn.lane` parameter that defaults to `subagent`, validates lane names, forwards the resolved lane through both native and ACP spawn paths, supports maintainer-approved per-lane subagent concurrency while preserving `agents.defaults.subagents.maxConcurrent`, and updates docs plus regression tests.
