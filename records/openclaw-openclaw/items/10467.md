## Summary

Keep open. Current main still does not expose a `sessions_spawn.lane` parameter, and both native sub-agent and ACP spawn paths still dispatch the child `agent` run on the fixed `subagent` lane. The config/docs/tests also still model sub-agent concurrency as one global `agents.defaults.subagents.maxConcurrent` scalar, so the requested multi-lane sub-agent support remains unimplemented.

## Best Possible Solution

Keep this issue open until OpenClaw adds an additive `sessions_spawn.lane` API that defaults to `subagent`, validates custom lane names, forwards the resolved lane through both native and ACP spawn paths, supports per-lane subagent concurrency configuration and reload, preserves the existing `agents.defaults.subagents.maxConcurrent` behavior for the default lane, and updates docs plus regression tests. Coordinate the config model with the adjacent lane/concurrency work before implementation.
