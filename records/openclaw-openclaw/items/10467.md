## Summary

Keep open. Current main still lacks the requested caller-selectable `sessions_spawn` lane: the tool schema has no `lane` input, both native and ACP spawn paths dispatch on the fixed `subagent` lane, and config/docs still expose one scalar subagent concurrency cap. The related open per-agent lane PR is adjacent infrastructure, not a replacement for this sub-agent API gap.

## What This Changes

Request adds an optional `lane` parameter to `sessions_spawn` with per-lane subagent concurrency while preserving the existing `subagent` default.

## Best Possible Solution

Add an additive `sessions_spawn.lane` API that validates a bounded lane name, defaults omitted calls to `subagent`, threads the resolved lane through both native and ACP child dispatch, applies compatible per-lane subagent concurrency config, and updates docs plus regression tests without changing existing callers.
