## Summary

Keep open. Current main still exposes `sessions_spawn` without a caller-selectable lane, and both native and ACP spawn paths hard-code the shared `subagent` queue lane while config/docs remain scalar. The adjacent per-agent lane work is not a replacement because it targets inbound agent runs rather than sub-agent lanes spawned through this tool.

## What This Changes

The issue requests an optional `sessions_spawn.lane` parameter with per-lane subagent concurrency configuration and a default fallback to the existing `subagent` lane.

## Best Possible Solution

Add a backward-compatible `sessions_spawn.lane` API that validates a bounded lane name, defaults omitted calls to `subagent`, threads the resolved lane through native and ACP child dispatch, applies compatible per-lane subagent concurrency config, and updates docs plus regression tests while preserving existing callers.
