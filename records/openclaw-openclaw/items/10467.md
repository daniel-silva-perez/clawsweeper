## Summary

Current main still does not implement multi-lane `sessions_spawn`. The tool schema has no `lane` parameter, native and ACP spawn paths still dispatch child `agent` runs on the fixed `subagent` lane, and config/docs/tests still describe one global `agents.defaults.subagents.maxConcurrent` limit. The prior maintainer-side Codex review in the issue remains accurate on current main.

## Best Possible Solution

Keep this issue open until OpenClaw adds an additive `sessions_spawn.lane` parameter that defaults to `subagent`, validates lane names, forwards the resolved lane through native and ACP spawn paths, applies per-lane subagent concurrency while preserving the existing scalar default, and updates config schema, docs, and regression tests.
