## Summary

Keep this issue open. Current main still has the reported `/agent` read-guard gap when the sandboxed read tool is wrapped by the workspace-root guard: the guard maps only `containerWorkdir`, the read-tool wiring passes only that option, and current regression coverage still expects `/agent/secret.txt` to fail with `Path escapes sandbox root`. The related fix PRs #39571 and #41033 were closed unmerged, while merged PR #44307 explicitly scoped out `/agent` read-guard remapping.

## Best Possible Solution

Keep the issue open. The best fix belongs in the sandboxed file-tool guard/wiring: validate through the sandbox mount table, or pass an explicit `/agent` mapping only for the `workspaceAccess: "ro"` agent workspace mount. Add regression coverage for both the direct guard and `createOpenClawCodingTools` proving `/agent/...` reads succeed while write/edit/apply_patch remain blocked. Treat `/tmp` reads as a separate policy decision.
