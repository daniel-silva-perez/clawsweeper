## Summary

Keep open. Current main still has the reported authorization-policy gap in the inline skill tool-dispatch path, and the linked fix PR #75125 is open and unmerged, so this issue should remain open until that PR or an equivalent reviewed fix lands.

## What This Changes

Make `command-dispatch: tool` skill slash commands resolve tools through the session-effective tool policy instead of constructing a raw OpenClaw tool set.

## Best Possible Solution

Review and land #75125, or an equivalent narrow patch, so inline tool-dispatched skill commands use the same session-effective tool surface as normal agent turns and add regression coverage for restricted profiles and leaf subagent sessions.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: configure a restricted session or leaf subagent, add a user-invocable skill with `command-dispatch: tool` for a restricted tool such as `sessions_spawn`, and observe that current main resolves it through raw `createOpenClawTools(...)` plus owner-only filtering.

## Solution Assessment

No for current main. The maintainable fix is to reuse the existing effective policy resolver used by normal turns and `/tools`; the linked PR appears to target that boundary, but it is not merged yet.
