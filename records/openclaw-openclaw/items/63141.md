## Summary

Current main still matches the reported behavior: parent thread/session forking uses a static 100000-token default, oversized parents are skipped with a log warning, and the child starts fresh. The related compaction-provider work in #56224 and checkpoint work in #62146 are present, but they are not wired into the parent-fork overflow branch.

## Best Possible Solution

Keep this issue open. The fix belongs in the session owner path, likely src/auto-reply/reply/session.ts and src/auto-reply/reply/session-fork.ts: compact/checkpoint the oversized parent before forking, emit a one-time visible/operator warning when inheritance is skipped, or resolve a model-aware cap from existing context-window metadata. Update focused session tests and the public session/config docs for the chosen behavior, reusing the shipped compaction provider and checkpoint infrastructure where appropriate.
