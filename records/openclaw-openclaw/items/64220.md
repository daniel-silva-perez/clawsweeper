## Summary

Keep this issue open. Current main still has the reported stale Control UI status gap: active/retrying compaction state is not age-bounded, terminal/reconnect/clear-history paths clear run or stream state without clearing compaction/fallback status, and the existing tests cover only partial lifecycle cleanup rather than a missed compaction-end event after the reply is done.

## What This Changes

The issue requests Control UI cleanup so `Compacting context...` cannot remain visible after the current session/run is no longer compacting.

## Best Possible Solution

Control UI should own a timer-aware, session/run-scoped cleanup path for compaction and fallback indicators: active/retrying compaction should get a stale watchdog, terminal chat/reconnect/clear-history/session or agent switches/send-failure paths should call the same cleanup, and regression tests should cover missed compaction-end after final reply plus reconnect and clear-history recovery.
