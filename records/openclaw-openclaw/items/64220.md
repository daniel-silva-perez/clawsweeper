## Summary

Keep this issue open. Current main has partially improved the originally proposed cleanup because session switches now clear compaction and fallback status, but the core stale-state bug remains: active/retrying compaction has no stale watchdog, and terminal chat, reconnect, and clear-history paths still clear run/stream state without clearing the compaction indicator.

## What This Changes

The issue asks for Control UI compaction and fallback indicators to clear when the current session/run is no longer compacting, including missed compaction end events and UI transition paths.

## Best Possible Solution

Add a small, shared Control UI cleanup/watchdog path for compaction and fallback indicators: active/retrying compaction should be age-bounded and session/run-scoped, terminal chat/reconnect/clear-history/send-failure paths should clear the same transient status, and regression tests should cover a missed compaction end after final reply plus reconnect and clear-history recovery.
