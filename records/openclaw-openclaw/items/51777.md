## Summary

Keep this PR open. Current main still has the Feishu explicit-default persistence gap, and the PR addresses a real onboarding bug, but the patch is stale because the DM policy prompt logic moved out of `src/commands/onboard-channels.ts` into `src/flows/channel-setup.prompts.ts`. The prior review discussion is useful: the author acknowledged the Twitch setter side effect, so a focused Feishu fix or explicit presence seam is safer than a blanket unconditional setter.

## Best Possible Solution

Keep the PR open for maintainer review. Rebase or rewrite it against `src/flows/channel-setup.prompts.ts`, either as a focused Feishu missing-key write or as a small explicit-policy-presence contract on `ChannelSetupDmPolicy`, and add a regression test for fresh Feishu config selecting `pairing`. Avoid a global unconditional setter until non-idempotent channel setters such as Twitch are fixed or the setup contract can distinguish missing persisted keys from fallback defaults.
