## Summary

Keep this PR open. Current main still has the Feishu onboarding persistence gap: the DM policy prompt skips `setPolicy` when the selected default equals the fallback current value, and Feishu still reports missing `dmPolicy` as `"pairing"`. The PR is not obsolete, but it needs to be rebased/adapted to the current `src/flows/channel-setup.prompts.ts` flow and should include a regression test. The existing review discussion is material: a blanket unconditional `setPolicy` call is unsafe today because some channel setters are not idempotent.

## Best Possible Solution

Keep the PR open for maintainer review. Rebase it onto current main, implement a focused Feishu explicit-default persistence fix in the current channel setup prompt path, or add a small generic `ChannelSetupDmPolicy` presence/missing-policy seam so the prompt can persist an explicitly selected default only when the backing key is absent. Add a regression test for fresh Feishu config selecting `pairing`, and avoid a blanket unconditional `setPolicy` call until non-idempotent channel setters are fixed or isolated.
