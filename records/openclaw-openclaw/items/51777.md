## Summary

Keep this PR open. Current main still has the Feishu explicit-default persistence gap, and the PR addresses a real onboarding bug, but the patch is stale because the active DM-policy prompt code now lives in `src/flows/channel-setup.prompts.ts` while the PR targets the old `src/commands/onboard-channels.ts` implementation path.

## Best Possible Solution

Keep this PR open for maintainer follow-up. Rebase or rewrite the fix against `src/flows/channel-setup.prompts.ts`, preferably as a focused Feishu missing-key write or a small `ChannelSetupDmPolicy` presence contract, and add a regression test for a fresh Feishu config where the user explicitly selects `pairing`. Avoid a global unconditional setter until non-idempotent channel setters are repaired or bypassed safely.
