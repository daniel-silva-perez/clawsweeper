## Summary

Keep open. Current main still exposes partial Dreams visibility, but it does not expose the requested last-run timestamp, run outcome or partial failure state, processed trace count, or diary updated/workspace status in the Dreams UI.

## What This Changes

Request to expose Dreaming run metadata in the Control UI, including last run time, items processed, diary update/workspace status, run outcome, and next scheduled run.

## Best Possible Solution

Add a memory-core-owned Dreams run summary, expose it through the doctor Dreams status surface or a sibling status method, normalize it in the Control UI controller, render it in the Dreams tab, and update focused backend/UI tests plus the Dreaming docs.
