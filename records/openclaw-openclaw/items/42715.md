## Summary

Keep this issue open. Current main has fixed the most dangerous silent fallback behavior for explicit agent model selections and remote custom providers without an apiKey now fail loudly, but the requested Cloudflare AI Gateway multi-provider credential sharing/documentation is still not implemented: model credentials remain provider-scoped and Cloudflare onboarding only creates the bundled provider/profile.

## Best Possible Solution

Keep this open for a small provider-auth follow-up: document that Cloudflare AI Gateway custom providers need their own models.providers.<id>.apiKey or SecretRef, add an explicit shared credential mechanism only if maintainers want same-gateway reuse, and preserve the current loud failure/strict-selection regression coverage so missing remote custom-provider auth does not silently use another model.

## Risks / Open Questions

- Naively inheriting credentials by matching baseUrl could leak one provider's credential to an unrelated custom provider; any sharing should be explicit, such as a SecretRef reuse or named scoped gateway credential.
- Changing all 401/403 behavior globally could break intentional fallback chains, profile rotation, and cooldown recovery; strict failure should stay tied to explicit model selections or clear operator policy.
