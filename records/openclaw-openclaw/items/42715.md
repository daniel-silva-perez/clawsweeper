## Summary

Keep this issue open. Current main has already fixed the most dangerous silent-fallback part for explicit agent model selections and missing remote custom-provider auth, but the remaining Cloudflare AI Gateway multi-provider credential sharing/documentation request is still not implemented: credentials are still provider-scoped, no sibling provider inherits by matching the same Gateway URL, and Cloudflare docs/onboarding still describe the single bundled provider path rather than additional same-gateway custom providers.

## Best Possible Solution

Keep this open for a small provider-auth follow-up: document that Cloudflare AI Gateway custom providers need their own models.providers.<id>.apiKey or SecretRef, consider an explicit shared credential mechanism if maintainers want same-gateway reuse, and preserve the current strict missing-auth/no-hidden-fallback regression coverage.

## Risks / Open Questions

- Naively inheriting credentials by matching baseUrl could leak one provider credential to an unrelated custom provider; any sharing should be explicit, for example by reusing a SecretRef or a named gateway credential.
- Changing all 401/403 behavior globally could break intentional fallback chains and auth-profile recovery; the strict failure behavior should stay tied to explicit model selections or clear operator policy.
