## Summary

Keep open. Current main still defines and injects the exact `[image data removed - already processed by model]` text into provider replay context, WebChat still forwards the same literal text as ordinary user input, and there is no marker-specific output/input sanitizer or canonical merged fix. The related #66702 is open and explicitly describes a different repro in the same multimodal-context family, not a full superseding fix for this marker recursion report.

## Best Possible Solution

Keep this issue open and fix it in the agent replay/sanitization layer: avoid sending an English bracketed internal marker as ordinary model-visible text, add a focused regression for user-authored literal marker input through WebChat/Kimi Coding, and coordinate with the broader stale-image context work in #66702 without treating that issue as a complete replacement.
