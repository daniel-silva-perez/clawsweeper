## Summary

Keep open. Current main still exposes the exact reported marker as model-visible replay text, WebChat still accepts that literal as normal user input, and Kimi Coding/k2p5 remains an active Anthropic Messages provider path. The related #66702 is useful context for the broader multimodal-history family, but it is open and describes stale image contamination rather than this literal marker recursion case.

## Best Possible Solution

Keep this issue open. The best fix is in the agent replay/sanitization path: stop using an English bracketed internal marker as ordinary model-visible text, or escape/tag it so user-authored literal text cannot collide with internal image-pruning state. Add a focused regression for WebChat literal marker input with Kimi Coding/k2p5 and coordinate with #66702 for the broader stale-image-history behavior without treating it as a complete replacement.
