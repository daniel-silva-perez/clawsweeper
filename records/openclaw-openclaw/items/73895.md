## Summary

Keep open. Current main still does not emit the requested promotion-grade Gateway v2 metadata surface; the typed WS chat event, agent RPC response path, OpenAI Chat Completions adapter, OpenResponses adapter, shared usage shape, and docs all expose narrower response/usage data. The adjacent diagnostics and trajectory work from #72936 is useful context, but it is a support/export path rather than the live response envelope this benchmark request needs.

## What This Changes

The issue requests an additive Gateway v2 response metadata surface with config lineage, routing decisions, fallback/policy/version fields, timing data, decision trace data, and token usage including reasoning tokens for live benchmark smoke tests.

## Best Possible Solution

Define a redacted, additive Gateway v2 metadata envelope in the typed protocol layer, then wire it consistently through the benchmark-facing live response surfaces. Reuse diagnostics or trajectory internals only where their privacy model is explicit, and document missing-provider behavior and backward compatibility.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is an instrumentation feature request. Current-main source inspection gives a high-confidence verification path because the relevant schemas and response builders do not contain the requested fields.

## Solution Assessment

Unclear because there is no patch yet. The suggested routing-layer envelope is directionally right, but the maintainable fix needs maintainer-approved redaction, versioning, target surfaces, and provider-specific token semantics before implementation.
