## Summary

Keep open. Current main still does not emit the requested Gateway v2 promotion metadata envelope; the typed chat event schema, agent RPC acknowledgement, OpenAI-compatible Chat Completions response builder, OpenResponses resource schema, and shared usage normalization remain narrower than the requested config lineage, routing policy, timing, decision trace, and reasoning-token surface.

## What This Changes

The issue requests a Gateway v2 response metadata envelope with config lineage, route/policy identifiers, timing metrics, and token usage including reasoning tokens for live benchmark smoke tests.

## Best Possible Solution

Define an additive, redacted Gateway v2 metadata envelope at the typed protocol boundary, then wire it consistently through the benchmark-facing live response surfaces with documented compatibility behavior and provider-specific missing-field semantics.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is an instrumentation feature request. Source inspection gives a high-confidence verification path because the current schemas and response builders do not contain the requested fields.

## Solution Assessment

Unclear because there is no patch yet. The suggested routing-layer envelope is directionally right, but the maintainable fix needs maintainer-approved schema, redaction, versioning, target surfaces, and reasoning-token semantics.
