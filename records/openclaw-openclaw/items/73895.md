## Summary

Keep open. Current main still lacks the requested promotion-grade Gateway v2 live response metadata, and the remaining work needs maintainer approval because it changes public Gateway protocol/HTTP response contracts and could expose sensitive config or routing details.

## What This Changes

The issue requests additive Gateway v2 response metadata for routing/config lineage, route decisions, timing, policy, decision trace, and token usage so a benchmark smoke harness can validate live Gateway output.

## Best Possible Solution

Define a redacted, additive Gateway v2 metadata envelope in the typed protocol layer, then wire it through the maintainer-approved live surfaces such as agent RPC, WS chat final events, and the OpenAI/OpenResponses compatibility endpoints targeted by the benchmark. Add focused contract tests and docs for missing-provider fields, backward compatibility, and redaction behavior.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is an instrumentation feature request. Current-main source inspection gives a high-confidence verification path because the relevant schemas, response builders, docs, and tests do not contain the requested fields.

## Solution Assessment

Unclear because there is no implementation patch yet. The suggested routing-layer response envelope is directionally right, but the best fix needs maintainer-approved schema, redaction policy, versioning, and endpoint scope before code is written.
