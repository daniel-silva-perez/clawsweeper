## Summary

Keep open. Current main has model selection, usage totals, model fallback logs, and owner-approved trajectory export, but it does not emit the requested promotion-grade Gateway v2 response metadata surface. The WS chat schema, agent RPC result path, and OpenAI/OpenResponses HTTP handlers lack the requested config lineage, route class, fallback chain, policy, timing, and reasoning-token fields.

## Best Possible Solution

Keep this open and design an additive Gateway v2 routing metadata envelope. The best implementation should define redacted config lineage, route class/fallback chain, policy id/version, gateway version, bounded decision trace or trace id, timing fields, and token usage including reasoning tokens when available, then wire it consistently through the WS agent/chat path and whichever HTTP compatibility endpoints the benchmark targets. Existing fields should remain backward compatible, and trajectory/export internals should only be reused if the privacy model stays explicit.
