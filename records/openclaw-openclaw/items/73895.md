## Summary

Keep open. Current main still lacks the requested live Gateway v2 metadata envelope across the typed WebSocket chat event, agent acknowledgement path, OpenAI-compatible Chat Completions output, OpenResponses output, and shared usage normalization; this also needs maintainer-owned schema and redaction decisions before implementation.

## What This Changes

The issue requests a Gateway v2 response metadata envelope with config lineage, routing policy fields, timing metrics, decision trace data, gateway version, and token usage including reasoning tokens for live benchmark smoke tests.

## Best Possible Solution

Define a typed, additive, redacted Gateway v2 metadata envelope at the protocol boundary, then wire it consistently through the approved benchmark-facing live response surfaces with docs and tests for compatibility, omitted-field semantics, and provider-specific usage details.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is an instrumentation feature request. We do have a high-confidence verification path from source inspection because the current schemas and response builders do not contain the requested fields.

## Solution Assessment

Unclear because there is no patch yet. The suggested routing-layer envelope is directionally maintainable, but the final solution should first settle schema shape, redaction, versioning, target surfaces, and reasoning-token semantics.
