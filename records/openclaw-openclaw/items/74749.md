## Summary

Keep open. Current main still sends the reported flag through the bundled Google Gemini CLI backend, the regression test currently locks that behavior in, and upstream Gemini CLI 0.38.2 parser source does not define `--skip-trust`. The issue is fresh, has no protected labels, and exact duplicate searches for `skipTrust`/the yargs error only found this issue.

## What This Changes

The issue asks for the Google Gemini CLI backend to stop passing the unsupported `--skip-trust` argument so `gemini-cli` 0.38.x can run successfully.

## Best Possible Solution

Remove the unsupported flag from the Google plugin’s Gemini CLI initial and resume defaults, update the backend assertion and changelog, and keep approval/trust behavior conservative rather than substituting broader auto-approval.
