## Summary

Keep open. Current main and the latest release still pass `--skip-trust` to the Google Gemini CLI backend, the runner forwards that backend config directly into the spawned `gemini` argv, and the linked open PR #74775 is the active implementation path rather than a reason to close this issue before merge.

## What This Changes

The issue requests removing the unsupported `--skip-trust` flag from the Google Gemini CLI backend defaults so modern `gemini-cli` runs no longer fail before model execution.

## Best Possible Solution

Review and land or replace PR #74775 so the Google plugin removes `--skip-trust` from both fresh and resume defaults, updates the backend default assertions, and keeps Gemini trust/approval behavior conservative and plugin-owned.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current main configures `--skip-trust`, the runner passes configured backend args directly to `gemini`, and upstream Gemini CLI 0.38.2 does not define that option.

## Solution Assessment

Yes. The narrow maintainable solution is to drop only the unsupported flag from `extensions/google/cli-backend.ts` and update `src/agents/cli-backends.test.ts`; substituting a broader approval mode would change security semantics beyond this bug fix.
