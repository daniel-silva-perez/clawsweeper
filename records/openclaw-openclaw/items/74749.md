## Summary

Keep open. Current main still registers `google-gemini-cli` with the unsupported `--skip-trust` flag, the CLI runner forwards those configured args into the spawned `gemini` process, and the latest tagged release remains affected; open PR #74775 is the active unmerged implementation path rather than a reason to close this issue early.

## What This Changes

The issue requests removing the unsupported `--skip-trust` argument from the Google Gemini CLI backend's fresh and resume defaults so gemini-cli 0.38.x calls stop failing.

## Best Possible Solution

Land PR #74775 or a maintainer replacement that removes only the unsupported Gemini trust flag from fresh and resume defaults, updates the backend-default test and changelog, and avoids substituting broader approval behavior such as `--yolo` unless that security posture is explicitly chosen.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current main configures `--skip-trust`, the runner forwards backend args into the `gemini` argv, and upstream Gemini CLI 0.38.2 does not define that option.

## Solution Assessment

Yes. The narrow maintainable fix is to drop the unsupported flag from `extensions/google/cli-backend.ts` and align `src/agents/cli-backends.test.ts`; replacing it with broader auto-approval or trust behavior would solve a different problem with different security semantics.
