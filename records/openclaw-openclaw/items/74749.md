## Summary

Keep open. Current main still registers `google-gemini-cli` with the unsupported `--skip-trust` flag, the CLI runner forwards that flag into the spawned `gemini` process, upstream `@google/gemini-cli` 0.38.2 documents and defines no such option, and the latest tagged release remains affected. Open PR #74775 is the active unmerged implementation path, so this issue should stay open until that PR or a replacement lands.

## What This Changes

The issue asks OpenClaw to remove the unsupported `--skip-trust` flag from the Google Gemini CLI backend defaults because gemini-cli 0.38.x rejects it and every Gemini CLI model call fails.

## Best Possible Solution

Land PR #74775 or a maintainer replacement that removes only `--skip-trust` from fresh and resume Gemini CLI defaults, updates the backend-default test and changelog, and avoids substituting broader approval behavior such as `--yolo` or `--approval-mode=yolo` unless that security posture is explicitly chosen.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current main configures `--skip-trust`, the runner forwards configured args into `gemini`, and upstream `@google/gemini-cli` 0.38.2 has strict yargs options without `skip-trust`.

## Solution Assessment

Yes. Removing the unsupported flag from `extensions/google/cli-backend.ts` and updating the matching test is the narrow maintainable fix; replacing it with broader auto-approval or trust behavior would solve a different problem with different security semantics.
