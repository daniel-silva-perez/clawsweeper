## Summary

Keep open. Current main and the latest release still pass the reported `--skip-trust` flag to the Gemini CLI, upstream Gemini CLI 0.38.2 does not define that flag, and open PR #74775 specifically proposes the fix but has not merged yet.

## What This Changes

The issue requests removing the unsupported `--skip-trust` argument from the bundled Google Gemini CLI backend's initial and resume command defaults.

## Best Possible Solution

Complete and land the linked fix PR or an equivalent maintainer patch that removes the unsupported flag from both Gemini CLI defaults, updates the default-args test, and avoids substituting broader approval behavior such as `--yolo` without explicit maintainer review.
