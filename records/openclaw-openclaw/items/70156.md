## Summary

Keep this PR open. Current main still lacks the proposed Swift `.gitignore` generalization, `check:no-large-files`, and `check:no-unjustified-exec` wiring. The latest PR patch is a plausible active repo-hygiene contribution, not implemented or superseded, but the exec-surface guard still needs maintainer/security review before it becomes a required CI check.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase on current main, refresh both allowlists against the current tree, remove stale allowlist entries, harden the exec guard with parser-backed or otherwise semicolon-safe import detection plus comment/string-safe `shell: true` handling, keep the checks wired through `scripts/run-additional-boundary-checks.mjs`, add regressions for the reviewed parser cases, then run the targeted script tests and the changed CI lane.
