## Summary

Keep open. Current main still does not pass an idle timeout from the PDF tool into the remote media download path. The lower-level fetch helper can abort stalled body reads when `readIdleTimeoutMs` is supplied, but the PDF tool and `loadWebMediaRaw` path do not plumb that option. The related fix PR #68747 is closed unmerged, and the broader open lane-stall report #73581 does not replace the concrete PDF-fetch bug.

## Best Possible Solution

Keep this issue open. The best path is to land a narrow fix like PR #68747: add `readIdleTimeoutMs` to `WebMediaOptions`, forward it through `loadWebMediaRaw` into `fetchRemoteMedia`, set a conservative default for remote PDF loads, and add a PDF-tool regression test that verifies the timeout is passed. Track slash-command bypass and session-level watchdog recovery separately in the broader lane-stall thread #73581.
