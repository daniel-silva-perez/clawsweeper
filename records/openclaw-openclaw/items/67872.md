## Summary

Keep this PR open. Current main still drops Slack no-mention channel messages before shared message hooks can run, so the PR is not obsolete, but the submitted patch is not mergeable because it gates the new hook on the generic group-policy surface instead of Slack's typed channel policy.

## What This Changes

The PR adds a Slack no-mention skip-path internal `message:received` hook gated by ingest policy and adds silent-ingest regression coverage.

## Best Possible Solution

Add an explicit Slack-owned `ingest` boolean to `channels.slack.channels` with wildcard plus channel-specific override semantics, wire it through Slack types/schema/metadata/docs and `resolveSlackChannelConfig`, then emit `message:received` from the no-mention skip path only when that resolved opt-in policy is true.

## Reproduction Assessment

Yes. A Slack room message with `requireMention` true and no bot mention reaches the no-mention branch in `prepareSlackMessage`, records pending history, returns `null`, and the caller returns before dispatch hooks run.

## Solution Assessment

No. The desired behavior is valid, but the submitted implementation reads the wrong config surface; the maintainable fix should resolve a Slack-specific ingest policy from `channels.slack.channels` rather than using generic `groups`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Resolve ingest from Slack channel config:** `extensions/slack/src/monitor/message-handler/prepare.ts:549-556`
  - body: This lookup uses `resolveChannelGroupPolicy`, which reads `channels.slack.groups`, but Slack's public and strict channel policy is `channels.slack.channels`. Valid Slack configs therefore cannot set the `ingest` value this branch checks, so the new mention-skip hook path remains disabled in production.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The diff is supply-chain clean, but the behavior is privacy-sensitive because it can emit skipped Slack channel content to internal hooks and needs an explicit documented Slack opt-in.

Concerns:

- **[medium] Document and type the Slack ingest opt-in:** `extensions/slack/src/monitor/message-handler/prepare.ts:557`
  - body: The patch emits unmentioned Slack channel content to internal hooks when its new ingest gate is true, but it does not add a valid Slack config key, schema entry, metadata, or docs describing that content exposure. This should be an explicit Slack channel policy before merge.
  - confidence: 0.82
