## Summary

Keep open. Current main still drops Slack channel messages that miss mention-gating before shared message hooks run, so the PR is not obsolete. The submitted patch is not merge-ready because it resolves `ingest` through the generic `groups` policy while Slack channel policy is `channels.slack.channels`, and the typed Slack config/schema/docs do not currently expose an `ingest` field. Security review did not show CI, dependency, lockfile, package, or secret-handling changes; the relevant risk is privacy/product behavior because no-mention Slack content would be forwarded to internal hooks and should be explicit, documented opt-in policy.

## What This Changes

The PR adds a Slack no-mention skip-path internal `message:received` hook gated by group `ingest` policy and adds silent-ingest regression coverage.

## Best Possible Solution

Keep the PR open and request a Slack-owned revision: add an explicit Slack `ingest` boolean to `channels.slack.channels` with wildcard plus channel-specific override semantics, wire it through Slack types/schema/metadata/docs and `resolveSlackChannelConfig`, then emit internal `message:received` from the no-mention skip path only when that resolved opt-in policy is true.
