## Summary

Keep this PR open. Current main still does not implement the PR's central ACP lifecycle fixes: `/acp spawn` reports success after `initializeSession` without a direct post-spawn runtime status probe, and stale `Gateway disconnected` failures are not recovered in close, passive cancel, active-turn cancel, or early-turn retry paths. The related `/acp` command-routing fix is present on main, but it is a separate surface and does not supersede this PR.

## Best Possible Solution

Keep this PR open for ACP/Gateway lifecycle review. The best path is a narrow fix that adds a direct post-init spawn liveness/status check with cleanup, treats resolved `dead`/`no-session` status as unhealthy, and applies an explicit stale gateway-disconnect close-code allowlist across close, passive cancel, active-turn cancel, and early-turn retry, with the focused tests proposed in the PR.
