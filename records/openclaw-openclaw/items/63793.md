## Summary

Keep this PR open. Current main still lacks the PR's central ACP lifecycle behavior: `/acp spawn` does not probe the freshly initialized runtime before reporting success, and `AcpSessionManager` still only treats `acpx exited ...` as stale-handle recovery for close/cancel/early-turn retry. The related ACP command-routing fix is present on main, but it is a separate command-dispatch issue and does not supersede this PR.

## Best Possible Solution

Keep this PR open for ACP/Gateway lifecycle review. The best path is a narrow fix that adds direct post-init runtime liveness/status probing with cleanup before `/acp spawn` reports success, treats resolved `dead`/`no-session` status as unhealthy, and applies an explicit stale gateway-disconnect close-code allowlist across close, passive cancel, active-turn cancel, and early-turn retry, with the focused tests proposed in the PR.
