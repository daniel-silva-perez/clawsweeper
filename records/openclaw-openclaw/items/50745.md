## Summary

Keep this PR open. Current main still does not implement the requested WhatsApp `lookupPnLidEntry` / `PnLidEntry` active-listener API, and the PR remains a focused implementation candidate with unresolved review items around authDir fallback, changelog preservation, and the plugin-visible privacy boundary.

## Best Possible Solution

Keep this PR open as the implementation candidate. Rebase it on current main, preserve existing changelog entries, implement lookup through the same `jidToE164` / `resolveJidToE164` authDir-first semantics used by current inbound normalization for `@lid` and `@hosted.lid`, add coverage for LID, hosted LID, phone JID, plain phone, cold-cache, and authDir fallback cases, and get maintainer review on whether this plugin-visible phone-number resolution needs opt-in gating or docs.
