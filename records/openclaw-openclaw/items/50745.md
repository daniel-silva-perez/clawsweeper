## Summary

Keep this PR open. Current main still does not implement the requested WhatsApp `lookupPnLidEntry`/`PnLidEntry` active-listener API, and the live PR remains open, unmerged, and dirty against main. The feature is plausible, but the current diff still needs maintainer review for the authDir LID fallback, changelog preservation, and plugin-visible privacy boundary.

## Best Possible Solution

Keep this PR open as a focused WhatsApp implementation candidate. The best path is to rebase it on current main, preserve existing changelog entries, implement lookup via the existing `jidToE164`/`resolveJidToE164` authDir-first semantics for `@lid` and `@hosted.lid`, add cold-cache/authDir fallback coverage alongside the LID, hosted LID, phone JID, and plain phone cases, and get maintainer product review on whether plugin-visible phone-number resolution needs gating or docs.
