## Summary

Keep this PR open. Current main still lacks the requested WhatsApp `lookupPnLidEntry` / `PnLidEntry` active-listener API, while the PR remains a focused implementation candidate with review items still unresolved around authDir fallback behavior, changelog preservation, and the plugin-visible privacy boundary.

## Best Possible Solution

Keep this PR open as the implementation candidate. Rebase it on current main, preserve the existing changelog entries, implement lookup through the same `jidToE164` / `resolveJidToE164` authDir-first semantics used by current inbound normalization for `@lid` and `@hosted.lid`, add coverage for LID, hosted LID, phone JID, plain phone, cold-cache, and authDir fallback cases, and get maintainer review on whether this plugin-visible phone-number resolution needs opt-in gating or docs.
