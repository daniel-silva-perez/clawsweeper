## Summary

Keep this PR open. Current main still does not implement the requested WhatsApp `lookupPnLidEntry` / `PnLidEntry` active-listener API, and the PR remains a focused implementation candidate rather than obsolete work. The remaining review concerns are real: the new plugin-visible lookup should follow current authDir-first LID resolution semantics, preserve the changelog, and get maintainer privacy review because it exposes phone-number resolution to plugins.

## Best Possible Solution

Keep this PR open as the implementation candidate. Rebase it on current main, preserve existing changelog entries, implement lookup through the same `jidToE164` / `resolveJidToE164` authDir-first semantics used by current inbound normalization for `@lid` and `@hosted.lid`, add coverage for LID, hosted LID, phone JID, plain phone, cold-cache, and authDir fallback cases, and get maintainer review on whether this plugin-visible phone-number resolution needs opt-in gating or docs.
