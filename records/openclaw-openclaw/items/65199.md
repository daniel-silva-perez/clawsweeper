## Summary

Keep open. Current main still lacks the PR's cron humanization in both targeted UI surfaces, and the PR already includes focused code plus follow-up fixes for the earlier review comments. The remaining work is maintainer review, rebase/CI, and product judgment because current Dreaming behavior now uses one managed sweep cron shared across phases rather than the older distinct per-phase schedules described in the original report.

## What This Changes

This PR adds a tested `describeCronExpression()` UI formatter and uses it in Cron Jobs subtitles and Dreaming phase status text while preserving fallback behavior for raw cron expressions and missing phase cron strings.

## Best Possible Solution

Keep the PR open for normal maintainer review and CI. The desired end state is a small, tested cron-description formatter for the Control UI, with Cron Jobs labels humanized and Dreaming schedule text adjusted to match the current one-managed-sweep model documented at https://docs.openclaw.ai/concepts/dreaming.
