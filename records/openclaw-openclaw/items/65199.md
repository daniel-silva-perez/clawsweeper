## Summary

Keep open. Current main still lacks the cron-expression humanization this PR proposes: cron jobs render raw `Cron <expr>` subtitles, Dreaming phase rows still show only `nextRunAtMs`, and the existing Dreaming status model already carries phase cron strings that the current view ignores. The PR remains a focused UI improvement, with prior review feedback about minute offsets and missing-cron fallback reflected in its follow-up commits, but maintainers should review the Dreaming copy against the newer one-managed-sweep model before landing.

## What This Changes

This PR adds a `describeCronExpression()` UI formatter with tests and uses it in Cron Jobs and Dreaming phase status text so common cron expressions render as concise human-readable schedules.

## Best Possible Solution

Keep the PR open for normal maintainer review and CI. The desired end state is a small, tested UI formatter for common cron expressions that improves Cron Jobs labels and Dreaming schedule clarity while preserving raw-expression fallback and aligning Dreaming copy with the current managed-sweep model.
