## Summary

Keep open. Current main still lacks the proposed sandbox Docker workspace mount propagation config, and the PR is not obsolete, but the proposed patch is incomplete because it only applies the option to the main sandbox container while browser sandbox containers still create the workspace mount without the propagation mode.

## What This Changes

The PR adds a `sandbox.docker.workspaceMountPropagation` config field with type/schema/generated-schema plumbing, Docker workspace mount formatting changes, and sandbox mount tests for `rprivate`, `rslave`, and `rshared`.

## Best Possible Solution

Finish the PR by applying the propagation mode consistently to both main and browser sandbox workspace mounts, keep `rprivate` as the explicit default-compatible value, update generated config/docs/changelog as required for the new public config surface, and validate the Docker create args and config-hash behavior for all three propagation modes.
