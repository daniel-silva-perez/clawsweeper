## Summary

Keep this PR open. Current main still has no `workspaceMountPropagation` config, schema, docs, or Docker create-arg plumbing, and the requested behavior belongs in the core Docker sandbox path rather than ClawHub. The PR is a focused implementation candidate, but it needs maintainer/security review and the remaining browser-container parity issue resolved before merge.

## Best Possible Solution

The best path is to finish the implementation by adding the config key consistently across public types, Zod schema, generated schema/help/labels, docs, config resolution, main sandbox container creation, browser container creation, config-hash recreation behavior, and targeted tests, then have secops/maintainers decide whether `rprivate`/`rslave`/`rshared` is the right public API and warning level.
