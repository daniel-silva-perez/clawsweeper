## Summary

Keep this open. Current main still has the reported closed-stdin failure shape, and open PR #73566 is already linked with closing syntax, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Make `openclaw plugins uninstall <id>` fail cleanly when confirmation stdin closes, while preserving piped `y`/`n` answers and `--force`.

## Best Possible Solution

Review and land PR #73566 or an equivalent narrow fix that makes the shared prompt helper settle on readline close/EOF, converts that case in plugin uninstall into exit 1 with interactive/`--force` guidance, and adds focused prompt and uninstall regression tests. Keep #73551 separate unless a PR intentionally covers both the stale-entry cleanup and this prompt failure.

## Reproduction Assessment

Yes. Current main still awaits the shared prompt in the non-forced uninstall path, and a read-only `node:readline/promises` closed-stdin smoke reproduces exit 13 while piped input resolves normally.

## Solution Assessment

Yes at the design level. The linked PR direction is the narrow maintainable fix because it handles EOF in the shared prompt primitive and catches that specific condition in `plugins uninstall` without using a blanket non-TTY gate that would break piped answers.
