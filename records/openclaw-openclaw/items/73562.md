## Summary

Keep this issue open. Current main still has the reported failure shape: `plugins uninstall` awaits the shared `promptYesNo()` confirmation when `--force` is absent, and `promptYesNo()` awaits `readline.question()` without handling EOF/readline close. An isolated Node/readline reproduction with closed stdin exits 13, while piped input still resolves, matching the report.

## Best Possible Solution

Fix the shared CLI prompt helper to settle cleanly on readline close/EOF, preferably with a distinct error/result for closed confirmation input. Then have `plugins uninstall` turn that into exit code 1 with an actionable message suggesting an interactive TTY or `--force`, and add focused tests in `src/cli/prompt.test.ts` and `src/cli/plugins-cli.uninstall.test.ts`. Keep #73551 separate unless a PR intentionally fixes both the stale-entry seed and this cleanup prompt failure.
