## Summary

Keep this PR open. Current main still has the runtime/context-report gap the PR targets: `loadWorkspaceBootstrapFiles` always includes `BOOTSTRAP.md` and converts any failed guarded read into a missing bootstrap entry, while the existing #17491 behavior is limited to the Control UI file-list RPC. The PR remains a focused implementation candidate, and the current diff does not introduce an obvious security or supply-chain concern.

## Best Possible Solution

Keep this PR open and review/land it, or land an equivalent change in `src/agents/workspace.ts`: use a read-only setup-completion lookup, omit only genuinely absent (`ENOENT`) `BOOTSTRAP.md` after setup completion, keep present files and non-ENOENT failures visible, and retain the regression tests in `src/agents/workspace.test.ts`.
