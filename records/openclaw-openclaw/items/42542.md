## Summary

Keep this PR open. Current main at e476523082a6fff3a0c3048077d2f0e52413fbf2 still loads BOOTSTRAP.md unconditionally for runtime bootstrap context and renders absent bootstrap files as [MISSING]. The already-shipped hide behavior is limited to the Control UI file-list path, so the PR remains a focused implementation candidate. The discussion adds useful independent validation from Kaspre, and the current PR diff is limited to src/agents/workspace.ts plus tests with no obvious security or supply-chain regression.

## Best Possible Solution

Keep this PR open and review or land it, or land an equivalent change in src/agents/workspace.ts: use a read-only setup-completion lookup, omit only absent ENOENT BOOTSTRAP.md after setup completion, preserve present files and non-ENOENT failures, and keep the regression tests in src/agents/workspace.test.ts.
