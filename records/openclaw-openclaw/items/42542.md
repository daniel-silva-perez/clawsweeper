## Summary

Keep this PR open. Current main still has the runtime `/context` gap the PR targets: `loadWorkspaceBootstrapFiles` always includes `BOOTSTRAP.md` and turns any guarded-read failure into a missing bootstrap entry, while the existing post-onboarding hide behavior is only in the Control UI `agents.files.list` path. The PR remains a focused implementation candidate, with useful independent production validation in the discussion and no obvious security or supply-chain concern in the provided diff.

## Best Possible Solution

Keep this PR open and review/land it, or land an equivalent change in `src/agents/workspace.ts`: use a read-only setup-completion lookup, omit only absent (`ENOENT`) `BOOTSTRAP.md` after setup completion, keep present files and non-ENOENT failures visible, and retain the regression tests in `src/agents/workspace.test.ts`.
