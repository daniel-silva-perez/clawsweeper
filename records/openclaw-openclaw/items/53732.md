## Summary

Keep open. Current main still builds BOOT.md into a normal model prompt, carries that prompt through fallback retries, and has no boot-specific literal-echo guard before final reply logging or message-tool delivery. The related PR #55016 is closed unmerged and targeted boot outbound delivery/send-policy behavior rather than suppressing echoed BOOT.md instructions.

## What This Changes

The issue requests that OpenClaw prevent fallback models from echoing generated BOOT.md startup instructions to users instead of executing them silently.

## Best Possible Solution

Add an internal boot-run context that follows the generated BOOT.md prompt through agent execution and tool setup, then suppress substantial literal echoes of that generated prompt before user-visible final replies and message-tool sends while preserving intentional BOOT.md-directed outbound notifications.
