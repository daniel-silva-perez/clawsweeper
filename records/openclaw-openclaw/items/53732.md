## Summary

Keep open. Current main still embeds BOOT.md as ordinary model prompt text, preserves that prompt through fallback retries, and has no boot-specific echo guard before message-tool delivery; the related outbound-delivery PR #55016 is closed unmerged and did not implement prompt-echo suppression.

## What This Changes

Fallback models can echo BOOT.md startup instructions to users instead of executing them silently.

## Best Possible Solution

Add a narrow internal boot-run context that carries a generated prompt fingerprint or comparable suppression signal through agent execution and message-tool/final-delivery paths, blocks substantial literal echoes of the BOOT.md prompt, and preserves intentional BOOT.md-triggered sends with focused regression tests.
