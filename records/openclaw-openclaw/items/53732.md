## Summary

Keep open. Current main still builds BOOT.md into ordinary model prompt text, carries that body through fallback attempts, and lacks a boot-specific literal-echo guard on the message-tool delivery path; the closely related PR #55016 is closed unmerged and addressed outbound delivery policy rather than prompt-echo suppression.

## What This Changes

The issue reports fallback models sometimes echo generated BOOT.md startup instructions to users and asks OpenClaw to make boot prompts model-agnostic or suppress literal prompt echoes.

## Best Possible Solution

Add a narrow internal boot-run context that carries the generated BOOT.md prompt or fingerprint through agent execution, suppresses substantial literal echoes before message-tool sends and any final-delivery/logging path, and preserves intentional BOOT.md-directed outbound notifications with focused regression tests.
