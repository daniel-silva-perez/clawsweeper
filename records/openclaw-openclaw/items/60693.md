## Summary

Keep open. Current main clearly covers parts of the report: Discord REST proxy wiring is implemented, ACPX now stages Codex/Claude ACP runtime dependencies, and model-selection precedence is documented. But the umbrella issue still contains remaining config/product asks that are not fully implemented on main: `openclaw models set` only updates `agents.defaults.model`, cron and per-agent overrides remain separate by design, and I did not find an ACP allowedAgents repair that prunes or skips missing/disabled harness entries during startup identity reconciliation.

## Best Possible Solution

Leave this issue open or split it: close/mark the Discord proxy and ACP package-staging portions as already handled, then either implement a deliberate bulk model-switch/cleanup command or document that per-agent and cron overrides are intentionally separate. For the ACP allowedAgents portion, add a doctor/fix or startup warning that identifies missing/unavailable ACP harness ids without repeatedly failing identity reconciliation, or decide explicitly that stale pending ACP sessions should be cleaned up instead.
