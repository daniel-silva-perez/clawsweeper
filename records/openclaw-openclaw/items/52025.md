## Summary

Keep open. Current main has adjacent managed assistant/generated-image support, but it still does not implement this PR's intended direct tool-result image behavior for `{ type: "image", data, mimeType }`, and tool-message images are still only rendered inside collapsed tool output. This remains a focused implementation candidate for open #50779 rather than something already shipped on main.

## Best Possible Solution

Keep this PR open as a focused candidate for #50779. The best fix should update against current main, handle direct and nested tool-result `{ type: "image", data, mimeType }` blocks, image-only results, accompanying text/captions, default visible inline previews outside collapsed tool details, live stream plus history reload paths, and regression coverage in the real grouped renderer while preserving managed-media safety constraints.

## Risks / Open Questions

- A renderer-only patch can still miss live tool-stream messages and history reload behavior because structured tool output is flattened or bounded in nearby paths.
- Rendering raw base64 tool media inline needs explicit MIME, SVG/data-URL, and size safeguards so it does not bypass managed media protections.
- The PR is useful but currently needs an update against current main; current main has evolved around managed assistant media without closing this specific tool-result gap.
