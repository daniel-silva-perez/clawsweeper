## Summary

Keep this PR open. Current main still does not implement the PR’s intended direct tool-result image rendering for `{ type: "image", data, mimeType }`, and the linked product bug remains open. The PR is a plausible focused implementation candidate, but it still needs maintainer review around default visibility, nested/live-streamed tool results, text preservation, and media safety.

## Best Possible Solution

Keep this PR open as a focused candidate for the open Control UI tool-image bug. The best fix should update against current main, render direct and nested `{ type: "image", data, mimeType }` tool-result blocks visibly outside collapsed tool details, preserve accompanying text/captions, cover image-only results plus live stream/history reload paths, and keep the existing Control UI media safety constraints intact.
