## Summary

Keep this PR open. Current main has adjacent managed assistant/generated-image support, but it still does not implement this PR's intended direct tool-result image behavior for `{ type: "image", data, mimeType }`, and tool-message images are still rendered only inside collapsed tool output. The PR remains a focused implementation candidate for open #50779, though it needs the review-thread gaps addressed before it is ready to land.

## Best Possible Solution

Keep this PR open as a focused candidate for #50779. The best fix should update against current main, handle direct and nested `{ type: "image", data, mimeType }` tool-result blocks, image-only results, accompanying text/captions, default visible inline previews outside collapsed tool details, live stream plus history reload paths, and targeted regression coverage while preserving the Control UI's managed-media and data-image safety constraints.

## Risks / Open Questions

- A renderer-only patch can miss live-streamed tool output because `ui/src/ui/app-tool-stream.ts` currently flattens structured tool results to text.
- Tool-result images need to be visible outside the collapsed tool-output details by default; tests that only assert DOM presence can pass while the user-visible bug remains.
- Rendering direct base64 tool media should preserve existing MIME, SVG/data-URL, size, and managed-media safety expectations rather than bypassing them.
