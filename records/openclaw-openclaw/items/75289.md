## Summary

Keep this open. Current main still has a plausible TUI lifecycle bug: the TUI handles SIGTERM but not SIGHUP, shutdown only resolves after terminal drain/stop succeeds, and active status/loader timers are not explicitly stopped on the exit path. Related open reports #61723 and #75137 overlap in TUI resource symptoms, but this issue adds a distinct macOS terminal-close/SIGTERM orphan path.

## What This Changes

The issue requests that `openclaw tui --session <name>` exit cleanly on terminal close, SIGTERM, and gateway restart instead of leaving CPU- and memory-heavy orphaned `openclaw-tui` processes.

## Best Possible Solution

Fix the TUI lifecycle itself: make shutdown idempotent, handle POSIX SIGHUP, perform bounded cleanup that always resolves or exits, stop status/loader timers before terminal drain/stop, and tolerate closed terminal streams without changing Gateway reconnect policy.

## Reproduction Assessment

Yes. The issue gives concrete macOS steps and current main has matching source evidence: no TUI SIGHUP handler, a SIGTERM path that waits on successful terminal drain/stop, and active render/status intervals that are not explicitly cleaned up on exit.

## Solution Assessment

Yes, a narrow TUI lifecycle repair is the best path. The Gateway client already clears reconnect/tick timers on stop, so broad Gateway restart or plugin-loader changes should stay out of scope unless profiling later proves a second root cause.

## Repair Work Prompt

Fix the TUI orphan/shutdown lifecycle bug reported in https://github.com/openclaw/openclaw/issues/75289. On current main, `runTui` handles SIGINT/SIGTERM but not SIGHUP, `requestExit()` only finishes after `drainAndStopTuiSafely(tui)` resolves, and status/loader timers are not explicitly stopped on exit. Implement a narrow TUI lifecycle repair in `src/tui/tui.ts`: add POSIX SIGHUP handling where supported, make shutdown idempotent, stop status/loader/waiting timers before terminal drain/stop, make drain/stop bounded and tolerant of closed terminal stream errors such as EPIPE/EBADF where appropriate, and ensure SIGTERM cannot hang indefinitely if terminal teardown rejects. Preserve Gateway reconnect/backoff behavior and plugin loading semantics. Add focused unit coverage in `src/tui/tui.test.ts` and, if launch-process signal forwarding changes, `src/tui/tui-launch.test.ts`. Update `docs/web/tui.md` or `docs/cli/tui.md` only if user-facing troubleshooting behavior changes. Add a single-line CHANGELOG fix entry for the user-facing runtime fix thanking @wolfgang-voss. Cross-link #61723 and #75137 in the PR body as related context, but do not collapse their distinct symptoms unless the fix demonstrably covers them.
