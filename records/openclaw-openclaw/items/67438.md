## Summary

Keep open. Current main still sends Feishu scan-to-create through a compact terminal QR (`small: true`), and the documented onboarding path still asks users to scan that QR with the Feishu/Lark mobile app. The only linked fix candidate, PR #67450, was closed unmerged, so the reported Windows onboarding scan failure still has a plausible and narrow unresolved code path.

## What This Changes

The issue asks OpenClaw to make the Feishu QR code shown during `openclaw onboard` scannable so the local OpenClaw setup can connect to a Feishu account.

## Best Possible Solution

Fix this in the Feishu plugin-owned setup path by making scan-to-create output reliably scannable, preferably with a larger terminal QR and a fallback users can open or view as an image when terminal rendering fails. Add focused regression coverage around the Feishu QR render option and keep docs aligned if a fallback becomes user-visible.
