## Summary

Keep this PR open. Current main still has single-target cron delivery, so the requested fan-out feature is not implemented, but the proposed branch is dirty against current main and still has concrete webhook-mode and session-routing bugs that should block merge.

## What This Changes

The PR adds optional cron `delivery.additionalTargets` fan-out across cron types, gateway schema, normalization, runtime dispatch, CLI edit flags, and tests, plus lazy channel-runtime access in `commands-session.ts`.

## Best Possible Solution

Land one canonical, documented cron fan-out implementation that keeps schema, normalization, delivery planning, runtime dispatch, trace/status reporting, CLI/UI surfaces, docs, changelog, and regression tests aligned, with explicit behavior for announce, webhook, no-delivery, primary-success gating, `channel: "last"`, and per-target failure isolation.

## Reproduction Assessment

Yes for the review findings. A static reproduction is enough: webhook delivery accepts `additionalTargets` in the PR schema but never reaches the direct-send `sendOccurred` fan-out gate, and a session-bound job using `channel: "last"` resolves additional targets with the run session instead of the job session.

## Solution Assessment

No. The direction is reasonable, but this patch is not the best mergeable solution until it either supports or rejects webhook fan-out, resolves `last` from the job delivery session, and is rebased onto current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Either support or reject webhook fan-out:** `src/cron/isolated-agent/run.ts:769-775`
  - body: `additionalTargets` is added to the shared delivery schema, so webhook-mode cron jobs can accept and persist extra targets. The runtime fan-out is gated on `deliveryResult.sendOccurred === true`, but webhook delivery is posted later by the gateway notification path, so webhook jobs silently skip all additional targets. Reject the field for unsupported modes or trigger fan-out from the webhook completion path too.
  - confidence: 0.9
- **[P2] Resolve last targets from the job session:** `src/cron/isolated-agent/delivery-dispatch.ts:762-767`
  - body: Additional targets can use `channel: "last"`, but `deliverToAdditionalTargets` passes the cron run's `agentSessionKey` into `resolveDeliveryTarget`. Primary delivery resolves against the job's configured session key, so session-bound jobs can resolve additional `last` targets from the wrong fresh cron session or fail closed. Pass the job delivery session key into this helper and cover it with a regression test.
  - confidence: 0.86

## Security Review

Status: needs_attention
