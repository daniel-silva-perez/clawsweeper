## Summary

Keep this PR open. Current main does not implement cron multi-target fan-out, so the PR is still a live implementation candidate, but the patch is not ready to merge because accepted configurations can silently skip additional deliveries.

## What This Changes

The PR adds `delivery.additionalTargets` cron fan-out across cron types, gateway schema, normalization, runtime delivery dispatch, CLI edit flags, and tests, plus lazy channel-runtime access in `commands-session.ts`.

## Best Possible Solution

Land one canonical, documented cron fan-out implementation that keeps schema, normalization, delivery planning, runtime dispatch, trace/status reporting, CLI/UI surfaces, docs, changelog, and regression tests aligned, with explicit behavior for announce, webhook, no-delivery, primary-success gating, `channel: "last"`, and per-target failure isolation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Either support or reject webhook fan-out:** `src/cron/isolated-agent/run.ts:769-772`
  - body: `additionalTargets` is added to the shared delivery schema, so webhook-mode cron jobs can now accept and persist extra targets. The runtime fan-out, however, only runs when `deliveryResult.sendOccurred === true`, and webhook delivery is posted later by the gateway notification path rather than by `dispatchCronDelivery`, so webhook jobs silently skip all additional targets. Please either reject `additionalTargets` outside supported modes or trigger fan-out from the webhook completion path too.
  - confidence: 0.9
- **[P2] Resolve last targets from the job session:** `src/cron/isolated-agent/delivery-dispatch.ts:914-920`
  - body: Additional targets can use `channel: "last"`, but `deliverToAdditionalTargets` passes the cron run's `agentSessionKey` into `resolveDeliveryTarget`. Primary delivery resolves against the job's configured session key, so session-bound jobs can resolve additional `last` targets from the wrong fresh cron session or fail closed. Pass the job delivery session key into this helper and cover `channel: "last"` fan-out with a regression test.
  - confidence: 0.84
