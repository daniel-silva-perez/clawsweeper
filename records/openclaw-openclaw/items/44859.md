## Summary

Keep open. Current main still has single-target cron delivery, and the PR remains useful but not merge-ready because the proposed fan-out contract has unresolved webhook-mode and session-routing bugs that can silently drop expected deliveries or resolve the wrong recipient.

## What This Changes

The PR adds optional cron `delivery.additionalTargets` fan-out across cron types, protocol schema, normalization, delivery planning, runtime dispatch, CLI edit flags, and tests, plus lazy channel-runtime access in `commands-session.ts`.

## Best Possible Solution

Land one canonical, documented cron fan-out implementation that aligns schema, normalization, delivery planning, runtime dispatch, trace/status reporting, CLI/UI surfaces, docs, changelog, and regression tests, with explicit behavior for announce, webhook, no-delivery, primary-success gating, `channel: "last"`, and per-target failure isolation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Either support or reject webhook fan-out:** `src/gateway/protocol/schema/cron.ts:191`
  - body: Adding `additionalTargets` to the shared delivery schema makes webhook-mode cron jobs accept the field, but the fan-out code only runs after direct `dispatchCronDelivery` sends. Webhook delivery is posted later by `dispatchGatewayCronFinishedNotifications`, so those jobs silently skip the extra targets. Please reject the field for unsupported modes or trigger fan-out from the webhook completion path too.
  - confidence: 0.9
- **[P2] Resolve last targets from the job session:** `src/cron/isolated-agent/delivery-dispatch.ts:766`
  - body: Additional targets can use `channel: "last"`, but this call passes the fresh cron run session into `resolveDeliveryTarget`. Primary delivery resolves against the job delivery session, so session-bound jobs can fan out to the wrong thread or fail closed. Pass the job delivery session key into this helper and cover it with a regression test.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff introduces routing-sensitive outbound fan-out, and one code path can resolve `channel: "last"` additional targets against the wrong session.

Concerns:

- **[medium] Fan-out can resolve the wrong last recipient:** `src/cron/isolated-agent/delivery-dispatch.ts:766`
  - body: `deliverToAdditionalTargets` passes the cron run session key to `resolveDeliveryTarget` instead of the job delivery session context. For session-bound cron jobs, `channel: "last"` can route cron output to an unintended channel or thread.
  - confidence: 0.86
