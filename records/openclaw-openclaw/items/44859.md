## What This Changes

The PR adds optional cron `delivery.additionalTargets` fan-out across cron types, gateway schema, normalization, runtime dispatch, CLI edit flags, and tests, plus lazy channel-runtime access in `commands-session.ts`.

## Best Possible Solution

Land one canonical, documented cron fan-out implementation that explicitly supports or rejects each delivery mode, resolves `last` targets from the same job delivery session context as primary delivery, reports per-target outcomes coherently, and keeps CLI, schema, docs, changelog, and tests aligned.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Either support or reject webhook fan-out:** `src/gateway/protocol/schema/cron.ts:191`
  - body: `additionalTargets` is added to shared delivery properties, so webhook-mode cron jobs validate and store it. The fan-out path only runs when `deliveryResult.sendOccurred === true`, while webhook delivery is posted later by gateway notifications, so webhook jobs silently skip every additional target. Please reject the field outside supported modes or fan out from the webhook completion path.
  - confidence: 0.9
- **[P2] Resolve last targets with the job session:** `src/cron/isolated-agent/delivery-dispatch.ts:766`
  - body: `deliverToAdditionalTargets` resolves each target with `params.agentSessionKey`, but primary cron delivery resolves `channel: "last"` from the job's configured delivery session. Session-bound jobs can therefore send fan-out output to the wrong last recipient or fail closed. Pass the job delivery session key into this helper and cover `last` fan-out with a regression test.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The PR introduces a routing-sensitive outbound fan-out path, and additional `channel: "last"` targets can resolve against the wrong session.

Concerns:

- **[medium] Fan-out can resolve the wrong last recipient:** `src/cron/isolated-agent/delivery-dispatch.ts:766`
  - body: `deliverToAdditionalTargets` passes the cron run session key into `resolveDeliveryTarget` for additional targets. For session-bound cron jobs, `channel: "last"` can resolve from the wrong session state and send cron output to an unintended channel or thread.
  - confidence: 0.86
