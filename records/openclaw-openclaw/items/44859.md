## Summary

Keep this PR open. Current main still has a single-target cron delivery contract and runtime path, so the proposed `delivery.additionalTargets` fan-out is not implemented; the PR is useful but still needs maintainer review for mergeability, semantics, and outbound-delivery security before it can land.

## What This Changes

The PR adds a `delivery.additionalTargets` cron fan-out field across cron types, gateway schema, normalization, delivery planning, service merge behavior, isolated-run dispatch, CLI edit flags, and tests, plus a lazy channel-runtime access tweak in command-session handling.

## Best Possible Solution

Land one canonical, documented fan-out implementation that keeps the cron contract consistent across schema, normalization, delivery planning, runtime dispatch, trace/status reporting, CLI/UI surfaces, docs, changelog, and regression tests, with per-target failure isolation and explicit webhook/primary-success semantics.
