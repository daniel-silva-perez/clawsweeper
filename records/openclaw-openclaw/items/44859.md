## Summary

Keep open. Current main still has single-target cron delivery, so the PR is not obsolete, but the latest PR head still has blocking delivery-mode and session-routing bugs that need author or maintainer fixes before merge.

## What This Changes

The PR adds optional cron `delivery.additionalTargets` fan-out across cron types, protocol schema, normalization, delivery planning, runtime dispatch, CLI edit flags, and tests, plus lazy channel-runtime access in `commands-session.ts`.

## Best Possible Solution

Land one canonical, documented cron fan-out implementation that keeps schema, normalization, delivery planning, runtime dispatch, trace/status reporting, CLI/UI surfaces, docs, changelog, and regression tests aligned, with explicit behavior for announce, webhook, no-delivery, primary-success gating, `channel: "last"`, and per-target failure isolation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Either support or reject webhook fan-out:** `src/cron/isolated-agent/run.ts:769-772`
  - body: `additionalTargets` is accepted through the shared delivery schema, so webhook-mode cron jobs can store extra targets. The runtime fan-out is gated on `deliveryResult.sendOccurred === true`, but webhook delivery is posted later by gateway notifications rather than by direct cron delivery, so webhook jobs silently skip every additional target. Reject the field outside supported modes or trigger fan-out from the webhook completion path too.
  - confidence: 0.9
- **[P2] Resolve last targets from the job session:** `src/cron/isolated-agent/delivery-dispatch.ts:914-920`
  - body: Additional targets can use `channel: "last"`, but the helper resolves each one with the cron run session key. Primary delivery resolves from the job delivery session context, so session-bound jobs can send fan-out output to the wrong last recipient or fail closed. Pass the job delivery session key into the helper and add a `last` fan-out regression test.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff introduces routing-sensitive outbound fan-out, and `channel: "last"` additional targets can resolve against the wrong session.

Concerns:

- **[medium] Fan-out can resolve the wrong last recipient:** `src/cron/isolated-agent/delivery-dispatch.ts:914`
  - body: `deliverToAdditionalTargets` resolves additional targets using the cron run session key rather than the job delivery session context. For session-bound cron jobs, `channel: "last"` can route cron output to an unintended channel or thread.
