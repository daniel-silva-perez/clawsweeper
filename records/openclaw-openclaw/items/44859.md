## Summary

Keep this PR open. The requested cron fan-out capability is not present on current main, and the proposed branch still has merge-blocking delivery semantics around webhook mode and session-based routing for `channel: "last"`. There is also an open duplicate-candidate branch, so this needs maintainer canonical-branch selection rather than cleanup closure.

## What This Changes

The PR adds optional cron `delivery.additionalTargets` fan-out across cron types, gateway schema, normalization, runtime dispatch, CLI edit flags, and tests, plus lazy channel-runtime access in `commands-session.ts`.

## Best Possible Solution

Land one canonical, rebased cron fan-out implementation that explicitly supports or rejects each delivery mode, resolves additional targets with the same job delivery session context as primary delivery, records/report failures coherently, and updates CLI/API docs and tests for announce, webhook, no-delivery, `channel: "last"`, accountId, and per-target failure isolation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Either support or reject webhook fan-out:** `src/gateway/protocol/schema/cron.ts:191`
  - body: `additionalTargets` is added to shared delivery properties, so webhook jobs validate and persist it. The only fan-out call is gated by `deliveryResult.sendOccurred`, which webhook delivery never sets because webhook posting happens later in gateway notifications. This silently drops configured extra targets; reject the field for webhook/noop modes or fan out from the webhook completion path.
  - confidence: 0.9
- **[P2] Resolve last targets with the job session:** `src/cron/isolated-agent/delivery-dispatch.ts:766`
  - body: `deliverToAdditionalTargets` resolves each target with `params.agentSessionKey`, but primary cron delivery resolves `channel: "last"` from the job's delivery session key. Session-bound jobs can therefore send additional output to the wrong last recipient or fail closed. Pass the job delivery session key into this helper and cover `last` fan-out with a regression test.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff introduces a routing-sensitive outbound fan-out path that can resolve `channel: "last"` against the wrong session.

Concerns:

- **[medium] Fan-out can resolve the wrong last recipient:** `src/cron/isolated-agent/delivery-dispatch.ts:766`
  - body: `deliverToAdditionalTargets` passes the cron run session key to `resolveDeliveryTarget` for additional targets. For session-bound cron jobs, `channel: "last"` can resolve from the wrong session state, which may send cron output to an unintended channel/thread.
  - confidence: 0.86
