## Summary

Keep open. Current main still passes startup-captured config into Discord native command routing and Slack slash command routing, while Telegram already reloads runtime config before native-command route resolution. The issue is also paired with open PR #39617 using closing syntax, so it should stay open until that PR is reviewed, landed, or replaced.

## What This Changes

Native slash commands in Discord, Telegram, and Slack DMs should route sessions using the current `session.dmScope` instead of a provider-startup config snapshot.

## Best Possible Solution

Review, land, or replace #39617 with a narrow channel-owned fix that refreshes route-sensitive config for Discord and Slack native/slash command routing, preserves Telegram’s already-fresh runtime path, and adds regression coverage for non-main DM session keys.
