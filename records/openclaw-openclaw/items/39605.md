## Summary

Keep open. Current main still resolves Discord native commands and Slack slash commands with provider-startup config while route selection depends on `session.dmScope`; Telegram already has a fresh-config native command path. The issue is also paired with open PR #39617 using closing syntax, so it should remain open until that PR is reviewed, landed, or replaced.

## What This Changes

Native Discord, Telegram, and Slack slash commands should route DM commands using the current `session.dmScope` instead of a provider-startup config snapshot.

## Best Possible Solution

Review and land PR #39617, or replace it with an equivalent narrow channel-owned fix that refreshes route-sensitive config for Discord and Slack native/slash route resolution while adding regression tests for non-main DM session keys.
