## Summary

Keep this issue open. Current main still has Discord native command and Slack slash command routing paths that resolve DM session keys from provider-startup config, while `resolveAgentRoute` derives direct-message session keys from the supplied `cfg.session.dmScope`. Telegram now has a fresh runtime-config path, but Discord and Slack remain unresolved. PR #39617 is also still an open closing-syntax fix candidate, so the issue should stay open until that PR is landed, closed, or replaced.

## Best Possible Solution

Keep this issue open and review, rebase, and land PR #39617 or an equivalent maintainer-reviewed fix in `extensions/discord/src/monitor/native-command.ts` and `extensions/slack/src/monitor/slash.ts`. Add focused regression tests proving Discord native DM commands and Slack slash DM commands honor an updated `session.dmScope` without recreating the provider; preserve the existing Telegram fresh-config behavior.
