## Summary

Keep this issue open. Current main still passes captured provider-startup config into Discord native-command routing and Slack slash-command route resolution, while route selection depends on `cfg.session.dmScope`; Telegram’s native command path already refreshes runtime config before resolving the route. The issue is also paired with open PR #39617 using closing syntax, so it should remain open until that PR is reviewed, landed, or replaced.

## What This Changes

Discord, Telegram, and Slack native slash commands in DMs should resolve the active session using the current `session.dmScope` instead of falling back to the agent main session.

## Best Possible Solution

The end state should be a narrow channel-owned fix that refreshes route-sensitive config for Discord and Slack native/slash command routing, keeps Telegram’s already-fresh native path intact, and adds regression coverage for non-main DM session keys.
