## Summary

Keep this issue open. Current main still routes Discord and Slack native slash-command DM targets through config objects captured at provider startup, while Telegram has already moved its native-command route/auth path to a fresh runtime config. The linked open PR #39617 explicitly targets this issue, so the right maintainer path is to review, land, or replace that PR rather than close the issue early.

## What This Changes

Make Discord, Telegram, and Slack native slash commands resolve DM routes with the current `session.dmScope` instead of falling back to the main session through stale startup config.

## Best Possible Solution

Review, land, or replace #39617 with a narrow channel-owned fix that refreshes the route config for Discord and Slack native slash-command routing, keeps Telegram’s fresh-config behavior intact, and adds per-channel DM regression coverage before closing this issue.
