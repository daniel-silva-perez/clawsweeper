## Summary

Keep this issue open. Current main still routes Discord native command interactions and Slack slash commands through provider-startup config objects, while Telegram native commands already reload runtime config before route resolution; the issue is also paired with open PR #39617, so it should not be closed before that implementation is reviewed or replaced.

## What This Changes

Discord, Telegram, and Slack native slash-command DMs should respect `session.dmScope` routing instead of falling back to the main session.

## Best Possible Solution

The end state should be a narrow channel-owned fix that resolves Discord and Slack native-command DM targets with fresh runtime config, keeps Telegram's already-fresh path intact, and adds regression coverage for DM session keys under non-main `session.dmScope`.
