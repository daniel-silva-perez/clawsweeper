## Summary

Keep open. Current main still passes startup-captured config into the Discord native command and Slack slash command routing paths, while Telegram already refreshes runtime config before native-command route resolution. The issue is also paired with open PR #39617 using closing syntax, so cleanup should wait for that PR to be reviewed, landed, or replaced.

## What This Changes

The issue requests that Discord, Telegram, and Slack native slash commands in DMs respect current `session.dmScope` routing instead of falling back to the main session key.

## Best Possible Solution

Review, land, or replace #39617 with a narrow channel-owned fix that refreshes route-sensitive config for Discord and Slack native/slash command routing, preserves Telegram’s already-fresh runtime path, and adds regression coverage for non-main DM session keys.
