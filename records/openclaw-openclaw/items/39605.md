## Summary

Keep this issue open. Current main still routes Discord native commands and Slack slash commands through provider-startup config while `resolveAgentRoute` derives DM session keys from the supplied `cfg.session.dmScope`. Telegram already has a fresh-runtime-config path, but the Discord and Slack portions remain unresolved. PR #39617 is still open with closing syntax for this issue, so the issue should stay open until that PR is rebased/landed or replaced by an equivalent fix.

## Best Possible Solution

Keep this issue open and review, rebase, and land PR #39617 or an equivalent maintainer-reviewed fix ported to `extensions/discord/src/monitor/native-command.ts` and `extensions/slack/src/monitor/slash.ts`. Add focused regression tests proving Discord native DM commands and Slack slash DM commands honor a changed `session.dmScope` without recreating the provider; keep Telegram covered if the final patch touches it.
