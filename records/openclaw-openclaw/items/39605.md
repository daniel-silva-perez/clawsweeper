## Summary

Keep this issue open. Current main still routes Discord and Slack native slash-command DM targets through provider-startup config objects, while Telegram has already moved native command routing to a fresh runtime config path; the open PR #39617 explicitly targets this issue, so the maintainer path is to review, land, or replace that PR rather than close the issue early.

## What This Changes

The issue requests that Discord, Telegram, and Slack native slash commands in DMs respect `session.dmScope` instead of routing command targets to the shared main session.

## Best Possible Solution

Review, land, or replace #39617 with a narrow channel-owned fix that refreshes the route config for Discord and Slack native slash-command target resolution, preserves Telegram’s already-fresh runtime behavior, and adds DM routing regression tests before closing this issue.
