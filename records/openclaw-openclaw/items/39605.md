## Summary

Keep this issue open. Current main still leaves the Discord native-command and Slack slash-command route-resolution paths tied to provider-startup config snapshots, while Telegram already refreshes runtime config for native commands. ## What This Changes

The issue requests Discord, Telegram, and Slack native slash commands in DMs to respect `session.dmScope` routing instead of falling back to the main session.

## Best Possible Solution

Finish a narrow channel-owned fix for Discord and Slack native/slash command route resolution using current runtime config, keep Telegram's already-fresh path intact, and add regression coverage for non-main `session.dmScope` values.
