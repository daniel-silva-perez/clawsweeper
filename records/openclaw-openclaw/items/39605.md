## Summary

Keep this issue open. Current main still routes Discord native command interactions and Slack slash commands through provider-startup config objects, so route resolution can miss a changed `session.dmScope`; Telegram already reloads runtime config before native command route resolution. The issue is also paired with open PR #39617, so it should remain open until that implementation is reviewed, replaced, or merged.

## What This Changes

Native Discord, Telegram, and Slack slash commands in DMs should route according to the current `session.dmScope` instead of falling back to the main session from stale startup config.

## Best Possible Solution

A narrow channel-owned fix should reload fresh runtime config before Discord and Slack native-command route resolution, preserve Telegram's already-fresh path, and add regression coverage for DM session keys under non-main `session.dmScope` values.

## Reproduction Assessment

Yes. The issue gives concrete channel steps, and static current-main verification shows `resolveAgentRoute` depends on `cfg.session.dmScope` while Discord and Slack native command paths still pass startup config into route resolution.

## Solution Assessment

Yes. Reloading route-sensitive runtime config at native command handling time for Discord and Slack is the narrowest maintainable fix; this does not need a core migration or a broader routing redesign.
