## Summary

Keep this issue open. Current main still routes Discord and Slack native slash-command sessions through config snapshots captured at provider startup, while the linked open PR #39617 explicitly closes this issue and is the right implementation candidate to review or replace. Telegram appears to have already moved its native-command route/auth path to a fresh runtime config, so the remaining fix should be verified per channel rather than closing this issue now.

## What This Changes

Native slash commands in Discord, Telegram, and Slack should route DMs using the current runtime config so `session.dmScope` affects the session key the same way normal messages do.

## Best Possible Solution

Review, land, or replace #39617 with a narrow fix that makes Discord and Slack native slash-command route resolution use the current runtime config, preserves Telegram’s fresh-config behavior, and adds regression tests for `session.dmScope: "per-channel-peer"` in native DM commands before closing this issue.
