## Summary

Keep this issue open. Current main still routes Discord native command and Slack slash command route resolution through config captured at provider startup, while `resolveAgentRoute` still derives DM session keys from `cfg.session.dmScope`. Telegram already uses a fresher runtime-config pattern. The issue is also paired with open closing-syntax PR #39617, so it should stay open until that PR is merged, closed, or replaced.

## Best Possible Solution

Keep this issue open and review, rebase, and land PR #39617 or an equivalent maintainer-approved fix. The narrow fix should refresh routing config for Discord native command and Slack slash command route resolution, preserve Telegram’s fresh-config behavior, and add focused regression tests proving DM native/slash commands honor `session.dmScope` without recreating the provider.
