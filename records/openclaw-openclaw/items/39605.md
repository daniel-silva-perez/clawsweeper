## Summary

Keep this open. Current main still leaves Discord native-command and Slack slash-command route resolution tied to provider-startup config objects while route computation reads `session.dmScope`; Telegram already refreshes runtime config for native commands. The issue is also paired with open PR #39617 using closing syntax, so it should not be closed before that implementation path is reviewed, rebased, replaced, or merged.

## What This Changes

The issue requests Discord, Telegram, and Slack native slash commands in DMs to respect `session.dmScope` and route to the same isolated DM session as normal text messages.

## Best Possible Solution

Review, rebase, or replace PR #39617 with a narrow channel-owned fix that makes Discord native commands and Slack slash commands resolve routes from current runtime config, preserves Telegram's already-fresh path, and adds Discord/Slack regression coverage for non-main `session.dmScope`.

## Reproduction Assessment

Yes. The issue gives concrete DM steps, and static current-main verification shows `resolveAgentRoute` depends on `cfg.session.dmScope` while Discord and Slack command paths still pass provider-startup config into route resolution.

## Solution Assessment

Yes. Refreshing route-sensitive runtime config at Discord/Slack native command handling time is the narrowest maintainable fix; because open PR #39617 already targets this issue, the issue should remain open until that PR is resolved.
