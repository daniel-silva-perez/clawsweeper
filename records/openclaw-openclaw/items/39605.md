## Summary

Keep open. Current main still routes Discord native command interactions and Slack slash commands with the monitor startup/context config, while Telegram native commands already reload runtime config before route resolution. The issue is also paired with open PR #39617 using closing syntax, so it should stay open until that PR is reviewed, landed, or replaced.

## What This Changes

Native slash commands in Discord, Telegram, and Slack should respect `session.dmScope` for direct-message session routing.

## Best Possible Solution

Review, land, or replace #39617 with a narrow channel-owned fix that refreshes route-sensitive config for Discord and Slack native-command DM routing, preserves Telegram’s already-fresh path, and adds regression coverage for non-main `session.dmScope` direct-message session keys.
