## Summary

Keep open. PR #73256 is a useful partial fix for observe-only Chrome joins and realtime speech-output verification, but current main still does not implement the central request: a reliable supported headless/listen-first Google Meet participant with live transcription health proof. The product surface remains explicit Meet URL joins through Chrome or Twilio, with post-meeting artifact/transcript export separate from live admission/audio/transcription proof.

## Best Possible Solution

Keep this open as Google Meet plugin product/bug work in `extensions/google-meet`. The best path is to define the supported identity/admission model, add a real listen-first path or explicitly document the Google API boundary, make observe/transcribe status prove live transcript or caption movement when available, and keep hardening Chrome-node/realtime with end-to-end admission plus actual audio input/output proof.
