## Summary

Keep open. Current main still has the isolated-cron awareness path that can mirror delivered isolated cron output into the main session, and the closed unmerged PR #61658 was an exact attempted fix rather than shipped implementation.

## Best Possible Solution

Keep this issue open and revive or reimplement the substance of PR #61658: only queue main-session awareness for isolated cron runs when the resolved delivery target is explicit. Default or implicit `announce`/`channel: last` delivery should send the cron result without mirrored main-session system events, with regression coverage for implicit last-target delivery, pinned channel without `to`, and explicit-target awareness preservation.
