## Summary

Keep open. Current main still has the reported mirror path: isolated agent-turn cron jobs default to announce delivery, implicit announce delivery resolves through the last chat route, and successful non-best-effort isolated delivery queues the delivered text into the main-session system-event queue. The related fix PR #61658 identified the same root cause but closed unmerged, so it does not supersede this issue.

## What This Changes

Cron jobs configured with `sessionTarget: isolated` are reported to write cron output into the main conversation instead of keeping each run isolated.

## Best Possible Solution

Implement a narrow cron delivery change so implicit/default isolated announce runs deliver to the resolved chat target and record run status without enqueueing the delivered text into the main session. Preserve explicit-target awareness only if maintainers still want that behavior, and update the regression tests, docs if needed, and changelog together.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: an isolated `agentTurn` without explicit delivery defaults to announce, resolves to the implicit last-chat route, and the delivery dispatcher enqueues the delivered text into `agent:main:main`; the current test suite asserts that queueing behavior.

## Solution Assessment

Yes. The best fix direction is the same narrow boundary described by the closed related PR: gate main-session awareness on explicit delivery intent instead of removing isolated sessions, delivery defaults, or outbound delivery behavior broadly.
