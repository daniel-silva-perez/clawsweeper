## Summary

Keep this PR open. Current main still creates `/home/node/.cache/ms-playwright` in the Docker runtime stage as root, then only chowns the child `ms-playwright` directory before switching the image to `USER node`; the proposed parent cache ownership fix has not been absorbed. The duplicate related PR #43846 was closed unmerged, and the author explicitly replied on April 28, 2026 that the problem still remains.

## Best Possible Solution

Review and land this focused Dockerfile ownership change if targeted Docker validation confirms the browser-enabled image leaves `/home/node/.cache` owned by or writable to the runtime `node` user. Add a narrow Dockerfile assertion or build proof for the parent cache ownership contract if feasible.
