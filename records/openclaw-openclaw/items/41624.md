## Summary

Keep open. Current main still has the parent-directory ownership bug: the Dockerfile chowns only `/home/node/.cache/ms-playwright`, while the root-run `mkdir -p` creates `/home/node/.cache` itself. This PR is a narrow, plausible fix and is not superseded by the closed duplicate #43846 or the broader open Docker env-wiring PR #61167.

## What This Changes

This PR widens the optional Docker browser-install chown from `/home/node/.cache/ms-playwright` to `/home/node/.cache`.

## Best Possible Solution

Land this focused parent-cache ownership fix, ideally with a small assertion in `src/dockerfile.test.ts` that the browser-install block chowns `/home/node/.cache`, then run the targeted Dockerfile test and a browser-enabled Docker build smoke if feasible.

## Reproduction Assessment

Yes. The current Dockerfile runs `mkdir -p /home/node/.cache/ms-playwright` before `USER node`, so Docker creates the parent `.cache` directory as root; the existing chown only covers the child and leaves runtime writes to sibling cache paths blocked for `node`.

## Solution Assessment

Yes for the reported bug. Widening ownership to `/home/node/.cache` is the narrowest Dockerfile-level fix; the only improvement before merge would be adding focused regression coverage for the ownership target.
