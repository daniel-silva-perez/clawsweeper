## Summary

Keep this PR open. Current main still creates `/home/node/.cache/ms-playwright` in the optional browser-install Dockerfile path and chowns only that child directory before switching the runtime image to `USER node`; the PR’s parent-cache ownership change has not landed. The latest release tag still has the same child-only chown, the duplicate child PR #43846 was closed unmerged, and #61167 is separate Docker env/setup wiring rather than proof that this image-layer ownership fix is implemented.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a focused Dockerfile ownership fix, plus a narrow `src/dockerfile.test.ts` assertion if feasible, after targeted Docker validation confirms that a browser-enabled image leaves `/home/node/.cache` writable by the runtime user. If maintainers instead choose #42771’s `/opt/playwright` direction, close or supersede this PR with that explicit canonical path.
