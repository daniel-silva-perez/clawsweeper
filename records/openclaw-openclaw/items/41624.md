## Summary

Keep open. Current main still chowns only `/home/node/.cache/ms-playwright` in the optional browser-enabled Docker build path, while runtime later switches to `USER node`; this PR is a narrow and plausible fix, but it remains Docker image permission work with an open broader alternative, so it needs maintainer review rather than cleanup closure.

## What This Changes

This PR widens the Dockerfile's optional `OPENCLAW_INSTALL_BROWSER` ownership fix from `/home/node/.cache/ms-playwright` to `/home/node/.cache` after installing Playwright Chromium.

## Best Possible Solution

Land this focused parent-cache ownership fix with a narrow Dockerfile test assertion and browser-enabled Docker build/run validation, unless maintainers explicitly choose #42771 as the broader path migration that replaces it.

## Reproduction Assessment

Yes. Current main shows the optional browser build path creates `/home/node/.cache/ms-playwright` as root and chowns only that child before switching to `USER node`; a browser-enabled Docker build or equivalent mkdir/chown ownership check would reproduce the parent cache ownership gap.

## Solution Assessment

Yes for the focused bug. Widening the chown to `/home/node/.cache` is the narrowest fix for the current documented cache path; the safer alternative only if maintainers want a larger behavior change is to land #42771 and move the browser directory to `/opt/playwright`.
