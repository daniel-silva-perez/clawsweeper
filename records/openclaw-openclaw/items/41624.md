## Summary

Keep this PR open. Current main still has the reported Docker ownership gap: the browser-install path creates `/home/node/.cache/ms-playwright` as root, chowns only that child directory, and then runs the image as the non-root `node` user. The duplicate child PR was closed unmerged, and the newer broader Docker PR does not currently absorb this exact Dockerfile fix.

## Best Possible Solution

Review and land this focused Dockerfile ownership change if targeted Docker validation confirms the browser-enabled image leaves `/home/node/.cache` owned by or writable to the runtime `node` user. Add a narrow Dockerfile assertion or build proof for the parent cache ownership contract if feasible.
