## Summary

Keep open. Current main still has the parent cache ownership gap, and this PR is a focused, plausible fix, but Docker image permissions plus overlapping open Docker browser-cache work need maintainer review rather than cleanup closure.

## What This Changes

This PR changes the optional Docker browser-install layer to chown `/home/node/.cache` instead of only `/home/node/.cache/ms-playwright`.

## Best Possible Solution

Land this narrow parent-cache ownership fix with a Dockerfile assertion and browser-enabled Docker build/run validation, unless maintainers explicitly choose #42771 as the replacement path migration.

## Reproduction Assessment

Yes. Current main shows a root-run `mkdir -p /home/node/.cache/ms-playwright` followed by a child-only chown and later `USER node`; a browser-enabled Docker build or equivalent ownership check can reproduce the parent cache ownership gap.

## Solution Assessment

Yes for the focused bug. Chowning `/home/node/.cache` is the smallest maintainable fix for the current documented path; the broader alternative is #42771 if maintainers want to move Playwright browsers to `/opt/playwright`.
