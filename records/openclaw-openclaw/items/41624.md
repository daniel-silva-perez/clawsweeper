## Summary

Keep this PR open. Current main still has the Dockerfile ownership gap the PR targets: the optional browser install path creates `/home/node/.cache/ms-playwright` while running as root, chowns only the child directory, and later switches the runtime image to the non-root `node` user. The related duplicate child PR was closed unmerged, and the broader open Docker setup PR does not prove this exact Dockerfile ownership change has landed.

## Best Possible Solution

Review and land this focused Dockerfile ownership change if targeted Docker validation confirms the browser-enabled image leaves `/home/node/.cache` owned by or writable to the runtime `node` user. Add a narrow Dockerfile assertion or build proof for the parent cache ownership contract if feasible.
