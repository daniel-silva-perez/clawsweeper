## Summary

Keep open. Current main still does not fully implement the requested hardening: OpenShell forwards generated command text into the shared SSH `/bin/sh -c` path without placeholder or malformed-shell preflight, and generic same-tool/same-args loops still warn until the later no-progress circuit breaker. The related open #58726 covers only the narrower loop-abort piece, not the OpenShell malformed-command preflight.

## Best Possible Solution

Keep this issue open for a focused runtime hardening patch: add shared SSH/OpenShell preflight before remote shell command construction for obvious generated placeholders and malformed shell structure; make repeated identical failed or no-progress generic calls capable of becoming a clear critical block at a configured threshold; add regression coverage for OpenShell/shared SSH command construction and before-tool-call blocking. Keep #58726 as the narrower loop-detector follow-up unless maintainers choose to consolidate that part.

## Risks / Open Questions

- A generated-command preflight needs careful scoping so legitimate shell redirection, comparisons, command substitution, and literal angle brackets in data are not rejected incorrectly.
- A generic repeat hard-abort should likely be failure/no-progress gated and configurable to avoid blocking legitimate repeated status or polling workflows.
