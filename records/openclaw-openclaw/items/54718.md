## Summary

Keep this PR open. Current main landed adjacent gateway-auth hotfixes for password fallback and explicit same-host loopback proxy opt-in, but it does not definitely implement this PR’s intended shared-secret/token fallback; current main explicitly rejects token + trusted-proxy mode as mutually exclusive. The PR also remains a security-sensitive auth design decision with active author follow-up and an open RFC, so automatic cleanup would be premature.

## Best Possible Solution

Keep the PR open for maintainer review or an explicit manual supersession decision. The concrete next step is to rebase it against current main and decide whether token fallback should remain unsupported, whether any remaining same-host reverse-proxy behavior should move into the current allowLoopback/password-fallback design, or whether #69066 should become the canonical follow-up for the remaining gateway-auth architecture work.
