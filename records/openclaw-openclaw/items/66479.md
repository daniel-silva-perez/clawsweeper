## Summary

Keep this issue open. Current main still rejects BOM-prefixed markdown frontmatter before skill parsing, and the affected skill is then dropped from discovery because the local loader requires a parsed description. Related #45432 is separate write/edit-tool BOM behavior, not skill frontmatter tolerance.

## Best Possible Solution

Fix the shared markdown frontmatter parser to tolerate a single leading UTF-8 BOM before the opening --- delimiter, then add focused coverage in src/markdown/frontmatter.test.ts and a skill discovery or skills list path test proving a BOM-prefixed SKILL.md remains discoverable. Keep #45432 closed as the separate write/edit-tool BOM report.
