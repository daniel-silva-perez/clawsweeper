## Repair Work Prompt

Add the `agents.setDefault` Gateway RPC requested by https://github.com/openclaw/openclaw/issues/52928. Implement it as an ADMIN-scoped additive core Gateway method, not as a startup migration and not by adding `agents.defaultId`. Params should be `{ agentId: string }`; result should be `{ ok: true, defaultId: string }`. Validate that `agentId` is a configured agent, set `default: true` on that entry, remove `default` from all other `agents.list` entries, persist with the normal config write path, and return the resolved default id. Keep `config.patch` as the bulk-edit escape hatch. Update the handler map, method registry, scope table, protocol schemas/types/validators/exports, generated protocol artifacts, public protocol docs, and focused tests. Include a user-facing changelog entry credited with `Thanks @samzong`. Do not broaden `agents.update` to accept arbitrary config fields or introduce core extension-specific behavior.
