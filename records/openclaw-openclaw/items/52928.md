## Summary

Keep open. Current main still lacks the requested `agents.setDefault` Gateway RPC across the handler map, advertised method list, ADMIN scope table, protocol schemas/exports, and public protocol docs. The related Control UI default-agent bug has been fixed by writing `agents.list[].default` through the config form, but that does not provide the dedicated RPC this issue requests for external desktop/web clients.

## What This Changes

Request to add an ADMIN-scoped `agents.setDefault` Gateway RPC that switches the configured default agent without requiring clients to use `config.patch`.

## Best Possible Solution

Add a small additive core Gateway method for default-agent selection while keeping `config.patch` as the advanced bulk-edit escape hatch. The method should validate the requested id against configured agents, persist exactly one `default: true` flag, return the resolved `defaultId`, and keep handler, schema, method advertising, scope mapping, docs, generated protocol artifacts, and tests synchronized.
