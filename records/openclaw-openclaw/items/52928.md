## Summary

Keep open. Current main still lacks the requested `agents.setDefault` Gateway RPC across the handler map, advertised method list, ADMIN scope table, protocol schemas/types/validators, generated client model surface, and public protocol docs. The related #59070 Control UI work is useful context because it confirms `agents.list[].default` is the right config representation, but it remains a config-edit path rather than the dedicated external-client RPC requested here.

## What This Changes

The issue requests an ADMIN-scoped Gateway `agents.setDefault` RPC that accepts `{ agentId: string }`, marks that configured agent as default, clears default flags from other agent entries, and returns the new `defaultId`.

## Best Possible Solution

Add a small additive Gateway method that validates the configured agent id, persists exactly one `agents.list[].default: true`, returns the resolved `defaultId`, and publishes the contract consistently for desktop/web clients while keeping `config.patch` as the bulk-edit escape hatch.
