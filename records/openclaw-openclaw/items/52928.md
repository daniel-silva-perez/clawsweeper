## Summary

Keep open. Current main still lacks the requested dedicated Gateway RPC: agent CRUD exists, the config representation already uses `agents.list[].default`, but there is no `agents.setDefault` handler, advertised method, ADMIN scope classification, protocol schema/type/validator, SDK helper, test coverage, or public protocol doc entry. ## What This Changes

Request to add an ADMIN-scoped `agents.setDefault` Gateway RPC that validates a configured agent id, updates `agents.list[].default`, clears other defaults, and returns the new `defaultId`.

## Best Possible Solution

Add a small additive core Gateway method that reuses the existing agent config representation, rejects unknown configured agents, persists exactly one default entry, returns the resolved `defaultId`, and keeps `config.patch` as the lower-level bulk-edit escape hatch.

Yes, the requested additive ADMIN-scoped Gateway method is the narrowest maintainable way to serve desktop/web clients without exposing them to raw config merge-patch mechanics. The existing Control UI config helper is useful precedent for storage semantics but is not sufficient for external Gateway clients.
