## Summary

Keep this PR open. Current main still performs an immediate Control UI `sessions.list` reload after each `sessions.changed` push event, so the PR is not obsolete or implemented elsewhere. The proposed direction is focused and useful, but it still needs test updates and lifecycle timer cleanup before it is merge-ready.

## Best Possible Solution

Review and refine this PR rather than closing it: add a named debounce for full `sessions.list` reconciliation after `sessions.changed`, keep `applySessionsChangedEvent` immediate for responsive UI updates, add fake-timer tests for delayed/coalesced reloads, and clear the pending timer in Control UI disconnect cleanup before merging.
