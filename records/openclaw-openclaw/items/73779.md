## Summary

Keep this PR open. Current main still handles `sessions.changed` by applying the incremental session update and immediately calling `loadSessions`, so the debounce is not implemented or superseded. The PR is a plausible focused UI optimization, but the submitted patch still needs test updates and lifecycle cleanup before it is merge-ready.

## Best Possible Solution

Keep the PR open and refine it: add a named debounce constant for full `sessions.list` reconciliation after `sessions.changed`, keep the immediate `applySessionsChangedEvent` update, add fake-timer tests for delay and coalescing, initialize the timer in the test host, and clear the pending timer in the Control UI lifecycle cleanup path before merge.
