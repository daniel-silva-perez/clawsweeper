## Summary

Keep this PR open. Current main still performs an immediate Control UI `sessions.list` reload for each `sessions.changed` push event, so the PR's intended debounce is not implemented or superseded. The PR is focused on the web UI event path for `sessions.changed` / `sessions.list`, but it still needs test updates and timer cleanup before it is merge-ready.

## Best Possible Solution

Review and refine this PR rather than closing it: implement a named, tested debounce for full `sessions.list` reconciliation after `sessions.changed`, keep the immediate `applySessionsChangedEvent` path for responsive UI updates, add fake-timer coverage for coalescing, and clear any pending timer in the Control UI lifecycle cleanup path.
