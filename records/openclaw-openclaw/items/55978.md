## Summary

Keep this PR open. Current main still has the reported cron workspace sticky offset bug: `.cron-workspace-form` uses a hardcoded `top: 74px` and height calculation, while the shell layout is driven by `--shell-topbar-height: 52px`. The PR remains a focused implementation candidate, and the discussion shows ProjectClownfish intentionally repaired the branch to preserve the contributor path.

## Best Possible Solution

Review and land this focused PR if CI and a quick desktop cron workspace check pass. The desired mainline change is to update `.cron-workspace-form` to use `top: 0` plus `var(--shell-topbar-height)`, keep the narrow regression guard and changelog entry, and then close after the merge reaches main.
