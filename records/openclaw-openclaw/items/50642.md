## Summary

Keep open. Current main still contains the reported macOS direct wss first-use TOFU path, still does not carry the documented remote TLS fingerprint into the macOS node session, and the linked closing PR is open rather than merged.

## What This Changes

The issue requests hardening macOS node direct wss connections so a fresh gateway certificate is not automatically trusted and gateway.remote.tlsFingerprint can protect first use.

## Best Possible Solution

Harden the macOS node direct wss path so first connection fails closed unless normal trust evaluation succeeds or an explicit configured or user-confirmed fingerprint matches, then add focused Swift coverage for both first-use rejection and configured fingerprint behavior.
