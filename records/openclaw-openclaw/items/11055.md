## Summary

Keep open. Current main still lacks the requested agent-visible pre-compaction warning: the config/type/schema surface has no `visibleWarning` or `warningPrompt`, the memory-flush turn is still run as a silent embedded memory turn, and the existing `notifyUser` path only sends user-channel status notices. This is valid, but should get maintainer product/API judgment because it changes core agent lifecycle prompting and overlaps broader pre-compaction lifecycle work.

## What This Changes

The issue requests an optional `memoryFlush.visibleWarning` and configurable warning prompt that is visible to the agent before automatic compaction, while keeping the existing silent memory flush behavior.

## Best Possible Solution

Maintainers should decide the lifecycle boundary, then either add a validated, deduped agent-visible warning under `agents.defaults.compaction.memoryFlush` before the silent flush/compaction path, or expose a documented runtime/plugin hook that can provide the same warning without overloading `notifyUser`.

## Reproduction Assessment

Not applicable as a feature request. There is a high-confidence verification path for the missing behavior by inspecting current main: the proposed config fields are absent, the schema is strict, and the memory flush remains `silentExpected: true`.

## Solution Assessment

Unclear. The requested config knob is plausible, but model-visible lifecycle prompting should be decided alongside the broader pre-compaction lifecycle design in #38520 before queueing an automated repair.
