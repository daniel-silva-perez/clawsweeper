## Summary

Keep open. Current main still lacks the requested agent-visible pre-compaction warning: the memory-flush config has no `visibleWarning` or `warningPrompt`, the strict schema would reject those keys, and the memory-flush runner still launches a silent embedded memory turn. The merged `notifyUser` work is adjacent but only posts start/completion notices to the user's channel, and the broader lifecycle design in #38520 remains open.

## What This Changes

Add optional `memoryFlush` config for a visible agent warning before compaction so the agent can save in-progress context consciously.

## Best Possible Solution

Maintainers should choose the boundary, then implement either a validated, deduped agent-visible pre-compaction warning under `agents.defaults.compaction.memoryFlush` or a documented runtime/plugin extension point that can safely add the warning while preserving silent memory flush and user-channel `notifyUser` behavior.
