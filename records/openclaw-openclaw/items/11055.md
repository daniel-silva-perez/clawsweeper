## Summary

Keep open. Current main still lacks the requested agent-visible pre-compaction warning: the memory-flush config type and strict schema have no `visibleWarning` or `warningPrompt` fields, the memory-flush runner still invokes a silent embedded memory turn, and the adjacent `notifyUser` path only sends user-channel status notices. The existing discussion already distinguishes this from user-channel compaction notices and routes it for maintainer product/API judgment.

## What This Changes

The issue requests optional `memoryFlush.visibleWarning` and `warningPrompt` config that injects an agent-visible warning before compaction so the model can checkpoint active work.

## Best Possible Solution

Maintainers should choose the boundary, then implement either a validated, deduped agent-visible pre-compaction warning under `agents.defaults.compaction.memoryFlush` or a documented runtime/plugin extension point that can safely add the warning while preserving silent memory flush and user-channel notices.
