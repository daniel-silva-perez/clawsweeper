## Summary

Keep open. Current main still lacks the requested agent-visible pre-compaction warning: the memoryFlush config and strict schema do not accept visibleWarning or warningPrompt, the memory-flush runner remains a silent embedded turn, and the merged notifyUser work only sends user-channel status notices. Existing comments also keep this open for maintainer product/API boundary judgment, with broader lifecycle work still tracked separately in #38520.

## What This Changes

The issue requests optional memoryFlush.visibleWarning and warningPrompt config that injects an agent-visible pre-compaction warning before the existing silent memory flush.

## Best Possible Solution

Maintainers should choose the boundary, then provide either a validated, deduped agent-visible warning surface under agents.defaults.compaction.memoryFlush or a documented runtime/plugin hook that can safely inject that warning while preserving silent memory flush and user-channel notices.
