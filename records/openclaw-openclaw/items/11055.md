## Summary

Keep open. Current main still lacks the requested agent-visible pre-compaction warning: the config/type/schema surface has no `visibleWarning` or `warningPrompt`, memory flush still runs as a silent embedded memory turn, and the shipped `notifyUser` behavior is user-channel status messaging rather than a model-visible warning moment. The issue is valid, but it needs maintainer product/API judgment before an automated repair PR.

## What This Changes

Add optional `memoryFlush.visibleWarning` and `warningPrompt` behavior so OpenClaw can inject an agent-visible warning before auto-compaction while keeping the existing silent memory flush.

## Best Possible Solution

Maintainers should decide the lifecycle boundary, then either add a validated, deduped agent-visible pre-compaction warning under `agents.defaults.compaction.memoryFlush` or expose/document a runtime/plugin hook that can safely inject the same warning before the silent flush and compaction path.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence verification path for the missing behavior: inspect the config type/schema and memory-flush runner on current main, where the requested keys are absent and the flush remains `silentExpected: true`.

## Solution Assessment

Unclear. The requested direction is plausible, but adding model-visible lifecycle prompting is not a narrow cleanup because maintainers need to choose between a core config knob, broader #38520 lifecycle work, or a new extension seam.
