## Summary

Keep open. Current main still lacks the requested agent-visible pre-compaction warning: the `memoryFlush` config surface has no `visibleWarning` or `warningPrompt`, the strict schema would reject those keys, and the memory-flush runner still uses a silent embedded run. The related shipped `notifyUser` behavior only sends user-channel compaction notices, which does not satisfy the requested model-visible warning moment.

## What This Changes

Add optional `memoryFlush.visibleWarning` and `warningPrompt` behavior so an agent receives a visible pre-compaction warning before the existing silent memory flush runs.

## Best Possible Solution

Maintainers should choose the lifecycle boundary, then implement either a validated, deduped agent-visible pre-compaction warning under `agents.defaults.compaction.memoryFlush` or a documented runtime/plugin extension point that can provide the same warning while preserving silent memory flush and existing user-channel notices.
