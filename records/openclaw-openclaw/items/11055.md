## Summary

Keep open. Current main still lacks the requested agent-visible pre-compaction warning: `memoryFlush` has no `visibleWarning` or `warningPrompt` config, the strict schema would reject those keys, and the flush runner still launches a silent memory-triggered embedded run. The shipped adjacent `notifyUser` path only posts start/completion notices to the user channel, so it does not solve the requested model-visible warning moment.

## What This Changes

Add optional `memoryFlush.visibleWarning` and `memoryFlush.warningPrompt` behavior so the agent receives a visible pre-compaction warning before auto-compaction while preserving the existing silent memory flush.

## Best Possible Solution

Maintainers should choose the product boundary, then implement either a validated, deduped `agents.defaults.compaction.memoryFlush.visibleWarning` / `warningPrompt` surface that preserves the existing silent flush, or a documented plugin/runtime extension point that can safely add an agent-visible pre-compaction warning.
