## Summary

Keep open. Current main still lacks the requested agent-visible pre-compaction warning: the memory-flush config has no `visibleWarning` or `warningPrompt`, the strict schema would reject those keys, and the memory-flush turn remains a silent embedded run. The shipped adjacent `notifyUser` work is useful user-channel UX, but it does not create the model-visible warning moment requested here.

## What This Changes

The issue requests an optional `memoryFlush.visibleWarning` / `warningPrompt` configuration that sends a visible pre-compaction warning to the agent so it can checkpoint in-progress work before context compaction.

## Best Possible Solution

Maintainers should choose the lifecycle boundary, then implement either a validated, deduped agent-visible pre-compaction warning under `agents.defaults.compaction.memoryFlush` or a documented runtime/plugin extension point that can safely add the warning while preserving the existing silent memory flush and user-channel `notifyUser` behavior.

## Reproduction Assessment

Not applicable as a feature request. There is a high-confidence verification path for the missing behavior by inspecting current main: the proposed config keys are absent, schema validation is strict, and memory flush remains `silentExpected: true`.

## Solution Assessment

Unclear. The requested config knob is plausible, but model-visible lifecycle prompting should be decided with the broader pre-compaction lifecycle design in #38520 before queueing an automated repair.
