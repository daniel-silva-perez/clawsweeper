## Summary

Keep open. Current main still lacks the requested agent-visible pre-compaction warning: the config type and strict schema do not accept `visibleWarning` or `warningPrompt`, the memory-flush runner still invokes a silent embedded memory turn, and the merged `notifyUser` work only sends user-channel compaction notices. Recent maintainer-facing comments also explicitly kept this open for product/API boundary judgment, with broader lifecycle design still tracked separately in #38520.

## What This Changes

The issue requests optional `memoryFlush.visibleWarning` and `warningPrompt` configuration so the agent receives a visible pre-compaction warning before the existing silent memory-flush turn.

## Best Possible Solution

Maintainers should choose the boundary, then either add a validated, deduped agent-visible warning surface under `agents.defaults.compaction.memoryFlush` or expose/document a runtime/plugin hook that can inject the warning while preserving the existing silent memory-flush turn and user-channel compaction notices.
