## Summary

Keep open. Current main still does not implement the requested agent-visible pre-compaction warning: the memory-flush config lacks visibleWarning/warningPrompt, the schema is strict, and the runtime memory flush remains a silent embedded run. The adjacent notifyUser work only sends fixed status notices to the user channel, so it does not satisfy the model-visible warning moment requested here.

## What This Changes

The issue requests optional memoryFlush config for a visible agent warning before compaction, including visibleWarning and warningPrompt fields.

## Best Possible Solution

Maintainers should choose the boundary, then implement either a validated, deduped agent-visible pre-compaction warning under memoryFlush or a documented runtime/plugin extension point that can safely add the warning while preserving the existing silent memory flush and user-channel notifyUser behavior.
