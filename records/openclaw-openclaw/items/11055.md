## Summary

Keep open. Current main still lacks the requested agent-visible pre-compaction warning: the memoryFlush config and strict schema do not accept visibleWarning or warningPrompt, the runtime memory flush is still a silent embedded run, and the existing notifyUser path only sends user-channel status notices. Prior discussion also keeps this open for maintainer product/API boundary judgment, with broader lifecycle work tracked separately in #38520.

## What This Changes

Request an optional memoryFlush.visibleWarning/warningPrompt config that injects a visible agent/model-facing warning before auto-compaction so the agent can checkpoint in-progress work.

## Best Possible Solution

Maintainers should choose the boundary, then implement either a validated, deduped agent-visible warning under agents.defaults.compaction.memoryFlush or a documented runtime/plugin extension point that can inject that warning before the soft threshold while preserving silent memoryFlush and user-channel notifyUser behavior.
