## Summary

Current main still does not implement the requested agent-visible pre-compaction warning. The existing `memoryFlush` path remains a silent `NO_REPLY` housekeeping turn, and the merged `notifyUser` work sends user-channel notices rather than a model-visible `memoryFlush.visibleWarning` / `warningPrompt` turn.

## Best Possible Solution

Keep this issue open for maintainer product/API judgment. A core implementation would add validated `agents.defaults.compaction.memoryFlush.visibleWarning` and `warningPrompt` config, document it separately from `notifyUser`, inject one deduped agent-visible warning before auto-compaction at the soft threshold, preserve the existing silent memory flush, and add focused config/runtime/docs tests. If maintainers decide this belongs outside core, first add or document a plugin/runtime API that can safely inject an agent-visible pre-threshold warning.
