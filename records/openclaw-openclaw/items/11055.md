## Summary

Keep open. Current main still lacks the requested agent-visible pre-compaction warning: the memory-flush config has no `visibleWarning` or `warningPrompt`, strict validation would reject those keys, and the memory-flush turn remains silent. The merged `notifyUser` work is adjacent user-channel UX, but it does not give the model a visible warning moment; the broader lifecycle/API shape still needs maintainer judgment, especially alongside #38520.

## What This Changes

The issue requests optional compaction memory-flush settings that inject an agent-visible warning prompt before auto-compaction while preserving the existing silent memory flush.

## Best Possible Solution

Maintainers should choose the compaction lifecycle boundary, then either add a validated, deduped agent-visible pre-compaction warning under memory flush or document/expose a runtime/plugin extension point that can provide the warning while preserving silent memory flush and existing user-channel notices.
