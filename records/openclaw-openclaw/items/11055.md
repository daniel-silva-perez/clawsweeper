## Summary

Keep open. Current main still lacks the requested agent-visible pre-compaction warning: the memory-flush config has no `visibleWarning` or `warningPrompt`, the strict config schema would reject those keys, and the flush path remains a silent embedded run. The merged `notifyUser` work is adjacent user-channel UX, but it does not satisfy this issue’s model-visible warning moment.

## What This Changes

Add an optional `memoryFlush.visibleWarning` and `warningPrompt` style config so an agent can receive a visible warning before compaction instead of only a silent memory flush.

## Best Possible Solution

Maintainers should define the lifecycle boundary, then either add a validated, deduped agent-visible warning surface under compaction memory flush or document a runtime/plugin extension point that can provide the warning while preserving the existing silent flush and user-channel notices.

## Reproduction Assessment

Not applicable as a feature request. There is a high-confidence verification path for the missing behavior by inspecting current main: the proposed config fields are absent, validation is strict, and the memory-flush turn is still silent.
