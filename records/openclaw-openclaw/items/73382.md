## Summary

Keep this PR open, but not merge-ready as-is. Current main still has the Bedrock Claude 3.x context-window gap, while the PR's new family fallback overclaims Claude 4 coverage because its regex misses the actual Claude 4 Bedrock ID shape; maintainers also need to choose how this relates to the open configurable-override alternative.

## What This Changes

The branch adds missing Amazon Bedrock Claude 3/3.5/Opus 4 context-window entries, a discovery family fallback, regression tests, and a changelog entry for the linked Bedrock context-window bug.

## Best Possible Solution

Keep this PR open for maintainer review alongside #73328 and #73959. If the built-in fallback route is chosen, widen the family matcher to cover actual Claude 4 Bedrock ID shapes, add a Claude 4 fallback regression test, preserve explicit 1M row precedence, and rerun the focused amazon-bedrock/changed validation before merge.

## Reproduction Assessment

Yes. A static reproduction is high-confidence: on current main, an unlisted ID such as `anthropic.claude-3-5-sonnet-20241022-v2:0` is absent from `KNOWN_CONTEXT_WINDOWS`, `resolveKnownContextWindow` returns undefined, and discovery falls through to the 32K default.

## Solution Assessment

No, not as-is. The explicit Claude 3.x rows are a narrow fix for the observed bug, but the family fallback should either correctly cover real Claude 4 Bedrock naming or be replaced/combined with the configurable override approach from #73959.
