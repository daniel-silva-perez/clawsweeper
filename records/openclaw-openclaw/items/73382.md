## Summary

Keep this PR open, but it is not merge-ready as written. Current main still has the Bedrock Claude context-window gap, and the PR is a focused implementation candidate, but its family fallback does not actually cover the real Claude 4 Bedrock ID shapes it claims to handle.

## What This Changes

The PR adds missing Amazon Bedrock Claude 3/3.5/Opus 4 context-window entries, a Claude family fallback, regression tests, and a changelog entry for the Bedrock discovery context-window bug.

## Best Possible Solution

Keep this PR open for maintainer review alongside #73328 and #73959. If the built-in fallback direction is chosen, revise the matcher to cover real Claude 4 Bedrock IDs or narrow the fallback scope to Claude 3.x, add a Claude 4 regression test if the claim remains, and preserve explicit 1M row precedence.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: an unlisted ID such as `anthropic.claude-3-5-sonnet-20241022-v2:0` is absent from `KNOWN_CONTEXT_WINDOWS`, `resolveKnownContextWindow` returns undefined, and discovery falls through to the 32000 default.

## Solution Assessment

No, not as-is. The explicit Claude 3.x rows are a reasonable narrow fix, but the family fallback should correctly cover the Claude 4 ID shapes it documents or be replaced by the configurable override approach being reviewed in #73959.
