## Summary

Keep open. Current main still preserves the Secure Context/WebCrypto gate and the documented rejection policy for remote plain-HTTP Control UI sessions without device identity; only the bypass-scope and implied operator-scope parts are already implemented. The remaining behavior change is real, but it needs maintainer product/security judgment rather than cleanup closure or an automated patch.

## What This Changes

The issue asks OpenClaw to generate browser Control UI device identity without depending on Secure Context/WebCrypto, preserve scopes when the explicit device-auth bypass is enabled, and normalize implied operator scopes.

## Best Possible Solution

Maintainers should make an explicit product/security call on non-secure LAN HTTP. If supported, add a browser-safe hashing path with explicit UI dependency metadata, narrow or remove the Secure Context identity-generation guard, update focused UI/gateway tests, and revise public Control UI docs. If not supported, keep the shipped bypass and scope-normalization fixes, document HTTPS/Tailscale Serve/trusted-proxy/break-glass as intended paths, and use #32473 as the broader UX follow-up.
