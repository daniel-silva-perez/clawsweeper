## Summary

Keep open. Current main still has the agent-run assistant-media supplement path that can append a gateway-injected transcript message from text fallback when no embeddable media resolves, and two open, unmerged PRs already target this issue with closing intent.

## What This Changes

Control UI/WebChat should stop persisting duplicate text-only assistant replies through the assistant-media delivery path after TTS/media references fail to resolve.

## Best Possible Solution

Land one narrow gateway fix that appends the agent-run assistant-media transcript only when resolved non-text media content exists, with regression coverage for stale local audio/media references and preserved command/non-agent TTS fallback behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is an agent-run final payload with visible text plus a stale trusted local audio mediaUrl; current main can resolve no media, then append a text fallback with an :assistant-media idempotency key.

## Solution Assessment

Yes. Guarding the supplemental assistant-media append on persisted non-text media content is the narrowest maintainable fix; the normal model transcript should carry the text, while non-agent command/TTS fallback remains separate.
