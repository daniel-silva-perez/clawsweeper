## Summary

Keep this PR open. Current main still accepts Feishu/Lark announce cron jobs without `delivery.to`, the linked bug #39599 remains open, and this PR is still not mergeable as submitted because its head replaces the cron CLI registration module with `404: Not Found`.

## Best Possible Solution

Keep this PR open for contributor repair, or supersede it with a clean PR linked to #39599. The fix should restore the cron CLI module, add service/gateway validation for Feishu/Lark announce jobs when no explicit or deterministic target exists, optionally add CLI fail-fast feedback, and include create/update regression coverage. Security review found no new third-party code execution or supply-chain changes in the listed diff, but the accidental CLI file replacement is a blocking functional regression.
