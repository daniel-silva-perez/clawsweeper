## Summary

Keep this PR open. Current main still registers Xiaomi MiMo Pro/Omni through the generic OpenAI-compatible path, and that parser maps `reasoning_content` to `thinking` blocks. I did not find a Xiaomi stream wrapper, payload patch, or regression test on main that implements the PR's intended behavior. The linked bug being closed is not enough to make this unmerged PR obsolete.

## Best Possible Solution

Keep the PR open for maintainer review or a refreshed patch. The likely best fix remains Xiaomi-owned: add a scoped stream compatibility wrapper or equivalent provider hook for `mimo-v2-pro` and `mimo-v2-omni`, with regression tests for both models and non-target Xiaomi models. If newer shared helpers can express the same thinking-only-final-to-text rule, factor it there and wire Xiaomi through that SDK surface rather than duplicating provider-local logic.
