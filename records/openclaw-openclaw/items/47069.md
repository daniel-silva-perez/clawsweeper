## Summary

Keep this PR open, but do not merge it as-is. Current main still treats Control UI `/model status` and `/model list` as model-setting arguments, so the user bug is real; the PR fixes the error path by collapsing those subcommands into the bare summary path, which does not preserve the documented shared `/model list` and `/model status` semantics.

## What This Changes

The PR adds a Control UI model-info subcommand set, routes `/model status`, `/model list`, and `/model info` to the existing bare `/model` info display, and adds slash-command executor tests including a normal model-selection regression.

## Best Possible Solution

Refresh the Control UI local `/model` implementation so it matches the shared slash-command contract: bare `/model` shows the summary, `/model list` follows the model-browser/list semantics, `/model status` shows detailed model/auth/endpoint status, normal model selection continues to patch the session, and `info` is either documented across surfaces or left available as a normal model id.
