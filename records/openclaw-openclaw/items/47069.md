## Summary

Keep open. Current main still has the Control UI `/model status` and `/model list` bug, so this PR is not obsolete, but the proposed branch should not merge as-is because it routes documented `list` and `status` modes into the bare summary path and reserves undocumented `info`.

## What This Changes

The PR adds a Control UI `MODEL_INFO_SUBCOMMANDS` guard and slash-command executor tests so `/model status`, `/model list`, and `/model info` show the existing model-info summary instead of calling `sessions.patch`, plus a regression test for real model names.

## Best Possible Solution

Refresh the Control UI local `/model` implementation so bare `/model` keeps the current summary, `/model list` follows the shared model-browser/list behavior, `/model status` shows detailed model/auth/endpoint diagnostics, normal model selection still calls `sessions.patch`, and `info` is only reserved if the shared command contract and docs add it consistently.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: `/model status` parses as local command `model` with args `status`, and `executeModel` sends any non-empty args to `sessions.patch`.

## Solution Assessment

No. The PR prevents the immediate bad patch call, but it is not the best fix because it collapses `list` and `status` into the bare summary path and reserves undocumented `info`; the safer path is to implement each documented mode explicitly while preserving current main's catalog and session-patch behavior.
