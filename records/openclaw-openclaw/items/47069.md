## Summary

Keep open. Current main still sends non-empty Control UI `/model` arguments to `sessions.patch`, so the underlying bug remains, but this PR should not merge as-is because it collapses documented `list` and `status` behavior into the bare summary path and reserves undocumented `info`.

## What This Changes

The PR adds a Control UI `MODEL_INFO_SUBCOMMANDS` guard for `status`, `list`, and `info`, routes those tokens to the existing model-info branch, and adds node tests including a normal model-selection regression case.

## Best Possible Solution

Align the Control UI local `/model` implementation with the shared command contract: bare `/model` keeps the current summary, `/model list` follows the model browser/list behavior, `/model status` shows detailed model/auth/endpoint diagnostics, ordinary model selections still call `sessions.patch`, and `info` is only reserved if the shared command contract and docs add it consistently.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: `/model status` parses as local command `model` with args `status`, and `executeModel` sends any non-empty args to `sessions.patch`.

## Solution Assessment

No. The PR prevents the immediate bad patch call, but it is not the best fix because it maps documented `list` and `status` modes to the bare summary and reserves undocumented `info`; the safer fix is explicit mode handling that matches the shared directive and docs.
