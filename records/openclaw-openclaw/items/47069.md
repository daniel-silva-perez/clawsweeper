## Summary

Keep open. Current main still routes Control UI `/model status` and `/model list` through the local model-setting path, so the PR is not obsolete, but the branch should not merge as-is because it collapses the documented `list` and `status` behaviors into the bare summary path and adds an undocumented `info` reservation.

## What This Changes

The PR adds a Control UI `MODEL_INFO_SUBCOMMANDS` set for `status`, `list`, and `info`, routes those `/model` arguments to the existing info summary path, and adds slash-command executor regression tests including normal model selection.

## Best Possible Solution

Refresh the Control UI local `/model` implementation so bare `/model` keeps the current summary, `/model list` follows the shared model-browser/list behavior, `/model status` shows detailed model/auth/endpoint diagnostics, normal model selection still calls `sessions.patch`, and tests cover each branch.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: `/model status` parses as local command `model` with args `status`, and `executeModel` sends any non-empty args to `sessions.patch`.

## Solution Assessment

No. The PR prevents the immediate bad patch call, but it is not the best fix because it collapses `list` and `status` into the bare summary path and reserves undocumented `info`.
