## Summary

Keep this PR open. Current main still has the Control UI local `/model` handler sending any non-empty argument to `sessions.patch`, so the linked Web UI bug is not implemented on main. The PR is useful source work, but its current approach should be refreshed because today’s shared directive/docs distinguish `/model list` from `/model status` rather than collapsing both to the bare model-info output.

## What This Changes

The PR adds a Control UI `MODEL_INFO_SUBCOMMANDS` set for `status`, `list`, and `info`, changes local `/model` handling to show model info for those arguments, and adds regression tests that they do not call `sessions.patch`.

## Best Possible Solution

Refresh the Control UI local `/model` implementation so it matches the current shared directive contract: bare `/model` shows the summary, `/model list` follows the model-browser/list semantics, `/model status` shows detailed model/auth/endpoint status, and normal provider-qualified model selection continues to update the session override. Add focused tests for the aliases and for a real model selection path, then land this PR or a replacement that closes #46894.
