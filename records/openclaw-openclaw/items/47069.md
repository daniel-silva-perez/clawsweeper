## Summary

Keep open. The Control UI bug is still real on current main and this PR is paired with the open bug report #46894, but the proposed patch should not merge as-is because it collapses `/model list` and `/model status` into the bare summary path instead of preserving the documented shared command semantics.

## What This Changes

The PR adds a Control UI model-info subcommand set for `status`, `list`, and `info`, routes those arguments to the existing bare `/model` info display, and adds slash-command executor tests including a normal model-selection regression.

## Best Possible Solution

Refresh the Control UI local `/model` implementation so bare `/model` shows the summary, `/model list` follows the model-browser/list behavior, `/model status` shows detailed model/auth/endpoint diagnostics, normal model selection continues to patch the session, and `info` is only reserved if it is added consistently across shared commands and docs.

## Reproduction Assessment

Yes. Current main provides a source-level reproduction path: Control UI parses `/model` as a local command and `executeModel` sends every non-empty argument to `sessions.patch`, so `/model status` and `/model list` are treated as model refs before any shared directive handling can run.

## Solution Assessment

No. The PR fixes the immediate bad `sessions.patch` call for a few tokens, but it is not the best fix because it loses the distinct documented behavior for `list` and `status` and reserves undocumented `info` as a model alias.
