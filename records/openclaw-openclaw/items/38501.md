## Summary

Keep open: the checked-out main branch still routes TUI Ctrl+C through clear/warn/exit handling without active-run awareness, while Escape and `/abort` use the separate abort path. A focused open PR with closing syntax and a broader open PR are already in play, so this should stay open for maintainer review rather than be auto-closed or queued as another repair.

## What This Changes

The issue asks for TUI Ctrl+C feedback to clarify that an active run continues unless stopped with Escape or `/abort`.

## Best Possible Solution

Choose the intended Ctrl+C policy, then land the matching TUI patch with focused regression coverage.
