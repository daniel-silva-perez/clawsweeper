## Summary

Current main still lacks the requested active-run Ctrl+C hint in the TUI. The Ctrl+C path still only clears/warns/exits, while Esc and `/abort` remain the active-run abort paths. This issue should stay open because it is already paired with open implementation work, especially the focused #38502 and the broader Ctrl+C policy alternative #44223.

## Best Possible Solution

Keep this issue open until #38502 or an equivalent focused patch lands with a regression test for active-run Ctrl+C status text. If maintainers prefer Ctrl+C to abort active runs instead, reconcile that direction with #44223, update docs/tests accordingly, and supersede the text-only path explicitly.
