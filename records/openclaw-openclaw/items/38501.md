## Summary

Keep open: current main still routes TUI Ctrl+C through clear/warn/exit feedback without active-run-aware status text, while Esc and `/abort` use the separate abort path. The reporter linked the focused fix PR #38502, and #44223 raises the broader behavior question, so this needs maintainer review rather than cleanup closure or another repair lane.

## What This Changes

The issue requests clearer TUI Ctrl+C feedback during an active run, stating that the run is still active and can be stopped with Esc or `/abort`.

## Best Possible Solution

Choose the intended TUI Ctrl+C policy and land one matching patch with focused regression coverage. If Ctrl+C remains clear/exit, the fix should add the active-run status hint; if Ctrl+C should abort, the broader behavior needs docs and tests aligned with that policy.
