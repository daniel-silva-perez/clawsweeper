## Summary

Keep open: current main still routes TUI Ctrl+C through clear/warn/exit status handling without checking active run state or showing the requested Esc/`/abort` hint, while Escape and `/abort` use the separate abort path. The issue is also paired with open PR #38502 using closing syntax, with #44223 proposing a broader Ctrl+C abort policy, so this should be resolved by maintainer review of the linked PRs rather than auto-closed or queued as a duplicate repair.

## What This Changes

The issue requests a TUI Ctrl+C status hint that says an active run is still running and can be stopped with Esc or `/abort`, without changing Ctrl+C abort semantics.

## Best Possible Solution

Choose the intended TUI Ctrl+C policy and land one matching patch with focused regression coverage. If Ctrl+C remains clear/exit, #38502 or an equivalent status-hint patch should close this after merge; if Ctrl+C should abort active runs, reconcile #44223 with docs and tests.
