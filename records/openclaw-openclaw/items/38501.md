## Summary

Keep open: current main still routes TUI Ctrl+C through clear/warn/exit handling without active-run awareness, and the issue is paired with a focused open closing PR plus a broader open Ctrl+C behavior PR that need maintainer review before this issue is closed.

## What This Changes

The issue requests clearer TUI Ctrl+C feedback during an active run so users know the run is still active and can use Esc or `/abort` to stop it.

## Best Possible Solution

Choose the intended TUI Ctrl+C policy and land one matching patch with focused regression coverage. If Ctrl+C remains clear/exit, #38502 or an equivalent text-only hint should close this after merge; if Ctrl+C should abort active runs, #44223 needs maintainer product review plus aligned tests and docs.

## Reproduction Assessment

Yes. A high-confidence source reproduction path exists: during an active run, Ctrl+C reaches `handleCtrlC` and `resolveCtrlCAction`, which do not inspect `activeChatRunId`, while Esc and `/abort` route through `abortActive`.

## Solution Assessment

Unclear pending maintainer policy. The narrowest fix for the reported UX mismatch is the focused status-text hint, while changing Ctrl+C to abort active runs is a broader behavior change that should be reviewed deliberately.
