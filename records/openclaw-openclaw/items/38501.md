## Summary

Keep open. Current main still has Ctrl+C wired to clear/warn/exit status handling without checking active run state or showing the requested Esc/`/abort` hint, while Escape and `/abort` use the separate abort path. The issue is also paired with open PR #38502 using closing syntax, plus broader open PR #44223 for an alternate Ctrl+C abort policy, so it should remain open for maintainer review rather than be auto-closed or queued as a separate repair.

## What This Changes

The issue requests a TUI Ctrl+C status-text clarification so users know an active run is still running and can be stopped with Esc or `/abort`.

## Best Possible Solution

Choose the intended TUI Ctrl+C policy and land one matching patch with focused regression coverage. If Ctrl+C remains clear/exit, #38502 or an equivalent status-hint patch should close this after merge; if Ctrl+C should abort active runs, reconcile #44223 with docs and tests.
