## Summary

Keep open: current main still has Ctrl+C clear/warn/exit feedback without active-run awareness, and the issue is paired with an open focused closing PR plus a broader open behavior PR. Maintainer review should choose between the text-only hint and the broader Ctrl+C abort semantics before this is closed.

## What This Changes

The issue requests clearer TUI Ctrl+C feedback that says an active run is still running and can be stopped with Esc or /abort.

## Best Possible Solution

Choose the intended TUI Ctrl+C policy and land one matching patch with focused regression coverage. If Ctrl+C remains clear/exit, #38502 or an equivalent text-only change should add the active-run hint; if Ctrl+C should abort active runs, #44223 needs maintainer product review plus aligned tests and docs.
