## Summary

Current main still has the reported TUI Ctrl+C UX gap: Ctrl+C resolves only to clear, warn, or exit and does not include active-run-aware status text. The issue should stay open because #38502 is an open focused PR with closing syntax for this issue, while #44223 is an open broader policy alternative that would change Ctrl+C to abort active runs.

## What This Changes

Request TUI Ctrl+C feedback that explicitly tells users when a run is still active and how to stop it with Esc or `/abort`.

## Best Possible Solution

Choose the intended Ctrl+C policy, then land the matching TUI patch with focused regression coverage. If maintainers want current keybinding semantics preserved, #38502 or an equivalent text-only patch should close this after merge; if Ctrl+C should abort active runs, reconcile and land the broader #44223-style behavior with docs/tests updated accordingly.
