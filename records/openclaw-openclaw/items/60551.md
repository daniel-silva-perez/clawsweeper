## Summary

Keep this PR open. Current main still lacks a plain-text leaked-reasoning preamble cleaner, and the related Gemini/output-filter reports show the problem is real, but this patch is not merge-ready because it adds broader structural scrubbing that can delete legitimate fenced JSON examples and has an inconsistent tilde-fence path in the preamble heuristic.

## What This Changes

The PR adds leaked-reasoning preamble detection plus structural-contamination regex cleanup to the shared assistant visible-text delivery sanitizer, with new unit coverage in the sanitizer test file.

## Best Possible Solution

Keep the last-mile delivery sanitizer direction, but narrow the PR to leaked reasoning/meta-planning removal, remove or separately design the broad structural-contamination scrubber, align all fenced-code handling with the shared code-region parser, and add regression tests for the linked Gemini/Telegram cases plus representative false positives.

## Reproduction Assessment

Yes. The linked reports provide concrete leaked-preamble examples, and the PR tests show deterministic sanitizer inputs; the review findings are also reproducible by passing a fenced JSON schema example or a tilde-fenced answer through the proposed delivery sanitizer.

## Solution Assessment

No, not as-is. The shared delivery sanitizer is a plausible seam, but the current patch mixes in an overbroad structural scrubber and has an inconsistent fence guard; a narrower preamble cleaner with explicit leak markers and code-region-consistent tests is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve fenced schema examples:** `src/shared/text/assistant-visible-text.ts:684`
  - body: The new structural scrubber removes every fenced `json` block containing a `"schema"` key before final delivery. Normal assistant replies often include plugin manifests or config schema snippets in fenced JSON, so this would silently delete valid user-facing examples; narrow this to the explicit leaked metadata envelope shape or keep structural cleanup out of this PR.
  - confidence: 0.9
- **[P2] Reject tilde fences as visible suffix starts:** `src/shared/text/assistant-visible-text.ts:584`
  - body: `findCodeRegions` and `hasFencedCode` both recognize `~~~` fences, but `looksUserFacingStart` only rejects suffixes starting with backtick fences. With two heuristic-triggering opening lines followed by a tilde-fenced example, the cleaner can strip the lead-in while the equivalent backtick case is preserved; treat `~~~` the same way and add coverage.
  - confidence: 0.82
