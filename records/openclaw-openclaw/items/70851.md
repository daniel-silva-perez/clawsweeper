## Summary

Current main does not implement this PR's intended picker behavior. The chat session picker still filters only global/unknown and cron rows before adding session options, current tests still expect subagent rows to appear as picker options, and the UI session row type does not carry the gateway's existing `spawnDepth` metadata. The PR remains a useful implementation candidate, but it needs maintainer review/refinement because the proposed UI-only heuristic can hide legitimate flexible keys and does not provide the source-side internal visibility contract described in the PR body.

## Best Possible Solution

Keep this PR open and revise it toward the stated minimal end-state: gateway/session rows should expose an explicit internal visibility/classification for subagent, ACP, and node-backed runtime sessions; the normal chat picker should hide those rows by default while still showing the currently active internal session; and the Sessions/admin/debug surface should keep a way to inspect internal rows. Use parsed session-key helpers or source metadata instead of broad substring matching, and leave the broader picker split tracked separately by #64121.
