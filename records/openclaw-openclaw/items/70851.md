## Summary

Keep this PR open. Current main still does not implement the intended behavior: the chat picker only filters global/unknown and cron rows, current picker tests still keep subagent rows visible, and the gateway/UI contract still lacks an explicit internal visibility classification for subagent, ACP, and node-backed runtime sessions. The PR is a useful implementation candidate, but the final fix should move from UI-side string heuristics toward source-side classification plus picker filtering and admin/debug visibility preservation.

## Best Possible Solution

Revise this PR or replace it with a narrow implementation that adds an explicit gateway session-row internal classification for subagent, ACP, and node-backed runtime sessions, propagates that contract through protocol/UI types and session events, hides inactive internal rows in the normal chat picker while keeping the active internal row visible, and keeps the admin/debug Sessions surface able to show internal rows. Keep the broader picker split tracked by #64121.
