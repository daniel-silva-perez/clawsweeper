## Summary

Keep this PR open. Current main still shows internal runtime sessions in the chat picker, and the PR is a useful implementation candidate, but the current patch is not safe to merge as-is because it uses whole-key substring matching and does not establish the gateway-side visibility/classification contract described in the PR body.

## What This Changes

This PR adds UI-side filtering for subagent, ACP, and node-backed sessions in the chat picker, keeps the selected internal session visible, adds `spawnDepth` to the UI session row type, and extends focused picker tests.

## Best Possible Solution

Add explicit gateway/session-row classification such as internal visibility for subagent, ACP, and node-backed runtime sessions; have the normal chat picker hide inactive internal rows by default; keep the currently selected internal row visible; and preserve an admin/debug Sessions path that can still list internals. Keep the broader Agent versus Agent Sessions split tracked separately in #64121.

## Reproduction Assessment

Yes. A focused source-level reproduction is to feed `resolveSessionOptionGroups` a main row plus an `agent:main:subagent:...` row on current main: the existing code only filters global/unknown and cron rows, and the current test suite still expects the subagent label to appear.

## Solution Assessment

No. The PR is directionally useful as an interim mitigation, but the best maintainable fix is source-side classification plus picker filtering based on that contract; if a key fallback remains, it should use parsed session-key helpers rather than whole-string substring matching.
