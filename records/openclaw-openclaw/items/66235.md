## Summary

Current main still does not expose `stopReason`, `hasToolCalls`, `agentContext`, or equivalent completion metadata to `message_sending` hook handlers. The requested signal exists in agent run metadata, but the outbound hook contract and delivery metadata remain delivery/routing-only. The related PR #39761 is still an open implementation candidate in the provided context, not shipped behavior.

## Best Possible Solution

Keep this issue open. Implement a backward-compatible core hook contract change that threads agent completion metadata from `EmbeddedPiRunMeta` through agent reply delivery, route reply/outbound delivery params, and `applyMessageSendingHook()`. Expose optional `stopReason` and `hasToolCalls`, or a broader typed `agentContext`, on `PluginHookMessageSendingEvent`; add focused tests for agent command/channel delivery hook payloads and update plugin hook docs. If #39761 is the chosen implementation, review and land it first, then close this issue after current main exposes the requested signal.
