## Summary

Keep open. Current main and the latest tagged release still have the pre-fix temp-directory repair behavior, while related #66867 remains an open production race report; this PR is a narrow implementation candidate, though maintainers should confirm it covers the adjacent already-secure re-stat race before landing.

## What This Changes

This PR changes the temp-dir permission repair path to catch selected chmod permission/removal errors, re-check whether the directory is now trusted, and adds a regression test asserting silent success when another process tightened permissions.

## Best Possible Solution

Land or refine a narrow temp-dir repair fix that treats concurrent repair as success only after revalidating ownership, symlink status, mode, and access; cover both chmod-error and already-secure re-stat races without weakening the trust checks.
