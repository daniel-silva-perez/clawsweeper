## Summary

Keep this PR open. Current main still does not implement the PR’s intended `session_status` behavior: the tool delegates to the shared status builder without any timestamp payload, the shared builder does not accept or forward `timeLine`, and the only renderer support is an optional `args.timeLine` that current status callers never provide. The PR remains a plausible focused implementation candidate, but it needs a rebase against the current status split.

## Best Possible Solution

Rebase and rework this PR against current main: add or reuse a small timezone-aware local ISO formatter, pass a machine-readable local ISO plus UTC timestamp into the shared status card path used by `session_status`, and add a targeted regression test that pins the clock and configured user timezone. Keep the change scoped away from system prompt injection, cron scheduler behavior, auth, and other runtime surfaces.
