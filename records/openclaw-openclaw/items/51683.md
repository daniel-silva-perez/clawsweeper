## Summary

Keep this PR open. Current main still does not implement the PR’s intended `session_status` behavior: the status renderer can display an optional `timeLine`, but `session_status` and the shared status builder do not compute or pass a machine-readable local ISO timestamp plus UTC time. The PR remains a focused implementation candidate, though it needs a rebase/rework against the current shared status modules.

## Best Possible Solution

Rebase and rework this PR against current main. Add or reuse a timezone-aware formatter that produces a local ISO timestamp with offset plus UTC ISO time, thread that value through the shared status-card path used by `session_status` (and `/status` if the docs continue to describe the shared card), and add a pinned-clock regression test for a configured non-UTC user timezone. Keep the change scoped away from scheduler behavior, system prompt injection, auth, CI, dependencies, and release surfaces.
