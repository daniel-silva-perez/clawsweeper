## Summary

Keep open for maintainer review. Current main still has the abort/timeout payload leak path, while this PR is a focused implementation with regression coverage, green PR checks, and no blocking review findings.

## What This Changes

The PR changes embedded run payload assembly so aborted assistant turns surface formatted error payloads and suppress partial answer/reasoning text, with regression tests for timeout and reply-tag leakage.

## Best Possible Solution

Review and land this PR or an equivalent abort-aware guard in buildEmbeddedRunPayloads, preserving the existing media/raw-answer selection for normal turns while preventing aborted assistant text and reasoning artifacts from reaching channel delivery.

## Reproduction Assessment

Yes. A focused static reproduction is to call buildEmbeddedRunPayloads on current main with non-empty assistantTexts, lastAssistant.stopReason set to "aborted", and a timeout-shaped errorMessage; current main can emit the partial assistant text instead of only a clean error payload.

## Solution Assessment

Yes. The PR fixes the leak at the final payload assembly decision point, reuses the existing assistant error formatter, and keeps the richer answer selection logic intact for non-aborted turns.
