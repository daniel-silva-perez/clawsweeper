## Summary

Current main still lacks the PR's intended start-args fallback for embedded read-image media. The linked same-author bug remains open, and the overlapping PR #46268 is another unmerged implementation candidate rather than a shipped replacement.

## Best Possible Solution

Keep this PR open for maintainer review or consolidation with #46268. The best fix is a narrow current-main rebase that makes media extraction start-args-aware, or makes the read tool attach structured `details.media`/`details.path` for successful image reads, while preserving the current exact-tool-name and non-external-result trust gates. Add regression coverage for `path`, `file_path`, `filePath`, and queued outbound media delivery, then resolve the linked bug once the chosen PR lands.
