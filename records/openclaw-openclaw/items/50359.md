## Summary

Keep this PR open. Current main still lacks the PR’s intended fallback from embedded read-image results to original tool start args, the linked bug report remains open with concrete reproduction evidence, and another overlapping open PR is not merged or canonical enough to make this one obsolete.

## Best Possible Solution

Keep this PR open for maintainer review. The best fix is a narrow current-main implementation that either passes stored `tool_execution_start` args into trusted read-result media extraction, or has `createOpenClawReadTool` attach structured `details.media` / `details.path` before extraction. Keep regression coverage for `path`, `file_path`, `filePath`, and final queued outbound media delivery, then close #41744 after the fix lands or consolidate with #46268 if that PR becomes canonical.
