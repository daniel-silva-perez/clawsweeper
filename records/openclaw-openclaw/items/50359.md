## Summary

Keep this PR open. Current main still lacks the intended fallback from embedded read-image tool results to the original tool start args, and the paired same-author bug #41744 remains open. The related #46268 is another open implementation candidate, not a shipped replacement.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase a narrow fix onto current main by making media extraction start-args-aware, or by having createOpenClawReadTool attach structured details.media/details.path for successful image reads. Add regression coverage for path, file_path, filePath, and final queued outbound media delivery, then resolve #41744 or consolidate with #46268 if that becomes canonical.
