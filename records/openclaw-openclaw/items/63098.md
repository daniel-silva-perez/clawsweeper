## Summary

Keep open. Current main still matches the reported Discord voice failure mode: `/vc join` waits on a fixed 15s Ready transition and destroys on failure, while the Disconnected recovery handler still gives Signalling/Connecting only 5s before clearing the session and destroying without a warning. Multiple recent issue comments confirm the behavior across releases and platforms, and no open PR was found that specifically fixes this issue.

## What This Changes

The issue requests a Discord voice fix so joins and reconnects tolerate slower voice handshake/recovery paths instead of aborting or silently disconnecting after the 2026.4.x timeout changes.

## Best Possible Solution

Keep this issue open and land a focused Discord plugin fix that lengthens or makes configurable the initial Ready wait, lengthens and logs the Disconnected recovery grace path, adds regression coverage for both abort/destroy paths, and updates Discord voice config/docs/changelog if new `channels.discord.voice.*TimeoutMs` fields are introduced.
