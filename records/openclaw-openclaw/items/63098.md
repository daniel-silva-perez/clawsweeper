## Summary

Current main still matches the reported Discord voice failure family: `/vc join` waits for `@discordjs/voice` Ready with a fixed 15s timeout, destroys the connection on abort, and the post-join disconnected handler still gives reconnect signalling only 5s before silently destroying the session. The requested configurable timeout/retry/logging path is absent from the strict Discord voice config, UI hints, generated metadata, and docs, while related reports remain open.

## Best Possible Solution

Keep this issue open and fix the Discord plugin voice join/recovery path. A focused fix should add regression coverage for the initial Ready abort and the disconnected recovery timeout/log path, then use a longer bounded default and/or a documented `channels.discord.voice.connectTimeoutMs` control without moving owner-specific behavior into core. If a new config knob lands, update the strict zod schema, exported types, plugin UI hints/generated metadata, Discord docs, and changelog, then reconcile #39825, #65039, and the Discord voice slice of #63125 based on the final root cause.
