## Repair Work Prompt

Fix the current Discord voice join/recovery timeout bug in the Discord plugin. Observable problem: after recent 2026.4.x releases, `/vc join` or autoJoin can visibly join a Discord voice channel, then abort or silently disconnect because the initial Ready wait is fixed at 15s and the Disconnected recovery path only waits 5s for Signalling/Connecting before clearing the session and destroying the connection.

Expected boundary: keep the fix inside `extensions/discord/**` plus config/docs/changelog if a new config field is added. Do not move owner-specific behavior into core, do not change `@discordjs/voice` dependency pins, and do not try to solve unrelated slash-command acknowledgement issue #73978 unless the same code path is directly fixed. If the implementation fully covers #39825 or #65039, mention that in the PR body.

Implementation direction: lengthen or make configurable the initial voice Ready timeout; lengthen and log the Disconnected recovery grace path before destroy; preserve bounded failure behavior; add regression tests that fail on the old 15s/5s destroy paths; update Discord voice schema/types/UI hints/docs and generated config metadata if adding `channels.discord.voice.*TimeoutMs` fields; add a user-facing changelog fix entry with proper attribution.

Validation: run targeted Discord voice tests first, then config schema/help tests if config changed, targeted oxfmt check for touched files, and the repo changed gate in Testbox per OpenClaw policy.
