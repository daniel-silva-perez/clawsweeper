## Summary

Current main still contains the reported Discord voice failure paths. Discord `/vc join` waits for `@discordjs/voice` Ready with a fixed 15s timeout, destroys the connection on failure, and the post-join Disconnected recovery path still waits only 5s for Signalling/Connecting before clearing the session and destroying the connection without a warning log. The issue also has multiple fresh confirmations through OpenClaw 2026.4.26, and no open PR was found that already fixes this exact problem.

## Best Possible Solution

Keep this open and land one focused Discord plugin fix. The fix should lengthen or make configurable the initial voice Ready wait, lengthen/log the Disconnected recovery grace path, add regression tests for both abort/destroy paths, update Discord voice config/docs if a new `channels.discord.voice.*TimeoutMs` field is added, and keep the behavior inside the Discord plugin owner module. If the implementation fully covers #39825 or #65039, close those from the PR; treat #73978 as a separate slash-command ack issue unless it is directly fixed.
