## Summary

Keep open. Current main still has the reported Discord voice failure shape: the initial `/vc join` path waits on a fixed 15s Ready transition and destroys on timeout, while the post-join Disconnected recovery path still waits only 5s for Signalling/Connecting before silently clearing and destroying the session. Recent issue comments report the same behavior through `2026.4.26`, so this remains an actionable Discord plugin reliability bug.

## What This Changes

The issue asks OpenClaw to stop Discord voice joins from aborting after the bot briefly enters the channel, likely by lengthening or configuring voice connection/recovery timeouts and adding recovery/logging.

## Best Possible Solution

Land a focused Discord plugin fix that lengthens and/or makes configurable the initial Ready wait and Disconnected recovery grace, logs the destroy path with guild/channel context, keeps the behavior inside the Discord plugin boundary, and adds regression coverage for both the initial join abort and post-join silent destroy paths.
