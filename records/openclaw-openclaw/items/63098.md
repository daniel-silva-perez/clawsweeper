## Summary

Current main still has the Discord voice abort/reconnect behavior described in this issue. `/vc join` waits for @discordjs/voice Ready with a fixed 15s timeout, destroys the connection on abort, and post-join disconnect recovery still gives Signalling/Connecting only 5s before silently destroying the session. The requested configurable timeout/retry/logging path is not present in the Discord voice config, UI hints, or docs, and recent issue comments confirm the failure on v2026.4.26.

## Best Possible Solution

Keep this issue open and fix the Discord plugin voice join/recovery path in the plugin owner module. A focused fix should add regression coverage for initial Ready aborts and disconnected recovery failure logging, use a longer bounded default and/or documented `channels.discord.voice.connectTimeoutMs`/recovery timeout control, and avoid moving Discord-specific policy into core. If the final root cause also resolves #39825 or #65039, close those from the implementation PR; keep the broader #63125 and slash-ack #73978 slices separate unless directly fixed.
