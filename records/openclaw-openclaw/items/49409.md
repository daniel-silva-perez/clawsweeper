## Summary

Keep this PR open. Current main still writes one slugged session-memory file per /new or /reset, still allows LLM slug generation by default outside tests, and still documents the slugged file behavior. Main has reader-side compatibility for exact daily files plus date-prefixed slugged artifacts, but it does not implement the PR's writer-side canonical daily append or opt-in/nonblocking slug path. The submitted PR is also not ready as-is because the review discussion identifies a daily-header TOCTOU race, timeout fallback bug, and a removed workspace routing regression test.

## Best Possible Solution

Keep this PR open as a useful implementation candidate, but require a refreshed patch against current main. The best path is to implement canonical `memory/YYYY-MM-DD.md` writer appends, make LLM slugging opt-in or otherwise nonblocking for /new and /reset, handle the first daily header atomically without the TOCTOU race, define timeout semantics clearly, preserve workspace routing coverage, update docs/HOOK metadata, and run targeted session-memory/fs-safe tests plus the appropriate changed gate.

## Risks / Open Questions

- The submitted daily-file implementation can still duplicate the top-level daily header under concurrent /new or /reset hooks because header detection and content append happen in separate opens.
- The submitted timeout fallback needs an explicit zero/invalid-value policy; the review discussion shows `0` falls through when `||` is used.
- Changing appendFileWithinRoot return semantics is security-sensitive and must preserve symlink, hardlink, alias, and root-boundary protections.
- The refreshed patch should keep the current workspaceDir/display-session-key regression coverage to avoid multi-agent workspace routing regressions.
