## Summary

Keep this PR open. The intended behavior is to make the bundled session-memory hook for `/new` and `/reset` avoid the blocking LLM slug path and write canonical daily memory files, but current main still defaults to LLM-backed slug generation and still writes per-session `YYYY-MM-DD-slug.md` files. The closed predecessor #41590 is not a merged canonical fix, and this PR remains a plausible implementation candidate once its review blockers are resolved.

## Best Possible Solution

Keep this PR open as a focused implementation candidate, but require a refreshed patch against current main. The best path is to implement canonical `memory/YYYY-MM-DD.md` appends for the bundled session-memory hook, make LLM slugging opt-in or otherwise nonblocking for `/new` and `/reset`, create the first daily header without a TOCTOU race, preserve workspaceDir/display-session-key coverage, update the public docs and bundled `HOOK.md`, and validate with targeted session-memory plus fs-safe tests and the appropriate changed gate.

## Risks / Open Questions

- The PR's daily-file implementation can duplicate the top-level daily header when concurrent `/new` or `/reset` hooks both probe before either writes.
- The proposed timeout config needs an explicit policy for zero and invalid values before adding `OPENCLAW_SESSION_MEMORY_SLUG_TIMEOUT_MS` or `slugTimeoutMs`.
- Removing workspaceDir/display-session-key coverage would risk regressing multi-agent workspace routing for reset snapshots.
- Changing `appendFileWithinRoot` return semantics touches a shared boundary-safe write helper, so root-boundary, symlink, hardlink, and alias protections should remain covered.
