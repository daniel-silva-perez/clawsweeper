## Summary

Keep this PR open. Current main still uses the old session-memory writer behavior: it allows LLM slug generation by default outside tests, writes one `YYYY-MM-DD-slug.md` file per `/new` or `/reset`, and the docs still describe that slugged output. Main has reader-side compatibility for date-prefixed slugged artifacts, but not the PR's writer-side canonical daily append or opt-in/nonblocking slug behavior.

## Best Possible Solution

Keep the PR open as a useful implementation candidate, but require a refreshed patch against current main. Preserve the reader compatibility for slugged artifacts, decide explicitly whether session-memory should write canonical `memory/YYYY-MM-DD.md`, make LLM slugging opt-in or otherwise nonblocking for `/new` and `/reset`, implement daily appends without the header race, keep workspace-context regression coverage, update hook docs/config help, and run targeted session-memory/fs-safe tests plus the appropriate changed gate.

## Risks / Open Questions

- The submitted patch can duplicate the daily top-level header because it probes with one `appendFileWithinRoot` call and writes content with a second call, leaving a TOCTOU window between concurrent `/new` or `/reset` hooks.
- The PR timeout fallback uses `||`, so a parsed `0` timeout falls through to another value; the refreshed patch needs an explicit policy for zero and invalid values.
- Removing the current workspaceDir/display-session-key regression test could reintroduce multi-agent workspace routing bugs.
- Changing `appendFileWithinRoot` return semantics is security-sensitive and should preserve existing symlink, hardlink, alias, and root-boundary guarantees.
