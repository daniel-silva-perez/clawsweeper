## Summary

Keep this PR open. Current main still does not implement Control UI LaTeX/KaTeX/MathJax rendering and #42840 remains the canonical open feature request, but this PR is not mergeable as-is because the diff removes existing markdown helper definitions, weakens sanitizer/link-hardening behavior, misses the lockfile update for a new dependency, and needs stronger delimiter parsing before becoming default chat behavior.

## What This Changes

This PR adds a KaTeX dependency and changes the Control UI shared markdown renderer and tests to render inline `$...$` and display `$$...$$` math in chat output.

## Best Possible Solution

Rework this PR, or a successor, against current main so math rendering is added in the shared Control UI markdown renderer while preserving the existing cache and DOMPurify link hardening, using robust delimiter rules, narrowing any KaTeX-specific style allowance, updating the lockfile, and covering inline math, display math, invalid LaTeX, currency prose, and code-span/code-fence cases.

## Reproduction Assessment

Yes. The feature gap is reproducible by inspecting current main: the UI package has no math renderer dependency and the shared markdown renderer has no math pipeline; the PR-head blocker is also statically reproducible because helper functions are removed while their call sites remain.

## Solution Assessment

No. The shared markdown renderer is the right surface, but this patch is not the best implementation because it deletes required helpers, broadens sanitizer attributes, omits the lockfile, and parses common dollar-amount prose as math; the safer approach is a reworked renderer change with preserved hardening and tighter parsing.

## Review Findings

  - body: This block removes `installHooks`, `getCachedMarkdown`, and `setCachedMarkdown`, but `toSanitizedMarkdownHtml` still calls all three. The PR will fail TypeScript/at runtime on non-empty rendering and drops the existing cache plus DOMPurify link-hardening hook.
  - confidence: 0.98

  - body: The inline scanner accepts any `$...$` pair without delimiter-boundary rules, so normal prose like `between $5 and $10` is transformed into KaTeX. That would corrupt common chat messages containing dollar amounts.
  - confidence: 0.93
- **[P2] Respect multi-backtick code spans:** `ui/src/ui/markdown.ts:128-130`
  - body: The code-span skip only looks for the next single backtick, so spans delimited with multiple backticks can still have `$...$` extracted and rendered. Literal examples using dollar signs inside code should remain untouched.
  - confidence: 0.9
- **[P2] Update the lockfile with the KaTeX dependency:** `ui/package.json:15`
  - body: The PR adds `katex` to `ui/package.json` but the changed-file list does not include `pnpm-lock.yaml`. Frozen-lockfile installs will not have a reproducible resolution for the new dependency.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff introduces sanitizer and supply-chain concerns that need rework before merge.

  - body: Removing `installHooks` drops the existing `afterSanitizeAttributes` hook that strips unsupported link schemes, sets `rel`/`target`, and applies the tail-link blur class.
