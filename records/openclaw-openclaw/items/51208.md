## Summary

This issue should stay open. Current `main` still does not provide a hook-mapping control that lets mapped agent hooks opt into continuation, and the active hook dispatch path still creates isolated cron jobs. The related stale-CLI-identity leak has been cleaned up in `resolveCronSession`, but that makes isolated rollover safer; it does not make repeated stable hook events resume the stored Codex CLI thread. There is also an open cross-referenced implementation candidate that explicitly says it closes this issue, so the right maintainer path is to review, land, or close that candidate before closing this report.

## Best Possible Solution

Keep this issue open and review the open implementation candidate. The best fix is to add a supported mapped-hook continuation control, such as `hooks.mappings[].sessionTarget`, plumb it through mapping normalization, transform overrides, request handling, and hook dispatch, then add regression coverage proving stable hook sessions with stored `cliSessionIds.codex-cli` invoke `codex exec resume` while truly isolated cron jobs still start fresh and clear stale CLI identity.
