## Summary

Keep this issue open. Current main still hardcodes hook-dispatched agent jobs as isolated, hook mapping config still has no `sessionTarget`/continuation field, and the CLI resume path still suppresses stored CLI session IDs whenever the cron session is new. The supplied GitHub context also shows an open cross-referenced implementation candidate that says it closes this issue, so the maintainer path is to review, land, or close that candidate before closing this report.

## Best Possible Solution

Keep this issue open and review the open implementation candidate. The concrete product/code path is to add a supported mapped-hook continuation control such as `hooks.mappings[].sessionTarget`, plumb it through config schema/help, mapping normalization, transform overrides, request replay scope, and hook dispatch, then add regression coverage proving stable hook sessions with stored `cliSessionIds.codex-cli` invoke CLI resume while default isolated cron jobs still start fresh and clear stale identity.
