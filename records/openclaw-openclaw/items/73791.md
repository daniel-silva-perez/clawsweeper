## Risks / Open Questions

- The PR changes a security-sensitive filesystem boundary. On Unix hosts without python3, it would intentionally use the legacy path-based atomic write/copy path instead of the pinned dir_fd Python helper, so maintainers should explicitly accept that fallback posture and rely on the existing hardlink/symlink checks plus post-write verification.
- The copy fallback tests mock runPinnedWriteHelper, which bypasses the helper's real stream/exitPromise behavior when spawn fails. That path should be validated or adjusted before merge so copyFileWithinRoot actually falls back in a python3-less runtime.
- At review time, sampled check-runs for the PR head included many green checks but also in-progress jobs; landing should wait for exact-head CI and ideally a python3-less Docker/manual smoke.
