## Summary

Current main still does not implement native support for the requested Grok 4.20 multi-agent model. The bundled xAI provider supports standard Grok 4.20 reasoning/non-reasoning IDs, but the catalog, dynamic resolver, modern-model classifier, provider wiring, docs, and tests all explicitly exclude multi-agent IDs. The issue discussion has also moved beyond an informational workaround into a request for native multi-agent support, so this needs xAI-provider maintainer judgment rather than cleanup closure.

## Best Possible Solution

Keep this issue open for xAI-provider maintainer judgment. If bundled support is desired, implement it inside `extensions/xai` as a provider-specific multi-agent path for canonical `grok-4.20-multi-agent` and relevant legacy aliases, with xAI Responses request shaping, model metadata/auth behavior, docs, and regression tests. If maintainers decide the beta/specialized surface should not be bundled, resolve that explicitly with provider-plugin or ClawHub guidance rather than closing as already implemented.
