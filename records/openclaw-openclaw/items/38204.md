## Summary

Current main still does not implement native Grok 4.20 multi-agent support. The bundled xAI plugin supports standard Grok 4.20 reasoning/non-reasoning IDs on the xAI Responses path, but current source and regression tests explicitly exclude `multi-agent` model IDs. Upstream xAI docs also describe `grok-4.20-multi-agent` as a beta, Responses/xAI-SDK-only surface, not Chat Completions. 

## Best Possible Solution

Keep this issue open for xAI-provider maintainer judgment. If bundled support is desired, implement it in `extensions/xai` as a provider-specific `grok-4.20-multi-agent` Responses path with legacy alias handling, unsupported-parameter filtering, clear built-in-tool behavior, docs, focused regression tests, and live xAI proof when keys are available. If maintainers decide the beta multi-agent surface should stay outside the bundled provider, close it only after explicit plugin/ClawHub guidance.
