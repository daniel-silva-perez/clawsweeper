## Summary

Current main still does not implement native Grok 4.20 multi-agent support. The bundled xAI plugin supports standard Grok 4.20 reasoning/non-reasoning IDs and explicitly documents the multi-agent ID as unsupported on the normal provider path; source and tests also continue to reject `multi-agent` IDs. This should remain open for xAI-provider maintainer judgment, not be closed as implemented or stale.

## Best Possible Solution

Keep this issue open for xAI-provider maintainer judgment. If bundled support is desired, implement it inside `extensions/xai` as a provider-specific multi-agent path for canonical `grok-4.20-multi-agent` plus relevant legacy aliases, with correct Responses payload shaping, unsupported-parameter filtering, tool behavior, docs, and regression/live tests. If maintainers decide this beta/specialized surface should not be bundled, resolve that explicitly with provider-plugin or ClawHub guidance rather than closing as already implemented.
