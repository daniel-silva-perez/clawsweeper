## Summary

Current main still does not support the requested Grok 4.20 multi-agent model. The bundled xAI provider supports standard Grok 4.20 reasoning/non-reasoning IDs, but dynamic model resolution, modern-model classification, docs, and tests still explicitly treat multi-agent IDs as unsupported. The issue discussion has also shifted from a workaround to native multi-agent endpoint/model support, which needs xAI-provider maintainer judgment rather than cleanup closure.

## Best Possible Solution

Keep this issue open for xAI-provider maintainer judgment. If bundled support is desired, implement it in `extensions/xai` as a provider-specific multi-agent path for the canonical `grok-4.20-multi-agent` model and relevant legacy aliases, with request shaping for xAI Responses API limits, docs, catalog/auth behavior, and regression tests. If maintainers decide the beta surface should not be bundled, resolve that explicitly with ClawHub/provider-plugin guidance rather than closing as already implemented.
