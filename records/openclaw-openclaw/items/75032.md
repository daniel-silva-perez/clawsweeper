## Summary

Keep open: current main still has only an internal loader opt-out, while the public `plugins` config schema, typed config contract, Gateway prestage path, docs, and package-level runtime-dependency repair default do not implement the requested operator controls. The linked field reports make this concrete enough for one focused repair PR rather than stale cleanup.

## What This Changes

The issue requests exposing a public `plugins.installBundledRuntimeDeps` opt-out, making bundled plugin runtime dependency installs use an isolated execution root by default, and documenting the operator behavior matrix.

## Best Possible Solution

Add a public boolean config field, thread it into Gateway prestage and plugin loading so `false` skips bundled runtime dependency installs and fails clearly on missing deps, make package-level repair run in an isolated execution root with atomic replacement by default, and update docs/tests/changelog for the supported matrix.

## Reproduction Assessment

Yes for the code-level gap: source inspection shows the config key is absent, the internal loader option is not public config, Gateway prestage ignores any user opt-out, and package-level repair does not pass an isolated execution root. The production stalls and disk bloat remain field-evidence rather than live reproduction in this review.

## Solution Assessment

Yes. The requested direction is the narrowest maintainable fix because it exposes an existing internal loader seam and changes the bundled runtime-dependency owner path, rather than adding another environment-only workaround.
