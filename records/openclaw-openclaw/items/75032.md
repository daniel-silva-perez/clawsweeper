## Summary

Keep open: current main still has the operator-facing gap. The loader has an internal bundled-runtime-deps opt-out, but the public config schema, typed config contract, Gateway prestage path, docs, and package-level install default do not yet expose or default the requested controls.

## What This Changes

Expose a public bundled runtime-dependency install opt-out, make isolated execution roots the default for bundled plugin runtime-dependency installs, and document the supported behavior matrix.

## Best Possible Solution

Add a public boolean config field with default-on behavior, thread it consistently through Gateway prestage and plugin loading so `false` skips bundled runtime-dependency installs and fails clearly on missing deps, switch package-level repairs to an isolated execution root with atomic replacement by default, and update docs/tests/changelog for the supported operator matrix.

## Reproduction Assessment

Yes for the code-level gap: source inspection shows the config key is absent, the internal loader option is not public config, Gateway prestage ignores any user opt-out, and package-level repair does not pass an isolated execution root. The reported production stalls and disk growth remain field evidence rather than live reproduction here.

## Solution Assessment

Yes. Exposing the existing loader seam through config and changing the bundled runtime-dependency owner path is the narrow maintainable fix, while avoiding another environment-only workaround.
