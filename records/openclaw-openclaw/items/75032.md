## Summary

Keep open: current main has internal seams for skipping bundled runtime dependency installs and for isolated install execution, but the requested operator-facing config field is absent and package-level external staging still runs installs directly in the shared install root by default.

## What This Changes

The issue requests a `plugins.installBundledRuntimeDeps` config escape hatch plus isolated bundled plugin runtime-dependency install execution by default, with documentation for the resulting behavior matrix.

## Best Possible Solution

Add the config field to the public plugin config contract, wire it through gateway prestage and per-plugin loader paths, default package-level bundled dependency installs to an isolated execution root with atomic `node_modules` replacement, and document the operator matrix with regression tests for the linked production symptoms.

## Reproduction Assessment

Partially. The missing config surface and non-default isolated execution path are high-confidence from source/schema inspection; the reported production stalls and disk bloat are supported by linked field reports but were not live-reproduced in this read-only review.

## Solution Assessment

Yes, as a focused implementation PR. The requested opt-out matches an existing loader option, and the isolated execution behavior already exists in the installer; the maintainable fix is to expose and consistently route those seams rather than adding a separate workaround path.

## Repair Work Prompt

Fix openclaw/openclaw#75032. Implement a public `plugins.installBundledRuntimeDeps?: boolean` config field defaulting to `true`, with type/schema/help/docs coverage. When it is `false`, gateway startup prestaging and per-plugin bundled runtime dependency repair must be skipped; missing runtime deps should fail clearly during load/import rather than being silently installed. Also make bundled runtime dependency installs use an isolated execution root by default for package-level/external install roots, not only plugin-root installs, while preserving the install-root lock, package-level merged dependency plan, path-list search roots, materialization short-circuit, and read-only packaged-root guarantees. Document the behavior matrix in the plugin/runtime dependency docs and add a user-facing changelog entry thanking @Charpup. Do not change package manager versions, `pnpm.patchedDependencies`, external plugin install behavior, or plugin ids.
