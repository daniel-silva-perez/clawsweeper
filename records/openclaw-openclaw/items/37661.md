## Summary

Keep this issue open. Current main still has tool-call loop detection, exact duplicate stream-event suppression, and heartbeat-specific duplicate-send handling, but I found no assistant-output repetition/ngram guard and no shared per-turn outbound flood cap that would stop pure text repetition or repeated channel sends during ordinary assistant streaming/final reply delivery.

## Best Possible Solution

Keep this issue open and implement a conservative assistant-output repetition guard in the streaming/final reply path, plus a shared per-turn outbound duplicate/message cap before channel delivery. Keep #34574 separate for tool-call loops and treat #39536 as the already-closed heartbeat-specific case, not as the generic fix for this issue.
