## Summary

Keep open. Current main still lacks an assistant text repetition or n-gram guard: tool-loop detection is still tool-call scoped, the assistant streaming path still emits any growing cleaned text, and block/outbound dedupe is exact-payload based rather than a per-turn repetition/message cap. The April 11 report says this reproduces as pure assistant text repetition outside subagent spawning, so the closed tool-loop and heartbeat-specific related issues do not supersede this one.

## Best Possible Solution

Keep this issue open and implement a conservative assistant-output repetition guard in the streaming/final reply path, plus a shared per-turn outbound duplicate/message cap before channel delivery. Keep #34574 scoped to tool-call loops and treat closed #39536 as heartbeat-specific context rather than the generic fix.

## Risks / Open Questions

- A text repetition guard can false-positive on legitimate repeated logs, code blocks, tables, quoted text, or short acknowledgements unless thresholds and exclusions are conservative.
- A per-turn delivery cap must count logical assistant replies carefully so normal long-message chunking, media delivery, and block streaming are not truncated incorrectly.
- The live trigger is intermittent and provider/model-specific, so validation should use synthetic repeated-delta, repeated-final-text, and repeated-outbound fixtures rather than depending on a live endpoint.
