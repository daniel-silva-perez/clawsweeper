## Summary

Keep open. Current main still only handles tool-call loops and exact duplicate stream/delivery cases; it does not add an assistant text repetition or n-gram guard, and it does not cap repeated outbound assistant block deliveries per turn. The April 11 report explicitly says the bug still reproduced on 2026.4.10 as pure text repetition during normal conversation, so the remaining problem is not covered by #34574 or the heartbeat-specific #39536 closure.

## Best Possible Solution

Keep this issue open and implement a conservative assistant-output repetition guard in the streaming/final reply path, plus a shared per-turn outbound duplicate/message cap before channel delivery. Keep #34574 scoped to tool-call loops and treat closed #39536 as heartbeat-specific context rather than the generic fix.
