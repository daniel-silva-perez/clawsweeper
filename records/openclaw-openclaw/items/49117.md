## Summary

Keep this PR open. Current main still does not implement the intended silent-completion logging behavior for the local delivery path, and the gateway CLI still emits a fallback summary/log line whenever the gateway result has no payloads. Exact NO_REPLY can still be filtered down to an empty payload list, so the PR remains a focused implementation candidate even though it likely needs a rebase around newer runner metadata.

## Best Possible Solution

Keep this PR open for maintainer review or update. The best path is to preserve a typed, delivery-consumable signal for deliberate exact NO_REPLY completions, or safely reuse current finalAssistantRawText/terminalReplyKind metadata, then gate both local delivery and gateway CLI empty-result logging with focused regression tests.
