## Summary

Keep open. Current main still does not implement this PR’s intended silent-completion logging behavior: exact NO_REPLY is recognized as intentional silence for incomplete-turn handling and then filtered out of reply payloads, but the local delivery and gateway CLI empty-payload paths still emit fallback output. The PR remains a focused runtime/test candidate, with no evident security or supply-chain-sensitive changes in the provided diff.

## Best Possible Solution

Keep this PR open for maintainer review or update. The best path is still to carry an explicit typed signal for exact NO_REPLY silent completions from the embedded runner into both local delivery and gateway CLI output, preferably through a shared predicate and focused regression tests for both logging surfaces.
