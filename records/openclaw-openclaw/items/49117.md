## Summary

Keep this PR open. Current main still does not implement the PR’s intended behavior: exact NO_REPLY silent completions can reduce to no outbound payloads, while both local delivery and the gateway CLI still log an empty-result message without checking the existing silent-reply metadata.

## Best Possible Solution

Keep this PR open for maintainer review or update. The best fix is to preserve or derive a typed silent-completion signal for deliberate exact NO_REPLY completions, then gate both deliverAgentCommandResult and agentViaGatewayCommand empty-result logging with focused regression tests on current main.
