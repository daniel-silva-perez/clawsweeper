## Summary

Keep this issue open. Current main still accepts `gateway.nodes.denyCommands` as a plain string array at schema/write time, while the unknown-command detection remains in `openclaw security audit`. There are also open PRs (#50076 and #50320) with closing syntax for this issue, so the issue should remain open until a candidate fix is reviewed and merged.

## Best Possible Solution

Review and land a focused config-set validation change, preferably one that handles direct `gateway.nodes.denyCommands` writes and parent `gateway.nodes` writes, warns rather than hard-fails for unknown names, preserves custom/plugin command names, and shares or improves the audit suggestion logic with a full command catalog.
