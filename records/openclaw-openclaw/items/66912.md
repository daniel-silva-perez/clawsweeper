## Summary

Keep this PR open. Current main still lacks the self-authored reply-target media guard, and the PR head is a plausible fix rather than obsolete. It should be cleaned up before merge: add focused regression coverage and remove or justify the unrelated generated Plugin SDK baseline change.

## Best Possible Solution

Keep this PR open and ask for a clean Telegram plugin patch: keep the self-authored reply-target guard in resolveReplyMediaForMessage, add regression coverage in extensions/telegram/src/bot.test.ts for a user-authored message replying to bot-authored media with no getFile/media fetch, preserve the existing ordinary reply-media ingestion behavior, and remove or explicitly justify/regenerate the docs/.generated/plugin-sdk-api-baseline.sha256 change against current main.
