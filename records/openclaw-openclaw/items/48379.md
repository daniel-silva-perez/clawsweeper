## Summary

Keep this issue open. Current main sends OpenRouter app-attribution headers, but the title is still a protected static `OpenClaw` value and there is no first-class per-agent `X-OpenRouter-Title` or equivalent OpenRouter app-title setting. The related reports in the supplied context cover static/correct OpenRouter header naming and a generic proposed `X-OpenClaw-Agent` header, but they do not show this OpenRouter dashboard attribution request implemented on main.

## Best Possible Solution

Implement this in the OpenRouter request-header path: pass the resolved agent id or identity into OpenRouter attribution resolution, preserve the current `OpenClaw` default, add either a documented auto title such as `OpenClaw/<agentId>` or an explicit per-agent OpenRouter app-title config, and cover it with provider-request/proxy-wrapper tests plus docs.
