## Summary

Keep this issue open. Current main still sends static OpenRouter app-attribution headers using the product title `OpenClaw`; neither the OpenRouter wrapper nor the provider-attribution policy receives agent identity, and the per-agent config schema has no OpenRouter app-title/header override. Related #50206 is adjacent generic `X-OpenClaw-Agent` work, but it is open and does not implement per-agent OpenRouter dashboard title attribution.

## Best Possible Solution

Keep this open and implement the attribution in the OpenRouter request-header path. Pass resolved agent identity into provider attribution or the OpenRouter provider wrapper, preserve the current `OpenClaw` default, add either a documented automatic title such as `OpenClaw/<agentId>` or an explicit per-agent OpenRouter app-title config, and cover the behavior with provider-request/proxy-wrapper tests plus docs.
