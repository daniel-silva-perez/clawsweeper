## Summary

Keep this issue open. Current main still uses static OpenRouter app-attribution headers, the LLM wrapper does not receive agent identity for title construction, and the per-agent config schema has no OpenRouter app-title/header override. Related #50206 is adjacent generic agent-header work, but it is open and does not satisfy the OpenRouter dashboard title request.

## Best Possible Solution

Keep this issue open and implement the attribution in the OpenRouter request-header path. Thread resolved `agentId` or sanitized agent display identity into the provider stream/header context, preserve the current `OpenClaw` default, add either a documented automatic title such as `OpenClaw/<agentId>` or an explicit per-agent OpenRouter app-title config, and cover it with wrapper/provider-attribution tests plus OpenRouter provider docs. #50206 can remain a related generic tracking PR, but it should not be treated as a complete fix for this OpenRouter title request unless expanded.
