## Summary

Keep open. Current main still has the reported Android 15/16 foreground-service crash path: the app targets SDK 36, uses a `dataSync` foreground service for the node connection, calls `ServiceCompat.startForeground` during `NodeForegroundService.onCreate`, and has no timeout or rejected-start fallback.

## What This Changes

Fix the Android companion app so `NodeForegroundService` does not crash with `ForegroundServiceStartNotAllowedException` when the `dataSync` foreground-service quota is exhausted.

## Best Possible Solution

Make the Android node service quota-aware: catch rejected foreground starts, implement the Android 15+ timeout callback, stop or demote cleanly without restart loops, surface any changed background-connection behavior in docs, and add focused unit plus device/emulator coverage for quota exhaustion.
