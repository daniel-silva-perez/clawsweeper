## Summary

Keep open. The underlying current-main bug is real: native `sessions_spawn` reads `cwd` but still passes only the inherited tool-context workspace to `spawnSubagentDirect`, while ACP receives `cwd`. The PR is useful but not merge-ready because its attachment override can write cross-agent attachments into a workspace different from the one the child run actually uses, and it also needs the required changelog entry for a user-facing fix.

## What This Changes

The PR forwards `sessions_spawn.cwd` into native subagent workspace inheritance and adds an attachment workspace override so inline attachments follow the overridden workspace.

## Best Possible Solution

Implement native `sessions_spawn.cwd` by computing the final child workspace once, honoring `cwd` for same-agent native subagents, preserving target-agent workspace selection for cross-agent spawns, and materializing attachments under that same final workspace with focused regression tests and a changelog entry.

## Reproduction Assessment

Yes. A high-confidence static reproduction is a native `sessions_spawn` from `main` to another allowed agent with attachments and `ctx.workspaceDir` or `cwd` set: the PR writes attachments under `ctx.workspaceDir`, while the child run still resolves to the target agent workspace, so the advertised relative attachment path is missing.

## Solution Assessment

No. Forwarding `cwd` to the native subagent path is the right direction, but the PR's attachment fix should use the resolved child workspace rather than raw `ctx.workspaceDir`, and it should add targeted tests plus the required changelog entry.
