## Summary

Keep open. Current main still reads the sessions_spawn cwd parameter but only forwards it to the ACP path; the native subagent path still passes opts.workspaceDir. The PR is aimed at a real remaining gap, but its latest attachment change needs more work because it can misplace attachments for cross-agent spawns.

## Best Possible Solution

Keep this PR open as a focused implementation candidate, but revise it so cwd affects native same-agent workspace inheritance without breaking the existing cross-agent target-workspace rule. The safest path is to compute one resolved child workspace once, use it consistently for spawned metadata and attachment materialization, and add targeted tests for sessions_spawn cwd forwarding plus cwd/cross-agent attachment cases.
