## Summary

Keep this PR open. Current main still has uncancelable memory batch polling, so the user problem is not obsolete, but the proposed branch is not merge-ready: the affected provider batch code has since moved into provider plugins, the production batch contract still has no disconnect signal source, and the PR's timer/listener pattern has correctness bugs already noted in prior review comments.

## What This Changes

The PR adds optional AbortSignal parameters through the memory-host batch runner and OpenAI, Voyage, and Gemini batch polling/fetch paths so batch work can stop when callers disconnect.

## Best Possible Solution

Implement the cancellation contract on current main by threading a real request/session AbortSignal into MemoryEmbeddingBatchOptions, then through the OpenAI, Voyage, and Gemini provider-owned batch helpers. The provider code should pass the signal to upload/create/status/error/download HTTP calls and use a leak-free polling sleep helper that handles already-aborted signals and removes listeners on every path.

## Reproduction Assessment

Yes. A code-level reproduction is high confidence: current main's OpenAI, Voyage, and Gemini wait loops use uncancelable setTimeout polling, and the PR's added listener pattern can be exercised with a long-running AbortController-based unit test to show stale listeners and missed already-aborted signals.

## Solution Assessment

No. The direction is right, but this old package-path patch is not the best current fix; the safer solution is a narrow replacement against the current extension provider files plus the shared memory batch contract and focused abort tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Wire the signal into the production batch contract:** `packages/memory-host-sdk/src/host/batch-runner.ts:10`
  - body: Adding `signal` to this helper type is not enough for disconnect cancellation: the production `MemoryEmbeddingBatchOptions` and memory-core `batchEmbed` call path still have no signal field, so provider adapters will continue passing `undefined` and polling will run until timeout.
  - confidence: 0.87
- **[P2] Clean up abort listeners after each sleep:** `packages/memory-host-sdk/src/host/batch-openai.ts:194-203`
  - body: Each polling interval adds an abort listener, but the normal timer path resolves without removing it. Long polls will leave one listener per interval on the same signal; name the handler and remove it when the timer fires or in a `finally` path. The same pattern appears in the Voyage and Gemini changes.
  - confidence: 0.9
- **[P2] Handle signals that are already aborted:** `packages/memory-host-sdk/src/host/batch-openai.ts:194-203`
  - body: This promise only listens for future abort events. If the signal has already aborted before the sleep starts, the event will not re-fire, so the code waits the full polling interval and may continue polling; check `signal.aborted` before scheduling the timer.
  - confidence: 0.84
