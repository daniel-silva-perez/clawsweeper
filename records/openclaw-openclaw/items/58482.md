## Summary

Keep this PR open, but do not merge it as-is. The cancellation goal is still valid, but the branch is stale against current main and the proposed implementation is partial: the live batch code moved into provider plugins, the production memory batch option path still has no signal, and the new polling sleeps leak abort listeners.

## What This Changes

The PR adds optional AbortSignal plumbing to memory-host batch execution and OpenAI, Voyage, and Gemini batch polling/fetch helpers.

## Best Possible Solution

Retarget the change to current main by adding a narrow optional cancellation contract through the memory batch runtime and provider-plugin batch helpers, using a shared cleanup-safe cancellable sleep pattern and focused tests for OpenAI, Gemini, and Voyage polling cancellation.

Yes. A code-level reproduction is high-confidence: current main shows bare setTimeout polling and no signal field in the production MemoryEmbeddingBatchOptions path, while the PR diff shows listener registration without cleanup; no runtime test was run because this was a read-only review.

## Solution Assessment

No. The proposed branch is not the best solution because it edits stale paths and only partially wires cancellation; a focused replacement against extensions/openai, extensions/google, extensions/voyage, and the memory batch option contract is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Thread the signal from the production batch API:** `packages/memory-host-sdk/src/host/batch-runner.ts:10`
  - body: Adding `signal` only to the low-level batch execution type does not make disconnect cancellation reachable. The production memory runtime still has no `signal` on `MemoryEmbeddingBatchOptions`, and the provider adapters build batch helper calls from that object, so `params.signal` remains undefined in normal runs.
  - confidence: 0.91
- **[P2] Clean up abort listeners after polling sleeps:** `packages/memory-host-sdk/src/host/batch-openai.ts:192-201`
  - body: Each polling interval now registers an abort listener, but the listener is never removed when the timer resolves normally and already-aborted signals are not rejected before sleeping. Long polls will retain one stale listener per interval; the same pattern appears in the Gemini and Voyage additions.
  - confidence: 0.9
- **[P2] Pass the signal into Gemini status polling:** `packages/memory-host-sdk/src/host/batch-gemini.ts:160`
  - body: `fetchGeminiBatchStatus` gains a `signal` parameter, but the patch does not attach it to the status request or pass it from `waitForGeminiBatch`. Aborts during an in-flight Gemini status request will still wait on the network call.
  - confidence: 0.84
