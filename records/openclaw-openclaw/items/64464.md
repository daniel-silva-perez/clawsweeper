## Summary

Keep this issue open. Current main does not implement the requested visible WSL2 Chrome fix. The browser plugin now has a Linux no-display headless fallback and an explicit headed-mode error, but it still does not set `DISPLAY=:0` or otherwise make `headless: false` WSL2 launches work automatically.

## Best Possible Solution

Keep the issue open for a maintainer-reviewed browser plugin fix or product decision. A good fix would add a scoped WSL2 headed-launch fallback that preserves existing `DISPLAY`/`WAYLAND_DISPLAY`, avoids changing non-WSL Linux/headless behavior, uses the existing WSL detection seam if appropriate, and adds tests around spawn env/headless resolution plus docs for WSL2 headed Chrome.
