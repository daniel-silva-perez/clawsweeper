## Summary

Keep open. Current main still has the client-tool handoff and streaming-argument gaps this PR targets, and the branch is directionally useful, but it should not merge until the newly added function-call-arguments event shape and changelog entry are corrected.

## What This Changes

This PR adds `terminate: true` to client-tool handoff stubs, emits dedicated `/v1/responses` function-call argument SSE events, teaches the Responses stream parser to consume `.done` arguments, and adds regression tests plus a changelog entry.

## Best Possible Solution

Land a corrected version of this PR that keeps the `terminate: true` handoff, the dedicated argument streaming events, and the parser/test coverage, while adding the documented `name` field to `.done` event schema/emission/tests and crediting the changelog entry.

## Reproduction Assessment

Yes. Current main source gives a high-confidence reproduction path: `/v1/responses` streaming client-tool calls still return a non-terminating pending stub and do not emit dedicated `function_call_arguments` events. A live check would POST to `/v1/responses` with `stream: true` and a client function tool, then inspect the SSE sequence and transcript.

## Solution Assessment

Mostly, but not yet. `terminate: true` matches the pi-agent-core termination contract and the event-sequence direction matches the Responses streaming convention; the safer merge path is to make the newly added `.done` event shape match the documented fields and fix the changelog attribution first.
