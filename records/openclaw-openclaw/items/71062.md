## Summary

Keep this PR open. Current main still has the reported `/v1/responses` client-tool behavior: the Pi adapter returns a normal pending stub without `terminate: true`, the Gateway streams function-call arguments only on `output_item` bookends, and the Responses stream parser handles argument deltas but not `.done`-only arguments.

## Best Possible Solution

Keep this PR open, review the focused implementation, and land it after CI plus the changelog attribution fix. The right fix boundary is the one described here: terminate Pi after client-tool handoff, emit dedicated `response.function_call_arguments.{delta,done}` events while preserving final snapshots, parse `.done`-only arguments, and keep regression coverage in the agent adapter, Gateway SSE, and upstream Responses stream parser tests.
