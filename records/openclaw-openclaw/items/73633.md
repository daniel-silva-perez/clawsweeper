## Summary

Keep this PR open, but the current patch is not sufficient to close the linked WebChat bug. The rendered-text copy path is useful hardening, but source inspection and the related discussion show the reported full-message copy can still occur when the click lands on the overlapping message-level copy action instead of the code-block button.

## What This Changes

The PR adds a Control UI helper and unit tests for resolving code-block copy text from rendered `<pre><code>` content, then updates the chat click handler to use that helper and stop propagation for code-block copy clicks.

## Best Possible Solution

Land one combined Control UI fix that both prevents the message-level action overlay from intercepting code-block Copy clicks and copies the rendered code text for clicks that reach the code-block control, with regression coverage for the overlap path and the rendered-text path.

## Reproduction Assessment

Yes. #69605 provides concrete Control UI steps, and current main shows a deterministic source path where `.chat-bubble-actions` can overlap the code-block copy control while copying full markdown through `.chat-copy-btn`.

## Solution Assessment

No. Reading rendered `textContent` is a reasonable part of the fix, but this PR does not handle the supported path where the click target is the message-level copy button and the new code-block handler never runs.
