## Summary

Keep this PR open. The patch improves the clipboard payload when a click reaches the code-block button, but it does not address the linked bug's still-plausible overlay path where the message-level copy button receives the click and copies the full markdown.

## What This Changes

The PR adds a Control UI helper and unit tests to resolve code-block copy text from rendered `<pre><code>` content, then updates the WebChat click handler to use that helper and stop event propagation.

## Best Possible Solution

The merged fix should cover both parts of the user-visible failure: keep the message-level copy/expand action from covering the visible code-block Copy control, and copy the rendered code text when the code-block button is activated, with regression coverage for both behaviors.

## Reproduction Assessment

Yes. The linked bug gives concrete WebChat steps, and current main shows a source-level reproduction path where the absolutely positioned message-level copy action can overlap the code-block copy control and write full markdown.

## Solution Assessment

No. Reading rendered `textContent` is a reasonable hardening change for valid code-button clicks, but it is not the narrowest complete fix unless it is paired with the layout/`has-copy` overlap fix or equivalent browser-verified coverage.
