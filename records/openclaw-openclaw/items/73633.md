## Summary

Keep open. The PR is useful hardening, but it does not fully close the linked WebChat bug because the click can still hit the message-level copy button before the new code-block handler runs.

## What This Changes

The PR adds a Control UI helper and tests for copying rendered code-block textContent, then updates the WebChat click handler to use that helper with a data-code fallback.

## Best Possible Solution

Land one canonical fix for the linked WebChat bug that combines reliable code-block text extraction with a layout or event-handling fix that prevents the message-level copy action from intercepting code-block copy clicks, plus regression coverage and the required changelog entry.

## Reproduction Assessment

Yes. The linked bug gives concrete Control UI steps, and current main shows a source-level reproduction path: an absolutely positioned message copy action can overlap the code-block copy button while the `has-copy` padding class is never applied.

## Solution Assessment

No. Reading rendered `textContent` is a reasonable improvement for clicks that reach `.code-block-copy`, but the PR is not the complete fix until it also handles the message-copy overlay path or applies the existing `has-copy` layout guard.
