## Summary

Keep this PR open. Current main still lacks the proposed rendered-text copy helper, but the patch appears incomplete for the linked WebChat symptom because the competing overlap path can still route clicks to the message-level copy button before the new code-block handler runs.

## What This Changes

The PR adds a Control UI code-block copy helper and tests, then updates the WebChat click handler to copy rendered `<pre><code>` textContent with a `data-code` fallback and stop event propagation.

## Best Possible Solution

Review this PR together with #69634 against a browser reproduction of the linked WebChat bug. The merged fix should ensure clicks on the visible code-block Copy control cannot hit the message-level copy action, and copying should use the rendered code text or an equally reliable source with regression coverage.

## Reproduction Assessment

Yes. The linked bug provides concrete WebChat steps, and current source shows a plausible deterministic path where the absolute message-level copy action can overlap the code-block copy button while writing full markdown.

## Solution Assessment

No, not as a complete fix yet. Reading rendered `textContent` is a reasonable hardening change for clicks that reach `.code-block-copy`, but it does not handle the supported overlap path where the event target is `.chat-copy-btn` and the new handler never runs.
