## Summary

Keep open. Current main still delivers Feishu streaming replies through CardKit interactive-card creation/update/close without a searchable plain-text fallback, and PR #74808 is open with closing syntax rather than merged.

## What This Changes

The issue asks for Feishu streaming card replies to be searchable in Feishu conversation search, either through an indexed fallback message or a documented limitation.

## Best Possible Solution

Review and either land or deliberately replace PR #74808 with a Feishu-owned opt-in searchable fallback plus docs that explain the visible duplicate-message tradeoff; avoid moving this into core unless another channel needs the same contract.

## Reproduction Assessment

Yes. The issue gives a concrete Feishu UI path, and current main matches it: streaming sends an interactive CardKit card and updates/closes it, while tests assert no plain-text fallback message is sent.

## Solution Assessment
