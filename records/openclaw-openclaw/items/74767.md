## Summary

Keep open. Current main still delivers Feishu streaming replies through CardKit interactive-card creation/update/close without an indexed plain-text fallback, and the linked fix PR #74808 is open with closing syntax rather than merged.

## What This Changes

The issue asks for Feishu streaming-card replies to remain searchable in Feishu conversation search, or for the limitation to be handled or documented.

## Best Possible Solution

Review and either land or supersede PR #74808 with a Feishu-owned opt-in searchable fallback plus docs, after confirming the Feishu search behavior and the visible duplicate-message tradeoff.

## Reproduction Assessment

Yes. The issue gives a concrete Feishu UI path: enable Feishu streaming, send a distinctive keyword in a streamed card reply, wait for closeout, then search the conversation; current main’s code path matches that streaming-card-only delivery mode, though this read-only review did not perform a live Feishu search.

## Solution Assessment

Yes, keeping this issue open while PR #74808 is reviewed is the right path. The narrow maintainable solution belongs in the Feishu plugin as an explicit fallback setting or documented limitation, not an unreviewed core behavior change.
