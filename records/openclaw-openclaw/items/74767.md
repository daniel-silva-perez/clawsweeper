## Summary

Keep open. Current main still uses Feishu CardKit interactive-card creation/update/close for streamed replies without a plain-text indexing fallback, and open PR #74808 is already the paired implementation candidate for this issue.

## What This Changes

Request that Feishu streaming interactive-card replies remain searchable in Feishu conversation search, either by adding an indexed fallback or documenting the limitation.

## Best Possible Solution

Review and either land or deliberately supersede PR #74808 with a Feishu-owned opt-in searchable fallback plus docs for the search-indexing limitation and duplicate-message tradeoff.

## Reproduction Assessment

Yes. The issue gives concrete Feishu UI steps, and current main matches the reported delivery path: streamed card replies are sent as CardKit interactive cards without a plain-text fallback. I did not perform a live Feishu search in this read-only review.

## Solution Assessment

Yes. Keeping this issue open while PR #74808 is reviewed is the narrowest maintainable path; the fix belongs in the Feishu plugin as an explicit fallback setting or documented limitation, not in generic core dispatch.
