## Summary

Keep open. Current main still sends Feishu streaming replies as CardKit interactive cards without a searchable plain-text fallback, and PR #74808 is open, unmerged, and explicitly targets this issue with closing syntax.

## What This Changes

Feishu streaming interactive-card replies should remain searchable in Feishu conversation search, either through an opt-in plain-text fallback or clear documentation of the limitation.

## Best Possible Solution

Review and either land or deliberately replace PR #74808 with a Feishu-owned opt-in searchable fallback plus docs that explain the visible duplicate-message tradeoff; keep the issue open until that PR or its replacement merges.

## Reproduction Assessment

Yes. The issue gives concrete Feishu UI steps, and current main can be source-verified to route streaming final text through CardKit interactive cards while tests assert the plain-text send path is not called. No Feishu/Lark credentials were present in env or profile for a live search-index check.

## Solution Assessment

Yes. The current best path is to keep this issue open while the linked Feishu plugin PR is reviewed, because it targets the owner module directly and avoids a core channel contract change unless another channel later needs the same behavior.
