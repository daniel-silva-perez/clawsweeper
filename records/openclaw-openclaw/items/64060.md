## Summary

Keep this PR open. Current main still has the cron normalization behavior this PR targets: text payloads are inferred as systemEvent before agent-turn hints such as model are considered, so model overrides can still be discarded. The PR is not obsolete cleanup; it remains an implementation candidate for the nested payload path and the legacy top-level path raised in review.

## Best Possible Solution

Keep this PR open and review or land this branch, or an equivalent mainline fix, that updates both coercePayload() and inferTopLevelPayload() and keeps regression coverage for nested and top-level create/patch text-plus-model inputs. The change belongs in src/cron/normalize.ts with colocated tests in src/cron/normalize.test.ts; the documented cron model precedence should remain unchanged.
