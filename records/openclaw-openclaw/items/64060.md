## Summary

Keep this PR open. Current main still has the cron normalization bug the PR targets: implicit `text` payloads are classified as `systemEvent` before agent-turn hints such as `model` are considered, so the model override can still be stripped. The PR is a focused implementation candidate, and the later branch update addressed the earlier top-level-path review gap.

## Best Possible Solution

Keep this PR open and review or land this branch, or an equivalent mainline fix, that updates both `coercePayload()` and `inferTopLevelPayload()` and keeps regression coverage for nested and top-level create/patch `text` plus `model` inputs. The fix belongs in `src/cron/normalize.ts` with colocated coverage in `src/cron/normalize.test.ts`, while preserving explicit `systemEvent` pruning and documented cron model precedence.
