## Summary

Keep open. Current main has a shared low-disk-space helper for update and bundled plugin runtime-dependency repair paths, but it still does not implement this PR's central behavior: a dedicated `doctor:disk-space` health contribution for the OpenClaw state directory before state-integrity checks. The PR is focused, prior review feedback was addressed in the PR discussion, and the security pass did not show dependency, workflow, package, script, or secret-handling changes.

## Best Possible Solution

Keep this PR open for maintainer review, but ask for a rebase/rework against current main: add a lazy `doctor:disk-space` contribution before `doctor:state-integrity`, reuse or extend `src/infra/disk-space.ts` for the filesystem probing, keep the 500 MB warning / 100 MB critical Doctor semantics if maintainers agree with those thresholds, and add focused tests for contribution ordering, threshold behavior, and the missing-state-dir ancestor path.
