## Summary

Keep this issue open. Current main still routes normal agent runs through the process-wide `main` global lane unless an explicit runtime lane is supplied, and there is still no shipped per-agent lane or per-agent lane concurrency config. The related open PR #52655 directly implements this area, but it is unmerged and should be reviewed or replaced before closing the user report.

## Best Possible Solution

Keep this issue open as the canonical product/runtime tracker. The best path is to review and land #52655 or an equivalent maintainer-owned change that adds a typed and documented per-agent lane config, applies lane concurrency on startup and reload, and threads the resolved lane through inbound, followup, and memory agent-run paths while preserving per-session serialization.
