## Summary

Keep open. Current main still turns fatal isolated cron/provider failures into `isError: true` delivery payloads and sends them through the normal announce dispatch path. Newer code appears to sanitize some raw provider JSON and adds failure-notification routing, but it does not implement the requested suppression of announce delivery for failed runs. The related implementation candidate #46236 is closed unmerged.

## Best Possible Solution

Keep this issue open. Add a guard at the isolated cron finalization or `dispatchCronDelivery` boundary so fatal/error-status isolated cron runs skip normal `delivery.mode: "announce"` fallback delivery regardless of `delivery.bestEffort`, while still recording `lastRunStatus: "error"` and allowing configured failure notifications through the separate alert path. Add focused regression coverage for provider/run-level failure with announce delivery and `bestEffort: false` asserting no outbound announce send occurs, plus coverage that explicit failure alerts still send sanitized messages.
