## Summary

Keep open. Current main still turns fatal isolated cron/provider failures into `isError: true` delivery payloads, records the run as `status: "error"`, and then passes those payloads through the normal announce dispatch path. The exact raw JSON text may now be partly formatted, but the core requested behavior remains missing: failed runs can still be delivered to user-facing announce channels instead of only being recorded and handled through the separate failure-alert path.

## Best Possible Solution

Keep this issue open. Add a guard at the isolated cron finalize or delivery-dispatch boundary so fatal/error-status runs skip normal `delivery.mode: "announce"` delivery, while still recording `lastRunStatus: "error"`, run history, counters/backoff, and configured failure-alert or failure-destination notifications. Add focused regression coverage for provider/run-level errors with announce delivery and `bestEffort: false` asserting no outbound announce send occurs, plus coverage that explicit failure alerts still send sanitized messages.
