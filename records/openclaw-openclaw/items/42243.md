## Summary

Keep open. Current main still turns fatal isolated-cron/provider failures into `isError: true` delivery payloads, records the run as an error, and then passes those payloads through normal announce delivery. The related implementation candidate #46236 is closed and unmerged, so there is no merged fix to close against.

## Best Possible Solution

Keep this issue open. Add a guard at isolated cron finalization or `dispatchCronDelivery` so fatal/error-status isolated cron runs skip normal `delivery.mode: "announce"` fallback delivery regardless of `delivery.bestEffort`, while still recording `lastRunStatus: "error"` and sending configured failure notifications through the separate alert path. Add regression coverage for provider/run-level failure with announce delivery and `bestEffort: false` asserting no outbound announce send, plus coverage that configured failure alerts still send sanitized messages.
