## Summary

Keep this PR open. Current main still has FIFO cron schedule cache behavior: cache hits return without reordering the Map entry, capacity eviction deletes the first Map key, and current tests only prove cache reuse/size rather than key-level LRU survival. The PR remains a focused implementation candidate, and the latest branch context addresses the earlier bot review by adding key-level cache assertions.

## Best Possible Solution

Keep this PR open as the canonical cron cache LRU fix, then land it or an equivalent maintainer patch that promotes cache hits with delete/re-set and includes key-level regression coverage proving the promoted cron expression survives eviction while the next-oldest entry is evicted.
