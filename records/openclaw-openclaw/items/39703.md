## Summary

Keep this PR open. Current main still has FIFO cron schedule cache behavior: cache hits return without Map promotion, and capacity eviction deletes the first insertion-order key. The PR remains a focused implementation candidate and its current head addresses the earlier test-review concern with key-level cache assertions.

## Best Possible Solution

Keep this PR open and review/land it, or land an equivalent maintainer patch that promotes cron cache hits with delete/re-set and keeps key-level regression coverage proving the promoted entry survives eviction while the next-oldest entry is removed.
