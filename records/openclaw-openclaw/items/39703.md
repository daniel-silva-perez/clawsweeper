## Summary

Keep this PR open. Current main still implements FIFO cron schedule cache eviction: cache hits return without refreshing Map insertion order, and eviction deletes the first inserted key. The PR remains a narrow implementation candidate, and the latest provided PR context shows the earlier test weakness was addressed with key-level cache assertions.

## Best Possible Solution

Keep this PR open and review/land it, or land an equivalent maintainer patch that promotes cron cache hits with delete/re-set and keeps key-level regression coverage proving the promoted cron expression survives eviction while the next-oldest entry is removed.
