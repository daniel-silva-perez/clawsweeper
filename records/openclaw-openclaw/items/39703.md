## Summary

Keep this PR open. Current main at 16906780fd2c481308ec4b80f96eb3b93d0add46 still has FIFO cron schedule cache behavior: cache hits return without refreshing Map order, and full-cache eviction deletes the first Map key. The current PR remains a narrow implementation candidate for #39679 and its branch now addresses the earlier review concern with key-level LRU assertions.

## Best Possible Solution

Keep this PR open and review/land it, or land an equivalent maintainer patch that promotes cron cache hits with delete/re-set and includes key-level regression coverage proving the promoted entry survives eviction while the next-oldest entry is removed.
