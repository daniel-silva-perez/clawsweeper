## Summary

Keep this PR open. Current main still does not expose structured non-self Feishu @mentions in inbound conversation metadata, and the submitted change addresses a real bundled Feishu multi-bot routing gap. It should be revised before merge to support all Feishu mention ID shapes, add focused coverage, and align with broader open Feishu arbitration work.

## Best Possible Solution

Keep this PR open and revise it into a tested Feishu mention extraction contract that excludes the receiving bot, preserves a stable identity across Feishu `open_id`/`user_id`/`union_id` shapes where available, serializes a structured `mentions` array in the untrusted inbound conversation metadata block, and coordinates the behavior with the broader arbitration policy in #38939.

## Risks / Open Questions

- Merging the PR exactly as submitted could silently omit mentions delivered only with `user_id` or `union_id`, because the proposed extraction requires `open_id`.
- The PR currently lacks focused regression coverage for Feishu mention extraction and inbound conversation-info serialization.
- The final metadata contract should be reconciled with open Feishu arbitration work in #38939 so prompt-visible metadata and plugin-level routing policy do not diverge.
- Mention display names are user-controlled; any implementation should keep them in the untrusted metadata block and use stable IDs for security-sensitive routing decisions.
