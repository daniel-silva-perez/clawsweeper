## Summary

Keep open. The report targets current Slack native slash-command behavior in the latest shipped release, and current main still registers Slack native commands by provider-level command name and routes the received command through the account/channel Slack delivered to OpenClaw. The related per-account native command naming request remains open, so there is no implemented fix or resolved canonical item to close against.

## What This Changes

The issue requests correct Slack native slash-command autocomplete and responder routing when multiple Slack Apps in one workspace register identical command names for different agents.

## Best Possible Solution

Maintainers should decide the supported Slack product contract: either document duplicate native command names across Slack Apps as unsupported and recommend unique per-app command names, or add account-aware native command naming plus tests so Slack-delivered commands resolve consistently to the channel-bound agent where Slack payload identity allows it.
