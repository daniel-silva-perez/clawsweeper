## Summary

Close as duplicate/superseded by the older open Slack native-command naming issue. Current main still lacks account-aware Slack native slash command names, and the observable cross-routing here follows the same root cause already tracked by #38302: Slack duplicate command ownership plus OpenClaw dispatching from the Slack app/account that receives the payload.

## What This Changes

The issue reports Slack native slash commands cross-routing between agents when multiple Slack Apps in one workspace register the same command names.

## Best Possible Solution

Keep #38302 as the canonical open work item, preserving this report's concrete multi-app Slack reproduction as linked context. The eventual fix should either implement account-aware Slack native command naming/mapping end to end, or document identical unprefixed Slack command names across apps as unsupported.
