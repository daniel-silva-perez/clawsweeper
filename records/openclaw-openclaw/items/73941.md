## Summary

Keep open. Current main still has ordinary path syntax normalization, workspace-boundary checks, and plugin before-tool-call hooks, but no shipped OpenClaw default or configurable middleware that removes model-inserted spaces between Han/CJK and ASCII characters in file paths. The nearby related work either covers UI markdown/link rendering or a still-open mixed CJK+ASCII read-path bug, so this request remains a product/security decision rather than a cleanup close.

## Best Possible Solution

Keep this issue open for maintainer product/security review. A good implementation would be opt-in and existence-aware, probably starting as a plugin or narrowly scoped before-tool-call path-param rewrite experiment, with core changes only if maintainers want a supported default policy. Raw exec command strings should be handled through explicit tooling or prompt guidance unless there is a carefully reviewed command-parser design.

## Risks / Open Questions
