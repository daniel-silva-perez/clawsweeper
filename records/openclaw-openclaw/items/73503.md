## Summary

Keep open. Current main still does not implement this PR’s intended CLI/docs clarification: `openclaw mcp list/show/set/unset` still use generic MCP wording, list output still says `MCP servers`, and the MCP docs still lack the prominent note that mcporter’s `config/mcporter.json` registry is separate. The linked issue #65209 is still open, and this PR remains a plausible implementation candidate.

## Best Possible Solution

Review this PR alongside #70495 and land one equivalent focused patch. The maintained change should label `openclaw mcp list/show/set/unset` as OpenClaw-managed `mcp.servers`, document that mcporter `config/mcporter.json` is handled by `mcporter list`, preserve JSON output exactly, and close #65209 only after the implementation merges.
