## Summary

Keep open. Current main still does not implement the PR’s intended CLI/docs clarification: `openclaw mcp list/show/set/unset` still use generic MCP wording, the human list output still says `MCP servers`, and the MCP docs still do not prominently state that mcporter’s `config/mcporter.json` registry is excluded. The PR remains a focused candidate for the linked open issue #65209.

## Best Possible Solution

Review and land this PR or an equivalent focused patch. The maintained change should label the human `openclaw mcp list/show/set/unset` surfaces as OpenClaw-managed `mcp.servers`, document that mcporter `config/mcporter.json` is a separate registry handled by `mcporter list`, keep JSON output unchanged, and close #65209 only after the implementation merges.
