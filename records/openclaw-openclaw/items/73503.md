## Summary

Keep this PR open. Current main still does not implement the intended CLI/docs clarification: `openclaw mcp list/show/set/unset` wording remains generic, list output still says `MCP servers`, the MCP docs do not call out the separate mcporter `config/mcporter.json` registry near the top, and #65209 is still open. The PR is a plausible focused implementation candidate, with an overlapping open PR (#70495) that maintainers should choose between.

## Best Possible Solution

Review this PR alongside #70495 and land one focused equivalent patch. The maintained change should label `openclaw mcp list/show/set/unset` as OpenClaw-managed `mcp.servers`, document that mcporter `config/mcporter.json` is handled by `mcporter list`, preserve JSON output unchanged, and close #65209 only after the implementation merges.
