## Summary

Keep this open. Current main still has immediate default-model setting, manual fallback management, OpenRouter-specific scan/probe helpers, status/auth probes, and runtime failover, but it does not provide the requested staged model migration workflow with arbitrary candidate testing, promote-with-old-primary-fallback, or user-facing rollback. The issue discussion also includes explicit maintainer keep-open guidance.

## What This Changes

Request a staged model migration workflow that can add and test a candidate fallback, promote it to primary while preserving the old primary as fallback, and roll back if needed.

## Best Possible Solution

Keep this open for a maintainer-designed core `openclaw models` migration workflow. It should build on existing status/probe, fallback, auth, and scan primitives, but add explicit staging of an arbitrary candidate, verification before promotion, old-primary fallback preservation, persisted rollback state, and docs/tests covering those config transitions.
