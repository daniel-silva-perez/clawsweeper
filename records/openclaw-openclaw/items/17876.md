## Summary

Keep this issue open. Current main still exposes immediate/default model setting, manual fallback primitives, runtime failover, OpenRouter-specific scanning, and auth-profile probes, but it does not provide the requested staged model migration workflow with arbitrary model testing, promotion, old-primary fallback insertion, or rollback. The issue also has an explicit maintainer comment saying to keep it open.

## Best Possible Solution

Keep this open for a core `openclaw models` workflow. The right implementation should build on existing status/probe, fallback, auth, and scan primitives: stage a candidate as fallback, test an arbitrary provider/model ref deeply enough to cover sub-agents/tool use, promote only after verification while preserving the old primary as fallback, persist enough previous config for rollback, and document/test the config transitions.
