## Summary

This issue should stay open. Current main still provides immediate default-model setting, manual fallback management, OpenRouter scan/probe helpers, auth/status probes, and runtime failover, but it does not provide the requested staged model migration workflow with arbitrary candidate testing, promote-with-old-primary-fallback, and user-facing rollback. The discussion also includes explicit maintainer keep-open guidance from steipete on 2026-04-26.

## Best Possible Solution

Keep this open for a maintainer-designed core `openclaw models` migration workflow. The implementation should build on existing status/probe, fallback, auth, and scan primitives, but add explicit staging of an arbitrary candidate, verification before promotion, old-primary fallback preservation, persisted rollback state, and docs/tests covering those config transitions.
