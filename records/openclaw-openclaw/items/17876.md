## Summary

Keep this issue open. Current main still has immediate model setting, manual fallback management, runtime failover, OpenRouter scan/probe primitives, and auth-profile probes, but no staged model migration workflow with arbitrary candidate testing, promotion, old-primary fallback insertion, or rollback. The issue discussion also includes explicit maintainer keep-open guidance.

## Best Possible Solution

Keep this open for a maintainer-designed core `openclaw models` workflow. The implementation should build on existing status/probe, fallback, auth, and scan primitives: stage a candidate as fallback, test an arbitrary provider/model ref deeply enough to cover sub-agents and tool use, promote only after verification while preserving the old primary as fallback, persist previous config for rollback, and add docs/tests for the config transitions.
