## Summary

Current main still does not implement the requested staged model migration workflow, and the issue discussion already includes explicit maintainer keep-open guidance. The existing code and docs cover immediate default-model setting, manual fallback management, OpenRouter-specific scan/probe helpers, status/auth probes, and runtime failover, but not arbitrary candidate staging, verification-before-promotion, old-primary fallback preservation, or user-facing rollback.

## What This Changes

The issue requests a safer model-switching workflow that stages a new model as a fallback, verifies it before promotion, preserves the old primary as fallback, and supports rollback.

## Best Possible Solution

Add a maintainer-designed core `openclaw models` migration workflow that stages an arbitrary candidate, verifies it before promotion, preserves the old primary as fallback, persists rollback state, and documents/tests the config transitions.
