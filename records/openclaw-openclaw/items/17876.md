## Summary

Keep open. Current main still provides immediate model setting, manual fallback editing, OpenRouter-specific scan/probe support, and runtime failover, but not the requested staged provider-generic test/promote/rollback workflow. The issue discussion also contains explicit maintainer keep-open guidance, so this needs product design before implementation.

## What This Changes

The issue requests a staged model migration workflow that tests a candidate model before promotion, preserves the previous primary as fallback, and supports rollback.

## Best Possible Solution

Add a maintainer-designed openclaw models migration flow that can stage any provider/model candidate, verify it before promotion, preserve the old primary in the fallback chain, store rollback state, and document/test the resulting config transitions.
