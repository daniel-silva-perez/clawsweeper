## Summary

Keep open. Current main still provides immediate default-model changes, manual fallback editing, OpenRouter-specific scan/probe support, and runtime failover, but it does not implement the requested provider-generic stage, test, promote, preserve-old-primary, and rollback workflow. The issue discussion also includes explicit maintainer guidance to keep it open for product follow-up.

## What This Changes

Request a safer `openclaw models` migration workflow that stages a new model as a fallback, verifies it before promotion, preserves the old primary, and supports rollback.

## Best Possible Solution

Add a maintainer-designed `openclaw models` migration flow that stages any provider/model candidate, verifies it before promotion, keeps the old primary in the fallback chain, records rollback state, and documents/tests the config transitions.
