## Summary

Keep open. Current main still has immediate default-model setting, manual fallback management, OpenRouter-specific scan/probe support, and runtime failover, but it does not implement the requested staged test, promote, preserve-old-primary, and rollback workflow. The issue discussion also includes explicit maintainer guidance to keep it open for product follow-up.

## What This Changes

The issue requests a staged `openclaw models` migration workflow that tests a new model as a fallback, promotes it only after verification, preserves the previous primary as fallback, and supports rollback.

## Best Possible Solution

Add a maintainer-designed `openclaw models` migration flow that stages any provider/model candidate, verifies it before promotion, keeps the old primary in the fallback chain, records rollback state, and documents/tests the config transitions.
