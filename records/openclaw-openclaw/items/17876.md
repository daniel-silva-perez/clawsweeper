## Summary

Keep open. Current main still has immediate default-model setting, manual fallback management, OpenRouter-specific scan/probe support, and runtime failover, but it does not provide a generic staged model migration workflow with verification-before-promotion, old-primary preservation, or config rollback. The existing discussion also contains explicit maintainer keep-open guidance.

## What This Changes

The issue requests a staged model migration workflow that can test a candidate model before promotion, preserve the old primary as fallback, and support rollback.

## Best Possible Solution

Add a maintainer-designed `openclaw models` migration flow that stages any provider/model candidate, verifies it before promotion, preserves the old primary in the fallback chain, records enough rollback state for a one-command restore, and documents/tests the resulting config transitions.
