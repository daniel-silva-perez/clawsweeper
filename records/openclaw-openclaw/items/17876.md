## Summary

Keep open. Current main still has immediate primary-model setting, manual fallback editing, OpenRouter-specific scan/probe support, and runtime failover, but not the requested provider-generic stage, test, promote, preserve-old-primary, and rollback workflow. The issue discussion also includes explicit maintainer guidance to keep this open for product follow-up.

## What This Changes

The issue requests a staged model migration UX that tests a new model before promotion, keeps the old primary as fallback, and supports rollback.

## Best Possible Solution

Add a maintainer-designed `openclaw models` migration flow that stages any provider/model candidate, verifies it before promotion, keeps the old primary in the fallback chain, records rollback state, and documents/tests the config transitions.
