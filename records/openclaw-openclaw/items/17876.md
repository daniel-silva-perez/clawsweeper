## Summary

Keep open. Current main still lacks the requested staged, provider-generic model migration workflow, and the issue discussion includes explicit maintainer guidance to keep it open; the existing code supports immediate primary swaps, manual fallback edits, OpenRouter-specific scan/probe behavior, runtime failover, and narrow raw-model smoke tests, but not candidate staging, verification-before-promotion, old-primary preservation, or rollback.

## What This Changes

Request a staged model migration UX that adds a new model as a fallback, verifies it before promotion, promotes it to primary, preserves the old primary as fallback, and supports rollback.

## Best Possible Solution

Add a maintainer-designed openclaw models migration flow that stages any provider/model candidate, verifies it before promotion against the intended agent surfaces, preserves the old primary in the fallback chain, stores rollback state, and documents/tests the config transitions.
