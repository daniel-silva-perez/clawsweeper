## Summary

Keep open. Current main still lacks the requested staged, provider-generic model migration workflow, and the issue discussion already includes maintainer guidance to keep it open. The existing implementation supports immediate primary swaps, manual fallback edits, OpenRouter-specific scan/probe behavior, and runtime failover, but not candidate staging, verification-before-promotion, old-primary preservation, or rollback.

## What This Changes

The issue requests a staged model migration workflow that can test a candidate model before promotion, preserve the old primary as fallback, and provide rollback.

## Best Possible Solution

Add a maintainer-designed `openclaw models` migration flow that stages any provider/model candidate, verifies it before promotion against the intended agent surfaces, preserves the old primary in the fallback chain, stores rollback state, and documents/tests the config transitions.
