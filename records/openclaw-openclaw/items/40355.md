## Summary

Keep open. Current main still lacks the requested doctor warning for default and per-agent configured primary model refs, and the live PR head is useful but not merge-ready because the diagnostic still needs alias-aware resolution, read-only/offline catalog behavior where feasible, and sanitized terminal output.

## What This Changes

The PR adds a config-validation helper, a doctor health contribution, and targeted tests to warn when `agents.defaults.model.primary` or `agents.list[].model` points at a model missing from the loaded catalog.

## Best Possible Solution

Land a focused doctor or shared validation path that warns on invalid default and per-agent primary model refs, resolves aliases the same way runtime/model-picker paths do, keeps catalog access read-only where feasible, sanitizes all terminal output, and keeps #39811 open until that behavior is merged.
