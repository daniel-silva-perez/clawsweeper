## Summary

Current main still lacks this PR's intended `openclaw doctor` warning for invalid configured default or per-agent model primaries. The linked canonical bug #39811 remains open, and this open contributor PR is still an implementation candidate rather than obsolete or already implemented work.

## What This Changes

This PR adds a `collectModelConfigWarnings` helper, a doctor health contribution, and targeted tests to warn when `agents.defaults.model.primary` or `agents.list[].model` references a model missing from the loaded catalog.

## Best Possible Solution

Keep this PR open and review or refresh it against current main. The best landing shape is a doctor health contribution or shared validation helper that warns for invalid default and per-agent primary model refs, uses alias-aware runtime-consistent resolution, keeps catalog access read-only/offline where feasible, sanitizes terminal output, and adds focused tests for defaults, per-agent models, aliases, empty catalogs, and terminal sanitization. #39811 should remain the canonical issue until the behavior ships.
