## Summary

Keep open. Current main still does not implement this PR’s requested doctor warning for default and per-agent configured primary model refs, and the live PR branch remains useful but needs maintainer review before merge because it still has alias-resolution, catalog side-effect, and terminal-output hygiene questions.

## What This Changes

The PR adds a config-validation helper, doctor health contribution, and tests to warn when `agents.defaults.model.primary` or `agents.list[].model` points to a model missing from the loaded catalog.

## Best Possible Solution

Land a focused doctor or shared validation path that warns on invalid default and per-agent primary model refs, resolves aliases consistently with runtime/model-picker behavior, keeps advisory catalog reads read-only where feasible, sanitizes terminal output, and leaves the broader fallback-notification work to its separate related item.
