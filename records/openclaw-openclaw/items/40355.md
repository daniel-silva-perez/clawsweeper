## Summary

Keep open. Current main still lacks the requested generic doctor warning for configured default or per-agent model refs, but the PR remains incorrect because the check uses a catalog that includes user-declared config rows, does not resolve aliases with runtime-equivalent semantics, and prints config-derived warning text without terminal sanitization.

## What This Changes

The PR adds a `collectModelConfigWarnings` helper, a `doctor:model-config` health contribution, and tests to warn when configured default or per-agent primary model refs are missing from the loaded model catalog.

## Best Possible Solution

Land a focused doctor or shared validation path that resolves model refs the same way runtime/model-picker paths do, distinguishes provider/runtime/static catalog evidence from user-declared config rows, reports terminal-safe warnings, and leaves broader fallback notification behavior to its separate tracking item.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Validate against catalog rows that do not come from config:** `src/flows/doctor-health-contributions.ts:407`
  - body: The new doctor check passes `loadModelCatalog({ config: ctx.cfg })`, but that catalog appends `models.providers.*.models` from the same config. In linked #39811, the bogus model is first declared there, so this can treat the bad ref as catalog-present and emit no warning. Use catalog origin/source metadata or a provider/runtime/static-only catalog for this diagnostic.
  - confidence: 0.88
- **[P2] Resolve aliases before checking configured primaries:** `src/config/validation.ts:1341`
  - body: `collectModelConfigWarnings` calls `resolveModelRefFromString` without `cfg` or an alias index, so a valid configured alias such as `fast` is parsed as `anthropic/fast` and warned as missing. Doctor should match runtime/model-picker semantics by resolving through `agents.defaults.models` aliases before checking the catalog.
  - confidence: 0.9
- **[P3] Sanitize model warning text before printing it:** `src/flows/doctor-health-contributions.ts:411`
  - body: The warning message includes model identifiers derived from local config and is passed straight to `note()`. A model id containing ANSI or control characters can spoof doctor output; route the path/message through the existing terminal sanitizer before display.
  - confidence: 0.82

## Security Review
