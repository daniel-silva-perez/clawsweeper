## Summary

Keep open. Current main still lacks a generic doctor/config warning for default or per-agent primary model refs, and the PR targets a real linked problem, but the reviewed head still has blocking catalog-source and alias-resolution correctness issues plus a low-risk terminal-output hardening issue.

## What This Changes

The PR adds a config validation helper, doctor health contribution, and tests to warn when configured default or per-agent primary model refs are missing from the loaded model catalog.

## Best Possible Solution

Land a focused doctor/config diagnostic that resolves configured model refs the same way runtime and model-picker paths do, distinguishes provider/runtime/static catalog evidence from user-declared config rows when warning about unsupported models, sanitizes terminal output, and adds regression coverage for #39811 plus alias-based configs without changing runtime failover behavior in this PR.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Validate against provider-backed catalog rows:** `src/flows/doctor-health-contributions.ts:407`
  - body: The new doctor check passes loadModelCatalog({ config: ctx.cfg }), but that catalog appends models.providers.*.models from the same config. In the linked #39811 reproduction, the bogus model is declared there first, so this can treat it as present and emit no warning. Use provider/runtime/static catalog evidence or origin metadata for this diagnostic.
  - confidence: 0.88
- **[P2] Resolve aliases before warning on primaries:** `src/config/validation.ts:1341-1344`
  - body: collectModelConfigWarnings resolves configured strings without cfg or an aliasIndex, so a valid alias from agents.defaults.models can be parsed as the default provider plus the alias text and warned as missing. Doctor should match the runtime/model-picker alias resolution path before checking catalog membership.
  - confidence: 0.9
- **[P3] Sanitize model warning output before printing:** `src/flows/doctor-health-contributions.ts:411`
  - body: The warning path and message include model identifiers derived from local config and are passed directly to note(). Route them through the existing terminal sanitizer or config issue formatter so control characters cannot spoof doctor output.
  - confidence: 0.82
