Current main still does not implement this PR’s intended `openclaw doctor` warning for invalid `agents.defaults.model.primary` and `agents.list[].model` primaries. The linked bug remains open, and this PR is still a plausible implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open and refresh it against current main. The best path is a doctor health contribution or shared validation helper that warns for invalid default and per-agent primary model refs, uses alias-aware runtime-consistent resolution, sanitizes all terminal-facing config-derived text, uses an explicit read-only/offline catalog policy for diagnostics, and adds focused tests for defaults, per-agent models, aliases, empty catalogs, and terminal sanitization. Keep #39811 as the canonical issue until this behavior ships.
