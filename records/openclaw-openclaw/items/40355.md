## Summary

Keep this PR open for maintainer and author follow-up. Current main still lacks a generic doctor/config warning for default or per-agent primary model refs, but the current PR patch has blocking semantic defects and a low-severity terminal-output security issue.

## What This Changes

The PR adds a config-validation helper, a doctor health contribution, and tests to warn when default or per-agent primary model refs are absent from the loaded model catalog.

## Best Possible Solution

Land a focused doctor or shared validation path that resolves model refs the same way runtime and model-picker paths do, distinguishes provider/static/runtime catalog evidence from user-declared config rows, keeps catalog inspection read-only where possible, sanitizes terminal output, and covers aliases plus the models.providers misconfiguration from #39811. Leave broader fallback-user-notification behavior to its separate tracking item.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Validate against catalog rows that do not come from config:** `src/flows/doctor-health-contributions.ts:408-409`
  - body: The new doctor check passes loadModelCatalog({ config: ctx.cfg }) into the warning helper, but that catalog appends models.providers.*.models from the same config. In the linked #39811 reproduction, the bogus model is first added under models.providers.openai.models, so this can treat it as present and emit no warning even though provider/runtime calls can still fail.
  - confidence: 0.88
- **[P2] Resolve aliases before checking configured primaries:** `src/config/validation.ts:1344`
  - body: collectModelConfigWarnings calls resolveModelRefFromString without cfg or an alias index, so a valid configured alias such as fast is parsed as anthropic/fast and warned as missing. Runtime and documented model input behavior resolve configured aliases first, so doctor would produce false positives for supported configs.
  - confidence: 0.9
- **[P3] Sanitize model warning text before printing it:** `src/flows/doctor-health-contributions.ts:411`
  - body: The warning message includes model identifiers derived from local config and is passed straight to note(). A model id containing ANSI or control characters can spoof doctor output; route the path/message through the existing terminal sanitizers before display.
  - confidence: 0.82

## Security Review
