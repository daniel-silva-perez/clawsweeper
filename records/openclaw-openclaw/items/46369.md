## Summary

Current main still does not clean up cron session-store entries or per-job run logs when `cron.remove` deletes a job. The issue is also paired with open, unmerged PR #46506 using closing syntax for the same bug, so it should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land PR #46506 or an equivalent narrow cron-owned patch. The implementation should remove the deleted job's base cron session-store entry and matching run entries for the relevant agent store, unlink the per-job `runs/<jobId>.jsonl` file through the existing safe path helper, preserve unrelated cron/session records, tolerate missing artifacts, and add focused regression tests. After the PR merges, this issue can close as implemented.
