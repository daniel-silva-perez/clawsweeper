## Summary

Keep open. Current main still routes `cron.remove` through the cron service remove path that only deletes the job definition, persists the cron store, re-arms the timer, and emits a removal event. It does not delete the matching cron session-store entry or unlink the per-job run log. The issue is also explicitly paired with open, unmerged fix PR #46506, so the best maintainer path is to review and land that PR or an equivalent narrow patch before closing this issue.

## Best Possible Solution

Keep this issue open and review/land #46506 or an equivalent narrow cron-owned patch. The implementation should remove the deleted job's base cron session-store entry and any matching run entries for the relevant agent store, unlink the per-job `runs/<jobId>.jsonl` file through the existing safe path helper, preserve unrelated cron/session records, tolerate missing artifacts, and add focused regression tests. After the PR merges, this issue can close as implemented.
