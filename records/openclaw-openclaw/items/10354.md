## Summary

Keep this issue open. Current main still exposes the shared message tool `channel` parameter as `Type.Optional(Type.String())` with no TypeBox description or enum, and the schema is still returned as the tool parameters. The existing prompt guidance and current-channel fallback reduce some failures but do not address the requested schema metadata, while the two linked implementation PRs are closed unmerged.

## What This Changes

Request to add guidance, and possibly registry-derived enum metadata, to the shared message tool `channel` parameter so agents pass a provider/plugin id such as `slack` rather than a destination conversation id.

## Best Possible Solution

Land a narrow core change in `src/agents/tools/message-tool.ts` that keeps `channel` optional but adds a TypeBox description saying it expects a messaging provider or channel/plugin id such as `slack` or `discord`, not the destination conversation/channel/user id. Add a focused assertion in `src/agents/tools/message-tool.test.ts`. Defer enum support unless it is derived from configured or registered channel/plugin ids.
