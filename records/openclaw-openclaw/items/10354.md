## Summary

Keep open. Current main still exposes the shared message tool `channel` parameter as an undescribed optional string, so the requested schema guidance is not implemented; prompt text and fallback routing help some cases but do not fix the tool schema metadata, and the related implementation PRs are closed unmerged.

## What This Changes

The issue requests adding schema guidance, and possibly an enum, for the shared message tool `channel` parameter so agents pass provider or channel plugin ids rather than destination channel IDs.

## Best Possible Solution

The message tool should keep `channel` optional, describe it as the messaging provider or channel plugin id such as `slack` or `discord`, clearly distinguish it from destination targets, and add a focused schema assertion; enum support should only be added if it can include dynamic plugin ids without blocking third-party channels.
