## Summary

Keep this issue open. Current main still exposes the shared message tool `channel` parameter as a bare optional string with no schema description or enum, while the prompt guidance and current-channel fallback do not satisfy the requested tool-schema metadata. The linked implementation attempts are closed unmerged, so the remaining work is still actionable.

## What This Changes

The issue requests adding clearer TypeBox schema metadata, and possibly an enum, for the shared message tool `channel` parameter so agents pass provider or plugin ids rather than destination conversation ids.

## Best Possible Solution

Land a narrow core update that keeps `channel` optional but adds a TypeBox description explaining it expects a messaging provider or channel/plugin id such as `slack` or `discord`, not the destination conversation/channel/user id. Add a focused assertion in `src/agents/tools/message-tool.test.ts`; defer enum support unless it is dynamically derived from the configured or registered channel/plugin ids.
