## Summary

Keep open. Current main and the latest release still expose the shared message tool `channel` parameter as an undescribed optional string, so the requested agent-visible schema guidance is not implemented. Runtime prompt guidance and channel-context fallback reduce adjacent `Unknown channel` failures, but they do not add schema metadata, and related PRs #10364 and #13584 are closed unmerged.

## What This Changes

Request to add a description, and possibly a plugin-aware enum, to the shared message tool `channel` parameter so agents pass provider/plugin ids rather than destination channel or user ids.

## Best Possible Solution

Land a narrow core schema/test update that keeps `channel` optional, adds TypeBox description text explaining it expects a messaging provider or channel plugin id such as `slack` or `discord` rather than a destination target, and defers enum support unless it can be generated from plugin-aware channel discovery.
