## Summary

Keep this issue open. Current main still exposes the shared `message` tool `channel` parameter as a bare optional string with no TypeBox description or enum, while the linked implementation PRs are closed unmerged. The narrow description-plus-test fix remains valid; a static enum should be avoided unless it is derived from the dynamic channel/plugin registry.

## Best Possible Solution

Land a narrow core change in `src/agents/tools/message-tool.ts` that keeps `channel` optional but adds a TypeBox description saying it expects a messaging provider or channel/plugin ID such as `slack` or `discord`, not the destination conversation/channel/user ID. Add a lightweight assertion in `src/agents/tools/message-tool.test.ts`. Defer enum support unless maintainers intentionally derive it from configured or registered channel/plugin IDs.
