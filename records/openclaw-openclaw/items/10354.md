## Summary

Keep this issue open. Current main still exposes the shared message tool `channel` parameter as a bare optional string with no schema description or enum, while the prompt-only guidance and runtime fallback do not fully address the reported tool-schema ambiguity. The two linked PR attempts are closed unmerged, and no open PR referencing this issue was found.

## Best Possible Solution

Land a narrow core change in `src/agents/tools/message-tool.ts` adding a `channel` schema description that says it expects a messaging provider or channel plugin id such as `slack` or `discord`, not a conversation/channel target id. Add a focused assertion in `src/agents/tools/message-tool.test.ts`. Defer enum support unless maintainers derive it dynamically from configured or registered channel/plugin ids.
