## Summary

Keep open. Current main and the latest release still expose the shared message tool `channel` parameter as an undescribed optional string, so the requested agent-visible schema guidance is not implemented. Prompt guidance and runtime fallback reduce adjacent failures, but they do not add schema metadata, and the two related implementation PRs are closed unmerged.

## What This Changes

Request to add description and possibly plugin-aware enum metadata to the message tool `channel` parameter so agents pass provider or channel plugin ids instead of destination conversation ids.

## Best Possible Solution

Land a narrow core schema/test update that keeps `channel` optional, adds TypeBox description text explaining it expects a messaging provider or channel plugin id such as `slack` or `discord` rather than a destination target, and defers enum support unless it can be derived from plugin-aware channel discovery.

In `src/agents/tools/message-tool.ts`, keep the `channel` parameter optional and string-typed, but add TypeBox description metadata explaining that it expects a messaging provider or channel plugin id such as `slack` or `discord`, not the destination channel/conversation/user id. Do not add a hard-coded enum unless it is derived from the active plugin/channel registry and remains compatible with external channel plugins.
