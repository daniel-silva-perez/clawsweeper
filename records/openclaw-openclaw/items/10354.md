## Summary

Keep open. Current main still exposes the shared message tool `channel` parameter as an undescribed optional string with no enum metadata, and the related implementation PRs are both closed unmerged. The runtime prompt and fallback routing reduce adjacent failures but do not add the requested agent-visible schema guidance.

## What This Changes

The issue requests a TypeBox description, and possibly a safe enum, for the shared message tool `channel` parameter so agents pass provider or channel plugin ids instead of destination conversation ids.

## Best Possible Solution

Add a focused core schema update that keeps `channel` optional, describes it as a messaging provider or channel plugin id such as `slack` or `discord` rather than a destination target, and covers that metadata with a narrow message-tool schema test. Defer enum support unless it can be generated from registered channel/plugin ids without blocking third-party plugins.
