## Summary

Keep this PR open. Current main now fixes the generic repeated-boundary merge corruption by appending non-prefix deltas directly, but it still does not implement the PR's directive-prefix whitespace normalization: inline directive display stripping returns leading whitespace as-is, and the shared live-chat projector merges that text without trimming or directive-aware cleanup. Because this is a PR, current main needs to definitely implement the intended change before cleanup closure, and the exact inline-directive WebSocket regression is not fully covered on main.

## Best Possible Solution

Keep this PR open as a stale but still relevant implementation candidate. Rebase or replace it with a small fix in src/gateway/live-chat-projector.ts that normalizes only whitespace introduced by stripped leading inline directive tags before live merge, preserves intentional delta separators, and adds an exact regression through the shared projector or Gateway event path for the reported xumum/GOOGLE corruption.
