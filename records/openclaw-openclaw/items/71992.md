## Summary

Keep open. Current main still has the Control UI terminal-event ordering described in the report: final and aborted events can write an assistant message into committed chat history while the live stream field is still populated. An open closing PR already targets the narrow UI race, so this issue should wait for that PR or an equivalent fix to land.

## What This Changes

The report requests a Control UI WebChat fix so final and aborted assistant chat events clear live stream state before writing the committed assistant message to chat history.

## Best Possible Solution

Review and land PR #72004, or apply the same narrow change directly: snapshot fallback stream text, clear `chatStream`, `chatRunId`, and `chatStreamStartedAt` before mutating `chatMessages` in both final and aborted terminal paths, with an order-sensitive regression test. Keep broader transcript idempotency work under #69208.
