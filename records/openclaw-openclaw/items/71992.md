## Summary

Current main still matches the reported Control UI rendering race. The final and aborted chat event paths append an assistant message to `chatMessages` before clearing `chatStream`, and the renderer still builds the chat list from both committed messages and the live stream. PR #72004 is open and unmerged as the implementation candidate, so this issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land PR #72004 or an equivalent patch. The fix should snapshot fallback stream text, clear `chatStream`, `chatRunId`, and `chatStreamStartedAt` before mutating `chatMessages` in both `final` and `aborted`, then add an order-sensitive regression test that observes the state at the exact `chatMessages` write. Broader transcript duplication should continue under #69208 rather than closing this focused Control UI race early.
