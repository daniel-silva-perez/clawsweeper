## Summary

Keep this PR open. Current main still leaves an existing automatic `provider:default` auth profile override pinned in `resolveSessionAuthProfileOverride` when the profile is valid, same-provider, not in cooldown, and compaction has not advanced; it does not canonicalize that legacy OAuth default to a non-default OAuth alias. The PR remains a plausible focused auth-routing fix, but it needs a current-main port and maintainer review of the identity-matching rules before merge.

## Best Possible Solution

Keep this PR open. Rebase or port it onto current main, add a narrowly scoped canonicalization step in `src/agents/auth-profiles/session-override.ts` for automatic legacy OAuth `:default` overrides, reuse or align with the existing OAuth identity/repair helpers, preserve the mismatch and rotation-progress regression coverage from the PR discussion, then run the targeted auth-profile test lane and changed gate before maintainer review.
