## Summary

Keep open. Current main still does not implement this PR’s intended auth-profile canonicalization: a steady-state automatic `provider:default` OAuth session override remains pinned unless the session is new, compaction advances, or the profile enters cooldown. The PR remains a focused, plausible fix, but it needs a current-main port and maintainer review of the credential identity-matching rules.

## Best Possible Solution

Keep this PR open. Rebase or port it onto current main, add a narrow canonicalization step in `src/agents/auth-profiles/session-override.ts` for automatic legacy OAuth `:default` overrides, preserve provider/order/cooldown semantics and explicit user overrides, align identity matching with existing OAuth helpers, keep the negative mismatch/conflict and rotation-progress regression tests, then run the targeted auth-profile test lane and changed gate before maintainer review.
