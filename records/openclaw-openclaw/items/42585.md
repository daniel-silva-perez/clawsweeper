## Summary

Keep open. Current main still does not implement this PR’s intended legacy OAuth `provider:default` session-override canonicalization; the resolver preserves a valid automatic override until new-session, compaction, or cooldown rotation, and there is no current regression coverage for the PR’s canonicalization and identity-safety cases.

## Best Possible Solution

Keep this PR open. Rebase or port the behavior onto current main by adding OAuth-only legacy `:default` canonicalization inside `src/agents/auth-profiles/session-override.ts`, while preserving explicit user overrides, auth order semantics, cooldown handling, and rotation progress. Keep the mismatch/conflict and non-OAuth-default regression coverage, then run the targeted auth-profile test lane plus the changed gate before maintainer review.
