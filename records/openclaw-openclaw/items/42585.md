## Summary

Current main still does not implement this PR’s intended auth-profile canonicalization. A valid automatic `provider:default` OAuth session override remains pinned unless the resolver enters the new-session, compaction, missing-profile, or cooldown paths, and the current session-override tests do not cover the PR’s legacy-default canonicalization cases.

## Best Possible Solution

Keep this PR open. Rebase or port the focused behavior onto current main by adding OAuth-only legacy `:default` canonicalization inside `src/agents/auth-profiles/session-override.ts`, preserving explicit user overrides, provider alias/order semantics, cooldown handling, and rotation progress. Align the identity-matching rule with the existing OAuth identity helpers or document a narrower canonicalization-specific rule, keep the negative mismatch/conflict and non-OAuth-default tests, then run the targeted auth-profile test lane and changed gate before maintainer review.
