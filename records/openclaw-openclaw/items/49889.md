## Summary

Current main has partial improvements: Telegram preview-finalized finals now feed the internal `message:sent` hook, and the lane code distinguishes `preview-finalized`, `preview-retained`, `sent`, and `skipped` internally. The central issue is still open, though, because the public hook/log surface does not expose an explicit delivery outcome, and paths like preview-retained or hook-canceled can still be opaque during incident review.

## Best Possible Solution

Keep this issue open and implement an additive Telegram delivery-observability contract: emit structured final outcome metadata for fresh sends, preview-finalized edits, preview-retained cases, hook-canceled skips, transport failures, and intentional skips; include tests for each path and document the new hook/log fields in the Telegram and hooks docs.
