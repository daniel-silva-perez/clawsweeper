## Summary

Keep this issue open. Current main has partial Telegram preview-finalization improvements, but it still lacks the explicit operator-visible delivery outcome contract requested here: `preview-finalized`, `preview-retained`, fresh `sent`, hook-canceled, transport-failed, and intentional skips are not all exposed through public hook/log/docs surfaces.

## Best Possible Solution

Keep this open and implement an additive Telegram delivery-observability contract: expose a structured final delivery outcome for fresh sends, preview-finalized edits, preview-retained cases, hook-canceled skips, transport failures, and intentional skips; emit it consistently through internal and plugin hooks or a companion delivery event; add lane/dispatch tests for each path; document the fields in Telegram and hooks docs.
