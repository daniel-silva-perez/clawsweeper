## Summary

Keep open. Current main still probes `/tmp/openclaw` before `os.tmpdir()` and can select the Windows `C:\tmp` mapping described; PR #73533 is still open and unmerged with closing syntax, so this issue should remain open until that fix or an equivalent one lands.

## What This Changes

Make `resolvePreferredOpenClawTmpDir()` skip the POSIX `/tmp/openclaw` preference on Windows and use the existing `os.tmpdir()`-based OpenClaw temp directory instead.

## Best Possible Solution

Land PR #73533 after review, or replace it with an equivalent narrow change that gates the POSIX preferred path off on `win32`, keeps the existing trusted `os.tmpdir()` fallback, and adds regression coverage for both Windows and non-Windows behavior.

## Reproduction Assessment

Yes. The current source gives a high-confidence reproduction path: when `/tmp/openclaw` is accessible, the resolver returns it before calling `tmpdir()`, which maps to `C:\tmp\openclaw` on the reported Windows setup.

## Solution Assessment

Yes. Skipping the POSIX preference on Windows and reusing the existing trusted fallback is the narrowest maintainable fix; the open PR appears aligned, but it still needs normal review and merge before this issue is resolved.
