## Summary

Keep open. Current main still blocks the reported project-local `.env` overrides for `OPENCLAW_CONFIG_PATH` and `OPENCLAW_STATE_DIR`, the docs and tests assert that fail-closed policy, and open PR #57416 remains the implementation candidate with closing syntax for this issue.

## Best Possible Solution

Keep this issue open and review PR #57416 as the implementation candidate. If maintainers want project-local config selection, land a narrow, security-reviewed exception for `OPENCLAW_CONFIG_PATH` with updated tests and docs while keeping `OPENCLAW_STATE_DIR` blocked. If the current fail-closed workspace `.env` policy is intentional, close the PR and answer this issue with the supported alternatives: process environment, service environment, trusted `$OPENCLAW_STATE_DIR/.env`, or `env.shellEnv`.
