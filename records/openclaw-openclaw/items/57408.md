## Summary

Keep open. Current main still blocks the reported project-local `.env` overrides for `OPENCLAW_CONFIG_PATH` and `OPENCLAW_STATE_DIR`, the security docs describe that fail-closed policy, and an open PR is already the implementation candidate with closing syntax for this issue.

## Best Possible Solution

Keep this issue open and review the linked open PR as the implementation candidate. If maintainers want project-local config selection, land a narrow security-reviewed exception for the intended safe `OPENCLAW_CONFIG_PATH` behavior with tests and docs. If the fail-closed workspace `.env` policy is intentional, close the PR and answer this issue with the supported alternatives: process env, service env, trusted `$OPENCLAW_STATE_DIR/.env`, or `env.shellEnv`.
