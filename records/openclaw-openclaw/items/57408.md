## Summary

Keep this issue open. Current main still blocks `OPENCLAW_CONFIG_PATH` and `OPENCLAW_STATE_DIR` from workspace/project `.env` loading, the tests now assert that behavior, and an open PR already references this issue with closing syntax. That PR needs maintainer review or closure before this issue should be resolved.

## Best Possible Solution

Review the linked open PR as the implementation candidate. If maintainers want project-local config selection, land a narrow, security-reviewed change that allows only the intended safe `OPENCLAW_CONFIG_PATH` shape with regression tests and docs updates. If the current fail-closed workspace `.env` policy is intentional, close the PR and respond on this issue with the supported alternatives: shell/process env, service env, or trusted `$OPENCLAW_STATE_DIR/.env`.
