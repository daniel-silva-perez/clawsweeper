## Summary

Keep open. Current main at 1b25dcf57a9fca5d00cf275046e20a2e3ea3730c still does not implement this PR's intended behavior: the Channels view treats snapshot channelMeta/channelOrder as the complete render list, so a custom-only snapshot can still omit built-in channel cards. The linked feature request #31379 remains open, and this PR is still the focused implementation lane.

## Best Possible Solution

Keep this PR open for maintainer review, rebase, or equivalent reimplementation. The fix should update ui/src/ui/views/channels.ts so snapshot-provided custom/loaded IDs are merged with the built-in channel order, deduped while preserving snapshot order first, and covered by focused regression tests for custom-only channelMeta, mixed channelOrder, and no-snapshot fallback. Prefer reusing the catalog-backed built-in order from src/channels/ids.ts if the UI import boundary allows it.
