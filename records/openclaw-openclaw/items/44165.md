## Summary

Keep open. Current main still does not implement this PR's intended behavior: the Channels view treats snapshot `channelMeta` or `channelOrder` as the complete render list, so a custom-only channel snapshot can still hide built-in channel cards. The linked feature request #31379 remains open, and this PR is still a focused implementation lane, though the patch should be refreshed to use the current catalog-backed built-in channel order rather than a hard-coded legacy list.

## Best Possible Solution

Keep this PR open for maintainer review, rebase, or equivalent reimplementation. The fix should update `ui/src/ui/views/channels.ts` so snapshot-provided custom/loaded IDs are merged with a catalog-backed built-in/bundled channel order, deduped while preserving snapshot order first, and covered by focused regression tests for custom-only `channelMeta`, mixed `channelOrder`, and no-snapshot fallback. Prefer a current source such as `src/channels/ids.ts` or a UI-safe generated/metadata seam over a hard-coded legacy list, and avoid changing backend plugin loading behavior unless maintainers explicitly choose that direction.
