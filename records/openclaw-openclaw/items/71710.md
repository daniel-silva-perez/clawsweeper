## Summary

Keep open. Current main has internal timeout and abort machinery for Gateway-owned agent runs, but the `openclaw agent` CLI entry point still does not bind external SIGTERM/SIGINT from cron/coreutils `timeout` to a bounded shutdown, a gateway-run abort, or embedded-fallback cancellation. The docs also do not warn OS-cron users to use `timeout -k`, so the reported operator problem is still actionable.

## Best Possible Solution

Keep this issue open and implement a focused `openclaw agent` shutdown path: on SIGTERM/SIGINT, abort the active gateway request and local embedded/fallback run, ask the Gateway to stop the accepted run when a runId is known, preserve session-lock cleanup, and exit with the conventional signal code after a bounded grace window. Add a short docs note for OS cron/supervisor wrappers, including `timeout -k`, as compatibility guidance for older releases and non-cooperative supervisors.
