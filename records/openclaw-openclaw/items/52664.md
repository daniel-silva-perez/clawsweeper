## Summary

Keep this PR open. Current main still does not expose `rawBody` on the requested plugin hook events, so the PR is not obsolete, but the proposed patch should not merge as-is because it forwards `rawBody` for system-event runs despite documenting that heartbeat, cron, CLI, and other non-channel sources should receive `undefined`.

## What This Changes

The PR adds optional `rawBody` to `before_prompt_build`, legacy `before_agent_start`, and `agent_end` hook events, threading it through reply, embedded-runner, follow-up, and collect queue paths with generated SDK baseline hash updates.

## Best Possible Solution

Land a rebased version that exposes `rawBody` only for user-originated channel runs across immediate, deferred follow-up, and collect paths, keeps non-channel sources undefined, documents the hook field for plugin authors, and adds narrow coverage for channel raw text plus heartbeat/cron omission.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Omit rawBody for system-event runs:** `src/auto-reply/reply/get-reply-run.ts:403-404`
  - body: The new hook contract says `rawBody` is undefined for CLI, heartbeat, cron, and other non-channel sources, but this computes the value for every `runPreparedReply` call and later forwards it into the hook pipeline. Heartbeat and cron paths already populate `CommandBody`/`RawBody`, so plugins would receive scheduler prompts as user raw text. Gate this value on a user/channel-originated run before storing or forwarding it.
  - confidence: 0.86
