## Summary

Keep open. Current main still lacks `rawBody` on the relevant plugin hook events, so the PR is not obsolete, but the provided latest diff still has a concrete contract bug: it derives and forwards `rawBody` for every run even though the new hook docs say non-channel/system runs must leave it undefined.

## What This Changes

The PR adds optional `rawBody` to `before_prompt_build`, legacy `before_agent_start`, and `agent_end` hook events and threads it through reply, embedded-runner, deferred follow-up, and collect queue paths with generated SDK baseline hash updates.

## Best Possible Solution

Land a rebased version that exposes `rawBody` only for user-originated channel runs across immediate, deferred follow-up, and collect paths, keeps CLI/heartbeat/cron/system-event runs undefined, documents the hook field for plugin authors, and adds focused tests for channel raw text plus non-channel omission.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Gate rawBody to user-originated channel runs:** `src/auto-reply/reply/get-reply-run.ts:403-404`
  - body: The new hook contract says `rawBody` is undefined for CLI, heartbeat, cron, and other non-channel sources, but these lines derive it from `CommandBody`/`RawBody`/`Body` for every run and later forward it into the hook pipeline. Heartbeat and cron callers already populate those fields, so plugins would receive scheduler/system prompts as raw user text unless this value is gated on a real user/channel-originated run.
  - confidence: 0.86
