## Summary

Keep this PR open. Current main still does not expose `rawBody` on `before_prompt_build`, legacy `before_agent_start`, or `agent_end` hook events, and the run/followup plumbing that would preserve it is absent. The PR remains a focused additive SDK/runtime change rather than obsolete cleanup; the main maintainer question is whether its run-level `rawBody` semantics are acceptable for steer/steer-backlog paths.

## Best Possible Solution

Keep the PR open and continue normal review. The best path is to land an updated version that adds `rawBody` to the public hook event types, threads it through immediate/deferred/collect runs, documents the SDK contract, updates generated API baselines, and either accepts or explicitly fixes the steer-path run-level semantics with focused tests.

## Risks / Open Questions

- The steer and steer-backlog path remains a product/API semantics question: `rawBody` can be run-level while `event.messages` may include later steered text, so maintainers should decide whether that is acceptable or require structured steer metadata before merge.
- `rawBody` exposes clean original user text directly to plugin hook handlers. Even though hooks already receive prompt/messages in these surfaces, the merge review should confirm this is aligned with the existing prompt-injection and conversation-access policy and docs.
