## Summary

Keep open. Current main still does not implement outbound Telegram album/media-group sending: the message action accepts one media URL, the core Telegram send helper loads one media item, reply delivery loops over media URLs one at a time, and outbound payload delivery intentionally fans out sequential sends. There is also an open draft external PR with closing syntax for this issue, so cleanup should wait for maintainers to review, port, land, or supersede that implementation.

## What This Changes

Add Telegram media-group/album sending to the message tool so 2-10 related photos or videos can be sent with a shared caption instead of separate messages.

## Best Possible Solution

Implement album support in the bundled Telegram plugin by using Telegram sendMediaGroup for eligible 2-10 photo/video groups across direct message actions, reply delivery, and outbound payload delivery, while preserving existing single-media and non-groupable fallbacks with focused tests and Telegram/message docs.
