## What This Changes

The issue requests Telegram album/media-group sending for multiple media files through the message tool, with a shared caption.

## Best Possible Solution

Review the open fork implementation at https://github.com/julia-meshcheryakova/openclaw/pull/1 and either land it, port it to current main, or explicitly supersede it. ## Reproduction Assessment

Yes. A high-confidence code-level reproduction exists: send two media URLs through the current Telegram reply or outbound payload path and the implementation fans them out into individual send calls; direct Telegram message actions also accept only one media string today.

## Solution Assessment

Unclear for the open fork PR until maintainers review its current fit. The best direction is to integrate album sending into the existing Telegram send/delivery pipeline, not add a separate action that leaves current `mediaUrls` reply and outbound payload paths sequential.
