## Summary

Keep this issue open. Current main still lacks outbound Telegram media-group/album support: the Telegram plugin only exposes singular `mediaUrl` direct sends, reply delivery loops through `mediaUrls` one item at a time, outbound payload delivery uses the shared sequential media helper, and docs only describe single-media Telegram sends. The issue is also paired with an open external PR that uses closing syntax, so the right cleanup path is maintainer review/port/merge or close of that implementation, not closing this issue now.

## Best Possible Solution

Keep this issue open and review, port, and land or close the open external implementation PR that references this issue. The eventual fix belongs in the bundled Telegram plugin: add a guarded `sendMediaGroup` path for 2-10 groupable photos/videos across reply delivery, direct message actions, and outbound payload delivery, with focused Telegram tests and docs while preserving current single-media and non-groupable fallbacks.

## Risks / Open Questions

- A partial implementation could leave inconsistent behavior between reply delivery, direct Telegram `sendMessage` actions, and outbound payload delivery.
- Telegram albums have API-specific constraints, especially 2-10 items, groupable media types, caption placement, caption length, threading/reply parameters, and no inline keyboard on the album itself.
- Batch loading multiple local/remote media items should reuse the existing media loading limits and local-root checks so album support does not expand the media fetch surface unexpectedly.
