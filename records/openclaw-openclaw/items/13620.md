## Summary

Keep this issue open. Current main still does not implement outbound Telegram media-group/album sending: the direct message action accepts one media value, `sendMessageTelegram` loads and sends one media item, reply and outbound payload paths fan out `mediaUrls` sequentially, and docs/tests still describe or assert singular/sequential behavior. The issue also has an open external PR with closing syntax, so cleanup should wait for maintainers to review, port, land, or explicitly close that implementation.

## What This Changes

The issue requests adding Telegram album/media-group support to the message tool so multiple media paths can be sent as a Telegram `sendMediaGroup` album with a shared caption.

## Best Possible Solution

Review the open external implementation and either land/port a current-main version or explicitly supersede it. The end state should be a guarded `sendMediaGroup` path in the bundled Telegram plugin for 2-10 groupable photos/videos across direct message actions, reply delivery, and outbound payload delivery, with preserved single-media/non-groupable fallbacks plus focused tests and Telegram/message docs.
