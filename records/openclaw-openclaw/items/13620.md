## Summary

Keep open. Current main still lacks outbound Telegram album/media-group sending: the Telegram action, send helper, reply delivery, and outbound payload paths all send one media item at a time, and an open fork PR is already linked as the implementation candidate.

## What This Changes

The issue requests Telegram message-tool support for sending multiple media files as a Telegram album/media group with a shared caption.

## Best Possible Solution

Review the open fork implementation and either land it, port a current-main version, or explicitly supersede it. The finished behavior should use Telegram sendMediaGroup for eligible grouped media across direct message actions, reply delivery, and outbound payload delivery, with focused tests and Telegram/message docs while preserving existing fallbacks.
