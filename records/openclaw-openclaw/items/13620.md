## Summary

Keep this issue open. Current main still lacks outbound Telegram media-group/album support: the direct message action and docs expose a single media value, reply delivery and outbound payload delivery fan out media URLs sequentially, and an open fork PR already references this issue with closing syntax, so maintainers should review, port, land, or explicitly close that implementation before cleanup closure.

## What This Changes

The issue requests Telegram outbound album support in the message tool so multiple photos or videos can be sent via Telegram media groups with a shared caption.

## Best Possible Solution

Keep this issue open. The best path is to review, port, land, or explicitly close/supersede the open fork implementation; the actual fix belongs in the bundled Telegram plugin by adding a guarded `sendMediaGroup` path for 2-10 groupable photos/videos across direct message actions, reply delivery, and outbound payload delivery, while preserving single-media and non-groupable fallbacks plus focused tests and Telegram/message docs.
