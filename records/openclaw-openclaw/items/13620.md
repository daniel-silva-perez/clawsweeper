## Summary

Keep this issue open. Current main still sends Telegram outbound media one item at a time: direct message actions accept a single mediaUrl, reply delivery iterates mediaUrls through individual sendPhoto/sendVideo/sendAudio/sendDocument calls, outbound payload delivery uses the shared sequential media helper, and docs/tool schemas do not expose a media-group action or array parameter. There is also an open fork PR with closing syntax for this issue, so maintainer review, port, landing, or closure of that implementation should happen before closing this issue.

## Best Possible Solution

Keep this issue open. The best path is to review, port, land, or close the open fork implementation, with the actual fix in the bundled Telegram plugin: add a guarded sendMediaGroup path for 2-10 groupable photos/videos across direct message actions, reply delivery, and outbound payload delivery; preserve existing single-media and non-groupable fallbacks; add focused tests and docs.
