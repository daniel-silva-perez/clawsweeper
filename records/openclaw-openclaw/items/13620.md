## Summary

Keep this issue open. Current main still lacks outbound Telegram album/media-group support: no `sendMediaGroup` call or InputMedia builder exists in the Telegram plugin, reply delivery still loops through `mediaUrls` with single-media Telegram APIs, direct Telegram message actions accept one `mediaUrl`, outbound payload delivery sequences media one item at a time, and docs only describe single-media sends. An open external PR already references this issue with closing syntax, so the issue should remain open until that work is reviewed, ported, and merged or closed.

## Best Possible Solution

Keep this issue open and review, port, and land or close the open external implementation PR that references this issue. The eventual fix belongs in the bundled Telegram plugin: add a guarded `sendMediaGroup` path for 2-10 groupable photos/videos across reply delivery, direct message actions, and outbound payload delivery, with focused Telegram tests and docs while preserving current single-media and non-groupable fallbacks.
