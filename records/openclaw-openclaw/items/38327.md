## Summary

Keep open. Current main still only sanitizes Google thinking payloads for `model.api === "google-generative-ai"`, so the reported `google-vertex` and `google-gemini-cli` wrapper paths remain uncovered. Open PR #73416 is the focused unmerged implementation candidate and uses closing syntax for this issue, so the issue should remain open until that PR is reviewed and landed or rejected.

## Best Possible Solution

Keep this issue open. Review and either land or reject PR #73416 as the focused wrapper-guard fix, with regression coverage for `google-vertex/gemini-3.1-pro-preview`, `google-vertex/gemini-3-flash-preview`, and `google-gemini-cli` through the embedded wrapper path. If crashes remain after that, split the gaxios/native-fetch fallback into a separate dependency-patch PR with explicit maintainer approval.
