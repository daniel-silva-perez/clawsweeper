## Summary

Keep open. Current main still gates the Google thinking payload wrapper to `model.api === "google-generative-ai"`, while current Google Vertex paths can resolve through `google-vertex`, `google-gemini-cli`, or OpenAI-compatible metadata. An open PR, #73416, explicitly closes this issue, so the issue should stay open until that PR is reviewed and either landed or rejected.

## Best Possible Solution

Keep this issue open. Review #73416 as the current focused implementation candidate for the Google thinking-wrapper gap, require targeted regression coverage for `google-vertex/gemini-3.1-pro-preview` and `google-vertex/gemini-3-flash-preview`, then let GitHub close this issue after merge. Track the gaxios/native-fetch auth path separately if maintainers confirm it still reproduces after the wrapper fix.
