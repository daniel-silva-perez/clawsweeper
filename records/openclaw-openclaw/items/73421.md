## Summary

Keep open. This is a fresh v2026.4.26 regression report with concrete Gateway WebSocket, Control UI/WebChat, browser.request, node.list, and Slack Socket Mode symptoms. Current main has adjacent post-release fixes, including Slack pong-timeout tuning and startup/runtime work, but I did not find source, changelog, test, or history evidence that clearly fixes the combined NAS Docker stall and delayed-delivery regression.

## Best Possible Solution

Keep the issue open and investigate it as a Gateway control-plane/browser-path regression with Slack as one visible downstream symptom. The best next step is to collect timestamped traces for node.list, browser.request, WebChat reconnects, Gateway WebSocket handshakes, event-loop delay, and Slack socket reconnects on 2026.4.23, 2026.4.26, and current main, then add a targeted regression test or latency guard once the blocking path is identified.
