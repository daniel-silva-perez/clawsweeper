## Summary

Keep open. Current main still lacks the requested OpenRouter Auto Router allowlist support, but this PR is not merge-ready because the payload injection is broader than the documented Auto-model surface, the new core test crosses the plugin API boundary, and the wildcard reasoning guard misses a supported class of OpenRouter patterns.

## What This Changes

The PR adds an OpenRouter `autoRouter.allowedModels` extra-param path that injects `plugins: [{ id: "auto-router", allowed_models: [...] }]`, exports the helper, and adds OpenRouter/provider extra-param tests.

## Best Possible Solution

Revise the PR so the allowlist plugin is applied only to the OpenRouter Auto model, expose any reusable helper through `extensions/openrouter/api.ts` or keep the coverage inside OpenRouter plugin tests, conservatively handle unresolved Grok/x-ai wildcard patterns, and document the accepted config with a changelog entry if maintainers accept this parameter shape.

## Reproduction Assessment

Yes for the blocking review findings. The source-backed reproduction is to configure `autoRouter.allowedModels` in shared/default params and run a fixed OpenRouter model: the PR head wraps any OpenRouter request because the injection block has no Auto-model guard.

## Solution Assessment

No. The proposed implementation is directionally useful, but it is not the narrowest maintainable solution until it is gated to the Auto model, aligned with the plugin API boundary, and hardened for supported wildcard patterns.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Guard auto-router injection to Auto model requests:** `extensions/openrouter/stream.ts:99-107`
  - body: `autoRouter.allowedModels` is documented as an OpenRouter Auto constraint, but this block runs for every OpenRouter model whenever the params are present. Since shared/default params are merged before provider wrapping, a global allowlist can add an Auto Router plugin payload and reasoning suppression to fixed-model calls. Gate this path to the Auto model request.
  - confidence: 0.9
- **[P2] Import the helper through the plugin API seam:** `src/agents/pi-embedded-runner-extraparams.test.ts:4`
  - body: This core agent test imports `injectAutoRouterPlugin` from the OpenRouter runtime entrypoint. The extension boundary requires core tests needing bundled plugin helpers to consume `api.ts` or another stable public seam, so routine entrypoint refactors can break unrelated core tests. Export the helper from `extensions/openrouter/api.ts` or keep this coverage in plugin tests.
  - confidence: 0.86
- **[P2] Treat broad Grok wildcard allowlists as reasoning unsafe:** `extensions/openrouter/stream.ts:57-59`
  - body: OpenRouter allowed-model patterns support provider-agnostic wildcards, so a pattern like `*/grok-*` can still route Auto Router to x-ai/Grok models while this helper returns false. That leaves proxy reasoning enabled for the same family the existing guard treats as unsafe. Treat unresolved Grok/x-ai wildcards conservatively or resolve the allowlist against a model registry.
  - confidence: 0.82
