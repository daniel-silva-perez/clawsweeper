## Summary

Keep open. Current main does not implement `params.autoRouter.allowedModels`, and the PR addresses a real OpenRouter Auto Router capability, but the branch is not ready to merge because the new payload injection is applied to every OpenRouter model instead of only `openrouter/auto`, and the new core-side agent test imports the OpenRouter plugin entrypoint rather than a public plugin seam.

## What This Changes

The PR adds an OpenRouter auto-router allowlist path that injects an `auto-router` plugin payload from `params.autoRouter.allowedModels`, adjusts reasoning handling for x-ai allowlists, exports the helper, and adds OpenRouter/provider extra-param tests.

## Best Possible Solution

Revise the PR so `autoRouter.allowedModels` is only honored for the OpenRouter Auto model, expose any helper needed by core tests through `extensions/openrouter/api.ts` or keep the coverage inside the OpenRouter plugin tests, add the user-facing docs/changelog entry if maintainers accept the config shape, then rebase and run focused OpenRouter plus extra-param validation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Guard auto-router injection to auto requests:** `extensions/openrouter/stream.ts:99-107`
  - body: `autoRouter.allowedModels` is documented and described as an `openrouter/auto` constraint, but this wrapper runs for any OpenRouter model as soon as the params are present. Because OpenClaw merges shared/default params before calling the provider wrapper, a shared `autoRouter` config can add an Auto Router plugin payload, and possibly suppress reasoning, on fixed-model calls that should not be routed through Auto Router. Add a `ctx.modelId === "auto"` guard before injecting the plugin or changing reasoning behavior.
  - confidence: 0.9
- **[P2] Import the helper through the plugin API seam:** `src/agents/pi-embedded-runner-extraparams.test.ts:4`
  - body: This core agent test now imports from the OpenRouter extension entrypoint. The scoped extension rules require core tests to use an exported `api.ts` seam instead of deep-importing bundled plugin internals, otherwise entrypoint/runtime refactors can break unrelated core tests. Export the helper from `extensions/openrouter/api.ts` or move this coverage into the OpenRouter plugin test suite.
  - confidence: 0.86
