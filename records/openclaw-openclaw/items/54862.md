## Summary

Keep open. Current main does not implement OpenRouter Auto Router allowlists, and the PR is a useful provider-owned feature, but the current branch still has concrete merge-blocking issues around non-Auto model behavior, plugin boundary coupling, and wildcard reasoning safety.

## What This Changes

The PR adds an `autoRouter.allowedModels` OpenRouter extra-param path that injects `plugins: [{ id: "auto-router", allowed_models: [...] }]` into OpenRouter request payloads, exports the helper, and adds OpenRouter/provider extra-param tests.

## Best Possible Solution

Revise the PR so the allowlist plugin is applied only to the OpenRouter Auto model, expose any helper needed by core tests through `extensions/openrouter/api.ts` or keep that coverage inside OpenRouter plugin tests, treat unresolved Grok/x-ai wildcard patterns conservatively, and add OpenRouter provider docs plus a credited changelog entry if maintainers accept this config shape.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Guard auto-router injection to Auto model requests:** `extensions/openrouter/stream.ts:99-107`
  - body: `autoRouter.allowedModels` is documented as an `openrouter/auto` constraint, but this block runs for every OpenRouter model when the params are present. Because shared/default params are merged before provider wrapping, a shared `autoRouter` config can add an Auto Router plugin payload and suppress reasoning on fixed-model calls. Gate the injection and reasoning change to the Auto model path.
  - confidence: 0.9
- **[P2] Import the helper through the plugin API seam:** `src/agents/pi-embedded-runner-extraparams.test.ts:4`
  - body: This core agent test imports `injectAutoRouterPlugin` from the OpenRouter extension entrypoint. The repo boundary requires core tests to use `api.ts` or a generic SDK seam for bundled plugin helpers, otherwise entrypoint refactors can break unrelated core tests. Export the helper from `extensions/openrouter/api.ts` or keep this coverage in OpenRouter plugin tests.
  - confidence: 0.86
- **[P2] Treat broad Grok wildcard allowlists as reasoning unsafe:** `extensions/openrouter/stream.ts:57-59`
  - body: OpenRouter allowed-model patterns support provider-agnostic wildcards, so patterns like `*/grok-*` can still route Auto Router to x-ai/Grok models. This helper only flags entries containing `x-ai/`, leaving proxy reasoning enabled and allowing the same x-ai reasoning payload failure the guard is meant to avoid. Treat unresolved broad Grok/x-ai patterns as unsafe or resolve the allowlist against a model registry.
  - confidence: 0.82
