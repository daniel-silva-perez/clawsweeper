## Summary

Keep this PR open. Current main still does not implement the Feishu outbound-level `NO_REPLY` suppression that this PR is trying to add, and the similar open PR #44828 is not a merged or clearly superior canonical replacement. The PR itself also still needs a follow-up pass for the `sendMedia` no-media fallback called out in review.

## Best Possible Solution

Keep this PR open and have the author or a maintainer complete the Feishu outbound guard across `sendText` and every `sendMedia` text path, including the no-`mediaUrl` fallback and any comment-thread caption behavior maintainers want. Add regression tests for exact and whitespace-padded `NO_REPLY`, substantive text passthrough, media-with-silent-caption delivery, and the no-media fallback. Then land this PR or explicitly consolidate it with #44828 once one canonical patch is complete.
