## Summary

Keep this PR open. Current main still does not implement the Feishu outbound-level NO_REPLY suppression this PR is trying to add, and the similar open PR #44828 is not merged or clearly canonical. The PR remains useful, but it needs a follow-up pass for the unguarded sendMedia no-media fallback called out in review.

## Best Possible Solution

Keep this PR open and ask the author or a maintainer to complete the Feishu outbound guard across sendText and every sendMedia text path, including the no-media fallback and any comment-thread caption behavior maintainers want. Add regression tests for the main Feishu outbound paths, then either land this PR or explicitly consolidate it with #44828 once one canonical patch is complete.
