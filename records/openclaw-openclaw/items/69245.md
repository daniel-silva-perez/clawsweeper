## Summary

Current main still does not implement this PR's intended behavior. The cache-TTL eligibility helper has plugin, Anthropic-family, KiloCode Anthropic, and Google prompt-cache paths, but no generic `modelApi === "openai-completions"` branch. The PR remains a focused source/test change, with no CI, dependency, script, or supply-chain-sensitive changes observed; the remaining question is maintainer policy on broad OpenAI-compatible eligibility versus provider-scoped hooks.

## Best Possible Solution

Keep this PR open for maintainer review. If the broad policy is accepted, land a rebased version that adds the generic `modelApi === "openai-completions"` eligibility branch, positive helper tests for direct OpenAI-compatible providers such as OpenAI, DeepSeek, and a local/custom provider, and a small clarification to the existing rejection test. Avoid changing provider plugin contracts, OpenRouter subset eligibility, prompt-cache-key compatibility behavior, or unrelated pruning defaults in this PR.
