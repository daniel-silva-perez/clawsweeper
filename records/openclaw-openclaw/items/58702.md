## Summary

Keep this issue open. Current main still has the reported Control UI/WebChat action-button overlap mismatch: the renderer computes and renders copy/expand actions for assistant bubbles, while the only CSS padding hook, `.chat-bubble.has-copy`, is never applied.

## Best Possible Solution

Fix the grouped message renderer so assistant bubbles with copy or expand actions reserve right-side space, likely by adding the existing `has-copy` class when `hasActions` is true, then add a focused Control UI test for assistant messages with action buttons. Carry over the useful related #66155 iPad/Safari reproduction detail when validating the touch path.
