## Summary

Keep this issue open. Current main still matches the reported Control UI/WebChat overlap: assistant bubbles render copy/expand actions, but the existing `.chat-bubble.has-copy` spacing hook remains unused, so the stale label is not enough to close it.

## What This Changes

Fix WebChat/Control UI assistant message bubbles so copy/expand action icons do not remain visibly overlaid on, truncate, or cover message text after the 2026.3.31 upgrade.

## Best Possible Solution

Land #69634 or an equivalent narrow Control UI patch against current main: add `has-copy` when `hasActions` is true, preserve the current tool-message bubble class, and add a focused regression test for assistant bubbles with copy/expand actions on hover and non-hover layouts.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes: render an assistant message in the Control UI with copy or expand actions, then inspect the bubble DOM/CSS path; current main still renders the action bar without the `has-copy` class that reserves space.

## Solution Assessment

Is this the best way to solve the issue? Yes: reusing the existing `.chat-bubble.has-copy` contract is the narrowest maintainable fix, as long as the current `chat-bubble--tool-shell` class and a regression test are preserved.
