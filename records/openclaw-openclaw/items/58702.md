## Summary

Keep this issue open. Current main still renders the chat bubble action buttons from the assistant-message path without applying the existing `has-copy` padding class, so the reported overlap/truncation remains a valid narrow Control UI bug rather than a stale or already-fixed report.

## What This Changes

The issue asks for Control UI/WebChat copy and expand action icons to stop covering message text after the 2026.3.31 upgrade.

## Best Possible Solution

Apply the existing CSS contract consistently: compute the action state before building `bubbleClasses`, add `has-copy` whenever copy/expand actions are rendered, and add a focused grouped-render test that would fail if the action buttons can render without the padding class.

## Reproduction Assessment

Yes. The report's UI behavior is reproducible by static source inspection: assistant markdown messages render action buttons, the CSS only reserves space for `.chat-bubble.has-copy`, and current main never adds that class.

## Solution Assessment

Yes. The narrowest maintainable fix is the commenter’s suggested class wiring plus a render test; this belongs in the Control UI chat renderer, not the Gateway or WebChat protocol.
