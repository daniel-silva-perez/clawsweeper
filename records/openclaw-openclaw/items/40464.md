## Summary

Keep this PR open. Current main has useful destructive-write and last-known-good recovery guardrails for the catastrophic wipe symptom, but it still lacks the PR’s central serialized, fresh-read ownerDisplaySecret persistence path, so the proposed work is not obsolete. The branch is stale against current config-write contracts and needs maintainer-guided rebase or narrowing before merge.

## What This Changes

The PR adds a module-level config write queue, changes ownerDisplaySecret auto-persistence to reread and write inside the queued critical section, and adds config write/runtime snapshot regression tests.

## Best Possible Solution

Implement the ownerDisplaySecret auto-persist as a current-main transaction: serialize the relevant write, read the fresh source snapshot inside that critical section with the existing write-snapshot/mutation machinery, merge only the generated `commands.ownerDisplaySecret`, and keep current config write safety, runtime refresh, plugin migration, and notification behavior intact.
