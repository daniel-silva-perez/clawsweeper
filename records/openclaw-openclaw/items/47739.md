## Summary

Keep this PR open. Current main still does not implement the generic gateway trailing-colon suppression, and the PR branch still misses the immediate item-start path that can flush the same buffered text after the tool-start path defers it. The linked Discord symptom has a shipped preview-streaming mitigation, so this needs maintainer product judgment before merge.

## What This Changes

The PR adds an optional `suppressIncompleteNarration` option to gateway buffered-chat flushing, enables it for tool-start flushes when buffered text ends with `:`, and adds gateway event tests for colon suppression plus period-ending flush behavior.

## Best Possible Solution

Maintainership should choose one direction: close the PR as superseded by the shipped channel-preview behavior, or revise it into an explicit generic gateway policy that defers incomplete narration across approved tool and item boundaries while preserving final/error flushing and adding focused regression coverage.
