## Summary

Keep open: current main still lacks the generic gateway colon-ending suppression, and this PR is not ready as-is because normal tool execution emits a follow-up item-start event that can flush the same buffered colon-ending text through an unsuppressed path. The linked Discord symptom has a shipped preview-streaming mitigation, so maintainers should decide whether the remaining generic gateway policy is still wanted before merge.

## What This Changes

The PR adds a scoped `suppressIncompleteNarration` option to gateway buffered-chat flushing, enables it for tool-start flushes when buffered text ends with `:`, and adds gateway event tests for colon suppression plus period-ending flush behavior.

## Best Possible Solution

Either close this PR as superseded after explicit maintainer agreement that the shipped preview-streaming behavior is sufficient, or update the gateway policy so incomplete narration is deferred across the actual tool-plus-item event boundary while preserving final/error flushing and adding focused regression coverage for that event sequence.
