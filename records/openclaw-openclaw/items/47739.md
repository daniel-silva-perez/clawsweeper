## Summary

Keep open. Current main still has the unguarded gateway flush behavior, so the PR is not obsolete, but the proposed patch should not merge as-is because it suppresses only the tool-start flush while the normal embedded tool path immediately emits an item-start event that can still flush the throttled colon-ending text.

## What This Changes

The PR adds an optional `suppressIncompleteNarration` option to the gateway buffered-chat flush helper, enables it for tool-start flushes when buffered assistant text ends with a colon, and adds gateway event tests for colon suppression and period-ending flushing.

## Best Possible Solution

Decide whether the shipped preview-streaming behavior is sufficient and close this PR as superseded, or accept an explicit generic gateway policy that defers incomplete narration across the full tool/item boundary while preserving terminal flushing and adding focused regression coverage for the adjacent event sequence.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes. A static event sequence with a throttled assistant update ending in `:`, followed by tool `start` and item `start`, shows why the current PR can still flush the colon tail at the unchanged item-start boundary.

## Solution Assessment

Is this the best way to solve the issue? No. The approach is narrow, but the implementation is incomplete because suppression is applied only to the tool-start call site and not carried through the adjacent item-start flush that current tool execution emits.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Carry colon suppression through item-start flushes:** `src/gateway/server-chat.ts:761-763`
  - body: The embedded tool path emits a tool `start` and then an item `start` for the same call. This option is only passed to the tool-start flush; because suppression returns without advancing `deltaLastBroadcastLen`, the unchanged item-start branch can immediately flush throttled colon-ending text as a chat delta. Please carry the suppression through that adjacent item boundary and cover the sequence in the gateway event test.
  - confidence: 0.88
