## Summary

Keep this PR open for maintainer judgment. Current main does not contain the gateway colon-ending suppression proposed here, but the linked Discord user bug was closed after a different shipped preview-streaming implementation; deciding whether to also add a generic gateway colon-deferral rule is a product/behavior call rather than conservative cleanup.

## What This Changes

The PR adds an optional `suppressIncompleteNarration` flag to the gateway buffered-chat flush helper, enables it for tool-start flushes when buffered text ends with `:`, and adds gateway event tests for colon suppression plus normal period-ending flushing.

## Best Possible Solution

Decide whether the shipped preview-streaming behavior is sufficient and close this PR as superseded, or explicitly accept a generic gateway policy that defers incomplete narration only at approved tool/item boundaries while preserving final and error flushing with focused regression coverage.
