## Summary

Keep open: current main still does not implement the generic gateway colon-ending suppression, and the PR branch remains incomplete because the normal tool path emits an immediate item-start event that can flush the same buffered colon-ending text through an unsuppressed call. The linked Discord symptom has a shipped preview-streaming mitigation, so this needs maintainer judgment before merge.

## What This Changes

The PR adds a scoped `suppressIncompleteNarration` option to gateway buffered-chat flushing, enables it for tool-start flushes when buffered text ends with `:`, and adds gateway event tests for colon suppression plus period-ending flush behavior.

## Best Possible Solution

Decide whether the shipped preview-streaming behavior is sufficient and close this PR as superseded, or explicitly accept a generic gateway policy that defers incomplete narration across approved tool/item boundaries while preserving final and error flushing with focused regression coverage.
