## Summary

Keep this PR open for maintainer review. Current main does not implement the proposed generic gateway colon suppression, and the PR still targets a real gateway buffering behavior, but the linked Discord symptom has a shipped channel-preview mitigation and the proposed patch misses the adjacent item-start flush path, so it is not ready to merge as-is.

## What This Changes

The PR adds an optional gateway buffered-chat flush flag, enables it for tool-start flushes when buffered assistant text ends with a colon, and adds two gateway event tests for colon suppression and period-ending regression coverage.

## Best Possible Solution

Either close this PR as superseded if the shipped preview-streaming behavior is considered sufficient, or revise it into an explicit generic gateway policy that defers incomplete narration across the full tool/item boundary while preserving final and error flushing and adding focused regression coverage for throttled colon-tail sequences.

## Reproduction Assessment

Yes. A high-confidence static reproduction path is to emit a throttled colon-ending assistant update, then the normal embedded tool sequence of `tool` start followed by `item` start; the PR suppresses only the first flush, leaving the second flush able to broadcast the orphan colon text.

## Solution Assessment

No. The proposed direction is narrow, but the implementation is incomplete because it scopes suppression to the tool-start flush and does not cover the adjacent item-start flush that current tool execution emits immediately afterward.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Carry colon suppression across item-start flushes:** `src/gateway/server-chat.ts:761-763`
  - body: The embedded tool path emits a tool `start` and then an item `start` for the same call. This branch suppresses the colon-ending buffer only for the tool-start flush, so when the colon tail was throttled and `deltaLastBroadcastLen` is behind, the following item-start flush can still broadcast `Let me check:` as the orphan message this PR is trying to prevent. Please carry the suppression across that adjacent item boundary and cover the sequence in the gateway event test.
  - confidence: 0.88
