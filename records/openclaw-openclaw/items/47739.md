## Summary

Keep this PR open. Current main still has no colon-ending narration guard in the gateway pre-tool flush helper, and tool-start events still call the flush directly. The PR remains relevant and narrow, but it needs maintainer review/rebase against current gateway chat paths, especially because main now also flushes buffered chat text before item-start events.

## What This Changes

The PR adds an optional `suppressIncompleteNarration` flag to `flushBufferedChatDeltaIfNeeded`, applies it to tool-start flushes, and adds gateway event tests for colon-ending suppression plus period-ending normal flushing.

## Best Possible Solution

Keep this PR open and review/rebase the narrow fix: add a scoped incomplete-narration suppression option for the intended pre-tool flush path, preserve final/error flushing, decide the item-start policy explicitly, and add focused gateway event tests for throttled colon-ending text, period-ending normal flushing, and any item-start behavior chosen.
