## Summary

Keep open. Current main still flushes any grown buffered assistant text before tool and item start events with no colon-ending suppression, and the existing main tests only cover the generic pre-tool flush behavior. The PR remains a narrow candidate for maintainer review; the provided diff is small and does not introduce an obvious security or supply-chain-sensitive change, but the item-start policy and stronger throttled-colon regression coverage should be settled before merge.

## What This Changes

The PR branch adds a scoped `suppressIncompleteNarration` option to the gateway buffered-chat flush helper, enables it for tool-start flushes, and adds gateway agent-event tests for colon-ending suppression plus period-ending normal flushing.

## Best Possible Solution

Review and update the branch so colon-ending narration is deferred only on the maintainer-approved pre-tool or item-start boundary, while final and error flushing still deliver complete text. Add a throttled colon-ending regression test plus a normal punctuation control, then run the targeted gateway test and the changed gate before merge.
