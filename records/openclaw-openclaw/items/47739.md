## Summary

Keep open. Current main does not implement the generic gateway colon-suppression behavior, so this contributor PR is not obsolete, but the proposed patch is not merge-ready: a normal tool-start followed by item-start sequence can still flush a throttled colon-ending buffer. The linked Discord symptom also has a shipped preview-streaming mitigation, so the remaining generic gateway policy needs maintainer judgment.

## What This Changes

The PR adds an optional buffered-chat flush flag, enables it for gateway tool-start flushes when buffered text ends with `:`, and adds gateway event tests for colon suppression plus normal period-ending flushing.

## Best Possible Solution

Either close this PR as superseded by the shipped #69611 preview-streaming behavior, or revise it into an explicit generic gateway policy that defers incomplete narration across the complete tool/item boundary while preserving final/error flushing and adding focused regression coverage for throttled colon-tail sequences.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Carry colon suppression across item starts:** `src/gateway/server-chat.ts:761-763`
  - body: The normal tool path emits a tool `start` and then an item `start` for the same call. This PR suppresses the colon-ending buffer only on the tool-start flush; if the colon arrived in a throttled assistant update, `deltaLastBroadcastLen` is still behind, so the following item-start flush can still broadcast `Let me check:` as the orphan message this PR is trying to prevent. Please carry the suppression across that adjacent item boundary and cover the sequence in the gateway event test.
  - confidence: 0.88
