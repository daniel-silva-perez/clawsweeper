## Summary

Keep this PR open. Current main fixes the Telegram/private-proxy regression path, but it does not fully implement the PRâ€™s broader requested behavior: explicit public proxies are still validated against the target URL `hostnameAllowlist` unless `allowPrivateProxy` is true. The Greptile discussion correctly describes a small, focused PR, but the remaining policy semantics need maintainer review rather than cleanup closure.

## Best Possible Solution

Review or adapt the PR rather than closing it automatically. A maintainer should either land the broader proxy-policy fix with tests for public and private explicit proxies, or document that `allowPrivateProxy` is the required trust bit for any proxy host outside the target allowlist and update tests/docs to make that narrower policy intentional.
