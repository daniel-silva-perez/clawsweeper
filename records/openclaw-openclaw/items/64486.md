## Summary

Current main still preserves unsupported reminder/follow-up commitment language and appends a transparency note. It does not implement the requested gate that forces the agent to either revise the reply or create a concrete scheduled action before delivery.

## Best Possible Solution

Keep this issue open. The best implementation is a focused pre-send gate in the auto-reply finalization path that reuses the existing commitment detector plus successfulCronAdds and session-cron coverage, allows one revision before delivery, and sends only after the agent either removes unsupported commitment language or creates a concrete cron/task/heartbeat-backed action. If maintainers prefer a plugin-first policy, first extend the documented reply/finalization hook API to expose the required per-turn scheduling evidence.
