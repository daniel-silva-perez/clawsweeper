## Summary

Keep open. Current main still implements an append-only transparency guard: unsupported reminder/follow-up language remains in the reply and the note is appended when no successful scheduling signal is present. The prior ClawSweeper comment reached the same conclusion, and rechecking the newer main SHA shows the requested pre-send forced-choice gate is still absent.

## What This Changes

The issue requests a pre-send gate that blocks unsupported future-commitment language unless the agent removes it or creates a concrete scheduled action.

## Best Possible Solution

Implement an opt-in pre-send finalization gate that reuses the existing detector, successfulCronAdds, and same-session cron coverage, with focused tests proving unsupported commitment text is revised or backed by a real schedule before delivery. If maintainers choose plugin-first, extend the documented finalization/message hook API to expose the required scheduling evidence before relying on an external plugin.
