## Summary

Keep open. Current main still does not implement the PR's intended followup dispatch barrier, but the proposed patch is stale against current reply code and introduces an abort/cancellation hole where a queued followup can start after a user stop request.

## What This Changes

The PR passes a dispatch-idle waiter through reply options, adds followup dispatch-idle helpers/tests, and makes queued followups wait before starting and after dispatcher-based followup delivery.

## Best Possible Solution

Implement a turn-scoped, abort-aware dispatch barrier on current main: it should wait only for the final reply delivery that caused the followup, keep parked followups cancellable until they either start or are discarded, track followup outbound sends until they drain, and cover dispatch ordering plus stop-during-wait regressions.

## Reproduction Assessment

Yes. The code-level reproduction is a busy session that queues a followup while the original final reply is still draining, then receives /stop during the PR's dispatch-idle wait; source inspection shows the followup is already handed to run() but has not created a reply operation yet.

## Solution Assessment

No. The proposed direction addresses the right ordering hazard, but this patch is not the best current solution because the pre-run wait is not abort-aware and the patch no longer applies cleanly to current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Cancel followups parked behind dispatch-idle waits:** `src/auto-reply/reply/followup-runner.ts:140`
  - body: The new pre-run wait runs before the followup creates a reply operation or starts the embedded agent. Because the queue drain has already handed this item to run(), /stop can clear queued items and abort active runs without cancelling this parked followup; once the dispatcher drains, it can still start and reply after the stop request. Make the idle wait abort-aware or register the parked followup with the stop path until it starts or is cancelled.
  - confidence: 0.9
