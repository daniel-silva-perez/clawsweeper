## Summary

Keep open. Current main has static, bundle-backed LSP support, but it still does not implement the requested zero-configuration project scanning, PATH discovery, opt-in installation/fix flow, user override/update semantics, or LSP health/dashboard surface. The remaining work is a product/API design item, not a conservative cleanup close.

## What This Changes

The issue requests automatic LSP server detection and setup from project files, package managers, installed server discovery, language/editor heuristics, optional install/fix actions, overrides, update notices, and diagnostics.

## Best Possible Solution

Keep this open for maintainer product/API design. A good end state would add safe project and PATH-based discovery, keep installs and fixes explicitly opt-in, support user overrides, expose LSP health through CLI/UI diagnostics, and define a plugin or bundle API if language-specific catalogs should live outside core.
