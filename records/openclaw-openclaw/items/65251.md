## Summary

Keep open. Current main has bundle-backed LSP support, but it is still explicit/static configuration from enabled bundles rather than the requested zero-configuration project scanning, server discovery, opt-in install/fix workflow, user overrides, update notices, or LSP health dashboard/API surface.

## What This Changes

The issue proposes automatic project-language detection, LSP server discovery and optional installation, smart configuration, overrides, and status/diagnostic surfaces so common language support works without manual LSP setup.

## Best Possible Solution

Keep the issue open for product/API design. A good end state would add safe project detection and PATH discovery, keep installs and fixes explicitly opt-in, support user overrides, and surface LSP health through CLI/UI diagnostics; if maintainers want language-specific logic outside core, first add and document a plugin or bundle API for dynamic workspace LSP configuration.
