## Summary

Current main still has the reported Discord mention reliability gap: outbound Discord sends rewrite cached handles, but there is no deterministic config-backed alias map and unknown handles remain plaintext. The issue is valid and narrow enough for a focused Discord plugin repair.

## What This Changes

Discord outbound messages should deterministically convert locally known `@AgentName` references into real Discord `<@userId>` mentions before sending.

## Best Possible Solution

Add a deterministic Discord mention-alias resolver in the Discord plugin, integrated ahead of the transient directory cache and scoped by account with a guild-aware shape where feasible; keep unknown/reserved/code mentions unchanged and document the supported config surface.

## Repair Work Prompt

Fix the Discord outbound mention reliability gap from https://github.com/openclaw/openclaw/issues/67587. Current behavior rewrites `@Name` only when the process-local Discord directory cache already has a handle-to-userId mapping; otherwise plaintext slips through. Implement a conservative deterministic alias source in the Discord plugin, preferably `channels.discord.mentionAliases` plus account-scoped `channels.discord.accounts.<id>.mentionAliases`, with a shape that can express guild-specific aliases if practical. Integrate it into `rewriteDiscordKnownMentions` before the transient directory cache. Preserve existing behavior for unknown handles, `@everyone`, `@here`, and code spans/fences. Keep the work extension-owned; do not add Discord-specific defaults or migrations in core startup. Update `DiscordAccountConfig`, the Discord zod schema/help/UI hints as appropriate, docs at `docs/channels/discord.md`, and focused tests for configured aliases, account isolation, optional guild scoping/collisions, normal sends, webhook sends, structured sends, reserved mentions, unknown names, and markdown code spans/fences. Changelog entry is expected because this changes user-facing Discord behavior; credit the issue reporter with `Thanks @McoreD`.
