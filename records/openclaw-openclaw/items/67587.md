## Summary

Current main still leaves the Discord outbound mention reliability gap open. Discord sends do run outbound text through the known-mention rewriter, but that rewriter only resolves handles from the process-local account-scoped directory cache; unknown handles are intentionally preserved as plaintext, and I did not find a config-backed alias map or normal inbound-message cache seeding path.

## Best Possible Solution

Keep this issue open and implement the fix in the Discord plugin. A conservative path is an explicit account-scoped, preferably guild-aware mention alias map such as channels.discord.mentionAliases or channels.discord.accounts.<id>.mentionAliases, integrated into rewriteDiscordKnownMentions before the transient directory cache, with optional trusted inbound author/member cache seeding. Add tests for configured aliases, account isolation, unknown-name preservation, reserved mentions, code spans/fences, normal sends, webhook sends, and structured sends.

## Risks / Open Questions

- A fix must avoid accidental real Discord notifications: preserve @everyone, @here, unknown handles, and text inside code spans/fences unless an alias is explicitly trusted.
- Alias resolution should be account-scoped and likely guild-aware because Discord usernames, global display names, and nicknames can collide.
- Adding a deterministic alias map should update the Discord config type, zod schema/generated metadata, docs, and regression tests together.
