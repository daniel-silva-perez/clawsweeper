## Summary

Current main still matches the reported regression: `requests-in-flight` heartbeat skips are retried through the shared 1 second wake retry, and there is no heartbeat-specific busy-session retry/backoff config in source, schema, tests, or docs. The focused fix PR #40629 is closed unmerged, while #13744 and #14051 are broader follow-ups rather than a shipped fix for this issue.

## Best Possible Solution

Keep this issue open and fix the periodic heartbeat busy-session retry path directly. A narrow path is to revive the useful parts of #40629: add a heartbeat-specific `requests-in-flight` retry delay or backoff separate from generic wake error retry, choose a conservative default, expose it under `agents.defaults.heartbeat` and per-agent heartbeat config, and update schema/types, generated config help/metadata, public heartbeat docs, and scheduler/wake regression tests. Leave #13744 for broader session-lock behavior and #14051 for activity-based heartbeat semantics.
