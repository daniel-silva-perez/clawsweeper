## Summary

Current main still has the reported heartbeat contention path: `requests-in-flight` skips are retried through the shared 1 second wake retry, tests still assert that timing, and there is still no heartbeat-specific retry/backoff config in schema, types, docs, or generated help. A closed unmerged PR already attempted the requested 60s/configurable delay, but that work did not land.

## Best Possible Solution

Keep this issue open and implement a narrow heartbeat busy-session retry policy. The fix should separate `requests-in-flight` retry delay/backoff from generic wake error retry, choose a conservative maintainer-approved default, expose it under `agents.defaults.heartbeat` and per-agent heartbeat config, update schema/types/generated config help and https://docs.openclaw.ai/gateway/heartbeat, and add wake/scheduler regression tests. Leave #13744 for broader session-lock scope and #14051 for activity-based heartbeat semantics.
