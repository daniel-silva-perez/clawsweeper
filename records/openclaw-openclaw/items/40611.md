## Summary

Keep open. Current main still routes `requests-in-flight` heartbeat skips through the shared 1 second wake retry, and there is still no heartbeat-specific busy-session retry/backoff config in schema, types, tests, or docs. The focused fix PR #40629 is closed unmerged, while #13744 and #14051 are broader open follow-ups rather than a shipped fix for this regression.

## Best Possible Solution

Keep this issue open and fix the periodic heartbeat busy-session retry path directly. The narrow path is to revive the useful parts of #40629: add a heartbeat-specific `requests-in-flight` retry delay separate from generic wake error retry, choose a default and config surface under `agents.defaults.heartbeat` and per-agent heartbeat config, then update schema/types, generated config metadata/help, public heartbeat docs, and scheduler/wake regression tests. Leave #13744 for broader session-lock scope/configuration and #14051 for activity-based heartbeat semantics.
