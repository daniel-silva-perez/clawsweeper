## Summary

Keep this issue open. Current main still retries `requests-in-flight` heartbeat skips through the shared 1 second wake retry path, and the heartbeat config/schema/docs still do not expose a heartbeat-specific busy-session retry/backoff setting. The focused fixing PR #40629 remains closed unmerged, while #13744 and #14051 are broader open designs rather than a shipped fix for this regression.

## Best Possible Solution

Keep this issue open and fix the periodic heartbeat busy-session retry path. The narrow maintainer path is to revive the useful parts of #40629: add a heartbeat-specific `requests-in-flight` retry delay separate from generic wake error retry, decide the default and config surface under `agents.defaults.heartbeat` and per-agent heartbeat config, then update schema/types, generated config metadata/help, public heartbeat docs, and scheduler/wake regression tests. Leave #13744 for broader session-lock scope/configuration and #14051 for activity-based heartbeat semantics.
