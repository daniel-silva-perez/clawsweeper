## Summary

Keep this PR open, but it should not be merged as-is. Current main has not replaced sharp with @napi-rs/image: image attachment operations still delegate to the media-understanding-core plugin, that plugin still imports and depends on sharp, install docs/scripts still carry the SHARP_IGNORE_GLOBAL_LIBVIPS workaround, and the PR discussion already identified production regressions in the proposed migration.

## Best Possible Solution

Keep this PR open for maintainer or author follow-up, but require a reworked migration before merge: target extensions/media-understanding-core as the sharp owner, preserve existing media-runtime contracts including source format/alpha and withoutEnlargement behavior, keep or explicitly migrate legacy OPENCLAW_IMAGE_BACKEND=sharp semantics, update install docs/scripts and dependency ownership together, and provide focused tests plus a security/supply-chain review for the new native dependency.
