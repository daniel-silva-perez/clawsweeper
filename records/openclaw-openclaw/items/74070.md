Current main still matches the reported ACP background-task behavior: progress is capped at 240 characters, ordinary non-blocked progress falls through to no terminalSummary, and successful task delivery only prints terminalSummary. Keep this issue open because #74103 is an open implementation PR linked with GitHub closing syntax; the right next step is to review, improve, and land or close that PR.

## Best Possible Solution

Keep this issue open while #74103 is reviewed. The best fix is a narrow ACP control-plane change that derives a sanitized terminal summary from useful successful ACP output, preserves blocked/permission detection and no-leak behavior, raises the progress accumulation cap to a bounded useful length, and adds focused regression tests before merge.
