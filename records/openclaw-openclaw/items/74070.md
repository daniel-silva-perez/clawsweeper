## Summary

Keep open: current `main` and the latest release still contain the reported ACP background-task summary behavior, and open PR #74103 already references this issue with closing syntax. The issue should remain open until that PR, or an equivalent sanitized and tested fix, lands.

## What This Changes

The issue requests that successful ACP background-task completions derive a useful `terminalSummary` from progress output and raise the `progressSummary` cap from 240 characters to about 1200.

## Best Possible Solution

Land PR #74103 after addressing sanitization and coverage, or replace it with an equivalent narrow ACP manager patch that copies cleaned successful progress into `terminalSummary`, preserves blocked-outcome handling, and adds regression coverage for useful summaries plus suppressed internal/control text.

## Reproduction Assessment

Yes. The static current-main path is high-confidence: non-thought ACP output populates `taskProgressSummary`, ordinary non-blocked success returns `{}`, terminal finalization stores a null summary, and delivery formats only `terminalSummary`.

## Solution Assessment

Yes on the boundary, with a caveat. Falling back from cleaned progress output and raising the bounded cap is the right maintainable direction, but the final patch should strip control/internal text and include focused tests before closing this issue.
