## Summary

Keep open: current main still has the reported ACP background-task behavior, and an open unmerged PR already references this issue with closing syntax, so maintainers should resolve that PR before closing the issue.

## What This Changes

The issue requests ACP background-task completions to populate useful successful terminal summaries from sanitized progress output and to raise the progress-summary cap beyond 240 characters.

## Best Possible Solution

Land a narrow ACP background-task fix via PR #74103 or an equivalent patch that derives a safe successful terminal summary, preserves blocked-outcome handling, raises the bounded progress cap, and adds regression coverage for useful summaries plus suppressed internal chatter.

## Reproduction Assessment

Yes. The static current-main path is high-confidence: non-thought output builds `progressSummary`, non-blocked successful text returns `{}`, the terminal summary is stored as null, and terminal delivery then emits the bare completion label.

## Solution Assessment

Unclear as proposed. Falling back from progress output and raising the cap is the right boundary, but the maintainable fix should include sanitization and tests so internal progress text does not leak into channel notifications.
