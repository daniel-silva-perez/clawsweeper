## Summary

Current main still has the reported ACP background-task behavior, and open PR #74103 references this issue with closing syntax, so the issue should remain open until that PR is reviewed and merged or closed.

## What This Changes

The issue requests that successful ACP background-task completions use sanitized progressSummary as terminalSummary and that the progress summary cap increase from 240 to about 1200 characters.

## Best Possible Solution

Review and land PR #74103 or an equivalent narrow ACP manager patch that preserves blocked-outcome detection, derives a bounded sanitized terminal summary for useful successful output, raises the progress cap intentionally, and adds regression tests for both useful summaries and suppressed internal chatter.

## Reproduction Assessment

Yes. Current-main source gives a high-confidence reproduction path: non-thought ACP output accumulates into progressSummary, successful completion calls `resolveBackgroundTaskTerminalResult`, non-blocked text returns `{}`, and the delivered terminal message ignores progressSummary when terminalSummary is null.

## Solution Assessment

Yes, the best direction is a narrow fix in the ACP background-task completion path plus targeted tests. PR #74103 is the active implementation candidate, but it should be reviewed for sanitization and coverage before this issue is closed.
