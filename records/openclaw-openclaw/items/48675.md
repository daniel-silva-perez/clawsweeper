## Summary

Keep this PR open. Current main still does not pre-create the daily memory file before gateway or CLI agent runs, and the submitted PR remains a plausible but stale candidate rather than work already implemented on main.

## Best Possible Solution

Keep the PR open for maintainer follow-up, but rebase or replace it against current main. A good fix should derive the target from `resolveMemoryFlushPlan`, skip work when memory flush is disabled or the workspace is not writable, wire both `getReplyFromConfig` and `src/agents/agent-command.ts`, preserve existing daily memory content, treat concurrent creation as success, and add focused tests for fresh, existing, disabled, read-only/sandboxed, and concurrent cases.
