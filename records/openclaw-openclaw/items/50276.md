## Summary

Keep open. The requested Slack `[[reply_to_current]]` behavior is not implemented on current main for outbound message-action sends, and the open PR still has source-proven compile/test defects plus an unresolved Slack `replyToMode="off"` policy mismatch before it can merge.

## What This Changes

The PR adds Slack-specific `[[reply_to_current]]` handling in outbound message-action sends and adds same-channel and cross-channel Slack threading tests.

## Best Possible Solution

Rework this PR through the existing `message-action-threading` seam, remove the redeclared helper, add runnable tests for same-channel, cross-channel, explicit `threadId`/`replyTo` precedence, and first-mode accounting, then update Slack docs and plugin metadata only if maintainers intentionally change off-mode semantics.

## Reproduction Assessment

Yes. Applying the PR diff would reproduce the compile issue because it redeclares an imported binding, and the new tests would fail because they call helpers that are not defined or imported in the file.

## Solution Assessment

No. The direction is plausible, but the current patch is not the narrowest maintainable fix because it duplicates an existing helper, adds unrunnable tests, and silently changes documented Slack off-mode behavior.
