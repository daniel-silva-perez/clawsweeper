## Summary

Keep open. Current main still lacks the requested Slack `[[reply_to_current]]` outbound mapping, but this PR head is not merge-ready because it redeclares an imported resolver and its new tests call helpers that are not defined in the test file. The behavior also changes the documented Slack `replyToMode="off"` policy, so maintainer review is needed before merge.

## What This Changes

The PR changes outbound Slack message-action sends and threading tests so `[[reply_to_current]]` can resolve to the active same-channel Slack `thread_ts` while preserving cross-channel marker behavior.

## Best Possible Solution

Land a narrow Slack threading fix through the existing outbound threading seam after deciding the Slack off-mode policy, with coverage for same-channel resolution, cross-channel sends, explicit `threadId`/`replyTo` precedence, and first-mode reply accounting; update Slack docs if the policy changes.
