## Summary

Keep open. Current main still does not map outbound Slack `[[reply_to_current]]` to a thread target, so the PR is not obsolete, but the proposed patch has compile/test blockers and conflicts with the current Slack off-mode threading policy.

## What This Changes

This PR adds Slack-only outbound handling for `[[reply_to_current]]` and new same-channel/cross-channel threading tests.

## Best Possible Solution

Rework the implementation through the existing `message-action-threading` seam, avoid redeclaring imported helpers, add runnable same-channel/cross-channel/explicit-precedence/first-mode tests, and only change Slack off-mode behavior with a coordinated docs and plugin metadata update.

## Reproduction Assessment

Yes. Applying the PR diff has a static ESM/TypeScript failure from redeclaring an imported binding, and the added tests reference helpers that are not defined or imported in the test file.

## Solution Assessment

No. The intended direction is plausible, but this patch is not the best fix because it duplicates an existing core threading helper, adds unrunnable tests, and silently changes documented Slack off-mode semantics.
