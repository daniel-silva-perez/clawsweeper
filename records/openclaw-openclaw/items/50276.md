## Summary

Keep open. The PR targets a real Slack outbound-threading gap on current main, but the proposed patch is not mergeable because it redeclares an imported helper, adds tests that call undefined helpers, and changes Slack's documented off-mode threading policy without a coordinated policy or docs update.

## What This Changes

The branch adds Slack-specific `[[reply_to_current]]` handling for outbound message-action sends and adds same-channel and cross-channel threading tests.

## Best Possible Solution

Rework the change through the existing outbound threading helper, remove the duplicate local resolver, add runnable coverage for same-channel, cross-channel, explicit `threadId`/`replyTo` precedence, and either preserve Slack off-mode behavior or intentionally update the Slack policy, docs, and tests together.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Remove the duplicate thread resolver declaration:** `src/infra/outbound/message-action-runner.ts:71`
  - body: `message-action-runner.ts` already imports `resolveAndApplyOutboundThreadId` from `./message-action-threading.js`. Declaring another function with the same name in this module is invalid ESM/TypeScript and prevents the runner from compiling or loading.
  - confidence: 0.97
- **[P2] Define the helpers used by the new threading tests:** `src/infra/outbound/message-action-runner.threading.test.ts:169-171`
  - body: The added tests call `mockHandledSendAction()` and `runThreadingAction(...)`, but this file does not define or import those helpers. As written, the new coverage fails before it can validate the Slack behavior.
  - confidence: 0.94
- **[P2] Align the off-mode override with Slack policy:** `src/infra/outbound/message-action-runner.ts:116-120`
  - body: Forcing `replyToMode: "all"` makes `[[reply_to_current]]` thread even when Slack `replyToMode` is `off`, while current Slack docs and plugin metadata say explicit reply tags are disabled in off mode. The patch needs to honor that policy or include a coordinated policy/docs change.
  - confidence: 0.87
