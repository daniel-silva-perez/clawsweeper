## Summary

Keep this PR open. Current main still does not route `parsed.replyToCurrent` into Slack outbound thread targeting, so the PR is not obsolete, but the branch is not merge-ready because it redeclares an imported resolver and adds tests that call undefined helpers. The remaining Slack `replyToMode="off"` policy change also needs maintainer review.

## What This Changes

The PR branch changes outbound message-action Slack sends and threading tests to map `[[reply_to_current]]` to the active same-channel Slack thread target while preserving cross-channel marker behavior.

## Best Possible Solution

Keep the PR open, settle the Slack explicit-tag/off-mode policy, then land a narrow implementation through the existing outbound threading seam with coverage for same-channel, cross-channel, explicit `threadId`/`replyTo` precedence, and single-use reply mode accounting; update Slack docs if the policy changes.
