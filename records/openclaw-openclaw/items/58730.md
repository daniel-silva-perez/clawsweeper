## Summary

Keep open. The issue has an open upstream draft PR #62937 with closing syntax for the host exec sandbox proposal, so the issue should stay open until that implementation candidate is reviewed, merged, or closed. Current main also only partially covers the umbrella request: sandbox backends, exec approvals, tool policies, compaction, transcripts, and hooks exist, but the proposed macOS/Linux host exec sandbox wrapper and the exact session resume CLI are not implemented on main.

## Best Possible Solution

Keep this issue open while maintainers review PR #62937 or replace it with a preferred sandbox/exec-permission design. If the PR is declined, split or supersede the remaining work into narrower issues for host exec sandboxing, permission semantics, and general session resume, while acknowledging that compaction, transcript persistence, hooks, tool policy, and exec approvals already exist on main.
