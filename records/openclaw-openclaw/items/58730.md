## Summary

Keep open. Current main has substantial adjacent safety and lifecycle mechanisms, but this umbrella request is only partially covered: exec sandboxing is optional, host exec still defaults to no approval unless tightened, the exec schema has no per-call sandbox profile, and the linked closing PR is still an open draft that only addresses macOS sandbox-exec.

## What This Changes

The issue requests OS-level exec sandbox isolation, granular tool permissions, automatic context compaction, session persistence/resume, and lifecycle hooks for OpenClaw agent runs.

## Best Possible Solution

Keep this open as a security/product design item or split it into narrower tracked issues, with any implementation aligned to the existing Docker/SSH/OpenShell sandbox backend and exec approval architecture instead of treating a deprecated macOS-only wrapper as the whole fix.

## Reproduction Assessment

Not applicable as a feature/security design request. Static verification on current main is high-confidence: the existing docs and schema show optional sandbox/approval controls but no per-call bwrap or sandbox-exec profile support.

## Solution Assessment

No, the currently linked PR is not the complete solution for this issue. A macOS-only sandbox-exec wrapper may be useful source work, but the maintainable fix needs a cross-platform policy decision and should account for current sandbox backends, exec approvals, and plugin hook approval surfaces.
