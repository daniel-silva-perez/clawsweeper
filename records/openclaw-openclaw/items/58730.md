## Summary

Keep open. Current main has related safety infrastructure, including optional sandbox backends, host exec approval modes, session transcripts/compaction, and hook systems, but the central host-process OS sandbox wrapper proposal is not implemented on main. More importantly, an open draft PR by the same author explicitly uses closing syntax for this issue, so the issue should remain open while maintainers review, land, or close that PR.

## Best Possible Solution

Keep this issue open while maintainers review the open same-author closing PR #62937. If that PR lands, close this issue as implemented by the merged sandbox change and split any still-desired permission/compaction/session/hooks work into narrower follow-ups; if the PR is rejected, keep or retitle this issue around the desired exec sandbox and permission-model direction.
