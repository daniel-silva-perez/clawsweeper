## Summary

Keep open. This is a broad security and UX feature request, and the upstream draft PR #62937 explicitly claims to close it, so the issue should not be closed before that implementation is reviewed, merged, replaced, or rejected. Current main already has adjacent sandbox backends, exec approvals, auto-compaction, session listing, persisted transcripts, and hook surfaces, but the requested macOS `sandbox-exec` wrapper is not implemented on main and the remaining security-design question needs maintainer/security review.

## What This Changes

The issue requests exec sandbox isolation, a richer tool permission model, automatic context compaction, session resume commands, and a broader hook system based on lessons from a Claude Code source-leak analysis.

## Best Possible Solution

Keep this issue open as the umbrella until maintainers decide the intended scope: either land or replace the linked sandbox-exec work for the exec-isolation slice, and split any remaining permission, resume, compaction, or hook gaps into narrower issues that match current OpenClaw sandbox, approvals, session, and plugin-hook architecture.
