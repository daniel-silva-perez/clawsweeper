## Summary

Keep this PR open. Current main still does not expose the active session UUID in the system prompt Runtime line, and the refreshed PR branch remains a narrow implementation candidate rather than obsolete or superseded. The open #66312 overlaps conceptually, but it is also unmerged and the latest maintainer comment says this contributor branch was repaired so the original path can stay canonical.

## Best Possible Solution

Keep this PR open for maintainer review/landing, or apply the same narrow change directly on main: add sessionId to runtime prompt types, pass the active session entry UUID into buildSystemPromptParams, render session=<uuid> in buildRuntimeLine, and keep the new regression assertion. If maintainers prefer the broader sessionKey design in #66312, close one PR only after the chosen canonical implementation lands.
