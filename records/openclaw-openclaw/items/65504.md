## Summary

Keep open. Current main has partial mitigations for auth-profile timeout poisoning and deferred context-engine maintenance, but the live gateway/agent.wait contract still lacks the requested phase-aware timeout attribution. The linked OpenClaw PR #65511 was closed unmerged, and related #54488/external lossless-claw work covers only part of the umbrella.

## Best Possible Solution

Keep this open as the OpenClaw tracking item for an additive timeout-attribution contract. The best maintainer path is to split useful pieces from the closed PR #65511 into focused PRs that add phase-aware timing/timeout metadata to embedded-run telemetry, lifecycle events, gateway dedupe snapshots, agent.wait responses, protocol schema/docs, caller types, and regression tests, while leaving lossless-claw scheduling work to #54488 or the external lossless-claw PR.

## Risks / Open Questions

- Closing now would hide the remaining core API/product gap: live gateway callers still cannot tell queue delay, preflight stalls, provider transport timeout, post-turn cleanup, or gateway draining apart.
- Partial fixes for auth-profile timeout health and deferred maintenance reduce symptoms, so future work should preserve those shipped guards instead of reviving the closed all-in-one PR unchanged.
- Related #54488 and the external lossless-claw PR cover session-lane/LCM scheduling, but they do not replace the missing OpenClaw gateway timeout-attribution contract.
