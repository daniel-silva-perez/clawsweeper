## Summary

Keep this PR open. Current main still resolves and reports one aggregate session default model without iterating configured agents, even though per-agent model overrides are supported and row-level session model resolution is agent-aware. The PR remains a plausible narrow fix, but it likely needs rebase/adaptation to current overview formatting internals.

## Best Possible Solution

Keep this PR open or replace it with a narrow maintainer patch. Compute `sessions.defaults` from each configured agent's effective status-safe default model, keep a concrete model/context only when all effective defaults match, expose an optional variation marker through `StatusSummary` and `GatewayStatusSummary`, render `varies by agent` in CLI and TUI status surfaces, and add divergent/uniform per-agent regression tests while preserving the read-only status path.
