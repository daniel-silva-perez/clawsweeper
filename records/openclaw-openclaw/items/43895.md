## Summary

Keep this PR open. The reported status bug is still present on current main, and the PR targets a valid narrow display fix, but the branch is stale against the current status overview path and would not update the human `openclaw status` Sessions row after rebase.

## What This Changes

The PR derives status session defaults from each agent’s effective model, adds an optional `variesByAgent` marker to status and TUI summary types, renders `varies by agent`, and adds divergent/uniform regression tests.

## Best Possible Solution

Implement the fix against the current status summary/runtime shape: compute `sessions.defaults` across the effective configured agents, expose a backward-compatible variation marker through CLI JSON and TUI types, render that marker in the current overview helper and TUI summary, and cover both divergent and uniform defaults in summary and overview tests.
