## Summary

Keep this PR open. The reported status bug is still present on current main, but the proposed branch is stale against the current renderer split: it updates the old inline `status.command.ts` Sessions row while current human status output is built through the overview helper.

## What This Changes

The PR derives status session defaults from each agent’s effective model, adds an optional `variesByAgent` marker to status and TUI summary types, renders `varies by agent`, and adds divergent/uniform regression coverage.

## Best Possible Solution

Update the PR against the current status summary/runtime shape: compute `sessions.defaults` across effective configured agents, expose a backward-compatible variation marker through CLI JSON and TUI types, render it in `status-overview-values.ts` and the TUI summary, and cover divergent plus uniform defaults in summary and overview tests.
