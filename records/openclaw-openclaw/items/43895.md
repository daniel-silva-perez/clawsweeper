## Summary

Keep this PR open. The reported status bug is still present on current main, but the PR branch is stale against the current status rendering path and would miss the human `openclaw status` overview after rebase.

## What This Changes

The PR derives status session-default model summaries from each configured agent's effective default, adds optional `variesByAgent` fields through CLI/TUI summary types, renders `varies by agent`, and adds status-summary regression tests.

## Best Possible Solution

Update the branch against current main by aggregating `sessions.defaults` across effective configured agents while preserving the lazy `status.summary.runtime` boundary, expose a backward-compatible variation marker through CLI JSON and TUI types, render it in `status-overview-values.ts` and the TUI summary, and cover divergent and uniform defaults in summary plus overview tests.

## Reproduction Assessment

Yes. A config with multiple `agents.list` entries whose effective model primaries resolve to different models still flows through the current single no-`agentId` aggregate resolver, while the per-session row path already has an agent-aware resolver.

## Solution Assessment

No, not as currently proposed. The per-agent default aggregation is the right direction, but the branch must be rebased to update the current overview helper and preserve the current lazy runtime boundary before it is mergeable.
