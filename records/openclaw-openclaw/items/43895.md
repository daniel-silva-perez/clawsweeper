## Summary

Keep this PR open. The bug it targets is still present on current main: status computes one aggregate session default without iterating configured agents, even though agent-specific model defaults are supported and row-level status resolution can be agent-aware. The PR remains useful, but it needs rebase/adaptation because the CLI overview rendering moved out of `status.command.ts`.

## What This Changes

The PR derives status session defaults from each agent’s effective model, adds an optional `variesByAgent` marker to status/TUI types, renders `varies by agent`, and adds divergent/uniform regression tests.

## Best Possible Solution

Keep the PR open or replace it with a narrow maintainer patch that computes `sessions.defaults` from each configured agent’s effective status-safe default model, returns concrete model/context values only when all agents agree, carries an optional variation marker through CLI JSON and TUI types, renders `varies by agent` in both CLI overview and TUI status, and adds divergent/uniform regression coverage.
