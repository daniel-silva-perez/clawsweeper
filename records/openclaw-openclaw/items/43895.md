## Summary

Keep this PR open. The reported status display bug still exists on current main, but the branch is stale against the current overview helper and lazy status-summary runtime shape, so it needs an update before it is mergeable.

## What This Changes

The PR derives status session-default display from each agent’s effective model, adds an optional `variesByAgent` marker through CLI/TUI summary types, renders `varies by agent`, and adds summary regression tests.

## Best Possible Solution

Update the PR against current main by aggregating `sessions.defaults` across effective configured agents while preserving the lazy `status.summary.runtime` boundary, exposing a backward-compatible variation marker through CLI JSON and TUI types, rendering it in `status-overview-values.ts` and the TUI summary, and covering both divergent and uniform defaults in summary plus overview tests.

## Reproduction Assessment

Yes. A config with multiple `agents.list` entries whose model primaries resolve to different providers/models will still flow through the current single no-`agentId` aggregate resolver, while the per-agent session row path already has an agent-aware resolver.

## Solution Assessment

No, not as currently proposed. The intended per-agent default aggregation is the right direction, but the branch must be rebased to update the current overview helper and current lazy runtime boundary before it is mergeable.
