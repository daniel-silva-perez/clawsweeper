## Summary

Keep this PR open. The reported status bug is still valid on current main because the aggregate status summary resolves one default model without agent context even though per-agent model defaults are documented and the status runtime can resolve them. The PR is useful source work, but it is stale against the current status overview refactor: the main human `openclaw status` Sessions row now renders through `status-overview-values.ts`, while the patch changes the older inline row in `status.command.ts`.

## What This Changes

The PR derives status session defaults from each agent’s effective model, adds an optional `variesByAgent` marker to status/TUI types, renders `varies by agent`, and adds divergent/uniform regression tests.

## Best Possible Solution

Implement the fix in the current status summary/runtime shape: compute `sessions.defaults` from each configured agent’s effective status-safe default model, expose a backward-compatible variation marker through CLI JSON and TUI types, render that marker through `buildStatusSessionsOverviewValue()` and the TUI summary, and add divergent/uniform regression coverage for both summary construction and overview formatting.
