## Summary

Keep this PR open. Current main still computes one aggregate status default model without iterating configured agents, while the resolver and public docs support per-agent model overrides. The PR remains a focused fix candidate, though it likely needs adaptation to the current `status-overview-values.ts` rendering path.

## Best Possible Solution

Keep this PR open or replace it with a narrow maintainer patch. Compute `sessions.defaults` from each configured agent's effective status-safe default model, return a concrete model/context only when all effective defaults match, expose an optional variation marker through `StatusSummary` and `GatewayStatusSummary`, render `varies by agent` in CLI and TUI status surfaces, and add divergent/uniform per-agent regression tests while preserving the read-only status path.
