## Summary

Keep this PR open. Current main does not contain the proposed reusable orphan-transcript pruning utility; existing cleanup paths only handle missing store entries, disk-budget pressure cleanup, or doctor’s interactive archive warning. The PR is an active implementation candidate with a security-sensitive unlink path, so it needs normal maintainer review rather than cleanup closure.

## What This Changes

This PR adds an opt-in `pruneOrphanedTranscripts` session-maintenance helper and unit tests for warning about or unlinking old unreferenced JSONL transcripts while preserving caller-supplied live paths, checkpoint snapshots, and unrelated JSONL files.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to decide whether the opt-in utility is acceptable as a staged building block, or whether same-branch CLI wiring should build the full preserved-path union across neighboring stores, topic transcripts, explicit session files, and compaction checkpoints before merge.
