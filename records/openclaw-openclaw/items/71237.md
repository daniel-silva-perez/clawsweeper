## Summary

Keep this PR open. Current main still lacks the proposed orphan-transcript pruning primitive, and the adjacent cleanup paths only cover stale/capped store entries, missing transcript entries, disk-pressure cleanup, or doctor archival rather than the opt-in direct JSONL orphan cleanup this PR adds. The PR appears relevant but should get maintainer review because it introduces a direct file-unlink maintenance helper and deliberately defers CLI wiring.

## What This Changes

This PR adds an opt-in `pruneOrphanedTranscripts` session-maintenance utility plus unit tests for warning about or unlinking old unreferenced JSONL transcript files while preserving caller-supplied live paths, checkpoint snapshots, and unrelated JSONL files.

## Best Possible Solution

Keep this PR open and have a maintainer decide whether to merge the opt-in utility as a staged building block or require the first CLI wiring in the same branch. The final accepted path should preserve all live transcript forms before deleting orphan files and should validate the targeted pruning tests plus the changed gate before merge.
