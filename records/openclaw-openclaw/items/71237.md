## Summary

Keep this PR open. Current main has adjacent session cleanup, pressure-gated disk-budget eviction, and doctor orphan archival, but it still does not implement the PR’s reusable `pruneOrphanedTranscripts` primitive or an ordinary on-demand path that scans already-unindexed JSONL transcripts and directly reclaims their bytes.

## What This Changes

This PR adds an opt-in `pruneOrphanedTranscripts` session-maintenance helper plus unit tests for old orphan `.jsonl` transcript cleanup, preserved paths, checkpoint snapshots, nested traversal, symlink containment, and unrelated JSONL files.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to decide whether the tested utility is acceptable as a staged building block, or whether the same branch should also wire `openclaw sessions cleanup` with a union of all live transcript paths across neighboring stores, topic transcripts, and compaction checkpoint metadata.
