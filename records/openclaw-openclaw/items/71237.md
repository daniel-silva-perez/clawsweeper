## Summary

Keep this PR open. Current main still lacks the proposed standalone orphan-transcript pruning helper, and the PR addresses a real cleanup gap not covered by existing missing-entry cleanup or pressure-only disk-budget eviction. The latest patch is mostly safe, but one small resilience issue remains in the recursive traversal before merge.

## What This Changes

The PR adds an opt-in `pruneOrphanedTranscripts` session-maintenance helper plus unit tests for warn/enforce cleanup, preserved paths, nested files, checkpoint skips, and symlink containment.

## Best Possible Solution

If maintainers accept a staged primitive, land the helper after making traversal best-effort per subtree; otherwise fold the helper into `openclaw sessions cleanup` in the same branch so operators get a dry-run/enforce path with multi-store preserved-path unioning from the start.

## Reproduction Assessment

Yes. The proposed behavior is covered by PR tests that create old valid session-header JSONL files outside `preservedPaths`, and the remaining traversal issue is reproducible when a nested unreadable directory makes recursive `readdir` reject before otherwise-readable orphans are considered.

## Solution Assessment

Mostly yes. An opt-in, age-gated, header-gated, path-preserved helper is a narrow maintainable building block, but a per-directory resilient walker is safer than relying on all-or-nothing recursive `readdir`; CLI wiring can remain a maintainer product call.

## Review Findings

Overall correctness: patch is correct

Overall confidence: 0.84

Full review comments:

- **[P3] Handle unreadable subtrees without aborting the sweep:** `src/config/sessions/store-maintenance.ts:473-478`

Because this catch returns the empty result, one unreadable subdirectory under `sessionsDir` prevents pruning of otherwise-readable orphan transcripts. Walk directories per subtree and skip failed descendants instead.
