## Summary

Keep this PR open. Current main still routes bare configured model-list entries and bare fallback entries through DEFAULT_PROVIDER in src/commands/models/list.configured.ts, so the PR’s central bug fix is not implemented on main. The shared model-selection layer already has the provider-inference helper and uses it in other paths, but the models-list configured-entry path has not been updated. The PR also has a relevant unresolved review note about alias handling that should be addressed before landing.

## Best Possible Solution

Keep the PR open and either revise/land it or implement the same fix on main by centralizing bare-ref provider inference for resolveConfiguredEntries, including alias-index alignment. Add focused tests for bare configured keys, bare fallbacks, ambiguous duplicate model IDs, explicit provider prefixes, and aliases on inferred bare keys.
