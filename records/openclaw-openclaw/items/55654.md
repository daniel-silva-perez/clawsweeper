## Summary

Keep open. Current main still does not expose `feishu_bitable_delete_record` as an agent-facing Feishu Bitable tool, and the linked implementation PRs are still open rather than merged. The existing delete calls are only internal cleanup for newly created Bitable apps, so the user-facing CRUD gap remains.

## What This Changes

The issue requests adding a `feishu_bitable_delete_record` tool that deletes one Feishu Bitable record through Feishu's record-delete API.

## Best Possible Solution

Keep this issue as the canonical tracker until a Feishu-owned implementation lands: add a typed `feishu_bitable_delete_record` helper in the bundled Feishu plugin, route it through the existing Bitable account-resolution helper, call the Lark/Feishu record-delete API with `app_token`, `table_id`, and `record_id`, return a structured success result, and cover registration, routing, SDK parameters, and error handling in focused Feishu plugin tests.
