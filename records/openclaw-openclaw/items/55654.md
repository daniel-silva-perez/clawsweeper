## Summary

Current `main` still does not expose an agent-facing `feishu_bitable_delete_record` tool. The Feishu Bitable registry provides list/get/create/update plus app/field creation, while the only record deletion calls are internal cleanup after creating a new Bitable app. Related PRs #40697 and #41368 remain open implementation candidates, and closed duplicate #61330 points back to this issue as the canonical tracker.

## Best Possible Solution

Keep this issue open as the canonical tracker. The likely fix belongs in `extensions/feishu/src/bitable.ts`: add a typed helper around `client.bitable.appTableRecord.delete`, add schema fields for `app_token`, `table_id`, `record_id`, and optional `accountId`, register `feishu_bitable_delete_record` through the existing Bitable tool helper so contextual routing is preserved, return a structured success result, and add focused Feishu plugin tests for registration, routing, SDK call parameters, and error handling. Review or consolidate #40697 and #41368 before landing.
