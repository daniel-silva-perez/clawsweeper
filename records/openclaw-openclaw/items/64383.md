## Summary

Keep open. Current main still implements the sandbox browser CDP path through socat, a derived Chromium CDP port, and cdpSourceRange-based ingress restrictions. Related #62873 shipped readiness/lifecycle hardening, but its own discussion says the socat proxy and CDP port offset remained for a later architectural follow-up.

## Best Possible Solution

Keep this issue open as the sandbox browser CDP architecture tracker. If maintainers accept direct CDP, remove socat and the CDP port-offset logic from Dockerfile.sandbox-browser and scripts/sandbox-browser-entrypoint.sh, update src/agents/sandbox/browser.ts orchestration/config hashing, preserve equivalent access-control and readiness guarantees, and update sandbox docs plus focused tests. If maintainers decide the relay remains necessary, document that decision and close this issue then.
