## Summary

Keep open. Current main still uses the sandbox browser socat relay, the derived Chromium CDP port offset, and the cdpSourceRange ingress model this issue asks maintainers to evaluate. The related #62873 hardening shipped readiness/lifecycle fixes, but it did not remove the relay design.

## Best Possible Solution

Keep this issue open as the core sandbox browser CDP architecture tracker. If maintainers accept direct CDP, remove socat and the CDP port-offset logic from Dockerfile.sandbox-browser and scripts/sandbox-browser-entrypoint.sh, update src/agents/sandbox/browser.ts orchestration/config hashing, preserve equivalent access-control and readiness guarantees, and update sandbox docs plus focused tests. If maintainers decide the relay remains necessary, document that decision and close this issue then.
