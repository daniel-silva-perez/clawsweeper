## Summary

Keep open. Current main still exposes ordinary Discord message sends as a scalar `mediaUrl`/`filename` path and builds one uploaded file per REST message. The newer Discord component `media-gallery` support is a separate component surface and does not solve native multi-file uploads for ordinary message actions or shared outbound reply delivery. The existing maintainer-side review comment from steipete also explicitly kept this open for the same gap.

## Best Possible Solution

Implement native multi-file sending in the bundled Discord plugin: accept an ordered multi-attachment input while preserving existing single `media`/`path`/`filePath`/`filename` behavior, load each file through the existing outbound media access helpers, batch eligible uploads into one Discord `files` array up to the verified Discord limit, split overflow into additional messages, and update focused Discord tests plus docs.
