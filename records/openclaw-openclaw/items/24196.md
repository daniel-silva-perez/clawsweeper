## Summary

Keep open. Current main still has the Discord ordinary-message upload path wired around one `mediaUrl` and one optional `filename`; shared outbound `mediaUrls` are preserved but delivered as one media send per URL instead of one Discord message with multiple native attachments. The existing maintainer-side discussion also already kept this open for the same remaining gap, and there is no open PR mentioning this issue with closing syntax.

## Best Possible Solution

Implement native multi-file sending in the bundled Discord plugin: accept ordered multi-attachment input while preserving existing single `media`/`path`/`filePath`/`filename` behavior, load every file through the existing outbound media helpers and access checks, send up to the verified Discord per-message attachment limit in one `files` array, split overflow into follow-up messages, and update focused Discord tests, docs, and changelog.
