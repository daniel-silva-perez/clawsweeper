## Summary

Keep open. Current main still exposes Discord ordinary-message uploads as one `mediaUrl` plus one optional `filename`, and the REST send helper builds a `files` array with exactly one uploaded file. Shared outbound `mediaUrls` are preserved, but the current planner/deliverer fans them out as one media send per URL, which does not satisfy the requested native multi-attachment Discord message behavior.

## Best Possible Solution

Implement native multi-file sending in the bundled Discord plugin: accept ordered multi-attachment input while preserving existing single `media`/`path`/`filePath`/`filename` behavior, load every file through the existing outbound media helpers and access checks, send up to the verified Discord per-message attachment limit in one `files` array, split overflow into follow-up messages, and update focused Discord tests, docs, and changelog.
