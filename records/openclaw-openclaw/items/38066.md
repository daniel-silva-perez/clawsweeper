## Summary

Current main still does not expose Telegram streamThrottleMs or minInitialChars through Telegram config, strict schema, generated metadata, docs, or dispatcher wiring. createTelegramDraftStream accepts throttleMs internally, but production dispatch does not pass a config-derived value and still uses the fixed 30-character initial threshold. PR #38255 remains open and unmerged as the active implementation path, so this issue should stay open for maintainer review rather than being closed.

## Best Possible Solution

Keep this issue open until PR #38255, or an equivalent maintainer-approved change, lands. The fix should add documented top-level and per-account Telegram preview cadence settings, validate them in the strict config schema, update generated config metadata and public Telegram docs, merge account overrides through the existing Telegram config path, pass resolved values into createTelegramDraftStream, and cover defaults, validation, fallback behavior, final flush behavior, and account overrides with focused tests.
