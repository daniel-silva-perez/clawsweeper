## Summary

Keep this issue open. Current main still does not expose Telegram `streamThrottleMs` or `minInitialChars` through config types, strict schema, docs, UI metadata, or dispatcher wiring. The draft stream has internal optional parameters, but production dispatch still passes the fixed `minInitialChars: 30` value and no `throttleMs`, so the 1000ms default remains. PR #38255 is the active open implementation path rather than a merged fix.

## Best Possible Solution

Keep this issue open until PR #38255 or an equivalent maintainer-approved change lands. The fix should add documented top-level and per-account Telegram preview cadence settings, validate them in the strict config schema, update generated metadata/docs/UI hints, merge account overrides through the existing Telegram config path, pass the resolved values into `createTelegramDraftStream`, and cover defaults, validation, fallback behavior, and account overrides with focused tests.
