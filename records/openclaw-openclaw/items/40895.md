## Summary

Keep open. Current main still has two Discord thread paths that can bypass per-message mention gating: bot-owned autoThread replies and bound thread sessions. The overlapping #52568 is open but not maintainer-marked as the canonical replacement, and closed PR #41072 explicitly scoped away from mention gating.

## What This Changes

The issue asks Discord thread follow-up messages to honor `requireMention` per message, or to add an explicit thread-specific mention policy for replies.

## Best Possible Solution

Define an explicit Discord thread-reply mention policy, preferably aligned with the existing Slack `thread.requireExplicitMention` pattern, then update Discord config types, strict schema, metadata/help, docs, and focused preflight/allow-list tests so autoThread and bound-thread replies can require explicit mentions without breaking intentional thread-bound session continuity.
