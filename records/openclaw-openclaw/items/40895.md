## Summary

Keep open. Current main still preserves the reported Discord thread mention-gating bypasses: bot-owned autoThread threads force the mention requirement off, and bound Discord thread sessions lower the configured mention requirement before the no-mention drop path runs. The related #52568 is useful but narrower and still open, while #41072 is closed unmerged and explicitly avoided Discord mention-gating changes.

## What This Changes

Requests that Discord thread follow-up messages honor `requireMention` on each message, or expose a thread-specific mention requirement config.

## Best Possible Solution

Define one explicit Discord thread-reply mention policy, preferably aligned with Slack's `thread.requireExplicitMention` semantics, then update Discord preflight/allow-list behavior, config types, strict schema, generated/help metadata, docs, and regression tests for both bot-owned autoThread replies and bound thread sessions.
