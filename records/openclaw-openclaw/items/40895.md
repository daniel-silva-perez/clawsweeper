## Summary

Keep open. Current main still bypasses Discord `requireMention` for bot-owned `autoThread` threads and bound thread sessions, and there is still no Discord thread-reply mention policy/config option that would satisfy the reported behavior. The related #41072 PR was scoped to ACP thread binding and did not supersede this mention-gating bug.

## Best Possible Solution

Keep this issue open and fix the Discord plugin behavior explicitly. A conservative path is to add a Discord thread-reply mention policy under the guild/channel config, wire it through config types, the strict Zod schema, generated config metadata/help, Discord docs, and preflight mention resolution, then add regression tests for bot-owned autoThreads and bound thread follow-ups with `requireMention: true`. Coordinate with #52568 if maintainers want that narrower product-shape issue to become the canonical tracker.
