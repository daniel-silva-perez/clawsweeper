## Summary

Keep this issue open. Current main still treats Discord bot-owned autoThreads and bound Discord thread sessions as implicit mention contexts, so follow-up thread messages can bypass `requireMention: true`. The supplied discussion’s `autoThread: true` observation matches the current source, and the overlapping #52568 enhancement is not marked as a canonical replacement for this bug.

## Best Possible Solution

Keep this issue open and fix the Discord plugin behavior explicitly. A conservative path is to add a Discord thread-reply mention policy under guild/channel config, wire it through config types, strict Zod schema, generated metadata/help and Discord docs, then add regression coverage for bot-owned autoThreads and bound thread follow-ups with `requireMention: true`. If maintainers prefer #52568 as canonical, mark that clearly first and close this issue as a duplicate afterward.
