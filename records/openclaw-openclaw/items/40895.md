## Summary

Keep open. Current main still preserves the reported Discord behavior: bot-owned autoThreads and bound Discord thread sessions can turn a thread reply into an implicit mention context, and there is still no Discord thread-reply mention policy/config that would make follow-up replies require their own mention. The closed #41072 PR was adjacent thread-binding work and explicitly scoped away from mention gating; #52568 is overlapping but remains an open enhancement rather than a maintainer-marked canonical replacement.

## What This Changes

The issue requests Discord thread messages to honor per-message `requireMention` rules, optionally via thread-specific config, so follow-up thread replies do not remain active after an initial mention.

## Best Possible Solution

Define a Discord thread-reply mention policy explicitly, decide whether #52568 is the canonical tracker, then update the Discord config contract, strict schema, generated metadata/help, docs, and preflight behavior so autoThread and bound-thread follow-up replies can require explicit mentions without breaking intended thread-binding continuity.
