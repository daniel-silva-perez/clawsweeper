## Summary

Keep open. Current main still preserves the reported Discord thread paths that bypass per-message mention gating: bot-owned autoThread threads disable the mention requirement, and bound Discord thread sessions lower the configured mention requirement before the no-mention drop path. The related #52568 is useful but narrower, while #41072 closed unmerged and explicitly avoided Discord mention-gating changes.

## What This Changes

The issue requests Discord thread follow-up messages to respect mention requirements instead of staying implicitly active after an initial bot mention.

## Best Possible Solution

Define an explicit Discord thread-reply mention policy, preferably aligned with Slack's `thread.requireExplicitMention` pattern, then update Discord preflight and allow-list behavior plus config types, strict schema, generated/help metadata, docs, and focused regression tests so autoThread and bound-thread replies can require explicit mentions without breaking intentional thread-bound sessions.
