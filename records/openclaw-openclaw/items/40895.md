## Summary

Keep open. Current main still has Discord thread paths that can bypass per-message mention gating: bot-owned autoThread threads disable `requireMention`, and bound thread sessions lower the mention requirement before the no-mention drop path. The overlapping #52568 is open and narrower, while #41072 was closed unmerged and explicitly scoped away from Discord mention gating.

## What This Changes

The issue requests that Discord thread follow-up messages obey mention gating, optionally through a thread-specific configuration override.

## Best Possible Solution

Define an explicit Discord thread-reply mention policy, preferably aligned with the existing Slack `thread.requireExplicitMention` pattern, then update Discord preflight/allow-list behavior, config types/schema/help metadata, docs, and focused regression tests so autoThread and bound-thread replies can require explicit mentions without breaking intentional thread-bound sessions.
