## Summary

Keep this PR open. Current main still hardcodes session-memory snapshots to `<workspace>/memory/`, while the PR addresses a real workflow problem confirmed in discussion. The PR is not ready to close as obsolete or implemented; it needs a focused revision for safe directory creation and repo docs policy before merge.

## Best Possible Solution

Keep this PR open and request a narrow revision: preserve the default `memory` output, add optional workspace-relative `hooks.internal.entries.session-memory.outputDir`, reject empty/absolute/traversal values with fallback to `memory`, write through `writeFileWithinRoot` rooted at `workspaceDir` with `relativePath` set to `outputDir/filename`, remove the separate raw pre-write mkdir or use an alias-safe helper, add tests for custom output, invalid fallback, and symlink alias behavior, and drop localized docs edits. Coordinate duplicate-indexing guidance with #54524, but do not treat that open docs/design issue as a replacement for this implementation.
