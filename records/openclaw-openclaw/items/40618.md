## Summary

Keep this PR open because the feature is still useful and current main does not implement configurable session-memory output directories, but the proposed patch is not merge-ready due to a filesystem-boundary issue and stale localized docs edits.

## What This Changes

This PR adds optional `hooks.internal.entries.session-memory.outputDir` support for the bundled session-memory hook, adds handler tests for custom and invalid output paths, and updates hook documentation.

## Best Possible Solution

Keep the optional `outputDir` feature, defaulting to `memory`, but revise the implementation so directory creation and writes go through the existing root-aware filesystem helpers, add custom-output/fallback/symlink-alias regression tests, update English docs and bundled hook metadata only, and leave broader duplicate-indexing guidance to #54524.

## Reproduction Assessment

Yes. Configure `hooks.internal.entries.session-memory.outputDir`, trigger `/new` or `/reset`, and current main still writes under `<workspace>/memory/`; for the PR-introduced bug, set `outputDir` to an in-workspace symlink prefix such as `link/archive` and the raw mkdir can affect the symlink target before the safe writer rejects the final file.

## Solution Assessment

No, not in the current patch shape. The feature direction is narrow and maintainable, but the implementation should use `mkdirPathWithinRoot` or rely on `writeFileWithinRoot` parent creation and should drop localized docs edits before merge.
