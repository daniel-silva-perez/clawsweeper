## Summary

Keep this PR open because current main still hardcodes `session-memory` output under `memory/`, and the requested configurable output directory is a real user-facing improvement. The PR is not merge-ready as written because it introduces a filesystem-boundary side effect before the safe writer runs and edits localized docs that repo policy says are not maintained here.

## What This Changes

This PR adds optional `hooks.internal.entries.session-memory.outputDir` support to the bundled `session-memory` hook, updates handler tests, and documents the option across hook docs including localized zh-CN files.

## Best Possible Solution

Keep the optional output-directory direction, but revise it against current main so `outputDir` validation covers empty, absolute, traversal, and symlink-alias cases before any filesystem side effects; create parents through `writeFileWithinRoot` or `mkdirPathWithinRoot`; preserve the default `memory` output; add custom, fallback, and symlink tests; update English docs plus bundled hook metadata only; and coordinate wording with #54524 for duplicate-indexing guidance.

## Reproduction Assessment

Yes. Configure `hooks.internal.entries.session-memory.outputDir`, trigger `/new` or `/reset`, and current main still writes under `<workspace>/memory/`; for the PR bug, use an in-workspace symlink prefix in `outputDir` and observe the raw mkdir side effect outside the workspace before the final safe write rejects it.

## Solution Assessment

No, not as currently implemented. The feature shape is maintainable, but the PR should rely on the existing boundary-aware filesystem helpers and drop localized docs edits before it is the safest solution.
