## Summary

Keep this PR open. Current main still lacks a configurable session-memory output directory, and the discussion includes a real workspace use case for separating session snapshots from durable memory, but the proposed patch should not merge as-is because it introduces a filesystem-boundary regression and edits localized docs that this repo does not maintain.

## What This Changes

The PR adds an optional workspace-relative `hooks.internal.entries.session-memory.outputDir` for the bundled session-memory hook, updates session-memory tests, and documents the new option in English and zh-CN hook docs.

## Best Possible Solution

Preserve the backwards-compatible default `memory` output, add the optional workspace-relative `outputDir` in the session-memory handler, validate empty/absolute/traversal values with fallback to `memory`, write through the existing root-scoped safe writer without raw pre-write mkdir side effects, cover custom output, invalid fallback, and symlink alias behavior in tests, and update only English source docs plus bundled hook metadata.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Guard output directory before creating it:** `src/hooks/bundled/session-memory/handler.ts:213`
  - body: Calling `fs.mkdir(outputRoot, { recursive: true })` can follow an in-workspace symlink such as `link/archive` and create directories outside the workspace before `writeFileWithinRoot` rejects the final file write. Please remove this raw mkdir or replace it with an alias-safe root-scoped helper so the new workspace-relative guarantee has no external side effects.
  - confidence: 0.91
- **[P3] Drop localized docs edits:** `docs/zh-CN/automation/hooks.md:39`
  - body: `docs/AGENTS.md` says localized docs under `docs/<locale>/**` are not maintained in this repo. Please keep the English docs as source of truth and remove the zh-CN changes so the i18n pipeline can regenerate them separately.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The patch has a concrete filesystem-boundary concern in the new output directory creation path.

Concerns:

- **[medium] Raw mkdir can escape through workspace symlinks:** `src/hooks/bundled/session-memory/handler.ts:213`
  - body: The new `fs.mkdir(outputRoot, { recursive: true })` runs before the alias-safe write path. If `outputDir` starts with an in-workspace symlink that points outside the workspace, this can create external directories even though the later `writeFileWithinRoot` call blocks the final file write.
