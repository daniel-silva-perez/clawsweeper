## Summary

Keep this PR open. Current main still writes session-memory snapshots only under `memory/`, and the discussion includes a credible live-workspace report that configurable output would reduce memory indexing noise. The proposed branch is useful, but it should not merge as-is because it creates the configured directory with raw `fs.mkdir` before the root-aware writer runs, and it also carries stale localized docs edits that current repo policy excludes.

## What This Changes

This PR adds optional `hooks.internal.entries.session-memory.outputDir` support for the bundled session-memory hook, adds handler tests for custom and invalid output paths, and updates hook documentation and metadata.

## Best Possible Solution

Keep the optional `outputDir` feature, defaulting to `memory`, but revise the implementation so directory creation and writes use the existing root-aware filesystem helpers, add a symlink-alias regression test alongside the custom-output and fallback tests, update English docs and bundled hook metadata only, and leave broader duplicate-indexing guidance to #54524.

## Reproduction Assessment

Yes. Configure `hooks.internal.entries.session-memory.outputDir`, trigger `/new` or `/reset`, and current main still writes under `<workspace>/memory/`; for the PR-introduced bug, use an in-workspace symlink prefix such as `link/archive` and the raw mkdir can affect the symlink target before the safe writer rejects the final file.

## Solution Assessment

No, not in the current patch shape. The feature direction is narrow and maintainable, but the implementation should use `mkdirPathWithinRoot` or rely on `writeFileWithinRoot` parent creation and should drop localized docs edits before merge.
