## Summary

Keep open. The requested capability is still missing on current main, and the live-workspace comment confirms the use case is real, but this PR should not merge as-is because the branch creates the configured directory with raw `fs.mkdir` before the root-scoped safe writer and still edits localized docs that current repo policy excludes.

## What This Changes

This PR adds optional `hooks.internal.entries.session-memory.outputDir` support to the bundled `session-memory` hook, updates handler tests, and documents the option across hook docs including localized zh-CN files.

## Best Possible Solution

Keep the optional output-directory direction, but revise it against current main so `outputDir` validation covers empty, absolute, traversal, and symlink-alias cases before any side effects; create parents through the pinned safe writer or `mkdirPathWithinRoot`; preserve the default `memory` output; add custom, fallback, and symlink tests; update English docs plus bundled hook metadata only; and coordinate wording with #54524 for the broader duplicate-indexing guidance.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Guard output directory before creating it:** `src/hooks/bundled/session-memory/handler.ts:213`
  - body: Calling `fs.mkdir(outputRoot, { recursive: true })` before the root-scoped write can follow an in-workspace symlink prefix such as `link/archive` and create directories outside the workspace. The later `writeFileWithinRoot` rejection does not undo that side effect, so create parents through the pinned writer or a boundary-aware mkdir helper before accepting this workspace-relative setting.
  - confidence: 0.91
- **[P3] Remove localized docs edits:** `docs/zh-CN/automation/hooks.md:42`
  - body: `docs/AGENTS.md` says foreign-language docs under `docs/<locale>/**` are not maintained in this repo. Drop the zh-CN edits and let the docs publish/i18n pipeline regenerate translations from the English source docs.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The diff introduces a concrete filesystem-boundary concern in the configured output-directory creation path.

Concerns:

- **[medium] Raw mkdir can escape through workspace symlinks:** `src/hooks/bundled/session-memory/handler.ts:213`
  - body: The new pre-write `fs.mkdir(outputRoot, { recursive: true })` runs before the alias-safe write path. If `outputDir` starts with an in-workspace symlink that points elsewhere, it can create external directories before `writeFileWithinRoot` blocks the final file write.
  - confidence: 0.91
