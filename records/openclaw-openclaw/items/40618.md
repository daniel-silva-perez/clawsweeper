## Summary

Keep open. Current main still lacks a configurable session-memory output directory, and the discussion shows this solves a real memory-indexing/noise problem, but the PR should not merge as-is because it introduces a filesystem-boundary side effect and includes localized docs edits that conflict with repo policy.

## What This Changes

This PR adds optional `hooks.internal.entries.session-memory.outputDir` support for the bundled `session-memory` hook, adds handler tests for custom and invalid paths, and updates hook documentation including localized docs files.

## Best Possible Solution

Keep the optional `outputDir` feature, defaulting to `memory`, but revise it on top of current main so all directory creation and writes go through the existing root-aware filesystem helpers, add custom-output/fallback/symlink-alias regression tests, update English docs and bundled hook metadata only, and leave the broader duplicate-indexing guidance to #54524.

## Reproduction Assessment

Yes. Configure `hooks.internal.entries.session-memory.outputDir`, trigger `/new` or `/reset`, and current main still writes under `<workspace>/memory/`; for the PR-introduced bug, set `outputDir` to an in-workspace symlink prefix such as `link/archive` and the raw mkdir can affect the symlink target before the safe writer rejects the final file.

## Solution Assessment

No, not in the current patch shape. The feature direction is narrow and maintainable, but the implementation should rely on `mkdirPathWithinRoot` or `writeFileWithinRoot` for parent creation and drop localized docs edits before merge.
