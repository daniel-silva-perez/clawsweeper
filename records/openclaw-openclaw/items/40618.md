## Summary

Keep this PR open. Current main still hardcodes session-memory output under `memory/`, and the discussion includes a real live-workspace use case for a configurable directory, but the proposed patch needs revision because it creates the configured output directory before the boundary-safe write path and also edits localized docs that current repo policy says are not maintained here.

## What This Changes

This PR adds optional `hooks.internal.entries.session-memory.outputDir` support to the bundled `session-memory` hook, extends handler tests, and updates hook documentation including localized docs files.

## Best Possible Solution

Revise this PR against current main to keep the optional `outputDir` behavior, preserve the default `memory` directory, validate empty/absolute/traversal values, perform parent creation through `writeFileWithinRoot` or `mkdirPathWithinRoot` under the workspace root, add custom-output, fallback, and symlink-alias tests, update English docs plus bundled hook metadata only, and coordinate duplicate-indexing guidance with #54524.

## Reproduction Assessment

Yes. Configure `hooks.internal.entries.session-memory.outputDir`, trigger `/new` or `/reset`, and current main still writes under `<workspace>/memory/`; for the PR-introduced bug, set `outputDir` to an in-workspace symlink prefix such as `link/archive` and observe directory creation at the symlink target before the final safe write rejects it.

## Solution Assessment

No, not as currently implemented. The feature shape is the narrow maintainable fix, but the patch should rely on existing boundary-safe filesystem helpers and drop localized docs edits before it is safe to merge.
