## Summary

Keep this PR open. The requested configurable session-memory output directory still is not implemented on current main, and the discussion includes a real live-workspace use case for separating session snapshots from durable memory, but the current PR head has a blocking filesystem-boundary issue and stale localized-doc edits that need revision before merge.

## What This Changes

The PR adds an optional workspace-relative `hooks.internal.entries.session-memory.outputDir` setting for the bundled `session-memory` hook, updates handler tests, and documents the option in hook docs.

## Best Possible Solution

Keep the optional `outputDir` feature, but implement it against current main by validating empty, absolute, traversal, and symlink-alias cases before any side effects; create parents only through the root-scoped safe writer or `mkdirPathWithinRoot`; preserve the default `memory` output; add custom, fallback, and symlink tests; and update English source docs plus bundled hook metadata only. Leave #54524 as the broader duplicate-indexing design/docs follow-up.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Guard output directory before creating it:** `src/hooks/bundled/session-memory/handler.ts:213`
  - body: Calling `fs.mkdir(outputRoot, { recursive: true })` before the root-scoped writer can follow an in-workspace symlink such as `link/archive` and create directories outside the workspace even though the later write is rejected. Remove this raw mkdir or replace it with a boundary-aware helper so the workspace-relative guarantee has no external side effects.
  - confidence: 0.91

  - body: `docs/AGENTS.md` says localized docs under `docs/<locale>/**` are not maintained in this repo. Keep the English docs as source of truth and drop the zh-CN edits so the i18n pipeline can regenerate them separately.
  - confidence: 0.87
