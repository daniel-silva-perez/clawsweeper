## Summary

Keep open. Current main still has the shared `read` action, but Telegram does not expose or implement it through discovery, adapter mapping, runtime dispatch, config gating, tests, or docs. The prior inbound-store PR and duplicate report are useful context, but the PR closed unmerged and the remaining design needs maintainer product/privacy review.

## What This Changes

The issue requests Telegram support for the shared `message read` action so agents can read recent Telegram group messages through `openclaw message read`.

## Best Possible Solution

Implement Telegram `message read` inside the Telegram plugin with a bounded runtime-owned inbound message store or equivalent cache, avoid secondary `getUpdates` polling, wire the action through discovery, mapping, runtime dispatch, config gating, CLI target/topic semantics, docs, and focused tests, and document the platform limitation that it can only read observed recent messages.
