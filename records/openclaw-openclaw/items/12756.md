## Summary

Keep open. Current main still lacks Telegram support for the shared message read action across discovery, adapter mapping, runtime dispatch, config gating, docs, and focused tests. The related inbound-store PR is closed unmerged and the duplicate report is closed, so this issue remains the live tracker rather than a cleanup-close candidate.

## What This Changes

Add Telegram support for the shared message read action so `openclaw message read --channel telegram --target ...` can return recent Telegram chat messages.

## Best Possible Solution

Implement Telegram message read inside the Telegram plugin with explicit observed-message semantics, a bounded privacy-reviewed inbound cache or equivalent runtime-owned store, no secondary getUpdates polling, and end-to-end wiring through action discovery, mapping, runtime dispatch, config schema/help, CLI and channel docs, and focused tests.
