## Summary

Keep this issue open. Current main still has `read` as a shared message action, but Telegram does not advertise, map, gate, dispatch, test, or document a Telegram message-history read path. The closed inbound-store PR and closed duplicate confirm the same gap, but neither landed in current main.

## What This Changes

The issue requests adding Telegram support for the shared `message read` action so agents can retrieve recent Telegram group messages with target and limit-style parameters.

## Best Possible Solution

Implement Telegram `message read` in the Telegram plugin with a bounded runtime-owned inbound message store or equivalent cache, avoid secondary `getUpdates` polling, and wire the feature through action discovery, action mapping, runtime dispatch, config gating, CLI target/topic semantics, docs, and focused tests.
