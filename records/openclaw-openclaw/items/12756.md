## Summary

Keep open. Current main still lacks Telegram support for the shared `message read` action across action discovery, adapter mapping, runtime dispatch, config gating, docs, and tests. The prior inbound-store PR is useful context but closed unmerged, and the duplicate report is closed, so this issue remains the live tracker.

## What This Changes

Add Telegram support for the shared `message read` action so Telegram chats can return recent messages through the message tool.

## Best Possible Solution

Implement Telegram `message read` inside the Telegram plugin using a privacy-reviewed bounded observed-message cache, avoid secondary `getUpdates` polling, expose it through discovery/mapping/runtime/config/docs/tests, and document that Telegram can only return recent messages observed by the running bot.
