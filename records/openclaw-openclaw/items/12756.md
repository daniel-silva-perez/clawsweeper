## Summary

Keep this issue open. Current main still does not implement Telegram `message read`: the shared `read` action exists, but the Telegram plugin does not advertise it, map it, gate it, dispatch it, or document it. The prior implementation PR #14367 is relevant design context, but it is closed unmerged, and no open PR currently references this issue with closing syntax.

## Best Possible Solution

Keep this issue open. The best path is a maintainer-reviewed Telegram-plugin implementation: advertise `read`, map it to a runtime action, add a bounded inbound message store or approved equivalent, support `--target`/`to`, `limit`, `before`, `after`, and topic/thread semantics, update docs, and add focused adapter/runtime/contract tests. Treat #14367 as prior art, not shipped proof.
