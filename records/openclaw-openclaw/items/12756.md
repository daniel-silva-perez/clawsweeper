## Summary

Keep open. Current main still has the shared `read` message action, but the Telegram plugin does not expose, map, gate, dispatch, test, or document Telegram message-history reads. The related inbound-store PR is closed unmerged and the duplicate report is closed, so this issue remains the live tracker for the remaining Telegram-specific work.

## What This Changes

The issue requests Telegram support for the shared `message read` action so agents can read recent Telegram group/chat messages through `openclaw message read --channel telegram`.

## Best Possible Solution

Implement Telegram `message read` inside the Telegram plugin with explicit observed-message semantics, a bounded privacy-reviewed inbound cache or equivalent runtime-owned store, no secondary `getUpdates` polling, and end-to-end wiring through action discovery, mapping, runtime dispatch, config schema/help, CLI and channel docs, and focused tests.
