## Summary

Keep open. Current main still has no before-prompt debug log for the `route="fits"` preemptive-compaction path, and the reported large-context/cache-TTL behavior is not clearly fixed or superseded by another item.

## What This Changes

The issue requests debug logging for every preemptive auto-compaction decision, especially `route="fits"`, and asks to investigate whether cache-TTL pruning can suppress large-context compaction estimates.

## Best Possible Solution

Add low-noise debug observability to the before-prompt precheck so every evaluation, including `route="fits"`, records the route and budget metrics. Treat any pruning/threshold behavior change as a second step that requires a focused failing test or live proof from the reported large-context setup.

## Reproduction Assessment

Partially: the missing `route="fits"` log is reproducible by static source inspection of current main. The Opus/cache-TTL failure path has detailed reported steps, but I do not have a high-confidence local reproduction of the long live session behavior.

## Solution Assessment

Unclear for the full compaction behavior, but the narrow logging request is the best first fix. Changing thresholds or estimating against a different message view would need stronger proof that cache-TTL pruning is actually masking the before-prompt estimate on current main.
