## Summary

Keep open. Current main still lacks the stale transient async notice policy and session-advance filtering proposed here, so this PR is not obsolete. The branch is relevant, but it still needs maintainer review/revision around non-completion exec notices, event consumption order, and route-context preservation before merge.

## What This Changes

The PR branch adds stale-policy metadata to selected transient async system events, filters stale queued events based on session activity in reply and heartbeat paths, adjusts exec-event heartbeat prompting, and adds infra regression tests.

## Best Possible Solution

Keep this PR open and revise it against current main. The best implementation should mark only transient success/completion notices as stale-droppable, pass latest session activity into reply and heartbeat drain paths, drop only marked stale entries, preserve durable blocked/error/timeout/denied/started/running events and delivery context, and add regression coverage for stale success, non-completion exec notices, mixed cron queues, and route-context preservation.
