## Summary

Keep open. Current main still lacks the requested `/restore` command, and the latest PR head is useful but not merge-ready because it can restore into a transcript file the session store does not read and can expose keyless or unreadable archives across sessionKey boundaries. The open related feature request #58028 reinforces that this is still desired work, not a cleanup close.

## What This Changes

The PR adds a built-in `/restore [number]` chat command, archive listing/restoration helpers, sessionKey transcript headers, docs, and focused tests for archived session transcript restoration.

## Best Possible Solution

Revise the branch so restore uses and validates `resetResult.entry.sessionFile` as the destination, enforces a private and explicit policy for legacy/keyless archives, distinguishes unreadable headers from legitimate legacy headers, scans archives until enough matching entries are found under a bounded policy, and lets a single reset path own hook emission. Add coverage for multi-channel isolation, persisted sessionFile destinations, topic/thread sessions, long/corrupt headers, and archive-list drift.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Restore into the persisted reset session file:** `src/gateway/session-restore.ts:248-252`
  - body: `performGatewaySessionReset` persists the active `entry.sessionFile`, often preserving the current transcript path, but this derives a new path from `newSessionId` and topic. The archive can be moved to a file the session store never reads, so `/restore` reports success while the next turn starts from the reset-created transcript. Use and validate `resetResult.entry.sessionFile` as the destination.
  - confidence: 0.94
- **[P1] Require owned archive headers before listing or restoring:** `src/gateway/session-restore.ts:201`
  - body: `readSessionKeyFromHeader` returns `null` for missing keys, parse failures, truncated headers, and read errors, and this condition accepts `null` for any requested `sessionKey`. In a multi-channel instance that can expose or restore another channel's archived transcript. Distinguish safe legacy archives from unreadable headers, or skip keyless archives unless ownership is explicit.
  - confidence: 0.91
- **[P2] Continue scanning until enough matching archives are found:** `src/gateway/session-restore.ts:191-193`
  - body: The scan ceiling is applied before the sessionKey filter, so newer archives from other channels can consume the whole window and make valid current-channel history disappear. Bound the total work if needed, but enforce `limit` after filtering or keep scanning until enough matches are found.
  - confidence: 0.86
- **[P2] Avoid emitting reset hooks twice:** `src/auto-reply/reply/commands-restore.ts:103-113`
  - body: This handler emits reset command hooks and then `performSessionRestore` immediately calls `performGatewaySessionReset`, which also triggers internal command and before-reset hooks. Hook consumers can observe duplicate side effects or inconsistent pre-reset snapshots; let one reset path own hook emission for restore.
  - confidence: 0.82
