## Summary

Keep this PR open. Current main still does not implement the proposed `/restore` chat command, restore handler, archive browser, or reset/deleted transcript restore helper. The shipped restore surface is limited to compaction checkpoints, while `/new` and `/reset` still archive old transcripts, so the PR remains a plausible implementation candidate rather than obsolete or superseded.

## Best Possible Solution

Keep this PR open for maintainer review as the implementation candidate for a first-party `/restore [number]` flow. The best path is to land or replace it with an equivalent sessionKey-safe restore implementation that lists recent reset/deleted archives, restores into the active session entry’s resolved transcript path, preserves expected reset semantics and hooks, and includes focused tests for multi-channel isolation, topic/thread sessions, legacy archives, corrupt headers, and restore selection races.
