## Summary

Keep open. Current main still does not solve the reported BytePlus/Volcengine Ark `patternProperties` rejection: schema cleanup is automatic only for Google/Gemini, generic keyword stripping requires explicit model compat metadata, and the bundled Ark catalogs do not declare `patternProperties` unsupported. There is also an open unmerged PR with closing syntax for this issue, so the best maintainer path is to review, land, or replace that implementation candidate before closing.

## Best Possible Solution

Keep this issue open and review PR #57480. If maintainers do not want universal non-Anthropic schema cleaning, implement a narrower current-main fix by adding BytePlus/Volcengine Ark model compatibility metadata for `unsupportedToolSchemaKeywords: ["patternProperties"]` and a regression test proving `exec.env` no longer sends `patternProperties` for those provider/model refs while preserving Anthropic behavior.
