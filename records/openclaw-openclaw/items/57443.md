## Summary

Keep open. Current main has a generic `modelCompat.unsupportedToolSchemaKeywords` path, but the bundled BytePlus/Volcengine Ark catalogs do not declare `patternProperties` unsupported, and the normalizer still only applies Gemini cleaning automatically for Google/Gemini providers. There is also an open unmerged PR with closing syntax for this issue, so the best maintainer path is to review or replace that implementation candidate rather than close the issue pre-merge.

## Best Possible Solution

Review and either land or close PR #57480. If maintainers do not want universal non-Anthropic schema cleaning, implement a current-main patch that adds explicit BytePlus/Volcengine Ark compatibility metadata for `patternProperties` and a regression test showing the `exec.env` schema no longer sends `patternProperties` for those provider/model refs while preserving Anthropic behavior.
