## Summary

Keep open. Current main still leaves the reported BytePlus/Volcengine Ark path uncovered: automatic schema cleaning remains Google/Gemini-only, generic keyword stripping depends on explicit model compatibility metadata, and the bundled Ark catalogs do not declare `patternProperties` unsupported. An open unmerged PR already targets this issue with closing syntax, so this should not be closed before that implementation candidate is reviewed, landed, replaced, or closed.

## What This Changes

The issue requests that tool JSON Schema cleanup strip `patternProperties` for BytePlus/Volcengine Ark and similar OpenAI-compatible providers instead of limiting that cleanup to Google/Gemini providers.

## Best Possible Solution

Ship a focused fix that removes `patternProperties` from outgoing tool schemas for BytePlus/Volcengine Ark models while preserving Anthropic’s full JSON Schema behavior, with regression coverage around the `exec.env` schema and the selected provider/model compatibility path.

## Reproduction Assessment

Yes. The static path is high-confidence: `exec.env` still uses a record schema, current normalization does not clean non-Gemini providers without compat metadata, and the Ark provider manifests do not provide that metadata; final validation should add a targeted unit test and, if credentials are available, a live Ark tool-call smoke.

## Solution Assessment

Unclear. The open PR’s universal non-Anthropic approach may solve the reported behavior, but explicit Ark-family compat metadata plus a regression test is likely the narrower maintainable fix unless maintainers deliberately want the broader policy.
