## Summary

Keep this PR open, but do not merge it as-is. Current main still has the fallback-context persistence bug the PR targets, while the PR branch is stale enough to regress current-main session-store behavior around runtime context windows and cost snapshots.

## What This Changes

The PR changes agent session-store persistence to avoid writing fallback contextTokens when lookup is unresolved, adds regression tests, and also carries unrelated plugin/provider type cleanup plus an ambient @copilotkit/aimock declaration.

## Best Possible Solution

Rebase or replace this with a focused session-store patch that only persists positive resolved context windows, preserves runtime agentMeta.contextTokens, clears stale stored contextTokens only when the authoritative runtime model/provider changes without a resolved replacement, and keeps current-main compaction, interaction, harness, and cost-snapshot semantics intact.

## Reproduction Assessment

Yes. The current-main source path is enough to reproduce the bug at code level: unresolved lookup returns the caller fallback, updateSessionStoreAfterAgentRun persists it, and status later treats the stored value as an override.

## Solution Assessment

No, not as currently proposed. The target fix is the right direction, but this branch needs a rebase and scope cleanup before it is the narrowest maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve runtime-provided context windows:** `src/agents/command/session-store.ts:62-69`
  - body: Current main stores `result.meta.agentMeta.contextTokens` before falling back to lookup, which covers runtimes that know their window even when discovery/config lookup is unresolved. This branch resolves only from `contextTokensOverride` and lookup, so merging it would drop runtime-provided budgets and persist `undefined` for those runs.
  - confidence: 0.88
- **[P2] Keep cost as a snapshot, not an accumulator:** `src/agents/command/session-store.ts:133-135`
  - body: Current main fixed #69347 by assigning `estimatedCostUsd = runEstimatedCostUsd` because the usage object is already cumulative for the run. This branch still adds the run cost to the existing entry, which would reintroduce multi-persist cost inflation on a stale merge.
  - confidence: 0.9
