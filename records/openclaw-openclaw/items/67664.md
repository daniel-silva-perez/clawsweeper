## Summary

Keep this PR open. Current main still persists the default context-token fallback into session-store entries when lookup is unresolved, so the targeted bug remains real; the PR is a plausible implementation candidate but needs maintainer review because current main has newer runtime-context and cost-snapshot behavior that the branch must preserve, and the branch also includes unrelated extension/type cleanup.

## What This Changes

The PR changes agent session-store persistence and tests to avoid writing default fallback contextTokens on unresolved lookup, while also carrying extension/provider type cleanup and a new aimock ambient type declaration.

## Best Possible Solution

Rebase or replace this with a focused session-store fix that only persists positive, resolved context-token values, preserves an existing value when the runtime model/provider has not changed, clears stale values when the persisted runtime model changes without a resolved replacement, and keeps current-main runtime metadata and cost snapshot semantics intact. Add regression tests for unresolved no-write, preserve-existing, model/provider drift clearing, and authoritative on-disk merge behavior.
