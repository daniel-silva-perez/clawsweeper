## Summary

Keep open. Current main still writes `cwd: process.cwd()` in all three transcript-header creation paths named by this PR, so the intended regression fix is not implemented on main. The PR remains a focused candidate fix; the provided discussion shows the earlier test cleanup concern was addressed, and the security pass did not find obvious supply-chain-sensitive changes in the provided file list.

## Best Possible Solution

Review and land a cleaned-up version of this PR or an equivalent maintainer patch that uses `resolveAgentWorkspaceDir(cfg, agentId)` for sessions.create, chat-created transcript injection, and channel/DM assistant-message transcript append paths. Prefer threading config/workspace context through existing call paths, then add focused regression tests proving new headers use the agent workspace instead of the gateway process cwd.
