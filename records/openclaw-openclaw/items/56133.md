## Summary

Keep open. Current main still writes transcript header cwd from process.cwd() in the gateway session, mirrored transcript, and chat transcript creation paths, so the underlying bug is real and not implemented on main. The PR is useful but not merge-ready: GitHub reports the branch dirty against current main, current main has since changed the same transcript helper, and the patch still leaves at least one no-agent chat transcript creation path able to fall back to process.cwd().

## What This Changes

The branch replaces process.cwd() with resolveAgentWorkspaceDir(...) in several session transcript header helpers and adds a gateway regression test for the session transcript helper.

## Best Possible Solution

Land one rebased, current-main-compatible fix that removes process.cwd from all transcript header creation paths, threads or derives the target agent workspace consistently, preserves current lazy imports, and adds focused regression coverage for mirrored delivery transcripts, sessions.create, and chat-created/abort-created transcripts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Derive cwd for callers without agentId:** `src/gateway/server-methods/chat.ts:769-771`
  - body: `ensureTranscriptFile` still falls back to `process.cwd()` when `agentId` is absent. Existing create-if-missing callers such as aborted partial persistence do not pass `agentId`, so a missing transcript can still be stamped with `/app` or another process-global cwd. Resolve the agent from the session key or require/pass `agentId` for every creation path.
