## Summary

Keep open. The underlying transcript-cwd bug is still present on current main, but this PR is not merge-ready as the canonical fix: its chat transcript helper still falls back to process.cwd() when callers omit agentId, and the aborted-partial path is one such caller. The work also overlaps the open #49523/#50454 cluster, so a maintainer should consolidate the current-main fix before landing.

## What This Changes

The PR replaces process.cwd() with resolveAgentWorkspaceDir(...) in several session transcript header creation helpers and adds a focused regression test for ensureSessionTranscriptFile.

## Best Possible Solution

Land one current-main-compatible fix that removes process.cwd() from all session transcript header creation paths, derives the target agent workspace from config plus session key or explicit agent id, preserves current lazy imports, and adds regression coverage for mirrored delivery, sessions.create, chat-created transcripts, and abort-created transcripts.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a missing transcript on current main through sessions.create, mirrored delivery, or chat abort persistence while the process cwd is /app or another non-workspace directory; the header records that process cwd instead of the agent workspace.

## Solution Assessment

No. The proposed direction is right, but the PR's current patch is incomplete because it keeps a process.cwd() fallback for missing-agent callers and does not consolidate with the related open #49523/#50454 work.
