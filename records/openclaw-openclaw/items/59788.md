## Summary

Keep open. Current main has outbound-session mirroring infrastructure, but `openclaw message send` still seeds the action with the default agent and does not reverse-resolve `--channel whatsapp --account realtor-whatsapp` through inbound routing bindings to choose the bound non-default agent. That means the requested account-routed outbound-first WhatsApp workflow is not fully implemented on main.

## Best Possible Solution

Keep this issue open and implement account-aware outbound session start for `openclaw message send`: resolve the selected channel/account/peer through the same routing contract used by inbound messages or add an explicit `--agent`/session flag, then mirror the delivered outbound assistant turn into that resolved agent session with coverage for WhatsApp `dmScope` and non-default agent bindings.
