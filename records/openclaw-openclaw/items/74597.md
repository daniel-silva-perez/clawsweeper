## Summary

Keep open. The reporter narrowed the original incident to a manual source-level global install, but current main still lacks the requested diagnostic guard: update/status classify a symlinked package root that resolves to a git checkout as a git install, and doctor only compares service entrypoint realpaths rather than flagging an npm-global package root that points into a mutable source checkout.

## What This Changes

The issue requests status, doctor, and update safeguards for npm-global OpenClaw installs whose global package root resolves through a symlink into a source checkout.

## Best Possible Solution

Add a shared install-layout diagnostic that records both the lexical global package root and its realpath, detects when an npm-global package root points into a git/source checkout unexpectedly, and surfaces clear status, doctor, update pre/post verification, and repair guidance while preserving intentional dev-channel behavior.
