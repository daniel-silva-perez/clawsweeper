## Summary

Keep open. Current main still lacks the mixed install-layout diagnostic the narrowed report asks for: package-root resolution can classify a symlinked npm-global root as a git install, and doctor/status do not flag that the global package root itself resolves into a mutable source checkout.

## What This Changes

The issue asks OpenClaw to prevent or diagnose npm-global package installs whose `openclaw` package root resolves to a git/source checkout symlink, especially across update/status/doctor flows.

## Best Possible Solution

Add a small shared install-layout diagnostic that preserves both the package-root path and its realpath, detects npm-global package roots that resolve into source checkouts when the expected layout is package-based, and surfaces a clear warning/failure signal in status, doctor, and update verification without changing intentional dev-channel source installs.
