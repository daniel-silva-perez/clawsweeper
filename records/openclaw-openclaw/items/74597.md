## Summary

Keep open. The reporter narrowed the original incident to a manual source-level global install, but current main still does not provide the requested guard for an npm-global package root that resolves through a symlink into a source checkout; the active implementation candidate is still the open related PR #74920.

## What This Changes

The issue requests update/status/doctor safeguards so an npm-global OpenClaw package install cannot silently point at a mutable source checkout symlink and leave gateway runtime layout mixed.

## Best Possible Solution

Add a shared install-layout diagnostic that preserves both the lexical package root and resolved realpath, detects package roots pointing into git/source checkouts, and surfaces that state in status, doctor, and update pre/post verification while keeping documented dev-channel source installs explicit.

## Reproduction Assessment

Yes. The issue includes direct systemd log evidence for npm install -g /opt/openclaw-source, and current main has a focused code/test path showing a symlinked OpenClaw root into a git repo is classified as installKind git.

## Solution Assessment

Yes, the diagnostic/guard direction is the best narrow solution. Removing dev-channel source installs would conflict with documented behavior; preserving lexical root plus realpath lets OpenClaw distinguish intentional source installs from mixed package/source layouts.
