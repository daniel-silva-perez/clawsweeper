## Summary

Keep open. The reporter narrowed the original incident to a manual source-level global install, but current main still lacks the requested guard for an npm-global package root resolving to a mutable source checkout; status currently collapses that layout to a git install, and doctor only compares resolved entrypoints. Open PR #74920 appears to be the active implementation candidate, but it is still unmerged.

## What This Changes

The issue asks OpenClaw to detect and guard against npm-global installs whose package root resolves to a source checkout symlink, with clearer update/status/doctor diagnostics and reliable failure signaling.

## Best Possible Solution

Add a shared install-layout diagnostic that preserves both the lexical package root and its realpath, detects npm-global roots pointing into git/source checkouts, and surfaces clear status, doctor, update pre/post verification, and repair guidance while keeping documented dev-channel source installs explicit.

## Reproduction Assessment

Yes. The issue includes direct systemd log evidence for npm install -g /opt/openclaw-source, and current tests can reproduce the diagnostic gap by symlinking an OpenClaw root to a git repo; current main expects that case to classify as git.

## Solution Assessment

Yes, the diagnostic/guard direction is the best narrow fix. Removing dev-channel source installs would conflict with documented behavior, while preserving lexical root plus realpath lets status, doctor, and update distinguish intentional source installs from mixed package/source layouts.
