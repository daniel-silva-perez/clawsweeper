## Summary

Current main does not implement a clear fix for this issue. The bundled Grok web_search path still calls xAI Responses through the shared web-tool fetch helper, still inherits the shared 30 second default timeout, and still lets AbortError-shaped failures propagate through the tool adapter. The linked #58733 is useful but explicitly narrower than the reported immediate-abort symptom, and #43609 remains relevant unmerged follow-up work rather than shipped evidence.

## Best Possible Solution

Keep this issue open and use #58733 plus #43609 as related implementation evidence. The best fix is a focused current-main change in the bundled xAI/Grok web_search path: add a provider-appropriate default or provider-owned timeout config, wrap internal timeout AbortErrors into ordinary tool errors with useful diagnostics, preserve explicit user timeout overrides, and add focused regression tests for the xAI Responses path.
