## Summary

This issue should stay open. Current main still has the core heartbeat behavior the report describes: non-interval wake reasons are not gated by the configured interval, targeted/action-driven wakes still advance the next schedule as a fresh run, and the macOS launchd plist generator still lacks the proposed ProcessType=Interactive mitigation. The provided GitHub context also shows two open reporter-authored PRs, #62308 and #62310, directly targeting the remaining launchd and heartbeat-runner slices, so the best cleanup path is to review or supersede those PRs rather than close the parent bug.

## Best Possible Solution

Keep this issue open as the parent tracker. Review and either land or replace the two focused PRs with an equivalent current-main fix: add a defensible minimum-spacing rule for non-interval heartbeat wake reasons, preserve interval scheduling semantics, handle overdue timer drift if needed, and make an explicit maintainer decision on ProcessType=Interactive for macOS launchd installs.
