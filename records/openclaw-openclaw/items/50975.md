## Summary

Keep this PR open. Current main still rejects prefixless Anthropic setup tokens and still uses the legacy `sk-ant-oat01-` prefix plus an 80-character minimum. The PR also needs rework before merge because the proposed condition would allow too-short legacy-prefixed tokens, and the implementation has moved to the shared provider auth helper.

## Best Possible Solution

Rework the PR against `src/plugins/provider-auth-token.ts`: accept both legacy-prefixed and new prefixless Anthropic setup tokens, apply the minimum length check unconditionally, and add focused unit coverage for blank input, valid prefixless tokens, valid legacy-prefixed tokens, too-short prefixless input, and too-short legacy-prefixed input. Keep the patch limited to Anthropic setup-token validation and avoid unrelated auth, provider, dependency, or workflow changes.

- Merging the PR as written would weaken validation by accepting truncated `sk-ant-oat01-...` tokens and failing later during provider auth instead of at entry time.
