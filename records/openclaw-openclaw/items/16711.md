## Summary

Keep this issue open. Current main still has the inline-only Control UI chat composer: no Expand button, no large draft modal, no Apply/Cancel/Send modal workflow, and no modal-specific Enter/newline vs Ctrl/Cmd+Enter send behavior. The prior maintainer/Codex comment also explicitly kept it open, and the related items in the provided context are adjacent composer/session-modal work rather than a canonical replacement.

## Best Possible Solution

Keep this open for first-class Control UI work. The likely implementation belongs in the chat composer surface: add an Expand icon near the existing toolbar controls, introduce large-editor modal state with separate draft ownership, support Close/Cancel, Apply, and Send semantics, implement Enter-as-newline plus Ctrl/Cmd+Enter-to-send behavior in the modal, and cover draft ownership, queued sends, attachments, and dialog behavior in focused UI tests.
