## Summary

Keep this issue open. Current main still has the capped inline Control UI chat composer and does not implement the requested Expand button, large draft modal, Apply/Cancel/Send workflow, or modal keyboard behavior where Enter inserts a newline and Ctrl/Cmd+Enter sends. The prior maintainer/Codex comment also explicitly kept this open, and the related items are adjacent composer/modal topics rather than a canonical replacement.

## Best Possible Solution

Keep this open for first-class Control UI work. A good implementation should add an Expand icon near the composer actions, a large editor modal with separate draft ownership, Close/Cancel, Apply, and Send semantics, Enter-as-newline plus Ctrl/Cmd+Enter-to-send behavior, and focused UI tests covering draft ownership, send/queue behavior, attachments, and existing dialog patterns.
