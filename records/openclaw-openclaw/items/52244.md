## Summary

Keep open. Current main still lacks this PR's OpenRouter-specific wiring for dynamic reasoning models: OpenRouter dynamic model capabilities are used for runtime model resolution, but they are not fed into the model catalog or provider thinking default hooks that drive default reasoning visibility.

## Best Possible Solution

Keep this PR open for maintainer review. The best fix is a narrow OpenRouter-owned rework that feeds cached dynamic OpenRouter reasoning capabilities into the catalog/default thinking path, preferably using current provider thinking-profile/catalog-hook infrastructure without unnecessary public SDK expansion, plus focused regression coverage for a dynamic OpenRouter reasoning model absent from the static catalog.
