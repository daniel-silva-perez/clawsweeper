## Summary

Keep this PR open. Current main still leaves dynamic OpenRouter reasoning defaults unresolved: OpenRouter cached capabilities are used for runtime model resolution, but default thinking/reasoning still depends on catalog metadata, and the reported dynamic model is not in the static catalog. The proposed direction is relevant, but it needs maintainer review because it also expands the public provider-stream SDK surface.

## What This Changes

The PR adds OpenRouter catalog augmentation and default-thinking hooks backed by cached OpenRouter model capabilities, exports an all-cached OpenRouter model accessor through the provider-stream SDK, and reformats one daemon lifecycle test mock.

## Best Possible Solution

Land a narrow OpenRouter-owned fix that makes cached dynamic OpenRouter reasoning capabilities visible to default thinking and reasoning resolution for models absent from the static catalog, while keeping any SDK expansion intentional and documented.
