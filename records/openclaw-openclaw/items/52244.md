## Summary

Keep open. Current main still lacks the OpenRouter catalog/default hooks this PR proposes, and the proposed patch still needs maintainer review because the central reasoning-display path remains gated and the new cache accessor can return an empty catalog on fresh cache startup.

## What This Changes

This PR adds OpenRouter catalog augmentation and default-thinking fallback from cached OpenRouter model capabilities, exports an all-cached OpenRouter capability accessor through the provider-stream SDK surface, and reformats one daemon lifecycle test mock.

## Best Possible Solution

Revise or rebase the PR so dynamic OpenRouter capability metadata is available before thinking and reasoning defaults are resolved, preserve the intended reasoning visibility behavior in the embedded/TUI path, and keep any SDK export narrow, documented, and covered by fresh-cache and cached dynamic-model tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.8

Full review comments:

- **[P1] Preserve reasoning visibility when thinking is inferred:** `extensions/openrouter/index.ts:175-178`
  - body: On current main, reply defaults call `resolveDefaultReasoningLevel()` only while `thinkingActive` is false, and `subscribeEmbeddedPiSession()` only extracts native thinking when `reasoningMode` is `on` or `stream`. Returning `"low"` here for dynamic OpenRouter reasoning models therefore leaves `reasoningMode` at `off`, so the native thinking blocks this PR is trying to surface are still dropped unless the user explicitly sets `/reasoning`.
  - confidence: 0.83
- **[P2] Await the OpenRouter catalog before using it:** `src/agents/pi-embedded-runner/openrouter-model-capabilities.ts:310-311`
  - body: `getAllCachedOpenRouterModels()` only calls `ensureOpenRouterModelCache()`, which triggers an async fetch and immediately returns an empty map when no disk cache exists. Default resolution runs before `resolveModelAsync()` calls `prepareDynamicModel()`, so first use on a fresh cache still produces no supplemental row and the dynamic model defaults can remain off.
  - confidence: 0.78
