## Summary

Keep open: current main does not contain the intended OpenRouter dynamic-model default handling, and the proposed patch still has blocking default-resolution defects that can leave native reasoning hidden and first-use fresh caches unhandled.

## What This Changes

This PR adds OpenRouter catalog augmentation and default-thinking fallback from cached OpenRouter model capabilities, exports an all-cached OpenRouter capability accessor through the provider-stream SDK surface, and reformats one daemon lifecycle test mock.

## Best Possible Solution

Revise or rebase the patch so OpenRouter dynamic capability metadata is available before both thinking and reasoning defaults are resolved, native OpenRouter reasoning blocks remain visible by default when intended, and any SDK/cache surface is narrow, documented, and covered by cached plus fresh-cache dynamic-model tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P1] Preserve reasoning visibility when thinking is inferred:** `extensions/openrouter/index.ts:175-178`
  - body: Returning `"low"` for dynamic OpenRouter reasoning models makes `thinkingActive` true before model reasoning defaults run. Since native thinking is extracted only when `reasoningMode` is `on` or `stream`, the reasoning blocks this PR is meant to surface can still be dropped by default.
  - confidence: 0.83
- **[P2] Await the OpenRouter catalog before using it:** `src/agents/pi-embedded-runner/openrouter-model-capabilities.ts:310-311`
  - body: `getAllCachedOpenRouterModels()` calls the nonblocking cache ensure path, so a fresh machine with no disk cache can return an empty map while the API fetch runs in the background. Default resolution happens before the later dynamic-model warm-up, leaving first use of an uncached dynamic reasoning model at the old defaults.
  - confidence: 0.78
