## Summary

Keep open. Current main still lacks the OpenRouter-owned catalog/default-thinking wiring this PR is trying to add: dynamic OpenRouter capability metadata is used for runtime model resolution, but not fed into the catalog or provider thinking default/profile paths that drive default reasoning visibility. The PR remains useful but needs maintainer review and likely a rebase/rework against current provider-stream-family and thinking-profile infrastructure.

## What This Changes

The PR adds OpenRouter catalog augmentation and default-thinking hooks backed by cached OpenRouter model capabilities, exports an all-cached OpenRouter model accessor through the provider-stream SDK, and reformats one daemon lifecycle test mock.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a narrow OpenRouter-owned update that feeds cached dynamic OpenRouter reasoning capabilities into the catalog and thinking default/profile path on current main, avoids broad public SDK expansion unless the SDK contract is deliberately updated, and adds focused regression coverage for a dynamic OpenRouter reasoning model absent from the static catalog.
