## Summary

Keep open. The reported OpenRouter dynamic-model reasoning bug still appears plausible on current main, but this PR should not merge as-is because its proposed default-thinking fallback can still leave reasoning visibility off, and its all-cached capability accessor can return an empty map on a fresh cache before the async OpenRouter catalog fetch completes.

## What This Changes

This PR adds OpenRouter catalog augmentation and default-thinking fallback from cached OpenRouter model capabilities, exports an all-cached OpenRouter capability accessor through the provider-stream SDK surface, and reformats one daemon lifecycle test mock.

## Best Possible Solution

Revise the PR so dynamic OpenRouter capability metadata is available before thinking and reasoning defaults are resolved, keep reasoning visibility enabled when provider reasoning is inferred, and cover both warm-cache and fresh-cache dynamic-model paths without broadening the SDK surface beyond a reviewed provider-facing contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P1] Preserve reasoning visibility when thinking is inferred:** `extensions/openrouter/index.ts:175-178`
  - body: Returning `"low"` for dynamic OpenRouter reasoning models makes `thinkingActive` true before model reasoning defaults run. Since the embedded subscriber only extracts native thinking when `reasoningMode` is `on` or `stream`, the reasoning blocks this PR is meant to surface can still be dropped by default.
  - confidence: 0.83
- **[P2] Await the OpenRouter catalog before using it:** `src/agents/pi-embedded-runner/openrouter-model-capabilities.ts:310-311`
  - body: `getAllCachedOpenRouterModels()` only calls the nonblocking cache ensure path, so a fresh machine with no disk cache can return an empty map while the API fetch runs in the background. Default resolution happens before the later dynamic-model warm-up, leaving first use of an uncached dynamic reasoning model at the old defaults.
  - confidence: 0.78
