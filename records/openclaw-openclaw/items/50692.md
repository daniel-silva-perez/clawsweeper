## Summary

Current main still does not implement Anthropic native web search, and the linked feature request #49949 remains open. This PR is still a plausible implementation candidate, but the PR-head context has unresolved functional and security concerns around unconditional native search injection, ignored search config, stale registry wiring, and committed log artifacts.

## Best Possible Solution

Keep this PR open for maintainer review or replacement. The best path is provider-owned work in extensions/anthropic that updates the manifest/config/docs/onboarding surfaces, gates native server-tool injection behind global web_search enablement and Anthropic provider selection or auto-selection, passes Anthropic search config into the payload patcher, suppresses managed web_search only while native search is active, tests disabled/different-provider/direct-Anthropic/config-filter cases, removes the log artifacts, and rebases onto the current manifest-first registry.
