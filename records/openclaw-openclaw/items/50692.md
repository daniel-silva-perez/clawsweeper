## Summary

Keep this PR open. Current main still does not implement Anthropic native web search, and the PR remains a plausible implementation candidate for open feature request #49949, but its current head still needs maintainer review and functional/security cleanup before it could merge.

## Best Possible Solution

Keep this PR open for maintainer review or replacement. The best path is provider-owned work in extensions/anthropic that mirrors the existing native-search gating pattern: activate only when web search is enabled and Anthropic native search is selected or auto-selected, suppress the managed web_search tool only while native search is active, pass configured Anthropic search options into the payload patcher, update the manifest/config/docs/onboarding surfaces, and remove the unrelated log artifacts before review continues.

## Risks / Open Questions

- The PR head appears to inject Anthropic native web search for direct Anthropic requests without respecting tools.web.search.enabled or a non-Anthropic selected search provider, which could cause unexpected outbound searches and costs.
- The PR head builds the native search wrapper without passing configured Anthropic search options, so domain filters, maxUses, toolVersion, and userLocation may be ignored.
- The PR includes unrelated root log files, which should be removed and audited for local operational data before any merge.
- The linked feature request #49949 remains open, so closing this PR now would remove an active implementation candidate for unshipped work.
