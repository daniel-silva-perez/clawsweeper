## Summary

Keep this issue open. Current main still has the pair-aware loop-guard gap reported here: the global breaker is scoped to the current tool/signature, ping-pong critical blocking still requires stable result hashes on both sides, and the before-tool-call hook blocks only the current attempted call rather than persisting a block for both tools in an alternating pair.

## Best Possible Solution

Fix the agent tool-loop guard in src/agents/tool-loop-detection.ts and the before-tool-call enforcement path so sustained A/B alternation can be handled as a pair. The likely direction is a pair-scoped threshold or short-lived session-level pair block that blocks both tool signatures when a ping-pong breaker trips, plus regression tests for variable-result A/B alternation and for blocking both sides of the offending pair. Update docs/config only if a new user-facing threshold or behavior is introduced.
