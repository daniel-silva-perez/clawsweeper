## Summary

Keep this PR open. Current main still does not implement the PR's central logging fix: console capture forwards every console call to the file logger without checking `logging.level`, even though the repository already has `isFileLogLevelEnabled()` for that threshold check. The PR remains a plausible implementation candidate, but it should be rebased onto current main and add a regression test for the console-capture path.

## Best Possible Solution

Keep this PR open and ask for a current-main rebase that gates only the file-logger write in `src/logging/console.ts` with `isFileLogLevelEnabled(level)`, while preserving stdout/stderr passthrough. Add a focused regression test in `src/logging/console-capture.test.ts` for `logging.level: warn` suppressing captured `console.log` file output, and either drop or explain the unrelated test mock additions.
