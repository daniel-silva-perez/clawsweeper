Keep open. Current main still uses raw `http2.connect()` for direct APNs delivery, so the PR is not obsolete. ## What This Changes

The PR replaces the direct APNs HTTP/2 connection seam with a managed-proxy-aware CONNECT/TLS helper, exports active proxy state from the proxy lifecycle, adds tunnel/APNs tests plus lint/OpenGrep guardrails, and documents the APNs wrapper.

## Best Possible Solution

Land a maintainer-reviewed version of the APNs session wrapper that keeps the proxy logic at the low-level APNs HTTP/2 seam, preserves the managed-proxy lifecycle boundary, and passes the targeted APNs/tunnel tests plus the changed gate before merge.

## Solution Assessment
