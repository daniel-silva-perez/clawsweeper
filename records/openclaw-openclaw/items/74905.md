## Summary

Keep open. Current main still has the APNs managed-proxy gap this PR targets, but this draft MEMBER-authored PR has protected `security` and `maintainer` labels and latest head still contains blocking correctness defects in the APNs timeout/import path and CONNECT timeout handling.

## What This Changes

The branch routes direct APNs HTTP/2 delivery through an active managed-proxy CONNECT tunnel, adds active proxy URL state, docs, tests, and raw-HTTP2 lint/OpenGrep guardrails.

## Best Possible Solution

Keep the low-level APNs session wrapper direction, restore a valid `http2` constants binding or import `NGHTTP2_CANCEL` directly, replace the tunnel timeout with a real promise rejection/cleanup path, add a regression test for stalled CONNECT handshakes, then have maintainers perform the security-sensitive review before merge.

## Reproduction Assessment

Yes. Current main’s gap is reproducible by source inspection: direct APNs sends call `http2.connect(authority)` while the proxy docs call raw `http2` bypass-prone; the PR-head defects are also reproducible by static inspection at the cited lines.

## Solution Assessment

No, not as currently patched. The active managed-proxy APNs wrapper is the narrow maintainable direction, but the missing binding and ineffective timeout must be fixed before this is a safe solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Restore the http2 constants binding:** `src/infra/push-apns.ts:1-3`
  - body: This module still calls `http2.constants.NGHTTP2_CANCEL` in the request timeout handler, but the branch removes the `node:http2` import. Typecheck should fail and the timeout path would throw, so keep the import or import the constant directly.
  - confidence: 0.97
- **[P2] Reject tunnel timeouts without emitting request errors:** `src/infra/net/http-connect-tunnel.ts:44-48`
  - body: The timeout callback emits `error` on a bare EventEmitter that has no listener, and `https-proxy-agent.connect()` does not consume that event to reject the promise. A stalled CONNECT can therefore throw from the timer instead of rejecting and cleaning up; race the connect attempt against a real timeout and cleanup path.
  - confidence: 0.88
