## Summary

Keep open. Current main implements timing-based inbound debouncing for rapid same-sender messages, but this issue also asks for semantic long-message topic detection/splitting and adaptive aggregation behavior. Those parts are not present in current source, docs, config schema, or plugin hook contracts.

## Best Possible Solution

Keep this issue open, but narrow the remaining scope around semantic preprocessing beyond the shipped debounce layer. The best path is to decide whether OpenClaw should add a gateway-level input preprocessor or a typed plugin/SDK hook that can safely analyze long messages, choose split-vs-unified handling, support adaptive timing, and define media/control-command semantics without reimplementing the existing `messages.inbound` debounce.
