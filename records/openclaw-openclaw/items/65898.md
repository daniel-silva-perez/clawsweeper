## Summary

Keep this PR open. Current main still has a single-phase model idle watchdog and provider-scoped timeout guidance, while this PR proposes a new `agents.defaults.llm.firstTokenTimeoutSeconds` surface that conflicts with the current legacy-migration direction and still needs maintainer API/product review plus QA leak-detection alignment.

## What This Changes

This PR adds a two-phase LLM stream timeout with `agents.defaults.llm.firstTokenTimeoutSeconds`, phase-aware timeout reporting via `llmTimeoutPhase`, generated config schema updates, and embedded-runner/Codex fixture test updates.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to decide whether first-token versus mid-stream timeout belongs under provider-scoped model timeout config or a deliberately reintroduced agent-level LLM config, then adapt the branch to current main, preserve backwards-compatible single-phase behavior when unset, update QA timeout leak detection, and validate the embedded-runner/config/schema surfaces.
