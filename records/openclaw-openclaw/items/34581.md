## Summary

Keep this PR open. Current main still uses strict `express.json()` in the Microsoft Teams webhook, forwards non-413 parser errors, and has no raw-body JSON escape repair or authenticated HTTP 200 fallback for unrecoverable Bot Framework JSON payloads. The PR is a focused implementation candidate, but it changes a security-sensitive webhook parsing boundary and should get Teams maintainer/security review before landing.

## Best Possible Solution

Keep this PR open as the implementation candidate. Rebase/review it on current main, preserve bearer/JWT validation before body parsing and the size limit, repair only truly bare invalid JSON escapes, return HTTP 200 only for intended authenticated Bot Framework parse failures, and add integrated middleware coverage before landing.
