## Summary

Keep this PR open. Current main still strict-parses authenticated Microsoft Teams webhook bodies with `express.json()` and does not include the raw-body parser, bare JSON escape repair helper, or authenticated HTTP 200 fallback for unrecoverable malformed JSON. The PR remains a focused implementation candidate, but it needs maintainer review/rebase against the current auth-before-body-parse hardening and integrated middleware regression coverage.

## Best Possible Solution

Rebase this PR onto current main, keep the auth-before-body-parse and body-size-limit hardening, implement repair only for truly bare invalid JSON escapes, return HTTP 200 only after authenticated unrecoverable Bot Framework JSON, and add integrated webhook regression coverage before landing.

## Risks / Open Questions

- The PR changes an internet-facing Microsoft Teams webhook from strict JSON parsing to raw parsing and repair; review must preserve bearer precheck, Bot Framework JWT validation before body parsing, and `MSTEAMS_WEBHOOK_MAX_BODY_BYTES`.
- Returning HTTP 200 for unrecoverable authenticated JSON intentionally acknowledges and drops that activity, so logging and tests should make the tradeoff explicit.
- The provided PR test mirrors the repair helper directly; integrated middleware coverage is still needed for repair success, unrecoverable authenticated acknowledgement, adapter non-invocation, middleware ordering, and payload-size handling.
