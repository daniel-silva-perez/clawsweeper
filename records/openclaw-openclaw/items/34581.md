## Summary

Keep this PR open. Current main still uses strict `express.json()` in the Microsoft Teams webhook and only handles 413 parser errors, so invalid Bot Framework JSON escape payloads can still fail before the adapter sends its normal HTTP 200 acknowledgement. The PR remains a focused implementation candidate, but it changes an authenticated webhook parsing boundary and should get Teams maintainer/security review before landing.

## Best Possible Solution

Keep this PR open as the implementation candidate. Rebase/review it on current main, preserve bearer/JWT validation before body parsing and the existing size limit, repair only truly bare invalid JSON escapes, return HTTP 200 only for intended authenticated Bot Framework parse failures, and add integrated middleware coverage before landing.
