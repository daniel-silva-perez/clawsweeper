## Summary

Keep this PR open. Current main still strict-parses authenticated Microsoft Teams Bot Framework webhook bodies with `express.json()` and does not include the PR’s raw-body repair path or authenticated malformed-JSON HTTP 200 fallback. The PR remains a plausible focused implementation candidate, but it needs maintainer review against the current auth-before-body-parse hardening plus integrated middleware regression coverage.

## Best Possible Solution

Rebase and review this PR as the implementation candidate: keep auth-before-body-parse and size-limit hardening, repair only truly bare invalid JSON escapes, return HTTP 200 only for authenticated unrecoverable Bot Framework JSON, and add integrated webhook regression tests before landing.

## Risks / Open Questions

- The PR changes an internet-facing Microsoft Teams webhook from strict JSON middleware to raw parsing and repair; review must preserve the existing bearer precheck, Bot Framework JWT validation before body parsing, `MSTEAMS_WEBHOOK_MAX_BODY_BYTES`, and webhook timeout behavior.
- Returning HTTP 200 for unrecoverable authenticated malformed JSON intentionally acknowledges and drops that activity, so logging and tests should make that tradeoff explicit.
- The provided test mirrors the repair helper logic instead of exercising the Express middleware end to end; integrated coverage is still needed for repair success, unrecoverable authenticated acknowledgement, adapter non-invocation, middleware ordering, and payload-size handling.
