## Summary

Keep this PR open. Current main still strict-parses authenticated Microsoft Teams Bot Framework webhook bodies with `express.json()` and does not contain the PR’s raw-body repair middleware or malformed-JSON HTTP 200 fallback, so the PR’s central bug fix is not implemented on main. The PR is focused and still plausible, but it changes an internet-facing webhook parser/acknowledgement path and needs maintainer review plus integrated middleware coverage before landing.

## Best Possible Solution

Keep this PR open as the implementation candidate. Rebase it onto current main, review the webhook security boundary, keep auth-before-body-parse and size-limit behavior intact, repair only truly bare invalid JSON escapes, acknowledge unrecoverable authenticated Bot Framework JSON with HTTP 200 only where intended, and add integrated Express middleware regression tests before landing.
