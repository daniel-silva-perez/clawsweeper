## Summary

Keep open. Current main contains useful latency diagnostics and one partial packaged-loader cache improvement, but the maintainer discussion explicitly says the Docker/VPS chat latency regression is not proven fixed and affected installs still need to retest current main or the next package. The current source also still matches the reported shape: WebChat receives an immediate `chat.send` acknowledgement, then the agent turn continues asynchronously through dispatch, tool inventory, runtime/plugin loading, and embedded-run preparation paths where the reported delay can still live.

## What This Changes

The issue reports severe Docker/WebChat chat-turn latency on OpenClaw v2026.4.26 where OpenClaw takes 30-90 seconds to complete trivial OpenAI-backed replies while direct OpenAI calls from the same container complete in under two seconds.

## Best Possible Solution

Keep this open as the active Docker/WebChat latency tracker. The best path is to retest current main or the next Docker package on a constrained 2 vCPU / 4 GB profile with `OPENCLAW_LOG_LEVEL=trace`, capture the new `[trace:embedded-run]` startup/prep stage summaries, then land a narrow fix for the stage that still dominates pre-model latency while coordinating narrower follow-ups with #73477 and #73836.
