## Summary

Keep open. This is a fresh, actionable Docker/WebChat latency bug with multiple reproductions on v2026.4.26, and current main still contains the same async WebChat dispatch, stuck-session telemetry, ACP/session queue path, and pre-model bootstrap/tool/system-prompt work that match the reported 30-90s completion delay. The related items do not supersede it: #73540 is closed unmerged and the current tools.effective code still defers cold refreshes through setImmediate, #73607 tracks only config-param stripping, and #73477 tracks a narrower CLI/tool-inventory cold-load symptom.

## Best Possible Solution

Keep this issue open as the active user-facing WebChat/Docker latency tracker. Maintainers should collect timing around chat.send acknowledgement, dispatchInboundMessage, lane dequeue, runEmbeddedAttempt start, bootstrap/MCP/LSP/tool inventory, system-prompt assembly, extra-param transport setup, and first model I/O, then land targeted fixes with a regression or perf guard for a small Docker profile. Coordinate narrower work with #73477 and #73607, and either revive or replace the unmerged #73540 approach if tools.effective cold misses are confirmed to contribute.
