## Summary

Keep open. Current main still has no token-free static cron message payload: the cron contract, Gateway schema, agent tool schema, CLI, Control UI, and docs only model `systemEvent` and `agentTurn`, while runtime execution sends main jobs through heartbeat processing and non-main jobs through the isolated agent runner. Related #18160 is a broader open direct-exec design, not a shipped or exact replacement for fixed text-to-channel delivery.

## What This Changes

Add a `staticMessage` cron payload that sends fixed text directly to a configured channel target without running an AI turn.

## Best Possible Solution

Define an additive non-agent cron delivery contract for fixed text-to-channel delivery, then wire it consistently through cron types, normalization, Gateway protocol validation, job validation, runtime dispatch/run logs, CLI add/edit, Control UI, agent cron tool guidance, and public docs. Reuse the existing outbound delivery plumbing where possible, but keep the public contract clear and backwards compatible.
