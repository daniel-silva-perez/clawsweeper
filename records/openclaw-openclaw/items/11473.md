## Summary

Keep this issue open. Current main still has no token-free fixed-text cron payload: the cron type/schema/runtime, CLI, Control UI, agent cron tool, and docs still model jobs as either main-session system events or agent-backed turns. Related #18160 remains a broader open direct-exec design, not a shipped replacement for static text-to-channel delivery.

## What This Changes

Add a cron payload that sends a fixed message directly to a configured channel target without starting an AI/agent turn.

## Best Possible Solution

Keep this issue open until OpenClaw either implements an additive token-free fixed-text cron payload with direct channel delivery, or explicitly folds fixed text-to-channel delivery into a maintained non-agent cron design such as #18160. A complete design should update cron payload types, Gateway schemas/validators, normalization/patch handling, runtime delivery and run logs, CLI, Control UI, agent cron tool guidance, docs, and focused tests.
