## Summary

Keep open. Current main still only supports `systemEvent` and `agentTurn` cron payloads, and the execution paths remain model-mediated for channel-visible fixed text: main-session jobs enqueue a system event/heartbeat, while non-main jobs call the isolated agent runner before delivery. Related #18160 is a broader open direct-exec design, not a shipped replacement for static text-to-channel delivery.

## What This Changes

Add a token-free `staticMessage` cron payload that sends fixed text directly to a configured messaging channel and target without invoking an agent/model.

## Best Possible Solution

Define and implement an additive non-agent cron delivery contract that covers fixed text-to-channel delivery, either as a dedicated `staticMessage` payload or as part of a maintained broader direct cron design. The finished path should keep schema validation, runtime delivery/run logs, CLI, Control UI, agent cron tool guidance, and public docs aligned.
