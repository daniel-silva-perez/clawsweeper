## Summary

Keep open. Current main still lacks a token-free static cron message payload: the public cron payload contract, Gateway schema, runtime validation, execution paths, CLI, Control UI, tests, and docs still model only `systemEvent` and `agentTurn`. The related #18160 tracks a broader direct-exec/non-agent cron design and remains open, so it is not a shipped replacement for fixed text-to-channel delivery.

## What This Changes

The issue requests a `staticMessage` cron payload that sends fixed text directly to a channel target without invoking an agent or spending model tokens.

## Best Possible Solution

Define an additive non-agent cron delivery contract for fixed text-to-channel delivery, then wire it consistently through cron types, normalization, Gateway validation, job validation, runtime dispatch/run logs, CLI add/edit, Control UI, agent cron tool guidance, tests, and public docs. The design should stay backwards compatible and reuse existing outbound delivery plumbing where possible.
