## Summary

Keep this issue open. Current main still has no token-free `staticMessage` cron payload: the core cron type, Gateway schema, runtime execution, CLI, agent cron tool, Control UI, and docs remain built around `systemEvent` for main-session heartbeat work and `agentTurn` for isolated/current/session jobs. Related #18160 is useful broader direct-exec cron design, but it is open and not a shipped superseder for fixed text-to-channel delivery.

## Best Possible Solution

Keep this issue open until OpenClaw either implements an additive token-free static message cron payload or explicitly supersedes it with a broader non-agent cron design that covers fixed text-to-channel delivery. A complete path should update the cron payload types, Gateway TypeBox schemas/validators, normalization/patch handling, runtime delivery/run-log/failure behavior, CLI, Control UI, agent cron tool schema/guidance, docs, and focused tests. Cross-reference #18160, but do not close this narrower request before that work lands or is intentionally folded into a maintained design.
