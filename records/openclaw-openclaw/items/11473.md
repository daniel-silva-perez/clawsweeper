## Summary

Keep open. Current main and the latest shipped tag still only support `systemEvent` and `agentTurn` cron payloads, so the requested token-free fixed text-to-channel cron path is not implemented; related #18160 is broader direct-exec design work and remains open rather than a shipped replacement.

## What This Changes

Add a `staticMessage` cron payload that sends fixed text directly to a configured channel target without invoking an AI turn.

## Best Possible Solution

Define an additive non-agent cron delivery contract for fixed text-to-channel delivery, or deliberately fold that contract into the broader direct-exec cron design, then wire it through cron types, normalization, Gateway validation, runtime dispatch/run logs, delivery plumbing, CLI add/edit, Control UI, agent cron tool guidance, focused tests, and public docs.
