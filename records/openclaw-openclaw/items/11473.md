## Summary

Keep open. Current main still does not implement a token-free static cron message payload: the cron contract, Gateway schema, runtime validation and execution, CLI, Control UI, docs, and agent cron tool still model scheduled work as either `systemEvent` or `agentTurn`. The related direct-exec request is broader, open, and not a shipped replacement for fixed text-to-channel delivery.

## What This Changes

Requests a new `staticMessage` cron payload that sends fixed text directly to a configured channel target without invoking an AI model.

## Best Possible Solution

Define an additive non-agent cron delivery contract for fixed text-to-channel delivery, then wire it through cron types, normalization, Gateway validation, job validation, runtime dispatch and run logs, CLI add/edit, Control UI, agent cron tool guidance, focused tests, and public docs while explicitly deciding how it relates to #18160.
