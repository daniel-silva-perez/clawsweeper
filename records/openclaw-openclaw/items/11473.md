## Summary

Keep open. Current main still has no token-free static cron message path: cron payloads remain limited to `systemEvent` and `agentTurn`, and visible channel delivery is still mediated by the main heartbeat or an isolated agent turn. Related #18160 covers broader direct exec cron work and is still open, so it does not supersede or implement this fixed text-to-channel request.

## What This Changes

The issue requests a `staticMessage` cron payload that sends fixed text directly to a channel target without invoking an agent or spending model tokens.

## Best Possible Solution

Define an additive non-agent cron delivery contract for fixed text-to-channel delivery, or deliberately fold it into the broader direct-exec cron design, then wire it through cron types, normalization, Gateway validation, runtime dispatch/run logs, delivery plumbing, CLI add/edit, Control UI, agent cron tool guidance, focused tests, and public docs.
