## Summary

Keep open. Current main still does not provide a token-free static cron message payload: the public cron contract, runtime validation, execution paths, CLI, Control UI, agent cron tool guidance, and docs still model visible scheduled work as either a main-session system event or an isolated/current/session agent turn. Related #18160 is open and broader, but it is not a shipped fixed-text channel delivery replacement.

## What This Changes

The issue requests a new token-free `staticMessage` cron payload that sends fixed text directly to a channel target without running an agent/model turn.

## Best Possible Solution

Define an additive non-agent cron delivery contract for fixed text-to-channel messages, or explicitly fold fixed text into the broader non-agent cron design, then wire the chosen contract through cron types, normalization, Gateway validation, runtime dispatch/run logs, CLI add/edit, Control UI, agent cron tool guidance, focused tests, and public docs.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The missing behavior is high-confidence by source and docs inspection: current main exposes only `systemEvent` and `agentTurn`, and direct fixed text cannot be scheduled for channel delivery without an agent run.

## Solution Assessment

Unclear. `staticMessage` is a plausible narrow shape for fixed reminders, but maintainers should decide whether it stands alone or becomes part of the broader non-agent cron execution and delivery contract tracked by #18160.
