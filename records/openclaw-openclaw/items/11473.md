## Summary

Keep open. Current main still has no token-free static cron message payload: the cron type contract, Gateway schema, runtime validation/execution, CLI, Control UI, agent cron tool, tests, and docs still expose only `systemEvent` and `agentTurn`. Related #18160 is relevant, but it is a broader open direct-exec design rather than a shipped or exact replacement for fixed text-to-channel delivery.

## What This Changes

The issue requests a `staticMessage` cron payload that sends fixed text directly to a channel target without invoking an agent or model call.

## Best Possible Solution

Define an additive non-agent cron delivery contract for fixed text-to-channel delivery, then wire it consistently through cron types, normalization, Gateway validation, job validation, runtime dispatch/run logs, CLI add/edit, Control UI, agent cron tool guidance, focused tests, and public docs while explicitly deciding its relationship to #18160.
