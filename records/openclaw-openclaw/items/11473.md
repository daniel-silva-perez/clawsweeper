## Summary

Keep this issue open. Current main still has no token-free static cron payload: cron types, Gateway schema, runtime guards, runtime execution, CLI creation, agent cron tool guidance, and docs are still built around `systemEvent` for main-session events and `agentTurn` for isolated/current/session jobs. Related #18160 is open and broader, but it has not shipped and does not fully replace the narrower fixed text-to-channel request.

## Best Possible Solution

Keep this issue open until OpenClaw either implements an additive token-free static message cron payload or explicitly folds fixed text-to-channel delivery into a maintained non-agent cron design such as #18160. A complete implementation should update the cron payload types, Gateway TypeBox schemas/validators, normalization/patch handling, runtime delivery/run-log/failure behavior, CLI, Control UI, agent cron tool schema/guidance, docs, and focused tests.
