## Summary

Keep open. Current main still does not provide a token-free static cron message path: the cron contract, Gateway schema, validation, runtime execution, CLI, Control UI, agent cron tool, and public docs continue to model scheduled work as either `systemEvent` or `agentTurn`. Related #18160 is broader direct-exec cron work and remains open, so it does not implement or supersede fixed text-to-channel delivery.

## What This Changes

Add a `staticMessage` cron payload that sends fixed text directly to a channel target without starting an AI turn.

## Best Possible Solution

Define an additive non-agent cron delivery contract for fixed text-to-channel delivery, or deliberately fold this request into the broader direct-exec cron design, then wire it consistently through cron types, normalization, Gateway validation, job validation, runtime dispatch, run logs, delivery plumbing, CLI add/edit, Control UI, agent tool guidance, focused tests, and public docs.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The missing behavior is high-confidence by source inspection because current schemas, validation, runtime dispatch, CLI/UI, and docs expose only `systemEvent` and `agentTurn`.

## Solution Assessment

Unclear. A `staticMessage` payload is a plausible narrow solution for fixed reminders, but maintainers should decide whether it should stand alone or be part of the broader non-agent cron execution and delivery contract tracked by #18160.
