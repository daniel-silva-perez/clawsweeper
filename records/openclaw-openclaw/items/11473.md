## Summary

Keep this issue open. Current main still has no token-free static cron payload: cron payload types, Gateway validators, runtime dispatch, CLI, Control UI, agent cron tool schema, and docs still model cron payloads as either main-session system events or agent-backed turns. Related #18160 is still open and broader, not a shipped replacement for fixed text-to-channel delivery.

## Best Possible Solution

Keep this issue open until OpenClaw either implements an additive token-free fixed-text cron payload with direct channel delivery, or explicitly folds fixed text-to-channel delivery into a maintained non-agent cron design such as #18160. A complete design should update cron payload types, Gateway schemas/validators, normalization/patch handling, runtime delivery and run logs, CLI, Control UI, agent cron tool guidance, docs, and focused tests.
