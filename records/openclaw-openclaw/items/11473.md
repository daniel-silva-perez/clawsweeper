## Summary

Keep open. Current main still lacks a token-free static cron message payload: cron types, Gateway validation, runtime dispatch, CLI, Control UI, agent tool guidance, and docs all model scheduled user-visible work as either `systemEvent` or `agentTurn`. The related direct-exec tracker is broader and remains open, so it is not a shipped replacement for fixed text-to-channel delivery.

## What This Changes

Add a `staticMessage` cron payload that sends fixed text directly to a channel target without starting an AI turn.

## Best Possible Solution

Define an additive non-agent cron delivery contract for fixed text-to-channel delivery, or deliberately fold this into the broader direct-exec cron design, then wire the chosen contract through cron types, normalization, Gateway validation, job validation, runtime dispatch, run logs, delivery plumbing, CLI add/edit, Control UI, agent tool guidance, focused tests, and public docs.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The missing behavior is high-confidence by source inspection because current schemas, validation, runtime dispatch, CLI/UI, and docs expose only `systemEvent` and `agentTurn`.

## Solution Assessment

Unclear. A `staticMessage` payload is a plausible narrow solution for fixed reminders, but maintainers should decide whether it stands alone or becomes part of the broader non-agent cron execution and delivery contract tracked by #18160.
