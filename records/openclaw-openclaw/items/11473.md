## Summary

Keep open. Current main still has no token-free static cron message payload: the cron payload contract, Gateway schema, validation, runtime dispatch, CLI, Control UI, agent tool guidance, and public docs only support `systemEvent` and `agentTurn`. The issue discussion is useful evidence, including a second user confirming the token-cost problem; related #18160 is broader direct-exec cron work and remains open rather than implementing or superseding fixed text-to-channel delivery.

## What This Changes

Add a `staticMessage` cron payload that sends fixed text directly to a channel target without starting an AI turn.

## Best Possible Solution

Define an additive non-agent cron delivery contract for fixed text-to-channel delivery, or deliberately fold it into the broader direct-exec cron design, then wire it through cron types, Gateway validation, job normalization/validation, runtime dispatch, run logs, delivery plumbing, CLI add/edit, Control UI, agent tool guidance, focused tests, and public docs.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The missing behavior is high-confidence by source inspection because current schemas, validation, runtime dispatch, CLI/UI, and docs expose only `systemEvent` and `agentTurn`.

## Solution Assessment

Unclear. A `staticMessage` payload is a plausible narrow solution for fixed reminders, but maintainers should decide whether it should be a standalone payload or part of the broader non-agent cron execution/delivery contract tracked by #18160.
