## Summary

Keep open. Current main still has trusted per-run/session delivery context plumbing, but it does not have a typed per-agent delivery default that can seed targetless message-tool sends when a programmatic or cross-session run lacks origin routing.

## What This Changes

The issue requests an agent-level default delivery target and channel so cross-session, cron, or webchat-routed agent runs can send messages without repeatedly providing long target IDs.

## Best Possible Solution

Define a typed, documented agents.list[].delivery contract using DeliveryContext-style fields such as channel, to, accountId, and threadId; validate it in config/schema/help/docs, carry it through resolved agent config, and apply it only as the lowest-precedence trusted fallback after explicit message args and per-run or session delivery context.
