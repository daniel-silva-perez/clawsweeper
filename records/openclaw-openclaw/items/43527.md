## Summary

Keep open. Current main still renders a zero `bootstrapPendingCount` as `no bootstrap files`, and the count is still derived only from `BOOTSTRAP.md` presence; open PR #43614 already links this issue and contains the narrow wording/test fix, so the issue should stay open until that PR lands or is replaced.

## What This Changes

The issue asks for `openclaw status --deep` wording to distinguish absence of the one-shot `BOOTSTRAP.md` from missing workspace bootstrap file injection.

## Best Possible Solution

The end state should be status output that describes active bootstrapping or pending `BOOTSTRAP.md` counts, with focused regression coverage, while leaving workspace bootstrap file injection behavior unchanged.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: when `bootstrapPendingCount` is zero, `buildStatusAgentsValue` returns `no bootstrap files`, and that count is computed from `<workspace>/BOOTSTRAP.md` existence.

## Solution Assessment

Yes. A wording-only status helper change plus a focused test is the narrowest maintainable solution; broader bootstrap detection or injection changes are not needed for this bug.
