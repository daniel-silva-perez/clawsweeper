## Summary

Keep open. Current main and the latest release still render `bootstrapPendingCount === 0` as `no bootstrap files`, while the count is only derived from `BOOTSTRAP.md` presence; open PR #43614 already links this issue and contains the narrow wording/test fix, so the issue should remain open until that PR lands or is replaced.

## What This Changes

`openclaw status --deep` should stop saying `no bootstrap files` for mature workspaces where only the one-shot `BOOTSTRAP.md` file is absent.

## Best Possible Solution

The end state should be status output that describes active bootstrapping or pending `BOOTSTRAP.md` counts, with focused regression coverage, while leaving workspace bootstrap file injection behavior unchanged.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: when `bootstrapPendingCount` is zero, `buildStatusAgentsValue` returns `no bootstrap files`, and that count is computed only from `<workspace>/BOOTSTRAP.md` existence.

## Solution Assessment

Yes. A wording-only status helper change plus a focused test is the narrowest maintainable solution; broader bootstrap detection or injection changes are not needed for this bug.
