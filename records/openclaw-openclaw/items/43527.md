## Summary

Keep this open. Current main still renders the misleading `no bootstrap files` wording on the normal/deep status overview path, while the counted signal is only whether any agent workspace still has `BOOTSTRAP.md`; the issue is also paired with open PR #43614, so it should remain open until that PR lands or is superseded.

## What This Changes

The issue asks for `openclaw status --deep` to describe zero pending `BOOTSTRAP.md` files as no active bootstrapping instead of implying that all workspace bootstrap files are absent.

## Best Possible Solution

Land or supersede PR #43614 with a narrow change in `buildStatusAgentsValue` that uses active-bootstrapping language such as `0 bootstrapping` / `N bootstrapping`, plus a regression test for a mature workspace where `BOOTSTRAP.md` is absent but regular workspace bootstrap files still inject normally.
