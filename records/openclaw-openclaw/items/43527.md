## Summary

Keep open. Current main and the latest release still render the misleading `no bootstrap files` wording even though the measured signal is only `BOOTSTRAP.md` presence, and open PR #43614 already targets the narrow wording/test fix.

## What This Changes

The issue asks for `openclaw status --deep` to describe absent `BOOTSTRAP.md` as no active bootstrapping rather than implying all workspace bootstrap files are missing.

## Best Possible Solution

Land or adapt #43614’s narrow status wording and regression test so `bootstrapPendingCount` is described as active bootstrapping or pending `BOOTSTRAP.md`, while leaving workspace bootstrap injection behavior unchanged.

## Reproduction Assessment

Yes. The source path is high-confidence: with `bootstrapPendingCount` equal to zero, `buildStatusAgentsValue` returns `no bootstrap files`, while `bootstrapPendingCount` is computed only from `BOOTSTRAP.md` existence.

## Solution Assessment

Yes. A wording-only status helper change plus focused test coverage is the narrowest maintainable fix because the underlying `BOOTSTRAP.md` pending detection and broader bootstrap injection behavior are already separate.
