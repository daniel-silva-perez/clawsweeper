## Summary

Keep this open. Current main still has the misleading status wording: the overview helper renders `no bootstrap files` when the measured signal is only the count of workspaces with `BOOTSTRAP.md` present. The issue is also paired with open PR #43614, which uses closing syntax and remains the active implementation candidate, so the issue should stay open until that PR is reviewed, landed, or superseded.

## What This Changes

The issue asks `openclaw status --deep` to stop saying `no bootstrap files` when only the one-shot `BOOTSTRAP.md` file is absent from a mature workspace.

## Best Possible Solution

Land or supersede PR #43614 with a narrow `buildStatusAgentsValue` wording change that reports active bootstrapping state, such as `0 bootstrapping`, plus regression coverage for a mature workspace where `BOOTSTRAP.md` is absent but normal workspace bootstrap files still inject normally.
