## Summary

Keep this open. Current main still renders the misleading `no bootstrap files` wording on the normal/deep status overview path, while the underlying count only checks for pending `BOOTSTRAP.md` files; an open linked fix PR remains the right path to resolve it.

## What This Changes

The issue asks for `openclaw status --deep` to describe the absence of one-shot `BOOTSTRAP.md` as no active bootstrapping rather than implying all workspace bootstrap files are missing.

## Best Possible Solution

Land or supersede PR #43614 with a narrow status wording change in `buildStatusAgentsValue`, using active-bootstrapping language such as `0 bootstrapping` / `N bootstrapping`, plus regression coverage for a mature workspace where `BOOTSTRAP.md` is absent but regular workspace bootstrap files still inject normally.
