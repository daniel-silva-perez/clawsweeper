## Summary

Keep open. Current main still has only a global subagent concurrency cap at `agents.defaults.subagents.maxConcurrent`; `agents.list[]` supports per-agent subagent allow/model/require-agent settings, but no typed, validated, documented, or enforced per-target-agent `maxConcurrent` setting. The original TTS/local-model use case remains concrete and was also confirmed by another user.

## Best Possible Solution

Keep this issue open. Add a validated per-target-agent subagent concurrency setting, likely `agents.list[].subagents.maxConcurrent` if maintainers keep the existing namespace pattern; update `AgentConfig`, strict Zod schema, generated schema/help, and docs; then enforce it alongside the global subagent lane cap for native and ACP `sessions_spawn`. Add focused tests proving a capped target serializes while uncapped targets can still run concurrently.
