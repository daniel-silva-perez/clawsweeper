## Summary

Keep this issue open. Current main still lacks the requested schema-driven per-tool security classification contract. OpenClaw has adjacent controls such as owner-only gating, allow/deny tool policy, plugin before_tool_call approval hooks, plugin display risk metadata, provider-level parallel_tool_calls transport settings, and generic abort-signal wrapping, but I did not find a public or core metadata surface for isReadOnly, isDestructive, isConcurrencySafe, interruptBehavior, isOpenWorld, or toAutoClassifierInput.

## Best Possible Solution

Keep this issue open. The best path is a backwards-compatible AgentToolSecurityProps-style metadata contract on AnyAgentTool and the public plugin SDK, propagated through registration, wrappers, adapters, inventories, and runtime definitions. Annotate core and bundled plugin tools incrementally, add conformance tests, and wire policy, UI, scheduling, and interrupt consumers behind conservative fallbacks.

## Risks / Open Questions

- This would be a core and public plugin SDK contract change, so defaults and optionality need to preserve existing third-party plugin behavior.
- Concurrency metadata must stay conservative and must not weaken sandboxing, owner-only gating, allow/deny policy, before_tool_call hooks, approval flows, or provider safety controls.
- toAutoClassifierInput would need redaction rules for secrets, SecretRef values, media references, and sensitive tool arguments before any classifier prompt sees it.
