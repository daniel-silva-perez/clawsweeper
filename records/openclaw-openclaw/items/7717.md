## Summary

Current main still does not implement Discord role-mention triggers. The Discord config/schema only has sender allowlist `roles`, preflight records `mentionedRoles` as generic mention metadata, and mention gating still computes `wasMentioned` from explicit bot mentions, configured text patterns, @everyone/@here, and reply-to-bot state rather than configured role IDs.

## Best Possible Solution

Keep this issue open and implement the feature in the bundled Discord plugin as an additive guild/channel config surface, for example `triggerOnRoles?: string[]`. Wire it through config types, strict schema, generated metadata/help, docs, and preflight. Preflight should compare `message.mentionedRoles` or hydrated `mention_roles` against configured role IDs and treat a match as an explicit wake mention while preserving `requireMention`, `ignoreOtherMentions`, @everyone/@here behavior, reply-to-bot behavior, sender `users`/`roles` allowlists, and routing. Add focused schema tests plus preflight tests for matched role mentions, unmatched role mentions, and role-only messages with `ignoreOtherMentions`.
