## Summary

Keep this PR open. Current main still has the stale cached session runtime path after a transport closes unexpectedly, and this PR is the active implementation candidate. The patch is focused and security-sensitive surfaces are not touched, but the current PR head still has a grounded functional review concern: the new close callback can also fire during intentional cleanup after a failed server startup, which could dispose an otherwise healthy multi-server runtime.

## Best Possible Solution

Keep this PR open. The best path is to update the patch so transport-close eviction only runs for unexpected closes, preserve the SIGKILL eviction regression test, add coverage for multi-server partial startup failure, then run the targeted agent runtime test file and the normal changed gate before landing.
