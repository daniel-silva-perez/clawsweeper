## Summary

Keep this PR open. Current main still lacks transport-close eviction for cached session MCP runtimes, so this is not implemented or superseded. The PR is a focused fix candidate for a real stale-runtime path, but the current patch still needs to address the grounded review concern that intentional close during partial startup can fire the same onclose eviction path. Security review found no unrelated supply-chain-sensitive changes; the diff is limited to the MCP runtime and its test.

## Best Possible Solution

Keep this PR open. The best path is to update the patch so transport-close eviction fires only for unexpected closes, preserve the child-death stale-runtime regression test, add coverage for multi-server partial startup failure, then run the targeted runtime test and changed gate before maintainer review/landing.
