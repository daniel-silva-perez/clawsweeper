## Summary

Keep open. The PR targets a real stale session-scoped MCP runtime bug that current main does not handle, but the proposed unconditional transport-close eviction can also run during intentional startup-failure cleanup and retire a healthy multi-server runtime before merge.

## What This Changes

The PR wires MCP transport close/error callbacks into session MCP runtimes, routes close events through manager-level disposal with logging, and adds a SIGKILL-based regression test for stale stdio MCP child sessions.

## Best Possible Solution

Land a guarded version of this lifecycle fix: keep the pre-connect onclose wiring required by the MCP SDK chaining contract, but mark intentional cleanup before closing a failed session and have the close handler no-op when that session has already been removed. Add focused multi-server partial-startup coverage alongside the child-death regression test.

## Reproduction Assessment

Yes. The original stale-runtime bug is reproducible by connecting a stdio MCP server, killing its child process after catalog materialization, and observing the cached runtime remain on current main; the PR regression is reproducible with a multi-server config where one server fails during startup while another should remain usable.

## Solution Assessment

No, not yet. The callback ordering and manager-level eviction are the right shape, but the patch must guard intentional closes so failed startup cleanup does not dispose the whole in-flight runtime.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Guard intentional transport closes before eviction:** `src/agents/pi-bundle-mcp-runtime.ts:277-280`
  - body: This handler also runs when OpenClaw intentionally closes a transport in the startup-failure cleanup path, because the MCP transport contract says close() invokes onclose and OpenClawStdioClientTransport.close() waits for the child close event. In a multi-server config, one failed server can therefore call onTransportClosed, retire the in-flight session runtime, and leave callers with a disposed runtime or drop healthy server sessions. Mark intentional cleanup before closing and make this handler no-op when the session is already absent.
  - confidence: 0.88
