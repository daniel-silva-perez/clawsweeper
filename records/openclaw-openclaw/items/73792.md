## Summary

Keep this PR open. Current main still lacks transport-close eviction for cached session MCP runtimes, so the PR is not implemented or superseded. The proposed direction addresses a real stale-runtime path, but the current PR head still needs to handle the reviewed intentional-close case where partial startup cleanup can fire the same onclose eviction path. Security review found no unrelated supply-chain-sensitive changes; the PR is limited to the MCP runtime and its colocated test.

## Best Possible Solution

Keep this PR open. The best path is to update the patch so transport-close eviction fires only for unexpected closes, preserve the child-death stale-runtime regression test, add coverage for multi-server partial startup failure, then run the targeted MCP runtime test and the changed gate before maintainer review or landing.
