Keep this issue open. Current main still has the reported behavior: the plugin-bound `inbound_claim` handled path returns `queuedFinal: false`, Telegram still sends the silent-reply fallback when there is no queued final and no delivery, and the regression test still asserts the false result. The issue is also paired with open, unmerged PR #70288 using closing syntax, so it should remain open until that PR or an equivalent fix lands.
