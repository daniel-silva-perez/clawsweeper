## Summary

Keep this issue open. Current main still returns `queuedFinal: false` for the `plugin-bound-handled` inbound-claim path, while Telegram still uses `!queuedFinal && !deliverySummary.delivered` to send the silent-reply fallback. The issue is also paired with open closing PR #70288, so maintainer policy says the issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Review, rebase if needed, and land #70288 or an equivalent maintainer-owned patch that returns `queuedFinal: true` for successful plugin-bound inbound claims and updates the dispatch tests. Track typing indicators or timeout fallback for claimed-but-never-delivered async plugin responses as a separate follow-up if maintainers want that broader contract.
