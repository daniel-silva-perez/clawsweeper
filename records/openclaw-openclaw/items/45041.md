## Summary

Keep open. Current main still lacks the requested opt-in global/per-agent output filter, and the existing sanitizers target tagged reasoning, tool/runtime scaffolding, and flagged reasoning payloads rather than untagged natural-language meta-planning preambles. The related open PR #60551 is still the active implementation candidate for the nearby last-mile sanitizer work, so this should remain with maintainer review instead of being closed or queued as a competing repair.

## What This Changes

Request to add an opt-in global/per-agent output filter that strips configured reasoning or meta-planning patterns from assistant messages before channel delivery.

## Best Possible Solution

Keep this as the product/design tracker until maintainers decide whether the desired end state is a first-class opt-in config filter, a conservative shared sanitizer, or a documented plugin-hook/API pattern. Any implementation should include schema and docs if config is added, plus regression coverage for English and CJK preambles, code/quote pass-through, channel delivery, TTS/media payloads, and hook ordering.
