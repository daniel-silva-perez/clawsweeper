## Summary

Keep open. Current main still lacks the requested configurable outbound filter surface, while existing cleanup only covers structured reasoning visibility, reasoning tags, special tokens, memory/tool scaffolding, and reply directives. The related open PR #60551 overlaps the same plain-text reasoning-preamble leak class, so the remaining path needs maintainer product/API review rather than a competing automated repair.

## What This Changes

Request an optional per-agent or output-level regex/pattern filter that strips leaked reasoning and meta-planning text from assistant messages before channel delivery.

## Best Possible Solution

Keep this as the tracker until maintainers either accept an updated version of #60551, add a documented opt-in config/plugin hook for outbound text filtering, or explicitly document that supported cleanup will remain shared heuristic/provider-specific sanitization rather than user-defined regex filtering.

## Reproduction Assessment

Yes for the requested surface gap: current main has no outputFilters/outputFilter key in config types, strict schema, docs, or delivery normalization. A live multi-agent Discord run is not required to verify that the requested configurable filter surface is absent.

## Solution Assessment

Unclear. A configurable regex filter could solve the user’s examples, but it creates false-positive and config-contract risk; the narrower #60551 heuristic sanitizer is also plausible, so maintainers should choose the API boundary before implementation.
