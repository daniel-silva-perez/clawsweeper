## Summary

Keep open. Current main still lacks the requested opt-in global/per-agent output filter config, and the merged sanitizer paths still do not classify untagged natural-language reasoning/meta-planning preambles before delivery. Open PR #60551 remains the active implementation candidate for the related last-mile preamble cleanup, so this should stay with maintainer review rather than be closed or queued as a competing repair.

## What This Changes

Request an optional global/per-agent output-level regex or pattern filter that strips leaked reasoning and meta-planning prose from assistant messages before channel delivery.

## Best Possible Solution

Keep this as the product/design tracker until maintainers decide whether the desired end state is a first-class opt-in config filter, a conservative shared sanitizer, or a documented plugin-hook pattern. Any implementation should include schema/docs if config is added and regression coverage for English and CJK preambles, code and quote pass-through, channel delivery, TTS/media payloads, and plugin hook ordering.
