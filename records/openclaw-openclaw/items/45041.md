## Summary

Keep open. Current main still lacks the requested configurable output-filter surface, and the existing delivery path only suppresses structured reasoning payloads plus known tagged/internal scaffolding. The open related PR #60551 targets the same leak class, so this should stay with maintainer review rather than being closed or queued as a competing repair.

## What This Changes

The issue requests an optional per-agent output-level regex or pattern filter that strips leaked reasoning and meta-planning text from assistant messages before channel delivery.

## Best Possible Solution

Keep this as the product/design tracker until maintainers either accept a version of #60551, add a first-class opt-in config or plugin/API hook for output filtering, or document why the supported solution is heuristic/shared sanitization instead of user-defined regex filters.

## Reproduction Assessment

Yes for the requested surface gap: current main has no outputFilters/outputFilter key in types, strict schema, docs, or delivery sanitization. I did not live-run the multi-agent Discord scenario, but the static config and sanitizer checks are enough to show the requested feature is still absent.

## Solution Assessment

Unclear. The requested regex filter is a plausible opt-in solution, but #60551 shows a narrower heuristic sanitizer path; maintainers should choose the safest boundary before implementation lands.
