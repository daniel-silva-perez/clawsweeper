## Summary

Keep open. Current main still lacks the requested opt-in output-level pattern/filter config and does not include a merged plain-text reasoning preamble cleaner for untagged meta-planning text; existing protections cover flagged reasoning payloads, known reasoning tags, and fixed scaffolding/tool-call shapes. Open PR #60551 is an active implementation candidate for the related last-mile sanitizer gap, so this should stay open for maintainer review rather than be closed or queued as a competing repair.

## What This Changes

Request an optional global/per-agent output-level regex or pattern filter that strips leaked reasoning and meta-planning text from assistant messages before channel delivery.

## Best Possible Solution

Keep this as the product/design tracker until maintainers decide whether the right end state is a first-class opt-in config filter, a conservative shared sanitizer, or a documented plugin-hook pattern. Any implementation should include schema/docs if config is added and regression coverage for English and CJK preambles, code/quote pass-through, channel delivery, TTS/media payloads, and plugin hook ordering.
