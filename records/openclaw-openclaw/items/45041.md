## Summary

Keep open. Current main still has no global or per-agent outputFilters-style config surface, and the merged sanitizers handle tagged reasoning, explicit isReasoning payloads, tool/runtime scaffolding, and provider artifacts rather than untagged natural-language meta-planning preambles. The related open PR #60551 remains the nearest implementation candidate, so this should stay with maintainer review instead of being closed or queued as a competing repair.

## What This Changes

Request an optional global/per-agent output-level regex or pattern filter that strips leaked reasoning and meta-planning text from assistant messages before channel delivery.

## Best Possible Solution

Keep this as the product/design tracker until maintainers settle on one path: a first-class opt-in config filter, a conservative shared sanitizer, or a documented plugin-hook/API pattern. Any implementation should include schema and docs if config is added, plus regression coverage for English and CJK preambles, code/quote pass-through, delivery, TTS/media payloads, and hook ordering.
