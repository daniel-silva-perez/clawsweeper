## Summary

Keep open. Current main still suppresses commentary-phase assistant text before it can become a Discord partial, block, or final reply, and tests explicitly assert that commentary remains non-visible. The related closed report #55236 covered main-result delivery latency; it does not supersede this issue’s separate early-commentary commit/durability request.

## Best Possible Solution

Keep this issue open for scoped Discord/channel delivery work. The likely fix is to distinguish approved user-visible status commentary from internal commentary/reasoning, materialize that status before tool or restart-sensitive boundaries, route it through the existing Discord draft/dispatcher/restart-deferral machinery, and add regression coverage for early commentary plus restart-sensitive turns. If maintainers decide commentary suppression is intentional product policy, document that explicitly before closing.
