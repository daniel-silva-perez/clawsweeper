## Summary

Keep this issue open. Current main has normalized provider handling in core model resolution, but several Control UI usage/display paths still render raw `modelProvider`/`providerOverride` strings, and related #47620/#47621 covered narrower picker value/deduping bugs rather than this provider-display consistency problem.

## Best Possible Solution

Keep this open and fix the display boundary deliberately: either have the Gateway return canonical provider ids for usage/session display data, or add a shared Control UI provider-display normalizer backed by the same provider alias contract. Cover Usage overview/details/filter/export paths, model selector/agent model option labels where provider ids are shown, clarify the `bailian` alias question, and add regression tests for `z.ai`/`z-ai` and `aws-bedrock` canonical display.
