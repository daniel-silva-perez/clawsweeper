## Summary

Keep this issue open. Current main still has raw-provider paths in the Control UI Usage surface: the gateway returns raw `modelProvider`/`providerOverride`, Usage overview/detail/filter/query code renders and matches those values directly, and transcript usage aggregates group by raw provider strings. The current provider normalization contract exists for `z.ai`/`z-ai` and `bedrock`/`aws-bedrock`, but there is still no current `bailian` alias in source, so that alias should not be added without maintainer confirmation.

## Best Possible Solution

Keep this open and land a narrow fix that canonicalizes provider IDs for Control UI Usage display, provider filters/suggestions, and Usage aggregate grouping using the existing `normalizeProviderId` contract. Preserve stored config/session values, avoid adding a `bailian` alias without maintainer confirmation, and add regression coverage for current aliases such as `z.ai`/`z-ai` and `bedrock`/`aws-bedrock`.
