## Summary

Keep open. Current main has a canonical provider-id normalization contract, but the Usage/Control UI path still derives provider filters, suggestions, badges, display labels, and aggregate keys from raw provider strings, so supported aliases such as `z-ai`/`z.ai` and `aws-bedrock` can still display or group separately from their canonical providers.

## What This Changes

The issue requests consistent Control UI display of normalized model provider names instead of raw alias/config values across sessions, usage, model selection, and agent configuration surfaces.

## Best Possible Solution

Land a focused Usage-surface fix that reuses the existing provider-id canonicalization contract for display labels, filters, suggestions, query matching, and aggregate grouping, while preserving raw stored session/config fields and leaving any new alias contract to maintainer approval.
