Keep this issue open. Current main still has the canonical provider-normalization contract, but the Control UI Usage API, aggregation, display badges, filter options, suggestions, and query matching continue to use raw provider strings, so aliases can still appear or split in Usage views.

The issue requests consistent Control UI display of normalized model provider IDs instead of raw config aliases such as provider aliases for z.ai and Bedrock.

Land a focused Usage-surface fix that reuses the existing provider-id canonicalization contract for display labels, filters, suggestions, query matching, and aggregate grouping, while preserving raw persisted session/config values and leaving any new alias contract to maintainer approval.
