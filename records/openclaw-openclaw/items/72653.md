## Summary

Keep open. The report is recent, concrete, and current main still has browser operation paths where CDP readiness can be green while open/snapshot/navigate hit separate Playwright/CDP action timeouts; the navigate-specific part overlaps an existing open browser timeout issue, but this report is broader and is not clearly fixed or fully superseded.

## What This Changes

The issue asks OpenClaw to stop timing out browser open, snapshot, and navigate operations when the same Chromium CDP endpoint is reachable and works through direct Playwright CDP connection.

## Best Possible Solution

Keep this open until the browser plugin has a focused fix or canonical tracking path for CDP-backed operation timeouts: reproduce against a local/remote CDP endpoint, preserve SSRF guards, propagate operation-specific timeouts with outer request slack, and make failures operation-specific instead of generic control-service reachability errors.
