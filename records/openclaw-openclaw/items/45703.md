## Summary

Keep open. Current main has partial cleanup, but the central Control UI Agents regression remains: the agent picker is still a native select with label-only options, and the primary overview still omits agent identity/avatar metadata even though related model-selector behavior has been improved.

## What This Changes

The issue asks to restore the Control UI Agents section to a richer, readable selector and information display with agent emoji/avatar/metadata instead of the v2 plain dropdown regression.

## Best Possible Solution

Keep the issue open and repair the Control UI Agents page by replacing or augmenting the plain dropdown with an accessible rich agent selector that shows avatar/emoji, display name, ID/default status, and useful metadata, while preserving the current Copy ID, Set Default, refresh, and provider-qualified model picker behavior.

Yes. The source-level render path is high-confidence: rendering `renderAgents` with any non-empty agent list produces `.agents-select` options, and `renderAgentOverview` does not render identity name/avatar in the primary overview grid.
