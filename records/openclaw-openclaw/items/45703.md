## Summary

Keep open. Current main still renders the Control UI agent picker as a plain `<select>` and the main Overview card still omits agent emoji/avatar/identity metadata, so the central regression described in this issue remains. Some sub-symptoms have improved: the old inert action is now Copy ID / Set Default, and provider-qualified model picker behavior is covered separately, but that is not enough to close this issue.

## Best Possible Solution

Keep this issue open and fix the Control UI Agents page by replacing or augmenting the dropdown with an accessible rich agent list/card selector that shows emoji/avatar, display name, default status, and key metadata, and by showing selected-agent identity in the main Overview. Preserve the current provider-qualified model picker behavior and the functional Copy ID / Set Default actions.
