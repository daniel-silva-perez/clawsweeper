## Summary

Keep open. Current main has repaired some adjacent symptoms, but the central Control UI Agents regression remains: the main agent picker is still a native dropdown with label-only options, and the primary overview still does not surface the selected agent's avatar/emoji identity metadata.

## What This Changes

The issue requests restoring the v2 Control UI Agents section to a richer, more usable agent-selection and information display instead of the current plain dropdown-based presentation.

## Best Possible Solution

Keep the issue open and repair the Control UI Agents page with an accessible rich selector or visible desktop list/sidebar/card view that shows avatar/emoji, display name, ID/default status, and useful metadata, while preserving the current Copy ID, Set Default, refresh, panel switching, and provider-qualified model picker behavior.

Yes. Rendering `renderAgents` on current main with any non-empty agent list produces `.agents-select` label-only options, and `renderAgentOverview` does not render the selected agent's identity avatar/emoji in the primary overview grid.

## Solution Assessment

Yes. A focused Control UI Agents-view repair is the narrowest maintainable solution; a broad v2 rollback or core/plugin API change is unnecessary for the remaining observable problem.
