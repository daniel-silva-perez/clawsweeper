## Summary

Keep open. Current main has addressed part of the report around model provider qualification and explicit toolbar actions, but the central Agents view regression remains: agent selection is still a native label-only dropdown and the Overview panel still does not show the selected agent's avatar/emoji identity or richer agent metadata.

## What This Changes

The issue asks for the v2 Control UI Agents section to regain the richer old agent selection and identity/metadata display while keeping controls and model selectors usable.

## Best Possible Solution

Repair the Control UI Agents page with an accessible rich selector, list, or sidebar that shows avatar/emoji, display name, id/default state, and useful metadata, and surface the selected agent's identity in the Overview panel while preserving Copy ID, Set Default, refresh, tabs, and provider-qualified model picker behavior.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: rendering renderAgents with multiple agents still produces .agents-select label-only options, and renderAgentOverview does not render selected-agent identity/avatar/emoji in its primary overview grid.

## Solution Assessment

Yes. A focused Control UI Agents-view repair is the narrowest maintainable path; the model-provider sub-issue is already handled separately, and neither a core contract change nor a full dashboard-v2 rollback is needed for the remaining bug.
