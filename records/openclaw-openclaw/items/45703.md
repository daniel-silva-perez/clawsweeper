## Summary

Keep open. Current main still renders the Control UI Agents page around a single agent `<select>` and does not restore the visible list/card-style selector requested in the report. The overview panel also does not show the agent identity/emoji metadata that the issue calls out. Related open issues #57067 and #45755 cover parts of the same regression, but the remaining work is not fully implemented or clearly superseded enough for an automated close.

## Best Possible Solution

Keep this open unless a maintainer explicitly consolidates it. The best product path is to restore a desktop-visible Agents navigation surface, such as a sidebar/list/card grid with responsive mobile fallback, include agent emoji/identity metadata in both selection and the selected-agent info display, remove or wire any dead actions, and keep regression coverage for agent switching, default-agent persistence, and model selector behavior. If consolidating, link #57067 and #45755 as the canonical follow-ups before closing this broader report.
