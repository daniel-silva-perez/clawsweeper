## Summary

Keep this PR open. Current main still dual-binds loopback to 127.0.0.1 and ::1 when IPv6 is bindable, with no win32 guard and no Windows-specific regression test. The PR remains a narrow, clean implementation candidate for the linked Windows gateway hang; its final patch also addresses the earlier review note about a redundant Windows test. The security pass found only loopback bind-selection code and focused unit tests, with no CI, dependency, lockfile, package, release, generated, or secret-handling changes.

## What This Changes

The PR adds a Windows-only early return in resolveGatewayListenHosts so loopback gateway startup binds only 127.0.0.1 on win32, plus unit tests covering the Windows skip and preserved non-Windows ::1 behavior.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or request a rebase if needed. The best path is the narrow win32 guard in resolveGatewayListenHosts plus focused unit coverage proving Windows skips ::1 while non-Windows loopback behavior remains unchanged; do not close this PR unless this fix or an equivalent one lands on main.
