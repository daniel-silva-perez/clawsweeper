## Summary

Keep open. Current main still has a provider-boundary path where Anthropic-compatible message conversion can produce an empty `messages` array, and this issue is paired with open closing PR #74731, so it should remain open until that PR is reviewed and merged or rejected.

## What This Changes

MiniMax M2.7 intermittently receives an empty Anthropic-compatible chat payload after tool-call chains, returns 400/2013, and leaves the session needing manual repair or model switching.

## Best Possible Solution

Review #74731 as the focused Anthropic-compatible payload-boundary fix, require regression coverage for zero converted messages and valid tool-result continuations, then close this issue after the PR merges. Keep #73929/#73926 separate for the broader blank replay-history prompt guard.

## Solution Assessment
