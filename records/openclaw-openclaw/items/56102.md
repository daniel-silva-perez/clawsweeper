## Summary

Keep this issue open. Current `main` still passes the ACP stdio stream directly into `AgentSideConnection`, and the pinned `@agentclientprotocol/sdk@0.20.0` validates `initialize.params.protocolVersion` as a uint16 integer before OpenClaw's translator can respond. The reported VS Code/Cursor date-string `protocolVersion` remains a plausible current bug, and open PR #56176 is an implementation candidate with closing syntax, so the issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep the issue open and review PR #56176, or an equivalent current-main patch. The fix should add a narrow ACP stream compatibility layer before `AgentSideConnection`, preserve valid numeric ACP `protocolVersion` requests unchanged, and add a regression test for VS Code/Cursor-style `protocolVersion: "2025-11-25"`. Close the issue only after that lands on `main`.
