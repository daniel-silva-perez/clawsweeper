## Summary

Keep this issue open. Current main still passes the ACP stdio stream directly into `AgentSideConnection` with no pre-validation `protocolVersion` compatibility layer, while the pinned ACP SDK validates `initialize.params.protocolVersion` as a numeric uint16. The reported VS Code/Cursor date-string remains a plausible current bug, and open PR #56176 is an implementation candidate with closing syntax, so the issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review PR #56176, or land an equivalent current-main patch. The fix should add a narrow ACP initialize compatibility layer before `AgentSideConnection`, preserve valid numeric `protocolVersion` requests unchanged, handle VS Code/Cursor-style date-string versions, and add regression coverage plus manual IDE proof if feasible. Close the issue only after the fix is merged.
