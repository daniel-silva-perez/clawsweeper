## Summary

Keep this PR open: current main still has the TUI ACP lazy-init gap, but the branch is not merge-ready because the new CLI lazy-init path can send the wrong prompt shape, can start ACP during raw model runs, and the reply-dispatch guard misses first-turn inline-image-only ACP turns.

## What This Changes

This PR exposes agent runtime config through `resolveAgentConfig`, adds lazy ACP session initialization in reply dispatch and CLI agent command execution, and adds resolver/dispatch regression tests.

## Best Possible Solution

Keep the useful lazy ACP initialization direction, but make it narrow and side-effect controlled: initialize only after policy and no-op checks, include inline image attachments in the dispatch decision, keep raw model runs ACP-free, recompute or defer CLI ACP body materialization after lazy init, and add targeted tests for those cases before maintainer review/merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Recompute the CLI body after lazy ACP init:** `src/agents/agent-command.ts:498`
  - body: The CLI prepares `body` while `acpResolution` is still `none`, so it uses `prependInternalEventContext(...)`. After this new block initializes and re-resolves ACP, the ready branch sends that same body to the external ACP runtime. For subagent/task-completion turns this leaks the internal runtime envelope instead of the ACP-safe prompt from `resolveAcpPromptBody(...)`. Move body selection until after lazy init or recompute it when init succeeds.
  - confidence: 0.88
- **[P2] Skip lazy ACP init for raw model runs:** `src/agents/agent-command.ts:472`
  - body: This new condition is not gated by `!isRawModelRun`. `openclaw agent` calls using `modelRun` or `promptMode: "none"` with an ACP-configured agent and missing metadata will now start an external ACP runtime and write metadata, then skip the ACP ready branch and continue down the embedded/raw path. Raw model runs previously avoided ACP side effects, so the lazy-init block should be excluded for them.
  - confidence: 0.8
- **[P2] Include inline images in the empty-turn guard:** `src/auto-reply/reply/dispatch-acp.ts:391`
  - body: The new first-turn guard returns `null` when there is no prompt text and no inbound media, but the existing ACP path also supports `params.images` via `resolveAcpInlineImageAttachments(...)` a few lines later. An ACP-configured session whose first TUI/WebChat turn is image-only will therefore fall back to embedded instead of initializing ACP. Consider resolving attachments before this guard or checking inline images too.
  - confidence: 0.83
