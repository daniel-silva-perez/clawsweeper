## Summary

Current main still lacks the PR's central lazy ACP initialization behavior for TUI/reply-dispatch sessions, so this PR is not obsolete and should remain open for maintainer review.

## Best Possible Solution

Keep this PR open and review it as the active implementation candidate. The best path is a narrow lazy ACP initialization in `src/auto-reply/reply/dispatch-acp.ts` for sessions whose resolved agent has `runtime.type === "acp"`, with normalized ACP fields, config-aware session-to-agent mapping, policy/send-policy/empty-turn gates before side effects, unconditional re-resolve after initialization, focused regression tests for ACP and non-ACP agents, and explicit maintainer judgment on whether the CLI parity changes belong in the same PR.
