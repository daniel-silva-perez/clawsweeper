## Summary

Keep open: current main still has the TUI ACP activation gap, but the PR head is not merge-ready because it adds lazy external-runtime startup in paths that still have side-effect and payload-selection bugs.

## What This Changes

This PR adds lazy ACP session initialization for agents configured with `runtime.type: "acp"` in TUI reply dispatch and CLI agent command execution, exposes runtime/default fields through `resolveAgentConfig`, and adds resolver/dispatch regression tests.

## Best Possible Solution

Keep the lazy ACP initialization direction, but make it side-effect-safe: initialize only for non-empty turns after policy checks, exclude raw model runs, choose the ACP prompt body after any lazy re-resolve, include inline image attachments in first-turn detection, and add targeted CLI plus dispatch tests for those cases.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Recompute the ACP body after lazy init:** `src/agents/agent-command.ts:498`
  - body: After this re-resolve can turn a `none` resolution into `ready`, the later `runTurn` still sends `body` that was computed before ACP was ready. Internal/subagent event turns will use the embedded prompt wrapper instead of `resolveAcpPromptBody(...)`; recompute the body after lazy init or move body selection below this block.
  - confidence: 0.88
- **[P2] Skip lazy ACP init for raw model runs:** `src/agents/agent-command.ts:472`
  - body: This lazy-init condition is not gated by `!isRawModelRun`. For `modelRun` or `promptMode: "none"`, an ACP-configured session with missing metadata can start an external ACP runtime and write metadata, then skip the later ACP-ready branch and continue through the raw/embedded path.
  - confidence: 0.84
- **[P2] Include inline images in the empty-turn guard:** `src/auto-reply/reply/dispatch-acp.ts:391-392`
  - body: The new guard returns `null` when there is no prompt text and no inbound media, but inline chat images arrive through `params.images` and are converted later by `resolveAcpInlineImageAttachments(...)`. A first image-only TUI/WebChat turn can still fall back to embedded instead of initializing ACP.
  - confidence: 0.83
