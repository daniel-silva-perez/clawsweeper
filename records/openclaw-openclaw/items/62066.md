## Summary

Keep open. Current main partially addresses the Slack thread case with `channels.slack.thread.requireExplicitMention`, and it also has a generic SDK seam that can filter implicit mention kinds. However, the broader requested strict mode is not implemented: Discord still only exposes boolean `requireMention`, its preflight path still treats reply-to-bot as an implicit mention, and it does not pass an `allowedImplicitMentionKinds` policy to disable that behavior. The Discord commenter’s bound-channel-session no-mention bypass appears separately covered by current tests, but the strict no-implicit-mentions feature remains missing outside Slack threads.

## Best Possible Solution

Keep this issue open as the canonical strict mention-mode request. The best implementation path is to reuse the existing `resolveInboundMentionDecision` `allowedImplicitMentionKinds` seam, add a documented config surface with clear precedence, and wire it through Discord and the other implicit-reply-capable channels with focused tests for explicit mention, mention pattern, reply-to-bot, and bound-session cases.
