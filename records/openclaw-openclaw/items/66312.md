## Summary

Keep this PR open. Current main at 5828dcdb05aad587bff831467e7fa51f73312ff8 still does not expose `sessionKey` through system-prompt runtime metadata or render it in the `Runtime:` line. The existing `sessionKey="current"` resolver is adjacent, but it is not a model-visible identifier in the Runtime line, so this PR remains a plausible implementation candidate and needs maintainer product/security review.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase it onto current main, resolve the test constant cleanup, and make an explicit product/security choice between exposing raw `sessionKey` in Runtime metadata and documenting/improving the existing `sessionKey="current"` self-history path. Coordinate the Runtime metadata shape with #40044 before landing.
