## Summary

Keep this PR open. Current main at 17811480dae20f65b526454d75e971ae8424ec10 still does not implement the PR’s intended Runtime-line sessionKey exposure: the runtime input/type/rendering path omits sessionKey, and the embedded runner does not pass it into buildSystemPromptParams. Main has an adjacent sessionKey="current" resolver, but sessions_history still lacks model-facing self-history guidance, so that is not a landed replacement for this PR. The diff is narrow, but it needs maintainer product/security review because it would expose a raw session routing key in model-visible prompt text.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase it onto current main, address the minor test constant note, and make a deliberate product/security choice: either expose sessionKey in Runtime metadata, or land/document a safer canonical self-history path such as sessionKey="current" for sessions_history and then close this PR as superseded. Coordinate that decision with #40044.
