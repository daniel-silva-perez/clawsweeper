## Summary

This issue should stay open. Current main documents macOS, Linux, and Windows/WSL2 install paths, plus an Android companion node app, but it does not document or prove a supported Termux gateway install/run path. The existing Termux scripts are phone-side auth/SSH helpers, not OpenClaw-on-Termux availability. The issue is recent, author is not a maintainer, has no protected labels, and is not old enough for stale-insufficient-info cleanup.

## Best Possible Solution

Keep open and turn it into a concrete Termux support decision. The best path is to test current npm/pnpm install, onboarding, and foreground gateway startup under Termux; then either document a supported no-daemon Termux runbook with known limitations, or explicitly document Termux as unsupported and point phone users to the Android node/VPS/SSH-widget workflows. If support is added, update install/platform docs and add onboarding/daemon tests for the Termux or no-systemd path.
