## Summary

Keep this open: current main does not implement or document Termux as a supported OpenClaw Gateway host. The supported install path is macOS, Linux, or Windows/WSL2, while Android is documented as a companion node that connects to a Gateway running elsewhere, so this needs maintainer platform triage rather than cleanup closure.

## What This Changes

Request to make OpenClaw usable from Termux on Android and document the required commands if it is already supported.

## Best Possible Solution

Keep the issue open until maintainers decide the intended Termux support level: either a documented no-daemon/local-prefix Gateway path that is reproduced in Termux, or explicit docs saying Android/Termux is not a supported Gateway host and should connect as an Android node to a Gateway elsewhere.
