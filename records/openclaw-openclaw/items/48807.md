## Summary

Keep this PR open. Current main still lacks the PR's central drain-before-abort behavior for health-monitor restarts: the monitor records restart bookkeeping and calls `stopChannel`/`startChannel` directly, while `stopChannel` aborts before waiting for the channel task to settle. The PR remains a focused implementation candidate, not obsolete cleanup.

## Best Possible Solution

Keep this PR open as the implementation candidate. Rebase/review the latest head against current main, verify the drain-before-abort behavior, the 60-second cooldown floor, fresh timing resolution, and all post-drain rechecks in `src/gateway/channel-health-monitor.ts`, keep focused regression tests, and coordinate with #45712 before landing or requesting changes.
