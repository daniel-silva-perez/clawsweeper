## Summary

Keep this PR open. Current main still does not implement the PR's central behavior: the health monitor immediately restarts unhealthy running accounts through stopChannel/startChannel, the stop path aborts before waiting for graceful channel shutdown, cooldown is still cycle-based with no 60-second floor, and timing is resolved once from startup config. The live PR is still open at head e316f68fd6e81c6e21e70c30a0432e662199b5bd and remains a focused implementation candidate limited to the health monitor and its tests.

Keep this PR open as the implementation candidate.
