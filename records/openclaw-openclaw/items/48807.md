## Summary

Keep this PR open. Current main still does not implement the PR's central drain-before-abort behavior: the health monitor restarts unhealthy running accounts directly, the channel stop path aborts before graceful shutdown waiting, and cooldown remains cycle-based without a 60-second floor. The PR remains a focused implementation candidate; its diff is limited to the health monitor and tests with no CI, dependency, lockfile, release, or secret-handling changes observed in the security pass.

## Best Possible Solution

Keep this PR open as the implementation candidate. Rebase and review the latest head against current main, then verify the drain-before-abort behavior, 60-second cooldown floor, fresh timing resolution, and post-drain stopped/recovered/removed/disabled rechecks before landing or requesting focused changes, with coordination against #45712.
