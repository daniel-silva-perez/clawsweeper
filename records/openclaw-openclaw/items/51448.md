## Summary

Keep this PR open. Current main still does not forward configured `tool_choice`/`toolChoice` from agent model params into embedded runner stream options, so the user-visible bug is still real. The PR remains useful, but it is stale against current test layout and should be updated to preserve alias precedence, validate values, and fit the current dependency option types before merge.

## Best Possible Solution

Keep this PR open and ask for an update against current main. The best fix is to canonicalize `tool_choice`/`toolChoice` with the same source-precedence model used for `parallel_tool_calls`, preserve runtime and per-agent overrides over inherited config, validate or warn on invalid values, forward the resolved value into stream options using a type that actually includes the provider-specific option, and add regression coverage in the current extra-params test files for both configured forwarding and cross-alias override precedence.
