## Summary

Keep this PR open. Current main still does not forward configured `tool_choice`/`toolChoice` from agent model params into embedded runner stream options. The PR is directionally useful and security-narrow, but should be updated so alias precedence matches the existing `parallel_tool_calls` handling before merge.

## Best Possible Solution

Keep this PR open and update it against current main. The best fix is to canonicalize `tool_choice`/`toolChoice` with the same source-precedence model used for `parallel_tool_calls`, preserve runtime and per-agent overrides over inherited config, forward the resolved value into `streamParams.toolChoice`, and add regression coverage for both configured forwarding and cross-alias override precedence.
