## Summary

Keep this PR open. Current main still does not forward configured `tool_choice`/`toolChoice` from agent model params into embedded runner stream options, and the PR remains useful but needs an update for alias precedence and validation before merge.

## Best Possible Solution

Keep this PR open and update it against current main. The best fix is to canonicalize `tool_choice`/`toolChoice` with the same source-precedence model used for `parallel_tool_calls`, preserve runtime and per-agent overrides over inherited config, forward the resolved value into `streamParams.toolChoice`, validate or warn on invalid values, and add regression coverage for configured forwarding plus cross-alias override precedence.
