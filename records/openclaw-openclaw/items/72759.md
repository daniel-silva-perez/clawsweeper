## Summary

Keep open for maintainer review. The branch now uses the valid `off` mode rather than the invalid `final` value, but current `main` still intentionally documents and tests Telegram's unset preview-streaming default as `partial`, and this PR changes that user-visible default without updating the contract tests or docs.

## What This Changes

The PR changes Telegram's preview streaming fallback in `extensions/telegram/src/preview-streaming.ts` from `partial` to `off` when no stream mode is configured.

## Best Possible Solution

Maintainers should choose the desired Telegram default explicitly: either keep `partial` as the documented default now that current main has in-place preview finalization coverage, or accept final-only-by-default behavior and update the resolver tests, Telegram docs, and user-facing changelog together.

## Reproduction Assessment

Yes for the PR impact: current source and tests show unset Telegram streaming resolves to `partial`, while the PR diff would make it resolve to `off`. The original ghost-message report has external reproduction detail in #72706, but I did not perform a live Telegram run for this read-only review.

## Solution Assessment

Unclear. `off` is a valid way to suppress previews, but it disables a documented default feature; the narrower maintainable path depends on whether maintainers prefer preserving fixed partial previews or changing Telegram to final-only delivery by default.
