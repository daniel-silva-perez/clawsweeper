## Summary

Current main still lacks the requested SSRF middle policy for browser private-network access. The implemented behavior remains static allow/block via `dangerouslyAllowPrivateNetwork`, exact hostname exceptions, and hostname allowlist patterns; there is no browser confirmation mode, CIDR/port allowlist/blocklist contract, hard metadata override under broad opt-in, audit/rate-limit/timeout flow, or `openclaw ssrf` CLI surface.

## Best Possible Solution

Keep this issue open as the canonical browser private-network confirmation/security design request. Maintainers should first settle the policy contract and precedence rules for deny/allow/confirm, CIDR and port allowlists/blocklists, non-overridable cloud metadata blocks, timeout/rate-limit/audit behavior, redirect and DNS-rebinding handling, browser proxy/CDP behavior, and whether `openclaw ssrf` temporary controls are an accepted CLI surface. Then implement the contract across the shared SSRF policy, browser config/schema, browser navigation/control paths, UI approval flow, public docs, and focused security tests.
