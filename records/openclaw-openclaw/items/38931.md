## Summary

Current main still does not implement the requested browser SSRF confirmation mode, CIDR/port allowlist/blocklist policy, hard metadata blocking under broad opt-in, audit/rate-limit/timeout behavior, or temporary `openclaw ssrf` controls. The existing behavior remains static: strict by default, broad private-network opt-in, and hostname exceptions.

## Best Possible Solution

Keep this issue open as the canonical browser private-network confirmation/security design request. Maintainers should first settle the policy contract and precedence rules for deny/allow/confirm, CIDR and port allowlists/blocklists, non-overridable cloud metadata blocks, timeout/rate-limit/audit behavior, redirect and DNS-rebinding handling, browser proxy/CDP behavior, and whether `openclaw ssrf` temporary controls are an accepted CLI surface. Then implement the contract across `src/infra/net/ssrf.ts`, browser config/schema, `extensions/browser/src/browser/*` navigation/control paths, UI approval plumbing, public docs, and focused security tests.
