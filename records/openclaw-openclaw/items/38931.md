## Summary

Current main still implements static SSRF behavior only: default blocking, broad private-network opt-in, and hostname exceptions. It does not implement the requested confirmation mode, CIDR/port allowlist and blocklist policy, non-overridable metadata blocking under broad allow, timeout/rate-limit/audit behavior, or temporary `openclaw ssrf` CLI controls.

## Best Possible Solution

Keep this issue open as the canonical browser private-network confirmation/security design request. A complete solution should extend the browser SSRF config contract, define confirmation and allow/blocklist precedence, decide whether cloud metadata endpoints are hard-blocked in every mode, add timeout/rate-limit/audit behavior, cover redirects, DNS rebinding, browser proxy routing, CDP reachability, and related guarded fetches in tests, update public docs, and add `openclaw ssrf` temporary controls only if maintainers accept that CLI surface.
