## Summary

Keep this PR open: the UX change is still not implemented on current main, but the proposed branch is not correct as-is because it hard-codes an `allow-always` fallback command even when current approval policy filters that decision out for effective `ask=always`.

## What This Changes

The PR changes exec-approval forwarded fallback notifications and tests to render ready-to-copy `/approve <id> <decision>` command lines, with backtick wrapping only for markdown-capable channels.

## Best Possible Solution

Build the forwarded fallback command lines from the same allowed-decision source used by approval action descriptors, apply markdown wrapping only for markdown-capable channels, and keep regression coverage for markdown, plain-text, and `ask=always` fallback text.
