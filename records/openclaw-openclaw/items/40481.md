## Summary

Current main still has the generic exec-approval fallback text with `/approve <id>` instead of concrete copyable commands. The PR is not obsolete, but the submitted diff still needs rework because it hard-codes `allow-always` while current main intentionally omits that decision when effective `ask` is `always`. The supplied final PR file list only touches source and tests, so no CI, dependency, workflow, lockfile, or supply-chain regression is visible from the PR diff context.

## Best Possible Solution

Keep this PR open and rework it against current main. The best fix is to update the generic exec approval fallback in `src/infra/exec-approval-forwarder.ts` so it renders concrete per-decision `/approve <actual-id> <decision>` lines from the existing allowed-decision path, uses markdown wrapping only for markdown-capable channels, and preserves `ask=always` filtering. Add focused coverage in `src/infra/exec-approval-forwarder.test.ts` for markdown fallback, non-markdown fallback, and `ask=always` without `allow-always`, then review and land the PR if the diff stays scoped.
