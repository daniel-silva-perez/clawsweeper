## Summary

Keep open. Current main still sends the generic exec-approval fallback text, so the PR is not obsolete, but the submitted patch is not safe to merge as-is because it advertises `allow-always` even when the current `ask=always` policy deliberately removes that decision.

## What This Changes

This PR changes exec-approval forwarded fallback notifications to show full `/approve <id> <decision>` command lines, with markdown wrapping for markdown-capable channels and focused forwarder tests.

## Best Possible Solution

Generate the fallback command lines from the existing allowed-decision path, preferably by reusing or mirroring `buildExecApprovalActionDescriptors`; apply markdown formatting only when the target channel supports it, handle arbitrary non-empty approval ids safely, and add tests for markdown, plain-text, and `ask=always` variants.
