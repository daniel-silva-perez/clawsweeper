## Summary

Keep open: current main still renders the generic fallback hint, so the requested UX is not implemented there. The PR is useful but still has a correctness issue because it can advertise an `allow-always` command for requests whose effective policy filters that decision out, and the resolver rejects that decision.

## What This Changes

The PR changes forwarded exec-approval fallback notifications and tests to show ready-to-copy `/approve <id> <decision>` command lines, with backtick wrapping only for markdown-capable channels.

## Best Possible Solution

Generate forwarded fallback command lines from the same allowed-decision descriptors used by approval actions, apply markdown wrapping only for markdown-capable channels, and keep coverage for markdown, plain-text, and `ask=always` fallback text.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Preserve allowed decision filtering:** `src/infra/exec-approval-forwarder.ts:256`

That advertised command is rejected by the resolver, so build these lines from the allowed-decision descriptors and cover the `ask=always` fallback case.
