## Summary

Keep open. Current main still shows the generic `/approve <id>` placeholder in forwarded fallback approval messages, so the PR is not obsolete; however, the PR branch hard-codes all three decisions and would advertise `allow-always` even when the effective policy intentionally removes that option.

## What This Changes

The PR changes exec-approval forwarded fallback notifications and tests to render ready-to-copy `/approve <id> <decision>` command lines, with markdown wrapping only for markdown-capable channels.

## Best Possible Solution

Generate forwarded fallback command lines from the same allowed-decision source used by the existing approval action descriptors, apply markdown formatting only when the target channel supports it, and add regression coverage for markdown, plain-text, and `ask=always` fallback text.
