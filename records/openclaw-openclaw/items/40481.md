## Summary

Keep open. Current main still uses placeholder fallback text for chat approval commands, so the PR is not obsolete; however, the branch still hard-codes `allow-always` even when effective policy removes that decision, which would advertise a command the resolver rejects.

## What This Changes

The PR changes exec-approval forwarded fallback notifications and tests to render ready-to-copy `/approve <id> <decision>` command lines, with markdown wrapping only for markdown-capable channels.

## Best Possible Solution

Generate forwarded fallback command lines from the same allowed-decision source used by existing approval action descriptors, apply markdown formatting only when the target channel supports it, and keep regression coverage for markdown, plain-text, and `ask=always` fallback text.
