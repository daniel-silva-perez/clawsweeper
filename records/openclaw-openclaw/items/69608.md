## Summary

Keep open. Current main still lacks the PR's shared daily-memory/session-summary helper layer and several affected runtime paths remain canonical-file or local-rule based; the PR is also an open XL implementation touching gateway startup, session-memory hooks, memory-core, plugin-facing runtime files, docs, and scripts, so the right next step is maintainer review rather than cleanup closure.

## What This Changes

This PR adds shared daily-memory/session-summary parsing, probing, provenance, startup-selection, and backfill helpers, exports them through the memory host runtime surface, and rewires startup context, session-memory persistence, doctor, memory-core dreaming/REM/short-term promotion, docs, and scripts to use the shared rules.

## Best Possible Solution

Land, split, or replace this with a maintainer-approved implementation that centralizes daily-memory/session-summary classification and provenance, applies it consistently across startup/session-memory/doctor/dreaming/REM/short-term flows, and keeps plugin-facing file helpers narrowly scoped with explicit workspace-boundary guarantees and regression coverage.
