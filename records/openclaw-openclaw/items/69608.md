## Summary

Keep open. Current main does not implement the shared daily-memory/session-summary model, and the PR still carries useful work, but the current head has a blocking path-boundary defect in deleted-summary classification that should be fixed before merge.

## What This Changes

The PR adds shared daily-memory/session-summary helpers, exposes them through the memory host runtime surface, and rewires startup context, session-memory, doctor, dreaming/REM/short-term flows, bootstrap handling, docs, and scripts to use those rules.

## Best Possible Solution

Land, split, or replace this with a maintainer-approved implementation that centralizes daily-memory/session-summary classification and provenance, but first constrain remembered-summary fallback to normalized in-workspace aliases or a deliberately documented migrated-path path with regression coverage for foreign absolute basename collisions.

## Reproduction Assessment

Yes. A focused reproduction is to call isSessionSummaryDailyMemoryPath with a workspace whose recent-daily index remembers 2026-04-19-vendor-pitch.md as a session summary, then pass /old/workspace/memory/2026-04-19-vendor-pitch.md with no in-workspace probe candidates; the current PR-head code reaches the raw-path remembered fallback.

## Solution Assessment

No. The overall shared-helper direction is maintainable, but the current diff is not the best safe implementation until the remembered-summary fallback stops trusting arbitrary foreign absolute path basenames.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Constrain remembered-summary fallback to in-workspace aliases:** `src/memory-host-sdk/host/daily-session-summary-probes.ts:576`
  - body: When a persisted short-term entry points at an absolute path outside the workspace, buildSessionSummaryDailyMemoryProbePaths returns no candidates, but this fallback still extracts a memory filename from the raw input and checks the current workspace's recent-daily index. If the current workspace has a remembered session-summary file with the same basename, the foreign entry is treated as bookkeeping and removed or skipped. Only trust remembered provenance from normalized in-workspace probe candidates, or add an explicit migrated-path check that cannot match unrelated workspaces.
  - confidence: 0.89

## Security Review

Status: needs_attention

Summary: The PR introduces path-boundary classification logic that can still mix foreign absolute paths with current-workspace remembered provenance.

Concerns:

- **[medium] Foreign path can inherit local summary provenance:** `src/memory-host-sdk/host/daily-session-summary-probes.ts:576`
  - body: A foreign absolute memory path can be classified from the local .recent-daily-files index when basenames collide, which risks data-integrity errors in memory cleanup/promotion decisions across migrated or mixed workspaces.
  - confidence: 0.86
