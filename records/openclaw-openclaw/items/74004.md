## Summary

Keep open. Current main and the latest release still lack a post-refresh write-back path from the OAuth manager to Claude CLI storage, so this PR is not obsolete; however, the proposed patch writes OAuth secrets to an external credential store and needs maintainer/security follow-up before merge.

## What This Changes

The PR adds an optional OAuth manager post-refresh hook and wires it to write refreshed `anthropic:claude-cli` credentials back to Claude CLI file or Keychain storage via the existing Claude CLI credential writer.

## Best Possible Solution

If bidirectional Claude CLI sync is intended, keep the hook narrow and provider-scoped: write back only validated `anthropic:claude-cli` / `claude-cli` OAuth credentials after a successful refresh, redact or omit write-back error details, add focused tests around success/no-op/failure paths, and update docs or changelog to reflect the supported boundary. If maintainers prefer the token-sink model, document that decision and leave external CLI stores read-only.

## Reproduction Assessment

Yes for the code path: an expired `anthropic:claude-cli` OAuth profile on current main can refresh into OpenClaw’s store, but source inspection shows no hook capable of writing the rotated token back to Claude CLI storage. I did not run a live OAuth race because this review is read-only.

## Solution Assessment

No, not as written. The direction may be the narrow product fix, but the patch should first guard on both profile id and provider, sanitize write-back logging, rebase onto current main, and add regression tests around the new external secret write boundary.

  - confidence: 0.88
