## Summary

Keep this PR open. Current main still lacks the PR's stale already-executed slot guard in the cron runnable path, and the PR remains a focused candidate fix for the duplicate restart execution behavior described by the linked report and confirmed in discussion.

## Best Possible Solution

Review and land this PR, or port the same narrow strict stale-slot guard into current main's `isRunnableJob` startup catch-up path with the regression test. The implementation should skip already-executed stale persisted slots while still allowing genuine missed cron slots and same-millisecond retry slots to run.
