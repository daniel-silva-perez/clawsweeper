## Summary

Keep open. Current main still lacks the PR's already-executed stale-slot guard in the cron runnable path, and the PR remains a focused implementation candidate for the duplicate gateway-restart execution bug confirmed in the discussion.

## Best Possible Solution

Review and land this PR, or port the same narrow strict stale-slot guard plus regression test onto current main. The implementation should skip already-executed stale persisted slots while preserving genuine missed cron slots and same-millisecond retry slots.
