## Summary

Keep open. Current main still uses the fixed 16k/32k context-window guard that this issue asks to replace, and PR #72958 is an open, unmerged implementation candidate that explicitly targets the same behavior.

## Best Possible Solution

Keep this issue open while PR #72958 is reviewed. The best path is to land or revise a focused core change that derives warning and block thresholds from the effective context window with safe floors, preserves agents.defaults.contextTokens cap semantics, removes the fixed embedded-runner guard overrides, and updates the guard tests plus local-model docs. If PR #72958 merges, this issue can close as implemented by that merge; if it is rejected, keep this open for a revised implementation or explicit maintainer product decision.
