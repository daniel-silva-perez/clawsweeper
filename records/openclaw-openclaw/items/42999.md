## Summary

Keep open. Current main still implements the fixed 16k/32k context-window guard that this issue asks to replace, and the provided GitHub context shows open, unmerged PR #72958 as a closing-syntax implementation candidate for the same behavior.

## Best Possible Solution

Keep this issue open while PR #72958 is reviewed. The best path is to land or revise a focused core change that derives warning/block thresholds from the effective context window with safe floors, preserves agents.defaults.contextTokens cap semantics, removes the fixed embedded-runner guard overrides, and updates the guard tests plus local-model docs. If that PR merges, this issue can close as implemented by the merge; if it is rejected, keep this open for a revised implementation or explicit maintainer product decision.
