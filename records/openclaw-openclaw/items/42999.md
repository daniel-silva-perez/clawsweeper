## Summary

Current main still has the static 16k/32k context-window guard behavior that this issue asks to replace. There is also an open, unmerged implementation candidate, #72958, whose body uses closing syntax for this issue, so repository policy says to keep this issue open while that PR is reviewed.

## Best Possible Solution

Keep this issue open while #72958 is reviewed. The concrete path is to land or revise a focused core change that derives guard thresholds from the effective context window with safe floors, stops passing fixed 16k/32k overrides from the embedded runner, preserves `agents.defaults.contextTokens` cap semantics, and updates guard tests plus public local-model docs. If #72958 merges, this can close as implemented by that merge; if it is rejected, this should remain open for a revised implementation or explicit maintainer product decision.
