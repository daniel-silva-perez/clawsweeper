## Summary

Keep this PR open. Current main still has the timeout propagation gaps the PR targets, but the PR is not safe to close as implemented or merge as-is because the proposed patch conflicts with the current gateway timeout contract and has unresolved runtime-budget semantics.

## Best Possible Solution

Keep the PR open for revision or replacement. A mergeable fix should forward numeric agent timeouts only where the caller is actually waiting, preserve the existing fire-and-forget runtime contract unless maintainers intentionally change it, account for session-lock wait within the effective run budget, align A2A target and wait budgets, and add regression coverage for gateway validation, `timeoutSeconds: 0`, A2A helper waits, and contended session locks.

## Risks / Open Questions

- Merging the PR as-is would send `timeout` as a string even though the gateway contract requires an integer.
- The proposed fire-and-forget change would cap target runs at 30 seconds instead of preserving the normal configurable agent runtime budget when `timeoutSeconds` is 0.
- Passing the full run timeout into session-lock acquisition can let a contended run spend the caller's whole budget waiting for the lock and then still begin model execution.
- The proposed A2A helper change can grant target runs a larger budget than the helper waits for when `timeoutMs` exceeds the existing 60-second wait cap.
