## Summary

Keep this PR open. Current main still does not implement the ACP token-only/device-identity opt-out: the ACP server constructs `GatewayClient` without `deviceIdentity: null`, the CLI and standalone parser expose no `--no-device-identity`/`--skip-device-identity` option, and the ACP docs list no such flag. The linked feature issue is closed, but this unmerged PR remains a plausible implementation candidate rather than obsolete work.

## Best Possible Solution

Keep the PR open and either finish it or replace it with an equivalent maintainer patch. The final change should add a documented ACP CLI flag, pass `deviceIdentity: null` only when explicitly requested, preserve the current `undefined` auto-load behavior by default, and include focused startup tests for both paths plus the public docs entry.
