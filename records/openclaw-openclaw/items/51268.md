## Summary

Keep this PR open. Current main still lacks an ACP device-identity suppression option, and the existing GatewayClient null contract plus gateway role policy support the PR's intended token-only path. The branch remains useful, but it should not merge until the public flag name and ACP docs are aligned and the auth-sensitive behavior receives normal maintainer review.

## What This Changes

The PR adds an ACP server option and CLI flag that pass `deviceIdentity: null` to `GatewayClient`, plus startup tests for null versus undefined device-identity handling.

## Best Possible Solution

Finalize the public ACP flag name, document it in the ACP CLI reference, keep the server option, Commander registration, standalone parser/help, and null-versus-undefined tests aligned, then merge this PR or an equivalent maintainer patch after gateway-auth review.

## Reproduction Assessment

Yes. Current-main code inspection shows ACP omits `deviceIdentity`, and `GatewayClient` turns an omitted value into `loadOrCreateDeviceIdentity()`; a focused branch test can assert `deviceIdentity: null` for the new flag and `undefined` when absent.

## Solution Assessment

Yes for the core approach: using `deviceIdentity: null` is the narrowest maintainable way to use the existing GatewayClient and gateway policy contracts. The submitted patch is not complete until docs and final flag naming are aligned.
