## Summary

Keep open. Current main still leaves the eager lazy-CLI fallback racy and only partially addresses parent option preservation, so this PR has real remaining value; however, the branch needs maintainer/author follow-up before merge because it changes a public Plugin SDK helper contract without updating the generated API baseline and must be reconciled with newer main's parent-option fallback.

## What This Changes

The PR reparses lazy CLI actions from the root command path, awaits eager command-group registration through core and browser CLI wiring, and updates browser/CLI tests plus changelog entries.

## Best Possible Solution

Rebase onto current main, keep the root-reparse/full-command-path behavior while preserving current main's explicit parent-option fallback for empty-rawArgs cases, update the generated Plugin SDK API baseline if the async helper contract is intentional, and land after targeted CLI/browser tests plus the Plugin SDK API check pass.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Update the Plugin SDK API baseline:** `src/cli/program/register-command-groups.ts:70`
  - body: `registerCommandGroups` is exported through `openclaw/plugin-sdk/cli-runtime`, so changing it from a synchronous helper to an `async` function changes the generated declaration from `void` to `Promise<void>`. The patch does not update `docs/.generated/plugin-sdk-api-baseline.sha256`, so `pnpm plugin-sdk:api:check` should fail until the intentional public API change is regenerated and committed.
  - confidence: 0.9
- **[P3] Preserve parent options in the empty-rawArgs fallback:** `src/cli/program/action-reparse.ts:29-30`
  - body: The new root fallback builds the nested command path, but when `root.rawArgs` is empty it only appends leaf action args. Current main has fallback coverage that serializes explicit parent options, so a nested programmatic reparse such as `browser --json open` would drop `--json`/`--browser-profile` if this version replaces that logic during rebase.
  - confidence: 0.73
