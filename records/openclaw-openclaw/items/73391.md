## Summary

Keep this PR open. Current main still has both lazy CLI runtime behaviors the PR targets: nested lazy reparse starts from the immediate parent and can lose parent options, and eager command-group registration still fires async dynamic imports without awaiting them. The PR is open, unmerged, active at head 923670a6d1034714e9c3113645cb46496d07a786, and the latest diff addresses the earlier Greptile fallback-path review concern rather than being obsolete.

## Best Possible Solution

Keep this PR open as the active implementation candidate. Review the Promise-returning `plugin-sdk/cli-runtime` compatibility impact, confirm the latest nested fallback-path fix remains acceptable, then land after targeted CLI/browser tests and the changed gate are green. Current main does not yet implement the intended fix.
