## Summary

Keep open. Current main still uses the immediate parent command as the reparse root, so the PR's intended parent-option preservation is not implemented. The browser CLI still reads `cmd.parent.opts()` for `browserProfile`, current lazy tests do not cover parent option propagation, and the PR is a focused runtime/test/changelog change with no observed supply-chain or secrets-sensitive diff.

## Best Possible Solution

Review and land the focused root-reparse fix if targeted tests pass, keeping the nested Commander regression test and the browser lazy CLI parent-option test. The out-of-scope eager registration bug should be split into a separate issue or PR rather than blocking this parent-option repair.
