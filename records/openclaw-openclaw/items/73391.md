## Summary

Keep open. Current main has a separate partial fix for preserving browser parent flags, but the eager command-group path is still fire-and-forget and the PR remains the active implementation candidate for the documented eager fallback race. It should be rebased against the newer parent-flag work and reviewed for the public `plugin-sdk/cli-runtime` return-type change before landing.

## What This Changes

The PR changes lazy CLI reparsing to start from the root command with full fallback paths, makes eager command-group registration await dynamic imports, propagates async registration through core and browser CLI wiring, and adds focused CLI/browser regression tests plus changelog entries.

## Best Possible Solution

Rebase onto current main, reconcile the existing parent-flag fix with the PR’s root-reparse approach, keep regression coverage for parent options and eager dispatch, then land the awaited eager-registration fix once targeted CLI/browser tests and the appropriate changed gate pass.
