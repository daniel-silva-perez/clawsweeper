## Summary

Keep this issue open. Current main still depends on SQLite FTS5 for keyword/BM25 search, still skips keyword results when FTS is unavailable, and still has no better-sqlite3 or non-FTS lexical fallback. Related reports were closed as duplicates of this tracker, and an open PR explicitly references this issue with closing syntax, so it should stay open until that PR is reviewed/merged and maintainers decide whether visibility-only doctor work is enough or a real keyword fallback/backend change is still needed.

## Best Possible Solution

Keep this issue open as the canonical tracker. Review PR #62338 for the doctor warning improvement, then decide the product scope explicitly: either land visibility/documentation only and narrow this issue, or implement a true keyword fallback when FTS5 is unavailable, likely in the memory-core search/storage path with focused tests for missing FTS5 and hybrid/vector-only degradation.
