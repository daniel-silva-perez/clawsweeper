## Summary

Keep open. Current main still only records and degrades around missing FTS5 rather than restoring keyword/BM25 fallback, and the visible doctor warning proposed for this report is still an open PR rather than merged behavior.

## What This Changes

The issue reports that missing FTS5 support in Node SQLite breaks keyword/BM25 fallback in hybrid memory search and asks for a backend fallback, clearer documentation, or a more visible doctor warning.

## Best Possible Solution

Review and either land or replace the open doctor-warning PR, then decide whether the remaining product goal is documentation-only degraded-mode clarity or a separate backend change that restores keyword search on Node builds without FTS5.

## Reproduction Assessment

Yes. The report includes concrete logs and environments, and current main still marks FTS unavailable when `CREATE VIRTUAL TABLE ... USING fts5` fails, then omits keyword/BM25 results from hybrid search.

## Solution Assessment

Unclear. The open PR is a narrow maintainable fix for visibility in doctor, but it does not implement the stronger backend fallback requested for restoring BM25 on no-FTS Node builds.
