## Summary

Keep this issue open. Current main still has the reported TUI footer path: a known context window with no displayable total renders as `tokens ?/200k`, while the status/session_status path can still show `0/200k` through different zero-token semantics. The earlier fix PR #43044 is closed and unmerged, so there is no landed implementation to close against.

## Best Possible Solution

Keep this issue open. The narrow fix should make the TUI footer match status semantics for known-context/missing-total sessions, either by normalizing that display case to `0/<ctx> (0%)` in `src/tui/tui-formatters.ts` or by carrying an explicit zero total into `sessionInfo`. Add focused coverage in `src/tui/tui-formatters.test.ts` and a refresh-path regression in `src/tui/tui-session-actions.test.ts` where defaults provide `contextTokens` and the selected session row has no `totalTokens`.
