## Summary

Current main still has the reported TUI display fallback: when the footer has a known context window but no positive total token count, `formatTokens` renders `tokens ?/<context>` and omits the percent. Recent gateway transcript fallback improves some nonzero usage cases, but it does not implement the narrow TUI fallback proposed by the closed, unmerged related PR #43044.

## Best Possible Solution

Keep this issue open. Reapply the narrow TUI behavior from the closed related PR: in `src/tui/tui-formatters.ts`, when context is known and total is null or undefined, render `tokens 0/<ctx> (0%)` instead of `tokens ?/<ctx>`. Add a focused regression test in `src/tui/tui-formatters.test.ts`, plus a refresh-path test where defaults provide context but the selected session row has no `totalTokens`.
