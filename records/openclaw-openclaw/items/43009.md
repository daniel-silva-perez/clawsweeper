## Summary

Keep this issue open. Current main still renders a known-context/missing-total TUI footer as `tokens ?/<ctx>`, while the separate status path can report `0/<ctx> (0%)`; related PR #43044 is closed and unmerged rather than a shipped fix.

## Best Possible Solution

Keep this issue open and fix the TUI display layer. The narrow path is to normalize `formatTokens(null, knownContext)` to `tokens 0/<ctx> (0%)`, or to populate an explicit zero `totalTokens` before footer rendering when context is known and usage is absent. Add focused regression coverage in `src/tui/tui-formatters.test.ts` for the formatter behavior and in `src/tui/tui-session-actions.test.ts` for the refresh/default-context path.
