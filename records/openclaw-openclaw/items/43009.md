## Summary

Keep this issue open. Current main still has the reported TUI footer behavior: when the session has a known context window but no displayable total token count, the footer formatter renders `tokens ?/<ctx>`. The related fix PR #43044 is closed and unmerged, so there is no landed implementation to close against.

## Best Possible Solution

Keep this issue open. The best fix is the narrow TUI-layer change described by the closed PR #43044: either normalize `formatTokens` so a known context with a missing total displays `0/<ctx> (0%)`, or carry an explicit zero total into `sessionInfo` before the footer renders. Add focused coverage in `src/tui/tui-formatters.test.ts` for the formatter case and a refresh-path regression in `src/tui/tui-session-actions.test.ts` where defaults provide `contextTokens` and the selected session row has no `totalTokens`.
