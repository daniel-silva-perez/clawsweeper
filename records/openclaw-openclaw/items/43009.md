## Summary

Keep this issue open. Current main still has the reported TUI footer path: a known context window with missing total token usage renders as `tokens ?/<ctx>`, and related PR #43044 is closed and unmerged rather than landed.

## Best Possible Solution

Keep this issue open. The narrow maintainer path is to fix the TUI display layer: either normalize `formatTokens(null, knownContext)` to display `tokens 0/<ctx> (0%)`, or populate an explicit zero `totalTokens` before footer rendering when context is known and usage is absent. Add focused tests in `src/tui/tui-formatters.test.ts` for the formatter case and in `src/tui/tui-session-actions.test.ts` for the refresh/default-context path.
