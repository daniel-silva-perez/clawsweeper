## Summary

Keep this PR open. Current main still does not contain the Windows path/runtime alignment from the branch, but the proposed patch changes runtime executable and home-prefix resolution and still has blocking correctness issues around PATHEXT-gated executable checks and unmatched tilde tokens.

## What This Changes

The branch updates Windows scheduled-task and path-related tests while also changing runtime helpers for executable resolution, home-prefix expansion, and stable Homebrew Node path resolution.

## Best Possible Solution

Refresh or replace the branch with a focused patch that preserves the useful Windows taskkill test alignment, handles PATHEXT only where Windows extension lookup actually needs it, preserves unmatched tilde tokens, and adds targeted regression coverage for the executable and allowlist resolution behavior before landing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve unmatched tilde tokens:** `src/infra/home-dir.ts:73`
  - body: Wrapping the replacement in path.resolve also resolves inputs the regex intentionally does not expand. A token like `~alice/bin/tool` becomes a cwd-anchored absolute path, so executable and allowlist callers can target a different filesystem path instead of leaving the unresolved token unchanged.
  - confidence: 0.92
- **[P2] Bypass PATHEXT checks for qualified files:** `src/infra/executable-path.ts:41`
  - body: This Windows branch uses PATHEXT to decide whether an already-found file is executable. With a restricted env such as `PATHEXT=.CMD`, a concrete `node.exe` path can be rejected even though the file exists; PATHEXT should drive extension search, not invalidate qualified executable paths.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: No workflow, lockfile, dependency, publishing, or release-script changes were found, but the runtime command-resolution changes affect exec approval and executable lookup semantics and need maintainer review before merge.

Concerns:

- **[low] Home-prefix expansion can retarget approval paths:** `src/infra/home-dir.ts:73`
