## Summary

Keep this issue open. Current main and the latest release still pin @agentclientprotocol/sdk 0.20.0, and that SDK's generated Zod schemas still contain optional/nullish array fields without default empty arrays. OpenClaw still exposes the affected ACP bridge and ACPX/Claude surfaces, so there is no implemented-on-main or duplicate/superseding fix to close against.

## Best Possible Solution

Keep the issue open and have a maintainer decide whether OpenClaw should carry a temporary @agentclientprotocol/sdk patch/bump or wait for an upstream SDK release. The eventual fix should normalize optional array schema defaults for the affected ACP request/update/tool shapes, update the lockfile and any dependency patch metadata deliberately, and add focused ACP/Claude schema compatibility coverage around generated/defaulted array fields.
