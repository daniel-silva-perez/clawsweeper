## Summary

Keep this PR open for maintainer review. Current main still has the media URI workspace-join failure path, and the PR is a plausible narrow hardening of `loadWebMedia()`, but it does not cover every linked #74123 surface because the image tool and sandbox resolver can rewrite `media://` before `loadWebMedia()` sees it.

## What This Changes

The branch adds a `loadWebMediaInternal()` guard for unresolved `media://` references, skips `workspaceDir` resolution for classified scheme-bearing media refs, and adds three regression tests in `src/media/web-media.test.ts`.

## Best Possible Solution

Merge or revise a coordinated fix that treats `media://` as a managed URI before any workspace or sandbox path resolution, with regression coverage for `loadWebMedia()`, the image tool with a real `workspaceDir`, and the sandbox bridge path. The linked issue should only close once those user-visible paths are covered, or the PR should narrow its claim to supplemental loader hardening.
