## Summary

Keep open. Current main still lacks the proposed post-reset startup marker, so the PR is not obsolete, but the branch has blocking one-shot semantics and stale active-session-id behavior that should be fixed or consolidated before merge.

## What This Changes

The PR adds a persisted `postRotationStartupUntilMs` session field and uses it to wrap Discord channel/thread post-reset owner messages with a startup-recovery steer in reply/session code.

## Best Possible Solution

Consolidate this PR with #49001 into one durable Discord reset-window implementation that covers channel and thread resets, applies only to the first qualifying owner turn, clears after both normal and embedded-queue consumption paths, preserves active session-id resolution, and adds focused regression tests.

Full review comments:

- **[P2] Clear the window only after consuming the startup steer:** `src/auto-reply/reply/agent-runner.ts:850`
  - body: `startedRun` is true for any agent turn, but the new startup steer applies only to owner Discord messages. A non-owner message after reset can clear `postRotationStartupUntilMs` before the owner's first turn, so the recovery steer is skipped. Track actual steer consumption before clearing.
  - confidence: 0.88
- **[P2] Clear the marker on the embedded steer return path:** `src/auto-reply/reply/agent-runner.ts:273-277`
  - body: When `queueEmbeddedPiMessage` succeeds, this path returns before the `finally` clear runs. The marker stays active, so later owner messages in the 30 second window can be wrapped again as the first post-reset message. Clear the marker before this successful return.
  - confidence: 0.86
- **[P2] Preserve the active session id for embedded steer enqueue:** `src/auto-reply/reply/agent-runner.ts:273`
  - body: Current main resolves the active run id through `replyRunRegistry.resolveSessionId(sessionKey)` before queueing embedded steer. This branch queues with `followupRun.run.sessionId`, which can target a stale pre-reset id after session rotation. Keep the active-id lookup while swapping in the steer prompt.
  - confidence: 0.74
