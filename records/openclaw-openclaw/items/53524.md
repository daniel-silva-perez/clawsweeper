## Summary

Keep open. Current main still lacks the proposed reset-startup marker, so this PR is not obsolete, but the branch has blocking one-shot semantics and active-session routing issues that can leave the Discord channel/thread reset race unresolved or repeat the recovery steer.

## What This Changes

The PR adds a persisted `postRotationStartupUntilMs` session field and uses it to wrap Discord channel/thread post-reset owner messages with a startup-recovery steer prompt.

## Best Possible Solution

Consolidate this with #49001 into one durable Discord reset-window implementation that applies only to the first qualifying owner turn, clears after both normal and embedded-queue consumption paths, preserves active session-id resolution, and adds focused channel and thread reset regression coverage.

## Reproduction Assessment

Yes. There is a high-confidence source-level reproduction: a non-owner Discord run after reset can set `startedRun` and clear the marker before the owner's first turn, and a successful embedded steer enqueue returns before any marker clear.

## Solution Assessment

No. The proposed direction is useful, but this PR is not the best final fix until it preserves current active-session routing and makes the marker truly one-shot on steer consumption.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Clear the window only after an owner steer is consumed:** `src/auto-reply/reply/agent-runner.ts:850-857`
  - body: `startedRun` is set for any agent turn, but the startup steer only applies to owner Discord messages. If a non-owner channel message starts a run before the owner's first post-reset turn, this `finally` block clears `postRotationStartupUntilMs` and the owner turn never receives the recovery prompt.
  - confidence: 0.88
- **[P2] Clear the window before returning from embedded steer enqueue:** `src/auto-reply/reply/agent-runner.ts:273-277`
  - body: When `queueEmbeddedPiMessage` succeeds, this early return exits before the later clear path is reached. The marker remains active, so subsequent owner messages inside the 30 second window can be wrapped again as the first post-reset message.
  - confidence: 0.86
- **[P2] Preserve the active session id when queueing the steer prompt:** `src/auto-reply/reply/agent-runner.ts:273`
  - body: Current main resolves the active session id through `replyRunRegistry.resolveSessionId(sessionKey)` before queueing an embedded steer. This PR queues with `followupRun.run.sessionId`, which can target a stale pre-reset id after session rotation.
  - confidence: 0.74
