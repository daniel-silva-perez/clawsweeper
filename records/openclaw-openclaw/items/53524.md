## Summary

Keep open. Current main still lacks the proposed post-reset startup marker, and the PR addresses bundled Discord session behavior, but the patch leaves blocking one-shot and active-session routing gaps before it can merge.

## What This Changes

The PR adds a persisted `postRotationStartupUntilMs` session field and uses it to wrap Discord channel/thread post-reset owner messages with a startup-recovery steer in reply/session code.

## Best Possible Solution

Consolidate this with #49001 into one durable Discord reset-window implementation that covers channel and thread resets, applies only to the first qualifying owner turn, clears after both normal and embedded-queue consumption paths, preserves active session-id resolution, and adds focused regression tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Clear the window only after a steer is consumed:** `src/auto-reply/reply/agent-runner.ts:847`
  - body: `startedRun` is set for any agent turn, but the startup steer only applies to owner Discord messages. A non-owner channel message after reset can start a run and clear `postRotationStartupUntilMs` before the owner's first post-reset turn, so the recovery steer is skipped. Gate the clear on actual steer consumption instead of any started run.
  - confidence: 0.88
- **[P2] Clear the window before returning from embedded enqueue:** `src/auto-reply/reply/agent-runner.ts:273-277`
  - body: When `queueEmbeddedPiMessage` succeeds, this early return exits before the new `finally` clear path runs. The marker remains active, so later owner messages inside the 30 second window can be wrapped again as the first post-reset message. Clear the marker on this successful enqueue path too.
  - confidence: 0.86
- **[P2] Preserve the active session id for embedded steering:** `src/auto-reply/reply/agent-runner.ts:273`
  - body: Current main resolves the active run id through `replyRunRegistry.resolveSessionId(sessionKey)` before queueing an embedded steer. This PR queues with `followupRun.run.sessionId`, which can target a stale pre-reset id after session rotation; keep the active-id lookup while swapping in the steer prompt.
  - confidence: 0.74
