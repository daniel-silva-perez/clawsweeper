## Summary

Keep this PR open. Current main still does not implement the requested minimal/autonomous-mode context-pruning cap: `keepLastAssistants` remains 3 by default, the embedded extension factory has no prompt-mode input, and the run/compaction paths compute prompt mode but do not pass it into context-pruning setup. The PR discussion is relevant because the original `skillsSnapshot` signal concern was addressed by the second PR commit switching to `promptMode === "minimal"`.

## Best Possible Solution

Review this PR as an active implementation candidate. The best path is to rebase or recreate the small prompt-mode plumbing on current main: add `promptMode` to the embedded extension factory inputs, cap `settings.keepLastAssistants` to 1 only when the effective prompt mode is `minimal`, pass the effective run and compaction prompt modes into the factory, preserve explicit `keepLastAssistants: 0`, and keep focused tests for minimal versus full prompt behavior.
