## Summary

Keep this PR open. Current main at 39f810911c382f92ea80b2fb406fdf7dbd405721 still does not implement the requested minimal-prompt context-pruning cap: `keepLastAssistants` remains 3, the embedded extension factory has no `promptMode` input, and both run and compaction paths compute prompt mode without passing it into pruning setup. The PR head c1b7a16a64257d54ae6d9caa66b7522df3f64df3 is still a plausible narrow implementation candidate and addresses the earlier review concern by switching from `skillsSnapshot` to `promptMode === "minimal"`.

## What This Changes

The PR passes prompt mode into embedded extension setup, caps context-pruning `keepLastAssistants` to 1 for minimal prompt mode, forwards that mode from run and compaction paths, and adds focused extension-factory tests.

## Best Possible Solution

Review and land this PR, or an equivalent patch, if maintainers agree that effective minimal prompt mode is the right boundary. The landing path should pass the effective run and compaction prompt modes into `buildEmbeddedExtensionFactories`, cap `keepLastAssistants` to 1 only for minimal mode while preserving explicit `0`, keep focused minimal/full tests, and optionally address the stale Anthropic model-id nit in the touched test helper.
