## Summary

Keep open. The CLI gateway timeout portion overlaps the closed #51357 probe/auth discussion, and current main has newer gateway probe/auth hardening, but this issue also reports model switching failure in the web controller and `/model`. Current main supports `models.mode: "merge"` and has session-patch/UI coverage for provider-qualified model switches, but the issue lacks the exact model-switch input, error, endpoint response, or current-version reproduction needed to conclude it is fixed or fully duplicated.

## Best Possible Solution

Keep this issue open and ask for a current-main or latest-release reproduction focused on model switching: exact UI or `/model` input, expected versus actual selected model, `openclaw models list --provider <id>` and `openclaw models status --json` output with secrets redacted, the relevant `sessions.patch` or gateway log error, and whether the same provider/model works via `openclaw infer model run --model <provider/model>`. If it reproduces, the likely owner surface is the generic model catalog/allowlist/session patch path, not a provider-specific config syntax issue.
