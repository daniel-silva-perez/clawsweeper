## Summary

Keep open. Current main does not show OpenClaw stripping Telegram or Discord token keys from the state-dir `.env`, and the token loaders accept those keys when present, but the requested migration hardening is not implemented: the migration guide still lacks a state-dir `.env` integrity checklist, and doctor does not have a safe generic preflight for expected channel env keys.

## What This Changes

The issue asks OpenClaw to investigate missing Telegram/Discord bot token lines after a VPS rsync migration, add an `openclaw doctor` preflight where OpenClaw can know those tokens are expected, and document `.env` migration verification.

## Best Possible Solution

Add migration guidance to verify expected state-dir `.env` keys without exposing values and recommend `openclaw status --all` or channel-specific probes after cutover. If doctor support is added, scope it to channels or SecretRefs that are explicitly configured or service-managed so optional env fallbacks do not create noisy warnings.
