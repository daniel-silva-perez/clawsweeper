## Summary

Current main still has the focus-mode hamburger UX bug this PR targets. The proposed behavior is still useful, but the patch is stale against the current `.topbar-nav-toggle` surface and should be adapted before merge.

## Best Possible Solution

Keep this PR open or replace it with an equivalent current-main patch. The implementation should update the `.topbar-nav-toggle` click handler so that, when `settings.chatFocusMode` is active and onboarding is not controlling focus, clicking the hamburger clears `chatFocusMode` and opens the nav/drawer. Add a browser regression test that activates focus mode, clicks `.topbar-nav-toggle`, and verifies `.shell--chat-focus` is removed and the nav becomes visible.
