## Summary

Keep open. Current main still filters web-search setup entries by `onboardingScopes`, DuckDuckGo's shared provider metadata still does not set `onboardingScopes: ["text-inference"]`, docs still describe DuckDuckGo as selectable during CLI onboarding, and PR #65940 remains open/unmerged with closing syntax for this issue.

## Best Possible Solution

Keep this issue open and review or refresh PR #65940. The code fix should add `onboardingScopes: ["text-inference"]` to the shared DuckDuckGo web-search provider metadata so both runtime and lightweight contract surfaces inherit it, then add a focused regression assertion that the setup/web-search provider options include the real bundled DuckDuckGo provider when plugin policy allows it.
