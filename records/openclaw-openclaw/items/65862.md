## Summary

Current main still excludes DuckDuckGo from the interactive web-search setup list. The setup flow filters providers by `onboardingScopes`, the current DuckDuckGo shared provider metadata still lacks `onboardingScopes: ["text-inference"]`, and an open same-author PR (#65940) still references this issue with closing syntax, so the issue should remain open for PR review rather than be auto-closed.

## Best Possible Solution

Keep this issue open and review or refresh PR #65940. The best fix is to add `onboardingScopes: ["text-inference"]` to `extensions/duckduckgo/src/ddg-search-provider.shared.ts` so both runtime and lightweight contract surfaces inherit it, then add focused regression coverage proving the real bundled DuckDuckGo provider appears in web-search setup options when plugin policy allows it.
