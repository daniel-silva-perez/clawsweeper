## Summary

Current main still filters web-search setup providers by `onboardingScopes`, and DuckDuckGo's shared provider metadata still lacks `onboardingScopes: ["text-inference"]`. Because open PR #65940 uses closing syntax for this same bug and the fix has not merged, this issue should stay open for PR review or refresh.

## What This Changes

DuckDuckGo should be visible in the interactive web-search setup wizard by adding text-inference onboarding scope metadata to its web-search provider.

## Best Possible Solution

Keep this issue open and review or refresh PR #65940. The current-main fix should add `onboardingScopes: ["text-inference"]` to `extensions/duckduckgo/src/ddg-search-provider.shared.ts` so runtime and contract paths inherit the same metadata, then add focused regression coverage proving the real bundled DuckDuckGo provider appears in web-search setup options when plugin policy allows it.
