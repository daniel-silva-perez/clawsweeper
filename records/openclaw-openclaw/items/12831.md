## Summary

Keep open. Current main has useful gateway plumbing for tool-event delivery (`tool-events` and `session.tool`) and Control UI chat tool cards, but it still does not provide the requested first-class Activity panel / observer mode with separate UI state, per-connection verbose behavior, TUI split-pane support, or an external activity stream experience that fully decouples visibility from chat clutter. Related #46604 is closed and narrower; draft PR #60814 overlaps but is open/draft and does not supersede the full scope.

## Best Possible Solution

Keep this open as the broad observer-mode tracker. Build on the existing `tool-events` and `session.tool` plumbing, add a dedicated Control UI Activity panel with independent scroll/state, consume session-scoped tool events for late-joining observers, decide per-connection/tab verbose semantics, and add focused TUI/docs/tests. Treat #46604 as narrower/closed related context and #60814 as an open draft implementation candidate unless maintainers explicitly make one canonical for the full scope.
