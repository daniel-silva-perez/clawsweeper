## Summary

Keep open. Current main has useful backend plumbing for run-scoped `tool-events` and session-scoped `session.tool`, but the requested user-facing observer mode is still incomplete: Control UI still feeds live tool activity into chat state, there is no Activity panel tab/state/handler, and the TUI still gates visible tool cards on session verbose state. The existing discussion also already routes this as broad maintainer product work with open draft PR #60814 as a possible implementation path.

## What This Changes

Request a separated activity/tool-call observer mode so users can monitor tool calls without cluttering the main chat, including a Control UI side panel, connection-scoped verbose behavior, a dedicated stream, and a TUI split pane.

## Best Possible Solution

Keep this as the broad observer-mode tracker. Build a dedicated Control UI Activity panel on top of the existing `tool-events` and `session.tool` plumbing with independent state, scrolling, filtering, collapse/expand, and truncation behavior, then split per-connection verbose, TUI split-pane, and external stream work into deliberate follow-ups if they remain in scope.
