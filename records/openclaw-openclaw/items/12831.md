## Summary

Keep open. Current main still has only partial backend observer plumbing: tool events can be broadcast to capable WebSocket/session subscribers, but Control UI still renders live tool activity as chat tool messages, has no Activity/observer tab or `session.tool` handler, and the TUI still hides tool cards when session verbose is off. The related draft implementation PR #60814 is still open and unmerged in the provided context.

## What This Changes

Request to add a separate activity/tool-call observer mode so users can monitor agent tool work without cluttering the main conversation stream.

## Best Possible Solution

Keep this as the broad observer-mode tracker. Build a dedicated Control UI Activity panel on top of the existing `tool-events` and `session.tool` plumbing with independent state, scrolling, filtering, collapse/expand, and truncation behavior, then split per-connection verbose, TUI split-pane, and external stream work into deliberate follow-ups if they remain in scope.
