## Summary

Keep open. Current main has backend tool-event plumbing, but the requested user-facing observer mode is still missing: Control UI still renders live tool activity as chat tool messages, has no Activity/observer route, does not consume `session.tool`, and the TUI still gates tool cards on the session verbose setting.

## What This Changes

Request to separate verbose/tool-call activity from the main conversation through a Control UI activity panel, connection-scoped verbose state, observer stream, or TUI split pane.

## Best Possible Solution

Keep this as the broad observer-mode tracker. Build a dedicated Control UI Activity panel on top of existing `tool-events` and `session.tool` plumbing with independent state, scrolling, filtering, expand/collapse, and truncation behavior, then split per-connection verbose, TUI split-pane, and external-stream follow-ups if maintainers keep them in scope.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence inspection path for the missing behavior. Current main still routes live tool events into chat state/cards, lacks an Activity/observer route, and has TUI tests asserting tool-event suppression when verbose is off.

## Solution Assessment

No complete solution is on main yet. The narrowest maintainable first step appears to be a Control UI Activity panel over the existing backend event plumbing, with broader TUI and external-stream work scoped after maintainer review of draft PR #60814.
