## Summary

Keep open. Current main has useful backend plumbing for tool-event broadcast and session-scoped tool mirroring, but the user-facing observer mode is still missing: Control UI still feeds live tool events into chat items, has no Activity tab or `session.tool` handler, and the TUI still gates tool cards on session verbose state. The maintainer discussion also identifies this as the broad tracker, with draft PR #60814 still open and unmerged.

## What This Changes

The issue requests a separate activity/tool-call observer surface so users can monitor agent work without mixing verbose tool output into the main chat stream.

## Best Possible Solution

Keep this as the broad observer-mode tracker. Build a dedicated Control UI Activity panel on top of existing `tool-events` and `session.tool` plumbing with independent state, scrolling, filtering, collapse/expand, and truncation behavior, then deliberately split per-connection verbose, TUI split-pane, and external stream work into follow-ups if they remain in scope.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence inspection path for the missing behavior. Current main still routes live tool events into chat messages, lacks an Activity/observer route, and has TUI tests asserting tool-event suppression when verbose is off.

## Solution Assessment

No complete solution is on main yet. The narrowest maintainable first step appears to be a Control UI Activity panel over the existing backend event plumbing, with the broader TUI and external stream pieces scoped by maintainers after reviewing or splitting draft PR #60814.
