## Summary

Keep open. Current main has useful backend tool-event plumbing, but the requested user-facing observer mode is still missing: Control UI still routes live tool activity into chat state/cards, has no Activity/observer tab or `session.tool` consumer, and the TUI still suppresses tool cards when verbose is off.

## What This Changes

The issue requests a separate Activity/observer surface for tool calls and agent work so verbose tool output can be monitored without cluttering the main chat stream.

## Best Possible Solution

Keep this as the broad observer-mode tracker. Build a dedicated Control UI Activity panel on top of existing `tool-events` and `session.tool` plumbing with independent state, scrolling, filtering, collapse/expand, and truncation behavior, then split per-connection verbose, TUI split-pane, and external stream follow-ups if they remain in scope.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence inspection path for the missing behavior. Current main still routes live tool events into chat state/cards, lacks an Activity/observer route, and has TUI tests asserting tool-event suppression when verbose is off.

## Solution Assessment

No complete solution is on main yet. The narrowest maintainable first step appears to be a Control UI Activity panel over the existing backend event plumbing, with broader TUI and external-stream work scoped after maintainer review of draft PR #60814.
