## Summary

Keep open. Current main has backend plumbing for run-scoped `tool-events` and session-scoped `session.tool`, but the requested user-facing Activity panel / observer mode is still missing, and the supplied maintainer comment plus draft PR context treat this as broad product follow-up rather than completed work.

## What This Changes

The issue requests separating verbose tool and activity output from the main chat through a Control UI side panel, per-connection verbose state, an observer stream, or a TUI split pane.

## Best Possible Solution

Keep this as the canonical observer-mode tracker. Build a dedicated Control UI Activity panel on top of the existing `tool-events` and `session.tool` plumbing with independent state, scrolling, filtering, expand/collapse, and truncation behavior, then split per-connection verbose, TUI split-pane, and external observer stream work into deliberate follow-ups if maintainers keep them in scope.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence inspection path for the missing behavior. Current main still routes live tool events into chat state/cards, lacks an Activity/observer route, and has TUI behavior/tests that suppress tool events when verbose is off.

## Solution Assessment

No complete solution is on main yet. The narrowest maintainable first step appears to be a Control UI Activity panel over the existing backend event plumbing, with broader TUI and external-stream work scoped after maintainer review of draft PR #60814.
