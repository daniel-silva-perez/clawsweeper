## Summary

Keep open. Current main has partial backend observer plumbing for run-scoped `tool-events` and session-scoped `session.tool`, but the user-facing observer mode requested here is still missing: Control UI still feeds live tool events into chat rendering, there is no `session.tool` branch for independent activity state, and TUI tool cards remain gated by session verbose state. The linked real-time activity dashboard PR #60814 is still open and draft, so this should remain a maintainer-scoped tracker rather than be swept closed.

## What This Changes

The issue requests a separate activity/tool-call observer surface so users can monitor agent tool activity without mixing verbose output into the main chat stream.

## Best Possible Solution

Keep this as the broad observer-mode tracker. Build a dedicated Control UI Activity panel on top of existing `tool-events` and `session.tool` plumbing with independent state, scrolling, filtering, collapse/expand, and truncation behavior, then split follow-ups for per-connection verbose settings, TUI split-pane behavior, and any external observer stream.
