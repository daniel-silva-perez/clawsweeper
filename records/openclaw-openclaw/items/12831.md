## Summary

Keep open. Current main has useful backend event plumbing for run-scoped `tool-events` and session-scoped `session.tool`, but the requested user-facing observer mode is still missing: Control UI still routes tool activity into chat rendering, there is no dedicated Activity panel/state model, and the TUI still suppresses tool cards when verbose is off. The linked dashboard implementation PR remains open and draft, so this should stay as a maintainer-scoped tracker.

## What This Changes

Request to separate verbose tool-call activity from the main conversation via a Control UI Activity panel, per-connection verbose behavior, an observer stream, and a TUI split-pane style view.

## Best Possible Solution

Keep this as the broad observer-mode tracker. Build a dedicated Control UI Activity panel on top of the existing `tool-events` and `session.tool` plumbing with independent state, scrolling, filtering, collapse/expand, and truncation behavior, then deliberately split follow-ups for per-connection verbose settings, TUI split-pane behavior, and any external observer stream.
