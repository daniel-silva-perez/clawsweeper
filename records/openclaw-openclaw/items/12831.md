## Summary

Keep open. Current main has backend observer plumbing for run-scoped `tool-events` and session-scoped `session.tool`, but the requested user-facing observer mode is still missing: Control UI still routes live tool events into chat rendering, the Control UI dispatcher has no `session.tool` branch for a separate activity model, and the TUI still suppresses tool cards when verbose is off. The open draft dashboard PR #60814 is related but unmerged, so this remains a broad maintainer-scoped product/UX/API tracker.

## What This Changes

The issue requests separating verbose/tool-call activity from the main chat through a Control UI activity panel, per-connection verbose/observer behavior, a dedicated tool-activity stream, or a TUI split pane.

## Best Possible Solution

Keep this as the broad observer-mode tracker. Build a dedicated Control UI Activity panel on top of the existing `tool-events` and `session.tool` plumbing with independent state, scrolling, filtering, collapse/expand, and truncation behavior, then deliberately split or accept follow-ups for per-connection verbose settings, a TUI split pane, and any external observer stream.
