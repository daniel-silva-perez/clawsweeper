## Summary

Keep open. Current main has backend observer plumbing for run-scoped `tool-events` and session-scoped `session.tool`, but the requested user-facing observer mode is still missing: Control UI still renders live tool events as chat items, has no `session.tool` handler for separate activity state, and the TUI still suppresses tool cards when session verbose is off. The related real-time activity dashboard PR #60814 is open and unmerged, so this remains a maintainer-scoped product/UX/API tracker rather than a cleanup close.

## What This Changes

Requested a separate activity/tool-call observer mode that keeps verbose agent/tool activity visible outside the main conversation stream.

## Best Possible Solution

Keep this as the broad observer-mode tracker. Build a dedicated Control UI Activity panel on top of the existing `tool-events` and `session.tool` plumbing with independent state, scrolling, filtering, collapse/expand, and truncation behavior, then deliberately split follow-ups for per-connection verbose settings, a TUI split pane, and any external observer stream.
