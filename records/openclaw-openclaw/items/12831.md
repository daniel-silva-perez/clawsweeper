## Summary

Keep open. Current main has backend observer plumbing for tool-event delivery, but the requested user-facing observer mode is still missing: Control UI still renders tool activity inside the chat model, has no Activity tab or session.tool handling, and the TUI still hides tool cards when verbose is off. The provided discussion also already treats this as the broad tracker while draft PR #60814 remains open and unmerged.

## What This Changes

Request a first-class activity/observer mode that separates verbose tool-call activity from the main chat stream across Control UI, per-connection settings, external streams, and TUI.

## Best Possible Solution

Keep this as the broad observer-mode tracker. Build a dedicated Control UI Activity panel on top of the existing tool-events and session.tool plumbing with independent state, scrolling, filtering, collapse/expand, and truncation behavior, then split per-connection verbose, TUI split-pane, and external stream work into follow-ups if they remain in scope.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence inspection path for the missing behavior. Current main still routes live tool events into chat messages, lacks an Activity/observer route, and has TUI tests asserting tool-event suppression when verbose is off.

## Solution Assessment

No complete solution is on main yet. The narrowest maintainable first step appears to be a Control UI Activity panel over the existing backend event plumbing, with the broader TUI and external-stream pieces scoped after maintainers review or split draft PR #60814.
