## Summary

Keep this open. Current main has useful backend tool-event plumbing, but the user-facing observer mode is still missing: Control UI still routes live tool events into chat items instead of a dedicated Activity panel, `session.tool` is not consumed by the Control UI event dispatcher, and the TUI still gates tool cards on the session verbose setting. The linked draft activity-dashboard PR remains open in the provided context, so this issue should remain the broad tracker rather than be closed as implemented.

## What This Changes

The issue requests a separate activity/tool-call observer mode so users can watch agent work without mixing verbose tool output into the main chat stream.

## Best Possible Solution

Keep this as the broad observer-mode tracker. Build a dedicated Control UI Activity panel on top of the existing `tool-events` and `session.tool` plumbing, with independent state, scrolling, filtering, collapse/expand, truncation, and clear follow-up boundaries for per-connection verbose settings, TUI split-pane support, and any external observer stream.
