## Summary

Keep open. Current main still has useful partial plumbing for tool-event delivery, but it does not implement the requested first-class Activity panel / observer mode. Gateway broadcasts run-scoped `tool-events` and session-scoped `session.tool`, yet Control UI still converts tool events into `chatToolMessages` and appends them into the chat item stream, and the TUI still suppresses tool events when verbose is off. The prior maintainer/Codex comment already reached the same keep-open conclusion, and nothing in current main supersedes that: draft PR #60814 is open and does not close this issue, while related #46604 and #74018 are closed narrower overlaps.

## Best Possible Solution

Keep this open as the broad observer-mode tracker. The best path is to build on the existing `tool-events` and `session.tool` plumbing, add a dedicated Control UI Activity panel with independent state/scrolling/filtering, decide whether verbose is session- or connection-scoped for observer tabs, and handle TUI/external stream semantics deliberately. Treat #60814 as an open draft implementation candidate and #46604/#74018 as closed narrower related context, not as the canonical resolution.
