## Summary

Keep this open: current main still has no safe-cache-trace exception in the gateway config mutation guard, and the linked implementation PR is still open and unmerged, so closing this issue before that PR or an equivalent fix lands would be premature.

## What This Changes

Allow the agent-facing gateway config patch path to enable diagnostics.cacheTrace only when message, prompt, and system content capture remain disabled.

## Best Possible Solution

Review and land PR #73098, or an equivalent narrow change, so the gateway tool permits cache-trace enablement only when the resulting diagnostics config keeps all raw-content capture disabled, with regression coverage for safe enablement and unsafe attempts.

## Reproduction Assessment

Yes. Static inspection of current main is enough: the reported patch changes diagnostics.cacheTrace leaves, those leaves are absent from the gateway tool allowlist, and the guard rejects disallowed changed paths before writing.

## Solution Assessment

Yes. A conditional allow based on the resulting cacheTrace config is the narrowest maintainable direction; unconditionally allowing the subtree is less safe because the content-capture fields default to true.
