## Summary

Keep open: current main still rejects the reported safe cache-trace patch through the agent-facing gateway tool, and open PR #73098 is the active implementation candidate rather than a reason to close the issue before merge.

## What This Changes

The issue requests allowing `gateway config.patch` to enable `diagnostics.cacheTrace.enabled` when message, prompt, and system content capture are explicitly disabled.

## Best Possible Solution

Review and land PR #73098, or an equivalent narrow change, so safe cache-trace enablement is accepted only when all content-capture fields are explicitly false and regression tests cover both allowed observability and blocked sensitive capture.

## Reproduction Assessment

Yes. Static inspection of current main is enough: the reported patch changes `diagnostics.cacheTrace.enabled` and the three `include*` leaves, and the gateway tool rejects every changed leaf outside `ALLOWED_GATEWAY_CONFIG_PATHS`.

## Solution Assessment

Yes. Conditional safe enablement is the narrowest maintainable direction; unconditionally allowing the cache-trace subtree would be riskier because the content-capture flags default to true.
