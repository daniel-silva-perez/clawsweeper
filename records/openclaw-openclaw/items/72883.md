## Summary

Keep open: current main still rejects the safe `diagnostics.cacheTrace.*` patch through the agent-facing gateway tool, and open PR #73098 is the active closing implementation candidate rather than a reason to close before merge.

## What This Changes

The issue requests a safe way for `gateway config.patch` to enable `diagnostics.cacheTrace.enabled` while forcing `includeMessages`, `includePrompt`, and `includeSystem` off.

## Best Possible Solution

Review and land PR #73098, or an equivalent narrow change, so cache tracing can be enabled through the gateway tool only when the resulting config has message, prompt, and system content capture disabled, with regression coverage for blocked unsafe cases.

## Reproduction Assessment

Yes. Static inspection of current main is enough: the requested patch changes `diagnostics.cacheTrace.enabled` and the three `include*` leaves, while the gateway mutation guard rejects every changed path outside the allowlist.

## Solution Assessment

Yes. The narrow conditional allowlist used by the linked PR is the best direction; unconditionally allowing the cache-trace subtree would be riskier because the content-capture flags default to `true`.
