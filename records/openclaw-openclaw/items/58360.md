## Summary

Keep this PR open, but not as ready to merge. Current main still does not implement the proposed raw `before_model_call` / `after_model_call` payload hooks; it only ships sanitized `model_call_started` / `model_call_ended` telemetry, and this PR needs a rebase plus privacy/security fixes before it can be accepted.

## What This Changes

The PR adds `before_model_call` and `after_model_call` hook types and runner methods, wraps embedded-runner `streamFn` calls to emit request/response events, refactors replay tool-call normalization, and adds focused hook tests.

## Best Possible Solution

Rework this as a deliberate extension of the current model-call diagnostics surface: either add an explicitly gated raw-payload mode to `model_call_started` / `model_call_ended`, or add separate raw-payload hooks with conversation-access enforcement, public docs, loader policy tests, and embedded-runner tests covering provider-bound payload alignment, retries, errors, aborts, and early stream termination.

## Reproduction Assessment

Not applicable as a feature/API PR rather than a runtime bug report. The current-main check is high-confidence: searches and source inspection show only sanitized `model_call_started` / `model_call_ended`, while the PR diff adds the missing raw payload hook names and wiring.

## Solution Assessment

No, not as-is. The direction is plausible, but the current patch needs security gating for raw conversation payloads and should be rebased onto the newer metadata-only model-call hook implementation rather than landing a parallel stale API unchanged.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Gate raw model-call hooks behind conversation access:** `src/plugins/types.ts:1799-1800`
  - body: These new hooks expose `systemPrompt`, provider request messages, and the final response message to plugin code, but the patch only adds them to the general hook catalog. Existing raw conversation hooks are blocked for non-bundled plugins unless `allowConversationAccess=true`; without adding these hook names to that policy, an external plugin can receive more sensitive model-call content than `llm_input`/`llm_output` with no explicit opt-in.
  - confidence: 0.92
- **[P2] Use an attempt-unique token in model call IDs:** `src/agents/pi-embedded-runner/run/attempt.ts:268-270`
  - body: `buildModelCallId` still derives IDs from `runId`, `sessionId`, and a per-wrapper sequence. The outer retry loop re-enters `runEmbeddedAttempt` with the same `runId` and `sessionId`, while `modelCallCounter` starts over, so the first provider call in each retry can reuse the same `callId` and break trace correlation.
  - confidence: 0.9
- **[P2] Emit after hooks when streams are abandoned:** `src/agents/pi-embedded-runner/run/attempt.ts:1263-1272`
  - body: The after hook is only fired through a patched `.result()` method. If the stream is consumed with async iteration and then abandoned or returned early, `.result()` is never called and plugins get an unmatched `before_model_call`; current main's diagnostic wrapper handles this by proxying the async iterator and finalizing on early return.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The PR introduces a new plugin data-access path for raw model prompts, request messages, and responses without the policy gate used for existing conversation hooks.

Concerns:

- **[high] Raw model payloads bypass conversation-access gating:** `src/plugins/types.ts:1799`
  - body: `before_model_call` and `after_model_call` expose sensitive conversation/model-call content, but the diff does not add these names to the conversation-hook allowlist enforced by the plugin registry. Non-bundled plugins should need an explicit `plugins.entries.<id>.hooks.allowConversationAccess=true` opt-in before receiving these events.
  - confidence: 0.92
