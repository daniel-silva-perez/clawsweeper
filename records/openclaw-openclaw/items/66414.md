## Summary

Keep this PR open. Current main still does not implement the PR's two intended fixes: gateway image-capability lookup still misses provider-qualified catalog ids when the session model is bare, and Control UI attachment reads still have no pending-read guard before Enter/send. The PR remains useful but should not be merged as-is because review comments identify stale render-state bugs in the pending counter and attachment append logic, plus over-broad blocking of command sends. Security review found no supply-chain or secrets-sensitive changes in the provided PR file list.

## Best Possible Solution

Keep this PR open and request a focused revision: preserve the gateway provider-qualified catalog lookup and regression test, then rework Control UI attachment handling so pending-read deltas and attachment appends apply against live app state, limit pending-read blocking to user sends that depend on in-flight attachments, wire the current run-controls component, and add tests for concurrent paste/select/drop races plus command-bypass behavior.

## Risks / Open Questions

- Merging the current Control UI patch as-is can still re-enable send too early or leave the UI blocked because pending-count updates are computed from render-time props instead of live app state.
- Merging the current Control UI patch as-is can still drop attachments because FileReader callbacks append to captured attachment arrays rather than the latest attachment state.
- The current PR guard blocks all sends while attachment reads are pending, including messageOverride/local command paths such as new-session or compact actions that do not depend on in-flight attachments.
- The Control UI changed around the run-controls component on current main, so the PR needs a focused rebase/rework before it can be considered merge-ready.
