## Summary

Keep this PR open. Current main still does not implement the PR's central changes for #72015: Active Memory defaults remain 15s plus 30s setup grace, the hook envelope remains sized for the 120s cap, docs still recommend 15s budgets for recent/full modes, and QMD startup has no jitter path. The PR is also draft and has an unresolved functional review concern around the proposed fixed 4s hook timeout.

## Best Possible Solution

Keep this PR open for maintainer review as an implementation candidate for #72015. The best path is to revise the hook envelope so it respects the resolved per-agent timeout or the documented 120s cap, deliberately choose the default timeout/setup-grace policy, add bounded QMD startup jitter or concurrency control without regressing current deferral semantics, update docs, and land with targeted Active Memory plus gateway startup tests.

## Risks / Open Questions

- The proposed fixed 4_000ms before_prompt_build envelope can cancel valid Active Memory configurations above 3s before the plugin's own watchdog fires.
- The QMD startup change switches toward parallel initialization with random jitter but no explicit concurrency bound, so it may still create startup bursts or alter existing eager/deferred semantics without deterministic coverage.
- The PR is a draft implementation candidate for open #72015; closing it before review or replacement would prematurely discard potentially useful focused work.
