## Summary

Keep this PR open. Current main has related Active Memory timeout and Gateway/QMD startup work, but it does not implement this PR's central changes: the default recall timeout is still 15s, setup grace is still 30s, docs still keep 15s budgets for common modes, and QMD startup has no jitter path. The PR is also draft and paired with the still-open #72015, so the best outcome is maintainer review/fix-up rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review as an implementation candidate for #72015. The best path is to fix the hook-timeout envelope so custom timeoutMs values still work, add targeted QMD startup jitter/concurrency coverage, and then decide the product policy for the lower default timeout, setup-grace removal, and docs budget changes before landing or superseding it with a narrower canonical PR.
