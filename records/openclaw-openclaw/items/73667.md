## Summary

Keep this PR open. Current main has related Active Memory timeout work, but it does not implement this PR's central changes: the default recall timeout is still 15s, setup grace is still 30s, the hook registration still allows 150s, docs still recommend 15s budgets, and QMD startup has no jitter path. The PR remains an implementation candidate for #72015 rather than cleanup-closeable.

## Best Possible Solution

Keep this PR open for maintainer review as an active candidate for #72015. The best path is to refine or land a focused fix that lowers documented/default Active Memory budgets, keeps recall fail-open, deliberately resolves the setup-grace and hook-timeout policy, and adds targeted QMD startup jitter/concurrency coverage.
