## Summary

Keep this draft PR open. Current main has partial Active Memory timeout hardening and narrower implicit QMD startup, but it does not implement the PR's central changes: the default recall timeout is still 15s, setup grace is still 30s, the hook envelope is still 150s, the docs still recommend 15s for recent/full/default examples, and QMD startup still initializes eager agents sequentially with no jitter. The PR is also paired with open #72015 and has an unresolved functional review concern about truncating valid custom timeoutMs values.

## Best Possible Solution

Keep this draft PR open as an implementation candidate for #72015. The best path is to revise the hook envelope so it respects the resolved per-agent timeout or documented cap, make a deliberate maintainer decision on the default timeout and setup-grace policy, and add deterministic tests for high custom timeoutMs values plus QMD startup jitter/concurrency behavior before landing or replacing it with a narrower maintainer-owned fix.

- The PR is paired with open #72015 by the same author and should not be cleanup-closed while it remains an implementation candidate.
