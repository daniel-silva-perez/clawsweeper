## Summary

Keep open. Current main does not implement this PR's intended GitHub Copilot chat-model auto-discovery: the Copilot catalog hook still only resolves the base URL and returns an empty model list, while unknown Copilot model support is limited to a manual forward-compatible resolver after a user requests/configures a model. The PR still contains unique work, but it needs maintainer review and cleanup because it adds a new authenticated network call and also carries unrelated branch changes.

## Best Possible Solution

Keep this PR open, but ask for a narrow rebase. The best path is a focused GitHub Copilot provider change that adds chat-model `/models` discovery under `extensions/github-copilot`, honors discovery-disabled and explicit-model config, uses configured/token-derived base URLs and provider headers, validates and sorts the response, dedupes built-ins, routes APIs through `resolveCopilotTransportApi`, preserves existing stream/replay hooks, and adds targeted provider-discovery tests. Unrelated session-tool-result-guard and contract/testkit changes should move to separate PRs or be removed.
