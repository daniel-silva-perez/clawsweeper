## Summary

Keep open. Current main still lacks GitHub Copilot chat-model discovery, and this PR is useful source work, but the branch is not ready to merge because configured provider headers still replace the returned Copilot IDE headers during implicit-provider merging. The branch also mixes unrelated session tool-result guard changes into a provider feature, so the safer maintainer path is a narrow current-main implementation or replacement PR.

## What This Changes

The PR adds a GitHub Copilot `/models` discovery helper, wires discovered models and IDE headers into the provider catalog, expands provider-discovery tests, allowlists the new fetch call, and also changes session tool-result guard behavior/tests.

## Best Possible Solution

Land a narrow current-main implementation in the GitHub Copilot plugin: additive best-effort `/models` discovery, deterministic model ordering, current transport/auth semantics, configured baseUrl and header support, a provider-specific header merge proof, and no unrelated session guard changes.

## Reproduction Assessment

Yes for the blocking review finding. Configure `models.providers.github-copilot.headers`, run implicit provider resolution with the PR catalog result, and the shallow `mergeImplicitProviderConfig` path replaces `COPILOT_IDE_HEADERS` before final merging; no live Copilot account is needed to reproduce that config-loss path.

## Solution Assessment

No, not as-is. The feature direction is maintainable, but the best fix is a narrower provider-owned patch that deep-merges Copilot IDE headers with explicit provider headers and excludes the unrelated agent session changes.
