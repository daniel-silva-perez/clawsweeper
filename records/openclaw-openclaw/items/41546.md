## Summary

Keep this PR open. Current main still has the Control UI bug this PR targets: ordinary edits to the Overview WebSocket URL clear the manually entered Gateway Token before the user connects. The PR is also paired with open bug #41545 by the same author and has a maintainer-pushed, scope-aware repair on the PR branch, so it remains the active implementation candidate rather than something to close as obsolete.

## Best Possible Solution

Review and land this PR, or an equivalent focused repair, as the canonical fix for #41545: preserve manually entered Gateway Access tokens during same-scope URL formatting edits, clear them when the normalized gateway endpoint changes, keep regression coverage for both behaviors, preserve contributor credit, then close #41545 and supersede duplicate PRs such as #41718 and #42001.
