## Summary

Keep open. Current main still resolves `$VAR` and `${VAR}` user paths literally outside config-load substitution, so this PR is not obsolete; it is a valid small fix candidate, but it needs maintainer selection alongside the overlapping open PR and the resolver semantics called out in review.

## What This Changes

The PR changes `src/infra/home-dir.ts` to expand `$VAR` and `${VAR}` placeholders during shared path resolution and adds `src/infra/home-dir.test.ts` coverage for placeholder and edge-case behavior.

## Best Possible Solution

Land one focused shared path resolver update that expands env placeholders consistently before filesystem resolution, preserves intentional passthrough behavior for unresolved or unsafe cases, adds regression coverage across the resolver and ClawHub install path, then close #53628 and retire the unchosen overlapping PR.
