## Summary

Keep open. The ClawHub path bug still exists on current main, and this PR is a plausible implementation, but the shared resolver semantics need maintainer review, especially with overlapping PR #58281.

## What This Changes

The PR adds `$VAR` and `${VAR}` expansion to shared home-relative path resolution and adds resolver tests for braced, bare, unknown, inherited-key, empty-value, and tilde-valued placeholders.

## Best Possible Solution

Land one focused shared path resolver update that expands env placeholders consistently before filesystem resolution, preserves deliberate passthrough behavior for unresolved or unsafe placeholders, covers both resolver helpers and the ClawHub install path, then close #53628 and retire the unchosen overlapping PR.
