## Summary

The PR adds `$VAR` and `${VAR}` expansion to `resolveHomeRelativePath` and `resolveOsHomeRelativePath`, plus focused resolver tests for braced, bare, unknown, inherited-key, empty-value, and tilde-valued placeholders.

## Best Possible Solution

Land one focused shared path resolver update that expands env placeholders consistently before filesystem resolution, preserves deliberate passthrough behavior for unresolved placeholders, covers both resolver helpers and the ClawHub install path, then close #53628 and retire the unchosen overlapping PR.
