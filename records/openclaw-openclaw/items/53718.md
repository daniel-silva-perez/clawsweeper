## Summary

Keep open. Current main still does not expand `$VAR` or `${VAR}` in shared user path resolution, and this PR remains a plausible implementation candidate for the linked skill-install path bug. A newer open PR (#58281) appears to overlap, but there is no maintainer signal in the provided context making it canonical enough to close this PR as superseded.

## Best Possible Solution

Keep this PR open for maintainer review, compare it with #58281, and land one focused fix that expands env placeholders in shared user-path resolution with regression tests for braced `$ {VAR}`-style values, bare `$VAR`, unknown/empty env passthrough, inherited property names, and tilde-valued env values. After one PR merges and ships, close the other overlapping PR and the linked bug as implemented.
