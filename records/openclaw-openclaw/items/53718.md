## Summary

Keep open. Current main still resolves only blank, relative, absolute, and leading-tilde paths, so `$VAR` and `${VAR}` remain literal through `resolveUserPath`; this PR addresses a valid ClawHub install bug, but maintainers should choose one canonical implementation because #58281 overlaps the same resolver surface.

## What This Changes

This PR adds `$VAR` and `${VAR}` expansion to the shared home-relative path resolvers and adds focused tests for known, unknown, inherited, empty, and tilde-valued placeholders.

## Best Possible Solution

Land one focused shared resolver update that expands env placeholders consistently before filesystem resolution, preserves deliberate passthrough behavior for unresolved or unsafe placeholders, covers both resolver helpers and the ClawHub install path, then close #53628 and retire the unchosen overlapping PR.
