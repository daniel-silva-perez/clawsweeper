## Summary

Keep open. Current main still resolves `${VAR}` and `$VAR` path placeholders literally, so this PR remains a relevant implementation candidate for the open ClawHub skill-install bug. It should be reviewed alongside open alternative PR #58281 and closed-unmerged prior PR #56154 rather than closed as implemented or obsolete.

## What This Changes

This PR changes `src/infra/home-dir.ts` to expand `${VAR}` and `$VAR` environment placeholders during home-relative path resolution and adds `src/infra/home-dir.test.ts` coverage for braced, bare, unknown, empty, inherited-key, OS-home, and tilde-valued cases.

## Best Possible Solution

Keep this PR open for maintainer review. Compare it with #58281 and the closed prior attempt #56154, then land one focused `src/infra/home-dir.ts` fix that expands `${VAR}` and `$VAR` consistently before filesystem resolution while preserving intended unknown, empty, inherited-key, and tilde semantics.
