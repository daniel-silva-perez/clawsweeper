## Summary

Keep this PR open. Current main still does not implement the PR's intended session sync status observability: the memory status payload lacks `custom.sessionSync`, the CLI renderer has no session sync rows, JSON output therefore cannot include the proposed fields, and the current tests/docs do not cover that surface. The PR is stale against the current `extensions/memory-core` plugin layout, but the author did respond to the earlier timestamp and Map aggregation review comments, so it remains a plausible implementation candidate after a port.

## Best Possible Solution

Keep this PR open and ask for a rebase/port into `extensions/memory-core`. The best implementation is an additive typed session-sync status helper on the memory-core manager, surfaced through `MemoryProviderStatus.custom.sessionSync` or an equivalent backward-compatible extension field, with plain and JSON `openclaw memory status` output plus focused manager and CLI tests. Carry forward the author's fixes for timestamp consistency and Map aggregation, and add error sanitization before display or logging.
