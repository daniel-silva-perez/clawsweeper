## Summary

Keep this PR open. Current main still lacks the requested session sync observability in the memory status payload and `openclaw memory status` output. The PR is stale against the current `extensions/memory-core` plugin layout, but the provided discussion shows the author addressed the concrete timestamp and Map aggregation review bugs in `2df928fbc4df1b5e6f1137c04cb6cbb2bf547706`, so this remains a useful implementation candidate rather than cleanup-close material.

## Best Possible Solution

Keep this PR open and ask for a rebase into `extensions/memory-core`. The best implementation is a typed session-sync status helper on the memory-core manager, exposed under `MemoryProviderStatus.custom.sessionSync`, rendered by `openclaw memory status` including JSON and deep output, covered by focused manager and CLI tests, and carrying forward the PR's timestamp consistency and Map aggregation fixes while sanitizing displayed/logged error summaries.
