## Summary

Keep this PR open. Current main still does not implement the requested session sync observability in the active memory-core status payload, CLI renderer, docs, or focused tests. The PR remains a useful implementation candidate, but it targets the old `src/memory/*` and `src/cli/memory-cli.ts` layout rather than the current `extensions/memory-core` plugin surface. The prior bot review issues were acknowledged and fixed on the PR branch, so the remaining work is primarily a port plus maintainer review, not cleanup closure.

## Best Possible Solution

Keep this PR open and ask for a port into `extensions/memory-core`. The best implementation is an additive typed `MemoryProviderStatus.custom.sessionSync` object exposed in JSON status and rendered in plain `openclaw memory status`, with focused memory manager and CLI tests plus docs at `docs/cli/memory.md`. Carry forward the PR branch's timestamp consistency and `Map` aggregation fixes, and sanitize displayed/logged sync errors before merge.
