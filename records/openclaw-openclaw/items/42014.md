## Summary

Keep this PR open. Current main still does not implement the requested session sync status observability in the active memory-core status payload, plain/JSON CLI output, docs, or tests. The PR remains a focused implementation candidate, but it targets obsolete `src/memory/*` and `src/cli/memory-cli.ts` paths rather than the current `extensions/memory-core` plugin layout.

## Best Possible Solution

Keep this PR open and ask for a port into `extensions/memory-core`. The best implementation is an additive typed `MemoryProviderStatus.custom.sessionSync` object exposed in JSON status and rendered in plain `openclaw memory status`, with focused memory manager and CLI tests plus docs at `docs/cli/memory.md`. Carry forward the author's timestamp consistency and Map aggregation fixes, and sanitize displayed/logged sync errors before merge.
