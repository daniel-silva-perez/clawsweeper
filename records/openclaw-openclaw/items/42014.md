## Summary

Keep this PR open. Current main still does not implement session sync status observability in the memory status payload or CLI output, and the PR remains a plausible focused implementation candidate, but it must be ported from the old src/memory layout to extensions/memory-core before it can be reviewed for merge.

## Best Possible Solution

Keep this PR open and ask for a port into extensions/memory-core. The best implementation is an additive typed session-sync status helper surfaced through MemoryProviderStatus.custom.sessionSync or an equivalent backward-compatible extension field, with plain and JSON openclaw memory status output, focused manager and CLI tests, docs for the new user-visible status fields, and sanitized error display/logging. Carry forward the author's timestamp-consistency and Map aggregation fixes from the PR discussion.
