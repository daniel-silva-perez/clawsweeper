## Summary

Keep this PR open. Current main still does not expose session sync observability in the memory status payload, CLI output, tests, or docs. The PR remains a focused implementation candidate, but it targets the old pre-plugin memory paths and needs to be ported to the current memory-core plugin layout before it can be reviewed for merge.

## Best Possible Solution

Keep this PR open and ask for a port into extensions/memory-core. The best implementation is an additive typed session sync status object, likely under MemoryProviderStatus.custom.sessionSync, surfaced in JSON and plain openclaw memory status output with focused manager and CLI tests plus docs for the new user-visible fields. Carry forward the author's timestamp-consistency and Map aggregation fixes from the PR discussion, and sanitize displayed/logged sync errors.
