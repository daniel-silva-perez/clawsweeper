## Summary

Keep this PR open. Current main still does not implement session sync status observability in the memory status payload, JSON/plain CLI output, tests, or docs. The PR remains a focused implementation candidate, but it targets old `src/memory` and `src/cli/memory-cli.ts` paths that are no longer the active memory plugin layout.

## Best Possible Solution

Keep this PR open and ask for a port into `extensions/memory-core`. The best implementation is an additive typed `MemoryProviderStatus.custom.sessionSync` object surfaced in JSON and plain `openclaw memory status` output, with focused memory manager and CLI tests plus docs for the new fields. Carry forward the author’s timestamp-consistency and Map aggregation fixes from the review discussion, and sanitize displayed/logged sync errors before merge.
