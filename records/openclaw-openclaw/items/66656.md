## Summary

Keep open. Current main still derives `lastMessagePreview` from the latest user/assistant-shaped transcript tail row without skipping `diagnostic.*` entries, while this PR adds the missing filter, focused regression coverage, and contributor changelog credit. Related #66544 is complementary because it addresses display-name fallback, not transcript-preview derivation.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or land an equivalent focused change on main that skips `diagnostic.*` transcript entries before deriving `lastMessagePreview`, keeps the regression tests, and preserves the contributor changelog credit. Coordinate separately with #66544 because that PR fixes a different heartbeat display-name fallback path.
