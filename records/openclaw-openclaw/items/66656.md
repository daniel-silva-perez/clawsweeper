## Summary

Keep open. Current main still does not implement this PR's diagnostic transcript filtering for session previews: the tail scanner returns the latest user/assistant message without checking `type: "diagnostic.*"`, and the PR remains a focused candidate for the WebChat/Control UI heartbeat-preview bug. Related #66544 is complementary because it targets display-name fallback from `origin.label`, not `lastMessagePreview` derivation.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or land an equivalent focused change in `src/gateway/session-utils.fs.ts` that skips `diagnostic.*` transcript entries when deriving `lastMessagePreview`, with the included regression test and changelog credit. Coordinate separately with #66544 because it addresses a different display-name fallback path.
