## Summary

Keep open. Current main still derives `lastMessagePreview` from the latest user/assistant transcript row without filtering `diagnostic.*` entries, and the current tests do not cover diagnostic heartbeat/context rows. The provided PR diff adds the focused filter, regression coverage, and a changelog credit; related #66544 is complementary because it targets display-name fallback, not transcript-preview derivation.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or land an equivalent focused change on main that skips `diagnostic.*` transcript entries in `readLastMessagePreviewFromOpenTranscript`, keeps the regression tests, and preserves the contributor changelog credit. Coordinate separately with #66544 because that PR addresses a different display-name fallback path.
