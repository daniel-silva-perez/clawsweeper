## Summary

Keep open. Current main still derives `lastMessagePreview` from the newest user/assistant-shaped transcript tail row without excluding `diagnostic.*` entries, while this PR adds the missing helper-level filter, regression coverage, and changelog credit. Related #66544 is complementary because it addresses display-name fallback from origin metadata, not transcript-preview derivation.

## What This Changes

This PR skips `diagnostic.*` transcript entries when deriving session last-message previews, adds regression tests for diagnostic tail rows, and adds a changelog fix entry.

## Best Possible Solution

Land this PR or an equivalent focused patch on main so `diagnostic.*` transcript rows are ignored before deriving `lastMessagePreview`, keep the regression tests and contributor changelog credit, and review #66544 separately for the display-name fallback path.

## Reproduction Assessment

Yes. A source-level reproduction is a transcript whose last JSONL row has `type: "diagnostic.heartbeat"` and `message.role: "assistant"`; current main will return that row's content because it only checks `parsed.message.role`, while the PR test covers that exact shape.

## Solution Assessment

Yes. Filtering the diagnostic namespace inside the shared transcript-preview helper is the narrowest maintainable fix because it protects `sessions.list`, the agent sessions tool fallback, TUI, Control UI/WebChat consumers, and MCP conversation listings without changing normal preview extraction.
