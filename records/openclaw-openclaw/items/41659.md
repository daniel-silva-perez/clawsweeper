## Summary

Keep open. The PR targets a real remaining edge in the Control UI basePath path: current main still has root-relative favicon/apple-touch-icon links in `ui/index.html`, and gateway routing with a configured basePath does not handle root `/favicon.svg` requests. The branch is not merge-ready because it predates the current CSP-aware `serveResolvedIndexHtml` flow and must be rebased so the favicon rewrite preserves inline-script CSP hash handling.

## What This Changes

This PR adds a gateway HTML-serving helper that rewrites favicon and apple-touch-icon hrefs under `gateway.controlUi.basePath`, passes the base path through both index HTML serving paths, and adds gateway unit tests for rewrite and no-op behavior.

## Best Possible Solution

Rebase the favicon/apple-touch-icon rewrite onto the current `serveResolvedIndexHtml`, compute CSP hashes from the final served HTML, pass the normalized basePath through both direct index and SPA fallback paths, and add coverage that proves icon rewriting and CSP hash preservation together.

## Reproduction Assessment

Yes. At handler/source level, serve an index HTML containing `/favicon.svg` or `./favicon.svg` under `gateway.controlUi.basePath`; current main does not rewrite the link, and root `/favicon.svg` is outside the prefixed Control UI route.

## Solution Assessment

No, not as the branch currently stands. The runtime rewrite is a reasonable narrow direction, but the implementation must be rebased around the current CSP-aware HTML serving helper rather than replacing it with the older no-hash flow.
