## Summary

Keep this PR open. Current main still serves Control UI index HTML without rewriting favicon or apple-touch-icon hrefs for a configured basePath, while the source HTML still declares those icon links as root-relative paths. The PR remains a focused implementation candidate for the linked #41205 regression, with the earlier review concern addressed in its second commit.

## Best Possible Solution

Keep this PR open as a valid focused bug-fix candidate. Rebase it onto current main, integrate the favicon basePath rewrite into serveResolvedIndexHtml while preserving CSP hash/header behavior, keep the callback replacer, and run targeted gateway tests plus the changed gate before merge.
