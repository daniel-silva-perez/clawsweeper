## Summary

Keep this issue open. Current main still applies the cross-agent session visibility gate before the A2A allow policy for all actions, including `sessions_send`, and sandboxed label resolution still blocks cross-agent targets before A2A can authorize them. The provided GitHub context also shows open unmerged implementation candidate PR #57483 with closing syntax, so the issue should remain open until that candidate is reviewed, merged, or rejected.

## Best Possible Solution

Review and either land or close PR #57483. A complete fix should let A2A policy authorize cross-agent `sessions_send` under `tools.sessions.visibility=tree`, keep list/history/status read surfaces visibility-gated, and address sandboxed cross-agent label/session resolution with focused regression tests.
