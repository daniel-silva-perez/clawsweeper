## Summary

Keep open. Current main still blocks cross-agent `sessions_send` under non-`all` session visibility before the A2A policy can allow it, and it still restricts sandboxed cross-agent label resolution. There are also open PRs with closing syntax for this issue, so the issue should remain open until a candidate fix is merged or explicitly rejected.

## Best Possible Solution

Review and either land or close the open candidate PRs, with tests for `visibility=tree` plus A2A-allowed `sessions_send`, A2A-denied sends, and unchanged list/history/status isolation. The fix should also address sandboxed cross-agent label/session resolution or split that into a clear canonical follow-up.
