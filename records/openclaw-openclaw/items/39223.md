## Summary

Current main still asks `Where will the Gateway run?` before it handles `--section model`, so the reported configure path can still block before Anthropic model/auth setup. The direct `openclaw models auth ...` commands are real documented workarounds, but they do not fix the wizard path itself.

## Best Possible Solution

Keep this open and fix the configure path itself. The narrow implementation should let section-filtered model/auth configuration reach `promptAuthConfig` without first requiring the gateway-location select when the requested sections do not need gateway, daemon, channels, or health work. Add an early actionable non-interactive TTY guard for interactive configure paths that points Anthropic users to direct `openclaw models auth setup-token --provider anthropic` and `openclaw models auth paste-token --provider anthropic` workarounds, and update configure docs/tests so those direct auth commands remain documented as workarounds rather than the closing fix.

## Repair Work Prompt

Fix the configure wizard hang reported for section-filtered model/auth setup. On current main, `runConfigureWizard` asks `Where will the Gateway run?` before it handles `opts.sections`, so `openclaw configure --section model` can block before Anthropic auth setup. Update the wizard so section-filtered model/auth configuration can call `promptAuthConfig` without first requiring the gateway-location select when the requested sections do not need gateway, daemon, channels, or health work. Preserve existing behavior for full `openclaw configure` and for sections that genuinely need gateway mode. Add an early actionable non-interactive TTY guard for interactive configure paths that points Anthropic users to `openclaw models auth setup-token --provider anthropic` and `openclaw models auth paste-token --provider anthropic`. Add regression tests in `src/commands/configure.wizard.test.ts` proving `sections: ["model"]` reaches `promptAuthConfig` without consuming a gateway select, and keep or update tests for gateway/channel/daemon/health paths that should still request gateway mode. Update `docs/cli/configure.md` if behavior changes; keep direct `models auth` commands documented as workarounds, not as the fix itself. Add a user-facing changelog fix entry if this changes released configure behavior, crediting the reporter as `Thanks @LevityLeads`. Do not change provider auth semantics, token storage, or unrelated model selection behavior.
