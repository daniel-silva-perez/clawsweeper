## Summary

Keep this issue open. Current main has google-vertex as a bundled Google provider alias and supports per-model generic params, but it does not expose or consume a per-model Vertex location override. The current config schema rejects a top-level `location` under `agents.defaults.models[...]`, Vertex ADC auth still depends on a single `GOOGLE_CLOUD_LOCATION`, and the Google provider tests only cover model-id/baseUrl normalization, not per-model endpoint selection.

## Best Possible Solution

Keep this issue open and implement it as a Google-provider-owned per-model Vertex endpoint override. The likely path is to define a documented per-model config contract, resolve it with priority model override -> provider override -> `GOOGLE_CLOUD_LOCATION`, construct the correct global or regional Vertex base URL for google-vertex requests, and add tests covering preview/global plus GA/regional models coexisting under the same project/credentials.
