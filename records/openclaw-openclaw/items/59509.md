## Summary

Keep this issue open. Current main still has `google-vertex` only as part of the bundled Google provider path and does not expose or consume a per-model Vertex location override. The existing generic per-model `params` path exists, but there is no Google/Vertex resolver that turns model-level location into a global or regional Vertex endpoint, and the config schema still rejects the proposed top-level `location` field.

## Best Possible Solution

Keep this issue open and implement a Google-provider-owned per-model Vertex location contract. The likely fix is to resolve location with priority model override -> provider override -> `GOOGLE_CLOUD_LOCATION`, build the correct global or regional Vertex endpoint for `google-vertex`, document the chosen config shape, and add regression tests proving preview/global and GA/regional models can coexist under the same project and credentials.
