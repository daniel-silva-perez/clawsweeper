## Summary

Keep open. Current main still routes explicit `sessions_send` delivery requests through the generic `inter_session`/A2A path, still exposes broad skip-token guidance, and still suppresses visible delivery when the announce step returns `ANNOUNCE_SKIP`. Related #43848 overlaps the A2A skip/suppression area but is not a canonical replacement for this WhatsApp explicit-delivery bug.

## Best Possible Solution

Keep this issue open and fix the core `sessions_send`/A2A path with a backward-compatible delivery-intent signal or explicit prompt/runtime guard. Clear requests to emit a normal channel message should outrank generic `REPLY_SKIP`, `ANNOUNCE_SKIP`, and `NO_REPLY` guidance while preserving those tokens for real coordination chatter. Add a focused WhatsApp group regression covering authorized/always activation, explicit delivery wording, and forbidden skip-token outcomes.
