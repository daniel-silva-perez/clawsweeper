## Summary

Keep open. Current main still has no typed delivery intent or prompt/runtime carve-out for explicit channel-delivery requests; `sessions_send` routes the request through the same generic `inter_session`/A2A path where `REPLY_SKIP`, `ANNOUNCE_SKIP`, and `NO_REPLY` remain valid suppression outcomes. Related #43848 overlaps the A2A suppression area, but it is open and Telegram-topic-specific, so it is not a canonical replacement for this WhatsApp explicit-delivery issue.

## Best Possible Solution

Keep this issue open and fix the core sessions_send/A2A path with a backward-compatible delivery-intent signal or an explicit prompt/runtime guard. Clear requests to emit a normal channel message should outrank generic REPLY_SKIP, ANNOUNCE_SKIP, and NO_REPLY guidance while preserving those tokens for real coordination chatter. Add a focused WhatsApp group regression for authorized always/allowed activation, explicit delivery wording, and forbidden skip-token outcomes.
