## Summary

Keep this PR open. Current main still does not implement the PR's central behavior: inbound debounce buffers are not globally discoverable or flushable before SIGUSR1 restart, and there is no all-followup-queue drain waiter in the restart path. The PR remains a live implementation candidate, but it needs maintainer review because the proposed approach changes plugin-facing debounce contracts and has restart lifecycle/security concerns.

## Best Possible Solution

Keep this PR open as an implementation candidate, but require a narrower maintainer-reviewed revision. The best path is to add a lifecycle-safe, backwards-compatible drain surface for active inbound debouncers; preserve the existing plugin SDK contract; stop or gate external ingress early enough during restart; flush debounce buffers and then wait for active turns plus followup queue state, including dropped summaries; cover channel worker queues such as Discord; and add focused tests for SIGUSR1 ordering, followup drain behavior, teardown races, and SDK compatibility.

## Risks / Open Questions

- Current main can still lose messages buffered only in closure-local inbound debounce timers during SIGUSR1 teardown; the central user problem is not fixed on main.
- The PR's process-global debouncer registry is lifecycle-sensitive: stale handles, unregister ordering, and live ingress during restart can retain closures or make late buffers undiscoverable if not designed carefully.
- The PR changes a public plugin SDK-facing helper shape and flushKey semantics; third-party plugin compatibility needs an additive or versioned design.
- The proposed restart ordering has security/availability implications because accepting new external work during drain can extend restart work and incur extra agent cost; blocking new ingress must be balanced with allowing trusted internally flushed work to drain.
- The PR is broad and labeled dirty-candidate, with many channel/test files touched beyond the core restart path, so it should not be treated as a clean ready-to-land fix without maintainer review and green validation.
