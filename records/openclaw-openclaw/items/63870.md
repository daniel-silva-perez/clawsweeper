## Summary

Keep this PR open. Current main still does not expose `gateway.mcpLoopback.port`, and the remaining requested product behavior, an operator-configurable stable MCP loopback URL, is not implemented. Main partially addresses the related CLI session-reset symptom with `mcpResumeHash`, but that is not the same as a public gateway config field wired into lazy bundled-MCP startup.

## Best Possible Solution

Keep this PR open and require a rebase against current main. If maintainers still want operator-configured stable MCP URLs, add `gateway.mcpLoopback.port` to `GatewayConfig`, the strict Zod schema, config help/labels/generated schema, and the public gateway configuration docs; then pass `cfg.gateway?.mcpLoopback?.port ?? 0` into the current lazy `ensureMcpLoopbackServer(configuredPort)` path in `src/agents/cli-runner/prepare.ts`. Add tests for config parsing, prepared backend startup on a configured port, bind-failure diagnostics, and preservation of loopback-only bearer-token behavior. If maintainers decide `mcpResumeHash` fully supersedes the product need, record that explicit product decision or link a canonical follow-up before closing.
