## Summary

Keep this PR open. Current main still lacks the requested public `gateway.mcpLoopback.port` config surface and still starts bundled MCP with the default lazy `ensureMcpLoopbackServer()` path, so the intended stable operator-configured MCP URL is not implemented. The PR remains relevant, but the submitted branch is stale against current main and has a definite TypeScript scoping error in its latest diff.

## What This Changes

The PR adds `gateway.mcpLoopback.port` to gateway config types, schema/help/generated metadata, uses it when starting the MCP loopback server, and adds fixed-port loopback coverage plus a bind-failure log hint.

## Best Possible Solution

Rebase the PR onto current main and retarget the runtime wiring from the removed eager gateway startup path to the lazy CLI backend preparation path. The finished change should expose `gateway.mcpLoopback.port` consistently across types, strict schema, generated config metadata, help labels, and public docs, then feed that value into `ensureMcpLoopbackServer(configuredPort)` with focused tests for config validation, fixed-port startup, bind-failure logging, and unchanged loopback authentication behavior.
