## Summary

Keep this PR open. Current main still preserves structured connect-handshake failures for `onClose`, but pending RPC requests are still flushed with a generic `gateway closed (...)` error, so the PR addresses a real remaining UI gateway bug. The PR diff is narrow and does not introduce a security-sensitive surface.

## Best Possible Solution

Keep the PR open and review or recreate the same narrow fix on current main: in `GatewayBrowserClient`'s close handler, when `pendingConnectError` is present, reject pending requests with `new GatewayRequestError(pendingConnectError)` while preserving the generic close error for ordinary disconnects. Keep the targeted `ui/src/ui/gateway.node.test.ts` regression test for `details.code` propagation.
