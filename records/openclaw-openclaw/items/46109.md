## Summary

Keep open. Current main has a substantial SecretRef runtime, CLI, redaction/restore, and preflight foundation, but the requested first-class Control UI/config UX is still missing: structured SecretRefs still render as generic read-only placeholders, SecretInput schema analysis still collapses to string editing, and the docs still describe current limitations rather than source-aware env/config/unset controls.

## What This Changes

Request first-class `.env` / environment-backed secret configuration UX with masked source-aware display, env reference preservation, and clearer Control UI/CLI guidance.

## Best Possible Solution

Add a source-aware SecretInput renderer and metadata flow on top of the existing SecretRef contracts, with masked display, safe source labels, env variable identity where appropriate, resolved/unresolved status, explicit config/env/unset editing, and docs/tests aligned with the current secrets CLI and target registry behavior.
