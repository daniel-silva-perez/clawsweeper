## Summary

Keep open. Current main has a substantial SecretRef runtime/CLI/docs foundation and Control UI corruption guards, but it still does not deliver the requested first-class environment-backed secret UX: structured SecretRefs render as generic read-only redacted inputs, SecretInput unions are normalized to the string editor path, and the docs still describe raw/form limitations rather than source badges, resolved/unresolved status, or config/env/unset editing. The related merged PRs are useful foundation, not a full implementation of this issue.

## What This Changes

Request first-class `.env` and environment-backed secret configuration UX in the CLI and Control UI, including source-aware masked display, env-reference preservation, status visibility, and migration guidance.

## Best Possible Solution

Add a source-aware SecretInput renderer and supporting metadata flow on top of the existing SecretRef contracts: masked display, safe env variable labeling, resolved/unresolved status, explicit config/env/unset editing, and docs/tests that reuse the existing secret target registry and `openclaw secrets` audit/configure model.
