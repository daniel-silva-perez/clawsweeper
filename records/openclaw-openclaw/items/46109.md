## Summary

Keep this issue open. Current main has the SecretRef runtime, CLI, redaction/restore, preflight, and docs foundation, but the requested first-class Control UI/config UX is still missing: structured SecretRefs render as generic read-only placeholders, SecretInput schema analysis collapses to the string editor path, and the docs still describe those limitations rather than source-aware env/config/unset editing.

## What This Changes

Request first-class `.env` and environment-backed secret configuration UX that shows secret source state, preserves env references, and improves migration guidance in CLI and Control UI.

## Best Possible Solution

Add a source-aware SecretInput renderer and metadata flow on top of the existing SecretRef contracts, with masked display, safe source labels, env variable identity when appropriate, resolved/unresolved status, explicit config/env/unset editing, and aligned docs/tests for the Control UI and existing secrets CLI workflows.
