## Summary

Keep this issue open. Current main has a real SecretRef and env-template foundation plus config-write preflight checks, but the requested first-class Control UI/config UX is still missing: structured SecretRefs render as opaque read-only placeholders, SecretInput schema analysis still collapses to the plain string editor path, and the docs explicitly describe those form-mode limitations.

## What This Changes

Request first-class `.env` and environment-backed secret configuration UX with masked source-aware display, env/config/unset states, env-reference preservation, and CLI/Control UI guidance.

## Best Possible Solution

Add a source-aware SecretInput renderer and metadata flow on top of the existing SecretRef contracts: masked display, safe source labels, env variable identity where appropriate, resolved/unresolved status, explicit config/env/unset editing, and aligned docs/tests for Control UI, config RPC preflight, and the current secrets CLI workflows.
