## Summary

Keep this issue open. Current main has substantial SecretRef runtime, CLI, docs, and Control UI safety work, but the requested first-class env-backed secret UX is still not implemented: structured SecretRefs render as a generic redacted read-only value, SecretInput unions are still normalized to the string editor path, and the Control UI does not expose source badges, env/config/unset switching, resolved/unresolved status, or a dedicated secrets/environment helper for referenced credentials.

## Best Possible Solution

Keep this open for a focused Control UI/CLI design follow-up on top of the existing SecretRef contracts: render secret fields with `config` / `env` / `unset` source state, show only approved safe metadata and resolved/unresolved status, add controls for switching between inline config, structured SecretRef, and unset, and reuse the existing SecretRef target registry and `openclaw secrets` audit/configure model for any dedicated Secrets/Environment helper.
