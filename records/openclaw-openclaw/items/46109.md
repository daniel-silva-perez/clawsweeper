## Summary

Keep this open. Current main has the runtime SecretRef and CLI foundations, but the requested first-class source-aware Control UI/config UX is still missing and needs maintainer security/product judgment before implementation.

## What This Changes

The issue requests source-aware `.env` and environment-backed secret configuration UX in the CLI and Control UI, including masked env/config/unset state, env-reference preservation, and migration guidance.

## Best Possible Solution

Define a safe SecretRef metadata contract, then implement a source-aware SecretInput renderer and metadata flow that shows masked source labels, allowed env variable identity when appropriate, resolved/unresolved state, and explicit config/env/unset editing, with Control UI, CLI/docs, and regression tests aligned.

## Reproduction Assessment

Yes. The focused current-main check is the existing browser rendering path for an env SecretRef on `channels.discord.token`, which currently produces a read-only empty input with a generic Structured value placeholder.

## Solution Assessment

Unclear until maintainers set the disclosure boundary. The maintainable solution is a bounded metadata contract plus UI/docs/tests, not a placeholder-only change that guesses which secret-source details are safe to show.
