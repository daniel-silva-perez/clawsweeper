## Summary

Keep this PR open. Current main still has the agents overview picker gap the PR targets: `buildModelOptions` only uses `agents.defaults.models`, so provider-qualified default primary, fallback, or per-agent model refs can still appear through the synthetic `Current (...)` option unless they are also in the allowlist or catalog. The chat-header picker side is already covered on main, so the useful remaining work is the narrower overview fix plus regression coverage.

## Best Possible Solution

Refresh or reimplement the agents overview portion on current main: build overview picker options from the broader configured-model collector, preserve alias labels from `agents.defaults.models`, add the provider-qualified overview regression test, and leave the already-shipped chat-header model-ref path intact except for any needed test reconciliation.
