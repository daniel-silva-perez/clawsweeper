## Summary

Keep this PR open. Current main already covers the chat-header provider-qualified model picker behavior, but it still does not implement the PR's agents overview picker fix: the overview picker only collects options from `agents.defaults.models`, so a configured `agents.defaults.model.primary` or fallback can still appear as a synthetic `Current (...)` option instead of a normal selectable provider-qualified value.

## Best Possible Solution

Refresh or reimplement the narrow remaining agents overview change on current main: make the overview picker build configured options from the same broader collector used for cron suggestions, preserve `agents.defaults.models` alias labels, and add the provider-qualified overview regression test. Leave the newer chat-header implementation in place except for any small test reconciliation needed.
