## Summary

Keep this PR open. Current main still has the agents overview picker gap this PR targets: the broader configured-model collector exists, but the overview picker still builds options only from `agents.defaults.models` plus catalog entries, so provider-qualified primary, fallback, or per-agent refs can still fall through to a synthetic `Current (...)` option. The chat-header picker side is already covered on main, so the useful remaining work is the narrower overview fix and regression test.

## Best Possible Solution

Refresh or reimplement the agents overview portion on current main: make `resolveConfiguredModels` or `buildModelOptions` reuse the broader configured-model collector, preserve alias labels from `agents.defaults.models`, add the provider-qualified overview regression test, and leave the already-shipped chat-header model-ref path intact except for any needed test reconciliation.
