## Summary

Current main does not provide enough evidence to close this issue. The related report #72057 is closed as a duplicate of this issue, not a superseding tracker, and the current Google LLM path still goes through `buildGuardedModelFetch` with SSRF/proxy policy gates. Plain ambient `http_proxy`/`https_proxy` behavior remains a security-sensitive product decision because current docs describe opt-in proxy configuration paths instead.

## Best Possible Solution

Keep this issue open as the canonical Google LLM proxy-routing report. A maintainer should decide the intended contract first: either document and improve the supported workaround (`proxy.enabled` with `OPENCLAW_PROXY_URL` or `models.providers.google.request.proxy.mode=env-proxy`) and ensure failures surface clearly, or intentionally change the Google model fetch path to trust configured env proxy mode with focused SSRF/proxy regression coverage.
