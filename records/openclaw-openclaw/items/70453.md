## Summary

Current main still does not implement the requested Gemini/provider LLM ambient proxy behavior. Google LLM streaming uses `buildGuardedModelFetch`, and that path still calls `fetchWithSsrFGuard` in strict mode without the trusted env-proxy upgrade. #72480 is an open, focused implementation candidate, so this issue should remain open as the canonical report until that PR or an equivalent patch lands.

## Best Possible Solution

Keep this issue open as the canonical Google LLM ambient-proxy report. Maintainers should review and land #72480 or an equivalent narrowly gated patch in `buildGuardedModelFetch` that selects trusted env-proxy mode only when no explicit provider dispatcher policy exists and `shouldUseEnvHttpProxyForUrl(url)` is true, with focused regression tests. If ambient proxy env support is rejected for this path, document the supported provider proxy workaround and make Google LLM proxy failures surface explicit CLI/WebUI errors.
