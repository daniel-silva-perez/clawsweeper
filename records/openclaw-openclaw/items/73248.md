## Summary

Keep open. Current main has real mitigations for large skill catalogs, including prompt character limits, compact name/location fallback, truncation notices, deterministic sorting, and per-agent allowlists. It still injects a flat `<available_skills>` catalog and tells the model to scan descriptions, so the issue’s central request for hierarchical, tagged, or lazy skill discovery remains unimplemented.

## Best Possible Solution

Keep this open for a core skills prompt/discovery design. The best implementation would preserve existing budgets, truncation warnings, per-agent allowlists, plugin-bundled skills, and on-demand SKILL.md loading while adding a structured discovery layer such as categories/tags or lazy retrieval, with config schema, docs, and tests proving reduced prompt size and stable selection behavior.
