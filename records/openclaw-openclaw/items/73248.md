## Summary

Keep open. Current main still uses a flat `<available_skills>` catalog with compacting and prompt-budget mitigations; it does not yet provide the requested hierarchical, tagged, or lazy skill discovery design. The open related PR #68734 may address part of the problem through hook-driven narrowing, but it is unmerged and does not settle the broader discovery contract.

## What This Changes

Request to replace or augment flat skills prompt injection with hierarchical, tagged, or lazy skill discovery to reduce prompt overhead and model-selection noise for large skill sets.

## Best Possible Solution

Keep this as a core skills prompt/discovery design tracker. A good end state would add an additive, documented narrowing or discovery contract, such as tags/categories, hook-driven skill filtering, or lazy retrieval, while preserving deterministic ordering, explicit allowlists, prompt budgets, plugin-bundled skills, and SKILL.md on-demand loading.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the remaining structural behavior: current source and tests show skills are still represented as a flat `<available_skills>` prompt catalog. The exact 80-plus-skill token measurement is not reproduced because the issue does not include a fixture or logs.

## Solution Assessment

Is this the best way to solve the issue? Unclear at the design level: compact prompt budgeting is a useful mitigation, but it does not solve hierarchical or lazy discovery. The safer path is a reviewed additive API, potentially informed by #68734, rather than a broad rewrite of skill loading.
