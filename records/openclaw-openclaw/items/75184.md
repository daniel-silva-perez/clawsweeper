## Summary

Keep open. Current main still treats `agents.defaults.skipBootstrap` as a workspace-file creation flag while normal prompt injection is controlled by `contextInjection` and bootstrap character budgets; the generated schema also still exposes `skipBootstrap` as an unlabeled bare boolean. An open closing PR, #75217, is already paired with this issue, so this should remain open until that PR is reviewed and either merged or rejected.

## What This Changes

The issue reports that `agents.defaults.skipBootstrap: true` does not suppress existing workspace bootstrap files from normal agent prompt injection, leaving `systemPromptReport.injectedWorkspaceFiles` unchanged.

## Best Possible Solution

The config surface should have one unambiguous, tested contract: either `skipBootstrap` suppresses runtime bootstrap injection and the resolver, `systemPromptReport`, docs, generated schema/help, and tests all agree, or the flag stays creation-only and schema/help clearly direct prompt-size users to `contextInjection: "never"` plus the bootstrap budget knobs.

## Reproduction Assessment

Yes. The issue gives deterministic CLI steps, and current-main source inspection reproduces the root cause statically: existing workspace bootstrap files remain eligible for injection because the resolver path does not read `skipBootstrap`.

## Solution Assessment

Unclear until maintainers choose the intended contract. PR #75217 is a plausible implementation of the reporter's preferred behavior, but the narrowest compatibility-preserving repair would be a schema/help clarification if `skipBootstrap` is meant to remain creation-only.
