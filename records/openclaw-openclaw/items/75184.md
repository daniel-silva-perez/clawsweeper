## Summary

Keep open. Current main still has the reported mismatch: `skipBootstrap` is wired as a workspace bootstrap file creation flag, while prompt injection is controlled by `contextInjection` and bootstrap budgets. The public docs mostly describe that split, but the generated schema/help surface still exposes `skipBootstrap` as an unlabeled bare boolean, so the issue remains a narrow actionable config-surface cleanup rather than an implemented or superseded report.

## What This Changes

The issue asks for `agents.defaults.skipBootstrap` to either skip workspace bootstrap prompt injection or be clearly documented and named as a creation-only setting.

## Best Possible Solution

Keep `skipBootstrap` creation-only on current main, add schema label/help/description that says exactly that, update the `contextInjection` help text to include `"never"`, and point prompt-size users to `agents.defaults.contextInjection: "never"` plus bootstrap character budgets. Leave any runtime redesign to the tiered-bootstrap discussion.

## Reproduction Assessment

Yes. The issue has deterministic CLI steps, and source inspection gives a high-confidence static reproduction: `skipBootstrap` only reaches `ensureAgentWorkspace`, while the prompt injection helper ignores it and skips injection through `contextInjection: "never"`.

## Solution Assessment

No for repurposing `skipBootstrap` as an injection switch; that would conflict with its introduced and tested creation-only contract. Yes for a narrow schema/help/docs repair, which resolves the misleading config surface without changing runtime behavior.
