## Summary

Keep open. Current main still accepts `agents.defaults.skipBootstrap` but does not use it to suppress runtime workspace bootstrap injection, and the generated schema still exposes it as an unlabeled bare boolean. There is already an open contributor PR, #75217, with closing syntax for this issue, so the issue should remain open until that PR is reviewed and either merged or closed.

## What This Changes

The issue reports that `agents.defaults.skipBootstrap: true` does not prevent existing workspace bootstrap files from being injected into normal agent system prompts.

## Best Possible Solution

Resolve the open PR or an equivalent narrow change so runtime behavior and user-facing config metadata agree: either make `skipBootstrap` suppress injection and update docs/schema/tests accordingly, or preserve creation-only semantics and add schema/help text that points prompt-size users to `contextInjection: "never"` and bootstrap character budgets.

## Reproduction Assessment

Yes. The reporter provided deterministic CLI steps and `systemPromptReport` evidence, and current-main source inspection reproduces the root cause statically: `skipBootstrap` is wired to workspace file creation but not to `resolveBootstrapFilesForRun` or the embedded attempt injection skip path.

## Solution Assessment

Unclear. The linked PR is a plausible implementation candidate, but maintainers should first decide whether changing the runtime semantics is preferable to clarifying the existing creation-only flag and relying on `contextInjection: "never"` for injection control.
