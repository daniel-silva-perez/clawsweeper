## Summary

Keep open. Current main has a separate `agents.defaults.contextInjection: "never"` path that disables workspace bootstrap prompt injection, and prose docs describe `skipBootstrap` as creation-only, but the generated config schema still exposes `skipBootstrap` as a bare boolean with no title or description. The report's schema/discoverability problem remains valid even though the runtime already has a different injection-off knob.

## What This Changes

The issue requests that `agents.defaults.skipBootstrap` either skip workspace bootstrap prompt injection or be clearly scoped, renamed, or documented as affecting only bootstrap file creation.

## Best Possible Solution

Keep `skipBootstrap` creation-only, make the generated schema and UI help say that explicitly, and direct users who want zero workspace-file prompt injection to `agents.defaults.contextInjection: "never"`. Broader tiered loading should remain with #22438/#22439 rather than being folded into this narrow schema/discoverability fix.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: `skipBootstrap` is only wired through setup/onboarding workspace creation, while the prompt builder skips injection only through `contextInjection`; the generated schema still shows `skipBootstrap` without title or description.

## Solution Assessment

Yes for a narrow repair: schema labels/help plus docs clarification is the lowest-risk fix because current main already has `contextInjection: "never"` for injection suppression. Reusing `skipBootstrap` for runtime injection would conflate two existing config meanings and needs maintainer product judgment.
