## Summary

Keep open. Current main still treats `skipBootstrap` as a workspace-file creation flag, while prompt injection is controlled by `contextInjection` and bootstrap character budgets; the generated schema still exposes `skipBootstrap` as an unlabeled bare boolean, so the reported confusion remains actionable and is not superseded by the open tiered-bootstrap work.

## What This Changes

The issue reports that `agents.defaults.skipBootstrap` does not disable existing workspace bootstrap file injection and asks for runtime skip behavior, removal, or clearer config documentation.

## Best Possible Solution

Preserve the current creation-only meaning of `skipBootstrap`, add schema label/help that says it only suppresses automatic workspace bootstrap file creation, and point users who want prompt-size control to `agents.defaults.contextInjection: "never"` plus the bootstrap character-budget knobs. Runtime semantic changes should be left to the broader tiered-bootstrap discussion unless a maintainer explicitly chooses to repurpose the flag.

## Reproduction Assessment

Yes. The issue gives deterministic CLI steps, and current source inspection reproduces the root cause statically: `skipBootstrap` is only wired into `ensureBootstrapFiles`, while the prompt injection path ignores it.

## Solution Assessment

No, making `skipBootstrap` double as the injection-disable flag is not the best narrow fix because its origin, docs, and tests define creation-only behavior. The safer fix is a schema/help/docs clarification, with `contextInjection: "never"` remaining the supported injection-off control.
