## Summary

Keep open. The reported bug is real, but this PR needs updates before merge: current main routes gateway-rpc calls through the runtime module that the PR does not change, and the channels status patch would shorten omitted-timeout probe calls from 30s to 10s. The overlapping open timeout PRs make maintainer coordination and a rebase the right next step.

## What This Changes

The PR adds stricter CLI timeout parsing helpers, wires them into gateway, gateway-rpc, devices, and channels status timeout paths, and adds focused regression tests for invalid timeout rejection and passthrough behavior.

## Best Possible Solution

Land one rebased timeout-validation fix that rejects explicit invalid values at every affected gateway-facing CLI boundary, applies validation in the current active runtime module, preserves existing defaults including gateway-rpc 30s and channels status probe 30s, and keeps focused Commander plus direct-call regression coverage coordinated with #54646 and #59397.
