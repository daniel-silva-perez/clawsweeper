## Summary

Keep open. The PR is a useful focused timeout-validation fix, but it is not merge-ready against current main: the active gateway-rpc call path now lives in the lazy runtime file that this PR does not update, and the channels status change would shorten the existing probe fallback from 30s to 10s. The related open timeout PRs should be coordinated into one rebased canonical fix.

## What This Changes

The PR adds strict CLI timeout parsing helpers, wires them into gateway, gateway-rpc, devices, and channels status timeout paths, and adds focused regression tests for invalid timeout rejection and default passthrough.

## Best Possible Solution

Land one rebased timeout-validation fix that validates explicit invalid values at every affected gateway-facing CLI boundary, updates the active `gateway-rpc.runtime.ts` path, preserves existing defaults including gateway-rpc 30s and channels status probe 30s, and includes focused Commander plus direct-call regression coverage while coordinating #54646 and #59397.
