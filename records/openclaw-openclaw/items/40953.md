## Summary

Keep this PR open. Current main still has the gateway-facing timeout validation gap the PR targets: gateway call, gateway-rpc, devices, and channels status still accept raw string timeout options and pass `Number(...)` results toward `callGateway`. The related open PRs do not fully supersede it: #54646 covers gateway-rpc only, and #59397 covers the nodes CLI path outside this PR's stated scope. Security review of the provided PR file list found no unexplained workflow, dependency, package, secret, install, or release-surface changes; the diff is limited to CLI parsing/call sites and focused tests.

## Best Possible Solution

Keep this PR open for maintainer review or contributor rebase. The best path is to adapt the strict timeout parser to the current gateway-rpc runtime split, apply it consistently to the gateway call, gateway-rpc, devices, and channels status gateway-facing paths, preserve intended defaults, add focused regression tests for invalid input and valid passthrough, and coordinate with #54646 and #59397 so timeout-validation work converges cleanly.
