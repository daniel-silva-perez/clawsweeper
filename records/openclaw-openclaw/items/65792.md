## Summary

Current main still lacks the requested WebChat/WebSocket `message_sending` hook. The Gateway final chat path resolves buffered assistant text, flushes any pending delta, constructs a `state: "final"` payload, and sends it with `broadcast("chat", payload)` plus `nodeSendToSession(...)` without calling the plugin hook runner. The hook contract and other delivery paths already support `message_sending`, including optional `sessionKey`, so this remains a real consistency gap rather than a stale or duplicate issue.

## Best Possible Solution

Keep this issue open and implement a narrow Gateway WebChat fix: run `message_sending` for the final `jobState === "done"` WebSocket chat payload, pass `channelId: "webchat"` plus session/run correlation, apply rewritten content, honor cancellation, and isolate hook failures. Add focused Gateway chat tests for rewrite, cancel, thrown hook, and no-hook behavior, with an explicit decision on how final-only hooks interact with previously flushed deltas.

## Clownfish Work Prompt

Fix the WebChat/WebSocket `message_sending` hook gap described by this issue. Current `src/gateway/server-chat.ts` finalizes `jobState === "done"` chat output by directly broadcasting the final payload without running plugin hooks, while `src/infra/outbound/deliver.ts` and `src/auto-reply/dispatch.ts` already run `message_sending` for other delivery paths.

Expected scope:
- Update `src/gateway/server-chat.ts` so final WebChat delivery runs `getGlobalHookRunner()?.runMessageSending(...)` only when `hasHooks("message_sending")` is true.
- Use `channelId: "webchat"`; include `sessionKey`, run correlation where available, and a stable WebChat conversation/target value such as the session key.
- Apply returned `content` to the final assistant text.
- Honor `{ cancel: true }` by suppressing the final WebChat delivery, and explicitly test/handle the interaction with the existing pending-delta flush so a cancel does not newly flush text during finalization.
- Catch hook failures and continue delivering the original final text, matching the failure-isolation behavior of the shared outbound path.
- Do not change external channel delivery, plugin SDK types unless strictly necessary, or broader streaming semantics without focused tests.
- Add focused tests in `src/gateway/server-chat.agent-events.test.ts` for rewrite, cancel, thrown hook, and no-hook behavior. Mock the hook runner narrowly.
- Add a user-facing changelog fix entry if this repo’s current changelog policy requires one for plugin hook behavior; credit `@real3stone`.

Related refs: this issue and #38340 for adjacent prior discussion. Validate with the targeted Gateway chat test and changed gate according to OpenClaw policy.
