## Summary

Keep this PR open. Current main still resets the core model catalog cache on model-related hot reloads and then lets Gateway request handlers await the next cold catalog load, so the PR is relevant implementation work for the linked Gateway responsiveness report rather than obsolete cleanup.

## What This Changes

The PR adds a Gateway stale-while-revalidate model catalog wrapper, marks that Gateway cache stale during model-related hot reloads, adds retry/race coverage, and updates the changelog plus generated Plugin SDK API baseline hash.

## Best Possible Solution

Review and, if the branch checks out, land a narrowly scoped Gateway stale-while-revalidate catalog boundary that preserves the last successful catalog for latency-sensitive Gateway consumers while refreshing after hot reload. Keep the public Plugin SDK surface intentional and compiler-checked, and keep broader provider-discovery/off-thread work in separate follow-up issues if needed.
