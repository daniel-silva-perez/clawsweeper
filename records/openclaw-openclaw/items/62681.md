## Summary

Keep this issue open. Current main has Control UI i18n and Telegram custom menu entries, but the Telegram native command path still registers one English built-in command catalog and does not support per-locale command descriptions or Telegram `language_code` registration. Existing plugin/config seams do not let a ClawHub plugin localize the built-in Telegram command menu without a core/bundled Telegram change.

## Best Possible Solution

Keep this open as a Telegram command-surface/i18n feature request. Best path: add a backwards-compatible localized command metadata layer for built-in and plugin commands, register Telegram per-language command descriptions with `setMyCommands`, keep English command names as the stable default unless explicit alias semantics are designed, and update docs/tests for the Telegram config and Control UI command surfaces.
