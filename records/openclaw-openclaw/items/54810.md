Keep this issue open. Current main still lacks the streaming markdown fence handling proposed for the reported code-indentation bug, and the linked fix PR #73485 is closed without being merged.

The fix should prove that streaming/unclosed fenced code renders as a code block with leading indentation preserved before the final history refresh.
