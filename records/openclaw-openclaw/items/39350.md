## Summary

Keep this PR open. Current main still does not implement the intended lifecycle-end Gateway final-message fix: a suppressed `NO_REPLY` lead fragment can remain in the chat buffer, then finalization reprojects it with lead-fragment suppression disabled and emits it as a final assistant message. The PR remains the active implementation candidate, but the clientRunId-scoped suppression state noted by Aisle needs maintainer resolution before merge.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate. Resolve the clientRunId scoping concern, keep regression coverage for lifecycle-end `NO_`/`NO_RE` suppression and intentional uppercase `NO` preservation, then run the normal Gateway changed gate before merge.
