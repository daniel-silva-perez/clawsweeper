## Summary

Keep this PR open. Current main still does not implement the intended Gateway final-message suppression for lifecycle-ending NO_REPLY lead fragments: deltas buffer suppressed fragments, finalization explicitly disables lead-fragment suppression, and final chat payload emission only gates on exact/control suppression. The PR remains the relevant narrow fix candidate, but its latest provenance-state approach needs maintainer review because the new suppression state is keyed by caller-provided clientRunId/idempotencyKey.

## Best Possible Solution

Keep this PR open as the canonical narrow Gateway fix candidate. Revise or confirm the suppression provenance so it is scoped to a session/source run or server-owned run identity, keep the lifecycle-ending NO_ or NO_RE regression coverage plus the intentional uppercase NO preservation test, then run the targeted Gateway test and the normal changed gate before merge.
