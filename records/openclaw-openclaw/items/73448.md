## Summary

Keep open. Current main still exposes `asVoice` on the shared message tool schema but does not propagate it through the shared send runner, `executeSendAction`, `sendMessage`, or the gateway send schema. A linked open PR already claims this issue with closing syntax, so the issue should remain open until that PR is reviewed and merged or replaced.

## What This Changes

The issue requests that the shared message send path read the message tool `asVoice` parameter and preserve it as `audioAsVoice` for channel delivery.

## Best Possible Solution

Review and land PR #73483 if it is still correct, or replace it with an equivalent narrow fix that carries `asVoice` as `audioAsVoice` through the shared message action runner, outbound send service, `sendMessage`, gateway send contract, and focused regression tests.
