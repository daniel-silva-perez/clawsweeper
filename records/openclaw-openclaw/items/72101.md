## Summary

Keep open. Current main and v2026.4.27 already tolerate ACP-only `streamTo`/`resumeSessionId` on native `sessions_spawn`, but current main still forwards `model:"default"` as a literal native subagent model override, so this PR still carries useful remaining work.

## What This Changes

The PR adds shared model override normalization, applies it to `sessions_spawn` and `session_status`, gates `streamTo` to ACP runtime, and adds `sessions_spawn` regression coverage for `model:"default"`.

## Best Possible Solution

Land a rebased version that preserves the already-shipped ACP-only field tolerance and adds the missing native `sessions_spawn` `model:"default"` normalization, sharing the helper with `session_status` only if ACP model reset semantics are intentionally shared.

## Reproduction Assessment

Yes. The source-level reproduction is a native `sessions_spawn` call with `model:"default"`: current main reads that string unchanged, passes it to `spawnSubagentDirect`, and model selection treats it as the first non-empty override instead of falling back.

## Solution Assessment

Mostly yes. Normalizing empty/default model input at the tool boundary is narrow and matches `session_status`; the safer refinement is to either document/test the ACP `model:"default"` behavior or scope normalization to native subagent spawns if ACP should preserve literal runtime model strings.
