## Summary

Keep open. Current main already covers the ACP-only `streamTo`/`resumeSessionId` tolerance, but it still forwards `sessions_spawn` `model:"default"` as a literal subagent model override, so this PR remains useful for the remaining behavior. The patch is narrowly scoped and I did not find a blocking correctness issue in the proposed diff.

## What This Changes

The PR adds shared model override normalization, applies it to `sessions_spawn` and `session_status`, gates `streamTo` to ACP runtime, and adds `sessions_spawn` regression coverage for `model:"default"`.

## Best Possible Solution

Land a rebased, focused version that preserves the shipped ACP-only field tolerance and adds the missing native `sessions_spawn` `model:"default"` normalization, with regression coverage and shared helper use only where the reset semantics are intentionally common.

## Reproduction Assessment

Yes. A native `sessions_spawn` call with `model:"default"` is enough: current main reads that string unchanged, passes it to `spawnSubagentDirect`, and model selection treats the non-empty override as the first choice instead of falling back.

## Solution Assessment

Mostly yes. Normalizing empty/default model input at the tool boundary is the narrow maintainable fix; the safer refinement is to explicitly decide whether the same helper should apply to ACP model overrides.
