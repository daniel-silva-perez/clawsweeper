## Summary

Keep open. Current main and v2026.4.27 already cover the ACP-only `streamTo` tolerance, but current main still forwards `model:"default"` as a literal native subagent model override, so this PR still has useful remaining work.

## What This Changes

The PR adds shared tool model-override normalization, applies it to `sessions_spawn` and `session_status`, gates `streamTo` to ACP runtime, and adds `sessions_spawn` regression coverage for `model:"default"`.

## Best Possible Solution

Land a rebased version that preserves the shipped ACP-only field tolerance and adds the missing native `sessions_spawn` `model:"default"` normalization, with ACP `model:"default"` behavior either intentionally covered or scoped out.

## Reproduction Assessment

Yes. The source-level reproduction is a native `sessions_spawn` call with `model:"default"`: current main reads that string unchanged, passes it to `spawnSubagentDirect`, and model selection treats it as the first non-empty override before falling back.

## Solution Assessment

Mostly yes. Normalizing empty/default model input at the tool boundary is narrow and matches `session_status`; the safer refinement is to document/test the ACP behavior or limit normalization to native subagent spawns if ACP should preserve literal runtime model strings.
