## Summary

Keep open. Current `main` still does not auto-select the only enabled account for omitted-account message sends. The shared message runner preserves only explicit/default/bound account IDs, channel selection remains channel-only, and WhatsApp omitted-account sends still fall back to the configured/default account resolver. The related implementation PR #20801 is closed unmerged, so this remains a valid narrow fix candidate.

## Best Possible Solution

Implement a shared outbound message-account resolver after channel selection. Preserve explicit `accountId`, `input.defaultAccountId`, and agent binding first; then use the selected channel plugin config adapter to prefer an enabled configured default account, auto-select exactly one enabled/configured account, and throw clear zero/multiple-enabled-account errors. Add focused shared-runner/channel-selection tests, WhatsApp omitted-account regression coverage, routing/message docs updates, and a user-facing changelog fix entry.
