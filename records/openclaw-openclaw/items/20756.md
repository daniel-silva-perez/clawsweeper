## Summary

Keep open. Current main still does not auto-select the only enabled account for omitted-account message tool sends; the shared runner only preserves explicit/tool-default/agent-binding account IDs, WhatsApp still falls back to its default-account resolver, and the prior fix PR #20801 is closed unmerged.

## Best Possible Solution

Keep this issue open and implement a shared outbound message-account resolver after channel selection. Preserve explicit `accountId`, tool/default account, and agent-binding precedence; then prefer an enabled configured default, auto-select exactly one enabled account, and return clear errors for zero or multiple enabled accounts. Add focused runner/channel-selection regression tests plus WhatsApp coverage and update message/channel routing docs.
