## Summary

Keep open. Current main has Android chat, image attachments, assistant-launch routing into Chat, and Tailscale-aware connection guidance, but the official Android app still defaults and documents a Connect-first companion-node flow; the remaining request is Android product direction plus possible focused upstreaming, not completed cleanup.

## What This Changes

The issue requests maintainer discussion about making the official Android app chat-first or selectively upstreaming small Android UX, media, and connectivity improvements from independent forks.

## Best Possible Solution

Android/mobile maintainers should first choose the official app shape: default to Chat, add an explicit chat-first mode, or keep Connect-first while inviting focused PR slices for chat UX, media playback, and Tailscale/connectivity improvements. Broad fork imports should stay out of scope, and #70993 can be reviewed as one possible chat UX slice.
