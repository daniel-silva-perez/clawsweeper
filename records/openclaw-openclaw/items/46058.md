## Summary

Keep open. Current main has meaningful Android chat support, assistant-launch routing into Chat, image attachments, and Tailscale-aware connection guidance, but the official Android app still implements and documents a Connect-first companion-node flow. The linked external Android projects and open chat-parity PR are useful product context, not a shipped resolution of the requested Android direction.

## What This Changes

The issue asks whether OpenClaw maintainers want an official chat-first Android mode/surface or selective upstreaming of focused Android UX, media, and connectivity improvements from an external fork.

## Best Possible Solution

Android/mobile maintainers should choose the official product shape first: default to Chat, add an explicit chat-first mode, or keep Connect-first while inviting narrow PR slices for chat UX, media playback, and Tailscale/connectivity improvements. Broad fork imports should stay out of scope, and #70993 can be evaluated as one possible chat-UX slice.
