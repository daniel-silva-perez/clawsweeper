## Summary

Keep open. Current main has real Android chat support, assistant-launch routing into Chat, image attachments, and Tailscale-aware connection guidance, but the official Android app still defaults and documents a Connect-first companion-node flow. The remaining ask is Android product direction and possible focused upstreaming from external chat-first Android work, not cleanup that is already implemented.

## What This Changes

The issue asks maintainers to discuss whether the official Android app should add a chat-first surface or accept focused UX, media, and connectivity improvements from an independent Android fork.

## Best Possible Solution

Android/mobile maintainers should choose the official product shape first: keep Connect-first, make Chat the default after pairing, or add an explicit chat-first mode. Once that direction is set, focused PR slices for chat UX, media playback, and Tailscale/connectivity improvements can be reviewed without importing a whole fork.

Not applicable as a bug reproduction. The current product state is high-confidence from source and docs: post-onboarding defaults to Connect, docs describe companion-node pairing, and chat/media support exists but remains image-oriented.
