## Summary

Keep open. Current main has meaningful Android chat support, image attachments, assistant-launch routing into Chat, and Tailscale-aware connection guidance, but the official Android app still implements and documents a Connect-first companion-node flow. The linked external Android projects and open chat-parity PR are useful evidence, not a resolution of the requested Android product direction.

## What This Changes

The issue asks maintainers to discuss either a chat-first official Android mode/surface or focused upstreaming of Android UX, media, and connectivity improvements from an independent fork.

## Best Possible Solution

Android/mobile maintainers should choose the official shape first: default to Chat, add an explicit chat-first mode, or keep Connect-first while inviting focused PR slices for chat UX, media playback, and Tailscale/connectivity improvements. Broad fork imports should stay out of scope, and #70993 can be reviewed as one possible chat-UX slice.
