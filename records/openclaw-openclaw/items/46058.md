## Summary

This issue should stay open for maintainer/product judgment. Current main has an official Android Chat tab and several related connection/media pieces, but it does not settle the requested chat-first Android direction or focused upstreaming guidance.

## Best Possible Solution

Keep the issue open and have the Android maintainer/product owner decide the direction. The clean path is to ask the contributor to split the fork learnings into narrow proposals or PRs: one for chat-first/default navigation, one for media receive/render/playback behavior, and one for Tailscale/manual connection UX, using the linked external Android projects as prior art.
