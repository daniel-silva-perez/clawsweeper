## Summary

Keep open. Current main partially covers the requested direction with a functional Android Chat tab, assistant-launch routing into Chat, image attachments, and Tailscale-aware connection guidance, but the official app still launches into and documents a Connect-first companion-node flow. The linked external Android projects and open chat-parity PR are useful product context, not a shipped resolution of the Android direction being requested.

## What This Changes

The issue asks maintainers to discuss a chat-first Android mode or selective upstreaming of focused Android chat, connectivity, and media UX improvements from an external fork.

## Best Possible Solution

Android/mobile maintainers should decide the official product shape first: default to Chat, add an explicit chat-first mode, or keep Connect-first while inviting narrow PR slices for chat UX, media playback, and Tailscale/connectivity improvements. Broad fork imports should stay out of scope, and #70993 can be evaluated as one possible chat-UX slice.
