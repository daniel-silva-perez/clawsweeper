## Summary

Keep open. Current main has substantial Android chat, assistant-intent routing into Chat, image attachment support, voice/camera features, and Tailscale-aware connection handling, but the official Android app still documents and implements a Connect-first companion-node flow rather than the requested chat-first product direction. The linked external Android projects are useful reference context, not evidence that OpenClaw has made or shipped the official app decision.

## What This Changes

This issue asks maintainers to consider an official chat-first Android mode/surface or focused upstreaming of validated Android UX, media, and connectivity improvements from independent forks.

## Best Possible Solution

Keep this open for Android/mobile maintainer triage: decide whether the official Android app should default to Chat, offer an explicit chat-first mode, or only accept narrower UX/media/connectivity slices, then ask contributors for small PRs against the existing Android app surfaces rather than importing a broad fork.
