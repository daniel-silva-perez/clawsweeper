## Summary

Keep open. This is still a broad Android/mobile product-direction discussion, not an already-implemented narrow bug fix. Current main has substantial Android chat, assistant-intent routing into Chat, image attachments, voice/camera support, and Tailscale-aware Connect docs, but the official app still starts post-onboarding in a Connect-first companion-node flow and is documented as unreleased alpha. The linked clawchat2-public fork and AndroidForClaw comment are useful reference context, not evidence that OpenClaw has made the requested product decision.

## What This Changes

Request to discuss a chat-first official Android surface or selectively upstream small validated Android UX/media/connectivity improvements from an independent Android fork.

## Best Possible Solution

Keep this open for Android/mobile maintainer triage. Maintainers should decide whether the official Android app should add a chat-first default or explicit chat-first mode, then invite focused PR slices against existing Android surfaces such as PostOnboardingTabs, RootScreen/MainActivity assistant launch handling, ChatSheetContent/ChatController, ConnectTabScreen/Gateway docs, and chat media rendering. Do not import a broad fork wholesale.
