## Summary

Keep open. Current main has meaningful Android chat support, image attachments, assistant-launch routing into Chat, and Tailscale-aware pairing guidance, but the official Android app still defaults to and documents a Connect-first companion-node flow. The linked external Android projects and open chat UX PR are useful product evidence, not a shipped resolution of the requested Android direction.

## What This Changes

The issue asks maintainers to discuss a chat-first Android mode or surface, or selective upstreaming of focused Android UX, media, and connectivity improvements from an independent fork.

## Best Possible Solution

Android/mobile maintainers should choose the official app shape first: default to Chat, add an explicit chat-first mode, or keep Connect-first while inviting focused PR slices for chat UX, media playback, and Tailscale/connectivity improvements. Broad fork imports should stay out of scope, and #70993 can be evaluated as one possible chat-UX slice.
