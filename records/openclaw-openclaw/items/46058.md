## Summary

Keep open. Current main has a real Android Chat tab, assistant-intent routing into Chat, image attachment support, voice/camera features, and Tailscale-aware connection guidance, but the official Android app still documents and implements a Connect-first companion-node flow rather than the requested chat-first Android product direction. The linked external Android projects and the related open chat-parity PR are useful context, but they do not resolve whether OpenClaw should default Android to Chat, add an explicit chat-first mode, or accept only narrower UX/media slices.

## What This Changes

The issue asks maintainers to discuss an official chat-first Android mode or accept focused Android UX, media, and connectivity improvements from an independent fork.

## Best Possible Solution

Android/mobile maintainers should decide the official shape first: default to Chat, add an explicit chat-first mode, or keep Connect-first and invite focused PR slices for chat UX, media playback, and Tailscale/connectivity improvements. Related PR #70993 should be reviewed as one possible chat-UX slice, while broader fork imports should stay out of scope.
