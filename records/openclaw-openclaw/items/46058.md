## Summary

Keep open. This is a broad Android product-direction discussion, not a stale bug or an already-completed narrow request. Current main has meaningful Android chat, image attachment, voice/camera, Gateway, and Tailscale-related support, but the official Android app still defaults post-onboarding to a Connect-first multi-tab companion-node flow and is still documented as unreleased alpha. The external fork and AndroidForClaw comment are useful maintainer context rather than evidence that OpenClaw core can close the discussion.

## Best Possible Solution

Keep this issue open for Android/mobile maintainer triage. The best path is for maintainers to decide whether the official Android app should add a chat-first default or explicit chat-first mode, then invite focused PR slices against existing Android surfaces such as PostOnboardingTabs/RootScreen, ChatSheetContent/ChatController, ConnectTabScreen/Gateway docs, and media rendering. Accept small, testable UX/media improvements with Android unit tests and docs updates where behavior changes; do not import a broad fork wholesale.
