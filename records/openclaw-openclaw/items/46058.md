## Summary

Keep open. Current main has meaningful Android chat support and Tailscale-aware pairing guidance, but the official Android app still defaults and documents a Connect-first companion-node flow; the remaining ask is Android product direction and possible focused upstreaming from external chat-first Android work.

## What This Changes

The issue asks maintainers to discuss a chat-first Android mode or selective upstreaming of validated Android UX, media, and connectivity improvements from an external fork.

## Best Possible Solution

Android/mobile maintainers should choose the official shape first: keep Connect-first, make Chat the default after pairing, or add an explicit chat-first mode. After that, focused PR slices for chat UX, media playback, and Tailscale/connectivity can be reviewed without importing a whole fork.

## Reproduction Assessment

Not applicable as a bug reproduction. The current product state is high-confidence from source and docs: post-onboarding defaults to Connect, docs describe companion-node pairing, and chat/media support exists but remains image-oriented.

## Solution Assessment
