## Summary

Keep open. Current main has meaningful Android chat support, assistant-launch routing, image attachments, and Tailscale-aware connection guidance, but the official Android app still defaults to a Connect-first companion-node flow and the broader chat-first/media direction remains an Android product decision rather than an implemented cleanup item.

## What This Changes

The issue asks maintainers to consider a chat-first Android app surface or focused upstreaming of validated Android UX, media, and connectivity improvements.

## Best Possible Solution

Android/mobile maintainers should choose the official app shape first: default to Chat, add an explicit chat-first mode, or keep Connect-first while inviting focused PR slices for chat UX, media playback, and Tailscale/connectivity improvements. Broad fork imports should stay out of scope, and #70993 can be reviewed as one possible chat-UX slice.

## Reproduction Assessment

Not applicable as a bug reproduction. The current product state is high-confidence from source and docs: post-onboarding defaults to Connect, docs describe companion-node pairing, and chat/media support is present but image-oriented.

## Solution Assessment

No specific proposed patch is available in this issue, so there is no narrow implementation to bless. The maintainable path is a maintainer product decision followed by small Android PR slices, not an automated broad fork import.
