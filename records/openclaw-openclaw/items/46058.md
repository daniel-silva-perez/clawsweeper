## Summary

Keep open because current main has useful Android chat support, image attachments, assistant-launch routing, and Tailscale-aware pairing guidance, but it still presents and documents Android as a Connect-first companion-node app. The remaining ask is an Android product direction decision plus possible focused media/UX slices, not something already shipped or narrow enough to auto-close.

## What This Changes

The issue asks maintainers to discuss a chat-first Android app surface or accept focused upstream PR slices for chat UX, connectivity, and media handling.

## Best Possible Solution

Android/mobile maintainers should choose the official app shape first: default to Chat, add an explicit chat-first mode, or keep Connect-first while inviting focused PR slices for chat UX, media playback, and Tailscale/connectivity improvements. Broad fork imports should remain out of scope, with #70993 evaluated as one possible chat-UX slice.
