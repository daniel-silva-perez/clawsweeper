## Summary

Keep this PR open. Current main still does not implement WhatsApp inbound edit/revocation handling: the WhatsApp monitor only subscribes to `messages.upsert` and `connection.update`, and there is no `messages.update`, `editedMessage`, pending-message, or processed-message path in the current source or tests. The PR is a plausible implementation candidate, but it is not obsolete and still needs review fixes before it should land.

## Best Possible Solution

Keep the PR open and revise it rather than close it. The best path is a focused WhatsApp plugin change that adds `messages.update` handling with explicit buffered, in-flight, and processed states; imports or routes logging through existing runtime helpers; keeps late-edit notifications gated by successful inbound access-control and delivery; and adds targeted monitor tests for edit, revoke, self-chat, unauthorized/historical messages, and in-flight timing before running the changed WhatsApp checks.
