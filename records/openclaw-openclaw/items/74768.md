## Summary

Keep open for maintainer review. Current main still ignores legacy `embeddedHarness` in the shared runtime-policy resolver, while the PR addresses that central gap and adds focused tests for harness selection, startup plugin activation, and `sessions_spawn` staying on the native sub-agent route unless ACP is explicit.

## What This Changes

The PR maps legacy `embeddedHarness` runtime and fallback fields into `resolveAgentRuntimePolicy` and adds tests for native Codex app-server runtime selection, startup activation, and `sessions_spawn` routing.

## Best Possible Solution

Land this PR or an equivalent shared-resolver compatibility change after maintainer review, keeping explicit `agentRuntime` precedence, legacy field compatibility, plugin enablement semantics, and the native Codex versus explicit ACP routing boundary intact.

## Reproduction Assessment

Yes. Against current main, a config that only sets `embeddedHarness: { runtime: "codex", fallback: "none" }` reproduces by source inspection because the shared resolver ignores `embeddedHarness`, and both harness selection and startup activation consume that resolver.

## Solution Assessment

Yes. Updating the shared resolver is the narrowest maintainable solution because the affected consumers already centralize runtime policy there, and the PR keeps `sessions_spawn` ACP dispatch gated on an explicit `runtime: "acp"` argument.
