## Summary

Keep open. Current main still ignores legacy `embeddedHarness` in the shared agent runtime policy resolver even though that config remains typed and documented as legacy input, and this PR is a focused compatibility fix with no blocking correctness or security finding from the reviewed diff.

## What This Changes

The PR extends shared agent runtime policy resolution to derive missing policy fields from deprecated `embeddedHarness` config and adds harness selection, startup plugin, and `sessions_spawn` tests for native Codex app-server routing.

## Best Possible Solution

Land this PR or an equivalent shared-resolver compatibility change after maintainer review, preserving explicit `agentRuntime` precedence, plugin disablement semantics, and the documented split between native Codex app-server runs and explicit ACP/acpx sessions.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to configure an agent or defaults with only legacy `embeddedHarness`; on current main the shared resolver ignores that field, while harness selection and startup plugin activation consume only the resolver output.

## Solution Assessment

Yes. Resolving legacy compatibility in `resolveAgentRuntimePolicy` is the narrowest maintainable fix because the affected harness selection, metadata, startup activation, and route-preservation tests already converge on that shared policy path.
