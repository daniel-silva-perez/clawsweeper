## Summary

Keep open. Current main still has the legacy runtime-policy compatibility gap, while this PR is a narrow implementation with no concrete blocking review or security finding; the next step is maintainer code review and normal merge validation.

## What This Changes

The PR updates runtime-policy resolution to fold legacy `embeddedHarness` runtime/fallback values into missing `agentRuntime` fields, with harness-selection, startup-plugin, and `sessions_spawn` tests for native Codex routing.

## Best Possible Solution

Land this PR or an equivalent shared-resolver compatibility change after maintainer review, preserving explicit `agentRuntime` precedence, legacy fill-missing behavior, plugin disablement semantics, and the native Codex versus explicit ACP routing split.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: a config that only sets legacy `embeddedHarness` can be accepted and migrated by doctor, but the shared resolver ignores it before harness selection and startup activation consume policy.

## Solution Assessment

Yes. Centralizing the compatibility behavior in `resolveAgentRuntimePolicy` is the narrowest maintainable approach because the affected metadata, harness-selection, model-runtime, and startup-plugin paths already depend on that resolver.
