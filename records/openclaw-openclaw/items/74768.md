## Summary

Keep open for maintainer review. Current main still ignores legacy `embeddedHarness` in the shared runtime-policy resolver even though the config contract and doctor migration still support it; the PR targets that shared resolver and adds downstream regression coverage. I found no definite correctness or security defect in the proposed patch.

## What This Changes

The PR updates shared agent runtime policy resolution to merge deprecated `embeddedHarness` fields into `agentRuntime`, with regression tests for harness selection, startup plugin activation, and `sessions_spawn` routing.

## Best Possible Solution

Land this PR or an equivalent shared-resolver compatibility change after maintainer review, preserving explicit `agentRuntime` precedence, plugin enable/disable semantics, and the native Codex versus explicit ACP routing boundary.

## Reproduction Assessment

Yes. A current-main config that only sets legacy `embeddedHarness: { runtime: "codex", fallback: "none" }` reproduces by inspection because the shared resolver ignores `embeddedHarness`, while harness selection and startup activation both read that resolver.

## Solution Assessment

Yes. Updating the shared resolver is the narrowest maintainable fix because all affected consumers already centralize on it, and the merge behavior matches the existing doctor migration semantics.
