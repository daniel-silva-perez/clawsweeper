## Summary

Keep this PR open. Current main still has the reported backup output-path behavior, and the PR’s useful logic needs to be rebased into the active backup resolver rather than closed as implemented or obsolete.

## Best Possible Solution

Keep this PR open for maintainer follow-up. Rebase the useful behavior into `src/infra/backup-create.ts`: treat missing extensionless output paths as destination directories, keep existing files and non-empty suffixes such as `.tar` and `.bak` as explicit archive filenames, rethrow non-ENOENT stat errors, update backup help/docs if that is the intended contract, and add focused regression tests around the resolver plus existing safety checks.
