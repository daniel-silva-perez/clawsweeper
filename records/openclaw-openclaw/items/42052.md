## Summary

Keep this PR open. Current main still treats a missing extensionless backup output path as the final archive file path, while this PR’s patch targets the old command wrapper instead of the active resolver now in `src/infra/backup-create.ts`. The PR discussion is still useful because it settled the intended edge cases around `.tar` and custom suffixes like `.bak`, but the change needs a rebase into the current implementation before it can land.

## Best Possible Solution

Keep this PR open for maintainer follow-up. Rebase the intended behavior into `src/infra/backup-create.ts`: treat missing extensionless output paths as destination directories, keep existing files and non-empty suffixes such as `.tar` and `.bak` as explicit archive filenames, rethrow non-ENOENT stat errors, document the heuristic if it is product policy, and add focused regression tests around the resolver plus existing containment and no-overwrite safety checks.
