## Summary

Keep this PR open. The intended bug is still present on current main, but the PR patch targets the old `src/commands/backup.ts` resolver while the active resolver now lives in `src/infra/backup-create.ts`. The PR is still a plausible focused bug fix, but it needs a rebase onto the current backup implementation plus regression tests before it can be closed or merged.

## Best Possible Solution

Keep this PR open for maintainer follow-up. Rebase the intended behavior into `src/infra/backup-create.ts`: treat missing extensionless output paths as destination directories, preserve existing files and non-empty suffixes such as `.tar` and `.bak` as explicit archive filenames, rethrow non-ENOENT `fs.stat` errors, add focused resolver regression tests, and update CLI docs/help if this heuristic is the desired product policy.
