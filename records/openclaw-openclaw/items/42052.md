## Summary

Keep this PR open. Current main still treats a missing extensionless backup output as the final archive filename, and the PR patch targets the old command wrapper instead of the active resolver in infra.

## Best Possible Solution

Rebase the useful behavior into `src/infra/backup-create.ts`: treat missing extensionless output paths as destination directories, keep existing files and non-empty suffixes such as `.tar` and `.bak` as explicit archive filenames, rethrow non-ENOENT stat errors, update backup docs/CLI help, and add focused regression tests around the directory/file heuristic and safety checks.
