## Summary

Keep this PR open. The backup output-path bug still exists on current main, and the PR is a plausible contributor fix, but it targets the old command wrapper while the active resolver now lives in src/infra/backup-create.ts. It needs a rebase plus focused resolver tests before it can be merged or closed as obsolete.

## Best Possible Solution

Keep this PR open for maintainer follow-up. Rebase the intended behavior into src/infra/backup-create.ts: treat missing extensionless output paths as destination directories, preserve existing files and non-empty suffixes such as .tar and .bak as explicit archive filenames, rethrow non-ENOENT fs.stat errors, add focused resolver regression tests, and update CLI/docs wording if this heuristic is the desired policy.
