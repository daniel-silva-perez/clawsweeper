## Summary

Keep this PR open. Current main still treats a missing extensionless backup output path as the final archive file, while this PR's diff targets the old command wrapper instead of the active backup resolver.

## Best Possible Solution

Keep this PR open for maintainer follow-up. Rebase the intended behavior into `src/infra/backup-create.ts`: treat missing extensionless output paths as destination directories, preserve existing files and non-empty suffixes such as `.tar` and `.bak` as explicit archive filenames, rethrow non-ENOENT `fs.stat` errors, document the heuristic if it is product policy, and add focused regression tests around the resolver plus existing containment and no-overwrite safety checks.
