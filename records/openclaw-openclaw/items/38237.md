## Summary

Keep this PR open. Current main still has the Slack draft-preview identity gap: `sendMessageSlack` and normal `deliverReplies` support `username`/`iconUrl`/`iconEmoji`, and dispatch resolves `slackIdentity`, but the `createSlackDraftStream` params and initial draft `sendMessageSlack` call do not accept or forward identity. The PR is also paired with still-open #38235, so it should remain open until a clean port lands on the current `extensions/slack` paths.

## Best Possible Solution

Keep this PR and the paired #38235 open, then port only the identity fix to current `extensions/slack`: add optional `SlackSendIdentity` support to `createSlackDraftStream`, pass the resolved `slackIdentity` from `dispatchPreparedSlackMessage`, forward identity only on the initial draft `sendMessageSlack` call, and add a focused regression test. Split or drop the unrelated daemon cleanup and already-superseded duplicate-race work.
