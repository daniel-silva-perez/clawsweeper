## Summary

Keep this PR open. Current main still has the Slack draft-preview identity gap: normal Slack delivery resolves and forwards the active agent identity, and the Slack sender supports `username`/`iconUrl`/`iconEmoji`, but `createSlackDraftStream` has no identity parameter and its initial `sendMessageSlack` call omits identity. The PR is also paired with the still-open linked bug report, so it should not be closed before an implementation lands.

## Best Possible Solution

Keep this PR and the linked #38235 open until an identity-only fix is ported to current `extensions/slack`: add optional `SlackSendIdentity` support to `createSlackDraftStream`, pass the resolved `slackIdentity` from `dispatchPreparedSlackMessage`, forward identity only on the initial draft `sendMessageSlack` call, and add a focused regression test in `extensions/slack/src/draft-stream.test.ts`. Split out unrelated daemon cleanup and any already-superseded duplicate-race work.
