## Summary

Keep this PR open. Current main still has the Slack draft-preview identity gap: `sendMessageSlack` and normal delivery support Slack custom identity, and dispatch resolves `slackIdentity`, but `createSlackDraftStream` still neither accepts nor forwards identity for the initial draft `chat.postMessage`. The PR is also paired with open #38235 and should be ported cleanly to the current `extensions/slack` paths rather than closed.

## Best Possible Solution

Keep this PR and paired #38235 open, then port only the identity fix to current `extensions/slack`: add optional `SlackSendIdentity` support to `createSlackDraftStream`, pass the resolved `slackIdentity` from `dispatchPreparedSlackMessage`, forward identity only on the initial draft `sendMessageSlack` call, and add a focused regression test. Do not merge the unrelated daemon cleanup or already-superseded duplicate-race changes as part of this fix.
