## Summary

Keep open. Current main still treats Gemini web_search freshness/date filters as unsupported: the Google plugin schema advertises them as unsupported, the Gemini runtime returns the shared unsupported-filter response before any API request, the request body still sends an empty google_search tool, and the Gemini cache key omits temporal filters. The prior ClawSweeper comment remains accurate against current main; related #17925 is broader and still open for native-search passthrough work, but it is not a canonical replacement for this narrower Gemini freshness gap.

## Best Possible Solution

Keep this issue open. The likely fix belongs in the Google plugin: verify the Gemini Developer API time-range contract, map OpenClaw freshness/date_after/date_before into the supported Gemini Google Search grounding field, narrow or remove the unsupported-filter guard for Gemini, include normalized temporal filters in the cache key, add request-construction/cache/error tests, and update the Gemini/web_search docs.
