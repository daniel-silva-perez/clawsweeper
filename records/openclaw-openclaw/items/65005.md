## Summary

Keep open. Current main still has no `models explain` command, so this PR is not obsolete; however, the proposed implementation still has correctness issues around default-agent resolution and persisted-session overrides.

## What This Changes

Adds an `openclaw models explain` subcommand, command implementation, CLI/export wiring, and tests to report default, override, persisted-session, family-inference, and final provider/model resolution.

## Best Possible Solution

Add the diagnostic command in the current lazy-loaded models CLI structure, resolve omitted agents and session targets through the canonical helpers, make session-versus-CLI override precedence explicit, cover JSON/text behavior with focused tests, and document the command on the models CLI page.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Resolve omitted agents before explaining defaults:** `src/commands/models/explain.ts:24`
  - body: When `--agent` is omitted, `agentId` remains undefined, so both `defaults` and `resolved` call `resolveSessionModelRef` without the configured default agent. Runtime session paths resolve the default agent first, so configs whose default agent has a per-agent model override will get misleading explain output.
  - confidence: 0.86
- **[P2] Apply CLI overrides in session explain mode:** `src/commands/models/explain.ts:46-50`
  - body: In the `--session` branch the resolver entry is built only from stored session fields, but the payload still reports `opts.model` and `opts.provider` as applied overrides. Either merge those CLI values into the entry or reject the combination before reporting overrides that did not affect the result.
  - confidence: 0.83
