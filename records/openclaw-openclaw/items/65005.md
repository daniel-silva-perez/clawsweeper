## Summary

Keep open. Current main still does not ship a `models explain` command, so the PR is not obsolete; however the proposed patch still has correctness issues around default-agent resolution and session-mode overrides, and it also needs a rebase onto the current lazy-loaded models CLI structure.

## What This Changes

The PR adds an `openclaw models explain` CLI command with JSON/text output, session lookup support, command exports/wiring, and focused tests for provider/model resolution diagnostics.

## Best Possible Solution

Keep the diagnostic command in core if maintainers want this CLI surface, but rebase it onto the current lazy-loaded models CLI, resolve the default agent before no-agent explanations, define and enforce `--session` plus CLI override semantics, then add docs/changelog and regression coverage for default-agent and persisted-session cases.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists with a config whose configured default agent has its own model: the PR passes `undefined` into `resolveSessionModelRef` when `--agent` is omitted, while session flows route through the configured default agent; a second static reproduction is `models explain <model> --provider <p> --session <key>`, where the resolver ignores those CLI overrides but the payload reports them as applied.

## Solution Assessment

No. The command is a reasonable core diagnostic, but this PR is not the best current implementation until it matches default-agent/session resolution semantics and is rebased onto the current models command architecture.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Resolve the default agent before explaining defaults:** `src/commands/models/explain.ts:62`
  - body: When `--agent` is omitted, `agentId` remains undefined, so both `defaults` and `resolved` use the global configured model path. Real session routing resolves bare/main sessions through the configured default agent, so configs with a default-agent model override will get misleading `Default resolved` and `Final resolved` output.
  - confidence: 0.88
- **[P2] Apply CLI overrides in session explain mode:** `src/commands/models/explain.ts:50-54`
  - body: In the `--session` branch, the resolver input is built only from persisted session fields. If a user passes a positional model or `--provider` with `--session`, the payload reports those values as applied, but `resolved` was computed from the unchanged session entry, making the explanation internally inconsistent.
  - confidence: 0.9
