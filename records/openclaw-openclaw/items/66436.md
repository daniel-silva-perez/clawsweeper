## Summary

Keep this issue open. Current main still emits a macOS LaunchAgent with `KeepAlive=true` and `ThrottleInterval=1`, and the missing-config startup guard still exits with `EXIT_CONFIG_ERROR` before the gateway loop can apply any in-process restart/backoff behavior. The related #60885 launchd supervision issue is adjacent, but the provided context does not show it covering the missing-config crash-loop/log-storm case.

## Best Possible Solution

Keep this open for a macOS LaunchAgent/gateway startup fix that prevents repeated missing-config or config-error respawns while preserving intentional restarts. A good fix would add launchd-specific suppression, a sentinel or bounded backoff/max-retry path, or a degraded wait-for-config mode, then update launchd/gateway regression tests and gateway troubleshooting docs. Coordinate with #60885, but only close as duplicate if that work explicitly covers missing-config crash-loop/log-storm behavior.
