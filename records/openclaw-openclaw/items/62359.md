## Summary

Keep open. Current main still enables the packaged CLI Node compile cache without deriving an OpenClaw install-root/version-scoped cache directory, while this open PR is the active implementation candidate for that cache-hygiene change. The latest PR patch targets the right surface, adds focused tests and docs, and avoids workflow/dependency/lockfile changes, but it still needs normal maintainer review, rebase/current-head validation, and careful framing around #71348 because the fresh-process stale-bytecode root cause remains unproven.

## Best Possible Solution

Keep this PR open and review/land it or an equivalent patch after current-head validation. The desired fix is to derive one OpenClaw-scoped compile-cache directory per packaged install root and OpenClaw version in both bootstrap paths, keep source-checkout cache disable/respawn behavior intact, pass Node-compatible string cacheDir values, and document NODE_COMPILE_CACHE as the base for scoped entries while leaving #71348 as a reference unless the root cause is proven.
