## Summary

Keep open. Current main has a newer source-checkout compile-cache guard, but it still does not implement this PR's central behavior: scoping packaged CLI compile-cache directories by OpenClaw install root and OpenClaw version. The maintainer discussion also keeps this PR active as cache hygiene while asking for a refresh, clearer NODE_COMPILE_CACHE semantics, changelog credit if shipped, and stronger/reframed root-cause evidence.

## Best Possible Solution

Keep this PR open as the concrete implementation candidate. Refresh it onto current main, implement install-root plus OpenClaw-version scoped cache directories in the current wrapper and entry compile-cache paths, pass a Node-22-compatible string cacheDir, document the intentional NODE_COMPILE_CACHE base-directory semantics, add changelog credit if this ships as a user-facing cache fix, and either prove stale bytecode reuse or describe the change as preventative cache hygiene before review/landing.
