## Summary

Keep open. Current main still does not implement this PR's central behavior: scoping packaged CLI compile-cache directories by OpenClaw install root and OpenClaw version. The current launcher and entry helper only disable/respawn the compile cache for live source checkouts, while packaged installs still call Node's compile-cache API without a derived OpenClaw-scoped cache directory. Maintainer discussion also keeps this PR active as cache hygiene/hardening while asking for a refresh, clearer NODE_COMPILE_CACHE semantics, changelog credit if it ships, and stronger or reframed root-cause evidence.

## Best Possible Solution

Keep this PR open as the concrete implementation candidate. Refresh it onto current main, implement install-root plus OpenClaw-version scoped cache directories in the current wrapper and entry compile-cache paths, pass a Node-22-compatible string cacheDir, document the intentional NODE_COMPILE_CACHE base-directory semantics, add changelog credit if this ships as a user-facing cache fix, and either prove stale bytecode reuse or describe the change as preventative cache hygiene before review/landing.
