## Summary

Keep this open. Current main has generic plugin install, uninstall, stale-config cleanup, and channel replacement primitives, but I did not find an install-time migration that maps or removes a still-installed `openclaw-weixin` when installing `openclawwechat`; the requested automatic cleanup also needs maintainer/product judgment because it can affect explicitly installed third-party plugin code.

## What This Changes

The issue requests safer migration when installing the renamed WeChat plugin so the older `openclaw-weixin` install is not mistaken for, left beside, or allowed to conflict with `openclawwechat`.

## Best Possible Solution

Keep this open for a maintainer decision on the migration policy. The safest end state is an explicit, plugin-owned replacement story for the WeChat package rename, with OpenClaw using generic opt-in metadata or warnings and clear docs/uninstall steps rather than guessing from similar names.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Unclear: the report gives concrete install output, versions, and old/new plugin ids, but the external `openclawwechat` package is not in this repository and no current-main fixture reproduces that exact mixed install state.

## Solution Assessment

Is this the best way to solve the issue? No for automatic same/similar-name cleanup as proposed; the narrower maintainable path is explicit plugin replacement metadata plus docs or a generic install-time warning, with destructive cleanup left to explicit `plugins uninstall` or `doctor --fix` behavior.
