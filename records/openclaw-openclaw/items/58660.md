## Summary

Current main does not fully address the reported plugin rename/migration problem. OpenClaw now has normal same-id update, force reinstall, uninstall, stale-config doctor repair, and some tracked-update id migration support, but there is no explicit install-time replacement contract that maps `openclaw-weixin` to `openclawwechat` and safely removes the old managed plugin directory/config as requested.

## Best Possible Solution

Keep this open for maintainer/product decision. The safest path is an explicit plugin replacement contract, not fuzzy name matching: let a new plugin/package declare legacy plugin IDs it replaces, have install/update/doctor detect tracked old records, preview or perform safe uninstall of managed old paths, migrate config/channel references, and document the WeChat migration. If OpenClaw intentionally does not support `openclawwechat`, the docs should instead state the supported `openclaw-weixin` package and provide a manual cleanup sequence for old third-party installs.

## Risks / Open Questions

- Closing this as implemented would leave the exact renamed-plugin case unresolved: a user can still install a new plugin id while an old managed or untracked `openclaw-weixin` directory/config remains active or noisy.
- Any fix should avoid fuzzy “similar name” deletion. A safe implementation needs explicit replacement metadata and must only remove tracked managed install paths inside the OpenClaw plugin extensions root.
