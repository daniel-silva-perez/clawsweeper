## Summary

Keep this PR open. Current main at ab5c8025 still extracts session-memory transcript text verbatim and writes it under `## Conversation Summary` without sanitizer coverage for chat-template tokens, raw or dangling `<tool_call>` XML, role-marker-only turns, or `NO_REPLY`-style malformed turns. The PR is still a live implementation candidate, but it overlaps with #73768 and #71257 and has bot-reviewed sanitizer edge cases that need maintainer consolidation before merge.

## Best Possible Solution

Keep this PR open for maintainer review and consolidation. The best fix is a single session-memory artifact-sanitization path for #69943 that sanitizes transcript reads and, if retained, the final handler write path; reuses the shared assistant-visible/model-token sanitizer where behavior matches; handles raw and dangling tool-call XML; preserves short valid replies; elides truly malformed turns with visible tests; and coordinates with #73768/#71257 rather than landing parallel implementations. Add focused regression tests for transcript extraction and handler persistence, plus a changelog entry if the final patch changes user-visible memory-file content.
