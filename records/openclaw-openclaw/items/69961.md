## Summary

Keep this PR open. The reported session-memory poisoning bug is still present on current main, and the PR targets the right bundled-hook path, but the proposed sanitizer has correctness blockers that can drop legitimate short replies and miss truncated tool-call XML before persistence.

## What This Changes

The PR adds a session-memory sanitizer, wires it into transcript reads and the handler write path, and adds tests for stripping chat-template tokens, raw tool-call XML, role-marker-only lines, and NO_REPLY before memory persistence.

## Best Possible Solution

Land a single consolidated session-memory fix that sanitizes persisted memory on the transcript and writer boundaries, handles closed and truncated tool-call XML, preserves nonempty legitimate short content, adds regression tests for those edge cases, and includes the required changelog entry.

## Reproduction Assessment

Yes. The original bug has a high-confidence source-level reproduction because current main reads transcript content raw and writes it verbatim to the memory file; the PR findings are also reproducible with pure inputs such as `2+2=4<|im_end|>` and an unterminated `<tool_call>` block.

## Solution Assessment

No. The PR is aimed at the right seam, but the current sanitizer heuristic is too destructive for short valid content and the XML pattern is incomplete for truncated tool-call leaks; a narrow sanitizer repair is safer than merging as-is.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve short valid turns after sanitization:** `src/hooks/bundled/session-memory/sanitize.ts:72-76`
  - body: The elision heuristic replaces any short turn when more than half of the original string was removed. A concise valid reply like `2+2=4<|im_end|>` cleans to useful content but is still persisted as `[malformed turn elided]`, so session memory drops exactly the kind of short factual/tool answer users expect to carry forward. Preserve nonempty cleaned text unless the turn is truly empty or known housekeeping garbage.
  - confidence: 0.9
- **[P2] Strip truncated tool_call blocks:** `src/hooks/bundled/session-memory/sanitize.ts:27`
  - body: The tool-call regex only matches blocks with a closing `</tool_call>`. If the stream is cut after `<tool_call>` or mid-function, that malformed block survives sanitization and can still be written into the memory file, leaving the re-poisoning path open for the parser-race case this PR is meant to defend against.
  - confidence: 0.86
