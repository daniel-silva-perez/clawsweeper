## Summary

Keep open. Current main still reads session-memory transcript text verbatim and writes it under `## Conversation Summary` without sanitizing chat-template tokens, raw `<tool_call>` XML, orphan role markers, or `NO_REPLY`-style malformed turns. The later related PR #73768 is open and narrower, not merged into main, so this PR remains a live implementation candidate rather than conservative cleanup.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a consolidated session-memory fix for #69943 that sanitizes transcript reads and the final write path, preferably reusing the shared assistant-visible/model-token sanitizer where appropriate, handles dangling raw tool-call XML, preserves short valid replies, elides truly malformed turns with tests, and coordinates with #73768 instead of duplicating behavior. Treat #68751 as a separate broader trust-framing follow-up unless the fix intentionally includes untrusted historical markers.
