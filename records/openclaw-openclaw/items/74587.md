## Summary

Keep open. Current main still emits the reported Active Memory warning text, while the resolver, docs, and tests show `modelFallback` is the last candidate in model-resolution, not runtime failover. The issue is already covered by open PRs #74627 and #74602, so maintainer review should choose one fix rather than closing or queueing another repair job.

## What This Changes

The issue requests clarifying Active Memory's `modelFallbackPolicy` deprecation warning so `modelFallback` is described as a final model-resolution fallback, not runtime failover.

## Best Possible Solution

Land #74627, #74602, or an equivalent small copy/test change that keeps Active Memory's current model-resolution behavior unchanged while making the deprecation warning match the documented chain semantics. Any true retry-on-error runtime failover should remain a separate product feature discussion.
