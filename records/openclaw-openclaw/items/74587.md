## Summary

Keep open. Current main still emits the reported deprecation warning text, while the active-memory resolver, tests, and docs all show `modelFallback` is only the last model-resolution candidate, not runtime failover after a timeout or model error.

## What This Changes

The issue requests clearer active-memory `modelFallbackPolicy` deprecation text and docs so `modelFallback` is not mistaken for runtime failover.

## Best Possible Solution

Keep the current chain-resolution behavior, but update active-memory warning/config/docs copy to say `modelFallback` is a last-resort model-resolution default, not retry/failover for a failed or timed-out resolved model. Any real runtime failover setting should be tracked separately as a new feature.
