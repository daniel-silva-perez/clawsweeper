## Summary

Keep open. Current main still emits the reported Active Memory deprecation warning, while the resolver, docs, and tests show `modelFallback` is the final model-resolution candidate rather than runtime retry/failover. The issue is already paired with open PR #74602, which uses closing syntax and targets the warning copy, so this should wait for maintainer review/merge rather than be closed or queued as a separate repair.

## What This Changes

The issue asks for the Active Memory `modelFallbackPolicy` deprecation warning to clarify that `modelFallback` is a last-resort model-resolution fallback, not runtime failover.

## Best Possible Solution

Land #74602 or an equivalent small copy/test change that keeps Active Memory’s current model-resolution behavior unchanged while making the deprecation warning match the documented chain semantics. Any real retry-on-error runtime failover should remain a separate product feature discussion.
