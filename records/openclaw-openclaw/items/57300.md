## Summary

Keep this PR open. Current main still runs `openclaw agent --deliver` through an ACP or embedded agent turn and delivers the resulting reply payloads, not the original `--message`. The PR remains a relevant implementation candidate for #57284, but it still needs maintainer review around non-CLI delivery workflows such as exec approval followups.

## Best Possible Solution

Keep the PR open and revise it so direct delivery applies only to the explicit CLI notification use case from #57284. Preserve gateway-first execution, keep internal-event announcements and exec approval followups on the model-synthesis path, and add regression coverage for direct CLI `--deliver`, internal-event announcements, exec approval followups, and gateway routing before merge.
