## Summary

Keep this PR open. Current main does not implement the PR's intended direct-delivery behavior for `openclaw agent --deliver`; it still runs an agent/ACP turn and delivers the resulting reply payloads. The PR remains a plausible implementation candidate for #57284, but needs maintainer review and a narrower contract for non-CLI delivery workflows such as exec approval followups.

## Best Possible Solution

Keep the PR open and revise it so direct delivery only applies to the explicit CLI notification use case from #57284. Preserve gateway-first execution, keep internal-event and exec-approval followup deliveries on the model-synthesis path, and add regression coverage for direct CLI `--deliver`, internal-event announcements, exec approval followups, and gateway routing before merge.
