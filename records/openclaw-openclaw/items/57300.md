## Summary

Keep this PR open. Current main still runs `openclaw agent --deliver` through a normal agent turn and only delivers the resulting payloads, so the intended direct-delivery behavior is not already implemented. The PR also still needs maintainer review because the same `deliver: true` contract is used by other model-synthesis workflows, including exec approval followups and subagent announcements.

## Best Possible Solution

Keep the PR open and revise the implementation to target only the explicit CLI direct-notification use case from #57284. Preserve the gateway-first route, keep internal-event and exec-approval followup deliveries on the LLM synthesis path, and add regression coverage for direct CLI `--deliver`, `internalEvents`, exec approval followups, and gateway routing before merge.
