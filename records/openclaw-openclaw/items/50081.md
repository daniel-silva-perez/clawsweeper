## Summary

Keep this issue open. Current main still has the reported current-run path: Control UI chat attachments are parsed into `parsedImages`, passed into `replyOptions.images`, merged as `existingImages`, and then sent directly to `activeSession.prompt(..., { images })`. The only current-main HEIC normalization found is in the separate input-file/OpenResponses path, not in chat attachment prompt images. There is also an open implementation candidate, #50320, whose body uses closing syntax for this issue, so the maintainer path is to review/land or extract that fix rather than close the issue before merge.

## Best Possible Solution

Keep the issue open and review/land PR #50320 or extract a focused equivalent fix. The code change likely belongs in `src/gateway/chat-attachments.ts` before `parsedImages` are handed to `chat.send`/`agent.run`, with a regression test proving `image/heic` and `image/heif` attachments become `image/jpeg` before the current-run `activeSession.prompt` call.
