## Summary

Keep this issue open. Current main still leaves the reported current-run HEIC/HEIF attachment path unnormalized before provider send, and the linked implementation candidate is still open and unmerged, so this should not be closed as fixed yet.

## What This Changes

The issue asks OpenClaw to normalize HEIC/HEIF Control UI prompt images to JPEG before the current model call, rather than only after persistence or history processing.

## Best Possible Solution

Fix the current-run chat attachment path so HEIC/HEIF inline images are converted to JPEG before `parsed.images` reaches `activeSession.prompt`, recheck converted size against image limits, preserve existing non-HEIC behavior, and add regression coverage in the gateway attachment tests and prompt dispatch path as needed.
