## Summary

Keep this issue open. Current main still has an inline Control UI/WebChat attachment path where HEIC/HEIF data can be kept as `image/heic` through the current-run prompt handoff, and the provided GitHub context shows #50320 is still open, unmerged, and uses closing syntax for this issue.

## What This Changes

The issue asks for HEIC/HEIF images uploaded through Control UI/WebChat to be converted to JPEG before the current agent run sends prompt images to the provider.

## Best Possible Solution

Land #50320 or a focused replacement that normalizes HEIC/HEIF inline chat attachments to JPEG before they reach `activeSession.prompt`, rechecks converted size against image limits, preserves non-HEIC and offloaded-media behavior, and adds focused gateway attachment plus prompt dispatch regression coverage.

## Reproduction Assessment

Yes. The reporter provided concrete Control UI HEIC upload steps and hot-patch confirmation, and current main statically preserves inline HEIC attachments through the current-run prompt image path when the prompt has no file refs.

## Solution Assessment

Yes. Normalizing before the current-run provider call is the right fix boundary because persistence/history normalization happens too late for the in-flight model request; the remaining maintainer choice is whether to land, split, or replace #50320.
