## Summary

Current main still appears to have the reported current-run gap: inline HEIC/HEIF chat attachments can remain `image/heic`/`image/heif` in `parsed.images` and flow into `activeSession.prompt`. The issue should stay open because #50320 is still an open, unmerged implementation candidate linked with closing syntax for this issue.

## What This Changes

Normalize HEIC/HEIF images uploaded through Control UI/WebChat to JPEG before they are handed to the current model run.

## Best Possible Solution

Fix the current-run chat attachment path so HEIC/HEIF inline images are converted to JPEG before `parsed.images` reaches `activeSession.prompt`, recheck converted size against image limits, preserve non-HEIC behavior, and cover the regression in `src/gateway/chat-attachments.test.ts` plus the dispatch path if needed.
