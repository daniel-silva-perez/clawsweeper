## Summary

Keep this issue open. Current main still preserves inline WebChat HEIC/HEIF attachments into the current-run prompt image path, and #50320 remains the open implementation candidate that uses closing syntax for this issue.

## What This Changes

The issue asks for HEIC/HEIF images uploaded through Control UI/WebChat to be converted to JPEG before the current-run provider prompt send.

## Best Possible Solution

Land a narrow fix, from #50320 or a focused replacement, that converts HEIC/HEIF inline chat attachments to JPEG before `parsed.images` reaches `activeSession.prompt`, rechecks the converted output against image limits, preserves non-HEIC and offloaded-media behavior, and adds focused regression coverage.

## Reproduction Assessment

Yes. The reporter provided concrete Control UI HEIC upload steps and hot-patch confirmation, and current main statically preserves `image/heic` from chat attachment parsing through `replyOptions.images` into `activeSession.prompt` when the prompt has no file refs.

## Solution Assessment

Yes. Normalizing before the current-run provider call is the right fix boundary because persisted/history normalization happens too late for the in-flight model request; the remaining decision is whether #50320 should be landed as-is, split, or replaced.
