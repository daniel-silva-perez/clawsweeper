## Summary

Current main still has the reported gap: Control UI/webchat inline HEIC/HEIF attachments can be returned from `parseMessageWithAttachments` as `image/heic`/`image/heif`, forwarded into the current run, and handed to `activeSession.prompt`. The issue should stay open because #50320 is an open, unmerged implementation candidate with closing syntax for this issue.

## What This Changes

Normalize HEIC/HEIF images uploaded through Control UI/webchat before current-run prompt delivery to OpenAI vision models.

## Best Possible Solution

Keep this issue open and resolve it through #50320 or a focused successor: normalize HEIC/HEIF chat attachments to JPEG before `parsed.images` is returned for current-run prompt delivery, recheck the converted JPEG against the same image limits, preserve non-HEIC behavior, and add focused regression coverage in `src/gateway/chat-attachments.test.ts` plus a dispatch/path proof if needed.
