## Summary

Keep this issue open. Current main still preserves inline WebChat/Control UI HEIC or HEIF attachments through the current-run prompt-image handoff, and the provided GitHub context shows #50320 as an open, unmerged PR with closing syntax for this issue.

## What This Changes

Normalize HEIC/HEIF images uploaded through the Control UI/WebChat before the current-run prompt handoff so OpenAI vision models receive JPEG data instead of `image/heic` or `image/heif`.

## Best Possible Solution

Land a narrow fix, from #50320 or a split/replacement, that converts HEIC/HEIF inline chat attachments to JPEG before `parsed.images` reaches `activeSession.prompt`, rechecks converted size against image limits, preserves non-HEIC behavior, and adds regression coverage for the gateway attachment and prompt dispatch path.
