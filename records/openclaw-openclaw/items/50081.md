## Summary

Keep this issue open. Current main still preserves inline Control UI/WebChat HEIC or HEIF attachments as prompt images on the current-run path, while the existing HEIC-to-JPEG normalization lives in other media/input paths. The open #50320 PR still uses closing syntax for this issue, so maintainer follow-up should happen there before this issue is closed.

## What This Changes

Normalize HEIC/HEIF images uploaded through Control UI/WebChat to JPEG before the current-run model prompt image handoff.

## Best Possible Solution

Land a narrow fix, from #50320 or a split/replacement, that converts HEIC/HEIF inline chat attachments to JPEG before `parsed.images` reaches `activeSession.prompt`, rechecks converted output against image limits, preserves non-HEIC behavior, and adds regression coverage for chat attachment parsing and prompt dispatch.
