## Summary

Keep this issue open. Current main still preserves a path where inline Control UI/WebChat HEIC or HEIF attachments can enter the current-run prompt image list without conversion, and #50320 remains the open unmerged implementation candidate with closing syntax for this issue.

## What This Changes

The issue asks for HEIC/HEIF images uploaded through Control UI/WebChat to be converted to JPEG before the current-run prompt image handoff reaches OpenAI vision models.

## Best Possible Solution

Land #50320 or a focused replacement that converts HEIC/HEIF inline chat attachments to JPEG before `parsed.images` reaches `activeSession.prompt`, rechecks converted size against image limits, preserves non-HEIC and offloaded-media behavior, and adds focused gateway attachment and prompt dispatch regression coverage.

## Reproduction Assessment

Yes. The reporter provided concrete Control UI HEIC upload steps and hot-patch confirmation, and current main statically preserves inline `image/heic` attachments through the current-run prompt image path when the prompt has no file refs.

## Solution Assessment

Yes. Normalizing before the current-run provider call is the right fix boundary because persistence/history normalization happens too late for the in-flight model request; the remaining maintainer choice is whether to land, split, or replace #50320.
