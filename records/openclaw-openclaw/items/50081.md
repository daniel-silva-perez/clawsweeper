## Summary

Keep open. Current main still preserves inline Control UI/WebChat HEIC or HEIF attachments as prompt images without converting them to JPEG before the provider handoff, and the open implementation candidate #50320 already uses closing syntax for this issue but has not merged.

## What This Changes

The issue requests HEIC/HEIF images uploaded through Control UI/WebChat be normalized to JPEG before current-run prompt images are sent to OpenAI vision models.

## Best Possible Solution

Land #50320 or a narrow replacement that converts HEIC/HEIF inline chat attachments to JPEG before `parsed.images` reaches `activeSession.prompt`, rechecks converted output against image limits, preserves non-HEIC and offloaded-media behavior, and adds focused regression coverage for the chat attachment and prompt dispatch path.
