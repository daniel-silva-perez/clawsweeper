## Summary

Current main still lacks a normal allowlist satisfaction path for POSIX shell builtins such as cd/export/pwd when they appear as shell-chain segments. The related closed #40478 covers durable exact-command allow-always reuse after a prompt, but it does not solve preconfigured allowlist mode for fresh builtin-containing chains. A narrow docs clarification is safe; runtime auto-allowing builtins needs separate maintainer design.

## Best Possible Solution

Keep this issue open. The safest narrow next step is a docs-only clarification in the exec approvals allowlist section explaining that normal path/bare-name allowlist entries match filesystem executables, not POSIX shell builtins, and recommending forms such as git -C <path> instead of cd <path> && git status. If maintainers want runtime behavior changes, design an opt-in builtin policy separately with explicit cwd/env mutation semantics and regression tests.
