## Summary

Keep this issue open. Current main still intentionally returns INVALID_REQUEST / unknown agent id when tools.catalog or tools.effective is called with an agent ID that is not configured, while tools.profile: "full" remains unrestricted. The report still lacks the failing agentId, sessionKey, request payloads, effective tool inventory, redacted config, provider/model, and logs needed to decide whether this is stale local agent state, a UI/session mismatch, provider/search readiness, or a real runtime filtering bug.

## Best Possible Solution

Keep this open and ask for a fresh reproduction on current OpenClaw with the failing agentId, sessionKey, tools.catalog and tools.effective params/responses, openclaw agents list, redacted agents.list and tools config, selected provider/model/channel/plugin, web-search provider config, install method/path, and gateway logs. If the cause is stale local agent state, improve doctor or Control UI diagnostics; if a valid full-profile session still exposes only messaging/session tools, trace resolveEffectiveToolPolicy, createOpenClawCodingTools, provider/model filtering, group/channel policy, and web-search provider readiness.
