## Summary

Keep open. Current main still lacks native node file transfer, and this PR is a useful implementation candidate for the open feature request, but the branch is not merge-ready because it expands node-host filesystem read/write capability with blocking security and contract defects.

## What This Changes

The PR adds `file_pull` and `file_push` actions to the `nodes` tool, `file.read` and `file.write` node commands, node-host handlers, command-policy entries, and file-transfer helper tests.

## Best Possible Solution

Land a revised file-transfer design that keeps the useful single-shot workflow, treats node filesystem access as high-risk unless explicitly scoped, binds write approval to the requester or a deliberate host-wide policy, honors the documented write flags, supports platform-native paths, and adds focused tests for those contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate file.read before enabling it by default:** `src/gateway/node-command-policy.ts:63`
  - body: This adds `file.read` to the host-node default command set, but the new handler can read any absolute or `~` path under the node-host user's permissions. That creates a default file exfiltration primitive; make this an explicit dangerous/path-scoped opt-in before enabling it for paired hosts.
  - confidence: 0.92
- **[P1] Preserve requester-specific file.write approval:** `src/node-host/invoke.ts:588-590`
  - body: The write gate scans every configured agent and allows the request if any one has `file.write` allowlisted or `security: full`. Because the request is not resolved against the initiating agent/session, approval for one agent can unlock file writes for other callers once the gateway command is enabled.
  - confidence: 0.86
- **[P2] Honor mkdirs and overwrite in the node handler:** `src/node-host/invoke.ts:650-651`
  - body: The tool forwards `mkdirs` and `overwrite`, but the handler always creates parent directories and always overwrites via `writeFileSync`. Requests with `overwrite: false` can still replace existing files, breaking the declared contract and risking data loss.
  - confidence: 0.9
- **[P2] Accept Windows absolute paths for host file transfer:** `src/node-host/invoke.ts:524`
  - body: The path validation only accepts `/` or `~` prefixes. Windows is included in the default command policy, but normal Windows absolute paths such as `C:\tmp\a.txt` are rejected before read or write, so file transfer is broken on that advertised platform.
  - confidence: 0.88
- **[P2] Reject malformed base64 before writing files:** `src/node-host/invoke.ts:638`
  - body: `Buffer.from(data, "base64")` is permissive and can silently ignore invalid characters. A malformed upload can be accepted as a successful `file.write` with different bytes than the caller sent, so validate base64 strictly before decoding and writing.
  - confidence: 0.78
- **[P2] Recheck file.read bytes after reading:** `src/node-host/invoke.ts:564`
  - body: The size cap is checked against `stat.size` before `readFileSync`, but the file can grow or be swapped before the read. Check `buf.length` after reading or read through an opened descriptor so the hard cap cannot be bypassed by a race.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The diff introduces node-host filesystem read/write primitives, and the current policy/approval wiring has concrete security concerns before merge.

Concerns:

- **[high] Default-enabled arbitrary node file reads:** `src/gateway/node-command-policy.ts:63`
  - body: `file.read` is added to default host-node commands while the handler reads arbitrary absolute or home-relative paths. That exposes a default exfiltration primitive for paired node hosts instead of requiring an explicit high-risk command opt-in or path-scoped approval.
  - confidence: 0.92
- **[medium] file.write approval can leak across agents:** `src/node-host/invoke.ts:588`
  - body: The write gate checks all agents in the exec approvals file rather than the requester. If any agent has `file.write` allowlisted or `security: full`, other callers can pass the node-host gate once the gateway command is enabled.
  - confidence: 0.86
