## Summary

Keep open. Current main does not provide a native Plugin SDK `registerMcpServer` API; it only supports existing user-configured and bundle-provided MCP server wiring. The open related PR #62160 is the active implementation candidate for this request, so the right cleanup path is maintainer review of that PR rather than closing this issue now.

## What This Changes

Request to add a native Plugin SDK `registerMcpServer` surface so plugins can register managed stdio MCP servers for embedded OpenAI-backed runs.

## Best Possible Solution

Review and either land a safe native SDK implementation like #62160, including validation, deterministic precedence, registry/workspace scoping, embedded runtime materialization, documentation, and focused tests, or replace that PR with clearer maintainer guidance. After the implementation merges, this issue can close as implemented.
