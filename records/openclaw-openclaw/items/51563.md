## Summary

Keep this PR open. Current main still reports the package install manager from the installed package's repo metadata, so the linked #51401 bug remains present on the status path. The PR targets the right behavior, but it should be refreshed to use the stronger global-install detector already on main instead of adding a separate path-only detector.

## Best Possible Solution

Refresh this PR before merge: make `checkUpdateStatus` use repo package-manager detection only for git roots, and for package roots call or adapt the existing `detectGlobalInstallManagerForRoot` path used by update flows. Add regression coverage for an npm global OpenClaw package whose `package.json` declares pnpm, plus custom `BUN_INSTALL` coverage, then merge this PR or supersede it with that implementation.
