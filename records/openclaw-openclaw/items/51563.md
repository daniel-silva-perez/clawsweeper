## Summary

Keep this PR open. Current main still uses repo/package metadata to populate `openclaw update status` for package installs, so the reported npm-global misdetection is not fixed on main. The PR targets the right bug, but it should be refreshed to reuse the stronger global-install detector that now exists in current main rather than adding a weaker path-only detector.

## Best Possible Solution

Keep this PR open and refresh it before merge: make `checkUpdateStatus` use repo package-manager detection only for git roots, and use or adapt current main's `detectGlobalInstallManagerForRoot` for package roots. Add regression coverage for an npm global OpenClaw package whose `package.json` declares pnpm, plus custom `BUN_INSTALL` coverage, then merge this PR or supersede it with that implementation.
