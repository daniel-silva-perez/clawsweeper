## Summary

Keep this PR open. Current main still routes `openclaw update status` package installs through `detectPackageManager(root)`, which reads the installed package's `package.json.packageManager` before lockfiles, so an npm global install of the OpenClaw package can still be reported as pnpm. The PR addresses the right bug, but should be refreshed to reuse or adapt the newer global-install detector that already handles npm/pnpm roots and custom Bun roots.

## Best Possible Solution

Keep this PR open and refresh it before merge: make `checkUpdateStatus` use repo package-manager detection only for git checkouts, and use or adapt `detectGlobalInstallManagerForRoot` for package roots. Add regression tests for an npm global package root where the installed OpenClaw package declares pnpm in `package.json`, plus custom `BUN_INSTALL` coverage, then merge this PR or supersede it with that implementation.
