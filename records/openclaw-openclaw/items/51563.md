## Summary

Keep this PR open. Current main still routes package-install status through `detectPackageManager(root)`, which reads the installed package's `package.json.packageManager`; that leaves the linked #51401 behavior unresolved even though the issue is closed. The PR is relevant and has no obvious security-sensitive side effects, but it should be refreshed to reuse or adapt the newer global-install detection helper so supported custom Bun roots are not misclassified.

## Best Possible Solution

Keep this PR open, but refresh it before merge: make `checkUpdateStatus` use repo package-manager detection only for git checkouts and use or adapt `detectGlobalInstallManagerForRoot` for package roots. Add regression coverage for an npm global package root with OpenClaw metadata declaring pnpm, plus custom `BUN_INSTALL` coverage, then merge this PR or supersede it with that implementation.
