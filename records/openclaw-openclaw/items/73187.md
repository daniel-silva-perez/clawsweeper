## Summary

Current main still emits and tests the compact Gateway startup log shape, while the requested structured auth/server/model/plugins/log-file/service-status startup summary is not implemented. ## What This Changes

The issue requests restoring or adding a structured Gateway startup log summary that shows auth mode/source, server URL, agent model, loaded plugins, log path, and health-monitor/heartbeat startup status.

## Best Possible Solution

Add one redacted, deterministic Gateway-owned startup summary that reuses existing logger behavior and already-resolved runtime state; document the operator-visible fields and avoid a new config switch unless maintainers explicitly want both minimal and structured modes.
