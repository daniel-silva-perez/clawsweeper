## Summary

Keep open. Current main still emits and tests the compact Gateway startup log shape, and the requested structured auth/server/model/plugins/log-file/service-status startup summary is not implemented. The related #14861 is adjacent subsystem-outcome observability work, but it does not supersede this formatted startup display request.

## What This Changes

Request to restore or configure a structured Gateway startup log showing auth mode/source, server URL, agent model, loaded plugins, log file path, and health/heartbeat startup status.

## Best Possible Solution

Add one redacted, deterministic Gateway-owned startup summary that reuses existing logger behavior and resolved runtime state; document the operator-visible fields and avoid a new config switch unless maintainers explicitly want both minimal and structured modes.

Yes. ## Solution Assessment
