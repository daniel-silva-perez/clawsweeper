Keep open. Current main still emits the compact Gateway startup shape and lacks the requested structured startup snapshot with auth mode/source, server URL, agent model, plugins, log path, health-monitor status, and heartbeat status; the adjacent startup-observability work does not fully cover this formatted startup display request.

Add one redacted, deterministic Gateway startup summary in the Gateway-owned startup path, reusing the existing logger and already-resolved runtime state; keep plugin ordering stable, document the operator-visible fields if they become a public contract, and avoid a new config switch unless maintainers explicitly want both minimal and structured modes.
