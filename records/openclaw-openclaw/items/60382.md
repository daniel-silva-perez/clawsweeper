## Summary

Keep open. Current main has incremental TUI polish, light/dark accessibility work, and optional channel setup paths, but it does not implement the broad visual/interaction redesign requested here and I did not find a canonical duplicate that owns the remaining product/design decision.

## What This Changes

The issue asks to redesign the OpenClaw terminal UI experience and reduce the setup/channel friction that makes the current TUI feel off-putting.

## Best Possible Solution

Keep this as a product/design enhancement and scope it into concrete work: target TUI layout, visual language, accessibility goals, first-chat/no-channel path, and feasible terminal animation behavior, then split implementation across `src/tui/*`, TUI docs, and focused tests.

## Solution Assessment
