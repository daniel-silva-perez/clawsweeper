## Summary

Keep open. Current main still has the hard-coded `20000` overflow recovery hint, so the PR addresses a real gap, but the branch still formats suggested config values as grouped display strings like `50,000`; that is not the right shape for a JSON5 numeric config field validated as an integer.

## What This Changes

The PR replaces the hard-coded context-overflow `reserveTokensFloor` hint with model-context-window tiers and adds regression tests for 1M, 200k, 100k, smaller, and unknown context windows.

## Best Possible Solution

Keep the context-window-aware guidance if maintainers accept the tiers, but emit the suggested `reserveTokensFloor` as an ungrouped JSON5 numeric literal such as `50000` or as a full parseable config snippet, and make the tests assert the parseable output.

## Reproduction Assessment

Yes. Inspection of the PR diff shows `toLocaleString("en-US")` in the recovery hint path, while current config loading uses JSON5 and the schema requires `reserveTokensFloor` to be an integer.

## Solution Assessment

No. The direction is narrow and useful, but the current patch is not the best mergeable form because the suggested value is formatted for display instead of copy-pasteable configuration.
