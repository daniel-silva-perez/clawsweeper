## Summary

Keep open. Current main still has the reported `models status --probe --agent <id>` gap: `modelsStatusCommand()` resolves the target `agentDir`, but the probe pipeline does not accept or forward it, and `buildProbeTargets()` still calls `ensureAuthProfileStore()` with no agent directory. The related PR #67263 appears to be the right narrow fix, but it is closed and unmerged, so the issue is not implemented on current main.

## Best Possible Solution

Review and apply the narrow #67263 approach on current main: add an optional `agentDir` parameter through `runAuthProbes()` and `buildProbeTargets()`, pass the `modelsStatusCommand()` resolved `agentDir`, and add regression tests that assert both the supplied-agent path and default-store path.
