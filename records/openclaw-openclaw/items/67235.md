## Summary

Keep open. Current main still has the reported gap: `modelsStatusCommand()` resolves the target `agentDir` and uses it for the non-probe auth overview, but the probe target planner still calls `ensureAuthProfileStore()` with no `agentDir`, so probe planning can read the default/main auth profile store instead of the requested agent store. The directly related fix PR #67263 is closed and unmerged, so it is useful context but not shipped behavior.

## Best Possible Solution

Review and apply the #67263 approach on current main, but extend it far enough to keep the whole probe pipeline agent-scoped: pass the already-resolved `agentDir` and, if needed, `agentId`/workspace/session context from `modelsStatusCommand()` through `runAuthProbes()`, `buildProbeTargets()`, and probe execution. Add regression tests that assert `--agent <id>` forwards the target agent directory into `ensureAuthProfileStore(agentDir)` and that omitting `--agent` preserves the default-store path.
