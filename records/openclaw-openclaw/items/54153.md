## Summary

Keep open. Current main partially hardened the Signal installer by moving the archive download to the shared SSRF guard and adding a streamed byte cap, but the remaining timeout/test/cleanup expectations are not fully settled, and open PR #73489 is already the linked implementation candidate for this issue.

## What This Changes

The issue requests hardening the Signal CLI installer download path against hangs, redirect SSRF, oversized downloads, and proxy bypass by replacing its raw HTTP download helper with the shared guarded fetch path.

## Best Possible Solution

Review and land #73489, or an equivalent Signal plugin fix, that completes downloader hardening in the owner module using the existing plugin SDK SSRF seam: explicit timeout, declared and streamed size checks, conservative proxy-mode choice, partial-download cleanup, safe archive path handling, and focused Signal installer tests. Keep the issue open until that work is merged.
