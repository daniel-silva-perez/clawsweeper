## Summary

Keep open. Current main does not provide enough evidence that the reported Telegram group-session data loss is fixed: ordinary non-forum Telegram group messages still do not pass `updateLastRoute`, the related fix PR was closed unmerged, and the broader restart/persistence symptom from the issue discussion remains unverified. The issue’s expectation of multiple sessions is not aligned with current docs, but the missing-history problem is still actionable.

## Best Possible Solution

Keep this issue open and handle it in the Telegram plugin/session persistence path. The best next step is a targeted regression test for ordinary non-forum Telegram group messages that confirms triggered group turns append to the durable session transcript and survive gateway restart/reload, plus a docs or maintainer reply clarifying that groups intentionally use one stable session key rather than many sessions.
