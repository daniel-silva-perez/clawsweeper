## Summary

Keep this PR open for maintainer review, but do not merge the current patch: it adds generic runner-level `options.metadata` stamping, while current LiteLLM calls use the `openai-completions` transport path that does not serialize that metadata into the request payload, and the proposed fields include stable channel/session identifiers without a provider opt-in boundary.

## What This Changes

The PR adds a `runEmbeddedAttempt` `streamFn` wrapper that stamps `session_key`, `message_channel`, and `context_limit` into outbound model-call `options.metadata` when those run parameters are present.

## Best Possible Solution

Add a narrow, tested LiteLLM/OpenAI-completions metadata path that serializes only approved string fields into the actual request payload, preserves existing caller/provider metadata, gates the behavior by provider or explicit config, and documents the privacy semantics for session identifiers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Propagate metadata into the LiteLLM request payload:** `src/agents/pi-embedded-runner/run/attempt.ts:2458-2462`
  - body: This wrapper only writes to `options.metadata`, but LiteLLM is registered as `openai-completions` and that transport's payload builder does not copy `options.metadata` into chat-completion params. As written, LiteLLM callbacks still will not receive these fields.
  - confidence: 0.91
- **[P2] Gate session metadata before sending it to providers:** `src/agents/pi-embedded-runner/run/attempt.ts:2447-2451`
  - body: The wrapper is installed for any provider whenever session fields are present. Since `session_key` can include channel, group, topic, thread, or DM routing identifiers, this needs a LiteLLM-only or explicit opt-in boundary before arbitrary provider streams can receive it.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The diff has no dependency, workflow, or supply-chain changes, but it introduces an ungated outbound metadata path for stable session/channel identifiers.
