## Summary

Keep this issue open. Current main still treats an exit-0 `codex-cli` JSONL run as a normal completed CLI run: failure parsing is only reached for nonzero or non-exit process results, the JSONL parser returns text/session/usage without a semantic failure flag, and the runner records the attempt as `success` with `stopReason: "completed"`. The focused fix candidate #39460 is closed unmerged, and the related #66081 work explicitly says it does not change failure classification.

## Best Possible Solution

Keep this issue open and revive the substance of #39460 on current main: parse structured Codex JSONL records for sandbox-denied or failed command/tool/action/error/status signals after exit 0, fail before `buildCliRunResult`, add an explicit `sandbox_denied` failover/status/auth-profile policy if maintainers want that contract, avoid assistant-message keyword matching, and add focused parser plus runner regression tests. Treat #66081 as related final-answer reliability work, not the semantic failure fix.
