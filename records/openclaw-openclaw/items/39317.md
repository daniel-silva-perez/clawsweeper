## Summary

Keep this issue open. Current main still treats an exit-0 `codex-cli` JSONL run as successful after stdout parsing, with no structured sandbox-denial or semantic tool/action failure signal. The focused fix PR #39460 was closed unmerged, and the related #66081 work is scoped to final-message artifacts rather than failure classification.

## Best Possible Solution

Keep the issue open and revive the substance of #39460 on current main: parse structured Codex JSONL records for sandbox-denied or failed command/tool/action/error/status signals after exit 0, fail before `buildCliRunResult`, add an explicit `sandbox_denied` failover/status/auth-profile policy if maintainers want that contract, avoid assistant-message keyword matching, and add focused parser plus runner regression tests. Treat #66081 as related final-answer reliability work, not the semantic failure fix.
