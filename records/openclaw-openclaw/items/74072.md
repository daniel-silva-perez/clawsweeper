## Summary

Keep open. Current main still builds and runs one boot-md startup task per agent, so two agent entries that resolve to the same configured workspace will run the same BOOT.md more than once. Existing tests assert the per-agent behavior rather than unique-workspace behavior, and the reported release contains the same handler shape.

## Best Possible Solution

Keep the issue open and fix boot-md in the bundled hook handler by deduping startup tasks by resolved workspace path for a single gateway lifecycle, while preserving per-agent BOOT.md runs for distinct workspaces. Update the handler unit/integration tests to cover shared-workspace dedupe and distinct-workspace behavior, and add a user-facing changelog fix entry crediting the reporter.
