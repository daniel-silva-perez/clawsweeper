## Summary

Keep this open. Current main has adjacent provider 429 classification, cooldown/fallback handling, and usage/quota readouts, but it still lacks the requested configurable outbound rate limits, queueing/blocking, token or cost budgets, warning thresholds, alerts, and `openclaw limits` CLI. The issue discussion also has maintainer signal to keep this as the broad canonical tracker, with prior implementation work closed unmerged and narrower related issues still open.

## What This Changes

The issue requests configurable rate limiting, throttling, token/cost budgets, observability, alerts, and CLI controls for outbound external API calls.

## Best Possible Solution

Keep this as the canonical core design tracker. The desired end state is a maintainer-scoped, opt-in outbound policy layer that starts with model-provider dispatch, explicitly defines inclusion criteria for tools/search/channels/plugins, and aligns schema, persistence, status/CLI UX, warning thresholds, 429 backoff behavior, token/cost budgets, docs, and focused tests.
