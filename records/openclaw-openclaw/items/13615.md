## Summary

Keep this open. Current main has adjacent reactive provider 429/cooldown/fallback handling, provider usage readouts, gateway auth/control-plane throttles, and web_search caching/fallbacks, but it still lacks the requested configurable outbound API limiter, pre-dispatch queueing/blocking, token/cost budget enforcement, warning thresholds, alerts, and `openclaw limits` CLI. The issue discussion also has maintainer signal keeping this as the broad canonical tracker, while #13686 was closed unmerged rather than shipped.

## What This Changes

The issue requests configurable per-provider external API rate limits, throttling/backoff, token/cost budgets, observability, alerts, and `openclaw limits` CLI commands.

## Best Possible Solution

Keep this as the canonical core design tracker. The desired end state is a maintainer-reviewed, opt-in outbound policy layer that starts with model-provider dispatch, explicitly defines whether tools/search/channels/plugins are in scope, and aligns config schema, persistence, status/CLI UX, warning thresholds, 429 backoff, token/cost budgets, docs, and focused tests.
