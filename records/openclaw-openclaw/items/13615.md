## Summary

Keep open. Current main still has adjacent provider cooldown/fallback handling, usage readouts, gateway auth throttling, Codex account-limit readouts, and channel-specific retry helpers, but it does not implement the requested configurable outbound API rate limits, pre-dispatch queue/block behavior, token or cost budget enforcement, warning/alert thresholds, or `openclaw limits` CLI.

## What This Changes

The issue requests configurable rate limiting, throttling, retry/backoff, cost budgets, observability, and CLI controls for external API calls across providers, tools, and channels.

## Best Possible Solution

Keep this as the canonical core design tracker. The desired end state is a maintainer-scoped, opt-in outbound policy layer that starts with model-provider dispatch, explicitly defines which tool/search/channel/plugin calls are in scope, and aligns schema, persistence, status/CLI UX, warning thresholds, 429 backoff behavior, token/cost budgets, docs, and focused tests.
