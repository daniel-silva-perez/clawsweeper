## Summary

Keep this open. Current main still has reactive provider 429/cooldown/fallback handling, usage/cost readouts, gateway/control-plane auth throttles, and plugin-specific rate-limit helpers, but it does not implement the requested configurable outbound API policy layer for pre-dispatch rate limiting, queueing/blocking, token/cost budget enforcement, warning thresholds, alerts, or an `openclaw limits` CLI. The existing maintainer discussion also treats this as the broad canonical tracker rather than a stale or plugin-only request.

## What This Changes

Request to add configurable rate limiting, throttling, cost budgets, observability, alerts, and CLI controls for external API calls across providers, web search, and channel integrations.

## Best Possible Solution

Keep this as the canonical core design tracker. The desired end state is a maintainer-reviewed, opt-in outbound policy layer that starts with model-provider dispatch, explicitly defines inclusion criteria for tools/search/channels/plugins, and aligns config schema, persistence, status/CLI UX, warning thresholds, 429 backoff, token/cost budgets, docs, and focused tests.
