## Summary

Keep this issue open. Current main still lacks the requested configurable outbound API rate limiting/throttling layer, token/request budget enforcement, queueing/status controls, and `openclaw limits` CLI. Existing code covers adjacent reactive 429/failover handling, auth-profile cooldowns, inbound gateway/webhook rate limits, Codex-account quota readouts, and usage/cost reporting, but those do not satisfy the proposed cross-provider pre-dispatch policy.

## Best Possible Solution

Keep this issue open as the canonical core product/design tracker. The best path is an opt-in outbound policy layer at model-provider, tool/web-search, and channel request boundaries, with config/schema/docs for provider/model/global RPM, TPM, daily/monthly windows, queue/backpressure controls, 429 retry/backoff behavior, token/cost budget enforcement, warning/status reporting, and an `openclaw limits` CLI. Treat #13686 as prior art and #14206/#42475/#60450 as narrower related follow-ups, not replacements.
