## Summary

Keep open. Current main has adjacent reactive 429 classification, auth-profile cooldown/fallback behavior, provider usage readouts, and gateway/control-plane rate limits, but it still lacks the requested configurable outbound API rate limits, queueing/blocking, token or cost budget enforcement, warning thresholds, alerts, and `openclaw limits` CLI. The existing issue discussion also has maintainer signal to keep this as the broad canonical tracker, with prior implementation work closed unmerged and narrower related issues still open.

## What This Changes

The issue requests configurable rate limiting, throttling/backoff, cost budgets, observability, and CLI controls for OpenClaw outbound API calls.

## Best Possible Solution

Keep this as the canonical core design tracker. The desired end state is a maintainer-scoped, opt-in outbound policy layer that starts with model-provider dispatch, explicitly defines inclusion criteria for tools/search/channels/plugins, and aligns schema, persistence, status/CLI UX, warning thresholds, 429 backoff behavior, token/cost budgets, docs, and focused tests.
