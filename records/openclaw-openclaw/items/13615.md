## Summary

Keep this issue open. Current main still has adjacent reactive 429 handling, model failover/cooldowns, inbound gateway auth rate limits, provider quota readouts, and historical cost reporting, but it does not implement the requested configurable outbound API rate limiting/throttling layer, token/request budget enforcement, queueing/status controls, or `openclaw limits` CLI. The April 26 maintainer comment in the issue explicitly kept this open as the canonical tracker, and the related PR #13686 was closed unmerged.

## Best Possible Solution

Keep this open as the canonical core product/design tracker. The best path is an opt-in outbound policy layer at model-provider, tool/web-search, and channel request boundaries, with config/schema/docs for provider/model/global RPM, TPM, daily/monthly windows, queue/backpressure controls, 429 retry/backoff behavior, token/cost budget enforcement, warning/status reporting, and an `openclaw limits` CLI. Treat #13686 as prior art and #14206, #42475, and #60450 as narrower related follow-ups, not replacements.
