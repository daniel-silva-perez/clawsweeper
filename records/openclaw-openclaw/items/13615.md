## Summary

Keep this open. Current main has adjacent auth/control-plane/channel rate-limit guards, provider 429 cooldown/fallback handling, and usage/cost readouts, but it still does not implement configurable outbound rate limiting, queueing, token/cost budget enforcement, warning thresholds, alerts, or an `openclaw limits` CLI. The existing maintainer discussion also treats this as the canonical broad design tracker rather than a stale or superseded request.

## What This Changes

Add configurable rate limiting, throttling, usage visibility, and token/cost budget controls for external API calls across providers and integrations.

## Best Possible Solution

Keep this as the canonical core design tracker. The desired end state is a maintainer-reviewed, opt-in outbound policy layer that starts with model-provider dispatch, explicitly defines whether tools/search/channels/plugins are in scope, and aligns config schema, docs, persistence, status/CLI UX, warning thresholds, 429 backoff, and token/cost budget enforcement.
