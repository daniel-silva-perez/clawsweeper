## Summary

Keep this issue open. Current main still has only adjacent reactive provider 429 handling, auth-profile cooldown/fallback behavior, provider usage readouts, and inbound/control-plane/webhook rate limiters. It does not implement the requested configurable outbound API rate limiting/throttling layer, queueing, token/request budget enforcement, warning/status controls, or `openclaw limits` CLI, and a maintainer comment already kept this open as the canonical tracker.

## Best Possible Solution

Keep this open as the canonical core product/design tracker. The best path is an opt-in outbound policy layer at model-provider, tool/web-search, and channel/plugin request boundaries, with config/schema/docs for provider/model/global RPM, TPM, daily/monthly windows, queue/backpressure controls, 429 retry/backoff behavior, token/cost budget enforcement, warning/status reporting, and an `openclaw limits` CLI. Treat #13686 as prior art and #14206, #42475, #60450, and #45771 as narrower related follow-ups, not replacements.
