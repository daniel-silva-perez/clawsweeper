## Summary

Keep this PR open. Current main still has the false-positive path the PR targets: the security audit builds known node commands from platform allowlists with denyCommands cleared, but it does not add DEFAULT_DANGEROUS_NODE_COMMANDS. Since commands such as camera.snap, screen.record, and sms.send are intentionally not default-allowlisted, putting them only in gateway.nodes.denyCommands can still be reported as unknown/ineffective. The PR patch is narrowly aimed at the right function and does not introduce an obvious security or supply-chain concern, but it should gain a focused regression test before landing.

## Best Possible Solution

Keep the PR open and land or recreate its small listKnownNodeCommands() fix on current main, with a regression test in src/security/audit-node-command-findings.test.ts proving that denyCommands entries such as camera.snap, screen.record, and sms.send do not emit gateway.nodes.deny_commands_ineffective while true unknown and pattern-like entries still do.
