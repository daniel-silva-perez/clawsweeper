## Summary

Keep this PR open. Current main still has the false-positive path: `listKnownNodeCommands()` builds the audit known-command set from platform/default allowlists with `denyCommands` cleared, but does not union `DEFAULT_DANGEROUS_NODE_COMMANDS`. The PR is a focused one-file audit-classification fix and does not appear to broaden execution, dependency, CI, release, or secret-handling surfaces, but it should land with a regression test for dangerous-but-valid deny-only commands.

## Best Possible Solution

Keep the PR open and land this small `listKnownNodeCommands()` fix, or recreate it on current main, with a regression test in `src/security/audit-node-command-findings.test.ts` proving dangerous-but-valid deny-only entries do not emit `gateway.nodes.deny_commands_ineffective` while typos and pattern-like entries still do.
