## What This Changes

This PR adds channel-backed session detection to main-session restart recovery, marks matching interrupted sessions failed for manual recovery before synthetic resume, and adds a Vitest regression.

## Best Possible Solution

Merge a targeted restart-recovery guard once maintainers accept the channel metadata predicate, preserving non-channel resumable recovery and focused agents-core regression coverage.

## Reproduction Assessment

Yes. Seed a running `abortedLastRun` channel-backed main session with a resumable tool-result tail and run restart recovery; current main proceeds toward `callGateway`, while the PR’s expected behavior is failed/manual recovery without replay.

## Solution Assessment

Yes. The guard sits at the restart-recovery decision point before replay can re-enter agent run machinery, which is narrower than changing channel delivery paths; the remaining judgment is whether the classifier should cover every persisted channel-route field.
