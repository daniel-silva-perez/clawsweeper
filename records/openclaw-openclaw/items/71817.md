## Summary

Keep this PR open. Current main still leaves Telegram and Feishu preview gating tied to session-store state or hardcoded off defaults, so the PR addresses a real remaining channel bug. The branch should not be merged as-is without maintainer review because its new fallback lookup drifts from current main’s normalized agent config resolution and newly shipped global `agents.defaults.reasoningDefault` contract.

## What This Changes

The PR updates Telegram and Feishu reasoning-preview resolution to use `agents.list[].reasoningDefault` as a fallback, passes config and agent IDs into Feishu call sites, and adds extension tests for config fallback and explicit session off behavior.

## Best Possible Solution

Update the branch so Telegram and Feishu resolve the same effective reasoning default that current main exposes: explicit session state first, then normalized per-agent default, then global default, then `off`, while preserving Feishu’s stream-only preview behavior and adding focused tests for normalized agent IDs, global defaults, and explicit session `off`.
