## Summary

Keep this PR open. Current main still does not apply `agents.list[].reasoningDefault` in the Telegram or Feishu reasoning-preview paths, while the public config/docs contract says per-agent reasoning defaults should be part of resolution. The PR is focused on the affected plugin-owned files and remains a plausible implementation candidate for maintainer/channel-owner review, not something already implemented on main.

## Best Possible Solution

Keep this PR open for maintainer/channel-owner review. The best path is to land or lightly adjust the focused fix so Telegram and Feishu use `agents.list[].reasoningDefault` as the fallback while preserving explicit session overrides and Feishu's stream-only preview behavior. Broader `agents.defaults.reasoningDefault`, per-model defaults, and agent-id normalization should stay separate unless maintainers choose to expand the scope.
