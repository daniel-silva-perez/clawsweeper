## Best Possible Solution

Keep the channel behavior fix, but either fully register the new helper as a documented Plugin SDK subpath or keep the helper on an existing exported surface, then rerun the focused Telegram, Feishu, and Plugin SDK checks before maintainer review.

Yes. On current main, configure a per-agent or global `reasoningDefault`, leave the Telegram or Feishu session without a stored `reasoningLevel`, and source inspection shows Telegram resolves `off` while Feishu preview resolves `false`.

## Solution Assessment

No, not fully as proposed. The fallback resolution strategy is the narrow maintainable direction, but publishing a new SDK subpath without docs/API metadata leaves the public contract incomplete; adding those artifacts or using an existing SDK surface is safer.
