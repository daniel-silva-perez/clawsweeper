## Summary

Keep this issue open. Current main still does not include a built-in Accessible Control UI theme preset; the shipped surface is Claw, Knot, Dash, plus one browser-local tweakcn custom import slot, which does not satisfy the requested first-class, reusable high-contrast color-vision-deficiency-safe preset.

## What This Changes

Add a built-in Accessible Control UI theme preset with high-contrast, color-vision-deficiency-safe CSS tokens and Control UI picker support.

## Best Possible Solution

Add a focused built-in Accessible theme family through the existing Control UI theme contract: theme parsing/resolution, first-paint bootstrap, quick and advanced Appearance pickers, CSS variable overrides, targeted tests, docs, and a user-facing changelog entry. Keep the tweakcn custom import and #28300 theme-studio discussion as complementary surfaces rather than substitutes.

Not applicable as a runtime bug reproduction; this is a feature request. We do have a high-confidence current-main verification path: inspect the theme contract, bootstrap, picker lists, CSS theme blocks, docs, and tests, all of which currently omit Accessible.

## Solution Assessment

Yes, the narrow maintainable solution is to add a built-in theme preset using the existing theme-family pattern. The current browser-local tweakcn import is not an adequate substitute because it is local-only, not documented as a preset, and does not give all users a shipped accessible option.
