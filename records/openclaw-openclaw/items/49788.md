## Summary

Current main still does not ship a first-class Accessible Control UI theme preset. The theme contract, first-paint bootstrap, Appearance pickers, docs, and CSS tokens still cover Claw, Knot, Dash, and one browser-local tweakcn custom slot, which does not satisfy the requested built-in color-vision-deficiency-safe high-contrast preset.

## Best Possible Solution

Keep this issue open for a focused Control UI accessibility theme implementation: add an accessible theme family to ui/src/ui/theme.ts, ui/index.html, the quick and advanced Appearance pickers, CSS variable overrides in ui/src/styles/base.css, tests around parsing/resolution/application, and the public Control UI docs at https://docs.openclaw.ai/web/control-ui. Treat the existing tweakcn custom import and #28300 theme-studio discussion as complementary, not replacements.
