## Summary

Keep open. Current main does not implement an Accessible built-in Control UI theme preset: the theme model, advanced Appearance panel, quick settings panel, bootstrap script, docs, and CSS variables all show Claw/Knot/Dash plus a browser-local custom import slot, not a first-class color-vision-deficiency-safe high-contrast preset.

## Best Possible Solution

Keep this issue open for a focused Control UI implementation: add an `accessible` theme family to the theme type/resolver, first-paint bootstrap, advanced and quick Appearance pickers, CSS variable overrides for CVD-safe high-contrast dark tokens, tests, and Control UI docs. The broader custom theme studio thread can remain separate; it does not replace a built-in accessibility preset.
