## Summary

Current main still has the cron heartbeat merge behavior this PR targets: main-session systemEvent cron jobs force `heartbeat.target = "last"`, but the gateway cron adapter merges that override with inherited default/agent heartbeat config and keeps explicit `to`/`accountId` fields that outbound delivery treats as routing inputs. The PR is not obsolete and should be reviewed or replaced by an equivalent narrow fix.

## Best Possible Solution

Review and land this PR, or an equivalent narrowly scoped fix, so cron-forced `target: "last"` wakes drop inherited explicit heartbeat destinations while preserving non-routing heartbeat settings. Keep the regression coverage in `src/gateway/server-cron.test.ts` and the changelog credit for linked #73900.
