## Summary

Keep this PR open. Current main still has the cron heartbeat merge behavior described by #73900: main-session systemEvent cron wakes force `target: "last"`, but the gateway cron wrapper still shallow-merges that override over resolved heartbeat defaults and preserves inherited `to`/`accountId`, which downstream routing treats as explicit delivery input. The PR is a focused implementation candidate and is not obsolete.

## Best Possible Solution

Review and land this PR, or an equivalent narrowly scoped fix, so cron-forced `target: "last"` wakes drop inherited explicit heartbeat destinations while preserving non-routing heartbeat settings. Keep the gateway cron regression test and the changelog credit for #73900.
