## Summary

Keep this PR open. Current main still has the cron heartbeat merge behavior described by the linked #73900: main-session cron wakes force `heartbeat.target = "last"`, but the gateway wrapper merges that override over resolved default/agent heartbeat config and preserves inherited explicit destination fields. Downstream heartbeat routing treats `heartbeat.to` and `heartbeat.accountId` as explicit inputs, so the PR is not obsolete and should be reviewed/landed or replaced by an equivalent fix.

## Best Possible Solution

Review and land this PR, or an equivalent narrowly scoped fix, so cron-forced `target: "last"` wakes drop inherited explicit heartbeat destinations while preserving non-routing heartbeat settings. Keep the regression coverage in `src/gateway/server-cron.test.ts` and the changelog credit for #73900.
