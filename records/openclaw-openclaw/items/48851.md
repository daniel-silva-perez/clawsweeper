Keep this PR open. Current main still does not implement the intended API/model call-count feature in /status, the response usage footer, runner metadata, or session persistence. The PR is a plausible implementation candidate because the author addressed review feedback around followup persistence and tool-loop call counting, but it needs rebasing onto the current status/usage refactor and a clear decision on zero/usage-missing call semantics before merge.

Keep this PR open as an implementation candidate.
