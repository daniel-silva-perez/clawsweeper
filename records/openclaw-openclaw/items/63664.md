## Summary

Keep open. Current main still installs an exact-path memory-flush write wrapper that throws the reported restriction for every path except the active daily memory file, including unrelated workspace files. The direct fix PR is closed unmerged, and the related open issue about SESSION_HANDOFF.md is narrower than this broader workspace-write failure.

## What This Changes

Allow memory-triggered flush turns to append to the active daily memory file while still permitting unrelated workspace writes through normal write-tool safeguards.

## Best Possible Solution

Update `wrapToolMemoryFlushAppendOnlyWrite` so the active daily memory file is append-only, other `memory/` targets remain blocked, and non-memory workspace paths are delegated to the original write tool with regression coverage for that pass-through behavior.
