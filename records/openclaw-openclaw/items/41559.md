Keep open. Current main does not implement the requested ingress echo, nested relay guard, or dual-channel A2A relay behavior, and the open draft PR is still useful source work, but the current diff has blocking correctness and security-sensitive routing issues before it can merge.

## What This Changes

## Best Possible Solution

Land a rebased version only after the A2A delivery behavior remains opt-in, all new sends and history reads use the scoped gateway caller, strict relay delivery fails closed when configured, generated config artifacts are refreshed, and focused tests cover ingress echo, nested relay blocking, dual-channel relay, strict-failure, best-effort, and bound-channel protection paths.

## Security Review

Status: needs_attention

Summary: The diff does not add dependencies or CI execution paths, but it introduces new external channel relay sends with concrete gateway-context and strict-delivery concerns.

Concerns:

- **[medium] New delivery path bypasses injected gateway context:** `src/agents/tools/sessions-send-tool.ts:297`
  - body: The ingress echo send and nested guard history lookup call the module-level gateway helper instead of the injected caller selected by `createSessionsSendTool`, which can bypass scoped gateway resolution while sending to external channels or reading history.
  - confidence: 0.84
- **[low] Strict relay policy is parsed but not enforced:** `src/agents/tools/sessions-send-tool.a2a.ts:88`
  - body: Relay send failures are swallowed even when `relay.requireDelivery` is true. Because this feature mirrors inter-agent content to external channels, strict delivery mode should fail closed when configured.
  - confidence: 0.85
