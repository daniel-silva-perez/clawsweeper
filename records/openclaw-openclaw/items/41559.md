## Summary

Keep open. Current main still lacks the requested A2A ingress echo and dual-channel relay behavior, and this draft PR remains unmerged with merge-blocking functional and security concerns in the new external delivery paths.

## What This Changes

This draft PR adds opt-in `sessions_send` A2A ingress echo, nested relay guarding, dual-channel relay mirroring with announce suppression, config/docs updates, bound-channel protection, and focused tests.

## Best Possible Solution

Land a rebased version only after the A2A delivery design preserves scoped gateway injection, blocks nested recursive sends before any external delivery, enforces strict relay delivery fail-closed when configured, refreshes generated config artifacts, and adds focused tests for those paths.

## Reproduction Assessment

Yes. Enable ingress echo or relay on the PR head with an injected gateway caller to observe the new paths using the module-level gateway, configure `relay.requireDelivery=true` with a rejecting relay send to see the flow continue, and enable ingress echo during a nested `sessions_send` to see the external echo happen before the guard blocks the run.

## Solution Assessment

No. The opt-in feature direction is plausible, but the current PR is not the best mergeable solution until the delivery ordering, scoped gateway usage, strict failure semantics, and generated config baseline are corrected.

## Security Review

Status: needs_attention

Summary: The diff adds no new dependency or CI execution path, but it introduces external A2A channel sends that need scoped-gateway, guard-ordering, and strict-delivery fixes before merge.

Concerns:

- **[medium] New sends bypass scoped gateway context:** `src/agents/tools/sessions-send-tool.ts:297`
  - body: The ingress echo path calls the module-level gateway helper instead of the injected caller selected by `createSessionsSendTool`, which can bypass scoped gateway resolution while sending to external channels.
  - confidence: 0.86
- **[medium] Nested guard runs after external echo:** `src/agents/tools/sessions-send-tool.ts:357`
  - body: A recursive `sessions_send` can post the ingress echo before the guard returns `forbidden`, undermining the default nested relay guard and creating an external message side effect from a blocked request.
  - confidence: 0.84

  - confidence: 0.87
