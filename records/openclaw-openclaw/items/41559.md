  - body: The ingress echo send and nested guard history lookup call the module-level gateway helper even though createSessionsSendTool has selected opts.callGateway as gatewayCall.   - confidence: 0.83

## Security Review

Status: needs_attention

Summary: Security review needs attention because the new external channel relay paths bypass injected gateway context and do not enforce strict relay delivery failures.

Concerns:

- **[medium] New delivery path bypasses injected gateway context:** `src/agents/tools/sessions-send-tool.ts:297`
  - body: The new ingress echo send and nested guard history lookup use the module-level gateway caller instead of the caller injected into createSessionsSendTool. Contexts that intentionally scope gateway access can therefore send to external channels or read history through the wrong gateway resolution path.
  - confidence: 0.84
- **[low] Strict relay policy is parsed but not enforced:** `src/agents/tools/sessions-send-tool.a2a.ts:88`
  - body: Relay send failures are swallowed even when relay.requireDelivery is true. Because this feature mirrors inter-agent content to external channels, strict delivery mode should fail closed when configured.
  - confidence: 0.85
