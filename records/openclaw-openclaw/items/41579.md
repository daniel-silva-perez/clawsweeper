## Summary

Keep open. Current main still lacks a permanent Control UI/Mission Control File Gallery tab, route, or left-nav label, and there is no built-in workspace docs/images gallery. The reporter's workaround (`refresh_gallery` plus local HTML) remains outside the app. Current file support is chat attachments plus the Agents > Core Files editor for bootstrap/persona files, not a workspace-wide document/image gallery.

## Best Possible Solution

Keep this open. Implement as a core Control UI feature or first extend the plugin Control UI seam to support nav/view contributions. The likely product path is a scoped Gateway RPC/HTTP surface for indexing workspace docs/images, safe authenticated open/download URLs, a File Gallery tab in TAB_GROUPS or equivalent plugin-projected nav, Refresh Gallery and Open in Browser actions, friendly empty/error states, and tests/docs for workspace containment and unsupported files.
