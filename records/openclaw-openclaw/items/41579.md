## Summary

Keep open. Current main still lacks the requested Control UI/Mission Control File Gallery left-nav entry and standalone workspace docs/images gallery; existing file surfaces are chat attachments and the restricted Agents Core Files editor. Because a real implementation would expose workspace file indexing/opening through the Gateway, it needs maintainer product and security design rather than cleanup closure.

## What This Changes

Request a permanent Mission Control left-nav File Gallery view for workspace documents/images with refresh, open-in-browser, and friendly empty/error states.

## Best Possible Solution

Design a scoped workspace File Gallery surface: authenticated Gateway index/open/download APIs constrained to the configured workspace, a Control UI tab with refresh and open-in-browser actions, clear empty/error states, docs/images filtering, and tests for containment and unsafe paths.
