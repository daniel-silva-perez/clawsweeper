## Summary

Current main still does not provide a permanent Control UI/Mission Control File Gallery left-nav item or a workspace docs/images gallery. Existing UI file support is limited to chat attachments and the Agents > Files editor for bootstrap/persona files, so the requested daily workflow remains a real core UI feature gap.

## Best Possible Solution

Keep this open and implement it as a core Control UI feature or first add a supported Control UI extension seam. A good path is a scoped Gateway RPC/HTTP surface for indexing workspace docs/images, safe authenticated open/download URLs, a File Gallery tab in TAB_GROUPS with refresh/open-in-browser actions, empty/error states, and tests/docs covering workspace path containment and unsupported files.
