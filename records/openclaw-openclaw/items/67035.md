## Summary

Current main fixes some adjacent Control UI live-update problems, but the central report is still only partially implemented. The composer still lacks focused/IME draft preservation, and the chat error path still discards partial streamed text instead of leaving visible output or a terminal assistant message.

## Best Possible Solution

Keep this issue open or split it deliberately. Preserve the already-shipped optimistic-history, deferred active-run reload, and active-memory timeout fixes, then land focused Control UI work for IME/focused composer draft preservation in `ui/src/ui/views/chat.ts` with browser regression coverage and chat-error partial-stream retention in `ui/src/ui/controllers/chat.ts` with controller tests. Track the broader accepted/processing placeholder UX as a narrower follow-up if it is not included here.
