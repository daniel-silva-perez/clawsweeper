Keep this open. Current main has fixed several adjacent WebChat live-update races, but it still does not implement the remaining composer IME/focused-draft preservation or the error-path partial-stream retention described in the issue thread and the prior ClawSweeper review.

Treat the broader explicit accepted/processing placeholder model as a follow-up unless maintainers choose to include it deliberately.

## Repair Work Prompt

Fix the remaining Control UI regressions tracked by https://github.com/openclaw/openclaw/issues/67035. Keep the scope narrow: preserve the already-shipped active-run history reload deferral and active-memory timeout behavior, and do not change provider routing, gateway protocol, active-memory recall behavior, or broad accepted/processing UX unless a maintainer explicitly expands scope.

Implement two focused fixes:
1. In `ui/src/ui/views/chat.ts`, preserve focused/IME composer draft text. Add ephemeral focused/composing state for the chat textarea, use the live active textarea value for `.value` and `dir` while focused or composing, set focus state on input/focus/compositionstart, flush `onDraftChange` on compositionend and blur, and keep Enter-send behavior safe during IME composition.
2. In `ui/src/ui/controllers/chat.ts`, make the `payload.state === "error"` branch retain visible output: if `chatStream` has non-silent text, append it as an assistant text message before clearing stream state; otherwise, if `errorMessage` has visible text, append a concise assistant terminal message such as `Generation failed: ...`. Continue setting `lastError` and clearing active run state for the owned run.

Add browser/controller regression coverage in `ui/src/ui/views/chat.test.ts` and `ui/src/ui/controllers/chat.test.ts`. Update `docs/web/control-ui.md` if behavior changes are user-visible, and add a single-line changelog entry with proper reporter credit such as `Thanks @q7793527` if the project convention requires it for this user-facing fix.
