## Summary

Keep open. Current main still has only the existing user/plugin/workspace hook discovery and user-configured enable/disable path; the requested admin policy tier is a real core governance gap and needs maintainer security design before implementation.

## What This Changes

Request a machine-wide, admin-managed policy hook discovery tier that loads before user-controlled hooks and cannot be disabled by user config or `openclaw hooks disable`.

## Best Possible Solution

Design and implement a core policy-hook tier with hardened cross-platform discovery, policy-first ordering, immunity from user toggles, policy-aware CLI inventory, public docs, and regression coverage for POSIX and Windows trust-boundary behavior.

## Reproduction Assessment

Not applicable as a feature request. The gap is high-confidence from current source and docs: hook discovery, source typing, startup gating, trigger execution, and CLI status/toggle paths all lack a machine-wide policy tier.

## Solution Assessment

Unclear pending maintainer security design. If OpenClaw accepts this capability, a core runtime policy tier appears to be the maintainable layer because plugins and hook packs cannot enforce OS-owned roots, trusted precedence, or immunity from user configuration.
