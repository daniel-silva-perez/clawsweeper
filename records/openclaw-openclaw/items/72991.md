## Summary

Keep open. Current main and the latest stable tag still expose only bundled, plugin, managed, and workspace hook tiers, all without the requested machine-wide policy source, OS-owned discovery root, user-disable immunity, or policy-only CLI inventory. The request is concrete and valid, but it changes a security-sensitive core trust boundary, so it needs maintainer/security design rather than conservative cleanup or an automated fix lane.

## What This Changes

Add an admin-managed machine-wide policy hook discovery tier that always loads before user-controlled hooks and cannot be disabled through user configuration or `openclaw hooks disable`.

## Best Possible Solution

Design and implement a core policy-hook tier with hardened machine-wide discovery, policy-first ordering, immunity from user config disables, policy-aware CLI list/info output, public docs, and regression coverage across POSIX and Windows trust-boundary cases.
