## Summary

Keep open. The prior ClawSweeper assessment still matches current main: OpenClaw has no admin-owned machine-wide hook policy tier, and this is core security/product design work because it needs OS-owned discovery roots, policy-first ordering, user-disable immunity, and auditable CLI inventory.

## What This Changes

Request an admin-managed machine-wide hook discovery tier whose hooks always load first, cannot be disabled by user config or `openclaw hooks disable`, and are exposed in policy-aware CLI inventory.

## Best Possible Solution

Define a maintainer-approved core policy-hook design, then implement hardened POSIX and Windows discovery, deterministic policy-first dispatch, fail-open crash handling with honored block decisions, disable immunity, policy-aware CLI output, docs, and regression tests.

## Reproduction Assessment

Not applicable as a feature request. The missing capability is high-confidence from current source, tests, and docs because all existing hook tiers are user/plugin/workspace scoped and gated by user-controlled configuration.

## Solution Assessment

Unclear until maintainer security design is settled. If OpenClaw accepts this enterprise governance capability, the core hook loader/dispatcher/CLI is the right layer because current plugin and hook-pack surfaces cannot enforce OS-owned roots, trusted ordering, or user-disable immunity.
