## Summary

Keep open. Current main still lacks an admin-owned machine-wide policy hook tier: hook sources, discovery, load gating, trigger gating, CLI inventory/toggle behavior, tests, and public docs all remain built around bundled, plugin, managed, and workspace hooks. The request is concrete and core/security-sensitive because plugins or user-managed hook packs cannot enforce OS-owned roots, policy-first ordering, or immunity from user config.

## What This Changes

Request to add an admin-managed, machine-wide policy hook discovery tier that runs before user hooks and cannot be disabled by user-controlled hook settings.

## Best Possible Solution

Design a core policy-hook tier with hardened machine-wide discovery, policy-first execution, immunity from user hook toggles, policy-aware CLI inventory, public docs, and regression coverage for POSIX and Windows trust-boundary behavior.

## Reproduction Assessment

Not applicable as a feature request, but the gap is high-confidence. Static inspection of current main shows only the four existing hook sources and user-configured load/disable gates, with no policy root or policy inventory path.

## Solution Assessment

Unclear pending maintainer security design. If OpenClaw accepts this capability, a core runtime tier appears to be the right layer because current plugin and hook-pack surfaces cannot enforce OS-owned discovery roots, mandatory ordering, or immunity from user-controlled config.
