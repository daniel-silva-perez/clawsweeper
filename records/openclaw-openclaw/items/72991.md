## Summary

Keep open. Current main and the latest release still expose only the existing bundled, plugin, managed, and workspace hook tiers, with user-configurable enable/disable behavior and no admin-owned machine-wide policy tier. The prior ClawSweeper keep-open comment remains accurate after rechecking the current hook source model, discovery path, startup gate, CLI, docs, and related reports.

## What This Changes

The issue requests an admin-managed machine-wide hook policy tier that loads before user-controlled hooks and cannot be disabled by user config or `openclaw hooks disable`.

## Best Possible Solution

Design and implement a core policy-hook tier with hardened machine-wide discovery, policy-first ordering, user-config disable immunity, policy-aware CLI inventory, docs, and regression coverage across POSIX and Windows trust boundaries.
