## Summary

Current main still does not provide the requested public session lifecycle management. It has lower-level abort plumbing, Chat/TUI stop paths, and a specialized sub-agent kill HTTP path, but `openclaw sessions` still only lists sessions and exposes cleanup; there are no public `sessions stop`, `kill`, `pause`, or `resume` commands, and the Sessions UI does not render row-level Stop/Pause/Resume controls. The related open PR #35735 is deletion cleanup (`rm`/`clear`) and does not supersede this lifecycle-control request.

## Best Possible Solution

Keep this open as the broad session lifecycle tracker. Implement a narrow `openclaw sessions stop <session-key>` first, backed by existing `sessions.abort`/`chat.abort` behavior, with CLI docs and regression tests. Add Sessions-page Stop controls only when they reflect real active-run state and Gateway abort results. Track delete-only cleanup through #35735, and defer pause/resume/resource quotas until the runtime, provider cancellation, and accounting model can enforce them.
