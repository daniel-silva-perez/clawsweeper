## Summary

Current main still has only partial session cancellation plumbing. It exposes Chat/UI abort and background-task cancel paths, but not the requested public `openclaw sessions stop/kill/pause/resume` lifecycle commands, Sessions-page row controls, or enforceable pause/resume/resource-limit model.

## Best Possible Solution

Keep this issue open as the broad session lifecycle tracker. Implement `openclaw sessions stop <session-key>` first by wiring a narrow CLI command to existing `sessions.abort`/`chat.abort` behavior, document it in `docs/cli/sessions.md`, and add regression coverage in the sessions CLI tests. Add Sessions-page Stop controls only when the table can represent active-run state and Gateway abort results accurately. Leave delete-only cleanup to the related `rm`/`clear` PR, and treat pause/resume/resource quotas as later work requiring real runtime/provider/accounting support.
