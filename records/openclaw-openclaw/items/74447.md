## Summary

Keep this PR open. Current main does not already contain the intended cleanup, and the latest PR diff appears to weaken the unused-file guard by moving the stale required entries into the optional allowlist, where they are no longer stale-checked. Because this is a CI guard with `triage: risky-infra`, the right path is maintainer/author revision plus PR-head validation rather than closing or replacing it automatically.

## What This Changes

The PR empties the required Knip unused-file allowlist in `scripts/deadcode-unused-files.allowlist.mjs` and moves the prior required paths into the optional unused-file allowlist, adding one gateway test helper there.

## Best Possible Solution
