## Summary

Keep open: current main has not implemented the requested allowlist cleanup, and the PR branch weakens the unused-file guard by moving stale required paths into the optional allowlist. The patch should be revised before merge.

## What This Changes

The PR changes `scripts/deadcode-unused-files.allowlist.mjs` to empty the required Knip unused-file allowlist and move the previous required paths, plus `src/gateway/test/server-sessions-helpers.ts`, into the optional unused-file allowlist.

## Best Possible Solution

Delete stale entries from both required and optional allowlists unless a specific path has a documented platform-dependent Knip discrepancy, keep the optional list narrow, and merge only after the deadcode and formatting checks pass on the PR head.

## Reproduction Assessment

Yes for the introduced review finding: the PR patch plus current-main checker code show optional entries are allowed for unexpected reports but excluded from stale checks. I did not run Knip because this review is read-only and the bug is visible from the source-level contract.

## Solution Assessment

No. Emptying the required allowlist is directionally right, but moving the full stale required list to optional is not the narrowest maintainable fix because it suppresses future unused-file findings for those paths.
