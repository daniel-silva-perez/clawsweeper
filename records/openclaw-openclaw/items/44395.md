## Summary

Keep this open. Current main still indexes memory markdown through fixed token/overlap chunking with no heading-aware or section-aware strategy in the chunker, config schema, reindex metadata, or docs; the related multi-strategy implementation is an open, unmerged PR, and the entity-extraction/graph-memory stretch scope is not shipped.

## What This Changes

The issue requests heading-aware memory chunking for structured markdown, with broader follow-up ideas for entity extraction, graph memory, reranking, and nightly consolidation.

## Best Possible Solution

Ship a maintained markdown heading or section-aware strategy in the memory-host SDK path used by memory-core, expose it through config types/schema/help/docs, persist the effective strategy for reindex invalidation, and add focused tests proving adjacent `##`/`###` sections do not blend into one chunk. Track entity extraction, graph memory, cross-encoder reranking, and nightly consolidation as explicit follow-ups unless maintainers deliberately keep this as an umbrella.
