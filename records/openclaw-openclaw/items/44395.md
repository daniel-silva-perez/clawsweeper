## Summary

Keep this issue open. Current main still uses fixed token/character-window chunking for memory indexing, exposes only token/overlap chunking config, and has no shipped heading-aware/section-aware chunking strategy. The related entity-extraction/wiki discussion also remains broader than what current memory-wiki implements: wiki pages and digests exist, but automatic entity/concept extraction from source material is not present. The provided discussion also shows an open implementation candidate, #68019, while earlier focused PR attempts were closed unmerged, so this issue should continue tracking the user-visible gap until a PR lands.

## Best Possible Solution

Keep this issue open. The maintainer path is to review #68019 or request a narrower PR that adds a documented heading/section-aware chunking strategy in the memory-host SDK path used by memory-core, exposes it through config types/schema/help/docs, persists the effective strategy/config for reindex invalidation, and adds tests proving adjacent `##`/`###` sections do not blend into one chunk. Track entity extraction, graph memory, cross-encoder reranking, and consolidation as separate follow-ups unless maintainers intentionally keep this issue as the umbrella.
