## Summary

Keep this issue open. Current main still ships fixed token/overlap memory chunking with no heading-aware strategy in the runtime config, memory-core indexing, or reindex metadata, and the related multi-strategy implementation is still an open, unmerged PR.

## What This Changes

The issue requests markdown heading or section-aware memory-search chunking, with broader follow-up ideas for entity extraction, graph memory, reranking, and nightly consolidation.

## Best Possible Solution

Land a maintained, documented heading or section-aware chunking strategy in the package-backed memory-host SDK path used by memory-core, wire it through config types/schema/help/docs, persist the effective chunking strategy for reindex invalidation, and add tests proving adjacent `##`/`###` sections no longer blend into one chunk. Treat entity extraction, graph memory, cross-encoder reranking, and nightly consolidation as explicit follow-ups unless maintainers intentionally keep this as an umbrella.
