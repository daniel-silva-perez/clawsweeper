## Summary

Keep this open. Current main still indexes memory markdown through fixed token/overlap chunking with no heading-aware strategy in the chunker, config schema/help, or reindex metadata; #68019 is open and unmerged for the chunking portion, while the entity-extraction and graph-memory stretch scope is still not shipped.

## What This Changes

The issue requests heading-aware markdown chunking for memory search, with entity extraction, graph memory, reranking, and nightly consolidation as stretch improvements.

## Best Possible Solution

Land a maintained markdown heading or section-aware strategy in the shared memory-host path used by memory-core, expose and document the config, persist strategy-aware reindex metadata, and split entity extraction, reranking, graph memory, and nightly consolidation into explicit follow-up issues or plugin-owned work.
