## Summary

Keep open. Current main has command-scoped session-memory capture and startup hydration for recent memory files, but it still saves extracted user/assistant turns rather than synthesized durable summaries with transcript anchors; the related synthesis PR is open and unmerged, and the poisoning report should shape the design before this is treated as implemented.

## What This Changes

The issue requests automatic session memory preservation and synthesis on `/new`, including compact durable summaries, full transcript retention or links, and continuity across new sessions.

## Best Possible Solution

Land or explicitly reject a maintainer-owned design for compact, sanitized session-memory synthesis with clear transcript receipts. The end state should preserve useful continuity across `/new` without promoting raw chat scaffolding or noisy full transcripts into active context by default.
