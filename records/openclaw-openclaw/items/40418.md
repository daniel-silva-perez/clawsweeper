## Summary

Keep open. Current main has useful reset-memory plumbing, but it still writes recent session turns as raw role-prefixed text and loads matching memory files back into startup context as quoted notes. The requested structured synthesis, transcript receipts or anchors, and daily continuation package are not implemented, and open PR #50584 plus safety report #69943 make this maintainer-review work rather than a fresh automated repair lane.

## What This Changes

Add automatic session memory preservation and synthesis around `/new`, including compact summaries, durable links to full transcripts, and startup hydration from the synthesized continuation package.

## Best Possible Solution

Adopt a bounded, sanitized continuation package for reset/session transitions: keep full transcripts as receipts, write compact durable summaries with stable links or anchors, align the output with startup-context loading, and avoid reinjecting raw transcript artifacts as durable memory.
