## Summary

Keep this issue open. Current main has partial session-memory behavior for `/new` and `/reset`, plus startup hydration from recent memory files, but it still does not implement the requested preservation-and-synthesis workflow: no `.session_memory_preserve.json`, no full transcript export with anchors, no daily continuation package generation, and no LLM synthesis mode in the bundled hook.

## Best Possible Solution

Keep this issue open until maintainers either land a scoped implementation or explicitly redirect the design to a ClawHub/context-engine plugin. A maintainable path should coordinate with #50584 and #69943: opt-in synthesis, sanitization before persistence, compact daily continuation packages, receipt links to full transcripts, clear daily-file behavior, timezone-aware filenames, and regression coverage for `/new`/`/reset` preservation plus startup hydration.
