## Summary

Keep open. Current main has a bounded session-memory hook and startup-context hydration, but it still persists extracted user/assistant turns rather than producing synthesized durable summaries with transcript anchors; open #50584 is the active implementation candidate, and #69943 raises related sanitization concerns that should shape the design.

## What This Changes

The issue requests automatic session memory preservation and synthesis on `/new`, including durable transcript retention, compact summaries, and links between summaries and full conversation records.

## Best Possible Solution

Keep this issue open until maintainers either land a narrow session-memory synthesis/canonical-daily design or intentionally redirect the feature to a plugin/context-engine path; the end state should preserve continuity with compact, quoted/untrusted, sanitized memory and clear links to full transcripts when retained.
