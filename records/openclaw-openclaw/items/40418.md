## Summary

Keep open. Current main has a bundled `session-memory` hook and startup-context loader, but the remaining request is not implemented: the hook still writes extracted recent turns as a raw `Conversation Summary`, there is no structured synthesis mode or anchor generation on main, and the related implementation PR remains open rather than merged.

## What This Changes

The issue requests automatic preservation and synthesis of session memory across `/new` resets, including compact daily summaries and links back to full transcripts.

## Best Possible Solution

Adopt or replace the open synthesis design with a bounded, sanitized continuation package: keep full transcripts as receipts, write compact durable summaries with links/anchors, align output with startup-context loading, and avoid promoting raw transcript artifacts back into the next session.
