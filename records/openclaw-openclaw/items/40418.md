## Summary

Keep this issue open. Current main has partial `/new` and `/reset` session-memory behavior plus first-turn hydration from existing `memory/*.md` files, but it still does not implement the requested preservation and synthesis workflow: no `.session_memory_preserve.json`, no full transcript export with message anchors, no synthesized daily continuation package, and no LLM summary/anchor generation on session start. The related implementation PR #50584 is still open, and the open poisoning bug #69943 should shape any expansion of this feature.

## Best Possible Solution

Keep this open until maintainers either land a scoped implementation or explicitly redirect the design to a plugin/context-engine path. A strong solution should coordinate with #50584 and #69943, make synthesis opt-in, sanitize before persistence, generate compact daily continuation packages, link to full transcript receipts with anchors when enabled, and add regression coverage for `/new` and `/reset` preservation plus startup hydration.

## Risks / Open Questions

- Closing now would lose the broader tracker for full-fidelity transcript receipts, anchors, daily continuation synthesis, and startup behavior while #50584 remains open and partial.
- Expanding automatic persistence without resolving #69943 could preserve and re-inject unsafe raw transcript artifacts such as chat-template tokens, role markers, or malformed tool-call XML.
- A ClawHub/context-engine plugin may ultimately be the right product shape, but current main already ships bundled session-memory and startup-context behavior for this surface, so redirecting it needs maintainer judgment rather than conservative cleanup closure.
