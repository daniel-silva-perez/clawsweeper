## Summary

Keep open. Current main has partial reset memory preservation, but it still persists bounded raw role-prefixed transcript excerpts and reloads memory files as quoted notes; the requested structured synthesis, transcript receipts or anchors, and daily continuation package remain unresolved. Open PR #50584 already targets this area, and open #69943 adds safety constraints that should shape the design before another automated repair is queued.

## What This Changes

Request automatic memory preservation on `/new` that exports session transcript receipts and synthesizes compact durable summaries for continuity across sessions.

## Best Possible Solution

Adopt a bounded, sanitized continuation package for reset/session transitions: preserve full transcripts as receipts, write compact durable summaries with stable links or anchors, align output with startup-context loading, and avoid reinjecting raw transcript artifacts as durable memory.

## Reproduction Assessment

Yes. Source and tests give a high-confidence path: enable `session-memory`, trigger `/new` or `/reset` with a prior transcript, and current main writes raw role-prefixed user/assistant lines that startup context can later quote back into reset/startup turns.

## Solution Assessment

Unclear. The requested direction is reasonable, but the maintainable fix should be reviewed together with #50584 and the #69943 sanitization constraints before choosing exact hook behavior or a broader plugin/context-engine path.
