## Summary

Keep open. Current main preserves some reset context, but it still writes bounded raw `user:`/`assistant:` transcript excerpts and startup loads recent memory files as quoted notes; the requested structured synthesis, transcript receipts or anchors, and daily continuation package are not implemented. Open PR #50584 targets this feature area, and open issue #69943 raises safety constraints that should shape the design before another automated fix is queued.

## What This Changes

Add automated session memory preservation and synthesis around `/new`, including compact summaries, durable links to full transcripts, and startup hydration from the synthesized continuation package.

## Best Possible Solution

Adopt a bounded, sanitized continuation package for reset/session transitions: preserve full transcripts as receipts, write compact durable summaries with stable links or anchors, align output with startup-context loading, and avoid reinjecting raw transcript artifacts as durable memory.

## Reproduction Assessment

Yes for the current behavior. Source and tests show a high-confidence path: enable `session-memory`, trigger `/new` or `/reset` with a prior transcript, and the hook writes role-prefixed raw user/assistant lines that startup context can later quote back into the first reset/startup turn.

## Solution Assessment

Unclear as an implementation choice. The requested direction is reasonable, but the best maintainable fix needs maintainer review of #50584 plus the #69943 sanitization/security constraints before choosing core hook behavior versus a plugin/context-engine path.
