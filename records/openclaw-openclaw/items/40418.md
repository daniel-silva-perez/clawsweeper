## Summary

Keep open. Current main partially covers reset preservation with the bundled `session-memory` hook and startup-context preload, but it still persists bounded raw user/assistant turns in slugged Markdown files and does not implement structured synthesis, anchored transcript receipts, or a compact continuation package. Open PR #50584 is a live implementation candidate, and open #69943 makes sanitization and retention policy important before expanding this feature.

## What This Changes

The issue requests automatic session memory preservation across `/new`, including full transcript export, structured synthesis, and links from compact summaries to underlying session transcripts.

## Best Possible Solution

Land or replace #50584 with an opt-in, bounded design that synthesizes durable decisions, TODOs, preferences, and receipts into compact startup memory; records links to full transcripts without injecting them by default; sanitizes unsafe role/tool/control-token artifacts; and updates tests/docs for dedupe-aware startup loading.

## Reproduction Assessment

Yes. A source-level reproduction path is clear: enable `session-memory`, trigger `/new` or `/reset`, and current main writes recent raw user/assistant transcript text to a slugged memory file that startup context can later preload. No live run was attempted because this was a read-only review.

## Solution Assessment

Unclear as originally proposed. The layered summary-with-receipts direction from the discussion is safer than unconditional transcript promotion, and the current open PR or a maintainer-approved replacement should settle synthesis, privacy, retention, and sanitization details.
