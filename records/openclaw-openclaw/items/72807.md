## Summary

Keep open. Current main does not implement the PR’s intended `gitInit` opt-out: agent workspace creation still initializes git for brand-new workspaces, `agents.create` does not accept the field, and the handler does not forward it. The PR remains a plausible focused implementation candidate after rebase/regeneration.

## Best Possible Solution

Keep this PR open as the active implementation candidate. Best path is to rebase on current main, remove generated drift, keep the additive `gitInit?: boolean` contract across schema/server/workspace/generated clients, and land after normal maintainer review and targeted validation.
