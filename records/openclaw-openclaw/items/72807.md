## What This Changes

This PR adds an optional `gitInit` boolean to `agents.create` and `ensureAgentWorkspace`, forwards it into workspace setup, regenerates Swift protocol models, and tests that `gitInit: false` skips creating `.git`.

## Best Possible Solution

Land a rebased version of this additive opt-out that keeps default git initialization unchanged, updates the Gateway schema/server/workspace helper and generated protocol clients together, and includes focused regression coverage for both the helper and RPC forwarding path.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: on current main, `agents.create` cannot accept `gitInit` because the schema rejects extra properties, and the workspace helper still calls the git initialization path for brand-new bootstrapped workspaces.

## Solution Assessment

Yes, mostly. An optional boolean that defaults to existing behavior is the narrowest maintainable fix; before merge, the safer final path is to rebase and run the repo protocol/changed gates rather than replacing the PR with a broader workspace policy.
