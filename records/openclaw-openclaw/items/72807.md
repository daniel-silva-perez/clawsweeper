## Summary

Keep open. Current main still lacks a `gitInit` opt-out for `agents.create`, and the proposed direction is narrow and backwards-compatible; the concrete merge blocker I found is the missing required changelog entry for a user-facing protocol feature.

## What This Changes

The PR adds an optional `gitInit` boolean to `agents.create` and `ensureAgentWorkspace`, forwards it through the gateway handler, updates generated Swift protocol models, and adds a workspace regression test for skipping `.git` creation.

## Best Possible Solution

Land the opt-out as an additive protocol/workspace option with default behavior unchanged, after adding the required Unreleased changelog entry, preserving generated Swift model alignment, and running the repo validation path for the touched protocol and agents surfaces.

## Reproduction Assessment

Not applicable as a bug reproduction. Current-main inspection gives high-confidence behavioral evidence: `gitInit` is absent from the schema/handler and brand-new bootstrapped workspaces still run `git init` when git is available.

## Solution Assessment

Yes. An optional boolean defaulting to current behavior is the narrowest maintainable API shape for embedders that need to suppress automatic repo creation; the safer merge path is to add release-note coverage and repo-wrapper validation without broadening the feature.
