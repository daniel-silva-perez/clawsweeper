## Summary

Keep open. Current main has a partial backend mitigation for unique bare model ids, but the Control UI `/model` status output still renders `Available:` from `ModelCatalogEntry.id` only, which drops the provider and can still make slash-containing provider-native ids ambiguous or non-round-trippable. The related canonicalization PR was closed unmerged, so there is no merged implementation that fully satisfies the issue’s requested UX fix.

## Best Possible Solution

Implement the user-facing round-trip fix in the Control UI slash command and picker: display canonical `provider/model` values, including provider-native nested ids such as `vercel-ai-gateway/anthropic/claude-opus-4.6`, and add regression tests that copy an `Available:` entry back through `sessions.patch`. Keep the existing unique-bare-id backend inference as compatibility, but do not rely on it as the primary UX contract.
