## Summary

Keep open. Current main still prints `/model` `Available:` entries from bare `ModelCatalogEntry.id` values, while model parsing treats slash-containing input as `provider/model` and only infers a provider for inputs without `/`. The chat picker has a partial provider-qualified path, but the no-argument `/model` status output remains not reliably copy-paste round-trippable. Related PR #47249 targeted this but is closed unmerged.

## Best Possible Solution

Implement the remaining Control UI slash-command fix: format `Current model` and every `Available:` entry as the canonical value accepted by `sessions.patch`, using the catalog provider plus model id for nested IDs. Keep the current unique-bare-ID backend inference for compatibility, and add regression coverage that copies an `Available:` value back through `/model` or the session patch path.
