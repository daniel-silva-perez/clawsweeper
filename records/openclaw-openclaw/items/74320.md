## Summary

Keep this PR open. Current main still has the heartbeat task parser bug described here, the related merged PR #64488 fixed only the adjacent top-level interval/prompt bleed case, and this PR is a narrow parser/test change with no obvious security-sensitive or supply-chain surface.

## What This Changes

This PR changes `parseHeartbeatTasks` to test block-boundary indentation against the original source lines and adds two regression tests for unknown indented task fields and top-level task-block termination.

## Best Possible Solution

Land a narrow parser fix that uses the original source line for indentation-sensitive block-boundary checks, keeps top-level non-task content terminating the `tasks:` block, and preserves the added regression coverage for unknown indented fields before later tasks.
