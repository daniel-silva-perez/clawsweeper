## Summary

Keep this issue open. Current main still masks the OpenAI Codex device-pairing code in remote/SSH mode even though the provider helper receives the real user code, and open PR #74258 already uses closing syntax for the narrow fix.

## What This Changes

The issue asks OpenClaw to show the OpenAI Codex device-pairing code in the SSH/remote `models auth login --provider openai-codex` terminal flow instead of replacing it with a local-device placeholder.

## Best Possible Solution

Land #74258 or an equivalent narrow OpenAI plugin fix that puts `Code: <userCode>` in the remote prompter note, keeps `ctx.runtime.log` URL-only, and updates `extensions/openai/openai-codex-provider.test.ts` to guard both behaviors.
