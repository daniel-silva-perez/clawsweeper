Keep open. Current main receives the real `userCode` from `loginOpenAICodexDeviceCode`, but `extensions/openai/openai-codex-provider.ts` replaces it with `Code: [shown on the local device only]` when `ctx.isRemote` is true, making SSH/headless device pairing unusable. Update the provider-owned prompt so interactive remote/headless users can see the short-lived code in `ctx.prompter.note`. Preserve the existing token exchange helper unless a small typed adjustment is needed. Update `extensions/openai/openai-codex-provider.test.ts` so remote mode expects the note to include `Code: CODE-12345` while still asserting `runtime.log` does not contain `CODE-12345`. Keep the change scoped to the OpenAI Codex plugin; do not add/rename CLI flags or expand the docs `--device-code` mismatch unless the fix genuinely requires it.
