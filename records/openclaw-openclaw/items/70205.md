## Summary

Keep this PR open. Current main does not implement the intended fix: `usage.cost` still reads a single default-agent transcript scope and caches only by date range, while `sessions.usage` already scans configured agents. The PR remains a focused implementation candidate for the #41986 aggregation mismatch and supersedes the older closed/unmerged #42063 rather than being superseded by main.

## Best Possible Solution

Keep this PR open for maintainer review and likely landing, or land an equivalent focused fix on main. The best implementation path is the one the PR is already taking: resolve the `usage.cost` agent scope from `listAgentsForGateway`, validate explicit `agentId`, include the resolved agent set in the cache key, merge per-agent daily/totals with partial-failure tolerance, preserve date-only UI labels, and keep the broader per-model usage-log proposal in #13219 separate.
