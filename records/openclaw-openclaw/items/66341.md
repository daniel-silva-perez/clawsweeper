## Summary

Keep this PR open. Current main still requires admin scope for cron mutation methods and does not persist or enforce a creator-scope ceiling, so the PR’s central authorization change is not implemented or obsolete. The branch is still plausible, but it needs focused maintainer security review because the supplied diff also broadens cron update/remove/run and bundles unrelated browser and QA-lab changes.

## Best Possible Solution

Keep this PR open for maintainer review. Ask for a focused cron-only branch that lowers only the intended cron method scope, persists creator scopes at job creation, enforces the ceiling in embedded and CLI/MCP execution paths, and adds regression coverage proving write-scoped callers cannot update, remove, or run admin-created jobs to gain admin-level tools. Unrelated browser WSL2 and QA-lab formatting changes should be split out.
