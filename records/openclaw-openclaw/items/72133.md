## Summary

Keep open. Current main still satisfies the rich per-message token/cache/cost/model/context display only in the Control UI; channel replies have an opt-in usage footer but not model or context percentage, iOS preserves usage without rendering it, and Android chat history drops usage metadata entirely.

## What This Changes

Request to surface compact per-message token, cache, cost, model, and context metadata in native mobile chat views and messaging-channel reply surfaces.

## Best Possible Solution

Add one shared compact usage presentation contract for assistant replies, reuse it from the existing responseUsage channel footer and native Swift/Kotlin chat renderers, and gate model/context display on reliable metadata availability.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a feature request, but the implementation gap is high-confidence by source inspection across Control UI, channel footer, iOS chat rendering, Android chat parsing, and gateway history preservation.

## Solution Assessment

Is this the best way to solve the issue? Yes: extending the existing /usage footer and native chat message renderers is narrower than adding per-channel custom behavior, with context percentage omitted unless the run/history metadata can support it.
