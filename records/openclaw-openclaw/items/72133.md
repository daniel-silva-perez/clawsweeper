## Summary

Keep open. Current main still only satisfies the full per-message token/cache/cost/context/model display in the Control UI; channel replies have an opt-in usage footer, but it lacks context percentage and model, while native iOS and Android chat views still do not surface the metadata to users.

## What This Changes

Request to show compact per-message token, cost, cache, context percentage, and model metadata in native iOS/Android chat views and optional messaging-channel reply footers.

## Best Possible Solution

Add a shared compact usage presentation for assistant replies, reuse it from the generic channel responseUsage footer and native chat renderers, and include model/context percentage only when those values are available from reliable run or history metadata.

## Reproduction Assessment

Not applicable as a feature request; source inspection gives a high-confidence verification path. Current main shows full Control UI metadata, partial channel footer formatting, iOS usage preservation without rendering, and Android history parsing that drops usage entirely.

## Solution Assessment

Yes. The narrow maintainable path is to extend the existing shared /usage footer and native chat models/views rather than adding transport-specific channel code; if context percentage is unavailable, the safer result is to omit it.
