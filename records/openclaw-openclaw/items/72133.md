## Summary

Keep open. Current main still only provides the full per-message token/cache/cost/model/context display in the Control UI; channel replies have an opt-in usage footer, but it does not include model or context percentage, and native iOS/Android chat views still do not render the metadata for users.

## What This Changes

Request to surface Control UI-style token/cache/cost/model/context metadata under assistant replies in iOS/Android and optional messaging-channel reply footers.

## Best Possible Solution

Add a shared compact usage presentation for assistant replies, reuse it from the generic channel responseUsage footer and native chat renderers, and include model/context percentage only when those values are available from reliable run or history metadata.

## Reproduction Assessment

Not applicable as a feature request; source inspection gives a high-confidence verification path. Current main shows full Control UI metadata, partial channel footer formatting, iOS usage preservation without rendering, and Android history parsing that drops usage entirely.

## Solution Assessment

Yes, for a future fix: extending the existing /usage footer and native chat message renderers is the narrowest maintainable direction. A safer implementation should omit context percentage when the context limit or prompt-token source is unavailable.
