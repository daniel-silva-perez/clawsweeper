## Summary

Keep open. Current main still has full per-message token/cache/cost/model/context display only in the Control UI; channel replies expose an opt-in usage footer but omit model and context percentage, iOS preserves usage data without rendering it, and Android chat history does not retain usage metadata.

## What This Changes

Requested surfacing per-message token/cost/cache/model/context metadata in native mobile chat views and messaging channel replies.

## Best Possible Solution

Add a shared compact usage presentation for assistant replies, reuse it from the existing responseUsage channel footer and the native Swift/Kotlin chat renderers, preserve Android usage metadata during history parsing, and show model/context percentage only when reliable metadata is available.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a feature request, but the implementation gap is high-confidence by source inspection across Control UI, channel footer, iOS/Swift chat, and Android chat paths.

## Solution Assessment

Is this the best way to solve the issue? Yes: extending the existing /usage footer and native chat message renderers is the narrowest maintainable path, with context percentage gated on reliable run or history metadata.
