## Summary

Keep this issue open. Current main has the Control UI metadata the reporter references, and channel replies already have an opt-in `/usage` footer, but the native mobile chat surfaces do not yet show this metadata: iOS decodes usage without rendering it, and Android drops usage while parsing chat history. The channel footer also remains a partial match because it omits the Control UI-style context percentage and model label.

## Best Possible Solution

Implement a compact assistant-message usage footer in the native chat clients using existing transcript usage data: iOS should render the decoded `OpenClawChatUsage`, and Android should decode usage/cost/model fields from chat history and render the same compact footer. If matching the Control UI more closely, pass/use session context-window and model metadata to show context percentage and model labels where reliable. For channels, keep the existing `/usage` toggle and extend the shared `formatResponseUsageLine` path for context/model instead of touching individual channel plugins. Update token-use/mobile docs and focused tests.
