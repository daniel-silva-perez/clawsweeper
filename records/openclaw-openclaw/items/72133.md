## Summary

Keep open. Current main still implements the full per-message token/cache/cost/model/context display only in the Control UI; messaging-channel replies have an opt-in usage footer but it omits model and context percentage, iOS preserves usage data without rendering it, and Android drops usage metadata from chat history entirely.

## What This Changes

Request per-message token, cache, cost, model, and context-window metadata in native iOS/Android chat views and optional messaging-channel reply footers.

## Best Possible Solution

Add a shared compact usage presentation for assistant replies, reuse it from the channel responseUsage footer and native chat renderers, parse/preserve Android usage metadata, and document that model/context percentage appears only when reliable metadata is available.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a feature request, but yes for verification by source inspection: current main has full Control UI metadata, partial channel footers, iOS usage preservation without rendering, and Android history parsing that omits usage.

## Solution Assessment

Is this the best way to solve the issue? Yes for a future fix: extending the existing /usage footer and native chat message renderers is the narrowest maintainable path, with context percentage gated on trustworthy run or history metadata.
