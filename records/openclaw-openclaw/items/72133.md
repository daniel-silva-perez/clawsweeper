## Summary

Keep open. Current main still only fully implements the requested metadata in the Control UI; channel replies have an opt-in token/cache/cost footer but not the Control UI-style context percentage or model label, and the native mobile chat surfaces still do not render the metadata.

## What This Changes

Request to expose per-assistant-message token, cost, cache, context-window, and model metadata in iOS/Android native chat and optional messaging-channel reply footers.

## Best Possible Solution

Add a compact shared usage presentation that reuses existing transcript usage data across native chat and the generic channel /usage footer path, with context percent and model labels included only when the source data is reliable and clearly scoped to the displayed assistant reply.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a feature request, but source inspection gives a high-confidence verification path: iOS preserves usage without rendering it, Android does not preserve usage in ChatMessage, and the channel footer formatter omits context/model.

## Solution Assessment

Is this the best way to solve the issue? Yes: extending the existing generic /usage footer and native chat rendering paths is narrower and more maintainable than adding per-channel transport-specific footers or inventing a separate mobile telemetry channel.
