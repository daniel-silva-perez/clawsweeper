## Summary

Current main still lacks the requested global, per-cron, or max-age policy for disabling missed cron catch-up after gateway restart. The cron startup path still invokes missed-job catch-up, the planner still replays missed cron slots, config/schema/docs expose only full scheduler disablement plus existing scheduling/delivery controls, and tests still assert replay-on-restart behavior. A hardcoded stale-delivery guard exists for isolated fallback delivery after a 3-hour start delay, but it does not prevent the job from running and is not a configurable missed-run policy.

## Best Possible Solution

Keep this issue open. Implement a backward-compatible missed-run policy in the cron-owned surface: a global default plus an optional per-job override and/or max-age window for stale runs, wired through config validation/generated help, cron job protocol and CLI edit surfaces, docs, and runMissedJobs planning. Add tests for default replay, global skip, per-job override, and stale-window behavior.
