## Summary

Current main still replays missed cron jobs during gateway startup catch-up and still does not expose a global, per-job, or max-age/window option to skip stale missed cron runs. The issue is recent, concrete, has no protected labels or maintainer author association, and remains the canonical open tracker after same-author duplicate #73644 was closed.

## Best Possible Solution

Keep this issue open. Implement a backward-compatible missed-run policy in the cron-owned surface: preserve replay as the default, add a global cron default plus optional per-job override and/or max-age window, validate and document the config/protocol fields, expose CLI add/edit support where jobs are created, and add restart catch-up tests for default replay, global skip, per-job override, and stale-window behavior.

## Risks / Open Questions

- Changing the default behavior from replay to skip could cause users to miss recovery/reminder jobs after short outages. A conservative fix should preserve replay by default and add explicit opt-out or max-age policy controls.
- A complete fix needs aligned changes across cron config/schema/help/docs, gateway cron job protocol types, CLI add/edit surfaces, startup catch-up planning, and restart catch-up tests.
