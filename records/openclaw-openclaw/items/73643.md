## Summary

Current main still replays missed cron jobs on gateway restart and has no global, per-job, or max-age/window policy to skip stale missed runs. The report is recent, concrete, has no protected maintainer metadata, and the same-author duplicate #73644 is already closed, so this issue remains the actionable tracker.

## Best Possible Solution

Keep this issue open. Implement a backward-compatible missed-run policy in the cron-owned surface: preserve default replay, add a global cron default plus optional per-job override and/or max-age window, wire it through config validation/help, cron protocol schemas, CLI add/edit and Control UI forms, docs, and restart catch-up planning, then add tests for default replay, global skip, per-job override, and stale-window behavior.
