## Summary

Current main still runs missed cron jobs during Gateway startup and does not expose a global, per-job, or max-age/window policy to skip stale catch-up runs. The older #42039 overlaps on exposing internal missed-job throttling knobs, but it does not fully cover this issue's requested skip/window policy because the current deferred path still reschedules missed jobs after restart.

## Best Possible Solution

Keep this issue open for missed-run policy semantics. The best path is a backward-compatible cron-owned policy: preserve replay by default, add an explicit global opt-out and/or max-age window, optionally add a per-job override for time-sensitive jobs, define how any maxMissedJobsPerRestart=0 behavior relates to skipping versus deferring, update config/schema/help/docs and CLI/protocol surfaces, and coordinate with #42039 instead of closing this automatically.
