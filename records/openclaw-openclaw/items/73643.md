## Summary

Current main still replays missed cron jobs on gateway restart and does not expose a global, per-job, or max-age/window policy to skip stale missed runs. The issue is recent, concrete, and remains the canonical open tracker because the same-author duplicate #73644 is already closed.

## Best Possible Solution

Keep this issue open. Implement a backward-compatible missed-run policy in the cron-owned surface: preserve default replay, add a global cron default plus optional per-job override and/or max-age window, wire it through config validation/generated help, cron job protocol schemas, CLI add/edit options, docs, and restart catch-up planning, then add tests for default replay, global skip, per-job override, and stale-window behavior.

## Risks / Open Questions

- Changing the default from replay to skip could cause users to miss recovery/reminder jobs after short outages, so the current replay behavior should remain the default unless an explicit policy opts out.
- A fix needs to stay aligned across CronConfig, strict config schema/help/generated docs, cron job protocol schemas, CLI add/edit surfaces, docs, and restart catch-up tests.
