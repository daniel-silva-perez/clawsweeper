## Summary

Current main still replays missed cron jobs during gateway startup and does not expose a global, per-job, or time-window policy to skip stale missed runs. The issue is recent, concrete, unprotected, and remains actionable in the cron scheduler/config surface. The same-author duplicate #73644 is closed, so this issue is the remaining open tracker rather than something to close as duplicate.

## Best Possible Solution

Keep this issue open. Implement an explicit missed-run policy in the cron-owned surface: a backward-compatible global default under cron plus an optional per-job override and/or max-age window, wired through CronConfig, strict zod/generated schema help, cron job protocol/tool/CLI edit surfaces, docs, and runMissedJobs planning. Add tests for default replay, global skip, per-job override, and stale-window behavior.
