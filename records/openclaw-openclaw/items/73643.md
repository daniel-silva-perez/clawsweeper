## Summary

Current main still replays missed cron jobs during gateway startup and does not expose a global, per-job, or time-window policy to skip stale missed runs. This issue is recent, concrete, unprotected, and still actionable in OpenClaw cron scheduler/config surfaces rather than eligible for cleanup closure.

## Best Possible Solution

Keep this issue open. Implement an explicit missed-run policy in the cron-owned surface, preferably a backward-compatible global default under `cron` plus an optional per-job override and/or max-age window. Wire it through `CronConfig`, strict zod/schema help, cron job/protocol/CLI edit surfaces, docs, and `runMissedJobs` planning, with tests for default replay, global skip, per-job override, and stale-window behavior.
