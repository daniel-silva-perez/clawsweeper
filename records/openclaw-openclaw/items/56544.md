## Summary

Keep open. Current main still has the Slack DM preview-streaming gate this PR targets, and the draft-stream/send path supports posting a preview without a pre-existing Slack thread timestamp. The branch is technically narrow and no blocking code-review or security finding showed up, but it changes documented Slack DM preview behavior and should get maintainer review plus a rebase/validation before merge.

## What This Changes

The PR enables Slack draft preview streaming for non-off streaming modes in direct messages even when no thread timestamp exists, and updates the Slack streaming eligibility tests accordingly.

## Best Possible Solution

Land a Slack-owned fix that allows draft preview streaming in flat DMs when preview streaming is configured, while keeping native Slack streaming thread-gated and aligning Slack docs/changelog with the new DM behavior.

## Reproduction Assessment

Yes. Current main reproduces at helper level: shouldEnableSlackPreviewStreaming({ mode: "partial", isDirectMessage: true }) returns false, and the current test suite asserts that disabled path.

## Solution Assessment

Mostly yes. Removing the direct-message threadTs gate is the narrow technical fix because the draft preview path posts to the DM target with an optional threadTs; the remaining concern is product/docs alignment rather than code shape.
