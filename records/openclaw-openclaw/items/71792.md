## Summary

Keep open. Current main still does not implement this PR's intended automatic daily/idle rollover memory persistence: the rollover path archives/emits typed session lifecycle hooks, but it does not emit the internal `command:reset` event consumed by bundled `session-memory`. The PR is open, unmerged, maintainer-repaired in discussion, and remains a focused implementation candidate rather than something to close.

## Best Possible Solution

Keep this PR open for maintainer review or landing as a focused fix. The strongest path is to emit the existing internal `command:reset` event for non-command stale daily/idle rollovers with explicit `commandSource` values such as `auto:daily` and `auto:idle`, include `previousSessionEntry` and resolved workspace context for `session-memory`, preserve daily/idle/fresh no-op regression coverage, and add a docs note only if maintainers want the broader automatic command-event semantics documented.
