## Summary

Current main still does not implement this PR's intended automatic daily/idle rollover memory persistence. The stale rollover path archives the old transcript and emits plugin session lifecycle hooks, but it does not emit the internal command:new/reset hook that bundled session-memory listens to. The PR is active, maintainer-repaired in discussion, and linked to open issue #11960, so it should stay open for maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review and likely landing as the narrow contributor path, unless maintainers explicitly choose another overlapping PR as canonical. The concrete fix belongs in src/auto-reply/reply/session.ts: when stale daily/idle rollover creates a new session with a previousSessionEntry and was not caused by an explicit command, emit the existing internal command:reset hook with clear commandSource values such as auto:daily and auto:idle, pass cfg/workspaceDir/sessionEntry/previousSessionEntry consistently with manual reset, and keep focused regression coverage for daily, idle, and fresh-session no-op behavior.
