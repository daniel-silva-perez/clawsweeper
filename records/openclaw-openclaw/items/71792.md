## Summary

Keep open. Current main still does not implement this PR's intended automatic daily/idle rollover memory persistence. The stale rollover path preserves and archives the previous session and emits typed plugin lifecycle hooks, but it does not trigger the internal `command:reset` hook that bundled `session-memory` consumes. The PR is also active and maintainer-repaired in discussion, so it remains a focused implementation candidate rather than cleanup to close.

## Best Possible Solution

Keep this PR open for maintainer review and landing as the focused contributor path. The strongest product/code direction is to emit the existing internal `command:reset` event for non-command stale daily/idle rollovers with explicit `commandSource` values such as `auto:daily` and `auto:idle`, pass `previousSessionEntry` plus the resolved workspace context for `session-memory`, preserve daily/idle/fresh no-op regression coverage, and document the automatic command-event semantics only if maintainers want that behavior public.
