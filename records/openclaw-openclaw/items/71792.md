## Summary

Current main still does not implement this PR's intended automatic daily/idle rollover memory persistence. The rollover path records and archives the previous session and emits typed plugin lifecycle hooks, but it never emits the internal `command:reset` hook that bundled `session-memory` consumes. The PR remains an active, focused implementation candidate, with maintainer branch repair comments and reported validation in the discussion.

## Best Possible Solution

Keep this PR open and review or land it as the focused canonical fix for automatic daily/idle rollover memory persistence. The likely right path is to emit the existing internal `command:reset` event for non-command stale rollovers with explicit `commandSource` values such as `auto:daily` and `auto:idle`, pass the previous session entry and resolved workspace context to `session-memory`, keep daily/idle/fresh no-op regression coverage, and add a small docs note only if maintainers want the broader command-event semantics documented.
