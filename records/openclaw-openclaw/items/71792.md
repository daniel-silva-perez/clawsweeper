## Summary

Current main still lacks the PR's intended automatic daily/idle rollover memory persistence. Stale rollover archives the previous transcript and emits typed session lifecycle hooks, but it does not emit the internal command reset hook that the bundled session-memory handler consumes. This PR is active, targeted, and has maintainer repair context, so it should stay open for review/landing rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review and likely landing unless maintainers explicitly choose another overlapping PR as canonical. The best implementation direction is the narrow one here: emit the existing internal command:reset hook during non-command stale daily/idle rollover only when a previousSessionEntry exists, pass cfg/workspaceDir/sessionEntry/previousSessionEntry consistently with manual reset, keep typed session_end/session_start behavior intact, retain focused daily, idle, and fresh-session no-op coverage, and resolve the failing PR checks before merge.
