Current main still retains the reported iOS LAN plaintext and mixed-auth precedence behavior, setup-code generation remains bootstrap-token-only, and the provided GitHub context shows open PR #65185 with closing syntax targeting this issue. Keep this issue open while maintainers review PR #65185 or a successor.
