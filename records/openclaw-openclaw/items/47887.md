## Summary

Keep open. Current main still retains multiple reported behaviors, and the provided GitHub context shows an open, unmerged implementation PR with closing syntax targeting this issue. There is partial bootstrap-race mitigation on main, but LAN plaintext handling, iOS setup-code parsing, and Swift mixed-auth precedence remain unresolved or inconsistent with the docs/generator behavior.

## Best Possible Solution

Keep this issue open while maintainers review PR #65185 or a successor. The best fix is a coherent iOS/shared Swift change for LAN plaintext handling and password-over-bootstrap precedence, plus a product/security decision on bootstrap-only setup codes versus shared-token/password payloads, with matching TypeScript tests, Swift tests, and docs.
