## Summary

Keep open. Current main does not implement these guardrails, and the PR remains a focused implementation candidate; the only definite issue found is low-priority duplicate wording in the new supported-action guidance.

## What This Changes

The PR adds actionable missing-path guidance to the read tool, pre-dispatch unsupported-action validation to the message tool, and focused regression tests for both paths.

## Best Possible Solution

Land a focused version that keeps the read-tool missing-path guidance and message-action preflight, preserves existing pass-through cases, deduplicates the supported-action display list, and retains the regression tests.

## Reproduction Assessment

Yes. ## Solution Assessment

Yes, with one small polish fix. The proposed changes sit in the existing read/message tool wrappers rather than changing channel plugins or outbound dispatch broadly; the supported-action display list should use a Set before sorting.
