## Summary

Keep this issue open. Current main still exposes Telegram group history as flat sender/body/timestamp context without per-message bot target attribution, and #72510 remains the open implementation candidate that explicitly closes this issue.

## Best Possible Solution

Keep this issue open and review #72510. If that PR is not the right patch, the core path should add stable bot-target metadata to shared history and reply context, populate it from Telegram mention entities and reply-chain bot identity, render it in inbound prompt metadata, and cover current-bot, other-bot, and ambient group-history cases with focused tests.
