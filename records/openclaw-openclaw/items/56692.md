## Summary

Keep this issue open. Current main still has only current-turn mention/reply signals plus unannotated group history, and the open PR #72510 is an implementation candidate that explicitly says it closes this issue. The community triage gate and multi-agent kit mentioned in discussion are useful adjacent workarounds, but they do not close the bundled Telegram/core prompt-context gap.

## Best Possible Solution

Keep the issue open and review #72510. If that PR is not the right patch, the core fix should add stable target metadata to the shared history/reply context, populate it from Telegram mention entities and reply-chain bot identity, render it in inbound prompt metadata, and cover current-bot, other-bot, and ambient group-history cases with focused tests.
