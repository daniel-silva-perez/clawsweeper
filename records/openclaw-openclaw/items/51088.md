## Summary

Keep open. Current main supports iMessage acknowledgements through the recommended BlueBubbles channel and exposes generic inbound message hooks, but the legacy `channels.imessage`/`imsg` path described here still has no channel-owned ack/read-receipt config, capability, schema, or inbound tapback/read behavior. Since that path is documented as legacy but still supported, this needs maintainer product direction rather than cleanup closure or an automatic repair.

## What This Changes

Request native iMessage message acknowledgement for inbound messages, either by extending `ackReactionScope` to legacy iMessage or by adding an inbound hook that can drive read receipts/tapbacks.

## Best Possible Solution

Make the supported iMessage direction explicit. If legacy `channels.imessage` remains supported, add channel-owned acknowledgement with config/schema/docs and direct/group coverage; if BlueBubbles is the intended answer, document that read receipts and tapback acknowledgements require `channels.bluebubbles` and explicitly scope legacy `imsg` out.
