## Summary

Keep open. Current main supports iMessage acknowledgements through the recommended BlueBubbles channel and exposes generic inbound message hooks, but the legacy `channels.imessage` / `imsg` path described here still has no native acknowledgement setting, capability, schema support, or inbound tapback/read-receipt behavior. Because that legacy path is still documented while deprecated, the remaining question is maintainer product direction rather than cleanup closure.

## What This Changes

Request native acknowledgement for legacy iMessage inbound messages, via `ackReactionScope` tapbacks/read receipts or an inbound hook, so senders can tell the agent received messages while processing is queued.

## Best Possible Solution

Make the supported iMessage direction explicit. If legacy `channels.imessage` remains supported, add channel-owned acknowledgement with config/schema/docs and direct/group tests; if BlueBubbles is the intended answer, document that read receipts and tapback acknowledgements require `channels.bluebubbles` and explicitly scope legacy `imsg` out.
