## Summary

Keep open. Current main still has the reported gap for the legacy `imessage` channel: BlueBubbles supports read receipts and configured ack tapbacks, but `channels.imessage` does not expose ack/read config, advertise reaction capability, or perform any ack/read action before dispatching inbound messages.

## Best Possible Solution

Keep this open for maintainer follow-up. If legacy `channels.imessage` remains supported, implement acknowledgment in `extensions/imessage`: add config/schema/type support, trigger read receipt or tapback handling before queue/dispatch according to `messages.ackReactionScope`, and cover direct/group scopes with tests. If legacy `imsg` should not grow new behavior, document that acknowledgment support is available through BlueBubbles and close only after explicit maintainer triage.
