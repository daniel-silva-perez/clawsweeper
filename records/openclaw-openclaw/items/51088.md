## Summary

Keep open. Current main still does not implement native acknowledgment/read-receipt support for the legacy `channels.imessage` / `imsg` path. BlueBubbles already has ack reactions and read receipts, and related iMessage typing/tapback PRs exist, but none closes the specific legacy iMessage `ackReactionScope` / read-receipt gap described here.

## Best Possible Solution

Keep this open for maintainer decision. If legacy `channels.imessage` remains supported, implement channel-owned acknowledgment after access/mention gating and before dispatch: typed config/schema/docs for read/tapback/ack behavior, imsg capability/version handling, echo/reflection safeguards, and tests for direct/group scope. If maintainers choose BlueBubbles as the only supported iMessage acknowledgment path, document that explicitly and close after that product decision.
