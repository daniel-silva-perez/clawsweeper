## Summary

Keep open. Current main still does not add native acknowledgment behavior to the legacy `imessage`/`imsg` channel: its capabilities do not advertise reactions/read receipts, its strict config type/schema has no ack/read setting, and the inbound monitor records and dispatches messages without sending a tapback or read receipt. BlueBubbles covers read receipts and tapback reactions, but that does not close the legacy iMessage gap requested here.

## Best Possible Solution

Keep this issue open for maintainer product decision. If legacy `channels.imessage` remains supported, add channel-owned acknowledgment after access/mention gating and before dispatch, with typed config/schema/docs, echo/reflection safeguards, and direct/group tests. If BlueBubbles is the intended supported path for iMessage read receipts and tapbacks, document that explicitly on https://docs.openclaw.ai/channels/imessage and then close after the maintainer decision.
