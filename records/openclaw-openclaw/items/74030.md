## What This Changes

This PR adds a repeatable `openclaw mcp serve --scope` option, plumbs `operatorScopes` through the MCP server and bridge, and adds CLI/bridge tests while validating against all `KNOWN_OPERATOR_SCOPES`.

Use #74428 as the canonical implementation path: keep the default MCP bridge scopes unchanged, allow explicit least-privilege selection only among read/write/approvals, document the new flag, include the changelog attribution, and land with focused CLI and bridge regression tests.
