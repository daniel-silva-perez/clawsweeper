## What This Changes

This PR parses outbound `[[react:...]]` and `[[react_to_current:...]]` directives, carries Telegram reaction metadata through outbound payloads, sends native Telegram reactions, permits reaction-only replies, and adds focused normalization/adapter tests.

## Best Possible Solution

Rebase the PR onto current main, preserve current markdown image extraction behavior, carry reaction intent through every shared reply payload construction path that consumes `parseReplyDirectives`, resolve `react_to_current` to the triggering message, type any forwarded Telegram delivery scopes, add the required changelog entry, and land it with tests covering reaction-only and text/media-plus-reaction replies.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: reaction directives are not parsed into reaction metadata, no Telegram reaction data is added during outbound normalization, and reaction-only Telegram replies are rejected before `reactMessageTelegram` can run.

## Solution Assessment

No, not as currently submitted. The implementation direction is right, but this branch loses reaction metadata in existing parser consumers, does not preserve the `react_to_current` target, forwards an undeclared delivery field, and needs a clean rebase plus changelog before it is the narrow maintainable fix.
