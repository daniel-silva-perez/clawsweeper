Keep this PR open. The PR also needs maintainer review because the diff includes unrelated guard-script and delivery-routing changes plus unresolved review concerns.

## Best Possible Solution

Keep the PR open, but ask for a focused revision: split unrelated guard and delivery-routing changes out, keep the memory feature to `memoryInjection` plus `memorySearch.autoRecall`, resolve the existing review comments, and add targeted tests for bootstrap filtering, config schema validation, disabled-memory-search warnings, per-agent overrides, and hot config reload behavior.
