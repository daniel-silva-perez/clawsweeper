## Summary

Keep open. Current main still does not classify the built-in `commands` chat slash command before the generic plugin allow-list fallback, so the PR is not obsolete. The branch is relevant but not merge-ready because it routes control flow through user-facing diagnostic text and carries unrelated Matrix ACP test expectation changes.

## What This Changes

The PR changes CLI missing-command diagnostics/tests so `openclaw commands` is described as a built-in runtime slash command, and also updates Matrix ACP spawn test target expectations from `channel:` to `room:` prefixes.

## Best Possible Solution

Land a focused CLI fix in the current policy structure that returns a structured result/code for built-in runtime slash-command hints, uses that structured signal on the Commander unknown-command path, covers `commands` with and without `plugins.allow` in policy and exit tests, and leaves unrelated ACP Matrix expectations out of the branch.
