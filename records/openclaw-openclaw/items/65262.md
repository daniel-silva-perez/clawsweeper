## Summary

Keep this PR open. Current main still returns the generic `plugins.allow` diagnostic for `openclaw commands` instead of recognizing `commands` as a built-in chat slash command, so the PR is not obsolete. The branch remains useful, but it still needs maintainer review because it branches on user-facing diagnostic text and includes unrelated ACP Matrix test expectation changes.

The PR changes CLI missing-command diagnostics/tests so `openclaw commands` is described as a runtime slash command, adds exit-path coverage for that hint, and also changes Matrix ACP spawn test expectations.

## Best Possible Solution

Land a focused CLI fix that classifies built-in runtime slash commands with structured data, preserves plugin allow/disable diagnostics for real plugin command aliases, covers `commands` with and without `plugins.allow` on both the policy and Commander exit paths, and leaves unrelated ACP Matrix tests out of the branch.
