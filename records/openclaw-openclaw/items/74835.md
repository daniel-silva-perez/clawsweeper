## Summary

Keep open. Current main still has per-provider model request overrides for proxy/TLS/auth and `allowPrivateNetwork`, but no schema, type, docs, or runtime merge source for a global/default model-provider request policy. The linked top-level proxy and Fake-IP reports are relevant operator evidence, but they do not supersede this narrower request.

## What This Changes

Add a global/default model-provider request policy so explicit proxy routing and private-network allowance can be configured once and overridden per provider.

## Best Possible Solution

Design a `models`-scoped provider request defaults layer that merges before per-provider `request` overrides, keeps local/self-hosted providers explicitly scoped, is visible in effective config inspection, and updates schema/help/docs/redaction/tests with a clear SSRF threat-model note.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure one of the proposed global defaults and omit `models.providers.<id>.request`; the strict schema has no accepted default field, and the runtime only reads the selected provider's request block.

## Solution Assessment

Unclear. The direction is plausible and narrower than relying on the linked process-wide proxy path, but the config shape and global SSRF inheritance semantics need maintainer security/product design before this should become an automated fix PR.
