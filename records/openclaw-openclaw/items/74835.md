## Summary

Keep open. Current main still supports only per-provider model request overrides for proxy/TLS/auth and `allowPrivateNetwork`; the schema, types, docs, and runtime paths do not expose or merge a global/default model-provider request policy. The related Fake-IP and top-level proxy reports are useful operator evidence, but they do not supersede this narrower request.

## What This Changes

The issue requests a global/default request policy for model providers so proxy and private-network SSRF settings can be configured once and inherited by providers unless overridden.

## Best Possible Solution

Design a `models`-scoped provider request defaults layer that merges before `models.providers.<id>.request`, preserves per-provider overrides, handles local/self-hosted providers explicitly, appears in effective config inspection, and includes SSRF threat-model docs plus schema/redaction/runtime tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure one of the proposed global default paths and omit `models.providers.<id>.request`; the strict schema has no accepted default field, and the runtime only reads the selected provider's request block.

## Solution Assessment

Unclear. The direction is plausible and narrower than relying on the process-wide proxy path, but the config shape and global private-network inheritance semantics need maintainer/security design before an automated implementation is the best fix.
