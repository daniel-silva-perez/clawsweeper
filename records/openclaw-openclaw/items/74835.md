## Summary

Keep open. Current main still supports only per-provider model request overrides for proxy/auth/TLS and `allowPrivateNetwork`; there is no accepted schema/type/docs/runtime merge point for a global or default model-provider request policy. The linked top-level proxy and Fake-IP reports are useful operator evidence, but they do not supersede this narrower ergonomics and security-design request.

## What This Changes

Add a global/default model-provider request policy so proxy routing and private-network SSRF allowance can be configured once and overridden per provider.

## Best Possible Solution

Design a `models`-scoped provider request defaults layer that merges before `models.providers.<id>.request`, keeps local/self-hosted providers explicitly scoped, exposes effective per-provider policy for audit, and updates schema/help/docs/redaction/tests with a clear SSRF threat-model note.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure one of the proposed global defaults and omit `models.providers.<id>.request`; current strict schema has no accepted default field, and runtime request resolution only reads the selected provider's `request` block.

## Solution Assessment

Unclear. The requested direction is plausible and narrower than relying on the linked process-wide proxy path, but the config shape and global SSRF inheritance semantics need maintainer security/product design before implementation.
