## Summary

Keep this PR open. Current main does not already implement the PR's intended attribution change: `FailoverError` still lacks `sessionId`/`lane`, `describeFailoverError` still omits provider/model/profile/session fields, and the exhausted model-fallback path still throws `FallbackSummaryError` without request-origin session metadata. The linked issue remains open, and this PR is an active implementation candidate rather than obsolete.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to either extend this PR or supersede it with a narrow observability fix that carries normalized request-origin metadata through the actual model-fallback path: `runWithModelFallback` inputs, `FallbackSummaryError`, `logModelFallbackDecision`, and the relevant runner call sites, with focused tests for exhausted multi-provider fallback structured logs. If maintainers decide `FailoverError` is still the right surface, this PR's additive fields can be part of that implementation, but it should prove the linked issue's gateway log attribution path before landing.
