## Summary

Keep this PR open. Current main still does not implement the PR's intended attribution propagation: `FailoverError` lacks `sessionId`/`lane`, `describeFailoverError` still drops provider/model/profile/session/lane fields, and `coerceToFailoverError` cannot accept session/lane or fall back to `signal.provider`. The linked issue remains open, and this PR is an active implementation candidate rather than obsolete.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to either extend this PR or replace it with a narrow observability fix that carries normalized request-origin `sessionId`/`lane` plus last-attempt provider/model/profile through the actual fallback path: `FailoverError`/`describeFailoverError`/`coerceToFailoverError`, `runWithModelFallback`, `FallbackSummaryError`, `logModelFallbackDecision`, and the runner call sites that know the originating session. Add focused regression coverage for the linked issue's exhausted multi-provider fallback structured-log case before landing.
