## Summary

Keep this issue open. Current main still does not preserve or forward `extraBody`/`extra_body` for OpenAI-compatible TTS endpoints: config can accept object-valued provider fields, but the bundled OpenAI speech provider normalizes only known TTS fields and `openaiTTS` serializes a fixed `/audio/speech` request body.

## Best Possible Solution

Implement a narrow bundled OpenAI speech-provider change: accept `extraBody` and `extra_body` as JSON records under `messages.tts.providers.openai` (and through the existing legacy OpenAI config normalization path), preserve them through `resolveConfig`/provider reads, merge them into both normal and telephony `/audio/speech` request bodies with documented collision precedence, add focused request-body tests, and document the field on https://docs.openclaw.ai/tools/tts and the OpenAI provider docs.
