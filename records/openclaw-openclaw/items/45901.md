## Summary

Keep open. Current main still creates managed session store, transcript, lock, setup, and Gateway agent session directories with plain recursive mkdir paths, so the hardening is not obsolete. The PR branch addresses a real remaining security hardening gap, but it still has a likely TypeScript type error in the new helper and should get maintainer review before merge.

## What This Changes

The PR adds managed session-directory hardening helpers, wires setup/onboarding/Gateway/session store/transcript/lock/probe flows through them, and adds permission, symlink, and custom-path regression tests.

## Best Possible Solution

Land a focused, maintainer-reviewed hardening that centralizes managed session directory creation with private POSIX permissions, preserves plain mkdir behavior for caller-supplied custom paths, rejects unsafe symlink traversal before permission mutation, fixes the FileHandle type import, and passes targeted session/setup/Gateway tests plus the changed gate.
