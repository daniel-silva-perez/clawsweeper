## Summary

Keep open. Current main still creates managed session directories through plain recursive mkdir paths, so the hardening is not already implemented; this PR addresses a real gap but the latest head still has a TypeScript type-import blocker that should be fixed before merge.

## What This Changes

The PR adds managed session-directory hardening helpers, wires setup/onboarding/Gateway/session store/transcript/lock/probe flows through them, and adds permission, symlink, custom-root, and custom-path regression tests.

## Best Possible Solution

Land a focused hardening that centralizes managed session directory creation with private POSIX permissions, rejects unsafe managed-path symlink traversal before permission mutation, preserves plain mkdir behavior for caller-supplied custom paths, and passes targeted session/setup/Gateway tests plus the changed gate.

## Reproduction Assessment

Yes. On current main, a POSIX first-run or managed session-store write with a normal umask can follow the plain recursive mkdir paths shown above and produce default directory permissions; the PR body also gives a concrete setup/store-write inspection path.

## Solution Assessment

No, not as currently proposed. The design direction is maintainable, but the PR head should first fix the FileHandle type import and then rerun the targeted tests and changed gate before merge.
