## Summary

Keep open. Current main still creates several managed session directories with plain recursive mkdir, so the hardening work is not obsolete; however, this PR is not ready to merge because the branch drops current-main validation that rejects compaction-checkpoint-shaped session IDs.

## What This Changes

The PR adds private-permission helpers for managed session directories, routes setup/onboarding/agent creation/session store/transcript/lock/probe flows through them, and adds permission, symlink, custom-root, and custom-path coverage.

## Best Possible Solution

Rebase or update this PR, keep the managed-session hardening centralized and scoped to managed paths, preserve plain mkdir behavior for truly caller-supplied custom paths, carry forward the current compaction-checkpoint session ID guard and test, then land after maintainer security review and targeted session/setup/gateway validation.

## Reproduction Assessment

Yes. Current main still has managed session store, transcript, and lock paths that create the sessions directory without an explicit private mode, and the PR regression is reproducible by comparing current main's checkpoint-shaped session ID rejection with the PR head's regex-only validation.

## Solution Assessment

Yes as a direction, but no as the current branch. A centralized managed-session-directory helper is the narrow maintainable fix, but the branch must preserve current session ID validation before it is safe to merge.
