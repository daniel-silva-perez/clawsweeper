## Summary

Keep open. Current main still has default-permission recursive mkdir paths for managed session stores, transcripts, locks, onboarding, and Gateway agent creation, so the hardening is not obsolete; the PR head is useful but still has a concrete TypeScript type issue that should be fixed before merge.

## What This Changes

The PR adds managed session-directory permission hardening helpers, routes setup/onboarding/agent creation/session store/transcript/lock/probe flows through them, and adds permission, symlink, custom-root, and custom-path regression tests.

## Best Possible Solution

Land a focused hardening that centralizes managed session directory creation with private POSIX permissions, rejects unsafe managed-path symlink traversal before permission mutation, preserves plain mkdir behavior for caller-supplied custom paths, and has targeted session/setup/Gateway tests plus changed-gate proof.

## Reproduction Assessment

Yes. On current main, a POSIX setup, managed store write, transcript header write, or lock acquisition can follow the plain recursive mkdir call sites above and create a managed sessions directory with the process default mode; the PR body also provides a concrete stat-based verification path.

## Solution Assessment

No, not as currently proposed. The design direction is maintainable, but the PR head should first fix the FileHandle type annotation and then rerun the targeted tests and changed gate before merge.
