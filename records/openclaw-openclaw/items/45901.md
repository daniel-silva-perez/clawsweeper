## Summary

Keep open. Current main still creates several managed session directories with plain recursive mkdir, while this PR addresses the real privacy hardening gap with a centralized helper and focused regression coverage; the remaining action is maintainer rebase, security review, and validation rather than cleanup closure.

## What This Changes

The PR adds private-permission session directory helpers, routes managed setup/onboarding/agent creation/store/transcript/lock flows through them, and adds permission, symlink, custom-root, and custom-path tests.

## Best Possible Solution

Rebase or update this PR, keep hardening centralized around managed session directories, preserve plain mkdir behavior for truly caller-supplied custom paths, and land after maintainer security review plus targeted session/setup/Gateway tests and the changed gate.

## Reproduction Assessment

Yes. On POSIX, Node mkdir defaults directory creation to 0o777 before umask, and current main still has setup/store/transcript/lock paths that create managed session directories without explicit private mode or chmod; the stat-based reproduction described in the PR is coherent.

## Solution Assessment

Yes as a product/code direction. A centralized helper for managed session directories is the narrow maintainable solution, but this branch still needs maintainer rebase and validation before it should be considered landable.
