## Summary

Keep open. Current main still lacks the PR's central managed-session-directory hardening, and the open PR remains a plausible security-hardening candidate, but it needs maintainer review and at least one likely type fix before it can merge safely.

## What This Changes

The PR adds a shared private-session-directory helper, wires managed session store/transcript/setup/onboarding/agent-create/lock/probe paths through it, preserves plain mkdir behavior for custom paths, and adds permission and symlink regression tests.

## Best Possible Solution
