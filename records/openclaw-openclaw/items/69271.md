## Summary

Keep this issue open. Current main still exposes Control UI session deletion through row selection plus the bulk delete bar, not a per-row delete/archive/actions affordance. The Gateway delete path archives transcript files as a side effect, but there is still no first-class archive state, archive filter, or recovery-oriented archive action in the Sessions view.

## Best Possible Solution

Keep this issue open. The narrow UI fix is to add a compact row Actions column in ui/src/ui/views/sessions.ts with a Delete action that calls the existing deleteSessionsAndRefresh(state, [key]) path and preserves confirmation, loading, refresh, and selected-key cleanup behavior. Treat the requested archive option as a coordinated Gateway/session API follow-up unless maintainers accept the existing transcript-archive-on-delete behavior as sufficient wording for the first row action.
