## Summary

Keep open. Current main still has only checkbox-plus-bulk-bar deletion in the Control UI Sessions view, with no per-row Delete/Archive/Actions affordance. The existing delete helper can already handle a single key, so the row-delete part is a narrow implementation candidate; first-class Archive remains broader Gateway/session API and product work.

## Best Possible Solution

Keep this issue open. Implement the narrow per-row Delete or compact Actions affordance in the Control UI Sessions table by reusing deleteSessionsAndRefresh(state, [key]) and preserving current confirmation/loading/error/refresh semantics. Treat first-class Archive as a separate Gateway/session API and product-design follow-up before adding archive UI.
