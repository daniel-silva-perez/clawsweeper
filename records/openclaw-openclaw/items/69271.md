## Summary

Current main still supports Control UI session deletion through row selection plus the bulk delete bar only. There is no per-row Delete/Archive/Actions affordance, and the Gateway delete path archives transcript files as a delete side effect rather than exposing first-class archived session state, filtering, or recovery.

## Best Possible Solution

Keep this issue open. The narrow implementation path is to add a compact per-row Actions/Delete affordance in the Sessions table that calls the existing deleteSessionsAndRefresh(state, [key]) flow and preserves current confirmation/loading/error/refresh semantics. Treat first-class Archive as a separate Gateway/session API design or coordinate it with the related archive work before adding archive UI.
