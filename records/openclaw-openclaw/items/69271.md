## Summary

Keep open. Current main still exposes session deletion only through row checkboxes plus the bulk delete bar in the Control UI Sessions view; there is no per-row Delete, Archive, or Actions control. The existing delete helper already supports a single key, so the per-row Delete affordance is a narrow fix candidate, while first-class Archive still needs separate Gateway/session API and product design work.

## What This Changes

Request adds a per-session Delete, Archive, or Actions affordance to the Control UI Sessions table so individual sessions can be cleaned up without bulk selection.

## Best Possible Solution

Keep this issue open and implement the narrow per-row Delete or compact Actions affordance in the Control UI Sessions table by reusing deleteSessionsAndRefresh(state, [key]) and preserving the current confirmation/loading/error/refresh behavior. Treat first-class Archive as a separate Gateway/session API and product-design follow-up before adding archive UI.
