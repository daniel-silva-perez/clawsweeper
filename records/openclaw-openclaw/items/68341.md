## Summary

Keep open. Current main still lacks the core poll inflight dedupe behavior, and send/message.action still register inflight work only after earlier async validation. The latest PR diff is not obsolete, but it remains mixed: the prior MCP owner-token downgrade appears gone from the final diff, while the failover classifier change conflicts with current main’s tested plain-status behavior.

## Best Possible Solution

Keep this PR open, but ask for a narrower maintainer-reviewable version. The best path is to split or reduce it to the gateway idempotency race, with focused tests for poll, send, and message.action concurrent duplicates that prove the inflight promise is registered before the first await. Leave the failover and cron changes out unless they have a concrete reproduction and tests aligned with current behavior.
