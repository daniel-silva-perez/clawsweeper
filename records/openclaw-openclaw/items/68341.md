## Summary

Keep open. Current main still does not implement the central poll inflight dedupe behavior, and the PR remains an active contributor implementation candidate. It is not ready to close as obsolete because the gateway race it targets is still present, but it should be narrowed before maintainer review because the latest diff mixes gateway idempotency, cron trust, maintenance cleanup, and failover classification changes.

## Best Possible Solution

Keep this PR open, but ask for a narrower maintainer-reviewable revision. The best path is to split out the gateway idempotency race and implement one shared pattern that registers the inflight promise synchronously before any async validation/channel resolution for `send`, `message.action`, and `poll`, with concurrent duplicate tests for each. Leave the failover, cron, and maintenance changes out unless each has a concrete reproduction and tests aligned with current main behavior.
