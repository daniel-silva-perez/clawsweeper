## Summary

Keep this PR open. Current main at ab5c8025c9d0a31873d311ae3e73f3664b0a1185 still has the gateway inflight dedupe race this PR targets: `send` and `message.action` await channel resolution after the inflight lookup but before registration, and `poll` still has only completed-result dedupe with no inflight path. The provided PR context shows the latest diff is focused on `src/gateway/server-methods/send.ts` and `src/gateway/server-methods/send.test.ts`, and the author has said it will be reworked, so maintainer review or replacement is the right outcome.

## Best Possible Solution

Keep this PR open for maintainer review or replacement. The fix should synchronously claim/register the inflight promise before the first await for `send`, `message.action`, and `poll`; duplicate callers should await the same result with cached metadata; existing authorization, scope, channel selection, and dedupe cache semantics should be preserved; and focused concurrent regression tests should land in `src/gateway/server-methods/send.test.ts`.
