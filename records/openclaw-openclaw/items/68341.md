## Summary

Keep this PR open. Current main still has the gateway inflight race the PR targets: `message.action` and `send` resolve channel/provider state before registering work in the inflight map, and `poll` only checks the completed dedupe cache with no inflight path. The live PR is still open, narrowed to `src/gateway/server-methods/send.ts` and `src/gateway/server-methods/send.test.ts`, and its latest diff does not touch workflows, dependencies, lockfiles, release scripts, or other supply-chain-sensitive files.

## Best Possible Solution

Keep this PR open for maintainer review or replacement. The best implementation path is to register the inflight promise synchronously before any async channel/provider resolution for `send`, `message.action`, and `poll`, return shared inflight results with cached metadata for duplicate callers, preserve existing owner-authorization and gateway scope propagation semantics, and land focused concurrent regression tests in `src/gateway/server-methods/send.test.ts`.
