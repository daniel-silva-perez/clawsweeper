## Summary

Keep open. Current main still does not implement the PR's narrowed gateway inflight-dedupe behavior: `message.action` and `send` can still do async channel work before registering inflight work, and `poll` still has no inflight map at all. The latest PR head is now a focused implementation candidate for `src/gateway/server-methods/send.ts` plus tests, but its body still describes unrelated MCP/cron/failover/maintenance changes that were reverted or moved elsewhere, so it needs maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open as an active implementation candidate, but ask for a focused title/body matching the latest diff and a maintainer-reviewable gateway idempotency patch. The best path is to validate/register inflight work before async channel/provider work for `send`, `message.action`, and `poll`, preserve current owner-authorization semantics, add or retain targeted concurrent regression tests, and run the targeted gateway send test file before merge.
