Keep this PR open. Current main still has the gateway inflight dedupe race for send/message.action and no inflight path for poll, and the PR is an active implementation lane rather than work already implemented on main. The current PR diff is narrowed to the gateway send handler and its tests, with no apparent workflow, dependency, lockfile, release, or script changes in the security/supply-chain pass.

## What This Changes

The PR refactors gateway send, message.action, and poll dedupe handling to distinguish cached/inflight/ready states, register shared work promises before async dispatch, and add concurrent regression tests in src/gateway/server-methods/send.test.ts.

## Best Possible Solution

Keep this PR open for maintainer review or a focused rework. The fix should synchronously claim/register the inflight promise before the first await for send, message.action, and poll; duplicate callers should await the same result with cached metadata; all three handlers should consistently handle cached, inflight, and ready states; and existing authorization, senderIsOwner propagation, scope forwarding, channel selection, and dedupe cache semantics should be preserved with focused concurrent tests.
