## Summary

Keep open. Current main does not implement the central poll/message-action inflight dedupe work, and the PR is still an active implementation candidate. It should not merge as-is: the current PR head still leaves a race window before inflight registration and conflicts with the existing MCP owner-token contract.

## Best Possible Solution

Keep this PR open for maintainer review, but require it to be narrowed or split. The gateway dedupe fix should register the shared work promise synchronously before any await for send, message.action, and poll, with focused concurrent idempotency tests. MCP bearer-token owner propagation should be preserved, and failover/cron/maintenance changes should be handled separately with concrete repros and regression tests.
