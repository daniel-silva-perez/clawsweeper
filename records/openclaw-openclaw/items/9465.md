## Summary

Keep open. Current main still lacks the persisted cron hook lifecycle requested here, Gateway cron schemas still omit hook fields, scheduler/startup/manual execution paths still call the core executor without hook phases, and the discussion plus open PR #45465 show this is active product/security design work rather than cleanup-ready.

## What This Changes

Add cron job lifecycle hooks that can run custom pre, post, and error logic such as agent turns, system events, shell commands, and webhooks.

## Best Possible Solution

Land or explicitly supersede an additive cron hook lifecycle covering persisted job/config types, Gateway protocol schemas and RPCs, scheduled/startup/manual execution paths, CLI, Control UI, docs, tests, and an operator-controlled security model for user hooks.

## Security Review

Status: needs_attention

Summary: The requested cron shell/webhook lifecycle is security-sensitive and needs explicit design review before implementation.

Concerns:

- **[medium] Define user hook execution boundaries:**
  - body: Shell and webhook cron hooks can run local commands or outbound network calls before and after scheduled jobs. The design needs clear permission, timeout, logging, failure, and skip semantics so hooks do not become an implicit privileged execution path.
  - confidence: 0.9
