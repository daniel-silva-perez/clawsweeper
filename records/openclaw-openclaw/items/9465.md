## Summary

Keep this issue open. Current main at f2405c830bd925bd24f91b6bac5d417c6d36e5ba still lacks a persisted per-job cron hooks contract: cron types and Gateway schemas have no hooks field, scheduled/startup/manual paths call the core executor directly, and CLI/Control UI only expose existing schedule, payload, delivery, timeout, tools, and failure-alert controls. The discussion also has concrete zero-token pre-gate use cases and open implementation PRs #45129 and #45465, so cleanup closure would be premature.

## Best Possible Solution

Keep this issue open until maintainers either land an additive first-class cron hook lifecycle across persisted cron job types, Gateway schema/RPC, scheduled/startup/manual execution paths, CLI, Control UI, docs, and tests, or explicitly supersede it with a narrower canonical tracker. Existing plugin hooks are useful adjacent workarounds, but they do not replace the requested job-defined cron hook contract.
