## Summary

Keep open. Current main still lacks the persisted cron hook lifecycle requested here, and the latest maintainer review plus related open PR show this is active product/security design work rather than cleanup-ready stale work.

## What This Changes

Add configurable cron job hooks that can run before, after, or on error around scheduled execution, including deterministic pre-run gates that can skip an agent/model call.

## Best Possible Solution

Land or explicitly supersede an additive cron hook lifecycle that covers persisted job/config types, Gateway protocol schemas and RPCs, scheduled/startup/manual execution paths, CLI, Control UI, docs, tests, and a clear security model for user-configured hooks.
