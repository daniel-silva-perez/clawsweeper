## Summary

Keep this issue open. Current main still does not expose a persisted cron hook lifecycle or a deterministic pre-run gate that can skip an agent/model call, and the related open PR is an implementation candidate rather than merged proof.

## What This Changes

Request to add cron job pre/post/error hooks, especially deterministic pre-run shell/webhook-style gating that can skip LLM execution for no-op cron runs.

## Best Possible Solution

Land or explicitly supersede an additive cron hook lifecycle that covers persisted job/config types, Gateway protocol schemas and RPCs, scheduled/startup/manual execution paths, CLI, Control UI, docs, tests, and a clear security model for user-configured hooks.
