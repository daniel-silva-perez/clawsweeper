## Summary

Keep open. Current main still lacks a persisted cron hook lifecycle, the existing plugin cron hook is observe-only, and the discussion points to active product/security design rather than cleanup-ready work.

## What This Changes

Request to add configurable cron-job hooks around execution, including pre/post/error phases and shell/webhook/agent/system-event hook kinds for conditional runs, reminders, escalation, and preprocessing.

## Best Possible Solution

Land or explicitly supersede an additive cron hook lifecycle that covers persisted job/config types, Gateway protocol schemas and RPCs, scheduled/startup/manual execution paths, CLI, Control UI, docs, tests, and an operator-controlled security model for user hooks.

Summary: No patch is under review, but the requested shell/webhook cron lifecycle is security-sensitive and needs maintainer design before implementation.
