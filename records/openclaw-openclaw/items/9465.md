## Summary

Keep open. Current main still lacks the requested persisted cron hook lifecycle: cron job types and Gateway schemas have no hook fields, scheduled/startup/manual execution paths call the core executor directly, and the existing plugin cron hook is observe-only. The related open PR #45465 remains the implementation candidate, and the requested shell/webhook behavior needs maintainer security review before cleanup closure.

## What This Changes

Request adds pre/post/onError cron job hooks for agent turns, system events, shell commands, and webhooks so scheduled jobs can gate, retry, escalate, and run custom logic around execution.

## Best Possible Solution

Land or explicitly supersede #45465 with an additive cron hook lifecycle covering persisted job/config types, Gateway schemas and RPCs, scheduled/startup/manual execution paths, CLI, Control UI, docs, tests, and an operator-controlled security model for user-configured hooks.

## Security Review

Status: needs_attention

Summary: The requested shell/webhook cron lifecycle is security-sensitive and needs explicit design review before implementation.

Concerns:

- **[medium] Define user hook execution boundaries:**
  - body: Cron shell and webhook hooks would execute local commands or outbound network calls around scheduled jobs. The design needs clear permission, timeout, logging, skip, and failure semantics so hooks do not become an implicit privileged execution path.
