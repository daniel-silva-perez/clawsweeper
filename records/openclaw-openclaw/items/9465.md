## Summary

Keep open. Current main still lacks persisted cron execution hooks, the available plugin cron hook is observe-only, and #45465 is already the active open implementation candidate for this design-sensitive work.

## What This Changes

The issue requests cron job lifecycle hooks that can run custom pre/post/error logic, including shell/webhook gates before scheduled agent execution.

## Best Possible Solution

Land or explicitly supersede an additive cron hook lifecycle that covers persisted job/config types, Gateway protocol schemas and RPCs, scheduled/startup/manual execution paths, CLI, Control UI, docs, tests, and an operator-controlled security model for user hooks.

## Security Review

Status: needs_attention

Summary: No patch is under review, but the requested shell/webhook cron lifecycle is security-sensitive and needs explicit design review before implementation.

Concerns:

- **[medium] Define user hook execution boundaries:**
  - body: Shell and webhook cron hooks can run local commands or outbound network calls before and after scheduled jobs. The design needs clear permission, timeout, logging, failure, and skip semantics so hooks do not become an implicit privileged execution path.
  - confidence: 0.9
