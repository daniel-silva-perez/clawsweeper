## Summary

Keep open. Current main still lacks persisted cron pre/post/error hook support, and the issue discussion now contains concrete production use cases for deterministic pre-LLM gating. The latest maintainer comment plus open implementation candidate #45465 make this active product/security design work, not conservative cleanup.

## What This Changes

Add cron job lifecycle hooks, especially pre-run shell/webhook/agent/system hooks, so scheduled jobs can be gated or extended before and after execution.

## Best Possible Solution

Land or explicitly supersede an additive cron hook lifecycle that covers persisted job/config types, Gateway protocol schemas and RPCs, scheduled/startup/manual execution paths, CLI, Control UI, docs, tests, and a clear operator-controlled security model for user hooks.
