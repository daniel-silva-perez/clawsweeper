## Summary

Keep open. Current main still lacks persisted cron execution hooks, Gateway cron RPC schemas still reject hook fields, scheduled/startup/manual execution paths still call the core executor without hook phases, and the existing plugin cron hook is observe-only. The recent discussion also shows active real-world demand and an open implementation candidate, so this is not cleanup-ready.

## What This Changes

The issue requests a cron job hook lifecycle for pre-run gating plus post/error custom logic for reminders, escalation, conditional execution, shell checks, and webhooks.

## Best Possible Solution

Land or explicitly supersede an additive cron hook lifecycle covering persisted job/config types, Gateway protocol schemas and RPCs, scheduled/startup/manual execution paths, CLI, Control UI, docs, tests, and an operator-controlled security model for user hooks.

## Security Review

Status: needs_attention

Summary: No patch is under review, but the requested shell/webhook cron lifecycle is security-sensitive and needs explicit design review before implementation.

Concerns:

- **[medium] Define user hook execution boundaries:**
  - body: Shell and webhook cron hooks can run local commands or outbound network calls before and after scheduled jobs. The design needs clear permission, timeout, logging, failure, and skip semantics so hooks do not become an implicit privileged execution path.
