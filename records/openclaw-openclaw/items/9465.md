## Summary

Keep this issue open. Current main still lacks the requested persisted per-job cron hooks lifecycle: there is no `hooks` field in cron job types, Gateway cron schemas, CLI/UI create/edit payloads, or scheduler/manual/startup execution flow. Recent main added useful plugin hook surfaces (`before_agent_reply` with `ctx.jobId`, plus observation-only `cron_changed`), but those are global plugin workarounds/observers rather than a job-defined `hooks.pre/post/onError` cron contract. Open PRs #45129 and #45465 remain implementation candidates and need maintainer/security review before this issue can close.

## Best Possible Solution

Keep this issue open until maintainers either land an additive first-class cron hook lifecycle across persisted cron types, Gateway protocol/schema/RPC, scheduler/manual/startup paths, CLI, Control UI, docs, and tests, or explicitly supersede it with a canonical narrower tracker. The existing `before_agent_reply`/`cron_changed` plugin surfaces should be documented as partial workarounds, not treated as the replacement for job-defined cron hooks.
