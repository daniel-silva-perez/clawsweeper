## Summary

Keep open. Current main still lacks persisted cron lifecycle hooks or a deterministic pre-execution gate before cron jobs invoke the agent/model path, while the issue discussion documents concrete no-op and debounce use cases and the related implementation PR #45465 remains open for maintainer review.

## What This Changes

Add a cron job hook lifecycle so jobs can run configured pre/post/error logic, including shell or webhook gates that can skip or augment scheduled execution.

## Best Possible Solution

Review and either land or supersede #45465 with an additive cron hook lifecycle covering persisted job/config types, Gateway schemas and RPCs, scheduled/startup/manual execution paths, CLI, Control UI, docs, tests, and an explicit operator-controlled security model for user-configured hooks.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. We do have a high-confidence verification path: source and schema inspection on current main shows no persisted hooks field and no hook phase before executeJobCoreWithTimeout.

## Solution Assessment

Unclear for the exact proposed schema because this is active product and security design work. The narrow maintainable path is a maintainer-reviewed additive cron lifecycle design, likely through #45465 or a successor, rather than a cleanup close or a ClawHub-only plugin workaround.

## Security Review

Status: needs_attention

Summary: The requested shell/webhook cron lifecycle is security-sensitive and needs explicit design review before implementation.

Concerns:

- **[medium] Define user hook execution boundaries:**
  - body: Cron shell and webhook hooks would execute local commands or outbound network calls around scheduled jobs. The design needs clear permission, timeout, logging, skip, and failure semantics so hooks do not become an implicit privileged execution path.
