## Summary

Keep open. Current main still lacks persisted cron lifecycle hooks or a deterministic cron pre-execution gate before agent/model execution, and the discussion plus related open #45465 show this is active product and security design work rather than cleanup-ready stale work.

## What This Changes

Request adding cron job hooks that can run custom pre, post, and error logic, including deterministic shell/webhook gates that can skip or react to scheduled execution.

## Best Possible Solution

Review and either land or explicitly supersede #45465 with an additive cron hook lifecycle covering persisted job/config types, Gateway schemas and RPCs, scheduled/startup/manual execution paths, CLI, Control UI, docs, tests, and an operator-controlled security model for user-configured hooks.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. We do have a high-confidence verification path: source and schema inspection on current main shows no persisted hooks field and no hook phase before executeJobCoreWithTimeout.

## Solution Assessment

Unclear for the exact proposed schema because this is active product and security design work. The narrow maintainable path is a maintainer-reviewed additive cron lifecycle design, likely through #45465 or a successor, rather than a cleanup close or a ClawHub-only workaround.

## Security Review

Status: needs_attention

Summary: The requested shell/webhook cron lifecycle is security-sensitive and needs explicit design review before implementation.

Concerns:

- **[medium] Define cron hook execution boundaries:**
  - body: Cron shell and webhook hooks would execute local commands or outbound network calls around scheduled jobs. The design needs clear operator controls for permissions, timeouts, logging, skip behavior, failure handling, and secret exposure so hooks do not become an implicit privileged execution path.
