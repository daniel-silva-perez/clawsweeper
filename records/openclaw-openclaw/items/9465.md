## Summary

Keep open. Current main still lacks persisted cron lifecycle hooks or a deterministic pre-execution gate, and the discussion plus open #45465 show this is active product/security design work rather than cleanup-ready.

## What This Changes

Add cron job lifecycle hooks such as pre, post, and onError so custom logic can run before or after cron execution, including deterministic skip gates before model calls.

## Best Possible Solution

Review and either land or explicitly supersede #45465 with an additive cron hook lifecycle covering persisted job/config types, Gateway schemas and RPCs, scheduled/startup/manual execution paths, CLI, Control UI, docs, tests, and an operator-controlled security model.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. There is a high-confidence verification path: source and schema inspection on current main shows no persisted hooks field and no hook phase before executeJobCoreWithTimeout.

## Solution Assessment

Unclear for the exact proposed schema because this is active product and security design work. The narrow maintainable path is a maintainer-reviewed additive lifecycle design through #45465 or a successor, not a cleanup close or a separate automated repair branch.

## Security Review

Status: needs_attention

Summary: The requested shell/webhook cron lifecycle is security-sensitive and needs explicit design review before implementation.

Concerns:

- **[medium] Define cron hook execution boundaries:**
  - body: Cron shell and webhook hooks would execute local commands or outbound network calls around scheduled jobs. The design needs clear operator controls for permissions, timeouts, logging, skip behavior, failure handling, and secret exposure so hooks do not become an implicit privileged execution path.
