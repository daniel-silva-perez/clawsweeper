## Summary

Keep this issue open. Current main at 39f810911c382f92ea80b2fb406fdf7dbd405721 still lacks a persisted per-job cron hooks contract: cron job types, Gateway schemas, scheduler/manual/startup execution paths, CLI, Control UI, and docs do not expose hooks.pre/post/onError or an equivalent zero-token pre-gate. The concrete debounce/no-op fleet use cases remain valid, and open PR #45465 is an implementation candidate rather than proof that the issue is done.

## What This Changes

Add a cron job hooks system so jobs can run custom pre/post/error logic, including shell or webhook gates that can skip model execution before a scheduled run.

## Best Possible Solution

Keep this issue open until maintainers either land an additive first-class cron hook lifecycle across persisted cron job types, Gateway schema/RPC, scheduled/startup/manual execution paths, CLI, Control UI, docs, and tests, or explicitly supersede it with a narrower canonical tracker. The near-term path is to review, revise, or close #45465 rather than auto-closing this issue.
