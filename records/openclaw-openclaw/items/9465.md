## Summary

Keep open. Current main still lacks the requested persisted cron hook lifecycle, and the discussion plus open PR #45465 show this is active product/security design work rather than cleanup-ready. The recent maintainer review also explicitly kept this open for follow-up.

## What This Changes

Add cron job lifecycle hooks so custom pre, post, and error logic can gate or react to scheduled job execution.

## Best Possible Solution

Land or explicitly supersede #45465 with an additive cron hook lifecycle covering persisted job/config types, Gateway schemas and RPCs, scheduled/startup/manual execution paths, CLI, Control UI, docs, tests, and an operator-controlled security model for user hooks.

## Security Review

Status: needs_attention

Summary: The requested shell/webhook cron lifecycle is security-sensitive and needs explicit design review before implementation.

Concerns:

- **[medium] Define user hook execution boundaries:**
  - body: Cron shell and webhook hooks would run local commands or outbound network calls around scheduled jobs. The design needs clear permission, timeout, logging, skip, and failure semantics so hooks do not become an implicit privileged execution path.
  - confidence: 0.9
