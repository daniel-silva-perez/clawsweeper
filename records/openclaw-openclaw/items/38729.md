## Summary

Keep this PR open. Current main is now provider-profile driven for Anthropic thinking levels, and Claude Opus 4.6 remains adaptive-only in source, tests, and docs. The PR's current `supportsXHighThinking` hook is a deprecated compatibility hook and would be bypassed by Anthropic's non-empty `resolveThinkingProfile`, so the remaining question is maintainer policy/rework rather than conservative cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer decision and rework. If OpenClaw should advertise Claude Opus 4.6 max effort, implement it through `resolveClaudeThinkingProfile` or the Anthropic provider-profile path, add focused Opus 4.6/Sonnet 4.6 boundary tests, and update `docs/tools/thinking.md`. If the current Opus 4.7-only `xhigh`/`max` boundary is intentional, close this PR with that rationale. Do not merge the current hook-only diff as-is.
