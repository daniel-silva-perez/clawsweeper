## Summary

Keep this PR open for maintainer review. Current main has moved Anthropic thinking support to provider profiles: Claude Opus 4.7 advertises `xhigh`/`max`, while Claude Opus/Sonnet 4.6 advertise `adaptive` only. The PR's hook-only change uses a deprecated compatibility path and would be bypassed by the existing non-empty Anthropic `resolveThinkingProfile` result, so this is not implemented on main and still depends on a provider-capability decision.

## What This Changes

The PR adds an Anthropic `supportsXHighThinking` hook in `extensions/anthropic/register.runtime.ts` for Claude Opus 4.6 model prefixes so the legacy xhigh adapter can advertise `/think xhigh`.

## Best Possible Solution

Keep Claude thinking support defined in `resolveClaudeThinkingProfile` as the provider-profile source of truth, with Anthropic tests and thinking docs matching the selected model boundary. If maintainers want Opus 4.6 max effort exposed, update the profile path rather than adding only the deprecated legacy hook; otherwise close the PR with the Opus 4.7 rationale.
