## Summary

Keep open. Current main is profile-driven for Anthropic thinking support, and this PR adds only a deprecated legacy hook that is bypassed once Anthropic returns a non-empty `resolveThinkingProfile`; existing source and tests still keep Claude Opus 4.6 adaptive-only rather than advertising `xhigh` or `max`.

## What This Changes

The PR adds an Anthropic provider `supportsXHighThinking` callback for Claude Opus 4.6 model IDs in `extensions/anthropic/register.runtime.ts`.

## Best Possible Solution

Keep Claude thinking capabilities centralized in `resolveClaudeThinkingProfile`. If maintainers confirm Opus 4.6 should expose max effort, update that profile plus Anthropic tests and thinking docs; otherwise close this PR with the rationale that Opus 4.6 remains adaptive-only in the advertised profile.
