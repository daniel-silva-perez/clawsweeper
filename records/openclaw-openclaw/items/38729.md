## Summary

Keep open for maintainer decision. Current main has moved Anthropic thinking capability advertisement to provider profiles, and those profiles intentionally expose `xhigh`/`max` for Claude Opus 4.7 while keeping Claude Opus/Sonnet 4.6 on `adaptive` only; this PR's legacy hook-only change would not affect the effective level list while Anthropic supplies a non-empty `resolveThinkingProfile`.

## What This Changes

The PR adds a `supportsXHighThinking` hook to the Anthropic provider registration for Claude Opus 4.6 so the legacy xhigh adapter can advertise `/think xhigh`.

## Best Possible Solution

Settle the Claude Opus 4.6 capability boundary in the provider-profile source of truth. If Opus 4.6 should advertise max effort, update `resolveClaudeThinkingProfile` and the affected Anthropic/proxy tests and docs; otherwise close this PR with the rationale that Opus 4.7 is the supported `xhigh`/`max` model family.
