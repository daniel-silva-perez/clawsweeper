## Summary

Keep this PR open for maintainer decision and rework. Current main is provider-profile driven for Anthropic thinking levels, and the PR’s added `supportsXHighThinking` hook is now a deprecated compatibility hook that would be bypassed by Anthropic’s non-empty `resolveThinkingProfile`; current source, tests, and docs intentionally advertise xhigh/max for Claude Opus 4.7, not Claude Opus 4.6.

## What This Changes

The PR adds an Anthropic provider `supportsXHighThinking` compatibility hook for Claude Opus 4.6 so OpenClaw can advertise `/think xhigh` and map it to max effort.

## Best Possible Solution

Keep this PR open for maintainer decision and rework. If OpenClaw should advertise Claude Opus 4.6 max effort, implement it through `resolveClaudeThinkingProfile` or the Anthropic provider-profile path, add focused Opus 4.6/Sonnet 4.6 boundary tests, and update `docs/tools/thinking.md`. If the current Opus 4.7-only `xhigh`/`max` boundary is intentional, close this PR with that rationale. Do not merge the current hook-only diff as-is.
