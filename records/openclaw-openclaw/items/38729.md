## Summary

Keep this PR open, but it is not merge-ready against current main. Current main routes Anthropic thinking options through `resolveThinkingProfile`, and that profile is returned before the legacy `supportsXHighThinking` hook this PR adds, so the proposed callback would not make Opus 4.6 advertise `xhigh` in the current profile-driven path.

## What This Changes

The PR adds an Anthropic `supportsXHighThinking` callback so Claude Opus 4.6 model IDs can advertise `xhigh` thinking through the legacy provider hook.

## Best Possible Solution

Keep Claude thinking capabilities centralized in `resolveClaudeThinkingProfile`. If Opus 4.6 should expose max effort, update that profile plus the Anthropic/provider thinking tests and thinking docs; otherwise close this PR with the explicit policy that Opus 4.6 remains adaptive-only in the advertised profile.
