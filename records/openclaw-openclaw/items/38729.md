## Summary

Keep open. The PR is not mergeable as written: current main resolves Anthropic thinking through `resolveThinkingProfile`, and the generic resolver returns a non-empty provider profile before consulting the deprecated `supportsXHighThinking` hook. Current tests and docs intentionally keep Claude Opus 4.6 adaptive-only while reserving `xhigh`/`max` advertisement for Opus 4.7, so maintainers need to decide the provider capability before changing the profile source of truth.

## What This Changes

The PR adds an Anthropic provider `supportsXHighThinking` callback in `extensions/anthropic/register.runtime.ts` for Claude Opus 4.6 model ID prefixes.

## Best Possible Solution

Keep Claude thinking capabilities centralized in `resolveClaudeThinkingProfile`. If maintainers confirm Opus 4.6 `xhigh`/`max` should be user-facing, update that helper plus Anthropic regression tests and the thinking docs; otherwise close this PR with the rationale that Opus 4.6 remains adaptive-only in the advertised provider profile.
