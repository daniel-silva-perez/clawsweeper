## Summary

Keep this PR open. Current main is profile-driven for Anthropic thinking levels, advertises `xhigh`/`max` only for Claude Opus 4.7, and keeps Claude 4.6 adaptive-only. The PR's legacy `supportsXHighThinking` hook is stale against that path and would be bypassed by the current non-empty provider profile, so this needs maintainer policy/rework rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer decision and rework. If OpenClaw should advertise Claude Opus 4.6 max effort, implement it through `resolveClaudeThinkingProfile` or the Anthropic provider-profile path, add focused Opus 4.6/Sonnet 4.6 boundary tests, and update `docs/tools/thinking.md`. If the current Opus 4.7-only `xhigh`/`max` boundary is intentional, close this PR with that rationale. Do not merge the current hook-only diff as-is.
