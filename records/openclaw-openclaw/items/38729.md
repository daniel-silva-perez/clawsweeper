## Summary

Current main still does not implement this PR's intended Anthropic Claude Opus 4.6 `xhigh`/`max` advertisement. The current provider-profile path intentionally exposes `xhigh`/`max` for Claude Opus 4.7 only, keeps Claude 4.6 adaptive-only, and returns that profile before legacy `supportsXHighThinking` hooks are consulted. The PR remains plausible because the Anthropic transport can map an explicit Opus 4.6 `xhigh` request to provider effort `max`, but the submitted hook-only diff is stale against main and needs maintainer policy/rework rather than closure as implemented.

## Best Possible Solution

Keep this PR open for maintainer decision/rework. If OpenClaw should advertise Claude Opus 4.6 max effort, implement it through `resolveClaudeThinkingProfile` or the Anthropic provider-profile path, add focused tests for Opus 4.6 and Sonnet 4.6 capability boundaries, and update the public thinking docs. If the current Opus 4.7-only `xhigh`/`max` boundary is intentional, a maintainer should close this PR with that rationale. Do not merge the current hook-only diff as-is.
