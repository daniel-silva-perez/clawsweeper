## Summary

Keep this PR open for maintainer handling. Current main does not implement the requested Anthropic Opus 4.6 `xhigh` advertisement, but the hook-only patch is not correct against the current profile-driven thinking path: Anthropic already returns a non-empty `resolveThinkingProfile`, and that profile explicitly keeps Claude 4.6 on base levels plus `adaptive`.

## What This Changes

The PR adds an Anthropic provider `supportsXHighThinking` callback in `extensions/anthropic/register.runtime.ts` that returns true for Claude Opus 4.6 model ID prefixes.

## Best Possible Solution

Keep Claude thinking capabilities centralized in `resolveClaudeThinkingProfile`. If maintainers confirm Opus 4.6 max effort should be user-facing, update that helper plus Anthropic regression tests and thinking docs; otherwise close this PR with the rationale that Opus 4.6 remains adaptive-only in the advertised profile.
