## Summary

Keep this PR open, but the current patch is not sufficient for current main. Anthropic already uses the provider thinking profile path, and that profile takes precedence over the deprecated `supportsXHighThinking` hook the PR adds, so this change would not expose or validate `xhigh` for Claude Opus 4.6 in the current UI/directive path.

## What This Changes

The PR adds an Anthropic provider `supportsXHighThinking` callback that returns true for Claude Opus 4.6 model ID prefixes.

## Best Possible Solution

Keep Claude thinking capabilities defined through `resolveClaudeThinkingProfile`. If maintainers approve Opus 4.6 max effort as user-facing, update that profile plus Anthropic regression tests and thinking docs; otherwise close this PR with the rationale that Claude 4.6 remains adaptive-only in the advertised provider profile.
