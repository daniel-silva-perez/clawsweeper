## Summary

Keep this PR open for maintainer review. Current main uses provider thinking profiles as the source of truth, and the proposed legacy hook would be bypassed by Anthropic's existing non-empty `resolveThinkingProfile`, so the patch does not actually expose `xhigh` for Claude Opus 4.6 under the current runtime path.

## What This Changes

The PR adds an Anthropic `supportsXHighThinking` callback for Claude Opus 4.6 model IDs in `extensions/anthropic/register.runtime.ts`.

## Best Possible Solution

Keep Claude thinking support defined through `resolveClaudeThinkingProfile`. If maintainers want Opus 4.6 max effort user-facing, update that profile and matching Anthropic tests/docs; otherwise close this PR with the rationale that Opus 4.6 remains adaptive-only in the advertised profile.
