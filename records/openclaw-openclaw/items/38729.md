## Summary

Keep this PR open, but do not merge it as-is. Current main is profile-driven for thinking levels, so adding only Anthropic's legacy `supportsXHighThinking` hook will not expose `xhigh` for Claude Opus 4.6 while Anthropic already returns a non-empty `resolveThinkingProfile`.

## What This Changes

The PR adds an Anthropic provider `supportsXHighThinking` callback that returns true for Claude Opus 4.6 model ID prefixes.

## Best Possible Solution

Keep Claude thinking capabilities centralized in `resolveClaudeThinkingProfile`. If maintainers confirm Opus 4.6 max effort should be user-facing, update that helper plus Anthropic regression tests and thinking docs; otherwise close this PR with the rationale that Opus 4.6 remains adaptive-only in the advertised profile.
