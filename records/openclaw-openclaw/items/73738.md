## Summary

Keep this PR open. The ephemeral sandbox workspace feature is not implemented on current main and the branch is the active implementation path for the linked enhancement, but the latest PR head still has a concrete package URL redirect validation gap in a security-sensitive Package Acceptance download path.

## What This Changes

The branch adds `workspaceLifecycle: "persistent" | "ephemeral"` across sandbox config, docs, schema, runtime cleanup, and tests, and also touches embedded-run cleanup, Discord system-event trust annotations, Docker E2E setup, and package-candidate URL download hardening.

## Best Possible Solution

Keep this PR as the implementation path for ephemeral sandbox workspaces, but require the package URL download path to reject redirects or validate and repin every redirected HTTPS target, add focused blocked/private redirect coverage, and then rerun the targeted sandbox/config/scripts gates before merge.

## Reproduction Assessment

Yes: the blocking issue has a high-confidence source-inspection reproduction path. PR head calls `undiciFetch` without overriding the default follow-redirect behavior, while its pinned lookup only validates the original host and falls back to ordinary DNS for redirected hosts.

## Solution Assessment

No: the sandbox lifecycle design is a plausible narrow solution, but the current package URL hardening is not safe enough for a CI/package download path. The safer path is to set redirects to `error` or `manual`, or explicitly validate and repin every redirected HTTPS target before streaming it.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Reject unvalidated redirect hosts:** `scripts/resolve-openclaw-package-candidate.mjs:440`
  - body: The new dispatcher validates the original `package_url` host, but `undiciFetch(parsed, { dispatcher })` still follows redirects by default. When a redirect points at a different host, `createPinnedLookup` falls back to normal DNS, so an allowed URL can redirect to a private/internal HTTPS host and bypass the blocked-host/address checks. Set redirects to `error`/`manual`, or validate and repin each redirect target before downloading.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: Security review needs attention because the PR’s package URL hardening still permits unvalidated redirected download targets.
