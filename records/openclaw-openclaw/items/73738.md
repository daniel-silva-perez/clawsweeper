## What This Changes

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

No, not as currently proposed. The sandbox lifecycle design is a plausible narrow solution for #14804, but the security-sensitive package URL hardening should reject redirects or validate and repin every redirected HTTPS target before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Reject unvalidated redirect hosts:** `scripts/resolve-openclaw-package-candidate.mjs:235-241`
  - body: The new package URL guard only validates the original host. Because undici fetch follows redirects by default, a URL can redirect to a different HTTPS host and this fallback path delegates that host to normal DNS without the blocked-host/address checks, bypassing the SSRF protection added here. Set redirects to `error`/`manual`, or re-run the HTTPS, host, DNS, and pinning checks for every redirect target before downloading.
  - confidence: 0.88

## Security Review

Status: needs_attention
