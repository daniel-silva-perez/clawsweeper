## Summary

Keep this PR open. Current main does not implement ephemeral sandbox workspaces, and this PR is the active implementation path for the linked enhancement, but the latest head is not merge-ready because the package URL hardening still allows redirects to bypass the host/address policy.

## What This Changes

The PR adds sandbox `workspaceLifecycle` config, run-scoped ephemeral sandbox workspace/runtime cleanup, schema/docs/tests, Discord system-event trust annotations, a Docker E2E fixture tweak, and package URL download hardening.

## Best Possible Solution

Keep this PR as the implementation path for #14804, but make the package URL resolver either reject redirects or manually validate and repin every redirected HTTPS target before download, add focused redirect-to-private/internal-host coverage, and then rerun the relevant package, sandbox, config, and secops gates.

## Reproduction Assessment

Yes for the blocking review finding: source inspection of the PR head plus Undici 8.1.0 source shows default redirect following and a lookup guard that only pins the original hostname. I did not run a live exploit because this review is read-only.

## Solution Assessment

No, not as currently proposed. The sandbox lifecycle design is a plausible narrow solution, but the package URL hardening is incomplete until redirects are rejected or every redirect target is validated with the same SSRF policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Reject unvalidated package URL redirects:** `scripts/resolve-openclaw-package-candidate.mjs:440`
  - body: The new dispatcher validates and pins only the original `package_url` host, but `undiciFetch(parsed, { dispatcher })` follows redirects by default. When a 30x points at another HTTPS host, `createPinnedLookup` falls back to ordinary DNS for that host, so an allowed URL can redirect to a private/internal address and bypass the blocked-host/address checks. Set redirects to `error`/`manual`, or validate and repin each redirect target before downloading.
  - confidence: 0.9
