## Summary

Keep open. Current main still lacks ACP skill-context injection: `sessions_spawn` has no skill selector for ACP, the ACP spawn parameter and runtime turn contracts carry only the task text, and the native skill snapshot path is separate from the ACP early-return path. The earlier ClawSweeper review remains consistent with current main.

## What This Changes

Allow `sessions_spawn` with `runtime: "acp"` to inject selected, automatic, or default OpenClaw skill context into the spawned ACP session prompt.

## Best Possible Solution

Add a documented ACP skill-context capability after policy is settled: start with a typed explicit selector, reuse existing skill snapshot filtering and prompt limits, pass context through a structured ACP seam or tightly bounded prompt-prefix fallback, and add tests/docs for explicit lists, missing skills, truncation, isolation, and host-runtime safety. Treat automatic relevance and default injection as follow-up policy work unless maintainers choose them up front.
