## Summary

Keep open. Current main still does not implement ACP skill-context injection: the ACP `sessions_spawn` schema has no skill selector, ACP spawn params have no skill/context field, the dispatch path forwards only the raw task, and the normal skill snapshot prompt path is bypassed for ACP turns. The prior ClawSweeper review remains accurate on current main; the checked compare from that reviewed SHA to current main shows no changes in the relevant ACP, skills, or docs files.

## What This Changes

Request to let `sessions_spawn` with `runtime: "acp"` inject selected, automatic, or default OpenClaw skill context into the spawned external harness session.

## Best Possible Solution

Add a documented ACP skill-context capability only after policy is chosen: start with a typed explicit selector, reuse `buildWorkspaceSkillSnapshot` with existing agent skill filters and prompt limits, pass context through a structured ACP seam or tightly bounded task-prefix fallback, and add tests/docs for explicit lists, missing skills, truncation, isolation, and host-runtime safety. Treat automatic relevance and default skill injection as separate follow-up policy choices unless maintainers explicitly want them in the first API.
