## Summary

Keep open. Current main still does not implement ACP skill-context injection: the public `sessions_spawn` schema has no skill selector, ACP spawn parameters carry only task/session controls, and the ACP dispatch/runtime path forwards text without a structured skill snapshot or rendered skills prompt.

## What This Changes

Request ACP sessions spawned with `sessions_spawn(runtime: "acp")` support injecting selected, automatic, or default rendered OpenClaw skill context into the initial external-harness prompt.

## Best Possible Solution

Add a documented ACP skill-context capability after maintainer policy is settled: start with a typed explicit selector, reuse the existing skill snapshot filtering and prompt limits, carry the context through a bounded ACP seam or tightly scoped prompt-prefix fallback, and cover missing skills, truncation, isolation, and host-runtime safety in tests and docs.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The current-main static check is high-confidence: `sessions_spawn` does not advertise or read a skill selector for ACP, and ACP dispatch forwards only `params.task`.

## Solution Assessment

Unclear for the full proposal. Explicit skill selection looks like the narrowest maintainable first step, while automatic relevance and default injection need maintainer product and safety decisions before they can be called the best solution.

## Security Review

Status: needs_attention

Summary: The requested feature is security-sensitive because it would move local skill context across the ACP host-runtime boundary into external harness prompts.

Concerns:

- **[medium] Constrain skill context crossing the ACP boundary:** `src/agents/acp-spawn.ts:1387`
  - body: ACP spawns currently send only task text to the Gateway `agent` call. Any implementation that injects skill context should preserve skill allowlists, prompt budgets, path exposure expectations, and secret/config isolation before sending local skill content to the external harness.
  - confidence: 0.86
- **[medium] Preserve ACP skill-secret isolation:** `src/acp/client.ts:135`
  - body: The ACP client already strips active skill env keys from spawned ACP server environments. Adding prompt-level skill injection should not reintroduce secret values or sensitive local configuration through rendered skill context.
  - confidence: 0.8

## Risks / Open Questions

- Prompt-level skill injection into external ACP harnesses needs policy for allowlists, prompt budget, truncation, local path exposure, and secret/config isolation.
- The current ACP runtime contract has no structured context field, so a durable implementation may need an additive ACP seam rather than only concatenating text into `task`.
