## Summary

Keep open. Current main still does not implement ACP skill-context injection: the public spawn schema has no skill selector, ACP spawn/runtime types carry no skill prompt field, and ACP dispatch forwards only the task text. The existing native sub-agent `context: "fork"` capability is explicitly not available for ACP, so it does not satisfy this request.

## What This Changes

Request adding optional rendered OpenClaw skill context to `sessions_spawn` calls that use `runtime: "acp"`, with explicit, automatic, or default skill selection.

## Best Possible Solution

Add a documented ACP skill-context capability after maintainer policy is settled: start with an explicit typed selector, reuse existing skill filtering and prompt-budget logic, pass rendered context through a bounded ACP seam or tightly scoped prompt prefix, and cover missing skills, truncation, isolation, secret handling, and docs.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The current-main static check is high-confidence because the spawn schema, ACP params, dispatch call, and ACP runtime turn input all lack a skill-context path.

## Solution Assessment

Unclear for the full proposal. Explicit skill selection looks like the narrowest maintainable first step, while automatic relevance and default injection need maintainer product and safety decisions before implementation.

## Security Review

Status: needs_attention

Summary: The requested feature is security-sensitive because it would move local skill context across the ACP host-runtime boundary into external harness prompts.

Concerns:

- **[medium] Constrain skill context crossing the ACP boundary:** `src/agents/acp-spawn.ts:1387`
  - body: ACP spawns currently send only task text to the Gateway `agent` call. Any implementation that injects skill context should preserve skill allowlists, prompt budgets, path exposure expectations, and local configuration isolation before sending rendered skill content to an external harness.
  - confidence: 0.86
- **[medium] Preserve ACP skill-secret isolation:** `src/acp/client.ts:135`
  - body: The ACP client strips active skill env keys from spawned ACP server environments. Prompt-level skill injection should not reintroduce secret values or sensitive local configuration through rendered skill context.
  - confidence: 0.8
