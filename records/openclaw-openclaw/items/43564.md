## Summary

Keep open. Current main still lacks ACP skill-context injection: the `sessions_spawn` schema has no skill selector, the ACP spawn parameters and runtime turn contracts carry only task text/session controls, and the existing skill snapshot prompt path is separate from the ACP dispatch path.

## What This Changes

Request to let `sessions_spawn` ACP calls include selected, default, or automatically chosen OpenClaw skill context in the spawned ACP harness prompt.

## Best Possible Solution

Add a documented ACP skill-context capability after policy is settled: start with a typed explicit selector, reuse the existing skill snapshot filtering and prompt limits, pass context through a structured ACP seam or tightly bounded prompt-prefix fallback, and cover explicit lists, missing skills, truncation, isolation, and host-runtime safety in tests and docs.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The current-main static check is high-confidence: `sessions_spawn` does not advertise or read a skill selector for ACP, and ACP dispatch forwards only `params.task`.

## Solution Assessment

Unclear for the full proposal. Explicit skill selection looks like the narrowest maintainable first step, while automatic relevance and default injection need maintainer policy decisions before they can be called the best solution.

## Security Review

Status: needs_attention

Summary: The requested feature is security-sensitive because it would move local skill context across the ACP host-runtime boundary into external harness prompts.

Concerns:

- **[medium] Constrain skill context crossing the ACP boundary:** `src/agents/acp-spawn.ts:1387`
  - body: ACP spawns currently send only the task text to the Gateway `agent` call. Any implementation that injects skill context should preserve skill allowlists, prompt budgets, path exposure expectations, and secret/env isolation before sending local skill content to the external harness.
  - confidence: 0.84
- **[medium] Preserve ACP skill-secret isolation:** `src/acp/client.ts:135`
  - body: The ACP client already strips active skill env keys from spawned ACP server environments; adding prompt-level skill injection should not reintroduce secret values or sensitive local configuration through rendered skill context.
  - confidence: 0.78
