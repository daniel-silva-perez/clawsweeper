## Summary

Keep this PR open. Current main still lacks the proposed `agents.defaults.systemPromptFile` config/runtime path, and the PR remains a relevant implementation candidate, but it is not ready to merge because the public generated config/docs surface is incomplete and the prompt-file immutability boundary needs maintainer-approved semantics.

## What This Changes

The PR adds an optional `agents.defaults.systemPromptFile` config field, reads the configured local file into CLI and embedded system prompt composition, and adds a helper unit test.

## Best Possible Solution

Land a tightened version after maintainer review: define the config contract and file trust model, rebase onto the current prompt override and extra-prompt flow, update generated schema/help/labels/docs, and keep focused coverage for CLI and embedded prompt composition.

## Reproduction Assessment

Yes for the feature gap: current main has no `agents.defaults.systemPromptFile` schema/type/runtime path, so config cannot drive file-based prompt injection. The PR behavior itself would be verified with a configured prompt file through CLI and embedded sessions, but this sweep stayed read-only.

## Solution Assessment

No, not in its current form. File-based global prompt injection may be the right product direction, but the patch needs public config-surface regeneration/docs and an explicit immutable-file deployment boundary before it is the narrow maintainable solution.
