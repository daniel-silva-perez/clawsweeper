## Summary

Keep this PR open. Current main still does not implement `agents.defaults.systemPromptFile`: the config type/schema omit the key, both CLI and embedded prompt assembly only pass through existing `extraSystemPrompt`, and current `systemPromptFile*` hits are CLI-backend plumbing for writing OpenClaw's already assembled prompt to a temporary file. The PR is still a focused implementation candidate for the closed companion feature request #36190, not something already shipped on main.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a tightened version that defines read timing and path-resolution semantics, keeps the file path outside agent-tunable config mutation surfaces, updates docs/generated schema/labels, and covers both CLI and embedded runner prompt composition with targeted tests. If maintainers decide this should not be core `agents.defaults` behavior, they should explicitly redirect the design to a plugin/API hook rather than closing it as already implemented.
