## Summary

Keep this PR open. Current main still lacks a global file-backed prompt default, and the branch is a useful implementation candidate, but it is not mergeable as-is because the public config surface is incomplete, the prompt-file trust model is unresolved, and CLI session reuse can miss prompt-file changes.

## What This Changes

The PR adds an optional `agents.defaults.systemPromptFile` config key, reads the configured local file into CLI and embedded system prompt construction, and adds a helper unit test.

## Best Possible Solution

Land a tightened version only after maintainer review: define the global prompt config API, enforce or clearly validate/document the immutable outside-workspace file boundary, include prompt-file content in CLI session reuse invalidation, regenerate schema/help/docs/baselines, and keep focused CLI plus embedded prompt coverage. If #50103 becomes the canonical direction, fold this file-based use case into that design instead of merging a narrower incompatible surface.

## Reproduction Assessment

Yes. Current main can be checked statically: there is no `agents.defaults.systemPromptFile` schema/type/runtime path, and the PR's remaining defects are visible from the diff plus current CLI session reuse and generated-schema code paths.

## Solution Assessment

No, not in its current form. The proposed hook point is plausible, but the patch is missing the public config generation/docs work and needs a defined trust boundary plus CLI session reuse behavior before it is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Regenerate and document the config schema surface:** `src/config/zod-schema.agent-defaults.ts:41`
  - body: Adding this public key only to Zod/types leaves the served config schema, field labels/help, Control UI metadata, docs, and config-doc baselines stale. `buildBaseConfigSchema()` serves the committed generated schema, so `openclaw config schema` will not expose the new key until those generated/docs surfaces are updated.
  - confidence: 0.91
- **[P2] Define the prompt-file trust boundary:** `src/agents/system-prompt-file.ts:14-16`
  - body: The helper rereads the configured path on each session while the feature is positioned as operator-unmodifiable guidance. If the configured path is workspace-relative or agent-writable, an agent can affect future prompts, so the patch should enforce, snapshot, or document and validate the required outside-workspace deployment model.
  - confidence: 0.84
- **[P2] Include prompt-file content in CLI session hashing:** `src/agents/cli-runner/helpers.ts:78-82`
  - body: The file content is appended inside `buildSystemPrompt()`, after `prepare.ts` has already computed `extraSystemPromptHash`. Reusable CLI sessions therefore are not invalidated when this config is added or the file changes, and `execute.ts` skips resending the system prompt on resume, so some CLI sessions can keep running without the new operator prompt.
  - confidence: 0.83
