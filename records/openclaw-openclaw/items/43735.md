## Summary

Keep open. Current main does load workspace skills and has prompt filtering/limit behavior that can explain some partial lists, but the restart-safe stale snapshot path remains: persisted snapshots are still validated only against process-local version counters and the agent/gateway paths can reuse an old version-0 snapshot after a restart. The related proposed fix in #48089 was closed unmerged, and the open #74161 only tightens prompt path usage rather than rebuilding stale inventories.

## What This Changes

The issue requests that valid workspace skills be discovered and reflected in the agent `<available_skills>` context instead of leaving agents with a stale or partial skills list.

## Best Possible Solution

Add a durable, deterministic skill-inventory fingerprint to skill snapshots and compare it in both CLI and gateway session paths, keeping the current version watcher as a fast path while making restarts self-heal when workspace, managed, personal, extra-dir, plugin, filter, eligibility, or prompt-limit inputs change.

## Repair Work Prompt

Fix the stale skillsSnapshot restart path for this issue. Current main persists session skills snapshots with only a process-local version counter; after a restart, a stored version-0 snapshot can compare fresh against a new process whose in-memory version is also 0. Implement restart-safe invalidation by persisting and comparing a deterministic skill inventory fingerprint or equivalent durable freshness key. The fingerprint should cover the visible skill roots and relevant inputs that affect prompt-visible skills, including workspace/project/personal/managed/extra/plugin skill inventories, SKILL.md metadata used for gating/exposure, agent skill filter, eligibility/config inputs, and prompt limits as appropriate. Keep the existing watcher/version path as a fast path, and do not promise every directory always appears in `<available_skills>` when filters, invalid frontmatter, dependency gating, `disable-model-invocation`, or prompt budgets exclude it. Update both the CLI agent path and gateway session path, add focused regression coverage for an existing persisted version-0 snapshot surviving a process restart after a skill is added while no watcher is active, and update docs or changelog only if user-visible behavior wording changes. If adding a changelog entry, use a single-line active-version Fixes entry with appropriate reporter attribution such as Thanks @che1404, and do not change unrelated skill install target behavior from #74124 or prompt path wording from #74161.
