## Summary

Keep open. Current main still relies on process-local skills snapshot versions plus agent skill filters, and there is no durable inventory fingerprint on persisted `skillsSnapshot` data to prove the effective skill catalog changed across restarts. The related stale-restart PR is closed unmerged; draft maintainer PR #71497 targets the version-0 restart root cause, but it is still open and does not explicitly close this issue, while #74161 only tightens prompt path wording and does not rebuild stale inventories.

## What This Changes

The issue asks for valid workspace skills under `~/.openclaw/workspace/skills/` to appear consistently in agent `<available_skills>` context instead of disappearing from partial or stale session skill snapshots.

## Best Possible Solution

Keep this issue open until #71497 or an equivalent fix lands and proves that CLI, gateway, and cron skill snapshots refresh after restart when the effective workspace skill inventory changed. If maintainers want the stronger contract described here, extend that work with a deterministic inventory fingerprint stored on `SessionSkillSnapshot` while preserving precedence, realpath containment, allowlists, eligibility gates, and prompt limits.
