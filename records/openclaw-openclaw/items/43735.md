## Summary

Keep open. Current main still has a restart-stale skill snapshot path: workspace skills are loaded and watched, but persisted session snapshots are refreshed only by process-local version counters and agent skill filters, so a version-0 snapshot can survive a restart without being compared to the current skill inventory. The related stale-snapshot fix PR was closed unmerged, and the open prompt-path PR is adjacent rather than a fix for rebuilding stale inventories.

## What This Changes

The issue requests reliable discovery of valid workspace skills in the agent `<available_skills>` context instead of showing only a partial or stale subset.

## Best Possible Solution

Add a deterministic skill-inventory fingerprint to skill snapshots and compare it in the CLI and gateway session paths so restarts self-heal when workspace, project-agent, personal, managed, extra-dir, plugin, filter, eligibility, or prompt-limit inputs change, while preserving existing gating and precedence behavior.

## Repair Work Prompt

Fix the stale skill snapshot bug tracked by https://github.com/openclaw/openclaw/issues/43735. Current main loads workspace skills, but persisted session snapshots are refreshed only by process-local version counters and skill filters, so after a restart a version-0 snapshot can be reused even when the skill inventory changed. Add a durable, deterministic inventory fingerprint/hash to `SkillSnapshot` and `SessionSkillSnapshot`, compute it from the effective skill inventory inputs, and compare it in both the CLI agent path and gateway `ensureSkillSnapshot` path before reusing a persisted snapshot. Preserve existing source precedence, symlink containment, invalid frontmatter handling, agent allowlists, eligibility gates, remote-node behavior, compact prompt limits, and watcher fast-path behavior. Treat #48089 as useful prior art but do not use a restart timestamp alone as the fix, because that rebuilds every restart instead of detecting actual inventory drift. #74161 is adjacent prompt wording work and should not be treated as solving this bug. Add focused tests for restart/version-0 stale snapshots in CLI and gateway paths plus any helper/fingerprint tests needed.
