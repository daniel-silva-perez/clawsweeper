## Summary

Keep open. Current main still has a restart-stale skill snapshot path: workspace skills are documented and loaded, but persisted session snapshots are refreshed from process-local version counters plus agent skill filters, with no durable inventory fingerprint. The direct stale-restart fix PR is closed unmerged, and the open prompt-path PR is adjacent rather than a fix for rebuilding stale inventories.

## What This Changes

The issue asks for valid workspace skills under the OpenClaw workspace skills directory to appear consistently in `<available_skills>` instead of disappearing from stale or partial agent context snapshots.

## Best Possible Solution

Add a deterministic skill-inventory fingerprint to session skill snapshots and compare it in the CLI, gateway, and cron session paths so restarts self-heal when skill roots, files, plugin skills, config filters, eligibility inputs, or prompt-limit-relevant inventory changes. Preserve current source precedence, realpath safety checks, agent allowlists, eligibility gates, and prompt limits.
