## Summary

Keep open. Current main still treats workspace skills as an in-repo skills/snapshot surface, and the gateway/CLI refresh path can still keep a persisted version-0 skills snapshot after restart when no in-process version bump has happened. The related maintainer PR is still draft/open and does not explicitly close this issue, so closing before that work lands would be premature.

## What This Changes

The issue requests that valid workspace skills under `~/.openclaw/workspace/skills/` all load into the agent’s `<available_skills>` context instead of only a partial subset appearing.

## Best Possible Solution

Keep this open until #71497 or an equivalent fix lands and proves that gateway, CLI, and cron skill snapshots rebuild after restart when the effective workspace skill inventory changes. If maintainers want the stronger contract requested here, store a deterministic inventory fingerprint on `SessionSkillSnapshot` while preserving source precedence, realpath containment, allowlists, eligibility gates, and prompt-size limits.
