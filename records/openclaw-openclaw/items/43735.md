## Summary

Keep open. Current main loads and documents workspace skills, but the reported stale persisted skillsSnapshot path is still plausible: CLI and Gateway session reuse compare cached snapshots against process-local version counters, and a restart can leave both cached and current versions at 0 with no inventory fingerprint or filesystem comparison. Related PR #48089 is closed unmerged, and #44898 is closed rather than an open canonical fix.

## Best Possible Solution

Keep this open until maintainers add restart-safe invalidation for persisted CLI/Gateway skillsSnapshot data or explicitly narrow the documented contract. A robust fix would persist and compare a deterministic inventory fingerprint covering visible skill roots, loaded skill metadata, relevant config/filter/eligibility inputs, and prompt limits, with regression coverage for an existing version-0 session snapshot after a process restart and an offline workspace skill change.
