## Summary

Keep open. Current main has a generic `openclaw pairing approve --notify` path, but WhatsApp still registers only a pairing ID label and no approval notifier, so the approved WhatsApp sender is not told they can start using the bot.

## What This Changes

Request WhatsApp pairing approvals to notify the approved sender after owner approval, with optional default/configurable notification and original-message replay behavior.

## Best Possible Solution

Wire WhatsApp into the existing pairing notification contract by adding a WhatsApp text approval adapter that sends `PAIRING_APPROVED_MESSAGE` through the active WhatsApp listener, and pass the approved request or selected account scope through the generic notify path.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: `openclaw pairing approve whatsapp <code> --notify` reaches `notifyPairingApproved`, but the WhatsApp pairing adapter lacks `notifyApproval`, so the function returns without sending anything.

## Solution Assessment

Yes for the narrow repair. Reusing the existing `--notify` pairing adapter path is the smallest maintainable solution; changing defaults or replaying blocked messages should remain separate product work.
