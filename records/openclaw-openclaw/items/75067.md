## Summary

Keep open: the current PR head still has concrete correctness regressions around disk-cache precedence and retry scheduling, and current main does not already implement the intended resilience. The direction is useful, but this needs author and Gateway maintainer revision before merge; the existing ClawSweeper review remains mostly applicable after the force-push.

## What This Changes

The PR lowers the Gateway model-pricing fetch timeout from 60s to 10s, adds disk-backed pricing cache warming and persistence plus an in-memory circuit breaker, adds `startupBlockedReason` to startup-pending readiness responses, and updates related Gateway tests.

## Best Possible Solution

Keep the resilience work, but revise it so disk warming overlays current configured and manifest pricing, retry timers respect both cache TTL remaining time and circuit-breaker cooldown, and the branch is coordinated with #73486 so optional pricing refresh cannot stay on the startup-critical path.

## Reproduction Assessment

Yes for the review findings. The PR head shows the fresh-disk-cache return before config seeding and both problematic paths calling the fixed 24h scheduler; focused fake-timer/unit coverage around fresh cache, changed configured cost, and open-circuit cooldown should reproduce them.

## Solution Assessment

No. The proposed architecture is reasonable, but this implementation is not the narrowest maintainable fix until it preserves current pricing precedence and schedules retries based on the actual TTL or breaker cooldown rather than the generic daily refresh helper.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Apply configured pricing before disk cache warm return:** `src/gateway/model-pricing-cache.ts:1182-1185`
  - body: This branch returns as soon as any fresh disk cache exists, before `collectSeededPricing` reads current `models.providers.*.models[].cost`. If a user changes local pricing while the cache is fresh, stale disk data wins for up to the cache window. Merge or overlay current configured and manifest pricing before returning.
  - confidence: 0.91
- **[P2] Retry when the circuit cooldown expires:** `src/gateway/model-pricing-cache.ts:1232`
  - body: The open-circuit path logs the remaining cooldown but schedules via `scheduleRefresh`, which waits the 24h cache TTL. After three failures, a recovered pricing service can be ignored until the daily timer instead of being retried after the 10-minute breaker cooldown.
  - confidence: 0.9
- **[P3] Schedule disk refresh for remaining TTL:** `src/gateway/model-pricing-cache.ts:1184`
  - body: A fresh disk cache always schedules a full 24h delay from process start. A 23-hour-old cache can therefore remain in use for almost another day, and repeated restarts can keep postponing live refreshes. Schedule from the loaded cache's remaining TTL instead.
  - confidence: 0.84
