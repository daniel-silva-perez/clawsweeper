## Summary

Keep this PR open for normal maintainer and author review. Current main still has the 60s Gateway pricing fetch path and lacks the disk cache/circuit breaker work, so the PR is not obsolete; however the proposed patch introduces two blocking P2 correctness issues and has an explicit sequencing dependency on the still-open pricing-startup deferral PR.

## What This Changes

The branch lowers the Gateway pricing fetch timeout to 10s, adds a disk-backed pricing cache and circuit breaker, adds `startupBlockedReason` to startup-pending readiness responses, and updates Gateway tests.

## Best Possible Solution

Keep the resilience direction, but revise the PR so disk-cache warming overlays current config and manifest pricing before returning, circuit-open refreshes retry at the cooldown boundary, and the branch is coordinated with #73486 so pricing refresh cannot stay on the startup-critical path.

## Reproduction Assessment

Yes for the review findings: the PR head shows the fresh-cache return before `collectSeededPricing`, and the circuit-open branch calls the 24h `scheduleRefresh` helper. Focused tests with a fresh cache fixture and fake timers would reproduce both; tests were not executed because this sweep was read-only.

## Solution Assessment

No. The proposed direction is useful, but this implementation is not the best fix until it preserves current local pricing precedence and makes the circuit breaker retry on the advertised cooldown rather than the daily cache TTL.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Apply config pricing before disk-cache short-circuit:** `src/gateway/model-pricing-cache.ts:1157-1159`
  - body: The fresh-cache path returns before manifest/config seeding runs. If a user changes `models.providers.*.models[].cost` while the disk cache is fresh, startup keeps stale cached pricing until the TTL expires even though local config is supposed to drive cost estimates. Overlay current configured pricing before returning from the disk warm path.
  - confidence: 0.91
- **[P2] Schedule retries at the circuit cooldown:** `src/gateway/model-pricing-cache.ts:1199-1200`
  - body: This branch logs a remaining 10-minute pause but calls `scheduleRefresh`, which always waits the 24h cache TTL. After the breaker opens, a recovered pricing service can be ignored until the daily timer instead of the cooldown; schedule the next attempt for `circuitOpenUntil`.
  - confidence: 0.89
