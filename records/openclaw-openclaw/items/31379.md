## Summary

Current main still has the reported Control UI behavior: `resolveChannelOrder()` treats `channelMeta` or `channelOrder` as the complete render list, while `channels.status` builds those fields only from loaded channel plugins. A custom-only channel snapshot can still hide unloaded built-in channel cards. PR #44165 is an open focused implementation candidate with closing syntax for this issue, so this issue should remain open until that PR or an equivalent fix merges.

## Best Possible Solution

Keep this issue open until PR #44165 or an equivalent focused Control UI fix merges. The fix should update `ui/src/ui/views/channels.ts` so snapshot-provided custom/loaded IDs are merged with the built-in channel order, deduped while preserving snapshot order first, and still promoted by enabled state. Add regression coverage in `ui/src/ui/views/channels.test.ts` for custom-only `channelMeta`, custom-only `channelOrder`, and no-snapshot fallback. Backend `channels.status` behavior can remain unchanged unless maintainers prefer a catalog-backed built-in order.
