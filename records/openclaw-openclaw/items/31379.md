## Summary

Keep open. Current main at ab5c802 still renders exactly the ids returned by channelMeta or channelOrder and the backend still derives those fields from loaded channel plugins only, so a custom-only channel snapshot can still hide unconfigured built-in channel cards. The focused fix PR #44165 is open and directly tracks this issue; the earlier PR #31350 was closed unmerged.

## Best Possible Solution

Keep this issue open until PR #44165 or an equivalent focused Control UI fix merges. The fix should update ui/src/ui/views/channels.ts to merge snapshot-provided custom/loaded ids with the built-in channel order, dedupe while preserving snapshot/custom order first, retain enabled-channel promotion, and add regression coverage in ui/src/ui/views/channels.test.ts for custom-only channelMeta, custom-only channelOrder, and no-snapshot fallback. Backend channels.status behavior and plugin loading can remain unchanged unless maintainers prefer a shared catalog-backed built-in order.
