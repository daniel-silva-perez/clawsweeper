## Summary

Current main still has the reported Channels settings behavior: the Control UI renders exactly the ids from `channelMeta` or `channelOrder`, while `channels.status` still derives those fields from loaded channel plugins only. The focused fix PR #44165 is still open and directly targets this issue; the earlier PR #31350 was closed unmerged, so this issue should remain open until a fix lands.

## Best Possible Solution

Keep this issue open until PR #44165 or an equivalent focused Control UI fix merges. The fix should update `ui/src/ui/views/channels.ts` to merge snapshot-provided custom/loaded ids with the built-in channel order, dedupe while preserving snapshot/custom order first, retain enabled-channel promotion, and add regression coverage in `ui/src/ui/views/channels.test.ts` for custom-only `channelMeta`, custom-only `channelOrder`, and no-snapshot fallback.

There is no merge or append step for built-in channel ids when custom metadata exists.
