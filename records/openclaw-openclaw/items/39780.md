## Summary

Keep open. Current main still has the reported Control UI gap: the Overview insecure-HTTP recommendation renders manual text for `gateway.controlUi.allowInsecureAuth: true`, while the existing diff and base-hash guarded config write flow remains scoped to Config and Quick Settings rather than recommendation callouts.

## Best Possible Solution

Keep this issue open until Control UI recommendation callouts can stage narrow base-hash guarded config changes, show before/after values, offer accept/dismiss controls, apply through the existing config write path, and confirm the saved value. The likely path is a shared recommendation/apply component reused by the insecure HTTP Overview hint and future config-hardening suggestions, with focused UI/controller tests and docs updates.
