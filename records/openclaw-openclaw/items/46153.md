Keep open. Current main still lacks daemon-safe stdin handling in the macOS LaunchAgent plist path, and the linked broader report was closed in favor of this focused startup-hang issue rather than superseding it. The remaining work is narrow and code-local.

The issue requests that the macOS Gateway LaunchAgent startup path avoid a silent launchd-only hang, with the reported fix being explicit `/dev/null` stdin handling for generated service plists.
