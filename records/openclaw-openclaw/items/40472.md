## Summary

Keep this PR open. Current main has the gateway-side `sessions.changed` reset event and subscription method, but the TUI still does not subscribe to that event, dispatch it, or clear same-session stale run/stream state after an external `/new` or `/reset`. The PR remains the active implementation candidate for #38966 and recently had maintainer repair work to preserve the contributor path.

## Best Possible Solution

Keep this PR open and review/land it after any needed rebase, or land an equivalent patch on main. The fix should make the TUI subscribe to `sessions.changed` on connect/reconnect, expose the backend subscription hook, handle active-session `new`/`reset` events, clear stale run/stream/BTW/local-run state, suppress late old-run events where feasible, reload history/session info, and keep the targeted TUI regression tests and changelog entry. After that ships, close #38966 as implemented.
