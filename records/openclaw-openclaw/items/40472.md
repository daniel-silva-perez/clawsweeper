## Summary

Keep this PR open. Current main has the gateway-side `sessions.changed` event and subscription RPCs, but the TUI still does not subscribe to session lifecycle events, dispatch `sessions.changed`, or clear same-session stale run/stream state after an external `/new` or `/reset`. The provided discussion also shows this PR is the active implementation candidate for #38966 and had recent maintainer repair work to preserve the contributor path.

## Best Possible Solution

Keep this PR open and review/land it after any needed rebase or land an equivalent patch on main. The fix should add the TUI backend `sessions.subscribe` wrapper, subscribe on connect/reconnect, dispatch `sessions.changed`, handle active-session `new`/`reset` events by clearing stale run/stream/local BTW state, reload history/session info, keep targeted regression tests, and then close #38966 once the implementation ships.
