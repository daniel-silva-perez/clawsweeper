## Summary

Keep this PR open. Current main has the gateway-side `sessions.changed` reset event and advertised subscription contract, but the TUI still does not subscribe to, dispatch, or handle that event, so the user-visible stale TUI state bug from #38966 remains unresolved without this PR or an equivalent patch.

## Best Possible Solution

Review and land this PR, or an equivalent TUI-side implementation, so the TUI subscribes to `sessions.changed` on connection/reconnection, handles active-session `new`/`reset` events, clears stale run/stream/BTW/local-run state, and reloads history/session info. After that lands, #38966 can close as implemented.
