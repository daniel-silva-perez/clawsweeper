## Summary

Keep this PR open. Current main has the gateway-side subscriber-scoped `sessions.changed` event, but the TUI still does not subscribe to it, dispatch it, or clear stale run/stream state when the active session is reset externally. The PR remains the active implementation candidate for #38966, with a recent maintainer repair and merge-ready labeling in the provided discussion.

## Best Possible Solution

Review and land the repaired branch if its current diff and checks are acceptable: add a TUI backend wrapper for `sessions.subscribe`, subscribe on Gateway connection/reconnection, dispatch `sessions.changed`, and for active-session `new`/`reset` events clear stale run, stream, watchdog, BTW, and local-run state before reloading history/session info. After it merges, #38966 can close as implemented.
