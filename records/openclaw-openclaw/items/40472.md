## Summary

Keep this PR open. Current main still has the gateway-side `sessions.changed` reset event contract, but the TUI does not subscribe to it, dispatch it, or clear same-session stale run state after an external `/new` or `/reset`. The PR remains the active implementation candidate for the linked #38966 flow, and maintainer discussion shows the branch was recently repaired to preserve the contributor path.

## Best Possible Solution

Review and land this PR, or an equivalent TUI-side patch, so the TUI subscribes to `sessions.changed` on connection/reconnection, dispatches active-session `new`/`reset` events, clears stale active run, stream assembler, local run, and BTW state, suppresses late old-run events where feasible, and reloads history/session info. After that ships, close #38966 as implemented.
