## Summary

Keep this PR open. Current main does not contain the proposed visionOS node app or docs, and the related visionOS platform-policy PR is still open. The PR remains a plausible implementation candidate, but it is an XL native-app/security-sensitive change with unresolved protocol-alignment review findings against current gateway contracts.

## Best Possible Solution

Keep this PR open for maintainer review. If OpenClaw wants a first-party visionOS node app, the best path is to resolve the paired platform-policy work in #55944, align the app with current node protocol contracts, address the remaining native security/reconnect findings, add the missing app/docs/labeler surfaces, and require real Apple Vision Pro validation. If maintainers decide visionOS should live outside core, that should be an explicit product/plugin-boundary decision rather than automated cleanup.
