## Summary

Keep this PR open. Current main still does not implement the Mattermost message-tool thread inheritance that this PR targets, and the paired bug report #45082 remains open. The PR is abandoned by its author, but it remains a plausible implementation candidate for maintainer triage rather than something to close as implemented or obsolete.

## Best Possible Solution

Keep this PR open for maintainer triage or contributor pickup. The best path is to adapt the Mattermost-owned fix to current main: add a Mattermost `threading.buildToolContext` that supplies normalized current channel id, active thread root, account-scoped `replyToMode`, and `hasRepliedRef`; add `resolveAutoThreadId` if mirror/session routing needs the implicit thread before send; preserve explicit reply precedence; and add focused regression tests for all/first/off, explicit reply targets, same/different-channel guards, and first-reply state.
