## Summary

Keep this PR open. Current main still does not implement the intended Mattermost message-tool thread inheritance: Mattermost does not build channel-specific tool threading context, and its send action only uses explicit reply targets. The author has abandoned the PR, but the linked Mattermost bug remains open and this is still a useful implementation candidate rather than conservative cleanup close.

## Best Possible Solution

Keep this PR open for maintainer triage or contributor pickup. The best implementation path is to adapt the Mattermost-owned fix to current main: add a Mattermost threading.buildToolContext that exposes normalized current channel id, active thread root, replyToMode, and hasRepliedRef; add resolveAutoThreadId if delivery/mirror routing needs a thread id before send; make same-channel implicit message-tool sends preserve the Mattermost root_id; and add focused regression tests for all/first/off, explicit replyTo precedence, same/different-channel guards, and first-reply state. If maintainers decide bundled Mattermost is no longer supported, close this together with #45082 and related Mattermost PRs after that explicit product decision.
