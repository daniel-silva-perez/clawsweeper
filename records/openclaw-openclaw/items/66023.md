## Summary

Keep this PR open. Current main still lacks custom cron job IDs end-to-end: the CLI has no `cron add --id`, the gateway add schema rejects unknown `id`, `CronJobCreate` omits `id`, and the service still generates UUIDs unconditionally. The PR remains a concrete implementation candidate for #65636, but it should resolve the case-variant run-log collision concern before landing; #65669 is closed unmerged and does not supersede it.

## Best Possible Solution

Keep this PR open as the implementation lane for #65636. Before merge, revise it to canonicalize custom cron job ids or reject case-insensitive duplicates before persistence/run-log use, keep the CLI/gateway/service/normalization tests, add run-log collision coverage, and update user-facing docs/changelog if maintainers accept the feature.
