## Summary

Keep open. Current main still lacks an OpenRouter provider thinking profile, while this PR makes a narrow provider-owned change that matches the linked bug and the existing plugin contract; it should receive normal maintainer review and focused validation rather than cleanup closure.

## What This Changes

The PR adds an OpenRouter `resolveThinkingProfile` hook that exposes `off`, `low`, `medium`, `high`, and `xhigh` for reasoning-capable OpenRouter models while excluding the known unsupported Hunter Alpha proxy route.

## Best Possible Solution

Land a provider-owned OpenRouter thinking profile with targeted provider tests for a reasoning DeepSeek V4 model, a non-reasoning OpenRouter model, and the Hunter Alpha unsupported proxy route, keeping the change inside the OpenRouter plugin boundary.

## Reproduction Assessment

Yes. Current-main code inspection gives a high-confidence reproduction path: select `openrouter/deepseek/deepseek-v4-pro`, send `/think xhigh`, and the shared thinking resolver has no OpenRouter provider profile that can advertise `xhigh`, so directive validation rejects the level before the existing transport wrapper can send `reasoning.effort`.

## Solution Assessment

Yes. `resolveThinkingProfile` is the narrowest maintainable seam for provider-specific level sets; the safer completion is to add focused tests rather than move OpenRouter-specific policy into core.
