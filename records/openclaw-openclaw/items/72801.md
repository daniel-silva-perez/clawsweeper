## Summary

Keep open. Current main still generates an empty isolated Codex config for ACPX, so the PR is not obsolete, but the proposed parser misses supported Codex project-trust shapes and the branch is draft/dirty against current main.

## What This Changes

The PR copies trusted Codex project declarations, plus the ACPX workspace cwd trust entry, into ACPX’s isolated Codex home while adding auth bridge tests that ensure auth and notify hooks are not copied.

## Best Possible Solution

Keep the fix owner-local in `extensions/acpx`, rebase it onto current main, and parse Codex config with a TOML-aware path or equivalent coverage for explicit tables, `[projects]` inline entries, and top-level inline `projects` maps while continuing to exclude auth and notify hooks.

## Reproduction Assessment

Yes. Current main can be verified by inspecting `prepareIsolatedCodexHome`: it never reads the source Codex config, so a trusted project in `CODEX_HOME/config.toml` is not propagated to the isolated ACPX Codex home.

## Solution Assessment

No, not as written. The direction is narrow and maintainable, but the ad hoc parser is incomplete for Codex’s supported TOML project-trust forms; a TOML-aware parser or added inline-project support would be safer.
