## Summary

Keep open. Current `main` at `e583db63c602d8d21b4ebdcd119061571a2e40a9` still has unbounded `memory_search` manager setup and search awaits, existing memory tool tests cover rejected errors but not never-settling promises, and open PR #49794 still explicitly targets this issue with closing syntax.

## Best Possible Solution

Review, update, and land PR #49794 or replace it with an equivalent fix in the memory plugin: wrap `loadMemoryToolRuntime`, `getMemoryManagerContext`, QMD manager lookup/creation, and `memory.manager.search` in a hard deadline that always returns the existing unavailable payload, then add fake-timer regression coverage for never-settling manager-init and search promises. Close this issue only after that lands on `main`.
