## Summary

Keep open. Current main still lacks a hard timeout around the full `memory_search` manager-init/search path, and the supplied GitHub context shows an open implementation PR with closing syntax for this issue, so this issue should remain open until that PR, or an equivalent fix, lands.

## Best Possible Solution

Review, update, and land PR #49794 or implement an equivalent fix in `extensions/memory-core/src/tools.ts`/manager boundaries: wrap runtime load, manager lookup, and `manager.search` in a hard deadline that always returns the existing `{ disabled: true, unavailable: true }` result, then add regression coverage for never-settling manager init and search promises. Close this issue only after that lands on `main`.
