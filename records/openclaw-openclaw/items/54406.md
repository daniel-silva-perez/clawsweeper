## Summary

Keep open. Current main still lacks the richer compaction event observability this PR is trying to add, but the submitted branch is stale against the current `compaction_start` / `compaction_end` path and still misses part of its promised end-event contract.

## What This Changes

The PR branch adds embedded compaction start/end event metadata, structured compaction lifecycle logging, dispatcher forwarding for start details, and a focused compaction payload test file.

## Best Possible Solution

Rework the change on current main so the embedded compaction dispatcher passes the `compaction_start` event into the handler, start events expose a safe `reason`, end events expose `completed`, `hasResult`, `willRetry`, and sanitized `errorMessage`, and compaction count, retry, hook, liveness, UI, and fallback behavior stay unchanged.

## Reproduction Assessment

Yes. The source-level check is high confidence: current main emits the narrow payload, and the PR patch still targets old event names while omitting `hasResult` from emitted end events; no runtime test was run because this was a read-only review.

## Solution Assessment

No. The requested direction is useful, but this exact patch is not the best solution because it needs to be rebased onto the current `compaction_start` / `compaction_end` path, include `hasResult`, and keep routine lifecycle logs below `warn`.
