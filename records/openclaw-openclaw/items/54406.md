## Summary

Keep open. Current main only partially covers the PR’s observability goal: compaction end events expose `completed` and `willRetry`, but start events still lack `reason`, end events still lack `hasResult` and sanitized `errorMessage`, and the current tests assert the narrower payload shape. The PR also has unresolved functional review comments that should be addressed before it is mergeable.

## Best Possible Solution

Keep the PR open for a rebase/rework against current main. The best path is to pass the compaction start event into `handleCompactionStart`, emit `reason` on start, emit `hasResult` and sanitized `errorMessage` on end through both event paths, preserve the existing `completed`/`willRetry` semantics, keep healthy-path logging at an appropriate level, and add focused tests for no-result, aborted-with-result, and willRetry-with-result cases.
