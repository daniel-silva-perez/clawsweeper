## Summary

Keep open. Current main still has the narrower compaction event contract, while the PR branch is stale against the renamed `compaction_start` / `compaction_end` event path and omits one field promised by its own contract, so it is useful source work but not merge-ready.

## What This Changes

The PR branch adds compaction start/end event metadata, structured compaction lifecycle logging, dispatcher forwarding for compaction start details, and focused tests for compaction event payloads.

## Best Possible Solution

Rework the change on current main so the embedded compaction dispatcher passes the `compaction_start` event into the handler, start events expose a safe `reason`, end events expose `completed`, `hasResult`, `willRetry`, and sanitized `errorMessage`, and existing compaction count, retry, hook, liveness, UI, and fallback behavior stay unchanged.

Full review comments:

- **[P2] Emit `hasResult` in compaction end events:** `src/agents/pi-embedded-subscribe.handlers.compaction.ts:120-128`
  - body: The patch computes and logs `hasResult`, and the PR body promises it, but both emitted event payloads still omit it.   - confidence: 0.9

  - body: The patch raises normal compaction start and successful end lifecycle logs from debug to warn. Healthy compaction is expected behavior, so warning on every normal cycle would create production noise; reserve warn for retry, abort, or failure paths.
