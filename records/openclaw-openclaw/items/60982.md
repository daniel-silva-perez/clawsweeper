## Summary

Keep open. Current main still does not implement this PR's intended Discord final-reply sanitization: Discord final delivery forwards raw reply payloads into shared outbound delivery, the Discord outbound adapter does not register a sanitizer hook, and draft preview final edits only strip inline directive tags. The PR remains a plausible Discord-owned fix, but it needs maintainer review for the remaining sanitizer and sanitized-empty media edge cases noted in the review discussion.

## Best Possible Solution

Keep this PR open and finish it as a Discord-owned fix: wire a code-region-aware outbound sanitizer through Discord final delivery or the Discord outbound adapter, apply the same sanitizer to draft preview final edits, preserve media finals when sanitized text becomes empty, and land focused tests for final text, preview edits, <thinking>, <relevant-memories>, code fences, and media payloads.
