## What This Changes

The PR changes legacy memory path helpers and tests so `memory/` directory prefix checks are case-insensitive and dated temporal-decay paths are checked after lowercasing.

## Best Possible Solution

Port the directory-prefix behavior to the active memory host SDK and memory-core plugin surfaces: make default `memory/` directory checks case-insensitive, make dated temporal-decay matching case-insensitive without exempting dated files, cover QMD/read-file/temporal paths in tests, and preserve the current canonical `MEMORY.md` root policy unless maintainers intentionally reopen that contract.
