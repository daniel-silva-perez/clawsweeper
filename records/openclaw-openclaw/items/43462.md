## Summary

Keep open. The underlying case-variant memory-directory bug still exists on current main, while this patch targets stale implementation paths that are no longer shipped, so it is useful source work but not merge-ready as-is.

## What This Changes

The PR changes the old memory helpers and tests so `memory/` directory prefix checks are case-insensitive and dated temporal-decay paths are tested after lowercasing.

## Best Possible Solution

Port the behavior to the current memory surfaces: make default `memory/` directory-prefix checks case-insensitive in the memory host SDK and memory-core temporal/QMD gates, ensure dated files under case-variant directories still decay, and keep the canonical root `MEMORY.md` policy unless maintainers intentionally change that broader contract.
