Keep this PR open, but do not merge it as-is. Current main still has the reported memory path casing bug in the active memory modules, while the PR branch targets old `src/memory/*` paths and still misses the first dated-path parse needed for mixed-case daily memory files.

Port the narrow normalization fix to the active memory-host SDK and memory-core modules: normalize once, compare the memory directory prefix case-insensitively, lowercase before dated filename parsing, preserve canonical root memory semantics, and add focused regression tests for mixed-case evergreen and dated memory paths.
