## Summary

Keep this PR open. Current main still has the silent typing-TTL expiry path: the timer logs and cleans up the typing loop, but there is no TTL-expiry callback, no reply option for caller-supplied feedback, and no dispatch-level still-working notice. The PR is still relevant, but it needs a focused rebase before it is mergeable.

## Best Possible Solution

Keep this PR open as a useful implementation candidate, but require a fresh or heavily rebased auto-reply-only patch against current main. The best patch would add the chosen TTL-expiry feedback contract, contain sync throws and async rejections from custom handlers, preserve routed-reply ordering, add focused tests for TTL expiry, suppressed typing, and routed replies, drop unrelated macOS/Tlon/package-resolution changes, and coordinate the behavior with #40760.
