## Summary

Keep this PR open. Current main at 189c91eae6e1f4f280179961bca2c4153cf42e86 still does not implement the PR's intended typing-TTL expiry feedback path: the TTL timer only logs and cleans up, the reply options have no TTL-expiry callback, dispatch does not inject a still-working notice, and the public typing docs only cover start modes and refresh cadence. The PR remains relevant, but should not be merged as-is because the head mixes the auto-reply fix with unrelated macOS/onboarding and stale Tlon dependency/package-resolution changes, and one bot review concern about synchronous throws from the async handler wrapper remains unresolved in the PR diff.

## Best Possible Solution

Keep this PR open as reference, but do not merge it as-is. The maintainable path is a fresh or heavily rebased auto-reply-only change against current main that adds the chosen TTL-expiry feedback contract, handles both synchronous throws and async rejections from custom handlers, preserves routed-reply ordering, adds focused tests for TTL expiry, suppressed typing, and routed replies, drops unrelated macOS/Tlon/package-resolution changes, and coordinates product behavior with #40760.
