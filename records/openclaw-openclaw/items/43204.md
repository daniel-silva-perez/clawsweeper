## Summary

Keep this PR open. Current main still has the silent typing-TTL expiry behavior this PR targets, and the PR is a useful implementation candidate. It is not ready to close or merge because the branch still needs a focused rebase, an unresolved sync-throw guard fix for custom TTL handlers, and removal/splitting of unrelated macOS/Tlon/package-resolution changes. Related #40760 overlaps the timeout-visibility area but does not supersede this exact TTL-expiry feedback contract.

## Best Possible Solution

Keep this PR open as a useful implementation candidate, but require a fresh auto-reply-focused rebase against current main. The best patch would add the chosen TTL-expiry feedback contract, contain both synchronous throws and async rejections from custom handlers, preserve routed-reply ordering, add focused tests for TTL expiry, suppressed typing, custom handlers, and routed replies, drop unrelated macOS/Tlon/package-resolution changes, and coordinate the product behavior with #40760.
