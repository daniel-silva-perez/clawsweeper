## Summary

Keep this PR open. Current main still has the silent typing-TTL expiry behavior this PR targets, and the PR is still a plausible implementation candidate. It is not ready to close or merge because the branch needs a fresh auto-reply-focused rebase, a fix for synchronous throws from custom TTL handlers, and removal or splitting of unrelated macOS/Tlon/package-resolution changes. ## Best Possible Solution

Keep this PR open as a useful implementation candidate, but require a fresh rebase focused on auto-reply only. The best patch should add the chosen TTL-expiry feedback contract, guard both synchronous throws and async rejections from custom handlers, preserve routed-reply ordering, add focused tests for TTL expiry, suppressed typing, custom handlers, and routed replies, drop unrelated macOS/Tlon/package-resolution changes, and coordinate the final UX with #40760.
