## Summary

Keep this open. Current main still treats cron as a Gateway-local scheduler with local job/state files, process-local serialization, local status, and local CLI commands. There is no distributed lease, leader election, shared coordination backend, HA health/status surface, or cron leader command, and the broader open horizontal-scaling proposal does not replace this cron-specific feature request.

## What This Changes

The issue requests distributed coordination for cron execution across multiple OpenClaw Gateway instances so HA deployments do not run the same scheduled job more than once.

## Best Possible Solution

Define a supported distributed cron ownership model, then implement it with a versioned config or plugin API boundary, health/status visibility, CLI inspection, docs, and tests that simulate separate Gateway processes sharing scheduled jobs.
