## Summary

Keep this open. Current main still implements cron as a Gateway-local scheduler backed by local job/state files and process-local serialization; there is no distributed lock, leader election, coordination config, HA health/status, or CLI leader surface. The related closed same-store duplicate issue and the broader open horizontal-scaling proposal do not replace this narrower HA cron ownership request.

## What This Changes

Request to add distributed cron ownership or locking so multiple OpenClaw gateways in HA deployments do not execute the same scheduled jobs multiple times.

## Best Possible Solution

Define a supported distributed cron ownership model: either core-owned coordination config and health/CLI visibility, or a documented, versioned scheduler-ownership or lock-provider API that plugins can implement, with tests that simulate separate gateway processes/nodes and clear interaction with #25883.
