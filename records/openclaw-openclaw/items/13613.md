## Summary

Keep open. The current implementation and docs still leave cron ownership local to each Gateway process, while the related horizontal-scaling proposal is open and the open slot-dedup PR explicitly does not cover cross-process races. This remains a valid feature gap that needs maintainer architecture direction before implementation.

## What This Changes

The issue requests distributed coordination for scheduled cron jobs so HA or multi-Gateway deployments do not execute the same job multiple times.

## Best Possible Solution

Define a supported distributed cron ownership model for HA deployments, then implement it behind a versioned config or plugin/API boundary with backend health, leader/lease visibility, CLI inspection, docs, and tests that simulate separate Gateway processes sharing scheduled jobs.
