## Summary

Keep open. Current main still implements cron as a Gateway-local scheduler with local job/state files, process-local serialization, local status/CLI surfaces, and no distributed lease, leader election, shared backend, or HA health/status contract. The broader horizontal-scaling proposal remains open and the related slot-dedup PR is explicitly process-local, so neither supersedes this cron-specific HA coordination request.

## What This Changes

The issue requests distributed cron job coordination across multiple OpenClaw gateways for high-availability deployments.

## Best Possible Solution

Define a supported distributed cron ownership model for HA deployments, then implement it behind a versioned config or plugin/API boundary with health/status visibility, CLI inspection, docs, and tests that simulate separate Gateway processes sharing scheduled jobs.
