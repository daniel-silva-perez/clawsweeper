## Summary

Keep this open. Current main still implements cron as a Gateway-local scheduler with local job/state files, process-local serialization, local status, and local CLI commands; it has not added distributed leases, leader election, shared coordination backends, HA health/status, or a cron leader surface. The existing same-store duplicate test only covers two CronService instances inside one Node process, and the broader #25883 horizontal-scaling proposal is still open rather than a merged replacement for this cron-specific request.

## What This Changes

The issue requests distributed coordination for cron execution across multiple OpenClaw Gateway instances so HA deployments do not run the same scheduled job more than once.

## Best Possible Solution

Define a supported distributed cron ownership model, then implement it with a versioned config or plugin API boundary, health/status visibility, CLI inspection, docs, and tests that simulate separate Gateway processes sharing scheduled jobs.
