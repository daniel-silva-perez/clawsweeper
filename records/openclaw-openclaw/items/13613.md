## Summary

Keep this issue open. Current main still has gateway-local cron execution with process-local/store-path serialization, not distributed cron ownership across HA gateways. The later #69738 fix documents and supports per-instance state isolation to avoid accidental shared-store duplicate fires, but it does not provide leader election, Redis/etcd/DynamoDB locks, lock TTLs, coordination health, or a cron leader/status CLI. The broader #25883 horizontal-scaling proposal may inform the design, but it does not supersede this narrower cron coordination request.

## Best Possible Solution

Keep this open for maintainer design. A complete solution should either implement distributed cron ownership in the core scheduler or add a documented, versioned lock-provider/scheduler-ownership API. It should define config/schema/docs, lease and expiry semantics, behavior when coordination is unavailable, health/status and CLI visibility, multi-process or multi-node regression coverage, and how this relates to the broader #25883 horizontal-scaling discussion. Treat #69738 as the per-instance state-isolation fix, not the HA cron coordination solution.
