## Summary

Keep this issue open. Current main still has gateway-local cron scheduling with only process-local serialization around the JSON store; it does not expose distributed cron ownership, leader election, Redis/etcd/DynamoDB lock configuration, lock TTLs, health/status, or a CLI leader surface. The related #13582 work is local scheduler reliability, not HA coordination across gateways.

## Best Possible Solution

Keep this open until OpenClaw either implements distributed cron ownership in the core scheduler or adds a documented scheduler/lock-provider API that can support it. A complete fix should define config/schema/docs, due-job ownership semantics across gateways, leader or lock health in status/CLI, failure/lease expiry behavior, and multi-process or multi-node regression coverage. Treat #13582 as shipped local scheduler reliability work, not the canonical HA coordination tracker.
