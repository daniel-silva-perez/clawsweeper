## Summary

Keep open. Current main still treats cron as Gateway-local work backed by local JSON state and process-local serialization; there is no distributed ownership contract, backend lease, leader election, HA status, or `cron leader` surface. The related horizontal-scaling proposal remains open rather than a merged replacement, so this issue still needs maintainer architecture direction.

## What This Changes

The issue requests distributed coordination for cron jobs across multiple Gateway instances so HA deployments do not execute the same scheduled jobs multiple times.

## Best Possible Solution

Define a supported distributed cron ownership model for HA deployments, then implement it behind a versioned config or plugin/API boundary with backend health, leader or lease visibility, CLI inspection, docs, and tests that simulate separate Gateway processes sharing scheduled jobs.
