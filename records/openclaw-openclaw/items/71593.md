## What This Changes

The issue requests a plugin-based SecretRef provider mechanism so users can cover secret-bearing OpenClaw config fields without repeated core schema changes.

## Best Possible Solution

Review PR #72548 as the current implementation candidate, then either land a versioned plugin SecretRef provider contract with docs/tests/security review or split the remaining broad field-coverage request into a narrower canonical follow-up before closing this issue.

## Solution Assessment
