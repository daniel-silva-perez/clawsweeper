## Summary

Keep open. Current main and the latest release still match the reported Telegram DM regression: the dispatcher forces DM answer streaming through message-preview transport even though the lower draft stream would choose native draft transport for DM threads when left on auto, and #47631 is already the focused open implementation candidate.

## What This Changes

The issue asks Telegram DM reply streaming to preserve auto or draft-capable preview transport for answer lanes instead of forcing message transport, with a possible config preference for DM reply transport.

## Best Possible Solution

Land #47631 or an equivalent Telegram plugin change that lets the DM answer lane pass auto so createTelegramDraftStream can choose native draft transport, keeps reasoning on message transport, preserves duplicate-final safeguards, and aligns Telegram docs/config wording with the chosen behavior.
