## Summary

Keep open. Current main and the latest release still force Telegram DM answer streaming through message-preview transport, while the lower draft stream would choose native draft transport for DM threads when left on auto. The open focused PR #47631 is already the implementation candidate for the same behavior, so this should stay open for maintainer review rather than be closed or queued separately.

## What This Changes

Telegram DM answer streaming should preserve auto or draft-capable preview transport instead of forcing message transport, with optional configuration if maintainers want an explicit transport preference.

## Best Possible Solution

Land #47631 or an equivalent Telegram plugin change that lets the DM answer lane pass auto so createTelegramDraftStream can choose native draft transport, keeps reasoning on message transport, preserves duplicate-final safeguards, and updates Telegram docs/config wording to match the selected behavior.
