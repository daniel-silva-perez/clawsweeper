## Summary

Keep open. Current main still matches the reported Telegram DM regression: the dispatcher forces DM answer streaming through message-preview transport, while the lower Telegram draft-stream helper would choose native draft transport for DM threads when left on auto. The latest release tag still contains the same selector, and #47631 is already the focused open implementation candidate for this boundary.

## What This Changes

Request that Telegram direct-message answer preview streaming preserve auto or draft-capable transport instead of forcing message transport, with a possible DM transport preference config.

## Best Possible Solution

Review and land #47631 or an equivalent Telegram plugin change that lets the DM answer lane pass auto so createTelegramDraftStream can choose native draft transport, keeps reasoning on message transport, preserves duplicate-final safeguards with regression tests, and updates Telegram docs/config wording to match the selected behavior.
