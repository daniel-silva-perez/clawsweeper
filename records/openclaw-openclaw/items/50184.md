## Summary

Keep open. Current main still matches the reported regression: the Telegram dispatcher forces DM answer streaming through message preview transport, while the lower draft-stream helper would choose native draft transport for DM threads when left on auto. An open focused PR, #47631, is already the implementation candidate for this same boundary, so this should stay open for maintainer review rather than be closed or queued as a separate repair.

## What This Changes

Telegram DM streamed answer previews should preserve draft-capable auto behavior instead of the reply layer forcing message preview transport.

## Best Possible Solution

Land #47631 or an equivalent Telegram plugin change that lets the DM answer lane pass auto so createTelegramDraftStream can choose native draft transport, keeps reasoning on message transport, preserves duplicate-final safeguards, and aligns the Telegram docs/config surface with the selected behavior.
