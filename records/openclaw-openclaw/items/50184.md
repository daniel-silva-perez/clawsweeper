## Summary

Keep open. Current main still matches the reported Telegram DM regression: the dispatcher forces DM answer streaming through message-preview transport, while the lower draft-stream helper would choose native draft transport for DM threads when left on auto. The latest release tag still contains the same selector, and #47631 is already the focused open implementation candidate.

## What This Changes

The issue requests that Telegram DM answer preview streaming preserve auto/draft-capable transport instead of forcing message preview transport, ideally with a config option for the DM reply transport preference.

## Best Possible Solution

Land #47631 or an equivalent Telegram plugin change that lets the DM answer lane pass auto so createTelegramDraftStream can choose native draft transport, keeps reasoning on message transport, preserves duplicate-final safeguards, and aligns Telegram docs/config wording with the selected behavior.
