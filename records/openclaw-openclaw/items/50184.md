## Summary

Keep open. Current main still forces Telegram DM answer streaming through message preview transport, while the lower draft-stream helper still defaults auto DM previews to native draft transport. The latest shipped release has the same selector, and PR #47631 is already open as a focused implementation candidate for restoring draft-capable answer lanes, so this should stay open for maintainer review rather than be swept closed or queued as a separate repair.

## What This Changes

Telegram DM answer streaming should stop forcing message preview transport and preserve auto or draft-capable transport selection so native Telegram draft previews can be used for direct chats.

## Best Possible Solution

Resolve this through PR #47631 or an equivalent Telegram plugin change that lets the DM answer lane pass `auto` so `createTelegramDraftStream` can choose native draft transport, while preserving the reasoning-lane message transport decision and duplicate-final safeguards.
