## Summary

Keep open. Current main still contains the reported transport override and tests assert it, while the lower Telegram draft stream remains draft-capable and the latest release still has the same selector. PR #47631 is an open, unmerged implementation candidate, so the issue should stay open for that review or an equivalent fix.

## What This Changes

The issue asks for Telegram DM reply streaming to stop forcing answer previews to message transport and preserve the lower draft-capable auto behavior, with an optional user-facing transport preference.

## Best Possible Solution

Keep the issue open and resolve it through PR #47631 or an equivalent Telegram plugin patch that lets the DM answer lane pass `auto` so `createTelegramDraftStream` can choose native draft transport, while explicitly preserving or deciding reasoning-lane transport and duplicate-final safeguards.
