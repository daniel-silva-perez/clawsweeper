## Summary

Keep open. Current main still has the TUI `/model list` and `/model status` fallthrough bug, but this PR should not merge as-is because it fixes the literal model-id path by bypassing the shared session-aware model command behavior.

## What This Changes

The PR changes the TUI `/model` command handler and tests so `list` opens the TUI model selector and `status` prints cached session model information instead of patching the session model.

## Best Possible Solution

TUI should reserve `list` and `status` before model selection while preserving the shared slash-command contract: listing should use the active agent/session/workspace context, status should keep the detailed model/auth/endpoint diagnostics, and tests should assert those subcommands do not call `sessions.patch`.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: `/model list` and `/model status` parse to the TUI `model` case with non-empty args, then the handler forwards those words to `sessions.patch` as model refs.

## Solution Assessment

No. The PR fixes the immediate literal model-ref failure, but it is not the narrowest maintainable solution because it routes `list` through the bare TUI picker and replaces detailed shared `status` diagnostics with cached session info.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Route `/model list` through the session-aware command path:** `src/tui/tui-command-handlers.ts:298-299`
  - body: The new `args === "list"` branch calls `openModelSelector()`, which sends a bare `models.list` request and resolves visibility from the default workspace rather than the active TUI session. In multi-agent sessions this can show the wrong catalog, so route this through the shared model command path or pass current session context into the list API.
  - confidence: 0.84
- **[P2] Preserve `/model status` diagnostics:** `src/tui/tui-command-handlers.ts:300-306`
  - body: The new `status` branch replaces the shared status response with one cached `state.sessionInfo` line or `unknown`. That drops selected vs active runtime model, default model, agent, auth file, provider auth state, and endpoint details, and can report unknown before the gateway can answer the command correctly.
  - confidence: 0.88
