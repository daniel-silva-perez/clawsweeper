## Summary

Keep this PR open. Current main still does not implement the PR's intended TUI behavior: `/model list` and `/model status` are parsed as non-empty `/model` arguments and routed to `sessions.patch` as model references. The shared chat directive path now handles these aliases, but the TUI command switch intercepts `/model ...` first, so this PR is not obsolete.

## Best Possible Solution

Keep this PR open for maintainer review. The best direction is to fix the TUI-specific interception so `/model list` and `/model status` are reserved subcommands with regression tests, ideally preserving the shared agent-aware model browsing and detailed status diagnostics instead of duplicating a narrower TUI-only response.
