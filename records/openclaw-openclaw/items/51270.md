## Summary

Keep open. Current main still has the reported TUI behavior, so the PR is not obsolete, but this revision should not merge as-is because it fixes the literal model-id fallthrough by bypassing the session-aware shared model command path and by replacing the detailed `/model status` diagnostics with a cached one-line TUI response.

## What This Changes

The PR changes the TUI `/model` command handler and tests so `/model list` opens the model selector and `/model status` prints session model information instead of treating those words as model IDs.

## Best Possible Solution

TUI should reserve `list` and `status` before model selection while preserving the shared slash-command contract: listing should use the active agent/session/workspace context, status should keep the detailed model/auth/endpoint diagnostics, and tests should assert those subcommands do not call `sessions.patch`.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: `/model list` and `/model status` parse to `name: "model"` with non-empty args, and the TUI handler forwards those args to `sessions.patch` as model refs.

## Solution Assessment

No. The PR fixes the literal model-id failure, but it is not the best implementation because it uses the non-session-aware TUI picker for `list` and drops the shared detailed diagnostics for `status`.
