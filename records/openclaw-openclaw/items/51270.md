## Summary

Keep open. Current main still has the reported TUI parsing bug, and this PR targets the right surface, but the proposed fix is not merge-ready because it bypasses the shared model command semantics for both listing and status output.

## What This Changes

The PR changes the TUI `/model` command handler and tests so `/model list` opens the model selector and `/model status` prints session model information instead of treating those words as model IDs.

## Best Possible Solution

TUI should reserve `list` and `status` before model selection while preserving the shared slash-command contract: listing should use active agent/session context, status should keep the detailed model/auth/endpoint diagnostics, and tests should assert those subcommands do not call `sessions.patch`.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: `/model list` and `/model status` parse to non-empty `/model` args and the TUI handler forwards those args to `client.patchSession({ model: args })`.

## Solution Assessment

No. The PR fixes the literal model-id failure, but it does so by using the non-session-aware TUI picker for `list` and replacing the shared detailed status diagnostics with a single cached `sessionInfo` line.
