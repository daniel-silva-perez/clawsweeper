## Summary

Keep open. Current main still sends TUI `/model list` and `/model status` through the model-setting path, so the PR is not obsolete, but the branch should not merge as-is because it bypasses the shared model command semantics for session-aware listing and detailed status diagnostics.

## What This Changes

The PR changes the TUI `/model` command handler and colocated tests so `/model list` opens the model selector and `/model status` prints session model information instead of treating those words as model IDs.

## Best Possible Solution

TUI should reserve `list` and `status` before model selection while preserving the shared slash-command contract: `/model list` should browse models with the active agent/session context, `/model status` should show selected/runtime/default/agent/auth/endpoint details, and tests should assert those subcommands do not call `sessions.patch`.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: `/model list` parses to `name: "model", args: "list"`, then the TUI handler sends the non-empty arg to `client.patchSession({ model: args })`; the same applies to `status`.

## Solution Assessment

No. The PR fixes the literal model-id failure, but `/model list` uses the non-session-aware picker path and `/model status` replaces the shared detailed diagnostics with one `state.sessionInfo` line or `unknown`.

## Review Findings
