## Summary

Keep this issue open. Current main partially addresses memory indexing concurrency through `agents.defaults.memorySearch.remote.nonBatchConcurrency`, but the embedding retry parameters requested here are still hardcoded. There is also an open, unmerged implementation candidate, #68822, whose body uses closing syntax for this issue, so the issue should remain open until that PR or a successor is reviewed and merged or explicitly closed.

## Best Possible Solution

Review #68822 or a successor against the current `agents.defaults.memorySearch` config shape, add schema/types/docs/tests for configurable embedding retry max attempts/base delay/max delay plus any remaining concurrency behavior, and close the issue only after the implementation merges. Coordinate any broader indexing-throughput scope with #14207.
