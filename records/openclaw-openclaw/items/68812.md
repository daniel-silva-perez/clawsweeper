## Summary

Keep this issue open. Current main only partially addresses the requested concurrency concern through `agents.defaults.memorySearch.remote.nonBatchConcurrency`; the embedding retry attempts/base delay/max delay are still hardcoded constants and are not exposed in config, schema, docs, or tests. The provided GitHub context also shows open PR #68822 as an unmerged implementation candidate using closing syntax for this issue, so the issue should remain open until that PR or a successor is reviewed and merged or closed.

## Best Possible Solution

Keep the issue open and review #68822 or a successor. The best implementation should fit the existing `agents.defaults.memorySearch` config shape, add typed/schema/help/docs coverage for retry max attempts, base delay, and max delay, wire those values into `MemoryManagerEmbeddingOps` retry calls and wait clamping, and keep broader indexing-throughput work coordinated with #14207.
