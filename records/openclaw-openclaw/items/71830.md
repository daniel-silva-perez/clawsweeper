## Summary

Keep this PR open. Current main still lacks the WhatsApp-specific XML tool-call sanitizer expansion, `channels.whatsapp.exposeErrorText`, and the opt-in path for text-only error payloads that lets WhatsApp apply channel-owned suppression policy. The PR remains the active implementation candidate for the production report in #71715, and the latest diff appears to address the earlier account-level override review note.

## Best Possible Solution

Keep this PR open for normal maintainer review and landing, or replace it with an equivalent focused patch that sanitizes WhatsApp-visible plural tool/function XML across auto-reply, direct outbound, and sendPayload paths; adds documented `channels.whatsapp.exposeErrorText` with channel/account inheritance and default-compatible behavior; and treats suppressed text-only WhatsApp error payloads as no-op deliveries without false message-sent bookkeeping.
