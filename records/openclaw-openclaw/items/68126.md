## Summary

Keep open. Current main still follows the reported Discord DM path: auto-thread reply planning produces a channel-shaped reply target, non-guild DMs have no auto-thread context, and the inbound context still assigns `ctx.To` from that channel-shaped target before the user-shaped DM conversation target is computed. The focused related fix PR is closed unmerged, so this remains actionable source-level work.

## What This Changes

The issue requests that inbound Discord DMs set semantic `ctx.To` to `user:<userId>` while preserving the channel ID for Discord API delivery.

## Best Possible Solution

Make an extension-local Discord message-handler change that resolves `dmConversationTarget` before `effectiveTo`, uses it for semantic inbound DM `ctx.To`, preserves channel-shaped delivery targets for Discord API calls, and adds a focused regression test for `To`, `From`, and `ChatType` on inbound DMs.

## Reproduction Assessment

Yes. The current source path deterministically reproduces the bug for direct messages: `replyTarget` is `channel:<dmChannelId>`, `autoThreadContext` is null, and `ctx.To` is assigned before the user-shaped DM target is available.

## Solution Assessment

Yes. The narrowest maintainable fix is the same direction as the closed PR: change only the semantic inbound context fallback for DMs, leave delivery routing channel-based, and add a regression assertion for the dispatch context shape.
