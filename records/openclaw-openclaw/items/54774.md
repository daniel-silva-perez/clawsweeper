## Summary

Keep this PR open. Current main still treats a pre-hello code-1000 empty-reason gateway close as fatal in one-shot gateway calls, while this PR is a focused implementation candidate with regression coverage and a bounded client-side suppression path. It also overlaps with #54475, so maintainers should choose one canonical branch before merging or closing either one.

## Best Possible Solution

Keep this PR open for maintainer review and consolidate it with #54475. The best path is a narrow shared gateway fix: track `hello-ok` in `src/gateway/call.ts`, suppress only pre-hello code-1000 empty-reason closes in `src/gateway/client.ts`, keep suppression bounded and keyed to a specific connect attempt, and retain focused regression tests in `src/gateway/call.test.ts` and `src/gateway/client.test.ts`. After one branch is selected and lands, close the other overlapping PR as superseded.
