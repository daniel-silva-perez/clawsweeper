## Summary

Current main still appears to have the fallback edge case this PR targets: a primary transient cooldown probe can consume the per-run provider probe slot, causing a later same-provider fallback model to be skipped. The PR remains useful and should be refreshed or reimplemented rather than closed.

## Best Possible Solution

Keep this PR open. A maintainer should refresh or reimplement the narrow behavior on current main: primary cooldown probes should not check or consume `cooldownProbeUsedProviders`, while non-primary same-provider fallbacks should still deduplicate among themselves. Keep the same-provider-after-primary-probe regression test and the non-primary dedup regression test, then run the targeted `src/agents/model-fallback.test.ts` coverage.
