## Summary

Keep open. Current main still appears to have the edge case this PR targets: a primary transient cooldown probe can populate the per-run provider probe slot, and a later same-provider fallback can then be skipped. The PR discussion is useful and positive: the bot review and reviewer both identified the same narrow fix, and the author followed up with comments plus an extra dedup regression test.

## Best Possible Solution

Keep this PR open and have a maintainer either refresh it or reimplement the same narrow behavior on current main: primary cooldown probes should not check or consume `cooldownProbeUsedProviders`, while non-primary same-provider fallbacks should still deduplicate among themselves. The refreshed change should include the exact same-provider-after-primary-probe regression plus the non-primary dedup regression, then run the targeted model fallback tests.
