## Summary

Keep open: current main still rejects a first-level `~/.openclaw` symlink in the exec approvals write path, and the issue is already paired with open fixing PRs that should be reviewed or superseded before closure.

## What This Changes

Support a trusted first-level `~/.openclaw` symlink for local exec-policy and exec-approvals writes while preserving deeper symlink traversal rejection.

## Best Possible Solution

Land a single reviewed fix that permits only the immediate `.openclaw` child of the trusted home root after ownership and mode validation, continues traversal from the real target, keeps deeper symlinks and symlinked destination files rejected, and adds focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. The high-confidence reproduction is a temp-home test where the trusted home is real, `.openclaw` is a symlink to a user-owned config directory, and `saveExecApprovals()` or `exec-policy preset yolo` reaches the current `lstatSync(...).isSymbolicLink()` rejection.

## Solution Assessment

Yes, directionally. The narrowest maintainable solution is the issue's proposed trusted first-hop exception with ownership and permission checks; broader macOS/socket hardening should only be included if maintainers want that larger scope in the same PR.
