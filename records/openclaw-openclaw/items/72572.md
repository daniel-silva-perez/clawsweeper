## What This Changes

The issue asks OpenClaw to allow a trusted, first-level `~/.openclaw` symlink for local exec approvals while preserving deeper symlink and destination-write hardening.

## Best Possible Solution

Land one reviewed implementation that permits exactly one trusted, user-owned, non-group/other-writable first-level `.openclaw` symlink under the home root while continuing to reject chained or deeper symlinks and symlinked approvals-file destinations, with focused store tests and docs if the supported layout changes.

Yes. A source-level reproduction path is high-confidence: `openclaw exec-policy preset yolo` calls `saveExecApprovals()`, `ensureDir()` walks from the home root, and a symlinked `.openclaw` child is rejected before the approvals file can be written.
