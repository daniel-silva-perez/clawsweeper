## Summary

Keep open. Current main still builds subagent and cron announce handoffs from the child completion result plus a rewrite instruction; the internal event, active-session steer path, dormant direct-agent call, docs, and tests do not show an explicit requester-conversation context summary. The related reports in the provided context cover completion routing and delivery resilience, not this prompt-context failure.

## What This Changes

Request that subagent and isolated cron announce completion handoffs include enough parent/requester conversation context so generated user-facing completion replies stay on topic.

## Best Possible Solution

Add a bounded requester-context handoff for subagent and cron announce replies, using the parent/requester session state where possible and covering active and dormant delivery paths with regression tests, without leaking unrelated raw transcript content into child completion events.
