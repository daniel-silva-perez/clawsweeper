## Summary

Keep open. Current main still only partially overlaps with this PR: it has an internal agent event stream and host-owned plugin event subscriptions, but it does not implement the PR's proposed public channel-plugin API shape, typed thinking lifecycle hooks, or enriched agent_end metadata. The PR also changes a security-sensitive event surface and has unresolved review concerns, so it should remain available for maintainer review rather than be closed as obsolete.

## Best Possible Solution

Keep this PR open for maintainer review or author follow-up. The best path is to rebase it onto current main, decide whether the public SDK should use the existing registerAgentEventSubscription/runtime.events path or a new api.onAgentEvent shape, add typed thinking lifecycle hooks only if maintainers want that contract, thread token/tool metadata through agent_end with SDK docs and baseline updates, and require focused tests for event scoping, cleanup, native/tagged reasoning, duplicate/orphaned end events, and message_end fallback behavior.
