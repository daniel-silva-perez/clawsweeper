## Summary

Keep open. Current main partially overlaps with the PR through the internal agent event stream and host-owned plugin event subscriptions, but it does not implement the PR's requested thinking_start/thinking_end plugin hooks, top-level api.onAgentEvent contract, or agent_end token/tool metadata. The PR also touches a security-sensitive event surface that still needs explicit scoping review.

## Best Possible Solution

Keep this PR open for maintainer review or author follow-up. The best path is to rebase onto current main, reconcile the proposed streaming surface with the existing registerAgentEventSubscription/api.runtime.events.onAgentEvent seams, decide whether typed thinking_start/thinking_end hooks are still desired, add tokenUsage/toolCallCount to agent_end only with aligned docs/API baselines, and include tests for event scoping plus native/tagged reasoning edge cases.

## Risks / Open Questions

- Agent event subscriptions can carry assistant text, thinking deltas, and tool activity, so any public plugin API needs explicit session/run scoping and terminal cleanup tests.
- Adding api.onAgentEvent beside registerAgentEventSubscription and api.runtime.events.onAgentEvent could create overlapping public SDK contracts unless maintainers choose one shape.
- Thinking lifecycle hooks need focused edge-case coverage for native events, tagged reasoning, duplicate or orphaned end events, message_end fallback, and code-span filtering.
