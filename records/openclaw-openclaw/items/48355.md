## Summary

Keep this PR open. Current main only partially overlaps with the proposal: it has a global agent event bus, `stream: "thinking"` events, and the host-owned `registerAgentEventSubscription` seam, but it does not implement the PR's public top-level `api.onAgentEvent` shape, typed `thinking_start`/`thinking_end` plugin hooks, or `agent_end` token/tool metadata. The PR also touches a security-sensitive streaming surface and still needs maintainer review for event scoping and lifecycle edge cases.

## Best Possible Solution

Keep this PR open for maintainer review or author follow-up. The best path is to rebase it onto current main, decide whether the public SDK should extend the existing `registerAgentEventSubscription` and runtime event surfaces or add a new top-level `api.onAgentEvent` API, add typed thinking lifecycle hooks only if maintainers want that public contract, thread token/tool metadata through `agent_end` with docs and SDK baseline updates, and require focused tests for event scoping, listener cleanup, thinking lifecycle dedupe, orphaned end events, message-end fallback, and channel-plugin usage.
