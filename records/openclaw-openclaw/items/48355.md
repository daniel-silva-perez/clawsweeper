## Summary

Keep open. Current main only partially overlaps with the PR: it has an internal thinking event stream and host-owned plugin event subscriptions, but it still lacks the PR's public thinking lifecycle hooks, top-level plugin API shape, and enriched agent_end payload. The PR remains relevant, but the proposed patch is not merge-ready because it exposes a sensitive live event stream without the same conversation-access gate used by comparable hooks and still has thinking lifecycle edge-case bugs.

## What This Changes

The PR adds public plugin thinking_start/thinking_end hooks, a session-filtered api.onAgentEvent stream subscription, thinking stream emissions for plugin listeners, and token/tool metadata on agent_end with tests and SDK baseline updates.

## Best Possible Solution

Keep the work available, but land it only after maintainers settle the public SDK shape against the existing registerAgentEventSubscription/runtime.events path, enforce the intended conversation-data permission boundary, and make thinking lifecycle delivery deterministic across native, tagged, duplicate, and fallback stream shapes.

## Reproduction Assessment

Not applicable as a feature PR, but the current-main gap is source-verifiable. The main review defects have code-level reproduction paths, such as delivering two native thinking_end events to the PR handler after one thinking phase and observing a second synthetic start/end pair.

## Solution Assessment

No, not as proposed. The direction is useful, but the patch should first align with the current host-owned plugin event subscription model, reuse the conversation-access policy for raw event data, and harden the thinking lifecycle before exposing the public contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Gate agent event streaming behind conversation access:** `src/plugins/registry.ts:1000-1007`
  - body: This exposes live thinking, assistant, and tool events to any full plugin that can call api.onAgentEvent with a sessionKey, while comparable raw conversation hooks require the plugin conversation-access gate. A non-bundled plugin can bypass that policy through the new stream API, so the wrapper should enforce the same permission boundary or dispatch only a sanitized contract.
  - confidence: 0.86
- **[P2] Deduplicate native thinking_end lifecycle events:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:223-239`
  - body: After the first thinking_end, emitReasoningEnd clears reasoningStreamOpen. If the provider sends a duplicate thinking_end, this branch reopens the stream, emits a synthetic thinking_start, and then emits another thinking_end, which double-counts timers and flickers stateful channel UIs. Track the active block or ignore duplicate ends until the next real start/delta.
  - confidence: 0.9
- **[P2] Respect code spans in fallback thinking extraction:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:463-465`
  - body: The message_end fallback extracts tagged thinking with a regex-only helper over the final assistant text. When a native thinking stream is still open and the final answer contains a literal `<think>...</think>` example in code, the new thinking_end hook can receive fake reasoning text. Reuse the code-span-aware tag scanner or limit fallback text to provider thinking fields.
  - confidence: 0.78
