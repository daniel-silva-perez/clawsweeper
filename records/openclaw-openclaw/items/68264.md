## Summary

Keep open. Current main still treats `canvas present` and `canvas navigate` as node canvas controls: they select a node first and return JSON success rather than a chat-visible page preview. The browser plugin has separate `snapshot` and `screenshot` actions that can return text or image output, but `browser navigate` also returns JSON, so the reported product gap is not fully implemented.

## What This Changes

The issue asks for canvas/browser UI commands, especially `canvas present` and `canvas navigate`, to visibly render or confirm webpage state in chat instead of surfacing node/validation errors without a usable preview.

## Best Possible Solution

Define the intended chat-visible contract for canvas and browser controls, then either add a preview-producing result or optional snapshot after state-changing actions, or tighten tool descriptions, docs, and `node required` guidance so users reliably select a node and call the explicit preview action. Add focused regression coverage for the chosen behavior.
