## Summary

Keep open. Current main has adjacent streaming, typing indicators, channel health/run-state tracking, and outbound delivery hooks, but it still does not provide a configurable runtime beacon keyed to operator-visible silence; because this would change shared channel notification behavior, it needs maintainer product/design review rather than cleanup closure.

## What This Changes

Request to add a configurable per-channel silence watchdog that sends repeatable status beacons during active work when no operator-visible output has been delivered for a threshold interval.

## Best Possible Solution

Define an opt-in, default-off generic channel/runtime watchdog with a documented operator-visible delivery timestamp, shared reset/cancel/repeat semantics, per-channel config/schema/docs, and focused tests across outbound delivery, preview/block streaming, active run lifecycle, and max-beacon behavior.

## Reproduction Assessment

Not applicable as a feature request rather than a bug report. The high-confidence check is source/docs search on current main, which shows the requested watchdog contract is absent while adjacent liveness mechanisms exist.

## Solution Assessment

Unclear until maintainer approval. The maintainable direction is a generic default-off runtime seam tied to real visible delivery, not per-channel ad hoc messages or agent prompt rules.
