## Summary

Current main still does not implement the requested configurable channel silence watchdog. Existing streaming, tool-progress previews, channel run-state health, and transport watchdogs are adjacent, but none send a repeatable operator-visible beacon keyed to elapsed visible-output silence during active work.

## Best Possible Solution

Keep this open for maintainer product/design review. The best path is an opt-in, default-off generic channel/runtime silence watchdog that observes active reply lifecycle and a clearly defined operator-visible delivery timestamp updated by final replies, block/preview streaming, tool-progress previews, visible tool-result delivery, and plugin-owned visible sends, with docs and focused tests for reset, repeat, cancellation, and max-beacon behavior.
