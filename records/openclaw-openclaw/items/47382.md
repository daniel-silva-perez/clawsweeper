## Summary

Keep this issue open. Current main has adjacent liveness mechanisms, including typing indicators, preview/block streaming, tool-progress preview updates, status reactions, and transport stall watchdogs, but I did not find the requested configurable gateway/channel silence watchdog keyed to operator-visible delivery silence during active work.

## Best Possible Solution

Keep the issue open for maintainer/product design. A good fix would add a generic channel/gateway delivery watchdog with strict config schema, per-channel/account overrides, visible-delivery timestamp tracking, threshold/max-beacon controls, and tests proving visible sends/edits/block replies reset the timer while internal model/tool activity does not. It should document how the watchdog interacts with existing preview streaming, typing indicators, and status reactions.
