## Summary

Keep open. Current main has adjacent streaming, typing indicators, run-state health tracking, and outbound delivery hooks, but it still lacks a configurable channel silence watchdog keyed to operator-visible output; because this would alter shared notification behavior, it needs maintainer product/design review rather than cleanup closure.

## What This Changes

Request a gateway/channel silence watchdog that sends configurable operator-visible status beacons during active work after a period without visible channel output.

## Best Possible Solution

Define an opt-in, default-off generic runtime watchdog with a documented operator-visible delivery timestamp, shared reset/cancel/repeat semantics, per-channel config/schema/docs, and focused coverage for outbound delivery, preview/block streaming, active run lifecycle, and max-beacon behavior.

## Reproduction Assessment

Not applicable as a feature request rather than a bug report. The high-confidence check is source/docs search on current main, which shows the requested watchdog contract is absent while adjacent liveness mechanisms exist.

## Solution Assessment

Unclear until maintainer approval. The maintainable direction is a generic default-off runtime seam tied to real visible delivery, not per-channel ad hoc messages or agent prompt rules.
