## Summary

Keep open. Current main has pinned Docker base-image digests and a weekly Dependabot Docker entry, plus brief Docker docs saying Dependabot refreshes the Node base digest, but the requested stale-image alerting and full update runbook are not implemented. This is still an actionable repo/CI maintenance issue rather than a duplicate, ClawHub item, or already-shipped fix.

## Best Possible Solution

Keep this issue open and implement it as a small secops-reviewed maintenance change: verify Dependabot behavior for the Dockerfile ARG-based pinned images, add a scheduled/manual workflow or script that resolves the current `node:24-bookworm` and `node:24-bookworm-slim` manifest-list digests and flags stale pins, dedupe any created alert issue, and add a concise public Docker update runbook to the existing Docker docs.
