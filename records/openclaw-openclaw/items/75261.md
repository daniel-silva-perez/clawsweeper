## Summary

Keep open. Current main and the latest tagged release still omit the canonical current channel target from the Claude CLI bundle-MCP loopback path, so implicit cron delivery can still fall back to the lowercased session-key parser for Matrix room IDs. The closed #75303 PR is useful source context, but it was not merged.

## What This Changes

The issue asks OpenClaw to preserve case-sensitive Matrix room IDs when Claude CLI bundle-MCP cron creation infers delivery from `sessionTarget: "current"`.

## Best Possible Solution

Thread canonical current delivery context through the Claude CLI bundle-MCP loopback pipeline generically: run context to `RunCliAgentParams`, env/config headers, request context, loopback cache, `resolveGatewayScopedTools`, and `createOpenClawTools`. Add regression coverage for mixed-case Matrix targets without adding Matrix-specific cron special cases.

## Reproduction Assessment

Yes. The issue provides concrete v2026.4.27 steps and stored job/error evidence, and current main source still shows the same lowercased fallback path when loopback requests omit a current target. I did not run a live Matrix/Claude CLI loop because this review is read-only.

## Solution Assessment

Yes. Extending the existing `currentDeliveryContext` seam into loopback tool resolution is the narrowest maintainable fix; reconstructing Matrix room IDs inside cron from lowercased session keys would fight the documented routing contract.
