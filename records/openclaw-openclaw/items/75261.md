## Summary

Keep open. Current main still has a credible MCP loopback gap: the in-process cron tool can preserve canonical delivery context, but the Claude CLI bundle-MCP loopback path does not forward the current chat target into tool resolution, so Matrix cron creation can still fall back to lowercased session-key inference.

## What This Changes

The issue reports that cron creation through MCP loopback with `sessionTarget: "current"` still derives a lowercased Matrix `delivery.to` in v2026.4.27.

## Best Possible Solution

Preserve the canonical chat-origin delivery target across the MCP loopback boundary, preferably by threading current delivery context through the generated loopback headers/env and tool cache, with a fallback lookup from preserved session origin metadata for case-sensitive channels.

## Reproduction Assessment

Yes. A source-level reproduction is clear: invoke the cron tool through MCP loopback with only the lowercased Matrix session key and no current delivery target, then `createCronTool` has no context target and falls back to lowercased session-key inference.

## Solution Assessment

Yes, the best fix is a narrow loopback-context repair rather than another Matrix-specific delivery workaround. The fix should keep core generic by passing or recovering canonical current delivery context before cron infers `delivery.to`.
