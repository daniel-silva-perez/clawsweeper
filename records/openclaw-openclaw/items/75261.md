## Summary

Keep open. Current main still preserves mixed-case cron delivery only when `currentDeliveryContext` reaches `createCronTool`, but the Claude CLI MCP loopback path still does not carry a canonical current target/thread into tool resolution, so Matrix delivery can fall back to the lowercased session key. The closed, unmerged #75303 is useful source work but not an implemented or canonical open fix.

## What This Changes

Reports that `mcp__openclaw__cron action=add` with `sessionTarget: "current"` and no explicit `delivery` lowercases mixed-case Matrix room IDs when invoked through Claude CLI MCP loopback.

## Best Possible Solution

Carry a generic, case-preserving delivery context through Claude CLI MCP loopback into `createOpenClawTools`/`createCronTool`, or resolve the same context from persisted session metadata before cron falls back to session-key parsing. The fix should stay channel-agnostic and include regression coverage for mixed-case Matrix room IDs through the loopback cron path.

## Reproduction Assessment

Yes. The issue provides concrete runtime steps and failure artifacts, and current-main source shows the source-level reproduction path: no loopback current target reaches cron, so implicit delivery can fall back to lowercase session-key parsing.

## Solution Assessment

Yes. Extending the existing delivery-context seam into loopback tool resolution is the narrowest maintainable direction; a Matrix-specific cron casing special case would be less reusable and would leave the generic case-sensitive target gap open.
