## Summary

Keep this issue open. Current main still hardwires native Talk, push-to-talk, and wake transcription to Apple/Android platform recognizers, while the existing configurable STT surfaces cover inbound audio/media and Voice Call rather than native app voice input. The issue comment is accurate: this needs a provider-swappable ASR seam, not only a settings field.

## Best Possible Solution

Keep the issue open. The likely implementation path is to introduce a provider-swappable ASR/transcription layer for native Talk, push-to-talk, and wake flows; keep platform speech as the default; reuse existing OpenAI-compatible multipart helpers or media/realtime transcription provider contracts where they fit; add config/settings for endpoint/auth/model/language/fallback behavior; and cover config parsing plus provider-routing tests.
