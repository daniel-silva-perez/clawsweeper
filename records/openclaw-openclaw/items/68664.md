## What This Changes

The issue asks OpenClaw to migrate or strip legacy `memorySearch` config keys before validation so gateway startup does not fail after schema changes.

## Best Possible Solution

Keep this issue open while maintainers review #68685 or an equivalent focused fix that strips only the obsolete memory-search keys before validation and adds regression coverage; split any update-command automation into a follow-up if maintainers want that broader behavior.

## Reproduction Assessment

Yes. A config containing `agents.defaults.memorySearch.chunkSize`, `chunkOverlap`, or root-level `maxResults` reaches a strict `MemorySearchSchema` with no current migration that removes those keys; I verified this statically in source rather than executing tests because this review is read-only.

## Solution Assessment
