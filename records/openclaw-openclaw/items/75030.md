## Summary

Keep open. Current main still lacks the masked OAuth-profile diagnostic, and this PR is a narrow implementation with regression coverage and no blocking code-review finding; the remaining action is maintainer review plus deciding whether the non-touched CI failures matter before merge.

## What This Changes

The PR adds a final missing-auth hint for config/store auth-profile mode mismatches and covers the `openai-codex:default` OAuth profile masked by `mode: "api_key"`.

## Best Possible Solution

Land a small auth-layer diagnostic once maintainer review is satisfied, while keeping broader legacy OAuth credential-shape normalization tracked in #47964 unless maintainers intentionally expand this branch.

## Reproduction Assessment

Yes. On current main, configure `auth.profiles.openai-codex:default.mode` as `api_key` while the store contains that same profile as `type: "oauth"`; `resolveAuthProfileOrder` filters it as `mode_mismatch`, then `resolveApiKeyForProvider` falls through to the generic missing API key error.

## Solution Assessment

Yes for the masked-mode subcase. The patch preserves credential precedence, only augments the final missing-auth error, and reports profile/provider/mode/type metadata rather than token material; broader alias normalization is a separate policy choice.
