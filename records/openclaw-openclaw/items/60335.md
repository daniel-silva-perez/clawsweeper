## Summary

Keep this issue open. Current main still has the reported Slack thread-media path: text-only thread replies can hydrate the starter message files and then use them as the effective media payload. The same reporter also has open PR #60353 with closing syntax and live dogfooding evidence, so the right maintainer path is to review/land or close that PR, not close this issue early.

## Best Possible Solution

Keep this issue open until PR #60353, or an equivalent Slack plugin patch, is reviewed and merged. The fix should remove starter-media hydration/fallback for thread replies, keep direct reply media working, and add regression coverage for first and later text-only replies plus replies with their own media.
