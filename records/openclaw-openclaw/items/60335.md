## Summary

Keep this open. Current main still has the reported Slack media fallback, and the linked fix PR is open and unmerged, so the issue should remain open until that PR or an equivalent patch lands.

## What This Changes

The issue requests that Slack thread replies include only media attached to the current reply, not media from the thread starter message.

## Best Possible Solution

Land PR #60353 or an equivalent narrow Slack plugin change that stops parent thread-starter media from being attached to reply turns while preserving starter text/history context and direct media on replies, then update the Slack attachment docs and regression tests.

## Reproduction Assessment

Yes. The current source gives a high-confidence reproduction path: a thread reply with no effectiveDirectMedia and a starter containing files downloads the starter files, then prepare.ts uses them as effectiveMedia for the reply.

## Solution Assessment

Yes, the linked direction is the narrowest maintainable fix: keep direct reply media, keep textual thread context, and remove the parent-media fallback for replies. It is not implemented on current main yet.
