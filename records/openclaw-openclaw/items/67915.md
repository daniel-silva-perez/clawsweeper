Closing now would bypass the normal issue/PR lifecycle before the fix lands or is deliberately replaced.

## What This Changes

Fix Control UI local assistant media rendering so server-authorized `MEDIA:` attachments are not permanently marked outside allowed folders when bootstrap media preview roots have not loaded yet.

## Best Possible Solution

Keep this issue open until PR #67916, or an equivalent focused replacement, lands. The desired fix should treat empty `localMediaPreviewRoots` as a not-yet-loaded bootstrap state, fall through to the authenticated server metadata endpoint, preserve the populated-roots client-side denial for genuinely outside paths, and add regression coverage for the empty-roots case.
