## Summary

Current main does not implement the PR’s intended Feishu video-thumbnail behavior. Feishu outbound video is routed as native `media`, but the message content still contains only `file_key`; there is no current thumbnail extraction, thumbnail image upload, or `image_key` inclusion for outbound video bubbles. The PR remains a plausible implementation candidate, though it should be updated against current main and use the existing ffmpeg execution helpers/security constraints.

## Best Possible Solution

Keep this PR open as a focused Feishu bug-fix candidate. Update it against current main, preferably by reusing or extending the existing ffmpeg runtime helper, add tests that verify outbound video uploads a JPEG thumbnail and sends Feishu `media` content with both `file_key` and `image_key`, and then run the targeted Feishu media test lane plus normal changed-gate validation before merge.
