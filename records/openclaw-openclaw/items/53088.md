## Summary

Keep open. Current main still renders the Control UI Sessions `Updated` column with the relative timestamp formatter and has no `YYYY-MM-DD HH:mm` full date-time path on that surface. The issue is also paired with open, unmerged PR #53445, so it should remain open until that PR or an equivalent main-branch fix lands.

## What This Changes

Request that the Control UI Sessions list show each row's updated timestamp as full local date/time, such as `2026-03-20 14:30`, instead of only relative time.

## Best Possible Solution

Land a narrow Control UI change that formats the Sessions `Updated` column as local `YYYY-MM-DD HH:mm`, optionally preserving the relative label as hover/title context, without changing the sessions API, row sorting, or unrelated session-list behavior.
