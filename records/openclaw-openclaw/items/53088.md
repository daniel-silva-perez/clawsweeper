## Summary

Keep this issue open. Current main still renders the Control UI Sessions `Updated` column with the relative timestamp formatter, not a full `YYYY-MM-DD HH:mm` date/time, and the provided discussion/context plus public PR metadata show #53445 is still open and unmerged with closing syntax for this work.

## What This Changes

The issue requests that the Control console Sessions list show full local date and time, such as `YYYY-MM-DD HH:mm`, instead of only a time/relative timestamp.

## Best Possible Solution

Review and land #53445 if acceptable, or replace it with the same narrow Control UI change on main: add a tested local full date-time formatter, render the Sessions `Updated` column as `YYYY-MM-DD HH:mm`, and preserve the existing API, sorting, and session row data contract.
