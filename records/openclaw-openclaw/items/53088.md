## Summary

Keep this issue open. Current main still renders the Control UI Sessions `Updated` column with the relative timestamp formatter, not the requested full `YYYY-MM-DD HH:mm` date/time, and the issue is already paired with open, unmerged implementation PR #53445 plus a fork mirror with the same head work.

## What This Changes

The issue requests that the Control console Sessions list show a full local date/time such as `2026-03-20 14:30` instead of only a relative or time-only timestamp.

## Best Possible Solution

Land a narrow Control UI change that formats the Sessions `Updated` column as local `YYYY-MM-DD HH:mm`, optionally keeps the relative value as hover text, and preserves the existing sessions API, row sorting, and session data contract.
