## Summary

Current main still does not implement the requested Control UI Sessions full date/time display. The Sessions table still computes the Updated column with the relative timestamp formatter and renders that value directly, while the row already carries updatedAt data. The provided GitHub context also shows open, unmerged implementation candidates #53445 and copied fork PR #1305 with closing syntax for this issue, so this should stay open until one lands or maintainers choose a replacement fix.

## Best Possible Solution

Keep this issue open while #53445 remains open. Maintainers should review and land that PR if acceptable, or implement the same narrow change on main: add a tested local YYYY-MM-DD HH:mm formatter, render the Sessions Updated column with it, preserve existing sort/API behavior, and optionally keep the relative timestamp as hover/title context.
