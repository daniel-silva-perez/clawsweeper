## Summary

Keep this issue open. Current main implements the chat/session model picker and a global `agents.defaults.models` model allowlist, but it still does not implement account-scoped provider/model allowlists, per-account defaults, wildcard account policy, or account-aware server-side rejection. The related #7424 is still narrower per-agent allowlist work and is not a substitute for this admin/account policy request.

## Best Possible Solution

Keep this issue open for maintainer design. Define an effective account/user provider-model policy in config/schema, expose it in Control UI admin settings, filter `models.list` and the chat picker by that effective policy, and enforce the same policy in `sessions.patch`, `/model`, `chat.send`/run override paths, and OpenAI-compatible model override paths. Coordinate with #7424 so per-agent and per-account policy compose cleanly instead of creating competing allowlist layers.
