## Summary

Keep this issue open. Current main has a chat/session model picker and a global `agents.defaults.models` model allowlist, with some agent-scoped model handling, but it still does not implement account-scoped provider/model allowlists, wildcard account policy, per-account defaults, or account-aware server-side rejection. The related #7424 remains open and is narrower per-agent allowlist work, so it is not a substitute for this admin/account policy request.

## What This Changes

The issue requests provider and model selection per session/account, with admin-controlled per-account provider/model allowlists, defaults, global fallback defaults, wildcard support, and server-side enforcement.

## Best Possible Solution

Keep this issue open for maintainer design. Define one effective provider/model policy that composes global defaults, account/user overrides, and per-agent policy; add typed config/schema/docs; expose it in Control UI admin settings; filter `models.list` and the chat picker by the effective policy; and enforce the same policy in session patching, slash-command model switching, chat/run override paths, and OpenAI-compatible model override paths. Coordinate with #7424 so per-agent and per-account controls do not diverge.
