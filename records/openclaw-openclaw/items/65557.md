## Summary

Keep open. Current main has a chat/session model picker, per-agent default model routing, and a global `agents.defaults.models` allowlist enforced for session overrides, but it still does not implement the requested per-account/admin provider and model allowlists, account-scoped defaults, wildcard account policy, or account-aware server-side rejection. The related #7424 is narrower per-agent allowlist work, not a full substitute for this account/admin policy request.

## Best Possible Solution

Keep this issue open for a core/admin design and implementation: define an effective account/user provider-model policy in config/schema, expose it in Control UI admin settings, filter `models.list` and the chat picker by the effective policy, and enforce the same policy in `sessions.patch`, `/model`, `chat.send` or run overrides, and OpenAI-compatible model override paths. Coordinate with #7424 so per-agent and per-account allowlists compose cleanly rather than becoming separate policy systems.
