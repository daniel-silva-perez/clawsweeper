## Summary

Keep open. Current main has partial mitigation around stop/abort handling, queue clearing, and denied exec messaging, but it does not fully implement the reported denial-loop behavior. Denied exec completions still become agent-session work by default, the proposed direct PR was closed unmerged as the wrong production seam, and the exact reported Chinese stop phrases are not recognized by the current abort trigger list.

## Best Possible Solution

Keep this issue open as the canonical denial-loop tracker. The best fix should target the production exec approval/system-event/session queue path, not the unused `runTaskInFlow` helper: recognize the reported stop phrases as priority aborts, clear queued session work on stop, prevent repeated denied exec completions from starting new agent turns after repeated denials or an abort, and coordinate any broader async-completion batching/cancellation with #69570.
