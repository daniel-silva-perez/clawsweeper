## Summary

Keep this issue open. Current main has shipped important mitigations for the silent-loss symptom: source-to-runtime write projection now preserves source keys missing from the runtime shape, destructive size-drop writes are rejected, and plugin installs add the installed plugin to an existing restrictive allowlist. However the requested root-schema behavior is not implemented: `OpenClawSchema` is still strict, validation still fails on unknown keys rather than passing them through, and plugin-provided CLI command surfaces are still intentionally blocked when their parent plugin is excluded by `plugins.allow`. The remaining behavior needs maintainer product/security judgment rather than an auto-close.

## Best Possible Solution

Keep this open for maintainer judgment. The best path is to split the remaining decisions: either explicitly keep strict root validation and add a regression proving `plugins install` fails closed without writing when unknown top-level config is present, or deliberately change the root schema/write contract to preserve unknown top-level keys with tests and docs. Separately decide whether plugin-provided bundled CLI commands should continue honoring `plugins.allow` or get a narrow core-owned exemption.
