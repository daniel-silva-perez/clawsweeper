## Summary

Keep this PR open. Current main still lacks the PR's central behavior: the status-card usage lookup does not receive the session's active OAuth profile, while the auth label path does. The PR may need a refresh because current main has already absorbed the unrelated Swift `ModelChoice.alias` protocol update, but the provider-usage/status fix itself is not implemented on main.

## Best Possible Solution

Keep the PR open and ask for a refresh onto current main that drops already-landed generated Swift noise, preserves the preferred OAuth profile through `status-text` -> `loadProviderUsageSummary` -> `resolveProviderAuths` -> plugin/direct OAuth resolution, and includes regression coverage for same-provider selection plus mismatched/cross-provider fallback.
