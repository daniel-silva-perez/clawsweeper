## Summary

Keep open. Current main still has the reported `memory status --deep` probe path, and this PR is a relevant implementation attempt for #74544, but the diff has blocking correctness and policy issues that should be fixed before merge.

## What This Changes

The PR branch adds a budgeted embedding-probe helper for `memory status --deep`, renders a timeout embedding state, extends memory probe result types, adds a probe-budget test, and updates memory CLI docs plus the changelog.

## Best Possible Solution

Put the short diagnostic budget in a manager/provider probe path that avoids or cancels the normal batch timeout, keep non-TTY progress visible with a `log` fallback, and add the required changelog credit plus focused tests covering the real timer behavior.

## Reproduction Assessment

Yes. #74544 provides user-level steps, and source inspection gives a focused reproduction path: cache-miss `memory status --deep` awaits `probeEmbeddingAvailability()`, which calls the batch embedding ping through the local 10-minute timeout path.

## Solution Assessment

No. The current PR-level `Promise.race` is not the best solution because it can return an 8-second timeout while the underlying batch probe and its normal timer remain alive; a cancellable or short-budget manager probe is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Avoid leaving the inner batch timeout alive:** `extensions/memory-core/src/cli.runtime.ts:122`
  - body: This races `manager.probeEmbeddingAvailability()`, but on a cache miss that call has already entered `embedBatchWithRetry(["ping"])` and created the normal local batch timeout. The command can report the 8s timeout while the ref-counted inner timer keeps the process open, so runtime is not actually bounded.
  - confidence: 0.92
- **[P2] Keep verbose non-TTY progress visible:** `extensions/memory-core/src/cli.runtime.ts:732`
  - body: When `--verbose` is set this uses `fallback: "line"`, but `createCliProgress` only renders `line` on TTY streams and returns a no-op for non-TTY unless the fallback is `log`. `openclaw memory status --deep --verbose | cat` would still suppress the probe labels.
  - confidence: 0.9
- **[P2] Add the required changelog credit:** `CHANGELOG.md:9`
  - body: The repo changelog policy requires every added user-facing changelog entry to include at least one `Thanks @author` attribution. This new fix entry only has `Fixes #74544`, so it needs contributor credit before merge.
  - confidence: 0.97
