## Summary

Keep this PR open. Current main still does not document cron output-language troubleshooting, so the PR is not obsolete. The useful change is salvageable, but the current PR should be revised because OpenClaw cron exposes prompt/message fields, not a cron language field or template-substitution workflow.

## Best Possible Solution

Keep the PR open for revision. The best path is a narrow English-only update to docs/automation/cron-jobs.md explaining that cron output language is controlled by explicit prompt text in --message, with an example such as openclaw cron edit <jobId> --message "...Respond in Chinese...". Remove template-file, template-variable, and language-config claims unless those surfaces are implemented first in cron types, Gateway schema, CLI, tests, and docs. Localized docs should stay out of this repo per the current i18n policy.
