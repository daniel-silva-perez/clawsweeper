## Summary

Keep open. The PR addresses a real current-main gap and the latest head fixes the first PDF URL-normalization defects by reusing the shared resolver, but the streaming path still makes transport policy, beta-header, and payload-policy decisions from `model.baseUrl` while the actual request URL can now come from `ANTHROPIC_BASE_URL`. That can misclassify env-configured custom proxies as direct Anthropic traffic and can still block loopback/local gateways, so the patch is not ready to merge as-is.

## What This Changes

The PR exports and tests an Anthropic Messages URL resolver with `baseUrl` → `ANTHROPIC_BASE_URL` → default precedence, then reuses it for native Anthropic PDF requests.

## Best Possible Solution

Resolve the effective Anthropic Messages base URL once, then use that same effective base for URL construction, guarded-fetch policy, private-network allowance, beta-header gating, and payload-policy classification. Add streaming tests for env-configured custom and loopback endpoints, plus native PDF regression tests for env fallback and `/v1` suffix handling, then land the narrow provider-routing fix with the normal changed-surface validation.
