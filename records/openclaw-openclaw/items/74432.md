## Summary

Keep this PR open. The latest branch fixes the earlier PDF URL-normalization review issue, but the central env-var behavior still does not work for normal Anthropic catalog models because those models already carry the public Anthropic base URL before the new helper is called.

## What This Changes

The PR exports and tests an Anthropic Messages URL resolver with explicit `baseUrl` > `ANTHROPIC_BASE_URL` > default precedence and reuses it for native Anthropic PDF requests.

## Best Possible Solution

Land a revised provider-routing fix that distinguishes user/provider-configured Anthropic base URLs from catalog defaults, honors a non-empty `ANTHROPIC_BASE_URL` for default Anthropic streaming and native PDF calls, preserves explicit config precedence, shares one `/v1/messages` normalization path, and adds the required release-note entry.

## Reproduction Assessment

Yes. With a default Anthropic catalog model, `ANTHROPIC_BASE_URL` set, and no `models.providers.anthropic.baseUrl`, the pinned PI catalog still supplies `https://api.anthropic.com` as `model.baseUrl`, so the new resolver never reaches the env fallback.

## Solution Assessment

No. Reusing the shared resolver is the right direction, but the current diff applies the env fallback too late in the call chain; the fix must account for catalog-provided defaults and add production-path streaming/PDF coverage.
