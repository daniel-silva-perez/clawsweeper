## Summary

Keep this PR open. Current main still has the response-before-deferred-close window for device removal, token rotation, and token revoke, while the PR is a targeted gateway auth hardening change with no concrete blocking review finding from this read-only pass.

## What This Changes

The PR adds synchronous device-client invalidation before device removal, token rotation, and token revoke responses, adds a WebSocket per-RPC invalidation guard, and extends gateway request-context and device-handler tests.

## Best Possible Solution

Review and land this PR, or an equivalent narrow mainline patch, so device-token clients are synchronously marked invalid before successful invalidation responses are emitted and any later queued RPC on that socket is rejected before business logic runs.
