## What This Changes

The PR adds synchronous invalidation of affected gateway device-token clients before device removal, token rotation, and token revocation responses, adds a WebSocket dispatch guard for invalidated clients, and extends device-handler/request-context tests.

## Best Possible Solution

Review and land this PR, or an equivalent narrow mainline patch, so device-token clients are marked invalid synchronously before successful invalidation responses and any later queued RPC on that socket is rejected before handler logic runs.

## Reproduction Assessment

The PR’s focused tests reproduce the critical ordering invariant without needing a timing-sensitive live WebSocket pipeline test.

## Solution Assessment

Yes: the branch uses a minimal in-memory flag and a dispatch guard aligned with the existing shared-auth recheck pattern, while preserving response flushing and the queued hard close. I did not find a smaller safer fix for the identified race.
