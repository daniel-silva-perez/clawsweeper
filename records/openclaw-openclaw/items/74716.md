## Summary

Keep this PR open. It has the protected `maintainer` label and current `main` does not already contain the node-hosted MCP path, so it is not cleanup-closeable. The latest PR head still has a concrete lifecycle bug in the new loopback proxy: HTTP aborts are not honored while node MCP startup is pending, which can leave the Mac-hosted Computer Use backend open after the caller disconnects.

## What This Changes

The PR adds node-hosted MCP protocol and proxy plumbing, Mac app Computer Use MCP hosting, Codex thread MCP config injection, pairing approval updates, generated Swift protocol models, regression tests, and docs.

## Best Possible Solution

Keep the PR open, patch the node MCP HTTP proxy so request aborts cancel pending startup and close any created transport/session, add a regression test for abort-before-open, then rerun the targeted Gateway MCP tests, protocol check, and exact-head CI before merge.

## Reproduction Assessment

Yes. On the PR head, start an owner-authenticated `/mcp/node/<node>/<server>` HTTP request, let it send `node.mcp.session.open`, then abort the HTTP client before `node.mcp.session.open.result`; `forward()` is still awaiting startup and the already-aborted signal is not checked before the proxy can continue.

## Solution Assessment

No. The architecture may be the right product direction, but this is not the best mergeable form until the new node MCP HTTP proxy treats request aborts as lifecycle cancellation during startup.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Cancel MCP startup when loopback requests abort:** `src/gateway/node-mcp-http-proxy.ts:116`
  - body: `forward()` awaits `ensureStarted()` before registering any abort-aware waiter or checking `signal.aborted`. If the HTTP client disconnects while `node.mcp.session.open` is pending, the Gateway can still finish opening the Mac-hosted MCP process and retain/send through the abandoned proxy session until timeout or shutdown. Check the abort signal during startup and close the transport/session when startup belongs to an aborted request.
  - confidence: 0.88
