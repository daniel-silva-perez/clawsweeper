## Summary

Keep open. The PR has the protected `maintainer` label and the latest head still has a blocking protocol mismatch in the new macOS node-hosted MCP path: the Swift host can send a numeric `signal`, while the Gateway schema and handler accept only a string or null.

## What This Changes

The PR adds node-hosted MCP session protocol and HTTP proxy plumbing, macOS app hosting for Codex Computer Use packages, owner-only Codex thread MCP config injection, Swift protocol models, docs, and regression tests.

## Best Possible Solution

Make the closed-session protocol contract consistent on the PR branch, then review the native node-hosted MCP architecture, macOS process ownership, generated protocol models, docs, and CI/macOS validation before merge.

## Reproduction Assessment

Not applicable as a feature PR, but the blocking finding has a high-confidence static reproduction path: an uncaught-signal backend exit makes the Swift host send a numeric `signal`, and the Gateway validator accepts only string or null.

## Solution Assessment

No, the PR is not mergeable as-is. The narrow fix is to encode a protocol-valid string signal from Swift, or intentionally broaden the schema and regenerate clients if numeric signals are the desired contract.
