## Summary

Keep open. This is a protected maintainer-labeled XL PR with privileged macOS/Gateway protocol changes, current main does not already implement the native node-hosted MCP path, and the latest head still has a concrete MCP lifecycle bug that should block merge.

## What This Changes

The PR adds node-hosted MCP session protocol and loopback proxy support, macOS Computer Use package hosting and transfer, Codex thread MCP injection, generated Swift protocol models, docs, SDK/runtime types, and regression tests.

## Best Possible Solution

Keep the native node-hosted MCP architecture only after terminal native MCP lifecycle frames settle pending opens immediately, then complete maintainer/platform review of the protocol, macOS host, package validation, generated models, docs, and CI before merge.

## Reproduction Assessment

Yes for the review finding: start a node MCP session and deliver `node.mcp.session.closed` for the same session before `node.mcp.session.open.result`. The current PR head leaves the pending open unresolved in that path, so startup waits for the open timeout instead of failing on the terminal close frame.

## Solution Assessment

No. Routing permission-sensitive Computer Use through OpenClaw.app matches the stated direction, but the current lifecycle handling is not the narrowest maintainable solution until `handleMcpSessionClosed` also clears and resolves matching pending opens with a closed/error result.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Settle pending MCP opens when the node closes:** `src/gateway/node-registry.ts:452-459`
  - body: `handleMcpSessionClosed` only removes the active session and notifies `onClosed`; it leaves a matching entry in `pendingMcpOpens`. If the native host exits and sends `node.mcp.session.closed` before or instead of `node.mcp.session.open.result`, `NodeMcpClientTransport.start()` keeps waiting until the open timeout even though the session has already reached a terminal state. Clear the pending open and resolve it with the close error in this handler, with a regression test for closed-before-open.
  - confidence: 0.86
