## What This Changes

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Forward mcpServers from pair requests:** `src/gateway/server-methods/nodes.ts:538-552`
  - body: `NodePairRequestParamsSchema` now accepts `mcpServers`, but the `node.pair.request` handler still omits `mcpServers: p.mcpServers` when it calls `requestNodePairing`. Direct pair requests therefore drop the declared MCP surface, do not record it for approval, and do not trigger the intended admin-scope requirement; later reconnects will treat the same server as unapproved again.
  - confidence: 0.91
- **[P2] Call onclose when closing the transport:** `src/gateway/node-mcp-client-transport.ts:70-83`
  - body: The MCP SDK `Transport` contract says `onclose` should be invoked when `close()` is called, but this implementation only marks the transport closed and sends the node close frame. MCP clients or protocol layers waiting for the close callback will miss caller-initiated shutdowns even though node-initiated closes do call it.
  - confidence: 0.86
