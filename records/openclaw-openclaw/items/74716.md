## Summary

Keep open. This protected maintainer-labeled PR is still useful and current main does not already implement its native node-hosted MCP path, but PR head still has a blocking protocol mismatch in the macOS closed-session frame.

## What This Changes

The PR adds node-hosted MCP session protocol and proxy plumbing, macOS app hosting for Codex Computer Use, owner-only Codex thread MCP config injection, Swift protocol models, docs, and regression tests.

## Best Possible Solution

Fix the closed-session `signal` contract on the PR branch, then have a maintainer review the native node-hosted MCP architecture, macOS process ownership, generated protocol models, docs, and CI/macOS validation before merge.

## Reproduction Assessment

Not applicable as a feature PR, but the review finding has a high-confidence static reproduction path: an uncaught-signal backend exit makes the Swift host send a numeric `signal`, while the Gateway validator accepts only a string or null.

## Solution Assessment

No, this PR is not in its best mergeable form as-is. The narrow fix is to encode a protocol-valid string signal from Swift, or deliberately broaden the schema and regenerate clients if numeric signals are the intended contract.
