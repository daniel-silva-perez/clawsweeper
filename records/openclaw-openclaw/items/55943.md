## Summary

Keep open. Current main still has the reported gap: subagent completion emits the typed plugin `subagent_ended` lifecycle hook, but the bundled `session-memory` internal hook is only registered for and implemented around `command:new` and `command:reset`, so it will not save a subagent transcript when a normal subagent run reaches `done`.

## What This Changes

The issue requests that the bundled session-memory hook also save memory files when subagent sessions end, matching the behavior users see after `/reset` in a main session.

## Best Possible Solution

Teach the bundled session-memory feature to consume a well-defined subagent end signal with enough context to resolve the child session transcript and workspace, then update the hook metadata/docs and add focused tests proving a completed `sessions_spawn` run writes exactly one memory file for the child session.
