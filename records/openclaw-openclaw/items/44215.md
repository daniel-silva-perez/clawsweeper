## Summary

Keep this PR open. Current main still lacks Windows-specific PATH bootstrap directories in `ensureOpenClawCliOnPath`, so the central requested behavior is not implemented. The submitted PR is still not ready to merge because it changes command-resolution behavior in a PATH security boundary, prepends user-writable Windows locations/WindowsApps ahead of the existing PATH, has test coverage/delimiter gaps, and carries an unrelated schema-label hunk that is already present on main.

## Best Possible Solution

Keep this PR open for maintainer-guided rework. The fix should preserve the hardened PATH model, add only carefully chosen Windows system/runtime bootstrap directories, avoid prepending user-writable package-manager or WindowsApps aliases by default, handle Windows semicolon delimiters deterministically, remove the unrelated schema-label hunk, and add focused Windows regression tests. The related `env.vars.PATH` part of #19910 should remain governed by the existing rejected-PATH-override and `tools.exec.pathPrepend` policy.
