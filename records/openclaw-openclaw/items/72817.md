## Summary

Keep open. Current main still lacks the PR's guarded retry path for source public artifacts, while the PR is focused, matches a real bundled source-artifact pattern, and addresses the earlier bot concern by narrowing the Jiti fallback guard. The remaining action is maintainer review of this loader behavior, not cleanup closure.

## What This Changes

This PR adds a guarded Jiti fallback for bundled source public-surface loads with emitted-style `.js` imports, adds regression tests and mock cleanup, and extracts bundled entry load-profile emission into a helper.

## Best Possible Solution

Have a maintainer review the guarded fallback as the intended plugin-loader boundary, keep the focused regression coverage, and land this PR if that execution-path behavior is accepted.

## Reproduction Assessment

Yes. The source check is high-confidence: current main can select a source `.ts` public artifact that imports a sibling with a `.js` specifier, then calls `sourceArtifactRequire` without a retry path; the PR adds a unit test that models that module-not-found path.

## Solution Assessment

Yes. The PR preserves the existing source-require fast path and only retries via the existing Jiti loader for module-not-found errors that reference a `.js` specifier, which is narrower than removing the fast path or broadly retrying every failure.
