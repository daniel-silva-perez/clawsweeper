## Summary

Keep open. Current main still lacks the guarded source-artifact require-to-Jiti fallback, while this PR is active, mergeable, and has green CI; I found no blocking review finding, but the loader code-execution path deserves maintainer review before landing.

## What This Changes

This PR adds a guarded Jiti fallback for bundled source public-artifact loads with emitted-style `.js` imports, adds focused regression tests and mock hygiene, and extracts bundled entry load-profile emission into a helper.

## Best Possible Solution

Have a maintainer review the guarded fallback as the intended product boundary, keep the regression tests, and land the PR if that execution-path behavior is accepted; no separate replacement branch is needed while this PR remains clean and mergeable.

## Reproduction Assessment

Yes. Current main selects source public artifacts and then directly calls `sourceArtifactRequire`, and bundled source artifacts such as `extensions/anthropic/provider-policy-api.ts` use emitted-style `.js` sibling imports; the PR's new test models that module-not-found path and verifies the Jiti fallback.

## Solution Assessment

Yes. The proposed fix preserves the existing source-require fast path and only retries through the existing Jiti loader for module-not-found errors that mention a `.js` specifier, which is narrower than removing the fast path or broadly retrying every failure.
