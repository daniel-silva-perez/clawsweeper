## Summary

Keep open. Current main still lacks the guarded fallback from source public-artifact `require()` to Jiti, while the PR is open, mergeable, focused, covered by regression tests, and has green check runs; I found no blocking correctness or security finding.

## What This Changes

This PR adds a guarded Jiti fallback for bundled source public-artifact loads with emitted-style `.js` imports, adds focused loader/mock regression coverage, and extracts bundled entry load-profile emission into a helper.

## Best Possible Solution

Have a maintainer review the guarded loader fallback as an accepted plugin-runtime boundary and land this PR if that tradeoff is intended, preserving the focused regression tests and existing changelog credit.

## Reproduction Assessment

Yes. Current main directly requires source public artifacts when a TS require hook exists, and a real bundled source artifact imports a sibling with a `.js` specifier; the PR’s new test models that module-not-found path and verifies the Jiti fallback.

## Solution Assessment

Yes. The PR preserves the existing source-require fast path, retries only for module-not-found errors that point at a `.js` specifier, and explicitly keeps syntax errors on the original path instead of broadening every failure into Jiti.
