## Summary

Keep open. Current main still lacks the guarded source-public-artifact require fallback that this PR adds, while the PR is focused, has regression coverage for the known emitted-style `.js` import failure, and the latest PR head reports green checks with no blocking review finding.

## What This Changes

The PR adds a guarded Jiti fallback when source public-artifact `require` fails on emitted-style `.js` imports, adds public-surface loader regression tests, extracts bundled entry load profiling, and adjusts a few extension test mocks/awaits.

## Best Possible Solution

Have a maintainer review the guarded source-artifact Jiti retry as an accepted bundled-plugin loader boundary and land this PR if the message-based guard is acceptable, preserving the focused regression tests and existing changelog credit.

## Reproduction Assessment

Yes. Current main directly requires source public artifacts when a TypeScript require hook exists, and a real source public artifact imports a sibling with a `.js` specifier; the PR's tests model that module-not-found path and verify the Jiti fallback plus syntax-error preservation.

## Solution Assessment

Yes. The PR keeps the existing source-require fast path, retries only for module-not-found errors that look like emitted-style `.js` specifier misses, and avoids turning unrelated syntax failures into Jiti retries.
