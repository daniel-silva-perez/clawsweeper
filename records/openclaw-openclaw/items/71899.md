## Summary

Keep open. Current main still has the reported nested workspace default path, and this PR is an active implementation for that bug, but the proposed resolver-level guard is not ready because it still misses profiled setup workspaces when the later command runs under a different profile environment.

## What This Changes

The PR changes agent workspace resolution to treat setup-created main workspaces as managed rather than shared bases for added agents, adds resolver regression tests, documents peer-level workspaces, and adds a changelog entry for #71889.

## Best Possible Solution

Land one canonical fix for #71889 that makes brand-new `agents add` wizard defaults peer-level, preserves explicit per-agent workspaces and intentional shared-base runtime behavior, covers profile transitions directly, and includes wizard `initialValue` coverage plus resolver/helper edge coverage.

## Reproduction Assessment

Yes. The source path is high-confidence: setup persists `agents.defaults.workspace`, the wizard passes `resolveAgentWorkspaceDir(cfg, agentId)` to the prompt, and the current resolver joins that fallback with the non-default agent id.

## Solution Assessment

No. This patch fixes the default-profile setup case but changes the runtime resolver broadly and still misses a setup workspace saved under a different profile; a narrower wizard prefill helper or a fully profile-aware managed-workspace check would be safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Recognize profiled setup workspaces before nesting:** `src/agents/agent-scope-config.ts:72-73`
  - body: The guard only treats `<stateDir>/workspace` and the current profile's default workspace as managed. If setup ran with `OPENCLAW_PROFILE=dev`, it can persist `<stateDir>/workspace-dev`; later running `agents add work` without that profile leaves this fallback unrecognized, so the resolver still returns `<stateDir>/workspace-dev/work`, preserving the nested layout this PR is meant to avoid.
  - confidence: 0.84

## Security Review

Status: needs_attention
