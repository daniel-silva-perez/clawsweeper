## Summary

Keep this PR open. Current main still has the legacy file-watcher behavior where `gateway.reload.mode: "hot"` warns and returns for restart-required plans, and current config/docs surfaces do not expose `gateway.reload.fallbackToRestart`. The PR remains a focused implementation candidate, but it needs maintainer review and alignment with the newer config contract/docs/generated schema surfaces before merge.

## Best Possible Solution

Keep this PR open and review it as a real gateway fix candidate. The likely path is to rebase it onto current main, confirm the desired default for hot-mode restart fallback, wire `fallbackToRestart` through all public config contract surfaces and docs, and keep targeted tests for hot fallback, opt-out legacy skip behavior, and unchanged hybrid behavior.
