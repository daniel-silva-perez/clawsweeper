## Summary

Keep this PR open. Current main does not implement the proposed `gateway.reload.fallbackToRestart` behavior: file-watcher hot mode still warns and returns without restarting for restart-required plans. The PR remains a plausible focused gateway fix, though it needs maintainer review against the newer config contract/docs/generated schema surfaces.

## Best Possible Solution

Keep this PR open and review it as a real implementation candidate. The likely path is to rebase it onto current main, decide the desired default for hot-mode restart fallback, update all public config contract surfaces and docs for the new option, and keep/extend the focused gateway reload tests around hot fallback, opt-out, and unchanged hybrid behavior.
