## Summary

Keep this PR open. Current main still compares the configured bound node and per-call requested node as raw strings before canonical resolution, while the config/docs contract allows node selectors by id/name and the shared resolver supports multiple selector forms. The PR remains a focused fix candidate for the linked node-exec bug, with the main merge-review concern being the command-execution boundary’s fail-closed behavior for stale or ambiguous bindings.

## What This Changes

The PR canonicalizes both `boundNode` and `requestedNode` before node-exec binding comparison and adds regression tests for display-name/prefix matches, mismatch rejection, not-found errors, and stale bound-node handling.

## Best Possible Solution

Land this PR after normal maintainer review, or port the same canonicalization and tests onto current main: resolve both configured and requested node selectors through the shared node resolver, compare canonical ids, reject different nodes, preserve not-found causes, and keep stale configured bindings from selecting a default node.
