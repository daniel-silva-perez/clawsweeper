## Summary

Keep this PR open. Current main still performs the bound-node guard on raw selector strings before node lookup, while the repo’s config and docs allow node ids or names; the PR appears to address that remaining bug with focused production code and regression tests, but it needs maintainer review because it changes node command-execution target selection.

## What This Changes

The PR canonicalizes configured and requested node selectors through `resolveNodeIdFromList` before enforcing node exec binding, reuses resolved ids for target selection, preserves requested-node lookup errors, and adds regression coverage.

## Best Possible Solution

Land this PR after normal maintainer review, or port the same canonicalization and tests onto current main: resolve configured and requested node selectors through the shared resolver, compare canonical ids, reject different nodes, preserve lookup causes, and keep stale configured bindings from selecting a default node.
