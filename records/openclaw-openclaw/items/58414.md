## Summary

Keep open. Current main fixes or hardens several parts of the report, especially `openclaw gateway` / `openclaw gateway run` routing and supervised stale-lock/update recovery, but it still does not implement the concrete macOS LaunchAgent request: generated gateway plists have no `StandardInPath=/dev/null`. The related launchd hang tracker #46153 is also still open, so this should be narrowed or consolidated by a maintainer rather than auto-closed.

## Best Possible Solution

Keep this open and narrow it. The best maintainer path is either to add `StandardInPath=/dev/null` to `buildLaunchAgentPlist` with launchd plist tests and repair/rewrite coverage, or explicitly consolidate this into #46153 after preserving the self-update trigger, v2026.3.28 routing regression history, and manual workaround details. Do not close as implemented while the requested plist key is absent.
