## Summary

Current main still implements persistent user model overrides only. It does not parse `/model <name> --turns N`, store a turn budget, warn near expiry, auto-clear the override, notify on revert, or validate a `models.limits` / `maxTurns` chat-model config contract. The prior ClawSweeper review reached the same conclusion, and the newer main snapshot still matches it.

## Best Possible Solution

Keep this open for maintainer design. A complete solution likely belongs in core model-selection and session state: parse `/model <provider/model> --turns N`, validate any configured per-model budget shape, persist a temporary override budget, decrement it after completed user turns, warn near expiry, clear the override through the existing helper when the budget expires, send a revert notification, and document/test the exact counting semantics.
