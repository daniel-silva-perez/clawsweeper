## Summary

Current main still does not implement per-model turn limits or auto-revert for `/model`. The existing behavior remains persistent session model overrides, with no `--turns` parsing, countdown state, warning/revert notification, or chat-model `models.limits`/`maxTurns` config contract.

## Best Possible Solution

Keep this open for maintainer design. A complete solution likely belongs in core model directive parsing and session state: parse `/model <provider/model> --turns N`, validate a documented per-model budget config shape, persist a temporary override budget, decrement only on the chosen completed-turn semantics, warn near expiry, clear the override via the existing helper when the budget expires, send a revert notification, and update docs/tests.
