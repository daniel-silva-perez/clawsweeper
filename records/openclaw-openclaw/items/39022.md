## Summary

Current main still supports persistent user model overrides, but not per-model turn budgets or automatic revert. `/model` parsing returns only model/profile/runtime, native command metadata exposes only a single model argument, session state has no countdown/revert fields, and the documented/configured model surfaces do not include the proposed `models.limits` or `maxTurns` contract.

## Best Possible Solution

Keep this open for maintainer design. A complete solution likely belongs in core model-selection/session state: parse `/model <provider/model> --turns N`, persist a temporary override budget, decrement it only after completed user turns, warn near expiry, clear the override through the existing helper when the budget expires, send a revert notification, and add a validated config contract once maintainers settle the shape. If maintainers prefer plugin extensibility, first define a safe plugin API for model-selection interception and per-turn session accounting.
