## Summary

Keep this issue open. Current main still implements persistent session model overrides, but it does not implement `/model <name> --turns N`, turn countdown persistence, warning/revert notifications, or the proposed `models.limits` / per-model `maxTurns` config contract. The previous ClawSweeper comment reached the same conclusion, and the current checkout has not added the requested behavior since then.

## Best Possible Solution

Keep this issue open for maintainer design. A complete solution likely belongs in core model-selection/session state: parse `/model <provider/model> --turns N`, persist temporary override budget state, decrement after completed user turns, warn near expiry, clear the override through the existing helper when the budget expires, send a revert notification, and add a validated config contract only after maintainers settle the shape. If maintainers prefer plugin extensibility, first define a safe plugin API for model-selection interception and per-turn session accounting.
