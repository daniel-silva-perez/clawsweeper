## Summary

Keep this issue open. Current main still routes Discord inbound user turns through a Carbon MessageCreateListener-only path, does not register MessageUpdateListener/MESSAGE_UPDATE handling, has no edited-message config or docs surface, and the replay guard still keys by account/channel/message id, so a same-message edit would need explicit edit-aware dispatch and dedupe semantics before it can trigger a second agent turn.

## Best Possible Solution

Keep this issue open and implement the feature in the bundled Discord plugin. Add an opt-in edited-message config/type/schema/docs surface, register a Carbon MessageUpdateListener path, normalize eligible edited-message payloads through the existing Discord preflight and inbound worker pipeline, preserve current bot/self/access/mention filters, make replay dedupe edit-aware, and add focused tests for edited DMs, mention-added guild messages, unchanged-content updates, bot/self edits, duplicate update delivery, and config validation. Carry over #48894's requireMention/new-mention-on-edit acceptance criteria; coordinate with #53654 only if delete-to-cancel is intentionally included.
