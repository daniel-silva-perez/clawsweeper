## Summary

Keep open. Current main still has no Discord MESSAGE_UPDATE/messageUpdate inbound path, no edited-message config or docs surface, and the existing same-message replay guard would suppress an edit unless the implementation makes updates edit-aware. The issue is active, not protected, and is a good focused fix-PR candidate rather than a cleanup close.

## Best Possible Solution

Keep this issue open and implement the feature in the bundled Discord plugin. Add an opt-in edited-message config/type/schema/docs surface, register a Carbon MessageUpdateListener path, route eligible edited payloads through the existing Discord preflight and inbound worker pipeline, make replay dedupe edit-aware, and cover DMs, guild mention edits, unchanged updates, bot/self edits, duplicate delivery, and config validation. Treat #38505 as overlapping and carry forward #48894's new-mention-on-edit acceptance criteria; coordinate with #53654 only if delete-to-cancel is intentionally included.
