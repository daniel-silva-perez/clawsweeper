## Summary

Keep open for maintainer review: current main still has the reported Docker build-time remote Bun installer, and this PR is a useful active fix, but it touches Docker supply-chain behavior and still needs maintainer validation plus CI/Docker build proof before merge.

## What This Changes

The PR replaces the root Dockerfile Bun installer with a digest-pinned `oven/bun` source stage and expands Docker image digest tests to cover ARG-backed `FROM` stages.

## Best Possible Solution

Land a reviewed Docker hardening change that removes build-time execution of the remote Bun installer, keeps the Bun source image digest-pinned and aligned with CI, tightens the new ARG-backed digest test helper, and closes #74356 through the merge after CI and Docker build proof pass.

## Reproduction Assessment

Yes. The issue is reproducible by static inspection of current main: `Dockerfile:47` still executes the remote Bun installer with `curl ... | bash`.

## Solution Assessment

Yes, the proposed direction is the narrow maintainable fix: copy Bun from a digest-pinned official image stage instead of executing a network script during the build. The test helper should be tightened to respect Docker ARG scope, and maintainers should require Docker build or install-smoke validation before merge.

## Review Findings

Overall correctness: patch is correct

Overall confidence: 0.8

Full review comments:

- **[P3] Limit ARG default collection to global scope:** `src/docker-image-digests.test.ts:36-43`
  - body: Docker only makes ARG defaults declared before the first `FROM` available for later `FROM` substitution. This helper now scans ARG instructions from every stage, so a future stage-local ARG could satisfy the digest assertion even though Docker would not resolve that `FROM` as written. Stop collecting defaults once the first `FROM` is reached, or model ARG scope explicitly.
  - confidence: 0.74
