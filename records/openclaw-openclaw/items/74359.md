## Summary

Keep this PR open for maintainer review. Current main still executes the Bun installer via `curl | bash`, while the PR moves the Docker build to a digest-pinned `oven/bun` source stage and looks directionally correct; the remaining blockers are maintainer/security ownership, current failing CI checks, and Docker build/install-smoke proof rather than conservative cleanup closure.

## What This Changes

The PR adds a digest-pinned `OPENCLAW_BUN_IMAGE` source stage to the root Dockerfile, copies `/usr/local/bin/bun` into the build stage instead of running the remote installer, and expands Docker image digest tests to inspect all ARG-backed `FROM` stages.

## Best Possible Solution

Land a reviewed Docker hardening change that removes build-time execution of the remote Bun installer, keeps the Bun source image digest-pinned and aligned with CI, preserves multi-arch behavior, keeps digest coverage for ARG-backed stages, and closes #74356 through the merge after CI plus Docker build or install-smoke proof passes.

## Review Findings

Overall confidence: 0.78

Full review comments:

- **[P3] Limit ARG defaults to Dockerfile-global scope:** `src/docker-image-digests.test.ts:36-43`
  - body: Docker only makes ARG defaults declared before the first `FROM` available for later `FROM` substitution. This helper now scans ARG instructions from every stage, so a future stage-local ARG could satisfy the digest assertion even though Docker would not resolve that `FROM` as written. Stop collecting defaults once the first `FROM` is reached, or model ARG scope explicitly.
  - confidence: 0.74
