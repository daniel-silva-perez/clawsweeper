## Summary

Keep open. Current main still installs Bun in the root Dockerfile by piping `https://bun.sh/install` into `bash`, so this PR is not obsolete; the PR head now appears to remove that path with a digest-pinned Bun image stage, but it is an active external-contributor Docker supply-chain change that needs maintainer/security review and image-build validation before merge.

## What This Changes

This PR changes the root `Dockerfile` to add a digest-pinned `oven/bun:1.3.9` source stage and copy `/usr/local/bin/bun` into the build stage instead of running the Bun installer script from `bun.sh`.

## Best Possible Solution

Land a Docker hardening change that removes build-time execution of the remote Bun installer, uses a digest-pinned and updateable Bun source consistent with the existing Docker image policy, extends digest coverage to all relevant `FROM` stages, and validates the root Docker image path before closing #74356.
