## Summary

Keep this PR open. Current main still uses the Mattermost bundled-entry sync loader for slash route registration, keeps slash command state in a module-local Map, and has no equivalent static-import or shared-state fix. The related alternate PR #69038 is closed/unmerged and was treated as duplicate work in favor of this PR plus the canonical bug #68113, so it does not supersede this implementation candidate.

## Best Possible Solution

Keep this PR open for maintainer review and consolidate it with #68113. The best path is to land one focused Mattermost fix that guarantees slash route registration and monitor activation share the same account state across supported loader paths, adds a targeted dual-loader regression test, and explicitly resolves the import-graph tradeoff before closing the linked bug.

## Risks / Open Questions

- The PR still lacks the dual-loader regression test described in its body, so maintainers should add coverage before or with the final fix if feasible.
- The static import is a plausible functional fix but broadens the Mattermost entrypoint import graph into `slash-state.ts` and `slash-http.ts`; maintainers should decide whether this tradeoff is acceptable or whether a narrower runtime seam is preferable.
- The current loader's native-require fast path may mitigate some packaged dist installs, but source and fallback loader paths remain relevant and should be covered by the final design.
