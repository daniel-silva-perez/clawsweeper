## Summary

Current main still does not provide an admin-facing WhatsApp contacts/permissions view. Existing surfaces are partial: `pairing list` is pending-only, `directory` is lookup-only and config-backed, channel/status output is account-level, and the related #56621 issue is a narrower approved-pairing list/revoke request that does not cover group policy or per-contact activity.

## Best Possible Solution

Keep this open and implement a conservative read-only admin surface first, likely `openclaw contacts list --channel whatsapp [--account <id>] [--json]`. The implementation should merge configured `allowFrom`, account-scoped pairing allow-store entries, `dmPolicy`, `groupPolicy`, `groupAllowFrom`/`groups`, and whatever per-sender activity data is actually available. Coordinate the approved-pairing subset with #56621, and add revoke/edit verbs only after the read model, privacy posture, and account-scoping behavior are clear.
