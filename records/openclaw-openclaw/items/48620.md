## Summary

Current main has adjacent pairing, directory, channel status, and config/UI surfaces, but none provides the requested approved WhatsApp contact/permission management view. Pairing list is pending-only, approved senders are stored separately in allowFrom files, WhatsApp group authorization is a separate policy surface, and status/directory output does not merge per-contact DM/group permission state with last-active data.

## Best Possible Solution

Keep this open and implement a conservative admin-facing contact surface, likely starting read-only: merge configured `allowFrom`, account-scoped pairing allow-store entries, WhatsApp `dmPolicy`, `groupPolicy`, `groupAllowFrom`/`groups`, and available per-sender activity into a channel-aware `openclaw contacts list --channel whatsapp` or equivalent expanded admin command. Add management verbs only after the read path and security/privacy model are clear.

## Risks / Open Questions

- A future implementation needs careful privacy handling because the requested output would expose phone numbers, allowlists, and activity timestamps.
- The feature should keep DM pairing approvals and group sender/group membership authorization distinct so admins do not infer group access from a DM approval alone.
