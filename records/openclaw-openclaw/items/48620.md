## Summary

Current main still lacks the requested admin-facing view for approved WhatsApp contacts and their permissions, so this should stay open. The available surfaces remain split: pairing only lists pending requests, directory is lookup/config-backed, and WhatsApp policy state is documented and implemented across separate DM and group allowlists; the related approved-pairing request is narrower and still open.

## What This Changes

Request a CLI/admin command, such as `openclaw contacts list --channel whatsapp`, that lists paired or connected WhatsApp sender IDs with DM policy, group policy, and last-active information.

## Best Possible Solution

Add a read-only, account-scoped admin view that merges configured `allowFrom`, pairing allow-store entries, `dmPolicy`, `groupPolicy`, `groupAllowFrom` or group membership state, and any reliable per-sender activity signal. Keep revoke/edit behavior in a separate scoped pairing-management follow-up unless maintainers intentionally combine the surfaces.

## Reproduction Assessment

Not applicable as a feature request. The missing surface is reproducible by inspecting current main: no `contacts` CLI command exists, `pairing list` is pending-only, and the existing directory/status/docs surfaces do not combine approved senders with DM/group permissions and activity.

## Solution Assessment

No proposed patch is under review. A conservative read-only list command is the best starting direction, but maintainers should first settle naming, account scoping, privacy posture, and how it coordinates with #56621.
