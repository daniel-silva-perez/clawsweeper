## Summary

Current main still lacks an admin-facing command or UI that lists approved WhatsApp contacts together with DM policy, group policy, and per-contact last-active data. Adjacent surfaces exist, and #56621 tracks a narrower approved-pairing list/revoke workflow, but that does not cover the broader contact/permission/activity view requested here.

## Best Possible Solution

Keep this open and implement a conservative admin contact/permission surface, likely starting read-only: `openclaw contacts list --channel whatsapp [--account <id>] [--json]` or an equivalent admin command that merges config `allowFrom`, account-scoped pairing allow-store entries, WhatsApp `dmPolicy`, `groupPolicy`, `groupAllowFrom`/`groups`, and any available per-sender activity. Coordinate the approved-pairing subset with #56621, and add management verbs only after the read model and privacy/security behavior are clear.
