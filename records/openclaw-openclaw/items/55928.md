## Summary

Keep this PR open. Current main still does not implement the PR's intended HTTP history guard: the `/sessions/:key/history` handler authorizes `operator.read`, resolves the requested session key, loads the store, and returns history without rejecting subagent, cron, or ACP internal namespaces. The PR discussion also shows the contributor addressed the bare `cron:*` bypass and narrowed the change back to HTTP after review found WebSocket `chat.history` is still needed by internal subagent control flows.

## Best Possible Solution

Review and land this focused HTTP guard, or replace it with an equivalent main-branch patch that rejects raw and canonicalized internal subagent, cron, and ACP session keys before any HTTP history store read while preserving WebSocket `chat.history` for internal runtime flows that still depend on it.
