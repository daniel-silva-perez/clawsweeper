## Summary

Keep this PR open. Current main still lacks an internal-session namespace guard on the active HTTP session-history endpoint, and the latest PR diff addresses the observed raw and canonical subagent, cron, and ACP cases without reintroducing the earlier WebSocket regression.

## What This Changes

The PR adds an internal-session key predicate, rejects HTTP `/sessions/:key/history` requests for subagent, cron, and ACP namespaces using raw and canonical keys, and adds regression coverage for those blocked keys.

## Best Possible Solution

Land this focused HTTP-only guard, or an equivalent main-branch patch, with a central internal-session predicate that covers raw and canonical subagent, cron, and ACP keys while preserving WebSocket `chat.history` for internal runtime flows until a separate caller-trust distinction exists.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: seed an internal entry such as `agent:main:cron:daily`, request `/sessions/cron%3Adaily/history` with `operator.read`, and current main canonicalizes the key before reading and returning history instead of rejecting it.

## Solution Assessment

Yes. The latest PR shape is the narrowest maintainable fix because it blocks the external HTTP read surface and preserves the WebSocket path that current subagent-control code still depends on.
