## Summary

Keep this PR open. Current main still does not implement the PR's intended HTTP history guard: `/sessions/:key/history` is an active REST surface, and after auth/scope checks it resolves the requested key, loads the session store, reads messages, and returns history without rejecting subagent, cron, or ACP internal namespaces. The PR discussion is useful: the contributor addressed the bare `cron:*` bypass and narrowed the change back to HTTP after review showed WebSocket `chat.history` is still used by internal subagent flows.

## Best Possible Solution

Review and land this focused HTTP guard, or replace it with an equivalent main-branch patch that rejects raw and canonicalized subagent, cron, and ACP session keys before any REST history store read while preserving WebSocket `chat.history` for internal runtime flows that still depend on it. Keep the related session-override PR separate unless maintainers choose to share the helper between both HTTP hardening changes.
