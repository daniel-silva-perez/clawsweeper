The branch exports a Telegram bindings adapter, accepts positive direct-chat IDs when compiling and matching Telegram ACP configured bindings, preserves topic matching and bare negative group rejection, adds adapter tests, and updates one ACP binding doc reference.

## Best Possible Solution

Keep the PR open, rebase it onto current main, retain the channel-owned adapter fix, add or preserve focused coverage for direct DM compile/match plus message-context routing, and update the current Telegram and ACP Agents docs so they no longer describe Telegram persistent ACP bindings as topic-only.
