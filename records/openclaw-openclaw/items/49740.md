## Summary

Keep open. Current main still lacks the requested per-job recurring cron retry controls, and PR #51001 is an open, unmerged implementation candidate that explicitly links this issue for closure. The existing cron retry/backoff behavior is not equivalent to the requested `--retry-count`, `--retry-delay`, and optional backoff flow for daily recurring jobs.

## Best Possible Solution

Keep this issue open and review PR #51001 as the active implementation candidate. A complete fix should add a durable per-job recurring retry contract, CLI add/edit flags, scheduler state that retries before the next natural run when appropriate without cadence drift, docs, and focused regression coverage. Close this issue only after that PR or an equivalent implementation lands.
