## Summary

Keep open. Current main does not implement the requested per-job cron retry controls, and PR #51001 is an open, unmerged implementation candidate that explicitly closes this issue. Per cleanup policy, the issue should stay open until that PR is reviewed and merged or closed.

## Best Possible Solution

Keep the issue open and review PR #51001 as the active implementation candidate. A complete solution should add a durable per-job retry contract, CLI add/edit flags, scheduler behavior that retries before the next natural run when appropriate without cadence drift, docs, and targeted regression coverage. Close the issue only after that PR or an equivalent implementation lands.

## Evidence

- **Current recurring behavior:** Recurring cron failures only apply the existing error backoff as a lower bound and choose the later of the natural next run and the backoff delay. For a daily cron, that still means the natural next daily slot can win instead of a configurable retry before the next scheduled run.
