## Summary

Keep open. Current main still lacks the requested per-job recurring cron retry controls, and PR #51001 is an open, unmerged implementation candidate that explicitly links this issue for closure. Existing recurring error handling applies an error-backoff floor but can still wait until the next natural daily run, and the Gateway schema/CLI do not expose `--retry-count`, `--retry-delay`, or `--retry-backoff`.

## Best Possible Solution

Keep this issue open and review PR #51001 as the active implementation candidate. A complete fix should add a durable per-job recurring retry contract, CLI add/edit flags, protocol/schema support, scheduler state that retries before the next natural run without cadence drift, docs, and focused regression coverage. Close this issue only after that PR or an equivalent implementation lands.
