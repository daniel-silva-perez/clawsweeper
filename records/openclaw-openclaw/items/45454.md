## Summary

Keep this PR open. Current main at bdfb408ce6eeb724a638c106104dbbcae50b612c still has the early-abort responsePrefix interpolation gap, the linked bug remains open, and this draft PR plus #45315 are implementation candidates that need maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review or a rebased equivalent. The best fix is a current-main implementation that threads session-aware prefix context through createReplyPrefixContext/createChannelReplyPipeline and routeReply, seeds early abort replies from accurate runtime metadata before delivery, preserves onModelSelected as the later authoritative override, and adds focused regression tests for direct fast-abort replies, routed fast-abort replies, stored metadata, and post-selection override behavior. Maintainers should then choose between this PR and #45315, land one canonical fix, and close the remaining duplicate candidate after merge.
