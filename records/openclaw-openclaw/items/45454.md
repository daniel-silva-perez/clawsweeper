## Summary

Keep this PR open. Current main at e476523082a6fff3a0c3048077d2f0e52413fbf2 still has the early-abort responsePrefix interpolation gap, the linked bug remains open, and this draft PR plus #45315 are implementation candidates that need maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review or a rebased equivalent. The best fix is a current-main implementation that threads session-aware prefix context through createReplyPrefixContext/createChannelReplyPipeline and routeReply, seeds early abort replies from accurate runtime metadata or an accepted fallback before delivery, preserves onModelSelected as the later authoritative override, and adds focused regression tests for direct fast-abort replies, routed fast-abort replies, stored/default metadata, and post-selection override behavior. Maintainers should then choose between this PR and #45315, land one canonical fix, and close the remaining duplicate candidate after merge.
