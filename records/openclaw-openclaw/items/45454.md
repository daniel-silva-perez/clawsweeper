## Summary

Keep this PR open. Current main still has the reported early-abort responsePrefix gap: prefix context is not seeded with model/thinking metadata before fast abort, the abort path returns before onModelSelected runs, and the linked bug plus alternate implementation PR remain open rather than shipped. The PR likely needs rebase/adaptation, not cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review or a rebased equivalent, and review it alongside #45315. The best fix is a current-main implementation that threads a session-aware prefix context through the shared reply pipeline, seeds early abort replies from accurate session/runtime metadata before delivery, keeps onModelSelected as the later authoritative override, covers both direct and routed fast-abort replies, and adds focused regression tests for stored metadata plus post-selection override behavior.

## Risks / Open Questions

- The PR is draft and its provided diff includes legacy channel paths such as src/discord and src/slack that no longer exist on current main; it needs a rebase onto the extensions/* channel layout before it can land.
- There are two open implementation candidates with different seeding strategies. Maintainers should choose the source of truth carefully: persisted session/runtime metadata may be more accurate for early abort than static config defaults, but it needs correctness and hot-path review.
- A partial fix could still leak placeholders on routed replies unless route-reply receives or derives the same template context used by direct dispatcher delivery.
