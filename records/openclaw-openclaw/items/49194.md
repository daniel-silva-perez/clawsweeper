## Summary

Keep this PR open. Current main still does not implement the PR's intended Google Vertex ADC fix: OpenClaw now uses a `gcp-vertex-credentials` marker instead of the old `<authenticated>` sentinel, but that marker is still resolved and stored as an `api-key` credential on the main runtime and compaction paths, and the Google transport serializes any non-JSON api key string as `x-goog-api-key`. The PR is also stale against current main and includes an unrelated `skills/bird` addition that expands the review and security surface.

## Best Possible Solution

Keep this PR open only as a candidate for a cleaned, current-main fix. The best path is to remove the unrelated `skills/bird` change, rebase the auth fix onto the current `gcp-vertex-credentials` marker flow, ensure Google Vertex ADC is represented as provider-owned OAuth/no literal API key before pi/provider transport calls, and add focused tests proving the marker is not stored or serialized as `x-goog-api-key` in main chat, compaction, and simple-completion/provider-call paths.
