## Summary

Keep open. Current main has adjacent session-lock hardening, including `SessionWriteLockTimeoutError` and failover suppression, but it still does not implement this PR's central timeout propagation: `runEmbeddedAttempt` calls `acquireSessionWriteLock` without `timeoutMs: params.timeoutMs`, so the lock wait can still fall back to 10 seconds. The SDK export concern also remains for the current error shape.

## Best Possible Solution

Keep this PR open for maintainer review or contributor update. Rebase it onto current main, decide whether embedded attempts should pass `params.timeoutMs` as the session-lock wait or use a narrower/configurable wait, reuse/expose the existing `SessionWriteLockTimeoutError`/code/helper if that is the intended public signal, and add focused tests for timeout propagation plus the supported SDK export.
