## Summary

Keep open. Current main supports Anthropic prompt-cache retention, cache-ttl pruning, and heartbeat-based keep-warm behavior, but it does not implement the proposed dedicated Cache TTL Warmer: no `cacheWarmer` config/schema/docs surface exists, and current heartbeat runs are still full agent turns with heartbeat prompt, delivery/visibility handling, and `bootstrapContextRunKind: "heartbeat"`. The related heartbeat/cache reports are useful context, but this issue remains a maintainer design decision rather than conservative cleanup.

## Best Possible Solution

Keep the issue open for maintainer design. If approved, implement a narrow opt-in core/provider cache-maintenance path or a new plugin API seam that can build the same request prefix as a real user turn, append a bounded minimal marker, consume/discard the response, and write no session/channel/memory state. The design should decide whether this is separate from heartbeat or a heartbeat mode, document `cacheRetention` interactions, and include schema/docs plus targeted scheduler/request-shape tests and an explicit live Anthropic validation plan.
