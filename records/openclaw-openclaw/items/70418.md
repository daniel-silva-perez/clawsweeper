## Summary

Keep open. Current main has prompt-cache retention, cache-ttl pruning, heartbeat keep-warm guidance, and provider cache-control shaping, but it still has no dedicated Anthropic Cache TTL Warmer subsystem or `cacheWarmer` config/runtime/docs surface; the remaining decision is product and provider-policy design, not conservative cleanup closure.

## What This Changes

The issue proposes an opt-in per-agent background task that periodically sends a minimal Anthropic request matching the active conversation prefix to refresh prompt-cache TTL without changing session state or delivering a channel reply.

## Best Possible Solution

Keep the design discussion open. If maintainers approve the feature, implement a narrow opt-in cache-maintenance path or explicit plugin/API seam that reuses the same stable prefix/tools/system/messages through the cache-control boundary, appends only a bounded marker, consumes and discards the response, blocks tool execution and transcript/session/channel/memory writes, documents cost and retention behavior, and includes focused unit tests plus live Anthropic validation.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature/design proposal. The absence check is high-confidence from current-main source, schema, docs, runtime, and synonym searches showing no dedicated `cacheWarmer` surface.

## Solution Assessment

Unclear because there is no patch and the core ownership boundary needs maintainer judgment. A dedicated warmer may be viable, but it should not be automated until maintainers decide whether OpenClaw should own paid background model calls or expose a safer plugin/provider seam.
