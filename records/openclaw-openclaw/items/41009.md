## Summary

Current main still has the Tailscale Serve/Funnel loopback validation, but does not have the legacy migration this PR is trying to add. The PR remains a plausible implementation candidate, though it is stale against the current doctor migration layout and needs maintainer review around `gateway.bind="auto"` semantics.

## Best Possible Solution

Keep this PR open and refresh it against current main. The best path is to port the migration into `src/commands/doctor/shared/legacy-config-migrations.runtime.gateway.ts` before the non-loopback `allowedOrigins` seed, add a source-literal legacy rule for explicit `gateway.bind=lan|tailnet|custom` with `gateway.tailscale.mode=serve|funnel`, preserve the current runtime validation invariant, and add regression coverage in the current doctor/config test files. Maintainer review should resolve `auto` bind semantics before landing.
