## Summary

Keep this PR open, but do not merge it as-is. The underlying crash-loop bug is still valid on current main, but the branch is stale against the current doctor/shared migration layout and PR head weakens the existing Tailscale Serve/Funnel loopback validation for explicit `gateway.bind="auto"`.

## What This Changes

The PR adds a legacy config rule/migration and shared bind-resolution helper to normalize literal non-loopback `gateway.bind` values to `loopback` for Tailscale Serve/Funnel, reuses that helper in validation, and adds regression tests.

## Best Possible Solution

Refresh the fix on current main by adding a doctor/shared runtime gateway migration and matching source-literal legacy rule for `lan`, `tailnet`, and non-loopback `custom` under `serve` or `funnel`; preserve the current rejection of explicit `auto`; and cover migration ordering before `allowedOrigins` seeding plus startup/doctor detection with focused tests.
