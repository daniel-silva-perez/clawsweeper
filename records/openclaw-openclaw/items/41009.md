## Summary

Keep this PR open because current main still lacks the legacy migration it is trying to add, but do not merge it as-is: the branch is stale against the current doctor/shared migration layout and its helper relaxes the Serve/Funnel loopback validation for explicit `gateway.bind="auto"`.

## What This Changes

The PR adds config migration, legacy detection, shared bind-resolution helpers, validation reuse, and regression tests to normalize explicit legacy non-loopback `gateway.bind` values to `loopback` for Tailscale Serve/Funnel configs.

## Best Possible Solution

Refresh the PR against current main by porting the migration into the doctor/shared runtime gateway migration list, adding a matching source-literal legacy rule for explicit `gateway.bind=lan|tailnet|custom` with `gateway.tailscale.mode=serve|funnel`, preserving strict validation for explicit `auto`, and covering both migration ordering and validation with focused tests.
