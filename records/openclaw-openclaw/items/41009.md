## Summary

Keep open. The crash-loop recovery gap is still real on current main, but this PR is not safe to merge as-is because its head weakens the Serve/Funnel loopback validation for explicit `gateway.bind="auto"` and the branch predates the current doctor/shared migration-file split.

## What This Changes

The PR adds legacy config detection, migration logic, shared bind-resolution helpers, validation reuse, and regression tests to normalize explicit legacy non-loopback `gateway.bind` values to `loopback` for Tailscale Serve/Funnel configs.

## Best Possible Solution

Implement the migration in the current doctor/shared runtime gateway migration specs with co-located source-literal legacy detection for `lan`, `tailnet`, and non-loopback `custom` under `serve` or `funnel`; preserve the explicit `auto` validation rejection; and cover migration ordering, source-literal detection, and validation with focused tests.
