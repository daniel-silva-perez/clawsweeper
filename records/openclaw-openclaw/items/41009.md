## Summary

Keep this PR open. Current main still enforces the Tailscale Serve/Funnel loopback bind contract, but it still does not include the PR's intended legacy migration for explicit non-loopback `gateway.bind` values. The PR is stale against the current doctor/shared migration layout, but it remains a plausible implementation candidate for the crash-loop recovery gap described in the linked issue and PR discussion.

## Best Possible Solution

Keep this PR open and refresh it against current main. The likely path is to port the migration into `src/commands/doctor/shared/legacy-config-migrations.runtime.gateway.ts`, add a matching source-literal legacy rule for explicit `gateway.bind=lan|tailnet|custom` with `gateway.tailscale.mode=serve|funnel`, preserve strict validation for unsupported non-loopback Serve/Funnel binds, and add focused regression coverage in the current doctor/config tests.
