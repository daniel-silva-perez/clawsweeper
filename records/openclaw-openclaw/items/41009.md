## Summary

Keep open. Current main still lacks the requested Serve/Funnel legacy bind migration, so the PR targets a real recovery gap, but the current PR head is not mergeable as-is because it weakens the explicit `gateway.bind="auto"` validation path and needs refresh into the current doctor/shared runtime migration layout.

## What This Changes

The PR adds config migration, legacy detection, shared bind-resolution helpers, validation reuse, and regression tests to normalize literal legacy non-loopback `gateway.bind` values to `loopback` for Tailscale Serve/Funnel configs.

## Best Possible Solution

Port the migration into the current doctor/shared runtime gateway migration list, add a source-literal legacy rule for explicit `gateway.bind=lan|tailnet|custom` with `gateway.tailscale.mode=serve|funnel`, keep explicit `auto` invalid for Serve/Funnel, and cover migration ordering plus validation with focused tests.
