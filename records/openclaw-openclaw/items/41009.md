## Summary

Keep open. The underlying crash-loop fix is still useful because current main rejects Tailscale Serve/Funnel configs with non-loopback binds and has no migration for that combination, but this PR head is not merge-ready because it weakens the explicit `auto` bind validation path and targets an old migration-file layout.

## What This Changes

The PR branch adds shared loopback-bind helpers, a literal legacy-rule trigger, a legacy migration, validation reuse, and regression tests for Tailscale Serve/Funnel configs with non-loopback `gateway.bind` values.

## Best Possible Solution

Refresh the fix on current main by adding a doctor/shared runtime gateway migration and matching source-literal legacy rule for `lan`, `tailnet`, and non-loopback `custom` under `serve` or `funnel`; keep explicit `auto` rejected by validation; and cover migration ordering before `allowedOrigins` seeding plus startup/doctor detection with focused tests.

## Review Findings

Overall correctness: patch is incorrect
