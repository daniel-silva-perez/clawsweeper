## Summary

Keep open. Current main still has the reported visible-transcript behavior, but source, tests, and changelog show it is now an intentional #72019 single-source prompt contract, so changing it needs maintainer direction rather than conservative cleanup.

## What This Changes

Requests that native subagent spawns include the delegated task text in the first visible user message while keeping the system-prompt task block as the canonical instruction source.

## Best Possible Solution

Define the subagent task visibility contract explicitly. The maintainable end state is either a transcript/channel diagnostic that exposes the delegated task without duplicating model input, or a deliberate move of the canonical task into the visible user turn with source, tests, docs, and changelog updated together.

## Reproduction Assessment

Yes. Source and regression-test inspection on current main reproduce the reported symptom: the first visible user message omits task text while the system prompt contains it under `## Your Role`.

## Solution Assessment

Unclear. The suggested local patch is narrow, but it is not clearly the best fix because it conflicts with the shipped token-dedupe contract; a transcript-only diagnostic may be safer if maintainers want auditability without duplicate model-visible input.
