## Summary

Keep open. Current main still has the reported visible-transcript behavior, but it is an intentional prompt-token dedupe contract from the #72019 fix, so this is not cleanup-closeable without maintainer agreement on the subagent task-placement contract.

## What This Changes

The issue requests adding the delegated task text to the first visible subagent user message while retaining the system-prompt task block as the canonical instruction source.

## Best Possible Solution

Define the subagent task-placement contract explicitly. A maintainable end state should either expose the delegated task in channel/transcript diagnostics without sending a second full task copy to the model, or deliberately move the canonical task into the visible user message and update the system prompt, tests, docs, and #72019 token-cost expectations together.

## Reproduction Assessment

Yes. Source and test inspection on current main gives a high-confidence reproduction: the first visible user message omits the delegated task and points to `## Your Role`, while the task is carried in `extraSystemPrompt`. No live run was attempted because this review was required to stay read-only.

## Solution Assessment

Unclear. The suggested local patch would satisfy visible auditability, but it is not clearly the best fix while the system prompt still carries the full task; a safer direction needs maintainer agreement on transcript-only visibility versus a model-prompt contract change.
