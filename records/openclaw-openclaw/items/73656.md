## Summary

Keep open. Current main still matches the reported visible-transcript behavior, and the omission is deliberate from the #72019 token-dedup fix, so this needs maintainer prompt-contract judgment rather than conservative cleanup closure.

## What This Changes

Request that the initial visible native subagent user message include the delegated task text, or an equivalent visible fallback, instead of only pointing to the hidden `Your Role` system prompt.

## Best Possible Solution

Define an explicit subagent task visibility contract: either expose the delegated task through channel/transcript metadata that is not resent to the model, or add a concise visible fallback while preserving exactly one long-form task copy in the model payload. Update the current omission tests and spawn-path coverage to encode that decision.

## Reproduction Assessment

Yes. A current-main static check reproduces the issue: the visible-message helper has no `task` input, `spawnSubagentDirect` sends that helper output as the child `message`, and tests assert the delegated task token is absent from the first visible user text.

## Solution Assessment

Unclear. The requested visible `[Subagent Task]` copy would solve transcript diagnosability, but it conflicts with the existing #72019 dedupe invariant unless implemented as non-model-visible metadata or a carefully bounded fallback.
