## Summary

Keep open. Current main still intentionally omits the delegated task from the first visible subagent user message, and tests assert that omission to preserve the #72019 token-dedup fix. The related #73624/#73637 path concerns preserving `extraSystemPrompt` under `systemPromptOverride`; it does not implement the visible transcript fallback requested here.

## Best Possible Solution

Keep this issue open for maintainer prompt-contract judgment. The best path is to choose an explicit subagent task visibility contract: either add a concise visible `[Subagent Task]` fallback while preserving #72019's single long-form model-input copy, or expose task text through channel/transcript metadata that is visible for diagnostics without duplicating tokens. Any implementation should update the current omission tests and cover native spawn with `message` plus `extraSystemPrompt`, especially alongside the #73624/#73637 prompt-override fix.

## Risks / Open Questions

- Adding the full task back to the model-visible first user message would likely reintroduce the #72019 duplicate-token regression unless the design changes where the visible transcript text is stored or keeps only one model-visible long-form task copy.
- A visible fallback would improve auditability but cannot replace fixing the hidden canonical prompt path; #73624/#73637 still need separate handling so `extraSystemPrompt` reliably reaches the compiled system prompt.
