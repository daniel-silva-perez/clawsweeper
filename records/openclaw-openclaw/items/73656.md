## Summary

Keep open. Current main still has the reported visible-transcript behavior, but source, tests, and changelog show it is intentional single-source prompt placement from #72019, so changing it needs maintainer prompt-contract direction rather than conservative cleanup.

## What This Changes

Request to include the delegated task text in the first visible subagent user message as an audit trail and fallback while keeping the system prompt as canonical context.

## Best Possible Solution

Define the subagent task visibility contract explicitly: either add a transcript/channel diagnostic that exposes the assignment for audit without duplicating model-visible input, or deliberately move/copy the task into the visible kickoff message and update source, tests, docs, and changelog together.

## Reproduction Assessment

Yes. Source and regression-test inspection on current main reproduce the symptom: the first visible user message omits task text, while the task is rendered under ## Your Role in extraSystemPrompt.

## Solution Assessment

Unclear. The suggested local patch is narrow, but it is not clearly the best fix because it conflicts with the shipped #72019 single-source task contract; a transcript-only diagnostic may better satisfy auditability without duplicating prompt tokens.
