## Summary

Keep open. Current main still has the reported visible-transcript behavior, but it is now an intentional single-source prompt contract for #72019, so closing as stale or implemented would be wrong without maintainer direction on whether the task should become model-visible again or be exposed through a transcript-only diagnostic path.

## What This Changes

The issue asks OpenClaw to include delegated task text in the first visible native subagent user message as an auditable fallback while still preserving the canonical subagent task in the system prompt.

## Best Possible Solution

Define the subagent task placement contract explicitly. The maintainable end state is either a transcript/channel diagnostic that exposes the delegated task without duplicating model input, or a deliberate move of the canonical task into the visible user turn with the system prompt, tests, docs, and changelog contract updated together.

## Reproduction Assessment

Yes. Source and test inspection on current main reproduces the issue: the visible first user message is built without task text, while the hidden system prompt carries the task and tests enforce single-source placement.

## Solution Assessment

Unclear. The proposed local patch is understandable for auditability, but it is not clearly the best fix because it conflicts with the shipped #72019 dedupe contract; a transcript-only visibility seam may be safer if maintainers want auditability without duplicate model tokens.
