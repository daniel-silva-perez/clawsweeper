## Summary

Keep open. Current main still intentionally omits the delegated task from the initial visible subagent user message, and tests assert that the task remains single-sourced in the system prompt. The related #73624 / #73637 path addresses preserving `extraSystemPrompt` under `systemPromptOverride`, not this visible transcript fallback. This needs maintainer product/prompt-contract judgment rather than conservative cleanup closure.

## Best Possible Solution

Keep this issue open for maintainer review. The best path is to decide the subagent task placement contract explicitly: either add a concise visible `[Subagent Task]` fallback while preserving #72019 dedupe guarantees, or add a channel/UI transcript diagnostic surface that exposes the delegated task without duplicating model input. Any implementation should update the current omission tests and include spawn-path coverage for `message` plus `extraSystemPrompt`, especially alongside the #73624 prompt-override fix.
