## Summary

Keep open. Current main still deliberately omits the delegated task from the first visible subagent user message, and the tests assert that the task remains single-sourced in the system prompt; the related prompt-override work does not implement this visible transcript fallback. The requested direction conflicts with the prior #72019 token-dedup design, so this needs maintainer prompt-contract judgment rather than cleanup closure.

## What This Changes

The issue requests that native subagent spawns include the delegated task text in the first visible user message while keeping the canonical task context in the system prompt.

## Best Possible Solution

Define the subagent task placement contract explicitly and preserve the #72019 single-source model-payload guarantee. A maintainable end state would either expose the delegated task through channel/transcript diagnostics without sending a second full task copy to the model, or intentionally move the canonical task into the visible user message and update the system-prompt/tests/docs contract together.

## Reproduction Assessment

Yes. On current main, inspecting `buildSubagentInitialUserMessage` and the spawn tests gives a high-confidence reproduction: the visible first user message omits the delegated task and points to `Your Role`, while the hidden system prompt carries the task.

## Solution Assessment

Unclear. The suggested local patch is understandable as a UX fallback, but passing the full task into the visible model message while retaining the system prompt copy conflicts with the #72019 dedupe fix; a safer implementation needs maintainer agreement on whether visibility should be transcript-only or model-visible.
