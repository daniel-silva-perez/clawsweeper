## Summary

Keep this PR open. Current main still has the Debug Snapshots UX gap from #39777: `loadDebug` returns silently while disconnected, the Debug view only exposes `loading` to the Refresh button, and the Snapshots card body still renders Status/Health/Last heartbeat blocks without a body-level loading or disconnected callout. The PR remains a focused implementation candidate, but it should be reviewed for layout quality and accurate connection-state handling before merge.

## Best Possible Solution

Keep this PR open, or replace it with an equivalent focused fix that passes connection state or a refresh-disabled reason into the Debug view, shows a body-level loading/progress state while `debugLoading` is true, shows a clear disconnected callout when refresh cannot run, preserves the existing Status/Health/Last heartbeat layout, and closes #39777 after merge.
