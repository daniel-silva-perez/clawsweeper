## Summary

Keep this PR open. Current main does not implement the Cursor SDK harness, and the PR is an active contributor implementation paired with open #75044, but the branch is not merge-ready: static tracing shows the plugin can fail before its harness runs because it claims a provider without registering provider/auth/catalog runtime, and the Cursor runner does not preserve OpenClaw timeout or abort semantics through the full SDK run. The bundled trusted local/cloud runtime and dependency override also need maintainer security and product review.

## What This Changes

The PR adds a bundled `extensions/cursor-sdk` plugin with a manifest, AgentHarness registration, Cursor SDK runner, error-classification tests, an isolated `@cursor/sdk` dependency, lockfile updates, a workspace `undici` override, and one formatting-only config schema change.

## Best Possible Solution

Maintainers should first decide whether Cursor SDK belongs as a bundled trusted plugin or as an external ClawHub/npm plugin. If bundled, the plugin should provide a real provider/auth/model catalog contract, preserve OpenClaw abort/timeout/fallback semantics, document the runtime policy boundary for Cursor-owned tools, and land after dependency/security review.

## Reproduction Assessment

Yes for the review findings. Static tracing against current main plus the PR head shows `cursor-sdk/composer-2` needs model/auth resolution before harness execution, while the proposed runner leaves `agent.send()`/`run.wait()` outside the timeout guard and never observes `params.abortSignal`.

## Solution Assessment

No, this is not the best way to solve the linked feature request as-is. The extension boundary is directionally right, but the implementation needs provider/auth/catalog wiring, full timeout/abort handling, and maintainer-approved runtime/dependency policy before it can safely merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Register cursor-sdk as a real provider:** `extensions/cursor-sdk/index.ts:9`
  - body: The plugin declares and supports a `cursor-sdk` provider, but only registers an AgentHarness. Current main resolves the model before invoking the harness, so `cursor-sdk/composer-2` has no provider catalog/auth metadata and can fail with `model_not_found` before this runner is reached.
  - confidence: 0.9
- **[P2] Keep the timeout active through send and wait:** `extensions/cursor-sdk/src/run-attempt.ts:103-106`
  - body: The timeout starts only after `agent.send()` returns and is cleared before `run.wait()`. Cursor SDK initializes runtime on first send, so a stalled startup/send or final wait can exceed `params.timeoutMs` without cancellation or failover.
  - confidence: 0.88
- **[P2] Cancel Cursor runs on OpenClaw abort:** `extensions/cursor-sdk/src/run-attempt.ts:99`
  - body: `runCursorSdkAttempt` never observes `params.abortSignal`. `/stop`, gateway shutdown, or cron cancellation can leave Cursor work running until upstream completion; wire an abort listener through send/stream/wait and call `run.cancel()` promptly.
  - confidence: 0.85
