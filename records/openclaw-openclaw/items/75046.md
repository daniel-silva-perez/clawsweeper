## Summary

Keep this PR open. It is an active contributor implementation paired with open #75044, but the current branch still has blocking correctness issues in provider/model wiring and runtime cancellation semantics, plus unresolved bundled-plugin dependency and security-policy questions.

## What This Changes

The PR adds a bundled `cursor-sdk` plugin with a manifest, AgentHarness registration, Cursor SDK runner and error classification tests, an isolated `@cursor/sdk` dependency, lockfile updates, and a workspace `undici` override.

## Best Possible Solution

Maintainers should first decide whether Cursor SDK belongs as a bundled trusted plugin or as an external ClawHub/npm plugin. If bundled, the implementation should add a real provider/auth/model catalog contract, preserve OpenClaw abort/timeout/fallback semantics, document the runtime policy boundary, and land only after dependency/security review.

## Reproduction Assessment

Yes for the review findings. Static tracing on current `main` plus the PR head shows model resolution happens before harness execution, while the Cursor runner leaves `agent.send()`/`run.wait()` outside the timeout guard and never observes `params.abortSignal`.

## Solution Assessment

No, this PR is not the best merge path as-is. The extension-only boundary is reasonable, but the branch needs provider/auth/catalog wiring, OpenClaw runtime-policy handling, and dependency/security approval before it safely solves the linked feature request.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Register a provider before claiming cursor-sdk:** `extensions/cursor-sdk/index.ts:9`
  - body: The plugin only registers an AgentHarness. Current main resolves provider/model/auth before invoking a harness, so `cursor-sdk/composer-2` has no provider catalog, dynamic model resolver, or auth metadata and can fail with `model_not_found` before this runner is reached.
  - confidence: 0.9
- **[P2] Keep the Cursor timeout active through send and wait:** `extensions/cursor-sdk/src/run-attempt.ts:99-124`
  - body: The timeout starts only after `agent.send()` returns and is cleared before `run.wait()`. Cursor SDK initialization is deferred until first send, so a stalled startup/send or terminal wait can exceed `params.timeoutMs` without cancellation or failover.
  - confidence: 0.88
- **[P2] Cancel Cursor runs when OpenClaw aborts:** `extensions/cursor-sdk/src/run-attempt.ts:99-106`
  - body: `runCursorSdkAttempt` never observes `params.abortSignal`. `/stop`, cron aborts, and gateway shutdown can leave Cursor work running until upstream completion; register an abort listener and cancel/dispose the SDK run promptly.
  - confidence: 0.84
