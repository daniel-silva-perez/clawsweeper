## Summary

Keep this PR open. Current main does not implement the proposed Cursor SDK agent harness, but the PR is not merge-safe as-is: the harness can be selected without a provider/model/auth catalog, the Cursor run is not bounded by OpenClaw abort/timeout semantics, and the dependency/security boundary needs maintainer approval.

## What This Changes

The PR adds a bundled `cursor-sdk` plugin with an AgentHarness runner, manifest/config schema, Cursor SDK dependency and lockfile changes, plus harness and error-classification tests.

## Best Possible Solution

If maintainers want this bundled, land it as a trusted plugin only after it registers the provider/auth/model catalog surface, preserves OpenClaw abort/timeout/fallback policy, documents the Cursor-owned tool/subagent execution boundary, uses reviewed dependency pins, and updates the relevant plugin docs/labels/changelog. If that bar is too high for core, move the implementation to an external ClawHub plugin and keep any needed SDK seams generic.

## Reproduction Assessment

Yes for the review findings. A static reproduction is to configure `cursor-sdk/composer-2` with the new harness: current main can select the harness, then `resolveModelAsync` has no provider catalog and throws `model_not_found`; the PR runner also visibly leaves `agent.send()`/`run.wait()` outside the timeout and never observes `params.abortSignal`.

## Solution Assessment

No, this is not the best merge path as-is. The extension-only shape is directionally right, but the implementation needs provider/auth/catalog wiring, runtime-policy handling, and dependency/security approval before it safely solves #75044.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Register a provider before claiming cursor-sdk:** `extensions/cursor-sdk/index.ts:9`
  - body: The plugin only registers an AgentHarness. Current main still requires a resolved provider/model before invoking the harness attempt, so `cursor-sdk/composer-2` has no catalog, dynamic model resolver, or auth metadata and can fail with `model_not_found` before this runner is reached.
  - confidence: 0.9
- **[P2] Keep the Cursor timeout active through send and wait:** `extensions/cursor-sdk/src/run-attempt.ts:99-126`
  - body: The timeout starts only after `agent.send()` returns and is cleared before `run.wait()`. Cursor SDK initialization is deferred until first send, so a stalled startup/send or terminal wait can exceed `params.timeoutMs` without cancellation or failover.
  - confidence: 0.88
- **[P2] Cancel Cursor runs when OpenClaw aborts:** `extensions/cursor-sdk/src/run-attempt.ts:99-106`
  - body: `runCursorSdkAttempt` never observes `params.abortSignal`. `/stop`, cron aborts, and gateway shutdown can leave Cursor work running until upstream completion; register an abort listener and cancel/dispose the SDK run promptly.
  - confidence: 0.84
