## Summary

Keep this PR open. It is an active contributor implementation paired with open #75044, and current main does not already provide the Cursor SDK backend; however the proposed bundled plugin still has blocking provider/auth/model resolution, timeout/abort, replay-safety, and security-review gaps before it is mergeable.

## What This Changes

The branch adds a bundled `extensions/cursor-sdk` AgentHarness plugin with Cursor SDK runner/tests, isolated `@cursor/sdk` dependency metadata, lockfile updates, a workspace `undici` override, and one formatting-only config schema change.

## Best Possible Solution

Maintainers should first decide the product/security boundary for Cursor SDK. If it is bundled, it should land as a provider-plus-agent-harness plugin with manifest auth/catalog metadata, full send/stream/wait timeout and abort handling, conservative side-effect/replay metadata, explicit docs for Cursor-owned tools, exact reviewed dependency policy, and targeted tests for those contracts.

## Reproduction Assessment

Yes for the review findings. Static tracing against current main and the PR head shows a `cursor-sdk/composer-2` runtime would need model/auth resolution before harness execution, while the runner leaves `agent.send()`/`run.wait()` outside the timeout guard and never observes `params.abortSignal`.

## Solution Assessment

No, this is not the best way to solve the linked feature request as-is. The extension boundary is directionally right, but the implementation needs provider/auth/catalog wiring, runtime cancellation/timeout fixes, replay policy fixes, and maintainer-approved dependency/security handling before merge.

- **[P2] Register cursor-sdk as a real provider:** `extensions/cursor-sdk/index.ts:9`
  - body: The plugin declares a `cursor-sdk` provider, but the entrypoint only registers an AgentHarness. Current main resolves the model before invoking the harness, so `cursor-sdk/composer-2` has no provider catalog/auth metadata and can fail with `model_not_found` before this runner receives a resolved API key.
  - confidence: 0.9

  - body: Cursor SDK defers runtime initialization until the first `send`, but this timer starts only after `agent.send()` returns and is cleared before `run.wait()`. A stalled startup/send or terminal wait can exceed `params.timeoutMs` without cancellation or failover.
  - confidence: 0.88
- **[P2] Cancel Cursor runs on OpenClaw abort:** `extensions/cursor-sdk/src/run-attempt.ts:99`
  - body: `runCursorSdkAttempt` never observes `params.abortSignal`. `/stop`, gateway shutdown, or cron cancellation can leave a Cursor local/cloud run active until upstream completion; wire an abort listener through send/stream/wait and call `run.cancel()` promptly.
  - confidence: 0.86
- **[P2] Mark Cursor runs replay-unsafe after execution:** `extensions/cursor-sdk/src/run-attempt.ts:210`
  - body: The result helpers report `hadPotentialSideEffects: false` and `replaySafe: true` even though Cursor may edit files, run shell/MCP tools, or open PRs outside OpenClaw telemetry. That can make recovery/retry logic treat a side-effecting run as replay-safe.
  - confidence: 0.82
