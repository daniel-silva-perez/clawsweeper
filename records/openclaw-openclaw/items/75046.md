## Summary

Keep open. This is an active contributor PR paired with open feature request #75044; current main does not implement the proposed Cursor SDK local/cloud harness, but the patch still violates the current harness/provider contract and needs maintainer product/security review before merge.

## What This Changes

The PR adds a bundled `extensions/cursor-sdk` plugin with manifest config, AgentHarness registration, Cursor SDK runner/error classification/tests, an isolated `@cursor/sdk` dependency, lockfile changes, and a workspace `undici` override.

## Best Possible Solution

Maintainers should first decide whether Cursor SDK belongs as a bundled trusted plugin or as an external ClawHub/npm plugin. If bundled, the implementation should add a real provider/auth/model catalog or documented config contract, preserve OpenClaw abort/timeout/fallback semantics, document the runtime support and policy boundary, update ownership/labeling surfaces, and land only after dependency/security review.

## Reproduction Assessment

Yes for the review findings. Static tracing on current main plus the PR diff shows model resolution happens before harness execution, and the Cursor runner does not cover `agent.send()`/`run.wait()` with the timeout or observe `params.abortSignal`; no live Cursor key is needed for those checks.

## Solution Assessment

No, this PR is not the best merge path as-is. The extension-only shape is a reasonable boundary, but the patch needs provider/auth/catalog wiring, OpenClaw runtime-policy handling, and dependency/security approval before it safely solves the linked feature request.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Register a provider before claiming cursor-sdk:** `extensions/cursor-sdk/index.ts:9`
  - body: The plugin only registers an AgentHarness. Current main resolves provider/model/auth before invoking a harness, so `cursor-sdk/composer-2` has no provider catalog, dynamic model resolver, or auth metadata and can fail with `model_not_found` before this runner is reached.
  - confidence: 0.9
- **[P2] Keep the Cursor timeout active through send and wait:** `extensions/cursor-sdk/src/run-attempt.ts:97-119`
  - body: The timeout starts only after `agent.send()` returns and is cleared before `run.wait()`. A stalled SDK startup/send or terminal wait can therefore exceed `params.timeoutMs` without cancellation or failover.
  - confidence: 0.88
- **[P2] Cancel Cursor runs when OpenClaw aborts:** `extensions/cursor-sdk/src/run-attempt.ts:97-104`
  - body: `runCursorSdkAttempt` never observes `params.abortSignal`. `/stop`, cron aborts, and gateway shutdown can leave Cursor work running until upstream completion; register an abort listener and cancel/dispose the SDK run promptly.
  - confidence: 0.84
