## Risks / Open Questions

- The supplied PR diff includes unrelated Z.AI wrapper and scoped Vitest config changes alongside the Gemini safety-settings work; those should be split or explicitly justified before merge.
- Review comments on the PR raised wrapper-scope concerns where stale provider-specific wrappers could mutate later provider payloads. That is functional and security-adjacent because it can send unexpected request fields to a different backend.
