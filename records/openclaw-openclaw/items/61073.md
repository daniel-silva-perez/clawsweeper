## Summary

Keep this PR open. Current main has absorbed part of the Instances localization work, but it still leaves Instances fallback/scope labels hardcoded and still has the empty-reason placeholder path that reviewers called out. The PR remains a plausible implementation candidate, but it should be rebased and adjusted to the current generated-locale pipeline before merge.

## Best Possible Solution

Keep the PR open, rebase it onto current main, and update it to finish the remaining Instances localization through the current i18n pipeline: add English source keys for unknown host/mode and scope labels if those labels should be localized, regenerate non-English locale bundles, and either guard the empty reason line or harden the translator placeholder replacement with focused tests.
