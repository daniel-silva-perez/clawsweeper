## Summary

Keep open. Current main still does not implement xAI as a `/voiceclaw/realtime` provider: source and docs only wire Gemini Live, and xAI docs in this repo explicitly mark realtime voice as not exposed. The issue is also paired with open draft PR #73032 using closing language, so it should remain open until that PR is reviewed, merged, or closed.

## What This Changes

Add xAI Realtime Voice Agent support to the `/voiceclaw/realtime` gateway surface, ideally through a reusable OpenAI-Realtime-protocol adapter that could also support OpenAI Realtime later.

## Best Possible Solution

Review the paired draft PR as the canonical implementation path. If accepted, the end state should add provider-aware `/voiceclaw/realtime` dispatch, xAI adapter coverage, docs for `XAI_API_KEY`, and tests for audio, transcripts, tools, reconnect, and secret redaction while keeping existing Gemini behavior stable.

## Reproduction Assessment

Not applicable as a feature request, but the missing behavior is reproducible by static inspection: current main only accepts and forces Gemini for `/voiceclaw/realtime`, and the docs say xAI realtime voice is not exposed.

## Solution Assessment

Unclear. A shared OpenAI-Realtime-protocol adapter may be a good implementation, but the repo currently has both a core VoiceClaw gateway adapter and a plugin realtime-voice provider surface, so the surface choice needs maintainer review before closure.
