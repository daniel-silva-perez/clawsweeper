Keep open. Current main still does not implement xAI as a `/voiceclaw/realtime` provider, and the issue is already paired with open draft PR #73032 using closing language, so this should remain open until that PR is reviewed, merged, or closed.

## What This Changes

Add xAI Realtime Voice Agent support to `/voiceclaw/realtime`, preferably through a shared OpenAI-Realtime-protocol adapter that can also support future OpenAI Realtime work.

## Best Possible Solution

Use draft PR #73032 as the canonical implementation path. If accepted, the end state should add provider-aware `/voiceclaw/realtime` dispatch, an xAI adapter or shared OpenAI-Realtime protocol layer, `XAI_API_KEY` documentation, and tests covering audio, transcripts, tools, reconnect behavior, and secret redaction without regressing Gemini.
