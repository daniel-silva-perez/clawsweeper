## Summary

Keep open. Current main still has a cost-sensitive preview Google default and default-setting paths that can overwrite an existing primary unless the caller explicitly opts into preservation; the discussion also includes a second paid-model surprise report, so this is not conservative cleanup material.

## What This Changes

The issue asks OpenClaw to prevent silent migration or setup flows from replacing a user’s default model with a paid preview model without explicit consent, warning, or a revert path.

## Best Possible Solution

Define a model-default safety policy that preserves existing global and per-agent primaries/fallbacks across upgrades unless the user explicitly accepts a preview or higher-cost replacement, separates provider ID normalization from default promotion, adds a warning/revert path for migration-set preview defaults, and tracks cron control plus tool-grounding visibility as separate scoped items.

## Reproduction Assessment

Partial yes. Current main source and tests reproduce the remaining policy risk around preview defaults and opt-in-only preservation; the exact historical billing artifact needs the reporter’s saved backups and session logs.

## Solution Assessment

Unclear until maintainers choose the product policy. A narrow patch could broaden preservation in setup/migration paths, but consent prompts, pricing-tier detection, changelog policy, and the adjacent cron/tool-grounding asks need explicit maintainer scope first.
