## Summary

Keep open. Current main still defaults Google API-key setup and the `gemini` alias to `google/gemini-3.1-pro-preview`, the Google helper still overwrites an existing primary, and the only preservation path I found is caller-scoped to configure-style flows rather than a general upgrade/migration consent policy. The added user comment reporting a similar paid-model surprise through agent state reinforces that this needs maintainer scoping rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to prevent silent default-model migrations that promote users to higher-cost preview models without explicit consent, plus related cost-control follow-ups for cron control and tool-grounded completion claims.

## Best Possible Solution

Treat the default-model migration and cost consent path as the primary product bug: preserve existing user primaries across upgrades unless the user explicitly opts into a new preview or higher-cost default, keep alias normalization separate from default promotion, add a one-time warning/revert path for migration-set preview defaults, document pricing-impact expectations, and split cron/tool-grounding work into narrower follow-ups if maintainers want those changes.
