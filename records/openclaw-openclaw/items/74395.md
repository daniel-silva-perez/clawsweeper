## Summary

Keep open. Current main still wires Google API-key setup to `google/gemini-3.1-pro-preview`, the Google default helper still replaces an existing primary, and the broader migration/default-promotion consent policy is not resolved by the newer configure-only preservation path. A second user comment reports a similar paid-model surprise through agent state, which reinforces that this needs maintainer scoping rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to prevent upgrades or migrations from silently changing a user's configured default model to a paid preview/higher-cost model, with related cost-control warnings and follow-up surfaces for cron and tool-grounding visibility.

## Best Possible Solution

Treat model migration and cost consent as the primary product bug: preserve existing user primaries across upgrades unless the user explicitly opts into a new preview or higher-cost default, keep alias normalization separate from default promotion, add a one-time warning/revert path for migration-set preview defaults, and split cron/tool-grounding work into narrower issues if maintainers accept those directions.
