## Summary

Keep open. Current main still has a cost-sensitive Google preview model as a default, and default-setting paths can still overwrite an existing primary unless the caller explicitly opts into preservation; the discussion also includes a second paid-model surprise report, so this is not conservative cleanup material.

## What This Changes

Request to prevent upgrades or migrations from silently replacing user default models with higher-cost preview models, add pricing/consent warnings, and improve adjacent cost-control surfaces.

## Best Possible Solution

Define a model-default safety policy that preserves existing global and per-agent primaries/fallbacks across upgrades unless the user explicitly accepts a preview or higher-cost replacement, separates provider ID normalization from default promotion, adds a visible warning/revert path for migration-set preview defaults, and tracks cron control plus tool-grounding visibility as separate scoped follow-ups.

## Reproduction Assessment

Partial yes. Current main source and tests reproduce the remaining policy risk around preview defaults and opt-in-only primary preservation; the exact historical billing artifact needs the reporter's saved backups and session logs.

## Solution Assessment

Unclear until maintainers choose the product policy. A safer path is to decide the consent/pricing-tier rule first, then implement a narrow migration/setup/config change with docs and regression coverage.
