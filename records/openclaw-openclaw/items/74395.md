## Summary

Keep open. Current main still has a preview Google default and several default-setting paths that can replace an existing primary unless a caller opts into preservation, while the issue discussion includes a second paid-model surprise report. This is a cost-sensitive model-default and migration policy issue, not conservative cleanup material.

## What This Changes

The issue asks OpenClaw to prevent upgrades or setup flows from silently changing configured default models to higher-cost preview variants, and also raises related cron-stop and tool-grounding visibility gaps.

## Best Possible Solution

Define a default-model safety policy that preserves existing global and per-agent primaries/fallbacks across upgrades unless the user explicitly accepts a higher-cost or preview replacement, separates provider ID normalization from default promotion, adds a visible warning/revert path for migration-set preview defaults, and tracks cron control plus tool-grounding visibility as narrower follow-ups.

## Reproduction Assessment

Partial yes. Current main source and tests reproduce the remaining policy risk: Google still defaults to a preview model and default helpers overwrite existing primaries unless preservation is explicitly requested; the exact historical billing artifact needs the reporter's saved backups/session logs.

## Solution Assessment

Unclear until maintainers choose the product boundary. A narrow code fix could extend preservation around provider default promotion, but pricing-tier detection, interactive consent, changelog policy, cron mutation tools, and completion-claim grounding need explicit scope first.
