## Summary

Keep open. Current main still carries a Google Gemini preview default and alias, the Google setup helper still overwrites an existing primary, and the newer preservation behavior is scoped to configure re-runs rather than a general upgrade/migration consent policy. The corroborating user comment makes this a maintainer-scoped cost-control policy issue, not conservative cleanup.

## What This Changes

The issue requests migration/default-model safeguards so upgrades do not silently promote existing user defaults and fallbacks to higher-cost preview models without consent, pricing notice, or a revert path.

## Best Possible Solution

Adopt a model-default migration policy that preserves existing global and per-agent primaries/fallbacks unless the user explicitly selects a higher-cost or preview replacement, keeps provider ID normalization separate from default promotion, and adds a one-time warning/revert path for migration-set preview defaults while splitting cron control and tool-grounding indicators into narrower follow-ups.
