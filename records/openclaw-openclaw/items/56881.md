## Summary

Keep open. Current main appears to have resolved the OpenAI websocket fallback and Discord interaction-listener timeout symptoms, and the plugin loader still passes schema-defaulted config through `api.pluginConfig`, but there is not enough proof that the remaining external `memory-lancedb-pro` `sessionStrategy` regression is fixed on current main.

## What This Changes

The issue asks maintainers to identify a regression after 2026.3.28 where `memory-lancedb-pro` resolved `sessionStrategy` as `none`, OpenAI Codex websocket fallback logged HTTP 500s, and Discord interactions slowed on Windows.

## Best Possible Solution

Verify `memory-lancedb-pro@1.1.0-beta.10` against current main with and without the explicit `sessionStrategy` override. If it works, close this with the shipped websocket/Discord evidence; if it still fails, split a focused compatibility issue that states whether OpenClaw must preserve a config-resolution behavior or the external plugin should normalize its own default from `api.pluginConfig`.

## Reproduction Assessment

Unclear. The original report has credible reproduction evidence on Windows 10 with 2026.3.28 versus 2026.3.24 and an external linked report, but there is no high-confidence current-main reproduction for `memory-lancedb-pro@1.1.0-beta.10` in this checkout.

## Solution Assessment

Unclear. Current main looks like the right solution for the websocket and Discord symptoms, but the `sessionStrategy` part should be solved only after confirming whether the external plugin is using the documented `api.pluginConfig` contract or needs an OpenClaw compatibility seam.
