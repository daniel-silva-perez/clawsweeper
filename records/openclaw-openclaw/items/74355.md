## Summary

Current main still omits pdf.js `standardFontDataUrl` in the bundled `document-extract` plugin, while the dependency contract and source confirm the parameter is the standard-font data hook. This PR is not obsolete; it should stay open for maintainer review and landing or an equivalent narrow fix.

## What This Changes

The PR updates the bundled `document-extract` PDF extractor to resolve `pdfjs-dist/standard_fonts/`, pass it as `standardFontDataUrl` to `getDocument`, and adds a real Helvetica-PDF integration test for the standard-font path.

## Best Possible Solution
