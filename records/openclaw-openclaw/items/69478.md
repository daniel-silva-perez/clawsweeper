## Summary

Keep open. Current main and latest release still only suppress consecutive duplicate system-event text, while the report is about retry/failover fanout where interleaved events or fresh approval/run IDs can requeue the same user-visible approval. The issue is also paired with open PR #73040, which directly targets this bug; per cleanup policy the issue should stay open until that PR lands or maintainers choose a different fix.

## Best Possible Solution

Keep this issue open and use #73040, or a maintainer-approved equivalent, as the implementation path. The fix should add queue-wide dedupe in `src/infra/system-events.ts` for identical text within the same stable context, avoid mutating `lastContextKey` when skipping or when no context is supplied, and add regression tests for non-consecutive duplicates, different-context same text, and context preservation. If maintainers decide approval IDs themselves need stable coalescing, track that as a narrower follow-up after the system-event chokepoint fix lands.
