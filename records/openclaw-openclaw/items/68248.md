## Summary

Keep open. Current main already covers much of the agent/avatar plumbing, and the overlapping avatar-only work in #53380 and #17298 supports narrowing the scope. The remaining requested behavior is still real: Control UI favicons are static, the agent picker/sidebar path does not render avatar images next to agent names, and the browser-local user display name is not exposed or sent as the Control UI client display name.

## What This Changes

Request to customize Control UI agent avatars from identity data, per-agent/browser-tab favicons, and the user display name for easier multi-agent distinction.

## Best Possible Solution

Keep the shipped agent avatar model, then split or narrow the remaining work into a small Control UI enhancement: safe current-agent favicon updates, a browser-local user display-name editor that feeds rendering and client displayName metadata, and an explicit decision on whether agent/session selectors should render avatar images rather than text-only labels.
