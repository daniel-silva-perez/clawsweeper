## Summary

Keep this PR open. Current main still lacks the proposed OpenAI WebSocket first-response watchdog: the WS stream options have no watchdog budget, the post-send wait only handles abort/close/message/error events, and auto fallback only follows runtime failures rather than a connected socket that never sends an initial server event. The PR is focused on source/tests with no observed supply-chain or CI surface changes, but it needs rebase/adaptation to the current `resolveEmbeddedAgentStreamFn` path before it can be merged.

## Best Possible Solution

Keep this PR open and adapt it to current main. The best implementation is a narrowly scoped OpenAI WS first-response watchdog wired through `resolveEmbeddedAgentStreamFn`, with auto mode resetting/degrading the WS session and falling back to HTTP/SSE, forced websocket mode surfacing a timeout, and regression tests for disabled watchdogs, first server/bookkeeping events, late WS completion isolation, single `start` delivery, and abort-signal forwarding.
