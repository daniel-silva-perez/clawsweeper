## Summary

Keep this PR open. Current main still lacks the OpenAI WebSocket first-response watchdog proposed here: there is no watchdog option or helper, the post-send wait only listens for abort/close/message events, and fallback is still driven by connect/send/runtime errors or the generic idle timeout rather than a silent first server response stall. The provided diff is focused on source and tests, with no supply-chain or CI surface changes, but it needs rebase/adaptation to current stream-resolution wiring.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase it onto current main, wire a first-response watchdog through the current `resolveEmbeddedAgentStreamFn` path, preserve the existing generic idle timeout, and keep focused regression coverage for auto HTTP/SSE fallback, forced-websocket timeout errors, disabled watchdog handling, first server/bookkeeping events, late WS completion isolation, single `start` delivery, and abort-signal forwarding. No changelog is strictly required unless the merged fix is treated as user-visible recovery behavior.
