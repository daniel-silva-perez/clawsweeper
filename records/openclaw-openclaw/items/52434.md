## Summary

Keep open. Current main still lacks typing feedback for inline tool-dispatched skill commands, but the PR is not ready because the new awaited typing start can block tool execution and bypasses the existing typing-suppression policy.

## What This Changes

The PR adds an awaited `typing.startTypingLoop()` call before executing a tool-dispatched skill command in `src/auto-reply/reply/get-reply-inline-actions.ts`.

## Best Possible Solution

Implement inline tool-dispatch typing through the existing typing policy/signaler semantics, keep typing-start failures non-fatal to skill execution, and add focused regression coverage for allowed, suppressed, and rejected typing-start paths.

## Reproduction Assessment

Yes. A focused unit test can make `typing.startTypingLoop()` reject in the inline tool-dispatch branch and assert that `tool.execute()` still runs; another can set `typingMode: "never"` or `opts.suppressTyping` and assert no typing start is attempted.

## Solution Assessment

No. The current PR's direct awaited call is too broad; the safer fix is a policy-aware tool-start signal whose failure cannot prevent the skill tool from executing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep typing failures from aborting tools:** `src/auto-reply/reply/get-reply-inline-actions.ts:244-246`
  - body: `startTypingLoop()` can reject because the typing controller uses `rethrowOnError: true`. With this await inside the same `try` block as `tool.execute()`, a transient channel typing failure returns the error reply before the skill tool runs.
  - confidence: 0.93
- **[P2] Respect typing policy before starting inline typing:** `src/auto-reply/reply/get-reply-inline-actions.ts:246`
  - body: `handleInlineActions` runs before the normal run path resolves typing policy, so this unconditional call can emit typing for `typingMode: "never"`, explicit `suppressTyping`, heartbeat, system-event, or internal-webchat paths that should stay silent.
  - confidence: 0.9
