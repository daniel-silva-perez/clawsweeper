## Summary

Keep this PR open. The user-visible goal is valid because current main still executes inline tool-dispatched skill commands without starting typing, but the submitted patch starts typing unconditionally inside the tool execution try/catch, which can both abort the skill when typing fails and bypass existing typing suppression policy.

## What This Changes

The PR adds an awaited `typing.startTypingLoop()` call in `src/auto-reply/reply/get-reply-inline-actions.ts` before executing a tool-dispatched skill command.

## Best Possible Solution

Implement inline tool-dispatch typing through the existing resolved typing policy/signaler semantics, keep typing-start failures non-fatal to skill execution, and cover the behavior with narrow inline-action tests.
