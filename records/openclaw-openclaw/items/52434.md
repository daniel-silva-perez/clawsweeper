## Summary

Keep open. The user-visible goal is still valid on current main, but the submitted approach needs changes because it can turn a typing-indicator failure into a failed skill run and can bypass configured typing suppression.

## What This Changes

The PR adds an awaited `typing.startTypingLoop()` call in `src/auto-reply/reply/get-reply-inline-actions.ts` before executing a tool-dispatched skill command.

## Best Possible Solution

Implement inline tool-dispatch typing through existing typing policy/signaler semantics, keep typing-start failures non-fatal to skill execution, and add narrow inline-action tests for allowed, suppressed, and rejected typing-start behavior.
