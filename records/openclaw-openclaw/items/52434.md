## Summary

Current main still does not implement inline tool-dispatch typing, and this PR remains useful but is not merge-ready because the submitted change starts typing unconditionally and lets typing-start failures abort tool execution.

## What This Changes

The PR adds an awaited `typing.startTypingLoop()` call before executing an inline skill command whose `dispatch.kind` is `tool` in `src/auto-reply/reply/get-reply-inline-actions.ts`.

## Best Possible Solution

Keep this PR open and update it narrowly: start typing for inline tool-dispatched skill execution through the existing typing mode/suppression policy, make typing-start failure non-fatal so the tool still executes, and add focused inline-action tests for the allowed, suppressed, and rejection paths.
