## Summary

Keep open: current main still has the Telegram native-command policy bypass, the latest release still ships it, and open PR #56741 already references this issue with closing syntax. This should stay open until that PR or an equivalent fix lands.

## What This Changes

Telegram native slash commands in forum topics should honor per-topic and per-group `groupPolicy: "disabled"` so only the intended bot/account processes commands for that topic.

## Best Possible Solution

Land #56741 or an equivalent Telegram-plugin fix that makes native command authorization use the same effective topic/group policy as regular group messages, with regression coverage for a forum topic where inherited or explicit `groupPolicy: "disabled"` must block `/reset` and related native commands.

## Reproduction Assessment

Yes. The issue gives concrete Telegram forum reproduction steps, and current main statically reproduces the bypass because the native-command auth path resolves topic config but disables topic/group policy overrides.

## Solution Assessment

Yes, with coverage. The narrow maintainable fix is to reuse the existing override-aware policy resolver for native commands; a regression test should lock the forum-topic disabled-policy case.
