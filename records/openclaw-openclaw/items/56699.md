## Summary

Keep open. Current main still has the reported Telegram native-command bypass: the native command auth path passes `useTopicAndGroupOverrides: false`, while the policy helper only reads topic/group `groupPolicy` when that flag is true. The issue is also paired with open PR #56741, so it should stay open until the fix lands or is replaced.

## What This Changes

Native Telegram slash commands in forum topics should respect per-topic and per-group `groupPolicy: "disabled"` so bots for other topics do not process the command.

## Best Possible Solution

Land or replace the linked Telegram plugin fix so native command auth uses the same effective topic/group policy resolution as regular Telegram messages, with focused regression coverage for forum-topic and group-level `groupPolicy: "disabled"` cases.

## Reproduction Assessment

Yes. The issue gives concrete Telegram forum reproduction steps, and current main statically reproduces the bypass because the native command path resolves `topicConfig` but tells the policy helper not to use topic/group overrides.

## Solution Assessment

Yes, with a small test addition. The maintainable fix is local to `extensions/telegram/src/bot-native-commands.ts`: enable topic/group overrides for native command policy evaluation and cover the forum-topic disabled case in native command auth tests.
