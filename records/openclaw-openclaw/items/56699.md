## Summary

Keep open. Current main still has the reported Telegram native-command policy bypass: the command auth path resolves topic/group config but passes `useTopicAndGroupOverrides: false`, so topic or group `groupPolicy: "disabled"` is ignored for `/reset`, `/new`, `/status`, and plugin/native commands. An open linked PR, #56741, references this issue with closing syntax and targets the exact flag, so the issue should remain open until that fix or a replacement lands.

## What This Changes

Telegram native slash commands in forum topics should respect per-topic and per-group `groupPolicy: "disabled"` so only the bot assigned to that topic processes the command.

## Best Possible Solution

Review and land #56741 or an equivalent current-layout fix, with regression coverage that sends a native command in a forum topic where `topicConfig.groupPolicy` or inherited group policy disables commands while the account-level policy would otherwise allow them.

## Reproduction Assessment

Yes. The issue gives concrete Telegram forum reproduction steps, and current main statically reproduces the bypass because native command auth resolves `topicConfig` but tells the policy helper not to use topic/group overrides.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing override-aware policy resolver for native commands, matching the regular message path, plus a focused regression test for disabled forum-topic policy.
