## Summary

Keep this issue open. Current main has added useful diagnostics, allowlist fail-closed behavior, provider docs, and a narrow strict-agentic retry path, but it still does not generally enforce real tool invocation when a model emits prose, JSON/XML-looking text, or success-like text instead of structured tool calls. The related #45049 remains open and may become the cleaner canonical enforcement tracker, but this issue has broader AgentMail, GitHub, Composio, and custom plugin context that should not be auto-closed without maintainer consolidation.

## Best Possible Solution

Keep this issue open and ask for a current redacted trajectory export from latest main or the latest release. Maintainers should classify the failure as one of: no callable tools after policy filtering, tools sent but no structured tool call emitted, structured call blocked/normalized incorrectly, real tool execution succeeded but the external side effect failed, or provider-specific local-model behavior. If the remaining core problem is exactly the enforcement gap tracked in #45049, consolidate there after maintainer review; otherwise add a guarded, provider-aware retry/tool-choice path without executing arbitrary assistant prose.
