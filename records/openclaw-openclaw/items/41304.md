## Summary

Keep this issue open. Current main has useful partial mitigations for missing allowlisted tools and some local/proxy OpenAI-compatible tool-call paths, but I did not find evidence that the broader reported failure is fully fixed when registered plugin/MCP tools are available and the model replies in prose or simulated success instead of emitting a structured tool call.

## Best Possible Solution

Keep this issue open and request a redacted trajectory export from a current failing run on the latest release or current main. Maintainers should classify the failure as one of: no callable tools after policy filtering, tools sent but no structured tool call emitted by the provider/model, structured call blocked or normalized incorrectly, or real tool execution succeeded in OpenClaw but the external side effect failed. If the remaining repro reduces to provider-specific local-model behavior, split it to the relevant provider docs/runtime path or consolidate it with #45049 after maintainer review.
