## Summary

Keep this issue open. Current main has partial mitigations for missing explicit allowlists and local/proxy OpenAI-compatible `tool_choice` defaults, plus diagnostics that make fake tool-call text visible. It still does not generally enforce real tool invocation when a registered plugin/MCP tool is available but the model replies in prose or simulated success instead of emitting a structured tool call.

## Best Possible Solution

Keep this issue open and request a redacted trajectory export from a current failing run on the latest release or current main. Maintainers should classify the failure as one of: no callable tools after policy filtering, tools sent but no structured tool call emitted, structured call blocked or normalized incorrectly, real tool execution succeeded but the external side effect failed, or provider-specific local-model behavior. If it reduces to the same enforcement problem as #45049, consolidate there after maintainer review.
