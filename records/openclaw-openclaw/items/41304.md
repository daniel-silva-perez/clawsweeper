## Summary

Keep this issue open. Current main has improved diagnostics, plugin-tool validation, allowlist fail-closed behavior, vLLM workaround docs, and a narrow strict-agentic retry/blocked path for OpenAI GPT-5-family runs, but it still does not generally enforce real tool invocation for the reported AgentMail/GitHub/Composio/local-model class when a model emits prose or JSON-looking text instead of structured tool calls. The narrower related #45049 remains open and may be the right canonical enforcement tracker after maintainer consolidation.

## Best Possible Solution

Keep this issue open for maintainer follow-up and ask reporters for current redacted trajectory bundles from latest main or the latest release. Maintainers should classify each failure as no callable tools after policy filtering, tools sent but no structured call emitted, structured call blocked/normalized incorrectly, real tool execution succeeded but the external side effect failed, or provider-specific local-model behavior. If the remaining core gap matches #45049 exactly, consolidate there after maintainer review; otherwise add a guarded provider-aware retry/tool-choice path without executing arbitrary assistant prose.
