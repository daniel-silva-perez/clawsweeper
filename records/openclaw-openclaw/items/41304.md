## Summary

Keep open. Current main has several partial safeguards for tool registration, allowlist failures, OpenAI-compatible tool_choice, and text-shaped local-model tool calls, but the active discussion now includes a wire-level repro showing zero toolCall events for plausible tool-use text and narrower open follow-ups for prompt size/order. I would not close this as implemented or duplicate yet because those follow-ups do not prove the original AgentMail/GitHub/Composio action-tool reports are resolved.

## What This Changes

The issue requests a fix for agents that claim write/action tool success without actually invoking registered tools such as AgentMail, GitHub, Composio, weather, or web_fetch.

## Best Possible Solution

Keep this open until the prompt-budget/template follow-ups or an equivalent fix land, then verify with a small side-effect reproduction that the session transcript contains toolCall/toolResult events before any claimed write/action success. The durable end state should combine clear diagnostics for no-tool-call success claims with a prompt/bootstrap shape that preserves tool-use rules for small local models.

## Reproduction Assessment

Yes, partially. The 2026-04-30 comment gives a high-confidence wire-level reproduction for the local-model failure path using `openclaw agent --json` plus session jsonl inspection, but the original AgentMail/GitHub/Composio reports still need a current-main trajectory bundle to confirm the same root cause across those plugins.

## Solution Assessment

Unclear. Current main's partial guardrails are useful but not the best complete solution because they warn on tool-shaped text and allowlist errors while still allowing plausible natural-language success claims without a matching execution event; a safer fix would validate claimed action success against tool execution telemetry and reduce or prioritize bootstrap prompt content for small-model routes.
