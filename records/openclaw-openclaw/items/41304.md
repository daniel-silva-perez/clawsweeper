## Summary

Keep this issue open. Current main has useful partial mitigations for missing explicit allowlists, proxy/local OpenAI-compatible `tool_choice` defaults, trajectory diagnostics, and visible warnings for fake tool-call text, but it still does not generally enforce real tool invocation when a registered plugin/MCP/built-in tool is available and the model replies in prose or simulated success instead of emitting a structured tool call.

## Best Possible Solution

Keep this issue open and ask for a current redacted trajectory export from a failing run on the latest release or current main. Maintainers should classify the failure as one of: no callable tools after policy filtering, tools sent but no structured tool call emitted, structured call blocked or normalized incorrectly, real tool execution succeeded but the external side effect failed, or provider-specific local-model behavior. If it reduces to the same enforcement gap as #45049, consolidate there after maintainer review.
