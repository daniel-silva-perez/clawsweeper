## Summary

Keep this issue open. Current main still lacks a per-agent or per-route-binding session reset policy: the config contract, strict schemas, docs, and runtime reset resolver remain scoped to global session settings, chat type, and channel override, while route bindings only accept `session.dmScope`.

## What This Changes

The issue requests configurable session reset policy per agent or per binding so one-shot workflow agents can start fresh without changing global or channel-wide session behavior.

## Best Possible Solution

Define an additive core config/runtime contract for reset-policy overrides after routing has identified the effective agent and binding. The implementation should update TypeScript types, strict Zod schemas, schema help/labels/generated metadata, docs, and focused tests for precedence and the one-shot workflow case, while preserving existing global/type/channel behavior by default.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence source verification path: proposed fields such as `agents.list[].session.reset` or `bindings[].session.reset` are absent from strict schemas and the runtime resolver cannot select policy by agent or binding.

## Solution Assessment

Unclear until maintainers choose the public config shape, but the maintainable direction is a generic core reset-policy resolver rather than a ClawHub plugin or ACP-only workaround. Binding and agent precedence should be made explicit before implementation.
