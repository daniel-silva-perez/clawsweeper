## Summary

Keep this issue open. Current main still supports automatic session reset policy only at the global session level, by chat type, and by channel override. There is no accepted config field or runtime resolution path for per-agent or per-binding reset policy, and the request is a core session-routing/config capability rather than a ClawHub-only optional plugin.

## Best Possible Solution

Keep this issue open. The best path is a generic core config/runtime change: add a documented reset-policy override at `agents.list[]` and/or `bindings[].session`, extend strict schema/help/docs, then resolve the effective reset policy after inbound routing identifies the target agent and binding. Tests should cover precedence against global `session.reset`, `resetByType`, and `resetByChannel`, including the one-shot workflow case described here.
