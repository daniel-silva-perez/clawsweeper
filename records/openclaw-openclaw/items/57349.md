## Summary

Keep open. Current main has useful mitigations and diagnostics, but it does not prove the broad long-uptime gateway OOM is fixed. Recent reports are still current and span Docker, macOS, webchat/Discord, Telegram-only heavy compaction/tool use, and memory-plugin-heavy deployments; current source still has memory-relevant polling, whole-store, whole-transcript, and watcher/listener paths.

## Best Possible Solution

Keep this issue open as the broad gateway heap-exhaustion tracker. The maintainer path should be to collect a current diagnostics export plus heap snapshots from an affected build, keep #55334 as the sessions.json/session-store subtracker, and split proven retainers into owner fixes: bounded or push-based node refresh, lazy/paged sessions.list, tail/page chat.history reads, compaction snapshot deduplication, and memory-plugin watcher/cache/session cleanup where profiling confirms retention.
