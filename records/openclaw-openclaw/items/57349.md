## Summary

Keep open. Current main has useful mitigations for session-store capping, diagnostics, and one known agent-event sequence leak, but the reported broad long-uptime gateway OOM is still plausible and not proven fixed. The discussion includes recent multi-user data beyond Discord and webchat, including Telegram-only heavy tool/compaction crashes, and current source still has the relevant polling, whole-store, and whole-transcript hot paths.

## Best Possible Solution

Keep this issue open as the broad gateway heap-exhaustion tracker. Link it with #55334 for the sessions.json/session-store portion, but do not close it unless the Telegram-only compaction-heavy and general long-uptime OOM paths are split into narrower canonical issues or fixed. The best maintainer path is to collect a diagnostics export plus heap snapshots from a current build, then add targeted fixes such as bounded/push-based node state refresh, lazy or paged session-store listing, streaming/tail-limited chat history reads, snapshot deduplication, and cleanup tests for any proven retained runtime state.
