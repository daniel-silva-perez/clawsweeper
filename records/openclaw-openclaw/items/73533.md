## Summary

Keep open. Current main still prefers `/tmp/openclaw` before any Windows platform guard, so this PR's intended Windows temp-dir fix is not implemented on main. The PR remains a focused implementation candidate for #60713, but the hydrated PR diff still has the `path.win32.join` test expectation mismatch already flagged in review.

## Best Possible Solution

Keep this PR open as the implementation candidate for #60713. Update the Windows regression test to construct the expected fallback with native `path.join`, then run the targeted `src/infra/tmp-openclaw-dir.test.ts` coverage and the changed gate before merging the focused resolver change.
