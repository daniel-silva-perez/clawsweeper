## Summary

Keep this PR open. Current main at c0ec58f4 still resolves `/tmp/openclaw` before any Windows platform guard, while logs, media, TTS, and several plugin temp paths depend on that resolver and gateway locks independently use `os.tmpdir()`. The PR is a focused implementation candidate for the linked Windows split-temp-state bug, not obsolete on main.

## Best Possible Solution

Keep this PR open as the focused implementation candidate for #60713. Review the Windows guard, run targeted temp/media tests plus the changed gate/API proof, then land or request revisions; once merged, the linked Windows temp-directory issue can close as implemented.
