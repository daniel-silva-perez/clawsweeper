## Summary

Keep open. Current main still has the Windows split-state behavior described by the linked bug, while this PR is the focused implementation candidate and its latest head addresses the review feedback shown in the discussion.

## What This Changes

The PR updates the shared OpenClaw temp-dir resolver to skip the POSIX `/tmp/openclaw` preferred path on `win32`, adds a platform test seam and regression coverage, uses a Windows joiner for injected Windows fallback paths, and adjusts a WhatsApp media test that mocks `process.platform`.

## Best Possible Solution

Review and land the focused PR if maintainers are satisfied, preserving the existing POSIX behavior on non-Windows platforms while routing Windows temp files through the `os.tmpdir()/openclaw-<uid>` fallback so #60713 closes through the merge.
