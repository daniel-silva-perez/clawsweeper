## Summary

Keep this PR open. Current main still sends every Discord inbound job through a per-session `KeyedAsyncQueue` before downstream reply queue settings resolve `steer` or `steer-backlog`, so the PR’s intended behavior is not implemented. The PR also targets the old `src/discord` layout while current Discord code lives under `extensions/discord`, so it needs a rebase or replacement rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer follow-up or replace it with a rebased change against `extensions/discord`. The best fix is to resolve the effective queue mode before the Discord inbound worker queue, bypass the outer per-session `KeyedAsyncQueue` for `steer` and `steer-backlog`, preserve serialization for non-steer modes including session overrides, and add regression tests for steer, steer-backlog, session override precedence, replay dedupe, timeout fallback, deactivation, and run-state accounting.
