## Summary

Keep this PR open. Current main still routes every Discord inbound job through a per-session KeyedAsyncQueue before downstream queue-mode handling can apply steer or steer-backlog behavior, and the PR's intended fix is not implemented on the current extensions/discord path. The PR is also stale against the current plugin layout because it edits old src/discord paths, so the useful work needs a rebase or replacement rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer follow-up or replace it with a rebased implementation under extensions/discord. The best fix is to resolve the effective Discord queue mode early enough to bypass the outer per-session KeyedAsyncQueue for steer and steer-backlog, while honoring session-level overrides and preserving serialization for collect, followup, interrupt, and other non-steer modes. Add regression coverage for steer, steer-backlog, session override precedence, replay dedupe, timeout fallback, deactivation, and busy/run-state accounting.
