## Summary

Keep open. Current main still implements restart-sentinel wake and restart-continuation recovery on a session-scoped path; it does not enumerate all agent session stores or broadcast a GatewayRestart system event to recently active sessions. This PR remains meaningful unmerged work, but the branch is stale against the current restart-sentinel startup task, durable session-delivery queue, and newer main-session recovery behavior.

## Best Possible Solution

Keep this PR open for maintainer review or replace it with a current-main implementation. The best fix should integrate with `loadRestartSentinelStartupTask`, `enqueueRestartSentinelWake`, durable restart-continuation delivery, and main-session restart recovery; enumerate known agent session stores using the current `SessionEntry` timestamp/status contract; skip pseudo, cron, ACP, subagent, and otherwise non-resumable sessions; enqueue GatewayRestart and request a heartbeat wake for every selected session; and add focused tests for cross-session selection, dedupe, no-route behavior, and stale-session filtering.
