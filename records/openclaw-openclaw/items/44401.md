## Summary

Keep open. Current main still wakes only the restart-sentinel session for the Gateway restart system event; it does not enumerate recently active agent session stores or broadcast GatewayRestart across active sessions. This PR remains meaningful unmerged work, but it likely needs a current-main rebase over the durable restart-continuation queue and focused cross-session wake tests.

## Best Possible Solution

Keep this PR open for maintainer review or replace it with an updated current-main patch. The best fix should integrate with the current restart-sentinel startup task, durable session-delivery queue, and heartbeat wake plumbing; enumerate known agent session stores using the agreed session timestamp/status contract; skip pseudo, cron, ACP, and otherwise non-resumable sessions; enqueue GatewayRestart plus heartbeat wake only for selected sessions; and add focused tests for cross-session selection, dedupe, and no-route behavior.
