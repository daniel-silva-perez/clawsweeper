## Summary

Keep this issue open. Current main has append-only heartbeat transcripts plus selective filtering of `HEARTBEAT_OK` no-op pairs, but the reported default path still runs heartbeats in the main session and does not remove or isolate prior successful `sent` heartbeat turns. The opt-in `isolatedSession: true` setting is a workaround/alternate mode, not an implementation of the expected default behavior for the reported config.

## Best Possible Solution

Keep this as an OpenClaw core heartbeat bug. The best fix is to make scheduled heartbeat cycles independent in the default main-session path, either by filtering/pruning all heartbeat-generated sent turns and their tool context before the next heartbeat prompt or by making an equivalent isolated heartbeat context the default without losing delivery routing and workspace HEARTBEAT.md injection. Add a regression test for a `sent` heartbeat followed by the next scheduled cycle proving the prior sent/tool activity is not visible enough to cause a false `ok-token` skip.
