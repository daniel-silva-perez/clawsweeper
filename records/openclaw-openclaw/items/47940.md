## Summary

Keep open. Current main still runs default heartbeats in the main session, makes isolated heartbeat context opt-in, and filters only HEARTBEAT_OK no-op pairs from replay. Meaningful sent heartbeat/tool turns remain eligible for subsequent heartbeat context, so the reported effective 2x interval is not resolved on current main.

## Best Possible Solution

Keep this as an OpenClaw core heartbeat bug. Maintainers should decide whether scheduled heartbeats should default to isolated context or whether heartbeat-triggered replay should filter prior heartbeat-generated sent/tool turns while preserving HEARTBEAT.md/workspace injection, delivery routing, dedupe state, and append-only transcripts. Add regression coverage proving a meaningful sent/tool heartbeat cannot cause the next scheduled heartbeat to falsely short-circuit to HEARTBEAT_OK.
