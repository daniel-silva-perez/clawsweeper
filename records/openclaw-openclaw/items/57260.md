Keep this PR open. Current main still lacks the Discord-specific mutation fingerprint normalization described by the PR, so the stale `Message failed` path is not implemented on main. The PR is also draft and mixes an unrelated manual cron lane change into the Discord fix, so the best path is focused maintainer review/rebase rather than cleanup closure.

## Best Possible Solution

Keep this PR open as a plausible focused bug fix candidate, but require a rebase onto current main, split or remove the unrelated cron commit, and land the Discord mutation-fingerprint normalization with the two regression tests described in the PR body. The related ClawSweeper reports about inbound Discord dedupe are separate problems and do not supersede this outbound stale-warning fix.
