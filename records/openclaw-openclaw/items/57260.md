## Summary

Keep this PR open. Current main still does not implement the PR's core Discord mutation-fingerprint behavior: `message` tool `to`/`target` values are fingerprinted through the generic alias path, so a bare numeric Discord target and the matching `channel:` retry remain different actions. Main does normalize Discord targets in the normal outbound send path, which may reduce the original stale-warning path, but that is not the same as the retry/error-dedupe contract proposed here. The PR is also draft and includes an unrelated cron lane change, so it needs focused maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer judgment. Ask for a rebase and split/remove the unrelated cron lane commit, then either land a focused Discord mutation-fingerprint normalization with the two proposed regression tests or close it manually after confirming current message-action target normalization fully supersedes the stale-warning behavior.
