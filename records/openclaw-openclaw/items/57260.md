## Summary

Keep this PR open. Current main still does not implement the PR’s actual Discord mutation-fingerprint normalization: bare numeric Discord `target`/`to` and `channel:<id>` still fingerprint differently, so the regression tests proposed by the PR would not pass as written. Main does now normalize Discord message-action targets before dispatch in the normal send path, which may make the original ambiguous-recipient symptom less reachable, but that is not the same as implementing the retry/error-dedupe behavior this PR changes. The PR is also draft and includes an unrelated cron lane change, so it needs focused maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer judgment, but require a rebase and split/remove the unrelated cron commit. Then either land a focused Discord mutation-fingerprint normalization with the two regression tests, or close it manually after confirming current message-action target normalization fully supersedes the stale-warning path.
