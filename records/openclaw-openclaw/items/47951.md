## Summary

Keep open. Current main still accepts ACP spawns after dispatch/background-task setup, while the child session transcript user row is only appended after `runTurn` completes. That does not close the reported accepted-but-empty `sessions_history` failure mode for stalled ACP/Codex turns.

## Best Possible Solution

Keep this issue open and change the ACP spawn/turn contract so an accepted child session has durable diagnostic state before acceptance or before `runTurn` can stall: either append a first user/task transcript row up front, or make an equivalent task/session event visible through `sessions_history` or status. Add a regression test where ACP dispatch is accepted but `runTurn` stalls or fails before output, proving the child session is no longer empty and the eventual post-turn persistence does not duplicate the prompt.
