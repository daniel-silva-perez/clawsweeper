## Summary

Keep open. Current main still has the minimum visibility gap: native subagent orchestration exits the parent turn on `sessions_yield`, does not expose a requester-scoped waiting/progress state, and leaves child-run lifecycle/tool events scoped away from the parent Control UI/TUI chat surface.

## What This Changes

The issue requests parent-session progress or streaming while `sessions_yield` waits on a spawned native subagent so Control UI and TUI do not appear frozen.

## Best Possible Solution

Add a generic requester-scoped native subagent waiting/progress signal that starts when the parent yields with active child work and clears on child completion, error, kill, pause, or parent resume; render that state in Control UI and TUI without changing ACP `streamTo` semantics or forwarding raw child token deltas.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: spawn a native subagent, call `sessions_yield`, and inspect the parent Control UI/TUI session while the child is active; current code terminates the parent run and lacks a parent-scoped wait indicator.

## Solution Assessment

Yes for the minimum bug fix: a requester-scoped lifecycle/progress event is the narrowest maintainable solution. Full native `streamTo: "parent"` forwarding is a larger product/API decision and should stay separate unless maintainers explicitly choose it.
