## Summary

Keep open. Current main still has the reported parent-session visibility gap: native subagent orchestration treats `streamTo` as ACP-only, `sessions_yield` ends the parent attempt as an idle/end-turn state, and Control UI/TUI event handling does not maintain a requester-scoped waiting indicator while the child run continues.

## What This Changes

The issue requests progress visibility, and ideally child output forwarding, in Control UI and TUI while a yielded parent session waits for a native subagent.

## Best Possible Solution

Add a generic requester-scoped native subagent wait/progress signal that starts when the parent yields with active child work and clears on child completion, error, kill, pause, or parent resume; render that state in Control UI and TUI without changing ACP `streamTo` semantics or forwarding raw native child token streams.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: spawn a native subagent, call `sessions_yield`, then observe that current main ends the parent run and exposes no parent-scoped wait/progress state to Control UI or TUI while the child remains active.

## Solution Assessment

Yes for the minimum repair. A requester-scoped lifecycle/progress indicator is the narrowest maintainable fix; native `streamTo: "parent"` output forwarding should remain a separate maintainer API decision.
