## Summary

Keep open. Current main still ends the parent run on `sessions_yield` and does not emit a requester-scoped waiting/progress signal while a native subagent is running; `streamTo: "parent"` remains ACP-only, and both Control UI and TUI currently filter child-run events away from the parent chat surface.

## What This Changes

The issue requests parent-session progress or streaming while `sessions_yield` waits on a spawned native subagent so Control UI and TUI do not appear frozen.

## Best Possible Solution

Implement a generic requester-scoped subagent waiting/progress signal for native `sessions_yield` orchestration, render it in Control UI and TUI as an active waiting state, and leave native child stream forwarding as a separate explicit contract if maintainers want it later.

## Reproduction Assessment

Yes. A high-confidence reproduction path is to spawn a native subagent, call `sessions_yield`, and observe the parent Control UI/TUI session while the child is active; the code ends the parent run and currently emits no parent-scoped wait indicator.

## Solution Assessment

Yes for the minimum bug fix: a parent-session wait/progress lifecycle signal is the narrowest maintainable solution. Full native `streamTo: "parent"` forwarding is broader and should not be bundled into the first repair unless maintainers explicitly choose that API.
