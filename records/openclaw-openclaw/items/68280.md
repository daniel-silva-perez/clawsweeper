## Summary

Keep this PR open. Current main still lacks the PR's proposed fail-fast local probe auth result, and the PR remains an active implementation candidate, but it needs maintainer review before merge because it is far behind current main and still sits on credential-forwarding semantics for explicit probe URLs.

## What This Changes

The PR adds a `failureReason` path to gateway probe auth resolution, uses it to synthesize failed local probe/status results instead of attempting anonymous loopback probes when required auth is missing, and adds focused regression tests for auth-mode guards and explicit URL overrides.

## Best Possible Solution

Keep this PR open as the implementation candidate.
