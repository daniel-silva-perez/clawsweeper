## Summary

Keep open: current main still contains the reported source-level failure path, and the linked implementation candidate has not landed. The gateway tool still relies on an emit hook to write the restart sentinel continuation, while the already-in-flight coalescing branch returns success before any hook can run.

## What This Changes

The issue reports that `gateway` restart calls with `continuationMessage` can return successful `coalesced: true` responses while dropping the continuation when another restart is already in flight.

## Best Possible Solution

Keep this issue open until PR #73812 or an equivalent narrow fix lands, with behavior that either durably persists or merges the continuation for already-in-flight coalesced restarts, or explicitly reports that no continuation was queued. The merged fix should include a regression test for the unconsumed SIGUSR1 branch.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to create an unconsumed SIGUSR1 restart signal, then invoke the gateway restart tool with `continuationMessage`; current main returns from the in-flight coalescing branch before the gateway tool's sentinel-writing hook can run.

## Solution Assessment

No. Current main is not the best solution because it can report a successful coalesced restart while losing the accepted continuation; the narrow maintainable fix is to make continuation queuing explicit and tested in the coalesced path.
