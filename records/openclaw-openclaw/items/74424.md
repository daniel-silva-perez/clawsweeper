## Summary

Keep open. Current main still has the reported in-flight coalescing gap: the gateway restart tool accepts a continuation and returns the scheduler result, but the already-emitted restart path returns `coalesced: true` before the tool's sentinel-writing hook can run. The linked open PR is adjacent and may be the right place to handle it, but no fix is merged on current main, and the closed follow-up PR was not merged.

## What This Changes

The issue requests that `gateway` restart calls with `continuationMessage` either durably deliver that continuation after a coalesced restart or explicitly report that it was not queued.

## Best Possible Solution

Keep restart coalescing, but make the accepted continuation contract truthful: atomically attach or queue the explicit continuation for the already in-flight restart and drain it once after startup, or return a clear not-queued/error result so the agent can recover, with focused regression coverage for the in-flight coalesced path.
