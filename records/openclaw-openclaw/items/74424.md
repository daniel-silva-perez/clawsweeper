## Summary

Keep open: current main still has the reported source-level failure path, and the linked implementation candidate has not landed. The gateway restart tool still writes the continuation only from a restart emit hook, while the already-in-flight coalescing branch returns a successful `coalesced: true` result before any hook can run.

## What This Changes

The issue reports that a gateway restart request with `continuationMessage` can return success with `coalesced: true` while failing to persist or deliver that continuation after restart.

## Best Possible Solution

Land PR #73812 or an equivalent narrow fix that either durably persists or merges the continuation for coalesced restarts, or explicitly reports that no continuation was queued, with a regression test for the already-in-flight SIGUSR1 branch.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to create an unconsumed SIGUSR1 restart signal, then invoke the gateway restart tool with `continuationMessage`; current main returns from the in-flight coalescing branch before the gateway tool's sentinel-writing hook can run.

## Solution Assessment

No. Current main is not the best solution because it can report a successful coalesced restart while losing an accepted continuation; the maintainable fix is to make continuation queuing explicit and tested in the coalesced path.
