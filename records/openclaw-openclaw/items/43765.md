## Summary

Keep open. Current main does not already implement the PR’s intended runtime-recovery changes, and the PR head is not merge-ready because it contains high-confidence static blockers plus ACPX dependency drift that needs maintainer review before any merge or replacement.

## What This Changes

The PR changes heartbeat system-event consumption, Feishu runtime health snapshots, exec/process terminal status semantics, ACPX/Codex ACP launch behavior, ACP spawn guidance, first-visible reply diagnostics/config/docs, lockfile entries, and related tests.

## Best Possible Solution

Rebase or replace this as smaller current-main changes: handle Feishu health reporting through #39489 or a reconciled successor, decide the exec non-zero exit semantic change with compatibility-focused tests, keep heartbeat event consumption aligned with current prefix-consumption semantics, and review ACPX/Codex ACP launch changes only against the current dependency baseline.

## Reproduction Assessment

Yes for the PR defects. The PR head raw files show duplicate top-level Feishu declarations, an import of a missing session-generation module, and a drained-event path that returns without restoring pending heartbeat events; this review stayed read-only and did not run tests.

## Solution Assessment

No. The current PR is not the best way to solve the remaining runtime issues because it combines several stale runtime, dependency, diagnostics, docs, and config changes; a maintainer-directed split or replacement is safer and easier to validate.
