## Summary

Keep this PR open for maintainer handling, not cleanup closure. Current main only covers the heartbeat replay slice and has a newer ACPX Codex adapter path; Feishu health reporting, exec/process terminal-status semantics, and first-visible diagnostics are still not implemented on main, while the PR head itself has import/parse blockers and supply-chain-sensitive ACPX executable changes.

## What This Changes

The PR changes heartbeat system-event handling, Feishu channel health snapshots, exec/process terminal status semantics, ACPX Codex command resolution, ACP spawn guidance, reply first-visible diagnostics, config schema, and related docs/tests.

## Best Possible Solution

Ask for a current-main rebase or replacement split: drop the already-covered heartbeat replay work, compare ACPX changes against main's newer Codex ACP implementation, review or land the narrower Feishu health PR, and handle exec terminal-status semantics plus first-visible diagnostics as separate product-reviewed changes with focused tests and docs.
