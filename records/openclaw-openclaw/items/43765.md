## Summary

Keep this PR open as candidate work, not cleanup-closeable. Current main has independently covered the heartbeat replay slice and now has a newer ACPX Codex adapter implementation, but Feishu websocket health snapshots, exec foreground non-zero failure semantics, first-visible diagnostics/design docs, and some ACP spawn/tooling semantics from this PR remain unmerged or need separate maintainer/security review.

## Best Possible Solution

Keep this PR open only as candidate work. Ask for a current-main rebase or replacement split: drop/compare the heartbeat slice against main's shipped consume-on-success implementation, compare/drop ACPX Codex-adapter work against main's newer implementation, land Feishu websocket health as a focused bundled-plugin PR, handle exec terminal-status semantics as a separate product-reviewed PR, and design first-visible diagnostics independently with tests and docs.

## Risks / Open Questions

- The PR is XL and mixes heartbeat, Feishu channel health, exec/process semantics, ACPX dependency and process-launch behavior, ACP spawn metadata, first-visible diagnostics, and docs, which makes conservative review and rollback difficult.
- The ACPX portion touches executable third-party packages and command resolution. Even though current main has a newer bundled Codex ACP adapter, any remaining branch changes in that area need explicit maintainer/security review after rebase.
- Changing all non-zero foreground exec exits from completed to failed is a user-visible tool semantics change and may break workflows that intentionally use non-zero exit codes such as grep no-match or diff differences.
- Provided review comments report module-load blockers on the branch, so this PR cannot be treated as already-implemented or ready-to-close from current main evidence.
