## Summary

Keep this PR open as candidate work, not conservative cleanup. Current main has independently covered the heartbeat replay slice and now has newer ACPX/Codex ACP support, but the Feishu websocket health snapshot/restart path, foreground exec failure semantics, process terminal-status preservation, and first-visible diagnostics/design work from this PR are still not implemented on main. The PR also remains broad and security-sensitive enough that it needs a rebase/split and maintainer review rather than closure as obsolete.

## Best Possible Solution

Keep this PR open only as candidate work. Ask for a current-main rebase or replacement split: drop/compare the heartbeat slice against main's shipped consume-on-success implementation, drop/compare ACPX/Codex ACP work against main's newer acpx implementation, land Feishu websocket health as a focused bundled-plugin PR, handle exec terminal-status and non-zero exit semantics as a separate product-reviewed PR, and design first-visible diagnostics independently with tests and docs.

## Risks / Open Questions

- The PR is XL and mixes heartbeat, Feishu channel health, exec/process semantics, ACPX dependency and launch behavior, ACP spawn guidance, diagnostics, and docs, which makes review and rollback difficult.
- Changing all non-zero foreground exec exits from completed to failed is user-visible and may break workflows that intentionally use non-zero codes such as grep no-match or diff differences.
- The ACPX portion touches executable third-party packages and command resolution; current main has newer Codex ACP support, so retained branch changes need fresh supply-chain/security review after rebase.
- Provided review comments report branch-level P0 blockers, so this PR should not be treated as ready or already implemented without a rebase and validation.
