## Summary

Keep this open. Current main has the intended Anthropic plus `claude-cli` runtime mapping, dispatch path, plugin auto-enable coverage, and unit tests, but that is not enough to prove the packaged 2026.4.26 cold-start Telegram path is fixed. The report describes a privacy-sensitive provider substitution before normal fallback telemetry, and I found no current-main marker or regression proof for this exact cold-start/auth/session/hot-reload path.

## What This Changes

The issue asks OpenClaw to honor `agents.defaults.agentRuntime.id = "claude-cli"` on a fresh 2026.4.26 install and avoid silently routing Anthropic CLI-configured turns to the OpenRouter free fallback without diagnostic events.

## Best Possible Solution

Add a focused packaged-install reproduction path for 2026.4.26 or newer that captures loaded config, session entry, plugin registry, auth-profile resolution, selected provider/model/runtime, and trajectory events for the first Telegram turn; if reproduced, make explicit `claude-cli` runtime selection fail closed or emit trajectory-visible runtime-skip diagnostics before any provider substitution can run.
