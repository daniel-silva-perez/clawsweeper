## Summary

Keep this open. Current main has the intended canonical Anthropic plus `claude-cli` runtime dispatch path and focused unit coverage, but that does not prove the fresh packaged 2026.4.26 cold-start report is fixed: the reported behavior appears to select OpenRouter before normal fallback telemetry, and I found no current-main fix marker for this exact packaged hot-reload/auth/session path.

## What This Changes

The issue reports that a cold-start 2026.4.26 install with `agents.defaults.agentRuntime.id = "claude-cli"` silently routes Anthropic turns to OpenRouter `gpt-oss-120b:free` instead of using Claude CLI or logging a failover decision.

## Best Possible Solution

Add a focused triage/reproduction path for a clean packaged 2026.4.26-or-newer install that captures loaded config, session entry, plugin registry, auth-profile store, selected model/runtime, and trajectory events; if confirmed, make explicit `claude-cli` runtime selection fail closed or emit trajectory-visible runtime-skip diagnostics before any provider/model substitution is allowed.
