## Summary

Keep this PR open. Current main still indexes usage-counted session transcripts, including reset/deleted archive variants, in both the source SDK and the package SDK copy. The PR is therefore not obsolete; it remains an unmerged behavior/product change that needs maintainer review, especially because the latest PR context also includes unrelated plugin loader, doctor, gateway, and browser test-isolation changes plus a security-sensitive bundled-plugin trust seam.

## Best Possible Solution

Keep this PR open for normal maintainer review. If maintainers accept primary-transcript-only memory indexing, land a scoped version that updates both `src/memory-host-sdk` and `packages/memory-host-sdk`, keeps exact compaction-checkpoint filename filtering, updates tests and any needed docs/changelog, and separates or explicitly justifies the unrelated plugin/gateway/doctor/browser changes. If maintainers want reset/deleted transcripts to remain part of shipped QMD recall, close or revise the PR on that product basis.
