## Summary

Keep this PR open. Current main still does not implement primary-transcript-only memory-host SDK session discovery: both source copies still use the usage-counted transcript helper, tests still assert reset/deleted transcript inclusion, and the changelog records that reset/deleted QMD session export was intentionally added to preserve historical recall. The PR remains a plausible implementation candidate if maintainers want to reverse or narrow that product behavior, but its latest head is broader than the title and needs normal review or splitting.

## Best Possible Solution

Review this PR normally rather than closing it. If maintainers accept primary-transcript-only session indexing, land a scoped version that keeps `src/memory-host-sdk` and `packages/memory-host-sdk` synchronized, preserves exact checkpoint matching and dotted session-id coverage, updates the tests, and either splits or clearly justifies the unrelated browser, Signal, doctor, channel, and plugin SDK changes. If maintainers want the shipped reset/deleted QMD recall behavior to remain, close or revise the PR on that explicit product basis.
