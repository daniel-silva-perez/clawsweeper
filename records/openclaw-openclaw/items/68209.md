## What This Changes

The issue asks OpenClaw to keep a switch from the PI `openai-codex/gpt-5.4` route to the native `codex/gpt-5.4` route from causing runaway context reporting, off-task workspace contamination, token/cost growth, or auth-mode drift.

## Best Possible Solution

Keep this as a focused child of #66251 until a maintainer captures a clean current-main route-switch trace showing stable `agentHarnessId`, fresh or explicitly unknown context accounting, app-server auth/runtime events, and Codex login status before and after the switch; any remaining narrow defect should then be split to the Codex app-server, status, or auth path.
