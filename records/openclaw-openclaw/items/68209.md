## Summary

Keep open. Current main has adjacent hardening for Codex harness selection, stale context display, cumulative usage accounting, and OAuth/API-key isolation, but it does not prove the full route-switch regression is fixed. Native Codex compaction still does not record a post-compaction token count from completion events, so this remains a useful focused child of the broader Codex harness stability tracker.

## What This Changes

The issue asks OpenClaw to handle switching from the PI OpenAI Codex OAuth path to the native Codex harness without runaway context growth, off-task workspace contamination, auth-mode drift, or misleading context reporting.

## Best Possible Solution

Keep this issue open as a focused repro under #66251. The desired end state is a current-main route-switch proof with a clean session boundary, stable agentHarnessId/runtime state, fresh or explicitly unknown context accounting after native compaction, and clear auth-mode isolation; any remaining narrow defect should be split to the relevant Codex app-server, status, or auth path.
