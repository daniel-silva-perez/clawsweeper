## Summary

Keep open. Current main supports the related merged PR’s `autoThreadName: "generated"` strategy, but it still uses a fixed Discord thread-title prompt and does not expose `threadPromptGuideline` or an equivalent customization field in config, schema, runtime wiring, tests, or docs.

## Best Possible Solution

Implement this as a focused Discord bundled-plugin enhancement: add a bounded optional channel-level guideline field to Discord config, validate length in the schema, carry it through resolved channel config into thread-title generation, compose it with the existing safe system prompt and channel/message context, and document/test the backward-compatible fallback behavior.
