## Summary

Keep open. Current main supports generated Discord auto-thread titles, but the generation prompt is still fixed and there is no Discord channel config field, schema entry, docs entry, or runtime wiring for a custom title guideline; the linked merged PR introduced the generated-title mode but does not cover this follow-up customization request.

## What This Changes

Requested an optional Discord `threadPromptGuideline` channel setting to customize the LLM prompt used when `autoThreadName` is `generated`.

## Best Possible Solution

Add a backwards-compatible Discord channel config field that is treated as an operator-supplied title guideline, clamps length, preserves the default concise/title-only guardrails, and is documented with schema/help/tests so absent config keeps the current generated-title behavior unchanged.

## Reproduction Assessment

Not applicable as a bug reproduction, but there is a high-confidence verification path for the missing feature: current main has no config key and the generated-title code always uses the built-in prompt.

## Solution Assessment

Yes, a narrow additive per-channel Discord config field is a maintainable fit for this request, provided implementation appends or scopes the guideline under the existing title-generation constraints instead of replacing the whole safety/format prompt.

Implement optional `threadPromptGuideline` support for generated Discord auto-thread titles. Scope the change to the bundled Discord/config/docs surfaces: add the optional string field to `DiscordGuildChannelConfig` and the zod config schema with a 500-character max; update config help/generated metadata as required by existing scripts; pass the resolved channel config value from `maybeCreateDiscordAutoThread` into `generateThreadTitle`; and compose it as an operator guideline while preserving the existing concise/title-only default constraints and absent-field behavior. Add focused tests showing the field is accepted, overlong values are rejected, the auto-thread runtime passes the guideline, and the completion context includes the guideline only when configured. Update `docs/channels/discord.md` and add a user-facing changelog entry thanking @stanislasbdx. Do not move Discord behavior into core, do not change the default generated-title prompt for users without the new field, and do not add broad template-engine behavior.
