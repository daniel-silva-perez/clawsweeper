## Summary

Keep open. Current main still has a hardcoded generated Discord thread-title prompt and only supports `autoThreadName: "generated"`; the requested per-channel prompt guideline is not present in the config type, schema, resolved channel config, or title-generation call path. The feature is narrow and belongs in the bundled Discord plugin surface rather than a core migration or ClawHub close.

## What This Changes

The issue requests an optional Discord channel configuration field for customizing the prompt used when `autoThreadName` is `generated`.

## Best Possible Solution

Add a Discord-owned, backward-compatible `threadPromptGuideline` channel config option with a bounded string schema, carry it through resolved channel config into the generated-title helper, combine it with the existing message/channel context without changing default behavior, and document/test the new config surface.

## Reproduction Assessment

Not applicable as a bug reproduction. The feature gap is high-confidence from source inspection: current main has no accepted `threadPromptGuideline` config key and no runtime path that can influence the hardcoded generated-title prompt.

## Solution Assessment

Yes, the narrowest maintainable direction is an additive Discord channel config field wired through the existing generated-title helper. This should not require core IDs, startup migrations, or a new plugin API because the behavior is already owned by the bundled Discord plugin.
