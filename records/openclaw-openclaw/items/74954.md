## Summary

Keep open. The PR targets a valid current-main bug, but it is not safe to merge as-is because it blanket-removes deferred provider `agentModelOverride` behavior and would lose the selected provider default when there is no inherited `agents.defaults.model.primary` to use.

## What This Changes

The PR changes provider auth-choice/default-model handling and related tests so `setDefaultModel: false` no longer returns provider defaults as `agentModelOverride` during agent auth setup.

## Best Possible Solution

Keep the PR open but require a narrower implementation: suppress `agentModelOverride` only when an inherited `agents.defaults.model.primary` exists, preserve the provider default override when no inherited primary exists, and cover both string/object global defaults plus the no-default case.

## Reproduction Assessment

Yes. The current-main source path is clear: deferred provider auth returns `agentModelOverride`, and `agents add` writes it into `agents.list[]`; the linked #24170 steps exercise the configured-global-primary case. The PR regression is also source-reproducible by following a deferred provider setup with no global primary, where the PR would leave no explicit agent model to inherit.

## Solution Assessment

No. The PR’s direction is right for the configured-default case, but removing every deferred provider override is broader than the product contract; the safer fix is conditional suppression based on whether an inherited primary model exists.
