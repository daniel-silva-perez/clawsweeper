## Summary

Keep open. The PR targets a real current-main bug, but the patch is too broad: it removes deferred provider model overrides even for configs with no inherited `agents.defaults.model.primary`, while the linked history and earlier fix attempt preserve the provider default in that no-global-default case.

## What This Changes

The PR changes provider auth-choice/default-model handling and tests so `setDefaultModel: false` no longer returns provider defaults as `agentModelOverride` during agent auth setup.

## Best Possible Solution

Keep the PR open, but narrow it to suppress `agentModelOverride` only when an inherited `agents.defaults.model.primary` exists; preserve the provider default override when no inherited primary exists, and cover object-form defaults, string-form defaults, and the no-default case.

## Reproduction Assessment

Yes. The current-main source path is clear: deferred provider auth returns `agentModelOverride`, and `agents add` writes it into `agents.list[]`; the linked issue steps exercise the configured-global-primary case. The PR regression is source-reproducible by following a deferred provider setup with no global primary, where the patch leaves no explicit agent model to inherit.

## Solution Assessment

No. The PR direction is right for configs with a global primary, but removing every deferred provider override changes the no-global-default contract; the safer fix is conditional suppression based on whether an inherited primary model exists.
