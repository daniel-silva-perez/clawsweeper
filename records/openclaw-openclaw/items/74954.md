## Summary

Keep open. Current main still has the reported write-time model override path, and this PR targets the right area, but the patch is not safe to merge because it suppresses provider default overrides unconditionally instead of only when a configured inherited default model exists; the branch is also dirty against current main.

## What This Changes

The PR changes provider auth/default-model handling and related tests so `setDefaultModel: false` no longer returns provider defaults as `agentModelOverride` during new-agent auth setup.

## Best Possible Solution

Land a narrow conditional fix: when `agents.defaults.model.primary` already exists, omit the per-agent model override so the new agent inherits it; when no inherited primary exists, keep writing the selected provider default so the newly configured provider remains usable. Update focused tests for both cases before closing the related stale PRs.

## Reproduction Assessment

Yes. The current-main path is source-reproducible: `openclaw agents add` calls auth setup with `setDefaultModel: false`, the auth helper can return `agentModelOverride`, and the add command writes that override into `agents.list[]`; #24170 supplies concrete config and CLI steps for the configured-default case.

## Solution Assessment

No. The proposed direction is correct for configs that already have `agents.defaults.model.primary`, but the implementation removes the no-default fallback behavior as well; a conditional override based on the inherited primary is the narrower maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve provider defaults when no global model exists:** `src/plugins/provider-auth-choice.ts:437-440`

For `agents add` with no inherited `agents.defaults.model.primary`, the new agent will not receive the provider default it just configured and runtime selection can fall back to the product default instead. Suppress the override only when an inherited primary exists, and keep the provider override for the no-default case.
