## Summary

Keep open. The report is new, concrete, has a second reporter tying the regression window to 2026.4.24, and current main does not prove the latency class is fixed. Main has leaner raw model-run paths than v2026.4.26, but `openai-codex/gpt-5.5` still resolves to the Codex Responses transport with very large context/output metadata, and the Responses payload still forwards the selected max token cap as `max_output_tokens`.

## What This Changes

Investigate and fix a regression causing high latency for `openai-codex/gpt-5.5` across local infer, gateway infer, and agent turns after 2026.4.24.

## Best Possible Solution

Keep this open for maintainer reproduction on current main. Capture sanitized local infer, gateway infer, and `openclaw agent` timings; log request payload shape, `max_output_tokens`, first byte, total stream duration, prompt/cache/session setup, and Pi version. If confirmed, make a narrow provider/transport or raw-model-run fix with targeted regression coverage, likely around raw probe token caps or Codex Responses payload policy.
