## Summary

Keep this PR open. Current main still lacks the proposed `SO_NOSIGPIPE` hardening and the companion regression test for macOS exec approval socket descriptors, while the PR discussion supports the change as a focused defensive fix with one minor cleanup note.

## Best Possible Solution

Keep this PR open as a focused implementation candidate: rebase onto current main, apply SIGPIPE suppression to the client fd and each accepted server-side fd, avoid implying listener-socket inheritance, keep the descriptor regression test, and get targeted macOS IPC validation before merge.
