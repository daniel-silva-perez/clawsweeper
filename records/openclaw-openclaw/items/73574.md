## Summary

Current main still has the reported memory-wiki lint gap. The resolver only accepts exact extensionless relative paths and basenames, while Obsidian targets are kept raw and imported source-path/title/slug variants are not added to the lookup set. The issue also has same-day contributor follow-up, so conservative cleanup should leave it open.

## Best Possible Solution

Keep this issue open. Implement the normalization in `extensions/memory-wiki/src/lint.ts`, preferably by extracting shared target/key helpers that handle title keys, slugified title/path/basename keys, fragment-stripped Obsidian targets, normalized path variants, and imported `sourcePath`/bridge/unsafe-local suffix keys. Add focused positive and negative coverage in `extensions/memory-wiki/src/lint.test.ts`, then let the active contributor's focused fix proceed through normal review.
