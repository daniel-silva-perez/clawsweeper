## Summary

Current main still has the reported memory-wiki lint resolver gap. `wiki lint` validates raw wikilink targets only against exact extensionless relative paths and basenames, while imported pages already carry title and source provenance that the lint resolver does not use. A contributor has also commented that they can take a focused fix, so the best maintainer outcome is to keep this open for that implementation.

## Best Possible Solution

Keep this issue open. Implement the fix in `extensions/memory-wiki/src/lint.ts` by adding shared normalized lookup helpers for title, slugified title/path basename, fragment-stripped Obsidian targets, path/basename variants, and bridge/unsafe-local source-path suffixes. Add focused positive and negative regression tests in `extensions/memory-wiki/src/lint.test.ts`, ideally reusing or sharing normalization logic with compile/query where appropriate.
