## Summary

Keep this PR open. Current main at bdfb408ce6eeb724a638c106104dbbcae50b612c still does not implement the central watchdog behavior: ACP and subagent task creation omit `notifyPolicy: "state_changes"`, new running task records do not emit an immediate start update, and the direct-delivery fallback path still records `deliveryStatus: "failed"` even after queueing fallback work. The predecessor PRs are closed unmerged, so this remains the active implementation candidate.

## Best Possible Solution

Keep this PR open for maintainer review. If the watchdog behavior is desired, land or amend the PR to opt completion-visible ACP/subagent task creation into `state_changes`, emit an initial running/start state-change update from task creation, preserve `session_queued` when fallback queueing succeeds, update docs if defaults change, drop already-main or unrelated test churn, and verify with focused task-registry/ACP/subagent tests plus normal CI.
