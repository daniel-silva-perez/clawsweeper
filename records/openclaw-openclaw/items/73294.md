## Summary

Keep this PR open. Current main still does not implement the proposed Discord response watchdog behavior: ACP and subagent task creation do not opt into state-change notifications, newly-created running tasks do not emit an immediate start update, and direct-delivery fallback still records `failed` after a queued session fallback succeeds. The prior related PRs are closed unmerged, so this PR remains the active implementation candidate.

## Best Possible Solution

Keep this PR open for maintainer review. If the watchdog behavior is desired, land or amend the PR to explicitly set ACP/subagent notify policies, emit the initial running update from task creation, preserve `session_queued` after successful fallback queueing, update the background-task docs for any changed defaults, and verify with focused task-registry/ACP tests plus a scoped Discord delivery smoke.
