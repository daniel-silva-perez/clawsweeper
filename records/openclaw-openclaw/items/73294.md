## Summary

Keep this PR open. Current main does not implement the Discord response watchdog behavior proposed here: ACP/subagent task creation still defaults to `done_only`, newly-created running tasks do not emit an immediate state-change update, and direct delivery fallback still records `failed` even after queuing a session fallback. The related PRs #73232 and #73238 are closed unmerged, so this PR is the active implementation candidate rather than obsolete.

## Best Possible Solution

Keep this PR open for maintainer review. If the watchdog behavior is desired, land or amend this PR with focused task-registry/ACP/subagent tests, align the background task docs with the new notify default, and do a scoped Discord or heartbeat delivery smoke before merge.
