## Summary

Keep this PR open. Current main at f256eeba431b5f59e6085b7c11c9c0a309e408b7 still does not implement the watchdog behavior this PR targets: ACP/subagent task creation does not opt into state-change notifications, newly-created running task records do not emit an immediate started update, and direct-delivery fallback still records `failed` even when the session fallback is queued. The linked predecessor PRs are closed unmerged, so this PR remains the active implementation candidate.

## Best Possible Solution

Keep this PR open for maintainer review. If the watchdog behavior is desired, land or amend the PR to set `state_changes` for completion-visible ACP/subagent task creation, emit the initial running/start update from task creation, preserve `session_queued` after successful fallback queueing, update the background-task docs for changed defaults, remove or justify the unrelated active-memory test edit, and verify with focused task-registry/ACP/subagent tests plus normal CI.
