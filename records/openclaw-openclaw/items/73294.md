## Summary

Keep this PR open. Current main at 14f140d6f0d31fed802869622c38c7e8f8e19587 still does not implement the central watchdog behavior: ACP/subagent task creation does not opt into `state_changes`, newly-created running tasks do not emit an immediate start update from `createTaskRecord`, and direct-delivery fallback still records `failed` after queueing the requester-session fallback. The predecessor PRs #73232 and #73238 are closed unmerged, so this PR remains the active implementation candidate.

## Best Possible Solution

Keep this PR open for maintainer review. If the watchdog behavior is desired, land or amend it to set `state_changes` for the intended ACP/subagent task-creation paths, emit an immediate start update for newly-created running tasks, preserve `session_queued` after successful direct-delivery fallback queueing, update the background-task docs if defaults change, and verify with focused task-registry/ACP/subagent tests plus normal CI and any approved live Discord smoke.
