## Summary

Keep this PR open. Current main at 189c91eae6e1f4f280179961bca2c4153cf42e86 still lacks the proposed Discord response watchdog behavior: ACP and subagent task creation do not opt into state-change notifications, newly-created running tasks do not emit an immediate started update, and the direct-delivery fallback still records failed even after queue fallback succeeds. The related PRs #73232 and #73238 are closed unmerged, so this PR remains the active implementation candidate rather than obsolete.

## Best Possible Solution

Keep this PR open for maintainer review. If the watchdog behavior is desired, land or amend this PR with the focused task-registry and ACP/subagent coverage, update the background-task docs for the new notification default, and do a scoped Discord or delivery smoke before merging.
