## Summary

Keep this PR open. Current main still lacks the requested response-prefix context variables, and this PR is a real implementation candidate, but the current patch is not merge-ready because it can compute context usage from aggregate totals that include output tokens and appears to wire the new callback through only part of the channel delivery surface.

## What This Changes

This PR adds `{context}` and `{contextPercent}` response-prefix template variables, an `onContextUsage` callback, runtime usage population, and focused tests for the new interpolation path.

## Best Possible Solution

Revise the PR to derive usage from prompt-side context counters, thread the new callback wherever `onModelSelected` is passed, document the exact `{context}` and `{contextPercent}` semantics in config docs/type help, add a changelog entry, and validate the changed auto-reply/channel/plugin-sdk lanes.

## Reproduction Assessment

Yes. On current main, configuring a response prefix with `{context}` or `{contextPercent}` will leave those placeholders unresolved because the template resolver and prefix context lack those fields; on the PR branch, a mocked `lastCallUsage` with a large `output`/`total` value reproduces the inflated percentage bug.

## Solution Assessment

No. The PR direction is the right narrow area, but the implementation should use existing prompt-context helpers such as `deriveContextPromptTokens` and should centrally propagate `onContextUsage` through all channel delivery paths instead of relying on partial call-site wiring.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Compute context from prompt tokens only:** `src/auto-reply/reply/agent-runner-execution.ts:1250-1254`
  - body: Using `lastCallUsage.total` here makes `{context}` include completion/output tokens for providers that report aggregate totals. That can overstate context-window pressure and clamp `{contextPercent}` to 100% after a long answer even when the prompt side is still below the limit. Reuse the prompt-side context helper or input/cache counters consistently.
  - confidence: 0.93
- **[P2] Thread usage callbacks through channel replies:** `src/channels/reply-prefix.ts:24`
  - body: Adding `onContextUsage` to `ReplyPrefixOptions` is not enough for existing channel callers that destructure only `onModelSelected` and pass only that into `replyOptions`; the new callback remains outside `runAgentTurnWithFallback`, so `{context}` stays literal on those surfaces. Sweep the channel call sites or centralize the reply option wiring beside `onModelSelected`.
  - confidence: 0.86
