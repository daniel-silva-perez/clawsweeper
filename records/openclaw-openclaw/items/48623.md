## Summary

Keep open. Current main still records `lastGood` but intentionally does not use it for auth-profile ordering; session override and embedded-runner rotation continue to walk the precomputed round-robin order. Slug timeout handling has a partial general timeout improvement, but the issue's main lastGood/failover-ordering request and per-task model override remain unresolved.

## Best Possible Solution

Keep this issue open. Implement failover-aware auth-profile selection that can prefer the most recently good profile after rate-limit/auth failures while preserving normal round-robin behavior where intended. Add focused coverage for the resolver, session override selection, and embedded-runner profile rotation path. Treat slug-generator model override and embedded-run cooldown isolation as follow-up scope or split issues if maintainers want a narrower fix.
