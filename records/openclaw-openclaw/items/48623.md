## Summary

Keep open. Current main still records lastGood but intentionally does not use it to order auth-profile candidates, and both session override selection and embedded-runner rotation consume that round-robin/precomputed order. The slug generator now honors the general agent timeout, and embedded attempts appear to get their own timeout timer, but the central lastGood/failover-ordering request and per-task model override remain unresolved.

## Best Possible Solution

Keep this issue open. Implement failover-aware auth-profile selection that can prefer the most recently good profile after rate-limit/auth failures while preserving normal round-robin behavior where intended. Add focused coverage for resolveAuthProfileOrder, resolveSessionAuthProfileOverride, and embedded-runner profile rotation. Split or explicitly scope the slug-generator model override and embedded-run cooldown isolation follow-up if maintainers want a narrower fix.
