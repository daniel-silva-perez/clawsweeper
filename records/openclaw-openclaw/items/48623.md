## Summary

Keep open. Current main still intentionally ignores `lastGood` during auth-profile ordering, and tests assert that behavior. The embedded-runner timeout path and slug-generator timeout have partial improvements, but they do not resolve the primary failover ordering problem or the broader embedded-run/cooldown isolation concern raised in the discussion.

## Best Possible Solution

Keep this issue open. Implement failover-aware auth-profile ordering that can prefer the most recently good profile after rate-limit/auth failures while preserving normal round-robin behavior, and add coverage in both the resolver tests and embedded-runner auth-profile rotation tests. Treat the slug-generator model override and embedded-run cooldown isolation as follow-up scope or split issues if maintainers want them handled separately.
