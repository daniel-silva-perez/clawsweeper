## Summary

Keep open. Current main still does not implement this PR's intended detached Windows `update.run` handoff for Control UI/Gateway updates. The current Gateway RPC path runs the update inside the live gateway, the global npm updater still swaps the active package tree with `fs.rename`, and the newer CLI service-stop/update/restart mitigation is a separate shell command path. The PR also has unresolved security-sensitive process/command-lifecycle concerns, so it needs maintainer review or contributor revision rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review or contributor revision. The best path is a current-main Windows package-update design for `update.run` that avoids replacing the live package tree while the gateway is running, reuses the CLI service-stop/update/restart flow where appropriate, preserves `createGlobalInstallEnv`, npm `--omit=optional` fallback, safe argv escaping, supervisor/fallback restart behavior, and startup/result reconciliation, with explicit tests for Windows Gateway RPC package updates. If a newer canonical PR fully fixes #40540 for the Gateway RPC surface, this PR can then be closed as superseded by that merged implementation.
