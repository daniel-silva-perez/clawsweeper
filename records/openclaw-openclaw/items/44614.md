## Summary

Keep open. Current main still does not implement this PR's intended Windows `update.run` detached package-update handoff. The Gateway RPC path still runs the update inside the live gateway and then schedules the normal restart after success, while the shared npm package updater still swaps the live package tree with `fs.rename`. Current CLI service-stop/restart improvements help shell-driven package updates, but they do not cover the Control UI/Gateway RPC surface this PR targets. The PR also has unresolved security-sensitive command/process lifecycle review concerns, so it needs maintainer review or contributor revision rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review or contributor revision. The best path is a current-main Windows package-update design for `update.run` that avoids replacing the live package tree while the gateway is running, aligns with the existing CLI service-stop/update/restart helper where appropriate, preserves `createGlobalInstallEnv`, npm `--omit=optional` fallback, safe argv escaping, supervised and fallback restart behavior, startup/result reconciliation, and adds explicit tests for Windows Gateway RPC package updates. If a newer canonical PR fully fixes #40540 for the Gateway RPC surface, this PR can then be closed as superseded by that merged implementation.
