## Summary

Keep open. Current main does not definitely implement this PR's intended Windows `update.run` detached update flow, and the related Windows EBUSY bug remains open as #40540. Main has improved the CLI package-update path, but the Control UI/Gateway RPC path still calls `runGatewayUpdate()` from inside the live gateway and then schedules a SIGUSR1 restart only after success.

## Best Possible Solution

Keep this PR open for maintainer review or contributor revision. The best fix is a current-main Windows package-update design that either reuses the CLI's service-stop/update/restart flow or adds a supervised detached handoff for `update.run` before the live package-tree swap, while preserving `createGlobalInstallEnv`, npm omit-optional fallback, safe argv escaping, startup/result reconciliation, and explicit tests for supervised and unsupervised Windows gateway installs. If a newer canonical PR fully fixes #40540, this PR can then be closed as superseded by that merged implementation.
