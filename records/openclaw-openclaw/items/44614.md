## Summary

Keep open. Current main does not implement this PR’s intended Windows `update.run` detached package-update handoff. The Control UI/Gateway RPC path still runs the package update inside the live gateway and only schedules the normal restart after success, while the related Windows EBUSY bug remains tracked by open issue #40540. The PR also contains security-sensitive process/command changes with unresolved review concerns, so it needs maintainer review or contributor revision rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review or contributor revision. The best path is a current-main Windows package-update design for `update.run` that avoids replacing the live package tree while the gateway is running, reuses or aligns with the CLI service-stop/update/restart flow where appropriate, preserves `createGlobalInstallEnv`, npm omit-optional fallback, safe argv escaping, supervised/unsupervised restart behavior, startup/result reconciliation, and adds explicit tests for Windows Gateway RPC package updates. If a newer canonical PR fully fixes #40540, this PR can then be closed as superseded by that merged implementation.
