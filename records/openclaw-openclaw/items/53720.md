## Summary

Keep this PR open. Current main still does not implement the intended per-agent enabled/disabled lifecycle control: there is no typed `agents.list[].enabled` field, no `openclaw agents disable`/`enable` commands, no `[disabled]` list status, and no active-agent filtering in routing, cron, or heartbeat paths. The PR remains a relevant implementation candidate for the closed #21680 request, but it needs revision before merge because the supplied review discussion identified remaining functional gaps around all-disabled routing, cron/heartbeat suspension, and clean `--json` output.

## Best Possible Solution

Keep this PR open and ask for a refresh against current main. The best implementation path is a typed `agents.list[].enabled?: boolean` contract in config types, strict schema, generated baseline, docs, and CLI; separate helpers for all configured agents versus active agents; routing, cron, heartbeat, and channel polling that exclude disabled agents; management commands that still operate on disabled agents; clean JSON output; and regression tests covering data preservation, all-disabled fallback, scheduler suspension, and CLI round trips.
