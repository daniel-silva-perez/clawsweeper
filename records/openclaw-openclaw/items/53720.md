## Summary

Keep open. Current main still does not implement per-agent suspension, and the proposed patch is not merge-ready because it leaves background scheduler paths active for disabled agents and has a machine-readable CLI output regression.

## What This Changes

The branch adds an optional `agents.list[].enabled` flag, `openclaw agents disable/enable` commands, disabled status in agent listing/JSON output, schema/docs/generated baseline/changelog updates, and routing/config regression tests.

## Best Possible Solution

Land a refreshed implementation that defines a typed, schema-validated `agents.list[].enabled?: boolean` contract, keeps disabled agents visible to management operations, excludes them consistently from routing and background execution, preserves machine-readable CLI output, updates generated docs, and adds regression coverage for config preservation plus scheduler suspension.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Honor disabled agents in cron and heartbeat:** `src/config/types.agents.ts:62-64`
  - body: The new contract documents `enabled: false` as stopping cron and heartbeat, but the PR does not touch `src/gateway/server-cron.ts` or `src/infra/heartbeat-runner.ts`; both current paths iterate raw `cfg.agents.list`, so background jobs for a disabled agent keep running.
  - confidence: 0.9
- **[P2] Route empty enabled sets to the fallback:** `src/routing/resolve-route.ts:140-142`
  - body: After filtering disabled agents out of the lookup, `pickFirstExistingAgentId` still returns the requested ID when the lookup map is empty. If every configured agent is disabled and a binding targets one of them, routing continues to that disabled ID instead of falling back.
  - confidence: 0.87
- **[P2] Keep disable and enable JSON output parseable:** `src/commands/agents.commands.disable.ts:58-60`
  - body: `agents disable --json` and `agents enable --json` call `logConfigUpdated(runtime)` before `writeRuntimeJson`, which writes a human `Updated ...` line to stdout ahead of JSON. Guard that log the same way add/delete do.
  - confidence: 0.88
