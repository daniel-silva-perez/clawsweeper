## Summary

Keep open. Current source and the latest release tag contain the installer hardening for noninteractive apt and gum ioctl fallback, but the user-facing hosted installer still serves the stale v22-era script that matches the report, and an open linked PR with closing syntax remains unmerged.

## What This Changes

The issue reports that the hosted Linux installer on WSL2 can report Node.js as installed after ioctl-related installer failure, then exit because Node.js is still missing from PATH.

## Best Possible Solution

Redeploy or sync `https://openclaw.ai/install.sh` from the hardened `scripts/install.sh`, then review and either land or replace #73855 with a drift guard that checks the hosted script after merge/schedule without blocking normal installer PRs. Close this issue only after the hosted script no longer reports success unless `node` is actually usable.

## Reproduction Assessment

Yes. The runtime reproduction is WSL2 Ubuntu without preinstalled Node using the hosted installer, and the current hosted script still contains the same v22 false-success and Node-required strings plus stale spinner handling.

## Solution Assessment

No complete solution is on main for the reported user-visible path. Source and release hardening are the right direction, but the hosted installer must be synced and the linked guard reviewed before this is solved.
