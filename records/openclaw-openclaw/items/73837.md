## Summary

Keep open. Current main and the latest release contain source-side hardening for noninteractive apt and gum ioctl fallback, but the live hosted installer still serves the stale Node 22/raw-apt script that matches the report, and open PR #73855 uses closing syntax for this issue.

## What This Changes

The report asks the Linux/WSL installer to detect Node.js auto-install failures accurately and avoid printing Node.js installed before a usable Node binary is available.

## Best Possible Solution

Redeploy the hosted `install.sh` from the hardened source script, verify the WSL/Linux no-preinstalled-Node path cannot print Node success unless `node` is usable, then land or replace PR #73855 with a drift guard so hosted/source divergence fails visibly.

## Reproduction Assessment

Yes. The live hosted installer still exposes the stale raw apt and Node 22 markers that match the reported WSL2 failure path; a final WSL smoke would confirm the runtime path after redeploy.

## Solution Assessment

No, the current state is not complete. Source-side hardening is present, but the maintainable fix needs hosted installer sync plus a guard such as PR #73855 so users stop receiving the stale script.
