## Summary

Keep open. Current main and the latest release contain installer-side hardening for noninteractive apt and gum ioctl fallback, but the live hosted installer still serves the older v22/raw-apt script that matches the report. ## What This Changes

The issue asks for the Linux/WSL Node.js auto-install path to detect install failures accurately instead of printing success and then crashing when `node` is unavailable.

## Best Possible Solution

Redeploy the hosted `install.sh` from the hardened repository script, verify the WSL/Linux no-preinstalled-Node path cannot print Node success unless `node` is usable, then land or replace the drift guard so future hosted/source divergence fails visibly.
