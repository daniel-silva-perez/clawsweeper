## Summary

Keep open. Current source and the v2026.4.27 tag contain installer hardening for noninteractive apt and gum ioctl fallback, but the live hosted installer still serves the older v22/ioctl-era script that matches the report. There is also an open unmerged closing PR, so repository policy says the issue should stay open until that PR is resolved and the hosted installer is synced.

## What This Changes

The issue reports that the hosted Linux/WSL installer can report Node.js installation success after ioctl/apt failures, then fail because Node.js is still absent.

## Best Possible Solution

Keep this issue open until maintainers sync or redeploy `https://openclaw.ai/install.sh` from the hardened source and resolve #73855 or an equivalent hosted-installer drift guard. Final proof should compare live/source hashes and ideally cover WSL2 Ubuntu without preinstalled Node so the installer never reports Node success unless a valid Node binary is active.

## Reproduction Assessment

Yes. The reporter provided concrete WSL2 Ubuntu steps and logs, and a live fetch of `https://openclaw.ai/install.sh` still shows the stale v22/raw-apt/ioctl-era markers that match the failure family; I did not run the full installer because it mutates the host.

## Solution Assessment

Unclear/partial. The current source-side hardening is the right direction, but the linked drift-check PR alone is not sufficient unless the hosted script is actually redeployed or synced from that source.
