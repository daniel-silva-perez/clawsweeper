## Summary

Keep open. Current source and the latest release tag contain installer hardening for noninteractive apt and gum ioctl fallback, but the live hosted installer still serves the older v22-era script that matches the report. There is also an open linked PR with closing syntax, so this issue should remain open until that candidate is reviewed and the hosted installer is actually synced.

## What This Changes

The issue asks the Linux installer to detect Node.js auto-install failures accurately instead of printing Node.js success after apt/gum ioctl failures when `node` is still unavailable.

## Best Possible Solution

Sync or redeploy `https://openclaw.ai/install.sh` from the hardened `scripts/install.sh`, then review and either land or replace #73855 with a maintainable hosted-installer drift guard. Close this issue only after the hosted script matches source and the no-preinstalled-Node WSL path no longer reports success unless `node` is actually usable.

## Reproduction Assessment

Yes. The remaining user-visible problem has a high-confidence reproduction path because the live hosted installer still contains the stale v22/raw apt markers and the same Node.js error string; the full runtime repro is WSL2 Ubuntu without preinstalled Node using the hosted installer.

## Solution Assessment

No, source hardening alone is not the best complete solution while the hosted installer is stale. The narrow maintainable fix is to sync the hosted installer and keep a drift guard so source and hosted behavior cannot silently diverge again.
