## Summary

Keep this PR open. Current main does not implement the proposed Bedrock auth-config surface, and the PR branch is useful source work, but it still needs maintainer review before merge because the proposed manifest, typed helper, secret handling, and pending pi-ai resolver contract are not yet aligned.

## What This Changes

The PR adds Amazon Bedrock auth config types and normalization tests, re-exports a setup type alias, declares Cline-parity Bedrock manifest config fields and uiHints, updates Bedrock docs/changelog, and bumps the private Bedrock plugin package version.

## Best Possible Solution

Amend the PR so the manifest schema, uiHints, docs, tests, and exported BedrockAuthConfig describe one consistent plugin-owned contract; mark or route credential fields through the existing sensitive/SecretRef-aware surfaces; and merge only after the pi-ai resolver contract is either shipped and pinned or the OpenClaw change is scoped to the currently supported BedrockOptions.
