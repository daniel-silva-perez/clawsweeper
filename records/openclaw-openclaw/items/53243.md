## Summary

Keep open. Current main still does not implement the PR's intended default behavior: chat-history display projection uses an 8,000-character default and character-only truncation before the later JSON-byte oversized placeholder pass. The PR is useful source work, but it targets the older server-methods-local helper and still needs maintainer review/rebase onto the shared projection path, with one concrete correctness issue around raw UTF-8 budgeting versus serialized-message bytes.

## What This Changes

This PR raises the old `chat.history` text cap to 64,000, adds raw UTF-8 byte-aware truncation before the oversized-message placeholder, and adds gateway regression tests for long ASCII and emoji-heavy assistant text.

## Best Possible Solution

Implement the behavior in `src/gateway/chat-display-projection.ts`, preserve long assistant text that fits the serialized history budget, truncate byte-heavy or JSON-escape-heavy text to a visible prefix before placeholder replacement, and align the chosen default/config documentation with focused gateway and UI history coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Measure truncation against serialized history bytes:** `src/gateway/server-methods/chat.ts:425-427`
  - body: The fallback trims candidates with `Buffer.byteLength(candidate, "utf8")`, but the later oversized gate measures `JSON.stringify(message)`. Escape-heavy text such as control characters can stay under this raw text budget, expand past the 128 KiB single-message cap when serialized, and still be replaced entirely by the placeholder. Budget the prefix against the JSON-encoded message/string bytes so these messages degrade to visible truncation instead of omission.
  - confidence: 0.91
