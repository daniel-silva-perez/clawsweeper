## Summary

Keep this PR open. Current main still uses an 8,000-character default for chat.history text projection, truncates by character count before the later JSON byte-budget placeholder pass, and Control UI/WebChat does not request a larger maxChars override. The PR is still relevant to the linked bug, but it is stale against the current chat-display-projection split and has an unresolved JSON-encoded byte-budget review concern.

## Best Possible Solution

Keep this PR open for maintainer follow-up, but do not land it as-is. The best path is to rebase or replace it with an equivalent fix in src/gateway/chat-display-projection.ts: decide the intended default/config behavior, budget truncation against the serialized chat.history message path enough to avoid recoverable oversized placeholders, preserve long assistant text that fits the history budget, and keep #53242 paired until the fix lands.
