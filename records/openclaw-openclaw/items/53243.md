## Summary

Keep open: current main still does not implement the requested long-text preservation or serialized-byte-aware visible truncation, and this PR remains paired with open #53242. The branch is useful source work, but it is not merge-ready because current main moved the behavior into the shared projection helper and the proposed byte budget still measures raw UTF-8 text rather than the serialized history message that the hard cap enforces.

## What This Changes

The PR raises the old gateway `chat.history` text cap from 12,000 to 64,000, adds raw UTF-8 byte-aware truncation in gateway chat-history text handling, and adds gateway tests for long ASCII and emoji-heavy assistant replies.

## Best Possible Solution

Implement the behavior in the shared chat-display projection path, preserve long assistant text when the fully serialized history entry fits the hard cap, truncate UTF-8-heavy and JSON-escape-heavy text against the same serialized message budget used by the oversized guard, and cover the gateway plus Control UI default/config path with focused tests and docs/config help as needed.
