## Summary

Keep open. Current main has not implemented the default cap increase or byte-aware visible-prefix behavior; the later configurable-limit work only added config/RPC controls and preserved the existing default payload limits. The branch is still useful, but it needs rework because the truncation logic now belongs in the shared projection path and the proposed raw-byte fallback still misses JSON-serialized oversize cases.

## What This Changes

The PR raises the chat.history text cap to 64,000, adds raw UTF-8 byte-aware truncation in the gateway chat-history path, and adds gateway regression tests for long ASCII and emoji-heavy assistant replies.

## Best Possible Solution

Implement visible-prefix truncation in the shared chat-display projection path, preserve long assistant text when the fully serialized history entry fits the hard cap, truncate UTF-8-heavy and JSON-escape-heavy text against the serialized message budget before placeholder replacement, and align focused gateway/UI tests plus docs or config comments with the chosen default behavior.

## Review Findings

Overall correctness: patch is incorrect
