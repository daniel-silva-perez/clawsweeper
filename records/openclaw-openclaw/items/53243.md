## Summary

Keep open. Current main has related configurable-limit work, but it does not implement the requested higher default or serialized-byte-aware fallback, and this branch still has a blocking correctness issue because it budgets raw UTF-8 text while the later oversized-message guard budgets JSON-serialized bytes.

## What This Changes

The PR raises the `chat.history` per-text cap to 64,000, adds raw UTF-8 byte-aware truncation in the gateway server method, and adds gateway regression tests for long ASCII and emoji-heavy assistant history.

## Best Possible Solution

Put the visible-prefix truncation logic in the shared chat display projection path, choose and document the intended default/config semantics, and budget truncation against the serialized history message bytes before the placeholder replacement step runs.

## Review Findings
