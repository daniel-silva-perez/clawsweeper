## Summary

Keep this PR open. Current main still truncates chat.history display text by character count with an 8,000-character default and still runs that projection before the 128 KiB JSON byte placeholder pass, so the linked user problem is not implemented on main. The PR remains relevant, but it is stale against the current chat-display-projection split and carries an unresolved review concern about budgeting raw UTF-8 bytes rather than JSON-encoded bytes.

## What This Changes

This PR raises the chat.history text cap from 12,000 to 64,000 characters, adds raw UTF-8 byte-aware truncation before the single-message cap, and adds regression tests for long ASCII and emoji-heavy assistant history.

## Best Possible Solution

Keep this PR open for maintainer follow-up, but do not land it as-is. The best path is to rebase or replace it with a focused fix in src/gateway/chat-display-projection.ts that preserves long assistant text that fits the serialized chat.history budget, truncates byte-heavy or JSON-escape-heavy text to a readable prefix before placeholder replacement, aligns the intended default/config behavior, and keeps #53242 paired until an equivalent fix lands.
