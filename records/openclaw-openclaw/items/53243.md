## Summary

Keep this PR open. Current main does not implement the default 64,000-character cap or byte-aware readable-prefix truncation, and the later serialized-byte placeholder path still runs after the character projection. Related merged work made the limit configurable, but it preserved existing default payload semantics rather than superseding this change.

## What This Changes

This PR raises the `chat.history` per-text cap to 64,000 characters, adds UTF-8 byte-aware prefix truncation before oversized placeholder replacement, and adds gateway regression tests for long ASCII and emoji-heavy assistant history.

## Best Possible Solution

Current main should handle chat-history display projection in the shared projection helper, preserve long assistant text that fits the serialized history budget, degrade byte-heavy or JSON-escape-heavy text to a readable truncated prefix before placeholder replacement, and cover the default Control UI/WebChat path plus related history endpoints and docs.
