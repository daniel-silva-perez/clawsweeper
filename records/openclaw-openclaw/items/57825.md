## Summary

Keep this PR open. The intended bug fix is still relevant on current main, but the branch targets the older `attempt-execution.ts` layout while the active shared persistence helper still deletes omitted `clearedFields` without an opt-in gate.

## What This Changes

The PR adds opt-in missing-field clearing for session persistence and tests preserving model/provider overrides unless a reset or cleanup path explicitly clears them.

## Best Possible Solution

Adapt the PR onto current main by adding an explicit missing-field clear option to the shared persistence helper, threading it only through reset/cleanup paths, and adding focused regression tests for both preserve-by-default and explicit-clear behavior.

## Reproduction Assessment

Yes. A source-level reproduction is direct: start with a stored session entry containing `providerOverride` and `modelOverride`, then persist a patch that omits those fields while passing them in `clearedFields`; current main deletes them in the shared helper.

## Solution Assessment

No for the branch exactly as written, because it edits the old helper location. The intended opt-in deletion semantics are the narrow maintainable fix once moved to the current shared helper and audited at the wrapper call sites.
