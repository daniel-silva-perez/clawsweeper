## Summary

Keep this PR open. Current main still does not implement the PR’s intended opt-in deletion semantics: the shared persistence helper deletes omitted fields whenever `clearedFields` is supplied, and the agent wrapper still supplies the override-field list on every wrapper persistence call. The PR likely needs rebase/adaptation to the current shared helper layout, not closure as already implemented. Security pass: the provided PR diff is limited to agent session persistence code and tests, with no workflow, dependency, lockfile, package publishing, secret handling, or external code execution changes.

## Best Possible Solution

Review and adapt this PR to current main by adding an explicit `clearMissingFields` or equivalent option to `src/agents/command/attempt-execution.shared.ts`, threading an explicit-clear flag only through reset/cleanup paths in `src/agents/agent-command.ts` and the Claude CLI binding cleanup path, and adding focused regression tests for preserve-by-default plus explicit clearing in the current test layout.
