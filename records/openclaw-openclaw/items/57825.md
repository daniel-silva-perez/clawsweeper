## Summary

Keep this PR open. Current main does not yet implement the PR’s opt-in clear semantics: the shared session persistence helper still deletes omitted fields whenever they are listed in `clearedFields`, and the agent wrapper still passes the override field list on every wrapper call. The PR likely needs a small rebase/adaptation to the current `attempt-execution.shared.ts` layout rather than closure as implemented.

## Best Possible Solution

Review and adapt this PR to current main by adding an explicit `clearMissingFields` or equivalent option to the shared persistence helper, threading an explicit-clear flag only through reset/cleanup paths, and adding regression tests in the current `src/agents/command` test layout for preserve-by-default plus explicit clearing.
