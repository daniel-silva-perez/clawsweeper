## Summary

Keep this PR open. Current main still only has startup light/dark TUI palette selection through `OPENCLAW_THEME` and `COLORFGBG`; it does not implement this PR's central runtime behavior: no `/theme` command, no `/theme list`, no runtime theme application API, no searchable theme picker with live preview, and no cancel/revert or commit flow. The branch discussion is still useful because the author addressed earlier bot findings, but the remaining repaint concern and unrelated `.secrets.baseline` churn need maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open and coordinate it with #44382. The best path is one focused TUI theming implementation that adds a runtime theme API, registers `/theme`, supports `/theme list`, opens a searchable picker with live preview, commits on select, reverts on cancel, repaints all TUI chrome consistently, documents the public TUI behavior at `https://docs.openclaw.ai/web/tui`, and includes targeted command/theme tests. If maintainers choose #44382 as canonical, mark this PR superseded only after that decision or merge path is explicit.
