## Summary

Keep this issue open. Current main still exposes only global session maintenance retention plus a cron-run-specific reaper; it does not implement ordered per-pattern or per-type `session.maintenance.rules`. The KeepMyClaw comment is useful backup context and open PR #50755 is adjacent per-agent work, but neither provides the requested core session-store retention policy.

## What This Changes

Request support for ordered per-pattern or per-type session maintenance retention rules so ephemeral cron/subagent sessions can be pruned more aggressively than interactive sessions.

## Best Possible Solution

Add an additive core `session.maintenance.rules` contract after maintainer approval, with ordered first-match retention by a well-defined key/type matcher, global `pruneAfter` fallback, strict schema/help/generated-docs coverage, and pruning call sites that preserve active/running-session protections and existing cron-specific retention behavior.
