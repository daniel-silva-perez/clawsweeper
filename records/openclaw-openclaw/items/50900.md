## Summary

Keep this issue open. Current main still has only global session maintenance age/count settings plus a separate cron-run reaper, so ordered per-pattern or per-type retention rules remain unimplemented; the KeepMyClaw comment is useful backup context, but it does not replace a core retention policy.

## What This Changes

Request support for ordered per-pattern or per-type session maintenance retention rules so ephemeral cron and subagent sessions can be pruned more aggressively than interactive sessions.

## Best Possible Solution

Add an additive core session.maintenance.rules contract after maintainer approval, with ordered first-match retention by a well-defined matcher, global pruneAfter fallback, strict schema/help/docs coverage, and pruning call sites that preserve active/running-session protections plus existing cron-specific behavior.
