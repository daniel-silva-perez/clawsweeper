## Summary

Keep this issue open. Current main still implements session maintenance as a global retention policy plus a separate cron-run-only retention sweep; it has no ordered `session.maintenance.rules`/`match` contract for per-pattern pruning. The KeepMyClaw comment is useful backup context, and open PR #50755 is adjacent per-agent work, but neither replaces the requested core session-store retention rule behavior.

## What This Changes

Add ordered per-pattern or per-type session maintenance retention rules so matching session keys can use shorter `pruneAfter` values before falling back to the global `session.maintenance.pruneAfter`.

## Best Possible Solution

Keep the issue open for maintainer design approval. If accepted, implement an additive `session.maintenance.rules` contract with ordered first-match retention by session key or well-defined type, global `pruneAfter` fallback, strict schema/type/help/generated docs coverage, and pruning call sites that preserve active/running session safeguards and existing cron-specific retention behavior.
