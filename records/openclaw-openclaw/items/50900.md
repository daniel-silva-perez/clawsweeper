## Summary

Keep this issue open. Current main still has global session maintenance age/count/disk controls plus a separate cron-run retention reaper, but it does not implement ordered per-pattern or per-type retention rules; the proposed `session.maintenance.rules` shape is still outside the strict config contract. The KeepMyClaw comment is useful backup-retention context, but it does not replace a core pruning policy for `sessions.json`.

## What This Changes

The issue requests ordered per-pattern or per-type session maintenance retention rules so ephemeral cron/subagent sessions can be pruned faster than interactive sessions.

## Best Possible Solution

Add an additive core `session.maintenance.rules` contract after maintainer design approval, with a documented matcher, ordered first-match retention, global `pruneAfter` fallback, strict schema/generated-help/docs coverage, and pruning call sites that preserve active/running-session protections plus existing cron-specific behavior.
