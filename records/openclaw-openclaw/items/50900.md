## Summary

Keep this issue open. Current main still exposes only global session-store retention plus a separate cron-run reaper; it does not implement ordered per-pattern or per-type `session.maintenance.rules`. The KeepMyClaw comment is useful backup context and #45210/#50755 are adjacent, but neither replaces the requested core retention policy.

## What This Changes

Request additive session maintenance retention rules so selected session-key patterns or types can use shorter `pruneAfter` values while unmatched sessions fall back to the global setting.

## Best Possible Solution

Add an additive core `session.maintenance.rules` contract after maintainer approval, with ordered first-match retention by a documented matcher, global `pruneAfter` fallback, strict schema/generated-docs coverage, and pruning call sites that preserve active/running-session protections plus existing cron-specific behavior.
