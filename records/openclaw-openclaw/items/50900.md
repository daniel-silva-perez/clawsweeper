## Summary

Keep this issue open. Current main still has only a global `session.maintenance.pruneAfter` path: the config schema is strict and has no `rules`/`match` field, the resolver produces one `pruneAfterMs`, and save/load cleanup applies that single age cutoff across the whole store. Cron run sessions have a separate `cron.sessionRetention` reaper, and subagents have archive controls, but neither implements the requested first-match per-pattern retention rules for general session maintenance. The KeepMyClaw comment is useful backup/workaround context, but it does not close the core session-store retention gap.

## Best Possible Solution

Keep the issue open for a core session-maintenance design: add a typed, documented `session.maintenance.rules` or equivalent policy with ordered matching over session keys/types, reuse the existing locked session-store maintenance path, preserve active/running sessions, and add schema, resolver, cleanup, CLI dry-run, and docs coverage. Coordinate with the narrower subagent/ACP retention and per-agent maintenance override work rather than treating the KeepMyClaw backup workaround as a core fix.
