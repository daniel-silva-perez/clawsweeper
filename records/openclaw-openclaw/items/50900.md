## Summary

Keep this issue open. Current main still models session maintenance as global `pruneAfter`/`maxEntries` settings plus a separate cron-run-only reaper; it does not expose or resolve ordered per-pattern retention rules. The KeepMyClaw comment is useful backup context, and open PR #50755 is adjacent per-agent work, but neither satisfies the requested core session-store rule contract.

## What This Changes

Request to add ordered per-pattern or per-type `session.maintenance.rules` so ephemeral cron/subagent session entries can prune sooner than long-lived interactive sessions while falling back to the global `pruneAfter`.

## Best Possible Solution

After maintainer approval, add an additive core `session.maintenance.rules` contract with ordered first-match retention by a well-defined key/type matcher, global `pruneAfter` fallback, strict schema/help/generated docs coverage, and pruning call sites that preserve active/running-session protections and existing cron-specific retention behavior.
