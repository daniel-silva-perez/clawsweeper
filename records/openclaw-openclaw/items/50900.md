## Summary

Keep this issue open. Current main still has a single global session maintenance age/count policy plus a separate cron-run reaper; there is no ordered per-pattern or per-type retention-rule surface, and the strict config schema would reject the proposed `session.maintenance.rules` shape. The KeepMyClaw comment is useful backup/retention context, but it does not implement the requested core pruning policy.

## What This Changes

Add ordered per-pattern or per-type session maintenance retention rules with per-rule `pruneAfter` values and a global `pruneAfter` fallback.

## Best Possible Solution

Add an additive core `session.maintenance.rules` contract after maintainer approval, with a documented matcher, ordered first-match retention, global `pruneAfter` fallback, strict schema/generated-help/docs coverage, and pruning call sites that preserve active/running-session protections plus existing cron-specific behavior.
