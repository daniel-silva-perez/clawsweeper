## Summary

Keep this PR open. Current main still does not implement the intended behavior: abort replies can still be normalized before model/thinking metadata is populated, so templated response-prefix placeholders can remain literal. The linked same-author bug #45314 is still open, and #45454 is an open draft implementation candidate rather than shipped proof.

## Best Possible Solution

Keep this PR open until maintainers choose the canonical fix for #45314. The landed change should populate response-prefix context before stop/abort reply normalization, add focused regression coverage for templated abort prefixes, and either handle or explicitly defer session overrides, channel overrides, routed replies, and catalog-backed thinking defaults. If #45454 becomes the preferred implementation, close this PR only after that work lands or a maintainer explicitly marks it superseded.
