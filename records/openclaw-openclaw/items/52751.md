## Summary

Keep open. Current main still does not implement per-agent `compaction` or `contextPruning` overrides: the per-agent type and strict runtime schema omit both keys, and the Pi compaction/pruning consumers still read `agents.defaults`. The PR is useful but needs maintainer consolidation with the overlapping open #69988 rather than cleanup closure.

## Best Possible Solution

Land one canonical implementation that accepts both per-agent keys, merges each agent override over `agents.defaults`, threads the effective settings into Pi compaction and context-pruning consumers, updates docs/generated config surfaces, and closes #52732 only after that implementation merges.
