## Summary

Keep open. Current main still does not implement per-agent `compaction` or `contextPruning`: those settings exist only under `agents.defaults`, `agents.list[]` rejects them via the strict agent entry schema, and the embedded Pi compaction/pruning runtime reads global defaults. The PR discussion is useful prior art, including the circular-import fix, but later review comments still identify a real runtime no-op gap.

## Best Possible Solution

Keep this PR open for maintainer review or contributor revision. A complete implementation should add typed and strict-schema support for both per-agent keys, avoid schema import cycles, merge per-agent values with defaults in the effective runtime config, pass that effective config through normal compaction/pruning paths, and add focused regression tests plus config docs. If a newer PR becomes the canonical implementation for both compaction and contextPruning, this PR can be closed as superseded then.
