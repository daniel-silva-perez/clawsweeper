## Summary

Keep open. Current main still does not implement the intended per-agent `compaction` and `contextPruning` overrides, and the PR branch remains an incomplete/stale implementation because it validates or resolves the new keys without threading effective per-agent settings into the standard Pi compaction and pruning consumers. The related open PR #69988 overlaps on compaction but does not cover `contextPruning`, so this needs maintainer consolidation rather than cleanup closure.

## What This Changes

The PR adds per-agent `compaction` and `contextPruning` type/schema fields, extracts shared Zod schemas, and returns those fields from agent config resolution.

## Best Possible Solution

Land one canonical implementation that accepts both per-agent keys, merges each agent override over `agents.defaults`, passes the effective settings through Pi settings and embedded extension factories, updates config docs/generated surfaces, and keeps #52732 open until that implementation merges.
