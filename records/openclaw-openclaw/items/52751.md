## Summary

Keep open. Current main still does not implement this PR’s intended per-agent `compaction` and `contextPruning` overrides: the per-agent type and strict runtime schema omit both keys, the resolver does not return them, and the Pi compaction/pruning runtime still reads `agents.defaults`. The overlapping #69988 remains open and appears to cover only the compaction half, so it is not enough to close this PR as implemented or fully superseded.

## Best Possible Solution

Keep this PR open for maintainer review or consolidation. The best implementation should add one canonical per-agent config path for both `compaction` and `contextPruning`, avoid schema cycles, merge `agents.list[]` overrides over `agents.defaults`, thread the effective settings through Pi compaction/pruning consumers, and update tests plus `docs/gateway/config-agents.md`. Maintainers should decide whether #69988 becomes the canonical compaction half and whether this PR should be rebased to cover the remaining `contextPruning` scope.
