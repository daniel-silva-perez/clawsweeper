## Summary

Current main still matches the reported LINE multi-account failure mode: `accounts.default` is listed but not resolved as the default LINE account, named LINE accounts without explicit `enabled: true` resolve disabled, and gateway startup returns before LINE reaches webhook route registration. The related fix PR #47528 is closed and unmerged, so it does not supersede this issue.

## Best Possible Solution

Keep this issue open and implement the #47528 direction on current main: resolve default LINE credentials from `channels.line.accounts.default`, treat configured named LINE accounts as enabled unless explicitly disabled while still respecting top-level `channels.line.enabled: false`, and add focused regression tests for `accounts.default` credentials plus a named account with a custom `webhookPath` reaching route registration.
