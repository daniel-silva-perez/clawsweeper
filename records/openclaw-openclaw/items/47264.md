## Summary

Current main still leaves the reported LINE multi-account config broken. The default account resolver still skips `channels.line.accounts.default`, named LINE accounts still default to disabled unless they explicitly set `enabled: true`, and gateway startup returns before `monitorLineProvider` can register the webhook route. The related fix PR #47528 was closed unmerged, so it does not supersede or implement this issue.

## Best Possible Solution

Keep this issue open and implement the #47528 direction on current main: make LINE resolution read `channels.line.accounts.default` for the default account, treat configured named LINE accounts as enabled unless explicitly disabled while still respecting top-level `channels.line.enabled: false`, and add focused regression coverage for `accounts.default` plus a named account with a custom `webhookPath` reaching route registration.
