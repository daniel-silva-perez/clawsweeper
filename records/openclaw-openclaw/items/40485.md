## Summary

Keep this PR open. Current main still lacks the narrow Telegram direct-chat override for native `/new` and `/reset`, so the deleted-message reply bug remains plausible when `replyToMode` is configured. The PR is useful, but it needs to be ported from removed legacy `src/telegram` paths into the active `extensions/telegram` plugin and should cover both commands in regression tests.

## Best Possible Solution

Port the PR’s narrow behavior to `extensions/telegram/src/bot-native-commands.ts`: set delivery `replyToMode` to `"off"` only for private-chat native `/new` and `/reset` replies, while preserving configured threading for groups, topics, plugin commands, and other native commands. Add regression coverage for both `/new` and `/reset` with configured `replyToMode: "first"`.
