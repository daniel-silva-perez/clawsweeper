## Summary

Keep this PR open, but do not merge it as written. The intended Telegram UX fix is still relevant on current main, but the branch modifies removed legacy `src/telegram` files while current runtime code lives under `extensions/telegram/src`, where the native-command path still uses the configured reply mode and still injects the command message as the reply target.

## What This Changes

The PR adds a legacy Telegram native-command reply-mode override so private `/new` and `/reset` replies use `replyToMode: "off"`, plus a regression test for private `/new`.

## Best Possible Solution

Port the narrow override into the active Telegram plugin: direct-chat native `/new` and `/reset` should deliver the first assistant reply without implicit reply threading, while groups, forum topics, plugin commands, explicit reply tags, and other native commands keep their current threading behavior.
