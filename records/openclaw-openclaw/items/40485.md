## Summary

Keep this PR open. The reported Telegram direct-chat bug still appears unfixed on current main: active native command delivery still uses the configured reply mode and injects the slash-command message as the reply target, while this PR’s patch targets removed legacy `src/telegram` paths rather than the active Telegram plugin.

## What This Changes

The PR adds a Telegram native-command reply-mode override under legacy `src/telegram` paths so private `/new` and `/reset` replies use `replyToMode: "off"`, plus a regression test for private `/new`.

## Best Possible Solution

Port the narrow override into `extensions/telegram/src/bot-native-commands.ts`: for native direct-chat `/new` and `/reset` only, pass `replyToMode: "off"` into delivery while preserving configured threading for groups, forum topics, plugin commands, and all other native commands. Add regression coverage for both commands with configured `replyToMode: "first"`.
