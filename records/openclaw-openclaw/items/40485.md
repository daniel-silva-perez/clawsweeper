## Summary

Keep open, but do not merge as-is. The PR’s intent is plausible, and prior discussion links it to the Telegram deleted-message UX, but the diff changes legacy `src/telegram` files that are absent from current main while the active Telegram plugin still lacks this per-command direct-chat reply-mode override.

## What This Changes

The PR adds a `replyToModeOverride` path in legacy Telegram native-command code so direct-chat native `/new` and `/reset` replies use `replyToMode: "off"`, plus a private `/new` regression test.

## Best Possible Solution

Port the narrow behavior into `extensions/telegram/src/bot-native-commands.ts`: direct-chat native `/new` and `/reset` should deliver the first assistant reply without implicit reply threading, while groups, forum topics, plugin commands, explicit reply tags, and other native commands keep current threading semantics. Add active-plugin regression coverage for both `/new` and `/reset`.
