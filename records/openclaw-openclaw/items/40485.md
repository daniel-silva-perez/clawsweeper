## Summary

Keep this PR open. Current main still routes Telegram native command replies through the account-level `replyToMode` and still injects the triggering slash-command message as the reply target when the payload has no target. The PR addresses a plausible still-present Telegram direct-chat bug, but its diff targets removed legacy `src/telegram` paths and needs to be ported to the active `extensions/telegram` plugin with `/reset` coverage added.

## Best Possible Solution

Keep this PR open and port the fix to `extensions/telegram/src/bot-native-commands.ts`. Add a narrow delivery override that sets `replyToMode` to `"off"` only for private-chat native `/new` and `/reset` replies, preserve configured threading for groups, topics, plugin commands, and other native commands, and add regression coverage for both `/new` and `/reset` when configured `replyToMode` is `"first"`.
