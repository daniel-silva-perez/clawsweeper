## Summary

Keep this PR open, but it needs changes before merge. The requested Telegram behavior is still valid; however, the branch edits removed legacy `src/telegram` files while current `main` ships Telegram native commands from `extensions/telegram`, where direct-chat `/new` and `/reset` still use the configured reply mode and inject the command message as the reply target.

## What This Changes

The branch changes legacy Telegram native-command delivery so direct-chat native `/new` and `/reset` use `replyToMode: "off"`, and adds a legacy `/new` regression test.

## Best Possible Solution

Port the narrow override into `extensions/telegram/src/bot-native-commands.ts` so direct-chat native core `/new` and `/reset` suppress implicit reply threading, while group/forum topics, plugin commands, explicit reply targets, progress placeholders, menus, and other native commands keep current behavior.

## Reproduction Assessment

Yes. A code-level reproduction exists on current `main`: configure Telegram `replyToMode` to a threaded mode and invoke direct-chat native `/new` or `/reset`; the active handler adds `replyToId: String(msg.message_id)`, and delivery only strips it when `replyToMode` is `off`.

## Solution Assessment

No. The behavior change is the right narrow direction, but this PR is not the best implementation because it patches removed legacy files instead of the active `extensions/telegram` plugin path; the safer fix is to port the override and add active-path tests for both reset commands.
