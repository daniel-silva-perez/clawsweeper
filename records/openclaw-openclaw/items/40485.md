## Summary

Keep open. The PR is useful, but it is not correct against current main: it changes legacy `src/telegram/*` files while the active Telegram plugin still lives under `extensions/telegram/src` and still threads native command replies through the configured reply mode plus the command message id.

## What This Changes

The PR adds a legacy Telegram native-command `replyToMode` override so direct-chat native `/new` and `/reset` replies are sent without reply threading, plus a private `/new` regression test.

## Best Possible Solution

Port the narrow behavior into the active Telegram plugin: direct-chat native `/new` and `/reset` should deliver the first assistant reply without implicit reply threading, while groups, forum topics, plugin commands, explicit reply tags, and other native commands keep their existing threading semantics. Add active-plugin regression coverage for both reset commands before replacing or superseding this stale branch.
