## Summary

Keep open. Current main has the main Telegram/TTS primitives, but the issue is specifically about inconsistent normal-turn voice delivery and the remaining official contract. A maintainer already reviewed it on April 25 and explicitly kept it open until there is a current-main repro or a narrower policy/docs bug. Source and docs still show normal voice delivery is policy-dependent, so there is not enough evidence for an implemented-on-main close.

## Best Possible Solution

Keep this issue open and ask for a current-main repro that includes messages.tts config, /tts status output, chat override state, provider/model, Telegram chat type, whether the triggering inbound message was audio/text, and delivery logs. Then either fix the specific normal-reply branch or document the intended contract for auto, inbound, tagged, and final/all modes.
