## Summary

Keep this PR open. Current main still accepts Telegram `agentId` only for topic config and binding-based routing; it does not accept or consume `channels.telegram.accounts.<id>.agentId`, while the PR remains a focused fix for the concrete regression reported in #62985.

## What This Changes

The PR adds account-level Telegram `agentId` support by extending config schema/types, applying account-default routing when no explicit binding matches, keeping top-level `channels.telegram.agentId` invalid, and adding regression tests.

## Best Possible Solution

Review, rebase, and land this PR or an equivalent focused patch for #62985: accept `channels.telegram.accounts.<id>.agentId` in the account schema/type, keep top-level `channels.telegram.agentId` rejected, consume the account default only when no explicit binding matches, preserve explicit binding and topic-specific precedence, and add schema plus Telegram route/message-context regression coverage.
