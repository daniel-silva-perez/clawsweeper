## Summary

Keep open. Current main still drops the trusted requester agent override on the ACP `sessions_spawn` path, while this PR is a focused parity fix with no blocking review findings found in the latest diff.

## What This Changes

The PR adds `requesterAgentIdOverride` to ACP spawn context, forwards it from `sessions_spawn`, uses override-first requester resolution for ACP heartbeat and relay eligibility, and adds regression tests for tool passthrough and implicit parent-stream gating.

## Best Possible Solution

Land this PR or an equivalent rebased patch after normal maintainer review and CI completion, keeping the requester override as trusted internal tool context and not exposing it as a model-callable `sessions_spawn` argument.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `sessions_spawn` with trusted `requesterAgentIdOverride` and `runtime: "acp"`; current main forwards that override for native subagents but omits it from the ACP context, and ACP heartbeat/relay logic derives the requester from the parent session key.

## Solution Assessment

Yes. The PR is the narrow maintainable fix for the observed gap: it mirrors the existing subagent override-first pattern, updates the two ACP requester-resolution paths that gate heartbeat/relay behavior, and covers both tool passthrough and relay gating.
