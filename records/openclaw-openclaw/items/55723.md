## Summary

Keep open. Current main still drops the trusted requester agent override on the ACP `sessions_spawn` path, while this PR applies a narrow fix that matches the existing native subagent behavior and is covered by focused regression tests plus green PR check runs.

## What This Changes

This PR adds `requesterAgentIdOverride` to ACP spawn context, forwards it from `sessions_spawn`, resolves ACP heartbeat/relay requester state from the override before falling back to the parent session key, and adds regression tests for passthrough and implicit parent-stream gating.

## Best Possible Solution

Land this PR or an equivalent rebased patch after maintainer review, preserving `requesterAgentIdOverride` as trusted internal tool context and keeping it out of the model-callable `sessions_spawn` schema.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to construct `sessions_spawn` with trusted `requesterAgentIdOverride` and call it with `runtime: "acp"`; current main accepts the override at the tool boundary but omits it from ACP context and requester heartbeat/relay resolution.

## Solution Assessment

Yes. The PR is the narrowest maintainable fix I found: it mirrors the existing native subagent override path, updates both ACP requester checks that gate heartbeat/relay behavior, and adds targeted regression tests.
