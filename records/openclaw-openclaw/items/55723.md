## Summary

Keep this PR open. Current main still does not preserve `requesterAgentIdOverride` through the ACP `sessions_spawn` path: the tool accepts the trusted override context and forwards it for native subagents, but the ACP context type, ACP tool call, heartbeat gating, and requester-state resolution still ignore it. The PR remains a focused implementation candidate for the exact gap described by the PR body, the Greptile review, and the earlier ClawSweeper review.

## Best Possible Solution

Review and land this PR, or apply an equivalent rebased patch: add `requesterAgentIdOverride` to `SpawnAcpContext`, forward it from the ACP `sessions_spawn` branch, use override-first requester resolution in ACP heartbeat and requester-state logic, and keep regression coverage for ACP tool passthrough plus implicit parent-stream gating.
