## Summary

Keep this PR open. Current main still does not implement the binding-implies-Telegram-group-authorization behavior, the PR's main handler patch targets the removed legacy Telegram path, and the hot-reload 409 change needs reconciliation with current polling 409 recovery before merge.

## What This Changes

The PR branch maps Telegram `supergroup` to `group`, lets explicit Telegram peer bindings bypass group allowlist drops, and adds a Telegram-only 1.5s pause between channel stop/start during gateway reload.

## Best Possible Solution

Keep this PR open for maintainer review and request a current-main rebase. The best path is to split or rework the change in current paths: add any still-needed `supergroup` normalization with focused route/config tests, make an explicit maintainer security decision on whether Telegram peer bindings should authorize groups outside `channels.telegram.groups`, and add a Telegram-scoped reload pause only if a current-main regression test shows the existing 409 recovery is insufficient.
