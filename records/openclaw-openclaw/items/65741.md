Keep this PR open. Current main at 7ddd815e469e5f5a9407d67cd8dc306eff771b84 still does not contain the PR's three observable changes: `normalizeChatType()` does not accept `supergroup`, Telegram group allowlist rejection still happens before any explicit binding bypass, and channel hot reload still restarts immediately after `stopChannel()`. The PR also targets the removed `src/telegram/bot-handlers.ts` path, so it needs a current-main rebase into `extensions/telegram` plus focused tests and maintainer review for the binding-implies-authorization behavior.

## Best Possible Solution

Keep this PR open and ask for a current-main rebase. The best path is to split or clearly test the three behaviors: add `supergroup` normalization only where route/binding inputs can legitimately carry Telegram chat types; make Telegram group allowlist gating binding-aware in the current `extensions/telegram` handler or helper without weakening disabled/sender authorization; and add a Telegram-scoped reload pause only if it is still needed on top of existing polling 409 recovery.
