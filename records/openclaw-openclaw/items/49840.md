## Summary

Keep this PR open. Current main still does not implement the proposed `maxCostPerRun` config surface or a `cost-cap-exceeded` pre-dispatch skip path for heartbeat, main-session cron wakeups, or isolated cron agent turns. The linked tracking issue #49823 is closed as superseded by this PR, so this remains the canonical implementation thread. The submitted diff also still has unresolved functional review concerns around estimating the actual run cost and resolving the effective model, although its file list does not show a supply-chain or CI/security-sensitive expansion.

## Best Possible Solution

Keep this PR open for maintainer review or replacement. The best implementation should add the config type plus strict zod schema, generated schema/help, and public docs; enforce the cap at a shared embedded-run dispatch point covering heartbeat, main-session cron wakeups, isolated cron, and future triggers; price the effective resolved model using the existing pricing catalog; and estimate the actual prompt/transcript/tool/system context or clearly document a conservative limitation with focused tests.
