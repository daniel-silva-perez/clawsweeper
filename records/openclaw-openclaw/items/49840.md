## Summary

Keep this PR open. Current main still has no `maxCostPerRun` config, no `cost-cap-exceeded` skip path, and no equivalent per-run cost cap for heartbeat or isolated cron dispatch. The same-author tracking issue #49823 is already closed as superseded by this PR, so closing this PR would drop the canonical thread for the feature. The proposed diff also still needs maintainer-quality follow-through before merge: schema/help/docs coverage, shared embedded-run placement, effective model resolution, and transcript/system/tool-aware estimation or clear documentation.

## Best Possible Solution

Keep this PR open for maintainer review or replacement. The best implementation should add the config type plus strict schema, generated schema/help, docs, and focused tests, then enforce the cap at a shared embedded-run dispatch point using the effective resolved model and the same session context the actual run will use. It should either include transcript/system/tool overhead in the estimate or document the limitation plainly, and it should resolve the existing review comments before merge.
