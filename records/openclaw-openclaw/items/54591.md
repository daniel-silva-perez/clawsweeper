Keep this PR open. Current main still enforces the sandboxed-requester to unsandboxed-child block, the public docs and runtime tests still describe/assert that guard, and the requested config key is absent from the strict sandbox schema and resolved sandbox runtime status. The PR adds a dangerous sandbox config flag and runtime/schema/help plumbing so a sandboxed requester can opt into spawning unsandboxed subagents, plus Zod validation coverage for the new config key.
