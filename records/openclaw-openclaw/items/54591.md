## Summary

Keep this PR open. Current main still has the sandboxed-requester to unsandboxed-child hard block in `sessions_spawn`, and the proposed `dangerouslyAllowUnsandboxedSubagentSpawn` config surface is absent from the sandbox config type, resolver, runtime status, Zod schema, docs, and help metadata. The linked feature issue is closed, but this PR is still the visible implementation candidate rather than something already shipped on main.

## Best Possible Solution

Keep the PR open for maintainer review. If accepted, land a security-conscious version that adds the config/type/schema/help surface, threads the flag through resolved sandbox runtime status, preserves `subagents.allowAgents` and `sandbox="require"` as hard gates, documents the dangerous operator tradeoff, and adds focused runtime tests for both the blocked and explicitly allowed paths.
