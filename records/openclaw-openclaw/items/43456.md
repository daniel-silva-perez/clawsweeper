## Summary

Keep this issue open. Current main still does not wake Mattermost `oncall` agents for unmentioned replies to the bot's own post; `root_id` is used for thread/session routing, but the wake gate still only sees explicit mention, authorized command bypass, and `onchar` trigger state.

## Best Possible Solution

Implement this in the bundled Mattermost plugin: resolve `root_id` and any supported inline/parent reply metadata, verify the referenced post was authored by the Mattermost bot user, feed `reply_to_bot` into the shared mention gate or an equivalent Mattermost gate extension, add regression tests for unmentioned bot-thread replies in default `oncall` mode, and update the Mattermost/group-channel docs when shipped.

## Risks / Open Questions

- A Mattermost fix should not trust `root_id` or `parent_id` alone; it should resolve the referenced post safely and verify the referenced `user_id` is the bot user before bypassing the `oncall` mention requirement.
- The implementation should preserve existing self-message filtering, access checks, command authorization bypass, `onchar` behavior, pending-history recording, and thread/session routing.
