## Summary

Keep this issue open. Current main still does not wake default Mattermost `oncall` agents for unmentioned replies to the bot's own message. Mattermost preserves `root_id` for thread/session routing, but the wake gate only receives explicit mention state, authorized command bypass, and `onchar` trigger state; the shared `reply_to_bot` implicit mention contract exists but is not wired into the bundled Mattermost plugin.

## Best Possible Solution

Implement this in the bundled Mattermost plugin: resolve supported root/parent reply metadata, verify the referenced post was authored by the Mattermost bot user, pass `reply_to_bot` into the shared mention-gating contract or an equivalent Mattermost gate extension, then add regression tests for unmentioned bot-thread replies in default `oncall` mode and update the Mattermost/group-channel docs when shipped.
