## Summary

Current main still does not implement Mattermost reply-to-bot wake behavior. The Mattermost inbound path uses `root_id` for thread/session routing only, computes mention state from explicit mentions/onchar/commands, and has no bot-authored referenced-post check or implicit `reply_to_bot` mention fact. Docs and tests still describe and cover Mattermost oncall as mention-only.

## Best Possible Solution

Keep this issue open. Implement the behavior in the bundled Mattermost plugin by resolving safe reply metadata for `root_id` and any available parent reply field, fetching or otherwise proving the referenced post author is the Mattermost bot user, feeding a `reply_to_bot` implicit mention fact into the shared mention gate, and adding regression tests for unmentioned threaded and inline replies in default oncall mode plus non-bot-root negative cases. Update Mattermost/channel docs when the behavior ships.
