## Summary

Keep this issue open. Current main still treats Mattermost thread metadata as routing/session context, but the oncall mention gate does not resolve the referenced post, verify it was authored by the bot, or pass a `reply_to_bot` implicit mention into the wake decision.

## Best Possible Solution

Implement this in the bundled Mattermost plugin: add safe referenced-post resolution for `root_id` and any supported inline/parent reply metadata, verify the referenced post was authored by the Mattermost bot user, feed `reply_to_bot` into the shared mention gate or an equivalent Mattermost gate extension, add regression tests for unmentioned threaded and inline replies in default `oncall` mode, and update the Mattermost and group-channel docs when it ships.
