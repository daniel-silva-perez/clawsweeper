## Summary

Keep this issue open. Current main still leaves the reported Apple Silicon local-embedding stability and guidance problem unresolved: built-in local embeddings still initialize node-llama-cpp in-process, still run each local batch with Promise.all, expose no CPU/GPU/Metal backend selector, and the public memory docs do not document a supported Metal-safe or CPU-only path. The April 2026 comments add fresh reproductions, and the related GPU-backend PR only covers one slice of the requested work.

## What This Changes

The issue requests safer official support for local memory embeddings on Apple Silicon, including Metal/GPU guidance, CPU-only configuration, batch serialization, and gateway crash containment for node-llama-cpp/ggml-metal failures.

## Best Possible Solution

Keep this as the canonical Apple Silicon local-embedding stability tracker. The desired end state is a documented macOS support matrix for builtin local and QMD embeddings, a validated CPU/GPU/backend config path based on the node-llama-cpp contract, serialized or otherwise bounded local GGUF embedding calls, and a gateway-safe failure boundary or fallback behavior for native embedding crashes.
