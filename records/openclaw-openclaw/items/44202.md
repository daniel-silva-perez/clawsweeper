## Summary

Keep this issue open. Current main still has the reported core problem: builtin local embeddings initialize node-llama-cpp in the gateway process, do not expose a CPU/GPU/Metal backend selector, and still have no official Apple Silicon Metal-safe guidance. The recent comments include fresh macOS reproductions, and the open GPU-backend PR only covers one part of the requested fix surface.

## What This Changes

The issue requests safer official support for local memory embeddings on Apple Silicon, including Metal/GPU guidance, CPU-only configuration, serialized local embedding batches, and gateway crash containment for node-llama-cpp/ggml-metal failures.

## Best Possible Solution

Keep this as the canonical Apple Silicon local-embedding stability tracker. The desired end state is a documented macOS support matrix for builtin local and QMD embeddings, a validated node-llama-cpp CPU/GPU/backend config path, bounded or serialized local GGUF embedding calls, and a gateway-safe failure boundary or fallback for native embedding crashes.
