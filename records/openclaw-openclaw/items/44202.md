## Summary

Keep this open. Current main still leaves the reported Apple Silicon local-embedding stability gap unresolved: the built-in local provider loads node-llama-cpp without a CPU/GPU/Metal selector, batch embedding still calls the shared embedding context concurrently, and the public docs provide only partial QMD workarounds rather than an official Metal-safe support path. Fresh issue comments and open #60347 show the problem is active, not stale or implemented.

## What This Changes

Provide a stable, documented Apple Silicon path for local memory embeddings, including Metal/GPU guidance, safer failure behavior, CPU-only/backend configuration, and serialized local embedding batches where needed.

## Best Possible Solution

Keep this as the canonical Apple Silicon local-embedding stability tracker. The desired end state is a documented macOS support matrix for builtin local and QMD embeddings, a validated node-llama-cpp CPU/GPU/backend config path, bounded or serialized local GGUF embedding calls, and a gateway-safe failure boundary or fallback for native embedding crashes.
