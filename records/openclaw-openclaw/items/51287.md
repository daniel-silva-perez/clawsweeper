## Summary

Current main no longer has the original 2-minute exec approval window; it uses a 30-minute default and the docs say pending exec approvals expire after 30 minutes. However, the requested user-configurable timeout is still not implemented: the approvals config type/schema only cover forwarding, and the agent-side approval request still passes the hardcoded default timeout. The open related PR #25789 tracks the same feature but is not merged, and the provided context says it is incomplete, so this issue should stay open.

## Best Possible Solution

Keep this issue open until OpenClaw exposes a supported exec approval timeout setting, validates it in the config schema, wires it through the agent/gateway approval request path, and documents the behavior. The most direct path is to complete or replace #25789 so the configured value actually affects approval expiration and wait-request budgets.
