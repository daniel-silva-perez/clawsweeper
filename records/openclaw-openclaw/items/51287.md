## Summary

Keep this open. Current main has mitigated the original two-minute pain by using a 30-minute default, but it still does not expose a user-configurable exec approval timeout in config, and the agent-side request path continues to pass hardcoded timeout constants. The related implementation candidates are not merged: #25789 is still open and described as incomplete, while #51292 was closed unmerged.

## Best Possible Solution

Keep this issue open until OpenClaw exposes a supported exec approval expiration setting, validates it in the config schema, resolves it from the chosen config surface, threads it through `ExecToolDefaults`/`pi-tools` into the exec approval request payload and wait RPC budgets, covers gateway/CLI/node approval paths, and documents the behavior. The practical path is to complete or replace #25789 with end-to-end wiring and tests.

## Risks / Open Questions

- The default is now longer, so the urgency is lower than the original two-minute report, but the requested configurability is still missing.
- A future implementation should keep the client transport timeout aligned with the approval expiration timeout plus buffer; otherwise long approval windows can still fail early.
- Maintainers should choose one stable config surface before landing this, because prior unmerged attempts used different paths such as `approvals.timeoutMs` and `tools.exec.approvalTimeoutMs`.
