## Summary

Keep open. I rechecked current main at 1b25dcf57a9fca5d00cf275046e20a2e3ea3730c: preview streaming remains config-driven, the built-in command registry and runtime handler list still have no `/stream` or `/streaming` command, session persistence has no preview-streaming override field, and Telegram/Slack/Discord preview paths still resolve from channel/account config rather than a current chat/session override. The related items in the issue are adjacent streaming or command work, not a canonical duplicate for this per-chat/session control.

## Best Possible Solution

Keep this open for maintainer product/design review. The best path is to define a shared preview-streaming override contract first, likely a session/conversation-scoped field resolved before channel config, then add a built-in `/stream` command with status/off/partial/progress/reset semantics, `/status` visibility, native command arg menus, and opt-in support in Telegram, Slack, and Discord. Unsupported channels should reply clearly instead of silently accepting the override.
