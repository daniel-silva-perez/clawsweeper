## Summary

Keep open. Current main still only supports config-driven preview streaming, with no built-in `/stream` or `/streaming` command, no session/conversation override field, and channel preview resolvers that read channel/account config rather than runtime session state. The linked reports are adjacent streaming or command-routing work, but none currently tracks this shared per-chat/session streaming-mode control as the canonical implementation.

## What This Changes

The issue requests a slash/native command to show, set, and reset preview streaming mode for the current chat or session without editing channel config or restarting the gateway.

## Best Possible Solution

Add a shared preview-streaming override contract that resolves before channel config, expose it through a built-in `/stream` command with status/set/reset behavior, surface active overrides in `/status`, and let preview-capable channels opt in while unsupported channels return a clear unsupported response.
