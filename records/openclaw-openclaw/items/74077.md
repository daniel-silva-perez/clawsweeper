## Summary

Keep this issue open. Current main still has only config-driven preview streaming: no built-in `/stream` or `/streaming` command, no loaded handler, no persisted session/conversation override, and Telegram/Slack/Discord preview paths still resolve from channel/account config. The linked items are adjacent command-alias, slash-routing, and preview-streaming reports, but none is a canonical implementation for this per-chat/session control.

## What This Changes

The issue requests a shared slash/native command such as `/stream` to show, set, or reset preview streaming mode for the current chat/session without editing gateway config or restarting the gateway.

## Best Possible Solution

Add a shared preview-streaming override contract that resolves before channel config, expose it through a built-in `/stream` command with show/set/reset behavior, surface active overrides in `/status`, and let preview-capable channels opt in while unsupported channels return a clear unsupported response.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high-confidence current behavior: current main has no `/stream` command/handler or session override field, and preview-capable channel dispatch still reads channel/account config.

## Solution Assessment

Unclear. The requested user-facing command is a good direction, but the best implementation needs maintainer design for scope, storage, authorization, and cross-channel opt-in before a narrow code fix is safe.
