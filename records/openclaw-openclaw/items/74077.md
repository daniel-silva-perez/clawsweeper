## Summary

Keep open. Current main still has only config-driven preview streaming: there is no built-in `/stream` or `/streaming` command, no session/conversation override field, and the Telegram, Slack, and Discord preview paths resolve from channel/account config. The related items in the issue body are adjacent streaming, default, observability, and slash-routing work, but they do not supersede this shared per-chat/session streaming-mode control.

## What This Changes

Request adds a slash/native command to view, set, and reset preview streaming mode for the current chat/session while keeping channel config as the fallback.

## Best Possible Solution

Add a shared preview-streaming override contract that resolves before channel config, expose it through a built-in `/stream` command with status/set/reset behavior, show active overrides in `/status`, and let preview-capable channels opt in while unsupported channels return a clear unsupported response.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The absence is high-confidence by source inspection: `/stream` is not registered or handled, session state has no override field, and current channel resolvers read config only.

## Solution Assessment

Unclear until maintainers choose the storage and scope model. The narrow maintainable direction is a shared resolver/override seam plus channel opt-in, not a Telegram-only command or a plugin-only command that cannot influence existing preview dispatch.
