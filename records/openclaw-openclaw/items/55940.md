## Summary

Keep open. Current main still returns the Homebrew-missing failure for `kind: "brew"` skill installers when `brew` cannot be resolved; only the Go installer path has an apt-get fallback. The linked Docker skill-install bug remains open, and the alternative Docker/Linuxbrew PR #42303 is also open, so this PR is neither implemented nor clearly superseded. The PR also needs maintainer security judgment because its current approach would run root/passwordless-sudo `apt-get install` from skill installer metadata.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to either revise the apt fallback so it is explicitly gated, allowlisted, or limited to trusted/bundled installers with regression tests, or decide that #42303 is the canonical Docker/Linuxbrew fix for #14593 and close this PR only after that superseding path is explicit.
