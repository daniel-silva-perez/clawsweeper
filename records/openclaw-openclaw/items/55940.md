## Summary

Keep this PR open. Current main still returns the Homebrew-missing failure for `kind: "brew"` installers when Homebrew cannot be resolved; only the Go prerequisite path has an apt-get fallback. The linked Docker skill-install bug #14593 and alternative Linuxbrew Docker PR #42303 are both still open, and this PR still needs maintainer security judgment because it changes a user-level missing-dependency failure into root/passwordless-sudo `apt-get install` from skill installer metadata.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to either revise the apt fallback so it is explicitly gated, allowlisted, limited to trusted/bundled installers, and non-interactive with regression tests, or decide that #42303 is the canonical Docker/Linuxbrew fix for #14593 and close this PR only after that superseding path is explicit or merged.

## Risks / Open Questions

- The PR can turn a missing Homebrew dependency into root or passwordless-sudo system package installation from skill metadata, including non-bundled skill sources that current main only warns about.
- The PR's sudo probe uses `sudo -n true`, but the later diff invokes `sudo apt-get ...` without `-n`; current `runCommandWithTimeout` inherits stdin when no input is supplied, so maintainer review should decide whether to force non-interactive sudo for this path.
- Closing this PR now would drop a still-relevant candidate fix for #14593, while #42303 remains open and unmerged rather than a shipped superseding path.
