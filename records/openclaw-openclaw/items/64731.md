## Summary

Keep open. Current main still only surfaces model fallback and fallback-cleared notices through the verbose-notice path, and current tests still assert that verbose-off replies suppress those notices. This PR remains an implementation candidate, but it overlaps with open PR #69399, which proposes a broader opt-in config direction that maintainers should choose between before landing either change.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to choose between this Telegram-specific exception and the broader `agents.defaults.notifyOnModelFallback` opt-in proposed in #69399, then rebase and validate the selected implementation with focused reply-runner coverage.
