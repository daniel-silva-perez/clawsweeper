## Summary

Keep open. Current main still only prepends model fallback and fallback-cleared user notices when verbose mode is enabled, and the current tests explicitly assert that verbose-off replies suppress those notices. This PR remains a focused implementation candidate; its latest patch includes the missing import fix and is limited to the reply runner plus e2e coverage. A broader overlapping PR #69399 is open, but it has not merged, so it is not a basis to close this PR yet.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to decide whether OpenClaw wants a Telegram-specific exception or the broader `agents.defaults.notifyOnModelFallback` configuration approach in #69399, then rebase and validate the chosen implementation rather than closing this still-unmerged fix as cleanup.
