## Summary

Keep this issue open. Current main still defaults `commands.nativeSkills` to `"auto"`, still resolves `auto` on for Discord and Telegram, and still documents/generated-publishes that default. Existing overflow mitigations reduce failure impact, but they do not make native skill command registration opt-in, and the narrower warning/default decision remains represented by open PR #50191.

## Best Possible Solution

Keep this open for maintainer judgment. The concrete path is to review and either land or close #50191 for targeted overflow guidance. If maintainers choose the default change, update schema defaults, generated config metadata, public docs, resolver tests, Discord/Telegram behavior expectations, and changelog/migration notes together; if they keep `auto`, make overflow diagnostics consistently point at the narrower `nativeSkills` setting.
