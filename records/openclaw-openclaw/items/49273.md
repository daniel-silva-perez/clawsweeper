## Summary

Keep this issue open. Current main has meaningful quota mitigations for Telegram and Discord, but it has not implemented the requested default change: `commands.nativeSkills` still defaults to `"auto"`, Discord/Telegram still opt into native skill command registration under auto, and the remaining warning/default product decision is still represented by the open related PR #50191.

## Best Possible Solution

Keep the issue open for maintainer judgment. The narrow near-term path is to review, update, and land or close PR #50191 for actionable overflow guidance. Separately, maintainers should decide whether to change `commands.nativeSkills` from `"auto"` to false/none for Discord and Telegram, including schema defaults, docs, generated config metadata, tests, and changelog/migration notes.
