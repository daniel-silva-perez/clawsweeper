## Summary

Keep this issue open. Current main still defaults `commands.nativeSkills` to `"auto"`, Discord and Telegram still resolve `auto` to native skill command registration, and the docs/generated schema still publish that default. Current main has overflow mitigations, but it has not made native skill command registration opt-in by default; the remaining warning/default decision is still represented by the open related PR #50191.

## Best Possible Solution

Keep this issue open for maintainer judgment. The near-term path is to review, update, and either land or close PR #50191 for actionable overflow guidance. If maintainers choose the default change, it should update schema defaults, generated config metadata, public docs, resolver tests, channel behavior expectations, and changelog/migration notes together.
