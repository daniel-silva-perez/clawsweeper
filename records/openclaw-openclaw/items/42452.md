## Summary

Keep this PR open. Current main still does not expose provider rate-limit headroom from `openclaw models status --probe`: there is no `--rate-limits` CLI option, no `rateLimit` probe result field, no RPM/TPM output columns, and no docs/tests for that contract. The PR remains a plausible implementation candidate, but it needs rebase and maintainer review because it adds a new live credentialed provider request path.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is to rebase it onto current main, decide whether rate-limit header capture belongs in the existing provider runtime/transport seam or a narrow shared seam, wire both CLI routing paths, document the `--rate-limits` JSON/table contract on the public models CLI docs, add focused parsing/output/base URL/failure cleanup tests, and remove unrelated `.gitignore` churn.
