## Summary

Keep this PR open. Current main still does not expose provider rate-limit headroom in `openclaw models status --probe`: there is no `--rate-limits` flag, no probe-result `rateLimit` payload, no provider header parsing, and no RPM/TPM table or JSON contract. The PR remains a plausible implementation candidate, but it needs rebase and maintainer/security review before merge.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rebase it onto current main, wire `--rate-limits` through both CLI paths, decide whether header capture belongs in the existing provider runtime/usage seam or a narrow shared probe seam, add focused tests for parsing/output/base URL/failure cleanup, document the public JSON/table contract, and drop unrelated `.gitignore` changes.
