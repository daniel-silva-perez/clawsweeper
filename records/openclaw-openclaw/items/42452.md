## Summary

Keep this PR open. Current main still lacks the requested `openclaw models status --probe --rate-limits` surface: there is no CLI flag, no routed-parser support, no `rateLimit` probe payload, no `x-ratelimit-*` header parsing, and no RPM/TPM table or JSON contract. The PR remains a plausible implementation candidate, but it needs rebase plus functional/security review before merge.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rebase it onto current main, wire `--rate-limits` through Commander and routed CLI paths, decide whether header capture belongs in the existing provider runtime/usage seam or a narrow probe seam, add focused parsing/output/base URL/failure-cleanup tests, document the public JSON/table contract and billing behavior, and remove unrelated `.gitignore` changes.
