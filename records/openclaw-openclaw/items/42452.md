## Summary

Keep this PR open. Current main still does not expose provider rate-limit headroom from `openclaw models status --probe`, and the PR remains a relevant implementation candidate rather than obsolete work. It needs rebase and maintainer/security review because it adds a new credentialed direct provider request path and still misses the routed CLI parser/docs surface.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is to rebase it onto current main, wire `--rate-limits` through both Commander and the routed `models status` parser, decide whether header capture belongs in the existing provider runtime/transport seam or a narrow shared seam, add focused tests for parsing/output/base URL/failure cleanup and both CLI paths, document the public `--rate-limits` JSON/table contract in the models CLI docs, and remove unrelated `.gitignore` churn.
