## Summary

Keep this PR open. Current main still does not implement optional Slack run-start timing for ack reactions: there is no `ackReactionTiming` config type/schema/help/doc surface, Slack monitor context/provider do not carry timing, and the Slack runtime still starts ack/status reactions before the agent run begins. The PR discussion remains useful: the Greptile scope-gate concern was addressed in the PR with `ackReactionAllowed`, and companion channel PRs #45449 and #46258 are still open rather than shipped replacements.

## Best Possible Solution

Keep this PR open as the Slack implementation candidate. The best path is to rebase it onto current main, keep the default receipt-time behavior, accept `received | run-start` through strict config/schema/docs, defer Slack ack/status reaction startup from `onAgentRunStart` only when configured, and keep targeted tests for scope gating and cleanup semantics.
