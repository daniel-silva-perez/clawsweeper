## Summary

Keep this PR open. Current main still does not implement Slack ack reaction run-start timing, strict Slack config does not accept the proposed key, Slack runtime still starts ack/status reactions at receipt time, and the same contributor still has open companion channel PRs for the same feature. The PR needs maintainer review and a rebase, not conservative cleanup closure.

## Best Possible Solution

Keep this PR open and ask for a rebase against current main. The best path is a Slack-scoped implementation that keeps received as the default, adds a validated ackReactionTiming config contract through the public config/plugin-sdk surface, defers the initial ack/status reaction from onAgentRunStart when configured, preserves existing scope and cleanup semantics, and updates Slack docs plus targeted prepare/dispatch tests. Coordinate the shared type/schema decision with #45449 and #46258 before merge.
