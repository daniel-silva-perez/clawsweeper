## Summary

Keep this issue open. Current main has adjacent browser tab cleanup for primary idle/cap sweeps and lifecycle/reset paths, but it still does not close temporary primary-agent browser tabs when an ordinary chat or agent task completes, and the tracked-tab model still lacks ephemeral/preserved ownership metadata.

## Best Possible Solution

Implement a focused browser lifecycle change: add task-scoped ephemeral/preserved metadata to tracked browser tabs, make all browser open paths including proxy-routed opens participate, invoke best-effort cleanup for ephemeral primary-agent tabs on successful chat/agent turn completion, preserve explicit keep-open/debug/user-owned/existing-session tabs, and add regression tests for completion cleanup, keep-open preservation, and existing-session safety.

## Risks / Open Questions

- A naive close-on-completion change could close user-intended or existing-session tabs unless the registry gains explicit ephemeral versus preserved/user-owned state.
- Proxy-routed browser opens currently return before the direct registry tracking call, so a future completion cleanup path should first make all relevant open paths consistently track temporary tabs.
- The current docs promise reusable primary tabs until idle/cap cleanup, so changing the default should include an explicit keep-open/debug preservation path and docs update.
