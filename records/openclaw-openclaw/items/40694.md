## Summary

Keep this issue open. Current main still has browser cleanup for idle/cap sweeps and lifecycle-owned subagent/cron/reset paths, but not automatic close-on-successful-completion for ordinary primary chat or agent browser work. The related closed #56271 does not supersede this broader remaining gap.

## Best Possible Solution

Implement task-scoped browser tab ownership in the bundled browser plugin and gateway run lifecycle: classify opened tabs as temporary or preserved, track direct and proxy-routed opens consistently, run best-effort cleanup after successful primary chat/agent completion, preserve explicit keep-open/debug/user-owned tabs, and add focused regression tests plus docs/config updates.

## Risks / Open Questions

- A naive completion hook could close existing-session, user-owned, labeled, or debug tabs unless the registry first gains explicit temporary/preserved ownership semantics.
- Proxy-routed browser opens currently bypass the direct registry tracking path, so task-completion cleanup would be incomplete unless node/proxy opens are tracked consistently.
