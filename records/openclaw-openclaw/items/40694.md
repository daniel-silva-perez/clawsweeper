## Summary

Keep this issue open. Current main still has browser tab cleanup for idle/cap sweeps and lifecycle/reset paths, but not automatic close-on-successful-task-completion for ordinary primary chat/agent browser work. The later related #56271 is closed and does not supersede the remaining broader gap here.

## Best Possible Solution

Implement a focused browser lifecycle change in the bundled browser plugin and gateway run lifecycle: add task-scoped tab ownership metadata, make direct and proxy-routed open paths track temporary vs preserved tabs, invoke best-effort cleanup after successful primary chat/agent turn completion, preserve explicit keep-open/debug/user-owned/existing-session tabs, and add regression tests plus docs/config updates for the new behavior.

## Risks / Open Questions

- A naive close-on-completion patch could close existing-session, user-intended, labeled, or debug tabs unless the registry first gains explicit ephemeral/preserved ownership metadata.
- Proxy-routed browser opens currently do not participate in the same direct registry tracking path, so completion cleanup could be incomplete unless those opens are tracked consistently.
