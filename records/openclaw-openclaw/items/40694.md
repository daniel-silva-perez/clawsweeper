## Summary

Keep this issue open. Current main still implements partial browser cleanup for idle/cap sweeps, session reset/delete, cron, and subagent lifecycle paths, but not the requested default close-on-task-completion behavior for ordinary primary chat or agent browser work. The related closed #56271 does not supersede the broader remaining gap.

## Best Possible Solution

Implement task-scoped browser tab ownership in the browser plugin and gateway run lifecycle: classify opened tabs as temporary or preserved, track direct and proxy-routed opens consistently, run best-effort cleanup after successful primary chat/agent completion, preserve explicit keep-open/debug/user-owned tabs, and add focused regression tests plus docs/config updates.

## Risks / Open Questions

- A naive completion hook could close user-owned, existing-session, labeled, debug, or intentionally preserved browser tabs unless the registry first gains explicit temporary/preserved ownership semantics.
- Proxy-routed browser opens currently bypass direct registry tracking, so completion cleanup would be incomplete unless node/proxy opens are tracked consistently.
