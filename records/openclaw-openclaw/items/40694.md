## Summary

Keep open. Current main has meaningful browser tab cleanup for idle/cap sweeps, subagent completion, isolated cron completion, and session reset/delete paths, but it still does not implement the issue's requested default close-on-task-completion behavior for ordinary primary chat or primary agent browser work. Related closed items are narrower or superseded old-path attempts, not a canonical remaining-work tracker.

## Best Possible Solution

Keep this issue open and implement task-scoped browser tab ownership in the bundled browser plugin plus gateway run lifecycle. Mark automation-opened tabs as temporary or preserved, track direct and node/proxy opens consistently, run best-effort cleanup after successful primary chat/agent completion, preserve explicit keep-open/debug/user-owned tabs, and add focused tests plus docs/config updates. If maintainers prefer narrower tracking, open a canonical implementation issue that explicitly covers the remaining primary chat/agent gap and then supersede this issue with that link.

## Risks / Open Questions

- Closing now would leave ordinary primary chat/agent browser workflows without the requested close-on-task-completion default; tabs can remain until idle/cap cleanup or manual close.
- A naive completion hook could close user-owned, existing-session, debug, or intentionally preserved tabs because current tracked-tab records do not distinguish temporary automation ownership from preserved/user-owned tabs.
- Node/proxy-routed browser opens currently bypass direct session-tab registry tracking, so a completion cleanup implementation that only closes locally tracked tabs would be incomplete.
