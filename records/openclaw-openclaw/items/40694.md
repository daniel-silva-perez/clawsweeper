## Summary

Current main still does not implement the requested default close-on-task-completion behavior for ordinary primary chat or primary agent browser work. Browser cleanup is implemented for idle/cap sweeps and selected lifecycle paths such as subagent, isolated cron, and session reset/delete cleanup, but primary sessions intentionally keep active tabs reusable until background cleanup reclaims idle or excess tabs.

## Best Possible Solution

Keep this issue open. Implement task-scoped browser tab ownership in the bundled browser plugin and gateway run lifecycle: mark automation-opened tabs as temporary or preserved, track direct and node/proxy opens consistently, run best-effort cleanup after successful primary chat/agent completion, preserve explicit keep-open/debug/user-owned tabs, and add focused tests plus docs/config updates. If maintainers prefer narrower tracking, open a canonical implementation issue for the remaining primary chat/agent gap and then supersede this issue with that link.
