## Summary

Keep this issue open. Current main has partial browser tab cleanup for idle/cap sweeps, session reset/delete, subagent completion, and isolated cron completion, but it still does not implement the requested default close-on-task-completion behavior for ordinary primary chat or primary agent/browser work. The related closed #56271 is narrower and does not supersede this remaining gap.

## Best Possible Solution

Implement task-scoped browser tab ownership in the bundled browser plugin and gateway run lifecycle: mark automation-opened tabs as temporary or preserved, track direct and node/proxy opens consistently, run best-effort cleanup after successful primary chat/agent completion, preserve explicit keep-open/debug/user-owned tabs, and add focused regression tests plus docs/config updates. Keep this issue open until that broader behavior lands or a canonical open implementation issue/PR supersedes it.

## Risks / Open Questions

- Closing now would leave the ordinary primary chat/agent browser workflow without the requested close-on-completion behavior.
- A naive completion hook could close user-owned, existing-session, debug, or intentionally preserved tabs because the registry does not yet record temporary/preserved ownership semantics.
- Node/proxy-routed browser opens currently bypass direct registry tracking, so a completion cleanup implementation would be incomplete unless those opens are tracked consistently.
