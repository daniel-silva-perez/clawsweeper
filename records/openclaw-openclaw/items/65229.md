## Summary

Keep this PR open for maintainer handling, but it should not be merged as-is. Current main still has the doctor false-positive path this PR targets, so it is not implemented on main. The PR’s final approach also has an unresolved review concern: it would call a throwing SecretRef resolver from the doctor health path, which can turn the intended read-only warning into a doctor abort when the ref is unavailable. The branch is also dirty for this stated fix because it includes unrelated memory-core, generated schema, gateway scope, provider-contract, and secrets test-support changes.

## Best Possible Solution

Refresh or replace this PR with a focused doctor fix: resolve `gateway.auth.token` through the non-throwing SecretRef resolution path, pass the resolved token as an auth override only when available, preserve the current warning when unavailable, add a regression test for a resolvable env-backed SecretRef, and drop unrelated dirty-branch files. If #67463 becomes the maintainer-chosen implementation, use that PR as the canonical follow-up for this bug and close this one only after that decision is explicit or after it lands.
