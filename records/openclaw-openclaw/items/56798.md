## Summary

Keep open. The current PR branch is not implemented on main and still has a concrete correctness issue: it treats any observed 250 ms Node network-family autoselection attempt timeout as the runtime default, even though Node also returns 250 ms when an operator explicitly sets that value via CLI flag or NODE_OPTIONS. The diff also no longer matches the PR body’s module-load autoSelectFamily description, so this needs maintainer review rather than cleanup closure.

## What This Changes

The PR adds a guarded call inside `ensureGlobalUndiciStreamTimeouts()` to raise Node's default network-family autoselection attempt timeout from 250 ms to 300 ms, plus tests for default-vs-custom timeout handling.

## Best Possible Solution

Keep the PR open and revise it so process-wide Node network-family timeout changes either preserve explicit 250 ms operator policy or become opt-in. The branch should be rebased, the PR body should match the actual implementation, and validation should cover the provider/guarded-fetch path plus existing dispatcher behavior.
