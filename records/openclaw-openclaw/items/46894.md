## Summary

Current main still treats non-empty Control UI `/model` args as model changes, so `/model status` and `/model list` would still call `sessions.patch`. The issue should stay open because open implementation PRs #47069 and #46895 are already paired with this behavior and should be reviewed or closed rather than bypassed.

## Best Possible Solution

Keep this issue open and review the paired implementation candidates, especially #47069. The best fix is a narrow Control UI change: route `/model status`, `/model list`, and likely `/model info` through the same model-info path as bare `/model`, add regression tests proving those inputs do not call `sessions.patch`, and preserve normal `/model <model>` behavior.
