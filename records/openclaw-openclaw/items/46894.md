## Summary

This issue should stay open. Current `main` still intercepts Control UI `/model ...` commands locally and sends any non-empty argument through `sessions.patch`, while open unmerged PRs #46895 and #47069 are already implementation candidates for the reported behavior.

## Best Possible Solution

Review and either land or close the linked implementation PRs. The likely fix belongs in `ui/src/ui/chat/slash-command-executor.ts`: route `/model status`, `/model list`, and probably `/model info` to the local model-info path without calling `sessions.patch`, with focused regression tests preserving `/model <provider/model>` switching.

## Evidence
