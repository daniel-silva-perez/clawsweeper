## Summary

Current main still has the reported Web UI behavior: `/model` is a local Control UI command, and the local executor only uses the informational model path when the argument string is empty. Non-empty args such as `status` and `list` are still sent to `sessions.patch` as model names. The issue should stay open because open PRs #47069 and #46895 are already paired implementation candidates for the same behavior.

## Best Possible Solution

Keep this issue open and review the paired implementation candidates, especially #47069. The desired fix belongs in the Control UI local slash-command executor: route `/model status`, `/model list`, and likely `/model info` through the same model-info path as bare `/model`, add regression tests proving these inputs do not call `sessions.patch`, and leave normal `/model <model>` patching unchanged.
