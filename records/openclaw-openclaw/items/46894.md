## Summary

Current main still routes Control UI `/model status` and `/model list` through the non-empty `/model <name>` path that calls `sessions.patch`, and open PR #47069 still targets this exact regression. The issue should stay open until maintainers land, replace, or close that paired fix path.

## What This Changes

The issue requests that the Control UI treat `/model status` and `/model list` as informational slash-command subcommands instead of passing `status` or `list` as model ids to `sessions.patch`.

## Best Possible Solution

Keep the issue open and land or replace a narrow Control UI fix that routes normalized informational model subcommands such as `status`, `list`, and any maintainer-approved alias through the existing model-info path, with focused executor tests proving those inputs do not call `sessions.patch` while real `/model <provider/model>` changes still work.

## Reproduction Assessment

Yes. The source-level reproduction is deterministic on current main: `/model status` parses as command `model` with args `status`, then `executeModel` calls `sessions.patch` for that non-empty arg, matching the reporter's gateway error logs.

## Solution Assessment

Yes. The requested direction is the narrowest maintainable fix: update `ui/src/ui/chat/slash-command-executor.ts` and its colocated tests, while leaving the already-correct shared directive path and core protocol alone.
