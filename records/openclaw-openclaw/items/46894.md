## Summary

Keep open. Current main still has the reported Control UI behavior: `/model` is executed locally, `/model status` and `/model list` arrive as non-empty args, and the local executor still sends any non-empty arg to `sessions.patch` as a model name. The issue is already paired with open implementation candidates, especially #47069, so maintainers should review or update those PRs instead of closing this issue early.

## What This Changes

The issue reports that the Control UI local `/model` slash-command handler treats `/model status` and `/model list` as model IDs and sends them through `sessions.patch` instead of showing model information.

## Best Possible Solution

Land or replace the paired Control UI fix so `status`, `list`, and `info` route through the model-info path, add focused executor regression tests proving those inputs do not call `sessions.patch`, and preserve existing model-change behavior for real model identifiers.

Yes. The code-level path is deterministic on current main: `/model status` is parsed as local command `model` with args `status`, and `executeModel` sends any non-empty args to `sessions.patch`, matching the reporter's gateway error logs.
