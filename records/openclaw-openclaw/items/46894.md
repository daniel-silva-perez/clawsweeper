## Summary

Keep open. Current main still reproduces the reported Control UI path: `/model` is local, `/model status` and `/model list` are passed as non-empty args, and the executor sends any non-empty value to `sessions.patch` as a model name. The issue is also paired with open implementation PRs, especially #47069, so it should remain open until a fix lands.

## What This Changes

The issue requests that the Control UI treat `/model status` and `/model list` as informational `/model` subcommands instead of model identifiers sent to `sessions.patch`.

## Best Possible Solution

Land or replace the paired Control UI fix so `status`, `list`, and any maintainer-approved informational aliases route through the model-info path, with focused executor tests proving those inputs do not call `sessions.patch` while real `/model <provider/model>` changes still work.

## Reproduction Assessment

Yes. The code-level path is deterministic on current main: `/model status` is parsed as local command `model` with args `status`, and `executeModel` sends that non-empty arg to `sessions.patch`, matching the reporter's gateway error logs.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Control UI local slash-command executor with colocated regression tests; the already-open PR #47069 targets that boundary and should be reviewed, rebased, or replaced.
