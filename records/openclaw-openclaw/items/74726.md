## Summary

Keep open. Current main supports configured MCP servers as prefixed `bundle-mcp` tools in embedded Pi/main-agent tool profiles, but it does not implement the specific behavior requested here: the built-in `image` tool still falls back only across configured image model/provider candidates and does not dispatch failed image understanding to an arbitrary MCP `understand_image` tool.

## What This Changes

The issue requests that configured `mcp.servers` tools, especially an MCP `understand_image` tool, be usable by the main agent as a fallback when the built-in `image` tool's provider path fails.

## Best Possible Solution

Maintainers should either document direct use of exposed prefixed MCP tool names as the intended contract, or add a generic media-understanding MCP adapter/fallback contract that can discover and invoke configured MCP tools without hardcoding MiniMax-specific server names or schemas into core.
