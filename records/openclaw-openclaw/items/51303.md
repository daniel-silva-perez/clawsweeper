## Summary

Keep this PR open. The original `child.ts` patch is stale against current main, but the underlying UTF-8 split-chunk problem is still present in the shared default decoder path, so current main does not yet implement the PR’s intended fix.

## Best Possible Solution

Keep this PR open, but ask for a port into `src/infra/windows-encoding.ts`: make the default UTF-8 path streaming/boundary-safe, preserve the Windows legacy-codepage fallback, and add focused tests for split UTF-8 chunks when `platform` is non-Windows or `windowsEncoding` is null/UTF-8.
