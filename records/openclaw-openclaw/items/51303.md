## Summary

Keep this PR open. Current main moved child stdout/stderr through a shared decoder, but the default non-Windows UTF-8 path still decodes each Buffer independently with `buffer.toString("utf8")`, so main does not implement the PR’s intended boundary-safe streaming fix. The PR should be ported into the shared decoder while preserving the newer Windows legacy-codepage behavior.

## Best Possible Solution

Keep the PR open and port the fix into the current shared child-output decoder: make the default UTF-8 path streaming/boundary-safe, preserve Windows legacy-codepage fallback behavior, and add focused regression coverage for split UTF-8 chunks plus stdout/stderr flush behavior before maintainer review or landing.
