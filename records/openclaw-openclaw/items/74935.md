## Summary

Current main still routes tool-card sidebar payloads through the markdown renderer that collapses JSON fences, while this PR adds a focused tool sidebar renderer with targeted tests and no blocking review findings. The remaining action is maintainer review and landing validation, not cleanup closure.

## What This Changes

This PR adds a `tool` `SidebarContent` variant, routes tool-card sidebar actions to it, renders expanded tool input/output with empty states and raw fallback, updates sidebar CSS, and adds focused UI tests.

## Best Possible Solution

Land a reviewed version that keeps the generic markdown JSON-collapse behavior intact while routing tool-card details through an escaped, dedicated sidebar renderer with regression coverage for expanded input, output, empty states, JSON formatting, and raw fallback.

## Reproduction Assessment

Yes. Current main builds tool sidebar content as markdown with fenced JSON and then renders it through the markdown sidebar path whose fence rule wraps JSON in a collapsed `<details>` block; a focused tool-card fixture is enough to reproduce without a live gateway run.

## Solution Assessment

Yes. The dedicated `tool` sidebar content kind is the narrowest maintainable fix because it avoids disabling markdown JSON collapse globally and preserves the existing canvas and markdown sidebar routes.
