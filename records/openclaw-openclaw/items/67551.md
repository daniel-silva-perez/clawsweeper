## Summary

Keep this PR open. Current main still lacks structural Unicode smart-quote normalization for tool-call argument JSON; it only parses raw ASCII JSON or extracts a balanced ASCII-delimited JSON prefix. The merged related #70294 broadened when the existing malformed-argument repair wrapper runs for openai-completions, but it did not change the parser/scanner behavior this PR targets.

## Best Possible Solution

Keep this PR or an equivalent follow-up open. Rebase onto current main, implement delimiter-aware Unicode smart-quote repair in the current shared/parser structure, preserve #70294's openai-completions wrapper enablement, and add focused regression tests for U+201C/U+201D structural delimiters, content-level smart quotes inside ASCII strings, and trailing-junk repair before maintainer review.
