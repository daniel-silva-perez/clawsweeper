## Summary

Keep open. Current main still lacks the proposed TUI user-message color overrides, and the PR remains a useful narrow contribution, but the head patch has an unresolved readability defect: a background-only override can combine with the base text color at very low contrast on the active user-message rendering path. Existing review comments already flagged the same area, so this needs maintainer policy review or author follow-up before merge.

## What This Changes

The PR adds `OPENCLAW_TUI_USER_BG` and `OPENCLAW_TUI_USER_TEXT` TUI palette overrides with strict `#RRGGBB` validation, theme tests, and environment/TUI documentation updates.

## Best Possible Solution

Land the customization only once the theme layer preserves readable user-message text for single-variable overrides, defines malformed and low-contrast override behavior, and keeps the environment and TUI docs aligned with that policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Derive readable text for background-only overrides:** `src/tui/theme/theme.ts:141`
  - body: The docs advertise setting `OPENCLAW_TUI_USER_BG` and/or text, but this spread lets a background-only override keep the base `userText`. In dark mode, `#ffffff` keeps `#F3EEE0` text at about 1.16:1 contrast; in light mode, `#000000` keeps `#1E1E1E` at about 1.26:1. Since `UserMessageComponent` renders both colors together, derive or require readable text, or reject unsafe background-only overrides.
