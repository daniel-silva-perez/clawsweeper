## Summary

Keep this PR open. Current main still lacks the proposed user-message color override env vars and docs, while the PR is a focused implementation but has an unresolved readability edge case around background-only and low-contrast overrides.

## What This Changes

The PR adds `OPENCLAW_TUI_USER_BG` and `OPENCLAW_TUI_USER_TEXT` palette overrides for TUI user-message colors, documents the env vars, and adds theme tests for valid and invalid hex values.

## Best Possible Solution

Land the env-var customization only after the theme code preserves readable user-message text for single-var and paired overrides, keeps invalid input falling back to defaults, and documents the final behavior consistently in the environment and TUI docs.
