## Summary

Keep this PR open. Current main still lacks the proposed TUI user-message color env overrides, and the PR is a narrow additive implementation, but active review feedback identifies an unresolved readability policy for low-contrast and background-only overrides that needs maintainer or author follow-up before merge.

## What This Changes

The PR adds `OPENCLAW_TUI_USER_BG` and `OPENCLAW_TUI_USER_TEXT` palette overrides for TUI user-message colors, documents them, and adds theme tests for valid and invalid hex values.

## Best Possible Solution

Land the customization only once the TUI theme code keeps user-message text readable for single-var and paired overrides, keeps malformed values falling back to defaults, and documents the final behavior consistently in the environment and TUI docs.
