## What This Changes

The issue reports that Matrix E2EE recovery on OpenClaw 2026.4.26 cannot be completed with a recovery key, and that `matrix encryption setup` then fails during bootstrap with a Matrix one-time-key upload conflict.

## Best Possible Solution

Keep this with Matrix maintainer triage. Reproduce the recovery-key verify and encryption setup sequence on current main or a current packaged build, then fix the narrow owner module behavior: either validate and report recovery-key acceptance consistently when backup material is usable, improve the CLI next steps for the accepted-no/usable-yes state, or add explicit handling for the Matrix one-time-key upload conflict during setup bootstrap.
