## Summary

Keep open. Current main still has the duplicated Control UI header surfaces this PR targets and the related Usage/config PRs do not supersede the broad cleanup, but this patch is not correct as-is because it drops dynamic Sessions store-path information, leaves blank header scaffolding in the UI, lacks the required user-facing changelog entry, and needs a careful rebase over current breadcrumb link/basePath work.

## What This Changes

This PR changes Control UI rendering, CSS, and one cron test to add tab subtitles to the dashboard breadcrumb, remove view-local title/subtitle blocks across multiple tabs, and simplify the Config pending-changes/status presentation.

## Best Possible Solution

Land a narrower, rebased Control UI cleanup that moves top-level page context into the breadcrumb, removes only truly duplicate page headers, preserves dynamic per-page details and meaningful section labels, keeps chat/dream controls intact, intentionally handles Config diff/status behavior, and adds focused render/layout coverage plus the required changelog entry.

## Reproduction Assessment

Yes. Source inspection against current main and the PR head gives a high-confidence reproduction path: current main renders both dashboard-header and content-header, while the PR head removes the Sessions store path and leaves blank header/row scaffolding without needing a live Gateway.

## Solution Assessment

No as-is. The consolidation direction is plausible, but the best fix should be a current-main rebase that preserves dynamic Sessions information, avoids blank layout placeholders, keeps current breadcrumb navigation semantics, and documents the user-facing UI change.
