## Summary

Keep this PR open as useful source work, but it is not merge-ready. Current main still lacks configurable Slack assistant thread status text, while this branch targets the removed core `src/slack` path and exposes a Slack-specific public config key through shared agent identity instead of the active Slack plugin/config surface.

## What This Changes

The PR adds `identity.typingStatuses` to core identity config/schema, adds a random typing-status resolver in `src/agents/identity.ts`, and changes the old Slack dispatch handler to use that resolver instead of the hardcoded typing status.

## Best Possible Solution

Define a validated Slack-owned status/loading-message config, or a generic channel status contract if maintainers want this across channels, then implement it in `extensions/slack`, update generated config metadata and Slack docs, and cover default plus configured status payloads in Slack monitor tests.

## Reproduction Assessment

Yes. The current-main gap is reproducible by source inspection and the existing Slack monitor test: the active plugin path always sends `status: "is typing..."`, and there is no accepted config field for custom Slack status messages.

## Solution Assessment

No. The feature is useful, but this patch targets an inactive pre-plugin Slack path and puts Slack-specific public API in shared identity config; a Slack config field or explicit generic channel status contract is the narrower maintainable path.
