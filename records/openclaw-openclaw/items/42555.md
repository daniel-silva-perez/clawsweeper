## Summary

Keep this PR open, but do not merge it as submitted. The requested Slack typing-status customization is still absent from current main, while the branch targets the removed `src/slack` tree and adds a Slack-specific public key to core agent identity instead of the active Slack plugin/config surface now discussed in #67129 and #33413.

## What This Changes

The PR adds optional `identity.typingStatuses`, a resolver that randomly selects one configured status, and changes the old Slack dispatch path to send that status instead of the hardcoded typing text.

## Best Possible Solution

Implement this in the active Slack plugin with a typed, validated Slack status/loading-message config contract, wire that contract through `setSlackThreadStatus`, cover default and configured payloads in Slack monitor tests, and update Slack docs/generated config metadata. If maintainers prefer per-agent personality, define a generic channel status contract first rather than adding a Slack-only field to core identity.
