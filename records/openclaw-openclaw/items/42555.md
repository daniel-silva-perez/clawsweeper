## Summary

Keep this PR open, but it is not merge-ready. The requested Slack typing-status customization is still missing on current main and latest release, while this branch edits an obsolete Slack path and adds a Slack-specific public key to shared core identity config instead of the active bundled Slack plugin/config surface.

## What This Changes

The PR adds `identity.typingStatuses` to core agent identity config/schema, adds a random `resolveTypingStatus()` helper, and changes Slack dispatch to send the selected typing status.

## Best Possible Solution

Define a validated Slack-owned status/loading-message contract, or a generic channel status contract if maintainers want this across channels, then route it through `extensions/slack`, validate non-empty entries and Slack limits, update generated config metadata/docs, and cover default plus configured status payloads in Slack monitor tests.
