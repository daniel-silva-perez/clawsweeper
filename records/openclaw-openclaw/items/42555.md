## Summary

Keep open: current main still lacks configurable Slack assistant status messages, but this PR is not merge-ready because it targets the removed pre-plugin Slack path and adds a Slack-specific public field to shared identity config. The related Slack status issues remain open for maintainer product/API direction.

## What This Changes

The PR adds `identity.typingStatuses`, a random typing-status resolver in agent identity helpers, and changes the old Slack dispatch handler to use the selected status string.

## Best Possible Solution

Define a validated Slack-owned status/loading-message config, or a generic channel status contract if maintainers want this across channels, then implement it in the active `extensions/slack` plugin path with schema/metadata/docs updates and tests for default, configured, and invalid empty status entries.

## Reproduction Assessment

Yes. Source inspection and the existing Slack monitor test reproduce the current-main gap: the active plugin always sends `status: "is typing..."`, and no accepted config field allows custom Slack status messages.

## Solution Assessment

No. The feature is useful, but this patch targets an inactive pre-plugin Slack path and makes a Slack-specific setting part of shared identity config; a Slack config field or explicit generic channel status contract is the narrower maintainable solution.
