## Summary

Keep this PR open, but do not merge it as-is. Current main still lacks configurable Slack assistant-thread status messages, yet the branch targets a removed Slack shim path and adds a Slack-specific public key to core agent identity instead of resolving the Slack plugin or generic channel-status config contract.

## What This Changes

The PR adds `identity.typingStatuses` to core identity config/schema, adds a random `resolveTypingStatus()` helper, and changes the old Slack dispatch path to send that selected status when typing starts.

## Best Possible Solution

Define a validated Slack status/loading-message config contract in the bundled Slack plugin, or first add a generic channel status contract if this should work across channels. The implementation should route through `extensions/slack`, validate non-empty entries and Slack limits, update generated config metadata and Slack docs, and cover default plus configured status payloads in Slack monitor tests.
