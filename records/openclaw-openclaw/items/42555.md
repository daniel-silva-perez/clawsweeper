## Summary

Keep this PR open for maintainer handling, but do not merge it as-is. Current main still lacks configurable Slack typing status text, and the submitted patch targets the removed `src/slack` tree while adding a Slack-specific public key to core agent identity.

## What This Changes

The PR adds optional `identity.typingStatuses`, a helper that randomly chooses one configured typing status, and a Slack dispatch change that sends that status instead of the hardcoded typing text.

## Best Possible Solution

Implement this in the active Slack plugin with a typed and validated Slack status/loading-message config contract, route it through `setSlackThreadStatus`, cover default and configured payloads in Slack monitor tests, and update Slack docs plus generated config metadata. If per-agent personality is preferred, define a generic channel status contract before expanding core identity.
