## Summary

Keep this PR open because the requested Slack typing-status customization is still absent from current main. The branch is not merge-ready as submitted: it targets the removed `src/slack` layout and adds a Slack-specific public config key to core agent identity, while related open threads are now framing the feature as Slack channel status/loading-message configuration.

## What This Changes

The PR adds an optional `identity.typingStatuses` array, a resolver that randomly selects a status string, and changes the old Slack dispatch path to send that selected status instead of the hardcoded typing text.

## Best Possible Solution

Implement the feature in the active Slack plugin surface with a typed, validated Slack status/loading-message config contract, wire it through `setSlackThreadStatus`, add focused Slack monitor tests for default and configured payloads, and update Slack configuration docs and generated metadata. If maintainers still want per-agent personality rather than channel config, first define a generic channel status contract instead of adding Slack-only fields to core identity.
