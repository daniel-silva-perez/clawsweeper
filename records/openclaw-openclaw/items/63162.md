## Summary

Keep this issue open. Current main has adjacent Control UI file handling, but not the requested workspace file explorer toggle/sidebar. The existing surfaces are chat upload/drop for user-selected local files and an Agents > Core Files editor for a fixed allowlist of bootstrap/persona files, not a collapsible workspace tree with search, context menu, refresh, and open-in-chat behavior.

## Best Possible Solution

If maintainers want this feature, implement it as a first-class Control UI and Gateway capability: add a scoped workspace tree/list/read API with safe-root enforcement, then add a collapsible explorer sidebar in the Chat view with refresh/search/open-in-chat behavior and tests for path safety, large trees, and attachment/read-command integration.
