## Summary

Keep this issue open. Current main still lacks the requested Control UI file explorer toggle/sidebar next to Cron and the chat-adjacent workflow for browsing workspace files, selecting them into chat, search/refresh, and context actions. The current related surfaces are chat attachment upload/drop and an Agents > Files editor backed by the allowlisted `agents.files.*` bootstrap-file API. Related #61368 and open PR #62417 are relevant, but they track a broader Agents-panel workspace manager and are not merged proof that this chat/topbar explorer request is implemented.

## Best Possible Solution

Keep this open as the chat-adjacent Control UI explorer request. The best path is to first settle or land a safe generic workspace tree/list/read API, likely through the #61368/#62417 design after review fixes, then decide whether the product surface belongs in Agents > Workspace, a Chat sidebar toggle, or both. If maintainers choose the sidebar, implement a scoped explorer with refresh/search/open-in-chat behavior and tests for path safety, large trees, attachment/read-command integration, and permission boundaries.

## Risks / Open Questions

- Any implementation would expose gateway workspace filesystem browsing from a browser UI, so it needs explicit safe-root enforcement, symlink and hardlink handling, scope separation for read/write/delete, file-size limits, and careful path redaction.
- Related PR #62417 is unmerged and has bot review comments about overwrite enforcement and UTF-8 byte-size limit checks; it should not be treated as implemented until those concerns are reviewed and resolved.
- Product shape is still unsettled: #61368/#62417 point toward an Agents-panel Workspace tab, while this issue asks for a chat/topbar explorer sidebar with open-in-chat behavior.
