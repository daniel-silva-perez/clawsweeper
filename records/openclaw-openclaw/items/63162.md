## Summary

Keep open. Current main still lacks the requested Control UI file explorer toggle/sidebar and the backing arbitrary workspace tree/read API; the existing file surfaces are chat upload/drop attachments and an Agents > Files bootstrap-file editor, while the broader workspace-file-management design remains open in related work.

## What This Changes

Add a Control UI top-bar folder button near Cron/Settings that toggles a workspace file-tree sidebar with search, refresh, open-in-chat, context actions, and drag/drop support.

## Best Possible Solution

Land or revise a generic, scoped workspace tree/list/read API first, then add the chosen Control UI surface for browsing files with search/refresh/open-in-chat behavior and focused tests around path safety, large directories, permissions, and attachment/read-command integration.
