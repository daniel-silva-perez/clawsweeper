## Summary

Keep this PR open. Current main still does not expose Matrix MessagePresentation metadata: the Matrix runtime outbound adapter only has text/media/poll sends, and the Matrix channel facade only forwards those same three methods. The PR remains a plausible bundled Matrix plugin feature, but it still needs maintainer review because the submitted change is runtime-only and the Matrix extraContent path can override base event content if forwarded without constraints.

## Best Possible Solution

Keep this PR open and request revisions: expose Matrix presentationCapabilities, renderPresentation, and sendPayload through the Matrix channel facade or a shared delegate helper; add facade/core-delivery coverage proving MessagePresentation reaches Matrix events; preserve mediaUrls and audioAsVoice behavior; and constrain Matrix extraContent forwarding to the intended com.openclaw.presentation metadata without allowing overrides of core event fields.
