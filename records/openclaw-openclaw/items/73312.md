## Summary

Keep this PR open. Current main still does not implement Matrix MessagePresentation metadata delivery, and the submitted work is not obsolete. The PR remains a plausible bundled Matrix plugin feature, but it still needs revisions so core delivery can actually reach the new Matrix payload methods and so Matrix event-content metadata is constrained safely.

## Best Possible Solution

Keep this PR open and request revisions: expose Matrix presentationCapabilities, renderPresentation, and sendPayload through the Matrix channel facade or extend createRuntimeOutboundDelegates; add facade/core-delivery coverage proving MessagePresentation reaches Matrix events; keep the mediaUrls and audioAsVoice fixes; and restrict Matrix extraContent forwarding to the intended com.openclaw.presentation payload so event content cannot be overridden.
