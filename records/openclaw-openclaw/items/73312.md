Keep this PR open. Current main still does not implement Matrix MessagePresentation metadata, and the PR remains a plausible bundled Matrix plugin feature rather than obsolete work. The open concerns are implementation completeness and Matrix event-content safety, not whether the request is already shipped.

## Best Possible Solution

Keep this PR open and request revisions: expose Matrix presentationCapabilities, renderPresentation, and sendPayload through the Matrix channel facade or extend the shared runtime delegate helper; add facade/core-delivery coverage proving MessagePresentation reaches Matrix events; preserve the existing mediaUrls and audioAsVoice fixes; and constrain Matrix extraContent forwarding to the intended com.openclaw.presentation metadata instead of accepting arbitrary event-content overrides.
