## Summary

Keep open. Current main still has the streaming-gated steer behavior, while this PR is an open, dirty implementation branch that overlaps newer maintainer-labeled work in #73365, so the right outcome is maintainer review rather than cleanup closure.

## What This Changes

This PR changes auto-reply steer handling to attempt injection for active runs regardless of streaming state, abort the embedded run on non-heartbeat injection failure, remove the runner `isStreaming` parameter, and add focused queue/agent-runner regression tests.

## Best Possible Solution

Use the maintainer-reviewed lifecycle-aware steer path from #73365 or a narrow successor, preserving this PR's useful heartbeat guard/test context if needed, then document the final queue semantics and close the linked soft-steer tracker after merge.
