## Summary

Keep this PR open. Current main still has the reported steer-mode gap, and this PR remains a focused implementation candidate for the open soft-steer request, but it is currently dirty against main and changes active-run interruption semantics that should be accepted explicitly before merge. The prior review comments were useful: the branch has addressed the misleading test name, removed the unused `isStreaming` parameter, and added the non-heartbeat guard for the abort fallback.

## What This Changes

The PR changes steer handling so active runs attempt steer injection regardless of streaming state, aborts the active embedded run when non-heartbeat steer injection fails, removes the now-unused `isStreaming` runner parameter, and adds focused auto-reply queue regression tests.

## Best Possible Solution

The best outcome is a rebased steer-mode fix that preserves successful mid-stream injection, aborts only the accepted failed-injection cases so queued followups drain promptly, keeps heartbeat drops non-destructive, updates the public queue docs if behavior changes, and lands with focused auto-reply tests.
