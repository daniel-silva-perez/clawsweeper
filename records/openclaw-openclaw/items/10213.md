## Summary

Keep open. Current main still lacks the requested Anthropic-native server-side compaction support: there is no config/schema surface, Anthropic request construction does not inject `context_management.edits`, streaming output does not preserve compaction blocks or `compaction_delta`, and `compaction` is still not a handled stop reason. The linked implementation PR #10214 is closed unmerged, while #54041 and #65287 are broader related trackers rather than a clean replacement for this focused request.

## What This Changes

The issue requests opt-in support for Anthropic server-side compaction in OpenClaw, including configuration, Anthropic request payload/header integration, streaming response handling, logging, and docs.

## Best Possible Solution

Keep this as the focused Anthropic server-side compaction tracker, then implement a maintainer-approved opt-in path in the Anthropic provider/transport/config/docs surface with current beta handling, `context_management.edits`, compaction block history round-trip, streaming delta support, `compaction` stop-reason handling, usage/logging support, and targeted regression tests.

Not applicable as a feature request rather than a bug report. The current-main verification path is high-confidence: source and docs inspection show the requested config, request payload, stream parsing, and docs support are absent.

## Solution Assessment

Unclear for an autonomous patch. The implementation area is bounded, but config/API naming, passthrough versus first-class integration, and coordination with #54041 and #65287 need maintainer judgment first.
