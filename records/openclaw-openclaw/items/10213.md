## Summary

Keep open. Current main still does not implement Anthropic-native server-side compaction: the public compaction config/schema has no Anthropic native option, the Anthropic provider wrapper only handles existing beta params, the native Anthropic request builder does not send `context_management.edits`, and the stream parser still lacks compaction block and `compaction` stop-reason handling. The linked implementation PR is closed unmerged, while the newer related issues remain open and need maintainer coordination rather than auto-closure.

## What This Changes

The issue requests opt-in Anthropic-native server-side compaction support via config/schema, Anthropic request parameters, beta headers, streamed compaction handling, logging, and docs.

## Best Possible Solution

Keep this as the focused Anthropic server-side compaction tracker and implement a maintainer-approved opt-in path through Anthropic provider params/config/schema/docs, native request construction, streamed compaction block history round-trip, `compaction` stop-reason handling, usage/logging, and targeted regression tests coordinated with #54041 and #65287.
