## Summary

Keep open. Current main still lacks the requested Anthropic-native server-side compaction path: the config/schema surface has only local compaction controls, Anthropic request construction does not add `context_management.edits`, Anthropic beta handling only covers configured betas and 1M context, and the native stream parser still has no compaction block, compaction delta, or `compaction` stop-reason support. The report remains valid but should stay in maintainer review because the public config/API shape overlaps with broader open Anthropic context-management work.

## What This Changes

Add an opt-in Anthropic server-side compaction mode that sends Anthropic compaction beta/request parameters, handles compaction response events, logs the behavior, and limits the feature to Anthropic providers.

## Best Possible Solution

Keep this as the focused Anthropic server-side compaction tracker until maintainers approve an opt-in API shape, then implement it in the bundled Anthropic provider/config/schema/docs surface with beta/header handling, `context_management.edits` injection, compaction block round-trip, streaming delta support, `compaction` stop-reason handling, usage/logging support, and targeted regression tests.
