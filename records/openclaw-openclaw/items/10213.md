## Summary

Keep open. Current main still lacks Anthropic-native server-side compaction support: there is no config/schema knob, the Anthropic request builder does not send `context_management.edits`, and the stream parser does not preserve compaction blocks or handle the `compaction` stop reason. The linked implementation PR #10214 is closed unmerged, while #54041 and #65287 are broader context-management trackers rather than a clean replacement for this focused request.

## What This Changes

Request opt-in Anthropic server-side compaction support, including configuration, Anthropic request payload/header integration, compaction response handling, and logging/docs/tests.

## Best Possible Solution

Keep this as the focused Anthropic server-side compaction tracker until maintainers approve the public API shape, then implement an opt-in Anthropic provider path with beta handling, `context_management.edits`, compaction block history round-trip, streaming delta support, `compaction` stop-reason handling, usage/logging support, docs, and targeted regression tests.

## Reproduction Assessment

Not applicable as a feature request rather than a bug report. The current-main verification path is high-confidence: source and docs inspection show the requested config, request payload, stream parsing, and docs support are absent.

## Solution Assessment

Unclear for an autonomous patch. The implementation area is bounded, but the config/API names, passthrough versus first-class integration scope, and coordination with #54041 and #65287 need maintainer judgment first.
