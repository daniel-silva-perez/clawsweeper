## Summary

Keep this issue open. Current main at 1b25dcf57a9fca5d00cf275046e20a2e3ea3730c still does not implement Anthropic server-side compaction: the typed config and strict schema have no server-side Anthropic compaction option, the Anthropic plugin wrapper only handles existing beta/header/service-tier/thinking-prefill behavior, the native Anthropic transport does not inject `context_management.edits`, and streamed content/stop handling still lacks compaction block round-trip support. The closed unmerged #10214 remains useful prior art, while open #54041 and #65287 are broader related trackers rather than a clear maintainer-designated replacement for this focused request.

## Best Possible Solution

Keep this issue open as the focused Anthropic server-side compaction tracker until maintainers choose the config/API shape and how it relates to #54041 and #65287. A complete fix should add an explicit opt-in direct-Anthropic setting or model param, validate and document it, inject the required beta plus `context_management.edits` in the relevant Anthropic request paths, preserve and replay provider compaction blocks through native and wrapper streaming, handle compaction-related stop behavior and usage metadata, and add focused tests.
