## Summary

Keep open. Current main still lacks the requested Anthropic-native server-side compaction path: no config/schema option exists, Anthropic request construction does not inject `context_management.edits`, streaming output does not preserve compaction blocks or `compaction_delta`, and `compaction` remains an unhandled stop reason. The linked implementation PR #10214 is closed unmerged, while #54041 and #65287 are broader related trackers rather than a clean replacement for this focused request.

## What This Changes

The issue requests opt-in Anthropic server-side compaction support through OpenClaw config, Anthropic beta/header handling, `context_management.edits`, stream/response handling, logging, docs, and tests scoped to Anthropic rather than proxy providers.

## Best Possible Solution

Keep this as the focused Anthropic server-side compaction tracker until maintainers approve the public API shape, then implement an opt-in Anthropic provider path with `compact-2026-01-12` beta handling, `context_management.edits`, compaction block history round-trip, streaming delta support, `compaction` stop-reason handling, usage/logging support, docs, and targeted regression tests.

## Reproduction Assessment

Not applicable as a feature request rather than a bug report. The current-main verification path is high-confidence: source and docs inspection show the requested config, request payload, stream parsing, and docs support are absent.

## Solution Assessment

Unclear for an autonomous patch. The implementation area is bounded, but config/API naming, passthrough versus first-class integration, and coordination with #54041 and #65287 need maintainer judgment first.
