## Summary

Keep open. Current main still lacks the requested Anthropic-native server-side compaction path: the public compaction config/schema has no Anthropic server-side option, Anthropic request construction does not add `context_management.edits`, beta handling only covers explicit `anthropicBeta`/`context1m`, and the native stream parser does not preserve compaction blocks or accept the `compaction` stop reason. The closed unmerged #10214 and the broader open #54041/#65287 overlap with this work, but they do not make this focused tracker obsolete.

## What This Changes

Request opt-in support for Anthropic server-side compaction, including configuration, Anthropic beta/header and `context_management.edits` request support, stream event handling, logging, docs, and tests while preserving existing local compaction defaults.

## Best Possible Solution

Keep this as the focused Anthropic server-side compaction tracker and implement a maintainer-approved opt-in path in the Anthropic provider/transport/config/docs surface, including beta/header handling, `context_management.edits`, trigger/instructions/pause options, compaction block history round-trip, `compaction` stop reason handling, usage/logging, and targeted tests coordinated with #54041 and #65287.
