## Summary

Keep open. Current main still lacks Anthropic-native server-side compaction: there is no public config/schema surface, the Anthropic wrapper only handles existing beta params, the native transport does not send `context_management.edits`, and streamed compaction blocks or `compaction` stop reasons are not handled. The closed linked PR did not merge, and the remaining work needs maintainer coordination with the broader Anthropic context-management trackers before choosing the public API shape.

## What This Changes

Request opt-in Anthropic server-side compaction support, including config, Anthropic request headers/body, streamed response handling, logging, and Anthropic-only scoping.

## Best Possible Solution

Keep this as the focused Anthropic server-side compaction tracker and implement a maintainer-approved opt-in path through Anthropic provider params/config/schema/docs, native request construction, streamed compaction block round-trip, `compaction` stop-reason handling, usage/logging, and targeted regression tests coordinated with #54041 and #65287.
