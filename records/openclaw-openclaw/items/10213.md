## Summary

Keep this issue open. Current main still lacks an Anthropic-native server-side compaction option: config/schema do not expose it, the Anthropic plugin wrapper only handles existing beta/header/service-tier behavior, the native Anthropic transport does not inject `context_management.edits`, and streamed/history content handling still has no compaction block or compaction stop-reason path. Related PR #10214 is closed unmerged; #54041 and #65287 are relevant broader trackers but are not a clear canonical replacement for this focused request.

## Best Possible Solution

Keep this issue open as the focused Anthropic server-side compaction tracker until a maintainer chooses the exact config/API shape and relationship to #54041/#65287. A complete fix should add an explicit opt-in direct-Anthropic setting, validate and document it, inject the required Anthropic beta and `context_management.edits`, preserve provider compaction blocks through streaming and transcript replay, handle the `compaction` stop reason/pause flow, and add focused tests for both the Anthropic plugin wrapper and native Anthropic transport.
