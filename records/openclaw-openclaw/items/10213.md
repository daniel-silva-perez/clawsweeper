## Summary

Keep this issue open. Current main still lacks Anthropic-native server-side compaction: there is no opt-in config/schema field, the Anthropic plugin wrapper only handles existing beta/service-tier/fast-mode behavior, the native Anthropic transport does not inject `context_management.edits`, and the transport content model does not round-trip provider compaction blocks. The linked implementation PR #10214 is closed unmerged; #54041 and #65287 are related broader trackers, but the provided discussion does not mark either as the canonical replacement for this focused issue.

## Best Possible Solution

Keep this issue open as the focused Anthropic server-side compaction tracker. A maintainer-quality fix should add an explicit opt-in config/model parameter, validate and document it, inject the Anthropic beta plus `context_management.edits` only for eligible direct Anthropic requests, preserve provider compaction blocks through streamed output and replay history, handle pause/stop/logging behavior, and add focused tests for both the bundled Anthropic plugin wrapper and native Anthropic transport. Treat #54041 and #65287 as related broader design work unless maintainers explicitly mark one canonical.
