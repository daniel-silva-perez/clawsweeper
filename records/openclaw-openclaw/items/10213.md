## Summary

Keep this issue open. Current main still has no Anthropic server-side compaction config, schema support, request injection, or streaming round-trip support for Anthropic compaction blocks. The closed unmerged #10214 remains useful prior art, while #54041 and #65287 are broader related trackers rather than a maintainer-designated replacement for this focused request.

## Best Possible Solution

Keep this issue open as the focused Anthropic server-side compaction tracker. The best fix should add an explicit opt-in direct-Anthropic setting or model param, validate and document it, inject the required beta plus `context_management.edits` in the relevant Anthropic request paths, preserve provider compaction blocks through streaming/history replay, handle the compaction stop reason and usage metadata, and coordinate the chosen config shape with #54041 and #65287.
