## Summary

Keep open. Current main still does not implement this PR’s intended sessions_history fix: the tool-output sanitizer retains `thinking` blocks after truncating/redacting them, has no `redacted_thinking` omission path, and the current regression test still asserts the old retained-block behavior. The PR remains a plausible implementation candidate for the closed linked bug report, but it should be updated before landing because the review context flags functional and safety blockers.

## Best Possible Solution

Keep this PR open as an implementation candidate, but require an update against current main. The best fix should strip `thinking` and `redacted_thinking` from `sessions_history` output, preserve assistant turn shape when all blocks are omitted, keep replay-preservation controls internal, decide whether `sessions_list` previews need the same sanitizer/backfill behavior, and add focused regression tests for direct tool output and any preview behavior changed.
