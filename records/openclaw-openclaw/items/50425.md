## Summary

Keep open. Current main still does not implement this PR's central behavior: `sessions_history` truncates and redacts `type: "thinking"` blocks but returns them in the tool payload, has no `redacted_thinking` omission path for this tool surface, and existing tests still expect a returned thinking block. The PR remains a plausible implementation candidate, but it needs updates before landing because review context shows unresolved functional and security-sensitive concerns.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is to update it against current main so `sessions_history` strips `thinking` and `redacted_thinking` from returned assistant messages, preserves a valid assistant turn shape when every content block is omitted, keeps any replay-preservation behavior internal rather than model-callable, decides and tests whether `sessions_list` previews should share the same sanitized/backfilled behavior, and addresses the existing review comments before merge.
