## Summary

Keep open. Current main still does not implement this PR’s intended `sessions_history` output fix: the tool sanitizes/truncates `thinking` blocks but keeps them in the returned payload, has no `redacted_thinking` omission path, and current tests still assert the old retained-block behavior. The PR remains a plausible implementation candidate, but the supplied review context also shows unresolved landing risks that need author or maintainer handling first.

## Best Possible Solution

Keep this PR open as an implementation candidate, but require an update against current main. The best fix should strip `thinking` and `redacted_thinking` from `sessions_history` tool output, preserve assistant turn shape when all blocks are omitted, keep any provider-replay preservation out of the public model-callable tool schema, resolve the reported syntax/review issues, and add focused regression tests for direct `sessions_history` output plus any `sessions_list` preview behavior the PR changes.
