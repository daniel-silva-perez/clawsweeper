## Summary

Keep open. Current main still does not implement this PR’s central sessions_history behavior: the tool sanitizer keeps `thinking` blocks in returned assistant content after truncating/redacting them, has no `redacted_thinking` omission path, and the current regression test still asserts that a returned `thinking` block exists. The PR remains a plausible implementation candidate for linked issue #50415, but it needs update before landing because its own review context includes functional and safety blockers.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is to update it against current main so `sessions_history` strips `thinking` and `redacted_thinking` from returned assistant messages, preserves valid assistant turn shape when all blocks are omitted, keeps replay-preservation controls out of the public tool schema, decides whether `sessions_list` previews need the same sanitized/backfilled behavior, and adds focused regression coverage for the direct tool output and any preview behavior changed.
