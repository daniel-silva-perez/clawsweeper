## Summary

Keep open. Current main does not implement namespaced `/models` grouping, and this PR is useful in scope, but the PR head still lets channel-specific interactive model lists return before the new grouped/scoped text path runs, so Telegram can bypass the grouping and lose namespace scope on pagination.

## What This Changes

The PR adds `/models <provider>/<group>` parsing, grouped text output for namespaced model catalogs, scoped model listings, and colocated command tests.

## Best Possible Solution

Keep the namespace grouping feature, but either make the channel list adapter and Telegram callback data carry the namespace scope end to end, or bypass interactive list rendering for grouped/scoped views. Add Telegram coverage for grouped catalogs and scoped pagination, and update https://docs.openclaw.ai/tools/slash-commands for the new drilldown syntax.

## Reproduction Assessment

Yes. A high-confidence static repro is Telegram `/models litellm` on the PR branch with multiple namespaced groups: the command can return provider-only channel data before grouped text is built. A scoped list with more than one page also loses scope because Telegram pagination callbacks encode only provider and page.

## Solution Assessment

No. The text fallback implementation is close, but the maintainable fix needs scope-aware interactive channel contracts and Telegram callback handling, or an intentional text-only path for grouped/scoped model lists.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Thread namespace scope through interactive model lists:** `src/auto-reply/reply/commands-models.ts:608-621`
  - body: The new grouping block runs after `buildModelsListChannelData` can return, while that adapter receives only `provider` and Telegram callbacks encode only `mdl_list_<provider>_<page>`. On Telegram, `/models litellm` can still show the old flat keyboard, and paginating a scoped view such as `litellm/deepseek` reloads the full provider list. Please pass the namespace through the adapter/callback path or skip interactive rendering for grouped/scoped views.
