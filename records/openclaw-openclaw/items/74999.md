## Summary

Keep open. Current main does not implement namespaced `/models` grouping, and the PR branch contains useful work but is not merge-ready because the new scoped model list still returns through the interactive channel adapter before grouped text is built, which breaks the feature on Telegram and loses scope during pagination.

## What This Changes

The PR adds parsing for `/models <provider>/<group>`, grouped text output for namespaced provider catalogs, scoped listings, and tests for aggregator and local-model behavior.

## Best Possible Solution

Keep the namespace grouping feature, but make the channel list adapter and Telegram callback data scope-aware, or intentionally bypass interactive list rendering for grouped and scoped views. Then add Telegram coverage for top-level grouped catalogs and scoped pagination, and update the slash-command docs for the new drilldown syntax.

## Reproduction Assessment

Yes. A high-confidence static repro is Telegram `/models litellm` on the PR branch with multiple namespaced groups: the command can return provider-only channel data before grouped text is built. A scoped list with more than one page also reproduces scope loss because Telegram pagination callbacks encode only the provider.

## Solution Assessment

No. The text fallback implementation is close, but the maintainable fix needs to thread namespace scope through the interactive channel contract and Telegram callback handling, or avoid that renderer for grouped/scoped cases.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Thread namespace scope through interactive model lists:** `src/auto-reply/reply/commands-models.ts:577-585`
  - body: The PR computes scoped `models`, but still calls `buildModelsListChannelData` with only `provider` before grouped text is built. Telegram implements this hook, so `/models litellm` can show the old flat keyboard, and scoped pagination loses the group because callbacks remain `mdl_list_<provider>_<page>`. Please pass the namespace scope through the adapter/callback path or skip interactive rendering for grouped/scoped views.
  - confidence: 0.89
