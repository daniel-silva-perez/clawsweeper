## Summary

Keep this PR open: current main still does not implement namespaced `/models` grouping, but the proposed branch still has a concrete Telegram/channel regression where interactive rendering can bypass the grouped text and scoped pagination cannot retain the namespace.

## What This Changes

The PR adds `/models <provider>/<group>` parsing, grouped text output for namespaced aggregator catalogs, scoped model listings, and colocated auto-reply command tests.

## Best Possible Solution

Land the namespace grouping feature only after grouped and scoped `/models` views either preserve namespace scope through the channel adapter and Telegram callbacks or intentionally bypass interactive rendering for those views, with focused Telegram coverage and updated slash-command docs.

## Reproduction Assessment

Yes. A high-confidence static reproduction is Telegram `/models litellm` on the PR branch with multiple namespaced groups: the command can return provider-only interactive channel data before grouped text is built, and scoped pagination callbacks encode only provider/page.

## Solution Assessment

No. The text fallback implementation is close, but the maintainable fix needs namespace-aware channel rendering/callbacks, or an explicit text-only path for grouped and scoped listings.
