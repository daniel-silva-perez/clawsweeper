## Summary

Keep this PR open: the requested namespace grouping is not on current main, and the PR is useful but still has a concrete P2 correctness issue in the interactive `/models` path that should be fixed before merge.

## What This Changes

The PR adds `/models <provider>/<group>` parsing, grouped text output for namespace-heavy provider catalogs, scoped list tests, and keeps `/model <provider/model>` plus `/models <provider> all` behavior intact.

## Best Possible Solution

Keep the feature direction, but land it only after namespace scope is represented in the channel adapter and Telegram callback data/handler, or after grouped/scoped views intentionally bypass interactive list rendering. Add Telegram coverage for top-level grouped catalogs and scoped pagination, and update the public slash-command docs.

## Reproduction Assessment

Yes. A high-confidence static reproduction is Telegram `/models litellm` on the PR branch with multiple namespaced groups: the command can return provider-only channel data before grouped text is built, and scoped pagination loses `modelPrefix` because callbacks encode only provider plus page.

## Solution Assessment

No. The text fallback implementation is close, but the proposed patch is not yet the best maintainable solution because interactive channel rendering needs to preserve namespace scope or be bypassed for grouped/scoped views.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Thread namespace scope through interactive model lists:** `src/auto-reply/reply/commands-models.ts:577-585`
  - body: The PR computes scoped `models`, but still calls `buildModelsListChannelData` with only `provider` before grouped text is built. Telegram implements this hook, so `/models litellm` can show the old flat keyboard, and scoped pagination loses the group because callbacks remain `mdl_list_<provider>_<page>`. Please pass the namespace scope through the adapter/callback path or skip interactive rendering for grouped/scoped views.
  - confidence: 0.89
