## Summary

Keep this PR open. It targets a real Synology Chat compatibility bug: current `main` and `v2026.4.27` still reject `HEAD` probes and ACK accepted webhook POSTs with `204`, while the PR changes that behavior to the reported Synology-compatible `200` responses. The remaining decision is maintainer branch selection and rebase because the same Synology ACK work overlaps the broader open #53522 and the paired #53439 issue remains open.

## What This Changes

The PR changes the Synology Chat webhook handler and related tests to answer `HEAD` availability probes with `200` and return a `200 {"success":true}` ACK instead of `204` for accepted POST webhooks.

## Best Possible Solution

Land one current-main Synology Chat change that accepts Synology `HEAD` availability probes, returns the exact `200 {"success":true}` ACK for accepted POSTs, preserves POST token validation, allowlist, rate-limit, and pre-auth guard behavior, and updates all current regression coverage for `HEAD`, ACK status, and ACK body. After that lands, #53439 can close through the merged fixing PR.
