## Summary

Keep this PR open. Current main has partially superseded the export-template and Google Vertex ADC parts, but it still does not implement the full intended change: current-run HEIC/HEIF prompt images can still reach the model path unchanged, and `config set` still has no targeted `gateway.nodes.denyCommands` warning. The PR also has unresolved review findings on security-sensitive auth, attachment-size validation, and node command validation behavior.

## Best Possible Solution

Keep this PR open, but require a rebase and narrower update before merge: drop or adapt the export/auth pieces already changed on main, implement HEIC/HEIF normalization in `parseMessageWithAttachments` with converted-size validation before `parsed.images` is returned, add `config set` warnings for `gateway.nodes.denyCommands` using a full node command catalog, and resolve the existing review comments.

## Risks / Open Questions

- Provider auth and node command policy are security-sensitive surfaces; the PR review comments identify a Google Vertex ADC runtime breakage risk if the branch returns oauth mode without a usable runtime credential.
- The PR's HEIC/HEIF conversion path needs post-conversion size validation to match the existing media input guard and avoid over-limit model payloads after JPEG expansion.
- Using the effective node allowlist as the denyCommands validation source can warn on valid dangerous commands that operators may intentionally deny, making hardening guidance misleading.
