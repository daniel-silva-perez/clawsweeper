## Summary

Keep this PR open. Current main has already moved the export-template placeholder handling and google-vertex ADC marker behavior away from the exact PR shape, but it still does not implement the full intended change: current-run chat attachments can still return inline HEIC/HEIF payloads unchanged, and `openclaw config set gateway.nodes.denyCommands ...` still has no targeted warning path.

## Best Possible Solution

Keep this PR open, but ask for a rebase and narrower update before merge: drop or adapt the export/auth pieces that current main already changed, implement HEIC/HEIF normalization inside `parseMessageWithAttachments` before `parsed.images` is returned with post-conversion size checks, and add `gateway.nodes.denyCommands` config-set warnings against a real node command catalog. The open review findings should be addressed before maintainers consider landing it.
