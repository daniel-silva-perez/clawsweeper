## Summary

Keep this PR open. Current main has partially superseded the export-template and google-vertex sentinel shape, but it still does not implement the full intended change: current-run chat attachments can still hand inline HEIC/HEIF payloads to the model unchanged, and config set still has no targeted gateway.nodes.denyCommands warning path. The PR is also not merge-ready as-is because review comments identify security-sensitive correctness issues in the auth, HEIC, and denyCommands changes.

## Best Possible Solution

Keep this PR open, but ask for a rebase and narrower update before merge: drop or adapt the export/auth pieces already changed on main, implement HEIC/HEIF normalization in parseMessageWithAttachments with converted-size validation before parsed.images is returned, add config set warnings for gateway.nodes.denyCommands using a full node command catalog, and resolve the open review comments.

## Risks / Open Questions

- Provider auth and node command policy are security-sensitive surfaces; the auth and denyCommands parts should not be merged without resolving the existing review findings.
- The PR's HEIC/HEIF conversion approach needs a post-conversion size check to match the existing media input guard and avoid over-limit model payloads after JPEG expansion.
- Using the effective allowlist as the denyCommands validation source can warn on valid dangerous commands that operators may intentionally deny, making hardening guidance misleading.
- The provided PR file list does not show workflow, dependency, lockfile, install/build/release, publishing, generated, or vendor changes, so the security concern is behavioral rather than supply-chain.
