## Summary

Keep open. Current main has a zh-CN Control UI locale and some Settings navigation is translated, but the Settings surface still renders many English literals and schema-derived English titles/descriptions, so the requested Chinese Settings localization is not implemented or superseded.

## What This Changes

Translate the Chinese Control UI Settings section and its configuration labels/descriptions into Chinese.

## Best Possible Solution

Keep the item open and define a scoped Settings i18n pass: move visible Settings and Quick Settings strings into the English locale source, regenerate non-English bundles through the existing pipeline, add coverage that prevents new hardcoded Settings copy, and decide how schema title/description metadata should be localized for core and plugin-provided config fields.

## Reproduction Assessment

Yes. Set the Control UI locale to zh-CN and open Settings/Advanced Settings; current main statically renders English literals from the Settings view files and English schema title/description metadata from the generated config schema.

## Solution Assessment

Unclear. The existing Control UI i18n pipeline is the right path for visible UI strings, but full field-label localization needs a maintainer decision on whether to translate schema metadata client-side, add locale-aware schema responses, or accept a staged rollout.
