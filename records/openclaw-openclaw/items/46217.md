## Summary

Keep open. Current main supports zh-CN for the Control UI shell, but the Settings/Config form still renders section names, field labels, help text, and several controls from English-only metadata or hard-coded English strings. The related local title-search reports concern duplicate Settings headers, not Chinese localization, so they do not supersede this request.

## Best Possible Solution

Keep this issue open and implement a proper localized Settings/Config form path: add locale-aware keys or metadata for section labels, schema titles/help, placeholders, buttons, validation text, and plugin/channel config hints, with zh-CN generated through the existing Control UI i18n pipeline rather than hand-editing generated locale bundles.
