## Summary

Keep this issue open. Current main still rejects provider-only `models.providers.openai` entries because `models` is required in the Zod schema, generated JSON schema, and TypeScript config contract. The memory embedding runtime would use `models.providers.<id>.baseUrl` once the config is accepted, and live GitHub state shows #39766 is still open/unmerged as the active implementation candidate with closing syntax for this issue.

## Best Possible Solution

Keep this issue open until #39766 or an equivalent focused fix lands. The fix should make `ModelProviderSchema.models` optional with a default empty array, align `ModelProviderConfig` and generated schema/help artifacts, add a regression test for `models.providers.openai` with only `baseUrl`/`apiKey`/`api`, and keep or add an embedding test proving provider `baseUrl` is used when `memorySearch.remote.baseUrl` is absent.
