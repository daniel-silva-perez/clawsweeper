## Summary

Keep this issue open. Current main still requires `models.providers.<id>.models` in the Zod schema, generated schema, and TypeScript config contract, so the reporter's provider-only OpenAI baseUrl config is still rejected before memory embeddings can use it. There is also an open implementation candidate, #39766, that explicitly targets this issue and is not merged.

## Best Possible Solution

Keep this issue open until #39766 or an equivalent focused fix lands. The fix should allow provider-only `models.providers.<id>` entries by defaulting `models` to an empty array at validation time, align the generated schema/help and TypeScript config contract, and add regression coverage proving the custom OpenAI provider baseUrl is accepted and used by memory embeddings when `memorySearch.remote.baseUrl` is absent.
