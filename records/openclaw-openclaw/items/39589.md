## Summary

Current main still rejects provider-only `models.providers.openai` entries because `models` remains required in the Zod schema, generated schema, and TypeScript config contract. The embedding runtime would honor `models.providers.<id>.baseUrl` once validation accepts the config, and there is still an open unmerged implementation candidate, #39766, that targets this issue with GitHub closing syntax.

## Best Possible Solution

Keep this issue open until #39766 or an equivalent focused fix lands. The fix should allow provider-only `models.providers.<id>` entries by defaulting `models` to an empty array at validation time, regenerate and align the schema/config contract, and add regression coverage proving the custom OpenAI provider baseUrl is accepted and used by memory embeddings when `memorySearch.remote.baseUrl` is absent.
