## Summary

Keep this issue open. The report is recent, concrete, and environment-specific: outbound works, inbound never reaches logs/status in a Docker to k3s OpenShell-style deployment. Current main has important WhatsApp reconnect/append fixes, but those only prove OpenClaw can process Baileys upserts after they are emitted; they do not prove the nested k3s inbound/WebSocket path is fixed. The related OpenShell deployment docs PR #54959 is still open and references this issue as sandbox networking context, so it is not shipped evidence for closure.

## Best Possible Solution

Keep this issue open and handle it as a WhatsApp/OpenShell deployment triage item. The best path is to reproduce or rule out Baileys WebSocket/event delivery in nested k3s, then either land clear OpenShell deployment/networking guidance via #54959 or add a WhatsApp plugin/runtime fix if current APIs need proxy, socket, or health diagnostics support for this environment.
