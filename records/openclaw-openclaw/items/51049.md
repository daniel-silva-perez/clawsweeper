## Summary

Keep this issue open. Current main has WhatsApp reconnect and recent `append` upsert handling, but the reported nested Docker to k3s failure is still plausible because inbound delivery depends on Baileys emitting `messages.upsert` from the WhatsApp Web socket. The discussion already narrowed this toward container networking/event delivery, and the related OpenShell deployment docs PR #54959 is still open rather than shipped.

## Best Possible Solution

Keep this issue open as a WhatsApp/OpenShell deployment triage item. Reproduce or rule out Baileys WebSocket/event delivery inside the nested k3s pod, then either land clear OpenShell/k3s networking guidance via #54959 or add WhatsApp plugin diagnostics/runtime support if current APIs need proxy, socket, or health visibility for this environment.
