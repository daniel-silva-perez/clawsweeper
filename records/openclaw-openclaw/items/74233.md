## Summary

Keep open. Current main still applies context-engine assembly, then feeds the larger pre-assembly snapshot into the hard overflow precheck, and the existing helper test still locks in that larger-unwindowed estimate. The issue is also paired with open closing PR #74255, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

Make context-engine assembled messages prompt-authoritative for overflow admission, with raw pre-assembly history handled as opt-in hard pressure or separate maintenance telemetry.

## Best Possible Solution

Land or adapt #74255 so assembled context is the default prompt-admission authority, raw pre-assembly history only participates in hard overflow gating through an explicit generic opt-in, and the contract docs plus precheck regressions are updated together.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: return a small assembled message set from a context engine while keeping a larger pre-assembly transcript, then observe the precheck helper choose the larger unwindowed estimate.

## Solution Assessment

Yes, the requested direction is the narrow maintainable fix. It should be implemented as a generic context-engine contract/policy seam, not as a lossless-claw-specific bypass or by removing overflow protection entirely.
