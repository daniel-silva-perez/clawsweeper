## Summary

Keep this open. Current main still contains the reported source path and an explicit unit test that locks in raw pre-assembly history as the larger prompt-admission estimate, while open PR #74255 is the paired implementation candidate and should be reviewed before the issue is closed.

## What This Changes

The issue asks OpenClaw to stop hard-failing context-engine prompt admission against larger pre-assembly history when the context engine has already assembled a budget-fitting prompt.

## Best Possible Solution

Review and land a narrow contract-aware fix like #74255: make assembled context-engine output prompt-authoritative by default or via an explicit capability, keep raw transcript pressure as a separate maintenance/debt signal, and update the locking tests plus context-engine docs.

## Reproduction Assessment

Yes. The source-level path is reproducible by the existing unit shape in `preemptive-compaction.test.ts`: small assembled messages plus larger unwindowed messages currently triggers compaction against the larger estimate. The live LCM session logs are not needed to prove the host-side admission bug.

## Solution Assessment

No, current main is not the best solution because it conflates prompt admission with raw-history pressure after context-engine assembly. The safer direction is the open PR's contract split, reviewed by maintainers for compatibility with engines that intentionally want pre-assembly overflow gating.
