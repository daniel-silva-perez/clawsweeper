## Summary

Current main still contains the reported behavior, and an open closing PR already tracks the intended host-side fix, so this should remain open until that PR or an equivalent implementation lands.

## What This Changes

The issue reports that context-engine assembled prompts can be bypassed because the embedded runner’s overflow precheck still hard-gates on larger pre-assembly history.

## Best Possible Solution

Land or adapt #74255 so prompt admission checks the assembled prompt by default, keeps raw pre-assembly history as an explicit opt-in or maintenance/debt signal, and updates the locking unit test plus context-engine docs to match the final contract.

## Reproduction Assessment

Yes. The current helper test reproduces the decision shape by pairing a small assembled message window with a larger unwindowed history, and current source shows the embedded runner still passes that unwindowed snapshot into the hard precheck.

## Solution Assessment

No, current main is not the best solution because it still conflates prompt-admission size with raw-history pressure. The best path is a generic context-engine contract change such as the one proposed in #74255, reviewed against the documented safeguard semantics.
