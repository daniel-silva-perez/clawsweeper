## Summary

Keep open: current main still has the reported path where context-engine assembly updates the active messages, but the overflow precheck also receives the pre-assembly snapshot and can promote the larger estimate. The issue is also paired with open closing PR #74255, so it should not be closed before that PR is reviewed and either merged or rejected.

## What This Changes

The issue asks OpenClaw to make context-engine assembled messages authoritative for prompt overflow admission, while treating raw pre-assembly history as a separate maintenance or debt signal.

## Best Possible Solution

Land a generic prompt-authority contract that lets assembled context be the default admission surface, preserves explicit raw-history overflow gating only when an engine opts in, and updates the locking regression plus context-engine docs. PR #74255 is the current candidate implementation for that end state.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: the runtime passes a pre-assembly snapshot into precheck after assembly, and the focused unit test asserts that a large unwindowed history overrides a small assembled window.

## Solution Assessment

No for current main; it still treats raw pre-assembly history as a hard prompt-admission gate after context assembly. The linked PR's assembled-by-default, raw-history-opt-in direction is a plausible narrow fix, but it needs maintainer review because it affects third-party context-engine semantics.
