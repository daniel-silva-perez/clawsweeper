## What This Changes

The issue reports a packaged plugin-loader/runtime-deps hot loop where repeated bundled runtime mirror preparation, manifest parsing, and runtime-deps staging starve the Gateway event loop.

## Best Possible Solution

Keep this as the canonical runtime-deps/plugin-loader starvation tracker until maintainers retest an affected packaged host on current main or the next release. If it still reproduces, add timing/cache-hit instrumentation around plugin registry loading, runtime-deps materialization, dist-mirror locks, chat-completions preparation, and channel startup, then close or split against the smallest proven fixes.
