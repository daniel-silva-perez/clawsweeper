## Summary

Keep open. Current main has additional runtime-deps convergence hardening after the last sweep, especially e311ffdc for the repeated reinstall loop, but the broad gateway starvation report still is not proven fixed: the request-time plugin-tool path, exact loader cache scoping, runtime-deps locks, sync install fallback, and runtime mirror preparation paths remain reachable, while related narrower items #74860 and #75032 still track remaining work.

## What This Changes

The issue asks OpenClaw to stop a gateway/plugin-loader hot loop where bundled runtime-deps mirror preparation and JSON5 manifest parsing repeatedly saturate CPU, starve the event loop, and break Telegram, WS RPC, and OpenAI-compatible request paths.

## Best Possible Solution

Keep this as the canonical runtime-deps/plugin-loader starvation tracker until maintainers retest an affected packaged host on current main or the next package with CPU, RSS, event-loop delay, runtime-deps activity, registry cache hits, and mirror timing enabled. If it still reproduces, land the smallest proven fix in the narrowed owner path while leaving #74860 and #75032 as focused follow-ups for their split-out surfaces.

## Reproduction Assessment

Yes for the released affected range: the thread gives concrete Linux, Docker, and Raspberry Pi setups with plugin counts, profiles, logs, and v2026.4.24-v2026.4.29 behavior. It is unclear for exact current main at 359d8712 because this review did not run the affected host and current main has e311ffdc after the latest field reports.

## Solution Assessment

Unclear. Current main contains sensible mitigations and one new convergence fix, but the broad issue is not proven solved; maintainer-led affected-host retest and instrumentation is safer than closing or assigning one automated branch to the whole cluster.
