## Summary

Keep open. Current main still has the length-only prompt-submission guard, so this PR is not obsolete. The linked empty-prompt race remains plausible because blank replay user content can satisfy the guard before replay normalization drops it. The PR is a narrow candidate direction, but it still needs maintainer review/changes: the patch reassigns `promptSubmission` even though current main declares it as `const`, adds inline `any` content inspection instead of a typed shared helper, and has no targeted regression coverage for blank replay content. The security pass found no supply-chain-sensitive changes because the diff only touches the embedded runner TypeScript path.

## Best Possible Solution

Keep this PR open and request changes. The best fix is a typed content-aware prompt-submission helper used by `runEmbeddedAttempt`, with focused helper tests for blank string replay content, blank text-block replay content, normal non-empty replay content, prompt text, and image-only submissions. The fix should preserve existing `/new` and `/reset` startup behavior, avoid unrelated provider/plugin/CI/dependency changes, and either revise this PR or let a maintainer deliberately supersede it with a narrow fix for #73926.
