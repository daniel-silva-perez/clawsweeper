## Summary

Keep open. Current main still lacks the requested read-only `routing.list` Gateway API and the selected-agent Channel Bindings display in the Agents Channels panel. Existing config/CLI/docs prove route and ACP bindings are real OpenClaw data, but they are not exposed through the requested Gateway UI surface. There is also a same-author open fork PR, `Billmvp73/openclaw#1`, that is an implementation candidate, so the best maintainer path is to review, port, land, or supersede that work rather than close this issue.

## Best Possible Solution

Keep this issue open. Maintainers should review and either land or port `Billmvp73/openclaw#1`, or close that candidate with rationale and implement an equivalent current-pattern solution: add an additive read-only Gateway method for route and ACP bindings, wire it into protocol schemas/types and operator READ scope, load it in the Control UI Agents Channels panel, render only the selected agent’s bindings, and add focused Gateway/UI coverage.
