## Summary

Keep this PR open. Current main and the latest release still have the unsafe OpenAI Codex legacy-alias canonicalizer, and this PR is a narrow defensive fix for that path; the main remaining gap is regression coverage for malformed resolved-model metadata.

## What This Changes

The PR adds runtime `typeof` guards before trimming/lowercasing `model.id` and `model.name` in the OpenAI Codex legacy alias canonicalizer.

## Best Possible Solution

Add a focused regression test in `src/agents/pi-embedded-runner/model.test.ts` for malformed OpenAI Codex resolved-model metadata, then land this guard or an equivalent maintainer patch.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by feeding an `openai-codex` resolved model with undefined or non-string `id`/`name` through the resolved-model path that reaches `canonicalizeLegacyResolvedModel`; current main will call string methods before guarding.

## Solution Assessment

Yes, with a test added. The PR’s guard is the narrowest maintainable fix for the immediate crash because it preserves normal alias canonicalization while avoiding string-method calls on malformed runtime metadata.
