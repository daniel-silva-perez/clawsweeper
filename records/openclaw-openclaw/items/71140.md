## Summary

Keep open. Current main still lacks the reaction-directive path for normal Telegram outbound replies: media/GIF handling is already present, but [[react...]] / [[react_to_current:...]] is not parsed into structured data, reaction-only replies are rejected, and the Telegram outbound adapter ignores reaction channel data. Open draft PR #71141 is already linked with closing syntax, so this issue should stay open until that PR is reviewed and merged or replaced.

## What This Changes

The issue requests shared outbound normalization and Telegram delivery support so reaction directives and media directives become native Telegram reactions/GIF/media instead of leaked or dropped text.

## Best Possible Solution

Review, land, or replace PR #71141 so the shipped end state is shared directive parsing for reactions, preserved structured Telegram reaction data, reaction-only delivery support, and Telegram adapter execution through the existing native reaction primitive with focused tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: [[react_to_current:...]] is not parsed by the inline directive parser, and a reaction-only Telegram reply still reaches the no-text/no-media rejection path.

## Solution Assessment

Yes. The requested shared-parser plus Telegram delivery/adapter wiring is the narrow maintainable direction; a Telegram-only string special case would leave outbound normalization inconsistent across reply paths.
