## Summary

Keep this PR open. Current main still does not implement channel icons in Sessions rows or Channels card titles, and the PR remains a plausible focused Control UI contribution, but its head still has a functional blocker and should not be treated as merge-ready without fixes.

## Best Possible Solution

Keep this PR open for contributor or maintainer follow-up. The best path is to rebase onto current main, fix direct/dm peer extraction, use a shared channel title/icon helper backed by existing `channelMeta`/`channelSystemImages` where possible, wire `channel` through the browser session type and session event field list, then add focused UI tests for Sessions direct/dm rendering and channel card titles.
