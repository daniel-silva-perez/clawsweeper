## Summary

Keep this PR open. Current main still does not render channel icons in the Sessions list or channel card titles, so the PR is not obsolete or implemented. It remains useful, but it should not merge as-is because the submitted diff has a real `dm`-key peer extraction bug, duplicates channel icon mapping, and needs to align with the current browser session row contract.

## Best Possible Solution

Keep this PR open for contributor or maintainer follow-up. The best path is to rebase onto current main, implement one shared channel title/icon helper that can use existing channel metadata where possible, render icons in generic and channel-specific cards, add a Sessions metadata row that handles both `direct` and `dm` session-key shapes, update the browser session row contract if `row.channel` remains part of the design, and add focused UI tests for channel-card titles plus Sessions direct/dm rendering.
