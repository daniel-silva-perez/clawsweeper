## Summary

Keep this PR open. Current main still awaits lane tasks directly without a task-level timeout, the public enqueue API has no `taskTimeoutMs` option, and the paired same-author bug #48488 remains open with this PR as its implementation candidate. The branch is still relevant, but it should be reworked before landing to honor caller timeout budgets and address zombie-task overlap/cancellation semantics.

## Best Possible Solution

Keep this PR open for maintainer review, but require a rework before landing: implement queue-level recovery or an equivalent health mechanism in `src/process/command-queue.ts`, derive or pass task timeout budgets from callers instead of imposing a short global default, document and test zombie-task/cancellation behavior, and retain regression tests for lane unblock, reset generation handling, disabled/custom timeouts, and diagnostics. Once a corrected implementation lands, close #48488 as implemented.
