## Summary

Keep this PR open. Current main still does not implement the intended gateway default-agent model behavior: `sessions.list.defaults` is computed from the global configured fallback instead of the effective default or filtered agent, so the linked Control UI default-label bug path remains plausible. The PR is still a useful candidate, but its latest provided diff also bundles an unrelated hook path canonicalization change and still needs the defaults resolver to derive model, contextTokens, and thinking from the same effective agent source.

## Best Possible Solution

Keep the PR open for maintainer follow-up. Rebase it on current main, narrow it to the gateway session-defaults bug, make `getSessionDefaults` accept and use the normalized effective agent id, derive model, contextTokens, and thinking defaults from the same selected/default agent source, pass the filtered `agentId` through from `listSessionsFromStore`, and keep focused regressions for both default-agent and filtered-agent defaults. Split the hook path canonicalization into a separate PR if maintainers want that behavior.
