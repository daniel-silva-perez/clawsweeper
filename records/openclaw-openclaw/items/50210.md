## Summary

Keep this PR open. Current main still does not implement the intended `sessions.list.defaults` behavior: gateway defaults are resolved from global configured model defaults, not from the effective default or filtered agent. The PR remains useful, but it needs a current-main rebase, preservation of newer `modelCatalog`/thinking-default behavior, selected-agent context-token handling, and the unrelated hook-loader change split out.

## Best Possible Solution

Keep this PR open for maintainer follow-up. Rebase on current main, narrow the diff to the gateway session-defaults bug, make `getSessionDefaults` accept the normalized selected/default agent id while preserving `modelCatalog`, derive model/contextTokens/thinking defaults from the same effective agent source, pass filtered `agentId` through from `listSessionsFromStore`, and keep focused regressions for default-agent and filtered-agent defaults. Split the hook path canonicalization into a separate PR if maintainers want that behavior.
