## Summary

Keep this PR open. Current main still computes `sessions.list.defaults` from the global configured fallback, not from the effective default or requested agent model, so the linked Control UI bug path is not implemented on main. The PR remains useful, but its latest head also bundles an unrelated hook-loader path change and still needs the defaults resolver to align model, contextTokens, and thinking from the same effective agent source.

## Best Possible Solution

Keep the PR open for maintainer follow-up. Rebase it on current main, narrow it to the gateway session-defaults bug, make `getSessionDefaults` resolve model, contextTokens, and thinking defaults from the same effective default or requested agent, pass the normalized `agentId` from `listSessionsFromStore`, and keep focused regressions for default-agent and filtered-agent defaults. Move the hook path canonicalization to a separate PR if maintainers want that behavior.
