## Summary

Keep open. Current main still reports `sessions.list.defaults` from the global configured fallback rather than the effective default/requested agent model, and the linked user bug was already consolidated onto this PR. The PR is useful but still needs maintainer cleanup before merge because the latest head includes an unrelated hook-loader path change and has unresolved feedback about `contextTokens` using a different default source than the resolved model.

## Best Possible Solution

Keep the PR open for maintainer follow-up, rebase it on current main, and narrow it to the gateway session-defaults fix. `getSessionDefaults` should resolve model, contextTokens, and thinking defaults from the same effective default/requested agent; `listSessionsFromStore` should pass the normalized `agentId` filter into that resolver; focused regressions should cover default-agent and filtered-agent defaults. Move the hook path canonicalization into a separate PR if maintainers still want that behavior.

## Risks / Open Questions
