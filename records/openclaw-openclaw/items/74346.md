## Summary

Keep open. Current main still treats any live recorded PID as an active bundled runtime-deps lock owner, which leaves the Docker PID-reuse hang plausible, and the issue is now paired with open PR #74361 using closing syntax. The right cleanup path is to review that PR or an equivalent narrow fix before closing this issue.

## What This Changes

The issue asks OpenClaw to make bundled plugin runtime-deps lock staleness detection safe when Docker container restarts reuse the same numeric PID.

## Best Possible Solution

Review and land #74361 or an equivalent narrow runtime-deps lock hardening change that records process starttime, treats live-PID/starttime mismatches as stale, preserves legacy lock behavior when evidence is missing, and adds focused tests for Docker PID reuse plus active-install protection.
