## Summary

Keep open. Current main has related global-install hardening, especially staged npm verification and swap rollback before post-update work, but it still does not implement this PR's intended previous-version reinstall when the post-update doctor step fails after a package update. The submitted patch is still plausible core update-safety work, but it needs to be rebased onto the shared package update helper and should address the existing guard/security concern before merge.

## Best Possible Solution

Keep this PR open for contributor or maintainer revision. The best implementation is to move rollback support into the shared package update flow, carry the known `beforeVersion`, attempt `packageName@beforeVersion` rollback only after a successful live replacement/direct install followed by post-update doctor failure, rerun doctor when the restored entrypoint exists, preserve the overall error result, and add focused CLI/helper tests for rollback and no-rollback cases.
