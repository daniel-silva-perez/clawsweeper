## Summary

Keep this PR open. Current main still rejects symlinked skill directories whose realpath leaves `~/.agents/skills` or `<workspace>/.agents/skills`, and the PR’s intended user-managed symlink support is not implemented. The diff is narrow and test-backed, but it deliberately relaxes skill path containment for two sources, so the remaining action is maintainer review rather than automated cleanup.

## Best Possible Solution

Keep the PR open for maintainer review. If maintainers want user-managed `.agents/skills` symlinks, land this narrow implementation or an equivalent one, keep the positive personal/project symlink tests and managed-source negative test, document the security exception, and clarify the relationship to the already-closed #61585 and the still-open #46690.
