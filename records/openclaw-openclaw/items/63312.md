## Summary

Keep this PR open. Current main still rejects symlinked skill directories whose realpath escapes the configured root, and the personal/project `.agents/skills` roots flow through that same containment helper. The linked bundled dist-runtime issue was closed by a separate copy-based fix, but this PR’s user-managed `.agents/skills` symlink behavior is not implemented on main.

## Best Possible Solution

Review and either land this PR, or land an equivalent narrow change that accepts symlinked personal/project `.agents/skills` entries using logical containment while keeping bundled, managed, workspace, plugin, and extra-dir physical containment strict. If accepted, add or retain the symlink regression tests and consider a short docs/security clarification.
