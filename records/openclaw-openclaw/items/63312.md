## Summary

Keep open. Current main still uses strict realpath containment for the skills loader and does not implement the PR's user-managed `.agents/skills` symlink behavior. The PR is narrow and relevant, but it changes a skill-loading trust boundary, so the remaining action is maintainer review rather than conservative cleanup.

## What This Changes

The PR adds a logical-path containment fallback for personal and project `.agents/skills` sources, plus tests for symlinked personal/project skills and managed-source symlink rejection.

## Best Possible Solution

Land this PR or an equivalent narrow patch only if maintainers want personal and project `.agents/skills` symlinked directories to be a supported workflow. Keep strict physical containment for managed, bundled, workspace, extra, and plugin-owned skill roots; preserve focused regression tests; and update the public Skills security docs to describe the exception.
