## Summary

Keep this PR open. Current main still lacks the proposed logical-path fallback for user-managed personal/project `.agents/skills` symlinks, and the PR remains a narrow implementation candidate, but it deliberately relaxes a skill path-containment boundary and needs maintainer review rather than automated cleanup closure.

## What This Changes

The PR adds a user-managed skill-source exception in `src/agents/skills/workspace.ts` for symlinked personal/project `.agents/skills` directories and adds tests for personal/project symlink acceptance plus managed-source symlink rejection.

## Best Possible Solution

Keep this PR open for maintainer review. If maintainers want user-managed `.agents/skills` symlinks, land this narrow implementation or an equivalent one, preserve the personal/project positive tests and managed-source negative test, update the Skills security docs for the exception, and clarify how it relates to #46690 and the already-closed #61585.

- The PR body links #61585, but that closed report was about bundled `dist-runtime` symlinks, not personal/project `.agents` symlink behavior, so maintainers should not treat the closed related issue as proof this PR is obsolete.
