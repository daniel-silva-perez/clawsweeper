## Summary

Keep this PR open. Current main still uses strict realpath containment for personal and project `.agents/skills` entries, so the proposed user-managed symlink behavior is not implemented, but the patch needs maintainer/security approval plus two concrete fixes before merge: Windows-safe symlink tests and public documentation for the relaxed trust boundary.

## What This Changes

The PR adds a logical-path containment fallback for `agents-skills-personal` and `agents-skills-project`, with tests for symlinked personal/project skills and managed-source symlink rejection.

## Best Possible Solution

Land this PR or an equivalent narrow patch only if maintainers want symlinked personal and project `.agents/skills` directories to be supported. Keep strict physical containment for managed, bundled, workspace, extra, and plugin-owned skill roots; make the symlink tests match the repo's cross-platform pattern; and update the public Skills security docs at `https://docs.openclaw.ai/tools/skills` to describe the accepted exception.

  - body: The new tests call `fs.symlink()` unconditionally, but nearby skill symlink coverage is skipped on Windows because symlink creation and link type behavior are platform-sensitive there.   - body: This fallback allows personal/project `.agents/skills` entries whose real target is outside the configured root, but the public Skills security docs do not describe that exception or which roots remain strict.
