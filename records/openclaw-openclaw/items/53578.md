## Summary

Keep open. The issue is recent and concrete, and current main still routes Gemini CLI OAuth setups through the external `gemini` CLI backend. The report’s fixed-delay evidence also appears to come from standalone Gemini CLI/macOS Keychain behavior, but OpenClaw currently depends on that same CLI path for this setup, so there is not enough current-main or release evidence to close this as fixed, unreproducible, duplicate, or out of scope.

## Best Possible Solution

Keep this issue open for maintainer triage. The best path is to separate standalone `gemini` startup/auth latency from OpenClaw turn overhead, then either document a Gemini CLI/macOS Keychain remediation, add diagnostics that flag slow external CLI startup, or implement a supported mitigation that avoids repeated per-turn CLI credential initialization for Gemini CLI OAuth setups.
