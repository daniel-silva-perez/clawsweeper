## Summary

Keep open. Current main has a `session:patch` hook for the privileged `sessions.patch` RPC, including model patch support, but the two reported normal model-change paths still bypass that RPC/hook pipeline. `/model` applies the model override directly to the session entry and writes the session store; `session_status(model=...)` does the same. Existing tests verify the RPC hook path and model persistence behavior, but there is no regression coverage showing a hook fires for `/model` or `session_status` model changes.

## Best Possible Solution

Keep the issue open and implement a shared session-model patch path or hook-emission helper used by `sessions.patch`, `/model`, and `session_status(model=...)`. Add focused regression tests proving `session:patch` fires with `patch: { model: ... }` for both chat `/model` and the agent tool path, while preserving current live-switch and queued-followup semantics.
