## Summary

Keep this PR open. Current main still applies `--profile` by setting `OPENCLAW_PROFILE`, but it does not clear a stale inherited OpenClaw-managed `OPENCLAW_LAUNCHD_LABEL`; launchd plist resolution still gives that inherited label precedence. The PR remains a focused implementation candidate for open issue #65643, and the latest provided PR context shows the custom-label regression review was addressed by preserving non-OpenClaw operator labels.

## Best Possible Solution

Review and land this PR, or an equivalent narrow patch, as the canonical fix for #65643. The implementation should clear stale OpenClaw-managed launchd labels during explicit profile setup, preserve custom operator `OPENCLAW_LAUNCHD_LABEL` overrides, and keep regression coverage for both profile env cleanup and custom launchd label precedence.
