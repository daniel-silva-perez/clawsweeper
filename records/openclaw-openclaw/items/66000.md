## Summary

Keep this PR open. Current main still applies `--profile` by setting `OPENCLAW_PROFILE` but does not clear a stale inherited OpenClaw-managed `OPENCLAW_LAUNCHD_LABEL`, while launchd plist resolution still gives that env label precedence. The provided PR context shows a focused, recently repaired implementation candidate for open issue #65643, with no security-sensitive workflow/dependency/publishing changes in the PR file list.

## Best Possible Solution

Review and land this PR, or an equivalent narrow patch, as the canonical fix for #65643. The implementation should clear stale OpenClaw-managed launchd labels during explicit profile setup, preserve explicit custom `OPENCLAW_LAUNCHD_LABEL` overrides, and keep regression coverage for both profile env cleanup and custom launchd label precedence.
