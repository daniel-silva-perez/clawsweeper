## Summary

Keep this PR open. Current main still has the profile/launchd precedence bug: `--profile` applies profile env before command dispatch, but it does not clear inherited `OPENCLAW_LAUNCHD_LABEL`, and launchd resolution still prefers that env label over `OPENCLAW_PROFILE`. The repaired PR remains a focused implementation candidate for #65643 and the provided diff does not show CI, dependency, lockfile, release-script, or other supply-chain-sensitive changes.

## Best Possible Solution

Review and land this PR, or an equivalent patch, as the canonical fix for #65643. The implementation should clear only stale OpenClaw-managed launchd labels during explicit profile setup, preserve custom operator labels, and keep focused regression coverage in `src/cli/profile.test.ts` plus launchd override coverage.
