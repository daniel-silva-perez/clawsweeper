## Summary

Keep this PR open. Current main still has the macOS profile/launchd precedence bug from #65643, and this PR is a focused implementation candidate, but its current production change is too broad because it would clear explicit custom OPENCLAW_LAUNCHD_LABEL overrides that main currently supports and tests.

## Best Possible Solution

Keep this PR open and request a narrower fix. The best implementation should make explicit --profile invocations ignore or clear stale OpenClaw-managed launchd labels from a different profile, while preserving explicit custom OPENCLAW_LAUNCHD_LABEL overrides. Add regression coverage for stale managed labels, matching managed labels, custom labels, absent labels, and the gateway status plist-resolution path from #65643; then land the revised PR or close it only after a canonical fix lands elsewhere.
