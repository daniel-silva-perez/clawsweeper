## Summary

Keep open. Current main still has the reported OpenRouter owner-catalog mismatch: the bundled OpenRouter provider emits a bare `auto` model id while onboarding/docs use `openrouter/auto`, and the `/models` picker/session path still carries raw catalog model ids into `modelOverride`. The related fix PR #62745 is closed and unmerged, and its regression test is not present on main.

## Best Possible Solution

Review/reapply the closed #62745 approach or equivalent OpenRouter-owned fix: change the bundled default catalog id to `openrouter/auto`, add a regression test that OpenRouter catalog ids are canonical/namespaced, and add picker or catalog coverage proving `/models` exposes only one OpenRouter auto choice and cannot persist bare `auto` for this provider.
