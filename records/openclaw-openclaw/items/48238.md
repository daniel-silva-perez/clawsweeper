## Summary

Current main has adjacent safeguards, including summary quality retries, recent-turn preservation, tool-failure summary capture, and separate tool-loop detection, but it still does not implement the requested loop-aware saturated-session compaction guard. The requested config block, transcript-tail risk detector/scorer, guarded action mapping, post-compaction health validator, and recommend-reset escalation are not present, and the issue is paired with an open staged PR series for the same work.

## Best Possible Solution

Keep this issue open as the canonical tracker while maintainers review, consolidate, or replace the staged PR series. The best implementation path is a conservative config-gated guard with pure transcript-tail detection/scoring, guarded compaction instruction augmentation, post-compaction health validation, non-noisy recommend-reset escalation, incident fixture coverage, and docs once behavior lands.
