## Summary

Keep this issue open. Current main has a compaction safeguard path and separate tool-loop detection, but the requested loop-aware saturated-session guard is not complete: there is no `agents.defaults.compaction.guard` surface, no transcript-tail risk classifier/scorer/action mapping, no reset recommendation path, and no incident-specific regression fixture. The provided timeline also shows a same-author open PR stack for this exact breakdown, so the issue should stay open while those implementation candidates are reviewed.

## Best Possible Solution

Keep the issue open and review the linked PR stack as the likely implementation path. The best product direction is a conservative core guard layered on the existing compaction safeguard: typed config/schema/docs, tail-signal detector, scorer/action mapping, guarded instruction augmentation, post-compaction validation, last-resort reset recommendation, and a sanitized incident regression fixture.
