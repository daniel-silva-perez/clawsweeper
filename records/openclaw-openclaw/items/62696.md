## Summary

Keep this PR open. Current main does not implement the buffered text-only block reply collapse described by this PR, and the PR still appears to need the run-loop propagation called out in review before it can safely land.

## Best Possible Solution

Keep this PR open and either update it or land an equivalent fix that threads explicit buffered text-only block reply state from `runAgentTurnWithFallback` through `runEmbeddedPiAgent`, `runEmbeddedAttemptWithBackend`, and `subscribeEmbeddedPiSession`. The subscriber should collapse buffered text-only chunks back to the final assistant text when block streaming is disabled, with focused subscriber and runner-propagation regression coverage.
