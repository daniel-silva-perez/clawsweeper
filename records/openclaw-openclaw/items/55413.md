## Summary

Keep this PR open. Current main still lacks the intended Discord PluralKit canonical inbound-id behavior, and the PR is a focused implementation candidate rather than obsolete. It also still has a material review concern about PluralKit lookup ordering before the bot-drop gate that should be resolved before merge.

## Best Possible Solution

Keep the PR open and have the author resolve the remaining review concern: preserve the intended canonical inbound id behavior for PluralKit webhook proxy messages, avoid PluralKit calls for known OpenClaw bound-thread webhook echoes, and still classify PluralKit identity before bot-drop in degraded payload cases. The final patch should keep `MessageSid` canonicalized to PluralKit `original`, preserve the provider id in `MessageSidFull`, and include regression tests for webhook-authored PK messages, own webhook echoes, and the degraded missing-webhook-id bot-gate path.
