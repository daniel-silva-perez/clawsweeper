## Summary

Keep open. Current main has meaningful partial coverage: persisted session labels, `sessions.patch` label updates, `sessions_spawn` labels, and explicit `cleanup: "delete"`. The broader request is still not implemented: there is no `/session rename` or persisted auto-title flow, default keep-mode subagent sessions still remain visible, `archiveAfterMinutes` is not a general cleanup path for keep-mode runs, and the Control UI sessions surfaces still lack channel/kind/status filters or requested channel/kind grouping.

## What This Changes

Request to improve session management with human-readable names or titles, subagent auto-cleanup, and grouped/filterable Control UI session lists.

## Best Possible Solution

Keep this open as a core session-management enhancement, but split or explicitly scope the remaining work before implementation: define the intended naming contract, add a first-class rename or auto-title path, decide the default retention/visibility policy for completed run-mode subagents, and extend the sessions API plus Control UI with channel/kind/status filtering or grouping.

## Reproduction Assessment

Partially yes. The missing `/session rename`, absent persisted auto-title flow, and missing Control UI channel/kind/status filters are reproducible by static source/docs inspection; the reported `archiveAfterMinutes` timer behavior still needs a live Gateway reproduction, but source confirms keep-mode subagents do not receive archive timers.

## Solution Assessment

Not applicable as a patch review, and the requested direction is too broad for one automatic fix. The maintainable path is to keep the existing label and explicit cleanup support, then land narrower changes for naming, retention, and UI filtering after maintainers choose the product contract.
