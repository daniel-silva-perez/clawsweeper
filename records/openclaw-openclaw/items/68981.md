## Summary

Keep open. Current main has partial session-management improvements, including stored labels, `sessions.patch` label edits, derived read-only titles, and explicit `sessions_spawn` cleanup, but it still does not implement the central request: no `/session rename` or persisted auto-title flow, keep-mode run subagents are not automatically removed, and the Control UI sessions table still lacks channel/kind/status filters or grouping.

## What This Changes

Request to improve session management with human-readable session naming or auto-title, automatic subagent cleanup or TTL, and grouped/filterable Control UI session lists.

## Best Possible Solution

Keep this open as a core session-management enhancement, then split or explicitly scope the remaining pieces: a first-class rename or persisted title contract, a chosen default retention/visibility policy for completed run subagents, and sessions API plus Control UI filters/grouping for channel, session kind, and run status.

## Reproduction Assessment

Partially yes. Static inspection reproduces the missing `/session rename`, non-persisted auto-title behavior, explicit-only delete cleanup, and missing Control UI filters; the commenter’s OpenClaw 4.20 data adds live evidence for the archive cleanup concern, while a fresh live Gateway timer run was not needed to see the remaining source-level gaps.

## Solution Assessment

Not applicable as a patch review. The current implementation is useful partial groundwork, but the best maintainable solution is not to close this issue until maintainers choose the naming, retention, and UI filtering contract and land narrower changes against those decisions.
