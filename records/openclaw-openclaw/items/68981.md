## Summary

Keep this issue open. Current main has useful partial coverage: `sessions_spawn` supports human-readable labels and `cleanup: "delete"`, and session rows can persist/display labels. But the issue’s central UX request is broader: there is still no `/session rename` command or persisted auto-title flow, the Control UI sessions page only has active/limit/global/unknown plus text search, and the chat session selector is grouped by agent with a cron hide toggle rather than channel/kind/status filters or collapsible channel/kind sections. The related Telegram topic-name tracker is narrower and does not supersede this broader session-management item.

## Best Possible Solution

Keep the issue open as a core Control UI/session-management enhancement. The best path is to preserve the existing label and `cleanup: "delete"` support, then add or clarify a first-class rename/auto-title UX, verify/fix the `archiveAfterMinutes` sweeper behavior separately from explicit cleanup, and extend the Control UI session selector/sessions page with channel, kind, status, and label/name filtering or grouping. The narrower Telegram topic-name issue can remain separate for Telegram-specific display metadata.
