## Summary

Keep this issue open. Current main has useful partial support for session labels, label lookup/display, `sessions_spawn` labels, and explicit `cleanup: "delete"`, but it does not implement the broader requested UX: no `/session rename` command or persisted auto-title flow, no true channel/kind/status filtering in the Gateway sessions list or Control UI Sessions page, and no channel/kind grouping beyond the chat selector’s agent grouping and cron hide toggle. The follow-up comment’s `cleanup: "delete"` workaround is real and supported by current code/tests, but the reported `archiveAfterMinutes` timing concern still deserves separate verification rather than closing this umbrella issue.

## Best Possible Solution

Keep this open as a core session-management and Control UI enhancement. Preserve the existing label and explicit `cleanup: "delete"` support, then add a first-class rename or persisted auto-title UX, decide whether labels should remain separate from immutable keys, add Gateway list parameters and UI controls for channel/kind/status filtering, make subagent/cron kinds first-class enough for filtering/grouping, and separately verify or fix `archiveAfterMinutes` sweeper behavior.
