## Summary

Keep this PR open for maintainer review: current main does not implement the intended Anthropic setup-token sibling sync, but the PR branch is stale and over-broad in credential propagation, with an unrelated contract-registry change mixed into the auth fix.

## What This Changes

The PR routes plugin auth profile writes through a new sibling-sync helper, syncs Anthropic token and all OAuth credentials to sibling agent directories, adds Anthropic token sync coverage, and adds DeepSeek to the plugin contract registry.

## Best Possible Solution

Land a narrower maintainer-approved fix that centralizes token auth-profile persistence for the intended Anthropic setup-token/manual-token contexts, covers interactive plugin auth, non-interactive Anthropic setup-token, and paste-token as appropriate, keeps OAuth copy behavior opt-in, and leaves unrelated provider registry changes out of the auth PR.
