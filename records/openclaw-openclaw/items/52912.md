## Summary

Keep this PR open for maintainer changes. The branch addresses a real Anthropic token propagation problem, but its current approach broadens OAuth refresh-token copying across sibling agents and does not cover the models auth token entrypoints called out by the linked regression.

## What This Changes

The PR routes plugin provider auth profile writes through a new shared helper, syncs Anthropic token and OAuth credentials to sibling agent directories, adds Anthropic token sibling-sync coverage, and adds DeepSeek to the bundled provider contract registry.

## Best Possible Solution

Land a narrower auth persistence change that covers the intended Anthropic setup-token/manual-token flows across interactive plugin auth, models setup-token, paste-token, and non-interactive setup where needed; keep OAuth refresh-token copying opt-in; and split unrelated contract-registry cleanup from the auth fix.

## Reproduction Assessment

Yes. The PR diff itself reproduces the main review finding: an OAuth-returning plugin auth method would call the new writer with sibling sync enabled, while current docs and portability code require OAuth copy opt-in; the linked Anthropic token bug remains source-reproducible through the single-store models auth paths.

## Solution Assessment

No. This is not the best way to solve the issue because it broadens OAuth credential propagation and leaves relevant token entrypoints untouched; a safer fix should centralize token-profile persistence only for the intended Anthropic contexts and preserve the existing OAuth portability contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

  - body: `shouldSyncSiblingAgentsForCredential` returns true for every OAuth credential, so generic plugin OAuth login would copy refresh material into all sibling agent stores. Current docs and `resolveAuthProfilePortability` keep OAuth profiles non-portable unless `copyToAgents: true`, so sibling sync should be scoped to Anthropic token auth or an explicit provider/context opt-in.

  - confidence: 0.84
