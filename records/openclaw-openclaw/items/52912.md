## Summary

Keep open and do not merge as-is. The linked Anthropic token sync problem is valid, but this patch broadens credential propagation to all OAuth profiles, conflicts with the current portability policy, and does not cover every current token-auth persistence path involved in the linked report.

## What This Changes

The PR routes plugin auth-choice profile writes through a shared auth-profile writer, enables sibling-agent sync for Anthropic token and OAuth credentials, adds Anthropic token sync coverage, and adds DeepSeek to the provider contract registry.

## Best Possible Solution

Revise or replace the PR with a narrow Anthropic token persistence fix that covers the intended interactive setup-token, non-interactive setup-token, and paste-token surfaces, keeps OAuth copying explicitly opt-in under the existing portability contract, and splits unrelated registry cleanup into separate work.
