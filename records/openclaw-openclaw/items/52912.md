## Summary

Keep open. The linked regression is real and current main still writes the relevant token/setup-token paths to a single selected agent store, but this PR is not merge-ready because it broadens OAuth refresh-token copying across sibling agent stores, misses token entrypoints named in the linked report, and mixes an unrelated contract-registry change into the auth fix.

## What This Changes

The PR routes plugin provider auth profile writes through a new shared helper, syncs Anthropic token and OAuth credentials to sibling agent directories, adds Anthropic token sibling-sync contract coverage, and adds DeepSeek to the bundled provider contract registry.

## Best Possible Solution

Land a narrower maintainer-approved auth fix that centralizes token profile persistence for the intended Anthropic setup-token/manual-token contexts, keeps OAuth copying explicitly opt-in, covers plugin auth plus models auth entrypoints where needed, and moves unrelated registry cleanup to a separate change.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Restrict OAuth auth-profile sync to explicit opt-ins:** `src/plugins/provider-auth-helpers.ts:222-223`
  - body: `shouldSyncSiblingAgentsForCredential` returns true for every OAuth credential, so plugin OAuth logins would copy refresh material into all sibling agent stores. That conflicts with the documented default that OAuth profiles are non-portable unless explicitly opted in and weakens per-agent credential isolation; gate sibling sync to the Anthropic token case or a provider/context opt-in.
  - confidence: 0.92
- **[P2] Cover the models auth token entrypoints:** `src/plugins/provider-auth-choice.ts:133-135`
  - body: The linked regression uses `models auth setup-token` and `paste-token`, but this patch only changes `runProviderPluginAuthMethod`. Current models auth and Anthropic non-interactive token writers still persist to one agent store, so the reported multi-agent failure can remain after merging this branch.
  - confidence: 0.86
