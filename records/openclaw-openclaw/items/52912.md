## Summary

Keep this PR open for maintainer review. Current main still lacks the intended Anthropic setup-token sibling sync, but this branch is stale, incomplete for the broader linked regression, and over-broad because it would copy all OAuth credentials across sibling agent directories.

## What This Changes

The PR routes plugin auth profile writes through a new sibling-sync helper, syncs Anthropic token and all OAuth credentials to sibling agent directories, adds Anthropic token sync coverage, and adds a DeepSeek provider contract-registry import.

## Best Possible Solution

Land a narrower maintainer-approved fix that centralizes token auth-profile persistence for the intended Anthropic setup-token and manual-token contexts, covers interactive plugin auth, non-interactive setup-token, and paste-token as appropriate, keeps OAuth copying explicitly opt-in, and leaves unrelated registry cleanup out of the auth PR.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Restrict OAuth auth-profile sync to intended contexts:** `src/plugins/provider-auth-helpers.ts:222-223`
  - body: The new sync policy returns true for every OAuth credential, so any plugin auth method that returns OAuth credentials will write refresh material into all sibling agent stores. That breaks the documented default that OAuth profiles are non-portable unless explicitly opted in and can overwrite credentials for agents that should remain isolated. Scope sibling sync to the Anthropic token case or an explicit provider/context opt-in.
