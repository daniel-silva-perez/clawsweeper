## Summary

Keep this PR open, but do not merge it as-is. The branch targets a real open Anthropic setup-token propagation problem, yet the patch broadens OAuth credential copying across sibling agents, misses the models auth and Anthropic non-interactive token paths described by #51911, and includes a stale unrelated provider registry change that no longer matches current main.

## What This Changes

The PR routes plugin auth-choice profile writes through a new sibling-sync helper, syncs Anthropic token and all OAuth credentials to sibling agent directories, adds Anthropic token sync coverage, and adds DeepSeek to the old provider contract registry.

## Best Possible Solution

Land a narrower maintainer-approved fix that centralizes Anthropic token profile persistence for the intended setup-token and manual-token contexts, covers plugin auth-choice, models auth setup-token, paste-token, and non-interactive setup-token where appropriate, keeps OAuth copying explicitly opt-in under the existing portability contract, and leaves unrelated registry cleanup out of the auth PR.
