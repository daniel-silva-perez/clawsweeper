## Summary

Keep open. Current main still persists provider auth results, including Anthropic setup-token profiles, only to the resolved agent directory. The PR is still aimed at a real remaining gap, but it should be revised before merge because the proposed helper would also propagate all plugin OAuth credentials to sibling agents, which conflicts with per-agent auth isolation and was already flagged in review. The PR also includes an unrelated DeepSeek contract-registry edit that should be split out.

## Best Possible Solution

Revise this PR or replace it with a scoped fix that syncs Anthropic setup-token/paste-token credentials to sibling agent dirs only in the intended default setup context, preserves explicit per-agent auth isolation, adds the sibling-dir regression test, and removes the unrelated DeepSeek registry edit. Once that lands, it can close #51911’s credential-propagation portion.

## Risks / Open Questions

- The PR’s proposed sync policy would broaden credential propagation for all plugin OAuth credentials, creating a risk of leaking or overwriting credentials across sibling agents when users intentionally configure per-agent auth.
- The current code and tests preserve `--agent` targeted auth writes; a fix that syncs siblings unconditionally could regress that behavior.
- The PR mixes an unrelated DeepSeek contract-registry change into an Anthropic auth fix, which increases review risk and should be split out.
