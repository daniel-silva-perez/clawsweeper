## Summary

Keep this PR open. Current main still lacks the stale first-attempt age guard, and ordinary outbound startup recovery still replays retryCount 0 entries with no lastAttemptAt regardless of enqueuedAt. The PR is a plausible implementation candidate, but it is dirty against current main and still needs a fix for budget-deferred never-attempted entries before merge.

## Best Possible Solution

Keep the PR open, rebase it onto current main, and revise the recovery logic so stale filtering runs against the re-read currentEntry while preserving the active-claim race fixes. Track true attempted-vs-deferred state, or preserve first-attempt status for maxRecoveryMs budget deferrals, then add tests for stale first-attempt skip, retried-entry bypass, and budget-deferred never-attempted entries. Align the default/config story with the open stale replay trackers #55564 and #16555 before landing.
