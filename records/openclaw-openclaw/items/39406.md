## Summary

Current main still does not implement the requested normal-chat `toolErrorDelivery` / final / off policy. The active payload policy still preserves mutating tool warnings for success-looking replies, `messages.suppressToolErrors` remains ineffective for those mutating warnings, and the changelog explicitly leaves this issue for configurable delivery-policy follow-up.

## Best Possible Solution

Keep this issue open for maintainer policy review. The best implementation path is to decide how `messages.suppressToolErrors`, heartbeat-only `suppressToolErrorWarnings`, and a possible normal-chat delivery policy should interact, then make one documented config/schema/help/test change that preserves visible unresolved mutating failures by default while allowing an explicit managed-deployment suppression mode.
