## Summary

Keep this issue open. Current main still does not implement the requested normal-chat delivery policy such as `toolErrorDelivery: "all" | "final" | "off"`, and the current `messages.suppressToolErrors` path still does not suppress mutating write/edit-style warnings. Recent related work only covers narrower acknowledged-failure/retry cases and explicitly leaves this broader policy open.

## Best Possible Solution

Keep this issue open for a maintainer-chosen normal-chat tool-error delivery policy. The best path is a documented config/schema/help/docs change near `resolveToolErrorWarningPolicy` and `buildEmbeddedRunPayloads` that preserves the safe default, makes mutating write/edit failures visible unless explicitly suppressed, and defines clear behavior for final-success suppression before closing the broader request.
