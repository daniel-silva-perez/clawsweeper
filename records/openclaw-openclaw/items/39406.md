## Summary

Current main partially mitigates the original symptom by clearing same-action retry failures and hiding acknowledged failed-action warnings, but it still does not implement the requested normal-chat configurable delivery policy. The payload policy keeps unresolved mutating failures visible even with `messages.suppressToolErrors`, tests assert success-looking replies still receive mutating warning payloads, and the changelog explicitly leaves this issue for configurable delivery-policy follow-up.

## Best Possible Solution

Keep this issue open for maintainer policy review. The best path is to define one documented normal-chat tool-error delivery policy that explains how `messages.suppressToolErrors`, heartbeat-only `suppressToolErrorWarnings`, mutating failure safety, and any `all` / `final` / `off` style option interact, then update config types, zod/schema help, public docs, and focused payload/tool-handler tests while preserving visible unresolved mutating failures by default.
