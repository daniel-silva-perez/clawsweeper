## Summary

Keep this issue open. Current main still does not implement the central auto-recovery request: stuck-session diagnostics warn and emit events but do not abort runs or clear queues. The report also includes broader startup compaction, large-transcript, Feishu liveness, and Linux service resource-control gaps; some nearby pieces exist, but not enough to close the item as implemented or as a clean duplicate.

## Best Possible Solution

Keep this issue open and consolidate it with the active stuck-session recovery work. The best path is to revise or replace #73243 for diagnostics-owned auto-abort/queue cleanup, handle startup/large-transcript recovery alongside #64585 and #71555, verify whether Feishu transport liveness needs a channel-owned watchdog, and make a maintainer decision on default versus documented Linux service memory guardrails.
