## Summary

Keep open. Current main still preserves existing generated provider `apiKey` and same-surface `baseUrl` values from `agents/*/agent/models.json` in default merge mode, so the broader Onboard overwrite/persistence bug remains valid. The narrower related baseUrl API-surface case was discussed around #39103, but the same-provider edit path still needs an explicit update/overwrite contract across config, generated models, and auth profiles.

## Best Possible Solution

Keep this issue open and implement an explicit Onboard/provider-auth update path that distinguishes deliberate same-provider edits from omitted fields. New `apiKey` and `baseUrl` values should persist through `openclaw.json`, regenerated `agents/*/agent/models.json`, and auth-profile handling, while preserving existing runtime values only when the new config truly omits them or when SecretRef/auth-profile ownership requires it. Add regression coverage for Moonshot .ai/.cn base URL edits, same-API-surface custom base URL changes, plaintext key replacement, SecretRef-managed credentials, and auth-profile precedence.
