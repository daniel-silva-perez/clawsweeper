## Summary

Current main still preserves existing generated provider credentials/endpoints in the default merge path, so the broader Onboard overwrite/persistence bug remains valid. The related #39103/#37309 work addressed a narrower explicit-config baseUrl precedence case, but same-provider edits can still be masked by existing `agents/*/agent/models.json` values and the docs/tests still encode that behavior.

## Best Possible Solution

Keep this issue open and define an explicit provider-update contract for Onboard/configure flows. Deliberate same-provider edits should update `openclaw.json`, regenerated `agents/*/agent/models.json`, and auth-profile state as appropriate, while preserving existing generated values only when the new config omits them or when SecretRef/auth-profile ownership requires preservation. Add regression coverage for Moonshot .ai/.cn baseUrl edits, same-API-surface custom baseUrl changes, plaintext key replacement, SecretRef-managed credentials, and auth-profile precedence.
