## Summary

Current main still preserves existing generated provider credentials/endpoints in the default merge path, so the broader Onboard/configure overwrite issue remains valid. Related PR #39103 covered a narrower stale-baseUrl case, and #67893 later restored/documented agent models.json baseUrl preservation, leaving this issue as an unresolved product/contract decision rather than an already-implemented fix.

## Best Possible Solution

Keep this issue open and define a narrow provider-update contract for Onboard/configure flows. Deliberate same-provider edits should update `openclaw.json`, regenerated `agents/*/agent/models.json`, and auth-profile state as appropriate, while ordinary merge-mode regeneration should keep preserving user-managed generated values unless the edit path explicitly opts into overwrite. Cover Moonshot `.ai` to `.cn` baseUrl changes, plaintext key replacement, SecretRef-managed credentials, and auth-profile precedence.
