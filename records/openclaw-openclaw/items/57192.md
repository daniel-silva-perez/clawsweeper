## Summary

Keep this PR open. Current main still does not allow `screen.record` for macOS by default, while the macOS app still declares the command, so the linked bug remains plausible. The PR also needs revision before merge because the existing wizard deny-list and security audit still treat `screen.record` as dangerous; the Greptile review correctly points out that the proposed one-line platform default would be negated for wizard-created gateways.

## Best Possible Solution

Keep the PR open as a valid implementation candidate, but require revision before merge: decide the product/security policy for macOS `screen.record`, update `node-command-policy`, wizard defaults, security audit expectations, docs, and tests consistently, and include a regression that covers both bare config and wizard-generated `denyCommands` for the reported `macOS 26.4.0` platform string.
