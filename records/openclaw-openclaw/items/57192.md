## Summary

Keep this PR open. Current main still blocks `screen.record` for macOS by default while the macOS node declares the command, so the linked behavior remains plausible. The submitted PR is still an implementation candidate, but it needs revision because the wizard deny-list and security audit still classify `screen.record` as dangerous, and the PR also includes unrelated/stale OpenAI WebSocket warm-up changes.

## Best Possible Solution

Keep the PR open, but require a focused revision: decide the product/security policy for macOS `screen.record`, update `node-command-policy`, wizard defaults, security audit expectations, docs, and regression tests consistently, and cover both bare macOS `26.4.0` config and wizard-generated `denyCommands`. Split or drop the unrelated OpenAI WebSocket warm-up hunks before landing.
