## Summary

Keep this PR open. Current main still does not implement the PR's intended behavior: macOS defaults allow `screen.snapshot` but keep `screen.record` gated as a dangerous opt-in command. The linked bug remains plausible, and this PR is still an implementation candidate, but the submitted diff is incomplete because wizard-generated configs would still deny `screen.record`, security audit policy still treats it as dangerous, docs still describe explicit opt-in, and the PR mixes in unrelated OpenAI WebSocket warm-up changes.

## Best Possible Solution

Keep the PR open, but require a focused revision: decide the product/security policy for macOS `screen.record`, then update `src/gateway/node-command-policy.ts`, wizard default deny-list behavior, security audit expectations, docs, and regression tests consistently. Coverage should include the reported `macOS 26.4.0` metadata and a wizard-generated config with `denyCommands`. Split or drop the unrelated OpenAI WebSocket warm-up changes before landing; close the linked #57169 only after the revised fix merges.
