## Summary

Keep open. Current main has adjacent grounding, provenance, memory-citation, and completion-reporting pieces, but it still does not implement per-factual-claim confidence tiers, output-wide source attribution, known-vs-inferred semantics, or autonomous reporting for unverified claims. This remains a broad product/runtime trust contract rather than conservative cleanup.

## What This Changes

The issue requests an agent truthfulness contract that annotates factual claims with confidence, source attribution, known/inferred/assumed semantics, and autonomous-mode reporting for unverified claims.

## Best Possible Solution

Define a versioned truthfulness contract that specifies which outputs carry confidence/source metadata, how that metadata crosses agent runtime, task, protocol, and UI boundaries, and which parts are enforceable code versus documented prompt guidance.
