## Summary

Keep this open. Current main has adjacent prompt grounding, input provenance, memory-scoped citations, trajectory export, and task completion delivery, but it still does not implement the requested output-wide known/believed/uncertain/unknown claim contract or autonomous completion reporting for unverified claims.

## What This Changes

The RFC requests per-factual-claim confidence tiers, source attribution, known-versus-inferred wording, and autonomous-mode surfacing of unverified claims.

## Best Possible Solution

Keep this as a maintainer design item until OpenClaw has a documented truthfulness contract defining which outputs require confidence/source metadata, where that metadata is represented, how it survives task/protocol/completion delivery, and what is enforceable by code versus advisory prompt guidance.

## Reproduction Assessment

Not applicable: this is an RFC, not a concrete bug with runtime steps. The high-confidence check is source/docs inspection of current main, which shows adjacent trust features but no implemented per-claim confidence or unverified-claim reporting contract.

## Solution Assessment

Unclear: the requested direction is valid, but the implementation boundary is not defined. A prompt-only change would be too weak for the stated trust goal; maintainers should first choose the runtime, protocol, reporting, and enforcement model.
