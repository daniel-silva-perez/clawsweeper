## Summary

Keep this open. Current main has adjacent grounding and auditability features, including memory citation footers, input provenance, task-completion status metadata, trajectory bundles, and prompt guidance to verify mutable facts, but it does not implement the RFC's central request: explicit known/believed/uncertain/unknown confidence tiers and source attribution on every factual claim, nor a completion-report surface for unverified claims in autonomous/background operation.

## Best Possible Solution

Keep this issue open for maintainer/product design. The best path is to define a first-class trust/grounding contract for agent output: what counts as a factual claim, how known/believed/uncertain/unknown should be represented, how file/tool provenance is attached, how autonomous/background completion reports surface unverified claims, and how this interacts with existing memory citations, internal task-completion events, trajectory export, and channel/UI rendering.
