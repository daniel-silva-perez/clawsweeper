## Summary

Keep this open. Current main has adjacent prompt-level verification guidance, memory-scoped citations, inbound input provenance, trajectory export, and background task completion machinery, but it still lacks the requested output-wide per-factual-claim confidence/source contract and autonomous completion reporting for unverified claims.

## What This Changes

Request an RFC-level truthfulness contract that distinguishes known, believed, uncertain, unknown, and inferred claims, attributes sources, and reports unverified autonomous-mode claims.

## Best Possible Solution

Keep this open for maintainer design. The best end state is a documented truthfulness contract that defines which outputs must carry confidence/source data, how that metadata survives runtime/protocol/completion delivery, and which parts are enforced in code versus advisory prompt guidance.

## Reproduction Assessment

Not applicable: this is an RFC rather than a concrete bug with execution steps. The high-confidence check is source/docs inspection of current main, which shows adjacent grounding features but no per-claim confidence or unverified-claim completion contract.

## Solution Assessment

Unclear: the requested direction is valid, but the implementation boundary is not defined. A prompt-only update would be too weak for the stated trust goal; maintainers should first choose the runtime, protocol, reporting, and enforcement model.
