## Summary

Keep open. Current main has adjacent grounding guidance, input provenance, memory citations, trajectory export, and background-task completion summaries, but it still does not implement the RFC's per-factual-claim confidence tiers, output-wide source attribution, known-versus-inferred semantics, or autonomous reporting for unverified claims.

## What This Changes

The RFC requests an agent truthfulness contract with per-claim confidence tiers, source attribution, known-versus-inferred wording, and autonomous-mode surfacing of unverified claims.

## Best Possible Solution

Define a versioned truthfulness contract across agent runtime, task records, Gateway protocol, trajectory/export data, and user-facing completion reports, then implement the smallest enforceable slice with schema tests, runtime tests, and docs once maintainers choose the semantics.

## Reproduction Assessment

Not applicable: this is an RFC rather than a concrete bug report. The high-confidence check is current-main source and docs inspection, which shows adjacent mechanisms but no implemented per-claim confidence or unverified-claim contract.

## Solution Assessment

Unclear: the requested direction is reasonable, but the best implementation boundary has not been designed. A prompt-only change would be weaker than the requested verifiable trust contract; this needs a protocol/runtime/reporting design first.
