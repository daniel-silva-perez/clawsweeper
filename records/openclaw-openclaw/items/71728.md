## Summary

Keep this open. Current main has adjacent grounding and provenance features, but it still does not implement the requested per-factual-claim confidence tiers, output-wide source attribution, known-vs-inferred semantics, or autonomous completion reporting for unverified claims; the remaining work is a broad product/runtime trust contract rather than conservative cleanup.

## What This Changes

The issue requests a verifiable trust contract for agent output: confidence tiers and source attribution on factual claims, explicit known/inferred wording, and autonomous-mode reporting of unverified claims.

## Best Possible Solution

Keep this open for maintainer design. The desired end state is a documented truthfulness contract that defines which outputs must carry confidence/source data, how that metadata flows through runtime and protocol boundaries, and where code can enforce it versus relying on prompt guidance.

## Reproduction Assessment

Not applicable: this is an RFC rather than a concrete bug with steps. The high-confidence check is source and docs inspection of current main, which shows adjacent grounding/provenance features but no implemented per-claim confidence or unverified-claim completion contract.

## Solution Assessment

Unclear: the requested direction is reasonable, but the best implementation is not yet specified. A prompt-only change would be too weak for the stated trust goal; maintainers should first choose the runtime/protocol/reporting boundary and enforcement model.
