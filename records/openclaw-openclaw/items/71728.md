## Summary

Keep this issue open. Current main has adjacent grounding mechanisms for memory citations, inter-session input provenance, trajectory export, background task status, and prompt-level verification discipline, but it does not implement the requested per-factual-claim known/believed/uncertain/unknown contract or autonomous completion reporting of unverified claims. This is a broad product/runtime trust design item, not conservative cleanup.

## What This Changes

The issue requests confidence tiers, source attribution, known-vs-inferred wording, and autonomous completion reporting for unverified factual claims in agent output.

## Best Possible Solution

Keep this open for maintainer design. The best end state is a documented truthfulness contract that defines which outputs must carry confidence/source data, how that data survives agent runtimes and completion delivery, and which parts are enforceable versus advisory before any broad code changes start.
