## Summary

Keep open. Current main has adjacent prompt guidance, memory citations, input provenance, trajectory capture, and task completion summaries, but it still does not implement the RFC’s per-factual-claim confidence tiers, output-wide source attribution, known-versus-inferred semantics, or autonomous unverified-claim reporting contract. The existing ClawSweeper comment reached the same conclusion, and the source inspection still supports keeping this as a maintainer design item rather than closing it as cleanup.

## What This Changes

The issue requests confidence tiers, source attribution, known-versus-inferred labeling, and autonomous reporting of unverified claims for agent output.

## Best Possible Solution

Define a versioned truthfulness contract across agent runtime, task records, gateway protocol, and user-facing reports, then implement the smallest enforceable slice with tests and docs once maintainers choose the claim/source/confidence semantics.

## Reproduction Assessment

Not applicable: this is an RFC, not a concrete bug report. The high-confidence check is source and docs inspection of current main, which shows adjacent grounding/provenance mechanisms but no implemented per-claim confidence or unverified-claim completion contract.

## Solution Assessment
