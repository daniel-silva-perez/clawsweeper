## Summary

Keep this issue open. Current main has adjacent prompt-level grounding, memory citations, input provenance, trajectory export, and background task notification machinery, but it does not implement the requested per-factual-claim known/believed/uncertain/unknown contract or autonomous completion reporting for unverified claims. This remains a broad product/runtime trust design item rather than conservative cleanup.

## What This Changes

The issue requests an agent truthfulness contract with per-claim confidence tiers, source attribution, known-vs-inferred wording, and autonomous completion reporting for unverified claims.

## Best Possible Solution

Keep this open for maintainer design. The best end state is a documented truthfulness contract that defines which agent outputs must carry confidence/source data, how that data survives runtime/protocol/completion delivery, and which parts are enforceable by code versus advisory prompt guidance.

## Reproduction Assessment

Not applicable: this is an RFC rather than a concrete bug with execution steps. The high-confidence check is source/docs inspection of current main, which shows adjacent grounding features but no implemented per-claim confidence or unverified-claim completion contract.

## Solution Assessment

Unclear: the requested direction is valid, but the best implementation is not defined yet. A prompt-only change would be too weak for the stated trust goal; maintainers should first choose the runtime/protocol/reporting boundary and enforcement model.
