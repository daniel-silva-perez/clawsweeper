## Summary

Keep open. Current main has adjacent truthfulness mechanisms, but not the requested claim-level confidence tiers, source attribution for every factual claim, or autonomous completion reporting of unverified claims.

## Best Possible Solution

Keep this RFC open for maintainer product/design review. The best path is an additive core or SDK-visible trust metadata contract spanning reply payloads, gateway agent/completion events, prompt guidance, and channel rendering, while reusing memory citations and memory-wiki claim metadata only as input evidence sources.
