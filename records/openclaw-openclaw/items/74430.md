## Summary

Keep open. Current main still does not implement the proposed private skill archive upload RPC, so the PR is not obsolete; however, the patch is an XL security-sensitive Gateway protocol/code-install feature and introduces a blocking ClawHub archive compatibility regression that needs maintainer review before merge.

## What This Changes

The PR adds admin-scoped `skills.upload.begin/chunk/commit` RPCs, extends `skills.install` with `source: "upload"`, adds a durable upload store and shared archive install helper, and updates docs/tests for private skill archive installs.

## Best Possible Solution

Fix or explicitly version the ClawHub root-marker compatibility change, then have maintainers review the additive Gateway protocol contract and security model before landing the upload lifecycle with focused store/handler tests and changed-gate validation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve legacy skill root filename compatibility:** `src/agents/skills-archive-install.ts:63-64`
  - body: Current main accepts ClawHub archives whose extracted root contains `skill.md`, `skills.md`, or `SKILL.MD` via `ensureSkillRoot`. This helper only accepts exact `SKILL.md`, and `installSkillFromClawHub` now routes through it, so existing packages using those legacy marker names fail on Linux with `archive is missing SKILL.md`. Keep the legacy variants for the ClawHub path or make the breaking change explicit and versioned.
  - confidence: 0.91
