## Summary

Keep this issue open. Current main still has the reported Control UI crash shape: `cron.list` rows are accepted into UI state without runtime row validation, and `renderApp` still reads `job.payload.kind` while building Cron model suggestions before the later Cron-tab render gate. The two focused fixes linked in the discussion, #54550 and #54552, are closed and unmerged, so no implemented-on-main close is justified.

## What This Changes

The issue reports a Control UI regression where visiting sidebar pages such as Cron, Sessions, or Nodes can leave Chat unresponsive because a render path reads `kind` from an undefined Cron payload.

## Best Possible Solution

Implement a narrow robustness fix in the Control UI Cron path: validate or filter malformed `cron.list` rows before storing them in UI state, guard any remaining Cron render helpers before reading `payload.kind`, and add a jsdom or unit regression test proving malformed Cron rows cannot block switching back to Chat.
