## Summary

Keep this issue open. Current main still has the reported Control UI crash path: Cron navigation loads `cron.list` rows into UI state without row-shape validation, and the shared render path still reads `job.payload.kind` before the Cron-tab render branch or any payload guard. The focused fixes linked in the discussion, #54550 and #54552, are closed and unmerged.

## Best Possible Solution

Keep this issue open. The best maintainer path is to revive or reimplement the focused #54552 approach on current main: validate or filter malformed `cron.list` rows at the UI and/or server boundary, guard Control UI cron helpers before reading `payload.kind`, and add a jsdom or browser regression test proving malformed Cron rows do not prevent switching back to Chat.
