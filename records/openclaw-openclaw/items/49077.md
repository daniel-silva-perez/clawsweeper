## Summary

Keep open. Current main at e583db63c602d8d21b4ebdcd119061571a2e40a9 still does not expose a configurable delay between model API request starts. The model config type, strict schema, docs, and guarded model fetch path cover provider timeouts, headers, auth, request transport overrides, SSRF policy, and Retry-After retry bypass behavior, but not global or per-provider request pacing. PR #49221 remains the matching open implementation candidate, so this issue should stay open until that PR lands or maintainers choose a different design.

## Best Possible Solution

Review PR #49221 as the current implementation candidate, or replace it with an equivalent core design that adds validated global and per-provider model request delay config, applies it to embedded agent runs and compaction, documents the public config surface, and tests provider-scoped request spacing including zero overrides and provider normalization.
