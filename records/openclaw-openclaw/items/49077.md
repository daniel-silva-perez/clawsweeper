## Summary

Keep open. Current main does not expose a configurable delay between model API request starts. The current model config type, strict schema, and user docs cover provider timeouts, headers, auth, proxy/TLS, and private-network request transport overrides, but not a global or per-provider request-delay setting. The related PR #49221 is open and unmerged and appears to be the implementation candidate for the same request, so the issue should stay open until that PR lands or maintainers choose a different design.

## Best Possible Solution

Review PR #49221 as the current implementation candidate, or replace it with an equivalent core design that adds a validated global and per-provider model request delay, applies it to embedded agent runs and compaction, documents the config surface, and includes focused tests for provider-scoped request spacing.
