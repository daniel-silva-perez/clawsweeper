## Summary

Keep open. Current main at ef58307f843eed1a7832e9c23c519591f1b7cdbd still does not expose a configurable delay between consecutive model API request starts. The model config type, strict schema, docs, guarded fetch path, and embedded run/compaction stream setup cover timeouts, headers, request transport, SSRF guarding, and Retry-After retry bypass behavior, but not request pacing. PR #49221 remains the matching open implementation candidate.

## Best Possible Solution

Keep this issue open and review PR #49221 as the current implementation candidate, or replace it with an equivalent core design that adds validated global and per-provider model request delay config, applies it to embedded agent runs and compaction, documents the public config surface, and adds focused tests for provider-scoped request spacing and zero overrides.
