## Summary

Keep this PR open. Current main does not implement the PR’s client-side WebChat fixes: the Control UI still uses a case-sensitive `NO_REPLY` regex, has no `NO_REPLY` prefix-stream guard, and current tests still expect lowercase `no_reply` assistant payloads to remain visible. The included `/activate` alias work is also not present on main.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land an updated or split change that brings Control UI silent-token handling into parity with the server helper, adds the lowercase and prefix-fragment regression tests, and separately decides whether the `/activate` alias belongs in this PR or a focused command-control PR.
