## Summary

Keep this PR open. Current main has the backend `sessions.create` method, but it does not implement this PR’s intended Control UI flow: a discoverable New Session action in the chat header/session selector that opens a naming modal, persists via `sessions.create`, and immediately switches to the created session. The existing visible plus button is a composer toolbar action that sends `/new` with reset confirmation, which is materially different from named persistent session creation. The provided PR discussion is useful and includes several functional review comments that should be resolved before merge, but there is no current-main implementation that makes this PR obsolete.

## Best Possible Solution

Keep the PR open as an implementation candidate, or replace it with an equivalent main-branch Control UI change that adds a discoverable New Session action near the chat session selector/header, creates a unique labeled dashboard session through `sessions.create`, switches to it on success, handles RPC failures visibly, and covers agent resolution/reconnect edge cases with focused UI tests.
