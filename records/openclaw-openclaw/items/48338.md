## Summary

Keep open. Current main still gives generic custom providers a 16,000-token context window and 4,096 max output tokens by default, while the interactive and non-interactive custom-provider setup paths do not expose contextWindow/maxTokens prompts or flags. The related #43493 PR is a plausible implementation candidate for this UX, but it is still open and not present on main.

## Best Possible Solution

Review and land #43493 or an equivalent main-branch change that lets custom-provider setup configure and display effective context metadata, warns when OpenClaw is using the 16K fallback, and updates docs/tests. After that merges, close this as implemented with the merge commit and shipped release provenance.

## Risks / Open Questions

- The issue is broader than the open PR: landing metadata prompts would address setup visibility, but maintainers should also decide whether runtime context-overflow copy and docs need follow-up coverage.
- Raising the generic fallback too high could overrun genuinely small third-party endpoints, so the safest path is an explicit guided metadata/warning flow rather than only changing the default.
