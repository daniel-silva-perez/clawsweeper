## Summary

Keep this open. Current main still writes generic custom-provider models with a 16,000-token context window and 4,096 max output tokens, and the custom-provider setup paths still do not expose contextWindow/maxTokens configuration or fallback warnings. The open #43493 PR remains the clearest implementation candidate, but it is not present on main.

## Best Possible Solution

Review and land #43493 or an equivalent main-branch change that lets custom-provider setup configure and display effective context metadata, warns when OpenClaw uses the conservative fallback, and updates docs/tests so interactive and non-interactive setup match the documented config model.

## Risks / Open Questions

- Docs currently describe manual custom-provider metadata and, in one section, say omitted optional custom-provider fields default to larger values, while the onboarding path writes explicit 16K/4096 metadata. That mismatch can keep the UX confusing even for users who read the docs.
- Landing metadata prompts alone may not cover the full issue. Runtime overflow copy and explicit fallback warnings may still need follow-up if maintainers want the complete guided experience described in the report.
