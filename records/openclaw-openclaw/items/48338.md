## Summary

Keep this issue open. Current main still configures non-Azure custom/OpenAI-compatible providers with the hard-minimum 16,000-token context window and 4,096 max output tokens, and the setup paths still do not collect or display contextWindow/maxTokens metadata or fallback warnings. The open #43493 PR remains the clearest implementation candidate, but it is not present on current main.

## Best Possible Solution

Review and land #43493 or an equivalent main-branch change that lets custom-provider setup configure and display effective context metadata, warns when OpenClaw uses the conservative fallback, updates non-interactive flags/docs/tests, and improves runtime overflow copy if maintainers want the full guided experience described here.
