## Summary

Keep open: current main still has the reported misleading fallback, and this PR is a narrow implementation candidate for the open linked bug rather than something already implemented or superseded.

## What This Changes

The PR adds CLI-side missing media-understanding provider detection for `infer image describe` and `infer audio transcribe`, with tests that separate no-provider errors from configured-provider empty output.

## Best Possible Solution

Land a focused fix that emits configuration-focused errors only when no suitable media-understanding provider is configured, while preserving the existing empty-result errors when a configured provider is actually called and returns no text.
