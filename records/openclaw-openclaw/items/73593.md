## Summary

Keep this PR open: current main still emits the generic path-focused image/audio empty-result errors, while this PR is a focused implementation candidate for the linked no-provider-configured bug.

## What This Changes

The PR adds CLI error selection and regression tests so `infer image describe` and `infer audio transcribe` report missing media-understanding provider configuration when no provider is configured, while preserving the existing empty-result errors for configured providers.

## Best Possible Solution

Keep this PR open for maintainer review and targeted validation, then land it or an equivalent focused change that reports missing audio/image-understanding provider configuration while preserving the existing empty-result errors when a configured provider returns empty text.
