## Summary

Keep this PR open. Current main still lacks the proposed synchronous delegation tools, so the branch is not obsolete; however, the patch is not merge-ready because it adds a security-sensitive attachment input path without matching transcript redaction and still has an output-capture cleanup race for fast child runs.

## What This Changes

The PR adds `sessions_delegate` and `sessions_delegate_batch` agent tools, registers them in tool catalog/display surfaces, wires delegate-style subagent cleanup, and adds tests for synchronous child-result collection.

## Best Possible Solution

Keep this PR open as the implementation candidate for synchronous subagent delegation, but require maintainer API review plus a rebase onto current main, delegate attachment redaction, pre-cleanup output-capture synchronization, gate release in failure paths, and focused docs/tests before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Redact delegate attachment payloads before transcript persistence:** `src/agents/tools/sessions-delegate-tool.ts:51-61`
  - body: `sessions_delegate` accepts inline `attachments[].content`, but current transcript repair only redacts attachment payloads for `sessions_spawn`. Persisted assistant tool-call inputs for the new delegate tool can therefore retain raw file contents or secrets and bloat replay context; extend `sanitizeToolCallInputs` to cover delegate inputs or avoid persisting raw content.
  - confidence: 0.9
- **[P1] Register the output-capture gate before cleanup can run:** `src/agents/tools/sessions-delegate-tool.ts:191`
  - body: `registerOutputCaptureGate` is called after `spawnSubagentDirect` resolves, but spawn registration and lifecycle cleanup can already happen before that return path completes. A very fast child can reach delegate cleanup while no gate exists, so cleanup can delete the child session/transcript before `readChildOutput` falls back to history.
  - confidence: 0.86
- **[P2] Release the output-capture gate when output reading fails:** `src/agents/tools/sessions-delegate-tool.ts:220-221`
  - body: `signalOutputCaptured` only runs after `readChildOutput` resolves. If the history fallback throws, delegate cleanup waits for the 30-second lifecycle timeout before it can finish deleting or archiving the already-finished child run; wrap the output read in `try/finally` so the gate is always released.
  - confidence: 0.8
