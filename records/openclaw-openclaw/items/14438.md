## Summary

Keep open. Current main still lacks the requested live plugin reload or plugin cache-clear workflow: plugin docs remain restart-only, the plugin CLI exposes no reload/cache-clear command, plugin config changes still plan a Gateway restart, and the latest maintainer discussion explicitly keeps this item open for plugin-loader and Gateway lifecycle design.

## What This Changes

Request a supported plugin-development reload workflow that invalidates plugin/Jiti cache and re-registers plugin runtime surfaces without restarting the OpenClaw process or container.

## Best Possible Solution

Define a supported plugin development reload path in the plugin loader and Gateway lifecycle model, including cache invalidation, active registry replacement, disposal semantics, CLI/RPC UX, and public docs; if live runtime reload remains unsupported, keep restart-only behavior documented with accurate cache/container guidance.
