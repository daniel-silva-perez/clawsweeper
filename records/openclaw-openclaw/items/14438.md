## Summary

Current main still matches the requested feature gap: plugin runtime edits remain restart-only, the plugin CLI has no reload or cache-clear command, and plugin config changes still route to Gateway restart behavior. The latest maintainer discussion also explicitly keeps this open for core plugin loader and Gateway lifecycle design, so this should not be closed by cleanup.

## What This Changes

Request support for reloading TypeScript plugins, invalidating Jiti/plugin loader cache, or clearing the relevant cache without restarting the OpenClaw process/container.

## Best Possible Solution

Design a supported plugin development reload workflow backed by a Gateway/API path that invalidates loader/Jiti/module state and safely swaps plugin registrations, then expose it through `openclaw plugins reload <id>` and/or `openclaw plugins clear-cache` with docs; if live runtime reload remains unsupported, document restart-only behavior with precise container/cache guidance.
