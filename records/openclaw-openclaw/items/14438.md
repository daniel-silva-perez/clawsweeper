## Summary

Keep open. Current main still documents native plugin runtime changes as restart-only, exposes no `openclaw plugins reload` or plugin cache-clear command, and treats plugin config changes as restart-required rather than a live plugin-code swap. The maintainer discussion already kept the item open for plugin loader and Gateway lifecycle design; adjacent debounce work does not implement the requested reload workflow.

## What This Changes

Request a plugin development workflow that invalidates Jiti/plugin loader cache and reloads TypeScript plugin code without restarting the OpenClaw process or container.

## Best Possible Solution

Define a supported plugin development reload path in the plugin loader and Gateway lifecycle model, including cache invalidation, active registry replacement, disposal semantics, CLI/RPC UX, and public docs; if live runtime reload remains unsupported, keep restart-only behavior documented with accurate cache/container guidance.
