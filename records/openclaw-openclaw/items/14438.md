## Summary

Keep open. Current main still documents and implements plugin runtime changes as restart-required, has no supported plugin reload or cache-clear CLI/RPC path, and the maintainer discussion explicitly leaves this for plugin-loader and Gateway lifecycle design rather than conservative cleanup closure.

## What This Changes

The issue requests a supported plugin development reload or cache-clear workflow so TypeScript plugin changes can take effect without a full OpenClaw process or container restart.

## Best Possible Solution

Define a supported plugin development reload path in the plugin loader and Gateway lifecycle model: invalidate Jiti/module/runtime caches, safely replace the active registry, run plugin lifecycle cleanup for stale registrations, expose an additive Gateway/CLI UX such as openclaw plugins reload or clear-cache, and document the container and security behavior. If live runtime reload remains unsupported, keep restart-only behavior documented with accurate cache guidance.
