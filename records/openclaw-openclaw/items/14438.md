## Summary

Keep open: current main still documents native plugin runtime edits as restart-only, exposes no `openclaw plugins reload` or cache-clear command, and plans `plugins.*` config changes as Gateway restarts. The latest maintainer comment also explicitly keeps this open for core plugin loader and Gateway lifecycle follow-up.

## What This Changes

Request a supported plugin development workflow that reloads native TypeScript plugin code or clears Jiti/module cache without restarting the OpenClaw process/container.

## Best Possible Solution

Add a supported plugin development workflow: either `openclaw plugins reload <id>` and/or `openclaw plugins clear-cache` backed by a Gateway/API path that invalidates loader/Jiti/module state and safely swaps plugin registrations, or explicitly document restart-only behavior with precise Docker/cache guidance if native runtime hot reload remains unsupported.
