## Summary

Keep open. Current main still has no supported plugin code hot-reload or plugin cache-clear command, the docs explicitly describe plugin runtime/config changes as restart-only, and the maintainer discussion already marked this as Gateway/plugin-loader lifecycle design work rather than cleanup closure.

## What This Changes

Request a supported plugin development reload workflow that invalidates Jiti/plugin loader caches and re-registers plugin runtime state without restarting the Gateway/container.

## Best Possible Solution

Define a supported plugin development reload path in the plugin loader and Gateway lifecycle model: invalidate Jiti/module/runtime caches, safely replace active plugin registrations, dispose stale lifecycle resources, expose an additive CLI/RPC UX such as `openclaw plugins reload <id>` or `openclaw plugins clear-cache`, and document the container/cache/security behavior. If live runtime reload remains unsupported, keep restart-only behavior documented with accurate cache guidance.
