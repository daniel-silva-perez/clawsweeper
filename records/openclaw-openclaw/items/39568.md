## Summary

Keep this issue open. Current main still has the reported Feishu reply-mention gap, and the linked implementation PR #39569 remains open and unmerged in the provided GitHub context with closing syntax for this issue. The source still treats a non-empty Feishu `mentions` array as authoritative and still carries `root_id` into thread-reply dispatch.

## Best Possible Solution

Keep this issue open until PR #39569 or an equivalent Feishu plugin fix lands. The concrete path is to update `checkBotMentioned()` so reply events with orphaned bot mention placeholders are detected, suppress `rootId`/thread routing only for normal group reply @mentions that should answer in the main chat, and add regression tests for both mention detection and non-thread reply dispatch while preserving explicit topic/thread behavior.
