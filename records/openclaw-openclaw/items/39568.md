## Summary

Keep this issue open. Current main still has the reported Feishu reply-mention failure path: structured mentions are treated as authoritative when non-empty, orphaned `@_user_N` placeholders are not detected, and preserved `root_id`/`thread_id` still force Feishu thread replies. The linked implementation PR #39569 remains open and unmerged in the provided GitHub context, so it should be reviewed or replaced rather than closing the report early.

## Best Possible Solution

Keep this issue open until PR #39569 or an equivalent Feishu plugin fix lands. The concrete maintainer path is to update `checkBotMentioned()` so Feishu reply events with orphaned bot mention placeholders are detected, clear or suppress `rootId`/`threadReply` only for normal group reply @mentions that should answer in the main chat, and add regression tests for both the mention-detection false negative and the non-thread reply dispatch behavior.
