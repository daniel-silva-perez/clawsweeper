## Summary

Keep this PR open, but do not merge the current revision. The reported invalid-input timing hardening target is real on current main, and the added null/undefined-versus-empty tests are useful, but the patch replaces the recently remediated padded byte comparison with a fixed-key HMAC digest comparison that has active GitHub Advanced Security CodeQL comments.

## What This Changes

The PR replaces `safeEqualSecret`'s padded UTF-8 byte comparison with fixed-key HMAC-SHA256 digest comparison for normalized null/undefined values and adds null/undefined-versus-empty regression cases.

## Best Possible Solution

Keep the padded timing-safe byte comparison as the equality primitive, make invalid inputs perform comparable bounded dummy work before returning false, and retain the new null/undefined-versus-empty regression coverage.

## Reproduction Assessment

Yes. Current main visibly returns before buffer allocation and `timingSafeEqual` for non-string inputs, while the string path performs buffer conversion and padded timing-safe comparison.

## Solution Assessment

No. The tests are useful, but the implementation reverses the recent CodeQL remediation by hashing candidate credentials again; the safer fix should preserve the padded-buffer primitive.
