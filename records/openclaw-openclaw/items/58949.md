## Summary

Keep open. Current main still has the non-string early return in `safeEqualSecret`, and the latest tagged release has the same behavior, so this PR is not implemented on main or superseded. The PR is a narrow security hardening change to a shared auth helper, but it should be revised to address the existing review feedback: add null/undefined-versus-empty regression tests and remove the small `&&` short-circuit timing asymmetry.

## Best Possible Solution

Keep this PR open and ask for a small revision: normalize inputs before hashing, compute type/equality booleans without short-circuiting, add regression tests for null/undefined versus empty-string and both-sides null/undefined cases, then run the targeted security test and the normal changed gate before merge.
