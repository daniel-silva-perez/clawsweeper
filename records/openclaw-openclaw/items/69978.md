## Summary

Keep open. Current main still only exposes the TUI streaming watchdog as an internal `streamingWatchdogMs` handler option, does not register a `/watchdog` TUI command or persisted user setting, and does not track warned run ids to suppress repeated warnings for the same run. Related #68596 and #69081 remain open and do not fully supersede this issue.

## Best Possible Solution

Keep this issue open and implement a focused TUI change: add per-run watchdog-warning suppression, add a local `/watchdog` command with status/off/duration validation, wire it into the event handler with tests, and either persist it in the existing OpenClaw state-dir pattern or explicitly leave broader config policy to #68596.
