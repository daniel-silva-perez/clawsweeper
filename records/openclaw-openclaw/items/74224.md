## Summary

Keep this PR open. Current main still lacks the reconnect/watchdog resync behavior targeted by the PR: watchdog arming is tied to chat deltas, quiet tool events return before any watchdog refresh, lifecycle events do not rearm the watchdog, and TUI reconnect/disconnect handlers do not pause or rearm watchdog state. The PR is a narrow implementation attempt for open bug #69081, with no obvious supply-chain or security-sensitive file changes, so it should go through normal maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for normal maintainer review. If the approach is sound, land a narrow TUI fix that treats active-run tool and non-terminal lifecycle events as streaming watchdog proof-of-life, pauses the watchdog while disconnected, rearms it on reconnect without dropping a legitimate in-flight activeChatRunId, and reloads history when a reconnect-stale run times out. Keep watchdog threshold/message UX changes such as #69026 separate.
