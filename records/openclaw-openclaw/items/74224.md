## Summary

Keep this PR open. Current main still lacks the reconnect/watchdog resync behavior targeted by the PR and by #69081: watchdog proof-of-life is still tied to chat deltas, quiet tool events return before refreshing the watchdog, lifecycle events do not rearm it, and reconnect/disconnect handling does not pause or rearm watchdog state. The PR is a narrow TUI implementation candidate with no obvious supply-chain-sensitive file changes, so it should proceed through normal maintainer review.

## Best Possible Solution

Keep this PR open for normal maintainer review. If the latest approach is sound, land a narrow TUI fix that treats active-run tool and non-terminal lifecycle events as watchdog proof-of-life, pauses the watchdog during disconnects, rearms it on reconnect without dropping a legitimate in-flight activeChatRunId, and reloads history when a reconnect-stale run times out. Keep watchdog threshold/message UX changes such as #69026 separate unless maintainers intentionally combine them.
