## Summary

Keep this draft PR open. Current main still does not implement the PR's intended TUI watchdog/reconnect behavior: the watchdog is armed from chat deltas, active-run tool events can return before any watchdog refresh when tool verbosity is off, lifecycle events update status without refreshing watchdog liveness, and reconnect/disconnect handlers do not pause or rearm the watchdog. The PR is a narrow implementation attempt for open bug #69081, with no security-sensitive files changed, so it needs normal maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this draft PR open for normal maintainer review. The best path is to review and, if sound, land a narrow TUI fix that treats active-run tool and non-terminal lifecycle events as watchdog proof-of-life, pauses the watchdog during disconnect, rearms on reconnect without dropping valid in-flight runs, and reloads history when a reconnect-stale run times out. Keep watchdog UX/config follow-ups such as #69026 separate.
