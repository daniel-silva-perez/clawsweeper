## Summary

Keep open. Current main still keeps implicit ACP parent streaming behind the subagent-requester gate and has no `acp.progressForward` config surface, so the PR's intended behavior is not already implemented. The branch remains potentially useful, but it changes a user-visible event-delivery default and is stale against the current generated config-docs pipeline, so it needs maintainer review and a rebase before merge.

## What This Changes

The PR default-enables implicit ACP parent progress forwarding with an `acp.progressForward` opt-out, adds thread-bound forwarding exclusions, expands ACP spawn tests, and updates config schema/help/labels plus ACP docs and generated config docs artifacts.

## Best Possible Solution

If maintainers want this default, port the branch onto current main, keep `acp.progressForward: false` as the opt-out, preserve the session-binding/session-key/delivery-context thread exclusions, update the published ACP agent docs and current config-docs hash artifact, coordinate the parent-stream event contract with #54471, then merge after targeted ACP tests and the changed gate pass.
