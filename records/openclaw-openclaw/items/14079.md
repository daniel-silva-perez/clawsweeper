## Summary

Keep this issue open. Current main still does not implement Linux desktop notification delivery for the existing `system.notify` / `openclaw nodes notify` surface: Linux is allowlisted at the gateway policy layer, but the headless node host neither advertises nor handles `system.notify`, and docs/CLI still describe notification delivery as macOS-node behavior. The related closed #24616 covered the misleading error path, not the requested Linux `notify-send`/libnotify support.

## Best Possible Solution

Keep this issue open and implement a focused Linux headless node-host `system.notify` path: detect a usable desktop session, advertise the command only when delivery is available, deliver via a fixed-binary `notify-send` path first, degrade cleanly on headless/no-backend hosts, keep macOS/iOS/Android behavior unchanged, and update CLI help, node docs, and targeted tests. If maintainers want this outside core, core first needs a documented way for plugins to implement or augment the reserved `system.notify` node-host command.
