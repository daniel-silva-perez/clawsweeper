## Summary

Keep open. Current main still lacks Linux desktop notification delivery for the core `system.notify` node surface: the headless node host neither advertises nor dispatches that command, the CLI/docs still describe notify as macOS or non-headless behavior, and plugins cannot fill the exact gap because `system.notify` is reserved by core. The related closed #24616 covered the rejection/error experience, not Linux notification delivery.

## What This Changes

Request Linux desktop notification support for OpenClaw nodes, likely via `notify-send`/libnotify or D-Bus with graceful no-display degradation.

## Best Possible Solution

Add a focused Linux headless node-host `system.notify` implementation with desktop-session/backend detection, safe argv-only process spawning, structured unavailable results for missing display or backend, targeted tests, docs/CLI updates, and unchanged macOS/iOS/Android behavior.

Treat #24616 as related context only: it covered the error experience, not delivery. Add a Linux-focused node-host notify handler, likely in a new `src/node-host/invoke-system-notify.ts`, wire it into `src/node-host/invoke.ts`, and advertise `system.notify` from `src/node-host/runner.ts` only when the node host can support the command or with well-defined unavailable behavior.
