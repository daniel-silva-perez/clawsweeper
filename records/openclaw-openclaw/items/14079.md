## Summary

Keep open. Current main still does not implement Linux desktop notification delivery for the existing node `system.notify` surface: Linux is allowlisted at the gateway policy layer, but the headless node host neither advertises nor handles `system.notify`; CLI/docs still describe notify as mac-only or headless nodes as run/which only; and plugins cannot fill the exact gap because `system.notify` is reserved by core.

## Best Possible Solution

Implement a focused core Linux headless node-host `system.notify` path: detect a desktop session (`DISPLAY` or `WAYLAND_DISPLAY`) and a resolved `notify-send` backend, advertise `system.notify` only when delivery is available, return structured `UNAVAILABLE`/`NOT_AUTHORIZED` style failures when the backend disappears or cannot deliver, preserve macOS/iOS/Android behavior, and update CLI help, node docs, targeted tests, and changelog.

## Repair Work Prompt

Implement Linux desktop notification support for the headless node host. Current main allowlists `system.notify` on Linux but the headless node host does not advertise or handle it, so `openclaw nodes notify` cannot deliver notifications on Linux desktop sessions.

Scope:
- Add a focused core Linux node-host `system.notify` implementation, preferably via a small helper such as `src/node-host/invoke-system-notify.ts` plus colocated tests.
- Detect desktop availability with `DISPLAY` or `WAYLAND_DISPLAY` and resolve `notify-send` from the node host PATH. Advertise `system.notify` from `src/node-host/runner.ts` only when the backend is available.
- Spawn the backend without a shell, pass sanitized environment, set a timeout, and convert missing/disappearing backend or headless sessions into structured `UNAVAILABLE` errors.
- Preserve existing macOS/iOS/Android behavior and do not change `system.run` approval semantics.
- Do not add new dependencies or implement direct D-Bus/dunstify support unless it remains small, tested, and clearly within the same PR.
- Update CLI help and docs so Linux support is described accurately. Add a single-line user-facing changelog entry under the active version with `Thanks @holistic-medical-supply-base`; include `Thanks @origamium` only if the fix also materially improves the related Linux error path.

Related refs: this issue and #24616. Keep the fix owner-specific to the node-host/core `system.notify` path; do not open plugin command registration for core-reserved commands in this PR.
