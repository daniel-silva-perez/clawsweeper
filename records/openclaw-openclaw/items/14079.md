## Summary

Keep open. Current main still lacks Linux desktop notification delivery for the reserved core `system.notify` node command: Linux is allowlisted at the gateway policy layer, but the headless node host neither advertises nor dispatches `system.notify`, and the CLI/docs still describe local node notifications as macOS-only or non-headless behavior.

## What This Changes

Add Linux desktop notification support for OpenClaw node hosts, using `notify-send` or an equivalent desktop notification backend with graceful headless fallback.

## Best Possible Solution

Implement a focused core Linux headless node-host `system.notify` path with desktop-session detection, safe backend invocation, structured unavailable results for missing display/backend, docs and CLI updates, and tests proving advertisement plus invocation behavior while leaving macOS, iOS, and Android notification behavior unchanged.
