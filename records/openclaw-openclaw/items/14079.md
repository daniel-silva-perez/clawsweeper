## Summary

Keep open. Current main still does not provide Linux desktop notification delivery for the core `system.notify` node surface: Linux is allowlisted by policy, but the headless node host neither advertises nor handles the command, and docs/CLI still present notification delivery as macOS-app behavior. The related closed report addressed the error message, not Linux notification delivery.

## What This Changes

The issue requests Linux desktop notification support for `system.notify`, preferably through `notify-send`/libnotify with graceful degradation on headless Linux hosts.

## Best Possible Solution

Add a focused Linux headless node-host `system.notify` implementation with desktop-session/backend detection, safe helper invocation, structured unavailable results for missing display or backend, and docs/CLI/tests aligned with the new supported surface while preserving current macOS, iOS, Android, and Windows behavior.

## Repair Work Prompt

Fix Linux desktop notification support for the core `system.notify` node command tracked by https://github.com/openclaw/openclaw/issues/14079. Implement the fix in the core headless node-host path, not as a plugin: advertise `system.notify` on Linux-capable headless node hosts, dispatch it safely from `src/node-host/invoke.ts` or a focused helper such as `src/node-host/invoke-system-notify.ts`, and use a minimal backend such as `notify-send` only when a desktop session is available (`DISPLAY` or `WAYLAND_DISPLAY`). Do not use shell interpolation; pass all user-controlled title/body/sound/priority/delivery values as argv or structured fields. Return structured unavailable/invalid-request errors when no display or backend exists. Update CLI/docs/changelog user-facing text as needed, keeping macOS, iOS, Android, and Windows behavior unchanged. Do not relax plugin reserved-command handling for `system.notify` unless maintainers explicitly ask for a broader API change. Related context: closed #24616 fixed the misleading unsupported-command error, but did not add Linux delivery.
