## Summary

Keep open. Current main still lacks Linux desktop notification delivery for the reserved core `system.notify` node command: the headless node host does not advertise or dispatch it, CLI/docs still describe notification delivery as macOS or non-headless behavior, and the plugin registry reserves the exact command so a plugin cannot fill the gap cleanly.

## What This Changes

Add Linux desktop notification support for OpenClaw node hosts, primarily via `notify-send`, with graceful handling for headless sessions and missing notification backends.

## Best Possible Solution

Implement a focused core Linux headless node-host `system.notify` path with desktop-session detection, safe backend invocation, structured unavailable results for missing display/backend, docs and CLI updates, and tests proving advertisement plus invocation behavior without changing macOS, iOS, or Android notification behavior.
