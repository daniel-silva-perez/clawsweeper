## Summary

Keep this issue open. Current main still does not implement Linux desktop notification delivery for the existing `system.notify` / `openclaw nodes notify` surface: Linux is allowlisted at the gateway policy layer, but the headless node host does not advertise or handle `system.notify`, docs still describe headless node hosts as `system.run` / `system.which` only, and the plugin node-host API reserves `system.notify` for core so an external plugin cannot fill the exact command gap today.

## Best Possible Solution

Implement a focused core Linux headless node-host `system.notify` path: detect a desktop session (`DISPLAY` or `WAYLAND_DISPLAY`) and a resolved `notify-send` backend, advertise `system.notify` only when delivery is available, handle disappearing/unavailable backends with structured `UNAVAILABLE` errors, spawn without a shell using sanitized env and a timeout, preserve macOS/iOS/Android behavior, and update CLI help, node docs, targeted tests, and changelog.
