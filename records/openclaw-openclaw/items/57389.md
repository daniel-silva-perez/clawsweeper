## Summary

Keep open. Current main still does not accept `api_key` as an alias in `auth-profiles.json`: persisted auth-profile normalization maps only `apiKey` to `key`, while eligibility and runtime resolution still require `key` or `keyRef`. The prior related PR #57418 targeted this exact fix but was closed unmerged, so there is no shipped implementation to close against.

## Best Possible Solution

Patch `src/agents/auth-profiles/persisted.ts` so `normalizeRawCredentialEntry` maps `api_key` to canonical `key` when `key` is absent, add a regression case beside the existing `mode/apiKey` alias tests, and consider a warning or doctor check for unknown auth-profile credential fields so future mistakes are visible.
