## Summary

Keep open. Current main has fixed the unsupported Codex ACP timeout handoff, but the isolated Codex auth gap remains: the wrapper still forces `CODEX_HOME` to a generated `codex-home`, that home is populated with generated config only, and the regression test still asserts source Codex auth is not copied. The open #73943 PR references this issue with closing syntax, so this should wait for that PR or a successor to merge.

## What This Changes

Fix managed Codex ACP sessions so the isolated Codex home can use valid Codex authentication and unsupported timeout config is not sent to the Codex ACP adapter.

## Best Possible Solution

Land or replace #73943 with a narrow ACPX Codex auth fix that bridges only the required Codex `auth.json` or documents a supported isolated-home login flow, preserves generated config isolation and the current timeout guard, and updates regression tests plus the public ACP docs.
