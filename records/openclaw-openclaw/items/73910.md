## Summary

Keep open. Current main appears to have the Codex ACP timeout guard, but the isolated CODEX_HOME auth problem remains: the ACPX Codex wrapper still points Codex at a generated home that only gets OpenClaw's generated config, and an open closing PR already tracks the auth bridge work.

## What This Changes

OpenClaw-managed Codex ACP should safely reuse or document authentication for its isolated CODEX_HOME and avoid sending unsupported timeout config to Codex ACP.

## Best Possible Solution

Keep the timeout guard already on main, then review and land or replace #73943 as a narrow Codex ACP auth fix: bridge only the required auth material or document a supported isolated-home login flow, keep generated config isolation, add regression coverage, and update the ACP docs.

Yes. The source-level reproduction is clear: set a source CODEX_HOME with auth.json, call prepareAcpxCodexAuthConfig, and inspect that current main still generates an isolated codex-home/wrapper without bridging auth while tests assert no auth copy.

## Solution Assessment
