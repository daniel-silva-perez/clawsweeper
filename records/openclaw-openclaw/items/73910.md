## Summary

Keep this issue open. Current main has fixed the unsupported Codex ACP timeout handoff, but the isolated CODEX_HOME auth problem remains: the acpx Codex wrapper still forces an isolated codex-home, writes only generated config there, and tests explicitly assert source Codex auth is not copied. The open #73943 PR references this issue with closing syntax, so the issue should wait for that PR or a successor to merge.

## What This Changes

OpenClaw-managed Codex ACP should safely reuse or bridge Codex authentication for its isolated CODEX_HOME and avoid sending unsupported timeout config options to Codex ACP.

## Best Possible Solution

Keep this issue open and review #73943 or a successor. The best fix is a narrow, security-reviewed Codex ACP auth path that either bridges only the required Codex auth.json into the isolated home or provides a documented isolated-home login flow, preserves generated config isolation, keeps the current Codex timeout guard, and updates regression tests plus public ACP docs.
