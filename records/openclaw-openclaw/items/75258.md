## Summary

Keep this PR open for maintainer review. The branch adds a narrow, useful defense-in-depth filter for marked reasoning payloads before final payload assembly and streaming normalization, and current main does not have those exact guards; the related WhatsApp privacy report remains broader than this patch because unmarked reasoning-like text may still need separate investigation.

## What This Changes

The PR filters `ReplyPayload.isReasoning` items out of final reply payload assembly and streaming/tool/block text normalization, with a regression test for mixed reasoning and visible payloads.

## Best Possible Solution

Merge a narrow defense-in-depth guard for marked reasoning payloads after the changelog and CI details are resolved, while keeping the broader WhatsApp regression tracked separately until the unmarked-text and fallback-state hypotheses are confirmed or ruled out.

## Reproduction Assessment

Yes for the marked-payload case: current main can be inspected to see `buildReplyPayloads()` lacks an `isReasoning` filter, and the PR adds a focused mixed reasoning plus visible payload regression test. No for the full WhatsApp bad-state report; the related issue still lacks a safe current-main reproduction.

## Solution Assessment

Yes for the stated defense-in-depth boundary: filtering `isReasoning` before final payload assembly and streaming normalization is narrow and consistent with the existing generic dispatch suppression. It is not a complete solution for the broader related issue if reasoning arrives as unmarked ordinary text.
