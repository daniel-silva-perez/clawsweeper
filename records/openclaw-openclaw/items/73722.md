## Summary

Keep this PR open. Current main still has the narrow memory-wiki lint behavior the PR is meant to fix: lint indexes only exact extensionless relative paths and basenames, while Obsidian wikilinks are extracted raw, so title-style, slug-style, fragment-bearing, and imported source-path suffix links can still produce false-positive `broken-wikilink` warnings. The PR is a focused implementation candidate for open #73574, and the provided diff does not show an obvious security or supply-chain regression.

## Best Possible Solution

Continue review of this PR as the implementation candidate for #73574. Keep the fix in the memory-wiki plugin lint resolver, keep or add the focused regression test for title, slug, fragment, and imported source-path suffix variants, and require targeted formatter plus `pnpm test extensions/memory-wiki/src/lint.test.ts` proof before merge.
