## Summary

Keep open. Current main still lacks the PR’s full own-line `NO_REPLY` suppression, and the PR is useful source work, but it has blocking correctness issues and overlaps with the still-open #42472/#42873 work that needs maintainer consolidation.

## What This Changes

The PR adds an own-line `NO_REPLY` helper, applies it across reply normalization, directive parsing, cron delivery, subagent follow-up, gateway live/history suppression, and adds/updates focused tests.

## Best Possible Solution

Consolidate the overlapping `NO_REPLY` work into one maintained fix: add a shared cached, case-insensitive own-line/trailing silent-intent helper, apply it consistently to reply normalization, directive/outbound planning, cron delivery, subagent follow-up, gateway live projection, and chat history, and preserve media, interactive, and channelData payload semantics with focused regression tests.

## Reproduction Assessment

Yes. The current-main source and tests show the core reproduction path: `normalizeReplyPayload({ text: "File's there. Not urgent.\n\nNO_REPLY" })` still preserves the preamble, while the PR-added inline directive test fails by static trace because `parseReplyDirectives("😄 NO_REPLY")` does not strip inline tokens.

## Solution Assessment

No. The direction is right, but this PR is not the best mergeable solution as written because it needs case-insensitive matching, parser/test alignment, channelData preservation, and a rebase around the cron suppression already present on main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Align the inline-token test with parser behavior:** `src/auto-reply/reply/reply-directives.test.ts:33-36`
  - body: The added test expects `parseReplyDirectives("😄 NO_REPLY")` to remove the token, but this patch only clears text when the reply is exact-silent or own-line silent. Inline text still returns unchanged, so this test will fail unless the parser also strips inline tokens or the expectation is corrected.
  - confidence: 0.97
- **[P2] Make own-line token matching case-insensitive:** `src/auto-reply/tokens.ts:64`
  - body: Existing silent-token helpers accept mixed-case forms such as `no_reply`, and current cron tests cover `No_Reply`. The new own-line regex has no `i` flag, so `summary\nNo_Reply` can still leak on paths that rely on this helper.
  - confidence: 0.9
- **[P2] Preserve channelData when marking own-line replies silent:** `src/auto-reply/reply/reply-directives.ts:42`
  - body: Marking own-line `NO_REPLY` text as `isSilent` makes outbound planning treat the entry as droppable when there is no media, before `channelData` can make the payload sendable. ChannelData-only cards, buttons, or custom payloads can therefore be discarded instead of delivered with empty text.
  - confidence: 0.86
