## Summary

Keep this PR open, but do not merge it as-is. Current main still lacks the proposed own-line silent-token handling on several reply/gateway paths, so the work is not obsolete; however, the branch has concrete correctness issues and overlaps with other open NO_REPLY proposals that need maintainer consolidation.

## What This Changes

The PR adds an `isSilentTokenOnOwnLine` helper, applies it across reply normalization, directive parsing, cron delivery, subagent follow-up, gateway streaming/history filtering, and adds token, normalization, and directive tests.

## Best Possible Solution

Consolidate the overlapping NO_REPLY work into one maintained fix: use a shared cached, case-insensitive helper for own-line/trailing silent intent, apply it consistently to reply normalization, directive/outbound planning, gateway live/history projection, cron delivery, and subagent follow-up, and preserve media, interactive, and channelData payload semantics with focused tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Align the inline-token test with parser behavior:** `src/auto-reply/reply/reply-directives.test.ts:33-36`
  - body: The added test expects `parseReplyDirectives("😄 NO_REPLY")` to remove the token, but this patch only clears text when the reply is exact-silent or own-line silent. Inline text still returns unchanged, so this test will fail unless the parser also strips inline tokens or the expectation is corrected.
  - confidence: 0.97
- **[P2] Make own-line token matching case-insensitive:** `src/auto-reply/tokens.ts:64`
  - body: Existing silent-token helpers accept mixed-case forms such as `no_reply`, and current cron tests cover `No_Reply`. The new own-line regex has no `i` flag, so `summary\nNo_Reply` still leaks on every path that relies on this helper.
  - confidence: 0.9
- **[P2] Preserve channelData when marking own-line replies silent:** `src/auto-reply/reply/reply-directives.ts:42`
  - body: Marking own-line `NO_REPLY` text as `isSilent` makes outbound planning treat the entry as droppable when there is no media, before `channelData` can make the payload sendable. ChannelData-only cards, buttons, or custom payloads can therefore be discarded instead of being delivered with empty text.
  - confidence: 0.86
