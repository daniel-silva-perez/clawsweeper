## Summary

Keep this PR open. Current main still renders stale last-exit details for a running systemd service, while the PR’s current formatter-level fix addresses the user-visible status output without deleting shared runtime diagnostics.

## Best Possible Solution

Keep `readSystemdServiceRuntime` returning the raw systemd exit fields for restart health and onboarding diagnostics.
