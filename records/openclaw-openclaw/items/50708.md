## Summary

Keep this PR open. Current main still shows last-exit details for a running runtime, so the user-visible bug is not implemented on main; the force-pushed PR now targets the right formatter layer, but the branch is dirty against current main and needs to preserve the newer signal-label formatting when rebased.

## What This Changes

The PR changes `src/daemon/runtime-format.ts` so formatted runtime output omits `lastExitStatus` and `lastExitReason` when `runtime.status` is `running`.

## Best Possible Solution

Rebase the PR onto current main, keep the formatter-level suppression while preserving raw systemd runtime fields and signal-labeled stopped diagnostics, then add focused `runtime-format` regression coverage before landing.

## Reproduction Assessment

Yes. A runtime object with `status: "running"`, `lastExitStatus`, and `lastExitReason` will still render those fields on current main because `formatRuntimeStatus` pushes them unconditionally; live systemd is not required to verify that formatter behavior.

## Solution Assessment

Yes in direction, but not yet as a merge-ready branch. Formatter-level suppression is the narrowest maintainable fix because restart diagnostics still consume raw `lastExitStatus`; the rebase should wrap `formatLastExitStatus(...)` instead of reverting to the older raw string.
