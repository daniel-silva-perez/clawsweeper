## Summary

Keep open. Current main still renders stale systemd last-exit details for a runtime whose status is `running`, so this PR’s intended user-visible fix is not implemented on main.

## Best Possible Solution

Review and land this PR or an equivalent formatter-level fix on main. The fix should suppress `lastExitStatus` and `lastExitReason` only when the displayed runtime status is `running`, preserve raw systemd runtime fields for restart/diagnostic consumers, and add focused `src/daemon/runtime-format.test.ts` coverage for both the running suppression case and stopped/failed preservation case.
