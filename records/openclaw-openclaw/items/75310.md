## Summary

Keep open. The linked bug remains valid on current main, but this PR still has a functional gap: the proposed detector uses materialization as a completion signal even though the generated install manifest is written before the package-manager install runs, so an interrupted install can still look complete for packages without a `main` entry.

## What This Changes

The PR adds an incomplete bundled runtime-deps install detector, wires it into gateway startup and `openclaw doctor --repair`, adds regression tests, and adds a changelog entry for #75309.

## Best Possible Solution

Use a completion signal that is written only after successful bundled runtime-deps staging, or strengthen materialization so it verifies the actual loadable package surface for packages without `main`; then gate the gateway and doctor early returns on that signal with regressions that include generated manifest present, package sentinels present, and package contents incomplete.

## Reproduction Assessment

Yes. A source-level reproduction can create an install root with the generated install manifest and `node_modules/<dep>/package.json` for a package that has no `main`, omit the package contents, and observe that current scan plus this PR's helper can still treat the plan as materialized and skip repair.

## Solution Assessment

No. The affected gateway and doctor surfaces are the right place to fix this, but `isRuntimeDepsPlanMaterialized` is not a reliable completion marker because current install code writes the generated manifest before install completion.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Use a post-success completion signal:** `src/plugins/bundled-runtime-deps.ts:225-226`
  - body: `isRuntimeDepsPlanMaterialized` is not a reliable incomplete-install detector here: current install/repair writes the generated manifest before spawning the package manager, and packages without a `main` field are treated as having satisfied entry files. If an install times out after the manifest and package.json sentinels exist, this helper returns false for `installIncomplete`, so gateway startup and doctor repair can still skip the reinstall. Use a marker written only after successful staging or strengthen the package verification before relying on this gate.
  - confidence: 0.9
