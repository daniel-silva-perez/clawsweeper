## Summary

Keep open. The PR is aimed at a real, narrow regression tracked by #75309, but the proposed retained-manifest check is stale against current main: `.openclaw-runtime-deps.json` is now removed as legacy state, while generated package manifests plus installed package checks define materialization. As written, the patch would treat normal completed stage roots as incomplete and force repeated full reinstalls.

## What This Changes

The branch adds a retained-manifest-based incomplete-install detector for bundled runtime deps, wires it into gateway startup and doctor repair, adds regression tests, and adds a changelog entry for #75309.

## Best Possible Solution

Land a current-main patch that detects incomplete bundled runtime dependency materialization using the generated package/materialization contract, or a new completion marker written only after successful staging. Keep the change scoped to gateway startup and `openclaw doctor --repair`, with regression tests for both paths, and do not revive `.openclaw-runtime-deps.json` as an authoritative completion signal.

## Reproduction Assessment

Yes. A focused unit reproduction can create a stage root whose dependency metadata looks satisfied while required package contents or a completion signal are incomplete, then verify gateway startup and doctor repair do not early-return without repair. This review verified the relevant current-main early returns and materialization helpers but did not run tests.

## Solution Assessment

No. The PR targets the right surfaces, but the retained-manifest absence check is not the best fix on current main because that file is deliberately removed as legacy state. The safer fix is to use the current materialization contract or add a new post-success marker with matching install/finalize tests.
