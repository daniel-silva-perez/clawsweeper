## Summary

Keep this issue open. Current main still has no `brave.http` diagnostics flag or Brave-specific request/status/timing/cache hit/miss logging, while the existing diagnostics helper and web-search provider context make the request actionable inside the bundled Brave plugin.

## Best Possible Solution

Keep this issue open. Implement the feature in `extensions/brave` by threading the full config or a precomputed `isDiagnosticFlagEnabled("brave.http", config)` boolean into the Brave runtime, then add sanitized diagnostic logs for outbound URL/query/params, response status and duration, and cache hit/miss for both `web` and `llm-context` modes. Add focused Brave tests and document the flag in `docs/diagnostics/flags.md` and `docs/tools/brave-search.md`.
