## Summary

Keep this PR open. Current main still enforces the old ping-pong ceiling, and the PR is a plausible focused implementation, but it is not merge-ready because it leaves public generated config surfaces stale and edits localized docs that this repo no longer owns.

## What This Changes

The PR raises the configurable `session.agentToAgent.maxPingPongTurns` ceiling from 5 to 20 across runtime, validation, help text, and session/tool docs while keeping the default at 5.

## Best Possible Solution

If maintainers accept the higher cap, land one canonical update that raises the runtime clamp and validation ceiling, regenerates checked-in config schema/help/docs baseline artifacts, updates current English source docs and config reference pages, drops localized docs edits from this repo, and adds focused boundary coverage for accepting 20 and rejecting or clamping values above 20.

## Reproduction Assessment

Yes. Static current-main inspection gives a high-confidence reproduction path: a config value above 5 is rejected by `SessionSchema` and would be clamped back to 5 by `resolvePingPongTurns`; tests were not run because this review was required to stay read-only.

## Solution Assessment

No, not as currently proposed. The direction is narrow and plausible, but the branch must be rebased and completed so all public generated config surfaces and current English docs agree with the new ceiling while localized docs remain in the publish pipeline.
