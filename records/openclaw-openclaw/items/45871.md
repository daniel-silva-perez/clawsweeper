## Summary

Keep open. Current main still enforces the old 5-turn ceiling, so the proposed behavior is not obsolete, but the PR is not merge-ready because it leaves generated config surfaces/current public config docs stale and edits localized docs that this repo does not maintain.

## What This Changes

The PR raises the configurable `session.agentToAgent.maxPingPongTurns` ceiling from 5 to 20 across the runtime clamp, Zod validation, type/help text, and session/tool docs while keeping the default at 5.

## Best Possible Solution

If maintainers accept a 20-turn ceiling, land one rebased update that changes the runtime clamp and validation ceiling, regenerates checked-in config schema/help/doc baselines, updates current English source docs and config reference pages, drops localized docs edits from this repo, and adds focused boundary coverage for accepting 20 and rejecting or clamping values above 20.

Yes. Static current-main inspection gives a high-confidence reproduction path: a config value above 5 is rejected by `SessionSchema` and would be clamped to 5 by `resolvePingPongTurns`; tests were not run because this review was required to stay read-only.

## Solution Assessment
