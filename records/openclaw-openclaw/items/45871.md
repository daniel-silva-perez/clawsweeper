## Summary

Keep open. Current main still caps `session.agentToAgent.maxPingPongTurns` at 5, so this PR is not obsolete, but the proposed branch is incomplete because it does not update the generated/public config schema or the current config reference and it edits localized docs that this repo no longer maintains.

## What This Changes

The PR raises the configurable `session.agentToAgent.maxPingPongTurns` ceiling from 5 to 20 in runtime clamping, source config validation, type/help text, and session-tool docs while leaving the default at 5.

## Best Possible Solution

If maintainers accept the 20-turn ceiling, land a rebased branch that updates runtime clamping and source validation together, regenerates the checked-in config schema/docs baselines, updates the current English config reference and session-tool docs, removes localized docs edits from this repo, and adds focused boundary coverage for accepting 20 and rejecting or clamping values above 20.

## Reproduction Assessment

Yes. Static inspection of current main gives a high-confidence reproduction path: a value above 5 is rejected by `SessionSchema` and would be clamped to 5 by `resolvePingPongTurns`; tests were not run because this review was required to stay read-only.

## Solution Assessment

No, not as currently proposed. Raising the runtime and validation ceiling is the narrow technical direction if the cap is accepted, but the branch must also update generated/public config surfaces, current English docs, and i18n-policy compliance before merge.
