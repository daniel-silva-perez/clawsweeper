## Summary

Keep open. Current main still enforces the 5-turn ceiling, so the PR is not obsolete, but the branch is not merge-ready because it updates only part of the public config surface and violates the repo's localized-docs policy.

## What This Changes

The PR raises `session.agentToAgent.maxPingPongTurns` from 5 to 20 in runtime clamping, source validation, type/help text, and related docs while leaving the default at 5.

## Best Possible Solution

If maintainers accept the 20-turn cap, land a rebased branch that updates runtime clamping, source validation, generated schema/help, current English config docs, and focused boundary tests together, while removing localized-doc edits from this repository.

## Reproduction Assessment

Yes. A static reproduction path is clear on current main: configuring `session.agentToAgent.maxPingPongTurns` above 5 is rejected by `SessionSchema`, and the runtime resolver would clamp values above 5 back to 5. Tests were not run because this review was required to stay read-only.

## Solution Assessment

No, not as currently proposed. Raising the runtime and source schema ceiling is the right narrow technical direction if the product cap is accepted, but the PR also needs generated public config surfaces, current docs, i18n-policy cleanup, and boundary coverage before merge.
