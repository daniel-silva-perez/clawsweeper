## Summary

Keep open. Current main and the latest release still cap `session.agentToAgent.maxPingPongTurns` at 5, so the PR is not obsolete, but the branch should not merge as-is because it updates the source limit without regenerating checked-in config schema/help artifacts and edits localized docs that current repo policy excludes.

## What This Changes

The branch raises `session.agentToAgent.maxPingPongTurns` from 5 to 20 in the runtime clamp, Zod validation, type/help text, and session/tool docs while leaving the default at 5.

## Best Possible Solution

If maintainers accept the higher cap, land one canonical update that raises the runtime clamp and validation ceiling, regenerates checked-in config schema/help/docs baseline artifacts, updates current English source docs and config reference pages, leaves localized docs to the publish pipeline, and adds focused boundary coverage for accepting 20 and rejecting or clamping values above 20.
