## Summary

Keep open. Current main still clamps and validates `session.agentToAgent.maxPingPongTurns` at 5, so the PR is not obsolete, but the patch is not ready to merge because it misses generated/public config surfaces and edits localized docs that are out of scope for this repo.

## What This Changes

The PR raises the maximum `session.agentToAgent.maxPingPongTurns` value from 5 to 20 in runtime clamping, source validation, type/help text, and related docs while keeping the default at 5.

## Best Possible Solution

If maintainers accept the 20-turn cap, land a rebased branch that updates the runtime clamp, source validation, generated base schema/help, current English config/docs references, and focused boundary tests together, while removing localized-doc edits from this repository.

## Reproduction Assessment

Yes. A static current-main reproduction path is clear: values above 5 are rejected by `SessionSchema` and also clamped back to 5 by `resolvePingPongTurns`. Tests were not run because the review was read-only.

## Solution Assessment

No, not as currently proposed. Raising the runtime and source schema ceiling is the narrow technical direction if the cap is accepted, but the PR needs generated/public config surfaces, current docs, i18n-policy cleanup, and boundary coverage before merge.
