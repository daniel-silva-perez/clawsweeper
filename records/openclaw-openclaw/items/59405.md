## Summary

Keep open. Current main has related Telegram progress/event plumbing and current OpenAI provider docs point users toward newer OpenAI/Codex model routes, and a commenter reports a model-switch workaround. I did not find a source-backed fix, regression test, release note, or canonical duplicate proving the reported Telegram tool-result loss is solved for the original setup.

## Best Possible Solution

Keep this issue open until maintainers either reproduce it on current main or add a targeted regression/live QA case. A good fix path is to exercise a Telegram-shaped inbound session with the same tool policy and assert that exec/cat or grep output becomes model-visible and is reflected in the final reply; if the original model route is no longer supported, document or validate the supported OpenAI/Codex model selection and close only after confirming the failure is obsolete.
