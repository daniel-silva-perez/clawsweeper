## Summary

Keep open. The PR targets a real memory-core data-loss path, but the patch is still too broad: it treats every empty session store as an unsafe load and skips the orphan transcript scan even when `{}` is a valid store state.

## What This Changes

This PR adds a guard in `extensions/memory-core/src/dreaming-narrative.ts` that warns and skips dreaming orphan archival when an agent's loaded session store has no keys.

## Best Possible Solution

Keep the data-loss protection, but make it failure-aware: only suppress orphan archival when the session store read/parse was untrustworthy, while preserving normal stale transcript cleanup for a valid empty store.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: corrupt `sessions.json` returns `{}` and can make current cleanup treat all old dreaming transcripts as unreferenced; the PR regression is a valid `{}` store with an old unreferenced dreaming transcript that should still be archived.

## Solution Assessment

No. The direction is right, but `Object.keys(store).length === 0` is not the narrowest maintainable signal because an empty object is also a valid session-store result.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
