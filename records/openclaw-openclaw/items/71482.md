## Summary

Current main does not implement this PR's intended protection. `loadSessionStore()` can still return `{}` after a failed/malformed/missing read, and `scrubDreamingNarrativeArtifacts()` still proceeds with an empty reference set and can archive old dreaming transcripts. The PR should stay open, but the proposed unconditional empty-store skip likely needs revision because reviewers correctly flagged that it also disables cleanup for legitimate empty stores.

## Best Possible Solution

Keep this PR open and request a tighter memory-core fix: distinguish a failed/untrusted session-store load from a valid empty store before destructive archival, or expose loader status/throwing behavior for this cleanup path. The final patch should preserve legitimate orphan cleanup while skipping archival when the reference set is empty because the store could not be trusted, and it should add focused regression tests for malformed/unreadable store data plus a true empty-store cleanup case.
