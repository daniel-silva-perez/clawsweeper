## Summary

Keep this PR open. Current main still has the destructive-memory-core path the PR is trying to address: `loadSessionStore()` can collapse unreadable or invalid session-store data to `{}`, and `scrubDreamingNarrativeArtifacts()` then builds an empty reference set and archives old dreaming transcript files as orphans. The proposed PR is directionally useful but too broad because it skips cleanup for every truly empty store too, which would preserve stale orphan dreaming transcripts indefinitely. The safe path is a tighter load-trust/status distinction plus focused regression tests.

## Best Possible Solution

Keep this PR open and request a tighter memory-core revision: distinguish failed or untrusted session-store loads from valid empty stores before destructive orphan archival, preserve legitimate orphan cleanup for truly empty stores, and add regression tests for malformed/unreadable store data plus true empty-store orphan cleanup.
