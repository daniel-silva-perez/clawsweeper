## Summary

Keep this issue open. Current main still has strict FIFO command-lane behavior for the relevant session lanes: `sessions_send` and A2A follow-up turns are marked as `inter_session`, but that provenance is not used to demote queued agent-originated work behind later human/channel turns. The KinthAI production comment is useful supporting evidence, and the existing foreground/background priority code I found is limited to deferred context-engine maintenance, not A2A session turns.

## What This Changes

Request to prevent human/channel messages from being indefinitely delayed behind agent-originated `sessions_send` loops in per-session command lanes.

## Best Possible Solution

Add a small generic command-lane priority option, default current callers to foreground behavior, explicitly mark `sessions_send` / `inter_session` A2A turns as background, and test that foreground human/channel/control turns overtake queued background A2A work while preserving FIFO inside each tier.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to block one per-session lane entry, enqueue one or more `inter_session` background turns and then a later human/channel turn on the same session lane; current `push`/`shift` FIFO ordering will run the later human turn after the earlier agent-originated work.

## Solution Assessment

Yes. A two-tier command-lane priority keyed from trusted provenance is the narrowest maintainable fix; budget throttling and loop/back-pressure policy are useful follow-ups but should not be bundled into the first repair.

## Repair Work Prompt

Fix the per-session command-lane starvation reported at https://github.com/openclaw/openclaw/issues/70634. Implement a conservative two-tier priority in the command queue: preserve FIFO for equal-priority work, do not preempt already-active tasks, and allow later foreground human/channel/control turns to run before queued background agent-originated `sessions_send` / `inter_session` work for the same target session. Prefer a generic `CommandQueueEnqueueOptions` priority/tier seam in `src/process/command-queue.ts` and pass background priority from the trusted `sessions_send` / A2A gateway paths rather than brittle payload inspection. Add focused tests proving foreground-over-background ordering and stable background FIFO, plus gateway/tool tests proving `sessions_send`/A2A turns are marked background. Do not bundle budget throttling, token back-pressure, provider policy, or unrelated channel changes. Add a user-facing changelog fix entry crediting @ewrurwteurwU and @kinthaiofficial unless maintainers decide this should remain internal-only.
