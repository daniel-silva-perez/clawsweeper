## Summary

Keep this issue open. Current main still uses a strict FIFO command-lane queue with no source/priority/tier on queue entries, while `sessions_send` and A2A turns are already distinguishable as `inputProvenance.kind === "inter_session"` but are not demoted before entering the same session lane. The KinthAI comment is useful production confirmation, and no open PR appears to be paired with this issue.

## Best Possible Solution

Implement a conservative two-tier command-lane priority in core queue/session runtime: preserve FIFO within each tier, place human/channel/control foreground turns ahead of queued `inter_session`/`sessions_send`/A2A background turns for the same target session, and add regression coverage for both the queue primitive and gateway/session-send path.

## Clownfish Work Prompt

Implement a conservative two-tier command-lane priority for this issue. Current main enqueues all command-lane work FIFO in `src/process/command-queue.ts`; `sessions_send`/A2A work is identifiable via `inputProvenance.kind === "inter_session"` and `sourceTool`, but it is not demoted before entering the target session lane. Add an explicit queue priority/tier option or equivalent trusted boundary so foreground human/channel/control turns run ahead of queued background `inter_session`/`sessions_send` work for the same target session. Preserve FIFO order within each tier. Do not preempt an already-active model run. Do not add budget throttling/back-pressure in this PR. Add regression tests proving a later foreground turn runs before queued background A2A work while background FIFO remains stable, and that `sessions_send`/A2A paths use the background tier. Add a user-facing changelog fix entry crediting the reporter and useful KinthAI context if maintainers accept that attribution.
