## Summary

Keep this PR open for maintainer review and merge consideration. Current main still drops the nested Schema 2.0 identity shape because the Feishu parser requires a flat `open_id`, while the PR targets that remaining gap, preserves the existing token/tag/action-value guard, adds focused lifecycle coverage, and matches the maintainer guidance that this is the better base than the narrower closed alternative.

## What This Changes

The PR updates Feishu card-action parsing to normalize nested Schema 2.0 operator/event identity fields, adds lifecycle regression coverage, and expands the Feishu changelog entry with contributor attribution.

## Best Possible Solution

Land the narrow Feishu parser, lifecycle-test, and changelog update after normal maintainer validation, while keeping instant callback responses/card disabling in the separate #71677 enhancement thread.

## Reproduction Assessment

Yes. On current main, a `card.action.trigger` payload with token, record-valued `action.value`, `context.open_chat_id`, and only `operator.user_id.open_id` leaves `openId` undefined, so parsing returns null and the monitor handler drops the callback before dispatch.

## Solution Assessment

Yes. Normalizing Schema 2.0 identity fields at the Feishu monitor parser boundary is the narrowest maintainable fix because downstream card-action handling already consumes the normalized `FeishuCardActionEvent` and the PR keeps the existing malformed-payload filters.
