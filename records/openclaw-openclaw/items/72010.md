## Summary

Keep open. Current main still uses age-only stuck-session diagnostics, while this PR adds useful progress/backoff work but still leaves several real progress callbacks unmarked; the latest public PR state is also open, unmerged, and dirty against the base.

## What This Changes

The PR adds diagnostic session progress tracking and repeated stuck-warning backoff, wires progress marks into ACP and selected generic dispatch callbacks, updates config docs/generated schema text, and adjusts related diagnostics/channel/agent tests.

## Best Possible Solution

Land this PR or an equivalent patch after rebasing and making the generic dispatch progress marker cover every non-typing real progress callback, while keeping typing keepalives and reply-start lifecycle callbacks excluded and keeping ACP progress gated by `diagnostics.enabled`.

## Reproduction Assessment

Yes. Current main can be reproduced with fake timers by marking a session processing, advancing past `diagnostics.stuckSessionWarnMs`, and observing `session.stuck`; the PR-specific gap is source-reproducible with a resolver that invokes only callbacks such as `onPartialReply`, `onToolStart`, `onItemEvent`, or `onCommandOutput`.

## Solution Assessment

No. The progress/backoff design is at the right layer, but this patch is incomplete because it wraps only a subset of the real progress callbacks; the safer fix is one shared wrapper strategy for all non-typing progress callbacks in generic dispatch.
