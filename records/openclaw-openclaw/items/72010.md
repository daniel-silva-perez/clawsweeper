## Summary

Keep open. Current main does not implement this PR's intended diagnostic progress semantics: stuck-session checks are still age-based, there is no diagnostic progress marker or repeated-warning backoff state, ACP runtime events and generic reply callbacks do not refresh diagnostic activity, and docs/schema help still describe the old age-threshold behavior. The related #71969 was auto-closed, and this PR is the active replacement.

## Best Possible Solution

Keep this PR open for normal review and landing, or land an equivalent implementation that adds diagnostic session progress marking, repeated stuck-warning backoff, ACP/runtime progress wiring gated by `diagnostics.enabled`, and matching docs/generated config updates. The prior #71969 is superseded by this active PR, not by current main.
