## Summary

Keep this PR open. Current main still lacks the PR’s intended Codex PDF extraction fallback behavior: the PDF fallback context is user-message-only, both fallback call sites call it without the resolved model, and current PDF tests cover generic OpenAI fallback but not Codex instructions. The PR is a focused implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open and review it as the implementation candidate. The best path is to land this PR or an equivalent change that threads the resolved model into both PDF extraction fallback branches, adds a Codex-only `systemPrompt` when `model.api === "openai-codex-responses"`, preserves non-Codex behavior, and keeps regression coverage for both the normal and text-only Codex fallback branches.
