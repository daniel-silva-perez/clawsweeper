## Summary

Keep open. Current main still implements `isAnnounceSkip` as strict trimmed equality, so a multi-line assistant reply ending with `ANNOUNCE_SKIP` remains visible announce content. The issue is also paired with open PR #74136, which explicitly targets this behavior, so it should remain open until that PR lands or is replaced.

## What This Changes

Allow sub-agent announce suppression when a multi-line sub-agent reply ends with `ANNOUNCE_SKIP` on its own final line.

## Best Possible Solution

Review and land PR #74136, or an equivalent narrow patch, so `isAnnounceSkip` accepts the canonical exact token plus a standalone final-line `ANNOUNCE_SKIP` while preserving `REPLY_SKIP` and `NO_REPLY` semantics. Include focused helper tests and announce-flow regression coverage.
