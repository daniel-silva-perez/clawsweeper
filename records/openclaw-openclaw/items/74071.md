## Summary

Keep open. Current main still treats ANNOUNCE_SKIP as an exact trimmed reply, so multi-line sub-agent output ending with a standalone final token is still handled as visible announce content. The issue also has open PR #74136 with closing syntax for this behavior, so the right maintainer action is to review that PR or replace it, not close the issue early.

## What This Changes

The issue requests that announce suppression recognize multi-line sub-agent output ending with ANNOUNCE_SKIP as a final standalone line.

## Best Possible Solution

Review and land PR #74136, or an equivalent narrow patch, so announce suppression accepts the canonical exact token plus a standalone final-line ANNOUNCE_SKIP while preserving REPLY_SKIP and NO_REPLY semantics. Include focused helper coverage, announce-flow regression coverage, and docs/prompt alignment.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is isAnnounceSkip("DM summary block\nANNOUNCE_SKIP"), which returns false on current main because the helper compares the entire trimmed string to the token.

## Solution Assessment

Yes. The narrowest maintainable fix is to change only isAnnounceSkip to recognize a standalone final-line token, keep REPLY_SKIP strict, and cover both the helper and announce-flow paths; PR #74136 is already the active candidate for that shape.
