## Summary

Keep open. Current main and the latest release still require the entire trimmed sub-agent reply to equal `ANNOUNCE_SKIP`, while the issue is paired with open PR #74136 that explicitly closes this behavior; the issue should stay open until that PR lands or is replaced.

## What This Changes

The issue requests that sub-agent announce suppression treat a standalone final-line `ANNOUNCE_SKIP` token as skip output instead of requiring the whole trimmed output to equal the token.

## Best Possible Solution

Review and land PR #74136, or an equivalent narrow patch, so `isAnnounceSkip` accepts the canonical exact token plus a standalone final-line `ANNOUNCE_SKIP` while preserving `REPLY_SKIP` and `NO_REPLY` semantics, with focused helper tests and announce-flow regression coverage.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: `"summary\nANNOUNCE_SKIP".trim()` does not equal `ANNOUNCE_SKIP`, and both output capture and announce delivery rely on that strict helper.

## Solution Assessment

Yes for keeping this issue open. The narrow maintainable solution is a helper-level final-line check plus targeted tests and docs/prompt alignment, and the already-open PR #74136 is the active path to review first.
