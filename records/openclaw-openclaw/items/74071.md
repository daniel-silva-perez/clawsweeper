## Summary

Keep open. Current main still implements `isAnnounceSkip` as strict trimmed equality, and the subagent announce paths rely on that helper before delivering captured output. The issue is narrow, reproducible from source inspection, and not covered by the current exact/whitespace-only tests. The provided related items are only title-search overlaps and do not track this `ANNOUNCE_SKIP` final-line behavior.

## Best Possible Solution

Implement a narrow shared-token fix in `src/agents/tools/sessions-send-tokens.ts` so `ANNOUNCE_SKIP` is accepted either as the whole trimmed reply or as a standalone final line after a newline, then add focused announce/capture tests for the multi-line final-token case and keep `REPLY_SKIP` / `NO_REPLY` semantics unchanged unless deliberately reviewed.
