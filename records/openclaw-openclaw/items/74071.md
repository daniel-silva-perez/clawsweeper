## Summary

Keep open. Current main still uses strict trimmed equality for `ANNOUNCE_SKIP`, so multi-line sub-agent output ending with the token remains deliverable content. There are already open closing PRs for this bug, so the right next step is maintainer review/dedupe of those PRs, not closing this issue or queueing another fix lane.

## Best Possible Solution

Review and land one focused fix PR for this issue, then close or supersede the duplicate PR attempts. The implementation should update `isAnnounceSkip` to accept either the exact token or a standalone final non-empty line equal to `ANNOUNCE_SKIP`, preserve `REPLY_SKIP` semantics unless intentionally changed, add focused helper and announce-flow regression tests, and update the subagents docs if the final-line form is now supported.
