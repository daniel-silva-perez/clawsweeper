## Summary

Keep open. Current main still implements announce suppression as an exact trimmed-token comparison, so a multi-line sub-agent reply ending with a standalone ANNOUNCE_SKIP is still treated as visible announce content. The issue is also paired with open PR #74136 using closing syntax, so the right maintainer path is to review and land or replace that PR.

## What This Changes

Allow sub-agent announce suppression to recognize ANNOUNCE_SKIP as a standalone final line in multi-line output, not only as the entire trimmed reply.

## Best Possible Solution

Land PR #74136 or an equivalent narrow patch that updates isAnnounceSkip, adds focused token-helper and announce-flow regression coverage, and aligns the announce prompt/docs once final-line token support is accepted as contract.

## Reproduction Assessment

Yes. The source-level reproduction is isAnnounceSkip("DM Summary\nANNOUNCE_SKIP"), which returns false on current main because the helper compares the whole trimmed string to the token.

## Solution Assessment

Yes. Accepting a standalone final-line ANNOUNCE_SKIP while preserving exact-token behavior is the narrowest maintainable fix; the active linked PR is the current implementation candidate to review.
