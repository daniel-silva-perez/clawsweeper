## Summary

Keep this PR open. Current main still uses the old Telegram per-chat/per-topic sequential key and the inbound debouncer still serializes same-key work, so the PR’s central Telegram interrupt/steer delivery bypass is not implemented. The PR is useful, and a maintainer comment already says the two-layer design is sound and mergeable after rebase/check, but the branch is stale/dirty and includes unrelated release/install/script changes that need splitting or careful review before landing.

## Best Possible Solution

Keep this PR open as a useful implementation candidate. The best path is to rebase it onto current main, adapt it to the current `bot-core.ts` structure, drop unrelated commits/files, land or replace the core steer/interrupt prerequisites first, then review and test the narrow Telegram changes: active-run session lookup, conditional per-message sequential keys, and inbound debouncer bypass for immediate-delivery queue modes.
