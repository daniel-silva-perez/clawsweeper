## Summary

Keep open. Current main still does not implement per-spawn tool restrictions for `sessions_spawn`: the tool schema has no `tools`/allow/deny parameter, the execution path does not parse or forward one, child session metadata has no per-spawn policy carrier, and the subagent policy resolver only uses global `tools.subagents.tools` plus stored depth/role capabilities. The linked Claw Drive use case and closed PR #23506 remain relevant context, while open related issue #23906 is a companion request rather than a shipped replacement.

## Best Possible Solution

Keep this issue open as the canonical core API/security gap. Implement an additive `sessions_spawn` tool-policy parameter, likely `tools: { allow?: string[]; deny?: string[] }` plus consideration for `tools: "none"` from #23906. Thread it through the schema, spawn params, immutable child-session metadata, and `resolveSubagentToolPolicyForSession`; enforce deny-wins and narrow-only semantics; add focused unit/e2e coverage; update https://docs.openclaw.ai/tools/subagents.
