## Summary

Keep open. Current main still does not implement per-spawn tool restrictions for `sessions_spawn`: the schema has no `tools`/allow/deny input, the native spawn path does not parse or forward a policy, child session metadata has no policy carrier, and the subagent policy resolver still reads only global `tools.subagents.tools` plus stored depth/role capabilities. The linked Claw Drive use case and closed PR #23506 remain useful context; #23906 is a companion request, not a shipped replacement.

## Best Possible Solution

Keep this open as the canonical core API/security gap. Implement an additive `sessions_spawn` policy parameter, likely `tools: { allow?: string[]; deny?: string[] }`, while considering the `tools: "none"` shape requested in #23906. Thread it through the tool schema, spawn params, immutable child-session metadata, and `resolveSubagentToolPolicyForSession`; enforce deny-wins and narrow-only behavior; add focused unit/e2e tests; update https://docs.openclaw.ai/tools/subagents.
