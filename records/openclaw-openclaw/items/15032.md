## Summary

Current main still does not implement per-spawn tool restrictions for `sessions_spawn`. The schema, execution path, spawn params, child session metadata, subagent policy resolver, tests, and public docs all still describe only global/session-role based subagent tool policy. The linked Claw Drive use case and closed PR #23506 remain useful implementation context, while #23906 is a related open request rather than a shipped replacement.

## Best Possible Solution

Keep this open as the canonical `sessions_spawn` core API/security gap. Implement an additive per-spawn policy parameter, likely `tools: { allow?: string[]; deny?: string[] }` plus consideration for `tools: "none"`; thread it through schema parsing, `SpawnSubagentParams`, immutable child-session metadata, and `resolveSubagentToolPolicyForSession`; add focused unit/e2e coverage; update https://docs.openclaw.ai/tools/subagents.
