## Summary

Keep open. Current main still renders every eligible visible skill into the prompt using either full XML or compact XML, and compact mode keeps per-skill location blocks rather than a names-only catalog or task-matched lazy selection. Existing budgets and per-agent allowlists reduce bloat, but they do not implement the requested prompt visibility strategy, and the related prompt-cache/tool-list items in the provided context do not supersede this specific skills optimization.

## What This Changes

Request a lower-token skills prompt mode by lazy-loading skill descriptions, using a names-only skill catalog, or adding configurable skill prompt visibility.

## Best Possible Solution

Add a backwards-compatible skills prompt visibility strategy after maintainer review: keep current full/compact behavior as the default, add an explicit opt-in names-only or stricter compact mode if desired, update config schema/help/docs, and cover prompt rendering, reports, snapshots, limits, and skill-selection guidance in tests.
