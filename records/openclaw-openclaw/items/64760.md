## Summary

Keep open. Current main still has compact formatting, count and character budgets, and per-agent skill allowlists, but it does not implement task-matched lazy loading, a names-only skill catalog, or a configurable prompt visibility mode. The reporter’s follow-up correctly identifies existing mitigations, but those mitigations do not satisfy the requested optimization.

## What This Changes

Request to reduce per-turn skills overhead in the system prompt by lazy-loading or otherwise shrinking the available skills catalog for users with many installed skills.

## Best Possible Solution

Add a backwards-compatible core skills prompt visibility strategy after maintainer product review. A conservative end state would keep the current full/compact behavior as the default, add an explicit opt-in names-only or stricter compact mode, update config schema/help/docs, and cover prompt rendering, snapshots, reports, per-agent limits, and skill-selection guidance in tests.
