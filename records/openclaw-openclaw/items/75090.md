## Summary

Keep this PR open: it addresses a real current-main bundled-skill bug, but the proposed service-account example now tells agents to run `op vault list`, which is not in 1Password's documented service-account command set; `op whoami` or `op read` is the safer documented probe. ([developer.1password.com](https://developer.1password.com/docs/service-accounts/use-with-1password-cli/))

## What This Changes

This PR rewrites the bundled `1password` skill and its get-started reference to choose direct `op` execution or tmux by auth mode, and adds an Unreleased changelog fix entry for #52540.

## Best Possible Solution

Land the auth-mode-aware bundled skill update after replacing the unsupported service-account `op vault list` probe with a documented `op whoami` or `op read` check, keeping the fix limited to the skill, reference note, and changelog.

## Reproduction Assessment

Yes. Current main plainly contains the tmux-only instruction, and #52540 gives a concrete desktop-app reproduction where direct `op vault list` succeeds while the tmux-wrapped call fails; the PR finding is reproducible by comparing the new service-account example against 1Password's documented supported commands.

## Solution Assessment

No, not as written. The overall direction is the narrowest maintainable fix for the reported bug, but the service-account example should use documented service-account-safe commands before merge.
