## Summary

Keep open. Current main still lacks the requested teardown routing, while this PR is a focused implementation with no blocking correctness or security finding; maintainer review should decide the public event documentation and merge readiness.

## What This Changes

The PR adds a read-scoped `socket.drain` gateway event for session teardown, routes it to session event and message subscribers, exposes message-subscriber lookup on `GatewayRequestContext`, adjusts teardown slow-consumer handling, and adds gateway coverage.

## Best Possible Solution

Merge this PR or an equivalent focused patch after documenting or explicitly accepting the new event contract, preserving the read-scoped routing, teardown delivery semantics, and targeted tests.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: on current main, `sessions.messages.subscribe` clients are tracked separately from session event subscribers, while teardown lifecycle broadcasts only use the event subscriber set and `socket.drain` is not registered as an allowed gateway event.

## Solution Assessment

Yes, with documentation follow-through. Reusing the existing subscriber registries and session mutation helper is the narrow maintainable approach; the safer remaining step is to document the event if clients are expected to rely on it.
