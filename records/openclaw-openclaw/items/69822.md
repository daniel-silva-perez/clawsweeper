## Summary

Keep this PR open. Current main does not implement the proposed `socket.drain` path for message-only session subscribers: `socket.drain` is not registered as a gateway event or scope-guarded broadcast event, `GatewayRequestContext` does not expose message-subscriber lookup, and `emitSessionsChanged` still only broadcasts `sessions.changed` to `sessions.subscribe` clients with `dropIfSlow: true`. The PR is a focused implementation candidate for the closed related feature request #63374 and a cleaner successor to closed unmerged PR #65539, but it needs maintainer review/rebase rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase it onto current main, review the `socket.drain` event-scope and slow-buffer behavior explicitly, then either land a focused gateway implementation with the proposed regression tests or replace it with an equivalent maintainer-owned patch. Current main still lacks the central behavior, so cleanup closure would be premature.
