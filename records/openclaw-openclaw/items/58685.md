## Summary

Keep this PR open. Current main has not implemented the intended human-readable warning behavior for `openclaw channels list`, and the PR is not obsolete. The useful direction is real, but the current PR needs iteration: the reviewed code path still renders Claude `user:profile` failures as usage snapshot errors rather than thrown loader failures, matching the later review comments in the PR discussion.

## Best Possible Solution

Keep the PR open for normal review iteration. The best fix is to classify usage snapshot errors in the human-readable `channels list` rendering path as well as thrown loader failures, show Claude-specific guidance only for explicit `user:profile` scope signals, preserve raw diagnostics for generic failures, add regression coverage for both thrown and snapshot error paths, and drop unrelated file changes. Coordinate the separate JSON-output robustness concern with #59472 rather than folding it into this PR.
