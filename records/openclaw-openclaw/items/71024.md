## Summary

Keep this PR open. Current main still lacks a Mattermost read-only account inspector: Mattermost status inspection can still fall back to strict `resolveMattermostAccount`, which dereferences `botToken` through `normalizeResolvedSecretInputString` and throws for unresolved SecretRefs. The PR remains a relevant, narrow implementation for the linked Mattermost status crash, and the fetched PR diff shows the Greptile env-token precedence suggestion has been applied. Security review found no workflow, dependency, package-script, lockfile, or third-party execution changes in the PR diff; the sensitive surface is limited to secret status metadata reporting.

## Best Possible Solution

Review and land this PR or an equivalent Mattermost-owned fix. Keep runtime Mattermost token resolution strict, add the read-only inspector through the shared Mattermost config adapter so setup/status fallback paths see it, preserve `botTokenStatus` in snapshots, and keep the focused Mattermost account/status regression tests.
