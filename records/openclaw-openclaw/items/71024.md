## What This Changes

This PR adds a Mattermost read-only account inspector, wires it into the Mattermost config adapter, preserves `configured_unavailable` token metadata in status snapshots, and adds unit/contract tests for unresolved SecretRef bot tokens.

## Best Possible Solution

Land this PR or an equivalent Mattermost-owned fix after normal CI/review, keeping runtime token resolution strict while read-only status uses `inspectAccount` to report configured-but-unavailable credentials. Track the Telegram variant separately in #74832.

## Reproduction Assessment

Yes. The report has a concrete config shape, and current main statically reproduces the failure path: Mattermost has no `inspectAccount`, status inspection falls back to strict `resolveAccount`, and strict SecretRef resolution throws for unresolved refs.

## Solution Assessment

Yes. The proposed fix is the narrowest maintainable direction because it adds the missing Mattermost read-only inspector in the plugin boundary, mirrors existing Slack/Discord/Telegram patterns, and avoids weakening runtime credential resolution.
