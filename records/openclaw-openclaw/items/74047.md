## Summary

Keep open. Current main has partial safeguards for stale gateway service paths and current GPT-5.5 model resolution, but it does not yet resolve the reported split user/system install plus newer-path `/v1/chat/completions` timeout. The related open PRs are adjacent and unmerged, and there is no current source, docs, or test evidence proving the newer service-path reliability problem fixed.

## What This Changes

The issue asks OpenClaw to clarify and harden upgrades where the CLI and gateway service run from different install paths, especially when moving from an older system install to a newer user install makes `/v1/chat/completions` unreliable.

## Best Possible Solution

Keep the issue open while maintainers review the related service diagnostics/restart work, decide the supported user-vs-system install migration policy, and collect gateway logs or a minimal reproduction for the newer-install `/v1/chat/completions` timeout. The end state should make the active CLI install, service entrypoint, systemd scope, model route, and health/rollback path visible before users promote a newer model through the gateway.

## Reproduction Assessment

Unclear. The report has concrete machine state and rollback evidence, but there is not yet a high-confidence current-main reproduction for the newer-install `/v1/chat/completions` timeout without the reporter's gateway logs, config, service unit, and request trace.

## Solution Assessment

No current-main solution is complete. The existing doctor/update safeguards are useful but partial; the safer path is diagnostics plus explicit migration guidance before any automatic service repointing behavior is expanded.
