## Summary

Keep open. Current main has useful partial guardrails for stale gateway service entrypoints, update-time service refresh, OpenAI Codex model metadata, and some newer Codex transport failures, but it does not yet resolve the reported split user/system install policy or prove that repointing the gateway service to the newer install no longer makes `/v1/chat/completions` time out.

## What This Changes

The issue asks OpenClaw to clarify and harden upgrades when a gateway service is pinned to an older system install while a newer user install exists, especially because repointing the service to the newer install made `/v1/chat/completions` unreliable.

## Best Possible Solution

Keep this issue open until OpenClaw has a clear, documented migration path for split user/system installs, status/doctor output that exposes the active service entrypoint/package root/version, and either a reproduced fix for the newer-install `/v1/chat/completions` timeout or enough gateway logs to route the failure to update/doctor, systemd supervision, or the OpenAI-compatible runtime endpoint.
