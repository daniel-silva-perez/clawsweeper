## Summary

Current main still does not provide the real-time bidirectional mirror described here. Control UI connects as WebChat and sends chat messages with `deliver: false`; the Gateway keeps those turns on the internal WebChat surface and explicitly prevents WebChat clients from inheriting external channel delivery routes. Feishu inbound messages are routed to Feishu-scoped agent sessions, while the Control UI only reloads chat history when the `session.message` event matches the currently selected session key. The related #36640 context is useful, but it does not close this Feishu-specific report or settle the product decision around dashboard-to-channel mirroring.

## Best Possible Solution

Keep this open for maintainer product triage. Reproduce on current main with Feishu DM, group, and topic sessions while recording the active Control UI session key, resolved Feishu session key, gateway logs, and `session.message` events. If bidirectional mirroring is desired, add an explicit opt-in Control UI delivery/mirror flow for Feishu-backed sessions with tests for inbound refresh and dashboard-to-Feishu delivery. If mirroring is intentionally out of scope, document that boundary in Control UI, WebChat, channel routing, and Feishu docs before closing.
