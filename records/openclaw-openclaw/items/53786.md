## Summary

Keep this open. Current main still preserves the WebChat/internal delivery boundary, so Control UI chat sends do not deliver to Feishu, and Feishu inbound visibility depends on the resolved Feishu session matching the active Control UI session rather than a true bidirectional mirror. The related #36640 Telegram/WebChat report is useful context, but it does not settle this Feishu-specific behavior.

## Best Possible Solution

Keep this open for maintainer product triage. Reproduce on current main with Feishu DM, group, and topic sessions while recording the active Control UI session key, resolved Feishu session key, gateway logs, and session.message events. If bidirectional mirroring is desired, add an explicit opt-in Control UI delivery/mirror flow for Feishu-backed sessions with tests for Feishu inbound-to-active-session refresh and dashboard-to-Feishu delivery. If mirroring is intentionally out of scope, document that boundary in Control UI, WebChat, channel routing, and Feishu docs before closing.
