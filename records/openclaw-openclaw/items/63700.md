## Summary

Keep this issue open. Current main still does not implement repeatable `--file` input for `openclaw infer model run`: the capability metadata and registered command are prompt-only, local execution sends only a text message through the lean completion path, and gateway execution calls `agent` without `attachments`. The recent local `model run` work addresses adjacent raw/lean completion behavior, but not the requested multimodal file workflow.

## Best Possible Solution

Keep the issue open and implement the core CLI capability. Add repeatable `--file` to `model.run` metadata and Commander registration, read and MIME-validate supported files, forward image files through the local lean completion path or an explicitly supported image-content adapter, pass gateway files as `agent.attachments`, add focused CLI tests for local/gateway forwarding and unsupported files, and update `https://docs.openclaw.ai/cli/infer`. Define the audio model-run content contract separately before documenting `wav`/`mp3` support.
