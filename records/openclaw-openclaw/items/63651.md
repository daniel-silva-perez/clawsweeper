## Summary

Keep this PR open. Current main still has the Windows schtasks restart stdout writes that the PR targets, and the stale-PID retry path can still invoke the restart implementation again. The PR addresses a plausible gateway bug, but it should not be closed as cleanup because main does not implement the intended fix; it also needs a focused revision because the provided PR context shows unrelated browser CDP/SSRF changes mixed into the Windows restart-output patch.

## Best Possible Solution

Keep this PR open and ask for a focused gateway-only revision: rebase onto current main, split any browser CDP/SSRF work into its own review path, preserve exactly one human completed-restart success message, de-duplicate stdout across stale-PID retries, and add a targeted regression test for the Windows schtasks retry/output behavior.
