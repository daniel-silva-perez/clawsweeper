## Summary

Keep open. Current main does not implement this PR’s intended log-level downgrade: pre-handshake closes still warn unless they match the existing SwiftPM helper filter, while `probeGateway` still creates a `PROBE`-mode `GatewayClient` and calls `client.stop()` after settling. The provided PR is focused and its second commit addresses the review concern about wildcard bind hosts, so the remaining best path is maintainer review/landing or an equivalent tested change.

## Best Possible Solution

Keep this PR open for maintainer review, or land an equivalent change in `src/gateway/server/ws-connection.ts` that recognizes internal loopback health-probe closes with code 1000, normalizes wildcard gateway bind hosts to the loopback probe origin, downgrades only that case to debug, and adds focused regression coverage.
