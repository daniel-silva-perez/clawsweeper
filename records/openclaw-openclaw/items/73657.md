## Summary

Current main still matches the reported failure path: the CLI runner keeps the raw CLI model string in `agentMeta.model`, the reply runner persists that value into session state, and `runWithModelFallback` adds the primary candidate without alias resolution even though configured fallback entries are resolved through the alias index. A persisted bare alias such as `sonnet` can therefore remain a primary `anthropic/sonnet` candidate on a later run.

## Best Possible Solution

Keep this issue open and fix the current code path. The narrow fix should resolve the primary candidate in `src/agents/model-fallback.ts` through `resolveModelRefFromString` using the same alias index used for fallbacks, with a regression test for a persisted bare alias such as `sonnet` resolving to the configured `anthropic/claude-*` model. A complementary improvement is to have the CLI runner persist canonical model identity instead of the raw CLI alias.
