## Summary

Keep open. Current main still forwards strikethrough content from injected context files, so the PR is addressing a real gap, but the latest patch still strips literal `~~...~~` inside backtick-fenced Markdown code blocks and can drop valid context examples before they reach the model.

## What This Changes

This PR updates `sanitizeContextFileContentForPrompt()` to remove Markdown and HTML strikethrough content from injected context files and adds system prompt tests for inline, multiline, HTML-attribute, and tilde-fence cases.

## Best Possible Solution

Revise the sanitizer so it removes actual rendered strikethrough while preserving Markdown code fences and inline code, preferably by using an existing Markdown-aware tokenizer or by first masking code spans/fences. Keep the HTML attribute coverage and add regression tests for backtick fences, inline code, tilde fences, multiline strikethrough, and attributed HTML tags before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

- **[P2] Preserve backtick-fenced code examples:** `src/agents/system-prompt.ts:85`
  - body: The new `~~` regex runs over raw Markdown, so it also removes literal `~~...~~` inside triple-backtick fences. Context files can contain fenced examples, and Markdown does not render content inside those fences as strikethrough, so this can silently drop valid prompt context. Please skip code fences/inline code or use a Markdown-aware pass before removing strikethrough spans.
