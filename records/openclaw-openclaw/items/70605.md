## Summary

Keep open. The PR targets a real current-main gap and follows useful discussion from #47668, but the proposed raw regex sanitizer still deletes literal strikethrough examples inside Markdown code spans and fenced code. Because these context files are user-authored prompt input, that would silently remove valid instructions or examples before they reach the model.

## What This Changes

The PR changes `sanitizeContextFileContentForPrompt()` to remove Markdown and HTML strikethrough content from injected context files and adds system-prompt tests for inline, multiline, tilde-fence, and attributed-HTML cases.

## Best Possible Solution

Make the prompt sanitizer remove rendered strikethrough content while preserving Markdown code regions, keep the existing heartbeat cleanup and blank-line collapse behavior, retain HTML attribute support, and cover inline code plus both fenced-code styles in focused tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve code regions before stripping strikethrough:** `src/agents/system-prompt.ts:83`
  - body: The new regex runs over raw context text, so `~~...~~` inside inline backticks or fenced code is removed even though Markdown renders it as code, not strikethrough. Context files can contain literal examples or instructions with these markers, so this silently deletes valid prompt text. Mask code regions, for example with `findCodeRegions()`, before removing rendered strikethrough.
  - confidence: 0.91
