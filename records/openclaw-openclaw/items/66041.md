## What This Changes

The PR makes Telegram treat the default `group-mentions` ack reaction scope as `group-all` when a group is already configured with `requireMention: false`, and adds a regression test plus a Telegram docs note.

## Best Possible Solution

Keep the Telegram-owned effective-scope fix and regression test, add a single-line `CHANGELOG.md` entry under Unreleased Fixes with contributor credit, then run the focused Telegram test and the normal changed gate before merge.

## Solution Assessment

Mostly yes. The code fix is the narrowest maintainable place to align Telegram ack behavior with Telegram's own mention gate, but the PR is not merge-ready until the missing changelog entry is added.
