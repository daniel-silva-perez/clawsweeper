## Summary

Keep this PR open for maintainer review. Current main does not implement the proposed segment-offset behavior, and the observable Control UI duplication path still exists: the gateway emits full-snapshot chat deltas, the UI commits pre-tool stream text before the tool card, and the next full-snapshot delta can be rendered as the active stream after that committed segment. The PR is not safe to close as implemented, and the branch still needs protocol/UI judgment because generic WS, TUI, ACP, and node/session consumers rely on monotonic full-snapshot chat deltas.

## What This Changes

The PR adds per-run gateway segment offset tracking, threads that state through chat abort/runtime/maintenance helpers, and emits segment offset hints in chat delta messages while preserving final full-text payloads.

## Best Possible Solution

Land a current-main fix that preserves full-snapshot chat deltas for generic subscribers while making the Control UI render only the uncommitted post-tool segment after a tool card, backed by focused gateway and UI regression tests for text -> tool -> text runs.
