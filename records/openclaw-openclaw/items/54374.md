## Summary

Keep this PR open. Current main does not implement the PR’s segment-boundary behavior or an equivalent WebChat-side dedupe, and the current gateway/UI path can still combine a committed pre-tool stream segment with a later full-snapshot delta. The PR is a focused bug fix with useful review discussion, but it still needs maintainer review, regression coverage, and protocol/client compatibility decisions before it can be considered obsolete or safe to close.

## Best Possible Solution

Keep the PR open for review or rework. The best path is to land an implementation that preserves full-snapshot chat deltas for generic subscribers, gives WebChat a tested way to render only post-tool text after a committed pre-tool segment, and adds focused regression coverage across `src/gateway/server-chat.agent-events.test.ts` plus a UI/tool-stream test for the rendered segment ordering.
