## Summary

Keep open. Current main still has full-snapshot gateway chat deltas with no segment-offset state, and the Control UI path can still render committed pre-tool text plus the later full active stream. The PR contains useful direction, but its latest diff only adds a gateway hint without a Control UI consumer or UI regression tests, so it is not a complete fix as-is.

## What This Changes

The PR branch adds per-run gateway chat segment-offset state, threads cleanup through abort and maintenance paths, and includes a `segmentOffset` hint in chat delta message payloads while preserving full final assistant text.

## Best Possible Solution

Land a current-main fix that keeps generic chat deltas as full snapshots while giving the Control UI a documented or locally derived way to render only the post-tool active segment after a tool card, with focused gateway and UI tests for text-to-tool-to-text runs.
