## Summary

Keep open. Current main still emits full-snapshot chat deltas after tool boundaries and the Control UI still renders committed pre-tool stream segments alongside the next active full snapshot, so the requested behavior is not implemented on main; the PR also needs maintainer review because the current branch is gateway-only and protocol-adjacent while the visible fix likely needs UI consumer coverage.

## What This Changes

The PR branch adds per-run gateway chat segment-offset state and cleanup plumbing, then includes segment-offset hints in chat delta messages while preserving full final assistant text.

## Best Possible Solution

Land a current-main fix that keeps generic chat deltas as full snapshots, gives the Control UI a documented or locally derived way to render only the post-tool active segment after a tool card, and covers text-to-tool-to-text runs in both gateway and UI tests.
