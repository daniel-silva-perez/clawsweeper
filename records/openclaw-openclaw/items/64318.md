## Summary

Keep this PR open. Current main still does not ship the unified Amazon Polly, Transcribe, and Nova Sonic plugin, the plugin architecture can support a vendor multi-capability plugin, and the discussion contains an explicit maintainer keep-open signal with unresolved docs, CI, runtime, protocol, and security-review blockers.

## Best Possible Solution

Keep this PR open as the active Amazon voice/speech integration candidate. The contributor should rebase against current main, add provider docs/changelog/labeler metadata, fix disabled Transcribe behavior, align Nova Sonic with AWS' documented bidirectional event schema and OpenClaw realtime audio contracts, resolve audio codec/sample-rate and reconnect/close review comments, and return with green changed-surface CI. If maintainers decide AWS voice/speech should not be bundled, make that an explicit ClawHub/plugin-placement decision after product and security review.
