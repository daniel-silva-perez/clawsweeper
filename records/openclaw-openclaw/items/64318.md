## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Follow Nova Sonic event framing:** `extensions/amazon/nova-sonic/bridge.ts:126-190`
  - body: The bridge sends a custom `{ event: { type: "sessionStart" } }` envelope, then `audioInput` chunks, and closes with only `sessionEnd`. AWS Nova Sonic requires the structured bidirectional flow with `sessionStart`, `promptStart`, `contentStart` carrying `promptName`/`contentName`, content events, `contentEnd`, `promptEnd`, and `sessionEnd`, so this stream shape will not be interpreted as a valid voice session. citeturn0search1.
  - confidence: 0.94
- **[P1] Honor the realtime audio format before conversion:** `extensions/amazon/nova-sonic/bridge.ts:67-73`
  - body: `sendAudio` always treats inbound audio as 8 kHz G.711 u-law, but `RealtimeVoiceBridgeCreateRequest.audioFormat` can also be PCM16/24 kHz for the Talk relay. A PCM caller will be decoded as u-law garbage, and telephony input is converted to 16 kHz PCM without resampling from the 8 kHz source, so both supported sinks can receive the wrong audio shape.
  - confidence: 0.91
- **[P1] Do not close the call while reconnecting:** `extensions/amazon/nova-sonic/bridge.ts:267-277`
  - body: On stream errors, `processOutputStream` starts `attemptReconnect()` and returns, but the `finally` block immediately emits `onClose("error")`. Voice-call treats that close as a hangup, so transient Bedrock stream failures end the call before the reconnect attempt can succeed.
  - confidence: 0.9
- **[P2] Stop registering Transcribe when disabled:** `extensions/amazon/transcribe/media-understanding-provider.ts:53-58`
  - body: `transcribe.enabled` is parsed but never used in the returned provider. Media-understanding providers have no `isConfigured` hook, so keeping `capabilities`, `autoPriority`, and `transcribeAudio` present leaves Amazon Transcribe auto-selectable and callable even when config disables it.
  - confidence: 0.95
- **[P2] Merge plugin defaults into Nova Sonic resolution:** `extensions/amazon/nova-sonic/realtime-voice-provider.ts:33-55`
  - body: The plugin entry passes `api.pluginConfig`, but this builder discards it as `_pluginConfig`. In normal realtime provider resolution, an empty provider config therefore ignores `plugins.entries.amazon.config.novaSonic.enabled`, `region`, `voice`, and `model`, so the documented single Amazon config block does not actually control this provider.
  - confidence: 0.84
- **[P2] Use a valid PCM rate for Polly telephony:** `extensions/amazon/polly/speech-provider.ts:206-212`
  - body: The telephony path requests `outputFormat: "pcm"` with sample rate `22050`, but Amazon Polly only accepts `8000` or `16000` for PCM. Normal voice-call synthesis through this provider will fail validation instead of returning audio. citeturn1search0.
  - confidence: 0.94
- **[P2] Use the actual FLAC sample rate:** `extensions/amazon/transcribe/stt.ts:95-97`
  - body: The FLAC pass-through path still sends `MediaSampleRateHertz: 16000` for every FLAC file. Amazon Transcribe requires the specified sample rate to match the audio, so common 44.1/48 kHz FLAC inputs will be rejected or transcribed incorrectly. citeturn0search5.
  - confidence: 0.86
