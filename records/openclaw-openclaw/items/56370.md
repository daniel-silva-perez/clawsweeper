## Summary

Current main implements the reported default-agent routing behavior: `openclaw agent --to` without `--agent` now derives the session key and store from the configured default agent, while explicit `--agent` remains explicit. The related merged PR #72414 carried forward the earlier #56453 analysis and the current checkout has regression coverage. The fix appears main-only in the local tag set: latest tag is `v2026.4.26`, and the changelog entry is under `2026.4.27` with no containing release tag yet.

## Best Possible Solution

Close this issue as implemented on current main. Keep the default-agent routing fix and regression coverage from #72414, and track broader channel delivery-context lookup problems separately under their existing issues.

## Evidence
