## Summary

Keep open. Current main still leaves the reported DeepSeek V4 disabled-thinking/extra_body path unresolved: boolean `params.thinking: false` is not resolved to off, the DeepSeek wrapper decides whether to strip before `extra_body` can overwrite `thinking`, and two open closing PRs already target the remaining behavior.

## What This Changes

The issue asks OpenClaw to prevent DeepSeek V4 Pro multi-turn failures when `extra_body.thinking.type=disabled` and prior assistant history contains `reasoning_content`.

## Best Possible Solution

Review #74403 and #74418 together and land one narrow fix that derives the effective disabled-thinking decision before the final DeepSeek V4 payload is sent, covers `off`/`none`/`disabled`/boolean false plus `extra_body.thinking.type=disabled`, and adds regression coverage for the late extra_body override and multi-turn replay behavior.
