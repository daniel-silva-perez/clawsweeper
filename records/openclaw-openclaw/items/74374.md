## Summary

Keep open. Current main still leaves the reported disabled-by-extra_body path unresolved: DeepSeek V4 replay repair only strips `reasoning_content` when the resolved `thinkingLevel` is `off` or `none`, while the reported `params.thinking=false` shape is ignored and `extra_body` can overwrite `thinking` after the DeepSeek wrapper has added `reasoning_content`. Two open PRs explicitly target this issue, so the issue should remain open until a fix merges.

## What This Changes

Fix DeepSeek V4 Pro disabled-thinking multi-turn requests so replayed assistant `reasoning_content` is stripped or made consistent when `extra_body` disables thinking.

## Best Possible Solution

Land one consolidated fix, or choose between #74403 and #74418 after review, so DeepSeek V4 derives a single effective disabled-thinking state from resolved thinking level and final request payload, strips replayed `reasoning_content` whenever the final payload disables thinking, and has regression coverage for the reported config shape.
