## Summary

Keep this issue open. Current main still implements `/usage` as a per-response token/cache/cost footer, with `full` adding only the session key. It does not include the active model, cumulative context used/window/percentage, a `/usage context` mode, or an inherited `agents.defaults.responseUsage` default. Open PR #69618 uses closing syntax for this issue and remains unmerged, so the issue should stay open while that implementation candidate or a smaller replacement is reviewed.

## Best Possible Solution

Keep the issue open while #69618, #48635, or a smaller replacement is reviewed. The maintainable path is to enrich `/usage full` with the active model plus cumulative context used/window/percentage, add documented inherited `agents.defaults.responseUsage` semantics while preserving session overrides, and cover formatter, runner composition, slash command behavior, config schema/docs, and reply/session tests.
