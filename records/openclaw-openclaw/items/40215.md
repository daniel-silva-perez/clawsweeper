## Summary

Current main still does not implement the requested normal channel `/usage full` footer: the formatter only accepts per-response token/cache/cost inputs, the runner only appends that line plus the session key in `full` mode, and `agents.defaults` has no `responseUsage` default. Open PR #69618 explicitly closes this issue and remains the implementation candidate, so this issue should stay open until that PR or a replacement lands.

## Best Possible Solution

Keep this issue open while maintainers review and either land #69618 or replace it with a focused patch. The implementation should enrich the normal channel `/usage full` footer with active model plus cumulative context used/window/percentage, add documented inherited `agents.defaults.responseUsage` semantics while preserving session overrides, and cover formatter, reply-runner, slash-command, config schema/docs, gateway/TUI/Web UI, and session tests.
