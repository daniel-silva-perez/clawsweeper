## Summary

Keep this issue open. Current main still does not put active model plus cumulative context used/window/percentage into the normal channel `/usage full` reply footer, and it still has no inherited `agents.defaults.responseUsage` default. Open PR #69618 explicitly closes this issue and remains an implementation candidate, so the issue should stay open until that PR or a replacement lands.

## Best Possible Solution

Keep this issue open while maintainers review and either land #69618 or replace it with a focused patch. The concrete path is to enrich the normal channel `/usage full` footer with active model plus cumulative context used/window/percentage, add documented inherited `agents.defaults.responseUsage` semantics while preserving session overrides, and cover the formatter, reply runner, slash command, config schema/docs, gateway patch, TUI/Web UI, and session tests.
