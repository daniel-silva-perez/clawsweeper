## Summary

Keep this issue open. Current main still appends only per-response token/cache/cost data in the `/usage` footer, with `full` adding only the session key. It does not surface active model, cumulative context used/window/percentage, or an inherited `agents.defaults.responseUsage` default. Open PR #69618 remains an unmerged closing implementation candidate, so the issue should not be closed before that lands or is replaced.

## Best Possible Solution

Keep the issue open while #69618, #48635, or a smaller replacement is reviewed. The maintainable path is to enrich `/usage full` with active model plus cumulative context used/window/percentage, add documented inherited `agents.defaults.responseUsage` semantics while preserving session overrides, and cover formatter, runner composition, slash command behavior, config schema/docs, and reply/session tests.
