## Summary

Keep this issue open. Current main still does not enrich the normal channel `/usage full` response footer with active model plus cumulative context used/window/percentage, and it still lacks an inherited `agents.defaults.responseUsage` default. Open PR #69618 explicitly targets this request with closing syntax and remains an implementation candidate, not merged behavior.

## Best Possible Solution

Keep this issue open while maintainers review and either land #69618 or replace it with a smaller focused patch. The concrete product path is to enrich the normal channel `/usage full` footer with active model and cumulative context used/window/percentage, add documented inherited `agents.defaults.responseUsage` behavior while preserving session overrides, and cover formatter, reply-runner, slash command, config schema/docs, gateway patch, TUI/Web UI, and session tests.
