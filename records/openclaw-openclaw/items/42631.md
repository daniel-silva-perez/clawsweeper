## Summary

Keep this issue open. Current main still has no supported job-level model/provider override for cron jobs that run as sessionTarget="main" with payload.kind="systemEvent". The protocol only allows text on systemEvent payloads, normalization removes model/provider-like fields from system events, main-session cron execution only enqueues text and wakes heartbeat, and the closed related PR #42708 was not merged.

## Best Possible Solution

Keep this issue open. The best path is an additive cron contract for per-job model/provider selection on main-session systemEvent jobs, with allowed-model validation and transient runtime plumbing through schema/types, normalization, main-session cron execution, heartbeat/system-event dispatch, CLI, Control UI, docs, and regression tests. PR #42708 is useful prior design context, but it is not shipped.

## Risks / Open Questions

- Closing now would hide a still-current feature gap for users who need cheaper recurring main-session cron runs without editing session metadata.
- A future implementation needs to keep the override job-scoped/transient so it does not silently mutate the shared interactive main-session model selection.
- The change crosses protocol, normalization, runtime heartbeat/system-event plumbing, CLI, Control UI, docs, and validation tests, so a partial fix could create inconsistent behavior across surfaces.
