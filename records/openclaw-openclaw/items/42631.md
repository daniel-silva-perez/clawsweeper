## Summary

Current main still does not support job-level model/provider overrides for `sessionTarget: "main"` cron jobs with `payload.kind: "systemEvent"`. The contract still keeps model selection on `agentTurn` payloads, normalization prunes model/provider fields from `systemEvent`, and the main-session execution path only enqueues system-event text plus heartbeat wake metadata. The related implementation PR #42708 is closed unmerged, so it is useful design context but not a shipped fix.

## Best Possible Solution

Keep this issue open. The best path is an additive cron contract for per-job model/provider selection on `main + systemEvent` jobs, validated against the existing allowed-model behavior and passed transiently through the main-session cron wake/system-event path without mutating unrelated interactive session defaults. Use #42708 as prior design context, but do not treat it as shipped.

## Risks / Open Questions

- Closing now would hide a still-current feature gap for cheaper recurring main-session cron runs without manual session metadata edits.
- A future implementation should avoid persisting a cron job's transient model/provider choice into shared main-session state where it could affect later interactive turns.
- The fix is cross-surface: protocol schema, cron runtime types, normalization, main-session system-event/heartbeat plumbing, CLI, UI, docs, and regression tests need to change together.
