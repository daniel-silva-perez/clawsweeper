## Summary

Keep this issue open. Current main still does not support job-level model/provider selection for cron jobs that run as `sessionTarget: "main"` plus `payload.kind: "systemEvent"`; the existing contract keeps model selection on `agentTurn`/isolated-style jobs, and the prior implementation PR #42708 is closed unmerged.

## Best Possible Solution

Keep this issue open. The best implementation is an additive cron contract for per-job model/provider selection on `main + systemEvent` jobs, with allowed-model validation and transient plumbing through normalization, protocol schema, runtime types, main-session system-event/heartbeat execution, CLI, UI, docs, and regression tests. PR #42708 is useful prior design context, but not shipped.
