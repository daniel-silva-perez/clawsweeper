## Summary

Keep this PR open. Current main still does not implement the smoke-only Control UI build skip: the Dockerfile runs `pnpm ui:build` unconditionally, and install-smoke passes only `OPENCLAW_EXTENSIONS=matrix` to the root Dockerfile smoke image. The PR remains a plausible focused CI performance change, but it needs a rebase and careful scoping against the current split fast/full install-smoke workflow because full install-smoke reuses `/app/dist` from the Docker image for package/update smoke paths.

## Best Possible Solution

Keep the PR open as a focused performance candidate. The best path is to rebase it onto current main, keep `OPENCLAW_SKIP_UI_BUILD` re-declared inside the Docker build stage, pass the skip arg only to the intended fast Docker smoke image build, and preserve real Control UI asset coverage for package/update and release-adjacent lanes.

## Risks / Open Questions

- The PR is stale against current main: the Dockerfile now uses Node 24 base images and runs `pnpm qa:lab:build` after `pnpm ui:build`, while install-smoke now has split fast/full paths and tags one matrix-extension image as both root and extension smoke images.
- If `OPENCLAW_SKIP_UI_BUILD` is applied to the full install-smoke image, package/update smoke paths may pack placeholder Control UI assets because they copy `/app/dist` from `openclaw-dockerfile-smoke:local` and run `npm pack --ignore-scripts`.
- The build arg should remain smoke-only and should not be used for release, package acceptance, published Docker images, or any lane intended to validate real Control UI assets.
