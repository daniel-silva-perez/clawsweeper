## Summary

Keep this PR open. Current main still does not implement the smoke-only Control UI build skip: the Dockerfile runs `pnpm ui:build` unconditionally, and the install-smoke workflow does not pass `OPENCLAW_SKIP_UI_BUILD`. The PR remains a plausible focused CI performance change, but it needs a rebase against the current install-smoke workflow shape and careful scoping so placeholder Control UI assets cannot be reused by package/update smoke paths.

## Best Possible Solution

Keep this PR open as a focused performance candidate. The best path is to rebase it onto current main, keep `OPENCLAW_SKIP_UI_BUILD` re-declared inside the Docker build stage, pass the skip arg only to the intended fast Docker smoke image build if maintainers still want this optimization, and preserve real Control UI asset coverage for package/update and release-adjacent lanes.
