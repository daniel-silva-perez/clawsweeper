## Summary

Keep this PR open. Current main still only backs off and suspends Telegram sendChatAction for 401 errors; it does not implement the PR's transient network / 429 / 5xx cooldown path. The PR is a focused implementation candidate and its latest branch also addresses the bot-review concern by continuing to reject during cooldown so the shared typing start guard can trip.

## Best Possible Solution

Keep this PR open for maintainer review and landing, or land an equivalent narrow patch in `extensions/telegram/src/sendchataction-401-backoff.ts` that adds transient network / 429 / 5xx cooldown, preserves the existing 401 suspension semantics, keeps rejected starts visible to the shared typing guard, and ships the focused regression tests. After it lands on main, use it as the canonical fix for the remaining transient sendChatAction reports such as #56096.
