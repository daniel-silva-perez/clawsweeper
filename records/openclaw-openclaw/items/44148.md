## Summary

Current main still does not implement WhatsApp human-takeover pause behavior. The WhatsApp config type/schema has no `humanTakeover` surface, manual outgoing `fromMe` direct messages are filtered as non-actionable access-control noise, and the auto-reply path has no account/conversation pause guard before dispatching replies.

## Best Possible Solution

Keep this issue open and implement the behavior in the bundled WhatsApp plugin: add a typed/schema-backed `channels.whatsapp.humanTakeover` config surface, detect non-gateway manual outgoing direct `fromMe` messages per account/conversation, store a pause-until timestamp, and skip WhatsApp auto-replies while the pause is active. Update config metadata/docs and focused regression tests; define group behavior explicitly rather than changing it incidentally.
