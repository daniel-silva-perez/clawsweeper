## Summary

Keep this PR open. Current main at 230f7122dd5cd8a37a55e671254bd9be8278f6e8 still does not implement the PR's intended recommend-reset escalation path: there is no compaction guard config surface, no runtime guard/escalation wiring, no transcript-tail scorer/validator modules, and no post-compaction recommendation diagnostic. The PR is also draft and stacked on open prerequisite PRs (#48278, #48293, #48312, #48335, #48350), with useful reviewer discussion noting that reset recommendations need a recovery/restore path; main now has successor transcript/checkpoint support, but that is adjacent rather than the requested diagnostic escalation.

## Best Possible Solution

Keep this PR open for maintainer review or supersession by a consolidated loop-aware compaction guard implementation. The concrete path is to resolve or replace the prerequisite guard PR chain, decide how recommend-reset should relate to existing compaction checkpoints/successor transcripts, then land a focused internal diagnostic path with tests and no automatic reset by default.
