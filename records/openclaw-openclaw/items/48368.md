## Summary

Keep this PR open. Current main still does not implement the proposed recommend-reset escalation path: the compaction config has no `guard` or `escalation` surface, safeguard runtime wiring has no guard/escalation fields, and the post-compaction path does not run a validator or emit a recommend-reset diagnostic. The PR is also a draft stacked on still-open prerequisite PRs, with useful review discussion about needing a recovery/restore point; main now has adjacent compaction checkpoint and successor transcript support, but that does not replace this PR's internal diagnostic escalation.

## Best Possible Solution

Keep this PR open for maintainer review or replacement by a consolidated loop-aware compaction guard implementation. The concrete path is to resolve the prerequisite guard PR chain, decide how recommend-reset should use existing compaction checkpoints/successor transcripts, then land a focused internal diagnostic path with tests and no automatic reset by default.
