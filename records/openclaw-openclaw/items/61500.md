## Summary

Keep open. Current main still only routes node-host shell execution through `system.run`/`system.which`; there is no `system.read`/`system.write` implementation, advertisement, or built-in read/write/edit routing to a node filesystem. This is a core node protocol and file-tool routing gap, not a ClawHub-only item, because plugin node-host commands exist but the standard file tools are not wired to use plugin/node commands for filesystem operations.

## Best Possible Solution

Keep this issue open and handle it as a core design/implementation task. The likely path is to add a typed node filesystem protocol surface such as `system.read`/`system.write` plus any needed `stat`/`mkdir`/`access` semantics, advertise and policy-gate those commands, add node-backed file operation factories for read/write/edit, preserve gateway-owned workspace/bootstrap files with explicit routing, and cover the behavior with protocol, node-host, agent-tool, and docs tests.
