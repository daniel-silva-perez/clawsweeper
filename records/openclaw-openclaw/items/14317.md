## Summary

Keep open. Current main still does not implement Slack `groupPolicy: "private"`: the shared policy type/schema reject it, Slack routing only evaluates `open`, `disabled`, and `allowlist`, and the Slack docs/metadata expose only the existing three values. The prior maintainer review comment that kept this open remains consistent with the current checkout.

## What This Changes

The issue requests a Slack `groupPolicy: "private"` mode that allows private Slack channels, blocks unlisted public channels, and still permits explicitly allowlisted public channels.

## Best Possible Solution

If maintainers approve the trust boundary, implement a Slack-scoped private group policy through the config type/schema/metadata, normal message routing, slash-command routing, security audit guidance, public Slack docs, and focused tests while preserving the missing-provider fail-closed fallback and documenting Slack Connect caveats.
