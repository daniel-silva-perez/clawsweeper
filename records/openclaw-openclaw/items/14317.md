## Summary

Keep open. Current `main` still does not accept or implement Slack `groupPolicy: "private"`: the shared config contract, strict schema, generated metadata, public Slack docs, normal message routing, slash-command routing, and exposure audit all still model only `open`, `disabled`, and `allowlist`. The request is concrete, but it changes Slack access-control and audit semantics, so it needs maintainer/security approval before an automated fix changes the trust boundary.

## What This Changes

Add a Slack-only `groupPolicy: "private"` mode that allows invited private channels while blocking public channels unless explicitly allowlisted.

## Best Possible Solution

After maintainer/security approval, implement the Slack-scoped private policy end to end: config type/schema/metadata support, normal and slash routing that carries Slack privacy metadata, public-channel allowlist fallback, exposure-audit classification, Slack docs with Slack Connect caveats, and focused regression tests while preserving the current fail-closed default.
