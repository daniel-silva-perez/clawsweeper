## Summary

Keep this issue open. Current main still does not implement Slack `groupPolicy: "private"`: the shared policy type/schema, Slack config schema, generated metadata, runtime channel routing, slash routing, docs, and tests still model only `open`, `allowlist`, and `disabled`. The report's default-open motivation is now obsolete because current Slack docs/runtime use `allowlist` as the fail-closed default, but the requested private-channel trust boundary remains missing.

## Best Possible Solution

Keep this issue open and implement a Slack-scoped `groupPolicy: "private"` option in the bundled Slack plugin. The narrow fix should accept the value in Slack config types/schema/generated metadata, evaluate Slack `is_private` for message and slash routing, allow private channels where the bot is present, preserve explicit public-channel allowlist and explicit deny behavior, update security audit/fix guidance so `private` is not treated like `open`, and document/test Slack Connect caveats. Do not change the current Slack default from `allowlist` as part of this issue without maintainer product approval.
