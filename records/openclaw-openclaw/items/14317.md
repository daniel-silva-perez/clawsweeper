## Summary

Keep open. Current main still rejects Slack `groupPolicy: "private"` in the shared config contract, exposes only the existing Slack channel-policy modes in docs and metadata, and routes normal and slash-command channel messages through an open/disabled/allowlist-only gate. The requested behavior also changes Slack access-control and exposure-audit semantics, so it needs maintainer/security approval before implementation.

## What This Changes

Add a Slack `groupPolicy: "private"` mode that allows invited private channels, blocks public channels by default, permits explicit public-channel allowlist entries, and adjusts security-audit treatment.

## Best Possible Solution

If approved, implement a Slack-scoped private policy end to end: config type/schema/metadata support, normal and slash-command routing that preserves Slack privacy metadata, explicit public-channel allowlist fallback, security-audit guidance, public docs with Slack Connect caveats, and focused regression tests.
