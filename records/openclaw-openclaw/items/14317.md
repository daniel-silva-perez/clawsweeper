## Summary

Keep open. Current main still does not implement Slack `groupPolicy: "private"`: the shared config contract rejects it, Slack docs and generated metadata expose only the existing modes, normal and slash-command routing use the current open/disabled/allowlist helper, and the security audit only classifies open group policy exposure.

## What This Changes

Request to add a Slack-only `groupPolicy: "private"` mode that allows private channels by membership while requiring explicit allowlist entries for public channels.

## Best Possible Solution

If maintainers approve the Slack trust boundary, implement a Slack-scoped private policy end to end through config type/schema/metadata, normal and slash-command routing using Slack privacy metadata, explicit public-channel allowlist fallback, security-audit guidance, docs with Slack Connect caveats, and focused tests.
