## Summary

Keep this issue open. Current main still does not implement Slack `groupPolicy: "private"`: shared config types/schema, generated Slack config metadata, Slack message routing, slash-command routing, security audit guidance, docs, and tests still model only `open`, `allowlist`, and `disabled`. Slack discovery code already captures private-channel metadata, but the access-control paths do not consume it.

## Best Possible Solution

Keep this issue open for maintainer judgment. If approved, implement a Slack-scoped `groupPolicy: "private"` that accepts the config value, preserves the current fail-closed `allowlist` default, uses Slack private-channel metadata for normal message and slash-command routing, still permits explicitly allowlisted public channels and explicit denies, updates security audit guidance so `private` is not treated like `open`, and documents/tests Slack Connect caveats.
