## Summary

Keep open. Current main still does not accept or implement Slack `groupPolicy: "private"`: the shared type/schema omit it, Slack docs and generated metadata expose only the existing modes, normal and slash-command routing use the open/disabled/allowlist-only gate, and exposure auditing only classifies the current `open` case. The request is concrete, but it changes Slack access-control semantics and needs maintainer/security product approval before implementation.

## What This Changes

Add a Slack `groupPolicy: "private"` mode that responds in invited private channels, ignores public channels unless allowlisted, and updates Slack routing, config, docs, tests, and security-audit behavior.

## Best Possible Solution

After maintainer/security approval, implement a Slack-scoped private group policy end to end: config type/schema/metadata support, normal and slash-command routing that carries Slack privacy metadata, public-channel allowlist fallback, exposure-audit classification, public Slack docs with Slack Connect caveats, and focused regression tests.
