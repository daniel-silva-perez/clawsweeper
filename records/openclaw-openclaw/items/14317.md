## Summary

Keep open. Current main still rejects and omits Slack `groupPolicy: "private"` at the shared config contract, generated metadata, docs, normal channel routing, slash-command routing, and exposure-audit surface. The request is concrete, but it changes Slack authorization and audit semantics, so it should wait for maintainer/security approval rather than being auto-closed or sent straight to an automated repair lane.

## What This Changes

Add a Slack-only `groupPolicy: "private"` mode that allows private Slack channels the bot is invited to while requiring explicit allowlist entries for public channels.

## Best Possible Solution

After maintainer/security approval, implement the Slack-scoped private policy end to end: config type/schema/metadata support, route authorization that carries Slack privacy metadata for normal and slash paths, public-channel allowlist fallback, exposure-audit classification, Slack docs with Slack Connect caveats, and focused regression tests while preserving fail-closed behavior for missing Slack config.
