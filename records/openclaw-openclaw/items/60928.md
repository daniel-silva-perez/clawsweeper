## Summary

Keep open. Current main still lacks a CLI-side manual skills snapshot bump, so the PR targets a real remaining gap after the earlier stale-snapshot work, but the proposed patch should not merge as-is because it starts the persistent skills watcher from a one-shot CLI path without a teardown.

## What This Changes

The PR adds a force-refresh option to the agent command path, calls skills refresh helpers before snapshot reuse, opts the CLI entrypoint into the refresh, and extends the agent command test to assert the watcher and manual bump calls.

## Best Possible Solution

Land a corrected CLI-only fix that manually bumps the workspace skills snapshot version for trusted local `openclaw agent` runs before cached snapshot reuse, without starting a persistent watcher from the one-shot CLI path, and keep ingress/gateway refresh semantics explicit and unchanged.

## Reproduction Assessment

Yes. The PR body gives a concrete `skills.load.extraDirs` plus fixed-session CLI flow, and current main source confirms the CLI path can read version `0` and reuse an existing snapshot without any local bump source. I did not execute it because this review was required to stay read-only.

## Solution Assessment

No. The manual version bump is the right narrow mechanism, but starting `ensureSkillsWatcher` in the one-shot command path is not the best implementation; avoid the watcher there or guarantee teardown, and add a negative ingress/no-watcher assertion.
