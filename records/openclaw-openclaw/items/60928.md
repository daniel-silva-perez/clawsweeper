## Summary

Keep open. Current main has the general stale-snapshot rebuild path, but it does not implement this PR's CLI-specific one-shot manual skills snapshot bump. The PR also has unresolved functional review concerns around starting a persistent watcher in `openclaw agent` and broadening ingress behavior, so it is not safe to close as implemented or merge as-is.

## Best Possible Solution

Keep this PR open, but ask for a rebase and a narrower implementation: in the trusted `agentCommand` CLI path, call `bumpSkillsSnapshotVersion` from the lightweight `refresh-state` runtime before reading `getSkillsSnapshotVersion`, avoid starting `ensureSkillsWatcher` for one-shot CLI runs, keep ingress behavior unchanged, and add a focused regression test that asserts the CLI path bumps while ingress does not.

## Evidence

- Closing as implemented would lose a still-unimplemented CLI-specific refresh path: current main refreshes only after the in-process snapshot version changes.
