## Summary

Keep this PR open. Current main still caps the QMD vector status probe at 5000 ms, so configured `memory.qmd.limits.timeoutMs` values above that are ignored. The PR head removes that cap and adds focused regression coverage. The checkpoint-export concern has been narrowed after review: current main already centralizes exact compaction-checkpoint exclusion in the session artifact helpers, and the PR now adds coverage that valid session IDs containing `checkpoint` are still exported.

## Best Possible Solution

Keep this PR open, rebase or refresh the branch, update the PR body to match the narrowed diff, and land the QMD status-timeout fix with its regression tests. Keep checkpoint transcript exclusion centralized in the session artifact/session-file helpers rather than reintroducing a broad export-layer `.checkpoint.` substring filter.
