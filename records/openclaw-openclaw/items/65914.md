## Summary

Keep open. Current main still caps the QMD vector status probe at 5000 ms, while this PR carries the focused code change that makes the probe honor `memory.qmd.limits.timeoutMs`; the earlier broad checkpoint export regression has been removed from the branch and replaced with narrower coverage. The remaining concerns are normal maintainer review, one nonblocking test-strengthening finding, and exact-head CI failures that should be resolved or proven unrelated before merge.

## What This Changes

The PR removes the 5000 ms cap from QMD vector status probes, adds QMD manager regression tests for timeout and checkpoint-named session export behavior, and adds an unreleased changelog entry.

## Best Possible Solution

Land the focused QMD status-timeout fix after changing the timeout regression test to use a value above 5000 ms and obtaining clean current changed-gate/CI proof; keep checkpoint filtering centralized in the session artifact helpers and leave the separate CLI file-count inconsistency tracked by #69759.
