## Summary

Keep open. The later discussion narrowed the reproducible failure to background exec completion when `agents.defaults.heartbeat.every = "0m"`; current `main` still enqueues an `exec-event` wake on background exec exit but `runHeartbeatOnce` returns `skipped: disabled` before it inspects that event when the heartbeat interval resolves to null. The related PR #67913 attempted this exact fix but was closed unmerged, and current tests only cover scoped exec wakes with enabled heartbeat intervals.

## Best Possible Solution

Keep this issue open and land a focused fix that lets targeted `exec-event` wakes bypass the per-agent interval-disabled check while preserving the global heartbeat enable switch. Add regression coverage for background exec notify-on-exit with `heartbeat.every: "0m"`, including captured delivery context, then re-test the Codex/Telegram scenario and separately investigate the assistant-phase/planning-only path if status-only behavior remains.
