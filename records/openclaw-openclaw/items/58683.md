## Summary

Keep this PR open. Current main still only supports fixed `heartbeat.every` plus optional `activeHours`; the requested `heartbeat.schedule[]` daypart intervals are not present in types, schema, docs, or runner behavior. The linked feature requests #58581 and #51844 are closed in the provided context, but current source does not show the feature shipped, so this PR is still a plausible implementation candidate rather than obsolete.

## Best Possible Solution

Keep the PR open and review it as the implementation candidate: rebase onto current main, integrate `heartbeat.schedule[]` with the current phase-based heartbeat scheduler, keep the config schema/types/docs aligned, and resolve or explicitly accept the DST and boundary-transition review findings before merge.
