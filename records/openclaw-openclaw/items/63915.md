## Summary

Keep open. Current main still routes the bundled ZAI image provider through the generic image-model completion path, with no merged ZAI-specific fix for the reported `zai/glm-4.6v` `Request was aborted` failure. The related #70886/#70915 work is an open, unmerged implementation candidate for a newer 404 variant, and #66608 was closed as a duplicate of this issue, so this issue should remain open until a fix lands and proves it covers the abort symptom.

## Best Possible Solution

Keep this issue open while maintainers review the ZAI vision-model fix candidate. The likely product/code path is in the bundled ZAI plugin/media-understanding boundary: prove `zai/glm-4.6v` imageModel calls succeed through the `image` tool for the relevant ZAI endpoint configuration, add regression coverage for the selected route and streaming behavior, then close this issue as implemented or explicitly consolidate it with the canonical ZAI vision routing issue after the fix merges.
