## Summary

Keep this issue open. Current main still routes `zai/glm-4.6v` image understanding through the generic OpenAI-compatible image-model path, with no merged ZAI-specific endpoint, streaming, or abort handling for the reported `Request was aborted` failure. The duplicate report context strengthens the reproduction, and the related route-fix PR is still only an open/unmerged candidate for a related 404 variant.

## Best Possible Solution

Keep this issue open while maintainers review the ZAI vision-model route/streaming path. The likely fix belongs in the bundled ZAI media-understanding provider or a generic provider-facing image payload seam if multiple providers need it: prove `zai/glm-4.6v` works through the `image` tool with the reported endpoint setup, add regression coverage for the selected endpoint/stream behavior, then close this issue as implemented or explicitly supersede it with the canonical ZAI vision-routing issue after a fix merges.
