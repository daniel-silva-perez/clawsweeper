## Summary

Keep this PR open. Current main still has only the older secrets-audit finding codes and summary fields, still classifies SecretRef-capable config targets from the resolved runtime config, and does not compare against the authored pre-substitution config needed to distinguish exact `${ENV_VAR}` templates from literal plaintext.

## Best Possible Solution

Keep this PR open for normal maintainer review. The best path is to land or revise the focused implementation so exact `${ENV_VAR}` templates on SecretRef-capable config fields produce explicit migration guidance, mixed strings remain plaintext findings, CLI JSON/text summaries and docs include the new count/code, and the `config.env` plaintext-source edge is either guarded or split into a tracked follow-up before merge.
