## What This Changes

Ensure WebChat and openclaw-tui outbound assistant replies fire the expected message lifecycle hooks, especially `message_sent` and `before_message_write`, with parity to hook-aware delivery paths.

## Best Possible Solution

Land a narrow Gateway chat hook coverage fix before the broader outbound observer work: route `chat.send` final replies through a hook-aware wrapper or equivalent local helper, emit plugin `message_sent` and internal `message:sent` once for final delivery, run `before_message_write` before Gateway-injected assistant transcript appends, and cover WebChat plus Gateway TUI with focused regression tests.
