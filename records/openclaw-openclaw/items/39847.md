## Summary

Keep this issue open. Current main still lacks the requested shared outbound safety net: the shared outbound planner parses raw reply text before any inbound-metadata stripping, delivery projection does not add stripping, and the Discord adapter sanitizer only removes Discord runtime scaffolding. There are also open same-author implementation PRs with closing syntax for this issue, so the issue should remain open until one is reviewed and merged or explicitly closed.

## Best Possible Solution

Keep this issue open as the canonical bug. Review and land one focused source-level fix in `src/infra/outbound/payloads.ts` that strips inbound metadata before directive parsing/formatting in the shared outbound plan, with regression coverage in `src/infra/outbound/payloads.test.ts` for Conversation/Sender metadata while preserving MEDIA directives, reply tags, audioAsVoice, silent replies, presentation/interactive payloads, and channelData. Then close or supersede the duplicate open fix PRs with that rationale.
