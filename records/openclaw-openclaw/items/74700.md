## Summary

Keep this PR open. It addresses a real ACP run-session cleanup gap that current main still has, but the proposed fixed 24h fallback cleanup can terminate legitimate no-timeout ACP runs, so the patch should not merge unchanged.

## What This Changes

The PR adds lifecycle-triggered cleanup for ACP `mode: "run"` runtimes, passes the initialized runtime close handle into dispatch-failure cleanup, and adds regression tests in `src/agents/acp-spawn.test.ts`.

## Best Possible Solution

Revise the PR so ACP run runtimes close promptly on confirmed terminal lifecycle events and dispatch failures, while any fallback cleanup either proves terminal task state or derives from an explicit timeout; keep persistent `mode: "session"` behavior unchanged and leave broader orphan recovery to #68916.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve no-timeout ACP runs in fallback cleanup:** `src/agents/acp-spawn.ts:1031-1033`
  - body: The fallback timer always calls cleanup after 24h when no lifecycle terminal event is seen. `sessions_spawn` documents and forwards `runTimeoutSeconds: 0` as the ACP no-timeout path, so a legitimate long-running run can be closed while still active. Gate the fallback on terminal task state or an explicit timeout instead of a fixed wall-clock close.
  - confidence: 0.86
