## Summary

This issue should stay open. Current main has native backup archive creation and verification, plus docs for those two commands, but it still does not provide the requested native restore/import/list/cloud/encryption/scheduled backup workflow or component-aware cron/session restore semantics. The discussion already identifies useful community workarounds, especially `openclaw-workspace-sync` and `keepmyclaw`, and an open restore PR, but those are not a merged replacement for the first-party umbrella.

## Best Possible Solution

Keep this issue open as the native umbrella for first-party backup restore/import and disaster recovery. The next concrete path is maintainer review of the narrower restore and migration/profile portability work, then an accepted design for dry-run validation, overwrite/rollback rules, component-aware cron/session semantics, cloud/encryption scope, and docs. Community options such as `openclaw-workspace-sync` and `keepmyclaw` should remain documented workarounds unless maintainers explicitly move the remaining native scope to ClawHub/plugin API discussion.

## Risks / Open Questions

- Native restore/import for archives is security-sensitive: path traversal, overwrite policy, permissions, secret handling, and rollback behavior need explicit design and tests before landing.
- Community options cover much of the disaster-recovery workflow, but they do not settle whether OpenClaw should provide first-party session-history and cron merge semantics.
- The umbrella is broad, so an implementation should be split into narrower PRs or canonical follow-ups rather than attempting cloud storage, encryption, scheduling, restore, cron merge, and session import all at once.
