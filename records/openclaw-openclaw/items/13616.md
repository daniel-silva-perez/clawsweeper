## Summary

Keep this issue open. Current main has a native local backup archive creator and verifier, but it still does not provide the requested first-party restore/import/list/cloud/encryption/scheduled backup workflow or component-aware cron/session restore semantics. Community options linked in the discussion, including openclaw-workspace-sync and Keep My Claw, are useful workarounds, but the issue remains a native backup/restore umbrella and the latest maintainer-side comment already kept it open.

## Best Possible Solution

Keep this issue open as the native umbrella for first-party backup restore/import and disaster recovery. Maintainers should coordinate the narrower restore, migration/profile portability, and per-agent backup work, decide what belongs in core versus plugin/ClawHub APIs, then land a reviewed restore design with dry-run validation, overwrite/rollback rules, path safety, secret handling, and explicit cron/session component semantics. Community tools such as openclaw-workspace-sync and Keep My Claw should remain documented workarounds unless maintainers decide the remaining native scope moves out of core.
