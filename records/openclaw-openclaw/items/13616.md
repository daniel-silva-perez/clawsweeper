## Summary

Keep open. Current main has first-party local backup archive creation and verification, but it still lacks the requested native restore/import/export/list/cloud/encryption/scheduled lifecycle and component-aware cron/session restore semantics. The issue discussion also has useful community workarounds, but the latest maintainer-side review kept the native umbrella open.

## What This Changes

Request adds a built-in OpenClaw backup and restore lifecycle for config, cron jobs, workspace files, session history, approvals, pairing state, selective export/import, cloud storage, encryption, scheduling, and disaster recovery.

## Best Possible Solution

Keep this as the native umbrella for a reviewed first-party backup lifecycle built on the existing create/verify archive and migration/plugin seams, with safe extraction, dry-run validation, explicit component modeling for cron and sessions, secret/credential policy, overwrite and rollback rules, cloud/encryption/scheduling choices, and docs that point to community tools as optional workarounds.
