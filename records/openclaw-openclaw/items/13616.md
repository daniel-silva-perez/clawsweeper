## Summary

Keep open. Current main has first-party local backup archive creation and verification, but it still lacks the requested native restore/import/export/list/cloud/encryption/scheduled-backup lifecycle and component-aware cron/session restore semantics. The linked community plugin and ClawHub skill are useful workarounds, but they do not make the remaining native umbrella obsolete, and maintainer-side context already kept this open for scope and safety decisions.

## What This Changes

The issue requests built-in OpenClaw backup and restore commands for config, cron jobs, session history, workspace state, optional cloud storage, encryption, scheduling, and disaster recovery.

## Best Possible Solution

Keep this as the native backup lifecycle umbrella, built on the existing create/verify archive and migration/plugin seams, with safe extraction, dry-run validation, explicit cron/session component modeling, secret and credential policy, overwrite/rollback rules, cloud/encryption/scheduling choices, and docs that point to community tools as optional workarounds.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The current-main verification path is high-confidence: source, tests, docs, changelog, and issue discussion all show create/verify only, with no native restore/import/list/cloud/encryption/scheduling surface.

## Solution Assessment

Unclear as a single implementation because the requested lifecycle is broad and security-sensitive. The maintainable path is to settle archive format and restore safety first, then split cloud, encryption, scheduling, cron merge, and session-history semantics into reviewed follow-ups.

## Security Review

Status: needs_attention

Summary: No patch is under review, but the requested native restore/import workflow is security-sensitive and needs maintainer-approved safety rules before implementation.

Concerns:

- **[medium] Define restore archive safety before native import:**
  - body: A native restore command would consume archives and overwrite OpenClaw state, credentials, cron jobs, session stores/transcripts, pairing data, and workspace files. Any core implementation needs path containment, symlink handling, ownership/permissions, dry-run, conflict, rollback, and partial-failure rules before merge.
  - confidence: 0.92

  - body: The request includes credentials-adjacent state plus cloud storage and encryption. Maintainers should define what is excluded, how keys are managed, whether credentials are restorable, and how docs steer users away from plaintext cloud artifacts.
  - confidence: 0.86
