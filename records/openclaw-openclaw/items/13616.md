## Summary

Keep open. Current main provides first-party local backup archive creation and verification, but it still lacks the requested native restore/import/export/list/cloud/encryption/scheduled-backup workflow and component-aware cron/session restore semantics. The community options mentioned in the discussion are useful workarounds, and the maintainer-side review already kept this native umbrella open.

## What This Changes

Request a first-party backup/restore utility for OpenClaw config, cron jobs, session history, migration, and disaster recovery.

## Best Possible Solution

Keep this as the native umbrella for first-party backup restore/import and disaster recovery. The desired end state is a reviewed restore design built on the existing create/verify archive and migration seams, with dry-run validation, path traversal protection, secret handling, overwrite/rollback rules, and explicit cron/session component semantics, while documenting community tools as optional workarounds.
