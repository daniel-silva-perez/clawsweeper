## Summary

Keep open. Current main has first-party local backup archive creation and verification, but it still lacks the requested native restore/import/export/list/cloud/encryption/scheduled-backup lifecycle and component-aware cron/session restore semantics. Recent maintainer-side discussion also kept the native umbrella open, while the community tools linked in the comments remain useful workarounds rather than a full core resolution.

## What This Changes

Add native backup/export/import/restore/list workflows for OpenClaw config and state, cron jobs and run history, session history, cloud storage, encryption, and scheduled backups.

## Best Possible Solution

Keep this as the native umbrella for a reviewed first-party backup lifecycle built on the existing create/verify archive and migration/plugin seams, with safe extraction, dry-run validation, explicit component modeling for cron and sessions, secret/credential policy, overwrite and rollback rules, cloud/encryption/scheduling choices, and docs that point to community tools as optional workarounds.
