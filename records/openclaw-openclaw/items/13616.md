## Summary

Keep open. Current main has first-party local backup archive creation and verification, but it still lacks the requested native restore/import/export/list/cloud/encryption/scheduled-backup lifecycle and component-aware cron/session restore semantics. The linked `openclaw-workspace-sync` plugin and `keepmyclaw` skill are useful workarounds, and open related restore/design items remain active rather than making this umbrella obsolete.

## What This Changes

Add built-in backup/restore commands for OpenClaw config, cron jobs, skills/workspaces, session history, approvals, pairing state, cloud storage, encryption, scheduling, and disaster recovery.

## Best Possible Solution

Keep this as the native backup lifecycle umbrella, built on the existing create/verify archive and migration/plugin seams, with safe extraction, dry-run validation, explicit cron/session component modeling, secret and credential policy, overwrite/rollback rules, cloud/encryption/scheduling choices, and docs that point to community tools as optional workarounds.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The current-main verification path is high-confidence: source, tests, generated CLI index, docs, changelog, and related reports all show create/verify only, with no native restore/import/list/cloud/encryption/scheduling surface.

## Solution Assessment

Unclear as a single implementation because the requested native lifecycle is broad and security-sensitive. The maintainable path is to settle restore safety and archive format first, likely through review of #59280/#41111, then split cloud, encryption, scheduling, cron merge, and session history semantics into reviewed follow-ups.

- Community tools cover important disaster-recovery workflows today, but they do not prove that OpenClaw core supports component-aware cron/session import or the requested built-in CLI lifecycle.
