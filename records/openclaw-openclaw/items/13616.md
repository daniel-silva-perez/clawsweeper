## Summary

Keep open. Current main has first-party local backup archive creation and verification, but it still does not provide the requested native restore/import/export/list/cloud/encryption lifecycle or component-aware cron/session restore semantics. The discussion has useful community workarounds, but a recent maintainer-side review explicitly kept the native umbrella open and the remaining restore scope is security-sensitive.

## What This Changes

The issue requests a built-in backup/restore lifecycle for OpenClaw config, cron jobs, skills/workspace files, session history, cloud storage, encryption, scheduling, and disaster recovery.

## Best Possible Solution

Keep this as the native umbrella for a reviewed first-party backup lifecycle built on the existing create/verify archive and migration/plugin seams, with safe extraction, dry-run validation, explicit component modeling for cron and sessions, secret/credential policy, overwrite and rollback rules, cloud/encryption/scheduling choices, and docs that point to community tools as optional workarounds.
