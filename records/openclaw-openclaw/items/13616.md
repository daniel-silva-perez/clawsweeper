## Summary

Keep open. Current main has first-party local backup archive creation and verification, but it still does not implement the requested native restore/import/export/list/cloud/encryption/scheduled-backup lifecycle or component-aware cron/session restore semantics. The discussion has useful community workarounds, especially keepmyclaw and openclaw-workspace-sync, but a recent maintainer-side review also explicitly kept the native umbrella open and the remaining restore/import scope is security-sensitive.

## What This Changes

The issue requests built-in backup and restore commands for OpenClaw config, cron jobs, workspace files, session history, approvals, pairing state, and optional cloud, encrypted, incremental, and scheduled backups.

## Best Possible Solution

Keep this as the native umbrella for a reviewed first-party backup lifecycle built on the existing create/verify archive and migration/plugin seams, with safe extraction, dry-run validation, explicit component modeling for cron and sessions, secret/credential policy, overwrite and rollback rules, cloud/encryption/scheduling choices, and docs that point to community tools as optional workarounds.
