## Summary

Keep open. Current main has first-party local backup archive creation and verification, but it still lacks the requested native restore/import/export/list/cloud/encryption/scheduled-backup workflow and component-aware cron/session restore semantics; the latest maintainer-side review also explicitly kept the native umbrella open despite useful community workarounds.

## What This Changes

Add built-in backup and restore utilities for OpenClaw config, cron jobs, session history, workspace state, cloud storage, encryption, scheduling, and disaster recovery.

## Best Possible Solution

Keep this as the native umbrella for a reviewed first-party backup lifecycle built on the existing create/verify archive and migration seams, with safe extraction, dry-run validation, explicit component modeling for cron and sessions, secret/credential policy, overwrite and rollback rules, cloud/encryption choices, and docs that point to community tools as optional workarounds.

## Risks / Open Questions

- Native restore/import can overwrite config, credentials, cron jobs, sessions, and workspaces, so path traversal, secret handling, dry-run, conflict, rollback, and permissions rules need explicit maintainer design before implementation.
- Several overlapping open follow-ups and PRs are referenced in the thread; landing a narrow automated fix before choosing the canonical scope could create incompatible backup formats or restore semantics.
- Community plugin/skill options cover important backup and sync workflows today, but they do not prove that first-party session-history export/import and component-aware native restore are complete.
