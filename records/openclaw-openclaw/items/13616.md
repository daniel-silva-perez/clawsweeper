## Summary

Keep this issue open. Current main has first-party local backup archive creation and verification, but it still lacks the requested native restore/import workflow, backup list/restore cloud surface, encryption, scheduled/cloud backup policy, and component-aware cron/session restore semantics. The discussion already points to useful community workarounds, especially openclaw-workspace-sync and keepmyclaw, but the latest maintainer-side comment treats them as workarounds rather than a replacement for the native umbrella.

## Best Possible Solution

Keep this issue open as the native umbrella for first-party backup restore/import and disaster recovery. The next concrete path is maintainer review of the narrower restore and migration/profile portability PRs, especially #59280 for `openclaw backup restore`, then an accepted design for dry-run validation, overwrite/rollback rules, component-aware cron/session semantics, cloud/encryption scope, and docs. Community options such as openclaw-workspace-sync and keepmyclaw should remain documented workarounds unless maintainers explicitly move the remaining native scope to ClawHub/plugin API discussion.
