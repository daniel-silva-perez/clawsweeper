## Summary

Keep this issue open. Current main has first-party local backup archive creation and verification, but it still lacks the requested native restore/import/export/list/cloud/encryption/scheduled backup workflow and component-aware cron/session restore semantics. The linked community options, including openclaw-workspace-sync and Keep My Claw, are useful workarounds, but the most recent maintainer-side review explicitly kept this native backup/restore umbrella open.

## Best Possible Solution

Keep this issue open as the native umbrella for first-party backup restore/import and disaster recovery. Maintainers should coordinate narrower restore, migration/profile portability, per-agent backup, and backup-design follow-ups, then land a reviewed restore design with dry-run validation, overwrite/rollback rules, path safety, secret handling, and explicit cron/session component semantics. Community tools such as openclaw-workspace-sync and Keep My Claw should remain documented workarounds unless maintainers decide the remaining native scope belongs outside core.

## Risks / Open Questions

- A native restore/import feature would touch high-risk state: config, credentials, cron job definitions/run history, session transcripts, workspaces, and plugin installs.
- Restore archive extraction needs explicit path traversal, symlink, overwrite, rollback, dry-run, and secret-handling rules; the earlier closed comprehensive PR was reported to have unsafe tar extraction concerns.
