## Summary

Keep open. Current main has first-party local backup archive creation and verification, but it still lacks the requested native restore/import/export/list/cloud/encryption/scheduled-backup lifecycle and component-aware cron/session restore semantics. The discussion includes useful community workarounds, especially Keep My Claw and openclaw-workspace-sync, but recent maintainer-side review also kept the native umbrella open.

## What This Changes

Add native backup and restore utilities for OpenClaw configuration, cron jobs, session history, workspace state, and optional cloud/encrypted/scheduled backup workflows.

## Best Possible Solution

Keep this as the native umbrella for a reviewed first-party backup lifecycle built on the existing create/verify archive and migration/plugin seams, with safe extraction, dry-run validation, explicit component modeling for cron and sessions, secret/credential policy, overwrite and rollback rules, cloud/encryption/scheduling choices, and docs that point to community tools as optional workarounds.
