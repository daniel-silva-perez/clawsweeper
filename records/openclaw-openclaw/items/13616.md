## Summary

Keep this issue open. Current main has first-party local backup archive creation and verification, but it still lacks the requested native restore/import/export/list/cloud/encryption/scheduled backup workflow and component-aware cron/session restore semantics. The linked community options, including openclaw-workspace-sync and Keep My Claw, are useful workarounds, and the latest maintainer-side comment already kept this native umbrella open.

## What This Changes

Request a first-party backup/restore utility for OpenClaw config, cron jobs, session history, migration, and disaster recovery.

## Best Possible Solution

Keep this issue open as the native umbrella for first-party backup restore/import and disaster recovery. Maintainers should coordinate the narrower restore, migration/profile portability, per-agent backup, and backup-design follow-ups, then land a reviewed restore design with dry-run validation, overwrite/rollback rules, path safety, secret handling, and explicit cron/session component semantics. Community tools such as openclaw-workspace-sync and Keep My Claw should remain documented workarounds unless maintainers decide the remaining native scope belongs outside core.
