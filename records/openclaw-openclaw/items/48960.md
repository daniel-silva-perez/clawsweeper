## Summary

Keep this PR open. Current main still lacks the skill-loading diagnostics this PR targets: missing skill roots return silently, malformed or incomplete SKILL.md files are dropped without a warning, and there is no parse-error/status plumbing in SkillEntry, skills list/info/check, or the current loader summary. The PR remains relevant but is not merge-ready without a refresh against the newer loader shape and fixes for logging accuracy, test assertions, and log-safety concerns.

## Best Possible Solution

Keep this PR open and ask for a refreshed implementation against current main. The best path is to consolidate with the narrower #47327 work where useful, add diagnostics in the current local-loader/workspace loader flow, carry parse/read failures through SkillEntry and skills status, surface them consistently in skills list/info/check text and JSON, cap or aggregate repeated warnings, compact/sanitize path and error details, and update tests to assert the actual formatted subsystem logger output.
