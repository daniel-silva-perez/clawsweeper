## Summary

Keep this PR open. Current main has a generic preserve-keys mechanism for the single caller-provided active session, but it still does not collect active subagent child session keys before prune/cap/disk-budget maintenance. The PR targets a real remaining bug also tracked by #56034, but the current PR version needs the unresolved review issue fixed before merge: active subagent keys should be derived from the persisted run snapshot, not only in-memory registry state.

## Best Possible Solution

Keep this PR open as an implementation candidate for #56034. Update it or replace it with a patch that derives protected child session keys from `getSubagentRunsSnapshotForRead(subagentRuns)`, threads that set through prune, cap, and disk-budget enforcement, and adds regression coverage for stale, over-cap, and disk-budget active subagent sessions, including a persisted-snapshot case.
