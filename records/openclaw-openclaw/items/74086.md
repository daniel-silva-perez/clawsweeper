## Summary

Current main now implements the Telegram-owned fixes for the reported Windows/global-npm regression: Bot API transport honors ALL_PROXY/all_proxy and OPENCLAW_PROXY_URL, webhook cleanup can continue when getWebhookInfo confirms no webhook is set, and the unreleased changelog/docs explicitly reference this report. The fixes are on main only and are not contained in the latest tagged release v2026.4.26.

## What This Changes

The issue reports a Windows npm global-install regression where Telegram startup and command sync fail after v2026.4.23 with deleteWebhook, deleteMyCommands, setMyCommands, polling stalls, and occasional packaged runtime-loading errors.

## Best Possible Solution

Close this issue as fixed on current main, keep the Telegram-owned implementation, and let the next release carry the fix to npm users. If the Windows packaged runtime-loading symptom recurs after that release, it should be filed with a fresh installed-package artifact because the primary Telegram startup and proxy regressions are now covered on main.
