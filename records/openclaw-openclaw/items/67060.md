## Summary

Keep this open. Current main still has explicit provider request proxy support, but it does not implement the requested startup warning when proxy environment variables are present and remote model providers lack an explicit request proxy; the directly related implementation candidate, #68023, is closed and unmerged.

## What This Changes

Add a non-breaking startup warning when proxy environment variables are detected but remote model providers are not explicitly configured to use request proxy routing.

## Best Possible Solution

Implement a warning-only gateway startup check that uses existing proxy-env helpers, inspects configured model providers for remote HTTP(S) targets without explicit request proxy configuration, warns once with the `models.providers.<provider>.request.proxy.mode = "env-proxy"` remediation, and covers the warn/no-env/all-configured/local-provider cases in focused tests.

## Repair Work Prompt

Implement the warning requested by this issue as a narrow, non-breaking startup observability change. In or near `logGatewayStartup`, detect configured proxy environment variables with the existing infra proxy-env helpers, inspect configured `models.providers` entries for remote HTTP(S) `baseUrl` values, skip loopback/local/private/LAN-style providers and providers with any explicit `request.proxy`, and emit a single warning that names affected providers and suggests `models.providers.<provider>.request.proxy.mode = "env-proxy"`. Do not auto-enable env proxy, do not change provider request routing defaults, and do not add bundled provider-specific special cases. Use #68023 only as prior design context because it is closed and unmerged. Add focused tests in `src/gateway/server-startup-log.test.ts` for: proxy env + remote unconfigured provider warns, no proxy env does not warn, all remote providers already configured do not warn, and local/loopback/private providers do not warn. Add a one-line user-facing changelog entry if the repo's active changelog section expects one, crediting the reporter (`Thanks @wnlen`) and any borrowed prior PR context appropriately, never `@codex`.
