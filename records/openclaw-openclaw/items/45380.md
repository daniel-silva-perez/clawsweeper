## Summary

Keep open. Current main still has the exact Unix-style env-prefixed package scripts that the PR targets, no `scripts/with-env.mjs` helper exists on main, and the repo `.npmrc` does not configure a Windows-compatible script shell. The PR remains a focused implementation candidate, not obsolete cleanup.

## Best Possible Solution

Keep this PR open for review or ask the author to rebase it into the current package script layout. The best code direction is a small cross-platform env wrapper or equivalent Node-owned script entrypoint for the env-prefixed package scripts, with focused validation for parsing, child errors, and signal exit behavior. Coordinate with #53664 if maintainers want one consolidated Windows package-script cleanup, but do not close this PR until a main-branch implementation lands.
