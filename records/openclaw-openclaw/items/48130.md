## Summary

Keep this PR open. Current main still has the same Windows registry command parsing bug in the moved browser plugin file, so the PR remains functionally useful, but it needs a rebase and targeted regression coverage before it can land.

## Best Possible Solution

Rebase the PR onto current main, apply the two regex fixes in extensions/browser/src/browser/chrome.executables.ts, and add focused Vitest coverage for quoted and unquoted Windows registry command strings through resolveBrowserExecutableForPlatform or a narrow test seam. After that, the patch is small and appropriate to land.
