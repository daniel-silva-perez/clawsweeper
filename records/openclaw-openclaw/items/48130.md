## Summary

Keep this PR open. Current main still has the Windows default-browser registry command parsing bug, and the PR remains a plausible focused fix, but it targets the old source path and should be rebased with regression coverage before landing.

## Best Possible Solution

Rebase the PR onto current main, apply the two regex fixes in extensions/browser/src/browser/chrome.executables.ts, and add focused Vitest coverage for quoted and unquoted Windows registry command strings through resolveBrowserExecutableForPlatform or a narrow exported test seam. Then review and land the small fix if the tests pass.
