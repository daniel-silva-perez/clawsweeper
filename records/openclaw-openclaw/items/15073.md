## Summary

Keep open. Current main still handles fallback as provider/model candidate selection and reuses the original agent run context; there is no config, schema, or runtime path for a fallback attempt to switch agent identity, workspace, session file, skills snapshot, or model-specific context profile. The recent issue discussion also treats this as maintainer design work, not cleanup closure.

## What This Changes

Request a fallback mode where a failed primary model can hand off to another agent/workspace or model-specific context profile so smaller fallback models receive minimal context.

## Best Possible Solution

Design a first-class fallback-context contract, then update config types, strict schemas, schema help/docs, fallback candidate metadata, reply-runner/session plumbing, and regression tests so fallback retries can use alternate context without corrupting the original agent state.
