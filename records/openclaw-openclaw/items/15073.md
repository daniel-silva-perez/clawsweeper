## Summary

Keep open. Current main still implements model fallback as provider/model candidate selection while preserving the original agent identity, workspace, session file, agentDir, and skills snapshot; the recent issue discussion also explicitly kept this as maintainer design work rather than cleanup closure.

## What This Changes

The issue requests a way for model fallback to use a different agent, workspace, or model-specific context profile when the primary model fails.

## Best Possible Solution

Design a first-class fallback-context contract, then update config types, strict schemas, schema help/docs, fallback candidate metadata, reply-runner/session plumbing, and regression tests so fallback retries can use alternate context without corrupting the original agent state.
