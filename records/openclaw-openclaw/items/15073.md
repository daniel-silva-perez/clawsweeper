## Summary

Keep this issue open. Current main still implements model-only fallback selection: per-agent config can override fallback model strings, but fallback retries continue under the same agent id, workspace, agentDir, sessionFile, and skills snapshot. The requested fallback-agent, fallback-workspace, or model-specific context-profile behavior is still absent from config types, strict schemas, fallback candidate metadata, runner plumbing, docs, and tests. The provided discussion also includes a recent maintainer/Codex comment explicitly keeping this open.

## Best Possible Solution

Keep this open for maintainer design. The best path is to choose a first-class core model: either fallbackAgent/fallbackWorkspace semantics or a narrower model-specific context profile/budget mechanism. A complete implementation likely belongs in config types and strict schemas, schema help/docs, fallback candidate metadata, reply-runner/session plumbing that can safely swap agentDir/workspaceDir/sessionFile/skills/context per fallback attempt, and regression tests proving fallback retries use alternate context without corrupting the original session.
