## Summary

Keep open. Current main still treats model fallback as provider/model candidate selection while preserving the original agent, session, workspace, and skills context; the recent discussion also explicitly kept this as maintainer design work rather than cleanup closure.

## What This Changes

Request per-agent or model-specific fallback context so fallback attempts can use a smaller agent/workspace/bootstrap profile instead of the primary agent's full prompt.

## Best Possible Solution

Design a first-class fallback-context contract, then update config types, strict schemas, schema help/docs, fallback candidate metadata, reply-runner/session plumbing, and regression tests so fallback retries can use alternate context without corrupting the original agent state.
