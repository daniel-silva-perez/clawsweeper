## Summary

Keep open. Current main still treats fallback as provider/model retry, and the config/schema/fallback-runner surfaces have no fallback-time agent, workspace, session, skills, or context-profile selector; recent issue discussion also kept this for maintainer API design.

## What This Changes

The issue requests fallback behavior that can switch to a different agent, workspace, or model-specific context profile when the primary model fails.

## Best Possible Solution

Design a first-class core fallback-context contract, then update config types, strict schemas, schema help/docs, fallback candidate metadata, reply-runner/session plumbing, and regression tests so fallback retries can use alternate context without corrupting the original session or agent state.
