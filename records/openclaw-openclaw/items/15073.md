## Summary

Keep this issue open. Current main still implements fallback as provider/model candidate selection, with only coarse global or per-agent context-budget knobs; it does not let a fallback attempt switch agent identity, workspace, agentDir, session file, or skills snapshot. The recent discussion also explicitly kept this open for maintainer design rather than cleanup closure.

## What This Changes

The issue requests an agent-level or model-specific fallback context/workspace mechanism so smaller fallback models do not receive the primary agent's full workspace prompt context.

## Best Possible Solution

Design a first-class core fallback-context contract, then update config types, strict schemas, schema help/docs, fallback candidate metadata, reply-runner/session plumbing, and regression tests so fallback retries can use alternate context without corrupting the original session or agent state.
