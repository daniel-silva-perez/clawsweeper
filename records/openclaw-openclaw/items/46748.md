## Summary

Current main still makes local timestamp rendering opt-in for `openclaw logs`: the parser prefers the log file's local `time` field, but the default formatter converts it back to UTC unless `--local-time` is passed or the CLI process has a valid `TZ`. The issue should stay open because open PRs #46782 and #47365 explicitly target this behavior with closing syntax and remain unmerged in the provided GitHub context.

## Best Possible Solution

Keep this issue open as the canonical tracker, review and consolidate #46782 and #47365, then land one narrow fix that makes human `openclaw logs` output default to local time while preserving explicit UTC output and updating tests/docs. Decide separately whether Docker console timestamp behavior belongs in the same PR or a follow-up.
