## Summary

Keep this issue open. Current main has partially addressed the original field-selection concern by writing and parsing a local `time` field, but `openclaw logs` still defaults to UTC display unless `--local-time` is passed or the CLI process has a valid `TZ`. The provided GitHub context also shows open implementation PRs #46782 and #47365 with closing syntax, so this issue should remain open until one is reviewed and merged or maintainers explicitly choose a different product direction.

## Best Possible Solution

Keep this issue open as the canonical tracker. Review and land one focused implementation PR, or make an explicit maintainer decision to keep UTC as the default and update docs/tests accordingly. If maintainers also want Docker/gateway console output covered, track that path explicitly because it is adjacent to, but not identical with, the `openclaw logs` formatter.
