## Summary

Keep open. Current main still defaults human `openclaw logs` output to UTC unless `--local-time` is passed or the CLI process has a valid `TZ`, and the issue is paired with open unmerged PRs #46782 and #47365 that reference it with closing syntax.

## What This Changes

The issue requests changing `openclaw logs` so human-readable timestamps display in the system local timezone by default instead of UTC.

## Best Possible Solution

Review and consolidate #46782 and #47365, then land one focused fix that makes human-readable `openclaw logs` output local time by default while preserving JSON/tooling output, existing `--local-time` compatibility, and a clear UTC opt-out.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: run `openclaw logs --plain` without `--local-time` and without a valid `TZ`; current main leaves local-time mode false and formats the timestamp through the UTC `toISOString()` path.

## Solution Assessment

No, closing this now is not the best way to solve it. The narrow maintainable fix is to change the human-output default/flag behavior while preserving raw JSON output and an explicit UTC path.
