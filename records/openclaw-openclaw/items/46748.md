## Summary

Keep this issue open. Current main still renders human `openclaw logs` timestamps as UTC by default unless `--local-time` is passed or the CLI process has a valid `TZ`, and open unmerged PRs already reference this issue with closing syntax.

## What This Changes

The issue requests that `openclaw logs` display timestamps in the user’s local timezone by default instead of UTC.

## Best Possible Solution

Review and consolidate #46782 and #47365, then land one narrow fix that makes human `openclaw logs` output default to local time while preserving explicit UTC output and JSON/tooling behavior.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: default `openclaw logs` leaves local-time mode off, `formatLogTimestamp` returns `toISOString()`, and the current tests assert UTC output by default.

## Solution Assessment

No for the literal suggestion to only prefer the raw `time` field, because current main already parses `time` before `_meta.date`. The maintainable fix is to change the human-output default/flag behavior while preserving an explicit UTC mode.
