## Summary

Keep open. Current main still lacks the proposed `memory_refresh` tool, so the feature is not obsolete, but the PR branch is dirty against current main and needs rework before it is mergeable.

## What This Changes

This PR adds a `memory_refresh` tool to `memory-lancedb`, adds replace/search tests, serializes related forget/refresh operations, and adds session transcript metadata stripping changes.

## Best Possible Solution

Keep the feature under review, rebase it onto the current memory-lancedb and memory-host SDK layout, move or drop the stale session-file portion based on what current main already implements, restrict audit-log permissions, update the memory-lancedb docs for the fourth tool, and then run targeted plugin and memory-host tests.

## Reproduction Assessment

Not applicable as a feature PR rather than a bug report. The current-main gap is high-confidence from source and docs: only three memory-lancedb tools are registered and documented, while the PR adds the missing fourth tool.

## Solution Assessment

No, not as currently shaped. The `memory_refresh` direction is plausible, but the branch is not the best mergeable solution until it is rebased onto current memory-host SDK paths and the audit-log permission behavior is fixed.
