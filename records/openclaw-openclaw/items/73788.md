## Summary

Keep open. Current main still lacks the requested `memory_refresh` tool, so this contributor PR is not obsolete, but the proposed patch is not merge-ready: it targets stale session-ingestion paths, needs adaptation to the current memory-lancedb/memory-host layout, and writes a new local audit log without restrictive permissions.

## What This Changes

This PR adds a `memory_refresh` tool to the bundled memory-lancedb plugin with search-only conflict preview and atomic replace modes, serializes related forget/refresh operations, adds tests, and changes session transcript metadata stripping.

## Best Possible Solution

Rebase the feature onto the current memory-lancedb and memory-host SDK layout, keep the `memory_refresh` implementation inside the plugin, move or drop the session-ingestion changes based on current SDK coverage, write the audit log with private permissions or an existing private state helper, update memory-lancedb docs and changelog if accepted, then run targeted plugin and memory-host validation.

## Reproduction Assessment

Not applicable as a feature PR rather than a bug report. The current-main gap is high-confidence from source and docs: only three memory-lancedb tools are registered and documented, while the PR adds a fourth.

## Solution Assessment

No, not as currently shaped. The `memory_refresh` direction is plausible, but this branch needs a rebase onto current memory-host paths and a permission fix before it is the narrowest maintainable solution.
