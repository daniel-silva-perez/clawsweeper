## Summary

Keep open. Current main still lacks the requested `memory_refresh` LanceDB tool, so this PR is not obsolete, but the branch is not merge-ready: it changes a session-ingestion path that current main no longer tracks and introduces a local audit log with default filesystem permissions for memory metadata.

## What This Changes

The PR adds a `memory_refresh` tool to `memory-lancedb`, tests search/replace/rollback behavior, serializes forget/refresh operations by memory ID, and adds transcript metadata stripping under `src/memory/session-files.ts`.

## Best Possible Solution

Land a rebased, maintainer-reviewed version that adds `memory_refresh` in `extensions/memory-lancedb`, writes any audit metadata with private directory/file permissions, documents the fourth tool at `https://docs.openclaw.ai/plugins/memory-lancedb`, and routes any remaining session-ingestion changes through the current memory-host SDK files.

## Reproduction Assessment

Not applicable as a feature PR rather than a bug report. The current-main gap is high-confidence from source and docs: only three memory-lancedb tools are registered and documented, while the PR adds a fourth.

## Solution Assessment

No, not as currently shaped. The `memory_refresh` direction is plausible, but the PR needs a rebase onto the current memory-host SDK layout and a private-permissions audit-log fix before it is the narrowest maintainable solution.
