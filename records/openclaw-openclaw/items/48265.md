## Summary

Keep this PR open. Current main does not implement the central `[[SPLIT]]` directive or the proposed WhatsApp @mention context enrichment, so the PR is not obsolete. It is also far broader than the title and includes security-sensitive filesystem, plugin SDK, dependency, auth, runtime, and test-infra changes with unresolved review concerns, so the best path is maintainer/contributor follow-up by splitting the work.

## Best Possible Solution

Keep open as a maintainer/contributor follow-up item, but split it before merge. The best path is one focused PR for a generic `[[SPLIT]]` parser and outbound delivery semantics with tests for streaming boundaries, code fences, split-only payloads, thread overrides, and media captions; one focused PR for WhatsApp @mention context enrichment through explicit `TemplateContext` fields, prompt metadata, and docs; and separate reviewed PRs for `/powernap`, media localRoots, shoarchive, auth/model defaults, memory fixes, macOS dependency changes, and test-infra changes.
