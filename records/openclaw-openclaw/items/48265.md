## Summary

Keep this PR open. Current main does not implement the central `[[SPLIT]]` directive, and it only has partial WhatsApp mention plumbing: it extracts mention JIDs for gating/debugging, but the inbound agent context still exposes only `WasMentioned`, not mentioned JIDs/self/contact-resolution fields. The PR also remains too broad and risky to treat as obsolete or safe cleanup: it is open, unmerged, non-mergeable, spans 153 files, and mixes the stated feature with unrelated commands, auth, macOS dependency, media-root, shoarchive, gateway, and provider changes plus multiple unresolved review findings.

## Best Possible Solution

Keep the PR open for maintainer/contributor follow-up, but ask for it to be split. The best path is a focused WhatsApp/channel PR for `[[SPLIT]]` with parser, streaming/final/media delivery, thread-context preservation, split-only suppression, and tests; a separate focused PR for mention context enrichment that exposes safe fields through `buildWhatsAppInboundContext` and prompt metadata; and separate owner-reviewed PRs for unrelated powernap, auth, shoarchive, media roots, macOS, provider, and gateway changes.
