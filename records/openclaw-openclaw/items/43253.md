## Summary

Keep this PR open. Current main still has the exact skill-creator scaffold bug the PR targets: the generated `description` placeholder is an unquoted YAML flow value, and the skill-creator guide still lacks the proposed YAML quoting guidance. The Python validator/package path makes this functional because PyYAML parses the current placeholder as a list and `validate_skill()` rejects non-string descriptions. The PR is narrow and low security risk, but the unresolved wording review and missing focused regression coverage remain valid maintainer follow-up items.

## Best Possible Solution

Keep this PR open. Ask the author to resolve the wording review, preferably add a focused scaffold-output or validator regression test proving the generated `SKILL.md` has a string `description`, then land the narrow skill-creator docs/template fix.
