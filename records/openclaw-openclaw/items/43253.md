## Summary

Keep this PR open. Current main still has the exact skill-creator scaffold issue the PR targets: generated `SKILL.md` frontmatter emits an unquoted `description` value that PyYAML parses as a list, and the skill-creator guide still has no YAML quoting guidance. The PR is narrow and low security risk, but the existing wording review and missing focused regression coverage are still valid maintainer follow-up before landing.

## Best Possible Solution

Keep this PR open. Ask the author to address the wording review and preferably add a focused scaffold-output or validator regression test proving the generated `SKILL.md` has a string `description`, then land the narrow skill-creator docs/template fix.
