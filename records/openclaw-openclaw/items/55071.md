## Summary

This should stay open because current main still does not implement skill-driven plugin context visibility. The requested public `metadata.openclaw.activates.plugins` contract is absent from the skill metadata type, parser, snapshots, docs, and request-time plugin tool assembly, so this is an active design/API request rather than conservative cleanup.

## What This Changes

The issue requests a new skill metadata contract where matched skills can activate plugin tool visibility per turn through `metadata.openclaw.activates.plugins`.

## Best Possible Solution

Keep this open for a maintainer-owned design of a backward-compatible activation contract that parses skill activation metadata, carries activated plugin ids through skill snapshots, applies them during request-time plugin tool planning, and documents/tests the policy precedence.
