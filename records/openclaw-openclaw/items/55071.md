## Summary

Current main still does not implement skill-driven plugin context visibility. The requested `metadata.openclaw.activates.plugins` contract is absent from the skill metadata type, parser, snapshots, docs, and request-time plugin tool assembly; the existing issue discussion also points to a related unmerged PR that should be handled separately rather than closing this issue now.

## What This Changes

The issue requests a new `metadata.openclaw.activates.plugins` skill metadata contract so matched skills can dynamically activate plugin tool schemas and bundled plugin skills per turn.

## Best Possible Solution

Keep this issue open for maintainer design review of a public, backward-compatible skill activation contract that threads parsed activation metadata through skill snapshots into request-time plugin tool planning, with docs and focused tests for deny/allow precedence and plugin-skill behavior.
