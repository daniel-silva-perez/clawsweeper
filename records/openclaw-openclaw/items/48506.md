## Summary

Keep this issue open. Current main still exposes `message_sending` as a rewrite/cancel hook with raw routing fields only; resolved target kind, external/cross-context status, and same-account context remain internal outbound-routing concerns. Because this is an additive public Plugin SDK and security-policy surface, it needs maintainer API/security review rather than stale cleanup or an automated fix PR.

## Best Possible Solution

Keep this open for maintainer API/security review. The best path is an additive optional target-context contract on `PluginHookMessageSendingEvent`, populated from existing target-resolution and cross-context logic across generic and channel-specific delivery paths, documented in the public plugin hooks docs, and coordinated with #48503 and #48517 before the SDK shape is frozen.
