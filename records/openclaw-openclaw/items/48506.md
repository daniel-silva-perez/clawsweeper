## Summary

Keep this issue open. Current main still exposes `message_sending` as a content rewrite/cancel hook with routing and metadata only; resolved target and cross-context information exists internally, but it is not part of the public plugin hook event. Because this is a public plugin SDK and security-policy API shape, it needs maintainer review rather than stale cleanup or an automated narrow fix.

## Best Possible Solution

Keep this open for maintainer API/security review. The best path is an additive optional target-context surface on `PluginHookMessageSendingEvent`, populated from existing target-resolution and cross-context logic, documented at the plugin hook API level, and tested across generic outbound delivery plus channel-specific delivery paths. Coordinate the shape with #48503 and #48517 before codifying the public SDK contract.
