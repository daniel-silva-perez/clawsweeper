## Summary

Keep this PR open for maintainer review. Current main still has the data-loss behavior, and the latest PR revision addresses the earlier cleanup-semantics concern by pruning state contents around preserved workspaces instead of skipping state cleanup wholesale.

## What This Changes

The PR adds a workspace-preservation option to uninstall state cleanup, passes unselected workspace directories from `openclaw uninstall`, and adds regression coverage for preserving a nested workspace while deleting state config and credentials.

## Best Possible Solution

Land a focused uninstall cleanup fix that preserves unselected workspaces nested under state while deleting selected state/config/auth siblings, then close the paired bug through the merge.

## Reproduction Assessment

Yes. Current main provides a high-confidence source-level reproduction: default workspaces are under `~/.openclaw`, uninstall treats state and workspace as separate scopes, and state cleanup recursively removes the whole state directory.

## Solution Assessment

Yes. The revised PR is the narrow maintainable direction because it only passes a preserve list when the workspace scope is not selected and prunes around preserved workspaces instead of leaving all selected state data behind.
