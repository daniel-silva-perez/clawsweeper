## Summary

Keep this PR open. Current main still rejects bare time-only `cron add --at` values because `parseAt` only applies `--tz` to offset-less ISO datetimes and then falls through to absolute-time or duration parsing. The PR remains the focused open implementation candidate for the related open bug #59441, and its diff is limited to cron parser logic plus unit coverage.
