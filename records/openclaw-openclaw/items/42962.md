## Summary

Keep open. Current main still lacks the targeted gateway schema lookup warning suppression, and landing this PR alone likely would not fix the production path because `config.schema.lookup` is still classified as mutating before payload rendering. The related open PR #42898 covers the classification half, so the best path is to combine or coordinate both fixes rather than close this PR as implemented or superseded.

## Best Possible Solution

Keep this PR open as an implementation candidate, but update it or fold it together with #42898 so both required pieces land together: classify compound read-only gateway actions by leaf verb, suppress only the exact benign `gateway` `config.schema.lookup` + `config schema path not found` miss, and add regression coverage for the full runtime path plus inverse cases.
