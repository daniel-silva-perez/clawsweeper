## Summary

Keep this PR open. Current main still does not implement the PR’s targeted `gateway` + `config.schema.lookup` + `config schema path not found` warning suppression, and the live mutation-classification path still treats the compound lookup action as mutating. The PR is relevant, but it needs to be coordinated with #42898 or equivalent mutation-classification work before it fully fixes the reported Telegram/user-facing warning noise.

## Best Possible Solution

Keep this PR open as an implementation candidate, but coordinate it with #42898 or fold both fixes together. The best fix is to classify compound gateway read actions by leaf verb, suppress only the exact benign `gateway` `config.schema.lookup` + `config schema path not found` miss, and add regression coverage for the runtime path plus inverse cases for mutating and unrelated gateway failures.
