## Summary

Keep open. Current main still lacks this PR's targeted gateway schema lookup warning suppression, and landing this PR alone likely would not fix the production path because `config.schema.lookup` is still classified as mutating before payload rendering. Related PR #42898 covers the classification half, but this PR still contains the targeted warning-policy piece.

## Best Possible Solution

Keep this PR open as an implementation candidate, but coordinate it with #42898 or fold the two together: classify compound read-only gateway actions by leaf verb, suppress only the exact benign `gateway` `config.schema.lookup` + `config schema path not found` miss, and add regression coverage for the runtime path plus inverse cases.
