## Summary

Keep this PR open. Current main still lacks the targeted gateway `config.schema.lookup` path-not-found warning suppression, and the live runtime path still classifies `config.schema.lookup` as mutating before payload rendering. The PR is useful but incomplete by itself; related PR #42898 covers the mutation-classification half while this PR covers the payload warning-policy half.

## Best Possible Solution

Keep this PR open as an implementation candidate, but coordinate it with #42898 or fold the two together. The best fix is to classify compound read-only gateway actions by leaf verb, suppress only the exact benign `gateway` `config.schema.lookup` + `config schema path not found` miss, and add regression coverage for the runtime path plus inverse cases for mutating and unrelated gateway failures.
