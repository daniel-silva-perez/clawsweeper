## Summary

Keep this PR open. Current main still lacks explicit Active Memory chat-type support, and the provided PR remains a focused implementation candidate for the linked bug, but its resolver change should parse structured explicit session keys before broad marker checks before landing.

## Best Possible Solution

Keep this PR open as the active implementation candidate. Update the resolver to parse structured `agent:<agent>:explicit:<sessionId>` keys before broad marker checks, then land the manifest schema, TypeScript config, normalization, docs, and regression-test changes for explicit Active Memory sessions.
