## Summary

Keep this PR open. Current main still rejects and strips the `explicit` Active Memory chat type and cannot classify `agent:...:explicit:...` session keys as explicit. The current PR head is still a useful implementation candidate for linked bug #65775 and appears to address the earlier resolver-review concern by checking structured explicit keys before broad marker checks.

## Best Possible Solution

Keep this PR open as the active implementation candidate for linked bug #65775. Add or confirm the public Active Memory docs coverage for `allowedChatTypes: ["explicit"]`, then review and land the focused schema, runtime resolver, normalization, changelog, and regression-test changes after normal CI.
