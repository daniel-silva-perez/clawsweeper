## Summary

Keep open. Current main still exposes compaction lifecycle hooks, but the bundled session-memory hook only subscribes to and handles `command:new` and `command:reset`; `/compact` runs the compaction path and emits `session:compact:before` / `session:compact:after`, which session-memory does not consume. The KeepMyClaw comment is useful complementary backup context, but it does not implement the requested native hook behavior.

## What This Changes

The issue requests firing the bundled session-memory hook when `/compact` runs so conversation state is captured before compaction.

## Best Possible Solution

Keep this as the compact-specific tracker and implement a narrow bridge through the existing `session:compact:before` lifecycle event. The session-memory writer should be refactored or extended to consume pre-compaction session file/workspace context, update its hook metadata and docs, and add focused regression coverage that `/compact` writes one memory artifact before summarization without changing broader daily/idle/prune behavior.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: enable or inspect the bundled `session-memory` hook and follow `/compact`; current metadata and handler guards exclude compact, while `/compact` only reaches the compaction runtime and its separate `session:compact:*` events.

## Solution Assessment

Yes, the requested native preservation behavior is appropriate, but the best implementation should use `session:compact:before` rather than inventing only `command:compact`. That preserves the before-summarization timing and also covers non-chat compaction callers that share the runtime path.
