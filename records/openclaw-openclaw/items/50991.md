## Summary

Keep this issue open. Current main has a real compaction hook surface, but the bundled session-memory hook still only subscribes to and handles `command:new`/`command:reset`; `/compact` currently goes through the compaction runtime and emits `session:compact:before` / `session:compact:after`, not a `command:compact` event that session-memory consumes. The discussion’s backup-tool comment is useful context, but it is complementary and does not implement the requested native session-memory preservation path.

## Best Possible Solution

Keep this issue open as the compact-specific session-memory tracker. The best implementation is a narrow, tested bridge that saves the active transcript before compaction, preferably by subscribing session-memory to `session:compact:before` and reusing/refactoring its writer so it can consume the compaction session file/workspace context. Update the hook metadata, docs, and regression tests to prove `/compact` creates the expected `<workspace>/memory/YYYY-MM-DD-*.md` file without duplicating `/new` or `/reset` behavior.
