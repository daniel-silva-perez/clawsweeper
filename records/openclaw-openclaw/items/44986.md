## Summary

Keep open. Current main still does not implement the same-version installer skip, so this PR is not obsolete, but the patch is not merge-ready because the skip guard can trust an arbitrary PATH binary and the skipped path bypasses shared installer finalization.

## What This Changes

This PR adds `--force` / `OPENCLAW_FORCE_INSTALL` and a same-version no-op path to the npm branch of `scripts/install.sh`, leaving git installs unchanged.

## Best Possible Solution

Keep the optimization direction, but make the no-op decision depend on the npm-managed install itself, such as reading the global npm package `package.json` or proving the resolved binary belongs to npm's global bin/root. Preserve the shared finalization flow even when the npm package install is skipped, and add focused regression tests for the fake-PATH/no-package case and finalization behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Verify the npm package before skipping install:** `scripts/install.sh:1989-1990`
  - body: The skip path accepts the version from `resolve_openclaw_version()`, which first uses `OPENCLAW_BIN` or any `openclaw` on `PATH`. A wrapper or stale binary that prints the target semver can set `INSTALL_SKIPPED=1` even when npm's global `openclaw` package is missing, so the installer never repairs the official package.
  - confidence: 0.94
- **[P2] Keep finalization on skipped installs:** `scripts/install.sh:2340-2341`
  - body: Returning when `INSTALL_SKIPPED=1` bypasses shared finalization, including duplicate install diagnostics, gateway refresh, doctor/onboarding checks, and the final resolver path. Same-version reruns should skip only the npm install or continue through finalization.
  - confidence: 0.88

## Security Review

Status: needs_attention
