## Summary

Keep this PR open for author or maintainer follow-up, but do not merge it as-is: the proposed skip path can suppress the official npm install based on an arbitrary `openclaw` executable on `PATH`, which is an install-integrity regression in risky installer infrastructure.

## What This Changes

This PR adds `--force` / `OPENCLAW_FORCE_INSTALL` and an npm-install skip to `scripts/install.sh` when the detected local OpenClaw version equals the resolved target version.

## Best Possible Solution

Keep the optimization direction, but make the no-op decision depend on the npm-managed installation itself, such as reading the global npm package `package.json` or proving the resolved binary belongs to npm's global bin/root. Preserve shared finalization behavior and add a regression test for the fake-PATH/no-package scenario before relying on installer smoke coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Verify the npm package before skipping install:** `scripts/install.sh:1989`
  - body: The new skip branch compares the target version with `resolve_openclaw_version()`, which accepts `OPENCLAW_BIN` or the first `openclaw` on `PATH` before falling back to `$(npm root -g)/openclaw/package.json`. A wrapper or stale binary that prints the target semver can set `INSTALL_SKIPPED=1` while the npm-managed package is missing, so the official install never gets repaired. Base the skip on the npm package tree or prove the binary belongs to npm's global install first.
  - confidence: 0.94

## Security Review

Status: needs_attention

Summary: The installer diff introduces an install-integrity concern because the skip check can trust an arbitrary PATH binary instead of the npm-managed package.

Concerns:

For an installer path, that can leave the official package missing or unrepaired.
