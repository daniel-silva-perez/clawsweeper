## Summary

Keep open. Current main does not implement the no-op installer optimization, and the PR branch still has review-blocking installer issues: the skip decision can be satisfied by an arbitrary PATH binary, and the no-op path does not preserve the installer’s shared finalization contract.

## What This Changes

This PR adds `--force` / `OPENCLAW_FORCE_INSTALL` and skips the npm install path in `scripts/install.sh` when the detected local OpenClaw version equals the resolved target version.

## Best Possible Solution

Keep the optimization direction, but make the no-op decision depend on the npm-managed install itself, such as reading the global npm package `package.json` or proving the resolved binary belongs to npm's global bin/root. Preserve shared finalization behavior and add a fake-PATH/no-package regression test before relying on installer smoke coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Verify the npm package before skipping install:** `scripts/install.sh:1989-1991`
  - body: The skip branch trusts `resolve_openclaw_version()`, which first accepts `OPENCLAW_BIN` or the first `openclaw` on `PATH`. A wrapper or stale binary that prints the target semver can set `INSTALL_SKIPPED=1` while the npm-managed package is missing, so the official install path never gets repaired. Base the skip on the npm package tree or prove the binary belongs to npm's global install first.
  - confidence: 0.94
- **[P2] Keep finalization on skipped installs:** `scripts/install.sh:2338`
  - body: The new early return after `install_openclaw` exits before the shared finalization block, so same-version reruns no longer run duplicate-install diagnostics, gateway refresh, doctor, bootstrap/onboarding checks, or the final resolver path. Skip only the npm install or route the no-op case through shared finalization.
  - confidence: 0.86
