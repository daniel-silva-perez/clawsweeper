## Summary

Keep this PR open. Current main still performs the npm installer path unconditionally, so the contribution is not obsolete, but the PR as written has concrete correctness and supply-chain concerns already backed by maintainer review: the skip decision can trust an arbitrary `OPENCLAW_BIN` or PATH executable, and the skipped path exits before normal installer finalization.

## What This Changes

The PR adds `--force`/`OPENCLAW_FORCE_INSTALL` handling and skips the npm install path in `scripts/install.sh` when the resolved local version matches the target registry version, leaving git installs unchanged.

## Best Possible Solution

Keep the optimization direction, but base the no-op decision on the npm-managed package tree or a verified npm-global binary, skip only the package download/install step, continue through normal finalization, and add focused installer tests for fake PATH/no-package and skipped-finalization cases.

## Reproduction Assessment

Yes. The high-confidence reproduction is to put a fake same-version `openclaw` earlier on PATH while `npm root -g` has no `openclaw` package; the current resolver order and the PR skip guard show why that suppresses repair.

## Solution Assessment

No. The PR solves a real redundant-install request, but the safest implementation needs npm-package proof and preserved finalization rather than trusting the general version resolver and returning early.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Verify the npm package before skipping install:** `scripts/install.sh:1989-1990`
  - body: The skip guard accepts `resolve_openclaw_version()`, which can come from `OPENCLAW_BIN` or any `openclaw` on PATH before npm package metadata. A wrapper or stale binary that prints the target semver can set `INSTALL_SKIPPED=1` even when the npm global package is missing, so the installer never repairs the official package.
  - confidence: 0.94
- **[P2] Keep finalization on skipped installs:** `scripts/install.sh:2340-2342`
  - body: Returning when `INSTALL_SKIPPED=1` bypasses shared finalization, including duplicate-install diagnostics, gateway refresh, doctor/onboarding checks, and the final resolver path. Same-version reruns should skip only the npm install work and then continue through finalization.
  - confidence: 0.9

## Security Review
