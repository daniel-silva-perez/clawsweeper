## Summary

Keep open. Current main still does not implement the same-version installer skip, and this PR has concrete blocking installer concerns: the skip trusts a PATH or `OPENCLAW_BIN` version result instead of proving the npm-managed package is installed, then exits before normal finalization.

## What This Changes

The PR adds `--force`/`OPENCLAW_FORCE_INSTALL` handling and a same-version npm install skip path to `scripts/install.sh`, leaving git installs unchanged.

## Best Possible Solution

Keep the optimization direction only if the no-op decision is based on the npm-managed package tree or a verified npm-global binary, skip only the package download/install step, continue through normal finalization, and add focused installer tests for fake PATH/no-package and skipped-finalization cases.

## Reproduction Assessment

Yes. The maintainer review gives a high-confidence reproduction using a fake same-version `openclaw` earlier on PATH with no npm package tree, and the PR code confirms that resolver result is enough to set `INSTALL_SKIPPED=1`.

## Solution Assessment

No. The requested optimization is useful, but this PR is not the best implementation because it trusts the wrong proof source and exits too early; the safer solution is npm-managed install verification plus preserved finalization.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Verify the npm package before skipping install:** `scripts/install.sh:1989-1990`
  - body: The skip guard accepts `resolve_openclaw_version()`, which can come from `OPENCLAW_BIN` or any `openclaw` on PATH before npm package metadata. A wrapper or stale binary that prints the target semver can set `INSTALL_SKIPPED=1` even when the npm global package is missing, so the installer never repairs the official package.
  - confidence: 0.94
- **[P2] Keep finalization on skipped installs:** `scripts/install.sh:2340-2342`
  - body: Returning when `INSTALL_SKIPPED=1` bypasses shared finalization, including duplicate-install diagnostics, gateway refresh, doctor/onboarding checks, and the final resolver path. Same-version reruns should skip only the npm install work and then continue through finalization.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The installer diff introduces a supply-chain/integrity concern because the no-op decision can trust an arbitrary PATH or `OPENCLAW_BIN` executable instead of the npm-managed package.

Concerns:

For the official installer path, that can leave the npm package missing or unrepaired.
