## Summary

Current main still does not implement the requested runtime fallback from a gateway catalog miss to explicit `models.providers.<provider>.models[].input` metadata. The issue discussion includes a real deployment reproduction and a confirming comment that this is distinct from onboarding issue #61716. Related PR #67766 remains an implementation candidate, but the checked-out main code still fails closed on catalog miss, so this issue should stay open.

## Best Possible Solution

Keep this issue open. The best maintainer path is to review and land #67766 if it is sound, or replace it with a focused patch in `src/gateway/session-utils.ts` that lets `resolveGatewayModelSupportsImages()` consult normalized explicit provider model metadata on catalog miss. Coverage should include explicit provider refs, unique provider inference from config, catalog miss plus config hit, text-only config, catalog-hit precedence, and the chat, agent RPC, and node-event attachment paths.

## Evidence

- **Issue discussion confirms real runtime gap:** The reporter confirmed a real deployment where attachments are stored and the configured custom model declares `input: ["text", "image"]`, but the gateway behaves as text-only until a downstream fallback patch reads `models.providers[provider].models[].input`. A later commenter confirmed this is complementary to #61716 and that `resolveGatewayModelSupportsImages()` should fall back to configured provider model input metadata.
- **Resolver still only consults catalog and hardcoded shims:** `resolveGatewayModelSupportsImages()` accepts only `loadGatewayModelCatalog`, `provider`, and `model`; it checks a catalog entry plus Foundry/claude-cli shims, then returns `false` on catalog miss or catalog-load failure. It does not accept config, call `getRuntimeConfig()`, or inspect `models.providers.*.models[].input`.
  - file: [src/gateway/session-utils.ts:1118](https://github.com/openclaw/openclaw/blob/68561a8c948b49e85ae012646de550a49d6c63a3/src/gateway/session-utils.ts#L1118)

Explicit configured model metadata is not available to this decision point when the catalog misses.
