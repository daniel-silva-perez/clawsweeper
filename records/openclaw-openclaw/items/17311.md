## Summary

Keep this issue open. Current main ships the SecretRef foundation with env/file/exec providers and documents 1Password through the generic exec bridge, but it still does not include native keyring/Keychain, native 1Password, or a plugin-owned SecretRef provider-source seam. The provided thread context also identifies same-author open PR #72548 as the active implementation candidate, so closing this issue now would race that review.

## Best Possible Solution

Review #72548 as the active implementation candidate. If it lands, close this issue as implemented for the env/keyring/plugin-seam portion and open or link a focused follow-up for native 1Password if maintainers still want it bundled. If maintainers decline that PR, narrow this issue to the accepted native keyring/Keychain, native 1Password, or plugin SecretRef provider API work, with broader cloud-provider tracking left to #13610 or dedicated provider issues.
