## Summary

Keep this issue open. Current main has the env/file/exec SecretRef foundation and a documented 1Password-through-exec path, but it still does not ship native keyring/Keychain, native 1Password, or a plugin-owned SecretRef provider/source seam. The same author has open PR #72548 as the active implementation candidate for the remaining architecture and bundled keyring work, so closing this issue now would race that review.

## Best Possible Solution

Review PR #72548 as the active same-author implementation candidate. If it lands, close this issue as implemented for the env/keyring/plugin-seam portion and open or link a focused follow-up for native 1Password if maintainers still want it bundled. If the PR is declined, narrow this issue to the accepted native keyring/Keychain, native 1Password, or plugin SecretRef provider API work, leaving broader cloud-provider tracking to #13610 or dedicated provider issues.
