## Summary

Keep this issue open. Current main has the SecretRef foundation with built-in env/file/exec providers and documents 1Password through exec, but it still lacks native keyring/Keychain, native 1Password, and a plugin-owned SecretRef provider-source seam. The same author also has open PR #72548 as the active implementation candidate for the remaining work, so closing this issue now would race that review.

## Best Possible Solution

Review #72548 as the active implementation candidate. If it lands, close this issue as implemented for the pluggable SecretRef provider-source seam and native keyring/Keychain path, then open or link a focused native 1Password follow-up only if maintainers want that bundled. If maintainers decline that PR, narrow this issue to the accepted native keyring/Keychain, native 1Password, or plugin SecretRef provider API work, with broader cloud-provider tracking left to #13610 or dedicated provider issues.
