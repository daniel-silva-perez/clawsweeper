## Summary

Keep this issue open. Current main still supports only built-in env/file/exec SecretRefs; native keyring/Keychain, native 1Password, and a plugin-owned secret-provider source seam are not shipped. The same author has open PR #72548 as the active implementation candidate, so closing this issue now would race that review.

## Best Possible Solution

Review PR #72548 as the active implementation candidate. If it lands, close this issue as implemented for the env/keyring/plugin-seam portion and open or link a focused follow-up for native 1Password if maintainers still want bundled support. If it is declined, narrow this issue to the accepted native keyring/Keychain, native 1Password, or plugin SecretRef provider API work, leaving broader cloud-provider tracking to #13610 or dedicated provider issues.
