## Summary

Keep this PR open for maintainer judgment, but do not merge it as-is. Current main still does not implement the submitted mutex-based appendMessage serialization, and the submitted approach would change a synchronous SessionManager append contract into an asynchronous runtime path while also tracking pending tool calls before the write path is accepted.

## Best Possible Solution

Do not auto-close or merge this PR as-is. Maintainers should either close it manually as obsolete if the current idle-before-flush implementation fully covers the reported synthetic-result failures, or ask for a rework that preserves the synchronous `SessionManager.appendMessage` contract, tracks pending tool calls only after accepted persistence, and adds a current-main regression test through a real async boundary or an explicit async append adapter.
