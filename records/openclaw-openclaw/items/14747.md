## Summary

Current main still lacks a configurable lane wait diagnostic threshold. The command queue defaults missing warn thresholds to 2,000ms, diagnostics config/schema/docs do not expose a lane wait setting, manual cron still has a separate fixed 5,000ms override, and the related implementation PRs are closed unmerged while the broader concurrency issue does not replace this narrow request.

## Best Possible Solution

Keep this open and land a focused core patch adding documented `diagnostics.laneWaitWarnMs` as a positive millisecond value. Preserve the current 2,000ms default when unset, preserve explicit per-call `warnAfterMs` overrides, decide and test how the manual cron 5,000ms override interacts with the new config, apply the configured default to queue enqueues without an override, and update schema/types/help/labels/generated metadata, docs, tests, and changelog. Treat #53023 as the broader concurrency/auto-yield follow-up, not the canonical fix for this request.
