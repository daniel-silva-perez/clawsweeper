## Summary

Current main still does not expose a configurable lane wait diagnostic threshold. The queue warning path defaults missing `warnAfterMs` to 2,000ms, scheduled cron isolated agent work reaches the cron lane without an override, manual cron uses a separate fixed 5,000ms override, and the diagnostics schema/docs only expose `stuckSessionWarnMs`. Related implementation PRs #14889 and #16366 are closed unmerged, and #53023 is broader concurrency/auto-yield work rather than a replacement for this narrow request.

## Best Possible Solution

Keep this open and land a focused core patch adding a documented `diagnostics.laneWaitWarnMs` positive-millisecond setting. Preserve the current 2,000ms default when unset, preserve explicit per-call `warnAfterMs` overrides, apply the configured default to scheduled cron and other queue enqueues without an override, deliberately handle the manual cron 5,000ms override, and update config schema/types/generated metadata, docs, tests, and changelog. Keep #53023 as the broader concurrency/auto-yield follow-up.
