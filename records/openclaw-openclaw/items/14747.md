## Summary

Current main still does not provide a user-configurable lane wait diagnostic threshold. The queue warning uses each entry's `warnAfterMs`, but `enqueueCommandInLane` still defaults missing values to `2_000`; scheduled cron agent work reaches the queue without an override, manual cron enqueue has its own fixed `5_000`, the strict diagnostics config schema exposes no lane-wait field, and the related implementation PRs are closed unmerged. This remains a narrow, valid fix-PR candidate.

## Best Possible Solution

Keep this issue open and land a focused core patch adding a documented `diagnostics.laneWaitWarnMs` positive-ms setting. Preserve the existing 2000ms default, apply the setting whenever `enqueueCommandInLane` receives no explicit `warnAfterMs`, preserve per-call overrides, make gateway start/reload update the process queue default, cover scheduled and manual cron behavior, update generated config metadata/docs/changelog, and keep #53023 as the broader concurrency/auto-yield follow-up.

## Clownfish Work Prompt

Implement a focused fix for openclaw/openclaw#14747: add a configurable lane wait diagnostic threshold without broadening into session concurrency or auto-yield work.

Expected behavior:
- Add a positive integer config setting, preferably `diagnostics.laneWaitWarnMs`, defaulting to the current behavior of 2000ms when unset.
- `enqueueCommandInLane` should use the configured process default only when callers do not pass `opts.warnAfterMs`; explicit per-call overrides must continue to win.
- Scheduled cron isolated agent work should inherit the configured default through the existing cron -> cron-nested embedded-runner lane path.
- Manual cron enqueue currently passes `warnAfterMs: 5_000`; route it through the same configurable behavior unless there is a clear owner-path reason to keep it explicit, and document/test the chosen behavior.
- Gateway start and config reload must apply the setting so a config change takes effect without requiring a full restart if that matches the existing reload model.
- Update config types, zod schema, labels/help/generated schema metadata, public docs, and changelog. This is user-facing, so add a changelog entry with reporter credit; do not credit Codex/OpenClaw.
- Do not implement #53023 session concurrency, auto-yield, or preemption in this PR.

Likely implementation areas: add a queue default setter/state in `src/process/command-queue.ts`; apply it from gateway lane/config startup and reload code; extend diagnostics config schema/metadata/docs; add regression tests for default, override, scheduled cron lane, and manual cron behavior.
