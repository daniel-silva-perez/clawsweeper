## Summary

Keep this PR open. Current main already covers the stop-button visibility problem through the refactored `renderChatRunControls` path, but it still does not implement the PR’s other central behavior: preserving typed drafts across session switches. The PR also should not be landed as-is because the proposed draft cache is only keyed by `sessionKey`, and the later display-name changes are unrelated to the title and had concrete review concerns.

## Best Possible Solution

Keep the PR open but require a revision against current main: drop the stop-button patch because `ui/src/ui/chat/run-controls.ts` already solves it, split or remove the unrelated display-name changes, and implement draft persistence in the current session-switch path with a key scoped beyond bare `sessionKey` plus tests for A -> B -> A restore, empty-draft cleanup, and no cross-gateway/workspace draft bleed.
