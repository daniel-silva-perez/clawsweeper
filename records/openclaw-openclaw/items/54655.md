## Summary

Keep this PR open. Current main already covers the Stop button visibility part through the extracted chat run controls, but it still clears composer drafts on session switch, so the central draft-preservation work is not implemented. The branch is not merge-ready as-is because it adds an under-scoped module-level draft cache and folds unrelated session-display-name behavior into the same patch while overlapping with open PR #47992.

## What This Changes

The PR adds an in-memory per-session draft map for Control UI session switches, changes the chat Stop button condition, adds UI tests, adjusts session display-name fallbacks/filtering, and includes the helper node test in Vitest config.

## Best Possible Solution

Ship one rebased, narrow Control UI change that preserves composer drafts in the centralized session-switch reset path, scopes or clears draft state across gateway/auth/workspace changes, preserves existing queued-message behavior, and leaves display-name cleanup to a separate reviewed change unless it is required by the same user-visible bug.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Scope the draft cache beyond sessionKey:** `ui/src/ui/app-render.helpers.ts:495-502`
  - body: The new module-level map is keyed only by sessionKey, then restored by nextSessionKey. Control UI can reconnect or switch gateway/auth contexts without a page reload, and common keys like main can be reused, so a draft from one backend can reappear in another. Scope the cache to the active connection/session namespace or clear it on gateway/auth changes.
  - confidence: 0.87
- **[P2] Narrow the generated group-name filter:** `ui/src/ui/app-render.helpers.ts:677-678`
  - body: buildGroupDisplayName emits provider-prefixed g-* names for normal channel groups, not only WebChat internals. Treating every <provider>:g-* displayName as internal drops legitimate group labels such as Discord or Slack group names and falls back to generic text.
  - confidence: 0.9
- **[P2] Use the last subagent segment for short IDs:** `ui/src/ui/app-render.helpers.ts:686-688`
  - body: extractShortSessionId takes the first UUID-like segment in the full key. Nested subagent keys can include both parent and child segments, so child sessions can display the parent's short ID and become indistinguishable. Derive the ID from the last :subagent: segment instead.
  - confidence: 0.86
- **[P3] Assert the Stop button against null:** `ui/src/ui/views/chat.test.ts:661`
  - body: querySelector returns null, not undefined, so this presence assertion would pass if the Stop button were absent. Use not.toBeNull() to make the test's intent and failure mode correct.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: No supply-chain or code-execution risk was found, but the new in-memory draft cache has a low-severity draft isolation issue because it is keyed only by sessionKey.

Concerns:

- **[low] Drafts can bleed across gateway contexts:** `ui/src/ui/app-render.helpers.ts:495`
  - body: The PR stores unsent text in a module-level cache keyed only by sessionKey; after reconnecting the same Control UI tab to another gateway or workspace with a reused key like main, previously typed draft text can be restored in the wrong context. This is local to the browser session but can expose sensitive unsent text to the wrong workspace or gateway view.
  - confidence: 0.76
