## Summary

Keep this PR open. Current main already has the Stop button behavior via the refactored chat run controls, but it still clears composer drafts on session switches, so the central draft problem is not implemented on main. The PR is not safe to merge as-is because its draft cache is only keyed by sessionKey and its extra display-name changes can regress legitimate channel group labels and nested subagent labels; the overlapping open PR #47992 also needs maintainer consolidation.

## What This Changes

The PR adds an in-memory per-session draft map in Control UI session switching, changes chat Stop button rendering to follow canAbort, adds UI tests, expands session display-name fallback/filtering, and includes the helper node test in Vitest config.

## Best Possible Solution

Ship one rebased Control UI change that preserves composer drafts in the centralized session-switch reset path, scopes or clears draft state across gateway/auth/workspace changes, keeps the current main Stop-button behavior, and handles session display-name cleanup only in a separate narrowed review if still needed.

## Reproduction Assessment

Yes. A source-level reproduction is clear: set a non-empty chatMessage, call switchChatSession, and current main delegates to resetChatStateForSessionSwitch, which clears chatMessage; the Stop-button gap no longer reproduces on main because Stop renders from canAbort.

## Solution Assessment

No. The Stop-button part matches the maintainable current-main direction, but the draft cache should be scoped beyond sessionKey and wired through the shared reset path; the display-name changes should be narrowed or split because they introduce unrelated regressions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Scope the draft cache by connection context:** `ui/src/ui/app-render.helpers.ts:495-502`
  - body: The new module-level map is keyed only by sessionKey, then restored by nextSessionKey. Control UI can reconnect or switch gateway/auth contexts without a page reload, and common keys like main can be reused, so a draft from one backend can reappear in another. Scope the cache to the active connection/session namespace or clear it on gateway/auth changes.
  - confidence: 0.87
- **[P2] Narrow the generated group-name filter:** `ui/src/ui/app-render.helpers.ts:677-678`
  - body: buildGroupDisplayName emits provider-prefixed g-* names for normal channel groups, not only WebChat internals. Treating every <provider>:g-* displayName as internal drops legitimate group labels such as Discord or Slack group names and falls back to generic text.
  - confidence: 0.9
- **[P2] Use the last subagent segment for short IDs:** `ui/src/ui/app-render.helpers.ts:686-688`
  - body: extractShortSessionId takes the first UUID-like segment in the full key. Nested subagent keys can include both parent and child segments, so child sessions can display the parent's short ID and become indistinguishable. Derive the ID from the last :subagent: segment instead.
  - confidence: 0.86
- **[P3] Assert the Stop button against null:** `ui/src/ui/views/chat.test.ts:661`
  - body: querySelector returns null, not undefined, so this presence assertion would pass if the Stop button were absent. Use not.toBeNull() to make the test's intent and failure mode correct.
  - confidence: 0.82
