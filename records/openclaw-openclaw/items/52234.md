## Summary

Keep this PR open. Current main still does not implement the central reset-followup behavior: reset cleanup clears followup queues and drain callbacks, queued followups still execute against their original run/session binding, and streaming/tool/block delivery lacks reset-aware stale-output suppression. The PR is a plausible implementation candidate and a refresh of closed unmerged #51655, but the review thread and large diff mean it needs maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open as an implementation candidate, but require a revised maintainer-reviewed patch before merge. The best path is a smaller, race-tested change that preserves queued followups across sessions.reset, pauses/resumes preserved drains with ownership for overlapping resets, rebinds queued and in-flight work through canonical latest-session resolution, suppresses or aborts stale partial/reasoning/block/tool/final deliveries, preserves threading and session overrides, and adds targeted regression coverage for the reset races.
