## Summary

Keep this PR open. Current main still does not implement the PR's central behavior: session reset still clears followup queues and drain callbacks, and the main/followup reply paths still lack reset-aware rebind plus stale-output suppression. The PR remains a relevant implementation candidate, but the provided review context shows it still needs revision before it would be safe to merge.

## Best Possible Solution

Keep the PR open for maintainer review or a revised smaller patch. The best implementation should preserve queued followups across `sessions.reset`, pause/resume drains with overlap-safe ownership while the replacement session is being written, rebind queued and in-flight work through canonical latest-session resolution, suppress or abort stale partial/reasoning/block/tool/final deliveries, preserve existing per-session overrides, and add focused regression coverage for each reset race before merge.
