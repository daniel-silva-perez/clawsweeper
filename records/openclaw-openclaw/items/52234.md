Keep this PR open. Current main still does not implement the PR's central behavior: session reset clears followup queues and drain callbacks, and the reply/followup paths still lack the PR's reset-aware rebind and stale-output suppression. The PR is a relevant fix candidate, but the current head also has unresolved reset-path blockers from review context, so it is not safe to close as implemented or merge without revision.

## Best Possible Solution

Keep the PR open for maintainer review or a revised, smaller current-main patch. The best implementation should preserve queued followups across sessions.reset, pause and resume drains around the reset mutation with overlap-safe ownership, rebind queued and in-flight work through canonical latest-session resolution, abort or suppress stale partial/block/tool/final deliveries, preserve existing session overrides, and add targeted regression tests for each race.
