## Summary

Keep this PR open. Current main still does not implement the PR's central behavior: reset cleanup still clears followup queues and drain callbacks, queued followups are still bound to their original session snapshot, and the main/followup delivery paths still lack a reset-aware latest-session rebind plus stale-output suppression. The PR is a relevant refresh of the closed unmerged #51655 work, but the review history shows it needs careful maintainer review before merge rather than conservative cleanup closure.

## Best Possible Solution

Keep this PR open as an implementation candidate, but require a revised maintainer-reviewed patch before merge. The best fix should preserve queued followups during sessions.reset while clearing command lanes and stale system events, pause/resume preserved drains with overlap-safe ownership until the replacement session is written, rebind queued and in-flight reply work through canonical latest-session resolution, suppress or abort stale partial/reasoning/block/tool/final deliveries, preserve existing threading, compaction metadata, and per-session overrides, and add targeted reset-race regression coverage.
