## Summary

Keep open. Current main does not implement the PR's intended optimization: builtin memory search still calls `hasIndexedContent()` during every search preflight, and that method still directly queries SQLite without a cached field. The linked performance issue remains open, and the PR is a focused owner-module change rather than obsolete work. The remaining maintainer path is review/land or adapt the PR with dedicated cache hit/miss and invalidation tests.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or adapt the memory-core cache in the owner module, preserve the PR head's DB-handle-swap invalidation idea, and add focused tests proving repeated searches avoid redundant SQLite preflight queries while deletes/resets/reindex correctly invalidate the cache. Once that lands, the linked performance issue can close as implemented.
