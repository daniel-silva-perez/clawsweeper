## Summary

Keep open. Current main still matches the reported group-chat no-visible-final signature, and the Mac versus Windows question appears platform-independent in core because reply visibility is resolved from chat type and config, not OS. Changing explicit mention auto-posting would affect the intentional private-by-default group/channel policy, and POPO is not bundled in this repository, so this needs maintainer support/product judgment.

## What This Changes

The issue requests restoring visible replies when the bot is mentioned in a POPO group chat after upgrading to 2026.4.27, while P2P chats still reply normally.

## Best Possible Solution

Clarify support guidance for v2026.4.27 POPO users: the current core behavior is platform-independent, messages.groupChat.visibleReplies: "automatic" restores legacy group auto-posting, and any change that lets explicit mentions bypass tool-only visible replies should be a maintainer-approved follow-up.

## Reproduction Assessment

Yes for the core delivery signature: current tests cover a group/channel turn resolving to message_tool_only, queuedFinal=false, and no sendFinalReply. No for a live POPO-specific reproduction, because that channel is external to this checkout.

## Solution Assessment

Unclear. The documented config restore path is the narrowest support answer if the private-by-default policy is intended; changing explicit mentions to auto-post would be broader and should be a maintainer product decision.
