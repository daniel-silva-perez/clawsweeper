Keep open. Current main still matches the reported symptom for default group/channel source turns, and the follow-up platform question needs a maintainer response rather than conservative cleanup.

## What This Changes

The issue reports that after updating to 2026.4.27, a POPO group-chat mention reaches the agent but produces no visible reply while direct P2P chat still works.

## Best Possible Solution

Maintainers should clarify the intended behavior for POPO users, answer that the shared visible-reply policy is not Mac/Windows-specific, and either point affected users to the documented automatic group-reply config or open a deliberate policy change for explicit mentions in group/channel rooms.

## Reproduction Assessment

Yes for the shared OpenClaw symptom: the current resolver and regression tests show a default group/channel turn can reach the agent and still queue no final visible reply. No live POPO-specific reproduction is available because the POPO implementation is outside the bundled source tree.

## Solution Assessment

Unclear. The current implementation is a deliberate privacy-oriented default with a documented compatibility knob, but changing mention-triggered group replies for an external POPO channel needs maintainer product judgment rather than an automatic fix.
