## Summary

Keep open. Current main still matches the reported signature: group/channel turns can reach the agent and finish with no visible final reply because the shipped default is message-tool-only delivery. The documented compatibility knob can restore legacy room posting, but changing explicit-mention behavior for an external POPO channel is a maintainer product/support decision rather than conservative cleanup.

## What This Changes

The reporter asks for POPO group-chat bot mentions on OpenClaw 2026.4.27 to produce visible replies like P2P chats.

## Best Possible Solution

Clarify the intended 2026.4.27 behavior for POPO users: either document/support `messages.groupChat.visibleReplies: "automatic"` as the compatibility path, or open a deliberate follow-up to change explicit-mention handling across group/channel turns without weakening the message-tool privacy goal.
