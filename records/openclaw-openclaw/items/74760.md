## Summary

Keep open. The report matches the current shipped group/channel visible-reply behavior rather than an already-fixed regression: group turns still process the agent but suppress normal visible final replies by default. Because the reported POPO channel is not bundled here and changing the new group default for explicit mentions is a product/privacy decision, this should get maintainer review rather than automatic cleanup closure.

## What This Changes

The issue reports that after updating to 2026.4.27, a POPO group-chat mention dispatches to the agent but produces no visible group reply, while direct chat still works.

## Best Possible Solution

Clarify the intended 2026.4.27 behavior for this reporter: either keep the shipped private-by-default group policy and point POPO users who want legacy group auto-posting to `messages.groupChat.visibleReplies: "automatic"`, or open a deliberate follow-up to change explicit-mention handling across group/channel turns without weakening the message-tool privacy goal.
