## Summary

Keep open. Current main still matches the reported signature: a group/channel turn can reach the agent and finish with no visible final reply because group visible replies default to message-tool-only delivery, while direct chats still auto-deliver. The prior ClawSweeper comment correctly treated this as a maintainer product/support decision because POPO is not bundled here and changing explicit-mention room output would affect the privacy behavior shipped in v2026.4.27.

## What This Changes

The issue reports that after updating to 2026.4.27, a POPO group chat mention reaches the agent but produces no visible reply, while private chat still replies.

## Best Possible Solution

Clarify the intended 2026.4.27 behavior for POPO and similar external channels: either document/support `messages.groupChat.visibleReplies: "automatic"` as the compatibility path, or intentionally change explicit-mention room delivery through a generic, tested channel policy without weakening the message-tool privacy default for always-on group contexts.
