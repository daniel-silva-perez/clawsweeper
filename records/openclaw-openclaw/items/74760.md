## Summary

Keep open. Current main still matches the reported group-chat no-visible-reply signature: group/channel turns default to message-tool-only visible output, and regression coverage asserts queuedFinal=false with no automatic final reply. Because the report is about explicit group mentions after the v2026.4.27 privacy change, deciding whether POPO users should use the documented config restore path or whether explicit mentions should auto-post needs maintainer product/support judgment.

## What This Changes

The issue reports that after updating to 2026.4.27, mentioning an external POPO bot in a group reaches dispatch but produces no visible reply, while P2P still works.

## Best Possible Solution

Maintainers should choose the support policy: either keep the shipped private-by-default behavior and direct affected group users to set messages.groupChat.visibleReplies: "automatic", or implement a narrow explicit-mention/channel rule that restores visible replies without changing always-on room privacy semantics.

## Reproduction Assessment

Yes for the core behavior: current regression coverage reproduces a group/channel turn resolving to message_tool_only with queuedFinal=false and no sendFinalReply. Live POPO reproduction is not available from this repo because POPO is external.

## Solution Assessment

Unclear. The documented config restore path is the narrowest solution if the v2026.4.27 privacy default is intended, but auto-posting explicit mentions may be the better user experience and needs maintainer approval.
