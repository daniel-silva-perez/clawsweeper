## Summary

Keep open. Current main still advertises Feishu `read`, but that action is implemented only as a single-message lookup by `messageId`; the requested paginated chat-history path for `chatId`/`limit`/`before`/`after`/`pageToken` is not implemented.

## What This Changes

The issue requests paginated Feishu chat history support for the shared `message` tool `read` action instead of returning only one message by ID.

## Best Possible Solution

Add a Feishu-owned paginated read path that preserves the existing `messageId` single-message lookup and uses the Feishu message-list endpoint for chat history when the caller provides a chat target plus pagination/filter parameters, with tests and docs updated for the new behavior.
