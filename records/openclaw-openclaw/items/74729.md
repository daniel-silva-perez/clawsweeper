## Summary

Keep open. Current main still advertises Feishu `read`, but the handler only accepts a `messageId` and calls the single-message lookup path; there is no chat-history branch for `chatId`/`limit`/`before`/`after`/`pageToken`. Request Feishu `message read` support for paginated chat history retrieval with chat targets, limits, cursors, and time filters instead of only single-message `messageId` lookup.

Add a Feishu-owned paginated read path that preserves the existing `messageId` lookup and uses the Feishu message-list endpoint for chat history when a chat target plus pagination or time filters are supplied, with focused unit tests and user-facing docs for the new parameters.
