## Summary

Keep open. Current main still recreates array-backed message objects and text-part objects in `stripInlineDirectiveTagsFromMessageForDisplay` even when no inline directive tags are stripped, and the linked attempted fixes are closed unmerged.

## Best Possible Solution

Implement the focused identity-preservation fix in `src/utils/directive-tags.ts`: reuse unchanged text-part objects, track whether any text part changed, and return the original message object when nothing was stripped. Add colocated Vitest coverage with `toBe` assertions for unchanged array-backed messages and mixed changed/unchanged text parts. Keep the SDK surface/docs unchanged unless maintainers deliberately decide to de-export or replace this helper contract.
