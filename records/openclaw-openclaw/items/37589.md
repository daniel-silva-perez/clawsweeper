## Summary

Keep open. Current main still recreates array-backed message content and returns a new message object even when no inline directive text changes, the focused identity tests are still absent, and the related implementation PRs are closed unmerged.

## What This Changes

Preserve object identity in `stripInlineDirectiveTagsFromMessageForDisplay` by rebuilding only text parts whose directive-stripped text changes and returning the original message when nothing changed.

## Best Possible Solution

Land a narrow helper and test update in `src/utils/directive-tags.ts` and `src/utils/directive-tags.test.ts` that reuses unchanged text parts, tracks whether any part changed, returns the original message when no directives were stripped, and keeps all existing stripping semantics intact.
