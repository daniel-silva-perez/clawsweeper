## Summary

Keep open. Current main still rebuilds every array-backed message content object and returns a new message object even when no inline directive tag is stripped; the latest release has the same behavior, the identity-preservation tests are still absent, and the related implementation PRs are closed unmerged.

## What This Changes

Request identity-preserving `stripInlineDirectiveTagsFromMessageForDisplay` behavior so unchanged text parts and unchanged messages are returned by reference when no inline directive tags are stripped.

## Best Possible Solution

Implement the identity-preserving path in `src/utils/directive-tags.ts`: reuse unchanged text-part objects, track whether any text part changed, return the original message object when nothing was stripped, and add focused Vitest coverage for unchanged array-backed messages plus mixed changed/unchanged parts.
