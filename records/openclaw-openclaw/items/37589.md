Current main still rebuilds every array-backed text part and returns a new message object even when no inline directive tag was stripped; the message-level tests still do not assert unchanged object identity, and the focused implementation PRs are closed unmerged with no open replacement found.

## What This Changes

Preserve object identity in `stripInlineDirectiveTagsFromMessageForDisplay` by only rebuilding text parts whose stripped text changes and returning the original message when nothing is stripped.

## Best Possible Solution

Land a narrow utility and regression-test update in the directive-tag helper that reuses unchanged text-part objects, returns the original array-backed message when no directives are stripped, and leaves existing directive stripping semantics unchanged.
