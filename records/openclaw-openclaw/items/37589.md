## Repair Work Prompt

Fix openclaw/openclaw issue #37589. In `src/utils/directive-tags.ts`, update `stripInlineDirectiveTagsFromMessageForDisplay` so it preserves object identity when stripping inline directive tags makes no change: keep the existing early returns for undefined messages and non-array content, call `stripInlineDirectiveTagsForDisplay` for text parts, reuse the original text-part object when `changed` is false, track whether any text part changed, and return the original message object when no part changed. When at least one part changes, return a new message with a new content array while preserving unchanged non-text and unchanged text-part references. Do not change the public helper signature, regex behavior, delivery parsing, plugin SDK exports, or unrelated directive semantics. Add focused tests in `src/utils/directive-tags.test.ts` asserting that a message with no directives is returned with `toBe(input)`, that unchanged text parts are reused in mixed content, and that changed text parts are recreated while existing stripping output remains the same. Changelog is not required unless maintainers decide this internal performance fix is user-facing.
