## Summary

Current main still rebuilds every array-backed text part and returns a new message object from `stripInlineDirectiveTagsFromMessageForDisplay` even when no directive tag was stripped. The linked implementation PRs are closed unmerged, and the latest release tag still has the same unresolved implementation.

## Best Possible Solution

Implement the focused identity-preservation fix in `src/utils/directive-tags.ts`: reuse unchanged text-part objects, track whether any text part changed, and return the original message object when nothing was stripped. Add colocated Vitest coverage using `toBe` for unchanged array-backed messages and mixed changed/unchanged text parts.
