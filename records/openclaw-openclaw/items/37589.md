## Summary

Keep open. Current main still recreates array-backed message content and returns a new message object from `stripInlineDirectiveTagsFromMessageForDisplay` even when no inline directive tag changed; the latest release has the same implementation, identity-preservation tests are still missing, and the related implementation PRs are closed unmerged.

## What This Changes

The issue requests that `stripInlineDirectiveTagsFromMessageForDisplay` preserve object identity by rebuilding only changed text parts and returning the original message object when nothing is stripped.

## Best Possible Solution

Implement the identity-preserving path in `src/utils/directive-tags.ts`: reuse unchanged text-part objects, track whether any text part changed, return the original message object when nothing was stripped, and add focused Vitest coverage for unchanged array-backed messages and mixed changed/unchanged parts.

## Repair Work Prompt

Fix the identity churn reported by openclaw/openclaw#37589. In `src/utils/directive-tags.ts`, update `stripInlineDirectiveTagsFromMessageForDisplay` so it calls `stripInlineDirectiveTagsForDisplay` for text parts, reuses each unchanged text-part object when `changed` is false, tracks whether any text part changed, and returns the original message object unchanged when array-backed content contains no stripped directive tags. Preserve all existing stripping output, empty-string behavior, non-array behavior, and the public plugin-facing API exported through `openclaw/plugin-sdk/text-runtime`; do not change delivery stripping or `parseInlineDirectives`. Add colocated Vitest coverage in `src/utils/directive-tags.test.ts` for an unchanged array-backed message returning the exact same message object, and for mixed content where only changed text parts are recreated while unchanged text/non-text parts keep identity. No changelog entry is expected unless maintainers decide this plugin-facing performance behavior should be user-visible. Validate with targeted tests/format first, then the changed gate per repo policy.
