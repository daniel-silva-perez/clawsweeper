## Summary

Keep open. Current main and the latest release still rebuild array-backed message content and text-part objects on the no-op display-stripping path, the focused identity regression tests are still absent, and the direct implementation PRs are closed unmerged with no open replacement found.

## What This Changes

The issue requests preserving the original message object and unchanged text-part objects when inline directive stripping makes no textual changes.

## Best Possible Solution

Land a narrow helper and regression-test update that reuses unchanged text-part objects, returns the original array-backed message when no text changed, and keeps all directive-stripping output unchanged.

## Reproduction Assessment

Yes. A focused unit test can pass an array-backed message with plain text to `stripInlineDirectiveTagsFromMessageForDisplay` and assert `result === input` plus unchanged part identity; current source necessarily fails because it always maps the array and spreads text parts.

## Solution Assessment

Yes. The narrowest maintainable fix is to use the existing `{ changed }` result, allocate only changed text parts, and return the original message when no part changed; no core API or product decision is needed.
