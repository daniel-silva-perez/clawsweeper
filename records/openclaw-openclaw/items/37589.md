## What This Changes

Avoid recreating unchanged message objects and unchanged text parts when stripping inline directive tags for display.

Implement the focused identity-preservation fix in src/utils/directive-tags.ts: reuse unchanged text-part objects, track whether any text part changed, and return the original message object when nothing was stripped. ## Repair Work Prompt

Implement the focused identity-preservation fix for this issue. In `src/utils/directive-tags.ts`, update `stripInlineDirectiveTagsFromMessageForDisplay` so it calls `stripInlineDirectiveTagsForDisplay` once per text part, reuses unchanged text-part objects when `changed` is false, tracks whether any text part changed, and returns the original message object when nothing was stripped. When a change is needed, return a new message with a new `content` array, recreating only changed text parts and preserving non-text/unchanged part references. Add colocated Vitest coverage in `src/utils/directive-tags.test.ts` using `toBe` for: unchanged array-backed messages returning the exact original message object; mixed changed/unchanged text parts preserving unchanged part identity while recreating changed parts; and the existing non-array behavior if useful. Do not change directive regexes, delivery stripping, parse whitespace normalization, plugin SDK exports, package metadata, or unrelated docs. Changelog is likely not needed because this is an internal performance/identity fix plus tests.
