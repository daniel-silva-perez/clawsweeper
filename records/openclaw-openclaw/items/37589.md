## Summary

Keep this issue open. Current main still rebuilds every array-backed text part and returns a new message object from `stripInlineDirectiveTagsFromMessageForDisplay` even when no directive tag was stripped; the linked implementation PRs are closed unmerged, and the latest release has the same implementation.

## Best Possible Solution

Implement the focused identity-preservation fix in `src/utils/directive-tags.ts`: reuse unchanged text-part objects, track whether any text part changed, and return the original message object when nothing was stripped. Add colocated Vitest coverage using `toBe` for unchanged array-backed messages and mixed changed/unchanged text parts.

## Clownfish Work Prompt

Fix this issue by preserving object identity in `stripInlineDirectiveTagsFromMessageForDisplay` when no inline directive tags are stripped. In `src/utils/directive-tags.ts`, use `stripInlineDirectiveTagsForDisplay(record.text).changed` to decide whether to recreate a text part, track whether any text part changed, and return the original message object when no array content changed. Preserve existing behavior for undefined messages, non-array content, non-text parts, stripped directive text, and empty-string post-strip text. Do not change directive regexes, SDK exports, package exports, or unrelated display sanitizers. Add focused tests in `src/utils/directive-tags.test.ts` covering: unchanged array-backed messages return the same message object; unchanged text parts are reused in mixed content; changed text parts are recreated while unchanged neighboring parts keep identity; existing empty-string directive-only behavior still passes. Changelog likely not needed unless maintainers want a public SDK behavior note.
