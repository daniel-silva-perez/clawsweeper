## Summary

Keep open. Current main still rebuilds every array-backed message content array and returns a new message object even when no directive text changes, the latest stable release has the same behavior, and the direct implementation PRs are closed unmerged with no open direct replacement found.

## What This Changes

The issue requests that `stripInlineDirectiveTagsFromMessageForDisplay` preserve object identity by rebuilding only text parts that change and returning the original message when nothing is stripped.

## Best Possible Solution

Land a narrow utility and regression-test update that uses the existing `changed` result, reuses unchanged text-part objects, returns the original array-backed message when no text changed, and keeps directive stripping output unchanged.

## Reproduction Assessment

Yes. A focused unit test can pass an array-backed message with plain text to `stripInlineDirectiveTagsFromMessageForDisplay` and assert `result === input` plus unchanged part identity; current source necessarily fails because it always maps the array and spreads text parts.

## Solution Assessment

Yes. The proposed direction is the narrowest maintainable fix: keep the helper API and stripping semantics stable, track whether any text part changed, and only allocate new objects for changed content.
