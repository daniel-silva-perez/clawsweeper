## Summary

Keep open. Current main still supports explicit `sessions_spawn` parameters plus global/per-agent sub-agent defaults, but it does not implement reusable sub-agent templates, a `sessions_spawn.template` selector, prompt-file prepending, or template discovery. The linked AIpexForge/agent-swarm PR is useful evidence of the orchestration pattern, but the requested platform feature changes public config/tool contracts and introduces workspace file reads into model context, so it needs maintainer API and security-boundary review before implementation.

## What This Changes

The issue requests reusable sub-agent template definitions under agent config that can prepend prompt files and provide default model, timeout, and label values for `sessions_spawn`.

## Best Possible Solution

Maintainers should first decide whether this belongs in core or a plugin/skill extension seam. If accepted for core, the end state should be an additive typed template contract with safe workspace-relative prompt-file resolution, bounded reads, explicit override precedence, template discovery, generated schema/docs updates, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. The current-main verification path is high confidence: schema, execution, native spawn, docs, and discovery inspection all show no `template` selector, `promptFile` handling, or template discovery.

## Solution Assessment

Unclear until maintainers decide the product/API and security boundary. The requested direction is plausible, but a safer implementation needs a typed additive contract and file-read guardrails rather than ad hoc prompt prepending.
