## Summary

Keep open. Current main still supports explicit `sessions_spawn` parameters and global/per-agent defaults for model, thinking, timeout, allowlists, and discovery, but it does not provide reusable prompt/model/timeout sub-agent templates, prompt-file prepending, or a `sessions_spawn.template` parameter. Because the missing behavior changes public config/tool API and introduces workspace file reads into model context, this needs maintainer product/API review rather than stale cleanup or a ClawHub-only close.

## What This Changes

Add reusable config-defined sub-agent templates that combine a prompt file with model, timeout, and label defaults, then let `sessions_spawn` select a template while preserving explicit parameter precedence.

## Best Possible Solution

Maintainers should decide the product/API boundary first. If accepted for core, add an additive typed template contract with safe prompt-file resolution relative to the selected agent workspace, bounded reads, explicit override precedence, template discovery, generated schema/docs updates, and focused tests; otherwise document the skill/plugin orchestration pattern and leave the built-in tool contract unchanged.
