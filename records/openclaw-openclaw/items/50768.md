## Summary

Keep open. Current main still supports explicit `sessions_spawn` parameters plus global/per-agent sub-agent defaults for model, thinking, timeout, allowlists, and discovery, but it does not implement reusable sub-agent template definitions, a `sessions_spawn.template` input, prompt-file prepending, or template discovery. The request is concrete, but it changes public config/tool API and would add workspace file reads into model context, so it needs maintainer product/API and security-boundary review before implementation.

## What This Changes

The issue requests reusable sub-agent templates that provide a prompt file plus model, timeout, and label defaults for repeated `sessions_spawn` calls.

## Best Possible Solution

Maintainers should decide the product/API boundary first. If accepted for core, add an additive typed template contract with safe prompt-file resolution relative to the selected agent workspace, bounded reads, explicit override precedence, template discovery, generated schema/docs updates, and focused tests; otherwise document the supported skill/plugin orchestration pattern and leave the built-in `sessions_spawn` contract unchanged.
