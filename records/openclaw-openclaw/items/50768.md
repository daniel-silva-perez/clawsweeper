## Summary

Keep open. Current main still has explicit `sessions_spawn` parameters plus global/per-agent sub-agent defaults, but it does not implement reusable sub-agent template definitions, a `sessions_spawn.template` input, prompt-file prepending, or template discovery. The request is concrete, but it changes public config/tool API and introduces workspace file reads into model context, so it needs maintainer product/API and security-boundary review before implementation.

## What This Changes

Add reusable sub-agent templates under `agents.list[].subagents` with prompt/model/timeout/label defaults and a `sessions_spawn.template` parameter that prepends template prompt content while explicit spawn args override defaults.

## Best Possible Solution

Maintainers should decide the product/API boundary first. If accepted for core, add an additive typed template contract with safe prompt-file resolution relative to the selected agent workspace, bounded reads, explicit override precedence, template discovery, generated schema/docs updates, and focused tests; otherwise document the supported skill/plugin orchestration pattern and leave the built-in `sessions_spawn` contract unchanged.
