## Summary

Keep open. Current main still has explicit sub-agent spawn params plus global/per-agent defaults for model, thinking, timeout, allowlists, and discovery, but it does not implement reusable sub-agent templates, a `sessions_spawn.template` parameter, or prompt-file prepending. The external AIpexForge/agent-swarm#19 cross-reference is useful prior art, and open PR #72128 is a narrower label-template proposal, but neither resolves the requested prompt/model/timeout template API in OpenClaw.

## What This Changes

The issue requests reusable `agents.list[].subagents.templates` entries and a `sessions_spawn.template` parameter that prepends a configured prompt file and applies default model, timeout, and label values unless explicit spawn parameters override them.

## Best Possible Solution

Decide the product/API boundary first. If accepted for core, add an additive typed template contract with explicit override precedence, safe prompt-file resolution relative to the selected agent workspace, bounded reads, discovery, docs, generated schema updates, and focused tests. If maintainers prefer keeping this as orchestration policy, document the skill/plugin pattern and leave the built-in `sessions_spawn` contract unchanged.
