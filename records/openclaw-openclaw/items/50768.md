## Summary

Keep open. Current main still supports explicit `sessions_spawn` parameters and global/per-agent defaults for model, thinking, timeout, allowlists, and discovery, but it does not provide reusable prompt/model/timeout sub-agent templates, a `sessions_spawn.template` parameter, prompt-file prepending, or template discovery. Because the missing behavior touches public config/tool API and secure prompt-file resolution, this needs maintainer product/API review rather than stale cleanup or a ClawHub-only close.

## What This Changes

Request to add reusable sub-agent templates under `agents.list[].subagents.templates`, plus a `sessions_spawn.template` parameter that prepends a prompt file and supplies model, timeout, and label defaults.

## Best Possible Solution

Decide the product/API boundary first. If accepted for core, add an additive typed template contract with safe prompt-file resolution relative to the selected agent workspace, bounded reads, explicit override precedence, discovery, docs, generated schema updates, and focused tests; otherwise document the skill/plugin orchestration pattern and leave the built-in tool contract unchanged.
