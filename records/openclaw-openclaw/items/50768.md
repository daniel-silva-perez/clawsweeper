## Summary

Keep open. Current main still supports explicit sub-agent spawn parameters plus global/per-agent defaults for model, thinking, timeout, allowlists, and discovery, but it does not implement reusable sub-agent templates, a `sessions_spawn.template` parameter, or prompt-file prepending. The linked AIpexForge/agent-swarm#19 item is useful external prior art, not an OpenClaw implementation.

## What This Changes

The issue requests reusable sub-agent templates in agent config, with prompt-file, model, timeout, and label defaults applied by a new `sessions_spawn` template parameter.

## Best Possible Solution

Keep this issue open for maintainer product/API review. If accepted for core, implement an additive typed `agents.list[].subagents.templates` contract plus `sessions_spawn.template`, with explicit tool params overriding template defaults, prompt-file resolution relative to the selected agent workspace, bounded and normalized file reads, discovery via `agents_list` or a narrow new field, and schema/docs/tests updates. If maintainers decide this is orchestration policy rather than core platform API, document a ClawHub/plugin pattern instead.
