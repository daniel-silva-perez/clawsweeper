## Summary

Keep open. Current `origin/main` still supports explicit `sessions_spawn` model/thinking/timeout/label parameters plus global and per-agent sub-agent defaults, but it does not implement reusable sub-agent template definitions, a `sessions_spawn.template` input, prompt-file prepending, or template discovery. The request is clear and still relevant, but it changes public config/tool API and would introduce workspace file reads into model context, so it needs maintainer product/API review rather than stale cleanup or a ClawHub-only close.

## What This Changes

Add reusable sub-agent template definitions with prompt/model/timeout/label defaults and a `sessions_spawn` template parameter that can prepend a prompt file and allow explicit call parameters to override template defaults.

## Best Possible Solution

Maintainers should decide the product/API boundary first. If accepted for core, implement an additive typed template contract with safe prompt-file resolution relative to the selected agent workspace, bounded reads, explicit override precedence, template discovery, generated schema/docs updates, and focused tests; otherwise document the supported skill/plugin orchestration pattern and leave the built-in `sessions_spawn` contract unchanged.
