## Summary

Keep open. Current main still has explicit `sessions_spawn` parameters and global/per-agent sub-agent defaults, but not reusable sub-agent template definitions, `sessions_spawn.template`, prompt-file prepending, or template discovery. The request is concrete, but it changes public config/tool API and introduces workspace prompt-file reads into model context, so it needs maintainer product/API and security-boundary review before implementation.

## What This Changes

Request to add reusable sub-agent templates under `agents.list[].subagents` with prompt-file, model, timeout, and label defaults plus a `sessions_spawn` template parameter.

## Best Possible Solution

Maintainers should decide the product/API boundary first. If accepted for core, add an additive typed template contract with safe prompt-file resolution relative to the selected agent workspace, bounded reads, explicit override precedence, template discovery, generated schema/docs updates, and focused tests; otherwise document the supported skill/plugin orchestration pattern and leave the built-in `sessions_spawn` contract unchanged.
