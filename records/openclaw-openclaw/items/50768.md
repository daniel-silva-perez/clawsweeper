## Summary

Keep open. Current main still has sub-agent defaults for model, thinking, timeout, allowlists, and target discovery, but it does not implement reusable sub-agent templates, a `sessions_spawn.template` parameter, or prompt-file prepending. The AIpexForge/agent-swarm#19 cross-reference is useful prior art, but it is external and not an OpenClaw implementation.

## Best Possible Solution

Keep this issue open for maintainer product/API review. If accepted for core, implement an additive `sessions_spawn.template` flow backed by typed `agents.list[].subagents.templates`, with prompt-file resolution relative to the selected agent workspace, explicit tool params overriding template defaults, discovery via `agents_list` or a narrow new field, security limits for file reads, and aligned schema/docs/tests. If maintainers decide this is orchestration policy rather than core platform API, document a ClawHub/plugin pattern instead.
