## Summary

Keep open. Current main has sub-agent defaults and target-agent selection, but it does not implement reusable named sub-agent templates: there is no `sessions_spawn.template` parameter, no `subagents.templates` config schema, no `promptFile` resolution, and no template discovery through `agents_list`. The related external AIpexForge/agent-swarm cross-reference is useful context for the orchestration use case, but it is not an OpenClaw core implementation.

## Best Possible Solution

Keep the issue open for a maintainer product/API decision. A good implementation would add an additive, validated `subagents.templates` schema, extend `sessions_spawn` with a `template` selector, resolve `promptFile` safely relative to the agent workspace, merge template defaults with explicit call parameters taking precedence, expose templates through `agents_list` or a dedicated discovery tool, and cover the behavior in focused config/tool tests plus docs.
