## Summary

Keep open. Current main still supports explicit `sessions_spawn` arguments plus global and per-agent sub-agent defaults, but it does not implement reusable sub-agent templates, a `sessions_spawn.template` selector, prompt-file prepending, or template discovery. The request is concrete and supported by linked orchestration context, but it changes public config/tool contracts and introduces workspace file reads into model context, so it needs maintainer product/API and security-boundary review before implementation.

## What This Changes

Request adding reusable sub-agent templates under `agents.list[].subagents` with `promptFile`, model, timeout, and label defaults plus a `sessions_spawn.template` selector whose values can be overridden by explicit call arguments.

## Best Possible Solution

Maintainers should decide the product/API boundary first. If accepted for core, implement an additive typed template contract with safe workspace-relative prompt-file resolution, bounded reads, explicit override precedence, template discovery, generated schema/docs updates, and focused tests; otherwise document the supported skill/plugin orchestration pattern and leave the built-in tool contract unchanged.
