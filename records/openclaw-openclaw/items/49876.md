## Summary

Keep open. Current main has several relevant cron hardening fixes, but it still does not implement the requested fail-closed cron/session delivery policy for fabricated final assistant text, missing tool evidence, or marker phrases like "placeholder", "mock data", and "simulate".

## Best Possible Solution

Keep this issue open for maintainer design. The best path is a narrow, opt-in unattended cron/session delivery policy in core, or a small plugin API extension if maintainers decide the policy belongs in ClawHub. Acceptance should include regression tests proving that failed/missing required tool evidence, unsupported success claims, and self-referential draft markers are suppressed or surfaced as clean cron failures before announce/webhook delivery.
