## Summary

Keep open. Current main still has no schema-backed config policy for opt-in group-chat tool-result delivery: ordinary group text summaries are suppressed by a hard-coded surface gate, the config schemas do not expose the requested option, and related #44027 is only an open `/verbose` behavior candidate rather than the requested configuration policy.

## What This Changes

The issue requests a typed configuration option to let selected group chats, or channel defaults, opt into verbose tool-result delivery while preserving silent group behavior by default.

## Best Possible Solution

Add a narrow, typed, fail-closed group tool-result delivery policy that can be set through channel defaults and channel/group/topic overrides, wire it through dispatch and ACP projection without changing media or exec-approval exceptions, cover ordinary groups and forum topics with regression tests, and document the trusted-room security model.
