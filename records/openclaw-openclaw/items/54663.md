## Summary

Keep this issue open. Current main still has no schema-backed channel/default/group config option for allowing verbose tool-result delivery in ordinary group chats. The dispatch path still derives tool-summary delivery from chat surface state, suppresses text-only tool summaries for non-forum groups, and passes the same unresolved boolean into ACP/plugin dispatch. The already-posted Codex review and related open PR #44027 are useful context, but #44027 is only an open `/verbose` behavior candidate and does not implement the requested config policy.

## Best Possible Solution

Keep this issue open. The best path is a narrow typed config policy, defaulting to false, with channel defaults plus provider/group/topic overrides where appropriate. Wire that policy into `dispatch-from-config` and ACP/plugin projection, add regression tests proving ordinary groups stay silent by default but deliver text summaries when explicitly enabled, preserve media and exec-approval behavior, and update public channel/verbose docs with a trusted-group warning while coordinating with #44027.

## Risks / Open Questions

- Verbose tool-result output in shared chats can expose commands, paths, URLs, plugin diagnostics, and tool-returned data. Any implementation should default to false and document trusted-group use only.
- The config policy should be coordinated with #44027 so OpenClaw does not grow inconsistent mechanisms for group tool-result delivery.
