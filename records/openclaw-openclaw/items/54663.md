## Summary

Keep open. Current main still hard-codes ordinary group chats to suppress text tool-result summaries, does not expose the requested channel-default or Telegram group/topic config option, and the adjacent open PR #44027 only covers `/verbose` behavior rather than this typed configuration policy.

## What This Changes

The issue requests an opt-in config option that allows verbose tool-call results to be delivered in group chats while keeping the default suppressed.

## Best Possible Solution

Add a narrow typed group tool-result delivery policy that defaults to false, supports channel defaults plus channel/group/topic overrides, feeds the resolved policy into normal dispatch and ACP projection, preserves media and exec-approval exceptions, covers ordinary groups and forum topics with tests, and documents the trusted-group security model.
