## Summary

Keep this PR open. Current main has the related fresh-session transcript rotation behavior, but it still lacks the archive-on-rotation flow for prior heartbeat and isolated cron transcript files; the PR contains unique remaining work and needs revision for the cron run-history reference-set issue before merge.

## What This Changes

The PR adds archive-on-rotation helpers and tests so prior isolated-session heartbeat and cron transcripts are renamed into `.reset` archives, while separating heartbeat stale-suffix deletion archival from reset archival.

## Best Possible Solution

Keep the shipped fresh-session behavior, then land a narrow archive-on-rotation implementation that captures the prior entry before overwrite, persists the new entry, archives the prior transcript with `reason: "reset"` and `restrictToStoreDir: true`, and treats historical cron run rows as inactive for that reset-archive reference check.
