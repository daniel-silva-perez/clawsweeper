## Summary

Keep this PR open. Current main has the fresh-session transcript rotation behavior from the related merged work, but it still does not archive the prior isolated heartbeat or cron transcript on rotation. The PR targets a real remaining gap, but the current diff still has a blocking cron reference-set bug and needs maintainer/author revision before merge.

## What This Changes

The PR adds archive-on-rotation helpers and tests so prior isolated cron and heartbeat transcripts are renamed into `.reset` archives, while separating heartbeat suffix-collapse deletion archival from reset archival.

## Best Possible Solution

Keep the shipped fresh-session behavior, then land a narrow archive-on-rotation implementation that captures the prior entry before overwrite, persists the new entry, archives the prior transcript with `reason: "reset"` and `restrictToStoreDir: true`, and excludes historical cron run rows from the reset-archive reference set.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a store containing the prior stable cron session entry plus an older `...:run:<id>` row with the same prior `sessionId`; the PR helper includes that row in `referencedSessionIds`, and `archiveRemovedSessionTranscripts` skips the reset rename.

## Solution Assessment

No. The direction is right, but the current PR is not the best mergeable solution until it narrows the reset reference set to active rows or otherwise ignores historical cron run rows, then rebases onto current main's stable-key persistence behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Ignore historical run rows before reset archival:** `src/cron/isolated-agent/session.ts:148-152`
  - body: `referencedSessionIds` is built from every session-store row, but isolated cron runs can leave historical `...:run:<id>` rows that still reference the just-rotated `sessionId`. When that happens, `archiveRemovedSessionTranscripts` skips the rename, so the new reset-archive path is a no-op for the common cron flow. Build the reset reference set from active rows or exclude run-history keys for this rotation archive path.
  - confidence: 0.91
