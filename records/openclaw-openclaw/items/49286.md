## Summary

Keep this PR open. Current main still does not implement the PR's intended Discord native system-message audit: the Discord mapper still omits ChannelNameChange, ChannelIconChange, Call, ChannelFollowAdd, discovery warnings, invite reminders, role/app subscription notices, StageRaiseHand, and related MessageType cases, while preflight only drops/enqueues messages that the mapper resolves. The overlapping #49308 is open, narrower, and by the same author, so it is coordination work rather than a shipped superseder.

## Best Possible Solution

Keep this PR open and review it as the broader Discord system-message fix, or explicitly choose #49308 as the smaller canonical fix if maintainers only want rename/icon/follow coverage. Before landing, rebase onto current main, update the enqueue-system-event mock to system-event-runtime, add the required changelog attribution, and keep regression coverage for both the mapper and the preflight drop/enqueue path.
