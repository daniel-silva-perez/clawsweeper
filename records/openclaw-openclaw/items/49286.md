## Summary

Current main still does not implement the Discord native system-message mapping this PR proposes. The preflight path still only queues and drops system events when resolveDiscordSystemEvent returns text, and the mapper still falls through to null for ChannelNameChange, ChannelIconChange, ChannelFollowAdd, Call, GuildDiscovery*, GuildInviteReminder, RoleSubscriptionPurchase, InteractionPremiumUpsell, StageRaiseHand, and GuildApplicationPremiumSubscription. The related narrower #49308 is open and unmerged, so it is an implementation candidate, not a superseder.

## Best Possible Solution

Keep this PR open for maintainer review as the broader Discord native system-message audit, or explicitly choose open #49308 as the narrower canonical fix. The implementation path should add the missing MessageType cases in extensions/discord/src/monitor/system-events.ts, keep user-authored/content-carrying types as passthroughs, add mapper and preflight enqueue/drop regression tests against the current system-event-runtime import, and fix the changelog attribution before landing.
