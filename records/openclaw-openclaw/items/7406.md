## Summary

Keep open. Current main has the Telegram topic-name source and exposes it to prompt/plugin metadata, but the stored session metadata, gateway session row, and Control UI selector still do not carry or prefer a durable human-readable Telegram topic label, so the reported dropdown problem remains valid.

## What This Changes

The issue asks OpenClaw to show Telegram forum topic sessions with human-readable group and topic names in the session dropdown instead of raw topic-qualified session keys.

## Best Possible Solution

Add an additive, generic thread/topic display-label path through session metadata and sessions.list, populate it from Telegram's cached TopicName for forum-topic sessions, and have the Control UI selector prefer that explicit label while preserving existing non-Telegram fallbacks.
