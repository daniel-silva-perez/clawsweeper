## Summary

Keep open. Current main has the Telegram topic-name data source, but it still stops at inbound prompt/context metadata and is not carried through stored session display metadata, gateway session rows, or the Control UI session selector.

## What This Changes

Display Telegram forum-topic sessions with human-readable topic names in the session dropdown instead of raw or group-only topic identifiers.

## Best Possible Solution

Add a narrow generic session display-label/thread-label path for known thread or topic names, have the Telegram plugin populate it from the existing cached TopicName for forum-topic sessions, expose it through sessions.list, and make the Control UI selector prefer that explicit label while preserving existing non-Telegram labels.
