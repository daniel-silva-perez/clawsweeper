## Summary

Keep open. Current main still has the data source for Telegram forum topic names, but the value stays in inbound prompt/context metadata and is not carried into stored session display metadata, gateway session rows, or the Control UI selector. The recent issue discussion also treats this as the canonical tracker: duplicate reports were closed in favor of it, the direct implementation PRs are closed unmerged, and the remaining open UI cleanup PR is adjacent rather than a complete metadata fix.

## What This Changes

Display Telegram forum topic sessions with human-readable group/topic labels in the session dropdown instead of raw topic-qualified session keys.

## Best Possible Solution

Add a narrow generic session display-label path for known thread/topic names, have the Telegram plugin populate it from the existing cached TopicName for forum-topic sessions, expose it through sessions.list, and make the Control UI selector prefer that explicit label while preserving existing non-Telegram session labels.
