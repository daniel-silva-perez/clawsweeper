## Summary

Current main still has the reported Telegram forum-topic label gap. Telegram now learns and exposes `TopicName` for inbound context and prompt metadata, but that value is not carried into stored session metadata, gateway session rows, or the Control UI session selector label path.

## Best Possible Solution

Keep this issue open as the canonical tracker and implement a narrow Telegram/session metadata fix: reuse the existing cached Telegram `TopicName`, persist or derive a generic topic/thread display label for topic-qualified sessions, expose it through `sessions.list`, and update the Control UI selector to prefer explicit `label`/`displayName` before the automatic topic label. Add focused tests for Telegram topic-name capture/cache reuse, session metadata persistence, gateway row projection, and UI label resolution.
