## Summary

Keep open. Current main already learns Telegram forum topic names and exposes them to inbound prompt metadata, but the name still does not flow into durable session display metadata, gateway session rows, or the Control UI selector, so the reported dropdown problem remains. The related implementation PRs were closed unmerged, and open PR #51124 is adjacent selector cleanup rather than the missing metadata path.

## What This Changes

The issue requests human-readable Telegram forum topic names in the session dropdown instead of raw topic-qualified session keys.

## Best Possible Solution

Add a narrow generic thread/topic display-label path in session metadata, populate it from Telegram's cached TopicName for forum-topic sessions, expose it through sessions.list, and make the Control UI prefer that explicit label while preserving existing non-Telegram fallbacks.
