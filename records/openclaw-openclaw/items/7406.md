## Summary

Current main still does not carry Telegram forum `TopicName` into a stored session label or Control UI session selector label. The issue remains valid: Telegram now learns and exposes topic names in message context, but the session metadata, gateway session row, and UI selector still use `label` / `displayName` / generic group fallback fields that do not include the known topic name.

## Best Possible Solution

Keep this issue open as the canonical tracker and implement a narrow Telegram/session metadata fix: reuse the existing cached `TopicName`, persist or derive a human topic/thread display label for topic-qualified sessions, expose it through `sessions.list`, and update the Control UI selector to prefer explicit labels while showing the topic name when known.
