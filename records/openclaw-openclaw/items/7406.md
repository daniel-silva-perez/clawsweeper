## Summary

Keep open. Current main learns Telegram forum topic names and exposes them to inbound/prompt metadata, but the title still is not persisted into durable session display metadata or preferred by the gateway/UI session selector, so the reported dropdown problem remains actionable.

## What This Changes

Show Telegram forum-topic sessions in the session dropdown with human-readable group/topic labels instead of raw topic-qualified session keys.

## Best Possible Solution

Add a narrow generic thread/topic display label in session metadata, populate it from Telegram's cached TopicName for forum-topic sessions, expose it through sessions.list, and have the Control UI prefer that explicit label while preserving existing non-Telegram fallbacks.
