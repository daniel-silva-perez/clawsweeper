## Summary

Keep open. Current main already learns Telegram forum topic names and exposes them to inbound prompt/hook metadata, but that value is not carried into stored session display metadata or preferred by the gateway/UI session selector, so the user-visible dropdown problem remains.

## What This Changes

Request to show Telegram forum-topic sessions with human-readable group/topic labels instead of raw topic-qualified session keys or indistinguishable group-only labels.

## Best Possible Solution

Add a narrow generic thread/topic display-label path in session metadata, have the Telegram plugin populate it from the existing cached TopicName for forum-topic sessions, expose it through sessions.list, and make the Control UI selector prefer that explicit label while preserving existing non-Telegram labels and topic-id fallback behavior.
