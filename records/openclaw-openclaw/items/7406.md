## Summary

Current main still has the reported gap. Telegram learns `TopicName` for inbound agent context, but session metadata, gateway session rows, and the Control UI session dropdown still derive labels from group subject/channel/id without a topic-name display field.

## Best Possible Solution

Keep this issue open as the canonical tracker and implement a narrow Telegram/session metadata fix: reuse cached Telegram `TopicName` from inbound context, persist or derive topic display metadata for topic-qualified sessions, expose it through gateway `sessions.list`, and let the Control UI session dropdown prefer explicit label/displayName before the automatic topic label. Add focused tests for Telegram topic extraction/cache reuse, session metadata persistence, gateway row projection, and Control UI label resolution.
