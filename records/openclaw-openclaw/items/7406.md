## Summary

Keep open. Current main captures Telegram forum topic names and carries them into inbound/prompt metadata, but the value still is not persisted as durable session display metadata or surfaced through the gateway/UI session selector, so the reported dropdown problem remains valid. The direct implementation PRs are closed unmerged, and the open adjacent UI cleanup PR does not add the missing Telegram topic metadata path.

## What This Changes

Show Telegram forum topic sessions in the session dropdown with human-readable group/topic labels instead of raw topic-qualified keys or numeric topic IDs.

## Best Possible Solution

Add a narrow generic thread/topic display-label path in session metadata, populate it from Telegram's cached topic name for forum-topic sessions, expose it through sessions.list, and have the Control UI prefer that explicit label while preserving existing non-Telegram label fallbacks.
