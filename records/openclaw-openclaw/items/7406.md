## Summary

Keep open. Current main can learn Telegram forum topic names and exposes them as `TopicName` for prompt/plugin metadata, but it still does not persist a durable topic display label into session metadata, the gateway session row, or the Control UI selector, so the dropdown problem remains actionable. The related implementation PRs were closed unmerged, and the remaining open UI PR is adjacent cleanup rather than a complete source-to-UI topic-label fix.

## What This Changes

Show human-readable Telegram forum topic names in session dropdowns instead of raw or indistinguishable topic-qualified session keys.

## Best Possible Solution

Add an additive, generic thread/topic display-label path through session metadata and `sessions.list`, populate it from Telegram's cached `TopicName` for forum-topic sessions, and have the Control UI selector prefer that explicit label while preserving existing non-Telegram fallbacks.
