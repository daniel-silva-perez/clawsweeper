## Summary

Keep this issue open. Current main still routes every non-Claude GitHub Copilot model through `openai-responses`, and the docs explicitly say Gemini uses Responses, so the reported Gemini `/responses` failure is not fixed. The second header path appears partly addressed in current core transport code because OpenAI-completions clients now inject Copilot IDE headers for `provider === "github-copilot"`, but there is no focused regression coverage for Copilot chat-completions and the primary Gemini routing bug remains.

## Best Possible Solution

Keep this issue open and make a focused GitHub Copilot provider fix: route Gemini/chat-completions-only Copilot models to `openai-completions`, preserve Copilot IDE headers on that transport with regression coverage, update the GitHub Copilot docs and changelog, and leave the optional repeated-fallback spend warning as a separate product follow-up unless maintainers want it bundled.
