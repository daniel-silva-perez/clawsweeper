## Summary

Keep open. Current main still sends GitHub Copilot Gemini models through the Responses transport by default, so the primary reported failure remains valid; the manual completions header path appears partly addressed in shared transport code but lacks the provider regression coverage needed to consider this resolved.

## What This Changes

The issue requests that GitHub Copilot Gemini and other chat-completions-only Copilot models use the completions endpoint with required Copilot IDE headers instead of failing through `/responses` and falling back to paid providers.

## Best Possible Solution

Land a provider-local GitHub Copilot fix that routes Gemini and other confirmed chat-completions-only Copilot model refs to `openai-completions`, proves Copilot IDE headers on that transport, updates provider docs and changelog, and tracks any repeated fallback-cost warning separately.
