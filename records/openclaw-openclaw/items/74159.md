## Summary

Keep this issue open. Current main still routes every non-Claude GitHub Copilot dynamic/default model through `openai-responses`, so Gemini/chat-completions-only Copilot models still hit the unsupported Responses path. The shared OpenAI-completions client now injects Copilot IDE headers for `github-copilot`, but the Gemini routing failure remains and tests/docs still encode the old behavior.

## Best Possible Solution

Make a focused GitHub Copilot provider fix: route Gemini/chat-completions-only Copilot model ids such as `gemini-3-flash-preview`, `gemini-3.1-pro-preview`, and `gemini-2.5-pro` to `openai-completions`; keep Claude on `anthropic-messages` and verified GPT/o/Codex models on Responses; add regression coverage that Copilot IDE headers are present on the completions path; update the GitHub Copilot docs and changelog. Leave the repeated fallback spend warning as a separate product follow-up.
