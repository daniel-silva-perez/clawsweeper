## Summary

Keep open. Current main has fixed the session-store cache/load structuredClone hot path, but auth-profile cache and runtime snapshot clone helpers still call structuredClone, and maintainer discussion explicitly kept the item open for that remaining path plus benchmark/soak confirmation. The remaining work touches credential and OAuth snapshot semantics, so it should go through maintainer review rather than an automatic close.

## What This Changes

Replace structuredClone with JSON-shaped cloning or serialized reuse in session and auth store clone paths to reduce native memory growth and gateway OOMs.

## Best Possible Solution

Finish or explicitly defer the auth-profile clone cleanup in the auth-profile owner area, add focused mutation-isolation/no-structuredClone coverage for persisted and runtime auth stores, and validate with the reported large-store benchmark/soak before closing. Broader gateway memory accumulation should remain tracked on #54155 and #51264.
