## Summary

Keep open. Current main has fixed the reported session-store cache/load structuredClone hot path, but auth-profile cache, runtime snapshot, and external OAuth overlay paths still use structuredClone, and maintainer discussion explicitly kept the issue open for that remaining work plus benchmark or soak confirmation.

## What This Changes

Replace structuredClone-based cloning in session-store and auth-profile hot paths with JSON-equivalent cloning or serialized JSON reuse to reduce Gateway native memory/RSS growth.

## Best Possible Solution

Finish or explicitly defer the auth-profile clone cleanup in the auth-profile owner area, add focused mutation-isolation and no-structuredClone coverage for persisted/runtime auth stores, then validate with the reported large-store benchmark or a gateway soak before closing. Keep broader Gateway memory accumulation tracked on #54155 and #51264.
