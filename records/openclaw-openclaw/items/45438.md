## Summary

Keep open. Current main has fixed the session-store cache/load structuredClone paths, but the auth-profile store and runtime snapshot clone paths still call structuredClone, and a maintainer explicitly kept this open for that remaining cleanup plus benchmark/soak confirmation.

## What This Changes

Request to replace structuredClone cloning in session-store and auth-profile cache hot paths with JSON-shaped cloning to reduce Node native-memory RSS growth and gateway OOM risk under large stores.

## Best Possible Solution

Finish or explicitly defer the auth-profile structuredClone cleanup in the auth-profile owner area, add focused mutation-isolation/no-structuredClone coverage for persisted and runtime auth stores, and then validate the reported large-store workload before closing. Broader gateway memory accumulation should remain tracked on #54155 and #51264.
