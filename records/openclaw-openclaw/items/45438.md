## Summary

Keep open. Current main has removed the reported session-store cache/load structuredClone hot path, but the auth-profile store/cache and runtime auth snapshot paths still call structuredClone, and a maintainer explicitly kept the issue open for that remaining cleanup plus benchmark/soak confirmation.

## What This Changes

Replace structuredClone-based cloning in session/auth store hot paths with JSON-safe cloning to reduce native memory growth and gateway OOMs under large session stores.

## Best Possible Solution

Finish or explicitly defer the auth-profile clone cleanup in the auth-profile owner area, add focused mutation-isolation/no-structuredClone coverage for persisted and runtime auth stores, and then run the reported large-store benchmark/soak before closing. Broader gateway memory accumulation should stay on #54155 and #51264.
