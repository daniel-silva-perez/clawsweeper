## Summary

Keep open. Current main has addressed the session-store cache/load structuredClone hot path, but auth-profile cache, runtime snapshot, and external OAuth overlay clone paths still use structuredClone, and a maintainer explicitly kept the issue open for that remaining work plus benchmark or soak confirmation.

## What This Changes

The issue requests replacing structuredClone in large JSON-shaped session/auth store clone paths with JSON-backed cloning or serialized-store reuse to reduce native-memory growth and gateway OOM risk.

## Best Possible Solution

Finish or explicitly defer the auth-profile clone cleanup in the auth-profile owner area, add focused no-structuredClone and mutation-isolation coverage for persisted/runtime auth stores, then validate with the reported large-store benchmark or a production-shaped gateway soak before closing.
