Keep open. Current main fixed the session-store cache/load structuredClone hot path, but the auth-profile cache and runtime auth snapshot helpers still clone with structuredClone, and maintainer discussion explicitly kept this issue open for that remaining path plus benchmark or soak confirmation.

## What This Changes

The issue requests replacing structuredClone-based cloning in session/auth-profile store hot paths to stop gateway native-memory/RSS growth and OOM crashes.

## Best Possible Solution

Finish or explicitly defer the auth-profile clone cleanup in the auth-profile owner area, add mutation-isolation and no-structuredClone coverage for persisted and runtime auth stores, then validate with the reported large-store benchmark or a production-shaped gateway soak before closing. Keep broader gateway memory accumulation tracked on #54155 and #51264.
