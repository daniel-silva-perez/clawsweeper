## Summary

Keep open. Current main exposes useful session/subagent lineage, status, timing, token, and task metadata, but the user-facing surfaces still render flat lists rather than a CLI or TUI hierarchy for active agents and child sessions.

## What This Changes

The issue requests a new `openclaw tree` command with optional JSON, watch, session-detail, and resource-usage views for active agent/session hierarchies.

## Best Possible Solution

Add a core inspection surface that renders the existing session/subagent lineage as a stable hierarchy, with text and `--json` output first, plus focused tests and docs. Treat live watch mode and per-process resource accounting as follow-up work unless maintainers define those contracts upfront.
