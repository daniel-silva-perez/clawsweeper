## Summary

Keep open. Current main exposes useful lineage, status, timing, token, and cost metadata for sessions and subagents, but the user-facing surfaces still render flat session/task/subagent lists rather than a CLI or TUI hierarchy of active agents and their child sessions.

## What This Changes

Request a user-facing `openclaw tree` style command to visualize active agents, subagents, session bindings, status, and resource details in tree, JSON, and watch modes.

## Best Possible Solution

Add a core inspection surface that renders a session/agent hierarchy from existing `sessions.list` and subagent metadata, with stable text and `--json` output first, plus focused tests and docs. Treat live watch mode and per-process CPU/RSS accounting as follow-up work unless maintainers first define the data contract.
