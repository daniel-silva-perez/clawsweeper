## Summary

Keep open. Current main has session and subagent lineage metadata, but the CLI, slash-command, docs, and Control UI surfaces still render flat session/subagent lists rather than the requested active-agent hierarchy. The remaining work needs maintainer direction on command namespace and whether live watch/resource accounting belong in the first version.

## What This Changes

Add a user-facing tree view command for active agents and subagents, including hierarchy, session status, optional JSON/watch output, session details, and resource usage.

## Best Possible Solution

Add a core inspection surface that renders the existing session/subagent lineage as a stable hierarchy, with text and `--json` output first, focused tests, and CLI docs. Treat live watch mode and per-process CPU/RSS metrics as follow-up work unless maintainers define those contracts upfront.
