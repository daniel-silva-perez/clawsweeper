## Summary

Keep this issue open. Current main has prompt-cache shaping, cache-break telemetry, cache-ttl pruning, and a documented heartbeat workaround, but it does not implement the requested invisible non-turn keep-warm ping mechanism, opt-in `promptCacheKeepWarm` config, adaptive TTL timer, cost cap, or Anthropic `max_tokens: 1` warm request path between user turns. The related work is useful foundation rather than a superseding implementation: #62179 exposed prompt-cache telemetry, #62501 was closed unmerged and only proposed timestamps, and open #56575 covers compact/reset session maintenance rather than ping-based cache refresh.

## Best Possible Solution

Keep the issue open as the canonical request for core prompt-cache keep-warm pings. A maintainer-quality implementation would likely live in the agent/gateway runtime, reuse the existing Anthropic payload policy and prompt-cache observability, add an opt-in config surface with cost/TTL safeguards, document the behavior in Prompt Caching, and include focused tests proving no transcript turn/tool call/session-history entry is created.
