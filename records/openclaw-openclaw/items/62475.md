## Summary

Keep open. Current main has related prompt-cache controls, observability, cache-ttl pruning, and heartbeat keep-warm guidance, but it does not implement the requested invisible non-turn keep-warm ping mechanism; the existing heartbeat path is a full agent turn. The feature also adds autonomous paid provider traffic, so it needs maintainer product/cost review before implementation.

## What This Changes

The issue requests an opt-in Anthropic prompt-cache keep-warm mechanism that sends invisible minimal pings between user turns to avoid cache TTL expiry.

## Best Possible Solution

Define a maintainer-approved opt-in provider-owned keep-warm mode, or explicitly decide that heartbeat plus cache-ttl pruning is the supported strategy. If approved, the implementation should reuse the existing Anthropic payload policy, avoid transcript/history mutation, expose bounded cost controls, and document how it differs from heartbeat.

Unclear. The requested ping design is plausible, but current main’s heartbeat workaround is not equivalent because it runs full agent turns; the narrowest maintainable solution needs a maintainer decision on background paid provider traffic before code work starts.
