## Summary

Keep open. Current main still lacks the central macOS VM auto-login guidance in the English VM docs and LaunchAgent bootstrap error, but the PR needs refresh before merge because it edits obsolete localized docs and the runtime auto-login hint is too broad for a security-sensitive setting.

## What This Changes

The PR adds macOS VM auto-login guidance to the English and Chinese macOS VM setup docs and adds an auto-login hint to the macOS LaunchAgent bootstrap failure message.

## Best Possible Solution

Refresh the PR into an English-source-docs-only change: add scoped post-setup auto-login guidance for isolated headless macOS VMs, decide whether the generic LaunchAgent bootstrap error should mention that VM-specific workaround, update the launchd guidance test if the runtime text changes, and leave localized docs to the i18n publish pipeline.

## Reproduction Assessment

Unclear for the live headless macOS VM bootstrap failure because I did not run a macOS VM reproduction. Static verification against current main is high confidence: the docs, runtime error, and current test all lack the proposed auto-login guidance.

## Solution Assessment

No, not as-is. The intended guidance is useful, but the maintainable path is to drop the docs/zh-CN edit, scope the auto-login recommendation to isolated VM use, and add test coverage for any runtime message change.
