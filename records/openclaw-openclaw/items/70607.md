## Summary

Keep this PR open. Current main does not implement the PR's intended behavior: `resolveRequesterOriginForChild` still resolves the target agent's bound account for cross-agent spawns, and current tests assert that behavior. However, the proposed one-character guard flip is not clearly safe because it would make same-agent spawns re-resolve bindings and override the active requester account, contradicting current same-agent coverage and the nearby comment. A correct fix likely needs to target the completion/announce misroute separately from the child session's initial routing.

## Best Possible Solution

Keep this PR open for maintainer review, but require a revised fix. The safer direction is to preserve current same-agent active-account behavior, preserve or deliberately re-evaluate cross-agent target-account routing for the child run itself, and add focused coverage for the actual completion announce misroute from linked issue #70574. If another PR lands that targeted fix first, this PR can then be closed as superseded or implemented on main.
