## Summary

Keep open. The original Feishu-specific log text is gone on current main, but the current embedded tool lifecycle logger still cannot attribute tool start/end logs to the originating Telegram/Discord/WhatsApp/etc. channel because the subscription layer receives no channel origin and logs through a static `agent/embedded` logger.

## What This Changes

The issue asks for tool call lifecycle logs to use the subsystem of the channel that originated the request instead of being attributed to the wrong channel/logger.

## Best Possible Solution

Thread the existing originating channel/provider identity into the embedded subscription layer, derive a normalized `gateway/channels/<channel>` logger for valid chat-channel runs, keep `agent/embedded` as the fallback, and add regression tests that prove non-Feishu channel attribution while preserving sanitized tool event logging.
