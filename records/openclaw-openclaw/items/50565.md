## Summary

Keep open. The old Feishu-specific `tool call` strings are gone on current `main`, but the same attribution bug remains in the embedded tool lifecycle path: tool start/end debug logs inherit a static `agent/embedded` logger and the subscription layer still receives no originating channel identity.

## What This Changes

The issue requests that tool lifecycle log entries be tagged with the originating channel subsystem, such as `gateway/channels/telegram`, instead of a static Feishu or embedded subsystem.

## Best Possible Solution

Add a typed channel-origin or logger seam from `runEmbeddedAttempt` into `subscribeEmbeddedPiSession`, derive a safe channel subsystem logger for valid message-channel runs, keep `agent/embedded` as the fallback, and cover tool start/end attribution with focused regression tests.
