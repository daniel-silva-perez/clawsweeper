## Summary

Keep open. Current main no longer contains the exact old `feishu: tool call` text, but the current embedded tool lifecycle path still logs through a static `agent/embedded` logger and the subscription API still has no originating channel/provider field, so tool lifecycle log attribution remains unresolved.

## What This Changes

The issue asks for tool call lifecycle logs to be tagged with the originating messaging channel subsystem instead of a hardcoded or static subsystem.

## Best Possible Solution

Thread the existing origin channel/provider identity into the embedded subscription layer, derive a normalized `gateway/channels/<channel>` logger when the origin is a valid chat channel, keep `agent/embedded` as the fallback, and add regression coverage for non-Feishu attribution plus existing redaction behavior.
