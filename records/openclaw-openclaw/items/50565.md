## Summary

Keep open. Current `main` no longer contains the old Feishu-specific `tool call` text, but the remaining bug is still present in the embedded tool lifecycle path: tool start/end debug logs inherit a static `agent/embedded` logger because the subscription layer is not given the originating channel identity.

## What This Changes

The issue asks for tool call, completion, and failure logs to be tagged with the originating channel subsystem instead of a Feishu-specific or static subsystem.

## Best Possible Solution

Add a typed channel-origin or logger seam from `runEmbeddedAttempt` into `subscribeEmbeddedPiSession`, derive a safe `gateway/channels/<channel>` logger for valid message-channel runs, keep `agent/embedded` as fallback, and cover start/end tool lifecycle attribution in focused regression tests.
