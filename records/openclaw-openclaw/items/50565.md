## Summary

Keep open. Current main removed the old Feishu-specific `tool call` wording, but the underlying attribution problem remains in the embedded tool lifecycle path: tool start/end logs still inherit a static `agent/embedded` logger because the subscription layer does not receive the originating `messageChannel` or `messageProvider`.

## What This Changes

The issue requests that tool call lifecycle logs be tagged with the subsystem for the channel that originated the run instead of a static or wrong channel logger.

## Best Possible Solution

Add a typed channel-origin or logger seam from `runEmbeddedAttempt` into `subscribeEmbeddedPiSession`, derive a safe channel subsystem logger for message-channel runs, keep `agent/embedded` as the fallback, and add focused regression tests for tool start/end attribution.
