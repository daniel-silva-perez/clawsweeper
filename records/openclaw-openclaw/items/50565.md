## Summary

Keep open. The original Feishu-specific `tool call` wording is gone on current main, but the remaining diagnostics bug is still present: embedded tool lifecycle logs inherit a static `agent/embedded` logger because the subscription layer is not given the originating message channel.

## What This Changes

The issue asks for embedded tool lifecycle logs to be attributed to the originating message channel subsystem instead of a static Feishu or embedded subsystem.

## Best Possible Solution

Thread a typed channel-origin or logger seam from runEmbeddedAttempt into subscribeEmbeddedPiSession, derive a safe channel subsystem logger for valid message-channel runs, keep agent/embedded as fallback, and add focused regression tests for tool start/end attribution.
