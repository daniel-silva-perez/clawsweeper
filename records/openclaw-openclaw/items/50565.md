## Summary

Keep open. Current main no longer contains the old Feishu-specific `tool call` log text, but the attribution bug remains in the embedded tool lifecycle path: start/end logs inherit a static `agent/embedded` logger because the subscription layer is not passed the originating channel identity.

## What This Changes

The issue requests that tool lifecycle log entries be attributed to the originating channel subsystem, such as `gateway/channels/telegram`, instead of Feishu or another static subsystem.

## Best Possible Solution

Thread a typed channel-origin or logger seam from `runEmbeddedAttempt` into `subscribeEmbeddedPiSession`, derive a safe channel logger only for valid gateway message-channel runs, keep the embedded fallback, and add focused regression tests for tool start/end attribution.
