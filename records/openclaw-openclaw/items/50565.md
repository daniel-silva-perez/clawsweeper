## Summary

Keep open. Current main no longer contains the original `feishu: tool call` string, but embedded tool lifecycle logs still use a static `agent/embedded` logger; the originating channel context is available upstream and is not passed into the subscription/logger path.

## What This Changes

Tool call lifecycle logs should be attributed to the originating gateway channel subsystem instead of a Feishu or static embedded subsystem.

## Best Possible Solution

Land a narrow generic core fix: thread `messageChannel`/`messageProvider` from the embedded run attempt into subscription params, derive a channel-aware logger for `gateway/channels/<origin>` when a valid origin exists, keep `agent/embedded` as fallback, and add regression tests covering non-Feishu attribution plus existing redaction behavior.
