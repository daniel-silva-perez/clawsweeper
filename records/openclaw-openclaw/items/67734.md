## Summary

Keep this PR open. Current main still routes Feishu setup/status through the probe path, the probe still creates the SDK HTTP client without a custom logger, and the client factory has no probe-only logger option or cache-bypass behavior. The paired same-author bug remains open and this PR is a focused implementation candidate rather than cleanup-close material.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or request revisions to the focused Feishu plugin fix: add a probe-only silent logger option to createFeishuClient, avoid mutating or reusing the shared runtime client cache when that custom logger is supplied, keep the regression tests, and let the linked Feishu bug close only after the PR merges.
