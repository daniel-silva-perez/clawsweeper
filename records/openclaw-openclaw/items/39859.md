## Summary

Keep open. Current main still returns positive contextTokensOverride values before resolving a known provider/model context window, so the central over-reporting bug from linked #39857 remains. The provider-alias lookup piece appears present on main, but status/session/runtime paths can still preserve an oversized override.

## Best Possible Solution

Keep this PR and linked #39857 open. Refresh or replace the branch with a narrow fix on current main: resolve the known model context window before honoring contextTokensOverride, return min(override, modelWindow) when both are known, preserve override behavior when the model window is unknown, align session-store/status/runtime persistence, keep normalized provider lookup, and add focused resolver/status/session regression tests while dropping unrelated CI or baseline churn.
