## Summary

Keep this PR open. Current main still reuses a single Telegram draft-stream reply params object across rotated preview sends, and `forceNewMessage()` does not consume or clear the reply target. The PR remains the focused canonical implementation for the Telegram streaming `Deleted message` artifact, with recent maintainer repair/merge-ready context and no evidence that main has absorbed the behavior.

## Best Possible Solution

Keep this PR open for maintainer review/landing. The best path is to land the repaired Telegram draft-stream reply gating if validation stays green, while deciding whether the existing thread-not-found fallback should be fixed in this PR or split into a focused security follow-up.
