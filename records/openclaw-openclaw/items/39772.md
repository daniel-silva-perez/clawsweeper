## Summary

Keep this PR open. Current main still does not implement the Telegram draft-stream reply gating this PR adds: `createTelegramDraftStream` accepts `replyToMessageId` but not `replyToMode`, builds one reusable `replyParams` object, and `forceNewMessage()` does not consume or clear reply context before the next preview send. The PR also has recent maintainer repair/merge-ready context, so the best path is maintainer review and landing rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review or landing. The best implementation path is to land the repaired branch after validating the Telegram draft-stream tests and changed gate, while explicitly deciding whether the thread-not-found fallback warning is in scope for this PR or should become a narrow follow-up.
