## Summary

Keep this PR open. Current main still does not implement the Telegram draft-stream reply gating this PR is trying to land: `reply_to_message_id` is still baked into one reusable preview `replyParams` object, `forceNewMessage()` does not consume or recompute that reply target, and dispatch still does not pass `replyToMode` into `createTelegramDraftStream`. The PR also has active maintainer/contributor context, including a narrow repair on the branch and an unresolved security finding around thread fallback behavior that should be resolved or split before landing.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or port the repaired Telegram draft-stream change: pass `replyToMode` into `createTelegramDraftStream`, apply `reply_to_message_id` only to the first successful rotated preview for single-use modes, preserve `message_thread_id` on later previews, keep `replyToMode: "all"` semantics, and retain the regression tests. Before landing, resolve the thread-not-found fallback security concern in this PR or split it into a clear follow-up that maintainers accept.
