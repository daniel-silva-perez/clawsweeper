## Summary

Keep this PR open. Current main still does not implement the Telegram draft-stream reply gating: rotated preview messages reuse the same `reply_to_message_id`, `forceNewMessage()` does not consume or recompute single-use reply state, and dispatch still does not pass `replyToMode` into `createTelegramDraftStream`. The PR branch remains a focused implementation candidate, with maintainer repair context and green live checks, but the thread fallback security concern should be resolved or explicitly split before merge.

## Best Possible Solution

Keep this PR open for maintainer review and either land or port the repaired branch: pass `replyToMode` into `createTelegramDraftStream`, apply `reply_to_message_id` only to the first successful rotated preview for single-use modes, preserve `message_thread_id` on later previews, keep `replyToMode: "all"` semantics, and retain the regression tests. Resolve the thread fallback security concern in this PR or split it into a maintainer-accepted follow-up before merge.
