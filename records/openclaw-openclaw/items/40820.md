## Summary

Keep open: current main still lacks a Telegram `/settings` panel, so the feature is not obsolete, but this branch is not merge-ready because it targets the removed `src/telegram` runtime instead of the current Telegram plugin and writes a legacy streaming scalar path.

## What This Changes

The PR adds a Telegram-only `/settings` inline keyboard panel for access-control and streaming settings, plus callback parsing/auth/config-write helpers, native command registration changes, shared command registry provider filtering, and tests.

## Best Possible Solution

Port the panel into `extensions/telegram/src` using the current Telegram plugin command, menu, auth, callback dispatch, and config-write seams; keep the UI DM-only and gated by native command/config-write policy, write canonical `streaming.mode` paths for default and named accounts, and add Telegram plugin tests plus docs/changelog coverage before merge review.

## Reproduction Assessment

Yes for the merge-blocking defects: current-main static checks show no active `src/telegram` runtime or settings symbols, while the PR head adds and edits `src/telegram/*` files. The streaming-path issue is also reproducible by inspecting the PR resolver against current-main `streaming.mode` canonicalization.

## Solution Assessment

No, this branch is not the best way to solve the issue as-is. The maintainable solution is a current-main port inside the Telegram plugin with canonical config paths and renewed auth review for the config-mutating callbacks.
