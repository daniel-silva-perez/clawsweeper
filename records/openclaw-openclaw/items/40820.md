## Summary

Keep this PR open. Current main still does not implement a Telegram `/settings` inline keyboard panel, so the feature is not obsolete, but the proposed branch is not merge-ready against current main because it targets the old `src/telegram` layout and writes the legacy scalar streaming setting instead of the current nested config shape. The prior callback-auth review rounds are useful context and the latest provided diff appears to address those specific bypasses, but the port needs maintainer/author revision before merge.

## What This Changes

The PR adds a Telegram-only `/settings` inline keyboard panel for DM policy, group policy, streaming, and allowlist display, plus callback auth/config-write helpers, native command registration, provider filtering in the command registry, and focused tests.

## Best Possible Solution

Keep the feature work open, port it into `extensions/telegram/src` using the existing Telegram plugin boundaries and callback patterns, write `channels.telegram.streaming.mode` and account-scoped `streaming.mode`, then rerun focused Telegram command/callback/config-write tests plus targeted formatting before maintainer merge review.
