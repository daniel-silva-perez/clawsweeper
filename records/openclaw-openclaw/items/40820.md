## Summary

Keep open. Current main at 17811480dae20f65b526454d75e971ae8424ec10 still does not implement the Telegram `/settings` inline-keyboard control panel: there is no settings native command, no `cfg_*` callback parser/handler, and no settings panel/buttons/auth module in the current `extensions/telegram` plugin layout. The PR remains a plausible implementation candidate, especially because the supplied review discussion shows the author iterated on callback authorization hardening, but it needs porting and another security-focused review against current main.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best maintainer path is to port the feature into `extensions/telegram`, implement settings buttons/panel/auth against the current native command and callback structure, add any provider-specific native-command seam deliberately rather than as a broad incidental registry change, preserve the hardened authorization and config-write guards from the review discussion, and add focused tests for rendering, `cfg_*` callbacks, auth parity, config writes, and current nested streaming config.
