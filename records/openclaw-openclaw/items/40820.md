## Summary

Keep open: current main still lacks a Telegram `/settings` inline keyboard panel, so the feature is not implemented, but this branch is not merge-ready because it targets the removed `src/telegram` tree and still writes the legacy scalar streaming key. The idea remains useful, but it needs a current-main port into the Telegram plugin plus maintainer review of the privileged config-write callback path.

## What This Changes

The PR adds a Telegram-only `/settings` inline keyboard panel under `src/telegram`, with callback parsing, config mutation/auth helpers, native command registration changes, provider filtering in the shared command registry, and button/panel/auth tests.

## Best Possible Solution

Port the panel into `extensions/telegram/src` using current Telegram plugin command/menu and callback patterns, preserve `nativeEnabled`, `configWrites`, DM-only, and command-auth guards, write canonical `streaming.mode` paths for default and named accounts, and add matching Telegram tests plus user-facing docs/changelog coverage before merge review.
