## Summary

Keep open. Current main still does not implement the Telegram `/settings` inline-keyboard control panel, and this PR remains a useful implementation candidate, but the branch is stale against the current Telegram plugin layout and should not be merged as-is.

## What This Changes

This PR adds a Telegram-only `/settings` inline keyboard panel with settings callback parsing, panel rendering, authorization helpers, config writes for DM policy, group policy, and streaming, and focused tests.

## Best Possible Solution

Keep the PR open as source work, but port the implementation into the current Telegram plugin tree, keep Telegram-owned behavior out of core unless a generic command-provider seam is intentionally accepted, write canonical current config paths, and add focused tests for rendering, callback authorization, config writes, native command registration, and `configWrites` denial.
