## Summary

Keep open. The feature is still useful and current main does not implement a Telegram `/settings` panel, but this PR is not merge-ready because it targets the old `src/telegram` layout while current Telegram runtime is plugin-owned under `extensions/telegram/src`, and one config write path still uses the legacy scalar streaming key.

## What This Changes

The PR adds a Telegram-only `/settings` inline keyboard panel for DM policy, group policy, streaming, and allowlist display, plus callback auth/config-write helpers, native command registry plumbing, and tests.

## Best Possible Solution

Port the panel into `extensions/telegram/src` using current Telegram plugin command/menu and callback patterns, preserve the command-level auth/configWrites/nativeEnabled guards, write canonical `streaming.mode` paths for default and named accounts, and add matching Telegram tests plus user-facing docs/changelog coverage before maintainer merge review.
