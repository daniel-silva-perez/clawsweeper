## Summary

Keep open. Current main at 68561a8c948b49e85ae012646de550a49d6c63a3 still does not implement the PR’s intended Telegram `/settings` inline-keyboard control panel. The existing Telegram inline callback path supports model selection and plugin/approval callbacks, but there is no `cfg_*` settings callback parser, no settings panel module, and no `/settings` native command registration. The PR remains a plausible implementation candidate, but it needs porting from the old `src/telegram` layout into the current `extensions/telegram` plugin layout and another security-focused review because it adds a privileged config-write callback surface.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best maintainer path is to port the feature into `extensions/telegram`, reuse the current native command and callback structure, preserve fresh-config loading plus command/owner/topic authorization parity, keep `configWrites` as a hard guard, and add focused tests for the settings panel before review.
