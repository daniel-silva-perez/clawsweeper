## Summary

Keep open. Current main still lacks the Telegram `/settings` panel, so this PR is not obsolete, but the branch is not merge-ready because it adds the feature under the removed `src/telegram` tree while current Telegram runtime lives under `extensions/telegram/src`; the streaming write path also targets the legacy scalar `streaming` key instead of the current nested contract.

## What This Changes

The PR adds a Telegram-only `/settings` inline keyboard panel for viewing and changing DM policy, group policy, streaming mode, and allowlist state, with new callback parsing, config-write logic, native command wiring, and tests.

## Best Possible Solution

Implement the panel inside the current Telegram plugin using the existing plugin command, callback, auth, and config-write seams; keep it DM-only and gated by native-command/config-write settings, write canonical nested streaming paths, and add plugin-local tests plus docs/changelog coverage before merge review.

## Reproduction Assessment

Yes. Static current-main checks are enough to reproduce the blocking integration problem: `src/telegram` is absent and the exact settings symbols are absent, while the PR file list adds the feature under `src/telegram/*`.

## Solution Assessment

No. The feature direction is reasonable, but this branch is not the best implementation as-is; the maintainable path is a current-main port inside `extensions/telegram/src` with canonical `streaming.mode` writes and renewed privileged-surface review.
