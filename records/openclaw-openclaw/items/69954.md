## Summary

Keep this PR open. Current main still lacks the canonical `<sessionId>.jsonl` fallback for `sessions cleanup --fix-missing`, and the PR addresses a real unsafe cleanup path linked to #50248. The remaining blocker is the review finding that repair-only runs can mutate `sessionFile` metadata while reporting `wouldMutate: false`.

## Best Possible Solution

Keep this PR open, fix the repair-only mutation accounting, then review and land the canonical transcript fallback with focused regression coverage. The final behavior should preserve safe pruning for truly missing transcripts while preventing `sessions cleanup --fix-missing` from deleting entries whose canonical `<sessionId>.jsonl` transcript still exists.
