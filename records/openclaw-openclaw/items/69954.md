## Summary

Keep this PR open. Current main still lacks the stale `sessionFile` fallback that the PR adds: cleanup resolves one transcript path from the stored entry and prunes when that path is missing, while existing read-path helpers already consider canonical sessionId transcript candidates. The PR is still useful, but it should address the repair-only JSON accounting issue before merge.

## Best Possible Solution

Keep this PR open, ask for a small fixup that reports repair-only cleanup as a mutation, and then review/land the fallback with focused coverage. The implementation should either reuse the shared transcript candidate resolver or clearly preserve the narrower canonical fallback boundary, and it should close the linked #50248 class of unsafe `--fix-missing` pruning once merged.
