## Summary

Keep open. Current main still has the reported CLI-backend prompt bug, and the PR branch is stale against the current prepare-based runner path, so it does not actually suppress heartbeat instructions for cron or subagent CLI runs.

## What This Changes

The PR adds session-key-aware prompt-mode selection to CLI system-prompt construction, shares the prompt-mode resolver with the embedded runner, forwards the session key from the CLI runner, and adds helper-level prompt tests.

## Best Possible Solution

Refresh or replace the PR so the current prepareCliRunContext path passes a shared cron/subagent prompt-mode decision into CLI system-prompt construction, while preserving full mode for normal sessions and adding regression coverage around prepare-based CLI runs.

## Reproduction Assessment

Yes. Static current-main reproduction is high-confidence: an isolated cron run passes a cron sessionKey into runCliAgent, prepareCliRunContext receives it, but buildSystemPrompt cannot consume it and buildAgentSystemPrompt defaults to full mode with Heartbeats when heartbeatPrompt is present.

## Solution Assessment

No. The intended direction is correct, but this branch updates the old prompt construction site and helper-level tests only; the maintainable fix needs to flow through prepareCliRunContext and share the resolver used by embedded runs.
