## Summary

Keep this PR open, but require changes before merge. The underlying CLI-backend heartbeat bug is still valid on current main, while the PR branch is stale: it forwards the session key at the old direct prompt-building site, but current main builds CLI system prompts inside `prepareCliRunContext`, where the available `params.sessionKey` still does not reach prompt-mode selection.

## What This Changes

The PR adds session-key-aware prompt-mode resolution for CLI system prompts, forwards `sessionKey` from the CLI runner path, moves/re-exports the resolver, and adds helper tests for full versus minimal prompt sections.

## Best Possible Solution

Adapt the fix to the current prepare-based CLI runner: derive `minimal` prompt mode from cron and subagent session keys before CLI prompt construction, pass that mode into the shared prompt builder while preserving full mode for normal sessions, and cover the prepare path with regression tests.
