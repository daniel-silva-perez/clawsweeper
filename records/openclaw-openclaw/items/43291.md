## Summary

Keep this PR open, but it needs changes before merge. The reported cron/subagent CLI-backend prompt bug is still real on current main, and the PR's intended direction is right, but the branch targets an old prompt construction site while current main builds CLI prompts inside prepareCliRunContext.

## What This Changes

The PR adds session-key-aware prompt-mode selection to CLI system prompt building, shares the prompt-mode resolver, forwards the session key from the CLI runner, and adds helper-level prompt tests.

## Best Possible Solution

Update the current prepareCliRunContext flow to derive a shared cron/subagent prompt-mode decision from params.sessionKey and pass it into CLI system-prompt construction, preserving full mode for normal sessions and systemPromptOverride behavior.

## Reproduction Assessment

Yes. Static current-main reproduction is high-confidence: isolated cron passes a cron sessionKey into runCliAgent, prepareCliRunContext receives it, but buildSystemPrompt cannot consume it and buildAgentSystemPrompt defaults to full mode with Heartbeats when heartbeatPrompt is present.

## Solution Assessment

No. The intended direction is correct, but this patch forwards sessionKey at an obsolete prompt-construction site and only adds helper-level tests; the safer fix is to wire promptMode through prepareCliRunContext and cover that path.
