## Summary

Keep this PR open. Current main still has the CLI-backend prompt-mode gap described by the PR and the prior ClawSweeper comment: cron/subagent session keys reach the CLI runner, but CLI system-prompt construction does not pass a session key or derived prompt mode into buildAgentSystemPrompt, so the prompt defaults to full mode and can include the Heartbeats section.

## Best Possible Solution

Keep this PR open and adapt it to current main. The best landing shape is to centralize resolvePromptModeForSession in a shared routing or agent prompt helper, pass params.sessionKey or the derived promptMode through prepareCliRunContext/buildSystemPrompt into buildAgentSystemPrompt, and add CLI regression tests for no key, regular agent key, cron run key, and subagent key.
