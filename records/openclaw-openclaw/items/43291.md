## Summary

Keep this PR open. The underlying CLI-backend prompt bug still exists on current main, but this branch is stale against the current prepare-based CLI runner path and therefore does not actually suppress heartbeat instructions for cron or subagent CLI sessions.

## What This Changes

The PR adds session-key-aware prompt-mode selection to CLI system-prompt construction, shares the prompt-mode resolver with the embedded runner, forwards the session key from the CLI runner, and adds helper-level prompt tests.

## Best Possible Solution

Refresh or replace the PR so the current prepareCliRunContext path derives a shared cron/subagent prompt-mode decision from params.sessionKey and passes it into CLI system-prompt construction, while preserving full mode for normal sessions and systemPromptOverride behavior.

## Reproduction Assessment

Yes. Static current-main reproduction is high-confidence: isolated cron passes a cron sessionKey into runCliAgent, prepareCliRunContext receives it, but buildSystemPrompt cannot consume it and buildAgentSystemPrompt defaults to full mode with Heartbeats when heartbeatPrompt is present.

## Solution Assessment

No. The intended direction is correct, but the patch updates an old prompt construction site and helper-level tests only; the maintainable fix needs to flow through prepareCliRunContext and share the resolver used by embedded runs.
