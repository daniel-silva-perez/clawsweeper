## Summary

Keep this PR open. The targeted bug still exists on current main: CLI backend cron/subagent runs carry a session key into the prepare path, but the CLI system-prompt builder still receives no session key or prompt mode, so `buildAgentSystemPrompt` defaults to full mode and can inject `## Heartbeats` / `HEARTBEAT_OK`. The PR is useful source work, but it is now stale against the moved current-main prompt construction path and needs adaptation before merge.

## What This Changes

The branch adds a shared cron/subagent prompt-mode resolver, forwards CLI session keys into system-prompt construction, and adds helper tests asserting heartbeat sections are kept for normal sessions and suppressed for cron/subagent sessions.

## Best Possible Solution

Adapt the change to the current prepare-based CLI runner: derive cron/subagent minimal prompt mode from `params.sessionKey`, forward that mode into the shared system-prompt builder, keep normal CLI sessions in full mode, and add current-path regression coverage before landing.
