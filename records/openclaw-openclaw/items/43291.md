## Summary

Keep this PR open, but it still needs changes before merge. The reported CLI-backend heartbeat problem remains on current main, while this branch forwards the session key at an older direct prompt-building site; current main now builds CLI system prompts through prepareCliRunContext, where the available sessionKey still does not affect promptMode.

## What This Changes

The PR adds session-key-aware prompt-mode selection to CLI system-prompt construction, forwards sessionKey from the CLI runner, moves/re-exports the prompt-mode resolver, and adds helper tests for full versus minimal prompt sections.

## Best Possible Solution

Move prompt-mode resolution into the current prepare-based CLI runner path, reuse one shared resolver for cron and subagent session keys, preserve full mode for normal sessions, and add regression tests that exercise prepareCliRunContext rather than only the helper.
