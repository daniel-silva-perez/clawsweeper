## Summary

Keep this PR open. Current main still has the CLI-backend prompt-mode gap, but the branch targets an older prompt-construction site; after the prepare-based CLI runner refactor, `prepareCliRunContext` still calls `buildSystemPrompt` without `sessionKey` or `promptMode`, so the patch would not fix cron/subagent CLI sessions on main.

## What This Changes

The branch moves cron/subagent prompt-mode resolution toward a shared helper, passes a session key into CLI system-prompt construction, and adds helper tests for heartbeat section suppression.

## Best Possible Solution

Adapt the change to the current prepare-based CLI runner: derive cron/subagent minimal prompt mode from `params.sessionKey`, forward that mode into the shared system-prompt builder, keep normal CLI sessions in full mode, and add current-path regression coverage before landing.
