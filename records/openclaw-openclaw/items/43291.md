## Summary

Keep this PR open. Current main still has the CLI-backend prompt-mode gap: cron run session keys are passed into `runCliAgent`, and the embedded runner already maps cron/subagent keys to minimal prompt mode, but CLI prompt construction drops the session key and calls `buildAgentSystemPrompt` without `promptMode`, so it can still inject `## Heartbeats` / `HEARTBEAT_OK`.

## Best Possible Solution

Keep this PR open and adapt it to current main. The best landing shape is to centralize the cron/subagent prompt-mode resolver in a shared owner-neutral helper, pass `params.sessionKey` or the derived prompt mode through `prepareCliRunContext` into `buildSystemPrompt`, forward that mode to `buildAgentSystemPrompt`, and add CLI regression tests covering no key, regular agent key, cron run key, and subagent key before running the changed checks.
