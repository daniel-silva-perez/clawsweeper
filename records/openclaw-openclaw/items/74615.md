## Summary

Keep open. Current main still has static group mention gating plus implicit mention/reply handling, but no configurable timed attention window that temporarily processes all follow-up group messages after a mention. The request is valid feature work, but the cross-channel trigger-policy and safety semantics need maintainer design before implementation.

## What This Changes

Request a configurable group-chat attention window that starts after a mention, temporarily treats follow-up messages as mention-authorized, resets on new mentions, and expires back to mention-only mode across supported channels.

## Best Possible Solution

Design a shared group-attention policy that runs after group allowlists and before mention skipping, stores per-agent/account/conversation expiry state, resets only on accepted mentions or approved implicit mentions, and exposes a documented config surface with tests for expiry, reset, disabled mode, and at least Slack/Telegram plus the shared mention-gating helper.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The current gap is source-verifiable: with `requireMention: true`, an unmentioned follow-up that is not an implicit reply/quote is skipped or stored as pending history rather than dispatched.

## Solution Assessment

Unclear until maintainers choose the product and safety contract. The narrow maintainable direction would be a shared mention-gating/admission seam with per-channel adapters, not separate ad hoc timers in each channel.
