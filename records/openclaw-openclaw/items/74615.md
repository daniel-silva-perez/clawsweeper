## Summary

Keep open. Current main still uses static group mention gating plus existing implicit reply/thread mention exceptions; it does not expose or enforce a time-based attention window after a mention. The request is valid but broad enough to need maintainer design review because it would change group-listening and token-spend behavior across multiple channel plugins.

## What This Changes

Add a configurable group-chat attention window that temporarily treats follow-up messages after a mention as eligible even when mention gating is enabled.

## Best Possible Solution

Define an opt-in shared attention-window policy for group/channel turns, add typed config/schema/docs, and have channel handlers consult that shared state before treating an unmentioned message as mention-gated, while preserving existing groupPolicy and sender authorization checks.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a feature request, but there is a high-confidence source-level verification path: current main has no attention-window config and representative handlers still skip unmentioned group/channel messages when `requireMention` is true.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose the product boundary; if accepted, the maintainable direction is a shared opt-in policy rather than separate ad hoc timers in each channel plugin.
