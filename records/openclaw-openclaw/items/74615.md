## Summary

Keep open. Current main still has static per-message mention gating and explicit/implicit mention bypasses, but no configurable time-limited attention window after a group mention; the related reports cover mention detection, slash-command bypasses, or model routing rather than this cross-channel conversation-window behavior.

## What This Changes

The issue requests a configurable attention window that temporarily processes unmentioned group-chat follow-ups after one mention, then reverts to mention-only mode.

## Best Possible Solution

Define an opt-in shared attention-window model with per-conversation expiry semantics, documented defaults, tests for mentioned and unmentioned follow-ups, and channel integrations that preserve existing authorization and mention-gating safety boundaries.

## Reproduction Assessment

Yes. Configure a group/channel with requireMention enabled, send one mentioned message, then send an unmentioned follow-up; current main evaluates only the follow-up's own mention facts and takes the no-mention skip/store path.

## Solution Assessment

Unclear. The requested direction solves a real workflow gap, but the exact config shape and shared state model should be designed before coding because it spans channel plugins, shared mention policy, privacy behavior, and documentation.
