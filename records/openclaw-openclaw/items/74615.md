## Summary

Keep open. Current main still implements static group mention gating plus narrow explicit/implicit mention and command bypasses, but it does not implement a configurable, time-limited attention window after a mention. The request remains valid and is not covered by the related mention-detection or model-routing reports.

## What This Changes

The issue requests a configurable group-chat attention window that temporarily processes follow-up messages after an initial mention before returning to mention-only mode.

## Best Possible Solution

Define an opt-in shared attention-window model with per-conversation expiry semantics, explicit config/defaults, and platform-specific limitations, then wire it through channel plugins and docs while preserving existing authorization and mention-gating safety boundaries.

## Reproduction Assessment

Yes. A source-level reproduction is clear: configure a group/channel with `requireMention: true`, send one mentioned message, then send an unmentioned follow-up; current main evaluates only the follow-up's own mention facts and takes the no-mention skip/store path.

## Solution Assessment

Unclear. The user problem is well-defined, but the proposed per-channel `attentionWindow` shape needs maintainer design before implementation because the best solution likely spans shared mention policy, plugin contracts, timer state, privacy semantics, and platform-specific delivery constraints.
