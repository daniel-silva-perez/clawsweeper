## Summary

Keep this issue open. Current main has useful adjacent discoverability through slash-command autocomplete and the command palette, but it does not implement user-defined shortcut keywords/prompts, `gateway.webchat.shortcuts`/`webchat.shortcuts` config, or a persistent shortcut sidebar/drawer. The plugin Control UI descriptor seam is descriptor-only and is not consumed by the current Control UI as an interactive click-to-insert panel, so this is not safely closable as ClawHub-only yet.

## Best Possible Solution

Keep the issue open and route it as a Control UI/product design item. The likely best implementation is either a core WebChat shortcut/snippet panel backed by a typed config/schema and docs, or a small additive plugin UI contribution API that lets external plugins provide clickable draft-insertion actions with labels, descriptions, icons, categories, and mobile placement.
