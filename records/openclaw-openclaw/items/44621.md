## Summary

Keep this PR open. Current main still parses and strips mixed-message `/model <provider/model> <message>`, then clears the model directive before model selection can apply it as a one-shot override. The PR is a relevant implementation candidate, but its current diff has unresolved review findings around auth-profile validation/mutation and stored session model overrides.

## Best Possible Solution

Keep this PR open as the implementation candidate. The best fix is to preserve mixed-message `/model` as explicit one-shot state, apply provider/model/auth profile only to the current run, validate one-shot auth profiles through the existing provider/cooldown rules, avoid persistent session model/auth-profile mutation, recompute thinking/reasoning defaults without reapplying stored session overrides, and add regressions for prior persisted model/profile overrides plus follow-up turn reversion.
