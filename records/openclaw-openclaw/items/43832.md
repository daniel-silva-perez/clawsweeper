## Summary

Keep open. Current main still does not implement this PR's intended graceful cron drain: cron hot reload stops the old service synchronously, the cron contract exposes no stopGraceful API, and timer/manual runs still have out-of-lock work that can later persist. The PR is not obsolete; it needs maintainer review and a bounded-drain decision before merge.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to refresh it against current main, preserve coverage for timer ticks and CommandLane.Cron manual runs, add a bounded drain or explicit fallback for stuck/no-timeout runs, update the CronService contract and gateway reload tests, and then use the graceful stop path in src/gateway/server-reload-handlers.ts if maintainers want that stronger guarantee. If maintainers prefer the current force-reload merge behavior as the shipped fix for linked #30098, document that decision before closing this PR.
