## Summary

Keep open. Current main still does not implement this PR's omitted-coreGatewayHandlers reuse path: the loader cache key includes gateway method names, but active runtime state only stores the cache key/workspace/mode and the compatibility fallback cannot reconstruct the gateway method set when a helper/runtime caller opts into gateway binding while omitting the handler map. The PR remains a focused implementation candidate for maintainer review, with the main review risk being preservation of the default-mode capability boundary.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase onto current main, persist active core gateway method names with active registry state, reconstruct compatibility only when the caller explicitly opts into gateway binding and method-name equality proves compatibility, keep the default-mode non-inheritance regression coverage, and then run the targeted plugin loader/runtime tests plus the changed gate before landing.
