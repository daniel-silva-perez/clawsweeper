## Summary

Keep open. Current main still has the cache-key mismatch this PR targets: gateway startup records core gateway method names in the load key, but active runtime state does not persist those names and the compatibility check cannot reconstruct an omitted handler map. The PR branch is a plausible implementation candidate and its latest diff appears to address the earlier default-mode capability-boundary concern with explicit tests, so the remaining action is maintainer review and normal validation rather than cleanup closure.

## What This Changes

The PR records active core gateway method names in plugin runtime state and updates loader compatibility tests so opt-in gateway-bindable helper loads that omit handler maps can reuse a matching active registry while default-mode loads do not inherit gateway bindings.

## Best Possible Solution

Merge a reviewed version of this PR or equivalent code that stores the active registry's gateway method set and only reconstructs compatibility for callers that explicitly opted into gateway-bindable runtime loading. Keep the regression tests for omitted handlers, mismatched method sets, and default-mode non-inheritance.
