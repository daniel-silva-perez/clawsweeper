## Summary

Keep open. Current main has related gateway-bindable registry reuse logic, but it still does not persist the active registry's gateway method names, so a helper/runtime load with allowGatewaySubagentBinding and no coreGatewayHandlers cannot match a gateway-startup registry whose cache key included those handlers. The PR remains a focused implementation candidate, with tests and a review-thread fix for the default-mode capability boundary, but it should be reviewed/rebased against current main before landing.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase it onto current main, preserve the existing activatePluginRegistry hook-runner guard, keep method-name equality as the compatibility gate, retain the default-mode non-inheritance regression test, and land after the targeted plugin loader/runtime tests and changed gate pass.
