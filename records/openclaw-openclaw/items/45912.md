## Summary

Keep this PR open. Current main still rejects cross-configured-root skill symlinks, so the requested behavior is not obsolete, but the proposed patch would weaken bundled-skill allowlist enforcement by allowing bundled realpaths to load under non-bundled source labels.

## What This Changes

The PR adds a cross-configured-root realpath allowlist to skills containment checks and adds regression tests for workspace-to-project-agent and managed-to-extra-dir skill symlinks.

## Best Possible Solution

Implement one source-preserving symlink policy for configured skill roots: allow symlinks into explicitly trusted non-bundled roots, keep unknown external escapes rejected, make bundled-target behavior explicit, and cover cross-root success, external rejection, and `skills.allowBundled` enforcement in tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve bundled policy for symlink targets:** `src/agents/skills/workspace.ts:476`
  - body: Adding `bundledSkillsDir` to `allowedSkillRootRealPaths` lets a workspace or managed symlink load bundled skill files under the caller source. Since `skills.allowBundled` only filters entries whose source is `openclaw-bundled`, an excluded bundled skill can be reintroduced through a non-bundled symlink.
  - confidence: 0.9

## Security Review

Status: needs_attention

Concerns:

  - body: Including `bundledSkillsDir` in the trusted symlink target roots accepts bundled files from non-bundled roots, while later filtering treats them as workspace/managed skills and does not apply `skills.allowBundled`.
