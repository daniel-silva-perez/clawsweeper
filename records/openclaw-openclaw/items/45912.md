## Summary

Keep open. Current main still enforces same-root realpath containment, so the PR is not obsolete, but the proposed patch should not merge as written because the configured-root allowlist appears to include the bundled skills root while source classification remains tied to the scanning root. That can bypass `skills.allowBundled`, and the overlapping open symlink reports need one source-preserving policy.

## What This Changes

The PR changes the skills workspace loader to allow symlinked skill realpaths under configured skill roots and adds regression tests for workspace and managed symlink cases.

## Best Possible Solution

Implement one source-preserving symlink policy for configured skill roots: allow cross-root links among explicitly trusted non-bundled roots, keep external escapes rejected, and either exclude bundled targets from non-bundled aliases or relabel bundled targets so bundled allowlist checks still apply. Cover cross-root success, external rejection, bundled allowlist enforcement, grouped skills, and the final docs contract.

## Reproduction Assessment

Yes. Configure `skills.allowBundled` to exclude a bundled skill, then create a workspace or managed symlink to that bundled skill; under this PR’s trusted-root logic the realpath is accepted while the loaded source remains non-bundled. I did not run tests because this review was required to stay read-only.

## Solution Assessment

No. Accepting cross-root symlinks may be the right direction for non-bundled configured roots, but including bundled roots without preserving bundled source semantics is not the narrowest maintainable fix.
