## Summary

Keep this PR open. Current main still lacks the requested configured-root symlink allowance, so the branch is not obsolete, but the proposed implementation is not merge-ready because it includes bundled skills in the cross-root allowlist without preserving bundled source policy, which can bypass `skills.allowBundled`.

## What This Changes

The branch adds a configured-root realpath allowlist to skills containment checks and regression tests for workspace and managed skill symlinks that point into project-agent or extra-dir skill roots.

## Best Possible Solution

Implement one source-preserving symlink policy for configured skill roots: allow symlinks into explicitly trusted non-bundled roots when the target root's policy is preserved, keep unknown external escapes rejected, make the bundled-target behavior an explicit maintainer decision, and cover cross-root success, external rejection, and `skills.allowBundled` enforcement in tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve bundled policy for symlink targets:** `src/agents/skills/workspace.ts:476`
  - body: Adding `bundledSkillsDir` to the cross-root allowlist lets a workspace or managed symlink load a bundled skill under the caller source. Because `skills.allowBundled` only filters entries whose source is `openclaw-bundled`, a bundled skill excluded by the allowlist can be reintroduced through the symlink.
  - confidence: 0.9

## Security Review
