## Summary

Keep this PR open, but do not merge it as-is. Current main still enforces same-root realpath containment, while the PR’s cross-root allowlist changes a security-sensitive skills boundary and can bypass the bundled-skill allowlist when a non-bundled symlink points at a bundled skill.

## What This Changes

The PR threads an allowed configured-root realpath list through skills containment checks and adds regression tests for workspace and managed symlinks that resolve into other configured skill roots.

## Best Possible Solution

Implement one source-preserving symlink policy for configured skill roots: allow safe cross-root links among explicitly trusted non-bundled roots, keep external escapes rejected, and either exclude bundled targets from non-bundled aliases or relabel bundled targets so bundled allowlist checks still apply. Cover cross-root success, external rejection, bundled allowlist enforcement, grouped skills, and the docs contract.

## Reproduction Assessment

Yes. Configure `skills.allowBundled` to exclude a bundled skill, then create a workspace or managed symlink to that bundled skill; under this PR’s trusted-root logic the realpath is accepted while the entry source remains non-bundled. I did not run tests because this review was required to stay read-only.

## Solution Assessment

No. Accepting cross-root symlinks may be the right direction for non-bundled configured roots, but including bundled roots without preserving bundled source semantics is not the narrowest maintainable fix.
