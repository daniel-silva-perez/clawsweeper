## Summary

Keep this PR open, but it should not merge as-is. Current main still lacks the requested cross-configured-root symlink behavior, while the proposed patch broadens the skills containment boundary in a way that can bypass bundled-skill allowlist enforcement.

## What This Changes

The PR adds an allowed-realpath-root list to skills containment checks so symlinked skill paths can resolve inside other configured skill roots, plus regression tests for workspace-to-project-agent and managed-to-extra-dir symlinks.

## Best Possible Solution

Implement one source-preserving symlink policy for configured skill roots: allow cross-root symlinks among explicitly trusted non-bundled roots, keep external escapes rejected, and either exclude bundled targets from non-bundled aliases or relabel them so `skills.allowBundled` still applies. Cover cross-root success, external rejection, and bundled allowlist enforcement in tests.

## Reproduction Assessment

Yes. A high-confidence reproduction path is a bundled skill excluded by `skills.allowBundled`, plus a workspace or managed symlink pointing at that bundled skill; this PR accepts the realpath while the local loader keeps the source as non-bundled.

## Solution Assessment

No. Allowing realpaths into other configured roots is plausible for non-bundled roots, but including `bundledSkillsDir` without preserving bundled source semantics is not the narrowest maintainable fix.
