## Summary

Keep open. Current main still has the older prompt-submission check that treats replay history as sufficient content for a whitespace-only visible prompt, while this PR adds the missing embedded-runner guard with focused helper and integration coverage. I found no blocking review findings; the remaining action is maintainer review and CI completion for the linked blank-prompt bug.

## What This Changes

The PR replaces the embedded runner prompt-content boolean with a skip-reason helper, skips blank visible non-runtime prompt submissions before provider calls, adds focused tests, and adds a changelog entry for #74137.

## Best Possible Solution

Review and land this PR, or an equivalent embedded-runner pre-provider guard, once CI is green. The end state should keep runtime-only and media-only turns working, skip blank visible user prompts before provider submission, and leave session-file repair as a safety net rather than the first line of defense.
