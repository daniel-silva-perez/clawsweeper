## Summary

Keep this PR open. Current main already covers the `sessions_spawn` ACP-only-field behavior, but it still rejects the central `sessions_send` case where a model sends both `sessionKey` and `label`, so the PR is not obsolete or implemented on main.

## Best Possible Solution

Keep the PR open and rebase/narrow it onto current main. The best path is to carry forward the `sessions_send` normalization so `sessionKey` wins over redundant `label`/`agentId`, keep the regression test that skips `sessions.resolve`, and avoid reintroducing obsolete `sessions_spawn` churn that main already covers.
