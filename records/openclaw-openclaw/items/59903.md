## Summary

Keep this PR open. Current main does not fully implement the PR’s intended behavior: `/status` can still render a recent successful terminal task as “recently finished” and include its `terminalSummary`. The PR’s original target moved during refactors, so the useful fix needs rebasing/adapting to the current `src/status/status-text.ts` path.

## Best Possible Solution

Keep the PR open and adapt its behavior to the current status-text implementation: suppress completed-success-only task lines, keep active and recent failure summaries, update the agent-local fallback consistently, and add the PR’s missing regression test for a recent successful cron task with `terminalSummary`.
