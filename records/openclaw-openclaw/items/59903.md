## Summary

Keep this PR open. The requested behavior is still not implemented on current main: chat `/status` task-line formatting now lives in `src/status/status-text.ts`, and that code still renders a recent completed-success task as `recently finished` plus the task `terminalSummary`. ## What This Changes

The PR suppresses completed-success background task output from chat `/status` task lines, preserves active and recent-failure summaries, and adds regression coverage for completed cron output and agent-local recent failures.

## Best Possible Solution
