## Summary

Keep this PR open. Current main at e476523082a6fff3a0c3048077d2f0e52413fbf2 still implements the runtime logger adapter with message-only wrappers, while the exported RuntimeLogger contract accepts optional structured metadata and repo logging conventions expect structured fields as the first logger argument.

## Best Possible Solution

Keep this PR open, review exact-head CI, and land or adapt the focused fix: forward non-empty RuntimeLogger metadata to the underlying logger as method(meta, message), omit undefined or empty metadata, and keep the colocated regression coverage in src/plugins/runtime/runtime-logging.test.ts.
