## Summary

Keep open, but this PR is not merge-ready. Current main still does not add Defuddle, and the branch now targets the old core extraction path instead of the plugin-based web-content-extractor path that main uses. The review discussion did improve the branch by adding extractor assertions, switching the fallback tag to unknown, and disabling Defuddle async fetches, but the remaining implementation boundary and dependency-scope questions need maintainer review.

## What This Changes

This PR adds Defuddle as the primary `web_fetch` HTML extractor with Readability fallback, updates extractor tagging/config/docs/tests/scripts, and adds the `defuddle` dependency.

## Best Possible Solution

Rework the idea around the current web-content-extractor contract: either add a plugin-owned Defuddle extractor with dependency ownership, ordering, docs, config/help text, and tests, or first extend the plugin API/runtime so an external ClawHub plugin can provide the extractor without a bundled-only path. Preserve local-only guarded extraction semantics and keep Readability as a fallback if maintainers choose Defuddle as the preferred default.
