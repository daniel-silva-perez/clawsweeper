## Summary

Keep open. Defuddle is a useful direction and the earlier async-network concern was addressed with `useAsync: false`, but this PR is not merge-ready against current `main`: HTML extraction now runs through plugin-owned `webContentExtractors`, while the PR adds Defuddle to the old utility-local path and adds the parser dependency to the root runtime surface.

## What This Changes

This PR adds Defuddle as the primary `web_fetch` HTML extractor with Readability fallback, updates extractor tagging/config/docs/tests/scripts, and adds the `defuddle` dependency.

## Best Possible Solution

Implement Defuddle through the current `webContentExtractors` contract as a maintainer-approved extractor plugin, with its dependency scoped to that plugin, `autoDetectOrder` ahead of Readability if chosen as default, `useAsync: false`, docs/config/help text aligned to the plugin model, and focused tests for Defuddle success plus Readability fallback.
