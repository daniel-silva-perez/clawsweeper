## Summary

Keep this PR open for normal maintainer review. Current main still lacks the Debug Event Log sticky-at-bottom state, scroll handler, lifecycle scheduling, and focused tests, and the related chat-scroll work does not cover this debug-log surface.

The PR adds sticky-at-bottom scroll state and handlers for the Control UI Debug Event Log, changes the shared event buffer to append order with Overview recency preserved, constrains the debug log to in-panel scrolling, and adds focused scroll tests.

## Best Possible Solution

Land this PR, or adapt the same narrow changes onto main, after normal UI validation confirms Debug stays stable while scrolled up and Overview still shows the newest events first. Keep the related chat-scroll work separate because it addresses a different surface.
