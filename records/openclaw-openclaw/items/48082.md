Keep this issue open. The report is recent, concrete, and describes a native SIGSEGV on a specific Feiniu NAS Linux kernel where both PTY and non-PTY exec paths crash while native spawn and a manual node-pty probe work. Current main has relevant hardening, but I found no source, docs, tests, changelog, or history evidence that this platform-specific crash is fixed or no longer reproducible.

Keep the issue open as a platform-specific exec crash.
