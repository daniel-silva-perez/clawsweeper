## Summary

Keep this PR open. Current main still does not provide an equivalent deep `doctor --check-config` path, and the PR remains a relevant implementation candidate, but the submitted patch is not merge-ready because it imports a non-existent TTS export and still lets some unresolved env-backed active config failures exit successfully.

## What This Changes

The PR adds a `doctor --check-config` CLI option, a new deep config validation module for schema/plugin/model/TTS/channel checks, and formatter-focused tests for the new result output.

## Best Possible Solution

Land or replace this with a core doctor deep config validation mode that uses the current plugin/provider registry APIs, documents the new CLI surface, and has behavior tests for schema/plugin warnings, model resolution/catalog status, fallback parsing, TTS availability, channel summaries, and non-zero exits for active config failures.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Use an exported TTS provider source:** `src/commands/doctor-check-config.ts:18`
  - body: `../tts/tts.js` does not export `TTS_PROVIDERS`; the TTS facade and speech-core runtime expose functions like `resolveTtsConfig`/`resolveTtsProviderOrder`, not a static provider list. This new import will fail typecheck/build before `doctor --check-config` can run. Use the registered provider/runtime API instead of importing a non-existent constant.
  - confidence: 0.97
- **[P2] Fail active unresolved env placeholders:** `src/commands/doctor-check-config.ts:72`
  - body: Missing `${ENV}` substitutions are preserved in the snapshot and emitted here as `warn`, while `hasFailures` only considers `fail`. Because speech providers treat a non-empty `apiKey` string as configured, an active TTS key like `${OPENAI_API_KEY}` with the env unset can still exit 0, missing the config degradation this flag is meant to catch. Make unresolved env refs on checked active paths fail, or exclude unresolved placeholders from provider configured checks.
  - confidence: 0.86
