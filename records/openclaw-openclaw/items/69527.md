## What This Changes

The issue asks for `openclaw infer model run` to honor a configured OpenRouter primary such as `openrouter/auto` or `openrouter/@preset/...` and to show fallback attempts when execution falls through to another model.

## Best Possible Solution

Implement a focused model-run routing and diagnostics fix: `infer model run` should use the configured or explicitly requested model, preserve raw local/gateway probe behavior, and return meaningful fallback-attempt details when fallback occurs. OpenRouter `openrouter/auto` and `@preset/...` handling should be covered by regression tests and checked against current OpenRouter behavior before changing the provider's model-ID normalization.
