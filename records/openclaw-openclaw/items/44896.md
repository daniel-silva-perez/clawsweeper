## Summary

Keep open. Current main still has the Moonshot/Kimi payload inconsistency: the Moonshot thinking wrapper disables `thinking` for pinned tool-choice objects, while the OpenAI-compatible completions transport can independently emit `reasoning_effort` for reasoning-capable models. The current docs describe normalizing incompatible `tool_choice` values to `auto`, and the related fix PRs were closed unmerged.

## What This Changes

The issue asks OpenClaw to stop sending contradictory Moonshot/Kimi reasoning payloads where `reasoning_effort` remains set while `thinking.type` is forced to `disabled` for pinned tool choices.

## Best Possible Solution

Update the Moonshot thinking wrapper to produce one coherent contract for enabled thinking plus incompatible tool choices, preferably by normalizing pinned `tool_choice` objects to `auto` as the docs already state. Add regression coverage that prevents `reasoning_effort` or nested reasoning fields from coexisting with `thinking.type: disabled` in this path, and keep the Moonshot docs and changelog aligned.
