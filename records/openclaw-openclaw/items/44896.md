## Summary

Keep open. Current `main` still has the reported Moonshot/Kimi payload inconsistency: enabled thinking with a pinned tool-choice object is downgraded to `thinking.type: disabled`, while the OpenAI-compatible transport can independently emit `reasoning_effort` for reasoning-capable models. The docs and the earlier discussion point toward normalizing incompatible tool choices to `auto`, and the related fix PRs were closed unmerged.

## What This Changes

The issue requests coherent Moonshot-compatible reasoning payload handling so enabled reasoning does not produce `reasoning_effort` together with `thinking.type: disabled` when a pinned tool choice is present.

## Best Possible Solution

Update the Moonshot thinking wrapper to keep enabled thinking coherent by normalizing pinned or otherwise incompatible `tool_choice` values to `auto` when thinking is enabled, while preserving explicit `/think off` behavior. Add regression coverage that prevents `reasoning_effort` or equivalent reasoning fields from coexisting with `thinking.type: disabled` in this path, and align Moonshot docs/changelog if behavior text changes.
