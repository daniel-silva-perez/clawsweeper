## Summary

Keep open. Current main still lacks this PR's html/body viewport lock, openclaw-app 100dvh minimum, .shell--chat min-height removal, and targeted iOS chat textarea font-size fix. Related #64673 remains open and covers only the input-font-size part, so it does not supersede the viewport/global-scrollbar work. Security review did not find a supply-chain or permissions concern because the live PR file list is CSS-only and the earlier viewport maximum-scale change was reverted.

## What This Changes

This PR changes Control UI CSS to use dynamic viewport heights for the app/chat shell, hide body overflow, remove a redundant chat-shell min-height, and raise the agent chat textarea font size on Apple mobile.

## Best Possible Solution

Keep this PR open for maintainer review. Reconcile the input-font-size portion with #64673, then either land a rebased version of the remaining viewport/global-scrollbar fix with focused CSS coverage and iPhone Safari verification, or split the viewport behavior into a narrower tracked PR if #64673 lands first.
