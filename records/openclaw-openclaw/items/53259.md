## Summary

Keep this PR open. Current main still does not implement the PR’s intended Discord elevated allowFrom fallback, while the docs still promise it and the linked bug remains a valid implementation candidate for review/landing.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a reviewed version that adds one shared Discord fallback resolver, uses it from both runtime elevated auth and `sandbox explain` with account context, and keeps regression tests for top-level fallback, account-level fallback, explicit empty override, non-channel session-key diagnostics, and human output guidance.
