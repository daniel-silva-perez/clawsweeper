## What This Changes

The PR adds an opt-in `transientRetry` path to `executeWithApiKeyRotation`, introduces transient provider-operation classification, enables bounded same-key retries for media audio/video and Google embeddings, and adds retry/rotation regression tests.

## Best Possible Solution

Land a narrow opt-in same-key transient retry path for idempotent provider operations after maintainer review, with CI/targeted tests green and docs or changelog wording updated to distinguish alternate-key rate-limit rotation from same-key transient retries.

## Reproduction Assessment

Yes. Against current main, call `executeWithApiKeyRotation` with one API key and an `execute` function that throws an HTTP 500-style error or AbortError; the wrapper calls `execute` once and rethrows because the default predicate is rate-limit-only.

## Solution Assessment

Yes, the PR's explicit `transientRetry` option is the narrow maintainable direction because it preserves 429 alternate-key rotation and avoids globally retrying all provider calls. The safer completion path is to keep the opt-in scope small and finish validation plus docs/changelog policy follow-up before merge.
