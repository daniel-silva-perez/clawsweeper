## Summary

Keep this PR open for maintainer/author follow-up. The change targets a real current-main reliability gap from #60422, but the proposed patch appears to introduce a type-incompatible Google embedding test mock that should be fixed before merge.

## What This Changes

The PR adds an opt-in transient provider-operation retry helper, extends API-key rotation with bounded same-key retries, enables it for media audio/video and Google embeddings, and adds regression coverage for retry behavior.

## Best Possible Solution

Land a narrow opt-in same-key transient retry path for idempotent provider operations, preserving 429 alternate-key rotation and fail-fast behavior for permanent/auth/caller-aborted errors, after fixing the typed test mock and confirming targeted plus changed-lane validation.

## Reproduction Assessment

Yes. Against current main, call `executeWithApiKeyRotation` with one key and an execute function that throws an HTTP 500 or timeout-like error; the wrapper calls execute once and rethrows because the default retry predicate is rate-limit-only.

## Solution Assessment

Mostly yes. Extending the existing wrapper with an explicit opt-in transient retry option is the narrow maintainable direction, but the current PR should make the Google embedding HTTP mock type-compatible before it is merged.
