## Summary

Keep open. Current main still lacks transient same-key retries, and this active PR is a plausible implementation for the linked provider reliability bug; it needs maintainer review and a required changelog entry before merge.

## What This Changes

The PR adds an opt-in transient same-key retry policy to provider API-key execution, enables it for media audio/video and Google embeddings, and adds focused retry/rotation tests.

## Best Possible Solution

Land this PR or an equivalent narrow change after adding the required changelog entry and maintainer review, keeping transient retry opt-in for idempotent provider operations while preserving 429 alternate-key rotation and fail-fast behavior for permanent/auth/caller-aborted errors.

## Reproduction Assessment

Yes. Against current main, call executeWithApiKeyRotation with one API key and an execute function that throws a provider-style HTTP 500 error or AbortError; the wrapper calls execute once and rethrows because the default predicate is rate-limit-only.

## Solution Assessment

Yes for the code direction: an explicit opt-in same-key retry policy is narrower than globally retrying every provider call. The current PR is not merge-ready until the required changelog entry is added and maintainers accept the SDK/docs surface.
