## Summary

Current `main` still lacks REST-native session reset/status/compact/model/reasoning endpoints for HTTP API integrations. The issue remains a valid Gateway HTTP feature request: slash-command dispatch is intentionally not enabled for `/v1/chat/completions`, while Gateway RPC/session protocol methods already provide the underlying session operations.

## Best Possible Solution

Keep this issue open as the specific Gateway HTTP session-management tracker. Implement a focused authenticated REST surface for reset/status/compact/model/reasoning, map it onto existing Gateway session RPC semantics, preserve default-deny slash-command handling for `/v1/chat/completions`, add route/request/response tests, and update `docs/gateway/openai-http-api.md`.
