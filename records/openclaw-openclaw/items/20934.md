## Summary

Keep open. Current main still lacks REST-native session reset/status/compact/model/reasoning endpoints for HTTP `/v1/chat/completions` integrations, while the underlying Gateway RPC/session methods already exist. Related PR #23941 was closed unmerged, and #29985 does not ship the requested REST surface, so this remains a valid canonical Gateway HTTP API tracker.

## Best Possible Solution

Keep this as the canonical Gateway HTTP API tracker. The best implementation is a focused, authenticated REST surface that maps to existing `sessions.resolve`/`sessions.list`, `sessions.reset`, `sessions.compact`, and `sessions.patch` semantics, with explicit route predicates, operator-scope behavior, request/response schemas, tests, and public docs updated together.
