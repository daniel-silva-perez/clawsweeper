## Summary

Keep open. Current main still resolves `openclaw agent --channel discord --to ...` before channel/account delivery metadata is part of session selection, and the resolver still has no delivery-context scan for an existing Discord channel/thread session. The reporter's manual dist patch and the related #44450/#44451 duplicates point at the same remaining source-level gap; #44452 only covers propagation/persistence after a session has already been selected.

## Best Possible Solution

Implement generic delivery-context session routing for `openclaw agent` when a channel target is provided and no explicit session selector should win. Normalize the target through the channel parser, compare against persisted `deliveryContext` and legacy `lastChannel`/`lastTo`/`lastAccountId`/`lastThreadId` fields, preserve explicit selector precedence, and update the CLI docs/changelog to describe the supported behavior.
