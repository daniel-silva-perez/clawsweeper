## Summary

Keep open. Current main still has the source-level failure described in the report: Telegram captionless photos can become media-bearing inbound payloads, but group mention matching is computed from empty Telegram text/caption and returns false before a catch-all regex such as `.*` can activate the turn. The prior fix PR #44883 directly targeted this path but was closed unmerged, and no open replacement PR was found.

## What This Changes

The issue requests that captionless Telegram group photo messages trigger image understanding under normal group mention policy instead of requiring a caption workaround.

## Best Possible Solution

Adjust shared mention matching or Telegram activation so captionless image messages with an intentional catch-all group mention pattern pass the mention gate, while specific patterns, explicit native mentions, reply-to-bot implicit mentions, sender authorization, and media attachment delivery keep their current semantics.
