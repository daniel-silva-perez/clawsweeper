## Summary

Keep open. Current main still appears to drop captionless Telegram group photos before image understanding when group activation depends on a configured catch-all mention pattern: the photo is represented as media, but mention matching receives empty caption/text, skips regex evaluation, and the group path returns `null` on the no-mention decision. The prior direct fix PR #44883 is closed unmerged, so this remains a narrow valid regression.

## What This Changes

Telegram group photo-only messages should trigger image understanding consistently with DMs and captioned group photos when normal group activation policy allows the turn.

## Best Possible Solution

Implement a narrow activation fix so configured catch-all mention patterns can authorize captionless media-only Telegram group turns, with shared mention-helper coverage and a Telegram inbound regression test proving the image placeholder/media attachment reaches the normal handling path.

## Reproduction Assessment

Yes. The issue gives concrete steps, and current main provides a deterministic static reproduction: a captionless Telegram photo yields empty `messageTextParts.text`, `.*` is not evaluated on empty input, and the group handler returns `null` before image understanding.

## Solution Assessment

Yes. The narrow maintainable fix is in mention matching or Telegram inbound activation for configured catch-all media-only group turns; media download, image understanding, group allowlists, and provider routing do not need broad changes.
