## Summary

Keep open. Current main still drops the reported Telegram path before media understanding: captionless photos become media placeholders with media paths available, but group mention matching is computed from text/caption or audio transcript only; when that text is empty, the shared mention helper returns false before testing catch-all regexes, so `requireMention` groups configured with `.*` still skip. A closed unmerged PR, #44883, already identified the same narrow fix.

## What This Changes

Telegram group photo-only messages should trigger image understanding without requiring a caption when the group reply policy is configured to allow them.

## Best Possible Solution

Update mention matching or Telegram activation so captionless image messages with catch-all group mention patterns pass the mention gate, while preserving sender authorization, specific mention-pattern behavior, audio preflight semantics, and media attachment delivery.

## Repair Work Prompt

Fix the Telegram group captionless-image activation bug described in this issue. Current main drops photo-only Telegram group messages before media understanding when `requireMention` is true and activation depends on catch-all `mentionPatterns` such as `.*`, because empty text is not tested against configured mention regexes. Keep the fix narrow: adjust shared mention matching or Telegram-specific activation so catch-all patterns can intentionally match empty/captionless media input, while specific patterns like `\bbot\b` still do not match empty text. Preserve sender/group authorization, explicit native mentions, reply-to-bot implicit mention behavior, audio preflight behavior, and existing media attachment delivery. Use #44883 as source context but do not copy broad or unrelated changes. Add focused regression coverage for captionless Telegram image messages in a requireMention group with `messages.groupChat.mentionPatterns: [".*"]`, plus coverage that a specific mention pattern still skips empty/captionless input.
