## Summary

Keep open. Current main still appears to drop captionless Telegram group photos before image understanding when group activation relies on configured mention patterns such as `.*`; the related fix PR was closed unmerged and no open replacement was found in the provided context or read-only search.

## What This Changes

Telegram captionless photo messages in configured group chats should trigger the same downstream media/image understanding path as DMs or captioned group photos.

## Best Possible Solution

Implement a narrow Telegram/media activation fix so configured catch-all mention patterns can authorize captionless media-only group turns without weakening normal group allowlists, explicit mentions, or media download/provider routing; cover it with shared mention-helper tests and a Telegram inbound regression test.

## Reproduction Assessment

Yes. The issue gives concrete steps, and current main provides a deterministic static reproduction: captionless Telegram photos have empty caption/text, the configured catch-all regex is not evaluated, and the Telegram group handler returns `null` on the no-mention decision.

## Solution Assessment

Yes. The narrow maintainable fix is to adjust mention matching or Telegram inbound activation for configured catch-all media-only group turns; broad image understanding, provider routing, Telegram download, or group allowlist behavior does not need to change.
