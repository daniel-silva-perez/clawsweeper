## Summary

Keep open. Current main still appears to skip captionless Telegram group photos before image understanding when group activation depends on configured mention patterns: the photo becomes media context, but mention matching checks the empty caption/text, `.*` is not evaluated, and the Telegram group path returns `null` on the no-mention decision. The direct source PR #44883 is closed unmerged, so the regression remains actionable.

## What This Changes

Telegram group photo-only messages should trigger image understanding under normal group mention policy without requiring a caption workaround.

## Best Possible Solution

Implement a narrow Telegram/media activation fix so configured catch-all mention patterns can authorize captionless media-only group turns, with shared mention-helper coverage and a Telegram inbound regression test proving the image placeholder/media attachment reaches the normal agent path.

## Reproduction Assessment

Yes. The issue gives concrete steps, and current main provides a deterministic static reproduction: captionless Telegram photos have empty mention text, `.*` is not evaluated on that empty input, and the group handler returns `null` before image understanding.

## Solution Assessment

Yes. The narrow maintainable solution is in mention matching or Telegram inbound activation for configured catch-all media-only group turns; media download, provider routing, and broad group-policy behavior do not need to change.
