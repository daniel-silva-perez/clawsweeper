## Summary

Keep open. Current main still has the reported activation bug: Telegram captionless photos get a media placeholder and downloaded media, but group mention matching is evaluated against empty Telegram text/caption and returns false before a catch-all pattern such as `.*` can activate the turn.

## What This Changes

Ensure Telegram group photo-only messages trigger image understanding when normal group mention policy is satisfied, without requiring a caption.

## Best Possible Solution

Update shared mention matching or Telegram activation so captionless media with an intentional catch-all mention pattern can pass the group mention gate, while preserving specific pattern behavior, native mentions, reply-to-bot implicit mentions, sender authorization, command gating, and media attachment delivery.
