## Summary

Keep open. Current main still appears to drop captionless Telegram group photos before image understanding: the media placeholder is assembled, but mention evaluation uses empty caption/text, the shared mention matcher does not let configured catch-all patterns match empty input, and the group branch returns null on a no-mention decision. The prior direct fix PR #44883 is closed unmerged, so this remains a narrow valid regression.

## What This Changes

Telegram group photo-only messages should trigger image understanding under configured group mention policy without requiring a caption.

## Best Possible Solution

Implement a narrow activation fix so configured catch-all mention patterns can authorize captionless media-only Telegram group turns, preferably with shared mention-helper coverage plus a Telegram inbound regression test that proves the image attachment reaches the normal path.

## Reproduction Assessment

Yes. The issue gives concrete steps, and the current code provides a deterministic static reproduction: a captionless Telegram photo produces empty `messageTextParts.text`, `.*` is not evaluated on empty text, and the group handler returns `null` before image understanding.

## Solution Assessment

Yes, the narrowest maintainable solution is to adjust mention matching or Telegram inbound activation for configured catch-all media-only group turns, not to change media download, group allowlist, or downstream image-understanding behavior.
