## Summary

Keep this issue open. Current main still appears to have the reported gap: Telegram DM topics using the default main DM scope build keys like `agent:main:main:thread:{chatId}:{topicId}`, while Active Memory only classifies explicit/group/channel/direct token keys or unthreaded 3-part main sessions as direct. That means the threaded main DM key still resolves to no chat type and is skipped by the `before_prompt_build` guard before memory lookup or transcript persistence.

## Best Possible Solution

Fix the Active Memory classifier so it treats Telegram default-main DM topic sessions as direct, preferably by stripping the generic `:thread:<id>` suffix before applying the existing session-key classification rules. Add regression coverage for `agent:main:main:thread:{chatId}:{topicId}` with `messageProvider: "telegram"` and `allowedChatTypes: ["direct"]`, without changing Telegram session-key generation or core routing semantics.
