## Summary

Keep this issue open. Current main at f2405c830bd925bd24f91b6bac5d417c6d36e5ba still has the reported Active Memory classifier gap: Telegram DM topic sessions can use `agent:main:main:thread:{chatId}:{topicId}`, but `resolveChatType` still only recognizes explicit/group/channel/direct tokens or an unthreaded 3-part main session key. That threaded default-main DM key still resolves to no chat type and is skipped before recall or transcript persistence.

## Best Possible Solution

Fix the Active Memory session classifier in `extensions/active-memory/index.ts` so Telegram default-main DM topic sessions like `agent:main:main:thread:{chatId}:{topicId}` are classified as `direct` when provider/channel context indicates an interactive direct-message channel. Prefer stripping the generic thread suffix before applying the existing rules, add focused regression coverage in `extensions/active-memory/index.test.ts`, and avoid changing Telegram session-key generation or core routing semantics.
