## Summary

Keep open. Current main still has the reported server-side leak path: malformed store entries can be loaded without required session fields, canonicalized into an agent-scoped key such as `agent:sean:sessions`, retained by `listSessionsFromStore`, and serialized as a Gateway session row with `updatedAt: null` and no `sessionId`. The related implementation PR #57419 is closed and unmerged in the provided context, so there is no shipped fix to close against.

## Best Possible Solution

Keep the issue open and fix this server-side in `listSessionsFromStore` or the combined-store normalization boundary. Require a real non-empty `sessionId` and valid activity timestamp, or an explicitly supported synthetic/session row contract, before building a Gateway session row. Add a regression test with an `agent:<id>:sessions` placeholder lacking `sessionId` and `updatedAt` alongside a real session for that agent, using #57419 as useful context but reviewing the final predicate against current main.
