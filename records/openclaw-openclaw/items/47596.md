## Summary

Keep open. Current main has ACP `streamTo="parent"` support and a parent relay, but the relay still only surfaces assistant text and lifecycle events. Tool starts/results, command output, patch summaries, approvals, and thinking are already represented in the shared event stream, but `startAcpSpawnParentStreamRelay()` ignores them. This is a real remaining ACP parent-relay visibility gap, not a stale or implemented request.

## Best Possible Solution

Keep this open and decide the ACP parent-relay visibility policy first. The likely implementation belongs in `src/agents/acp-spawn-parent-stream.ts`: consume existing `tool`, `item`, `command_output`, `patch`, `approval`, and `thinking` events and emit bounded, redacted parent-session summaries, with tests for default/minimal behavior, verbose behavior, result truncation/redaction, and continued assistant/lifecycle delivery.
