## Summary

Keep open. Current main still wires ACP `streamTo="parent"` through the parent relay, but that relay only surfaces assistant text plus lifecycle status and still drops the structured tool, command output, patch, approval, item, plan, and thinking streams. The existing discussion correctly identifies a real gap; the related closed channel-visibility issue is broader and does not supersede this ACP parent-relay request.

## What This Changes

Add tool call, tool result, and thinking visibility to ACP `runtime="acp"` runs when `streamTo="parent"` sends progress back to the requester session.

## Best Possible Solution

Add a narrow ACP parent-relay policy that keeps existing assistant and lifecycle summaries intact while emitting bounded, redacted summaries for structured progress streams, gated by the maintainer-chosen verbose/default behavior, with focused relay tests and ACP docs updates.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to start `startAcpSpawnParentStreamRelay()`, emit a matching `tool` or `thinking` agent event for the child run, and observe that the callback returns at the non-lifecycle guard without enqueueing a parent system event.

## Solution Assessment

Unclear until maintainers choose the visibility policy. The implementation likely belongs in `src/agents/acp-spawn-parent-stream.ts`, but raw full tool/result/thinking relay would be a weaker solution than redacted summaries with explicit verbosity gating.
