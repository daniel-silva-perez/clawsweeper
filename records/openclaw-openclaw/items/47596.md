## Summary

Keep open. Current main still wires ACP `streamTo="parent"` through a parent relay, but that relay only surfaces assistant text and lifecycle status; existing structured tool, command, patch, approval, and thinking streams are ignored, so the requested visibility gap remains.

## What This Changes

The issue requests parent-session visibility for ACP child-agent tool calls, tool results, and thinking output when `streamTo="parent"` is enabled.

## Best Possible Solution

Add a maintainable ACP parent-relay policy that emits bounded, redacted summaries for existing structured progress streams, keeps assistant/lifecycle behavior intact, and gates sensitive detail behind an explicit verbose or equivalent control before documenting the behavior.

## Reproduction Assessment

Yes. A focused static reproduction is to start `startAcpSpawnParentStreamRelay()`, emit a matching `tool` or `thinking` agent event, and observe from the code that the callback returns at the non-lifecycle guard without enqueueing a parent system event.

## Solution Assessment

Unclear until maintainers choose the visibility policy. The narrow implementation belongs in `src/agents/acp-spawn-parent-stream.ts`, but raw full tool/result/thinking relay would be riskier than redacted summaries with explicit verbosity gating.
