## Summary

Keep open. Current main wires ACP `streamTo="parent"` and has a parent relay, but the relay still only surfaces assistant text plus lifecycle start/end/error style events. Tool starts/results, command output, patch summaries, approvals, and thinking already exist in the shared event stream, but `startAcpSpawnParentStreamRelay()` still drops those streams before they can reach the parent session.

## Best Possible Solution

Keep this open for maintainer policy review. The likely implementation belongs in `src/agents/acp-spawn-parent-stream.ts`: consume existing `tool`, `item`, `command_output`, `patch`, `approval`, and `thinking` events and emit bounded, redacted parent-session summaries according to a clear default/verbose/full policy, with docs and targeted relay tests.
