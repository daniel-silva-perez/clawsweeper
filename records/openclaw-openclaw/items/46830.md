## Summary

Keep open. Current main has useful partial work: Microsoft Teams injects a delivery-failure system event, and shared outbound delivery emits diagnostics and message_sent hooks. The broad feature requested here is still not implemented for normal Feishu, Telegram, or Discord reply delivery, where failures are still logged locally rather than fed back into the agent context.

## Best Possible Solution

Keep this issue open and implement a shared delivery-failure contract for normal channel reply delivery. The best path is to reuse the existing system-event primitive, add a compact cross-channel delivery-failure helper that includes channel, target/thread, error summary, retry hint, and message preview, then wire Feishu, Telegram, Discord, MSTeams, and shared outbound/message-tool paths with focused regression coverage. MSTeams can serve as the reference implementation, but the issue should not close until the main bundled channels or the shared dispatcher cover the behavior.

## Risks / Open Questions

- Closing this now would leave Feishu, Telegram, and Discord reply-delivery failures without the requested model-visible failure signal.
- Existing diagnostics and message_sent hooks could be mistaken for the requested behavior, but they require observers/hooks and do not by themselves tell the agent that the user missed a reply.
- A channel-by-channel solution risks inconsistent retry and recovery semantics unless the shared reply pipeline or plugin SDK defines the delivery-failure contract.
