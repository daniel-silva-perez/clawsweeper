## Summary

Keep open. Current main has the system-event primitive needed for model-visible delivery failures and a Microsoft Teams-specific partial implementation, but the requested cross-channel delivery-failure notification is still missing from the normal shared, Feishu, Telegram, and Discord reply paths. Those paths still primarily log, emit observer hooks, or attempt a same-channel fallback rather than enqueueing a failure into the agent context or returning delivery status to the agent.

## Best Possible Solution

Keep this issue open and implement a shared delivery-failure contract for normal channel reply delivery. Reuse the existing system-event primitive, add a compact cross-channel helper carrying channel, target/thread, error summary, retry hint, and message preview, then wire the shared dispatcher/outbound path plus Feishu, Telegram, Discord, and MSTeams with focused regression coverage. MSTeams can be the reference implementation, but the issue should close only after the main bundled reply paths or a shared dispatcher cover the requested behavior.
