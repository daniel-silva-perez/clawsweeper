## Summary

Keep open. Current main has the right primitive for model-visible system events and a Microsoft Teams-specific partial implementation, but the requested cross-channel delivery-failure notification is still missing for normal Feishu, Telegram, and Discord reply delivery. Those paths still log or emit hook/diagnostic observability rather than enqueueing a failure into the agent context or returning delivery status to the agent.

## Best Possible Solution

Keep this issue open and implement a shared delivery-failure contract for normal channel reply delivery. Reuse the existing system-event primitive, add a compact cross-channel helper with channel, target/thread, error summary, retry hint, and message preview, then wire Feishu, Telegram, Discord, MSTeams, and shared outbound/message-tool paths with focused regression coverage. MSTeams can serve as the reference implementation, but the issue should not close until the main bundled reply paths or the shared dispatcher cover the requested behavior.
