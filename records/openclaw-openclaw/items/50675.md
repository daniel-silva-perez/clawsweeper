## Summary

Keep this PR open. Current main implements the central plugin-command continuation behavior through the public `continueAgent` field, so this PR should not merge as-is with the stale `shouldContinue` API name. However, current main still lacks the PR's nullish command-result guard and still has stale public API/docs wording, so the PR is not fully obsolete.

## Best Possible Solution

Keep this PR open for maintainer follow-up, but do not merge it as-is. The best path is a small rebase or follow-up patch that keeps the current `continueAgent` public API, ports the nullish-result guard, updates `PluginCommandResult` and `registerCommand` documentation plus the stale SDK overview wording, and settles reply-suppression behavior with focused tests.
