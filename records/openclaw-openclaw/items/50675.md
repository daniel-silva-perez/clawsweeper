## Summary

Keep this PR open. Current main already implements the central continuation path for plugin commands, but it does so through the public `continueAgent` field rather than this PR's stale `shouldContinue` name, and the PR still contains useful work to port or explicitly reject: a nullish command-result guard and public API/docs cleanup. The PR should not merge as-is, but it is not fully superseded.

## Best Possible Solution

Keep this PR open for maintainer follow-up, but do not merge it as-is. The best path is a small rebase or follow-up patch that keeps the current `continueAgent` API, ports the nullish-result guard, updates `PluginCommandResult`/`registerCommand` documentation and the stale SDK overview wording, and settles the reply-suppression behavior with focused tests.
