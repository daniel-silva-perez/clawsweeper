## Summary

Current main already implements the central plugin-command continuation behavior, but it standardized the public plugin result field as `continueAgent`, not this PR's `shouldContinue`. The PR should not merge as-is, and it still carries useful cleanup that main has not fully absorbed: a nullish plugin handler result guard and clearer public SDK/JSDoc wording.

## Best Possible Solution

Keep this PR open, but do not merge it as-is. The best path is a small rebase or follow-up patch that preserves main's `continueAgent` public API, ports the nullish command-result guard if maintainers want that tolerance, updates the stale `registerCommand`/SDK wording, and explicitly settles whether replies should be suppressed in the command-handler result when continuation is requested.
