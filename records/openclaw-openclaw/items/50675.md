## Summary

Keep this PR open for maintainer follow-up. Current main already implements plugin-command agent continuation, but it standardized the public API as `continueAgent`, not this PR's `shouldContinue`. The PR should not merge as-is, yet it still contains useful cleanup that current main has not fully absorbed: nullish handler-result hardening and public API/docs wording cleanup.

## Best Possible Solution

Keep the PR open, but do not merge it as-is. The best path is a small rebase or follow-up patch that preserves the current `continueAgent` public API, ports the nullish-result guard, updates `PluginCommandResult` and `registerCommand` documentation plus the stale SDK overview wording, and settles the reply-suppression behavior with focused tests.
