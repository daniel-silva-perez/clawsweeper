## Summary

Keep this PR open because current main still matches Feishu bot mentions by app-scoped open_id only, so the reported multi-bot bug is not implemented away. The PR is useful source work, but it is incomplete and stale against the current Feishu monitor/refactor surface, with blocking gaps in identity sourcing, startup propagation, mention stripping, and debounce handling.

## What This Changes

The PR threads a Feishu bot union_id through probe, startup identity, monitor state, and message parsing so @mention detection can fall back from app-scoped open_id matching in multi-bot groups.

## Best Possible Solution

Refresh or replace the branch on current main after verifying the Feishu stable bot identity contract. The final fix should carry the chosen identifier through probe/startup, monitor state, message parsing, bot mention stripping, mention-forward extraction, debounced merge handling, and sequential command parsing, with focused Feishu regression tests.
