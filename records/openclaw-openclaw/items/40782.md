## Summary

Keep this PR open because the underlying Feishu multi-bot mention bug remains present on current main, and this branch is useful source work but not safe to merge as-is. The proposed union_id fallback targets the right channel-specific failure mode, but the patch leaves important mention paths open_id-only and still needs maintainer/API verification of the stable Feishu identity source.

## What This Changes

The PR threads a Feishu bot union_id through probe results, monitor identity/state, account handling, and message parsing so @mention detection can fall back from open_id to union_id in multi-bot groups.

## Best Possible Solution

Land a refreshed Feishu fix that uses a verified stable bot identity or documented cross-app mention resolution strategy, then carries that identity through startup, monitor state, direct parsing, debounce merge handling, mention stripping, mention-forward extraction, and command parsing with focused Feishu regression coverage.

## Reproduction Assessment

Yes. The paired report gives concrete steps with multiple Feishu bot accounts in one group, and current main has an inspectable open_id-only requireMention path that would drop the message when Feishu emits a non-self mention open_id.

## Solution Assessment

No. The proposed direction is plausible, but this patch is incomplete until the stable Feishu identity contract is verified and the fallback is applied to all mention-derived paths, not only the final checkBotMentioned call.
