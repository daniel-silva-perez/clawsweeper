## Summary

Keep open. The paired Feishu bug still appears valid on current main, but this stale PR is not merge-ready: it assumes a `union_id` source that current main's probe no longer exposes and leaves several `open_id`-only paths in place, including command stripping and debounce mention merging.

## What This Changes

The PR threads a Feishu bot `union_id` from probe and monitor state into message handling so bot mention detection can fall back from `open_id` in multi-bot group chats.

## Best Possible Solution

Refresh or replace the PR after verifying the Feishu/Lark stable bot identity contract. The final fix should carry the chosen identifier through probe results, monitor identity state, message parsing, mention normalization, mention-forward extraction, debounce merging, and sequential command parsing, with regression coverage for mismatched mention `open_id` plus matching stable identity.
