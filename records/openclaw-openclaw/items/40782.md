## Summary

Keep this PR open. Current main still has the reported Feishu multi-bot mention gap, but this PR is not mergeable as-is because the fallback identity source is unverified and the patch does not cover all mention-derived paths.

## What This Changes

The PR threads a Feishu bot union_id through probe results, monitor identity/state, account handling, and message parsing so group @mention detection can fall back from open_id to union_id.

## Best Possible Solution

Land a refreshed Feishu fix that uses a verified stable bot identity or documented cross-app mention-resolution strategy, then applies it consistently to mention detection, bot-mention stripping, mention-forward extraction, debounce merging, startup/recovery state, and focused Feishu regression tests.

## Reproduction Assessment

Yes. The paired report provides concrete multi-bot Feishu group steps, and current main has an inspectable code path where a synthetic group event with a mismatched mention open_id is rejected under requireMention.

## Solution Assessment

No. The union_id fallback direction is plausible, but this patch is not the best mergeable solution until the identity source is verified and all related mention paths use the same fallback semantics.
