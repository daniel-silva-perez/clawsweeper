## Summary

Keep this PR open. Current main still uses Feishu bot `open_id` only for group @mention recognition and related parsing paths, while the same-author bug report #40768 remains open. The PR is a plausible implementation candidate, but it is stale/incomplete against current main because mention detection has moved into `bot-content.ts`, startup/probe state still has no stable bot identifier beyond `botOpenId`, and debounce/mention-forward/mention-stripping paths remain `open_id`-oriented.

## Best Possible Solution

Keep this PR open and ask for a refresh against current main. First verify the Feishu/Lark stable bot identity contract, then thread the chosen identifier through probe results, `FeishuMonitorBotIdentity`, monitor state, message receive/debounce plumbing, `parseFeishuMessageEvent`, `checkBotMentioned`, `normalizeMentions`, mention-forward detection/extraction, and debounce mention merging. Add focused tests for mismatched mention `open_id` with matching stable identity, slash-command stripping, mention-forward targets, and a debounced message batch.
