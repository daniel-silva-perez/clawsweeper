## Summary

Keep open. Current main still uses Feishu open_id-only bot mention detection, while this PR is a plausible implementation candidate for the still-open same-author bug report. The branch is stale against the current Feishu refactor and the proposed fix is incomplete because debounce, mention-forward, and mention stripping paths still need stable-identifier handling.

## Best Possible Solution

Keep this PR open and ask for a refreshed implementation tied to #40768. The best path is to verify the Feishu/Lark identity contract first, then thread the chosen stable bot identifier through probe results, monitor identity state, message parsing, mention stripping, mention-forward detection, debounce merging, and sequential-key parsing. Add focused regression tests for a multi-bot group where the mention open_id differs but the stable identifier matches, including slash-command stripping and a debounced message batch.
