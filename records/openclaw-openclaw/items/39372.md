## Summary

This PR should stay open. Current main still rejects `mode="session"` without `thread=true` in both native subagent and ACP spawn paths, and the PR’s intended behavior is not implemented on main. The PR is relevant, but the proposed diff still needs an ACP delivery fix and maintainer product review before it is mergeable.

## Best Possible Solution

Keep this PR open as an implementation candidate, but require an update before merge. If maintainers want headless persistent sessions, the fix should remove both guards, make ACP inline delivery require an actual bound conversation/thread, preserve failed-start cleanup, emit accurate headless accepted notes, and update tests plus the public subagent/ACP docs. If maintainers prefer the current thread-bound-only model, close the PR with that product rationale and leave the broader discussion in #53548.
