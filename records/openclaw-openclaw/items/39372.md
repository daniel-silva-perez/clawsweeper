## Summary

Current main still rejects the PR's requested headless `mode="session"` behavior in both native subagent and ACP `sessions_spawn` paths. The PR remains useful, but it is not safe to close or merge as-is because the discussion includes an acknowledged ACP inline-delivery blocker and current docs/tests still encode the thread requirement.

Keep this PR open as an implementation candidate, but require an update before merge. If maintainers want headless persistent sessions, the fix should remove the native and ACP guards, make ACP inline delivery require an actual bound thread/conversation, preserve failed-start cleanup, use accurate headless accepted notes after the current accepted-note refactor, and update tests plus the public docs.
