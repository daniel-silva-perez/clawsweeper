## Summary

Current main still rejects the requested headless mode="session" behavior in both native subagent and ACP sessions_spawn paths, and docs/tests still encode the thread=true requirement. The PR remains relevant, but it should not be closed or merged as-is because the ACP inline-delivery path would still treat every session-mode spawn as deliverable to the requester conversation once the guard is removed, and that risk was acknowledged in review discussion.

## Best Possible Solution

Keep this PR open as an implementation candidate, but require an update before merge. If maintainers want headless persistent sessions, the fix should remove both guards, make ACP inline delivery require a real bound thread/conversation, preserve failed-start cleanup, add accurate headless accepted notes through the current helper structure, and update docs/tests. If maintainers prefer the current non-thread workflow, close this PR with that explicit product rationale and leave the broader discussion in #53548.
