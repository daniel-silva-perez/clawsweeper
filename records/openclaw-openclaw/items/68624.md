## Summary

Keep open. Current main does not implement the plugin-neutral session-run cancel seam proposed here, and this PR is still the active implementation candidate for the closed #68550 scope. It is also stacked on the still-open #68622 sessions_send hook PR and has unresolved correctness review around the plugin SDK boundary and late cancel handler registration.

## Best Possible Solution

Keep this PR open as the implementation candidate, but require a rebase or split after #68622 is resolved, remove `emitSessionRunCancel` from the public plugin SDK subpath, make cancel delivery robust for late handler registration, add or keep focused tests for both directions, and then run the appropriate changed gate before merge.

## Risks / Open Questions

- Closing this PR now would leave #68550's requested cancel fan-out seam unimplemented on current main.
- The submitted SDK shape exposes a core-owned emit helper to plugin code, which review comments correctly identify as a boundary and correctness risk because it clears handler buckets.
- The proposed cancel fan-out is not sticky for late subscribers in the reviewed diff, so delegated task runtimes can still miss an abort race unless the implementation records terminal cancellation state or otherwise replays it.
- The PR is stacked on open #68622; merging or closing it independently needs maintainer handling so the sessions_send hook work and cancel seam do not get tangled or lost.
- Security review pass: the provided diff does not add dependencies, workflows, downloaded artifacts, lockfile changes, or lifecycle scripts, but it does broaden public plugin SDK exports and plugin hook dispatch surfaces, so normal maintainer review of plugin authority boundaries is still needed.
