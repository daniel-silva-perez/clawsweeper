## Summary

Keep open. Current main does not implement the requested bidirectional session-run cancel seam, and the PR is still useful, but the proposed patch has blocking correctness issues around public SDK exposure and lost cancellation fan-out for late subscribers. The branch is also stacked on the still-open sessions_send hook PR, so it needs author or maintainer revision rather than cleanup closure.

## What This Changes

The PR branch adds a sessions_send delegation hook plus a plugin-neutral session-run cancellation runtime, SDK subpath/export metadata, gateway chat-abort wiring, shutdown cleanup, generated API baseline updates, and focused tests.

## Best Possible Solution

Keep the PR open, rebase it onto current main after resolving the #68622 dependency, remove the core-owned emitter from the public SDK subpath, make cancellation state replay-safe for late handler registration, guard requester wiring in minimal test gateways, then rerun the targeted session/gateway/plugin-SDK tests, API generation, and build.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Replay cancellation to late session-run handlers:** `src/sessions/session-run-cancel.ts:84-85`
  - body: When core aborts a run, `emitSessionRunCancel` deletes the handler bucket and does not record that the target is already cancelled. If delegated task startup registers `onSessionRunCancel` after that abort, the task never observes cancellation and can continue running after its owning run stopped. Store terminal cancel state and invoke late subscribers immediately or otherwise make the registration path replay-safe.
  - confidence: 0.88
- **[P2] Keep the core cancel emitter out of the SDK:** `src/plugin-sdk/session-run-cancel-runtime.ts:11`
  - body: `emitSessionRunCancel` is the core-to-plugin fan-out primitive and clears registered handlers when called. Re-exporting it from the public plugin SDK lets plugin code consume handlers before the real core abort, so later `abortChatRunById` notification can be silently lost. Export only `onSessionRunCancel`, `requestSessionRunCancel`, and the public types from this runtime subpath.
  - confidence: 0.91
- **[P2] Skip requester wiring for minimal test gateways:** `src/gateway/server-startup-early.ts:124-134`
  - body: `startGatewayEarlyRuntime` skips discovery, skill registry, task maintenance, and maintenance timers when `minimalTestGateway` is true, but this new call still installs a module-level abort requester. Minimal test gateways can therefore claim the singleton cancel requester slot in-process and route requests to the wrong gateway state. Guard this wiring the same way as the other side-effectful startup work.
  - confidence: 0.78
