## Summary

Keep open. Current main does not already implement the requested session-run cancel seam, and the PR is useful source work for #68550, but the proposed patch is not merge-ready because it exposes a core-owned cancel emitter through the public Plugin SDK and loses cancellation events for late delegated-task subscribers.

## What This Changes

The PR adds a plugin-neutral session-run cancel target, gateway chat-abort wiring in both directions, a public Plugin SDK runtime subpath, a sessions_send delegation hook surface, generated SDK baseline updates, and focused tests.

## Best Possible Solution

Land a narrow, plugin-neutral cancel seam after fixing the boundary and lifecycle issues: keep emitSessionRunCancel core-internal, expose only onSessionRunCancel and requestSessionRunCancel to plugins, persist and replay terminal cancellation for late subscribers until cleanup, guard global requester wiring for non-minimal gateways, and document/changelog the new public SDK surface without adding A2A-specific core branches.

## Reproduction Assessment

Yes. The blocking issues are reproducible by static inspection of the PR head: the public SDK subpath exports emitSessionRunCancel, and emitSessionRunCancel deletes or ignores handler buckets without storing already-cancelled state for later onSessionRunCancel calls.

## Solution Assessment

No. The neutral seam is the right product direction, but this patch needs a narrower public SDK boundary and sticky cancel semantics before it is the safest maintainable implementation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Keep the core cancel emitter out of the plugin SDK:** `src/plugin-sdk/session-run-cancel-runtime.ts:11-13`
  - body: The public runtime subpath exports `emitSessionRunCancel`, which lets plugin code fire the core-to-plugin fan-out directly. Since `emitSessionRunCancel` clears the handler bucket before dispatch, an accidental plugin call can consume handlers before the real core abort and leave delegated-task cleanup unnotified.
  - confidence: 0.92
- **[P1] Replay cancels for late session-run handlers:** `src/sessions/session-run-cancel.ts:86-94`
  - body: `emitSessionRunCancel` forgets cancellation when no handler is registered yet and deletes the bucket after dispatch. If a delegated runtime registers its handler just after the owning run is aborted, it never observes the abort and can keep running; store terminal cancel state long enough for late subscribers to receive it once.
  - confidence: 0.86
- **[P2] Skip requester wiring for minimal test gateways:** `src/gateway/server-startup-early.ts:124-134`
  - body: `startGatewayEarlyRuntime` wires the module-level cancel requester even when `minimalTestGateway` is true, while the other side-effectful startup hooks are skipped. A minimal gateway can therefore replace the process-global requester with test-local chat state and make later plugin cancellation route to the wrong gateway state.
  - confidence: 0.8
- **[P3] Add a changelog entry for the new SDK surface:** `package.json:700-703`
  - body: The patch publishes a new Plugin SDK subpath and hook behavior for plugin authors, but it does not update `CHANGELOG.md`. Repo policy requires a changelog entry for user-facing feature/API changes, with appropriate contributor credit.
  - confidence: 0.78
