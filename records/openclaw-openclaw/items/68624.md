## Summary

Keep open. Current main still lacks the requested session-run cancel seam, so the PR is not obsolete, but the current head is not merge-ready: it exposes a core-owned emitter through the public Plugin SDK, drops cancellation for late handlers, wires a process-global requester in minimal gateways, and is stale against current sessions_send behavior on main.

## What This Changes

The PR adds a typed sessions_send hook, a plugin-neutral session_run cancel API, gateway chat-abort wiring in both directions, Plugin SDK export metadata, generated SDK baseline updates, and focused tests.

## Best Possible Solution

Repair or replace the PR with a rebased, plugin-neutral implementation: keep #68622's sessions_send hook dependency explicit or land it first, expose only plugin-safe cancel APIs, make cancellation terminal state replay to late subscribers, guard global requester wiring in minimal gateways, preserve current main sessions_send behavior, and update the SDK docs/changelog/API baseline before validation.

## Reproduction Assessment

Yes. Static inspection of the PR diff reproduces the blockers: the public SDK subpath exports `emitSessionRunCancel`, `emitSessionRunCancel` drops cancellation when no handler exists and clears handlers on dispatch, and requester wiring is added without the existing `minimalTestGateway` guard pattern.

## Solution Assessment

No. The neutral cancel seam is the right direction, but this PR is not yet the safest implementation because the public boundary and lifecycle semantics are too broad, and the stale branch needs a careful rebase over current sessions_send fixes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep the core cancel emitter out of the plugin SDK:** `src/plugin-sdk/session-run-cancel-runtime.ts:11-13`
  - body: The public runtime subpath exports `emitSessionRunCancel`, allowing plugin code to fire and clear the core-to-plugin fan-out without a real core abort. Expose only `onSessionRunCancel`, `requestSessionRunCancel`, and the related types so plugins cannot consume handlers before the owning run is actually cancelled.
  - confidence: 0.92
- **[P1] Replay cancels for late session-run handlers:** `src/sessions/session-run-cancel.ts:82-94`
  - body: `emitSessionRunCancel` returns without recording terminal state when no handler is registered and deletes the bucket before dispatch. A delegated runtime that registers just after the owning run aborts will never observe the cancellation, so store and replay the terminal cancel once for late subscribers until cleanup.
  - confidence: 0.86
- **[P2] Skip requester wiring for minimal test gateways:** `src/gateway/server-startup-early.ts:124-134`
  - body: `startGatewayEarlyRuntime` wires the process-global cancel requester even when `minimalTestGateway` is true, unlike the other side-effectful startup hooks in this function. A minimal gateway can replace the global requester with test-local chat state and route later plugin cancellation to the wrong maps.
  - confidence: 0.8
- **[P3] Add the SDK changelog entry:** `package.json:697-703`
  - body: The patch publishes a new Plugin SDK subpath and hook behavior for plugin authors but does not add a `CHANGELOG.md` entry. Add a user-facing API entry under the active version with valid contributor credit.
  - confidence: 0.78
