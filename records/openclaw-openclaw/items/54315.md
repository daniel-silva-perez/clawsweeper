## Summary

Keep this PR open. Current main has bootstrap truncation warning/reporting support, but it does not implement the PR's central runtime behavior: deterministic priority ordering for SOUL.md/HARD_EXECUTION_RULES.md/AGENTS.md under the total bootstrap budget, nor loading HARD_EXECUTION_RULES.md as a recognized workspace bootstrap file.

## Best Possible Solution

Keep this PR open for maintainer review or contributor rebase. The best implementation path is a focused runtime change that sorts bootstrap files before total-budget consumption, decides whether HARD_EXECUTION_RULES.md should become a recognized/documented bootstrap file, integrates skipped-file diagnostics with the existing bootstrap-budget warning system, and adds regression tests for tight total-budget ordering.
