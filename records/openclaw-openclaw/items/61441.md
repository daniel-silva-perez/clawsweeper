## Summary

Keep this PR open. Current main has part of the lightweight bootstrap plumbing for normal heartbeat/cron runs, but it does not fully implement the PR’s stated compaction and system-prompt-estimate changes. In particular, current main still resolves command system-prompt context and direct compaction context without forwarding `contextMode`/`runKind`, so the PR may still contain useful work after a rebase and cleanup.

## Best Possible Solution

Keep the PR open, rebase it onto current main, drop or split the unrelated cron `NO_REPLY` commits, and focus the patch on the remaining bootstrap-context gaps: command system-prompt estimates plus direct/queued/overflow compaction runtime context. Add focused regression tests for those current paths before merge.
