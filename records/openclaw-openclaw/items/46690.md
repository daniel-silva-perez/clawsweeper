## Summary

Keep open. Current main still documents and enforces bounded skill load roots, but it does not detect arbitrary skill-like directories such as `~/my-skill/SKILL.md` outside those roots. The skills CLI/status path only reports discovered skills from the active workspace/config roots, and skill-creator packaging/scaffolding still accepts caller-chosen paths without warning that they are outside OpenClaw-loaded locations.

## Best Possible Solution

Keep this open for maintainer product/security judgment. The best path is a narrow, bounded diagnostic, likely in `openclaw skills check` or doctor/status, that compares candidate `SKILL.md` directories against documented load roots and reports folders that will not be loaded. Avoid broad implicit home scans unless maintainers approve the privacy/performance tradeoff, and update focused tests plus `https://docs.openclaw.ai/tools/skills` / `https://docs.openclaw.ai/cli/skills` once the policy is chosen.
