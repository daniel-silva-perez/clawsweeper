## Summary

Keep open. Current main documents and enforces canonical skill roots, but it still does not detect arbitrary skill-like directories such as `~/my-skill/SKILL.md` outside those roots. The CLI `skills list/info/check` path only reports skills discovered from active workspace/config roots, and the bundled skill-creator scripts still accept caller-chosen paths without a canonical-root warning.

## Best Possible Solution

Keep this open for maintainer product judgment. The best path is a narrow, bounded diagnostic, likely in `openclaw skills check` or doctor/status, that compares candidate `SKILL.md` directories against the documented load roots and reports folders that will not be loaded. Avoid broad implicit home scans unless maintainers explicitly approve the privacy/performance tradeoff, and update docs/tests around the final canonical-root behavior.
