## Risks / Open Questions

- The PR should not land as-is until the review-thread correctness issues are addressed; several edge cases can report a false up-to-date or low-risk result when registry, GitHub release notes, git fetch, channel fallback, or downgrade state is actually unknown or review-worthy.
- The feature adds a new network dependency on GitHub release metadata for CLI guidance. That is acceptable for a read-only review command, but the implementation needs robust timeout cleanup and conservative recommendations when release data cannot be fetched.
