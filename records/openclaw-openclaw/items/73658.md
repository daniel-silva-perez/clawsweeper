## Summary

Keep this PR open. Current main still has only partial blank-text protection: replay normalization removes blank user text blocks and the direct Anthropic transport filters final payload blocks, but `sanitizeSessionMessagesImages` still does not apply the PR’s sanitizer-level blank-text filtering across the user, toolResult, assistant error, images-only assistant, and preserve-signatures-with-thinking paths. The linked bug remains open and this PR is still a focused implementation candidate.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate for the open Anthropic blank ContentBlock bug. The likely path is a focused sanitizer-level fix in `src/agents/pi-embedded-helpers/images.ts` with regression coverage for preserveSignatures, toolResult, images-only assistant, and assistant error handling, plus maintainer confirmation of skip-versus-placeholder semantics before merge.
