## Summary

Keep this PR open. Current main still recreates `HEARTBEAT.md` whenever it is missing, while runtime behavior and docs preserve a meaningful distinction between a missing heartbeat file and an empty/comment-only one. The PR remains a focused implementation candidate, but it needs a rebase onto the current test layout and a tighter regression fixture before landing.

## Best Possible Solution

Keep this PR open as the implementation candidate. Rebase it onto current main, move the regression coverage into `src/agents/workspace.test.ts`, tighten the negative fixture to create `AGENTS.md`, `SOUL.md`, `TOOLS.md`, `IDENTITY.md`, and `USER.md` while leaving `HEARTBEAT.md` absent, then land the guard that only writes `HEARTBEAT.md` when `isBrandNewWorkspace` is true.
