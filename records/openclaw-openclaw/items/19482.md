## Summary

Keep this PR open. Current main still recreates `HEARTBEAT.md` for any existing workspace where the file is absent, so the PR’s intended behavior is not implemented. The change remains a focused fix candidate for the linked heartbeat regression, but it needs a rebase onto the current test layout and a tighter negative fixture.

## Best Possible Solution

Keep this PR open as the implementation candidate. Rebase it onto current main, move the regression coverage into `src/agents/workspace.test.ts`, tighten the existing-workspace fixture to preseed every canonical workspace file except `HEARTBEAT.md`, then land the `isBrandNewWorkspace` guard around the heartbeat template write.
