## Summary

Keep this PR open. Current main still computes `isBrandNewWorkspace` but writes `HEARTBEAT.md` unconditionally whenever it is missing, so the PR's intended behavior is not implemented. The PR is a focused implementation candidate for the heartbeat regression, but it needs a rebase onto the current test layout and a more exact negative fixture.

## Best Possible Solution

Keep this PR open as the implementation candidate for the linked heartbeat regression. Rebase it onto current main, move the regression coverage into `src/agents/workspace.test.ts`, tighten the negative fixture to create `AGENTS.md`, `SOUL.md`, `TOOLS.md`, `IDENTITY.md`, and `USER.md` while leaving `HEARTBEAT.md` absent, then land the guard that only writes `HEARTBEAT.md` when `isBrandNewWorkspace` is true.
