## Summary

Keep this PR open. Current main still computes `isBrandNewWorkspace` but writes `HEARTBEAT.md` unconditionally, so an existing workspace that intentionally removed the file can still get an empty template recreated. The PR remains a focused implementation candidate, with a prior maintainer/Codex comment already identifying the same remaining work and the needed test refresh.

## Best Possible Solution

Keep this PR open as the implementation candidate. Rebase it onto current main, move the regression coverage into `src/agents/workspace.test.ts`, tighten the negative fixture to preseed the canonical workspace files while leaving `HEARTBEAT.md` absent, then land the `isBrandNewWorkspace` guard around the heartbeat template write.
