## Summary

Keep this PR open. Current main still recreates `HEARTBEAT.md` whenever an existing workspace lacks it, while the runtime and docs intentionally distinguish a missing heartbeat file from an effectively empty one. The PR remains a narrow implementation candidate, but it needs a rebase onto the current workspace test layout and a tighter negative fixture.

## Best Possible Solution

Keep this PR open as the implementation candidate. Rebase it onto current main, move the regression coverage into `src/agents/workspace.test.ts`, model a fully initialized workspace with `AGENTS.md`, `SOUL.md`, `TOOLS.md`, `IDENTITY.md`, and `USER.md` present while `HEARTBEAT.md` is absent, then land the `isBrandNewWorkspace` guard around the heartbeat template write.
