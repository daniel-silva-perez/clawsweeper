## Summary

Keep this PR open. Current main has a later partial WebKit light-mode scrollbar fix from #61753, but it still lacks this PR's root-level Firefox scrollbar support, 8px/theme-token global scrollbar styling, and config overflow cleanup. The PR is still a focused CSS-only implementation candidate for the remaining scrollbar behavior rather than something already implemented on main.

## Best Possible Solution

Keep this PR open, or replace it with an equivalent maintainer-owned Control UI patch. The best path is to intentionally choose the global scrollbar width and colors, add root-level Firefox `scrollbar-width`/`scrollbar-color` support if desired, remove or justify the config `overflow: clip` declarations, and capture a small manual QA note or visual check for light/dark Chrome and Firefox.
