## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.79

Full review comments:

- **[P1] Re-check containment at the write boundary:** `extensions/skill-workshop/src/skills.ts:239`
  - body: `prepareProposalWrite()` does the realpath checks, but the final write still uses the saved string path after that check. If the skill directory is swapped for a symlink after the second prepare and before `atomicWrite()` opens the temp path, `fs.writeFile`/`rename` can still land outside `<workspace>/skills/<name>`, so the advertised path/TOCTOU invariant is not enforced at the commit boundary. Move the containment/open-with-root check into the final write helper or use a no-follow root-bound write primitive.
  - confidence: 0.78
