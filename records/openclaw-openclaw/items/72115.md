## Summary

Keep open. Current main still lacks the requested Skill Workshop hardening, and the PR is a useful implementation candidate, but the latest head is not merge-ready because it introduces a proxy fail-open regression, still has the bundled override path-normalization bug, and GitHub reports the branch as dirty against main despite green checks.

## What This Changes

The PR hardens Skill Workshop writes with realpath containment, write-boundary revalidation, and prompt-budget preflight through plugin SDK exports, adds docs/tests, and carries related QA, plugin-loader, proxy, channel, and CI stabilization changes.

## Best Possible Solution

Land the Skill Workshop hardening after the proxy activation path fails closed when global-agent cannot bootstrap, bundled override comparisons use the same resolveUserPath/realpath normalization as bundled-dir resolution, and the branch is rebased or split so maintainers can review the hardening separately from broad support changes.

## Reproduction Assessment

Yes. For the PR findings, enable proxy mode in a runtime where global-agent cannot be resolved and startProxy still returns an active handle, or set OPENCLAW_BUNDLED_PLUGINS_DIR to a value like ~/bundled and compare bundled-dir's resolveUserPath expansion against facade-loader's raw path.resolve check.

## Solution Assessment

No, not yet. The Skill Workshop hardening direction fits the existing plugin/SDK boundary, but the proposed support changes need the proxy fail-closed behavior and bundled override normalization fixed before this is the narrowest maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Fail closed when global-agent cannot bootstrap:** `src/infra/net/proxy/proxy-lifecycle.ts:254-258`
  - body: The network proxy docs rely on global-agent to cover node:http/node:https callers, but this change logs a warning and still returns an active proxy handle when the package cannot be loaded. In that state OpenClaw reports proxy routing as active while Node core HTTP clients can bypass it, so proxy activation should fail instead of continuing without the hook.
  - confidence: 0.86
- **[P2] Normalize bundled overrides before comparing paths:** `src/plugin-sdk/facade-loader.ts:149-160`
  - body: OPENCLAW_BUNDLED_PLUGINS_DIR is expanded through resolveUserPath in bundled-dir, but this new check compares against path.resolve of the raw env value. Values like ~/bundled point at the real override for discovery but fail this comparison, so modules from that override can be treated as outside it and runtime dependency staging gets disabled.
  - confidence: 0.88
