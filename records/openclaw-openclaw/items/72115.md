## Summary

Keep open. Current main does not implement the Skill Workshop write-path, TOCTOU, and prompt-budget hardening, and the PR is substantively useful, but the latest head is still not merge-ready because it mis-normalizes bundled-plugin override paths and is currently dirty against main.

## What This Changes

The PR hardens Skill Workshop workspace skill writes with realpath containment, write-boundary revalidation, prompt-budget parity, SDK exports, docs, tests, and related QA/plugin-loader stabilization changes.

## Best Possible Solution

Land the Skill Workshop hardening after the bundled override comparison uses the same user-path and realpath normalization as bundled-dir resolution, the branch is made mergeable against current main, and maintainers accept or split the support changes intentionally.

## Reproduction Assessment

Yes for the PR finding: set OPENCLAW_BUNDLED_PLUGINS_DIR to a home-relative value such as ~/bundled and load a bundled public surface from the expanded override. bundled-dir expands the value with resolveUserPath, but the new facade-loader comparison uses path.resolve on the raw string and can incorrectly disable runtime dependency staging.

## Solution Assessment

Not yet. The Skill Workshop hardening direction fits the existing root-bound write and SDK seam model, but the proposed loader change should normalize the override with the same resolver before deciding whether it is safe to skip runtime dependency staging.

## Review Findings

- **[P2] Normalize bundled override paths before comparing:** `src/plugin-sdk/facade-loader.ts:149-156`
