## Summary

Keep open. Current main does not contain the EV-002B Skill Workshop hardening, and the latest PR head still has a concrete write-boundary race in the security-critical part of the patch, so this needs maintainer-requested changes before merge rather than cleanup closure.

## What This Changes

The PR adds realpath containment, stale-write checks, prompt-budget preview/enforcement, SDK exports, docs, tests, and several merge-support changes across Skill Workshop, QA, gateway RPC, bundled plugin resolution, proxy, channel, and test surfaces.

## Best Possible Solution

Keep Skill Workshop as the procedural writer, but commit `SKILL.md` and support files through a pinned root-bound SDK write primitive such as `writeFileWithinRoot` or an equivalent no-follow directory-fd writer, and add a regression test that proves no out-of-root temp file or directory is created during a symlink-swap race.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Pin the final Skill Workshop write to the verified root:** `extensions/skill-workshop/src/skills.ts:140-143`
  - body: `atomicWriteInsideRoot()` still performs `mkdir` and `writeFile(tempPath, ...)` through path strings after a containment check. If the skill directory is swapped to a symlink between the check and those operations, the helper can create directories or a temp file outside `<workspace>/skills/<name>/` before the later check throws. Route the final commit through a root-bound no-follow writer and assert that no outside temp path is created.
  - confidence: 0.87

## Security Review
