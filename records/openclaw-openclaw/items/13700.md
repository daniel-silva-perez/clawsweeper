## Summary

Keep open. Current main has useful adjacent primitives, especially compaction checkpoint branch/restore and native subagent transcript forking, but it still lacks the requested named `/session save` and `/session load` workflow, reusable snapshot/template support, and restore/import from reset or deleted transcript archives. The recent maintainer comment in the discussion also explicitly kept the design open.

## What This Changes

Request a first-class session snapshot workflow that can save, load, restore, and fork conversation/context checkpoints by name.

## Best Possible Solution

Define a native session snapshot design on top of SessionManager, session store metadata, compaction checkpoint artifacts, reset/deleted transcript archives, and subagent fork support, then add coherent save/list/load/branch semantics across slash commands, Gateway protocol, CLI/Control UI where appropriate, docs, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. The current-main gap is high-confidence from source and docs: `/session` only accepts `idle`/`max-age`, while the available checkpoint and fork primitives are narrower than the requested named snapshot workflow.

## Solution Assessment

Unclear. The requested direction is reasonable, but the exact best solution needs maintainer design for restore semantics, retention/import, permissions, and cache-preserving forks before a narrow implementation can be chosen.
