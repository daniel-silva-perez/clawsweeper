## Summary

Keep open. Current main has adjacent compaction checkpoint branch/restore and native subagent transcript-fork primitives, but it still lacks the requested named `/session save` and `/session load` workflow, reusable snapshot templates, and restore/import from `.reset` or `.deleted` transcript archives; the recent maintainer comment also explicitly kept this issue open for design follow-up.

## What This Changes

Request adds explicit session snapshot commands and APIs to save, load, restore, and fork conversation state including transcript, context, model state, and reusable checkpoints.

## Best Possible Solution

Define a native session snapshot design that builds on SessionManager, session store metadata, compaction checkpoint artifacts, reset/deleted transcript archives, and subagent fork support, then add coherent save/list/load/branch semantics across slash commands, Gateway protocol, CLI/Control UI where appropriate, docs, and focused tests.
