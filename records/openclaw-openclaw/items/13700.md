## Summary

Keep open. Current main has useful adjacent primitives for compaction checkpoint branch/restore and native subagent transcript forking, but it still does not implement the requested named `/session save` and `/session load` workflow, reusable session templates, or restore/import from `.reset` and `.deleted` transcript archives; the recent maintainer comment also kept this open for design follow-up.

## What This Changes

The issue requests explicit session snapshots that can save, load, restore, and fork a session's messages, context state, model choice, and related checkpoint data.

## Best Possible Solution

Define a native session snapshot design that builds on SessionManager, session store metadata, compaction checkpoint artifacts, reset/deleted transcript archives, and subagent fork support, then add coherent save/list/load/branch semantics across slash commands, Gateway protocol, CLI/Control UI where appropriate, docs, and focused tests.
