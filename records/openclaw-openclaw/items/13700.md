## Summary

Keep open. Current main has adjacent primitives, especially compaction checkpoint list/get/branch/restore RPCs, Control UI checkpoint restore, and native `sessions_spawn` transcript forking, but it still does not provide the requested named `/session save` and `/session load` workflow, reusable session templates, or restore from `.reset`/`.deleted` transcript archives. The 2026-04-26 maintainer comment already kept this open, and current main at `9bb1e59447a5fdf2c0164672df6c748188b26442` still matches that assessment.

## Best Possible Solution

Keep this open and design a native session snapshot feature that builds on SessionManager, the session store, compaction checkpoint metadata, reset/deleted archive handling, and subagent fork primitives. The design should define save/list/load/branch semantics, slash/Gateway/CLI and possibly Control UI surfaces, destructive-vs-branch restore behavior, restore/import rules for `.reset` and `.deleted` archives, permissions, retention, docs, and focused tests.

## Risks / Open Questions
