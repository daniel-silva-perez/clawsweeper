## Summary

Keep open. Current main has adjacent compaction checkpoint branch/restore and native subagent transcript forking, but it still does not provide the requested named `/session save` and `/session load` workflow, reusable session templates, or restore/import from `.reset` and `.deleted` transcript archives; a recent maintainer comment also explicitly kept this open for design follow-up.

## What This Changes

Add explicit session snapshot commands and APIs for saving, loading, restoring, and branching conversation checkpoints across long-running OpenClaw sessions.

## Best Possible Solution

Define a native session snapshot design that builds on SessionManager, session store metadata, compaction checkpoint artifacts, reset/deleted transcript archives, and subagent fork support, then add coherent save/list/load/branch semantics across slash commands, Gateway protocol, CLI/Control UI where appropriate, docs, and focused tests.
