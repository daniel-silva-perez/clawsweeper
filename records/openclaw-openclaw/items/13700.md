## Summary

Keep open. Current main has adjacent compaction checkpoint branch/restore and parent transcript fork primitives, but it still does not provide the requested named `/session save` and `/session load` workflow, reusable snapshot templates, or restore/import from `.reset` and `.deleted` transcript archives. The issue discussion also broadened the request into context-poison rollback, prompt-cache-sensitive forks, batch reuse, and restart recovery, and the recent maintainer comment explicitly kept the design open.

## What This Changes

Request to add explicit session snapshot commands and APIs for saving, loading, restoring, and branching conversation context checkpoints.

## Best Possible Solution

Define a native session snapshot design on top of SessionManager, session store metadata, compaction checkpoint artifacts, reset/deleted transcript archives, and subagent fork support, then add coherent save/list/load/branch semantics across slash commands, Gateway protocol, CLI/Control UI where appropriate, docs, and focused tests.
