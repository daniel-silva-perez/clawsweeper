## Summary

Keep this issue open. Current main still has adjacent compaction checkpoint branch/restore and native subagent transcript forking, but it does not implement arbitrary named session snapshots, `/session save`, `/session load`, `/session restore`, reusable snapshot templates, or restore from `.reset` / `.deleted` transcript archives. The recent maintainer-facing issue comment also explicitly kept this open after reviewing the same product gap.

## Best Possible Solution

Keep this issue open until maintainers either implement a deliberate native session snapshot surface or split it into smaller canonical issues. A complete path would add typed Gateway snapshot save/list/load/delete methods, `/session save|load|restore` command handling, UI/CLI controls, deterministic transcript and model/session metadata preservation where feasible, reusable spawn-from-snapshot support, and a safe restore path for `.reset` / `.deleted` transcript archives.
