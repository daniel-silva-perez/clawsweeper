## Summary

Keep open. Current main has adjacent compaction checkpoint branch/restore and native subagent transcript-fork support, but it still lacks the requested named `/session save` and `/session load` workflow, reusable templates, and import/restore from reset or deleted transcript archives; a recent maintainer comment also kept the design open.

## What This Changes

Request to add explicit session snapshot save/load commands and related restore, branch, and reusable-template workflows for OpenClaw session context.

## Best Possible Solution

Define one native session snapshot model that reuses SessionManager, session metadata, compaction checkpoint artifacts, reset/deleted transcript archives, and subagent fork support, then expose coherent save/list/load/branch/import semantics through chat commands, Gateway protocol, CLI/Control UI where appropriate, docs, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. The current-main gap is high-confidence from source and docs: `/session` only accepts `idle`/`max-age`, no exact save/load command surface exists, and the available checkpoint/fork primitives are narrower than named snapshots.

## Solution Assessment

Unclear. The requested direction is reasonable, but the best implementation needs maintainer decisions on destructive restore versus branch/import, retention, permissions, archive import, and prompt-cache-preserving fork invariants before a maintainable fix can be chosen.
