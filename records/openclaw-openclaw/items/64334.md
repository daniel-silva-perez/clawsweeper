## Summary

Keep this issue open. Current main still does not add a cumulative transcript-backed session cost line to `/status` or `session_status`; it only shows the existing per-reply estimated `Cost:` line when API-key pricing is visible. The separate `/usage cost` command and transcript cost aggregation already exist, and the previously referenced PR #72517 is now closed as a README-only non-implementation.

## What This Changes

Request to add cumulative session cost to `/status` by summing transcript `usage.cost.total` values across the current session.

Add a cumulative session-cost line in the shared status renderer used by chat `/status` and `session_status`, reusing the existing transcript/session cost aggregation path, keeping `/usage cost` as the detailed summary command, and updating focused tests, docs, and changelog credit.
