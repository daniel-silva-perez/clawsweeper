## Summary

Keep this issue open. Current main still does not surface cumulative transcript-backed session cost in `/status` or `session_status`; it only renders the existing estimated `Cost:` field from status token fields and pricing, while transcript cost aggregation and `/usage cost` remain separate paths.

## What This Changes

The issue requests a cumulative session cost line in `/status`, derived by summing `usage.cost.total` from the session transcript.

## Best Possible Solution

Add a cumulative, transcript-backed session-cost line in the shared status renderer used by chat `/status` and `session_status`, reusing existing transcript/session cost aggregation and keeping `/usage cost` as the detailed summary view.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: transcript cost is already read into `snapshot.costUsd`, but the status helper discards it and the renderer never emits a cumulative session-cost line.

## Solution Assessment

Yes. The narrowest maintainable fix is to extend the shared status data path and focused tests, not duplicate logic in the chat command or change session persistence semantics.
