## Summary

Keep this issue open. Current main still does not surface a cumulative transcript-backed session cost in `/status` or `session_status`; it only renders the existing estimated `Cost:` field from status token fields and pricing. The transcript cost aggregation and `/usage cost` command already exist, so this remains a narrow status-rendering feature rather than an implemented request.

## What This Changes

The issue requests a cumulative session cost line in `/status`, derived by summing `usage.cost.total` from the session transcript.

## Best Possible Solution

Add a cumulative, transcript-backed session-cost line in the shared status renderer used by chat `/status` and `session_status`, reusing the existing session-log cost aggregation path while keeping `/usage cost` as the detailed summary view.
