## Summary

Keep this open. Current main still has only the partial WhatsApp reconnect/group metadata fallback from the related merged work, while the Baileys socket retry/cache options and proactive group/LID event handlers requested here are still absent from the WhatsApp plugin.

## What This Changes

Add WhatsApp/Baileys reliability hooks for group messages, including retry `getMessage`, socket-level group metadata caching, proactive group metadata updates, participant updates, and LID mapping handling.

## Best Possible Solution

Implement this as a WhatsApp plugin-owned reliability patch: provide a bounded recent-message lookup for Baileys retry handling, pass the existing group metadata cache through `cachedGroupMetadata`, update that cache from group and participant events, integrate LID updates through the existing auth/LID mapping helpers, and add focused plugin tests for those paths.
