## Summary

Keep this issue open. Current main still does not wire the Baileys `getMessage` retry callback, socket-level `cachedGroupMetadata`, proactive group metadata/participant event handlers, or `lid-mapping.update` persistence in the bundled WhatsApp plugin. Open PR #73732 is relevant to the group inbound reconnect subset, but it remains open and does not cover the broader retry/cache/LID reliability request.

## Best Possible Solution

Keep this issue open. Review, land, or supersede #73732 for the group inbound reconnect subset, then handle the broader Baileys reliability hooks in a focused WhatsApp plugin follow-up: account-scoped bounded message retry storage wired through `getMessage`, a group metadata cache exposed through socket-level `cachedGroupMetadata`, proactive cache updates from group/participant events, and an explicit decision on `lid-mapping.update` persistence beyond the current LID lookup paths. Add focused regression tests before closing.
