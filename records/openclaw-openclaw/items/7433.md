## Summary

Keep this issue open. Current main contains the partial reconnect/group metadata fallback from the merged WhatsApp work, but the requested Baileys socket-level retry/cache hooks and proactive group/LID event handling are still absent from the WhatsApp plugin.

## What This Changes

Request to improve WhatsApp/Baileys group reliability by wiring Baileys `getMessage`, socket-connected group metadata caching, group update listeners, participant update handling, and LID mapping updates.

## Best Possible Solution

Land a focused WhatsApp plugin patch that wires Baileys retry/cache hooks through existing session and inbound-monitor boundaries, keeps any recent-message store bounded and privacy-preserving, updates group metadata from Baileys group events, persists or consumes LID mappings where the auth store expects them, and adds targeted regression coverage for those socket options and events.
