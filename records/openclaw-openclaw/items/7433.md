## Summary

Keep this issue open. Current main still has only the partial WhatsApp reconnect/group metadata fallback from the related merged work; the requested Baileys `getMessage`, socket-level `cachedGroupMetadata`, proactive group/participant update listeners, and `lid-mapping.update` handling are still absent. The newest discussion also adds useful implementation guidance: `getMessage` needs outbound message content retained before send acknowledgement to cover sender-side retransmit requests.

## What This Changes

Request to improve WhatsApp/Baileys group reliability by adding retry-message lookup, socket-level group metadata caching, proactive group metadata updates, participant updates, and LID mapping persistence in the WhatsApp plugin.
