## Summary

Keep this open. Current main still has only the partial reconnect-time group metadata fallback from the merged WhatsApp recovery work; the broader Baileys retry/cache hooks and proactive group/LID event handling requested here remain absent from the WhatsApp plugin.

## What This Changes

The issue requests WhatsApp/Baileys reliability improvements for group messages by adding retry message lookup, socket-connected group metadata caching, proactive group metadata/participant event handling, and LID mapping updates.

## Best Possible Solution

Land a focused WhatsApp plugin patch that wires Baileys retry/cache hooks through the existing session and inbound-monitor boundaries, keeps recent-message retention bounded, updates group metadata from Baileys group events, consumes or persists LID mappings where the auth store expects them, and adds targeted regression coverage for those socket options and events.

## Solution Assessment

## Repair Work Prompt

Implement the remaining WhatsApp/Baileys group reliability hooks requested by openclaw/openclaw#7433. Scope the fix to the WhatsApp plugin; do not change core, plugin SDK contracts, dependency versions, package manager patches, or unrelated channel behavior.

Expected boundaries:
- Wire a bounded, privacy-preserving `getMessage` callback into `createWaSocket` so Baileys retry requests can retrieve recent sent message content without unbounded retention.
- Wire `cachedGroupMetadata` into the Baileys socket using the existing plugin-owned group metadata cache or a narrowly shared helper.
- Add inbound socket event handling for `groups.upsert`, `groups.update`, and `group-participants.update` so the cache stays fresh across reconnects and group churn.
- Handle `lid-mapping.update` consistently with the existing Baileys LID mapping store/auth-dir lookup behavior; avoid duplicating persistent state if Baileys already owns it, but add the missing event consumption/persistence if needed.
- Keep #73732's reconnect fallback behavior intact, including bounded metadata cache behavior and no broad core migration.
- Add targeted regression tests for socket options plus group/LID event handling.
- Add a user-facing changelog fix entry if behavior changes, crediting @octopuslabs-fl and any relevant related reporters; do not credit bots.
