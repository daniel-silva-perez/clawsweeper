## Summary

Keep open. Current main still has no configurable operator-scope selection for `openclaw mcp serve`, and the issue is already paired with the open replacement implementation PR #74428 after #74030 closed unmerged, so the conservative maintainer path is to review or land the linked PR rather than close this issue or queue duplicate repair work.

## What This Changes

Request a supported way for `openclaw mcp serve` to request narrower operator scopes or explicit least-privilege modes instead of always requesting read, write, and approvals.

## Best Possible Solution

Land #74428 or an equivalent narrow implementation that validates repeatable MCP bridge scopes, threads `OperatorScope[]` through the CLI, server, bridge, and `GatewayClient`, preserves the current read/write/approvals default when omitted, and updates docs plus focused CLI/server tests for explicit, invalid, default, and cached-token behavior.
