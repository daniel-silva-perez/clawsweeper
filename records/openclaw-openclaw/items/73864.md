## Summary

Keep open. Current main still lacks configurable `openclaw mcp serve` operator scopes, but the issue is now paired with open PR #74428, which is the active replacement implementation for the closed-unmerged #74030; the maintainer path is to review or land that PR rather than close this issue or queue a separate repair.

## What This Changes

Request configurable operator-scope selection for `openclaw mcp serve` so MCP bridge profiles can ask for read/write/approvals subsets instead of always requesting the fixed three-scope set.

## Best Possible Solution

Land #74428 or an equivalent narrow implementation that validates repeatable MCP bridge scopes, threads `OperatorScope[]` through CLI, server, bridge, and `GatewayClient`, preserves the default read/write/approvals request when omitted, documents the contract, and covers explicit/default/cached-token behavior with tests.
