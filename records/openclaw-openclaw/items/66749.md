## Summary

Keep this PR open. Current main still has the targeted unguarded exec-event heartbeat wake behavior, and the same-author linked issue #66748 remains open, so this PR remains a plausible implementation candidate rather than cleanup-close material.

## Best Possible Solution

Keep this PR open for maintainer review. The likely best path is a narrow guard that skips requestHeartbeatNow for isSubagentSessionKey(sessionKey) in both exec completion paths while preserving enqueueSystemEvent, plus regression coverage for explicit exec system events and notify-on-exit. If accepted, landing this PR or an equivalent patch should resolve #66748.
