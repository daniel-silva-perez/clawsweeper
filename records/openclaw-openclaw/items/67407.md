## Summary

Keep this issue open. Current main still has adjacent config/startup safeguards and the cited lossless-claw regression is fixed separately, but there is still no first-class `openclaw gateway preflight [--json]` command or shared no-bind startup validation routine that resolves configured plugins/context engines and checks bindability before serving traffic.

## What This Changes

The issue requests a no-bind `openclaw gateway preflight [--json]` command that validates gateway config, plugin and context-engine startup paths, and port availability before serving traffic.

## Best Possible Solution

Add `openclaw gateway preflight [--json]` as a Gateway CLI command backed by a shared no-bind startup validation routine. It should parse and validate config, resolve auth/SecretRefs, load the same startup plugin and context-engine registration paths in a no-mutation mode, probe configured port bindability without accepting connections, emit stable human/JSON results with exit code 0/1, and document the command at the public Gateway CLI page before composing it into a later safe-update flow.
