## Summary

Keep open. Current main has generic stale-socket health policy improvements, but the Mattermost WebSocket path still lacks active ping/pong keepalive and does not publish `lastTransportActivityAt`, so the silent half-open/deaf-socket failure described here is not demonstrably fixed. The open related PR #57621 is the right implementation candidate to review rather than closing this issue now.

## Best Possible Solution

Review and either land or supersede #57621 with a Mattermost-owned fix: add active WebSocket ping/pong or equivalent transport liveness, publish `lastTransportActivityAt` from real transport activity, and add focused tests in `extensions/mattermost/src/mattermost/monitor-websocket.test.ts` plus health-policy/monitor coverage if the shared contract changes. After the implementation merges, close this issue as implemented or consolidate it under #41837 if that remains the canonical tracker.
