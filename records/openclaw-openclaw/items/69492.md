## Summary

Current main still lacks a SystemEvent audience/classification primitive, and the related implementation is still an open partial PR. The existing code still represents system events with text, timestamp, contextKey, deliveryContext, and trusted only; exec completion and heartbeat paths classify by text/routing flags rather than per-event audience. This should stay open for maintainer design review alongside #72201, #72218, and #72253.

## Best Possible Solution

Keep this open as the canonical maintainer discussion. Review #72201 together with the heartbeat tactical fixes (#72218 and #72253), decide whether OpenClaw wants a SystemEvent audience primitive and which producers it covers, then close this only after the accepted design lands or the remaining exec/cron/heartbeat leakage work is fully superseded by narrower canonical issues.
