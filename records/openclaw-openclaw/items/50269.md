## Summary

Keep open. Current main still does not implement the PR’s intended `openclaw gateway status --deep` text behavior: the probe does not measure elapsed time, `DaemonStatus.rpc` cannot carry a duration, the renderer prints probe status/capability without any `RPC duration:` line, and current tests do not cover duration rendering or JSON stability for that field. The related health-duration items are separate command-surface work and do not supersede this gateway status PR.

## Best Possible Solution

Keep this PR open as a still-relevant focused CLI fix. The best path is to review or land a patch that measures probe elapsed time in `src/cli/daemon-cli/probe.ts`, carries it through `DaemonStatus.rpc`, prints it for successful and failed text status probes in `src/cli/daemon-cli/status.print.ts`, and preserves the JSON output contract with targeted gather/print tests.
