## Summary

Current main still does not implement this PR’s intended behavior. `openclaw gateway status --deep` text output still has no RPC probe duration line, `probeGatewayStatus` does not measure elapsed time, and `DaemonStatus.rpc` has no field to carry it. The PR remains a focused implementation candidate rather than cleanup-close material.

## Best Possible Solution

Review and land this PR, or an equivalent focused patch, so `probeGatewayStatus` measures elapsed time, `DaemonStatus.rpc` carries `durationMs`, `printDaemonStatus` shows `RPC duration: <n>ms` for successful and failed text probes, and JSON output remains stable with targeted gather/print tests.
