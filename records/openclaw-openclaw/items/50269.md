## Summary

Keep this PR open. Current main still does not implement the intended `openclaw gateway status --deep` text behavior: the probe does not measure duration, `DaemonStatus.rpc` has no duration field, and the renderer has no `RPC duration:` output. The PR discussion and diff show a focused CLI/test change, not an obsolete or already-shipped change.

## Best Possible Solution

Keep this PR open as a still-relevant focused CLI fix. The best path is to review or land a patch that measures elapsed probe time in `src/cli/daemon-cli/probe.ts`, carries it through `DaemonStatus.rpc`, prints it for both successful and failed text status probes in `src/cli/daemon-cli/status.print.ts`, and preserves the JSON contract with targeted gather/print tests.
