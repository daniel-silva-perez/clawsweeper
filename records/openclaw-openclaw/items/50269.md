## Summary

Keep this PR open. Current main still does not implement the intended gateway status text behavior: the daemon status probe does not measure elapsed time, `DaemonStatus.rpc` has no duration field, and the text renderer prints probe status/capability without any duration line. The PR remains a focused implementation candidate, and the provided discussion/diff show no unrelated security-sensitive changes.

## Best Possible Solution

Review and land this PR or an equivalent focused patch that measures elapsed probe time in `src/cli/daemon-cli/probe.ts`, carries it through `DaemonStatus.rpc`, prints duration for both successful and failed text status probes in `src/cli/daemon-cli/status.print.ts`, and preserves the JSON contract with targeted gather/print tests.
