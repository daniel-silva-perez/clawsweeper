## Summary

Keep open. Current main has not implemented this PR's intended Dockerfile simplification, and the proposed change is not an obvious cleanup to auto-close or auto-accept: it removes digest pinning and the documented `OPENCLAW_EXTENSIONS` build contract while current code, docs, CI, and tests still depend on those behaviors.

## Best Possible Solution

Keep this PR open for maintainer review or author revision. A viable version should either preserve the current digest pinning and `OPENCLAW_EXTENSIONS` contract, or explicitly update the Docker docs, setup scripts, workflows, and tests with maintainer-approved migration notes and equivalent validation. If maintainers do not want this risky infra simplification, it should be closed manually as an explicit product/build-policy decision rather than by automated cleanup.
