## Summary

Keep open. Current main still has the reported cold-registry ordering bug, and open PR #73904 is already paired with this issue through closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that cliBackend-only cold starts can hit `sessions_spawn` before the lazy context-engine registry has registered the built-in `legacy` engine.

## Best Possible Solution

Review and land PR #73904, or an equivalent narrow patch, so native subagent spawn preparation initializes built-in context engines before resolving the configured engine. Keep PR #75129 as the separate follow-up for the CLI compaction lane.

## Reproduction Assessment

Yes. A cold process that reaches native `sessions_spawn` before any PI embedded path calls `ensureContextEnginesInitialized()` can call `resolveContextEngine()` against an empty registry and hit the exact reported `legacy` error.

## Solution Assessment

Yes. Initializing at the spawn-side resolution boundary is the narrowest maintainable fix for this path; making built-in registration a top-level side effect would have broader bundling/runtime implications.
