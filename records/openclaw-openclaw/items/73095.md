## Summary

Keep open. Current main still resolves the context engine in native subagent spawn preparation before initializing the built-in registry, and the provided GitHub context shows open PR #73904 is the focused fix candidate with closing syntax for this issue.

## What This Changes

This issue requests initializing built-in context engines before `prepareContextEngineSubagentSpawn` resolves the configured engine on cold cliBackend-only `sessions_spawn` calls.

## Best Possible Solution

Land #73904 or an equivalent narrow change that initializes built-in context engines at the native subagent spawn preparation boundary before resolving the configured engine, then review #75129 separately for the adjacent CLI compaction path.

## Reproduction Assessment

Yes. Current main has a source-level reproduction path: a cold process can enter `sessions_spawn`, reach `spawnSubagentDirect()`, then resolve the default `legacy` context engine before `ensureContextEnginesInitialized()` has registered it, which matches the reported error and the independent reporter confirmations.

## Solution Assessment

Yes. Initializing with `ensureContextEnginesInitialized()` at the spawn-side resolution boundary is the narrow maintainable fix for this report; moving registration to a top-level side effect would be broader and riskier.
