## Summary

Keep open. Current main and the latest release still have the reported spawn-prep ordering bug, and open PR #73904 is already linked with closing syntax as the implementation candidate.

## What This Changes

The issue requests initializing built-in context engines before native `sessions_spawn` subagent preparation resolves the configured context engine on cliBackend-only cold starts.

## Best Possible Solution

Review and land PR #73904, or an equivalent narrow patch, so native subagent spawn preparation initializes built-in context engines before resolving the configured engine. Keep PR #75129 as a separate follow-up for the CLI compaction lane.

## Reproduction Assessment

Yes. A cold cliBackend-only process that reaches native `sessions_spawn` before any PI embedded path initializes context engines can call `resolveContextEngine()` against an empty registry and hit the exact reported `legacy` error.

## Solution Assessment

Yes. Initializing at the subagent spawn preparation boundary is the narrowest maintainable fix for this path; making built-in registration a top-level side effect would be broader and riskier.
