## Summary

Keep open. Current main still has the reported resolve-before-init path, and the issue is already paired with open PR #73904 using closing syntax, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Initialize the built-in context-engine registry before native `sessions_spawn` subagent preparation resolves the configured engine on cliBackend-only cold starts.

## Best Possible Solution

Land #73904 or an equivalent narrow change that wires `ensureContextEnginesInitialized()` into the native subagent spawn preparation dependency path before resolving the context engine, with the cold-registry regression test retained.

## Reproduction Assessment

Yes. A cold process using cliBackend-routed chat can reach `sessions_spawn` without PI runner initialization; current main then calls `resolveContextEngine()` for the default `legacy` slot before registration and hits the explicit unregistered-engine throw path.

## Solution Assessment

Yes. The linked PR's initializer call at the spawn-side resolution boundary is the narrowest maintainable fix and mirrors adjacent initialized-first subagent paths; making registration a top-level side effect would be a broader bundling/runtime change.
