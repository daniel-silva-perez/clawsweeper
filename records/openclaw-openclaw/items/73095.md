## Summary

Keep open. Current main still has the cold context-engine registry ordering bug on the native `sessions_spawn` path, and open PR #73904 is already the focused implementation candidate linked to this issue; the issue should remain open until that PR, or an equivalent fix, lands.

## What This Changes

The issue requests initializing built-in context engines before `prepareContextEngineSubagentSpawn()` resolves the configured context engine for cliBackend-only cold starts.

## Best Possible Solution

Land PR #73904, or an equivalent narrow patch, so the native subagent spawn preparation boundary calls the idempotent built-in context-engine initializer before resolving the configured engine. Keep PR #75129 as the separate CLI compaction follow-up unless maintainers choose to combine the two call sites explicitly.

## Reproduction Assessment

Yes. A cold process that reaches native `sessions_spawn` before any PI embedded path or subagent registry path has initialized context engines will resolve the default `legacy` engine against an empty registry and produce the reported error.

## Solution Assessment

Yes. Initializing at the spawn-side resolution boundary is the narrow maintainable fix for this path; making legacy registration a top-level import side effect would have broader bundling/runtime implications.
