## Summary

Keep open. Current main still has the reported resolve-before-initialize path, and open PR #73904 uses closing syntax for this issue, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Initialize built-in context engines before `prepareContextEngineSubagentSpawn` resolves the configured context engine on cliBackend-only cold starts.

## Best Possible Solution

Land #73904 or an equivalent narrow change that initializes built-in context engines at the spawn-preparation resolution boundary, with the regression test retained; review #75129 separately for the adjacent CLI compaction path.

## Reproduction Assessment

Yes. A cold cliBackend-only process can reach native `sessions_spawn` without PI runner initialization; current main then resolves the default `legacy` engine before registration and hits the registry’s explicit unregistered-engine error.

## Solution Assessment

Yes. Calling the existing idempotent initializer before spawn-prep resolution is the narrowest maintainable fix for the reported path and avoids changing context-engine registration into a broader top-level side effect.
