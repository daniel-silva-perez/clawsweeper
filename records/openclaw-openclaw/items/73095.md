## Summary

Keep open. Current main still has the reported cold-registry ordering bug in the native subagent spawn path, and open PR #73904 is already paired with this issue through closing syntax as the focused fix candidate.

## What This Changes

The issue requests initializing the built-in context-engine registry before `prepareContextEngineSubagentSpawn` resolves the configured engine for cliBackend-only cold starts.

## Best Possible Solution

Land PR #73904 or an equivalent narrow patch that adds `ensureContextEnginesInitialized()` to the native subagent spawn preparation boundary before resolving the configured context engine, while reviewing PR #75129 separately for the CLI compaction lane.

## Reproduction Assessment

Yes. A cold process that reaches `sessions_spawn` through the native subagent path can call `resolveContextEngine()` before the lazy `legacy` registration runs, producing the exact reported error string.

## Solution Assessment

Yes. Initializing at the spawn-side resolution boundary is the narrow maintainable fix for this path; making registration a top-level side effect would have a broader bundling/runtime blast radius.
