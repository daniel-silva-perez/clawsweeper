## Summary

Keep this PR open. Current main still recurses into the first object-valued `anyOf`/`oneOf` variant during Control UI config form coercion, so the PR's discriminator-aware object variant selection remains useful unmerged work. The provided follow-up commit also appears to address the review concern about unconstrained variants winning before constrained matches.

## Best Possible Solution

Keep this PR open and review or land the discriminator-aware object-variant selection in `ui/src/ui/controllers/config/form-coerce.ts`. The best implementation should preserve existing primitive union and array fallback behavior, prefer object variants with matching top-level `const` constraints, fall back to the first object variant when no discriminator match exists, and include focused `anyOf`/`oneOf` regression tests including the unconstrained-variant edge case raised in review.
