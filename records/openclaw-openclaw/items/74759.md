## Summary

Keep this open. Current main has configured SecretRef resolution with env/file/exec providers, but it still lacks lazy agent-callable `secret.get`/`secret.has`/`secret.list` tools and the trusted redaction/session-scope contract needed to return live credentials safely.

## What This Changes

Request a first-class secret broker plugin surface that resolves named secrets from Bitwarden, macOS Keychain, 1Password, or Vault without storing secret values in transcripts or unsafe sessions.

## Best Possible Solution

Define a trusted secret capability contract on top of the existing SecretRef provider foundation, including `has/list` semantics, redacted or non-persistent `get` results, and a pre-execution session-scope gate; then implement the first backend either as a bundled trusted plugin or after extending the plugin API to safely express that trust boundary.

## Reproduction Assessment

Not applicable as a feature and security-design request. The current-main gap is high-confidence from source and docs inspection: configured SecretRefs exist, while lazy agent-callable `secret.*` tools and a safe redaction/session-scope API are absent.

## Solution Assessment

Unclear. The requested generic plugin direction is not sufficient by itself because external plugins cannot currently register the trusted pre-model result middleware; the safer path is a maintainer-approved bundled implementation or a core plugin-API extension first.

Status: needs_attention

Summary: This issue is security-sensitive because it proposes returning live credential material through agent tools before OpenClaw has a safe result-redaction and trusted-session contract for that surface.

  - body: A `secret.get` tool would intentionally return credential material.   - confidence: 0.93
- **[high] Gate secret tools by trusted session scope:** `docs/channels/groups.md:81`
  - body: OpenClaw has group and channel sessions with separate authorization behavior. Secret tools should be hidden or blocked before execution for group/channel and otherwise untrusted sessions, not only redacted after the tool result is produced.
  - confidence: 0.88
