## Summary

Keep this open. Current main has configured SecretRef resolution with env/file/exec providers, but it still lacks lazy agent-callable `secret.get`/`secret.has`/`secret.list` tools; the existing external plugin surface also cannot safely provide the requested pre-model redaction because tool-result middleware remains bundled-only. The follow-up comment correctly identifies this as a maintainer product/security choice rather than a narrow implementation task.

## What This Changes

Add a secret broker plugin exposing first-class agent-callable `secret.get`, `secret.has`, and `secret.list` over Bitwarden, macOS Keychain, 1Password, and Vault with transcript redaction and trusted session-scope guards.

## Best Possible Solution

Define a trusted secret capability contract on top of the existing SecretRef provider foundation, including `has/list` semantics, redacted or non-persistent `get` results, and a pre-execution trusted-session gate; then implement the first backend either as a bundled trusted plugin or after extending the plugin API to express that trust boundary safely.

## Reproduction Assessment

Not applicable as a feature and security-design request. The current-main gap is high-confidence from source and docs inspection: configured SecretRefs exist, while lazy agent-callable `secret.*` tools and a safe external redaction/session-scope API are absent.

## Solution Assessment

Unclear. The requested generic plugin direction is not sufficient by itself because external plugins cannot currently register the trusted pre-model result middleware; the safer path is a maintainer-approved bundled implementation or a core plugin-API extension first.

## Security Review

Status: needs_attention

Summary: This issue is security-sensitive because it proposes returning live credential material through agent tools before OpenClaw has a safe result-redaction and trusted-session contract for that surface.

Concerns:

- **[high] Define redacted secret result semantics before get:** `src/plugins/registry.ts:369`
  - body: A `secret.get` tool would intentionally return credential material. Current external plugin tools return normal tool content, while pre-model tool-result rewriting is restricted to bundled plugins, so core needs a trusted redacted or non-persistent result contract before any backend returns secrets.
  - confidence: 0.93
- **[high] Gate secret tools by trusted session scope:** `docs/channels/groups.md:23`
  - body: OpenClaw supports group and channel sessions with separate authorization behavior. Secret tools should be hidden or blocked before execution for group/channel and otherwise untrusted sessions, not only redacted after a tool result is produced.
  - confidence: 0.88
