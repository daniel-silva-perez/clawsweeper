## Summary

Keep this open. Current main has an eager configured SecretRef system with env/file/exec providers, but it still lacks lazy agent-callable `secret.get`/`secret.has`/`secret.list` tools and the trusted redaction plus session-scope contract needed before returning live credentials to an agent.

## What This Changes

Request to add a secret broker plugin exposing `secret.get`, `secret.has`, and `secret.list` over Bitwarden, macOS Keychain, 1Password, and Vault, with transcript redaction and trusted-session guards.

## Best Possible Solution

Define a trusted secret capability contract on top of the existing SecretRef provider foundation, including redacted or non-persistent result semantics and a pre-execution session-scope gate, then implement the first backend either as a trusted bundled plugin or after extending the plugin API safely.

## Reproduction Assessment

Not applicable as a feature and security-design request. The current-main gap is high-confidence from source and docs inspection: configured SecretRefs exist, while lazy agent-callable `secret.*` tools and safe redaction/session-scope APIs are absent.

## Solution Assessment

Unclear. A generic plugin is not enough by itself today because external plugins cannot register the trusted pre-model middleware; the safer path is a maintainer-approved bundled implementation or a core plugin-API extension first.

## Security Review

Status: needs_attention

Summary: This issue is security-sensitive because it proposes an agent-callable tool that intentionally returns credential material before OpenClaw has a safe result and session-trust contract for that surface.

Concerns:

- **[high] Define secret-result redaction before exposing get:** `src/plugins/registry.ts:369`
  - body: A `secret.get` tool would return credential material. Current external plugin tools produce normal tool content, while pre-model tool-result rewriting is restricted to bundled plugins, so core needs a trusted redacted or non-persistent result contract first.
  - confidence: 0.93
- **[high] Gate secret tools by trusted session scope:** `docs/channels/groups.md:76`
  - body: OpenClaw has distinct group/channel sessions and group authorization behavior. Secret tools should be hidden or blocked for group and otherwise untrusted sessions before execution, not only redacted after the result exists.
  - confidence: 0.86
