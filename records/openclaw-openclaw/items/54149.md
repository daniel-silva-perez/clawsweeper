## Summary

Current main still does not resolve the reported Volcengine/BytePlus Kimi catalog mismatch. The old shared catalog path has been removed and the model rows now live in the provider plugin manifests, but Kimi K2.5 and Kimi K2 Thinking still have `maxTokens: 4096` and omit `reasoning`, which the manifest loader defaults to `false`. Together still advertises Kimi K2.5 as reasoning-capable with `maxTokens: 32768`, so the issue remains real provider-catalog work pending vendor verification.

## Best Possible Solution

Keep this issue open. The best fix is a provider-owned catalog update in `extensions/volcengine/openclaw.plugin.json` and `extensions/byteplus/openclaw.plugin.json`, verified against current Volcengine/BytePlus vendor limits and reasoning controls, plus focused regression tests that assert Kimi K2.5 and Kimi K2 Thinking metadata propagates through the provider builders and augmented model catalog. Update provider docs if corrected reasoning or output-token metadata changes user-visible model/status expectations.
