## Summary

Keep open. Current main still does not implement the intended direct-compaction behavior: the shared direct compaction helper computes counts from `session.messages`, but the plugin `before_compaction` event still receives only counts. The PR is coherent and has supportive discussion, but its patch is now stale against the refactored helper and needs a defensive pre-compaction message snapshot plus stronger tests before merge.

## Best Possible Solution

Keep this PR open and request an update against current main: thread a defensive pre-compaction message snapshot through `runBeforeCompactionHooks`, assert exact pre-compaction contents plus the empty transcript case, document or type the intended read-only/snapshot semantics for `PluginHookBeforeCompactionEvent.messages`, and then run the focused compaction hook tests before normal changed-gate validation.
