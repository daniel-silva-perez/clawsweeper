## Summary

Keep this PR open. Current main still does not implement the intended direct-compaction behavior: the direct before_compaction helper computes counts from session.messages but sends only messageCount/tokenCount to plugin hooks. The hook type and other compaction paths already support messages, so the feature is coherent, but the PR still needs a defensive pre-compaction snapshot, stronger content assertions, and explicit transcript exposure/mutation semantics before it should land.

## Best Possible Solution

Keep this PR open and request an update: thread a defensive pre-compaction message snapshot through compactEmbeddedPiSessionDirect/runBeforeCompactionHooks, assert exact pre-compaction contents plus the empty-transcript case in compact.hooks tests, and clarify transcript exposure/mutation semantics before running the focused compaction hook tests. Related #55745 covers the typed hook API, while related #44152 and #44163 are separate compaction-hook payload gaps, so none of them supersedes this PR.
