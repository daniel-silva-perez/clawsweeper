## Summary

Keep open. Current main and the latest release still route Telegram DM answer previews through message-edit transport, while the lower-level draft stream still supports the requested DM `sendMessageDraft` path; the focused open fix candidate remains #47631, so this issue should not be closed before that PR or an equivalent fix lands.

## What This Changes

The issue reports a Telegram DM partial-streaming regression where previews changed from smooth `sendMessageDraft` behavior to chunkier `sendMessage`/`editMessageText` updates after OpenClaw 2026.3.8.

## Best Possible Solution

Review, update, and land #47631 or an equivalent Telegram plugin patch that lets normal DM answer partial previews use draft-preferring auto transport again, keeps reasoning/tool-progress behavior stable, materializes final text without duplicate replies, and updates Telegram streaming tests and docs to make the intended DM transport explicit.
