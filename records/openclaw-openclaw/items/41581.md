## Summary

Keep open. Current main and the latest release still force Telegram DM partial answer previews through `sendMessage`/`editMessageText`, while the lower-level Telegram draft stream still supports the requested `sendMessageDraft` path. There is now an open focused implementation candidate in #47631, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw to restore smooth `sendMessageDraft`-style Telegram DM partial streaming instead of coarse `sendMessage`/`editMessageText` preview updates.

## Best Possible Solution

Review, update if needed, and land #47631 or an equivalent Telegram plugin patch that restores native draft transport for normal DM answer partial previews, keeps reasoning/group/topic behavior stable, preserves finalization and stale-preview safeguards, and updates Telegram streaming docs to match the intended runtime behavior.
