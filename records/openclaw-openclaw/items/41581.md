## Summary

Keep open. Current main and the latest stable tag still force Telegram DM partial answer previews through `sendMessage`/`editMessageText` message transport, while the lower-level draft stream still has the DM-preferring `sendMessageDraft` path when the dispatcher leaves `previewTransport` on `auto`. The issue also has multiple user confirmations and an open unmerged fix PR, so it should stay open for maintainer review rather than be swept closed.

## What This Changes

The issue asks to restore smooth Telegram DM `streaming: "partial"` behavior by avoiding the post-2026.3.8 forced message-edit transport for normal DM answer previews.

## Best Possible Solution

Restore the Telegram plugin so normal DM answer partial previews use the draft-preferring `auto` transport again, while preserving message transport where needed for reasoning/tool-progress, native quote exclusions, stale preview cleanup, duplicate-final suppression, and fallback behavior. The intended transport should be locked in with Telegram regression tests and reflected in the Telegram and streaming docs.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: in a DM with preview streaming enabled and no native-quote or block-streaming exclusion, current main passes `previewTransport: "message"`; leaving it as `auto` would select the draft transport in `createTelegramDraftStream`.

## Solution Assessment

Yes, the best fix direction is a narrow Telegram plugin transport-selection and finalization fix, not a core streaming rewrite. #47631 is the active candidate, but it needs review/rebase and coverage before this issue can close.
