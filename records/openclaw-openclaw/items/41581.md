## Summary

Keep open. Current main and the latest stable tag still force Telegram DM partial answer previews through `sendMessage`/`editMessageText`, while the lower draft-stream layer still prefers `sendMessageDraft` for DM threads when left on `auto`; the report also has several user confirmations and an open unmerged fix PR.

## What This Changes

The issue requests restoring smooth Telegram DM partial streaming so normal DM answer previews use the draft-style `sendMessageDraft` path instead of coarse message-edit updates.

## Best Possible Solution

Restore the Telegram plugin so normal DM answer partial previews use the draft-preferring `auto` transport again, while preserving message transport for cases that need it, including reasoning/tool-progress conflicts, native quote exclusions, fallback paths, stale preview cleanup, and duplicate-final suppression. Update Telegram regression tests and the public Telegram/streaming docs to lock in the intended DM transport.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: with DM scope and preview streaming enabled, current main passes `previewTransport: "message"`; leaving it as `auto` would select draft transport in `createTelegramDraftStream`.

## Solution Assessment

Yes, the best fix direction is a narrow Telegram plugin transport-selection and finalization change, not a core streaming rewrite. PR #47631 is the active candidate, but it remains open and unmerged, so the issue should stay open until a reviewed fix lands.
