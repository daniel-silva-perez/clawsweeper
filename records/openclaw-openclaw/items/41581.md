## Summary

Keep open. Current main and the latest release still route Telegram DM partial answer previews through `sendMessage`/`editMessageText`, while the lower draft-stream helper still supports native `sendMessageDraft` for DM threads when the dispatcher leaves transport on `auto`. The issue is already the canonical report for this regression, with #47631 open as the focused implementation candidate and #62207/#62210 closed unmerged.

## What This Changes

Restore Telegram DM `channels.telegram.streaming = "partial"` previews to the smoother draft-style `sendMessageDraft` behavior instead of coarse message-edit updates after 2026.3.8.

## Best Possible Solution

Land #47631 or an equivalent Telegram plugin change that lets normal DM answer lanes use `auto` so `createTelegramDraftStream` can choose native draft transport, keeps reasoning/tool-progress behavior stable, preserves duplicate-final and media/button/error fallback safeguards, and updates Telegram streaming tests plus docs to state the intended DM transport clearly.
