## Summary

Keep open. Current main still forces Telegram DM partial answer previews through message/edit transport, while the lower draft stream still supports native `sendMessageDraft` for DMs when left on `auto`; latest release evidence and current tests/docs still match the reported regression, and #47631 remains the open implementation candidate rather than a merged fix.

## What This Changes

The issue reports that Telegram direct-message partial streaming regressed from native draft-style previews to chunkier `sendMessage`/`editMessageText` updates after OpenClaw 2026.3.8.

## Best Possible Solution

Restore the Telegram plugin so normal DM answer partial previews use the draft-preferring `auto` transport again, while keeping reasoning/tool-progress and fallback finalization safeguards stable; update Telegram streaming tests, docs, and changelog so the intended DM transport is explicit.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: with Telegram preview streaming enabled in a DM and no native quote/block-streaming exclusion, current main passes `previewTransport: "message"` to draft lanes, while `auto` would select native draft transport for DM threads.

## Solution Assessment

Yes, restoring `auto` for the normal DM answer lane while preserving message transport for reasoning and existing finalization safeguards is the narrow maintainable fix. A broad core change is not warranted because the regression is Telegram-plugin transport selection and lifecycle behavior.
