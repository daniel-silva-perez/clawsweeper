## Summary

Keep open. Current main still routes Telegram DM partial streaming through message-preview transport for answer previews, while the lower-level draft stream still supports DM `sendMessageDraft`; the dispatcher tests and public docs currently lock in the reported choppy `sendMessage`/`editMessageText` behavior rather than a shipped fix.

## What This Changes

Restore Telegram DM `streaming: "partial"` previews to the smoother `sendMessageDraft`-style behavior reported from 2026.3.2 instead of coarse `editMessageText` updates.

## Best Possible Solution

Implement a targeted Telegram plugin fix that lets normal DM answer partial previews use the draft-preferring auto transport again, while preserving final materialization, duplicate-final suppression, stale-preview cleanup, native-quote exclusions, and the intended reasoning/tool-progress behavior. Update the Telegram streaming docs and regression tests so the expected DM transport is explicit.

## Repair Work Prompt

Fix the Telegram DM partial streaming regression tracked by openclaw/openclaw#41581. Current main forces DM preview lanes through `previewTransport: "message"`, making `channels.telegram.streaming: "partial"` use coarse `sendMessage`/`editMessageText` updates instead of the draft-preferring `sendMessageDraft` path that still exists in `createTelegramDraftStream`. Implement a narrow Telegram plugin fix, likely in `extensions/telegram/src/bot-message-dispatch.ts`, so normal DM answer partial previews can use `previewTransport: "auto"`/draft transport again. Do not blindly remove all transport guards: preserve final materialization, duplicate-final suppression, stale-preview cleanup, native quote behavior, media/buttons fallback, and decide explicitly via tests whether reasoning/tool-progress lanes stay message-based or use draft transport. Update regression tests in `extensions/telegram/src/bot-message-dispatch.test.ts`, `extensions/telegram/src/draft-stream.test.ts`, and/or `extensions/telegram/src/lane-delivery.test.ts` to prove DM answer partial streaming chooses the draft path and finalizes without duplicate messages. Update `docs/channels/telegram.md` and `docs/concepts/streaming.md` so Telegram DM transport docs match behavior. Add a user-facing changelog fix entry with reporter credit, not bot credit. Do not move Telegram behavior into core and do not change unrelated channel streaming semantics.
