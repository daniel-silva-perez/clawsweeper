## Summary

Keep open. Current main still forces Telegram DM partial answer previews through `sendMessage`/`editMessageText` message transport, while the lower draft stream still supports native `sendMessageDraft` when the dispatcher leaves DM previews on `auto`; the latest release contains the same forced path and the focused fix PR remains open rather than merged.

## What This Changes

The issue asks to restore Telegram DM `partial` streaming to the smoother native `sendMessageDraft`/draft-preferring path instead of forced `sendMessage`/`editMessageText` preview edits.

## Best Possible Solution

Restore the Telegram plugin so normal DM answer partial previews use the draft-preferring `auto` transport again, while preserving message transport where needed for reasoning/tool-progress, duplicate-final suppression, stale-preview cleanup, and native quote exclusions. Update Telegram regression tests, public docs, and the changelog so the intended DM transport is explicit.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: with Telegram preview streaming enabled in a DM and no native quote or block-streaming exclusion, current main passes `previewTransport: "message"`; leaving it on `auto` would select native draft transport for DM threads.

## Solution Assessment

Yes. The narrow maintainable solution is in the Telegram plugin transport selection and finalization logic, not a broad core change: restore `auto` for the normal DM answer lane while preserving existing safeguards around reasoning lanes and final delivery.
