## Summary

Keep open. Current main still forces Telegram DM partial answer previews through `sendMessage`/`editMessageText` message transport, while the lower draft stream still supports native `sendMessageDraft` whenever the dispatcher leaves DM transport on `auto`; the latest release tag shows the same forced-message path, and #47631 remains the focused open implementation candidate rather than a merged fix.

## What This Changes

Restore Telegram direct-message partial streaming so normal answer previews use draft-capable `auto`/`sendMessageDraft` transport instead of being forced through `sendMessage` plus `editMessageText`.

## Best Possible Solution

Land #47631 or an equivalent Telegram plugin change that lets normal DM answer partial previews use the draft-preferring `auto` transport again, while keeping reasoning lanes and final-delivery safeguards stable and updating Telegram streaming tests plus docs to state the intended DM transport clearly.
