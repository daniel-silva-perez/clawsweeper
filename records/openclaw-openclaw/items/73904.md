## Summary

Keep this PR open for maintainer review and CI. Current main still resolves the context engine during `sessions_spawn` preparation without first initializing built-in engines, while the PR adds the missing idempotent initializer call and focused regression coverage; the linked reports and latest maintainer-adjacent comment support this as the canonical fix.

## What This Changes

This PR adds `ensureContextEnginesInitialized()` to the subagent spawn runtime dependency boundary, calls it before `resolveContextEngine()` during native `sessions_spawn` preparation, and adds a regression test for the cold registry ordering.

## Best Possible Solution

Land this PR, or an equivalent patch, after CI confirms the touched agent/context-engine tests remain green; then close #73095 and #74865 as implemented if they are still tracking the same cold registry regression.

## Reproduction Assessment

Yes. #73095 gives a concrete cliBackend-only cold-start path with no prior PI warm-up, and current source confirms an empty default registry throws before spawn preparation can proceed.

## Solution Assessment

Yes. The PR uses the same narrow initializer-before-resolver ordering already present in adjacent context-engine paths, avoiding a broader side-effect registration change in the context-engine module.
