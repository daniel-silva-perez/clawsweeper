## Summary

Keep this PR open for maintainer review and CI. Current main still has the cold-registry ordering bug in the `sessions_spawn` path, and the PR is a narrow source/test fix with no blocking review findings from this read-only pass.

## What This Changes

The PR wires `ensureContextEnginesInitialized()` into the subagent spawn dependency/runtime seam, calls it before `resolveContextEngine()` during spawn preparation, and adds a focused cold-registry regression test.

## Best Possible Solution

Land this PR, or an equivalent initializer-before-resolve patch with the same regression coverage, after CI confirms the touched agent runtime and tests remain green. Keep the CLI compaction follow-up separate unless maintainers prefer one combined landing.

## Reproduction Assessment

Yes. A cold cliBackend-only process can reach `sessions_spawn` without any prior PI path registering the legacy context engine; current main then resolves the default `legacy` engine against an empty registry and returns the reported error.

## Solution Assessment

Yes. Calling the existing idempotent initializer immediately before the spawn path resolves the context engine is the narrowest maintainable fix; moving registration to top-level module side effects would be broader and riskier.
