## Summary

Keep this PR open. Current main still calls `resolveContextEngine()` in the `sessions_spawn` preparation path without first initializing built-in context engines, while the linked report and follow-up comment describe a concrete cliBackend-only cold-start failure that this PR narrowly addresses. The diff is source/test-only and does not introduce security-sensitive supply-chain changes.

## Best Possible Solution

Keep this PR open for normal maintainer review and CI completion, then land it or an equivalent patch that initializes built-in context engines immediately before `sessions_spawn` context-engine preparation resolves the configured engine. After merge, #73095 can close as implemented.
