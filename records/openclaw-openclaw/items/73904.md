## Summary

Keep open for maintainer review and merge consideration. Current main still resolves the context engine in the `sessions_spawn` preparation path before calling the built-in initializer, while this PR adds the missing idempotent initializer call plus focused regression coverage; it is not implemented on main yet. The only concrete merge cleanup I found is the missing changelog entry for a user-facing fix, and the newly filed #75129 should remain a separate companion for the CLI compaction site.

## What This Changes

The PR adds `ensureContextEnginesInitialized()` to the subagent spawn runtime dependency path, calls it before `prepareContextEngineSubagentSpawn()` resolves the context engine, and wires a cold-registry ordering regression test through the subagent spawn test helper.

## Best Possible Solution

Land this PR, or an equivalent one-line initializer call with the same regression coverage, after adding the missing changelog entry and confirming the normal changed gate remains green. Keep #75129 as the separate follow-up unless a maintainer prefers one combined landing.

## Reproduction Assessment

Yes. The high-confidence reproduction is a cold process where cliBackend-routed chat has not initialized context engines, followed by `sessions_spawn`; current main then resolves the default `legacy` engine against an empty registry and returns the reported error.

## Solution Assessment

Yes for the spawn endpoint. Calling the idempotent built-in initializer immediately before resolving the context engine is the narrowest maintainable fix and matches adjacent initialized-first paths; the separate CLI compaction site should be handled by #75129 or a follow-up, not used to block this focused patch.
