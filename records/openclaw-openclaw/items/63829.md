## Summary

Keep this issue open. Current main still does not provide per-agent memory-wiki vault isolation: the memory-wiki plugin resolves one plugin config at registration, the vault schema only supports the existing global modes, agent entries do not accept per-agent plugin overrides, and wiki read/write/gateway paths continue to use the captured vault path. The related open PR #67584 is an implementation candidate, but it is not merged into current main and should be reviewed rather than using it as a close reason.

## Best Possible Solution

Keep this issue open as the memory-wiki vault isolation tracker. Review and land #67584 or an equivalent memory-wiki-owned change that resolves the vault root per invocation context or provides explicit per-agent vault mapping, preserves the current single-vault default, updates schema/docs, and adds regression tests proving two agents on one gateway cannot read or write each other's vaults through wiki_search, wiki_get, wiki_apply, and gateway/CLI management surfaces.
