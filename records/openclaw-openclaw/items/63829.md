## Summary

Keep this issue open. Current main at c0ec58f4 still captures one memory-wiki plugin config at registration, accepts memory-wiki config only under the global plugin entry, and uses the captured vault path across wiki search/get/apply plus gateway and prompt/corpus surfaces. The related open PR #67584 is an implementation candidate, but it is not merged in the reviewed checkout.

## Best Possible Solution

Keep this issue open as the memory-wiki per-agent vault isolation tracker. Review and land #67584 or an equivalent memory-wiki-owned change that resolves the vault root per invocation context or provides explicit per-agent vault mapping, preserves the existing single-vault default, updates schema/docs, and adds regression tests covering `wiki_search`, `wiki_get`, `wiki_apply`, gateway/CLI management paths, prompt supplements, and corpus supplements across two agents on one gateway.

## Risks / Open Questions

- Closing now would remove the active tracker while current main still permits cross-agent wiki visibility when multiple agents share one memory-wiki vault on a single gateway.
- A partial fix limited to `wiki_search`/`wiki_get` would still leave `wiki_apply`, gateway methods, CLI/status/lint/compile/ingest, prompt supplements, and corpus supplements on the captured vault path.
- The related PR should receive normal maintainer review before this issue is closed, including tests that prove two agents on one gateway cannot read or write each other's wiki vaults.
