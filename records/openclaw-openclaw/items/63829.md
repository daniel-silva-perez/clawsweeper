## Summary

Keep this open. Current main still does not provide per-agent memory-wiki vault isolation: memory-wiki resolves one top-level plugin config at registration, the vault schema only has the existing global modes, agent config does not accept per-agent plugin entries, and both wiki read and write paths continue to operate on the captured vault path. The discussion includes a concrete multi-agent reproduction and #67584 is an open implementation candidate, but it has not landed on main.

## Best Possible Solution

Keep this issue open as the memory-wiki vault isolation tracker. Review and land #67584 or an equivalent memory-wiki-owned change that resolves the vault root per invocation context or adds explicit per-agent vault mapping, preserves the current single-vault default, updates schema/docs, and adds regression coverage proving two agents on one gateway cannot read or write each other's vaults through `wiki_search`, `wiki_get`, `wiki_apply`, and gateway/CLI management surfaces.
