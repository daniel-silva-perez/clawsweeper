## Summary

Keep this issue open. Current main still rejects unknown Feishu per-account config keys through the generated channel schema, and the doctor unknown-key repair path still does not remove plugin-owned AJV channel-schema additional-property failures. The issue is recent and plausible, but it still needs the offending extra key or a sanitized config snippet before maintainers can choose a safe migration or schema expansion.

## Best Possible Solution

Keep this issue open and ask for a sanitized `channels.feishu.accounts.main` snippet or logs showing the extra property. Then fix it in the Feishu channel/plugin-owned config path: add a targeted doctor migration/repair for a known legacy key, or intentionally support the field in `extensions/feishu/src/config-schema.ts`, regenerate bundled channel metadata, update docs, and add focused validation/doctor coverage.
