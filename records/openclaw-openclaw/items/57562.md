## Summary

Keep this issue open. Current main still validates Feishu channel account config with a strict account schema, and the existing doctor unknown-key repair only strips core OpenClaw Zod unknown keys, not plugin-owned `channels.feishu.accounts.<id>` schema failures. The report is recent and plausible, but it lacks the exact extra property/config snippet needed for a safe migration or schema change.

## Best Possible Solution

Keep this issue open and ask for a sanitized `channels.feishu.accounts.main` config snippet or logs showing the extra property. Then fix it in the Feishu channel/plugin-owned config path: either add a targeted doctor migration/repair for a known legacy key, or intentionally support the account-level field in `extensions/feishu/src/config-schema.ts`, generated channel metadata, docs, and focused tests.
