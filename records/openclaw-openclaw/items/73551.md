## Summary

Keep open. Current main still has the reported write-before-validate behavior for `openclaw plugins enable <id>` and `plugins disable <id>`, and the related open PRs #73554 and #73623 are implementation candidates that reference this issue with closing syntax. The issue should stay open until one canonical fix is merged or maintainers choose a different direction.

## Best Possible Solution

Review and land one canonical fix from #73554 or #73623. The fix should resolve the requested id against the discovered plugin snapshot before `enablePluginInConfig` or `setPluginEnabledInConfig`, exit non-zero with a clear `Plugin not found` message on a miss, avoid config writes and registry refreshes on failure, and add regression tests for both enable and disable. Cleanup of stale entries already written by older versions can remain a separate doctor/fix concern.
