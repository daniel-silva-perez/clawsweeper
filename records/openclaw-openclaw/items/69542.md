## Summary

Keep open. Current main does not implement ACP backend failover or an `acp.fallbacks` config key, and this PR is the active implementation attempt. It should not be closed as implemented, but it also is not ready to merge: the proposed branch still has unresolved P1/P2 review findings in the ACP turn loop, links the unrelated #4698 reasoning-default issue, and mixes the ACP feature with unrelated install-smoke/runtime-deps changes for #70756.

## Best Possible Solution

Keep this PR open but request changes: split the unrelated install-smoke/runtime-deps work into its own PR or drop it here, correct the wrong closing reference, and rework ACP failover around resolved session metadata. The ACP change should close old backend handles before switching, avoid failover once output/tool effects have started, define whether init-stage `ACP_SESSION_INIT_FAILED` unavailable errors are failover-worthy, use precise retry classification, and add focused manager tests for primary failure, fallback success, all-backends failure, observability, metadata-backed sessions, and no-output-after-partial-stream cases.
