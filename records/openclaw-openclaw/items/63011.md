## Summary

Current main still does not provide one canonical outbound send path that is hook-bearing, deliverable, and auditable. The gateway path still turns hook-cancelled empty delivery into `No delivery result`, delivery cancellation still produces no structured blocked outcome, `message_sending` still lacks the requested agent/request identity, and direct CLI sends can still be handled by plugin action dispatch before the shared core delivery path. The same author also has open draft PR #63007 targeting these gaps, so the issue should remain open until that PR is reviewed, landed, or explicitly superseded.

## Best Possible Solution

Keep this issue open. Review and either land #63007 or replace it with a maintainer-approved canonical outbound send contract that threads agent/session/request identity into `message_sending`, returns real provider delivery results on success, returns explicit guarded-denial results on hook cancellation, and aligns direct CLI, gateway send, and plugin action behavior or documents the intentional canonical path with regression coverage.
