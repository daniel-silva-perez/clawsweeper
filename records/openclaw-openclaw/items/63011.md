## Summary

Keep open. Current main still leaves the requested outbound send contract unresolved: gateway `send` still converts hook-cancelled empty delivery into `No delivery result`, `message_sending` still does not receive the requested outbound identity, cancellation still returns no structured blocked/auditable outcome, and the same-author draft PR #63007 remains the focused implementation candidate.

## What This Changes

Define one canonical outbound `message.send` path that runs through `message_sending`, returns provider delivery on allow, reports explicit guard denial on block, and carries session/request identity for audit correlation.

## Best Possible Solution

Review and either land #63007 or replace it with a maintainer-approved canonical outbound-send contract that threads agent/session/request identity into `message_sending`, returns explicit guarded-denial outcomes on hook cancellation, preserves real provider delivery results, and aligns or documents direct CLI, gateway `send`, and plugin-action paths.
