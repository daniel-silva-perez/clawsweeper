## Summary

Keep open. Current main still leaves the requested canonical outbound-send contract unresolved: gateway `send` still collapses hook-cancelled empty delivery into `No delivery result`, delivery-layer cancellation still returns no structured blocked/auditable outcome, the `message_sending` call does not receive the requested outbound identity, and the same-author draft PR #63007 remains open as the focused implementation candidate.

## What This Changes

The issue requests one canonical outbound `message.send` path that runs through `message_sending`, returns real delivery results, surfaces guarded denial, and preserves agent/session/request identity for audit correlation.

## Best Possible Solution

Review and either land #63007 or replace it with a maintainer-approved canonical outbound-send contract that threads outbound identity into `message_sending`, returns explicit guarded-denial outcomes, preserves real provider delivery results, and aligns or documents direct CLI, gateway `send`, and plugin-action behavior.
