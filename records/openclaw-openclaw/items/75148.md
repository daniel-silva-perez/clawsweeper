## Summary

Keep open. The PR addresses a real current-main gap in the message-tool fallback path and adds useful group/sandbox coverage, but the sandbox availability check should use the same runtime policy session key as the embedded runner before this is safe to merge.

## What This Changes

The PR updates dispatch reply tool-availability checks to include group-scoped, sandbox, and subagent tool policies, and adds regression tests for group and sandbox policies that remove `message`.

## Best Possible Solution

Update the PR so dispatch determines `messageToolAvailable` from the same effective policy inputs used by the embedded runner, including the runtime policy session key for sandbox/subagent checks, then keep the focused group/sandbox regression tests and add coverage for the missed runtime-policy-key shape.

## Reproduction Assessment

Yes. The current-main code trace shows `messageToolAvailable` excludes group/sandbox/subagent policies, and the PR patch can be checked directly against the runner path that uses `runtimePolicySessionKey` for sandbox resolution.

## Solution Assessment

No as written. Reusing the existing group, sandbox, and subagent policy resolvers is the right direction, but the sandbox portion should be keyed to the runner's runtime policy session key or extracted into a shared effective-availability helper.
