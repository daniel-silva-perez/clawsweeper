## Summary

Keep open. Current main still has the Discord `thread-reply` attachment alias gap: the handler reads only `media`, while Discord `send` accepts `media`, `path`, and `filePath`; the linked implementation PR #43574 remains open and unmerged, so this issue should stay open until that PR is reviewed, landed, replaced, or closed.

## What This Changes

The issue requests that Discord `thread-reply` honor `path` and `filePath` attachment parameters the same way Discord `send` already does.

## Best Possible Solution

Review and land #43574 or apply an equivalent Discord-owned fix that normalizes `thread-reply` attachments with `media` first, then `path`, then `filePath`, and adds regression coverage for both aliases and precedence.
