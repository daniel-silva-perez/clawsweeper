## Summary

Keep open. Current main still has the Discord `thread-reply` attachment alias gap: the handler reads only `media`, while Discord `send` and the docs support `media`, `path`, and `filePath`; the focused fixing PR #43574 remains open and unmerged, so this issue should stay open for that review/landing path.

## What This Changes

Discord `thread-reply` should honor `path` and `filePath` attachment parameters with the same precedence as `send`.

## Best Possible Solution

Land #43574 or an equivalent Discord-owned patch that resolves `thread-reply` attachments as `media` first, then `path`, then `filePath`, and adds focused regression coverage for alias handling and precedence.

## Reproduction Assessment

Yes. A high-confidence source trace reproduces the issue: `thread-reply` reaches the Discord plugin handler, shared hydration does not rewrite its aliases, and the handler only reads `media`.

## Solution Assessment

Yes. The narrowest maintainable fix is in the Discord `thread-reply` handler because the runtime already sends an attachment when `mediaUrl` is populated.
