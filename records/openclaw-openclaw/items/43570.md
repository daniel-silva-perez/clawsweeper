## Summary

Current main still has the Discord `thread-reply` attachment alias gap: the handler only maps `media`, while `send` supports `media`, `path`, and `filePath`. The linked open PR with closing syntax is the right implementation candidate, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Discord `thread-reply` should honor `filePath` and `path` attachment parameters with the same `media` precedence used by Discord `send`.

## Best Possible Solution

Review and land #43574, or apply an equivalent Discord-owned patch that maps `thread-reply` attachments with `media` precedence followed by `path` and `filePath`, plus regression tests for both aliases and precedence.
