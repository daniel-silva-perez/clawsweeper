## Summary

Keep open. Current main still has the reported Discord `thread-reply` attachment alias gap: the Discord handler reads only `media`, while `send` accepts `media`, `path`, and `filePath`; the runtime would send the attachment if `mediaUrl` were populated. The linked implementation PR #43574 remains open and unmerged, so the issue should stay open until that PR is reviewed, landed, replaced, or closed.

## What This Changes

The issue requests that Discord `thread-reply` accept `filePath` and `path` attachment aliases with the same precedence as Discord `send`.

## Best Possible Solution

The Discord plugin should normalize `thread-reply` attachments locally to `mediaUrl` using `media`, then `path`, then `filePath`, preserve the existing runtime media access forwarding, and add regression tests for `filePath`, `path`, and precedence when `media` is also present.
