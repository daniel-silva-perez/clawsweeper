## Summary

Keep open. Current main still has the Discord thread-reply attachment alias gap: `thread-reply` maps only `media`, while Discord `send` already accepts `media`, `path`, and `filePath`. The open fixing PR #43574 still references this issue and remains unmerged, so this issue should stay open until that PR or an equivalent Discord-owned patch lands.

## What This Changes

Discord `message` tool `thread-reply` should accept `path` and `filePath` attachment aliases with the same precedence as Discord `send`.

## Best Possible Solution

Review and land #43574, or refresh an equivalent Discord-owned patch, so `thread-reply` maps attachments with `media` precedence followed by `path` and `filePath`, forwards the needed media options, and has regression coverage for aliases and precedence.

## Reproduction Assessment

Yes. A high-confidence source trace reproduces the issue: the message tool dispatches `thread-reply` through the plugin path, shared hydration does not rewrite `path` or `filePath`, and the Discord handler reads only `media`.

## Solution Assessment

Yes. The requested handler alias chain is the narrowest maintainable fix because the runtime already sends an attachment when `mediaUrl` is populated; the patch should stay in the Discord plugin with focused regression tests.
