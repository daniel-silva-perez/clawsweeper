## Summary

Keep open. Current main still has the Discord `thread-reply` attachment alias gap: the handler reads only `media`, while Discord `send` accepts `media`, `path`, and `filePath`. The linked fix PR #43574 remains open and uses closing syntax, so this issue should stay open until that PR is reviewed, landed, replaced, or closed.

## What This Changes

The issue reports that Discord `thread-reply` ignores `path` and `filePath` attachment parameters even though Discord `send` supports them.

## Best Possible Solution

Review and land #43574, or apply an equivalent Discord-owned fix that normalizes `thread-reply` attachments with `media` first, then `path`, then `filePath`, and adds regression coverage for the aliases and precedence.
