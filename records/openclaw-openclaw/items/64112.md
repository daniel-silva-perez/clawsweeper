## Summary

Keep this PR open. Current main still has the false plugin allowlist diagnostic path for arbitrary missing primary commands, and this PR remains a plausible implementation candidate, though it needs adaptation to the current `run-main-policy.ts` split and consolidation with overlapping CLI diagnostic PRs.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best fix on current main is to update `src/cli/run-main-policy.ts` so `plugins.allow` and `plugins.entries.<id>.enabled=false` diagnostics are emitted only for real bundled plugin CLI roots or known command aliases, preserve parent-plugin guidance for aliases such as `voicecall` or `ltm`, return `null` for unknown roots such as `run`, and add focused regression coverage. Consolidate that final behavior with #63336 and #65111.
