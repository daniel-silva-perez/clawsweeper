## Summary

Keep this PR open. Current main still computes both preflight compaction and memory-flush token gates from `reserveTokensFloor`, while the config/Pi settings contract treats configured `reserveTokens` as the effective reserve with the floor as a minimum. The live PR is a focused replacement for closed, unmerged prior attempts #69379 and #67297, and the paired bug #66830 remains open.

## What This Changes

This PR adds an effective Pi compaction reserve resolver, updates preflight compaction and memory-flush gates to use the effective reserve instead of the floor-only value, renames the gate parameter, adds focused regression tests, and records the fix in the changelog.

## Best Possible Solution

Keep this PR open and review/land it, or land an equivalent narrow core fix. The fix should resolve one effective compaction reserve from configured `reserveTokens`, `reserveTokensFloor`, and the small-context cap, then use that value consistently in preflight compaction and memory-flush gates with focused regression tests and preserved contributor credit for #69379/#67297.
