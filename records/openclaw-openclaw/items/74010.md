## Summary

Keep this PR open. Current main still computes the preflight compaction and memory-flush gates from `reserveTokensFloor`, while the config/Pi settings contract treats configured `reserveTokens` as the effective reserve with the floor as a minimum. The live PR is a focused replacement for closed, unmerged prior attempts #69379 and #67297, and should proceed through normal maintainer review and CI rather than conservative cleanup closure.

Keep this PR open and review/land it, or land an equivalent narrow core fix. The fix should resolve one effective compaction reserve from configured `reserveTokens`, `reserveTokensFloor`, and the small-context cap, then use that value consistently in preflight compaction and memory-flush gates with focused regression tests and preserved contributor credit for #69379/#67297.
