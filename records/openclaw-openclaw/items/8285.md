## Summary

Keep open. Current main still does not provide a supported cross-channel text acknowledgement before agent processing. The existing surface covers ack reactions, status reactions, typing indicators, terminal before_dispatch handling, and final reply dispatch, but not a messages.ackText-style config or send-and-continue hook for Slack, WhatsApp, WebChat, Feishu, or other channels.

## Best Possible Solution

Keep this issue open as a core/product or plugin-API feature request. The best implementation path is either a supported messages.ackText configuration with scope and per-channel/account overrides, or a documented non-terminal pre-processing hook that can send interim text and then continue normal model dispatch. It should update config types, zod schema, generated schema/help, public docs, channel docs, and regression tests for the affected messaging surfaces.
