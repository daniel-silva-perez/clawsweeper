## Summary

Keep this PR open. The contributor is addressing a real current-main behavior: skills onboarding filters installers only when OS metadata exists, and several bundled skill manifests still expose unscoped Homebrew installers that remain eligible on Windows. The PR is not ready as-is because it is stale against current main and includes an installer-kind change for xurl that needs explicit review.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase or replace it on current main, audit every current bundled Homebrew install spec, add per-installer os metadata or top-level metadata.openclaw.os where appropriate, preserve Windows-capable installers where they exist, explicitly review the xurl node installer change, and add a small manifest audit test for unscoped brew-only installers.
