## Summary

Keep this PR open. Current main still has the Windows onboarding behavior the PR is trying to address: installer filtering only removes specs with OS metadata, and several bundled skills still have unscoped Homebrew install specs that remain eligible on Windows. The PR is useful but needs a refresh against current main and an intentional review of the xurl installer-kind change.

## Best Possible Solution

Keep this PR open for maintainer follow-up. The best path is to rebase or replace it on current main, audit every current bundled Homebrew install spec, add per-installer `os` metadata or top-level skill OS metadata where appropriate, preserve Windows-capable installers such as node/go/download where they exist, explicitly review or split the xurl node installer change, and add a small manifest audit test for unscoped brew-only installers.
