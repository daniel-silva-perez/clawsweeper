## Summary

Keep this PR open. Current main still has the Windows onboarding problem the PR targets: install filtering only hides installers that already carry `os` metadata, and several bundled skills still declare unscoped Homebrew install specs. The PR remains directionally useful, but it is stale against the current skill inventory and includes an `xurl` installer-kind change that should be reviewed deliberately rather than swept in as metadata-only cleanup.

## Best Possible Solution

Keep this PR open for maintainer follow-up or replacement. The best path is to refresh against current main, audit every bundled Homebrew install spec, add top-level or per-installer OS metadata where appropriate, preserve or add Windows-capable installers when available, split or explicitly review the `xurl` node installer change, and add a small manifest audit test for unscoped Homebrew-only installers.

## Risks / Open Questions

- There is no visible manifest audit test that would prevent future unscoped Homebrew-only install specs from reappearing.
