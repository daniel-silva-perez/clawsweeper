## Summary

Keep this PR open. Current main still has the Windows onboarding problem the PR targets: skill installer filtering only hides installer specs that carry OS metadata, and current bundled skills still include unscoped Homebrew installer specs. The PR is also stale against newer skill inventory and includes an `xurl` installer-kind change that deserves deliberate maintainer review rather than being treated as metadata-only cleanup.

## Best Possible Solution

Keep this PR open for maintainer follow-up or replacement. The best path is to refresh against current main, audit all bundled Homebrew installer specs, add top-level or per-installer OS metadata where appropriate, preserve or add Windows-capable installers when available, explicitly review the `xurl` node installer change, and add a small manifest audit test for unscoped Homebrew-only installers.
