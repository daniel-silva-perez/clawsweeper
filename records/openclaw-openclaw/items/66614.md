## Summary

Keep this issue open. Current main at 17811480dae20f65b526454d75e971ae8424ec10 still has fence-aware block chunking only, with no markdown table span parser or table-safe break guard in EmbeddedBlockChunker. The related implementation PR #66344 is still open and explicitly closes this issue, so it is a candidate fix rather than shipped behavior.

## Best Possible Solution

Keep this issue open and review or land #66344, or land a smaller equivalent fix. The fix should add markdown table span detection in the shared markdown/chunker layer, guard EmbeddedBlockChunker paragraph/newline/sentence/whitespace/max-length break paths with table-safe boundaries as well as fence-safe boundaries, and add focused regression coverage for text + table + text, partial streamed tables, fenced tables, and oversized-table fallback. #66605 remains a useful integration-level follow-up.
