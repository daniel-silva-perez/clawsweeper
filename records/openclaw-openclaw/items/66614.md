## Summary

Keep open. Current main still has fence-only block-streaming safe-break logic, no markdown table span parser or table-safe break guard, and the related implementation PRs are still open rather than shipped.

## What This Changes

Block streaming should keep Markdown tables intact instead of splitting one assistant reply into multiple channel messages around table paragraph boundaries.

## Best Possible Solution

Land one focused fix that adds markdown-table span detection in the shared markdown/chunker layer, guards EmbeddedBlockChunker paragraph/newline/sentence/whitespace/max-length break paths alongside fence protection, preserves existing fence behavior, and adds unit plus subscribe-level regression coverage for text + table + text, partial streamed tables, fenced tables, and oversized-table fallback.
