## Summary

Keep this issue open. Current main at 5828dcdb05aad587bff831467e7fa51f73312ff8 still implements block-streaming safe breaks only for fenced code blocks, with no markdown table span parser or table-safe break guard. The related implementation PR #66344 is open and unmerged, so it is a candidate fix rather than shipped behavior.

## Best Possible Solution

Keep this issue open and review/land #66344, or land a smaller equivalent fix. The implementation should add markdown table span detection in the shared markdown/chunker layer, guard EmbeddedBlockChunker paragraph/newline/sentence/whitespace/max-length break paths with table-safe boundaries as well as fence-safe boundaries, and add focused regression coverage for text + table + text, partial streamed tables, fenced tables, and oversized-table fallback. #66605 is a useful follow-up for integration-level coverage.
