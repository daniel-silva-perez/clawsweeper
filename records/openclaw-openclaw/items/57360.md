## Summary

Keep this issue open. Current main still has pi-ai Bedrock `requestMetadata` support available underneath OpenClaw, but OpenClaw does not expose a config key, validate it, document it, or inject it into Bedrock Converse calls. Related #60602 overlaps on cost attribution, but it is still open and focuses on automatic per-agent injection rather than replacing this static provider/plugin metadata request.

## Best Possible Solution

Keep this open and implement it in the Amazon Bedrock plugin as a plugin-owned config surface, likely `plugins.entries.amazon-bedrock.config.requestMetadata`, with AWS limit validation, UI/schema metadata, docs on the public Bedrock provider page, and focused tests proving the configured map is injected into Bedrock Converse options or payloads. Maintainers should decide whether #60602 becomes the canonical follow-up for dynamic per-agent templating.
