## Summary

Keep this issue open. Current main still does not expose or inject Bedrock `requestMetadata` from OpenClaw config: the Amazon Bedrock plugin schema/runtime only model discovery and guardrails, generic agent/model `params` are filtered to known stream options, and the Bedrock docs do not document cost-allocation metadata. Related #60602 remains useful context, but it is open and focused on automatic per-agent attribution rather than fully replacing this provider/static metadata request.

## Best Possible Solution

Implement this in the Amazon Bedrock plugin as a plugin-owned config surface, likely `plugins.entries.amazon-bedrock.config.requestMetadata`, with validation for AWS limits, schema/UI metadata, docs on the public Bedrock provider page, and focused tests proving the configured map is injected into Bedrock Converse options or payloads. Maintainers should also decide whether #60602 becomes the canonical follow-up for dynamic per-agent templating.
