## Summary

Keep this issue open. Current main still does not expose or inject Bedrock `requestMetadata` from OpenClaw config. The Bedrock plugin config surface only covers discovery and guardrails, and the generic agent/model `params` path is filtered to known stream options before calling `pi-ai`, so adding `requestMetadata` to `openclaw.json` would not reach Bedrock today. Related #60602 is useful context for per-agent cost attribution, but it is still open and does not fully replace this provider/static metadata request without maintainer triage.

## Best Possible Solution

Implement this in the Amazon Bedrock plugin as a plugin-owned config surface, likely under `plugins.entries.amazon-bedrock.config.requestMetadata`, with optional per-agent or templated values if maintainers want to merge it with #60602. Add schema/UI metadata, docs on the public Bedrock provider page, focused tests proving the configured map is validated and injected into Bedrock Converse payloads/options, and a maintainer decision on whether #60602 becomes the canonical per-agent follow-up.
