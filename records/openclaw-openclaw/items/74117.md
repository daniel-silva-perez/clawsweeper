## Summary

Keep this issue open. Current main still calls `ensureRuntimePluginsLoaded` on inbound dispatch and that helper still unconditionally calls `resolveRuntimePluginRegistry` with explicit load options, so the reported cache-miss path has not been fixed. The provided GitHub context also shows #74118 is an open, unmerged PR paired as the fix, so the cleanup path is to review or land that PR rather than close this issue early.

## Best Possible Solution

Keep this issue open and review #74118. If maintainers accept the narrow fix, land the `getActivePluginRegistry()` short-circuit in `src/agents/runtime-plugins.ts` with focused regression coverage in `src/agents/runtime-plugins.test.ts`; if they prefer a broader compatibility fix, keep this issue as the tracker until the loader-side change lands.
