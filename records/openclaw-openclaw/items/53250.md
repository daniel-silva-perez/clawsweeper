## What This Changes

The issue requests that exec approval timeout or unavailable messages tell operators where to approve, how to open the Control UI, and how to inspect or enable approval routes.

## Best Possible Solution

Update the shared exec approval unavailable reply so no-route, disabled, and unsupported paths give concrete recovery steps: open the dashboard via `openclaw dashboard --no-open`, inspect effective policy with `openclaw approvals get` or its alias, and enable the appropriate channel `execApprovals.enabled` path only when that channel actually supports native approvals.
