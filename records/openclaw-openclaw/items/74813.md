## Summary

Keep open. The PR addresses the marker-only continuation gap, but the new assistant-message fallback is broader than the current heartbeat contract: a shared-session lightweight heartbeat can leave an assistant row without full bootstrap context, and the next normal `continuation-skip` turn would now skip workspace bootstrap injection.

## What This Changes

The PR changes `hasCompletedBootstrapTurn` to treat assistant messages without the full-bootstrap marker as completed continuation evidence and adds regression tests for assistant-only and post-compaction tails.

## Best Possible Solution

Keep the assistant-message compatibility path, but constrain it so heartbeat-only lightweight turns do not count as completed bootstrap evidence. A narrow fix would ignore recognized heartbeat user/assistant pairs while continuing to accept ordinary completed assistant turns and preserving the post-compaction guard.

## Reproduction Assessment

Yes. A source-level reproduction is a session tail with a heartbeat user prompt and `HEARTBEAT_OK` assistant response, no full-bootstrap marker, `contextInjection: "continuation-skip"`, and a following non-heartbeat run; the PR fallback would make `hasCompletedBootstrapTurn` return true and skip workspace bootstrap injection.

## Solution Assessment

No for the PR as written. The marker fallback is the right area to fix, but treating every assistant row as safe continuation evidence is broader than the heartbeat/lightweight bootstrap contract; the safer alternative is to distinguish non-heartbeat assistant turns or keep scanning past heartbeat-only pairs.
