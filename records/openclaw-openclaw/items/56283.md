## Summary

Keep this PR open. Current main still lacks the proposed `markdown.strip` config contract and shared outbound markdown-stripping stage, while related open items only cover narrower pieces and the PR discussion identifies real correctness issues that need rework before merge.

## Best Possible Solution

Keep the PR open for maintainer judgment or contributor rework. A good revision should add a validated `markdown.strip` contract with account-scoped resolution, use channel/plugin capability metadata without hard-coded core-only assumptions, avoid stripping channels with native or lightweight markdown rendering, preserve identifiers and URLs, and add focused delivery tests for iMessage/SMS/IRC plus WhatsApp and markdown-capable plugin-channel non-regressions. The narrower fenced-code work can continue separately in #58948, and the iMessage-specific report remains tracked by #24943.
