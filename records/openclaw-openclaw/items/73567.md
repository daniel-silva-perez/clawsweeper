## Summary

Keep this PR open. Current main still uses the QQBot startup config snapshot for inbound routing and outbound dispatch, while the linked bug #69546 describes dynamic routing bindings not taking effect until gateway restart. The PR is a focused implementation candidate for that remaining bug, not obsolete cleanup.

## Best Possible Solution

Review this PR as the implementation candidate for #69546. The likely product/code path is to land the QQBot per-inbound runtime-config lookup with focused regression coverage, after removing the unrelated UI import churn and confirming the targeted QQBot tests/changed gate pass.
