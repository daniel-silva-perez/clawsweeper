## Summary

Keep open. Current main still limits managed Gateway service adapters to macOS, Linux, and Windows, and unsupported platforms still throw from resolveGatewayService(). Both `openclaw status` and `openclaw gateway status` reach that resolver before their later catch-based degraded status paths can run. The related PR #25690 is useful prior art, but it is closed and unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep this issue open and land a narrow Gateway service/status fix: either add a conservative unsupported-platform `GatewayService` adapter or guard the read-only status paths so `openclaw status` and `openclaw gateway status` can render degraded diagnostics on OpenBSD while managed lifecycle operations still fail with actionable unsupported-platform errors. Add regression tests for unsupported-platform status behavior and update docs/changelog if the user-facing behavior changes.
