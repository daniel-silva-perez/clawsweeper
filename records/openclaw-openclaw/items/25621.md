## Summary

Keep open. Current main still throws for OpenBSD and every platform outside darwin/linux/win32 in `resolveGatewayService()`. Both `openclaw status` and `openclaw gateway status` still call that resolver before their degraded status paths can render, and the related implementation PR #25690 is closed and unmerged.

## Best Possible Solution

Land a narrow gateway service/status fix: either return a conservative unsupported-platform `GatewayService` adapter whose read-only methods report safe unknown/not-installed state while mutating methods reject clearly, or guard status-only call sites before resolving managed service adapters. Cover both `openclaw status` and `openclaw gateway status` on an unsupported `process.platform`, and update docs/changelog only if user-visible wording changes.

## Repair Work Prompt

Fix the unsupported-platform Gateway service status failure tracked by openclaw/openclaw#25621. Current `resolveGatewayService()` throws on platforms outside darwin/linux/win32, which prevents read-only `openclaw status` and `openclaw gateway status` diagnostics from rendering on OpenBSD and similar platforms. Implement a narrow fix either by returning a conservative unsupported-platform `GatewayService` adapter or by guarding status-only paths before resolving managed service adapters. Read-only status should render safe unknown/not-installed/degraded service information; mutating lifecycle operations (`install`, `uninstall`, `stop`, `restart`, service rewrite/stage`) must still reject with a clear unsupported-platform message and must not imply OpenBSD managed service support. Use #25690 as prior art, but do not import unrelated CI/infra changes. Add regression tests for unsupported `process.platform` behavior covering both the resolver/service state and the status call sites. Update user-facing docs/changelog only if visible status output or lifecycle error wording changes; any changelog entry needs normal OpenClaw attribution, not Codex/OpenClaw/steipete. Keep core changes generic and platform-neutral.
