## Summary

Keep open. Current main still supports managed Gateway service adapters only for macOS, Linux, and Windows, and unsupported platforms still throw from resolveGatewayService(). Both openclaw status and openclaw gateway status still resolve that adapter before their later catch-based degraded paths can run. The closed, unmerged PR #25690 remains useful prior art but does not supersede the issue.

## Best Possible Solution

Keep this issue open until OpenClaw adds a graceful unsupported-platform GatewayService adapter or guards the read-only status paths so openclaw status and openclaw gateway status work on OpenBSD and similar platforms. Reuse the shape from #25690 as prior art, but keep managed lifecycle operations honest with actionable unsupported-platform errors and add current-main regression coverage.
