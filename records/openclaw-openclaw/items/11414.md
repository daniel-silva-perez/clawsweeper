## Summary

Keep this issue open. Current main still implements only Telegram forum topic creation and editing through the message plugin; close, reopen, and delete topic lifecycle actions are missing from the shared action list, Telegram action map/discovery, runtime dispatch, transport exports, config schema, docs, and focused tests. Open PR #41410 is relevant deletion-only prior art, but it is unmerged and does not cover the full requested lifecycle.

## What This Changes

The issue requests Telegram forum topic lifecycle support in the message plugin for creating, editing, closing, reopening, and deleting topics.

## Best Possible Solution

Keep this as the canonical Telegram forum-topic lifecycle tracker and land one coherent Telegram plugin implementation that completes close, reopen, and delete alongside the existing create/edit path across shared action names, Telegram discovery/map, runtime dispatch, transport helpers, config schema/metadata, docs, and focused contract/runtime tests.
