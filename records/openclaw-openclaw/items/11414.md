## Summary

Keep open. Current main still exposes Telegram forum topic creation and editing only; close, reopen, and delete topic lifecycle actions are absent from the shared message action contract, Telegram discovery/map, runtime dispatch, transport exports, config schema, docs, and focused tests. The related PR #41410 is useful deletion-only prior art, but it is open and does not cover the full requested lifecycle.

## What This Changes

The issue requests Telegram forum topic management through the message tool: create, edit, close, reopen, and delete forum topics.

## Best Possible Solution

Keep this as the canonical Telegram forum-topic lifecycle tracker and land one coherent Telegram plugin update that adds close, reopen, and delete alongside the existing create/edit path across shared action names, Telegram discovery/map, runtime dispatch, transport helpers, config schema, docs, and focused contract/runtime tests.
