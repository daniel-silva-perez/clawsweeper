## Summary

Keep open. Current main still implements only the create/edit part of Telegram forum topic management, while close, reopen, and delete are missing from the message action names, Telegram action mapping/discovery, runtime dispatch, transport exports, config schema, docs, and focused tests. Related PR #41410 is relevant delete-only prior art, but it is still open and does not cover the full lifecycle.

## What This Changes

The issue requests Telegram forum topic create/edit/close/reopen/delete support through the message plugin.

## Best Possible Solution

Keep this as the canonical Telegram forum-topic lifecycle tracker and land one coherent Telegram plugin update that adds close, reopen, and delete alongside the existing create/edit path across shared action names, Telegram discovery and mapping, runtime dispatch, transport helpers, config schema/generated metadata, docs, and focused contract/runtime tests.

## Reproduction Assessment

Yes. Static current-main verification is enough: topic-close/topic-reopen/topic-delete are not registered or advertised, closeForumTopic/reopenForumTopic/deleteForumTopic are not runtime aliases or handlers, and the existing delete path requires messageId for message deletion.

## Solution Assessment

No. Current main is only a partial solution for create/edit, and the open deletion-only PR is not the full lifecycle fix; the maintainable path is a coordinated Telegram plugin update for the remaining lifecycle actions.
