## Summary

Keep this issue open. Current main still implements only Telegram forum topic creation and editing through the message plugin; close, reopen, and delete are missing from the shared action contract, Telegram discovery, runtime dispatch, transport exports, config schema, docs, and focused contract coverage. Open PR #41410 is useful deletion-only prior art, but it remains open/unmerged and does not cover the full requested lifecycle.

## What This Changes

Add Telegram forum topic lifecycle management to the message plugin, covering create, edit, close, reopen, and delete actions.

## Best Possible Solution

Keep this as the canonical Telegram forum-topic lifecycle tracker and land one coherent implementation that adds close, reopen, and delete alongside the existing create/edit path across shared action names, Telegram action discovery, runtime dispatch, send/runtime-api helpers, config schema and generated metadata, docs, and focused tests, reusing #41410 only where it still fits current main.
