## Summary

Keep this issue open. Current main has Telegram forum topic create/edit support, but still does not expose close, reopen, or delete forum-topic lifecycle actions through the shared message action names, Telegram action discovery/map, runtime dispatch, transport exports, config schema, docs, or contract tests. PR #41410 is relevant deletion-only prior art, but it remains open/unmerged and does not cover close or reopen.

## Best Possible Solution

Keep this issue open as the canonical Telegram forum topic lifecycle tracker. The best path is to review PR #41410 as deletion-only prior art, then either expand it or land a complete follow-up that adds close, reopen, and delete forum topic support through the shared message action contract, Telegram action discovery/map, runtime dispatch, transport helpers using Telegram Bot API methods, config schema/metadata, public docs, and focused Telegram contract/runtime/transport tests.
