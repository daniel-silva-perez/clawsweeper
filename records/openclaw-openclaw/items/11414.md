Current main implements only the create/edit slice of Telegram forum topic management; close, reopen, and delete topic lifecycle support is still absent across the shared action contract, Telegram action mapping/discovery, runtime dispatch, transport exports, config schema, docs, and focused tests. Add Telegram Bot API forum topic lifecycle methods to the message plugin for create, edit, close, reopen, and delete topic actions.
