## Summary

Keep open. Current main still supports only Telegram forum topic creation and editing through the message plugin; close, reopen, and delete topic lifecycle support is absent across action names, Telegram discovery/mapping, runtime dispatch, transport exports, config schema, docs, and focused tests. Related PR #41410 is useful deletion-only prior art, but it remains open and does not cover the full requested lifecycle.

## What This Changes

The issue requests Telegram forum-topic lifecycle support in the message plugin, including create, edit, close, reopen, and delete actions.

## Best Possible Solution

Keep this as the canonical Telegram forum-topic lifecycle tracker and land one coherent Telegram plugin update that adds close, reopen, and delete alongside the existing create/edit path across shared action names, Telegram discovery and mapping, runtime dispatch, transport helpers, config schema, docs, and focused contract/runtime tests.

## Reproduction Assessment

Yes. Static current-main verification is enough: topic-close/topic-reopen/topic-delete are not registered action names, closeForumTopic/reopenForumTopic/deleteForumTopic are not runtime aliases or handlers, and the existing delete path requires messageId for message deletion.

## Solution Assessment

No. Current main only solves the create/edit portion, and the open deletion-only PR is a partial implementation candidate; the maintainable fix should cover the remaining lifecycle actions coherently in the Telegram plugin surface.
