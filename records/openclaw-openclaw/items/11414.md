## Summary

Keep this issue open. Current main supports Telegram forum topic creation and editing through the message plugin, but close, reopen, and delete topic lifecycle actions are still absent from the shared action names, Telegram action map/discovery, runtime dispatch, transport exports, config schema, docs, and focused tests; open PR #41410 is relevant deletion-only prior art but is unmerged and does not cover the full requested lifecycle.

## What This Changes

Expose Telegram forum topic create, edit, close, reopen, and delete management through the message plugin.

## Best Possible Solution

Keep this as the canonical Telegram forum-topic lifecycle tracker and land one coherent Telegram plugin implementation that completes close, reopen, and delete alongside the existing create/edit path across shared action names, Telegram discovery/map, runtime dispatch, transport helpers, config schema/metadata, docs, and focused contract/runtime tests.
