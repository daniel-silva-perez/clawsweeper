## Summary

Keep this open. Current main still exposes only Telegram forum topic creation and editing through the message plugin; close, reopen, and delete are missing from the shared action names, Telegram action map/discovery, runtime dispatch, transport exports, config schema, docs, and focused tests. Open PR #41410 is relevant deletion-only prior art, but it remains open and does not cover the full requested lifecycle.

## What This Changes

Add Telegram forum topic lifecycle management to the message plugin, including create, edit, close, reopen, and delete operations for Telegram supergroups with topics enabled.

## Best Possible Solution

Keep this as the canonical Telegram forum-topic lifecycle tracker and land one coherent Telegram plugin implementation that adds close, reopen, and delete alongside the existing create/edit path across shared action names, Telegram discovery, runtime dispatch, transport helpers, config schema, generated metadata/docs, and focused contract/runtime tests.
