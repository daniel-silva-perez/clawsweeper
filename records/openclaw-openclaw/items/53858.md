## Summary

Current main still matches the reported Nostr restart-loop bug. The bundled Nostr channel still passes an array-wrapped DM filter to `nostr-tools.subscribeMany`, while the pinned dependency expects a single `Filter` object and serializes the supplied value into the Nostr REQ frame. The gateway start path also still resolves immediately after startup, which fits the reported supervisor auto-restart loop. The related fix PR #42832 is open and unmerged, so this issue should remain open.

## Best Possible Solution

Keep this issue open as the current Nostr restart-loop/subscription tracker. Review and either refresh/land PR #42832 or prepare a current-main patch that passes a plain DM `Filter` object to `SimplePool.subscribeMany`, removes the unsafe cast, keeps the Nostr gateway account task pending until abort, closes subscription and pool resources during cleanup, and adds regression tests for the serialized REQ filter shape and gateway lifecycle.
