## Summary

Keep this open: current main still matches the reporter's later root-cause comments. The Nostr bus still passes an array-wrapped DM filter into `nostr-tools.subscribeMany`, the gateway still restarts channel tasks whose `startAccount` promise resolves, and the related implementation PR is open rather than merged.

## What This Changes

The issue reports that the Nostr channel connects to relays, sends a malformed array-wrapped DM subscription filter, immediately falls into the gateway auto-restart loop, and can leave relay WebSocket connections behind.

## Best Possible Solution

Keep this as the canonical Nostr restart-loop tracker until PR #42832 or an equivalent maintainer-approved fix lands. The durable fix should pass a plain DM `Filter` object to `SimplePool.subscribeMany`, keep the channel account task pending until abort/stop, close subscription and pool relay resources during cleanup, and add targeted regression tests for REQ filter shape plus channel lifecycle.
