## Summary

Current main still matches the reported Nostr restart-loop failure: the Nostr bus passes an array-wrapped DM filter into `nostr-tools.subscribeMany`, while the pinned dependency accepts a single `Filter` and serializes the value into the REQ frame. The gateway supervisor also still restarts a channel whose `startAccount` promise settles, and the related implementation PR #42832 is open rather than merged.

## What This Changes

The issue reports that the bundled Nostr channel connects to relays but immediately restarts because its DM subscription is sent in an invalid nested REQ filter shape and leaves relay connections behind.

## Best Possible Solution

Keep this issue open as the current Nostr restart-loop tracker. The fix should pass a plain DM `Filter` object to `SimplePool.subscribeMany`, keep the gateway account task pending until abort/stop, close both the subscription and pool relay resources on cleanup, and add focused regression tests for the serialized REQ filter shape and channel lifecycle.
