## Repair Work Prompt

Implement the logging enhancement requested by #32438. Keep the existing `logging.file` JSONL file and all CLI/Control UI/script consumers unchanged. Add a derived human-readable companion log path beside the active JSONL file by inserting `-formatted` before the extension, including for the currently active dated rolling file. Write one redacted plain-text line for each file-log event using `YYYY-MM-DD HH:mm:ss [level] [subsystem] message`. Do not add a config key unless an existing schema pattern makes it unavoidable. Preserve existing redaction before persistence, and make an explicit tested decision for date-boundary, rotation, pruning, and `logging.maxFileBytes` behavior; the issue requested the size cap only apply to the main JSONL file, so avoid silently changing JSONL rotation semantics. Update user-facing logging docs and configuration references, and add a CHANGELOG entry crediting @maweibin if the behavior is user-facing. Do not change unrelated log parsing, Control UI display, channel-specific behavior, or plugin APIs.
