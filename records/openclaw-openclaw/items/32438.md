## Summary

Keep open. Current main still persists Gateway file logs as a single redacted JSONL stream, and the CLI/Control UI/docs continue to parse or describe that JSONL file rather than writing the requested durable plain-text companion file. The earlier duplicate is closed, the same-author implementation PR is closed unmerged, and the merged log fallback/date-boundary work is adjacent but does not satisfy the requested sidecar.

## What This Changes

Request to add a plain-text `-formatted` companion log file derived from `logging.file` while preserving the existing JSONL log file for CLI, Control UI, and scripts.

## Best Possible Solution

Land a focused logger-layer enhancement that derives a readable companion path from the active resolved log file, writes redacted plain-text lines beside the unchanged JSONL stream, handles date rollover and rotation intentionally, and updates operator docs plus regression tests for redaction and unchanged JSONL consumers.

## Solution Assessment

Yes, the requested logger-layer sidecar is the narrowest maintainable direction. The existing CLI fallback and Control UI JSONL parsing are useful adjacent behavior, but they do not help operators who need a durable plain-text file while the Gateway is down.
