## Summary

Keep open. Current main still persists Gateway file logs as one JSONL stream, while the CLI and Control UI pretty-print that JSONL at read time; there is no durable `-formatted` companion file beside `logging.file`. The earlier duplicate is closed, the same-author implementation PR is closed unmerged, and the merged dated-log discovery work is adjacent rather than a substitute.

## What This Changes

The issue requests a derived plain-text formatted companion log file next to the existing JSONL `logging.file`, without changing current JSONL consumers.

## Best Possible Solution

Land a focused logger enhancement that derives a readable companion path from the active `logging.file`, writes redacted plain-text lines beside the unchanged JSONL stream, handles dated rolling files and rotation/pruning intentionally, and updates operator docs plus regression coverage for redaction and unchanged JSONL consumers.

## Reproduction Assessment

Yes. A source-level reproduction is clear: enable file logging and emit a log event; current `buildLogger` writes a single redacted JSONL payload to `activeFile`, and no durable formatted sidecar path exists. I did not run a live reproduction because this review is read-only and that would write log files.

## Solution Assessment

Yes. The narrow maintainable fix is in the logger/file-log layer, with docs and tests, while keeping `logs.tail`, CLI formatting, and Control UI JSONL parsing unchanged; the existing CLI fallback and dated-log lookup are helpful but do not solve the requested persistent sidecar.
