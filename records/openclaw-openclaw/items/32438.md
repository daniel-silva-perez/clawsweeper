## Summary

Keep open. Current `main` still writes only the configured JSONL Gateway log, the CLI and Control UI continue to consume that JSONL path, and the docs still describe file logs as JSON lines. The earlier duplicate was closed, the paired implementation PR was closed unmerged, and the recent channel-log rolling-file fix is adjacent but does not create the requested persistent formatted companion file.

## What This Changes

The issue requests a derived plain-text formatted log file beside the existing Gateway JSONL log so operators can inspect logs directly when the Control UI is unavailable.

## Best Possible Solution

Land a focused logger-transport enhancement that derives a human-readable companion path from the active `logging.file`, writes redacted formatted lines beside the unchanged JSONL file, documents how to find and interpret that file, and adds regression coverage for redaction, day-boundary/rotation behavior, and unchanged JSONL consumers.

Implement openclaw/openclaw#32438 as a focused logging enhancement. Current main writes only the configured JSONL Gateway file log; add a human-readable formatted companion file derived from the active JSONL log path by inserting `-formatted` before the extension, for example `openclaw.log` -> `openclaw-formatted.log` and dated rolling files analogously.
