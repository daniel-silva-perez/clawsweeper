## Summary

Keep this issue open. Current main still writes only JSONL gateway file logs, docs still describe file logs as JSON objects per line, and the CLI's local-file fallback only covers local loopback pairing/scope errors rather than the general gateway-down path described here. The earlier duplicate pointer #29378 is closed, and the paired implementation PR #32442 is also closed unmerged, so there is no canonical open or shipped fix to close against.

## Best Possible Solution

Keep the issue open and implement a focused logging enhancement: derive a human-readable sidecar path from the active JSONL log path, write formatted redacted lines alongside the unchanged JSONL log, and document the behavior. The implementation should preserve existing CLI/Control UI JSONL consumers and should align sidecar rotation/retention with current main's rolling log behavior to avoid unbounded disk growth.
