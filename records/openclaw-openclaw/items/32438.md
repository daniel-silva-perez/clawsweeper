## Summary

Current main still does not implement the requested formatted sidecar log file. The logger resolves and writes only the configured JSONL file, docs/UI/CLI still describe Gateway file logs as JSONL, and the related earlier issue is closed while the paired implementation PR was closed unmerged. Recent logging improvements are adjacent but do not create a persistent human-readable file beside `logging.file`.

## What This Changes

The issue requests a derived human-readable formatted log file beside the existing JSONL Gateway log file so operators can inspect logs when the Control UI is unavailable.

## Best Possible Solution

Keep this issue open and land a focused logging enhancement in the logger transport: derive a human-readable sidecar path from the active `logging.file`, write redacted formatted lines beside the unchanged JSONL file, keep day-change and rotation lifecycle bounded, document the path, and add regression coverage for redaction, rotation, and unchanged JSONL consumers.
