## Summary

Keep open. Current main still persists Gateway file logs as a single JSONL stream and the operator-facing UI/docs continue to treat that file as JSONL; the newer CLI local fallback helps when the Gateway RPC path is unavailable, but it still reads and pretty-prints the configured JSONL file rather than creating the requested persistent formatted companion file. The earlier duplicate was closed, the paired implementation PR was closed unmerged, and the merged channel-log date-boundary fix is adjacent but does not supersede this request.

## What This Changes

Add a derived human-readable formatted companion log file next to the existing JSONL Gateway log without changing JSONL consumers or adding a config key.

## Best Possible Solution

Land a focused logger enhancement that derives a human-readable companion path from the active `logging.file`, writes redacted plain-text lines beside the unchanged JSONL file, handles rolling date changes and rotation/pruning deliberately, and updates operator docs plus regression coverage for redaction and unchanged JSONL consumers.
