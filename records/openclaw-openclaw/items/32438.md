## Summary

Keep this issue open. Current main still writes only the configured JSONL gateway log file; `openclaw logs` can format JSONL for stdout, but there is no durable sibling `-formatted` log file. The earlier duplicate candidate #29378 is closed and PR #32442 was closed unmerged, so this remains the active tracker for the requested core logging behavior.

## Best Possible Solution

Implement or explicitly reject the durable formatted-file behavior. A complete implementation should derive a sibling path from `logging.file`, write redacted human-readable lines whenever the JSONL file is written, preserve existing JSONL consumers, define rotation/pruning for the sibling file, add focused logger/log-tail tests, and update https://docs.openclaw.ai/logging.

## Clownfish Work Prompt

Implement the durable formatted gateway log file requested by this issue. Current main writes only JSONL in `src/logging/logger.ts`; `openclaw logs` formatting in `src/cli/logs-cli.ts` is presentation-only. Add a sibling formatted file whenever file logging is enabled, deriving the path from the active JSONL log path by inserting `-formatted` before the extension. Preserve the existing JSONL file format and all `logs.tail`, Control UI, CLI, and script consumers. Write redacted plain-text lines in the form `YYYY-MM-DD HH:mm:ss [level] [subsystem] message` whenever the JSONL file is written. Reuse existing timestamp, parsing/flattening, and redaction behavior where practical; do not add a new config key. Define and test rotation/pruning behavior for default rolling files and `logging.maxFileBytes` so the formatted file does not grow unbounded or drift from the active log. Update public logging docs and add a single-line changelog entry under the active version `### Changes`, crediting `Thanks @maweibin`. Do not change unrelated logging semantics, provider/channel behavior, or the `logs.tail` wire contract.
