## Summary

Keep open. Current main still writes Gateway file logs as a single JSONL stream; the CLI and Control UI format that JSONL only when reading it, and the docs still describe file logs as JSONL. The earlier duplicate is closed, the same-author implementation PR is closed unmerged, and the merged active-log discovery work is adjacent but does not provide the requested persistent formatted companion file.

## What This Changes

Requests a derived plain-text companion log file beside the existing `logging.file` JSONL log so operators can inspect readable logs when the Gateway or Control UI is unavailable.

## Best Possible Solution

Land a focused logger enhancement that derives a readable companion path from the active `logging.file`, writes redacted plain-text lines beside the unchanged JSONL file, preserves existing CLI/Control UI/script consumers, handles rolling-date and rotation behavior deliberately, and updates operator docs plus regression coverage.
