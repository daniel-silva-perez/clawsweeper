## Summary

Keep open. Current main still implements `agents.defaults.subagents.runTimeoutSeconds` as a fallback only: explicit `sessions_spawn.runTimeoutSeconds` is accepted and forwarded without a configured minimum or lock, and the current tests/docs preserve that precedence. Related PRs #37973 and #38048 are useful prior art but are closed unmerged, so they do not resolve the issue.

## Best Possible Solution

Keep this issue open until OpenClaw implements a maintainer-approved subagent timeout constraint. A focused fix should add an explicit config contract such as `agents.defaults.subagents.minRunTimeoutSeconds` or a lock option, clamp model-supplied `sessions_spawn.runTimeoutSeconds` before subagent run registration, preserve existing zero/unset compatibility, and add regression coverage for omitted timeout fallback, explicit values above and below the floor, and schema acceptance/rejection. Closed PRs #37973 and #38048 are useful prior art but do not close the issue on current main.
