## Summary

Current main still lacks an enforced subagent timeout minimum or lock. `agents.defaults.subagents.runTimeoutSeconds` remains a fallback used only when `sessions_spawn.runTimeoutSeconds` is omitted, and current schema/docs/tests preserve that behavior. Related PRs #37973 and #38048 are useful prior art but are closed unmerged, so they do not resolve the issue.

## Best Possible Solution

Keep this issue open until OpenClaw implements a maintainer-approved subagent timeout constraint. A focused fix should add an optional contract such as `agents.defaults.subagents.minRunTimeoutSeconds` or an explicit lock option, clamp model-supplied `sessions_spawn.runTimeoutSeconds` before subagent run registration, preserve current zero/unset behavior, and add coverage for omitted timeout fallback plus explicit values above and below the configured constraint. Closed PRs #37973 and #38048 are useful prior art but do not close the issue on current main.
