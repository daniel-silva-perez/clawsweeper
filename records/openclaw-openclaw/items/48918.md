## Summary

Keep open. Current main still has a SKILL.md-only loading, prompt, and watcher contract, so there is no supported skill-scoped user preferences/conventions companion file yet. The issue discussion is actionable and specifically supports the markdown companion-file direction; the related #50199 priority-configuration issue is adjacent, not a replacement for this user-context request.

## What This Changes

Add a supported mechanism for user-level, skill-scoped preferences or conventions to be loaded only when the matching skill is used, without replacing SKILL.md or putting skill-specific notes into global context files.

## Best Possible Solution

Define a small core skills contract for a markdown companion file under existing skill roots, append it as lower-precedence local context only for the selected skill, and update discovery, prompt rendering, watcher refresh, sandbox/plugin materialization, docs, and focused tests so the behavior is consistent across runners.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence check is source and docs inspection on current main: a companion preferences/conventions file has no loader, prompt, or watcher path today.

## Solution Assessment

Unclear until maintainers approve the exact public contract. Option A, a plain markdown companion file, is the best-supported direction in the discussion, but the implementation should explicitly preserve SKILL.md authority and cover refresh/materialization behavior.
