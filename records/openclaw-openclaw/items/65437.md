## Summary

Keep this issue open. Current main still drops Slack room/group text control commands before the elevated allowlist path can evaluate `tools.elevated.allowFrom.slack`; the existing fallback seam is present in core but Slack does not register an elevated fallback adapter, and the docs still only document the Discord fallback. The related Discord and tiered-command-auth items are useful context, but they do not supersede this Slack-specific bug.

## Best Possible Solution

Fix the Slack room/group `/elevated` path narrowly. Either let `/elevated`-bearing Slack messages from `tools.elevated.allowFrom.slack` reach `resolveElevatedPermissions`, or register/add an elevated-specific authorizer/fallback for Slack without granting broader control-command access. Add a regression test for a Slack channel/group sender listed only in `tools.elevated.allowFrom.slack`, and update the public elevated/Slack docs to describe the resulting behavior.
