## Summary

Keep this issue open. Current main still drops Slack room/group control commands before `tools.elevated.allowFrom.slack` can be evaluated, and the related Discord/tiered-auth issues do not supersede this Slack-specific bug.

## Best Possible Solution

Fix the Slack room/group `/elevated` path narrowly. The safest product direction is to let `/elevated`-bearing Slack messages from `tools.elevated.allowFrom.slack` reach `resolveElevatedPermissions`, or add an elevated-specific authorizer to the inbound command gate, without widening authorization for unrelated control commands. Add a regression test for a Slack channel/group sender listed only in `tools.elevated.allowFrom.slack`, and update the public elevated/Slack docs to describe the resulting behavior.
