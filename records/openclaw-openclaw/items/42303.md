## Summary

Keep open. Current main still does not implement this PR's intended opt-in Linuxbrew Docker build/setup support, and the linked Docker skill-install failure path remains possible for brew-only skills. The PR is relevant, but it is outdated against the current Docker setup layout and has functional plus supply-chain blockers before it could be merged.

## Best Possible Solution

Keep this PR open for maintainer follow-up, but do not merge it as-is. Choose the canonical fix for #14593: either a safe opt-in Linuxbrew Docker build/setup flow in the current Dockerfile and scripts/docker/setup.sh, with conditional runtime environment handling, docs, regression coverage, and installer integrity policy, or a different shipped path such as hiding brew-only installs in Linux containers or providing non-brew Linux installers. After one path lands, close or supersede the remaining PRs against that shipped implementation.

## Risks / Open Questions

- Merging this PR as-is would add unpinned remote installer execution during Docker builds; maintainers need an explicit trust, pinning, or integrity policy for that path.
- The PR targets an older Docker setup shape and would need to be ported to scripts/docker/setup.sh plus the current multi-stage Dockerfile.
- The Dockerfile changes described in the PR discussion have concrete functional blockers: unconditional Homebrew environment pollution and fragile installer quoting.
- Closing now would discard a relevant implementation candidate while #14593 and alternative fix PRs remain open and unshipped.
