## Summary

Current main still does not implement this PR’s intended opt-in Linuxbrew installation support for Docker builds, and it does not ship an equivalent fix for brew-based skill installs in Linux containers. Keep this PR open for maintainer follow-up, but do not merge it as-is because the branch is stale against the current Docker setup layout and has unresolved functional and supply-chain review concerns.

## Best Possible Solution

Keep this PR open for maintainer follow-up, but do not merge it as-is. The best path is to pick the canonical fix for #14593: either safe opt-in Linuxbrew support in the current Dockerfile plus scripts/docker/setup.sh, with conditional runtime environment handling, docs at https://docs.openclaw.ai/install/docker, Docker proof, and an explicit installer integrity policy; or a different shipped product path such as apt fallback or hiding brew-only skill installs in Linux containers. Once that path lands on main, close or supersede the remaining implementation PRs against the shipped fix.
