## Summary

Keep this PR open. Current main still does not implement the PR's intended opt-in Linuxbrew support for Docker builds/setup, and the linked Docker skill-install bug remains unresolved by current main. The PR is relevant, but it is stale against the current Docker setup layout and has functional and supply-chain review blockers, so it should not be merged as-is.

## Best Possible Solution

Keep this PR open for maintainer follow-up, but do not merge it as-is. The best path is to pick the canonical fix for #14593: either safe opt-in Linuxbrew support in the current Dockerfile and scripts/docker/setup.sh with conditional runtime environment handling, docs, regression coverage, and an explicit installer integrity policy, or a different shipped product path such as hiding/falling back from brew-only skill installs in Linux containers. Once that path lands on main, close or supersede the remaining PRs against the shipped implementation.

## Risks / Open Questions

- Merging this PR as-is would add unpinned remote code execution from Homebrew's HEAD install script during Docker builds; maintainers should decide whether this is acceptable or require pinning/checksum/trusted-source controls.
- The Docker setup script has moved to scripts/docker/setup.sh and uses the shared Docker build helper; this PR edits the old root docker-setup.sh body and would need to be rebased onto the current setup layout.
- The PR's unconditional HOMEBREW_* ENV/PATH changes would affect non-brew images if kept in that form.
- Closing this PR now could lose a relevant implementation candidate while #14593 is still open and the alternative PRs #55940 and #57553 are still unmerged.
