## Summary

Keep this PR open. Current main still does not implement opt-in Linuxbrew installation for Docker builds or another shipped path that fixes brew-based skill installs in Linux containers, but the PR is stale against the current Docker setup layout and has functional and supply-chain blockers, so it should not merge as-is.

## Best Possible Solution

Keep this PR open for maintainer follow-up, but do not merge it as-is. The best path is to choose the canonical fix for #14593: either safe opt-in Linuxbrew support in the current Dockerfile plus scripts/docker/setup.sh, with conditional runtime environment handling, docs, regression/manual Docker proof, and an explicit installer integrity policy; or a different shipped product path such as hiding or falling back from brew-only skill installs in Linux containers. Once that path lands on main, close or supersede the remaining PRs against the shipped implementation.
