## Summary

Keep this PR open. Current main does not implement split model routing, but the proposed patch is not merge-ready because it only swaps the model object after the primary provider transport/auth/wrapper pipeline has already been built, and it routes only after a tool result exists. That leaves the advertised local-primary plus API-tool workflow broken and creates a provider/auth boundary risk in cross-provider configurations.

## What This Changes

The PR adds a `model.tool` config field, resolver/schema support, and Pi embedded runner stream-wrapper tests intended to route tool-continuation LLM calls to a configured alternate model.

## Best Possible Solution

Keep the PR open while maintainers decide whether split model routing belongs in the core `agents.defaults.model` contract or behind a plugin/API seam. If it stays core, the effective per-call model should drive the whole model-call pipeline: tool exposure, schema normalization, provider stream dispatch, auth profile/API-key selection, provider-specific wrappers, transcript policy, context budgeting, fallbacks, diagnostics, trajectory metadata, docs, generated config contracts, and integration coverage.
