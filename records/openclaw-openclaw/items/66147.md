## Summary

Keep this PR open. Current main still lacks split model routing, and the existing review discussion points to real design and correctness gaps in the branch rather than evidence that it is obsolete. The remaining work changes core agent model routing plus provider/auth boundaries, so it needs maintainer review or author revision.

## What This Changes

The PR adds a `model.tool` config value, resolver/schema support, and a Pi embedded runner stream wrapper intended to route tool-continuation LLM calls to a configured alternate model.

## Best Possible Solution

Keep the PR open while maintainers decide whether split model routing belongs in the core `agents.defaults.model` contract or behind a new plugin/API seam. If it stays core, the implementation should make the effective model drive the entire model-call pipeline, not only the `Model` argument passed to one stream wrapper; if maintainers prefer plugin routing, #13225 is the right API design thread to continue.

- If `model.tool` becomes public core config, generated config baselines, public docs, and config validation/docs tests need to be updated with the new contract.
- The PR body still lists manual Ollama plus Anthropic integration proof and full repo validation as incomplete.
