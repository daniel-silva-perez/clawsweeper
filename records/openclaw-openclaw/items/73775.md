## Summary

Keep this PR open. Current main still documents iMessage as supported for `dmHistoryLimit`, while the current iMessage schema/type surface still accepts the field and the runtime path only applies in-memory history context to group messages. The PR remains a focused docs/changelog correction for #73172 and its provided diff does not introduce a security-sensitive change.

## Best Possible Solution

Review and merge this focused docs/changelog PR if maintainers accept the option-2 resolution for #73172: keep the schema/type field for compatibility, stop documenting iMessage as a supported `dmHistoryLimit` runtime provider, and leave any future iMessage direct-message history adapter to a separately testable runtime change.
