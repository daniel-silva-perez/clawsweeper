## Summary

Keep this PR open. Current main still documents iMessage as supported for `dmHistoryLimit`, while the current iMessage schema/type surface accepts the field and the runtime path still only applies `historyLimit` to group history plus live `watch.subscribe` events. The PR remains a focused docs/changelog fix for #73172 and does not appear to introduce a security-sensitive change.

## Best Possible Solution

Review and merge this focused docs-only PR if maintainers accept the option-2 resolution for #73172: keep the schema/type field backward-compatible, stop documenting iMessage as a supported `dmHistoryLimit` runtime provider, and leave any future iMessage direct-message history adapter to a separate testable code change.
