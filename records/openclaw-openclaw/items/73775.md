## Summary

Keep this PR open. Current main still lists iMessage as supported for `dmHistoryLimit`, the iMessage config schema and type still accept the field, and the iMessage monitor path still only shows group pending-history handling rather than a direct-message history replay adapter. The PR is not already implemented on main and remains a focused docs/changelog candidate for the linked iMessage history issue.

## Best Possible Solution

Review and likely merge this focused docs-only PR if maintainers accept the option-2 resolution for #73172: keep `channels.imessage.dmHistoryLimit` accepted for compatibility, remove iMessage from the documented runtime-supported DM-history provider list, and track any future iMessage direct-message history replay adapter as a separate macOS-testable code change.
