## Summary

Keep open. Current main still does not implement the PR's ACP stream legacy-key migration: the schema and type surface only accept the new names, config load only runs registered runtime migrations, and the current runtime migration registry has no ACP stream migration. The PR remains useful, but it needs maintainer review/rebase into the current modular migration layout and a scope check against the broader #35957 gateway-control-ui report.

## What This Changes

The PR adds a runtime legacy-config migration and regression tests to move older `acp.stream.maxTurnChars` and `acp.stream.maxToolSummaryChars` to the current `acp.stream` names while removing retired `acp.stream` keys during config load.

## Best Possible Solution

Keep this PR open for maintainer follow-up. The best path is to rebase it into the current modular legacy-config migration system, likely with an ACP runtime migration module plus tests for rename, non-overwrite, no-replacement removal, and wrong-typed removed keys, then clarify the relationship to #35957 before merge.
