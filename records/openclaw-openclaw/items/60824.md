## Summary

Keep this PR open. Current main still has no ACP stream legacy-key migration: the schema only accepts the new `acp.stream.maxOutputChars` and `maxSessionUpdateChars` fields, the runtime migration registry does not include an ACP migration, and generic doctor unknown-key stripping would lose the two renamed limits instead of preserving them. The PR needs a rebase into the current modular migration files, but the author already addressed the bot-reviewed no-replacement cleanup edge case in `e6a344ef7c3debbf972d7a2049b41791edd79ff9`, so the work remains plausibly useful.

## Best Possible Solution

Keep the PR open and ask for a rebase into the current modular legacy-config migration system, likely as an ACP/runtime migration plus tests covering rename, non-overwrite, no-replacement removal, and wrong-typed removed keys. Also clarify whether the PR should stop claiming to close #35957 or whether a separate gateway.controlUi legacy-key migration is needed for that broader report.
