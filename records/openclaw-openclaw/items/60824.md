## Summary

Keep open. Current main still has the user-facing gap this PR is trying to address: legacy `acp.stream` keys are not accepted by the strict schema and there is no registered ACP runtime migration. The PR is directionally valid, but it targets the old pre-modular migration file layout and needs to be ported before it is merge-ready. Security review found no unexplained workflow, dependency, package, secret, or code-execution surface changes in the provided PR diff.

## What This Changes

The PR adds a legacy ACP stream config migration and regression tests to rename `maxTurnChars`/`maxToolSummaryChars`, remove retired ACP stream keys, and preserve existing replacement values.

## Best Possible Solution

Port the migration into the current modular legacy-config migration system, register it from the runtime migration registry, and keep regression coverage for raw legacy detection, rename/non-overwrite behavior, no-replacement removal, and wrong-typed retired keys before merge.
