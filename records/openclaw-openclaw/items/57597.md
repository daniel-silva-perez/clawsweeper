## Summary

Keep this PR open. Current main still has the requester-store label write that the PR is trying to fix, while ACP metadata is already stored by resolving the spawned target session key. The PR is a plausible focused bug fix and is not obsolete on main.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate. The likely best fix is exactly this direction: resolve the target session store from the spawned ACP sessionKey before persisting the label, only mutate the in-memory requester store when it is the same store, and keep or land a regression test that proves requester-store and target-store mismatch behavior.
