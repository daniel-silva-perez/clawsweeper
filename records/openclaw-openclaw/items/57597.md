## Summary

Current main still has the requester-store label write that this PR is trying to fix. The PR remains a focused implementation candidate: the spawned ACP session key is for the target agent, ACP metadata already resolves that target store, but label persistence still writes through the requester command store path.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate. The likely best fix is to resolve the target session store from the spawned ACP sessionKey before updateSessionStore, only mutate the requester in-memory store when it is the same store, and land regression coverage for requester-store/target-store mismatch.
