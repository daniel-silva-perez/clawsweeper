## Summary

Keep open. Current main still does not implement the PR’s intended vLLM fix: JSON-stringified chat content-block arrays are returned as ordinary strings, while only real arrays are normalized into text. The PR remains useful, but its latest heuristic still needs maintainer tightening before merge.

## Best Possible Solution

Keep this PR open and revise it against current main. The best path is a conservative shared normalization helper that parses stringified JSON only when it has a recognizable chat content-block shape, mirrors real-array extraction for text and mixed media/text blocks, preserves arbitrary JSON strings, routes relevant string-body callers through the same helper, and adds focused regression tests for vLLM text-only blocks, mixed blocks, invalid JSON fallback, and non-chat typed JSON arrays.
