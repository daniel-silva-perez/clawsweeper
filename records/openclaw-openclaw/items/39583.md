## Summary

Keep open. Current main still does not implement the PR’s intended vLLM fix: JSON-stringified chat content-block arrays are treated as ordinary strings, while only real arrays are normalized into text. The PR remains a plausible focused fix, but it still needs maintainer review because its latest heuristic can over-classify arbitrary JSON arrays of objects with a `type` key and drop valid string content.

## Best Possible Solution

Keep this PR open and revise it against current main. The best path is a conservative shared normalization helper that only parses stringified JSON arrays when they have a recognizable chat content-block shape, mirrors the real-array extraction behavior for text and mixed media/text blocks, preserves arbitrary JSON strings, routes relevant string-body callers through the same helper, and adds focused regression tests for vLLM text-only blocks, mixed blocks, invalid JSON fallback, and non-chat typed JSON arrays.
