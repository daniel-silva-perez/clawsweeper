## Summary

Keep open. Current main still routes Discord reaction add/remove events into session-scoped system events, while internal hooks only expose message lifecycle events such as `message:received` and plugin hooks have no reaction hook name. Related reports #52658 and #60805 were closed as duplicates of this canonical hook/API gap, and #17840 separately tracks reaction-triggered agent turns.

## Best Possible Solution

Keep this issue open as the canonical Discord reaction hook/API tracker. The best fix is an additive, documented reaction event surface that fires only after existing Discord reaction authorization/filtering and route resolution, includes structured reaction metadata, and has focused tests/docs. Keep opt-in reaction wake/auto-response behavior in #17840 unless maintainers explicitly merge the designs.
