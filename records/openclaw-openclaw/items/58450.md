## Summary

Keep open. Current main has partial mitigations for adjacent plan-only and reminder-promise failures, and the reporter later noted the OpenAI/Codex GPT path looked healthier on 2026.4.23, but the requested general runtime guard for unbacked follow-up promises is not implemented. The existing reminder guard is cron-specific, and strict-agentic retry is scoped to specific provider/model lanes and narrow tool-activity cases.

## Best Possible Solution

Keep this issue open and implement a narrow runtime consistency guard in the reply pipeline: classify unbacked follow-up/future-work commitments in final user-visible text, check same-turn evidence for real follow-up mechanisms such as cron creation, sessions_spawn/background task creation, queued follow-up, or committed message-tool delivery, then retry/rewrite/annotate only when the promise is unsupported. Add focused tests for memory_search-plus-'come back' text, sessions_spawn-backed text, cron-backed reminders, and at least one non-OpenAI provider/default-contract path so this is not limited to strict-agentic GPT heuristics.
