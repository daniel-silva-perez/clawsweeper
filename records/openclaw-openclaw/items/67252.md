## Summary

Keep open. Current main and the latest tagged release still clear Feishu documents with a single non-paginated `documentBlock.list` call before `writeDoc` inserts replacement content; the locked Feishu SDK supports `page_token`, the bundled skill promises replace-all behavior, and the external fork PR is useful prior art but is not merged into `openclaw/openclaw`.

## What This Changes

The issue requests that `feishu_doc` write fully replace large Feishu documents by paginating the clear-before-write block listing before deleting old top-level blocks.

## Best Possible Solution

Land a focused Feishu plugin fix in `extensions/feishu/src/docx.ts`: paginate `client.docx.documentBlock.list` inside `clearDocumentContent` until no next page token remains, delete the full top-level range, and add a regression test where `feishu_doc` write receives multiple block-list pages before inserting replacement content.
