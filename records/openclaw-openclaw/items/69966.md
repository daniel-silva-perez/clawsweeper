## Summary

Current main still treats `contextInjection`, `bootstrapMaxChars`, and `bootstrapTotalMaxChars` as `agents.defaults`-only bootstrap profile settings. The strict per-agent config schema does not accept those fields under `agents.list[]`, runtime resolvers read only global defaults, Control UI presets patch only `agents.defaults`, and docs list nearby per-agent budget knobs but not these bootstrap profile overrides. The adjacent per-agent compaction work in #69988 does not supersede this bootstrap-profile request.

## Best Possible Solution

Keep this issue open and implement a core config/runtime feature: add optional direct per-agent overrides for `contextInjection`, `bootstrapMaxChars`, and `bootstrapTotalMaxChars` under `agents.list[]`, resolve them by the effective session agent id with fallback to `agents.defaults`, update Control UI to edit global defaults or a selected agentâ€™s bootstrap profile, and document/test inheritance. Defer named reusable profile registries unless maintainers explicitly want that product layer.
