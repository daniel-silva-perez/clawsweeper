## Summary

Keep open. Current main still treats bootstrap profile settings as defaults-only: `agents.list[]` does not accept `contextInjection`, `bootstrapMaxChars`, or `bootstrapTotalMaxChars`; runtime bootstrap/context-injection resolution reads `agents.defaults`; the Control UI profile presets patch only defaults; and docs still present these knobs as global defaults. The adjacent open compaction PR #69988 is related precedent, but it does not implement this bootstrap-profile request.

## Best Possible Solution

Implement a focused core config/runtime/UI feature: add optional direct per-agent overrides for `contextInjection`, `bootstrapMaxChars`, and `bootstrapTotalMaxChars` under `agents.list[]`; resolve them by effective session agent id with fallback to `agents.defaults`; update Control UI so bootstrap profiles can apply globally or to the selected agent; and document/test inheritance. Defer named reusable profile registries unless maintainers explicitly choose that product layer.
