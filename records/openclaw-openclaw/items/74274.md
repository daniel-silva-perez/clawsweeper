## Summary

Keep this PR open. Current main still lacks the requested Control UI/Gateway behavior, and the latest PR head is a focused implementation rather than obsolete work; it still needs normal maintainer review, especially around how download URLs are formed for external attachments.

## What This Changes

The PR normalizes assistant top-level `mediaUrl`/`mediaUrls` into Control UI attachment items, changes non-image attachment anchors to download links, and adds `assistant-media` attachment-disposition handling plus Gateway/UI tests.

## Best Possible Solution

Land a narrow version that renders assistant top-level media fields as attachment cards, makes local non-image assistant attachments download-friendly without weakening Control UI auth/root checks, and avoids mutating external attachment URLs in a way that can break signed links.
