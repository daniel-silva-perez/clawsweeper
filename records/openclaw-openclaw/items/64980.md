## Summary

Keep open. Current main has partial Matrix-side safeguards, but it does not implement the requested agent-visible streaming/editing metadata. Outbound Matrix draft previews can carry MSC4357 live markers, and inbound Matrix replacement edit events are dropped before routing, but the agent context still receives plain BodyForAgent/RawBody plus WasMentioned, with no is_streaming/is_final/m.relates_to/live-marker field. Block-streamed Matrix reply chunks are still sent as normal Matrix messages without extra content metadata, so the central multi-agent coordination request remains actionable.

## Best Possible Solution

Keep the issue open and implement a narrow Matrix/core inbound contract for streaming state. The safest direction is either to suppress initial MSC4357 live draft events from agent routing until finalization, or to add a typed agent-context field such as MessageStreamingState/IsStreaming/IsFinal plus relation target metadata, then cover initial live sends, m.replace edits, and blockStreaming bot-to-bot room behavior in Matrix monitor tests.
