## Summary

Keep open. Current main still has Matrix outbound live preview markers and accurate mention metadata, but it does not expose streaming, finality, or edit-relation state to the inbound agent context or message hook contract, so the reported multi-agent coordination gap remains.

## What This Changes

Request to expose Matrix streaming/edit/finality metadata to agents so multi-agent rooms can avoid acting on partial preview messages.

## Best Possible Solution

Add a narrow typed inbound streaming/relation contract, or gate Matrix live draft events until finalization, then propagate the chosen state through Matrix inbound routing, `MsgContext`, and inbound hook mapping. Cover MSC4357 live initial events, `m.replace` edits, `allowBots` plus `requireMention` rooms, and thread/reply cases in Matrix monitor and hook/context tests.

## Reproduction Assessment

Unclear. The issue body gives a concrete Matrix config and event sequence, and current source shows the requested metadata is absent, but this read-only sweep did not run a live Matrix multi-agent room to reproduce the truncated first-chunk response end to end.

## Solution Assessment

Unclear as proposed. The requested `is_streaming`/`is_final` style fields address a real gap, but the maintainable fix should be a typed Matrix/core inbound state or routing suppression policy rather than raw ad hoc content keys picked without maintainer API review.
