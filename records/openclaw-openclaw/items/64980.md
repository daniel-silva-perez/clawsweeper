## Summary

Keep this issue open. Current main still has Matrix client-facing draft/final preview behavior, but it does not expose the requested streaming/finality/edit-relation state to the agent context or plugin hook contracts. The prior ClawSweeper comment remains accurate against current main; the relevant Matrix/channel paths have not gained an IsStreaming/IsFinal/m.relates_to-style inbound field since that review.

## Best Possible Solution

Keep this open and implement a narrow Matrix/core inbound contract for streaming state. The safest product path is either to suppress live/partial Matrix events from agent routing until finalization, or to add typed fields such as MessageStreamingState, IsStreaming, IsFinal, and relation target metadata to the finalized inbound context and plugin hook mapping. Cover Matrix m.replace, MSC4357 live drafts, blockStreaming bot-to-bot rooms, allowBots mention gating, and thread/reply handling in Matrix monitor tests.
