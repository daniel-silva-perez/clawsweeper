## Summary

Keep this issue open. Current main still turns an explicit `pty:true` request into `usePty=false` whenever exec resolves to a sandbox runtime, and I found no warning/error branch or user-facing sandbox caveat. The related fix PR #39349 is still open and unmerged with closing syntax for this issue, so the issue should stay open until that PR, or a replacement, lands.

## Best Possible Solution

Keep this issue open and review #39349 or replace it with an equivalent narrow fix. The fix should either deliberately pass `pty:true` through to sandbox backends that support it or keep suppression but emit a clear warning/rejection before `usePty` is set false. Add focused regression coverage for sandboxed `pty:true`, and align `docs/tools/exec.md`, `src/agents/bash-tools.schemas.ts`, and `src/agents/bash-tools.descriptions.ts` with the chosen contract.
