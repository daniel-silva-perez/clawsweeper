## Summary

Keep this issue open. Current main still masks explicit `pty:true` requests whenever exec resolves to a sandbox runtime, the user-facing docs/schema still do not state the sandbox limitation, and the same-repo fix PR #39349 remains open and unmerged with closing syntax for this issue.

## Best Possible Solution

Keep this issue open and review/land #39349 or replace it with an equivalent narrow fix. The fix should choose the product contract explicitly: either pass `pty:true` through to sandbox backends that support it, or keep suppression but emit a clear warning/rejection. Add focused regression coverage for sandboxed `pty:true` and align `docs/tools/exec.md`, `src/agents/bash-tools.schemas.ts`, and `src/agents/bash-tools.descriptions.ts` with the chosen behavior.
