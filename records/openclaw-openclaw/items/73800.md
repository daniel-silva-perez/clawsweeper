## Summary

Keep this PR open. Current main has the #73228 target-agent routing piece, but it still uses the coarse `!result.delivered` hook fallback gate, and current tests still expect a successful `deliver:false` hook result to enqueue a system event. The PR is a focused implementation candidate for the remaining shared-hook announcement policy work, not something already implemented or superseded.

## Best Possible Solution

Keep the PR open for normal review. The likely best path is to land a scoped gateway fix that replaces the `!result.delivered` fallback gate with an explicit announcement policy, keeps #73228 target-agent routing composed with that policy, adds the included regression tests, and explicitly decides whether the broader #55761 prefix/`announceToMain` pieces belong in this PR or a follow-up.
