## Summary

Keep this PR open. Current main still uses the coarse `!result.delivered` gateway hook fallback and still has tests expecting a successful `deliver:false` hook result to enqueue a system event. The PR remains a focused implementation candidate for the shared-hook announcement policy, with one review question around `skipped` handling and no obvious security-sensitive file changes in the provided diff.

## Best Possible Solution

Keep this PR open for normal maintainer review. The best path is to land or amend a scoped gateway fix that replaces the `!result.delivered` fallback with an explicit announcement policy, keeps the already-merged target-agent routing composed with that policy, documents or tests the `skipped` behavior, and decides whether the broader #55761 `announceToMain` and prefix cleanup pieces belong here or in a follow-up.
