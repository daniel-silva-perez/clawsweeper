## Summary

Keep open. Current main still handles verified same-path/pattern QMD conflicts, but the reported metadata-free different-name conflict can still fall through to a skipped add while later searches query the configured collection name.

## What This Changes

Fix QMD boot-time collection reconciliation so a stale different-name collection such as `life-main` does not leave searches querying the configured `life` collection and failing.

## Best Possible Solution

Keep this issue open until PR #72297 or an equivalent memory-core change is merged: parse the stale conflict name from QMD stderr, surface a targeted manual repair hint when list metadata is unverified, keep automatic rebind limited to verified path/pattern matches, and cover the path with regression tests.

## Reproduction Assessment

Yes. The issue includes concrete QMD stderr for the `life` versus `life-main` conflict, and current tests model the metadata-free add-conflict path that still produces a skipped-add warning.

## Solution Assessment

Yes for the linked implementation direction, not for current main. The narrow maintainable fix is metadata-gated rebind plus a clear repair hint for unverified conflicts, rather than blindly removing a name seen only in stderr.
