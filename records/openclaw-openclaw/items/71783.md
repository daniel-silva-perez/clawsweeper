## Summary

Keep this issue open. Current main still does not parse the conflicting QMD collection name from add-error text, and the name-only/list-metadata-missing path still logs `qmd collection add skipped` while searches continue to target the managed config name. Open PR #72297 is an active candidate that addresses this conservatively with a manual repair hint, but it is not merged.

## Best Possible Solution

Keep the issue open and review #72297 or an equivalent narrow memory-core change. The fix should either safely repair only when QMD exposes verifiable path/pattern metadata, or clearly surface the stale collection name and exact manual repair when only add-error text identifies the conflict. Avoid broad core migrations or unverified destructive removal.
