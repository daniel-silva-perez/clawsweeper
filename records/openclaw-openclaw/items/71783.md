## Summary

Current main still does not fully resolve the reported QMD different-name path/pattern conflict. The code only auto-rebinds same-name conflicts or different-name conflicts when QMD list output exposes verifiable path and pattern metadata; the metadata-free conflict path still logs a skipped add, while search continues to target the configured managed collection name. Open PR #72297 remains the focused implementation candidate, so this issue should stay open for review/landing of that PR or an equivalent narrow fix.

## Best Possible Solution

Keep this issue open and review/land PR #72297, or an equivalent narrow `memory-core` change. The fix should parse QMD path/pattern conflict errors enough to show a clear manual repair hint for metadata-free conflicts, while preserving automatic rebind only when QMD exposes verifiable path/pattern metadata.
