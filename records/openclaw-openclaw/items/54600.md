## Summary

Keep open. Current `main` still forwards the cron `agentTurn` message value directly into the isolated runner, so this PR is not obsolete; however, the branch needs maintainer review because the proposed defensive extraction logs serialized prompt content on the fallback path and lacks focused regression coverage for the malformed runtime shape.

## What This Changes

This PR modifies `src/cron/service/timer.ts` to unwrap object-shaped cron `agentTurn.payload.message` values from `text`, `message`, or `content` fields, then JSON-stringify and debug-log unrecognized shapes before calling the isolated cron runner.

## Best Possible Solution

Land a tested cron fix that preserves the public `message: string` contract: either normalize/repair malformed stored payloads before execution, or add a narrow defensive extraction at the isolated execution boundary without logging prompt content.
