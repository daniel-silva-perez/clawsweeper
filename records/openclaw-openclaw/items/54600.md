## Summary

Keep this PR open. Current main does not implement the PR's intended runtime guard: isolated cron execution still forwards `job.payload.message` directly into `runIsolatedAgentJob`, while normalization only trims string messages and leaves non-string payload message values alone. The linked bug issue is closed, but I did not find an equivalent fix on main, and the PR is a focused one-file runtime change with no visible supply-chain or CI surface.

## Best Possible Solution

Keep the PR open and either land a tested version of this defensive extraction in `src/cron/service/timer.ts` or implement an equivalent normalization/repair path that guarantees isolated cron receives a string prompt. A focused regression test should cover a persisted `agentTurn` job whose `payload.message` is an object such as `{ text: "Say: HELLO WORLD" }` and assert that `runIsolatedAgentJob` receives the extracted text, not `[object Object]`.
