## Summary

Keep this PR open. Current main still forwards the cron agentTurn payload message directly into isolated execution, so the branch is not obsolete; the remaining maintainer decision is whether to land this defensive extraction with a focused regression test or repair the malformed payload earlier in normalization/doctor.

## What This Changes

The PR updates `src/cron/service/timer.ts` so cron isolated agentTurn execution extracts a string from object-shaped `payload.message` values before calling the isolated agent runner, with JSON fallback logging for unknown shapes.

## Best Possible Solution

Land a tested fix that keeps the public cron `message: string` contract intact, either by normalizing/repairing malformed stored payloads before execution or by adding a narrow defensive extraction at the isolated execution boundary without logging sensitive prompt content.
