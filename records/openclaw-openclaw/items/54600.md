## Summary

Keep open. Current main still forwards the cron `agentTurn` payload message directly into isolated execution, where object-shaped runtime values can be interpolated as `[object Object]`; this PR is a plausible targeted fix, but it is unmerged and still needs maintainer review for root cause, tests, and prompt-content logging.

## What This Changes

This PR changes `src/cron/service/timer.ts` to extract string content from object-shaped cron `agentTurn.payload.message` values using `text`, `message`, or `content` fields, with JSON fallback debug logging before calling the isolated agent runner.

## Best Possible Solution

Land a tested fix that preserves the public `message: string` cron contract: either normalize/repair malformed stored payloads before execution, or add a narrow defensive extraction at the isolated execution boundary without logging sensitive prompt content.
