## Summary

Keep open. Current main still lacks the defensive extraction for malformed cron `agentTurn` messages, and the PR targets a plausible user-visible failure, but the proposed patch introduces prompt-content logging and should not merge as-is.

## What This Changes

The branch modifies `src/cron/service/timer.ts` to unwrap object-shaped cron `agentTurn.payload.message` values from common string fields and fallback-stringify before calling the isolated runner.

## Best Possible Solution

Land a tested cron fix that preserves the public string-valued message contract: normalize or repair malformed stored payloads before execution where possible, and keep any defensive extraction at the isolated execution boundary narrow and free of prompt-content logging.
