## Summary

Keep this PR open. Current main still passes isolated cron agentTurn messages straight through to the isolated runner, and the runner still interpolates `input.message` into the prompt; the PR is a focused defensive hardening change but needs a regression test and maintainer verification because the linked bug report says persisted `jobs.json` already contained a correct string.

## What This Changes

The PR updates `src/cron/service/timer.ts` to defensively extract cron `agentTurn` payload text from a string or common object fields before passing it to the isolated agent runner, with JSON fallback logging for unknown object shapes.

## Best Possible Solution

Keep this PR open. Add a focused regression test that loads or runs an isolated cron job with malformed object-shaped `payload.message`, then either land this defensive extraction in the current `executeDetachedCronJob` path or implement an equivalent normalization/doctor repair path while preserving the public `message: string` cron contract.
