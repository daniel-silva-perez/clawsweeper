## Summary

Keep open. Current main and the latest release still install Playwright Chromium under `/home/node/.cache/ms-playwright`, not `/opt/playwright`, and the Docker docs still document the hidden cache path. This PR is paired with the same-author open bug #42819 via closing syntax, and the PR patch still needs maintainer review for Docker ownership/cache behavior before it can be merged or rejected.

## Best Possible Solution

Keep this PR open for contributor and maintainer follow-up. A complete path would rebase on current main, set a stable Docker Playwright browser path, create it with node ownership in default builds, recursively chown after optional preinstall, avoid deleting apt list cache contents from the BuildKit cache mount, add the Chromium symlink only after a successful install, update `docs/install/docker.md` and `docs/tools/browser-control.md`, and add focused Dockerfile assertions for path, ownership, symlink, and cache behavior.
