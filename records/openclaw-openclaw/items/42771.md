## Summary

Keep this PR open. Current main still installs Docker Playwright browsers under `/home/node/.cache/ms-playwright`, docs still tell Docker users to persist that cache path, and the same-author paired bug remains open. The PR is a plausible in-repo Docker/docs fix, but the current patch still needs Dockerfile corrections before it should merge.

## Best Possible Solution

Keep this PR open for maintainer review and contributor fixes. The best path is to rebase onto current main, set `/opt/playwright` as the Docker Playwright browser path, create it with `node:node` ownership for default builds, recursively chown after browser preinstall, avoid deleting apt lists inside the BuildKit cache mount, create the Chromium symlink only after a successful install, update both Docker and browser-control docs, and add focused Dockerfile assertions for path, ownership, symlink, and cache behavior.
