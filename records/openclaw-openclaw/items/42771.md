## Summary

Keep open. Current main still uses the hidden Playwright cache path, the linked bug remains open, and this PR is useful but not merge-ready because the Dockerfile changes leave the new default path unwritable for runtime installs and delete a BuildKit apt cache mount.

## What This Changes

The PR changes the Docker Playwright browser path to `/opt/playwright`, creates that directory with `mkdir`/`chmod`, recursively chowns preinstalled browser files, adds a `/usr/local/bin/chromium` symlink, and updates Docker/browser documentation.

## Best Possible Solution

Land a corrected Docker/browser fix that sets a stable `/opt/playwright` default, creates it with `node:node` ownership, recursively reassigns ownership after optional root-run preinstall, preserves BuildKit apt caches, creates the chromium symlink only after a successful install, and updates the current Docker/browser docs plus focused Dockerfile assertions.

## Reproduction Assessment

Yes. The high-confidence reproduction is to apply the PR, build without `OPENCLAW_INSTALL_BROWSER`, then run `node /app/node_modules/playwright-core/cli.js install chromium` as the image's `node` user; the PR-created `/opt/playwright` is root-owned and Playwright writes registry metadata there.

## Solution Assessment

No. The direction is maintainable, but this exact patch is not the best fix until it restores runtime-writable ownership, removes the apt-list deletion from the cache mount, rebases the docs, and adds focused assertions for the Docker contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Set node ownership when creating /opt/playwright:** `Dockerfile:165-166`
  - body: The default build path creates `/opt/playwright` as `root:root` with mode `775`, and no later `chown` runs when `OPENCLAW_INSTALL_BROWSER` is unset. Since the image later runs as `node`, documented runtime installs into the new default `PLAYWRIGHT_BROWSERS_PATH` will fail when Playwright tries to write its registry metadata there.
  - confidence: 0.93
- **[P2] Remove apt-list cleanup from the cache mount:** `Dockerfile:176`
  - body: `/var/lib/apt` is mounted as a BuildKit cache in this same `RUN`, so deleting `/var/lib/apt/lists/*` wipes the named cache for later builds while not reducing the final image layer. This regresses browser-enabled rebuild performance and should be removed.
  - confidence: 0.88
