## Summary

Keep this PR open. Current main at 891c7d9f1c1f5e0f52935fadfeacf695b0ef1eb8 still installs Docker Playwright browsers under `/home/node/.cache/ms-playwright`, has no default `/opt/playwright` browser path, and still documents the hidden cache path for Docker persistence. The PR is paired with the same-author open bug #42819 and is a plausible in-repo Docker/docs fix candidate, but the current patch still needs Dockerfile corrections before it should merge.

## Best Possible Solution

Keep this PR open for contributor and maintainer follow-up. The best path is to rebase on current main, set `/opt/playwright` as the Docker Playwright browser path, create it with node ownership before default runtime installs, recursively chown after optional browser preinstall, remove apt-list deletion inside the BuildKit cache mount, keep the Chromium symlink behind a successful install, update the current Docker and browser-control docs, and add focused `src/dockerfile.test.ts` assertions for path, ownership, symlink, and cache behavior.
