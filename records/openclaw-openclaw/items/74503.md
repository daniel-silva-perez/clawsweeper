## Summary

Keep open. The retry-disable half now matches the upstream SettingsManager contract, but the new repeated-thinking mitigation is wired into `transformTransportMessages`, while the Qwen/default `openai-completions` path still builds payloads through the OpenAI completions converter and does not call that helper. That leaves the PR’s advertised Qwen thinking-loop layer ineffective for the target surface.

## What This Changes

This PR disables pi-coding-agent retry for prepared embedded runs via an in-memory settings snapshot and adds repeated-thinking loop-hint injection in transport message transformation.

## Best Possible Solution

Land the in-memory retry override with focused coverage, and move or duplicate the repeated-thinking guard into a shared stream/replay seam that actually covers default `openai-completions` providers such as Qwen before adding provider-specific loop hints.
