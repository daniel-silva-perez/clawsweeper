## Summary

Keep open. The PR targets a real agent replay-loop report, and the post-review update fixes the earlier Qwen early-return problem in shape, but the latest diff still introduces a blocking settings-persistence regression: the proposed retry toggle can write into the operator's file-backed pi settings instead of staying a per-run OpenClaw override.

## What This Changes

The PR disables pi-coding-agent auto-retry in `createPreparedEmbeddedPiSettingsManager` and adds repeated-thinking loop-hint injection to `transformTransportMessages`.

## Best Possible Solution

Land a narrow, tested fix for #73781 that disables pi-coding-agent's inner retry through a non-persistent per-run override or sanitized in-memory snapshot, then add any thinking-loop mitigation behind a tested transport/runtime seam that covers Qwen's `openai-completions` path without surprising strict-provider transcript shape or user settings.
