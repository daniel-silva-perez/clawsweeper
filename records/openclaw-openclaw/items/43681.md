## Summary

Keep open. Current main has same-provider auth-profile rotation once multiple profiles exist, but it still does not provide a supported way to create distinct OpenAI Codex OAuth profiles for the same email/team subscription through `models auth login`. The cross-referenced alias PR #43694 was closed unmerged, and the remaining operator-chosen profile-id work is still tracked separately by #40402/#49315 rather than shipped on main.

## Best Possible Solution

Keep this issue open and resolve it with a supported OAuth profile discriminator for `models auth login`: either a generic `--profile-id`/`--profile-alias` passed through provider-owned auth context, or OpenAI Codex-specific team/account/org id extraction that beats plain email when multiple subscriptions share one account. Add tests proving two same-email Codex OAuth logins persist distinct `openai-codex:*` profiles and that existing rate-limit rotation can use them.
