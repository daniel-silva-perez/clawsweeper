## Summary

Keep open. Current main still returns exec stdout/stderr-derived output through foreground, background/process, node-host, and dynamic-tool paths before an exec-owned redaction boundary; downstream event/UI sanitization and the related HTTP/logging PR do not close the reported agent-context exposure.

## What This Changes

The issue reports that secret-bearing exec stdout/stderr can be incorporated into tool-result context without redaction.

## Best Possible Solution

Add an exec-owned output-redaction seam that sanitizes stdout/stderr-derived content and details before foreground results, process poll/log results, node-host results, Codex dynamic-tool responses, and approval follow-up summaries are returned, while preserving status metadata, caps, truncation behavior, and any maintainer-approved verbatim-output policy.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: a command that prints an env-style secret flows into `session.aggregated`/`outcome.aggregated` and then into exec result text or details before an exec-boundary redaction call.

## Solution Assessment

Yes. Redacting stdout/stderr before exec result construction and at shared process/node-host/dynamic-tool boundaries is the narrowest maintainable fix; blocking specific secret-file reads could supplement it but would not replace output redaction.

## Security Review

Status: needs_attention

Summary: This security issue remains actionable because exec stdout/stderr can still become raw tool-result text/details before an exec-owned redaction pass.

Concerns:

- **[high] Redact foreground exec output before returning tool results:** `src/agents/bash-tools.exec.ts:79`
  - body: Foreground exec result construction still uses raw outcome reason and aggregated output for `AgentToolResult` content/details, so secret-bearing stdout can cross the tool-result boundary before downstream event sanitizers run.
  - confidence: 0.9
- **[medium] Cover background process output paths:** `src/agents/bash-tools.process.ts:317`
  - body: Process poll/log results return stdout/stderr-derived aggregates and slices directly, so a foreground-only fix would leave comparable leak paths open.
  - confidence: 0.86
- **[medium] Cover node-host exec output formatting:** `src/agents/bash-tools.exec-host-node-phases.ts:82`
  - body: Node-host result formatting renders raw stdout/stderr/error text and stores the joined raw fields in `details.aggregated`, which can expose remote command output in the same tool-result shape.
  - confidence: 0.84
- **[high] Redact before dynamic-tool responses:** `src/agents/pi-tool-definition-adapter.ts:249`
  - body: The Pi adapter and Codex dynamic-tool bridge start from raw tool execution results; event-facing sanitization alone does not prove the model/tool-result payload is redacted before consumption.
  - confidence: 0.82
- **[medium] Redact approval follow-up summaries:** `src/agents/bash-tools.exec-host-gateway.ts:216`
  - body: Gateway approval follow-up summaries include tails of `outcome.aggregated`, so approved exec flows can reintroduce secret-bearing command output into system events.
  - confidence: 0.78
