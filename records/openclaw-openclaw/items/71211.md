## Summary

Keep open. Current main still builds exec-owned foreground, background/process, and node-host tool results from raw stdout/stderr-derived strings; downstream UI/event redaction exists, but it does not close the raw exec result boundary described here. Related PRs #75033 and #75230 are adjacent logging/redaction work and remain open, so neither supersedes this issue.

## What This Changes

Request that exec stdout/stderr be redacted or blocked before being returned to agent/tool result context.

## Best Possible Solution

Add a shared exec/tool-payload redaction helper at the exec-owned output boundary: foreground result construction, runtime outcomes, process poll/log responses, node-host formatting, and approval follow-up summaries. Cover env-style assignments and known token formats in focused tests while keeping existing downstream event/UI redaction as defense in depth.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: a command that prints `TOKEN=...` flows through `session.aggregated` into foreground, background/process, or node-host tool result content/details without an exec-boundary redaction call. I did not run a live secret-printing command because this review is read-only.

## Solution Assessment

Yes. The issue’s primary direction, redacting stdout/stderr before exec result construction and at shared process/node-host boundaries, is the narrowest maintainable fix; blocking specific secret-file reads could supplement it but would not replace output redaction.

## Security Review

Status: needs_attention

Summary: This security issue remains actionable because exec stdout/stderr can still become raw tool result text/details before an exec-owned redaction pass.

- **[high] Redact foreground exec output before returning tool results:** `src/agents/bash-tools.exec.ts:79`
  - body: Foreground exec result construction still uses raw `outcome.reason` and `outcome.aggregated` for tool result content/details, so a secret printed by a command can reach internal tool-result context before downstream sanitizers run.
  - confidence: 0.9

  - body: `process poll` and `process log` return aggregate output and slices from raw process buffers, so a foreground-only fix would leave comparable background leak paths open.
  - confidence: 0.86

  - body: Node-host exec formatting renders raw stdout/stderr/error text and stores the joined value in `details.aggregated`, leaving remote exec output outside the downstream UI-only redaction boundary.
  - confidence: 0.84
