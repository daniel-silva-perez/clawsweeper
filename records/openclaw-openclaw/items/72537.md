## Summary

## What This Changes

The PR preserves OpenAI TTS provider `timeoutMs`, resolves chat/non-telephony TTS attempt timeouts with explicit/provider/global precedence, and adds speech-core regression tests.

## Best Possible Solution

Merge the narrow fix after maintainer review, rebase, and gates, keeping the behavior split between the generic speech-core synthesis loop and the OpenAI plugin normalizer rather than adding a new core API or provider-specific core special case.

## Solution Assessment
