## Summary

Keep this open. Current main has partial mitigations for startup prewarm and macOS no-prompt keychain reads, but the central `models.list` problem is not proven fixed: the RPC still awaits the full gateway model catalog, and the catalog path can still run provider discovery hooks without a normal non-live timeout.

## What This Changes

The issue requests faster Gateway startup and `models.list` responses with many configured providers by making provider validation/catalog work lazy, parallel, or optional.

## Best Possible Solution

Keep this open until normal Gateway operation can answer the default/configured `models.list` view from bounded work: configured rows and auth summaries should not require full catalog discovery, provider catalog hooks should be scoped, parallelized, cached, or time-boxed, and the separate prewarm and first-message stalls should stay tracked through narrower follow-ups.
