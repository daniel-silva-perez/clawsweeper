## Summary

Keep this open. Current main contains useful mitigations for the newest prewarm and macOS Keychain reports, but the central user-visible problem is not proven fixed because `models.list` still waits for the gateway model catalog and that catalog path can still run provider discovery without a normal default timeout.

## What This Changes

The issue asks OpenClaw to keep gateway startup and `models.list` fast when several provider plugins or slow provider/auth discovery paths are configured.

## Best Possible Solution

Keep this open until the model catalog path is bounded for normal gateway operation: `models.list` should avoid nonessential full-catalog work for configured/default views, provider catalog hooks should be scoped, parallelized, or time-boxed, and regression coverage should include deliberately slow provider catalog/auth/prewarm hooks.
