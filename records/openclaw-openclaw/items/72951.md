## Summary

Keep open. The PR is a focused fix for a real dashboard/history sender-label regression and it addresses the earlier Greptile edge-case feedback, but the current diff only wires runtime-context recovery into the raw JSONL parser. Current v3 transcripts take the active-branch `SessionManager.getBranch()` path first, and that path still ignores hidden `custom_message` runtime-context entries before returning.

## What This Changes

The PR adds gateway session-history logic and tests to recover `senderLabel` from hidden `openclaw.runtime-context` rows while keeping those rows out of `readSessionMessages()` output.

## Best Possible Solution

Handle `openclaw.runtime-context` consistently in both `SessionManager.getBranch()` and raw JSONL history reads, preferably through one small shared helper, then cover both transcript shapes with focused gateway regression tests.

## Reproduction Assessment

Yes. A v3 transcript with a user message followed by a hidden `openclaw.runtime-context` `custom_message` entry takes the active-branch reader, where this PR currently skips the custom row and never attaches `senderLabel`.

## Solution Assessment

No. Recovering from raw JSONL rows is useful, but the narrow maintainable fix is to apply the same runtime-context filtering and sender-label attachment before the active-branch path returns.
