## Summary

Keep open. Current main still lacks the sender-label recovery this PR is trying to add, and the PR is a scoped contributor implementation with relevant tests rather than obsolete work. It needs maintainer review before merge, especially because the current active-branch `SessionManager.getBranch()` path returns before the raw JSONL parser and the PR diff only adds runtime-context handling in the raw parser path.

## What This Changes

This PR changes gateway session history reading to recover `senderLabel` from hidden `openclaw.runtime-context` custom rows and adds focused `readSessionMessages()` regression tests for hidden, no-label, compaction, and stale-label cases.

## Best Possible Solution

Merge a reviewed version that recovers `senderLabel` from hidden runtime-context rows for every history shape that can contain them, keeps those rows hidden from visible history output, and preserves the model-visible prompt cleanup that moved runtime metadata out of user text.
