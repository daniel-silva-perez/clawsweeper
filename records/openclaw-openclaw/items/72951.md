## Summary

Keep this PR open, but the current patch is not ready to merge. It fixes the legacy/raw JSONL reader path, while current v3 session files with `id`/`parentId` take the `SessionManager.getBranch()` path and return before the new runtime-context recovery code runs.

## What This Changes

The PR adds gateway session-history logic and regression tests to recover `senderLabel` from hidden `openclaw.runtime-context` custom rows while keeping those rows out of `readSessionMessages()` output.

## Best Possible Solution

Update `readSessionMessages()` so runtime-context `custom_message` entries are handled consistently in both the active-branch and raw JSONL paths, keep hidden runtime-context rows out of returned history, and cover both storage shapes with focused regression tests.

## Reproduction Assessment

Yes. A v3 transcript with a user message followed by a hidden `openclaw.runtime-context` `custom_message` entry will take the active-branch path, where this PR currently ignores the custom row and never attaches `senderLabel`.

## Solution Assessment

No. Recovering from raw JSONL rows is useful, but the maintainable fix is to share the runtime-context handling with the `SessionManager.getBranch()` reader so current active-branch transcripts are covered too.
