## Summary

Keep open. Current main still routes Gemini CLI OAuth token exchange and project/userinfo calls through the default strict guarded fetch path, which intentionally ignores ordinary HTTP_PROXY/HTTPS_PROXY env proxy routing unless the caller opts into trusted env-proxy mode or OpenClaw's managed proxy lifecycle is active. The linked #47518 PR is an open implementation candidate for this exact issue, but it is not merged, so this issue should remain open for review/landing or an equivalent fix.

## Best Possible Solution

Review and land #47518 if its implementation is still correct after current-main changes, or apply an equivalent focused fix in the Google plugin's OAuth HTTP helper that conditionally uses trusted env-proxy mode for Gemini CLI OAuth/Code Assist calls with regression coverage. Keep this issue open until that fix merges and ships or is otherwise superseded.
