## Summary

Keep this PR open, but it needs changes before merge. Current main still lacks dedicated `read`, `write`, and `edit` inventory entries in `docs/tools/index.md`, so the requested docs gap is real; however, the proposed entries describe unsupported aliases, an outdated fixed read cap, a top-level edit payload shape, and `MEDIA:<path>` image behavior that do not match current main.

## What This Changes

This PR adds dedicated `read`, `write`, and `edit` entries to `docs/tools/index.md` with descriptions, parameters, and usage notes for the file tools.

## Best Possible Solution

Revise the docs-only patch so the new inventory entries document the current canonical contracts: `read` and `write` use `path`, `write` uses string `content`, `edit` uses `edits: [{ oldText, newText }]`, read output is adaptively paged/capped, and image reads are image content blocks with no guaranteed `MEDIA:` marker.

## Reproduction Assessment

Not applicable as a docs-only PR, but there is a high-confidence verification path: compare the proposed docs text against current runtime validation, read execution, and colocated tests on main.

## Solution Assessment

No, not as written. Adding dedicated inventory entries is the right direction, but the patch must be corrected to match current main before it is a maintainable docs fix.

- **[P2] Document adaptive read caps:** `docs/tools/index.md:190-202`
  - body: The added `read` docs say text output is always truncated to `2000 lines or 50KB`, but current read execution auto-pages up to an adaptive byte budget and tests cover returning a 5,000-line file when context allows. Please describe the adaptive cap/default continuation behavior instead of a fixed 50KB limit.
  - confidence: 0.94
- **[P2] Remove unsupported legacy path aliases:** `docs/tools/index.md:194`
  - body: The new entries document `path or file_path`, but runtime validation only accepts canonical `path` and tests reject `file_path`. Keeping this alias in the user docs will cause copy-paste tool calls to fail.
  - confidence: 0.93
- **[P2] Describe the current edits array shape:** `docs/tools/index.md:222-224`
  - body: The `edit` section documents top-level `oldText`/`newText` and legacy snake_case aliases, but current validation requires `edits: [{ oldText, newText }]` and supports multiple replacements. Users following this shape will get a missing `edits` error.
  - confidence: 0.95
- **[P2] Document image reads as image blocks:** `docs/tools/index.md:202`
  - body: The `read` notes say images return `MEDIA:<path>`, but current read results are image content blocks and may have no media path; extraction tests only fall back to `details.path` when present. Please avoid promising a `MEDIA:` token.
  - confidence: 0.91
