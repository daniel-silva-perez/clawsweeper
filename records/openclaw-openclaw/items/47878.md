## What This Changes

The PR adds dedicated `read`, `write`, and `edit` entries to `docs/tools/index.md` with descriptions, parameters, and usage notes for the file operation tools.

## Best Possible Solution

Revise the docs-only patch so the inventory entries match current main: canonical `path`, string `content`, `edits: [{ oldText, newText }]`, adaptive read paging/caps, and image reads as image content blocks with no guaranteed `MEDIA:` token.

  - confidence: 0.95

  - body: The `read` notes say images return `MEDIA:<path>`, but current read results are image content blocks and may have no media path; extraction tests only fall back to `details.path` when present. Please avoid promising a `MEDIA:` token.
  - confidence: 0.91
