## Summary

Keep open. Current main still contains the exact slash-form alias bypass described by the PR and linked issue #73573, and the PR is a focused implementation candidate rather than obsolete work. The provided diff is small and does not introduce security- or supply-chain-sensitive file changes.

## Best Possible Solution

Review and land this PR or an equivalent patch that resolves slash-form aliases before provider/model parsing in the shared model-selection path, keeps the focused regression tests, and then let the linked issue #73573 close through the merge. If slash-form aliases are meant to work across every model-selection surface, add matching coverage for the separate subagent-spawn alias helper before or shortly after landing.

## Risks / Open Questions

- The proposed alias-first behavior means a configured slash-form alias can intentionally shadow an otherwise valid typed `provider/model` string; maintainers should confirm that precedence is desired for this config surface.
- There is a separate subagent-spawn helper in `src/agents/model-selection.ts` that also returns slash-containing values before alias lookup, so maintainers may want to decide whether slash-form aliases should be fixed there too or scoped only to the shared runtime resolver in this PR.
