## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Restore existing Gateway and protocol surfaces:** `src/gateway/server-methods.ts:74`
  - body: The patch adds `memoryHandlers` while dropping existing plugin/session/protocol registrations such as `plugins.uiDescriptors` and `sessions.pluginPatch` from the method lists, scopes, and schemas. Those methods exist on current main and are public Control UI/plugin surfaces, so this would make existing clients lose methods unrelated to the memory page; keep the memory RPC additions additive.
  - confidence: 0.9
- **[P1] Guard pathless search hits before sorting:** `src/gateway/server-methods/memory.ts:442-444`
  - body: `decorateSearchHit` can leave `path` undefined, but the equal-score sort fallback calls `left.path.localeCompare(right.path)`. A pathless hit with the same score will throw and fail `memory.search.debug`; normalize with `left.path ?? ""` and `right.path ?? ""` before comparing.
  - confidence: 0.93
- **[P2] Bound the index job registry:** `src/gateway/server-methods/memory.ts:50-52`
  - body: `indexJobs` is process-global and entries are added by `memory.index.run` without TTL, pruning, or a size cap. A long-running Gateway that receives repeated index requests will retain completed/failed jobs indefinitely; evict old jobs or cap the map like the `sourceRefs` TTL path.
  - confidence: 0.88
- **[P2] Keep the memory host SDK self-contained:** `packages/memory-host-sdk/src/host/memory-schema.ts:2`
  - body: The package-local memory schema changes from `./error-utils.js` to a deep `../../../../src/infra/errors.js` import. Published package consumers should not depend on the repository's root `src` tree being present; keep the local helper or expose a supported package boundary.
  - confidence: 0.84
