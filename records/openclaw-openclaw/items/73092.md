## Summary

Keep this PR open. Current main still has the unconditional outer no-real-conversation compaction skip before any token-threshold bypass, and the manual `/compact` path still resolves fresh token usage only after compaction instead of passing it into the compaction engine.

## What This Changes

The PR adds a reserve-threshold-aware compaction skip helper, forwards fresh manual `/compact` token usage into `compactEmbeddedPiSession`, and adds focused regression tests for over-threshold tool-heavy slices and manual token forwarding.

## Best Possible Solution

Land a narrow compaction fix that preserves the cheap idle-session skip for genuinely idle or below-threshold sessions, while allowing fresh over-threshold manual compaction to proceed even when the limited tail is tool-heavy.

## Reproduction Assessment

Yes. The linked issue gives a concrete reproduction shape, and current-main source inspection confirms that a limited tail of heartbeat/tool-result messages with no meaningful user message in the 20-entry lookback can hit the outer skip while persisted tokens are over the context window.

## Solution Assessment

Yes. The PR uses the existing fresh session-token contract and Pi reserve-token setting to bypass only at or above the compaction threshold, which is narrower than changing the shared message classifier or disabling the idle pre-check entirely.
