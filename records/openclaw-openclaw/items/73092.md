## Summary

Keep this PR open. Current main still has the unconditional no-real-conversation skip and manual `/compact` still does not pass fresh persisted token usage into compaction, while the PR is a narrow implementation of the linked bug with no concrete review finding from this read-only sweep.

## What This Changes

The PR adds a token-threshold-aware no-real-conversation compaction guard, forwards fresh manual `/compact` token totals into embedded compaction, and adds focused tests for tool-heavy over-threshold slices and manual token forwarding.

## Best Possible Solution

Land a narrow fix that preserves the cheap idle-session skip for absent or below-threshold token usage, but lets fresh over-threshold sessions reach compaction even when the limited tail is tool-heavy.

## Reproduction Assessment

Yes. A code-level reproduction is a limited session slice containing only heartbeat text plus tool results outside the 20-message meaningful-user lookback, combined with a fresh persisted total at or above `contextWindow - reserveTokens`; current main takes the idle skip in that shape.

## Solution Assessment

Yes. The proposed direction uses the existing fresh session token contract and the already-prepared Pi reserve-token setting, avoiding broader changes to the real-conversation classifier or inner safeguard behavior.
