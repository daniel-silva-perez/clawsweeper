## Summary

Keep this PR open. Current main still has the unconditional outer compaction skip for tool-heavy slices with no nearby meaningful user turn, and manual `/compact` does not pass fresh persisted token usage into the compaction engine. The proposed change matches the linked bug shape and I found no blocking correctness or security findings in the patch context.

## What This Changes

The PR adds a reserve-threshold-aware no-real-conversation compaction guard, forwards fresh manual `/compact` token usage into `compactEmbeddedPiSession`, and adds focused tests for over-threshold tool-heavy slices and manual token forwarding.

## Best Possible Solution

Land a narrow fix that keeps idle heartbeat-only sessions cheap while allowing over-threshold, fresh-token sessions to reach the compaction engine even when the limited tail is tool-heavy.

## Reproduction Assessment

Yes. The linked issue gives a concrete reproduction shape for current main: a long Discord-thread session at 236k/200k tokens where the limited tail is mostly tool results and heartbeat content, and source inspection confirms that such a tail can hit the unconditional no-real-conversation skip.

## Solution Assessment

Yes. The PR uses the existing fresh session token contract and Pi reserve-token setting to bypass only at or above the compaction threshold, while preserving the existing skip when token usage is absent or below threshold.
