## Summary

Do not close this PR. Current main does not fully implement the PR’s intended fixes: wiki fallback for `memory_search`/`memory_get` is still missing when the memory-wiki supplement registration is absent, and plugin tool resolution still has a gateway-bindable active-registry reuse path that bypasses compatibility checks.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is to review or split the PR into the two concrete fixes: add a policy-respecting memory-wiki fallback through a supported public/plugin API, and update plugin tool resolution to use compatible registries or a guarded non-activating snapshot load with targeted regression tests. The package export and script additions should receive explicit maintainer review before landing.
