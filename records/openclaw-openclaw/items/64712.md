## Summary

Keep this PR open. Current main still has the channel summary fallback/default-account behavior and does not contain the PR guard or regression test. The proposed patch also is not yet a high-confidence fix for the linked Feishu report because Feishu account listing currently uses the shared helper that returns a synthetic default account when no configured accounts exist, and review feedback identifies a real risk of hiding status for explicit but unresolved channel configs.

## Best Possible Solution

Keep this PR open and request changes. The fix should distinguish authored or persisted account configuration from synthetic default-account fallbacks, preserve status diagnostics for explicit-but-unresolved configs, and add regression coverage that exercises the real Feishu/shared account-helper path for #64707.
