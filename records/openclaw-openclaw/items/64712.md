## Summary

Keep this PR open. Current main does not contain the proposed guard or regression test, and the patch is not yet a high-confidence fix for the linked Feishu report. In current Feishu code, `listFeishuAccountIds` is built with the shared helper that falls back to `default` when no configured accounts are present, so the PR's `accountIds.length > 0` guard would still treat the synthetic default as real. Review feedback also identifies a real regression risk for channels whose config exists but account IDs cannot be resolved yet.

## Best Possible Solution

Keep this PR open and request changes. The fix should distinguish authored or persisted account configuration from synthetic default-account fallbacks, preserve status diagnostics for explicit-but-unresolved configs such as SecretRefs, and add regression coverage against the real Feishu/shared account-helper path for #64707.
