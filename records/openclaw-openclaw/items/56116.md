## Summary

Keep open. Current main has Feishu streaming cards, static card notes, duplicate-final suppression, and transient body status lines, but it does not implement this PR’s central additions: localized status/last-activity footer updates, account-level statusLanguage config, long-task completion ping, or Feishu-specific Codex scratchpad filtering. The PR is open, unmerged, and unique, but needs revision before merge because the proposed scratchpad heuristic can suppress valid final replies and the new heartbeat timer is not cleaned up on all paths.

## Best Possible Solution

Keep this PR open. Rebase it onto current main, then revise the Feishu plugin implementation so scratchpad suppression cannot drop final user-facing replies and heartbeat cleanup runs on all dispatcher cleanup paths. The status footer, last activity, statusLanguage config, and long-task completion ping belong in the Feishu plugin with focused tests covering final-reply safety, note lifecycle cleanup, localization, and long-task completion behavior.
