## Summary

Keep open. Current main has Feishu streaming cards, notes, duplicate-final suppression, and transient tool/compaction status, but it does not implement this PR's central additions: localized status/last-activity footer updates, account-level statusLanguage config, long-task completion ping, or Feishu-specific Codex scratchpad filtering. The PR is active and relevant, but it needs revision before merge because review found a final-reply suppression bug and heartbeat cleanup leak in the proposed diff.

## Best Possible Solution

Keep this PR open, rebase it onto current main, then revise the implementation so scratchpad suppression cannot drop final user-facing replies and heartbeat cleanup runs on all dispatcher cleanup paths. The status footer/last activity/completion ping work belongs in the Feishu plugin, with focused tests covering final-reply safety, note lifecycle cleanup, statusLanguage config, and long-task completion behavior.
