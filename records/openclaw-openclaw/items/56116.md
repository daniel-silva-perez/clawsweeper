## Summary

Keep open. Current main still does not provide the PR's Feishu status/last-activity footer, localized statusLanguage setting, completion ping, or Codex scratchpad filtering, so the work is not obsolete; however, the proposed patch is not merge-safe because it can suppress legitimate final replies and can leak its heartbeat timer across cleanup.

## What This Changes

The PR adds Feishu streaming-card status and last-activity notes, account-level statusLanguage localization, long-task completion pings, public note updates, Codex scratchpad-text filtering, and focused Feishu tests.

## Best Possible Solution

Keep the Feishu-plugin implementation, but revise it so scratchpad suppression cannot drop final user-facing replies, heartbeat cleanup runs on every teardown path, the new account config is reflected in plugin metadata/docs if accepted, and the branch is rebased over current main before targeted Feishu validation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate scratchpad filtering off final replies:** `extensions/feishu/src/reply-dispatcher.ts:519-520`
  - body: The new `hasText` check runs `isLikelyCodexScratchpadText` before considering `info?.kind`, so a normal final answer from `openai-codex` can set `hasText` false. When there is no media, the deliver path returns without sending anything, leaving the Feishu user with no final reply; restrict the heuristic to non-final/internal chunks or provide a final fallback.
  - confidence: 0.93
- **[P2] Stop heartbeat timers during dispatcher cleanup:** `extensions/feishu/src/reply-dispatcher.ts:434`
  - body: `startHeartbeat()` creates an interval whenever streaming is attempted, including cases where the async start later fails. `closeStreaming()` stops it for idle/error paths, but `onCleanup` only delegates typing cleanup, so a torn-down dispatcher can keep the interval alive and continue queuing note refreshes.
  - confidence: 0.87
