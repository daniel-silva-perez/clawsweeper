## Summary

Keep open. Current main still awaits auth-profile failure marking before advancing to the next profile in both reported failover paths, and the same-author implementation PR that explicitly closes this issue remains open and unmerged.

## Best Possible Solution

Keep this issue open and review PR #57283 as the implementation candidate. The likely fix belongs in the embedded runner failover paths: advance the auth profile before any awaited cooldown marking, make cooldown marking non-blocking or bounded where appropriate, and add focused regression coverage proving rate-limit failover is not blocked by auth-store lock contention while cooldown state is still recorded eventually.
