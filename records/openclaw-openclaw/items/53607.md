## Summary

Keep this PR open. Current main has general outbound transcript mirroring, but it does not implement this PR’s intended richer Discord Components v2 mirroring: component labels/select options/modal trigger text are still not appended to the mirrored assistant transcript. The PR remains potentially useful, but it needs rebase/review and the security-sensitive transcript-meta design should be tightened before merge.

## Best Possible Solution

Keep the PR open as a valid implementation candidate, rebase it onto current main, and revise the approach so Discord-owned component summaries are bounded, explicitly marked as adapter-enriched, and merged without changing ordinary media transcript semantics. Add/keep focused regression tests for buttons, selects, modal trigger defaults, media-with-components, and no double-write behavior before considering merge.

## Risks / Open Questions

- The PR’s `meta.transcriptText` design is security-sensitive because `OutboundDeliveryResult.meta` is arbitrary channel/plugin-owned data; persisting it as assistant transcript text needs clear ownership, bounds, and injection-safe framing before merge.
- The PR diff’s `resolveMirroredTranscriptText` change appears to broaden caption-plus-media mirroring for ordinary media sends, not only enriched component sends, matching the unresolved review concern about transcript noise/regression.
- The linked bug issue is closed, but current main still lacks the PR’s enriched Components v2 transcript behavior, so closing this PR now would drop the remaining implementation candidate rather than recognizing shipped code.
