## Summary

Keep open. Current main still sends and registers Discord Components v2 messages without mirroring a component summary into the target session transcript, so the PR targets a valid bug. The branch is not merge-ready because it broadens captioned-media transcript behavior for all callers and introduces an unbounded, unlabeled adapter-metadata-to-assistant-transcript path that needs maintainer security/product review.

## What This Changes

The branch adds Discord component transcript summarization, returns that summary through outbound delivery result metadata, changes shared transcript mirroring to prefer adapter text, and adds Discord/outbound regression tests.

## Best Possible Solution

Land a narrow enriched-transcript implementation that mirrors Discord component text and interactive labels through one idempotent write path, gates text-plus-media merging to explicit enriched adapter summaries, preserves ordinary captioned-media mirror semantics, and bounds or labels UI-derived text before it becomes assistant session memory.

## Security Review

Status: needs_attention

Summary: The diff introduces a new adapter-result metadata-to-assistant-transcript path that needs bounded, labeled handling before merge.

Concerns:

- **[medium] Bound adapter transcript metadata before persistence:** `src/infra/outbound/deliver.ts:808`
  - body: `deliverOutboundPayloadsCore` would join `meta.transcriptText` from delivery results and persist it as assistant transcript text without a size or line cap. Because channel adapters/plugins can populate this metadata, the path should bound stored text before it can inflate session files or downstream context.
  - confidence: 0.78
- **[medium] Treat UI-derived transcript text as lower-trust context:** `src/infra/outbound/deliver.ts:816`
  - body: The PR stores component labels/options and adapter-provided transcript metadata as an assistant message. That may be acceptable for context recovery, but it should be explicitly labeled or constrained so UI-derived text is not silently promoted to high-authority assistant instructions.
  - confidence: 0.72
