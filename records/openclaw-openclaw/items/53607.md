## Summary

Keep open. Current main still lacks the PR's intended enriched Discord Components v2 transcript mirroring, and the branch is useful source work for the closed linked bug report, but it needs focused maintainer/security review before merge because it turns generic adapter metadata into persisted assistant transcript text and changes shared media mirroring semantics.

## What This Changes

The PR adds a Discord component transcript summary helper, attaches that summary as `meta.transcriptText` in Discord outbound payload results, teaches shared outbound mirroring to prefer adapter-provided transcript text, updates media mirror resolution, and adds Discord/outbound regression tests.

## Best Possible Solution

Land a narrow implementation that preserves Discord component button/select/modal context in the target session transcript through an explicit enriched-transcript path, with size limits, provenance/labeling for UI-derived text, single-write idempotency, and media filename merging limited to enriched component sends.
