## Summary

Keep open. Current main has internal stable heartbeat phasing and cron-expression staggering, but the requested public scheduling surface is still missing: no prime cadence mode, no configurable heartbeat phase/jitter controls, no resolved phase/status exposure, and cron `every` remains a fixed interval without stagger/jitter support.

## Best Possible Solution

Keep this open as a scheduler/API design item. Preserve the current internal stable heartbeat phase, then decide backward-compatible public controls for heartbeat phase/jitter, optional prime cadence, and cron `every` jitter/prime semantics. If accepted, implement the contract through config normalization, Gateway protocol/schema, CLI, Control UI/status output, docs, and focused scheduler tests.
