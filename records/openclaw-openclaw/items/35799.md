## Summary

Keep open. Current main has partial anti-herd mitigation through internal stable heartbeat phasing and cron-expression staggering, but the requested public scheduling surface is still missing: no prime cadence mode, no configurable heartbeat phase/jitter controls, no resolved heartbeat phase/status exposure, and cron `every` remains a fixed interval without stagger/jitter support.

## Best Possible Solution

Keep this open as a deliberate scheduler/API design item. Preserve the existing internal stable heartbeat phase, then decide backward-compatible public controls for heartbeat phase/jitter and optional prime cadence. If accepted, carry the semantics through cron `every`, Gateway protocol/schema, normalizer/store migration, CLI, Control UI/status output, docs, and tests with resolved schedule observability.
