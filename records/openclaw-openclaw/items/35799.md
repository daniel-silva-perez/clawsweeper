## Summary

Keep open. Current main has internal stable per-agent heartbeat phasing and deterministic cron-expression staggering, but the requested public scheduling contract is still missing: no prime cadence mode, no configurable heartbeat phase/jitter fields, no resolved phase/status exposure, and cron `every` remains a fixed interval with CLI/protocol/tests still treating stagger as cron-only.

## Best Possible Solution

Keep this open as a scheduler/API design item. Preserve the current internal stable heartbeat phase, then decide additive public controls for heartbeat phase/jitter, optional prime cadence, and cron `every` jitter/prime semantics. If accepted, implement the contract through config normalization/types/help, Gateway protocol/schema, CLI, status or Control UI exposure, docs, and focused heartbeat/cron scheduler tests.
