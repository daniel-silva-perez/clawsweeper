## Summary

Keep open. Current main still shows the reported unquoted bracket-notation `openclaw config` examples in the relevant docs, and a shell reproduction still fails before OpenClaw receives the argument. The underlying config path is valid once quoted or written with dot-index notation, so this remains an actionable docs shell-quoting bug; related PR #40642 proposed the right docs-only direction but was closed unmerged.

## Best Possible Solution

Land a small docs-only fix that quotes bracket-notation config paths in `docs/cli/config.md`, `docs/tools/exec.md`, and `docs/nodes/index.md`, or switch those examples to shell-safe dot-index paths such as `agents.list.0.tools.exec.node`. Add a short note in the config paths docs that paths containing `[]` need shell quoting, then let the docs i18n pipeline update localized output.
