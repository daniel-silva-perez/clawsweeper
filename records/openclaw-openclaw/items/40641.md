## Summary

Current main still contains the reported shell-unsafe docs examples. The config path itself is valid once OpenClaw receives it, but unquoted bracket notation is still rejected by zsh/failglob-style shells before the CLI can parse it, so this remains an actionable docs bug rather than a close candidate.

## Best Possible Solution

Keep this issue open and land a docs-only fix in the English source docs: quote bracket-notation paths such as `agents.list[0].tools.exec.node`, or switch examples to shell-safe dot-index paths such as `agents.list.0.tools.exec.node`. Update `docs/cli/config.md`, `docs/tools/exec.md`, and `docs/nodes/index.md`, add a short Paths note that `[]` paths need shell quoting, and let the docs i18n pipeline handle localized output.
