## Summary

Current main still contains the reported unquoted bracket-notation config examples in the English docs, and a failglob-style shell still rejects the argument before OpenClaw can parse it. The underlying CLI parser and config type path are valid once the path is quoted or written as dot-index notation, so this remains an actionable docs bug. Related PR #40642 proposed the right docs-only fix but was closed unmerged.

## Best Possible Solution

Land a docs-only fix that quotes bracket-notation config paths in `docs/cli/config.md`, `docs/tools/exec.md`, and `docs/nodes/index.md`, or switch those examples to shell-safe dot-index paths such as `agents.list.0.tools.exec.node`. Add a short note in the config Paths section that paths containing `[]` need shell quoting, then let the docs i18n pipeline handle localized output.
