## Summary

Keep open. Current main still contains the reported unquoted bracket-notation `openclaw config` examples, and a shell-only reproduction still fails before OpenClaw receives the argument. The underlying config path and CLI parser are valid once the argument is quoted or written as a dot-index path, so this remains an actionable docs shell-quoting regression. Related PR #40642 proposed the right fix but was closed unmerged.

## Best Possible Solution

Keep this issue open and land a small docs-only fix: quote bracket-notation config paths in `docs/cli/config.md`, `docs/tools/exec.md`, and `docs/nodes/index.md`, or switch examples to shell-safe dot-index paths like `agents.list.0.tools.exec.node`. Add a short note that paths containing `[]` must be quoted in shells that treat brackets as glob patterns, then let the docs i18n pipeline update zh-CN output.
