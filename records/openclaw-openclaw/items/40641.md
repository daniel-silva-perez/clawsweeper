## Summary

Keep open. Current main still contains the reported unquoted bracket-notation `openclaw config` examples in the English source docs, and the argument still fails in failglob-style shells before OpenClaw receives it. The underlying config path and CLI parser are valid once the path is quoted or written as dot-index notation, so this remains a small actionable docs bug rather than an implemented or stale item. Related PR #40642 proposed the right docs-only direction, but it was closed unmerged.

## Best Possible Solution

Land a docs-only fix that quotes bracket-notation config paths in `docs/cli/config.md`, `docs/tools/exec.md`, and `docs/nodes/index.md`, or switch those examples to shell-safe dot-index paths such as `agents.list.0.tools.exec.node`. Add a short note in the config paths section that paths containing `[]` need shell quoting, then let the docs i18n pipeline update localized output.
