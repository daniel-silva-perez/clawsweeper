## Summary

Keep open. Current main still documents the reported unquoted bracket-notation config path in the config, exec, and nodes docs; failglob-style shells reject that argument before OpenClaw can parse it. The CLI parser supports bracket paths once they reach argv, so this is still an actionable docs bug rather than an implemented fix.

## Best Possible Solution

Keep this issue open and land a docs-only fix in the English source docs. Quote bracket-notation path arguments such as `agents.list[0].tools.exec.node`, or switch examples to shell-safe dot-index paths such as `agents.list.0.tools.exec.node`; add a short Paths note that bracket notation must be shell-quoted; update `docs/cli/config.md`, `docs/tools/exec.md`, and `docs/nodes/index.md`; then let the docs i18n pipeline regenerate localized output.
