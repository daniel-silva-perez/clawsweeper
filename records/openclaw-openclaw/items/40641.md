## Summary

Keep open. Current main still documents unquoted bracket-notation config paths such as `agents.list[0].tools.exec.node`, and a failglob-style shell rejects that argument before OpenClaw receives it. The CLI and config schema support the path once quoted, so this remains an actionable docs bug rather than an implemented fix.

## Best Possible Solution

Land a docs-only fix in the English source docs: quote bracket-notation paths such as `agents.list[0].tools.exec.node`, or switch the examples to shell-safe dot-index paths such as `agents.list.0.tools.exec.node`. Add a short note in the `openclaw config` Paths section that paths containing `[]` need shell quoting; let the docs i18n pipeline handle localized output.
