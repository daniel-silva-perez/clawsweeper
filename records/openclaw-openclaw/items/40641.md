## What This Changes

The issue asks OpenClaw to fix official docs examples for `openclaw config set agents.list[0].tools.exec.node ...` because copied bracket-notation paths fail in common shells.

## Best Possible Solution

Land a docs-only fix in the English source docs: quote bracket-notation path arguments such as `'agents.list[0].tools.exec.node'` or use shell-safe dot-index paths such as `agents.list.0.tools.exec.node`, add a short Paths note explaining bracket notation must be shell-quoted, and let the docs localization pipeline regenerate translated output.

## Solution Assessment
