## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Avoid defaulting unknown Discord DMs to one-to-one routing:** `extensions/discord/src/monitor/message-handler.preflight.ts:295-310`
  - body: When channel metadata lookup fails and `guild_id` is absent, the PR forces `ChannelType.DM`. Discord group DMs also omit `guild_id`, so this can bypass group-DM policy and collapse distinct group chats into direct-user sessions.
  - confidence: 0.86
- **[P1] Ignore message-type values in Discord channel fallback:** `extensions/discord/src/monitor/message-utils.ts:129-139`
  - body: `message.rawData.type` is a Discord message type, not a channel type. Using it as `ChannelType` can make a real DM look non-DM when REST metadata is missing, which directly regresses the DM fallback being added.
  - confidence: 0.9
- **[P2] Guard mirror registration when agent delivery is already enabled:** `src/gateway/server-methods/agent.ts:509-517`
  - body: For `agent` RPC calls with `deliver: true`, normal outbound delivery already sends the final reply. Registering `webchatMirrorTarget` as well makes the lifecycle handler mirror the same text a second time.
  - confidence: 0.84
- **[P1] Preserve stored routes for main and per-peer DM sessions:** `src/gateway/server-methods/webchat-mirror.ts:18-27`
  - body: The mirror target resolver bails out before consulting stored delivery context for channel-agnostic session heads like `main` and supported per-peer direct keys. That drops mirroring for common DM session layouts even when the session entry has a valid external route.
  - confidence: 0.86
- **[P2] Preserve colon-containing peer IDs in mirror targets:** `src/gateway/server-methods/webchat-mirror.ts:120-160`
  - body: Splitting the session suffix on `:` truncates IDs such as Matrix room IDs (`!room:example.org`). The rebuilt target can point to `channel:!room` instead of the real room, so mirrored WebChat replies fail or go to the wrong destination.
  - confidence: 0.82
- **[P2] Trim command text before mirroring it externally:** `src/gateway/server-methods/chat.ts:1375-1385`
  - body: The PR skips mirroring only when raw text literally starts with `/`, while command detection trims whitespace first. Inputs like ` /help` can be executed as local commands and still leaked into the external chat.
  - confidence: 0.88
- **[P2] Strip nested Discord files from payload JSON:** `extensions/discord/src/monitor/rest-fetch.ts:82-85`
  - body: The proxied multipart serializer supports `body.data.files`, but only removes top-level `files` before serializing `payload_json`. Nested file descriptors can be embedded in JSON, producing oversized or invalid Discord multipart uploads.
  - confidence: 0.9
- **[P2] Use a compatible agent for Baileys fetchAgent:** `extensions/whatsapp/src/session.ts:128-130`
  - body: The PR casts an undici `ProxyAgent` to a Node `https.Agent` for Baileys `fetchAgent`. If Baileys uses Node HTTP APIs on that path, proxy-enabled WhatsApp sessions can fail at runtime because undici dispatchers do not implement native agent methods.
  - confidence: 0.8
- **[P1] Keep Matrix shutdown persistence intact:** `extensions/matrix/src/matrix/monitor/index.ts:131-145`
  - body: Replacing `releaseSharedClientInstance(client, "persist")` with a plain stop path drops the Matrix `stopAndPersist()` behavior. Graceful restarts can then look like cold starts and miss backlog replay.
  - confidence: 0.86
- **[P1] Keep Matrix binding setters on the active registry:** `extensions/matrix/src/matrix/thread-bindings.ts:60-120`
  - body: The PR moves manager state into a new registry while existing exported setters still target the old shared registry. Session lifecycle commands that set idle/TTL can update an empty map instead of live Matrix bindings.
  - confidence: 0.84
- **[P2] Restore removed public account-resolution exports:** `src/plugin-sdk/account-resolution.ts:10-16`
  - body: `openclaw/plugin-sdk/account-resolution` is a public SDK subpath. Removing Discord, Slack, Telegram, and Signal account resolver exports can break third-party plugins that import those helpers after upgrade.
  - confidence: 0.82
