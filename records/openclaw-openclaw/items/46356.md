## Summary

Keep this PR open, but do not merge it as-is. Current main does not already contain the proposed WebChat mirror or WhatsApp proxy support, while the PR branch introduces several concrete routing, Matrix lifecycle, command-leak, and dependency-surface regressions that need maintainer-led splitting or replacement.

## What This Changes

The PR branch adds WebChat-to-channel mirroring, Discord DM/proxy REST handling, WhatsApp proxy configuration and docs, Matrix lifecycle/thread-binding refactors, SDK export changes, Tlon dependency cleanup, and related tests.

## Best Possible Solution

Use this branch as source material, but land a focused gateway/session fix for the Control UI external-reply routing bug first, then handle Discord proxy hardening, WhatsApp proxy config, and Matrix lifecycle/dependency work in separate owner-reviewed PRs with targeted tests and live or near-live channel proof where feasible.

## Reproduction Assessment

Yes for the review blockers: the PR diff and existing current-main code provide static reproduction paths for the Discord metadata fallback, whitespace-prefixed slash command mirroring, duplicate delivered-run mirroring, and Matrix persistence/registry regressions. ## Solution Assessment

No. The direction is useful, but this PR is too broad and introduces regressions; the safer solution is a narrow WebChat routing change followed by channel-owned proxy and Matrix follow-up PRs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Do not default metadata misses to direct DMs:** `extensions/discord/src/monitor/message-handler.preflight.ts:309-312`
  - body: When channel lookup fails and guild_id is absent, this defaults to ChannelType.DM. Discord group DMs also omit guild_id, so this can bypass group-DM policy and collapse group conversations into direct-user sessions.
  - confidence: 0.86
- **[P1] Ignore message type when deriving channel type:** `extensions/discord/src/monitor/message-utils.ts:132-139`
  - body: message.rawData.type is a Discord message type, not a channel type. Using it as ChannelType can make real DMs look non-DM or apply arbitrary channel semantics when REST metadata is unavailable.
  - confidence: 0.9
- **[P2] Skip mirror registration for delivered agent runs:** `src/gateway/server-methods/agent.ts:512-519`
  - body: For agent RPC calls with deliver: true, normal outbound delivery already sends the final reply. Registering webchatMirrorTarget here makes the lifecycle handler mirror the same text again.
  - confidence: 0.84
- **[P2] Preserve full peer IDs when rebuilding mirror targets:** `src/gateway/server-methods/webchat-mirror.ts:94-107`
  - body: Splitting the session key on every colon truncates peer IDs such as Matrix room IDs. The rebuilt target can point to channel:!room instead of the full room ID, so mirrored replies can fail or go to the wrong destination.
  - confidence: 0.82
- **[P2] Trim command text before mirroring externally:** `src/gateway/server-methods/chat.ts:1378`
  - body: This guard checks rawMessage.startsWith("/") before command detection trims whitespace. Inputs like " /help" can execute as local commands and still be mirrored to the external chat.
  - confidence: 0.88
- **[P1] Preserve Matrix sync persistence on shutdown:** `extensions/matrix/src/matrix/monitor/index.ts:134-143`
  - body: Replacing the persist release path with stopSharedClientInstance drops Matrix stopAndPersist behavior. Graceful restarts can then look like cold starts and miss backlog replay.
  - confidence: 0.86
- **[P1] Keep Matrix lifecycle setters on the live registry:** `extensions/matrix/src/matrix/thread-bindings.ts:65-66`
  - body: This file creates a new local manager registry, while existing exported lifecycle setters still use the shared registry. Session lifecycle commands can update an empty registry instead of the active Matrix bindings.
  - confidence: 0.84
- **[P2] Restore public account-resolution exports:** `src/plugin-sdk/account-resolution.ts:10-19`
  - body: The published account-resolution SDK subpath is reduced by this change. Existing external plugins importing Discord, Slack, Telegram, or Signal account resolvers from that entrypoint can fail after upgrade.
  - confidence: 0.78
- **[P2] Strip nested files before serializing Discord payloads:** `extensions/discord/src/monitor/rest-fetch.ts:82-85`
  - body: The multipart proxy path clears top-level files but leaves data.files in payload_json for nested upload payloads, which can serialize attachment descriptors or binary-like objects into the JSON body.
  - confidence: 0.78
