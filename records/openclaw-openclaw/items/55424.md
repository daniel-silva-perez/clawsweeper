## Summary

Keep open. Current main still has the reported false warning path: mutating write/edit tool errors are surfaced as user-facing warnings without a response-timeout exception, and the latest release snapshot still contains a test case expecting `write` plus `connection timeout` to show a warning. The linked fix PR was closed unmerged, while the later related issue appears to be a duplicate of this original report rather than a canonical fix.

## What This Changes

The issue asks OpenClaw to stop showing Write/Edit failed warnings when a write/edit tool response times out after the file was actually saved successfully.

## Best Possible Solution

Keep this as the canonical open bug and land a narrow agent payload-policy fix that suppresses false mutating file-tool warnings only for response/ack timeout errors, while preserving visible warnings for actual write/edit failures and adding regression coverage for both sides.

Boundary: keep the fix in the agent payload/tool-error warning path. Prefer a small helper that recognizes response/ack timeout errors from structured `lastToolError.timedOut` and known messages such as `invoke timed out`, `Response timeout`, and `connection timed out waiting for response`.
