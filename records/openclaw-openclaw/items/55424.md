## Summary

Keep open. Current main still has the reported false-warning class: write/edit remain mutating tools, timeout metadata is recorded but the shared warning policy does not use it, and current tests still expect a write connection-timeout warning. The linked fix PR is closed and unmerged.

## What This Changes

The issue requests that OpenClaw stop showing write/edit failure warnings when file writes actually completed successfully and only the tool response acknowledgement timed out.

## Best Possible Solution

Add a narrow timeout-aware suppression in the shared agent payload warning policy for write/edit response-ack timeout errors, while preserving warnings for real filesystem/edit failures and covering both `timedOut` metadata and known response-timeout message shapes with focused tests.
