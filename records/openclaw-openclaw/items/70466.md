## Summary

Keep this PR open. Current main still does not implement the PR's intended runtime behavior: both the update.run supervised restart branch and the generic supervised SIGUSR1 restart branch exit with code 0. The packaged systemd unit now uses Restart=always, which mitigates OpenClaw-managed services, but that is not the same as this PR's proposed runtime exit-code behavior for supervised systemd restarts.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is either to update it for current main so both generic supervised SIGUSR1 restarts and update.run supervised restarts get the intended systemd non-zero exit behavior with focused non-launchd tests, or to close it only after an explicit maintainer decision that Restart=always fully supersedes the runtime exit-code change.
