## Summary

Keep this PR open. Current main does not implement the PR's intended runtime exit-code change: supervised gateway restart paths still call `exitProcess(0)`. The linked issue was closed because the canonical OpenClaw-managed systemd unit now uses `Restart=always`, but that is a different mitigation and leaves this PR's proposed supervised `exit(1)` behavior as an unresolved maintainer decision.

## Best Possible Solution

Keep this PR open for maintainer review rather than closing it as implemented. The best path is either to update the PR for current main so both generic supervised SIGUSR1 restarts and `update.run` supervised restarts exit non-zero on systemd with focused tests, or to close it after an explicit maintainer decision that the canonical `Restart=always` unit fully supersedes the runtime exit-code change.
