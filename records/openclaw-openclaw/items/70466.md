## Summary

Keep this PR open. The PR is aimed at a real gateway restart failure mode, but its current diff is stale/incomplete against current main: the update.run supervised restart path still exits 0, and the generic supervised SIGUSR1 path on main also still exits 0. The packaged systemd unit now uses Restart=always, which mitigates OpenClaw-managed installs, but that is not the same runtime behavior this PR proposes for Restart=on-failure units.

## What This Changes

The PR splits spawned and supervised gateway restart handling so spawned restarts exit 0 while supervised restarts exit 1 after the launchd delay, and updates the supervised launchd restart test to expect exit code 1.

## Best Possible Solution

Update this PR or replace it with a narrow current-main patch that makes systemd-supervised update.run and generic SIGUSR1 restarts exit non-zero, keeps spawned handoffs at exit 0, preserves launchd delay semantics, and adds focused non-launchd/systemd tests for both restart reasons.

## Reproduction Assessment

Yes. The code path is reproducible by forcing a systemd-marker supervised respawn after an update.run or authorized SIGUSR1 restart: current main reaches supervised mode and exits 0, and systemd Restart=on-failure does not restart clean exits.

## Solution Assessment

No, not as currently proposed against main. The patch handles the old generic supervised branch but misses the newer update.run branch and should be narrowed or tested around systemd-specific behavior before merge.
