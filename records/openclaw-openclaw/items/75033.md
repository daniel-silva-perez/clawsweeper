## Summary

Keep this PR open. The change is on the right shared-redaction path and is narrowly scoped, but the new object-diagnostic regex misses common camelCase HTTP secret keys such as `apiKey`, `accessToken`, and `refreshToken`, so the patch should be updated before merge.

## What This Changes

The PR expands default logging redaction patterns for HTTP-client config/header fields and adds regression coverage for `redactSensitiveText` and `formatErrorMessage` cause-chain output.

## Best Possible Solution

Extend the new object-diagnostic redaction pattern and tests to cover the common camelCase aliases alongside snake/kebab case, then run the targeted redaction/error tests plus formatting before maintainer/security review.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a util.inspect-style string such as `config: { apiKey: '...' }` or `config: { accessToken: '...' }`; the PR's new regex does not match those names, while the affected formatter path returns shared-redacted text.

## Solution Assessment

No, not as-is. Shared default redaction is the right maintainable seam, but this implementation should cover the common camelCase token aliases before it is considered complete.
