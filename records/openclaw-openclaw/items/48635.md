## Summary

Keep this PR open. Current main still does not implement the PR's central behavior: the per-response `/usage` footer formats usage/cache/cost/session data only, and `agents.defaults` still has no `responseUsageDefault` config key. The PR remains a plausible implementation candidate for #12578 and the model-name portion of #40215/#65477, but the existing review comments identify correctness issues that need a rebase and cleanup before merge.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is to rebase onto current main, move the formatter change into `src/auto-reply/reply/agent-runner-usage-line.ts`, add typed/schema/generated-doc support for `agents.defaults.responseUsageDefault`, preserve an explicit session-level off override, wire model and resolved run thinking level into the footer under a tested `tokens` versus `full` contract, update `/usage` docs/tests, and remove the unrelated log artifact.

## Risks / Open Questions

- Closing now would discard a focused implementation candidate while #12578, #40215, and #65477 still track related unmet behavior on current main.
- As drafted, the PR can make `/usage off` ineffective when a non-off config default exists, because current command handling deletes the session value to represent off.
- The PR needs a tested mode contract for whether model/thinking appears only in `full` mode or also in `tokens` mode.
- The PR should use the resolved run thinking level, not only `activeSessionEntry?.thinkingLevel`, so default-driven sessions are reported accurately.
- The unrelated root log artifact should be removed before merge.
