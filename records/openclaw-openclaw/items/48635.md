## Summary

Keep this PR open. Current main still does not implement the PR’s central behavior: the response usage footer does not show model or thinking metadata, and there is no `agents.defaults.responseUsageDefault` config surface. The PR is still a relevant implementation candidate for #12578 and part of #65477/#40215, but it needs a rebase and correctness fixes before it should land.

## Best Possible Solution

Keep this PR open as a useful implementation candidate. The best path is to rebase it on current main, remove the root log artifact, add a typed and schema-backed `agents.defaults.responseUsageDefault`, preserve explicit session `/usage off` precedence, wire model and resolved run thinking level into the footer according to a tested mode contract, and update the usage docs/tests. That would directly address #12578 and the model-footer part of #65477/#40215 without closing the broader context-usage requests prematurely.

## Risks / Open Questions

- As drafted, the config fallback using nullish coalescing conflicts with current `/usage off` semantics because `handleUsageCommand` deletes `responseUsage` for off; with a non-off default, that can re-enable the footer instead of preserving a session-level off override.
- The PR’s thinking footer uses `activeSessionEntry?.thinkingLevel`, which can miss or misreport the resolved run thinking level when defaults or provider fallback are involved.
- The PR needs a product/test decision on whether model and thinking metadata belong only in `full` mode or also in `tokens` mode; existing review comments flagged this contract mismatch.
- The PR branch includes an unrelated root log file with local runtime paths, which is not appropriate to merge even though it is not a dependency or workflow supply-chain change.
- The code has moved enough that the PR body’s referenced formatter path is stale; a rebase should update the implementation to the current `agent-runner-usage-line.ts` structure.
