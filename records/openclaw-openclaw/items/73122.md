## Summary

Keep open. Current main does not implement this PR's central harness-registration change: CLI backend registration still only populates the CLI backend registry, while harness selection maps CLI runtime aliases away from embedded harness selection. Main does cover some of the observable routing/startup behavior through the separate CLI path and installed-plugin startup index, so this PR needs maintainer design review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to choose one contract: either keep current main's separate CLI-backend model and ask for narrower regression tests around CLI routing/startup activation, or intentionally promote CLI backends into harnesses and update the docs, harness semantics, and lifecycle/replay tests in the same change.
