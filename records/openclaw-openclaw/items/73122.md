## Summary

Keep this draft PR open. Current main does not implement the PR's central change of registering CLI backends such as `claude-cli` as same-id embedded agent harnesses; main instead preserves a separate CLI-backend execution path and documents that `claude-cli` must not be passed to AgentHarness selection.

## Best Possible Solution

Keep the PR open for maintainer design review. The best path is to choose and document one contract: either keep main's separate CLI-backend model and ask for narrower regression tests around CLI routing/startup activation, or intentionally promote CLI backends into harnesses and update docs, harness semantics, and lifecycle/replay tests in the same change.
