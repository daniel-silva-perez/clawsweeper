## Summary

Keep this issue open. Current main still does not filter imsg RPC tapback reactions before dispatch, and the provided context plus remote ref check show an open implementation candidate, #60677, that explicitly closes this issue but has not merged.

## Best Possible Solution

Keep the issue open and use #60677, or an equivalent maintainer-reviewed patch, as the implementation path. The narrow fix belongs in extensions/imessage: add typed optional imsg reaction/tapback fields, preserve and read them in parse/decision code, drop tapbacks before dispatch, and add regression tests for explicit metadata, associated_message_type ranges, Loved/Liked/Laughed/Emphasized/Questioned text, removal patterns, and false-positive prevention. Leave full reaction-event forwarding to #39031.
