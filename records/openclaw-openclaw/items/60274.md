## Summary

Keep this issue open. Current main still lacks imsg RPC tapback filtering: the bundled iMessage plugin parses ordinary message fields, subscribes with only the attachments option, and dispatches allowed non-empty inbound text without a tapback/reaction gate. The provided GitHub context also shows open PR #60677 explicitly closing this issue, so the right cleanup path is to review or land that PR rather than close the issue early.

## Best Possible Solution

Keep the issue open and use #60677 as the active implementation candidate. The narrow fix belongs in the bundled imessage plugin: add typed optional imsg reaction/tapback fields, preserve them in parse-notification, filter tapbacks in resolveIMessageInboundDecision before dispatch, and add regression tests for explicit metadata, associated_message_type ranges, quoted Loved/Liked/Laughed/Emphasized/Questioned text, removal patterns, and false-positive prevention. Leave full reaction-event forwarding to #39031.
