## Summary

Keep this PR open. Current `main` and the latest release still lack the proposed segmented-final flush, so the branch is not obsolete, but the proposed early-return flush would bypass the existing preview-finalized `message:sent` hook path because `flushBufferedFinalAnswer()` discards the `LaneDeliveryResult`.

## What This Changes

This PR adds a Telegram dispatcher flush for buffered final-answer text before the segmented final-delivery early return, adds a skipped-reasoning regression test, and adds a changelog fix entry.

## Best Possible Solution

Update this PR or land an equivalent Telegram-owned patch so `flushBufferedFinalAnswer()` captures or returns the `LaneDeliveryResult` and routes `preview-finalized` results through `emitPreviewFinalizedHook()`, while preserving the missing final-answer flush, compaction-boundary handling, focused tests, and changelog entry.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Emit the hook when flushing buffered final answers:** `extensions/telegram/src/bot-message-dispatch.ts:703`
  - body: The new segmented-final early return calls `flushBufferedFinalAnswer()` and exits, but that helper discards the `deliverLaneText()` result. When the buffered answer finalizes an existing preview, Telegram shows the answer but `emitPreviewFinalizedHook(...)` never runs, so `sessionKeyForInternalHooks` flows miss the internal `message:sent` hook unlike normal final segment delivery. Capture the result and route preview-finalized deliveries through the hook before resetting state.
  - confidence: 0.88
