## Summary

Keep open. Current main and the latest release still return from Telegram segmented final delivery without flushing a buffered final answer, so this PR's intended bug fix is not obsolete. It should still get maintainer review because the proposed flush path appears to need the same preview-finalized `message:sent` hook handling as the normal final segment path.

## What This Changes

This PR adds a Telegram dispatcher flush for buffered final-answer text before the segmented final-delivery early return, adds a regression test for skipped reasoning after a tool boundary, and adds a changelog fix entry.

## Best Possible Solution

Land this PR or an equivalent Telegram-owned patch after routing flushed buffered final-answer deliveries through the same preview-finalized hook handling as normal final segment deliveries, with scope limited to Telegram dispatch, its focused regression coverage, and the user-facing changelog entry.
