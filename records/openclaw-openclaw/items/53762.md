## Summary

Keep this PR open: it targets a real Telegram partial-streaming message-loss path that is still present on current main and in the latest release, but the proposed patch still has a P2 correctness issue because its new flush path can skip the internal preview-finalized message hook.

## What This Changes

This PR adds a Telegram dispatcher flush for buffered final-answer text before the segmented final-delivery early return, plus a focused regression test and changelog entry.

## Best Possible Solution

Land this PR or an equivalent Telegram-owned patch after the buffered-final-answer flush captures or returns the LaneDeliveryResult and sends preview-finalized results through emitPreviewFinalizedHook(), while preserving current-main compaction-boundary handling and keeping the regression coverage focused on the skipped-reasoning path.

## Reproduction Assessment

Yes. The PR and linked report give a concrete two-turn Telegram partial-streaming scenario, and current main statically shows the buffered final answer can remain pending when reasoning delivery is skipped and the segmented final branch returns without flushing it.

## Solution Assessment

No, not as currently proposed. The missing flush is the right narrow fix for the dropped answer, but the implementation should preserve the same preview-finalized hook behavior as normal final segment delivery.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Emit the hook when flushing buffered final answers:** `extensions/telegram/src/bot-message-dispatch.ts:703`
  - body: The new final-segment early return calls `flushBufferedFinalAnswer()` and exits, but that helper discards the `deliverLaneText()` result. When the buffered answer finalizes an existing preview, Telegram shows the answer but `emitPreviewFinalizedHook(...)` is never run, so `sessionKeyForInternalHooks` flows miss the internal `message:sent` hook unlike normal final segment delivery. Capture the result and route preview-finalized deliveries through the hook before resetting state.
  - confidence: 0.88
