## Summary

Keep this PR open. Current main and the latest release still lack the proposed Telegram final-answer flush, so the branch is not obsolete, but the current patch introduces an actionable hook-preservation bug before merge.

## What This Changes

This PR adds a Telegram dispatcher flush for buffered final-answer text before the segmented final-delivery early return, adds a skipped-reasoning regression test, and adds a changelog fix entry.

## Best Possible Solution

Land this PR or an equivalent Telegram-owned patch after the buffered-final-answer flush captures or returns the LaneDeliveryResult and routes preview-finalized results through emitPreviewFinalizedHook(), while preserving the missing final-answer delivery fix, focused regression coverage, and changelog entry.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Emit the hook when flushing buffered final answers:** `extensions/telegram/src/bot-message-dispatch.ts:703`
  - body: The new segmented-final early return calls `flushBufferedFinalAnswer()` and exits, but that helper discards the `deliverLaneText()` result. When the buffered answer finalizes an existing preview, Telegram shows the answer but `emitPreviewFinalizedHook(...)` never runs, so `sessionKeyForInternalHooks` flows miss the internal `message:sent` hook unlike normal final segment delivery.
  - confidence: 0.88
