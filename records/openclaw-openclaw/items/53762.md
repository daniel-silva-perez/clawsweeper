## Summary

Keep this PR open. Current main still has the segment-processed final path that can buffer an answer after skipped reasoning and return without flushing it, so the PR is not obsolete. The proposed change also needs the existing review comment addressed so buffered preview-finalized answers still emit the internal message-sent hook.

## Best Possible Solution

Keep the PR open and update it, or replace it with an equivalent patch, so the final segment early-return path flushes buffered final answers while preserving `emitPreviewFinalizedHook` behavior. Add regression coverage for skipped reasoning after a tool boundary, including the hook case when `sessionKeyForInternalHooks` is present, then run the targeted Telegram dispatch tests and the appropriate changed gate.
