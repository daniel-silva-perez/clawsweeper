## Summary

Keep this PR open. Current main has partial bundled Groq support, but it still does not implement the PR’s central user-facing change: Groq is not exposed as a `groq-api-key` onboarding/auth choice, `--groq-api-key` is not generated, and there is no Groq text model catalog/default-model setup path on the bundled Groq plugin.

## Best Possible Solution

Keep this PR open for rework or replacement against current main. The implementation should move primarily into `extensions/groq`: add plugin-owned `providerAuthChoices` metadata for `groq-api-key` and `--groq-api-key`, runtime API-key auth metadata through the provider plugin helper, a Groq text `modelCatalog` or catalog registration, default model setup for `groq/llama-3.3-70b-versatile`, and focused tests for picker visibility, non-interactive flag inference/forwarding, env-ref storage, auth profile creation, and catalog behavior. Drop the unrelated secret-scanning baseline and allowlist-comment changes.
