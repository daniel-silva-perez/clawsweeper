## Summary

Keep this PR open. Current main has partial bundled Groq support, but it still does not implement the PR’s central onboarding/auth UX: no `groq-api-key` auth choice, no `--groq-api-key` flag, no Groq text model catalog/default setup, and no runtime API-key auth method on the Groq provider plugin.

## Best Possible Solution

Keep this PR open for rework or replace it with a fresh PR against current main. The implementation should live primarily in `extensions/groq`: add plugin-owned `providerAuthChoices` metadata for `groq-api-key` and `--groq-api-key`, runtime auth metadata through the provider plugin helper, a Groq text `modelCatalog`/catalog registration, default model setup for `groq/llama-3.3-70b-versatile`, and focused tests for interactive picker visibility, non-interactive flag inference/forwarding, env-ref storage, auth profile creation, and catalog behavior. Drop the unrelated secret-scanning baseline and allowlist-comment churn.
