## Summary

Keep this PR open. Current main has bundled Groq support for provider ownership, docs, env-var metadata, media-understanding, and reasoning compatibility, but it still does not implement the PR's central onboarding/auth UX: no `groq-api-key` auth choice, no `--groq-api-key` flag, no Groq text model catalog/default setup, and no plugin-owned API-key auth method.

## Best Possible Solution

Keep this PR open for maintainer or contributor rework, or replace it with a fresh PR against current main. The best implementation belongs in `extensions/groq`: add plugin-owned `providerAuthChoices` metadata for `groq-api-key` and `--groq-api-key`, runtime auth metadata via the provider plugin helper, Groq text `modelCatalog`/catalog registration, default model setup for `groq/llama-3.3-70b-versatile`, and focused tests for interactive picker visibility, non-interactive flag inference/forwarding, env-ref storage, auth profile creation, and catalog behavior. Drop the unrelated secret-scanning baseline and allowlist-comment churn.
