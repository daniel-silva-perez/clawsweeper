## Summary

Keep this PR open. Current main has partial bundled Groq support through a plugin, docs, env-var metadata, media-understanding registration, and reasoning compatibility, but it still does not implement the PR's central user-visible onboarding/auth behavior: `groq-api-key`, `--groq-api-key`, an interactive text-provider picker entry, or a Groq text model catalog/default setup path.

## Best Possible Solution

Keep the PR open for maintainer or contributor rework, or replace it with a fresh PR against current main. The implementation should live in `extensions/groq`: add plugin-owned `providerAuthChoices` metadata for `groq-api-key`/`--groq-api-key`, a Groq `modelCatalog` and runtime/static catalog for text models, API-key auth via the current provider plugin helper, default-model config for `groq/llama-3.3-70b-versatile`, and focused tests for picker, CLI flag inference, non-interactive onboarding, env-ref storage, and catalog behavior. Drop the unrelated secret-scanning and allowlist-comment churn.
