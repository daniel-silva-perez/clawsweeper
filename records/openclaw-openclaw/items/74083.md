## Summary

Keep this PR open. Current main still does not add Docker `--init` in the sandbox create path, and the linked zombie-process report remains a live implementation target. The PR is narrow and matches Docker's documented create flag, but it changes the security-owned sandbox startup path and needs maintainer/secops review, especially on whether existing containers should be recreated or manually refreshed.

## What This Changes

The PR adds Docker `--init` to Docker sandbox container creation arguments and adds unit coverage asserting the flag is present after `create`.

## Best Possible Solution

Keep this PR open for maintainer/secops review. The best fix is likely to add Docker `--init` through `buildSandboxCreateArgs`, keep the focused unit assertion, and make an explicit rollout choice: either bump a sandbox hash/version/epoch so existing containers are recreated safely, or document/communicate `openclaw sandbox recreate` for existing runtimes.
