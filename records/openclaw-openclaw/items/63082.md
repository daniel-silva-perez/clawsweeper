## Summary

Keep open: the PR targets a real shell-completion path bug, but the current patch adds the resolver outside the runtime code that actually checks and writes completion profiles, so the observable install/update behavior remains broken and doctor can print a path different from the file it modified.

## What This Changes

The branch adds an environment-aware shell profile resolver in the completion CLI, tests zsh/fish/bash/PowerShell path cases, and changes doctor completion notes to print that resolved path.

## Best Possible Solution

Move the profile-path resolver into the completion runtime owner and have installCompletion, isCompletionInstalled, usesSlowDynamicCompletion, doctor, update, and setup onboarding share it; preserve completion cache generation behavior and leave broader PowerShell onboarding polish to #44296 unless the shared resolver naturally covers it.

## Review Findings

  - confidence: 0.94
