## Summary

Keep open: the bug report and PR intent are valid, but the current patch does not fix the runtime path used by completion install/check/update. The branch adds a resolver in the CLI generator module and changes doctor text, while the installer and status checks still use the hardcoded resolver in `completion-runtime.ts`.

## What This Changes

The branch adds an environment-aware shell profile resolver and tests in `src/cli/completion-cli.ts`, then changes `src/commands/doctor-completion.ts` to print the resolved profile path in completion install and upgrade notes.

## Best Possible Solution

Revise the PR so profile-path resolution is owned by `completion-runtime.ts` and reused by install, status checks, doctor, update, and setup onboarding. Preserve the intended zsh, fish, and bash path behavior, keep completion script generation and cache placement unchanged, and leave broader PowerShell onboarding polish to #44296 unless a shared resolver naturally covers it.
