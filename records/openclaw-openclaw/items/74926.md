## Summary

Keep open. This is a protected maintainer-labeled, MEMBER-authored implementation PR, current main still lacks the artifact RPC implementation, and the branch appears useful but not ready to close or replace automatically because run/task-scoped artifact queries have a correctness blocker around real transcript provenance.

## What This Changes

The PR adds transcript-derived Gateway `artifacts.list/get/download` RPCs with read-scope registration, protocol and Swift models, SDK `oc.artifacts` helpers/types/docs, tests, changelog coverage, and hardening for task-scope provenance and unsafe download sources.

## Best Possible Solution

Land a maintainer-approved revision that preserves the transcript-derived projection, `operator.read` gating, unsupported unsafe/local download behavior, SDK/docs/protocol synchronization, and generated client artifacts, while also making the advertised `runId` and `taskId` scopes work against real transcript data through explicit persisted provenance.

## Reproduction Assessment

Yes. Current main reproduces the missing API by inspecting the unsupported SDK `oc.artifacts.*` methods and absent Gateway method discovery; the PR finding is reproducible by following the new run/task filters and current transcript reader/writer metadata paths.

## Solution Assessment

No, not as-is. The overall direction is narrow and maintainable, but the PR should first persist or read real run/task provenance for transcript artifacts, or narrow the advertised scope to session-key queries until that provenance exists.
