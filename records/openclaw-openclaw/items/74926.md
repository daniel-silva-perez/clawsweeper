## Summary

Keep open. Current main still lacks the artifact RPC implementation, and this MEMBER-authored, maintainer-labeled PR needs maintainer revision rather than cleanup closure; the latest head still has a blocking taskId lookup defect and an incomplete public protocol docs update.

## What This Changes

The PR adds Gateway `artifacts.list`, `artifacts.get`, and `artifacts.download` RPCs over transcript media, protocol schemas and generated Swift models, read-scope registration, SDK artifact helpers/types, SDK docs, changelog coverage, and focused handler/SDK tests.

## Best Possible Solution

Land a revised artifact RPC hardening PR that keeps the transcript-provenance and unsafe-download guardrails, resolves taskId queries through `TaskRecord` ownership before transcript filtering, repairs the taskId regression test, and documents the new public Gateway protocol methods before closing #74706.

## Reproduction Assessment

Yes. On the latest head, create a task record whose `taskId` differs from its `runId` and requester session, then call `artifacts.list({ taskId })`; the handler sends the task id to a run/session resolver and can return `artifact_scope_not_found` despite matching transcript artifacts.

## Solution Assessment

No, not yet. The overall Gateway/SDK artifact RPC direction is appropriate, but the narrow maintainable fix is to resolve taskId through the task registry/requester session and update the public protocol docs before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Resolve taskId queries from task records:** `src/gateway/server-methods/artifacts.ts:274-275`
  - body: `taskId` is advertised as a supported artifact query, but this path still passes it to `resolveSessionKeyForRun`, which only resolves run/session ids. Real task records keep `taskId`, `runId`, and `requesterSessionKey` separately, so task-scoped artifact queries can return `artifact_scope_not_found` even when transcript messages are tagged with that task id. Look up the task record and use its owning session before filtering artifacts.
  - confidence: 0.9
- **[P2] Fix the taskId regression test fixture:** `src/gateway/server-methods/artifacts.test.ts:158-195`
  - body: The new taskId test does not exercise the intended resolver path: it leaves the run resolver undefined, mocks `loadSessionEntry` without an entry/session file, and asserts `label` even though artifact summaries expose `title`. As written, the test fails or proves the wrong contract instead of guarding taskId provenance.
  - confidence: 0.86
- **[P3] Document the artifact RPC methods:** `src/gateway/server-methods-list.ts:74-76`
  - body: The PR adds public Gateway methods to method discovery, but the protocol reference still omits `artifacts.list`, `artifacts.get`, and `artifacts.download`. Add their params/results and scope notes to `docs/gateway/protocol.md` so SDK and operator clients can discover the new surface from the public docs.
  - confidence: 0.78
