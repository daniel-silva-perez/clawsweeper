## Summary

Keep this issue open. Current main still has only static binding match fields, strict config validation rejects a proposed content matcher, the resolver/cache have no message-content input or rewritten-body output, and the related model-routing or mention-routing items do not supersede this broader content-based agent-binding request.

## Best Possible Solution

Keep this issue open for maintainer product/design work. A good implementation should define a validated content matcher for route bindings, decide precedence across existing static tiers and mention behavior, make route caching content-aware or safely bypassed, pass normalized message text into representative channel/plugin routing calls, define whether stripMatch rewrites BodyForAgent only or a broader inbound context, update config schema/help/docs/generated metadata, and add resolver plus validation coverage.
