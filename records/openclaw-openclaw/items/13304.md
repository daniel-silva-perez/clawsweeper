## Summary

Keep open. Current main still implements bindings as static route selectors only, and the issue discussion already has a maintainer/bot review agreeing that the remaining work crosses config, routing, cache, channel body handling, and API semantics. The related open mention-routing request is narrower, and the older linked items cover model routing/middleware rather than agent binding routing.

## What This Changes

The issue requests an optional binding content matcher, including regex or prefix matching and optional matched-prefix stripping, so inbound message text can choose the target agent.

## Best Possible Solution

Define a first-class validated binding content matcher with deterministic precedence, safe regex/prefix semantics, content-aware or bypassed route caching, representative channel/plugin body plumbing, documented `stripMatch` behavior, and regression coverage for static and content-routing combinations.
