## Summary

Keep open. Current main still treats bindings as static route selectors: the public config type, strict validation schema, route resolver, route cache key, docs, and representative channel call sites do not support matching or stripping message content. The linked mention-routing request is narrower, and the older related reports cover model routing or middleware rather than agent binding routing.

## What This Changes

Request to add an optional content matcher to agent bindings so inbound message prefixes or regexes can route to different agents and optionally strip the matched text before the agent turn.

## Best Possible Solution

Define a first-class, validated binding content matcher with deterministic precedence, safe regex/prefix semantics, content-aware or bypassed route caching, representative channel/plugin body plumbing, documented stripMatch behavior, and regression coverage for static and content-routing combinations.
