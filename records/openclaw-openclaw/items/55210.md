## Summary

Keep open. Current main already has the normal empty-delta stop chunk and nullable non-final `finish_reason`, but it still keeps the streaming error text on the same `finish_reason: "stop"` choice. The PR is useful source work for that remaining error-path bug, but the submitted branch would regress current main's usage-gated finalization and disconnect cleanup, so it needs a bounded current-main replacement or repair before merge.

## What This Changes

This PR changes the Gateway `/v1/chat/completions` streaming helper and tests so streamed chat chunks use top-level `finish_reason`, terminal stop chunks use empty deltas, lifecycle end emits a stop chunk, and streaming errors emit error text before a separate stop signal.

## Best Possible Solution

Carry forward the error-path split as a small current-main patch: emit the error text as a nullable non-final content chunk, let the existing finalization path emit the empty-delta stop chunk, preserve `stream_options.include_usage` and disconnect cleanup, and update the focused Gateway streaming tests plus a changelog fix entry.
