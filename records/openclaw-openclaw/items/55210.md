## Summary

Keep open. Current main has already implemented part of this PR’s normal streaming behavior: `finish_reason` is at the choice level and normal completion emits an empty stop chunk. However, the streaming error path still sends `Error: internal error` in the same chunk that has `finish_reason: "stop"`, and the current test suite still asserts that shape. The PR is therefore not fully obsolete, but it needs to be rebased/narrowed against main’s newer finalization, usage, and disconnect-cleanup logic before it is safe to land.

## Best Possible Solution

Keep this PR open, but ask for a rebase/narrowing: preserve main’s current normal-stream finalization, usage, and disconnect cleanup, then fix the remaining streaming error-path shape with focused tests for a separate error-content chunk followed by an empty final stop chunk.
