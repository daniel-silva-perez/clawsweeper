## Summary

Keep open. Current main covers the normal streaming stop-chunk path, but the streaming error path still combines the error content with `finish_reason: "stop"`; the PR is useful source work but needs rework before merge because its tests conflict with its nullable `finish_reason` output and its lifecycle-end branch drops disconnect cleanup.

## What This Changes

This PR updates the Gateway OpenAI-compatible chat streaming response helper and tests so non-terminal content chunks carry nullable `finish_reason`, terminal stop chunks use empty deltas, and streaming errors emit error text before a separate stop chunk.

## Best Possible Solution

Carry the useful error-path split forward against current main by reusing the existing `writeAssistantStopChunk`/`maybeFinalize()` shape, preserving current usage-chunk finalization and `watchClientDisconnect()` cleanup, and adding focused assertions for the error content chunk plus the terminal empty-delta stop chunk.
