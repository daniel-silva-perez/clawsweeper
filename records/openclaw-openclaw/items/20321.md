## Summary

Current main still does not implement Telegram groupPolicy "members". Config types, strict schema, generated metadata, docs, inbound authorization, and native command authorization all still expose only open/disabled/allowlist, and the related implementation PRs are closed unmerged.

## Best Possible Solution

Keep this open for explicit bundled Telegram plugin and shared config-contract work. A good fix should add members to the exported GroupPolicy, strict schema, generated metadata/help, docs, and Telegram examples; implement fail-closed all-current-members verification in inbound and native command auth with bounded Telegram API calls, caching, and join/leave invalidation; update security audit behavior; and add focused config/runtime/docs tests. Treat #12045, #28351, #28392, #33551, and the openclaw-chat-only-trigger plugin as prior art, not close evidence.
