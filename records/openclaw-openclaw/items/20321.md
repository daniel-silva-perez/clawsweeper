## Summary

Current main still does not implement Telegram `groupPolicy: "members"`. The exported config type, strict schema, generated Telegram metadata, Telegram runtime authorization helper, inbound/native command paths, tests, and docs still expose only `open`, `disabled`, and `allowlist`. The related implementation PRs are closed unmerged, and the linked `openclaw-chat-only-trigger` plugin is useful prior art/workaround rather than the requested bundled Telegram group policy.

## Best Possible Solution

Keep this open for explicit bundled Telegram plugin plus shared config-contract work. A good fix should add `members` to the exported group-policy type, strict schema, generated metadata/help, docs, and Telegram examples; implement fail-closed all-current-members verification for inbound and native command authorization with bounded API calls, caching, and join/leave invalidation; update security audit behavior; and add focused config/runtime/docs tests. Treat #12045, #28351, #28392, #33551, and the linked `openclaw-chat-only-trigger` plugin as prior art, not close evidence.
