## Summary

Keep this issue open. Current main still does not implement Telegram `groupPolicy: "members"`: config types/schema/metadata only accept `open`, `disabled`, and `allowlist`, the Telegram runtime helper only enforces those policies, and the inbound/native command paths delegate to that helper. The linked implementation PRs are closed unmerged; the chat-only-trigger plugin is useful prior art/workaround but not the bundled Telegram policy requested here.

## Best Possible Solution

Keep this open for explicit bundled Telegram plugin plus shared config-contract work. A good fix should add `members` to the exported group-policy type, strict schema, generated metadata/help, docs, and Telegram examples; implement fail-closed all-current-members verification in inbound and native command authorization with bounded API calls, caching, and membership invalidation; update security audit behavior; and add focused config/runtime/docs tests. Treat #12045, #28351, #28392, #33551, and the linked `openclaw-chat-only-trigger` plugin as prior art, not close evidence.
