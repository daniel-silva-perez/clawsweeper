## Summary

Keep this issue open. Current main still does not implement Telegram `groupPolicy: "members"`: the public config type, strict schema, generated metadata, shared group-policy resolver, Telegram runtime authorization helper, inbound/native command paths, docs, and focused tests still expose only `open`, `disabled`, and `allowlist`. The closed related PRs and the linked chat-only-trigger plugin are useful prior art/workarounds, not an implementation of the requested bundled Telegram member-allowlist policy.

## Best Possible Solution

Keep this open for explicit bundled Telegram plugin plus shared config-contract work. A good fix should add `members` to the exported group-policy type, strict schema, generated metadata/help, docs, and Telegram examples; implement fail-closed all-current-members verification for inbound messages and native command authorization with bounded API calls, cache TTL, and join/leave invalidation; update Telegram security audit behavior; and add focused config/runtime/docs tests. Treat #12045, #28351, #28392, #33551, and https://github.com/constansino/openclaw-chat-only-trigger as prior art rather than close evidence.
