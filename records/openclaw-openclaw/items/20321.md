## Summary

Keep open. Current main still does not ship a bundled Telegram `members` group policy: the public config contract, strict schema, generated metadata, docs, inbound group handling, native-command authorization, and Telegram security audit all remain limited to `open`, `disabled`, and `allowlist`. The closed implementation PRs and the external chat-only-trigger plugin are useful prior art/workarounds, but they are not a merged OpenClaw implementation.

## What This Changes

The issue requests a bundled Telegram `groupPolicy: "members"` mode that authorizes newly created groups when every current participant is in the trusted user allowlist.

## Best Possible Solution

Define and land an explicit bundled Telegram `members` policy that updates config types/schema/generated metadata, inbound group-message and native-command authorization, fail-closed Telegram membership verification, security-audit semantics, docs, and focused tests while preserving the existing `open`, `disabled`, and `allowlist` behavior.
