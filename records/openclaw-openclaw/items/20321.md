## Summary

Keep open. Current main still lacks a bundled Telegram `groupPolicy: "members"`: the public config type, strict schema, generated metadata, Telegram inbound/native-command authorization, security audit, tests, and user docs remain limited to `open`, `disabled`, and `allowlist`. The closed implementation PRs and constansino's `openclaw-chat-only-trigger` plugin are useful prior art/workarounds, but they do not satisfy the requested OpenClaw behavior.

## What This Changes

Add a Telegram `groupPolicy: "members"` mode that allows newly created groups when every current participant is trusted, instead of manually allowlisting each Telegram group chat ID.

## Best Possible Solution

Define and land an explicit bundled Telegram `members` policy with updated config types/schema/generated metadata, inbound group-message and native-command authorization, fail-closed Telegram membership verification, security-audit semantics, docs, and focused tests while preserving existing `open`, `disabled`, and `allowlist` behavior.
