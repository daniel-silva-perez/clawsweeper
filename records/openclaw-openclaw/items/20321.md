## Summary

Keep open. Current main still does not include a bundled Telegram `members` group policy: the public config contract, strict schema, generated metadata, Telegram runtime authorization paths, native-command authorization, security audit, and user docs all remain limited to `open`, `disabled`, and `allowlist`. The prior closed PRs and the external chat-only-trigger plugin are useful prior art/workarounds, but they do not satisfy the requested bundled OpenClaw behavior.

## What This Changes

Add a Telegram `groupPolicy: "members"` mode that allows newly created groups when every current participant is trusted, instead of manually allowlisting each Telegram group chat ID.

## Best Possible Solution

Define and land an explicit bundled Telegram `members` policy that updates config types/schema/generated metadata, inbound group-message and native-command authorization, fail-closed Telegram membership verification, security-audit semantics, docs, and focused tests while preserving existing `open`, `disabled`, and `allowlist` behavior.
