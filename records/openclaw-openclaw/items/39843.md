## Summary

Keep open. Current main still has the Google Chat typing-placeholder cleanup gap for exact silent replies, and open PR #39891 references this issue with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks Google Chat to clear its typing indicator when an agent uses an emoji reaction as the acknowledgment and returns `NO_REPLY` with no text reply.

## Best Possible Solution

Review and land #39891, or an equivalent Google Chat-owned fix, so a leftover `typingMessageName` is deleted after silent dispatch completion and a monitor-level regression test covers reaction-only exact `NO_REPLY`.

## Reproduction Assessment

Yes. The source-level path is deterministic on current main: Google Chat creates a typing message, exact `NO_REPLY` can be skipped before delivery, and no Google Chat cleanup callback or post-dispatch deletion handles the leftover message.

## Solution Assessment

Yes. The narrowest maintainable fix is in the Google Chat monitor lifecycle, either by wiring `onTypingCleanup` or deleting any still-held `typingMessageName` after dispatch completes without delivery; this does not require a core rewrite.
