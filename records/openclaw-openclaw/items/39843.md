## Summary

Keep open. Current main still creates a Google Chat typing placeholder message and only clears it from the delivery callback, while exact `NO_REPLY` final payloads can be skipped before delivery. Open PR #39891 still references this issue with closing syntax and remains the concrete implementation path, so the issue should not be closed before that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land #39891, or an equivalent Google Chat monitor fix, after maintainer review. The fix should delete any remaining `typingMessageName` after `dispatchReplyWithBufferedBlockDispatcher` completes without delivery, or wire `onTypingCleanup`, and add a monitor-level regression test for reaction-only exact `NO_REPLY`.
