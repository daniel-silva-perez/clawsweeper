## Summary

Keep open. Current main still has the WebChat history/stream ordering problems this PR is trying to address, so the PR is not obsolete; however, the current PR head still has two blocking correctness issues in the new stream-prefix and deferred-final-message handling.

## What This Changes

The PR updates Gateway chat-history projection/session transcript timestamp metadata plus Control UI chat reconciliation and rendering to stabilize WebChat history refreshes, active streams, tool-boundary stream segments, deferred final messages, visible error bubbles, and related regression tests.

## Best Possible Solution

Revise the PR so live stream text has exactly one representation boundary, either cumulative until render/final time or suffix-only with downstream readers aware of that contract, and preserve distinct deferred finals using stable run/message metadata or by allowing repeated visible final messages; then land after targeted UI/gateway validation and maintainer review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep stream-prefix trimming at one boundary:** `ui/src/ui/controllers/chat.ts:837`
  - body: This stores an already-trimmed suffix in `state.chatStream`, but `streamedAssistantText` and `buildChatItems` trim the same stream against committed segments again. If the committed segment is `A` and the next cumulative snapshot is `AA reply`, state becomes `A reply`; the second trim drops the leading `A` from the live bubble and fallback final text. Store cumulative text until render/final time, or make downstream readers treat `chatStream` as already trimmed.
  - confidence: 0.88
- **[P2] Preserve distinct deferred final messages:** `ui/src/ui/controllers/chat.ts:309`
  - body: Deferred finals are deduped by role plus rendered text/content, so two unrelated background runs that both produce text such as `Done` collapse in the queue and only one is flushed after the active run ends. Use run/message identifiers when available, or avoid text-only dedupe for deferred finals.
  - confidence: 0.84
