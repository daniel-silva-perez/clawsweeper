## Summary

Keep open. Current main still has the WebChat history and tool-stream ordering risks this PR targets, but the current PR head still has two blocking UI state bugs around double stream-prefix trimming and text-only deferred-final deduplication.

## What This Changes

The PR changes Gateway chat-history projection/session metadata and Control UI chat reconciliation/rendering to stabilize WebChat history reloads, active streams, tool-boundary stream segments, deferred final messages, and visible error display.

## Best Possible Solution

Revise the PR so live stream text has exactly one representation contract, either cumulative until render/final time or suffix-only with all downstream readers aware of that contract, and preserve distinct deferred finals using stable run/message metadata or by avoiding text-only dedupe; then land after targeted WebChat/Gateway validation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep stream-prefix trimming at one boundary:** `ui/src/ui/controllers/chat.ts:837`
  - body: This stores `state.chatStream` as an already-trimmed suffix, but `streamedAssistantText` and `buildChatItems` trim the same stream against committed segments again. If the committed segment is `A` and the next uncommitted suffix also starts with `A`, the live bubble and fallback final text lose that leading text. Keep `chatStream` cumulative until render/final time, or stop trimming it downstream.
  - confidence: 0.9
- **[P2] Preserve distinct deferred final messages:** `ui/src/ui/controllers/chat.ts:309`
  - body: Deferred finals are queued through `appendUniqueDisplayMessages`, which dedupes only by role plus rendered text/content. Two unrelated background runs that both finish with text like `Done` collapse before flush, so one visible result is lost. Use run/message identifiers when available, or avoid text-only dedupe for deferred finals.
  - confidence: 0.86
