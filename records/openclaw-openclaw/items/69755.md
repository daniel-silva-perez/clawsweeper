## Summary

Current main still emits the bare `Run: openclaw doctor --non-interactive` restart-sentinel hint from config write, update, chat restart, and gateway-tool restart paths, so the reported approval-constrained follow-up UX remains valid and narrow.

## What This Changes

The issue asks OpenClaw to make post-restart `openclaw doctor --non-interactive` guidance distinguish recommended follow-up advice from commands that are actionable in the current surface.

## Best Possible Solution

Keep this issue open and make a narrow restart-sentinel UX fix: keep recommending `openclaw doctor --non-interactive`, but label it as a recommended follow-up to run from a terminal or approvals-capable OpenClaw UI when the current surface cannot complete approvals. Prefer centralizing the wording in `formatDoctorNonInteractiveHint()` and updating focused tests across config/update/restart delivery paths.
