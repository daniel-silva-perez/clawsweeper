## Summary

Keep open. Current main still emits the bare `Run: openclaw doctor --non-interactive` restart-sentinel hint for config writes, update/restart paths, and gateway-tool restarts, without distinguishing a terminal or approvals-capable surface from a heartbeat/chat context.

## Best Possible Solution

Keep this issue open and implement a narrow restart-sentinel UX fix: keep the doctor recommendation, but label it as a recommended follow-up that should be run from a terminal or approvals-capable OpenClaw UI when the current surface cannot complete approvals. Add focused tests around `formatDoctorNonInteractiveHint()` and the restart/config/update sentinel paths, and coordinate with adjacent doctor-summary automation in #64684.
