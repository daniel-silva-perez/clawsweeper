## Summary

Keep open. Current main still matches the reported custom-session cron transcript gap, and open PR #74913 plus the linked fork PR already target the same bug, so this issue should remain open until a focused fix lands.

## What This Changes

The issue reports that announced cron jobs targeting `--session session:<id>` deliver visible replies to the destination channel but do not persist those replies into that session transcript.

## Best Possible Solution

Review and land PR #74913, or an equivalent focused fix, so successful direct cron deliveries mirror their visible assistant text into the resolved destination session transcript using the existing outbound mirror contract, an idempotency key, and regression coverage for `session:<id>` jobs.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: `session:<id>` resolves to a persistent cron session and direct delivery omits `mirror`, while transcript append requires `params.mirror`; the reporter's live steps exercise that path.

## Solution Assessment

Yes. Keeping this issue open while maintainers review the existing focused PR is the narrow maintainable path; the fix should reuse outbound mirroring rather than adding a separate cron-only transcript writer.
