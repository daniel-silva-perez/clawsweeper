## Summary

Keep open. Current main still matches the reported cron transcript gap: custom `session:<id>` jobs resolve to a persistent cron session and direct delivery sends to the channel, but the direct outbound call still omits `mirror`, while transcript persistence only runs when `mirror` is present. The issue is also paired with open PR #74913 using closing syntax, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that cron jobs targeting `session:<id>` persist direct-delivered assistant replies into the destination session transcript.

## Best Possible Solution

Land or revise PR #74913, or an equivalent focused patch, so successful non-empty direct cron deliveries pass the existing outbound mirror contract for the resolved destination session and use an idempotency key to avoid duplicate transcript writes.

## Reproduction Assessment

Yes. The reporter's steps map to current main: `session:<id>` is resolved into the cron run session key, direct delivery sends through `deliverOutboundPayloads`, and the only transcript append path requires `params.mirror`, which that call does not supply.

## Solution Assessment

Yes. Reusing the existing outbound `DeliveryMirror` contract is the narrow maintainable fix; it avoids adding a separate cron-only transcript path and can be covered in `delivery-dispatch.double-announce.test.ts`.
