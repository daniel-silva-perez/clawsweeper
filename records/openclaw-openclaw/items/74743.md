## Summary

Keep open. Current main still matches the reported cron custom-session transcript gap, and PR #74913 plus the linked fork PR already target the same bug, so this issue should remain open until one focused fix lands.

## What This Changes

The issue reports that announced cron jobs targeting `--session session:<id>` deliver visible replies to the destination channel but do not persist those replies into that session transcript.

## Best Possible Solution

Review and land PR #74913, or an equivalent focused fix, so successful direct cron deliveries mirror their visible assistant text into the resolved destination session transcript using the existing outbound mirror contract, idempotency key, and regression coverage for `session:<id>` jobs.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: `session:<id>` resolves to a persistent cron session and direct delivery omits `mirror`, while transcript append requires `params.mirror`; the reporter's live steps exercise that path.

## Solution Assessment

Yes. Keeping this issue open while maintainers review the existing focused PR is the narrowest maintainable path; a separate cron-only transcript writer would duplicate the outbound mirror contract.
