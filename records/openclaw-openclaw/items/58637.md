## Summary

Keep this PR open. Current main still lacks the Control UI sessions-table Model column and still does not recover a transcript model when runtime model fields are missing but usage data is already persisted; however, the PR should not merge as-is because its new transcript fallback remains gated behind the existing usage-fallback read.

## What This Changes

The PR adds a Model column to the Control UI sessions table and changes `buildGatewaySessionRow` to prefer transcript model identity when no runtime model or explicit override is recorded.

## Best Possible Solution

Refresh the patch against current main, add a localized Model column to the sessions table, update responsive table column hiding if needed, and make transcript model recovery trigger whenever runtime model fields are absent and no explicit override is set, independent of token/cost fallback. Add focused gateway and UI regression tests before merging.

## Reproduction Assessment

Yes. A source-level reproduction is clear: build a session row with no `entry.model`, no `entry.modelProvider`, no `entry.modelOverride`, complete persisted usage fields, and a transcript assistant usage record with a model; current main skips the transcript read and returns the configured default model. The UI side is also directly reproducible from current render code because `row.model` is never rendered in the sessions table.

## Solution Assessment

No, not as proposed. The direction is right, but the PR needs to load transcript usage for the model-only missing case before checking `transcriptUsage?.model`, and it needs a current-main UI refresh that preserves newer sessions-table structure and localization.
