## Summary

Keep open. Current main still lacks the sessions-table model column and still cannot recover transcript model identity when no runtime or override model is stored, but this PR is not ready to merge because the proposed fallback only works when another usage field also forces a transcript read.

## What This Changes

The PR adds a Model column to the Control UI sessions table and changes gateway session-row model fallback to prefer transcript model data over the configured default when stored runtime and override model metadata are absent.

## Best Possible Solution

Rebase or replace the PR on current main, keep the requested model column, use the existing Control UI i18n/table patterns, trigger transcript reading for model-only recovery while preserving runtime, override, and live-subagent precedence, and add a focused regression plus changelog entry.

## Reproduction Assessment

Yes. A high-confidence reproduction path is a session entry with persisted token/context/cost data, no entry.model, no entry.modelOverride, and a transcript assistant usage snapshot with model data; the proposed fallback would not read that transcript because the usage-needed gates are false.

## Solution Assessment

No. The product direction is right, but the patch is not the best fix until transcript fallback is decoupled from missing usage data and the UI/changelog changes are adapted to current main conventions.
