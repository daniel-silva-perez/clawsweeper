## Summary

Keep this PR open. Current main still does not show a Model column in the Control UI sessions table, and `buildGatewaySessionRow` still only reads transcript usage when token/context/cost fields need fallback, so it does not implement the PR’s intended model-visibility and missing-runtime-model fallback changes. The bot review also identifies a real gap in the PR as proposed, so the best path is a refreshed, narrower patch rather than cleanup closure.

## Best Possible Solution

Refresh this PR against current main, add the Model column in `ui/src/ui/views/sessions.ts`, and update `buildGatewaySessionRow` so transcript model recovery is triggered when runtime model fields are absent and no explicit override is set, independent of whether token/context/cost fallback is needed. Add focused tests for complete-persisted-usage/no-runtime/no-override transcript recovery, override precedence, and the sessions-table model cell.
