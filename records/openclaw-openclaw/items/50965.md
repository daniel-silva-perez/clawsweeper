## Summary

Keep this PR open. Current main still has the reported cron-remove false-success behavior, and this PR is a focused but incomplete implementation candidate: it changes the service-layer result, but it does not update the CLI remove aliases to exit non-zero or add regression coverage for missing-job removal. The PR diff is not suspicious from a security or supply-chain perspective because it only touches cron service logic, but it needs functional follow-through before merge.

## Best Possible Solution

Update this PR or replace it with a complete fix: make cron service removal return a typed not-found failure without persisting or rearming on a no-op, propagate that result through the cron service/gateway contract, make `cron rm/remove/delete` exit non-zero when `ok` or `removed` is false, and add focused service plus CLI regression tests for missing-job removal.
