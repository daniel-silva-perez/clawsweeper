## Summary

Keep this PR open. Current main still returns a successful cron removal payload for missing jobs and still leaves the CLI remove aliases without a non-zero exit path, so main does not implement the intended fix. The submitted PR is directionally useful at the service layer, but the provided review discussion is right that it is incomplete without CLI propagation and regression tests.

## Best Possible Solution

Update this PR or replace it with a complete fix: make cron service removal return a typed not-found failure without persisting or rearming on a no-op, propagate the richer result through the cron service/gateway contract, make `cron rm/remove/delete` exit non-zero when `ok` or `removed` is false, and add focused service plus CLI regression tests for missing-job removal.
