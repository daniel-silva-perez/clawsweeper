## Summary

Keep this PR open. Current main still has the cron removal silent-success behavior, and the submitted PR is a plausible service-layer fix but does not complete the user-visible CLI failure path reviewers already flagged.

## Best Possible Solution

Update this PR or replace it with a complete fix: make cron service removal return a real not-found failure without persisting or rearming on no-op, propagate the typed result through gateway/tool contracts, make cron rm/remove/delete exit non-zero when ok/removed is false, and add targeted service plus CLI regression tests for the missing-job path.
