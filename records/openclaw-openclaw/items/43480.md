## Summary

Keep open: current main still matches the reported VoiceWakeOverlay render-loop path. The SwiftUI view reacts to attributed-text changes by synchronously updating the window frame, and that measurement path still mutates observable overflow state that the view reads.

## What This Changes

The issue reports that the macOS VoiceWakeOverlay can enter a SwiftUI render loop because attributed-text changes synchronously trigger frame measurement that mutates observable overflow state.

## Best Possible Solution

Fix the macOS owner code so overlay measurement is pure or schedules/coalesces overflow-state changes outside SwiftUI view-update handling, then add focused regression coverage plus manual macOS verification for repeated VoiceWakeOverlay open/update/reorder cycles.

## Repair Work Prompt

Fix the macOS VoiceWakeOverlay SwiftUI render-loop bug reported in https://github.com/openclaw/openclaw/issues/43480. Current main still has `VoiceWakeOverlayView` reacting to `controller.model.attributed` by synchronously calling `controller.updateWindowFrame(animate: true)`, while `VoiceWakeOverlayController+Window.measuredHeight()` directly writes `model.isOverflowing`. Keep the fix scoped to the macOS VoiceWakeOverlay owner code: make height measurement pure or guard/defer/coalesce `isOverflowing` changes outside the SwiftUI view-update cycle, and avoid broad architecture work unless necessary. Also consider coalescing attributed-text frame updates so live transcript changes cannot schedule multiple frame updates per SwiftUI frame. Likely files are `apps/macos/Sources/OpenClaw/VoiceWakeOverlayView.swift`, `apps/macos/Sources/OpenClaw/VoiceWakeOverlayController+Window.swift`, `apps/macos/Sources/OpenClaw/VoiceWakeOverlayController+Session.swift`, and focused tests under `apps/macos/Tests/OpenClawIPCTests/`. Add regression coverage where practical for guarded/deferred overflow updates or frame-update coalescing; document manual macOS verification if the SwiftUI loop cannot be fully unit-tested. Add a single-line `CHANGELOG.md` Unreleased Fixes entry for the user-facing macOS pinwheel fix, crediting `@gdiab` and any additional credited contributor whose workaround materially informs the patch. Do not change unrelated menubar watchdog work from #60990, bundled gateway behavior, or broad VoiceSessionPublisher migration docs unless the narrow fix requires a small doc clarification.
