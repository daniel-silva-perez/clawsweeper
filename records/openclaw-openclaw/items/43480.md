## Repair Work Prompt

Fix the macOS VoiceWakeOverlay render-loop/pinwheel reported in openclaw/openclaw#43480. Keep the change scoped to the overlay owner files. Break the synchronous cycle where attributed text changes trigger `updateWindowFrame()`, `targetFrame()` calls `measuredHeight()`, and `measuredHeight()` mutates `model.isOverflowing` on the `@Observable` model during SwiftUI view updates. Prefer making measurement side-effect free or moving overflow-state updates behind a guarded/coalesced MainActor update; also defer or coalesce attributed-text frame updates so SwiftUI `onChange` does not synchronously recurse. Add focused tests that would catch unguarded overflow updates or redundant frame/update calls where feasible, and preserve existing voice overlay behavior. Do not change unrelated WorkActivityStore/menubar watchdog behavior or broaden this into provider/gateway work.
