Fix https://github.com/openclaw/openclaw/issues/43480. The macOS VoiceWakeOverlay can enter a SwiftUI render loop because `VoiceWakeOverlayView` handles attributed-text changes by synchronously calling `updateWindowFrame(animate: true)`, while `measuredHeight()` mutates `model.isOverflowing` during layout measurement.
