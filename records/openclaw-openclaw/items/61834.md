## Summary

Keep open. Current main still supports QMD search mode selection, direct CLI querying, and mcporter/MCP querying, but it does not expose any config-level or `memory_search` request-level way to disable QMD query reranking.

## What This Changes

Expose QMD 2.1.0's no-rerank capability through OpenClaw's QMD memory query mode.

## Best Possible Solution

Add a QMD-owned config-level control, thread it through the typed config/schema/resolver, apply it only when the effective QMD search command is `query`, and update direct CLI plus MCP v2 query calls with focused regression coverage and user-facing docs.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature gap. The gap is high-confidence by static verification: current config, tool schema, direct `qmd query` args, and MCP query args all lack a no-rerank path.

## Solution Assessment

Yes for the config-level path: it is the narrowest maintainable solution because it stays in the QMD backend contract and avoids changing model tool behavior per call. A request-level toggle can be considered later if maintainers want per-search latency/quality control.

## Repair Work Prompt

Implement a narrow fix for https://github.com/openclaw/openclaw/issues/61834. Add a typed config-level QMD rerank control, preferably `memory.qmd.noRerank?: boolean`, and resolve it through the memory-host SDK. Apply it only when the effective QMD search mode is `query`: direct CLI mode should append `--no-rerank`; mcporter/MCP v2 `query` calls should send `rerank: false`; `search` and `vsearch` should ignore it. Preserve existing older-QMD fallback behavior and do not add a request-level `memory_search.rerank` parameter unless it is explicitly kept compatible and covered. Update config schema/help/generated docs, public QMD memory docs, focused backend-config and qmd-manager tests, and add a user-facing changelog entry credited with `Thanks @kouka-t0yohei`. Do not change QMD collection behavior, memory scoping, or unrelated builtin memory reranking/MMR code.
