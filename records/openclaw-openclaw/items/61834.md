## Summary

Keep open. Current main supports QMD `searchMode` selection and MCP tool compatibility, but it still does not expose a config-level or `memory_search` request-level way to disable reranking for QMD query mode.

## Best Possible Solution

Add a QMD-owned memory-core implementation: a typed config option such as `memory.qmd.noRerank` and/or a `memory_search` request-level `rerank` boolean, resolve it through `packages/memory-host-sdk`, append `--no-rerank` only for direct `qmd query`, pass `rerank: false` only for MCP v2 `query`, and add focused config/schema/docs plus direct-CLI and mcporter regression tests with older-QMD fallback behavior.
