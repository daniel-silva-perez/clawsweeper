## Summary

Keep open. Current main still renders native chat bubble bodies as plain SwiftUI `VStack` and Android Compose `Column` content, with scrolling only at the transcript/list level. The latest release tag and current main both lack an always-scrollable bounded body path for large/tall bubbles, and the prior issue discussion already reached the same keep-open conclusion.

## Best Possible Solution

Keep the issue open for a focused native chat UI fix. Define the current equivalent of `.large` or tall bubbles, wrap the message body area in a bounded vertical scroll container in `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatMessageViews.swift`, add Android parity in `apps/android/app/src/main/java/ai/openclaw/app/ui/chat/ChatMessageViews.kt`, preserve transcript-level scrolling in `ChatView.swift` and `ChatMessageListCard.kt`, and add focused render coverage or documented manual mobile verification for tall-but-not-truncated content.
