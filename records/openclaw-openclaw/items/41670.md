## Summary

Keep this PR open. Current main still omits `encoding_format` from the active shared remote embedding helper and from the OpenAI direct/batch embedding request builders, so the user-visible compatibility problem is not implemented on main. The PR is a plausible small fix, but landing it exactly as submitted would only change the `src/memory-host-sdk` mirror path and would not cover the package-backed SDK export or the OpenAI-specific paths. #43910 is a related open candidate, not merged replacement evidence.

## Best Possible Solution

Keep this PR open for maintainer follow-up. The best path is to rework the fix against current main so the active package helper and the OpenAI direct/batch paths send `encoding_format: "float"` only where appropriate, preferably through a provider-scoped or configurable `encodingFormat` option with focused regression tests for a strict OpenAI-compatible proxy. If maintainers prefer #43910's configurable design, review and land that first, then close this PR as superseded after the replacement is merged.
