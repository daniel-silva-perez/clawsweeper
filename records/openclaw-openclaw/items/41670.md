## Summary

Keep this PR open. The underlying compatibility request is valid, but current main at 14f140d6 still does not send `encoding_format` from the active package-backed remote embedding helper or the OpenAI direct/batch embedding paths. The submitted one-line patch targets the mirrored `src/memory-host-sdk` file, while the public plugin SDK facade re-exports the package implementation used by bundled providers, so merging it as-is would likely miss the active helper path.

## Best Possible Solution

Keep this PR open for maintainer follow-up. Rework the fix against current main so the active package-backed helper and OpenAI direct/batch embedding paths handle `encoding_format` where appropriate, ideally through a provider-scoped or configurable `encodingFormat` option with focused regression tests. If #43910 becomes the accepted implementation, land that and then close this PR as superseded.
