## Summary

Keep this PR open. Current main still does not implement the requested outbound `encoding_format: "float"` behavior for the active shared embedding helper or the OpenAI direct/batch embedding request builders. The PR is a plausible focused compatibility fix, but as submitted it only edits the `src/memory-host-sdk` mirror path and would miss the package-backed SDK export plus separate OpenAI paths. A newer configurable approach in #43910 is only an open candidate from the provided context, not merged replacement evidence.

## Best Possible Solution

Keep this PR open for maintainer follow-up. The best path is to rework the fix against current main so the active package helper and OpenAI direct/batch embedding paths send `encoding_format: "float"` only where appropriate, ideally via a provider-scoped or configurable `encodingFormat` option with focused regression tests for a strict OpenAI-compatible proxy. If maintainers prefer #43910's configurable approach, review and land that first, then close this PR as superseded after it is merged.
