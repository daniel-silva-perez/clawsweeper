## Summary

Keep this PR open. The compatibility request is real, and the PR is narrowly scoped, but current main still omits `encoding_format` from the active package-backed remote embedding helper and from the OpenAI direct/batch embedding paths. The submitted patch only changes the mirrored `src/memory-host-sdk` helper, while the public SDK facade exports the package implementation. A broader related implementation candidate, #43910, is still open rather than merged, so it is not yet a reason to close this PR as implemented or superseded.

## Best Possible Solution

Keep this PR open for maintainer follow-up. Rework the fix against current main so the active `packages/memory-host-sdk` helper and the relevant OpenAI inline/batch embedding paths support an owner-approved `encodingFormat`/`encoding_format` policy, preferably configurable or provider-scoped rather than hardcoded globally. Update schema/types/docs and focused tests. If #43910 becomes the accepted implementation and merges, close this PR as superseded by that merged work.
