## Summary

Keep this PR open. Current main still does not implement the requested `encoding_format` support in the active package-backed remote embedding helper, the OpenAI direct path, or the OpenAI batch path. The PR also targets the `src/memory-host-sdk` mirror path only, while current docs/code show that hard-coding this field globally needs maintainer care because some compatible providers reject it.

## Best Possible Solution

Keep this PR open for maintainer follow-up. Rework the fix against current main so the active package helper and OpenAI direct/batch paths send `encoding_format: "float"` only where appropriate, ideally through a provider-scoped or configurable `encodingFormat` option with focused regression tests for strict OpenAI-compatible proxies. If #43910 becomes the accepted configurable implementation, land that first and then close this PR as superseded.
