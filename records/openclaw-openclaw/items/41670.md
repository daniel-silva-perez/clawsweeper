## Summary

Keep this PR open. Current main at bdfb408ce6eeb724a638c106104dbbcae50b612c still omits `encoding_format` from the active package-backed remote embedding helper and from the OpenAI direct and batch request builders. The one-line PR diff targets the `src/memory-host-sdk` mirror path, while the public SDK entrypoint re-exports the package implementation. #43910 is related and still open, so it is not shipped replacement evidence.

## Best Possible Solution

Keep this PR open for maintainer follow-up. The best path is to rework the fix against current main so the active package helper and the OpenAI direct/batch paths send `encoding_format: "float"` only where appropriate, ideally through a provider-scoped or configurable `encodingFormat` option with focused regression tests for a strict OpenAI-compatible proxy. If maintainers prefer #43910's configurable design, review and land that first, then close this PR as superseded after the replacement is merged.
