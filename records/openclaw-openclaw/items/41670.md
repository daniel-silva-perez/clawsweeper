## Summary

Keep open. The compatibility request is coherent, but current main still does not send `encoding_format` from the active package-backed remote embedding helper, and the submitted one-line patch targets the mirrored `src/memory-host-sdk` file rather than the public SDK package implementation used by bundled providers. Related #43910 is a broader open implementation candidate, not a merged replacement.

## Best Possible Solution

Keep this PR open for maintainer follow-up. Rework the fix against current main so the active package-backed helper and the relevant OpenAI direct/batch embedding paths handle `encoding_format` through a provider-scoped or configurable `encodingFormat` option, with schema/types/docs and focused regression tests. If #43910 becomes the accepted implementation and merges, close this PR as superseded by that implementation.
