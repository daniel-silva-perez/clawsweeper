## What This Changes

The PR adds a `suggestPeerAgentWorkspaceDir` helper, uses it for the brand-new `agents add` workspace prompt, re-exports it, and adds regression tests plus a changelog entry.

## Best Possible Solution

Land exactly one focused fix for #71889 that changes the brand-new `agents add` prompt default to the documented peer-level workspace while preserving explicit per-agent workspaces and the #59789 shared-base runtime contract unless a maintainer deliberately changes that contract.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: after setup writes `agents.defaults.workspace` to the default workspace, `agents add <new-id>` seeds the prompt from `resolveAgentWorkspaceDir`, whose non-default fallback appends the id under that workspace.

## Solution Assessment

No as submitted. The scoped wizard-default approach is maintainable, but this PR should not merge until the environment-dependent test is fixed, attribution is clarified, and a maintainer chooses it over the competing open fixes.
