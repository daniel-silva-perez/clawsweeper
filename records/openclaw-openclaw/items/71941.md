Current main still has the reported `agents add` wizard behavior: the prompt uses `resolveAgentWorkspaceDir`, whose `agents.defaults.workspace` fallback appends the new agent id as a child path. This PR is a plausible fix for #71889, but it has unresolved review feedback and a newer competing open PR (#73620) with different workspace-default semantics, so it needs maintainer review rather than cleanup closure.
