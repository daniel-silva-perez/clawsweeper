## Summary

Keep open. Current main still matches the reported Feishu voice-duration bug: the message tool and Feishu plugin can send Opus/Ogg as native Feishu audio, but the outbound media path does not compute or pass a duration into the Feishu file upload call. The prior ClawSweeper comment remains consistent with current main.

## Best Possible Solution

Keep this issue open and fix the bundled Feishu plugin media path. Compute a finite audio duration for native Opus/Ogg sends and successful `audioAsVoice` transcode outputs, pass it to `uploadFileFeishu`, and add focused regression tests proving `client.im.file.create` receives `duration` for native voice audio and voice-intent transcodes.
