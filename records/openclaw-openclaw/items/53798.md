## Summary

Current main still appears to have the reported Feishu voice-duration bug. The message tool and Feishu plugin route `media`/`asVoice` into native Feishu Ogg/Opus audio delivery, and the upload helper can send `duration`, but the outbound media path does not compute or pass it. The prior Codex review in the issue remains consistent with current main; the only later Feishu media commit in this checkout fixes filename decoding, not audio duration.

## Best Possible Solution

Keep this issue open and fix the bundled Feishu plugin media path. Compute a finite audio duration for native Opus/Ogg sends and for successful `audioAsVoice` transcode outputs, pass it to `uploadFileFeishu`, and add focused regression tests proving `client.im.file.create` receives `duration` for native voice audio and voice-intent transcodes.
