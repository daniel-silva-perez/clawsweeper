## Summary

Keep open. Current main still routes message-tool Feishu media/asVoice sends into native Ogg/Opus audio delivery, and the upload helper can forward duration, but the outbound media path never computes or passes it. The related PR #50399 targets the same bug but remains open and unmerged, so it is not implementation evidence.

## What This Changes

The issue asks Feishu voice messages sent through the shared message tool with Opus media to include duration metadata so Feishu displays the normal audio duration.

## Best Possible Solution

Keep this issue open and fix the bundled Feishu plugin with a narrow patch: preserve existing media routing/download metadata, compute a finite millisecond duration for native Opus/Ogg uploads and successful audioAsVoice transcode outputs, pass it to uploadFileFeishu, and add focused regression tests proving client.im.file.create receives duration on those voice paths.
