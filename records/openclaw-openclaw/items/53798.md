## Summary

Keep open. Current `main` still routes message-tool Feishu `media`/`asVoice` sends into native Ogg/Opus audio delivery, and the Feishu upload helper can forward `duration`, but the outbound media path never computes or supplies it. The related open PR #50399 targets the same bug, but it has maintainer-reviewed regression/security concerns and is not merged, so it is not evidence that this issue is implemented.

## What This Changes

The issue reports that Feishu voice messages sent through the shared message tool with an Opus media file are delivered but do not show an audio duration in the Feishu UI.

## Best Possible Solution

Keep this issue open and fix the bundled Feishu plugin with a smaller patch than #50399 if needed: preserve the existing media routing/download metadata helpers, compute a finite millisecond duration for native Opus/Ogg uploads and successful `audioAsVoice` transcode outputs, pass it to `uploadFileFeishu`, and add focused regression tests proving `client.im.file.create` receives `duration` for those voice paths.
