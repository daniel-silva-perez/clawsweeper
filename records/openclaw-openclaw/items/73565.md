## Summary

Keep this issue open. Current main still has unguarded transcript append sinks, and the linked open PR #73563 is the active closing implementation candidate, so the issue should remain open until that PR or an equivalent maintainer-approved fix lands.

## What This Changes

Apply the existing session transcript redaction guard to core session, CLI attempt, gateway inject, and bundled plugin mirror appendMessage sinks.

## Best Possible Solution

Review and land PR #73563 or an equivalent maintainer-approved fix that routes every bare transcript append sink through guardSessionManager with config where applicable, plus targeted regression tests for redaction and redactSensitive="off" behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path exists: send token-bearing text through the config transcript append, CLI text-turn transcript, gateway inject helper, or Codex mirror while transcript redaction is enabled; current main shows direct appendMessage calls on those paths while the guarded run paths apply the redaction hook.

## Solution Assessment

Yes. Reusing guardSessionManager is the narrowest maintainable direction because it applies the existing before-write hook, redaction, and tool-result guard infrastructure instead of adding a parallel transcript scrubber.
