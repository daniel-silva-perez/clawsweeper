## Summary

Keep open: current main still has direct transcript append sinks that bypass the existing guard redaction path, and the linked open PR #73563 is the active closing implementation candidate rather than a merged fix.

## What This Changes

Request to route core session, CLI attempt, gateway inject, and bundled Codex transcript append paths through the existing redaction guard before JSONL persistence.

## Best Possible Solution

Review and land PR #73563 or an equivalent maintainer-approved fix that wraps every bare transcript append sink in guardSessionManager, threads config where applicable, and adds regression coverage for redaction plus redactSensitive="off" behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path exists by sending token-bearing text through the config transcript append, CLI text-turn transcript, gateway inject helper, or Codex mirror while transcript redaction is enabled; current main shows direct appendMessage calls on those paths.

## Solution Assessment

Yes. Reusing guardSessionManager is the narrowest maintainable direction because it applies the existing before-write hook, transcript redaction, and tool-result guard infrastructure rather than adding a parallel scrubber.
