## Summary

Keep open. Current main has opt-in exponential temporal decay, hybrid BM25/vector scoring, and weighted dreaming promotion, but it does not implement the requested write-time 1-10 importance scoring, Weibull decay, access-boost persistence, or importance-weighted retrieval across the builtin and QMD memory paths. Related open work covers pieces of this, especially #71590 for Weibull decay and #43306 as a broader importance-weighted decay candidate, but nothing is merged or shipped enough to close this issue.

## Best Possible Solution

Keep this issue open as the focused feature request. The best path is a memory-core design/PR that adds durable importance metadata and access tracking, introduces a backward-compatible configurable Weibull decay model, threads the ranking signal through builtin retrieval and either QMD reranking or a documented QMD limitation, and updates config schema, docs, and targeted tests. #71590 and #43306 are useful implementation candidates, but this issue should remain open until a merged PR or a narrower canonical issue fully owns the remaining work.
