## Summary

Keep this issue open. Current main has opt-in exponential temporal decay, hybrid BM25/vector scoring, and short-term recall/dreaming promotion signals, but it still does not implement durable write-time importance scoring, Weibull decay, access-boost persistence that affects retrieval, or importance-weighted retrieval across the builtin and QMD memory paths. Related #71590 and #43306 are still open implementation candidates, not shipped replacements; the KinthAI comment adds useful production design context rather than proving OpenClaw core behavior exists.

## Best Possible Solution

Keep this issue open as the focused tracker. The best path is a memory-core design/PR that adds backward-compatible durable importance metadata and access tracking, implements configurable Weibull decay and retrieval weighting for builtin search, defines the QMD behavior or limitation explicitly, updates config schema/docs, and adds targeted tests. #71590 and #43306 should be reviewed as possible pieces of that work rather than treated as shipped closure.
