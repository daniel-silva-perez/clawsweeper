## Summary

Keep this issue open. Current main has opt-in exponential temporal decay, hybrid BM25/vector scoring, and dreaming promotion signals, but it still does not implement durable write-time importance scoring, Weibull decay, persisted access boosts, or importance-weighted retrieval across the builtin and QMD memory paths. Related #71590 and #43306 are open PRs, not shipped replacements; the KinthAI comment is useful production design context, not evidence that OpenClaw main already has the feature.

## Best Possible Solution

Keep this issue as the focused tracker. The best path is a memory-core design/PR that adds backward-compatible durable importance metadata and access tracking, implements configurable Weibull decay and importance-weighted retrieval for builtin search, defines QMD behavior or limitations explicitly, updates config schema/docs, and adds targeted tests. Review #71590 and #43306 as possible pieces of that work rather than treating them as shipped closure.
