## Summary

Keep this issue open. Current main still drops the requested Feishu card-action fields before agent dispatch, and the provided GitHub context shows open implementation PRs, including same-author PR #43953 with closing syntax for this issue.

## Best Possible Solution

Keep this issue open and review, land, or supersede the linked Feishu implementation PRs, especially same-author PR #43953. The durable fix should preserve `form_value`, `input_value`, `name`, `option`, and `options` from the raw `card.action.trigger` callback through parsing and typed event dispatch, serialize them only when present so simple button behavior stays backward compatible, and add parser-to-handler regression coverage.
