## Summary

Keep this issue open. Current main still drops the requested Feishu `card.action.trigger` action sibling fields before dispatch, and the provided GitHub context shows open implementation candidates, including same-author PR #43953 with closing syntax for this issue.

## Best Possible Solution

Keep this issue open and review, land, or supersede the linked Feishu implementation PRs, especially same-author PR #43953. The durable fix should preserve `form_value`, `input_value`, `name`, `option`, and `options` from raw callback parsing through typed dispatch, serialize them only when present to preserve simple button behavior, and add parser-to-handler regression coverage.
