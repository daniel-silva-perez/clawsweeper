## Summary

Keep this issue open. Current main still has a credible Windows path-containment failure path for valid workspace-relative IDENTITY.md avatars, and live GitHub state shows open PR #38923 still references this issue with closing syntax. Per cleanup policy, that PR is an implementation candidate, not a reason to close the issue before merge.

## Best Possible Solution

Keep this issue open. Review and either land PR #38923 or extract an equivalent narrow fix that normalizes Windows namespace and drive-letter path forms before workspace containment checks, preserves symlink and 2 MB protections, handles backtick-wrapped IDENTITY.md avatar values if maintainers want to cover the related report shape, and adds regression coverage for a Windows non-default agent resolving a workspace-relative IDENTITY.md avatar through both /avatar/<agentId>?meta=1 and /avatar/<agentId>. It would also be useful to surface too_large/outside_workspace avatarReason in the UI instead of leaving users with only a broken image.
