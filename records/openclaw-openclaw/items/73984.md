## Summary

Keep open. Current main still does not hydrate TUI slash-command autocomplete from the runtime command inventory: the TUI uses `getSlashCommands()` from static built-ins plus the chat-command registry, and the TUI backend/client do not expose `commands.list`. The runtime `commands.list` path already includes plugin commands, and Active Memory registers `/active-memory`, so this PR addresses a real remaining TUI gap rather than duplicating current main.

## Best Possible Solution

Keep this PR open as the focused implementation candidate. The best path is to review and validate wiring TUI autocomplete to `commands.list` in Gateway mode plus an embedded equivalent in local mode, refresh it when agent/provider context changes, preserve local argument completions on name collisions, and add a user-facing changelog entry if this lands as a TUI autocomplete fix.
