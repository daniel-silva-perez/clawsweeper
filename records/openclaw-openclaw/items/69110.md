## Summary

Keep open. Current main still does not pass resolved runtime provider/model identity into the awaited outbound `message_sending` delivery hook or `OutboundSessionContext`. Related open PRs propose overlapping hook/context changes, but none are merged on main and the remaining fix needs coordinated maintainer review.

## Best Possible Solution

Keep this issue open and coordinate one maintainer-approved provenance design: carry the resolved provider/model from the agent run/fallback result into the final reply assembly and/or awaited delivery hook context, document the public plugin contract, add focused tests in outbound delivery plus agent-runner final payload paths, and reconcile the overlap with #65149, #39761, and #44011 before merging competing hook APIs.
