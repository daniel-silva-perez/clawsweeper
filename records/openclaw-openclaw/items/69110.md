## Summary

Keep open. Current main still does not pass resolved runtime provider/model identity into the awaited outbound `message_sending` delivery hook, `OutboundSessionContext`, or the typed `PluginHookMessageSendingEvent`. The runner does know the actual provider/model after fallback, but that data is used for persistence/diagnostics rather than the final delivery enforcement seam. Related open PRs are implementation candidates, not merged fixes.

## Best Possible Solution

Keep this issue open and coordinate one maintainer-approved provenance design: carry the resolved provider/model from the agent run or fallback result into the final reply assembly and/or awaited delivery hook context, document the public plugin contract, add focused tests in outbound delivery plus agent-runner final payload paths, and reconcile overlap with #65149, #39761, and #44011 before merging competing hook APIs.
