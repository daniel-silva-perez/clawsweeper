## Summary

Keep this issue open. Current main still does not pass resolved runtime provider/model identity to the awaited `message_sending` delivery hook, even though that identity is available elsewhere for usage, diagnostics, fallback notices, and response-prefix interpolation. The related hook-context PRs are still open implementation candidates, not merged fixes.

## What This Changes

The issue requests gateway-level resolved provider/model identity to be passed into delivery hooks so plugins can detect or prevent mismatched model self-tags.

## Best Possible Solution

Keep this issue open and coordinate one maintainer-approved provenance design. Thread resolved provider/model identity from the agent run/fallback result into final reply assembly and the awaited `message_sending` delivery hook, expose additive typed fields for plugins, cover central outbound plus Telegram/direct paths with tests, update plugin-hook docs, and reconcile overlap with #39761, #65149, and #44011 before merging competing APIs.
