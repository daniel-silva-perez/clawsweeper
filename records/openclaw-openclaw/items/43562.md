## Summary

Keep open. Current main and the latest release still put the hardcoded personal-assistant identity before SOUL.md project context, and the supported config surface is still full system-prompt replacement rather than a focused identity/preamble override. The linked open PR #48442 remains the active implementation candidate, while #44497 was closed unmerged.

## What This Changes

The issue asks for the top system-prompt identity preamble to be configurable or neutralized when SOUL.md defines a persona, instead of always identifying the agent as a personal assistant.

## Best Possible Solution

Land #48442 or a narrow successor that adds a documented identity-line configuration path while preserving the current default output and full systemPromptOverride behavior, with focused tests for configured overrides, SOUL.md ordering, and promptMode none.
