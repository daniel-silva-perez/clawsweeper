## Summary

Keep open. Current main and the latest release still put the hardcoded personal-assistant identity ahead of SOUL.md context, and the only supported config path is full system prompt replacement. Open PR #48442 is the existing implementation candidate, so the issue should stay open until that PR or a narrow successor lands.

## What This Changes

Make the agent system-prompt identity preamble configurable so custom SOUL.md personas are not overridden by the hardcoded personal-assistant line.

## Best Possible Solution

Review and land #48442 or a narrow successor that adds a documented identity-line configuration path with focused tests for default behavior, configured overrides, SOUL.md interaction, promptMode none, and full prompt replacement compatibility.
