## Summary

Keep open. Current main and the latest release still emit the hardcoded personal-assistant identity before SOUL.md project context, and the supported config surface is still full system prompt replacement rather than a focused identity/preamble override. Open PR #48442 remains the active implementation candidate, while #44497 was closed unmerged.

## What This Changes

The issue requests a focused agent identity/preamble configuration path so custom agent personas are not overridden by the hardcoded personal-assistant line at the start of the system prompt.

## Best Possible Solution

Land #48442 or a narrow successor that adds a documented identity-line/preamble configuration path while preserving the current default output and full systemPromptOverride behavior, with focused coverage for configured overrides, SOUL.md ordering, promptMode none, and docs/schema alignment.

## Reproduction Assessment

Yes for the prompt-construction issue: current source deterministically emits the personal-assistant sentence before Project Context, and tests assert the SOUL.md guidance text in that lower section. I did not run live model calls in this read-only review, but the reporter and confirmer describe the same observable behavior.

## Solution Assessment

Unclear until maintainers choose the API shape. A narrow configurable identity line is more maintainable than forcing users to replace the entire system prompt, but #48442’s broader disabledSections addition needs normal maintainer review.
