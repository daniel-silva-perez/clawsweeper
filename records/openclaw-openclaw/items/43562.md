## Summary

Keep open. Current main still starts the assembled agent system prompt with the hardcoded personal-assistant identity, SOUL.md remains later Project Context with lower-priority wording, and the public config surface still only offers full prompt replacement rather than a focused identity-line setting. The matching open implementation PR #48442 is the right path to review before this issue can close.

## What This Changes

The issue requests a configurable leading agent identity preamble, or a neutral SOUL.md-aware default, instead of always starting prompts with the hardcoded personal-assistant sentence.

## Best Possible Solution

Review and either land #48442 or replace it with an equivalent focused config surface for the leading identity line, with documented precedence across defaults, per-agent config, SOUL.md interaction, and full prompt replacement behavior.
