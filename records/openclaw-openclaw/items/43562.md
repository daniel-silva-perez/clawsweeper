## Summary

Keep open. Current main still starts the assembled agent system prompt with the hardcoded personal-assistant identity, still places SOUL.md later under Project Context with lower-priority wording, and only exposes full system prompt replacement rather than a narrow identity-line/preamble setting. The matching open implementation PR #48442 is the right maintainer review path before this issue can close.

## What This Changes

The issue requests a configurable replacement for the hardcoded leading agent identity preamble so custom SOUL.md personas or per-agent/default config can define the agent identity.

## Best Possible Solution

Review and either land #48442 or replace it with an equivalent focused config surface for the leading identity line, with documented precedence across defaults and per-agent config plus tests covering SOUL.md interaction, default fallback, and full prompt replacement behavior.
