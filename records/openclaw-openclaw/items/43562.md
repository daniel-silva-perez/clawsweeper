## Summary

Keep open. Current main still puts the hardcoded personal-assistant identity at the top of generated prompts, SOUL.md guidance remains lower in Project Context, and the supported config surface is still full prompt replacement rather than a focused identity/preamble override. The related open PR #48442 is the active implementation candidate, so this should stay open for maintainer review instead of being closed or duplicated by a new repair lane.

## What This Changes

Make the first agent system-prompt identity line configurable or neutral so SOUL.md personas are not overridden by the hardcoded personal-assistant preamble.

## Best Possible Solution

Review and land #48442 or a narrow successor that adds a documented default and per-agent identity-line/preamble configuration path, preserves current default output and full `systemPromptOverride` behavior, and covers SOUL.md ordering plus `promptMode: "none"` in tests and docs/schema.

## Reproduction Assessment

Yes. The prompt construction path is deterministic: current source prepends the personal-assistant line and later emits SOUL.md guidance inside Project Context, so this can be reproduced by inspecting or invoking `buildAgentSystemPrompt` with a SOUL.md context file.

## Solution Assessment

Unclear until maintainers choose the final config shape. A narrow configurable identity line or preamble is more maintainable than requiring users to replace the entire assembled system prompt, while the active PR’s broader section-disabling surface needs review.
