## Summary

Keep open. Current main still emits the hardcoded personal-assistant identity before Project Context, the supported config surface still only offers full system-prompt replacement, and the latest release remains affected. The open related PR #48442 is the active implementation candidate, so this should stay with maintainer review rather than being closed or duplicated by a repair branch.

## What This Changes

Make the agent identity preamble configurable so SOUL.md-defined personas are not overridden by the hardcoded personal-assistant line.

## Best Possible Solution

Review and land #48442 or a narrow successor that adds a documented default and per-agent identity-line/preamble configuration path, preserves the existing default output and full `systemPromptOverride` behavior, and covers SOUL.md ordering plus prompt-mode behavior in tests and docs/schema.

## Reproduction Assessment

Yes. The prompt construction path is deterministic: inspecting or invoking `buildAgentSystemPrompt` with a SOUL.md context file shows the personal-assistant line emitted before the SOUL.md Project Context guidance.

## Solution Assessment

Unclear until maintainers choose the final config shape. A focused identity-line or preamble setting is more maintainable than requiring full prompt replacement, while the active PR’s broader section-disabling surface needs review.
