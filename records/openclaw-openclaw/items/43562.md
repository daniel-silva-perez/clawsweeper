## Summary

Keep open. Current main still starts the agent system prompt with the hardcoded personal-assistant identity and still places SOUL.md later under Project Context with lower-priority wording. The public config surface remains full-prompt replacement only, and the matching open implementation PR #48442 is the right item for maintainer review before this issue can close.

## What This Changes

Make the leading agent system-prompt identity line configurable, or neutral when SOUL.md exists, so custom agent personas are not overridden by the hardcoded personal-assistant preamble.

## Best Possible Solution

Add a documented, narrowly scoped identity-line configuration path with clear precedence and tests covering default behavior, per-agent/default overrides, SOUL.md interaction, promptMode none, and full prompt replacement compatibility.
