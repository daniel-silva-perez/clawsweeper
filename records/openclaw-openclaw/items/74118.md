## Summary

Keep open. The PR targets a real hot-path cache miss for inbound dispatch, but the proposed unconditional truthy active-registry fast path is too broad and can treat an empty placeholder registry as a completed runtime load.

## What This Changes

The PR imports `getActivePluginRegistry`, adds an early return in `ensureRuntimePluginsLoaded` when an active registry exists, and updates the focused runtime-plugin unit test to expect no resolver call on that path.

## Best Possible Solution

Keep the fast-path direction, but make the guard distinguish a real activated runtime registry from placeholder or unrelated active state, then add a regression test for the placeholder-first path while preserving the no-rebuild gateway dispatch case from #74117.

## Reproduction Assessment

Yes. The reported performance issue is source-reproducible: gateway boot and dispatch construct different loader option sets, and current main falls through to `loadOpenClawPlugins` on a strict cache-key miss. The PR-introduced regression is also reproducible with a focused unit path that calls `listChannelSetupPlugins()` before `ensureRuntimePluginsLoaded()`.

## Solution Assessment

No. Short-circuiting agent dispatch once a gateway registry is actually loaded is a good narrow fix, but `getActivePluginRegistry()` truthiness alone bypasses the loader compatibility contract and mistakes empty placeholders for completed runtime loads.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Require a loaded registry before short-circuiting:** `src/agents/runtime-plugins.ts:31-33`
  - body: `getActivePluginRegistry()` can be the empty placeholder created by `requireActivePluginRegistry()` before any real plugin load. If channel setup/status code runs first in a long-lived process, this new guard skips `resolveRuntimePluginRegistry`, so plugin hooks, tools, channels, and policies never load. Gate the fast path on a real activated/loaded registry, such as an active registry key or explicit loaded marker, and cover the placeholder-first path.
  - confidence: 0.86
