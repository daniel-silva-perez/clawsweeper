## Summary

Keep this PR open. Current main still rebuilds through `resolveRuntimePluginRegistry(loadOptions)` from `ensureRuntimePluginsLoaded`, so the PR is not obsolete and remains the focused fix vehicle for the linked performance bug #74117. The proposed diff is narrow and security-neutral, but its unconditional active-registry fast path still needs review against existing gateway-subagent compatibility and empty-placeholder registry behavior before merge.

## Best Possible Solution

Keep this PR open and revise it rather than close it. The best fix is a constrained fast path in `src/agents/runtime-plugins.ts` that treats an already-loaded active registry as sufficient for inbound dispatch, while preserving explicit gateway-binding fall-through and avoiding empty-placeholder registries. Add regression coverage for the zero-call dispatch no-op and for the gateway-binding/placeholder cases before landing; keep #74117 open until a PR merges.
