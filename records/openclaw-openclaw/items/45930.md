## Summary

Keep open: the PR targets a real Control UI churn problem, but the current patch weakens Logs tab live-tail behavior. Current main still polls logs every 2 seconds, while the PR changes that to 5 seconds even though log fetching remains capped to 500 lines / 250 KB and the server resets to the newest window when the cursor falls behind. The broader churn work in #69454 is still open, so this needs maintainer direction rather than cleanup closure.

## What This Changes

The PR replaces three inline Control UI polling interval literals with named constants and changes nodes/logs/debug polling from 5s/2s/3s to 30s/5s/10s in `ui/src/ui/app-polling.ts`.

## Best Possible Solution

Reduce unnecessary Control UI gateway churn while preserving Logs tab live-tail semantics: keep the logs cadence short unless the tail window/reset behavior is made lossless for slower polling, and handle nodes/debug load through tab-scoped or coordinated churn changes such as the #69454 direction.
