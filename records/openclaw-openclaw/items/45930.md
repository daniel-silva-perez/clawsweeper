## Summary

Keep this PR open for maintainer/author follow-up. The churn problem is real, but the patch is not mergeable as-is because slowing the Logs tab poll to 5 seconds can turn the existing bounded log-tail contract into skipped log lines on busy gateways, and the smaller interval-only change overlaps the broader open Control UI churn work in #69454.

## What This Changes

This PR extracts Control UI polling interval literals into named constants in `ui/src/ui/app-polling.ts` and changes nodes/logs/debug polling from 5s/2s/3s to 30s/5s/10s.

## Best Possible Solution

Reduce unnecessary Control UI gateway churn without weakening live-tail semantics: preserve the short Logs tab cadence or pair a slower cadence with a larger/safer tail contract, and handle nodes/debug load through tab-scoped polling or immediate refresh behavior coordinated with #69454.

## Review Findings

- **[P1] Restore the shorter logs polling cadence:** `ui/src/ui/app-polling.ts:21`
