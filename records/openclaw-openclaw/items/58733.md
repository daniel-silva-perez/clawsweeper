## Summary

Keep this issue open. Current `main` still gives the bundled Grok/xAI Responses web-search path the shared 30-second default when no timeout override is configured, and the related PR #43609 remains open/unmerged rather than shipped. The report is recent, specific, contributor-authored, and not covered by a protected-label exception or an implemented fix on `main`.

## Best Possible Solution

Keep this issue open and review/land PR #43609 or an equivalent focused fix. The maintainer path should add a Grok-specific no-override default, preserve explicit `tools.web.search.timeoutSeconds` or plugin-config overrides, and include regression coverage proving Grok no longer inherits the shared 30-second default while other providers keep their current behavior.
