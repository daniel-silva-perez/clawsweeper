## Summary

Keep open. Current main has useful partial hardening for web/file/hook content and inbound marker spoofing, but it still does not implement the central requested shared trust-boundary contract for non-owner current message bodies, generic/open-world tool results, and stable system-prompt guidance. The narrow linked implementation PR is closed unmerged, while the broader CaMeL PR remains open and is not a shipped replacement.

## What This Changes

Add structural prompt-injection defenses that mark untrusted message and tool-result content as data, add stable system-prompt guidance, and validate high-confidence injection patterns at ingest boundaries.

## Best Possible Solution

Implement or explicitly supersede this with a core-owned prompt-injection boundary design: stable pre-cache guidance, tested non-owner current-message marking during prompt assembly, shared classification/wrapping for open-world tool results, and provider-shape regression coverage across OpenAI, Anthropic, and Google while preserving existing external-content sanitizers.
