## Summary

Keep open. Current main has several partial defenses, including external-content wrappers for web/hook/media content, inbound system-tag neutralization, inter-session provenance labels, and owner-only tool policy. It still does not implement the central request: a shared trust-boundary contract for non-owner current message bodies and generic/open-world tool results, plus stable system-prompt guidance. The implementation PR #62973 is closed unmerged, and the broader CaMeL PR #39231 is still open rather than a shipped replacement.

## Best Possible Solution

Keep this issue open and implement a core-owned, tested trust-boundary strategy: add stable pre-cache system-prompt guidance, label or wrap non-owner current message bodies during prompt assembly, classify/wrap open-world tool results at a shared transport or tool-result boundary, preserve existing external-content sanitizer behavior, and add provider-payload regression tests. Treat #62973 as a closed unmerged attempt and #39231 as related broader CaMeL work, not as shipped coverage for this issue.
