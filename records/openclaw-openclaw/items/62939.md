## Summary

Keep open. Current main has useful targeted prompt-injection hardening for external web/file/hook content, but it still does not implement the requested shared trust-boundary contract for non-owner current message bodies, generic tool results, or stable system-prompt guidance. The narrow linked implementation PR is closed unmerged, and the broader CaMeL implementation remains open rather than shipped.

## What This Changes

The issue requests structural delimiters and stable prompt guidance so untrusted tool results, incoming non-owner messages, web/file/API content, and suspicious injection patterns are treated as data rather than instructions.

## Best Possible Solution

Implement or explicitly supersede this with a core-owned prompt-injection boundary design: stable pre-cache guidance, tested non-owner current-message marking, shared classification/wrapping for open-world tool results, and provider-shape regression coverage across OpenAI, Anthropic, and Google while preserving existing external-content sanitizers.
