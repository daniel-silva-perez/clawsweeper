## Summary

Keep open. Current main has useful targeted prompt-injection hardening for web fetches, hook payloads, media/file extraction, and inbound marker spoofing, but it still does not implement the requested shared trust-boundary contract for non-owner current message bodies, generic/open-world tool results, or stable system-prompt guidance. The narrow implementation PR #62973 is closed unmerged, and the broader CaMeL implementation PR #39231 remains open rather than shipped.

## What This Changes

Request to add structural trust delimiters, stable prompt guidance, source classification, and injection-pattern handling for externally sourced tool results, non-owner messages, web/file/API content, and persisted transcript inputs.

## Best Possible Solution

Implement or explicitly supersede this with a core-owned prompt-injection boundary design: cache-stable guidance, tested non-owner current-message marking during prompt assembly, shared classification/wrapping for open-world tool results, and provider-shape regression coverage across OpenAI, Anthropic, and Google while preserving existing external-content sanitizers.
