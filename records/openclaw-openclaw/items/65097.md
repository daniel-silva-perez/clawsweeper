## Summary

Keep this issue open. Current main still has the reported Dream Diary language gap: the narrative prompt and prompt builder are English-only, the narrative APIs and phase call sites only carry timezone/model, the resolved memory-core Dreaming config has no language or locale field, the strict plugin manifest would reject the requested key, and public docs list only enabled/frequency/model.

## What This Changes

Add a configurable Dreaming narrative language or locale option under `plugins.entries.memory-core.config.dreaming`, with broader optional ideas for auto-detection and custom prompt templates.

## Best Possible Solution

Implement a schema-backed explicit Dream Diary `language` hint in memory-core, resolve it through the memory-host Dreaming config path, pass it through light/REM/deep narrative generation, inject a concise output-language directive into the narrative prompt/run path, update docs/tests/changelog, and leave auto-detection plus custom prompt templates to separate product follow-up.
