## Summary

Keep this issue open. Current main still has the reported Dream Diary language gap: the narrative prompt is fixed English, the narrative runner and phase call sites only carry timezone/model, the resolved Dreaming config has no language or locale field, the strict manifest would reject the requested key, and docs expose no language setting.

## What This Changes

Add a configurable Dreaming narrative language or locale option under `plugins.entries.memory-core.config.dreaming`, with broader optional ideas for auto-detection and custom prompt templates.

## Best Possible Solution

Implement a schema-backed explicit Dream Diary `language` hint in memory-core, resolve it through the memory-host Dreaming config path, pass it through light/REM/deep narrative generation, inject a concise output-language directive into the narrative prompt/run path, update docs/tests/changelog, and keep auto-detection plus custom prompt templates as separate follow-up scope.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature request. The current gap is high-confidence from source inspection because the schema rejects language/locale keys and narrative generation never reads or carries an output-language setting.

## Solution Assessment

Yes, with scope adjustment. The maintainable path is a narrow memory-core config contract for an explicit language hint; auto inference and custom prompt templates are broader product choices and should not be mixed into the first fix.
