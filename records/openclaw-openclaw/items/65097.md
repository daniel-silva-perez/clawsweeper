## Summary

Keep this open because current main still has the reported Dreaming language gap: Dream Diary prompt text is fixed English, the narrative APIs only carry timezone/model, the resolved config has no language field, the manifest rejects unknown Dreaming keys, and docs expose only enabled/frequency/model as user-facing settings.

## What This Changes

The issue requests a configurable language or locale option for memory-core Dreaming narrative generation, with possible explicit language, auto-detection, or custom prompt-template support.

## Best Possible Solution

Add a documented, schema-backed explicit Dream Diary language hint in memory-core, resolve it through the memory-host Dreaming config path, pass it through light/REM/deep narrative generation, and inject a concise language directive into the subagent prompt path while leaving auto-detection and full prompt-template customization for separate product follow-up.
