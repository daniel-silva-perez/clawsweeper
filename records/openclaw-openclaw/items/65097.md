## Summary

Keep this issue open. Current main still has no Dreaming diary language/locale configuration: the narrative system prompt and user prompt are English-only, the narrative generation API only accepts timezone/model, the memory-core config resolver and manifest schema omit language/locale, and the public docs only document enabled/frequency/model. The related #69354 was closed as a duplicate of this feature request, not as a shipped fix.

## Best Possible Solution

Keep this open and implement the feature in `memory-core`: add a documented `dreaming.language` or `dreaming.locale` config contract, decide whether `auto` is supported, resolve the hint in the memory-host config path, thread it into Dream Diary subagent and request-scoped fallback narrative generation, update `extensions/memory-core/openclaw.plugin.json`, update the Dreaming and memory config docs, refresh generated config metadata if required, and add focused resolver/narrative/phase tests.
