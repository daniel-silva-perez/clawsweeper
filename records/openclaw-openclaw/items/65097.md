## Summary

Keep this issue open. Current main still has no Dreaming diary language or locale setting: the narrative prompt is fixed English, the narrative runner and phase call sites only carry timezone/model, the resolved memory-core config has no language field, the strict plugin manifest would reject language/locale keys, and the public docs only document enabled/frequency/model. Related #69354 adds duplicate user signal for the same language gap, while #69533 is the broader custom-prompt request.

## What This Changes

Add a configurable Dreaming diary language or locale setting so memory-core narrative generation can output in the user's preferred language instead of defaulting to English.

## Best Possible Solution

Implement this in memory-core as a documented Dream Diary language hint: add a schema-backed config field, resolve it through the memory-host Dreaming config path, pass it through light/REM/deep narrative generation, inject a concise language directive into both subagent and fallback prompt paths, update docs and focused tests, and leave full prompt-template customization to #69533.
