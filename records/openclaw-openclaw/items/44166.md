## Summary

Keep this issue open. Current main still does not classify the reported transient embedding transport failures as retryable, and the batch embedding path still has no recursive split-after-retry fallback. The provided GitHub context also shows same-author PR #44167 is open with closing syntax, so that PR or an equivalent fix should be reviewed before this issue is closed.

## Best Possible Solution

Keep this issue open and review or update #44167 as the immediate implementation candidate. The best fix belongs in memory-core: extend the retry classifier for narrow transient transport failures, add recursive split-after-exhausted-retry behavior for multi-item batches while preserving order, and add focused regression tests for retry-before-split, terminal single-item failures, and non-transport service errors that should not split.
