## Summary

Current main still does not implement the requested transient embedding transport retry classification or recursive batch-split fallback. This issue should remain open because same-author PR #44167 is still open and explicitly proposes the targeted fix with closing syntax.

## Best Possible Solution

Keep this issue open and review or update #44167 as the immediate implementation candidate. The memory-core fix should narrowly extend retry classification for transient transport failures, add recursive split-after-exhausted-retry behavior for multi-item embedding batches while preserving result order, and add focused regression tests for retry-before-split, terminal single-item failures, and non-transport service errors that should not split.
