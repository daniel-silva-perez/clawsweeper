## Summary

Keep open. Current main at 9bb1e59447a5fdf2c0164672df6c748188b26442 still has the narrow memory embedding retry classifier and still lacks the requested recursive split fallback for exhausted multi-item embedding batches. The issue is also paired with open PR #44167, so the right next step is review, update, land, or explicitly replace that PR rather than close this issue now.

## Best Possible Solution

Keep this issue open and review, update, or land PR #44167 as the focused implementation candidate. The fix should remain in memory-core: classify narrow transient embedding transport failures as retryable, reuse the existing backoff loop, recursively split exhausted multi-item text and structured embedding batches while preserving order, and add targeted regression tests for retry-before-split, single-item terminal failures, structured inputs, and non-transport service errors that should not split. Broader rollback/cache preservation work can remain tracked by adjacent PRs such as #59137.
