## Summary

Keep open. Current main at 1b25dcf57a9fca5d00cf275046e20a2e3ea3730c still has the narrow memory embedding retry classifier and still lacks the requested recursive split fallback for exhausted multi-item embedding batches. The issue is also paired with open PR #44167, which is the immediate implementation candidate and should be reviewed or replaced before this issue closes.

## Best Possible Solution

Keep this issue open and review, update, or land PR #44167 as the focused implementation candidate. The fix should remain in memory-core: classify narrow transient embedding transport failures as retryable, reuse the existing backoff loop, recursively split exhausted multi-item text and structured embedding batches while preserving order, and add targeted regression tests for retry-before-split, single-item terminal failures, and non-transport service errors that should not split.
