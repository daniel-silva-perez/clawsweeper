## Summary

Keep this issue open. Current main still lacks the requested transient embedding transport retry classification and recursive multi-item batch split fallback, and same-author PR #44167 remains open with explicit closing syntax as the implementation candidate.

## Best Possible Solution

Keep this issue open and review or update #44167 as the immediate implementation candidate. The fix should narrowly classify transient transport failures as retryable, reuse the existing backoff loop, recursively split exhausted multi-item embedding batches while preserving order, and add regression tests for retry-before-split, terminal single-item failures, and non-transport service errors that must not split.
