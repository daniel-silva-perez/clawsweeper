## Summary

Current main still performs cron startup catch-up and does not expose a global or per-job way to skip missed recurring slots after a gateway restart. The request is recent, concrete, unprotected, and belongs in OpenClaw cron scheduling/config rather than cleanup closure.

## Best Possible Solution

Keep this issue open. Implement an explicit startup missed-run policy, preferably defaulting to the current `run` behavior with a global config override and/or per-job `missedPolicy: "run" | "skip"`. Wire it into `planStartupCatchup` before jobs are marked running, update config/types/protocol/CLI/docs, and add restart-catch-up regression tests for global disabled and per-job skip behavior.

## Risks / Open Questions

- Changing the default globally could cause users to miss important reminders or operational jobs after short outages; default behavior should likely remain `run` for compatibility.
- A per-job policy touches persisted cron job shape, gateway protocol schema, CLI/API validation, config docs, and restart-catch-up tests, so the implementation should be coordinated across those surfaces.
