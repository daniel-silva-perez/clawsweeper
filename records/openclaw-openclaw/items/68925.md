## Summary

Current main does not implement this PR's intended UI changes: approval buttons still use the old labels, `handleExecApprovalDecision` still removes the active id only, and it does not refresh overview state or add an `approval-resolved` UI log. The PR is small and security-sensitive only in the approval-UX sense; it does not add dependency, workflow, release, or secret-handling changes. Keep it open for normal maintainer review rather than closing as implemented.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to keep the focused approval UI polish if maintainers want it, but update `handleExecApprovalDecision` so any synthetic approval-resolved entry is written to the canonical event buffer or is otherwise covered by the gateway broadcast path, preserve the restored `loadOverview({ refresh })` and reactive state fields, and add a small targeted UI/state test for the approval-resolution flow before merge.
