## Summary

Keep this PR open. Current main still does not fire internal `command:daily`/`command:idle` hooks or typed `before_reset` hooks from the lazy daily/idle rollover path, and the bundled `session-memory` hook still only accepts manual `new`/`reset` command actions. The live PR is an active, maintainer-repaired, merge-ready candidate for the remaining behavior rather than obsolete work.

## Best Possible Solution

Keep this PR open and review/land it as the canonical fix for lazy daily/idle session reset hooks. After it lands, use it to close or supersede the older overlapping PR #50933 and the related daily/idle session-memory issues it covers.
