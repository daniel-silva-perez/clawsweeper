## Summary

Keep this PR open. Current main still does not implement the PR's intended behavior: stale daily/idle rollovers only fire the plugin `session_end`/`session_start` lifecycle hooks, while internal command hooks and the typed `before_reset` plugin hook remain tied to manual `/new` and `/reset` handling. The PR is a focused implementation candidate for a still-real gap, and the supplied discussion shows review issues were addressed rather than making the work obsolete.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The best path is to rebase it onto current main, preserve the fixes from the review discussion, verify targeted tests for lazy daily/idle internal hooks plus `before_reset`, and then review/land it or replace it with an equivalent core patch that wires the same behavior.
