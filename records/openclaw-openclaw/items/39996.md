## Summary

Current main still does not implement this PR's central status-output change. The status overview path is now shared through `status-overview-surface.ts`, `status-overview-rows.ts`, and `status-all/report-data.ts`; that pipeline does not carry `tools.profile`, has no `Global tools profile` row, and has no formatter equivalent to the PR's `status.tool-profile.ts`. Related docs/config-help PRs clarify tool profiles but do not supersede the requested `openclaw status` / `openclaw status --all` overview behavior.

## Best Possible Solution

Keep this PR open. Refresh it against the current shared status overview pipeline, decide whether status should show only the global baseline or an effective-profile summary, and add focused regression coverage for both `openclaw status` and `openclaw status --all`.
