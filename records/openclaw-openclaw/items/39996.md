## Summary

Keep open. Current main still does not implement the requested status overview row, and the related docs/config-help work does not supersede the CLI status-output behavior. The PR remains useful, but its branch targets the older per-command overview construction and needs to be refreshed for the current shared status overview pipeline.

## What This Changes

The PR adds a shared formatter and inserts a `Global tools profile` overview row into `openclaw status` and `openclaw status --all` when `tools.profile` is configured.

## Best Possible Solution

Implement the status hint in the current shared overview pipeline: carry the chosen profile summary through `StatusOverviewSurface`, render it once for both status commands, and add focused tests for unset, `messaging`, and `full` cases. The final wording should explicitly distinguish a global config baseline from per-agent/provider effective profile overrides, or intentionally switch to an effective-profile summary if maintainers prefer that diagnostic surface.

## Reproduction Assessment

Yes. The requested behavior can be reproduced by configuring `tools.profile` and checking the Overview section, and current-main source/tests show there is no formatter or row that could render it today; I used source inspection rather than running the CLI to keep the checkout read-only.

## Solution Assessment

No. The PR's direct insertion was plausible for its older base, but current main now routes both commands through shared overview helpers, so the maintainable fix is to thread this through the shared surface and row builder with tests.
