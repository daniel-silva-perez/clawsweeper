## Summary

Keep this PR open. Current main still exposes a `filename` parameter on the message tool but does not carry it through normal `send` delivery into `MessageSendParams`, outbound payload/context, or Feishu `sendMediaFeishu`. The PR targets a real gap, but it is not safe to close as implemented or merge as-is because the supplied diff still mixes unrelated security-sensitive repo changes and the review thread identifies Feishu media regressions.

## Best Possible Solution

Keep this PR open as an implementation candidate, but request a narrow revision or replacement. The right fix should thread `filename` through `executeSendAction`, `MessageSendParams`, outbound payload/context, and Feishu `sendPayload`/`sendMedia` into `sendMediaFeishu`; preserve `loadWebMedia`/`mediaLocalRoots` and `sanitizeFileNameForUpload`; add regression tests for explicit filename forwarding, basename fallback, remote non-image URLs, local-root rejection, and upload sanitization; and remove unrelated workflow, CODEOWNERS, local-script, docs, skills, lockfile, and OpenAI WebSocket changes.
