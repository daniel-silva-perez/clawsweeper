## Summary

Keep open. The filename-plumbing idea is useful and current main still does not carry an explicit `filename` through the normal send path into Feishu media delivery, but this PR is not mergeable as-is because it mixes the Feishu fix with broad unrelated repository changes and introduces confirmed Feishu remote-upload, local-root, CODEOWNERS, cache-isolation, and secret-handling regressions.

## What This Changes

The PR threads optional filename values through outbound message contracts into Feishu media upload, adds Feishu `sendPayload` and local-file handling, and also changes workflows, CODEOWNERS, local docs/scripts, skills, OpenAI WebSocket headers, and a uv lockfile.

## Best Possible Solution

Land a narrow Feishu/outbound fix that carries explicit filenames through the send contracts and adapter, while preserving `loadWebMedia`, `mediaLocalRoots`, and `sanitizeFileNameForUpload`, dropping unrelated repo metadata/local tooling/provider changes, and adding regression tests for explicit filename forwarding, basename fallback, remote non-image URLs, local-root rejection, and filename hardening.
