## Summary

Keep open. Current main still lacks the filename-preservation path that this PR is trying to add, but the proposed patch is not mergeable as-is because it mixes the Feishu fix with unrelated repository/workflow/local-tool changes and introduces concrete Feishu media and repository-safety regressions.

## What This Changes

The PR threads attachment filename values through outbound send/Feishu delivery while also adding a Feishu local-file upload path and unrelated workflow, CODEOWNERS, local docs/scripts, skill, lockfile, and OpenAI WebSocket changes.

## Best Possible Solution

Request a narrow revision or replacement that only threads filename through the existing outbound send contracts into Feishu media upload, keeps `loadWebMedia`/`mediaLocalRoots` and `sanitizeFileNameForUpload` intact, and adds focused regression coverage for explicit filename forwarding, basename fallback, remote non-image URLs, local-root rejection, and upload sanitization.
