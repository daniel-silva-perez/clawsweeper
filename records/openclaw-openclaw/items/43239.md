## Summary

Keep open. The underlying Feishu filename-preservation work is still useful because current main does not carry `filename` through the normal send pipeline, but this PR is not safe to merge: it mixes that fix with broad unrelated repository changes and introduces Feishu media/security regressions already called out in the discussion.

## What This Changes

The PR threads optional attachment filename values through outbound send contracts into Feishu media upload, adds Feishu `sendPayload` and local non-image upload handling, and also changes workflows, CODEOWNERS, local helper docs/scripts, skills, OpenAI WebSocket headers, and a lockfile.

## Best Possible Solution

Ask for a narrow rebase or replacement that only carries `filename` from the message tool/outbound payload contracts into Feishu media sending, keeps `loadWebMedia`, `mediaLocalRoots`, and `sanitizeFileNameForUpload` intact, and adds focused regression tests for explicit filename forwarding, basename fallback, remote non-image URLs, local-root rejection, and upload filename hardening.
