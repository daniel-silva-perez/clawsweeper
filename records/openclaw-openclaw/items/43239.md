## Summary

Keep open. Current main still does not thread a caller-provided filename through the normal `send` delivery path, so the PR is not obsolete; however, this branch is not mergeable as-is because it mixes the useful Feishu filename work with unrelated repository changes and introduces blocking Feishu media and security regressions.

## What This Changes

The PR threads optional attachment filename values through outbound send contracts into Feishu media upload, adds Feishu payload/local-file handling, and also changes workflows, CODEOWNERS, local helper docs/scripts, skills, OpenAI WebSocket headers, and a lockfile.

## Best Possible Solution

Ask for a narrow rebase or replacement that only carries filename from the message tool/outbound payload contracts into Feishu media sending, preserves `loadWebMedia`, `mediaLocalRoots`, and `sanitizeFileNameForUpload`, removes unrelated CI/local docs/scripts/skills/lockfile changes, and adds focused regression tests for filename forwarding, basename fallback, remote non-image URLs, local-root rejection, and upload filename hardening.
