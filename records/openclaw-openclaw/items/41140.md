## Summary

Keep open. Current main still has adjacent exec-approval hardening, but not the requested pre-allowlist deny rules that match full command content. The broader #6615 tracker is open and relevant, and the CRE/SafeClaw comments plus closed #47848 are useful prior art, but none is a merged OpenClaw core implementation or an explicit maintainer supersession of this focused report.

## What This Changes

The issue requests exec-approval deny patterns that evaluate full command content before allowlist decisions, so shell-wrapped dangerous commands can be blocked even when the shell binary is allowed.

## Best Possible Solution

Define an additive native exec approvals policy that evaluates deny rules before broad allow decisions, matches canonical command text plus argv/shell payload consistently across gateway and node execution, exposes the policy in the approvals CLI and Control UI, documents precedence and limitations, and either folds this focused shell-bypass report into #6615 or keeps this as the canonical narrow tracker until the reviewed implementation ships.
