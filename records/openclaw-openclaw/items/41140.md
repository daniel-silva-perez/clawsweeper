## Summary

Keep this issue open. Current main has several adjacent exec-approval hardening changes, but it still does not implement configurable command-content deny patterns evaluated before allowlist matches. The discussion is useful: commenters linked CRE as external prior art for full-command regex blocking, and PR #47848 attempted similar work, but that PR is closed unmerged and the core behavior is not shipped.

## Best Possible Solution

Keep the issue open as focused core exec-approval hardening. A complete fix should add first-class deny rules to the exec approvals schema/storage and protocol, evaluate them before allowlist and ask-fallback decisions against a canonical command string plus argv/shell payloads, expose management in the approvals CLI and Control UI, document precedence, and add regression coverage for shell-wrapper bypasses such as `zsh -c "openclaw gateway stop"` when the shell binary is allowlisted. Coordinate with #6615, but close only after maintainers pick a canonical tracker or the reviewed implementation ships.
