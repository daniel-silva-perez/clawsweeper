## Summary

Keep open. Current main still lacks native pre-allowlist full-command deny rules: the exec approvals contract, gateway/node enforcement, protocol schema, CLI/UI, and docs remain allowlist-oriented. The broader #6615 tracker is relevant and #47848 was a closed unmerged attempt, so there is no implemented fix or clear maintainer supersession of this focused shell-wrapper bypass report.

## What This Changes

The issue requests exec-approval deny patterns that match full command content before allowlist decisions so shell-wrapped dangerous commands cannot be allowed merely because the shell binary is allowed.

## Best Possible Solution

Define an additive native exec approvals policy that evaluates deny rules before broad allow decisions, matches canonical command text plus argv and shell payload consistently across gateway and node execution, exposes the policy in the approvals CLI and Control UI, documents precedence and limitations, and either folds this focused shell-bypass report into #6615 or keeps it as the narrow tracker until the reviewed implementation ships.
