## Summary

Keep open. Current main still has no native pre-allowlist full-command deny rule surface: the policy types, config model, protocol schema, approval decision helper, CLI/UI, tests, and docs remain allowlist-oriented. The broader #6615 tracker is relevant and #47848 was a closed unmerged implementation attempt, but neither supersedes this focused shell-wrapper bypass report.

## What This Changes

The issue requests exec-approval deny patterns that match full command content before allowlist decisions so dangerous shell-wrapped commands cannot be allowed solely because the shell binary is allowlisted.

## Best Possible Solution

Define an additive native exec approvals policy that evaluates deny rules before broad allow decisions, matches canonical command text plus argv and shell payload consistently across gateway and node execution, exposes the policy in the approvals CLI and Control UI, documents precedence and limitations, and either folds this focused report into #6615 or keeps it as the narrow tracker until reviewed code ships.
