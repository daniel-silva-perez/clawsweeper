## Summary

Keep this PR open. Current main at 14f140d6f0d31fed802869622c38c7e8f8e19587 still does not implement Telegram forum topic deletion through the message action surface: the Telegram plugin exposes `delete`, `topic-create`, and `topic-edit`, but no `topic-delete`/`deleteForumTopic` action, runtime handler, transport helper, docs, or contract-test coverage. The PR remains a useful implementation candidate for the deletion slice of #11414, but it needs a current-main port and maintainer review of the destructive ID-parsing feedback.

## Best Possible Solution

Keep this PR open as useful implementation work. The best path is to port it onto current main by adding a Telegram `topic-delete` or `deleteForumTopic` action path in `extensions/telegram/src/channel-actions.ts`, implementing `deleteForumTopicTelegram` in `extensions/telegram/src/send.ts`, wiring a runtime branch in `extensions/telegram/src/action-runtime.ts`, updating shared message-action metadata only if maintainers want the public alias, documenting the user-facing behavior, and adding focused tests for message delete versus topic delete routing, delete gating, cfg/account pass-through, and decimal-only explicit ID validation. Keep #11414 as the broader umbrella for remaining Telegram forum topic lifecycle work.
