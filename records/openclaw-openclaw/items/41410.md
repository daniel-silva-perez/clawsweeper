## Summary

Keep this PR open. Current main at e583db63c602d8d21b4ebdcd119061571a2e40a9 still does not implement Telegram forum topic deletion through the message action surface: `delete` maps only to `deleteMessage`, `topic-create` and `topic-edit` exist, but there is no `topic-delete`/`deleteForumTopic` action, runtime branch, transport helper, public docs entry, shared action metadata, or focused test coverage. The PR remains a useful implementation candidate, but it needs a current-main port and security review around destructive ID parsing.

## Best Possible Solution

Keep this PR open as useful implementation work. The best path is to port the deletion slice onto current main by adding a Telegram `topic-delete` or `deleteForumTopic` action path in `extensions/telegram/src/channel-actions.ts`, implementing `deleteForumTopicTelegram` in `extensions/telegram/src/send.ts`, wiring a runtime branch in `extensions/telegram/src/action-runtime.ts`, updating shared message-action metadata only if maintainers want the public alias, documenting the user-facing behavior, and adding focused tests for message delete versus topic delete routing, delete gating, cfg/account pass-through, proxy behavior, and decimal-only explicit ID validation. Keep #11414 as the broader forum topic lifecycle umbrella.
