## Summary

Current main at 955b4df0939f90b4b6de5e7e6f3149dd125e1e50 still does not implement the PR's intended Telegram forum topic deletion path. The current Telegram message action map routes `delete` only to `deleteMessage`, advertises `topic-create`/`topic-edit` but not `topic-delete`, has no `deleteForumTopic` runtime branch or transport helper, and the docs/tests likewise only cover message deletion. The PR remains useful implementation work, but it should stay open for a current-main port and a final security pass on decimal-only destructive ID parsing.

## Best Possible Solution

Keep this PR open as useful implementation work. The best path is to port the Telegram forum-topic deletion slice onto current main by adding a dedicated `topic-delete` or approved `deleteForumTopic` action path in `extensions/telegram/src/channel-actions.ts`, implementing `deleteForumTopicTelegram` in `extensions/telegram/src/send.ts`, wiring `extensions/telegram/src/action-runtime.ts`, updating shared action metadata and `docs/channels/telegram.md` only for the accepted public surface, and adding regression tests for gating, cfg/account/proxy pass-through, message-delete fallback behavior, and decimal-only explicit ID validation. Keep #11414 as the broader forum topic lifecycle umbrella.
