## Summary

Keep this PR open. Current main at 891c7d9f1c1f5e0f52935fadfeacf695b0ef1eb8 still does not implement Telegram forum topic deletion through the message action surface, and this PR remains a plausible implementation candidate for one concrete part of open umbrella #11414. ## Best Possible Solution

Keep this PR open as useful implementation work. The best path is to rebase/port it onto the current Telegram plugin layout, implement `deleteForumTopicTelegram` in `extensions/telegram/src/send.ts`, add a `deleteForumTopic` runtime branch in `extensions/telegram/src/action-runtime.ts`, expose `topic-delete` in the shared action contract if maintainers want that alias, document the user-facing action, and add focused tests for message delete vs topic delete routing, gating, cfg pass-through, and decimal-only destructive ID validation. Keep #11414 as the broader umbrella for remaining Telegram forum topic lifecycle work.
