## Summary

Keep this PR open. Current main at 6d323ee73640afdf02ed9fa3aded13da23359b98 still does not implement Telegram forum topic deletion through the message action surface: there is no `topic-delete` action, no `deleteForumTopic` runtime action, and no `deleteForumTopicTelegram` transport helper. The PR remains a plausible implementation candidate for the deletion slice of open umbrella #11414, but it needs a current-main port and the remaining destructive-ID parsing/security review feedback resolved before merge.

## Best Possible Solution

Keep this PR open as useful implementation work. The best path is to rebase/port it onto current main by adding a Telegram `topic-delete` or `deleteForumTopic` action path, implementing `deleteForumTopicTelegram` in `extensions/telegram/src/send.ts`, updating the shared action contract only if maintainers want the public alias, documenting the user-facing behavior, and adding focused tests for message delete vs topic delete routing, gating, cfg pass-through, and decimal-only explicit ID validation. Keep #11414 as the broader umbrella for remaining Telegram forum topic lifecycle work.
