## Summary

Keep this PR open. Current main at bdfb408ce6eeb724a638c106104dbbcae50b612c still lacks Telegram forum topic deletion through the message action surface: `delete` maps only to message deletion, `topic-delete` is not a recognized shared or Telegram action, and there is no `deleteForumTopicTelegram` transport helper. The PR remains a plausible implementation candidate for the deletion slice of open umbrella #11414, but it needs maintainer review, a current-main port, and the destructive ID parsing concern resolved before merge.

## Best Possible Solution

Keep this PR open as useful implementation work. The best path is to port it onto current main by adding a Telegram `topic-delete`/`deleteForumTopic` action path, implementing `deleteForumTopicTelegram` in `extensions/telegram/src/send.ts`, updating shared action-name/target-mode support if maintainers want the public alias, documenting the user-facing behavior, and adding focused tests for message delete vs topic delete routing, gating, cfg pass-through, and decimal-only explicit ID validation. Keep #11414 as the broader umbrella for remaining Telegram forum topic lifecycle work.
