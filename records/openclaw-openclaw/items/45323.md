## Summary

Keep open. Current main has Control UI slash-command autocomplete, but not Slack-style @agent mention autocomplete or mention chips in the chat composer/history. The existing `agents.list` RPC already exposes agent ids and identity metadata that could back the feature, but the UI still uses a plain textarea and only calls the slash-menu updater on input.

## Best Possible Solution

Keep the issue open. A good implementation would add Control UI @agent autocomplete and keyboard/a11y behavior in the chat composer, likely backed by the existing `agents.list` metadata unless maintainers want a smaller additive directory RPC. If rich chips are added, keep `chat.send` serialization as plain text, add parser/rendering coverage for chat history, and update `docs/web/control-ui.md` plus protocol docs only if a new Gateway method is introduced.
