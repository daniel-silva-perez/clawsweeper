## Summary

Keep this open. Current main still has slash-command autocomplete and reusable agent-list data, but it does not implement @mention suggestions, mention chips, known-agent validation, or mention rendering in Control UI chat history.

## What This Changes

Add Slack-style @agent-name autocomplete, styled mention chips, plain-text serialization, and known-agent mention rendering to the Control UI chat composer/history.

## Best Possible Solution

Treat this as a Control UI product task: reuse agents.list where possible, add a tested @mention autocomplete flow in the composer, serialize selected mentions back to plain @agent text for chat.send, render only known-agent mentions in history, and make any Gateway protocol change additive only if current agent summaries lack required data.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a runtime bug, but yes for the current-main absence check: typing @ has no code path to open a mention menu because the composer only calls updateSlashMenu and searches found no mention chip or @mention implementation.

## Solution Assessment

Is this the best way to solve the issue? Unclear for the exact proposed agents.directory/contenteditable plan; the maintainable path is to decide the composer UX first and extend the Gateway protocol only if agents.list cannot support the directory needs.
