## Summary

Keep open. Current main still has a plain textarea composer with slash-command-only autocomplete and markdown-only history rendering; the available agent-list data is not wired into @mention autocomplete, chips, validation, or history rendering.

## What This Changes

The request is to add Slack-style @agent-name autocomplete in Control UI chat, including known-agent filtering, inline mention chips, plain-text serialization, history rendering, keyboard support, accessibility, and any needed gateway directory metadata.

## Best Possible Solution

Keep the issue open for a Control UI product/UI pass that reuses agents.list where possible, adds a tested @mention autocomplete flow, and only extends the gateway protocol additively if the existing AgentSummary contract is insufficient. The end state should cover composer behavior, serialization to plain @agent text, known-agent history rendering, accessibility, and focused UI tests.

## Reproduction Assessment

Not applicable as a bug reproduction, but there is a high-confidence current-main absence check: the composer is a textarea, updateSlashMenu only handles leading slash commands, and search found no mention-chip or @mention autocomplete implementation.

## Solution Assessment

Unclear for the proposal as written. The requested UX is reasonable, but adding a duplicate agents.directory endpoint is probably not the narrowest first step while agents.list already exposes ids, names, and identity metadata; the safer path is to decide the composer UX first and keep any protocol change additive.
