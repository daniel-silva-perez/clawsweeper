## Summary

Keep open. Current main still has a textarea-based Control UI composer with slash-command-only autocomplete and markdown-only chat history rendering; the existing agent directory data is available through `agents.list`, but it is not wired into `@` mention suggestions, mention chips, validation, or known-agent rendering.

## What This Changes

Add Slack-style `@agent` autocomplete, inline mention chips, plain-text serialization, and known-agent mention rendering to the Control UI chat.

## Best Possible Solution

Keep this as a Control UI product task: reuse `agents.list` where possible, add a tested @mention autocomplete flow, serialize selected mentions back to plain `@agent` text for `chat.send`, render only known-agent mentions in history, and add an additive Gateway protocol change only if the current agent summary data is insufficient.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a runtime bug, but yes for the current-main absence check: typing `@` in the textarea has no code path to open a mention menu because input handling only calls the slash-command updater and repository search found no mention chip or @mention implementation.

## Solution Assessment

Is this the best way to solve the issue? Unclear for the proposal as written because a separate `agents.directory` endpoint may duplicate `agents.list`; the safer maintainable direction is to decide the composer UX first and extend the protocol additively only if the existing agent summary data is not enough.
