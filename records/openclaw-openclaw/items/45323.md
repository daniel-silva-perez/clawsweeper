## Summary

Keep open. Current main still only has slash-command autocomplete plus reusable agent-list data; it does not implement @mention autocomplete, mention chips in the composer, validation, or known-agent mention rendering in chat history.

## What This Changes

The request is to add Slack-style @agent mention autocomplete, styled mention chips, known-agent validation, and mention rendering to the Control UI chat experience.

## Best Possible Solution

Keep this open for a Control UI product/UI pass that reuses agents.list where possible, adds a tested @mention autocomplete flow, and only expands the gateway protocol if AgentSummary lacks required metadata. The end state should include composer behavior, serialization to plain @agent text, known-agent history rendering, accessibility, and focused UI tests.

## Reproduction Assessment

Not applicable as a bug reproduction, but there is a high-confidence current-main absence check: the composer is still a textarea, updateSlashMenu only handles leading slash commands, and repository search found no mention-chip or @mention autocomplete implementation.

## Solution Assessment

Unclear for the proposal as written. The requested user experience is reasonable, but adding a new agents.directory RPC is probably not the narrowest solution unless agents.list proves insufficient; the safer path is to decide the composer model first and keep protocol changes additive.
