## Summary

Keep open. Current main still allows omitted cache pricing fields to reach the shared cost estimator, where they make the estimate non-finite and suppress the OpenTelemetry cost metric; the timeline also links an open external fix proposal, so this should be reviewed, landed, or replaced rather than closed.

## What This Changes

The issue requests defaulting omitted `cacheRead` and `cacheWrite` model cost fields to zero so OpenTelemetry cost metrics are still emitted when input/output pricing exists.

## Best Possible Solution

Normalize omitted cache price multipliers to zero at the shared pricing boundary or defensively inside `estimateUsageCost`, preserve tiered pricing behavior, and add regression coverage for direct config/models lookup plus diagnostics cost emission.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to pass a cost object with finite `input`/`output` but omitted `cacheRead`/`cacheWrite` through `estimateUsageCost`; current main multiplies zero cache usage by undefined price fields and returns `undefined`.

## Solution Assessment

Yes. Defaulting omitted cache price fields to zero is the narrow maintainable fix; the safest implementation is shared cost normalization or estimator-side coercion with focused regression tests.
