## Summary

Keep this PR open. Current main still lacks the NUL-byte stripping that the PR proposes for iMessage echo-cache text normalization, so the branch remains a narrow implementation candidate rather than obsolete or already shipped.

## Best Possible Solution

Review and land this focused iMessage echo-cache normalization PR, or land an equivalent patch that strips NUL-prefixed echoed text and adds regression coverage for single and repeated leading NUL bytes. Keep broader U+FFFD or general control-character normalization as a separate maintainer decision unless intentionally combined.
