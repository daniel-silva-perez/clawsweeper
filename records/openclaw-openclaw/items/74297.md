## Summary

Keep open. Current main appears to avoid the exact wrong-header path for API-key-shaped Anthropic credentials, but it still accepts and can select Anthropic `type: "token"` profiles without a targeted Enterprise/API-key diagnostic or clear docs warning.

## What This Changes

The issue requests validation, diagnostics, and documentation so Anthropic Enterprise/API-only credentials use `type: "api_key"` with `key` instead of `type: "token"` in `auth-profiles.json`.

## Best Possible Solution

Preserve supported Anthropic setup-token/Claude CLI token auth, but add a provider-owned warning/doctor/status path for Anthropic token profiles whose stored value looks like an Anthropic API key, and document that Enterprise/API-only Anthropic accounts should use `api_key` with `key`.

## Reproduction Assessment

Partially yes. Current-main source gives a high-confidence static reproduction for the missing validation/docs path, but not for the live Anthropic 529 loop because that needs an Enterprise credential and provider response.

## Solution Assessment

Not applicable as a patch review. Removing `token` from the global schema is not the best fix because Anthropic setup-token auth is intentionally supported; a narrow Anthropic diagnostic plus docs update is safer.
