## Summary

Keep open. Current main still has the gateway shutdown watchdog and bounded websocket/HTTP shutdown hardening, but the remaining reported child-process/pipe-handle path is not implemented: ProcessSupervisor-managed exec/CLI children are still not enumerated, cancel-all'ed, or bounded-drained by gateway shutdown. The issue discussion also explicitly says merged PR #61565 fixed an adjacent websocket close blocker, not this subprocess blocker.

## Best Possible Solution

Keep this issue as the canonical tracker for gateway shutdown blocked by active child-process or pipe handles. The best fix is a narrow core shutdown contract: expose active ProcessSupervisor run enumeration plus bounded cancel/drain APIs, wire that into gateway shutdown before the force-exit watchdog can fire, add privacy-safe diagnostics for active run ids/backends/pids/scope, and cover the behavior with focused supervisor and gateway shutdown tests. Treat #61565 as adjacent websocket hardening only.

Expected boundary:
- Add a generic ProcessSupervisor shutdown surface in `src/process/supervisor/types.ts` and `src/process/supervisor/supervisor.ts`, such as active-run enumeration plus cancel-all and bounded wait/drain. Keep it generic; do not add channel/plugin-specific policy.
