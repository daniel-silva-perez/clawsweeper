## Summary

Current main still does not implement the child-process shutdown cleanup this issue tracks. The gateway has bounded shutdown watchdogs and adjacent websocket/HTTP/MCP shutdown hardening, but stop/restart still does not enumerate, cancel, or bounded-drain active ProcessSupervisor-managed exec/CLI child processes. The discussion also explicitly separates this from the websocket shutdown fix in #61565.

## Best Possible Solution

Keep this open as the canonical tracker for gateway shutdown blocked by active child-process or pipe handles. The best implementation is a narrow core shutdown contract: expose active ProcessSupervisor run enumeration plus bounded cancel/drain APIs, wire that into gateway stop/restart before the force-exit watchdog can fire, add privacy-safe diagnostics for run id/backend/pid/scope without command text, and cover the behavior with focused supervisor and gateway shutdown tests. Treat #61565 as adjacent websocket hardening only.
