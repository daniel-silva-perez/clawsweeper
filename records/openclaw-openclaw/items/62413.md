## Summary

Keep open. Current main and the latest release still render Dream Diary body text through Lit text interpolation after `flattenDiaryBody`, so markdown markers like `**bold**` remain literal. The linked fix PR #62433 was closed unmerged and its approach needs updating for current backfill-flattening behavior.

## What This Changes

The issue requests rendering markdown formatting correctly in the Control UI Dream Diary tab instead of showing markdown source literally.

## Best Possible Solution

Render Dream Diary prose through the existing sanitized markdown pipeline, add focused Dreaming view tests for rendered bold/italic and sanitization behavior, and preserve the current structured-backfill cleanup so internal headings and source citations do not leak back into the UI.

## Reproduction Assessment

Yes. A high-confidence reproduction is to pass diary content containing `**bold**` or `*italic*` through the current `renderDreaming` diary-tab path; current source shows the body becomes escaped Lit text in `.dreams-diary__para` rather than sanitized markdown HTML.

## Solution Assessment

No, current main is not the best solution because it still treats diary body text as plain text. The best narrow fix is the existing sanitized markdown renderer, adapted to current `flattenDiaryBody` behavior and backed by a non-vacuous view test.
