## Summary

Keep open. The related PR/comment claimed the Dream Diary markdown bug was fixed, but that PR is closed and unmerged in the provided context, and current main still renders diary body text as escaped Lit text after flattening rather than through the existing sanitized markdown renderer.

## Best Possible Solution

Keep this issue open and implement the Dream Diary rendering path in `ui/src/ui/views/dreaming.ts`, ideally by reusing `toSanitizedMarkdownHtml()` for diary body markdown while preserving the newer backfill-flattening behavior where intended. Add a focused `ui/src/ui/views/dreaming.test.ts` regression case that feeds diary content with `**bold**`, `*italic*`, and a list, then verifies rendered markdown and sanitizer-safe output.
