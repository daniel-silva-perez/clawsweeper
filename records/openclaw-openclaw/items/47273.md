## Summary

Keep this issue open. Current main still gates `noteStartupOptimizationHints` on `platform === "linux"`, so a Darwin host is not an eligible startup optimization target. The regression tests also only cover Linux and Windows for this path. There is an open implementation PR, #47291, with closing syntax for this issue, so the right cleanup is to review and land or close that PR rather than closing the issue ahead of a merged fix.

## Best Possible Solution

Keep this issue open and review #47291 or an equivalent patch that adds Darwin to the startup optimization target logic with focused regression tests. If maintainers want actual agent-visible free/total memory data beyond doctor hints, split that into an explicit product/API design item rather than treating the one-line Darwin gate change as the full behavior.
