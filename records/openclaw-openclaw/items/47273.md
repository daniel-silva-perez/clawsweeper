## Summary

Keep open. Current main still excludes Darwin from the active doctor startup optimization helper, and the issue is paired with open PR #47291 using closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks to include macOS Darwin in the memory-backed startup optimization detection path so OpenClaw can use available OS memory values instead of skipping non-Linux hosts.

## Best Possible Solution

Review and land PR #47291, or replace it with an equivalent focused patch that includes Darwin ARM and low-memory hosts in `noteStartupOptimizationHints` and adds Darwin regression tests. Keep broader agent memory telemetry separate unless maintainers explicitly expand the scope.

## Reproduction Assessment

Yes. A direct unit-level call to `noteStartupOptimizationHints` with `platform: "darwin"`, `arch: "arm64"`, low `totalMemBytes`, and a `noteFn` would hit the Linux-only target check and return before calling `noteFn`.

## Solution Assessment

Yes for the proven bug surface. Updating the doctor helper predicate and colocated startup optimization tests is the narrow maintainable fix; changing prompt injection, `MEMORY.md` behavior, or general telemetry would exceed the evidence in this report.
