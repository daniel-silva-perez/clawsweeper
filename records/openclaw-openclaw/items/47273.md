## Summary

Keep open. Current main still excludes Darwin in the active doctor startup optimization helper, and the issue is paired with open PR #47291 as the implementation path, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests macOS/Darwin support in the startup memory/optimization detection path so host memory data is not skipped outside Linux.

## Best Possible Solution

Review and land #47291, or replace it with an equivalent focused patch that includes Darwin ARM/low-memory hosts in `noteStartupOptimizationHints` and adds Darwin regression tests. Keep broader agent memory telemetry separate unless maintainers explicitly expand the product scope.

## Reproduction Assessment

Yes. A direct unit-level call to `noteStartupOptimizationHints` with `platform: "darwin"`, `arch: "arm64"`, low `totalMemBytes`, and a `noteFn` would return at the Linux-only target check before calling `noteFn`.

## Solution Assessment

Yes for the proven bug surface. Updating the doctor helper predicate and colocated startup optimization tests is the narrow maintainable fix; changing prompt injection, MEMORY.md behavior, or general telemetry would exceed the evidence in this report.
