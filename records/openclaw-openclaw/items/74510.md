## Summary

Current main still leaves the reported phaseless commentary path open: phased commentary is filtered, but a separate assistant text message with no `phase`/`textSignature` and `stopReason="toolUse"` remains display-eligible and can be rendered by Control UI before the final answer. The linked #74928 is broader audience-boundary work, not a replacement for this narrower regression.

## What This Changes

Prevent Control UI from rendering intermediate assistant commentary/progress messages as normal same-level assistant replies when a single user turn also has a final answer.

## Best Possible Solution

Preserve commentary/final-answer phase metadata through provider conversion, transcript persistence, history projection, and UI extraction, plus a tightly scoped legacy fallback for the reported phaseless intermediate-commentary shape so the final answer is the primary visible reply.

## Reproduction Assessment

Yes. A source-level fixture with a user message, an assistant text-only entry with no phase/textSignature and `stopReason="toolUse"`, then a final assistant answer follows the current code path: phase is undefined, chat.history keeps it, and Control UI extracts it as visible text.

## Solution Assessment

Yes. Metadata preservation with a narrow legacy fallback is the maintainable fix; broadly hiding every assistant `toolUse` message would be too risky because some tool-use turns can include legitimate visible content.
