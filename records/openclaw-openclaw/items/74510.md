## Summary

Keep open. Current main still has the reported visibility gap for phaseless persisted assistant commentary: the shared extractor treats unphased assistant text as visible legacy text, chat.history only drops entries with explicit commentary phase metadata, and Control UI renders through that same shared extractor. The related #74928 report is broader audience-boundary work, not a replacement for this narrower Control UI regression.

## What This Changes

Prevent Control UI from rendering intermediate assistant commentary or progress entries as normal same-level assistant replies alongside the final answer for one user turn.

## Best Possible Solution

Preserve commentary and final_answer metadata through provider conversion, transcript persistence, history projection, and UI extraction, then add a constrained legacy fallback for the reported phaseless intermediate-commentary shape so the final answer is the primary visible reply without suppressing legitimate tool-use assistant content.

## Reproduction Assessment

Yes. A source-level fixture with a user message, an assistant text-only entry with no phase/textSignature and stopReason="toolUse", then a final assistant answer exercises the current path: phase resolution is undefined, chat.history keeps the intermediate entry, and Control UI extraction can render it.

## Solution Assessment

Yes. Metadata preservation plus a narrow legacy fallback is the best maintainable direction; hiding every assistant stopReason="toolUse" entry would be too broad because some tool-use turns may contain legitimate visible text or non-text content.
