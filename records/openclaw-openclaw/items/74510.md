## Summary

This should stay open: current main still handles phased commentary correctly, but the reported persisted shape with missing `textSignature` remains renderable as ordinary assistant text through both `chat.history` and the Control UI fallback path.

## What This Changes

The issue requests that Control UI stop showing intermediate assistant commentary as same-level assistant replies alongside the final answer for one user turn.

## Best Possible Solution

Preserve `commentary` and `final_answer` metadata through provider output, Pi transcript persistence, and `chat.history`, then add a narrow legacy fallback only for the reported unphased intermediate-commentary shape so the final answer remains the primary visible reply.
