## Summary

Keep open: current main suppresses commentary when explicit phase metadata is present, but the reported persisted shape with missing `textSignature` can still pass through `chat.history` and render as normal assistant text in Control UI.

## What This Changes

Prevent Control UI from showing intermediate assistant commentary/progress as same-level assistant replies beside the final answer for one user turn.

## Best Possible Solution

Preserve `commentary` and `final_answer` metadata through provider output, Pi transcript persistence, and `chat.history`, then add a narrow legacy fallback for unphased intermediate tool-use assistant text only when the final answer remains available as the primary visible reply.
