## Summary

Keep this open. Current main still has the reported failure mode for persisted assistant commentary that has no `phase` or phased `textSignature`: the shared extractor treats unphased assistant text as visible legacy text, `chat.history` only drops explicit commentary, and Control UI delegates to that shared extractor.

## What This Changes

The issue requests that Control UI avoid rendering intermediate assistant commentary as normal same-level replies alongside the final answer for one user turn.

## Best Possible Solution

Preserve `commentary` and `final_answer` metadata through provider conversion, transcript persistence, `chat.history`, and Control UI extraction, then add a narrow legacy fallback for the reported phaseless intermediate-commentary shape so the final answer remains the primary visible reply.

## Reproduction Assessment

Yes. A source-level fixture with an unphased assistant text entry plus `stopReason: "toolUse"` reproduces the path: phase resolution is undefined, history projection keeps the message, and Control UI extraction can render the text.

## Solution Assessment

Yes. The narrow maintainable fix is metadata preservation plus a constrained legacy fallback; hiding all tool-use assistant turns by stop reason alone would be too broad.
