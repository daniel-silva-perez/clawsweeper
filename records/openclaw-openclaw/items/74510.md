## Summary

This issue should stay open: current main has phase-aware rendering and history filtering, but the reported persisted shape explicitly lacks `textSignature`, and the gateway/UI fallback path still renders unphased assistant text as a normal assistant message. The existing coverage proves phased commentary is handled, not the unphased `stopReason=toolUse` commentary row described in the report.

## What This Changes

Control UI should avoid rendering intermediate assistant commentary/progress as normal same-level assistant replies when one user turn later produces a final answer.

## Best Possible Solution

Preserve `commentary` and `final_answer` metadata through provider output, Pi transcript persistence, and `chat.history`, with a narrowly tested gateway/UI fallback for legacy or unphased intermediate commentary rows so the final answer remains the primary reply.
