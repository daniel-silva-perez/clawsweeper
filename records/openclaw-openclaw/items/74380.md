## Summary

Keep open. Current main and the latest tagged release still contain the reported onboarding failure path, and open PR #74408 is already the active closing fix candidate, so this issue should not be closed before that PR or an equivalent fix lands.

## What This Changes

The issue reports that `openclaw onboard` fails while installing bundled skill dependencies for `model-usage` and `summarize` on macOS.

## Best Possible Solution

Review and land PR #74408, or an equivalent narrow fix, so bundled skill metadata can select a working macOS CodexBar cask path and the maintained npm installer for `summarize`, with focused coverage for command generation, metadata propagation, status labels, and onboarding behavior.

## Reproduction Assessment

Yes. On macOS with missing `codexbar` and `summarize`, current main will offer these missing dependencies through `setupSkills()` and then run formula-only Homebrew commands through `installSkill()`, matching the reported failures.

## Solution Assessment

Yes. The linked repair direction appears to be the narrowest maintainable fix: add structured cask support instead of ad hoc flags, switch `summarize` to the maintained node installer, and keep install-security metadata aligned.
