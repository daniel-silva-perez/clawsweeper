Keep open. Current main still has the reported onboarding failure path for the confirmed hard failures: `model-usage` still resolves to a plain Homebrew formula install for CodexBar, and `summarize` still points at a removed tap formula. ## What This Changes

The issue reports that `openclaw onboard` fails while installing bundled skill dependencies, especially `model-usage` and `summarize`, and asks for those installs to succeed.

## Best Possible Solution

Review and land PR #74408 if it keeps cask selection structured, updates summarize to the maintained npm package, audits the remaining selected skill installers, and adds focused installer/status/onboarding tests. Close this issue only after that fix is merged and verified on current main.

## Solution Assessment

Yes. A typed cask option plus corrected node installer metadata is the narrow maintainable direction; it is safer than embedding raw Homebrew flags in skill metadata and should be completed through the open fix PR.
