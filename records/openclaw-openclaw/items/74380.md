## Summary

Keep open. Current main still advertises invalid installers for the reported onboarding path: `model-usage` is macOS-scoped but points at a Linux-only Homebrew formula while the macOS delivery is a cask the installer cannot invoke, and `summarize` still points at a deleted tap formula despite upstream docs/npm pointing to `@steipete/summarize`. The onboarding flow still calls those selected installer recipes and surfaces failures, so the reported 2026.4.26 behavior remains valid.

## What This Changes

The issue asks for `openclaw onboard` skill dependency installation to avoid failing when installing bundled skills such as `model-usage` and `summarize` on macOS.

## Best Possible Solution

Update the bundled skill recipes and installer contract so onboarding selects working, validated install actions: support a safe Homebrew cask path for CodexBar or stop offering `model-usage` as installable on macOS until that path exists, replace `summarize`'s deleted brew formula with the current `@steipete/summarize` node installer, audit the other selected skills, and add focused tests around metadata parsing, installer command generation, status selection, and onboarding failure handling.
