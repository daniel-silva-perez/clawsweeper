## Summary

Keep open. Current main still has the formula-only Homebrew installer behavior and bundled skill manifests that match the reported `model-usage` and `summarize` failures, and PR #74408 is already the open unmerged closing fix candidate.

## What This Changes

The issue reports that `openclaw onboard` fails when installing bundled skill dependencies because `model-usage` invokes a Homebrew cask as a formula and `summarize` points to a removed Homebrew formula.

## Best Possible Solution

Land PR #74408 or an equivalent narrow fix that adds explicit Homebrew cask support for skill install specs, switches `summarize` to its maintained npm package, and keeps installer/status/security metadata plus focused tests aligned.

## Reproduction Assessment

Yes. On macOS with the reported missing dependencies, current main's onboarding flow still routes `model-usage` and `summarize` through the same bundled install specs and command builder that produce the reported Homebrew failures.

## Solution Assessment

Yes. The linked repair direction is the narrowest maintainable solution because it fixes the generic skill installer contract for casks and corrects stale bundled metadata instead of adding one-off onboarding exceptions.
