## What This Changes

The issue reports that `openclaw mcp serve` fails strict stdio MCP clients such as Hermes Agent when they send a correlated `tools/list` request.

## Best Possible Solution

Keep this issue open until PR #68800 is reviewed and either landed or replaced by an equivalent narrow fix. The desired end state is for `openclaw mcp serve` to route all non-protocol logging away from stdout before any config/log output can occur, with a regression test that a strict client receives a valid `tools/list` JSON-RPC response with the matching id.

## Reproduction Assessment

Yes for the user-visible failure path: the issue gives stdio MCP steps, and PR #68800 provides a raw JSON-RPC before/after diagnosis for non-JSON stdout pollution. ## Solution Assessment

Unclear until the linked PR's stdout-pollution diagnosis is verified against a strict client trace. If that diagnosis is correct, routing logs to stderr before config loading is the narrowest maintainable fix because it matches the existing stdio MCP server pattern.
