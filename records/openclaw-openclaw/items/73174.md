## Summary

Keep open. The PR addresses a real GitHub-rendered-source annoyance, but current main has not implemented it and the proposed Markdown target conflicts with the scoped docs authoring rule for root-relative Mintlify links.

## What This Changes

The PR changes the `installer script` link in `docs/install/node.md` from `/install#alternative-install-methods` to `index.md#alternative-install-methods`.

## Best Possible Solution

The desired end state is a maintainer-approved docs link convention for this case: either keep the Mintlify root-relative route, or document and test a GitHub-source readability exception before using a `.md` file anchor in `docs/install/node.md`.

## Reproduction Assessment

Yes. Current main contains the leading-slash link in `docs/install/node.md`, and the target heading exists in `docs/install/index.md`; the reported GitHub-source behavior follows from that source page resolving `/install#alternative-install-methods` outside the docs host.

## Solution Assessment

No. The patch likely fixes GitHub source browsing, but as written it is not the narrowest maintainable repository fix because it conflicts with the scoped docs link rule without updating the policy or guardrails.
