## Summary

Keep this PR open, but do not merge it as-is. It fixes a real GitHub-rendered-source annoyance, and Greptile reached the same functional conclusion, but the proposed `index.md#...` target conflicts with the scoped docs rule that internal docs links stay root-relative and omit `.md`/`.mdx` suffixes.

## What This Changes

This PR changes the `installer script` link in `docs/install/node.md` from `/install#alternative-install-methods` to `index.md#alternative-install-methods`.

## Best Possible Solution

The desired end state is a maintainer-approved docs link convention: either preserve the root-relative published-docs form, or explicitly document and apply a GitHub-source readability exception before changing this page to a file-relative or absolute docs URL.

## Reproduction Assessment

Yes. Current main contains `/install#alternative-install-methods` in `docs/install/node.md`, and the destination heading exists in `docs/install/index.md`; the GitHub-rendered-source complaint follows from that leading-slash link being interpreted outside the docs host.

## Solution Assessment

No. The proposed `index.md#alternative-install-methods` likely fixes GitHub source browsing, but it is not clearly the best repository solution because it conflicts with `docs/AGENTS.md`; a policy-consistent alternative or documented exception should come first.
