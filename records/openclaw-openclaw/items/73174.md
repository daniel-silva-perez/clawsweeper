## Summary

Keep this PR open for maintainer review. Current main still has the reported GitHub-source navigation problem, but the proposed `index.md#alternative-install-methods` link conflicts with the scoped docs rule that internal docs links in `docs/**/*.md` stay root-relative and omit `.md` suffixes, so this is a docs-policy decision rather than a conservative cleanup close.

## What This Changes

This PR changes the `installer script` link in `docs/install/node.md` from the Mintlify root-relative `/install#alternative-install-methods` target to the local Markdown target `index.md#alternative-install-methods`.

## Best Possible Solution

The desired end state is a maintainer-approved docs link convention: either preserve the published-docs root-relative link, or explicitly document and apply a GitHub-source readability exception before changing `docs/install/node.md` to a relative Markdown target.

## Reproduction Assessment

Yes. Current main still contains `/install#alternative-install-methods` in `docs/install/node.md`, and the destination heading exists in `docs/install/index.md`; the GitHub-rendered-source complaint follows from that leading-slash link being interpreted outside the Mintlify docs host.

## Solution Assessment

No. The proposed link likely fixes GitHub source browsing, but it is not clearly the best repository solution because it conflicts with `docs/AGENTS.md`; a policy-consistent alternative or documented exception should come first.
