## Summary

Keep open. Current main still leaves the central bootstrap relevance request unresolved: oversized bootstrap files are trimmed by deterministic prefix and suffix slices, while the related compaction and observation-extraction parts are only partially covered by existing plugin surfaces and need maintainer scoping.

## What This Changes

The issue requests relevance-aware bootstrap file selection, less fragile incremental compaction, and observation extraction from existing LLM responses.

## Best Possible Solution

Split the request into narrower tracks: design relevance- or structure-aware bootstrap selection around buildBootstrapContextFiles and embedded bootstrap injection; keep compaction work aligned with the existing compaction-provider/context-engine contracts; and leave observation extraction plugin-owned unless maintainers identify a specific missing SDK hook.
