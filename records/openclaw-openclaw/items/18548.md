## Summary

Keep open. Current main has improved the bootstrap truncation ratio math and has plugin surfaces for compaction and response observation, but the central bootstrap relevance request is still unresolved: oversized bootstrap files are still reduced to deterministic prefix/suffix slices with the middle dropped positionally. The provided maintainer discussion already reached the same conclusion, and the issue bundles multiple architecture topics that need maintainer scoping before implementation.

## What This Changes

The issue requests relevance-aware bootstrap file selection, more incremental compaction behavior, and a plugin-oriented way to extract observations from existing LLM responses.

## Best Possible Solution

Split the work into narrower trackers: design and implement relevance- or structure-aware selection around bootstrap context building, keep compaction changes aligned with the existing compaction-provider/context-engine contracts, and treat observation extraction as plugin-owned unless a concrete missing SDK hook is identified.
