## Summary

Keep open. Current main still trims oversized bootstrap files by deterministic prefix and suffix slices, so the central bootstrap relevance request is unresolved; the compaction and observation-extraction parts have adjacent plugin surfaces but still need maintainer scoping.

## What This Changes

The issue requests relevance-aware bootstrap file selection, less fragile incremental compaction, and observation extraction from existing LLM responses.

## Best Possible Solution

Split this into narrower tracks: implement relevance- or structure-aware bootstrap selection around buildBootstrapContextFiles and embedded bootstrap injection; keep incremental compaction aligned with the compaction-provider and context-engine contracts; and leave observation extraction plugin-owned unless a concrete missing SDK hook is identified.
