## Summary

Keep open. Current main still truncates oversized bootstrap files by fixed prefix/suffix ratios and the embedded run path still injects that output; the compaction and observation pieces have plugin/context-engine paths and related open trackers, but they do not resolve the core bootstrap relevance request.

## Best Possible Solution

Keep this issue open, or split it into narrower trackers. The unresolved core piece belongs around `buildBootstrapContextFiles`, `resolveBootstrapContextForRun`, and the embedded bootstrap injection path. Incremental compaction should continue through the context-engine/compaction work and related open reports, while observation extraction should remain plugin/hook-driven unless a concrete missing SDK API is identified.
