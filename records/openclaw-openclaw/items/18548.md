## Summary

Keep open. Current main has improved the bootstrap ratio math since the report, and current compaction/context-engine plugin surfaces cover part of the discussion, but the central bootstrap relevance request is still unresolved because large bootstrap files are still trimmed by fixed prefix/suffix selection with the middle dropped positionally. The issue also bundles separate compaction and observation-extraction architecture questions, so it needs maintainer splitting or prioritization before any automated repair lane is appropriate.

## What This Changes

The issue requests relevance-aware bootstrap file selection, incremental compaction behavior, and plugin-driven observation extraction from existing model responses.

## Best Possible Solution

Split or scope the work into narrower trackers: implement a relevance- or structure-aware bootstrap selection path around buildBootstrapContextFiles and embedded bootstrap injection, keep incremental compaction work aligned with the existing compaction provider and context-engine surfaces, and leave observation extraction plugin-driven unless maintainers identify a concrete missing SDK/API seam.
