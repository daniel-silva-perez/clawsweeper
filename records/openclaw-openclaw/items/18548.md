## Summary

Keep open. Current main has fixed the original bootstrap ratio arithmetic and has added compaction/context-engine/plugin hook surfaces for parts of the discussion, but bootstrap file injection still selects fixed prefix and suffix slices and drops the middle positionally, so the central relevance-aware bootstrap request is unresolved. The existing maintainer and ClawSweeper comments also identify the same remaining gap, and the bundled compaction plus observation-extraction topics need maintainer scoping rather than automatic closure.

## What This Changes

Requests three architecture improvements: relevance-aware bootstrap file selection, incremental or non-bulk compaction, and lower-overhead observation extraction from existing LLM output.

## Best Possible Solution

Split the work into narrower trackers: implement a relevance- or structure-aware bootstrap selection path around buildBootstrapContextFiles and embedded bootstrap injection, keep incremental compaction aligned with the existing context-engine and compaction-provider contracts, and leave observation extraction plugin-driven unless maintainers identify a concrete missing SDK/API seam.
