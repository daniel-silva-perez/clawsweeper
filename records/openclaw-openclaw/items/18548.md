## Summary

Keep open. Current main fixes the original bootstrap ratio arithmetic and has plugin surfaces for compaction and response observation, but the central bootstrap relevance request is still unresolved: large bootstrap files are still trimmed by fixed prefix/suffix slices with the middle dropped positionally. The issue also bundles separate architecture topics, and existing maintainer review already points to splitting or prioritizing the work rather than closing it.

## What This Changes

The issue requests relevance-aware bootstrap file injection, more incremental/robust compaction, and observation extraction from existing model responses.

## Best Possible Solution

Split this into narrower maintainer-owned trackers: implement relevance- or structure-aware bootstrap selection around buildBootstrapContextFiles and embedded bootstrap injection, keep compaction work aligned with the existing compaction-provider/context-engine contracts and community plugin path, and treat observation extraction as plugin-driven unless a concrete missing SDK hook is identified.
