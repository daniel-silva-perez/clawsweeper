## Summary

Keep open. Current main still truncates oversized bootstrap files by fixed head/tail position and the live embedded run path still injects that output; context-engine and hook surfaces make the compaction/observation parts plausible as plugin or follow-up work, but they do not close the core bootstrap relevance request.

## Best Possible Solution

Keep this issue open, or split it into narrower trackers. Bootstrap relevance belongs in core around `buildBootstrapContextFiles` and the embedded bootstrap injection path. Incremental compaction should follow the context-engine/compaction work and related reports such as #58398, #58455, and #67716. Observation extraction should remain plugin/hook-driven unless a concrete missing SDK API is identified.
