## Summary

Keep open. Current main has improved bootstrap budget math and provides plugin surfaces for compaction and observation work, but the central bootstrap relevance request remains unresolved: large bootstrap files are still reduced by fixed prefix/suffix selection, with the middle dropped positionally. The issue also bundles three separate architecture topics, so it needs maintainer splitting or prioritization before implementation.

## What This Changes

The issue requests relevance-aware bootstrap file injection, incremental compaction improvements, and lower-overhead observation extraction from existing LLM responses.

## Best Possible Solution

Split the work into narrower trackers: implement relevance- or structure-aware bootstrap selection around buildBootstrapContextFiles and embedded bootstrap injection, align any incremental compaction changes with the context-engine and compaction-provider surfaces, and keep observation extraction plugin-driven unless a concrete missing SDK/API seam is identified.
