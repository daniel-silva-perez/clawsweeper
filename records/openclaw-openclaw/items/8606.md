## Summary

Current main still lacks a context-pruning lifecycle hook. The pruning extension calls the pruner and returns rewritten messages only, the pruner returns `AgentMessage[]` without metadata, hook catalogs/docs still omit `session:prune` or `context:prune`, and the direct implementation candidate #12581 remains open and unmerged. Because an open PR explicitly targets this issue, the issue should stay open until that PR or a maintainer-approved replacement lands.

## Best Possible Solution

Keep this issue open and treat #12581, or a maintainer-approved replacement, as the implementation path. The fix should live in `src/agents/pi-hooks/context-pruning/*`, collect safe pruning metadata from the pruner, emit a `session:prune` or equivalent lifecycle event only after successful pruning, add focused tests, and document the event in `docs/automation/hooks.md` plus plugin docs if it becomes a plugin hook.
