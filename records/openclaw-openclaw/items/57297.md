## Summary

Keep this issue open. Current main has partial WebChat TTS/audio handling, but the requested structured generation/routing/delivery telemetry, tool-to-delivery correlation id, and TTS-specific user-visible delivery/playback status are still missing. The linked same-author WebChat playback bug #57296 is also still open, so this companion observability request should not be closed independently.

## Best Possible Solution

Keep this issue open. Implement structured WebChat TTS events for generation start/success, route selection, and delivery success/failure with actionable reason codes and a run/tool correlation id; persist or broadcast enough state for the Control UI to show final TTS delivery/playback status; add focused tests around the TTS tool result path, `src/gateway/server-methods/chat.ts`, `src/gateway/server-methods/chat-webchat-media.ts`, and `ui/src/ui/chat/grouped-render.ts`; coordinate with #57296 and #65859.
