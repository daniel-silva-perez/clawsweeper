## Summary

Keep this PR open. Current main still does not implement the PR’s core behavior: default-model config changes do not archive old transcripts, rotate affected session IDs, create one-shot sanitized handoff files, or inject handoff context on the first post-switch turn. The related target-repo PR #41907 is closed unmerged and only covered the narrower stale model-field reset, so it does not supersede this larger handoff/archive work.

## Best Possible Solution

Keep this PR open for maintainer review or a focused replacement. The best path is a core session/gateway change that detects default-model config changes, preserves explicit user overrides, clears stale runtime model/context metadata where needed, archives or rotates affected transcripts, stores handoff data with explicit privacy and retention semantics, and delivers sanitized one-shot user context only when an agent run will proceed. Add targeted coverage for hot reload, restart-based reload, missing transcript files, multi-session retention, explicit overrides, and first-turn handoff delivery.
