## Summary

Keep this PR open. Current main still lacks the model-change transcript archive, session-id rotation, and one-shot sanitized user-message handoff that the PR is trying to add. The narrower related PR #41907 is closed unmerged, and the current checkout does not show a successor implementation.

## Best Possible Solution

Keep this PR open for maintainer review or replacement. The best path is a focused core session/gateway implementation that detects default-model config changes, preserves explicit user model overrides, clears stale runtime model/context metadata, rotates or archives affected transcripts, stores any handoff data with the same privacy expectations as session state, and delivers sanitized one-shot user context only when an agent run will actually proceed. Add tests for hot reload, restart reload, multi-session retention, missing transcript files, and first-turn delivery.
