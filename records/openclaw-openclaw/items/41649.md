## Summary

Keep this PR open. Current main still does not implement the PR’s core model-change handoff behavior: no model-change session reset action, no transcript archive/rotation on default-model changes, no one-shot sanitized handoff file, and no prompt consumption path. The related target-repo PR #41907 is closed unmerged and only covered the narrower stale model-field reset, so it does not supersede this larger PR.

## Best Possible Solution

Keep this PR open for maintainer review or a focused replacement. The best implementation should detect default-model config changes across hot reload and restart paths, preserve explicit user overrides, clear stale runtime model/context metadata where appropriate, archive or rotate affected transcripts, store sanitized handoff data with explicit privacy and retention semantics, and deliver it only when a model run will actually proceed. Add targeted coverage for hot reload, restart-based reload, missing transcript files, multi-session retention, explicit overrides, and first-turn handoff delivery.
