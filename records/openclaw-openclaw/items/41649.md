## Summary

Keep this PR open. Current main still lacks the model-change transcript archive, session-id rotation, and one-shot sanitized user-message handoff that this PR is trying to add. The related narrower PR #41907 is closed unmerged, and no successor implementation is present on main.

## Best Possible Solution

Keep this PR open for maintainer review or replacement. The best path is a focused core session/gateway change that detects default-model config changes, preserves explicit user overrides, clears stale runtime model/context metadata, archives or rotates affected transcripts, stores any handoff data with clear privacy/retention semantics, and delivers sanitized one-shot user context only when an agent run will actually proceed. Add coverage for hot reload, restart reload, missing transcript files, multi-session retention, explicit overrides, and first-turn handoff delivery.
