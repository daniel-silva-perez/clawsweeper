## Summary

Keep this PR open. Current main still returns a hard `timeout` from `sessions_send` when the initial `agent.wait` times out, so the intended behavior is not implemented. The PR is a focused implementation attempt, but it has unresolved correctness concerns around terminal timeout misclassification and stale assistant-history delivery. Related #68196 is open and unmerged, so it is not yet a canonical replacement.

## Best Possible Solution

Keep the PR open for maintainer review or author revision. The best fix belongs in the `sessions_send`/`run-wait` surface: distinguish soft wait-window expiry from terminal aborted timeout, preserve or pass the pre-run assistant baseline through delayed delivery, and add regression coverage for delayed reply delivery, second-wait timeout/no-delivery, stale-history prevention, and terminal aborted timeout behavior. If #68196 becomes the accepted canonical fix after its own review concerns are resolved, then this PR can be closed as superseded with that link.
