## Summary

Keep this open. Current main still has the reported LiteLLM provider-docs gap: the manual setup example remains an unpinned install and there is no advisory/remediation text for the compromised LiteLLM versions. The runtime evidence still supports the issue's narrower framing that OpenClaw talks to a user-hosted LiteLLM proxy rather than bundling the PyPI package, so this is a docs/security guidance follow-up rather than a runtime dependency fix.

## What This Changes

Add LiteLLM supply-chain incident guidance to the provider docs, including safer install wording and remediation steps for users who ran affected proxy versions.

## Best Possible Solution

Land a maintainer-reviewed docs-only update to `docs/providers/litellm.md` that replaces or qualifies the unpinned install example and adds a dated LiteLLM incident notice with version-check, affected-version, credential-rotation, reinstall/downgrade/upgrade, and host-audit guidance. The OpenClaw runtime/plugin code can remain unchanged unless that review identifies a separate dependency or proxy-safety issue.
