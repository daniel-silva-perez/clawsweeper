## Summary

Keep this issue open. Current main still has the reported docs gap: the LiteLLM provider page uses an unpinned proxy install command and I found no advisory or remediation text for the reported v1.82.7/v1.82.8 compromise. The runtime integration is network/proxy-based and does not bundle the PyPI package, so the remaining work is docs-only.

## Best Possible Solution

Keep this issue open and land a docs-only update in `docs/providers/litellm.md`: pin or qualify the LiteLLM proxy install guidance, add a dated advisory for the reported v1.82.7/v1.82.8 compromise, and include version-check, downgrade or reinstall, key-rotation, and server-audit steps. No OpenClaw runtime dependency change appears needed because the bundled integration talks to a user-hosted LiteLLM proxy over HTTP.
