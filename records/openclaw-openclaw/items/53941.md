## Summary

Keep this open. Current `origin/main` still has the reported LiteLLM docs gap: the provider page uses an unpinned `pip install 'litellm[proxy]'` command and no advisory/remediation text for the reported compromised versions. The runtime/plugin side supports the issue's claim that OpenClaw talks to a user-hosted LiteLLM proxy rather than bundling the PyPI package, so the remaining work is a docs/security-guidance update.

## What This Changes

The issue requests a docs-only LiteLLM provider advisory that pins or qualifies the proxy install command and tells self-hosted proxy users how to respond to the reported LiteLLM v1.82.7/v1.82.8 compromise.

## Best Possible Solution

Keep this issue open and land a docs-only update in `docs/providers/litellm.md` after upstream verification: replace or qualify the unpinned proxy install example, add a dated security notice for the affected LiteLLM versions, and include version-check, downgrade/reinstall, credential-rotation, and host-audit guidance. No OpenClaw runtime dependency change appears needed.
