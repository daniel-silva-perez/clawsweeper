## Summary

Keep open. Current main still treats Signal `groupAllowFrom` through sender matching and mention gating still falls back to `requireMention: true` when no explicit group/default value is configured. Open PR #73538 is linked with closing syntax but unmerged, so this issue should wait for maintainer review, merge, revision, or replacement rather than being closed.

## What This Changes

The issue requests Signal group admission by inbound group id instead of only sender identity, plus a default of no mention requirement for explicitly configured groups.

## Best Possible Solution

Keep this issue open until the Signal-local group-id matching behavior is either merged or intentionally rejected, and handle the `requireMention` default as an explicit group-policy decision with tests and docs updated to match.
