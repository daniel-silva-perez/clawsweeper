## Summary

Keep open. Current main still evaluates Signal `groupAllowFrom` through sender matching before a Signal group id can admit the inbound event, and the mention-default behavior still falls back to `requireMention: true` unless config or an override says otherwise. Open PR #73538 is already paired with this issue via closing syntax, so this should stay open until that PR is reviewed, merged, revised, or replaced.

The issue requests fixing Signal group admission so allowlisted group messages are not silently dropped by sender/group-id mismatch or mention-default behavior.
