## Summary

Keep this issue open. Current main still accepts arbitrary model params, but the OpenAI/OpenAI-compatible stream paths only forward first-class temperature/max token/topP-style options and do not directly normalize or forward frequency_penalty/presence_penalty from config. The closed PRs mentioned in the discussion were not merged, and the broader related issues do not supersede this narrow request.

## Best Possible Solution

Keep this issue open as the canonical narrow request for OpenAI/OpenAI-compatible sampling-penalty passthrough. Implement a focused transport change that normalizes finite numeric frequency_penalty/frequencyPenalty and presence_penalty/presencePenalty from the already-merged params sources, forwards them on supported OpenAI-family payloads, adds targeted regression tests, and documents the supported scope without folding in the broader per-agent generation-parameter programs in #47852 or #71350.
