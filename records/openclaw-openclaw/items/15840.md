## Summary

Keep this issue open. Current main still synthesizes visible media placeholders for blank media-only Signal and iMessage sends, audio MIME types still resolve to the `audio` placeholder kind, and the strict `messages` config/schema/docs surface still has no suppression option. The related implementation PRs #15853, #15914, and #15935 are closed unmerged, while the existing maintainer-side comment treats this as the canonical cross-channel request.

## Best Possible Solution

Keep this issue open as the canonical product/config request. A conservative fix should add a documented default-off `messages` option, update `MessagesConfig`, the strict zod schema, generated config metadata/help/docs, audit outbound media-only channel paths, and wire the setting through Signal and iMessage placeholder construction while preserving existing fallback behavior by default and adding focused tests for audio/media-only sends.
