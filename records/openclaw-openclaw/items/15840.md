## Summary

Keep this issue open. Current main still synthesizes visible `<media:audio>` / `<media:${kind}>` placeholder text for blank media-only Signal and iMessage sends, and the strict `messages` config surface still has no suppression option. The issue also has recent maintainer-side review context treating it as the canonical cross-channel request, while related implementation PRs were closed unmerged.

## Best Possible Solution

Keep this issue open as the canonical product/config request. A conservative implementation should add a documented default-off `messages` option, update `MessagesConfig`, the strict zod schema, generated config metadata/help/docs, audit outbound media-only channel paths, and route the setting through the bundled Signal and iMessage placeholder paths while preserving the existing fallback by default and adding focused tests for audio/media-only sends.
