## Summary

Keep this issue open. Current main still synthesizes visible media placeholder text for blank media-only Signal and iMessage sends, audio MIME still maps to the `audio` kind, and the strict `messages` config/schema/docs surface has no suppression option. The related implementation PRs in the provided context were closed unmerged, while the maintainer-side review comment treats this as the canonical cross-channel request.

## Best Possible Solution

Keep this issue open as the canonical product/config request. A conservative fix should add a documented default-off `messages` option, update `MessagesConfig`, the strict zod schema, generated config metadata/help/docs, audit outbound media-only channel paths, and wire the setting through Signal and iMessage placeholder construction while preserving existing fallback behavior by default and adding focused tests for audio/media-only sends.
