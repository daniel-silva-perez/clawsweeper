## Summary

Keep open. Current main still routes bound ACP follow-ups directly to the ACP harness: bound conversation dispatch selects the ACP session, uses the incoming `BodyForAgent`/command text as the ACP prompt, and tests assert `runTurn` receives the raw follow-up text. The docs also describe direct bound follow-up routing. There is an open same-author PR, #69675, covering docs/policy clarification, so the best next step is maintainer review rather than cleanup closure or a separate automated fix PR.

## Best Possible Solution

Keep this issue open and review the paired open PR #69675. The best product path is to decide whether parent-orchestrated rewrite is intended for parent-owned thread-bound ACP follow-ups, then align runtime behavior, accepted-note wording, docs, and tests around that boundary without breaking explicit interactive ACP binding semantics or verbatim relay requests.
