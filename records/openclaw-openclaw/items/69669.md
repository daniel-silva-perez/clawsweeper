## Summary

Keep open. Current main still routes bound ACP follow-ups directly into the ACP session prompt path: bound conversation dispatch retargets the ACP session, ACP prompt resolution reads the inbound message fields, and regression coverage asserts the raw follow-up text reaches `runtime.runTurn`. The docs also describe direct bound follow-up routing. The same-author PR #69675 is still open in the provided context and is docs/policy-only, so it is not a reason to close this issue before maintainer review and a runtime decision.

## Best Possible Solution

Keep this issue open, review the paired open PR #69675 first, and make an explicit maintainer decision on ACP semantics. If parent-orchestrated follow-ups are the intended default, implement a focused runtime change that distinguishes parent-owned shorthand continuation from explicit interactive/verbatim ACP relay, then align `sessions.send`, bound dispatch, accepted-note wording, docs, and regression tests.
