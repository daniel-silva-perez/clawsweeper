## Summary

Keep open. Current main still passes `logging.file` through to the file logger without home-directory expansion before `fs.mkdirSync`, and the latest release tag has the same behavior. There is also an open PR, #73614, that explicitly targets this issue, so the right cleanup path is to review and land or close that PR rather than closing the issue early.

## Best Possible Solution

Keep this issue open while maintainers review #73614 or an equivalent fix. The logger should expand a leading `~` before rolling-path detection and directory creation, with regression coverage for stable and rolling `logging.file` values. After the logging crash-loop fix lands, maintainers should decide whether the broader config-derived path audit belongs in the same PR or a narrower follow-up.
