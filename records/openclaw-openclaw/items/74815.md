## Summary

Keep open. Current main still lacks the requested SDK-facing task ledger RPCs: the SDK task helpers still throw unsupported errors, Gateway does not register `tasks.*` handlers, and docs/tests still describe the task namespace as unsupported. The PR is an active draft implementation for linked member-authored #74707, with a parallel open implementation in #74847, so the right next step is maintainer API/security/native-client review and reconciliation rather than cleanup closure.

## What This Changes

The PR adds Gateway `tasks.list`, `tasks.get`, and `tasks.cancel` RPCs, SDK task helpers/types, protocol schemas/generated Swift models, docs, changelog, scopes, and tests for SDK-facing task ledger access.

## Best Possible Solution

Land one polished version of the SDK-facing task ledger RPC contract for #74707, reconciling this draft with #74847 and keeping Gateway schemas, method scopes, SDK types, generated native models, docs, and tests aligned.

## Reproduction Assessment

Yes. Current main reproduces the gap through the SDK unsupported errors and matching docs/tests, while the PR's added tests provide a focused reproduction path for dispatch, validation, filtering, get, cancel, and scope classification.

## Solution Assessment

Unclear. Additive Gateway RPCs backed by the existing task registry/cancel executor are the right architectural direction, but this draft needs maintainer review of the public metadata surface, the Swift generated-model shape, and the duplicate #74847 implementation before it is the best merge candidate.
