## Summary

Keep this PR open. Current main still stages inbound attachments into `media/inbound/*`, but the read-tool path returns normalized/sanitized read results without classifying those paths or wrapping text blocks as untrusted data. The PR remains a plausible implementation candidate for the linked open security issue, with prior bot-review findings addressed in its follow-up commits but still needing maintainer security review before merge.

## Best Possible Solution

Keep this PR open for maintainer review or replacement by a maintained fix. The fix should live in the read-tool path, detect sandbox-staged inbound media, wrap only text blocks with the existing untrusted-data wrapper, sanitize path labels, preserve image blocks, pass any needed container workdir/root context through sandboxed reads, and include focused regression tests. The linked security issue should remain open until such a fix lands.
