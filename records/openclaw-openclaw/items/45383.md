## Summary

Keep this PR open. Current main still does not implement the PR's inbound-media read guard: staged `media/inbound/*` files are a live agent read surface, but `createOpenClawReadTool` currently normalizes/sanitizes image blocks and returns the result without classifying inbound media paths or wrapping text blocks as untrusted prompt data.

## Best Possible Solution

Keep this PR open for maintainer review or replacement by a maintained fix. The fix should live in the read-tool path, detect sandbox-staged inbound media, wrap only text blocks with the existing untrusted-data wrapper, sanitize path labels, preserve image blocks, pass any needed container workdir/root context through sandboxed reads, and include focused regression tests. The linked security issue should remain open until such a fix lands.
