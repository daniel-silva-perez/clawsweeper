## Summary

Keep this PR open. Current main still stages inbound attachments into `media/inbound/*`, but the read tool returns normalized/sanitized read results without detecting inbound media paths or wrapping text blocks as untrusted data. The PR remains a plausible implementation candidate for the linked open security report, with prior review findings addressed in discussion and no obvious supply-chain-sensitive file changes in the provided diff.

## Best Possible Solution

Keep this PR open for maintainer review or replacement by a maintained fix. The fix should live in the read-tool path, detect sandbox-staged inbound media, wrap only text blocks with the existing untrusted-data wrapper, sanitize path labels, preserve image blocks, pass any needed container workdir/root context through sandboxed reads, and include focused regression tests. The linked security issue should stay open until such a fix lands.
