## Summary

Keep this open. Current main still lacks a user-facing `voice.autoJoin[].sessionChannelId` config field, still routes Discord voice sessions from the joined voice channel, and still suppresses Discord text delivery for voice transcript turns; an open implementation PR also references this issue with closing syntax, so the issue should remain open until that work lands or is superseded.

## What This Changes

Add optional `sessionChannelId` support to Discord voice auto-join entries so voice transcripts and agent replies can route to a paired text channel while audio playback continues in the voice channel.

## Best Possible Solution

Add a typed, documented Discord plugin config field for the companion text channel, validate it against the guild/channel contract, route voice ingress through that session when configured, enable Discord delivery only for that paired route, and cover the behavior with focused config/schema/manager tests while reconciling the scope with #73699.
