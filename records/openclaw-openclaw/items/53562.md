## Summary

Keep this open. Current main still lacks the requested `autoJoin` companion text-channel field and still processes Discord voice turns through the joined voice channel route with text delivery disabled; because an open implementation PR already references this issue with closing syntax, maintainers should review, land, or explicitly supersede that path rather than close the issue now.

## What This Changes

Add optional `channels.discord.voice.autoJoin[].sessionChannelId` support so Discord voice transcripts and replies can route to a companion text channel while audio playback stays in the joined voice channel.

## Best Possible Solution

Implement the feature in the Discord plugin/config surface: add a typed and schema-validated `autoJoin[].sessionChannelId`, propagate it through auto-join and voice session creation, route paired sessions to the configured text channel, enable Discord delivery only for that paired route, preserve voice playback, update docs/config metadata, and cover the behavior with focused config plus voice manager/segment tests while accounting for #73699 and #47095.
