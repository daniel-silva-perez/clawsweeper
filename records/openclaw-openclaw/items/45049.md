## Summary

Keep open. Current main has diagnostics, tests, docs, and a merged proxy `tool_choice: "auto"` compatibility fix, but it still does not implement the requested strict protocol-boundary behavior: no generic retry, forced tool choice, deterministic routing, or block happens when assistant text merely looks like tool use. The discussion also frames this as a security-sensitive executable-tool boundary, so the next step is maintainer design rather than automatic cleanup.

## What This Changes

The issue requests stricter agent-loop handling for models that emit simulated tool-use text instead of structured tool calls, including detection, optional forced `tool_choice` retry, and deterministic routing for obvious fetch/search tasks.

## Best Possible Solution

Define a maintainer-approved protocol-boundary policy: structured tool calls remain the generic executable path, simulated tool-call text is detected and surfaced, forced `tool_choice` retries are opt-in and provider/model scoped, and any provider-local text-promotion exception is hook-preserving, documented, and covered by focused tests.
