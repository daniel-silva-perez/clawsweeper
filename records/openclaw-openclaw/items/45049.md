## Summary

Keep open. Current main has diagnostics for assistant text that looks like a tool call and has local-model `tool_choice` workarounds, but it still lacks a maintainer-approved strict/retry/routing policy for requiring a real protocol-level tool invocation when a model emits simulated tool-use prose. The related comments also raise security-boundary concerns that need explicit maintainer design before an automated fix should be attempted.

## What This Changes

The issue requests stricter agent-loop handling when a model emits text that looks like tool use instead of a real protocol-level tool call.

## Best Possible Solution

Define a maintainer-approved protocol-boundary policy: only structured tool calls may execute, text-shaped tool calls should be detected and surfaced, and any retry with `tool_choice: "required"` should be opt-in, provider/model scoped, hook-preserving, and covered by focused tests and local-model docs.
