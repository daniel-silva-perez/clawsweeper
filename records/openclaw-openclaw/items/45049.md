## Summary

Keep open. Current main has diagnostics and local-model compatibility workarounds for text-shaped tool calls, but it still does not define or enforce the requested protocol-boundary policy for strict retry/routing when a model emits simulated tool-use prose. The discussion also raises security-boundary concerns around hook-preserving execution that need maintainer design before this should become an automated fix.

## What This Changes

The issue asks OpenClaw to stop accepting simulated text-form tool calls as normal agent output and add stricter enforcement or retry behavior for real protocol-level tool invocation, especially with LiteLLM/vLLM local models.

## Best Possible Solution

Define a maintainer-approved protocol-boundary policy: structured tool calls are the only generic executable path, text-shaped tool calls are detected and surfaced, forced `tool_choice` retries are opt-in and provider/model scoped, and any compatibility exceptions remain hook-preserving and covered by focused tests plus local-model docs.
