## Summary

Current main still has the reported compaction overflow path: it passes `params.reserveTokens` directly into the pinned pi-coding-agent summarizer, whose source computes API `maxTokens` as `Math.floor(0.8 * reserveTokens)`. The PR is relevant and includes focused tests, but it is dirty against current main and its patch predates the current headers-aware compaction call shape, so it needs a refresh before landing.

## Best Possible Solution

Refresh this PR or create an equivalent narrow replacement: clamp the reserve tokens passed to built-in compaction summarization using the resolved model output limit before pi-coding-agent computes `Math.floor(0.8 * reserveTokens)`, preserve current `params.headers` threading, keep focused clamping coverage, and keep a user-facing changelog entry crediting @adzendo.
