## Summary

Keep open. Current main has partial Chrome MCP hardening, but the existing-session `/tabs` path can still exceed the route/client budget and failed sticky sessions are not immediately process-tree-cleaned. The Windows Defender discussion is useful related context, but it does not resolve the original macOS/Node 25 npx-spawn report.

## Best Possible Solution

Keep this issue open and implement the fix in the browser plugin: propagate timeout and abort signals through Chrome MCP reachability, tab listing, and tab-management calls; close failed or timed-out pending and sticky sessions immediately; use a plugin-safe process-tree-aware transport or browser-plugin-local equivalent for Chrome MCP; and add a regression that simulates a hung default `npx`/`client.connect`/`list_pages` path through `/tabs` with bounded response time and cleanup proof.
