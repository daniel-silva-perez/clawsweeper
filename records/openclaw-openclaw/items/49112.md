## Summary

Keep this PR open. Current main still lacks `openclaw doctor` coverage for gateway lock inspection/repair, and the linked #49036 product gap remains, but the submitted implementation needs revision before it is safe to merge.

## Best Possible Solution

Keep this PR open and revise it. The best path is to add a gateway-lock doctor health contribution on `src/flows/doctor-health-contributions.ts`, reuse or expose the same owner-status semantics used by `acquireGatewayLock`, add stale-without-repair and stale-with-repair tests, and update the doctor/gateway docs after the repair behavior is safe.
