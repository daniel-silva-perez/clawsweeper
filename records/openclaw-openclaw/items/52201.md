## Summary

Keep this issue open. Current main still resolves workspaces at agent scope, not route-binding scope: route bindings can select an agent and optional session dmScope, but there is no typed or validated `bindings[].workspace` field, no route result field carrying a workspace override, and reply execution still calls `resolveAgentWorkspaceDir(cfg, agentId)`. The related implementation PRs in the provided GitHub context are closed unmerged, and the contributor's latest comment says a fresh narrower PR is planned.

## Best Possible Solution

Keep the issue open and review a fresh, narrower implementation. If maintainers accept binding-level workspace isolation, the implementation should add a typed and documented `bindings[].workspace` field, validate/trim it in the config schema and generated docs, carry the matched override through route resolution, apply it when computing bootstrap/context/tool workspace, persist it only where outbound interactions need it, clear it on re-route/rebind paths, and cover precedence/rebind behavior with focused routing and channel tests.
