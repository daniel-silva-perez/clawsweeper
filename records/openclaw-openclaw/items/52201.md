## Summary

Keep this issue open. Current main still supports per-agent workspace isolation through routing bindings, but it does not implement the requested same-agent, per-binding `workspace` override. The earlier workspace override PRs were closed unmerged, and the later linked PR #61460 is also closed unmerged and tracks a different outbound-firewall shape, with the author saying they plan a fresh narrower PR.

## Best Possible Solution

Keep the issue open and ask maintainers to review a fresh narrower implementation. If the binding-level workspace design is accepted, the likely implementation should add a typed `bindings[].workspace` field, validate/trim it in the config schema, return or carry it through route resolution, thread it through channel message contexts and outbound interaction persistence where needed, and cover precedence/rebind behavior with focused routing and channel tests.
