## Summary

Keep open. Current main still has a sequential `finally` cleanup path where a rejected CLI live-session cleanup prevents bundle MCP cleanup from running, and this PR is a narrow implementation with a focused regression test for that seam. I found no blocking correctness or security finding in the diff; the remaining action is normal maintainer review and CI/targeted validation.

## What This Changes

The PR wraps `runCliAgent` live-session and bundle MCP run-end cleanup calls in independent warning-only guards and adds a regression test that later cleanup still runs after a live-session cleanup rejection.

## Best Possible Solution

Land a narrow cleanup-isolation change in `runCliAgent` so each run-end resource cleanup gets its own best-effort guard and diagnostics, with the added seam test covering continuation after a cleanup rejection.

## Reproduction Assessment

Yes. The high-confidence reproduction is a unit seam: call `runCliAgent` with both cleanup flags enabled, make `closeClaudeLiveSessionForContext` reject, and current main's sequential `finally` path prevents `closeMcpLoopbackServer` from running.

## Solution Assessment

Yes. Independent guards around each cleanup hook are the narrowest maintainable fix for preserving later cleanup and the successful agent result; a shared cleanup helper with timeout could be a future consistency improvement, but is not required for this rejection-continuation bug.
