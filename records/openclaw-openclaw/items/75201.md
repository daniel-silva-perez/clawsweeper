## Summary

Keep open. Current main still awaits the live-session cleanup before the bundle MCP cleanup in one shared `finally` path, so the PR is not obsolete; the proposed change is a narrow follow-up with a focused regression test and no definite blocking defect found in the patch.

## What This Changes

The PR wraps CLI live-session and bundle MCP run-end cleanup calls in independent warning-only guards and adds a seam test that later cleanup still runs after live-session cleanup rejects.

## Best Possible Solution

Land this PR or an equivalent narrow change that keeps each `runCliAgent` run-end cleanup hook best-effort and independently observable, with the added seam test preserving continuation after a cleanup rejection.

## Reproduction Assessment

Yes. On current main, call `runCliAgent` with both cleanup flags enabled, make `closeClaudeLiveSessionForContext` reject, and the later `closeMcpLoopbackServer` await is skipped because it is after the rejecting await in the same `finally` block.

## Solution Assessment

Yes. Independent guards around the two run-end cleanup hooks are the narrowest maintainable fix for preserving later cleanup and the successful run result while still logging cleanup diagnostics.
