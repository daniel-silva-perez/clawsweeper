Keep open. Current main at ef58307f843eed1a7832e9c23c519591f1b7cdbd still does not implement the proposed Node.js SEA packaging path, and this PR remains a live implementation candidate rather than obsolete cleanup. ## Best Possible Solution

If maintainers prefer the existing Docker image and npm tarball packaging path, make that product call explicitly before closing the PR.
