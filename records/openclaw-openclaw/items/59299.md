## Summary

Keep open. This is a recent native Windows Gateway crash report, and current main still uses the schtasks/Startup-folder Windows daemon path with no source, docs, tests, or changelog evidence that AC-to-battery power transitions are fixed.

## Best Possible Solution

Keep this issue open for a native Windows maintainer pass. Reproduce on current main/latest release with npm install plus Gateway service install, capture Task Scheduler history, Event Viewer, `openclaw gateway status --json`, Gateway logs, and `openclaw gateway stability --bundle latest`; then fix `src/daemon/schtasks.ts` or the accepted successor path to keep the gateway allowed to start and continue on battery, with focused tests and Windows docs. PR #68149 may be the right implementation vehicle if maintainers accept that PowerShell ScheduledTasks direction.
