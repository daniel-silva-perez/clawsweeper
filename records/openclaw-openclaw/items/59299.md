## Summary

Keep open. Current main still installs the native Windows Gateway through `schtasks` plus a Startup-folder fallback and has no implemented battery-power task settings; the related PowerShell ScheduledTasks PR is open but unmerged, so the reported AC-to-battery crash should not be closed as fixed.

## What This Changes

The issue requests that the native Windows OpenClaw Gateway daemon keep running when a laptop switches from AC power to battery instead of exiting or crashing.

## Best Possible Solution

Keep this issue open for native Windows daemon handling: reproduce on current main or latest release with npm install plus `openclaw gateway install`, collect Task Scheduler history/Event Viewer/Gateway stability evidence, then either land a narrow battery-settings fix or an accepted PowerShell ScheduledTasks successor with focused tests and Windows docs.

## Reproduction Assessment

Unclear. The likely reproduction path is native Windows npm install, `openclaw gateway install`, then switching from AC power to battery while monitoring Task Scheduler history and Gateway logs, but this review could only verify the source-level Scheduled Task gap rather than live-run it on Windows hardware.

## Solution Assessment

Unclear. The current implementation is not the best end state because it leaves Task Scheduler battery defaults implicit; PR #68149 may solve the battery portion, but its broader admin/fallback/PowerShell execution changes need maintainer review or narrowing before adoption.
