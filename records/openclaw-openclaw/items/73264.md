## Summary

Keep open. This PR is maintainer-protected by both author association and label, and current main does not already implement the intended hardening. The current code still resolves a bare `trash` executable through PATH in both browser cleanup helpers, while service PATH construction still accepts env-configured toolchain dirs and only filters existing PATH entries for relative/tmp paths. The PR is therefore not obsolete; it needs explicit maintainer/security review or an equivalent hardened replacement on main.

## Best Possible Solution

Keep the PR open for maintainer review. If the finding is accepted, land this PR or an equivalent patch that removes PATH-resolved trash execution from both cleanup helpers, filters workspace-derived service PATH entries with realpath-aware tests, and keeps unrelated UI churn out of the security change.
