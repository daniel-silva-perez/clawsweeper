## Summary

Keep open. Current main already solves the Codex WebSocket transport part, but it does not implement the proposed soft/hard `before_tool_call` contract, and the PR branch still has two concrete P2 defects in hook approval merging and live profile-env fallback behavior.

## What This Changes

The PR changes OpenAI WebSocket transport selection, adds soft/hard `before_tool_call` block semantics with docs and tests, and updates live test `.profile` environment loading.

## Best Possible Solution

Refresh the branch against current main, keep the existing endpoint-aware WebSocket guard, implement `blockMode` in the current hook contract and runner/consumer paths, clear approval state whenever a soft block is cleared, make profile loading continue until profile-derived variables or static parsing succeed, and update the public plugin docs plus focused regression tests.

## Reproduction Assessment

Yes. Static inspection gives a high-confidence reproduction path: merge a soft block with `requireApproval`, then a lower-priority `{ block: false }`, and the PR still preserves approval state; similarly, a first shell that emits an unrelated new env var can stop `.profile` fallback early.

## Solution Assessment

No, not as-is. The direction is plausible, but current main already has the better Codex transport guard, and the remaining hook/env changes need a rebase plus the two P2 defects fixed before this is the narrow maintainable solution.
