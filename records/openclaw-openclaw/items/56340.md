## Summary

Keep open. Current main already contains a stronger Codex WebSocket exclusion than this branch, but the proposed soft/hard `before_tool_call` API and behavior are still absent; the PR remains useful but cannot merge as-is because the branch is stale and introduces two concrete functional defects.

## What This Changes

This PR changes OpenAI WebSocket transport selection, adds soft/hard `before_tool_call` block semantics with docs and tests, and updates live test `.profile` env loading across shell candidates.

## Best Possible Solution

Refresh the useful parts against current main: keep the endpoint-gated WebSocket predicate, implement `blockMode` in the current hook contract/runner/consumer with ownership regression tests, update the public docs, and make the live env loader continue until profile-defined variables are loaded or static parsing has run.

## Solution Assessment

Is this the best way to solve the issue? No, not as-is. The direction is plausible, but the branch needs to be refreshed onto current main and fixed so soft-clear approval state and shell fallback behavior match the documented semantics.

- **[P2] Clear approval state when a soft block is cleared:** `src/plugins/hooks.ts:725-728`
  - body: When a higher-priority plugin returns a soft block with `requireApproval` and a lower-priority hook returns `{ block: false }`, this branch clears the block fields but leaves `acc.requireApproval` in the merged result. The consumer then still prompts for approval, so `{ block: false }` does not actually clear a soft approval block.
  - confidence: 0.79
