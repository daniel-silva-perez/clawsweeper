## Summary

Keep this issue open. Current main still has both reported parameter-conflict paths: `sessions_send` exposes `sessionKey`, `label`, and `agentId` as independent optional schema fields and still rejects mixed `sessionKey` + `label`; the `message` tool still exposes poll creation fields on the shared action schema while send actions reject poll-creation params. The provided GitHub context also shows open implementation PRs covering the two halves, including open closing-syntax coverage, so maintainers should review or merge those before this issue is closed.

## Best Possible Solution

Keep this issue open while maintainers review and dedupe the linked implementation PRs. The practical path is to land or intentionally close one `sessions_send` fix from #41255 or #59324, then review #46904 or an equivalent message-tool change. Close only after both conflicts are fixed on main, or after any remaining half is split into a narrower canonical issue.
