## Summary

Keep this issue open. Current main still has both reported parameter-conflict behaviors: `sessions_send` exposes and rejects mixed `sessionKey`/`label` inputs, and the `message` tool still exposes poll fields on the shared action schema while send actions reject poll-creation params. The provided related context also shows open implementation PRs for the two halves, including an open PR with closing syntax for this issue, so this should not be closed before a fix lands or the remaining work is split.

## Best Possible Solution

Keep this issue open while maintainers review and dedupe the linked implementation PRs. The practical path is to land or intentionally close one `sessions_send` fix from #41255 or #59324, then review #46904 or an equivalent message-tool change. Close only after both conflicts are fixed on main, or after any remaining half is split into a narrower canonical issue.
