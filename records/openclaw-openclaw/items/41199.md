## Summary

Keep this issue open. Current main still has both reported parameter-conflict paths: `sessions_send` exposes independent optional `sessionKey` and `label` fields and rejects mixed inputs, while `message` exposes poll creation fields in the shared schema and still throws when `action: send` includes those fields. Open implementation PRs also still reference the issue, so this should wait for maintainer review/dedupe and merge rather than be closed early.

## Best Possible Solution

Keep this issue open while maintainers review and dedupe the linked implementation PRs. The practical path is to land or intentionally close one `sessions_send` fix from #41255 or #59324, then land #46904 or an equivalent message-tool fix, or split any remaining half into a narrower canonical issue before closing this one.
