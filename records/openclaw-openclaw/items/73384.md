## Summary

Keep open. Current main has the merged #72287 host-hook foundation, but it still does not implement this PR's workflow follow-up: typed plugin session actions, host-mediated session attachments, scheduled session turns, plugin-emitted workflow events, bounded finalize retry objects, advanced workflow fixtures, or host-hook recipe docs. The PR is recent, active, security-sensitive SDK/Gateway work, and has same-author stacked follow-up #73526, so it needs maintainer API/security review rather than cleanup closure.

## Best Possible Solution

Keep this PR open as the canonical consolidated workflow follow-up to merged #72287. Maintainers should review the SDK session-action, attachment, scheduling, finalize-retry, registry, Gateway protocol, generated client, docs, fixture, cleanup, authorization, and side-effect changes together, then either land it after validation or explicitly close it with a product/API rationale. Coordinate that decision with stacked follow-up #73526.
