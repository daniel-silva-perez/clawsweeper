## Summary

Keep this PR open. Current main still does not implement the PR’s intended payload fallback: `buildEmbeddedRunPayloads` prefers any non-empty `assistantTexts` over the final visible text from `lastAssistant`, so the unit-level `/reasoning on` + earlier block-reply-content scenario can still drop the final answer from assembled payloads. The linked bug issue is closed, but I did not find an equivalent main-branch implementation or regression coverage for this exact assembled-payload path.

## Best Possible Solution

Keep this PR open and either review/rebase it or supersede it with an equivalent main-branch fix in the reply runtime. The fix should ensure `/reasoning on` plus block replies cannot leave the assembled final payload without the final visible assistant answer, while deduping existing single-chunk and multi-chunk current-message content. Add regression coverage for earlier block text, already-covered final answers, chunk suffixes, hard/soft chunk joins, and the reasoning-off guard before closing.
