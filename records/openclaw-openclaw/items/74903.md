## Summary

Keep open. Current main still has the reported Feishu topic mismatch: message-tool-only group prompting steers visible replies to `message(action=send)`, while the Feishu `send` action path only enables native topic/thread delivery for explicit `thread-reply`.

## What This Changes

The issue requests that Feishu `group_topic` and `group_topic_sender` message-tool replies post back into the native Feishu/Lark topic thread instead of as top-level group messages.

## Best Possible Solution

Add Feishu-owned topic-aware message-tool delivery so same-conversation visible sends from `group_topic` and `group_topic_sender` sessions carry the topic root into Feishu `reply_in_thread` transport, while ordinary Feishu group sends keep the existing top-level behavior.

## Solution Assessment
