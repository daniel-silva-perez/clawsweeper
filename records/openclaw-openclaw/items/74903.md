## Summary

Keep open. Current main still has the reported Feishu topic mismatch: group tool-only prompting steers visible output to `message(action=send)`, while the Feishu `send` handler only sets Feishu thread delivery for explicit `thread-reply`, so tool-driven visible replies from `group_topic` and `group_topic_sender` sessions can still post at the group level instead of in the topic thread.

## What This Changes

The issue requests Feishu group topic sessions to preserve the topic/thread target when the assistant uses the message tool for visible group replies.

## Best Possible Solution

Add Feishu-owned topic-aware tool-delivery behavior so same-conversation visible sends from `group_topic` and `group_topic_sender` sessions carry the topic root into Feishu `reply_in_thread` transport, while ordinary group sends keep the existing top-level behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: trigger a Feishu `group_topic` message-tool-only turn, observe the prompt asks for `message(action=send)`, then follow the Feishu send action path where `replyInThread` remains false unless the action is explicitly `thread-reply`.

## Solution Assessment

Yes. The narrow maintainable fix should stay at the Feishu/plugin threading boundary, likely by adding Feishu threading tool context or action-resolution behavior; changing the generic group prompt for every channel would be broader and less robust.
