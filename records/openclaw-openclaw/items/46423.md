Keep this PR open. Current main still does not implement the intended recurring transient retry shortcut, but this branch should not merge as-is because it changes the one-shot-only `cron.retry.backoffMs` contract for recurring jobs and bundles unrelated guardrail allowlists outside the cron fix.

## What This Changes

The PR changes recurring `cron`/`every` error handling to classify transient failures and retry on a short backoff path, adds cron restart/regression coverage, and carries unrelated Discord/test/script CI-drift edits.

## Best Possible Solution

Land a focused cron scheduler fix that retries transient recurring failures before the next natural schedule while preserving permanent-error cadence, force-run cadence, restart catch-up behavior, and an explicit recurring retry contract. The final diff should keep targeted cron tests and remove unrelated guardrail or CI-drift changes.
