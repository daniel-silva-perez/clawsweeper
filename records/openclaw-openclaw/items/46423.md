## Summary

Keep this PR open. Current main still does not implement the PR’s central behavior: recurring `cron`/`every` jobs do not classify transient errors and retry on the short backoff window before the next natural schedule. The PR is paired with open bug #46341 by the same author and uses closing syntax, so it should remain an implementation candidate until maintainers either land a focused version or close the paired work together.

## Best Possible Solution

Keep this PR open, but require maintainer review or a focused refresh. The best path is a small cron-only change in `src/cron/service/timer.ts` that classifies recurring transient failures, retries `cron` and `every` jobs before long natural schedules, preserves manual force-run and restart catch-up semantics, decides whether recurring retry uses a separate config contract or fixed ladder, updates docs if the contract changes, and adds regression coverage for long-interval recurring jobs, internal timeout errors, and restart behavior. Remove unrelated script, Discord, auto-reply, macOS, generated, and CI guardrail changes from this branch before considering merge.
