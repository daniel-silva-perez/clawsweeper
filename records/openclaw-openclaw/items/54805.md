## Summary

Keep this PR open. Current main still does not implement a `plugin_updated` lifecycle hook, and the related feature issue was closed as covered by this PR, but the branch needs changes before merge because its dispatch path loads all plugin runtimes before invoking targeted update hooks and the branch is stale against the current hook contract location.

## What This Changes

The PR adds a typed `plugin_updated` hook contract, updater lifecycle event emission, CLI dispatch after plugin updates, docs/tests, and a Windows formatter `maxBuffer` change.

## Best Possible Solution

Refresh the implementation onto current main by adding the hook contract in `src/plugins/hook-types.ts`, emitting lifecycle events only for real successful version changes, persisting the installed-plugin index through the current update path, loading only updated plugin IDs for targeted dispatch, and keeping docs/tests aligned. Split the unrelated formatter buffer change unless it is still needed and justified by this PR's validation.

## Reproduction Assessment

Yes. Current main can be verified by source inspection: `plugin_updated` is absent from the hook registry, `updateNpmInstalledPlugins` has no lifecycle callback, and `runPluginUpdateCommand` has no dispatch step; the PR-specific regression is visible in the added `scope: "all"` registry load before targeted dispatch.

## Solution Assessment

No, not as written. The product direction is sound, but the current patch should use the current `hook-types.ts` contract surface and a scoped `onlyPluginIds` load before targeted `plugin_updated` dispatch.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.78

Full review comments:

- **[P2] Load only updated plugins for lifecycle dispatch:** `src/cli/plugins-update-command.ts:198`
  - body: The added dispatch path materializes the plugin registry with `scope: "all"` before invoking targeted `plugin_updated` handlers. Use the updated plugin IDs with the scoped loader before calling `runPluginUpdatedForPlugin`.
