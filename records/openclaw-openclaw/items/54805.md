## Summary

Keep open. Current main still lacks a `plugin_updated` hook in the typed hook contract and plugin update pipeline, so the PR is not obsolete; the branch is useful but needs maintainer review for the broad registry-load behavior and a rebase onto the current plugin hook/update structure.

## What This Changes

The PR adds a typed `plugin_updated` lifecycle hook, emits update lifecycle events from plugin updates, dispatches targeted hooks from `openclaw plugins update`, adds docs/tests, and raises a Windows formatter `spawnSync` buffer.

## Best Possible Solution

Adapt the hook addition to the current `hook-types.ts` contract and current install-record persistence flow, then dispatch `plugin_updated` after successful writes using a registry load scoped to the updated plugin ids only, with regression coverage for no-op updates, targeted dispatch, and unrelated broken plugins.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.8

Full review comments:

- **[P2] Load only updated plugins before dispatching hooks:** `src/cli/plugins-update-command.ts:195`
  - body: `ensurePluginRegistryLoaded({ scope: "all" })` makes `plugins update` load every configured plugin before running a hook for one updated plugin. The runtime loader throws on plugin load errors, so one broken unrelated plugin can prevent the updated plugin's `plugin_updated` hook from running, and the command also executes unrelated plugin registration code. Scope the registry load to the updated plugin ids before calling `runPluginUpdatedForPlugin`.
  - confidence: 0.83

## Security Review

Status: needs_attention

Summary: The diff introduces a broader-than-needed plugin code execution path during plugin updates.

Concerns:

- **[medium] Broad plugin loading during update:** `src/cli/plugins-update-command.ts:195`
  - body: The PR loads the full runtime plugin registry after a plugin update to discover `plugin_updated` hooks. Because `plugins update` otherwise avoids plugin preload, this can execute unrelated installed plugin code during a targeted update and can let unrelated load failures suppress the lifecycle dispatch. Limit the load to the updated plugin ids.
  - confidence: 0.78
