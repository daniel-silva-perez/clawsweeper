## Summary

Keep this PR open because current main still lacks the requested `plugin_updated` lifecycle hook, but the patch needs changes before merge. The branch is stale against the active hook contract and its dispatch path loads all plugin runtimes before targeted hook execution, which broadens update-time plugin code execution.

## What This Changes

The PR adds a typed `plugin_updated` hook for successful plugin updates, emits updater lifecycle events, dispatches them from `openclaw plugins update`, adds docs/tests, and includes a formatter `spawnSync` buffer tweak.

## Best Possible Solution

Refresh the feature onto current main as a narrow plugin contract change: define `plugin_updated` in `src/plugins/hook-types.ts`, emit events only for real successful plugin updates, persist install records through the current ledger path, load only the updated plugin IDs before targeted dispatch, document the public hook behavior, and keep unrelated script changes separate.

## Reproduction Assessment

Yes. Source and patch inspection are enough: the PR head calls `ensurePluginRegistryLoaded({ scope: "all" })` before targeted dispatch, while current main already supports `onlyPluginIds` for scoped runtime loading.

## Solution Assessment

No, not as written. The feature direction is sound and the earlier no-op emission bug was fixed, but the maintainable solution needs the current hook contract file and scoped runtime loading for only the updated plugins.
