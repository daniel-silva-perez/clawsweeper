## Summary

Keep this PR open. Current main has a related symlink-stability fix, but it does not implement this PR's versioned `.pnpm/.../node_modules/openclaw/openclaw.mjs` fallback, so the proposed regression path can still resolve to a version-pinned package path instead of the stable global `node_modules/openclaw` tree.

## Best Possible Solution

Keep the PR open as a focused implementation candidate. The best path is to adapt it onto current main by adding stable pnpm package-root candidate discovery in `src/daemon/program-args.ts`, preserving the existing symlinked-dist behavior, and landing the versioned global `openclaw.mjs` regression test with targeted daemon tests.
