## Summary

Current main implements the requested behavior: the TUI model catalog asks the Gateway for the configured view, and the shared visibility helper filters to configured, allowlisted, or auth-backed providers unless an explicit all-catalog view is requested. The same behavior is covered by Gateway and slash-command tests, so this issue can close as fixed on main.

## What This Changes

The request is for the TUI `/models` flow to hide providers and models that are not configured or authenticated, matching `openclaw models list`.

## Best Possible Solution

Keep the shared configured-model visibility path as the default for TUI and slash-command model browsing, with explicit `all` views reserved for full catalog inspection.

## Reproduction Assessment

Yes. The original older-version reproduction is clear, and current main can be verified through the TUI `models.list` configured request plus tests that hide unauthenticated providers by default.

## Solution Assessment

Yes. Centralizing the filtering in `resolveVisibleModelCatalog` is the narrow maintainable fix because it makes TUI, Gateway, and slash-command model browsing share the same configured/auth-backed behavior while preserving an explicit full-catalog escape hatch.
