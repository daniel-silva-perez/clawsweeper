## Summary

Keep open. Current main still does not make the terminal TUI `/models` picker auth-status-aware. The gateway-backed TUI requests the default `models.list` view, and that default is explicitly tested to return the full catalog when no `agents.defaults.models` allowlist exists. The Control UI already requests the configured view, but that path has not been applied to the TUI, and related #40412 only covers search/configured-allowlist behavior, not this auth-only filtering request.

## Best Possible Solution

Keep this issue open and implement a shared auth-aware model-listing path for the terminal TUI `/model` picker and `/models` command. The likely fix should make both gateway-backed and embedded TUI modes hide unauthenticated provider/catalog rows when usable auth exists, preserve explicit `agents.defaults.models` allowlist behavior, retain a sensible fresh-install fallback, and add regression coverage for an auth-only setup with no allowlist.
