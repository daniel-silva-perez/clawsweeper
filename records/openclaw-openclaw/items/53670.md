## Summary

Keep open. Current main and v2026.4.26 still probe channel accounts serially, so this PR addresses a real remaining performance gap, but the branch needs maintainer review because its concurrency test is timing-sensitive, the health snapshot path now has runtime-state and redaction semantics that must be preserved, and the diff includes unrelated agent-test formatting churn.

## What This Changes

The PR replaces the serial per-account health probe loop in `getHealthSnapshot()` with `Promise.all`, adds a Telegram multi-account concurrency test, and reformats one unrelated subagent workspace test.

## Best Possible Solution

Land the performance improvement only after rebasing it onto current main, preserving account summary order, runtime snapshot merging, and admin-only raw probe payload exposure; replace the timing-based test with deterministic concurrency proof and keep the diff scoped to the health implementation and its focused tests.
