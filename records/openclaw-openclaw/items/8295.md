## Summary

Keep this issue open. Current main still lacks Telegram `allowBots` in the Telegram config types, strict Zod schema, generated config metadata/help, docs, and inbound group/topic runtime policy. The Plaiground ClawHub skill mentioned in the discussion is a useful Discord workaround, PR #68325 is the active candidate to review, and PR #8310 was closed unmerged, but none of those ships the requested bundled Telegram plugin behavior on main.

## Best Possible Solution

Keep this issue open and review/land PR #68325 or a fresh Telegram plugin fix that adds `allowBots` across account/group/topic config types, strict schema, generated metadata/help, docs, and inbound group/topic runtime filtering. The implementation should explicitly document Telegram-side requirements and include regression tests for bot-authored group/topic messages, same-bot echoes, mention gating, and loop-safe defaults. If maintainers decide Telegram group bot-to-bot delivery is not supportable, document that product boundary and keep the Discord Plaiground/ClawHub workaround as an alternative rather than closing this as implemented.
