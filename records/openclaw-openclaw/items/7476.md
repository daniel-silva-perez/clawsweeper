## Summary

Keep this issue open. Current main still lacks an outbound WhatsApp sticker send path: WebP media is still routed through the generic image payload, there is no `asSticker` or sticker send option in the active WhatsApp send contract, and the WhatsApp action surface still advertises/handles react and poll behavior rather than `sticker`. The reporter’s local patch and related PR #7502 are useful implementation context, but that PR is closed and unmerged.

## Best Possible Solution

Keep this issue open and implement the capability in the bundled WhatsApp plugin. The clean path is to add an explicit outbound sticker contract, such as `action=sticker` or `asSticker`, preserve intended WebP sticker media before generic image optimization, route outbound sends through Baileys `{ sticker: buffer }`, advertise and dispatch the WhatsApp sticker action or option consistently, and add focused tests for media loading, send-api payload selection, message-action dispatch, and docs.

## Risks / Open Questions

- No live WhatsApp send was performed during this read-only review; the keep-open decision is based on current source, docs, tests, related PR state, and Baileys source/types.
- A naive automatic WebP detection implementation could misclassify ordinary WebP image sends; an explicit `action=sticker` or `asSticker` contract would reduce that risk.
- Changes to `src/media/web-media.ts` are shared across channels, so WebP preservation should be scoped carefully to WhatsApp sticker intent rather than changing all image optimization behavior.
