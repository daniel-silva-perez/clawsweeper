The PR changes the Control UI slash-command executor and tests so `/model status` and `/model list` display model information instead of calling `sessions.patch`, while reusing a trimmed argument for normal model changes.

## Best Possible Solution

Land one rebased Control UI fix that recognizes `/model status` and `/model list` before the model-setting branch while preserving current `SlashCommandContext`, configured model catalog loading, provider-qualified override resolution, and focused regression tests for both aliases.

Yes. Source inspection gives a high-confidence reproduction: in current main, Control UI `/model status` is parsed as local command `model` with args `status`, and `executeModel` sends any non-empty arg to `sessions.patch`, matching the linked bug logs.
