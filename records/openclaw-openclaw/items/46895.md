## Summary

Keep open. Current main still routes Control UI `/model status` and `/model list` through the model-setting path, so this PR is not obsolete, but the branch is stale against the current executor and would bypass the configured-catalog model-info path now on main.

## What This Changes

The PR changes the Control UI slash-command executor and tests so `/model status` and `/model list` display model information instead of calling `sessions.patch`, while reusing a trimmed model argument for normal model changes.

## Best Possible Solution

Land a rebased Control UI fix that recognizes `/model status` and `/model list` before the model-setting branch while preserving `SlashCommandContext`, `{ view: "configured" }` catalog loading, local catalog reuse, provider-qualified model override behavior, and focused regression tests for both aliases.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: on current main, `/model status` and `/model list` are non-empty args and therefore reach `sessions.patch`; the paired bug report includes matching gateway error logs.

## Solution Assessment

No. The requested behavior is correct, but this PR is not the best implementation as-is because it is based on an older executor path; the safer fix is to rebase the alias guard onto current main's model-info/catalog path and update the tests to expect the configured catalog contract.
