## Summary

Keep open. Current main still has the WebUI-local `/model` path treating non-empty `status` and `list` arguments as model selections, so this PR is not obsolete and remains paired with the open same-author bug report #46894. The branch is stale against current main, though: its new tests still assert the old `models.list` payload shape, while main now requests the configured model list.

## What This Changes

The branch changes the Control UI slash-command executor to treat `/model status` and `/model list` as informational local commands and adds executor tests that assert `sessions.patch` is not called for those aliases.

## Best Possible Solution

Land a focused Control UI fix on current main that treats `/model status` and `/model list` as informational local commands without calling `sessions.patch`, preserves current configured-catalog and provider-qualified model resolution behavior, and adds regression tests for both aliases with the current RPC payload contract.
