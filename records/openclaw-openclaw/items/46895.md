## Summary

Keep open. Current main still sends non-empty WebUI `/model` arguments through `sessions.patch`, so the reported `/model status` and `/model list` bug is not implemented on main. The PR targets the right behavior, but the branch is stale against the current Control UI executor and would bypass current configured-catalog handling if merged as-is.

## What This Changes

The PR changes the Control UI slash-command executor and tests so `/model status` and `/model list` display model information instead of calling `sessions.patch`, while reusing a trimmed model argument for the normal set-model path.

## Best Possible Solution

Land a rebased Control UI fix that handles `/model status` and `/model list` before the model-setting branch while preserving the current `SlashCommandContext`, configured catalog request, local catalog reuse, and provider-qualified model override behavior, with focused regression tests for both aliases.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: on current main, `/model status` and `/model list` are non-empty args and therefore reach `sessions.patch`; the paired bug report includes matching gateway error logs.

## Solution Assessment

No. The requested behavior is correct, but this PR is not the best implementation as-is because it is based on an older executor path; the safer fix is to rebase the alias guard onto current main's model-info/catalog path and update the tests to expect the configured catalog contract.
