## Summary

Keep open. Current main has useful mitigations for the CLI timeout symptom and better gateway CPU/event-loop diagnostics, but the full 2026.4.8 umbrella is not proven fixed or shipped: the same-author Discord voice report #63098 is still open, the gateway CPU/RPC-stall tracker #72338 is still open, and the current Discord voice join path still uses a hard-coded 15s Ready wait that destroys the connection on timeout.

## What This Changes

Fix regressions reported after upgrading from OpenClaw 2026.4.1 to 2026.4.8 affecting gateway CPU/RPC responsiveness, `openclaw agent --json` timeouts, Discord voice joins, and a Messenger bridge that depends on the CLI response.

## Best Possible Solution

Keep this umbrella open until maintainers either fix and ship the Discord voice failure in #63098 and the Gateway CPU/RPC-stall family in #72338, or explicitly split the remaining work into accepted canonical follow-ups and close this as superseded with retest evidence.

## Reproduction Assessment

No, not for the full umbrella against current main. The report gives concrete reproduction steps, but high-confidence verification would require an affected low-core Linux host plus live Discord voice and Messenger bridge coverage; this review only source-checked matching failure paths and open trackers.

## Solution Assessment

Not applicable as a patch, and current main is not enough to call the issue solved. The narrowest maintainable path is to land or close the two narrower trackers with packaged-release proof, then retest the CLI/Messenger flow as downstream confirmation.
