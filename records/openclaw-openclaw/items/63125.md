## Summary

Keep open. Current main still has unresolved evidence for both major remaining parts of the umbrella: the Discord voice join path still matches the reported Ready-timeout/abort failure, and the gateway high-CPU/RPC-stall class is still represented by the newer open #72338 current-release tracker. The CLI timeout path has mitigation via longer Gateway timeout and embedded fallback, but that does not prove the Gateway CPU/RPC-stall root cause or Messenger bridge failure is fixed.

## Best Possible Solution

Keep this issue open as the umbrella until maintainers either fix the Gateway CPU/RPC-stall on current main/latest release or explicitly split that remaining work to #72338. Route the Discord voice portion through #63098 with focused coverage for the initial Ready abort/timeout path plus a longer/configurable timeout and bounded retry behavior. Keep the CLI embedded fallback, but do not treat it as proof that the Gateway CPU/RPC root cause is fixed.
