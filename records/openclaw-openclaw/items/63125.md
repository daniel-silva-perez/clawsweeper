## Summary

Keep open. Current main has mitigations for the CLI timeout symptom, diagnostics for gateway event-loop pressure, and plugin-runtime mirror caching work, but it does not prove the full gateway CPU/RPC-stall regression is fixed or shipped. The Discord voice part is still actively tracked by same-author #63098, and the current-release gateway CPU/status-timeout family is still actively tracked by #72338.

## What This Changes

The issue reports gateway CPU/RPC stalls, CLI gateway timeouts, Discord voice join aborts, and Messenger bridge reply failures after upgrading from 2026.4.1 to 2026.4.8.

## Best Possible Solution

Keep this umbrella open until the Discord voice failure in #63098 and the gateway CPU/RPC-stall family in #72338 are either fixed and shipped with release provenance or explicitly split into maintainer-accepted canonical follow-ups.
