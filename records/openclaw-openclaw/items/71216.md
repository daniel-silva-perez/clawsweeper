## Summary

Keep this issue open. Current main still does not provide the requested first-class config/runtime surfaces, and the request bundles product, routing, and security-boundary decisions that should not be closed or auto-repaired as a single cleanup item.

## What This Changes

The issue requests config schema and runtime support for root `sandbox`, tag-based `routing.rules`, multi-gateway `instances`, and `gateway.nodes.denyPaths`.

## Best Possible Solution

Keep the issue open and split it deliberately: decide whether root `sandbox` should alias or replace `agents.defaults.sandbox`, design tag-aware model/provider routing with explicit metadata propagation, decide whether `instances.*` belongs in config or gateway/profile management, and implement `gateway.nodes.denyPaths` only with schema, docs, doctor coverage, runtime enforcement, and security tests.

## Reproduction Assessment

Not applicable as a runtime bug report. The schema gap is high-confidence to verify by inspecting the current strict config schema/types and the sandbox/routing/node-policy runtime paths on main.

## Solution Assessment

Unclear that the issue’s bundled shape is the best single solution. The maintainable path is to split the four asks and require runtime enforcement for the security-sensitive path-deny surface rather than accepting a schema-only PR.
