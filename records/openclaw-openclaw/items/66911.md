## Summary

Keep this PR open. Current main still does not implement `openclaw models auth clean` or an equivalent user-facing stale auth-profile pruning command, although main does include fixes for the underlying stale runtime overwrite and auth-state persistence problems. The remaining decision is maintainer review: whether this cleanup command is still desired as a CLI affordance after the underlying #41634 corruption path was fixed.

## What This Changes

This PR adds an `openclaw models auth clean` command with dry-run/JSON/agent support, auth-profile sanitizer/validation helpers and tests, and auth-store loading changes intended to prune stale persisted profiles without leaking inherited or runtime-only credentials.

## Best Possible Solution
