## Summary

Current main still does not strip a leading `mcp_` from tool calls such as `mcp_exec`, while this PR's dependency patch is obsolete for the current dependency graph. The reported bug remains plausible, but the PR should not merge as-is because it patches `@mariozechner/pi-agent-core@0.55.3` and changes exec-capable dispatch through a package patch rather than the current OpenClaw stream-normalization seam.

## Best Possible Solution

Keep this PR open, but rework it against current main. The best fix is a guarded OpenClaw-side extension to `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts`: resolve exact registered names first, then strip one leading `mcp_` only when the suffix is non-empty and resolves unambiguously to an allowed tool. Avoid a pnpm dependency patch, preserve future exact `mcp_` tool names, and add focused regression tests for prefixed, bare, missing, bundled/plugin, and normal tool names.
