## Summary

Keep this PR open. The reported `mcp_exec`/`mcp_read`/`mcp_write` dispatch miss is still not implemented on current main, but this PR should not merge as-is: it patches `@mariozechner/pi-agent-core@0.55.3`, while current main uses `@mariozechner/pi-agent-core` 0.70.5 and has no pi-agent-core patchedDependency. The right fix is a guarded OpenClaw-side normalization change in the embedded runner.

## Best Possible Solution

Rework this PR against current main by extending `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts`: resolve exact registered names first, then strip a leading single-underscore `mcp_` only when the suffix is non-empty and resolves unambiguously to an allowed tool such as `exec`, `read`, or `write`. Do not add a pnpm dependency patch. Preserve exact `mcp__bundle__*` names and add focused regression tests for `mcp_exec`, `mcp_read`, `mcp_write`, bare `mcp_`, nonexistent `mcp_foo`, exact `mcp__bundle__*`, and normal `exec/read/write`.
