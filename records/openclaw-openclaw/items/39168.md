## Summary

Keep this PR open. The reporter provided useful live repro context for `mcp_exec`/`mcp_read`/`mcp_write`, and the underlying bug remains plausible on current main: OpenClaw normalizes `functions.*` and `tools/*` names before pi-agent-core dispatch, but not `mcp_` aliases. The submitted patch should not merge as-is because it targets `@mariozechner/pi-agent-core@0.55.3`, while current main uses `0.70.5`, and it changes a compiled dependency/patch configuration instead of the current OpenClaw stream-normalization seam.

## Best Possible Solution

Keep this PR open, but rework it against current main. Drop the pnpm dependency patch and extend `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts` so exact registered names resolve first, then one leading `mcp_` alias is stripped only when the suffix is non-empty, not a `mcp__...` plugin-style name, and resolves unambiguously to an allowed tool. Add focused tests for prefixed, bare, missing, bundled/plugin, and normal tool names.
