## Summary

Keep this PR open. The reported `mcp_exec`/`mcp_read`/`mcp_write` dispatch miss is still not implemented on current main, but the submitted patch is obsolete and should not merge as-is because current main uses `@mariozechner/pi-agent-core` 0.70.5 and no longer has a pi-agent-core patchedDependency path. The right fix is a guarded OpenClaw-side normalization extension in the embedded runner, with security-sensitive care around exact names and `mcp__bundle__*` names.

## Best Possible Solution

Rework this PR against current main by extending `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts`: resolve exact registered names first, then strip a leading single-underscore `mcp_` only when the suffix is non-empty and resolves unambiguously to an allowed tool such as `exec`, `read`, or `write`. Do not add a pnpm dependency patch. Preserve `mcp__bundle__*` exact names and add focused regression tests for `mcp_exec`, `mcp_read`, `mcp_write`, bare `mcp_`, nonexistent `mcp_foo`, exact `mcp__bundle__*`, and normal `exec/read/write`.
