## Summary

Keep open. Current main still applies symmetric jitter after a finite `retryAfterMs`, so the reported lower-bound violation is not already implemented or superseded; the PR diff appears to address the central `src/infra/retry.ts` behavior and adds focused regression coverage, with no blocking review findings from source inspection.

## What This Changes

This PR adds a private positive-only jitter mode to `src/infra/retry.ts` for satisfiable `retryAfterMs` delays and adds a dedicated regression test file for Retry-After lower-bound cases.

## Best Possible Solution

Review and land this narrow retry-helper fix after rebase and CI, preserving symmetric jitter for ordinary exponential backoff while enforcing server-supplied Retry-After as a lower bound whenever the configured max delay can honor it.

## Reproduction Assessment

Yes: current main has a deterministic source-level reproduction by mocking `generateSecureFraction` to 0 and calling `retryAsync` with `retryAfterMs: () => 1000` and `jitter: 0.5`, which computes a 500 ms delay. The PR’s new regression test encodes that path directly.

## Solution Assessment

Yes: the proposed positive-only jitter mode is a narrow, private implementation change that keeps existing public API behavior for non-Retry-After callers and handles the `retryAfterMs === maxDelayMs` boundary without undercutting the server hint. A broader follow-up may align the newer memory-host duplicate helper, but that does not make this patch unsafe.
