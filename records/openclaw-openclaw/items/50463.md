## Summary

Keep this PR open. Current main still does not implement the requested model-visible hint when a registered skill is called as an unknown tool, so the PR is not obsolete; however, the current patch needs rework because its hook approach does not match the pinned `@mariozechner/pi-agent-core@0.70.5` public/runtime contract.

## Best Possible Solution

Keep this PR open as a focused implementation candidate. The best path is to rebase onto current main, wire the hint through a supported missing-tool result path for the pinned pi-agent packages or update the dependency contract explicitly, preserve existing extension/plugin `afterToolCall` behavior, define visible-skill matching deliberately, and add targeted regression tests for skill-match, non-match, and collision cases.

## Risks / Open Questions

- The current PR head appears incompatible with the pinned published `pi-agent-core@0.70.5` API because it calls `activeSession.agent.setAfterToolCall(...)`, which is not present in the published Agent type/source.
- The common missing-tool path in the pinned agent loop returns an immediate `Tool X not found` result before the `afterToolCall` finalization path, so the current hook may not cover the user-visible behavior it is meant to improve.
- Skill matching still needs maintainer/product judgment for `resolvedSkills` versus `skills`, `$`-prefixed spelling, punctuation/spaces, casing, and tool/skill name collisions.
- Current main has no regression coverage for skill-match rewrite, non-skill unknown-tool behavior, or collision handling.
