## Summary

Keep open. Current main fixes the adjacent `failoverReason: null` symptom for bare `404 status code (no body)` errors by classifying HTTP 404s as `model_not_found`, but I still did not find source, test, changelog, or history evidence that the broader post-`openclaw doctor`/update regression causing all OpenAI and Google calls to return 404 has been root-caused or fixed.

## Best Possible Solution

Keep this issue open for maintainer/root-cause verification of the post-`openclaw doctor` update path. A good resolution would reproduce or rule out the 2026.3.24 doctor/update mutation with OpenAI and Google configured, add a targeted regression test for the implicated config/runtime path if one exists, or explicitly close after maintainer confirmation that the only remaining observable bug was the already-shipped HTTP 404 fallback classification from 2026.4.7.
