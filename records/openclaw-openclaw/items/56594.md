## Summary

Keep open. Current main fixes the adjacent bare-HTTP-404 failover classification, but the issue report is broader: after `openclaw doctor` updated the install to 2026.3.24, all OpenAI and Google model calls failed from Telegram/WhatsApp. I did not find source, test, changelog, or history evidence that this doctor/update-triggered all-provider regression has been reproduced and fixed.

## Best Possible Solution

Keep this issue open for maintainer root-cause verification of the 2026.3.24 `openclaw doctor`/update path with OpenAI and Google configured. A good resolution would reproduce or rule out doctor/update mutation of provider config/auth/model refs, add a targeted regression test for the implicated path, or explicitly close after maintainer confirmation that the only remaining observable bug was the already-shipped HTTP 404 fallback classification.
