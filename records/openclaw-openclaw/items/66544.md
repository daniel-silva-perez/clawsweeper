## Summary

Keep this PR open. Current main still lets `origin.label` flow into the Gateway session row `displayName`, and the heartbeat sender fallback can still be the literal `"heartbeat"`. The PR is a plausible fix candidate, but it should be revised or reviewed because the current patch filters the label globally rather than by synthetic heartbeat provenance; related #66656 addresses preview text, not this display-name fallback.

## Best Possible Solution

Keep this PR open and revise it into a provenance-aware fix: either avoid persisting system-event heartbeat origin metadata, or ignore `origin.label` only when the origin/provider/surface proves it came from a synthetic heartbeat event. Add focused tests for `buildGatewaySessionRow` covering both the synthetic heartbeat fallback and a legitimate chat named `heartbeat`, and coordinate separately with #66656 for transcript preview filtering.
