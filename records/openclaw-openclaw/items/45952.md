## Summary

Keep open. Current main has better WebChat reconnect handling and a server-side `chat.send` ACK/idempotency path, but the requested client-side outbound queue with reconnect replay is still missing. The Control UI either returns early while disconnected or reports a failed send; it does not retain unACKed sends and replay them after reconnect.

## Best Possible Solution

Keep this issue open and implement a Control UI outbound send queue keyed by stable client-generated idempotency keys. Enqueue before clearing the composer, send only on an authenticated open socket, remove an item only after the `chat.send` ACK, replay unACKed items after reconnect, and show per-message failure state after bounded retries. Reuse the existing gateway `chat.send` ACK/idempotency contract and add focused reconnect tests in the Control UI send path.

## Evidence

There is no outbound replay queue carried to the new socket.
