## Summary

Keep this issue open. Current main still labels the Control UI composer plus action as “New session”, confirms it as “Start a new session?”, and routes it through `/new`/`sessions.reset` for the current session key. The gateway does have a separate `sessions.create` API for creating a new dashboard session entry, but the Control UI chat composer does not use it, so the requested UX distinction remains unresolved.

## Best Possible Solution

Keep the issue open and update the Control UI explicitly. The conservative path is to rename the composer plus tooltip/aria/confirmation from “New session” to reset-context wording and adjust tests/docs; the fuller product path is to add a separate “new session entry” action using `sessions.create` while keeping reset-context as its own action.
