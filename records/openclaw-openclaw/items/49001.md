## Summary

Keep this PR open. Current main still lacks the durable post-reset startup window and queued-owner-message recovery steer that the PR is trying to add, and the broader related Discord channel/thread PR is still only an open implementation candidate in the provided GitHub context.

## Best Possible Solution

Keep this PR open for maintainer review or fold it into a combined successor with #53524: add a durable post-reset startup marker for Discord thread and channel resets, wrap queued owner steers with startup-recovery instructions, clear the marker only after the intended recovery lifecycle succeeds, and keep focused regression tests for both surfaces.
