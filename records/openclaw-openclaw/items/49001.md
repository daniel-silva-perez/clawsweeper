## Summary

Keep this PR open. Current main still lacks the durable post-reset startup window, queued owner-message recovery steer, clearing lifecycle, and focused regression coverage described by this PR. The discussion usefully links the broader Discord channel/thread follow-up in #53524 and the related memory-lancedb-pro plugin fix, but those are context and implementation candidates rather than shipped proof on main. Security pass found no obvious supply-chain or CI-sensitive changes in this PR's file list; the remaining concerns are functional review issues around the startup-window lifecycle.

## Best Possible Solution

Keep this PR open for maintainer review, or supersede it with a corrected combined implementation that covers Discord thread and channel resets. The implementation should persist a post-reset startup marker, wrap queued owner steers with startup-recovery instructions, clear the marker only after the intended recovery lifecycle succeeds, and retain focused regression tests for both surfaces.
