## Summary

Keep open: current main still exposes the documented generic MCP channel tools with sparse text in `content` and the actionable payloads only in `structuredContent`. The related fix PR #57576 is closed unmerged, so there is no implemented or superseding fix to close against.

## What This Changes

The issue asks for MCP channel tool results to include their full structured payload as JSON text in `content` while preserving `structuredContent` for clients that support it.

## Best Possible Solution

Keep the issue open and update the core MCP channel tool result shape so documented generic MCP clients receive self-sufficient JSON text in `content` while existing `structuredContent` and Claude channel behavior remain intact.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: call an affected MCP channel tool such as `conversations_list` with one returned conversation, and the handler returns `content[0].text` as `conversations: 1` while the conversation array is only in `structuredContent`.

## Solution Assessment

Yes. The narrow maintainable fix is a shared success-result helper that mirrors the existing structured payload into JSON text `content` without changing gateway route semantics, error responses, or Claude-specific notifications.
