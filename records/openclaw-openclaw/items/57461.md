## Summary

Keep open. Current main still matches the reported failure mode: `openclaw mcp serve` exposes generic MCP channel tools, but the affected tools return their actionable payloads in `structuredContent` while `content` remains count/status text. The related fix PR #57576 was closed unmerged, so it is not a shipped implementation or a canonical replacement.

## Best Possible Solution

Keep this issue open and fix the core MCP channel tool result shape. A conservative implementation should add a shared success-result helper, likely in `src/mcp/channel-shared.ts`, that preserves `structuredContent` and mirrors the same payload into JSON text `content`; then add focused `src/mcp/channel-server.test.ts` coverage for representative tools proving `content[0].text` is parseable and `structuredContent` remains present.
