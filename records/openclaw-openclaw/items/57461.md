## Summary

Keep open. Current main still documents generic MCP clients as supported, but the MCP channel tools return actionable payloads in `structuredContent` while `content` remains count or short status text. The related fix PR #57576 is closed unmerged, so there is no shipped implementation to close against.

## Best Possible Solution

Keep this issue open and fix the core MCP channel tool result shape. A conservative implementation should add a shared success-result helper, likely in `src/mcp/channel-shared.ts`, that preserves `structuredContent` and mirrors the same payload into parseable JSON text `content`; then add focused `src/mcp/channel-server.test.ts` coverage for representative tools proving `content[0].text` is usable while `structuredContent` remains present.
