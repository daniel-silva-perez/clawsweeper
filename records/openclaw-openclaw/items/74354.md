## Summary

Keep open. Current main still lacks the Control UI composer safeguards and regression coverage for numeric-only input, and the linked open PR is the active implementation path that should be reviewed or replaced before this issue closes.

## What This Changes

The issue reports that pure numeric Control UI webchat input such as `123456789` is being rewritten with separators before it is sent.

## Best Possible Solution

Land PR #74369, or an equivalent narrow Control UI change, so the composer discourages browser/OS rewriting of numeric text and tests exact numeric payload preservation without adding server-side deformatting that could corrupt intentionally hyphenated input.

## Reproduction Assessment

Unclear. The issue provides a concrete Windows Control UI path and screenshot, and the current code path explains how a rewritten textarea value would be sent, but this read-only review did not live-reproduce the browser/IME formatting behavior.

## Solution Assessment

Yes. The best fix path is the existing narrow Control UI composer/test change; a Gateway sanitizer that rewrites `123-456-789` back to digits would be riskier because hyphenated IDs and prose can be intentional.
