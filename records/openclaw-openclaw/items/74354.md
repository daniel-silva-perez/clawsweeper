## Summary

Keep open. Current main still lacks the Control UI composer safeguards and regression coverage for numeric-only chat input, and the linked open PR is the right implementation path before this issue can close.

## What This Changes

The issue requests that Control UI preserve numeric-only chat messages such as `123456789` instead of allowing them to become separator-formatted text such as `123-456-789`.

## Best Possible Solution

Land PR #74369, or an equivalent narrow Control UI change, so the chat composer discourages browser/OS rewriting of numeric text and tests exact numeric payload preservation without adding server-side deformatting that could corrupt intentional hyphenated input.
