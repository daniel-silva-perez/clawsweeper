## Summary

Keep open. Current main still lacks the Control UI composer attributes and regression coverage needed for numeric-only preservation, and open PR #74369 explicitly targets this issue, so the issue should remain open until that PR lands or is replaced.

## What This Changes

The issue requests that Control UI chat preserve pure numeric input such as `123456789` instead of allowing it to be sent as grouped text like `123-456-789`.

## Best Possible Solution

Review and land PR #74369, or an equivalent narrow Control UI change, adding plain-text composer attributes and tests for numeric payload preservation while leaving intentional hyphenated input and Gateway sanitization semantics unchanged.
