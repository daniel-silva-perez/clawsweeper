## Summary

Keep open. Current main improves one gateway-status classification path, but it does not prove the reported `kimi/k2p6` embedded-agent failure is fixed; the Kimi Coding catalog still has no `k2p6` model id, Fireworks K2.6 still strips reasoning payload fields, and two open PRs already reference this issue with closing syntax.

## What This Changes

The issue asks OpenClaw to make `kimi/k2p6` agent turns complete normally and make `openclaw gateway status` finish its local WebSocket probe when the gateway service is running.

## Best Possible Solution

Keep this issue open while maintainers identify the real failing route and choose one canonical fix. The end state should include provider-level regression coverage for the K2.6 payload behavior, clear model-ref handling or docs for the intended provider, and enough gateway-status proof to separate service reachability from read-probe failures.
