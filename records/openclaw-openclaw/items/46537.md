## Summary

Keep open: current main still has the reported pending ensure-identity behavior, but this PR as written introduces a merge-precedence regression by making ensure-derived IDs authoritative over already resolved status/event IDs.

## What This Changes

The PR changes ACP ensure-derived identity creation to mark handles with stable backend or agent session IDs as resolved, adds focused tests, and updates a direct ACP spawn fixture expectation.

## Best Possible Solution

Land a corrected ACP identity change that resolves stable-ID ensure results for new or unresolved identities while preserving status/event authority over already resolved identities, with focused coverage for stable ensure, record-only ensure, stale ensure versus fresh status, startup skip behavior, and status-refresh failure behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Prevent ensure identities from overwriting resolved IDs:** `src/acp/runtime/session-identity.ts:164-166`
  - body: Marking ensure-derived identities as `resolved` makes `mergeSessionIdentity()` accept ensure values even when the current identity is already resolved. A later `ensureRuntimeHandle()` can replace fresher status/event IDs with stale `ensureSession()` IDs, and the `failOnStatusError: false` path can return before status repairs them.
  - confidence: 0.91
