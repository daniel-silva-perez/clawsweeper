## Summary

Keep open: current main still has the ACP ensure-identity behavior this PR targets, but the patch as written has a blocking merge-precedence bug because ensure-derived IDs would gain authority to overwrite already resolved status-derived IDs.

## What This Changes

This PR marks ACP identities returned from ensureSession as resolved when stable backend or agent session IDs exist, adds createIdentityFromEnsure unit coverage, and updates the direct ACP spawn test fixture.

## Best Possible Solution

Land a corrected version that promotes stable-ID ensure results only without letting ensure-sourced values outrank existing resolved status or event identities, with focused coverage for stable-ID ensure, record-only ensure, startup skip, and stale ensure/status precedence.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Prevent ensure identities from overwriting resolved IDs:** `src/acp/runtime/session-identity.ts:164-166`
  - body: Marking ensure-derived identities as resolved makes them authoritative in mergeSessionIdentity. A later ensureRuntimeHandle call can then replace fresher status-derived acpxSessionId/agentSessionId values with stale ensureSession values, and the failOnStatusError: false reconciliation path can leave those stale IDs persisted.
