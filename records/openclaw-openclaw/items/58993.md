## Summary

Keep this PR open for maintainer review. Current main still uses only the deprecated Google Chat `space.type` field for direct-vs-group routing, and the PR’s two-file change is a plausible narrow fix for modern direct-message payloads, but it should be reconciled with the broader linked Space-delivery report and overlapping #42330 work before landing.

## What This Changes

The branch adds `spaceType` and `singleUserBotDm` to the Google Chat space type and changes inbound routing so any legacy or modern direct-message signal is classified as direct instead of group.

## Best Possible Solution

Land or fold in a small Google Chat-owned classifier helper with focused coverage for legacy DMs, modern `DIRECT_MESSAGE`, `singleUserBotDm`, `SPACE`, and `GROUP_CHAT`; then keep the broader Space/group delivery behavior tracked through #58514 or #42330 if symptoms remain.

## Reproduction Assessment

Yes for the PR’s stated classification problem: a current-main event with `space.spaceType: "DIRECT_MESSAGE"` and no legacy `space.type` still computes `isGroup = true` by inspection. The broader #58514 Space/group silence is only partially reproducible from source review because it needs live Google Chat webhook delivery or a fuller integration test.

## Solution Assessment

Unclear as a complete fix for the linked issue, but yes as a narrow fix for modern DM classification. The safer landing shape is to extract or expose the classification seam enough to test the field combinations and coordinate with #42330 before treating #58514 as fully resolved.
