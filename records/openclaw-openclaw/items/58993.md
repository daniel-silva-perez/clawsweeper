## Summary

Keep this PR open. Current main still does not implement the PR's intended Google Chat `spaceType`/`singleUserBotDm` classification change, and the upstream Google Chat Space API does expose those fields while marking legacy `type` deprecated. The PR is a focused implementation candidate, but it needs regression tests and a maintainer check that it actually addresses the linked Space/group-message bug rather than only modern DM payload classification.

## Best Possible Solution

Keep the PR open as a focused implementation candidate, but require a small tested classification seam in the Google Chat plugin: add `spaceType` and `singleUserBotDm` to `GoogleChatSpace`, classify direct messages from legacy `type`, modern `spaceType`, or `singleUserBotDm`, and add unit/integration coverage that asserts the resulting peer kind/session key for legacy DM, modern direct message, bot DM boolean, and SPACE/GROUP_CHAT payloads. Also verify whether the linked Space/group-message issue needs a separate fix if group messages are still dropped after classification is corrected.
