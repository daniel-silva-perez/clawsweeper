## Summary

Keep this issue open. Current main has several ACP/Feishu mitigations since the reporter's version, but it still advertises Cursor as an ACP harness route in the prompt/skill surface when ACP is available, and I did not find a negative regression test or deterministic guard proving that a plain Cursor-related link cannot be interpreted as a Cursor/ACP session request.

## Best Possible Solution

Keep the issue open and add a focused regression around ACP intent routing: a Feishu/custom-agent message containing a Cursor GitHub link or Cursor-related text should remain in the current custom agent session and must not call `sessions_spawn(runtime: "acp", agentId: "cursor")` unless the user explicitly asks to run/use Cursor or ACP. If maintainers consider the existing ACP gating sufficient, close only after linking that test or a live reproduction result on current main.
