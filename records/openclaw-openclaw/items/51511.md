## Summary

Keep this issue open. Current main has meaningful ACP/Feishu mitigations, but Cursor remains advertised as an ACP harness route when ACP is available, and there is still no deterministic guard or regression test proving that a plain Cursor-related link in a custom Feishu session cannot trigger `sessions_spawn(runtime: "acp", agentId: "cursor")`.

## Best Possible Solution

Keep this issue open and add a focused regression around ACP intent routing. A Feishu/custom-agent message containing a Cursor GitHub link or incidental Cursor text should remain in the current custom-agent session and must not call `sessions_spawn(runtime: "acp", agentId: "cursor")` unless the user explicitly asks to run/use/open work in Cursor or ACP. The likely implementation belongs around `src/agents/system-prompt.ts`, `extensions/acpx/skills/acp-router/SKILL.md`, and a targeted prompt/tool-routing or Feishu dispatch test.
