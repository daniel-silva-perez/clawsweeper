## Summary

Keep open. Current main still permits the reported installer outcome: the Unix curl installer can warn that the npm global bin directory is missing from the user's original PATH, skip post-install verification by default, and still print an installed-successfully banner while existing non-root smoke coverage adds the PATH fixup after the installer runs.

## What This Changes

The issue reports that the Unix curl installer on Ubuntu 26.04 says OpenClaw installed successfully even though `openclaw` is not available on PATH afterward.

## Best Possible Solution

Make the Unix installer's default success path align with the user's next command: either ensure the CLI is discoverable without a manual post-install PATH export or turn the missing original-PATH case into a clear action-required/non-success outcome, with focused non-root installer coverage and matching install troubleshooting docs.

## Reproduction Assessment

Yes, at source level. Run the npm-method installer with the npm global bin directory absent from the parent shell PATH and `OPENCLAW_VERIFY_INSTALL` unset; current main can warn about that PATH gap and still print a success banner.

## Solution Assessment

Yes, a narrow installer fix is the best direction. Docs-only guidance or optional `--verify` does not address the misleading default success path for new users.

Current main can warn that the npm global bin dir is missing from the original PATH, skip verification by default, and still print `OpenClaw installed successfully`; the non-root smoke then exports `$HOME/.npm-global/bin` before verifying, masking the user's next-command failure.
