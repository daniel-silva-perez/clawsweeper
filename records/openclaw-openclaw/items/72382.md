## Summary

Keep open. Current main still has the reporter’s observable installer mismatch: the Unix installer can warn that the npm global bin directory is missing from the user’s original PATH, resolve the CLI only inside the installer process, and still continue to the installed-successfully path, so the next `openclaw` command in the parent shell can remain unavailable.

## What This Changes

The issue reports that the Unix curl installer says OpenClaw installed successfully on Ubuntu 26.04 while `openclaw` is not on PATH afterward.

## Best Possible Solution

Make the Unix installer’s normal success path match what the user can run next: either make the installed CLI discoverable after `curl | bash` returns, or make a missing original-PATH bin directory an action-required/non-success outcome before the success banner, with focused smoke coverage and docs aligned to that behavior.

## Reproduction Assessment

Yes, at source level. With the npm install method, original PATH missing the npm global bin directory, and `OPENCLAW_VERIFY_INSTALL` unset, current main can warn about the missing bin dir and still continue to the success banner.

## Solution Assessment

Yes, a narrow installer fix is the best direction. Docs-only guidance or opt-in verification would not address the misleading default success path for first-time installs.
