## Summary

Keep this PR open. Current main at 5828dcdb05aad587bff831467e7fa51f73312ff8 still enqueues 160-character inbound body previews in Slack, MS Teams, and Mattermost system-event labels, while each handler separately delivers the full message body through the inbound envelope path. The PR remains a focused implementation candidate for the paired open bug #67503, not an obsolete or already-implemented change.

## Best Possible Solution

Keep this PR open and review or rebase it as the implementation candidate for #67503. The best product fix is to remove inbound body previews from Slack, MS Teams, and Mattermost system-event labels while preserving full message delivery through the normal inbound envelope path and retaining preview variables only where they still support logging or type-compatible return data.
