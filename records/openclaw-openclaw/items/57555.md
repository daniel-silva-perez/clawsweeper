## Summary

Keep open. Current main still ships the bundled `summarize` skill with a hard `summarize` binary requirement and a single Homebrew installer, with no Linux/download alternative or macOS-only metadata. The gateway enable path only writes `skills.entries.<skillKey>.enabled` and returns success; missing binaries are still handled later as eligibility blockers, so the requested requirement-specific enable feedback is not implemented. Related #14593 and #52894 overlap the broader Docker/Homebrew installer problem, but they do not fully supersede the summarize-specific metadata and Control UI feedback gap.

## Best Possible Solution

Keep this issue open. A focused fix should either remove or move the bundled `summarize` skill to ClawHub, mark it accurately as platform-limited, or add a maintained Linux/download installer path; separately, the Control UI/gateway enable flow should surface a clear missing-requirement message when enabling leaves a skill ineligible.
