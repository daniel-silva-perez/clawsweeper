## Summary

Keep this PR open. Current main still has the default-agent/auth-store mismatch that the PR targets, the linked #74313 report is valid, and this open implementation PR is the right object for maintainer review rather than cleanup closure. The security pass did not find dependency, workflow, lockfile, package, or third-party code-execution changes, but the PR intentionally changes which auth store `models status` reads, so it needs normal maintainer review and green CI before merge.

## What This Changes

This PR changes `models status` and `models status --probe` to resolve a shared agent scope from explicit `--agent`, then compatibility env overrides, then the configured default agent, passes that scope into auth probes, and exempts the live compatibility agent directory from doctor orphan warnings with regression tests.

## Best Possible Solution

Review and land this PR, or a maintainer-equivalent patch, after CI is green. The end state should align `models status` and `--probe` with the documented default-agent contract, preserve explicit `--agent` and compatibility env override precedence, and keep doctor warnings for real orphan agent dirs while exempting only the live compatibility store.
