## Summary

Keep this PR open. Current main still has the documented default-agent mismatch in `models status` and the doctor orphan-dir false positive, while the latest PR patch targets those surfaces with focused scope and no definite blocking bug found in review. Because the change intentionally shifts local auth/model store selection, it should get maintainer review and exact-head validation before merge rather than cleanup closure.

## What This Changes

The PR adds shared agent-scope resolution for `models status` and probes, exempts the live compatibility agent directory from doctor orphan warnings, filters Knip unused-file parser noise, and adds focused regression tests.

## Best Possible Solution

Land this PR, or a maintainer-equivalent patch, after auth compatibility review and green exact-head validation so `models status`, `models status --probe`, and doctor share the documented precedence while doctor suppresses only the live compatibility store and still warns on real orphan agent directories.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a non-main default agent, leave `agents/main/agent` on disk, and current main will still route omitted `models status` through `resolveOpenClawAgentDir()` while doctor can classify the compatibility `main` dir as orphaned.

## Solution Assessment

Yes. The proposed shape is the narrow maintainable fix: centralize status/probe scope resolution, preserve explicit `--agent` and env override precedence, and add a targeted doctor exemption instead of a broad startup migration or compatibility-store removal.
