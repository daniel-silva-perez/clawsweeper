## Summary

Keep this PR open. Current main still has partial text/envelope cleanup for visible chat history, but it does not implement the PR's source-tagged contract: heartbeat/cron/exec-event dispatch contexts are not marked with `InputProvenance.kind = "internal_system"`, and neither the shared gateway projection nor the Control UI history filter hides user-role messages by that provenance. The PR remains a plausible implementation candidate for the linked internal-prompt leak class, with the author's pivot toward trusted source tags matching the safer direction raised in review comments.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate, or land an equivalent fix: set trusted `InputProvenance.kind = "internal_system"` at internal heartbeat/cron/exec/subagent origins, persist it through the existing session-manager path, hide those user-role messages in the shared gateway projection and Control UI defense-in-depth, and add regression coverage for full history snapshots, session-history SSE appends, and normal human messages without that provenance.
