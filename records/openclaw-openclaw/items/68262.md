## Summary

Keep this PR open. Current main has partial display cleanup for legacy internal envelopes and the default heartbeat prompt, but it has not implemented the PR’s provenance-based fix: heartbeat/cron/exec-event turns are not source-tagged as `internal_system`, and gateway/UI history sanitizers do not drop user-role messages by that provenance. The PR remains a plausible implementation candidate for the linked leak class, with the author’s later pivot away from text heuristics toward source tags matching the safer direction raised in review comments.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate or fold an equivalent fix into main: source-tag internally generated heartbeat/cron/exec/subagent user-role turns at origin with `InputProvenance.kind = "internal_system"`, persist that provenance through the existing session-manager path, and have the shared gateway projection plus Control UI defense-in-depth hide those provenance-marked user messages without broad text-prefix heuristics.
