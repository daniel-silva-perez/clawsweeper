## Summary

Keep this PR open. Current main still has only partial display cleanup for legacy internal envelopes, default heartbeat prompts, and heartbeat acknowledgements; it does not implement the PR’s provenance-based source-tagging and hide-by-provenance contract. The author’s later pivot from text heuristics to source tags matches the safer direction raised in review comments, and the PR remains a plausible implementation candidate for the linked history-leak class.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate, or fold in an equivalent fix on main: source-tag internally generated heartbeat/cron/exec-event user-role turns at origin with `InputProvenance.kind = "internal_system"`, persist that provenance through the existing guard/session-manager path, and have the shared gateway projection plus Control UI defense-in-depth hide those messages by provenance rather than broad text heuristics.
