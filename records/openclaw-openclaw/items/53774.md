## Summary

Keep this issue open. Current main still has a fixed TUI footer renderer and no config, CLI, plugin, or template surface for persistent cost, provider balance, or rate-limit status fields; the related open plugin-status work is a useful prerequisite but explicitly does not wire the TUI footer or settle the broader product/security design.

## What This Changes

Request a configurable TUI footer template/custom field surface that can show model and token state plus estimated cost, provider balance, and rate-limit cooldown/status.

## Best Possible Solution

Keep this as the broader product/API tracker. A complete solution should add a reviewed TUI footer customization surface, probably split into a generic footer template/render path plus a plugin/provider status-contribution API, with provider-specific cost, balance, and quota semantics kept behind explicit, cached, secret-safe integrations and documented under the public TUI/plugin docs.
