## Summary

Keep this open. Current main still renders the TUI footer from a fixed field list and has no config, CLI, or plugin-status surface that can persistently show custom cost, provider balance, or cooldown/rate-limit fields; existing `/status`, `/usage`, and `usage.cost` paths are useful adjacent cost/quota surfaces but not the requested footer customization.

## What This Changes

The issue requests a configurable TUI footer template that can show model/tokens plus estimated cost, provider balance, and rate-limit or cooldown status.

## Best Possible Solution

Keep this as the broader product/API tracker. A complete solution should add a reviewed TUI footer customization path, likely split between a generic footer renderer/template and a plugin/provider status contribution API, while keeping provider-specific cost, balance, and quota calls cached, secret-safe, and documented.
