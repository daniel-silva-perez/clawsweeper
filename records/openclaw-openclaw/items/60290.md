## Summary

Keep open. Current main does not implement this PR’s primary session-status fix: `updateSessionStoreAfterAgentRun` still builds `next` from the stale in-memory entry and passes the whole object to `mergeSessionEntry`, whose merge policy spreads the patch over the disk entry. That can still overwrite a lifecycle terminal status with stale `running`. The PR is also only partially obsolete: Mattermost DM threading and MiniMax remaining-percent handling are on main, but the generic-repeat critical escalation and Xiaomi `enable_thinking: false` wrapper are not.

## Best Possible Solution

Keep this PR open, but do not merge it as-is. The best path is to rebase and narrow it to the still-missing session-status fix plus its regression test, then handle the remaining unlanded side fixes such as `generic_repeat` critical escalation and Xiaomi `enable_thinking` in separate focused PRs if maintainers still want them. The already-landed Mattermost and MiniMax portions should be dropped from this branch.
