## Summary

Keep open. Current main still has the session-store write pattern that this PR is trying to fix, while only some bundled changes have independently landed. The PR also carries a concrete MiniMax regression and mixes five unrelated fixes, so it should not be closed as implemented or merged in its current shape.

## What This Changes

This PR updates session-store merging to avoid stale in-memory run status overwriting lifecycle terminal status, and also bundles Mattermost DM threading, generic tool-loop escalation, MiniMax usage percentage, and Xiaomi reasoning-model stream changes with tests.

## Best Possible Solution

Use the session-store fix as the primary landing target: omit lifecycle-owned `status` from the post-run token/model patch and add the regression test. Split or replace the stale bundled PR so already-landed Mattermost/MiniMax work is not reintroduced, the MiniMax scorer gap is fixed, and the remaining tool-loop and Xiaomi fixes can be reviewed on their own merits.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to preseed disk with terminal `status`, `endedAt`, and `runtimeMs`, keep the in-memory `sessionStore` at `status: "running"`, then call `updateSessionStoreAfterAgentRun`; current main still merges the stale in-memory status over the disk entry.

## Solution Assessment

No. The session-store fix direction is the narrow right fix, but this PR is not the best landing shape because it bundles unrelated fixes and contains a MiniMax candidate-scoring bug; a rebased split or narrow replacement would be safer.
