## Summary

Keep open. Current main still routes successful no-visible-response Telegram direct-message turns through a synthetic `NO_REPLY` fallback, the regression test still expects that visible rewrite, and the same-author implementation PR #70630 is open and unmerged.

## What This Changes

The issue requests that Telegram direct-message turns with no visible final reply and no real dispatch or delivery error remain silent instead of synthesizing a `NO_REPLY` payload that becomes visible filler text.

## Best Possible Solution

Review and land PR #70630, or apply an equivalent narrow Telegram adapter change that treats successful no-visible-response direct turns as handled silent no-ops, preserves real error and non-silent delivery failure fallbacks, and flips the Telegram dispatch regression test to assert no DM delivery for this path.
