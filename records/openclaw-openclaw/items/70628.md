## Summary

Keep open. Current main still synthesizes and visibly delivers a Telegram DM `NO_REPLY` fallback for successful no-visible-response turns, the regression test still asserts that behavior, and open same-author PR #70630 remains the unmerged implementation candidate with closing syntax for this issue.

## Best Possible Solution

Keep this issue open and review or land PR #70630, or apply an equivalent narrow Telegram adapter fix. The fix should treat successful no-visible-response Telegram direct turns as handled silent no-ops, avoid synthesizing a transport-visible `NO_REPLY`, preserve the real dispatch-error and non-silent delivery-failure fallback paths, finalize status reactions as done for the silent handled case, and update the Telegram dispatch regression test to assert no DM delivery for this path.
