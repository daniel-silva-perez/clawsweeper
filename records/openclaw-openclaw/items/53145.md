## Summary

Keep open. Current main still sends fire-and-forget `sessions_send` through the A2A announce path, records the literal announce placeholder as an inter-session agent turn, and only treats `ANNOUNCE_SKIP` / `REPLY_SKIP` as sessions-send control replies instead of reusing the shared silent-reply contract. The related open #73550 appears to be the same feedback-loop family and can likely be handled by the same focused fix.

## What This Changes

The issue asks OpenClaw to suppress internal announce placeholders, `NO_REPLY`-style control replies, and acknowledgement echo traffic in async inter-session `sessions_send` flows.

## Best Possible Solution

Centralize the sessions-send non-deliverable control-reply check, suppress those replies before ping-pong or announce delivery, and make announce bookkeeping non-user-visible while keeping real peer replies delivered once.

## Repair Work Prompt

In the sessions-send agent-to-agent flow, add or reuse a shared non-deliverable control-reply predicate for exact `NO_REPLY`, `HEARTBEAT_OK`, `ANNOUNCE_SKIP`, and `REPLY_SKIP`; apply it before ping-pong forwarding and before announce delivery. Preserve legitimate peer A2A replies and the requester-owned background ACP skip path. Do not broaden channel delivery behavior, dependency patches, or unrelated chat-history display filters.
