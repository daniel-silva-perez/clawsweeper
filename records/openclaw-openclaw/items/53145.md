## Summary

Keep open. The discussion and prior Codex review still match current main: peer `sessions_send({ timeoutSeconds: 0 })` continues to start the A2A announce flow, the ping-pong stop condition only recognizes `REPLY_SKIP`, the announce step still submits the literal placeholder as an inter-session user turn, and the sessions-send token helpers still do not classify `NO_REPLY` or `HEARTBEAT_OK` as non-deliverable control replies.

## Best Possible Solution

Keep this open. The best fix belongs in `src/agents/tools/sessions-send-tool.a2a.ts`, `src/agents/tools/sessions-send-tokens.ts`, and the focused sessions-send tests: classify `NO_REPLY`, heartbeat acknowledgements, `ANNOUNCE_SKIP`, and `REPLY_SKIP` through a shared non-deliverable control-reply predicate before ping-pong and announce delivery; avoid submitting the announce placeholder as a normal persisted inter-session user turn; and add regression coverage for peer `sessions_send({ timeoutSeconds: 0 })` covering placeholder suppression, `NO_REPLY`/heartbeat suppression, and no echo loop.
