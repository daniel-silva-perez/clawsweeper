## Summary

Keep open. Current main still has the same peer `sessions_send({ timeoutSeconds: 0 })` A2A announce gap described in the issue and in the prior Codex review: fire-and-forget sends enter the announce flow, ping-pong only stops on `REPLY_SKIP`, the announce phase still submits the literal placeholder as an inter-session agent turn, and the sessions-send token helper still does not treat `NO_REPLY` or `HEARTBEAT_OK` as non-deliverable control replies.

## Best Possible Solution

Keep this open. The best fix belongs in the sessions-send A2A path and token helpers: add a shared non-deliverable control-reply predicate for `NO_REPLY`, heartbeat acknowledgements, `ANNOUNCE_SKIP`, and `REPLY_SKIP`; apply it before ping-pong and announce delivery; avoid submitting the announce placeholder as a normal persisted inter-session user turn; and add focused regression coverage for peer `sessions_send({ timeoutSeconds: 0 })` covering placeholder suppression, control-token suppression, and no echo loop.
