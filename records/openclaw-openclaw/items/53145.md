## Summary

Keep open. Current main still matches the report: fire-and-forget `sessions_send` enters the A2A announce flow, the ping-pong loop only suppresses `REPLY_SKIP`, the announce phase submits the literal placeholder as an inter-session agent turn, and the sessions-send token helper does not classify `NO_REPLY` or `HEARTBEAT_OK` as non-deliverable control replies. The related open #73550 looks like the same duplication/feedback-loop family and can likely be fixed by the same focused change.

## What This Changes

Suppress internal announce placeholders, `NO_REPLY`/acknowledgement control replies, and echoed follow-up traffic in the async inter-session `sessions_send` agent-to-agent flow.

## Best Possible Solution

Add a shared non-deliverable control-reply predicate for sessions-send A2A, suppress exact `NO_REPLY`, `HEARTBEAT_OK`, `ANNOUNCE_SKIP`, and `REPLY_SKIP` before ping-pong and announce delivery, avoid persisting the raw announce placeholder as a normal inter-session user turn, and cover async `timeoutSeconds: 0` plus duplicate/echo regressions in focused tests.
