## Summary

Keep this PR open. Current main still does not implement the intended default-agent behavior: the built-in default workspace resolver ignores OPENCLAW_STATE_DIR, while non-default fallback workspaces already use the normalized state-dir contract. The submitted patch is also stale against the current workspace-default.ts split and would use OPENCLAW_STATE_DIR raw, so it needs revision or an explicit maintainer product decision that workspaces remain a separate knob.

## Best Possible Solution

Keep this PR open and request a focused revision against current main: update src/agents/workspace-default.ts to derive the default workspace base from the normalized state-dir contract if maintainers accept that product behavior, preserve unset-env and OPENCLAW_HOME fallback behavior, keep OPENCLAW_PROFILE suffix behavior, and add targeted tests for unset env, absolute state dir, tilde/relative state dir, default profile, and non-default profile. If maintainers want workspaces to remain explicitly separate from state dirs, document that decision and close the PR after that product call.
