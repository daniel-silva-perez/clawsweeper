## Summary

Keep this PR open. Current main still resolves the default/main agent workspace under the effective home directory instead of `OPENCLAW_STATE_DIR`, while the existing state-dir contract already normalizes `OPENCLAW_STATE_DIR`. The PR targets a real gap, but the submitted patch should not merge as-is because it uses the env value raw and is now stale against the split `src/agents/workspace-default.ts` implementation.

## Best Possible Solution

Keep this PR open and request a focused revision against current main: update `src/agents/workspace-default.ts` to derive the default workspace base from `resolveStateDir(env, homedir)` or equivalent normalized `OPENCLAW_STATE_DIR` handling, preserve existing unset-env and `OPENCLAW_HOME` behavior, keep `OPENCLAW_PROFILE` suffix behavior, and add targeted tests for unset env, absolute state dir, tilde/relative state dir, default profile, and non-default profile. If maintainers prefer workspace isolation to remain explicit, close only after documenting that product decision.
