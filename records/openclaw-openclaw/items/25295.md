## Summary

Keep open. Current main still does not expose Mattermost message-tool edit/delete actions, even though Mattermost now has internal post update/delete helpers for preview streaming. The PR is still a plausible implementation candidate, but it needs a focused refresh before maintainer review because the provided diff mixes unrelated changes and crosses the plugin boundary.

## Best Possible Solution

Keep this PR open and ask for a focused refresh against current main: wire Mattermost-only edit/delete through `describeMattermostMessageTool()`, `supportsAction()`, and `handleAction()`; expose update/delete through an approved Mattermost runtime seam; validate `messageId`, target/account, and non-empty edit text; add focused Mattermost action tests and docs; remove unrelated infra/provider/loader/changelog changes; and review destructive delete semantics explicitly.
