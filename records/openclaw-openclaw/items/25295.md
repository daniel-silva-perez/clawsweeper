## Summary

Current main still does not implement Mattermost message-tool edit/delete actions. Mattermost has internal update/delete post helpers for preview streaming, but the user-facing action adapter only advertises and handles send/react, so this PR is still a live implementation candidate rather than obsolete.

## Best Possible Solution

Keep this PR open and ask for a focused refresh against current main: wire Mattermost `edit` and `delete` through `describeMattermostMessageTool()`, `supportsAction()`, and `handleAction()`; expose any needed update/delete runtime seam without crossing the plugin boundary; validate `messageId`, target/account, and non-empty edit text; add Mattermost action tests and docs; remove unrelated infra/provider/loader/changelog changes; and review destructive delete semantics explicitly.
