## Summary

Current main still does not implement the PR's user-facing Mattermost message-tool edit/delete actions. Mattermost has internal update/delete helpers for preview streaming, but action discovery/support only exposes send/react, handleAction rejects edit/delete, docs do not advertise Mattermost edit/delete, and tests only cover send/react. The PR remains a useful implementation candidate, but it needs a focused refresh and security/scope review before landing.

## Best Possible Solution

Keep this PR open as the implementation candidate, but ask for a focused refresh against current main: wire Mattermost edit/delete through describeMattermostMessageTool, supportsAction, and handleAction; route API calls through SDK-safe helpers or a Mattermost runtime seam; validate messageId, target/account, requester permissions/trust, and non-empty edit text; add Mattermost edit/delete action tests; update public message docs; and drop unrelated infra/provider/loader changes from the branch.
