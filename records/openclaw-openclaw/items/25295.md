## Summary

Keep this PR open. Current main still lacks general Mattermost `edit` and `delete` message actions, so the branch is not obsolete; however, the submitted patch is stale against the current `describeMessageTool` action contract, crosses the bundled plugin boundary, carries unrelated infra/test changes, and needs maintainer review for destructive delete semantics before it can merge.

## What This Changes

This PR branch adds Mattermost edit/delete message actions and post mutation helpers with tests and a changelog entry, while also changing provider-usage tests, plugin-loader tests, Bonjour code, and auth-choice test code outside Mattermost.

## Best Possible Solution

Land a narrowed Mattermost-only implementation on current main that exposes edit/delete through `describeMessageTool`, uses SDK-safe parameter readers or local helpers, validates account routing, post id aliases, and non-empty edit text, updates public CLI and Mattermost docs, adds focused Mattermost action/client tests, and gets explicit approval for destructive delete behavior.

## Reproduction Assessment

Yes. The current-main reproduction is source-level: configured Mattermost accounts advertise only `send` and gated `react`, `handleAction` rejects unsupported actions, and docs/tests still omit general Mattermost edit/delete actions.

## Solution Assessment

No, not as submitted. The feature direction is reasonable, but the patch should be rebased and narrowed to the current action contract and plugin SDK boundary without unrelated infra/test changes.
