## Summary

Current main still does not implement the PR's intended user-facing Mattermost message-tool edit/delete actions. Mattermost now has internal update/delete helpers for preview streaming, but action discovery/support/handling, docs, and tests still cover only send/react for Mattermost, so this PR remains a useful implementation candidate rather than obsolete.

## Best Possible Solution

Keep this PR open as the implementation candidate, but refresh and narrow it against current main: wire Mattermost edit/delete through `describeMattermostMessageTool`, `supportsAction`, and `handleAction`; route API calls through an SDK-safe runtime/helper seam; validate messageId, target/account, requester trust, and non-empty edit content; add focused Mattermost edit/delete tests and public docs; and remove unrelated infra/provider/loader changes from the branch.
