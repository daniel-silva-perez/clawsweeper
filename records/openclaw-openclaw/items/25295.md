## Summary

Keep open. Current main still does not expose general Mattermost edit/delete message actions, so the feature is not obsolete, but the submitted branch is not merge-ready because it targets the older action-discovery shape, imports a core internal from plugin production code, carries unrelated test/infra changes, and exposes destructive delete behavior that needs maintainer sign-off.

## What This Changes

This PR adds Mattermost edit/delete message actions and post mutation helpers with tests and a changelog entry, while also changing provider-usage tests, plugin-loader tests, Bonjour code, and auth-choice test code outside Mattermost.

## Best Possible Solution

Land a narrowed Mattermost-only implementation on current main that exposes edit/delete through `describeMessageTool`, uses SDK-safe parameter readers or local helpers, validates account routing, post id aliases, and non-empty edit text, updates public CLI and Mattermost docs, adds focused Mattermost action/client tests, and gets explicit approval for destructive delete behavior.

## Reproduction Assessment

Yes. The current-main reproduction is source-level: configured Mattermost accounts advertise only `send` and gated `react`, tests assert that same contract, and `handleAction` rejects any other Mattermost action as unsupported.

## Solution Assessment

No, not as submitted. The feature direction is reasonable, but the patch should be rebased and narrowed to the current action contract and plugin SDK boundary without unrelated infra/test changes.
