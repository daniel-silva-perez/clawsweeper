## Summary

Current main still does not implement the Mattermost message-tool edit/delete actions targeted by this PR. Main has Mattermost post update/delete helpers for preview streaming cleanup, but the public message-action adapter still advertises and supports only send/react, and the dispatcher blocks unsupported actions before Mattermost handleAction can run. The PR remains useful, but needs a focused refresh because the provided diff also contains unrelated infra/provider/loader changes and a plugin-boundary violation.

## Best Possible Solution

Keep this PR open and ask for a focused refresh against current main. The best path is Mattermost-only wiring through describeMattermostMessageTool(), supportsAction(), and handleAction(); expose any needed post update/delete helper through a local runtime seam or approved SDK-facing helper; validate messageId/target/account and non-empty edit text; add focused Mattermost action tests and docs; and remove unrelated infra/provider/loader/changelog changes before maintainer review.
