## Summary

Current main partially overlaps with the PR by having Mattermost internal post update/delete helpers and preview streaming, but it still does not implement the PR’s user-facing Mattermost message-tool edit/delete actions. The PR remains an implementation candidate, not something to close as implemented.

## Best Possible Solution

Keep this PR open as the implementation candidate, but ask for a focused refresh against current main: wire Mattermost `edit` and `delete` through `describeMattermostMessageTool`, `supportsAction`, and `handleAction`; use SDK-safe helpers or a Mattermost runtime seam for post update/delete; validate `messageId`, target/account, and non-empty edit text; add Mattermost edit/delete action tests and update the public message docs; and drop unrelated infra/provider/loader changes from the branch.
