## Summary

Keep open. Current main still rejects root-level `paperclip` metadata on `agent` params, the PR is a narrow compatibility fix with corroborating live reports, and no merged/main implementation supersedes it. The remaining maintainer decision is whether to accept this Paperclip-specific top-level shim or land a generic adapter metadata namespace instead.

## What This Changes

This PR adds an optional `paperclip` record to `AgentParamsSchema`, adds a gateway agent-handler regression test for Paperclip metadata, and adjusts one existing reset-prompt assertion in `agent.test.ts`.

## Best Possible Solution

Keep this PR open for maintainer review. The best outcome is one additive Gateway protocol compatibility fix for adapter metadata in `agent` params, preferably a generic extension-safe metadata namespace unless maintainers explicitly approve the Paperclip-specific shim, followed by consolidating #62102, #72565, and #73653 after the accepted fix merges.
