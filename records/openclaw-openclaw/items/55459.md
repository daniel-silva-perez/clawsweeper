## Summary

Keep this issue open. Current main still documents the 1Password exec provider as `args: ["read", ...]` without `--cache=false` or `OP_CACHE=false`, and the exec resolver still only passes configured `args`, `passEnv`, and `env` without any 1Password-specific cache default. The existing generic config surface can express the workaround, but the reported docs/defaults problem remains actionable and is not fully superseded by #72278.

## Best Possible Solution

Keep the issue open. Update the 1Password exec-provider documentation to show `--cache=false` before `read` and/or `env: { OP_CACHE: "false" }` for LaunchAgent/headless setups, then consider a narrow doctor warning or explicit helper for 1Password exec providers. Coordinate separately with #72278 for the broader external-secret batching model.
