## Summary

Keep this PR open. Current main still resolves exec-approvals file and socket defaults from `~/.openclaw` instead of `OPENCLAW_STATE_DIR`, so the PR is not obsolete. The submitted change is plausible but overlaps #65736 and includes security-sensitive symlink validation changes that need maintainer consolidation/review before landing.

Keep this PR open for maintainer handling. The best path is to consolidate it with #65736 or explicitly choose one canonical branch, then land a focused fix that derives exec-approval file and socket defaults from `resolveStateDir()` when `OPENCLAW_STATE_DIR` is set, preserves default and `OPENCLAW_HOME` fallback behavior, adds regression coverage, and resolves the symlink/trusted-root review concerns before merge.
