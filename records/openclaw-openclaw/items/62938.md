Keep this PR open. Current main still hardcodes exec-approvals file and socket defaults under `~/.openclaw`, and the real load/save/ensure paths still flow through those resolvers. The PR targets a current bug linked to #62917, but it overlaps #65736 and its later symlink-validation changes need maintainer/security review before merge.

Keep this PR open for maintainer handling.
