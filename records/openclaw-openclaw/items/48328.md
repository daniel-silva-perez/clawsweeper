## Summary

Keep open. Current main still applies the provider/model filter to lexical FTS keyword search, which is the behavior this PR is trying to remove, and the linked bug remains a plausible current memory-search issue. The PR is stale against the moved memory-core plugin paths, but the intended fix is still useful.

## Best Possible Solution

Keep this PR open and ask for a rebase onto `extensions/memory-core/src/memory/manager-search.ts`, `extensions/memory-core/src/memory/manager.ts`, and `extensions/memory-core/src/memory/manager-search.test.ts`. The best fix is to stop passing/applying provider-model filters in FTS keyword search, keep vector model filtering intact, and add the old-model/new-model keyword regression in the current memory-core tests.
