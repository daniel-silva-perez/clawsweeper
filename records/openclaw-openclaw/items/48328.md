## Summary

Keep open. Current main still applies the active provider model as a lexical FTS keyword-search filter when an embedding provider is configured, so the behavior this PR is trying to fix is still present. The patch is stale against the current memory-core plugin paths, but the intended fix remains relevant to open bug #48300.

## Best Possible Solution

Keep this PR open and ask for a rebase onto `extensions/memory-core/src/memory/manager-search.ts`, `extensions/memory-core/src/memory/manager.ts`, and `extensions/memory-core/src/memory/manager-search.test.ts`. The best fix is to remove the provider-model parameter and SQL predicate from FTS keyword search, preserve vector model filtering, and add a regression that proves keyword hits indexed under old and new embedding models both surface in hybrid search.
