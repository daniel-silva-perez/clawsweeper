## Summary

Keep open. Current main still contains the behavior this PR is trying to remove: lexical FTS keyword search is filtered by the active embedding provider model when a provider is configured, so keyword matches indexed under a different embedding model can still be hidden. The PR is stale because it targets the old `src/memory/*` paths, but the intended memory-core fix remains relevant to the open linked bug.

## Best Possible Solution

Keep this PR open as a stale but still useful implementation candidate. Ask for a rebase onto `extensions/memory-core/src/memory/manager-search.ts`, `extensions/memory-core/src/memory/manager.ts`, and `extensions/memory-core/src/memory/manager-search.test.ts`, removing the provider-model predicate from lexical FTS keyword search while preserving vector model filtering and adding a regression that old-model and new-model keyword hits both appear in hybrid search.
