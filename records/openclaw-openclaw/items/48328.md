## Summary

Keep open. Current main still has the behavior this PR is trying to remove: keyword FTS search accepts the active provider model and adds a `model = ?` predicate, so lexical hits indexed under an older embedding model can still be hidden. The PR is stale because its patch targets legacy `src/memory/*` paths, but the linked #48300 bug remains relevant and the PR is still a useful implementation candidate after rebase.

## Best Possible Solution

Keep this PR open as a stale but still useful bug-fix candidate. Rebase it onto `extensions/memory-core/src/memory/manager-search.ts`, `extensions/memory-core/src/memory/manager.ts`, and `extensions/memory-core/src/memory/manager-search.test.ts`; remove the provider-model predicate from lexical FTS keyword search, preserve vector model filtering, and add a regression showing old-model and new-model keyword hits both appear in hybrid search for #48300.
