## Summary

Keep open. Current main still blocks IPv6 ULA DNS answers even when the web_fetch RFC2544 fake-IP opt-in is enabled, and the directly related fix PR is closed unmerged; because the requested change relaxes an SSRF/private-network boundary, it needs maintainer/security direction rather than cleanup closure.

## What This Changes

The issue requests that fake-IP proxy SSRF handling cover IPv6 fc00::/7 ULA DNS answers, either under the existing allowRfc2544BenchmarkRange option or through a separate opt-in.

## Best Possible Solution

Define the intended SSRF policy contract first, then either add a separate default-off IPv6 ULA fake-IP opt-in with schema/types/help/docs/cache-key/audit/test coverage, or document that ULA remains intentionally blocked for web_fetch.
