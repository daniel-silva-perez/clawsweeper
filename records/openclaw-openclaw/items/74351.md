## Summary

Keep open. Current main still blocks IPv6 ULA DNS answers under web_fetch even when the existing RFC2544 fake-IP opt-in is enabled, and changing that behavior would broaden an SSRF policy boundary that needs maintainer/security design rather than automatic cleanup.

## What This Changes

The issue requests that the web_fetch SSRF fake-IP exemption cover IPv6 ULA fake-IP proxy answers, or add a separate configuration option for that range.

## Best Possible Solution

Keep this open for maintainer/security design. The safest product direction is either a separate default-off IPv6 ULA fake-IP opt-in with schema/types/docs/audit/cache/test coverage, or an explicit decision to keep ULA blocked and document that allowRfc2544BenchmarkRange intentionally covers only 198.18.0.0/15.
