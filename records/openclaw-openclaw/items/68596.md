## Summary

Keep this issue open. Current main still uses a 30,000 ms TUI streaming watchdog by default, production TUI does not pass a user/config override, and the ACP stream settings/docs cover coalescing/projection rather than this client-side watchdog threshold. Related #69978 and PR #69026 overlap, but neither is merged nor a clear canonical replacement for this older active configurability request.

## Best Possible Solution

Keep the issue open and implement a supported TUI streaming watchdog control. A conservative path is to read a documented `openclaw.json` field or persisted TUI preference, plumb the resolved value into `createEventHandlers` as `streamingWatchdogMs`, support a disable mode if accepted, update schema/help/docs when config-backed, and add focused tests for default 30s behavior, a longer threshold, disabled mode, and any per-run warning dedupe chosen from #69978.
