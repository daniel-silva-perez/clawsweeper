## Summary

Keep this open. Current main still uses a 30-second TUI streaming watchdog by default, exposes only an internal handler option for tests/custom callers, and has no user-facing config, CLI option, TUI setting, or slash command for threshold/disablement. The related #69026 is still an open implementation attempt, and #69978 is a broader follow-up rather than a merged or maintainer-designated replacement.

## Best Possible Solution

Keep open and implement a user-facing TUI streaming watchdog control. A conservative path is to add validated config or a local TUI setting for threshold and disablement, plumb it through `TuiOptions` into `createEventHandlers`, document it on the public TUI/config pages, and add regression tests for default, configured threshold, disabled watchdog, and any `/watchdog` command if maintainers choose that UX. Coordinate with #69026 for the focused code change and #69978 for duplicate-warning/runtime-command design.
