## Summary

Keep open. Current main still uses a 30-second TUI streaming watchdog fallback, and normal TUI startup does not pass the existing internal override. The recent watchdog liveness work reduces false idle during active tool/lifecycle events, but there is still no user-facing config, CLI flag, slash command, or docs surface for tuning or disabling the threshold.

## What This Changes

Expose a user-facing configurable TUI streaming watchdog threshold or disable control for long silent reasoning and tool-call phases.

## Best Possible Solution

Add a validated user-facing TUI streaming watchdog control, plumb it through TUI startup into `createEventHandlers`, document it on the public TUI/config surfaces, and cover default, configured threshold, disabled watchdog, and any chosen runtime command behavior with focused tests.

## Reproduction Assessment

Yes. The current source path shows ordinary TUI startup uses the 30-second fallback, and existing fake-timer tests demonstrate the watchdog warning/reset behavior through the same handler path.

## Solution Assessment

Unclear. The internal handler option is the right low-level seam, but the best user-facing solution still needs maintainer choice between config, CLI option, persistent TUI setting, and/or `/watchdog` command.
