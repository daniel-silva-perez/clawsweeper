## Summary

Keep this issue open. Current main still defaults the TUI streaming watchdog to 30 seconds and only supports an internal `streamingWatchdogMs` handler option; there is no user-facing `openclaw.json`, ACP stream, CLI, TUI settings, or slash-command control for the threshold. The related #69978 is a broader follow-up, not a maintainer-designated canonical replacement, and #69026 is an open implementation attempt rather than a merged fix.

## Best Possible Solution

Keep open and implement a user-facing TUI streaming watchdog control: add a validated config or local TUI setting for threshold and disablement, plumb it into `TuiOptions`/`createEventHandlers`, document it on the public TUI/config pages, and add regression tests for configured default, disabled watchdog, and any runtime `/watchdog` command if maintainers choose that UX. Coordinate with #69978 for duplicate-warning suppression and runtime command design, and evaluate whether #69026 can be adapted rather than closing this before a merged fix exists.
