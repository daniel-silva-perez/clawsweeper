## Summary

This should stay open. Current main has several adjacent hardening pieces for temporary paths and durable trajectory exports, but it still lacks the requested general provenance/classification model for agent-created temp artifacts, cleanup/janitor semantics, and broad runtime-critical volatile-path validation.

## What This Changes

Request adding a default temp artifact provenance/classification system plus durable-path and runtime-critical temp-path guardrails for OpenClaw-created files outside the workspace.

## Best Possible Solution

Define a small core artifact-provenance contract first: a durable registry or sidecar schema under OpenClaw state, a plugin/agent-tool API for declaring scratch/reproducible/durable artifacts, doctor/security/janitor reporting that understands those classes, and a targeted runtime-path validator for service wrappers, module targets, and active install/build paths that resolve into volatile temp directories.

## Reproduction Assessment

Not applicable as a conventional bug reproduction. This is a feature/safety request, and the current-main gap is reproducible by inspecting the existing temp helpers, trajectory export path handling, service/security checks, and the absence of a general temp artifact provenance registry or classifier.

## Solution Assessment

Unclear until maintainers choose the product and API boundary. The requested direction is plausible, but the maintainable path should start with a narrow provenance contract and reporting surface before attempting to cover every agent-created file and runtime-critical temp path category.
