## Summary

This should stay open. Current main has adjacent safety work, including a private OpenClaw temp root, a static temp-path guardrail, durable trajectory exports, and service/security checks, but it does not provide the requested general provenance/classification model for agent-created temp artifacts or a broad durable-path/runtime-critical temp guardrail.

## What This Changes

Add first-class provenance, cleanup classification, durable-path routing, and runtime-critical temp-path warnings for agent-created artifacts outside the workspace.

## Best Possible Solution

Define a small core artifact-provenance contract first: a durable registry or sidecar schema under OpenClaw state, a plugin/agent-tool API for declaring scratch/reproducible/durable artifacts, doctor/security/janitor reporting that understands those classes, and a targeted runtime-path validator for service wrappers, module targets, and active install/build paths that resolve into volatile temp directories.

## Reproduction Assessment

Not applicable as a conventional bug reproduction. This is a feature/safety request, and the current-main review verifies the gap by inspecting existing temp helpers, trajectory export behavior, doctor/service checks, security audit checks, and the absence of a general temp artifact provenance registry.

## Solution Assessment

Unclear without maintainer product design. The requested direction is plausible and aligned with safe defaults, but the maintainable path likely starts with a narrow provenance API and doctor/security reporting contract before trying to cover all agent-created files or every runtime-critical path category.
