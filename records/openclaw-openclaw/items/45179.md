## Summary

Keep open. Current main has useful partial temp-path hardening, but it still lacks the requested first-class provenance/classification model, durable rerouting semantics, cleanup/janitor awareness, and broad runtime-critical temp-path validation; the remaining work is core/SDK/security policy design rather than a stale or ClawHub-only request.

## What This Changes

Add first-class provenance and cleanup semantics for agent-created temp artifacts, plus guardrails for runtime-critical paths that resolve into `/tmp` or `/private/tmp`.

## Best Possible Solution

Define a core temp artifact contract with classes such as scratch, reproducible, and durable; expose an additive SDK/runtime seam; migrate OpenClaw-owned temp writers to record provenance; teach doctor/security/cleanup how to interpret artifact classes; and add owned runtime-critical path checks where OpenClaw can reliably warn or fail.

## Reproduction Assessment

Not applicable as a feature/security-hardening request rather than a single executable bug. The high-confidence verification path is source inspection: current helpers, SDK exports, audit checks, docs, and targeted searches show the requested generic provenance and durable-path contract is still absent.

## Solution Assessment

Unclear for immediate implementation because the issue asks for the right direction but spans core, SDK, doctor/cleanup, and security enforcement. The narrowest maintainable path is a maintainer-approved contract and policy design first, then focused implementation PRs against that design.
