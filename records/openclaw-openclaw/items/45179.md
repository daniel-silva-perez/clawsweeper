## Summary

Keep open. Current main has a secure shared temp directory helper, cleanup closures for temporary downloads, an optional diffs plugin with TTL metadata, and a few narrow `/tmp` warnings, but it does not implement the requested built-in temp artifact classification/provenance registry, janitor awareness, or comprehensive runtime-critical temp-path guardrails.

## What This Changes

Request built-in provenance/classification for agent-created temp artifacts, cleanup awareness, durable-path defaults, and runtime-critical `/tmp` guardrails.

## Best Possible Solution

Keep the issue open and split the desired end state into a maintainer-approved design: define a first-class temp artifact registry/classification contract, decide which agent/tool writes must use it by default, add janitor/doctor reporting semantics, and add explicit runtime-critical temp-path checks where OpenClaw owns the path.
