## Summary

Keep open. Current main has partial temp hardening and cleanup primitives, but it still does not provide the requested default temp-artifact classification/provenance registry, janitor semantics, durable rerouting, or broad runtime-critical temp-path guardrails.

## What This Changes

The issue requests built-in provenance classes for agent-created temp artifacts, cleanup-aware metadata, durable-path defaults, and warnings or failures for runtime-critical paths under `/tmp` or `/private/tmp`.

## Best Possible Solution

Define a first-class temp artifact provenance contract, expose it through the OpenClaw-owned temp/artifact helpers and plugin SDK where appropriate, teach cleanup/doctor tooling to report by class, and add explicit runtime-critical temp-path checks at the path-validation points OpenClaw owns.
