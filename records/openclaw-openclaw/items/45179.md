## Summary

Keep open. Current main has useful temp-path hardening, but the requested first-class provenance/classification contract, durable rerouting semantics, cleanup awareness, and broad runtime-critical temp-path validation are still missing. The reporter's `skill-audit` workaround is useful context, but this is not ClawHub-only because the remaining work needs core/SDK, doctor/cleanup, and security policy design.

## What This Changes

The issue asks OpenClaw to add first-class provenance/classification for temp artifacts, durable-path routing guardrails, cleanup awareness, and runtime-critical `/tmp` path detection.

## Best Possible Solution

Define a first-class temp artifact contract in core, expose a backwards-compatible SDK seam for classification/provenance, migrate OpenClaw-owned temp writers to record it, teach doctor/cleanup/security reporting how to interpret artifact classes, and add explicit runtime-critical temp-path checks at the validation points OpenClaw owns.
