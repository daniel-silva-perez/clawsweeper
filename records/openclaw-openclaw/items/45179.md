## Summary

Keep open. Current main has useful partial temp hardening, including a preferred OpenClaw temp root, temp-download cleanup helpers, diffs artifact TTL cleanup, safe-bin temp warnings, and a static temp-path guard check, but it still lacks the requested default temp-artifact classification/provenance registry, durable rerouting semantics, cleanup/janitor classification, and broad runtime-critical temp-path validation.

## What This Changes

Request to add default temp-artifact provenance, scratch/reproducible/durable classification, durable-path routing, cleanup awareness, and runtime-critical temp-path guardrails.

## Best Possible Solution

Define a first-class temp artifact contract in core, expose a backwards-compatible SDK seam for artifact classification and provenance, update OpenClaw-owned temp writers to use it, teach cleanup/doctor/security reporting to distinguish artifact classes, and add explicit runtime-critical temp-path checks at the path-validation points OpenClaw owns.
