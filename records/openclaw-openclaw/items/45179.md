## Summary

Keep open. Current main has useful partial temp hardening and artifact cleanup, but it still lacks the requested default temp-artifact classification/provenance registry, janitor semantics, and comprehensive runtime-critical temp-path guardrails.

## What This Changes

Request a built-in temp artifact provenance/classification model plus durable-path and runtime-critical `/tmp` guardrails for agent-created artifacts.

## Best Possible Solution

Define a maintainer-approved design for a first-class temp artifact provenance contract, wire it into OpenClaw-owned artifact creation paths, add janitor/doctor reporting semantics, and add explicit runtime-critical temp-path checks where OpenClaw owns or validates the path.
