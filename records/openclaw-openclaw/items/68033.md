## Summary

Keep open. Current main has addressed the OAuth-scope/login half and improves Cloudflare/HTML error classification, but it still routes `openai-codex/*` through the ChatGPT Codex backend and does not implement a headless-container Cloudflare challenge bypass or a public OpenAI Responses fallback with new scopes. The maintainer comment already made that split and explicitly left the Cloudflare/runtime path open; related #72966 is a different gpt-5.5 hang report, not a clean replacement.

## Best Possible Solution

Keep this issue open and narrow it to the remaining `openai-codex` Cloudflare/headless-container runtime path. The likely owner is the OpenAI provider/PI transport layer: either implement a supported Codex transport path that avoids the challenge, add accurate `cf-mitigated: challenge`/403 diagnostics and fallback behavior, or document a confirmed workaround. Treat the `auth.json` wipe as a separate reverify/split item if it still reproduces on current main.
