## Summary

Keep open. Current main still rejects Docker bind sources outside the sandbox/workspace allowlist by default, while the user-facing sandbox docs still show arbitrary host-path examples without documenting the required break-glass flag. The related #59718 is useful context for the broader least-privilege direction, but it does not fully supersede this narrower docs/runtime mismatch.

## Best Possible Solution

Keep this open. The best maintainer path is to decide the intended product contract, then either document the current default allowlist and the `agents.defaults.sandbox.docker.dangerouslyAllowExternalBindSources` break-glass option on https://docs.openclaw.ai/gateway/sandboxing plus the FAQ/config docs, or change the Docker sandbox policy to accept explicitly configured non-blocked external bind sources by default. Coordinate any behavior change with #59718 for `workspaceAccess: "none"` and writable custom-bind semantics.
