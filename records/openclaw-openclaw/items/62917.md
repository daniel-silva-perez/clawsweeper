## Summary

Keep this issue open. Current main still resolves exec approval JSON and socket paths through the home-based `~/.openclaw` defaults and does not consult `OPENCLAW_STATE_DIR`, while the shared state resolver and doctor warning path still make the reported split possible. The open implementation candidate #62938 remains unmerged and is linked with GitHub closing syntax for this issue.

## Best Possible Solution

Keep this issue open and review, land, or close #62938. The best code path is to update `src/infra/exec-approvals.ts` so the approvals file and socket default resolve under `OPENCLAW_STATE_DIR` / the shared state-dir resolver before falling back to the current home default, add focused regression coverage in `src/infra/exec-approvals-store.test.ts`, preserve the existing symlink-hardening behavior, and update exec approvals docs if the storage wording changes.
