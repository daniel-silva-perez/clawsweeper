## Summary

Keep this PR open. Current main partially covers the thread/auth side, but it still does not implement the PR's central Telegram `all_group_chats` command registration/cleanup, and the plugin-command `Command not found.` fallback is still sent without topic thread params. The PR is also stale against the current `extensions/telegram` plugin layout, so the useful work needs a rebase or replacement rather than cleanup closure.

## Best Possible Solution

Keep this PR open and rebase or replace it with a focused `extensions/telegram` patch: register and clear both default and `all_group_chats` command scopes, preserve thread params for the plugin-command not-found fallback via `resolveTelegramThreadSpec` and `buildTelegramThreadParams`, and add regression tests for dual registration, dual cleanup, retry/cache failure behavior, and General-topic id 1 handling.
