Keep open. Current main only partially absorbed this PR: auth/diagnostic native-command replies now use Telegram thread params, but command menu registration/cleanup still targets only the default Telegram scope and the plugin-command “Command not found.” path still sends without topic params. The PR is stale against the current `extensions/telegram` layout, but the remaining behavior is still useful and should be rebased or reimplemented rather than closed.

## Best Possible Solution

Keep this PR open for a focused rebase or replacement in `extensions/telegram`: register and clear Telegram commands for both default and `all_group_chats` scopes, preserve topic params for the plugin-command not-found fallback via the existing thread helpers, and add regression tests for dual registration, dual cleanup, cache/retry behavior, and General-topic id 1 handling.
