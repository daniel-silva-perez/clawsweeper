## Summary

Keep this PR open. Current main still lacks the central Telegram `all_group_chats` command registration/cleanup, and the plugin-command `Command not found.` fallback still sends without topic thread params. Related #6833 and current tests cover part of the auth/threading side, but the remaining group-scope and fallback-threading work is still useful and should be rebased into the current `extensions/telegram` layout.

## Best Possible Solution

Rebase or replace this PR with a focused `extensions/telegram` patch: register and clear both default and `all_group_chats` command scopes, preserve thread params for the plugin-command not-found fallback via `resolveTelegramThreadSpec` and `buildTelegramThreadParams`, and add regression tests for dual registration, dual cleanup, retry/cache behavior, and General-topic id 1 handling.
