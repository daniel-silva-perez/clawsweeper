## Summary

Keep open. Current main still does not add image-specific timestamp, sender, or thread position/total annotations when Slack thread history is assembled, and the directly related implementation PR #48725 was closed unmerged.

## Best Possible Solution

Keep this issue open. Implement a channel-owned Slack fix that annotates image-bearing prior thread-history entries with sent time, sender, and message position/total when they are included in model context, with focused tests and an opt-out only if maintainers want that config surface. Reuse or extract a shared helper later only if another bundled channel needs the same behavior.
