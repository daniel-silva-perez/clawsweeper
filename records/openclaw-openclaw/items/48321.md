## Summary

Keep this issue open. Current main still formats Slack thread history as text envelopes and carries only current-message or thread-starter media into `MediaPaths`; it does not add image-specific timestamp, sender, or thread position annotations, and the related implementation PR was closed unmerged.

## Best Possible Solution

Keep the issue open and implement a channel-owned fix starting with Slack: annotate prior thread-history image/media entries with sent time, sender, and message position/total when included in model context, add focused Slack tests, and document/configure any opt-out. Extract a small shared helper only if multiple channel plugins need the same behavior.
