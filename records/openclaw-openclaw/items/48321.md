## Summary

Keep open. Current main still does not add per-image sent-time, sender, or thread position metadata for Slack thread-history images, and the directly related implementation PR was closed unmerged rather than shipped.

## What This Changes

Request annotation of images included from channel thread history with sent timestamp, message position/total, and sender so vision-capable models can reason about image freshness.

## Best Possible Solution

Implement the Slack behavior in the Slack plugin first: annotate image-bearing prior thread-history entries with sanitized sent time, sender, and message position/total when they are represented in model context, with focused Slack tests and docs/config only if maintainers want an opt-out. Generalize later only after another bundled channel needs the same contract.

## Reproduction Assessment

Yes. A code-level reproduction is clear: a Slack thread history message with `files` flows through `resolveSlackThreadHistory`, but current `resolveSlackThreadContextData` emits only formatted text/message id and no per-image timestamp, sender, or position metadata.

## Solution Assessment

The requested direction is reasonable, but the best maintainable implementation is a Slack-owned fix first rather than copying the closed PR's core-helper shape into extension production code. Any shared helper should be promoted through a documented plugin SDK seam only if multiple channel plugins need it.
