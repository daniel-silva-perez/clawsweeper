## Summary

Keep this issue open. Current main has Slack thread-history text envelopes with sender/timestamp and a thread starter media fallback, but it does not implement image-specific timestamp plus position annotations, `thread.annotateImages`, or a cross-channel image metadata path. The related implementation PR was closed unmerged, so it is useful context but not a resolved canonical tracker.

## Best Possible Solution

Keep the issue open and implement a channel-owned fix, starting with the bundled Slack path from the reproduction: annotate image/media entries with sent time, sender, and thread position/total when they are included as prior thread context. If multiple channels share the same need, add a small generic helper or SDK-safe seam, then cover it with Slack-focused tests and docs/config for any opt-out.
