## Summary

Keep open. Current main still has the linked missing-provider error behavior, and this PR is the right narrow area to fix it, but the patch appears to leave one reported no-provider reproduction on the old generic error path when the input path is not selected as image/audio media.

## What This Changes

The PR threads media-understanding decisions through file runtime results, converts skipped/no-attempt image and audio infer results into missing-provider CLI errors, adds regression tests, and updates the changelog.

## Best Possible Solution

Land a narrow version that gives clear configuration errors for every no-provider image/audio infer path while preserving the existing empty-description/transcript errors only after a provider attempt actually runs; keep #73569 open until the fix merges.
