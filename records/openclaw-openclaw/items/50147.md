## Summary

Keep this PR open. Current main still has the interrupt-mode stale lastAssistant fallback and the auto-reply delivery path still lacks an abort-specific gate. The PR is paired with the same-author open bug #50145, so it remains an active implementation candidate, but the submitted early return is too broad and can regress timeout replies and compaction bookkeeping.

## Best Possible Solution

Keep this PR open and request a narrower revision. The fix should suppress stale `lastAssistant` fallback only for externally interrupted, non-timeout runs with no new assistant text; prevent delivery of stale interrupted-run output; preserve timeout payload delivery, usage persistence, compaction count updates, and post-compaction context injection; and add regression coverage for the #50145 interrupt scenario plus timeout and compaction safeguards.
