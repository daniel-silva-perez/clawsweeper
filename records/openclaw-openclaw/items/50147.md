## Summary

Keep this PR open. Current main still has the stale `lastAssistant` fallback path that the PR targets, and the paired same-author bug #50145 is still open, so this remains an active implementation candidate. The submitted diff should be revised before merge because its abort early-return is too broad and can regress timeout replies and compaction bookkeeping.

## Best Possible Solution

Keep this PR open and request a narrower revision. The fix should suppress stale `lastAssistant` fallback only for externally interrupted, non-timeout runs with no new assistant text; prevent stale interrupted-run output delivery; preserve timeout payload delivery, usage persistence, compaction count updates, and post-compaction context injection; and add regression coverage for #50145 plus timeout and compaction safeguards.
