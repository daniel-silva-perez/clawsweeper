## Summary

Keep open. Current main still has the stale `lastAssistant` fallback and still lacks an aborted-run delivery gate in the auto-reply path, so this PR is not obsolete. It is also paired with the same author's still-open bug report #50145, and the PR as written has concrete functional risks around timeout payload delivery and post-compaction bookkeeping. Security review found no supply-chain or CI-sensitive changes in the PR diff.

## Best Possible Solution

Keep this PR open but request a narrower revision. The fix should suppress stale `lastAssistant` fallback only for externally interrupted runs with no new assistant text, avoid delivering stale interrupted-run output, preserve timeout error payload delivery, preserve usage and compaction bookkeeping, and add regression coverage for the #50145 interrupt scenario plus timeout and post-compaction safeguards.
