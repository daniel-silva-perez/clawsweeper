## Summary

Keep this PR open. Current main still has the stale last-assistant fallback and no aborted-run delivery gate for auto-reply payload delivery, and this PR is paired with the same-author open bug #50145. The patch is not obsolete, but the proposed early return is too broad because it would also suppress timeout payloads and skip compaction follow-up bookkeeping.

## Best Possible Solution

Keep this PR open and request a narrower revision. The fix should suppress stale lastAssistant fallback only for externally interrupted non-timeout runs with no new assistant text, avoid delivering stale interrupted-run output, preserve timeout payload delivery, preserve usage and compaction bookkeeping, and add regression coverage for the #50145 interrupt scenario plus timeout and post-compaction safeguards.
