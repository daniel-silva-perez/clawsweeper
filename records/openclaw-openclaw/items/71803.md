## Summary

Keep open. Current main still applies the CLI no-output watchdog to silent Claude CLI turns without any Monitor-aware or pending-tool expected-idle signal, and the linked open PR only raises timeout constants rather than fixing the reported healthy idle state.

## What This Changes

The issue asks OpenClaw to keep CLI backend sessions alive while a Monitor/tool wait is legitimately idle instead of terminating them after the no-output watchdog window.

## Best Possible Solution

Add an explicit expected-idle/proof-of-life contract for pending Monitor or MCP/tool waits in the CLI runner path, while preserving the overall request timeout and no-output protection for CLIs that are truly stalled.

## Reproduction Assessment

Yes. The issue includes a concrete Monitor wait command and matching watchdog logs, and current main source/tests show silent CLI processes still trip the no-output timeout without a Monitor-aware exemption.

## Solution Assessment

Unclear. Treating verified pending Monitor work as expected idle is the right direction, but maintainers should choose whether the state is detected in the Claude JSONL/live-session layer, a generic CLI tool-wait signal, or a bounded heartbeat mechanism.
