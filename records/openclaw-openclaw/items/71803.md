## Summary

Keep open. Current main and the latest tagged release still apply the CLI no-output watchdog to silent Claude CLI live-session turns with no Monitor-aware expected-idle exemption, and the related open cap-raising work is only a mitigation rather than a complete fix for this report.

## What This Changes

The issue asks OpenClaw to stop killing healthy CLI sessions that are deliberately idle while waiting for a long-running Monitor task to complete.

## Best Possible Solution

Add a CLI-runner proof-of-life or expected-idle contract for pending Monitor/MCP tool waits so healthy long-running background work does not kill the supervising CLI process, while preserving the no-output watchdog and overall timeout for genuinely stalled backends.

## Reproduction Assessment

Yes. The issue gives a concrete Monitor wait plus watchdog log sequence, and current-main source/tests still show silent CLI processes hitting the no-output-timeout path without any Monitor-aware exemption.

## Solution Assessment

Unclear. The issue’s semantic direction is stronger than merely raising caps, but maintainers should choose the contract boundary before implementation because Monitor state may need to come from Claude stream-json events, the MCP wrapper, or a bounded heartbeat.
