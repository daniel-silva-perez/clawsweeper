## Summary

Keep open. Current main still applies the CLI no-output watchdog to silent claude-cli/live-session turns with no Monitor or pending-tool expected-idle exemption, so the reported session-kill path remains plausible; #40982/#41022 may reduce the symptom by raising caps but do not clearly solve the Monitor-specific contract.

## What This Changes

The issue asks OpenClaw to stop the CLI no-output watchdog from killing a claude-cli session that is deliberately idle while waiting for a long-running Monitor/background task.

## Best Possible Solution

Add a CLI-runner proof-of-life or expected-idle contract for pending Monitor/MCP tool waits so healthy long-running background work does not kill the supervising CLI process, while preserving the no-output watchdog for truly stalled backends. Cover both claude live-session and generic supervised CLI paths with focused regression tests, and coordinate the result with the broader watchdog-cap work.

## Reproduction Assessment

Yes. The issue gives a concrete Monitor wait plus watchdog log sequence, and current-main source/tests show that silent CLI processes still hit the no-output-timeout path without a Monitor-aware exemption.

## Solution Assessment

Unclear. Raising timeout caps is a useful mitigation, but the narrower maintainable fix is likely an expected-idle/proof-of-life signal tied to pending Monitor or background-task state so stalled CLI processes can still be terminated.
