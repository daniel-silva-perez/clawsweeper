## Summary

Keep open. Current main still treats a silent CLI process as stalled without any Monitor-aware or pending-tool expected-idle signal, and the related open timeout-bump work does not solve the healthy-idle state described here.

## What This Changes

The issue asks OpenClaw to stop killing Claude CLI sessions that are deliberately silent while waiting for a long-running Monitor task to complete.

## Best Possible Solution

Add an explicit expected-idle or proof-of-life contract for verified pending Monitor/MCP/tool waits in the CLI runner/live-session path, while preserving the overall request timeout and no-output protection for truly stalled CLIs.

## Reproduction Assessment

Yes. The issue includes a concrete Monitor wait command plus matching watchdog logs, and current-main source/tests show silent CLI processes still trip the no-output timeout without a Monitor-aware exemption.

## Solution Assessment

Unclear. Treating verified pending Monitor work as expected idle is the right direction, but the current constant-only related PR is not the best fix; maintainers should choose the contract boundary before implementation.
