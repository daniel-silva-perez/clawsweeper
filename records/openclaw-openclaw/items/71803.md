## Summary

Keep open. Current main still enforces CLI no-output timers for both Claude live sessions and generic supervised CLI runs with no Monitor, pending-tool, or expected-idle exemption; the related timeout-cap work is still open and only mitigates duration, not the reported state mismatch.

## What This Changes

The issue requests that CLI watchdog handling stop terminating sessions that are intentionally idle while waiting for a long-running Monitor/background task.

## Best Possible Solution

Keep this open until the CLI runner can distinguish a stalled backend from expected idle during long-running tool/background waits, or maintainers explicitly decide that a documented watchdog override is the product contract. The end state should include regression coverage for no-output watchdog behavior while a long-running tool wait is pending.

## Reproduction Assessment

Yes. The issue provides concrete logs for a 180000 ms watchdog kill during a Monitor wait, and current-main source/tests show silent CLI runs still reach the no-output timeout path.

## Solution Assessment

Unclear. A Monitor or pending-tool expected-idle signal is likely narrower than raising the global cap alone, but maintainers should choose between tool-state detection, a heartbeat from the Monitor wrapper, and configurable per-backend/session watchdog policy.
