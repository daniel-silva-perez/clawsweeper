## Summary

Keep this PR open. Current main still does not implement the requested `agent.wait` usage/cost metadata, and the PR remains a focused implementation candidate for the closed feature request #49494. The existing review thread also identified concrete issues to resolve before merge, especially lifecycle/dedupe race behavior, best-effort cost calculation, tests, and docs.

## Best Possible Solution

Keep this PR open and refresh it against current main. The best path is an additive gateway protocol change that threads existing `EmbeddedPiAgentMeta` into `AgentWaitTerminalSnapshot`, preserves current terminal fields, keeps cost calculation non-fatal and clearly estimated, adds regression coverage for cached and lifecycle-race paths, and updates the public gateway/agent-loop docs.

## Risks / Open Questions

- The branch predates current main behavior that added `stopReason`, `livenessState`, and `yielded`; a refresh must preserve those terminal fields while adding usage metadata.
- The review comments identify a real determinism risk if lifecycle completion wins before the dedupe payload with usage metadata is written.
- Cost calculation should be best-effort and must not convert a successful agent run into an error if config/pricing lookup fails.
- The new response shape needs focused tests for cached dedupe snapshots and lifecycle/dedupe race paths, plus public gateway/agent-loop docs.
- Estimated `costUsd` may be affected by the separate open cost-accuracy work for OpenRouter tier-priced models (#68066), so naming/docs should make the estimate semantics clear.
