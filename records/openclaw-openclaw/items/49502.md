## Summary

Keep this PR open. Current main still does not implement the requested `agent.wait` usage/cost metadata, while the runtime already carries the needed `EmbeddedPiAgentMeta` usage fields. The PR is still useful, but it needs a refresh against current main, tests/docs, and the existing review concerns around lifecycle/dedupe races and best-effort cost calculation should be resolved before merge.

## Best Possible Solution

Keep this PR open and refresh it against current main. The best path is an additive gateway protocol change that threads existing `EmbeddedPiAgentMeta` into `AgentWaitTerminalSnapshot`, preserves current terminal fields, treats cost calculation as best-effort, adds focused tests for cached and lifecycle-race paths, and updates the public agent-loop/gateway docs.
