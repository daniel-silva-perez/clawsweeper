## Summary

Keep this PR open. Current main still does not implement the requested `agent.wait` usage/cost metadata, while the PR remains a focused implementation candidate for the closed feature request #49494. It should be refreshed and tightened before merge because the review thread identifies real race and best-effort-cost concerns, and current main has since added terminal fields that the PR must preserve.

## Best Possible Solution

Keep this PR open as the implementation candidate. The best path is an additive gateway protocol update that threads existing `EmbeddedPiAgentMeta` into `AgentWaitTerminalSnapshot`, returns optional usage/cost/session/provider/model metadata from cached and lifecycle-race `agent.wait` paths, keeps cost estimation non-fatal, preserves current terminal fields, adds regression coverage for cached and lifecycle-first paths, and updates the public agent-loop/protocol docs.

## Risks / Open Questions

- The PR branch needs a refresh against current main so it preserves the newer `stopReason`, `livenessState`, and `yielded` terminal fields while adding usage metadata.
- Review comments in the PR identify a real nondeterminism risk when lifecycle completion wins before the dedupe payload with metadata is available.
- Cost calculation should stay best-effort; config/pricing lookup failures must not convert a successful agent run into an error response.
- Docs and protocol-facing tests should be updated because this is an additive gateway wire-contract change.
- Estimated `costUsd` should be named/documented carefully because separate OpenRouter tier-pricing accuracy work is still tracked in #68066.
