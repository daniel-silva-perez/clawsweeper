## Summary

Keep this PR open. Current main still does not implement the requested `agent.wait` terminal metadata: the wait snapshot and both response paths expose status/timing/error plus newer stop/liveness/yield fields, but no session, usage, last-call, provider/model, or cost metadata. The PR remains a plausible implementation candidate for the closed feature request #49494, but it needs refresh against current main and the existing review concerns before merge.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is an additive gateway protocol update that threads existing `EmbeddedPiAgentMeta` into `AgentWaitTerminalSnapshot`, returns optional session/usage/last-call/provider/model/estimated-cost metadata from both cached and lifecycle-race `agent.wait` paths, makes cost estimation non-fatal, preserves current terminal fields, and adds focused regression tests plus docs for the new response shape. Maintainers should also decide whether plugin/runtime `waitForRun` wrappers should expose the new metadata.
