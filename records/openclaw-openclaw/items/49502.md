## Summary

Current main still does not implement this PR’s intended gateway protocol change. The `agent.wait` terminal snapshot and both wait response paths expose terminal status/timing/error plus stop/liveness/yield fields, but no optional `meta` payload for `sessionId`, token usage, last-call usage, provider/model, or estimated cost. This PR remains a plausible implementation candidate for the closed feature request #49494, but it should stay open for maintainer review, refresh against current main, tests/docs, and resolution of the race/non-fatal-cost review concerns.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is an additive gateway protocol update that threads existing `EmbeddedPiAgentMeta` into `AgentWaitTerminalSnapshot`, returns optional session/usage/last-call/provider/model/estimated-cost metadata from both cached and lifecycle-race `agent.wait` paths, keeps cost estimation non-fatal, preserves the current stop/liveness/yield fields, and adds focused regression tests plus public docs for the new response shape. Maintainers should also decide whether direct `agent` responses and plugin/runtime `waitForRun` wrappers should expose the same metadata.

## Risks / Open Questions

- The PR branch is stale relative to current main’s `agent.wait` terminal fields and must preserve `stopReason`, `livenessState`, and `yielded` while adding metadata.
- Review comments flagged a likely lifecycle-vs-dedupe race where in-flight waits can still lose metadata unless the dedupe fallback stays alive long enough to read the completed terminal payload.
- Review comments flagged that cost estimation must remain best-effort; config or pricing lookup failures should not convert a successful agent run into an error response.
- Related open issue #68066 reports OpenRouter cost undercounting on tier-priced models, so any `costUsd` field should be documented or named as estimated unless authoritative provider billing is handled.
- The consuming wrapper in `src/agents/run-wait.ts` currently drops any new metadata unless that API is intentionally extended too.
