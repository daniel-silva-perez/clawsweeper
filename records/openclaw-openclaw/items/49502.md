## Summary

Keep this PR open. Current main still does not expose usage/cost metadata on `agent.wait`, while this PR remains a focused implementation candidate for the closed feature request #49494. It needs a refresh against current gateway wait behavior and the review-thread concerns before merge.

## Best Possible Solution

Keep this PR open as the implementation candidate. The best path is an additive gateway protocol update that threads existing `EmbeddedPiAgentMeta` into `AgentWaitTerminalSnapshot`, returns optional session/usage/last-call/provider/model/estimated-cost metadata from both cached and lifecycle-race `agent.wait` paths, keeps cost estimation non-fatal, preserves current terminal fields, and adds focused tests plus public docs for the new response shape.

## Risks / Open Questions

- The PR branch is behind current main and must preserve newer terminal fields such as `stopReason`, `livenessState`, and `yielded` when adding `meta`.
- Review comments identify a real lifecycle-vs-dedupe race risk where in-flight `agent.wait` calls can resolve before metadata is available unless the fallback waits correctly.
- Cost estimation should remain best-effort; config/pricing lookup failures must not convert a successful agent run into an error response.
- `costUsd` should be named and documented as estimated because OpenRouter billed-cost accuracy is separately tracked in #68066.
- This is an additive gateway wire-contract change, so protocol-facing docs and regression tests should land with the implementation.
