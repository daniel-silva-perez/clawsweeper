## Summary

Keep open. Current main still parses browser evaluate input as a parenthesized expression in both ref-based and page-level Playwright paths, so statement bodies beginning with `var`, `let`, or `const` still fail before execution. The related closed #55598 request does not supersede this because current browser tool/schema still exposes `act` with `kind: "evaluate"`, not a separate `browser.exec` or statement-mode API.

## Best Possible Solution

Keep this issue open and implement statement support in the browser plugin evaluate path, likely around `evaluateViaPlaywright`, by detecting expression versus statement bodies or adding a compatible explicit mode. Preserve existing function-expression/ref behavior and return values, keep timeout/abort/navigation guards intact, and add focused tests for page and ref evaluate with `var`, `let`, `const`, async statement bodies, and existing expression/function inputs.

## Risks / Open Questions

- The evaluate path executes arbitrary page JavaScript, so a fix must preserve `browser.evaluateEnabled`, timeout clamping, abort disconnect behavior, and navigation safety checks.
- Existing-session / Chrome MCP evaluate uses a separate `evaluate_script` adapter, so any broader behavior change should verify managed and existing-session contracts separately.
