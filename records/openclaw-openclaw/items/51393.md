## Summary

Keep open. Current main still evaluates the managed browser path by wrapping the supplied `fn` in parentheses before `eval`, so statement inputs such as `var`, `let`, and `const` are still syntax errors. The related closed #55598 thread does not supersede this request because current main does not expose a `browser.exec` action that would cover arbitrary statement execution.

## Best Possible Solution

Implement statement support in the browser plugin evaluate path, likely in `extensions/browser/src/browser/pw-tools-core.interactions.ts`, by detecting expressions versus statement bodies or by adding a compatible explicit mode. Preserve existing function-expression/ref behavior and return values, keep timeout and navigation safety behavior intact, and add focused tests for page and ref evaluate with `var`, `let`, `const`, async statements, and existing expression/function inputs.
