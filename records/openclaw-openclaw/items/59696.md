## Summary

Keep open. Current main still uses the Host-header-derived URL base in the five targeted gateway handlers, while this PR is a narrow hardening change that is not obsolete; it needs maintainer review/rebase and a quick completeness check before landing.

## What This Changes

The PR replaces Host-header-derived URL bases with the literal `http://localhost` in five gateway HTTP handlers that only read path or query data.

## Best Possible Solution

Update this PR against current main, keep the literal `http://localhost` base for request-target parsing where only `pathname` and `searchParams` are consumed, decide whether to include the MCP loopback parser for full gateway consistency, and then run the targeted gateway HTTP handler tests.

## Reproduction Assessment

Yes. Current-main source inspection reproduces the reported condition directly: the five target handlers still build `URL` objects with `req.headers.host`, and a simple Node URL check confirms a literal `http://localhost` base preserves pathname/query parsing for relative request URLs.

## Solution Assessment

Yes for the touched handlers: the literal base is the narrowest maintainable change because these handlers only use path/query fields. For the broader title claim, the solution should either include `src/gateway/mcp-http.request.ts` or explicitly scope that parser out during maintainer review.
