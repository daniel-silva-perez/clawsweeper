## Summary

Keep this PR open. Current main still contains the exact Host-header URL base pattern in every gateway handler this PR edits, so it is not implemented on main. The PR is a focused, security-positive hardening with no dependency/workflow/package changes in the provided diff, but it should likely be updated before merge because another gateway HTTP request helper still has the same pattern.

## Best Possible Solution

Keep the PR open, ask for a small completeness update to cover `src/gateway/mcp-http.request.ts` or explicitly split it to a follow-up, add a focused regression test for hostile/malformed Host header parsing, then review and land if CI passes.
