## Summary

Keep this issue open. Current main still constrains OpenProse `model:` values to the aliases `sonnet`, `opus`, and `haiku` in the bundled VM/compiler docs, while the underlying OpenClaw `sessions_spawn` model path already preserves explicit provider/model refs such as `openai/gpt-5.4`. The remaining bug is an actionable bundled OpenProse plugin/spec/docs gap, not an already-fixed core model-selection issue.

## Best Possible Solution

Fix the bundled OpenProse plugin/spec/docs surface so `model:` accepts canonical `provider/model` refs anywhere it already accepts `sonnet`, `opus`, or `haiku`, while keeping the existing aliases. Update the VM spec, compiler reference, public docs, and at least one example or lightweight regression artifact to show distinct OpenProse agents passing exact model overrides through to `sessions_spawn.model` without falling back to the default subagent model.
