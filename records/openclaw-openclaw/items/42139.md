## Summary

Keep open. Current main now documents ACP/acpx as a Gateway-host harness path, but it still does not provide first-class remote Mac node placement for Discord `/acp spawn`; the config schema and runtime contract only carry agent/backend/mode/cwd-style options, with no node selector or remote execution contract.

## What This Changes

The issue asks for the official ACP/Discord flow to either support or clearly document a Raspberry Pi Gateway dispatching Claude/Codex ACP harness execution to a remote Mac node.

## Best Possible Solution

Keep this as the canonical maintainer decision point. The near-term product state should explicitly document that bundled Discord `/acp spawn` and acpx harnesses execute on the Gateway host today, with workarounds such as running the Gateway/acpx on the Mac or using an operator-managed wrapper. If first-class support is desired, add a typed remote ACP runtime node selector and process-execution contract across config schema, ACP runtime backend APIs, node-host/RPC handling, Discord docs, and regression tests.
