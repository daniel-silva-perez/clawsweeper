## Summary

Current main still does not provide an official ACP/acpx configuration or runtime contract for placing a Discord-spawned ACP harness process on a separate remote Mac node while the Gateway runs elsewhere. The report remains an actionable product/docs gap, not conservative cleanup.

## Best Possible Solution

Keep this open for maintainer product/docs judgment. Short term, document that bundled ACP/acpx harnesses run on the Gateway host today, recommend running the Gateway/acpx on the Mac or using an operator-managed wrapper for multi-machine deployments, and clarify Discord channel versus thread-bound ACP guidance. If OpenClaw should support this topology, add an explicit ACP runtime node selector and process-execution contract across config schema, ACP runtime/acpx integration, and Discord `/acp spawn` docs.
