## Summary

Keep open. Current main documents and implements ACP/acpx as a Gateway-host external harness runtime, while remote node-host support is scoped to `exec host=node`/`system.run`; there is still no typed ACP runtime node selector, Discord binding host selector, or acpx remote process-placement contract. The prior ClawSweeper discussion also kept this open and the stale label was removed afterward, so this remains a real product/docs follow-up rather than cleanup closure.

## What This Changes

The issue asks whether Discord `/acp spawn` can launch Claude/Codex ACP harnesses on a remote Mac node while the Gateway runs on another host, and asks for docs or feature tracking if that topology is unsupported.

## Best Possible Solution

Keep this open for maintainer product/docs judgment. The short-term end state should explicitly document that bundled Discord `/acp spawn` and acpx harnesses run on the Gateway host today, with practical workarounds such as running the Gateway/acpx on the Mac or using an operator-managed wrapper. If first-class support is desired, it should add a typed remote ACP runtime node selector and process-execution contract across config schema, ACP runtime SDK, acpx integration, node-host/RPC handling, Discord docs, and targeted tests.
