## Summary

Keep open. Current main still only suppresses the reminder guard for an enabled cron job with the same session key, so the sweep-backed reminder gap is not implemented on main. The PR is useful source work, but the current patch has a blocking regex-byte defect and applies the new heuristic more broadly than its isolated-cron contract.

## What This Changes

The PR adds a reminder-sweep-like cron-job heuristic to the reminder guard and a runReplyAgent regression test so an existing sweep cron can suppress the unscheduled-reminder note.

## Best Possible Solution

Preserve the existing same-session cron suppression and add a narrow path for enabled isolated agentTurn reminder-sweep jobs, with the regex bytes fixed, the isolated boundary enforced, positive and negative regression coverage, and coordination with the overlapping reminder-guard PRs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use word-boundary escapes for sweep detection:** `src/auto-reply/reply/agent-runner-reminder-guard.ts:36`
  - body: The added regex contains literal U+0008 backspace bytes around the alternation instead of `\b` word-boundary escapes. Normal cron names/messages such as `family-reminder sweep` will not match, so the new suppression path remains inactive and the guard note can still be appended.
  - confidence: 0.97
- **[P2] Limit sweep suppression to isolated cron jobs:** `src/auto-reply/reply/agent-runner-reminder-guard.ts:25`
  - body: The helper applies the reminder/sweep heuristic to every enabled `agentTurn` job. A main, custom, or other-session cron whose prompt mentions reminders or sweeps could suppress the warning for an unrelated reply, which is broader than the PR's stated isolated-cron contract.
  - confidence: 0.86
