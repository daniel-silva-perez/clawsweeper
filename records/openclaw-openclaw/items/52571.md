## Summary

Keep open. The PR targets a real reminder-guard false-positive path, but the proposed patch is not merge-ready: its sweep-detection regex contains literal backspace bytes, so the new suppression path does not work for normal cron names/messages, and the heuristic is broader than the stated isolated-cron contract.

## What This Changes

The PR adds a reminder-sweep-like cron heuristic to `hasSessionRelatedCronJobs` and a runReplyAgent regression test so an existing sweep cron can suppress the unscheduled-reminder note.

## Best Possible Solution

Preserve the existing same-session cron suppression, add a narrow path for enabled isolated `agentTurn` reminder-sweep jobs with real `\b` word-boundary escapes, cover both positive and negative regression cases, and coordinate the final shape with the adjacent reminder-guard PRs.

## Reproduction Assessment

Yes. The PR diff itself contains the reproducer: the added regex has byte `0x08` where `\b` was intended, and a direct Node probe shows the PR's representative sweep job text does not match.

## Solution Assessment

No. The direction is plausible, but this patch is too broad and currently ineffective; the safer fix is to require `sessionTarget === "isolated"`, keep the `agentTurn` and enabled checks, fix the regex bytes, and add negative coverage for non-reminder isolated jobs.
