## Summary

Keep open. Current main still only suppresses the reminder guard for same-session enabled cron jobs, so this PR is not obsolete, but the proposed patch has blocking correctness issues in the new sweep heuristic and overlaps with related open reminder-guard work that needs maintainer consolidation.

## What This Changes

The PR adds a reminder-sweep-like cron heuristic to the reminder guard and a runReplyAgent regression test so an existing sweep cron can suppress the unscheduled-reminder note.

## Best Possible Solution

Preserve the existing same-session cron suppression and add a narrow, documented path for enabled isolated agentTurn reminder-sweep jobs, with positive and negative regression coverage and coordination with the overlapping reminder-guard fixes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use word-boundary escapes for sweep detection:** `src/auto-reply/reply/agent-runner-reminder-guard.ts:36`
  - body: The added regex contains literal U+0008 control bytes around the alternation instead of `\b` word-boundary escapes, so names/messages like `family-reminder sweep` will not match. The new suppression path therefore stays inactive for the regression case and the guard note can still be appended.
  - confidence: 0.97
- **[P2] Limit sweep suppression to isolated cron jobs:** `src/auto-reply/reply/agent-runner-reminder-guard.ts:24-26`
  - body: The helper applies the reminder/sweep heuristic to every enabled `agentTurn` job, not only isolated jobs. A current/custom/session cron from another context whose prompt mentions reminders or sweeps could suppress the warning for an unrelated reply, which is broader than the PR's stated contract.
  - confidence: 0.86
