## Summary

Keep open. Current main and latest release still suppress the unscheduled-reminder note only for successful same-turn cron.add calls or enabled cron jobs with the same sessionKey; they do not recognize an existing isolated reminder sweep. The PR is a useful focused fix candidate, but its head patch needs repair before merge because the regex contains raw backspace bytes, the sweep heuristic is broader than the stated isolated-cron intent, and the added coverage lacks a negative non-reminder isolated-cron case.

## What This Changes

The PR modifies the reminder guard to treat enabled reminder-sweep-like cron jobs as backing reminder promises and adds one regression test for a sweep cron owned by another session.

## Best Possible Solution

Keep this PR open and update it before merge: fix the regex, constrain sweep suppression to enabled isolated agentTurn cron jobs, add positive and negative tests, preserve existing same-session behavior, and compare the final behavior with #43404 so the reminder guard has one coherent isolated-cron contract.
