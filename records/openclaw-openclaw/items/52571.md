## Summary

Keep open. Current main still only suppresses the unscheduled-reminder note for successful cron.add calls or enabled jobs sharing the current sessionKey, so the PR is not obsolete; however, the proposed patch has a definite regex control-byte bug and broadens suppression beyond isolated cron jobs, so it should not merge as-is.

## What This Changes

The PR adds a reminder-sweep-like cron-job heuristic to the reminder guard and a runReplyAgent regression test so an existing sweep cron can suppress the unscheduled-reminder note.

## Best Possible Solution

Preserve the existing same-session cron suppression and add a narrow, documented path for enabled isolated agentTurn reminder-sweep jobs, with the regex fixed, the sessionTarget boundary enforced, and positive plus negative regression coverage coordinated with the overlapping reminder-guard PRs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use word-boundary escapes for sweep detection:** `src/auto-reply/reply/agent-runner-reminder-guard.ts:36`
  - body: The added regex contains literal U+0008 control bytes around the alternation instead of `\b` word-boundary escapes. Normal cron names/messages such as `family-reminder sweep` will not match, so the new suppression path remains inactive and the new regression case would still append the guard note.
  - confidence: 0.97
- **[P2] Limit sweep suppression to isolated cron jobs:** `src/auto-reply/reply/agent-runner-reminder-guard.ts:24-26`
  - body: The helper applies the reminder/sweep heuristic to every enabled `agentTurn` job. A main, custom, or other-session cron whose prompt mentions reminders or sweeps could suppress the warning for an unrelated reply, which is broader than the PR's stated isolated-cron contract.
  - confidence: 0.86
