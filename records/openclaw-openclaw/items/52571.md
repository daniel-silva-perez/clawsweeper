## Summary

Keep open. Current main and the latest release still only suppress the reminder note for same-session cron jobs, so this PR is not obsolete; however, the PR head has correctness gaps called out in review and overlaps the isolated-cron contract in #43404, so it needs changes before merge.

## What This Changes

The PR adds a reminder-sweep-like cron heuristic to the reminder guard and a regression test for suppressing the unscheduled-reminder note when a separate sweep cron exists.

## Best Possible Solution

Keep this PR open and update it so the sweep heuristic is narrowly tied to enabled isolated agentTurn reminder-sweep jobs, the regex works as intended, same-session behavior stays unchanged, and positive/negative tests document the final isolated-cron contract alongside #43404.
