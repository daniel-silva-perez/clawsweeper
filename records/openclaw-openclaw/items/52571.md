## Summary

Keep open. Current main still only suppresses the reminder guard for enabled jobs with the same session key, so the sweep-backed reminder behavior is not implemented on main; however, this PR head has blocking review defects in the new heuristic and overlaps other open reminder-guard PRs that need maintainer consolidation.

## What This Changes

The PR adds a reminder-sweep-like cron-job heuristic to the reminder guard and a runReplyAgent regression test so an existing sweep cron can suppress the unscheduled-reminder note.

## Best Possible Solution

Preserve the same-session cron suppression and add a narrow, documented path for enabled isolated agentTurn reminder-sweep jobs, with the regex fixed, the sessionTarget filter enforced, and positive plus negative regression coverage coordinated with the overlapping reminder-guard work.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use word-boundary escapes for sweep detection:** `src/auto-reply/reply/agent-runner-reminder-guard.ts:35`
  - body: The added regex contains literal U+0008 control bytes around the alternation instead of `\b` word-boundary escapes. Normal cron names/messages such as `family-reminder sweep` will not match, so the new suppression path remains inactive for the intended case.
  - confidence: 0.97
- **[P2] Limit sweep suppression to isolated cron jobs:** `src/auto-reply/reply/agent-runner-reminder-guard.ts:25`
  - body: The helper applies the reminder/sweep heuristic to every enabled `agentTurn` job. A main, custom, or other-session cron whose prompt mentions reminders or sweeps could suppress the warning for an unrelated reply, which is broader than the PR's stated contract.
  - confidence: 0.86
