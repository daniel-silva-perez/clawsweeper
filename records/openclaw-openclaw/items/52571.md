## Summary

Keep this PR open. Current main still only suppresses the unscheduled-reminder note for successful same-turn cron.add calls or enabled cron jobs with the same sessionKey, so the sweep-backed reminder false positive from #52528 remains. The PR is still a focused implementation candidate, but its head diff needs repair before merge: the regex contains raw backspace bytes, the sweep heuristic is not limited to isolated cron jobs, and the added test lacks a negative non-reminder isolated-cron case.

## Best Possible Solution

Keep this PR open and update it. The best path is to replace the raw backspace regex bytes with real word-boundary behavior or a simpler safe pattern, require enabled isolated agentTurn cron jobs for sweep-backed suppression, add a negative test for a non-reminder isolated agentTurn job, preserve the existing same-session behavior, and compare with related #43404 before landing so isolated-cron reminder guard behavior is not split across competing fixes.
