## Summary

Keep this PR open, but it should not merge as-is without maintainer review. Current main still has the shared adaptive-effort mapper that returns low for minimal/low when a reasoning-enabled Claude Opus 4.7 model reaches the Anthropic Messages transport, and the paired bug remains open; however, the later related Copilot report says the affected provider/model only accepts medium, so this model-only low/minimal clamp is probably the wrong boundary and incomplete for high/xhigh.

## What This Changes

The PR changes the Anthropic Messages transport to map Claude Opus 4.7 minimal/low adaptive thinking effort to medium and adds three transport tests for Opus 4.7 and Opus 4.6.

## Best Possible Solution

Revise the fix around a provider/model effort capability for GitHub Copilot Claude Opus 4.7, likely coercing all unsupported adaptive efforts to medium or omitting output_config for that provider/model, without changing direct Anthropic behavior. Add regression coverage for low, minimal, high/xhigh, and a direct Anthropic or non-Copilot guard; treat auth-profile cooldown from schema rejections as a separate follow-up.

## Reproduction Assessment

Yes for payload-level reproduction: a reasoning-enabled github-copilot/claude-opus-4.7 model with reasoning low reaches mapThinkingLevelToEffort and emits output_config.effort="low" on current main. The provided #70322 and #69928 reports supply live Copilot 400 logs, but I did not run a live Copilot request in this read-only review.

## Solution Assessment

No. The PR is a useful narrow candidate for the low/minimal symptom, but the best fix should be provider-aware and should cover the broader medium-only Copilot evidence instead of changing every Opus 4.7 transport by model id.
