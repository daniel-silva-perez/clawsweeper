## Summary

Keep this issue open. Current main still does not resolve a bare Feishu username/display name like `jaydenli` to an `open_id` for outbound message delivery, and the current error path still matches the reported `Unknown target` behavior. The related Feishu/cron/channel fixes in the provided context address target forwarding, registry bootstrapping, heartbeat routing, or missing channel support, but they do not supersede this bare-username target resolution gap.

## Best Possible Solution

Implement this in the Feishu plugin, not generic core: add a Feishu target resolver that can resolve explicit user-name forms such as `@name` or `user:<name>` through the Feishu live contact directory to a stable `open_id`/`user_id`, return a clear ambiguous/unknown result, and update the message-tool hint/docs/tests. If maintainers choose not to support name lookup, improve the error to explicitly say bare usernames are unsupported and point users to `user:open_id`, `chat:chat_id`, and the existing `channel-list`/member lookup path.
