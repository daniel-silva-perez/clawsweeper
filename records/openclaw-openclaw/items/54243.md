## Summary

Keep this issue open. Current main still makes `--no-include-workspace` remove workspace candidates from the backup plan, but the broad state asset remains included and the archive filter only excludes installed plugin `node_modules`, so default workspaces nested under `~/.openclaw` can still be archived.

## Best Possible Solution

Keep this issue open. The best fix is to carry resolved workspace paths into archive creation as explicit exclusions when `includeWorkspace` is false, including default/profile/custom/symlinked workspace layouts, and add a real archive regression test showing a file under `~/.openclaw/workspace` is absent. If maintainers instead choose the broader #40786 exclude-pattern implementation, it should explicitly cover this default nested-workspace semantic before this issue closes; otherwise the docs/help should be changed only if the intended contract is state-only backup rather than archive-level workspace exclusion.
