## Summary

Keep this PR open. Current main still routes Discord account IDs through explicit binding tiers and then the default agent; it does not implement the Discord accountId-to-existing-agent fallback proposed here. The PR remains a focused candidate for #39428, but the existing review blockers are real: the proposed patch can synthesize a non-existent agent when no agents are configured, and it labels an implicit naming-convention match as an explicit binding match.

## Best Possible Solution

Keep this PR open, or supersede it with a maintainer-owned fix that adds a narrow Discord-only fallback after explicit binding tiers: skip accountId "default", require the candidate to exist in agents.list, return a distinct match reason such as implicit.account, and add regression coverage for positive matching, default-account behavior, and the empty-agent-list guard. Keep #39428 as the canonical bug until that guarded behavior lands.
