## Summary

Keep this PR open. Current main still routes Discord account IDs only through explicit bindings and then the default agent; it has not implemented the requested Discord accountId-to-existing-agent fallback. The PR remains a focused implementation candidate for open #39428, but the reviewed blockers should be fixed before merge.

## Best Possible Solution

Keep this PR open, or supersede it with a maintainer-owned fix that adds a narrow Discord-only fallback after explicit binding tiers: skip `accountId: "default"`, require the candidate to exist in `agents.list`, use a distinct match reason such as `implicit.account`, and add regression coverage for positive matching, default-account behavior, and the empty-agent-list guard. Keep #39428 as the canonical bug until that guarded behavior lands.
