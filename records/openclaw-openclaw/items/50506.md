## Summary

Keep this PR open. Current main still does not implement the requested media-server hardening headers, and the PR remains a focused security-hardening candidate rather than obsolete. It needs rework for the current `createMediaRequestHandler` implementation and maintainer review on CSP/cache policy before merge.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase or rework the header hardening onto `createMediaRequestHandler`, add shared assertions for success and error responses, and make an explicit media-server CSP/cache policy decision that accounts for active-content attachment handling and one-shot URL fallback behavior.
