## Summary

Keep this PR open: current main does not yet implement explicit plugin trust tiers, and the proposed branch still has a correctness/security issue where omitted-format native plugins can be labeled as content-tier metadata.

## What This Changes

This PR adds plugin trust-tier types and resolution, stores the resolved tier on plugin records, prints it in plugin CLI output, and adds trust-tier docs and tests.

## Best Possible Solution

If maintainers want this taxonomy, land a rebased implementation that computes tiers from the normalized effective plugin format, defines the supported public CLI/status contract, documents current behavior separately from reserved future sandboxing, and covers native, bundle, missing-format fallback, list/info output, and relevant loader paths in tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Resolve trust tiers from the effective plugin format:** `src/plugins/loader.ts:849-853`
  - body: The loader records missing formats as `openclaw`, but this new call passes raw `manifestRecord.format` into `resolveTrustTier`. For manifests without an explicit format, the record can show `format: openclaw` with `trustTier: content`, under-reporting a native-capable plugin in CLI/security metadata. Use the same effective format at both loader call sites and update the undefined-format docs/tests to match.
  - confidence: 0.92
