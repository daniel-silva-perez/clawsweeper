## Summary

Keep open: current main still has no trust-tier implementation, and the PR branch adds security-facing metadata that can under-report omitted-format native plugins as content-tier. The branch needs maintainer product/security review plus a normalized-format fix before merge.

## What This Changes

This PR adds plugin trust-tier types and resolution, stores the tier on plugin records, shows it in plugin list/info output, and documents content, sandboxed, and native tiers.

## Best Possible Solution

Land a rebased version only if maintainers want this public taxonomy, with trust tiers computed from the same effective plugin format used in PluginRecord, docs that distinguish current capabilities from reserved future behavior, and tests covering native, bundle, omitted-format fallback, loader record paths, and CLI inspect/list output.
