## Summary

Keep open. Current main still lacks a trust-tier model, so the PR is not obsolete, but the proposed patch has a correctness/security issue: it resolves trust from the raw manifest format while the loader normalizes missing formats to native `openclaw`, which can show native-capable plugins as `content`.

## What This Changes

This PR adds plugin trust-tier types and resolution, stores `trustTier` on plugin records, prints it in plugin CLI output, and adds trust-tier docs and tests.

## Best Possible Solution

Adopt the trust-tier contract only after rebasing and making the recorded tier match the effective plugin format used by the loader, with tests for bundle, native, missing-format fallback, verbose list, info/inspect output, and docs that distinguish current behavior from reserved future sandboxing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Resolve trust tiers from the normalized plugin format:** `src/plugins/loader.ts:849-853`
  - body: The loader records missing formats as `openclaw` via `createPluginRecord`, but this new call passes raw `manifestRecord.format` into `resolveTrustTier`. For manifests without an explicit format, the resulting record can show `format: openclaw` with `trustTier: content`, under-reporting a native-capable plugin in CLI/security metadata. Use the same effective format at both loader call sites and align the undefined-format docs/tests with that behavior.
  - confidence: 0.92
