## Summary

Keep open for maintainer review. Current main still lacks an explicit plugin trust-tier model, CLI field, or trust-tier docs, so the PR is not obsolete; however, it changes operator-facing security metadata and the branch has a concrete correctness issue where missing-format plugins can be labeled as lower-trust than the native behavior they actually get.

## What This Changes

This PR adds plugin trust-tier metadata, stores it on plugin records, displays it in plugin CLI output, and adds docs/tests for content, sandboxed, and native tiers.

## Best Possible Solution

If maintainers want an explicit trust taxonomy, land a rebased version that computes the tier from the normalized effective plugin format, defines the supported public CLI/JSON/status contract, documents only available capabilities plus clearly reserved future behavior, and covers native, bundle, missing-format fallback, list/info output, and inspect/status behavior in tests.
