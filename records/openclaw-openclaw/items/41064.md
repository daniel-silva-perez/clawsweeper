## Summary

Keep this issue open. Current main still clears Dashboard chat run state on gateway hello, the hello snapshot schema/build path still has no active chat-run recovery field, and the UI derives Stop/Queue availability from `chatRunId`, so the reported tab-return/reconnect busy-state loss remains plausible. The issue is also paired with open, unmerged PR #41067, which uses closing syntax for this issue, so the maintainer path is to review, update, land, or close that PR rather than close this issue independently.

## Best Possible Solution

Keep this issue open and treat PR #41067 as the implementation candidate. The best fix should add an additive active chat-run snapshot to the Gateway hello contract, populate it from server active run state, reconcile matching UI `chatRunId`/busy state on reconnect or tab return, preserve Stop/Queue behavior, update generated protocol/client artifacts, and cover the focused Gateway/UI tests before maintainer review.
