## Summary

Keep this issue open. Current main still clears Dashboard chat run state after gateway hello, the hello snapshot has no active chat-run recovery field, and the composer derives Queue/Stop from `chatRunId`. The issue is also paired with open, unmerged PR #41067 using closing syntax, so the right maintainer path is to review, update, land, or close that PR rather than close this issue independently.

## Best Possible Solution

Keep this issue open and treat #41067 as the active implementation candidate. The best fix should add an additive active chat-run snapshot to Gateway hello, populate it from server active run state, restore matching UI `chatRunId`/busy state on reconnect or tab return, preserve Stop/Queue behavior, update protocol/client artifacts if generated outputs are affected, and cover the Gateway/UI tests before maintainer review.
