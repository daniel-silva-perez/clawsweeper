## Summary

Keep this issue open. Current main still clears the Dashboard chat run state on gateway hello, the hello snapshot does not expose active chat runs for recovery, and the chat controls ultimately depend on `chatRunId` for abortability. There is also an open same-author implementation PR, #41067, whose body uses closing syntax for this issue, so the right maintainer path is to review/land or close that PR rather than stale-close this issue.

## Best Possible Solution

Keep this issue open and treat #41067 as the implementation candidate. The best path is to update/review that PR or equivalent work on main: add an additive active-chat-run snapshot field, populate it from active abortable runs, recover the matching session's `chatRunId`/busy state on hello, make the composer preserve Stop/Queue when a run is still abortable, and cover the reconnect/tab-return path with targeted Gateway/UI tests.
