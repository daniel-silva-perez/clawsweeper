## What This Changes

The PR adds Gateway and SDK artifact list/get/download RPCs with cursor pagination and type filtering, generated Swift models, App SDK docs/tests, plus unrelated browser Hyprland and Windows executable-resolution changes.

## Best Possible Solution

Land the artifact API through the canonical maintainer path after fixing cursor semantics to identify a unique artifact position, preserving explicit provenance for run/task scoped queries, and separating unrelated browser or Windows fixes into their own PRs.

## Reproduction Assessment

Yes for the PR bug: create one transcript message with two artifact blocks, request artifacts.list with limit 1, then request the next page with the returned cursor. Because the cursor only carries messageSeq, the second artifact from the same message is skipped.

## Solution Assessment

No. The feature direction matches the open maintainer request, but this PR is not the best final form until pagination uses a stable tiebreaker such as messageSeq plus contentIndex or artifact id, and the unrelated Hyprland/Windows changes are split or reconciled with their owning surfaces.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve artifacts that share a message sequence:** `src/gateway/server-methods/artifacts.ts:388`
  - body: The cursor only encodes messageSeq. If one transcript message contains more artifacts than the page limit, nextCursor encodes that message sequence and the next request filters with messageSeq > cursorSeq, so remaining artifacts from the same message are never returned. Include a tiebreaker such as contentIndex or artifact id and add a page-split regression test.
  - confidence: 0.95
- **[P3] Align the Hyprland docs with the implemented capture path:** `docs/tools/browser-linux-troubleshooting.md:183-186`
  - body: This section says OpenClaw creates a virtual output and captures it with grim -o, but the implementation only inspects hyprctl clients and runs grim -g against the current window geometry. Either implement the documented virtual-output lifecycle or change the docs to describe the actual fallback path.
  - confidence: 0.88
