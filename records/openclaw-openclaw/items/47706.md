Full review comments:

- **[P2] Restore the process global after stubbing architecture:** `src/infra/platform-profile.test.ts:34`
  - body: `vi.stubGlobal("process", ...)` is not undone by `vi.restoreAllMocks()`, so later tests in the same worker can keep seeing the mocked ARM `process.arch`. Add `vi.unstubAllGlobals()` in teardown or avoid replacing the whole process object.
  - confidence: 0.93
- **[P2] Clamp the low-profile resize grid before the first pass:** `src/media/image-ops.ts:28`
  - body: The low-profile grid still starts with `sideStart`, so a 2000px input still performs the expensive 2000px encode before trying reduced sizes. Clamp the first low-profile candidate to the low ceiling, for example `Math.min(sideStart, 1200)`.
  - confidence: 0.82
- **[P3] Validate systemd memory-limit values before rendering:** `src/daemon/systemd-unit.ts:53-58`
  - body: The new `resourceLimits` strings are appended to the unit without the CR/LF checks used for description, args, env, and EnvironmentFile values. Keep these values as a closed set or validate them before rendering so future callers cannot inject extra unit lines.
  - confidence: 0.74
