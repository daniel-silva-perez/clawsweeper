## Summary

Keep this PR open. Current main still lacks the proposed cron chaining fields/runtime path and the smaller cron UI i18n fixes, but the PR is not ready as-is because its direct manual-run service path does not dispatch configured chains and the branch is stale against current test/protocol layout.

## What This Changes

This PR adds cron job-to-job chaining fields across the cron model, gateway schema, Swift protocol models, runtime dispatch, and Control UI, while also fixing cron UI i18n strings and cron UI test registration.

## Best Possible Solution

If maintainers still want cron-native chaining, land it as a rebased protocol/runtime/UI change that consistently handles scheduled, queued manual, and direct service manual runs, updates current generated clients and i18n artifacts, and documents how it relates to Task Flow. If Task Flow is the preferred product direction, supersede the chaining portion with a narrower Task Flow-oriented follow-up while keeping the small cron UI i18n fixes.
