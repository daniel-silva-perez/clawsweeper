## Summary

Keep this PR open. Current main still does not implement opt-in WhatsApp live commentary delivery: there is no WhatsApp `commentaryDelivery` config/schema field, no `onCommentaryReply` reply seam, WhatsApp dispatch only wires existing block/final delivery, and the embedded agent subscription still suppresses commentary-phase visible output. The related #57484 is closed unmerged, so this draft PR remains the active implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The best path is maintainer review of a rebased, tightly scoped PR that adds a generic `onCommentaryReply` seam, wires it only for WhatsApp behind `channels.whatsapp.commentaryDelivery = "live"`, preserves current default suppression, documents the config, updates SDK/config generated artifacts intentionally, and lands the targeted suppression, text-only normalization, timeout/abort, source-policy, and final-dedupe tests.
