## Summary

Keep open. Current main still lacks the WhatsApp live-commentary config and callback path, and this PR targets the right product boundary, but the proposed dedupe helper can resend already-delivered text for long commentary segments because it ignores the recorded delivered length once the stored text prefix is truncated.

## What This Changes

The PR adds opt-in WhatsApp live commentary delivery with channel/account config, a generic commentary callback seam, text-only WhatsApp delivery, final-reply dedupe, abort/timeout handling, tests, docs, changelog, and Plugin SDK runtime exports.

## Best Possible Solution

Keep the opt-in, WhatsApp-only, text-only design, but fix the long-segment dedupe math and add a focused regression test before maintainer review, CI/Testbox, and live WhatsApp validation decide whether to merge.

## Reproduction Assessment

Yes. The current-main gap is reproducible by source inspection, and the PR regression is reproducible at helper level with a delivered commentary segment longer than the stored 8,000-character prefix followed by an extended current text.

## Solution Assessment

No as-is. The proposed architecture is likely the right seam, but the delivered-commentary delta helper needs to honor the recorded delivered length when the stored prefix is truncated.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Honor delivered lengths for truncated commentary text:** `src/agents/pi-embedded-commentary.ts:44-46`
  - body: `rememberDeliveredCommentarySegment` stores only the first 8,000 characters of a delivered segment while also recording the full delivered length. When a later update extends that same segment, this check rejects the larger `deliveredTextLength` because it is greater than the stored prefix length, so `safeDeliveredLength` falls back to the truncated prefix length and `currentText.slice(...)` resends everything after 8,000 characters. Use the recorded delivered length when the current text still starts with the stored prefix, and add a long-commentary regression test.
  - confidence: 0.88
