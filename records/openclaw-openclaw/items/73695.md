## Summary

Keep this PR open. Current main does not implement the requested opt-in WhatsApp live commentary path: the WhatsApp config/schema/account model has no `commentaryDelivery` field, the reply option surface has no `onCommentaryReply` callback, WhatsApp inbound dispatch only wires block/final delivery, and the embedded agent path still suppresses commentary-phase text. The related earlier PR #57484 is closed unmerged, and the current PR is an open draft implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The best path is maintainer review of a rebased, tightly scoped version that adds the generic `onCommentaryReply` seam, wires it only for WhatsApp behind `commentaryDelivery: "live"`, preserves current default suppression, documents the config, and lands the targeted tests for source-policy suppression, text-only normalization, timeout/abort handling, and final dedupe.
