## Summary

Keep open. Current main has implemented the safer generated Linux systemd default (`KillMode=control-group`) and that partial fix is documented and tested, but the central doctor/status observability request remains missing: current systemd runtime inspection does not query `KillMode`, `TasksCurrent`, or `MemoryCurrent`, and the service audit does not warn on existing `KillMode=process` units or suspicious cgroup size.

## Best Possible Solution

Keep this issue open and implement the staged observability work: extend the Linux systemd service runtime/audit path to read `KillMode`, `TasksCurrent`, `MemoryCurrent`, and `MainPID`; surface those fields in `openclaw gateway status --deep` and the broader `openclaw status --deep`/doctor surfaces; warn when `KillMode=process` combines with suspicious task or memory counts; and offer a clear doctor repair path that rewrites existing managed units to the current maintainer-approved default.
