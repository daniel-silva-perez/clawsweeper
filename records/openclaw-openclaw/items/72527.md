## What This Changes

The issue requests downgrade-aware warnings and recovery for `openclaw.json` files written by OpenClaw 2026.4.x but later read by OpenClaw 2026.3.2, covering unknown keys, changed enum/value shapes, Discord streaming config, stale plugin entries, and JSON-safe repair output.

## Best Possible Solution

Keep this issue open until PR #72535 is reviewed, revised, landed, or replaced. The desired end state is a clear supported downgrade recovery path: newer-config detection before unsafe service/startup behavior, actionable doctor/manual guidance, and narrowly scoped repair only for known safe compatibility rewrites.

## Reproduction Assessment

Yes. The issue gives exact versions and failing config fields, and current source inspection confirms the relevant paths: valid configs get only a simple newer-version warning, invalid configs can fail before that warning, and doctor only generically strips unknown keys.

## Solution Assessment
