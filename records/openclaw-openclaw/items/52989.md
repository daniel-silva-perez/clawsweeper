## Summary

Keep open. Current main still uses the Windows-broken `new URL(process.argv[1] ?? "", "file://")` entrypoint guard in `scripts/generate-base-config-schema.ts`, and package scripts still invoke that file for config schema check/gen. The PR is focused on the implicated code path, is not implemented on main, and is not superseded by the related same-author `commands.shellProfile` PR #52154.

## Best Possible Solution

Keep this PR open for maintainer review. Either land the focused `pathToFileURL` fix with a small regression test/helper coverage, or apply the same change directly on main; do not close it as implemented or superseded because the current main branch and latest release still contain the old guard.
