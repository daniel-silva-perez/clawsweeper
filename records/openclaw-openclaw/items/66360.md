## Summary

Keep open. Current main still lacks a `session.maintenance` size or line cap for active transcript `.jsonl` files; the available `agents.defaults.compaction.maxActiveTranscriptBytes` path is a semantic compaction preflight guard, not the requested maintenance cleanup control, and it requires `truncateAfterCompaction`.

## What This Changes

The issue requests a `session.maintenance` mechanism to cap, rotate, or tail-trim individual active transcript `.jsonl` files so long-running chat sessions cannot grow without bound.

## Best Possible Solution

Add an explicit, documented transcript maintenance path for active `.jsonl` files, with disabled-by-default settings, `warn`/`enforce` behavior matching existing maintenance, dry-run visibility in `openclaw sessions cleanup`, and regression coverage for header preservation, tail retention, archive cleanup, and concurrency failure paths.
