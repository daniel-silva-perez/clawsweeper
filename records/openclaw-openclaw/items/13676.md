## Summary

Keep this issue open. Current main still lacks the requested first-class Project object and the matching Control UI, Gateway protocol, session binding, workspace list, project-scoped skill/tool policy, and runtime-constraint surfaces. Existing sessions, agents, per-agent workspaces, skill allowlists, tool policy, sandbox settings, and sub-agent workstreams are useful adjacent primitives, but they do not satisfy the durable Project model described here. The discussion also adds a useful port/service registry idea, and the recent maintainer/Codex comment explicitly kept this open while treating #12333 and #13535 as related narrower items rather than replacements.

## Best Possible Solution

Continue tracking this as the canonical core/product dashboard request. A complete solution should design and add explicit Project storage and Gateway protocol methods, expose a Projects list/route in Control UI, bind projects to a home session plus optional workstreams, associate one or more workspace paths, and surface project-scoped tool/skill/runtime constraints while reusing existing agent, session, workspace, tool-policy, sandbox, and sub-agent primitives where they fit. The port/service registry suggestion from the discussion should be considered as a follow-up or part of the project detail panel.
