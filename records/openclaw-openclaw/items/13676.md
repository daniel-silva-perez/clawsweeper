## Summary

Keep this issue open. Current main still lacks the requested first-class Project object, Projects dashboard surface, Gateway `projects.*` protocol family, project-to-home-session/workstream binding, multi-workspace project model, and project-scoped tool/skill/runtime policy. Existing agents, sessions, workspaces, skill allowlists, sandbox settings, and sub-agent metadata are adjacent primitives, but they do not satisfy the durable Project model requested here. The discussion also remains useful: davstr1 added a project-scoped port/service registry idea, and a recent maintainer/Codex comment explicitly kept this open while treating #12333 and #13535 as related narrower items rather than replacements.

## Best Possible Solution

Keep this as the canonical core/product dashboard request. A good solution should design explicit Project storage and additive Gateway protocol methods, add a Projects route/list/detail surface in Control UI, bind projects to a home session plus optional workstreams, associate one or more workspace paths and docs links, and surface project-scoped tool/skill/runtime constraints while reusing existing agents, sessions, workspaces, skill allowlists, sandbox settings, and sub-agent metadata where they fit. The port/service registry idea from the discussion should be considered during project-detail design or split into a focused follow-up once the Project object exists.
