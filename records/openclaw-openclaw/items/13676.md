## Summary

Keep open. Current main still has adjacent agents, sessions, workspace paths, skill allowlists, and sandbox/tool policy, but it does not implement the requested first-class Project object, Projects dashboard surface, or Gateway project API. The recent issue discussion also explicitly keeps this as the canonical core/product dashboard request rather than treating narrower closed issues as replacements.

## What This Changes

Add first-class Projects to the Gateway Dashboard with durable project records, project-to-session/workspace bindings, scoped skills/tools, and runtime constraint visibility or enforcement.

## Best Possible Solution

Keep this as the canonical core/product request until OpenClaw has an explicit Project domain model, additive Gateway project protocol, Control UI list/detail/switching surface, project-to-home-session and workstream bindings, workspace/doc links, and project-scoped tool/skill/runtime constraint display or warning behavior that reuses existing agents, sessions, workspaces, skill allowlists, sandbox settings, and sub-agent metadata where appropriate.
