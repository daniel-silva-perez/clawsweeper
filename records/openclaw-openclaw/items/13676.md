## Summary

Keep open. Current main still has only adjacent sessions, agents, workspaces, skill allowlists, and sandbox/tool policy; it does not provide the requested first-class Project domain model, Projects dashboard surface, project-to-session/workspace bindings, or Gateway project API. The recent discussion also treats this as the canonical broader product request rather than a duplicate of narrower closed project/session items.

## What This Changes

Add first-class Projects to the Gateway dashboard with project/session/workspace bindings, scoped skills/tools, runtime constraints, and fast project switching.

## Best Possible Solution

Keep this as the canonical core/product request until OpenClaw has an explicit Project storage model, additive Gateway projects.* protocol methods, a Control UI Projects list/detail/switching surface, project-to-home-session and workstream bindings, workspace/doc links, and clearly labeled project-scoped tool/skill/runtime constraint display or enforcement built on the existing agents, sessions, workspaces, skill allowlists, sandbox settings, and sub-agent metadata.
