## Summary

Keep open. Current main has adjacent agents, sessions, workspaces, skill allowlists, and sandbox/tool policy, but it still lacks the requested first-class Project model, Projects dashboard surface, and Gateway project API. The recent issue discussion also treats this as the canonical broader product request rather than a duplicate of narrower closed project/session reports.

## What This Changes

The issue requests first-class Projects in the Gateway Dashboard with project/session/workspace bindings plus project-scoped skills/tools and runtime constraints.

## Best Possible Solution

Keep this as the canonical core/product request until OpenClaw has an explicit Project domain model, additive Gateway project protocol, Control UI project list/detail/switching surface, project-to-home-session and workstream bindings, workspace/doc links, and project-scoped tool/skill/runtime constraint display or warning behavior that reuses existing agents, sessions, workspaces, skill allowlists, sandbox settings, and subagent metadata where appropriate.
