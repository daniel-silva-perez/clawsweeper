## Summary

Keep open. Current main still lacks the requested first-class Project model, Projects dashboard surface, project-to-session/workspace bindings, and Gateway project API; the existing sessions, agents, workspace paths, skill allowlists, and sandbox policy are adjacent building blocks but do not satisfy the central request. The issue discussion also keeps this as the canonical broader product request rather than a duplicate of narrower closed project/session issues.

## What This Changes

The issue requests a first-class Project concept in the Gateway dashboard with persistent project records, home/workstream session bindings, workspace links, project-scoped skills/tools, and runtime constraints.

## Best Possible Solution

Keep this as the canonical core/product request until OpenClaw has an explicit Project domain model, additive Gateway projects.* protocol methods, a Control UI Projects list/detail/switching surface, project-to-home-session and workstream bindings, workspace/doc links, and project-scoped tool/skill/runtime constraint display or warning behavior that reuses existing agents, sessions, workspaces, skill allowlists, sandbox settings, and subagent metadata where appropriate.
