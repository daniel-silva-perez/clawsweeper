## Summary

Keep open. Current main still lacks the requested first-class Project domain model, Projects dashboard surface, project-to-session/workspace bindings, and Gateway project API. Existing sessions, agents, workspaces, skill allowlists, and sandbox/tool policy are useful building blocks, but they do not satisfy the central product request, and the discussion already treats this as the canonical broader issue rather than a duplicate of narrower closed items.

## What This Changes

The issue requests a first-class Project concept in the Gateway Dashboard, including project/session/workspace bindings plus project-scoped skill/tool and runtime constraints.

## Best Possible Solution

Keep this as the canonical core/product request until OpenClaw has an explicit Project storage model, additive Gateway projects.* protocol methods, a Control UI Projects list/detail/switching surface, project-to-home-session and workstream bindings, workspace/doc links, and clearly labeled project-scoped tool/skill/runtime constraint display or enforcement built on existing agents, sessions, workspaces, skill allowlists, sandbox settings, and sub-agent metadata.

Status: needs_attention

Summary: No patch is under review, but the requested project-scoped runtime and tool/skill constraints are security-sensitive and need explicit semantics.
