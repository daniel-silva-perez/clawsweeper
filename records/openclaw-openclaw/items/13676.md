## Summary

Keep open. Current main still lacks the requested first-class Project model, Projects dashboard surface, and Gateway project API; the existing agents, sessions, workspace paths, skill allowlists, and sandbox/tool policy are useful building blocks but do not satisfy the central request. The recent discussion also treats this as the canonical core/product dashboard request rather than a duplicate of the narrower closed project-session issues.

## What This Changes

The issue requests a first-class Project concept in the Gateway/Control UI dashboard with project-to-session, workspace, skill/tool, and runtime constraint scoping.

## Best Possible Solution

Keep this as the canonical core/product request until OpenClaw has an explicit Project domain model, additive Gateway project protocol, Control UI project list/detail/switching surface, project-to-home-session and workstream bindings, workspace/doc links, and project-scoped tool/skill/runtime constraint display or warning behavior that reuses the existing agents, sessions, workspaces, skill allowlists, sandbox settings, and subagent metadata where appropriate.
