## Summary

Keep this issue open. Current main still has no first-class Project object, Projects dashboard surface, projects.* Gateway protocol, project-to-session/home-session binding, multi-workspace project model, or project-scoped tool/skill/runtime-constraint policy. Existing agents, sessions, per-agent workspaces, skill allowlists, tool policies, sandbox settings, and sub-agent workstreams are useful adjacent primitives, but they do not satisfy the durable Project model requested here. The recent discussion also explicitly treats #12333 and #13535 as related narrower items rather than replacements, and adds a useful port/service registry idea for future design.

## Best Possible Solution

Continue tracking this as the canonical core/product dashboard request. A complete solution should design explicit Project storage and Gateway protocol methods, expose a Projects list/route in Control UI, bind projects to a home session plus optional workstreams, associate one or more workspace paths and docs links, and surface project-scoped tool/skill/runtime constraints while reusing existing agent, session, workspace, skill, tool-policy, sandbox, and sub-agent primitives where they fit. The port/service registry idea from the discussion should be considered during the project detail design or as a focused follow-up.
