## Summary

Keep open. Current main still has sessions, agents, workspaces, skill allowlists, and sandbox/tool policy, but not the requested first-class Project model, Projects dashboard surface, project-session/workspace bindings, or Gateway project API. The recent issue discussion also keeps this as the canonical broader product request rather than closing it in favor of narrower closed project/session reports.

## What This Changes

Add a first-class Project concept to the Gateway dashboard with durable project records, home-session and workspace bindings, project-scoped skills/tools, and runtime constraint display or enforcement.

## Best Possible Solution

Keep this as the canonical core/product request until OpenClaw has an explicit Project domain model, additive Gateway project protocol methods, a Control UI project list/detail/switching surface, project-to-home-session and workstream bindings, workspace/doc links, and project-scoped tool/skill/runtime constraint display or warn-only behavior that reuses the existing agents, sessions, workspaces, skill allowlists, sandbox settings, and subagent metadata where appropriate.
