## Summary

Keep open. Current main still has the cron-session filter in the desktop chat controls, but the mobile settings dropdown only renders the session selector plus thinking, tool calls, and focus buttons. The PR is a focused Control UI parity change against one UI helper file, and the provided discussion supports that it is intended to port existing desktop behavior rather than introduce a new surface.

## Best Possible Solution

Keep this PR open for maintainer review. The likely product path is to add the cron-session toggle to `renderChatMobileToggle`, reusing the existing desktop cron filter state/icon/labels and aligning any accessibility attributes with current Control UI conventions, then add focused render coverage for the mobile dropdown.
