## Summary

Current main still leaves the Feishu P2P direct-message thread reply regression open. Inbound parsing preserves `root_id` and `thread_id`, but reply routing remains group-gated: non-group messages force `replyInThread` and `threadReply` false, choose the non-thread DM target path, and strip reply metadata before dispatch. The issue is also already paired with open fixing PRs #73972 and #74001, so it should stay open until one canonical fix merges.

## Best Possible Solution

Keep this issue open and resolve it by reviewing and landing one canonical Feishu plugin patch from #73972, #74001, or an equivalent narrow fix. The fix should detect P2P direct-message thread events from `thread_id`, target the thread root, set `replyInThread` and `threadReply`, preserve reply metadata for that P2P thread path, keep root-id-only P2P quote replies and group behavior unchanged, add focused regression tests, run `pnpm check:changed`, then close this issue as implemented and supersede the overlapping PR.
