## Summary

Keep open. Current main still parses Feishu `root_id`/`thread_id`, but direct-message reply routing remains group-gated: P2P replies do not enable `replyInThread`, do not target the thread root, and strip reply metadata before dispatch. The issue is also paired with open closing PRs #73972 and #74001, so it should stay open until one canonical fix merges.

## Best Possible Solution

Keep this issue open and resolve it by reviewing and landing #73972 or an equivalent narrow Feishu patch. The fix should detect P2P direct-message thread events from `thread_id`, preserve plain root_id-only P2P quote replies, set `replyInThread`/`threadReply`, preserve reply metadata for P2P threads, keep group behavior unchanged, add focused regression tests, and then supersede the overlapping open PR.
