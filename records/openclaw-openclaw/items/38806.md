## Summary

Current main still leaves the reported Feishu P2P thread-reply regression open. Feishu inbound parsing preserves `root_id` and `thread_id`, but reply routing remains group-gated: non-group messages force `replyInThread` and `threadReply` false, choose the non-thread DM target path, and strip reply metadata before dispatch. The issue also has open fixing PRs #73972 and #74001, so it should stay open until one canonical fix merges.

## Best Possible Solution

Keep this issue open and resolve it by reviewing and landing one canonical Feishu patch, preferably the implementation that detects P2P direct-message thread events from `thread_id`, targets the thread root, sets `replyInThread`/`threadReply`, preserves reply metadata for that P2P thread path, keeps root-id-only P2P quote replies and group behavior unchanged, adds focused regression tests, and supersedes the overlapping open PR.
