## Summary

Keep this PR open. Current main still sends regular Telegram videos through both outbound paths without probing or forwarding `width`/`height`, and the submitted diff is not merge-ready because it introduces bare `ffprobe` execution, crosses the bundled-plugin boundary, changes a shared media limit unrelated to the stated fix, and lacks targeted regression coverage.

## Best Possible Solution

Keep the PR open and request a focused revision: implement video-dimension probing through `openclaw/plugin-sdk/media-runtime` using the trusted `runFfprobe`/resolver path, pass `width` and `height` only to regular Telegram `sendVideo` in `extensions/telegram/src/send.ts` and `extensions/telegram/src/bot/delivery.replies.ts`, leave GIF animations and video notes untouched, split or remove the unrelated `MAX_VIDEO_BYTES` change, and add targeted regression tests for both outbound paths plus exclusions.
