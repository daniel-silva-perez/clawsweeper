## Summary

Keep this PR open. Current main at 189c91eae6e1f4f280179961bca2c4153cf42e86 still does not pass video width/height to Telegram sendVideo in either outbound path, and the PR addresses a real focused Telegram bug. The submitted patch still needs revision for plugin-boundary compliance, trusted ffprobe execution, unrelated media-limit churn, and regression tests.

## Best Possible Solution

Keep this PR open and request a focused revision: use a plugin-safe media helper through openclaw/plugin-sdk/media-runtime or a Telegram-owned runtime barrel, rely on the trusted runFfprobe/resolveSystemBin path, pass width and height only for regular sendVideo calls in extensions/telegram/src/send.ts and extensions/telegram/src/bot/delivery.replies.ts, keep GIF animations and video notes excluded, add regression tests for both outbound paths, and split or remove the unrelated MAX_VIDEO_BYTES change.
