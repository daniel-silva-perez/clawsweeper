## Summary

Keep this PR open. Current main still does not implement the intended Telegram fix: both outbound Telegram video paths call `sendVideo` without probing or passing `width`/`height`. The contribution addresses a plausible current bug, but the submitted patch is not merge-ready because it crosses the bundled-plugin boundary, adds bare `ffprobe` execution instead of the trusted media-runtime resolver path, changes a shared video-size constant outside the stated fix, and lacks targeted regression coverage.

## Best Possible Solution

Keep this PR open and request a focused revision: implement or expose video-dimension probing through `openclaw/plugin-sdk/media-runtime` using the trusted `runFfprobe`/resolver path, pass `width` and `height` only for regular Telegram `sendVideo` in `extensions/telegram/src/send.ts` and `extensions/telegram/src/bot/delivery.replies.ts`, leave GIF animations and video notes untouched, add targeted regression tests for both outbound paths, and remove the unrelated shared `MAX_VIDEO_BYTES` change.
