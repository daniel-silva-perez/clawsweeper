## Summary

Keep this PR open. Current main still does not pass `width`/`height` to Telegram `sendVideo` in either outbound path, so the intended bug fix is not implemented. The direction is valid, but the submitted diff still needs revision for the plugin boundary, trusted `ffprobe` execution, unrelated media-limit change, and regression coverage.

## Best Possible Solution

Keep this PR open and request a focused revision: add or expose a video-dimension helper through `openclaw/plugin-sdk/media-runtime` using the existing trusted `runFfprobe`/`resolveSystemBin` path, import it through the Telegram runtime/local barrel, pass `width` and `height` only for regular `sendVideo` calls in `extensions/telegram/src/send.ts` and `extensions/telegram/src/bot/delivery.replies.ts`, keep GIF animations and video notes excluded, add regression tests for both outbound paths, and remove or split the unrelated `MAX_VIDEO_BYTES` change.
