## Summary

Keep this PR open. Current main still sends regular Telegram videos without probing or passing width/height on either outbound path, and the provided maintainer discussion already identifies this PR as a useful implementation candidate. The submitted patch should not be merged as-is because it crosses the plugin boundary, runs a bare ffprobe binary, and bundles an unrelated global media-limit change.

## Best Possible Solution

Keep this PR open and request a focused revision: add or expose a plugin-SDK media helper that probes video dimensions through the trusted ffprobe resolver path, pass width and height only to regular Telegram `sendVideo` in `extensions/telegram/src/send.ts` and `extensions/telegram/src/bot/delivery.replies.ts`, leave GIF/animation and video-note paths untouched, remove or split the global media-limit change, and add targeted regression coverage for both outbound paths and exclusions.
