## Summary

Keep this PR open. Current main still does not probe Telegram video dimensions or pass width/height on either regular sendVideo path, and the PR remains a useful implementation candidate, but the submitted patch needs revision for the plugin boundary, trusted ffprobe execution, unrelated media-limit change, and targeted regression coverage.

## Best Possible Solution

Keep this PR open and request a focused revision: implement or add a plugin-sdk media-runtime helper that probes video dimensions through the trusted ffprobe resolver path, pass width and height only to regular Telegram sendVideo in extensions/telegram/src/send.ts and extensions/telegram/src/bot/delivery.replies.ts, leave sendAnimation/GIF and sendVideoNote paths untouched, split or remove the unrelated MAX_VIDEO_BYTES change, and add targeted regression tests for both outbound paths plus exclusions.
