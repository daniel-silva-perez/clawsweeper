## Summary

Keep open. Current main still lets the run-side typing TTL expire during a quiet active run because keepalive ticks call the typing callback without refreshing the controller TTL. The PR is a narrow, relevant fix with no concrete code-review finding, but it still needs maintainer review and exact-head CI reconciliation before merge.

## What This Changes

The PR refreshes the shared typing controller TTL from the active keepalive tick and adds a regression test for long runs that continue past the original TTL window.

## Best Possible Solution

Land this narrow shared typing TTL-renewal with its regression test after maintainer review and green exact-head CI, while leaving the broader active-run presence lease work tracked in #65919.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to create a typing controller with a 6s keepalive interval and 10s TTL, start the loop, advance fake timers to the first keepalive tick, then advance past 10s total; current main cleans up because the keepalive tick does not refresh the TTL.

## Solution Assessment

Yes. For this specific Discord typing-liveness bug, renewing the controller TTL inside the existing guarded keepalive tick is the narrowest maintainable fix; broader cross-channel presence leasing belongs in #65919.
