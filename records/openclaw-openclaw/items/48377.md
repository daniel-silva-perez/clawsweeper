## Summary

Keep this PR open. Current main still does not implement create-with-content for `feishu_doc`: the create schema has no `content` field, and the runtime create branch only creates the document without calling the existing markdown write path. The PR remains a focused implementation candidate for the still-open linked bug, but it needs maintainer review around tests, empty-content semantics, response shape, and whether to reuse `writeDoc` or a populate helper.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate for the linked Feishu bug. The maintainer-quality path is to add a create-with-content regression test, decide explicit empty-content behavior, avoid or intentionally accept the extra clear call, return a clean create response, and align the Feishu doc skill documentation with the final supported create schema.

## Risks / Open Questions

- The PR still needs focused regression coverage for `feishu_doc` create with `content` before it is merge-quality.
- The provided implementation uses a truthiness check, so maintainers should decide whether `content: ""` or whitespace-only content is considered provided and should be handled explicitly.
- Reusing `writeDoc` is functionally plausible but clears a newly created document first; a create-populate or append-style helper could avoid extra Feishu API calls and response fields such as `blocks_deleted`.
- If create-with-content becomes supported, the Feishu doc skill docs should be updated so examples and permission guidance match the implemented schema and response behavior.
