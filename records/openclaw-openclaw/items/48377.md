## Summary

Keep this PR open. Current `main` still does not implement `feishu_doc` create-with-content behavior: the create schema has no `content` field, the create dispatcher returns `createDoc(...)` directly, and the existing write path remains separate. The PR is still the active implementation candidate for the linked open bug #48373, and the later branch repair in the provided context addresses the earlier empty-content and misleading `blocks_deleted` review concerns.

## Best Possible Solution

Keep this PR open for maintainer review or landing as the canonical implementation candidate for #48373. The best path is the branch direction shown in the provided context: add optional `content` to the Feishu create schema, write the created document when the property is present, preserve explicit empty-content behavior, avoid returning misleading `blocks_deleted` from create, and keep the focused tests/docs/changelog credit.
