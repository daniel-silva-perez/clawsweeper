## Summary

Keep this PR open. Current main still does not implement the Feishu `feishu_doc` create-with-content behavior: the create schema has no `content` field and the create branch only calls `createDoc`. The PR remains the active implementation candidate for the linked open bug, and the current diff is scoped to Feishu plugin behavior, tests, docs, and changelog.

## Best Possible Solution

Keep this PR open and review or land it as the canonical implementation candidate for #48373. The best path is the current branch direction: add optional create `content` schema support, populate the created document when the property is present, preserve explicit empty-content behavior, avoid misleading `blocks_deleted` in create responses, and keep the focused Feishu regression tests/docs/changelog credit.
