## Summary

Keep this PR open. Current main still does not implement create-with-content for `feishu_doc`: the create schema has no `content` field, and the runtime create branch only calls `createDoc`. The PR is still a focused implementation candidate for the linked open Feishu bug, but it needs maintainer review around tests, empty-content semantics, docs, response shape, and whether `writeDoc` is the right helper for a freshly-created document.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate for #48373. The maintainer-quality path is to add `content` to the create schema, populate the new document through a tested helper, define empty-content behavior, return a clean create response, avoid unnecessary Feishu API calls if practical, and update the Feishu doc skill documentation to match the final schema and permission behavior.

## Risks / Open Questions

- The PR still needs a focused regression test proving `feishu_doc` create with `content` creates the document and inserts converted markdown blocks in one tool call.
- The proposed truthiness check for `p.content` would ignore an explicitly provided empty string; maintainers should define whether empty or whitespace-only content is valid, rejected, or equivalent to omitted content.
- Reusing `writeDoc` is plausible but it clears/list-deletes content before writing; on a freshly-created document this may add unnecessary Feishu API calls, and the final response shape should stay clean.
- The Feishu doc skill documentation is stale around create parameters and requester permissions, so the final behavior should update the skill docs when the schema is settled.
