Keep this issue open. Current main still defines the collapsed Control UI rail at 78px and lacks a content-vs-rail bounds regression test, while open PR #55861 is an unmerged implementation candidate from the same reporter and explicitly targets this issue.

Keep this issue open and treat PR #55861, or a refreshed successor, as the implementation path. The best fix is to verify the Safari collapsed-sidebar overlap, apply the layout adjustment in the correct desktop scope, add a browser regression that waits for the layout transition and asserts content starts to the right of the rail, then close the issue after the fix lands on main.
