## Summary

Keep this PR open. Current main still targets the Feishu typing reaction at the reply target, while topic-mode and configured thread replies intentionally make that reply target the root message; the related open thread-routing PR explicitly leaves this typing-target fix out of scope.

## What This Changes

This PR adds an optional Feishu inbound message target, passes `ctx.messageId` from inbound dispatch into the reply dispatcher, uses it for typing-reaction targeting with `replyToMessageId` fallback, and adds focused Feishu tests.

## Best Possible Solution

Feishu should preserve existing reply/thread routing to the topic root while targeting the typing reaction at the actual inbound message, with a safe fallback for DM and non-topic flows plus regression tests for both paths.
