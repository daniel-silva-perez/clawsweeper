## Summary

Keep this PR open. Current main still has the Feishu typing-reaction target bug, but the proposed patch has an actionable edge-case regression: when an inbound path intentionally suppresses reply targeting, the new fallback can still try to add a typing reaction to the inbound id, including synthetic card-action ids.

## What This Changes

This PR adds an optional Feishu inbound message id to the reply dispatcher, passes `ctx.messageId` from Feishu inbound dispatch, targets typing reactions at that inbound id with `replyToMessageId` fallback, and adds focused tests.

## Best Possible Solution

Revise the patch so Feishu keeps replies anchored to the topic/root while typing reactions target the real inbound child message only when there is a valid message-reaction target; preserve no-target suppression for synthetic or suppressed flows, then land with focused regression tests and the required user-facing changelog entry if maintainers require one.

## Reproduction Assessment

Yes. The reported bug reproduces from source by combining topic/replyInThread routing, where `replyTargetMessageId` becomes `ctx.rootId`, with the dispatcher typing path that reacts to `replyToMessageId`; the PR regression can be covered by a dispatcher test with `inboundMessageId` present and `replyToMessageId` absent.

## Solution Assessment

No, not as written. The main fix direction is narrow and appropriate, but the fallback should not bypass existing no-reply-target suppression for synthetic card-action or other suppressed inbound paths.

## Review Findings
