## Summary

Keep open. Current main does not implement per-agent `env.scope` isolation: agent config types/schema do not accept `agents.list[].env`, agent resolution does not carry an env block, and host exec still builds its base environment from `process.env`. The PR is a plausible core security change, not obsolete, but the proposed diff needs revision for the unresolved review findings and regression tests before it could be merged or superseded.

## Best Possible Solution

Keep the PR open or replace it with a maintained core implementation. The best path is to add the `AgentConfig` env contract and strict schema, resolve it through the canonical agent config helpers, use `collectConfigRuntimeEnvVars`, wire an explicit scoped base env into gateway/node exec without regressing sandbox behavior, update docs/schema/redaction surfaces if agent vars can carry secrets, and add regression tests for the collaborator-security cases and the bot-review edge cases.
