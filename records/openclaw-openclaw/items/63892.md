## Summary

Keep open: current `main` still returns from the fresh persisted-token preflight path before evaluating token-threshold compaction, and #64384 is an open closing PR from the reporter for this bug. #66716 also overlaps the same fresh-token compaction path, so this needs maintainer selection, landing, or superseding rather than cleanup closure or a new repair job.

## What This Changes

The issue reports that proactive preflight compaction stops re-firing after token totals become fresh, causing long sessions to rely on overflow-retry compaction instead of the configured reserve threshold.

## Best Possible Solution

Land or replace a focused preflight fix so fresh persisted totals at or above `contextWindow - reserveTokensFloor - softThresholdTokens` reach `shouldRunPreflightCompaction`, while fresh below-threshold totals still skip expensive transcript estimation and transcript-byte compaction remains independent. Add a regression test for `totalTokensFresh: true`, cached tokens above threshold, and no transcript-byte trigger.
