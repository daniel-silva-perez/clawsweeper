## Summary

Keep open: current main still returns from the fresh-token preflight path before evaluating token-threshold compaction, and an open same-author PR uses closing syntax for this issue, so this should remain open until a fix is merged or explicitly superseded.

## What This Changes

The issue reports that proactive preflight compaction stops re-firing after a checkpoint because fresh cached token totals short-circuit before the token threshold is reevaluated.

## Best Possible Solution

Land or replace the focused preflight fix so fresh persisted totals at or above `contextWindow - reserveTokensFloor - softThresholdTokens` reach `shouldRunPreflightCompaction`, while fresh below-threshold totals still skip expensive transcript estimation and transcript-byte compaction remains independent. Add a regression test for `totalTokensFresh: true`, cached tokens above threshold, and no transcript-byte trigger.
