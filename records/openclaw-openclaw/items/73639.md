## Summary

Current main still has the reported gap. Non-interactive onboarding only accepts `token` and `password` gateway auth, while trusted-proxy is supported by the gateway config/runtime, documented for manual configuration, and available in interactive gateway setup. The same-author draft PR #49107 remains the active implementation candidate, and duplicate-looking #73638 is still open, so this issue should stay open for maintainer triage rather than cleanup closure.

## Best Possible Solution

Keep this issue open. Maintainers should review, land, or replace PR #49107 with a focused implementation that adds non-interactive onboarding support for `gateway.auth.mode="trusted-proxy"`, `gateway.trustedProxies`, `gateway.auth.trustedProxy.userHeader`, `requiredHeaders`, `allowUsers`, an explicit `allowLoopback` decision, and any needed `gateway.controlUi.allowedOrigins` automation, with targeted tests and docs aligned to the trusted-proxy auth guide. If #73638 is intended to be canonical, dedupe that pair deliberately after deciding how to handle the open PR.
