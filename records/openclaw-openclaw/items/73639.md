## Summary

Keep this issue open. Current main still rejects `--gateway-auth trusted-proxy` in non-interactive onboarding, and the same-author draft PR #49107 is still the active implementation candidate for this exact gap rather than a shipped fix.

## Best Possible Solution

Keep this issue open and review, land, or replace PR #49107 with a focused implementation that adds non-interactive onboard flags for `gateway.auth.mode="trusted-proxy"`, `gateway.trustedProxies`, `gateway.auth.trustedProxy.userHeader`, `requiredHeaders`, `allowUsers`, an explicit `allowLoopback` decision, and any needed Control UI allowed-origin automation, with targeted tests and docs aligned to the trusted-proxy auth guide.
