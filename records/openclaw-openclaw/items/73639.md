## Summary

Keep this issue open. Current main still rejects `--gateway-auth trusted-proxy` in non-interactive onboarding even though trusted-proxy auth is a supported gateway runtime and interactive configuration mode. The same-author draft PR #49107 is still open and unmerged, so cleanup should not close this tracking issue before that implementation is reviewed, landed, or replaced.

## Best Possible Solution

Keep this issue open and review, land, or replace PR #49107 with a focused implementation that adds non-interactive onboarding support for `gateway.auth.mode="trusted-proxy"`, `gateway.trustedProxies`, `gateway.auth.trustedProxy.userHeader`, `requiredHeaders`, `allowUsers`, an explicit `allowLoopback` decision, Control UI allowed-origin automation where needed, and targeted tests/docs aligned with https://docs.openclaw.ai/gateway/trusted-proxy-auth.
