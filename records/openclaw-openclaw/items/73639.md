## Summary

Keep this issue open. Current main still rejects `--gateway-auth trusted-proxy` in non-interactive onboarding, while trusted-proxy is already a supported gateway auth mode elsewhere. The same-author draft PR #49107 remains the active implementation candidate, so this issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review, land, or replace PR #49107 with a focused implementation that adds non-interactive onboarding support for `gateway.auth.mode="trusted-proxy"`, `gateway.trustedProxies`, `gateway.auth.trustedProxy.userHeader`, `requiredHeaders`, `allowUsers`, an explicit `allowLoopback` decision, and any needed Control UI allowed-origin automation, with targeted tests and docs aligned to the trusted-proxy auth guide.
