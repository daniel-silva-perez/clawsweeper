## Summary

Current main still has the reported gap: non-interactive onboarding rejects `--gateway-auth trusted-proxy`, while trusted-proxy is supported by gateway runtime/config and by the interactive gateway flow. The same-author draft PR #49107 remains open as the implementation candidate, and duplicate-looking #73638 is still open, so this should stay open for maintainer triage rather than cleanup closure.

## Best Possible Solution

Keep this issue open. Maintainers should review, land, or replace PR #49107 with a focused core onboarding implementation that accepts `--gateway-auth trusted-proxy`, adds explicit non-interactive flags for `gateway.trustedProxies` and `gateway.auth.trustedProxy.*`, handles Control UI allowed origins where needed, updates `docs.openclaw.ai` onboarding guidance, and adds targeted tests for valid writes plus security-field failures. If #73638 is intended as the canonical duplicate, dedupe the pair deliberately after deciding the PR path.
