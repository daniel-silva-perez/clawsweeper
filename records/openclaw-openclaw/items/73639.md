## Summary

Keep this issue open. Current main still rejects `--gateway-auth trusted-proxy` in non-interactive onboarding, even though trusted-proxy is supported in the gateway config/runtime and interactive onboarding. The same-author draft PR #49107 remains the active implementation candidate, and the exact duplicate-looking #73638 is also still open, so this needs maintainer triage or a landed fix rather than cleanup closure.

## Best Possible Solution

Keep this issue open. Maintainers should review, land, or replace PR #49107 with a focused implementation that adds non-interactive onboarding support for `gateway.auth.mode="trusted-proxy"`, `gateway.trustedProxies`, `gateway.auth.trustedProxy.userHeader`, `requiredHeaders`, `allowUsers`, an explicit `allowLoopback` decision, and any needed Control UI allowed-origin automation, with targeted tests and docs aligned to the trusted-proxy auth guide. If #73638 is intended to be canonical, dedupe that pair deliberately rather than auto-closing this while the linked PR remains open.
