## Summary

Keep this issue open. Current main still rejects `--gateway-auth trusted-proxy` in non-interactive onboarding, while trusted-proxy remains a supported runtime and interactive gateway auth mode. The same-author draft PR #49107 is still the active implementation candidate, and the same-author sibling issue #73638 is also open, so conservative cleanup should not close this tracking item before maintainer review/dedupe or a merged fix.

## Best Possible Solution

Keep this open until maintainers review and either land, replace, or close PR #49107, then dedupe this issue with #73638 as appropriate. The product fix should add focused non-interactive onboard support for `gateway.auth.mode="trusted-proxy"`, `gateway.trustedProxies`, `gateway.auth.trustedProxy.userHeader`, `requiredHeaders`, `allowUsers`, an explicit `allowLoopback` decision, and any needed Control UI allowed-origin automation, with targeted tests and docs aligned to the trusted-proxy auth guide.
