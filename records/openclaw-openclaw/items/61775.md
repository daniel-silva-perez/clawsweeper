## Summary

Keep this PR open. Current main still has only the minimal build-only Makefile, so the PR's intended developer workflow wrapper has not been implemented elsewhere. The PR is focused and plausibly useful, but it needs maintainer review against current command policy and at least one correctness fix before landing.

## Best Possible Solution

Keep the PR open for maintainer review. If OpenClaw wants a Makefile convenience layer, revise this PR against current main by fixing the format target semantics, adding landing-gate to .PHONY, and smoke-checking make help plus the non-mutating wrapper targets. Then land it as a focused developer-experience change if maintainers want make wrappers alongside the documented pnpm commands.

## Risks / Open Questions

- As written, make format would mutate files despite being described as a formatting check; it should call pnpm format:check or have its help text changed.
- landing-gate should be added to .PHONY before merge, matching the bot review finding.
- The deps target intentionally runs pnpm install when node_modules is out of date; that is a developer convenience but should be an explicit maintainer-approved Makefile behavior.
