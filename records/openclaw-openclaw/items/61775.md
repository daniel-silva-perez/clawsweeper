## Summary

Keep open. Current main still has only a minimal root Makefile, so this PR is not obsolete, but the proposed wrapper is not merge-ready because the new `make format` quality target would rewrite files instead of checking formatting, and `landing-gate` is missing from `.PHONY`.

## What This Changes

This PR replaces the root Makefile with contributor convenience targets for help, dependency install, build/UI, clean, quality checks, tests, development, version output, and a landing gate that delegates to existing pnpm scripts.

## Best Possible Solution

Keep the convenience Makefile direction if maintainers want a Make entry point, but make each target accurately mirror the existing pnpm contract: `make format` should run `pnpm format:check`, `format-fix` should remain the writing path, and every logical command target should be listed in `.PHONY` before rebase and CI review.

## Reproduction Assessment

Yes. The key findings are statically reproducible from the PR diff plus current `package.json`: the proposed `make format` line invokes the writing formatter script, and `landing-gate` appears as a target while missing from `.PHONY`.

## Solution Assessment

No. A Makefile wrapper is a narrow maintainable solution for contributor convenience, but this patch should first align the formatter target with the repo's check/write split and fix the phony target list.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Route make format to format:check:** `Makefile:51`
  - body: `format` is documented as a check-only target, but it runs `pnpm format`; in current `package.json` that script is `oxfmt --write --threads=1`. As a result `make format` rewrites files instead of failing on formatting drift, so the quality gate can pass only after mutating the tree. Please call `pnpm format:check` here and leave `format-fix` as the write path.
  - confidence: 0.94
- **[P3] Declare landing-gate as phony:** `Makefile:1-3`
  - body: `landing-gate` is a command target, but it is missing from the `.PHONY` list. If a file or directory named `landing-gate` appears at the repo root, `make landing-gate` will be treated as up to date and skip `check test build`.
  - confidence: 0.88
