## Summary

Keep open. Current main has adjacent timeout and cleanup improvements, but it still does not make the Discord timeout outcome unambiguous: normal Discord direct pre-reply failures still use generic copy, diagnostic stuck-session recovery remains observe-only for active embedded/reply/lane work by default, and related active-abort/recovery PRs are still open rather than merged into a chosen policy.

## What This Changes

The issue requests a clear terminal timeout/session-health outcome for Discord-routed agent sessions so wedged sessions stop receiving future work and users can distinguish possible partial success from total failure.

## Best Possible Solution

Add a shared terminal timeout/session-health transition in the core agent/session runner that marks or rotates the affected session before later routing, and surface channel-appropriate timeout copy that warns previous side effects may have completed. Keep Discord regression coverage, but avoid a Discord-only workaround or unconditional active-run abort policy.

## Solution Assessment

Unclear because no patch is proposed here and the remaining fix is policy-sensitive.
