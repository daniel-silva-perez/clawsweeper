## Summary

Keep this issue open. Current main still does not implement `heartbeat.model: { primary, fallbacks }`: the config schema, public types, generated schema, docs, and heartbeat runtime plumbing remain string-only for heartbeat-specific model selection. Existing fallback machinery and agent-level fallback overrides exist, including the narrower behavior described by related #62017, but heartbeat does not expose or forward its own fallback chain. Related PR #40875 is open and docs-only, so it does not supersede the runtime feature request.

## Best Possible Solution

Keep the issue open and implement heartbeat-specific fallback-chain support in core: update heartbeat schema and public config types to use `AgentModelConfig`, regenerate schema/help, document the object format on the public heartbeat/config docs, resolve heartbeat primary and fallbacks with the existing model-input helpers, pass a heartbeat fallback override into the reply/run path and `runWithModelFallback` while preserving explicit empty-array semantics, and add focused config plus heartbeat runtime tests. PR #40875 can remain useful documentation context, but it should not be treated as the implementation.
