## Summary

Keep open. This is a fresh WhatsApp direct-message delivery report against v2026.4.26, and current main does not provide source, release, or reporter-retest proof that the observed typing-without-send behavior is fixed. The current WhatsApp path can still start composing before final delivery, and delivery errors in the shared dispatcher path still depend on an optional onError hook that the WhatsApp inbound dispatch path does not wire.

## What This Changes

The issue reports that after WhatsApp QR setup on OpenClaw v2026.4.26, WhatsApp DMs show typing but never deliver the generated reply while Telegram works.

## Best Possible Solution

Keep this issue open until maintainers mirror the Discord findings and capture logs around a failed WhatsApp turn. The fix should stay in the WhatsApp delivery/dispatch path if the turn reaches a sendable payload and fails to send, or move to the shared reply/provider runtime only if logs show the agent run never produces deliverable output.
