## Summary

Keep this RFC open. Current main has an owner-only cron tool and gateway cron RPCs, but it does not implement the proposed agent-facing scheduling API with gateway-injected non-forgeable provenance, playbook allowlists, recurring activeUntil enforcement, per-agent quotas, or schedule-create audit entries. The AS76 comment reinforces that this is relevant to multi-agent orchestration, and related #71711 remains open for recurring-job expiration.

## Best Possible Solution

Keep this open as a maintainer-reviewed security/API RFC. The best path is to design a gateway-mediated scheduling surface, or a constrained mode of the existing cron tool, that injects actor/session provenance server-side, requires registered playbooks and param schemas, depends on or incorporates #71711 for recurring expiration, adds per-agent quotas, records audit entries, and documents how this differs from the unrestricted operator cron CLI.

## Risks / Open Questions
