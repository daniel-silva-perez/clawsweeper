## Summary

Keep open. Current main still lacks the requested constrained Gateway API for agent-created schedules, and the proposal is a security/API design item rather than stale cleanup or a narrow implemented bug.

## What This Changes

Request an agent-facing scheduling API that creates constrained cron jobs through the Gateway with server-injected provenance, playbook allowlists, quotas, recurring lifetime semantics, and audit logging.

## Best Possible Solution

Keep this as a maintainer-owned security/API RFC and design an additive Gateway scheduling contract that preserves unrestricted operator cron while adding a constrained agent path with server-side actor/session provenance, registered playbooks, parameter validation, quotas, audit records, and documented recurring lifetime semantics.

## Reproduction Assessment

Not applicable as a feature/security RFC. The current-state verification is high-confidence: source, protocol schemas, tests, and docs on current main show the proposed method and fields are absent.

## Solution Assessment

Unclear for implementation, but keeping the issue open is the right triage outcome. The requested direction is plausible, yet the lifetime, provenance, migration, and permission model need maintainer design before a safe PR boundary exists.
