## Summary

Current main has partial mitigations for context visibility and pruning, but it still does not implement the central request: pruning preview/history, pruning notifications, pruning metadata/events, or keyword/tag retention strategy. The existing Codex review comment also kept this open and correctly identified #8606 and #9409 as related narrower work rather than full replacements.

## Best Possible Solution

Keep this open and narrow it into a maintainer-approved pruning observability/control design. The likely core path is to extend src/agents/pi-hooks/context-pruning/* to return safe pruning metadata, add a pruning lifecycle event coordinated with #8606, surface concise Control UI/chat notifications when pruning happens, document existing and new knobs, and leave overflow-message specifics to #9409.
