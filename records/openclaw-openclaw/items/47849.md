## Summary

Keep open. Current main still has gateway-wide authentication plus device/operator scopes, not built-in Control UI user accounts or server-side per-user session authorization. The repo has adjacent isolation tools for multi-agent routing and messaging DM scopes, but they do not satisfy the requested Web UI multi-user login with isolated session visibility.

## Best Possible Solution

Keep this open for explicit maintainer product and security design. The best implementation would add a server-side account/user model, a Control UI login flow, user-to-agent/session ownership metadata, filtering/authorization for sessions.* and chat.history, trusted-proxy/Tailscale identity mapping, tests around cross-user denial, and updated docs/security guidance. If maintainers decide OpenClaw should stay personal-assistant-first without built-in Web UI user accounts, that should be an explicit product decision rather than automated cleanup.
