## Summary

Keep open. Current main still does not provide built-in Control UI multi-user login or server-side per-user session isolation. The implemented model is gateway-level auth plus device pairing, with multi-agent isolation available as an agent/workspace/routing concept rather than Web UI user accounts.

## Best Possible Solution

Keep this open for maintainer product and security design. A real implementation would belong in core Gateway and Control UI, likely adding a server-side user/account model, login flow, user-to-agent/session ownership mapping, filtering/authorization for sessions.* and chat.history, trusted-proxy identity mapping, and updated security/docs. If maintainers decide OpenClaw should not support this model, that should be an explicit maintainer close rather than an automated cleanup close.
