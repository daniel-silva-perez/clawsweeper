## Summary

Keep open. Current main still has Gateway-wide operator authentication and scoped operator permissions, but no built-in Control UI user accounts or server-side per-user session authorization for the requested Web UI multi-user login/session isolation.

## Best Possible Solution

Keep this open for explicit maintainer product and security design. A complete implementation would add a server-side user/account model, Control UI login/session state, user-to-agent/session ownership metadata, authorization/filtering for sessions.list/sessions.preview/chat.history and related read/write RPCs, trusted-proxy/Tailscale identity mapping, cross-user denial tests, and updated security/docs guidance. If maintainers decide OpenClaw should remain personal-assistant-first without built-in Web UI multi-user accounts, that should be recorded as an explicit product decision.
