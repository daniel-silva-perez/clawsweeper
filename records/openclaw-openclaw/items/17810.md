## Summary

Keep open. Current main still renders SOUL/IDENTITY/USER/MEMORY-style bootstrap files as ordered but flat Project Context entries, with only a fixed SOUL.md guidance sentence and no configurable elevation/exclusion surface. The related implementation PR is closed unmerged, and the latest maintainer-side review context keeps the item open for prompt/API product direction.

## What This Changes

Add configurable context-file elevation so selected bootstrap files can render as dedicated prompt sections before Project Context and be omitted from the flat Project Context block unless explicitly retained.

## Best Possible Solution

Maintainers should choose the accepted prompt/context surface, then either add a documented typed core config for deterministic per-file elevation and exclusion or extend the plugin/context-engine API so an external implementation can map bootstrap files to dedicated sections without replacing the whole OpenClaw system prompt.
