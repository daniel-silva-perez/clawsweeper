## Summary

Keep open. Current main still prioritizes these bootstrap filenames only by sort order; it does not implement configurable context-file elevation, dedicated Doctrine/Identity/User Context/Memory sections, or exclusion of elevated files from the flat Project Context block. The latest maintainer-side comment also explicitly keeps this open for product/API direction, and the related implementation PR is closed unmerged.

## What This Changes

Add configurable context-file elevation so selected bootstrap files such as SOUL.md, IDENTITY.md, USER.md, and MEMORY.md can render as dedicated prompt sections before Project Context and optionally be excluded from the flat Project Context block.

## Best Possible Solution

Keep this open for maintainer product/API direction. If accepted in core, add a documented typed config surface for deterministic per-file elevation before Project Context, preserve current behavior when unset, exclude elevated files from the flat block unless explicitly retained, and update prompt rendering tests, config schema/help/generated docs, and public docs. If maintainers prefer externalization, expose or document a stable plugin/context-engine path that can map bootstrap files to prompt sections and remove them from Project Context without replacing the whole OpenClaw system prompt.
