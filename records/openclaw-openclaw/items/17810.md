## Summary

Keep open. Current main still renders the requested bootstrap files as ordered but flat Project Context entries, has no configurable elevation or exclusion surface, and the related implementation PR is closed unmerged. ## What This Changes

The issue requests configurable elevation of selected bootstrap context files into dedicated prompt sections instead of rendering them as ordinary Project Context entries.

## Best Possible Solution

Define a documented deterministic context-elevation contract, then implement it where prompt assembly and agent config or the plugin context engine meet, with coverage for ordering, exclusion, missing files, cache placement, and instruction-priority wording.
