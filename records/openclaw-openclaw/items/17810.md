## Summary

Keep open. Current main still renders the requested bootstrap files as ordered but flat Project Context entries, has no configurable elevation/exclusion surface, and the related implementation PR is closed unmerged. The latest maintainer-side discussion also points to an unresolved prompt/API product boundary rather than a cleanup-closeable item.

## What This Changes

The issue requests configurable elevation of selected bootstrap/context files such as SOUL.md, IDENTITY.md, USER.md, and MEMORY.md into dedicated prompt sections ahead of the flat Project Context block.

## Best Possible Solution

Define a documented deterministic context-file elevation contract with matching semantics, ordering, exclusion behavior, missing-file behavior, cache placement, and priority-scoping, then implement it either in agent prompt assembly/config or as a narrow plugin/context-engine extension point with aligned tests and docs.
