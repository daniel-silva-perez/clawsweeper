## Summary

Keep open. Current main still renders bootstrap/context files as flat Project Context entries with only fixed basename ordering and a SOUL.md guidance sentence; it does not provide configurable file elevation, dedicated Doctrine/Identity/User Context/Memory sections, or exclusion from the flat block. The related implementation PR is closed unmerged, and the latest maintainer-side review comment keeps the item open for product/API direction.

## What This Changes

Add configurable context-file elevation so selected bootstrap files can render as dedicated prompt sections before Project Context and be omitted from the flat context block unless explicitly retained.

## Best Possible Solution

Choose a maintainer-approved direction for this prompt/context surface: either add a documented typed core config for deterministic per-file elevation and exclusion, or extend/document the plugin/context-engine API so an external implementation can map bootstrap files to sections without replacing the whole OpenClaw system prompt.
