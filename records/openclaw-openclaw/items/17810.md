## Summary

Keep open. Current main still only prioritizes SOUL.md, IDENTITY.md, USER.md, and MEMORY.md in flat Project Context rendering; it does not provide configurable context-file elevation, dedicated sections, or Project Context exclusion. The directly related implementation PR #18110 is closed and unmerged.

## Best Possible Solution

Keep this issue open for maintainer product direction. If accepted in core, add a documented config/schema surface for deterministic context-file elevation before Project Context, preserve current behavior when unset, exclude elevated files from the flat block unless explicitly retained, and cover prompt rendering, config schema/help/generated docs, and public docs. If maintainers prefer externalization, document the plugin/context-engine or prompt-hook path clearly enough that contributors can add stable prompt sections without replacing the whole system prompt.
