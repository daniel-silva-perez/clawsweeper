## Summary

Keep open. Current main still renders SOUL/IDENTITY/USER/MEMORY-style bootstrap files as ordered but flat Project Context entries, exposes no configurable elevation/exclusion surface in prompt params or agent-default config, and the direct implementation PR linked from the discussion is closed unmerged. The remaining blocker is maintainer product/API direction, not cleanup closure.

## What This Changes

The issue requests configurable elevation of selected workspace context files into dedicated system prompt sections, with matched files optionally excluded from the flat Project Context block.

## Best Possible Solution

Define a documented deterministic context-file elevation contract covering match semantics, order, exclusion, missing-file behavior, prompt-cache placement, and priority scoping, then implement it in the maintainer-approved surface with aligned tests and docs.
