## Summary

Keep open. Current main still sorts SOUL.md, IDENTITY.md, USER.md, and MEMORY.md ahead of other context files, but renders them as ordinary `## <path>` entries under `# Project Context`; there is no documented `context.elevation` config/schema surface, and the related implementation PR #18110 is closed unmerged.

## Best Possible Solution

Keep this issue open for maintainer product direction. If accepted in core, add a documented config/schema surface for deterministic context-file elevation before Project Context, preserve current behavior when unset, exclude elevated files from the flat block unless explicitly retained, and cover prompt rendering, config schema/help/generated docs, and public docs. If maintainers prefer externalization, document the plugin/context-engine or bootstrap-hook path clearly enough that contributors can add stable prompt sections without replacing the whole system prompt.
