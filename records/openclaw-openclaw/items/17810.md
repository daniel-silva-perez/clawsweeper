## Summary

Keep open. Current main still renders SOUL/IDENTITY/USER/MEMORY-style bootstrap files as ordered but flat Project Context entries, the requested configurable elevation/exclusion surface is absent from source/config/docs, and the related implementation PR is closed unmerged. The remaining question is product/API direction, not a cleanup-closeable stale item.

## What This Changes

Request configurable elevation of selected bootstrap/context files into dedicated prompt sections, with optional exclusion from the flat Project Context rendering.

## Best Possible Solution

Define a documented deterministic context-file elevation contract covering match semantics, ordering, exclusion, missing-file behavior, prompt-cache placement, and priority-scoping, then implement it either in agent prompt assembly/config or as a narrow plugin/context-engine API with aligned tests and docs.
