## Summary

Keep open. Current main still prioritizes SOUL.md, IDENTITY.md, USER.md, and MEMORY.md only by sort order, then renders them as flat files under Project Context. There is no typed `context.elevation`/`contextElevation` config surface, tests and docs still assert Project Context injection, and the related implementation PR #18110 is closed unmerged. The remaining question is product/API direction: core prompt config versus a stable plugin/context-engine path that can also exclude elevated files from the flat block.

## Best Possible Solution

Keep this issue open for maintainer product direction. If accepted in core, add a documented typed config/schema surface for deterministic context-file elevation before Project Context, preserve current behavior when unset, exclude elevated files from the flat block unless explicitly retained, and cover prompt rendering, config schema/help/generated docs, and public docs. If maintainers prefer externalization, document or expose a stable plugin/context-engine path that can map bootstrap files to prompt sections and remove them from Project Context without replacing the whole OpenClaw system prompt.
