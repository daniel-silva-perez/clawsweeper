## Summary

Keep open. Current main still renders the named bootstrap files as ordered but flat Project Context entries, has no configurable elevation/exclusion surface in prompt params or agent-default config, and the related implementation PR #18110 is closed unmerged. The latest issue discussion also points to maintainer product/API direction as the remaining blocker rather than cleanup closure.

## What This Changes

The issue requests configurable elevation of selected context/bootstrap files such as SOUL.md, IDENTITY.md, USER.md, and MEMORY.md into dedicated prompt sections instead of flat Project Context entries.

## Best Possible Solution

Define a documented deterministic context-file elevation contract covering match semantics, order, exclusion, missing-file behavior, prompt-cache placement, and priority scoping, then implement it in the maintainer-approved surface with aligned tests and docs.
