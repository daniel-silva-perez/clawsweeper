## Summary

Keep open. Current main still sorts the named bootstrap files but renders them as flat Project Context entries, exposes no `context.elevation`-style config or prompt parameter, and the direct implementation PR is closed unmerged; the remaining decision is the accepted core config versus plugin/context-engine boundary.

## What This Changes

Request to add configurable context-file elevation so files matching patterns such as `SOUL.md`, `IDENTITY.md`, `USER.md`, and `MEMORY.md` render as dedicated prompt sections and can be excluded from flat Project Context.

## Best Possible Solution

Define a deterministic, documented context-file elevation contract, then implement it in the maintainer-approved surface with prompt rendering, exclusion behavior, prompt-cache placement, schema/help labels, tests, and public docs kept in sync.

## Reproduction Assessment

Yes. A focused static reproduction is to pass `SOUL.md`, `IDENTITY.md`, `USER.md`, and `MEMORY.md` as `contextFiles` into `buildAgentSystemPrompt`; current main renders them as `## <path>` entries under `# Project Context`, and existing tests assert that flat model.

## Solution Assessment

Unclear. The requested typed config is plausible, but the best maintainable fix depends on whether maintainers want this as core prompt config or as a plugin/context-engine/hook contract that can both elevate and exclude bootstrap files.
