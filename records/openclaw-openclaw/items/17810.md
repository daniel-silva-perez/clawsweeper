## Summary

Keep open. Current main still treats the named bootstrap files as ordered Project Context entries rather than configurable elevated prompt sections, and the direct implementation PR is closed unmerged; the remaining blocker is maintainer product/API direction, not cleanup closure.

## What This Changes

The issue requests configurable elevation of SOUL.md, IDENTITY.md, USER.md, and MEMORY.md into dedicated prompt sections with optional exclusion from the flat Project Context block.

## Best Possible Solution

Define a deterministic, documented context-file elevation contract covering match semantics, ordering, exclusion, missing-file behavior, prompt-cache placement, and priority scoping; then implement it in the maintainer-approved surface with aligned config schema, tests, and public docs.

## Reproduction Assessment

Yes. A focused static reproduction is to pass SOUL.md, IDENTITY.md, USER.md, and MEMORY.md through buildAgentSystemPrompt; current main sorts them and emits them as `## <path>` under `# Project Context`, and existing tests assert that flat rendering model.

## Solution Assessment

Unclear. The requested behavior is narrow, but the best maintainable solution still needs maintainer direction: a typed core config could solve it directly, while a plugin/context-engine route would need a clear file-level elevation and exclusion contract.
