## Summary

Keep open. Current main still renders the requested bootstrap files as ordered but flat Project Context entries, with only fixed SOUL.md guidance and no configurable elevation or exclusion surface. The contributor supplied a concrete design and PR, but that PR is closed unmerged and the latest maintainer-side discussion still points to an unresolved prompt/API product boundary.

## What This Changes

The issue asks for configurable elevation of selected context files such as SOUL.md, IDENTITY.md, USER.md, and MEMORY.md into dedicated prompt sections instead of rendering them as ordinary Project Context files.

## Best Possible Solution

Define a documented deterministic context-elevation contract, then implement it where prompt assembly and agent config or the plugin context engine meet, with tests proving ordering, exclusion behavior, missing-file handling, cache placement, and instruction-priority wording.
