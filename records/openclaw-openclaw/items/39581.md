## Summary

Keep this PR open. Current main still emits the embedded-runner `llm_output` hook whenever the hook is registered, with no guard for actual current-attempt assistant output. The proposed PR is directionally useful but should be narrowed: it guards full-history `lastAssistant`, which can be stale after a no-output attempt, and it bundles unrelated `chat-content` JSON parsing changes that already drew review concerns.

## Best Possible Solution

Keep this PR open, but request a focused update: guard embedded-runner `llm_output` on actual current-attempt assistant output, avoid stale full-history `lastAssistant`, preserve normal-output hook emission, add embedded-runner regression coverage for timeout/no-output and normal-output cases, and split or drop the unrelated `chat-content` JSON parsing change unless it is fixed and justified separately.
