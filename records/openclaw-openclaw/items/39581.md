## Summary

Keep this PR open. Current main still emits the embedded-runner `llm_output` hook whenever the hook is registered and passes `lastAssistant` even when the current attempt produced no assistant output. The core bug remains real, but the patch should be narrowed: the guard should use actual current-attempt assistant output rather than full-history `lastAssistant`, and the unrelated `chat-content` JSON parsing changes still have unresolved review concerns. Security pass found no supply-chain-sensitive files in the provided PR diff.

## Best Possible Solution

Keep this PR open, but request a focused patch: guard embedded-runner `llm_output` on actual current-attempt assistant output, avoid stale full-history `lastAssistant`, preserve normal-output hook emission, add regression coverage for timeout/no-output and normal-output cases, and split or drop the unrelated `chat-content` JSON parsing change unless it is fixed and justified separately.
