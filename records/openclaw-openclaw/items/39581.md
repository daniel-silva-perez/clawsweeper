## Summary

Keep this PR open. Current main still emits the embedded-runner `llm_output` hook whenever the hook is registered, even when the current attempt produced no assistant output. The PR targets a real gap, but the proposed guard is too broad because it keys off full-history `lastAssistant`, and the PR also bundles unrelated `chat-content` JSON parsing changes with review concerns.

## Best Possible Solution

Keep this PR open, but request a focused update: guard embedded-runner `llm_output` on actual current-attempt assistant output, avoid stale full-history `lastAssistant`, preserve normal-output hook emission, add embedded-runner regression coverage for timeout/no-output and normal-output cases, and split or drop the unrelated `chat-content` JSON parsing change unless it is fixed and justified separately.
