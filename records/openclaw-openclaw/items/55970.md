## Summary

Keep this PR open. Current main still does not implement the intended Telegram inbound document fallback explanation: file-too-big and retry-exhausted getFile failures collapse to null, the final body renderer has no unavailable-reason branch, and the options type has no field to carry that reason. The linked bug #55917 remains open, and the PR review already identified a debounce propagation gap that should be addressed before merge.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to revise the Telegram document fallback so failed inbound document getFile cases return a structured unavailable reason, final body rendering preserves user text plus <media:document> plus the reason, and both single-entry and multi-entry debounce paths intentionally propagate or merge that reason with focused resolver and final-body tests.
