## Summary

Keep this PR open. Current main still does not implement the PR’s intended Telegram document fallback explanation, and the linked open bug #55917 remains the right product context. The PR is still potentially useful, but it should address the existing debounce propagation review before it is mergeable.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to revise the Telegram document fallback so failed inbound document downloads return a structured unavailable reason, final body rendering preserves user text plus `<media:document>` plus the reason, and both single-entry and multi-entry debounce paths intentionally propagate or combine that reason with focused tests.
