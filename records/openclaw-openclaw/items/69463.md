## Summary

Keep this PR open. Current main has useful related observability work, including OTEL export, model-call hooks, and trusted W3C traceparent propagation, but it does not implement the PR’s specific opt-in `diagnostics.sessionTracing` config or configurable `x-session-id` / `x-session-name` style headers on outgoing LLM provider requests. The PR is not obsolete; it needs maintainer review around config contract, docs/changelog, and header/privacy hardening before merge.

## Best Possible Solution

Review this PR as a still-relevant narrow feature. If maintainers want core-level session tracing headers, land it behind an opt-in diagnostics config, reuse or mirror existing provider header validation/forbidden-header behavior, preserve trusted `traceparent` precedence, add docs/changelog for the new config, and keep focused tests around disabled-by-default behavior, caller-header precedence, invalid session keys, subagent/run IDs, and privacy-sensitive fallbacks. If maintainers decide this belongs outside core, route it to a provider/observability plugin design explicitly rather than closing as already implemented.
