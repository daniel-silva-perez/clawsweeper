## Summary

Keep PR #72051 open. Current main at 998e37fcb3d5b937fcf70d7509b677efba939f10 still lacks both substantive PR behaviors: mirrored TaskFlow terminal timestamps can still violate the audit invariant, and the packaged build still does not declare a stable root dist entry for task-registry-control.runtime. The PR also has a valid review concern that should be addressed before merge: avoid rewriting legacy terminal endedAt values to the current sync time when there is no newer event timestamp.

## Best Possible Solution

Keep the PR open and request a narrow revision: preserve historical terminal endedAt when no newer lastEventAt exists, clamp only the lastEventAt > endedAt mirrored-flow case that violates TaskFlow audit invariants, keep the stable task-registry-control.runtime dist entry, and retain regression tests for both timestamp normalization and the build entry. If the packaging failure is urgent, split that root dist entry into a small fix tied to #72320 while the timestamp semantics are finalized.
