## Summary

Keep this PR open. Current main already has the stable `task-registry-control.runtime` dist entry and a broader tsdown config regression check, but it still lacks the TaskFlow timestamp hardening that is central to this PR: terminal mirrored flows can still be written with `endedAt < updatedAt` after task delivery updates `lastEventAt`.

## Best Possible Solution

Keep this PR open, rebase it over current main, drop or adapt the packaging-only part now covered by main, and revise the timestamp helper so mirrored terminal flows clamp only the `lastEventAt > endedAt` case while preserving historical terminal `endedAt` when no newer event timestamp exists. Retain a focused regression test for `lastEventAt > endedAt`.
