## Summary

Keep open. Current main still has the Slack thread media fallback this PR is trying to remove: text-only thread replies can resolve the root message files into `threadStarterMedia`, then use that as the effective media payload. The linked bug #60335 is still open, and this PR is still a plausible implementation candidate rather than obsolete.

## Best Possible Solution

Review and either land this PR after rebase/validation, or apply an equivalent Slack plugin fix on main: stop hydrating `threadStarterMedia` for thread replies, use only the reply's direct media for `effectiveMedia`, keep direct reply attachments working, and add the targeted regression tests described in the PR. Keep linked issue #60335 open until an implementation merges.
