## Summary

Keep open. Current main has generic Docker env support, workspace inheritance plumbing, and sandbox recreate filtering, but the specific subagent inheritance gap described in the issue is still not clearly solved: `SpawnedToolContext` carries workspace/group metadata only, `sessions_spawn(sandbox="inherit")` only enforces sandboxed/unsandboxed policy, and child sandbox config is still resolved independently rather than from the parent resolved sandbox context.

## Best Possible Solution

Keep this open for maintainer design review. The narrow technical path is to define `sandbox="inherit"` semantics explicitly, add regression coverage for same-agent and cross-agent subagent spawns with `sandbox.docker.env` plus `setupCommand`, and fix the owner boundary without leaking parent env into unrelated target agents.
