## Summary

Keep open. Current main still treats `sessions_spawn` sandbox inheritance as a sandboxed/unsandboxed guard and forwards only workspace metadata into the child session; Docker env/setup state is still resolved independently from global/target-agent sandbox config. The issue discussion’s root-cause analysis remains consistent with the code, and there is no current-main regression coverage proving same-agent subagents inherit effective Docker env/setup after recreate without leaking cross-agent env.

## Best Possible Solution

Keep this open for maintainer design review, then implement a focused fix that defines `sandbox="inherit"` semantics for native subagents. The likely path is same-agent inheritance of the effective resolved Docker env/setup context, explicit cross-agent non-inheritance unless reviewed, and regression coverage for env/setup after recreate plus workspace freshness.
