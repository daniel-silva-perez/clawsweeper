## Summary

Keep this issue open. Current main still lacks an official Docker/Linuxbrew install path or Linux-native fallback for brew-only skill installers, and the reporter’s example skill remains brew-only. The thread also has active implementation candidates and a maintainer comment saying to keep this issue open for follow-up.

## Best Possible Solution

Keep the issue open while maintainers choose and land one focused fix: an opt-in Linuxbrew Docker build/setup flow, Linux-native installer metadata or fallback for affected bundled skills, or onboarding/install filtering that avoids offering brew-only installs in Linux containers without brew. Add regression coverage for Docker skill dependency handling and update Docker/skills docs when the chosen path lands.
