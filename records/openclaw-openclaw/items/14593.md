## Summary

Keep this issue open. Current main still matches the reported Docker skill-install failure: the official Docker setup has no Linuxbrew/Homebrew install path, the reported openai-whisper bundled skill remains brew-only, and the installer still returns the brew-not-installed failure when no brew executable is present. The thread also has active open fix PRs with closing syntax, plus a maintainer comment explicitly saying to keep this issue open for follow-up.

## Best Possible Solution

Keep this issue open while maintainers review or close the linked implementation PRs and choose a focused fix. The likely paths are an opt-in Linuxbrew Docker build/setup flow, Linux-native installer metadata for brew-only bundled skills such as openai-whisper, or onboarding behavior that avoids offering brew-only installs in Docker when brew is unavailable. Add Docker/skills regression coverage and update Docker docs when the chosen fix lands; then close as implemented.
