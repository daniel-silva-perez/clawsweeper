## Summary

Keep this issue open. Current main still lacks an official Docker/Linuxbrew path or a Linux-native fallback for brew-only skill installers, and the reporter’s `openai-whisper` example remains brew-only. There are also active implementation candidates with closing intent, including #42303 and #55940, plus the thread’s later #57553 path, so the best maintainer action is to review or reject those fixes before closing this issue.

## Best Possible Solution

Keep this issue open while maintainers choose one focused path: land or close the active PRs, or implement an official opt-in Linuxbrew Docker setup, Linux-native installer metadata/fallbacks for affected bundled skills, or onboarding/install filtering that avoids offering brew-only installs in Linux containers without brew. Add Docker skill dependency regression coverage and update the Docker and skills docs when the chosen path lands.
