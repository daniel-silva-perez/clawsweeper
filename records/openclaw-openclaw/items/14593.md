## Summary

Keep open. Current main still lacks an official Docker/Linuxbrew path or a Linux-container fallback for brew-only skill installers, and related implementation PRs are still active in the provided GitHub context. The reporter’s `openai-whisper` example remains brew-only, so onboarding can still offer an install path that fails when `brew` is absent in the Docker container.

## Best Possible Solution

Keep this issue open while maintainers choose one focused path: review and either land or close #42303, #55940, and #57553, or implement an official opt-in Linuxbrew Docker setup, Linux-native installer metadata/fallbacks for affected bundled skills, or container-aware onboarding filtering. The accepted fix should include Docker skill-dependency regression coverage and updates to Docker/skills docs.
