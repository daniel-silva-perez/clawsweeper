## Summary

Keep open. Current main still classifies Unix gateway listeners from `lsof -Fpc` command-name tokens before any argv inspection, and the tests still assert that non-`openclaw` command names are dropped. That matches the macOS `cnode`/rewritten-argv failure described in the report. There is also an open paired implementation PR, #70681, with `Fixes` linkage, so the issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land #70681 or an equivalent narrow fix. The fix should stop using the lsof `c` field as the Unix gateway classifier, preserve self/ancestor exclusion, verify Unix listener PIDs through full argv inspection using the existing gateway argv contract, and update restart-stale-pids coverage for macOS-style `cnode` plus unrelated non-gateway listeners.
