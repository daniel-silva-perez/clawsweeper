## Summary

Current main still matches the reported macOS/Unix stale gateway PID bug. The Unix path still classifies lsof `-Fpc` listeners by the `c` command token containing `openclaw`, so a Node-backed gateway reported as `cnode` is dropped before the existing argv verifier can inspect it. The issue is also paired with open closing PR #70681, so it should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land #70681 or an equivalent narrow fix. The fix should stop using lsof `c`/`p_comm` as the Unix gateway classifier, parse listener PIDs separately from gateway classification, verify Unix listener PIDs through full argv inspection before treating them as OpenClaw gateways, preserve self and ancestor exclusions, and update coverage for macOS-style `cnode` plus unrelated non-gateway listeners.
