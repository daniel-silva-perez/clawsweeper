## Summary

Keep this PR open. Current main does not implement the proposed cron maintenance-window config, scheduler role isolation, deferred replay, manual-run/status diagnostics, or heartbeat queueing. The PR is an implementation candidate, but it is not merged and the provided review context includes unresolved functional issues plus an unrelated security-sensitive CI workflow change that should be split or reviewed separately.

## Best Possible Solution

Keep the PR open as useful but unfinished work. The best path is to rebase or recreate it as a focused cron/heartbeat maintenance-window PR, address the existing review comments with tests, and split the `.github/workflows/ci.yml` secret-scanning change into a separate security/CI PR for explicit maintainer review. If the linked WhatsApp cron incident #65774 is the motivating production bug, the maintainer path should also confirm whether this role-isolation design actually solves that user-facing stop/out-of-hours failure or whether a narrower emergency-stop/active-run cancellation fix belongs first.
