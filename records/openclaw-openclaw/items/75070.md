Keep open. Current main still has a bare session-reset model prompt path without an explicit anti-fabrication or tool-verification rule, and an open linked PR already uses closing syntax for this issue, so the issue should remain open until that PR lands or is closed.

## What This Changes

The issue asks OpenClaw to harden session resets so the first reset/startup response does not present invented calendar, email, or JSON data as user-provided fact.

## Best Possible Solution
