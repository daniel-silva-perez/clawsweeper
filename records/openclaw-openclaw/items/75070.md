## Summary

Keep open. Current main still sends a bare reset/startup prompt to the model without an explicit no-fabricated-user-data or verify-before-acting guardrail, and a linked open PR is already the implementation candidate for that narrow fix.

## What This Changes

The issue asks OpenClaw to prevent models from presenting invented calendar, email, or JSON-like data as user-provided context during bare session resets.

## Best Possible Solution

Land #75127 or an equivalent narrow patch that adds reset/startup anti-fabrication wording plus prompt regression tests, then close this issue after the fix reaches main.
