## Summary

Keep open. Current main still sends a reset/startup prompt for bare `/new` and `/reset` without the requested anti-fabrication or tool-verification guardrail, and the issue is already paired with open PR #75127 using closing syntax, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

Add a session-reset guardrail so bare `/new` and `/reset` prompts do not let the model present invented calendar, email, JSON, or other structured data as real without tool verification.

## Best Possible Solution

Land PR #75127, or an equivalent narrow prompt/test change, so all bare reset prompt variants explicitly forbid presenting invented data as real and require tool verification before acting on calendar, email, task, or structured-data claims.
