## Summary

Keep open. Current main still has reset-startup prompt variants without an explicit anti-fabrication or tool-verification guardrail, and the gateway reset path sends that prompt; linked PR #75127 remains the open unmerged implementation candidate, so this should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw to prevent session reset/bootstrap prompts from letting the model present invented calendar, email, or structured data as user-provided facts.

## Best Possible Solution

Review and land PR #75127, or an equivalent narrow prompt/test change, so every reset-startup prompt variant explicitly forbids presenting invented data as real and requires tool verification before acting on calendar, email, task, or structured-data claims.

## Reproduction Assessment

Unclear. There is a high-confidence static reproduction for the missing prompt guardrail on current main, but the actual fabricated response depends on the reported Xiaomi model/provider and lacks transcript evidence.

## Solution Assessment

Yes. A narrow reset-prompt contract guardrail with prompt regression tests is the most maintainable repo-level fix supported by the evidence; broader runtime permissions, provider routing, or tool-policy changes are not justified by this report alone.
