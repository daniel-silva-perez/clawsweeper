## Summary

Keep this issue open. Current main has a partial prompt-snapshot fix for the embedded Pi runner, but the reported Claude CLI/status path still appears actionable: CLI runs pass only normalized provider usage into session persistence, the session store derives `totalTokens` from that usage when no `promptTokens` snapshot exists, and the status card renders that value against the context window. The related ClawSweeper items are adjacent stale-zero/window-reporting fixes, not a clear canonical replacement for this cumulative-numerator bug.

## Best Possible Solution

Add a CLI/status contract that carries current working-set prompt tokens separately from provider/session cumulative usage, likely as `agentMeta.promptTokens` or a structured `context.used` snapshot for CLI runners. Then render `Context:` from that snapshot, keep cumulative session tokens only as a clearly labeled separate metric if desired, and add a regression test for `claude-cli`/`session_status` where cumulative usage exceeds the context window but the latest prompt snapshot does not.
