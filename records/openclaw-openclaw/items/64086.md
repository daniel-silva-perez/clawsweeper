## Summary

Keep open. Current main still lacks the requested PluginStatusProvider/registerStatusProvider API, registry storage, SDK/docs exposure, and TUI footer consumption. The same author also has open PR #72557 with resolving syntax for this issue, so the maintainer path is to review, land, or close that PR rather than close this issue early.

## Best Possible Solution

Keep this issue open. Review #72557 as the active implementation candidate, decide whether the status provider API and TUI footer consumption should ship together or with a tightly linked follow-up, and coordinate UX details with #53774. Close only after main gives ContextClaw and other plugins a supported way to contribute footer status without monkeypatching.
