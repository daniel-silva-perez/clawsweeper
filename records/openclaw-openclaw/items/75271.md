## Summary

Current main already addresses the Anthropic 400 loop by stripping stale trailing assistant turns from provider-bound Anthropic replay before the request is sent. The on-disk session repair pass remains narrower than the issue's suggested repair, but the observable prefill failure is covered by source, regression tests, docs, changelog, and release provenance.

## What This Changes

The issue asks OpenClaw to recover embedded sessions whose JSONL history ends with an assistant turn so Anthropic Opus 4.7 is not retried with unsupported assistant prefill.

## Best Possible Solution

Keep the shipped provider-bound replay hygiene as the canonical fix: preserve the stored transcript for user-facing history, but ensure Anthropic payloads never include stale assistant prefill tails. A separate janitor/quarantine feature would be optional operational cleanup rather than required for this failure mode.

## Reproduction Assessment

Yes. The issue includes concrete logs and counts, and the repository has a focused regression test that reproduces a user-to-assistant replay tail under Anthropic validation and verifies the provider-bound context is trimmed to the user turn.

## Solution Assessment

Yes. Stripping the stale tail immediately before provider replay is the narrowest maintainable fix because it protects Anthropic without rewriting historical transcript JSONL; on-disk repair can stay limited to malformed or known-poisoned records.
