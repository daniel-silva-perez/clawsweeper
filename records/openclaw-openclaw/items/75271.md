## Summary

Keep open. Current main still lacks the on-disk trailing failed-assistant-artifact repair requested here, and the issue is already paired with open PR #75284 using closing syntax, so it should remain open until that PR lands or is explicitly closed.

## What This Changes

The issue asks session repair to prevent embedded-agent JSONL files from remaining ended by a failed assistant turn that can be replayed to Anthropic as unsupported assistant prefill.

## Best Possible Solution

Land a narrow repair that drops only provable trailing failed-stream assistant repair artifacts while preserving normal completed trailing assistant turns, update the transcript-hygiene docs for that exception, and let the linked PR close the issue after validation.

## Reproduction Assessment

Yes for the session-file repair boundary: current main has a test shape where a failed empty assistant message is repaired into a final assistant fallback entry, and embedded attempts run that repair before session load. A live Anthropic replay was not run in this read-only review, but the reported logs and linked PR make the failure path concrete enough to keep open.

## Solution Assessment

Yes, the linked PR direction is the narrowest maintainable fix once docs and rebase follow-through are handled. Dropping only failed repair artifacts is safer than removing every completed trailing assistant turn or appending a synthetic user turn.
