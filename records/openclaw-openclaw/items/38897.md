## Summary

Current main still does not implement this PR's central Mission Control runtime-operations surface. The Gateway method/event contract, handler composition, request context, process registry, Control UI overview state, docs, and polling paths lack `dashboard.summary`, `dashboard.delta`, incident handlers, a global tool activity registry, finished-process session pivots, and the Tool Runtime / Execution Queue / Autonomy Posture panels. The PR also remains broad and carries known review issues plus security-sensitive data-surfacing questions, so it should stay open for maintainer judgment or contributor rework.

## Best Possible Solution

Keep this PR open for maintainer judgment or contributor rework. The best path is a narrower runtime-ops PR that adds a deliberate dashboard summary/delta contract, a global tool activity registry, process queue visibility with finished-session pivots, and an autonomy posture view backed by existing tool/exec policy data, while fixing the known `isError` and startup `requestContext` bugs, documenting the new surface, adding focused tests, and dropping unrelated workspace/script/test churn.
