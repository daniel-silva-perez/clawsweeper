## Summary

Keep open. Current main has the shared Anthropic thinking-prefill stripper and already applies it to direct Anthropic and Cloudflare AI Gateway routes, but the OpenRouter provider path still only composes OpenRouter routing/reasoning wrappers and does not apply that stripper for Anthropic model refs routed through `openai-completions`.

## What This Changes

The issue requests that OpenRouter-routed Anthropic Claude 4.6 models using the OpenAI-compatible adapter strip trailing assistant prefill turns before provider requests.

## Best Possible Solution

Add a provider-local OpenRouter stream wrapper that reuses the shared Anthropic thinking-prefill payload sanitizer for Anthropic OpenRouter model refs, while preserving existing OpenRouter reasoning injection, provider routing metadata, cache-control behavior, and Gemini pass-through replay policy.

## Reproduction Assessment

Yes. The issue gives a concrete configuration and error log, and source inspection shows current main lacks the OpenRouter prefill-strip path while direct Anthropic/Cloudflare paths have it.

## Solution Assessment

Yes. Reusing the existing shared payload wrapper in the OpenRouter provider boundary is narrower and more maintainable than adding broad core `openai-completions` stripping that could affect unrelated providers.
