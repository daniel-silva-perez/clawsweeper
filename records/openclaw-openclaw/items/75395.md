## Summary

Keep open. Current main has a shared Anthropic thinking-prefill sanitizer and applies it to direct Anthropic and Cloudflare AI Gateway routes, but the OpenRouter provider stream still only applies OpenRouter routing/reasoning wrappers, so Anthropic Claude models routed through OpenRouter's `openai-completions` path can still send a trailing assistant prefill.

## What This Changes

Strip trailing assistant prefill turns for Anthropic models routed through OpenRouter's OpenAI-compatible adapter when thinking is enabled.

## Best Possible Solution

Add an OpenRouter-owned stream wrapper that reuses the shared Anthropic thinking-prefill sanitizer for Anthropic OpenRouter model refs while preserving existing OpenRouter routing, reasoning, cache-control, and Gemini replay behavior.

## Reproduction Assessment

Yes. The issue gives a concrete config and provider error, and current-main source inspection shows the OpenRouter route does not apply the sanitizer that direct Anthropic and Cloudflare routes already use.

## Solution Assessment

Yes. A provider-local OpenRouter wrapper is the narrowest maintainable fix; broad `openai-completions` stripping would risk changing unrelated providers that may intentionally support assistant prefill.
