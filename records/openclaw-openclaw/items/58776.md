## Summary

Keep open. Current main has some hardening for aborted lifecycle events, including retry grace windows and subagent orphan recovery, but it still does not implement the requested additive `first-progress` or `startup-failed` lifecycle phases. The documented and source-backed contract remains `start` / `end` / `error`, and `agent.wait` still collapses aborted lifecycle end events into `timeout`.

## Best Possible Solution

Keep this issue open and implement an additive lifecycle/progress contract in the embedded runner and subagent registry: emit `first-progress` on the first qualifying tool call or visible assistant text, classify startup failures such as aborted prompt errors, empty startup output, startup timeout, and auth failure before `agent.wait`/task outcomes collapse them, then cover the behavior in `pi-embedded-subscribe`, `agent-job`, and subagent registry tests plus the agent-loop/subagents docs.
