## Summary

Keep open. This draft visual polish PR is still active and current main does not supersede it, but the diff has blocking CSS regressions: proposed light-mode warning/danger tokens become low-contrast foreground colors, and the exec approval scroll guard is removed.

## What This Changes

The branch changes Control UI accent, warning, and danger palette tokens, adjusts overview attention styling, and removes exec approval modal max-height and overflow CSS.

## Best Possible Solution

If maintainers want a calmer dashboard palette, keep the change scoped to theme/dashboard CSS, preserve exec approval overflow protections, and choose semantic warning/danger tokens that remain readable in both light and dark modes across all token consumers.

## Review Findings

  - body: The light theme changes `--warn` to `#f4c542` and `--danger` to `#ef6b6b`, but these tokens are used directly as text and icon colors across the UI.
