## Summary

Keep this draft PR open for maintainer/design review, but do not merge it as-is: current main has not adopted the palette change, and the proposed diff introduces concrete UI regressions in light-mode semantic contrast and exec approval overflow handling.

## What This Changes

The PR changes Control UI theme tokens toward blue/yellow/soft-red accents, updates overview attention item/link styling, and removes exec approval modal scroll and height CSS.

## Best Possible Solution

If maintainers want a calmer dashboard palette, keep the patch scoped to theme/dashboard styling, preserve exec approval overflow protections, and choose semantic warning/danger colors that remain readable in both light and dark modes across all token consumers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Keep semantic colors readable in light mode:** `ui/src/styles/base.css:166-169`
  - body: The light theme changes `--warn` to `#f4c542` and `--danger` to `#ef6b6b`, but these tokens are used directly as text and icon colors across the UI. Against the current light background they drop to 1.54:1 and 2.85:1 contrast, so warning and danger labels become inaccessible compared with the current AA-level values.
  - confidence: 0.95

  - body: This removes the max-height and overflow rules that keep long exec/plugin approval commands scrollable inside the modal. The shared modal caps its own height but leaves overflow visible, so long command text can push the action buttons off-screen instead of scrolling.
  - confidence: 0.88
