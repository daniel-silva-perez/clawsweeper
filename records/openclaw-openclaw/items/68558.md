## Summary

Keep this PR open and do not merge it as-is. Current main still has the misleading manual `/compact` output path, and the PR's label-clamping direction is useful, but the branch passes the model context window into `currentTokenCount`; the compactor treats that field as observed prompt usage and uses it for `tokensBefore`, so the fix can replace one misleading label with another inflated one.

## What This Changes

The PR changes `src/auto-reply/reply/commands-compact.ts` to pass a token count into manual compaction, suppress non-decreasing `before → after` labels, and use a JSONL file-size estimate when stored session totals look stale.

## Best Possible Solution

Land a corrected version that keeps the non-decreasing-label guard and stale-total display fallback, but only passes `currentTokenCount` when the handler has a real fresh prompt-token usage value; otherwise omit it and let the compactor use its own transcript estimates. Add focused tests for `before <= after`, stale stored totals, and not forwarding the context window as observed usage.
