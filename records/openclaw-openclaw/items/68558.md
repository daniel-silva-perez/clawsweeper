## Summary

Keep this PR open. Current main still has the misleading manual `/compact` reply path, so the contribution targets a real UX bug, but the branch is not safe to merge because it forwards the model context-window size as `currentTokenCount`, which the compactor treats as observed prompt usage for `tokensBefore` and token sanity checks.

## What This Changes

The PR modifies `src/auto-reply/reply/commands-compact.ts` to suppress non-decreasing compaction arrows, pass a token count into manual compaction, and estimate stale session totals from the session JSONL file size.

## Best Possible Solution

Land a corrected version that keeps the non-decreasing label guard and stale-total display fallback, but only passes `currentTokenCount` from a true fresh prompt-usage value; otherwise the manual handler should omit it and let the compactor use its transcript estimates.
