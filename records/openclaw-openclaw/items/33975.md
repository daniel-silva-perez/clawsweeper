## Summary

Keep open. Current main covers the model-attribution half through `messages.responsePrefix` templates such as `{modelFull}` and `{provider}`, but it still does not provide an opt-in approval gate before switching from the primary model to a fallback.

## Best Possible Solution

Keep this issue open as the fallback-approval tracker while treating model attribution as satisfied by `messages.responsePrefix` templates. A good implementation would add an opt-in per-agent fallback approval policy to `AgentModelConfig` and `AgentModelSchema`, pause `runWithModelFallback` before non-primary attempts, route approve/deny/timeout through an existing approval or notification surface, and add docs plus tests for approval, denial, timeout, cancellation, and channel delivery.
