## Summary

Keep open. Current main covers the model-attribution half through `messages.responsePrefix` templates such as `{modelFull}`, but the requested opt-in approval gate before switching to a fallback model is still missing.

## Best Possible Solution

Keep this open as the fallback-approval tracker. Treat message-level model attribution as satisfied by `messages.responsePrefix` templates such as `[{modelFull}]`, then implement an opt-in per-agent fallback approval policy in `AgentModelConfig`/`AgentModelSchema`, pause `runWithModelFallback` before non-primary attempts, route approve/deny/timeout/cancel through an existing or new approval surface, and add docs plus tests for approval, denial, timeout, cancellation, and channel delivery.
