## Summary

Keep open. Current main covers the model-attribution half through `messages.responsePrefix` template variables, but the fallback-approval half is still missing from config, runtime types, docs, and the fallback loop.

## Best Possible Solution

Keep this issue open as the fallback-approval tracker while treating model attribution as satisfied by `messages.responsePrefix` templates. A good implementation would add an opt-in per-agent fallback approval policy to `AgentModelConfig` and `AgentModelSchema`, pause `runWithModelFallback` before non-primary attempts, route approval through an existing approval or notification surface, define approve/deny/timeout proceed-or-fail semantics, and add docs plus tests for approval, denial, timeout, cancellation, and channel delivery.
