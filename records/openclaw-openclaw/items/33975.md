## Summary

Keep open. Current main covers the model-attribution part through `messages.responsePrefix` template variables such as `{modelFull}` and `{provider}`, but it still does not implement an opt-in approval gate before executing a fallback model.

## Best Possible Solution

Keep this issue open as the fallback-approval tracker while treating the model-attribution half as satisfied by `messages.responsePrefix` templates. A good implementation would add an opt-in per-agent fallback approval policy to `AgentModelConfig` and `AgentModelSchema`, pause `runWithModelFallback` before non-primary attempts, route approve/deny/timeout through an existing approval or notification surface, and add docs plus tests for approval, denial, timeout, cancellation, and channel delivery.
