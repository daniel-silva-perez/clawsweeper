## Summary

Keep open. Current main satisfies the model-attribution half through `messages.responsePrefix` templates such as `{modelFull}` and `{provider}`, but it still lacks an opt-in approval gate before moving from the primary model to a fallback candidate.

## Best Possible Solution

Keep this issue open as the fallback-approval tracker while treating model attribution as satisfied by `messages.responsePrefix` templates. A complete implementation should add an opt-in per-agent fallback approval policy to `AgentModelConfig` and `AgentModelSchema`, pause `runWithModelFallback` before non-primary attempts, route approve/deny/timeout through an existing approval or notification surface, and add docs plus tests for approval, denial, timeout, cancellation, and channel delivery.
