## Summary

Keep open. Current main still uses the broad poll-creation detector for `action="send"`, while this PR is a focused implementation candidate that narrows the guard to real poll question/options and adds regression coverage. I did not find a discrete blocking bug in the patch, but it still needs maintainer review of the intended poll-metadata semantics, rebase/validation against current main, and coordination with the broader open #46455 work.

## What This Changes

The PR adds a stricter `hasRealPollIntent` helper, switches the send-side poll guard to it, and updates poll/send regression tests so metadata-only poll fields are allowed on `send` while real poll question/options still reject.

## Best Possible Solution

Land this stricter send-side poll intent guard, or an equivalent action-scoped fix, if maintainers agree that metadata-only poll fields should not block normal sends. Keep real poll payloads on the corrective `action="poll"` path and preserve focused regression coverage for both allowed metadata-only sends and rejected question/options on `send`.
