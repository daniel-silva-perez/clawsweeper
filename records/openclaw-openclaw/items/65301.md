## Summary

Keep open. Current main still uses the broad poll-creation detector for `action="send"`, and current tests still expect metadata-only non-zero poll params to reject, so this PR's intended behavior is not implemented on main. The patch is a plausible narrow fix, but it changes current semantics for metadata-only and negative duration fields, so it needs maintainer review rather than automated closure.

## What This Changes

The PR adds a stricter real-poll-intent helper, switches the send-side poll guard to it, and updates regression tests so `send` only rejects non-empty poll question/options while allowing metadata-only poll fields.

## Best Possible Solution

Review and land this stricter send-side intent guard, or an equivalent action-scoped fix, if maintainers agree that metadata-only poll fields should not block normal sends. Preserve required question/options validation for real `poll` actions and keep regression coverage for both allowed metadata-only sends and rejected real-poll payloads on `send`.
