## Summary

Keep this PR open. Current main does not implement `tools.exec.denyPathPatterns`, and the linked production token-leak report is a valid security concern, but the submitted branch is not ready to merge because the new deny gate still has concrete bypasses in wrapped shell commands and bare cwd-relative secret filenames.

## What This Changes

The PR adds `tools.exec.denyPathPatterns` config/schema/docs/changelog support, deny-path helpers and tests, and enforcement before exec allowlist/approval checks in the agent and system.run paths.

## Best Possible Solution

Land a maintainer-reviewed version with one shared deny-path policy over parsed top-level argv and parsed inner shell-wrapper payloads for gateway, node, sandbox, and elevated paths; treat cwd-resolvable sensitive filenames as candidates; remove stale duplicate helpers; keep schema/docs/changelog aligned; and add integration tests for the bypass cases.

## Reproduction Assessment

Yes. Configure `tools.exec.denyPathPatterns: ["~/.openclaw/secrets/**", "**/.env"]`, then statically trace `bash -c "cat ~/.openclaw/secrets/foo.env"` through the new agent-side gate and `cat .env` through `isPathLikeArg`; both miss the deny match for the reasons shown above.

## Solution Assessment

No. The proposed config shape and pre-approval boundary are reasonable, but this branch is not the narrow maintainable fix until it reuses parsed shell-wrapper payloads and covers bare cwd-relative secret filenames.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Parse shell wrappers before deny matching:** `src/agents/bash-tools.exec.ts:1621-1623`
  - body: The agent-side gate passes the raw command string as `shellPayload`, so `bash -c "cat ~/.openclaw/secrets/foo.env"` is tokenized with the leading `cat ` text attached to the path token. That prevents a documented `~/.openclaw/secrets/**` deny pattern from matching in gateway/sandbox/elevated paths before host dispatch.
  - confidence: 0.88
- **[P2] Treat bare sensitive filenames as path candidates:** `src/infra/exec-deny-path.ts:86-95`
  - body: The matcher promises cwd-resolved relative matching and documents patterns like `**/.env`, but `isPathLikeArg` rejects bare names such as `.env` and `credentials.json`. Those arguments never reach cwd resolution, so common `cat .env` reads bypass the configured denylist.
  - confidence: 0.87
