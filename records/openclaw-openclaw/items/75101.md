## Summary

Keep open. Current main still lacks the requested exec path denylist, and the PR is directionally useful, but the latest patch is not merge-ready: the host-dispatch gate still misses wrapped shell payloads for gateway/sandbox/elevated execution, and the matcher skips bare sensitive filenames such as `.env` despite documenting cwd-resolved relative matching.

## What This Changes

The PR adds `tools.exec.denyPathPatterns` config/schema/docs/changelog support, a deny-path glob matcher, enforcement before exec policy checks, and focused matcher tests.

## Best Possible Solution

Land a maintainer-reviewed version where one shared deny-path policy evaluates parsed command segments and inner shell payloads before every host execution path, treats cwd-resolvable sensitive filenames as candidates, keeps schema/docs aligned, and adds integration tests for gateway, sandbox, elevated-full, and node execution.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to configure `tools.exec.denyPathPatterns: ["~/.openclaw/secrets/**"]` and run gateway/elevated `bash -c "cat ~/.openclaw/secrets/foo.env"`; the new top-level gate tokenizes the raw wrapper command instead of the inner shell payload, so the tilde pattern does not match.

## Solution Assessment

No. Moving the gate before host dispatch is the right boundary, but the implementation needs to reuse parsed shell-wrapper payloads and broaden candidate selection for cwd-relative sensitive filenames before it is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Parse shell wrappers before deny matching:** `src/agents/bash-tools.exec.ts:1622-1623`
  - body: The top-level gate passes the raw command string as `shellPayload`, so `bash -c "cat ~/.openclaw/secrets/foo.env"` is tokenized as one path-like token with a leading `cat ` prefix. With the documented `~/.openclaw/secrets/**` pattern this does not match, leaving gateway/sandbox/elevated execution able to read the denied path.
  - confidence: 0.88
- **[P2] Treat bare sensitive filenames as path candidates:** `src/infra/exec-deny-path.ts:86-95`
  - body: The matcher documents cwd-resolved relative matching and suggests `**/.env`, but `isPathLikeArg` skips bare filenames that lack `/`. As a result `cat .env` or `cat credentials.json` never reaches the cwd-resolution check and can bypass the configured deny pattern.
  - confidence: 0.86
