## Summary

Keep open. This PR targets a real exec-policy gap from #74379, and current main has no denyPathPatterns implementation, but the latest head still has blocking security bypasses in the gateway/sandbox/elevated path and in cwd-relative filename matching.

## What This Changes

The PR adds opt-in tools.exec.denyPathPatterns config, schema, docs, changelog support, a glob-based exec path deny matcher, gateway/node enforcement, and unit tests.

## Best Possible Solution

Land a maintainer-reviewed version with one shared deny-path policy fed by the same parsed command segments and inner shell payloads used by exec approval policy across gateway, sandbox, node, and elevated paths, with cwd-resolved bare sensitive filenames covered and stale duplicate helper files removed.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to configure tools.exec.denyPathPatterns: ["~/.openclaw/secrets/**"] and run gateway or elevated bash -c "cat ~/.openclaw/secrets/foo.env"; the new gate tokenizes raw wrapper text, so the inner path does not match the tilde glob. A second focused check is tools.exec.denyPathPatterns: ["**/.env"] with cat .env, which is skipped before cwd resolution.

## Solution Assessment

No. The requested direction is appropriate, but this PR is not yet the narrow maintainable fix because it does not reuse parsed shell-wrapper payloads for the gateway gate and filters out bare sensitive filenames before resolving them against cwd.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Parse shell wrappers before deny matching:** `src/agents/bash-tools.exec.ts:1622`
  - body: The top-level gate passes the raw command as shellPayload with argv empty. For `bash -c "cat ~/.openclaw/secrets/foo.env"`, tokenization yields a wrapper token containing the leading `cat ` text, so the documented `~/.openclaw/secrets/**` pattern does not match and gateway/sandbox/elevated runs can still read the denied file.
  - confidence: 0.9
- **[P2] Treat bare sensitive filenames as path candidates:** `src/infra/exec-deny-path.ts:86-95`
  - body: The matcher promises cwd-resolved relative matching and documents patterns like `**/.env`, but `isPathLikeArg` rejects bare names without `/`. That means `cat .env` and `cat credentials.json` are skipped before cwd resolution and bypass the advertised deny patterns.
  - confidence: 0.88
