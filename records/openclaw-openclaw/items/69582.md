## Summary

Keep open. Current main has related malformed streamed tool-call argument repair and opt-in loop detection, but it does not clearly solve the reported GLM/local behavior where valid string parameters are polluted with a literal trailing `{}` or where `process.action` is generated as `{}`. A blanket parameter cleaner would be risky because `{}` can be legitimate file/text content, and the exact root cause still needs maintainer/provider judgment or a current-main reproduction.

## Best Possible Solution

Keep this open for a focused maintainer pass. The best fix is probably not generic string cleanup; first reproduce on current main with the same local GLM/OpenAI-compatible route or capture the raw streamed tool-call deltas. If the `{}` comes from malformed transport chunks, extend the existing streamed argument repair with narrow tests. If it is model schema confusion, tighten tool schemas such as `process.action` to a flat enum and improve validation/retry guidance without mutating legitimate string payloads.
