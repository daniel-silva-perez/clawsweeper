## Summary

Keep this open. Current main has adjacent OpenAI-compatible malformed tool-call argument repair and opt-in loop detection, but it still does not clearly fix the reported local GLM failure: `process.action` is still only a broad string schema, non-string actions can flow to the same missing-session error the reporter saw, edit/string parameters ending in `{}` are preserved as valid strings, and the stream repair does not handle literal `{}` contamination inside an otherwise valid JSON argument object.

## What This Changes

The issue requests handling malformed local GLM tool-call parameters where valid string arguments gain a trailing `{}` and `process.action` may be emitted as an empty object, causing repeated failed tool invocation.

## Best Possible Solution

Keep this open for a focused maintainer/provider pass. Reproduce current main with the same local GLM/OpenAI-compatible route or capture raw streamed tool-call deltas. If `{}` is transport junk, extend `attempt.tool-call-argument-repair` with narrow provider-backed tests; if the model is being under-constrained, make `process.action` a flat enum and add runtime validation/retry coverage. Avoid generic string suffix cleanup unless the raw stream proves it is safe.
