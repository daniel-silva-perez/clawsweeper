## Summary

Keep this issue open. Current main has improved ACP docs, task records, timeouts, and parent-owned ACP cleanup, but the core ACPX/Codex reliability gaps remain: ACPX still defaults to approve-reads/fail, that config is passed directly into the runtime, parent-owned cross-agent ACP child visibility is still blocked by the generic all+A2A guard, and the docs still describe manual stale-process cleanup.

## Best Possible Solution

Keep this issue open as the canonical ACPX/Codex non-interactive reliability umbrella unless maintainers split it into narrower linked follow-ups. The best path is a maintainer-approved ACPX permission profile or spawn preflight with actionable remediation, parent-owned ACP child status/history visibility that does not require broad all+A2A access, and stale harness cleanup/reporting that removes the current manual ps/kill path.
