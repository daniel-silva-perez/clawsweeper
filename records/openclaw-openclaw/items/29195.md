## Summary

Keep this issue open. Current main has the ACP permission workaround documented and has improved task/session cleanup, but the central ACPX/Codex reliability gaps remain: the default ACPX config is still approve-reads/fail, the runtime path still passes that config through without a preflight or safer non-interactive profile, parent-owned cross-agent ACP child visibility still requires broad all+A2A access, and stale harness cleanup is still documented as a manual ps/kill operation.

## Best Possible Solution

Keep this issue open as the canonical ACPX/Codex non-interactive reliability umbrella unless maintainers split it into narrower linked follow-ups. The best path is a maintainer-approved ACPX permission profile or spawn preflight with actionable remediation, parent-owned ACP child status/history visibility that does not require broad all+A2A access, and stale harness cleanup/reporting that removes the current manual ps/kill path.
