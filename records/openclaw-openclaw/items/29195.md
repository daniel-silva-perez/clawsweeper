## Summary

Keep this issue open. Current main documents the ACPX permission workaround and has better ACP task/status/timeout handling, but the central ACPX/Codex reliability problem is not fully resolved: the embedded acpx plugin still defaults to approve-reads/fail, that policy is passed directly into the runtime, parent agents still hit cross-agent visibility gates when reading Codex ACP children, and docs still describe manual cleanup for stalled harness processes.

## Best Possible Solution

Keep this issue open as the canonical ACPX/Codex non-interactive reliability umbrella unless maintainers split it into narrower linked follow-ups. The best path is a maintainer-approved ACPX permission profile or preflight that surfaces actionable remediation before spawn, parent-owned ACP child visibility without requiring global all visibility or broad A2A allowlists, and automatic stale harness cleanup/reporting that removes the current manual ps/kill path. The existing permission docs should stay, but they are not enough to close the issue.
