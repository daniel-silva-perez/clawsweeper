## Summary

Keep this issue open. Current main documents the ACPX permission workaround and has partial ACP task/status/timeout improvements, but the central ACPX/Codex non-interactive reliability problem remains: ACPX still defaults to approve-reads/fail, those defaults are passed directly into the embedded runtime, parent agents still hit cross-agent visibility policy when inspecting Codex ACP children, and stale ACP process cleanup is still documented as manual.

## Best Possible Solution

Keep this issue open as the canonical ACPX/Codex non-interactive reliability umbrella unless maintainers split it into narrower linked follow-ups. The best product/code path is a maintainer-approved ACPX permission profile or preflight that surfaces actionable remediation before spawn, parent-owned cross-agent ACP child visibility without global `all` visibility or broad A2A allowlists, and automatic stale harness cleanup/reporting that removes the current manual `ps`/`kill` path. Keep the current permission docs, but do not close this until the remaining behavior is fixed or explicitly scoped into canonical issues.
