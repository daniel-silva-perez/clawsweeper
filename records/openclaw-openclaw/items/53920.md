## Summary

Keep open. Current main still mutates the tracked auth-monitor service template before copying it, and this PR targets the right setup-script bug, but the latest PR patch still misses a real rerun case for templates already mutated to user-specific notification values.

## What This Changes

The PR changes `scripts/setup-auth-system.sh` to copy the auth-monitor service template into the user systemd directory, render `ExecStart` and notification environment values in that copy, and leave the tracked template unchanged.

## Best Possible Solution

Keep the copy-then-render direction, but update the renderer to handle arbitrary existing `Environment=NOTIFY_NTFY=...` and `Environment=NOTIFY_PHONE=...` lines from previously mutated templates, preferably with a focused regression check covering commented placeholders, default uncommented placeholders, and custom uncommented values.

## Reproduction Assessment

Yes. Current main can be reproduced by inspecting or running the setup path with non-empty notification input: it applies `sed -i` to the tracked template before copying it. The remaining PR bug is reproducible with a template line already mutated to a custom `Environment=NOTIFY_NTFY=<value>` or `Environment=NOTIFY_PHONE=<value>`.

## Solution Assessment

No. The PR uses the right high-level fix, but the latest renderer is not yet the narrowest safe solution because it only accepts default already-uncommented values instead of any prior user-rendered notification value.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Handle previously rendered notification values:** `scripts/setup-auth-system.sh:113-116`
  - body: Older setup runs rewrite the tracked template to `Environment=NOTIFY_NTFY=<user topic>` or `Environment=NOTIFY_PHONE=<user phone>`. This helper only accepts the default placeholder values as already-uncommented lines, so rerunning setup from those dirty checkouts still exits with `placeholder not found` before enabling the timer; match any existing `Environment=<key>=...` line instead.
  - confidence: 0.93
- **[P2] Validate before overwriting the user unit:** `scripts/setup-auth-system.sh:72-75`
  - body: The script copies the unrendered template over the user's installed service file before checking `python3` or rendering placeholders. If that preflight or render step fails, a previously working unit can be replaced on disk with the hard-coded template while setup exits before reload/enable; preflight first or render to a temporary file and move it into place only after success.
  - confidence: 0.82
