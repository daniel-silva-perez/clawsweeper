## Summary

Current main still does not implement Workers AI model selection for `openclaw onboard --non-interactive --auth-choice cloudflare-ai-gateway-api-key`. The bundled Cloudflare provider continues to collect only account ID, gateway ID, and API key, then writes an Anthropic Messages endpoint with the Claude Sonnet default. The existing generic custom-provider path can be a workaround, but it is not documented as the supported Cloudflare Workers AI flow and does not satisfy the requested bundled auth-choice behavior.

## Best Possible Solution

Keep this open. The best fix belongs in `extensions/cloudflare-ai-gateway`: add a Workers AI endpoint mode or non-interactive model selector, generate an OpenAI-compatible provider config for `https://gateway.ai.cloudflare.com/v1/<account>/<gateway>/workers-ai/v1` with `@cf/...` model IDs, update `docs/providers/cloudflare-ai-gateway.md` and onboarding automation docs, and add focused tests for the generated provider/model config. If maintainers choose the generic custom-provider route instead, document the exact Workers AI command before closing.
