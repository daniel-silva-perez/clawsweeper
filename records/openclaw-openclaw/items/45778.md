## Summary

Keep open. Current main still lacks both requested platform paths: cron payloads are still limited to `systemEvent` and `agentTurn`, isolated/current/session cron jobs still require `agentTurn` and execute through `runIsolatedAgentJob`, and there is no `OPENCLAW_REPLY_CHANNEL` / `OPENCLAW_REPLY_TARGET` foreground skill env injection. The issue was also refreshed by the reporter on 2026-04-29, so the stale label should not drive cleanup. The right next step is maintainer product/security review rather than an automated fix PR.

## Best Possible Solution

Keep this open for maintainer product/security review. The likely path is a narrow platform seam: either scoped foreground reply-route env injection with explicit channel/account/thread semantics, or an opt-in directExec cron payload limited to approved installed-skill commands that reuses existing cron delivery, run logs, timeouts, and failure-notification behavior without invoking a model.
