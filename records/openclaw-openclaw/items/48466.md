## Summary

Keep this PR open. Current main still lacks the Telegram draft-stream HTML parse-error recovery that this PR targets: preview edit, first send, draft transport, and the outer preview catch still do not retry malformed HTML as plain text, and current draft-stream tests have no regression coverage for the fallback or MESSAGE_NOT_MODIFIED handling. The PR is a focused runtime/test change, not obsolete or superseded by current main.

## Best Possible Solution

Keep this PR open. Refresh it against current main, port the HTML parse-error fallback into `extensions/telegram/src/draft-stream.ts`, preserve current draft/message lifecycle behavior, and land it with focused regression tests for edit fallback, first-send fallback, draft transport fallback, escaped outer-handler parse errors, MESSAGE_NOT_MODIFIED, parse-mode disablement by generation, and forceNewMessage/race behavior. The security-sensitive review should stay focused on Telegram runtime behavior; no supply-chain or CI changes are part of the provided PR file list.
