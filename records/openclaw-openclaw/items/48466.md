## Summary

Keep this PR open. Current main still does not implement the Telegram draft-stream HTML parse-error recovery this PR targets: preview edits, first preview sends, native draft transport, and the outer preview catch still send formatted HTML directly and stop the stream on escaped errors instead of degrading to plain text. The PR is also still open and unmerged, with a focused two-file runtime/test diff rather than obsolete or superseded work.

## Best Possible Solution

Keep this PR open. Refresh it against current main, preserve the existing draft/message lifecycle, and land an equivalent fallback in `extensions/telegram/src/draft-stream.ts` with focused tests for edit fallback, first-send fallback, draft-transport fallback, escaped outer-handler parse errors, MESSAGE_NOT_MODIFIED, per-generation parse-mode disablement, and forceNewMessage race behavior.
