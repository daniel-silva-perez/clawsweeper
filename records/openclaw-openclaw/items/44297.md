## Summary

Current main still keeps Slack external arg-menu registration failure non-fatal, but the degraded mode remains visible only through `logVerbose`; it is not surfaced as a runtime warning, Slack account status/detail, doctor diagnostic, or Slack docs guidance.

## Best Possible Solution

Implement a narrow bundled Slack plugin fix: when external arg-menu options registration fails, keep falling back to static menus, emit a visible runtime warning, and publish a scoped degraded status/detail that `openclaw channels status --probe --json` can expose. Add focused tests for the non-fatal fallback plus the visible signal, and update Slack docs/changelog if the signal is user-facing.
