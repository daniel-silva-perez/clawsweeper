## Summary

Keep open. Current main still keeps Slack external arg-menu registration failure non-fatal, but the degraded mode is only emitted through verbose logging and is not surfaced in Slack account status, doctor diagnostics, or user-facing Slack docs.

## Best Possible Solution

Implement this in the bundled Slack plugin. Preserve the non-fatal fallback, but surface the degraded external-arg-menu state through a visible runtime warning and a scoped account status/detail that `openclaw channels status --probe --json` can expose. Add focused tests proving the fallback remains non-fatal and no longer relies only on `logVerbose`; update Slack docs and changelog if the signal is user-facing.
