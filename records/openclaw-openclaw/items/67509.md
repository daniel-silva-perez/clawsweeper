## Summary

Keep this PR open. Current main still does not implement the real/effective UID root guard this PR adds, and the linked root-execution corruption report #67478 remains open with a concrete DigitalOcean reproduction. The supplied PR diff is narrow CLI source/test work and does not show an obvious supply-chain, CI, dependency, lockfile, or release-script regression.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or an equivalent core fix that guards both `src/entry.ts` and `src/index.ts` before state/config side effects, checks both real and effective UID, preserves intentional container forwarding only, includes focused regression tests, and updates DigitalOcean/install guidance or wrapper behavior for root SSH users.
