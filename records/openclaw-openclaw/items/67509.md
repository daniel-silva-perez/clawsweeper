## Summary

Keep open. Current main still lacks the root guard this PR adds for the #67478 failure mode, but the branch should not merge as-is because it blocks root CLI invocations while the DigitalOcean guides still document root-session setup commands.

## What This Changes

The PR adds a CLI root guard with tests and wires it into the modern and legacy entrypoints to block real or effective UID 0 unless an explicit root override or container-forwarding hints are present.

## Best Possible Solution

Land this PR or an equivalent core fix after maintainer sign-off, with matching DigitalOcean/1-Click guidance: guard both CLI entrypoints before mutable state/config side effects, check both real and effective UID, preserve intentional container forwarding, and document the supported non-root workflow plus any explicit root override.
