## Summary

Keep this PR open. Current main still does not contain an equivalent real/effective UID root guard in either CLI entrypoint, the linked root-execution corruption bug remains open, and this PR is the active focused implementation candidate. Security review did not find CI, dependency, lockfile, release, install-script, package-resolution, or third-party code-execution changes in the PR file list, but the root/container bypass policy and DigitalOcean/root SSH docs still need maintainer review before merge.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or an equivalent core fix that guards both src/entry.ts and src/index.ts before state/config side effects, checks both real and effective UID, preserves intentional container forwarding only, includes focused regression tests, and updates DigitalOcean/install guidance or wrapper behavior so root SSH users get a clear supported command path.
