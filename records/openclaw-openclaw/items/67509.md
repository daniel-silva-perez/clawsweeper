## Summary

Keep open. Current main still has no CLI root guard, and this PR is a focused implementation for the active DigitalOcean/root-state corruption report. The latest diff resolves the earlier entrypoint, container-forwarding, and effective-UID review issues, but it still needs maintainer review because the new global root refusal conflicts with published root-based VPS setup docs and the override/container bypass semantics are product and security policy decisions.

## What This Changes

The PR adds a CLI root guard that blocks real or effective UID 0 unless an explicit root override or container-forwarding hints are present, wires it into both modern and legacy entrypoints, and adds unit coverage.

## Best Possible Solution

Land this PR or an equivalent core fix after maintainer sign-off, with a matching docs/install update: guard both CLI entrypoints before mutable state/config side effects, check both real and effective UID, preserve intentional container forwarding, and document the supported non-root service-user workflow plus any explicit root override.
