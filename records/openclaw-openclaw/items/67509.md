## Summary

Keep this PR open. Current main still does not implement an equivalent real/effective UID root guard for the CLI entrypoints, the linked root-execution corruption bug remains open, and this PR is the active focused implementation candidate. The provided PR surface is source/test-only and does not show CI, dependency, lockfile, release, install-script, package-resolution, or third-party code-execution changes.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or an equivalent core fix that guards both src/entry.ts and src/index.ts before state/config side effects, checks both real and effective UID, preserves intentional container forwarding only, includes focused regression tests, and updates DigitalOcean/install guidance or wrapper behavior for root SSH users.
