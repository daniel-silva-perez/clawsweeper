## Summary

Keep this PR open. Current main still does not contain an equivalent real/effective UID root guard in either CLI entrypoint, and the linked DigitalOcean root-execution corruption report #67478 is still the canonical open bug this PR is trying to fix. Security review of the provided PR surface did not show CI, dependency, lockfile, release, install-script, package-resolution, generated/vendor, or third-party code-execution changes; the remaining decision is maintainer policy around refusing root execution, explicit overrides, container forwarding, and matching install docs.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or an equivalent core fix that guards both src/entry.ts and src/index.ts before state/config side effects, checks both real and effective UID, preserves intentional container forwarding only, includes focused regression tests, and updates DigitalOcean/install guidance or wrapper behavior for root SSH users.
