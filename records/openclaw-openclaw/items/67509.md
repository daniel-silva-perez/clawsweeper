## Summary

Keep open. Current main still does not implement the root guard this PR adds, so the PR is not obsolete; however, it remains a live behavior/policy change that needs maintainer review because a global root refusal conflicts with the current DigitalOcean/VPS setup docs that still run OpenClaw commands from a root SSH session.

## What This Changes

The PR adds a CLI root guard with tests, then wires it into the modern and legacy CLI entrypoints to block real or effective UID 0 unless an explicit root override or container-forwarding hints are present.

## Best Possible Solution

Land this PR or an equivalent core fix after maintainer sign-off, with matching DigitalOcean/1-Click guidance: guard both CLI entrypoints before mutable startup side effects, check both real and effective UID, preserve intentional container forwarding, and document the supported non-root workflow plus any explicit root override.
