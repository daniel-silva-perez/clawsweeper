## Summary

Keep open. Current main still has no CLI-wide root guard, and the linked root-state corruption bug remains a real implementation target; however, this PR should not merge as-is until maintainers reconcile the new root refusal with the existing DigitalOcean/VPS setup docs and root-execution policy.

## What This Changes

The PR adds a CLI root guard with unit tests and wires the modern and legacy entrypoints to block real or effective UID 0 unless an explicit root override or container-forwarding hints are present.

## Best Possible Solution

Land this PR or an equivalent core fix after maintainer sign-off, with matching DigitalOcean/1-Click guidance: guard both CLI entrypoints before mutable state/config side effects, check real and effective UID, preserve intentional container forwarding, and document the supported non-root service-user workflow plus any explicit root override.
