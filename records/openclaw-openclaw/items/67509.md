## Summary

Keep this PR open. Current main still lacks a root-execution guard, and the PR targets the real DigitalOcean/root-state corruption report, but it is not merge-ready because the current DigitalOcean setup docs still instruct users to SSH as root and run root `openclaw` commands that this guard would now reject.

## What This Changes

The PR adds a CLI root guard with real/effective UID checks and root/container escape hatches, wires it into the modern and legacy CLI entrypoints, and adds unit coverage.

## Best Possible Solution

Land this PR or an equivalent core fix after maintainer sign-off, with both CLI entrypoints guarded before mutable state/config side effects, real and effective UID coverage retained, intentional container forwarding preserved, public install guidance updated to the chosen non-root or wrapper workflow, and a user-facing changelog entry if the core behavior ships.
