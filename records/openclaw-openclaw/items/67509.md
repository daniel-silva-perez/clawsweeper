## Summary

Keep open. Current main still does not implement the root guard, so this PR is not obsolete, but the patch needs maintainer review before merge because it globally refuses root CLI execution while the current DigitalOcean setup docs still tell users to SSH as root and run OpenClaw commands from that shell.

## What This Changes

The PR adds a CLI root guard module with tests and wires the modern and legacy entrypoints to block real or effective UID 0 unless an explicit override or container-forwarding hints are present.

## Best Possible Solution

Land this PR or an equivalent core fix after maintainer sign-off, with matching DigitalOcean/1-Click guidance: guard both CLI entrypoints before mutable startup side effects, check both real and effective UID, preserve intentional container forwarding, and document the supported non-root workflow plus any explicit root override.
