## Summary

Keep this PR open. Current main still does not implement the proposed root-execution guard in either CLI entrypoint, and the linked DigitalOcean corruption issue #67478 remains open. The PR is a focused implementation candidate and the discussion shows the author addressed prior review feedback around static imports, legacy entry coverage, container forwarding, and effective UID handling.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or an equivalent core fix that guards both src/entry.ts and src/index.ts before state/config side effects, checks real and effective UID, preserves intentional container forwarding only, includes focused regression tests, and updates DigitalOcean/install guidance for root SSH users.
