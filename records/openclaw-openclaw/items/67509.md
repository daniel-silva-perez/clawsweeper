## Summary

Keep open. Current main still lacks an equivalent root-execution guard in the modern and legacy CLI entrypoints, while the linked DigitalOcean bug #67478 remains open. This PR is a focused implementation candidate, and its discussion shows the author addressed prior review feedback around static imports, legacy entry coverage, container-forwarding bypass, and effective UID handling. The security review did not show supply-chain, CI, dependency, lockfile, publishing, or workflow changes in the provided PR file list.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or an equivalent core fix that guards both `src/entry.ts` and `src/index.ts` before state/config side effects, checks real and effective UID, preserves intentional container forwarding, includes focused regression tests, and updates DigitalOcean/install guidance for root SSH users.
