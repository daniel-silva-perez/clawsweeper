## Summary

Keep open. Current main still lacks the root-execution guard this PR adds, and the linked root-state corruption report remains open; however, the PR still needs maintainer review because it would make the current DigitalOcean/VPS docs' root-shell setup commands fail unless those docs or the guard policy are reconciled.

## What This Changes

The PR adds a CLI root guard with real/effective UID checks and escape hatches, wires it into the modern and legacy entrypoints, and adds unit coverage.

## Best Possible Solution

Land this PR or an equivalent core fix after maintainer sign-off: guard both CLI entrypoints before mutable state/config side effects, keep real and effective UID coverage, preserve intentional container forwarding, and update DigitalOcean/1-Click guidance to match the supported non-root service-user or explicit override workflow.
