## Summary

Keep this PR open. It is maintainer-authored and carries the protected `maintainer` label, so cleanup automation should not close it. Current `main` still lacks the bounded startup retry loop proposed here; memory-core still defers missing startup cron reconciliation to later heartbeat/cron-triggered runtime reconciliation. The linked bug #72841 and overlapping implementation PR #73170 are also still open, so the useful path is maintainer review, consolidation, and landing one fix rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or consolidate one bounded startup retry implementation in `extensions/memory-core/src/dreaming.ts`, keep the regression test proving cron can become available after `gateway_start` without heartbeat traffic, address the timer cleanup/stale-config review notes if still present on the latest head, then close #72841 and supersede #73170 if appropriate.
