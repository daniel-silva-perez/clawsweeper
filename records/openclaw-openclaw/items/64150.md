## Summary

Keep this PR open. Current main still lacks the requested Anthropic OAuth `Glob`/`Grep`/`LS` dispatch support, and the PR's submitted implementation is not merge-ready because it uses nonexistent dependency exports and would add filesystem discovery tools outside OpenClaw's workspace/sandbox and policy catalog handling.

## Best Possible Solution

Keep this PR open as a real but incomplete implementation candidate. The best maintainer path is a revised patch that uses the pinned dependency factory API (`createFindTool`, `createGrepTool`, `createLsTool`) or OpenClaw-owned guarded equivalents, routes the tools through workspace/sandbox filesystem policy, adds them deliberately to the core tool catalog/profile/docs if maintainers want them in `group:fs`, and adds regression tests for OAuth `Glob` -> `find`, `Grep` -> `grep`, `LS` -> `ls`, plus allow/deny and `tools.fs.workspaceOnly` behavior.
