## Summary

Keep this PR open. Current main still lacks the requested Anthropic OAuth `Glob` to `find` dispatch support, but the submitted patch is not merge-ready because it imports symbols the pinned dependency does not export and adds filesystem discovery tools without the existing sandbox, workspace, and policy-catalog protections.

## What This Changes

The PR adds Claude Code OAuth tool-name aliases and unconditionally registers pi-coding-agent `find`/`grep`/`ls` tools in OpenClaw's default coding tool set.

## Best Possible Solution

Revise the implementation so OpenClaw either creates guarded `find`/`grep`/`ls` tools from the pinned factory API or implements OpenClaw-owned equivalents, routes them through the same sandbox/workspace filesystem policy as existing file tools, deliberately adds them to the core catalog/profile/docs surface if maintainers want them in `group:fs`, and covers OAuth name mapping plus allow/deny and `tools.fs.workspaceOnly` behavior with targeted tests.
