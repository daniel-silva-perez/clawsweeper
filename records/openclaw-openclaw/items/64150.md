## Summary

Keep this PR open. Current main still has the same Anthropic OAuth Claude Code tool-name gap the PR is trying to fix: `Glob` is not mapped to a local `find` tool, `LS` is not advertised, and `createOpenClawCodingTools()` still does not register `find`/`grep`/`ls`. The submitted patch is not safe to merge as-is because it imports nonexistent dependency symbols and would add filesystem discovery tools outside the existing workspace/sandbox guard and tool-policy catalog path.

## Best Possible Solution

Keep this PR open as a real but incomplete implementation candidate. The best maintainer path is a revised patch that uses the dependency factory API or OpenClaw-owned guarded equivalents, applies workspace/sandbox filesystem policy, updates the core tool catalog/profile/docs if maintainers want these tools in `group:fs`, and adds regression coverage for OAuth `Glob` -> `find`, `Grep` -> `grep`, `LS` -> `ls`, plus policy and workspaceOnly behavior.
