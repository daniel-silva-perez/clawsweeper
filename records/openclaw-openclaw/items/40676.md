## Summary

Keep this PR open. Current main still does not implement the two central fixes: Discord voice auto-join is still READY-only, and the join path still calls `joinVoiceChannel` without stale `@discordjs/voice` registry cleanup. The PR is still useful but needs a rebase/rewrite into the current `extensions/discord/**` plugin layout and must address the review risks in its stale-connection and session-key changes.

## Best Possible Solution

Keep this PR or the linked bug open until a rebased, focused Discord plugin fix lands. The best path is to update `extensions/discord/src/voice/manager.ts`, `extensions/discord/src/voice/manager.runtime.ts`, and `extensions/discord/src/monitor/provider.ts` to add a `DiscordVoiceResumedListener`, register it beside the READY listener, add account-safe stale connection cleanup before `joinVoiceChannel`, and add targeted tests for RESUMED listener registration plus stale cleanup behavior. The session-key changes should be split out or removed unless separately designed and tested.
