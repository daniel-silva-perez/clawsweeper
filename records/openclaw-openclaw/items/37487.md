## Summary

Current main still lacks the central requested contract: per-profile browser proxy configuration plus per-agent default browser profile binding for concurrent browser runs. Named browser profiles, separate managed profile CDP/user-data settings, explicit per-call `profile`, and global `browser.extraArgs` proxy flags exist, but there is still no `browser.profiles.<name>.proxy`, no `agents.defaults.browser.profile` / `agents.list[].browser.profile`, and no profile-level proxy diagnostic. Related PR #42806 is closed unmerged, and the `m13v/browser-lock` discussion link is useful prior art rather than shipped OpenClaw behavior.

## Best Possible Solution

Keep this issue open. The best path is a maintainer-reviewed first-class browser/agent contract: add secret-aware `browser.profiles.<name>.proxy` for locally managed profiles, apply it only at managed browser launch, expose redacted diagnostics such as `proxyConfigured`, add `agents.defaults.browser.profile` and `agents.list[].browser.profile` with clear sub-agent inheritance rules, and document/test concurrent isolated profiles. Use PR #42806 and https://github.com/m13v/browser-lock as prior art, not as shipped fixes.
