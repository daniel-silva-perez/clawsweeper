## Summary

Keep open. Current main still has named isolated browser profiles and a global `browser.extraArgs` proxy workaround, but it does not implement the requested first-class `browser.profiles.<name>.proxy` contract or `agents.defaults.browser.profile` / `agents.list[].browser.profile` routing for concurrent agent browser runs. Related PRs #37534 and #42806 are closed unmerged, and the linked `m13v/browser-lock` project is useful prior art rather than shipped OpenClaw behavior.

## Best Possible Solution

Keep this open for maintainer-reviewed design. The strongest path is a secret-aware per-profile proxy contract for locally managed browser profiles, redacted profile/status diagnostics, explicit agent and sub-agent browser profile inheritance rules, and docs/tests that cover concurrent isolated profiles without changing remote CDP attach semantics unexpectedly.
