## Summary

Current main still lacks the central requested contract: per-profile browser proxy configuration plus per-agent default browser profile binding for concurrent browser runs. Named browser profiles and explicit per-call profile routing exist, and global `browser.extraArgs` can pass a Chrome proxy flag, but there is no `browser.profiles.<name>.proxy`, no `agents.defaults.browser.profile` / `agents.list[].browser.profile`, no profile-level proxy diagnostic, and the related PR #42806 is closed unmerged. The `m13v/browser-lock` link in the discussion is useful prior art for isolated user-data dirs, per-instance CDP ports, serialized launch, and per-process proxy args, but it is not a shipped OpenClaw implementation.

## Best Possible Solution

Keep this issue open. The best path is a maintainer-reviewed first-class browser/agent contract: add secret-aware `browser.profiles.<name>.proxy` for locally managed profiles, apply it only at managed browser launch, expose redacted proxy diagnostics such as `proxyConfigured`, add `agents.defaults.browser.profile` and `agents.list[].browser.profile` with clear sub-agent inheritance rules, and document/test concurrent isolated profiles. Use PR #42806 and https://github.com/m13v/browser-lock as prior art, not as shipped fixes.
