## Summary

Keep open. Current main at 1b25dcf still lacks the central requested contract: first-class per-profile browser proxy configuration and per-agent default browser profile binding for concurrent browser runs. OpenClaw has named browser profiles, isolated managed profile data/ports, explicit per-call `profile`, and a global `browser.extraArgs` proxy workaround, but not `browser.profiles.<name>.proxy`, not `agents.defaults.browser.profile` / `agents.list[].browser.profile`, and not a profile-level `proxyConfigured` diagnostic. Related PRs #37534 and #42806 are both closed unmerged; the posted `m13v/browser-lock` implementation is useful prior art, not shipped OpenClaw behavior.

## Best Possible Solution

Keep this issue open for maintainer-reviewed design and implementation. The best path is to add a secret-aware `browser.profiles.<name>.proxy` contract for locally managed profiles, apply it only during managed browser launch, expose redacted diagnostics such as `proxyConfigured`, add `agents.defaults.browser.profile` and `agents.list[].browser.profile` with clear sub-agent inheritance rules, and document/test concurrent isolated profiles. Use #37534, #42806, and `m13v/browser-lock` as prior art, not as shipped fixes.
