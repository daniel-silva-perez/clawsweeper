## Summary

Keep this open. Current main still lacks the Slack exact-message read path: `messageId` is not accepted by the Slack read dispatcher, not parsed by the runtime, and not used by `readSlackMessages`, so the reported thread-reply lookup remains unresolved.

## What This Changes

The issue requests Slack `message(action: "read")` support for `messageId`, including exact lookup/filtering for thread replies when `messageId` and `threadId` are provided.

## Best Possible Solution

Add exact-message read support in the Slack plugin path: expose and forward `messageId` for `read`, make `readSlackMessages` return only the requested timestamp for both threaded and channel reads, preserve existing `threadId` behavior, and update focused tests plus the public message docs.
