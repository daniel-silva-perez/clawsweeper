## Summary

Keep this open. Current main still does not implement group pairing: group policy remains open/disabled/allowlist, pairing remains DM-only in config, schema, docs, shared resolver, and CLI, and Telegram/WhatsApp group handling still either blocks or processes groups without a pending owner-approval state.

## What This Changes

Request to add a group-pairing flow with owner notification, CLI approve/reject, and optional read-only/reader-agent handling for unconfigured Telegram and WhatsApp groups.

## Best Possible Solution

Keep this open for maintainer/product design. The best path is an additive group-pairing contract: extend config/schema/docs with `groupPolicy: "pairing"`, persist pending group approvals keyed by channel/account/group, add CLI approve/reject operations, emit owner-visible group-add or first-seen-unconfigured-group notifications, and wire Telegram/WhatsApp through shared state. Coordinate the optional read-only reader-agent behavior with #42400 or split it into a narrower follow-up.
