## Summary

Keep open. Current main still has DM-only pairing and explicit group open/disabled/allowlist handling, with no owner notification on group add, pending group approval state, group approve/reject command, or pending read-only group routing. The existing maintainer-side and bot reviews reached the same conclusion, and the related open work is adjacent rather than a canonical replacement.

## What This Changes

Request adds a cross-channel group pairing flow with owner notification, approve/reject controls, and optional read-only reader-agent handling for unconfigured Telegram/WhatsApp groups.

## Best Possible Solution

Define an additive cross-channel group-pairing contract: extend config/schema/docs, persist pending approvals by channel/account/group, send owner-visible group-add notifications, add authorized approve/reject operations, and wire Telegram plus WhatsApp through shared policy state. Treat reader-agent observation as separately scoped unless maintainers explicitly include it in the approval contract.
