## Summary

Keep open. Current main still has DM-only pairing and explicit group open/disabled/allowlist behavior; it does not notify an owner on group add, persist pending group approvals, expose group approve/reject commands, or route unconfigured groups through a pending read-only agent. The related open work is adjacent, not a canonical replacement.

## What This Changes

Request an additive group pairing flow that notifies the owner when a bot is added to a group, lets the owner approve or reject that group, and optionally observes pending groups in read-only mode.

## Best Possible Solution

Define a cross-channel group-pairing contract: extend config/schema/docs, persist pending approvals by channel/account/group, send owner-visible group-add notifications, add authorized approve/reject operations, and wire Telegram plus WhatsApp through shared policy state. Treat reader-agent observation as a separately scoped mode unless maintainers explicitly include it in the approval contract.
