## Summary

Keep open. Current main still implements DM-only pairing while group handling remains open, disabled, or allowlist-based; there is no pending group approval state, owner notification on group add, group approve/reject CLI, or pending read-only group routing. The related #42400 and #8495 work is adjacent, but does not provide the requested cross-channel group-pairing approval flow.

## What This Changes

Request a cross-channel group-pairing flow that notifies the owner when a bot is added to a group, supports approve/reject decisions, and optionally routes unapproved groups through a read-only reader agent.

## Best Possible Solution

Define an additive cross-channel group-pairing contract: extend config/schema/docs, persist pending group approvals keyed by channel/account/group, send owner-visible group-add notifications, add authorized CLI or inline approve/reject operations, and wire Telegram/WhatsApp through shared policy state. Treat reader-agent or silent observation as a separately scoped mode unless maintainers explicitly choose to combine it with approval.
