## Summary

Keep open. Current main still implements DM-only pairing while group access remains limited to open, disabled, or allowlist behavior; there is no owner notification on group add, pending group approval state, group approve/reject CLI, or read-only pending-group routing. The existing maintainer-side Codex comment reached the same conclusion, and the related open work only covers adjacent group suppression/read-only observation rather than group pairing approval.

## What This Changes

Add a group pairing mode that notifies the owner when a bot is added to a group, holds unconfigured groups pending approval, supports approve/reject controls, and can optionally route pending groups through a read-only reader agent.

## Best Possible Solution

Define an additive cross-channel group-pairing contract: extend config/schema/docs, persist pending group approvals keyed by channel/account/group, deliver owner-visible group-add notifications, add authorized CLI or inline approve/reject operations, and wire Telegram/WhatsApp through shared policy state. Treat reader-agent or silent observation as a separately scoped mode unless maintainers decide it belongs in the same approval contract.
