## Summary

Keep open. Current main still lacks an injected `api.runtime.llm` or `api.runtime.llm.complete` Plugin SDK runtime helper, and the existing simple-completion helpers are public subpaths rather than the requested managed plugin runtime surface. The open #64294 PR is the active implementation candidate, so this issue should remain open until that PR lands or maintainers choose a different API shape.

## What This Changes

Request to add a Plugin SDK LLM inference helper so plugins can make provider-agnostic model calls through OpenClaw's existing inference stack instead of calling provider APIs directly.

## Best Possible Solution

Review and either land #64294 or replace it with the agreed narrow SDK/runtime implementation: a backwards-compatible `api.runtime.llm` completion surface wired through core model resolution, auth, fallback/rate-limit paths, test mocks, context-engine support if intended, and Plugin SDK docs.

## Reproduction Assessment

Not applicable as a feature request rather than a runtime bug. The missing capability is verifiable on current main by inspecting the plugin runtime type, runtime builder, docs, and targeted source search for `api.runtime.llm`.

## Solution Assessment

Unclear pending maintainer API review. The host-mediated runtime boundary is the right direction, and a single-shot completion API with no tool execution looks like a safer v1, but naming and budget/cost/permission semantics should be settled before merge.
