## Summary

Keep open. Current main does not expose an injected `api.runtime.llm` or `api.runtime.llm.complete` surface, and the related implementation candidate #64294 is still open and unmerged. Existing simple-completion helpers are lower-level SDK exports, not the requested plugin runtime API.

## Best Possible Solution

Keep this issue open as the canonical SDK/API request. Review, revise, and either land or replace #64294 with an additive plugin runtime LLM API, documented in `docs/plugins/sdk-runtime.md`, exported through the SDK type surface, and covered by tests for model alias/auth-profile resolution, multi-agent binding, and plugin attribution.

## Risks / Open Questions

- A plugin-facing LLM runtime would expose configured model credentials and token spend to plugins, so maintainers should decide permission, active-agent binding, observability, cost attribution, and concurrency semantics before landing it.
- Closing now would hide the canonical API request while the only visible implementation path remains an open PR.
