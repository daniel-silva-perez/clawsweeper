## Summary

Keep this PR open. Current main still has the reported ClawDock gap: `_clawdock_compose` only passes the base compose file and optional `docker-compose.extra.yml`, while the Docker setup path can create and retain `docker-compose.sandbox.yml` for opt-in sandbox deployments. The PR is a narrow additive fix that matches the existing optional-compose-file pattern, and I found no blocking functional or security issue in the diff.

## What This Changes

The PR updates `scripts/clawdock/clawdock-helpers.sh` so `_clawdock_compose` appends `docker-compose.sandbox.yml` to the `docker compose -f` list when that file exists.

## Best Possible Solution

Land a narrow helper change that loads `docker-compose.sandbox.yml` after `docker-compose.extra.yml` when present, ideally with a small helper-level assertion or smoke check, while leaving the broader `docker-compose.override.yml` behavior to #49909 or a deliberate follow-up.

## Reproduction Assessment

Yes. On current main, source inspection gives a high-confidence reproduction path: enable Docker sandbox setup so `docker-compose.sandbox.yml` exists, then any ClawDock command using `_clawdock_compose` will omit that overlay because the helper never appends it.

## Solution Assessment

Yes. The proposed conditional append is the narrowest maintainable fix for the sandbox overlay because it preserves the established `extra` overlay pattern and Docker Compose's ordered `-f` merge behavior; it should not be expanded in this PR to solve the separate override-file issue.
