## Summary

Keep this PR open. Current main still does not load `docker-compose.sandbox.yml` through ClawDock, while the Docker setup path can create that overlay for sandbox-enabled installs. The PR is a narrow, plausible fix rather than obsolete work.

## Best Possible Solution

Keep the PR open and review it as a narrow script bug fix. The likely best implementation is to include `docker-compose.sandbox.yml` in `_clawdock_compose()` after the base and extra compose files, with a small regression test or scripted proof for optional overlay loading and no broader Docker or CI changes.
