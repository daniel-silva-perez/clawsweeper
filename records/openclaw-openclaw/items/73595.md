## Summary

Keep open. Current main still lacks the requested final outbound sanitizer for leaked runtime XML scaffolding, and the Discord delivery path named in the report still has no plain-text sanitizer wiring. Existing sanitizers cover `<final>` tags, reasoning tags, or OpenClaw runtime-context markers, but they do not remove `<system-reminder>...</system-reminder>` bodies or `<previous_response>...</previous_response>` tags before Discord sends user-visible text.

## Best Possible Solution

Add a shared final outbound sanitizer at the channel delivery boundary after message_sending hooks and presentation rendering but before every send path. Seed it with explicit internal-only wrapper removal for `<system-reminder>...</system-reminder>` and `<previous_response>...</previous_response>` including empty/self-closing variants, wire Discord to the same sanitization expectations as other text channels, and add regression tests for the helper plus Discord/shared delivery after hook rewrites.
