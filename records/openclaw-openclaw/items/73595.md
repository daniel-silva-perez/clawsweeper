## Summary

Keep open. Current main still lacks the requested final outbound sanitizer for leaked runtime XML scaffolding, and the Discord delivery path named in the report still has no plain-text sanitizer wiring. The follow-up comments add useful evidence: the reporter traced the tags to Claude Code CLI harness scaffolding, and another commenter independently identified Discord's missing sanitizer plus the `<previous_response>` regex miss.

## Best Possible Solution

Keep this issue open. Implement a shared final outbound sanitizer at the channel delivery boundary after `message_sending` hooks and presentation rendering, before `sendPayload`, `sendFormattedText`, or `sendTextChunks`. Seed it with explicit internal-only wrapper removal for `<system-reminder>...</system-reminder>` and `<previous_response>...</previous_response>` including empty/self-closing variants, wire Discord to the same sanitization expectations as other text channels, and add focused helper plus Discord/shared delivery regression tests, including hook-rewritten content.
