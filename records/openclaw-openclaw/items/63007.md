## Summary

Keep this PR open. Current main still builds outbound session identity for gateway send, but it does not pass that identity into `message_sending`, does not expose `agentId` on the public message-hook context, and still collapses hook cancellation into an empty delivery result that gateway send reports as `No delivery result`. The same-author umbrella #63011 is also open, so this draft remains an implementation candidate rather than cleanup-close material.

## Best Possible Solution

Keep this draft open for maintainer review, or replace it with an equivalent focused patch reviewed alongside #63011. The implementation should add optional `agentId` to the public message hook context, pass outbound `agentId`/`sessionKey` into `message_sending`, preserve cancellation reason through a narrow delivery callback or blocked-result shape, surface explicit guarded gateway-send denial, update docs/types for the public contract, and add focused gateway/outbound tests.
