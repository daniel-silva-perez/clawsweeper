## Summary

Keep this draft open. Current main still builds outbound session identity for gateway sends, but `message_sending` does not receive `agentId`, delivery cancellation still collapses to an empty delivery result, and gateway send still reports `No delivery result`. The author also provided useful Telegram live verification, and the same-author umbrella #63011 remains open, so this PR is still an implementation candidate rather than cleanup-close material.

## Best Possible Solution

Keep this draft open for maintainer review, or replace it with an equivalent focused patch reviewed alongside #63011. The implementation should add optional `agentId` to the public message hook context, pass outbound `agentId`/`sessionKey` into `message_sending`, preserve cancellation reason through delivery, surface explicit guarded gateway-send denial, update public docs/types as needed, and add focused outbound/gateway regression tests.
