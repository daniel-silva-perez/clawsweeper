## Summary

Keep open. Current main still builds an outbound session for gateway send but does not pass that identity into `message_sending`, does not expose `agentId` on the public message-hook context, and still turns hook-cancelled delivery into `No delivery result`. The PR remains a useful draft implementation candidate with live Telegram verification from the author, but it is stale against the current hook type/mapper layout and should be reviewed alongside the open canonical outbound-send work in #63011 and the broader hook-identity request in #18571.

## What This Changes

The PR branch adds optional outbound `agentId`/`sessionKey` plumbing to `message_sending`, reports hook cancellation through outbound delivery, surfaces guarded gateway-send denial explicitly, and adds focused gateway/outbound mapper tests.

## Best Possible Solution

Rebase this draft, or replace it with an equivalent focused patch, so the current public hook context gains optional `agentId`, outbound delivery passes `agentId`/`sessionKey` into `message_sending` without dropping existing correlation fields, hook cancellation is represented explicitly for gateway send, and the public docs plus focused tests cover the contract.
