## Summary

Keep open. Current main still applies `extra_body` after the DeepSeek V4 payload wrapper without removing stale `reasoning_content`, and this PR is a narrow implementation candidate for the reported DeepSeek 400 regression rather than something already implemented or superseded.

## What This Changes

This PR adds OpenAI-completions `extra_body` post-merge cleanup that deletes `reasoning_content` from messages when the final payload has `thinking.type: "disabled"`, plus a DeepSeek V4 regression test.

## Best Possible Solution

Land a tested final-payload consistency fix so DeepSeek V4 disabled-thinking requests never send stale `reasoning_content`, while preserving enabled-thinking replay and coordinating any broader `thinking` collision cleanup through the related tracking issue.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is DeepSeek V4 with `extra_body: { thinking: { type: "disabled" } }` while the resolved thinking level is not `off`; current main can leave an assistant `reasoning_content` field after the final `thinking` value becomes disabled.

## Solution Assessment

Unclear. The PR is a plausible narrow defensive fix for the final payload shape, but maintainer review should decide whether generic `extra_body` cleanup is the best scope or whether the cleanup belongs in a DeepSeek-specific wrapper alongside the alias fix in #74418.
