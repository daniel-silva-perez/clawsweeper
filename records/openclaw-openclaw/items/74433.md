## Summary

Keep open. The latest PR head is still a useful but incomplete fix: it adds doctor and security-audit diagnostics, but the linked report asks for the confusing status/gateway-status path too, and the new detector can warn on intentional env-backed config that is not a real token split.

## What This Changes

The PR adds a doctor warning, a `gateway.env_token_overrides_config` security-audit finding, and two doctor-security tests for configurations with both `OPENCLAW_GATEWAY_TOKEN` and `gateway.auth.token`.

## Best Possible Solution

Land a shared, redacted local-gateway auth-shadow diagnostic that reads the trusted state-dir `.env`, ignores configs intentionally backed by the same env credential, surfaces the same remediation in doctor, status, gateway status, and security audit, and updates tests/docs/changelog without changing auth precedence or printing secrets.
