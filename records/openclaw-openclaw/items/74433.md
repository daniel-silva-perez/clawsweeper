## What This Changes

The PR adds a doctor security warning, a `gateway.env_token_overrides_config` security-audit finding, and doctor-security tests for cases where `OPENCLAW_GATEWAY_TOKEN` and configured `gateway.auth.token` coexist.

## Best Possible Solution

Land a shared redacted local-gateway auth-shadow diagnostic that reads the trusted state-dir `.env`, gates on configured local `gateway.auth.token`, surfaces consistently in doctor/status/gateway status/security audit, preserves existing auth precedence, and documents the remediation without printing or changing secrets.
