## Summary

Keep open. The PR addresses a real diagnostic gap, but the current branch is still too broad and incomplete for the linked local-gateway auth-shadow problem: it keys off generic process environment/config presence, can false-positive on env-backed config tokens, and does not add the requested status/gateway-status surfaces.

## What This Changes

The PR adds a doctor warning, a `gateway.env_token_overrides_config` security-audit finding, and doctor-security tests for cases where `OPENCLAW_GATEWAY_TOKEN` and `gateway.auth.token` are both configured.

## Best Possible Solution

Land a shared, redacted local-gateway auth-shadow diagnostic that reads the trusted state-dir `.env`, scopes detection to local gateway mode with a configured non-env-backed `gateway.auth.token`, surfaces the same remediation in doctor, status, gateway status, and security audit, and documents the behavior without changing credential precedence or printing secrets.
