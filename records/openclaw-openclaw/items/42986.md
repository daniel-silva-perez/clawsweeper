## Summary

Keep open. Current main still does not wire Telegram into the native subagent lifecycle hook path required by `sessions_spawn({ mode: "session", thread: true })`. Telegram has conversation/thread binding infrastructure and ACP topic binding, but the bundled Telegram entry lacks `registerFull` hook registration for `subagent_spawning`, `subagent_delivery_target`, and `subagent_ended`. The focused implementation candidate remains PR #42707, so this issue should stay open for maintainer review of that PR or an equivalent fix.

## Best Possible Solution

Keep this issue open and review/land PR #42707 or an equivalent Telegram plugin-boundary implementation. The fix should add Telegram `registerFull` wiring for `subagent_spawning`, `subagent_delivery_target`, and `subagent_ended`, use the existing Telegram thread-binding manager, honor `channels.telegram.threadBindings.spawnSubagentSessions`, clean up bindings on failed starts/end, and keep ACP topic binding behavior distinct in tests and docs.
