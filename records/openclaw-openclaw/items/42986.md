## Summary

Keep open. Current main still lacks Telegram native subagent lifecycle hook wiring, so `sessions_spawn({ mode: "session", thread: true })` cannot auto-bind a Telegram topic/conversation for native subagent sessions. Telegram has conversation binding infrastructure and ACP topic binding, but the requested native subagent spawn path remains unimplemented; PR #42707 is still the focused implementation candidate.

## Best Possible Solution

Keep this issue open and review/land PR #42707 or an equivalent Telegram plugin-boundary implementation. The fix should add Telegram full hook wiring for `subagent_spawning`, `subagent_delivery_target`, and `subagent_ended`, reuse the existing Telegram thread-binding manager, honor `channels.telegram.threadBindings.spawnSubagentSessions`, clean up bindings on failed starts/end, and add focused tests/docs so ACP topic binding and native subagent thread binding remain distinct.
