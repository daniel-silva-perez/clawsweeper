## Summary

Current main still does not implement Telegram native thread-bound `sessions_spawn({ mode: "session", thread: true })`. Telegram has topic routing, ACP topic binding, and a generic session-binding adapter, but the bundled Telegram entry still does not register the native subagent lifecycle hooks that the core spawn path requires. The open PR #42707 remains the focused implementation candidate, so this issue should stay open for maintainer review or an equivalent fix.

## Best Possible Solution

Keep this issue open and review or land PR #42707, or an equivalent Telegram plugin-boundary fix. The implementation should add Telegram `registerFull` wiring for `subagent_spawning`, `subagent_delivery_target`, and `subagent_ended`, use the existing Telegram thread-binding manager/session-binding adapter, honor `channels.telegram.threadBindings.spawnSubagentSessions`, clean up failed starts and ended sessions, and update tests/docs so Telegram native subagent binding is distinct from ACP topic binding.
