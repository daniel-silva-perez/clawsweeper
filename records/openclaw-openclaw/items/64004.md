## Summary

Keep open. Current main still contains the unresolved Control UI responsiveness surface: Sessions navigation still refreshes through `sessions.list`, background/session events still trigger more session refreshes, global node polling still calls `node.list`, and Overview/Usage/Cron still call the slow endpoint families named by the reporter. Related #71631 is broad and open, and #69454 is an open, unmerged implementation candidate rather than shipped behavior.

## Best Possible Solution

Keep this issue open as a concrete Control UI performance bug. The best maintainer path is to review and either land #69454 or an equivalent focused fix: scope node/device loading to the Nodes surface, debounce or stale-guard `sessions.changed` refreshes, bound chat/session list refreshes, lazy-load or isolate slow usage/cron panels, and add click-to-render plus RPC enqueue/send/receive/render timing instrumentation. Close only after a merged change proves the Sessions view updates promptly even when `node.list`, `cron.runs`, `sessions.usage`, or `usage.cost` are slow.
