## Summary

Keep this PR open. Current main does not implement the proposed doctor warning for a root/default config path diverging from a service-user gateway config path, and the PR is paired with an open same-author bug report that it closes. The supplied PR is focused, but the review-thread concerns about false-positive path advice and unsafe operator guidance mean the right maintainer path is review/rework plus tests, not conservative cleanup closure.

## Best Possible Solution

Keep the PR and paired bug open. The best fix is a robust doctor preflight or gateway/service audit that compares doctor’s actual resolved config path with the managed gateway’s effective config path, respects `OPENCLAW_CONFIG_PATH`/`OPENCLAW_STATE_DIR`/legacy config names, avoids destructive deletion advice, and adds focused regression tests for root HOME plus service-user config scenarios.
