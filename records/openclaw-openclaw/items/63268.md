## Summary

Keep this PR open. Current main at 955b4df0939f90b4b6de5e7e6f3149dd125e1e50 still does not implement the proposed `doctor --fix` warning for root/default config paths diverging from the running gateway/service-user config. There is adjacent CLI-vs-service config-path drift reporting under `openclaw gateway status`, but it is not on the doctor repair path. The PR is also paired with the open same-author bug #63265, so the right outcome is maintainer review/rework rather than cleanup closure.

## Best Possible Solution

Keep the PR and linked bug open. The best fix is a doctor preflight or shared service/config audit that compares doctor’s actual resolved config path with the managed gateway’s effective config path, respects `OPENCLAW_CONFIG_PATH`, `OPENCLAW_STATE_DIR`, and supported legacy config filenames, avoids destructive deletion advice, and adds focused regression tests for root `HOME` plus service-user gateway configs.
