## Summary

Keep this PR open for maintainer follow-up. Current main still does not add a `doctor --fix` warning for the root-vs-service-user config-path divergence reported in the paired bug, but the proposed patch is not merge-ready because it guesses from hard-coded file existence rather than comparing doctor’s actual resolved config path with the managed gateway’s effective config path.

## What This Changes

The PR adds a doctor config preflight that checks `/root`, `/home/node`, and `/home/openclaw` config file existence and emits warning/remediation text before reading the config snapshot.

## Best Possible Solution

Implement a doctor/service config audit that compares doctor’s actual resolved config file with the managed gateway’s effective config path from service metadata, respects `OPENCLAW_CONFIG_PATH`, `OPENCLAW_STATE_DIR`, `OPENCLAW_HOME`, and legacy candidates, and emits non-destructive guidance unless it can prove a config is stale and inactive.

## Reproduction Assessment

Yes. The current-main gap is reproducible by source inspection on the `doctor --fix` path, and the PR false-positive path follows from `HOME=/root` plus `OPENCLAW_CONFIG_PATH=/home/node/.openclaw/openclaw.json`: config IO targets the override while the PR warning still compares against `HOME/.openclaw`.

## Solution Assessment

No. A hard-coded existence scan for `/root`, `/home/node`, and `/home/openclaw` is not the best fix because it can warn on stale files and recommend the wrong override or deletion; a shared service/config-path audit is narrower and more maintainable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Use the resolved doctor config path:** `src/commands/doctor-config-preflight.ts:141`
  - body: This preflight derives `doctorConfigDir` from `resolveHomeDir()` even though doctor reads/writes through the config resolver. With `HOME=/root` and `OPENCLAW_CONFIG_PATH=/home/node/.openclaw/openclaw.json`, the command is already targeting the service-user file but this line still warns that doctor is writing to `/root/.openclaw`.
  - confidence: 0.9
- **[P2] Verify the active gateway config before suggesting overrides:** `src/commands/doctor-config-preflight.ts:71`
  - body: The check treats the first existing `/home/node` or `/home/openclaw` config as the gateway config without proving the running service loaded it. A stale file at either path will trigger a mismatch warning and suggest setting `OPENCLAW_CONFIG_PATH` to the wrong config, which can recreate the drift this guard is meant to prevent.
  - confidence: 0.86
- **[P2] Remove the unconditional root-config deletion advice:** `src/commands/doctor-config-preflight.ts:105`
  - body: This warning tells operators to delete `/root/.openclaw/openclaw.json` whenever doctor is using another config path, but the patch does not verify that a legitimate root-managed gateway is not using that file. Following the advice can remove a live service config on multi-instance or root-managed hosts.
  - confidence: 0.82
