## Summary

Keep this PR open. Current main still exposes `agentId` and `sessionKey` on the second hook context argument, but not on the `before_agent_start` or `agent_end` event payloads this PR is trying to extend. The PR is a narrow, additive public plugin API change, but it needs maintainer API review and a rebase onto the current split hook files/docs before it can merge.

## Best Possible Solution

Keep this PR open for maintainer review as an additive plugin API change. If accepted, rebase it onto current main by adding optional `sessionKey` and `agentId` to `PluginHookBeforeAgentStartEvent` and `PluginHookAgentEndEvent`, populate them from the existing hook context only when available, add regression coverage for present and omitted fields, and update `docs/plugins/hooks.md`.
