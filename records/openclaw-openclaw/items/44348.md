## Summary

Keep this PR open. Current main still exposes `agentId` and `sessionKey` on the second hook context argument, but not on the `before_agent_start` or `agent_end` event payloads this PR is trying to extend. The PR is an additive public plugin API change, so it needs maintainer API review and a rebase onto the current split hook files rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review as an additive plugin API change. If accepted, rebase it onto current main by adding optional `sessionKey` and `agentId` to `PluginHookBeforeAgentStartEvent` and `PluginHookAgentEndEvent`, populate them only when available (preferably via the hook runner’s existing correlation-field helper so all legacy paths stay consistent), add focused regression coverage for present and omitted fields, and update the plugin hooks docs. If maintainers prefer the existing second-argument context contract, close it manually with that API rationale.
