## Summary

Keep this PR open. Current main still lacks the requested behavior, and the PR branch has concrete TUI correctness issues plus a security-sensitive no-prompt exec-policy change that needs maintainer review before merge.

## What This Changes

The PR adds a hands-free exec-policy preset and docs, neutralizes CLI/TUI copy, prefixes TUI rows with timestamps, and changes TUI session activity/project-scoping behavior with tests.

## Best Possible Solution

Revise or split the PR so neutral copy and docs can be reviewed separately from security policy and TUI session changes; timestamp UI chrome should render outside Markdown, session matching should require agent-qualified identity, and configured session defaults should remain intact.

## Reproduction Assessment

Yes for the PR findings: the diff itself shows assistant Markdown is wrapped with `prefixTimestamp(...)`, and the session matcher compares only `parseAgentSessionKey(...).rest`. No local tests were run because this was a read-only review.

## Solution Assessment

No. The direction has useful pieces, but the current branch is not the narrowest maintainable solution because it mixes copy cleanup, exec security policy, timestamp rendering, and session routing changes with unresolved correctness issues.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve Markdown semantics when adding assistant timestamps:** `src/tui/components/assistant-message.ts:13`
  - body: Prepending the timestamp into the assistant Markdown payload changes parsing for replies that start with syntax such as code fences, headings, or lists. Render the timestamp as a separate TUI element instead of modifying the Markdown text.
  - confidence: 0.93
- **[P1] Match session events with the full agent-qualified key:** `src/tui/tui-session-actions.ts:149`
  - body: Comparing only the parsed rest key makes `agent:ops:main` match `agent:main:main`, which can update the active session and route later sends to the wrong agent. Require exact agent+rest equality for this path.
  - confidence: 0.9
- **[P2] Respect configured defaults when no TUI session is passed:** `src/tui/tui.ts:313-315`
  - body: Generating a project session whenever `--session` is omitted bypasses the empty-input path that applies configured global/main defaults, changing existing users' default TUI session behavior.
  - confidence: 0.86
- **[P2] Keep pending user rows timestamped after updates:** `src/tui/components/user-message.ts:7`
  - body: `UserMessageComponent` prefixes only the constructor text, but pending rows are later updated through the inherited raw `setText`, so edited pending entries drop the timestamp format introduced by this PR.
  - confidence: 0.84
- **[P3] Use a deterministic hour cycle for TUI timestamps:** `src/tui/components/timestamp.ts:2-6`
  - body: `toLocaleTimeString` inherits the host hour cycle, so the intended `[HH:MM:SS]` prefix can become locale-dependent output such as `02:30:45 PM`. Set `hour12: false` or otherwise format deterministically.
  - confidence: 0.78
