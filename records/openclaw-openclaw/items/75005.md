## Summary

Keep open. The linked bug is valid and the PR adds the right kind of media-provider seam, but the latest PR head still lets plugin-only media providers return synthetic auth before the established model-auth resolver can use real env or stored-profile credentials.

## What This Changes

The PR adds a media provider `resolveSyntheticAuth` hook, threads provider instances into media execution auth resolution, documents the no-auth media provider pattern, and adds audio/video regression tests.

## Best Possible Solution

Keep the media synthetic-auth hook, but make it a fallback behind the existing model-auth resolver: literal configured keys should win, explicit profile failures should remain hard failures, real profile/env/config credentials should be tried before synthetic markers, and focused tests should cover plugin-only env and stored-profile precedence.

## Reproduction Assessment

Yes. On PR head, configure a plugin-only audio provider with `resolveSyntheticAuth`, set `LOCAL_AUDIO_API_KEY` or a stored `local-audio` auth profile, omit `models.providers` and `entry.profile`, then run the local-no-auth media path; the branch at `src/media-understanding/runner.entries.ts:447` returns the synthetic marker before the resolver can see the real credential.

## Solution Assessment

No as-is. `resolveSyntheticAuth` is the right seam, but this implementation should call `resolveApiKeyForProvider(...)` first and use the hook only after the generic missing-provider-key case, while preserving explicit profile errors.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve resolver credentials before synthetic auth:** `src/media-understanding/runner.entries.ts:447`
  - body: This branch returns the provider hook's synthetic key whenever there is no provider config and no explicit profile selection, before `resolveApiKeyForProvider(...)` runs. A plugin-only media provider with a real env key or stored auth profile will ignore that credential and execute with the synthetic marker instead, which changes existing auth precedence. Please run the model-auth resolver first and use the hook only as the missing-key fallback, with env/profile precedence tests.
  - confidence: 0.9
