## Summary

Current main still does not satisfy the requested user-visible mid-thread compaction/reset signal plus assistant-side disclosure. The existing reply-layer notices remain opt-in, tests and docs explicitly preserve silent default behavior, post-compaction recovery is internal model guidance rather than guaranteed user disclosure, Control UI status is transient, and channel behavior remains surface-specific. Related PR #67830 added useful opt-in completion notices, but it does not implement this issue's default visibility and recovery-disclosure contract.

## Best Possible Solution

Keep this open for maintainer product work. A complete solution should define the default compaction/reset visibility contract, decide which durable UI or channel indicators are required, add assistant-side disclosure and re-grounding/reload-offer behavior after compaction, preserve safe suppression controls for customer-facing deployments, and update tests/docs for `agents.defaults.compaction.notifyUser`, post-compaction recovery, Control UI status, and supported chat surfaces.
