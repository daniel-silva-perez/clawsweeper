## Summary

Current main still does not implement the requested default, hard-to-miss mid-thread compaction/reset disclosure. The repo has partial pieces: opt-in start/completion notices, transient Control UI status, persisted compaction dividers in history, channel-specific reactions/status lines, and internal post-compaction model guidance. Those do not satisfy the issue's default user-visible signal plus assistant-side self-disclosure/reload-offer contract, and related #72043 shows the default/suppression behavior needs maintainer product judgment.

## Best Possible Solution

Keep this open for maintainer product work. Define the default compaction/reset visibility contract, decide the durable UI/channel signal model, add assistant-side disclosure and re-grounding/reload-offer behavior after compaction, preserve explicit suppression controls for customer-facing deployments, and update tests/docs for notifyUser, post-compaction recovery, Control UI status, and supported chat surfaces.
