## Summary

Keep this issue open. Current main still does not satisfy the requested default, hard-to-miss user signal plus assistant-side post-compaction disclosure. Compaction notices remain opt-in, the post-compaction recovery prompt is internal model guidance, Control UI status is transient, and channel behavior is still partial or surface-specific. Related #72043 is an open suppression/config discussion, not a superseding implementation of this request; #50790 is closed but tracks session reset controls, not mid-thread compaction visibility.

## Best Possible Solution

Keep this open for maintainer product work. A complete fix should define the default compaction/reset visibility contract, decide which UI or channel indicators must be durable, add assistant-side disclosure and re-grounding/reload-offer behavior after compaction, preserve safe suppression controls for customer-facing bots, and update tests/docs for `agents.defaults.compaction.notifyUser`, post-compaction recovery, Control UI status, and supported chat surfaces.
