## Summary

Keep this PR open for maintainer review. Current main still returns the effective `chat.history.verboseLevel` from the gateway but Control UI does not store it and still hard-codes tool-card auto expansion off; this PR is a narrow, directly relevant implementation with focused coverage and no security-sensitive diff surface found.

## What This Changes

The PR adds Control UI `chatVerboseLevel` state, populates it from `chat.history.verboseLevel`, resets it with chat history/session state, uses `full` to drive `autoExpandToolCalls`, adds focused UI regression tests, and records the fix in the changelog.

## Best Possible Solution

Review and land this narrow Control UI plumbing after normal maintainer review or rebase. The end state should keep the gateway API unchanged and consume the already-returned effective verbose level in the Control UI render path, with manual tool-card toggles still preserved by the existing expansion helper.
