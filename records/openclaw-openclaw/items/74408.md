## Summary

Keep this PR open. The linked onboarding bug is still present on current main, but the proposed cask fix is incomplete because the new boolean metadata is dropped by the skill frontmatter parser before the installer command builder sees it.

## What This Changes

This PR adds a `cask` flag to skill install metadata, uses it for Homebrew install/status handling, marks CodexBar as a cask, and changes the bundled summarize installer from Homebrew to the `@steipete/summarize` node package.

## Best Possible Solution

Revise the PR so the `cask` field is parsed and normalized end to end, align any public install-spec contract mirrors that should expose it, add focused parser/install/status tests, and then land the reviewed fix for the linked onboarding failure.

## Reproduction Assessment

Yes. The linked issue gives a concrete macOS `openclaw onboard` failure path, and the PR-specific defect is reproducible by parsing a skill manifest containing `"cask": true` and observing that current `parseInstallSpec()` never returns `spec.cask`.

## Solution Assessment

No, not yet. The direction is maintainable, but the patch misses the parser layer, so the CodexBar manifest change does not affect the command that onboarding will execute.

## Review Findings

Overall correctness: patch is incorrect
