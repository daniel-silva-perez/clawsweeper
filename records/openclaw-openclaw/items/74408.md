## Summary

Keep this PR open because it targets a valid linked onboarding bug, and current main still installs CodexBar as a Homebrew formula and summarize from the removed tap formula. The patch is not mergeable as-is: the new boolean `cask` field is added to the manifest and command builder, but the skill frontmatter parser never preserves that boolean, so the CodexBar install path would still run without `--cask`.

## What This Changes

This PR adds a `cask` skill install metadata flag, uses it when building and labeling Homebrew install commands, marks the bundled CodexBar installer as a cask, and changes the bundled summarize installer from Homebrew to the `@steipete/summarize` node package.

## Best Possible Solution

Revise this PR so `cask` is a parsed and normalized skill install property end to end, add focused regression tests for frontmatter parsing plus install command/status behavior, validate the npm source choice for summarize, and then land it as the fix for #74380.
