## Summary

Current main still has the reported model-usage cost parsing gap. Numeric-string costs are skipped, booleans are accepted as native numbers, and NaN/Infinity are not filtered before aggregation/current-model selection/latest-day output. The linked implementation PR #37877 is closed and unmerged, so this still needs a narrow fresh fix PR.

## Best Possible Solution

Keep this issue open and land a narrow parser fix: add a shared finite cost parser in skills/model-usage/scripts/model_usage.py that accepts finite numeric strings, rejects booleans, rejects non-finite values, and is used by aggregate_costs, pick_current_model, and latest_day_cost. Add focused tests in skills/model-usage/scripts/test_model_usage.py and a changelog fix entry crediting @shuofengzhang. The closed PR #37877 is useful prior art but needs a fresh PR against current main.
