## Summary

Current main still has the reported model-usage parser gap. The helper only accepts native int/float costs, so numeric strings are skipped, bool values are accepted through Python's bool-is-int behavior, and NaN/Infinity are not filtered before totals, model selection, or latest-day output. The linked fix PR #37877 is closed and unmerged, so this remains a valid narrow fix issue.

## Best Possible Solution

Keep this issue open and land a narrow parser fix in the model-usage skill: add one shared finite-cost parser that accepts finite numeric strings, rejects bools, rejects NaN/Infinity, and is used by aggregate_costs, pick_current_model, and latest_day_cost. Add focused tests for all three helper paths and a changelog fix entry crediting @shuofengzhang; PR #37877 is useful prior art but needs a fresh main-based PR.
