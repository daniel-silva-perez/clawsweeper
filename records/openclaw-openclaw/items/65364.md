## Summary

Keep this PR open. Current main still does not provide the proposed external plugin seam for runtime provider-auth overrides, and the PR is a plausible additive plugin-SDK direction, but the branch still has merge-blocking correctness/security concerns around provider alias matching and incomplete model-context propagation.

## What This Changes

The PR adds `api.registerProviderRuntimeAuthOverride`, registry/global storage for override registrations, model-auth resolution support with reentrancy and logging, per-call base URL/header passthrough, opt-in availability handling, and focused auth/runtime/media-understanding tests.

## Best Possible Solution

Land a revised additive plugin-SDK/core seam that lets trusted plugins provide per-call provider credentials, base URL, and request headers without replacing provider implementations, while preserving normalized provider matching, accurate model context, conservative availability semantics, and fallback behavior across main and side inference paths.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable for the feature request itself; for the review findings, yes: the PR patch shows raw provider matching and empty-string model fallback, and current main call sites show known model values are not passed into the new resolver parameter.

## Solution Assessment

Is this the best way to solve the issue? Unclear as-is. A dedicated additive SDK seam is a reasonable maintainable direction, but the current patch should normalize provider ids/aliases and thread known model ids before it is the best implementation to merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Normalize provider IDs before matching overrides:** `src/agents/model-auth.ts:390`
  - body: The override loop matches registrations with raw `includes(provider)`. Current auth resolution accepts normalized aliases such as `aws-bedrock` for `amazon-bedrock`, so a plugin registered for the canonical provider can be skipped and OpenClaw may fall back to normal credentials instead of the intended brokered auth. Normalize both the requested provider and registered provider ids before matching, including the availability check.
  - confidence: 0.88
- **[P2] Pass known model IDs into override callbacks:** `src/agents/model-auth.ts:399`
  - body: The callback receives `params.modelId ?? ""`, but the new `modelId` parameter is only populated through `getApiKeyForModel`. Media and image-generation paths already know the selected model yet still call `resolveApiKeyForProvider` without it, so model-scoped credential brokers see an empty model and can return null or the wrong credential. Thread the known model through those resolver calls or make the context explicitly optional.
  - confidence: 0.82
- **[P2] Keep image auto selection from pinning tentative providers:** `src/media-understanding/runner.ts:604`
  - body: `resolveAutoImageModel` now uses the override-aware candidate list but immediately returns `keyEntries[0]`. If that first provider is only conditionally override-backed and later fails at request time, callers of this single-model helper never consider later key-backed providers even though the audio auto path was updated to fall through. Preserve ordered fallback behavior or keep tentative override-only providers out of this single-result path.
  - confidence: 0.76

## Security Review

Status: needs_attention

Summary: The diff adds a trusted plugin auth-override capability and still has concrete auth-routing gaps that can bypass or mis-scope brokered credentials.

Concerns:

- **[medium] Alias calls can bypass brokered auth overrides:** `src/agents/model-auth.ts:390`
  - body: Raw provider-id matching means an alias-form request can skip a registered override and use default credentials instead, which undermines the policy boundary for plugins that are meant to broker all auth for a provider family.
  - confidence: 0.82
- **[low] Model-scoped auth context can be erased:** `src/agents/model-auth.ts:399`
  - body: Several side paths can call the new override surface with an empty model id even when the selected model is known, which can weaken model-scoped broker authorization or route a request through unintended credentials.
  - confidence: 0.74
