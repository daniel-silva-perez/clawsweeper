## Summary

Keep open. Current main does not implement the PR’s resource-cleanup behavior: `downloadToFile` still lacks the PR’s response draining, response-error handler, write-stream destroy, and single-settle guard, and the URL save path still has no outer temp-file cleanup catch. The PR is plausible, but it needs a rebase/update to preserve newer main-branch redirect-header stripping and custom `maxBytes` behavior.

## Best Possible Solution

Keep the PR open as a real implementation candidate, but ask for a rebase onto current main that preserves redirect credential stripping, keeps the `maxBytes` parameter behavior, and retains focused regression tests for failed URL-download temp cleanup. Then run the targeted media tests and changed gate before merge.
