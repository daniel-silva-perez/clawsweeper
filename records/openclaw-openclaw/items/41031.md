## Summary

Keep this PR open. Current main still resolves the heartbeat run context from the main, forced, or configured heartbeat session before delivery routing, so explicit Telegram delivery can target the requested chat while the model run still uses the main session. The PR remains directionally useful, but it needs maintainer semantics approval and an update for current isolated-heartbeat call sites before it is merge-ready.

## What This Changes

The PR makes heartbeat session resolution derive the run session from explicit `heartbeat.target` plus `heartbeat.to` through outbound session routing, and adds a Telegram regression test for that case.

## Best Possible Solution

Land an updated implementation only if maintainers accept target-derived run-session semantics: derive the session from `resolveHeartbeatDeliveryTarget` plus `resolveOutboundSessionRoute` only for explicit non-history `target` and `to` with no forced session or `heartbeat.session`, preserve global/subagent/`last`/`none`/isolated behavior, add Telegram and isolated-session regression coverage, and update the public heartbeat docs.
