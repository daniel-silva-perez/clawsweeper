## Summary

Keep open for maintainer review. Current main still lacks the requested Control UI session-list preview, but this PR is not merge-ready because it mixes an unrelated pnpm override with the UI change, enables transcript-tail reads for unbounded session lists, and renders unbounded preview text; it also overlaps the still-open #48396 implementation path.

## What This Changes

The PR adds a `lastMessagePreview` UI row field, requests `includeLastMessage` from `sessions.list`, renders `[MM/DD HH:mm]` preview text under each session key, and adds a `punycode` pnpm override.

## Best Possible Solution

Land one narrowed implementation for #47120: keep the dependency graph unchanged, request previews only through a bounded or capped path, propagate the row field deliberately, normalize and truncate displayed preview text with focused controller/view tests, and then close or supersede the duplicate PR path.

## Reproduction Assessment

Not applicable as a feature PR, but there is a high-confidence source verification path. Current main does not request, type, or render `lastMessagePreview`, while the live PR diff adds that path and the problematic override.

## Solution Assessment

No. The safer solution is a smaller Control UI change that uses the existing backend preview option only with bounded reads, clamps the displayed text, adds focused tests, and avoids unrelated dependency-resolution changes.

- **[P1] Remove the unpaired pnpm override:** `package.json:477`
  - body: Adding `punycode` to `pnpm.overrides` changes package resolution, but this PR does not update `pnpm-lock.yaml` and the change is unrelated to the Sessions UI. - **[P2] Gate last-message fetch behind bounded session lists:** `ui/src/ui/controllers/sessions.ts:42`
  - body: This always sends `includeLastMessage: true`, but the Control UI omits `limit` when the session limit is `0` for all sessions. Since the backend reads transcript tails per returned session for this flag, large stores can make a normal refresh perform unbounded file I/O; only enable previews for a bounded or capped list.
  - confidence: 0.9
- **[P2] Clamp rendered session preview text:** `ui/src/ui/views/sessions.ts:456`
  - body: The preview is rendered after only `trim()`, while the backend can return text from a 16KB transcript tail. A long final message can expand the Sessions table row and hurt scanning/performance, so normalize whitespace and truncate the visible text, with the full value in a tooltip if needed.
  - confidence: 0.86
