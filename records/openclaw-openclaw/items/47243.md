## Summary

Keep open. Current main still does not implement the Control UI session timestamp and last-message preview experience, but this PR should not merge as written because it includes an unrelated dependency override, enables transcript-tail reads on unbounded session-list refreshes, and renders preview text without a UI clamp. The linked feature issue and overlapping PR #48396 remain open, so this needs maintainer selection or author revision rather than cleanup closure.

## What This Changes

This PR adds Control UI session-list timestamp and last-message preview rendering by adding `lastMessagePreview` to the UI row type, requesting `includeLastMessage`, displaying `[MM/DD HH:mm]` preview text, and adding a `punycode` pnpm override.

## Best Possible Solution

Land one narrowed Control UI implementation for #47120: remove the dependency override, request previews only through a bounded or capped path, propagate `lastMessagePreview` through UI row and event handling deliberately, normalize and truncate displayed preview text, add focused controller/view coverage, and reconcile the duplicate open PR.

## Reproduction Assessment

Not applicable as a feature PR, but there is a high-confidence source verification path. Current main does not request `includeLastMessage`, does not type `lastMessagePreview` in the Control UI row, and does not render a preview lane, while the PR diff adds that feature path.

## Solution Assessment

No, this PR is not currently the best way to solve the issue. The safer solution is a smaller UI-focused change that uses the existing backend option with bounded reads, clamps display text, updates row/event typing deliberately, and avoids unrelated dependency resolution changes.

- **[P1] Remove the unpaired pnpm override:** `package.json:477`
  - body: Adding `punycode` to `pnpm.overrides` changes package resolution, but this PR does not update `pnpm-lock.yaml` and the change is unrelated to the Sessions UI.   - confidence: 0.94
- **[P2] Gate preview enrichment to bounded session lists:** `ui/src/ui/controllers/sessions.ts:42`
  - body: This always sends `includeLastMessage: true`, but the Control UI omits `limit` when the session limit is `0` for all sessions. Since the backend reads transcript tails per returned session for this flag, large stores can make a normal refresh perform unbounded file I/O; only enable previews for a bounded or capped list.
  - confidence: 0.9
- **[P2] Clamp the rendered preview text:** `ui/src/ui/views/sessions.ts:456`
  - body: The preview is rendered after only `trim()`, while the backend can return text from a 16KB transcript tail. A long final message can expand the Sessions table row and hurt scanning/performance, so normalize whitespace and truncate the visible text, with the full value in a tooltip if needed.
  - confidence: 0.86
