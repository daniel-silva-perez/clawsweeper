## Summary

Keep open. The requested Control UI UX is still missing on current main, but this patch is not merge-ready because it unconditionally enables per-session transcript-tail reads, renders unbounded preview text, and mixes in an unrelated pnpm override. The overlapping open implementation in #48396 also means maintainers should choose or reconcile the two PRs rather than let automation replace one blindly.

## What This Changes

This PR adds a Control UI session-list timestamp and last-message preview by adding a UI row field, requesting `includeLastMessage` from `sessions.list`, rendering a timestamped preview under the session key, and adding a `punycode` pnpm override.

## Best Possible Solution

Land one narrowed Control UI implementation for #47120: remove the dependency override, request previews only for a bounded or capped session set, propagate `lastMessagePreview` intentionally through the UI row type, normalize and truncate preview text with clear timestamp behavior, and add focused controller/view coverage while reconciling #48396.

## Reproduction Assessment

Not applicable as a feature PR, but there is a high-confidence source verification path: current main does not request `includeLastMessage` in the Control UI session list and does not render a preview lane, while the gateway can already supply `lastMessagePreview` when requested.

## Solution Assessment

No, this PR is not currently the best way to solve the issue. The maintainable path is a smaller UI-only change that uses the existing backend option with bounded reads, clamps display text, and avoids unrelated package-resolution changes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Remove the unpaired pnpm override:** `package.json:477`
  - body: Adding `"punycode": "npm:punycode.js@^2.3.1"` changes pnpm override settings without updating `pnpm-lock.yaml`, so frozen installs can fail with a lockfile config mismatch. This dependency-resolution change is also unrelated to the Sessions UI feature and should be removed or split into a dedicated dependency PR with lockfile proof.
  - confidence: 0.94
- **[P2] Bound last-message enrichment requests:** `ui/src/ui/controllers/sessions.ts:42`
  - body: This unconditionally requests `includeLastMessage`, but the Control UI omits `limit` when the session limit is `0` for all sessions. Because `sessions.list` performs transcript reads per returned session for this flag, large stores can turn a normal refresh into unbounded transcript I/O; only request previews when the result set is bounded or otherwise capped.
  - confidence: 0.9
- **[P2] Clamp rendered session previews:** `ui/src/ui/views/sessions.ts:456`
  - body: The rendered preview only calls `trim()`, while backend extraction can return a large transcript-tail message. A long final message can expand the table row and make the Sessions list hard to scan, so normalize whitespace and truncate the displayed text, with the full value in a tooltip if needed.
  - confidence: 0.86
