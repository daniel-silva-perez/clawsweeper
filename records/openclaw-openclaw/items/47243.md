## Summary

Keep open. The Control UI feature request is still valid on current main, but this PR is not mergeable as written: it mixes in an unrelated pnpm override without lockfile proof, turns last-message enrichment on for unbounded session-list refreshes, and renders preview text without a UI clamp. The linked feature issue and overlapping #48396 are both still open, so the maintainer action is to choose or revise an implementation rather than close this PR.

## What This Changes

This PR adds a Control UI `lastMessagePreview` row field, requests `includeLastMessage` from `sessions.list`, renders a `[MM/DD HH:mm]` preview under each session key, and adds a `punycode` pnpm override.

## Best Possible Solution

Land one narrowed Control UI implementation for #47120: remove the dependency override, request previews only for a bounded or capped session list, propagate `lastMessagePreview` intentionally through the UI row type, normalize and truncate displayed preview text, and add focused controller/view coverage while reconciling #48396.

## Reproduction Assessment

Not applicable as a feature PR, but there is a high-confidence source verification path. Current main does not request `includeLastMessage` in the Control UI Sessions list and does not render a preview lane, while the PR diff adds both.

## Solution Assessment

No, this PR is not currently the best way to solve the issue. The safer path is a smaller UI-only change that uses the existing backend option with bounded reads, clamps display text, and avoids unrelated package-resolution changes.

Full review comments:

- **[P1] Remove the unpaired pnpm override:** `package.json:477`
  - body: Adding `punycode` to `pnpm.overrides` changes package resolution, but this PR does not update `pnpm-lock.yaml` and the change is unrelated to the Sessions UI. Frozen installs can fail with a lockfile config mismatch, so remove this line or split it into a dependency PR with lockfile proof.
  - confidence: 0.94
- **[P2] Gate preview enrichment to bounded session lists:** `ui/src/ui/controllers/sessions.ts:42`
  - body: This always sends `includeLastMessage: true`, but the Control UI omits `limit` when the session limit is `0` for all sessions. Since the backend reads transcript tails per returned session for this flag, large stores can make a normal refresh perform unbounded file I/O; only enable previews for a bounded/capped list.
  - confidence: 0.9
- **[P2] Clamp the rendered preview text:** `ui/src/ui/views/sessions.ts:456`
  - body: The preview is rendered after only `trim()`, while the backend can return text from a 16KB transcript tail. A long final message can expand the Sessions table row and hurt scanning/performance, so normalize whitespace and truncate the visible text, with the full value in a tooltip if needed.
  - confidence: 0.86
