## Summary

Keep open. Current main still does not display last-message previews in the Control UI Sessions list, so the PR is addressing a real unresolved UX request, but the proposed patch is not ready to merge because it adds an unrelated package-resolution override and enables transcript-tail reads for every sessions refresh, including unbounded all-session views. The already-open #48396 overlaps the same feature, so maintainers should choose one corrected implementation path rather than close this PR as obsolete.

## What This Changes

This PR adds `lastMessagePreview` to the Control UI session row type, requests `includeLastMessage` from `sessions.list`, renders a timestamped preview below the session key, and adds a `punycode` pnpm override.

## Best Possible Solution

Land one narrowed Control UI implementation: remove the dependency override, request previews only for a bounded or capped session set, propagate `lastMessagePreview` through the row surface intentionally, normalize and truncate preview text with explicit timestamp behavior, and add focused controller/view coverage while coordinating with #47120 and #48396.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Remove the unpaired pnpm override:** `package.json:477`
  - body: The PR adds `"punycode": "npm:punycode.js@^2.3.1"` to pnpm overrides but does not update `pnpm-lock.yaml`. Frozen installs expect the lockfile override settings to match package configuration, and this package-resolution change is unrelated to the Sessions UI feature, so remove it or split it into a dedicated dependency PR with lockfile proof.
  - confidence: 0.94
- **[P2] Bound last-message enrichment requests:** `ui/src/ui/controllers/sessions.ts:42`
  - body: Setting `includeLastMessage: true` on every Sessions refresh makes the Gateway read transcript tails per returned session. The Control UI omits `limit` when users select all sessions, so large stores can turn a normal refresh into unbounded transcript I/O; only request previews when the result set is bounded or otherwise capped.
  - confidence: 0.9
- **[P2] Clamp rendered session previews:** `ui/src/ui/views/sessions.ts:456`
  - body: The view renders `lastMessagePreview` after `trim()` only. Backend extraction can return a large text body from the transcript tail, so a long final message can expand the key cell and make the table hard to scan; normalize whitespace and truncate the displayed preview, with the full value in a tooltip if needed.
  - confidence: 0.86
