## Summary

Keep this issue open. Current main still drops MIME types on Discord upload Blob construction in normal sends, component sends, and native-command replies, and shared web media still sends non-GIF images including WebP through an optimizer that returns PNG or JPEG. The issue is also already paired with open closing PRs (#41265, #41419, #41714), so the best cleanup path is to review or consolidate one of those fixes rather than close the issue or queue another duplicate repair.

## Best Possible Solution

Keep this issue open and review or consolidate the existing fix PRs (#41265, #41419, #41714). The selected patch should thread detected `contentType` into all Discord Blob construction paths, preserve static WebP and animated media in `src/media/web-media.ts` without silently converting to JPEG/PNG, keep size-cap and media safety checks intact, and add focused regression tests for normal sends, component sends, native-command replies, WebP preservation, and animated-image preservation.
