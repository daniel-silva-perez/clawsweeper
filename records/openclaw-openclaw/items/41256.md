## Summary

Keep this issue open. Current main at 1b25dcf57a9fca5d00cf275046e20a2e3ea3730c still constructs untyped Discord upload Blobs in normal sends, component sends, and native-command replies, and shared web media still sends WebP/non-GIF images through the optimizer that returns PNG or JPEG. The issue is also paired with open, unmerged fix PRs (#41265, #41419, #41714) that explicitly target and close this bug, so the right cleanup path is to review/consolidate one of those fixes rather than close the issue early.

## Best Possible Solution

Keep this issue open and review or consolidate the existing fix PRs (#41265, #41419, #41714). The best patch should thread detected `contentType` into all Discord Blob construction paths, preserve static WebP and animated media in the shared web media loader without silently converting to JPEG/PNG, keep size-cap enforcement intact, and add focused regression tests for normal sends, component sends, native-command replies, WebP preservation, and animated image preservation.
