## Summary

Keep open. Current main still lets the active-session filter drop a stale top-level main session before the Control UI dropdown renders it, and an open PR already uses GitHub closing syntax for this regression, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that the Control UI session dropdown lists only the heartbeat session and omits the main session.

## Best Possible Solution

Preserve the resolved top-level main session in `sessions.list` when active-session filtering is used, keep stale non-main sessions filtered so the dropdown remains compact, and retain regression coverage for stale main alongside active heartbeat.
