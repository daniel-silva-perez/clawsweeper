## Summary

Keep this issue open. Current main still has an active-session listing path that can filter a stale main row out while leaving heartbeat visible, and the chat dropdown only guarantees returned rows plus the current session key. The linked implementation PR remains open and unmerged, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for the Control UI session selector to keep the main session visible and selectable when the dropdown currently shows only heartbeat.

## Best Possible Solution

Land PR #43984, or replace it with an equivalent narrow fix, so active-filtered sessions.list results always retain the resolved top-level main session while preserving the compact recent-session filter for other stale sessions and covering the stale-main plus active-heartbeat regression.
