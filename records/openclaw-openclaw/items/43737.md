## Summary

Keep open. Current main still has a reachable path where `sessions.list` applies the active-session cutoff to every stored row, the chat refresh asks for active sessions after a final run, and the dropdown only renders returned rows plus the current session key. The related open closing PR is still the right implementation candidate rather than a reason to close this issue early.

## What This Changes

The issue asks for the Control UI session dropdown to keep the main session selectable instead of showing only heartbeat.

## Best Possible Solution

The durable fix is for `sessions.list` to preserve the resolved top-level main session when active-session filtering is requested, while continuing to filter stale non-main sessions so the dropdown stays compact and regression coverage protects stale main alongside active heartbeat.
