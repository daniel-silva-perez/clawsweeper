## Summary

Keep open. Current main still permits the reported Control UI state because `sessions.list` applies the active-session cutoff uniformly, the chat selector only renders returned rows plus the current session key, and the already-linked gateway fix remains open and unmerged.

## What This Changes

The issue reports that the Control UI session dropdown omits the main session and only shows heartbeat, preventing users from switching back to main.

## Best Possible Solution

Preserve the resolved top-level main session in the gateway `sessions.list` result when `activeMinutes` filtering is used, keep stale non-main sessions filtered so the dropdown stays compact, and add regression coverage for stale main alongside active heartbeat.
