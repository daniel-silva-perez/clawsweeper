## Summary

Keep open. Current main still has the active-session path that can drop a stale main session before the Control UI dropdown renders it, and an open closing PR already targets the exact regression, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for the Control UI session dropdown to show the main session alongside heartbeat and other active sessions.

## Best Possible Solution

Preserve the resolved top-level main session in the gateway `sessions.list` result when active-session filtering is requested, while continuing to filter stale non-main sessions and adding regression coverage for stale main alongside active heartbeat.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a store with stale `agent:main:main` and fresh heartbeat, then `sessions.list` with positive `activeMinutes` followed by dropdown rendering while heartbeat is the current session; current main filters out main and only adds the current key.

## Solution Assessment

Yes. Fixing the gateway active-list behavior is the narrowest durable solution because it keeps the API and all UI consumers consistent, rather than making the dropdown invent a missing main row locally.
