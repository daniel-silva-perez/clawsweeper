## Summary

Keep open. Current main still has the gateway active-session filter that can drop a stale top-level main session before the Control UI dropdown renders it, and open closing PR #43984 already targets the exact regression, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that the Control UI session dropdown omits the main session and only shows heartbeat after recent versions.

## Best Possible Solution

Preserve the resolved top-level main session in the gateway `sessions.list` result when active-session filtering is requested, while continuing to filter stale non-main sessions and covering stale main plus active heartbeat in regression tests.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a store with stale `agent:main:main` and fresh heartbeat, then `sessions.list` with positive `activeMinutes` followed by dropdown rendering while heartbeat is current; current main filters out main and only adds the current key.

## Solution Assessment

Yes. Fixing gateway active-list behavior is the narrowest maintainable solution because it keeps the API and all UI consumers consistent instead of making the dropdown invent a missing main row locally.
