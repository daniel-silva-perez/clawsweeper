## Summary

Keep open. Current main still has the gateway active-session filter that can drop a stale top-level main session before the Control UI dropdown receives it, and the dropdown only adds returned rows plus the current session key. Open PR #43984 already targets this regression with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Ensure the Control UI session dropdown keeps the main session visible and selectable alongside heartbeat and other active sessions.

## Best Possible Solution

Preserve the resolved top-level main session in gateway sessions.list results when active-session filtering is requested, while continuing to filter stale non-main sessions, and cover stale main plus active heartbeat with a regression test.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a store with stale agent:main:main and a fresh heartbeat session, followed by sessions.list with positive activeMinutes and dropdown rendering while heartbeat is current; current main filters out main and the dropdown has no fallback row for it.

## Solution Assessment

Yes. Fixing gateway active-list behavior is the narrowest maintainable solution because it keeps the API contract consistent for all consumers instead of making the Control UI invent a missing main row locally.
