## Summary

Keep open. Current main still contains the gateway active-session filter that can remove a stale top-level `main` session before the Control UI dropdown sees it, and open PR #43984 already targets this exact regression with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that the Control UI session dropdown lists only `heartbeat` and no longer offers the `main` session.

## Best Possible Solution

Preserve the resolved top-level main session in the gateway `sessions.list` result when active-session filtering is requested, continue filtering stale non-main sessions, and cover stale main plus active heartbeat in a regression test.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a store containing stale `agent:main:main` and fresh heartbeat, then `sessions.list` with positive `activeMinutes` followed by dropdown rendering while heartbeat is the current session; current main filters out main and the dropdown only adds returned rows plus the current key.

## Solution Assessment

Yes. Fixing gateway active-list behavior is the narrowest maintainable path because it keeps the API and all UI consumers consistent instead of making the dropdown invent a missing main row locally.
