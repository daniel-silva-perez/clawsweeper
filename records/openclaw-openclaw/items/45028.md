## Summary

Keep open: current main preserves external session metadata in some WebChat access paths, but the browser Control UI send path still uses session-only WebChat delivery and the dispatcher suppresses external route delivery unless an explicit deliver route is set. The open broad sync PR may address the workflow, but it is unmerged and needs maintainer review before this issue can be closed.

## What This Changes

The issue requests that Control UI replies sent while viewing a Telegram-backed session route back to the Telegram transport instead of staying in WebChat.

## Best Possible Solution

Add a narrow, explicit Control UI operator-reply affordance or routing policy for externally backed sessions that can deliver to the stored channel route without weakening existing WebChat leakage safeguards. The change should cover browser Control UI behavior, Gateway route resolution, dispatcher explicit-delivery handling, and regression tests for Telegram-style channel-scoped sessions plus WebChat-native sessions.
