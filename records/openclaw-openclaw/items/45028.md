## Summary

Current main still routes the browser Control UI chat path through internal WebChat semantics, so the reported Telegram-backed session reply problem is not safely closeable. The issue is also the canonical tracker used by a related WeChat duplicate, while the broad open implementation PR #46356 needs maintainer review before it can supersede this report.

## What This Changes

The issue asks for Control UI replies in a Telegram-backed direct session to preserve the underlying Telegram delivery route instead of staying on the internal WebChat surface.

## Best Possible Solution

Keep this issue open as the canonical tracker until maintainers either land a narrow, tested Control UI external-reply affordance or explicitly decide that browser WebChat must remain session-only. The implementation should distinguish ordinary WebChat viewing from an intentional operator reply to the stored channel route, and preserve the current cross-channel leakage protections.
