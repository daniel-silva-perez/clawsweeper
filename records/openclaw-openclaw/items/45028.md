## Summary

Keep open. Current main still preserves external route metadata for externally backed sessions, but ordinary browser Control UI sends are still internal WebChat turns and do not request explicit external delivery, so the dispatcher will not route the assistant reply to Telegram. The related open PR #46356 targets the broader WebChat mirroring/operator-reply behavior, so this is not cleanup-closeable yet.

## What This Changes

Control UI replies sent while viewing a Telegram-backed session should route to the session's Telegram delivery target instead of remaining on WebChat.

## Best Possible Solution

Define and implement a narrow operator-reply or WebChat-mirroring path for externally backed sessions that can use the stored channel route for Control UI replies without weakening the existing route-leakage safeguards. The end state should cover the browser composer, Gateway `chat.send` route resolution, reply dispatch, and regression tests for Telegram-style channel-scoped sessions plus WebChat-native sessions.
