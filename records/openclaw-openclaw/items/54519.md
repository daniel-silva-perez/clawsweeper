## Summary

Current main still does not implement the requested hard per-account read-only channel mode. Shared channel config, WhatsApp config/types/schema, gateway send/poll paths, and WhatsApp transport helpers lack a generic account-level outbound-deny gate. The related #71382 PR is open, unmerged, and WhatsApp-scoped, so it is an implementation candidate rather than a close reason for this broader issue.

## Best Possible Solution

Keep this issue open. The best product path is a maintainer-approved generic channel/account outbound policy exposed in shared config schema, docs, and UI metadata, then enforced before every outbound surface: automatic replies, gateway sends, polls, reactions, typing/presence, read receipts, heartbeat/status messages, and plugin-owned transport side effects where feasible. Coordinate with #71382 for the WhatsApp subset, but close this broader issue only after the generic policy or an explicitly scoped replacement ships with tests and docs.
