## Summary

Keep this PR open. Current main still has the reported heredoc approval behavior, but the proposed patch is not ready to merge because it routes timeout fallback approvals through raw shell text in a security-sensitive gateway exec path.

## What This Changes

The PR changes gateway exec approval follow-up so heredoc-required approvals execute the original command text instead of the rebuilt enforced allowlist command.

## Best Possible Solution

Land a narrow gateway approval fix that preserves raw heredoc command text only after explicit `allow-once` or `allow-always`, keeps `enforcedCommand` for allowlist timeout fallback approvals, and adds focused tests for both heredoc branches plus the unchanged non-heredoc path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Keep enforced command for heredoc timeout fallback:** `src/agents/bash-tools.exec-host-gateway.ts:285`
  - body: This switches every heredoc approval follow-up to `params.command`, but the same callback also runs when a request times out and `askFallback === "allowlist"` marks an allowlist match as approved without a human decision. That path should keep the rebuilt `enforcedCommand`; otherwise timeout fallback re-runs raw shell parsing instead of the pinned/quoted allowlist plan. Use raw command text only for explicit `allow-once` or `allow-always`.
  - confidence: 0.89

## Security Review

Status: needs_attention

Summary: The diff changes host exec approval security semantics and can bypass enforced allowlist execution on timeout fallback approvals.

Concerns:

- **[high] Raw heredoc execution applies to timeout fallback:** `src/agents/bash-tools.exec-host-gateway.ts:285`
  - body: The added `approvedExecCommand` uses raw `params.command` for every heredoc-required approval, including `askFallback: "allowlist"` timeout resolution where no human explicitly approved the run. That drops the enforced command's pinned and quoted argv boundary for a security-sensitive host exec path.
  - confidence: 0.87
