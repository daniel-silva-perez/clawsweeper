## Summary

Keep open. Current main still has the pre-PR behavior, and the PR addresses a real heredoc approval problem, but the proposed one-line change touches host exec approval security semantics and has an unresolved review finding for timeout fallback approvals.

## What This Changes

The PR changes gateway exec approval follow-up so heredoc-required approvals execute the original `params.command` instead of the rebuilt allowlist `enforcedCommand`, while leaving non-heredoc approvals on the enforced command path.

## Best Possible Solution

Land a narrow gateway approval fix that preserves raw heredoc command text only after explicit `allow-once` or `allow-always`, keeps the enforced command for allowlist timeout fallback approvals, and adds focused tests for both branches plus changelog attribution if treated as a user-visible bug fix.
