## Summary

Keep open. Current main still sends the approved gateway follow-up execution through `execCommand: enforcedCommand`, so the PR’s heredoc-preservation behavior is not implemented. The PR is plausible, but it needs maintainer/security review because the proposed raw-command switch would also affect `askFallback: "allowlist"` timeout approvals, not only explicit `allow-once` or `allow-always` decisions.

## What This Changes

The PR changes `src/agents/bash-tools.exec-host-gateway.ts` so heredoc-approved gateway exec follow-ups run the original `params.command` instead of the rebuilt allowlist-enforced command.

## Best Possible Solution

Keep this PR open, but revise before merge: preserve raw heredoc command text only for explicit `allow-once` or `allow-always` decisions, keep `enforcedCommand` for `askFallback: "allowlist"` timeout/unavailable fallback approvals, and add focused gateway-host tests for both branches. If the fix lands as a user-visible bug fix, include the normal changelog attribution.
