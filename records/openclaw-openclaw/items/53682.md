## Summary

Keep open. The PR targets a real gateway exec approval path, and current main does not implement its intended behavior: allowlisted heredoc commands still require explicit approval, but the asynchronous approved execution path still passes the rebuilt `enforcedCommand` rather than preserving the original heredoc command text. The PR also has a valid security review concern: its current diff would use raw `params.command` for timeout fallback approvals, not only explicit human approvals.

## Best Possible Solution

Keep the PR open and revise it so explicit human decisions (`allow-once` and `allow-always`) can execute the original heredoc command text, while timeout fallback approvals continue to use the enforced wrapper. Add focused gateway-host tests for both cases: approved heredoc preserves `params.command`, and `askFallback=allowlist` timeout keeps `enforcedCommand`.
