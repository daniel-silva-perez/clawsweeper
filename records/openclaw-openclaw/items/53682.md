## Summary

Keep open. Current main still executes the approved gateway follow-up with `execCommand: enforcedCommand`, so the PR’s intended heredoc preservation is not implemented. The PR is still plausible, but the current one-line change is security-sensitive because it would also use raw `params.command` for heredoc timeout approvals when `askFallback === "allowlist"`.

## Best Possible Solution

Keep this PR open, but revise it before merge: use raw `params.command` only after explicit `allow-once` or `allow-always` decisions for heredoc commands, keep `enforcedCommand` for `askFallback === "allowlist"` timeout/unavailable fallback approvals, and add focused gateway-host tests for both branches. If this lands as user-visible bug fix, include the normal changelog attribution.
