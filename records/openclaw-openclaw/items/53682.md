## Summary

Keep open. Current main still does not implement the PR’s intended heredoc-preserving approval execution, and the proposed one-line switch is not merge-ready because it would also use raw command text for non-human `askFallback: "allowlist"` timeout approvals on a security-sensitive host exec path.

## What This Changes

The PR changes the gateway exec approval follow-up so heredoc-required approvals execute the original `params.command` instead of the rebuilt allowlist `enforcedCommand`, while leaving non-heredoc approvals on the enforced command path.

## Best Possible Solution

Land a narrow gateway approval fix that preserves original heredoc text only after explicit `allow-once` or `allow-always`, keeps the enforced command for allowlist fallback approvals, and adds focused tests for both branches plus a normal changelog attribution if treated as a user-visible bug fix.
