## Summary

Keep this PR open. Current main still attributes `sessions.usage` channel buckets from `storeEntry.channel` before `origin.provider`, so the intended fix is not implemented. The PR is a plausible fix for the linked swapped webchat/Telegram bug, but it needs maintainer review because the same two-line change also changes visible session row semantics and uses mutable origin metadata as a whole-session aggregation key.

## Best Possible Solution

Keep this PR open for maintainer review or rework. The best path is a bounded usage fix that resolves the swapped webchat/Telegram aggregate bug while making the attribution contract explicit: preserve delivery metadata or add explicit delivery fields for rows, validate or cap origin-derived aggregate keys, and add regression coverage for swapped cross-channel sessions plus mixed-origin sessions. Coordinate with #61315 if maintainers want one consolidated usage aggregation fix.
