## Summary

Keep this PR open. Current main still does not implement the cron/subagent completion delivery-confirmation fix: gateway agent delivery still returns only payloads/meta, completion direct announce still treats a visible gateway payload as delivered, and tests still preserve that unconfirmed success path. The Langfuse tracing half is not present on main and now fits better as a plugin/ClawHub adapter using the existing diagnostics/OpenTelemetry and plugin hook surfaces, but that does not make the bundled delivery bug-fix half obsolete.

## Best Possible Solution

Do not close this PR as implemented and do not merge it as-is. Split a narrow core PR that makes gateway agent delivery return explicit external-delivery confirmation from deliverOutboundPayloads and requires that confirmation for completion-mode cron/subagent direct announces, while preserving silent duplicate/no-reply behavior and queue fallback. Move the Langfuse PoC to a ClawHub/plugin adapter using diagnostics/OpenTelemetry events and plugin hooks; propose a generic plugin API extension only if that adapter proves a missing seam.
