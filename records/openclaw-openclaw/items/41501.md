## Summary

Keep this PR open for maintainer follow-up, but do not merge it as-is. Current main has a backend-neutral OpenTelemetry/plugin path for the Langfuse observability goal, but it still does not implement the PR’s cron/subagent delivery-confirmation fix: gateway agent delivery drops the external delivery result, and completion direct announce still treats visible gateway payloads as delivered without explicit external-delivery metadata.

## Best Possible Solution

Do not merge this PR as-is. Split a small core PR that returns explicit external-delivery confirmation from gateway agent delivery, then require that confirmation for completion-mode subagent/cron direct announces or fall back to direct send when absent. Add regression coverage for payload-only gateway responses and best-effort delivery failures. Move the Langfuse PoC to a ClawHub/plugin adapter using diagnostics events and plugin hooks, and only propose a generic plugin API change if that adapter proves one is missing.
