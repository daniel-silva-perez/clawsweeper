## Summary

Keep this PR open for maintainer follow-up. Current main covers the Langfuse observability goal through backend-neutral diagnostics, OpenTelemetry export, and plugin hooks, but it still does not fully implement the PR’s cron/subagent completion delivery-confirmation fix: gateway agent delivery returns payloads/meta without explicit external-delivery confirmation, and completion direct announce still treats a visible gateway payload as delivered.

## Best Possible Solution

Do not close this PR as implemented and do not merge it as-is. Split a small core PR that returns explicit external-delivery confirmation from gateway agent delivery, then require that confirmation for completion-mode subagent/cron direct announces or fall back to direct send when absent. Move the Langfuse PoC to a ClawHub/plugin adapter using diagnostics events, plugin hooks, and agent event subscriptions; propose a generic plugin API change only if that adapter proves a missing seam.
