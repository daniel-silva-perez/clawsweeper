## Summary

Keep this PR open. Current main has backend-neutral observability through diagnostics/OpenTelemetry and plugin hooks, so the Langfuse-specific PoC should move out of core, but current main still does not implement the PR's explicit cron/subagent completion delivery-confirmation fix. The gateway agent delivery path still discards outbound delivery results and the subagent completion path treats a visible gateway payload as success without requiring external delivery metadata.

## Best Possible Solution

Do not close this PR as implemented and do not merge it as-is. Split a small core PR that makes gateway agent delivery return explicit external-delivery confirmation and require that confirmation for completion-mode subagent/cron direct announces, falling back to direct send when absent. Move the Langfuse PoC to a ClawHub/plugin adapter using diagnostics events, OpenTelemetry, plugin hooks, and agent event subscriptions; propose a generic plugin API change only if that adapter proves a missing seam.
