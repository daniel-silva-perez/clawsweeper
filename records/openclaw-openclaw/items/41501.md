## Summary

Keep this PR open. Current main still does not implement the cron/subagent completion delivery-confirmation fix, while the Langfuse tracing half is better handled outside core through the existing plugin, diagnostics, and OpenTelemetry surfaces. The PR should not be merged as-is because it bundles an optional tracing integration, a new third-party dependency, secret-bearing config, payload export/redaction code, and an unrelated core delivery bug fix.

## Best Possible Solution

Keep the PR open for maintainer follow-up, but do not merge it as-is. Split a narrow core PR that makes gateway agent delivery expose explicit external-delivery confirmation from deliverOutboundPayloads and requires that confirmation before completion-mode cron/subagent direct announces return delivered=true, with regression tests for payload-only gateway responses and fallback behavior. Move the Langfuse PoC to a ClawHub/plugin adapter using the existing diagnostics/OpenTelemetry and plugin hook surfaces; propose a generic plugin API extension only if that adapter proves a missing seam.
