## Summary

Keep this PR open for maintainer follow-up. Current main still does not implement the cron/subagent completion delivery-confirmation fix, and the Langfuse tracing half remains an optional observability integration better routed through ClawHub/plugin and OpenTelemetry surfaces rather than merged into core as-is.

## Best Possible Solution

Keep this PR open for maintainer follow-up, but do not merge it as-is. Split a narrow core PR that has the gateway agent command expose explicit external delivery confirmation from deliverOutboundPayloads and makes completion-mode cron/subagent direct announces require that confirmation before returning delivered=true, with regression tests for payload-only gateway responses and fallback behavior. Move the Langfuse PoC to a ClawHub/plugin adapter using the existing diagnostics-otel and plugin hook surfaces, and propose a small generic plugin API extension only if that adapter proves a missing seam.
