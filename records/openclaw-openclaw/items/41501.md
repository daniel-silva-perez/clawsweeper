## Summary

Keep this PR open for maintainer follow-up. Current main already covers the Langfuse goal through backend-neutral diagnostics/OpenTelemetry and plugin hooks, but it does not fully implement the PR’s cron/subagent delivery-confirmation fix: gateway agent delivery still returns payloads/meta without explicit external-delivery confirmation, and completion direct announce still treats visible gateway payloads as delivered.

## Best Possible Solution

Do not merge this PR as-is and do not close it as implemented. Split a small core PR that returns explicit external-delivery confirmation from gateway agent delivery, then require that confirmation for completion-mode subagent/cron direct announces or fall back to direct send when absent. Add regression coverage for payload-only gateway responses and best-effort external delivery failures. Move the Langfuse PoC to a ClawHub/plugin adapter using diagnostics events, plugin hooks, and agent event subscriptions; propose a generic plugin API change only if that adapter proves a missing seam.
