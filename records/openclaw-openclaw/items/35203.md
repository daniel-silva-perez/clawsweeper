## Summary

Current main has useful adjacent infrastructure, but it still does not implement the central RFC: learned capability profiling/routing, a queryable shared blackboard, private/team/global memory visibility boundaries, or token/cost budget enforcement. The recent Octo/teleconference and KinthAI comments are useful prior art, but they are partial, external, fork/prototype-level, or narrower than this umbrella request.

## Best Possible Solution

Keep this umbrella RFC open for maintainer product/API follow-up, then split it into focused issues: capability metrics and routing, blackboard/artifact search/query, memory visibility and promotion gates, token/cost budget aggregation and pre-call enforcement, and delegation privilege boundaries. Encourage plugin-first prototypes through context engines, memory plugins, lifecycle hooks, and api.runtime.subagent, and file narrower core/SDK issues where those seams cannot express the needed behavior.

## Risks / Open Questions

- The RFC is broad enough that maintainers should split it before accepting implementation work.
- Some pieces can be prototyped through plugins, context engines, memory plugins, and api.runtime.subagent today, but hard budget enforcement, pre-spawn routing, and visibility/privilege boundaries likely need explicit core or SDK design.
- Shared memory and blackboard features have privacy and privilege-escalation implications in multi-agent and multi-tenant setups; they need careful owner review rather than auto-close cleanup.
