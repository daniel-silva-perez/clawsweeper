## Summary

This issue should stay open. Current main still has adjacent multi-agent infrastructure, but it does not implement the RFC’s central capabilities: learned capability/performance routing, a queryable shared blackboard, private/team/global memory visibility boundaries, or token/cost budget enforcement. The Octo, teleconference, and KinthAI comments are useful prior art and implementation guidance, not a completed or fully superseding OpenClaw implementation.

## Best Possible Solution

Keep this umbrella RFC open for maintainer product/API follow-up, then split it into focused issues: capability metrics and routing, artifact/blackboard search and query APIs, memory visibility and promotion gates, token/cost aggregation with pre-call enforcement, and delegation privilege boundaries. Encourage plugin-first prototypes where current APIs are sufficient, and file narrower core/SDK issues where plugin surfaces cannot express the required behavior.

## Risks / Open Questions

- The RFC is an umbrella product/API design item and is too broad to implement safely as one change.
- Capability routing, shared blackboards, layered memory, and token budgets interact with delegation authority, privacy, and multi-tenant privilege boundaries, so they need explicit maintainer design review.
- Some pieces can be prototyped through plugins, context engines, memory plugins, lifecycle hooks, and api.runtime.subagent today, but hard pre-spawn enforcement and cross-agent visibility semantics likely need core or SDK design.
