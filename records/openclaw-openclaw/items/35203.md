## Summary

Keep this issue open. Current main still has isolated subagents, thread-bound sessions, plugin subagent/runtime hooks, memory search, and token/cost reporting, but it does not implement the RFC's central four layers: learned capability profiling/routing, a shared blackboard with query/pub-sub, private/team/global memory boundaries, or token/cost budget enforcement. The Octo/teleconference and KinthAI comments are useful prior art and implementation direction, but the referenced work is partial, external, fork-only, or still proposed rather than shipped in this checkout.

## Best Possible Solution

Keep this umbrella RFC open until maintainers split it into narrower product/API follow-ups: capability metrics and routing, blackboard or artifact search/query, memory visibility with promotion gates, and token/cost budget aggregation plus enforcement. Encourage plugin-first experiments through runtime.subagent, context-engine, memory, and subagent lifecycle hooks where they fit, and file focused core/plugin API issues where those seams cannot express pre-spawn routing, shared visibility, telemetry, or enforcement.
