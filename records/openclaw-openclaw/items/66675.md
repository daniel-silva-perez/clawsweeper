## Summary

Keep open. Current main still has the implicated followup-warning behavior and the restart lifecycle path still converts a failed post-restart health check into a failed restart result after systemd restart returns completed. The same-author PR #66685 remains an open, narrow warning-noise cleanup and explicitly does not claim to fix the false non-zero restart behavior.

## Best Possible Solution

Keep this issue open while maintainers review PR #66685 as a narrow expired-approval warning cleanup. The full fix should make a completed Linux/systemd restart exit successfully when follow-up status/probe proves the gateway is healthy, or add a regression/manual proof showing current main already behaves that way before closing.
