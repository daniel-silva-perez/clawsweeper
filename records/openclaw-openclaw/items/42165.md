## Summary

Keep open. Current main still widens restrictive tool profiles through profile-stage implicit grants: configured `tools.fs` and `tools.exec` sections are converted into additional allowed tools before final filtering, while the docs and catalog define `minimal` and `messaging` as narrower profiles.

## What This Changes

The issue asks for restrictive tool profiles, especially `minimal` and the related `messaging` case, to stop gaining filesystem or runtime tools merely because `tools.fs` or `tools.exec` configuration blocks are present.

## Best Possible Solution

Make `tools.fs` and `tools.exec` blocks configure behavior only for tools already granted by the effective profile or by explicit allow/alsoAllow policy, preserve deny precedence, and add regression coverage for both `minimal` and `messaging` profiles.
