## Summary

Keep open. Current main still has the reported profile-blind implicit grant path: configured `tools.fs` and `tools.exec` sections are converted into profile-stage allow entries before final filtering, even though the docs and catalog keep `minimal` and `messaging` narrower. Related PR #43061 is still open and #42308 was closed unmerged, so this is not implemented on main.

## What This Changes

The issue reports a regression where restrictive tool profiles, especially `minimal` and later `messaging`, still expose filesystem or runtime tools when `tools.fs` or `tools.exec` config blocks are present.

## Best Possible Solution

Make `tools.fs` and `tools.exec` blocks configure behavior only for tools already granted by the effective profile or an explicit allow/alsoAllow entry, preserve deny precedence, and add regression coverage for both `minimal` and `messaging` profiles.
