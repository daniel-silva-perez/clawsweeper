## Summary

Current main still appears to have the reported least-privilege regression. The documented profile contract excludes filesystem and runtime tools from `minimal` and `messaging`, but the policy code still implicitly adds `read`, `write`, `edit`, `exec`, and `process` whenever `tools.fs` or `tools.exec` config blocks exist, and current tests still assert that behavior for `messaging`. The issue should remain open while maintainers review or split the open implementation candidate #43061 and coordinate the related exec report #47487.

## What This Changes

The issue requests that restrictive tool profiles such as `minimal` and `messaging` not expose filesystem or runtime tools merely because `tools.fs` or `tools.exec` configuration blocks are present.

## Best Possible Solution

Keep this issue open until the policy code only treats `tools.fs` and `tools.exec` blocks as configuration for tools already granted by the effective profile or explicit `alsoAllow`, while preserving deny precedence. Add targeted regression tests for `minimal` and `messaging`, then land or split the relevant part of #43061 and use #47487 as the related runtime-tool coverage point.
