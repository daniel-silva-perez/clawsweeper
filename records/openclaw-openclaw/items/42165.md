## Summary

Keep open. Current main still has a profile-blind implicit grant path: configured `tools.fs` and `tools.exec` sections are merged into `profileAlsoAllow` before the normal profile filter, while the docs and catalog keep `minimal` to `session_status` and keep filesystem/runtime tools out of `messaging`. The issue is already paired with open implementation candidate #43061, so ClawSweeper should not queue a second repair lane.

## What This Changes

The issue asks OpenClaw to enforce restrictive tool profiles so `minimal` and `messaging` do not expose filesystem or runtime tools unless explicitly allowed.

## Best Possible Solution

Make `tools.fs` and `tools.exec` blocks configure behavior only for tools already granted by the effective profile or explicit allowlist; preserve explicit `alsoAllow` and deny precedence, add regression coverage for both `minimal` and `messaging`, and review or split #43061 rather than duplicating the work.
