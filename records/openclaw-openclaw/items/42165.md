## Summary

Current main still appears to match the reported regression. `resolveImplicitProfileAlsoAllow()` adds `read`, `write`, and `edit` whenever a `tools.fs` block is configured, without checking the effective `minimal` or `messaging` profile, and that implicit list is merged into the profile allowlist before the tool policy pipeline filters tools. Docs and the catalog still define `minimal` as `session_status` only and keep filesystem tools out of `messaging`; an open implementation candidate (#43061) still references this work, so this issue should remain open until a fix lands.

## Best Possible Solution

Keep this issue open. The best fix is to update `src/agents/pi-tools.policy.ts` so `tools.fs` and `tools.exec` configure already-granted tools but do not implicitly widen restrictive profiles like `minimal` or `messaging` unless the user explicitly opts in through `alsoAllow` or `allow`, while preserving deny precedence. Add regression coverage for both global and per-agent `minimal`/`messaging` profiles with configured `tools.fs`/`tools.exec`, then review or split #43061 as the active implementation candidate.
