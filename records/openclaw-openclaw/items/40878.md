## Summary

Keep this PR open. Current main still does not contain the PR's two active shared native chat UI fixes: the thinking picker still lacks an explicit adaptive option, and streaming assistant text still mutates scrollPosition synchronously inside the SwiftUI onChange handler. The current PR diff is narrow and not security-sensitive, but maintainers should clarify the stale OSLog wording before landing.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or replace the narrow native chat UI fix: add Adaptive consistently to the known picker levels, defer streaming auto-scroll past the current layout pass while revalidating pin state inside the deferred task, and clean up the stale OSLog wording. Longer term, move the shared native picker toward the gateway/provider thinkingLevels contract already documented for chat pickers.
