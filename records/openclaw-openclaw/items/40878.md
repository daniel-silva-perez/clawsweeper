## Summary

Keep this PR open. Current main still lacks the PR's two observable SwiftUI fixes: the native chat thinking picker does not include an explicit Adaptive option, and streaming assistant updates still set scrollPosition synchronously in the onChange handler. Adaptive is a valid current thinking level, and the broader provider-driven native picker path is not present in OpenClawChatSessionEntry yet.

## Best Possible Solution

Keep this PR open for maintainer review. The practical path is to validate and land the narrow native chat UI fix, or replace it with a provider-driven native thinking picker that consumes gateway thinkingLevels/thinkingOptions/thinkingDefault plus a deferred streaming-scroll update with pin-state revalidation. Also clean up the stale OSLog wording if that scope is no longer in the diff.
