## Summary

Keep this issue open. Current main still does not expose or thread a `tools.exec.shell` contract: shell selection is hard-coded in the exec helper, the runtime calls that helper directly, config types/schema do not accept `shell`, and the public exec docs describe only the current PowerShell/SHELL behavior. The request also has an open related implementation PR, so this needs maintainer review rather than stale cleanup or a duplicate automated repair lane.

## What This Changes

Add a configurable `tools.exec.shell` override so exec tool invocations can use a user-selected shell such as WSL Bash, Git Bash, zsh, or PowerShell instead of the built-in defaults.

## Best Possible Solution

Define a supported exec shell override contract with global and per-agent precedence, executable and argument semantics, startup or first-use validation, Windows/WSL cwd and PATH expectations, approval and safe-bin interactions, schema/help/docs updates, and targeted exec tests, then land it through PR #39413 or a maintainer-directed replacement.
