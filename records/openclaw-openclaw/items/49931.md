## Summary

Keep this issue open. Current main still hard-wires exec shell selection and has no `tools.exec.shell` type, schema, runtime wiring, or public docs entry, while the related implementation PR remains open and needs maintainer review because this is a command-execution semantics change.

## What This Changes

Request adding a `tools.exec.shell` config option so users can choose the default shell used by exec tool invocations instead of the current platform defaults.

## Best Possible Solution

Define a supported exec shell override contract, including global and per-agent precedence, executable and argument semantics, startup validation behavior, Windows/WSL cwd and PATH expectations, approval/safe-bin interactions, docs/schema metadata, and targeted exec tests, then land it through PR #39413 or a maintainer-approved rework.
