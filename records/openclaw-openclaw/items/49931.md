## Summary

Keep this open. Current main still has no `tools.exec.shell` config contract or runtime wiring, and the related implementation PR remains open/unmerged, so closing as stale or implemented would be premature.

## What This Changes

The issue requests a configurable `tools.exec.shell` override so exec commands can use a user-selected shell such as WSL bash, Git Bash, bash, zsh, or PowerShell instead of the platform default.

## Best Possible Solution

Define a supported exec shell override contract, including global and per-agent precedence, executable-plus-argument semantics, validation behavior, Windows/WSL cwd and PATH expectations, approval/safe-bin interactions, docs/schema metadata, and targeted exec tests, then land that through the existing related implementation path or a maintainer-approved rework.
