## Summary

Current main still does not expose or consume a configurable `tools.exec.shell` override. The exec runtime chooses shells through `getShellConfig()` with Windows PowerShell defaults and POSIX `$SHELL`/`sh` fallback, while the exec config type, strict zod schema, schema help/docs, and public exec docs have no `shell` field. Related PR #39413 remains open and unmerged in the provided GitHub context, so this issue should stay open for maintainer review rather than close as implemented or stale.

## Best Possible Solution

Keep this issue open as the canonical tracker and review or rework #39413. A complete fix should define the global and per-agent `tools.exec.shell` contract, shell executable and argument semantics, validation behavior, Windows/WSL cwd and PATH expectations, and approval/safe-bin interactions, then update config types, strict zod schema, schema help/labels/generated schema, public docs, and targeted exec tests.
