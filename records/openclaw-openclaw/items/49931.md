## Summary

Current main still does not implement a configurable `tools.exec.shell`. Host exec uses the platform-derived `getShellConfig()` path, the exec defaults and generated config schema expose no `shell` field, and docs still describe Windows as PowerShell-first. The related open PR #39413 is an implementation candidate, not landed behavior, so this issue should remain open.

## Best Possible Solution

Keep this issue open. Maintainers should review #39413 or a replacement implementation that adds a typed `tools.exec.shell` contract, schema/docs coverage, runtime propagation, validation, and tests for Windows, POSIX shells, WSL/path translation, approval/safeBins behavior, and config loss or invalid-shell diagnostics.
