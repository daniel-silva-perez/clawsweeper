## Summary

Current main still does not implement a configurable `tools.exec.shell` override. Host exec shell selection is hard-wired through `getShellConfig()`, the exec runtime calls that helper without config input, and the public config type/schema/help/docs do not expose a `shell` field. Related PR #39413 remains open and unmerged, so the right maintainer action is to review or rework that implementation rather than close this as stale or implemented.

## What This Changes

Add a global and/or per-agent `tools.exec.shell` config option so exec can use a caller-configured shell, especially for Windows users who want WSL, Git Bash, or another shell instead of PowerShell.

## Best Possible Solution

Keep this issue open as the canonical tracker and review or rework #39413. A complete fix should define global and per-agent shell override semantics, executable/argument handling, validation behavior, Windows/WSL cwd and PATH expectations, and approval/safe-bin interactions, then update config types, strict zod schema, schema help/labels/generated metadata, public exec docs, and targeted exec tests.
