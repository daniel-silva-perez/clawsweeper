## Summary

Keep open. The report is concrete and current main still lacks clear proof that the heartbeat transcript marker is filtered from Telegram delivery; additionally, open PR #75167 explicitly closes this issue, so this issue should stay open until that PR merges or is rejected.

## What This Changes

Prevent the raw heartbeat poll prompt from appearing in Telegram DM conversations while keeping default `HEARTBEAT_OK` acknowledgments suppressed.

## Best Possible Solution

Review and land PR #75167, or a narrower replacement, so heartbeat prompts and the configured heartbeat marker are suppressed before any external channel message is sent while preserving meaningful heartbeat alert delivery and optional `showOk` behavior.

## Reproduction Assessment

Yes. The issue includes a concrete v2026.4.27 Telegram DM configuration and wait-for-heartbeat path, and current main has enough seams to create a focused regression test that asserts the marker is not sent to Telegram.

## Solution Assessment

Unclear until the linked PR is reviewed. The safest maintainable solution is a shared heartbeat prompt visibility guard with Telegram regression coverage, unless review proves the leak is strictly confined to channel visible-text helpers.
