Keep open. The report is concrete, current main still lacks a clear guard that drops the heartbeat transcript marker before Telegram-visible delivery, and open PR #75167 explicitly closes this issue but remains unmerged.

## What This Changes

The issue requests that Telegram DMs never show the internal `[OpenClaw heartbeat poll]` prompt while continuing to suppress `HEARTBEAT_OK` acknowledgments by default.

## Best Possible Solution

The best end state is a narrow fix that suppresses `HEARTBEAT_TRANSCRIPT_PROMPT` before any external channel send, with Telegram regression coverage and preservation of meaningful heartbeat alert delivery plus optional `showOk` behavior.

## Reproduction Assessment

Yes. The issue provides a concrete v2026.4.27 Telegram DM configuration, and current main exposes a focused reproduction seam: heartbeat runs use the transcript marker while Telegram/outbound delivery lacks a marker filter.

## Solution Assessment

Unclear until PR #75167 is reviewed. A shared outbound guard with channel-level regression coverage appears safer than relying only on Telegram's visible-text accounting helper.
