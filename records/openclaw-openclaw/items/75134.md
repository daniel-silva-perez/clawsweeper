## Summary

Keep open. Current main still has the internal heartbeat transcript marker in the heartbeat run path, Telegram's visible-text helper does not filter by marker text, and the shared outbound delivery path still has no marker drop. Open PR #75167 explicitly targets this issue but remains open and unmerged, so this issue should stay open until that PR lands or is rejected.

## What This Changes

The issue asks for Telegram heartbeat poll prompts to remain internal and never appear as visible user messages in DM conversations, while preserving default suppression of OK-only heartbeat acknowledgments.

## Best Possible Solution

Review PR #75167 or a narrow equivalent that suppresses `HEARTBEAT_TRANSCRIPT_PROMPT` before any external channel send, with Telegram regression coverage and preservation of heartbeat alert delivery plus optional `showOk` behavior.

## Reproduction Assessment

Yes. The issue provides a concrete Telegram DM configuration on v2026.4.27, and current main still has a focused source path where the heartbeat transcript marker is produced without an outbound marker filter.

## Solution Assessment

Unclear for the linked PR until it is reviewed. The maintainable direction appears to be a shared outbound guard with channel-level regression coverage, rather than relying only on Telegram's visible-text accounting helper.
