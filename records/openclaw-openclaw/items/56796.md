## Summary

Keep this PR open. Current main has since solved the timeout lock-contention path with fresh embedded fallback sessions, but it has not fully resolved the gateway-closed ambiguity this PR targets, and the PR is now dirty against main with a broad string classifier that needs maintainer policy review before landing.

## What This Changes

The PR makes `openclaw agent` throw instead of using embedded fallback for gateway timeout, gateway request-timeout, and gateway-closed errors, with focused tests for those fail-closed cases.

## Best Possible Solution

Rebase the work onto current main and implement a structured, phase-aware policy for ambiguous agent gateway failures: keep current fresh-session timeout fallback, preserve recoverable pre-request fallback, and only fail closed or isolate fallback when the gateway-side agent request may actually be active.

## Reproduction Assessment

Yes. A focused unit path can mock `callGateway` to reject with timeout, request-timeout, pre-request `GatewayTransportError` close, and in-flight close shapes, then assert whether `agentCommand` is called with the original session, a fresh fallback session, or not called.

## Solution Assessment

No. The branch’s message-substring classifier is not the best current solution because main now has structured transport errors and a documented fresh-session timeout fallback; the safer direction is to classify transport phase/kind explicitly and preserve current timeout isolation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Distinguish pre-request gateway closes:** `src/commands/agent-via-gateway.ts:191`
  - body: The new classifier treats every `gateway closed` message as an in-flight remote agent and throws before embedded fallback. `callGateway` can also emit that close error from `onClose` before `onHelloOk` sends the request, so a startup/handshake close would lose the recoverable fallback even though no remote turn is active. Please classify the request phase instead of matching all close messages.
  - confidence: 0.84
