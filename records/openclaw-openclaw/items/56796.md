## Summary

Keep this PR open for maintainer review rather than close it as implemented. Current main already avoids the timeout lock-contention problem with a fresh embedded fallback session, but the PR branch still proposes broad fail-closed behavior for timeout and closed-message strings and introduces a recoverable pre-request close regression.

## What This Changes

The PR branch makes `openclaw agent` throw without embedded fallback for gateway timeout, gateway request-timeout, and gateway-closed error messages, with focused tests for those fail-closed cases.

## Best Possible Solution

Rebase the work onto current main and implement a structured, phase-aware policy for ambiguous agent gateway failures: preserve fresh-session timeout fallback, preserve embedded fallback for pre-request recoverable closes, and fail closed or isolate fallback only when the gateway-side agent request may actually still be active.

## Reproduction Assessment

Yes. A focused unit path can mock `callGateway` to reject with a timeout `GatewayTransportError`, `gateway request timeout for agent`, a pre-request closed transport error, and a pending-request close, then assert whether `agentCommand` is called with a fresh fallback session or not called.

## Solution Assessment

No. The branch's message-substring classifier is not the best current solution because main now has structured transport errors and a documented fresh-session timeout fallback; the safer fix is explicit phase/kind classification.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Distinguish pre-request gateway closes:** `src/commands/agent-via-gateway.ts:189`
  - body: The new classifier treats every `gateway closed` message as an in-flight remote agent and throws before embedded fallback. `callGateway` can also receive `onClose` before `onHelloOk` sends the agent request, so a startup or handshake close would lose recoverable fallback even though no remote turn is active. Please classify the request phase instead of matching all close messages.
