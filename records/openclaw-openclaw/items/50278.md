## Summary

Keep this PR open. Current main still treats leading Markdown image text like `![image](url)` as bash-bang input in the TUI, bash command handler, parser, and ACP bypass path. The PR head is still a relevant implementation candidate because it adds a narrower Markdown-image predicate and regression coverage while preserving shell-test bang inputs like `![ -f package.json ] && echo ok`.

## Best Possible Solution

Keep this PR open as a valid implementation candidate. The best path is to review or land the corrected head, or apply an equivalent narrow Markdown-image exclusion across TUI submit handling, bash request parsing, bash command routing, and ACP bypass, with tests for both `![alt](url)` normal text and `![ -f file ]` bash commands.
