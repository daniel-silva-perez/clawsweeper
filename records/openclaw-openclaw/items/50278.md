## Summary

Keep this PR open. Current main still treats leading Markdown image text like `![alt](url)` as bang/local-shell or bash-command input in the TUI and command paths, and there is no current-main regression coverage for the reported case. The PR is still a relevant implementation candidate, especially after the follow-up commits addressed the earlier review concern by preserving `![ -f ... ]` shell-test aliases and reverting the unnecessary inline command-detection regex change.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rebase it onto current main, put the TUI guard in `src/tui/tui-submit.ts` and the ACP guard in `src/auto-reply/reply/dispatch-acp-command-bypass.ts`, share a precise Markdown-image predicate for complete `![alt](url)` syntax across the bash parser/handler/bypass/TUI paths, and add regression coverage proving `![alt](url)` is normal text while `![ -f file ]` remains a valid bang bash command.
