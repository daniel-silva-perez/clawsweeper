## Summary

Keep this PR open. Current main still routes leading `!` text through the bash/command/ACP paths with no Markdown-image carve-out, and the current tests do not cover `![alt](url)`. The PR remains a relevant implementation candidate, but it needs rebasing/adaptation to current main's split files and must preserve valid bash aliases such as `![ -f package.json ] && echo ok`.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rebase it onto current main, put a shared Markdown-image predicate in the current TUI, bash parser/handler, command-auth, and ACP bypass surfaces as needed, and land regression coverage for `![alt](url)` as normal text plus `![ -f file ]` as a valid bash bang command.
