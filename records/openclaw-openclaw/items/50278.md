## Summary

Keep this PR open. Current main still has the reported behavior: TUI submit handling, bash request parsing, command routing, and ACP command bypass all treat leading bang text generically, with no Markdown-image carve-out. The PR is still a relevant implementation candidate, but it should be rebased/adapted to current main's split files and keep the precise predicate that preserves shell-test commands like `![ -f package.json ] && echo ok`.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is to rebase it onto current main, put the shared Markdown-image predicate in the current command-routing files, and land regression coverage for TUI submit handling, bash parser/routing, and ACP bypass, including both `![alt](url)` as normal text and `![ -f file ]` as a valid bash bang command.
