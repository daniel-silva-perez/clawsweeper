## Summary

Keep this PR open. Current main still treats leading Markdown image syntax such as `![alt](url)` as bang/bash input in the TUI, bash parser, command handler, and ACP bypass path, and current tests do not cover the reported regression. The PR remains a relevant implementation candidate, with the main maintainer concern being to keep the predicate narrow enough that valid bash aliases like `![ -f package.json ] && echo ok` still work.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rebase it onto current main, add a shared Markdown-image predicate used by the current TUI submit handler, bash parser/handler, command authorization or detection path if needed, and ACP command bypass path, then land regression coverage showing `![alt](url)` is normal text while `![ -f file ]` remains a valid bang bash command.
