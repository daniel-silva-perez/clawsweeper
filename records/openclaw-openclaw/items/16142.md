## Summary

Keep open: current main still routes main-session cron reminder text through the heartbeat cron-event prompt and model-backed reply path, so the reported repeated OpenAI/ChatGPT cost behavior remains present. The related open cost-cap PR may help bound expensive runs, but it does not replace a deterministic delivery boundary for simple reminders.

## What This Changes

The issue requests reducing repeated ChatGPT/OpenAI calls for scheduled Slack/email watcher reminders by replacing simple model-backed reminder relaying with deterministic string formatting.

## Best Possible Solution

Define a narrow low-cost delivery mode for simple main-session cron reminders, either through an explicit cron option or a conservative simple-payload detector, while preserving the existing model-backed cron-event path for complex, internal, tool-using, or reasoning-dependent events. Update cron/heartbeat docs and focused tests for both paths.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: enqueue a tagged cron system event, run heartbeat, and observe Provider cron-event plus a getReplyFromConfig call; existing tests assert that path.

## Solution Assessment

Unclear. Deterministic formatting is directionally right for simple reminders, but the best maintainable fix needs a defined boundary so complex cron events keep the current model-backed semantics.
