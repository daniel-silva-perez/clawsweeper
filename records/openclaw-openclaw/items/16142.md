## Summary

Keep open: current main still routes main-session cron reminder text through the heartbeat cron-event prompt and model-backed reply path, so the reported OpenAI/ChatGPT cost behavior remains present. Prior maintainer-visible reviews also kept this open, and the remaining fix needs a product boundary for when deterministic delivery may safely bypass the model.

## What This Changes

Request to stop using ChatGPT/OpenAI model calls as a formatter for frequent simple cron/heartbeat reminders and provide a lower-cost deterministic delivery path.

## Best Possible Solution

Define and implement a narrow low-cost delivery path for simple main-session cron reminders, preferably through an explicit cron option or conservative simple-payload detector, while preserving the model-backed cron-event path for complex, internal, tool-using, or reasoning-dependent events. Update focused cron/heartbeat tests and the public cron/heartbeat docs to lock down that boundary.
