## Summary

Keep open. Current main still routes main-session cron reminder text through the heartbeat cron-event prompt and model-backed reply path, and the relevant cron/heartbeat source, tests, and docs remain unchanged since the prior keep-open review. The report describes a real cost problem, but the fix needs maintainer product judgment about which cron events may safely bypass model reasoning.

## What This Changes

Request to stop using OpenAI/ChatGPT model calls to format simple recurring main-session cron/watch reminder text and use a lower-cost deterministic delivery path instead.

## Best Possible Solution

Define a narrow low-cost delivery mode for simple main-session cron reminders, preferably via an explicit cron option or conservative simple-payload detector, while preserving the existing model-backed cron-event path for complex, internal, tool-using, or reasoning-dependent events. Document the boundary and add focused cron/heartbeat tests for both paths.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: enqueue a tagged main-session cron system event, run heartbeat, and observe the cron-event prompt plus getReplyFromConfig call; current tests assert this model-backed behavior.

## Solution Assessment

Unclear. Deterministic formatting is directionally right for simple reminders, but the requested blanket model bypass is too broad unless maintainers first define which cron events are safe to deliver without model interpretation.
