## Summary

Keep open. Current main still routes main-session cron `systemEvent` reminders through the heartbeat reply/model path instead of a deterministic formatter, and the existing maintainer review comment also keeps this open for product follow-up. The related #15403 is closed unmerged and addressed shell rc/env loading, not model calls for reminders.

## Best Possible Solution

Keep this open for maintainer product judgment, then implement a narrow deterministic path only for clearly simple main-session reminder delivery, or add an explicit cron payload/option that chooses deterministic delivery. Preserve model-mediated cron events for payloads that need reasoning or tools, and cover routing, event draining, delivery visibility, and `target=last` behavior with focused regression tests.
