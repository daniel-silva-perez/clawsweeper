## Summary

Keep open. Current main still routes main-session cron system-event reminders through the heartbeat reply/model path, so short-interval reminder/watch jobs can still create paid model traffic for simple relay formatting. The earlier maintainer-visible comment also kept this open, and the related #15403 is closed unmerged and addresses shell rc/env loading rather than this cost path.

## Best Possible Solution

Keep this open for maintainer product judgment, then implement a narrow deterministic delivery path only for clearly simple main-session reminder payloads, or add an explicit cron option that chooses deterministic delivery. Preserve the model-mediated path for cron payloads that need reasoning or tools, and cover routing, event draining, delivery visibility, and target=last behavior with focused tests.
