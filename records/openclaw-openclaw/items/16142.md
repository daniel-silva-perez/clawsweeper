## Summary

Keep open: current main still sends main-session cron reminder text through the heartbeat cron-event prompt and model-backed reply path, so the reported OpenAI/ChatGPT cost behavior remains present. Prior maintainer-visible review also kept this open, and the remaining work needs a product boundary for when deterministic delivery is safe.

## What This Changes

The issue asks OpenClaw to avoid using ChatGPT/OpenAI as a formatter for frequent cron reminder delivery because simple recurring reminders can generate high API usage.

## Best Possible Solution

Define and implement a narrow low-cost delivery path for simple main-session cron reminders, preferably through an explicit cron option or conservative simple-payload detector, while retaining the current model-backed cron-event path for complex, internal, tool-using, or reasoning-dependent events. Update the cron/heartbeat docs and focused tests to make that boundary explicit.
