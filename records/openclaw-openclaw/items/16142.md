## Summary

Keep open: current main still sends main-session cron reminder text through the heartbeat cron-event prompt and model-backed reply path, so the reported cost behavior remains present. This is not closeable as implemented; the remaining work needs maintainer/product judgment about which cron reminders may bypass the model safely.

## What This Changes

The issue asks OpenClaw to stop using ChatGPT/OpenAI model calls as a formatter for simple recurring cron reminder text and add a lower-cost deterministic delivery path.

## Best Possible Solution

Define a narrow low-cost delivery mode for simple main-session cron reminders, either through an explicit cron option or a conservative simple-payload detector, while preserving the existing model-backed cron-event path for complex, internal, tool-using, or reasoning-dependent events. Update cron/heartbeat docs and focused tests for both paths.
