## Summary

Keep open: current main still routes main-session cron reminder text through the heartbeat `cron-event` prompt and `getReplyFromConfig(...)`, so the reported OpenAI/ChatGPT cost path is still present. Existing docs and tests preserve model-backed heartbeat behavior, and adjacent open PRs do not yet prove that a low-cost deterministic reminder delivery path has landed on main.

## What This Changes

Request to stop using ChatGPT/OpenAI model calls to reformat simple cron-driven Slack/email reminders because frequent watchers create excessive API traffic and throttling.

## Best Possible Solution

Add a narrowly scoped deterministic delivery path for simple main-session cron reminders, preferably behind an explicit cron option or conservative simple-payload detector, while keeping the existing model-backed cron-event path for complex, internal, tool-using, or reasoning-dependent events. Update heartbeat/cron docs and focused regression tests to define that boundary.
