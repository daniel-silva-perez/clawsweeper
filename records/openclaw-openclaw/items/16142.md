## Summary

Keep open: current main still routes main-session cron reminder text through the heartbeat cron-event prompt and model-backed reply path, so the reported cost/throttling behavior remains present. Prior review comments correctly identified that the remaining fix needs a product boundary for which cron reminders may bypass the model.

## What This Changes

The issue asks OpenClaw to stop using the ChatGPT/OpenAI model path as a formatter for recurring cron reminders that could be delivered deterministically.

## Best Possible Solution

Define a narrow low-cost delivery mode for simple main-session cron reminders, either as an explicit cron option or a conservative simple-payload detector, while preserving the existing model-backed cron-event path for complex, internal, tool-using, or reasoning-dependent events. Document the boundary and add focused cron/heartbeat coverage for both paths.
