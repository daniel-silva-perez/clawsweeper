## Summary

Keep open. Current main still routes main-session cron reminder text through the heartbeat cron-event prompt and model-backed reply path, so the reported repeated OpenAI/ChatGPT cost behavior remains present. Existing cost-reduction docs and the related cost-cap PR do not replace a deterministic delivery boundary for simple reminders.

## What This Changes

The issue asks OpenClaw to stop using ChatGPT/OpenAI model calls to rephrase simple scheduled Slack/email watcher reminders and use deterministic formatting instead.

## Best Possible Solution

Introduce an explicit low-cost mode or conservative simple-reminder detector for main-session cron reminders, with docs and tests defining which events may be delivered deterministically and which must keep the existing model-backed cron-event semantics.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: enqueue a tagged cron system event, run heartbeat, and observe Provider cron-event plus a getReplyFromConfig call; existing tests assert that behavior.

## Solution Assessment

Unclear. Deterministic formatting is directionally right for simple reminders, but the maintainable fix needs a defined boundary so complex cron events keep current model-backed behavior.
