## Summary

Keep open: current main still routes main-session cron reminder text through the heartbeat cron-event prompt and model-backed reply path, so the reported repeated OpenAI/ChatGPT cost behavior remains present. The related open cost-cap PR may help limit expensive runs, but it does not replace the requested deterministic delivery boundary for simple reminders.

## What This Changes

The issue requests a lower-cost deterministic formatting and delivery path for recurring cron reminder text instead of repeatedly sending simple reminder payloads through ChatGPT/OpenAI.

## Best Possible Solution

Define a narrow low-cost delivery mode for simple main-session cron reminders, either through an explicit cron option or a conservative simple-payload detector, while preserving the existing model-backed cron-event path for complex, internal, tool-using, or reasoning-dependent events. Update cron/heartbeat docs and focused tests for both paths.
