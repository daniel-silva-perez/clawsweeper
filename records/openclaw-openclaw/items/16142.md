## Summary

Keep open. Current main still routes main-session cron reminder text through the heartbeat cron-event prompt and model-backed reply path, so the reported repeated OpenAI/ChatGPT cost behavior remains present. Existing docs describe cost-reduction knobs for heartbeat runs, but they do not provide a deterministic delivery boundary for simple reminders.

## What This Changes

The issue requests avoiding ChatGPT/OpenAI model calls for simple recurring cron/reminder text formatting to reduce high API usage and throttling.

## Best Possible Solution

Introduce a narrowly scoped low-cost mode or conservative simple-reminder detector for main-session cron reminders, with docs and focused tests defining which events may be delivered deterministically and which must keep the existing model-backed cron-event behavior.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: enqueue a tagged cron system event, run heartbeat, and observe Provider cron-event plus a getReplyFromConfig call; existing tests assert this behavior.

## Solution Assessment

Unclear. Deterministic formatting is directionally right for simple reminders, but the maintainable fix needs a defined boundary so complex cron events keep current model-backed semantics.
