## Summary

Keep open. Current main still routes main-session cron system-event reminders through the heartbeat `cron-event` prompt and model-backed `getReplyFromConfig(...)` path, and the current tests preserve that behavior. The report is valid but the right fix needs maintainer product judgment around when deterministic delivery may bypass the model.

## What This Changes

Request reducing repeated OpenAI/ChatGPT API usage for simple scheduled reminder and watcher text that OpenClaw currently relays through the model.

## Best Possible Solution

Add a narrowly scoped deterministic delivery path for simple main-session cron reminders, preferably behind an explicit cron option or a conservative simple-payload detector, while preserving the existing heartbeat `cron-event` model path for complex, internal, tool-using, or reasoning-dependent events. Document the behavior and cover routing, draining, target `last`, and visibility cases with focused tests.
