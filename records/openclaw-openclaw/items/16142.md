## Summary

Keep open. Current main still routes main-session cron systemEvent reminders through the heartbeat reply/model path: cron enqueues the text, heartbeat wraps it as a scheduled reminder, and runHeartbeatOnce calls getReplyFromConfig. The prior Codex review comment reached the same conclusion, and the linked #15403 is closed unmerged and addressed shell rc/env loading rather than deterministic reminder delivery.

## Best Possible Solution

Keep this open until maintainers decide the product boundary for simple text reminders versus cron events that need reasoning. The likely fix is a small, explicit deterministic delivery path around executeMainSessionCronJob, resolveHeartbeatRunPrompt, or a new helper that bypasses getReplyFromConfig only for clearly simple reminder text while preserving model-mediated cron events, routing, visibility, event draining, and regression coverage.
