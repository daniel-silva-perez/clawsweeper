## Summary

Keep open. Current main still sends main-session cron reminder text through the heartbeat cron-event prompt and model-backed reply path, and the tests/docs preserve that behavior. The requested low-cost deterministic delivery path is still missing and needs maintainer product judgment around which reminders may bypass the model.

## What This Changes

The issue requests reducing paid OpenAI/ChatGPT usage for simple recurring cron reminders by formatting or delivering them deterministically instead of using the model as a string formatter.

## Best Possible Solution

Add a narrowly scoped deterministic delivery path for simple main-session cron reminders, ideally behind an explicit cron option or a conservative simple-payload detector, while keeping the existing cron-event model path for complex, internal, tool-using, or reasoning-dependent events. Update docs and focused cron/heartbeat tests to define the boundary.
