## Summary

Current main still appears to have the reported remote skill bin probe timing problem. The gateway starts `refreshRemoteNodeBins` during node connect before sending the successful connect response, while the Swift client consumes frames in `waitForConnectResponse` and only starts the normal listener afterward. Current main also has no co-located/loopback skip for remote bin probing.

## Best Possible Solution

Keep this issue open. The best fix is to defer the initial `refreshRemoteNodeBins` until after node clients have completed connect and started normal event handling, or buffer/dispatch early events received while waiting for the connect response. Then decide whether same-host loopback macOS app nodes should skip remote bin probing when the gateway already has local shell access, and add regression coverage for both the connect-time probe and any co-located skip policy.
