## Summary

Keep open. Current main still matches the reported behavior: omitted QMD `searchMode` resolves to lexical `search`, and both the embed run and embed timer paths return before `embedInterval` can trigger automatic embedding. The issue is also already paired with open PR #73512 using closing syntax, so maintainers should review that PR rather than closing this tracker or queueing duplicate work.

## What This Changes

The issue requests that QMD embedding run automatically according to `memory.qmd.update.interval` or `memory.qmd.update.embedInterval` after OpenClaw changes memory files.

## Best Possible Solution

Review PR #73512 as the canonical implementation candidate. The desired end state should either treat explicit `memory.qmd.update.embedInterval` as an embedding opt-in while preserving conservative defaults, or keep lexical-mode suppression and make the required `searchMode: "vsearch"` or `"query"` guidance more visible where users see pending embeddings.

## Reproduction Assessment

Yes. Current main has a focused unit-level reproduction: configure QMD with `searchMode: "search"` and `embedInterval: "5m"`, advance the timer, and the existing test expects no `update` or `embed` calls.

## Solution Assessment

Unclear. PR #73512 is a plausible narrow fix, but the best solution depends on maintainer judgment about whether explicit `embedInterval` should override the lexical-mode vector skip or whether the current behavior should remain with clearer guidance.
