## Summary

Keep open. Current main still treats the default QMD mode as lexical search and returns before the embed interval can schedule or run embedding, while open PR #73512 already targets this exact scheduler behavior with closing syntax, so maintainers should review that PR instead of closing this issue or queuing duplicate work.

## What This Changes

The issue asks OpenClaw to automatically run QMD embedding according to `memory.qmd.update.embedInterval` after memory changes, even when `memory.qmd.searchMode` is omitted and defaults to lexical `search`.

## Best Possible Solution

Review PR #73512 as the canonical implementation candidate. The desired end state should either make explicit `embedInterval` opt into scheduled embedding while preserving conservative defaults, or keep lexical-mode suppression and make the required `searchMode: "vsearch"` or `"query"` guidance visible where users see pending embeddings.

## Reproduction Assessment

Yes. Current main has a focused unit-level reproduction: configure QMD with lexical `searchMode: "search"` and `embedInterval: "5m"`, advance the timer, and the existing test expects no `update` or `embed` calls.

## Solution Assessment

Unclear. PR #73512 is a plausible narrow fix, but the best solution depends on maintainer product judgment about whether explicit `embedInterval` should override the lexical-mode vector skip.
