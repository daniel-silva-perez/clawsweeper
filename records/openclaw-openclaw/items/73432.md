## Summary

Keep open. Current main still gates QMD embedding before the configured embed interval can take effect in the default lexical QMD mode, and an open closing PR already targets the same scheduler behavior, so this should stay open for maintainer review rather than be closed or duplicated.

## What This Changes

The issue asks OpenClaw to automatically run QMD embedding according to `memory.qmd.update.interval` or `memory.qmd.update.embedInterval` after QMD-backed memory files change.

## Best Possible Solution

Review PR #73512 as the canonical implementation candidate. The desired end state should either treat an explicit `memory.qmd.update.embedInterval` as an embedding opt-in while preserving conservative defaults, or keep lexical-mode suppression and make the required `searchMode: "vsearch"` or `"query"` guidance visible where users see pending embeddings.

## Reproduction Assessment

Yes. Current main has a focused unit-level reproduction: configure QMD with `searchMode: "search"` and `embedInterval: "5m"`, advance the timer, and the existing test expects no `update` or `embed` calls.

## Solution Assessment

Unclear. PR #73512 is a plausible narrow fix, but the best solution depends on maintainer judgment about whether explicit `embedInterval` should override the documented lexical-mode vector skip.
