## Summary

Keep open. The reported config omits `memory.qmd.searchMode`, so current main uses the default `search` mode, and current code intentionally skips QMD vector probes plus `qmd embed` in that mode even when `memory.qmd.update.embedInterval` is configured. The existing dedicated embed timer only covers vector modes such as `query`/`vsearch`, so the reporter’s observable “pending embedding” state is still plausible on main rather than implemented away.

## Best Possible Solution

Keep this open for maintainer/product handling. A conservative fix would either make explicit `update.embedInterval` opt the QMD backend into embedding maintenance despite default `search` mode, or keep the skip but add a clear config/status diagnostic telling users to set `memory.qmd.searchMode` to `query` or `vsearch` when they expect automatic embeddings.
