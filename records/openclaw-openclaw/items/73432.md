## Summary

Keep open. Current main still gates QMD embedding on vector-capable search modes before `embedInterval` can matter, and an open closing PR already targets the same scheduler behavior, so this issue should wait for maintainer review of that implementation instead of being closed or queued as duplicate repair work.

## What This Changes

The issue requests that QMD automatic embedding run according to configured `memory.qmd.update.interval`/`embedInterval` after OpenClaw updates memory files.

## Best Possible Solution

Review PR #73512 as the canonical implementation candidate. The desired end state should either treat an explicit QMD embed interval as an embedding opt-in while preserving conservative defaults, or keep lexical-mode suppression and make the required vector search mode guidance visible where users encounter pending embeddings.

## Reproduction Assessment

Yes. Current main has a static and unit-level reproduction path: configure QMD in lexical `search` mode with `embedInterval: "5m"`, advance the timer, and the existing test expects no `update` or `embed` calls.

## Solution Assessment

Unclear. PR #73512 is a plausible narrow fix, but the best solution depends on maintainer product judgment about whether `embedInterval` should override the lexical-mode vector guard.
