## Summary

Keep open. Current main still defaults QMD to lexical `search` mode when `searchMode` is omitted, and the QMD manager returns before considering a positive `memory.qmd.update.embedInterval`, so the reported pending-embedding behavior remains reachable. Open PR #73512 targets this exact scheduler gate with closing syntax, so the issue should stay open until that PR is reviewed and merged or maintainers choose a docs/diagnostic-only path.

## What This Changes

The issue asks OpenClaw to automatically trigger QMD embedding according to `memory.qmd.update.interval` and `memory.qmd.update.embedInterval` after memory documents change, instead of leaving documents pending until manual `qmd embed`.

## Best Possible Solution

Review PR #73512 as the canonical implementation candidate. The desired end state should either treat an explicit `memory.qmd.update.embedInterval` as a user opt-in for automatic embeddings while preserving conservative defaults, or keep lexical `search` as no-embed mode and make the status/docs guidance clear enough that users know to set `searchMode` to `vsearch` or `query` for automatic embedding maintenance.
