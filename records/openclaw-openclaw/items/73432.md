## Summary

Keep open. Current main still matches the reported behavior: the QMD manager defaults omitted `searchMode` to lexical `search`, then skips both forced and periodic embedding before considering `memory.qmd.update.embedInterval`. The issue is also paired with open PR #73512 using closing syntax, so the tracker should stay open until maintainers review and merge or close that implementation path.

## What This Changes

The issue reports that OpenClaw does not automatically run QMD embedding according to `memory.qmd.update.embedInterval` when QMD documents become pending after memory changes.

## Best Possible Solution

Review PR #73512 as the canonical implementation candidate. The desired end state should either treat explicit `memory.qmd.update.embedInterval` as an embedding opt-in while preserving conservative defaults, or keep lexical-mode suppression and make the required `searchMode: "vsearch"` or `"query"` guidance visible where users see pending embeddings.

## Reproduction Assessment

Yes. Current main has a focused unit-level reproduction: configure QMD with `searchMode: "search"` and `embedInterval: "5m"`, advance the timer, and the existing test expects no `update` or `embed` calls.

## Solution Assessment

Unclear. PR #73512 is a plausible narrow fix, but the best solution depends on maintainer product judgment about whether explicit `embedInterval` should override the lexical-mode vector skip or whether the current behavior should remain with clearer status/errors.
