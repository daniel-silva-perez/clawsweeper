## What This Changes

The issue requests that QMD embedding run automatically according to `memory.qmd.update.embedInterval` when documents change, even when `memory.qmd.searchMode` is omitted and therefore defaults to lexical `search`.

## Best Possible Solution

Review PR #73512 as the canonical implementation candidate. The desired end state should either preserve the existing lexical `search` contract with clearer configuration guidance, or explicitly model `embedInterval` as a user opt-in to QMD embedding and update the scheduler, tests, and docs together.

## Reproduction Assessment

Yes. A high-confidence static and unit-level reproduction exists: configure QMD with `searchMode: "search"` and `embedInterval: "5m"`, advance the timer, and current main expects no `update` or `embed` subprocess calls.

## Solution Assessment

Unclear. The linked PR is a plausible narrow fix for the observable bug, but the best solution depends on whether maintainers want `embedInterval` to override the current lexical/BM25-only QMD contract.
