## What This Changes

The issue requests that QMD automatic embedding run according to `memory.qmd.update.embedInterval` even when `searchMode` is omitted and therefore defaults to lexical `search`.

## Best Possible Solution

Review PR #73512 as the canonical implementation candidate. The desired end state should either preserve the current lexical/BM25-only contract with clearer config guidance, or intentionally treat `embedInterval` as an embedding opt-in and update the scheduler, tests, and docs together.
