## Summary

Keep open. Current main still skips QMD embedding maintenance in default lexical `search` mode even when `memory.qmd.update.embedInterval` is configured, and open PR #73512 already references this issue with closing syntax, so the issue should stay open until that implementation is reviewed and merged or maintainers choose a docs/diagnostic-only path.

## What This Changes

The issue requests that OpenClaw automatically run QMD embedding on the configured `memory.qmd.update.interval` or `embedInterval` after memory document changes.

## Best Possible Solution

Review PR #73512 as the canonical implementation candidate. The preferred end state should either honor an explicit `memory.qmd.update.embedInterval` as an embedding opt-in while preserving no-surprise behavior for default BM25-only setups, or clearly warn/status-report that users must set `searchMode` to `query` or `vsearch` for automatic embeddings.
