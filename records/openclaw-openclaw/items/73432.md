## Summary

Keep open because current main still skips QMD embedding when `searchMode` is the default lexical `search` mode, even with `embedInterval` configured. Open PR #73512 already references this issue with closing syntax and targets the same scheduler guard, so the issue should wait for that PR's maintainer review rather than being closed or queued for a duplicate repair.

## What This Changes

The issue reports that QMD documents remain pending embedding because OpenClaw does not automatically run QMD embedding according to `memory.qmd.update.interval` or `memory.qmd.update.embedInterval`.

## Best Possible Solution

Review PR #73512 as the canonical implementation candidate, then land it with matching tests and docs if maintainers want `embedInterval` to override the lexical-mode embed guard; otherwise close or revise that PR and clarify the QMD configuration guidance.

## Reproduction Assessment

Yes. A high-confidence static and unit-level reproduction exists: configure QMD with default or explicit `searchMode: "search"` plus `embedInterval`, advance the manager timer, and current main expects no `update` or `embed` subprocess calls.

## Solution Assessment

Unclear. Treating a positive `embedInterval` as an explicit user opt-in is a narrow possible fix, but it changes the currently documented BM25-only `search` contract and should be decided in the linked PR review.
