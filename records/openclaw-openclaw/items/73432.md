## Summary

Keep open. Current main still skips QMD embedding before the interval checks in the default lexical mode, and an open closing PR already targets the same scheduler behavior, so maintainers should review that implementation path instead of closing or queueing duplicate work.

## What This Changes

The report asks OpenClaw to run QMD embedding automatically when `memory.qmd.update.embedInterval` is configured, even when `memory.qmd.searchMode` is omitted.

## Best Possible Solution

Review PR #73512 as the canonical implementation candidate. The desired end state should either treat an explicit `memory.qmd.update.embedInterval` as an embedding opt-in while preserving conservative defaults, or keep lexical-mode suppression and make the required `searchMode: "vsearch"` or `"query"` guidance visible where users see pending embeddings.

## Reproduction Assessment

Yes. Current main has a focused unit-level reproduction: configure QMD with `searchMode: "search"` and `embedInterval: "5m"`, advance the timer, and the existing test expects no `update` or `embed` calls.

## Solution Assessment

Unclear. PR #73512 is a plausible narrow fix, but the best solution depends on maintainer product judgment about whether explicit `embedInterval` should override the lexical-mode vector skip.
