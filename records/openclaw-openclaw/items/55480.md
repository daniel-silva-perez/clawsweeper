## Summary

Keep open. Current main still drops pure non-ASCII Discord channel names from the display path, so the PR targets a real bug, but the patch should not merge as-is because it changes Discord GroupSpace from stable guild IDs to mutable slugs and only fixes all-Unicode names, not mixed names covered by the PR description.

## What This Changes

The PR adds a Unicode-preserving fallback for Discord display channel slugs and changes Discord message/component GroupSpace construction to prefer guild slugs over guild IDs.

## Best Possible Solution

Implement a display-only Unicode-preserving Discord channel/thread label path on current main that covers pure and mixed non-ASCII names, while leaving normalizeDiscordSlug, configChannelSlug, allowlist matching, and ID-first GroupSpace policy routing unchanged.

## Reproduction Assessment

Yes. The current-main path is deterministic: displayChannelName is passed through the ASCII-only normalizeDiscordSlug, an all-Unicode channel name becomes empty, and GroupChannel is omitted when displayChannelSlug is empty.

## Solution Assessment

No. The display fallback idea is right, but this PR is not the best fix because it changes policy identity semantics and does not preserve mixed Unicode names; the safer fix is a display-only Unicode slug helper in the current preflight channel-context path.
