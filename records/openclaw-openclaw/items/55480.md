## Summary

Keep open: the underlying Discord display bug is still present on current main, but this PR is not safe to merge because it under-covers mixed Unicode names and changes guild GroupSpace identity in ways that can break ID-keyed policy routing.

## What This Changes

The PR adds an empty-normalized Discord displayChannelSlug fallback that preserves Unicode text and changes message/component GroupSpace construction to prefer guild slugs over guild IDs.

## Best Possible Solution

Implement a display-only Unicode-preserving Discord channel/thread label path on current main that covers pure and mixed non-ASCII names, while leaving normalizeDiscordSlug, configChannelSlug, allowlist matching, and GroupSpace ID-first policy routing unchanged.
