## Summary

Keep open. The display bug remains on current main because Discord display channel slugs still flow through the ASCII-only Discord slug normalizer, but this PR is not merge-safe as written: it changes Discord GroupSpace from guild ID to guild slug in two inbound paths, which can break documented ID-keyed guild policy lookup, and its display fallback only handles fully non-ASCII names.

## What This Changes

The PR adds a Unicode-preserving displayChannelSlug fallback when Discord channel-name normalization returns empty and changes message/component GroupSpace construction to prefer guild slugs over guild IDs.

## Best Possible Solution

Implement a display-only Unicode-preserving Discord channel label path on current main, with coverage for pure and mixed non-ASCII channel names, while leaving configChannelSlug, allowlist matching, and GroupSpace ID-first policy routing unchanged.
