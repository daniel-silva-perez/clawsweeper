## Summary

Keep open. Current main still lists only direct subagents from both the model tool and slash command path, while nested workers are only summarized as descendant or child-session metadata. The request is recent, corroborated by another user, and not covered by the merged nested-orchestration PR.

## What This Changes

Add recursive or depth-bounded `subagents list` support so a parent agent can inspect nested descendant agents and their hierarchy metadata.

## Best Possible Solution

Add a bounded traversal mode that preserves today’s direct-child default, supports finite `depth` and `recursive`, returns flat rows with `depth` and `parentSessionKey`, and documents the orchestrator workflow without expanding control permissions.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists by seeding a main-owned orchestrator run plus an orchestrator-owned worker run: the current list path returns only the orchestrator, with the worker visible only through descendant summary metadata.

## Solution Assessment

Yes. The requested depth/recursive list option is the narrowest maintainable direction if it reuses existing descendant traversal helpers and keeps non-list control actions scoped to directly controlled children.
