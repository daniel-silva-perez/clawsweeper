## Summary

Keep open. Current main still lacks recursive/depth-aware subagent listing: both the tool and slash-command paths list only direct children, while nested workers are only exposed as descendant counts or child-session summaries. The issue is valid, recent, and narrow enough for a focused repair PR.

## What This Changes

Request adding bounded recursive/depth support to `subagents list` and `/subagents list` so a parent session can inspect descendant subagents with hierarchy metadata.

## Best Possible Solution

Add a bounded, backward-compatible recursive list mode for both the tool and slash command: default to depth 1, accept finite `depth` plus `recursive`, return flat rows with `depth` and `parentSessionKey`, preserve direct-child control semantics, and document the orchestrator workflow.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: seed a main-owned orchestrator run plus an orchestrator-owned worker run, then current `listControlledSubagentRuns(main)` returns only the orchestrator while `buildSubagentList` reports the worker only through descendant summary metadata.

## Solution Assessment

Yes. The requested `depth`/`recursive` direction is the narrow maintainable solution as long as traversal is bounded, defaults remain unchanged, and control actions stay scoped to direct owned children.
