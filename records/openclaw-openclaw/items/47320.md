## Summary

Current main still does not implement recursive or depth-aware subagent listing. The `subagents` tool and `/subagents list` command only list direct controlled children; descendants are summarized as `pendingDescendants` / `childSessions`, not expanded into rows with depth or parent metadata. The request is valid, narrow, and suitable for a focused repair PR.

## Best Possible Solution

Keep this issue open and implement a bounded, backward-compatible recursive list path: default depth 1, accept `depth` and `recursive` on the `subagents` tool, parse `/subagents list --depth N` and `/subagents list --recursive`, return flattened rows with `depth` and `parentSessionKey`, and update tests/docs/changelog. Keep #67943 separate because it tracks stale lineage/depth-limit state, not recursive list visibility.
