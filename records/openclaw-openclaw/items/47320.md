## Summary

Current main still does not implement recursive/depth-aware subagent listing. The tool schema and slash command only expose direct-child listing, while descendants are only summarized through pending counts or child session metadata.

## Best Possible Solution

Keep this issue open and implement a bounded, backward-compatible recursive list path: default depth 1, accept `depth` and `recursive` on the `subagents` tool, parse `/subagents list --depth N` and `/subagents list --recursive`, return flattened rows with `depth` and `parentSessionKey`, and update docs/tests/changelog. Keep #67943 separate because it tracks stale lineage/depth-limit state rather than recursive visibility.
