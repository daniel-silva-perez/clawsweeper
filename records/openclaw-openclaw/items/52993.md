## Summary

Keep open. Current main still does not implement Feishu/Lark message recall or delete support, so this PR is not obsolete or implemented-on-main. It is also not ready to close as merged-equivalent: the PR adds a separate `feishu_message` tool while current main exposes Feishu through the channel message-action adapter, and the PR discussion records unresolved functional/runtime concerns that should be addressed before any landing decision.

## Best Possible Solution

Keep this PR open, but require a focused rework. The best path is to implement Feishu recall/delete through the current channel message-action adapter (`extensions/feishu/src/channel.ts` plus a runtime helper), advertise the supported action through discovery, route by the active `accountId`, add config/schema coverage for any destructive-action gate, use repo-supported JSON-schema/TypeBox tool surfaces, update focused Feishu tests/docs, and split unrelated localization changes out of the feature PR.
