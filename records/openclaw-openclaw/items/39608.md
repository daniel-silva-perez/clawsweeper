## Summary

Keep this issue open. Current main still creates the bundled WhatsApp Baileys socket with `syncFullHistory: false` and no explicit `shouldSyncHistoryMessage`, while Baileys 7.0.0-rc.9 defaults that omitted predicate from `syncFullHistory`. The issue has recent linked-device group-loss repro comments, OpenClaw docs still present WhatsApp group inbound as supported, and related open issues are adjacent rather than a clear canonical replacement.

## Best Possible Solution

Keep this issue open until the WhatsApp plugin intentionally fixes or documents linked-device group inbound behavior. The likely code path is to review Baileys 7 history-sync semantics, update `extensions/whatsapp/src/session.ts` to pass a deliberate `shouldSyncHistoryMessage` predicate or product-backed config option, add a regression assertion in `extensions/whatsapp/src/session.test.ts` for the `makeWASocket` options, and validate with a live linked-device WhatsApp group smoke. If maintainers decide linked-device group inbound cannot be supported reliably, document that limitation clearly in the public WhatsApp/group message docs instead.
