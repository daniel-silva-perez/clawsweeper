## Summary

Keep open. Current main still skips session hydration for Discord announce targets, so the final A2A announce can omit the target session account and route through the channel default instead. PR #73975 is already open as the focused closing implementation path, so this should stay in maintainer review rather than be closed or duplicated.

## What This Changes

The issue requests restoring Discord A2A announce routing so the announce send uses the target session's stored `deliveryContext.accountId` or `lastAccountId` instead of an omitted/default account.

## Best Possible Solution

Land PR #73975 or an equivalent narrow Discord-owned fix that opts Discord announce targets into session hydration and adds regression coverage proving `deliveryContext.accountId` or `lastAccountId` reaches the final gateway `send`.
