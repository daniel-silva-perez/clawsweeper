## Summary

Keep this issue open. Current main still has the Discord `sessions_send` A2A announce account-routing gap, and the open closing PR #73975 is the active implementation path rather than a reason to close the issue before merge.

## What This Changes

The issue requests preserving the target Discord session account identity during agent-to-agent announce delivery instead of falling back to the default account.

## Best Possible Solution

Ship #73975 or an equivalent narrow Discord-owned fix that opts Discord into announce-target session hydration and adds regression coverage proving `deliveryContext.accountId` or `lastAccountId` reaches the final gateway `send`.
