## Summary

Keep open. Current main still has the Discord announce-routing gap: parsed Discord announce targets return before session hydration, Discord does not opt into announce-target session lookup, and the linked open PR #73975 is the active closing implementation path rather than a reason to close this issue early.

## What This Changes

The issue asks for Discord agent-to-agent announce delivery to preserve the target session account identity instead of falling back to the default account.

## Best Possible Solution

Review and land PR #73975, or an equivalent narrow Discord-owned fix that opts Discord announce targets into session hydration and adds regression coverage proving deliveryContext.accountId or lastAccountId reaches the final gateway send.
