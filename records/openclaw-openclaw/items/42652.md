## Summary

Keep this issue open. Current main still has the Discord `sessions_send` announce account-routing gap: parsed Discord session targets return before session hydration, Discord has not opted into announce-target session lookup, and the final gateway send still forwards the missing `accountId` through the Discord outbound path.

## Best Possible Solution

Keep this issue open and fix the Discord-owned A2A announce routing path. The narrow product/code path is to opt Discord into `preferSessionLookupForAnnounceTarget` or merge missing `accountId` and thread fields from stored target-session metadata after parsing, then add a regression test where a Discord target session with `deliveryContext.accountId` or `lastAccountId` causes the final gateway `send` to receive that account id.
