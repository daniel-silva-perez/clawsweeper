## Summary

Current main still has the reported Discord announce-routing gap, and the issue should stay open because PR #73975 is an open, unmerged implementation path for the same bug.

## What This Changes

Discord A2A announce delivery should use the target session's stored Discord accountId instead of falling back to the default account.

## Best Possible Solution

Land PR #73975 or an equivalent narrow Discord-owned change that opts Discord announce targets into the existing session hydration contract and adds regression coverage proving the stored session account reaches the final gateway send.

## Reproduction Assessment

Yes. Current main has a unit-level reproduction path: Discord announce target resolution returns the parsed fallback without calling `sessions.list`, and the final send path forwards that missing accountId.

## Solution Assessment

Yes. The narrowest maintainable fix is to opt Discord into the existing `preferSessionLookupForAnnounceTarget` hydration path and cover the accountId regression, rather than changing the default behavior for every channel.
