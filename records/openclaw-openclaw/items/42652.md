## Summary

Keep open: current main still has the Discord announce account-routing gap, and PR #73975 is an open, unmerged implementation path that explicitly targets it. The closed related report #51626 supports the same failure mode but does not make this implemented.

## What This Changes

The issue asks for Discord A2A announce delivery to preserve the target session's account identity instead of falling back to the default Discord account.

## Best Possible Solution

Land PR #73975 or an equivalent narrow Discord-owned change that opts Discord announce targets into the existing session hydration contract and proves both `deliveryContext.accountId` and `lastAccountId` reach the final gateway send.

## Reproduction Assessment

Yes. Current main has a source-level reproduction path: a Discord session key resolves to a fallback target without `sessions.list`, and the final announce send forwards the missing accountId.

## Solution Assessment

Yes. The narrowest maintainable fix is a Discord-specific metadata opt-in plus regression coverage, rather than changing announce lookup defaults for every channel.
