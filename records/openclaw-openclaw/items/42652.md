## Summary

Keep open. Current main still skips session hydration for Discord announce targets, so the final A2A announce can omit the target session account and route through the configured/default Discord account. PR #73975 is already open with closing syntax for this exact bug, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw to preserve the target Discord session's accountId when the A2A sessions_send announce step posts back, instead of falling back to the configured/default Discord account.

## Best Possible Solution

Review and land PR #73975, or an equivalent narrow Discord-owned change that opts Discord announce targets into session hydration and adds regression coverage proving deliveryContext.accountId or lastAccountId reaches the final gateway send.
