## Summary

Keep this PR open. Current main still has the exact heartbeat dedupe load/mutate/save pattern that the PR targets, while `updateSessionStore` remains the lock-safe session-store API that re-reads under the write lock. The PR is a focused one-file bug fix and is not implemented, merged, or clearly superseded on main.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase if needed, land the `updateSessionStore` change for the `lastHeartbeatText` and `lastHeartbeatSentAt` dedupe write, and preferably add a focused regression test proving a concurrent session-store update is preserved.
