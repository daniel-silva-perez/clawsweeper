## Summary

Current main still has the heartbeat dedupe race this PR targets. The delivered heartbeat payload is still recorded through a load/mutate/save of a caller-provided session-store snapshot, while the session-store API documents updateSessionStore as the lock-safe path that re-reads inside the lock. The PR remains a focused one-file runtime fix and is not implemented, obsolete, or superseded on main.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase if needed, land the updateSessionStore change for the lastHeartbeatText/lastHeartbeatSentAt write, and ideally add a targeted regression test proving a concurrent session-store update is preserved.
