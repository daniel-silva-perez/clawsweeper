## Summary

Keep this PR open. The reported heartbeat dedupe race still exists on current main, and the PR’s core direction is the right narrow fix, but its import deletion is stale against current main and would break remaining `saveSessionStore` call sites after rebase or merge.

## What This Changes

The PR changes `src/infra/heartbeat-runner.ts` to record heartbeat dedupe metadata through `updateSessionStore` instead of a raw load/mutate/save sequence, and removes the `saveSessionStore` import from its original base.

## Best Possible Solution

Land a rebased, narrow heartbeat-runner fix that switches only the delivered heartbeat dedupe metadata write to `updateSessionStore`, preserves the remaining `saveSessionStore` import and call sites, and adds a focused regression test if maintainers want coverage for this race.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a concurrent writer can update the session store after heartbeat loads it and before heartbeat saves, causing the heartbeat to overwrite that update with the stale snapshot.

## Solution Assessment

No, not exactly as currently proposed against main. Replacing the dedupe write with `updateSessionStore` is the narrow maintainable fix, but the PR should keep `saveSessionStore` imported because current main still uses it elsewhere in the file.
