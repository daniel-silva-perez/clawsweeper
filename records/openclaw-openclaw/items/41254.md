## Summary

Keep open. Current main still has the heartbeat dedupe race the PR targets: `runHeartbeatOnce` records delivered heartbeat metadata by saving a caller-held session-store snapshot, while the lock-safe API re-reads inside the lock. The PR is directionally correct, but the branch is currently dirty against main and needs maintainer rebase/review before merge.

## What This Changes

The PR changes `src/infra/heartbeat-runner.ts` so delivered heartbeat dedupe metadata is written through `updateSessionStore` instead of a manual `loadSessionStore`/`saveSessionStore` snapshot, and removes the then-unused import on its original base.

## Best Possible Solution

Land a rebased, narrow heartbeat-runner fix that switches only the delivered heartbeat dedupe write to `updateSessionStore`, preserves unrelated current-main session-store writes/imports, and adds a small regression test for concurrent session-store writes during heartbeat delivery.
