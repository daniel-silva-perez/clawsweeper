## Summary

Keep open. The heartbeat dedupe race still exists on current main and in the latest release, so this PR is not obsolete; the proposed direction is right, but the branch is stale because current main still needs `saveSessionStore` elsewhere in the same file.

## What This Changes

The PR changes `src/infra/heartbeat-runner.ts` to record delivered heartbeat dedupe metadata through `updateSessionStore` instead of a raw `loadSessionStore`/`saveSessionStore` snapshot, and removes the `saveSessionStore` import on its original base.

## Best Possible Solution

Land a rebased, narrow heartbeat-runner fix that switches only the delivered heartbeat dedupe recording to `updateSessionStore`, preserves the remaining `saveSessionStore` import and calls, and adds a focused regression test for concurrent session-store writes during heartbeat delivery.
