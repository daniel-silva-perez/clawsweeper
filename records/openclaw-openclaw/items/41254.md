## What This Changes

The PR changes `src/infra/heartbeat-runner.ts` so delivered heartbeat dedupe metadata is recorded through `updateSessionStore` instead of a caller-held `loadSessionStore`/`saveSessionStore` snapshot.

## Best Possible Solution

Land a rebased, narrow heartbeat-runner fix that preserves current-main imports and unrelated `saveSessionStore` uses, switches only the delivered heartbeat dedupe write to `updateSessionStore`, and adds a small regression test for the concurrent-writer case.
