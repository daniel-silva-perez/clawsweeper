## Summary

Keep open. Current main still writes delivered heartbeat dedupe metadata through a stale `loadSessionStore`/`saveSessionStore` snapshot, while `updateSessionStore` is the lock-safe API for this race. The PR is focused and appears correct, but it needs maintainer rebase/review because current main has moved and added other `saveSessionStore` usages in the same file.

## What This Changes

The PR changes `runHeartbeatOnce` so delivered heartbeat dedupe metadata is written through `updateSessionStore` instead of a manual `loadSessionStore`/`saveSessionStore` snapshot, and removes the then-unused import on its original base.

## Best Possible Solution

Land a rebased, narrow heartbeat-runner fix that switches only the delivered heartbeat dedupe recording to `updateSessionStore`, preserves unrelated session-store writes, and adds a focused regression test for concurrent session-store writes during heartbeat delivery.
