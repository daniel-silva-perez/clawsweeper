## Summary

Keep open. Current main still has the heartbeat dedupe stale-snapshot write this PR targets, and the PR uses the existing lock-safe session-store update API rather than inventing a new path. It is still useful, but likely needs a rebase and ideally a focused concurrency regression test before landing.

## What This Changes

The PR changes `src/infra/heartbeat-runner.ts` so the delivered heartbeat dedupe metadata write uses `updateSessionStore` instead of a raw `loadSessionStore`/`saveSessionStore` snapshot write.

## Best Possible Solution

Land this as a rebased, narrow heartbeat-runner fix that preserves current-main imports, switches only the delivered heartbeat dedupe write to `updateSessionStore`, and adds a small regression test proving a concurrent session-store update is not clobbered.
