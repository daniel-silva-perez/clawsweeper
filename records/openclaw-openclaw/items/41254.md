## Summary

Keep open. The reported race is still present on current main, and the PR’s core change uses the right session-store API for the affected heartbeat dedupe write. The branch is not merge-ready as-is because its original import-removal hunk is now stale against current main, where `saveSessionStore` is still used by other heartbeat-runner paths.

## What This Changes

The PR changes `runHeartbeatOnce` to record delivered heartbeat dedupe metadata through `updateSessionStore` instead of a raw `loadSessionStore`/`saveSessionStore` snapshot, and removes the `saveSessionStore` import on its original base.

## Best Possible Solution

Land a rebased, narrow heartbeat-runner fix that switches only the delivered heartbeat dedupe recording to `updateSessionStore`, keeps unrelated session-store writes intact, and adds a focused regression test for concurrent session-store writes during heartbeat delivery.
