## Summary

Keep this PR open. The reported heartbeat dedupe race is still present on current main, and the PR’s core `updateSessionStore` direction matches the session-store locking contract, but the patch is stale against current main because it removes an import that new current-main code still needs.

## What This Changes

The PR changes `src/infra/heartbeat-runner.ts` to record delivered heartbeat dedupe metadata through `updateSessionStore` instead of a raw session-store snapshot save, and removes the `saveSessionStore` import from its original base.

## Best Possible Solution

Land a rebased, narrow heartbeat-runner fix that switches only the delivered heartbeat dedupe metadata write to `updateSessionStore`, preserves remaining `saveSessionStore` usage, and adds focused regression coverage for concurrent session-store updates if maintainers want a guard for this race.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a concurrent writer can update the session store after the heartbeat loads it and before the heartbeat saves, allowing the heartbeat to overwrite that update with the stale snapshot.

## Solution Assessment

No, not exactly as currently proposed against current main. Switching the dedupe write to `updateSessionStore` is the right maintainable fix, but deleting `saveSessionStore` from the import is stale because current main still calls it elsewhere in the file.
