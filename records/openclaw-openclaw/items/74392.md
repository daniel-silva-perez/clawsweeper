## Summary

Keep this PR open. Current main still has the manifest-less library runtime-deps gap in packaged scans and doctor repair, but this is an active contributor implementation with unresolved maintainer policy questions around plugin gate bypasses and it now needs rebasing/porting onto the split runtime-deps selection files.

## What This Changes

The PR adds package.json `openclaw.bundle.stageRuntimeDependencies` handling for manifest-less bundled library extensions in bundled runtime-deps scanning and doctor repair, adds regression tests, and documents library extensions as plugin-gate-bypassing infrastructure.

## Best Possible Solution

Land this PR or an equivalent narrow port after maintainer policy review: package.json-only bundled library packages that explicitly opt into staged runtime dependencies should be discoverable by packaged scans and doctor repair, with the chosen allow/deny/global-disable semantics documented and covered by tests while preserving the existing lazy public-artifact runtime preparation path.
