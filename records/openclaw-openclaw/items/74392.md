## Summary

Keep this PR open. Current main still skips manifest-less bundled library packages in the packaged runtime-deps scan and doctor repair selection, while this PR is a focused implementation for that remaining gap. I did not find a definite blocking correctness issue in the latest provided diff; the main remaining work is maintainer review of the library-extension gate policy and normal validation.

## What This Changes

The PR adds package.json `openclaw.bundle.stageRuntimeDependencies` handling for manifest-less bundled library packages in bundled runtime-deps scan and `openclaw doctor --fix`, with regression tests for scan, repair, and gate-bypass cases.

## Best Possible Solution

Land this PR or an equivalent narrow change that includes package.json-only bundled library packages with `openclaw.bundle.stageRuntimeDependencies: true` in packaged startup scans and doctor repair, with tests and docs aligned to the chosen gate semantics while keeping the existing lazy public-artifact runtime preparation intact.
