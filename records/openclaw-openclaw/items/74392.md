## Summary

Keep open. Current main still misses the manifest-less library-extension scan and doctor repair path, but the latest PR intentionally bypasses plugin disable/deny gates and is dirty against current main after the runtime-deps split; maintainer review is needed before merge.

## What This Changes

The PR adds package.json `openclaw.bundle.stageRuntimeDependencies` handling for manifest-less bundled library extensions in bundled runtime-deps scanning and doctor repair, plus docs and regression tests.

## Best Possible Solution

Port or rework the change on current main so package.json-only bundled library packages that opt into staged runtime dependencies are included in the intended packaged scan, doctor repair, and/or lazy public-artifact paths, with maintainer-approved gate semantics reflected consistently in code, docs, and tests.
