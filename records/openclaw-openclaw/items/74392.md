## Summary

Keep this PR open. Current main still misses the manifest-less library-extension case in the packaged startup scan and doctor repair paths, while the PR addresses a concrete runtime-deps bug. The remaining decision is maintainer review of the library-extension gate bypass and whether to fold this narrow fix into the broader open runtime-deps staging work in #74246.

## What This Changes

The PR adds package.json `openclaw.bundle.stageRuntimeDependencies` handling for manifest-less bundled library extension directories in bundled runtime-deps scanning and doctor repair, with regression tests for startup scan and doctor repair.

## Best Possible Solution

Review this as a narrow runtime-deps repair or fold it into #74246: manifest-less bundled library packages that explicitly opt into staged runtime dependencies should be discoverable by packaged startup scans and doctor repair, with tests locking the chosen allow/deny/global-disable semantics and the existing on-demand public-artifact preparation left intact.
