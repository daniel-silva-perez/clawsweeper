## What This Changes

The PR adds package.json `openclaw.bundle.stageRuntimeDependencies` detection for manifest-less bundled library extensions in bundled runtime-deps scan/doctor coverage, with docs and regression tests for the library-extension gate exception.

## Best Possible Solution

Land this PR or an equivalent narrow change after maintainer policy approval: manifest-less bundled library packages that explicitly opt into staged runtime dependencies should be discoverable by packaged scan and doctor repair, with docs/tests matching the chosen gate semantics and the existing on-demand public-artifact path left intact.

## Reproduction Assessment

Yes. A packaged-root fixture with `dist/extensions/media-understanding-core/package.json`, no `openclaw.plugin.json`, `stageRuntimeDependencies: true`, and `config: {}` exercises the gap because current main's configured scanner reads only the manifest and returns false for the package-only signal.

## Solution Assessment

Unclear until the maintainer policy decision is made. The package.json fallback in the shared scanner is a narrow technical fix, but bypassing `plugins.enabled`, `plugins.deny`, `plugins.allow`, and per-entry plugin gates is a product/security contract change that should be approved explicitly.
