## Summary

Keep this PR open. The reported runtime-deps gap is real on current main for packaged scans and doctor repair, but the PR is still an active, unmerged contributor implementation with maintainer-raised policy and startup-coverage questions rather than cleanup-close material.

## What This Changes

The PR adds package.json `openclaw.bundle.stageRuntimeDependencies` handling for manifest-less bundled library extensions in bundled runtime-deps scanning and doctor repair, adds regression tests, and documents library extensions as plugin-gate-bypassing infrastructure.

## Best Possible Solution

Land this PR or an equivalent narrow port after maintainer policy review: manifest-less bundled library packages that explicitly opt into staged runtime dependencies should be handled in the packaged scan and doctor repair paths, with the chosen disable/deny/allow semantics documented and covered by tests, and any startup-prestage claim backed by a `prepareGatewayPluginBootstrap` regression or removed.

## Review Findings

Overall correctness: patch is incorrect

  - body: The added docs promise that library extension deps are always installed by startup scans, but gateway prestaging calls `scanBundledPluginRuntimeDeps` with `selectedPluginIds` from the manifest lookup table and skips the scan when that set is empty.
