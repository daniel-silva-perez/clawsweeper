## Summary

Keep this PR open. The runtime-deps gap is real and current main still lacks the package.json-only scan/doctor fallback, but the latest branch needs maintainer resolution before merge because it bypasses explicit plugin disable/deny policy and does not yet prove the claimed startup prestage path.

## What This Changes

The PR adds `openclaw.bundle.stageRuntimeDependencies` handling for manifest-less bundled library packages in bundled runtime-deps scans and doctor repair, with regression tests including plugin-gate bypass cases.

## Best Possible Solution

Land this PR or an equivalent narrow fix after choosing and documenting the intended policy: manifest-less bundled library packages that opt into staged runtime dependencies should be included in the appropriate scan and doctor paths, with bootstrap coverage if startup prestage remains a claimed behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Honor explicit plugin disablement for runtime deps:** `src/plugins/bundled-runtime-deps.ts:969-981`
  - body: The new early return runs before `passesRuntimeDepsPluginPolicy`, so a manifest-less package with `stageRuntimeDependencies` is included even when `plugins.enabled: false` or `plugins.deny` would currently block automatic bundled runtime-deps repair. That contradicts the documented contract and the existing startup behavior for globally disabled plugins.
  - confidence: 0.9
- **[P2] Keep doctor repair behind the policy gate:** `src/commands/doctor-bundled-plugin-runtime-deps.ts:48-49`
  - body: This fallback adds the library directory to the doctor repair set unconditionally. As a result, `openclaw doctor --fix` will install `sharp` even when the operator disabled plugins globally or denied this package, while the manifest-backed path below still respects the policy gate.
  - confidence: 0.88
- **[P2] Prove startup selection before relying on selected ids:** `src/plugins/bundled-runtime-deps.ts:1093-1101`
  - body: This branch only helps if `selectedPluginIds` already contains `media-understanding-core`, but gateway startup builds that list from the manifest/index startup plan and this package has no `openclaw.plugin.json`. Without adding it to the bootstrap selection path or testing `prepareGatewayPluginBootstrap`, startup prestage still will not cover the library package.
  - confidence: 0.78
