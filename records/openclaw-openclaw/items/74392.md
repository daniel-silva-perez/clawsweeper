## Summary

Keep this PR open. Current main still lacks package.json `stageRuntimeDependencies` handling for manifest-less bundled library extension directories in both the packaged runtime-deps scan and doctor repair selection, while the PR is active implementation work for a valid runtime-deps gap. The remaining decision is maintainer review of the library-extension gate policy and overlap with #74246, not cleanup closure.

## What This Changes

The PR adds package.json `openclaw.bundle.stageRuntimeDependencies` handling for manifest-less bundled library extension directories in bundled runtime-deps scan and doctor repair paths, with regression tests for scan, repair, and gate-bypass semantics.

## Best Possible Solution

Review this as a narrow runtime-deps repair or fold it into #74246: manifest-less bundled library packages that explicitly opt into staged runtime dependencies should be discoverable by packaged startup scans and doctor repair, with the chosen allow/deny/global-disable semantics documented and covered by tests alongside the existing on-demand public-artifact preparation.
