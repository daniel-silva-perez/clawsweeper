## Summary

Keep this PR open. Current main still does not include manifest-less bundled library packages in the packaged runtime-deps scan or doctor repair selection, and the PR is a narrow implementation for a real `sharp`/image-runtime problem. It needs maintainer review because the final policy for manifest-less library extensions overlaps with the broader open runtime-deps staging work in #74246 and affects runtime dependency install eligibility.

## What This Changes

The PR adds package.json `openclaw.bundle.stageRuntimeDependencies` fallback handling for manifest-less bundled library packages in bundled runtime-deps scanning and doctor repair, plus regression tests.

## Best Possible Solution

Review this as a narrow runtime-deps repair or fold it into #74246: manifest-less bundled library packages that intentionally opt into staged runtime dependencies should be discoverable by packaged startup scans and doctor repair, while the existing on-demand public-artifact preparation remains intact and allow/deny/global-disable behavior is covered by tests.
