## Summary

Keep this PR open. Current main has overlapping on-demand runtime-deps preparation from the merged #74246 work, but it still does not implement the PR's scan and doctor repair fallback for manifest-less library extension packages. The remaining decision is maintainer policy: whether these library packages should bypass plugin allow/deny/global-disable gates during proactive startup scan and doctor repair.

## What This Changes

The PR adds package.json `openclaw.bundle.stageRuntimeDependencies` handling for manifest-less bundled library extension directories in bundled runtime-deps scan and doctor repair paths, with regression tests for scan, repair, and gate-bypass behavior.

## Best Possible Solution

Review this as a narrow follow-up to #74246 or fold it into that design: manifest-less bundled library packages that intentionally opt into staged runtime dependencies should either be included by packaged scan and doctor repair with documented gate semantics, or the project should explicitly rely on the on-demand public-artifact preparation path and close the proactive scan/doctor behavior as out of scope.
