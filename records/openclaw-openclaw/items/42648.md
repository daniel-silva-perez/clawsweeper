## Summary

Keep open. Current main has memory indexing/search, append-only pre-compaction flushes, thresholded short-term promotion, LanceDB duplicate checks, and memory-wiki provenance/contradiction reporting, but it still does not implement the requested resolver-backed durable memory write pipeline with typed candidate classification, required source refs, create/merge/update/conflict outcomes, and non-destructive revision history.

## Best Possible Solution

Keep this issue open. Implement a resolver-backed durable write pipeline in memory-core or the active memory contract after aligning with #42646 and #42647: typed candidate records, required source references, entity normalization, duplicate and contradiction lookup, explicit create/merge/update/conflict-for-review outcomes, and non-destructive revision/provenance history. Use #49108 as classifier/entity-extraction prior art and coordinate operator flows with #42650.
