## Summary

Keep open. Current main has related memory search, short-term promotion, Memory Wiki, and optional LanceDB storage, but the default durable write path still does not provide the requested conservative resolver with typed create/merge/update/conflict outcomes, mandatory source attachment, entity normalization, and append-only revision provenance.

## What This Changes

Build a conservative memory ingestion write pipeline that classifies candidates, attaches sources, dedupes or merges before durable write, and routes conflicts for review with revision history.

## Best Possible Solution

Design the durable memory write resolver as a coordinated memory contract: typed canonical records, required source references, normalized subjects/entities, near-duplicate and contradiction lookup, explicit create/merge/update/conflict-for-review outcomes, and append-only revision/provenance history, with Markdown, Memory Wiki, LanceDB, and indexing layers feeding or reflecting that contract rather than bypassing it.

## Reproduction Assessment

Not applicable as a feature architecture request. The high-confidence verification path is source and docs inspection: current main documents and implements Markdown/index/wiki/LanceDB-adjacent paths, but no canonical durable write resolver or storage model.

## Solution Assessment

Unclear for a concrete patch because no implementation is under review. The requested direction is valid, but the best solution needs maintainer design across schema, plugin/API boundaries, provenance, and review flows rather than a one-off change to promotion, wiki metadata, or LanceDB duplicate checks.
