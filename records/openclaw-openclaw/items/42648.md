## Summary

Keep open. Current main still has file/index/search memory, append-only compaction flushes, thresholded short-term promotion, memory-wiki provenance surfaces, and optional LanceDB duplicate checks, but it does not yet provide the requested resolver-backed durable write pipeline with required source attachment, typed canonical records, semantic dedupe/conflict lookup, create/merge/update/conflict outcomes, or revision history.

## What This Changes

Build a Memory MVP durable write pipeline that classifies candidate memories, attaches source/provenance, dedupes and conflict-checks before durable write, and records merge/update/conflict outcomes with revision history.

## Best Possible Solution

Keep this open and design the write path as the durable-memory resolver layer after aligning the canonical schema, provenance, and review/conflict model: typed candidates, required source refs, entity normalization, duplicate and contradiction lookup, explicit create/merge/update/conflict-for-review outcomes, and non-destructive revision history.
