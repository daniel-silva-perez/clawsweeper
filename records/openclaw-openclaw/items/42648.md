## Summary

Keep open. Current main still has memory search, append-style daily-note flushes, thresholded promotion into `MEMORY.md`, a provenance-rich `memory-wiki` companion, and optional LanceDB duplicate checks, but it does not provide the requested resolver-backed durable write pipeline with required source attachment, canonical records, merge/update/conflict outcomes, or revision history.

## What This Changes

Build a conservative candidate-memory write path that classifies candidates, attaches sources, dedupes or merges near duplicates, surfaces conflicts, and preserves revision history.

## Best Possible Solution

Design the memory write path as a first-class durable resolver layer coordinated with #42646, #42647, and #42650: typed candidates, required source refs, canonical SQLite records, entity normalization, duplicate and contradiction lookup, explicit create/merge/update/conflict-for-review outcomes, and non-destructive revision history.
