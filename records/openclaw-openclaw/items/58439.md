## Summary

Keep this PR open. Current main still drops `mediaLocalRoots` in the Mattermost `handleAction` send path even though core supplies the trusted roots and the Mattermost sender already honors them; the PR is a narrow, owner-local pass-through with a focused regression test. The related #67796 work is adjacent filePath/path fallback support, not a replacement for this media-root fix.

## What This Changes

The PR forwards `mediaLocalRoots` from the Mattermost message-action context into `sendMessageMattermost` and adds a Mattermost channel unit test for media sends through `handleAction`.

## Best Possible Solution

Land the focused Mattermost plugin pass-through and regression test after targeted validation, then keep the separate `filePath`/`path` fallback work in #67796 separate or rebase the two together if they conflict.

## Reproduction Assessment

Yes. The high-confidence reproduction path is a Mattermost shared message-tool send with `params.media` pointing at an agent `workspace-*` file: core computes and passes scoped roots, current main drops them in `handleAction`, and local media access rejects the workspace path when roots are undefined.

## Solution Assessment

Yes. Passing the server-provided `mediaLocalRoots` through the Mattermost action adapter is the narrowest maintainable fix because the sender already accepts the option and the existing Mattermost `sendMedia` path already forwards it.
