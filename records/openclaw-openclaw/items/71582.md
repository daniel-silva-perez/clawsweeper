## Summary

Keep this PR open. Current main still does not implement the PR's central exec-event heartbeat route-clearing behavior: the agent and node exec paths still enqueue old `scopedHeartbeatWakeOptions` wakes, the wake request type can only carry `heartbeat.target`, coalescing still preserves whole heartbeat objects rather than explicit route clears, cron still pre-merges heartbeat config before `runHeartbeatOnce`, and the shared `scopedExecEventWakeOptions` helper is absent. The closed duplicate #73851 is the same unmerged implementation candidate, not a superseding main-branch fix.

## Best Possible Solution

Keep this PR open as the active implementation candidate. Review the latest head, preserve the shared `scopedExecEventWakeOptions` direction if it matches maintainer intent, validate pinned route clears, `global`, synthetic node fallback, raw forced-session cron behavior, busy targeted wake retry/coalescing, and Telegram thread retry behavior, then merge or request targeted changes. The closed duplicate #73851 should remain closed and should not replace this PR.
