## Summary

Keep this issue open. Current main still does not visibly show the active config file path in Control UI Overview or prominently in Settings/Config; the backend and protocol already expose the path for admin-scoped sessions, but the UI only passes a config path into the Config view and uses it as an Open button tooltip.

## What This Changes

Request to show the active OpenClaw config file path prominently in Control UI Overview or Settings so operators can tell which instance/config is running.

## Best Possible Solution

Add a visible, copyable active config path to the Control UI Overview snapshot card and/or Settings/Config header, sourced from the existing admin-scoped hello snapshot configPath or config.get path, without widening backend exposure to low-privilege sessions.
