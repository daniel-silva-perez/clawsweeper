## Summary

Keep this issue open. Current main still does not visibly show the active config file path in Control UI Overview or prominently in Settings/Config; the backend and protocol already expose an admin-scoped path, but the UI only passes it to Config and currently uses it as an Open button tooltip.

## What This Changes

The issue requests a prominent visible active config file path in Control UI Overview or Settings to help users troubleshoot which OpenClaw instance is running.

## Best Possible Solution

Add a visible, copyable active config path to the Control UI Overview snapshot card and/or Settings/Config header, sourced from the existing admin-scoped hello snapshot configPath or config.get path without broadening backend exposure.
