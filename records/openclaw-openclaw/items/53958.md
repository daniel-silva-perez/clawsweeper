## Summary

Keep this issue open. Current `main` still does not visibly show the active config file path in Control UI Overview or prominently in Settings/Config. The backend already exposes the path for admin-scoped snapshots and `config.get` carries a path through to Config props, but the Overview view does not consume it and the Config view only uses it as an Open-button tooltip. The related implementation PR #54025 is closed unmerged in the provided GitHub context, so there is no shipped fix to close against.

## Best Possible Solution

Add a visible, copyable active config path to the Control UI Overview snapshot card and/or the Settings/Config header. Source it from the existing admin-scoped hello snapshot `configPath` or the existing `config.get` snapshot `path`, and add focused UI coverage plus privilege-boundary coverage so low-privilege sessions do not leak absolute paths.
