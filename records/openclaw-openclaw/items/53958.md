## Summary

Keep this issue open. Current main still does not visibly show the active config file path in Control UI Overview or prominently in Settings/Config. The backend already exposes an admin-scoped config path and `config.get` carries a path, but Overview does not consume it and Settings/Config only uses it as an Open-button tooltip. Related PR #54025 was closed unmerged, so there is no shipped fix to close against.

## Best Possible Solution

Add a visible, copyable active config path to the Control UI Overview snapshot card and/or the Settings/Config header. Source it from the existing admin-scoped hello snapshot `configPath` or the existing `config.get` snapshot `path`, and add focused UI coverage proving authorized sessions can see it while low-privilege snapshots do not leak path fields.
