## Summary

Keep this open. Current main still has the overflow diagnostic, bounded auto-compaction/retry behavior, memory promotion preview/apply primitives, and session archive/delete support, but it does not connect repeated context-overflow pressure to a deduped promote-then-rotate suggestion, precomputed candidate count, or operator confirmation flow.

## What This Changes

Request to surface a one-time promote-then-rotate lifecycle suggestion, including dry-run memory promotion candidates, when the same session repeatedly trips context overflow.

## Best Possible Solution

Approve a small lifecycle design first: persist per-session overflow pressure, emit a deduped structured suggestion with a stable id and dry-run candidate count, expose it through operator-visible CLI/UI surfaces, and require explicit confirmation before promotion or session rotation.

## Reproduction Assessment

Yes. The existing overflow loop test gives a high-confidence simulation path for repeated overflow, and current code inspection shows those attempts compact/retry or return `context_overflow` without a promote-then-rotate suggestion.

## Solution Assessment

Unclear pending maintainer design approval. The issue's opt-in suggestion and confirmation model is safer than automatic promotion/rotation, but the exact API, state storage, and UI surfaces should be settled before implementation.
