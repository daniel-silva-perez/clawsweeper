## Summary

Keep open. The prior ClawSweeper conclusion still matches current main: OpenClaw has overflow diagnostics, bounded auto-compaction, memory promotion preview/apply primitives, and session archive/delete support, but it still does not wire repeated context overflows into a deduped promote-then-rotate lifecycle suggestion or operator confirmation flow.

## What This Changes

This issue requests a lifecycle suggestion that detects repeated session context overflows, precomputes memory promotion candidates, and offers operator-confirmed promote/rotate actions through logs, CLI, and UI surfaces.

## Best Possible Solution

Approve a small lifecycle design first: persist per-session overflow pressure, emit a deduped structured suggestion with a stable id and dry-run candidate count, expose it in operator-visible CLI/UI surfaces, and require explicit confirmation before either promotion or session rotation.

## Reproduction Assessment

Yes. The current overflow loop test provides a high-confidence simulated reproduction path: repeated overflow attempts compact three times and then return `context_overflow`, while code/search inspection shows no lifecycle suggestion is emitted.

## Solution Assessment

Unclear pending maintainer design approval. The issue's opt-in suggestion and confirmation model is safer than automatic promotion/rotation, but the state storage, APIs, config, and user-facing surfaces should be settled before a fix PR.
