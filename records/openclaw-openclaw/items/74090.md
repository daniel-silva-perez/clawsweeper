## Summary

Keep this open. Current main has the overflow diagnostic and bounded auto-compaction path, memory promotion preview/apply primitives, rem-harness preview plumbing, and session archive/delete support, but it still does not connect repeated context overflows to a deduped promote-then-rotate lifecycle suggestion or an operator confirmation flow.

## What This Changes

Request to surface a one-time promote-then-rotate lifecycle suggestion, including dry-run memory promotion candidates, when the same session repeatedly trips context overflow.

## Best Possible Solution

Approve a small lifecycle design first: persist per-session overflow pressure, emit a deduped structured suggestion with a stable id and dry-run candidate count, expose it through operator-visible CLI/UI surfaces, and require explicit confirmation before either promotion or session rotation.

## Reproduction Assessment

Yes. The current overflow tests already provide a high-confidence simulation path for repeated overflow, and the code inspection shows those attempts compact/retry or return `context_overflow` without a promote-then-rotate suggestion.

## Solution Assessment

Unclear pending maintainer design approval. The issue's opt-in suggestion and confirmation model is safer than automatic promotion/rotation, but the exact API, state storage, and UI surfaces should be settled before implementation.
