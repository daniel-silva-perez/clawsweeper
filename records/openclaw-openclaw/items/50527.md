## Summary

Keep open. Current main still lacks the proposed per-field metadata length limits, and the PR remains useful security hardening, but as written it only validates direct `/hooks/agent` payloads while mapped hook agent dispatch can still carry oversized metadata fields.

## Best Possible Solution

Keep this PR open and revise it into a shared hook-agent metadata validator used by both direct `/hooks/agent` normalization and mapped `/hooks/<name>` agent dispatch. Add targeted tests for direct and mapped over-limit rejection, exact-boundary acceptance including `agentId` and `sessionKey`, and unchanged behavior for the overall 256 KB body limit.
