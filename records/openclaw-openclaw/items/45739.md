## Summary

Keep open. Current main still does not implement this PR's intended ACP parent relay fallback: the relay is still driven by local assistant/lifecycle events, while the newer task-completion path is separate terminal delivery rather than live recovery of missing child output.

## Best Possible Solution

Keep the PR open for maintainer review or a current-main rework. If live streamTo="parent" backfill is still desired, implement it in src/agents/acp-spawn-parent-stream.ts with scoped short-timeout chat.history and agent.wait calls, and add regression coverage for missing assistant events, missing terminal events, lifecycle-before-transcript persistence, whitespace-only local assistant events, history-first/local-second duplication, and the primeHistoryBaseline race. If maintainers decide task-completion delivery supersedes live relay backfill, close this PR manually as a product decision rather than automated implemented-on-main cleanup.
