## Summary

Keep this PR open. Current main still does not implement the PR's intended ACP parent relay fallback: `startAcpSpawnParentStreamRelay` remains driven by local `onAgentEvent` assistant/lifecycle events and has no `chat.history` or `agent.wait` backfill. The newer background task completion path is related, but docs and code show it is a separate terminal-delivery mechanism, not live relay recovery of missing child assistant output.

## Best Possible Solution

Keep the PR open for maintainer review or a current-main rework. If live `streamTo: "parent"` backfill is still desired, implement it in `src/agents/acp-spawn-parent-stream.ts` with scoped short-timeout `chat.history` and `agent.wait` calls, then add regression tests for missing assistant events, missing terminal events, lifecycle-before-transcript persistence, whitespace-only local assistant events, history-first/local-second duplication, and the `primeHistoryBaseline` race. If maintainers decide task-completion delivery supersedes live relay backfill, close it manually as a product decision rather than automated implemented-on-main cleanup.
