## Summary

Keep open. Current main still does not implement this PR's intended ACP parent relay fallback from gateway state: `streamTo: "parent"` remains driven by local `onAgentEvent` assistant/lifecycle events, while the task-completion path is separate terminal notification and does not recover missing live child output from `chat.history`/`agent.wait`.

## Best Possible Solution

Keep this PR open for maintainer review or a current-main rework. If live `streamTo: "parent"` backfill is still desired, implement it in `src/agents/acp-spawn-parent-stream.ts` with scoped short-timeout gateway history/wait reads and add regression coverage for missing assistant events, missing terminal events, lifecycle-before-transcript persistence, whitespace-only local assistant events, history-first/local-second duplication, and the `primeHistoryBaseline` race. If maintainers decide the task-completion path supersedes live relay backfill, close the PR manually as a product decision rather than automated implemented-on-main cleanup.
