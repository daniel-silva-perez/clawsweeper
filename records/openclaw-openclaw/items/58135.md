## Summary

Keep this issue open. Current main supports `promptMode` internally on direct agent runs and defaults sub-agent sessions to `minimal`, but the public `sessions_spawn` tool schema, native spawn params, and child `agent` gateway call do not accept or forward `promptMode`. This is not a ClawHub close: the request is for a missing core `sessions_spawn` API parameter, and the project vision explicitly points API-gap cases back to OpenClaw core/API design rather than external plugins.

## Best Possible Solution

Add an optional `promptMode?: "full" | "minimal" | "none"` to `sessions_spawn` for native sub-agent runs, keep the default as current `minimal`, include it in `SpawnSubagentParams`, validate it in the tool schema, forward it to the child `agent` gateway call, and document it in `docs/tools/subagents.md`. Add focused tests proving omitted `promptMode` preserves current behavior and `promptMode: "none"` reaches the child run config.

## Risks / Open Questions

- Exposing `promptMode: "none"` intentionally strips injected context and safety/workspace sections for child runs, so the implementation should document the security tradeoff and preserve the current default unless the caller explicitly opts in.
- ACP runtime semantics may differ from native sub-agents; maintainers should decide whether `promptMode` is native-only at first or has a defined ACP mapping.
