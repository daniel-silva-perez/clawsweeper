## Summary

Current main does not implement this PR’s intended change: `CliBackendConfig.command` and `CliBackendSchema.command` are still required, and the generated public config schema still requires `command`. The PR is useful but needs an update before merge, especially regenerating `src/config/schema.base.generated.ts` so schema-driven config consumers match the relaxed runtime contract.

## Best Possible Solution

Keep this PR open and ask for an update: make `command` optional in the Zod schema and TS input type, preserve a resolved type/guard for spawn-time `command: string`, add the partial `modelAliases` regression test, regenerate the base config schema, and run the targeted config schema check plus the CLI backend tests before review/merge.
