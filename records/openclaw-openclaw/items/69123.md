## Summary

Keep this PR open. Current main still rejects partial `agents.defaults.cliBackends.<id>` overrides at the source Zod/type/generated-schema boundary, while the runtime merge and resolver already support inheriting `command`; the PR addresses the right bug but remains incomplete because it does not update the generated config schema/docs baseline and should double-check the resolved-command guarantee across all CLI execution paths.

## What This Changes

The PR makes `CliBackendSchema.command` and `CliBackendConfig.command` optional, introduces a resolved-backend command guarantee, adds a defensive spawn-site guard, and adds a regression test for a `modelAliases`-only `cliBackends` override inheriting the base command.

## Best Possible Solution

Keep this PR open and require a focused update: make `command` optional for config input, preserve a resolved `command: string` contract across all CLI execution paths, keep the partial `modelAliases` regression test, regenerate `src/config/schema.base.generated.ts` and `docs/.generated/config-baseline.sha256`, add a one-line changelog fix entry with contributor attribution, then run the targeted CLI backend test, config schema/docs checks, core typecheck, and import-cycle check.
