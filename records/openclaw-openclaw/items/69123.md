## Summary

Current main still does not implement this PR's intended change: `CliBackendConfig.command`, `CliBackendSchema.command`, and the generated public config schema all still require `command`. The PR remains useful, but it needs a branch update that also regenerates `src/config/schema.base.generated.ts` and preserves the resolved non-empty command invariant before merge.

## Best Possible Solution

Keep this PR open and ask for an update: make `command` optional in the Zod schema and TypeScript input type, keep a resolved backend type or spawn-time guard that guarantees `command: string`, add the partial `modelAliases` regression test, regenerate `src/config/schema.base.generated.ts`, and run the targeted CLI backend/config schema validation before review or merge.
