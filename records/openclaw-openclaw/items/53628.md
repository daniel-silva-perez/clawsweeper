## Summary

Current main still leaves environment placeholders literal in the shared user-path resolver used by workspace resolution and ClawHub skill installs. The report also has open implementation PRs, including #58281 with closing syntax, so this should stay open until maintainers land or reject the chosen fix.

## Best Possible Solution

Keep this issue open. Review and either land or reject #58281 as the canonical fix, folding in useful coverage from #53718. The best code path is a deliberate shared resolver change in `src/infra/home-dir.ts` with tests for `${XDG_CONFIG_HOME}`, `$XDG_CONFIG_HOME`, unknown variables, and a ClawHub skill-install target that no longer creates literal placeholder directories.
