## Summary

Current main still leaves environment placeholders literal in the shared user-path resolver used by agent workspace resolution and ClawHub skill installs. The issue also has open implementation PRs with closing syntax, so it should stay open until maintainers review and merge or reject the chosen fix.

## Best Possible Solution

Keep this issue open. Review and either land or close PR #58281, folding in any useful coverage from #53718, with regression tests for ${XDG_CONFIG_HOME} and $XDG_CONFIG_HOME through the shared path resolver and a ClawHub skill-install path that no longer creates literal placeholder directories. After the selected PR merges, this can close as implemented.
