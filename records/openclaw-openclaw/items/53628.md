## Summary

Current main still leaves environment placeholders literal in the shared user-path resolver used by active workspace resolution and ClawHub skill installs, so the reported behavior remains valid. The issue is also paired with open implementation PRs that reference it with closing syntax, so it should stay open until maintainers review and land or close the chosen fix.

## What This Changes

The issue requests that ClawHub skill installation expand `${XDG_CONFIG_HOME}` and similar environment placeholders instead of creating literal placeholder directories.

## Best Possible Solution

Land one canonical shared resolver fix that expands supported environment placeholders before path resolution and adds regression coverage for the resolver plus the active-workspace ClawHub skill-install path. After the selected PR merges, this issue can close as implemented.
