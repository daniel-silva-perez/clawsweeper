## Summary

Keep open: current main still resolves configured workspace paths without expanding `${VAR}` or `$VAR` placeholders, and the reported ClawHub skill install path still flows through that resolver before installing into `skills/`. The issue also has open implementation PRs that reference it, so it should remain open until maintainers review and land or close the canonical fix.

## What This Changes

The issue requests that ClawHub skill installation expand `${XDG_CONFIG_HOME}` from the environment instead of treating it as a literal directory name.

## Best Possible Solution

Land one canonical resolver fix that expands supported environment placeholders before path resolution, with regression coverage for `resolveHomeRelativePath`/`resolveUserPath` and the active-workspace ClawHub skill-install path. After the selected PR merges and any separate ClawHub CLI surface is accounted for, this can close as implemented.
