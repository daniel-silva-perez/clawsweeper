## Summary

Keep this PR open. Current main still does not sanitize the shared channel logging helpers, so the PR is not obsolete or implemented-on-main. The PR also should not be merged as-is because its diff leaves `channel` unsanitized, duplicates an existing stronger sanitizer, and can introduce runtime errors for loosely typed plugin callers.

## Best Possible Solution

Keep this PR open and ask for a narrow revision: reuse the existing `sanitizeForLog` from `src/terminal/ansi.ts`, coerce each interpolated value with `String(...)` before sanitizing, sanitize every interpolated field including `channel`, `action`, `reason`, `target`, and `error`, and add focused tests for channel injection, DEL/C1/ANSI stripping, and non-string runtime inputs. After that, review the updated patch for merge instead of closing it.
