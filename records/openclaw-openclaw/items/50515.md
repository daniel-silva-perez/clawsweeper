## Summary

Keep this PR open. Current main still has unsanitized shared channel logging helpers, so the PR is not obsolete or implemented-on-main. The patch remains useful but needs revision before merge because it leaves `channel` unsanitized, duplicates a broader existing sanitizer, and may regress loose JavaScript plugin callers by calling `.replace` on non-string values.

## Best Possible Solution

Keep this PR open and request a narrow revision: reuse `sanitizeForLog` from `src/terminal/ansi.ts`, coerce interpolated values with `String(...)` before sanitizing, sanitize every interpolated field including `channel`, `reason`, `target`, `action`, and `error`, and add focused channel logging tests for channel injection, DEL/C1/ANSI stripping, and non-string runtime inputs.

## Risks / Open Questions

- Merging as-is would still leave `params.channel` as a log-forging surface in the shared helpers.
- Merging as-is would introduce a second sanitizer with weaker and different behavior than `src/terminal/ansi.ts`.
- Merging as-is could throw for JavaScript plugin callers that currently rely on template-literal stringification for non-string `target` or `reason` values.
