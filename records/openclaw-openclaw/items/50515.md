## Summary

Keep this PR open. Current main still does not apply log sanitization in the channel logging helpers, and the PR's own diff still needs revision before merge: it sanitizes only reason/target/error, leaves channel unsanitized, duplicates an existing project sanitizer, and can regress loosely typed plugin callers by calling .replace on non-string values.

## Best Possible Solution

Keep the PR open and ask for a narrow revision: reuse sanitizeForLog from src/terminal/ansi.ts, sanitize every interpolated field including channel/action/reason/target/error, coerce values with String(...) before sanitizing to preserve runtime compatibility, and add focused channel logging tests for channel injection, DEL/C1/ANSI coverage, and non-string runtime inputs. After that, review and merge the PR rather than closing it as implemented.
