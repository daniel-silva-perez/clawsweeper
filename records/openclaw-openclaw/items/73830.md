## Summary

Keep this PR open. Current main still has the raw `curl ... | bash` installer command, but the proposed `bash -s --` docs change does not protect child prompts from consuming the same pipe. The linked installer bug remains real enough to investigate, and the fix should either harden `scripts/install.sh` or update all install docs to a validated invocation pattern.

## Best Possible Solution

Keep this PR open for maintainer review or requested changes. The best fix is to validate the WSL/Linux failure against the hosted installer, then either harden `scripts/install.sh` so all interactive prompts and child commands use terminal-backed stdin where appropriate, or switch every public install surface to a tested caller-side pattern. Add a focused installer stdin regression check and update the install docs consistently.
