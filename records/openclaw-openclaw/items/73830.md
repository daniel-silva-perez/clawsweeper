## Summary

Keep this PR open for maintainer review. Current main still has the raw `curl | bash` Getting Started command, but the proposed `bash -s --` explanation is not a proven fix for stdin-consuming prompts and the install docs would remain inconsistent if this landed as-is.

## Best Possible Solution

Keep this PR open and request maintainer review or changes. The best fix is to validate the WSL/Linux `curl | bash` failure against the hosted installer, then either fix stdin handling in `scripts/install.sh` for all interactive prompts or switch the public docs to a tested caller-side pattern, while updating all public install surfaces consistently.
