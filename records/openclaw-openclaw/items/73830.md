## Summary

Keep this PR open for changes before merge. The linked installer bug appears real enough to investigate, but the proposed documentation fix is not correct because `bash -s --` does not isolate child process stdin from the pipe, and current main still has multiple public install commands that would remain inconsistent.

## What This Changes

This PR updates the Getting Started macOS/Linux install snippet to pipe the hosted installer into `bash -s --` and adds a note saying those flags prevent stdin consumption.

## Best Possible Solution

Validate the hosted installer failure, harden `scripts/install.sh` so interactive prompts and subprocesses that need user input use terminal-backed stdin or explicit noninteractive behavior, then update all public install commands consistently and add focused installer stdin regression coverage.

## Reproduction Assessment

Yes for the PR's claimed fix: the focused Bash pipe smoke shows `bash -s --` does not prevent a child read from consuming the script stream. The complete WSL2 hosted-installer hang from #73814 still needs live validation.

## Solution Assessment

No. Updating one quickstart snippet and adding a stdin-isolation guarantee is not the narrowest maintainable fix because the guarantee is false and the rest of the install documentation remains inconsistent.
