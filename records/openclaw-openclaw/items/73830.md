## Summary

Keep this PR open. Current main has not implemented the proposed Getting Started docs change, and this open PR is the active implementation candidate for the linked installer bug #73814. It needs normal maintainer review rather than cleanup closure, especially because `bash -s --` alone may not actually prevent stdin-consuming commands from reading the piped script, and other install docs still show the raw `curl | bash` form.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to verify the real WSL/Linux `curl | bash` failure mode against the hosted installer, decide whether the fix belongs in installer stdin handling, a safer downloaded-file/docs pattern, or a docs-only command update, and then align all public install docs if the docs command changes.
