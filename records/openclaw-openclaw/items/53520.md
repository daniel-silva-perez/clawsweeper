## Summary

Current main does not implement this PR's proposed portable `openclaw migrate export` / `openclaw migrate import <archive>` workflow. The current `migrate` command imports from plugin-owned providers such as Claude/Hermes, while `backup` only creates and verifies archives. The PR remains a product/security-sensitive restore/import design question rather than conservative cleanup.

## Best Possible Solution

Keep this PR open for maintainer product/security review. The best path is for maintainers to choose the intended scope first: rework into narrower backup export/convert improvements, or design a carefully gated restore/import flow with archive safety helpers, preview/conflict preflight, explicit trust prompts, backup/rollback behavior, and focused tests. Current main does not already solve the PR's central request.
