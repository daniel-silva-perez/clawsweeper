## Summary

Keep open. Current main does not implement archive-based OpenClaw-to-OpenClaw export/import; the shipped migrate command is provider-based import from other agent systems, backup only creates/verifies archives, and the public docs still direct machine-to-machine moves through manual copying. The PR also remains a broad restore/import change with maintainer-expressed security and product-scope concerns, so cleanup closure would be unsafe.

## What This Changes

The PR branch adds archive-based `openclaw migrate export` and `openclaw migrate import` commands with selective components, secret stripping, path remapping, config merge, CLI registration, archive import/export infrastructure, and tests.

## Best Possible Solution

Choose the recovery model first. If core should stay conservative, rework this toward smaller `backup` export/target/convert improvements plus clear manual restore docs. If OpenClaw should own automated import, design it as an explicitly trusted, preview-first restore flow with strong archive validation, rollback/backups, conflict handling, and security-focused tests before merge.
