## Summary

Keep this PR open for maintainer review. Current main does not implement the requested cross-device archive export/import flow, but the branch is not ready to merge: it overlaps the newer provider-based `openclaw migrate` surface, adds a security-sensitive archive restore path, and still has concrete correctness/security findings before merge.

## What This Changes

The PR branch adds archive-based `openclaw migrate export` and `openclaw migrate import` commands with selective components, secret stripping, path remapping, config merge, CLI registration, import/export infrastructure, and tests.

## Best Possible Solution

Choose the recovery model first. If core stays conservative, evolve `openclaw backup` with platform targeting, archive conversion/verification, and clearer manual restore docs. If OpenClaw should own automated import, integrate it with the existing provider-based migrate namespace and require preview-first UX, canonical target checks, archive validation, rollback/conflict handling, and security regression tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Canonicalize targets before destructive writes:** `src/infra/migrate-import.ts:328-333`
  - body: `assertTargetWithinRoot` only compares `path.resolve` strings, so a symlinked existing parent such as `~/.openclaw/agents` can pass validation while the later `fs.rm`/copy operates outside the OpenClaw state tree. Resolve existing target ancestors or reject symlinked target parents before any destructive import write.
  - confidence: 0.89
- **[P2] Do not export default workspace for unknown agents:** `src/commands/migrate-shared.ts:174-180`
  - body: When `--agents` contains an id that is not in `agents.list` and does not match an on-disk agent, this branch still adds the default workspace because the requested agent lacks an explicit workspace. That makes `openclaw migrate export --agents typo --include workspace` include unrelated workspace data despite matching no agent.
  - confidence: 0.88
- **[P2] Preserve existing provider migrate commands:** `src/cli/program/register.migrate.ts:45-58`
  - body: Current main already uses `openclaw migrate` for plugin-owned provider imports with `list`, `plan`, and `apply`. This registration defines the command as archive `export`/`import` only, so after rebase it would conflict with or drop the existing Claude/Hermes migration surface.
  - confidence: 0.84

## Security Review
