## Summary

Keep this PR open. Current main still hard-disables the Android Voice Tab manual mic path when `SpeechRecognizer.isRecognitionAvailable(context)` is false, and there is still no configured OEM speech-recognizer fallback. The PR addresses a real remaining gap with useful Xiaomi validation, but it needs a rebase onto the current Android package path and a no-backend retry safety path before merge.

## Best Possible Solution

Keep this PR open for revision. The best path is to rebase it onto `apps/android/app/src/main/java/ai/openclaw/app/voice/MicCaptureManager.kt`, keep the generic configured-service fallback using the device `voice_recognition_service` setting plus `ComponentName.unflattenFromString` and `SpeechRecognizer.createSpeechRecognizer(context, component)`, and add a no-backend safety path that disables manual mic or avoids retrying when no usable recognizer exists. Add focused Android/Robolectric coverage around the creation/error seam if practical; otherwise keep the Xiaomi manual proof and explain why platform mocking is not reliable.

- There is no focused `MicCaptureManager` recognizer fallback regression test, so the no-backend safety behavior could regress without either a small test seam or clearly documented manual device proof.
