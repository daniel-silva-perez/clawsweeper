## Summary

Keep this issue open. Current main has opt-in compaction lifecycle notices and a silent pre-compaction memory flush, but it still lacks the requested user-facing pre-threshold prompt, custom message/threshold config, and response window before flush/compaction.

## Best Possible Solution

Keep the issue open and implement this as a core compaction/memory-flush UX feature under agents.defaults.compaction: add configurable pre-notification text and threshold, deliver it to the active channel before memory flush or preflight compaction, optionally wait/timebox for the user's response, feed that response into the memory-flush/compaction context, and cover schema, docs, and reply-path tests.

## Risks / Open Questions

- The existing notifyUser option is easy to mistake for this request, but it is only fixed lifecycle status text and does not let the user influence what memory flush saves.
- Adding a response window must be carefully timeboxed so compaction does not stall active replies or create duplicate channel messages.
- The implementation will need schema/docs/tests because it changes a user-facing config contract and active-channel delivery timing.
