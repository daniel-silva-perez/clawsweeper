## Summary

Keep this issue open. Current main has an opt-in `agents.defaults.compaction.notifyUser` setting, but it only sends fixed start/end status notices when a compaction event is already underway. It does not implement the requested pre-threshold user prompt, custom message/threshold config, response window, or visible notification before the silent memory-flush path. The provided related reports cover different notification/config-safety issues and do not supersede this feature request.

## Best Possible Solution

Keep the issue open and implement this as a core compaction/memory-flush UX feature, likely under `agents.defaults.compaction`: add a configurable pre-notification message and threshold, send it to the active channel before memory flush or preflight compaction, optionally wait or timebox for a user response, fold that response into the memory-flush/compaction context, and cover the embedded Pi reply path plus channel delivery tests.
