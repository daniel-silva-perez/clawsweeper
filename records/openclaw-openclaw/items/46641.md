## Summary

Keep this issue open. Current main has opt-in compaction lifecycle notices and a silent pre-compaction memory flush, but it still does not implement the requested user-facing pre-threshold prompt, custom message/threshold config, or response window before memory flush/compaction proceeds.

## Best Possible Solution

Keep the issue open and implement this as a core compaction/memory-flush UX feature, likely under `agents.defaults.compaction`: add a configurable pre-notification message and threshold, deliver it to the active channel before preflight compaction or memory flush, optionally wait/timebox for the user's response, feed that response into the memory-flush/compaction context, and cover the reply path plus schema/docs tests.
