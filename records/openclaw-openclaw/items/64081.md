## Summary

Keep this issue open. Current main still has no typed, schema-backed, or documented config-level default for task notification policy, and ACP, subagent, Task Flow, and ACP control-plane task creation still fall back to hardcoded `done_only` behavior unless the caller supplies `notifyPolicy`. The related reports are adjacent, with #73905 covering a narrower subagent/cron inheritance case rather than this broader config default.

## What This Changes

Add a config-level default task notification policy so users can suppress or tune raw background task completion messages for ACP/subagent-style work while preserving explicit per-task overrides.

## Best Possible Solution

Add a backwards-compatible core config surface, likely under a maintainer-approved `agents.defaults.tasks` shape, that resolves the effective default before ACP, subagent, and Task Flow records are created. Keep current behavior when unset, keep cron/CLI/media silent defaults intact, let explicit task or flow `notifyPolicy` win, and document the behavior on the public tasks, subagents, and config pages.
