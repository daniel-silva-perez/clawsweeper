## Summary

Keep this issue open. Current main still has no typed, schema-backed, or documented config-level default for task notification policy, and ACP, subagent, Task Flow, and ACP control-plane task creation still fall back to hardcoded `done_only` behavior unless the caller explicitly supplies `notifyPolicy`. The related items are adjacent rather than canonical fixes: #73905 is narrower subagent/cron policy design, #46926 is background exec wake behavior, and #42438 is channel allowlist DM policy.

## What This Changes

The issue requests a config-level default task notification policy so users can suppress or adjust raw ACP/subagent completion notifications without racing per-task policy changes.

## Best Possible Solution

Add a backwards-compatible core config surface, likely under `agents.defaults.tasks` or a maintainer-approved subagent/defaults shape, and resolve the effective policy before ACP, subagent, and Task Flow records are created. Preserve current behavior when unset, keep cron/CLI/media silent defaults intact, let explicit per-task or flow `notifyPolicy` win, and document the behavior on the public tasks/subagents/config pages.
