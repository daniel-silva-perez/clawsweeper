## Summary

Current main still does not implement a config-level default task notification policy. ACP, subagent, and Task Flow paths still fall back to hardcoded `done_only` defaults unless an individual task or flow supplies `notifyPolicy`; cron and media paths remain explicitly silent. The issue should stay open for a core config/schema/docs implementation.

## Best Possible Solution

Keep this open and implement a core config/schema/docs change, likely under an agreed `agents.defaults.tasks` shape. Resolve the effective default before ACP, subagent, and Task Flow records are created; preserve current behavior when unset; keep cron, CLI, and media silent defaults intact; let explicit per-task or per-flow `notifyPolicy` overrides win; and add focused regression coverage for ACP, subagent, Task Flow, cron, media, and `openclaw tasks notify` override behavior.
