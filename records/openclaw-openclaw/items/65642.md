## Summary

Keep this PR open. Current main still lacks the intended OpenResponses behavior: the streaming handler only bridges assistant/lifecycle events, terminal streaming snapshots only include the assistant message, and the output schema still does not allow `function_call_output` as a response output item. The PR is still the active implementation candidate, but it needs maintainer/security review because it changes Gateway SSE semantics and exposes raw internal tool results to authenticated HTTP consumers.

## Best Possible Solution

Keep this PR open for maintainer/security review. The best path is to land this PR or an equivalent Gateway change that bridges internal `stream: "tool"` events into stable OpenResponses `function_call` and `function_call_output` SSE output items, includes those items in terminal snapshots, preserves the intended redaction boundary for shared agent events, normalizes response-shaped next-turn inputs, and adds focused gateway/event regression coverage.
