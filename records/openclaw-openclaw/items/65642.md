## Summary

Keep this PR open. Current main still does not implement the intended OpenResponses behavior: the streaming handler does not consume internal `stream: "tool"` events, terminal streaming snapshots only include the assistant message, and the output-item schema still lacks `function_call_output`. The PR remains a live, focused implementation candidate, but it needs maintainer/security review because it changes gateway SSE semantics and raw tool-result exposure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or an equivalent gateway change that bridges internal `stream: "tool"` events into stable OpenResponses `function_call` and `function_call_output` SSE output items, includes those items in terminal snapshots, preserves raw HTTP tool output without leaking it on shared event surfaces, normalizes response-shaped next-turn inputs, and adds focused gateway/agent-event regression coverage.
