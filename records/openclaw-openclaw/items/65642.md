## Summary

Keep this PR open. Current main still does not implement the PR's intended OpenResponses behavior: internal tool events are not bridged into `/v1/responses` SSE output items, terminal snapshots only include the assistant message, and the response output schema still lacks `function_call_output`. The branch remains the active implementation candidate, but it needs maintainer/security review because it changes Gateway SSE semantics and exposes raw internal tool results to authenticated HTTP consumers.

## What This Changes

The PR adds Gateway OpenResponses streaming support for internal tool `function_call`/`function_call_output` output items, a non-enumerable raw tool-result side channel for HTTP output, AI SDK-style input normalization, schema updates, and regression tests.

## Best Possible Solution

Keep this PR open for maintainer/security review, then land this branch or an equivalent Gateway change that emits internal `stream: "tool"` events as OpenResponses `function_call` and `function_call_output` SSE output items, includes those items in terminal snapshots, preserves the sanitized shared event-bus boundary, normalizes response-shaped next-turn inputs, and adds focused Gateway/event regression coverage.
