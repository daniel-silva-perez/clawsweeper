## Summary

Keep this PR open. Current main does not implement the PR's central behavior: reminder-like chat cron jobs with `payload.kind: "systemEvent"` and no explicit `sessionTarget`/`delivery` still normalize to `sessionTarget: "main"`, run as main-session system events, and record delivery as `not-requested`. Main only infers announce delivery for `agentTurn` jobs. SonicBotMan's comment is useful scheduler-drift context, but it describes a separate timing/load problem rather than the chat-reminder delivery promotion in this PR.

## Best Possible Solution

Keep this PR open for maintainer review or a successor patch. The best path is to either land a well-tested promotion flow that only affects implicit chat-created reminders and preserves explicit `sessionTarget: "main"` / `delivery` semantics, or close the PR after a different merged change implements the same behavior on main. The scheduler-drift feedback from the comment should become a separate issue or be linked to an existing cron timing tracker.
