## Summary

Keep this PR open, but do not merge it as-is. Current main still has the reminder delivery gap the PR is trying to address, while the proposed patch is stale against newer cron delivery-context work and introduces blocking routing bugs around wrapped job inputs and preserved chat targets.

## What This Changes

The PR changes the cron tool to promote reminder-like chat-created `systemEvent` jobs without explicit `sessionTarget` or `delivery` into isolated `agentTurn` announce jobs, with tests for promotion, explicit main jobs, and context history.

## Best Possible Solution

Rebase and narrow the PR so chat-created reminder promotion preserves all existing cron input contracts: respect explicit `sessionTarget` and `delivery` through unwrapped job shapes, reuse current delivery-context inference for exact channel targets, keep explicit internal `systemEvent` and `delivery.mode: "none"` paths internal, and add regression coverage for wrapped `job.data`, Matrix/current-context routing, and related internal system-event cases.

## Reproduction Assessment

Yes. The current-main behavior can be reproduced at the unit boundary by creating a cron tool with a chat `agentSessionKey` and adding an omitted-target `systemEvent` reminder: normalization leaves it as a main-session job with no delivery. The PR-specific regression is also reproducible with the already-supported `job.data` wrapper plus nested `sessionTarget: "main"`, which the new raw wrapper check would miss.

## Solution Assessment

No, not as written. Promoting chat reminders is a plausible direction, but the current patch is not the safest implementation because it bypasses current delivery-context preservation and does not reliably detect explicit internal intent inside accepted wrapper inputs.
