## Risks / Open Questions

- Merging as-is can leak secret-like config fields when a dictionary entry id collides with sanitizer container key names such as `entries`, `servers`, `models`, `cliBackends`, or `providerOptions`.
- Importing raw plugin install records from an archive can affect future plugin reinstall/update behavior and should validate schema and provenance before becoming trusted local state.
- Dry-run import currently appends would-write entries to `filesWritten`, which can mislead JSON consumers that treat that field as actual writes.
- The PR is an XL, security-sensitive archive import/export surface; it needs normal maintainer product and security review, not cleanup closure.
