## Summary

Keep open. Current main does not provide the proposed profile archive command surface, and the PR still has a blocking privacy/security bug in the branch-local sanitizer: fields that current main already treats as sensitive can survive into an archive marketed as privacy-safe.

## What This Changes

The branch adds `openclaw profile export/import`, a `.openclaw-profile.tar.gz` manifest/archive implementation for sanitized config, workspace markdown, and plugin install records, docs, CLI/command/infra tests, and adjacent gateway/plugin test fixture updates.

## Best Possible Solution

If core profile portability is accepted, land it as a documented archive format that reuses the shared sensitive-key and sensitive-URL semantics, has regression coverage for sanitizer misses, keeps plugin install metadata as a portable projection, preserves non-destructive import behavior, and stays clearly scoped away from full backup/restore or cloud sync.

## Review Findings

Overall correctness: patch is incorrect

  - body: The branch-local sanitizer only drops token/password/api-key style names.
