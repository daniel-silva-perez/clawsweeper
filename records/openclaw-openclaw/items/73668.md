## Summary

Keep open. Current main does not implement this profile export/import surface, and the latest PR head is still not merge-ready because the privacy sanitizer can leak host-local paths from arbitrary portable config, with smaller docs/changelog follow-ups also outstanding.

## What This Changes

The PR adds a new `openclaw profile export/import` CLI, profile archive creation/import infrastructure, docs, unit tests, and adjacent test fixture updates.

## Best Possible Solution

Land profile portability only after the archive format consistently removes secrets and host-local path material from all exported config projections, preserves non-destructive import behavior, keeps plugin install records as a portable projection, updates discoverability docs and changelog, and has regression tests for the sanitizer gaps.

## Reproduction Assessment

Yes. A unit-level `buildSafeProfileConfig` input with `plugins.entries.demo.config.sourcePath: "/home/alice/private"`, `mcp.servers.demo.localPath`, or `workspacePath` will pass through the current key filter because those normalized keys are neither exact local-path keys nor secret suffixes.

## Solution Assessment

No. The feature direction is plausible, but the proposed patch is not the best mergeable solution until path-like config keys are stripped or values are otherwise privacy-filtered, with regression coverage added before normal validation.

## Security Review

Status: needs_attention

Summary: Profile export can include host-local paths from arbitrary portable config because the sanitizer misses path-like key suffixes.

Concerns:

- **[medium] Profile archive can include local paths:** `src/infra/profile-portability.ts:363`
  - body: The privacy-safe archive sanitizer omits exact keys such as `path` and `cwd`, but it misses common path-like config keys such as `sourcePath`, `localPath`, `workspacePath`, and `configPath`. Because the exported projection includes arbitrary plugin, skill, MCP, and tool config, user names, private project names, or host-specific directories can be copied into the archive.
