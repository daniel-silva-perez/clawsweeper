## Summary

Keep open. Current main still does not provide privacy-safe `openclaw profile export/import`, and the existing `backup` and `migrate` commands cover different recovery/import models. The latest PR head has useful work and appears to address the earlier sanitizer collision review, but the remaining local-state import/export and plugin ledger handling need maintainer security and product review before merge.

## What This Changes

The PR adds a top-level `openclaw profile export/import` CLI, a `.openclaw-profile.tar.gz` archive implementation for sanitized config, workspace persona/memory files, and plugin install records, plus docs, tests, and related test harness adjustments.

## Best Possible Solution

Keep this PR open for maintainer product and security review. If profile portability belongs in core, land it with the sanitizer collision regression retained, plugin install records validated or rebuilt through the installed plugin index contract, docs navigation/index updates, and unrelated test trust changes split out or justified separately.
