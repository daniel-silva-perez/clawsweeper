## Summary

Keep open. Current main does not implement profile archive export/import, and this active PR adds a new privacy-sensitive CLI/archive surface that still has a blocking sanitizer gap on the latest head.

## What This Changes

The PR adds `openclaw profile export/import`, profile archive creation/import infrastructure, docs, CLI/command/infra tests, and adjacent test fixture updates.

## Best Possible Solution

If core profile portability is accepted, land it as a documented privacy-safe archive format that reuses the shared sensitive-key semantics, keeps plugin install metadata as a sanitized portable projection, preserves non-destructive import behavior, and adds regression coverage for credential-like names, dictionary collisions, traversal rejection, local path omission, and plugin install record rebuilding.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Reuse shared sensitive-key matching for profile export:** `src/infra/profile-portability.ts:360-372`
  - body: The profile sanitizer only drops a hard-coded subset of secret-looking keys. Fields such as `plugins.entries.<id>.config.privateKey`, `encryptKey`, `serviceAccount`, or `serviceAccountRef` are treated as sensitive by current main's shared matcher but survive this export path, so a privacy-safe archive can still leak credentials from arbitrary plugin/skill/MCP config. Reuse the shared sensitive-path logic or add equivalent patterns with regression coverage.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The PR introduces a new profile archive export path that can leak credential-like config fields because its sanitizer is narrower than OpenClaw's shared sensitive-key matcher.

Concerns:

- **[high] Profile export can include private keys:** `src/infra/profile-portability.ts:360`
  - body: `shouldOmitPortableConfigKey` omits tokens, passwords, secrets, and API keys, but not private-key, encryption-key, or service-account patterns that OpenClaw already treats as sensitive. Because the archive includes arbitrary portable plugin/skill/MCP config, those values can be written to a privacy-safe profile archive.
  - confidence: 0.9
