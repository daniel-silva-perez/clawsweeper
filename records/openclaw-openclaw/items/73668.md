## Summary

Keep open. Current main still has no top-level `profile` CLI command or profile-portability archive implementation, and the shipped `backup` and `migrate` surfaces are not equivalent to the PR's proposed privacy-safe OpenClaw-to-OpenClaw profile import/export flow. This is an active XL, privacy-sensitive implementation PR, so the next step is maintainer product and security review rather than cleanup closure or an automated replacement fix.

## What This Changes

The PR adds an `openclaw profile export/import` CLI, a `.openclaw-profile.tar.gz` portability archive layer for sanitized config, workspace profile/memory markdown, and plugin install records, plus docs and focused CLI/command/infra tests.

## Best Possible Solution

Keep this PR open for maintainer product and security review. If core profile portability is accepted, the desired end state is a documented privacy-safe archive format with sanitizer regression coverage, non-destructive import semantics, plugin install metadata handled through the existing plugin-index contract, and clear docs that distinguish profile portability from full backup/restore and external migration.
