## Summary

Keep this issue open. Current main has model fallback chains, automatic fallback session pinning, and fallback status/verbose notices, but it does not implement the requested user confirmation mode before switching from the primary model to a fallback. The provided related title matches are Feishu interactive-card/text-fallback items, not the same model-failover policy request.

## Best Possible Solution

Keep this open and implement it in core model-fallback policy rather than as a plugin-only feature. A solid path is to extend `AgentModelConfig` and gateway/config docs with a backwards-compatible `fallbackMode: "auto" | "confirm"`, gate only interactive chat sessions before trying a non-primary fallback candidate, reuse existing channel/native prompt or command-response infrastructure, define deterministic defaults for cron/non-interactive runs, and add focused tests around candidate ordering, declined fallback, picked fallback, and strict no-fallback behavior.

## Evidence

There is no confirmation/prompt branch before attempting a non-primary candidate.
