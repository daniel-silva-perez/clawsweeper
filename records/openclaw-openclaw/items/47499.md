## Summary

Keep open. Current main has related pieces: sandboxed browser noVNC observer links, Codex Computer Use setup, PeekabooBridge, direct cua-driver MCP, and early Control UI descriptor metadata. It does not implement the requested embedded Dashboard VNC viewer, agent-initiated desktop takeover permission flow, or full-desktop VNC control surface. The remaining work is product/security/API design, not conservative cleanup.

## Best Possible Solution

Keep the issue open and scope it as a maintainer-level design item. The best path is to split the work into a core Control UI/plugin-surface design for rich embedded viewers and explicit operator permission prompts, plus a backend strategy that treats VNC as visual preview while using semantic desktop-control backends such as Codex Computer Use, Peekaboo/accessibility, or cua-driver MCP for actions. Once that API exists, a VNC/noVNC implementation could plausibly live as a plugin/ClawHub package.
