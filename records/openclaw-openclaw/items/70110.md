## Summary

Current main still has the reported Nextcloud Talk reaction wiring gap. The plugin advertises reaction support and has a working HTTP sender helper, but it does not register a channel message action adapter, so shared `message` tool `react` dispatch returns `null` before `sendReactionNextcloudTalk` can run. Two open PRs already reference this issue with closing syntax, so the issue should stay open until one lands or an equivalent fix is merged.

## What This Changes

The issue asks for Nextcloud Talk emoji reactions to be sent when an agent uses the shared message reaction action.

## Best Possible Solution

Keep the issue open and review, land, or supersede #72348 and #72679 with one narrow fix that registers a Nextcloud Talk `ChannelMessageActionAdapter`, dispatches `react` to `sendReactionNextcloudTalk`, preserves existing sender/security behavior, and adds regression coverage for action discovery and dispatch.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: `react` dispatch for `nextcloud-talk` cannot reach the sender because the plugin has no `actions.handleAction`, and the dispatcher returns `null` in that case.

## Solution Assessment

Yes, the best maintainable direction is a plugin-owned action adapter rather than a generic core inference from `capabilities.reactions`. Closing this issue before one of the linked PRs lands would be premature.
