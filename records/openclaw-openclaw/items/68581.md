## Summary

Keep open. Current main still implements Android node location as foreground-only: the mode model and settings UI expose only Off and While Using, the handler returns `LOCATION_BACKGROUND_UNAVAILABLE` before attempting any location read when the app is backgrounded, and the manifest lacks the background-location and location foreground-service declarations requested here.

## What This Changes

Request adding Android `location.enabledMode: always` so background `nodes location get` can work through explicit background-location permission and a location foreground service.

## Best Possible Solution

Add an Android-owned Always mode end to end: enum/prefs/settings support, Android background-permission flow, a foreground service declared with location type, a background `location.get` path that preserves the existing While Using error, and aligned docs/tests with explicit privacy and release-channel treatment.

## Reproduction Assessment

Yes. Current-main source and unit tests reproduce the reported behavior: a background `location.get` request returns `LOCATION_BACKGROUND_UNAVAILABLE` before any location capture path is reached.

## Solution Assessment

Unclear until maintainer review. A foreground service plus background-location permission is the plausible Android implementation path, but the repo already treats `ACCESS_BACKGROUND_LOCATION` as a Play-restricted surface, so the final fix should be chosen with privacy and release-flavor review.
