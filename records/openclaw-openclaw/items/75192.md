## Summary

Keep open. Current main still has backend support for agent-scoped dashboard sessions, but the Control UI chat plus/New session action does not present configured agents or call `sessions.create` with a selected `agentId`.

## What This Changes

The issue requests a WebChat/Control UI flow to create a new chat session for a user-selected configured agent and switch to that session.

## Best Possible Solution

Add a focused Control UI new-chat flow that uses the existing `agents.list` data, calls `sessions.create` with the selected `agentId`, switches to the returned session key, refreshes session/history/avatar/tool state, and leaves broader sidebar or labeled-session redesigns to their separate issues.

## Reproduction Assessment

Yes. A source-level reproduction is clear: with multiple configured agents, the chat plus button still dispatches `/new` in the active session and no visible UI path passes a selected `agentId` to `sessions.create`.

## Solution Assessment

Yes. The narrowest maintainable fix is a UI addition over existing `agents.list` and `sessions.create`; no Gateway protocol change is indicated unless implementation discovers missing metadata or authorization handling.
