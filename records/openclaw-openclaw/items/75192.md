## Summary

Keep open. Current main still does not provide a WebChat/Control UI flow for choosing a configured agent when creating a new chat; the visible plus/New session path still sends `/new` in the active session, while the Gateway already has `sessions.create` support for an `agentId`. Related #73725 is an open maintainer PR for the generic create-and-switch slice, but it is not merged on main and does not establish that selected-agent creation is exposed in the UI.

## What This Changes

The issue requests a WebChat UI flow that lets users choose a configured agent, create a new chat session for that agent, and switch to it.

## Best Possible Solution

Sequence this with the active WebChat session-create work: land or update the generic `sessions.create` create/switch path, then add a focused selected-agent picker that reuses `agents.list`, calls `sessions.create` with the chosen `agentId`, switches to the returned key, and refreshes session/history/avatar/tool state without changing the Gateway contract unless a concrete missing field appears.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is available: configure multiple agents, open Chat, click the plus/New session control, and current main can only send `/new` in the active session; no visible UI path passes a selected `agentId` to `sessions.create`.

## Solution Assessment

Yes, with sequencing. The narrow maintainable solution is a Control UI addition over existing `agents.list` and `sessions.create`, while reusing or extending #73725 if it remains the maintainer-approved create/switch path.
