## Summary

Keep open. Current main still lacks a WebChat/Control UI flow that creates a new chat with a selected configured agent: the visible New Session affordance still sends `/new` to the active session, while the Gateway and protocol already support `sessions.create` with `agentId`. The related open #73725 covers the generic create-and-switch path, but it is not merged and does not make selected-agent creation available on main.

## What This Changes

The issue requests a WebChat UI flow that lets users choose a configured agent, create a new chat session for that agent, and switch to it.

## Best Possible Solution

Land or update the generic WebChat create/switch path first, then add a focused selected-agent picker that reuses `agents.list`, calls `sessions.create` with the chosen `agentId`, switches to the returned key, refreshes session/history/avatar/tool state, and avoids Gateway contract changes unless a concrete missing field appears.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is available: configure multiple agents, open Chat, click the plus/New Session control, and current main can only send `/new` in the active session; no visible UI path passes a selected `agentId` to `sessions.create`.

## Solution Assessment

Yes, with sequencing. The narrow maintainable solution is a Control UI addition over existing `agents.list` and `sessions.create`, while reusing or extending #73725 if it remains the maintainer-approved create/switch path.
