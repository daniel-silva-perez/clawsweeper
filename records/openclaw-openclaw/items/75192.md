## Summary

Keep open. Current main still lacks a WebChat/Control UI flow to create a new chat with a selected configured agent: the toolbar New session button only routes to the existing `/new` reset path, while the Gateway already supports `sessions.create` with `agentId`. The open related #73725 is an adjacent create/switch implementation candidate but is not merged and does not make selected-agent creation available on main.

## What This Changes

Add a WebChat UI flow that lets a user choose a configured agent when creating a new chat/session, then switches to that session.

## Best Possible Solution

Keep this issue open until the Control UI can create a dashboard session for a chosen configured agent using existing agents.list metadata and sessions.create(agentId), switch to the returned key, and refresh history/session/avatar/tool state without adding a new Gateway contract unless the UI exposes a concrete missing field.

## Reproduction Assessment

Yes. A source-level reproduction is clear: configure multiple agents, open the Control UI chat, and use the plus/New session control; current main only sends `/new` in the active session and has no UI path that passes a selected `agentId` to `sessions.create`.

## Solution Assessment

Yes, the requested UI-layer direction is the narrow maintainable solution, but it should be sequenced after #73725 or folded into that create/switch surface rather than adding a second divergent session-create path.
