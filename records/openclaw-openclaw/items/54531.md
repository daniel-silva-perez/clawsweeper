## Summary

Keep open. Current main has related route-preservation work, but it still intentionally blocks internal WebChat/dashboard turns from automatically routing replies to an external originating channel unless an explicit delivery route is set. The prior persisted-origin implementation in #57802 is closed and unmerged, so it is useful prior art rather than shipped behavior.

## What This Changes

The issue requests a configurable or automatic force-origin reply policy so agent responses are always delivered back to the messaging channel where the user asked, including Telegram, Discord, WhatsApp, and Web UI flows.

## Best Possible Solution

Keep this open for a maintainer-reviewed reply-routing design. The likely end state is either an explicit opt-in force-origin setting or an immutable first-external-origin route with stale-session and privacy safeguards, plus regression coverage for external-channel sessions while WebChat/dashboard is active and for explicit-delivery exceptions.
