## Summary

Keep open. Current main still blocks implicit WebChat/dashboard-originated turns from routing replies to a prior external originating channel unless an explicit delivery route is set, and the related persisted-origin PR was closed without merging. The remaining decision is not mechanical cleanup because changing the default could leak dashboard replies into stale or unintended external chats.

## What This Changes

The issue requests a configurable or automatic reply-routing behavior that forces agent responses back to the originating messaging channel such as Telegram, Discord, or WhatsApp when sessions are viewed from WebChat or dashboard surfaces.

## Best Possible Solution

Keep this open for a maintainer-reviewed reply-routing design: either an explicit opt-in force-origin setting or an immutable first-external-origin route with stale-session, account, thread, and privacy safeguards, plus docs and regression coverage for WebChat/dashboard sessions, channel-scoped sessions, and explicit-delivery exceptions.
