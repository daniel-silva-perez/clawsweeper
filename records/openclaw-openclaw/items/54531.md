## Summary

Keep this issue open. Current main has narrower route-preservation and explicit-delivery behavior, but it still does not implement the requested force-origin reply policy or immutable persisted-origin fields. WebChat/internal turns remain intentionally blocked from external origin routing unless an explicit delivery route is set, and the related #57802 PR was closed unmerged.

## Best Possible Solution

Keep this issue open for a maintainer-reviewed reply-routing design. The likely path is either a documented explicit force-origin setting or an immutable first-external-origin route with safeguards for stale shared sessions, plus regression tests covering Telegram/Discord/WhatsApp-origin sessions while WebChat/dashboard is active, explicit delivery, system-event completions, and stale route protection. #57802 is useful prior art but should be revived or replaced rather than treated as shipped.

- A fix needs to distinguish external user turns, WebChat/dashboard monitoring, explicit delivery, system-event completions, and stale shared-session route preservation.
