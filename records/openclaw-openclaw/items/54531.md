## Summary

Keep open. Current main still preserves the external-route metadata in some paths, but it deliberately blocks implicit WebChat/dashboard-originated turns from delivering to prior external chats unless an explicit delivery route is set; the related persisted-origin implementation is closed and unmerged.

## What This Changes

Request to make agent replies from Telegram, Discord, WhatsApp, and similar channel-originated sessions reliably deliver back to the originating channel even when WebChat or a dashboard is active.

## Best Possible Solution

Keep this open for a maintainer-reviewed reply-routing design: either an explicit opt-in force-origin setting or an immutable first-external-origin route with freshness/privacy safeguards, plus docs and regression coverage for external-channel sessions while WebChat/dashboard is active and for explicit-delivery exceptions.
