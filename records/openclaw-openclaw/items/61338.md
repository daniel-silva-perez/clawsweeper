## Summary

Keep open. Current main still compares skill OS requirements as raw platform strings, so a skill declaring `metadata.openclaw.os: ["macos"]` will not match macOS, where Node reports `process.platform === "darwin"`. The docs and tests cover `darwin`, but the reported `macos` alias path is still not implemented or validated with a clearer error.

## Best Possible Solution

Keep this issue open and fix the skills requirement platform contract in core. The likely maintainer path is to add a small canonical platform normalizer for skill `metadata.openclaw.os`, remote platform lists, and install-spec OS filters, covering at least `macos` -> `darwin` and preferably `windows` -> `win32`; then add regression tests for a `metadata.openclaw.os: ["macos"]` skill on Darwin and update the docs to state accepted aliases or only canonical values with validation guidance.
