## Summary

Current main still matches the reported bug: the before_tool_call plugin approval caller does not forward turn-source route metadata, while the gateway already supports those fields and fails closed with the reported no-route error when no route is present. This issue should remain open because #74150 is an open, unmerged implementation candidate that explicitly targets the same bug.

## What This Changes

The issue requests forwarding trusted origin-channel routing metadata from before_tool_call plugin approvals into plugin.approval.request so chat-channel approval prompts can be delivered.

## Best Possible Solution

Keep this issue open and review/land #74150 or an equivalent focused patch that adds trusted turn-source metadata to the before_tool_call hook context and forwards it to plugin.approval.request, with regression tests proving all route fields are forwarded and the no-route path still fails closed.
