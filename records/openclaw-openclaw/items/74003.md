## Summary

Current main still matches the reported before_tool_call plugin approval routing bug. The caller in `src/agents/pi-tools.before-tool-call.ts` still sends `plugin.approval.request` without turn-source routing metadata, while the gateway handler already accepts those fields and the shared approval path returns `decision: null` when no approval client, forwarder, or turn-source route exists. A referenced hclsys commit appears to implement the intended fix, but it is not contained in current main and no open PR referencing this issue was found.

## Best Possible Solution

Keep this issue open and land a narrow fix that forwards trusted turn-source metadata into `plugin.approval.request` for `before_tool_call` plugin approvals, matching exec approvals and preserving the current fail-closed no-route behavior. Add regression coverage for both forwarded same-chat metadata and absent-route callers.
