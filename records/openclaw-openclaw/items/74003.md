## Summary

Current main still matches the reported failure: the before_tool_call plugin approval caller does not carry or forward turn-source routing metadata, while the gateway already accepts those fields and fails closed with the reported no-route response when no route is present. This should stay open because #74150 is an open, unmerged implementation candidate with closing syntax for the same bug.

## What This Changes

Forward trusted originating channel, recipient, account, and thread metadata into before_tool_call plugin approval requests so plugin approval prompts can route back to Telegram and other channels.

## Best Possible Solution

Carry trusted turn-source metadata through the agent tool hook context and forward it to plugin.approval.request, with regression coverage for all route fields and for the no-route fail-closed behavior; reviewing and landing #74150 or an equivalent focused patch is the right end state.
