## Summary

Keep this PR open. Current main still preserves only the selected Node executable's own bin directory for daemon service PATH planning, while QMD defaults to resolving `qmd` from the gateway PATH; the PR remains a narrow implementation candidate with focused regression coverage and no unrelated dependency, workflow, lockfile, release, or secret-handling changes.

## What This Changes

This PR updates daemon install PATH planning so a selected Vite+-managed Node runtime also adds the matching `.vite-plus/bin` companion directory, and adds a regression test for that runtime shape.

## Best Possible Solution

Land this PR or an equivalent narrow fix that derives only the matching `.vite-plus/bin` companion directory from a selected `.vite-plus/js_runtime/node/.../bin/node` runtime, keeps non-Vite+ runtime behavior unchanged, and preserves focused regression coverage around daemon service PATH planning.
