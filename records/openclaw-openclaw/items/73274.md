## Summary

Keep open. Current main still lacks a durable transcript append helper on the public plugin runtime surface, while the issue and linked openclaw-plugin-cron-context workaround show the in-memory system-event path does not cover cross-session persistence. The request is real, but the API grants durable cross-session transcript authority and needs maintainer design before implementation.

## What This Changes

Expose a durable cross-session transcript append helper through the plugin runtime API so plugin lifecycle hooks can persist cron or announce output into another session's JSONL transcript.

## Best Possible Solution

Design a narrow additive runtime API around host-owned transcript writing: assistant-only by default, explicit trusted-plugin or operator opt-in, idempotency support, transcript update events, clear attribution, and docs/tests that distinguish permanent transcript writes from system events and next-turn injections. Maintainers should also decide whether isolated cron announce delivery should mirror into the target channel session without requiring a plugin workaround.

## Reproduction Assessment

Not applicable as a feature/API request. The high-confidence static check is that current main lacks api.runtime.agent.session transcript append support while the internal durable helper, ephemeral system-event path, and durable-but-once next-turn injection path are all present.

## Solution Assessment

Unclear. Exposing a persistent helper is plausible, but the proposed role-bearing arbitrary-session API is broader than the internal assistant-only delivery-mirror helper; a gated assistant-only mirror surface is the safer starting point.

## Risks / Open Questions

- A direct public helper with arbitrary session keys would let plugin code persist content into sessions that are not currently active.
- The proposed role option could let plugins forge user turns unless maintainers intentionally define attribution and role limits.
- The right product boundary may be a narrow assistant-only mirror helper, a trusted-plugin capability, or core cron announce mirroring rather than a general transcript write API.
