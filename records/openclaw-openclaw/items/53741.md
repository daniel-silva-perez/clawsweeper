## Summary

Keep open. Current main still lacks the central Discord voice behavior: `autoJoin` does not accept `sessionChannelId`, voice routing is still keyed to the voice channel, and voice turns still suppress text delivery. The PR is paired with open feature request #53562 and has unresolved functional/security review points plus possible overlap with broader design issue #73699, so this needs maintainer review rather than cleanup closure.

## What This Changes

This PR adds optional Discord voice `autoJoin.sessionChannelId` config/schema support, routes auto-joined voice sessions through a configured text channel, enables text delivery for routed voice replies, updates generated config baseline data, and adds Discord voice session-channel tests.

## Best Possible Solution

Land a revised narrow implementation if maintainers want this config option: keep the config/schema/generated-docs surface, document it in the Discord channel docs, validate a same-guild text-capable target with clean failure cleanup, pass an explicit Discord delivery target/run context, keep text delivery best-effort relative to voice playback, preserve routing during DAVE recovery, and cover those paths with focused Discord voice and delivery tests. If #73699 becomes the chosen product direction, supersede this PR with that broader design instead.
