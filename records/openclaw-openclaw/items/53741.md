## Summary

Keep open. Current main still lacks the requested Discord voice text-channel routing, and this contributor PR remains a plausible implementation path for #53562, but the patch needs maintainer revision before merge because the delivered-reply path, validation cleanup, guild-boundary check, and DAVE recovery behavior are not safe as written.

## What This Changes

The PR adds optional Discord voice `autoJoin.sessionChannelId` config/schema/generated-baseline support, routes auto-joined voice sessions through that text channel, enables delivered text replies for routed voice turns, and adds session-channel validation tests.

## Best Possible Solution

Land a revised narrow implementation only if maintainers want this config option: keep the config/schema/generated-docs surface, document the option in the Discord channel docs, validate a same-guild text-capable target with guaranteed cleanup on failure, pass an explicit Discord delivery target/run context, make text delivery best-effort relative to voice playback, preserve routing during DAVE recovery, and cover those paths with focused Discord voice and delivery tests. If #73699 becomes the chosen design, supersede this PR with that broader bridge instead.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Pass a Discord reply target when enabling delivery:** `extensions/discord/src/voice/manager.ts:666`
  - body: When `sessionChannelId` is set, the patch enables delivery but still omits `to`, `replyTo`, and `runContext.currentChannelId`. The delivery layer treats `messageChannel` as the source channel and has no recipient, so Discord target normalization fails before TTS playback can be queued.
  - confidence: 0.93
- **[P1] Destroy the joined connection on validation failure:** `extensions/discord/src/voice/manager.ts:420-428`
  - body: `sessionChannelId` validation happens after the voice connection reaches Ready. These new error returns leave that live connection undisposed and untracked by `this.sessions`, so later leave/status flows cannot clean it up.
  - confidence: 0.9
- **[P1] Require session channels to belong to the guild:** `extensions/discord/src/voice/manager.ts:425-428`
  - body: The same-guild check only rejects a mismatched `guildId`; channel objects without a guild id, such as DMs or group DMs, can fall through and become the routed session target. Reject non-guild targets explicitly and allow only messageable guild channels.
  - confidence: 0.86
- **[P2] Preserve routed sessions during DAVE recovery:** `extensions/discord/src/voice/manager.ts:788`
  - body: Decrypt recovery rejoins with only the voice `channelId`, so a session intentionally routed through `sessionChannelId` falls back to voice-channel routing after the first recovery and stops using the paired text channel.
  - confidence: 0.89
- **[P2] Make text delivery best effort for voice playback:** `extensions/discord/src/voice/manager.ts:666`
  - body: With `deliver` forced on, a transient Discord send failure can throw out of the agent command path before TTS is generated or queued. Voice playback should continue even when mirroring to text fails.
  - confidence: 0.82
