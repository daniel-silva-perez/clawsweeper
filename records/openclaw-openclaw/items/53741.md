## Summary

Keep open. Current main still lacks the requested Discord voice autoJoin text-channel routing behavior, and the supplied PR discussion shows this branch remains the active implementation candidate with unresolved review points rather than something to close as implemented or stale.

## What This Changes

This PR adds optional Discord voice `autoJoin.sessionChannelId` config/schema support, threads it through Discord voice join routing, enables delivered text replies for routed voice sessions, updates generated config baseline data, and adds session-channel validation tests.

## Best Possible Solution

Land a revised version if maintainers want the narrow autoJoin option: keep the config/schema/generated docs surface, add user-facing Discord docs, validate a same-guild text-capable target before committing to a live voice session or clean up on failure, pass an explicit Discord delivery target/run context, keep text delivery best-effort so TTS still plays, preserve routing across DAVE recovery rejoins, and cover those paths with focused Discord voice tests. If #73699 becomes the preferred product direction, supersede this PR with that broader design.
