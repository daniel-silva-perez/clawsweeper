## Summary

Keep this PR open. Current main still lacks the requested Discord voice autoJoin text-session routing, the paired feature issue #53562 is still open, and this PR remains the active implementation candidate but needs focused correctness/security review before merge.

## What This Changes

The PR adds optional Discord voice `autoJoin.sessionChannelId` config/schema/baseline support, threads it through `DiscordVoiceManager`, enables delivered text replies for routed voice sessions, and adds session-channel validation tests.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #53562, but require a focused revision before merge: add the config/schema/docs surface, pass `sessionChannelId` through autoJoin/join, validate the target as same-guild text-capable before or with cleanup after joining, pass an explicit Discord delivery target or run context when enabling delivery, make text delivery best-effort so TTS still plays, preserve the route across DAVE recovery rejoins, and add focused regression tests. Coordinate with #73699 if maintainers prefer the broader voice-as-text-session bridge over this narrow autoJoin option.
