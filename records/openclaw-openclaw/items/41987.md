## Summary

Keep open. The reported bug is still valid on current main, but this PR is stale against the current browser plugin architecture: it changes the old core browser-tool path while the active bundled browser tool still emits screenshots without passing image sanitization config.

## What This Changes

The PR threads a config option through the old core browser tool and adds tests so `browser:screenshot` passes `resolveImageSanitizationLimits` to `imageResultFromFile`.

## Best Possible Solution

Port the useful fix to the bundled browser plugin: pass the current runtime config from plugin tool context into the browser tool, apply `resolveImageSanitizationLimits()` to browser image results, and cover the plugin test path rather than the retired core browser tool files.
