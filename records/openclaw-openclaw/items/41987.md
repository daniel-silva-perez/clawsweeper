## Summary

Keep open. The reported bug is still present on current `main`: the active bundled browser plugin returns screenshot images without applying `agents.defaults.imageMaxDimensionPx`, while this PR modifies the removed legacy core browser tool path, so merging it as-is would not fix the shipped browser tool.

## What This Changes

The PR threads `config` into the legacy core browser tool and adds legacy tests so `browser:screenshot` passes `imageSanitization` from `agents.defaults.imageMaxDimensionPx` into screenshot image results.

## Best Possible Solution

Port the behavior into the active bundled browser plugin: pass the runtime config from `OpenClawPluginToolContext` through `extensions/browser/plugin-registration.ts` into the lazy/runtime browser tool, apply `resolveImageSanitizationLimits()` to screenshot image results, and cover the plugin path with focused tests.
