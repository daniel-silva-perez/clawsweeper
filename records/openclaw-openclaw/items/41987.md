## Summary

Keep open. The reported bug is still present on current main: the bundled browser plugin does not pass config or runtime config into its browser tool, and the screenshot branch still calls imageResultFromFile without imageSanitization. The PR is a valid bug-fix candidate, but its patch targets the old core browser-tool path, which is no longer present, so it needs to be ported to the current browser plugin architecture.

## Best Possible Solution

Port the fix to the current browser plugin: thread ctx.getRuntimeConfig or runtimeConfig from extensions/browser/plugin-registration.ts into createBrowserTool, compute imageSanitization from agents.defaults.imageMaxDimensionPx at execute time, pass it to imageResultFromFile for browser:screenshot, and add focused tests in extensions/browser/src/browser-tool.test.ts. Consider covering labeled snapshot image results in the same change or opening a narrow follow-up.
