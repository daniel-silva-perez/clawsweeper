## Summary

Keep this issue open. Current main still renders the chat `/status` and `session_status` paths through the shared status-card builder with no `session.statusCard`, `agents.defaults.statusCard`, JSON-only mode, hide flag, or documented equivalent. The related CLI JSON request is a separate `openclaw status --json` surface, and the privacy-focused #69445 was closed as superseded by this issue, so this remains the canonical tracker.

## Best Possible Solution

Keep this issue open until OpenClaw adds an explicit, documented status-card visibility control for chat `/status` and `session_status`, or a maintainer rejects the product direction. A complete implementation should preserve current rendering by default, add schema/types/help/docs for the chosen key, thread the setting through `buildStatusReply`, `buildStatusText`, `buildStatusMessage`, and `session_status`, and add focused tests for both chat-command and agent-tool paths while leaving CLI `openclaw status --json` as a separate diagnostics surface.
