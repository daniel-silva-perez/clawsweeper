## Summary

Keep open. Current main still does not implement the PR's central Dream Diary hardening: request-scoped fallbacks still write the first raw snippet/promotion, diary dedupe still keys on timestamp plus body, and daily dreaming block stripping still treats any heading as a boundary and only recognizes marker-started blocks. The PR is also not cleanly superseded by the related work: #70523 covers daily inline fallback pollution, while this PR has distinct Dream Diary fallback/dedupe/strip coverage and an unresolved review concern around italic diary lines.

## Best Possible Solution

Keep this PR open for normal maintainer review. The useful path is to revise the fingerprinting so only the timestamp line is ignored, coordinate the fallback/raw-dump boundary with #70523, and then land or close this PR based on whether the remaining Dream Diary dedupe and managed-block stripping changes are still desired.
