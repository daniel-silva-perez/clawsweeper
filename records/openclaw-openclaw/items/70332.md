## Summary

Current `main` still does not implement this PR's central Dream Diary hardening. Request-scoped fallback still writes the first raw snippet or promotion, diary dedupe still fingerprints timestamp plus body, append still inserts duplicate bodies, and managed daily dreaming stripping still starts from markers and treats any heading as a boundary. The related open PR #70523 covers only inline daily-memory fallback dump suppression for #70509 and does not supersede this PR's `DREAMS.md` fallback prose, body-only diary dedupe, append duplicate prevention, or heading-level managed-block stripping work.

## Best Possible Solution

Keep this PR open for normal maintainer review and revision. The best path is to coordinate with #70523 for fallback/raw-dump behavior, revise this PR so only timestamp lines are ignored in diary fingerprints while italic body prose remains part of the body, then land the narrow memory-core fixes for `DREAMS.md` fallback prose, body-level duplicate prevention, and heading-level managed-block stripping with focused memory-core tests plus the changed gate before merge.
