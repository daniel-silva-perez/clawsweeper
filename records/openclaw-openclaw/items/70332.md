## Summary

Keep open. Current main still lacks the Dream Diary hardening this branch targets, and the related open PR #70523 only covers the inline daily-memory fallback pollution slice, not diary fallback prose, body-only diary dedupe, duplicate append skipping, or heading-aware managed-block stripping. The branch is useful but should not merge as-is because its fingerprint change can collapse distinct diary entries that contain standalone italic body lines.

## What This Changes

The PR modifies memory-core dreaming narrative/phases code and tests to strip nested or legacy managed daily blocks, render request-scoped fallback snippets as diary prose, dedupe diary entries by body, and skip duplicate diary appends.

## Best Possible Solution

Land a revised branch or narrow replacement that preserves the useful diary hardening while fixing the fingerprint logic to ignore only the timestamp line, adding regression coverage for italic body lines, coordinating the fallback boundary with #70523, and validating the memory-core touched surface before merge.

## Reproduction Assessment

Yes. The PR diff itself gives a high-confidence reproduction: create two diary blocks with the same non-italic body but different standalone italic body lines, then run the patched fingerprint/dedupe path; those italic body lines are removed from the fingerprint and the entries can be treated as duplicates.

## Solution Assessment

No, not as currently written. The product direction is narrow and maintainable, but the fingerprint implementation should preserve italic body content while excluding only the date/timestamp line; the fallback overlap with #70523 also needs a clean landing plan.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

- **[P2] Preserve italic diary body lines in fingerprints:** `extensions/memory-core/src/dreaming-narrative.ts:392-393`
