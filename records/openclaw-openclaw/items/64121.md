## Summary

Current main still renders a combined chat session picker grouped by agent. The requested split into separate Agent and Agent Session controls is not implemented on desktop or mobile, and the related open agent-switcher issue is complementary rather than a complete superseding tracker.

## Best Possible Solution

Keep this issue open. Implement the split in the Control UI chat surfaces by adding a desktop and mobile Agent selector backed by agentsList, adding an Agent Session selector filtered from sessionsResult by parseAgentSessionKey(...).agentId, switching agent changes to the most recent session for that agent or buildAgentMainSessionKey({ agentId }) as fallback, and leaving Model/Thinking sessions.patch calls targeted at state.sessionKey. Add focused tests for agent filtering, agent-switch fallback, mobile parity, and preservation of session-bound Model/Thinking overrides. Coordinate with #68959, but do not close this issue unless a canonical item explicitly tracks the complete split-picker behavior.
