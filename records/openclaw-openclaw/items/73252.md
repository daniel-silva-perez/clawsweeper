## Summary

Keep this PR open. Current main still has the Mattermost draft-preview bug the PR targets, but the proposed patch is not merge-ready because it adds only object-form `streaming` schema support while current public Mattermost docs and related reports still use scalar `streaming: "partial"`, with no Mattermost doctor/compat path to migrate that documented shape.

## What This Changes

The PR adds a Mattermost `streaming` config object/type and uses `streaming.mode` to disable draft-preview callbacks and final preview editing when set to `off`, with schema and monitor regression tests.

## Best Possible Solution

Merge a Mattermost-owned fix that accepts the documented and canonical streaming shapes, resolves preview mode for top-level and per-account configs, disables draft preview cleanly for `off`, preserves explicit block-streaming fallback, and includes focused schema/runtime tests plus docs or doctor compatibility updates.

## Reproduction Assessment

Yes. Current-main source gives a high-confidence source-level reproduction: the strict Mattermost schema has no `streaming` key, and the inbound monitor unconditionally wires draft-preview updates and finalization.

## Solution Assessment

No. The implementation direction is right, but the merge-ready solution should either accept/migrate scalar `channels.mattermost.streaming` values or update the public docs and compatibility path so users are not left with a documented config shape that still fails.
