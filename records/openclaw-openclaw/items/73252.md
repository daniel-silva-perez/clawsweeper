## Summary

Keep this PR open. Current main still hardcodes Mattermost draft preview streaming and lacks a Mattermost `streaming` schema entry, so the PR is addressing a real remaining bug, but the proposed schema does not yet accept the scalar `channels.mattermost.streaming` shape that current Mattermost docs advertise.

## What This Changes

The PR adds a Mattermost `streaming` config object/type and uses `streaming.mode` to disable draft-preview callbacks and final preview editing when set to `off`, with schema and monitor regression tests.

## Best Possible Solution

Merge a Mattermost-owned fix that accepts the documented and canonical streaming shapes, resolves `streaming.mode` for top-level and per-account configs, disables draft preview cleanly for `off`, preserves explicit block-streaming fallback, and includes focused schema/runtime tests plus any required docs and changelog updates.

## Reproduction Assessment

Yes. Current-main source gives a high-confidence source-level reproduction: a strict Mattermost schema has no `streaming` key, and the inbound monitor unconditionally wires draft-preview updates and finalization.

## Solution Assessment

No. The implementation direction is right, but the merge-ready solution should either accept the documented scalar `channels.mattermost.streaming` value through schema/compat or update the public docs and migration path so users are not left with a still-rejected documented config.
