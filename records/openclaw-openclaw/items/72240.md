## Summary

Keep open. Current main has partial internal exec diagnostics, but the user-facing foreground exec result and process polling still do not expose enough structured termination metadata to distinguish timeout, manual cancellation, supervisor SIGKILL, or OS SIGKILL; the related open reports overlap but do not fully cover this macOS diagnostics request.

## What This Changes

The issue requests richer diagnostics for intermittent macOS exec SIGKILL failures, including termination metadata, preserved output tail, timeout context, and log/run correlation for broad state/log scans.

## Best Possible Solution

Add a focused exec diagnostics improvement that carries termination metadata through both foreground exec and background process results: exitSignal, failureKind, exitReason or terminationReason, timedOut/noOutputTimedOut, timeout limit, run/session id, useful output tail, and a diagnostics/log correlation point. Keep signal escalation and drain-timeout behavior in the broader supervisor issue unless maintainers explicitly fold it in.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Unclear: the issue describes a credible recurring macOS pattern, but not a deterministic command/environment that reliably SIGKILLs on current main. The missing user-facing metadata is still reproducible by inspecting and unit-testing the existing simulated SIGKILL/timeout paths.

## Solution Assessment

Is this the best way to solve the issue? Yes for the first step: enrich the exec/process result contract and tests without changing process-kill semantics. A broader SIGTERM/SIGKILL escalation change belongs in #66399 or a maintainer-approved combined PR.
