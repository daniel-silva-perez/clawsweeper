## Summary

Keep open. Current main still preserves more termination data internally than it exposes through foreground exec and process polling results, so the macOS SIGKILL diagnostics gap remains actionable. Related #66399 and open PR #68680 are adjacent, but neither is a merged/current-main fix for this user-facing exec/process diagnostics surface.

## What This Changes

The issue requests richer user-facing diagnostics when macOS exec/process commands are killed with SIGKILL, including timeout/cancellation/supervisor/OS cause metadata and bounded output context.

## Best Possible Solution

Add a focused diagnostics contract improvement that propagates termination metadata through foreground exec and background process results: exitSignal, failureKind, exitReason/terminationReason, noOutputTimedOut, timeout/no-output limits, run/session/log correlation, and bounded output tails. Preserve current kill/escalation policy in this repair, leaving graceful SIGTERM/SIGKILL behavior to #66399 unless maintainers choose to merge the scopes.

## Reproduction Assessment

Unclear: the report describes a credible recurring macOS pattern, but it lacks a deterministic current-main SIGKILL reproduction. The missing metadata is reproducible by inspecting and unit-testing the existing simulated signal, timeout, no-output-timeout, cancellation, foreground-result, and process.poll paths.

## Solution Assessment

Yes for the first repair step: enriching exec/process result metadata is the narrowest maintainable fix for the observable diagnostics gap. Broader process-kill escalation or broad-scan guardrails should remain separate unless maintainers intentionally expand the work.
