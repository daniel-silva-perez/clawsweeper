## Summary

Keep open. Current main still has only partial internal exec diagnostics: foreground exec failure details and process polling do not surface enough structured metadata to distinguish timeout, no-output timeout, manual cancellation, supervisor SIGKILL, or OS SIGKILL. The related open reports overlap with signal escalation and broad-search symptoms, but none is a merged or shipped fix for this macOS user-facing diagnostics gap.

## What This Changes

The issue requests richer diagnostics for intermittent macOS exec/process SIGKILL failures, including termination metadata, output tails, run/log correlation, and safer handling of broad scans over large OpenClaw state directories.

## Best Possible Solution

Add a focused diagnostics contract improvement that carries termination metadata through foreground exec and background process results without changing kill policy: exitSignal, failureKind, terminationReason/exitReason, timeout/no-output flags and limits, run/session/log correlation, and bounded stdout/stderr tails. Leave two-phase signal escalation or drain-timeout semantics to #66399 unless maintainers intentionally combine the work.

## Reproduction Assessment

Unclear: the issue describes a credible recurring macOS pattern, but it does not provide a deterministic command/environment that reliably SIGKILLs on current main. The missing user-facing metadata is reproducible by inspecting and unit-testing the existing simulated signal, timeout, no-output timeout, and cancellation result paths.

## Solution Assessment

Yes for the first repair step: enriching exec/process result metadata is the narrowest maintainable fix for the observable diagnostics gap. Broader SIGTERM/SIGKILL escalation changes should remain in the process-supervisor issue unless maintainers approve expanding this scope.
