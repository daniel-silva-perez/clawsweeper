## Summary

Keep this issue open. Current main has partial usage visibility in chat and Telegram can expose the shared `/usage` command, but the requested Telegram-visible breakdown by session/sub-agent with API/model-call counts is not implemented. The best path is to build this as a shared chat command enhancement backed by the existing session usage aggregation, then Telegram gets it through the existing native command surface.

## Best Possible Solution

Keep open and implement a shared chat-command enhancement, likely `/usage sessions` or `/usage subagents`, backed by `sessions.usage`. It should summarize the active session and child sub-agent sessions with tokens/cost where available plus model-call/tool-call counts, add command tests, and update usage/slash-command docs. Telegram should receive it through the existing native command registration path.

## Risks / Open Questions

- Scope needs a precise definition for “API call counts” because existing usage summaries expose assistant/model usage counts and tool-call counts, not a separately named provider API-call metric.
- There is partial overlap with `/usage cost`, `/status`, `/subagents list`, and the Control UI Usage tab, so implementation should extend one shared command surface instead of adding a Telegram-only duplicate.
