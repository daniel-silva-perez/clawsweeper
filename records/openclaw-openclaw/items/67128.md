## Summary

Keep open. Current main has a generic native `/usage` command and some sub-agent token visibility, but it does not yet provide the requested Telegram/chat breakdown by session or sub-agent with API/model-call counts; `/usage cost` only prints current session, today, and last-30-day totals.

## What This Changes

Request to expose Telegram-visible per-session or per-sub-agent usage statistics, such as API call counts and token consumption, via `/usage` or `/status`.

## Best Possible Solution

Add a compact, generic chat usage breakdown that works in Telegram through the existing command system, reusing session-log usage aggregation instead of building Telegram-specific accounting. The output should cover current session plus recent/active sub-agent sessions with tokens, cost when available, and a clear model-call count label.
