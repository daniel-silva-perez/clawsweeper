## Summary

Keep open. Current main has a Telegram-capable `/usage` command and sub-agent token snippets, but the requested Telegram-visible per-session/sub-agent breakdown with call counts is not implemented; the closest current surfaces are split between `/usage cost`, `/subagents list`, and the Gateway/Control UI usage API.

## What This Changes

The issue requests a Telegram `/usage` command or session stats panel that shows token consumption and API call counts broken down by session/sub-agent.

## Best Possible Solution

Add or review a compact, channel-safe usage summary built on the existing transcript-backed `sessions.usage`/`loadSessionCostSummary` data, preferably as a channel-agnostic slash command surface that Telegram can expose natively; it should show current session plus sub-agent/session rows with tokens, cost when available, and a clearly defined model-call count.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The current-main verification path is source/docs inspection: `/usage cost` only returns a compact session/today/30-day summary, while `/subagents list` shows token snippets but not the requested per-session/sub-agent call-count breakdown.

## Solution Assessment

Unclear. A Telegram-only panel is probably narrower than necessary because the command registry and usage APIs are channel-agnostic; the maintainable path is to reuse existing usage aggregation and expose a compact chat/native-command view, or adapt the related `/total_usage` PR if maintainers prefer that command name.
