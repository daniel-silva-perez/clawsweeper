## Summary

Keep open. Current main still rejects existing local `<qqmedia>` files outside the OpenClaw/QQ Bot media roots instead of mirroring them, and the discussion correctly flags that a broad auto-copy would risk arbitrary local-file disclosure.

## Best Possible Solution

Keep this open and design a QQ Bot-owned staging path: accept only files proven to be inside agent-scoped workspace/media allowlists, realpath-check them, enforce size/type limits, copy them into managed QQ Bot outbound media storage, and keep rejecting arbitrary outside-root or symlink-escape paths. Route `<qqmedia>` through that path and update QQ Bot tests plus `qqbot-media` skill guidance.
