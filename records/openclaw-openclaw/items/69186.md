Current main still does not implement the requested Control UI/WebChat completion sound. The issue should stay open because #73894 is an open, unmerged implementation candidate for the same browser-local response-completion cue work.

## What This Changes

Request an optional completion/success notification sound in Control UI/WebChat when an agent turn finishes, with settings such as enable/disable and possibly custom sound configuration.

## Best Possible Solution

Keep this issue open and route the fix through #73894 if that PR remains technically sound. The target shape is a focused, browser-local, default-off Control UI/WebChat response-completion cue for successful final events on the active run, with local settings and tests, without broadening gateway protocol, plugin hooks, channel delivery, or remote custom sound fetching.
