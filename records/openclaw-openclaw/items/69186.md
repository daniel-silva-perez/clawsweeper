## Summary

Current main does not implement the requested Control UI/WebChat completion sound. There is a post-turn plugin hook (`agent_end`) and generic browser Web Push subscribe/test plumbing, but neither provides a browser-local, optional audible cue when an active chat turn successfully reaches its final event.

## Best Possible Solution

Keep this open and implement a focused Control UI feature: a browser-local, default-off completion sound setting with a test button, playing a short generated tone when the active chat run in the current session reaches a successful final event. Do not broaden gateway protocol, channel delivery, plugin hooks, or remote custom sound URLs in the first pass.

## Clownfish Work Prompt

Implement optional Control UI/WebChat completion sound for agent turns.

Observable request: when the active chat run in the Control UI finishes successfully, users should be able to hear a short completion cue while the tab is in the background. The setting must be optional and default off.

Expected fix boundary:
- Add a browser-local `UiSettings` boolean such as `chatCompletionSoundEnabled`, persisted through `ui/src/ui/storage.ts` with backwards-compatible default `false`.
- Add a Notifications or Chat setting in `ui/src/ui/views/config.ts` with an enable toggle and a small test button.
- Add a tiny generated local Web Audio tone helper, likely in `ui/src/ui/chat/completion-sound.ts`, avoiding remote/custom sound URLs in this PR.
- Trigger playback only when the current session's active chat run transitions to successful `final` completion. Avoid duplicate playback when tool-output final events cause history reloads, and do not play for `aborted`, `error`, side-result terminal events, or unrelated background runs.
- Respect browser autoplay constraints: make the test/toggle path initialize or unlock audio from a user gesture where feasible, and fail silently if playback is blocked.
- Do not change gateway protocol, server Web Push behavior, channel delivery, plugin hook contracts, or `agent_end` semantics.
- Update `docs/web/control-ui.md` for the new setting. Add a user-facing changelog entry in the active version with `Thanks @canmaxice-maker` if the repo's changelog policy requires one for this feature.

Related refs: #69186, #59199.
