## Summary

Keep open. Current main still recognizes Feishu bot mentions by comparing the pushed mention `open_id` to the startup-resolved `botOpenId`; startup/probe state still stores only `botOpenId` and `botName`, so the reported app-scoped `open_id` mismatch is not fixed. The issue also has open PR #40782 with closing syntax for the same bug, so it should remain open while that PR is reviewed, refreshed, or replaced.

## Best Possible Solution

Keep this issue open and review, refresh, or replace PR #40782. The eventual Feishu-owned fix should verify the stable identity source with Feishu/Lark, store the needed per-account identity state in the Feishu monitor, update `checkBotMentioned()` plus mention stripping, mention-forward, and debounce paths consistently, add regression tests for app-scoped `open_id` mismatch, and include a user-facing changelog note if behavior changes.
