## Summary

Keep this issue open. Current main still wires WebChat code-block Copy directly to `navigator.clipboard.writeText` and silently ignores failures, so the Edge failure mode described by the report is not implemented away. The linked fallback PRs were closed unmerged, and current tests only assert that the button exists.

## Best Possible Solution

Keep the issue open and implement a shared UI clipboard helper for WebChat code-block Copy: try `navigator.clipboard.writeText`, handle missing or rejected Clipboard API calls, add an allowed fallback such as a temporary textarea plus `document.execCommand('copy')`, and add focused tests for the rejection/fallback path.
