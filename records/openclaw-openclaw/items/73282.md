## Summary

Keep this PR open. Current main does not implement the intended fix: the gateway agent handler still validates channel hints only with `isGatewayMessageChannel`, and current channel utilities only treat deliverable channels plus `webchat` as gateway channels. The PR is a focused implementation candidate for the still-open bug #73237, with no obvious security-sensitive diff surface in the provided file list.

## Best Possible Solution

Keep this PR open and review or land it, or land an equivalent main-branch fix. The implementation should accept `heartbeat`, `cron`, and `webhook` only as internal non-delivery agent channel hints, keep `isGatewayMessageChannel` semantics for actual delivery decisions, and include regression tests for both `channel` and `replyChannel` acceptance plus genuine unknown-channel rejection.
