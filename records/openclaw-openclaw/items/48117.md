## Summary

Keep open. Current main and latest release still route the Windows PowerShell installer through a plain `npm.cmd install -g ... --no-fund --no-audit` call and return failure on any nonzero npm exit, with no Windows TLS trust-store repair for `UNABLE_TO_VERIFY_LEAF_SIGNATURE`. The exact related PR #48416 is closed unmerged, while merged PR #51146 handled the adjacent Linux/nvm TLS startup path only.

## Best Possible Solution

Keep this issue open and implement a maintainer-reviewed Windows installer TLS recovery path in `scripts/install.ps1`. Prefer preserving strict verification by default and using a scoped CA trust repair such as npm `cafile` or `NODE_EXTRA_CA_CERTS` where feasible; only use a scoped `--strict-ssl=false` retry if maintainers explicitly accept that tradeoff. Update the installer docs and Windows smoke/release-check coverage or live proof before closing.
