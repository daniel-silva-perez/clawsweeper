## Summary

Keep open. Current main still does not handle the reported Windows installer TLS failure: the PowerShell installer runs one npm global install and returns a generic failure on any non-zero exit, without detecting `UNABLE_TO_VERIFY_LEAF_SIGNATURE`, setting a CA bundle/cafile, or retrying with a scoped TLS workaround. The related Windows PR that proposed a retry was closed unmerged, while the merged TLS work in #51146 addresses a different Linux/nvm runtime path.

## Best Possible Solution

Keep this issue open and implement a Windows installer fix in `scripts/install.ps1`: detect known npm TLS verification failures, preserve strict TLS by default, then either guide/set a trusted CA bundle or perform a clearly warned one-install scoped fallback. Add focused tests around the error classifier/retry path and update installer troubleshooting docs if user action is still required.
