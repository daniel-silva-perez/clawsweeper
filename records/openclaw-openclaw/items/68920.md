## Summary

Keep open. Current main still routes `/v1/chat/completions` through the normal agent ingress without an HTTP light-context or voice-mode prompt policy, and the related stable-session PR only addresses one session warm-cache subproblem.

## What This Changes

The issue requests a low-latency HTTP Chat Completions mode that can skip full framework context assembly and support stable sessions for realtime voice callers.

## Best Possible Solution

Keep this as the tracker for an explicit HTTP prompt/context policy that can intentionally reduce tools, skills, memory, bootstrap files, and system prompt sections for low-latency callers, with config/docs/tests and separate handling of the stable-session subproblem in #69060.

## Reproduction Assessment

Partial yes. A source-level reproduction is high-confidence: current main has no `x-openclaw-light-context` or equivalent HTTP config path, and `x-openclaw-session` is not parsed; the precise TTFB numbers require the reporter's model/workspace/runtime setup.

## Solution Assessment

Unclear. A bare header is not enough by itself; the maintainable solution should define safe HTTP prompt-policy semantics first, then wire existing `promptMode`/`bootstrapContextMode` seams and document the security boundary.
