## Clownfish Work Prompt

Fix the config validation numbering UX mismatch reported at https://github.com/openclaw/openclaw/issues/73971. Current Control UI config arrays label items one-based (`#1`, `#2`, ...), but config validation summaries and visible issue JSON expose raw zero-based paths such as `models.providers.<provider>.models.3.*`. Implement a narrow display-only fix so user-facing config validation messages align with the Control UI.
