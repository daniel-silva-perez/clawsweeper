## Summary

Current main still has the reported Telegram typing failure path. The generic typing callback wrapper has partial safeguards, but the auto-reply keepalive path can still rethrow through an error-unaware interval, and Telegram typing still uses the normal channel retry budget unless callers override it. The related fix PRs were closed unmerged, so this remains actionable.

## Best Possible Solution

Keep this issue open and land one focused reliability PR that bounds typing keepalive failures and makes Telegram typing cheap to fail, without changing normal Telegram message delivery retry semantics or the Telegram 401 `sendChatAction` backoff.
