## Summary

Keep this PR open. Current main at e583db63 still creates Feishu synthetic reaction IDs with `:reaction:<emoji>:<uuid>` suffixes, has no Feishu message-id normalizer, and still passes raw message IDs into external Feishu message/reaction/typing/media/pin/streaming-card API calls. The linked bug #34528 remains unresolved on main, and this PR is the active contributor implementation path rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open and review or land it as the canonical implementation path for #34528. The best fix is a Feishu-plugin-local message ID normalizer applied at every external Feishu API boundary, with focused tests for clean IDs, suffixed IDs, malformed partial suffixes, whitespace trimming, and UUID case handling; no core API change appears necessary.
