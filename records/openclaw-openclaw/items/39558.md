## Summary

Keep this PR open. Current main still generates Feishu synthetic reaction message IDs with `:reaction:<emoji>:<uuid>` suffixes and still passes raw message IDs into Feishu message, reaction, typing, media, streaming-card, and pin API calls. The refreshed PR branch is the active implementation candidate for the linked Feishu bug and has maintainer follow-up marking it merge-ready, so it should be reviewed or landed rather than closed.

## Best Possible Solution

Keep this PR open and review or land the refreshed branch as the canonical fix for the linked Feishu bug. The right product direction is a small Feishu-plugin-local normalizer applied only at Feishu API boundaries, with tests documenting valid suffix stripping and malformed/partial suffix passthrough, while preserving suffixed IDs for internal reaction identity.
