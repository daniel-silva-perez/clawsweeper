## Summary

Keep this PR open. Current main still generates Feishu synthetic reaction message IDs with the `:reaction:<emoji>:<uuid>` suffix and still passes caller-provided IDs directly into Feishu SDK message, reaction, typing, media, pin, and streaming-card API boundaries. The PR remains the active canonical contributor path for fixing open bug #34528, with recent maintainer context preserving this branch as the path to land.

## Best Possible Solution

Keep this PR open and review or land it as the canonical fix for #34528. The right product/code path is a Feishu-plugin-local `stripFeishuReactionSuffix()` helper applied at every Feishu API boundary while preserving suffixed synthetic IDs internally for dedupe/session identity, with focused tests for valid suffix stripping and malformed/partial suffix passthrough.
