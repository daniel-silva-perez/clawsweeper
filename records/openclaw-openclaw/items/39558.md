## Summary

Keep this PR open. Current main still creates Feishu reaction synthetic IDs with `:reaction:<emoji>:<uuid>` and still passes raw message IDs into Feishu message, reaction, typing, media, pin, and streaming-card API calls. The PR remains the active implementation candidate for open bug #34528, and the provided maintainer comment says the branch was repaired to preserve the contributor path. No unexplained security-sensitive changes were visible in the provided PR file list/diff.

## Best Possible Solution

Keep this PR open and review or land it as the canonical fix for #34528. The right implementation is a Feishu-plugin-local `stripFeishuReactionSuffix()` helper applied at every Feishu API boundary, while preserving suffixed synthetic IDs internally for dedupe/session identity, with focused tests for valid suffix stripping and malformed/partial suffix passthrough.
