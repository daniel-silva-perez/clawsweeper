## Summary

Keep this PR open. Current main still creates Feishu synthetic reaction message IDs with a `:reaction:<emoji>:<uuid>` suffix and still passes caller-provided message IDs directly into several Feishu SDK API paths. The PR is the current contributor implementation path for open bug #34528, with recent maintainer context saying the branch was refreshed to stay canonical.

## Best Possible Solution

Keep this PR open and review or land the refreshed branch as the canonical fix for #34528. The best code path is a Feishu-plugin-local `stripFeishuReactionSuffix()` helper applied at every Feishu API boundary, while preserving suffixed synthetic IDs internally for dedupe/session identity and keeping focused tests for valid suffix stripping plus malformed/partial suffix passthrough.
