## Summary

Keep this PR open. Current main still generates Feishu synthetic reaction message IDs with the reported `:reaction:<emoji>:<uuid>` suffix, has no `stripFeishuReactionSuffix` helper or equivalent normalizer, and still passes raw message IDs into Feishu message, reaction, typing, media, pin, and streaming-card API calls. The PR remains the active contributor path for open bug #34528; the recent ProjectClownfish repair and prior bot review context both support review/landing rather than cleanup closure.

## Best Possible Solution

Keep this PR open and review or land it as the canonical implementation path for #34528. The best fix remains a Feishu-plugin-local message ID normalizer applied at every external Feishu API boundary that accepts a message ID, while preserving suffixed synthetic IDs internally for dedupe/session identity and covering clean, suffixed, malformed, whitespace, and partial suffix cases with focused tests.
