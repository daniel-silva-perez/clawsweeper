## Summary

Keep this PR open. Current main still lacks both central behaviors proposed here: command delivery still decides “no reply” from raw upstream payload length before checking whether the outbound projection is deliverable, and Slack preview/progress turns with no delivered final still clear draft/history and return without the PR’s in-thread keepalive/status fallback. The branch is active, recently updated, and the review discussion shows the author addressed several real gating concerns, so it remains an implementation candidate rather than conservative cleanup.

## Best Possible Solution

Keep this PR open for maintainer review or a scoped rebase. The best path is to port the narrow fixes onto current main, preserve intentional silence/tool-sent success, add normalized-empty command fallback coverage, add Slack preview-only no-final fallback coverage, and split unrelated UI formatting churn out before merge. #45290 appears complementary from the provided context rather than a clear superseding implementation of the Slack no-final behavior.
