## Summary

Keep this PR open. Current main still lacks both central behaviors from the PR: command delivery can still no-op when raw payloads normalize to zero deliverable outbound payloads, and Slack preview/progress turns with no delivered final still clear state and return without the proposed in-thread keepalive/status fallback. The PR discussion shows active iteration on real review concerns, and related #45290 is open/complementary rather than a merged replacement for the Slack no-final path.

## Best Possible Solution

Keep this PR open for maintainer review or a scoped rebase. The best path is to extract the narrow normalized-empty command fallback and Slack preview-only no-final fallback onto current main, preserve intentional silence/tool-success semantics, keep Slack fallback messages in-thread, add focused regressions, and split unrelated UI formatting churn. If #45290 or another PR becomes the canonical replacement, close this PR only after that replacement merges and demonstrably covers both the command normalized-empty and Slack no-final fallback behaviors.
