## Summary

Keep open. Current main still lacks the two central behaviors proposed here: command delivery still checks raw payload presence rather than normalized deliverable output before deciding no-reply behavior, and Slack preview/progress turns with no delivered final still clear state and return without the proposed in-thread keepalive/status fallback. Related #45290 is still open and does not cover the Slack dispatch path, so it is complementary rather than a merged replacement.

## Best Possible Solution

Keep this PR open for maintainer review or a scoped rebase. The best path is to split out the narrow command normalized-empty fallback and Slack preview/progress no-final fallback, preserve intentional silence/tool-success semantics, keep Slack fallback messages threaded, add focused regressions, and separate unrelated UI formatting churn. Close this PR only after a merged replacement demonstrably covers both the command normalized-empty behavior and the Slack no-final fallback behavior.
