## Summary

Keep this PR open. Current main still lacks both central behaviors from the PR: command delivery can still silently no-op when non-empty raw payloads normalize to zero deliverable outbound payloads, and Slack preview/progress turns with no delivered final still clear state and return without the proposed in-thread keepalive/status fallback. The PR discussion is useful: automated review found real gating risks, the author iterated through several fixes, and the latest comment says the branch was rebased and the missing Slack sleep helper was added. That makes this an active implementation candidate, not something current main has made obsolete. The related open #45290 is not merged and does not clearly cover the Slack no-final fallback path.

## Best Possible Solution

Keep this PR open for maintainer review or a scoped rebase. The best path is to extract the narrow normalized-empty command fallback and Slack preview-only no-final fallback onto current main, preserve intentional silence/tool-success semantics, keep Slack fallback messages in-thread, add focused regressions, and split unrelated UI formatting churn before merge. If #45290 or another PR becomes the canonical replacement, only close this PR after that replacement is merged and demonstrably covers both the command normalized-empty and Slack no-final fallback behaviors.
