## Summary

Keep this PR open. Current main at 6d323ee73640afdf02ed9fa3aded13da23359b98 still lacks both central behaviors from the PR: command delivery can silently no-op when raw payloads normalize to zero deliverable outputs, and Slack preview/progress turns that end without any delivered final still clear state and return without an in-thread no-final fallback.

## Best Possible Solution

Keep this PR open for maintainer review or a scoped rebase. The best path is to port the narrow normalized-empty command fallback and Slack preview-only no-final fallback onto current main, keep intentional silence and tool-sent success paths silent, preserve same-thread Slack delivery, add focused regression coverage, and split unrelated UI formatting churn out before merge.
