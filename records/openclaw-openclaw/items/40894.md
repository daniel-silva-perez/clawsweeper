## Summary

Keep this PR open. Current main still does not implement the PR’s central media-staging request: the core media store/server and sandbox staging paths still use the hardcoded 5 MB `MEDIA_MAX_BYTES`, and the config contract still lacks `tools.media.maxBytes`. The PR remains a potentially useful implementation candidate for #40880, but it is bundled with unrelated transcript-hygiene work and a cross-agent workspace fix that is already present on main.

## Best Possible Solution

Keep this PR open as a useful but unsplittable implementation candidate. The best maintainer path is to rebase and split it: land the `tools.media.maxBytes` media store/server/sandbox staging fix for #40880 with schema/help/tests, drop the already-landed cross-agent workspace slice, and move consecutive assistant-error transcript hygiene into a separate focused PR with its own bug context and replay-history tests.
