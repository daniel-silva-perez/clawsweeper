## Summary

Keep this PR open. Current main still does not implement the PR title’s central #40880 request: the media store, media HTTP route, sandbox staging, and config contract still use the fixed 5 MB media cap and do not expose `tools.media.maxBytes`. The PR remains useful as an implementation candidate, but it should be rebased and split because it also carries unrelated transcript-hygiene work, an already-implemented cross-agent workspace fix, and secrets-baseline churn.

## Best Possible Solution

Keep the PR open as a useful candidate, but ask for a rebase and split: land the `tools.media.maxBytes` media store/server/sandbox staging fix for #40880 with schema/help/docs/tests, drop the already-landed cross-agent workspace slice, move consecutive assistant-error transcript hygiene into its own focused PR, and avoid or separately justify `.secrets.baseline` churn.
