## Summary

Current main still has the reported memory-core promotion gap. It filters some obvious dreaming artifacts and strips managed Light/REM blocks during daily ingestion, but promotion still writes the selected snippet verbatim to `MEMORY.md`, the detector still misses inline metadata-before-`Candidate:` shapes, rehydration does not reject ranges inside managed dreaming fences, and `minRecallCount` is enforced against aggregate signal count rather than actual recalls.

## Best Possible Solution

Keep this issue open until memory-core lands a tested fix or maintainers explicitly consolidate it with the broader distillation thread. The likely implementation path is to land or rework #68774's inline-metadata detector and dreaming-fence rehydration guard, then decide whether `minRecallCount` should mean actual recalls or documented aggregate signals, with regression tests for inline Candidate metadata, managed-fence rehydration, and `recalls=0` promotion cases.
