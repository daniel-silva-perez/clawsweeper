## Summary

Keep open. Current main still lacks an internal pre-send message hook bridge, so this PR is addressing a real gap tracked by #73542, but the proposed patch is not merge-ready because it can emit the new notification for cancelled sends and can report stale pre-rewrite content to internal hook subscribers.

## What This Changes

The PR adds a fire-and-forget internal `message:sending` hook emission in `src/infra/outbound/deliver.ts` immediately after the outbound `message_sending` plugin hook runs.

## Best Possible Solution

Land a revised outbound-delivery fix that exposes a documented pre-send internal message hook after plugin rewrites are applied and only for messages that will actually continue to delivery, with regression coverage for normal, rewritten, and cancelled sends plus clear docs for the supported event key.

## Reproduction Assessment

Yes. The current-main code path is directly reproducible by registering a user hook for the intended outbound pre-send event and sending a message: the plugin `message_sending` hook path runs, but no internal `message:sending` bridge exists in outbound delivery.

## Solution Assessment

No. Bridging this surface is the right direction, but this patch should not merge until it guards cancelled sends, reports the effective post-hook content, and resolves the internal event key/docs contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Skip the internal hook for cancelled sends:** `src/infra/outbound/deliver.ts:1032-1053`
  - body: The new `fireAndForgetHook` block runs before the `hookResult.cancelled` guard, so a plugin hook that cancels delivery still causes internal subscribers to hear that a message is being sent. Move the internal notification after the cancellation check or include `!hookResult.cancelled` in the guard.
  - confidence: 0.95
- **[P2] Send the rewritten payload content to internal hooks:** `src/infra/outbound/deliver.ts:1041`
  - body: This event is built from the outer `payloadSummary`, which was captured before `applyMessageSendingHook` can rewrite content. When a plugin returns replacement content, internal subscribers will receive the stale original text instead of the effective outbound text.
  - confidence: 0.93
- **[P2] Resolve the user hook event key mismatch:** `src/infra/outbound/deliver.ts:1036-1037`
  - body: `createInternalHookEvent("message", "sending", ...)` dispatches to `message` and `message:sending` listeners, while the reported setup registers `message_sending` and the loader stores event metadata strings verbatim. Without a compatibility alias or docs/test update for `message:sending`, this PR may still not fire for the reproduction it claims to fix.
  - confidence: 0.78
