## Summary

Keep this issue open. Current main still leaves the Feishu `write`/`append` failure actionable: those actions still depend on Feishu `document.convert`, public/manual setup guidance still omits the reporter-verified `docx:document.block:convert` permission, and the non-batch descendant mapper can still overwrite normalized `children` with the raw converted block value. The related fix PRs #64331 and #64336 were closed unmerged, so there is no shipped implementation to close against.

## Best Possible Solution

Keep this issue open and make a focused Feishu plugin cleanup: add `docx:document.block:convert` and the related document-writing scopes to the public Feishu setup docs and manual credential prompt, explicitly mention republishing the Feishu app after scope changes, add a clearer diagnostic for `document.convert` permission failures if feasible, fix `toDescendantBlock()` so normalized `children` cannot be overwritten, and add a narrow regression test for string-valued converted children.
