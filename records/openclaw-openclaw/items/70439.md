## Summary

Keep open. Current main still does not implement the requested Signal voice-note audio preflight: Signal preserves downloaded audio metadata for downstream media understanding, but mention gating still runs before attachment fetch/transcription and the Signal inbound context does not set `Transcript` or `MediaTranscribedIndexes`. Existing maintainer discussion also explicitly kept this open after the metadata-only Signal fix.

## Best Possible Solution

Keep this open and implement provider-backed Signal voice-note preflight. Fetch eligible audio attachments after Signal access checks but before requireMention skips, call the shared `transcribeFirstAudio` helper, use the transcript for mention/body/command handling, set `Transcript` and `MediaTranscribedIndexes`, preserve media metadata for downstream media understanding, and document Signal voice-note transcription. Do not add a shell `preflight_cmd` without a separate security-reviewed design.
