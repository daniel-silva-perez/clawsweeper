## Summary

Keep open. Current main does not contain the PR's microCompact pre-pass, analysis-scratchpad stripping, or transcript-reference behavior, so the work is not obsolete; however, the proposed patch is not merge-safe because the new summary-stripper can keep text from the analysis block instead of the final summary, and the branch also needs rebase/owner review against the newer compaction provider path.

## What This Changes

The PR adds a microCompact tool-result pruning pass, an analysis/scratchpad summary format with stripping, a full-transcript path suffix after compaction, and unit tests for the new helpers.

## Best Possible Solution

Rebase the branch onto current main, keep the microCompact pass scoped to summarizable messages, make summary extraction target the final summary block after analysis, and then have the compaction owners decide whether the transcript-path suffix is an acceptable product/security tradeoff before running the changed compaction gate.

## Reproduction Assessment

Yes. A focused input where the analysis text contains its own <summary>...</summary> pair before the real final summary deterministically exercises the parser bug, because the proposed regex takes the first match in the whole response.

## Solution Assessment

No, not as submitted. The overall direction may be useful, but the parser needs to select the intended final summary block and the branch needs to be reconciled with current main's provider-owned compaction path before it can be the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Extract the final summary block:** `src/agents/compaction-analysis-strip.ts:42-44`
