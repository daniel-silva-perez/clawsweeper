## Summary

Keep open. Current main does not contain the PR’s micro-compact or analysis-scratchpad implementation, and the linked contributor issue remains open, but the proposed patch is not safe to merge as-is because it prompts future agents to read raw transcript JSONL and its summary-tag parser can discard the real final summary.

## What This Changes

The PR adds microCompact and analysis-scratchpad helpers with tests, wires them into safeguard compaction, and appends a full-session transcript reference to compacted summaries.

## Best Possible Solution

A mergeable version should rebase onto current main, keep microCompact scoped to summarizable messages, use a robust final-summary extractor, avoid advertising raw transcript JSONL as the recovery path unless maintainers explicitly approve that security tradeoff, and add focused regression tests plus the required changelog entry.

## Reproduction Assessment

Yes for the blocking parser defect: pass `stripAnalysisBlock` a response where the `<analysis>` block mentions literal `<summary>...</summary>` before the real final `<summary>` section. The current first-match regex deterministically returns the analysis fragment instead of the final compaction summary.

## Solution Assessment

No. The microCompact direction is plausible, and the second commit fixed earlier ordering/path-construction feedback, but the raw transcript self-service path bypasses existing sanitization boundaries and the summary parser needs to target the intended final block.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Avoid prompting agents to read raw transcripts:** `src/agents/pi-hooks/compaction-safeguard.ts:943`
  - body: This suffix persists the exact JSONL path and tells future model context where to read the raw transcript. Current compaction code strips `toolResult.details` and runtime-context custom messages before LLM-facing summaries, while `sessions_history` is the bounded sanitized recall path; this new prompt can reintroduce hidden or credential-like raw transcript data through a later tool read. Keep recovery behind a sanitized recall/export path or require explicit maintainer approval for the raw path.
  - confidence: 0.82
- **[P2] Match the final summary block when stripping analysis:** `src/agents/compaction-analysis-strip.ts:39`
  - body: `text.match` extracts the first `<summary>...</summary>` anywhere in the response. If the model mentions or quotes summary tags inside `<analysis>` before writing the real final section, compaction keeps that analysis fragment and drops the actual summary. Target the summary after `</analysis>` or the last summary block, and add a regression test for this shape.
  - confidence: 0.9
- **[P3] Add the required changelog entry:** `src/agents/pi-hooks/compaction-safeguard.ts:685-689`
  - body: This changes user-visible compaction behavior, but the PR does not update `CHANGELOG.md`. OpenClaw policy requires user-facing feat/fix/perf changes to include a changelog entry with valid contributor credit, so add an active-version entry for the compaction improvement.
  - confidence: 0.76

## Security Review

Status: needs_attention

Summary: The patch introduces a concrete transcript-data exposure concern by steering future agents toward raw session JSONL instead of sanitized recall.

Concerns:

- **[high] Raw transcript path bypasses sanitized recall:** `src/agents/pi-hooks/compaction-safeguard.ts:943`
  - body: Appending the full JSONL path to persisted compaction summaries can cause later model/tool turns to read raw transcript entries that current compaction explicitly strips before LLM-facing summaries, including `toolResult.details` and runtime-context custom messages. Use a sanitized session-history mechanism or require explicit maintainer approval for this recovery path.
