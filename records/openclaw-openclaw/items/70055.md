## Summary

Current main no longer uses the old auth-store external CLI sync path that produced info-level "synced ... credentials from external cli" output. Commit a001b5343ff0499ff06aef0779fc21f06b0ce2c7 moved external CLI credentials to runtime-only auth overlays, removed the store-level sync calls, and shipped in the stable v2026.4.20 release; current main and v2026.4.26 retain that behavior.

## Best Possible Solution

Keep the shipped runtime-only external auth overlay behavior, which removes the old info-level sync noise while preserving CLI bootstrap compatibility. If maintainers still want a hard config/env provider opt-out for performance or policy reasons, track that as a narrower follow-up against the current overlay path.
