## Summary

Keep open. Current main has useful partial fixes for the reported noise, including scoped external CLI discovery and no external auth overlay in `models list`, but it still does not provide the requested user-facing config or environment opt-out for external CLI auth-profile discovery.

## What This Changes

The issue requests a user-facing way to disable or narrow external CLI credential sync for auth profiles, such as config, environment variable, or lower-noise logging.

## Best Possible Solution

Add a documented, default-on external CLI auth discovery policy under auth config, with an environment override if desired, that can disable all external CLI overlays or restrict them by provider/profile while preserving the existing scoped discovery behavior and runtime-only persistence model.

## Reproduction Assessment

Yes for the remaining feature request: current-main source and docs show no user-facing config or environment opt-out. The exact old `models list` info-log reproduction is no longer a high-confidence repro because that command now avoids external overlays and the old `synced ... from external cli` info message is absent.

## Solution Assessment

No: the current scoped discovery is a good partial fix, but it is not the requested first-class opt-out. A narrow config/env gate threaded into the existing scope and overlay path would be the maintainable follow-up.
