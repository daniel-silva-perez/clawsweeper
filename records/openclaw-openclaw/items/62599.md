## Summary

Keep open. Current main partially mitigates the original plain text status path by skipping memory inspection there, but the remaining status memory inspection path still resolves the selected memory plugin locally and calls the manager's vector probe/status in the CLI process. Also, #62643 is an open, same-author PR with closing syntax for this issue, so this issue should remain open until that implementation candidate is reviewed, merged, or explicitly superseded.

## Best Possible Solution

Keep this issue open and review #62643 or a stronger replacement. The durable fix should make status memory inspection either use the live gateway runtime or enforce a passive/read-only status contract that prevents third-party memory plugin startup side effects and false local vector state, with a targeted regression test for custom memory plugins.
