## Summary

Keep this PR open. Current main still generates the Windows config opener as `Start-Process -LiteralPath ...`, while PowerShell's `Start-Process` exposes `FilePath` and not `LiteralPath`. The PR is a narrow two-file fix aligned with linked bug report #69933 and does not add security-sensitive changes.

## Best Possible Solution

Keep this PR open for maintainer review and merge, or land the same minimal fix: change the Windows branch of `resolveConfigOpenCommand()` from `Start-Process -LiteralPath` to `Start-Process -FilePath`, preserve the existing single-quote escaping, update `src/gateway/server-methods/config.test.ts`, and run the targeted gateway config test before merge.
