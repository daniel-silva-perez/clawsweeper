## Summary

Keep this PR open. The docs direction is useful and matches a real Feishu/Lark mixed-proxy gap, but the proposed PowerShell examples can be wrong for installed gateway services because transient `$env:` values are not propagated by `openclaw gateway restart`.

## What This Changes

The PR adds Feishu/Lark troubleshooting guidance to `docs/channels/feishu.md` for using `NO_PROXY` when provider traffic needs a proxy but Feishu/Lark endpoints should go direct.

## Best Possible Solution

Revise the troubleshooting section so it distinguishes the launch environment clearly: use `NO_PROXY` for gateways started with those environment variables, avoid implying that a one-off PowerShell `$env:` plus `gateway restart` changes an installed service, and keep #72595 open for any future channel-level direct/ambient proxy policy.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable for the docs addition itself, but the review finding has a high-confidence static reproduction path: compare the proposed `$env:` + `gateway restart` snippet with the current managed-service env builder and restart path, which do not inject those transient values into the gateway service.

## Solution Assessment

Is this the best way to solve the issue? No as written: documenting `NO_PROXY` is a good narrow solution, but the examples should first explain how the gateway actually receives those env vars for manual versus managed launches.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Clarify how the proxy env reaches the gateway:** `docs/channels/feishu.md:235`
  - body: The examples set `$env:HTTPS_PROXY`, `$env:HTTP_PROXY`, and `$env:NO_PROXY` in the current PowerShell session and then run `openclaw gateway restart`. For an installed gateway service, OpenClaw deliberately does not persist HTTP_PROXY/HTTPS_PROXY/NO_PROXY in the service environment, and `gateway restart` restarts the existing service or signals the running process rather than relaunching it with the CLI's transient `$env:` values. Users following this snippet can see no routing change; please distinguish manual `openclaw gateway run` from persistent service launch or wrapper environment before telling them to restart.
  - confidence: 0.87
