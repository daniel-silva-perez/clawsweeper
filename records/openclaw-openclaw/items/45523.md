## Summary

Keep this PR open. Current main still does not implement the PR’s intended gateway startup dangerous-flag warnings for `gateway.auth.mode="none"` or trusted-proxy auth with empty/absent `allowUsers`, and the schema still accepts whitespace-only `trustedProxy.userHeader`. The PR remains a focused security hardening change, but it is currently dirty against main and needs a rebase into the refactored dangerous-flag collector plus targeted tests.

## Best Possible Solution

Keep this PR open as a focused gateway hardening candidate. Rebase it onto current main’s dangerous-flag collector, add collector and startup-log coverage for `gateway.auth.mode="none"` plus trusted-proxy allow-all states where both absent and empty `allowUsers` are treated the same, decide whether schema-level whitespace-only `userHeader` rejection should supplement the existing runtime guard, and add targeted regression tests before merge.
