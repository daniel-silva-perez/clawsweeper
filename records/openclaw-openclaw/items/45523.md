## Summary

Keep this PR open. Current main still does not implement the PR’s central request: gateway startup dangerous-config flags do not include `gateway.auth.mode="none"` or trusted-proxy allow-all `allowUsers` states. The related security audit paths recognize the risk, but the startup warning collector and its tests still omit these auth cases. The PR remains a focused gateway hardening candidate, though it needs a rebase into the current split collector shape, targeted tests, and generated schema follow-through if the `userHeader` schema refinement is kept.

## Best Possible Solution

Keep this PR open as a focused gateway hardening candidate. Rebase it onto the current split dangerous-config collector, add tests for collector output and gateway startup warnings for `gateway.auth.mode="none"` plus trusted-proxy omitted/empty `allowUsers`, and either include generated schema follow-through for whitespace-only `userHeader` rejection or split that schema hardening into a separate targeted change.
