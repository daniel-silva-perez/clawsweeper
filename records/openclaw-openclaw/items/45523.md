## Summary

Keep this PR open. Current main still does not implement the PR’s intended startup dangerous-flag warnings for `gateway.auth.mode="none"` or trusted-proxy auth with empty/absent `allowUsers`, and the config schema still accepts whitespace-only `trustedProxy.userHeader`. The security audit path already warns on related gateway auth risks, but gateway startup uses the dangerous-flag collector, which lacks these gateway auth cases. The provided PR diff is focused and security-relevant, but it needs a rebase into the current refactored collector shape plus targeted tests.

## Best Possible Solution

Keep this PR open and review it as a focused gateway hardening candidate. Rebase it onto the current dangerous-flag collector, add startup dangerous flags for `gateway.auth.mode="none"` and trusted-proxy allowlists where both `undefined` and `[]` are treated as allow-all, decide whether schema-level whitespace-only `userHeader` rejection should supplement the existing runtime guard, and add targeted tests for the collector, startup log, and config schema behavior before merge.
