## Summary

Keep open. Current main still has a context-overflow and compaction-failure path that rotates the session and returns the reported hard-reset message; the newer mitigation only replays a bounded user/assistant transcript tail and does not inject an explicit reset note, task goal, or progress recovery context. The open cross-reference #63216 also shows the hard-reset recovery loop is still an active area rather than a closed, shipped fix.

## What This Changes

The issue asks OpenClaw to carry minimal recovery context into the new session after an unrecoverable context-overflow reset so long-running tasks can resume instead of starting blank.

## Best Possible Solution

Keep the reset safety valve, but make the rotated session agent-visible and resumable: add a bounded recovery handoff that records the reset reason, the last user instruction, and safe recent assistant-visible progress without replaying tool/system/compaction records or expanding the prompt enough to re-trigger overflow.
