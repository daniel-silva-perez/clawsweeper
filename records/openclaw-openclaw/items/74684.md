## Summary

Current main still has the reported ACP process-correlation gap, and an open implementation PR already targets this issue. The issue should stay open until maintainers review, land, or replace that PR rather than being closed during cleanup.

## What This Changes

Expose host process metadata, such as ACP adapter or child-agent PIDs, for ACP sessions spawned through `sessions_spawn(runtime="acp")` and related status/control surfaces.

## Best Possible Solution

Define an additive ACP process-metadata contract, have ACPX populate adapter and child-agent process identifiers or explicit unavailability when possible, persist the metadata with the ACP session, and expose it through `sessions_spawn` plus status/control surfaces with focused regression coverage.
