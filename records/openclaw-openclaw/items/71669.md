## Summary

Keep open. Current main still rejects the described remote plain-HTTP Control UI path and has UI/docs guidance, but it does not emit the requested gateway startup warning. The open PR #73163 now targets this issue with closing syntax, so this issue should stay open until that PR lands or maintainers choose a different predicate.

## What This Changes

Request a gateway startup warning when Control UI is enabled in a remote plain-HTTP setup that will reject device-less browser connections.

## Best Possible Solution

Land a narrow gateway startup-log change, likely through #73163 after review, that warns only when the resolved bind/TLS/Control UI/device-auth configuration makes remote plain-HTTP Control UI connections likely to fail. Preserve the current authentication policy and keep coverage in the startup-log tests.
