## Summary

Keep this PR open. Current main still routes bare `openclaw` directly to Crestodian through an argv-only policy, and the current docs, unit tests, and Docker smoke expectation still describe or assert Crestodian for no-command starts. The PR remains a focused contributor implementation candidate for changing fresh or empty installs to classic onboarding first.

## Best Possible Solution

Keep this PR open for maintainer review. If maintainers accept the UX change, the best path is the PR’s scoped routing change: inspect the active config snapshot before the existing bare-root Crestodian branch, send missing or valid-empty configs to classic onboarding, preserve Crestodian for configured installs and invalid configs, and keep the paired tests, Docker smoke update, and docs updates.
