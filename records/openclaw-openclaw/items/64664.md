## Summary

Keep open. Current main still keeps approval records in fresh in-memory managers, so a gateway restart drops pending IDs; the gateway now includes structured remediation details, but the user-facing /approve path and Telegram callback path still surface or rethrow the raw `unknown or expired approval id` failure. The narrow related fix PR #67982 and Telegram callback PR #64406 are both closed unmerged, and the only currently open approval-related PR found is for followup warning logs, not this stale-button UX.

## What This Changes

The issue asks OpenClaw to handle stale Telegram approval callbacks after a gateway restart with clearer recovery guidance, and optionally to persist pending approvals across restarts.

## Best Possible Solution

Implement a narrow UX fix that translates approval-not-found failures into actionable re-request guidance in both the generic /approve reply path and the Telegram inline callback path, with focused regression tests. Leave disk-backed pending approval persistence for a separate maintainer product/storage decision.
