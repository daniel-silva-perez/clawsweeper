## Summary

Keep open. Current main still lacks the Gmail watcher excluded-label fix, and this PR is now cleanly rebased, but it still has one correctness gap: its documented empty-list override omits the flag, leaving upstream gog's default SPAM/TRASH exclusions active.

## What This Changes

The PR adds `hooks.gmail.excludeLabels` config/runtime plumbing, passes default Gmail watcher excluded labels to `gog gmail watch serve`, and updates docs, generated config metadata, and Gmail tests.

## Best Possible Solution

Land one canonical fix that restores default SPAM/TRASH/DRAFT/SENT watcher exclusions, preserves explicit empty-list opt-out at the gog CLI boundary, keeps schema/docs/tests/generated metadata aligned, and then closes the linked Gmail regression.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main resolves the Gmail runtime config and builds watcher serve args without any `--exclude-labels`, while upstream gog defaults an omitted flag to SPAM/TRASH.

## Solution Assessment

No, not as-is. The configurable runtime/config approach is the right narrow fix, but the builder and test need to preserve the documented empty-list opt-out or the docs/help must stop promising that behavior.
