## Summary

Keep open. Current main registers Telegram bot command menus and supports extra custom menu entries, but it does not support language-specific Telegram command descriptions or Chinese overrides for native commands like `/help` and `/tts`. The existing custom command path cannot override native commands, and `setMyCommands` is called without a `language_code` option.

## Best Possible Solution

Keep this issue open and implement it in the bundled Telegram plugin/config surface. A good fix would add a typed, validated locale-aware command-description override model, apply it to native/custom/plugin menu entries, and use grammY/Telegram `setMyCommands` options such as `language_code` while preserving current default-language behavior and command-limit handling.
