## Summary

Keep this issue open. Current main still exposes the reported failure class: Feishu wiki IDs are schema-declared as strings, but the plugin forwards received runtime values without rejecting numeric IDs after OpenAI-compatible tool-call JSON parsing can already have rounded unsafe integer literals.

## What This Changes

The issue reports that Feishu/Lark knowledge-base `space_id` values get rounded or altered when passed into Feishu wiki document tools.

## Best Possible Solution

Feishu wiki tools should treat wiki space identifiers as opaque strings end to end, reject numeric `space_id` and `target_space_id` values with a clear precision-loss message before any Lark API call, strengthen schema and skill wording so models quote IDs, and cover the behavior with focused Feishu tool tests.
