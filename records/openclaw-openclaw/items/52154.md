## Summary

Keep this PR open. Current main still does not implement `commands.shellProfile` or an equivalent custom shell-profile loader for exec commands, and this PR remains the visible implementation candidate for the linked feature request. The PR also changes command execution behavior by sourcing user-selected files, so it needs maintainer product/security review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review or rework. The best path is a deliberately scoped exec-profile configuration surface, documented in public exec/config docs, wired through the intended exec surfaces, and tested for PowerShell/POSIX command composition plus approval, allowlist, PTY, gateway, sandbox/node, and chat bash behavior.
