## Summary

Keep this PR open. Current main still does not implement `commands.shellProfile` or an equivalent per-command shell profile loading path, and the linked feature issue points back to this PR rather than to a shipped replacement. The PR is a plausible implementation candidate, but it touches the command execution surface and needs maintainer product/security review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review or rework. The best path is an explicit validated config surface for command profile loading, wired through every intended exec surface, documented with exact gateway/sandbox/node scope, and guarded so profile-defined aliases/functions cannot bypass approval or allowlist enforcement.
