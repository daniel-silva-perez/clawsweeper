## Summary

Keep open. Current main still does not implement `commands.shellProfile` or an equivalent per-command shell profile loading mechanism, and this PR remains the visible implementation candidate for the linked feature request. It also changes the command execution surface by sourcing user-selected files before exec commands, so it needs maintainer product/security review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review or rework. The best path is a deliberately scoped, validated exec-profile configuration surface, wired through all intended exec paths and docs, with tests for PowerShell/POSIX command composition, PTY and approval-mediated gateway execution, sandbox/node behavior, and explicit guards so profile-defined aliases/functions cannot bypass approval or allowlist enforcement.
