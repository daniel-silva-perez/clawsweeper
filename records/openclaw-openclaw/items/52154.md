## Summary

Keep open. Current main still does not implement `commands.shellProfile` or an equivalent per-command shell profile loader, and this PR remains the visible implementation candidate for the linked feature request. It also changes the command execution surface, so it needs maintainer product/security review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review or rework. The best path is a deliberately scoped exec-profile configuration surface, documented in public exec/config docs, wired through all intended exec paths, and tested for PowerShell/POSIX command composition plus direct, PTY, gateway approval, sandbox/node, allowlist, and chat bash behavior.
