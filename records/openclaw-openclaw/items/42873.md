## Summary

Keep this PR open. Current main still does not implement the PR’s intended normal auto-reply behavior: trailing own-line `NO_REPLY` is stripped, the preceding text remains deliverable, and the existing regression test still asserts that old behavior. The PR is also paired with same-author follow-up #46626 and overlaps open #69394, so maintainer coordination is still needed rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review or author follow-up. The best path is a rebased shared trailing/own-line silent-token detector in `src/auto-reply/tokens.ts` and `src/auto-reply/reply/normalize-reply.ts`, with tests that flip the current own-line trailing `NO_REPLY` expectation while preserving inline stripping, JSON-envelope suppression, glued-leading-token handling, case-insensitivity, media/channelData behavior, and scoped cron/outbound behavior. Coordinate with #46626 and #69394 before closing or landing duplicates.
