## Summary

Keep this PR open. Current main still does not implement the PR's intended normal auto-reply behavior: `normalizeReplyPayload()` strips a trailing own-line `NO_REPLY` and returns the preceding text as deliverable content, and the current colocated regression test still asserts that behavior. The PR remains a plausible implementation candidate, but it needs rebase/maintainer review against newer silent-token hardening and overlapping open work (#46626 and #69394).

## Best Possible Solution

Keep this PR open for maintainer review or author follow-up. The best path is a rebased shared silent-token detector, or equivalent trailing/own-line guard, in `src/auto-reply/tokens.ts` and `src/auto-reply/reply/normalize-reply.ts`, with tests flipping the current own-line trailing `NO_REPLY` expectation while preserving inline stripping, JSON envelope suppression, glued-leading-token handling, case-insensitivity, media/channelData behavior, direct silent-reply policy, outbound delivery behavior, and cron-specific suppression. Coordinate with #46626 and #69394 before closing duplicates.
