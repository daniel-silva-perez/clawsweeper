## Summary

Keep this PR open. Current main still strips a trailing own-line `NO_REPLY` and returns the preceding text from normal auto-reply normalization, and the current regression test still asserts that behavior. The PR is also paired with same-author follow-up #46626 and overlaps open #69394, so this needs maintainer coordination rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review or author follow-up. The best path is a rebased shared trailing/own-line silent-token detector in `src/auto-reply/tokens.ts` and `src/auto-reply/reply/normalize-reply.ts`, with tests that flip the current own-line trailing `NO_REPLY` expectation while preserving inline stripping, JSON-envelope suppression, glued-leading-token handling, case-insensitivity, media/channelData behavior, and scoped cron/outbound behavior. Coordinate with #46626 and #69394 before landing or closing duplicates.
