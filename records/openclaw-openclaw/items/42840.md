## Summary

Keep open. Current main still renders Control UI chat through markdown-it plus DOMPurify without a MathJax/KaTeX stage, and the related open PR #67103 is the active implementation candidate rather than a merged fix or a reason to queue duplicate automated work.

## What This Changes

Add MathJax/LaTeX rendering support to the Control UI chat so inline and display formulas render as equations instead of raw source text.

## Best Possible Solution

Add one sanitizer-aware math-rendering layer in the shared Control UI markdown renderer, with focused tests for inline math, display math, invalid formulas, code spans/fences, ordinary dollar amounts, sanitizer output, and normal markdown compatibility; then close this issue after PR #67103 or an equivalent implementation merges.

Yes. A high-confidence static reproduction exists: a chat message containing `$E=mc^2$` reaches `toSanitizedMarkdownHtml`, which currently renders markdown and sanitizes HTML without any MathJax/KaTeX processing stage.

## Solution Assessment

Unclear for the exact MathJax/config proposal. The narrow maintainable path is a shared frontend renderer change with sanitizer-aware tests, and maintainers should review, revise, land, or close PR #67103 before starting replacement work.
