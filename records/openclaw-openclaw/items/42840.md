## Summary

Keep open. Current main still renders Control UI chat through the generic markdown-it plus DOMPurify path with no MathJax, KaTeX, or equivalent math-rendering dependency, and the active KaTeX implementation candidate remains open as PR #67103. The earlier PR #61510 was closed unmerged, while related duplicate feature issues #56689 and #62118 are already closed, so this issue should remain the canonical tracker until a focused implementation lands.

## What This Changes

Request LaTeX/MathJax-style math rendering in the Control UI chat so inline and display equations render as formulas instead of raw delimiter text.

## Best Possible Solution

Land one focused Control UI renderer change, likely in the shared markdown path, with tests for inline math, display math, invalid formulas, code spans/fences, ordinary dollar amounts, sanitizer output, and normal markdown compatibility; then close this issue after that PR or an equivalent implementation merges.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: Control UI chat routes message text through toSanitizedMarkdownHtml, which currently only runs markdown-it and DOMPurify, while ui/package.json lacks any math-rendering dependency, so a message like `$E=mc^2$` remains ordinary text.

## Solution Assessment

Unclear for the exact MathJax/config proposal, but the narrow maintainable direction is a frontend math-rendering layer with sanitizer-aware tests. Open PR #67103 is the current implementation candidate, so the right next step is normal review of that PR rather than a separate automated repair branch.
