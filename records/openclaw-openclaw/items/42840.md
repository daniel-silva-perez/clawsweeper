## Summary

Keep open. Current main still has no Control UI math-rendering path, and open PR #67103 is the active implementation candidate rather than a merged fix or a reason to queue duplicate automated work.

## What This Changes

Add MathJax/LaTeX rendering support to Control UI chat so formulas display as rendered equations instead of raw source text.

## Best Possible Solution

Add one sanitizer-aware math-rendering layer in the shared Control UI markdown renderer, with focused tests for inline math, display math, invalid formulas, code spans/fences, ordinary dollar amounts, sanitizer output, and normal markdown compatibility; then close this issue after that PR or an equivalent implementation merges.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: a chat message containing `$E=mc^2$` flows through toSanitizedMarkdownHtml and then into chat rendering without any MathJax/KaTeX stage on current main.

## Solution Assessment

Unclear for the exact MathJax/config proposal. The maintainable path is a shared frontend renderer change with sanitizer-aware tests, and maintainers should review, land, revise, or close PR #67103 before starting replacement work.
