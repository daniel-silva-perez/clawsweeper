## Summary

Keep open. Current main still lacks Control UI math rendering, duplicate feature issues are already closed, and open PR #67103 is the active implementation candidate rather than a merged fix.

## What This Changes

Request LaTeX/MathJax-style formula rendering in Control UI chat so inline and display formulas render instead of appearing as raw source.

## Best Possible Solution

Add one sanitizer-aware frontend math-rendering layer in the shared Control UI markdown renderer, with focused tests for inline math, display math, invalid formulas, code spans/fences, ordinary dollar amounts, sanitizer output, and normal markdown compatibility; then close this issue after that PR or an equivalent implementation merges.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: Control UI chat message text flows through toSanitizedMarkdownHtml, which currently only runs markdown-it and DOMPurify, so input like `$E=mc^2$` has no math-rendering stage.

## Solution Assessment

Unclear for the exact MathJax/config proposal. The narrow maintainable path is a shared frontend renderer change with sanitizer-aware tests, and open PR #67103 is the current implementation candidate to review before creating any replacement work.
