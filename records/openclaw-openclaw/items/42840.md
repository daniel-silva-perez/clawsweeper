## Summary

Keep open. Current main still has no MathJax/KaTeX rendering path in the Control UI markdown renderer, and the related open PR #67103 is the active implementation candidate rather than a merged fix or a reason to start duplicate automation.

## What This Changes

Add MathJax/LaTeX rendering support to Control UI chat so inline and display formulas render as equations instead of raw source text.

## Best Possible Solution

Add one sanitizer-aware math-rendering layer in the shared Control UI markdown renderer, with focused tests for inline math, display math, invalid formulas, code spans/fences, ordinary dollar amounts, sanitizer output, and normal markdown compatibility; then close this issue after PR #67103 or an equivalent implementation merges.
