## Summary

Keep open. Current main still lacks stable behavioral capability descriptors for tools across the core tool shape, plugin tool metadata, before_tool_call payload/dispatch, tools.catalog/tools.effective protocol schemas, and effective inventory projection. The related #48503 remains complementary open design context rather than a merged implementation or superseding tracker.

## What This Changes

Add optional behavioral capability descriptors to tool definitions so policy plugins can classify tool reads, writes, external sends, privileged execution, and autonomous actions without brittle tool-name lists.

## Best Possible Solution

Design an additive, versioned Plugin SDK and Gateway protocol contract for behavioral tool capability descriptors, let built-in and plugin tools declare it optionally, project it through tools.catalog and tools.effective, and surface it to trusted policies plus before_tool_call payloads with focused contract tests, generated API checks, and public docs.
