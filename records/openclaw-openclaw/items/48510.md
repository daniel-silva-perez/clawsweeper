## Summary

Keep this issue open. Current main has adjacent display/policy metadata for tools (`displayName`, `description`, `risk`, `tags`) plus working `before_tool_call` enforcement, but it does not implement the requested stable behavioral capability descriptors on tool definitions, does not expose those descriptors through `tools.catalog` or `tools.effective`, and does not pass them to tool-call policy hooks. Related #48503 remains open and complementary, not a merged superseding fix.

## Best Possible Solution

Keep this issue open for maintainer design. The best path is an additive, versioned plugin SDK/tool contract that defines a small capability descriptor type, allows core and plugin tools to declare it, projects it through `tools.catalog` and `tools.effective`, and makes it available to `before_tool_call` and trusted tool policies. Coordinate the hook payload and provenance parts with #48503, then add focused SDK/protocol/docs/tests around the new contract.
