## Summary

Keep open. Current main still lacks stable behavioral tool capability descriptors across the core tool type, plugin tool metadata, before_tool_call/trusted-policy payloads, tools.catalog/tools.effective schemas, and effective inventory projection; the related design item remains complementary rather than a merged superseding implementation.

## What This Changes

Requested optional behavioral capability descriptors on tool definitions so policy plugins can reason about tool behavior without brittle tool-name matching.

## Best Possible Solution

Design an additive, versioned Plugin SDK and Gateway protocol contract for behavioral tool capability descriptors, allow built-in and plugin tools to declare it optionally, project it through tools.catalog and tools.effective, surface it to trusted policies and before_tool_call payloads, and update API baselines, contract tests, docs, and changelog together.
