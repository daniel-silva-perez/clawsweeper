Keep open. Current main still exposes the reported Kimi Coding path and has no merged targeted fix or regression test for repeated identical tool calls after a tool-result turn; the proposed root cause is plausible but incomplete because plain text-only final answers are already left unchanged.

## What This Changes

The issue requests fixing Kimi Coding so `kimi-code` stops repeating the same tool calls after tool results and produces a final `stop` response.

## Best Possible Solution

Keep the issue open until a current Kimi Coding transcript proves where the repeated call is introduced, then fix the owning Kimi path with a regression that covers tool-result replay and final-answer convergence. If the model legitimately re-emits identical tagged calls, the maintainable end state is likely a provider-specific normalization or bounded loop guard rather than a blanket stop-reason rewrite.
