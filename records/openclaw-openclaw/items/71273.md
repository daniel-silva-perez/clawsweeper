## What This Changes

The issue reports that the bundled Kimi Coding provider can repeat identical tool calls after receiving tool results instead of producing a final assistant answer.

Keep this open until a current Kimi Coding transcript or replay fixture shows whether Kimi is re-emitting identical tagged calls, OpenClaw is duplicating stale calls during normalization/replay, or another provider wrapper step is responsible; then fix the Kimi-owned path with focused regression coverage.
