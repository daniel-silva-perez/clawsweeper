Current main still has the active Kimi Coding stream wrapper and no targeted merged fix for the reported repeated tool-call loop, but the issue's proposed stop-reason root cause is not proven: current source only rewrites `stop` after tagged Kimi tool-call markup is parsed, and tests already cover plain final assistant text staying unchanged.

## What This Changes

The issue reports that Kimi Code via the bundled Kimi provider repeats identical tool calls after tool results instead of producing a final assistant answer.

## Best Possible Solution

Keep the issue open until a current Kimi Coding transcript or replay fixture captures the assistant response immediately after tool results; then fix the owning Kimi provider stream/replay path with focused regression coverage for repeated identical calls.

## Reproduction Assessment

Unclear: the issue gives concrete steps and repeated-log evidence, but there is no current replay fixture or live transcript in this review proving the first assistant response after tool results.
