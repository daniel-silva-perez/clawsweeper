## Summary

Current main still does not implement this PR's requested per-call `onComplete="notify"` exec API. Main has default background `notifyOnExit` behavior, but `onComplete` is absent from the schema, handler, docs, tests, and prompt guidance; successful empty-output completions are still suppressed unless explicitly configured. The PR remains potentially useful, but its head still has unresolved review concerns around session routing, node-host behavior, input validation, and overlap with #46926.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to update the existing PR so onComplete is a narrow validated option, the no-poll hint is only shown when completion routing is actually available or is paired with a fallback warning, host=node is explicitly supported or rejected, focused tests cover those cases, docs/schema surfaces are updated if the API is accepted, and overlap with #46926 is reconciled before merge.
