## Summary

Keep open. Current main still lacks the proposed per-call `onComplete` exec API, and the PR remains an open implementation that needs maintainer API/product review plus fixes for review-identified contract gaps before it can be merged.

## What This Changes

This PR adds an `onComplete="notify"` exec option that auto-backgrounds commands, forces completion notifications, and threads explicit completion state through bash process sessions.

## Best Possible Solution

Maintain a single exec completion-wake design: either land a revised per-call API or the default-notification direction from #46926, with literal validation, node-host semantics, docs, and focused tests covering routed and unrouted notification behavior.
