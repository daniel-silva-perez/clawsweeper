## Summary

Keep open for maintainer review. Current main still does not expose or handle an `onComplete` exec parameter, and the PR’s proposed API is not already implemented elsewhere. The branch is useful but still has unresolved review blockers around notification routing, `host=node` behavior, schema validation, and missing docs/tests.

## What This Changes

The PR adds an `onComplete="notify"` exec option that auto-backgrounds a command, forces completion notifications, adds no-poll response text, and threads explicit completion state through bash process sessions.

## Best Possible Solution

Keep this open until maintainers decide the exec completion-wake API. A mergeable version should validate `onComplete` as a narrow literal or reject unknown values, only promise no-poll notifications when routing can actually deliver them, explicitly implement or reject `host=node`, update `https://docs.openclaw.ai/tools/exec`, add focused coverage, and reconcile the overlap with #46926 before landing.
