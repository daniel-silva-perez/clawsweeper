## Summary

Keep open. Current main still arms the TUI streaming watchdog only from chat deltas; tool events do not refresh it, and the reconnect path reloads history without clearing the stale active run/watchdog state. The linked fix PR #69125 was closed unmerged, and #69978 is a separate watchdog UX/config follow-up rather than a fix for this bug.

## Best Possible Solution

Keep this issue open. The best fix is a small TUI change that treats active-run agent tool/lifecycle events as proof-of-life before verbose filtering, plus a reconnect reset that clears `activeChatRunId` and the streaming watchdog before `loadHistory()`. Add focused regression coverage in `src/tui/tui-event-handlers.test.ts` and a reconnect-path test around `src/tui/tui.ts` or its session/event seams.
