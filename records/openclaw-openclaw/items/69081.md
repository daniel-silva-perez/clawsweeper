## Summary

Keep open. Current main still refreshes the TUI streaming watchdog from chat delta events, not from active tool events, and the reconnect path reloads history without clearing stale active-run/watchdog state. The matching fix PR #69125 is closed unmerged, while #69978 is a separate watchdog UX/config request.

## Best Possible Solution

Keep this issue open. The focused fix should update `src/tui/tui-event-handlers.ts` so active-run tool, and likely lifecycle, events refresh the watchdog before verbose filtering, and update `src/tui/tui.ts` or the handler context so reconnect clears stale `activeChatRunId` plus the watchdog before history reload. Add regression coverage for active tool-event proof-of-life and reconnect stale-state recovery.
