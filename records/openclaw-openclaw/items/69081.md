## Summary

Current main still has the reported TUI desync paths. The streaming watchdog is armed only from chat delta handling, active-run tool/lifecycle events do not refresh it, and reconnect reloads history without clearing stale active run/watchdog state. The direct fix PR #69125 was closed unmerged, while #69978 and #69026 are related watchdog UX/config follow-ups rather than fixes for both reported bugs.

## Best Possible Solution

Keep this issue open and land a narrow TUI fix. The likely fix is to treat active-run tool and lifecycle agent events as proof-of-life before verbose filtering, rearming the streaming watchdog for the active run, and to handle reconnect by clearing stale active-run/watchdog state before history reload while allowing genuinely in-flight runs to re-establish themselves from subsequent events. Add focused regression tests for quiet tool events, lifecycle proof-of-life, and reconnect recovery; keep #69026/#69978 as separate watchdog UX/config work.

## Repair Work Prompt

Fix the TUI streaming status desync reported in openclaw/openclaw#69081. Current main still arms the streaming watchdog only from chat delta events, so active tool/lifecycle events do not keep the run alive, and reconnect reloads history without clearing stale active run/watchdog state.

Scope:
- In `src/tui/tui-event-handlers.ts`, treat active-run agent tool events as proof-of-life and rearm the streaming watchdog before verbose filtering or early returns. Quiet/default tool events must still refresh liveness even when no tool card is rendered.
- Also handle active-run lifecycle events as proof-of-life where appropriate without letting non-active/background runs steal the activity state.
- Add or expose a narrow reconnect reset seam so `src/tui/tui.ts` can clear stale `activeChatRunId` and pending watchdog state before `loadHistory()` on reconnect. Preserve legitimately in-flight runs by allowing subsequent chat/agent events to re-establish the active run; do not add gateway replay or broad protocol changes in this PR.

If reconnect is hard to test at `runTui` level, factor a small testable helper rather than broad UI refactoring.
- Do not implement #69026 or #69978 watchdog timeout/message product changes here.
- Add a single-line user-facing changelog fix entry in the active version section, crediting `Thanks @EenvoudJasper`.
