## Summary

Keep open. Current main still has the sibling-path legacy settings fallback, and this PR is relevant but not merge-ready because the current head still preserves unflagged legacy gateway URLs on first upgraded load and is behind newer Control UI browser-local storage behavior on main.

## What This Changes

This PR adds Control UI storage logic and tests for gateway URL derivation so path-prefixed reverse-proxy instances do not reuse a cached WebSocket URL from a sibling path.

## Best Possible Solution

Keep this PR open as the active implementation candidate, but update it before merge. The best path is to rebase onto current main and revise ui/src/ui/storage.ts plus ui/src/ui/storage.node.test.ts so page-derived defaults are distinguishable from explicit user overrides, unflagged legacy records are handled or explicitly documented, same-host custom gateway paths and cross-host overrides are preserved, gateway tokens remain scoped by normalized gateway URL, and current browser-local theme/identity fields are retained. Because this is a user-visible Control UI fix, add a changelog entry when landing.
