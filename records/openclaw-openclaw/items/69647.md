## Summary

Keep open. Current main still has the stale Control UI gateway URL fallback described by this PR: `loadSettings()` reads the current path-scoped key, then the default key, then the shared legacy key, and it preserves a legacy sibling-path `gatewayUrl` unless it exactly matches the current page-derived URL. The PR is narrow and aimed at the right storage surface, but the latest branch still needs maintainer review because review comments identify an unresolved legacy-upgrade case for unflagged stored gateway URLs.

## Best Possible Solution

Keep this PR open and review it as the active implementation candidate. The best fix belongs in `ui/src/ui/storage.ts` with focused tests in `ui/src/ui/storage.node.test.ts`: distinguish page-derived defaults from explicit user overrides, handle or deliberately document unflagged legacy records, preserve same-host custom gateway paths such as `/ws`, keep cross-host overrides, and keep tokens isolated per normalized gateway. Because this is a user-visible Control UI bug fix, a changelog entry should be considered before merge.
