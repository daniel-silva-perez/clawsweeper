## Summary

Keep open. Current main still has the stale sibling-path gateway fallback, so this PR is not obsolete or implemented. The PR is relevant, but the current PR head still preserves unflagged legacy gateway URLs on first upgraded load and appears behind newer Control UI storage fields on main, so it needs a focused update/rebase before merge.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The best fix belongs in ui/src/ui/storage.ts with regression coverage in ui/src/ui/storage.node.test.ts: distinguish page-derived defaults from explicit gateway overrides, handle or explicitly document unflagged legacy records, preserve same-host custom gateway paths, keep cross-host overrides, keep tokens isolated per normalized gateway, and preserve current main's browser-local settings fields. Because this is a user-visible Control UI bug fix, maintainers should consider a changelog entry when landing it.
