## Summary

Keep this PR open. Current main still lacks the central Slack `file_shared` fallback: there is no `file_shared` Bolt event registration, handler, setup manifest subscription, docs entry, source type, or focused test path in the current `extensions/slack` plugin. Adjacent Slack attachment, file-id download, and `app_mention` race handling work exists on main, but it does not replace the large-file Socket Mode fallback this PR is trying to add.

## Best Possible Solution

Keep this PR open as an implementation candidate, but require a rebase or replacement port into `extensions/slack`: add and register a `file_shared` handler under `extensions/slack/src/monitor/events`, hydrate the matching message with an event-timestamp-anchored bounded `conversations.history` lookup, dedupe Slack retries and normal `message`/`file_share` delivery, update `extensions/slack/src/setup-shared.ts` and https://docs.openclaw.ai/channels/slack for the required bot event, and add focused Slack plugin tests while preserving existing media/security protections.
