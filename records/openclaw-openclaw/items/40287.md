## Summary

Keep this PR open. Current main still does not implement the central Slack `file_shared` fallback: the Slack plugin has no `file_shared` event registration, handler source path, setup manifest entry, docs entry, or focused tests. Some adjacent Slack file/attachment and `app_mention` work exists on main, but it does not replace this PR’s intended large-file Socket Mode fallback, and the PR would need to be ported from the removed `src/slack` layout into the current `extensions/slack` plugin.

## Best Possible Solution

Keep this PR open as an implementation candidate, but require a rebase or replacement port into `extensions/slack`: add and register a `file_shared` handler under `extensions/slack/src/monitor/events`, hydrate the matching message with an `event_ts`-anchored bounded `conversations.history` lookup, dedupe Slack retries and normal `message`/`file_share` delivery, update `extensions/slack/src/setup-shared.ts` and https://docs.openclaw.ai/channels/slack for the `file_shared` bot event, and add focused Slack plugin tests for large-file, multi-file, duplicate-delivery, busy-channel, and `app_mention` paths while preserving existing media/security protections.
