## Summary

Keep this PR open. Current main still lacks the central Slack `file_shared` fallback this PR is trying to add: there is no `file_shared` Bolt event registration, no Slack setup/docs subscription, no message-handler source path for that event, and no focused test coverage. The PR is still an implementation candidate, but it cannot merge as-is because it targets the old `src/slack` tree while current main uses `extensions/slack`.

## Best Possible Solution

Keep this PR open as an implementation candidate, but require a rebase or replacement port into `extensions/slack`: add and register a `file_shared` handler, hydrate the matching message with a timestamp-anchored bounded `conversations.history` lookup, dedupe Slack retries and normal `message`/`file_share` delivery, update Slack setup/docs for the required bot event, and add focused Slack plugin tests with the existing media/security protections intact.
