## Summary

Keep this issue open. Current main has the underlying pieces for the pattern: local model configuration, sub-agent model overrides, sub-agent run timeouts, tool-policy narrowing, sandbox guards, and documentation about local model tool-call limitations. It does not yet provide the requested first-class cookbook/docs section that ties those pieces into a cloud-orchestrator plus local text-only worker architecture with sample prompting and config.

## Best Possible Solution

Add a concise docs page or cookbook section, likely linked from docs/gateway/local-models.md and docs/tools/subagents.md, showing a cloud primary/orchestrator model with local text-only sub-agent workers. Include sample config for agents.defaults.model, agents.defaults.subagents.model, local provider metadata with input:["text"] and compat.supportsTools:false when needed, tools.subagents.tools narrowing, sessions_spawn runTimeoutSeconds, sandbox:"require", and example task wording that tells the local worker to return plain text rather than attempt autonomous tool use.
