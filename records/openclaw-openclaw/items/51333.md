## Summary

Keep this issue open. Current main has the underlying pieces for the requested architecture: local model setup, hosted-primary/local-fallback examples, sub-agent model overrides, sub-agent run timeouts, sandbox gating, tool-policy narrowing, and local tool-call limitation docs. It still does not provide the requested first-class cookbook/docs section tying those pieces into a cloud-orchestrator plus local text-only worker pattern with sample config and worker prompting guidance.

## Best Possible Solution

Add a concise cookbook section, likely linked from `docs/gateway/local-models.md` and `docs/tools/subagents.md`, for a cloud primary/orchestrator with local text-only sub-agent workers. Include sample config for a hosted primary model, local provider metadata with `input: ["text"]` and `compat.supportsTools: false` when needed, `agents.defaults.subagents.model`, `runTimeoutSeconds`, sandbox `require`, safe tool-policy narrowing, and example task wording that asks the local worker to return plain text rather than attempting autonomous tool use.
