## Summary

Keep this PR open. Current main still matches the reported bug: main-session cron systemEvent jobs enqueue the raw event with a plain `cron:` context key, heartbeat prompt construction peeks that same queue and wraps cron-tagged events in `buildCronEventPrompt`, and the reply path can drain the same event into `System:` lines. The PR is still a focused implementation candidate for the linked bug, not obsolete or implemented on main.

## Best Possible Solution

Review and land this PR, or apply an equivalent focused fix on main that prevents `sessionTarget="main"` plus `payload.kind="systemEvent"` cron jobs from being wrapped a second time by heartbeat prompt construction. Preserve the existing cron reminder wrapper for cron events that are not already injected into the active main-session transcript, and keep regression coverage for `wakeMode="now"` and no-delivery/internal cron behavior.
