## What This Changes

The PR tags main-session cron systemEvent queue entries with a `:main-system-event` context-key suffix, filters those entries out of cron reminder prompt construction, and adds heartbeat regression coverage.

## Best Possible Solution

Update this PR or apply an equivalent focused fix on main that marks main-session cron `payload.kind="systemEvent"` entries as already injected, excludes only those entries from the heartbeat cron wrapper, preserves the wrapper for normal cron reminders, and updates the current cron timer seam expectations plus heartbeat regression coverage.
