## Summary

Keep open. Current main still lacks Brave Goggles support, so the contribution is not obsolete, but the branch is stale against the provider-owned web-search architecture and should not merge as-is. The prior review discussion is useful: the original old-surface implementation handled schema, forwarding, and cache concerns, but the same feature now needs to be ported into the active Brave plugin.

## What This Changes

The PR adds a Brave-only optional `goggles` parameter to `web_search`, forwards it to Brave web and llm-context requests, separates cache entries, and updates tests, docs, and changelog.

## Best Possible Solution

Land a rebased provider-owned port in `extensions/brave` that exposes per-query `goggles`, forwards the exact value to both Brave web and llm-context endpoints, separates cache entries without lowercasing or sentinel collisions, updates `docs/tools/brave-search.md` and `docs/tools/web.md`, and carries a credited changelog entry.

Yes. A static current-main trace reproduces the gap: `web_search` delegates to provider definitions, and the active Brave provider has no `goggles` schema field, URL forwarding, or cache-key component.

## Solution Assessment

No. The capability is a good fit, but this PR is no longer the narrow maintainable patch because it changes stale core and old docs surfaces instead of the active Brave plugin-owned contract and runtime.
