## Summary

Keep this PR open. Current main still does not expose Brave Goggles through `web_search`; the active implementation has moved to the provider-owned Brave plugin, so the useful PR work needs to be rebased or ported rather than closed as implemented.

## Best Possible Solution

Keep this PR open and ask for a rebase or port into the active `extensions/brave` provider. The implementation should add optional Goggles support to the Brave provider schema and shared schema, read it in `executeBraveSearch`, include it in cache keys without lossy normalization or missing-value collisions, append it to both Brave web and llm-context requests, update `docs/tools/brave-search.md` and `docs/tools/web.md`, and add focused tests for schema exposure, both URL builders, cache partitioning, and the chosen string/list behavior.
