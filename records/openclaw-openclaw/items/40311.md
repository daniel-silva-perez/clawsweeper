## Summary

Keep open. The Brave Goggles feature is valid and useful, but this PR is stale against current main: it wires the option into the old core web-search implementation while active Brave web_search behavior now lives in the Brave provider plugin. Current main still lacks a `goggles` schema field, URL forwarding, cache-key handling, tests, and public docs for the active Brave provider.

## What This Changes

The PR adds a Brave-only `goggles` parameter to the old core `web_search` schema/runtime, forwards it to Brave web and llm-context URL builders, updates cache behavior and tests, and edits Brave/web docs plus the changelog.

## Best Possible Solution

Land a rebased port that exposes a Brave-only per-query `goggles` option from the Brave plugin provider, forwards it to both current Brave endpoints, includes it in cache keys, and updates the public provider docs, overview docs, tests, and credited changelog while leaving non-Brave providers unchanged.
