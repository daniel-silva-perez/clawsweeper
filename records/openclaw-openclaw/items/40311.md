## Summary

Keep open. Current main still does not expose a Brave Goggles parameter in the active provider-owned Brave web_search implementation, so this PR remains a useful feature candidate. The current branch appears stale because its implementation targets the older core web-search path rather than the current Brave plugin boundary.

## Best Possible Solution

Keep this PR open as a useful feature candidate, but request a rebase/port into the active Brave plugin. Add optional goggles support in extensions/brave schema/shared schema/runtime, pass it through both Brave web and llm-context URL builders, include it in the cache key without lossy normalization or missing-value collisions, update https://docs.openclaw.ai/tools/brave-search and https://docs.openclaw.ai/tools/web, and add focused Brave provider tests. No new core/plugin API appears necessary.
