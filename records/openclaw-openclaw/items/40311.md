## Summary

Keep open. The feature direction is valid, and official Brave docs confirm Goggles support for LLM Context plus the `goggles` query parameter for Web Search, but current `main` still lacks the parameter in the active Brave provider schema, URL builders, cache key, tests, and public docs. This PR also targets stale core and legacy docs surfaces, so it needs a provider-owned port before merge. ([api-dashboard.search.brave.com](https://api-dashboard.search.brave.com/documentation/services/llm-context))

## What This Changes

The PR adds an optional Brave-only `goggles` parameter to `web_search`, forwards it to Brave web and llm-context requests, separates cache entries, and updates tests, docs, and changelog.

## Best Possible Solution

Land a rebased provider-owned implementation in `extensions/brave` that exposes per-query `goggles`, forwards the exact value to both Brave web and llm-context endpoints, includes it in cache identity, documents it on the current Brave and web tool pages, and carries a credited changelog entry.

## Reproduction Assessment

Yes. A static current-main trace reproduces the gap: `web_search` delegates to provider definitions, and the active Brave provider has no `goggles` schema field, forwarding path, or cache-key component.

## Solution Assessment

No. The requested capability is appropriate, but this PR is not currently the narrowest maintainable patch because it changes stale core and legacy docs surfaces instead of the active Brave plugin-owned provider.
