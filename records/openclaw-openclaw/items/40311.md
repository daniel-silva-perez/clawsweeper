## Summary

Keep open. The feature is valid, and Brave’s current API/docs support Goggles for the relevant endpoints, but this branch targets the stale core web-search implementation. Current main delegates `web_search` schema and execution to provider-owned definitions, and the active Brave provider still has no `goggles` schema field, URL forwarding, cache partitioning, or public Brave docs entry.

## What This Changes

This PR adds an optional Brave-only `goggles` parameter to the legacy core `web_search` path, forwards it to Brave web and llm-context requests, partitions cache entries, and updates web-search docs/tests/changelog.

## Best Possible Solution

Port the Goggles support into `extensions/brave` as provider-owned behavior: add an optional per-query `goggles` string to the active Brave schema, forward it to both Brave web and llm-context requests, include it in cache keys without lowercasing or missing-value collisions, document it on the public Brave and web-search pages, and leave the core wrapper extension-agnostic.
