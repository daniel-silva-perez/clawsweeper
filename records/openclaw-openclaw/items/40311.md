## Summary

Keep open. The feature remains useful, but current main moved managed web search onto provider-owned definitions, and this PR still edits the old core web-search surface plus the legacy Brave docs page. The patch needs a narrow port into the active Brave plugin before it can merge.

## What This Changes

The PR adds a Brave-only optional `goggles` parameter to `web_search`, forwards it to Brave web and llm-context requests, separates cache entries, and updates tests, docs, and changelog.

## Best Possible Solution

Land a rebased provider-owned port in `extensions/brave` that exposes per-query `goggles`, forwards the exact value to both Brave endpoints, includes it in cache separation without lowercasing or sentinel collisions, updates the current provider docs and web-search overview, and carries a credited changelog entry.

## Reproduction Assessment

Yes. A static current-main trace reproduces the gap: `web_search` delegates to provider definitions, and the active Brave provider has no `goggles` schema, URL forwarding, or cache-key component.

## Solution Assessment

No. The capability is a good fit, but this patch is no longer the narrow maintainable solution because it changes stale core files instead of the active Brave plugin-owned contract and runtime.
