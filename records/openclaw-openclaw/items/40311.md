## Summary

Keep open. The feature is valid and upstream-supported, but the branch is stale against current main: it edits the old core web_search implementation while main now delegates schema and execution to the provider-owned Brave plugin, where Goggles support is still missing.

## What This Changes

The PR branch adds an optional Brave-only `goggles` parameter to the legacy core web_search schema/runtime/cache path, updates Brave and web-search docs, adds tests, and adds a changelog entry.

## Best Possible Solution

Port the feature into `extensions/brave` as provider-owned behavior: expose a Brave-only per-query `goggles` string in the active tool schema, forward it to both Brave web and llm-context requests, keep cache entries distinct without lowercasing or missing-value collisions, and document it on the published Brave/web-search pages with a credited changelog entry.
