## Summary

Keep this PR open. The requested Brave Goggles capability is still absent from current main, but the branch targets the older core web-search implementation; the useful work needs to be rebased or ported into the active provider-owned Brave plugin before it can merge.

## What This Changes

This PR adds an optional Brave-only `goggles` parameter to the old core `web_search` path, forwards it to Brave web and llm-context requests, partitions caches by Goggles value, and updates web-search docs/tests/changelog.

## Best Possible Solution

Port the feature into `extensions/brave` so the active Brave provider exposes an optional per-query Goggles parameter, forwards it through both Brave web and llm-context requests, partitions cache entries correctly, documents the behavior on the Brave and web tools pages, and preserves the provider-owned boundary without adding core Brave-specific logic.
