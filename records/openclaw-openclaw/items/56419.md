## What This Changes

The issue requests a configurable image-analysis timeout so slow image providers are not hard-aborted by the shared `image` tool's fixed timeout.

## Best Possible Solution

Keep the shipped media-understanding timeout surface, reconcile #56454 against it, and then close this issue as implemented once the linked PR no longer conflicts with the issue state.

## Reproduction Assessment

Yes. The original failure path is high-confidence from source: a supplied image-description timeout drives an `AbortController` abort, and current tests show configured `tools.media.image` and per-model timeouts propagate to provider calls.

## Solution Assessment
