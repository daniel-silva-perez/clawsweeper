## What This Changes

The issue requests a configurable image-analysis timeout so slow image providers are not hard-aborted by the shared `image` tool's fixed timeout.

## Best Possible Solution

Yes. The original failure path is high-confidence from source because the image runtime aborts on the supplied timeout, and current tests prove configured tool/per-model timeouts propagate into provider calls.
