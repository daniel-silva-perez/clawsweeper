Keep open. Current main still lacks a supported `messages.modelExhausted` or equivalent config key, and the all-model billing/rate-limit exhaustion path still resolves to built-in reply text. The report is concrete, current code has the structured fallback data needed to implement it, and adjacent PR #70990 is plugin-hook work that does not change default user-facing fallback messaging.

Add a supported `messages.modelExhausted` string config key, wire it into the `FallbackSummaryError` all-model exhaustion path when every attempt reason is `billing` or `rate_limit`, preserve existing built-in messages when unset, update config types/schema/help/generated schema/docs/changelog, and add focused tests for pure billing, pure rate-limit, mixed billing+rate-limit, unset default behavior, and non-exhaustion failures.
