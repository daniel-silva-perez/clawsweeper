## Clownfish Work Prompt

Implement a configurable user-facing message for all-models-exhausted failures. Use this issue as the canonical user problem: when every configured fallback model fails for billing or rate-limit/exhaustion reasons, users should be able to set friendly text such as `messages.modelExhausted` instead of receiving the built-in billing/rate-limit copy. Keep the change in core config and the auto-reply fallback branch; do not add bundled-provider-specific behavior or plugin-only workarounds.

Expected boundary:
- Add a supported `messages.modelExhausted` string field, unless maintainers have already established a better nearby naming pattern by the time you work this.
- Update `MessagesConfig`, `MessagesSchema`, schema labels/help, generated schema artifacts if required by repo tooling, and public config docs.
- In `runAgentTurnWithFallback`, use the configured message for all-model fallback summaries that are pure billing, pure rate-limit/overloaded, or the agreed billing/rate-limit exhaustion bucket. Preserve existing raw-error sanitization and do not expose `All models failed` details.
- Keep existing hard-coded defaults when the config key is absent or blank/invalid per the chosen schema contract.
- Add regression tests for custom configured text, default billing behavior, default rate-limit behavior, and mixed billing/rate-limit behavior.
- Add a changelog entry only if repo policy treats this user-facing config addition as changelog-worthy; include a real reporter credit, not Codex.

Do not change provider failover classification, auth-profile cooldown semantics, plugin hook contracts, or unrelated message UX.
