## Summary

Keep this issue open. Current main still runs the reported WSL-sensitive pi-tools workspace-paths and safe-bins test paths without the WSL guard or temp-dir cleanup hardening proposed in the closed related PRs.

## Best Possible Solution

Keep this issue open until a focused test-only fix lands for src/agents/pi-tools.workspace-paths.test.ts and src/agents/pi-tools.safe-bins.test.ts. The best path is to harden temp-dir creation/cleanup, reuse src/infra/wsl.ts for any narrowly justified WSL guard, and verify the affected tests under WSL or an equivalent Windows/WSL lane rather than reviving the broader unrelated changes from PR #7085.
