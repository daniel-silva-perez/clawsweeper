## Summary

Keep this issue open. Current main still has the reported WSL-sensitive pi-tools test paths active, with plain temp-dir cleanup and only a native Windows skip in safe-bins. The related implementation PRs were closed unmerged, and WSL2 remains a documented supported/recommended path.

## Best Possible Solution

Keep this issue open until a focused test-only fix lands for src/agents/pi-tools.workspace-paths.test.ts and src/agents/pi-tools.safe-bins.test.ts. The best path is to harden temp-dir creation/cleanup, reuse src/infra/wsl.ts for any narrowly justified WSL guard, and verify the affected tests under WSL or a Windows/WSL lane without reviving unrelated changes from #7085.
