## Summary

Current main still has the unresolved WSL-sensitive test harness shape described in the issue: `workspace-paths` uses a local temp-dir helper with plain cleanup and no WSL guard, `safe-bins` skips only native Windows and still does plain sync cleanup, and the two focused related PRs were closed without merging. ## What This Changes

The issue asks to make the `pi-tools.workspace-paths` and `pi-tools.safe-bins` tests deterministic on Windows/WSL by hardening temp directory cleanup and platform-sensitive child-process handling.

## Best Possible Solution

Make the affected tests deterministic on supported hosts by reusing shared retrying temp-dir helpers and adding only narrow WSL skips where child-process behavior cannot be made reliable, with production behavior unchanged.
