Keep this issue open. Current main still has the reported WSL-sensitive tests without WSL guards or temp-dir cleanup hardening, and the two related mitigation PRs were closed unmerged rather than shipping a fix.

## Best Possible Solution

The best path is to harden temp-dir creation/cleanup around cwd-sensitive tests or add narrowly justified WSL guards using src/infra/wsl.ts, with targeted diagnostics or WSL verification and no unrelated changes.
