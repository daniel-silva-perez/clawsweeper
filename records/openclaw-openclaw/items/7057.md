## Summary

Keep open. Current main still leaves the reported WSL-sensitive test harness shape unresolved: the workspace-paths test file uses its own non-retrying temp-dir helper, and the safe-bins helper skips only native Windows while WSL remains on the Linux path. The focused fix PR referenced from the issue is closed unmerged, so there is still actionable test-only cleanup to do.

## What This Changes

The issue requests hardening or guarding the flaky `pi-tools.workspace-paths` and `pi-tools.safe-bins` tests that time out or hit temp-dir `ENOENT` under Windows/WSL environments.

## Best Possible Solution

The end state should make these two tests deterministic on supported hosts by using the shared retrying temp-dir helper and applying narrowly scoped WSL guards only where the child-process behavior cannot be made reliable, with production behavior unchanged.
