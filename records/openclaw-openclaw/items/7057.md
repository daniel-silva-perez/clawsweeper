## Summary

Keep this issue open. Current main still runs the reported workspace-paths and safe-bins tests on WSL without the WSL guards or cleanup hardening proposed in the related mitigation PRs, and the issue discussion includes a recent maintainer-side Codex review explicitly keeping it open for a focused test-only fix.

## Best Possible Solution

Keep this issue open until a focused test-only fix lands for src/agents/pi-tools.workspace-paths.test.ts and src/agents/pi-tools.safe-bins.test.ts. The likely maintainer path is to harden temp-dir creation/cleanup and add narrowly justified WSL guards using src/infra/wsl.ts, with targeted evidence that the affected tests no longer hang or race under WSL.
