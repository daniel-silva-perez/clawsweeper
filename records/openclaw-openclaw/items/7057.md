## Summary

Keep this issue open. Current main still has the reported WSL-sensitive workspace-path and safe-bins test paths without WSL guards or temp-directory cleanup hardening, and the related mitigation PRs were closed unmerged rather than shipped.

## Best Possible Solution

Keep this issue open until a focused test-only change lands for src/agents/pi-tools.workspace-paths.test.ts and src/agents/pi-tools.safe-bins.test.ts. The likely fix is to harden temp-dir creation/cleanup and/or add narrowly justified WSL guards using src/infra/wsl.ts, with targeted WSL diagnostics or clear code-level verification and no unrelated changes.
