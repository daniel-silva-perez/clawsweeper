## Summary

Keep open. Current main still has the unresolved WSL-sensitive test-harness shape described in the issue: the workspace-paths test keeps a file-local temp-dir helper with plain cleanup, the suite has no WSL guard for the cwd-sensitive cases, and safe-bins only skips native Windows. ## What This Changes

The issue reports WSL-sensitive timeouts and ENOENT failures in the pi-tools workspace-paths and safe-bins test suites and asks for deterministic hardening or narrow platform guards.

## Best Possible Solution

Make the affected tests deterministic on supported hosts by reusing the shared retrying temp-dir helpers and adding only narrowly scoped WSL skips where the child-process behavior cannot be made reliable, with production behavior unchanged.
