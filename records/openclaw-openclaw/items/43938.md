## Summary

Keep open. Current main still collapses channel-account config paths into whole-channel reloads, and this PR is a useful narrow fix, but the branch is dirty and the patch misses current gateway reload contracts that must be preserved before merge.

## What This Changes

The PR adds account-scoped channel restart planning for `channels.<channel>.accounts.<accountId>.*`, dispatches account-aware channel stop/start calls, and adds planner plus gateway reload tests.

## Best Possible Solution

Land a rebased account-scoped reload implementation that keeps full-channel precedence for mixed/global changes, applies active-work deferral to both account-scoped and whole-channel reloads, updates every current `GatewayReloadPlan` consumer, and covers hot reload plus secrets reload behavior with focused regression tests.

  - confidence: 0.84

  - confidence: 0.88
