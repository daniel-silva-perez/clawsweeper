## Summary

Keep open: the underlying account-scoped reload problem still exists on current main, and this PR is a plausible narrow fix, but it should not merge as-is because the new account-scoped plan state is not handled by the `secrets.reload` consumer.

## What This Changes

The PR adds `restartChannelAccounts` planning for account-scoped channel config paths, routes hot reload through account-aware channel start/stop calls, and adds planner plus hot-reload regression tests.

## Best Possible Solution

Rebase this PR onto current main and land it only after every `GatewayReloadPlan` consumer handles account-scoped restarts, full-channel precedence remains intact, active-work deferral still applies, and secrets reload has focused success, skip, and rollback coverage.
