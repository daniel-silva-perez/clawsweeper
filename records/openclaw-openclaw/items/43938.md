## Summary

Keep open. Current main still collapses account-scoped channel config paths into whole-channel restarts, so the PR targets a real unresolved gateway reload bug, but the proposed plan-shape change is not ready because `secrets.reload` is also a `GatewayReloadPlan` consumer and would ignore the new account-scoped restart map.

## What This Changes

The PR adds `restartChannelAccounts` planning for account-scoped channel config paths, routes hot reload through account-aware channel start/stop calls, and adds planner plus hot-reload regression tests.

## Best Possible Solution

Rebase and update the PR so every `GatewayReloadPlan` consumer handles account-scoped channel restarts consistently, including hot config reload and `secrets.reload`, while preserving full-channel precedence, active-work deferral, skip handling, rollback behavior, and focused regression coverage.
