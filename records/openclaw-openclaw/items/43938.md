## Summary

Keep open. The requested gateway reload optimization is still valid and current main still has the coarse channel-wide planning path, but this PR is not ready to land because it is dirty against current main and needs rebase fixes for the evolved reload-plan shape and active-work deferral behavior.

## What This Changes

The branch adds `GatewayReloadPlan.restartChannelAccounts`, detects `channels.<channel>.accounts.<accountId>.*` as account-scoped hot reloads, routes those changes through account-aware channel start/stop calls, and adds planner/handler tests.

## Best Possible Solution

Land a rebased account-scoped reload implementation that keeps full-channel precedence for mixed/global changes, updates every current `GatewayReloadPlan` literal/helper, preserves active-work deferral for account-scoped channel restarts, and keeps focused planner plus handler regression coverage tied to #43935.
