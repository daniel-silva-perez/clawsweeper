## Summary

Keep open. The account-scoped reload direction is valid and still missing from current main, but this PR needs changes before merge because the new account restart path bypasses existing active-work deferral and does not update all reload-plan consumers.

## What This Changes

The PR adds account-scoped channel reload planning for `channels.<channel>.accounts.<accountId>.*`, dispatches account-aware channel stop/start calls, and adds gateway reload planner plus hot-reload tests.

## Best Possible Solution

Land a rebased account-scoped reload implementation that keeps full-channel precedence for mixed/global changes, applies active-work deferral before both account-scoped and whole-channel reloads, updates secrets reload and typed plan helpers, and covers the hot reload plus secrets reload paths with focused regression tests.

## Reproduction Assessment

Yes. Current main gives a source-level reproduction path: a changed path like `channels.telegram.accounts.botA.token` matches Telegram's broad `channels.telegram` reload prefix and is planned as `restartChannels`, while the channel manager already supports account-scoped start/stop.

## Solution Assessment

No. The PR is pointed at the right narrow fix, but it is not the best mergeable solution until it preserves current reload deferral semantics and updates all consumers of `GatewayReloadPlan`.
