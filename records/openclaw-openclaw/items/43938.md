## Summary

Keep open. Current main still reloads channel account config changes at whole-channel granularity, and this PR is the open implementation candidate paired with the same-author issue #43935. The branch should not be cleanup-closed, but it needs rebase-aware review because current main now has additional GatewayReloadPlan consumers, especially the secrets reload path, that the proposed account-scoped plan shape must cover.

## What This Changes

The PR adds account-scoped channel restart planning for `channels.<channel>.accounts.<accountId>.*`, dispatches those changes through account-aware start/stop calls, preserves full-channel precedence for broader changes, and adds planner plus hot-reload tests.

## Best Possible Solution

Rebase this PR onto current main and land it only after the account-scoped plan representation is handled by every GatewayReloadPlan consumer, with full-channel precedence preserved, active-work deferral still applied, secrets reload behavior covered, and focused regression tests for planner, hot reload, and any auxiliary reload path touched.
