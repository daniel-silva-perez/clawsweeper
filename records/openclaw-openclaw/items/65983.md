## Summary

Keep open. Current main still has in-memory-only supervisor/process-session ownership and no real orphan reconciliation, and the docs still say background sessions are lost on restart. #66001 remains open and paired with this issue as the implementation candidate, so this issue should not be closed before that PR is reviewed, landed, or closed.

## Best Possible Solution

Review #66001 as the active implementation candidate. Maintainers should decide the intended lifecycle contract for PTY-backed background exec runs: in-process orphan reconciliation, durable startup reaping or re-registration after gateway restart, scope replacement for stable long-lived runs, no-output watchdogs where appropriate, and focused regression coverage for process-list visibility and cleanup after lost session ownership.
