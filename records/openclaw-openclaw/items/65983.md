## Summary

Keep open. Current main still has in-memory-only supervisor and process-session ownership, no production orphan reconciliation path, and docs still state background sessions are lost on restart. The related #66001 PR remains open and unmerged with closing syntax for this issue, so the issue should stay open until that implementation candidate is reviewed, landed, or closed.

## Best Possible Solution

Review #66001 as the active implementation candidate. The best path is a maintainer-approved lifecycle contract for PTY-backed background exec runs: reconcile or cancel supervisor runs whose bash session ownership disappears, decide whether cold-start stale process reaping or re-registration is required, add scope replacement and no-output watchdog behavior where appropriate, and cover process-list visibility plus cleanup with focused regression tests.
