## Summary

Keep open. Current main still has the no-op supervisor reconciliation and in-memory process/session ownership model described by the report, and #66001 remains an open implementation candidate with closing syntax, so this issue should not be closed before that PR is reviewed, landed, or closed.

## What This Changes

The issue requests lifecycle hardening for background PTY exec runs so process trees cannot remain alive and untracked after supervisor or session state is lost.

## Best Possible Solution

Land or revise the existing implementation candidate so the process supervisor has a clear lifecycle contract for PTY-backed background exec runs, with regression coverage for session-owner drift and process-list cleanup; if durable restart reaping remains out of scope, document that boundary and track any cold-start follow-up separately.
