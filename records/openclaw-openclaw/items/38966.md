## Summary

Keep this open. Current main still emits subscriber-gated `sessions.changed` reset events, but `openclaw tui` neither subscribes to those events nor dispatches/handles them, so an external `/new` or `/reset` can still leave stale TUI history/token/run state visible. The provided GitHub context also shows #40472 as an open, unmerged implementation candidate with closing syntax, so the issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land or close #40472. The right fix is to make the TUI subscribe to session change events, handle `sessions.changed` for the current session when the reason is `new` or `reset`, reload history/session metadata, clear stale run/stream/local-run state, ignore unrelated sessions, and add focused gateway/TUI regression coverage.
