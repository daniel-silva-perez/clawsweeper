## Summary

Current main still does not fully fix the reported TUI refresh gap. The gateway emits subscriber-gated `sessions.changed` events after `sessions.reset`, but the TUI gateway client neither subscribes to session-change events nor dispatches `sessions.changed` to a handler. The issue is also paired with open implementation PR #40472, which explicitly targets this behavior, so the issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review, land, or close #40472. The implementation should make the TUI subscribe to `sessions.changed`, dispatch and handle current-session `new`/`reset` events, refresh history/session metadata, clear stale active run/stream/local-run state, ignore unrelated sessions, and add focused TUI/gateway regression coverage.
