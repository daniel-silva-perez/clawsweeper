## Summary

Current main still does not refresh the TUI when the active session is reset externally. The gateway emits subscriber-gated `sessions.changed` reset events, and the protocol documents `sessions.subscribe`, but the TUI client neither subscribes to session-change events nor dispatches/handles `sessions.changed`. There is also an open unmerged implementation PR, #40472, that explicitly links this issue for automatic closure if merged, so this issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land or close #40472. The implementation should make the TUI subscribe to `sessions.changed`, dispatch session-change events, ignore unrelated sessions, refresh history/session metadata on current-session `new` or `reset`, clear stale active run/stream/local-run state, and add focused TUI/gateway regression coverage.
