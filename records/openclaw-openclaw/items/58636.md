## Summary

Keep this PR open. Current main still does not implement the TUI `/upload` command or any equivalent TUI file-attachment flow, and the PR is still open/unmerged. The work is plausibly useful, but it needs maintainer review and fixes for the remaining file-read safety and pending-context bugs before it should land.

## Best Possible Solution

Keep the PR open as a candidate implementation, but require a maintainer-quality revision: rebase onto current main, decide whether TUI should use the existing Gateway `attachments` contract or an explicit text-context prepend, add regular-file and size safeguards, fix pending-context lifecycle on failed/no-file uploads and send failures, then update TUI docs and targeted tests.
