## Summary

Keep this PR open. Current main still rejects the requested audio-level `allowPrivateNetwork` config, drops that flag from audio request overrides, and always builds an OpenAI-compatible audio Authorization header from the execution key; the PR remains an active implementation candidate for an SSRF/auth-sensitive media path.

## What This Changes

The PR adds audio-only private-network request config for media transcription, updates schema/generated metadata/docs/changelog, threads audio request overrides through provider execution, adjusts OpenAI-compatible audio local-auth handling, and adds regression tests.

## Best Possible Solution

Review or revise this PR as the narrow audio media fix: keep the opt-in audio-only, preserve image/video rejection, forward audio-level and per-model request config, align local OpenAI-compatible audio auth with the existing custom-local pattern, and tighten the test mock before merge.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: `ToolsSchema` rejects the requested media request key, runner execution drops audio-level `allowPrivateNetwork`, and OpenAI-compatible audio always builds a provider Authorization header.

## Solution Assessment

Yes, with a small test-fidelity follow-up. The PR uses the existing provider request contract instead of inventing a parallel transport path, but the `isLocalBaseUrl` mock should be aligned with production before maintainers treat the regression tests as complete.
