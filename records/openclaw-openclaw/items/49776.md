## Summary

Keep open. This is a recent Control UI freeze regression report with no browser console/network trace or verified root cause. Current main has adjacent freeze mitigations for live tool-result reload storms, oversized chat.history responses, and markdown ReDoS, but I did not find a targeted fix, shipped provenance, or exact duplicate that covers this Windows Chrome/Edge freeze after static assets load.

## Best Possible Solution

Keep this issue open and ask for a current-main reproduction with DevTools console errors, failed requests/WS frames, session file sizes, OpenClaw/browser versions, and whether resetting sessions still fixes it. If the root cause is accumulated history, fix it through an additive chat.history pagination/windowing contract plus Control UI load-older or virtualization work. If it is a Chromium first-render freeze, bisect the connected dashboard shell and add a targeted fallback or browser smoke. If it is the model selector/provider path, tie it to the already-fixed model-ref work only after reproducing the same failure mode.
