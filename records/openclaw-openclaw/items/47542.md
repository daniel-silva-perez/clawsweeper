## Summary

Keep this PR open. Current main still does not implement composing-aware inbound debounce for WhatsApp: there is no `composingDebounceMs` config, no WhatsApp inbound `presence.update`/`presenceSubscribe` path wired into debounce buffers, and `createInboundDebouncer` still exposes only `enqueue` and `flushKey`. The PR remains a plausible focused feature, but the provided review discussion identifies real blockers that still need branch updates before merge.

## Best Possible Solution

Keep this PR open and ask for a focused update on the existing branch: add runtime schema/config metadata and tests for `composingDebounceMs`, add covered `touch`/`touchByPrefix` debouncer behavior, wire WhatsApp `presence.update` using direct and group key formats that match current debounce buffers, dedupe `presenceSubscribe`, bound or TTL the LID-to-E164 cache, correct the security-impact section, and validate the shared inbound debounce plus WhatsApp plugin tests.
