## Summary

Current main partially addresses adjacent auto-fallback stickiness, but it still intentionally preserves explicit `/model` session overrides across `/new`, `/reset`, `sessions.reset`, and restarts. That leaves the reported user-set override versus changed config default behavior unresolved, along with incomplete `/status` source visibility.

## Best Possible Solution

Keep this open for a focused maintainer product/code decision. The clean path is either to make `/new`, `/reset`, and `sessions.reset` clear explicit user `providerOverride`/`modelOverride`, or to preserve pins but add first-class UX: `/status` labels for config default versus session override, a documented `/model clear` or `/model default` command, and docs under models/session/slash-commands that explain persistence and clearing.
