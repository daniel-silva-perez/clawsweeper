## Summary

Keep this PR open. Current main still does not implement the Feishu `type=file` HTTP 502 fallback to `type=media`, the linked bug remains concrete and open, and the overlapping fix attempt is also open/unmerged rather than a canonical supersession.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this patch or an equivalent Feishu-owned change in `extensions/feishu/src/media.ts`, with focused coverage in `extensions/feishu/src/media.test.ts` for HTTP 502 on `type=file`, retrying with `type=media`, preserving the original error if the retry fails, and leaving image/non-502 paths unchanged. After one fix merges, #49855 can close through the linked implementation and the duplicate open fix attempt can be resolved.

## Risks / Open Questions

- This PR does not add regression tests for the 502 `type=file` retry, successful `type=media` fallback, non-502 behavior, or preserving the original 502 error when the retry also fails.
- The patch relies on Axios-style `err.response.status`; OpenClaw's Feishu client uses the Lark SDK over Axios, but a small helper and tests would make that dependency contract explicit.
- Existing in-tree test commentary says message resources should use `image|file`; maintainers should reconcile that older assumption with the concrete Feishu report and the SDK's string `type` surface before landing.
- There is an overlapping open PR, #49967, for the same bug. Maintainers should choose one implementation path and close the other after a fix merges.
