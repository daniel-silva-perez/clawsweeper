## Summary

Keep open: current main still surfaces raw ClawHub 429 failures without an unauthenticated-login hint, and an open PR is already linked as the implementation candidate, so this issue should remain open until that PR is reviewed, landed, replaced, or closed.

## What This Changes

Request to append a ClawHub login/rate-limit hint to unauthenticated 429 errors from `openclaw skills search` and `openclaw skills install`.

## Best Possible Solution

Review and land #56394 or an equivalent narrow main-branch change that appends a login hint only for unauthenticated ClawHub 429 responses across JSON and archive-download paths, with regression coverage for unauthenticated 429, authenticated 429, and non-429 errors.

## Reproduction Assessment

Yes. A mocked ClawHub `fetchImpl` returning HTTP 429 through `searchClawHubSkills` or the archive download helpers would reproduce the raw `ClawHub ... failed (429): Rate limit exceeded` message; source inspection shows no hint path today.

## Solution Assessment

Yes, the proposed direction is the narrowest maintainable fix if implemented in the shared ClawHub request/error helper and covered at the infra boundary. The linked PR’s described boundary matches that approach, pending code review.
