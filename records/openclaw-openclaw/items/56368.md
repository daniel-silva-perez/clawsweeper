## Summary

Keep open: current main still emits raw ClawHub 429 failures without an unauthenticated-login hint, and the issue is already paired with open implementation PR #56394, so it should remain open until that PR is reviewed, landed, replaced, or closed.

## What This Changes

The issue requests a helpful ClawHub login hint when `openclaw skills search` or `openclaw skills install` receives an unauthenticated HTTP 429 rate-limit response.

## Best Possible Solution

Review and land #56394, or an equivalent narrow main-branch change, that appends a login hint only for unauthenticated ClawHub 429 responses across JSON and archive-download paths, with regression coverage for unauthenticated 429, authenticated 429, and non-429 errors.

## Reproduction Assessment

Yes. A mocked `fetchImpl` returning HTTP 429 through `searchClawHubSkills` or either ClawHub archive download helper would produce the raw `ClawHub ... failed (429): Rate limit exceeded` message because current error formatting has no hint path.

## Solution Assessment

Yes. The requested direction is the narrowest maintainable fix if implemented in the shared ClawHub request/error boundary so all CLI and gateway skill/plugin paths inherit the same unauthenticated-429 hint without changing authenticated rate-limit behavior.
