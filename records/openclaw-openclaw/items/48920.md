## Summary

Keep open. The original `isolatedSession` mismatch is fixed in the latest release, but the broader live-docs-versus-release drift is still present: current main documents and accepts `heartbeat.skipWhenBusy`, while the latest release tag does not, and the docs publish path mirrors main docs to the live site.

## What This Changes

The issue asks for live OpenClaw docs to stay aligned with released versions or clearly show version availability for documented heartbeat config keys.

## Best Possible Solution

Define an explicit compatibility policy for `docs.openclaw.ai`, then implement the matching publish workflow, site metadata, generated config-doc checks, or version-aware validation errors needed to keep documented config keys aligned with the intended user version. Treat #71301 as complementary bundled-docs work, not the sole replacement for the public live-docs policy gap.
