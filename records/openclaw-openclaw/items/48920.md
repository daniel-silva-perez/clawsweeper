## Summary

Keep open. The original `isolatedSession` mismatch is fixed in the latest release, but the broader live-docs-versus-release drift is still present: current main documents and accepts `heartbeat.skipWhenBusy`, while `v2026.4.27` does not.

## What This Changes

The issue requests that live OpenClaw docs stay aligned with released versions, or clearly mark version availability, so users do not copy config keys rejected by their installed release.

## Best Possible Solution

Define an explicit compatibility policy for `docs.openclaw.ai`, then implement the matching publish workflow, site metadata, generated config-doc checks, or version-aware validation errors needed to keep documented config keys aligned with the intended user version.
