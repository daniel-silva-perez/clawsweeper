## Summary

Keep open. The original `isolatedSession` mismatch is resolved in the latest release, but the broader live-docs-versus-release drift remains reproducible: current main documents and accepts `heartbeat.skipWhenBusy`, while the latest release tag does not.

## What This Changes

Request to keep live docs aligned with the latest shipped OpenClaw release, or clearly version-gate docs so users do not configure unreleased heartbeat keys.

## Best Possible Solution

Define an explicit compatibility policy for `docs.openclaw.ai`, then implement the matching workflow/site metadata and any generated config-docs or validation checks against the latest release tag. Treat #71301 as complementary bundled-docs work rather than the sole replacement for this live-docs policy gap.
