## Summary

Keep open. The concrete `isolatedSession` mismatch reported against 2026.3.13 is fixed in the latest release, but the broader live-docs-versus-release problem remains: current main documents and accepts a newer heartbeat key that `v2026.4.27` does not accept, while the docs publishing workflow mirrors main docs to the public docs repository.

## What This Changes

The issue asks for live docs to stay aligned with the latest release, or to show version availability, so documented heartbeat config keys do not fail strict validation on released builds.

## Best Possible Solution

Define an explicit compatibility policy for `docs.openclaw.ai`, then implement the matching publish workflow, version metadata, generated config-doc compatibility checks, and/or version-aware validation errors so users can tell whether a documented config key is available in their installed OpenClaw version. Treat #71301 as complementary bundled-docs work, not a complete replacement for the public live-docs policy gap.
