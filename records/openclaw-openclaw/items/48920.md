## Summary

Keep this issue open. The original isolatedSession example is now supported in the latest available release, but the broader reported product problem remains: live docs are still synced from main on docs changes, current main is already versioned ahead of the latest release, and the docs config does not expose an obvious user-facing docs-version or release-gating mechanism.

## Best Possible Solution

Keep the issue open as a docs/release process bug. The best path is to add a user-facing docs version or release-channel banner and either publish stable docs from release tags or add a CI/config-docs guard that prevents live docs from recommending config keys absent from the current stable release; a secondary improvement would be validation that can recognize known-but-unavailable keys and tell users which OpenClaw version is required.
