## Summary

Keep this issue open. The original `isolatedSession` mismatch reported against OpenClaw 2026.3.13 is fixed in current main and in the latest release, but the broader bug remains: docs.openclaw.ai is still synced from `main` docs changes, while current main is ahead of the latest release, so live docs can still describe unreleased behavior.

## Best Possible Solution

Keep this issue open for maintainer product/release-policy follow-up. Maintainers should choose an explicit docs.openclaw.ai policy: publish from release tags, visibly label the site as latest-main with release/version warnings, add per-feature availability metadata, or add CI that catches documented config examples absent from the latest release. Coordinate with #71301 for the separate bundled/local docs retrieval work.
