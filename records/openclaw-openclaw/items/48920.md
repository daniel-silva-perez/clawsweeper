## Summary

Keep open. The original `isolatedSession` mismatch is fixed in the latest release, but the broader live-docs-versus-release drift still exists: current main documents and accepts `heartbeat.skipWhenBusy`, while the latest release tag lacks that key and the docs publish workflow still mirrors docs from main.

## What This Changes

Request to keep live docs aligned with the latest shipped OpenClaw release, or visibly version/version-gate docs so users do not configure unreleased heartbeat keys.

## Best Possible Solution

Define an explicit compatibility policy for `docs.openclaw.ai`, then implement the supporting workflow/site metadata and any desired generated-docs/config compatibility checks against the latest release tag. Keep #71301 as a complementary bundled-docs discussion, not the sole replacement for this live-docs policy issue.
