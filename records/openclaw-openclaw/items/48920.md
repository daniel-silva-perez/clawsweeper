## Summary

Keep open. The original `isolatedSession` mismatch is fixed in the latest release, but the reported live-docs-versus-release drift still exists as a reproducible class: current main documents and accepts `heartbeat.skipWhenBusy`, while the latest shipped release `v2026.4.27` does not accept that strict-schema key.

## What This Changes

Ensure public live docs do not advertise config keys that the latest OpenClaw release rejects, or clearly mark the required OpenClaw version.

## Best Possible Solution

Define the compatibility contract for `docs.openclaw.ai`, then implement the matching publish workflow, site/version metadata, generated config-doc compatibility checks, and/or version-aware validation messages so released users can tell whether a documented config key is available in their installed OpenClaw version.

## Reproduction Assessment

Yes. The original `isolatedSession` reproduction is fixed in `v2026.4.27`, but the same class reproduces by comparing current main docs for `heartbeat.skipWhenBusy` with the strict `v2026.4.27` heartbeat schema that does not accept that key.

## Solution Assessment

Unclear because there is no patch and the next step is a docs/release policy decision. A validation-only error would reduce confusion, but it would not fully solve live docs advertising unreleased config without a versioning policy.
