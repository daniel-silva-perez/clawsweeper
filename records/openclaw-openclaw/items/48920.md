## Summary

Keep open. The original `isolatedSession` mismatch has shipped in `v2026.4.26`, but the broader live-docs-versus-release drift remains reproducible: current `main` documents and accepts `heartbeat.skipWhenBusy`, while the latest release tag does not, and the docs publish workflow still mirrors docs from `main`.

## What This Changes

The issue asks OpenClaw to keep live docs aligned with released versions, or otherwise show version availability, after heartbeat docs advertised config keys unsupported by the latest release.

## Best Possible Solution

Define an explicit compatibility policy for `docs.openclaw.ai`, then implement the matching workflow/site metadata and any generated config-docs or validation checks needed to keep documented config keys aligned with the intended user version. Treat #71301 as complementary bundled-docs work rather than the sole replacement for the live-docs policy gap.
