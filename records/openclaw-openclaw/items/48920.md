## Summary

Keep this issue open. The original `isolatedSession` mismatch is fixed in the current release, but the broader live-docs-versus-release drift is still present: current `main` documents `heartbeat.skipWhenBusy`, while the latest release tag `v2026.4.26` lacks that config key and the docs publish workflow still syncs docs from `main`.

## What This Changes

The issue asks OpenClaw to keep live docs aligned with released versions, or show/version-gate docs, so released users do not copy unsupported configuration keys.

## Best Possible Solution

Define an explicit compatibility policy for `docs.openclaw.ai`, then implement the supporting workflow/site metadata and, if desired, a generated-docs/config compatibility check against the latest release tag. Keep #71301 as the broader local bundled-docs retrieval discussion, not as the sole replacement for this live-docs policy issue.
