## Summary

Keep this issue open. The original `isolatedSession` mismatch is fixed in current main and the latest release, but the broader docs-versus-release drift is still real: current main publishes live docs from `main`, and current docs already mention `heartbeat.skipWhenBusy`, which is absent from the latest release tag `v2026.4.26`.

## What This Changes

The issue asks OpenClaw to keep live docs in step with released versions, or visibly mark/version-gate docs so users do not copy unreleased config keys into released builds.

## Best Possible Solution

Keep this issue open for maintainer release/docs policy follow-up. Maintainers should choose an explicit docs.openclaw.ai compatibility policy, then implement it in the docs publish workflow/site metadata and, if desired, add a config-docs compatibility check against the latest release. Coordinate with #71301 for the separate local bundled-docs retrieval proposal.
