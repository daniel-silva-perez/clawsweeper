## Summary

Keep this PR open, but do not merge it as-is. Current main still has the Gemma dynamic fallback path that can inherit the Gemini flash 1M context window, so the PR is not obsolete; however, the proposed 131,072-token fallback under-reports the affected Gemma 4 model compared with the linked report and the current models.dev Google catalog value.

## What This Changes

The PR adds a 131,072-token Gemma context-window fallback in the Google forward-compat resolver, preserves the Gemma 4 reasoning override, and adds a regression test for the Gemma 4 template-clone path.

## Best Possible Solution

Keep the fix in the Google provider boundary, but revise the fallback metadata so Gemma clones avoid Gemini flash's 1M window while using the best known Gemma value for the matched family or exact model, with regression coverage for configured metadata and registry-miss fallback behavior.
