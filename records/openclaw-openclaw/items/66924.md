## Summary

Keep this PR open for maintainer review. Current main still lets Gemma forward-compat clones inherit the Gemini flash template context window when no configured/catalog metadata wins, but the PR’s hardcoded 131,072-token fallback does not match the linked Gemma 4 report’s 256k metadata and needs provider metadata policy review before merge.

## What This Changes

The PR adds a 131,072-token Gemma context-window fallback in the Google forward-compat resolver, preserves the Gemma 4 reasoning patch, and adds a regression test for the Gemma 4 template-clone path.

## Best Possible Solution

Keep the fix in the Google provider boundary, but make the fallback metadata specific enough that Gemma clones avoid Gemini flash’s 1M window while preserving configured/catalog metadata and using the best known Gemma family values when the registry misses an exact id.
