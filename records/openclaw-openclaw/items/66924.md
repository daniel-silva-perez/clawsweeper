## Summary

Keep this PR open. Current main still maps `gemma-*` forward-compat models to the Gemini flash template and only patches Gemma 4 reasoning, while the shared template clone helper preserves inherited template metadata such as the 1M context window unless the resolver supplies an override. The PR is a narrow Google provider fix with a matching regression test; the only substantive maintainer question is whether the fallback should be a hardcoded 128k or should preserve configured/discovered Gemma metadata when available.

## What This Changes

The PR adds a 131,072-token Gemma context-window fallback in the Google forward-compat resolver, preserves the Gemma 4 reasoning override, and adds a regression test for the Gemma 4 template-clone path.

## Best Possible Solution

Keep the fix in the Google provider boundary and add regression coverage, ideally ensuring Gemma clones do not inherit Gemini flash's 1M context while preserving explicit configured or discovered Gemma context metadata when that metadata is available.
