## Summary

Keep this issue open. Current main still exposes Telegram inline keyboards through the agent/message send path, but it does not expose Telegram-native reply keyboards or keyboard removal semantics; the related closed items cover inline buttons/callback behavior rather than this separate reply-keyboard capability.

## What This Changes

Add native Telegram reply keyboard support, including keyboard rows/buttons, keyboard removal, resize_keyboard, and one_time_keyboard, to the Telegram agent/message send surface.

## Best Possible Solution

Implement this in the Telegram plugin as an additive, typed reply_markup path for outbound sends, with docs and tests proving reply keyboards, keyboard removal, resize_keyboard, and one_time_keyboard work through the message tool and Telegram action runtime without changing the generic presentation contract unless maintainers explicitly choose that broader design.
