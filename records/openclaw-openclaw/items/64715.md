## Summary

Keep this issue open. Current main still maps Telegram outbound presentation and action sends to inline keyboards only; there is no ReplyKeyboardMarkup, ReplyKeyboardRemove, resize_keyboard, or one_time_keyboard schema/runtime/docs surface. The related issues cover inline buttons and callback handling, so they do not supersede this separate reply-keyboard capability.

## What This Changes

Request first-class Telegram reply-keyboard rows, resize/one-time options, and keyboard removal support through outbound agent/message sends.

## Best Possible Solution

Implement this in the Telegram plugin as an additive, typed reply_markup path for outbound sends, with docs and tests proving reply keyboard rows, keyboard removal, resize_keyboard, and one_time_keyboard work without regressing existing inline keyboard behavior.
