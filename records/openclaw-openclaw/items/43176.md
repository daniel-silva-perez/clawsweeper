## Summary

Keep this PR open. Current main still does not implement the requested broad `find $HOME`/`find /` timeout cap or the matching prompt guidance, and the PR remains a focused implementation candidate. It should not be closed as obsolete; it needs maintainer review/changes because the patch touches shell execution timeout behavior and existing review comments identify remaining parser coverage gaps.

## Best Possible Solution

Keep this PR open and request changes before merge. The best path is to rebase onto current main, implement a shared broad-`find` timeout classifier used by local exec, gateway approval follow-up, and node execution planning, parse all `find` start roots, resolve relative roots against command `workdir`, support `~`, `$HOME`, and `${HOME}`, preserve explicit timeout and real traversal-limiting escape hatches only when they apply to the actual `find`, and add focused regression tests for local, gateway, node, relative-workdir, multiple-root, and prompt-guidance behavior.

## Risks / Open Questions

- Current main can still let broad `$HOME` or `/` discovery commands consume the configured default exec timeout when the agent omits an explicit timeout.
- Merging the PR as-is could leave inconsistent behavior across local exec, gateway approval follow-up, and node `system.run` paths.
- A broad-find classifier must avoid both false negatives for multi-root or `${HOME}` scans and false positives for legitimate workspace-relative searches.
