## Summary

Keep this PR open. Current main still lacks the requested broad `find` timeout cap and the matching prompt guidance, while the PR remains a plausible focused fix for a real exec-budget failure mode. It needs maintainer review and parser hardening rather than cleanup closure.

## Best Possible Solution

Keep this PR open and request changes before merge. The best path is to rebase onto current main, add a shared broad-`find` timeout classifier used by every foreground exec route, parse all `find` start roots, resolve relative roots against the command `workdir`, support `~`, `$HOME`, and `${HOME}` forms, preserve explicit `timeout`, `-prune`, `-xdev`, and `-mount` intent only when those tokens constrain the actual `find`, and keep focused regression tests for home, root, relative workdir, multiple roots, gateway follow-up, node routing, and prompt guidance.
