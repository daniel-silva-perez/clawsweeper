## Summary

Keep open. Current main still lacks Xiaomi MiMo VoiceDesign support: the bundled Xiaomi speech provider only normalizes built-in voice, model, format, and style fields, registers only the built-in TTS model ids, and always sends `audio.voice`, while Xiaomi's current MiMo v2.5 TTS docs define a separate `mimo-v2.5-tts-voicedesign` request shape that uses the `user` message as the voice description and omits preset voice selection.

## What This Changes

The issue requests Xiaomi MiMo v2.5 TTS VoiceDesign configuration support so OpenClaw can pass a voice-description prompt through the bundled Xiaomi TTS provider instead of only using preset voice ids.

## Best Possible Solution

Extend only the bundled `xiaomi` speech provider with a documented VoiceDesign config shape, map it to Xiaomi's `mimo-v2.5-tts-voicedesign` chat-completions contract, keep existing preset `voice` and `style` behavior unchanged, and add focused provider tests plus Xiaomi/TTS docs.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the gap: the provider does not read `voiceDesign`, does not register the VoiceDesign model id, and always sends `audio.voice` in the Xiaomi TTS request.

## Solution Assessment

Yes. A provider-local Xiaomi implementation is the narrowest maintainable path; a generic core `extraBody` passthrough or new SDK surface would be broader than the upstream provider-specific contract requires.

## Repair Work Prompt

Implement Xiaomi MiMo v2.5 TTS VoiceDesign support for this issue. Keep the change provider-owned in `extensions/xiaomi`; do not add a generic core `extraBody` passthrough or new SDK surface unless the existing speech provider config/override records prove insufficient.

Expected behavior: allow Xiaomi TTS config under `messages.tts.providers.xiaomi` to specify a documented VoiceDesign prompt shape, such as `voiceDesign: { prompt: string }` or an equivalent clearly documented field. When VoiceDesign is active, send Xiaomi's official chat-completions request for `mimo-v2.5-tts-voicedesign`: `messages` should include the voice description as a `user` message and the spoken text as an `assistant` message, and `audio` should include the output format without preset `voice`. Keep existing `voice` and `style` behavior unchanged for `mimo-v2.5-tts` and `mimo-v2-tts`.

Use the upstream contract from `https://platform.xiaomimimo.com/static/docs/usage-guide/speech-synthesis-v2.5.md`: VoiceDesign model id is `mimo-v2.5-tts-voicedesign`; the `user` message is required and is the voice design description; non-streaming examples use WAV and streaming examples use PCM16. Do not assume MP3 works for VoiceDesign unless a live/API check proves it.

Likely implementation files: `extensions/xiaomi/speech-provider.ts`, `extensions/xiaomi/speech-provider.test.ts`, `docs/providers/xiaomi.md`, `docs/tools/tts.md`, and `CHANGELOG.md`. Add focused tests for config normalization, request body construction, model metadata, and preserving old preset voice requests. If credentials are available, add or update live test coverage conservatively without committing secrets or generated audio.

Do not change unrelated TTS providers, core TTS architecture, plugin SDK contracts, package metadata, or lockfiles.
