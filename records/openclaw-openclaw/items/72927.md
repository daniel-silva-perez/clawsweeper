## Summary

Keep open. Current main still lacks Xiaomi MiMo VoiceDesign support: the Xiaomi speech provider only normalizes preset voice/model/format/style settings, advertises only the base MiMo TTS model IDs, and serializes requests with `audio.voice` rather than the VoiceDesign request shape. The existing discussion already identified the same provider-local gap and routed @zoujiejun and @leonchui; no linked merged PR or open PR supersedes the remaining work.

## What This Changes

The issue requests Xiaomi MiMo v2.5 TTS VoiceDesign configuration support in the bundled Xiaomi TTS provider while preserving existing preset `voice` behavior and documenting the new schema.

## Best Possible Solution

Extend only the Xiaomi speech provider with a documented VoiceDesign config shape mapped to Xiaomi's `mimo-v2.5-tts-voicedesign` chat-completions contract, keep existing `voice`, `style`, `format`, `mimo-v2.5-tts`, and `mimo-v2-tts` behavior unchanged, and add focused provider tests plus Xiaomi/TTS docs and a user-facing changelog entry.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the gap: a `voiceDesign` config value has no schema or normalization path, the VoiceDesign model is not advertised, and synthesis always serializes a preset `audio.voice`.

## Solution Assessment

Yes. A provider-local Xiaomi change is the narrowest maintainable solution; a generic core `extraBody` passthrough or new SDK seam would be broader than this provider-specific request contract requires.
