## Summary

Keep open. Current main has Xiaomi MiMo TTS with `style` guidance, but it still lacks first-class VoiceDesign support: no `voiceDesign` config field, no VoiceDesign model in provider metadata, and the request builder always sends a built-in `audio.voice`. The upstream request contract is now clear enough for a narrow provider-local fix.

## What This Changes

Add Xiaomi MiMo v2.5 TTS VoiceDesign configuration so users can describe a custom voice instead of selecting only preset voice IDs.

## Best Possible Solution

Extend the bundled `xiaomi` speech provider only: add a documented VoiceDesign config shape backed by Xiaomi's official `mimo-v2.5-tts-voicedesign` chat-completions contract, preserve existing `voice` and `style` behavior for built-in voices, and avoid adding a generic core passthrough unless another provider needs the same seam.

## Reproduction Assessment

Yes for the config gap: static inspection of current main shows `voiceDesign` is not normalized, documented, tested, or sent, and VoiceDesign requests cannot omit `audio.voice`. Live audio reproduction would require Xiaomi API credentials and model entitlement.

## Solution Assessment

Yes, a provider-local Xiaomi change is the narrowest maintainable path now that the upstream request shape is known. A generic TTS `extraBody` or core SDK change would be broader than needed for this issue.
