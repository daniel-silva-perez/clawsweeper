## Summary

Keep open. Current main has Xiaomi MiMo TTS support, including natural-language `style` guidance, but it does not preserve or forward a `voiceDesign` payload, and the exact upstream VoiceDesign request contract still needs maintainer/API confirmation before an automated repair is safe.

## What This Changes

The issue requests Xiaomi MiMo v2.5 TTS VoiceDesign passthrough from OpenClaw TTS provider configuration.

## Best Possible Solution

Decide the product boundary first: either extend the bundled `extensions/xiaomi` speech provider with a documented, upstream-verified VoiceDesign config/request mapping, or explicitly document that advanced MiMo v2.5 voice-design workflows should live in ClawHub plugins while the bundled provider remains preset-voice plus style guidance.

## Reproduction Assessment

Yes for the config gap: a static check of current main shows `voiceDesign` is not normalized, documented, tested, or included in the Xiaomi request body. Live reproduction of the actual VoiceDesign audio behavior would still require Xiaomi API credentials and the upstream request contract.

## Solution Assessment

Unclear. A provider-local Xiaomi change would be the narrowest in-repo path if maintainers want bundled support, but the safest next step is to confirm Xiaomi’s official VoiceDesign request schema before coding.
