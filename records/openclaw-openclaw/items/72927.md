## Summary

Keep open. Current main still supports only Xiaomi MiMo built-in TTS voices plus optional `style` guidance; it does not expose, normalize, document, or send the official `mimo-v2.5-tts-voicedesign` request shape, so the requested VoiceDesign capability remains valid and narrow.

## What This Changes

Add Xiaomi MiMo v2.5 TTS VoiceDesign support to the bundled Xiaomi TTS provider configuration while preserving existing `voice` behavior.

## Best Possible Solution

Extend only the bundled `xiaomi` speech provider: add a documented VoiceDesign config shape, map it to Xiaomi's official `mimo-v2.5-tts-voicedesign` chat-completions contract, keep preset `voice` and `style` behavior unchanged, and add focused tests/docs for the new request shape.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the gap: `voiceDesign` is not read, the VoiceDesign model id is not registered, docs/tests omit it, and the request builder always sends `audio.voice`.

## Solution Assessment

Yes. A provider-local Xiaomi implementation is the narrowest maintainable solution; a generic core `extraBody` passthrough or SDK change would be broader than this provider-specific upstream contract requires.
