## Summary

Keep open. Current main still lacks Xiaomi MiMo VoiceDesign support: the bundled Xiaomi speech provider only normalizes preset voice/model/format/style settings, registers only the base MiMo TTS model IDs, and always serializes a preset `audio.voice`. Xiaomi's current MiMo v2.5 TTS docs describe VoiceDesign as a separate model/request shape, so this remains a valid provider-local feature request.

## What This Changes

Add Xiaomi MiMo v2.5 TTS VoiceDesign configuration support so users can describe a desired voice instead of only selecting a preset Xiaomi voice ID.

## Best Possible Solution

Extend only the bundled `xiaomi` speech provider with a documented VoiceDesign config shape, map it to Xiaomi's current `mimo-v2.5-tts-voicedesign` chat-completions contract, preserve existing preset `voice`, `style`, `format`, `mimo-v2.5-tts`, and `mimo-v2-tts` behavior, and add focused provider tests plus Xiaomi/TTS docs and changelog coverage.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the gap: a `voiceDesign` config value has no normalization path, the VoiceDesign model is not advertised, and synthesis still always sends a preset `audio.voice`.

## Solution Assessment

Yes. A provider-local Xiaomi change is the narrowest maintainable solution; a generic core `extraBody` passthrough or new SDK surface would be broader than this provider-specific request contract requires.
