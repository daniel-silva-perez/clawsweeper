## Summary

Keep open. Current main still lacks Xiaomi MiMo VoiceDesign support: the bundled Xiaomi speech provider only normalizes preset voice/model/format/style settings, registers only the base MiMo TTS model IDs, and always sends `audio.voice` in the request body. Xiaomi's current MiMo page also advertises Voice Design as a distinct TTS-series capability, so this remains a valid provider-local feature request.

## What This Changes

Add Xiaomi MiMo v2.5 TTS VoiceDesign configuration support so users can describe a desired voice instead of selecting only a preset Xiaomi voice ID.

## Best Possible Solution

Extend only the bundled `xiaomi` speech provider with a documented VoiceDesign config shape, map it to the current Xiaomi MiMo VoiceDesign TTS request contract, preserve existing preset `voice` and `style` behavior, and add focused provider tests plus Xiaomi/TTS docs and changelog coverage.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the gap: a `voiceDesign` config value has no normalization path, is not represented in the request body, and the provider still requires/sends a preset `audio.voice`.

## Solution Assessment

Yes. A provider-local Xiaomi change is the narrowest maintainable solution; a generic core `extraBody` passthrough or new SDK surface would be broader than this provider-specific request contract requires.
