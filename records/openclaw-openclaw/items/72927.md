## Summary

Keep open. Current main supports Xiaomi MiMo TTS with fixed voice selection plus natural-language `style` guidance, but it does not normalize, document, test, or forward a `voiceDesign` payload. The remaining decision is whether bundled Xiaomi should own an upstream-verified VoiceDesign mapping or whether advanced MiMo voice-design workflows should live in a plugin.

## What This Changes

Add Xiaomi MiMo v2.5 TTS VoiceDesign configuration support so users can describe or configure custom voices instead of only selecting preset voice IDs.

## Best Possible Solution

Confirm Xiaomi's official VoiceDesign API contract first, then either extend `extensions/xiaomi` with a documented, provider-local config/request mapping and regression tests, or explicitly route advanced VoiceDesign workflows to a ClawHub/plugin implementation while keeping bundled Xiaomi on preset voices plus style guidance.

## Reproduction Assessment

Yes for the config gap: a static check of current main shows `voiceDesign` is not normalized, documented, tested, or included in the Xiaomi request body. Live audio reproduction would require Xiaomi credentials and the upstream VoiceDesign contract.

## Solution Assessment

Unclear. A provider-local Xiaomi change is the narrowest in-repo implementation if bundled support is desired, but the safe path requires upstream schema confirmation before coding.
