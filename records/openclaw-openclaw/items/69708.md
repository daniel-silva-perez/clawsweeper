## Summary

Keep open. Current main still rejects canonical persisted auth-profile entries whose normalized `type` is `aws-sdk`, while Bedrock provider config, runtime auth resolution, tests, and docs still expect `auth: "aws-sdk"` to use the AWS SDK default credential chain for IMDS/instance-role setups. The focused fix PR #69753 was closed unmerged, so this remains a valid narrow bug and a good fix-PR candidate.

## Best Possible Solution

Implement a narrow auth-profile store fix that treats `aws-sdk` as an SDK-managed non-credential marker during load, skips it without adding it to `AuthProfileCredential`, suppresses the invalid-entry warning only for that marker, preserves warnings for genuine invalid entries, and leaves Bedrock runtime auth on provider config/AWS SDK default-chain resolution. Add focused regression tests and a user-facing changelog fix entry crediting the reporter and useful prior PR context.
