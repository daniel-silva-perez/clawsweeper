## Summary

Current main still rejects persisted `auth-profiles.json` entries whose normalized `type` is `aws-sdk`. That conflicts with the Bedrock runtime and docs, which expect `auth: "aws-sdk"` to use the AWS SDK default credential chain for EC2/IMDS. The closed unmerged #69753 remains useful implementation context, but the fix is not present on main or in v2026.4.26.

## Best Possible Solution

Keep this issue open and land a narrow auth-profile store fix: treat `aws-sdk` as an SDK-managed marker during persisted store load, skip it without storing an `AuthProfileCredential`, suppress invalid-entry warning counts only for that marker, preserve warnings for real invalid entries, and add focused regression tests plus an Unreleased changelog fix entry crediting the reporter and useful #69753 context.
