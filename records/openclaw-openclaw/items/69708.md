## Summary

Keep open. Current `main` still rejects `auth-profiles.json` entries whose normalized `type` is `aws-sdk`, while the Bedrock runtime and docs still expect `auth: "aws-sdk"` to use the AWS SDK default credential chain for EC2/IMDS. The linked #69753 is useful implementation context, but it is closed and unmerged.

## What This Changes

The issue requests that persisted Bedrock auth profile entries with `type: "aws-sdk"` stop being rejected as invalid so EC2/IMDS AWS SDK credential-chain auth continues to work.

## Best Possible Solution

Land a narrow auth-profile store fix in `src/agents/auth-profiles/persisted.ts`: recognize `aws-sdk` as an SDK-managed marker during store load, skip it without storing an `AuthProfileCredential`, exclude only that marker from invalid-entry warnings, keep warnings for real invalid entries, add focused regression tests, and add an Unreleased changelog fix entry crediting the reporter and useful #69753 context.
