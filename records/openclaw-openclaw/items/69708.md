## Summary

Keep open. Current main still rejects persisted auth-profile entries whose normalized `type` is `aws-sdk`, while the Bedrock runtime path and docs still expect `auth: "aws-sdk"` to use the AWS SDK default credential chain. The related fix PR #69753 was closed unmerged, so this is still a valid narrow bug.

## Best Possible Solution

Implement a narrow auth-profile store fix that treats `aws-sdk` as an SDK-managed non-credential marker during load, skips it without warning, preserves existing warnings for genuine invalid entries, and leaves Bedrock runtime auth resolution on the provider config/AWS SDK default-chain path. Add focused regression tests and a user-facing changelog fix entry.
