## Summary

Keep open. Current main still lacks the requested Discord voice-to-text-session bridge: voice joins resolve a session from the voice channel id, voice ingress calls the agent with `deliver: false`, and the Discord voice config/schema/docs expose no `bridgeToText`, bridge channel, or text-reply fan-out option. Related #53562/#53741 overlap on text-channel routing but are narrower than this issue’s full transcript-posting plus text-and-TTS reply behavior.

## Best Possible Solution

Keep this open and implement the bridge in the bundled Discord plugin voice path: add typed/schema/docs config for bridge mode and target text channel, resolve voice input against the chosen text-channel session, post speaker-attributed transcripts, dispatch replies through normal Discord text delivery, and also enqueue the final reply for TTS playback. Coordinate with #53562/#53741 if maintainers choose to make that route work the canonical base.
