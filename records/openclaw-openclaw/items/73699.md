## Summary

Keep open. Current main still lacks the requested Discord voice-as-I/O bridge: voice auto-join only carries the voice guild/channel, the voice session route is still keyed from the joined voice channel, voice turns still use `deliver: false`, and the Discord voice config/schema/docs expose no bridge-to-text fields. The maintainer discussion explicitly leaves this as a dedicated feature, while related #53562/#53741 are narrower session-channel routing work.

## Best Possible Solution

Keep this open for a dedicated bundled Discord plugin feature. The best implementation path is typed config/schema/docs for bridge mode and bridge target selection, routing STT into the selected text-channel session, speaker-attributed transcript posting, normal Discord text delivery for the assistant reply, and enqueueing that same final reply for voice TTS. Coordinate with #53562/#53741 if their `sessionChannelId` work becomes the base, but do not close this tracker until the broader bridge behavior and security/product semantics are implemented or a maintainer chooses a canonical replacement.
