## Summary

Keep open. Current main still does not implement the requested Discord voice-as-I/O bridge: voice auto-join only joins a voice channel, the voice session route is still keyed from that voice channel, voice turns call `agentCommandFromIngress` with `deliver: false`, and the Discord voice config/schema/docs do not expose bridge-to-text fields. The maintainer discussion also explicitly leaves this open as a dedicated feature; related #53562/#53741 are narrower session-channel routing work, not a full replacement.

## Best Possible Solution

Keep this open for a dedicated bundled Discord plugin feature. The best path is to add typed config/schema/docs for bridge mode and bridge target selection, route STT into the selected text-channel session, post speaker-attributed transcripts, deliver the assistant reply through normal Discord text delivery, and enqueue that same final reply for voice TTS. Coordinate with #53562/#53741 if `sessionChannelId` becomes the base, but do not close this broader tracker until the full voice-as-I/O behavior is implemented or a maintainer chooses a canonical replacement.
