## Summary

Keep this issue open. Current main still only partially satisfies the request: the Control UI now has cron editing, run history, delete-after-run controls, Web Push subscription plumbing, cron finished Gateway events, and an inline high-context Compact notice, but it still lacks the central native in-app toast/popup notification layer, actionable cron result popups, and snooze/reschedule controls requested here.

## Best Possible Solution

Keep this issue open and split or explicitly scope the remaining work. The strongest next path is a native Control UI in-app notification/toast layer, Gateway cron finished events wired into actionable result notifications, validated snooze/reschedule controls, and a maintainer decision on whether context pressure should remain inline-only or also emit toast-style Compact prompts.
