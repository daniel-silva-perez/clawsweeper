## Summary

Keep open. Current main still only partially covers the request: Control UI cron editing, delete-after-run, run history, Web Push subscription/test plumbing, and an inline context-pressure compact notice exist, but there is still no generic in-app toast queue, actionable cron-result popup layer, or snooze/reschedule UI flow. ## What This Changes

Request a native Control UI notification/toast system with actionable cron-result popups, inline cron management including one-shot and snooze controls, and integrated context usage warnings with one-click compaction.

Define a connected-Control-UI notification event and queue contract, integrate cron-finished events into actionable result notifications, specify durable snooze/reschedule behavior, and decide whether context pressure remains inline-only or also emits notification prompts before implementing the work in focused slices with tests and docs.
