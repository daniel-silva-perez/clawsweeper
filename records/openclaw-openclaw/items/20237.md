## Summary

Keep this issue open. Current main partially covers the request with cron editing, delete-after-run, run history, backend cron-finished notification/delivery paths, Web Push subscription plumbing, and an inline context-pressure Compact notice, but it still lacks the central in-app Control UI toast/notification queue, actionable cron-result popup layer, and snooze/reschedule controls.

## What This Changes

The issue requests a native Control UI notification/toast system with actionable cron-result popups, inline cron management including delete-after-run and snooze/reschedule, plus context usage warnings with one-click flush/compact.

## Best Possible Solution

Keep the issue open and scope it into focused product work: a generic connected-Control-UI notification queue, cron-finished event integration with actionable result notifications, validated snooze/reschedule semantics, and a clear decision on whether context pressure remains inline-only or also emits notification prompts.
