## Summary

Keep open. Current main partially covers the request with cron editing, delete-after-run UI, run history, Web Push subscription/test plumbing, and an inline context pressure compact notice, but it still lacks the central generic in-app toast queue, actionable cron-result popup layer, and snooze/reschedule UI flow.

## What This Changes

Request to add a native Control UI toast/notification system with actionable cron result popups, inline cron management and snooze controls, one-shot cron exposure, and context pressure warnings with one-click compact/flush.

## Best Possible Solution

Define a connected-Control-UI notification event and queue contract, integrate cron-finished events into actionable result notifications, specify durable snooze/reschedule behavior, and decide whether context pressure remains inline-only or also emits notification prompts before implementing focused slices with tests and docs.
