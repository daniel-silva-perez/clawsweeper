## Summary

Keep this issue open. Current main still only partially covers the request: cron editing, delete-after-run, run history, cron finished delivery hooks, Web Push subscription plumbing, and an inline context-pressure compact notice exist, but there is still no generic in-app Control UI toast queue, no actionable cron-result popup layer, and no snooze/reschedule UI flow.

## What This Changes

Add a native WebUI notification/toast system with actionable cron job result popups, snooze/reschedule controls, inline cron management including one-shot jobs, and context usage warnings with one-click flush/compact.

## Best Possible Solution

Scope this into focused product work: a generic connected-Control-UI notification queue, cron-finished event integration with actionable result notifications, explicit snooze/reschedule semantics, and a decision on whether context pressure remains inline-only or also emits notification prompts.
