## Summary

Keep open. Current main still only partially satisfies the request: Control UI cron editing, delete-after-run, run history, Web Push subscription plumbing, cron finished backend delivery hooks, and an inline context-pressure compact notice exist, but there is still no generic in-app Control UI toast queue, no actionable cron-result popup layer, and no snooze/reschedule UI flow.

## What This Changes

Add a native Control UI notification/toast system with actionable cron job result popups, snooze/reschedule controls, inline cron management, one-shot cron UI, and context pressure warnings with a flush/save action.

## Best Possible Solution

Scope this into focused product work: define a connected-Control-UI notification event/queue contract, integrate cron-finished events into actionable result notifications, specify snooze/reschedule semantics, and decide whether context pressure remains inline-only or also emits notification prompts. Then implement the chosen slices with UI, Gateway, docs, and targeted regression coverage.
