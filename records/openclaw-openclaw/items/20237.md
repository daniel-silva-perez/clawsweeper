## Summary

Keep open. Current main still only partially satisfies the request: cron editing, delete-after-run, run history, backend cron-finished events, Web Push subscription plumbing, and an inline context-pressure compact notice exist, but the requested generic in-app toast queue, actionable cron-result popup layer, and snooze/reschedule UI remain missing. The existing issue discussion also points to related open notification/action work rather than cleanup-ready closure.

## What This Changes

Add a native Control UI notification/toast system for cron results, inline cron management and snooze controls, one-shot cron exposure, and context-usage warnings with a flush or compact action.

## Best Possible Solution

Define a connected-Control-UI notification event and queue contract, integrate cron-finished events into actionable result notifications, specify snooze/reschedule semantics, and decide whether context pressure remains inline-only or also emits notification prompts. Then implement the chosen slices with UI, Gateway, docs, and targeted regression coverage.
