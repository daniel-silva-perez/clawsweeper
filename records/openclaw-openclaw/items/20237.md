## Summary

Keep open. Current main has partial coverage, but the central user-visible notification work is still missing: cron events refresh the cron tab instead of entering a global in-app notification surface, and there is no actionable cron-result popup or snooze/reschedule flow.

## What This Changes

The issue requests a native Control UI notification/toast system with actionable cron result popups, inline cron scheduling controls, one-shot job exposure, snooze/reschedule controls, and context-pressure notifications with compact/flush actions.

## Best Possible Solution

Define a connected-Control-UI notification event and queue contract, integrate cron-finished events into actionable result notifications, specify persisted snooze/reschedule behavior, and decide whether context pressure should stay inline-only or also emit notifications before implementing focused slices with docs and tests.
