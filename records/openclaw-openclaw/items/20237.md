## Summary

Keep open. Current main still only partially satisfies the request: cron editing/history/delete-after-run, Web Push subscription/test plumbing, and an inline context pressure compact notice exist, but there is still no generic in-app Control UI toast queue, no actionable cron-result popup layer, and no cron snooze/reschedule UI flow.

## What This Changes

The issue requests a native WebUI notification/toast system with actionable cron-result popups, inline cron management including one-shot jobs and snooze/reschedule controls, plus context usage warnings with flush/save actions.

## Best Possible Solution

Define a connected-Control-UI notification event and queue contract, integrate cron-finished events into actionable result notifications, specify persisted snooze/reschedule behavior, and decide whether context pressure should also emit notification prompts before implementing the work in focused slices with tests and docs.
