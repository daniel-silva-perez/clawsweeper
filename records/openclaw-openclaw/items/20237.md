## Summary

Keep open. Current main still only partially covers the requested experience: cron editing/history/delete-after-run, Web Push subscription/test plumbing, and an inline context-pressure compact notice exist, but the generic in-app toast queue, actionable cron-result popup layer, and snooze/reschedule UI remain missing.

## What This Changes

Add a native Control UI notification/toast system with cron-result management popups, snooze/reschedule controls, one-shot cron UI exposure, and context-pressure alerts with flush/compact actions.

## Best Possible Solution

Define a connected-Control-UI notification event and queue contract, integrate cron-finished events into actionable result notifications, specify persisted snooze/reschedule behavior, and decide whether context pressure should also emit notification prompts before implementing the work in focused slices with tests and docs.
