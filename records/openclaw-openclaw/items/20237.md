## Summary

Keep open. Current main partially covers the request with cron editing, delete-after-run, run history, backend cron events, Web Push plumbing, and an inline context-pressure compact notice, but it still lacks the requested generic in-app toast queue, actionable cron-result popup layer, and snooze/reschedule UI.

## What This Changes

Request to add Control UI toast/popup notifications for cron results, inline cron management with snooze/delete-after-run controls, and context-usage warnings with a compact/flush action.

## Best Possible Solution

Define a connected-Control-UI notification event and queue contract, integrate cron-finished events into actionable result notifications, specify persisted snooze/reschedule semantics, and decide whether context pressure remains inline-only or also emits notification prompts before implementing focused slices with docs and regression coverage.
