## Summary

Keep this issue open. Current main has partial coverage: Control UI cron edit/run/history/delete-after-run, Web Push subscription support, Gateway cron finished events, and an inline high-context notice with Compact. The central request is still not implemented: a native in-app toast/popup layer, actionable cron result popups, and snooze/reschedule controls.

## Best Possible Solution

Keep this issue open and split or explicitly scope the remaining work. The best implementation path is a native Control UI in-app notification/toast layer, cron finished events wired into actionable result notifications, validated snooze/reschedule controls, and a maintainer decision on whether context pressure should stay inline-only or also emit toast-style Compact prompts.
