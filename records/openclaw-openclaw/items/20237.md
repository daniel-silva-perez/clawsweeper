## Summary

Keep open. Current main partially covers the request with cron editing/history/delete-after-run, Web Push subscription plumbing, and an inline context pressure compact notice, but it still lacks the central in-app toast queue, actionable cron-result popup layer, and snooze/reschedule UI flow.

## What This Changes

The issue requests a native Control UI notification/toast system with actionable cron-result popups, inline cron management including delete-after-run and snooze/reschedule, plus context usage warnings with one-click flush.

## Best Possible Solution

Define a connected-Control-UI notification event and queue contract, integrate cron-finished events into actionable result notifications, specify persisted snooze/reschedule behavior, and decide whether context pressure stays inline-only or also emits notification prompts before implementing focused slices with tests and docs.
