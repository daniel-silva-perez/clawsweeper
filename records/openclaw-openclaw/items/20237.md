## Summary

Keep open. Current main covers several pieces of the request, but the central UX remains missing: there is no generic Control UI in-app toast queue, no actionable cron-result popup layer, and no snooze/reschedule flow. The existing issue discussion and related open notification work in #53228 point to coordinated product scoping rather than conservative cleanup closure.

## What This Changes

The issue requests a WebUI notification/toast system with actionable cron result popups, cron snooze/reschedule and inline management, one-shot cron UI exposure, and context usage warning integration.

## Best Possible Solution

Define a connected-Control-UI notification event and queue contract, integrate cron-finished events into actionable result notifications, specify snooze/reschedule semantics, and decide whether context pressure remains inline-only or also emits notification prompts. Then implement the selected slices with UI, Gateway, docs, and targeted regression coverage.
