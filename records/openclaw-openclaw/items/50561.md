## Summary

Keep open. Current main still does not auto-apply general safe doctor fixes during ordinary gateway startup: startup performs targeted recovery/degraded loading and emits manual doctor hints, while the broader doctor repair sequence remains gated behind explicit repair/yes modes. The discussion adds credible safety counterexamples from a startup `doctor --fix` workaround, so this needs maintainer policy design before an automated fix PR is safe.

## What This Changes

Request to auto-apply safe, non-destructive doctor fixes when the gateway starts, or expose an opt-in setting while logging what changed.

## Best Possible Solution

Define a narrow startup repair contract first: classify which doctor mutations may run unattended, exclude sandbox/security/auth/allowlist/channel-availability/plugin-disablement changes unless explicitly opted in, log every changed config path with backup/recovery details, and keep broader repair behind manual `openclaw doctor --fix` or a clearly named opt-in setting.

## Security Review

Status: needs_attention

Summary: Unattended gateway-start repair is security-sensitive because it could silently change sandbox, auth, allowlist, or channel/plugin availability settings.

Concerns:

- **[medium] Define unattended repair safety policy first:** `src/commands/doctor/repair-sequencing.ts:52`
  - body: `runDoctorRepairSequence` can apply channel repair, allowlist/open-policy repair, stale plugin cleanup, invalid plugin quarantine, and exec safe-bin profile repair. Before any gateway-start auto-fix, maintainers should require an explicit safe-mutation allowlist, audit logging, backups, and opt-in for security or availability changes.
  - confidence: 0.86
- **[medium] Avoid silent plugin or channel disablement:** `src/commands/doctor/shared/invalid-plugin-config.ts:59`
  - body: Invalid plugin config repair deletes the plugin config payload and sets the plugin entry disabled. That is appropriate for explicit `doctor --fix`, but would be risky as an unattended startup mutation without a protected/intentional config policy.
