## Summary

Keep open. Current main still has the `update.run` producer gap: restart continuations are consumed and tested when present, and the agent restart path can create one, but `update.run` does not accept or write a `continuation`, and its tests still assert the field is absent. The closed, unmerged #71573 remains useful implementation context rather than proof that the issue is fixed.

## Best Possible Solution

Keep this issue open and land a narrow, additive `update.run` continuation fix: accept `continuationMessage` in the update protocol, parse it through the shared restart request helper, write `buildRestartSuccessContinuation({ sessionKey, continuationMessage })` into successful update sentinels, forward the field from the agent gateway tool, and update the tests that currently assert `continuation` is absent. After that lands, maintainers should decide whether graceful drain/fallback and post-reset bridge behavior remain here or move to narrower follow-ups such as #73918.
