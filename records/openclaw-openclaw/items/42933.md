## Summary

Keep open. Current main still does not implement the PR's intended configurable session-level circuit breaker. Main has adjacent protections such as provider/auth-profile cooldowns, cron retry backoff/failure alerts, tool-loop detection, heartbeat duplicate suppression, and context-overflow compaction guidance, but it lacks the requested `agents.defaults.circuitBreaker` / per-agent config, strict schema support, persisted per-session `cb*` state, and pause/reset/alert enforcement across heartbeat, cron, and interactive reply paths.

## Best Possible Solution

Keep this PR open and ask for a focused revision or superseding core implementation. The best path is a core-owned session-level breaker with typed `agents.defaults.circuitBreaker` and per-agent schema, persisted per-session breaker state, structured model-failure metadata from interactive reply, heartbeat, and cron execution, correct isolated-heartbeat session targeting, `/new`-equivalent reset behavior, sanitized external alert delivery, and targeted tests for heartbeat, cron, and interactive reply paths.
