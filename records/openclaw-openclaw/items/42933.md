## Summary

Keep open. Current main at 891c7d9f1c1f5e0f52935fadfeacf695b0ef1eb8 still does not implement this PR's intended configurable session-level circuit breaker. Main has adjacent mitigations for provider/auth-profile cooldowns, cron failure backoff, tool-loop protection, heartbeat duplicate suppression, and context-overflow recovery, but it lacks the requested `agents.defaults.circuitBreaker` / per-agent config, strict schema support, persisted per-session `cb*` state, and pause/reset/alert enforcement across heartbeat, cron, and interactive reply paths.

## Best Possible Solution

Keep this PR open for maintainer follow-up or a superseding core implementation. The best path is a core-owned session-level breaker with documented `agents.defaults.circuitBreaker` and per-agent schema, persisted per-session breaker state, structured model-failure metadata from interactive reply, heartbeat, and cron execution, correct isolated-heartbeat ownership, `/new`-equivalent reset behavior, sanitized external alert delivery, and targeted tests for all three execution paths.
