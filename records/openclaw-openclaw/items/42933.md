## Summary

Keep open. Current main still lacks the requested session-level model-error circuit breaker, and the PR remains a useful implementation candidate, but the proposed patch is not merge-ready because cron and heartbeat paths still miss important failure cases and interactive replies are not wired despite the PR's own design scope.

## What This Changes

The branch adds a new agent/session circuit-breaker module with config, state, tests, SessionEntry cb* fields, agents defaults/per-agent schema fields, and pause/reset/alert handling in heartbeat and isolated cron paths.

## Best Possible Solution

Keep the PR or a narrower replacement open until OpenClaw has a core-owned session-level breaker with documented config, persisted per-session state, structured model-failure metadata across interactive reply, heartbeat, cron, and auto-resume paths, correct isolated-session ownership, and generated config/docs updates.

## Reproduction Assessment

Yes. Current-main source inspection reproduces the missing feature by showing no circuitBreaker schema/type/session fields and no heartbeat or cron enforcement; the PR-specific bugs are statically reproducible from the proposed diff plus current runner tests showing error payloads without meta.error.

## Solution Assessment

No, not as currently proposed. The best implementation should use structured error metadata rather than partial text/meta heuristics, cover getReplyFromConfig and auto-resume, and align reset/pause semantics with main vs isolated session ownership before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Record cron error payloads before clearing breaker state:** `src/cron/isolated-agent/run.ts:779-794`
  - body: This branch clears the breaker whenever `finalRunResult.meta?.error` is absent, but embedded cron runs can still return fatal timeout/compaction failures as `payloads[].isError` with no `meta.error`. Those runs are later treated as job errors, yet this code would erase the accumulated counter, so repeated timeouts never trip the breaker.
  - confidence: 0.86
- **[P2] Wire the interactive reply path or narrow the design:** `docs/superpowers/specs/2026-03-11-session-circuit-breaker-design.md:87-92`
  - body: The new design lists `getReplyFromConfig()` as a required integration point, but the patch only wires heartbeat and isolated cron. Normal inbound replies can hit the same repeated model-error loop and will bypass pause/reset/alert entirely, so the feature is not actually session-level as documented.
  - confidence: 0.82
- **[P2] Avoid resetting the main session for isolated heartbeat failures:** `src/infra/heartbeat-runner.ts:746-754`
  - body: When `heartbeat.isolatedSession` is enabled, the model call runs against a fresh `:heartbeat` session, but this block records and executes actions against the preflight main `entry` and `sessionKey`. A breaker `reset` can therefore rotate the user's real conversation because only an ephemeral heartbeat run failed; the breaker needs to target the actual run session or define a separate base-session policy.
  - confidence: 0.74
