## Summary

Keep this PR open because current main still does not implement the PR's central behavior: update.run failure sentinels only include a doctor hint, restart sentinel payloads cannot carry or format a doctor summary, and config-reload restart-check failures only log and retry. The related issues do not supersede the contribution: #19992 is closed for broader config safety work, while #55347, #69755, and #50561 remain broader or adjacent requests rather than merged replacements.

## What This Changes

This PR adds a bounded non-interactive doctor-summary path for update.run failures, extends restart sentinel payload formatting/tests for doctor summaries, and attempts to add doctor-backed diagnostics around config-reload restart failure paths.

## Best Possible Solution

Keep the PR open for maintainer review or author refresh. The desired end state is a focused, safe diagnostic UX: failed update.run calls attach a concise bounded doctor summary to the restart sentinel/system event, restart sentinel formatting can display that summary, and config-reload restart-check failures either get an immediate non-stale diagnostic path or are explicitly left to a broader self-healing design such as #55347.
