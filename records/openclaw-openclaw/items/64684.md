## Summary

Keep this PR open for maintainer review or author refresh. Current main still lacks the proposed doctor-summary persistence/formatting and config-reload diagnostics, while the PR branch is stale against the current update handler and still has concrete correctness issues in the latest diff.

## What This Changes

This PR adds a bounded non-interactive doctor-summary subprocess for failed `update.run` results, extends restart sentinel payload formatting/tests for `doctorSummary`, and updates related test mocks; its latest net diff does not change config-reload production behavior.

## Best Possible Solution

Refresh the PR onto current main, keep the focused doctor-summary UX only where it is timeout-bounded and fixed-argv, fix early update-failure diagnostics and the duplicate mock key, and either implement a non-stale config-reload diagnostic path or explicitly defer that broader self-healing scope to #55347.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Use a fallback entry for early update failures:** `src/gateway/server-methods/update.ts:76`
  - body: If `loadConfig()` or root resolution throws before `root` is assigned, `result.root` stays undefined and this call passes `entry: undefined`, so `runDoctorNonInteractiveSummary` returns `null`. That leaves early `update.run` failures without the doctor summary this PR is trying to add; set a fallback root/entry before the try or pass a fallback entry here.
  - confidence: 0.78
- **[P2] Remove the duplicate runtime mock key:** `src/infra/outbound/target-resolver.test.ts:30`
  - body: This mock object now defines `getActivePluginChannelRegistry` twice. TypeScript/lint gates should reject the duplicate key, and if it transpiled the later entry would silently shadow the first mock, so the stale entry needs to be removed.
  - confidence: 0.9
