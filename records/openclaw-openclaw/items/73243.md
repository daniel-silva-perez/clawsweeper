## Summary

Keep this PR open. Current main does not implement the PR's intended stuck-session recovery: diagnostics still only warn on stuck `processing` sessions, while the abort config, `session.aborted` event, recovery callback, queue clearing, and regression tests are absent. The PR is an active implementation candidate for open #71127 and overlaps with open #73510, so the best path is maintainer review and validation, not cleanup closure.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #71127, with #73510 as overlapping production evidence. Maintainers should review the latest head, verify the async recovery fixes and abort-threshold semantics, confirm the config/event/docs surface, run the diagnostics/gateway/session-abort changed lanes, and land or revise the PR rather than closing it as obsolete.
