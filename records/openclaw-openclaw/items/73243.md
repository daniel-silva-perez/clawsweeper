## Summary

Keep this PR open. Current main still has warning-threshold diagnostics plus conservative stale-lane recovery, but it does not implement the PR's central behavior: `diagnostics.stuckSessionAbortMs`, active stuck-run aborts, `session.aborted` telemetry, and diagnostic state transition back to idle after abort. The PR remains useful for linked stuck-session reports, but it needs maintainer review and rework on top of main's newer recovery runtime before it is safe to land.

## What This Changes

The PR adds a `diagnostics.stuckSessionAbortMs` config/schema surface, `session.aborted` diagnostic telemetry, stuck-session abort/recovery handling, gateway reload handling, and diagnostics tests for abort, retry, and race behavior.

## Best Possible Solution

Keep this PR open and review it as an implementation candidate for #71127. The best path is to rebase onto current main's conservative stale-session recovery runtime, decide the active-abort product policy, move `session.aborted` emission behind the race guard, align docs/changelog/generated config surfaces, and validate diagnostics/recovery tests plus the changed gate before landing or replacing it.
