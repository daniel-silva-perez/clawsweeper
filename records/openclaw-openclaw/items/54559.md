## Summary

Keep open. Current main still matches the reported gap: `generic_repeat` is warning-only, warning-level detections do not block tool execution, and `process` with `action: "list"` is still outside the known-poll path. PR #60248 remains the open implementation candidate rather than merged behavior.

## What This Changes

The issue asks for `tools.loopDetection` to let the `generic_repeat` detector escalate to critical/blocking at `criticalThreshold`, and to consider `process(action="list")` plus a broader tool-only-turn circuit breaker.

## Best Possible Solution

Land or replace the scoped loop-detection fix so `generic_repeat` can block at `criticalThreshold` when repeated calls show no progress, update unit/e2e expectations plus docs/config help, and split `process(action="list")` and any tool-only-turn breaker into explicit follow-up decisions if they are not covered by the same PR.
