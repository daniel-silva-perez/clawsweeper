## Summary

Current main still matches the reported gap: generic_repeat only returns warning, warning-level loop detections do not block tool execution, and process action=list is still outside known poll handling. PR #60248 is an open, unmerged implementation candidate that references this issue, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue requests genericRepeat loop detection to escalate to critical/blocking at criticalThreshold and asks whether process action=list should be treated as a known polling call.

## Best Possible Solution

Land or replace the scoped loop-detection fix so generic_repeat can become critical at criticalThreshold when repeated calls show no progress, update the unit/e2e expectations and public docs/config help, and separately decide the process action=list classification plus any broader tool-only-turn circuit breaker follow-up.
