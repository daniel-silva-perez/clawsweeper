## Summary

Keep open. Current main still lacks the PR’s intended fallback image policy, prompt-image suppression, and failure-context threading, while the PR itself still needs maintainer review because its partial-execution replay protection is only plumbing and the branch is dirty against current main.

## What This Changes

The PR adds failure-aware fallback image forwarding, cross-provider prompt-image detection suppression, and partial-execution context plumbing across failover errors, model fallback options, agent attempts, embedded-runner params, and tests.

## Best Possible Solution

Land a rebased, maintainer-reviewed agent fallback fix that preserves same-provider images only for safe retry modes, suppresses prompt-detected images across provider boundaries, and either populates partialExecution from real runner failure sites or narrows that feature out of the PR before merge.
