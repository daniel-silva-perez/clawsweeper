## Summary

Current main addresses the reported QMD memory-search CLI hang. The CLI search path now uses a one-shot `purpose: "cli"` memory manager, QMD CLI managers return before starting watchers/background update timers, subprocess searches are bounded by QMD timeouts, and the manager is closed in a `finally` path. The follow-up comment usefully narrowed the problem to OpenClaw wrapper overhead rather than raw QMD; the current wrapper path has since been changed and shipped in v2026.4.26.

## Best Possible Solution

Keep the shipped transient one-shot memory CLI manager path. If users still see memory search failures on current releases, route remote embedding socket failures to the separate open #71784 thread, or ask for a new focused QMD report with current OpenClaw version, QMD version, searchMode, logs, and whether the process hangs before output or only exits slowly after output.
