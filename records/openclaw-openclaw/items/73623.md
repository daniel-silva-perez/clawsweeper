## Summary

Keep open. Current main still writes `plugins.entries.<id>` before proving the requested plugin exists, so this PR's intended fix is not implemented on main. The PR is a plausible implementation candidate for linked #73551, but it still needs review/fixup before landing because the posted diff validates display names while forwarding the raw user input to config mutation helpers.

## Best Possible Solution

Keep this PR open for review. Land a corrected version that resolves the requested plugin through `buildPluginSnapshotReport` before any config write, forwards the canonical `targetPlugin.id` to enable/disable and slot selection, exits 1 and skips `replaceConfigFile`/registry refresh on a miss, and adds regression coverage for missing ids plus display-name-to-canonical-id behavior. Keep linked #73551 open until the PR or an equivalent fix merges.
