## Summary

Keep this PR open. Current main still does not implement the requested pre-write registry validation for `plugins enable` or `plugins disable`, and the PR is still a focused implementation candidate for open bug #73551, but the posted patch needs the display-name/canonical-id review defect fixed before it is mergeable.

## Best Possible Solution

Keep this PR open for maintainer review and fixup. The best path is to land a corrected implementation that resolves the requested plugin through the registry before any config write, exits non-zero without `replaceConfigFile` or registry refresh on a miss, forwards the canonical `targetPlugin.id` to enable/disable and slot selection, and adds regression coverage for missing ids plus display-name-to-canonical-id behavior. Keep linked #73551 open until this PR or an equivalent fix merges.
