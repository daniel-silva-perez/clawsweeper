## Summary

Keep this PR open. Current main still does not implement the requested pre-write validation for `plugins enable` or `plugins disable`, and the posted PR remains a focused implementation candidate for open bug #73551, but it needs the existing review defect fixed before merge.

## Best Possible Solution

Keep this PR open for maintainer review and fixup. The best path is to land a corrected implementation that resolves the requested plugin through the registry before any config write, exits 1 without `replaceConfigFile` or registry refresh on a miss, forwards the canonical `targetPlugin.id` to enable/disable and slot selection, and adds regression coverage for missing ids plus display-name-to-canonical-id behavior. Keep linked #73551 open until this PR or an equivalent fix merges.
