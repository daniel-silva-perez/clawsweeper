## Summary

Keep this PR open. Current main still lacks the requested pre-write registry validation for `plugins enable` and `plugins disable`, and the PR is a focused candidate for open bug #73551, but the posted diff needs fixup because name-based lookup can still forward the raw display name into config mutation.

## Best Possible Solution

Keep this PR open for maintainer review and fixup. The best path is to resolve the user argument through the plugin snapshot before any config write, exit 1 without `replaceConfigFile` or registry refresh on a miss, forward the canonical `targetPlugin.id` to enable/disable and slot selection on a match, and add regression coverage for missing ids plus display-name-to-canonical-id behavior while preserving valid enable/disable semantics.
