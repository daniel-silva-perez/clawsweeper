## Summary

Keep this PR open. Current main still lacks the requested pre-write registry validation for `plugins enable` and `plugins disable`, so the PR targets a real open bug, but the posted patch still needs the canonical-id fix noted in review before it is mergeable.

## Best Possible Solution

Keep this PR open for maintainer review and fixup. The best path is to resolve the requested plugin through the registry before any config write, exit non-zero without `replaceConfigFile` or registry refresh on a miss, pass the canonical `targetPlugin.id` into enable/disable and slot selection, and add regression tests for missing ids plus display-name-to-canonical-id behavior. Keep linked #73551 open until this PR or an equivalent fix merges.
