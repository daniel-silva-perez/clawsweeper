## Summary

Keep this PR open. Current main still lacks the requested pre-write registry validation for `plugins enable` and `plugins disable`, and the PR is tied to the still-open linked bug #73551. The proposed patch is directionally useful but needs the canonical-id follow-up from review before it is mergeable.

## Best Possible Solution

Keep this PR open for maintainer review and fixup. The best patch should resolve the requested plugin through the discovered registry before any config write, exit non-zero without `replaceConfigFile` or registry refresh on a miss, pass the canonical `targetPlugin.id` into enable/disable and slot selection, and add regression tests for missing ids plus display-name-to-canonical-id behavior. Keep linked #73551 open until this PR or an equivalent fix merges.
