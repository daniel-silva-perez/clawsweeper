## Summary

Keep open: this is an apparently correct, test-only PR whose intended memory-slot regression coverage is still absent from current main. Current main already has the runtime fallback, but closing this PR would drop the focused `memory_search` coverage; the remaining path is normal maintainer review and merge.

## What This Changes

The PR adds Gateway HTTP `/tools/invoke` regression tests that mock `memory_search` and `memory_get`, record tool-factory contexts, and assert plugin-enabled fallback for `memory_search` while ordinary known core tools stay plugin-disabled.

## Best Possible Solution

Merge the focused regression test after maintainer review, keeping the existing runtime fallback on main. If maintainers want stronger confidence later, add a separate real-plugin memory slot integration test rather than expanding this small HTTP unit test.

## Reproduction Assessment

Yes. The PR creates a deterministic unit reproduction by making `memory_search` disappear when plugin tools are disabled, then verifying `/tools/invoke` retries with plugin tools enabled and succeeds.

## Solution Assessment

Yes. Because production fallback behavior is already present on main, adding a narrow regression test in `src/gateway/tools-invoke-http.test.ts` is the most maintainable fix boundary for this PR.
