## Summary

Keep this PR open. Current main still does not implement the PR’s central docs change on the tools overview: it mentions `gateway` as owner-only, but it does not explain that non-owner senders lose owner-only tools from the model tool list, does not document the `cron`/`nodes` ownership requirement there, and does not give the `commands.ownerAllowFrom` setup/troubleshooting path. The PR is still useful, though it should be refreshed against current source/docs before merge.

## Best Possible Solution

Revise and rebase the PR rather than close it: add a concise owner-only tools section to `docs/tools/index.md` that names current core owner-only tools (`cron`, `gateway`, `nodes`), explains non-owner filtering from the model tool list, points to `commands.ownerAllowFrom` using channel-native sender IDs, and notes that plugin tools can also declare `ownerOnly` where relevant. Use obviously fake IDs and keep the slash-command/config docs as the detailed reference.
