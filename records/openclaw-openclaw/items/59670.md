## Summary

Keep open. The docs gap is still real on current main because the tools overview does not explain owner-only tool filtering or how `commands.ownerAllowFrom` restores access, but this PR is not correct as submitted: its table reads as exhaustive while omitting the bundled WhatsApp `whatsapp_login` tool that is also `ownerOnly`, and one example uses a realistic Discord snowflake-looking ID.

## What This Changes

The PR adds an owner-only tools section to `docs/tools/index.md` documenting `gateway`, `cron`, and `nodes`, `commands.ownerAllowFrom`, channel sender ID examples, sender ID discovery, and silent fallback troubleshooting.

## Best Possible Solution

Land a revised docs-only update on `docs/tools/index.md` that explains owner-only filtering, links to the existing `commands.ownerAllowFrom` config docs, uses clearly synthetic sender IDs, and either lists `whatsapp_login` with the bundled owner-only tools or clearly scopes the table to core tools while noting plugin-owned `ownerOnly` tools.
