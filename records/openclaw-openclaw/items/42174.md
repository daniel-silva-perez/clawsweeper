## Summary

Keep this PR open. Current main still rejects the reported Windows drive-letter Docker bind form during config validation and runtime sandbox validation. The PR is a focused implementation candidate, but the touched code is security-sensitive host bind policy and needs maintainer review plus regression coverage before it can land or be superseded.

## Best Possible Solution

Keep this PR open for maintainer review or replace it with a maintainer-owned patch. The fix should use the shared bind-spec parser in config validation, add a shared Windows-aware absolute host-path helper, preserve blocked-path and allowed-root enforcement for drive-letter paths, canonicalize existing ancestors safely on Windows, handle drive-letter case correctly, and add regression tests for accepted and rejected Windows bind cases.
