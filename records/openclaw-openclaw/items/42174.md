## Summary

Keep this PR open. Current main still rejects Windows drive-letter Docker bind sources in both config validation and runtime sandbox validation, while the PR addresses a concrete Windows Docker Desktop custom-bind failure. The PR also touches sandbox host-bind security policy, so the best path is maintainer review or a replacement patch with regression coverage rather than conservative closure.

## Best Possible Solution

Keep this PR open for maintainer review, or replace it with a maintainer-owned patch. The fix should reuse splitSandboxBindSpec in config validation, define a shared Windows-aware absolute-host-path and canonicalization policy, preserve allowed-root enforcement for Windows drive paths, add regression tests for uppercase/lowercase/no-mode binds and rejected dangerous or outside-root binds, and update sandbox docs only if native Windows Docker Desktop bind support is intended.
