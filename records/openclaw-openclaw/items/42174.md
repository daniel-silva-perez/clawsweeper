## Summary

Keep this PR open. Current main still rejects Windows drive-letter Docker bind sources in the same two relevant paths: config validation splits `D:/...` at the drive-letter colon, and runtime sandbox validation still requires bind sources to start with `/`. The PR has a concrete Windows Docker Desktop repro and only touches sandbox source files, but because it changes sandbox host-bind security policy, the right path is maintainer review or a replacement patch with Windows allowed-root and canonicalization coverage.

## Best Possible Solution

Keep this PR open for maintainer review, or replace it with a maintainer-owned patch. The fix should reuse `splitSandboxBindSpec` in config validation, define a shared Windows-aware absolute-host-path helper, preserve blocked-path and allowed-root enforcement for drive-letter paths, canonicalize existing ancestors safely on Windows, compare drive-letter paths with the right case semantics, and add regression tests for uppercase/lowercase drive paths with and without `:ro`/`:rw`, plus rejected outside-root and dangerous bind sources.
