## Summary

Keep this PR open. Current main still rejects the reported Windows Docker bind form in both config validation and runtime sandbox validation, and I did not find a merged replacement or current-main regression coverage accepting drive-letter bind sources. The PR is still relevant, but the host-bind policy is security-sensitive and should be reviewed or replaced with a Windows-aware implementation that preserves allowlist, blocked-path, canonicalization, and case semantics.

## Best Possible Solution

Keep this PR open for maintainer review, or replace it with a maintainer-owned patch. The implementation should reuse the existing bind-spec parser in config validation, add a shared Windows-aware absolute host-path helper, preserve blocked-path and allowed-root enforcement for drive-letter paths, canonicalize existing ancestors safely on Windows, compare drive-letter paths with correct case semantics, and add regression tests for accepted and rejected Windows bind cases.
