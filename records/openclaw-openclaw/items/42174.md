## Summary

Keep this PR open. Current main still rejects Windows drive-letter Docker bind sources at both config validation and runtime sandbox validation, so the intended fix is not implemented on main. The PR addresses a concrete Windows Docker Desktop custom-bind failure, but it broadens the sandbox host-bind security boundary and still needs maintainer review plus regression coverage before merge.

## Best Possible Solution

Keep this PR open for maintainer review, or replace it with a maintainer-owned patch. The best fix should reuse the existing bind-spec parser in config validation, introduce a shared Windows-aware absolute-host-path and canonicalization policy, preserve allowed-root enforcement for Windows paths, add regression tests for accepted and rejected Windows binds, and update sandbox docs only if native Windows Docker Desktop bind support is officially supported.
