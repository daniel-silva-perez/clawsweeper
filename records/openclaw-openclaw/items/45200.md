## Summary

Keep open. The PR targets a valid current subagent delivery bug, but the branch should not merge as-is because its last-resort notifier routes recovered text through another gateway `agent` turn instead of the existing direct outbound delivery seam, so the user may receive rewritten or missing output rather than the raw give-up notice.

## What This Changes

The branch adds a `notifyAnnounceGiveUp` helper in `src/agents/subagent-registry.ts` and calls it from resumed and live announce retry-exhaustion cleanup to send a best-effort requester notification.

## Best Possible Solution

Implement the give-up notice in the current lifecycle and announce-delivery layers using direct outbound `sendMessage` delivery when a deliverable requester origin exists, while preserving killed-run suppression, idempotency, swallowed failure semantics, cleanup bookkeeping, and route normalization.
