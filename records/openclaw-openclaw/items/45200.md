## Summary

Keep open. The reported user-visible gap still exists on current main, but this PR should not merge as-is because its fallback notification goes through the gateway `agent` method, which starts another requester-session agent turn instead of sending the recovered text directly to the requester channel.

## What This Changes

The PR adds `notifyAnnounceGiveUp` in `src/agents/subagent-registry.ts` and invokes it from retry-exhaustion cleanup paths to send a best-effort requester notification containing recovered subagent result text.

## Best Possible Solution

Implement the final give-up notice in the current lifecycle and announce-delivery layers, using the existing direct `sendMessage` fallback seam when a deliverable requester origin exists, while preserving killed-run suppression, idempotency, swallowed-failure semantics, task cleanup, retry-limit bookkeeping, and route normalization.
