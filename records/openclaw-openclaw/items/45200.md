## Summary

Keep open. The silent subagent announce give-up problem still exists on current main, but this PR should not merge as-is because its fallback runs a new requester agent turn instead of delivering the recovered give-up notice directly, so the result can still be rewritten, omitted, or fail through the same model/session machinery.

## What This Changes

The PR adds a `notifyAnnounceGiveUp` helper in `src/agents/subagent-registry.ts` and calls it from resumed and live announce retry-exhaustion paths to send a best-effort requester notification with recovered frozen result text.

## Best Possible Solution

Implement the give-up notice in the current lifecycle and announce-delivery layers using direct outbound `sendMessage` delivery when a deliverable requester origin exists, with focused tests for resumed and live retry exhaustion, missing origin, killed-run suppression, idempotency, and failed last-resort delivery logging.
