## Summary

Keep open. The reported user-visible bug is still relevant on current main: retry-limit cleanup records subagent delivery as failed and logs give-up, but it does not notify the requester. The PR is useful source work, but it targets the older monolithic registry shape and routes the final notice through the gateway `agent` method, which starts another requester-session agent turn instead of using the current raw outbound completion fallback seam.

## What This Changes

The PR adds `notifyAnnounceGiveUp` to `src/agents/subagent-registry.ts` and calls it from retry-limit cleanup paths to send a best-effort requester notification with recovered subagent result text.

## Best Possible Solution

Implement the final give-up notice on current main in the lifecycle/announce-delivery layer, using the existing raw outbound delivery seam for recovered result text when a deliverable requester origin exists, while preserving killed-run suppression, swallowed failure semantics, idempotency, task cleanup, and non-duplicating behavior.
