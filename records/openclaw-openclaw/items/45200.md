## Summary

Keep open. The user-visible failure mode is still present on current main, but this PR should not merge as-is because its fallback sends the recovered result through another gateway agent turn instead of the existing raw outbound delivery path, and the branch targets an older file layout than current main.

## What This Changes

The PR adds `notifyAnnounceGiveUp` in `src/agents/subagent-registry.ts` and invokes it from retry-limit cleanup paths to send a best-effort requester notification containing recovered subagent result text.

## Best Possible Solution

Implement the final give-up notice in the current lifecycle and announce-delivery layers, using the existing direct `sendMessage` fallback seam when a deliverable requester origin exists, while preserving killed-run suppression, idempotency, swallowed-failure semantics, task cleanup, and retry-limit bookkeeping.
