## Summary

Keep this PR open for maintainer handling. The config-validation idea is not fully present on current main, but the branch mixes that change with Slack and TTS edits; the Slack behavior is already implemented on main, and the TTS regex change would silently drop currently documented inline directives.

## What This Changes

The PR centralizes validation known-plugin-ID collection, adds plugin-validation tests, disables Slack Bolt `ignoreSelf` in socket mode with test coverage, and changes TTS directive parsing/tests to require standalone directive lines except compact text blocks.

## Best Possible Solution

Split the useful config-validation change into a narrow rebased PR that uses current manifest/metadata contracts and focused validation tests, while preserving current inline TTS directive parsing and dropping the already-shipped Slack hunk unless maintainers want separate coverage.

## Reproduction Assessment

Yes. A source-level reproduction is a reply like `Hello [[tts:provider=edge]] world` with provider overrides enabled: current main's parser matches and strips it, while the PR's newline-bounded `directiveRegex` would ignore it.

## Solution Assessment

No. Centralizing known-ID resolution may be a reasonable fix direction, but this branch is not the best implementation because it combines unrelated Slack and TTS behavior changes with the config work; a config-only rebase is safer.
