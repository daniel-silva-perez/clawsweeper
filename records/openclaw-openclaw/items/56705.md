## Summary

Keep this PR open for maintainer review: the config-validation refactor is plausible, but the branch mixes unrelated Slack and TTS behavior, and the latest PR diff still introduces a silent TTS inline-directive regression. Current main already has the Slack `ignoreSelf: false` behavior and a broader TTS parser, while the exact config helper is not present on main.

## What This Changes

The PR centralizes validation known-plugin-ID collection around bundled alias and auto-enable metadata, adds config validation coverage, disables Slack Bolt `ignoreSelf` in socket mode, and tightens TTS directive parsing/tests.

## Best Possible Solution

Split the useful config-validation idea into a narrow, rebased PR against the current plugin metadata/registry code, keep current inline TTS directive behavior, and drop the Slack hunk unless a maintainer wants additional tests for already-shipped `ignoreSelf: false` behavior.

## Reproduction Assessment

Yes. A source-level reproduction is `Hello [[tts:provider=edge]] world` with provider overrides enabled: current main's parser matches it, while the PR's newline-bounded `directiveRegex` does not.

## Solution Assessment

No. The central known-ID helper may be a reasonable direction, but mixing it with Slack and TTS behavior changes is not the narrowest maintainable fix; the safer path is a rebased config-only change plus separate tests if needed.
