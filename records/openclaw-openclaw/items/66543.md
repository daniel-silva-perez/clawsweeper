## Summary

Keep open. The reported cron model-ID corruption is still not fixed on current main, but this PR’s blanket whitespace deletion would regress supported model aliases with spaces across primary and fallback paths, including persisted jobs that are normalized during store hydration. Prior Greptile/Codex comments identified the same unsafe shape, and the merged #73946 explicitly left this model-ID whitespace decision separate.

## What This Changes

The PR branch strips all whitespace from cron `payload.model`, `payload.fallbacks`, legacy top-level `model`, and legacy top-level `fallbacks`, and adds a normalization unit test for corrupted provider/model refs.

## Best Possible Solution

Implement an alias-preserving repair in the model-resolution path rather than rewriting stored cron payloads: preserve trimmed model strings during normalization, resolve aliases first, and only try a compacted whitespace-repaired candidate when the raw provider/model reference fails. Cover payload and legacy top-level create/patch inputs for both primary model and fallbacks, plus persisted-load behavior.

  - confidence: 0.9

  - body: Top-level `fallbacks` can also contain aliases. Stripping spaces while copying them into the payload rewrites valid fallback aliases and prevents the fallback runner from resolving the intended models.
