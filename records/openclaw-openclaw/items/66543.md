## Summary

Keep open. The latest PR is moving in the right alias-preserving direction, but it still does not fully fix the reported cron model-ID corruption: the new compaction path can be skipped in allow-any model configs, and payload fallback resolution bypasses the new helper entirely.

## What This Changes

The PR branch moves cron model-ID whitespace repair from cron normalization into model resolution, adds alias-preserving slash-ref compaction in shared model selection, and adds focused cron/model tests while keeping normalization trim-only.

## Best Possible Solution

Use one alias-safe slash-ref repair path after exact alias lookup and before accepting corrupted provider/model refs, then share it across explicit cron primary model resolution and fallback candidate resolution. Keep cron normalization trim-only, preserve spaced aliases, and cover allowlist, allow-any, multi-segment OpenRouter/custom provider refs, nested payloads, legacy top-level fields, and persisted hydration paths.
