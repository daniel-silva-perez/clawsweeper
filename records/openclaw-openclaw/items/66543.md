## Summary

Keep this PR open. Current main still trim-normalizes cron model and fallback strings without repairing internal whitespace, so the PR's core bug is not implemented on main. The PR is still not ready as-is because its blanket whitespace removal would rewrite valid spaced model aliases before alias lookup and during persisted cron job hydration.

## Best Possible Solution

Keep the PR open for revision or supersede it with a narrower maintainer patch. The fix should preserve aliases after normal trim, repair accidental whitespace only for clear provider/model refs or after alias lookup fails, cover payload and top-level create/patch paths for both model and fallbacks, and add regression tests for the OpenRouter corruption case plus spaced primary and fallback aliases.

## Risks / Open Questions

- Landing this PR as-is would break valid primary model aliases with internal spaces, such as custom aliases or documented provider aliases, by rewriting them before alias lookup.
- Landing this PR as-is would also break fallback aliases and can remove the intended fallback chain for cron jobs.
- Because cron store hydration calls normalizeCronJobInput(raw), the PR would rewrite already-persisted cron jobs on read, not only newly created jobs.
- Closing this PR now would drop a real, still-unfixed OpenRouter cron model-ID corruption report and a useful regression-test candidate.
