## Summary

Keep open. The reported cron/OpenRouter model-ID corruption is valid and the latest PR direction avoids the earlier blanket alias breakage, but the patch is still incomplete: allow-any configurations can accept a whitespace-corrupted slash ref before repair runs, and payload fallback candidates still bypass the new repair helper.

## What This Changes

The PR adds an alias-preserving compacted slash-ref retry to shared model selection and focused cron/model tests while keeping cron normalization trim-only.

## Best Possible Solution

Use one alias-safe resolution-layer repair: exact alias lookup first, then compact accidental whitespace only for clear slash-form provider/model refs before accepting an uncataloged allow-any ref, and share that behavior across cron primary model overrides and fallback candidate resolution while leaving cron normalization trim-only.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Repair slash refs before allow-any success:** `src/agents/model-selection-shared.ts:809-813`
  - body: The compacted retry only runs after the parsed ref is invalid or not allowed. With an allow-any config, `custom-1/ minimax/ minimax-m2.5-free` parses and `status.allowed` is true, so this function returns the corrupted ref unchanged and the provider can still reject it. Try the compacted slash form after exact alias lookup and before accepting uncataloged allow-any refs.
  - confidence: 0.91
- **[P2] Resolve fallback candidates through the repair path:** `src/cron/isolated-agent/run.payload-fallbacks.test.ts:49`
  - body: The new test expects spaced payload fallbacks to pass through, but those values are later resolved with `resolveModelRefFromString`, which does not compact slash refs. Payload or configured fallbacks such as `custom-1/ minimax/ minimax-m2.5-free` can still be attempted with spaces; route fallback candidate resolution through the same alias-safe repair and assert the repaired candidate.
  - confidence: 0.9
