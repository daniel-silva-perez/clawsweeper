## Summary

Keep open. The cron model-ID bug is still valid on current main, but this PR's blanket whitespace deletion would break supported model aliases with internal spaces, including persisted jobs re-normalized during store hydration. It needs an alias-preserving repair before merge.

## What This Changes

The PR branch changes cron normalization so nested payload and legacy top-level `model` and `fallbacks` values have all whitespace removed, and adds a cron normalization unit test for whitespace-corrupted provider/model IDs.

## Best Possible Solution

Revise or supersede this PR with an alias-preserving repair: preserve trimmed values as authored, resolve aliases first, and only repair accidental whitespace for clear provider/model references when the raw value fails resolution. Cover nested payload and top-level create/patch paths for primary model and fallbacks, plus persisted-load/runtime behavior.

## Review Findings

Full review comments:

- **[P1] Preserve spaced model aliases in payloads:** `src/cron/normalize.ts:181`
  - body: `payload.model` may be an alias, and alias lookup lowercases while preserving internal spaces. Rewriting values such as `DeepSeek R1` to `DeepSeekR1` makes valid cron jobs fail, including persisted jobs that are hydrated through this normalization path.
  - confidence: 0.94
- **[P2] Preserve spaced fallback aliases in payloads:** `src/cron/normalize.ts:205`
  - body: Fallback entries use the same model-resolution machinery as the primary model. Removing all internal whitespace turns aliases such as `Qwen3 8B` into non-matching strings, so jobs can lose their intended fallback chain.
  - confidence: 0.92
- **[P2] Preserve spaced top-level model aliases:** `src/cron/normalize.ts:351`
  - body: The legacy top-level `model` path is still accepted for create/patch normalization. Applying blanket whitespace deletion there breaks valid alias inputs before the runtime resolver can match them.
  - confidence: 0.9
- **[P2] Preserve spaced top-level fallback aliases:** `src/cron/normalize.ts:366`
  - body: Top-level `fallbacks` patches can also contain aliases. Stripping spaces while copying them into the payload rewrites valid fallback aliases and prevents the fallback runner from resolving the intended models.
  - confidence: 0.9
