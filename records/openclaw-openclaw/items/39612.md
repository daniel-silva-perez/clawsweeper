## Summary

Keep open. Current main still permits provider/runner failure details to be sent to external direct chats when verbose mode is enabled. The reported Anthropic thinking-block failure class has newer transcript hygiene and retry mitigation, but the central request, never exposing raw LLM/API failure details to WhatsApp/Telegram users, is not implemented.

## Best Possible Solution

Keep this open for an explicit maintainer decision on external-channel failure policy. A likely fix is to keep generic direct-chat fallback copy and silent group/channel behavior, but stop sending provider or runner exception details to external chat users even when `/verbose` is enabled; reserve detailed errors for logs, Control UI, TUI/operator-only surfaces, or exported diagnostics. Add regression coverage for WhatsApp and Telegram direct verbose LLM failures. Treat multi-agent auto-escalation and deeper session migration/recovery as separate follow-ups if maintainers want those in scope.
