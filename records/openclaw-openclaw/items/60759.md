## Summary

Current main still has the reported schema composition bug. The bundled qqbot and webhooks manifests keep `$defs` inside their plugin `configSchema` while using root-relative `#/$defs/...` references, and the schema merger still embeds those schemas under plugin config entries without hoisting definitions or rewriting refs.

## What This Changes

The issue asks OpenClaw to generate config schemas with plugin JSON Schema `$defs` at resolvable locations instead of nesting them under qqbot config while refs point to the document root.

## Best Possible Solution

Land a generic schema-composition fix in `src/config/schema.ts` that hoists or namespaces plugin/channel root `$defs` into the generated document root and rewrites affected local refs, with regression coverage in `src/config/schema.test.ts`. The existing open fork PR can be reviewed as a source candidate, but this issue should close only after a fix lands on main.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: `openclaw config schema` reads manifest metadata, nests plugin `configSchema` under `plugins.entries.<id>.config`, and current qqbot/webhooks schemas still contain `#/$defs/...` refs without root-level generated definitions.

## Solution Assessment

No current solution is landed. The narrowest maintainable fix is a generic schema merger update plus tests; directly editing only qqbot/webhooks manifests is a partial workaround unless maintainers explicitly choose that scope.
