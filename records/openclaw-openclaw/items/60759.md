## Summary

Current main still appears to reproduce the reported schema-composition bug. The qqbot manifest still contains root-relative `$ref` values inside a plugin `configSchema`, and the config schema merger still nests that schema under `plugins.entries.<id>.config` without hoisting `$defs` or rewriting refs. Related fix attempts in the provided GitHub context were closed unmerged, and an open fork PR still references this issue with closing syntax, so this should stay open until an implementation lands.

## Best Possible Solution

Keep this open and land a real schema fix. The strongest path is a generic update in `src/config/schema.ts` that hoists or namespaces plugin/channel root `$defs` into the generated document root and rewrites affected `$ref` values, with regression coverage in `src/config/schema.test.ts` for both plugin and channel schemas. The linked open fork PR can be reviewed as an implementation candidate, but this issue should close only after a fix lands on main.
