## Summary

Keep this issue open. Current main still nests plugin `configSchema` objects under the generated config tree without hoisting `$defs` or rewriting root-relative `$ref` pointers, and the bundled qqbot and webhooks manifests still contain the exact `$defs`/`#/$defs/...` shape from the report. Related implementation attempts in the provided timeline are closed unmerged, while an open fork PR still references this issue with closing syntax, so the right path is to review or land a fix before closing.

## Best Possible Solution

Keep this open and land a real schema fix. The strongest path is a generic update in `src/config/schema.ts` that hoists or namespaces plugin/channel root `$defs` into the generated schema root and rewrites affected `$ref` pointers, with regression coverage in `src/config/schema.test.ts` for plugin and channel schemas. The open fork PR can be reviewed as an implementation candidate, but this issue should close only after a fix lands on main.
