## Summary

Keep open. Current main still lacks the requested Feishu card-action sibling-field pass-through, and the posted patch is stale against the current parser/decoder flow because live callbacks are normalized before they reach the handler branch changed by this PR. The paired issue #43202 and overlapping PR #42908 are still open, so maintainers should choose a canonical Feishu fix rather than close this PR.

## What This Changes

The PR extends Feishu card-action handling to include form, input, name, option, and options fields in synthetic message payloads and adds handler-level tests for those paths.

## Best Possible Solution

Consolidate this work with #43202 and #42908 into one Feishu-owned fix that preserves `form_value`, `input_value`, `name`, `option`, and `options` from the raw callback parser through the event type, decoder/fallback, and handler, with parser-to-handler tests that lock in both interactive-field delivery and existing text/command behavior.
