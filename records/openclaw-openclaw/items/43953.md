## Summary

Keep open. The reported Feishu behavior is still not implemented on current main, and this PR is not complete against the current pipeline because it only updates the handler-level event shape and tests; live `card.action.trigger` payloads are normalized before that handler and current main still reconstructs `action` with only `value` and `tag`. The author did address the earlier Greptile name-only/handler-test feedback, but related #43202 and overlapping #42908 remain open, while #73514 and #74191 were closed without merging.

## What This Changes

The PR extends Feishu card-action handling and handler tests so `form_value`, `input_value`, `name`, `option`, and `options` fields can be forwarded in the synthetic message payload when present.

## Best Possible Solution

Consolidate this with #43202 and #42908 into one Feishu-owned fix that preserves `form_value`, `input_value`, `name`, `option`, and `options` from the raw callback parser through the event type, decoder/fallback, and handler, with parser-to-handler tests that lock in both interactive-field delivery and existing text/command behavior.
