## Summary

Keep open. Current main still drops Feishu card-action sibling fields before handler dispatch, and this PR only updates the handler/test surface after those fields would already have been stripped by the live parser. The PR is a valid implementation candidate for #43202, but it needs parser-to-handler coverage and consolidation with overlapping Feishu work before merge.

## What This Changes

The PR extends Feishu card-action handling and tests so `form_value`, `input_value`, `name`, `option`, and `options` can be forwarded as structured synthetic message content when those fields are present.

## Best Possible Solution

Update the Feishu card-action pipeline end to end: preserve `form_value`, `input_value`, `name`, `option`, and `options` in the raw callback parser, model them in the event/decoder types, decide one backward-compatible payload shape, and add parser-to-handler tests that lock in both interactive form/select data and existing button `text`/`command` behavior.
