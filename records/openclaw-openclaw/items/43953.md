## Summary

Keep this PR open. Current main still drops Feishu card-action fields outside `action.value`, and the same-author issue #43202 remains open. The PR is a plausible implementation candidate, but it is stale against current main because live callbacks now pass through parser, decoder, and fallback layers that this patch does not update.

## Best Possible Solution

Keep this PR open as an active implementation candidate for #43202, or have maintainers choose one canonical Feishu card-action PR if consolidating with #42908. A current fix should extend the Feishu callback parser/event type to preserve `form_value`, `input_value`, `name`, `option`, and `options`, update the decoder/fallback flow, and add parser-to-handler tests while preserving existing `value.text` and `value.command` extraction.
