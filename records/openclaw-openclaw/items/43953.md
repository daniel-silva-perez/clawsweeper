## Summary

Keep this PR open. Current `main` still drops Feishu card-action fields outside `action.value`, and the same-author issue #43202 remains open. The PR is a plausible implementation candidate, but it is not obsolete or implemented on main; a current fix also needs to account for the live parser path, not only the handler type and fallback behavior.

## Best Possible Solution

Keep this PR open as an implementation candidate for #43202, or consolidate it with overlapping PR #42908 if maintainers want one canonical Feishu card-action fix. The best fix should extend the Feishu callback parser/event type plus handler/fallback flow so `form_value`, `input_value`, `name`, `option`, and `options` survive from raw callback to synthetic agent message, with parser-to-handler tests and compatibility coverage for existing text/command extraction.
