## Summary

Keep this issue open. Current main still has manual profile/port isolation, Docker Compose setup, ClawDock single-project helpers, and container-targeted CLI execution, but not the requested first-class Docker multi-instance lifecycle with named create/list/delete/update, managed per-instance metadata, safe port/path allocation, or compose project/container isolation. The related profile-management PR #54126 was closed unmerged and is not a shipped replacement.

## Best Possible Solution

Keep this issue open until maintainers either implement or explicitly decline a supported Docker multi-instance lifecycle. A good solution would add a documented openclaw instance command or Docker/ClawDock equivalent that records per-instance metadata, validates or allocates unique host ports and paths, isolates compose project/container names, and provides safe create/list/update/delete cleanup with Docker install docs and targeted tests.
