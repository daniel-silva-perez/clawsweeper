## Summary

Keep this issue open. Current main still documents and implements manual profile/port isolation plus Docker/ClawDock helper flows, but it does not provide the requested named Docker instance lifecycle, managed instance metadata, safe port/path allocation, or compose project/container isolation.

## What This Changes

The issue requests first-class Docker multi-instance lifecycle commands for creating, listing, updating, and deleting named OpenClaw instances with isolated directories, ports, and compose/container names.

## Best Possible Solution

Add or explicitly decline a supported Docker instance lifecycle after maintainer design review. A good implementation would record per-instance metadata, reserve or validate unique host ports and paths, isolate Compose project/container names, provide safe create/list/update/delete behavior with dry-run or confirmation for destructive operations, and update Docker/ClawDock docs plus targeted tests.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection on current main showing only manual profile/port/container-targeting surfaces and no named Docker instance lifecycle.

## Solution Assessment

Unclear. The proposed openclaw instance direction is plausible, but the best maintainable solution should be designed around existing profile, Docker setup, ClawDock, Podman, and gateway service primitives before code is queued.
