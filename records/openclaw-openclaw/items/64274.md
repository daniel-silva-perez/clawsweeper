## Summary

Current main still has the runtime auth snapshot identity problem: runtime snapshots are keyed through resolved auth-store paths, the default path still depends on OPENCLAW_AGENT_DIR, and the runtime resolver still collapses requested-agent and default snapshots when those paths match. MiniMax portal discovery still reads through that auth-store path, and the direct fix PR was closed unmerged.

## What This Changes

The issue requests that MiniMax portal auth for an agent-specific dashboard/model session resolve from that agent's auth-profiles.json instead of the main agent's auth store.

## Best Possible Solution

Keep the issue open and land a focused auth-runtime fix that keys active auth snapshots by stable runtime identity, keeps main fallback explicit, and adds a regression covering distinct main and selected-agent minimax-portal profiles when OPENCLAW_AGENT_DIR points at the selected agent.
