## Summary

Keep open. Current main still resolves configured per-model `thinkingBudgets` but drops the field before calling pi-ai, so the PR addresses a real runtime gap; however, the latest diff still forwards any finite numeric budget, including zero, negative, and fractional token counts, so it should be tightened before merge.

## What This Changes

The PR adds a `thinkingBudgets` resolver in the pi embedded runner, forwards configured budgets into pi-ai stream options, and adds regression coverage for a MiniMax Anthropic-compatible model.

## Best Possible Solution

Land a narrow version of this PR after the new budget resolver accepts only positive integer token budgets and adds focused coverage for both valid pass-through and invalid-value filtering, without widening the config schema or changing unrelated provider defaults.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure `agents.defaults.models[model].params.thinkingBudgets`: current main merges that value into effective extra params but never maps it into `SimpleStreamOptions`, and the PR's added regression test exercises that exact missing pass-through path.

## Solution Assessment

No, not yet. Forwarding `thinkingBudgets` in this wrapper is the right narrow fix, but the proposed validation is weaker than the PR contract because malformed numeric token budgets can still enter provider request construction.
