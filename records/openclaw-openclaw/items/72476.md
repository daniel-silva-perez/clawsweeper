## Summary

Keep open. Current main still drops configured `thinkingBudgets`, and this PR targets the right embedded-runner forwarding path, but the latest diff still accepts malformed numeric token budgets despite the PR’s stated validation goal, so it should not merge as-is.

## What This Changes

The PR adds a `thinkingBudgets` resolver and stream-option forwarder in the pi embedded runner, plus regression coverage for configured MiniMax Anthropic-compatible budget pass-through.

## Best Possible Solution

Land a narrow version of this PR after the resolver keeps only positive integer token budgets and regression coverage proves both valid pass-through and invalid-value filtering, without changing provider defaults or widening the config schema.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure `agents.defaults.models[model].params.thinkingBudgets`: current main resolves the value into effective extra params but never maps it into `SimpleStreamOptions`; the malformed-value bug is reproducible from the PR diff with `0`, negative, or fractional budget entries.

## Solution Assessment

No, not as-is. Forwarding `thinkingBudgets` in this wrapper is the right narrow fix, but the proposed validation still lets malformed token counts reach downstream request construction.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88
