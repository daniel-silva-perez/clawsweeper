## Summary

Current main still drops configured `thinkingBudgets` before the pi-ai stream call, while config already permits provider-specific model params and OpenClaw transports consume `options.thinkingBudgets`. This PR remains a focused implementation candidate rather than obsolete or closeable, but its validation should be tightened before merge.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to tighten `resolveThinkingBudgetsParam` to accept only positive integer budget values, keep the regression proving configured MiniMax-style budgets reach pi-ai stream options, and then run the targeted agent test plus the normal changed gate before merge.
