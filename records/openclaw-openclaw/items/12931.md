## Summary

Keep this issue open. Current main only partially addresses the token-cost problem: ordinary bare text-channel `/new` and `/reset` now stop after a static acknowledgement, but OpenClaw still has a hardcoded reply-producing reset prompt, the gateway agent reset path still sends that prompt through an agent run, and the config schema has no typed reset-greeting suppression or cheaper-model policy. A maintainer review comment already reached the same conclusion and linked the remaining work to the broader reset-prompt API discussions.

## What This Changes

The issue requests configuration to either run bare `/new` and `/reset` greeting turns on a cheaper model or suppress that greeting/model invocation entirely.

## Best Possible Solution

Define one typed reset-greeting policy that covers static chat acknowledgements, reply-producing reset turns, and gateway agent resets, then wire it through config schema/types/help/docs/tests so users can suppress the model-run greeting or choose an explicit cheaper model without changing `/new <message>` and `/reset <message>` behavior.
