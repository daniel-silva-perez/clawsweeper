## Summary

Keep open. Current `main` only partially reduces the reported token-cost problem: bare text-channel `/new` and `/reset` now return a static acknowledgement without continuing to a model turn, but OpenClaw still has no typed reset-greeting suppression or cheaper-model policy, and the gateway agent path still turns bare resets into the hardcoded reset greeting prompt and dispatches an agent run.

## What This Changes

The issue requests a configurable reset greeting policy for bare `/new` and `/reset`, either to use a cheaper model for the greeting turn or to suppress the greeting/model invocation entirely.

## Best Possible Solution

Ship one typed public reset-greeting policy that covers chat acknowledgements, reply-producing reset turns, and gateway agent resets, with schema/types/help/docs/tests aligned and existing `/new <message>` and `/reset <message>` behavior preserved.
