## Summary

Keep open. Current main partially reduced the cost problem for ordinary bare text-channel resets by returning a static acknowledgement, but the requested configurable greeting policy is still missing and the gateway agent reset path still dispatches a hardcoded reset greeting prompt as a model turn. The current issue discussion also contains maintainer/bot reviews from April 26 and April 29, 2026 that explicitly kept this open because the remaining config shape crosses reset surfaces and overlaps related prompt-customization requests.

## What This Changes

Request to add configuration for bare `/new` and `/reset` greeting behavior so users can suppress the model-generated greeting or route it to a cheaper model.

## Best Possible Solution

Define one typed reset-greeting policy in the public config contract that covers static chat acknowledgements, reply-producing reset turns, and gateway agent resets, with schema/types/help/docs/tests aligned and existing `/new <message>` and `/reset <message>` behavior preserved.
