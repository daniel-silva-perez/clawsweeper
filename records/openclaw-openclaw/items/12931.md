## Summary

Keep open. Current main only avoids the model call for ordinary bare text-channel resets by returning a static acknowledgement; the shared prompt, reply-producing reset path, gateway agent reset path, and strict config schema still do not expose suppression or a cheaper greeting-model policy.

## What This Changes

The issue requests configurable `/new` and `/reset` greeting behavior so users can either use a cheaper model for the reset greeting or suppress the model-generated greeting entirely.

## Best Possible Solution

Add one typed reset-greeting policy that covers static chat acknowledgements, reply-producing reset turns, and gateway agent resets, then wire it through schema, exported types, help text, docs, and regression tests while preserving `/new <message>` and `/reset <message>` behavior.
