## Summary

Current main only partially addresses the request. Bare chat `/new` and `/reset` now return static acknowledgements without invoking the model, but gateway/CLI-style `agent` resets still build the reset greeting prompt and dispatch an agent run, and the strict config surface still has no `resetGreeting`, `resetGreetingModel`, `resetPrompt`, or equivalent reset-greeting policy/model override.

## Best Possible Solution

Keep this issue open for maintainer product follow-up. The best path is to decide one typed reset-greeting policy that covers chat and gateway agent reset paths, probably coordinated with #45501 and #51586, then implement schema/types/help/docs/tests so users can suppress the model-run greeting or route it through an explicit cheaper model without changing normal reset semantics.
