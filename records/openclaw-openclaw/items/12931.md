## Summary

Keep open. Current main only partially addresses the token-cost problem: ordinary bare text-channel `/new` and `/reset` now acknowledge without continuing to a model turn, but there is still no public typed reset-greeting suppression or model override policy, and the gateway agent reset path still builds the hardcoded greeting prompt and dispatches an agent run.

## What This Changes

The issue requests configuration to either use a cheaper model for bare `/new`/`/reset` greeting turns or suppress the model-generated greeting entirely.

## Best Possible Solution

Ship one typed public reset-greeting policy that covers chat acknowledgements, reply-producing reset turns, and gateway agent resets, with schema/types/help/docs/tests aligned and existing `/new <message>` and `/reset <message>` behavior preserved.
