## Summary

Current main only partially addresses the request. Bare chat `/new` and `/reset` now return static acknowledgements without invoking the embedded model, but gateway/CLI-style `agent` resets still build and dispatch the Session Startup greeting prompt, and the typed `session` config surface still has no `resetGreeting` or `resetGreetingModel` option.

## Best Possible Solution

Keep this issue open and resolve the remaining product decision. Either document that chat-only static acknowledgements are the intended scope and gateway `agent` reset greetings remain model-run behavior, or add a typed reset-greeting policy/model override and apply it consistently across chat and gateway agent reset paths with schema, docs, and tests.
