## Summary

Keep this issue open. Current main has removed the model call for ordinary bare chat /new and /reset commands, but it has not added a typed reset-greeting policy and the gateway agent reset path still constructs the hardcoded reset greeting prompt and dispatches an agent run.

## What This Changes

The issue requests a configurable model for bare /new or /reset greeting turns, or an option to suppress the greeting/model invocation entirely.

## Best Possible Solution

Define one typed reset-greeting policy that covers chat acknowledgements, reply-producing reset turns, and gateway agent resets, then wire it through schema/types/help/docs/tests so users can suppress the model-run greeting or choose an explicit cheaper model without changing normal reset behavior.
