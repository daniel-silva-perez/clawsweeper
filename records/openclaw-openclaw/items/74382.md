## Summary

Keep open. Current main still has a plausible onboarding UX bug: the skill dependency picker can show a keyless local OpenAI Whisper entry, then the same skills setup flow separately prompts for API keys for other missing env-gated skills such as `openai-whisper-api` and `sag`, without clearly separating dependency installation from credential setup.

## What This Changes

The issue requests that `openclaw onboard` stop asking for OpenAI Whisper or ElevenLabs API keys after showing a keyless OpenAI Whisper skill/dependency entry.

## Best Possible Solution

The skills onboarding UX should clearly split dependency installation from credential setup: local `openai-whisper` remains keyless, cloud `openai-whisper-api` and `sag` credential prompts are explicitly named/scoped, and tests cover the mixed keyless/keyed missing-skill path.
