## Summary

Keep open. Current main still has the confusing onboarding path, and the issue is already paired with open closing PRs #74407 and #74891, so it should not be closed before one of those lands or is replaced.

## What This Changes

The issue requests that `openclaw onboard` stop asking for OpenAI Whisper or ElevenLabs API keys after presenting the keyless local `openai-whisper` skill as not requiring an API key.

## Best Possible Solution

Keep the issue open until an implementation PR lands. The desired end state is a narrow `setupSkills` change with regression coverage for mixed keyless and keyed skills, preserving legitimate env-only credential prompts while avoiding misleading prompts for skills the user did not opt into configuring.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to mock missing keyless `openai-whisper` together with missing env-gated `openai-whisper-api` and `sag`; current `setupSkills` shows the install list first and then prompts for every missing `primaryEnv` skill.

## Solution Assessment

Unclear until the linked PRs are reviewed. The narrowest maintainable fix likely belongs in `src/commands/onboard-skills.ts` with tests in `src/commands/onboard-skills.test.ts`; prompt framing may be enough for env-only skills, but skipped installable skills likely need selected-skill gating.
