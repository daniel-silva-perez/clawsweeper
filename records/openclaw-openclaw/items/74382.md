## Summary

Keep open. Current main still has the plausible onboarding UX bug: the install picker can show the keyless local `openai-whisper` skill, then the later credential loop prompts for every missing env-gated skill such as `openai-whisper-api` and `sag`; the issue is also already paired with open closing PRs #74407 and #74891.

## What This Changes

The issue reports that `openclaw onboard` presents keyless local `openai-whisper` setup text but later asks for OpenAI or ElevenLabs API keys, making separate keyed skills look like a contradictory requirement.

## Best Possible Solution

Keep the issue open until one linked PR lands or is replaced by a narrow `setupSkills` fix with regression coverage: local `openai-whisper` remains keyless, credential prompts are clearly scoped to the actual env-gated skill being configured, and env-only skills still prompt when appropriate.
