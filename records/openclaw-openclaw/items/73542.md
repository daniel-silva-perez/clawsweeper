## Summary

Keep open. Current main still has plugin-only `message_sending` pre-send dispatch for the checked outbound paths, while internal/user hooks are registered and dispatched through the separate `message:<action>` registry. The open PR #73570 explicitly references this issue with closing syntax, so the issue should stay open until that PR lands or maintainers replace it.

## What This Changes

User-defined directory hooks registered for `message_sending` should fire before outgoing messages, matching plugin pre-send behavior and the existing internal `message:sent` bridge.

## Best Possible Solution

Review and land PR #73570 or a replacement that defines a canonical internal pre-send event, handles underscore-versus-colon compatibility or documentation clearly, preserves pre-delivery ordering, and adds focused tests for shared outbound plus inbound reply paths.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: HOOK.md event strings are registered directly, internal dispatch only reaches `message` and `message:<action>` listeners, and current pre-send delivery code only calls plugin `runMessageSending`.

## Solution Assessment

Unclear. The requested bridge is the right area to fix, but a bare trigger call is not enough unless it also resolves event-key compatibility, pre-delivery ordering, documented semantics, and non-shared reply-path coverage.
