## Summary

Keep open. Current main still has no internal/user-hook pre-send dispatch for the reported path, and open PR #73570 references this issue with closing syntax, so the issue should remain open until that implementation or a replacement lands.

## What This Changes

The issue reports that user-defined hooks registered for `message_sending` appear ready but never fire before outbound message delivery.

## Best Possible Solution

Land PR #73570 or a replacement that emits a documented internal pre-send hook before delivery, defines the supported event key or alias behavior, covers the main outbound and inbound reply paths, and adds focused regression tests plus docs for observe versus modify/cancel semantics.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current main registers directory hooks from HOOK.md metadata, but the active pre-send delivery paths only call plugin `message_sending` hooks and never emit an internal `message:sending` or `message_sending` event.

## Solution Assessment

Unclear. The general direction is correct, but a one-line bridge in one outbound path may miss the underscore/colon event-key mismatch and the separate inbound reply path, so maintainers should review PR #73570 against those requirements.
