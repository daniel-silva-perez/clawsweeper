## Summary

Keep open. Current main still installs Playwright Chromium under `/home/node/.cache/ms-playwright`, the Docker/browser docs still point users at that hidden cache path, and there is no stable Dockerfile-created `chrome` or `chromium` command on PATH. The issue is also paired with open implementation PR #42771, so it should not be closed before that PR is reviewed, landed, or explicitly closed.

## What This Changes

The issue asks Docker browser-enabled images to install Playwright browser files outside the hidden `.cache` directory and expose a stable Chrome/Chromium binary on PATH.

## Best Possible Solution

Review, refresh, and either land or close #42771 or a maintainer replacement. The desired end state is a Docker browser install path outside `/home/node/.cache`, stable `chrome`/`chromium` commands on PATH, aligned Docker/browser docs, regression coverage for the Dockerfile contract, and Docker image validation for browser-enabled and browser-disabled builds.
