## Summary

Keep open. Current main still installs Playwright Chromium under the hidden `/home/node/.cache/ms-playwright` path, the Docker/browser docs still document that path, and there is no Dockerfile-created stable `chrome` or `chromium` command on PATH. The issue is also paired with open implementation PR #42771 using closing syntax, so it should remain open until that PR is reviewed, refreshed, landed, replaced, or closed.

## What This Changes

Move Docker Playwright browser assets out of the hidden `/home/node/.cache/ms-playwright` cache path and expose a stable Chrome/Chromium binary on PATH.

## Best Possible Solution

Review, refresh, and either land or close #42771 or a maintainer replacement. The desired end state is a Docker browser install path outside `/home/node/.cache`, stable `chrome`/`chromium` commands on PATH, aligned Docker and browser-control docs, regression coverage for the Dockerfile contract, and Docker image validation for browser-enabled and browser-disabled builds.
