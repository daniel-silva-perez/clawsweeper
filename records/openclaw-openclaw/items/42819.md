## Summary

Keep open. Current main and the latest release still install Playwright Chromium under the hidden `/home/node/.cache/ms-playwright` path, the docs still tell Docker users to persist that path, and no stable Dockerfile-created `chrome` or `chromium` command is on PATH. The issue is also paired with open implementation PR #42771 using closing syntax, so it should remain open until that PR is reviewed, refreshed, landed, replaced, or closed.

## What This Changes

Request to move Docker Playwright browser binaries out of `/home/node/.cache/ms-playwright` into a stable non-hidden path and expose a stable Chrome/Chromium executable on PATH.

## Best Possible Solution

Review, refresh, and either land or close PR #42771 or a maintainer replacement. The desired end state is a Docker browser install path outside `/home/node/.cache`, stable `chrome` or `chromium` commands on PATH, aligned Docker and browser-control docs, and regression coverage for the Dockerfile contract.
