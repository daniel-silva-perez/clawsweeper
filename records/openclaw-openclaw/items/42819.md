## Summary

Keep this issue open. Current main still installs Playwright Chromium under `/home/node/.cache/ms-playwright`, the Docker/browser docs still document that hidden cache path, and no stable `chrome` or `chromium` executable is exposed on PATH. #42771 is also an open same-author implementation candidate with closing syntax, so the best maintainer path is to review, land, or close that PR first.

## Best Possible Solution

Keep this issue open and review or close #42771 as the implementation candidate. A complete fix should move Docker Playwright browser assets out of `/home/node/.cache`, expose a stable `chrome` or `chromium` executable on PATH, update Docker/browser docs and tests to the shipped path, and validate Docker builds with and without `OPENCLAW_INSTALL_BROWSER=1`.
