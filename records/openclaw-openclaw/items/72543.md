## Summary

Keep open. Current main still normalizes some main-session aliases, but the shared Control UI session dropdown builder does not inject the canonical main session when it is absent from `sessions.list`, so the reported UI gap remains valid.

## What This Changes

The issue requests a stable canonical main-session option in the Control UI chat dropdown and alias normalization to `agent:main:main`.

## Best Possible Solution

Add a shared Control UI path that derives the canonical main key from `hello.snapshot.sessionDefaults.mainSessionKey`, prepends or otherwise guarantees a stable main-session option before discovered sessions, keeps desktop and mobile selects aligned, and uses generic agent/session metadata for the label rather than hard-coding a persona name.

## Reproduction Assessment

Yes. A focused unit reproduction can call `resolveSessionOptionGroups` with active session `agent:main:subagent:x`, a `sessionsResult` that lacks `agent:main:main`, and Gateway defaults containing `mainSessionKey`; current code returns supplied rows plus the active key, so the canonical main option is absent.

## Solution Assessment

Yes. The narrow maintainable fix is to update the shared option-group builder and existing alias-normalization path using the Gateway snapshot defaults; adding a new Gateway API or hard-coding `Chelmsworth` would be broader than needed.

## Repair Work Prompt

Fix the Control UI chat session dropdown so the canonical main session is always available. In `ui/src/ui/chat/session-controls.ts`, derive a canonical main session key from `state.hello?.snapshot?.sessionDefaults?.mainSessionKey` when present, falling back only as needed, and ensure `resolveSessionOptionGroups` includes that key even when `sessionsResult.sessions` omits it and the active session is a subagent/channel/other non-main key. Keep desktop `renderChatSessionSelect` and mobile `renderChatMobileToggle` aligned by fixing the shared builder rather than duplicating UI code. Use generic agent/session metadata for the label, not the reporter's persona-specific `Chelmsworth` example. Review `ui/src/ui/app-gateway.ts` alias normalization and add/adjust tests only if settings/session-key normalization still leaves `main` aliases persisted after Gateway defaults arrive. Add regression coverage for a missing `agent:main:main` row while active on `agent:main:subagent:x`, plus any rendering-level coverage needed for the select options. Add a user-facing `CHANGELOG.md` fix entry with valid contributor attribution. Do not change Gateway protocol shape unless you first prove the existing `hello.snapshot.sessionDefaults.mainSessionKey` is insufficient.
