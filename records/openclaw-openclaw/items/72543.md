## Summary

Keep this issue open. Current main already normalizes some `main` aliases from the Gateway hello snapshot, but the shared Control UI dropdown builder still only adds discovered session rows plus the current active key, so `agent:main:main` can be absent when the user is viewing another session and the sessions list does not include the canonical main row.

## What This Changes

The issue requests that the Control UI chat session dropdown always include a stable canonical main-session option and normalize `main` aliases to the canonical session key.

## Best Possible Solution

Add a shared Control UI path that derives the canonical main key from `hello.snapshot.sessionDefaults.mainSessionKey`, prepends a stable main-session option before discovered sessions, keeps desktop and mobile selects aligned, and uses generic labeling from agent/session metadata rather than hard-coding a persona name.

## Reproduction Assessment

Yes. A focused unit reproduction can call `resolveSessionOptionGroups` with active session `agent:main:subagent:x` and a `sessionsResult` that lacks `agent:main:main`; current code returns only supplied rows plus the active key, so the canonical main option is absent.

## Solution Assessment

Yes. The narrow maintainable fix is to inject the canonical main option through the shared option-group builder and reuse the existing Gateway `sessionDefaults` normalization source; adding a new Gateway API or hard-coded label would be unnecessary.
