## Summary

Keep open. Current main still normalizes some `main` aliases, but the shared Control UI dropdown builder only renders rows returned by `sessions.list` plus the active session key, so the canonical `agent:main:main` option can still be absent when the active session is something else and the list omits main.

## What This Changes

The issue asks the Control UI chat dropdown to always include a stable canonical main-session option keyed as `agent:main:main` and to keep `main` aliases normalized to that key.

## Best Possible Solution

Add a shared Control UI path that derives the canonical main key from `hello.snapshot.sessionDefaults.mainSessionKey`, prepends a stable main-session option before discovered sessions, keeps desktop and mobile selects aligned, and uses generic agent/session metadata for the label instead of hard-coding a persona name.

## Reproduction Assessment

Yes. A focused code-level reproduction is to call `resolveSessionOptionGroups` with active session `agent:main:subagent:x` and a `sessionsResult` that lacks `agent:main:main`; current main returns only supplied rows plus the active key, so the canonical main option is absent.

## Solution Assessment

Yes. For this issue, injecting the canonical option through the shared Control UI option-group builder and reusing existing Gateway `sessionDefaults` is the narrowest maintainable fix; the related gateway active-list fix in #43984 is complementary but does not replace the UI guarantee requested here.
