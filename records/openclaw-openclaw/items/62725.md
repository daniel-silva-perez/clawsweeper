## Summary

Keep open. Current main still documents and validates typing controls at `agents.defaults` and `session`, but not at `agents.list[]`; the runtime also resolves reply typing from `session` before default agent settings, so the requested per-agent override is not implemented.

## What This Changes

Request support for `typingMode` and `typingIntervalSeconds` on individual `agents.list[]` entries, with per-agent values overriding `agents.defaults` when present.

## Best Possible Solution

Add `typingMode` and `typingIntervalSeconds` to the per-agent config contract, resolve them through the same agent-scoped path used for other per-agent defaults, and preserve precedence as session override first, then matching agent entry, then `agents.defaults`.

## Reproduction Assessment

Yes. A config containing `agents.list: [{ id: "buddy", typingMode: "message" }]` has a source-backed reproduction path through the strict `AgentEntrySchema`, which currently does not declare that key.

## Solution Assessment

Yes, the narrow maintainable fix is to extend the existing agent config/schema/resolver/runtime path rather than adding channel-specific behavior or a new top-level config key.

## Repair Work Prompt

Implement per-agent typing indicator overrides for openclaw/openclaw. The observable issue is that `agents.list[].typingMode` and `agents.list[].typingIntervalSeconds` are rejected as unrecognized keys and reply typing currently only uses `session` or `agents.defaults`. Add these fields to the per-agent config contract (`AgentConfig`, `AgentEntrySchema`, generated schema/help/labels as appropriate), expose them through `resolveAgentConfig`, and update reply typing resolution so precedence is `session.typingMode` / `session.typingIntervalSeconds` first, then the matched `agents.list[]` entry, then `agents.defaults`, then legacy defaults. Consider heartbeat typing consistency only if it can use the same agent-scoped resolution without broad behavior changes. Update docs under `docs/gateway/config-agents.md` and `docs/concepts/typing-indicators.md`; add focused tests for schema acceptance/rejection and runtime precedence. Do not add channel-specific logic, do not introduce startup/load-time migrations, and do not change unrelated typing defaults.
