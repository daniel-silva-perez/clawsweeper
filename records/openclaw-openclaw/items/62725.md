## Summary

Current main still does not support `typingMode` or `typingIntervalSeconds` on `agents.list[]`. The config schema and TypeScript type only expose those fields under `agents.defaults` and `session`, and the reply/heartbeat runtime reads typing settings from session/defaults rather than the resolved per-agent config.

## Best Possible Solution

Keep this issue open and implement it in the core agent configuration path: add `typingMode` and `typingIntervalSeconds` to `AgentConfig` and `AgentEntrySchema`, include them in resolved per-agent config, and resolve reply and heartbeat typing with clear precedence, likely `session` override first, then `agents.list[]`, then `agents.defaults`. Add schema/runtime tests and update the public typing/config docs.
