## Summary

Keep this open. Current main handles ordinary inbound WhatsApp media, but quoted/referenced media in a WhatsApp reply is still reduced to a reply-context placeholder, so the reported group-reply-to-image case is not implemented well enough to close.

## What This Changes

The issue reports that when a WhatsApp user replies to an earlier image and tags OpenClaw, OpenClaw does not receive or view the quoted image.

## Best Possible Solution

Extend the WhatsApp plugin inbound pipeline to hydrate visible quoted image media from `contextInfo.quotedMessage`, save it through the existing inbound media store, and pass it to the agent through the established media context without moving WhatsApp-specific behavior into core.

## Reproduction Assessment

Yes. A unit-level reproduction can construct an `extendedTextMessage` with `contextInfo.quotedMessage.imageMessage` and observe that current main produces `replyToBody: "<media:image>"` without any quoted media path for the agent to inspect.

## Solution Assessment

Yes. The narrowest maintainable fix is in `extensions/whatsapp` inbound extraction/enrichment, preserving existing access-control and `allowlist_quote` visibility rules while reusing the current media-save and media-context paths.
