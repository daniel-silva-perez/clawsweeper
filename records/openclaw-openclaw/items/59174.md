## Summary

Keep open. Current main still turns a quoted WhatsApp image into reply text context like `<media:image>` and only downloads media attached to the current inbound message, so the reported group-reply image case remains actionable.

## What This Changes

The issue requests that WhatsApp replies to earlier image messages expose the referenced image to OpenClaw when the bot is mentioned, instead of requiring the image to be re-sent.

## Best Possible Solution

Implement this in the WhatsApp plugin by downloading authorized visible `contextInfo.quotedMessage` media through the existing Baileys socket/media-save pipeline, preserving media size caps and context-visibility rules, then exposing the saved reply-target media to the agent alongside the reply text context with focused tests and docs/changelog updates.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: a WhatsApp text reply with `contextInfo.quotedMessage.imageMessage` becomes `ReplyToBody: "<media:image>"`, while the monitor only downloads media from the current WAMessage.

## Solution Assessment

Yes. The narrowest maintainable fix is WhatsApp-owned quoted-media handling using the existing inbound media pipeline; broad core changes or unrelated outbound native mention work are not needed.
