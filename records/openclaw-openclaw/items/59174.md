## Summary

Keep this issue open. Current main still only represents a quoted WhatsApp image as a reply-context media placeholder; it does not download or attach the referenced image for the agent to inspect.

## Best Possible Solution

Implement this in the bundled WhatsApp plugin, likely in `extensions/whatsapp/src/inbound/*`: when an authorized inbound message references a quoted image/video/document/audio, resolve and stage that quoted media into the agent context, either by caching media metadata/path by stanza id from the original inbound message or safely reconstructing/downloading the quoted Baileys message. Add tests for the exact group flow: unmentioned image arrives, a later reply mentions OpenClaw, and the resulting context includes the referenced image path/type for media understanding while respecting group allowlist/context visibility.

## Evidence

- **Issue context is actionable and unprotected:** The provided GitHub context describes a recent WhatsApp group regression on version 2026.3.31: user replies to an image and mentions OpenClaw, but the agent cannot see the referenced image. The author is not a maintainer and the labels are bug/regression, with no protected labels or closing PR supplied.
