## Summary

Keep this issue open. Current main has partial no-LLM direct invocation through Gateway HTTP `POST /tools/invoke` and skill `command-dispatch: tool`, but the requested umbrella capability is still not implemented across coding tools, cron direct tool payloads, CLI `--tool`/`--args`, and a generic `/tool ...` chat command. Open PR #63919 and open issue #18160 are useful narrower slices, not full replacements.

## Best Possible Solution

Keep this issue open as the umbrella product/API request. The best path is an explicit maintainer decision: either design one security-reviewed direct-invocation contract that consistently covers HTTP, cron, CLI, and slash-command surfaces, or split the remaining work into canonical narrower trackers while keeping #63919 and #18160 linked as active slices.
