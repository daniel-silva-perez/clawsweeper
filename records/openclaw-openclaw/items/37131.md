## Summary

Keep this issue open. Current main has partial no-LLM direct invocation through HTTP `/tools/invoke` and skill `command-dispatch: tool`, but it does not implement the requested generic direct invocation contract across coding tools, cron payloads, CLI flags, and generic slash-command syntax. Related #63919 and #18160 are active narrower slices, not complete replacements for the umbrella request.

## Best Possible Solution

Keep this issue open as the umbrella product/API request. The best path is either a single security-reviewed direct invocation contract that covers HTTP, cron, CLI, and slash-command surfaces consistently, or an explicit maintainer split into canonical narrower trackers for the remaining surfaces, with #63919 and #18160 linked as active slices rather than replacements.
