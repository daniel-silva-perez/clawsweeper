## Summary

Keep this issue open. Current main has partial no-LLM direct invocation through HTTP `POST /tools/invoke` and skill `command-dispatch: tool`, but it does not implement the requested umbrella contract across coding tools, cron payloads, CLI `--tool`/`--args`, and generic `/tool ...` slash-command syntax. Open PR #63919 and open issue #18160 are useful narrower slices, not complete replacements.

## Best Possible Solution

Keep this issue open as the umbrella product/API request. The best path is either a single security-reviewed direct invocation contract that covers HTTP, cron, CLI, and slash-command surfaces consistently, or an explicit maintainer split into canonical narrower trackers for the remaining surfaces, with #63919 and #18160 linked as active slices rather than replacements.
