## Summary

Keep this PR open. Current main still does not implement the PR's central hardening: attachment-cache local reads go through a custom readLocalBuffer helper instead of the shared readLocalFileSafely/openLocalFileSafely path, temp cache writes still omit an explicit 0o600 mode, and the current tests cover only partial nofollow/maxBytes behavior rather than the shared helper's identity checks or temp-file permissions. The PR head remains an unmerged, scope-limited implementation candidate, and the author already addressed the SafeOpenError("too-large") review concern in the PR discussion.

## Best Possible Solution

Port the PR onto current main by replacing or refactoring readLocalBuffer to use readLocalFileSafely/openLocalFileSafely identity checks while preserving allowed-root validation and MediaUnderstandingSkipError mappings, add { mode: 0o600 } to the temp fs.writeFile in getPath, add targeted tests for oversized SafeOpenError mapping, path/identity mismatch behavior, and temp-file mode, then run the focused media-understanding attachment tests plus the normal changed gate before merge.
