## Summary

Keep this PR open. Current main still lacks the intended Feishu-owned `channels.feishu.localRoots` and account-scoped `localRoots` support for media uploads; it only forwards caller-provided `mediaLocalRoots` arrays to `loadWebMedia`. The PR remains a focused implementation candidate, with a security-sensitive `"any"` bypass that needs maintainer review and docs/tests before merge.

## Best Possible Solution

Keep this PR open for maintainer review or land an equivalent Feishu plugin-owned patch: add `localRoots: "any" | string[]` to the shared top-level and account Feishu config, resolve it from merged account config in `sendMediaFeishu`, honor explicit empty arrays, inject raw `readFile` only for explicit `"any"`, update the Feishu docs/config reference with the security implications, and coordinate with #51375 for reply-path `mediaLocalRoots` forwarding.
