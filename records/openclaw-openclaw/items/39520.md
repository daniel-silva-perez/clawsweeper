## Summary

Keep this PR open. Current main still does not implement the PR's intended Feishu-owned `channels.feishu.localRoots` or account-scoped `localRoots` configuration for media uploads. Main only forwards caller-provided `mediaLocalRoots` arrays to `loadWebMedia`, while the PR adds the missing config field, merged-account resolution, `"any"` support, and targeted tests. The change is security-sensitive because `"any"` requires a raw file-read bypass, so it needs maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review or land an equivalent Feishu plugin-owned patch: add `localRoots: "any" | string[]` to shared top-level and account Feishu config, resolve it from the merged account config in `sendMediaFeishu`, honor explicit empty arrays, inject raw `readFile` only for explicit `"any"`, document the security implications, and test top-level config, account overrides, `"any"`, empty arrays, and the no-`readFile` array-root path. Coordinate with #51375 for reply-path root forwarding rather than treating it as a replacement.
