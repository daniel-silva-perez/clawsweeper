## Summary

Keep open. Current main still lacks the requested update-version behavior: update/status/startup paths call `compareSemverStrings()`, which delegates to the generic SemVer comparator, and the current regression test still ranks a numeric `-N` suffix below `-beta.1`. The PR remains useful, but it needs maintainer adaptation to current main because the parser has moved into the shared ClawHub/plugin SemVer path and the proposed numeric revision parsing has a precision issue.

## What This Changes

This PR changes update-version comparison and regression tests so numeric-only `-N` suffixes are treated as OpenClaw correction revisions newer than same-base stable and prerelease versions.

## Best Possible Solution

Adapt the fix on current main so OpenClaw update/status/startup/update-runner comparisons use correction-release ordering for OpenClaw package versions while preserving generic ClawHub/plugin SemVer semantics. The adapted patch should compare numeric suffixes without precision loss and cover correction-vs-stable, correction-vs-beta, revision ordering, and the affected update/status callers.

## Review Findings

Overall correctness: patch is correct

Overall confidence: 0.82

Full review comments:

- **[P3] Compare revision suffixes without lossy number coercion:** `src/infra/update-check.ts:405`
  - body: The new revision path stores an arbitrary numeric suffix with `Number.parseInt()`. Registry-provided versions above `Number.MAX_SAFE_INTEGER` can collapse adjacent revisions to the same JS number and produce the wrong update ordering; keep the suffix as a normalized string, use `BigInt`, or bound the accepted length before comparing.
