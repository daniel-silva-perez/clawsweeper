## Summary

Keep open. Current main still routes update availability through the generic semver comparator that treats numeric `-N` suffixes as prerelease identifiers, while OpenClaw's version parser and release policy define `YYYY.M.D-N` as a stable correction revision. The PR is useful, but it needs maintainer adaptation because the comparator is now shared with ClawHub/plugin SemVer checks and the proposed numeric parsing leaves a precision-safety question.

## What This Changes

This PR changes update-version comparison and regression tests so numeric-only `-N` suffixes rank as OpenClaw correction revisions newer than the base stable release and same-base prereleases.

## Best Possible Solution

Keep the PR open and adapt the implementation so OpenClaw update-specific version comparisons use correction-release ordering for `YYYY.M.D-N` without changing generic ClawHub/plugin SemVer behavior. Add regression coverage for correction-vs-stable, correction-vs-beta, revision ordering, and precision-safe numeric suffix comparison, then add an appropriate changelog credit if maintainers accept the behavior.
