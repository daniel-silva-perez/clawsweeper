## Summary

Keep this PR open. Current main still sends update availability through the shared SemVer comparator, which treats numeric `-N` suffixes as prerelease identifiers, while the OpenClaw-specific version parser already treats numeric suffixes as correction revisions newer than the base release. The PR is directionally valid, but it is stale against the current `src/infra/semver-compare.ts` extraction and should address the numeric precision concern before merge.

## What This Changes

The PR changes update version comparison to classify numeric-only `-N` suffixes as OpenClaw correction revisions newer than stable releases and adds regression tests in `src/infra/update-check.test.ts`.

## Best Possible Solution

Keep this PR open and ask for a rebase/adaptation onto current main: update the OpenClaw update-version comparison path so numeric correction revisions rank above stable releases, preserve generic ClawHub/plugin SemVer behavior, add/update regression coverage for `2026.3.23-1 > 2026.3.23` and revision-vs-beta ordering, and compare numeric suffixes without JavaScript number precision loss.
