## Summary

Keep open. Current main still treats numeric-only hyphen suffixes through the generic SemVer prerelease path used by update checks, while OpenClaw's release docs and `parseOpenClawVersion()` already define `YYYY.M.D-N` as a correction revision. The PR is useful, but it needs maintainer review/adaptation because current main moved the parser into a shared ClawHub-facing comparator and the numeric comparison should avoid JavaScript number precision loss.

## What This Changes

The PR modifies update-version comparison and regression tests so numeric-only `-N` suffixes rank as OpenClaw correction revisions newer than the base stable release.

## Best Possible Solution

Adapt the fix so OpenClaw update-version comparisons use correction-release ordering for `YYYY.M.D-N` without changing generic ClawHub/plugin SemVer behavior, add regression coverage for correction-vs-stable, correction-vs-beta, and precision-safe numeric suffix comparison, then land with an appropriate user-facing changelog credit if maintainers accept the behavior.
