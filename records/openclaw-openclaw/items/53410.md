## Summary

Keep this PR open. Current main still routes update availability through the shared semver comparator, which treats numeric `-N` suffixes as prerelease identifiers, while the OpenClaw-specific version parser already ranks numeric correction revisions above the base stable release. The PR is directionally valid, but it is stale against the current `src/infra/semver-compare.ts` extraction and should address the numeric precision concern before merge.

## Best Possible Solution

Keep this PR open and request a rebase/adaptation onto current main: update the OpenClaw update-version comparison path so numeric correction revisions rank above stable releases, preserve generic semver behavior for ClawHub/plugin comparisons, add/update regression coverage for `2026.3.23-1 > 2026.3.23` and revision-vs-beta ordering, and compare numeric suffixes without JavaScript number precision loss.
