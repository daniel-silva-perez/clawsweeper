## Summary

Keep open. Current main still has a Claude-only external CLI history import path, so the PR addresses a real remaining Gateway history gap, but the current branch is not merge-ready because the new Codex transcript resolver misses valid `CODEX_HOME`-based session stores.

## What This Changes

The PR adds Codex CLI transcript import support for Gateway `chat.history`, commentary filtering, multi-binding merge/reset suppression handling, persistence/type updates, tests, and a changelog entry.

## Best Possible Solution

Keep this PR as the implementation candidate, fix Codex home resolution to honor `CODEX_HOME` before falling back to `~/.codex`, preserve current-main CLI binding reuse fields during rebase, and land only after focused Gateway/agent tests plus changed-lane CI are refreshed.

## Reproduction Assessment

Yes. A source-level reproduction is clear: set `CODEX_HOME` to a directory outside `HOME/.codex`, place the Codex transcript under `$CODEX_HOME/sessions`, and the PR resolver still searches only `HOME/.codex/sessions`, so `chat.history` remains empty for that valid binding.

## Solution Assessment

No, not yet. The overall seam is the right maintainable direction, but the proposed resolver must use the same Codex home contract as the rest of OpenClaw before this is the best fix.
