## Summary

Keep open. The intended Telegram UX fix is valid, but this PR targets a legacy handler that is absent from current main; the active native approval path still only clears Telegram inline buttons and never deletes the approval message. The branch also mixes unrelated docs edits, including a contradictory sub-agent depth note and an edit under localized docs that this repo does not maintain.

## What This Changes

The branch adds a delayed Telegram `deleteMessage` call after approval resolution in the legacy Telegram exec approval handler and includes several documentation cleanup/update edits.

## Best Possible Solution

Land a narrow current-main replacement that wires Telegram approval finalization to delete the pending native message after a short visible resolved state, with timer cleanup and regression coverage, while keeping unrelated docs cleanup out of the runtime fix.

## Reproduction Assessment

Yes. Trigger a native Telegram exec approval and approve or deny it; current-main source inspection shows finalization returns `clear-actions` and only edits reply markup, so the approval message is not deleted.

## Solution Assessment

No. The requested behavior is the right direction, but this PR is not the best implementation because it patches a removed legacy handler and adds an uncancelled timeout; the fix belongs in the active `approval-handler.runtime.ts` path, using the existing approval final-action/delete seam or a tightly scoped Telegram delete transport.
