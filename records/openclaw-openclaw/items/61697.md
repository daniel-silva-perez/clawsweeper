## Summary

Keep open. Current main has a supported Gmail Pub/Sub path and tests showing `/hooks/gmail` dispatches when a valid request reaches the Gateway, but I did not find a shipped fix for this exact no-delivery report. The observed warning still follows from missing `hooks.gmail.topic`, and the related PR only proposes a log-message clarification and remains open.

## Best Possible Solution

Keep the issue open and ask for the missing delivery-path evidence: full current `hooks` config with secrets redacted, exact `gog gmail watch serve` command, gog serve logs during a fresh test email, the resolved hook URL/port, and whether the setup now uses `openclaw webhooks gmail setup` or intentionally opts out with `OPENCLAW_SKIP_GMAIL_WATCHER=1`. Separately, review or supersede PR #61704 if maintainers want the confusing missing-topic log clarified for external watcher setups.
