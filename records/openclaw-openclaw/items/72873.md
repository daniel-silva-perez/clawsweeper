## Summary

Keep open. Current main still subscribes to Telegram `edited_message` updates but only registers handlers for `message`, `channel_post`, `callback_query`, message migration, and reactions, so edited Telegram messages still do not enter the inbound pipeline. The linked open PR #73104 is the active closing implementation path and should be reviewed, landed, or replaced before this issue closes.

## What This Changes

Handle Telegram `edited_message` updates so rapidly edited streaming bot messages are coalesced and the final edited content reaches OpenClaw instead of only the initial partial message.

## Best Possible Solution

Keep the fix extension-owned in the Telegram plugin: add an `edited_message` handler that reuses the normal inbound path, coalesces rapid edits by chat/message id, avoids bot-authored echoes, and includes focused tests for registration, debounce, dedupe, and dispatch behavior.

## Reproduction Assessment

Yes. A synthetic grammY `edited_message` update is enough to reproduce the source-level gap because current main requests that update type but has no registered handler to pass it into the inbound message pipeline.

## Solution Assessment

Yes. A narrow Telegram plugin handler with debounce and dedupe coverage is the maintainable fix boundary; the linked closing PR should be reviewed, landed, or replaced before this tracker closes.
