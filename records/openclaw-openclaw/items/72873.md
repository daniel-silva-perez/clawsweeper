## Summary

Keep open. Current main still requests Telegram edited-message update types but does not register a Telegram `edited_message` runtime handler, so the reported truncation path remains valid. Open PR #73104 is already the active closing implementation candidate and should be reviewed, landed, or replaced before this issue closes.

## What This Changes

The issue requests Telegram `edited_message` update handling so edited or streaming bot messages are debounced and dispatched as inbound content instead of only processing the initial partial message.

## Best Possible Solution

Keep the fix owned by the Telegram plugin: add an `edited_message` handler that reuses the inbound message path, coalesces rapid edits by chat and message id, avoids bot-authored echoes, and locks the behavior with focused registration, debounce, dedupe, and dispatch tests.

## Reproduction Assessment

Yes. A synthetic grammY `edited_message` update is enough to reproduce the gap at source level because current main asks Telegram for that update type but has no registered handler to pass it into the inbound pipeline.

## Solution Assessment

Yes. A narrow Telegram plugin fix with debounce and dedupe coverage is the maintainable boundary; because PR #73104 already uses closing syntax for this issue, the issue should remain open until that PR lands or is replaced.
