## Summary

Keep this issue open. Current main still requests Telegram `edited_message` updates through the allowed-update path but has no runtime handler that dispatches edited messages into the inbound Telegram pipeline, and open PR #73104 is already the active closing implementation candidate.

## What This Changes

The issue requests Telegram inbound handling for `edited_message` updates, with debounce/deduplication, so streamed bot edits are processed instead of only the initial partial message.

## Best Possible Solution

Keep the change inside the Telegram plugin: add an `edited_message` handler that reuses the existing inbound pipeline, debounces rapid edits by chat/message id, avoids self-authored echoes, and adds focused tests for registration, coalescing, and dispatch behavior.
