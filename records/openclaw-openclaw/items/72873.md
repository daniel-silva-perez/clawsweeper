## Summary

Keep open. Current main still asks Telegram for `edited_message` updates but has no Telegram runtime handler that dispatches those updates into the inbound message pipeline, so the reported truncated streaming behavior remains valid. Open PR #73104 is the active closing implementation path and should be reviewed, landed, or replaced before this tracker closes.

## What This Changes

Handle Telegram `edited_message` updates so streaming bot edits are debounced, coalesced, and dispatched as inbound Telegram content instead of leaving only the first partial message.

## Best Possible Solution

Keep the fix extension-owned in the Telegram plugin: add an `edited_message` handler that reuses the normal inbound path, coalesces rapid edits by chat/message id, avoids bot-authored echoes, and includes focused tests for registration, debounce, dedupe, and dispatch behavior.

## Reproduction Assessment

Yes. A synthetic grammY `edited_message` update is a high-confidence reproduction path because current main requests that update type but has no registered handler to pass it into the inbound pipeline.

## Solution Assessment

Yes. A narrow Telegram plugin handler with debounce and dedupe coverage is the maintainable fix boundary; the open closing PR should be reviewed, landed, or replaced before this issue closes.
