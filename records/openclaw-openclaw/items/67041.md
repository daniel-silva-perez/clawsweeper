## Summary

Keep this PR open. The linked credential-redaction bug is still present on current main, and this PR is the relevant implementation candidate, but the proposed Basic auth regex can leave padded `=` characters outside the masked span and one new JWT test is covered by the pre-existing Bearer rule instead of the standalone JWT rule.

## What This Changes

The PR adds default log redaction regexes and tests for standalone JWTs, Basic auth headers, `X-OpenClaw-Token`, `x-pomerium-jwt-assertion`, `X-Api-Key`, and `X-Auth-Token`.

## Best Possible Solution

Keep this PR as the implementation candidate for the linked bug, but amend it so the shared default redaction fully covers padded Basic auth, standalone JWTs, and the named security headers, with focused text, line-batch, and `logs.tail` regression coverage before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Mask Basic auth padding with the credential:** `src/logging/redact.ts:44`
  - body: The Basic auth pattern ends with a word boundary after a base64 character class. For padded credentials such as `Authorization: Basic dXNlcm5hbWU6cGFzc3dvcmQ=`, the match backs off before the `=`, leaving padding outside the replacement span. Drop the boundary or use a delimiter that still includes padding in the captured credential.
  - confidence: 0.91
- **[P3] Exercise standalone JWT redaction directly:** `src/logging/redact.test.ts:215-216`
  - body: This test puts the JWT behind `Authorization: Bearer`, which is already covered by the existing Bearer regex. It would still pass if the new standalone JWT pattern stopped working, so use a standalone JWT or non-Bearer header sample to protect the behavior this PR adds.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: No CI, dependency, or supply-chain expansion was found, but this secret-redaction patch needs a correctness fix before merge.
