## Summary

Keep this issue open. Current main still routes Feishu/Lark file-typed video resources through `type=file` with no 5xx retry or fallback to `type=media`, and open unmerged PRs #49967 and #50164 explicitly target this same issue.

## Best Possible Solution

Keep the issue open and review the focused Feishu plugin fix. The best path is to add a conservative `extensions/feishu` retry from `type=file` to Feishu's media resource type on vetted 502/503/504 server errors, optionally use video-like metadata only when safe, preserve normal file behavior, and add targeted regression tests before closing after a PR lands on main.
