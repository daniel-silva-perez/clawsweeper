## Summary

Keep open. Current main and v2026.4.26 shipped the narrower hard-failure fix: `runtime="subagent"` now ignores ACP-only `streamTo`/`resumeSessionId` instead of erroring. But this issue asks for real `streamTo="parent"` live progress from native subagents, and current code, tests, and docs still define that as ACP-only with no native subagent relay/progress contract.

## Best Possible Solution

Keep this issue open as the canonical native subagent progress-streaming request. The best implementation would add an explicit supported contract for parent progress from `runtime="subagent"`, likely by extending `SpawnSubagentParams`, relaying gateway/agent lifecycle or task-progress events back to the requester session, preserving the shipped silent-ignore compatibility for unsupported ACP-only fields where needed, and adding focused tests for run mode, thread/session mode, missing requester context, cleanup/timeout, and non-streaming back compatibility.
