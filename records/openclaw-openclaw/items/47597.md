## Summary

Keep open. Current main has the compatibility fix that accepts and ignores ACP-only `streamTo` for native subagent spawns, but it still does not implement realtime parent streaming for `runtime="subagent"`: the schema and docs call `streamTo` ACP-only, the native dispatch path omits it, and the native spawn path only registers/completes a background run with announce delivery.

## What This Changes

The issue requests enabling `streamTo="parent"` for `sessions_spawn` with `runtime="subagent"` so native child runs stream progress to the parent session in real time.

## Best Possible Solution

Keep the shipped ACP-only field tolerance, then design and implement a native subagent progress mechanism in core, such as parent-visible task progress, a bounded heartbeat relay, or a gateway event stream with explicit visibility and isolation rules, followed by schema, docs, and focused tests for both `mode="run"` and `mode="session"`.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: current tests already call `runtime: "subagent"` with `streamTo: "parent"` and assert the field is stripped before `spawnSubagentDirect`, while the docs and native spawn code show announce/completion behavior rather than realtime streaming.

## Solution Assessment

No. The current main behavior is the right compatibility fix for provider-injected ACP-only fields, but it is not the requested feature; the issue's heartbeat suggestion is plausible but needs maintainer design before it is clearly the best implementation.
