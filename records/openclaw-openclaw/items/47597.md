## Summary

Keep open. Current main fixed the hard-error path by treating `streamTo` as ACP-only and ignoring it for native subagent spawns, but it does not implement realtime `streamTo="parent"` behavior for `runtime="subagent"`. The schema, docs, dispatch path, and tests all preserve ACP-only stream semantics while native subagents remain completion/announce based.

## Best Possible Solution

Keep this issue open for maintainer design. Preserve the current compatibility behavior that ignores ACP-only `streamTo` for native subagents, then choose a native progress mechanism, likely task progress notifications, a parent-session heartbeat relay, or a broader gateway event stream with explicit visibility and isolation rules. Update `sessions_spawn` schema/docs and focused agent/subagent tests once the design is selected.
