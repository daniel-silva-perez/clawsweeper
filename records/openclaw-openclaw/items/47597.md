## Summary

Keep open. Current main has the compatibility fix that accepts and ignores ACP-only `streamTo` on native subagent spawns, but it still does not implement realtime parent streaming for `runtime="subagent"`; source, docs, and tests continue to treat `streamTo` as ACP-only.

## What This Changes

Request to make `sessions_spawn` support `streamTo="parent"` for `runtime="subagent"` so native subagents can send realtime progress to the parent session.

## Best Possible Solution

Keep the shipped ACP-field tolerance, then design and implement a native subagent progress surface in core with explicit visibility and isolation rules, update the `sessions_spawn` schema and docs, and cover both `mode="run"` and `mode="session"` behavior with focused tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction path exists: current tests exercise `runtime="subagent"` plus `streamTo="parent"` and assert that the field is stripped, while the native spawn and registry code show completion/announce tracking rather than realtime parent streaming.

## Solution Assessment

Unclear. The current shipped behavior is a good compatibility fix for provider-injected ACP-only fields, but it is not the requested feature; the heartbeat suggestion is plausible, yet maintainers should choose the event source and visibility model before implementation.
