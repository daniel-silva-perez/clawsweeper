## Summary

Keep open. Current main fixes the hard-error compatibility path by ignoring ACP-only `streamTo`/`resumeSessionId` for native subagent spawns, but it still does not implement realtime `streamTo="parent"` behavior for `runtime="subagent"`. The schema, docs, dispatch path, and tests still describe or assert ACP-only streaming semantics.

## Best Possible Solution

Keep this issue open for maintainer design. Preserve the current compatibility behavior that ignores ACP-only `streamTo` for native subagents, then choose a native progress mechanism such as task progress notifications, a parent-session heartbeat relay, or a broader gateway event stream with explicit visibility and isolation rules. Update `sessions_spawn` schema/docs and focused agent/subagent tests once the design is selected.
