## Summary

Keep open. Current main still has an A2UI generator path that can vary by install layout, and the tracked hash is currently stale against the generator's own read-only hash inputs, so this is not safe to close as implemented.

## What This Changes

The issue asks for A2UI bundle and hash generation to stay byte-stable regardless of whether dependencies resolve through `ui/node_modules` or root `node_modules`.

## Best Possible Solution

Fix the generator rather than only refreshing artifacts: make dependency resolution canonical across UI-local and root installs, or complete the singleton alias set including `signal-polyfill` and `signal-polyfill/`; then regenerate the tracked A2UI bundle/hash and add regression coverage that proves stable generated bytes across both layouts.
