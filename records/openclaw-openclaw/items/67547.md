## Summary

Keep open. Current main has partial hash hardening, but the A2UI generator is still install-layout-sensitive: it prefers `ui/node_modules` when present, does not alias `signal-polyfill`, and has no regression coverage proving byte-stable generated artifacts across the two layouts. The related open PR #68934 remains an implementation candidate, not merged proof that this issue is fixed.

## Best Possible Solution

Keep this issue open. Review and land #68934 or a successor that canonicalizes A2UI dependency resolution across `ui/node_modules` and root `node_modules`, or completes the singleton alias set including `signal-polyfill` and `signal-polyfill/`. Add regression coverage that proves both `.bundle.hash` and `a2ui.bundle.js` stay stable across install layouts, then close as implemented after the fix merges.
