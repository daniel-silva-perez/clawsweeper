## Summary

Keep open. Current main partially improved the hash calculation, but the A2UI generator is still install-layout-sensitive: bundling still prefers `ui/node_modules`, still lacks a `signal-polyfill` singleton alias, and tests do not prove byte-stable generated artifacts across the two layouts. The related open PR #68934 looks like an implementation candidate, not a reason to close this issue before a fix lands.

## Best Possible Solution

Fix the generator path, not just the checked-in artifacts: make A2UI dependency resolution canonical across `ui/node_modules` and root `node_modules`, or add a complete singleton alias set including `signal-polyfill` and `signal-polyfill/`. Then add regression coverage that proves the generated bundle and hash stay stable with and without `ui/node_modules`. Review and land #68934 or a successor patch if it covers that full generator guarantee.
