## Summary

Keep open. Current main still has the source-level clobber path for real provider auth methods: provider `configPatch` output can write a new primary before the preservation helper reads the previous primary, and xAI/OpenRouter still emit that shape. PR #70793 established the intended preserve-vs-explicit-set-default contract, and the Keep My Claw comment is a useful recovery workaround, but neither removes the remaining OpenClaw config mutation bug.

## What This Changes

The issue asks OpenClaw onboarding/configure to preserve an existing `agents.defaults.model.primary` when adding a second provider unless the user explicitly chooses to change the default model.

## Best Possible Solution

Carry the pre-auth primary model through the provider-auth application path and use that original value for preservation decisions before merging provider-owned patches. Only replace the primary when there was no prior primary or the caller explicitly requested a default-model change, and lock the behavior with real xAI/OpenRouter-style provider-patch coverage.

## Repair Work Prompt

Fix the provider-auth default-model preservation bug tracked by this issue. Current main lets real provider auth methods such as xAI/OpenRouter return both `configPatch` and `defaultModel`; the patch can write `agents.defaults.model.primary` before `applyDefaultModelFromAuthChoice` reads the previous primary, so configure/onboard can preserve the wrong value. Keep the fix inside the generic provider-auth application path unless a minimal SDK helper adjustment is clearly needed. Preserve the pre-auth primary before merging provider config patches, and only change the primary for first setup or explicit default-setting flows such as `openclaw models auth login --provider <id> --set-default` and `openclaw models set`. Do not remove intentional `replaceDefaultModels` migration behavior. Add regression coverage using a provider method that returns a configPatch containing `agents.defaults.model.primary` plus `defaultModel`, and include real xAI/OpenRouter-style coverage where practical. Update the changelog with a user-facing fix entry crediting @rixcorp-oc and any source PR context required by repo policy. Relevant refs: this issue, PR #70793, and #70696.
