## Summary

Current main still does not fully fix the reported provider-default clobber. Configure now opts into `preserveExistingDefaultModel`, but provider auth first merges a provider `configPatch`; for real providers such as xAI/OpenRouter that patch can already contain `agents.defaults.model.primary`, so the later preservation step sees the newly clobbered primary as the baseline. Rerun onboarding also calls `applyAuthChoice` with `setDefaultModel: true` and no preservation flag. The Keep My Claw comment is a useful restore workaround, but it does not resolve the core overwrite path.

## Best Possible Solution

Keep this open and fix the generic provider auth flow so provider setup can add provider config, auth profiles, and allowlist entries without clobbering an existing `agents.defaults.model.primary` unless the caller explicitly chooses to change the default. The fix should cover `openclaw configure`, classic onboarding reruns, and real provider `configPatch` values that already contain `agents.defaults.model.primary`, with regression coverage using an xAI/OpenRouter-style patch.
