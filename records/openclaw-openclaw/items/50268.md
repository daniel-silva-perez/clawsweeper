## Summary

Keep open. Current main still has the provider-patch ordering gap: real xAI/OpenRouter-style auth methods can return a config patch that writes `agents.defaults.model.primary` before the preservation helper reads the previous primary. Merged #70793 documents and partly enforces the intended configure contract, and Keep My Claw is a useful recovery workaround, but neither removes this OpenClaw config mutation path.

## What This Changes

The issue reports that adding a second provider via onboard/configure can overwrite an existing `agents.defaults.model.primary`, leaving the previous provider unusable without manual config repair.

## Best Possible Solution

Carry the original pre-auth default model through the provider-auth application path and base preservation decisions on that original value, while still allowing explicit default-setting flows to overwrite it. Add regression coverage using a provider patch that includes `agents.defaults.model.primary`, preferably modeled after xAI/OpenRouter behavior.

## Repair Work Prompt

Fix the provider-auth default-model preservation bug tracked by openclaw/openclaw#50268. Current `openclaw configure` asks `applyAuthChoice` to preserve the existing primary, but `runProviderPluginAuthMethod` merges provider `configPatch` before preservation reads the previous primary, so xAI/OpenRouter-style patches that include `agents.defaults.model.primary` can replace Bedrock or another existing primary. Implement a narrow fix, likely in `src/plugins/provider-auth-choice.ts`, that bases preservation on the original pre-auth config while still allowing explicit default-setting flows such as `openclaw models auth login --provider <id> --set-default` and first-run/no-existing-primary onboarding to set the new provider default. Add regression coverage with a plugin/provider auth method whose `configPatch` includes `agents.defaults.model.primary` and returns `defaultModel`; use xAI/OpenRouter-style behavior where practical. Do not change provider SDK contracts except for a backward-compatible internal seam if needed, and do not weaken explicit `--set-default` semantics. Add a single-line changelog fix entry under the active version thanking @rixcorp-oc. Validate with the targeted provider-auth/configure tests and `pnpm check:changed` through the normal Testbox gate if promoted by a maintainer.
