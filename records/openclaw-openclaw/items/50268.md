## Summary

Keep open. Current main still has a deterministic clobber path: provider auth can merge a config patch that already rewrites `agents.defaults.model.primary`, then the preservation helper reads that mutated value as the previous primary. The recovery plugin mentioned in the discussion is useful context, but it does not fix the OpenClaw config mutation bug.

## What This Changes

Preserve an existing `agents.defaults.model.primary` when adding a second provider through onboard/configure unless the user explicitly chooses to change the default model.

## Best Possible Solution

Make preservation decisions from the pre-auth configuration, then merge provider-owned patches and default-model allowlist additions without changing an existing primary unless the flow is fresh setup or an explicit default-setting command. Cover both configure and re-run onboard with xAI/OpenRouter-style patch-plus-default regression tests.

## Repair Work Prompt

Fix the provider-auth default-model preservation bug tracked by https://github.com/openclaw/openclaw/issues/50268. Current main can merge a provider `configPatch` that already writes `agents.defaults.model.primary` before `applyDefaultModelFromAuthChoice` reads the previous primary, so `preserveExistingDefaultModel` is defeated for xAI/OpenRouter-style providers that return both `configPatch` and `defaultModel`. Carry the pre-auth primary/default model through the provider auth application path, or otherwise base preservation on the original config before applying provider-owned patches. Cover both `applyAuthChoiceLoadedPluginProvider` and `applyAuthChoicePluginProvider` as needed. Preserve explicit default-changing behavior for `openclaw models auth login --provider ... --set-default`, `openclaw models set`, and fresh setup with no existing primary. Do not move provider-owned catalog/default definitions into core and do not weaken SecretRef/auth-profile behavior. Add regression tests using a realistic provider patch that contains `agents.defaults.model.primary`, plus xAI/OpenRouter-style coverage where practical. Update docs only if the user-facing contract changes; otherwise add a one-line changelog Fixes entry credited with `Thanks @rixcorp-oc`.
