## Summary

Keep open. Current main still merges a provider auth `configPatch` before default-model preservation reads the previous primary; xAI/OpenRouter-style API-key providers can set `agents.defaults.model.primary` in that patch, so the documented configure preservation contract is still not guaranteed. The Keep My Claw comment is a useful recovery workaround, and merged #70793 established the desired contract, but the source-level clobber path remains.

## What This Changes

Adding a second provider through onboard/configure should preserve an existing primary model unless the user explicitly chooses to change it.

## Best Possible Solution

Carry the pre-auth primary model through the provider-auth application path and base preservation decisions on that original value, while still allowing explicit default-setting flows and no-existing-primary setup to choose the new provider default. Add regression coverage for provider auth methods whose config patches already include `agents.defaults.model.primary`.

## Repair Work Prompt

Fix the provider-auth default-model preservation bug for https://github.com/openclaw/openclaw/issues/50268. Current behavior: configure/onboard can merge a provider `configPatch` that writes `agents.defaults.model.primary` before `applyDefaultModelFromAuthChoice` reads the previous primary, so xAI/OpenRouter-style API-key providers can clobber an existing Bedrock or other primary despite `preserveExistingDefaultModel: true`. Keep the fix within the provider-auth/config application path; do not change explicit default-changing behavior such as `models auth login --provider <id> --set-default`, first-run/no-existing-primary setup, or agent override behavior. Preserve plugin API compatibility. Add regression coverage with a provider auth method whose `configPatch` already contains `agents.defaults.model.primary`, and include xAI/OpenRouter-style coverage where practical. Existing docs already describe the desired configure behavior, so only update docs/changelog if the implemented surface changes in a user-facing way.
