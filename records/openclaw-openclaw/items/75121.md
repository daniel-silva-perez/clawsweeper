## Summary

Keep open. The PR is moving in the right direction by carrying Telegram forward attribution as typed metadata, but latest head still maps forwarded-batch `body_line` from the original debounced entry index while `BodyForAgent` is built only from non-empty text entries, so mixed media-only/text forwarded bursts can produce misleading prompt attribution.

## What This Changes

The PR keeps Telegram forwarded-message text clean in `BodyForAgent`, adds per-item `UntrustedStructuredContext` for coalesced forwarded bursts, strips inherited forward metadata from synthetic burst messages, and adds forwarded-context regression tests.

## Best Possible Solution

Keep the typed structured-context approach, but compute `body_line` from the actual non-empty text entries that survive into `BodyForAgent`, or omit `body_line` for media-only forwarded entries and rely on `item_index` plus `message_id`; add a regression for media-only forward before text.

## Reproduction Assessment

Yes. A high-confidence static reproduction is a debounced Telegram forward burst where the first forwarded update has media but no text/caption and the second has text; the joined body has one line while the second entry reports `body_line: 2`.

## Solution Assessment

No, not yet. The structured metadata design is the narrow maintainable direction, but the line reference must match the final joined prompt body before the patch is safe to merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Align forwarded batch body_line with joined text:** `extensions/telegram/src/bot-handlers.runtime.ts:233`
  - body: Forwarded burst flushing builds `BodyForAgent` only from non-empty text entries, but this metadata uses the original debounced entry index. If a media-only forwarded message arrives before a text forward, the text appears on line 1 while its metadata says `body_line: 2`, giving the model misleading attribution. Track line numbers only for entries that contribute text, or omit `body_line` for media-only entries.
