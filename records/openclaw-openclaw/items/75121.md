Current main still represents a debounced Telegram forwarded burst as one synthetic message with only the first forwarded origin, while this PR adds the missing per-item forwarded attribution and appears to have addressed the earlier ClawSweeper text-fragment regression. ## What This Changes

The PR updates Telegram forwarded-message handling so direct forwards keep clean agent bodies and forwarded bursts carry per-message attribution through typed untrusted structured context, with regression coverage for direct, self-forwarded, burst, media-only, visibility-filtered, prompt-rendered, and long-fragment cases.

## Best Possible Solution

Land the typed metadata approach after normal maintainer review: preserve the existing ForwardedFrom* path for single forwards, use per-item UntrustedStructuredContext for multi-message forwarded bursts, keep group visibility filtering on forwarded origins, and retain the long-fragment regression so future helper changes do not drop attribution.
