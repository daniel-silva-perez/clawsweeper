## What This Changes

The PR changes Telegram forwarded-message handling so single forwards keep clean agent bodies and forwarded bursts carry per-message forwarded attribution through untrusted structured context.

## Best Possible Solution

Land the typed structured-context approach after maintainer review: keep single forwards on existing ForwardedFrom* fields, use forwardedBatchContext only for coalesced Telegram bursts, preserve group visibility filtering, and require the Telegram changed gate before merge.
