## Summary

Current main still does not expose WhatsApp forwarding metadata to plugin hooks. The Baileys `contextInfo` object supports `forwardingScore` and `isForwarded`, and OpenClaw's WhatsApp extractor can find `contextInfo`, but the normalized WhatsApp inbound message, finalized message context, canonical hook context, and `inbound_claim`/`message_received` event metadata do not carry those fields.

## Best Possible Solution

Keep this issue open and implement a narrow additive bridge: extract WhatsApp `contextInfo.forwardingScore` and `contextInfo.isForwarded`, normalize `isForwarded` as true when Baileys says true or `forwardingScore >= 1`, carry both through the WhatsApp inbound message/finalized context/canonical hook context, and expose them to `inbound_claim` and `message_received` plugin events, preferably in the existing metadata object unless maintainers want first-class typed event fields.
