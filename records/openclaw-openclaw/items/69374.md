## Summary

Keep open. Current main still does not expose WhatsApp Baileys `contextInfo.isForwarded` or `contextInfo.forwardingScore` to plugin inbound hooks: the WhatsApp plugin finds `contextInfo`, but forwarding metadata is not extracted, carried through `WebInboundMessage` or `FinalizedMsgContext`, added to `CanonicalInboundMessageHookContext`, or emitted in `inbound_claim` / `message_received` metadata. Related PR #46291 is open and targets the same WhatsApp forwarding source path, so the right next step is maintainer review/adaptation rather than closing this issue or opening a duplicate automated repair.

## What This Changes

Expose WhatsApp Baileys `contextInfo.isForwarded` and `contextInfo.forwardingScore` through inbound plugin hook payloads so plugins can distinguish forwarded messages from user-authored text.

## Best Possible Solution

Keep this issue open and review/adapt #46291 or land an equivalent current-main change that extracts WhatsApp `contextInfo.isForwarded` / `contextInfo.forwardingScore`, normalizes `isForwarded` when Baileys says true or `forwardingScore >= 1`, carries both through `WebInboundMessage`, `FinalizedMsgContext`, and `CanonicalInboundMessageHookContext`, and exposes them in `inbound_claim` and `message_received` metadata with focused WhatsApp hook tests and plugin docs if the public hook contract changes.
