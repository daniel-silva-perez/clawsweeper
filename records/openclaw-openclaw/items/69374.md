## Summary

Current main still does not expose WhatsApp `contextInfo.forwardingScore` or `contextInfo.isForwarded` to plugin inbound hooks. The WhatsApp extractor can locate Baileys `contextInfo`, but forwarding metadata is not read, stored on `WebInboundMessage`, bridged into `FinalizedMsgContext`, added to `CanonicalInboundMessageHookContext`, or emitted in `inbound_claim` / `message_received` metadata. There is also an open related implementation PR, #46291, so this issue should stay open while maintainers review or adapt that work rather than closing it or spawning a duplicate automated PR.

## Best Possible Solution

Keep this issue open. The best path is to review/adapt #46291 or land an equivalent current-main change that extracts WhatsApp `contextInfo.isForwarded` / `contextInfo.forwardingScore`, normalizes `isForwarded` when Baileys says true or `forwardingScore >= 1`, carries both through `WebInboundMessage`, `FinalizedMsgContext`, and `CanonicalInboundMessageHookContext`, and exposes them in `inbound_claim` and `message_received` metadata with focused tests and plugin docs.
