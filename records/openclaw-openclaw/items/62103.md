## Summary

Keep open. Current main has partial Google Vertex metadata, but it does not implement this PR's central runtime/onboarding behavior: no standalone `google-vertex` runtime provider is registered, `google-vertex` remains a hook alias of the Google AI Studio provider, there is no current Vertex region/base URL builder, and the auth choices still expose only Gemini API key and Gemini CLI OAuth. The linked #60736 remains open, and this PR is still a plausible implementation candidate rather than cleanup-closeable. Security pass: the supplied PR diff is scoped to Google plugin source/tests, not workflows/dependencies/scripts; it does introduce regional Google endpoint construction, and its second commit addresses the reviewed hostname/region-validation concern, but the later catalog/auth review comment still needs maintainer review before merge.

## Best Possible Solution

Keep this PR open for maintainer review as an implementation candidate for #60736. The best path is to rebase or fold it into the chosen Google Vertex implementation, preserving the useful region/base URL validation and tests, while aligning with current manifest ownership, runtime provider registration, auth-aware ADC/OAuth discovery, and onboarding/auth-choice metadata.
