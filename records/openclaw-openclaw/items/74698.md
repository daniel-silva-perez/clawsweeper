## Summary

Keep open. The report remains valid against current main: Telegram text command routing can still mark a whole-message `/status` command unauthorized when the effective group command allowlist is empty, then return no reply, and PR #74894 is still open and unmerged with closing syntax for this issue.

## What This Changes

The issue asks for Telegram group slash commands such as `/status` to produce a visible Gateway status reply in configured group sessions where ordinary group text still works.

## Best Possible Solution

Review and land PR #74894, or replace it with an equivalent focused Telegram fix, so configured group `/status` commands reply in the originating group or topic while explicit sender allowlists, owner-only commands, and disabled group/topic settings still fail closed.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a configured Telegram group where normal messages pass but `allowForCommands.hasEntries` is false; `/status` is denied by the current command gate and the reply path returns `undefined` for the whole-message control command.

## Solution Assessment

Yes. The best maintainer path is to review the linked PR or an equivalent narrow patch that adjusts the message-level gate without broadening the native command or owner authorization model.
