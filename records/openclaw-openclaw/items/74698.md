## Summary

Keep open. Current main still has the Telegram inbound-body command gate that can silently drop group control commands when no command/group allowlist is configured, and the diff from `v2026.4.27` to main does not touch that file. Open PR #74894 explicitly closes this issue, so the right maintainer path is to review, land, or supersede that PR rather than close the issue early.

## What This Changes

Telegram group `/status` slash commands no longer produce a visible status reply in configured group sessions even though normal group messages still reach the agent.

## Best Possible Solution

Review and land PR #74894, or replace it with an equivalent focused change in `extensions/telegram/src/bot-message-context.body.ts`, plus regression coverage for group `/status` when no command/group allowlist is configured and for explicit allowlist rejection cases.

## Reproduction Assessment

Yes. The source-level path is high confidence: with default `commands.useAccessGroups`, a group `/status` message with no effective command/group allowlist makes the early control-command gate unauthorized and returns `null`, while normal non-command text bypasses that block.

## Solution Assessment

Yes. Keeping the issue open for PR #74894 review is the best maintainer path because there is an open closing implementation candidate and current main does not yet contain its focused early-gate change.
