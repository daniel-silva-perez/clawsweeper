## Summary

Current main still does not implement the PR’s intended Slack behavior. Slack room history remains pending-only: skipped mention-gated messages are buffered, but processed `requireMention=false` channel messages are not retained after dispatch, and the current regression test still asserts that the second always-on channel message does not see the first as history. The PR remains a plausible behavior change, but it needs maintainer product judgment, a rebase onto the current `extensions/slack` layout, and renewed handling of the ordering/privacy semantics before cleanup closure would be appropriate.

## Best Possible Solution

Keep this PR open for maintainer review. If always-on Slack channel context is the desired product behavior, rebase the work into `extensions/slack/src/monitor/message-handler/*`, carry the resolved mention policy into dispatch or record the processed message at prepare time, retain `requireMention=false` room messages in both reply and no-reply outcomes with deterministic ordering, update the current Slack regression tests, and revise Groups/Slack docs away from the pending-only model. If maintainers prefer the current pending-only semantics, close the PR with that explicit product rationale rather than automated cleanup.
