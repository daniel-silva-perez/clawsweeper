## Summary

Keep open: current main still has a concrete OpenAI-compatible request-shaping gap for strict Mindie-style backends. The schema now accepts generic compat flags and preserves explicit streaming-usage overrides, but the Chat Completions payload builder still sends `stream_options.include_usage` unconditionally, so the reporter's strict 422 path remains actionable.

## What This Changes

The issue asks OpenClaw to stop sending unsupported OpenAI-compatible request parameters that cause a local Mindie DeepSeek endpoint to return HTTP 422.

## Best Possible Solution

Make the OpenAI Chat Completions transport honor `compat.supportsUsageInStreaming: false` by omitting `stream_options`, preserve usage reporting for native or explicitly compatible endpoints, and keep any additional strict-provider handling behind generic compat/schema-backed request-shaping rather than Mindie-specific IDs.

## Repair Work Prompt

Fix the strict OpenAI-compatible streaming request-parameter gap tracked by this issue. Current main accepts and preserves `models.providers.<provider>.models[].compat.supportsUsageInStreaming: false`, but `buildOpenAICompletionsParams()` in `src/agents/openai-transport-stream.ts` still unconditionally sends `stream_options: { include_usage: true }` for streaming Chat Completions requests. Change the transport so an explicit false compat value omits `stream_options`, while native OpenAI-compatible endpoints and explicit true custom providers continue to send streamed usage. Keep the fix generic; do not hardcode Mindie, DeepSeek, provider IDs, endpoint strings, or one-off migration behavior. Preserve existing tool-disable behavior through `compat.supportsTools: false`; do not add unsupported top-level config keys like `supportsStreamOptions` or `supportsToolChoice`. Add regression coverage for explicit false, keep existing true/native/local behavior covered, and update docs/changelog only if the public config guidance changes. Add a single-line changelog Fixes entry crediting @Zha-Miku if maintainers consider the behavior user-facing.
