## Summary

Keep open: current main still leaves strict OpenAI-compatible Mindie endpoints without an effective opt-out for the streaming usage request field. The repair is narrow, but it must preserve the local/proxy token-usage behavior added for compatible backends.

## What This Changes

A user reports HTTP 422 failures from a Mindie-hosted DeepSeek model using an OpenAI-compatible Chat Completions endpoint because OpenClaw sends request parameters the backend rejects.

## Best Possible Solution

Add generic OpenAI-compatible request shaping in the Chat Completions transport so strict backends can omit unsupported streaming usage parameters through schema-backed compat configuration, while preserving streamed usage for known-compatible, native, or intentionally opted-in endpoints and avoiding Mindie-specific provider IDs.

## Repair Work Prompt

Fix the strict OpenAI-compatible backend regression tracked by https://github.com/openclaw/openclaw/issues/45706. Current `buildOpenAICompletionsParams()` always sends `stream_options: { include_usage: true }` for streaming Chat Completions, even when a model is configured for a strict backend that rejects that field. Implement a generic fix that lets `models.providers.<provider>.models[].compat.supportsUsageInStreaming: false` omit `stream_options` for strict providers, while preserving token-usage reporting for native/default local/proxy endpoints and the behavior introduced by #68746 when no explicit strict opt-out is intended. Do not hardcode Mindie provider IDs. Do not solve #50374 message-key sanitization in this PR unless a minimal shared helper is unavoidable. Add focused regression coverage in `src/agents/openai-transport-stream.test.ts` for explicit false omitting `stream_options`, explicit true/native-compatible endpoints still sending it, and known local/proxy behavior remaining compatible. If the implementation needs to preserve whether a false value was user-explicit versus normalization-defaulted, keep that as a generic compat/schema-backed seam and update docs/config generation only if the public config contract changes. Changelog: add a single-line user-facing fix entry with reporter/contributor attribution if runtime behavior changes.
