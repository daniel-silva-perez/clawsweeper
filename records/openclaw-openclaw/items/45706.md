## Summary

Keep this issue open. Current main still has the Mindie/OpenAI-compatible 422 gap: the reporter's suggested `supportsStreamOptions` and `supportsToolChoice` config keys remain unsupported, `buildOpenAICompletionsParams` still always sends `stream_options.include_usage`, and proxy-like Chat Completions endpoints still default `tool_choice` to `auto` whenever tools are present.

## Best Possible Solution

Keep this issue open and fix the generic OpenAI-compatible request-shaping path. First, make `compat.supportsUsageInStreaming: false` omit `stream_options` in `buildOpenAICompletionsParams` while preserving it for native and known-compatible local providers. Then decide whether Mindie-style `tool_choice` rejection should be handled by a narrow documented compat flag or by documenting `compat.supportsTools: false` as the supported mitigation.
