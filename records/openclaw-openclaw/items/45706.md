## Summary

Current main has added some stricter OpenAI-compatible backend knobs, but this Mindie 422 report is not safely closeable. The exact workaround suggested in the issue discussion used unsupported keys, and current request construction still lacks a field-level way to suppress some parameters Mindie may reject, especially `stream_options` and `tool_choice` while keeping tools enabled.

## Best Possible Solution

Keep this issue open. The likely maintainer path is to add a tested, documented compatibility control for stricter OpenAI-compatible backends: make `compat.supportsUsageInStreaming: false` actually omit `stream_options`, decide whether a narrow `tool_choice` omit control is needed for backends that still accept tools, and add regression coverage using a strict OpenAI-compatible/Mindie-shaped mock provider.
