## Summary

Current main still leaves the Mindie/OpenAI-compatible 422 path actionable: the config schema has the relevant generic compat knobs, but the Chat Completions request builder still sends `stream_options.include_usage` unconditionally and still defaults proxy-like tool requests to `tool_choice: "auto"` when tools are present. The issue discussion is concrete and recent enough to keep open, and the previous workaround suggestion used unsupported key names.

## What This Changes

The issue requests that local Mindie/DeepSeek OpenAI-compatible requests avoid HTTP 422 validation failures caused by unsupported OpenAI request parameters.

## Best Possible Solution

Add a generic OpenAI-compatible request-shaping fix that honors strict endpoint compat without adding Mindie-specific IDs to core: `compat.supportsUsageInStreaming: false` should omit `stream_options`, compatible providers should keep streamed usage, and the tool-choice mitigation should be either an existing documented `compat.supportsTools: false` path or a schema-backed compat flag with tests and docs.

## Repair Work Prompt

Fix the Mindie/strict OpenAI-compatible 422 path from issue 45706. Keep the fix generic: do not add Mindie-specific provider IDs or dependency strings in core. The first required change is for `buildOpenAICompletionsParams()` to honor `compat.supportsUsageInStreaming: false` by omitting `stream_options`, while preserving `stream_options.include_usage` for native/known-compatible endpoints and explicit true overrides. Add focused regression tests for a custom `openai-completions` provider with `compat.supportsUsageInStreaming: false`. For `tool_choice`, preserve the PR #71472 default for proxy providers that need it, and either document/test `compat.supportsTools: false` as the supported strict-endpoint mitigation or add a narrowly named schema-backed compat opt-out with docs/tests if the existing mitigation is insufficient. Do not broadly strip message keys here; keep #50374 as separate follow-up unless the maintainer explicitly asks to fold it in. Update docs/changelog if runtime config behavior changes.
