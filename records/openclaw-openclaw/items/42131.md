## Summary

Keep this PR open. Current main still has the safe-bin casing mismatch the PR targets: configured safeBins are lowercased, while trusted directories and resolved executable directories are stored and compared with exact casing. The PR remains a focused candidate for a current core exec-policy bug, but its current tmpdir-wide case-sensitivity probe needs maintainer/security review before merge.

## Best Possible Solution

Keep this PR open for maintainer review or revision. The best path is a shared safe-bin trusted-path comparison helper that normalizes both sides consistently only for genuinely case-insensitive paths, preserves distinct paths on case-sensitive or mixed-sensitivity volumes, and adds regression coverage for mixed-case doctor/runtime matching plus a case-sensitive sibling non-trust boundary before merge.
