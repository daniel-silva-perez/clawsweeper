## Summary

Keep this PR open. Current main still has the exact safe-bin trust mismatch described by the PR: doctor lowercases configured safeBins, while trusted directories and resolved executable directories are stored and compared with exact casing. The PR is a focused candidate for a current core exec-policy bug, but its latest tmpdir-wide case-sensitivity probe still needs maintainer review because it can weaken the trust boundary on mixed-sensitivity mounts.

## Best Possible Solution

Keep this PR open for maintainer review or revision. The best path is a shared, testable safe-bin path comparison helper that compares configured trusted dirs and resolved executable dirs consistently on genuinely case-insensitive paths, preserves distinct paths on case-sensitive or mixed-sensitivity volumes, and adds deterministic regression coverage for doctor/runtime mixed-case matching plus the case-sensitive sibling trust boundary.
