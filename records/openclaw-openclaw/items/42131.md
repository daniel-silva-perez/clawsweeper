## Summary

Keep open. Current main still has the safe-bin casing mismatch this PR targets: doctor lowercases configured safeBins, while trusted directories and resolved executable directories are compared with exact casing. The PR is still a focused candidate for a real exec-policy bug, but its current tmpdir-wide filesystem case-sensitivity probe remains security-sensitive and should be revised or explicitly accepted by maintainers before merge.

## Best Possible Solution

Keep this PR open for maintainer review or revision. The best path is a shared safe-bin trusted-path comparison helper that normalizes both sides consistently only for genuinely case-insensitive trusted paths, preserves distinct paths on case-sensitive or mixed-sensitivity volumes, and adds regression coverage for mixed-case doctor/runtime matching plus a case-sensitive sibling non-trust boundary before merge.
