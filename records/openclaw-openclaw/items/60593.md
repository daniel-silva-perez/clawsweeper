## Summary

Keep this issue open. Current main has partial progress: Anthropic SSE JSON SyntaxErrors are converted to a stable transport sentinel and that sentinel is sanitized for users, but the sentinel is not classified as a failover reason, so the reported recovery/failover behavior is still incomplete.

## What This Changes

The issue requests consistent retry/failover for Anthropic streaming JSON parse failures, no raw JSON.parse SyntaxErrors in user-facing replies, and more resilient handling of malformed or truncated streaming JSON.

## Best Possible Solution

Add a focused agent/provider error-boundary fix: classify `OpenClaw transport error: malformed_streaming_fragment` as a retryable transport/failover reason, preserve the existing friendly user-facing message, and add regression coverage proving malformed Anthropic stream failures trigger profile/model failover without hiding normal provider request-validation diagnostics.

## Solution Assessment
