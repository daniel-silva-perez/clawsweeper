## Summary

Keep this issue open. Current main has some adjacent hardening, including tolerant tool-call partial parsing and a new exact "An unknown error occurred" stream fallback classifier, but it still does not classify the reported Anthropic streaming JSON SyntaxError shape as retryable/failover-worthy and can still format the raw SyntaxError text for channel delivery. The related PR #59118 is useful prior art but remains open and is scoped to Discord last-mile sanitization, not Telegram plus embedded failover recovery.

## Best Possible Solution

Fix this in the agent/provider error boundary rather than closing it: add a precise streaming JSON parse SyntaxError detector, classify it as retryable/failover-worthy for the embedded fallback path, sanitize the user-facing message across channels including Telegram, and add regression tests for `classifyFailoverReason`, `formatAssistantErrorText`, `sanitizeUserFacingText`, and the assistant failover decision path. Review #59118 as prior art, but it should not be treated as resolving this broader issue unless it is merged and extended beyond Discord formatting.
