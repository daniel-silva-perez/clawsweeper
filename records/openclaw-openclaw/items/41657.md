## Summary

Keep this issue open. Current main no longer fully strips Slack files: Slack file id/name/MIME/size are present at ingestion, and Slack can download media or expose filename/fileId placeholders. The requested passive structured attachment metadata delivery to agents is still missing from Slack final context and the shared MsgContext/plugin contract; the related open Slack PR is narrower media hydration work, not a replacement for this contract.

## Best Possible Solution

Keep this issue open and implement a backward-compatible shared inbound attachment metadata contract in MsgContext and the plugin SDK. Populate it from Slack message.files with provider, provider file id, file name, MIME type, size, downloaded/local-path state, and placeholder text while preserving existing BodyForAgent placeholders and MediaPath/MediaPaths/MediaTypes behavior. Add Slack and shared channel contract tests, update Slack/plugin docs, and review #62792 separately as the narrower Slack media hydration fix.
