## Summary

Keep open. Current main still handles hard sqlite-vec load failures but does not verify that a successful native load registered usable vec0/functions, which is the failure mode reported and independently confirmed in the discussion.

## What This Changes

The issue reports that OpenClaw v2026.4.11 on Linux can load sqlite-vec 0.1.9 against a newer SQLite runtime while registering no vec0/functions, degrading memory vector writes and semantic search.

## Best Possible Solution

Add a narrow post-load sqlite-vec health probe in the memory host/sdk path, base vector availability on that probe, preserve the existing in-process cosine fallback with an actionable load error when the probe fails, and add regression coverage for load success with missing vec0/functions.
