## Summary

Keep this PR open. Current main still slugifies the first embedded-agent payload text directly, strips edge dashes before truncation, and persists any non-null slug into the session-memory filename. The merged timeout work does not cover this error-payload or trailing-dash bug, and the broader #38161 slug-hardening PR remains open rather than shipped.

## Best Possible Solution

Keep this PR open as a focused bug-fix candidate. Review and land this PR or an equivalent change that rejects error-marked and non-slug-like embedded-agent payloads before filename construction, strips leading/trailing dashes after truncation, and keeps focused regression tests for the pure slugification behavior. Coordinate the final shape with #38161.
