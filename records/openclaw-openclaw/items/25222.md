## Summary

Keep this issue open. Current main still queues or drops same-session contention, with only a narrow shutdown-guard reply, and does not provide the requested immediate channel-visible busy/status response with cancel/new-session guidance.

## Best Possible Solution

Keep this open and implement a generic active-session busy-reply path around the active-run queue policy, runReplyAgent, and channel dispatch. It should produce configurable, channel-safe status text/action guidance for ordinary same-session contention, optionally expose reason codes, and add focused tests for queue policy, runner behavior, and at least one bridge/dispatch path.
