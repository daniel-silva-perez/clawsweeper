## Summary

Keep this issue open. Current main still queues or drops ordinary same-session contention and does not provide the requested immediate channel-visible busy/status reply with cancel or new-session guidance.

## Best Possible Solution

Keep this issue open and implement a generic active-session busy-reply path around active-run queue policy, runReplyAgent, and channel dispatch. It should return configurable channel-safe status text/action guidance for ordinary same-session contention, optionally expose reason codes, and add focused tests for queue policy, runner behavior, dispatch, and at least one bridge path.
