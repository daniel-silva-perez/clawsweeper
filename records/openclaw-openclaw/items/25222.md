## Summary

Keep this issue open. Current main still queues same-session active-run messages without an immediate channel-visible busy/status reply, cancel/new-session guidance, or stable busy reason payload. Related work exists, but it is not merged and does not close-reference this issue.

## Best Possible Solution

Keep this issue open as the canonical UX request. Implement a generic active-session busy-reply path around queue policy, runReplyAgent, and dispatch: configurable channel-safe status text, optional /stop and /new guidance, stable busy reason metadata, and focused tests for queue policy, runner return payloads, dispatch, and at least one channel path. Evaluate #39117 as an implementation candidate, and fold or duplicate-close #50600 only after maintainers choose the canonical design.
