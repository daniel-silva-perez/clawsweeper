## Summary

Keep this issue open. Current main still handles same-session active-run contention by steering, enqueueing followups, dropping heartbeats, or waiting for shutdown, but it does not return the requested immediate channel-visible busy/status reply with cancel/new-session guidance for ordinary same-session messages.

## Best Possible Solution

Keep this issue open. The best path is a generic, configurable active-session busy-reply path around `resolveActiveRunQueueAction`, `resolvePreparedReplyQueueState`, and `runReplyAgent`, with channel-safe text/action guidance plus focused runner and bridge regression tests for same-session active-run contention.
