## Summary

Keep this issue open. Current main still queues same-session contention instead of returning an immediate channel-visible busy reply with cancel/new-session guidance. The existing busy/status pieces are either internal queue/account state, Control UI local queue UX, or a narrow shutdown edge case, not the requested generic bridge response.

## Best Possible Solution

Keep this open and implement a generic active-session busy-reply path around queue policy, runReplyAgent, and dispatch. It should send configurable channel-safe status text when a same-session message is queued, include safe /stop and /new guidance where appropriate, optionally expose reason codes, and add focused tests for queue policy, runner return payloads, dispatch, and at least one bridge path.
