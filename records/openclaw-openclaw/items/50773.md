## Summary

Keep this issue open. Current main still only skips/retries heartbeats for the existing `requests-in-flight` path covering the main queue and resolved session lane; it does not implement the requested cron-lane guard, `cron-in-progress`/`lanes-busy` skip reasons, or `heartbeat.skipWhenBusy` config. The linked implementation PR #50774 was closed unmerged, so this remains actionable core scheduler work rather than already-shipped cleanup.

## Best Possible Solution

Keep the issue open and implement the behavior in core heartbeat scheduling: add an always-on guard for active cron/cron-nested work, add an opt-in `heartbeat.skipWhenBusy` config for broader lane checks if maintainers still want that policy, and update the wake retry/no-schedule-advance logic plus focused tests for cron and all-lane busy cases.
