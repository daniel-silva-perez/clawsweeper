## Summary

Keep this PR open. Current main does not implement the PR’s intended routing change: the A2A `sessions_send` announce target is still resolved from the target session/display key, and the existing regression test still expects delivery to the target-side route. The PR is a focused source/test change with no obvious security-sensitive or supply-chain surface, but it needs maintainer review/rebase against newer main behavior.

## Best Possible Solution

Keep the PR open and have maintainers review or ask for a rebase. The best fix is a targeted update in `src/agents/tools/sessions-send-tool.a2a.ts` that resolves final announce delivery from the requester session when `requesterSessionKey` is present, falls back to the target route when it is absent, keeps target-side channel metadata for prompt/provenance context, preserves current per-session nested lane behavior, and carries the requester-vs-target regression coverage in `src/agents/openclaw-tools.sessions.test.ts`.
