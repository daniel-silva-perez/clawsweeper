## Summary

Keep this PR open. Current main still resolves the A2A sessions_send announce delivery target from the target session/display key, while this PR specifically changes that delivery target to the requester session and adds requester-vs-target regression coverage. The PR is stale against newer lane-selection code, but its intended functional change is not implemented on main.

## Best Possible Solution

Keep this PR open for maintainer review or rebase. The best path is a targeted requester-side announce-routing fix in src/agents/tools/sessions-send-tool.a2a.ts, with fallback to the target route when requesterSessionKey is unavailable, separate target-side metadata for prompts/provenance, preservation of current per-session nested lane behavior, and regression coverage in src/agents/openclaw-tools.sessions.test.ts.
