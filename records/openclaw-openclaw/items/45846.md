## Summary

Keep open. Current main still leaves the sidebar markdown inline-image path less constrained than normal chat markdown and attachment image rendering, and an open PR with closing syntax directly targets that remaining CSS mismatch. The issue should stay open until PR #46947, or an equivalent verified fix, lands or is deliberately closed.

## What This Changes

The issue reports that Web UI chat images become abnormally large after extended conversations with multiple image messages.

## Best Possible Solution

Land PR #46947 or an equivalent narrow CSS fix that aligns sidebar markdown inline-image sizing with normal chat markdown, then add a focused CSS or visual regression check if the project has a practical UI test hook for that surface.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Unclear: the report gives broad extended-chat steps but no screenshot or log, and I did not run a browser session in this read-only review. The current CSS provides a focused reproduction candidate: render a base64 markdown image through the sidebar markdown reader on a wide sidebar and compare it with the normal `.chat-text` path.

## Solution Assessment

Is this the best way to solve the issue? Likely yes if the oversized image is the sidebar markdown path: matching the sidebar rule to the existing chat markdown cap is narrow and consistent. If live reproduction shows normal chat attachments or another renderer growing instead, the fix should move to that renderer rather than only changing sidebar CSS.
