## Summary

Keep open. Current main still has an uncapped sidebar markdown image width path, and open PR #46947 already references this issue with closing syntax, so the issue should stay open until that PR or an equivalent fix is reviewed and merged.

## What This Changes

The issue asks for Web UI chat images to stay within reasonable bounds during longer conversations instead of growing abnormally large.

## Best Possible Solution

Review and land PR #46947 or an equivalent Control UI CSS change that caps sidebar markdown images consistently with main chat, then verify an image-heavy transcript in both main chat and sidebar before closing. If the original main-chat virtual-scroll behavior still reproduces separately, track that as a follow-up fix in the chat renderer/style path.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? The original extended main-chat timing remains unclear without live browser proof.
