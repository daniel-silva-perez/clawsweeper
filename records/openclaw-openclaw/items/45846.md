Keep open. Current main still has a concrete oversized-image path in sidebar markdown, and open PR #46947 already references this issue with closing syntax, so this issue should remain open until that PR or an equivalent fix is reviewed and merged.

## What This Changes

The issue asks for Web UI chat images to remain at consistent, reasonable dimensions during image-heavy conversations.

## Best Possible Solution

The desired end state is a Control UI CSS change that caps `.sidebar-markdown .markdown-inline-image` consistently with main chat, followed by quick visual verification of image-heavy content in both main chat and the sidebar. Any remaining extended-session main-chat growth should be tracked separately once reproduced.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? ## Solution Assessment

Is this the best way to solve the issue? Yes for the known sidebar path: align the sidebar markdown image cap with the existing main chat rule. The broader issue should remain open until the linked PR is reviewed or maintainers confirm whether a separate main-chat virtual-scroll fix is needed.
