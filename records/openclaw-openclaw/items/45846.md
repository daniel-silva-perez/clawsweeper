## Summary

Keep open. Current main still has a looser sidebar markdown image sizing rule than the main chat renderer, and open PR #46947 already references this issue with closing syntax, so the issue should remain open until that PR or an equivalent fix is reviewed and merged.

## What This Changes

The issue requests that Web UI chat images stay consistently bounded instead of growing oversized during image-heavy conversations.

## Best Possible Solution

Review and land PR #46947 or an equivalent Control UI CSS change that caps sidebar markdown inline images consistently with main chat, then verify against a wide sidebar and an image-heavy transcript before closing. If the original main-chat virtual-scroll path can still reproduce separately, fix that renderer/style path as a follow-up rather than treating the sidebar change as complete.

## Reproduction Assessment

Yes for the sidebar markdown variant: render an allowed base64 data-image markdown image in the markdown sidebar on a wide panel and current CSS lets it expand to full panel width with only a 420px height cap. The original extended-chat timing remains unclear without live browser proof.

## Solution Assessment

Yes for the identified sidebar path: aligning `.sidebar-markdown .markdown-inline-image` with the existing main chat image cap is the narrowest maintainable fix. It should not close the broader issue until maintainers confirm that this is the affected surface or no separate main-chat growth remains.
