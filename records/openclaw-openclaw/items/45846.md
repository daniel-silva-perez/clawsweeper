Keep open. Current main still has a plausible CSS sizing mismatch for sidebar markdown inline images, and an open unmerged PR with closing syntax directly targets that mismatch, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports a Web UI regression where chat images become abnormally large after extended conversations with multiple images.

## Best Possible Solution

Review and land PR #46947, or an equivalent narrow CSS fix that aligns sidebar markdown inline-image caps with normal chat markdown, then close this issue after the fix is merged and verified.
