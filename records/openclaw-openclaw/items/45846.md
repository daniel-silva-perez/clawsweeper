## Summary

Keep open. Current main still leaves the sidebar markdown image rule looser than the main chat rule, and an open unmerged PR already references this issue with closing syntax, so the issue should stay open until that PR is reviewed and either landed or closed.

## What This Changes

Images in the Web UI chat should remain consistently bounded during extended conversations with multiple sent or received images.

## Best Possible Solution

Review and land PR #46947, or an equivalent sidebar markdown image sizing change, then close this issue after the merged fix is verified against a wide sidebar and image-heavy chat transcript. If reproduction shows the main chat or virtual-scroll recycling path is also affected, keep the fix in the Control UI renderer/style layer and broaden only that surface.

## Reproduction Assessment

Yes for the sidebar markdown variant identified by the linked PR: render an allowed base64 data-image markdown image in the markdown sidebar on a wide panel, and current CSS lets it expand to full panel width with a 420px height cap. The original extended-chat timing was not live-reproduced.

## Solution Assessment

Yes for the identified sidebar path. Aligning .sidebar-markdown .markdown-inline-image with the existing main chat image cap is the narrowest maintainable fix, with a manual visual check needed before closing the issue.
