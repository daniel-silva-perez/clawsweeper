## Summary

Keep open. Current main still replaces the Control UI with the login gate whenever `state.connected` is false, so the reported workflow problem remains. The chat view already has disabled-compose behavior for disconnected state, but the top-level gate prevents users from reaching it during a gateway restart/disconnect.

## Best Possible Solution

Keep this as a Control UI enhancement. The best implementation is to split first-load/auth failures from transient gateway disconnects, preserve the current dashboard/chat transcript shell after a successful session has existed, disable write/side-effect controls while offline, surface the reconnect/login action in a banner or overlay, and add focused UI tests for disconnect-after-connected behavior.
