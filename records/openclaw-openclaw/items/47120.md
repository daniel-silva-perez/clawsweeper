## Summary

Keep this issue open. Current main still does not show a last-message preview in the Control UI Sessions list: the UI controller does not request preview/last-message data, the UI row type does not expose `lastMessagePreview`, and the Sessions table renders only the existing relative Updated value plus the key/label columns. The provided timeline also shows open implementation candidates, including PR #47243 with closing syntax and PR #48396 from the issue author, so the issue should remain open until a canonical fix lands or maintainers close the candidate PRs.

## Best Possible Solution

Keep the issue open and review a focused Sessions UI change that wires `lastMessagePreview` into the Control UI safely: update the UI row type, request preview data through `sessions.list includeLastMessage` or a bounded visible-row `sessions.preview` call, render a compact timestamp plus preview in the Sessions table, and add controller/view tests. Once one canonical PR lands, close this issue as implemented and close the duplicate candidate PRs as superseded.
