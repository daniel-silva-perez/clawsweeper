## Summary

Keep open. Current main has backend support for session last-message previews, but the Control UI Sessions list still does not request or render those previews, and an open PR with closing syntax is already paired with this work.

## What This Changes

Request to show a timestamp and last-message preview in the Control UI Sessions list so sessions are easier to identify.

## Best Possible Solution

Land a small Control UI change that requests bounded preview data through the existing `sessions.list includeLastMessage` contract, updates the UI row type and rendering, adds focused controller/view tests, and then lets the merged PR close this issue.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The current-main source gives a high-confidence verification path: open the Control UI Sessions page with multiple sessions and the rows still render key/friendly label plus relative Updated time, without a last-message preview.

## Solution Assessment

Yes, the narrowest maintainable solution is to reuse the existing backend `includeLastMessage` session-list contract rather than adding per-row frontend RPC calls. The open PRs should be reviewed or replaced against that current-main shape.
