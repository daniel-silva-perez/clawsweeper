## Summary

Keep this issue open. Current main still drops `allowUnsafeExternalContent` in the direct `POST /hooks/agent` normalization path, even though the downstream dispatch and cron execution paths already honor the field when it is present.

## Best Possible Solution

Fix the direct `/hooks/agent` path by adding `allowUnsafeExternalContent?: boolean` to `HookAgentPayload`, copying `payload.allowUnsafeExternalContent === true` in `normalizeAgentPayload()`, and adding regression coverage that posts to `/hooks/agent` with the flag and verifies the cron job payload or final prompt is not externally wrapped. Consider updating the `/hooks/agent` docs field list if this body field is intended to be public.
