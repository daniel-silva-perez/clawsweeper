## Summary

Keep open. Current main still has the old opt-in behavior for successful empty-output background exec completion events, so the PR is not obsolete or implemented. ## What This Changes

The PR flips background exec empty-success completion notifications from opt-in to default-on, aligns the runtime default, updates tests/config help, and adds a 50 ms exec-exit heartbeat coalescing option.

## Best Possible Solution

If maintainers accept the default behavior change, rebase this work onto current main, preserve `reason: "exec-event"` with `coalesceMs: 0`, keep explicit suppression paths such as gateway/cron false, update generated and public config docs, and then follow with a narrow coding-agent docs update that treats parent-owned completion inspection as the reliable workflow.
