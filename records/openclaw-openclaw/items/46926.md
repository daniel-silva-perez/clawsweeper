## Summary

Keep open. Current main still defaults empty-success background exec completion notifications to opt-in, so the PR addresses a real workflow gap, but the proposed patch is not merge-ready because it is stale against current exec wake semantics and leaves generated/public config docs inconsistent.

## What This Changes

The PR changes exec empty-success completion notifications from opt-in to default-on, updates tests and config help, aligns the lower-level runtime default, and changes exec-exit heartbeat coalescing to 50 ms.

## Best Possible Solution

If maintainers accept the default behavior change, rebase the PR onto current main, keep the current `exec-event` wake reason with `coalesceMs: 0`, update generated config schema/docs and public exec/background-process docs, then follow with a narrow coding-agent skill/docs update that makes parent-owned completion inspection the reliable workflow while treating worker self-send as optional.
