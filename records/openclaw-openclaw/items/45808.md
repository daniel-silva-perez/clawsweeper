## Summary

Keep this PR open. Current main still does not implement Discord account-level `agentId` routing: Discord account config/schema do not accept `agentId`, routing still uses only explicit configured `bindings`, no Discord binding synthesizer exists, and the current docs still direct users to account-scoped routing bindings. The PR remains a plausible implementation candidate, but it needs maintainer review around routing security semantics, tokenless/disabled account behavior, and status/API propagation before merge.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is to rebase it onto current main, add Discord account-level `agentId` to the accepted config contract and generated metadata, synthesize effective route bindings only inside routing/runtime code without writing them back to config, preserve explicit binding precedence, define disabled/tokenless account behavior, expose effective account ownership in the real status/API surfaces, and update Discord/multi-agent docs plus focused routing/status tests.
