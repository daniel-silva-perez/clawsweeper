## Summary

Keep this PR open. Current main still does not implement Discord account-level `agentId` routing: routing only returns configured bindings, Discord account config/schema do not accept `agentId`, status snapshots do not expose an effective account binding, and docs still show manual Discord `bindings` entries for multi-agent bots. The PR remains a plausible implementation candidate, but it needs rebase and maintainer review for the remaining routing/status/security-sensitive edge cases.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is to rebase it onto current main, add Discord account-level `agentId` to the accepted config contract, synthesize effective route bindings in the routing layer without writing them back to config, preserve explicit binding precedence, define or filter disabled/tokenless account behavior, propagate effective binding visibility through the real status surfaces, and update Discord/multi-agent docs plus focused routing/status tests.
