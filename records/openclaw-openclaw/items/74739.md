## Summary

Keep this PR open for maintainer review. Current main still verifies package-update restart health from the already-loaded parent update process, while this PR moves that verification into the freshly installed CLI and appears to address the prior timeout and auth-env review feedback without introducing a blocking correctness or security finding.

## What This Changes

The PR adds a hidden restart-verification mode and tests so package-update restart health/version checks run from the updated OpenClaw CLI, with stale-PID retry and caller auth-env preservation while keeping current-process fallback for non-package updates.

## Best Possible Solution

Land this implementation or an equivalent maintainer-approved variant after final review and gate validation so package-update restart verification consistently runs from the updated install while preserving stale-PID recovery, expected-version checks, and auth-enabled gateway probes.

## Reproduction Assessment

Yes. Current main has a focused test path showing package restart verification still probes from the parent process, and the PR adds targeted tests that assert the updated entrypoint performs hidden verification instead. A live package self-update smoke would still be useful before merge.

## Solution Assessment

Yes. Delegating only restart health/version verification to the updated install is a narrow maintainable fix that matches the existing fresh-process update pattern while preserving the current-process fallback for non-package updates.
