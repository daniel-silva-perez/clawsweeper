## Summary

Keep this PR open for normal maintainer review. Current main still performs package-update restart verification in the already-loaded parent process, while this PR moves that verification into the freshly installed CLI and addresses the earlier timeout and auth-env review comments; I did not find a discrete blocking correctness issue in the latest diff.

## What This Changes

The PR changes the CLI package-update restart path and update CLI tests so package-mode restart verification is delegated to the freshly installed OpenClaw entrypoint through a hidden verification mode with stale-PID retry, timeout budgeting, and gateway auth-env preservation.

## Best Possible Solution

Land this implementation or an equivalent maintainer-approved variant after normal review and gate validation so package-update restart verification consistently runs from the updated install while preserving stale-PID recovery, expected-version checks, and auth-enabled gateway probes.
