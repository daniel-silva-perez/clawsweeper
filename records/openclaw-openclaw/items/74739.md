## Summary

Keep open for normal maintainer review. Current main still performs package-update restart health/version verification in the parent update process after invoking the updated install for the restart, while this PR moves that verification into a hidden updated-install `openclaw update` mode and covers the previously reviewed stale-PID timeout and auth-env cases. I did not find a concrete blocking correctness or security issue in the latest diff.

## What This Changes

The PR updates the CLI package-update restart path and tests so restart verification can re-enter the freshly installed OpenClaw CLI through a hidden verification mode, with stale-PID retry, timeout budgeting, and gateway auth-env preservation.

## Best Possible Solution

Land this implementation or an equivalent maintainer-approved variant after normal review and gate validation so package-update restart verification consistently runs from the updated install while preserving stale-PID recovery, expected-version checks, and auth-enabled gateway probes.
