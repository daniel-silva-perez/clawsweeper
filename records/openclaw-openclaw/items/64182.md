## Summary

Keep this PR open. Current main still suppresses tool summaries, tool-start statuses, and plan updates for ordinary non-forum group sessions through the shared verbose-progress gate, with no `CommandAuthorized === true` plus real control-command exception. The PR remains a focused implementation candidate, and the prior bot review/discussion correctly points toward a provider-agnostic fix rather than hardcoded Feishu/Lark handling.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate. Rebase it onto current main, implement a provider-agnostic authorized group command predicate at the current verbose-progress gate, apply it consistently to tool summaries and tool-start statuses, preserve Slack non-direct and normal group-chatter suppression, and keep regression coverage for authorized, unauthorized, mention-prefixed, forum-topic, and path-like slash inputs.
