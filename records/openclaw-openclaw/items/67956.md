## Summary

Keep open. Current main has partial CLI JSONL delta plumbing into the global agent event bus, but the auto-reply CLI path still does not forward those deltas into the channel preview hook, and the requested opt-in config/capability/diagnostic contract is absent.

## What This Changes

The issue requests an opt-in CLI backend incremental streaming mode so channels can show progressive preview updates instead of waiting for final buffered CLI output.

## Best Possible Solution

Implement a capability-gated CLI incremental preview contract: keep buffered output as the default, let eligible CLI JSONL/stdout deltas feed the existing channel `onPartialReply` preview hooks with coalescing and final-message dedupe, add diagnostics for produced-vs-dropped incremental output, and document the new backend config surface.

## Reproduction Assessment

Yes. A code-level reproduction is clear: a JSONL CLI backend can emit assistant deltas into agent events, but the auto-reply CLI branch never passes those deltas to `onPartialReply`, so Telegram/Feishu preview hooks stay final-only for CLI runs.

## Solution Assessment

Unclear. The requested opt-in direction is reasonable, but the exact public config/capability names and dedupe semantics should be settled before implementation rather than blindly forwarding every global CLI assistant event to channels.
