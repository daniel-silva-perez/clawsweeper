## Summary

Keep open. Current main still has CLI JSONL delta plumbing into the global agent event bus, but the auto-reply CLI branch does not route those deltas into channel preview hooks, and the requested opt-in backend config/capability/diagnostic contract is still absent.

## What This Changes

The issue requests an opt-in CLI backend incremental streaming mode so channels can show progressive preview updates during long-running CLI runs instead of waiting for final buffered output.

## Best Possible Solution

Implement a capability-gated CLI incremental preview contract: keep buffered output as the default, let eligible CLI JSONL/stdout deltas feed existing channel `onPartialReply` preview hooks with coalescing and final-message dedupe, add produced-vs-dropped diagnostics, and document the new backend config surface.

## Reproduction Assessment

Yes. A code-level reproduction is clear: a Claude stream-json CLI backend can emit assistant deltas into agent events, but the auto-reply CLI branch has no `onPartialReply` forwarding path, so Telegram and Feishu preview hooks remain final-only for CLI runs.

## Solution Assessment

Unclear. The requested opt-in direction is reasonable, but the exact public config/capability names and channel dedupe semantics should be chosen by maintainers before implementation rather than forwarding every global CLI assistant event directly to previews.
