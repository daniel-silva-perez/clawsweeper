## Summary

Keep open. Current main has partial CLI JSONL delta plumbing into the global agent event bus, but the CLI auto-reply branch still does not forward those deltas into channel `onPartialReply` preview hooks. Telegram and Feishu previews remain driven by `replyOptions.onPartialReply`, and there is no `streamingMode` or CLI incremental capability/diagnostic config surface in current source or docs.

## Best Possible Solution

Keep this open and implement a capability-gated CLI backend preview streaming contract: preserve buffered behavior by default, add an explicit backend config/capability for incremental preview, route eligible CLI deltas through the same `onPartialReply` path used by Telegram/Feishu/other channels with debounce/coalescing, add diagnostics for backend-no-delta vs dropped-before-dispatch, and cover the CLI branch plus at least Telegram/Feishu preview behavior in tests.
