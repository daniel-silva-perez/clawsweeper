## Summary

Keep open. Current main has partial CLI JSONL delta plumbing for Claude-style stream-json into the global agent event bus, but the channel preview path still depends on `replyOptions.onPartialReply`, and the CLI backend branch does not forward incremental CLI output into that hook. The requested configurable CLI-backend incremental channel preview mode and capability/diagnostic contract are still missing.

## Best Possible Solution

Keep this issue open and implement a capability-gated CLI backend preview streaming contract: keep buffered output as the default, let eligible CLI JSONL/stdout deltas feed the existing channel `onPartialReply` draft-preview hooks with debounce/coalescing, add diagnostics for “backend produced no incremental output” vs “output was dropped before channel dispatch,” and cover Telegram/Feishu plus a generic CLI backend test.
