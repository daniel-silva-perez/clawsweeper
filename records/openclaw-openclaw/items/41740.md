## Summary

Keep open. Current main still has the reported local self-signed TLS gap in the shared operator approvals GatewayClient path used by Discord approval buttons and native approval runtimes. The related fix PRs were closed unmerged, so they are useful prior art but not implementation evidence.

## Best Possible Solution

Keep this issue open. The best fix belongs in the shared operator approvals Gateway bootstrap/client path: resolve the same local TLS runtime fingerprint used by callGateway, include it in the bootstrap/client options, and add focused regression coverage for createOperatorApprovalsGatewayClient or resolveGatewayClientBootstrap plus the Discord/native approval paths. The closed #41741 and #46232 remain useful prior art, but they should not close this issue unless an equivalent tested fix lands on main.
