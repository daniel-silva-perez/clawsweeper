## Summary

Keep open. Current main still has the reported local self-signed TLS gap in the operator approval GatewayClient path used by Discord exec approval buttons. The code path resolves URL/auth bootstrap data but never resolves or passes the local gateway TLS fingerprint, while the regular callGateway path already does that for local wss:// gateways. The two related fix PRs, #41741 and #46232, are useful prior art but were closed unmerged, so they do not make this issue obsolete.

## Best Possible Solution

Keep this issue open. The best fix is to move the local TLS fingerprint resolution into the shared operator approvals Gateway bootstrap/client path, pass it to GatewayClient, and add focused regression coverage for resolveGatewayClientBootstrap or createOperatorApprovalsGatewayClient plus the Discord/native approval flows. The closed #41741 and #46232 remain useful prior art but should not close the issue unless an equivalent tested fix lands on main.
