## Summary

Keep open. Current main still has Azure Speech as a TTS-only bundled plugin, while Voice Call already has a generic realtime transcription provider registry. ## Best Possible Solution

Keep the PR open for maintainer review. The best path is to fix the lazy-connect first-frame delivery bug, add a regression assertion for first-frame write/no unexpected onError, review the Microsoft Speech SDK dependency, then run the targeted Azure Speech, Voice Call, config/docs, lint, and extension typecheck gates before merge consideration.
