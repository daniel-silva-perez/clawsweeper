## Summary

Keep this PR open. Current main still has Azure Speech as a TTS-only bundled plugin, while Voice Call already has a realtime transcription provider registry and uses it for streaming. The PR is not obsolete or implemented on main; it is a concrete implementation candidate that still needs maintainer review for the lazy-connect data-loss finding and the new Microsoft Speech SDK dependency.

## Best Possible Solution

Keep the PR open for normal maintainer review. The best path is to fix the lazy-connect audio delivery bug, verify the Azure Speech SDK dependency is acceptable and lazy-loaded for TTS-only users, review credential/endpoint/error logging behavior, then run targeted Azure Speech, Voice Call, config/docs, lint, and extension typecheck gates before considering merge.
