## Summary

Keep open. Current main still does not provide execute-time plugin tool identity context, so the PR addresses a real remaining gap, but the branch should not merge as-is because its adjacent compaction auth hunk drops header-based credentials and the branch is stale against current main paths. Related PR #56036 is adjacent hook-context work, not a replacement for this execute-time contract.

## What This Changes

The PR wraps resolved plugin tools so `execute` receives a frozen identity/session context as a fifth argument, adds an execute-context SDK type and tests, and updates an adjacent compaction auth call and mocks.

## Best Possible Solution

Ship a rebased, documented plugin-tool execute-context contract that appends a frozen identity-only context after the existing dispatcher arguments, with focused tests for context injection, signal/onUpdate preservation, own-property preservation, and config non-leakage; keep compaction auth header behavior intact or handle that migration separately.

## Reproduction Assessment

Yes. Static inspection of current main is enough: plugin factories receive `OpenClawPluginToolContext`, but resolved tools are returned unwrapped and the dispatcher calls `execute` with only `toolCallId`, params, `signal`, and `onUpdate`, so a plugin expecting execute-time identity context will not receive it.

## Solution Assessment

No, not as currently proposed. The execute-wrapper direction is narrow and plausible, but this branch also carries a stale compaction auth hunk that regresses header credentials; the safer solution is a current-main rebase that preserves existing compaction auth behavior or removes that unrelated change.
