## Summary

Keep open. Current main still does not append an execution-time identity context to plugin tool `execute` calls, so the core feature in this PR remains relevant; however, the PR is not merge-ready because its adjacent compaction auth change drops resolved header credentials and the branch is stale against the current compaction path.

## What This Changes

The PR wraps resolved plugin tools so `execute` receives a narrow identity/session context as a fifth argument, adds the corresponding plugin SDK type and tests, and updates an adjacent compaction-safeguard auth call and mocks.

## Best Possible Solution

Ship a rebased, documented plugin-tool execute-context contract that appends a frozen identity-only context after the existing dispatcher arguments, with focused tests for context injection, signal/onUpdate preservation, own-property preservation, and config non-leakage; keep the current header-aware compaction auth behavior intact or handle that migration in a separate PR.
