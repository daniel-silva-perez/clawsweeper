## Summary

Keep this PR open. The warning-only direction is a useful narrow fix for the open bootstrap `agentDir` issue, but the current patch warns even when the same bootstrap file exists in the workspace and is actually loaded, which contradicts its own test and would create false warnings.

## What This Changes

The PR adds `agentDir` bootstrap-file warning plumbing to the bootstrap resolver, passes `agentDir` from the embedded attempt runner, and adds tests for the warning behavior.

## Best Possible Solution

Repair the PR so the diagnostic only warns for recognized bootstrap filenames that exist in `agentDir` and do not also exist in the effective workspace, keep the current workspace-only loading contract unless maintainers choose a separate versioned merge/override design, and add the required changelog entry for the user-visible fix.

## Reproduction Assessment

Yes. The PR's own both-directories test is a high-confidence reproduction: create `SOUL.md` in both `workspaceDir` and `agentDir`; the current helper checks only `agentDir` and still emits the warning.

## Solution Assessment

No, not as currently implemented. A warning-only diagnostic is the narrowest backward-compatible fix for the silent-ignore behavior, but it must suppress false warnings when the workspace copy exists and is the file being loaded.
