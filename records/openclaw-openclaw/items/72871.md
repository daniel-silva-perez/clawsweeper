## Summary

Keep open. Current main still does not warn for bootstrap-named files under agentDir, and this PR is a useful Option B implementation for the open related bug, but the branch currently emits a false warning when the same bootstrap file also exists in the workspace and is actually loaded.

## What This Changes

The PR adds an optional agentDir parameter to bootstrap resolution, emits warning diagnostics for standard bootstrap filenames found under agentDir, passes agentDir from embedded attempts, and adds warning-focused tests.

## Best Possible Solution

Keep the PR open and repair the diagnostic so it warns only for recognized bootstrap filenames that exist in agentDir and do not also exist in the effective workspace. Preserve the existing workspace-only loading contract unless maintainers separately approve a versioned per-agent merge or override design.

## Reproduction Assessment

Yes. The PR's own both-directories test is a high-confidence reproduction: create SOUL.md in both agentDir and workspaceDir, call resolveBootstrapFilesForRun with both paths, and the current helper still warns after checking only agentDir.

## Solution Assessment

No, not as currently implemented. A warning-only diagnostic is a narrow backward-compatible solution for the silent-ignore behavior, but it must suppress warnings when the workspace copy exists and is loaded; changing agentDir loading semantics should remain a separate product/API decision.
