## Summary

Keep open. The PR targets a valid current-main gap: bootstrap context is still resolved from the workspace-only loader, while `agentDir` is documented as state/auth/session storage. The patch is not mergeable as-is because its new warning helper emits a warning for any matching file in `agentDir` without checking whether the same file exists in `workspaceDir`, contradicting the PR's own both-directories test and producing false warnings for correctly loaded workspace files.

## What This Changes

The PR adds an `agentDir` parameter to bootstrap resolution, warns when standard bootstrap filenames are found under `agentDir`, passes `agentDir` from the embedded attempt runner, and adds tests for the new warning behavior.

## Best Possible Solution

Keep the PR open and repair the diagnostic so it only warns for recognized bootstrap filenames that exist in `agentDir` and do not also exist in the effective workspace. Preserve current workspace-only loading semantics unless maintainers separately choose a versioned per-agent bootstrap merge/override design.

## Reproduction Assessment

Yes. The related issue has concrete setup steps, and the PR bug has a high-confidence reproduction path through the added both-directories test: create `SOUL.md` in both paths and the current PR helper still warns after checking only `agentDir`.

## Solution Assessment

No, not as currently implemented. A warning-only diagnostic is a narrow backward-compatible way to address the silent-ignore behavior, but it needs to suppress warnings when the workspace copy is present and loaded; broader per-agent bootstrap loading should remain a separate product/API decision.
