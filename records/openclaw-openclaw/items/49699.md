## Summary

Keep this PR open. Current main still uses the existing `DeviceAuthToken | null` and `{ ok, reason }` token contracts, so the structured error-object change proposed by this PR has not landed. The PR is still potentially useful, but it needs focused rework before merge because the provided diff/review context shows parse-breaking edits and a possible public resource-enumeration side channel in revoke errors.

## Best Possible Solution

Keep the PR open and ask for a focused rework: preserve or intentionally version the existing token result discriminators, add structured details only where callers can safely consume them, keep public rotate/revoke denial messages generic while logging detailed server-side context, fix the syntax errors, and update focused infra/gateway tests for scope escalation, missing device/role, and revoke-denial cases.
