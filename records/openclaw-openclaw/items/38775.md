## Summary

Current main still does not implement usage-aware multi-account auth-profile load balancing. Anthropic usage snapshots exist and are exposed through reporting/status paths, but request-time Gateway auth selection remains profile order, credential type, lastUsed, and reactive cooldown/failover based. The related open #41135 tracks broader provider-profile pool routing, but this issue still adds useful Anthropic usage-window details and should stay open for maintainer design/consolidation rather than be auto-closed.

## Best Possible Solution

Keep this open and consolidate design with #41135 rather than auto-closing. The best path is an additive, generic auth-profile routing contract that can use provider-owned usage snapshots as scoring inputs, preserve current defaults, support proactive utilization thresholds and fallback for profiles without usage data, expose observable route decisions, and document/configure the behavior before enabling it for Anthropic OAuth accounts.
