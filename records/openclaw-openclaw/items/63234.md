## Summary

Keep open. Current main still has the same generic plugin-config validation/log path and does not contain UnraidClaw or a regression proving that a valid required plugin config is preserved through startup plus post-ready tool catalog resolution. The adjacent invalid-plugin degradation work is useful but does not close this valid-config startup-spam report.

## What This Changes

The issue asks OpenClaw to stop repeatedly logging invalid-config errors during gateway startup when a plugin has valid required config under `plugins.entries.<id>.config` and later registers its tools successfully.

## Best Possible Solution

Land a narrow generic fix that preserves resolved `plugins.entries.<id>.config` for required native tool plugins across gateway startup, active registry reuse, `tools.catalog`, and post-ready plugin tool resolution, with regression coverage that fails if valid config is later seen as `{}` or logs repeated invalid-config errors.
