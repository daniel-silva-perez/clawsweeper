## Summary

Keep open. Current main still implements both reported surfaces: default AGENTS.md heading extraction is injected after compaction, and the safeguard appends extracted AGENTS.md sections to the compaction summary while normal Project Context remains in the system prompt. The current config/docs/tests provide an opt-out for one refresh path, but the default bloat and magic-heading behavior remains a live product decision rather than cleanup closure material.

## What This Changes

The issue requests removing or explicitly opting into the post-compaction AGENTS.md section parsing and reinjection introduced by #18049 because it duplicates retained prompt context and makes behavior depend on magic headings in a user-editable file.

## Best Possible Solution

Set an explicit maintainer-approved product boundary for post-compaction recovery. A conservative end state would make AGENTS.md section reinjection opt-in, or skip both the summary suffix and refresh prompt when normal Project Context is retained, then update docs and targeted tests to match the chosen default.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes. With an AGENTS.md containing Session Startup or Red Lines and no postCompactionSections override, readPostCompactionContext returns a refresh prompt and preflight compaction appends it; the compaction safeguard independently appends the same extracted sections to the compacted summary.

## Solution Assessment

Is this the best way to solve the issue? Unclear. The requested removal is directionally valid for bloat and magic-heading risk, but because #18049 intentionally added recovery enforcement, the safest fix needs maintainer agreement on whether the default should become opt-in, conditional, or removed from both injection paths.
