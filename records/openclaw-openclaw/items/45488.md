## Summary

Keep open. Current main still implements the reported behavior: after compaction it parses user-editable AGENTS.md section headings, injects extracted sections as a post-compaction system event, and also appends extracted AGENTS.md rules to the compaction summary. A config opt-out exists, but the default remains documented and tested, so the central session-bloat and magic-heading concern is still unresolved.

## Best Possible Solution

Keep this issue open for maintainer product/behavior judgment. The clean fix is to decide whether AGENTS.md section reinjection should be opt-in by default, skipped when normal Project Context is retained, or retained only through an explicit config/template path. Any change should update the parser, summary-suffix behavior, config docs, and tests together.
