## Summary

Keep this issue open. Current main still performs the behavior reported here: default post-compaction AGENTS.md section parsing is active, post-compaction text is injected after compaction, and the compaction safeguard still appends extracted AGENTS.md sections into the persisted summary. The discussion’s proposed direction, explicit opt-in/configured template behavior instead of default magic section parsing, is not implemented on main.

## Best Possible Solution

Keep this issue open and fix it in the compaction owner paths: stop default AGENTS.md section reinjection when the system prompt/bootstrap context is already retained, remove or gate the `<workspace-critical-rules>` summary append behind an explicit opt-in, and update tests/docs so the default behavior is no post-compaction magic-heading parsing. If maintainers still want this capability, make it an explicit config/template-file feature rather than implicit parsing of user-editable AGENTS.md.
