## Summary

Keep open. Current main still has the reported default behavior: AGENTS.md sections are parsed by heading, injected after compaction, and also appended to compaction summaries while normal Project Context still includes workspace bootstrap files. The issue is valid, but changing the default would partially reverse the intentional recovery behavior from #18049, so this needs maintainer product judgment rather than cleanup closure.

## What This Changes

The issue requests removing or replacing post-compaction AGENTS.md magic-heading parsing because it duplicates retained system-prompt content and makes session behavior depend on user-editable headings.

## Best Possible Solution

Set a maintainer-approved policy for post-compaction recovery, then update both injection paths together. A conservative end state would make AGENTS.md section reinjection opt-in, or skip both the refresh prompt and summary suffix when normal Project Context is retained, with docs and targeted tests updated to match.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: with default config and an AGENTS.md containing Session Startup or Red Lines, readPostCompactionContext returns a refresh payload, follow-up compaction appends it, and the safeguard independently appends workspace-critical-rules while Project Context remains injected.

## Solution Assessment

Unclear. Removing the magic parsing is directionally aligned with the bloat report, but because the current behavior was an intentional recovery feature, the safest solution needs maintainer agreement on whether the default becomes opt-in, conditional, or removed from both paths.
