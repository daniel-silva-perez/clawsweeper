## Summary

Keep open. Current main still has the reported default behavior: AGENTS.md sections are parsed and re-injected after compaction while normal Project Context continues to include AGENTS.md, so this is not implemented, obsolete, or stale cleanup. Because removing it would change intentional recovery behavior from the merged post-compaction enforcement work, the next step needs maintainer product judgment rather than an automatic close or narrow bot fix.

## What This Changes

The issue requests removing or changing post-compaction AGENTS.md section reinjection because it duplicates prompt content and makes behavior depend on magic section headings in user-editable AGENTS.md.

## Best Possible Solution

Set a maintainer-approved policy for post-compaction recovery, then update the refresh prompt, trusted system-event path, summary suffix, config docs, and targeted tests together. A conservative end state would make AGENTS.md section reinjection opt-in or skip it when normal Project Context is retained.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: with default config and an AGENTS.md containing Session Startup or Red Lines, readPostCompactionContext returns a refresh payload, follow-up or auto-compaction injects it, and the summary safeguard can append the same workspace-critical-rules while AGENTS.md is still present in Project Context.

## Solution Assessment

Unclear. Removing magic parsing is directionally aligned with the report, but the maintainable fix needs a maintainer decision on the intended default and must update both post-compaction injection paths rather than only one helper.
