## Summary

Keep open. Current main still has the reported default behavior, and the issue is valid; changing it would partially reverse the intentional post-compaction recovery behavior added by the merged post-compaction enforcement work, so this needs maintainer product judgment rather than cleanup closure.

## What This Changes

Request to remove or replace AGENTS.md heading-based post-compaction reinjection because it duplicates retained prompt context and makes behavior depend on user-edited headings.

## Best Possible Solution

Set a maintainer-approved policy for post-compaction recovery, then update both injection paths together. A conservative end state would make AGENTS.md section reinjection opt-in, or skip both the refresh prompt and summary suffix when normal Project Context is retained, with docs and targeted tests updated to match.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: with default config and an AGENTS.md containing Session Startup or Red Lines, readPostCompactionContext returns a refresh payload, follow-up or auto-compaction injects it, and the safeguard also appends workspace-critical-rules while Project Context still includes AGENTS.md.

## Solution Assessment

Unclear. Removing the magic parsing is directionally aligned with the bloat report, but the safest maintainable fix needs maintainer agreement on whether the default becomes opt-in, conditional, or removed from both paths.
