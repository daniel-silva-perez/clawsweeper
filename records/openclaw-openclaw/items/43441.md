## Summary

Keep open. Current main still has the bundled-layout template-resolution gap, and this PR is pointed at the right narrow area, but the branch is not mergeable as-is because the resolver/test seam changes existing fallback behavior and the new bundled-layout test does not reliably force package-root discovery to null.

## What This Changes

The PR changes workspace template directory resolution to add source and bundled module-relative fallback candidates before cwd-based lookup, and adds a bundled-layout regression test.

## Best Possible Solution

Repair this PR before merge: keep package-root resolution first, add reliable module-relative fallbacks for bundled and source layouts ahead of cwd, preserve the real-module fallback contract, and make the bundled-layout regression test deterministically bypass package-root discovery.

## Reproduction Assessment

Yes. The original bug is reproducible by simulating bundled dist execution with package-root and cwd template lookups missing; the PR-branch defects are reproducible by walking resolver candidate order through the existing missing-template test and the new test's default argv1 path.

## Solution Assessment

No. The production direction is narrow and appropriate, but the current patch is not the best form because the injected moduleUrl now changes all fallback directories and the new test does not isolate package-root discovery; a narrower module-dir/root-resolver test seam is safer.
