## Summary

Keep this PR open. The underlying bundled-layout problem is still present on current main, but the branch should not merge as-is because its testability refactor appears to break existing resolver coverage and the new regression test can be bypassed by the default argv1 package-root lookup.

## What This Changes

The PR changes workspace template directory resolution to try package-root docs, module-relative source and bundled `dist` docs paths, then cwd, and adds a bundled-layout regression test.

## Best Possible Solution

Keep the PR open and repair the resolver tests before merge: cover the bundled `dist` fallback with `packageRoot` genuinely null, preserve or intentionally update the existing missing-template fallback contract, and prefer the packaged `../docs/reference/templates` candidate before the source-layout overshoot when running from root `dist` artifacts.

## Reproduction Assessment

Yes. The original issue can be reproduced with a bundled-style module URL where package-root and cwd template lookups miss; the PR branch's CI failure can be reproduced by following the existing temp package-root test and the new test's default `argv1` path through `resolveOpenClawPackageRoot`.

## Solution Assessment

No. The production direction is narrow and appropriate, but the current patch couples the injectable module URL to production fallbacks and does not isolate `argv1`, so the automated coverage is not reliable enough to merge.
