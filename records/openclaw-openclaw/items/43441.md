## Summary

Keep open. Current main still has the single `../../docs/reference/templates` fallback after the cwd candidate, so the bundled `dist/` plus package-root-null case described by the PR is not implemented on main. The PR directly addresses that gap, and its second commit resolves the earlier Greptile testability objection by computing fallback candidates from the injected `moduleUrl`. I did not find a blocking functional or security regression in the patch, but it still needs normal maintainer validation because no CI statuses are recorded for the head SHA and this read-only review did not run tests.

## What This Changes

The PR changes workspace template resolution to try package-root templates, module-relative source and bundled `dist/` fallback paths, then cwd, and adds a Vitest case for the bundled-layout fallback when package-root lookup is null.

## Best Possible Solution

Land this PR or an equivalent narrow patch after maintainer validation so workspace template resolution does not depend on cwd when package-root discovery fails in bundled runtime layouts. The final behavior should reliably find root-packaged `docs/reference/templates` from both source and `dist/` modules, with regression coverage for the injected bundled module URL path.
