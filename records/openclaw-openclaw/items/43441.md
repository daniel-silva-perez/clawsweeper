## Summary

Keep this PR open. Current main still has the bundled `dist/` template-resolution gap, and the PR targets the right production seam, but the branch is not mergeable because its resolver seam changes existing fallback behavior and its new regression test can discover the real checkout through `argv1` before exercising the bundled fallback.

## What This Changes

The PR changes workspace template directory resolution to compute source-layout and bundled-`dist` module-relative fallback directories before cwd, and adds a bundled-layout resolver regression test.

## Best Possible Solution

Repair this PR before merge: keep package-root resolution first, add reliable module-relative fallbacks for both bundled and source layouts ahead of cwd, preserve the existing real-module fallback contract, and test the bundled fallback with package-root discovery forced to null.

## Reproduction Assessment

Yes. The original bug can be reproduced by simulating a bundled module URL with package-root and cwd template lookups missing; the PR-branch defects are reproducible by walking candidate order through the existing missing-template test and the new test’s default `process.argv[1]` fallback.

## Solution Assessment

No. The production direction is narrow and appropriate, but the current patch is not the best form because the injected `moduleUrl` changes existing fallback semantics and the new regression test does not isolate package-root discovery; a small resolver/test-seam adjustment is safer.
