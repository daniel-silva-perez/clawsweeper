## Summary

Keep open. Current main still treats a primary transcript as orphaned solely because it is absent from the active session-key map, so a valid historical chat transcript can still be offered for `.deleted.<timestamp>` archival; the interactive-only confirmation and archived-transcript indexing paths are mitigations, not the classifier fix.

## What This Changes

The issue asks OpenClaw doctor to stop archiving historical primary session transcripts as orphans, improve the prompt/recovery behavior, surface corrupted session-store reads, and harden doctor recommendation coverage.

## Best Possible Solution

Change the doctor state-integrity classifier so valid historical primary transcripts are preserved even when absent from the active session-key map. Add a regression fixture with an older real transcript, keep the sidecar/archive exclusions and interactive mitigation, then split restore tooling, malformed-store warnings, and doctor recommendation CI hardening into explicit follow-ups if they exceed the first repair boundary.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to create one active `sessions.json` entry plus an older valid `<id>.jsonl` in the same sessions directory; current main will include the older file in `orphanTranscriptPaths` because it is primary JSONL and not in `referencedTranscriptPaths`.

## Solution Assessment

No. Current main's interactive-only archival and QMD-specific suppression/indexing reduce the worst data-loss path, but the narrow maintainable fix is to stop using absence from the active session map as the definition of an orphan primary transcript.
