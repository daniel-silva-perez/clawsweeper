## Summary

Keep open. Current main still treats valid older primary transcript files as orphan candidates when they are absent from the active `sessions.json` map; the interactive confirmation and archived-discovery mitigations reduce impact but do not fix the classifier described in the issue.

## What This Changes

The issue asks OpenClaw to stop `doctor --fix` from archiving valid historical primary session transcripts as orphans and to harden the broader doctor recommendation surface.

## Best Possible Solution

Define a real orphan transcript in the doctor state-integrity path instead of using absence from the active session-key map. A focused first change should preserve valid historical primary transcripts and add regression coverage, while prompt wording, restore support, corrupt-store warnings, and doctor recommendation CI hardening should be split into scoped follow-ups.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is clear: create a sessions directory with one active `sessions.json` entry and an older valid `<sessionId>.jsonl` absent from the store, then the current orphan scan will classify that older transcript as orphan because it is missing from `referencedTranscriptPaths`.

## Solution Assessment

No. The current interactive-confirmation mitigation is safer than silent repair, but it does not solve the definition bug; the narrow maintainable fix is to change the classifier and tests first, then handle UX and CI hardening separately.
