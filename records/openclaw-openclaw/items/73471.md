## Summary

Keep open. Current main has impact mitigations, but it still identifies orphan transcript candidates by comparing primary `.jsonl` files against only the active `sessions.json` entries, so valid older primary transcripts can still be treated as archive candidates when they are no longer the current session for a key.

## What This Changes

The issue asks OpenClaw to stop `doctor --fix` from classifying valid historical session transcripts as orphan files, improve the archive prompt and recovery path, surface corrupt session-store reads, and harden doctor recommendations in CI.

## Best Possible Solution

Define a real orphan transcript in the doctor state-integrity path instead of using absence from the active session-key map. A focused first change should preserve valid historical primary transcripts with regression coverage; prompt wording, restore support, corrupt-store warnings, and CI recommendation hardening should be split into scoped follow-ups.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is clear: create a sessions directory with one active `sessions.json` entry and an older valid `<sessionId>.jsonl` absent from the store, and current main will classify that older transcript as orphan because it is missing from `referencedTranscriptPaths`.

## Solution Assessment

No. The current interactive-confirmation and archived-discovery mitigations reduce damage, but they do not solve the definition bug; the narrow maintainable fix is to change the classifier and tests first, then handle UX and CI hardening separately.
