## Summary

Keep open. Current main still provides raw workspace Markdown editing and a narrow plugin Control UI descriptor projection, but it does not implement a pinned key-value facts panel, structured pinned-facts storage, mounted persistent plugin panels, or an every-turn inclusion guarantee. The related #71736 RFC is relevant, but it remains an open design item and does not fully cover the pinned facts editor/storage/injection behavior requested here.

## What This Changes

Add a Control UI pinned context panel for user-defined facts backed by USER.md or a dedicated structured store and included in agent context on every turn.

## Best Possible Solution

Keep the issue open until maintainers either split or approve a concrete design: a safe manifest-first Control UI panel/contribution contract, plus a pinned facts editor backed by a documented structured store or USER.md section, with tests proving how those facts enter normal agent context under the intended scope and budget rules.
