## Summary

Keep open. Current main has raw workspace Markdown editing, browser-local pinned chat-message summaries, and a runtime plugin UI descriptor projection, but it still does not implement a Control UI pinned key-value facts panel, structured persistent-facts storage, guaranteed every-turn injection semantics, or the manifest-first persistent panel API discussed in the comments. The #46630/#49828 discussion is related but not a canonical shipped replacement, and the linked external fork PR was closed outside this repository.

## Best Possible Solution

Keep this open unless maintainers split a narrower canonical design item. The best path is to design a manifest-first Control UI panel contract that the Gateway exposes and the SPA mounts, then add a pinned facts editor backed by a documented structured store or USER.md section with tests proving inclusion in normal agent context under the intended scope and budget rules.

## Risks / Open Questions

- Pinned facts need explicit storage and injection semantics across agents, sessions, contextInjection modes, truncation budgets, memory tools, and prompt-injection policy.
