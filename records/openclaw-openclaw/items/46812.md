## Summary

Keep open. Current main has raw workspace Markdown file editing, bootstrap-file injection, and a narrow plugin Control UI descriptor registry, but it still lacks the requested pinned key-value facts panel, structured persistent-facts storage, manifest-first persistent panel mounting, and guaranteed every-turn injection semantics. The newer #71736 RFC is relevant for the generic Control UI contribution primitive, but it does not fully cover the pinned facts editor/storage/injection behavior requested here.

## What This Changes

Add a Control UI panel or sidebar widget for users to add, edit, delete, and persist key facts that are injected into agent context on every turn.

## Best Possible Solution

Keep this open unless maintainers split a narrower canonical design. The best end state is a manifest-first Control UI contribution or panel contract that the SPA actually mounts, followed by a pinned facts editor backed by a documented structured store or USER.md section, with tests proving the facts are included in normal agent context under the intended scope and budget rules.
