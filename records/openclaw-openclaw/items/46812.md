## Summary

Keep open. Current main has raw workspace Markdown editing, bootstrap file injection, and a runtime Control UI descriptor projection, but it still does not provide the requested pinned key-value facts panel, manifest-first persistent Control UI panel mounting, structured persistent-facts storage, or guaranteed every-turn injection semantics. The discussion usefully reframes the shared primitive with #46630/#49828 as a persistent Control UI panel surface; the linked external fork PR was closed outside openclaw/openclaw and is not shipped here.

## Best Possible Solution

Keep this open unless maintainers split a narrower canonical design item. The best path is to design a manifest-first Control UI panel contract that the SPA actually fetches and mounts, then add a pinned facts editor backed by a documented structured store or USER.md section, with tests proving the facts are included in normal agent context under the intended scope, contextInjection mode, and truncation rules.

## Risks / Open Questions

- A persistent Control UI panel contract needs product and security design for rendering mode, refresh cadence, plugin trust boundaries, operator scopes, and CSP/origin behavior.
- Pinned facts need explicit storage and injection semantics across agents, sessions, contextInjection modes, truncation budgets, memory tools, and prompt-injection policy.
- The existing registerControlUiDescriptor seam may be a starting point, but it is runtime metadata today rather than the manifest-first panel source requested in the discussion.
