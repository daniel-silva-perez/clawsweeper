## Summary

Keep open. Current main still allows same-provider cross-context message sends by default, so the central hardening is not obsolete, but the PR is not merge-ready because the branch mixes unrelated changes and has concrete Telegram topic and generated-config issues.

## What This Changes

The PR changes outbound message policy, config help, tests, and changelog to make same-provider cross-context message sends opt-in, adds Telegram topic handling, and also modifies unrelated failover classification and UI scroll behavior.

## Best Possible Solution

Refresh and narrow the branch to only outbound message-policy hardening: default-deny same-provider forwarding, preserve the legacy explicit override, compare threaded/provider targets through the loaded plugin route parser, require Telegram topic identity when a session is topic-bound, regenerate config schema help, credit the changelog entry, and validate focused outbound plus Telegram coverage before the changed gate.
