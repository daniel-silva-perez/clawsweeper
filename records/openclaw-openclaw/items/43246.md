Keep open: current main still has the permissive same-provider default, so the intended hardening is not obsolete, but this PR is not merge-ready. ## What This Changes

This PR flips message-tool same-provider cross-context sends to opt-in, updates config help/tests/changelog, adds Telegram topic handling, and includes unrelated failover classification plus UI scroll fallback changes.

## Best Possible Solution

Refresh and narrow the PR to the outbound message-policy hardening: make same-provider forwarding opt-in by default, preserve the legacy explicit override, compare Telegram/topic targets through the generic loaded-channel route parser, require explicit topic identity for topic-bound contexts, regenerate config schema/help outputs, add the required changelog attribution, and then run focused outbound plus Telegram coverage before the changed gate.
