## Summary

Keep this PR open. Current main and the latest release still allow same-provider cross-context message sends by default, so the central security hardening has not shipped. The PR is useful but not merge-ready because it mixes unrelated failover/UI changes and its Telegram handling reaches into provider-owned parsing while still leaving a topic-bound escape.

## What This Changes

The PR flips message-tool same-provider cross-context sends to opt-in, updates config help/tests/changelog, adds Telegram topic equivalence, and also changes agent failover classification plus UI scroll fallback coverage.

## Best Possible Solution

Refresh and narrow the PR to the outbound message-policy hardening: make same-provider forwarding opt-in by default, preserve the legacy explicit override, express Telegram/topic equivalence through the loaded channel route/session grammar or a generic additive seam, update generated config help and the changelog with required attribution, then run focused outbound tests plus the changed gate.
