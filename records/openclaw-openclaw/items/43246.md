## Summary

Keep open. Current main still has the permissive same-provider messaging default, so the intended hardening is still needed; the proposed branch is not merge-ready because its Telegram topic handling and generated config output are inconsistent with current main.

## What This Changes

The PR flips same-provider message cross-context sends to opt-in, updates config help, tests, and changelog, adds Telegram topic handling, and also changes failover classification plus Control UI scroll behavior.

## Best Possible Solution

Refresh and narrow the branch to the outbound message-policy hardening only: default-deny same-provider forwarding, preserve the explicit legacy override, compare threaded provider targets through the loaded plugin seam, require Telegram topic identity when a session is topic-bound, regenerate config schema help, credit the changelog entry, and validate focused outbound plus Telegram coverage before the changed gate.
