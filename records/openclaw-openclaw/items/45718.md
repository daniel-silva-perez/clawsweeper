## Summary

Keep open. Current main still writes full session entries to `sessions.json`, so `skillsSnapshot` and `systemPromptReport` remain part of the persisted store when present. The report is partially narrowed because `systemPromptReport` is overwritten rather than appended and skill snapshots are version/filter gated, but the active serialization-bloat fix is still open in PR #48425 with an explicit closing reference to this issue, so policy says the issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep the issue open and review PR #48425 or an equivalent maintainer patch. The best fix is a session-store serialization change that omits truly transient `skillsSnapshot` data from `sessions.json`, keeps disk-budget measurement aligned with the saved representation, and makes an explicit product decision for `systemPromptReport` persistence with regression coverage.
