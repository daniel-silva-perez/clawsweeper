## Summary

Keep open. Current main still does not provide a session import command, Gateway import/register method, or Control UI path that turns loose `.jsonl` transcripts into `sessions.json` entries visible in the session menu. The existing transcript discovery path is usage-reporting only, while `sessions.list`, previews, chat history, CLI listing, docs, and the Control UI remain store-backed.

## Best Possible Solution

Keep this issue open and implement a first-class import path, likely `openclaw sessions import <path>` plus a Gateway helper that validates an OpenClaw/PI JSONL transcript, derives or accepts a session key and label, safely copies or adopts the transcript under the selected agent sessions directory, writes the matching `sessions.json` entry, and makes it visible through `sessions.list`, `sessions.preview`, `chat.history`, CLI listing, and the Control UI. Add tests for successful registration, malformed JSONL rejection, duplicate handling, path safety, and docs on https://docs.openclaw.ai/cli/sessions and https://docs.openclaw.ai/concepts/session.
