## Summary

Keep open. Current main still does not provide a session-import command, Gateway method, or Control UI path that registers loose `.jsonl` transcript files into the session menu. The menu is backed by `sessions.list`, which reads `sessions.json` stores and maps store entries only. There is transcript-file discovery in the usage surface, but it is not connected to session listing, previews, chat history navigation, or CLI session listing. The related reports in the provided context cover adjacent JSONL/session maintenance and usage topics, not this import/menu workflow.

## Best Possible Solution

Keep this issue open and implement a first-class import path, likely `openclaw sessions import <path>` plus a Gateway helper that validates an OpenClaw/PI JSONL transcript, derives or accepts a session key/label, copies or adopts the transcript under the target agent sessions directory, writes the matching `sessions.json` entry, and makes it visible through `sessions.list`, `sessions.preview`, chat history, and the Control UI. Add tests for imported transcript registration, malformed JSONL rejection, duplicate session handling, and docs in the sessions CLI/concepts pages.
