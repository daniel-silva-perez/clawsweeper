## What This Changes

The issue asks for clearer plugin allowlist diagnostics when users configure channel-style names such as `feishu` instead of discovered plugin IDs such as `openclaw-lark`.

## Best Possible Solution

Keep this issue open until #68389 or a maintainer-approved replacement lands a focused diagnostic that uses discovered manifest/channel ownership to tell users when an allowlist entry looks like a channel ID and which plugin ID should be allowed.
