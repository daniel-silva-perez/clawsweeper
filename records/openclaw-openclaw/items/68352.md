## Summary

Keep open. Current main still lacks the requested channel-id/plugin-id mismatch diagnostic, and #68389 is an open PR that explicitly closes this issue, so the right cleanup path is to leave this issue open while that PR is reviewed, landed, or closed.

## What This Changes

The issue asks OpenClaw to make plugin allowlist warnings clearer when configured entries are channel IDs or other names that do not match discovered plugin IDs, and to improve plugin ID discovery/auto-load guidance.

## Best Possible Solution

Review #68389 as the active implementation candidate for a diagnostic-only mismatch warning. Keep allow/deny semantics and auto-discovery policy unchanged in that PR; handle any broader auto-load policy change as a separate maintainer product/security decision.

## Reproduction Assessment

Yes. A source-level reproduction is clear: configure a non-empty `plugins.allow` with an entry that is not a discovered plugin id, and current `warnWhenAllowlistIsOpen` returns before comparing it to discovered plugin ids; validation then has no generic channel-id-to-plugin-id hint.

## Solution Assessment

Not applicable for this issue alone because there is no patch on the issue. The linked PR's diagnostic-only direction appears to be the narrowest maintainable path for the misleading warning, while changing extension auto-load behavior would need separate maintainer judgment.
