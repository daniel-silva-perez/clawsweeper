## Summary

Keep this PR open. Current main still has the Mattermost draft-stream overwrite path, but the PR is not ready to merge because the boundary flush is asynchronous in a fire-and-forget callback path and the tool-status text contract changed without updating existing Mattermost tests. Related issue #43020 remains open, while overlapping PR #71645 was closed unmerged.

## What This Changes

The PR changes Mattermost draft streaming to track streamed content, rotate or flush draft preview posts at assistant/reasoning/tool boundaries, and render richer tool-status text from item-event titles/icons.

## Best Possible Solution

Land one focused Mattermost fix on current main that preserves streamed assistant text before assistant/reasoning/tool boundaries, serializes boundary and partial updates enough to avoid dropping first chunks, and adds targeted Mattermost monitor or draft-stream tests. Keep tool-status text unchanged unless maintainers intentionally accept the richer title/icon UX with updated tests and privacy expectations.

## Reproduction Assessment

Yes. The current-main source path is enough to reproduce the reported behavior: partial replies edit one Mattermost draft post, assistant/reasoning boundaries only clear lastPartialText, and tool starts update the same draft post. The PR's test-contract regression is also deterministically reproducible from the unchanged draft-stream.test.ts expectations.

## Solution Assessment

No. The intended boundary rotation is the right direction, but this patch is not the best mergeable form because it awaits boundary flushes in fire-and-forget callback paths and changes the status-text contract without corresponding tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Serialize boundary rotation before accepting new partials:** `extensions/mattermost/src/mattermost/monitor.ts:1702-1706`
  - body: `onDraftBoundary` now awaits `draftStream.flush()`, but the embedded subscriber invokes assistant starts, reasoning ends, and partial replies with `void`, so the next partial can run while this boundary is mid-flush. When the await resumes, `forceNewMessage()` and the state resets can discard or misplace the first text of the new assistant block; serialize the boundary with partial updates or keep the rotation synchronous.
  - confidence: 0.86
- **[P2] Update the Mattermost status-text tests:** `extensions/mattermost/src/mattermost/draft-stream.ts:66-71`
  - body: This changes `buildMattermostToolStatusText` to return emoji/plain-text strings, but the PR head still has `draft-stream.test.ts` expecting `Running `read`…` and `Running `exec`…`. The Mattermost test lane will fail deterministically unless the tests/contract are updated or the old output shape is preserved.
  - confidence: 0.98
