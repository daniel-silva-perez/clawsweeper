## Summary

Keep this PR open. Current main still has the Mattermost draft-stream overwrite path described in the PR: assistant-message and reasoning boundaries only clear `lastPartialText`, tool start still writes status text into the active draft post, and there is no Mattermost-side `hasStreamedMessage`/boundary rotation. The PR is not implemented on main and is not cleanly superseded: #43020 remains the canonical open bug and #71645 is an overlapping open PR, but this PR has a broader implementation attempt that still may be useful after maintainer review and test coverage.

## Best Possible Solution

Keep this PR open for maintainer review, reconcile it with #43020 and #71645, and land one focused Mattermost fix that flushes/seals streamed text before assistant/reasoning/tool boundaries, rotates with the existing `MattermostDraftStream.forceNewMessage()` primitive, and adds targeted Mattermost monitor tests for boundary and item-event race ordering. Update Mattermost streaming docs if the accepted behavior intentionally changes from one preview post to multiple preserved posts.
