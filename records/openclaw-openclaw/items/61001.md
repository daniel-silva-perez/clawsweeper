## Summary

Keep this issue open. Current main has useful partial boot-context controls, but it does not implement the requested per-file conditional bootstrap loading by session type or file frontmatter. The config/schema/docs expose broad `contextInjection` and lightweight heartbeat/cron/subagent modes, while the bootstrap loader still uses fixed file lists and hard-coded filters rather than configurable `direct`/`group`/`heartbeat`/`subagent` selectors.

## Best Possible Solution

Keep this open and design a core bootstrap-selection contract. The likely path is an additive `agents.defaults.bootstrapFiles` or named bootstrap-profile schema, normalized run/session kinds (`direct`, `group`, `channel`, `heartbeat`, `cron`, `subagent`, possibly `acp`), evaluation in `src/agents/bootstrap-files.ts`/`src/agents/workspace.ts`, and docs/tests that preserve default blanket loading while proving the issue’s examples.

## Risks / Open Questions

- Current main already has partial mitigations (`contextInjection`, heartbeat/cron/subagent `lightContext`, and hard-coded subagent/cron filtering), so any implementation should be additive and preserve the existing blanket-load default.
