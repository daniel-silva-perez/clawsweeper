## Summary

Keep this issue open. Current main still has only coarse bootstrap-context controls and hard-coded filtering; it does not provide per-file conditional loading by normalized session type or bootstrap-file frontmatter. The open related PR #74735 is useful adjacent work for exact `sessionKey` extra files, but its own scope explicitly does not add session-type selectors or a new `agents.defaults` bootstrap policy.

## What This Changes

Request configurable per-bootstrap-file loading gates by session type, or file-level frontmatter, so direct, group, heartbeat, and subagent sessions receive only relevant boot context while preserving default blanket loading.

## Best Possible Solution

Design an additive core bootstrap-selection contract that preserves current blanket loading by default, normalizes session/run kinds, applies selection in the workspace/bootstrap resolution path before context files are built, and documents/tests the issue's direct, group, heartbeat, and subagent examples.

## Reproduction Assessment

Not applicable as a feature request. The source-level check is high-confidence: current main can be inspected to see the schema, loader, docs, and tests lack the requested per-file session-type selectors.

## Solution Assessment

No for current main. The narrowest maintainable path is a core config/API design for default bootstrap-file selection; PR #74735 may help exact-session extra files but is not the full solution for this issue.
