## Summary

Keep open: current main still has the reported fast-abort prefix-context gap, and open implementation PRs already target this issue, so it should not be stale-cleaned or closed before maintainers choose and land a canonical fix.

## What This Changes

The issue requests that early abort replies such as `stop` or `/stop` populate dynamic `responsePrefix` variables like `{model}` and `{thinkingLevel}` instead of sending literal placeholders.

## Best Possible Solution

Land or supersede the linked fix PRs with one narrow implementation that supplies safe model/provider/thinking prefix context to early abort and routed command fast paths, then add regression coverage for dynamic abort prefixes and normal `onModelSelected` override behavior.

## Reproduction Assessment

Yes. The source path is deterministic: configure a dynamic `responsePrefix`, trigger `/stop` while a run is active, and the fast-abort branch sends before model selection populates the prefix context while the resolver preserves missing variables literally.

## Solution Assessment

Unclear which linked PR is the best implementation without maintainer review. The safest direction is to derive prefix context from actual runtime/session metadata where available, while preserving normal model-selection override behavior.
