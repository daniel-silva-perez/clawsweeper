## Repair Work Prompt

Implement syntax highlighting for WebChat/Control UI code blocks for this issue. Current main renders code blocks in `ui/src/ui/markdown.ts` through custom `fence` and `code_block` renderers that escape text directly while adding language labels, copy buttons, and JSON collapse; keep the work in the Control UI renderer/helpers, not core/channel code. Add a small theme-aware highlighting helper or markdown renderer integration that supports fenced languages such as `ts`, `js`, `cpp`, and `json`, plus a safe unknown-language fallback. Preserve DOMPurify sanitization, raw HTML escaping inside code, exact raw code in `data-code` for the copy button, JSON collapse behavior, markdown caching, and oversized-message plain-text fallback. If adding a highlighter dependency, add it explicitly through the repo package pattern and update the lockfile deliberately; do not rely on the existing transitive highlight.js used by transcript export. Add focused tests in `ui/src/ui/markdown.test.ts` for highlighted fenced code, unknown-language fallback/no crash, escaping of raw HTML inside code, copy button `data-code`, JSON collapse coexistence, and cache/fallback behavior as needed. Add Control UI styles in `ui/src/styles/components.css` that work in light and dark themes. Add a user-facing changelog entry with `Thanks @cxxzsh`. Do not change unrelated WebChat behavior, channel renderers, or the already implemented copy-button behavior from related copy-button issues.
