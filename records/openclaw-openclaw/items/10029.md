## Summary

Keep open. Current main still lacks syntax highlighting in the live WebChat/Control UI markdown path: code fences and indented blocks are escaped directly into `<pre><code>` output, while the only syntax-highlighting implementation found is for transcript export HTML and not the live chat renderer. The related copy-button work is present but does not satisfy the requested highlighting behavior.

## What This Changes

Add syntax highlighting for code blocks rendered in OpenClaw WebChat/Control UI markdown, ideally with a theme-aware highlighter and language handling.

## Best Possible Solution

Implement highlighting in the shared Control UI markdown layer used by live WebChat, with theme-aware token styles and focused regression coverage for fenced and indented code, language labels, copy buttons, JSON collapse, sanitization, cache behavior, and oversized-message fallback.

## Reproduction Assessment

Yes. Rendering a fenced block such as ```cpp through the live chat path currently flows through `toSanitizedMarkdownHtml` and emits escaped `<pre><code class="language-cpp">...` output without highlighted token markup; this was verified by static source inspection under the read-only review constraints.

## Solution Assessment

Yes. A narrow Control UI markdown-renderer/style change is the maintainable solution; backend rendering or transcript-export-only changes would not solve the live WebChat surface described here.
