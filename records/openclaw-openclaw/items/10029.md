## Summary

Keep open. Current main still lacks syntax highlighting in the live WebChat/Control UI markdown path: fenced and indented code blocks are escaped directly into `<pre><code>` output, and the only highlighter found is limited to transcript export HTML rather than the live chat renderer.

## What This Changes

Add syntax highlighting for code blocks rendered in OpenClaw WebChat/Control UI markdown.

## Best Possible Solution

Implement highlighting in the shared Control UI markdown layer used by live WebChat, with theme-aware token styles and regression coverage for fenced and indented code, language labels, copy buttons, JSON collapse, sanitization, cache behavior, and oversized-message fallback.

## Reproduction Assessment

Yes. Rendering a fenced block such as ```cpp through the live chat path currently flows through `toSanitizedMarkdownHtml` and emits escaped `<pre><code class="language-cpp">...` output without highlighted token markup; this is verifiable from the current source path.

## Solution Assessment

Yes. A narrow Control UI markdown-renderer/style change is the maintainable solution; backend rendering or transcript-export-only changes would not solve the live WebChat surface described here.
