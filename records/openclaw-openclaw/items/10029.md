## Summary

Current main still renders WebChat code blocks as escaped monospace text with language labels, copy buttons, and JSON collapse chrome, but it does not tokenize code or emit syntax-highlighting markup/styles. The prior Codex review comment remains accurate: the shipped copy-button work related to #30390 is separate from this syntax-highlighting enhancement.

## Best Possible Solution

Keep this issue open and implement syntax highlighting in the Control UI/WebChat markdown renderer or a narrow adjacent renderer helper. The fix should add sanitization-compatible highlighted markup/styles for fenced code blocks, preserve existing code-block chrome and fallback behavior, and add focused tests proving highlighted output without regressing copy buttons, JSON collapse, cache, and oversized-message handling.
