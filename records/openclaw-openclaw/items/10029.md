## Summary

Keep open. Current main still renders live WebChat/Control UI code fences as escaped plain code with code-block chrome, not syntax-highlighted token markup; the existing highlight.js-style code is limited to exported transcript HTML, and no open fixing PR was present in the provided context.

## What This Changes

The issue requests syntax highlighting for WebChat/Control UI code blocks, ideally by integrating a highlighter into the Markdown renderer with matching theme styles.

## Best Possible Solution

Implement highlighting in the shared Control UI markdown layer used by live chat, with theme-aware token styles and focused regression coverage for fenced and indented code, language labels, copy buttons, JSON collapse, sanitization, cache behavior, and oversized-message fallback.

## Reproduction Assessment

Yes. Rendering a fenced block such as ```cpp through `toSanitizedMarkdownHtml` on the live chat path currently produces escaped code inside `<pre><code class="language-cpp">` without highlighted token markup.

## Solution Assessment

Yes. The narrow maintainable path is a Control UI markdown-renderer/style change; backend rendering, channel-level changes, or transcript-export-only changes would not solve the live WebChat surface this issue describes.
