## Summary

Keep open. Current main still lacks syntax highlighting in the live WebChat/Control UI markdown path: code fences and indented code blocks are escaped into plain `<pre><code>` output, while the only syntax-highlighting code found is for transcript export HTML and does not affect live chat rendering.

## What This Changes

The issue requests syntax highlighting for code blocks in the WebChat/Control UI markdown renderer.

## Best Possible Solution

Implement highlighting in the shared Control UI markdown layer used by live WebChat, with theme-aware token styles and regression coverage for fenced and indented code, language labels, copy buttons, JSON collapse, sanitization, cache behavior, and oversized-message fallback.

## Reproduction Assessment

Yes. Rendering a fenced block such as ```cpp through the live chat path currently flows through `toSanitizedMarkdownHtml` and emits escaped `<pre><code class="language-cpp">` output without highlighted token markup.

## Solution Assessment

Yes for a renderer-layer fix, but the exact highlighter library should be chosen by maintainers or the fix PR based on bundle size, async/lazy behavior, security, and theme fit; transcript-export-only or backend rendering would not solve the live WebChat surface.
