## Summary

Current main still renders WebChat fenced and indented code blocks as escaped monospace text with copy-button/language/JSON-collapse chrome, but it does not tokenize code or emit syntax-highlighting spans/classes. The related copy-button work tracked by #30390 is separate and does not satisfy this enhancement.

## Best Possible Solution

Keep this issue open and implement syntax highlighting in the Control UI/WebChat markdown renderer or a narrow adjacent helper. The fix should add sanitization-compatible highlighted markup/styles for fenced code blocks, preserve the existing code-block chrome and fallbacks, and add focused tests proving highlighted output without regressing copy buttons, JSON collapse, cache behavior, and oversized-message handling.
