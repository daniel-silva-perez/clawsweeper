## Summary

Keep open. Current main still does not add syntax highlighting to live WebChat/Control UI code blocks: the live renderer emits escaped code with language class, copy button, and JSON collapse chrome, while the only highlight.js-style code found is for exported transcript HTML.

## What This Changes

The issue requests syntax highlighting for code blocks in OpenClaw WebChat/Control UI markdown rendering.

## Best Possible Solution

Implement syntax highlighting in the shared Control UI markdown layer with theme-aware token styles and regression coverage for fenced and indented code, language labels, copy buttons, JSON collapse, sanitization, cache behavior, and oversized-message fallback.

Yes. Source inspection gives a high-confidence reproduction path: render a fenced block such as ```cpp through `toSanitizedMarkdownHtml` on the live chat path and current main will output escaped code in `<pre><code class="language-cpp">` without highlighted token markup.

## Solution Assessment

Yes. The narrow maintainable fix is in the shared Control UI markdown renderer and styles; server-side highlighting or core/channel changes would be broader than this WebChat UI enhancement needs.
