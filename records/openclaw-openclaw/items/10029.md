## Summary

Keep open. Current main still routes WebChat/Control UI messages through the shared sanitized markdown renderer, and that renderer emits escaped plain `<pre><code>` blocks with copy/JSON chrome but no token markup or highlighter call. The only syntax-highlighting code found is for exported transcript HTML, not the live chat path.

## What This Changes

Add syntax highlighting to WebChat/Control UI markdown code blocks so fenced snippets are colorized instead of plain monospace text.

## Best Possible Solution

Implement syntax highlighting in the shared Control UI markdown layer with theme-aware token styles and regression coverage for fenced and indented code, language labels, copy buttons, JSON collapse, sanitization, cache behavior, and oversized-message fallback.
