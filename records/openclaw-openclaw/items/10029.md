## Summary

Current main still does not implement WebChat code-block syntax highlighting. The live Control UI markdown renderer preserves code-block chrome, language labels, copy buttons, JSON collapse, sanitization, caching, and large-message fallback, but fenced and indented code contents are still escaped directly into `<pre><code>` without highlighted token markup.

## What This Changes

Request syntax highlighting for fenced and indented code blocks in the WebChat markdown renderer.

## Best Possible Solution

Keep this open and implement a narrow Control UI/WebChat syntax-highlighting pass in the existing markdown renderer or a helper used by it. Preserve copy buttons, language labels, JSON collapse, sanitization, cache behavior, raw HTML escaping, and oversized/plain-text fallback, with focused regression tests and a user-facing changelog entry.
