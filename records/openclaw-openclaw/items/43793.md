## Summary

Keep open. Current main still does not implement the central TUI gateway-disconnect UX from the PR: ordinary disconnects still resolve to an idle activity state, there is no Ctrl+R editor hook, and no generic reconnect guidance message is posted. The PR is still useful and the author addressed earlier bot review feedback, but the branch is stale against the current TUI backend split and includes a macOS SSH probe host-key policy regression that needs maintainer-guided rebase/review before merge.

## What This Changes

The PR adds TUI gateway-disconnect guidance and Ctrl+R manual reconnect plumbing, gateway-client reconnect timer handling, macOS remote-gateway probe/auth UI changes, and a diffs-store cleanup test adjustment.

## Best Possible Solution

Land a rebased and narrowed version that adds a typed backend-level manual reconnect capability for gateway mode, wires Ctrl+R plus generic reconnectable-disconnect guidance, preserves auth/pairing-specific paused-reconnect guidance, keeps the current strict macOS SSH host-key policy, and splits unrelated diffs-store or macOS work unless it is required for the same user-facing fix.
