## Summary

Keep this PR open. Current main still does not implement the central TUI reconnect UX: generic gateway disconnects resolve to idle, the disconnect handler only emits pairing-specific guidance, there is no Ctrl+R retry hook, and the TUI backend does not expose a manual reconnect API. The PR discussion also shows active contributor follow-up on real review findings, so it remains an implementation candidate rather than conservative cleanup.

## Best Possible Solution

Keep this PR open for maintainer review, but ask the contributor to rebase and narrow it. The best path is a focused TUI/gateway change that exposes a safe manual reconnect API, wires a visible retry shortcut or command, shows retry/restart guidance only for reconnectable disconnects, preserves auth/pairing-specific guidance through the current reconnect-paused detail-code seams, resets activity status on reconnect, and includes regression tests. Split or separately justify the macOS remote-probe and diffs-store cleanup work unless a maintainer explicitly wants those surfaces reviewed together.
