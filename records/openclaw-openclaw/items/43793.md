## Summary

Keep this PR open. Current main still lacks the central TUI reconnect UX: generic gateway disconnects fall back to idle, the disconnect handler only emits pairing-specific guidance, there is no Ctrl+R retry hook, and the TUI backend still exposes start/stop without a manual reconnect API. The PR discussion shows active contributor follow-up on real review findings, so this remains an implementation candidate rather than conservative cleanup.

## Best Possible Solution

Keep this PR open for maintainer review, but ask for a rebase and likely split. The best path is a focused TUI/gateway change that exposes a safe manual reconnect API, wires a visible retry shortcut or command, shows retry/restart guidance only for reconnectable disconnects, preserves auth/pairing-specific guidance, resets activity status on reconnect, and includes regression tests. The macOS remote-probe and diffs-store cleanup work should be split or separately justified.
