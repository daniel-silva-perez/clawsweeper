## Summary

Keep open. Current main still uses the low-alpha `--accent-subtle` color for global browser text selection in the Control UI/WebChat styling, and the linked implementation candidate remains an open, unmerged PR. Repository cleanup policy also says an issue should stay open while an open PR explicitly targets it for closure.

## Best Possible Solution

Keep the issue open and review/land PR #60854, or apply an equivalent focused Control UI CSS fix that changes `::selection` to a visibly higher-contrast background/text pair and includes a quick visual accessibility check or lightweight CSS contract coverage.
