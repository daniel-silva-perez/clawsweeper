Current main still lacks any `<session-recap>` / `<session_recap>` handling in the shared inbound metadata stripper, while both Control UI and TUI user-message rendering still depend on that stripper. The supplied GitHub context also shows an open, unmerged implementation candidate, #41013, with closing syntax for this issue, so the right maintainer path is to review/land or close that PR rather than close this issue now.

Review #41013 or apply an equivalent narrow fix in `src/auto-reply/reply/strip-inbound-meta.ts`: add a session-recap quick check, strip only complete leading `<session-recap>` / `<session_recap>` blocks in both `stripInboundMetadata()` and `stripLeadingInboundMetadata()`, preserve mid-message literal tags, and add focused regression tests in `src/auto-reply/reply/strip-inbound-meta.test.ts` plus UI/TUI consumer coverage if maintainers want end-to-end proof.
