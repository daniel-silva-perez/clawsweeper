## Summary

Current main still does not strip leading `<session-recap>` / `<session_recap>` blocks from the shared inbound metadata sanitizer, while the Control UI and TUI continue to route user-visible user text through that sanitizer. This issue is also paired with open PR #41013 using closing syntax, so it should remain open until that PR lands or maintainers choose an equivalent replacement fix.

## Best Possible Solution

Keep this issue open and review PR #41013, or apply an equivalent narrow fix in `src/auto-reply/reply/strip-inbound-meta.ts`: add a session-recap quick check, strip only complete leading `<session-recap>` / `<session_recap>` blocks in both `stripInboundMetadata()` and `stripLeadingInboundMetadata()`, preserve mid-message and malformed literals, and add focused regression tests in `src/auto-reply/reply/strip-inbound-meta.test.ts` plus any UI/TUI consumer coverage maintainers want.
