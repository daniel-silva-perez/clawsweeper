## Summary

Current main no longer matches the reported 2026.4.1 behavior: exec approvals default to full/off/full, the exec tool falls back to those approval-file defaults when tools.exec is unset, and approval normalization preserves existing/legacy allowlists rather than replacing them with a strict empty allowlist.

## Evidence

- **Issue report describes strict reset in 2026.4.1:** The issue reports that after updating to 2026.4.1, exec-approvals.json was initialized with security: deny and a minimal allowlist, blocking cron/scheduler commands such as python3.
  - command: `web open https://github.com/openclaw/openclaw/issues/59701`
- **Current defaults are no-prompt host defaults:** Current main sets DEFAULT_SECURITY to full, DEFAULT_ASK to off, and DEFAULT_EXEC_APPROVAL_ASK_FALLBACK to full, so a missing/default approvals policy no longer becomes strict deny/on-miss.
  - file: [src/infra/exec-approvals.ts:3](https://github.com/openclaw/openclaw/blob/d6ef1fcf24eea3a5985d653a3baf400e32aa1b75/src/infra/exec-approvals.ts#L3)
  - command: `web open raw main src/infra/exec-approvals.ts`
  - sha: [d6ef1fcf24ee](https://github.com/openclaw/openclaw/commit/d6ef1fcf24eea3a5985d653a3baf400e32aa1b75)
- **Current normalization preserves allowlists:** normalizeExecApprovals migrates legacy agents.default into agents.main with mergeLegacyAgent, coerces legacy string allowlist entries, adds IDs, and sanitizes invalid policy fields without dropping valid allowlist entries.
  - file: [src/infra/exec-approvals.ts:3](https://github.com/openclaw/openclaw/blob/d6ef1fcf24eea3a5985d653a3baf400e32aa1b75/src/infra/exec-approvals.ts#L3)

(sandbox ? deny : full), with ask falling back through approvalDefaults?.ask ?? off. That aligns unset host exec with the current full/off defaults.
  - file: [src/agents/bash-tools.exec.ts:14](https://github.com/openclaw/openclaw/blob/d6ef1fcf24eea3a5985d653a3baf400e32aa1b75/src/agents/bash-tools.exec.ts#L14)

  - file: [docs/tools/exec-approvals.md:40](https://github.com/openclaw/openclaw/blob/d6ef1fcf24eea3a5985d653a3baf400e32aa1b75/docs/tools/exec-approvals.md#L40)
