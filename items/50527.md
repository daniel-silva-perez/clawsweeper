## Summary

Keep open. Current main does not implement the requested per-field length limits for hook agent metadata, and the open PR is not superseded. The PR is still useful, but it should not be closed as implemented because the submitted patch only validates direct `/hooks/agent` normalization while mapped hooks can still render and dispatch the same metadata fields without those limits.

## Best Possible Solution

Keep the PR open and ask for a shared hook-agent metadata length validator applied to both direct `normalizeAgentPayload()` results and mapped/transformed `HookAction` values before replay-key generation and `dispatchAgentHook()`. Add regression tests for direct and mapped paths, including exact-boundary coverage for all six metadata fields.
