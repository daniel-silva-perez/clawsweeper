## Summary

Current main does not yet satisfy #70475. There is a partial internal `MemoryDreamingExecutionConfig.model` shape, but the bundled `memory-core` manifest and public docs do not expose a Dreaming model setting, docs still say the Dream Diary uses the default runtime model, and the actual Dreaming narrative subagent calls do not pass a provider/model override. The issue comment is useful: local/embedding providers can reduce memory-search embedding cost, but that is separate from selecting a dedicated LLM for Dreaming's background narrative/subagent work.

## Best Possible Solution

Keep #70475 open. The best product/code path is to decide the supported config shape, likely under `plugins.entries.memory-core.config.dreaming.execution.defaults.model` with optional per-phase overrides, add it to the memory-core manifest/docs, thread the resolved model into Dream Diary subagent runs using the existing plugin subagent provider/model override API and trust policy, preserve default-model inheritance when unset, and add regression tests that a configured Dreaming model reaches `api.runtime.subagent.run`.
