## Summary

Keep open. Current main still omits `WorkingDirectory` on the normal packaged gateway install path and still resolves configured relative `agentDir` values against `process.cwd()`. Fresh setup appears to have moved away from writing a relative `agentDir` by default, but the issue's requested core fixes are not implemented and legacy/explicit relative configs can still hit the same class of launchd failure.

## Best Possible Solution

Keep #70223 open and fix the actual remaining boundary: set a deterministic `workingDirectory` for packaged macOS LaunchAgent installs, preferably HOME or the OpenClaw state/config directory, and defensively resolve configured relative `agentDir` values against the loaded config/state directory instead of `process.cwd()`. Add focused regression coverage for packaged `resolveGatewayProgramArguments()`/`buildGatewayInstallPlan()`, LaunchAgent plist generation, and `resolveAgentDir()` with relative config values under a simulated cwd of `/`.
