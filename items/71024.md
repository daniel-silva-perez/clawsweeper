## Summary

Current main still lacks the Mattermost read-only account inspector this PR adds, so the linked SecretRef status crash remains a real product bug rather than obsolete cleanup. The PR is a focused bug fix with targeted tests, though the Greptile env-token precedence review note should be checked before landing.

## Best Possible Solution

Keep #71024 open. The best maintainer path is to land or recreate a focused Mattermost `inspectAccount` implementation, preserve `botTokenStatus: configured_unavailable` in status snapshots, add the two proposed regression tests, and address the env-fallback ordering review note before merge.
