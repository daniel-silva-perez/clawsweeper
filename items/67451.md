## Summary

Keep this PR open. Current main still rejects the requested `group:` and `whatsapp:group:` WhatsApp group target forms, and the outbound messaging surface still depends on that normalizer. The PR is stale because it targets the old `src/whatsapp/*` path, but the underlying fix remains useful in the bundled WhatsApp plugin.

## Best Possible Solution

Keep the PR open as a focused implementation candidate, but ask the author to refresh it against current main: update `extensions/whatsapp/src/normalize-target.ts` to strip repeated `whatsapp:` and `group:` prefixes, add regression coverage in `extensions/whatsapp/src/resolve-target.test.ts` for valid group prefixes, mixed case, empty-group invalid cases, and repeated prefixes, then run the WhatsApp plugin test lane.
