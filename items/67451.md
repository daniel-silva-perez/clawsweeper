## Summary

Keep this PR open. Current main still does not handle `group:` or `whatsapp:group:` prefixes in WhatsApp target normalization: the normalizer strips only `whatsapp:`, and current tests explicitly expect the requested group-prefixed forms to be rejected. The PR is stale because it targets the old `src/whatsapp/*` path while current WhatsApp code lives under `extensions/whatsapp/src/*`, but the underlying fix remains useful and belongs in the bundled WhatsApp plugin.

## Best Possible Solution

Keep the PR open as a focused implementation candidate, but ask for a refresh against current main: update `extensions/whatsapp/src/normalize-target.ts` to strip repeated `whatsapp:` and `group:` target-kind prefixes, update `extensions/whatsapp/src/resolve-target.test.ts` for `group:<jid>`, `whatsapp:group:<jid>`, mixed-case, invalid empty-group, and repeated-prefix cases, then run the WhatsApp extension test lane.
