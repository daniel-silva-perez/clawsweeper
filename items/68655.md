## Summary

Keep #68655 open. Current main still routes web Control UI streaming text through the same full-message Markdown render path: each delta replaces `chatStream`, the stream item is rendered with `isStreaming: true`, but `renderGroupedMessage()` still calls `unsafeHTML(toSanitizedMarkdownHtml(markdown))` for streaming markdown. The only visible streaming guard in that function skips JSON detection, not Markdown subtree replacement. I did not find a merged fix, regression test, or canonical superseding item for this web UI flicker; the related reports are channel-specific markdown/chunking work, not this Control UI DOM-stability bug.

## Best Possible Solution

Keep the issue open and fix it in the Control UI chat renderer. A conservative path is to add a streaming-aware markdown render layer in `ui/src/ui/chat/grouped-render.ts` that preserves already-complete paragraph/code-fence DOM and only updates the trailing partial block, or uses a clearly scoped plain-text fallback while `isStreaming` is true. Add a jsdom regression test that rerenders a streaming markdown response with multiple paragraphs and an opening/closing code fence and asserts stable completed DOM nodes across updates. The related channel markdown/chunking items do not supersede this web UI DOM-stability bug.
