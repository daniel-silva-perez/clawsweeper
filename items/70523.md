## Summary

Keep open. Current main mitigates the linked daily-memory pollution for default users by defaulting Dreaming storage to `separate`, which explains why #70509 was closed. But #70523 targets the still-supported explicit `inline`/`both` storage path, and current main still writes raw light/REM phase bodies into `memory/YYYY-MM-DD.md` before narrative fallback with no fallback result or suppression path.

## Best Possible Solution

Keep #70523 open for maintainer reconciliation. The best path is either a narrow memory-core fix that prevents raw inline fallback dumps without transient sensitive persistence, or an explicit product decision to retire/deprecate inline and both-mode dreaming output. If this PR continues, strip unrelated docs changes and resolve the fallback/write-order review concerns.
