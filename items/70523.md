## Summary

Keep #70523 open. Current main still writes raw light/REM dreaming phase blocks into `memory/YYYY-MM-DD.md` before narrative fallback, and the fallback path does not report request-scoped fallback state back to the phase runner. The PR is not implemented on main, and the related open/broader dreaming hardening PR is not a clear canonical replacement for this exact daily-memory pollution fix.

## Best Possible Solution

Keep #70523 open for maintainer review. The best path is a scoped memory-core fix that either defers raw inline daily writes until request-scoped fallback is known safe or reliably removes only the managed raw block after a successful fallback, with regression coverage for light and REM daily-file pollution and without unrelated docs changes. If #70332 or another canonical PR lands the same daily-file fallback suppression first, then close #70523 as superseded with that link.
