## Summary

Keep #49109 open. The current main branch does not contain the PR's two central fixes: cron helper status checks still run during JSON-producing cron commands and do not use a quiet/probe helper call, and CLI gateway RPC still performs a single CLI-mode call with no retry path for transient transport failures. The PR is broad and outdated, but the underlying bug-fix work is still potentially useful rather than obsolete.

## Best Possible Solution

Keep the item open, but do not land the branch as-is. The best maintainer path is to ask for a small rebased PR or cherry-pick-equivalent patch that only updates src/cli/cron-cli/shared.ts, src/cli/gateway-rpc.ts, src/cli/gateway-rpc.runtime.ts, and focused CLI tests: skip cron.status helper checks under --json, run non-JSON helper probes with progress false, quiet/probe mode, and expectFinal false, and add a narrow bounded retry for pre-request transient gateway transport failures.
