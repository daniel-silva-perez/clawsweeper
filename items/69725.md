## Summary

Keep PR #69725 open. Current main still has the reported macOS node mismatch: the agent-side `exec host=node` path checks only for `system.run`, then always sends `system.run.prepare`; the macOS app still advertises and handles `system.run`/`system.which`/`system.notify` but not `system.run.prepare`; and the gateway still rejects undeclared node commands before forwarding. The PR is a plausible focused compatibility fix rather than obsolete or safely superseded by the open macOS-side prepare PR.

## Best Possible Solution

Keep the PR open for maintainer review. The best product direction is either to land a compatibility fallback like #69725 for existing macOS nodes that lack `system.run.prepare`, or explicitly choose the macOS-side implementation in #38781 and close this PR only after that path also handles already-shipped/legacy nodes or maintainers decide legacy compatibility is out of scope.
