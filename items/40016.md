## Summary

Keep #40016 open. Current main still lacks the requested user-facing `openclaw sessions set-subject` or `openclaw sessions rename` command, and the text `openclaw sessions` output still renders truncated raw keys without a label/displayName column. Gateway label patching exists, so the remaining work is a CLI/product surface, docs, and tests, not backend storage.

## Best Possible Solution

Keep the issue open and implement a focused CLI surface, likely `openclaw sessions rename <session-key> <label>` with `set-label` or `set-subject` as an alias if maintainers want compatibility with the issue wording. The command should call the existing `sessions.patch`/store label path, validate duplicate and empty labels through the existing session-label helpers, update `openclaw sessions` text/JSON output to surface label or displayName before the truncated key, and add CLI registration/unit tests plus `docs/cli/sessions.md` coverage.
