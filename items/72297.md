## Summary

Keep this PR open. Current main still has the QMD name-only collection-list gap, and the latest PR head is recent, focused work that now avoids the earlier destructive stderr-only removal path. It should not be auto-closed, but it needs maintainer review because the current patch is warning-only while the title/body still present it as a full rebind fix for #71783.

## Best Possible Solution

Keep this PR open for maintainer review. Either narrow it explicitly to a warning-only improvement by updating the title/body and removing the closing relationship to #71783, or implement a verified repair path that only rebinds when QMD exposes authoritative path/pattern metadata for the conflicting collection. If QMD cannot provide that metadata, the conservative warning-and-manual-repair path is the right product behavior.
