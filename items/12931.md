## Summary

Keep open. Current `main` still hard-codes a model-generated greeting for bare `/new` and `/reset`, and the session config schema does not expose any `resetGreeting`/`resetGreetingModel`-style option to suppress that turn or route it to a cheaper model.

## Evidence

- **Bare reset prompt explicitly tells the model to greet:** The bare session reset prompt unconditionally says: "Then greet the user in your configured persona..." for `/new` and `/reset`, so current behavior still spends a model turn on the greeting.
