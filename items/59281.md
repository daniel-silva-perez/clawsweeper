## Summary

Keep open. Current main still routes Windows plugin loading away from native jiti/import: `supportsNativeJitiRuntime()` returns false on `win32`, `shouldPreferNativeJiti()` therefore returns false, and the loader passes that `tryNative` value into the cached jiti loader. The issue is recent, concrete, not protected-label exempt, but the code evidence does not support an implemented/cannot-reproduce close.

## Evidence

- **Windows native jiti still disabled:** `supportsNativeJitiRuntime()` returns `typeof versions.bun !== "string" && process.platform !== "win32"`, so Windows cannot reach the native-jiti preference path on current main.
