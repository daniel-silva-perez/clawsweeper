## Summary

Keep open. Current main still contains actionable core doctor behavior matching the most important parts of #54740: launchd scanning can still report non-gateway OpenClaw ecosystem LaunchAgents as “Other gateway-like services,” and doctor still prints canonical gateway cleanup hints whenever any extra service is found. #52510 covers one narrower cleanup-hint symptom and #15849 was closed stale, so there is no canonical open/shipped item that fully supersedes this broader report.

## Best Possible Solution

Keep #54740 open and fix the core doctor path. The likely product/code direction is to make launchd discovery parse service kind plus ProgramArguments instead of whole-plist namespace matches, ignore non-gateway OpenClaw ecosystem services, suppress cleanup hints for the currently loaded/running canonical gateway service, and add macOS launchd regression tests. The gateway.mode warning can be verified separately, but it is not enough to close the broader issue while the service-scan and cleanup-hint false positives remain.
