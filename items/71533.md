## Summary

Keep #71533 open. Current main already passes a silent pino-like logger to Baileys by default, but it does not expose `channels.whatsapp.logLevel` or `channels.whatsapp.baileys.logLevel`, and the specific reconnect/watchdog lines cited by the issue are still emitted by OpenClaw’s WhatsApp monitor/runtime path rather than being controlled by a channel config knob.

## Best Possible Solution

Keep the issue open and implement it in the WhatsApp plugin surface: add a documented, validated `channels.whatsapp.logLevel` or narrower `channels.whatsapp.baileys.logLevel` field, decide whether it should govern only the Baileys pino-like socket logger or also OpenClaw’s own reconnect/watchdog lifecycle messages, then add focused schema, docs, and monitor/session tests. Because the remaining behavior is in the bundled WhatsApp plugin and its config contract, this is actionable in this repository rather than a ClawHub-only item.
