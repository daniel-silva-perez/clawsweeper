## Summary

Keep open. Current main already classifies the plain `An error occurred while processing...` provider message as a timeout/failover signal, but it does not implement the PR's user-facing rewrite or request-id preservation. The PR is stale against current helper layout, but the observable formatting problem remains useful to fix.

## Best Possible Solution

Keep the PR or a replacement patch open, rebase it onto current main, add a narrow shared helper for generic provider internal-error text plus request-id extraction, use it from both `formatAssistantErrorText` and `formatRawAssistantErrorForUi`/sanitization as appropriate, and add regression tests for full help-center wording and request-ID-only wording while preserving the existing timeout classification.
