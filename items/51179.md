## Summary

Keep open. Current `main` still truncates `allowFrom` entries in the live `openclaw channels status --probe` formatter and in the channel-summary helper, so the PR is not obsolete or implemented. The PR also has a still-relevant review concern around exposing full allowlists through non-admin status summaries, so it needs maintainer review or revision rather than cleanup closure.

## Best Possible Solution

Keep the PR open or consolidate it with the overlapping `allowFrom` status PR. The best code path is a small maintainer-reviewed fix that removes the hard truncation from the live `channels status --probe` formatter and channel-summary helper, adds regression coverage for 3+ entries, and gates full allowlist rendering behind sensitive/admin status summaries so non-admin `health.status` responses do not expose complete allowlists.
