## Summary

Keep open. Current main still omits `sessionKey` from the system-prompt runtime types, the embedded-runner prompt call, and the `Runtime:` line renderer, so this PR remains a focused implementation candidate rather than obsolete or already shipped.

## Best Possible Solution

Keep this PR open for maintainer review. The likely path is to rebase it onto current main, resolve the minor test model constant note, run the changed agents prompt checks, and land if maintainers want the Runtime line to expose the concrete session key; alternatively, explicitly choose the `sessions_history("current")` design and update prompt/tool docs before closing the PR.
