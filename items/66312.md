## Summary

Keep this PR open. Current main still does not thread `sessionKey` into the system-prompt runtime metadata or `Runtime:` line, and the PR remains a focused implementation candidate for the linked feature request. The existing `sessionKey: "current"` resolver is adjacent, but current prompt text does not expose either the concrete runtime key or clear guidance that solves the requested Runtime-line behavior.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase it onto current main, address the minor test model constant note, make a maintainer product/security decision on raw `sessionKey` versus the `current` alias design, then run the changed agents prompt checks and land if OpenClaw wants the Runtime line to expose the concrete session key. If maintainers prefer the alias design, update prompt/tool guidance accordingly and close the PR with that rationale.
