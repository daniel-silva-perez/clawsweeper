## Summary

Keep open. Current `main` still reproduces the reported logic bug: both the state-dir and config-dir resolution paths append `.openclaw` to `OPENCLAW_HOME` unconditionally, so `OPENCLAW_HOME=/home/user/.openclaw` resolves to `/home/user/.openclaw/.openclaw`. The current tests also still assert that nested behavior.

## Evidence

If `OPENCLAW_HOME` already points at a `.openclaw` directory, this still yields a nested `.openclaw/.openclaw` path.
