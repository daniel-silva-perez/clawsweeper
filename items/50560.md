## Summary

Keep open. Current `main` still reads stale systemd exit metadata (`ExecMainStatus`/`ExecMainCode`) into the runtime snapshot and always renders it in `gateway/daemon status`, so the reported post-restart stale-state symptom is not clearly fixed on `main`.

## Evidence

- **systemd runtime still captures last exit metadata on every status read:** `readSystemdServiceRuntime()` calls `systemctl show` for `ActiveState,SubState,MainPID,ExecMainStatus,ExecMainCode` and returns `lastExitStatus`/`lastExitReason` even when `ActiveState` is `active` and `status` becomes `running`.
  - file: [src/daemon/systemd.ts:683](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/daemon/systemd.ts#L683)
  - command: `nl -ba src/daemon/systemd.ts | sed -n '682,709p'`
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **runtime formatter does not suppress stale exit details for running services:** `formatRuntimeStatus()` always appends `last exit` and `reason` when those fields exist; there is no guard for `runtime.status === "running"`.
  - file: [src/daemon/runtime-format.ts:19](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/daemon/runtime-format.ts#L19)
  - command: `nl -ba src/daemon/runtime-format.ts | sed -n '15,44p'`
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **daemon status prints runtime summary verbatim:** `printDaemonStatus()` prints the `formatRuntimeStatus(service.runtime)` result directly as the `Runtime:` line, so any stale systemd exit metadata reaches user-visible status output.
  - file: [src/cli/daemon-cli/status.print.ts:177](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/cli/daemon-cli/status.print.ts#L177)
  - command: `nl -ba src/cli/daemon-cli/status.print.ts | sed -n '176,190p'`
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **tests only cover parsing inactive exit details, not suppressing stale details while active:** The current systemd runtime parsing tests validate inactive/dead output with exit metadata and malformed numeric rejection, but there is no coverage for an active/running unit carrying stale exit details after restart.
  - file: [src/daemon/systemd.test.ts:392](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/daemon/systemd.test.ts#L392)
  - command: `nl -ba src/daemon/systemd.test.ts | sed -n '390,417p'`
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
