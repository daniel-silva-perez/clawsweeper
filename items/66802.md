## Summary

Keep open. Current main still does not implement `agents.defaults.compaction.memoryFlush.model`: the nested config type/schema/generated schema/docs omit the field, the memory flush plan has no model slot, and the runtime still resolves the flush turn from the session run model. PR #66846 is an open, unmerged implementation candidate for the same request.

## Best Possible Solution

Keep this issue open until PR #66846 is reviewed and merged, or land an equivalent implementation on main: add `agents.defaults.compaction.memoryFlush.model` to config types, zod/generated schema, labels/help/docs; carry it through the memory flush plan or runtime boundary; resolve provider/model/auth consistently with `compaction.model`; and add regression coverage proving override and fallback behavior.
