## Summary

Keep open. Current main still does not implement `agents.defaults.compaction.memoryFlush.model`: the strict config schema and exported type omit it, the memory-core flush plan has no model field to pass through, and the runtime memory flush path still calls `resolveModelFallbackOptions()` from the session run model. The related open PR #66846 is an implementation candidate for this same request, but it is not merged and the feature is not shipped in the checked-out main.

## Best Possible Solution

Keep this issue open until #66846 is reviewed and merged, or implement an equivalent change on main: add `agents.defaults.compaction.memoryFlush.model` to config types/schema/generated help/docs, carry the override through the memory flush plan or runtime config boundary, resolve provider/model/auth consistently with `compaction.model`, and add regression coverage that proves the flush turn uses the override and falls back to the session model when unset.
