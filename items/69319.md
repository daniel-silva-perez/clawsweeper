## Summary

Keep #69319 open. Current main still lacks a registered `skills.uninstall` Gateway RPC, protocol schema/type/validator, admin scope entry, Swift protocol model, docs entry, handler tests, and ClawHub backend uninstall helper. The PR is a focused implementation for the still-open #69299 request, but it also has substantive review feedback about destructive deletion of non-ClawHub skill folders and error classification that should be addressed before merge rather than closed as obsolete.

## Best Possible Solution

Keep the PR open and request changes. The best path is to continue from #69319, gate deletion on ClawHub provenance (`.clawhub/origin.json` or workspace lockfile membership), classify malformed slug/path input as `INVALID_REQUEST`, keep the idempotent not-installed behavior, and retain/update the backend, Gateway handler, scope, protocol, docs, Swift model, and regression tests before merge. #69299 should remain the canonical product request until this or an equivalent fixed PR lands.
