## Summary

Keep this issue open. Current main partially fixes the durable `.env` / config-service-env and gateway-token drift class, but the macOS LaunchAgent install path still has remaining plaintext projection risk for non-durable auth-profile env refs and lacks proof for the broader FILE/structured SecretRef and reinstall/update vectors raised in the discussion.

## Best Possible Solution

Keep this issue open and finish the macOS install-path hardening: avoid serializing secret-bearing auth-profile, provider, plugin, channel, durable `.env`, and config service values into LaunchAgent `EnvironmentVariables`; load them through a launchd-safe runtime source or wrapper; scrub legacy inline copies during repair/reinstall/update; and add regression coverage for `gateway install --force`, update/npm reinstall, ENV/FILE/structured SecretRefs, auth profiles, provider/channel/plugin secrets, and gateway token behavior.
