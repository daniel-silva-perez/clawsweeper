## Summary

Current main only partially overlaps with the request. It has a chat-session select grouped by agent/other, a Sessions table with label editing, filtering, pagination, and column sorting, and backend session labels/search. It does not implement a left conversation-list manager with drag ordering, user-defined folders/tags/custom categories, or move-to-category behavior, so the feature request remains actionable core Control UI work.

## Best Possible Solution

Keep the issue open and scope it as a Control UI/session-management feature. A complete path would add a persisted session-organization model, additive gateway protocol fields/methods for folder/tag/category/order updates, UI affordances in the chat/session list surface, and focused tests for ordering, grouping, renaming, moving, and backward-compatible sessions without organization metadata.
