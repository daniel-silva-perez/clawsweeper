## Summary

Current main still does not implement the requested left-side conversation list management: no manual drag ordering, no persistent folder/tag/custom category grouping, no move-between-groups API, and no double-click/context-menu rename flow in the chat sidebar. Existing functionality covers session listing, filtering/sorting, select-based switching, and label editing through sessions.patch.

## Best Possible Solution

Keep this open as a Control UI conversation-management enhancement. The likely path is to design a persistent session organization contract, extend SessionEntry/GatewaySessionRow plus gateway protocol APIs for folder/tag/category assignment and manual ordering, then add a left conversation list with drag ordering, move/group actions, and double-click/context-menu rename. Existing label support can be reused for display-name editing, with focused UI/gateway tests and docs once the product contract is settled.
