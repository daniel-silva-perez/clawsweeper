## Summary

Keep open. Current `main` still runs the installer finalization probes synchronously with no outer timeout wrapper, so the requested 'never hang indefinitely during finalization' behavior is not implemented. One reported probe is partially mitigated because `openclaw daemon status` itself defaults to a 10s timeout, but `scripts/install.sh` still invokes both `npm prefix -g` and `openclaw daemon status --json` directly during finalization.
