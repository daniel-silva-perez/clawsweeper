## Summary

Keep open. Current `main` still has a Telegram send fallback that strips `message_thread_id` and retries threadless after `400: Bad Request: message thread not found`, which can deliver a forum-topic send to the parent chat/general topic instead of failing hard. The test suite explicitly locks in that behavior for text, media, stickers, and polls, so this issue is not implemented-on-main or unreproducible.

## Evidence
