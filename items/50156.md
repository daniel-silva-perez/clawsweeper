## Summary

Keep open. Current `main` still appears to have the exact mismatch described: gateway session defaults are computed from the global configured model, while the main session row resolves against the agent-specific default model, and the Control UI renders both sources in the model picker.

## Evidence

- **Gateway defaults still use global configured model:** `getSessionDefaults()` calls `resolveConfiguredModelRef(...)` with no agent context, so `sessions.list.defaults` is derived from the global configured default rather than the selected/default agent's model.
  - file: [src/gateway/session-utils.ts:1065](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/src/gateway/session-utils.ts#L1065)
  - command: `nl -ba src/gateway/session-utils.ts | sed -n '1058,1090p'`
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **Session rows still use the agent-specific resolved model:** `buildGatewaySessionRow()` resolves `sessionAgentId` from the session key, then uses `resolveSessionModelIdentityRef(..., sessionAgentId, ...)`. For a main-session row with no explicit override, that path falls back to `resolveSessionModelRef(cfg, entry, agentId)`, which uses `resolveDefaultModelForAgent({ cfg, agentId })`.
  - file: [src/gateway/session-utils.ts:1082](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/src/gateway/session-utils.ts#L1082)
  - command: `nl -ba src/gateway/session-utils.ts | sed -n '1280,1375p'`
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **Control UI still renders both the defaults entry and the session model entry:** `resolveChatModelSelectState()` reads `currentOverride` from the active session row and `defaultModel` from `sessionsResult.defaults`, then includes both in `options`. `renderChatModelSelect()` separately renders the top `Default (...)` option plus every option from that list, so differing values surface as two model-specific entries in the dropdown.
  - file: [ui/src/ui/chat-model-select-state.ts:34](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/ui/src/ui/chat-model-select-state.ts#L34)
  - command: `nl -ba ui/src/ui/chat-model-select-state.ts | sed -n '34,104p'`
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **UI select still shows the default row plus all resolved options:** The model picker always renders `<option value="">${defaultLabel}</option>` before iterating through `options`, so any mismatch between `sessions.list.defaults` and the active session model remains visible instead of being collapsed to one default entry.
  - file: [ui/src/ui/chat/session-controls.ts:83](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/ui/src/ui/chat/session-controls.ts#L83)
