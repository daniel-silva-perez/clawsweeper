## Summary

Keep open. Current `main` has partial overlap with the PR by wiring proxy agents into the WhatsApp socket setup, but it does not contain the PR's proxy-backed WhatsApp Web version refresh path, and the current fetch-dispatcher code still appears to flatten env proxy config into a fixed `ProxyAgent`, which would not preserve `NO_PROXY` behavior. That leaves unmerged, plausibly useful work in the PR.
