## Summary

Current main still hardcodes the managed media-store cap at 5 MB and has no schema-backed `media.maxBytes` or equivalent global inbound media cap. Existing per-call/channel `mediaMaxMb` support does not cover all affected paths: the media server and sandbox staging still use `MEDIA_MAX_BYTES` directly, and tests still assert rejection/skipping at `MEDIA_MAX_BYTES + 1`. Related PR #40950 is only an unresolved implementation attempt in the provided context, and related issue #55068 is narrower rather than a canonical replacement for this broader config request.

## Best Possible Solution

Keep #55074 open. The likely maintainer path is to add a schema-backed core media-store size setting with a 5 MB default, thread the resolved value through `saveMediaSource`, `saveMediaBuffer`, `src/media/server.ts`, and sandbox staging, preserve explicit per-call/channel overrides, and update config docs plus focused regression tests for default and raised limits.
