## Summary

Keep open. Current `main` still denies Feishu group text commands by default when no group/top-level `allowFrom` is configured, which matches the reported `/new`/`/status`/`/model` failure mode in group chats. The message-body loss part looks improved on `main` because Feishu now preserves parsed text into `ctx.content` and forwards it as `RawBody`/`CommandBody`, but the command path itself is still not clearly fixed for the reported group setup.

## Evidence

- **Feishu group command auth still depends on allowFrom:** `extensions/feishu/src/bot.ts:554-569` builds `commandAllowFrom` from `groupConfig?.allowFrom ?? configAllowFrom` and sets `commandAuthorized` only when an authorizer is both configured and allowed. With an empty allowlist, group text commands are not authorized.
