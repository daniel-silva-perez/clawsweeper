## Summary

Current `main` has macOS remote-gateway token UX, token-sync logic, and docs for stale token/env override problems, but I did not find strong evidence that the specific local-Docker token-mismatch path from #62422 is fixed end-to-end. The linked open issue #64973 reports the same stale-config/token-mismatch family, so closing this as resolved would be too aggressive.

## Evidence

- **Schema-unavailable is still just the downstream symptom of a failed gateway auth/connect path:** The macOS config screen fetches schema over `GatewayConnection.shared.requestDecoded(method: .configSchema)` and, on failure, only stores the error; the UI then renders `Schema unavailable.`. That matches the report's symptom, but these files do not prove the underlying token-mismatch bug is gone.
