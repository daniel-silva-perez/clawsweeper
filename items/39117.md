## Summary

Keep #39117 open. Current main still lacks the PR's queued busy receipt, per-session stuck-lane recovery API/path, aborted embedded-run stale-reply suppression, and richer sessions.reset audit metadata. The related #39669 tracks only the queued-feedback slice, and the PR also has an unresolved P1 review concern that needs maintainer or author judgment rather than cleanup closure.

## Best Possible Solution

Keep #39117 open for maintainer judgment or author revision. The best path is to split or revise the work: implement queued-message feedback in line with #39669, add focused aborted-run reply suppression, design stuck-lane recovery without marking a still-streaming session idle, and separately decide whether sessions.reset should persist/expose audit metadata with protocol, docs, and tests.
