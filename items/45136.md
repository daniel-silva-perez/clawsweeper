## Summary

Current main already addresses the Mattermost leak through the newer draft-preview delivery path: Mattermost disables generic block streaming for inbound replies, streams partial/tool/thinking updates into one preview post, and finalizes that post in place. That behavior is documented, covered by Mattermost tests, present in the shipped `v2026.4.22` release, and the old generic `text_end` semantics this PR changes are still intentionally preserved by core tests.

## Close Comment

Closing as implemented on main. Codex review checked current main and the shipped `v2026.4.22` release (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`): Mattermost no longer uses the generic `text_end` `onBlockReply` path this PR patches. The current Mattermost monitor disables block streaming and routes partial text, thinking, and tool status into a single draft preview post via `onPartialReply` / `onReasoningStream` / `onToolStart`, then finalizes that preview in place (`extensions/mattermost/src/mattermost/monitor.ts:1742`, `extensions/mattermost/src/mattermost/draft-stream.test.ts:42`, `extensions/mattermost/src/mattermost/monitor.test.ts:347`). The shipped docs also describe that single-preview behavior explicitly (`docs/channels/mattermost.md:247`). At the same time, main still intentionally preserves immediate generic `text_end` block replies in core (`src/agents/pi-embedded-subscribe.subscribe-embedded-pi-session.emits-block-replies-text-end-does-not.test.ts:102`), so this branch's global buffering approach no longer matches the current architecture.
