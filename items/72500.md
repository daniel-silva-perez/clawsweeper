## Summary

Keep this issue open. Current main still renders the credentials gate whenever `connected` is false, and the gateway still flips `connected` to false at reconnect start and on close. PR #72522 is an open, unmerged implementation candidate with closing syntax for this issue, so the issue should remain open until that PR is reviewed and either merged or closed.

## Best Possible Solution

Keep the issue open and use PR #72522 as the active implementation candidate. The likely fix belongs in `ui/src/ui/app-gateway.ts`, `ui/src/ui/app-render.ts`, and the app/view state types: preserve the chat UI after a successful session across transient reconnects, while restoring the credentials gate on explicit auth or pairing failures. After the PR merges, GitHub can close this issue as implemented.
