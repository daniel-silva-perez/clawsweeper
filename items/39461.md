## Summary

Keep #39461 open. Current main still ignores `QQBOT_DATA_DIR` for QQ Bot storage while diagnostics recommends that variable, so the reported Windows path problem remains actionable.

## Best Possible Solution

Implement `QQBOT_DATA_DIR` support in the central QQ Bot storage helper, likely `extensions/qqbot/src/engine/utils/platform.ts:getQQBotDataPath()`, with path normalization/validation and preserved lazy directory creation. Add focused tests for the override and diagnostics output, then either document `QQBOT_DATA_DIR` on `https://docs.openclaw.ai/channels/qqbot` or remove the diagnostic recommendation if maintainers do not want to support it.
