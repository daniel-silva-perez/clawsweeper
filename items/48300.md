## Summary

Keep #48300 open. The reporter later found a stale-index workaround, and current defaults now enable watch/on-search/session-start sync, but current main still has the same plausible suppression path: provider-backed hybrid FTS searches are filtered to the active embedding model. That can explain why a direct `chunks_fts MATCH 'Clyde'` query finds rows while `memory_search` returns none after provider/model/index history drift. The linked PR #48328 remains the direct open fix attempt and is not implemented on main.

## Best Possible Solution

Keep #48300 open until maintainers either land a tested version of #48328 or choose a different fix that makes provider-backed hybrid FTS robust across provider/model/index history changes. The fix should include active regression coverage where an FTS row exists under a previous model and `memory_search` still returns the exact keyword hit, plus coverage for keyword-only hybrid hits near the default minScore.
