## Summary

Not strong enough to close. Current main has multiple freshness safeguards around memory sync, but the hybrid keyword path still applies a provider-model filter to FTS queries and the exact keyword-only hybrid cases are still covered only by skipped tests. That leaves the reported behavior plausible on current main, even if the original reporter later worked around it with sync/watch + restart.

## Evidence

- **Current defaults favor fresh memory indexes:** `resolveSyncConfig` defaults `memorySearch.sync.onSessionStart`, `onSearch`, and `watch` to `true`, and the schema help describes those knobs as the normal freshness path.
  - file: [src/agents/memory-search.ts:355](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/src/agents/memory-search.ts#L355)

- **Builtin manager now bootstraps sync before first empty search:** `MemoryIndexManager.search()` checks for missing indexed content and forces `sync({ reason: "search", force: true })` before returning results, which addresses one stale-after-restart failure mode.
