## Summary

Keep open. Current main still gives non-Azure custom/OpenAI-compatible providers a 16k default context window during the custom onboarding path, and the verification UX still stops at generic success/failure without showing the effective context window or warning that a fallback/default is being used. Runtime low-context warnings/blocks exist now, but they are a later mitigation, not a full fix for the setup UX described in #48338.
