## Summary

Keep open. The discussion shows a useful workaround via `openclaw node install && openclaw node restart`, but current main still appears to leave the original macOS companion app node path unable to satisfy `system.run.prepare`. The agent exec host=node path now explicitly invokes `system.run.prepare`, while the macOS app node advertises and handles `system.run`/`system.which` but not `system.run.prepare`; the gateway rejection path matches the reporter's error when a node has not declared that command.

Another commenter confirmed the `node-host/node` workaround on v2026.4.2.
