## Summary

Keep open. Current main still does not surface session `label`/`displayName` in the CLI status sessions table or TUI `/status` summary, while the gateway/session-listing layer already carries that metadata. The PR is not obsolete, but it likely needs rework before merge because the existing review comments identify valid product/safety concerns around preferring unresolvable `displayName` over resolvable `label` and printing unsanitized friendly names.

## Best Possible Solution

Keep the PR open but request a small rework: pass `label` and `displayName` through the status summary types/builders, render a sanitized friendly value, and prefer the resolvable `label` over `displayName` or otherwise show enough information for users to copy a valid selector. Add focused regression coverage for CLI status and TUI `/status` output. The linked feature issue #51184 should remain the canonical product request until the statusline/session-picker portions are either implemented or split into narrower follow-ups.
