## Summary

Keep PR #66712 open. Current main still has only execution-level direct cron delivery dedup keyed by `createCronExecutionId(jobId, runStartedAt)`, and I did not find an equivalent scheduled-slot guard or the PR's `deliveryAttempted` propagation fix on main. The supplied PR discussion also says the slot-level guard is complementary rather than redundant, and the latest PR head appears to address the earlier review asks.

## Best Possible Solution

Keep #66712 open for maintainer review. The likely best path is to land a focused slot-level direct-delivery dedup guard keyed by job, scheduled slot, and delivery target, keep it explicitly process-local, preserve the existing execution-level cache, and include the `deliveryAttempted` startup-catchup propagation plus regression coverage. Persistent cross-process dedup should remain a separate product/design discussion if needed.
