## Summary

Keep this PR open. Current main still does not implement `skills.allowExternalSkillsIn`: workspace and extra-dir skill loading still requires the candidate realpath to stay inside the configured root, the strict skills config schema/types/generated schema omit the requested key, and the user-facing docs still document the opposite containment behavior. The PR remains an open contributor implementation candidate for a core skills/config change, with provided review context showing the initial containment and generated-schema findings were followed up on rather than making the work obsolete.

## Best Possible Solution

Keep this PR open for maintainer review. If OpenClaw wants trusted external symlinked skill roots, land a focused core skills/config change with safe path containment, generated config/docs artifacts, and tests for default rejection, allowed external containment, prefix-boundary bypasses, canonical prefix resolution, empty entries, and target-workspace sync behavior. If maintainers do not want this trust expansion, close it only after explicit product/security judgment rather than automated cleanup.
