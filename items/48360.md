## Summary

Keep open. Current `main` contains a partial mitigation for local loopback probe budgets, but `openclaw gateway probe` still runs the full detail probe path and still treats post-connect timeout/close failures as unreachable. That means the false-negative class from #48360 is still plausible on current `main`, so there is not strong evidence to close this issue.

## Evidence

- **`gateway probe` still uses the full gateway-status path:** The CLI subcommand `gateway probe` dispatches to `gatewayStatusCommand`; it does not use the lightweight daemon-status probe path.
