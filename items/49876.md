## Summary

Keep open. Current `main` still treats cron delivery as 'deliver the final reply unless it is `NO_REPLY`, heartbeat-only, or a fatal error payload'. There is no fail-closed gate that blocks delivery when non-mutating tool calls fail, when the model fabricates unsupported success text, or when `--no-deliver` is used but the `message` tool remains available.
