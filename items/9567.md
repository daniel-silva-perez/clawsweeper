## Summary

Current `main` already implements the core of this RFC as shipped semantic UI/browser snapshot surfaces: AI snapshots, AX-tree snapshots, token-reduced role snapshots with refs, and A2UI for semantic canvas rendering.

## Risks / Open Questions

- The implementation is scoped to shipped browser/canvas surfaces and is not a repo-wide named 'SCP' standard.
- I did not identify a single historical commit or release where this first landed, only that current main clearly contains it and the latest release still ships it.

## Close Comment

Closing from a Codex review because current `main` already implements the substance of this RFC. OpenClaw ships semantic browser snapshots through `extensions/browser/src/browser/routes/agent.snapshot.ts:344` with both `format=aria` and `format=ai`; the backends use the accessibility tree and Playwright AI snapshots in `extensions/browser/src/browser/pw-tools-core.snapshot.ts:23`. Those snapshots are then compacted into ref-addressable role trees in `extensions/browser/src/browser/pw-role-snapshot.ts:335`, and `mode=efficient` already provides the token-reduced path in `extensions/browser/src/browser/routes/agent.snapshot.plan.ts:37`. This is also documented for users as `openclaw browser snapshot` plus ref-based UI automation in `docs/cli/browser.md:122`. Separately, OpenClaw already ships A2UI as a semantic canvas protocol in `docs/platforms/mac/canvas.md:65`. Given that the requested direction is already present on main in shipped surfaces, there is not a separate actionable core item left to track here.
