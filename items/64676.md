## Summary

Keep open. Current main still has a gateway startup path that can regenerate the active agent `models.json`, and the merge planner still rewrites the file from config/catalog output while preserving only selected secret/base URL fields for matching providers. That leaves the reported user-pruned model list behavior unresolved unless maintainers decide `models.json` is generated cache rather than supported editable config.

## Best Possible Solution

Do not close yet. Maintainers should define the intended persistence contract for per-agent `models.json`: if user-pruned provider model arrays are supported, add a gateway-restart regression around `ensureOpenClawModelsJson`/`planOpenClawModelsJsonWithDeps` and preserve deletions through a tombstone or explicit replace mechanism. If `models.json` is generated cache, update docs, doctor, and CLI guidance so persistent edits go through `models.providers` or `models.mode: "replace"` rather than hand-editing generated agent files.
