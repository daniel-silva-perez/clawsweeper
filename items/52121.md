## Summary

Keep open. Current `main` still does not implement the PR's requested startup warning pass for semantic `tools.exec`/sandbox/elevated conflicts. Some adjacent behavior exists, such as gateway no-auth startup refusal and manual doctor/security/sandbox inspection surfaces, but those do not cover the startup feedback requested here. The PR is plausible but needs maintainer review because the discussion identifies remaining correctness risks around per-agent effective config, runtime bind overrides, and warning-only code becoming fatal.

## Best Possible Solution

Keep the PR open for maintainer review or replacement by a narrower implementation. The best path is a warning-only startup check that evaluates effective per-agent sandbox/tool/elevated settings, uses effective runtime bind/auth state for exposure checks, avoids throwing while collecting diagnostics, and keeps guidance aligned with current exec-security semantics.
