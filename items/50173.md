## Summary

Current `main` already fixes this. The memory-core plugin now registers `memory_search` and `memory_get` independently, so one tool failing to initialize no longer suppresses the other in new sessions, and the shipped changelog records that exact fix for `#50198`.

## Close Comment

Closing from a Codex review: this is already implemented on current `main` and present in the shipped `v2026.4.22` release. In `extensions/memory-core/index.ts:42` and `extensions/memory-core/index.ts:51`, `memory_search` and `memory_get` are now registered with separate `api.registerTool(...)` calls, so one tool returning `null` no longer suppresses the other in new sessions. `extensions/memory-core/src/tools.shared.ts:43` and `extensions/memory-core/src/tools.shared.ts:96` show each tool is created independently from config/session context, and `src/plugins/tools.ts:141` now logs null-returning tool factories instead of silently skipping them.
