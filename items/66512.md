## Summary

Keep open. The report is recent, concrete, non-maintainer-authored, and not protected by labels, but current main still does not implement systemd readiness notification. OpenClaw's Linux unit builder omits Type=notify/NotifyAccess/WatchdogSec, the gateway startup path only logs readiness after binding/post-attach work, and a targeted search found no NOTIFY_SOCKET, sd_notify, READY=1, or Type=notify support in the daemon/gateway/CLI paths. Existing restart health polling helps OpenClaw CLI lifecycle commands, but it does not make systemctl restart block on gateway RPC readiness or satisfy the requested systemd contract.

## Best Possible Solution

Keep this issue open for a focused core daemon/gateway implementation. The best path is to add a small systemd notification helper that detects NOTIFY_SOCKET, sends READY=1 only after the gateway has bound its WebSocket listener and completed the intended startup readiness boundary, updates the generated Linux unit to Type=notify with appropriate NotifyAccess, adds tests for the unit renderer and readiness hook behavior, and treats WatchdogSec/WATCHDOG=1 either as a follow-up or coordinates it with the existing open WatchdogSec issue.
