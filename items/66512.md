## Summary

Keep open. Current main still does not implement the requested systemd readiness notification contract: the generated Linux unit has no Type=notify/NotifyAccess/WatchdogSec directives, the gateway readiness boundary only updates internal readiness/probe behavior and logs the ready line, and the service restart path still completes after systemctl restart plus OpenClaw-side polling rather than making systemd block until sd_notify READY=1. The related WatchdogSec item in the provided context is relevant but narrower/adjacent, not a replacement for this READY=1 request.

## Best Possible Solution

Keep this issue open for a focused core daemon/gateway change: add a small systemd notification helper that detects NOTIFY_SOCKET, sends READY=1 only after the gateway listener and intended startup sidecars/readiness boundary are ready, update the generated Linux unit and docs to Type=notify with appropriate NotifyAccess, add unit tests for the renderer and notification hook, and coordinate WATCHDOG=1/WatchdogSec with the separate open watchdog-tracking item rather than burying it in this change.
