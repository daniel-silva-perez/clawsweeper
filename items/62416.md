## Summary

Keep open. Current `main` still has concrete screen-reader/accessibility gaps in the Control UI settings surface, so there is not strong evidence the issue is already fixed or obsolete.

## Evidence

- **Form fields are still rendered with visually adjacent labels rather than programmatic label bindings:** Text, number, select, and textarea fields render `<label class="cfg-field__label">...` as a sibling before the control, but the controls shown in the same render paths do not get matching `id`/`for` or `aria-labelledby` wiring. That leaves common settings inputs without an explicit accessible name source from the form renderer.
  - file: [ui/src/ui/views/config-form.node.ts:700](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/ui/src/ui/views/config-form.node.ts#L700)
  - command: `nl -ba ui/src/ui/views/config-form.node.ts | sed -n '698,858p'`
  - sha: [ecac696643c6](https://github.com/openclaw/openclaw/commit/ecac696643c604c6d854341bfe6424bfa31423e9)
- **Enum/choice controls still use plain button groups without selection semantics:** Small enums are rendered as `.cfg-segmented` button groups with an `active` CSS class, but the buttons do not expose radio/tab semantics such as `role`, `aria-checked`, or `aria-pressed`. A screen reader will not get reliable state information from these controls.
  - file: [ui/src/ui/views/config-form.node.ts:495](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/ui/src/ui/views/config-form.node.ts#L495)
  - command: `nl -ba ui/src/ui/views/config-form.node.ts | sed -n '491,590p'`
  - sha: [ecac696643c6](https://github.com/openclaw/openclaw/commit/ecac696643c604c6d854341bfe6424bfa31423e9)
- **Appearance selectors in Settings are also visual-only button groups:** Theme and roundness options are rendered as plain buttons whose current state is communicated through CSS classes like `settings-theme-card--active` and `active`, without corresponding ARIA state. That matches the report's complaint that the settings interface is difficult to navigate with assistive tech.
  - file: [ui/src/ui/views/config.ts:587](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/ui/src/ui/views/config.ts#L587)
  - command: `nl -ba ui/src/ui/views/config.ts | sed -n '580,632p'`
  - sha: [ecac696643c6](https://github.com/openclaw/openclaw/commit/ecac696643c604c6d854341bfe6424bfa31423e9)
- **Several icon/destructive controls still rely on `title` instead of explicit accessible labels:** Reset/remove controls in the config form use icon-only buttons with `title` attributes such as `Reset to default`, `Remove item`, and `Remove entry`, but they do not provide explicit `aria-label` text in those render paths.
