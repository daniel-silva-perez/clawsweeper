## Summary

Keep #39248 open. The report is concrete, the current main implementation still makes native `sessions_spawn` child sessions sandbox candidates under `agents.defaults.sandbox.mode: "non-main"`, and I found no source/test/history evidence proving the child initialization now progresses under that mode.

## Best Possible Solution

Keep the issue open until maintainers add a targeted regression test or live verification for native `sessions_spawn` under `agents.defaults.sandbox.mode: "non-main"`. The fix should prove the child run starts, persists transcript/message state, updates token/session state, and surfaces sandbox initialization or dispatch failures instead of silently timing out; if the repro still fails, patch the Docker-backed `resolveSandboxContext` or child dispatch path.
