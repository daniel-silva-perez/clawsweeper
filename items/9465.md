## Summary

Keep open. Current `main` does not implement cron job `pre`/`post`/`onError` hooks or shell/webhook hook execution. The cron job types, gateway schema, CLI surface, UI form state, docs, and execution path all still model only schedule/payload/delivery/failure-alert behavior, so the enhancement request remains unmet on `754acc4478fe361db1e5303fd60f589d52beea7a`.

## Evidence

- **Cron job type has no hook field:** `CronJob` is built from `CronJobBase` with `schedule`, `sessionTarget`, `wakeMode`, `payload`, `delivery`, and `failureAlert`; there is no `hooks` property or hook config union.
  - file: [src/cron/types.ts:112](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/src/cron/types.ts#L112)
