## Summary

Current main still omits MCP server tools from the /tools inventory, while the runtime path materializes and exposes bundled MCP tools. This PR targets a real gap, but it should stay open for a focused revision because it still mixes the inventory fix with unrelated plugin installer changes on a security-sensitive npm install path.

## Best Possible Solution

Keep this PR open, but request a focused revision or supersede it with a narrow PR for the linked /tools MCP inventory bug. The fix should drop the installer changes, add an MCP-aware inventory path that is lifecycle-safe, materialize tools with the same reserved-name rules as the run path, apply the same final bundled-tool policy, render a clear MCP servers group, and add compact plus verbose regression tests.
