## Summary

Keep PR #70876 open. Current main still appears to have the reported `/tools` inventory gap: runtime MCP tools are materialized and appended for agent runs, but `/tools` only renders `resolveEffectiveToolInventory()` output from `createOpenClawCodingTools()` and the inventory model has no MCP source/group. The PR is useful in direction, but it is not clean merge material as provided because it also carries unrelated plugin-install peer-dependency changes and had review discussion around those changes.

## Best Possible Solution

Keep the item open, but ask for a focused PR or maintainer cherry-pick: implement `/tools` MCP inventory against current main using the existing session runtime only, pass the real `sessionId`, preserve `reservedToolNames`, filter through `applyFinalEffectiveToolPolicy`, and add targeted `/tools` tests for MCP display, policy filtering, and collision-adjusted names. Split or drop the unrelated `install-package-dir` peer-dependency changes from this PR.
