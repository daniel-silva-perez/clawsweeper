## Summary

Keep open. Current `main` still does not support arbitrary `.jsonl` files in `memorySearch.extraPaths` for the builtin memory indexer. The code path only accepts `.md` plus configured multimodal media, docs still describe `extraPaths` as Markdown-only, and the existing JSONL support is limited to the separate session-transcript indexing path.

## Evidence

- **`extraPaths` filter still rejects generic JSONL files:** `packages/memory-host-sdk/src/host/internal.ts:91` allows only `.md` or multimodal-classified files, `packages/memory-host-sdk/src/host/internal.ts:143` hard-codes the root-memory helper to `.md`, and `packages/memory-host-sdk/src/host/internal.ts:182` only adds extra-path files when `isAllowedMemoryFilePath()` passes. There is no `.jsonl` branch here.
