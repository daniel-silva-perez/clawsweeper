## Summary

Keep open. Current main has a strong partial fix: default external direct-chat failures now use generic copy, non-direct group/channel failures are silenced, and Anthropic/Bedrock replay now strips invalid thinking blocks. But the issue’s central “never expose raw LLM/API errors to users” policy is still not true because `/verbose on|full` in external direct chats intentionally forwards sanitized provider/runner failure detail, and both tests and docs lock that behavior in.

## Best Possible Solution

Keep the issue open for an explicit maintainer decision on external-channel failure policy. The likely code path is to keep generic default direct-chat copy and silent group/channel behavior, but stop forwarding provider/runner exception detail to external chat users even under /verbose, reserving detailed errors for logs, Control UI, or operator-only/internal surfaces. Add regression coverage for WhatsApp and Telegram direct verbose failures. Treat the already-improved thinking-block replay repair as partial progress, and split multi-agent auto-escalation into a narrower follow-up if maintainers want it.
