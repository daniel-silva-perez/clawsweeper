## Summary

Current main does not support the report’s root-cause theory. Feishu is now loaded as a single bundled channel entry, and duplicate-id warnings are emitted from manifest discovery when the same plugin id is discovered from multiple install roots, not when a channel plugin shares an id with its owning entry.

## Close Comment

Closing from a Codex review. Current main does not reproduce the reported plugin-vs-channel duplicate path: Feishu is a single bundled channel entry in `extensions/feishu/index.ts:56`, and that entry’s contract only loads the channel plugin and calls `api.registerChannel(...)` during registration (`src/plugin-sdk/channel-entry-contract.ts:476`). The duplicate-id warning in current OpenClaw comes from manifest discovery when the same plugin id is discovered from multiple roots, and Doctor/config validation surfaces that under `plugins.entries.<id>` (`src/plugins/manifest-registry.ts:668`, `src/config/validation.ts:784`; documented in `docs/plugins/manifest.md:664`). That means a bundled Feishu plus a separately installed `~/.openclaw/extensions/feishu` copy is a real duplicate install, not a false positive caused by channel registration. If this still reproduces on current main with only one discoverable Feishu plugin copy, please open a new issue with the exact plugin roots, config, and fresh `openclaw doctor` output.
