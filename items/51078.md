## Summary

Keep open. This PR is a focused Android behavior fix, not a conservative auto-close candidate. Current main has not implemented the PR’s two central changes: the Screen tab still renders `Unit` on the first active composition until `screenTabStarted` flips, and initial chat load still reaches `ChatController.bootstrap(... refreshSessions = true)` with the default `fetchSessions(limit = 50)`. The PR also has useful review history: a maintainer requested the chat hunk be moved into the existing load/bootstrap path, the author followed up with that change, and a later Codex review flagged that the Screen-tab fallback still needs rework to avoid moving `ScreenTabScreen` between composition slots.

## Best Possible Solution

Keep the PR open for maintainer or author follow-up. The best path is to rebase/update it against current main, keep any chat session-limit change inside the existing `loadChat -> ChatController.load() -> bootstrap` path, and rework the Screen-tab first-visit fix so `ScreenTabScreen` is not mounted in one slot and immediately moved to the keep-alive slot. Add at least manual Android verification or focused tests before merge.
