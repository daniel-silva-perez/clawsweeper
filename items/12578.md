## Summary

Current main does not implement a config default for per-session usage footers. The agent defaults schema still has `thinkingDefault`/`verboseDefault` but no `usageDefault` or `responseUsageDefault`, `/usage` still only mutates the current session entry, and session creation only preserves an existing `responseUsage` value instead of sourcing one from config.
