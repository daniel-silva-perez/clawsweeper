## Summary

Current `main` does not implement this PR’s fix. `startAcpSpawnParentStreamRelay()` still relays only `assistant` and `lifecycle` events, so `system_event` payloads would still be dropped. The open PR also has an unresolved review note about counting `system_event` as progress for stall detection, so there is not strong evidence to close it conservatively.

## Evidence

- **Main still filters out non-lifecycle events after the assistant branch:** In `startAcpSpawnParentStreamRelay()`, after handling `assistant`, the code immediately returns for anything that is not `lifecycle`. There is no `event.stream === "system_event"` branch on current `main`. That means incoming `system_event` events are still ignored by the parent relay.
