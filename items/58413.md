## Summary

Keep open. Current main supports static provider/request/model headers and provider-specific attribution headers, but I found no implementation that automatically adds an agent id header to outgoing Anthropic/provider requests and no dynamic `{{agentId}}` header expansion. The issue is recent, externally authored, concrete, and plausible as either a core runtime enhancement or a provider/plugin SDK design discussion, so it should not be conservatively closed.
