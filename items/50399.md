## Summary

Keep PR #50399 open. Current main still lacks end-to-end Feishu duration propagation from `sendMediaFeishu` into `uploadFileFeishu`, so the central bug is not implemented. The PR is not landable as-is, but it is a focused, active Feishu bug-fix contribution with a recent maintainer review describing the smaller patch needed, so conservative cleanup should not auto-close it.

## Best Possible Solution

Rework this as a small Feishu-only patch on current main: keep `extractFeishuDownloadMetadata` and `resolveFeishuOutboundMediaKind`, derive audio/video duration safely after media is loaded or transcoded, pass it to `uploadFileFeishu` for proven Feishu `opus`/`mp4` uploads, and add focused tests that duration is included while MIME routing, MP3 fallback, and metadata preservation continue to work. Issue #53798 should remain the canonical bug tracker if maintainers decide to replace rather than revise this PR.
