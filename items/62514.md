## Summary

Keep open. Current `main` still has explicit mixed-source image handling gaps that match the report and the follow-up analysis, so there is not strong evidence this bug is fixed well enough to close conservatively.

## Evidence

- **Prompt refs are still appended after attachment images:** `mergePromptAttachmentImages` preserves inline/offloaded attachment order first, then unconditionally appends `promptRefImages`. Mixed turns are still not assembled into one fully ordered image set before the model call.
  - file: [src/agents/pi-embedded-runner/run/images.ts:100](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/src/agents/pi-embedded-runner/run/images.ts#L100)
  - command: `nl -ba src/agents/pi-embedded-runner/run/images.ts | sed -n '96,148p'`
  - sha: [ecac696643c6](https://github.com/openclaw/openclaw/commit/ecac696643c604c6d854341bfe6424bfa31423e9)
- **Remote image URLs are still excluded from native image injection:** `detectImageReferences` explicitly returns early for `http://` and `https://` values, and the file comments that remote HTTP(S) URLs are intentionally ignored. That leaves uploaded-image-plus-URL cases dependent on later tool-calling behavior.
  - file: [src/agents/pi-embedded-runner/run/images.ts:240](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/src/agents/pi-embedded-runner/run/images.ts#L240)

  - sha: [ecac696643c6](https://github.com/openclaw/openclaw/commit/ecac696643c604c6d854341bfe6424bfa31423e9)
- **Current code still documents the limitation as unfinished:** `chat-attachments.ts` contains a live `Known ordering limitation` note saying mixed large/small batches can reach the model in a different order and that a future refactor should unify all image references into a single ordered list.
  - file: [src/gateway/chat-attachments.ts:284](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/src/gateway/chat-attachments.ts#L284)

That is positive evidence the problem is still being worked rather than already resolved on `main`.
