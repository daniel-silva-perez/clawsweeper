## Summary

Current `main` already fixes the non-loopback Control UI startup failure that this PR targets, using explicit `allowedOrigins` seeding instead of the PR's dangerous Host-header fallback. The remaining script-side Windows symlink handling is also already implemented on `main`, and the current Docker image preserves the non-root runtime plus `HEALTHCHECK` that this PR would regress.

Closing from a Codex review because current `main` already addresses the startup problem this PR is trying to solve. OpenClaw now seeds `gateway.controlUi.allowedOrigins` before gateway runtime validation for non-loopback/container starts (`src/gateway/server.impl.ts:321`, `src/config/gateway-control-ui-origins.ts:46`), and that shipped in `v2026.2.27` (`CHANGELOG.md:3659`). Given that, there is no remaining mainline change from this PR to merge.
