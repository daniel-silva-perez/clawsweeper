## Summary

Keep this PR open. Current main still requires full contiguous query-string matches in the memory-wiki digest gate and full-page scoring path, so the token-overlap behavior proposed here is not implemented or superseded. The linked bug was closed as superseded by this PR, while the earlier same-scope PR is closed unmerged, leaving this PR as the active implementation candidate.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a refined memory-wiki fix that adds normalized token-overlap matching across digest and full-scan paths with focused regression tests, or ask the author to address the candidate-broadening and minimum-threshold concerns before merge.
