## Summary

Current main still appears to have the reported TUI behavior: failure details are written to the chat log, but the status bar receives only bare `error`, and error status is rendered as a non-busy state with no elapsed timer. The PR should not be closed as obsolete, but its current branch needs rework because it includes broader stale-run/gap-recovery behavior and an unrelated AGENTS.md change that drew multiple review concerns.

## Best Possible Solution

Keep the item open, but ask for a narrower patch: thread a sanitized single-line error summary from chat/lifecycle error events into the TUI status bar, preserve the failed run's elapsed time across status re-renders, add focused event-handler/status-render regression coverage, and drop unrelated AGENTS.md, stale-run timeout, and gap-recovery changes unless they are split into a separate maintainer-reviewed PR.
