## Summary

Keep open. Current main does not implement `publicMode`, prompt redaction, or the PR's per-agent public ingress behavior. Main has adjacent scope-based owner handling for OpenAI-compatible HTTP endpoints, but docs still describe those endpoints as operator/full-access surfaces and the system prompt still emits workspace/runtime details.

## Best Possible Solution

Keep the PR open, but require it to rebase onto current main and align with the existing scoped HTTP auth model. The best path is to implement `agents.defaults.publicMode` and `agents.list[].publicMode` through config types, Zod schemas, schema help/labels, effective routed-session resolution, prompt/runtime/docs/skills redaction, and OpenAI/OpenResponses tests, while addressing the already-raised default-only and streamed responses edge cases.
