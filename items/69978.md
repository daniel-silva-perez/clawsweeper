## Summary

Keep #69978 open. Current main still has the hardcoded TUI streaming watchdog path, no user-facing `/watchdog` command or persisted setting, and no per-run warning dedupe; the overlapping #68596 only tracks timeout configurability and does not fully cover the duplicate-warning part of this issue.

## Best Possible Solution

Keep this issue open or split it explicitly: implement a TUI-owned watchdog preference/command surface, likely coordinated with #68596 for timeout configuration, and add a focused per-run warning-dedupe guard in `src/tui/tui-event-handlers.ts` with tests covering repeated deltas after a watchdog fire for the same `runId`.
