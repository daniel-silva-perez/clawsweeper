## Summary

Current main still does not implement the requested generic user-visible feedback for messages queued behind an active session run. The active-run enqueue-followup branch enqueues and returns without a callback, GetReplyOptions has no onQueued-style extension point, and Feishu/Lark typing still hangs off onReplyStart, which only fires when processing actually starts. PR #39676 proposed the exact hook but was closed unmerged and its head commit is not present in this checkout.

## Best Possible Solution

Keep this issue open. The best fix is a generic non-blocking queued-message feedback seam on the active-run enqueue-followup path, likely an onQueued callback on GetReplyOptions/replyOptions, with channel wiring for Feishu/Lark typing or queue reactions and preservation of Telegram/Discord status reactions. Add focused tests proving the callback fires when a message is queued behind an active run, failures do not block enqueueing, and normal run-start typing remains unchanged.
