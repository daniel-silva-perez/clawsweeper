## Summary

Keep open. Current main does not contain this PR's core changes: isolated cron delivery still relies on loaded-channel explicit target parsing, Telegram target parsing still lacks the slash topic form, and cron run results still do not carry a delivery-specific error field. The PR also has unresolved review feedback about the shorthand `chatId:topicId` pre-parser gap, so it is not safely obsolete or closeable as implemented.

## Best Possible Solution

Keep the PR open but request an update: rebase onto current main, add a complete isolated pre-parser or a config-backed plugin parsing path that covers `chatId:topicId`, `chatId:topic:topicId`, and slash targets, retain slash support in the Telegram parser if still desired, and add focused cron delivery tests plus delivery-error persistence coverage.
