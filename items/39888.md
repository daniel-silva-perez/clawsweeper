## Summary

Keep open. Current main still emits `model.usage` diagnostics from the reply-runner path, but the CLI/ingress `agentCommandInternal` path still only updates session state and persists/delivers the result without emitting the diagnostic event this PR is trying to add. The PR itself is stale because `src/commands/agent.ts` is now only a re-export and the implementation lives in `src/agents/agent-command.ts`, but the underlying observability gap remains real.

## Best Possible Solution

Refresh the PR or replace it with a small main-targeted change in `src/agents/agent-command.ts` that emits a trusted `model.usage` diagnostic after the CLI/ingress run has updated session state. The implementation should reuse the current context-token resolution and cost logic, include `agentId` and an appropriate CLI/ingress channel label, preserve reply-runner behavior, and add a focused regression test for CLI agent diagnostic emission.
