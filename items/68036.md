## Summary

Keep open. Current main partially supersedes the largest profiled hot path by making dreaming/cron transcript classification parse `sessions.json` directly, but it does not implement this PR's `readonly` session-store cache option and `readSessionUpdatedAt` still clones through `loadSessionStore`. The PR still contains potentially useful remaining performance work and is not clearly duplicated or fully implemented on main.

## Best Possible Solution

Keep the PR open, but ask for a rebase against current main and narrow the patch to the remaining useful session-store read-only path, especially `readSessionUpdatedAt` and any generic `loadSessionStore` cache API change maintainers still want. The best implementation should add readonly overloads or `Readonly<Record<string, SessionEntry>>` return types plus focused cache hit/miss regression tests, while preserving the direct transcript-classification path already present on main.
