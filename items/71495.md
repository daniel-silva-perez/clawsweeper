## Summary

Keep open. The issue is recent, unprotected, and has concrete follow-up evidence that v2026.4.24 still returns an empty `/subagents list` while subagent runs complete. Current `main` still uses the same command-to-registry read path, and the only relevant post-v2026.4.24 change I found is deterministic sorting in `subagent-list.ts`, not a registry writer/reader fix.

## Best Possible Solution

Keep the issue open. Add a targeted regression test or QA scenario that spawns a subagent from a real requester session, immediately checks `listControlledSubagentRuns` and `/subagents list`, then verifies parent completion delivery. Use that to isolate whether the failure is module-instance split, persisted snapshot divergence, controller/requester key mismatch, or command context wiring.
