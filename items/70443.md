## Summary

Keep #70443 open. Current main still has no `--offline` or equivalent Docker setup mode: `scripts/docker/setup.sh` always builds `openclaw:local` or pulls a non-local `OPENCLAW_IMAGE` before continuing to permissions, onboarding, config sync, and compose startup. The issue is recent, concrete, core-installer scoped, and not superseded by the related cross-platform `OPENCLAW_DOCKER_PLATFORM` thread.

## Best Possible Solution

Add a narrow `--offline` flag to `scripts/docker/setup.sh`, parse it before setup side effects, and when enabled replace the main build/pull step with `docker image inspect "$IMAGE_NAME"` plus a clear `docker load` error if absent. For sandbox mode, inspect `openclaw-sandbox:bookworm-slim` and warn rather than fail if missing. Keep all later env upsert, permissions, onboarding, gateway config sync, compose startup, and sandbox config behavior unchanged. Update `src/docker-setup.e2e.test.ts` to pass argv and cover the offline paths, and document the flag in `docs/install/docker.md`.
