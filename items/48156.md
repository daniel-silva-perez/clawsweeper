## Summary

Keep open. Current `main` does not ship the proposed Context Book bootstrap pipeline, so there is no strong basis to close this PR as already implemented or obsolete. The work may need reshaping around existing hook/plugin seams, but the feature itself is still unresolved on `main`.

## Evidence

- **No Context Book implementation on current main:** A repo search on `main` found no `context-books`, `CONTEXT_BOOK`, or `alwaysActive` hits in the tracked source/docs/tests for this feature, and `src/agents` has no `context-books.ts` file.
  - command: `rg -n "context-books|CONTEXT_BOOK|Context Book|alwaysActive" src docs test packages ui -g '!node_modules' ; find src/agents -maxdepth 1 -type f | sort | grep 'context-books'`
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **Bootstrap types still only model built-in filenames:** `WorkspaceBootstrapFileName` is still restricted to the built-in bootstrap files, and `VALID_BOOTSTRAP_NAMES` still contains only `AGENTS.md`, `SOUL.md`, `TOOLS.md`, `IDENTITY.md`, `USER.md`, `HEARTBEAT.md`, `BOOTSTRAP.md`, and `MEMORY.md`. There is no shipped `CONTEXT_BOOK:*` support on `main`.
  - file: [src/agents/workspace.ts:136](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/src/agents/workspace.ts#L136)
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **Bootstrap loading still reads only the fixed workspace files:** `loadWorkspaceBootstrapFiles()` still enumerates only the fixed built-in bootstrap files, and subagent/cron filtering still works from that fixed set. Current `main` does not load `context-books/*.yaml|yml|json` as described by the PR.
  - file: [src/agents/workspace.ts:477](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/src/agents/workspace.ts#L477)

- **Current extension points exist, but they are not an equivalent shipped implementation:** `resolveBootstrapFilesForRun()` still loads the normal workspace bootstrap set and then applies `agent:bootstrap` hooks. The docs also explicitly advertise `agent:bootstrap` and `before_prompt_build` as the supported ways to add or mutate prompt/bootstrap context, which means there is an extension path today but not the specific core feature this PR adds.
  - file: [src/agents/bootstrap-files.ts:227](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/src/agents/bootstrap-files.ts#L227)
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **Submitted PR still has unresolved functional review concerns:** The PR discussion already identifies behavior gaps in the proposed patch: the validated `position` field is discarded, and appending always-on Context Book files after existing bootstrap files can cause them to be trimmed by bootstrap budget ordering. That argues for revision, not closure as obsolete.
  - sha: [dc86100fc547](https://github.com/openclaw/openclaw/commit/dc86100fc54708ad2a9d65f50f749f3db1b28306)

## Risks / Open Questions

- The feature may belong in a plugin or hook-pack instead of core under `VISION.md`'s 'core stays lean' guidance, but current `main` does not provide an equivalent shipped solution, so closing on scope alone would be premature.
- If maintainers want to pursue the PR, it likely needs rework around prompt-budget ordering and the currently no-op `position` field before it is ready to merge.
