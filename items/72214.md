## Summary

Keep #72214 open. The report is fresh, detailed, and against v2026.4.24, and current main still shows only partial Telegram mitigations rather than source/test proof that gateway health/RPC stays responsive when Node/undici hangs on api.telegram.org while curl succeeds.

## Best Possible Solution

Keep the issue open and fix it in the Telegram channel lifecycle/transport path: add bounded startup/menu/polling setup timeouts, ensure Telegram network failures only degrade channel account health rather than gateway RPC/health responsiveness, expose the active Telegram transport/fallback state in channel status diagnostics, and add a regression test around a hanging Telegram fetch or undici dispatcher.
