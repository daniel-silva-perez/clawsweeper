## Summary

Keep this PR open, but require a revision. Current main still lacks Vertex ADC marker stripping and still drops empty final payloads without a friendly error, so the work is not obsolete. The submitted patch is not safe as-is because the PR discussion identifies two real P1 risks: replacing the existing Google thinking stream wrapper, and treating valid already-delivered empty-final-payload paths as failures.

## Best Possible Solution

Revise this PR rather than close it. The Google change should compose a google-vertex ADC marker-stripping wrapper with the existing Google thinking wrapper, with regression tests for both Vertex ADC and non-Vertex thinking behavior. The empty-payload change should only surface a friendly retry/context error when a non-silent run truly produced no delivered output and no renderable final payload, while preserving successful block streaming, direct block-key dedupe, same-target messaging-tool suppression, and silentExpected turns.
