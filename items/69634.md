## Summary

Keep #69634 open. Current main still defines the `.chat-bubble.has-copy` padding and renders the floating message-level copy action, but the rendered bubble class list does not apply `has-copy`, so the overlap fix proposed by this PR is not implemented on main. Related duplicate PRs in the PRtags group are closed unmerged, and linked bug #69605 remains open in the provided context.

## Best Possible Solution

Keep the PR open and either merge an adapted version or ask the contributor to rebase: apply `has-copy` when assistant message actions render, preserve the current `chat-bubble--tool-shell` class behavior, and add a regression test in `ui/src/ui/chat/grouped-render.test.ts` that assistant bubbles with copy/expand actions get `has-copy` while bubbles without actions do not. Then close #69605 when the fix lands.
