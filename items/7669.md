## Summary

Keep #7669 open. Current main still lacks a supported npm `dev` dist-tag publishing path: the public npm release workflow only offers `beta` and `latest`, the publish helper maps every non-`latest` request to `beta`, release-plan types and tests model only stable/beta with latest/beta tags, docs still describe npm dev as optional, and the live npm registry currently has no `openclaw@dev` dist-tag.

## Best Possible Solution

Keep #7669 open until release maintainers either implement a supported main-snapshot `openclaw@dev` publish path with workflow/input or trigger support, release-plan validation, publish-helper preservation of the `dev` tag, postpublish verification, and docs; or explicitly retire npm dev snapshots and update the release-channel/update/release docs to make dev git-only.
