## Summary

Keep open. Current `main` does not implement an internal `session:end` hook event. OpenClaw now has plugin-level `session_end` hooks, but the internal HOOK.md hook surface still lacks `session:end`, and the existing plugin event is wired for session lifecycle rollovers/deletes rather than the requested internal completion event with completion metadata.

## Evidence

- **Internal hook docs do not expose session:end:** The internal hook event table lists `command:*`, `session:compact:*`, `session:patch`, `agent:bootstrap`, `gateway:startup`, and `message:*`, but no `session:start` or `session:end`.
  - file: [docs/automation/hooks.md:36](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/docs/automation/hooks.md#L36)
  - command: `nl -ba docs/automation/hooks.md | sed -n '34,55p'`
  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
- **Internal hook runtime only dispatches session compaction and patch events:** Local search found `createInternalHookEvent("session", ...)` only for `compact:before` and `compact:after`, while session mutation code emits `action: "patch"`. There is no dispatch site for `action: "end"`.
  - file: [src/agents/pi-embedded-runner/compaction-hooks.ts:185](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/src/agents/pi-embedded-runner/compaction-hooks.ts#L185)

  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
- **Plugin hook exists, but it is not the internal hook requested:** Docs and types show `session_start` / `session_end` under plugin hooks, not internal hooks. The event shape is `sessionId`, `sessionKey`, `messageCount`, `durationMs`, `reason`, `sessionFile`, and next-session fields; it does not match the issue's internal-hook payload with `stopReason`, `lastMessage`, and arbitrary start metadata.
  - file: [docs/concepts/agent-loop.md:91](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/docs/concepts/agent-loop.md#L91)

Tests cover reasons like `new`, `reset`, `daily`, `idle`, and `deleted`, which is different from "fires when a session completes" for workflow orchestration.
