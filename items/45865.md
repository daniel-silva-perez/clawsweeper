## Summary

Current `main` does not expose a plugin SDK `api.runtime.llm()` surface. The nearest existing helper is `api.runtime.agent.runEmbeddedAgent(...)`, which starts a full embedded agent turn rather than a narrow inference API, so the feature request remains unresolved.

## Evidence

- **Plugin runtime type has no `llm` namespace:** `PluginRuntimeCore` lists `agent`, `tts`, `mediaUnderstanding`, `imageGeneration`, `videoGeneration`, `musicGeneration`, `webSearch`, `stt`, `events`, `logging`, `state`, `tasks`, `taskFlow`, and `modelAuth`, but no `llm` helper.

## Risks / Open Questions

- Closing this as implemented would hide a real SDK gap: plugins still do not have a narrow provider-agnostic inference call on `api.runtime`.
- `api.runtime.agent.runEmbeddedAgent(...)` is only a partial workaround and changes the abstraction level from direct inference to a full agent turn, so it does not clearly satisfy the request.
