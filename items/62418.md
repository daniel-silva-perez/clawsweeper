## Summary

Current `main` does not have strong evidence for a full fix. The WebChat-side history filter only removes exact silent replies and one synthetic transcript-repair tool result, while gateway history sanitization is assistant-only and does not strip generated `System:`/`System (untrusted):` wrapper user messages or `/new`/`/reset` startup prompts. A nearby subsystem (`memory-host-sdk`) added explicit filters for those generated wrapper/exec-completion messages, which suggests this class of leakage is still recognized elsewhere rather than solved generically in WebChat.

## Evidence

There is no filter here for exec completion notifications, `System:` wrappers, or `/new`/`/reset` startup prompts.
