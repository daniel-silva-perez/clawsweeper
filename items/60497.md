## Summary

Keep open. Current main still suppresses ordinary non-timeout exec failures at default verbosity and can preserve assistant text after the failed tool result, so the reported user-visible failure mode is not resolved. The related open PR #60507 is a companion destructive-command HIL catalog, not a merged implementation of fail-closed post-exec behavior.

## Best Possible Solution

Keep this issue open and fix the embedded agent loop/reply policy for exec-like mutating tool failures. The likely change belongs around `src/agents/pi-embedded-runner/run/payloads.ts`: surface the exact failed exec result for default local runs, suppress or override success-looking assistant text when the matching exec action failed, and add regression coverage for exit 127/`python: command not found` in local embedded mode. The NemoClaw sandbox Python symlink workaround can remain upstream; OpenClaw core still needs fail-closed user-facing behavior after failed exec calls.
