## Summary

Keep open. Current `main` already has the external-secrets baseline (`SecretRef`, provider config, reload/audit/configure/apply flows), but it still only supports native secret sources `env`, `file`, and `exec`. The issue explicitly asks for first-class integrations like AWS Secrets Manager, Vault, Azure Key Vault, and GCP Secret Manager, and those are not implemented as distinct built-in providers on `main`.

## Evidence

- **Current source types are limited to env/file/exec:** `SecretRefSource` on current `main` is a closed union of only `"env" | "file" | "exec"`; there is no built-in `aws`, `vault`, `azure`, or `gcp` source type.
  - file: [src/config/types.secrets.ts:3](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/src/config/types.secrets.ts#L3)

  - file: [docs/gateway/secrets.md:84](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/docs/gateway/secrets.md#L84)
