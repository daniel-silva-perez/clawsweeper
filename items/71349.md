## Summary

Keep open. The issue is recent, detailed, and current `main` still leaves a plausible memory-lancedb auto-capture/product-contract gap: auto-capture only stores user messages that match narrow heuristics, agent-end persistence is dispatched fire-and-forget, and the adjacent doctor warning path still depends on the active memory-runtime backend rather than the selected `memory-lancedb` slot.

## Best Possible Solution

Keep the issue open for a focused maintainer fix or clarification: either document/log that memory-lancedb autoCapture is heuristic and not per-turn, or change the capture path to match the advertised expectation; also align `openclaw doctor` with the selected memory slot or have memory-lancedb expose the runtime/status contract needed by doctor.

## Evidence
