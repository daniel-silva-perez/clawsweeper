## Summary

Keep open. Current `main` at `40be5ad58187fd20b8defe5fc30e649190c131a8` does not contain the PR‚Äôs UI-side attachment buffering/pending-read handling or the gateway-side webchat media staging. The underlying image-loss path is still plausible on `main`, and the gateway still only injects `MediaPath`/`MediaPaths` for plugin-bound sessions, not general webchat sends.
