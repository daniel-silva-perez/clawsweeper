## Summary

Keep open. Current main still lacks Feishu inbound sibling mention arbitration, and this PR remains a focused attempt at a real multi-account group-routing gap. It should not be closed as implemented, duplicate, or stale cleanup; it needs a corrected implementation before merge.

## Best Possible Solution

Keep this PR open or replace it with a corrected Feishu implementation. The maintainer path should derive the coordinator from Feishu default-account selection, arbitrate on structured mentioned bot open_ids before per-account dispatch, avoid substring-only implicit mention matching, and add regression coverage for specialist-only, coordinator-plus-specialist, and multi-specialist group mentions.
