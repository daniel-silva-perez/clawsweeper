## Summary

Keep #71619 open. Current main still treats Linux version-manager directories as part of the daemon minimal service PATH, and the current tests still lock in that older behavior. The PR remains a focused fix for the #71578 Linux doctor/minimal-PATH contradiction rather than obsolete work.

## Best Possible Solution

Keep #71619 open for maintainer review or land an equivalent patch: make Linux minimal service PATH include stable user bins, pnpm/NPM prefix, Nix profiles, and system dirs while preserving macOS version-manager behavior; update service-env and service-audit regression tests, and align any user-facing docs that still describe Linux version-manager PATH inclusion.
