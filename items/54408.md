## Summary

Keep open. Current main still runs pre-compaction memory flush as a normal embedded-agent prompt on the active session, and repo-local code/tests do not show a fix that keeps that user-side flush prompt out of the main transcript. The existing protections only suppress silent assistant output and ignore assistant-side NO_REPLY rows in visible history; they do not sanitize or exclude leaked user flush prompts from history or compaction input.

## Evidence

- **Memory flush still reuses the active session and only marks the run silent:** `runMemoryFlushIfNeeded()` builds the flush run from `params.followupRun.run` and calls `runEmbeddedPiAgent()` with `silentExpected: true`, `trigger: "memory"`, and the flush `prompt`. That makes the flush a normal run on the current session rather than a separate hidden transcript lane.
