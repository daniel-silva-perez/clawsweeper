## Summary

Current `main` still appears to match the reported bug: `memoryFlush` is only invoked in the pre-turn path, while in-turn auto-compaction is handled later by compaction event accounting and post-compaction side effects that do not write durable `memory/*.md` notes.

## Evidence

- **Pre-turn-only memory flush callsite:** `runReplyAgent()` runs `runPreflightCompactionIfNeeded(...)` and then `runMemoryFlushIfNeeded(...)` before the main turn proceeds.
  - file: [src/auto-reply/reply/agent-runner.ts:1104](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/src/auto-reply/reply/agent-runner.ts#L1104)
  - command: `nl -ba src/auto-reply/reply/agent-runner.ts | sed -n '1098,1138p'`
  - sha: [ecac696643c6](https://github.com/openclaw/openclaw/commit/ecac696643c604c6d854341bfe6424bfa31423e9)
- **In-turn compaction counted from runtime events:** During execution, `evt.stream === "compaction"` increments the attempt compaction counter on completed in-turn compaction events.
  - file: [src/auto-reply/reply/agent-runner-execution.ts:1247](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/src/auto-reply/reply/agent-runner-execution.ts#L1247)
  - command: `nl -ba src/auto-reply/reply/agent-runner-execution.ts | sed -n '1247,1266p'`
  - sha: [ecac696643c6](https://github.com/openclaw/openclaw/commit/ecac696643c604c6d854341bfe6424bfa31423e9)
- **Post-turn auto-compaction handling has no memory flush retry:** After `autoCompactionCount > 0`, the code updates compaction counts, refreshes queued followups, and injects post-compaction context, but does not call `runMemoryFlushIfNeeded()` or any equivalent durable memory-file write path.
  - file: [src/auto-reply/reply/agent-runner.ts:1551](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/src/auto-reply/reply/agent-runner.ts#L1551)
  - command: `nl -ba src/auto-reply/reply/agent-runner.ts | sed -n '1551,1597p'`
  - sha: [ecac696643c6](https://github.com/openclaw/openclaw/commit/ecac696643c604c6d854341bfe6424bfa31423e9)
- **Post-compaction side effects are index sync only:** `runPostCompactionSideEffects()` emits a transcript update and runs post-compaction session-memory index sync; it does not perform the agentic `memoryFlush` write used for `memory/YYYY-MM-DD.md` notes.
  - file: [src/agents/pi-embedded-runner/compaction-hooks.ts:82](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/src/agents/pi-embedded-runner/compaction-hooks.ts#L82)

  - sha: [ecac696643c6](https://github.com/openclaw/openclaw/commit/ecac696643c604c6d854341bfe6424bfa31423e9)
- **Current contract still defines memoryFlush as pre-compaction:** The config types and reference docs still describe `agents.defaults.compaction.memoryFlush` as a pre-compaction/pre-threshold flush, which does not resolve the reported in-turn bypass path.
  - file: [src/config/types.agent-defaults.ts:448](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/src/config/types.agent-defaults.ts#L448)
