## Summary

This request is better handled as a plugin/hook distributed through ClawHub rather than by adding a new core `rules.inject` config surface. Current main already provides a per-agent prompt-injection seam that can prepend or append stable system-context text before prompt submission.

## Evidence

- **Vision keeps optional capability out of core:** `VISION.md` says OpenClaw has an extensive plugin API, core should stay lean, and optional capability should usually ship as plugins.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/VISION.md#L54)
  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
- **Documented hook for prompt injection:** `before_prompt_build` is explicitly documented as the hook for injecting `prependContext`, `systemPrompt`, `prependSystemContext`, or `appendSystemContext` before prompt submission.
  - file: [docs/concepts/agent-loop.md:93](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/docs/concepts/agent-loop.md#L93)
  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
- **Hook API supports stable system-prompt rules:** The hook result type already supports `prependSystemContext` and `appendSystemContext`, described as static plugin guidance that lives in system-prompt space.
  - file: [src/plugins/hook-before-agent-start.types.ts:28](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/src/plugins/hook-before-agent-start.types.ts#L28)
  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
- **Hook context is agent-aware:** Plugin hook context includes `agentId`, so a plugin can apply different injected rules per agent.
  - file: [src/plugins/hook-types.ts:154](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/src/plugins/hook-types.ts#L154)
  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
- **Runtime already applies injected system context:** The agent runner resolves prompt-build hook results, passes `agentId` into the hook context, and applies `prependSystemContext` / `appendSystemContext` to the final system prompt before submission.
  - file: [src/agents/pi-embedded-runner/run/attempt.ts:1931](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/src/agents/pi-embedded-runner/run/attempt.ts#L1931)
  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)

## Risks / Open Questions

- I did not identify an already-published ClawHub plugin that implements this exact rule-injection pattern; packaging still needs to happen outside core.

## Close Comment

Closing as `clawhub` after Codex review. Current `main` already exposes a plugin/hook path for this exact kind of behavior: `before_prompt_build` can inject `prependSystemContext` / `appendSystemContext` into system-prompt space, the hook context includes `agentId` for per-agent behavior, and the runner applies those additions before prompt submission (`docs/concepts/agent-loop.md:93`, `src/plugins/hook-before-agent-start.types.ts:28`, `src/plugins/hook-types.ts:154`, `src/agents/pi-embedded-runner/run/attempt.ts:1931`). `VISION.md:54` also sets the scope rule that optional capability should usually ship as plugins rather than growing core config surface. Given that, a ClawHub plugin/hook is the right home for persistent rules injection, not a new core `agents.defaults.rules` feature.
