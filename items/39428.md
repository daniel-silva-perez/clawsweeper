## Summary

Keep #39428 open. Current main still has no Discord accountId-to-agentId implicit routing tier: Discord inbound preflight passes the accountId into the generic route resolver, the resolver evaluates explicit binding tiers, then falls through to the configured default agent. The issue discussion also points to PR #39485 as the direct fix, but that PR is open/unmerged in the provided context, so the user-visible bug is not implemented on main.

## Best Possible Solution

Keep #39428 open until PR #39485 or an equivalent maintainer-approved change lands on main. The likely implementation belongs after explicit binding tiers and before default-agent fallback, should preserve the literal default account behavior, and should add regression coverage for Discord accountId matching plus the default-account guard.
