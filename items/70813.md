## Summary

Keep #70813 open. Current main still does not support WhatsApp `edit` or `unsend`: discovery advertises only `react`/`poll`, execution supports only `react`, and the active WhatsApp send API has no edit/unsend methods. The PR is a focused bundled WhatsApp plugin implementation with relevant tests and recent author follow-up, so this is not implemented, superseded, or a ClawHub-only cleanup candidate.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a focused WhatsApp bundled plugin change, or request changes on that PR, that adds `edit` and `unsend` to WhatsApp action discovery, gateway dispatch, the active Web listener/send API, and targeted tests while preserving own-message keys, outbound target authorization, and send-message gating.
