## Summary

Keep this PR open. Current main at e962381dbf1ecdb1cd4784fdeb6c2e07ddd3435b still does not implement WhatsApp edit or unsend actions: discovery advertises only react/poll, the WhatsApp action adapter supports only react, runtime rejects non-react actions, the active Web send API has no edit/unsend methods, and docs list only reaction/poll support. The PR remains a plausible WhatsApp-plugin-owned implementation candidate, with a bounded security-sensitive capability that still needs maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or revise a WhatsApp-plugin-owned implementation that advertises edit/unsend only when message sends are enabled, dispatches both actions through the existing gateway action surface, reuses outbound target authorization, constructs own-message mutation keys with fromMe: true and no caller-controlled participant, adds focused unit/seam tests, and updates the public WhatsApp and message CLI docs.
