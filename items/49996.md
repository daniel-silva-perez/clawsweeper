## Summary

Keep open. Current main still drops the tail of an unclosed reasoning tag in strict mode, and Telegram still uses that strict stripping to derive answer text. The PR is not mergeable as-is because it carries large unrelated churn, but the underlying bug is not implemented or safely superseded on main.

## Best Possible Solution

Keep the item open, but ask for a narrow replacement patch: change only src/shared/text/reasoning-tags.ts and its tests, preserve strict fail-closed behavior when visible content was already captured before an unclosed tag, recover the zero-capture answer-loss cases, include the whitespace-prefix safety case from review, and verify the Telegram split/dispatch path. If maintainers prefer the narrower #51915 thread as canonical, close this PR only after explicitly linking that PR as the replacement.
