## Summary

Current main still emits user-visible warning payloads for failed mutating tools such as `edit`, including when the assistant already produced a normal reply. The prior ClawSweeper keep-open comment remains consistent with current main, and the related fix PRs in the provided context were closed unmerged.

## Best Possible Solution

Keep the issue open until the payload policy is changed deliberately. The likely fix belongs in the agent reply payload policy: suppress or normalize mutating-tool warning payloads when the assistant already produced a user-facing explanation, while preserving a fallback warning when there is no assistant reply and keeping details in logs/debug surfaces. Update the existing payload tests and Telegram dispatch tests to encode that behavior.
