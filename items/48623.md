## Summary

Keep open. Current `main` still intentionally ignores `lastGood` when ordering auth profiles, and the slug generator still uses the agent primary model with only a timeout override. The timeout-budget subclaim appears outdated on current `main`: each embedded attempt gets a fresh `timeoutMs` abort timer.

## Evidence

- **`lastGood` still not used for ordering:** `resolveAuthProfileOrder()` explicitly documents and implements round-robin by `lastUsed`, with the comment `lastGood is NOT prioritized - that would defeat round-robin`.
  - file: [src/agents/auth-profiles/order.ts:155](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/agents/auth-profiles/order.ts#L155)
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Behavior is locked by a test:** There is a dedicated test asserting `does not prioritize lastGood over round-robin ordering`, expecting `anthropic:default` ahead of the `lastGood` profile.
  - file: [src/agents/auth-profiles.resolve-auth-profile-order.does-not-prioritize-lastgood-round-robin-ordering.test.ts:36](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/agents/auth-profiles.resolve-auth-profile-order.does-not-prioritize-lastgood-round-robin-ordering.test.ts#L36)
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Selection paths still rely on that ordering:** Model auth resolution calls `resolveAuthProfileOrder()` directly when choosing provider credentials, so the non-`lastGood` ordering remains live behavior, not dead code.
  - file: [src/agents/model-auth.ts:497](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/agents/model-auth.ts#L497)
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Timeout budget is per attempt on current main:** Inside the retry loop, each call to `runEmbeddedAttemptWithBackend()` is passed `timeoutMs: params.timeoutMs`, and the attempt code creates a new abort timer with `scheduleAbortTimer(params.timeoutMs, "initial")` for that attempt.
  - file: [src/agents/pi-embedded-runner/run.ts:795](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/agents/pi-embedded-runner/run.ts#L795)
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **No per-task slug model override exists:** The slug helper resolves the agent primary model via `resolveAgentEffectiveModelPrimary()` and forwards only a timeout override into `runEmbeddedPiAgent()`. There is no `slugModel` or `embeddedModels.slugGenerator` config path in the checked files.
  - file: [src/hooks/llm-slug-generator.ts:59](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/hooks/llm-slug-generator.ts#L59)
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
