## Summary

Keep #71350 open. Current main implements the first P0 via `agents.list[].thinkingDefault` and provides flat per-agent stream params through `agents.list[].params`, but the broader request is not fully shipped: there is no per-agent `models` map, no `thinkingStrategy` schema, and no rules/escalation adaptive thinking implementation.

## Best Possible Solution

Keep the issue open, but narrow it. Treat `agents.list[].thinkingDefault` and flat `agents.list[].params` as already implemented, decide whether a per-agent `models` map is still needed, and track rules/escalation adaptive thinking through PR #44126 or a focused follow-up design coordinated with #32496 for broader provider parameter pass-through.

## Risks / Open Questions

- The issue bundles already-implemented config pieces with unimplemented adaptive strategy work, so closing it would likely lose the remaining feature request.
- Flat `agents.list[].params` may not satisfy users who need different params per agent per selected/fallback model.
- Adaptive thinking needs an explicit product decision for schema, runtime/session semantics, UI visibility, and provider-owned `adaptive` interactions.
