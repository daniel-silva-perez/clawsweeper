## Summary

Keep open. Current `main` still snapshots bootstrap file contents per `sessionKey`, and heartbeat runs use that session-scoped bootstrap path without automatic invalidation on ordinary heartbeat ticks. Because `HEARTBEAT.md` is loaded into the cached bootstrap file content, plain-text checklist changes can remain stale for a long-lived heartbeat session until the session snapshot is explicitly cleared or the session key changes.

## Risks / Open Questions

- I did not run a live reproduction because this review was constrained to read-only inspection and the checkout environment does not have `pnpm` available.
- Structured `tasks:` parsing in `src/infra/heartbeat-runner.ts` re-reads `HEARTBEAT.md` directly for task scheduling, so the stale-content risk here is specifically the bootstrap-prompt/session-snapshot path for ordinary HEARTBEAT.md instructions, not every heartbeat feature equally.
