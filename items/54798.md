## Summary

Keep #54798 open. Current main has a same-cron-job running marker and configurable cron concurrency, but it does not provide the requested shared atomic counter, session KV increment command, spawn mutex/exclusive parameter, or agent-facing file lock primitive. The related #54325 skip/already-running concern is narrower than this issue’s shared counter request.

## Best Possible Solution

Keep the issue open for maintainer product/API judgment. A focused fix would add one documented coordination primitive, preferably a gateway-backed workspace/session-scoped atomic KV increment tool/CLI with bounded keys, durable locking, tests for concurrent cron/subagent increments, and docs. If maintainers decide this belongs outside core, the existing plugin registerTool plus file-lock/json-store SDK pieces look sufficient for a ClawHub plugin, but that should be an explicit product-boundary decision rather than an auto-close.
