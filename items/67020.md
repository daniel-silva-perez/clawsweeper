## Summary

Keep #67020 open. Current main still accepts and documents Slack `dmHistoryLimit`, but the Slack channel monitor prompt-history path still only uses `historyLimit` behind `isRoomish`, so true 1:1 DMs (`im`) do not get the configured DM history in `combinedBody` or `InboundHistory`. The linked PR #67054 is closed and unmerged, and I did not find an equivalent fix on main.

## Best Possible Solution

Keep the issue open and implement the Slack monitor wiring: derive `dmHistoryLimit` in `provider.ts`, add it to `SlackMonitorContext`, compute an effective history limit for 1:1 DMs in `message-handler/prepare.ts`, record/read `channelHistories` for DM history when enabled, clear using that effective limit in dispatch, and add focused regression tests for non-threaded Slack DM replies. The cross-channel `dmHistoryLimit` gap noted in the comments can be tracked separately if maintainers want parity across other messaging plugins.
