## Summary

Keep open. Current main still matches the reported Slack bug: Slack config/schema/docs accept `dmHistoryLimit`, but the Slack monitor only derives and carries `historyLimit`, and the prompt-history paths are still gated on `isRoomish`, which excludes 1:1 DMs. The related Slack fix PR #67054 is closed and unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep this issue open and land a focused Slack monitor fix: derive `account.config.dmHistoryLimit`, carry it through `SlackMonitorContext`, compute an effective room-vs-DM history limit, record pending 1:1 DM history, use that effective limit for `combinedBody`, `InboundHistory`, and history clearing, and add a regression test for a non-threaded 1:1 Slack DM with `dmHistoryLimit > 0`. Track broader cross-channel `dmHistoryLimit` parity separately after the Slack behavior is fixed.
