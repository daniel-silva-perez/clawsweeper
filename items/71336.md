## Summary

Keep open. Current main still stores the WebChat pending `chatQueue` only in browser memory and explicitly clears it on session switches; there is no current localStorage queue persistence or restore/drain path. The PR is a recent, focused contributor update after maintainer feedback, and it targets a real remaining slice of #51549 rather than being obsolete. Current main already has chat history reload support, so the PR should be reviewed as queue persistence work and not treated as a complete fix for draft/history unless that scope is added or separately tracked.

## Best Possible Solution

Keep #71336 open for maintainer review as the focused WebChat queue-persistence implementation. Require scoped tests and adjust the PR wording so it either only fixes the queue-loss portion of #51549 or links separate follow-up work for draft persistence; keep the session-store rotation work delegated to #71328 rather than re-bundling it here.
