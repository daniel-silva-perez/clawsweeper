## Summary

Keep this PR open. Current main still has memory-only WebChat queued-message and draft state: queue mutations only update `host.chatQueue`, session switches clear queue/draft state, and reconnect only drains an already in-memory queue after a seq-gap. The linked bug #51549 remains open with concrete reproduction steps, and the latest PR diff is now focused on UI queue persistence rather than the earlier session-store changes, so this is an active implementation candidate rather than cleanup-close material.

## Best Possible Solution

Keep this PR open as the active focused implementation candidate for the queued-message persistence part of #51549. Review the UI-layer restore timing, persistence on every queue mutation, localStorage cleanup and privacy tradeoff, and add targeted tests for restore, TTL, dedupe, empty-queue cleanup, reconnect drain, and session-switch restore. Maintainers should also decide whether draft persistence must be included here or tracked as a separate follow-up before #51549 can close.
