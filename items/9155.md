## Summary

Current `main` does not implement a global default primary cron delivery target. The only global cron delivery-style config on `main` is `cron.failureDestination` for failure notifications; primary cron delivery still comes from each job's `delivery` block, and when that is omitted isolated agent-turn jobs default to `announce` with channel `"last"` resolved from session state.

## Evidence

- **Cron config type has no `defaults` or primary delivery default:** `CronConfig` defines `enabled`, `store`, `retry`, `webhook`, `sessionRetention`, `runLog`, `failureAlert`, and `failureDestination`, but no `defaults.delivery` field or equivalent global primary-delivery config.
