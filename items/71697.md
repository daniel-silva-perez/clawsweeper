## Summary

Keep #71697 open. The earlier #66505 work appears to cover direct Ollama Cloud setup, but current main still does not add a native `api: "ollama"` replay policy or send-time normalization for strict user/assistant alternation in the agent/Gateway path.

## Best Possible Solution

Keep the issue open and fix the native Ollama provider path. The likely maintainer path is an Ollama-owned replay policy or converter-level normalization for `api: "ollama"` that reshapes agent/Gateway histories into Ollama Cloud's accepted turn order while preserving native Ollama tool-call behavior, plus regression coverage for the failing replay shape.
