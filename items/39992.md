## Summary

Keep open. Current main still does not implement the requested general `openclaw doctor` warning for synthetic/runtime catalog models that are available but missing from an active `agents.defaults.models` allowlist. Main has partial related improvements for listing already-allowlisted synthetic entries, and a narrow hooks-specific doctor warning, but not the operator-wide diagnostic requested in #39992. The related PR #46235 attempted this exact feature but was closed unmerged per the provided GitHub context.

## Best Possible Solution

Implement a read-only doctor health contribution in core that loads the runtime model catalog, detects provider-owned synthetic/forward-compatible catalog entries with a stable marker or equivalent contract, compares them against an active non-empty `agents.defaults.models` allowlist, and emits a warning with model ids and `openclaw models allowlist add <model-id>` guidance. It should never auto-modify the allowlist, should skip when no allowlist is configured, and should include focused tests for no allowlist, empty allowlist, all synthetic models already allowed, gap detection, non-synthetic filtering, and command wording. The cost-profile concern raised in follow-up comments may be a separate doctor warning unless maintainers want to fold it into this check.
