## Summary

Keep open. Current main still does not implement this PR's requested behavior: `mergeProviderModels` continues to overwrite explicit model `input` with the implicit catalog value, and the existing merge test still asserts that explicit `input: ["image"]` becomes `input: ["text"]`. The downstream Gateway/WebChat media-ref mitigation helps avoid losing attachments for text-only runs, but it does not preserve configured native vision capability for custom provider models.

## Best Possible Solution

Keep the PR open or supersede it with a tested fix in `src/agents/models-config.merge.ts` that preserves source-authored `models.providers.*.models[].input` while still refreshing implicit catalog modalities when the user omitted `input`. Add focused regression coverage in `src/agents/models-config.merge.test.ts` for explicit vision input, omitted input, and defaulted-config/source-snapshot behavior before landing.
