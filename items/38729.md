## Summary

Keep PR #38729 open. Current main does not implement the requested Anthropic Opus 4.6 xhigh/max advertisement; it intentionally exposes xhigh/max only for Claude Opus 4.7 while Claude 4.6 gets adaptive only. The PR itself is stale against the current provider-thinking API because it adds deprecated supportsXHighThinking wiring that is bypassed when resolveThinkingProfile returns a profile, but the underlying product question remains plausible because the Anthropic transport still maps forced xhigh to max for Opus 4.6. No high-confidence close reason fits without a maintainer decision or canonical replacement item.

## Best Possible Solution

Keep the PR open for maintainer judgment, but require a rework before merge: if OpenClaw should expose Opus 4.6 max effort, implement it in extensions/anthropic/register.runtime.ts through resolveThinkingProfile, update the Anthropic profile tests and thinking docs, and keep Sonnet 4.6 excluded unless transport/runtime behavior genuinely supports a distinct xhigh/max level. If the intended product direction is to reserve xhigh/max for Opus 4.7 only, a maintainer should close this PR with that rationale or open/link a canonical policy issue.
