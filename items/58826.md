## Summary

Keep #58826 open. Current main has adjacent safeguards and telemetry, but not the requested built-in token/request budget enforcement at per-request, per-session, per-user/day, or per-provider scopes. The related metabolism PR was closed unmerged and is not present in current main, while the related max-turn/tool-call item only covers a narrower iteration-limit slice.

## Best Possible Solution

Keep this open and design a core budget/limit contract rather than closing on partial telemetry. The likely path is a documented config surface with global, per-agent, and per-provider/account overrides; preflight request counting; estimated prompt/output caps; post-response accumulated usage accounting; warn/block/fallback actions; and integration with existing model fallback, cooldowns, session usage, and tool-loop detection. Cross-link the narrower maxTurns/maxToolCalls work if maintainers want to split iteration limits from token/request budgets.
