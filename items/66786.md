## Summary

Keep this issue open. Current main still does not route the default agent workspace fallback through OPENCLAW_WORKSPACE_DIR, and that fallback feeds the runtime workspace value used by the system prompt when no explicit per-agent or agents.defaults.workspace is configured. The linked fix PR #66831 was closed unmerged, and current tests still cover OPENCLAW_HOME rather than the reported OPENCLAW_WORKSPACE_DIR override.

## Best Possible Solution

Implement the closed PR's intended behavior on current main: make resolveDefaultAgentWorkspaceDir prefer a non-empty OPENCLAW_WORKSPACE_DIR before OPENCLAW_PROFILE / OPENCLAW_HOME fallback, preserve explicit per-agent workspace and agents.defaults.workspace precedence, and add regression tests in the workspace resolver plus resolveAgentWorkspaceDir / resolveRunWorkspaceDir consumer seams so the system prompt and runtime workspace stay aligned.
