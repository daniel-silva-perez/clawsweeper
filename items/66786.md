## Summary

Keep this issue open. Current main still does not honor OPENCLAW_WORKSPACE_DIR in the default agent workspace resolver, and the system prompt’s workspace text is fed by that resolver when no explicit agent/default workspace is configured. The linked fix PR #66831 was closed unmerged, and the current tests still only lock OPENCLAW_HOME fallback behavior, not the reported env override.

## Best Possible Solution

Implement the closed PR’s intended behavior on current main: make resolveDefaultAgentWorkspaceDir prefer a non-empty OPENCLAW_WORKSPACE_DIR before OPENCLAW_PROFILE/OPENCLAW_HOME fallback, preserve explicit per-agent and agents.defaults.workspace precedence, and add regression tests in src/agents/workspace.test.ts plus the resolveAgentWorkspaceDir consumer seam so the system prompt and workspace-run fallback stay aligned.
