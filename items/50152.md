## Summary

Keep open. Current `main` has a narrower opt-in `compat.requiresStringContent` normalization path, but it does not implement the PR's claimed out-of-the-box NVIDIA/OpenAI-compatible fix. Bundled NVIDIA models still do not set that compat flag, and the NVIDIA docs still describe the provider as working directly with `api: "openai-completions"`. There is also still reviewer-level judgment needed on the PR's block-joining semantics versus the newline-preserving helper now on `main`.
