## Summary

Keep open. Current `main` does not implement the PR's requested install-method-aware update recommendation in the healthcheck skill, and the proposed PR approach still has unresolved correctness gaps.

## Evidence

- **Healthcheck skill lacks install-method-aware update guidance:** The current healthcheck workflow only says to run `openclaw update status` and report the channel/update availability. It does not instruct the agent to detect Homebrew vs global package-manager installs before recommending an update command.
  - file: [skills/healthcheck/SKILL.md:92](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/skills/healthcheck/SKILL.md#L92)

  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **No fallback guidance for Homebrew in periodic version-check section:** Later in the same skill, the only version-check options are `openclaw update status` and `npm view openclaw version`; there is still no Homebrew-specific update path or install-method detection step.
  - file: [skills/healthcheck/SKILL.md:199](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/skills/healthcheck/SKILL.md#L199)

That means current docs do not show that the PR request has already been implemented elsewhere on `main`.
