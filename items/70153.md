## Summary

Keep PR #70153 open. Current main still has the reported gap: the auto-reply CLI path calls `runCliAgent` directly and returns the result after lifecycle events without calling `persistCliTurnTranscript`. The helper and direct CLI command path already persist CLI turns, so the PR remains a plausible, focused fix rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open and have a maintainer review or adapt it onto current main. The best fix is still to persist successful auto-reply CLI turns with `persistCliTurnTranscript`, pass `transcriptCommandBody` and `followupRun.originatingThreadId`, update the active session entry when persistence succeeds, keep persistence failures best-effort, and add auto-reply tests for success, rejected persistence, and threadId propagation.
