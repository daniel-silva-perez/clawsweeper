## Summary

Keep open. Current main still lacks the requested Dreaming diary language/locale control: the narrative prompt is English-only, the narrative API/config resolver carries timezone but no language or locale, the strict memory-core plugin schema does not allow the requested key, and public docs expose no language setting. Related #69354 was closed as a duplicate of this same remaining gap, not as an implementation.

## Best Possible Solution

Keep this issue open and implement the feature in `memory-core`: add a documented `dreaming.language` or `dreaming.locale` config contract, decide whether `auto` is supported, resolve the hint in the memory-host config path, thread it into the Dream Diary system/user prompt path and request-scoped fallback behavior, update `extensions/memory-core/openclaw.plugin.json`, public Dreaming/memory config docs, generated config metadata if required, and focused resolver/narrative tests.
