## Summary

Keep open. Current `main` strips ANSI only in the overview log-tail card, but the user-facing logs view still preserves and renders raw log text without ANSI sanitization, so the reported Web UI leak is not clearly fixed across the current surface.

## Evidence

- **Overview card now strips ANSI:** `renderOverviewLogTail` removes OSC-8 and SGR escape sequences before joining lines for display.
  - file: [ui/src/ui/views/overview-log-tail.ts:5](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/ui/src/ui/views/overview-log-tail.ts#L5)
  - command: `nl -ba ui/src/ui/views/overview-log-tail.ts | sed -n '1,120p'`
  - sha: [4630ce3d9e2f](https://github.com/openclaw/openclaw/commit/4630ce3d9e2f089062ccd386b0ae241325961d1a)
- **Logs tab still renders unsanitized message/raw text:** `renderLogs` outputs `entry.message ?? entry.raw` directly into `.log-message` with no stripping/parsing step.
  - file: [ui/src/ui/views/logs.ts:132](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/ui/src/ui/views/logs.ts#L132)
  - command: `nl -ba ui/src/ui/views/logs.ts | sed -n '1,220p'`
  - sha: [4630ce3d9e2f](https://github.com/openclaw/openclaw/commit/4630ce3d9e2f089062ccd386b0ae241325961d1a)
- **Log ingestion preserves raw ANSI-bearing lines:** `parseLogLine` falls back to `{ raw: line, message: line }` for non-JSON lines, and `loadLogs` stores those entries as-is.
  - file: [ui/src/ui/controllers/logs.ts:50](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/ui/src/ui/controllers/logs.ts#L50)
  - command: `nl -ba ui/src/ui/controllers/logs.ts | sed -n '1,180p'`
  - sha: [4630ce3d9e2f](https://github.com/openclaw/openclaw/commit/4630ce3d9e2f089062ccd386b0ae241325961d1a)
- **Existing test does not cover ANSI stripping:** The only logs controller test verifies message selection from structured JSON; there is no assertion that ANSI escapes are removed anywhere in the logs UI path.
  - file: [ui/src/ui/controllers/logs.test.ts:1](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/ui/src/ui/controllers/logs.test.ts#L1)

  - sha: [4630ce3d9e2f](https://github.com/openclaw/openclaw/commit/4630ce3d9e2f089062ccd386b0ae241325961d1a)

## Risks / Open Questions

- Closing as implemented would be premature because a current Web UI log surface still displays raw `entry.message`/`entry.raw` text.
- The current code has no focused regression test proving ANSI sanitization for the logs UI path.
