## Summary

Keep open. Current main still renders the one-shot cron schedule field as a native browser `datetime-local` input, and the latest release tag in this checkout shows the same implementation. I did not find evidence on `main` that a fix for the reported "can not select time" regression has landed, so there is not strong enough proof to close this issue.
