## Summary

Keep #71417 open. Current main no longer treats `--channel` as session routing, and it has a stale-session freshness guard that rolls daily/idle-expired sessions, but the central reported problem remains for fresh sessions: `openclaw agent --agent <id> -m ...` without `--session-id` still resolves the agent main session key and reuses the current session when it is fresh. There is still no documented `--new-session`/`--fresh` flag and no resume banner in the CLI path.

## Best Possible Solution

Keep the issue open and implement an explicit operator-safe fresh-session path for `openclaw agent`: either default `--agent` one-shot CLI runs to a new session, or add and document `--new-session`/`--fresh`, plus a one-line resume banner when an existing session is reused. The change should cover both Gateway and embedded fallback paths and add regression tests for `--agent` with an existing fresh main session.
