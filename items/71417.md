## Summary

Keep open. Current main at 4b9c85776d29b1cc81ecbdacd7feaca6a46c0941 still has the reported behavior: omitted `--channel` normalizes to `last`, omitted `--session-id` with `--agent` resolves to the agent main session key, and both Gateway and local embedded paths reuse a fresh stored session id without a `--new-session`/`--fresh` control or resume banner. The reporter’s follow-up proposes a concrete backward-compatible fix path and #71415 is related but does not supersede this remaining omitted-session/default-channel issue.

## Best Possible Solution

Keep this issue open. Implement a backward-compatible fresh-session path for `openclaw agent`: add `--new-session` with optional `--fresh` alias, plumb a force-new flag through `AgentCliOpts`, Gateway `AgentParamsSchema`, Gateway handler, and local embedded `resolveSession`, short-circuit session reuse when set, and print a clear stderr resume banner when an existing session is reused. Update `docs/cli/agent.md` and `docs/tools/agent-send.md`, and add focused Gateway plus `--local` coverage for omitted session, blank `--session-id`, explicit existing `--session-id`, and fresh-session controls.
