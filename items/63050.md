## Summary

Keep open. Current main still does not implement the PR's two observable fixes: `models auth paste-token` has no `--token` flag and still always prompts interactively, and invalid auth-profile load warnings do not surface the accepted credential types. The PR remains a focused, useful fix; the bot review concerns in the provided discussion were addressed by follow-up commits on the PR head.

## Best Possible Solution

Keep the PR open and ask for maintainer review or a small test/docs follow-up. The best path is to land the narrow CLI/command/warning changes after regression coverage confirms non-interactive `paste-token` works, invalid non-interactive Anthropic tokens are rejected, empty `--token` fails clearly, and invalid-type warnings include the accepted credential types.
