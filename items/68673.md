## Summary

Keep this PR open. Current main still hashes full `update_plan` params and echoed result details, so wording-only plan churn can still evade the global no-progress breaker. The PR is a focused implementation candidate, but the proposed shape-only normalization has a real false-positive review concern that needs maintainer/code review rather than conservative closure.

## Best Possible Solution

Keep the PR open for maintainer review. The right path is to preserve the useful fix direction: normalize `update_plan` args and outcome details enough to collapse wording-only no-progress churn, but keep enough signal to distinguish material plan changes, then land focused regression coverage for both no-progress churn and real structural progress.
