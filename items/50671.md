## Summary

Keep open. Current `main` does not implement a `--no-device-identity` ACP path: `openclaw acp` exposes no such flag, `serveAcpGateway()` still creates `GatewayClient` without `deviceIdentity`, and `GatewayClient` still auto-loads one by default. The gateway side still supports token-only operator auth when no device is sent, so the requested behavior remains unimplemented rather than obsolete.

## Evidence

- **ACP CLI has no opt-out flag:** `src/cli/acp-cli.ts:14-26` defines ACP flags (`--url`, `--token`, `--token-file`, `--password`, `--password-file`, `--session`, `--session-label`, `--require-existing`, `--reset-session`, `--no-prefix-cwd`, `--provenance`, `--verbose`) and there is no `--no-device-identity` option.
