## Summary

Keep open. Current main at 76cf013 does not implement bidirectional node file transfer: the `nodes` agent tool has no `file_pull`/`file_push` actions, gateway node command policy has no `file.read`/`file.write` command IDs, and the headless node host only advertises/handles system, exec-approval, and plugin-registered commands. The PR is an active implementation attempt for the still-open #41716 core node-transfer request, not obsolete, duplicate, or a clean ClawHub-only close.

## Best Possible Solution

Keep #67495 open for maintainer review against #41716. The best path is a focused core implementation that adds `file.read`/`file.write` command constants and gateway policy, built-in headless/native node handlers with conservative size/path/write semantics, `nodes` agent `file_pull`/`file_push` actions, and matching tests/docs; if maintainers reject this PR’s shape, leave #41716 as the canonical feature request and ask for a narrower replacement PR.
