## Summary

Current main does not fully implement the requested Bedrock Claude vision capability. The Bedrock plugin still derives `input` only from Bedrock `inputModalities`, and inference profiles only inherit image support when the backing foundation model is present and already image-capable. Existing tests even assert an Opus 4.6-backed profile falls back to `input: ["text"]` when the foundation model is absent, which matches the reported failure mode. The attachment/transport paths then drop or reject images unless the resolved model input includes `image`, so this should stay open for a provider-owned fix.

## Best Possible Solution

Keep the issue open and fix it in the bundled `amazon-bedrock` plugin: add a normalized known-vision capability override for Bedrock-hosted Anthropic Claude 4.x IDs and their regional/global profile prefixes, then add focused discovery tests for the listed Opus 4.7, Opus 4.6, and Sonnet 4.6 variants where Bedrock metadata is text-only or the foundation model summary is absent. Update the Bedrock provider docs/changelog if the generated registry behavior changes.
