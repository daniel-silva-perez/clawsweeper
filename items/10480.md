## Summary

Current `main` still does not implement Cloudflare AI Gateway-specific Workers AI model selection during `openclaw onboard --non-interactive`. The Cloudflare AI Gateway path remains hardcoded to the Anthropic endpoint and default Claude model, with no onboard `--model` flag for this flow. There is now a generic custom-provider onboarding path that can configure an arbitrary OpenAI-compatible base URL and model ID during onboard, which may cover the practical workaround, but that is not the feature this issue requested.
