## Summary

Keep #70607 open. Current main still has the cross-agent target-agent account lookup that the linked bug report is about, so the requested behavior is not implemented on main. However, the PR’s one-character `!==` to `===` change is not clearly mergeable as-is because existing source comments and tests preserve caller account only for same-agent spawns while multiple cross-agent/thread tests intentionally use the target agent’s bound account for child delivery setup.

## Best Possible Solution

Keep the PR open for maintainer judgment, but do not merge the one-character guard flip as-is. The likely product fix is to separate the parent `requesterOrigin` used for completion routing from the child/session delivery origin used for target-agent bound setup, then update coverage for cross-agent completion, same-agent caller-account preservation, thread bindings, and ACP/session spawn paths.
