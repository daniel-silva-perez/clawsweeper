## Summary

Keep open. Current main still classifies only numeric `sessions.json.bak.<digits>` store backups via `LEGACY_STORE_BACKUP_RE`, while PR #59997 broadens that to all non-empty `sessions.json.bak.*` suffixes and adds targeted regression coverage for the reported incident patterns. This is a plausible small bug fix, not already implemented on main.

That does not cover the PR's `session-maint-*`, `safe-cleanup-*`, or `pre-bot-session-detach-*` backup names.
