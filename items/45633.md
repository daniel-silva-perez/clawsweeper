## Summary

Keep open. Current `main` has clearer MiniMax CN/global docs and onboarding labels, but I did not find a code path that detects or repairs stale MiniMax endpoint mismatches, or that turns a MiniMax 401 into the clearer CN-vs-global guidance requested in #45633.

## Evidence

- **MiniMax now exposes explicit CN/global auth choices:** The bundled MiniMax manifest declares separate `minimax-global-oauth`, `minimax-global-api`, `minimax-cn-oauth`, and `minimax-cn-api` choices, each with explicit `api.minimax.io` vs `api.minimaxi.com` hints. That improves setup clarity, but it is not itself a stale-config fix.
  - file: [extensions/minimax/openclaw.plugin.json:12](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/extensions/minimax/openclaw.plugin.json#L12)
- **Docs spell out the regional split:** The provider docs have separate International and China onboarding flows and explicitly say which host each flow configures. This covers the documentation part of the report, but not automatic recovery from old wrong-endpoint configs.
  - file: [docs/providers/minimax.md:47](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/docs/providers/minimax.md#L47)
- **Provider auth remains static, not region-detecting:** `createMinimaxApiKeyMethod` and `createMinimaxOAuthMethod` register fixed Global and CN methods. I did not find logic here that inspects the API key, prior config, or a 401 response to infer the correct region or warn about a mismatch.
  - file: [extensions/minimax/provider-registration.ts:184](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/extensions/minimax/provider-registration.ts#L184)
- **Onboarding writes the chosen base URL but does not repair stale MiniMax configs:** The MiniMax onboarding helpers always apply the base URL associated with the selected flow and merge existing provider state. I did not find a migration or doctor-style repair path here for residual MiniMax configs that still point CN credentials at the global endpoint.
  - file: [extensions/minimax/onboard.ts:19](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/extensions/minimax/onboard.ts#L19)
- **Review stayed read-only and checkout stayed clean:** `git status --short` returned no changes after inspection.
  - command: `git status --short`
