## Summary

Keep open. Current main still does not include the proposed symptom-first troubleshooting section, and the PR remains a plausible docs contribution rather than a cleanup close. It does need revision before merge: current docs/source treat `minimal` as a first-class tool profile, while the PR section only explains `messaging`, `coding`, and `full`; it also points users at a `Global tools profile` status row that current main does not document or build.

## Best Possible Solution

Keep #40039 open as a salvageable docs PR. Revise the troubleshooting section to include `minimal`, align the diagnosis steps with status behavior currently shipped on main or wait for #39996, and keep the detailed profile definitions linked back to the canonical tools/config docs so the troubleshooting page stays compact and accurate.
