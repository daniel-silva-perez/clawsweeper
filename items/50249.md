## Summary

Keep open. Current main still does not implement the proposed script-backed CLI banner tagline mode: runtime mode parsing, config schema/types, tests, and docs only support random/default/off. The earlier related PR #41147 is closed and unmerged, while this PR is the active focused follow-up, so it is not superseded. The idea also needs maintainer product/security judgment because it would add opt-in arbitrary JavaScript execution during CLI startup, and the current plugin surfaces do not expose a CLI banner tagline hook that would make a ClawHub-only close safe.

## Best Possible Solution

Keep the PR open for maintainer product review. If accepted, land it as a core CLI config contract with schema/types/help/docs/tests aligned and careful startup suppression/timeout behavior. If maintainers prefer extensibility instead, design a narrow CLI banner/tagline plugin hook first, then route this kind of customization through a plugin rather than closing the current PR without a viable extension point.
