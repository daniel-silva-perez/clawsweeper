## Summary

Current main no longer probes both `MEMORY.md` and `memory.md` for bootstrap injection. Root memory bootstrap only loads an exact `MEMORY.md`, lowercase `memory.md` is treated as legacy repair input, regression tests cover the case, and the changelog says this shipped in 2026.3.13 as the fix for duplicate root-memory injection on case-insensitive filesystems.

## Evidence

- **Bootstrap loader only accepts exact canonical root memory file:** `loadWorkspaceBootstrapFiles()` only enqueues `DEFAULT_MEMORY_FILENAME` (`MEMORY.md`) and skips memory bootstrap entirely unless `exactWorkspaceEntryExists(resolvedDir, DEFAULT_MEMORY_FILENAME)` succeeds. That removes the old dual-candidate `MEMORY.md`/`memory.md` behavior that could duplicate injection on case-insensitive filesystems.
