## Summary

Keep this issue open. Current main has the runtime guard and generic plugin/gateway documentation, but the requested one-place BlueBubbles guidance is still missing: the BlueBubbles page does not compare channel disablement with plugin disablement, does not show the composed local loopback/token example with both BlueBubbles layers disabled, and does not explain startup config mutation behavior in that BlueBubbles context.

## Best Possible Solution

Add a focused section to docs/channels/bluebubbles.md that compares channels.bluebubbles.enabled=false with plugins.entries.bluebubbles.enabled=false, explains which layer controls channel startup versus plugin activation/runtime-dependency repair, includes a JSON5 local example with gateway.mode="local", gateway.bind="loopback", gateway.auth.mode="token", channels.bluebubbles.enabled=false, and plugins.entries.bluebubbles.enabled=false, and briefly states what startup may persist or repair while linking to the public plugin and gateway config docs.
