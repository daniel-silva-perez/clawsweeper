## Summary

Keep open. Current `main` still rejects `channels.discord.guilds.*.channels.*.agentId`, the Discord docs still point per-channel/per-conversation routing to top-level `bindings[]`, and the Discord doctor compatibility rules do not add any migration hint or auto-fix for this unsupported shape.

## Evidence

- **Discord channel config type excludes `agentId`:** `DiscordGuildChannelConfig` defines channel overrides like `requireMention`, `enabled`, `users`, `roles`, and `systemPrompt`, but no `agentId` field exists in the public config type.
  - file: [src/config/types.discord.ts:39](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/src/config/types.discord.ts#L39)
  - command: `nl -ba src/config/types.discord.ts | sed -n '35,95p'`
  - sha: [ecac696643c6](https://github.com/openclaw/openclaw/commit/ecac696643c604c6d854341bfe6424bfa31423e9)
- **Discord channel schema is strict and does not allow extra keys:** `DiscordGuildChannelSchema` enumerates allowed keys and ends with `.strict()`, so a channel-level `agentId` is still rejected as an unrecognized key on current `main`.
  - file: [src/config/zod-schema.providers-core.ts:448](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/src/config/zod-schema.providers-core.ts#L448)
