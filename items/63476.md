## Summary

Keep open. Current main has lower-level media-store URI hydration and a direct image-tool test for `media://inbound`, but the actual agent-session tool path still passes `workspaceDir` into the image tool. In that path, `media://inbound/<id>` is treated as a workspace-relative path before `loadWebMedia` can hydrate the managed media URI, and the existing regression test does not cover `createOpenClawCodingTools`/`createOpenClawTools` with `workspaceDir`.

## Best Possible Solution

Keep #63476 open until the image tool preserves `media://inbound/<id>` refs when `workspaceDir` is set, with a regression through `createOpenClawCodingTools` or `createOpenClawTools` using `workspaceDir` and `workspaceOnly`. The implementation should align image-tool behavior with the PDF/image-generate tools, which leave managed media refs intact before calling `loadWebMedia`.
