## Summary

Current main still appears to have the reported gap: `completeSubagentRun` always calls `cleanupBrowserSessionsForLifecycleEnd` when cleanup is triggered, and `SubagentRunRecord` only has the ended-hook marker, not a browser-cleanup dispatch marker. This should stay open because linked PR #68669 is still open, by the same contributor, and its body uses closing syntax for this issue.

## Best Possible Solution

Keep this issue open and review/land or close #68669. If maintainers accept the approach, the fix belongs in `src/agents/subagent-registry-lifecycle.ts`, `src/agents/subagent-registry.types.ts`, and the run replacement path in `src/agents/subagent-registry-run-manager.ts`, with a regression test proving parallel `completeSubagentRun(..., triggerCleanup: true)` only invokes the browser cleanup wrapper once.
