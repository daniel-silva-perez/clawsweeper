## Summary

Current main still implements numbered config backup rotation, not datetime-suffixed backups. The implementation rotates `.bak` into `.bak.1` through `.bak.4`, cleanup preserves only numeric suffixes and would remove timestamp-like `.bak.*` files, and tests still assert the numeric ring behavior. The related implementation PRs are not merged, so this remains a valid open feature request.

## Best Possible Solution

Keep this issue open until one of the datetime-suffix implementations lands, or until maintainers explicitly decide to keep the numeric backup ring. The implementation should update `src/config/backup-rotation.ts`, refresh `src/config/config.backup-rotation.test.ts`, and handle pruning, permission hardening, collision behavior, and legacy `.bak.N` migration deliberately.
