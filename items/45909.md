## Summary

Current `main` already suppresses `NO_REPLY` on the direct-delivery path used for structured-content and threaded cron delivery, and the latest release tag contains the same implementation and regression test coverage.

## Close Comment

Closing after Codex review: current `main` already implements this. In `src/cron/isolated-agent/delivery-dispatch.ts:44`, `normalizeSilentReplyText()` strips silent-token replies, and `src/cron/isolated-agent/delivery-dispatch.ts:525` applies that normalization to every direct-delivery payload before any outbound send. Silent-only payloads exit via `finishSilentReplyDelivery()` at `src/cron/isolated-agent/delivery-dispatch.ts:536`, so the structured/threaded `useDirectDelivery` branch at `src/cron/isolated-agent/delivery-dispatch.ts:825` no longer leaks literal `NO_REPLY`. The regression is covered in `src/cron/isolated-agent/delivery-dispatch.double-announce.test.ts:821` (structured direct delivery), with additional direct-delivery cases at `:1046` and `:1131`. I also compared the latest release tag `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`) against current `main` for the affected code/test files and found no diff, so this is already shipped. If you can still reproduce it on `v2026.4.22` or newer, please open a new issue with the exact cron config and payload shape.
