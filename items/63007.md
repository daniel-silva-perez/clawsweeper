## Summary

Keep this PR open. Current main still builds outbound session identity for gateway send, but does not pass agent identity into the public `message_sending` context and still collapses hook-cancelled gateway sends into the generic `No delivery result` error. The PR is also paired with the same-author broader outbound-send issue #63011, which remains open.

## Best Possible Solution

Keep this PR open, or replace it with an equivalent focused maintainer patch: add optional `agentId` to the public message hook context, pass outbound `session.agentId` and `session.key` into `message_sending`, preserve cancellation reason through a narrow delivery callback or blocked-result shape, surface an explicit guarded gateway-send denial, and cover the gateway/outbound path plus docs. Review it alongside the same-author broader #63011 rather than closing one half prematurely.
