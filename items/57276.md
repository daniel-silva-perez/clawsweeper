## Summary

Keep open. Current `main` still treats Linux gateway services as user-scoped only, while the docs explicitly describe system-scoped `/etc/systemd/system/openclaw-gateway*.service` installs. The PR targets a real gap and is not already implemented on `main`.

## Evidence

- **User-unit-only enable check:** `isSystemdServiceEnabled()` only probes the user unit path via `fs.access(resolveSystemdUnitPath(env))` and then runs `systemctl --user is-enabled ...`; there is no `/etc/systemd/system` detection or system-scope branch.
  - file: [src/daemon/systemd.ts:646](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/daemon/systemd.ts#L646)
  - command: `nl -ba src/daemon/systemd.ts | sed -n '646,667p'`
