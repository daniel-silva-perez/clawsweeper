## Summary

Keep open. Current main still performs the broad `schtasks /Query` availability preflight inside `readScheduledTaskRuntime()` before the task-scoped `/Query /TN ... /V /FO LIST` lookup, so PR #51486 remains a relevant narrow fix for the Windows status bug reported in #49187. The refreshed PR also addresses the bot review concern by catching `execSchtasks` rejections before Startup-folder fallback handling.

## Best Possible Solution

Keep PR #51486 open or land an equivalent narrow fix: remove the broad `/Query` preflight only from `readScheduledTaskRuntime()`, keep install/stop/restart preflight behavior intact, catch `execSchtasks` rejections so Startup fallback and `unknown` status handling remain defensive, and add regression tests for the single task-scoped runtime query plus spawn-error fallback.
