## Summary

Keep #71142 open. Current main expanded Control UI/WebChat uploads to non-video file attachments, but the requested configurable upload-size limit is not implemented: `chat.send` still passes a literal 5,000,000-byte attachment cap, and the config schema/docs expose no `gateway.controlUi` or `gateway.webchat` upload-size setting.

## Best Possible Solution

Add a first-class Gateway/WebChat upload limit config, likely under `gateway.webchat` or `gateway.controlUi`, validate it in the strict config schema, thread it into `chat.send`/session attachment parsing instead of the literal `5_000_000`, and document how it relates to `tools.media.*.maxBytes` and channel-specific media caps.
