## Summary

Keep open. Current main still only enriches BlueBubbles group participant phone handles, does not resolve 1:1 inbound sender names, and explicitly skips email-addressed participants. The maintainer discussion also points toward a contributor PR rather than cleanup closure.

## Best Possible Solution

Leave this issue open and review a focused BlueBubbles plugin PR. The likely path is to extend `extensions/bluebubbles/src/participant-contact-names.ts` with reusable phone/email sender resolution, call it from `monitor-processing.ts` only for authorized explicit 1:1 inbound senders with unusable native names, add the broader config flag with legacy fallback, and cover group plus DM phone/email behavior in tests and docs.
