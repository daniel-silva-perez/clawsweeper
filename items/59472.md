## Summary

Current main still directly awaits provider usage loading in the `channels list --json` path, so this PR is not obsolete or already implemented. The PR appears to address a real robustness gap, but existing review comments about silent JSON-mode error reporting and test mock cleanup still need maintainer judgment rather than cleanup closure.
