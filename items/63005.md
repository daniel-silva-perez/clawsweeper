## Summary

Keep #63005 open. Current main still has a real split path: explicit raw inbound session keys can be stored and looked up as raw keys, while gateway/session mutation paths canonicalize non-agent keys to `agent:<default>:...` and prune the raw alias. The related #49202 remains open and is narrower to Open WebUI bare UUID chats, so it does not fully supersede this structured external-key report.

## Best Possible Solution

Keep open. The best fix is a small session-identity invariant across inbound and gateway paths: either canonicalize explicit external session keys consistently before inbound lookup, or preserve raw external aliases that future inbound traffic still uses instead of pruning them. Add a regression that starts with an existing raw structured key, exercises a gateway mutation path such as `chat.send`/`sessions_send`, then verifies the next inbound turn reuses the same `sessionId` and transcript rather than creating a second record.
