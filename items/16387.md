## Summary

Keep open. Current `main` still treats custom provider headers as static strings and merges them verbatim into request config; there is no implemented runtime interpolation path for `{{session.key}}`, `{{agent.id}}`, `{{run.id}}`, or similar per-session/per-turn placeholders.

## Evidence

- **Header sanitization is verbatim-only:** `sanitizeModelHeaders()` accepts string values and copies them directly into the next header map. It does not parse or interpolate runtime placeholders.
  - file: [src/agents/pi-embedded-runner/model.inline-provider.ts:47](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/src/agents/pi-embedded-runner/model.inline-provider.ts#L47)

No session, agent, or run context is supplied for interpolation.
