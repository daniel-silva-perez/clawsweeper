## Summary

Keep #58110 open. Current main has useful pieces for multi-agent routing, per-agent sessions, session switching, and sub-agent work, but it still does not implement the requested product shape: agents as chat contacts with visible availability plus first-class group chats where users invite multiple agents into one shared conversation. The related open items track narrower gaps, so they do not supersede this umbrella request.

## Best Possible Solution

Keep this open as a product/UI umbrella unless maintainers intentionally split it. Define concrete acceptance criteria for a Control UI agent-contact/sidebar experience, per-agent session switching/history, visible agent status, and first-class multi-agent conversation or invitation semantics. Reuse existing multi-agent routing, sessions.list, sub-agent, and thread-binding primitives where they fit, while linking narrower follow-ups for attribution, command targeting, and per-agent queue isolation.
