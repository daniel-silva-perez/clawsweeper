## Close Comment

Current `main` already implements this Discord feature, so I'm closing it from this Codex review.

- `extensions/discord/src/monitor/message-handler.process.ts:627` builds a `Working…` preview and appends live tool/progress lines into the same edited draft message.
- The Discord reply pipeline wires `onToolStart`, `onItemEvent`, `onPlanUpdate`, `onApprovalEvent`, `onCommandOutput`, and `onPatchSummary` into that preview stream, and suppresses duplicate default tool-progress messages when preview-tool-progress is enabled (`extensions/discord/src/monitor/message-handler.process.ts:977`).
- The config contract exposes `streaming.preview.toolProgress` with default `true`, and the Discord docs describe the behavior (`src/config/types.base.ts:37`, `src/plugin-sdk/channel-streaming.ts:119`, `docs/channels/discord.md:538`).
- The changelog records this as shipped in `2026.4.21` under “Channels/preview streaming” (`CHANGELOG.md:317`, `CHANGELOG.md:326`).
