## Summary

Current main still does not implement the documented Telegram inline-button callback contract for the reported path. Generic callback_query data is forwarded to the agent as the raw payload, while the Telegram docs promise `callback_data: <value>`. The linked PR #54962 tracks the exact fix but is still open/unmerged, so this issue should stay open until the code and regression test are updated or the docs contract is intentionally changed.

## Best Possible Solution

Keep #54909 open and land a narrow Telegram plugin fix equivalent to #54962: forward only generic inline-button callbacks to the agent as `callback_data: ${data}` or deliberately update the public docs if raw callback text is the intended contract. Add a regression test that fails on the current raw-data behavior while preserving native command and built-in callback handling.
