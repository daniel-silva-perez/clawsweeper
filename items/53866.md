## Summary

Keep PR #53866 open. It is a valid Firecrawl web-search onboarding feature request, but current main has moved the setup surface away from `src/commands/onboard-search.ts` into provider-owned `runSetup`; the PR’s core-command implementation is therefore not the best merge path and the feature is not implemented on main yet.

## Best Possible Solution

Do not close or merge this branch as-is. Rewrite the feature inside `extensions/firecrawl`: add a Firecrawl-owned PKCE/browser-auth helper and `createFirecrawlWebSearchProvider().runSetup`, persist only canonical plugin config paths, respect `secretInputMode: "ref"` by avoiding plaintext storage, sync web-search/web-fetch credentials through plugin-owned config as appropriate, add Firecrawl plugin tests plus shared search-flow coverage, and then update `docs/tools/firecrawl.md` to document browser sign-in.
