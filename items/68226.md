## Summary

Keep open. Current main still renders the shared status footer with only the session key plus last-updated/no-activity text; neither `/status` nor `session_status` shows elapsed duration or an absolute start time. Existing session metadata has enough start-time plumbing to implement this cleanly, but it has not been wired into the status renderer yet.

## Best Possible Solution

Implement this in the shared status renderer so both `/status` and `session_status` benefit. Resolve a start timestamp from `sessionEntry.sessionStartedAt` first, with a safe fallback through existing lifecycle/header timestamp plumbing if needed, then render either `started <absolute time>` or a compact `duration <elapsed>` segment before the existing `updated ...` text. Add focused coverage around `buildStatusText()` or `buildStatusMessage()` for the session footer, and consider whether #57442 should become the broader canonical tracker for response-time/status timing beyond this specific session-start/duration field.
