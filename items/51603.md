## Summary

Keep PR #51603 open. Current main already fixes the chat model picker side of the PR and has supported custom-provider onboarding/configuration paths, but it does not implement the requested top-level `llm` config key or runtime expansion into `models.providers` and `agents.defaults.model.primary`. The remaining config shorthand is a real unmerged API/product decision, and the PR discussion also identified unresolved correctness issues in the proposed implementation.

## Best Possible Solution

Keep this open for maintainer judgment rather than auto-close. If the shorthand is still desired, refresh the PR on current main, split out or drop the already-implemented UI picker change, add schema/types/docs/tests for the top-level `llm` contract, and fix the review gaps around existing provider merges, fallback preservation, provider-id validation, and API enum validation. If maintainers prefer the existing onboarding/configure workflow, close with that explicit product decision instead of treating it as already implemented.
