## Summary

Keep #49870 open. Current main has related channel-owned inbound conversation resolution, but it still does not implement the PR's central agent-hook contract: PluginHookAgentContext lacks conversationId/accountId, the embedded runner hook contexts do not pass those fields, and after_agent_complete does not exist on main.

## Best Possible Solution

Keep the PR open but ask for a rebase/rework: add conversationId/accountId to the public PluginHookAgentContext and every agent hook call site, decide whether after_agent_complete belongs in this PR and implement it fully if so, preserve current channel-owned inbound conversation resolution, and add regression tests for multi-account agent hooks plus Discord DM/native-command routing.
