## Summary

Current main already uses the host local date for default rolling gateway log filenames, and the shipped docs/tests match that behavior, so the UTC-based filename split described in #50151 is already implemented away on main.

## Close Comment

Closing from a Codex review because current OpenClaw main already implements the requested behavior. The default rolling gateway log filename is now built from the host local calendar date in `src/logging/logger.ts:330`, and `resolveSettings()` uses that local-date path when `logging.file` is unset (`src/logging/logger.ts:127`). There is also coverage in `src/logger.test.ts:73` asserting the default file is `openclaw-YYYY-MM-DD.log` for today's local date, and the shipped docs say explicitly that the date uses the gateway host's local timezone (`docs/logging.md:22`, `docs/gateway/logging.md:20`). This behavior is present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). If you still hit UTC-split files on a newer build, please open a fresh issue with the exact version, effective `logging.file` setting, timezone, and a minimal repro.
