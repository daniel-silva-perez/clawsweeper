## Summary

Keep PR #51546 open. Current main still does not implement progressive disclosure for long markdown messages: chat markdown is rendered directly in `.chat-text`, the markdown renderer still hard-truncates at 140,000 chars, and tests still document the 40,000-char plain-text fallback. The PR is not obsolete or implemented on main, but the supplied review discussion shows it needs revision before merge.

## Best Possible Solution

Keep the PR open for focused UI maintainer review or request a revision. A best implementation should avoid eagerly parsing/inserting hundreds of thousands of characters, keep expanded state in Lit/reactive state, keep the toggle control outside masked content, handle streaming and completed messages predictably, and add regression coverage for long-message rendering.
