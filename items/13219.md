## Summary

Keep open. Current `main` has transcript-backed usage summaries and per-session/per-model aggregation, but it does not implement the requested dedicated append-only per-call usage ledger/log file. The issue’s core ask (`~/.openclaw/logs/usage.jsonl`-style native usage logging independent of transcript scanning) is still unmet.

## Evidence

- **Cost summary still scans session transcript files:** `loadCostUsageSummary` reads `resolveSessionTranscriptsDirForAgent(...)`, filters transcript filenames, and iterates each file with `scanUsageFile(...)`; there is no separate ledger path or writer in this flow.
  - file: [src/infra/session-cost-usage.ts:362](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/src/infra/session-cost-usage.ts#L362)
  - sha: [3361593442e2](https://github.com/openclaw/openclaw/commit/3361593442e295e9bb1f55cbfb38861ea9413e45)
- **Session usage/log APIs are transcript-backed:** `usage.cost`, `sessions.usage`, and `sessions.usage.logs` all route to transcript-derived loaders (`loadCostUsageSummaryCached`, `loadSessionCostSummary`, `loadSessionLogs`) rather than a dedicated usage ledger.
  - file: [src/gateway/server-methods/usage.ts:397](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/src/gateway/server-methods/usage.ts#L397)
  - sha: [3361593442e2](https://github.com/openclaw/openclaw/commit/3361593442e295e9bb1f55cbfb38861ea9413e45)
- **Per-session logs are reconstructed from transcript message content:** `loadSessionLogs` opens the session transcript, walks message rows, extracts content, and derives token/cost values only for assistant transcript entries. This is not a normalized per-call accounting log with fields like `eventId`, `runId`, `source`, or `status`.
  - file: [src/infra/session-cost-usage.ts:929](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/src/infra/session-cost-usage.ts#L929)
  - sha: [3361593442e2](https://github.com/openclaw/openclaw/commit/3361593442e295e9bb1f55cbfb38861ea9413e45)
- **Docs still describe `/usage cost` as session-log / transcript-backed:** The docs say `/usage cost` shows a local cost summary from OpenClaw session logs, and explicitly say assistant transcript entries are the stable source for `/usage cost` and session status.
  - file: [docs/reference/token-use.md:71](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/docs/reference/token-use.md#L71)
  - sha: [3361593442e2](https://github.com/openclaw/openclaw/commit/3361593442e295e9bb1f55cbfb38861ea9413e45)
- **Usage tracking docs do not describe a native cost ledger:** Current docs describe provider quota endpoints plus transcript fallback, and say `/usage cost` is aggregated from session logs. No dedicated usage ledger/log file is documented.
  - file: [docs/concepts/usage-tracking.md:11](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/docs/concepts/usage-tracking.md#L11)
  - sha: [3361593442e2](https://github.com/openclaw/openclaw/commit/3361593442e295e9bb1f55cbfb38861ea9413e45)
- **No dedicated usage-ledger symbols or `usage.jsonl` path found on main:** Repository search for `usage.jsonl`, `UsageLedger`, and similar ledger-specific identifiers returned no relevant implementation on current `main`. `git status --short --branch` also showed the checkout remained clean.
  - command: `rg -n "usage\.jsonl|UsageLedger|usage ledger|eventId|costStatus|costUsd" src docs && git status --short --branch`
  - sha: [3361593442e2](https://github.com/openclaw/openclaw/commit/3361593442e295e9bb1f55cbfb38861ea9413e45)

## Risks / Open Questions

- `main` now provides adjacent functionality: transcript-backed cost summaries, per-session usage APIs, and aggregate `byModel` views. That reduces urgency, but it does not satisfy the specific request for a dedicated append-only per-call usage ledger.
- GitHub CLI inspection of related issues/PRs was not available in this environment because `GH_TOKEN` was unset, so this review is based on the checked-out `main` tree, docs, and local history only.
