## Summary

Keep open. Current main checkpoints daily/session ingestion, so it avoids re-ingesting unchanged source files, but the light-phase diary/report path still selects recent short-term entries from the lookback window every sweep and sends those snippets to the narrative writer without a per-phase last-emitted cutoff. Existing diary dedupe intentionally keeps identical bodies when timestamps differ, which matches the reported repeated work-summary entries across hourly cycles. The related #70737 cron decoupling and #68882 promotion-contamination issue are orthogonal, not canonical fixes for this symptom.

## Best Possible Solution

Keep #72096 open. The best fix is in memory-core: persist a per-workspace/per-phase emitted cursor or emitted-snippet/body fingerprint for Dream Diary/light-phase narrative inputs, filter narrative/report input to newly emitted material, and add a regression test that runs two light sweeps with the same short-term entries at different timestamps and proves the duplicate work-summary body is not appended again.
