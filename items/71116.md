## Summary

Keep this issue open. Current main still has the privileged secrets RPC surface and existing SecretRef/SDK helper documentation, but it does not yet contain the requested focused `src/gateway/server-methods/AGENTS.md` secrets guidance or a central plugin credential-persistence contract. The issue also has an open same-author paired PR, #71156, that remains the active implementation candidate rather than a reason to close the issue before merge.

## Best Possible Solution

Keep this issue open until maintainers review and land #71156 or an equivalent focused documentation change. The best path is to add secrets-surface guidance under `src/gateway/server-methods/AGENTS.md` and add or cross-link public plugin docs that define the credential-persistence contract: SecretRef-backed config versus plugin-local storage, lock plus atomic-write expectations, TTL/lifecycle handling, and failure behavior that must not report success before persistence or revocation guarantees hold.
