## Summary

Keep #71116 open. Current main does not yet contain the requested secrets-specific maintainer guidance or a centralized extension credential-persistence contract. The related PR #71156 is still open and only partially covers the issue, so it is not a shipped implementation or a clear superseding canonical item.

## Best Possible Solution

Keep the issue open as the umbrella architecture/docs item. Maintainers should decide whether #71156 is the desired first slice, then either land a revised docs PR that covers the secrets RPC invariants plus extension credential persistence contract, or split the remaining helper/CI-audit ideas into explicit follow-up issues.
