## Summary

Keep open. Current `main` has media-understanding preprocessing and separate image/video media model configuration, but it does not implement the requested automatic per-message reply-model switch from a text model to a multimodal model and back based on inbound content type.

## Evidence

- **Model selection docs still describe only primary/fallback reply routing plus a separate image-model fallback:** `docs/concepts/models.md` says reply selection is `agents.defaults.model.primary` then `fallbacks`, and separately says `agents.defaults.imageModel` is used only when the primary model cannot accept images.
  - file: [docs/concepts/models.md:16](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/docs/concepts/models.md#L16)
  - command: `nl -ba docs/concepts/models.md | sed -n '10,40p'`
  - sha: [a59d1bd46d9f](https://github.com/openclaw/openclaw/commit/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3)
- **Inbound media is handled as preprocessing, not by switching the session reply model:** `docs/nodes/media-understanding.md` describes inbound media understanding as summarizing attachments before the reply pipeline, with ordered media-model fallback; it does not describe automatic session model changes. `get-reply.ts` calls `applyMediaUnderstanding(...)` before the normal reply flow using the already-resolved `activeModel`.
  - file: [docs/nodes/media-understanding.md:11](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/docs/nodes/media-understanding.md#L11)

- **Current image behavior skips preprocessing when the active model already has vision instead of switching models:** `src/media-understanding/runner.ts` explicitly skips image-understanding when the active model supports vision natively and otherwise resolves media-understanding entries; this is media preprocessing/routing logic, not automatic session model swapping.
