## Summary

Current `main` does not implement this PR's behavior. The fallback replay policy for unowned `anthropic-messages` providers still gates `dropThinkingBlocks` on Claude-like model IDs only, and current tests explicitly assert that newer Claude models behind a custom anthropic proxy keep thinking blocks. That means the PR is neither implemented nor obsolete on `main`; it would need rework against the newer replay-policy architecture rather than cleanup closure.

## Evidence

- **Unowned anthropic fallback still uses Claude model-id gating:** `resolveTranscriptPolicy()` falls back to `buildUnownedProviderTransportReplayFallback()`, where `dropThinkingBlocks` is only set when `isAnthropic` and `modelId.includes("claude")`, then further filtered by `shouldPreserveThinkingBlocks(modelId)`. There is no current-main guard that drops thinking blocks for all non-native `anthropic-messages` proxies.
