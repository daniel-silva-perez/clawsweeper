## Summary

Current main already unblocks same-origin microphone access in the default gateway Permissions-Policy header, but it still does not implement the PR's runtime configurability. The PR remains potentially useful and should stay open for rebase, scope cleanup, and maintainer review rather than being closed as obsolete.

## Best Possible Solution

Keep this PR open for maintainer review or contributor follow-up. The best path is to rebase it onto current main, preserve or explicitly re-decide the shipped `camera=(), microphone=(self), geolocation=()` default, add `gateway.http.securityHeaders.permissionsPolicy?: string | false` through config types, zod/generated schema/help/labels, public gateway docs, runtime resolution, and `createGatewayHttpServer`, and keep focused tests for default, override, and explicit-disable behavior. Split or remove unrelated auth-resolution/schema metadata changes before merge.
