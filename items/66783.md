## Summary

Keep this PR open. Current main already shipped the microphone-unblocking default `Permissions-Policy`, but it has not implemented the PR's requested runtime configurability: `gateway.http.securityHeaders.permissionsPolicy` is absent from config types, zod/schema metadata, docs, runtime resolution, and HTTP server thread-through. The supplied bot review also treated the core permissions-policy change as correct and tested, while flagging only scope-cleanup concerns, so the remaining work is still potentially useful rather than obsolete.

## Best Possible Solution

Keep this PR open for maintainer review or contributor follow-up. The best path is to rebase it onto current main, preserve or explicitly re-decide the shipped `camera=(), microphone=(self), geolocation=()` default, add `gateway.http.securityHeaders.permissionsPolicy?: string | false` through types, zod/generated schema/help/docs, runtime config, and `createGatewayHttpServer`, and keep focused tests for default, override, and explicit-disable behavior. Split or remove the unrelated auth-resolution/schema metadata changes noted in review.
