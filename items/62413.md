## Summary

Keep open. Current `main` still renders Dream Diary entries as plain text paragraphs and does not run diary content through the existing sanitized markdown pipeline, so the reported markdown bug remains present despite the linked comment claiming PR #62433 fixed it.

## Evidence

- **Dream Diary still renders raw text:** `renderDreamDiaryEntries()` builds the diary body by mapping `flattenDiaryBody(entry.body)` into `<p>` nodes and interpolating `${para}` directly. There is no markdown conversion or `unsafeHTML(...)` call in this path.
  - file: [ui/src/ui/views/dreaming.ts:1272](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/ui/src/ui/views/dreaming.ts#L1272)
  - command: `nl -ba ui/src/ui/views/dreaming.ts | sed -n '1272,1335p'`
  - sha: [ecac696643c6](https://github.com/openclaw/openclaw/commit/ecac696643c604c6d854341bfe6424bfa31423e9)
- **Preprocessing flattens prose instead of rendering markdown:** `flattenDiaryBody()` trims lines and strips list markers/labels before display. That preserves literal markdown markers like `**bold**` as text rather than converting them to formatted HTML.
  - file: [ui/src/ui/views/dreaming.ts:335](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/ui/src/ui/views/dreaming.ts#L335)
  - command: `nl -ba ui/src/ui/views/dreaming.ts | sed -n '335,360p'`
  - sha: [ecac696643c6](https://github.com/openclaw/openclaw/commit/ecac696643c604c6d854341bfe6424bfa31423e9)
- **Standard markdown renderer exists elsewhere in the UI:** OpenClaw already exports `toSanitizedMarkdownHtml()` and uses it in chat rendering via `unsafeHTML(toSanitizedMarkdownHtml(markdown))`. Dream Diary does not use that pipeline on current `main`.
  - file: [ui/src/ui/markdown.ts:477](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/ui/src/ui/markdown.ts#L477)

- **Current tests only cover plain-text diary rendering:** The Dream Diary view test asserts `.textContent` for parsed prose and has no assertion that markdown is rendered as formatted HTML, which matches the still-broken code path.
  - file: [ui/src/ui/views/dreaming.test.ts:371](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/ui/src/ui/views/dreaming.test.ts#L371)
