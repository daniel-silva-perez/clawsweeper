## Summary

Keep issue #71664 open. It is fresh, author association is NONE, and current main still has a plausible same-channel WhatsApp block-streaming delivery gap: `text_end` flush waits for the block pipeline, but the same-channel path only enqueues into the dispatcher and does not wait for the actual channel delivery before tool execution can continue.
