## Summary

Keep open. Current main still binds `TaskRecord.task` directly into the SQLite `task_runs.task` parameter, so the PR is not implemented or superseded. The PR is a small, focused storage-boundary fix with regression coverage, although maintainers should still decide whether structured task payloads are an accepted runtime contract because the current public task types say `task` is a string.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to either merge the narrow SQLite-boundary serialization with its regression test, or ask the contributor to add a concrete runtime/plugin path proving how a structured task value can reach `TaskRecord.task`. It should not be closed as implemented because current main still lacks the normalization.
