## Summary

Keep open. Current `main` has partial coverage for this idea: OpenClaw already retries some 429/transient external calls, applies provider auth-profile cooldown/backoff after rate-limit failures, and shows provider usage/quota in status surfaces. But the issue asks for configurable external-call rate limiting/throttling, token/request budgets, queueing/blocking behavior, alerts, and a dedicated `openclaw limits ...` CLI surface. I found no strong evidence that those requested control-plane features exist on `main`.

## Evidence

- **Channel/API retry exists, but only as per-request retry/backoff:** `createRateLimitRetryRunner` and `createChannelApiRetryRunner` handle retries, `retry_after`, and logging for rate-limited/transient requests. This is retry behavior, not a configurable global limiter/budget system.
  - file: [src/infra/retry-policy.ts:53](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/src/infra/retry-policy.ts#L53)
  - sha: [3361593442e2](https://github.com/openclaw/openclaw/commit/3361593442e295e9bb1f55cbfb38861ea9413e45)
- **Provider failover cooldowns exist, but they are post-failure cooldown state:** Auth-profile usage state records cooldown/disable windows after `rate_limit`, `billing`, or auth failures, with exponential backoff and model scoping. This is failover/cooldown logic, not the per-provider request/token budget enforcement proposed in the issue.
  - file: [src/agents/auth-profiles/usage.ts:733](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/src/agents/auth-profiles/usage.ts#L733)
  - sha: [3361593442e2](https://github.com/openclaw/openclaw/commit/3361593442e295e9bb1f55cbfb38861ea9413e45)
- **Config/docs expose failover knobs, not a global external API limiter:** The documented knobs are `auth.cooldowns.*` and cron retry settings such as `backoffMs`/`retryOn`. They do not match the requested `rateLimits` config with RPM/TPM/day budgets, queue sizes, or blocking policies.
  - file: [docs/gateway/configuration-reference.md:3510](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/docs/gateway/configuration-reference.md#L3510)
  - sha: [3361593442e2](https://github.com/openclaw/openclaw/commit/3361593442e295e9bb1f55cbfb38861ea9413e45)
- **Visibility is partially implemented through usage/quota status surfaces:** `openclaw status --usage` and provider usage tracking already show normalized provider usage/quota windows, and status text loads provider usage summaries for the current provider. That addresses part of the observability ask, but not enforcement or budget controls.
  - file: [docs/concepts/usage-tracking.md:11](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/docs/concepts/usage-tracking.md#L11)
