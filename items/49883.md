## Summary

Current `main` already lets onboarding users skip per-skill API-key setup without restarting. The shared wizard calls `setupSkills()`, and that flow asks for an explicit yes/no confirmation before prompting for a required skill API key; answering no continues onboarding. The same behavior is present in the `v2026.4.22` release commit.

## Evidence

## Close Comment

Closing from a Codex review because current OpenClaw already supports this. In `src/commands/onboard-skills.ts:205`, onboarding asks `Set <ENV> for <skill>?` with a default `false`, and `src/commands/onboard-skills.ts:209` continues the wizard when you decline; the required-value check at `src/commands/onboard-skills.ts:212` only runs after you explicitly opt in. This skills step is wired into the shared onboarding flow in `src/wizard/setup.ts:667`, and that shared wizard is what app/UI clients use per `docs/reference/wizard.md:193`. This behavior is also present in the shipped `v2026.4.22` release (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
