## Summary

Keep open. Current main still has only the source-channel `Timestamp` in the shared inbound context, agent-visible inbound metadata, plugin SDK direct-DM/envelope helpers, and hook payloads. Telegram has a plugin-local `receivedAtMs` used for ingress debug timing, but it is not a cross-channel Gateway-received metadata field and is not exposed to agents as `receivedAt` or `gatewayReceivedAt`.

## Best Possible Solution

Add an additive typed gateway receipt timestamp to the shared inbound context, such as `GatewayReceivedAtMs` or `ReceivedAtMs`, set once at channel ingress or in shared dispatch helpers, propagate it through plugin SDK helpers and hook payloads, render it in agent-visible inbound metadata as a stable ISO timestamp, and cover the metadata builder plus at least one buffered or long-poll-style channel path with focused tests.
