## Summary

Keep open. Current `main` does not implement the PR's SSRF guard: `firecrawl_scrape` still forwards the caller-supplied URL straight into the Firecrawl `/v2/scrape` request with no local hostname/protocol validation. The cache-growth part of the PR description is mostly obsolete because shared `writeCache()` already caps these maps at 100 entries, but that does not change the missing SSRF fix.
