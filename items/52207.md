## Summary

Keep #52207 open. The SearXNG half is already superseded by the cleaner merged SearXNG provider on current main, but the Tavily generic `web_search` freshness behavior requested by this PR is still missing. A maintainer also explicitly noted that #57317 was related but not a direct duplicate because #52207 combines SearXNG with Tavily freshness.

## Best Possible Solution

Keep the item open, but ask for a focused rework: drop the already-shipped SearXNG changes from this PR, rebase onto current main, and implement only Tavily generic `web_search` freshness. The likely path is to add `freshness` to `extensions/tavily/src/tavily-search-provider.ts`, normalize accepted values to Tavily's supported recency parameter or current `time_range` contract, include it in the Tavily cache key/body in `extensions/tavily/src/tavily-client.ts`, update `docs/tools/tavily.md`, and add focused provider/client tests. If a new focused PR is opened for that Tavily work, this mixed stale PR can then be closed as superseded.
