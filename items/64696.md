## Summary

Keep open. Current main still lacks the requested memory-wiki bridge import and lint durability fixes: relative Markdown links are not resolved against the containing page, report links are emitted vault-root-relative from nested reports, bridged source content is rendered raw, bridge render fingerprinting has no renderer-version token, and cached source-page skip logic still accepts any existing file. The related PR #65197 was closed unmerged, while open #70986 only covers part of the parser/resolver problem.

## Best Possible Solution

Keep this issue open and implement the bounded memory-wiki fixes in maintained source with regression coverage: make link extraction markdown-aware or otherwise prevent bridged reply directives from becoming lint targets, resolve native Markdown links relative to the containing page, emit report-local native links for reports/stale-pages.md, include a bridge renderer-version token in render fingerprinting, and validate existing imported source-page frontmatter before skip. Coordinate parser/resolver overlap with #70986, but do not close this issue until the bridge cache/render invalidation and malformed cached-page repair acceptance criteria are covered.
