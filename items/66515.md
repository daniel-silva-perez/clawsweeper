## Summary

Keep open. Current main does not implement a Yandex/YandexGPT provider, and this PR is the active replacement for the earlier closed PR #59768 rather than a duplicate of a newer canonical implementation. The provider-plugin surface exists and the project explicitly prioritizes major model-provider support, so there is not enough maintainer-level evidence to auto-close it as ClawHub-only. The right path is normal PR review: decide whether Yandex should be bundled, then require the remaining review issues to be fixed before merge.

## Best Possible Solution

Keep this PR open for maintainer review. If Yandex is accepted as a bundled provider, require the author to address the latest review comments, keep provider-specific auth/catalog/onboarding inside the plugin boundary, avoid unnecessary provider-named SDK surface unless a real public contract is needed, and add focused tests for folder ID resolution and config preservation. If maintainers decide Yandex is not a bundled provider, redirect it explicitly to a ClawHub/npm provider plugin rather than auto-closing from this cleanup pass.
