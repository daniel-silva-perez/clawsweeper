## Summary

Keep this PR open. Current main still does not ship a Yandex/YandexGPT provider, and this PR is an active replacement for the earlier closed same-author PR #59768 rather than obsolete work. Provider additions are supported by the plugin/provider architecture, while VISION.md makes major model-provider support a maintainer product direction, so this needs normal PR review rather than automated cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. If Yandex is accepted as a bundled provider, require the remaining auth/config preservation fixes, keep Yandex-specific behavior inside the provider plugin boundary, avoid unnecessary public SDK surface unless maintainers want that stable facade, and split or justify unrelated release-script changes. If maintainers decide Yandex should not be bundled, close it explicitly with direction to publish it as a ClawHub/npm provider plugin using the documented provider-plugin API.
