## Summary

Current main still does not implement configurable workspace bootstrap file order. Bootstrap discovery and final prompt rendering both use fixed ordering, the config schemas are strict and expose only injection/size/truncation controls, and the existing `agent:bootstrap` hook can mutate the file set but does not preserve arbitrary file ordering because the prompt renderer re-sorts files before rendering.

## Best Possible Solution

Keep #59050 open. A conservative implementation would add documented `agents.defaults.bootstrapFileOrder` and likely `agents.list[].bootstrapFileOrder`, validate it against supported bootstrap basenames, thread it through bootstrap context assembly and final prompt rendering before `sortContextFilesForPrompt()`, preserve deterministic fallback ordering for unlisted files, update generated config schema/help/docs, and add tests proving SOUL.md can render before AGENTS.md while default behavior stays unchanged.
