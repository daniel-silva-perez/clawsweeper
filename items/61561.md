## Summary

Keep open. The PR is a focused BlueBubbles bug fix with a concrete failure mode from the discussion, and current main still does not implement equivalent `iMessageLite` to `iMessage` normalization for satellite-message chat GUIDs.

## Best Possible Solution

Keep the PR open and either rebase/finish it or land an equivalent BlueBubbles plugin fix. The implementation should use one shared chatGuid service canonicalizer for `iMessageLite` to `iMessage`, apply it to webhook ingestion and target parsing, consider direct `chatGuid` action parameters/allowlist semantics deliberately, and add focused regression coverage for satellite-message inbound and target parsing paths.
