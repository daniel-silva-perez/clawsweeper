## Summary

Keep #63162 open. Current main has related pieces, but not the requested Control UI file explorer toggle/sidebar. The chat header controls only expose refresh/thinking/tool/focus/cron toggles, the global Control UI navigation has no file browser tab, and the existing agent Files panel is a narrow bootstrap-file editor rather than a workspace tree. The request also includes general file drag/drop and chat selection flows, while current chat attachments are image-only. This also should not be closed as ClawHub: current plugin APIs expose tools, channels, providers, HTTP routes, gateway methods, CLI, hooks, and services, but no Control UI topbar/sidebar/panel contribution point.

## Best Possible Solution

Keep the issue open as a Control UI product request. The likely path is a core UI change plus a bounded gateway workspace-tree/read API, or first designing a Control UI extension point if maintainers want this to be plugin-provided. The implementation should reuse existing safe file-boundary patterns, define workspace root and scope behavior, and decide separately whether general document/file chat attachments should be added beyond current image-only support.
