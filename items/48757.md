## Summary

Keep open. Current main does not implement the PR’s intended Vertex ADC runtime-auth fix: `google-vertex` ADC still resolves to the non-secret `gcp-vertex-credentials` marker, the run and compaction paths can still forward/store that marker as runtime auth when no provider runtime exchange returns a real credential, and the Google transport treats raw string credentials as API-key headers. The PR also needs rework for current main because the provided review context notes it targeted the old `<authenticated>` sentinel and included unresolved review concerns.

## Best Possible Solution

Keep the item open and rework the fix against current main: guard non-secret Vertex ADC markers in the run auth controller and compaction path, avoid passing the marker as `resolvedApiKey` to provider-owned streams, add focused tests for `google-vertex` ADC marker handling, and leave unrelated github-copilot behavior out of this PR.
