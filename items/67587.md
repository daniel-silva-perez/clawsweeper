## Summary

Keep this issue open. Current main has Discord outbound mention rewriting, but the implementation still depends on the account-scoped in-memory Discord directory cache; it preserves unknown handles as plaintext and does not provide a deterministic config-backed alias-to-userId mapping for generated `@Name` references. The issue is recent, concrete, unprotected, and the provided related items do not supersede this outbound mention reliability request.

## Best Possible Solution

Implement this in the Discord plugin by adding an account-scoped deterministic source of known mention aliases, such as `channels.discord.mentionAliases` or `accounts.<id>.mentionAliases`, and/or by seeding the existing directory cache from resolved allowlist or inbound Discord author metadata. Add tests covering configured aliases, account isolation, unknown-name preservation, code-span preservation, and outbound send/webhook/structured-message paths.

## Risks / Open Questions

- A fix should stay in the Discord plugin surface and preserve account scoping, because name-based Discord matching can collide across users.
- Care is needed to avoid broad outbound rewriting of `@everyone`, `@here`, code spans, or arbitrary unknown names.
