## Summary

Keep open. Current main still returns `Unable to resolve session target` immediately when a non-empty ACP target token fails session resolution, so the bound-thread fallback is not reached. The related fix PR #66338 was closed unmerged, and the local implementation still matches the reported failure mode.

## Best Possible Solution

Keep this issue open and land a focused resolver fix equivalent to the closed PR #66338: let unresolved explicit tokens fall through to `resolveBoundAcpThreadSessionKey`, preserve the existing explicit-token error when no thread binding resolves, and add a regression test for a bound-thread `/acp close <non-session-token>` plus a no-context bad-token case.
