## Summary

Keep open. Current main still matches the reported failure mode: a non-empty ACP target token that fails session resolution returns `Unable to resolve session target` before the thread-bound resolver can run. The related fix PR #66338 was closed unmerged, so this issue still needs a focused code change and regression coverage.

## Best Possible Solution

Keep this issue open and land a focused resolver fix equivalent to the closed PR #66338: when token resolution returns null, try `resolveBoundAcpThreadSessionKey` before returning the explicit-token error. Preserve the existing bad-token error when no thread binding resolves, and add regression tests for both `/acp close <non-session-token>` inside a bound thread and a bad explicit token with no binding context.
