## Summary

Keep open. Current main still resolves the user-visible `--version` commit from live git before `dist/build-info.json` and has no build-time `__OPENCLAW_GIT_COMMIT__` define, while this PR directly implements and tests that missing behavior. The linked bug is closed, but the current code does not show the fix shipped.

## Best Possible Solution

Keep this PR open for maintainer review and likely landing or rebasing. The best product/code path is to bake a build-time commit into node build outputs, consult it after explicit `GIT_COMMIT`/`GIT_SHA` env overrides and before live git, and keep regression coverage for env override, injected priority, fallback, and cache stability. If maintainers choose not to merge this PR, the remaining bug should stay tracked because current main still has the drift-prone priority order.
