## Summary

Keep open. Current main still has the exact behavior this PR changes: node `agent.request` passes `runId: sessionId`, and the current test asserts that equality. The surrounding gateway/event code keys per-run state by `runId`, so this remains a plausible focused core gateway bug fix rather than an obsolete, duplicate, or ClawHub item.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land or adapt the narrow core gateway fix so each node `agent.request` gets a unique run id, keep the focused regression test that proves `runId !== sessionId`, and run the relevant gateway changed/test lane before merge.
