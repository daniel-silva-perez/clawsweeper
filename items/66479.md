## Summary

Current main still matches the reported Windows BOM failure path. The shared frontmatter parser checks for the `---` delimiter before removing a leading UTF-8 BOM, skill parsing delegates directly to that parser, and skill loading drops entries whose parsed description is missing. The related BOM/write-tool report #45432 does not cover this skill-discovery path.

## Best Possible Solution

Keep this open and fix the shared markdown frontmatter parser by stripping a leading UTF-8 BOM before checking for the opening `---` delimiter. Add a focused `parseFrontmatterBlock` regression test for BOM-prefixed content and a skill-loader or skills-list test proving a valid BOM-prefixed `SKILL.md` remains discoverable.
