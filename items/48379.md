## Summary

Current `main` does not implement per-agent OpenRouter attribution. OpenRouter request shaping still forces `X-OpenRouter-Title: OpenClaw`, protects that header from override, and only exposes provider-scoped header config plus per-agent model params, so the feature request remains valid.

## Evidence

- **Static OpenRouter title on main:** `resolveProviderAttributionIdentity()` returns product `OpenClaw`, and `buildOpenRouterAttributionPolicy()` maps `X-OpenRouter-Title` to that product rather than any agent id or agent-specific setting.
  - file: [src/agents/provider-attribution.ts:391](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/src/agents/provider-attribution.ts#L391)
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **Attribution header is protected from override:** The request-header merge path builds a protected set from provider attribution headers, strips matching caller headers, then merges defaults back in. That prevents overriding `X-OpenRouter-Title` through normal caller/config header paths.
  - file: [src/agents/provider-request-config.ts:633](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/src/agents/provider-request-config.ts#L633)
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **Config surface is provider-scoped, not agent-scoped:** The config docs expose extra headers only under `models.providers.*.headers` and `models.providers.*.request.headers`, which are provider-level transport settings.
  - file: [docs/gateway/config-tools.md:409](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/docs/gateway/config-tools.md#L409)
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **Per-agent settings only cover model/stream params:** `agents.list[].params` is defined as per-agent stream params, and embedded-runner param resolution merges those params into model request params, not provider transport headers.
  - file: [src/config/types.agents.ts:117](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/src/config/types.agents.ts#L117)
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **OpenRouter image path is also static:** The bundled OpenRouter image provider still sends `X-OpenRouter-Title: OpenClaw` directly in its default headers, so non-LLM OpenRouter traffic is likewise not agent-attributed.
  - file: [extensions/openrouter/image-generation-provider.ts:263](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/extensions/openrouter/image-generation-provider.ts#L263)
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **User-facing docs still document a fixed title:** The OpenRouter provider docs say OpenClaw adds `X-OpenRouter-Title: OpenClaw` on real OpenRouter requests; there is no documented per-agent override.
  - file: [docs/providers/openrouter.md:87](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/docs/providers/openrouter.md#L87)
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)

## Risks / Open Questions

- Closing this would discard a still-valid enhancement request: multi-agent OpenRouter usage on current `main` remains indistinguishable in OpenRouter logs.
- Any future fix likely needs a new agent-aware transport/config seam because current provider-level header config and per-agent params do not cover this case.
