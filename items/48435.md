## Summary

Keep open. Current `main` does not implement Slack bot-message unfurl controls: the Slack config schema exposes no `unfurlLinks`/`unfurlMedia` fields, and the outbound Slack sender never passes `unfurl_links` or `unfurl_media` to `chat.postMessage`.

## Evidence

- **Slack config schema has no unfurl settings:** `SlackAccountSchema` and `SlackConfigSchema` enumerate Slack channel config keys such as `streaming`, `thread`, `slashCommand`, `responsePrefix`, `ackReaction`, and `typingReaction`, but nothing for `unfurlLinks` or `unfurlMedia`.
