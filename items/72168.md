## Summary

Keep #72168 open. The report is fresh, concrete, independently confirmed in the discussion, and current main still has the same packaged-build failure shape: browser Playwright routes soft-load `pw-ai`, the Playwright implementation still resolves `playwright-core` via normal ESM package resolution, and the root package does not declare `playwright-core`. PR #72238 is the right active fix path, but it is open and unmerged, so the issue should remain open until that or an equivalent fix lands.

## Best Possible Solution

Keep #72168 open and use PR #72238, or an equivalent maintainer patch, as the fix path. The fix should make browser Playwright runtime resolution work from packaged external plugin deps, preferably through a browser-local runtime shim or another resolver-visible mechanism, and should add regression coverage for packaged global/Homebrew-style installs where `doctor --fix` stages plugin deps outside the OpenClaw package tree.
