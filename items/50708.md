## Summary

Keep open. Current main still has the user-visible stale-exit behavior this PR targets: systemd runtime reads ExecMainStatus/ExecMainCode for active services, and the shared status formatter prints last exit/reason regardless of runtime.status. The latest PR diff in the provided GitHub context moved suppression to src/daemon/runtime-format.ts rather than deleting raw runtime fields, which addresses the Codex review concern about preserving restart/onboarding diagnostics. The remaining gap is test coverage, not a cleanup-close reason.

## Best Possible Solution

Keep the PR open. The best path is to add a small formatter regression test that suppresses last-exit fields only for status "running", preserves them for stopped/failed services, and keeps readSystemdServiceRuntime returning raw ExecMainStatus/ExecMainCode for restart/onboarding diagnostics. After that, run the changed gateway/daemon lanes and merge if green.
