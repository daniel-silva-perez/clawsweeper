## Summary

Keep PR #70884 open. Current main still lacks the PR's trusted-proxy WebSocket scope-clearing documentation, while the current Gateway code and tests confirm that device-less trusted-proxy Control UI WebSocket sessions can connect but have unbound scopes cleared unless a narrow exception applies. The provided PR discussion also supports the factual direction of the docs change, with later commits addressing the review comments about dangerous flag wording and the allowInsecureAuth exception.

## Best Possible Solution

Keep the PR open for maintainer review or merge after confirming the final docs wording. The best outcome is to document the existing Gateway behavior in docs/gateway/trusted-proxy-auth.md and docs/gateway/protocol.md: x-openclaw-scopes is an HTTP request header, WebSocket scopes come from the Gateway handshake/device binding, device-less trusted-proxy Control UI sessions can connect but lose unbound scopes, allowInsecureAuth only preserves local token/password scopes, and dangerouslyDisableDeviceAuth must remain framed as break-glass critical-risk behavior.
