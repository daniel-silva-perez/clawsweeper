## Summary

Keep open. This PR is a small docs clarification that still appears useful: current main documents generic Discord guild user/role allowlist behavior, but the bot-to-bot loop section does not yet include the PR's more specific allowBots/guild sender-gating clarification. The PR discussion also shows the initial placement/wording concerns were addressed in follow-up commits, so this is not a conservative close candidate.

## Evidence

They do not include the PR's clarification that guild users/roles are evaluated for member access and allowBots is orthogonal.
