## Summary

Keep open. Current `main` does not include this PR’s restart-reason propagation: `emitGatewayRestart` is still a zero-argument helper, deferred restarts do not accept or forward a reason, and the gateway config-reload path still emits restarts without attaching the config-change cause.

## Evidence

- **Restart emitter still has no reason parameter:** On current `main`, `emitGatewayRestart` remains `export function emitGatewayRestart(): boolean` and only emits SIGUSR1; there is no reason argument or logging of a propagated restart cause.
  - file: [src/infra/restart.ts:120](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/src/infra/restart.ts#L120)
  - command: `nl -ba src/infra/restart.ts | sed -n '112,170p'`
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **Deferred restart path cannot carry a reason:** `deferGatewayRestartUntilIdle` currently accepts `getPendingCount`, hooks, emitHooks, pollMs, and maxWaitMs only. There is no `reason` field to preserve restart context through idle deferral.
  - file: [src/infra/restart.ts:251](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/src/infra/restart.ts#L251)

- **Gateway config reload still emits restart without a reason:** The config-reload handler still calls `deferGatewayRestartUntilIdle(...)` with no reason and still uses `emitGatewayRestart()` with no arguments for the immediate path, so the changed config keys are not propagated into the restart emission path.
  - file: [src/gateway/server-reload-handlers.ts:257](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/src/gateway/server-reload-handlers.ts#L257)
