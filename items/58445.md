## Summary

Keep open. Current main does not implement the requested `before_block_reply` hook or trigger-bearing block reply context, and the PR is a plausible core plugin API extension rather than a ClawHub-only plugin. The PR discussion also contains unresolved implementation risk around preserving trigger context through coalesced block replies, so it should not be closed as already implemented or obsolete.
