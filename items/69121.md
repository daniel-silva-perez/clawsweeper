## Summary

Keep #69121 open. Current main still has only pre-restart deferral; it does not pass a remaining drain budget into `server.close()`, and the close handler cannot drain pending reply dispatchers or abort active chat runs before sockets/channels are closed. The PR is recent, focused, and not superseded by the related closed restart reports in the provided context.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a narrow gateway restart fix that carries the remaining restart drain budget into `GatewayServer.close`, drains `getTotalPendingReplies()` plus active `chatAbortControllers` before teardown, aborts still-active chat runs through the existing chat-abort path when the bounded budget is exhausted, and adds focused regression tests for both the run-loop handoff and close-time drain behavior.
