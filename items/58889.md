## Summary

Keep #58889 open. Current main still appears to do Telegram audio preflight transcription inside inbound body resolution before the reply pipeline starts typing. The issue is recent, author is not a maintainer, has no protected labels, and the requested UX improvement remains plausible and actionable. The one issue comment from SonicBotMan contains useful WeChat/Discord operational notes, but it does not provide a workaround or plugin that resolves this Telegram voice typing request.

## Evidence

- **Audio preflight still happens before body resolution completes:** Current Telegram inbound body handling detects audio media, computes needsPreflightTranscription, loads media understanding runtime, and awaits transcribeFirstAudio before returning the body text used for dispatch. This matches the reported delay window before the agent reply loop can begin.
