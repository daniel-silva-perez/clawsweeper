## Summary

Keep this PR open. Current main still only supports the memory backend at the root `memory` config plus per-agent `memorySearch`; `agents.list[].memory` is not in the type, strict schema, docs, generated schema, or backend resolver. The PR remains a focused implementation candidate for #64612, and the requester explicitly confirmed the intended boundary matches their staged-rollout need.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The best path is to refresh it on current main, fix any remaining typecheck/CI issue, preserve the intended boundary (`agents.list[].memory.backend` and `agents.list[].memory.qmd.*`, with `memory.citations` global), update generated schema/help and the package mirror, and land it with the included schema/runtime/resolver/docs regression coverage or an equivalent maintainer-owned patch.
