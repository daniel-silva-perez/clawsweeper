## Summary

Keep open. Current main does not contain this PR's registry-level sentinel-stripping fix, and the current Google Vertex ADC path still resolves ADC to a non-secret marker that is treated as an API-key-mode credential and can be passed into runtime transports. The PR is also stale against current main: it targets the old "<authenticated>" sentinel while current main uses "gcp-vertex-credentials", and the branch includes unrelated skill content plus unresolved review concerns.

## Best Possible Solution

Keep the PR open or replace it with a current-main fix: make google-vertex ADC resolve as a provider-owned OAuth/ADC credential rather than an API-key marker, ensure run, compact, and simple-completion paths never pass the marker to setRuntimeApiKey or transport options, and add regression tests proving ADC does not become x-goog-api-key for google-vertex.
