## Summary

Keep open. Current main and the latest release add prompt-level mitigations for config changes, but the repo still exposes the generic `exec` shell tool and I did not find a code-level guard that would prevent `openclaw config set ...` or `openclaw gateway restart` from being executed without confirmation. In a read-only review I cannot safely reproduce the report because a real reproduction would mutate config and restart the gateway.

## Risks / Open Questions

- A prompt-only mitigation may not fully eliminate the reported behavior if a model still chooses `exec` for config-changing shell commands.
- A conclusive `cannot_reproduce` determination would require a live session that changes config and restarts the gateway, which is outside this read-only review.
- The report targeted OpenClaw 2026.3.23-2; current main is different, but I found insufficient hard evidence to say the bug is fully fixed rather than merely mitigated.
