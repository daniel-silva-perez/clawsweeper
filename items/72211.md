## Summary

Keep PR #72211 open. Current main has general dashboard connectivity and gateway service troubleshooting, but it still does not include this PR's specific local-dashboard-after-update recovery runbook with lsof/curl checks, LaunchAgent reload recovery, and stale browser/deep-link guidance. The PR is focused docs work, and the existing review feedback is an actionable wording fix rather than a reason to close it as obsolete.

## Best Possible Solution

Keep PR #72211 open and ask for a small docs revision: preserve the local-dashboard-after-update recovery runbook, but split the flow so users first check whether `127.0.0.1:18789` is already listening and serving dashboard HTML, then run `openclaw gateway restart` or `openclaw gateway start` only when the gateway is not listening or not serving the dashboard. After that, this remains a focused docs improvement suitable for normal review.
