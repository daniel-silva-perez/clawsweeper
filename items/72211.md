## Summary

Keep PR #72211 open. Current main has general dashboard connectivity and gateway service troubleshooting, but it does not contain the PR's specific local-dashboard-after-update recovery runbook with lsof/curl checks, LaunchAgent reload recovery, and stale-browser-state guidance. The command surface and LaunchAgent recovery behavior exist in current code/tests, and the bot review identified an actionable wording improvement rather than a reason to close.

## Best Possible Solution

Keep the PR open and ask for a small docs revision: preserve the useful local-dashboard recovery runbook, but split the diagnostic flow so users first check whether 127.0.0.1:18789 is already serving the dashboard, then run `openclaw gateway restart` or `openclaw gateway start` only if the gateway is not listening or not serving HTML. After that, the PR can be reviewed as a focused docs improvement rather than closed as obsolete.
