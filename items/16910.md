## Summary

Keep open. Current `main` does not implement Cursor credits as a model provider, and the request is still coherent and in scope: OpenClaw treats Cursor today as a bundle ecosystem and ACP harness target, while provider integrations remain a supported plugin path rather than a clearly out-of-scope core request.

## Evidence

- **Vision keeps optional capability on the plugin path, not as an automatic core rejection:** The project vision says optional capability should usually ship as plugins, with external/community distribution preferred and a high bar for adding optional capability to core.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/VISION.md#L54)
  - sha: [a59d1bd46d9f](https://github.com/openclaw/openclaw/commit/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3)
- **Commercial integrations are only ruled out when they are not really model providers:** The same vision document rejects commercial service integrations only when they do not clearly fit the model-provider category. A Cursor-credits integration would be a provider request, so this is not a strong scope basis to close.
  - file: [VISION.md:99](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/VISION.md#L99)
  - sha: [a59d1bd46d9f](https://github.com/openclaw/openclaw/commit/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3)
- **OpenClaw docs treat model providers as plugins, including external/community plugins:** The plugin docs state that plugins can provide model providers and that some plugins are external/community packages. The current bundled provider list includes `github-copilot` but no `cursor`, which shows this request is not already implemented on `main`.
  - file: [docs/tools/plugin.md:11](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/docs/tools/plugin.md#L11)
  - sha: [a59d1bd46d9f](https://github.com/openclaw/openclaw/commit/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3)
- **Provider integrations are an explicit supported plugin surface, including ClawHub publication:** The provider-plugin guide documents how to add a provider plugin and explicitly says provider plugins published on ClawHub are supported. That means the request is technically actionable, not incoherent.
  - file: [docs/plugins/sdk-provider-plugins.md:11](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/docs/plugins/sdk-provider-plugins.md#L11)
  - sha: [a59d1bd46d9f](https://github.com/openclaw/openclaw/commit/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3)
- **Current Cursor support is not provider-credit support:** OpenClaw's Cursor docs describe Cursor support as bundle compatibility and ACP harness support. Bundles are content packs, not native provider plugins, and ACP runs the external Cursor harness rather than onboarding Cursor credits as an OpenClaw model provider.
  - file: [docs/plugins/bundles.md:10](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/docs/plugins/bundles.md#L10)
  - sha: [a59d1bd46d9f](https://github.com/openclaw/openclaw/commit/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3)
- **No bundled Cursor provider exists on current main:** A repo search for `cursor` under `extensions/` returned no Cursor provider/plugin directory on `main`, confirming there is no existing bundled implementation to justify `implemented_on_main`.
  - command: `rg --files extensions | rg 'cursor'`
  - sha: [a59d1bd46d9f](https://github.com/openclaw/openclaw/commit/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3)
