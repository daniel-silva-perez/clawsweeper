## Summary

Keep open. Current main still has the reported eager session-row materialization and synchronous transcript I/O paths. The older #57715 tracker overlaps the `sessions.list` hot path, but #72165 adds unresolved direct `sessionId` lookup and MCP `getConversation` scope, so it is not a safe conservative duplicate close without maintainer consolidation.

## Best Possible Solution

Keep #72165 open unless a maintainer explicitly consolidates its direct lookup and MCP scope into #57715 or another canonical tracker. The best implementation path is a focused session summary/index layer: persist or cache summary metadata at write time, filter/sort/limit before transcript hydration where semantics allow, provide direct lookup by `sessionId`, and change MCP `getConversation` away from a 500-row `sessions.list` scan, with regression coverage proving non-returned rows do not perform transcript reads.
