## Summary

Keep PR #50521 open. Current main still lacks explicit null-byte validation in `validateArchiveEntryPath()`, while both zip and tar extraction paths rely on that helper before resolving archive output paths. The PR is a narrow security hardening change with targeted tests, and the provided review comment supports the placement and behavior rather than showing it is obsolete or superseded.

## Best Possible Solution

Keep the PR open for maintainer/security review and likely merge or cherry-pick the small helper-level null-byte rejection with its two regression tests. The change belongs in `src/infra/archive-path.ts` because that validator is already the shared gate for zip and tar entry paths before output resolution.
