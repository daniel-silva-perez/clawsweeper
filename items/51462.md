## Summary

Keep open. Current main still has the reported gap: `handleMessageEnd` emits the final assistant stream event only for cleaned text or media, and there is no current tool-call-only branch. vLLM/OpenAI-compatible providers are documented first-class provider paths, so this belongs in core rather than ClawHub. The PR may still need maintainer refinement because empty assistant events remain invisible to some downstream relays.

## Best Possible Solution

Keep the PR open or supersede it with a maintainer patch that adds explicit tool-call-only detection in `handleMessageEnd`, covers ID-less tool-call blocks, and includes a regression test. Maintainers should also decide whether tool-only assistant updates need non-empty metadata or a marker for HTTP/ACP/UI relay consumers.
