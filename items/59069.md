## Summary

Keep open. Current main has the general plugin-backed topic parsing path, and the real Telegram plugin normalizes explicit `:topic:` targets through `parseExplicitTarget`, but the PR is specifically adding a fallback/defense-in-depth path for delivery resolution when Telegram parsing is unavailable in lightweight contexts. Current tests explicitly preserve raw plugin-owned `:topic:` targets when the plugin registry is unavailable, so this is not already implemented on main. The existing bot review also raised a small precedence concern with the PR patch, which means maintainer review or a revision is still useful rather than auto-closing.
