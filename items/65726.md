## Summary

Current main still does not implement either requested capability. ACP binding/per-agent ACP config has no `notifyPolicy`, bound ACP task creation still falls through to the task registry default of visible `done_only` notifications, and the remaining dedup logic is scoped to session/run or same Slack dispatch rather than channel/account/thread ownership across sessions.

## Best Possible Solution

Keep this issue open. Add typed `notifyPolicy` support to `AgentRuntimeAcpConfig` and/or `AgentAcpBinding.acp`, update strict Zod schemas and generated docs/help, persist/materialize the field through configured ACP bindings, and pass it into ACP task creation so `silent` can suppress bound-session completion notices. Separately add a channel/account/conversation/thread ownership guard that can detect an active ACP-bound session before another session delivers to the same Slack thread, with explicit policy and regression tests.
