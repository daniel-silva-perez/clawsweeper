## Summary

Keep open. The linked bug is still present on current main: PDF fallback extraction now runs through the bundled document-extract plugin, and that active PDF.js getDocument call still omits standardFontDataUrl. PR #70936 contains useful package-root resolution and regression-test intent, but its diff targets the older core src/media/pdf-extract.ts implementation and needs to be retargeted to the plugin-owned extractor.

## Best Possible Solution

Keep the PR open and ask for the package-root standardFontDataUrl fix to be retargeted to extensions/document-extract/document-extractor.ts, with the PdfJsModule type widened there and a plugin-level regression test in extensions/document-extract/document-extractor.test.ts. Keep #51455 open until that active plugin path passes a trailing-slash pdfjs-dist standard_fonts path into PDF.js.
