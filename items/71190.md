## Summary

Keep #71190 open. Current main still merges the full extracted orphan text into the next prompt, and a read-only simulation of the current predicate shows an orphan that already contains the queued marker would be merged again. The caller still removes the leaf after the merge result, and existing tests cover simple merge/dedupe cases but not accumulated queued-marker or abort-note snowballing. The reporter’s follow-up is also relevant: a full no-op would drop legitimate cross-channel orphaned messages, so this needs a targeted fix rather than closure.

## Best Possible Solution

Keep #71190 open. The best fix is a narrow change in `src/agents/pi-embedded-runner/run/attempt.prompt-helpers.ts` that normalizes orphan text before merging, preserving the latest legitimate orphan content while stripping previously accumulated queued markers and abort hints. Add focused tests in `attempt.test.ts` or the transcript repair contract test for the snowball case and for cross-channel orphan preservation, then run the changed agent test/typecheck lanes.
