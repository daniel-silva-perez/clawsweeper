## Summary

Keep #71190 open. Current main still prepends the raw orphan leaf text into the next prompt; it does not strip prior queued-message markers or the aborted-run note, so the reporter's snowball scenario remains plausible and unaddressed. The focused tests cover simple merge/de-dupe behavior but not repeated abort accumulation.

## Best Possible Solution

Keep the issue open. The likely fix belongs in `src/agents/pi-embedded-runner/run/attempt.prompt-helpers.ts`: preserve legitimate orphan text, but normalize/strip previously inserted queued markers and aborted-run hints before merging, then add regression coverage for repeated abort turns plus the cross-channel preservation case described in the reporter's update.
