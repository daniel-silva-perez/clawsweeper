## Summary

Keep open. Current main still unconditionally runs implicit provider discovery before consulting `models.mode`, so `replace` mode can still pay discovery cost and merge implicit plugin providers despite docs saying replace uses only configured providers. The related #68965 was closed as a duplicate of this issue, while #63552 is an overlapping performance PR that scopes discovery for explicit providers but is not a merged fix for the replace-mode skip contract.

## Best Possible Solution

Keep this issue open. The likely fix belongs in `src/agents/models-config.plan.ts`: branch on `cfg.models?.mode === "replace"` before invoking implicit provider discovery, use only explicit providers on that path, preserve existing normalization/secret enforcement behavior, and add a focused regression test proving the injected implicit resolver is not called in replace mode while merge/default mode still discovers providers. Review #63552 separately as an overlapping performance optimization, not as a substitute unless it is changed to explicitly satisfy this replace-mode contract.
