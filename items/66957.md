## Summary

Keep open. Current main still resolves implicit plugin providers before checking `models.mode`, so `replace` mode can still pay implicit discovery cost and include implicit providers despite the documented configured-providers-only contract. Related #68965 is already closed as a duplicate of this issue, and open PR #63552 is an overlapping performance optimization, not a merged fix for the replace-mode skip behavior.

## Best Possible Solution

Keep this issue open. The likely fix belongs in `src/agents/models-config.plan.ts`: branch on `cfg.models?.mode === "replace"` before invoking implicit provider discovery, use only explicit providers on that path, preserve normalization and source-managed secret enforcement, and add a focused regression test proving the injected implicit resolver is not called in replace mode while merge/default mode still discovers providers. Review #63552 separately unless it is updated to explicitly satisfy this replace-mode contract.
