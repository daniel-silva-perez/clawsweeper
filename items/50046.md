## Summary

Current `main` does not implement the PR's handshake-timeout fix. The Discord gateway still creates `ws` sockets without `handshakeTimeout`, and the existing lifecycle timeout only starts after a `Gateway websocket opened` event, so it would not recover a socket stuck forever in CONNECTING.

## Evidence

- **Current socket construction lacks handshake timeout:** `extensions/discord/src/monitor/gateway-plugin.ts:298-307` throws on empty URL, selects `ws.default`, then calls `new WebSocketCtor(url, params.wsAgent ? { agent: params.wsAgent } : undefined)`. No `handshakeTimeout` or equivalent timeout is passed.
