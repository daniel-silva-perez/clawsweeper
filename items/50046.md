## Summary

Keep open. Current main still does not implement the PR's Discord WebSocket handshake timeout, and the upstream Carbon closure does not currently remove the need for an OpenClaw-side fix: OpenClaw pins Carbon 0.16.0, Carbon's current gateway factory still constructs WebSockets without options, and OpenClaw's newer lifecycle guards cover startup readiness and opened-but-not-ready sockets, not a CONNECTING socket that never emits open/close/error.

## Best Possible Solution

Keep the PR open or replace it with an equivalent current-main patch: add a 30s `handshakeTimeout` to `extensions/discord/src/monitor/gateway-plugin.ts` while preserving the URL guard, proxy agent merge, testing WebSocket constructor, and transport activity capture. Add or update focused Discord tests so no-proxy and proxy WebSocket construction both assert the timeout. If Carbon later ships a verified release with the same behavior and OpenClaw upgrades to it, this PR can then be closed as superseded by that dependency update.
