## Summary

Keep open. Current main fixes the active-memory example by narrowing that embedded recall run to memory tools and a lightweight prompt path, but the broader requested least-privilege default is still not implemented: unset/`full` tool policy remains unrestricted, no policy still allows tools, and sub-agents remain broad unless a restrictive profile or allowlist is configured.

## Best Possible Solution

Keep this issue open for an explicit maintainer decision on least-privilege defaults. A complete solution would likely define per-agent or per-run default allowlists, preserve an explicit escape hatch such as `tools.profile: "full"`, and update config schema/help, onboarding or migration behavior, docs, and tool-policy tests across core, plugin, MCP, and subagent surfaces.
