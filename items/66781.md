## Summary

Keep open. Current main fixes the active-memory example by narrowing the blocking memory subagent to `memory_search`/`memory_get` and a lightweight prompt path, but the broader least-privilege default requested here is still not implemented: unset/`full` tool profiles remain unrestricted, no policy allows tools, and subagents still get broad tool access when no restrictive profile is configured.

## Best Possible Solution

Keep the active-memory narrowing already present on main, but leave this issue open for an explicit maintainer decision on broader least-privilege defaults. A complete fix would likely change the default tool profile or per-agent default allowlists, preserve an explicit escape hatch such as `tools.profile: "full"`, and update config docs, onboarding/migration behavior, and tool-policy tests for core, plugin, MCP, and subagent surfaces.
