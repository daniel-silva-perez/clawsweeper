## Summary

Keep open. Current main has not consolidated Vitest into a single mode-driven configuration: root vitest.config.ts delegates to test/vitest/vitest.config.ts, that config still enumerates many per-project config files, and separate top-level unit/e2e/extensions/gateway configs still exist. The issue is recent, actionable, unlabelled, and author association is NONE, so none of the conservative close reasons apply.

## Evidence

- **Root config delegates rather than consolidates:** vitest.config.ts only re-exports the default and helpers from ./test/vitest/vitest.config.ts, so it is not the single unified config described by the issue.
  - file: [vitest.config.ts:1](https://github.com/openclaw/openclaw/blob/d6ef1fcf24eea3a5985d653a3baf400e32aa1b75/vitest.config.ts#L1)

  - file: [test/vitest/vitest.config.ts:18](https://github.com/openclaw/openclaw/blob/d6ef1fcf24eea3a5985d653a3baf400e32aa1b75/test/vitest/vitest.config.ts#L18)
