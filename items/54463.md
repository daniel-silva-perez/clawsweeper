## Summary

Keep open. Current `main` updates the docs to say QMD still follows the underlying scanner behavior, but the runtime path still just hands QMD collections/path globs to `qmd update` without an OpenClaw-side cycle guard or exclusion mechanism. I found no evidence in `main` that the reported symlink-loop indexing failure has been fixed.

## Evidence
