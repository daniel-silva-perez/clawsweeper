## Summary

Current `main` does not implement pattern-based Discord guild channel allowlists. The Discord resolver only matches exact channel IDs/names/slugs plus a single `"*"` fallback, docs still describe explicit per-channel allowlisting, and status guidance still tells users to use numeric channel IDs for `channels.discord.guilds.*.channels`.

## Evidence

- **Discord channel matching only builds exact key candidates:** `buildDiscordChannelKeys()` only passes the channel `id`, `slug`, and `name` into the matcher, and `resolveDiscordChannelEntryMatch()` hard-codes only `wildcardKey: "*"`; there is no glob or pattern expansion path.
  - file: [extensions/discord/src/monitor/allow-list.ts:363](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/extensions/discord/src/monitor/allow-list.ts#L363)

  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
