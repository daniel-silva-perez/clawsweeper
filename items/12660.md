## Summary

Current `main` already ships this feature as the context-engine plugin slot: plugins register `api.registerContextEngine(...)`, users select it with `plugins.slots.contextEngine`, the runner replaces the model message array from `assemble(...)`, and post-turn capture runs through `afterTurn`/`ingestBatch`. The changelog says this landed in release 2026.3.7 via #22201.

## Risks / Open Questions

- The shipped config key is `plugins.slots.contextEngine`, not the issue's proposed `plugins.slots.context`.
- The shipped method names are `assemble` and `afterTurn`/`ingestBatch`, not the exact sketch names `buildContext` and `captureExchange`, but the behavior is present.

## Close Comment

Closing as implemented on `main`. Codex review verified that OpenClaw now ships this capability as the context-engine plugin slot rather than `plugins.slots.context`: the feature is documented in `docs/concepts/context-engine.md:10` and selected with `plugins.slots.contextEngine` (`docs/plugins/manifest.md:676`, `src/config/types.plugins.ts:24`). Plugins can register engines through `api.registerContextEngine(...)` (`src/plugins/types.ts:2005`), and the embedded runner replaces the model message array with the engine’s `assemble(...)` output before inference (`src/agents/pi-embedded-runner/run/attempt.ts:1636`). Post-turn capture is also wired through `afterTurn(...)` / `ingestBatch(...)` (`src/agents/pi-embedded-runner/run/attempt.context-engine-helpers.ts:238`). The changelog records this landing in release `2026.3.7` via `#22201` (`CHANGELOG.md:2878`).
