## Close Comment

Closing this PR as `clawhub` after a Codex review.

Current `main` now states that new skills should be published to ClawHub first and that OpenClaw will not merge new core skills when they can live on ClawHub (`VISION.md:68`, `VISION.md:101`). That matches this PR’s scope: it adds a new bundled `skills/gitlawb/SKILL.md` plus a changelog entry, but no core runtime changes.

Main already provides the external path for this work. The bundled `clawhub` skill documents `clawhub search`, `clawhub install`, and `clawhub publish` flows (`skills/clawhub/SKILL.md:23`, `skills/clawhub/SKILL.md:44`, `skills/clawhub/SKILL.md:67`), and current main also has native `openclaw skills search|install|update` support for ClawHub-backed skills (`CHANGELOG.md:2231`). There is also direct precedent for moving bundled skills out of core: `CHANGELOG.md:4146` removes the bundled `food-order` skill and tells users to manage/install it from ClawHub instead.

If you want to carry this forward, the right path now is a ClawHub-published skill/package rather than a bundled-skill PR in `openclaw/openclaw`.
