## Summary

Keep #70888 open. Current main still has the exact no-cfg early return in `resolveEntryForSessionKey()`, so persisted explicit requester session metadata cannot be read from the default on-disk session store. The PR’s central bug fix is still relevant, but it should be cleaned up before merge because its file list includes unrelated drift/test changes outside the stated subagent-depth scope.

## Best Possible Solution

Keep the PR open and ask for a scoped cleanup: retain the `src/agents/subagent-depth.ts` no-cfg default-store fix and the targeted regression tests in `src/agents/subagent-depth.test.ts` plus `src/agents/subagent-announce-delivery.test.ts`, add temp-dir cleanup, and remove or split unrelated drift changes before maintainer review.
