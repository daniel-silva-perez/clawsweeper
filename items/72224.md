## Summary

Keep open. Current main still lacks the three core behaviors this PR claims to add for non-systemd gateway restarts: native Linux listener discovery before lsof, accepting a bare `openclaw-gateway` process title, and carrying resolved probe auth into unmanaged restart health checks. The PR is recent, active, and directly tied to the closed source issue #72223, so it is not stale cleanup or an obsolete duplicate.

## Best Possible Solution

Keep #72224 open for maintainer review and validation. The right path is to land or adapt the focused fix if CI and review confirm it: add robust Linux unmanaged listener discovery, accept verified bare `openclaw-gateway` process titles, and thread resolved daemon probe auth through unmanaged restart health checks with regression tests.
