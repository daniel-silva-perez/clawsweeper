## Summary

Keep open. Current main has added explicit lifecycle metadata on several normal terminal/error branches, but it still does not satisfy the two requested acceptance points: there is no `cancelled` liveness state, user aborts without payloads still resolve to `blocked`, and an uncaught throw from `runEmbeddedAttemptWithBackend` can leave `runEmbeddedPiAgent` through the cleanup-only `finally` without setting terminal lifecycle metadata.

## Best Possible Solution

Keep #64888 open as a focused runner lifecycle bug. The best fix is still to add a `cancelled` liveness state through the runner meta/subscriber consumers and add a catch boundary around `runEmbeddedPiAgent`/`runEmbeddedAttemptWithBackend` that sets fallback terminal lifecycle metadata before rethrowing, with regression tests for a forced attempt throw and a simulated user abort.
