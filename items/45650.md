## Summary

Current main already implements the requested protection for Telegram polling: message-dispatch failures are caught and converted into a best-effort fallback reply, and update watermark persistence only advances for completed updates while failed updates block advancement until successfully retried.

## Evidence

- **Dispatch errors are swallowed in Telegram message processing:** `createTelegramMessageProcessor` wraps `dispatchTelegramMessage(...)` in `try/catch`, logs the failure, sends a fallback Telegram message, and does not rethrow, so provider errors do not escape the per-update handler.

## Close Comment

Closing from a Codex review. Current `main` already has the Telegram-side protection this issue asks for: `extensions/telegram/src/bot-message.ts:129` catches message-dispatch failures, logs them, sends a best-effort fallback reply, and does not rethrow into the polling loop. This behavior is already present in release `v2026.4.22` at commit `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`, so there is not a remaining maintainer cleanup item to track here.
