## Summary

Keep PR #58434 open. Current main does not implement the requested `/v1/responses` per-request `tool_deny` override: the strict OpenResponses schema omits `tool_deny`, the gateway runner does not accept or pass a deny override, and the agent tool factory has no per-request deny step. The PR is a plausible additive feature with useful discussion; Greptile's docs finding was acknowledged and fixed in the PR's second commit per the provided review context, so this is not a conservative cleanup close.
