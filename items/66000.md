## Summary

Keep this PR open. Current main still has the linked profile/launchd precedence bug, and the PR is a focused implementation candidate. However, the current PR behavior still needs maintainer review because it clears any nonmatching `OPENCLAW_LAUNCHD_LABEL`, including explicit custom launchd labels that current code and tests intentionally preserve.

## Best Possible Solution

Keep this PR open and request a narrower implementation: when `--profile` is explicitly supplied, ignore or clear stale OpenClaw-managed labels from another profile, while preserving explicit custom `OPENCLAW_LAUNCHD_LABEL` overrides. Add regression coverage for stale managed labels, matching managed labels, custom labels, absent labels, and the gateway status/plist-resolution path from #65643.
