## Summary

Keep open. Current main still lacks a Dockerfile-level fix for first-run Docker named-volume ownership: the runtime image runs as the non-root node user, Compose/docs mount the state/config directory at the reported container path, and config writes create temporary openclaw.json files there before rename. The provided GitHub context also shows open, unmerged PR #63959 as an implementation candidate with closing syntax for this issue, so this issue should remain open until a fix lands or that PR is closed/replaced.

## Best Possible Solution

Review and land a Dockerfile fix, or consolidate the open implementation candidates, that creates and owns the container OpenClaw state directory before USER node. Prefer preserving the existing 0700/0600 config-security expectations, add a named-volume Docker smoke or equivalent manual proof, and document the separate recovery path for already-created root-owned volumes. After that PR merges, this issue can close as implemented.
