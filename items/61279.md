## Summary

Keep open. Current main still does not pre-create or chown `/home/node/.openclaw` in the runtime image before switching to the non-root `node` user, while the Compose surface mounts that path and config writes create `openclaw.json.<pid>.<uuid>.tmp` files there. The provided GitHub context also shows open, unmerged PR #63959 as an implementation candidate linked to this issue, so the issue should remain open until a Dockerfile fix lands or that PR is closed/replaced.

## Best Possible Solution

Review and land #63959 or a replacement Dockerfile fix that creates `/home/node/.openclaw` with `node:node` ownership before `USER node`, preferably with a private directory mode consistent with the config/security expectations. Add a named-volume Docker smoke or equivalent manual proof, and document the separate recovery path for already-created root-owned volumes. After the implementation lands, this issue can close as implemented.
