## Summary

Keep open. Current main still preserves the Discord voice readiness path that matches the reported `/vc join` abort, the same-author Discord follow-up remains open, and the related high-CPU/stall report was closed in favor of this issue as the broader tracker. The agent CLI timeout path has been improved, but that partial fix does not resolve or invalidate the gateway CPU, Discord voice, or Messenger symptoms.

## Best Possible Solution

Keep this issue open as the umbrella tracker for the 2026.4.8 gateway CPU/RPC stall and downstream CLI/Messenger symptoms. Route the Discord voice portion through #63098 with a source fix in the Discord plugin, likely a documented/configurable ready/connect timeout plus bounded retry behavior. Validate Messenger only after the agent CLI and gateway request path are healthy on current main/latest release, and close this umbrella only after the CPU/RPC stall is fixed, reproduced as gone, or explicitly split into canonical narrower issues.
