## Summary

Keep open. Several post-2026.4.8 changes plausibly address parts of the gateway/CLI timeout cluster, including scoped plugin activation and QMD repair, but the mixed report cannot be conservatively closed: the Discord voice regression is still tracked as open in related issue #63098, and current main still uses the hardcoded 15s voice ready timeout that matches that report’s failure theory. A separate high-CPU report #63149 was also kept open in prior ClawSweeper review, so there is not strong enough evidence that all CPU/timeout symptoms from this umbrella issue are resolved or fully superseded.

## Best Possible Solution

Keep #63125 open until maintainers either split and close it against canonical narrower issues or verify the umbrella regression on current main/latest release. The likely path is: keep #63098 as the Discord voice connect-timeout/retry/configurability work; use #63149 or a more exact follow-up for any remaining gateway CPU saturation; verify `openclaw agent --json --timeout` against a current gateway with logs/profiles; then close this umbrella only after every sub-symptom is either fixed, unreproducible, or explicitly superseded.
