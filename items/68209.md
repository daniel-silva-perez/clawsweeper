## Summary

Keep open. Current main has important adjacent Codex hardening for route clarity, auth-profile forwarding, and cumulative-token accounting, but it does not prove the reported route-switch regression is fixed. The remaining symptoms span native Codex context accounting, compaction freshness, thread/runtime isolation, and external Codex auth-mode drift, and the related merged seam PR explicitly treated this issue as adjacent rather than closed.

## Best Possible Solution

Keep this issue open as a focused child of the broader Codex harness stability work. The best next maintainer path is to reproduce the exact current-main route switch from `openai-codex/gpt-5.4` PI/OAuth to native Codex app-server execution with a clean session, capture `/status`, app-server auth/runtime events, thread binding, and context usage before and after the switch, then either close with proof or split concrete fixes for native context freshness, stale thread binding, and Codex CLI auth isolation.
