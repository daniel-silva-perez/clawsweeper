## Summary

Keep open. Current `main` still appears to have the Windows command-resolution mismatch behind the report: Gmail setup checks for `gcloud` using PATHEXT-aware binary discovery, but the actual spawn path still invokes bare `gcloud` and only auto-appends `.cmd` for `corepack`/`pnpm`/`yarn`, not `gcloud`. I did not find a later fix in the relevant history.

## Evidence
