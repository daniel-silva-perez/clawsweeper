## Summary

Keep open. Current `main` still treats `USER.md` as a single workspace-wide bootstrap file injected for every run. The documented isolation options are per-session (`session.dmScope`) or per-agent workspaces, not per-user context files inside one shared agent, so the requested `users/<person>.md` behavior is not implemented on `main`.

## Evidence

- **Workspace bootstrap loading is fixed to one root-level USER.md:** `src/agents/workspace.ts` defines `DEFAULT_USER_FILENAME = "USER.md"`, and `loadWorkspaceBootstrapFiles()` loads a fixed root-level set of files (`AGENTS.md`, `SOUL.md`, `TOOLS.md`, `IDENTITY.md`, `USER.md`, `HEARTBEAT.md`, `BOOTSTRAP.md`, `MEMORY.md`). There is no `users/` directory scan, peer/frontmatter match, or sender-specific USER file resolution in this path.
  - file: [src/agents/workspace.ts:34](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/src/agents/workspace.ts#L34)
  - command: `nl -ba src/agents/workspace.ts | sed -n '24,40p'; nl -ba src/agents/workspace.ts | sed -n '477,560p'`
  - sha: [a59d1bd46d9f](https://github.com/openclaw/openclaw/commit/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3)
- **System prompt docs still describe global USER.md injection:** The current docs say workspace bootstrap injection includes `USER.md` and that these files are injected into the context window on every turn unless a file-specific gate applies. No per-user replacement mechanism is documented there.
  - file: [docs/concepts/system-prompt.md:99](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/docs/concepts/system-prompt.md#L99)

- **Current documented isolation model is per-agent, not per-user files within one agent:** `docs/concepts/multi-agent.md` defines an agent as owning its own workspace including `USER.md`, and later states that for one WhatsApp number with multiple people, true isolation requires one agent per person.
  - file: [docs/concepts/multi-agent.md:14](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/docs/concepts/multi-agent.md#L14)
