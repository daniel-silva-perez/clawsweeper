## Summary

Keep open. Current `main` has group-ID allowlisting and per-group/topic `groupPolicy` overrides, but it still does not implement the requested unknown-group handling flow with configurable warn/auto-leave behavior.

## Evidence

- **Docs describe only allowlist/group-policy controls for Telegram groups:** The Telegram docs document `channels.telegram.groups` as the group-ID allowlist and `channels.telegram.groupPolicy` as the sender-policy control. They do not document any unknown-group action/message/cooldown behavior.
  - file: [docs/channels/telegram.md:147](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/docs/channels/telegram.md#L147)
  - command: `nl -ba docs/channels/telegram.md | sed -n '140,210p'`
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Config surface lacks unknown-group action fields:** `TelegramAccountConfig` includes `groups`, `groupAllowFrom`, and `groupPolicy`, but there are no config fields for `unknownGroupAction`, `unknownGroupMessage`, or `unknownGroupCooldownMs`.
  - file: [src/config/types.telegram.ts:124](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/config/types.telegram.ts#L124)
  - command: `nl -ba src/config/types.telegram.ts | sed -n '80,190p'`
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Per-group/topic overrides already exist:** Group access resolution already considers topic and group overrides before account/default policy via `firstDefined(topicConfig?.groupPolicy, groupConfig?.groupPolicy, ...)`, so the report's claim that `groupPolicy = "disabled"` cannot be rescued by per-group overrides is no longer accurate on current `main`.
  - file: [extensions/telegram/src/group-access.ts:146](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/extensions/telegram/src/group-access.ts#L146)
  - command: `nl -ba extensions/telegram/src/group-access.ts | sed -n '90,180p'`
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Unknown groups are denied but only skipped/logged in inbound handling:** When chat allowlist checks fail, access returns `group-chat-not-allowed`. The main inbound handler then just logs `skipping group message` and returns, with no warning send and no leave call.
  - file: [extensions/telegram/src/group-access.ts:166](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/extensions/telegram/src/group-access.ts#L166)

- **No leave/warn config or `leaveChat` implementation found:** A repo search for `leaveChat`, `unknownGroupAction`, `unknownGroupMessage`, and `unknownGroupCooldownMs` returned no matches in Telegram code/docs/UI surfaces.
