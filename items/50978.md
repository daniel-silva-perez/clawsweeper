## Summary

Keep open. Current main still has the reported asymmetry: `enabled: false` short-circuits in `shouldIncludeSkill`, but `enabled: true` still falls through to runtime eligibility checks such as `requires.bins`, OS, env, and config. The PR remains a focused, plausibly useful fix, and its follow-up commit also addressed the review catch that skill status reporting must match runtime inclusion.

## Best Possible Solution

Keep the PR open or supersede it with an owner-approved patch that adds an `enabled: true` force-enable branch after `allowBundled`, updates `skills-status.ts` eligibility to match, and adds focused tests for force-disable, force-enable over missing bins, default runtime fallthrough, and allowBundled precedence. A short docs clarification for `skills.entries.<key>.enabled` would also remove the current ambiguity.
