## Summary

Keep open. Current main does not implement this change, and the PR’s proposed behavior conflicts with the repo’s documented and tested OPENCLAW_HOME contract by treating OPENCLAW_HOME as the config dir instead of the home root.

## Evidence

- **Current main still appends .openclaw under the resolved home:** `resolveConfigDir()` on `main` still resolves the default config directory as `path.join(resolveRequiredHomeDir(env, homedir), ".openclaw")`, with no special case for `OPENCLAW_HOME` already pointing at `.openclaw`. That means the reported nested-path case is not already implemented away on main.
  - file: [src/utils.ts:148](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/utils.ts#L148)
  - command: `nl -ba src/utils.ts | sed -n '136,156p'`
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Tests codify OPENCLAW_HOME as a home root, not a config root:** Current tests explicitly assert that `OPENCLAW_HOME=/custom/home` yields `resolveStateDir(...) === /custom/home/.openclaw` and `resolveConfigPathCandidate(...) === /custom/home/.openclaw/openclaw.json`. The PR’s "use OPENCLAW_HOME directly as config dir" behavior would break that contract for normal OPENCLAW_HOME users.
  - file: [src/config/config.nix-integration-u3-u5-u9.test.ts:53](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/config/config.nix-integration-u3-u5-u9.test.ts#L53)
  - command: `nl -ba src/config/config.nix-integration-u3-u5-u9.test.ts | sed -n '52,72p'`
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Docs describe OPENCLAW_HOME as replacing the home directory for internal path resolution:** The environment docs say `OPENCLAW_HOME` overrides the home directory used for internal path resolution of `~/.openclaw/`, agent dirs, sessions, and credentials. That matches the tested `${OPENCLAW_HOME}/.openclaw` behavior and contradicts the PR’s broader semantic change.
  - file: [docs/help/environment.md:108](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/docs/help/environment.md#L108)
  - command: `nl -ba docs/help/environment.md | sed -n '108,140p'`

- **Existing consumers assume resolveConfigDir stays under the .openclaw config root:** Plugin roots use `path.join(resolveConfigDir(env), "extensions")`, and global dotenv loading defaults to `path.join(resolveConfigDir(process.env), ".env")` while comparing against `${resolveRequiredHomeDir(...)}/.openclaw/.env`. Returning `OPENCLAW_HOME` directly would relocate these surfaces for existing installs.
