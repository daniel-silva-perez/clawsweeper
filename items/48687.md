## Summary

Keep open. Current `main` does not give strong evidence for a conservative close: Telegram runtime still treats per-group/topic `allowFrom` as an override, not as an intersection with top-level `groupAllowFrom`, so the issue's proposed suppression conflicts with the documented/runtime security model. At the same time, the current security-audit collector no longer flags per-group wildcard overrides, so the original report no longer maps cleanly to current behavior and is not clearly resolved on `main`.
