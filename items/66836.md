## Summary

Keep this PR open. Current main still does not implement a configurable sandbox initialization timeout or cooperative cancellation through sandbox startup, while the PR is a concrete implementation candidate rather than obsolete cleanup. The provided PR discussion also shows unresolved correctness review points, especially backend abort propagation and preserving the latest sandbox browser security hash epoch, so the right maintainer path is continued review rather than auto-closing.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The best maintainer path is to request/finish a revised patch that adds `agents.defaults.sandbox.initTimeoutMs` to the public config types and strict schema, wraps `resolveSandboxContext` in a bounded init timeout, threads a required/effective abort signal through docker, ssh, openshell, prune, browser startup, CDP polling, and port reads, preserves the current browser security epoch, and keeps focused regression tests such as `context.timeout.test.ts` plus updated browser behavior tests.
