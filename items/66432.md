## Summary

Keep open. Current main still has the Feishu thread-history media gap this draft PR targets: inbound current-message media is resolved into agent media payloads, but fetched thread-history messages are rendered from parsed text placeholders only, and `listFeishuThreadMessages` does not retain raw Feishu body content needed for media key extraction. The PR is a plausible focused fix, but it is still draft and the existing review comment asks for changes before merge.

## Best Possible Solution

Keep this draft PR open and finish it: retain raw Feishu message body content for thread-history items, resolve media for media-type history messages before building `ThreadHistoryBody`, inline runner-compatible attachment hints, and add both success and download-failure regression tests. After those review items are addressed, run the Feishu extension test lane and review/merge the PR rather than closing it as obsolete.
