## Summary

Keep open. Current main still gates user-facing model fallback and fallback-cleared notices behind verbose mode, tests explicitly assert that verbose-off replies suppress those notices, and the requested independent opt-in config is absent. The same-author open PR #69399 is the implementation candidate, so this issue should remain open until that PR lands or maintainers choose a different path.

## Best Possible Solution

Keep this issue open and review #69399. The best product path is a default-false agent-defaults opt-in, such as `agents.defaults.notifyOnModelFallback`, wired into `runReplyAgent` so fallback and fallback-cleared notices can be prepended when either verbose mode or the opt-in is enabled, with config schema/docs and e2e coverage updated.
