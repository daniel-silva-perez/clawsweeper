## Summary

Keep open. Current main still gates reply-visible model fallback and fallback-cleared notices behind verbose mode, the tests explicitly assert verbose-off suppression, and the requested independent opt-in config is absent. The same-author open PR #69399 is an implementation candidate using closing syntax for this issue, so this issue should remain open until that PR lands or maintainers choose another implementation.

## Best Possible Solution

Keep this issue open and review #69399. The likely product path is a default-false agent-defaults option such as `agents.defaults.notifyOnModelFallback`, wired into `runReplyAgent` so fallback and fallback-cleared notices can be prepended when either verbose mode or the opt-in is enabled, with config schema/docs and e2e coverage updated.
