## Summary

Keep PR #50463 open. Current main still does not implement a skill-specific hint for unknown tool calls, and the PR is a focused agent-runtime improvement rather than a cleanup item that can be closed. However, the current patch should not be considered ready or superseded: it wires the rewrite through `afterToolCall`, while the currently pinned `@mariozechner/pi-agent-core@0.70.2` still returns immediate `Tool X not found` results before `afterToolCall` for missing tools, so the central path needs revised wiring, an upstream dependency update with proof, or both.

## Best Possible Solution

Keep the PR open but require revision before merge. The best path is to implement the hint on the actual current missing-tool result path, or update `@mariozechner/pi-agent-core` to a version that provably calls `afterToolCall` for immediate missing-tool results, then add regression coverage for skill-name tool calls, non-skill unknown tools, tool/skill collisions, preserved prior after-tool behavior, and names containing punctuation or sanitized command forms.

## Risks / Open Questions
