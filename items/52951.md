## Summary

Keep open. Current main still only supports the binary `tools.fs.workspaceOnly` policy; it does not ship the proposed `tools.fs.roots` schema, policy resolution, host tool guards, root-scoped media reads, or deny-all empty-root semantics. The PR is active and potentially useful, but the provided review context still shows recent unresolved filesystem/security review concerns, so it needs maintainer review and narrowing rather than conservative cleanup closure.

## Best Possible Solution

Keep the PR open, but ask maintainers to decide whether `tools.fs.roots` is the desired core config contract. If yes, narrow the implementation around config/schema docs, host read/write/edit/apply_patch enforcement, media/outbound propagation, and focused tests; resolve the remaining P1/P2 review threads before merge. If maintainers want a broader filesystem sandboxing model instead, use the existing filesystem sandboxing discussion as the product-level follow-up rather than closing this as obsolete.
