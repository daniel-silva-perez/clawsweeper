## Summary

Keep this PR open. Current main still does not implement the proposed `google-genai` provider, `google-genai-sdk` auth mode, or `models.providers.google-genai.vertexai` config contract. The PR is still a plausible implementation path for unified Google AI Studio / Vertex AI auth, but it changes shared provider/auth contracts and still needs maintainer review plus correctness fixes before merge.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to either revise it so unified Google GenAI auth fits the current bundled Google plugin/provider architecture and passes the existing Google transport behavior bar, or supersede it with a narrower PR that fixes the remaining `google-vertex` ADC/proxy behavior in the existing provider path.

## Risks / Open Questions

- The PR changes shared model-provider config and auth contracts, so it needs maintainer design review rather than automated cleanup closure.
- The proposed keyless Google SDK auth mode must be propagated consistently across embedded runs, `/btw`, compaction, media/tool callers, and every `requireApiKey` call site before merge.
- The new SDK stream path must preserve existing Google transport behavior for system prompts, tools, generation controls, cancellation, usage, and finish metadata.
- The PR appears to import `@google/genai` from core without a matching root dependency declaration in the provided file list, which could become a build/package-resolution issue.
