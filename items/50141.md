## Summary

Keep open. Current main appears to have shipped the core webchat dropdown/provider-qualified picker fix, but it still does not contain this PR's `/model` slash-command qualification changes. The PR is therefore not safely auto-closeable as fully implemented or duplicated.

## Best Possible Solution

Keep the shipped main implementation for the dropdown path, but either land or port the narrow `/model` slash-command improvements from this PR: qualify current and available model refs from session/catalog provider data, keep the OpenRouter double-prefix guard, and avoid inferring a session provider from defaults. If maintainers do not want that behavior in webchat slash commands, close manually with that product decision rather than as automated cleanup.
