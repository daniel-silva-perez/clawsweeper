## Close Comment

Closing from a Codex review because current `main` already implements this fix. The system prompt no longer embeds a changing per-turn clock string (`src/agents/system-prompt.ts:248` now emits only `Time zone: ...`), cache-sensitive prompt composition keeps volatile context like `HEARTBEAT.md` below an explicit cache boundary (`src/agents/system-prompt.ts:54`, `src/agents/system-prompt.ts:945`), and the regression tests lock in that the formatted date/time must not appear in the system prompt (`src/agents/system-prompt.test.ts:497`). The shipped behavior is also documented in the prompt-caching docs (`docs/reference/prompt-caching.md:164`) and changelog (`CHANGELOG.md:1400`, under `2026.4.5`). Given that evidence, plus the reporter’s April 6 note that current versions seemed fixed again, this no longer needs to stay open as an active bug on `main`.
