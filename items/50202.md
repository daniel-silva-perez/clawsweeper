## Summary

Current `main` already supports heartbeat session routing and isolated heartbeat execution. The config schema accepts both `agents.defaults.heartbeat.session` and `isolatedSession`, the runner resolves explicit heartbeat session keys, and isolated runs execute in a dedicated `:heartbeat` session with regression coverage. The issue's documented-feature gap is no longer present on `main`.

## Risks / Open Questions

- Current main intentionally rejects subagent session keys for heartbeat routing and falls back to the main session, so existing subagent transcripts are still not a supported heartbeat target.

## Close Comment

Closing as implemented on current `main` after Codex review. OpenClaw now supports heartbeat session routing via `agents.defaults.heartbeat.session` and also supports isolated heartbeat execution via `agents.defaults.heartbeat.isolatedSession`. The current schema accepts both keys (`src/config/zod-schema.agent-runtime.ts:31-43`), the runner resolves explicit heartbeat session keys (`src/infra/heartbeat-runner.ts:312-372`), and isolated runs execute in a dedicated `:heartbeat` session (`src/infra/heartbeat-runner.ts:798-923`). The docs now describe both options (`docs/gateway/heartbeat.md:42`, `docs/gateway/heartbeat.md:227-230`), and regression tests cover them (`src/infra/heartbeat-runner.model-override.test.ts:209-235`, `src/infra/heartbeat-runner.returns-default-unset.test.ts:866-968`). One scope note: heartbeat routing to existing subagent session keys is still intentionally blocked on `main`, so if someone specifically wants heartbeats to run inside active subagent transcripts, that would need a new narrowly scoped request.
