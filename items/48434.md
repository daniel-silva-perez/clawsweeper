## Summary

Current `main` still lacks the broadcast `inbound_claim` dispatch path. The hook runner supports `runInboundClaim(...)`, but `dispatchReplyFromConfig(...)` only invokes the targeted `runInboundClaimForPluginOutcome(...)` path for plugin-owned bindings, and the tests explicitly assert that non-bound conversations do not broadcast inbound claims.

## Evidence

- **Dispatch only targets plugin-owned bindings:** In the inbound dispatch path, `dispatchReplyFromConfig` calls `hookRunner.runInboundClaimForPluginOutcome(...)` only inside the `pluginOwnedBinding` branch, then falls through to `message_received` and normal agent dispatch. There is no call to `hookRunner.runInboundClaim(...)` before agent dispatch for general conversations.
