## Summary

Keep open. Current `main` partially mitigates the report by compacting the skills list and documenting the prompt budget, but it still drops skills when the compact block overflows, does so by taking an alphabetized prefix with no pinning/prioritization, and only injects a warning into the prompt text rather than adding a separate operator-facing surface.

## Evidence

- **Overflow still drops skills from the prompt:** `applySkillsPromptLimits()` first caps by `maxSkillsInPrompt`, then if compact mode still does not fit it binary-searches the largest fitting prefix and slices `skillsForPrompt = skillsForPrompt.slice(0, lo)`. The prompt-visible list is alphabetically sorted before this, so later-named skills can still disappear with no priority handling.
