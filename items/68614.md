## Summary

Keep this PR open. Current main only partially overlaps with the branch: it already falls back to a new Google Chat message when updating the typing message fails, but it does not implement the PR's 25-second timeout/interim fallback behavior. The PR is a plausible focused Google Chat bug fix, with active author follow-up, but it needs a refresh against current main's extracted reply-delivery helper and should address the remaining timer-lifecycle review issue before merge.

## Best Possible Solution

Keep the PR open as an implementation candidate, but ask for a refresh against current main: port the timeout behavior into the current Google Chat monitor/reply-delivery split, remove the unrelated issue-number comments, clear the timer in a dispatch `finally`, and add targeted tests for expiry, final delivery after expiry, and no spurious fallback when dispatch exits without delivery.
