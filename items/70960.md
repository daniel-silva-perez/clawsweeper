## Summary

Current main only partially addresses the bundled report. The Agents page selector now prefers `agents.list[].name`, but the issue also reports a QClaw Windows upgrade deletion path that is not present in this repository and a Control UI topbar token/cost display regression that still appears missing. Session rows expose usage fields, but the topbar renders only mobile/theme controls and the chat token chip is only a draft input estimate.

## Best Possible Solution

Keep this issue open or split it into focused follow-ups. Route the destructive QClaw Windows upgrade/install report to the QClaw Electron installer owner with the supplied log and registry details. In OpenClaw Control UI, make all agent/session display surfaces consistently prefer `agents.list[].name` before identity-derived names, then restore the topbar current-session token/cost/cache usage display or intentionally replace it with an equivalent visible usage surface backed by the existing session usage fields.
