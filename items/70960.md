## Summary

Keep open. Current main appears to cover the configured agent-name part: the gateway includes `agents.list[].name`, and Control UI labels prefer that name before identity/id fallback. But the item also reports a QClaw Windows/Electron upgrade installer deletion problem that is not represented by this repository's OpenClaw installer, and the follow-up token-usage complaint still asks for a top-right status display; current main only exposes token/context details in message Context disclosure and the Usage view, while the topbar status area renders chat/mobile/theme controls.

## Best Possible Solution

Split the item into actionable tracks: route the QClaw Electron/Windows installer deletion report to the QClaw packaging owner with the provided install log details; treat the OpenClaw agent-name complaint as already covered by current main/release behavior; and either restore a concise session token/cost summary in the Control UI topbar or make a maintainer product decision that the current Context disclosure plus Usage tab is the intended replacement and document/comment that decision.
