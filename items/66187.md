## Summary

Keep #66187 open. Current main still lists endedAt-set orchestrator runs as active when they have pending descendants, and the same predicate still counts those runs toward the per-session active-child quota. The adjacent stale-unended-run fix for #71252 does not resolve this endedAt-plus-descendants case.

## Best Possible Solution

Keep the issue open for maintainer product/behavior work. The likely fix is to split user-visible liveness and per-session quota from descendant cleanup/announce tracking: endedAt-set parent runs should stop consuming `maxChildrenPerAgent` and should not appear as ordinary active work, while descendant relationships still remain available for cleanup, cascade stop, and completion delivery. Add focused regression coverage for endedAt-set parents with pending descendants and coordinate with the cleanup-pending semantics tracked around #57041 if that PR is revived.
