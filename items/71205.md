## Summary

Keep PR #71205 open. Current main still hardens the state/config directory to `0o700` on config writes and has no `OPENCLAW_SKIP_STATE_DIR_HARDEN` escape hatch, while the PR is a focused core config I/O fix with a regression test and an addressed bot-review suggestion to log accepted env usage.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a narrow core fix if maintainers accept the opt-in env escape hatch: add `OPENCLAW_SKIP_STATE_DIR_HARDEN` in `src/config/io.ts`, log accepted usage via `logAcceptedEnvOption`, keep the default `0o700` hardening behavior, and retain the regression test for preserving a setgid group-writable state dir.
