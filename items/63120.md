## Summary

Keep open. Current main partially mitigates this by hiding ACP-only fields when no ACP backend is available, but when ACP affordances are advertised it still exposes `streamTo` without ACP-only schema guidance and still rejects `runtime="subagent"` calls that include `streamTo:"parent"`. The linked fixes (#63121 and #68397) are described as open/unmerged in the provided context and their referenced commits are not present in this checkout.

## Best Possible Solution

Keep #63120 open until main lands an explicit fix. The best path is to either land a corrected version of #68397 that strips or ignores ACP-only fields for `runtime="subagent"`, or implement per-runtime schema/description guidance that reliably prevents providers from sending `streamTo` to native subagents. The regression test should cover a `sessions_spawn({ runtime:"subagent", streamTo:"parent" })` call through the current tool contract and assert that it no longer blocks subagent spawning, while preserving ACP `streamTo` behavior.
