## Summary

Current `main` does not implement the PR's multi-agent dropdown fix. The webchat session selector still builds options only from `sessions.list` rows plus the current `sessionKey`, while the backend `sessions.list` response still comes only from stored session entries. That means configured agents without a current/recent store row can still disappear from the dropdown, so closing as implemented or cannot-reproduce would be unsupported.

## Evidence

- **Dropdown builder still omits configured-agent main sessions:** `resolveSessionOptionGroups` populates options from `sessions?.sessions` and then adds only the current `sessionKey`. It never seeds one main-session option per configured agent. `state.agentsList` is only used later for group labeling.
