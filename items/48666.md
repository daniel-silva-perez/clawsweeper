## Summary

Keep #48666 open for maintainer triage. Current main still builds the chat session selector from `sessions.list` rows plus the active session key, not from all configured agents, so the underlying dropdown gap remains plausible even though linked issue #48634 has since been closed. The submitted branch is not merge-ready because it bundles unrelated risky changes, but the core UI fix should be extracted or reimplemented narrowly.

## Best Possible Solution

Keep this PR open as a useful but non-merge-ready signal. The best fix is a narrow current-main patch in `ui/src/ui/chat/session-controls.ts` that seeds one main-session option for every configured agent from `state.agentsList.agents` using the existing main-key/session-key helpers, dedupes against `sessions.list`, and adds focused regression tests for missing session rows and duplicate prevention. The author or maintainer should split away the unrelated Slack, JSON-file retry, model-picker/mobile-control, and log-file changes.
