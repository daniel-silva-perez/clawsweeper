## Summary

Keep open. Current `main` has protections for duplicate provider events, idle/stalled streams, manual chat aborts, and repeated tool-call loops, but it does not implement the requested stream-output repetition safeguard that counts repeated assistant phrases/chunks, halts generation, and requires manual resume/cancel confirmation.

## Risks / Open Questions

- This is an absence-of-feature conclusion; a hidden or externally provided implementation could exist outside the searched codepaths, but I found no docs, config, commands, or runtime hooks that expose the requested safeguard on current `main`.
- Current main does have adjacent protections (duplicate stream-event handling, idle timeout, tool-loop detection, manual `chat.abort`), so someone could mistakenly read those as covering this request even though they do not halt on repeated assistant output text.
