## Summary

Keep this PR open. Current main still contains the exact locked `skipCache: true` reads the PR removes, and `skipCache` still bypasses the object cache before synchronous disk read and JSON parse. The PR is a plausible focused performance fix, strengthened by the Greptile review and the related live symptom report in #70857, but it also intersects with an existing external-writer/stale-cache test contract that maintainers should explicitly accept, replace, or narrow before landing.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to remove the redundant locked-path disk parse if locked session-store updates are the supported write contract, or introduce a lock-aware fast path that avoids repeated large synchronous parses while preserving whatever external-writer protection maintainers still require. Add or update focused coverage for queued locked writes and the accepted stale-cache edge case before landing.
