## Summary

Keep open. Current main still does not render direct tool-result image blocks shaped as `{ type: "image", data, mimeType }` inline in Control UI chat, and #50779 remains open for that user-visible bug. PR #52025 is stale against the current renderer, but it is a focused implementation attempt and its later commits addressed the bot review concerns around image-only results, nested tool-result content, preserving nested text, and keeping images outside collapsed tool output.

## Best Possible Solution

Keep #52025 open for maintainer review or rework. The best path is to adapt the idea to current `ui/src/ui/chat/grouped-render.ts` and `ui/src/ui/chat/tool-cards.ts`, using the existing `appendImageBlock`/`buildBase64ImageUrl` helpers, render recognized tool-result images outside the collapsed tool output by default, preserve nested tool-result text, and add focused `grouped-render` tests for direct, image-only, and nested `{ type: "image", data, mimeType }` cases. If maintainers prefer not to carry this PR, #50779 should remain the canonical bug until another implementation lands.
