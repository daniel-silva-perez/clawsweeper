## Summary

Keep open. Current main still starts a Google Chat typing message before dispatch and only clears it from the delivery path; exact silent `NO_REPLY` final payloads can be dropped before channel delivery, and Google Chat does not wire the existing typing cleanup callback. The related open PR #39891 is still the concrete fix path rather than evidence that this issue is resolved.

## Best Possible Solution

Keep #39843 open and land PR #39891 or an equivalent Google Chat monitor fix that deletes any remaining `typingMessageName` after dispatch completes with no delivery, or wires the existing `onTypingCleanup` hook. Add a focused regression test for the reaction-plus-`NO_REPLY` path so the typing message cannot persist again.
