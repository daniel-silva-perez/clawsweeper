## Summary

Keep open. Current main has partial Copilot mitigation through reasoning-disabled default Copilot model metadata, but the transport code still maps Opus 4.7 `minimal`/`low` adaptive thinking to `output_config.effort: "low"` whenever a reasoning-capable Anthropic Messages model reaches that path. The linked bug #70322 is still plausibly actionable, and related #69928 raises a broader accepted-effort ambiguity that maintainers should reconcile before closing or landing this PR unchanged.

## Best Possible Solution

Keep the PR open and ask for maintainer/product review of the exact Copilot Opus 4.7 effort contract. The likely implementation belongs in the Anthropic Messages transport or a provider-owned compatibility policy with tests for `minimal`, `low`, and any confirmed unsupported higher efforts, while preserving the current default `reasoning: false` Copilot mitigation.
