## Summary

Current main still drops Mattermost `post_edited` WebSocket events before they can reach Mattermost mention detection and agent wake routing. This is a bundled Mattermost plugin behavior bug, not a ClawHub cleanup or duplicate of the related WebSocket keepalive/rawBody/plugin items.

## Best Possible Solution

Keep #71930 open and fix it in `extensions/mattermost`: accept `post_edited` events carrying a post payload, route qualifying edits through the existing mention gate and enqueue path, make replay dedupe edit-aware so an originally unmentioned post can wake after a later mention edit without re-waking on cosmetic edits, and add focused `monitor-websocket` plus mention/replay regression tests.
