## Summary

Keep open. Current main has only a partial generic mitigation for all-media-dropped cases, while the PR’s remaining product problem still exists for mixed valid/invalid media and catastrophic media-normalization failures. The related #70109 is parallel work, not a clearly canonical superseding PR in the provided discussion.

## Best Possible Solution

Keep this PR open for maintainer review or explicit consolidation with #70109. The best code direction is a narrow core fix that preserves dropped media through reply normalization, routing, outbound delivery, JSON/mirror projections, and followup/block paths, with sanitized user-visible notices for both partial and catastrophic media-normalization failures.
