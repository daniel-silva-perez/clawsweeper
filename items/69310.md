## Summary

Keep open. Current main only covers the narrow case where every normalized reply media item fails, using a generic warning string. It still lacks this PR's per-item dropped-media contract, sanitized user-visible notices, JSON/outbound/transcript propagation, and routing/followup preservation for dropped-media-only payloads. The related #70109 PR is closed and unmerged, so it does not supersede this active PR.

## Best Possible Solution

Keep this PR open. The best path is a maintainer-reviewed implementation that preserves sanitized per-item media drop details through reply media normalization, catastrophic fallback, block/routing/followup filters, outbound JSON/delivery, and transcript mirroring, while retaining the privacy and delivery-safety fixes already discussed in the review thread.
