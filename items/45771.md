## Summary

Keep open. Current `main` already exposes Anthropic usage/quota windows in `/status`, `session_status`, `openclaw status --usage`, `openclaw models status`, and `usage.status`, but I found no implementation of the issue's pace projection tiers or automatic autonomous-work throttling/blocking. This is partial overlap, not a full fix.

## Evidence

- **Anthropic quota windows are implemented:** Current main fetches Anthropic usage from dedicated usage endpoints, normalizes `five_hour` and `seven_day` windows, and returns them as provider usage snapshots.
  - file: [src/infra/provider-usage.fetch.claude.ts:19](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/infra/provider-usage.fetch.claude.ts#L19)
- **Status/session_status already surface provider usage:** `buildStatusText()` loads provider usage and formats it as a `📊 Usage:` line, and `session_status` calls that shared status builder for its output.
  - file: [src/status/status-text.ts:192](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/status/status-text.ts#L192)
- **session_status wires through the shared status card:** `session_status` invokes `buildStatusText()` with transcript usage enabled, so the issue's requested status visibility is already partially covered.
  - file: [src/agents/tools/session-status-tool.ts:546](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/agents/tools/session-status-tool.ts#L546)
- **Models status also shows usage summaries:** `openclaw models status` loads provider usage summaries and appends them per provider in the OAuth/token status section.
  - file: [src/commands/models/list.status-command.ts:609](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/commands/models/list.status-command.ts#L609)
- **Docs describe current scope as provider usage windows only:** The usage-tracking docs say OpenClaw pulls provider usage/quota from usage endpoints and shows normalized `% left` windows in `/status`, `session_status`, CLI status, and the menu bar.
  - file: [docs/concepts/usage-tracking.md:11](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/docs/concepts/usage-tracking.md#L11)
- **No pace-aware throttling found in subagent spawn guards:** Current subagent spawn protection is depth/child-count based (`maxSpawnDepth`, `maxChildrenPerAgent`). I found no usage-based pacing or throttle thresholds in the spawn path.
  - file: [src/agents/acp-spawn.ts:725](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/agents/acp-spawn.ts#L725)
- **Targeted search found no relevant pace config or tier implementation:** Searching current main for `pace`, `subagentThreshold`, `notifyOnChange`, `injectContext`, `comfortable`, `pressing`, and `runaway` under agent/status/config/docs paths produced no relevant implementation hits for this feature.
  - command: `rg -n '\bpace\b|subagentThreshold|notifyOnChange|injectContext|comfortable|pressing|runaway' src/agents src/status src/infra src/config docs/cli docs/concepts docs/gateway`

## Risks / Open Questions

- The item partially overlaps shipped usage visibility, so future work should be scoped to pace projection and autonomous throttling rather than raw usage display.
- A changelog note mentions 'provider usage headers', but the current Anthropic implementation is usage-endpoint based; that should not be mistaken for the requested header-capture and pacing feature.
