## Summary

Keep #51823 open. Current `main` does not implement the PR's central session-health lifecycle feature: there are no `session-health-*` infra modules, no derived `sessionHealthSurface` on health snapshots, no Mission Control session-health card, and `openclaw sessions cleanup` still has only the existing maintenance preview/enforce path without remediation plan execution. The PR is not ready to merge as-is because it is XL, mixed with now-divergent pre-commit hook work, and the provided review context still includes unresolved P1 concerns, but it remains unique product work rather than an implemented, duplicate, ClawHub-only, or non-actionable item.

## Best Possible Solution

Keep the PR open for maintainer/product judgment, but ask for a rebase and split. The strongest path is to separate session-health visibility/collector work from manual remediation execution and drop or realign the pre-commit hook changes with current hook policy. Before merge, the branch should address the remaining P1 review items against current archive timestamp, `sessionFile`, multi-agent, and `resetArchiveRetention` contracts.
