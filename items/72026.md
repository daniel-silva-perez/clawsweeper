## Summary

Current main still has the duplicated #72019 behavior. PR #72026 is a focused alternate fix that removes the task from the system prompt, and the Greptile review found it generally correct with only a dead-parameter cleanup. But overlapping PR #72053 is an active alternate in the opposite direction, and existing ClawSweeper context treats these as competing open fixes rather than a landed canonical replacement. Maintainers still need to choose the prompt contract.

## Best Possible Solution

Keep #72026 open until maintainers choose the canonical #72019 fix. The best path is to preserve exactly one long-form task copy in the first model request, preserve multiline and structured task content, add regression coverage across both prompt assembly surfaces, and then close the unchosen alternate PRs once a canonical design lands or is explicitly selected.
