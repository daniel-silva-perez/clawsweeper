## Summary

Keep open. Current main still lets a model-supplied exec `security` argument override the operator/session setting through `minSecurity(...)`, so the bug this PR targets is still present. The PR is a focused implementation candidate, but it needs maintainer review cleanup before merge.

## Best Possible Solution

Keep this PR open and review it as a focused bug fix. The best landing shape is a minimal change in `src/agents/bash-tools.exec.ts` that makes configured/session exec security authoritative for model tool calls, preserves the explicit elevated-full override, adds regression coverage for configured `full` with model `allowlist` and configured `allowlist`/`deny` with model attempts to change policy, and removes or documents the now-ignored model-call `security` parameter plus any unused `maxSecurity` helper.
