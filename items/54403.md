## Summary

Current `main` does not contain the proposed rollover fix. `createSubsystemLogger` still caches a child file logger once and reuses it forever, while the base logger can rotate when `getLogger()` rebuilds on a changed dated file path, so the reported stale-file behavior remains plausible. There is also no current test on `main` that exercises subsystem file-logger rebuild on base logger rollover.

## Evidence

- **Subsystem logger still caches child once:** `createSubsystemLogger` keeps `fileLogger` in a closure and only initializes it on first file emit; subsequent emits reuse the same child logger without checking whether the base logger changed.
  - file: [src/logging/subsystem.ts:327](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/logging/subsystem.ts#L327)
