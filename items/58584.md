## Summary

Keep #58584 open. Current main has cron error backoff after failed runs, and Ollama provider discovery can report an unreachable Ollama host, but I did not find a scheduler-time pre-flight that checks an agent's configured model provider before scheduling/accepting cron jobs or a periodic disabled-until-provider-online retry loop. The issue is recent, non-maintainer-authored, coherent, and still plausible as a core reliability request.
