## Summary

Current `main` already fixes this: heartbeat runs pass `heartbeat.model` as an explicit per-run override, reply selection prefers that heartbeat override over stored session model state, and live-session switching is now gated behind the explicit `liveModelSwitchPending` flag. The same implementation is present in release `v2026.4.23`.
