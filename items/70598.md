Keep PR #70598 open. Current main already shipped the central exec heartbeat prompt fix, but the PR still contains useful follow-up work not present on main: preserving deferred non-exec system events, including duplicate reminders, after an exec-event heartbeat consumes the inspected queue.

## Best Possible Solution

Keep #70598 open or ask the author to rebase/narrow it to the remaining behavior: after an exec-event heartbeat handles only exec-classified entries, preserve deferred non-exec pending events without routing them through consecutive-duplicate suppression. The final patch should keep focused tests for mixed exec+cron queues and duplicate deferred reminders, then close this PR only after that work lands or a canonical replacement tracks it.
