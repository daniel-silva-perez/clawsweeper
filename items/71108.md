## Summary

Keep open. Current main has history-side commentary filtering, but it does not implement this PR's live/history parity fixes for chat.inject broadcasts, live assistant deltas, unrelated session-scoped tool events, or unrelated terminal chat-event reloads. The PR discussion is also active and the review feedback about maxChars appears addressed in the submitted diff, so this is not obsolete cleanup.

## Best Possible Solution

Keep the PR open and continue normal maintainer review. The best path is to rebase on current main, preserve the core fixes that apply history-grade sanitization to chat.inject, suppress commentary-phase live chat output using shared phase resolution, and guard UI stream/history resets by the active run. Split or justify the unrelated Google Meet/QA-lab adjustments before merge.
