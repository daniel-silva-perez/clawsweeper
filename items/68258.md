## Summary

Keep open. Current main still lacks a top-level auth reload rule, so auth.profiles.* remains unmatched and falls into the conservative restart path. The issue is also paired with open PR #68257 from the same author, so the maintainer path is to review/land or reject that PR rather than close this issue early.

## Best Possible Solution

Keep this issue open and route maintainer attention to PR #68257. The likely fix belongs in the gateway reload planner by explicitly classifying top-level auth.profiles.* as a no-op while preserving gateway.auth.* as a restart-triggering gateway config path, with regression coverage for both behaviors.
