## Summary

Current `main` intentionally rejects loopback-source requests in `gateway.auth.mode="trusted-proxy"`, covers the exact localhost cases from this PR with fail-closed tests, and documents same-host loopback proxies as unsupported for trusted-proxy auth. That means the PR is not already implemented on `main`, but there is no supported close reason with high-confidence evidence.

## Evidence
