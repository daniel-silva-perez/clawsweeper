## Summary

Keep PR #50777 open. Current main still renders the Web UI code-block copy button with both `Copy` and `Copied!` spans in the HTML and relies only on CSS to hide `Copied!`, so the linked `CopyCopied!` failure is not implemented on main. The PR remains a relevant focused fix, though it should likely be revised to avoid globally allowlisting `hidden` in DOMPurify as noted in review.

## Best Possible Solution

Keep the PR open, but request a small revision: preserve the user-visible fix and regression test while avoiding a global `hidden` allowlist, for example by applying `hidden` only to generated `.code-block-copy__done` elements after sanitization or through a tightly scoped trusted-renderer path. Then verify both fenced and indented code blocks and the copy-state toggle.
