## Summary

Keep #71682 open. Current main has a real operator-facing fix for long idle model calls: `agents.defaults.llm.idleTimeoutSeconds` is accepted, honored by the embedded runner, documented, and covered by CLI write tests. But the issue also says the LLM cannot change the timeout, and current main still blocks model-facing `gateway config.patch/apply` edits to `agents.defaults.llm.idleTimeoutSeconds`. The timeout/fallback cooldown behavior is also still represented by open #51057 rather than fully resolved here.

## Best Possible Solution

Keep open for maintainer triage. The likely path is to reply with the operator workaround (`openclaw config set agents.defaults.llm.idleTimeoutSeconds <seconds>`, or `0` to disable) and decide whether the owner-only agent-facing gateway config tool should allow bounded edits to `agents.defaults.llm.idleTimeoutSeconds`. Keep the auth-profile timeout cooldown/faster fallback decision in the existing #51057 thread unless #71682 adds a distinct repro.
