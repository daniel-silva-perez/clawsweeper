## Summary

Keep #7433 open. Current main (4ad8b613c9b60466224d9b3737fff7d63ada652a) still does not implement the requested WhatsApp/Baileys reliability work: the bundled WhatsApp plugin creates the socket without `getMessage` or `cachedGroupMetadata`, has no `groups.upsert`/`groups.update`/`group-participants.update`/`lid-mapping.update` handlers, and only has narrower on-demand group metadata plus LID lookup behavior.

## Best Possible Solution

Implement the reliability work in the bundled `@openclaw/whatsapp` plugin: add bounded retry message storage and pass `getMessage`, share a group metadata cache through `cachedGroupMetadata`, update it from hydration plus `groups.upsert`/`groups.update`/`group-participants.update`, and decide whether current Baileys `signalRepository.lidMapping` persistence is sufficient or an explicit `lid-mapping.update` persistence handler is still needed. Add focused WhatsApp plugin tests covering socket options, cache seeding, event updates, and retry-message lookup behavior.
