## Summary

Keep PR #50975 open. Current main still rejects prefixless Anthropic setup tokens and keeps the 80-character minimum, so the central requested behavior is not implemented. The PR is a plausible focused fix, but it needs revision before merge because the submitted condition would let too-short legacy-prefixed tokens pass and it now targets an old wrapper path rather than the shared validator location.

## Best Possible Solution

Keep the PR open but request a refreshed patch against `src/plugins/provider-auth-token.ts`: lower the minimum only if the new Anthropic token shape is confirmed, accept prefixless tokens that meet the minimum, keep an unconditional too-short guard for all token formats, and add focused validator tests plus any necessary updates to live/QA setup-token helpers that currently assume the `sk-ant-oat01-` prefix.
