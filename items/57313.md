## Summary

Keep open. Current `main` does not expose `sessionModelOverride` or `sessionProviderOverride` on `PluginHookAgentContext`, and `before_model_resolve` still runs with a pre-session context that cannot directly tell a plugin whether a user-pinned session override is active.

## Evidence

- **Hook context type still omits requested fields:** `PluginHookAgentContext` currently includes `runId`, `agentId`, `sessionKey`, `sessionId`, `workspaceDir`, `modelProviderId`, `modelId`, `messageProvider`, `trigger`, and `channelId`, but no `sessionModelOverride` or `sessionProviderOverride`.
  - file: [src/plugins/hook-types.ts:155](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/plugins/hook-types.ts#L155)
  - command: `nl -ba src/plugins/hook-types.ts | sed -n '150,220p'`
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **Hook context builder cannot supply override fields:** `buildAgentHookContext()` only forwards `runId`, `agentId`, `sessionKey`, `sessionId`, `workspaceDir`, `messageProvider`, `trigger`, and `channelId`; there is no path to populate session override metadata for hooks.
  - file: [src/agents/harness/hook-context.ts:14](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/agents/harness/hook-context.ts#L14)
  - command: `nl -ba src/agents/harness/hook-context.ts | sed -n '1,220p'`
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **`before_model_resolve` is still invoked with the limited hook context:** `runEmbeddedPiAgent()` constructs `hookCtx` with provider/model ids and passes it into `resolveHookModelSelection()`, which then calls `runBeforeModelResolve(event, params.hookContext)` before model resolution. No session override fields are attached.
  - file: [src/agents/pi-embedded-runner/run.ts:324](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/agents/pi-embedded-runner/run.ts#L324)
  - command: `nl -ba src/agents/pi-embedded-runner/run.ts | sed -n '320,390p'`

- **Docs still describe `before_model_resolve` as pre-session routing only:** Current docs say `before_model_resolve` runs pre-session to override provider/model before model resolution; they do not document hook access to session-pinned override state.
  - file: [docs/concepts/agent-loop.md:93](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/docs/concepts/agent-loop.md#L93)
