## Summary

Keep this issue open. Current main still does not implement the documented Discord elevated fallback from `channels.discord.allowFrom`, `sandbox explain` still computes elevated allowlists only from `tools.elevated.allowFrom.<channel>`, and open PR #53259 remains the linked implementation candidate rather than a reason to close before merge.

## Best Possible Solution

Review and either land, revise, or close PR #53259. The fix should wire a Discord-owned `channels.discord.allowFrom` fallback into the elevated runtime path and make `sandbox explain` use the same effective allowlist resolution, with regression coverage for omitted `tools.elevated.allowFrom.discord`, explicit empty override behavior, and diagnostics consistency.
