## Summary

Keep open. Current main still only supports normal reply workspace selection at the agent level: route bindings do not accept `workspace`, resolved routes do not carry a binding workspace, and reply execution resolves cwd from the selected agent. This PR is a plausible core routing/config feature, not ClawHub work or an obsolete duplicate. The discussion is also material: the author addressed the initial uppercase peer-id bug in the PR head, but later review comments still identify remaining correctness issues around full binding scope and agent-id normalization before merge.

## Best Possible Solution

Keep this PR open as the implementation candidate. The best path is to revise it so `AgentRouteBinding` and the strict config schema expose an optional `workspace`, `resolveAgentRoute` carries the matched binding workspace through `ResolvedAgentRoute`, and reply execution consumes that resolved value instead of re-scanning bindings. Add config/help/docs updates plus targeted tests for peer, account/channel/team/guild scoped bindings, uppercase peer IDs, case-insensitive agent IDs, and fallback to the agent-level workspace.
