## Summary

Keep #38309 open. Current main still does not support SecretRef values in `sandbox.docker.env`: schema, generated schema, public types, secret collection/target registry, docs, and Docker launch handling remain string-only for this surface. PR #38310 is useful prior implementation context, but it was closed unmerged and is not shipped on main.

## Best Possible Solution

Keep #38309 open until OpenClaw implements SecretRef support end-to-end for Docker sandbox env values: accept `SecretInput` in `SandboxDockerSchema` and `SandboxDockerSettings.env`, keep resolved runtime Docker config as `Record<string, string>`, collect and resolve default/per-agent `sandbox.docker.env` refs through the secrets runtime, add target-registry coverage for audit/plan/configure, update docs/generated schema, and add focused schema/runtime/launch regression tests. PR #38310 remains useful prior design context but is not canonical because it closed unmerged.
