## Summary

PR #70681 should stay open. Current main still uses the Unix `lsof -Fpc` command-name field to classify gateway PIDs, so the macOS `p_comm` vs rewritten argv[0] bug described in #70664 is not implemented on main or superseded. The PR is a focused fix with concrete reproduction context, though it needs the existing bot-review timeout concerns addressed before merge.

## Best Possible Solution

Keep PR #70681 open. The best path is to update the PR or an equivalent maintainer patch so Unix stale-gateway detection parses listener PIDs from lsof, verifies candidate gateway processes with argv inspection, applies the short polling timeout on poll-path verifier calls, and updates the existing restart-stale-pids tests away from lsof command-name classification.
