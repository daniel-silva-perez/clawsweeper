## Summary

Keep open. Current main has a broader configurable sandbox/exec-approval architecture, but it does not implement this PR's requested macOS `sandbox-exec`/Seatbelt wrapper or `sandboxProfile` exec parameter. The PR is draft and likely needs redesign, but the remaining native macOS isolation question is real enough that no conservative auto-close reason fits.

## Best Possible Solution

Keep the item open for maintainer judgment, but do not merge #62937 as-is. If OpenClaw wants native macOS process isolation, implement it as a proper sandbox backend or backend capability behind `agents.defaults.sandbox`/`BashSandboxConfig.buildExecSpec`, with docs and regression tests, while leaving broader permission-model work tracked by #58730.
