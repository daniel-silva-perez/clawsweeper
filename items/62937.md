## Summary

Keep this PR open. Current main has backend-based sandboxing and exec approval hardening, but it does not implement this PR's macOS `sandbox-exec` wrapper or `sandboxProfile` exec parameter. The PR is also a draft paired with same-author open feature issue #58730, so conservative cleanup should leave it for maintainer product/security review.

## Best Possible Solution

Keep this PR open for maintainer review alongside #58730. If OpenClaw wants native macOS process isolation, design it as a sandbox backend or backend capability behind the existing `agents.defaults.sandbox` / `BashSandboxConfig.buildExecSpec` model, with config docs and regression coverage. If maintainers decide macOS Seatbelt support should not live in core, close this PR and the paired issue together after that explicit decision.

## Risks / Open Questions

- The draft patch is security-sensitive because it changes exec process isolation and adds a new per-call policy surface outside the current `agents.defaults.sandbox` backend model.
- The PR body notes `sandbox-exec` is deprecated on macOS, and the proposed `permissive` profile allows network/broad access, so this needs explicit maintainer product/security judgment before merge.
- The patch appears stale against the current backend-oriented sandbox architecture and should not be merged as-is without redesign and regression coverage.
