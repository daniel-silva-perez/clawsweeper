## Summary

Keep #70533 open. Current main has related startup improvements and fixes the separate #69250 register-thrash path, but the specific request remains: `loadOpenClawPlugins` still discovers and builds a manifest registry for all discovered plugin candidates before applying the scoped plugin filter. Discovery itself also reads package and plugin manifests for every directory it scans, so the requested discovery-time filtering is not implemented yet.

## Best Possible Solution

Keep the issue open and implement a true cold discovery scope: carry the startup plugin id set, `plugins.allow`/activation allowlist, and any necessary plugin-id entries from tool policy into discovery or manifest-registry construction so disallowed bundled plugin directories are skipped before `package.json`/`openclaw.plugin.json` reads and schema normalization. Preserve explicit channel/config/slot auto-enable exceptions, then add tests that spy on manifest reads for a many-plugin fixture.
