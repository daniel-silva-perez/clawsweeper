## Summary

Keep #50943 open. Current main has some Feishu topic-root handling, but it does not implement the PR's central behavior: recovering Feishu `:topic:<id>:sender:<id>` affinity for run context, ACP/subagent routing, and all outbound reply/media paths. The PR discussion also reflects real review iteration rather than a low-signal or duplicate drive-by.

## Best Possible Solution

Keep the PR open, or replace it with a narrower maintainer-owned patch that preserves Feishu topic affinity from sender-scoped session keys where routing needs it, threads the resolved context through run-context/ACP/subagent paths, preserves explicit Feishu inline replies for text/card/media sends, and adds focused regression tests while dropping unrelated formatting churn.
