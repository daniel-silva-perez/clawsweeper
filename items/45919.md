## Summary

Current main already provides a plugin SDK seam for sandbox backends and uses that seam for the managed OpenShell sandbox. This PR adds OpenSandbox as core-specific backend/config/env plumbing instead of using the plugin path, so it is out of scope for OpenClaw core and should be pursued as a plugin/backend package.

## Close Comment

Closing after Codex review. Current main already has the right extension seam for this: plugins can register sandbox backends through `src/plugin-sdk/sandbox.ts`, and the bundled OpenShell integration uses that pattern in `extensions/openshell/index.ts` with plugin-scoped config documented in `docs/gateway/sandboxing.md` and `docs/gateway/openshell.md`. VISION also sets the scope bar here: optional capability should usually ship as plugins, not new core-specific plumbing (`VISION.md`). This PR adds OpenSandbox-specific backend/config/env wiring directly to core, so the maintainable path is an OpenSandbox plugin/backend package rather than merging it into OpenClaw core.
