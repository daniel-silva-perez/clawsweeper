## Summary

Keep open. Current main has adjacent safeguards, including config validation before managed service start and a separate fix for the lossless-claw context-engine ID regression, but it does not implement the requested `openclaw gateway preflight [--json]` no-bind command. The gateway CLI still registers run/status/probe/call/diagnostics/service commands, and the real startup path ultimately calls `startListening()`, so there is no shared preflight-only surface that exercises startup/plugin paths and exits before binding.

## Best Possible Solution

Keep this issue open and implement a first-class `openclaw gateway preflight [--json]` command by extracting the current startup validation/plugin-bootstrap work into a shared routine that can run config/schema/auth/plugin/context-engine/port checks without `startListening()` or session-side effects. Reuse the existing config snapshot, startup auth, plugin bootstrap, and context-engine paths rather than adding a separate schema-only validator.
