## Summary

Keep this PR open. Current main still omits the sessionKey when memory-wiki shared search calls the shared memory manager, while the memory manager contract and QMD backend already accept and use that option for scope checks. The PR is a focused fix with a relevant regression test, but it likely needs a small polish so calls without an agentSessionKey do not accidentally change their option shape.

## Best Possible Solution

Review and land a scoped memory-wiki fix: forward a non-empty params.agentSessionKey into sharedMemoryManager.search in extensions/memory-wiki/src/query.ts, keep the regression coverage in extensions/memory-wiki/src/query.test.ts, and run the targeted memory-wiki/changed validation before merge. No core API or security-sensitive infrastructure change is needed.
