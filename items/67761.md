## Summary

Keep this PR open. Current main still has the reported behavior in all three affected channel plugins: Slack, MS Teams, and Mattermost build a 160-character inbound message preview and include it in the queued system event, while the full inbound body is delivered separately. The linked issue #67503 remains a valid product bug, and the PR body plus Greptile review point at the same focused fix path.

## Best Possible Solution

Keep the PR open and review a rebased version, or land an equivalent patch, that removes inbound body previews from Slack, MS Teams, and Mattermost system-event labels while preserving full body delivery through the existing inbound envelope path. Add focused regression coverage for the queued system event text so this does not reappear.
