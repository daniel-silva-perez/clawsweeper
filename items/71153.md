## Summary

Keep #71153 open. Current main still rejects any shell command containing a POSIX backslash-newline before allowlist analysis, and both gateway/node policy paths still surface analysis failure as an allowlist miss. The linked fix PR #71154 remains open, while the related #71225 was closed unmerged, so the reported bug is not implemented on main.

## Best Possible Solution

Keep the issue open until #71154 or an equivalent maintainer-approved fix lands. The fix should splice POSIX `\\<newline>` continuations before allowlist analysis, preserve quoted/comment/heredoc semantics, and split unsupported shell syntax from true allowlist misses in both gateway exec and node `system.run`, with regression coverage for LF/CRLF multi-line commands and unparseable syntax.
