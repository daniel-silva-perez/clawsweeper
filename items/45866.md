## Summary

Keep open. Current `main` already has a broader fix for reset behavior that clears only auto-sourced model/provider/auth overrides, but it does not implement the PR's `/new` fallback-notice cleanup in `initSessionState`. The PR as written is also stale against current behavior because blanket-clearing `modelOverride`/`providerOverride` would conflict with current tests that preserve user-selected overrides across `/new` and `/reset`.
