## Summary

Keep #71630 open. Current main has Web UI chat lifecycle primitives and abort/final/error handling, but it does not render the requested compact composer status chip, does not expose the requested In progress/Interrupted/Done states in the composer toolbar, and does not yet have the related gateway-owned sessions.activity API from #39127. The related #39127 and #52640 remain adjacent but not canonical replacements: #39127 is the lower-level activity API, while #52640 is a channel status-surface request rather than this Web UI composer UX.

## Best Possible Solution

Keep this issue open as the focused Web UI composer UX task. The best implementation path is to add a session-scoped composer status view model and compact chip in ui/src/ui/views/chat.ts, with tests around send start, final, abort/error, reconnect/gap interruption, mobile layout, and aria/title text. It can start from existing chatRunId/chatSending/chat event lifecycle if maintainers accept that as a first step, but the durable product direction should consume the gateway-owned sessions.activity.get/list plus WS statechange work tracked in #39127 when that API lands. Do not close this as #52640, because that tracks editable channel task-status surfaces rather than the Control UI composer.
