## Summary

Keep open. Current `main` still has the first-match package-root walk, so a copied `dist/package.json` with `name: "openclaw"` would still resolve `dist/` as the package root and feed double-`dist` runtime candidates downstream.

## Evidence

- **Root walker still returns the first matching OpenClaw package:** `findPackageRoot` and `findPackageRootSync` return immediately on the first ancestor whose `package.json` has an OpenClaw core name; there is no parent-peek or outermost-root guard in current `main`. That means `dist/package.json` would still win over the real package root when walking up from `dist/plugins/`.
  - file: [src/infra/openclaw-root.ts:30](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/infra/openclaw-root.ts#L30)
  - command: `nl -ba src/infra/openclaw-root.ts | sed -n '1,140p'`
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **Plugin runtime resolution still trusts that package root directly:** `resolveLoaderPackageRoot` first calls `resolveOpenClawPackageRootSync({ cwd })`, where `cwd` defaults to `path.dirname(modulePath)`. For a loader under `dist/plugins/`, that means the buggy root walk is used before any other hint.
  - file: [src/plugins/sdk-alias.ts:126](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/plugins/sdk-alias.ts#L126)
  - command: `git blame -L 126,151 -- src/plugins/sdk-alias.ts`
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **A bad root would produce the reported double-dist candidate:** `resolvePluginRuntimeModulePath` builds `path.join(packageRoot, "dist", "plugins", "runtime", "index.js")` for dist resolution. If `packageRoot` is already `.../dist`, the resulting candidate is `.../dist/dist/plugins/runtime/index.js`, matching the PR report.
  - file: [src/plugins/sdk-alias.ts:647](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/plugins/sdk-alias.ts#L647)
  - command: `nl -ba src/plugins/sdk-alias.ts | sed -n '640,675p'`
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **Current tests do not cover the dist-package shadow case:** `src/infra/openclaw-root.test.ts` has no case for a copied `dist/package.json`, and the existing plugin runtime test only checks the normal `dist/plugins/loader.js` path without seeding a nested `dist/package.json`. So there is no current regression proof that the reported scenario was fixed on `main`.
  - file: [src/infra/openclaw-root.test.ts:183](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/infra/openclaw-root.test.ts#L183)
