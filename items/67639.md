## Summary

Keep this PR open. Current main still preserves separate status `sourceConfig` and resolved `cfg`, but the memory probe path drops `sourceConfig` before bootstrapping the memory runtime. The PR discussion remains relevant, and its follow-up commits address the two runtime-contract review concerns; the branch likely needs scoping or rebasing, not automatic closure.

## Best Possible Solution

Keep the PR open and review a scoped version of the fix: thread `overview.sourceConfig` as `activationSourceConfig` through `executeStatusScanFromOverview`, `resolveStatusMemoryStatusSnapshot`, `resolveSharedMemoryStatusSnapshot`, the status memory runtime wrapper, and the internal memory runtime bootstrap, while keeping `activationSourceConfig` out of plugin runtime hook arguments and preserving omitted optional fields. Retain or add the focused regression tests for normal status, fast-json status, and memory-runtime bootstrap; split unrelated extension/type cleanup out of this branch.
