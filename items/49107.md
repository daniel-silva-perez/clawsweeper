## Summary

Keep #49107 open. Current main still does not implement non-interactive onboarding flags for trusted-proxy auth, trusted proxy IPs, trusted-proxy allowUsers, or Control UI allowed origins. The underlying gateway config/schema already supports trusted-proxy and allowUsers, so this PR is still a plausible onboarding/CLI plumbing change rather than obsolete or out of scope.

## Best Possible Solution

Keep the PR open or replace it with an updated focused PR that adds non-interactive onboarding support for `gateway.auth.mode="trusted-proxy"`, `gateway.trustedProxies`, `gateway.auth.trustedProxy.userHeader`, `requiredHeaders`, `allowUsers`, and `gateway.controlUi.allowedOrigins`, with tests and docs. Coordinate wording with #69066, but do not close #49107 as implemented or superseded because the concrete CLI behavior is still missing on main.
