## Summary

Keep #72148 open. Current main still contains and tests the user-facing `Automatic session resume failed, so sending the status directly.` prefix in the exec approval follow-up direct fallback path, while the PR is a recent focused attempt to remove that behavior for linked issue #72143. The provided PR context also shows active author follow-up after bot review, so this is not obsolete maintainer-cleanup material.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a focused fix that removes the internal prefix from user-facing exec approval follow-up fallback output and logs resume failures internally with safe sanitization, then leave retry/backoff as a separate follow-up or explicitly scope #72143 accordingly.
