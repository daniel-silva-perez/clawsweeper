## Summary

Keep #72148 open. Current main still sends the user-facing `Automatic session resume failed, so sending the status directly.` prefix when exec approval follow-up session resume fails, and tests still assert that behavior. The PR is focused on the linked #72143 user-visible leak and is not superseded by #72268, which addresses a separate nested-metadata parser bug.

## Best Possible Solution

Keep #72148 open for maintainer review. The best path is to land a focused fix that removes the internal resume-failure prefix from user-facing exec approval fallback messages, logs resume failures internally with safe redaction/sanitization, and tracks retry/backoff separately unless maintainers choose to expand this PR scope. Coordinate with #72268 only as a related parser fix; it does not replace this PR.
