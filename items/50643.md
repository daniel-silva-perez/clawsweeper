## Summary

Keep PR #50643 open. Current main still has the macOS first-use TLS path that stores and accepts a new WSS certificate before `SecTrustEvaluateWithError`, and the linked issue/PR discussion describes a concrete gateway impersonation chain. The PR is not obsolete or superseded; it is a focused security fix attempt, but the review thread also raises a real product/design concern around private/self-signed gateway enrollment that likely needs maintainer judgment or a broader macOS fingerprint config/UI path before landing.

## Best Possible Solution

Keep the PR open for maintainer/security review. The best path is to land a security fix for the macOS first-use WSS trust bypass, with regression coverage around `GatewayTLSPinningSession`, and either wire `gateway.remote.tlsFingerprint` into the macOS remote direct path or explicitly document/accept the breaking behavior for non-system-trusted first-use certificates. If maintainers prefer a different implementation, supersede this PR only after that fix lands on main.
