## Summary

Keep open. Current `main` at `ebd7f19a3b20001960474ec9ba9cc424e45f7b23` still has the vulnerable first-use TOFU path for macOS direct `wss://` node connections, so PR #50643 is not already implemented or obsolete.

## Evidence

- **Current TLS pinning code still trusts and stores first-use certs before any trust evaluation:** `apps/shared/OpenClawKit/Sources/OpenClawKit/GatewayTLSPinning.swift:120-143` computes the presented certificate fingerprint, then in the `allowTOFU` branch saves it with `GatewayTLSStore.saveFingerprint(...)` and returns `.useCredential` immediately. `SecTrustEvaluateWithError(...)` only runs afterward in the fallback block, so it is skipped on first-use TOFU accepts.
  - file: [apps/shared/OpenClawKit/Sources/OpenClawKit/GatewayTLSPinning.swift:120](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/apps/shared/OpenClawKit/Sources/OpenClawKit/GatewayTLSPinning.swift#L120)
  - command: `nl -ba apps/shared/OpenClawKit/Sources/OpenClawKit/GatewayTLSPinning.swift | sed -n '108,145p'`
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **macOS node still enables TOFU on first direct WSS connection:** `apps/macos/Sources/OpenClaw/NodeMode/MacNodeModeCoordinator.swift:174-186` loads a stored fingerprint by `host:port`, then sets `allowTOFU: stored == nil`. A new remote gateway therefore takes the insecure first-use branch above.
  - file: [apps/macos/Sources/OpenClaw/NodeMode/MacNodeModeCoordinator.swift:174](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/apps/macos/Sources/OpenClaw/NodeMode/MacNodeModeCoordinator.swift#L174)
