## Summary

Current `main` does not implement the requested native plugin SDK `registerMcpServer` surface. OpenClaw has adjacent MCP support for bundle manifests and ACPX bridges, but native plugins still cannot register managed MCP servers through `OpenClawPluginApi`.

## Evidence

- **No `registerMcpServer` symbol on main:** A repo-wide search for `registerMcpServer` returned no matches in `src`, `docs`, `extensions`, `packages`, or `test`.
  - command: `rg -n "registerMcpServer" src docs extensions packages test`
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **Plugin SDK API surface lacks MCP registration:** `OpenClawPluginApi` exposes registration methods for tools, commands, providers, channels, services, hooks, gateway methods, embedded extension factories, and memory slots, but no MCP server registration method.
  - file: [src/plugins/types.ts:2010](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/plugins/types.ts#L2010)
