## Summary

Keep this issue open. Current main implements the original Codex-native web-search feature and documents `mode: "live"`, but the concrete failure mode described in the issue/comment still appears present: `applyExtraParamsToAgent` receives `agentDir`/`workspaceDir` and uses them for provider preparation, yet does not forward them into the provider stream-wrapper context. The Codex native web-search wrapper depends on that context `agentDir` to detect persisted `openai-codex` auth, so valid auth in `auth-profiles.json` can still be treated as missing. The related PR #67843 is an open, focused candidate fix, not a merged/superseding resolution.

## Best Possible Solution

Keep the issue open. Review and either land or replace #67843 with a fix that forwards `agentDir` and `workspaceDir` into provider stream-wrapper context, with a regression proving persisted `openai-codex` auth activates native Codex web search. After that merges, verify whether a narrower follow-up is needed for diagnostics or end-to-end native tool-call visibility when the tool is attached but not invoked.
