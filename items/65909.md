## Summary

Keep this issue open. Current main still has the concrete auth-context failure mode described in the discussion: `agentDir` and `workspaceDir` are available before provider stream wrapping, but `applyExtraParamsToAgent` does not forward them into `wrapProviderStreamFn({ context })`. Codex native web search then cannot see persisted `openai-codex` auth through `ctx.agentDir`, so valid agent auth can still resolve as missing.

## Best Possible Solution

Keep the issue open. Review and either land or replace #67843 with a focused fix that forwards `agentDir` and `workspaceDir` into the provider stream-wrapper context, then add a regression proving persisted `openai-codex` auth activates native Codex web search. After that merges, decide whether a narrower follow-up is needed for diagnostics or end-to-end visibility when the native tool is attached but not invoked.
