## Summary

Keep this issue open. Current main still drops `agentDir` and `workspaceDir` when handing the stream wrapper context to provider hooks, while Codex native web-search activation still depends on `agentDir` to find persisted `openai-codex` auth profiles. The linked open PR #67843 remains the concrete candidate fix rather than evidence that the issue is already resolved.

## Best Possible Solution

Keep this issue open. Review and either land or replace #67843 with a focused fix that forwards `agentDir` and `workspaceDir` into `ProviderWrapStreamFnContext`, then add regression coverage proving persisted `openai-codex` auth activates native Codex web search through the embedded provider wrapper path. After that merges, decide whether any remaining diagnostics/tool-event visibility concern should become a narrower follow-up.
