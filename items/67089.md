## Summary

Keep open. Current main already covers the config-method scope part through reserved `config.*` admin scope handling, and it already has config write/observe audit logging, but it does not implement the PR's central config HMAC sidecar signing/verification or `integrityWarning` reload behavior. The PR also has review-raised correctness blockers, so the best path is request changes or security maintainer triage rather than auto-closing it.

## Best Possible Solution

Keep this PR open for maintainer/security review or request changes. A mergeable version should separate unrelated CodexBar onboarding from config security, preserve the existing reserved `config.*` admin-scope behavior, design a backwards-compatible HMAC bootstrap/migration path for existing configs, propagate signature-write failures consistently, thread injected filesystem/homedir dependencies through tests, and add focused tests for first-upgrade, tamper detection, reload behavior, and write failure handling.
