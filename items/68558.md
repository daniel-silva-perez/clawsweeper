## Summary

Keep open. Current main still appears to have the reported misleading `/compact` presentation path, so this PR is not obsolete; however, the PR head has a functional correctness issue called out in review: it passes `contextTokens`/session `contextTokens` as `currentTokenCount`, even though the compactor contract expects observed live prompt tokens. The right maintainer path is to request changes rather than close this cleanup item.

## Best Possible Solution

Keep this PR open for maintainer review and request a focused revision: preserve the useful label guard and stale-token summary idea, but either omit `currentTokenCount` for manual `/compact` or compute it from an actual observed prompt/session token source equivalent to the preflight path. Add a regression test for `before <= after` label rendering and, if keeping the file-size fallback, a test covering stale stored totals without passing context-window values as usage.
