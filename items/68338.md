## Summary

Keep this PR open. Current main still parses only same-line `MEDIA:<path-or-url>` tokens: an empty `MEDIA:` line has no regex match and is preserved as text, with no logic to inspect or consume the following line. The PR is a focused implementation candidate for a real remaining parser gap, and its review thread already addressed the main behavioral risk by limiting continuation consumption to one following line.

## Best Possible Solution

Review and refresh this PR against current main rather than closing it. The best implementation path is to add a narrow continuation branch in `src/media/parse.ts` for an otherwise empty `MEDIA:` directive, consume exactly one valid following media-path line outside fenced blocks, preserve ordered text/media segments, keep invalid continuations as text, add the two proposed regression tests, and update the public rich-output/media docs if maintainers accept multiline syntax as supported.
