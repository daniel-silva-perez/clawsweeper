## Summary

Keep #49117 open. Current main still has the empty-payload logging behavior this PR targets: local agent delivery still emits `No reply from agent.` with no intentional-silent-completion guard, and the gateway CLI still logs a summary/fallback for zero-payload results. Main does model exact `NO_REPLY` as deliberate silence elsewhere, so the PR is not obsolete or already implemented.

## Best Possible Solution

Keep the PR open and ask for a rebase/update against current main. The best implementation should preserve an intentional silent-completion signal for exact `NO_REPLY` empty-payload runs, use a shared helper or existing metadata shape rather than duplicated guards, and keep focused regression tests for local delivery and gateway CLI zero-payload logging.
