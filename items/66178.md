## Summary

Keep open. Current main still appears to have the reported failure class: the LM Studio/OpenAI-compatible chat-completions stream sends configured tools and executes structured `delta.tool_calls`, but bracketed plain-text blocks like `[mempalace_...] ... [END_TOOL_REQUEST]` are neither promoted to tool calls nor covered by the visible-text sanitizer.

## Best Possible Solution

Keep this issue open. Add a focused regression fixture for the OpenAI-compatible LM Studio path that emits the reported standalone bracket block, then decide a narrow safe behavior: either convert a standalone bracket block whose name exactly matches a registered tool into a normal toolCall before assistant text delivery, or explicitly reject/suppress unsupported local-model text-tool formats with a clear non-hanging error. Investigate the MemPalace `predicate` argument error separately against the advertised MCP tool schema.
