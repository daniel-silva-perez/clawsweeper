## Summary

Keep open. Current main still has the reported failure class: LM Studio uses the OpenAI-compatible stream path, that path only executes structured `delta.tool_calls`, and the current visible-text sanitizer covers XML and `[Tool Call: ...]` downgrade markers but not the reported `[mempalace_...] ... [END_TOOL_REQUEST]` bracket block. The issue is a concrete runtime/provider bug rather than ClawHub scope, because configured MCP tools are already materialized and executable through the core embedded run path.

## Best Possible Solution

Keep this issue open. Reproduce on current main with either the reported LM Studio/MemPalace setup or a focused OpenAI-compatible stream fixture that emits the reported bracket block. The fix should either safely promote recognized local-model plain-text tool blocks into normal tool-call events before assistant text delivery, or explicitly document/guard unsupported local-model text tool formats and prevent hangs. Also check the MemPalace `predicate` argument error against the advertised MCP tool schema as a separate root cause.
