## Summary

Keep open. Current main does not ship the Teams thread-copilot approval flow: Teams does not advertise or handle `thread-list`, the runtime has no `listThreadMSTeams`/thread action approval surface, and Adaptive Card invokes are still routed as normal agent messages rather than plugin interactive callbacks. The PR is useful but not safely closeable as implemented, duplicate, or ClawHub-only because the missing Teams approval callback seam is a real core/plugin-SDK question, and the supplied review context has unresolved correctness regressions.

## Best Possible Solution

Keep PR #66210 open for maintainer judgment, but do not land it as-is. The best path is to split the work: first add a small, tested Teams plugin-interactive approval callback seam that preserves current Adaptive Card fallback, SSO, and file-consent behavior; then add the thread-list/thread-action approval helper on top of that seam. If maintainers decide the long-thread copilot itself is optional plugin work, move that part to ClawHub after the missing Teams/API seam is available.
