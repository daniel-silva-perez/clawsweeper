## Summary

Keep #70479 open. Current main still contains the compaction-failure session reset path that the PR is trying to remove, and the linked bug #70472 remains unsolved by main: overflow/compaction failures can still call the reset hook, rotate the active session entry, and tell the user the conversation was reset.

## Best Possible Solution

Keep the PR open and either land a reviewed version of this narrow fix or make an equivalent mainline patch that preserves the active session mapping after unrecoverable overflow/compaction failures, surfaces explicit retry `/compact` `/new` guidance, removes the dead compaction-reset callback, and adds regression coverage for embedded overflow payloads and thrown compaction-failure errors. #70473 can remain separate because it addresses overflow-budget extraction, not the later session remap/orphaning path.
