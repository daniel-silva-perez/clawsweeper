## Summary

Keep open. Current main still has the Windows restart double-start path: CLI restart starts the Scheduled Task, while the gateway shutdown/respawn path can also generate a helper script that retries `schtasks /Run`. The PR is focused on the right file and addresses the linked duplicate-window report, but it still needs maintainer review and likely a small hardening/test pass rather than cleanup closure.

## Best Possible Solution

Keep #52487 open. The best path is a focused Windows restart fix in `src/infra/windows-task-restart.ts` that makes the self-restart helper skip `schtasks /Run` when the Scheduled Task is already running, hardens the PowerShell probe, and adds a targeted generated-script regression test. Then a maintainer can review/merge it against the current restart-intent flow.
