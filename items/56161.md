## Summary

Keep open. Current main still resolves `openclaw skills install` and `update` through the default agent workspace only, and `list`/`info`/`check` also build status from the default workspace. The cwd-to-agent helper exists and is used elsewhere, but `src/cli/skills-cli.ts` does not import or call it. The two related PRs in the provided context proposed this exact fix, but both are closed and unmerged, and their behavior is not present in the current checkout.

## Best Possible Solution

Keep the issue open and land a narrow skills CLI fix: add a shared active-workspace resolver in `src/cli/skills-cli.ts` that prefers `resolveAgentIdByWorkspacePath(config, process.cwd())` before falling back to `resolveDefaultAgentId(config)`, then use it consistently for install, update, list, info, and check. Add focused tests in `src/cli/skills-cli.commands.test.ts` for cwd-inferred sub-agent workspaces and default fallback, likely adapting the closed #56294 approach.
