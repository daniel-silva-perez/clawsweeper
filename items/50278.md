## Summary

Current main does not implement PR #50278’s requested behavior. Markdown image text that starts with `![` still matches the generic bang-command paths in TUI submit handling, bash chat command routing/parsing, and ACP command bypass logic. The PR remains a plausible focused bug fix, though it needs rebasing/adaptation to current file names and must preserve valid shell-test commands such as `![ -f file ]`.

## Best Possible Solution

Keep the PR open and ask for a rebase onto current main. The best fix is a small shared Markdown-image detector, applied to the current bang-routing sites (`src/tui/tui-submit.ts`, `src/auto-reply/reply/commands-bash.ts`, `src/auto-reply/reply/bash-command.ts`, and `src/auto-reply/reply/dispatch-acp-command-bypass.ts`), with focused tests proving `![alt](url)` stays normal text while `![ -f file ]` remains a valid bang shell command.
