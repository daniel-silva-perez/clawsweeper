## Summary

Keep open. Current `main` still has internal queue bookkeeping and log/diagnostic events, but it does not provide the requested agent-facing `queue_status` tool or an operator-facing queue snapshot surface. The issue remains a valid core enhancement, not a ClawHub/plugin-scope item.

## Evidence

- **No `queue_status` tool on current main:** The core tool catalog lists the current session-management tools (`sessions_list`, `sessions_history`, `sessions_send`, `sessions_spawn`, `sessions_yield`, `subagents`, `session_status`) and there is no `queue_status` entry. A repo search for `queue_status` returned no matches in `src` or `docs`.
  - file: [src/agents/tool-catalog.ts:145](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/src/agents/tool-catalog.ts#L145)
  - command: `nl -ba src/agents/tool-catalog.ts | sed -n '120,220p'; rg -n "queue_status" src docs`
  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
- **No queue snapshot RPC/CLI surface:** Gateway methods include `diagnostics.stability` but no queue diagnostics/snapshot method, and the CLI exposes `openclaw gateway stability` plus `gateway diagnostics export`, not a queue snapshot command.
  - file: [src/gateway/server-methods-list.ts:4](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/src/gateway/server-methods-list.ts#L4)
