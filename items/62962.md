## Summary

Keep open. Current main does not implement the requested tool security classification properties. The core AnyAgentTool metadata only carries ownerOnly/displaySummary, TOOL_POLICY_CONFORMANCE only snapshots tool groups, and existing safety paths are allow/deny filtering plus hook-driven approval, not schema-driven isReadOnly/isDestructive/isConcurrencySafe/interruptBehavior/isOpenWorld/toAutoClassifierInput behavior. The provided related items are adjacent security/tool-policy reports, not canonical duplicates for this feature.

## Best Possible Solution

Keep the issue open and implement it, if accepted, as a core/public SDK tool metadata contract: extend tool types and plugin/channel tool surfaces, preserve metadata through wrappers/adapters, annotate core tools, add conformance/tests/docs, and only then drive approval, interrupt, and parallel scheduling behavior from the declarations.
