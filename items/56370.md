## Summary

Keep open. Current main still takes `openclaw agent --to ...` without `--agent` through the hardcoded `main` session-key path, so the reported `agent:main:main` vs `agent:steward:main` mismatch remains actionable. The linked PR #56453 appears to be the right shape of fix, but the provided GitHub context says it is still open/unmerged and its head commit is not present in this checkout.

## Best Possible Solution

Review and refresh PR #56453 or land an equivalent fix: for `openclaw agent --to` without explicit `--agent`, resolve the configured default agent before deriving the session key and session-store path, producing `agent:<default>:main`; preserve explicit `--agent`, explicit `--session-key`, and cross-store `--session-id` behavior, and add a regression test with a non-main default agent such as `steward`.
