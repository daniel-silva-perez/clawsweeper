## Summary

Keep open. Current main still does not implement PR #51288's central user-message Control UI fix: user-role extraction strips runtime context, envelopes, and sentinel-based inbound metadata, but not `<relevant-memories>` XML blocks. The PR also has unresolved review feedback, so it should not be treated as already shipped or safe to close as obsolete.

## Best Possible Solution

Keep the item open, but request changes toward a shared, role-aware sanitizer: strip only OpenClaw-injected leading memory scaffolding from user-visible user messages, preserve literal user-authored tags and code-fence examples, fail open on malformed user tags, and leave assistant sanitization as global block stripping. Add focused tests in `ui/src/ui/chat/message-extract.test.ts` for injected user prefixes, literal non-prefix user tags, unterminated user blocks, and assistant mid-message blocks; consider whether `message-normalizer`/gateway sanitization need the same helper for consistent Control UI surfaces.
