## Summary

Keep open. Current `main` still implements the behavior this PR is trying to change: a dirty watched source tree forces a rebuild before any mtime check, and the test suite explicitly asserts that behavior.

## Evidence

- **Current main still short-circuits dirty watched trees to rebuild:** `resolveBuildRequirement()` returns `{ shouldBuild: true, reason: "dirty_watched_tree" }` when `hasDirtySourceTree()` is `true`, before `hasSourceMtimeChanged()` runs.
  - file: [scripts/run-node.mjs:333](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/scripts/run-node.mjs#L333)
  - command: `nl -ba scripts/run-node.mjs | sed -n '320,350p'`
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **Current tests still expect dirty source trees to rebuild:** There is an explicit test named `reports dirty watched source trees as an explicit build reason` asserting `shouldBuild: true` with reason `dirty_watched_tree` for a modified `src/index.ts`.
  - file: [src/infra/run-node.test.ts:1003](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/infra/run-node.test.ts#L1003)
