## Summary

Keep this PR open. Current main still has the affected temp-dir repair behavior: `tryRepairWritableBits` treats repair failure as `false` without re-checking whether the directory became valid, and the PR is a focused implementation candidate for that race. The provided PR context also shows the bot's only review suggestion was addressed by the second commit.

## Best Possible Solution

Keep the PR open for maintainer review and CI. The best path is to land or refine a narrow fix in `src/infra/tmp-openclaw-dir.ts` that re-checks directory availability after expected concurrent repair races, with regression coverage in `src/infra/tmp-openclaw-dir.test.ts`; if it is intended to close #66867, consider covering both chmod-throw and already-secure re-stat races.
