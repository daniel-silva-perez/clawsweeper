## Summary

Keep #54879 open. Current main still has only the existing browser act kinds (`type`, `fill`, `press`, etc.) and routes text entry through Playwright `fill`/`type` or Chrome MCP `fill`; there is no CDP/IME text insertion action, no `Input.insertText` call, and no docs/tests for the requested contenteditable IME path. The related CDP issues in the provided context concern attach/session/download/timeouts and do not supersede this request.

## Best Possible Solution

Keep the issue open and implement an additive browser act capability if maintainers accept the product direction: add a typed `insertText`/`insertTextIme` kind to the browser tool schema, client types, normalizer, CLI/docs, and tests; focus the target ref/tab and use a page-scoped CDP insertion path for managed CDP profiles, with an explicit existing-session strategy or documented limitation.
