## Summary

Keep #70061 open. Current main still misses the reported Telegram DM-topic session-key shape: Telegram DM topics can use `agent:main:main:thread:<chatId>:<topicId>`, but Active Memory only classifies keys containing `:group:`, `:channel:`, `:direct:`, `:dm:`, exact three-part main sessions, or webchat. The existing Active Memory regression test covers a `telegram:direct:...:thread:...` key, not the shared `main:thread` key from this report.

## Best Possible Solution

Patch `extensions/active-memory/index.ts` so Telegram/non-webchat interactive DM-topic sessions using shared main-session thread keys like `agent:main:main:thread:<chatId>:<topicId>` classify as `direct`. Add a focused Active Memory regression test for that exact key shape with `allowedChatTypes: ["direct"]`, and consider adding diagnostic logging/status coverage for unclassified chat-type skips so future session-key formats do not fail silently.
