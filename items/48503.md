## Summary

Keep open. Current `main` still does not expose action classification or input provenance on the `before_tool_call` event. The hook event surface remains limited to `toolName`, `params`, `runId`, and `toolCallId`, while provenance exists elsewhere in the runtime but is not threaded into this hook.

## Evidence

- **before_tool_call event type is still minimal:** `PluginHookBeforeToolCallEvent` only declares `toolName`, `params`, `runId`, and `toolCallId`; there is no `actionCategory` or `inputProvenance`.
  - file: [src/plugins/hook-types.ts:318](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/plugins/hook-types.ts#L318)
  - command: `nl -ba src/plugins/hook-types.ts | sed -n '308,326p'`
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **session identifiers live in hook context, not the event payload:** `PluginHookToolContext` carries `agentId`, `sessionKey`, `sessionId`, `runId`, `trace`, `toolName`, and `toolCallId`, confirming the current contract split and that the requested enrichment was not added to the event object itself.
  - file: [src/plugins/hook-types.ts:308](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/plugins/hook-types.ts#L308)
