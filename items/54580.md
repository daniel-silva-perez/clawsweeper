## Summary

Keep open. Current main does not provide strong evidence that either reported Feishu problem is fully fixed: in-place npm/pnpm updates still explicitly require a gateway restart to apply to a running service, and Feishu streaming-card creation still surfaces raw HTTP 400 failures without targeted card-create backoff, downgrade, or response-code handling.
