## Summary

Keep open. Current `main` already fixes the context banner to use fresh `totalTokens` instead of cumulative `inputTokens`, but the grouped per-message `% ctx` path still sums token usage across all assistant messages in a visual group. The PR is stale against current `main` for the banner portion, yet it still points at a real remaining gap in grouped rendering, so there is not strong evidence to close it as already implemented or obsolete.
