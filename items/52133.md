Keep open. Current main only has opt-in post-compaction JSONL truncation for #39953; it does not implement #52133's rolling pruning for oversized ordinary, never-compacted session transcripts. Keep #52133 open for maintainer-guided rebase or replacement. The best fix is a scoped core implementation of opt-in rolling transcript pruning for ordinary persisted sessions, with config schema/help/docs/tests, correct session-lock and retry timing, and preservation of sibling branches, compaction tails, tool-result spans, and custom transcript entries. Split or drop the unrelated plugin-sdk smoke-test change.
