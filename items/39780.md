## Summary

Current main still leaves #39780 unresolved. The Overview insecure-HTTP recommendation is text-only manual guidance, while the existing diff/apply controls are limited to the Config editor/profile flows and are not wired into Overview or hardening recommendation callouts. Related #38249 and #38420 are useful context, but they do not supersede this focused UI request.

## Best Possible Solution

Keep #39780 open until Control UI recommendation callouts can stage a narrow, base-hash guarded config patch, show a before/after diff, and let the user accept or dismiss the change with confirmation. The likely implementation should reuse the existing Config editor diff/apply primitives where possible, add focused tests for the insecure HTTP recommendation path, and update the public Control UI docs once the guided apply flow exists.
