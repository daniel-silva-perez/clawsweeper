## Summary

Current main does not implement Android `location.enabledMode: always`. Android still has only `Off` and `WhileUsing`, normalizes legacy `always` to `whileUsing`, returns `LOCATION_BACKGROUND_UNAVAILABLE` before attempting any location fetch when backgrounded, and does not declare Android background-location or location foreground-service permissions. The related closed #71015 is a different metadata-delivery request, not a canonical replacement for this background-location feature request.

## Best Possible Solution

Keep this issue open for maintainer product/platform decision. If accepted, the implementation likely belongs in `apps/android`: add an Android `LocationMode.Always`, an explicit settings and permission flow for `ACCESS_BACKGROUND_LOCATION`, manifest/flavor handling for `FOREGROUND_SERVICE_LOCATION`, a short-lived foreground-service-backed `location.get` path for background invocations, regression tests for both `whileUsing` and `always`, and updated location docs. If maintainers do not want to reintroduce background location after #39660, that should be an explicit maintainer close, not an automated cleanup close.
