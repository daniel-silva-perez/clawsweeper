## Summary

Keep this PR open. Current main still does not implement the requested save-time rewrite of persisted session transcript paths, and the PR is a plausible focused bug fix, but it needs revision before merge because the proposed resolver path can preserve some cross-root agent session paths instead of forcing the active sessions directory.

## Best Possible Solution

Keep this PR open and ask for a small update: add a save-time normalization pass in the session store write path, derive the canonical target directly from the active `sessionsDir` and `sessionId` rather than the compatibility resolver, persist both supported path fields if `transcriptPath` remains a compatibility field, and keep the regression test in `src/config/sessions/store.pruning.integration.test.ts`.
