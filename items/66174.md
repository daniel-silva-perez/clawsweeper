## Summary

Keep this PR open. Current main still lacks the requested save-time rewrite for persisted transcript paths, and the PR is a focused implementation candidate, but the current patch should be revised before merge because it relies on the compatibility resolver that can preserve canonical-looking cross-root session paths instead of forcing the active sessions directory.

## Best Possible Solution

Keep this PR open and request a small revision: add the save-time normalization in `src/config/sessions/store.ts`, derive the canonical path directly from the active `path.dirname(storePath)` and `sessionId` rather than `resolveSessionFilePath`, persist both `sessionFile` and compatibility `transcriptPath` if that alias remains supported, and keep the regression test while preserving external-transcript deletion safeguards.
