## Summary

Keep open. Current main still appears to have both reported problem paths: google-vertex ADC resolves to a synthetic marker that can be injected as the runtime apiKey and serialized as `x-goog-api-key`, and the bundled dependency path still includes gaxios 7.1.4, whose Node adapter imports `node-fetch` instead of preferring `globalThis.fetch`. The related ClawSweeper reports in the supplied context also kept the same ADC-marker bug open rather than identifying a shipped fix.

## Best Possible Solution

Keep the issue open and fix it in the bundled Google provider/runtime path: ensure google-vertex ADC never reaches transports as a fake API key, add provider-owned runtime auth that uses Vertex ADC/OAuth with `vertexai: true` plus project/location, and address gaxios Node 25 by updating/patching the dependency or injecting a native fetch implementation. Add focused tests covering ADC marker handling, absence of `x-goog-api-key` for ADC, Vertex client options, and the Node 25 fetch path.
