## Summary

Keep open. Current main still does not implement create/update-time Feishu/Lark missing-target rejection for #39599, but PR #39683 is not mergeable as the best fix against today’s architecture: cron core validation is generic, channel targets can be resolved from session/default plugin paths, and the unconditional `delivery.to` check would reject some valid or remediable jobs. The existing PR review also identified a concrete regression for disabling legacy Feishu/Lark jobs without `delivery.to`.

## Best Possible Solution

Keep the item open, but require a rework rather than merging the current diff. The best fix should validate unresolved announce targets through the current generic delivery-resolution seam, allow explicit `delivery.to` and deterministic session/default resolution, avoid blocking unrelated enable/disable updates for legacy jobs, keep provider-specific rules in the Feishu/Lark plugin surface where possible, and update only English docs plus focused regression tests for create-time failure and valid resolved-target acceptance.
