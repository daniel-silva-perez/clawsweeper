## Summary

Keep open. Current main has several stop/abort protections, but the issue reports an internal continuation after an explicit stop/no-action instruction and current main still contains automatic continuation/restart recovery paths that can synthesize resume prompts. With no reproduction details or logs, and with plausible continuation paths still present, this is not a high-confidence implemented or cannot-reproduce close.

## Best Possible Solution

Keep the issue open until maintainers add or verify an end-to-end regression for the affected surface. The likely hardening path is to use the shared abort phrase detector consistently in Control UI and recovery code, skip automatic continuation/restart recovery when the latest user instruction is an abort/no-action request, and add focused tests for active-run stop phrases plus restart/incomplete-turn continuation suppression after user abort.
