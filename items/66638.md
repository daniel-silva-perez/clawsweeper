## Summary

Keep this issue open. Current main has important partial fixes for subagent completion delivery, including direct completion announce delivery, retry, idempotency, task records, and plugin hooks for spawn/end/delivery routing. It still does not implement the requested framework-level immediate user-facing spawn ACK, per-run progress lifecycle notifications, or durable announce queue decoupled from in-process/session-lane delivery. Related #22358 and #54488 are still open and cover narrower parts of the same remaining area, but neither supersedes the full request.

## Best Possible Solution

Keep the issue open as an umbrella runtime architecture request. The best path is to build on the existing direct completion delivery and hook work by adding explicit spawned/progress/completed/failed/cancelled lifecycle events, a framework-owned user-visible spawn acknowledgement that does not wait for the parent session turn, and durable announce persistence/retry, likely by integrating subagent announce delivery with the existing session-delivery queue or a purpose-built durable run-lifecycle queue.
