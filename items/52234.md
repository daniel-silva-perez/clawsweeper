## Summary

Keep PR #52234 open. Current main still clears followup queues during session reset and lacks the preserve/pause/rebind/supersession behavior this PR is trying to add. The PR also has unresolved late review blockers in the provided discussion, so it should not be closed as implemented or merged as-is.

## Best Possible Solution

Keep the item open, but do not land the PR as-is. The best path is a smaller current-main patch that preserves queued followups across `sessions.reset`, pauses/resumes drains while the replacement session is being written, rebinds queued/in-flight runs through canonical session resolution, suppresses stale streaming/final output, and preserves existing session overrides, with focused tests for reset races and the review-thread regressions.
