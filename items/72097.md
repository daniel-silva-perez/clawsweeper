## Summary

Current main does not implement mid-session plugin tool injection after `openclaw plugins install`. Plugin install persists the new plugin and explicitly tells users to restart the gateway, while embedded Pi sessions still construct `customTools`/`tools` once when the agent session is created. The hot-reload path covers models, hooks, cron, heartbeat, MCP runtime disposal, and channel restarts, but I found no plugin-tool delta broadcast or active-session tool rebuild path.

## Best Possible Solution

Keep this issue open. The best fix is a core runtime/session change: after a successful plugin install/update, refresh the active plugin runtime registry, detect newly contributed plugin tool names, and either rebuild active embedded/CLI session tool allowlists before the next turn or invalidate/restart only the affected agent sessions with clear user-visible recovery. Add regression coverage around install/update plus an already-active Pi session seeing a newly registered plugin tool.
