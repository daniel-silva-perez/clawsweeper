## Summary

PR #60005 should stay open. Current main still exposes the channel MCP server and plugin-tools MCP bridge as tool-only surfaces, and the proposed `channel-resources.ts` / `plugin-resources.ts` files are not present. The related ClawSweeper matches are about third-party channel/plugin work and do not supersede MCP resource support. The PR discussion is useful: bot review found real sensitive-config exposure risks, and the author followed up saying env, URL, query-param redaction, JSDoc, and error handling were addressed, so this is a plausible feature PR that needs maintainer review/rebase rather than conservative cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase it onto the current `createToolsMcpServer` / `createPluginToolsMcpHandlers` split, add resource capability/list/read handling in the shared MCP server layer where appropriate, keep channel resource registration in the channel server, reuse `redactConfigObject` or equivalent schema-backed redaction for any config resource, add targeted resource/redaction tests, and update `docs/cli/mcp.md` if the feature is accepted.
