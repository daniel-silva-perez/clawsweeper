## Summary

Keep open. Current `main` still rewrites Discord mentions into `<@id>` form but does not send an `allowed_mentions` field on the production outbound bot message path, so the requested fix is not implemented on `main`.

## Evidence

- **Production outbound send rewrites mentions before sending:** `sendMessageDiscord` rewrites cached `@name` mentions to Discord ID mentions, then delegates to the shared send helpers.
  - file: [extensions/discord/src/send.outbound.ts:158](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/extensions/discord/src/send.outbound.ts#L158)
  - command: `nl -ba extensions/discord/src/send.outbound.ts | sed -n '138,170p'`
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Shared payload builder does not include `allowed_mentions`:** `buildDiscordMessagePayload` only sets `content`, `components`, `embeds`, `flags`, and `files`; there is no `allowed_mentions` field in the payload shape built on current `main`.
  - file: [extensions/discord/src/send.shared.ts:308](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/extensions/discord/src/send.shared.ts#L308)

  - file: [extensions/discord/src/send.shared.ts:374](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/extensions/discord/src/send.shared.ts#L374)
