## Summary

Keep open. Current `main` already has the PR's secondary cleanup in the agent/session paths, but it still does not invalidate skills snapshots across a process restart, so the core bug this PR targets is not implemented on `main`.

## Evidence

- **Restart baseline still resets to zero:** `src/agents/skills/refresh-state.ts` still initializes `globalVersion` to `0`, so a fresh process starts with the same baseline as an untouched prior run.
  - file: [src/agents/skills/refresh-state.ts:9](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/src/agents/skills/refresh-state.ts#L9)

  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)

That means a snapshot saved before restart can still look current immediately after restart if no bump happened.
