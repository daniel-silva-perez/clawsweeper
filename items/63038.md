## Summary

Keep this PR open. Current main still lacks a debouncer-wide drain before WhatsApp socket teardown, and the PR remains a plausible implementation path, but the existing PR review comments identify real functional gaps that should be fixed before merge.

## Best Possible Solution

Keep this PR open for author or maintainer follow-up. The best fix is a debouncer-wide drain/quiescence API that repeatedly flushes current buffers, waits for active keyed chains, accounts for late enqueues from already-running WhatsApp upsert callbacks, and is called from WhatsApp close after detaching inbound listeners but before closing the socket. Add focused regression coverage proving pending debounced WhatsApp messages can still send `reply` or `sendMedia` before socket teardown, including late-enqueue and in-flight keyed-task cases.
