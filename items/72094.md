## Summary

Keep #72094 open. Current main has a native Codex app-server harness, /codex chat controls, ACP fallback, and the PeekabooBridge path, but it does not expose the requested codex-desktop desktop-control provider, computer-use detection, sessions_spawn computerUse option, or channel-safe desktop-control routing contract.

## Best Possible Solution

Keep the issue open for maintainer product/API design. The likely implementation path is a small core desktop-control capability contract with explicit provider selection and security policy, implemented by provider plugins such as codex-desktop and peekaboo, then wired into channel-safe routing and sessions_spawn or /codex controls only after the Codex Desktop/App Server computer-use API surface is verified.
