## Summary

Keep open. Current main appears to have fixed the Control UI/WebChat side by reloading on `session.message`, but the TUI still does not subscribe to or handle `session.message`/`sessions.changed`, so the issue is not fully implemented on `main`.

## Evidence

- **Gateway emits transcript-update events for appended messages:** Transcript appends broadcast both `session.message` and `sessions.changed` with `phase: "message"`, so clients need to consume those events to surface external session updates.
  - file: [src/gateway/server-session-events.ts:115](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/gateway/server-session-events.ts#L115)
  - command: `nl -ba src/gateway/server-session-events.ts | sed -n '110,160p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Control UI now reloads history on transcript events:** The browser Control UI handles `session.message` by reloading `chat.history` for the active session, and also reloads sessions on `sessions.changed`. That covers the WUI half of the report.
  - file: [ui/src/ui/app-gateway.ts:481](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/ui/src/ui/app-gateway.ts#L481)

  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Control UI has explicit regression tests:** Current tests assert that `session.message` reloads chat history for the active session, skips mid-run reloads, and ignores other sessions.
  - file: [ui/src/ui/app-gateway.sessions.node.test.ts:129](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/ui/src/ui/app-gateway.sessions.node.test.ts#L129)
  - command: `nl -ba ui/src/ui/app-gateway.sessions.node.test.ts | sed -n '112,180p'`

  - file: [src/tui/tui.ts:1034](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/tui/tui.ts#L1034)
