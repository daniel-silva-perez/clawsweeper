## Summary

Keep PR #70134 open. Current main still fingerprints `auth-profiles.json` by file mtime in the models ready-cache, so the PR is not obsolete or implemented. The PR also has meaningful review discussion: earlier profile-order, mixed-audience, and API-key alias concerns were addressed, but the remaining OAuth token-content fingerprinting concern needs maintainer judgment before this can be considered safely resolved.

## Best Possible Solution
