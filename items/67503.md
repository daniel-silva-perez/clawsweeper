## Summary

Keep this issue open. Current main still includes the 160-character inbound message preview in Slack, MS Teams, and Mattermost system events, while also delivering the full body separately to the agent. The provided GitHub context also shows an open same-author PR with closing syntax for this issue, so the right maintainer path is to review, test, and either land or close that PR rather than preemptively closing the issue.

## Best Possible Solution

Keep the issue open and review PR #67761 on its merits. The likely fix belongs in the Slack, MS Teams, and Mattermost channel handlers: keep the full inbound body in the agent envelope, but remove message-body previews from notification-only system events, with focused regression tests for each affected channel.
