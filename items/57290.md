## Summary

Current `main` already implements a targeted fix for fake-IP proxy environments: `web_fetch` now has an opt-in `tools.web.fetch.ssrfPolicy.allowRfc2544BenchmarkRange` setting for `198.18.0.0/15`, the runtime wires it into the SSRF guard, tests cover both default-blocked and opt-in-allowed behavior, and the changelog says it shipped in `v2026.4.10`.

## Close Comment

This looks fixed on current `main`.

- Codex review of `main` found a new opt-in `web_fetch` setting, `tools.web.fetch.ssrfPolicy.allowRfc2544BenchmarkRange`, specifically documented for fake-IP proxy environments that resolve public hosts into `198.18.0.0/15` (`src/config/types.tools.ts:561`, `src/config/schema.help.ts:757`).
- The `web_fetch` runtime now passes that opt-in through to the SSRF/network guard, so RFC 2544 benchmark-range answers can be allowed without disabling the broader private-network protections (`src/agents/tools/web-fetch.ts:389`).
- Regression coverage exists for the exact behavior: blocked by default, allowed when the opt-in is enabled (`src/agents/tools/web-fetch.ssrf.test.ts:159`).
- The changelog says this shipped in `v2026.4.10` (`CHANGELOG.md:948`).
