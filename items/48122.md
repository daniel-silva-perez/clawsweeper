## Summary

Keep open. Current `main` still enforces device identity for remote insecure Control UI unless the break-glass bypass is active, and the issue report says that bypass was already set. The later comments also show a separate Docker bind-mount permission problem (`EACCES` under `/home/node/.openclaw`), which is a plausible environment cause, but that is not strong enough evidence to close the bug as fixed or non-reproducible on `main`.
