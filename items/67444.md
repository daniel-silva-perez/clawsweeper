## Summary

Keep this PR open. Current main still suppresses `ANNOUNCE_SKIP` before the subagent completion-delivery call, and current tests explicitly assert no completion delivery for that case. The PR targets a real remaining behavior difference, but the submitted branch is not ready as-is because review discussion identifies that the changed fallthrough would keep the literal `ANNOUNCE_SKIP` control token as completion content unless it is cleared.

## Best Possible Solution

Keep the PR open for maintainer review and require a follow-up patch before merge: allow `ANNOUNCE_SKIP` to skip only the soft announce step when completion delivery is required, clear the control token before building completion content, preserve `NO_REPLY` suppression, restore/strengthen test assertions and dependency cleanup, and update user docs if the behavior change is accepted.
