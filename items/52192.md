## Summary

Keep open. Current main still has the exact Feishu `sendMedia` behavior described by PR #52192: it sends companion text before checking `mediaUrl`, then falls through to the no-media text fallback. The PR is not implemented or superseded on main, and the existing tests do not cover the text-without-media `sendMedia` case.

## Best Possible Solution

Keep the PR open and finish it by adding the `text?.trim() && mediaUrl` guard in `extensions/feishu/src/outbound.ts`, plus a regression test in `extensions/feishu/src/outbound.test.ts` for text-only `sendMedia`. If maintainers decide `sendMedia` must never accept missing media, update the channel outbound contract and Feishu adapter accordingly instead of silently duplicating text.
