## Summary

Keep open. Current `main` does not already implement most of this PR: the macOS wizard timeout is still `0.75`, gateway restart deferral still does not account for browser-session work, UI nav width persistence is still capped at `400`, and `skills/skill-creator/SKILL.md` still does not document `quick_validate.py`. `main` did ship a separate `session-memory` hook in `v2026.4.22`, so the persistence goal is only partially superseded, not fully implemented as this PR proposes.

## Risks / Open Questions

- This PR is now a mixed bag: one persistence-related idea partly landed through the new bundled `session-memory` hook, while several unrelated DX/UI/gateway changes still have not. Closing as implemented would overstate what `main` actually contains.
- If someone revives the original memory-flush approach, it should be reworked before merge. Current `registerInternalHook` remains append-only (`src/hooks/internal-hooks.ts:220-224`), so the PR's registration pattern would still risk duplicate handlers on repeated gateway starts.
