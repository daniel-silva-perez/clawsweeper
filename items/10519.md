## Summary

Keep open. Current `main` still emits Telegram fenced code blocks as `<pre><code>...</code></pre>` with no language class, and the shared markdown IR/render path does not preserve fence language metadata needed to add `class="language-*"` per block.

## Evidence

- **Telegram formatter hardcodes plain code-block HTML:** `renderTelegramHtml()` maps `code_block` to a fixed `open: "<pre><code>"` / `close: "</code></pre>"`, so no language class is emitted today.
  - file: [extensions/telegram/src/format.ts:66](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/extensions/telegram/src/format.ts#L66)
  - command: `nl -ba extensions/telegram/src/format.ts | sed -n '50,80p'`
  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
- **Current test expectation omits language class:** The Telegram formatter test for a fenced ```js block expects exactly `<pre><code>const x = 1;\n</code></pre>`, confirming current main does not implement the requested output.
  - file: [extensions/telegram/src/format.test.ts:55](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/extensions/telegram/src/format.test.ts#L55)
  - command: `nl -ba extensions/telegram/src/format.test.ts | sed -n '45,65p'`
  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
- **Markdown IR drops fence language metadata:** `MarkdownStyleSpan` stores only `start`, `end`, and `style`, and `renderCodeBlock()` records only a `code_block` span from token content. The `fence` token path does not preserve the info string/language anywhere in the IR.
  - file: [src/markdown/ir.ts:36](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/src/markdown/ir.ts#L36)
  - command: `nl -ba src/markdown/ir.ts | sed -n '20,60p'; nl -ba src/markdown/ir.ts | sed -n '296,312p'; nl -ba src/markdown/ir.ts | sed -n '692,704p'`
  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
- **Renderer only supports static style markers:** `RenderStyleMarker` is a plain `{ open: string; close: string; }` shape, so the renderer cannot currently compute per-code-block opening tags from a captured language value.
  - file: [src/markdown/render.ts:3](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/src/markdown/render.ts#L3)
