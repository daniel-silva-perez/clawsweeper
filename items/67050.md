## Summary

Keep open. Current main still has the critical env-plus-network scanner rule and still walks all scannable plugin package files except hidden paths and node_modules, so the underlying false positive is not implemented away. The linked issue #66840 was closed specifically as superseded by this PR, making this PR the canonical active fix path. However, the proposed blanket test-name/test-directory skip is not safe to merge as-is because review comments identify a trivial scanner bypass and missing regression coverage.

## Best Possible Solution

Keep this PR open as the canonical fix path, but require a safer implementation before merge: preserve scan coverage for runtime-capable files, relax only credible test-file false positives or only the blocking decision for those findings, and add regression tests for both the @axonflow-style env-plus-fetch test mock and malicious test-named/runtime-reachable files.
