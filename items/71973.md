## Summary

Keep #71973 open. Current main already keeps the system prompt's date/time section cache-stable, but the narrower reported issue remains: exact `Current time:` lines are still injected into heartbeat, cron, `/new`/`/reset`, and post-compaction runtime prompt content, and some cache shaping still tags the trailing user turn.

## Best Possible Solution

Keep the issue open for a narrow core prompt/cache-shaping change. The best path is to preserve timezone-only stable system prompt behavior, then redesign heartbeat/cron/post-compaction exact-time delivery so wall-clock data is late-bound, omitted when unnecessary, or placed outside provider cache-control scope. Add tests for heartbeat, cron, post-compaction, `/new`/`/reset`, and Anthropic/OpenAI cache payload shaping so the daily-memory grounding fixes remain intact without unnecessary cache churn.
