## Summary

Keep open. Current main/release has added generic compat knobs and docs for stricter OpenAI-compatible backends, but there is no strong evidence that Mindie’s reported 422 is fixed. The shared Chat Completions transport still unconditionally sends `stream_options.include_usage`, there is no Mindie-specific handling in-tree, and the report’s exact rejected parameter is still unknown.

## Evidence

- **OpenAI-compatible transport still sends stream_options:** `buildOpenAICompletionsParams` always includes `stream_options: { include_usage: true }`, and still forwards `tool_choice` when tools are present. That means the suspected Mindie-incompatible field is not obviously stripped on current main.
