## Summary

Keep open. Current `main` copies skills into the sandbox workspace, but the run path still prefers a pre-sandbox `skillsSnapshot.prompt` built from the agent workspace and skips reloading sandbox-local skill entries when that snapshot exists. That means SKILL.md locations can still point at the original host/global paths instead of the sandbox-local copies described in the issue.

## Evidence

- **Sandbox now syncs skills into copied workspaces:** `ensureSandboxWorkspaceLayout` calls `syncSkillsToWorkspace({ sourceWorkspaceDir: agentWorkspaceDir, targetWorkspaceDir: sandboxWorkspaceDir, ... })` for non-`rw` sandboxes, so current `main` does attempt to copy skills into the sandbox workspace.
  - file: [src/agents/sandbox/context.ts:44](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/agents/sandbox/context.ts#L44)
  - command: `nl -ba src/agents/sandbox/context.ts | sed -n '34,78p'`
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Skill snapshot is built from the agent workspace before sandbox execution:** `agent-command` builds `skillsSnapshot` with `buildWorkspaceSkillSnapshot(workspaceDir, ...)`, where `workspaceDir` is the normal agent workspace resolved earlier, not the later sandbox `effectiveWorkspace`. That snapshot stores `prompt` and `resolvedSkills` from the host workspace.
