## Summary

Keep PR #41222 open. Current main already covers the inline reply/audio directive indentation part through the newer code-block-aware directive parser, but it does not fully implement this PR's media/audio extraction whitespace fix. `splitMediaFromOutput()` still collapses multi-space indentation and trims remaining body text after `MEDIA:` or audio-tag processing, so the PR still represents useful remaining work rather than a fully implemented or safely superseded change.

## Best Possible Solution

Keep the PR open, or replace it with a refreshed narrow patch that updates `splitMediaFromOutput()` to preserve meaningful multiline indentation after `MEDIA:` and audio directive extraction while stripping only directive-introduced whitespace. Add focused tests in `src/media/parse.test.ts` and the current outbound payload tests for indented text after media extraction, plus cases for the standalone directive-line cleanup raised in review.
