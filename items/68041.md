## Summary

Current main only partially covers this PR. The loader-side memory capability preservation is present, but the wiki CLI still does not preload runtime plugins or route bridge import through the gateway, and it still depends on an active memory public-artifacts capability. The remaining work should stay open or be explicitly consolidated with #67063/#69458.

## Best Possible Solution

Keep this PR open for maintainer reconciliation. Rebase or drop the already-implemented loader hunk, then solve the remaining wiki CLI bridge path with a narrow tested fix: either include the configured active memory slot for wiki-scoped CLI loads, likely along the #67063 shape, or route `wiki bridge import` through the gateway RPC. If #67063 or #69458 becomes the canonical landed fix, close this PR as superseded then.
