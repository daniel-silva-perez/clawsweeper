## Summary

Keep open. Current main still exposes dreaming as a global memory-core setting, fans automatic dreaming across all configured agent workspaces, and has no accepted `agents.list[].dreaming` contract. There is also an open same-author implementation PR with closing syntax for this issue, so the maintainer path is to review/land or close #67420 rather than close the issue first.

## Best Possible Solution

Keep this issue open while #67420 is reviewed. The likely implementation path is to add a documented `agents.list[].dreaming` contract, at minimum `enabled`, wire it through the strict config schema/types and `resolveMemoryDreamingWorkspaces`, and add resolver/runtime/docs tests. If per-agent custom schedules remain in scope, decide explicitly whether memory-core should manage multiple scoped cron jobs or whether schedule flexibility should be tracked separately.
