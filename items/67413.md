## Summary

Keep this issue open. Current main still fans automatic dreaming out across all configured agent workspaces, exposes only global dreaming enablement/frequency under the memory plugin config, and does not accept or use `agents.list[].dreaming`. There is also an open same-author PR with closing syntax for this issue, so the correct maintainer path is to review/land or close that PR rather than auto-close the issue first.

## Best Possible Solution

Keep this issue open while the linked open PR is reviewed. The likely product path is to add a documented per-agent dreaming contract, at minimum `agents.list[].dreaming.enabled`, wire it through the strict config schema/types and `resolveMemoryDreamingWorkspaces`, and add memory-core resolver/runtime/docs tests. If per-agent custom schedules remain in scope, that probably needs an explicit design for multiple scoped managed cron jobs or a scheduler-level workspace filter.
