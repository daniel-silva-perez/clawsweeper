## Summary

Keep open. Current main still does not strip inbound metadata in the shared outbound payload plan used by `normalizeReplyPayloadsForDelivery()` and channel delivery, so the PR targets a real remaining gap. The PR should not be auto-closed as implemented, but it also needs maintainer refinement because the proposed `stripInboundMetadata()` call would apply a user-role/UI sanitizer to every assistant outbound payload and can strip legitimate timestamp-prefixed assistant text.

## Best Possible Solution

Keep the PR open for maintainer refinement or consolidation with #39847/#50520. The best fix is a single shared sanitizer in `createOutboundPayloadPlanEntry()` before directive parsing or before final projection, but it should be outbound-specific or leading-only so it strips echoed `Conversation info` / `Sender` blocks without removing legitimate assistant timestamp text. Add regression tests for leading echoed metadata, MEDIA/reply directive preservation, all outbound projections, and timestamp-prefixed assistant output that must remain unchanged.
