## Summary

Keep open. Current main still does not implement the Telegram session selector cleanup from PR #51124, and the PR discussion records real correctness issues in the submitted patch, so this remains actionable maintainer/UI work rather than a conservative auto-close.

## Best Possible Solution

Keep the PR open for maintainer revision against the current `ui/src/ui/chat/session-controls.ts` implementation. A good fix should narrowly clean known Telegram-generated labels/display names, consume `origin.label` through the UI row type, preserve explicit user labels, avoid unconditional main-key remapping, and add focused regression tests for Telegram DM, slash, topic/group labels, and the `main` alias/canonical pair.
