## Summary

Keep #71562 open. Current main's normal workspace bootstrap path does not JSON-parse `TOOLS.md` or other allowlisted workspace Markdown files: it reads fixed workspace filenames as UTF-8 text, converts them into context-file `{path, content}` records, and injects the raw content into Project Context. However, the issue is new and the report does not identify the exact command, stack trace, config path, or runtime path that emitted the JSON parse errors. I also found a separate config `$include` path that intentionally parses included files as JSON5 regardless of filename, so an extension-check request is not conclusively implemented across every OpenClaw JSON-reading surface.

## Best Possible Solution

Keep the issue open and ask for the exact command/runtime path, stack trace, OpenClaw version, and any config that references `TOOLS.md` or other Markdown files. If the root cause is normal workspace bootstrap, add a regression test proving `TOOLS.md` content is loaded as plain text. If the root cause is config `$include`, either document that includes must be JSON/JSON5 or add a clearer extension/error guard for that config surface.
