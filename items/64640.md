## Summary

Keep open. Current main has superseded the webhooks portion through merged #67941, but the memory-search config-resolution fix is still not implemented: `resolveMemorySearchConfig` still uses the registered-only provider registry and does not forward the active `OpenClawConfig` for primary, fallback, or multimodal provider lookup.

## Best Possible Solution

Keep this draft PR open, but require a rebase and narrowing. Drop the obsolete webhooks changes covered by merged #67941, keep the memory-search work, switch `resolveMemorySearchConfig` to the cfg-aware memory embedding provider runtime resolver for primary, fallback, and multimodal validation lookup, add the focused runtime-config forwarding regression test, and run the changed agent/plugin gates before maintainer review.
