## Summary

Keep open. Current `main` already ships the earlier bare-model allowlist fix from `#51580`, but it does not include this PR's harder provider-inference handling, so there is not strong evidence to close the PR as already implemented or obsolete.

## Evidence

- **Inference still compares bare inputs literally:** `inferUniqueProviderFromConfiguredModels` trims the input and matches configured models, but on current `main` it does not strip trailing auth-profile suffixes and it skips bare allowlist keys entirely. That means inputs like `claude-opus-4-6@work` can still miss a unique configured match.
  - file: [src/agents/model-selection-shared.ts:52](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/src/agents/model-selection-shared.ts#L52)
  - command: `nl -ba src/agents/model-selection-shared.ts | sed -n '45,110p'`
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **Resolver infers provider before profile stripping:** `resolveAllowedModelRefFromAliasIndex` calls `inferUniqueProviderFromConfiguredModels({ model: trimmed })` before `resolveModelRefFromString`, and only `resolveModelRefFromString` strips the trailing auth profile. So a bare `model@profile` can still fall back to the wrong default provider first.
