## Summary

Keep open. Current main does pass `channels.telegram.apiRoot` into the grammY client and now dispatches Telegram messages eagerly again, but the issue’s migration-specific bugs are not fully addressed: the persisted polling offset is still keyed only by account/bot identity rather than `apiRoot`, offset cleanup only runs when the bot token changes, and `deleteWebhook` setup still exits on non-network errors such as `400 Logged out`. That is not strong enough evidence for closure.

## Evidence

- **`apiRoot` is wired into the Telegram client:** Current main still constructs the grammY client with `apiRoot`, so outbound calls like `deleteWebhook` should use the configured base URL rather than hardcoding `api.telegram.org`. This weakens only the reporter’s suspicion about the endpoint, not the rest of the bug.
  - file: [extensions/telegram/src/bot-core.ts:264](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/telegram/src/bot-core.ts#L264)
  - command: `nl -ba extensions/telegram/src/bot-core.ts | sed -n '256,276p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Persisted update offsets are not scoped by `apiRoot`:** The offset file path is `update-offset-<account>.json`, and the stored state carries only `lastUpdateId` plus `botId`. There is no `apiRoot` dimension, so switching the same bot token between official Telegram and a self-hosted Bot API server will reuse the old offset.
  - file: [extensions/telegram/src/update-offset-store.ts:27](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/telegram/src/update-offset-store.ts#L27)

- **Runtime still reloads offsets by account + bot token only:** Polling startup reads the persisted offset with `{ accountId, botToken }` and then keeps writing back the same way. There is no reset or namespacing when `apiRoot` changes.
  - file: [extensions/telegram/src/monitor.ts:175](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/telegram/src/monitor.ts#L175)

  - file: [extensions/telegram/src/channel.ts:725](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/telegram/src/channel.ts#L725)
