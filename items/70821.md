## Summary

Keep #70821 open. Current main still has the Bedrock-specific gap the PR targets: `amazon-bedrock` advertises only `off/minimal/low/medium/high` plus `adaptive` for Claude 4.6, while the shared Anthropic transport and Anthropic provider already have Opus 4.7 xhigh handling. The related closed Opus 4.7 work does not supersede this PR because the missing behavior is in the Bedrock plugin’s own `resolveThinkingProfile` seam.

## Best Possible Solution

Do not close this PR as implemented or duplicate. Merge or recreate the small Bedrock plugin change on current main: add an Opus 4.7 Bedrock predicate to `extensions/amazon-bedrock/register.sync.runtime.ts`, advertise `xhigh` from `resolveThinkingProfile` for matching models, keep 4.6 adaptive behavior unchanged, and add the focused `extensions/amazon-bedrock/index.test.ts` regression before running the amazon-bedrock extension test lane.
