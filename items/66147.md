## Summary

Keep this PR open. Current main still does not implement `model.tool` or split routing for tool-continuation turns, and the PR remains a plausible but incomplete implementation candidate rather than an obsolete, duplicate, or ClawHub-only item. The security pass found no new dependency, workflow, lockfile, release-script, or third-party code execution change in the provided PR file list, but the runtime/auth/transport boundary concerns are significant enough that it needs maintainer review before merge.

## Best Possible Solution

Keep this PR open for maintainer review or author revision. The best path is to decide whether split model routing belongs in the core `agents.defaults.model` contract or in a new plugin/API seam. If it stays core, the effective per-call model should drive tool enablement, model resolution, auth profile and API key selection, transport dispatch, provider wrappers, transcript policy, context budgeting, fallbacks, diagnostics, docs, generated config baselines, and tests. If maintainers prefer the plugin route, use #13225 as the design thread for the missing per-tool routing seam instead of closing this PR as already handled.

## Risks / Open Questions

- The PR’s advertised local-primary plus API-tool workflow may not work if the primary model cannot emit tool calls, because the first tool-emitting turn and `toolsEnabled` path are still governed by the primary model.
- Cross-provider routing is security-sensitive: the provided diff appears to swap only the `Model` argument while primary-provider transport, auth profile/API key lookup, text transforms, transcript policy, diagnostics, and stream wrappers may remain bound to the primary provider.
- The PR changes public config shape; generated config docs/baselines and user-facing documentation would need to be updated before accepting `model.tool` as a core contract.
- The provided PR body says manual Ollama plus Anthropic integration and full `pnpm build && pnpm check && pnpm test` validation were still pending.
- Security review did not find evidence of supply-chain expansion in the provided file list, but the auth/transport misrouting risk is enough to keep the PR under maintainer review.
