## Summary

Keep open. The requested validation is not implemented on current main: the nodes CLI gateway runtime still uses raw Number(opts.timeout ?? 10_000). The PR is still plausibly useful, though it likely needs rebase/adaptation after the rpc.runtime split and should address the existing review concern about parseInt semantics changing values like 1e3.

- The PR patch shown in the provided context targets src/cli/nodes-cli/rpc.ts, while current main has the gateway call in src/cli/nodes-cli/rpc.runtime.ts; the PR may need rebase/adaptation before it can land.
