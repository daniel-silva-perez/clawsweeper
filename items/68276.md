## Summary

Keep this PR open. Current main does not implement the Android-side cron marker filter or role-scoped `<final>` unwrapping: Android still turns every `chat.history` row into a `ChatMessage`, and text content is copied verbatim. The gateway history projection does some display normalization, but the inspected current path covers envelopes, inline directives, silent assistant tokens, heartbeat noise, phase filtering, and truncation rather than this PR's exact `<final>` wrapper behavior. The PR remains a focused implementation candidate, with review feedback already acknowledged by a later PR commit, so the right maintainer path is review/rework rather than conservative cleanup close.

## Best Possible Solution

Keep the PR open and review or rework it against current main. Prefer a tested fix that either normalizes these artifacts in the shared `chat.history` projection for all clients or adds narrowly scoped Android parsing with regression tests; cron filtering should use exact metadata or an exact marker shape, and `<final>` stripping should apply only to assistant/system output without mutating user-authored examples.
