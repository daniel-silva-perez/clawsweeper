## Summary

Keep #68596 open. Current main still has a 30,000 ms TUI streaming watchdog default, the production TUI does not pass a user/config override into the handler, and the config/schema/docs surfaces expose ACP stream coalescing and CLI backend watchdog tuning but not the requested client-side streaming watchdog threshold. The later #69978 is related and broader, while PR #69026 is a focused attempted fix, but neither is merged or clearly canonical enough to supersede this older, active configurability request.

## Best Possible Solution

Add a supported user-facing TUI streaming watchdog setting, likely by reading an `openclaw.json` or persisted TUI preference value and plumbing it into `createEventHandlers` as `streamingWatchdogMs`; document the setting, add schema/help coverage if it lives in config, and add focused tests for default 30s behavior, a longer threshold, and disabled mode. If `/watchdog` runtime control and duplicate-warning suppression are desired, coordinate with #69978 or use it as the UX follow-up rather than closing this issue prematurely.
