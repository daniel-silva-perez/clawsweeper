## Summary

Keep #71140 open. Current main still implements Telegram reaction primitives and MEDIA/GIF delivery, but it does not parse `[[react...]]` / `[[react_to_current:...]]` into structured reply payloads, does not allow reaction-only Telegram replies through the delivery path, and does not execute `channelData.telegram.reaction` in `telegramOutbound.sendPayload`. The linked PR #71141 is an open draft and the maintainer comment on #71140 explicitly says the reaction directive path remains unimplemented, so the issue is not obsolete.

## Best Possible Solution

Keep #71140 open as the canonical issue for the remaining Telegram outbound reaction-directive gap. The best fix is to extend shared reply directive parsing/normalization to emit structured reaction metadata, allow reaction-only Telegram replies when a reply target exists, execute that metadata through `reactMessageTelegram` in both reply delivery and `sendPayload`, and add focused parser, payload, delivery, and outbound-adapter tests. PR #71141 may be the candidate implementation, but it needs review/merge before this issue can close.
