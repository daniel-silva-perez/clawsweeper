## Summary

Keep open. The issue is recent, has multiple independent reproductions in the discussion, and current main still shows at least one reported failure class: subagent announce idempotency remains scoped to child run id, so remap/resume can plausibly create a new announce key for the same logical completion. Current main has some Telegram and Slack dedupe mitigations, but they do not prove the umbrella bug is fixed across the reported surfaces.
