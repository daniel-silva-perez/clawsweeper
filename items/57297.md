## Summary

Current `main` partially fixed WebChat TTS delivery by embedding audio into WebChat transcript/UI, but it does not implement the issue's requested explicit routing/playback telemetry or TTS-specific delivery outcome events. The acceptance criteria for structured gateway logs and correlation-stage visibility are still unmet.

## Evidence

- **`tts.convert` only returns synthesis output metadata:** The gateway TTS RPC returns `audioPath`, `provider`, `outputFormat`, and `voiceCompatible`. It does not emit or return a selected route, delivery outcome, playback result, or correlation id for WebChat delivery.
