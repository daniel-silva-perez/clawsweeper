## Summary

Keep open. Current main still appends the profile suffix under `OPENCLAW_HOME`, while the core config path resolver defaults `OPENCLAW_HOME` to an unsuffixed `.openclaw` state dir when `OPENCLAW_STATE_DIR` is unset. The PR remains a focused, relevant fix candidate and its latest diff addresses the bot review points rather than being obsolete or duplicated.

## Best Possible Solution

Keep the PR open for maintainer review. If maintainers agree that `OPENCLAW_HOME` is already the physical isolation boundary, land a normalized guard in `resolveProfileStateDir`, retain tests for named profiles, `dev`, and sentinel values, and update user-facing profile docs if the exception should be documented.
