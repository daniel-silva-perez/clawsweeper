## Summary

Current main still promotes the selected daily-memory snippet text directly into MEMORY.md. The deep promotion path rehydrates from the live daily note, filters some contaminated dreaming artifacts, and appends `candidate.snippet` with score/provenance metadata; there is no distillation/summarization step or config flag for distilled promotion. The related metadata-contamination thread/PR is narrower and does not supersede this issue's broader durable-memory distillation request.

## Best Possible Solution

Keep this issue open for maintainer product/implementation work in `extensions/memory-core`: decide whether deep promotion should write raw snippets by design or introduce a distilled-promotion step/flag, then update `buildPromotionSection()` or the apply pipeline, add regression coverage for distilled output versus provenance, and clarify the Dreaming/memory docs.
