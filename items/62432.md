## Summary

Keep open. Current `main` now has clearer exec/process guidance and an opt-in tool-loop detector, but the detector is disabled by default and I did not find a default runtime guard that forces `process` or rejects duplicate `exec` relaunches after a `status: "running"` result. That is not strong enough evidence to close this Xiaomi/MiMo orchestration bug as fixed.

## Evidence

- **Exec returns a running status plus process follow-up hint:** The exec tool still returns `Command still running ... Use process ... for follow-up` together with structured `status: "running"` and `sessionId`, so the intended follow-up path is present.
