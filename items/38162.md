## Summary

Keep this PR open. Current main still lacks the requested `sessionSaveRedirectPath` behavior and the deferred `postHookActions` / `blockSessionSave` / `sessionSaveContent` foundation it depends on, while the paired #38161 PR by the same contributor remains open. The remaining work is core bundled-hook filesystem policy with security-review context, not conservative cleanup closure material.

## Best Possible Solution

Keep this PR open. Resolve or explicitly supersede #38161 first, then review this redirect-specific core follow-up with maintainer-approved workspace-containment semantics, symlink/retraction behavior, late block/unblock behavior, public hook context documentation, and focused tests. If maintainers decide not to add this bundled-hook policy surface, close only after that product/security decision and point plugin authors to an approved external hook or plugin API replacement path.
