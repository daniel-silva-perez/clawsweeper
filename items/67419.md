## Summary

Current main does not implement the issue's primary default-change request: missing `agents.defaults.contextInjection` still resolves to `always`, tests assert that default, docs describe it as default, and `MEMORY.md` remains a normal injected bootstrap file. The related fix PR #67447 is closed unmerged in the provided context, while maintainer discussion links #69208 as an umbrella but explicitly says to keep this issue for its concrete repro unless it becomes a strict duplicate.

## Best Possible Solution

Keep this issue open. The narrow default-change path should be implemented or explicitly rejected in the agent bootstrap/config surface, likely by reviving or replacing the #67447 approach with updated tests and docs. The MEMORY.md stub/token-cost behavior and the configurable dynamic bootstrap file/cache-boundary work should either be implemented here or split to canonical follow-ups such as #65438, while #69208 remains the umbrella for broader transcript duplication analysis.
