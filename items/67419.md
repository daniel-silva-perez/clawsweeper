## Summary

Current main still has the reported behavior: missing `agents.defaults.contextInjection` resolves to `always`, tests and docs still describe that as the default, `MEMORY.md` remains part of normal bootstrap injection, and the dynamic bootstrap-file config requested here is not present. The maintainer-linked umbrella #69208 explicitly says concrete reports like this should not be automatically erased, and the current open closing PRs are unmerged implementation candidates, not a reason to close this issue.

## Best Possible Solution

Keep this issue open. The best path is to implement or explicitly reject the narrow default change in `src/agents/bootstrap-files.ts`, update config schema/help/docs and regression tests accordingly, and separately handle the `MEMORY.md` stub/token-cost work plus configurable dynamic bootstrap files through this issue or canonical follow-ups such as #65438. Review or close the open README-only closing PRs rather than letting them resolve the runtime bug.
