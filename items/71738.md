## Summary

Keep #71738 open. Current main still lacks the PR's central Matrix changes: room history is keyed only by room, Matrix history reads have no thread-aware path, thread sends still include fallback reply metadata unconditionally, and threaded edits still put reply context on the outer replace relation. The PR is a focused attempt to fix a real bundled Matrix behavior, but its current version has active P1/P2 review findings that should be addressed before merge rather than using maintainer cleanup to close it.

## Best Possible Solution

Keep #71738 open as a useful Matrix plugin fix candidate. The best path is to revise the PR against current main, address the active P1/P2 review comments, and land an equivalent implementation with tests for per-thread room-history queues and watermarks, thread-aware reads, main-room filtering without pagination loss, thread send fallback metadata, and threaded edit relation placement.
