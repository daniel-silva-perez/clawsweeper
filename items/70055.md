## Summary

Keep #70055 open. Current main has refactored the old sync into runtime-only external auth overlays and no longer appears to emit the exact info-level "synced ... credentials from external cli" log, but the requested first-class opt-out is still missing: the config schema is strict and only exposes auth.profiles/order/cooldowns, while external CLI credentials are still resolved from the hardcoded provider list whenever auth profiles are overlaid.

## Best Possible Solution

Add an explicit, default-on external CLI auth overlay control, preferably under auth.externalCliSync with boolean plus optional provider include/exclude semantics or a simpler env var first. Wire it into overlayExternalAuthProfiles/resolveExternalAuthProfileMap so disabled providers are not read at all, update the config type, zod schema, generated schema/help/docs, and add focused tests proving auth.externalCliSync: false prevents Codex/Claude/MiniMax external credential readers from being called while preserving current default behavior.
