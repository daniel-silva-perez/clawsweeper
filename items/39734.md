## Summary

Keep open. Current main partially covers the concrete Bedrock Claude 4.6 1M context metadata path, and the issue discussion notes the original 1M beta flag may no longer be required at GA. But the requested configurable `anthropic_beta` / provider-field passthrough for the standard `amazon-bedrock` provider is still absent from the current plugin schema, model schema, runtime wrapper, and docs.

## Best Possible Solution

Keep the issue open for maintainer product judgment or implementation. A concrete fix would add a Bedrock-owned, validated config/params surface for allowed `anthropic_beta` or additional Bedrock Converse request fields, merge it in the `amazon-bedrock` runtime wrapper, and cover it with Bedrock plugin tests plus `docs/providers/bedrock.md`. If maintainers decide arbitrary beta passthrough is outside scope now that the 1M GA use case no longer needs it, that should be an explicit product decision rather than an automated cleanup close.
