## Summary

Issue #9372 proposes a new bundled Webex channel under `extensions/webex/`, but current `main` treats new channel integrations as external plugins distributed via ClawHub or npm rather than additions to core.

## Evidence

- **Vision prefers external plugins:** Current vision says optional capability should usually ship as plugins, plugin authors should host and maintain them in their own repository, and the bar for adding optional plugins to core is intentionally high.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/VISION.md#L54)

## Close Comment

Closing as a Codex review because this request fits OpenClaw's external plugin path rather than core. Current `main` says optional capability should usually ship as plugins, plugin authors should host and maintain them in their own repo, and the bar for adding optional plugins to core is intentionally high (`VISION.md:54`). The plugin docs also say channel integrations do not need to be added to this repository and should be published on ClawHub or npm instead (`docs/plugins/building-plugins.md:16`, `docs/plugins/community.md:9`, `docs/plugins/sdk-channel-plugins.md:11`). Since this issue is asking for a new bundled `extensions/webex/` channel rather than a missing core plugin seam, the maintainable path on current main is a community Webex plugin distributed through ClawHub/npm. If you pursue that and hit a concrete SDK gap, please open a focused issue for the missing plugin API seam.
