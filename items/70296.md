## Summary

Keep this issue open. Current main still has no restart-sentinel channel-notification suppression surface, and BOOT.md startup runs still append the hard-coded OpenClaw silent token after workspace BOOT.md content. The Control UI model read-back portion appears covered on current main, but two substantive gateway/runtime concerns remain unresolved.

## Best Possible Solution

Keep this issue open and narrow it around the remaining work: add a documented restart-sentinel channel-notification policy/config key and guard the outbound notice path in src/gateway/server-restart-sentinel.ts, then settle and test the BOOT.md silent-token contract in src/gateway/boot.ts. Preserve the current Control UI effective-model read-back behavior unless a fresh reproduction shows a remaining UI regression.
