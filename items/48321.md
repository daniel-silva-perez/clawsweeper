## Summary

Keep open. Current main does not implement image timestamp/position annotations for thread history, and Slack still forwards thread-starter media into later replies without any added age/position metadata.

## Evidence

- **No annotation feature or config found:** A repo-wide search found no `annotateImages`, `thread.annotateImages`, or image-annotation marker matching the requested feature.
  - command: `rg -n "annotateImages|thread\.annotateImages|image metadata|message [0-9]+ of [0-9]+ in thread|sent .* EDT|from User\]" . -g '!**/dist/**' -g '!**/node_modules/**'`
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **Slack thread history is text-only envelopes:** Slack thread history is assembled from text envelopes with sender and timestamp, but there is no logic that injects metadata lines before image payloads.
  - file: [extensions/slack/src/monitor/message-handler/prepare-thread-context.ts:211](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/extensions/slack/src/monitor/message-handler/prepare-thread-context.ts#L211)
  - command: `nl -ba extensions/slack/src/monitor/message-handler/prepare-thread-context.ts | sed -n '120,235p'`
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **Slack still reuses root-thread media on replies:** When a reply has no direct media, Slack hydrates the thread starter files and later assigns them into `MediaPath`/`MediaPaths` for the agent run, which means an older root image can still be sent to the model without temporal/position annotation.
  - file: [extensions/slack/src/monitor/message-handler/prepare-thread-context.ts:124](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/extensions/slack/src/monitor/message-handler/prepare-thread-context.ts#L124)

  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **Slack context payload has no image-metadata field:** The finalized Slack inbound context includes `ThreadStarterBody`, `ThreadHistoryBody`, and media paths, but no per-image timestamp, author, or position metadata field.
  - file: [extensions/slack/src/monitor/message-handler/prepare.ts:689](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/extensions/slack/src/monitor/message-handler/prepare.ts#L689)
