## Summary

Keep open. Current main has a friendly catalog-label helper for the chat model picker, and Anthropic runtime code recognizes Claude Opus 4.7/Sonnet 4.6, but the Agents/Control UI primary-model dropdown path still builds labels from configured aliases or raw `provider/id` values. For configured `agents.defaults.models` entries, the raw configured option is added before catalog options, so a catalog display name would not replace the stale/raw label in the reported workflow.

## Best Possible Solution

Keep this issue open and fix the Agents/Control UI model option builder to share the catalog display-name logic already used by the chat model picker. The likely change belongs in `ui/src/ui/views/agents-utils.ts`: build a qualified catalog display lookup, render catalog rows with `entry.alias || entry.name`, and let configured `agents.defaults.models` entries use matching catalog names when available while preserving explicit user aliases where appropriate. Add a focused UI helper test for `anthropic/claude-opus-4-7` and `anthropic/claude-sonnet-4-6` so configured entries and catalog entries both render full display names.
