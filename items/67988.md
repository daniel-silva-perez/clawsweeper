## Summary

Keep this issue open. Current main still routes the Agents/Control UI primary-model dropdown through a separate option builder that labels configured models from aliases or raw model ids, dedupes those before catalog rows, and does not use the friendly catalog display-name helper used by the chat picker. Anthropic runtime recognition for Claude Opus 4.7 and Claude Sonnet 4.6 exists, but it does not fix this UI label path.

## Best Possible Solution

Keep this issue open and fix the Agents/Control UI model option builder in ui/src/ui/views/agents-utils.ts. Resolve configured model values against the qualified model catalog before falling back to aliases/raw ids, share the catalog display-name logic already used by the chat model picker, and add focused UI tests for configured anthropic/claude-opus-4-7 and anthropic/claude-sonnet-4-6 rendering as full Claude display names. If Opus 4.7 is still absent from the catalog, add provider-owned catalog metadata/augmentation for that Anthropic row rather than hard-coding Anthropic names in generic UI code.
