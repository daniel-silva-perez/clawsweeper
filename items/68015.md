## Summary

Keep open. Current main partially covers the config-enum guidance and has separate task/session/process primitives, but the central field-report requests are not fully implemented or clearly superseded: there is still no unified operator view for runtime fanout/MCP child processes, no documented model-free local maintenance-runner pattern, and no canonical related item that owns the same remaining work.

## Best Possible Solution

Keep this issue open for maintainer triage, then split it into narrower work: a config-validation/operator-hint slice for `tools.exec.host`, a runtime observability slice spanning process-tool sessions, MCP stdio/runtime sessions, subagent/task state, and Control UI/CLI status, plus a docs slice for external deterministic maintenance runners such as systemd user timers when model turns are unnecessary.
