## Summary

Keep #63112 open. Current main still does not implement the warning requested by the PR/linking issue: the CLI accepts main-session `--system-event` payloads and sends them to `cron.add`/`cron.update` without any shell-command heuristic or stderr warning, while the runtime path only enqueues the text as a system event. The provided PR context shows the PR is open/unmerged and its later commits addressed the bot review comments, so this remains useful work rather than an obsolete cleanup target.

## Best Possible Solution

Keep the PR open and have the contributor or maintainer refresh it against current main. The best implementation is still a shared shell-looking heuristic plus non-blocking stderr warning in both `cron add` and `cron edit`, including the edit path that infers an existing main-session target when `--session` is omitted. Add focused tests in `src/cli/cron-cli.test.ts` for warning and non-warning cases.
