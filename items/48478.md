## Summary

Current `main` already has the requested inbound interrupt behavior under the existing queue system: `messages.queue.mode: "interrupt"` aborts the active run for that session and restarts on the newest inbound message, with per-session `/queue interrupt` override support. The same behavior is present in the released `v2026.4.22` tree.
