## Summary

Current `main` has SecretRef support for `env`, `file`, and generic `exec` providers, and docs include a 1Password CLI example via `exec`, but there is no dedicated keyring provider or dedicated 1Password provider implementation matching this issue’s requested scope.

## Evidence

- **Secret provider types are limited to env/file/exec:** `src/config/types.secrets.ts` defines `SecretRefSource` as only `"env" | "file" | "exec"`, and `SecretProviderConfig` only unions `EnvSecretProviderConfig`, `FileSecretProviderConfig`, and `ExecSecretProviderConfig`.
  - file: [src/config/types.secrets.ts:3](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/src/config/types.secrets.ts#L3)
