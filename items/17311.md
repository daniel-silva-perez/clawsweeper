## Summary

Keep open. Current main ships the env/file/exec SecretRef foundation and documents 1Password through an exec recipe, but it still does not provide native keyring/Keychain or native 1Password sources, nor the pluggable SecretRef provider seam described as the remaining path. The provided discussion also links same-author PR #72548 as an open implementation candidate for the remaining work, so this issue should stay open until that PR is reviewed, landed, or explicitly declined.

## Best Possible Solution

Review #72548 as the concrete implementation candidate. If it lands and maintainers accept pluggable SecretRef sources plus the bundled keyring plugin as the intended architecture, close this as implemented and split any native 1Password follow-up if needed. If the PR is declined, record the product decision and either narrow this issue to the remaining native provider/API gap or close it only with an explicit maintainer rationale.
