## Summary

Current main does not implement the requested Nextcloud Talk rich-object mention handling. The webhook parser keeps `object.content` as an opaque string, while inbound mention gating only runs generic regex matching over that raw string and has no explicit mention signal for `{mention-userN}` placeholders or `parameters` metadata.

## Best Possible Solution

Keep this issue open. The fix should live in `extensions/nextcloud-talk`: parse `object.content` rich-object JSON into display/plain message text plus mention parameters, resolve `{mention-userN}` entries against the bot/agent identity or an explicit channel account bot-user config if needed, pass an explicit mention signal into the existing shared mention decision path, and add regression tests for group messages where `requireMention: true` should pass only when the placeholder metadata targets the bot.
