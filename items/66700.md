## Summary

Keep open. Current main still treats Nextcloud Talk webhook content as raw text and only applies regex mention matching, while the upstream bot webhook contract sends JSON-encoded rich-object content with placeholder parameters.

## Best Possible Solution

Keep this issue open. The fix should live in `extensions/nextcloud-talk`: parse webhook `object.content` as rich-object JSON, normalize display/plain text, preserve `parameters`, detect `{mention-userN}` placeholders that target the bot account identity (using `apiUser` or an explicit bot-user config if `apiUser` is not reliable), feed that as an explicit mention fact into the shared mention decision path, and add regression tests for group rooms with `requireMention: true` where placeholder mentions to the bot pass and non-bot placeholders still drop.
