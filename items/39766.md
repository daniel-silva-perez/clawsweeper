## Summary

Keep open. Current main still requires `models.providers.<id>.models`, so the bug this PR targets is not implemented on main. The PR discussion also shows the author narrowed the branch to the relevant config fix after dropping unrelated UI work, so it remains a plausible useful PR rather than stale noise.

## Best Possible Solution

Keep this PR or the linked bug #39589 open until main accepts credential/baseUrl-only `models.providers.<id>` entries. The best implementation is a small config-contract fix that defaults omitted provider `models` to `[]`, regenerates the schema contract, keeps exported parsed types coherent, and adds focused regression coverage for both schema parsing and memory embedding provider `baseUrl` fallback with SSRF/DNS stubs.
