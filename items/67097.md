## Summary

Current main still has the reported class of behavior: models.json planning resolves implicit bundled/plugin provider catalogs before mode-specific handling, Codex still exposes synthetic/fallback catalog paths, and Arcee still contributes provider catalog entries from Arcee/OpenRouter auth plus configured catalog augmentation. The related #66957 is narrower replace-mode work; this broader onboarding/configuration contract still needs maintainer judgment.

## Best Possible Solution

Keep this open for a maintainer decision on the models.json contract. If models.mode: "replace" is the intended opt-out, short-circuit implicit discovery before resolveImplicitProviders and add regression coverage. Separately decide whether default merge mode should include bundled Codex/Arcee catalogs based on auth/runtime intent, then either document that explicitly or gate those plugin catalog hooks accordingly.
