## Summary

Current main still has unresolved write-time implicit provider behavior: models.json generation resolves bundled provider catalogs before applying mode-specific merge behavior, and Codex still contributes a synthetic/fallback catalog path. The related replace-mode reports are narrower; this issue’s broader onboarding/configuration contract still needs maintainer product/API judgment.

## Best Possible Solution

Keep this issue open for a maintainer decision on the models.json contract. The likely fix path is to define whether generated models.json is allowed to include implicit bundled provider catalogs, gate write-time implicit catalogs on explicit provider/auth/config intent when required, and add focused regression coverage. If maintainers decide models.mode: "replace" is the intended opt-out, fold that piece into the existing #66957/#68965 replace-mode work rather than treating this broader onboarding/configuration concern as already resolved.
