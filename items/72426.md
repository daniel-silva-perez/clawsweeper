## Summary

Keep open. Current main still sends Google Live tool responses with only `id` and `response`, while the pinned `@google/genai` 1.50.1 type contract marks `FunctionResponse.name` as required. The PR is a focused implementation candidate with a regression test, is paired with open issue #72425 from the same author, and overlaps with broader open PR #72189, so it needs maintainer review/selection rather than automated closure.

## Best Possible Solution

Review this PR as the narrow fix for #72425, with the Google provider test updated to require `name` in `FunctionResponse` and a fail-closed path for unknown call IDs. If maintainers prefer the broader Google Live consult/continuation work, make #72189 the canonical implementation and close or supersede this PR explicitly after that decision; otherwise rebase and land this smaller PR with the targeted Google extension checks.
