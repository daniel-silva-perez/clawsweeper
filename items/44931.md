## Summary

Keep open. Current `main` adds better diagnostics for OAuth refresh failures, but I did not find an implementation for the core ask: a user-visible channel alert or automatic recovery when gateway/runtime OAuth refresh fails. The runtime refresh path still only logs a warning and retries once, and `gateway status` still reports reachability/capability rather than a distinct auth-failed state.
