## Summary

Keep this PR open. Current main still has the reported failed-copy cleanup gap in `syncSkillsToWorkspace`: per-skill `fsp.cp` failures only log and continue, leaving no cleanup of the destination path. The downstream Claude skills plugin still logs `claude skill plugin skipped missing skill file` for snapshot entries whose `filePath` is absent, so the reported symptom remains plausible. The PR is a narrow implementation candidate, with review follow-up focused on adding a regression test and deciding whether cleanup failures should be logged.

## Best Possible Solution

Keep this PR open. The best path is to add a focused regression test in the existing skills sync suite, address or explicitly accept the cleanup-failure logging concern, then land the narrow cleanup in `syncSkillsToWorkspace` after maintainer review.
