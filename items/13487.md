## Summary

Current `main` does not appear to implement the requested Discord single-target routing precedence. Inbound Discord routing is resolved before mention/reply evaluation, and the route resolver only considers bindings/account/guild/channel/default fallback. Reply metadata is used as an implicit mention signal, not as a route override, and I did not find regression tests covering mention > reply-target > default-owner suppression.

## Evidence

- **Preflight picks route before mention/reply checks:** `preflightDiscordMessage()` resolves `route`/`effectiveRoute` from conversation bindings and channel context first, then builds mention regexes from that chosen agent. There is no branch here that reroutes based on explicit Discord mentions or replied-to bot author.
