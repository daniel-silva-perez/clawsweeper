## Summary

Keep #7006 open. Current main still does not expose the concrete model selected by OpenRouter for `openrouter/auto`, and the response usage/cost surfaces still rely on normalized token usage plus local pricing estimates rather than OpenRouter's provider-reported `usage.cost`/`cost_details`. The ClawRouter link is a useful external workaround, but the reporter correctly clarified that it is a separate router and does not solve OpenRouter auto transparency; #9016 is cost-only and PR #43026 was closed unmerged.

## Best Possible Solution

Keep #7006 open. The best fix is to capture OpenRouter's routed `model` and numeric per-response cost (`usage.cost` plus relevant `usage.cost_details`) in the OpenAI-compatible transport, preserve selected versus resolved model identity in assistant/session metadata, and thread that through response footers, `/usage`, transcripts, status surfaces, and focused regression tests for `openrouter/auto`. Treat #9016 as related cost-only work, not a replacement for the resolved-model transparency request.
