## Summary

Keep open. Current main still lacks `allowedChatIds`/`deniedChatIds`, and the PR addresses a real Active Memory scoping/privacy gap, but the submitted implementation should not be closed as obsolete or merged as-is because the PR discussion identifies parser gaps for supported session-key shapes and threaded sessions.

## Best Possible Solution

Keep the PR open and ask for a focused revision: parse conversation ids through existing session-key utilities or a shared helper, strip generic `:thread:<id>` suffixes before matching, support all current direct/group/channel key shapes, and add regression tests for per-peer, per-channel-peer, per-account-channel-peer, group/channel, and threaded channel cases. Then the manifest/docs additions can land as a plugin-owned Active Memory feature.
