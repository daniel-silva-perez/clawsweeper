## Summary

Keep open. Current `main` already has the Gateway Logs/Event Log CSS classes and scroll constraints, but it does not implement the PR's whitespace-only line filtering in the log-tail renderer, so the reported layout inflation fix is only partially present.

## Evidence

- **Current log-tail renderer still preserves blank ANSI-stripped lines:** `renderOverviewLogTail` slices the last 50 lines, runs `stripAnsi`, and immediately joins with `"\n"`; there is no `.filter((line) => line.trim() !== "")` or equivalent on `main`. That means whitespace-only lines can still render as vertical space.
  - file: [ui/src/ui/views/overview-log-tail.ts:21](https://github.com/openclaw/openclaw/blob/40be5ad58187fd20b8defe5fc30e649190c131a8/ui/src/ui/views/overview-log-tail.ts#L21)
  - command: `nl -ba ui/src/ui/views/overview-log-tail.ts | sed -n '1,120p'`
  - sha: [40be5ad58187](https://github.com/openclaw/openclaw/commit/40be5ad58187fd20b8defe5fc30e649190c131a8)
- **Overview log loading still forwards raw string lines:** `loadOverviewLogs` takes `payload.lines`, filters only by `typeof line === "string"`, and appends them to `host.overviewLogLines`. There is no upstream trimming or blank-line removal before the renderer.
