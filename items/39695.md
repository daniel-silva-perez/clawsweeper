## Summary

Keep open. Current main has improved fresh local onboarding defaults, Discord/session plumbing, and some incomplete-turn error surfacing, but it does not fully close the reported legacy Discord exec failure: messaging profiles still exclude exec, existing explicit profiles are preserved, the targeted legacy migration PR #40202 was closed unmerged, and the silent-failure surfacing PR #55923 remains open/partial.

## Best Possible Solution

Keep #39695 open until maintainers either ship a targeted legacy local-onboard profile migration like #40202, add explicit diagnostics when SOUL/profile expectations require `exec` but policy excludes it, or land/split a canonical silent-failure surfacing fix such as #55923 and then re-evaluate this issue against that shipped behavior.
