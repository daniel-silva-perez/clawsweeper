## Summary

Keep open. Current main still wires the enabled-by-default Bonjour gateway discovery plugin to `@homebridge/ciao`, and the lockfile still resolves that dependency to `1.3.6`, the version whose Windows ARP probe calls `child_process.exec("arp -a | findstr /C:\"---\"")` without `windowsHide`. Upstream has since fixed this in `@homebridge/ciao@1.3.7`, but OpenClaw main has not absorbed that dependency update or added a local patch.

## Best Possible Solution

Keep #70197 open as the canonical OpenClaw tracker. The likely fix is to update the bundled Bonjour plugin dependency to `@homebridge/ciao@1.3.7` or apply an equivalent local patch, refresh the lockfile, run the relevant Bonjour/gateway changed checks, and verify on Windows with Bonjour enabled that the gateway no longer spawns visible `cmd.exe`/`conhost.exe` for the ARP probe.
