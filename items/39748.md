## Summary

Keep open. Current main still accepts arbitrary Nostr relay strings in the config schema, setup entry points only check ws/wss schemes without restricting plaintext ws:// to local hosts, and runtime forwards configured relays directly to the Nostr bus. The PR addresses a real remaining gap and the discussion improved the localhost-only ws:// behavior, but it needs a refresh against current main.

## Best Possible Solution

Keep the PR open and refresh it against current main. The best fix is to introduce one shared Nostr relay URL validator that allows wss:// for remote relays and ws:// only for localhost/loopback, then use it from NostrConfigSchema, CLI setup, Control UI setup, and focused regression tests.
