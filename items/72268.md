## Summary

Keep #72268 open. Current main still has the nested-parentheses parsing bug the PR targets: exec approval result parsing uses `([^)]*)`, while the gateway can emit denial metadata like `approval-timeout (allowlist-miss)`. That makes current main misclassify the denied async followup as non-denied. The related #72148 is complementary and does not supersede this parser fix.

## Best Possible Solution

Keep #72268 open for maintainer review. The best path is to land a focused parser fix with nested denied/finished metadata tests and followup-routing coverage, then handle the session-resume prefix behavior separately through #72148 or its eventual replacement.
