## Summary

Keep this PR open. Current main still has the nested-parentheses exec approval parsing bug, the gateway can still emit exactly the nested denial metadata described, and the PR remains a plausible focused fix. It should not be closed as implemented, duplicate, or obsolete. The main maintainer question is now a merge-review question: preserve the parser fix and regression tests, while addressing the security review concern that approval-wrapper parsing is also applied to generic tool output.

## Best Possible Solution

Keep this PR open for normal maintainer review. ## Risks / Open Questions

- Security review needs maintainer attention before merge: because `parseExecApprovalResultText()` is applied to generic tool output in `src/agents/pi-embedded-subscribe.handlers.tools.ts:1057`, a command can already spoof flat `Exec denied (...)` output, and this PR would also accept nested-parentheses spoof strings. The best fix is to scope approval-event parsing to trusted exec approval result sources or add a harder-to-spoof structured envelope, while preserving the nested parser for legitimate gateway/node events.
- The provided pull-file context still lists `docs/.generated/plugin-sdk-api-baseline.sha256` as modified even though the last commit message says unrelated baseline churn was dropped. If that file is still in the live diff, it should be removed before merge because this parser fix should not refresh plugin SDK baselines.
