## Summary

Keep open. Current main does not implement this PR's requested behavior; instead it intentionally keeps mutating tool failures visible when `messages.suppressToolErrors` is set. The PR is a focused attempt to address the still-open #51065 surface, but it conflicts with current tests and shipped behavior, so it needs maintainer product judgment or revision rather than conservative auto-close.

## Best Possible Solution

Keep the PR open for maintainer judgment, but do not merge the current one-line precedence change as-is. The best path is to decide #51065's product contract explicitly: either add a separate opt-in setting that suppresses mutating warnings with clear safety docs and tests, or preserve `messages.suppressToolErrors` as non-mutating-only and clarify the help/docs. Any revised PR should update `payloads.errors.test.ts` to lock the chosen behavior.
