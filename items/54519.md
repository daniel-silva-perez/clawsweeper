## Summary

Current main still does not implement the requested per-account channel read-only mode or hard gateway/transport no-outbound guarantee. The existing `sendPolicy` is session-scoped and suppresses some auto-reply/chat paths, while direct gateway sends, message actions, polls, and WhatsApp transport side effects still have no account-level `readOnly` gate. The open WhatsApp PR #71382 is a relevant subset implementation candidate, but it is not merged and does not cover the broader generic channel/account policy requested here.

## Best Possible Solution

Keep this issue open. The best product path is a generic channel/account outbound delivery policy, exposed in shared channel/account config and schema, enforced before gateway sends, direct delivery, message actions, polls, typing/presence, reactions, read receipts, and plugin-owned transport side effects where possible. It should include docs/config metadata/UI hints and regression coverage for at least the shared session send path plus one bundled account-based channel such as WhatsApp, coordinated with #71382.
