## Summary

Keep open. Current `main` does not provide the requested generic per-account inbound-only / no-outbound enforcement. There are some channel-specific action gates, but they are partial and do not cover normal auto-reply delivery paths.

## Evidence

- **Auto-reply still routes to outbound delivery:** Normal reply routing calls `deliverOutboundPayloads` with the resolved `channel`, `to`, and `accountId`, so external delivery remains the default reply path.
  - file: [src/auto-reply/reply/route-reply.ts:221](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/auto-reply/reply/route-reply.ts#L221)
  - command: `nl -ba src/auto-reply/reply/route-reply.ts | sed -n '200,235p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Outbound delivery has no generic read-only account check:** `deliverOutboundPayloads` sends through `handler.sendText` / `handler.sendPayload` directly. This is the core delivery path, and the shown send branches do not enforce a generic receive-only account mode before sending.
  - file: [src/infra/outbound/deliver.ts:789](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/infra/outbound/deliver.ts#L789)
  - command: `nl -ba src/infra/outbound/deliver.ts | sed -n '760,900p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Telegram gating is tool-specific:** Telegram does have `channels.telegram.actions.sendMessage=false`, but the check is in the manual action runtime for `action === "sendMessage"`, not in the shared reply-delivery path.
  - file: [extensions/telegram/src/action-runtime.ts:308](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/telegram/src/action-runtime.ts#L308)
  - command: `nl -ba extensions/telegram/src/action-runtime.ts | sed -n '304,314p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Telegram outbound adapter bypasses that gate:** Telegram's outbound adapter resolves `sendMessageTelegram` and calls it directly for reply delivery; the adapter code shown does not consult `actions.sendMessage` or any read-only flag.
  - file: [extensions/telegram/src/outbound-adapter.ts:190](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/telegram/src/outbound-adapter.ts#L190)

  - file: [src/config/types.whatsapp.ts:129](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/config/types.whatsapp.ts#L129)

  - file: [src/config/types.signal.ts:44](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/config/types.signal.ts#L44)
