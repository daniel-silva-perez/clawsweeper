## Summary

Keep open. Current `main` has only a partial, opt-in mitigation: post-compaction transcript truncation for Pi sessions when `agents.defaults.compaction.truncateAfterCompaction` is enabled. It is disabled by default, applies only after compaction, and there is still no general rolling-window or max-size pruning for ordinary session transcript files. Heartbeat transcripts are explicitly append-only on current `main`, so the issue is not implemented well enough to close conservatively.
