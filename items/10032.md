## Close Comment

Closing as a Codex review because this request fits OpenClaw's plugin/ClawHub scope rather than core. `VISION.md:54` says optional capability should usually ship as plugins and treats memory as a plugin slot; `docs/concepts/memory.md:9` shows current core memory is still Markdown-file based; and `docs/concepts/memory-honcho.md:14` already documents a plugin path that provides cross-session memory plus automatic per-user profiles. If you still want this behavior, the right path is a plugin/ClawHub solution rather than a built-in core feature.
