## Summary

Keep #54864 open. Current `main` has fixed the related `--agent` + `--session-id` routing problem, but it still does not expose the requested `--new-session` flag, generate an internal UUID for the CLI, or document JSON output containing a generated session id. The linked PR #69108 is useful related work, but the provided context shows it is still open/draft, so the requested user-facing behavior is not merged or shipped.

## Best Possible Solution

Keep the issue open until maintainers merge a fix or explicitly move tracking to PR #69108. The best implementation path is to add a mutually exclusive `--new-session` flag to `openclaw agent`, generate a fresh session id once in the shared CLI path before gateway/local dispatch, route it through the existing `sessionId` machinery, and include the generated `sessionId` in `--json` output with regression tests for default reuse, explicit `--session-id`, and generated sessions.
