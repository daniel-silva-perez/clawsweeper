## Summary

Keep this issue open. Current main has the lower-level `--agent` plus `--session-id` isolation fix, but it still does not expose `openclaw agent --new-session`, generate a CLI session id, or return that generated id as a top-level JSON field. The related implementation PR #69108 is closed and unmerged, so it is useful prior art rather than a superseding fix.

## Best Possible Solution

Keep this issue open and implement a narrow core CLI feature in the existing `openclaw agent` path: add a `--new-session` boolean option, reject it together with `--session-id`, generate one `crypto.randomUUID()` in `agentCliCommand`, pass that id as the effective `sessionId` to both gateway and embedded paths, surface it as top-level `sessionId` in `--json` output, and update parser/session-resolution/JSON-output tests plus `docs/cli/agent.md` and `docs/tools/agent-send.md`.
