## Summary

Current `main` already rejects the reported `config.patch`/`config.apply` SecretRef failure mode before writing to disk, and regression tests verify the config remains unchanged on failure.

## Evidence

- **Preflight before write for config.patch:** `ensureResolvableSecretRefsOrRespond` calls `prepareSecretsRuntimeSnapshot(...)` and `config.patch` returns early on failure before `writeConfigFile(...)` is reached.
  - file: [src/gateway/server-methods/config.ts:300](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/gateway/server-methods/config.ts#L300)
- **Preflight before write for config.apply:** `config.apply` also runs `ensureResolvableSecretRefsOrRespond(...)` before the write path, covering the full-config replacement case mentioned in the issue.
  - file: [src/gateway/server-methods/config.ts:637](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/gateway/server-methods/config.ts#L637)
- **Regression tests keep config unchanged:** Tests for both `config.patch` and `config.apply` inject a missing env-backed SecretRef, assert the RPC fails with `active SecretRef resolution failed`, and verify the config hash/raw are unchanged afterward.
  - file: [src/gateway/server.config-patch.test.ts:358](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/gateway/server.config-patch.test.ts#L358)
- **Docs state write RPC preflight keeps disk/runtime unchanged:** Gateway docs explicitly say config write RPC preflight rejects unresolved submitted SecretRefs before persistence and keeps both disk config and runtime snapshot unchanged.
  - file: [docs/gateway/secrets.md:386](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/docs/gateway/secrets.md#L386)
- **Shipped release note:** The changelog for `2026.3.31` includes: `Config/SecretRef + Control UI ... preflight submitted-config SecretRefs before config write RPC persistence. (#58044)`.
  - file: [CHANGELOG.md:1669](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/CHANGELOG.md#L1669)

## Close Comment

Closing from a Codex review because current `main` already covers the reported failure mode. In `src/gateway/server-methods/config.ts:300`, `config.patch` and `config.apply` both preflight active `SecretRef` resolution before `writeConfigFile(...)`, so unresolved env refs are rejected instead of being persisted. The regression coverage in `src/gateway/server.config-patch.test.ts:358` and `src/gateway/server.config-patch.test.ts:390` verifies the RPC fails and the config hash/raw stay unchanged for missing env vars. The behavior is also documented in `docs/gateway/secrets.md:386`, and the changelog says it shipped in `2026.3.31` under `#58044`. If you can still reproduce a post-write runtime-refresh failure on current `main`, please open a new issue with the exact config surface and error path, since the original SecretRef-bricking case appears fixed.
