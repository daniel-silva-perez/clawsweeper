## Summary

Keep #58626 open. Current main has a new doctor repair for some stale bundled plugin load paths, but the implementation appears narrower than the reported pnpm/npm global upgrade failure: it rewrites source-style bundled paths to the current packaged dist path, while the issue reports an already-dist path under a removed versioned pnpm package root. Config validation still treats missing plugins.load.paths as fatal, and I did not find evidence that doctor repairs plugins.installs.sourcePath/installPath for this case.
