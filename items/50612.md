## Summary

Keep open. Current main still appears to mishandle macOS dashboard token resolution when `gateway.auth.token` is configured as an env-substituted string such as `${OPENCLAW_GATEWAY_TOKEN}`: the Swift dashboard path returns the raw config string before any launchd fallback, so the reported literal-token URL remains plausible on current main.

## Evidence

- **macOS dashboard still uses raw config token before launchd fallback:** `GatewayEndpointStore.resolveGatewayToken` checks `process.env.OPENCLAW_GATEWAY_TOKEN`, then returns `resolveConfigToken(...)` as-is, and only after that falls back to `launchdSnapshot?.token`. A config value like `${OPENCLAW_GATEWAY_TOKEN}` would therefore win over the resolved launchd token and be propagated into the dashboard URL.
  - file: [apps/macos/Sources/OpenClaw/GatewayEndpointStore.swift:149](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/apps/macos/Sources/OpenClaw/GatewayEndpointStore.swift#L149)
  - command: `nl -ba apps/macos/Sources/OpenClaw/GatewayEndpointStore.swift | sed -n '145,190p'`
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **macOS menu action opens dashboard from that unresolved token-bearing config:** The menu-bar dashboard action calls `GatewayEndpointStore.shared.requireConfig()` and then `GatewayEndpointStore.dashboardURL(...)`, so whatever token `resolveGatewayToken` returns is what gets embedded in the opened URL fragment.
  - file: [apps/macos/Sources/OpenClaw/MenuContentView.swift:341](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/apps/macos/Sources/OpenClaw/MenuContentView.swift#L341)

- **Docs still advertise env substitution for `gateway.auth.token`:** The docs explicitly show `gateway.auth.token: "${OPENCLAW_GATEWAY_TOKEN}"` as a supported config pattern, so returning that string literally in the macOS dashboard path would contradict documented config behavior rather than an unsupported setup.
  - file: [docs/gateway/configuration.md:622](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/docs/gateway/configuration.md#L622)

That contrast suggests the Swift/macOS dashboard path has not been brought up to the same behavior yet.
