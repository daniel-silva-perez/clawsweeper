## Close Comment

Closing as implemented on `main` after a Codex review. The current control UI already has the copy-button state CSS that prevents both labels from rendering together: `ui/src/styles/components.css:1634` hides `.code-block-copy__done` by default, `ui/src/styles/components.css:1638` hides the idle label in copied state, and `ui/src/styles/components.css:1642` shows `Copied!` only when the button has `.copied`. The click handler in `ui/src/ui/views/chat.ts:784` adds/removes that `.copied` class after a successful clipboard write, and the copy-button markup is wired to those classes in `ui/src/ui/markdown.ts:434`. I also confirmed the same selectors are present in the latest tagged release, `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
