## Summary

Current `main` has partial mitigations, but not strong enough evidence to close. The code now applies Telegram `agentFilter`/`sessionFilter`, preserves Telegram origin targets, and sends an origin-chat notice when `target: "dm"` routes the approval to approver DMs. But native Telegram DM delivery is still keyed only by approver chat IDs, and `/approve` authorization is still based on sender/account authorization rather than the originating Telegram session/chat. That leaves the reported multi-agent misrouting/confusion plausible on current `main`, so conservative cleanup should keep the issue open.

- **Docs say Telegram `target: "dm"` intentionally delivers to approver DMs, not the origin chat:** `docs/tools/exec-approvals-advanced.md:291-315` and `docs/channels/telegram.md:742-751` document that native Telegram approvals default to approver DMs and only appear in the originating chat when `target` is `channel` or `both`.
