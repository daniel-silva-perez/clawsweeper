## Summary

Keep #71112 open. Current main does not implement the proposed non-vector `ltm query` command for `memory-lancedb`; the bundled plugin only registers `ltm list`, `ltm search`, and `ltm stats`, and `MemoryDB` does not expose the table. The PR is recent, focused, and targets an existing bundled memory plugin surface rather than a duplicate or clearly external-only request.

## Best Possible Solution

Keep the PR open for maintainer review and iteration. If accepted, land it as an extension-owned `memory-lancedb` CLI feature with tests and any needed docs/help updates, while addressing the remaining full-scan behavior before merge.
