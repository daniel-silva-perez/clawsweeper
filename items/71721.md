## Summary

Keep open. Current main still calls `os.networkInterfaces()` through the unsafe helper in the gateway LAN and tailnet discovery paths, and tests still assert that the reported `uv_interface_addresses` failure is propagated. The linked PR #71950 is relevant and targets the right fix, but the provided context shows it open and unmerged, so the issue is not implemented on main.

## Best Possible Solution

Keep #71721 open and land PR #71950 or an equivalent maintainer-reviewed patch that changes `src/gateway/net.ts` and `src/infra/tailnet.ts` to use `safeNetworkInterfaces()`, then update the gateway and tailnet tests to assert graceful fallback instead of thrown errors.
