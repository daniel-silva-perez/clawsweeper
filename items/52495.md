## Summary

Keep open. Current main does not contain the PR's docs row, and the PR addresses a real consistency gap between the memory docs/tool groups and the built-in tools overview, but the existing review finding is materially valid: memory_search/memory_get are exposed through the selected memory plugin path, and alternate or disabled memory slots change the available tools. That leaves the PR useful but not safely auto-closeable under the allowed cleanup reasons.

## Best Possible Solution

Keep the PR open for maintainer judgment or contributor revision. The best docs direction is to add memory_search/memory_get in a way that preserves the active-memory-plugin caveat, such as a memory row with precise default-slot wording or a plugin-provided/core-plugin note, rather than presenting the tools as unconditional built-ins.
