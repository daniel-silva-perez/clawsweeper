## Summary

Keep PR #60155 open. Current main still uses the workspace root as the audit containment boundary, while the skill loader enforces containment against the configured skill root. The PR addresses a real alignment gap and adds the missing regression coverage; it is not already implemented on main.

## Evidence

- **Current audit boundary still uses workspace root:** Current main's collectWorkspaceSkillSymlinkEscapeFindings resolves workspacePath, computes workspaceRealPath, and passes workspaceRealPath into isPathInside before warning. The finding title/detail/remediation still say the skill escaped the workspace root, so the PR's requested skills-directory boundary is not present on main.

That supports the PR's claim that loader containment is stricter than the current audit check.
