## Summary

Keep this issue open. Current main still has the reported macOS launchd storm shape: LaunchAgent plists are generated with KeepAlive=true and ThrottleInterval=1, while missing gateway config still exits with EX_CONFIG before the gateway starts. Recent launchd work around installed-but-unloaded supervision references related #60885 but does not add missing-config backoff, max retries, a degraded wait mode, or a launchd equivalent of systemd's RestartPreventExitStatus.

## Best Possible Solution

Keep this issue open for a macOS launchd-specific storm guard. The likely fix belongs in the LaunchAgent/gateway startup boundary: preserve fast intentional restarts, but stop repeated EXIT_CONFIG_ERROR/missing-config respawns via a supervised wrapper/backoff, config-error suppression, max-retry sentinel, or degraded wait-for-config mode. Update LaunchAgent tests and gateway troubleshooting docs, and coordinate with related #60885 without closing this as a duplicate unless a canonical issue explicitly covers the missing-config failure mode.
