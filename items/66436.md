## Summary

Keep open. Current main still matches the reported launchd storm path: generated macOS LaunchAgents keep `KeepAlive=true` with `ThrottleInterval=1`, and the gateway still exits with `EXIT_CONFIG_ERROR` on missing config before starting. Existing invalid-config recovery and systemd restart prevention do not cover this missing-config launchd case.

## Best Possible Solution

Keep this issue open for a macOS launchd-specific storm guard. The best implementation is to preserve fast intentional restarts while stopping repeated config-error respawns, either with a supervised wrapper/backoff, conditional suppression for `EXIT_CONFIG_ERROR` and missing config, or a degraded wait-for-config mode. Update LaunchAgent tests, daemon/service audit if the plist contract changes, and Gateway troubleshooting docs. Coordinate with adjacent launchd throttle work such as #60885, but do not close this as a duplicate because the missing-config failure mode remains distinct.
