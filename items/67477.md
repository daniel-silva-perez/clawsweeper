## Summary

Keep this PR open. Current main already supersedes the `message_sending` cancellation/transform part through the shared inbound reply dispatcher, and the changelog records that as shipped in 2026.4.24 via #70118. But this PR also explicitly adds plugin `message_sent` and internal `message:sent` emission for WhatsApp auto-replies, and current main still does not appear to emit those hooks on the WhatsApp auto-reply path. With unique remaining behavior and unresolved review concerns, this is not a conservative auto-close.

## Best Possible Solution

Keep this PR open, but ask the author or maintainer to rebase it onto current main and narrow it around the still-missing sent-hook parity. The `message_sending` part should reuse the shipped shared dispatcher behavior; any remaining implementation should emit plugin `message_sent` and internal `message:sent` with accurate delivered content, success/failure state, session/account/conversation/group metadata, and cancellation accounting.
