## Summary

Keep this issue open. Current main still runs direct QMD searches once per managed collection when more than one collection is selected, and the provided GitHub context shows an open, unmerged contributor PR with closing syntax for this issue. That PR is the implementation candidate, not a reason to close the issue before merge.

## Best Possible Solution

Keep this issue open until maintainers review PR #72485. If the PR is correct, land it or an equivalent patch that groups same-source direct QMD collection filters into fewer subprocesses while preserving separate memory/session inputs for source diversification; if it is not correct, close or revise that PR and keep this issue as the tracker for the remaining performance bug.
