## Summary

Keep open. Current main still implements and documents bound ACP follow-ups as direct prompt-text dispatch to the ACP session, which is the behavior this issue asks to change. The provided related-item context does not show a canonical duplicate for parent-orchestrated rewrite semantics.

## Best Possible Solution

Keep the issue open for maintainer product/design work. A good fix would narrow the intended surface, then update the ACP spawn accepted note/tool guidance and add regression coverage so `sessions_spawn({ runtime: "acp", thread: true, mode: "session" })` follow-ups from a parent orchestrator are rewritten into explicit child tasks by default, while explicit `/acp` chat bindings either remain documented as direct prompt-text routing or get a separately approved behavior change.
