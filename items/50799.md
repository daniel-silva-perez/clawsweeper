## Summary

Keep open. Current `main` still hard-restricts the macOS chat attachment flow to images, so the reported behavior is still implemented rather than fixed.

## Evidence

- **macOS paperclip picker is image-only:** The shared macOS chat composer labels the button `Add Image`, opens an `NSOpenPanel` titled `Select image attachments`, and sets `panel.allowedContentTypes = [.image]`, which would show non-image files but disable them in the picker.

  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
