## Summary

Keep open. Current `main` does not contain the PR's Control UI reload behavior: `runUpdate()` still only sends `update.run` and clears `updateRunning`, and the reconnect path still only re-establishes the WebSocket. I found no current-main helper/tests matching the PR's added reload flow, so there is not strong evidence to close this as already implemented or obsolete.
