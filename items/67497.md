## Summary

Keep this issue open. The GitHub context shows an open contributor PR, #67734, explicitly linked as the implementation candidate for this issue, and the workflow says not to close an issue while such a PR is still open. Current main also does not show the proposed logger-suppression path in the Feishu setup probe: setup status calls `probeFeishu`, `probeFeishu` constructs a normal Feishu client, and `createFeishuClient` passes no probe-specific silent/custom logger to the SDK client constructor.

## Best Possible Solution

Review #67734 as the implementation candidate. If it is correct, land a Feishu-owned fix that lets setup/status probes construct or use a client with a silent/custom SDK logger while preserving the existing `Feishu: connected as ...` status line, and keep or add focused regression coverage in `extensions/feishu/src/client.test.ts`, `extensions/feishu/src/probe.test.ts`, and/or `extensions/feishu/src/setup-surface.test.ts`. After that PR or an equivalent fix merges, this issue can close as implemented.
