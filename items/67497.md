## Summary

Keep this open. Current main still lacks a Feishu setup/status probe path that suppresses the SDK logger, and the provided GitHub context shows same-author PR #67734 is still open and unmerged as the implementation candidate for this issue.

## Best Possible Solution

Review #67734 as the implementation candidate. If it is correct, land a Feishu-owned change that lets setup/status probes use a quiet or custom SDK logger while preserving the existing `Feishu: connected as ...` status line, with focused regression coverage in the Feishu client/probe/setup-surface tests. After that merges, close this issue as implemented.
