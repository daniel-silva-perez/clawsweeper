## Summary

Keep open. Current `main` still routes normal Feishu agent replies through a Feishu-specific dispatcher that sends directly via `sendMessageFeishu`/`sendStructuredCardFeishu`/`sendMediaFeishu` and does not call the generic outbound delivery path where `message:sent` hooks are emitted. The issue’s reported behavior remains plausible on current `main`.

## Evidence

- **Feishu live reply path still uses custom dispatcher:** The main Feishu bot flow constructs `createFeishuReplyDispatcher(...)` for normal single-agent replies and passes it into `dispatchReplyFromConfig`, so outbound Feishu replies still go through this custom dispatcher.
  - file: [extensions/feishu/src/bot.ts:1197](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/extensions/feishu/src/bot.ts#L1197)
