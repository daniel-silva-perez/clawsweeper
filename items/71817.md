## Summary

Keep open. Current main still hardcodes Telegram and Feishu reasoning preview fallbacks to off/session-only behavior, while the config schema and docs already expose `agents.list[].reasoningDefault`. The PR is a focused, recent bug fix with tests and active review discussion, not obsolete cleanup.

## Best Possible Solution

Keep PR #71817 open for channel-owner maintainer review. The best path is a focused merge or update that makes Telegram and Feishu reasoning preview resolution fall back to `agents.list[].reasoningDefault`, preserves explicit session overrides including `off`, and keeps Feishu's stream-only preview semantics documented. Broader `agents.defaults` or per-model default behavior should stay with the related issues.
