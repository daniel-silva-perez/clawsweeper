## Summary

Keep open. Current main still has the unguarded `WebSocketTaskBox.sendPing()` checked continuation path that this PR targets, and the latest release contains the same behavior. The PR is a narrow, plausible crash fix and the provided review discussion supports it; the same-scope PR #51311 is an overlap to resolve, but not enough reason to discard this useful patch without maintainer selection of a canonical PR.

## Best Possible Solution

Keep this open for maintainer review. The best product/code path is to land a one-shot guard around `WebSocketTaskBox.sendPing()` in `apps/shared/OpenClawKit/Sources/OpenClawKit/GatewayChannel.swift` and add a focused regression test using a `WebSocketTasking` fake that invokes the ping callback twice; then close the overlapping PR #51311 or #51371 depending on which patch is chosen.
