## Summary

Keep open. Current main fixed/handles media understanding in the normal `getReplyFromConfig` path, but the ACP dispatch path named by the reporter still invokes media understanding without a routed `agentDir` or `activeModel`. That means provider/auth discovery can still fall back to ambient `OPENCLAW_AGENT_DIR` or the default main-agent directory for ACP/channel dispatch instead of the target session agent. The related ClawSweeper items cover WebChat image preservation, audio media, and followup queue behavior, not this remaining ACP agentDir gap.

## Best Possible Solution

Keep the issue open and fix the ACP dispatch path. The likely product/code direction is to resolve the `resolvedAcpAgent` directory with the same agent-scope helper used by normal replies, pass that `agentDir` into `applyMediaUnderstanding`, and add a focused `dispatch-acp.test.ts` case proving inbound image media forwards the routed agentDir and does not depend on `OPENCLAW_AGENT_DIR`. Consider also passing the active ACP model when available so native-vision skip and non-visual fallback behavior match the normal reply path.
