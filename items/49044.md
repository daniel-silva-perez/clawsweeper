## Summary

Keep #49044 open. Current main still does not implement per-account SOUL files: there is no `--soul` CLI surface, no `soulFile` config key, onboarding still returns after DM policy setup, and bootstrap injection still loads the fixed workspace `SOUL.md`. The related #42861 was the older closed-unmerged branch that #49044 superseded, not a canonical follow-up that tracks the remaining work. The PR also has unresolved review concerns in the provided discussion, especially schema validation and runtime bootstrap resolution, so it is not merge-ready but remains a plausible implementation path.

## Best Possible Solution

Keep the PR open or ask the author to refresh/supersede it with a complete core implementation: add a validated per-account `soulFile` contract for supported channel account schemas and metadata, wire `channels add --soul` and onboarding to that contract, pass channel/account context into bootstrap resolution and cache keys, preserve default `SOUL.md` behavior, document the config, and add regression tests for default and account-scoped SOUL prompt injection.
