## Summary

Keep this PR open. Current `main` still has the WhatsApp hardening gap in the model-facing path: WhatsApp builds a legacy `Body` envelope separately, but the reply pipeline prefers `BodyForAgent`, and WhatsApp currently fills `BodyForAgent` from the raw or transcribed inbound message body. The submitted patch is also not safe to merge as-is because it wraps only envelope/group-history strings and would put randomized wrapper markers into `combinedBody`, which is used for echo-dedup keys.

## Best Possible Solution

Keep this PR open as an incomplete WhatsApp security hardening candidate. The maintainer path is to rework the fix so the actual model-facing WhatsApp `BodyForAgent` text, reply-context body, and any model-visible group-history text are wrapped or represented as untrusted structured context, while keeping `combinedBody`/echo keys deterministic and keeping trusted operator `messagePrefix` outside the untrusted block. Review or replace this PR rather than closing it as implemented or obsolete.
