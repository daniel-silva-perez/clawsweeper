## Summary

Keep this PR open. Current main still leaves the WhatsApp model-facing body on the raw `BodyForAgent` path, so the requested hardening is not implemented. The submitted patch is also incomplete and risky as-is because it wraps the legacy envelope string used for combined echo keys, while the active prompt path prefers `BodyForAgent`.

## Best Possible Solution

Keep this PR open as an incomplete WhatsApp hardening candidate. The maintainer path is to revise or replace it so WhatsApp wraps or structurally isolates the actual model-facing `BodyForAgent`, reply-context body, and any model-visible group-history text, while keeping echo keys deterministic and leaving trusted operator `messagePrefix` outside the untrusted block.

## Risks / Open Questions

- Current main still sends WhatsApp inbound text or transcribed audio text to the model through raw `BodyForAgent`, so the security hardening requested by the PR remains unresolved.
- Merging the submitted patch unchanged would likely break combined-message duplicate suppression because `wrapExternalContent` adds random marker ids and the wrapped legacy body feeds echo keys.
- Merging the submitted patch unchanged would also mark trusted operator `messagePrefix` text as untrusted because the patch wraps the entire `baseLine` string.
