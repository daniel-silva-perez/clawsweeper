## Summary

Keep open. Current `main` still omits session context from the macOS exec approval popup, and the macOS popup path still drops richer requester metadata before presentation. The feature request is not implemented on `main` or in the latest release snapshot checked.

## Evidence

- **macOS popup request model is still narrow:** `ExecApprovalPromptRequest` only includes `command`, `cwd`, `host`, `security`, `ask`, `agentId`, `resolvedPath`, and `sessionKey`. It does not carry `turnSourceChannel`, `turnSourceTo`, `turnSourceAccountId`, `turnSourceThreadId`, or any sender/tool-context fields.
  - file: [apps/macos/Sources/OpenClaw/ExecApprovalsSocket.swift:8](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/apps/macos/Sources/OpenClaw/ExecApprovalsSocket.swift#L8)
  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
- **macOS popup still does not render Session:** `buildAccessoryView` adds rows for working directory, agent, executable, host, security, and ask mode, but never adds a row for `request.sessionKey`. This is the exact parity gap described in the issue.
  - file: [apps/macos/Sources/OpenClaw/ExecApprovalsSocket.swift:303](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/apps/macos/Sources/OpenClaw/ExecApprovalsSocket.swift#L303)
  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
- **Web UI still renders Session:** The Control UI exec approval view still renders `Session` from `request.sessionKey`, so the macOS/UI mismatch remains on current `main`.
  - file: [ui/src/ui/views/exec-approval.ts:33](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/ui/src/ui/views/exec-approval.ts#L33)
  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
- **Upstream approval payload already has requester-route metadata:** `ExecApprovalRequestPayload` still includes `sessionKey` plus `turnSourceChannel`, `turnSourceTo`, `turnSourceAccountId`, and `turnSourceThreadId`, so the requester/origin data exists upstream but is not surfaced by the macOS popup path.
  - file: [src/infra/exec-approvals.ts:97](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/src/infra/exec-approvals.ts#L97)
  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
- **macOS gateway prompt path still decodes into the narrow local request type:** `ExecApprovalsGatewayPrompter.GatewayApprovalRequest` uses `ExecApprovalPromptRequest` for `request`, so extra origin fields from gateway payloads are ignored before the popup is built.
  - file: [apps/macos/Sources/OpenClaw/ExecApprovalsGatewayPrompter.swift:14](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/apps/macos/Sources/OpenClaw/ExecApprovalsGatewayPrompter.swift#L14)
  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
- **Latest release still has the same omission:** The same `buildAccessoryView` block at release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`) also lacks any `sessionKey` row, so this was not already shipped there either.
  - command: `git show 00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5:apps/macos/Sources/OpenClaw/ExecApprovalsSocket.swift | sed -n '260,340p'`
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
