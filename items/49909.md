## Summary

Current `main` still appears to ignore `docker-compose.override.yml` in ClawDock helper commands. `_clawdock_compose()` only passes `docker-compose.yml` plus optional `docker-compose.extra.yml`, and `clawdock-start`/`clawdock-stop` both route through that helper, so the reported config drift remains plausible on current `main` and in the latest release snapshot checked.

## Evidence

- **Current helper omits override file:** `_clawdock_compose()` builds compose args from `docker-compose.yml` and optional `docker-compose.extra.yml` only; there is no conditional for `docker-compose.override.yml`. `clawdock-start` and `clawdock-stop` both call this helper.
  - file: [scripts/clawdock/clawdock-helpers.sh:151](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/scripts/clawdock/clawdock-helpers.sh#L151)
