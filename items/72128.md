## Summary

Keep PR #72128 open. Current main has no configurable `labelTemplate` support, and the linked request (#72074) targets built-in `sessions_spawn` behavior rather than a standalone ClawHub/plugin surface. The PR is useful but needs revision before merge: the existing review comments identify real control-flow issues because requester and resolved target agent IDs are determined after `sessions-spawn-tool.ts` forwards the call.

## Best Possible Solution

Keep the PR open and ask for a focused revision. The best implementation is likely to resolve the fallback label after requester identity and effective target agent are known, using requester-scoped `agents.list[].subagents.labelTemplate` over `agents.defaults.subagents.labelTemplate`, while preserving explicit `label` precedence. Add regression tests for explicit label, default template, requester override, omitted `agentId` native spawns, ACP default-agent spawns, repeated tokens, newline stripping, and task truncation; update user-facing subagents/config docs and generated schema artifacts if this config field is accepted.
