## Summary

Keep PR #51563 open. Current main still makes `openclaw update status` report the package manager from the installed package's `package.json.packageManager` for non-git installs, so the npm-global misdetection described in #51401 is not implemented on main. The PR is still a focused fix direction, but it needs maintainer/author follow-up because the existing review comment identifies a real Bun custom `BUN_INSTALL` gap in the proposed path-only heuristic.

## Best Possible Solution

Keep the PR open and refresh it against current main by making `checkUpdateStatus` use the existing global-install manager detection path for `installKind === "package"` or a small shared wrapper around it. Add regression coverage for npm global/Homebrew, pnpm global, default Bun, and custom `BUN_INSTALL`, then ensure `openclaw update status` renders the detected install manager while preserving git checkout package-manager detection.
