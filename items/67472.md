## Summary

Keep this PR open. Current main still uses the existing `tar.c` archive path, so the linked backup ENOENT race is not implemented on main. The PR is a focused implementation candidate for #67417, but it should be revised before landing because the review discussion raises real test and manifest/verification concerns.

## Best Possible Solution

Keep this PR open and ask for a revision or land an equivalent fix. The best path is to implement archive-time ENOENT/ENOTDIR skipping in `src/infra/backup-create.ts`, update the manifest/result semantics so skipped payloads do not break `backup verify`, and add a regression test that simulates a session transcript disappearing during packing. After that lands, #67417 can close as implemented.

## Risks / Open Questions

- Merging the submitted patch unchanged could leave `--verify` or later restore tooling with an archive whose manifest still declares payloads that were skipped during packing.
- The proposed mock change needs to distinguish Pack-level file errors from write-stream errors, otherwise the primary ENOENT behavior remains hard to test.
- A lower-level `tar.Pack` replacement must preserve the existing gzip, portable, preservePaths, path remapping, temporary archive cleanup, and atomic publish behavior.
