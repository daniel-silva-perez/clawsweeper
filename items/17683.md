## Summary

Keep open. Current `main` already has several building blocks the request asks for: isolated cron jobs with fresh sessions, optional light-context runs, per-job tool allowlists, explicit exec allowlists, delivery previews, run-history logs, and usage/cost visibility. But it does not expose a single scoped/script-limited mode, and I did not find core support for daily spend ceilings, graceful budget-abort behavior, or cron/agent cost-preview or plan-preview flows.

## Evidence

- **Isolated cron runs already support fresh-session, trigger-based execution with scoped tools:** Docs describe `isolated` jobs as dedicated `cron:<jobId>` sessions with fresh context, plus `--light-context` and `--tools exec,read` payload options.
  - file: [docs/automation/cron-jobs.md:85](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/docs/automation/cron-jobs.md#L85)
- **CLI and protocol carry the scoped cron payload fields:** `cron add` writes `lightContext` and `toolsAllow` into the agent-turn payload, and the gateway protocol schema accepts both fields for cron agent turns.
  - file: [src/cli/cron-cli/register.cron-add.ts:148](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/src/cli/cron-cli/register.cron-add.ts#L148)
- **Headless automation already supports explicit command allowlists:** Docs expose `openclaw approvals allowlist add ...`, and the exec host path explicitly tells cron/headless users to keep allowlist mode and add an explicit allowlist entry because cron runs cannot wait for interactive approval.
  - file: [docs/cli/approvals.md:155](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/docs/cli/approvals.md#L155)
- **Execution transparency exists, but as route preview and run-history diagnostics rather than plan/cost preview:** `openclaw cron list/show` preview the resolved delivery route, and `cron runs` records delivery diagnostics including resolved target, message-tool sends, fallback use, and delivered state.
  - file: [docs/cli/cron.md:19](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/docs/cli/cron.md#L19)
- **Cost controls are partial: visibility and token caps exist, but hard budget ceilings do not:** OpenClaw documents usage/cost summaries from session logs and supports `maxTokens` / `contextTokens` caps, but a repo search for `daily spend`, `spend ceiling`, `daily budget`, `graceful abort`, `estimated token`, `cost preview`, and `plan summary` under cron/agent/core surfaces returned no hits. The only `max_budget` example I found was LiteLLM proxy configuration, not OpenClaw core.
  - file: [docs/concepts/usage-tracking.md:23](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/docs/concepts/usage-tracking.md#L23)
  - command: `rg -n "daily spend|spend ceiling|daily budget|graceful abort|estimated token|cost preview|plan summary" docs/cli/cron.md docs/automation/cron-jobs.md src/cli/cron-cli src/gateway src/config`

## Risks / Open Questions

- This issue may be partially satisfied by composing existing cron, tool-policy, and exec-allowlist features, but the requested hard budget and preview behavior is still missing from core surfaces.
