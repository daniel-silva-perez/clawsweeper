## Summary

Keep #70630 open. Current main still has the Telegram no-visible-response path synthesizing a NO_REPLY fallback and delivering it for direct sessions, and the current regression test still asserts that behavior. The PR is a focused fix for the linked open bug #70628 and its review-thread nit was addressed in the second PR commit, so there is no strong close reason.

## Best Possible Solution

Keep the PR open for maintainer review and, if the policy choice is accepted, land the narrow Telegram extension fix: suppress only the synthetic NO_REPLY fallback for no-visible direct turns without real dispatch/delivery errors, mark that silent path as handled for status reactions, preserve fallback delivery for actual errors/failures, and keep the regression test update. If maintainers decide direct empty turns must remain visible/failing by contract, revise the PR and docs/tests explicitly rather than closing it as obsolete.
