## Summary

Keep open. Current main does not implement the PR’s central macOS Talk Mode ExecuTorch/Parakeet STT backend, does not include an ExecuTorch plugin/runtime, and still renders/sends Talk Mode prompts through the existing Apple Speech path. The PR discussion also contains substantive review blockers that the author claims to have addressed on the PR branch, so this is not a low-signal stale PR that can be conservatively closed as obsolete.

## Best Possible Solution

Keep the item open for maintainer judgment, but ask for a narrower path: split the generic Talk Mode STT backend/API and transcript-display/PermissionManager fixes from the ExecuTorch implementation, then publish or maintain the ExecuTorch Parakeet runtime as a plugin unless maintainers explicitly want it bundled. If Talk Mode should consume registered realtime transcription providers, define that core/native bridge first and test it independently.
