## Summary

Keep this PR open. Current main still routes gateway/TUI live assistant text through directive-tag stripping and the prefix/delta merge helper without the PR's leading-directive whitespace fix, and the linked bug report was closed as superseded by this PR rather than fixed on main.

## Best Possible Solution

Keep this PR open as the canonical gateway bug-fix thread. The best path is to revise or land a narrow shared `live-chat-projector` fix that normalizes only directive-introduced leading whitespace before merge, preserves intentional user-visible whitespace and delta separators, and adds a regression test through the real gateway event path or a well-scoped exported projector helper.
