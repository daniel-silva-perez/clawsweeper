## Summary

Keep this PR open. Current main still has the directive-tag leading-whitespace mismatch in the shared live chat projection path, and the linked bug report #66497 was closed as superseded by this PR rather than fixed on main. The PR is useful but needs to be rebased/adapted to the current `live-chat-projector` helper and narrowed so it preserves intentional leading whitespace.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate. The best path is to rebase it onto current main, move the fix into `src/gateway/live-chat-projector.ts`, normalize only whitespace introduced by stripped leading directive tags before merging, preserve intentional delta separators and leading user-visible formatting, and add a regression test through the real Gateway event path or the shared projector helper.
