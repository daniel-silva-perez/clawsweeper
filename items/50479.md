## Summary

Keep open. Current main still has the generic `fix_npm_permissions()` messages while silently writing `prefix=$HOME/.npm-global`, so PR #50479 is not implemented on main. Main does have adjacent duplicate-global-install detection, but that warns after multiple OpenClaw installs already exist; it does not replace this PR's preventive install-time warning about the `.npmrc` prefix and future sudo npm mismatch.

## Best Possible Solution

Keep #50479 open as the focused remaining tracker. The best path is to either merge an adapted install-time warning in `scripts/install.sh` after maintainer review, ideally with a small install-sh test, or implement an equivalent warning/detection in main before closing the PR as superseded.
