## Summary

PR #52200 should stay open. Current main still compacts home-relative skill paths by returning `~/` plus the original sliced remainder, so Windows paths can still become mixed-separator prompt locations. The available-skills prompt explicitly tells the model to read the `<location>` path, and current tests do not cover Windows backslash normalization. The PR’s proposed source change and follow-up test assertion directly address the remaining bug and review feedback.

## Best Possible Solution

Keep PR #52200 open and either land its focused normalization plus regression test, or apply an equivalent fix in `src/agents/skills/workspace.ts` that normalizes prompt-facing compacted Windows skill paths without changing canonical runtime paths.
