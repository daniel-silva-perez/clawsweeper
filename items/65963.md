## Summary

Keep this issue open. Current main partially addresses session sprawl by deleting dreaming narrative subagent sessions and scrubbing stale/orphaned dreaming artifacts, but the highest-leverage requests remain missing: no public `dreaming.model` override, no `retainDays`/batch/maxSessions control, no `sessions purge --kind dreaming`, and the session picker still has only a cron-session hide path rather than a dreaming-specific filter.

## Best Possible Solution

Keep this issue open and implement a memory-core-owned fix: expose and document a supported dreaming execution/model override, wire it into the subagent runtime call, add a bounded retention policy for generated narrative sessions/transcripts, and add a targeted UI/CLI way to hide or purge dreaming artifacts without changing unrelated session cleanup semantics.

## Risks / Open Questions

- A cleanup-only implementation could mask the original cost problem while still routing background dreaming through an expensive primary model.
- There is already an internal execution config type with a `model` field, but exposing it needs schema/docs/runtime wiring and tests so operators get a real supported contract rather than an ignored config key.
- A purge/filter design should avoid treating all non-human sessions the same; cron, heartbeat, subagent, and dreaming sessions have different retention expectations.
