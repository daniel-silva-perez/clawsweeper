## Summary

Keep this issue open. Current main partially mitigates the session-sprawl symptom by deleting dreaming narrative subagent sessions, scrubbing stale/orphaned dreaming artifacts, and excluding generated dreaming transcripts from memory ingestion, but the central requested product controls remain missing: no documented/public dreaming model override, no retain-days or burst/concurrency control, no `sessions purge --kind dreaming`, and no dreaming-specific session picker filter/collapse.

## Best Possible Solution

Keep this issue open for a memory-core-owned product fix: expose and document a supported dreaming execution/model override, pass it into the actual subagent/runtime path, add bounded retention or purge semantics for generated dreaming sessions/transcripts, and add a targeted UI/CLI way to hide or purge dreaming artifacts without broadening unrelated session cleanup behavior.
