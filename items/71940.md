## Summary

Keep #71940 open. Current main still routes normal viewport screenshots through CDP `Page.captureScreenshot` with no Hyprland/grim fallback, no `browser.hyprlandCapture` config/schema support, and no Hyprland troubleshooting docs. The linked bug #64317 is still open with concrete Omarchy/Hyprland reproduction and workaround evidence; the earlier same-title PR #67149 is closed unmerged, so it is not an active canonical replacement. This PR remains actionable but needs reviewer feedback addressed before merge.

## Best Possible Solution

Keep the PR open as the active implementation path for #64317. The best maintainer path is to rebase it onto current main, add `browser.hyprlandCapture` to the strict runtime schema and docs, address the signal/teardown/per-profile-headless review comments, keep the fallback opt-in and viewport-only, then validate with targeted browser route/config tests plus a real Hyprland grim capture smoke.
