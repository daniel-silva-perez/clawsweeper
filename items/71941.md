## Summary

Keep PR #71941 open. Current main still pre-fills `agents add` from `resolveAgentWorkspaceDir`, and that resolver still nests non-default agents under `agents.defaults.workspace` when the default workspace is configured. The linked bug #71889 is real against current main, and this PR is a focused implementation path, though its existing review comments should be addressed before merge.

## Best Possible Solution

Keep the PR open and have the contributor address the review comments: stub `OPENCLAW_HOME`/`HOME` in the wizard test, fix or clarify the changelog attribution, then run the normal changed gate before maintainer merge. The product/code direction should remain narrow: change only the `agents add` wizard default for brand-new agents in the documented post-setup default case, while preserving runtime `resolveAgentWorkspaceDir` and #59789 shared-base behavior.
