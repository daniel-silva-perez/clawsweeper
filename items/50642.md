## Summary

Keep open. Current `main` still has the reported macOS direct-`wss://` TOFU behavior: the first certificate fingerprint is saved and trusted before `SecTrustEvaluateWithError`, `MacNodeModeCoordinator` still enables `allowTOFU` whenever no stored fingerprint exists, and the macOS config path still does not read `gateway.remote.tlsFingerprint`.

## Evidence

- **TOFU acceptance still precedes trust evaluation:** `GatewayTLSPinningSession.urlSession(_:didReceive:completionHandler:)` still stores the presented fingerprint and returns `.useCredential` when `params.allowTOFU` is true; `SecTrustEvaluateWithError` is only reached afterward.
