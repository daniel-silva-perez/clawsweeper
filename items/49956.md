## Close Comment

Thanks for the detailed write-up. I reviewed this on current `main` as a Codex maintainer pass, and I’m closing it as `clawhub` rather than a core bug/feature for now.

The current architecture already provides supported extension points for this exact class of behavior: `agent:bootstrap` runs before bootstrap files are finalized (`docs/concepts/agent-loop.md:83`), `before_prompt_build` can inject prompt/context before model submission (`docs/concepts/agent-loop.md:94`), and `session_start` is part of the plugin hook contract (`src/plugins/hook-types.ts:396`). The hooks system is explicitly designed for custom automation and reusable hook packs/plugins (`docs/automation/hooks.md:56`, `docs/automation/hooks.md:144`).

More importantly, the project scope document says OpenClaw core should stay lean and that optional capability should usually ship as plugins instead of new core config knobs (`VISION.md:54`). Given that this is already achievable via a hook/plugin, and your report even notes a working `session_start` plugin path, this fits the plugin/ClawHub lane better than adding a new built-in `preSessionScripts` config surface to core.

If you want to take this further, the best next step would be packaging the behavior as a reusable hook pack or plugin rather than extending core config.
