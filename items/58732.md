## Summary

Keep this PR open. Current main has not implemented AbortSignal support for runTasksWithConcurrency, and the PR is a plausible additive API improvement. It should not be auto-closed, but it also should not be treated as ready as-is because the review discussion identifies a real contract risk: aborted runs would return partial results while reporting hasError: false.
