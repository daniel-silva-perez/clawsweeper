## Summary

Keep this issue open. Current main at e962381dbf1ecdb1cd4784fdeb6c2e07ddd3435b still has no `openclaw tree`, `openclaw sessions tree`, or `openclaw agents tree` command; the existing CLI, slash-command, gateway, and tool surfaces expose flat session/task/subagent listings with relationship metadata, not a hierarchical active-agent tree.

## Best Possible Solution

Keep this open for maintainer product judgment. A conservative first slice should add a core inspection command such as `openclaw sessions tree` or `openclaw agents tree`, build hierarchy from existing session/task/subagent relationship fields, support text and JSON output, and defer `--watch` plus per-process resource accounting unless maintainers choose a broader design.
