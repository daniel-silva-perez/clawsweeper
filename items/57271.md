## Summary

Keep open. Current `main` still rejects bind sources outside the sandbox workspace roots by default, tests lock that behavior in, and the main sandboxing docs still show arbitrary host-path examples that do not match the runtime default.

## Evidence

- **Runtime hard-codes the allowlist to workspace roots:** Sandbox container creation passes only `[workspaceDir, agentWorkspaceDir]` as `bindSourceRoots`, so custom bind sources outside those roots are rejected unless an override is enabled.
  - file: [src/agents/sandbox/docker.ts:488](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/agents/sandbox/docker.ts#L488)
  - command: `nl -ba src/agents/sandbox/docker.ts | sed -n '482,490p'`
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **Validator throws the same outside-allowed-roots error described in the issue:** `validateBindMounts()` checks allowed roots on both normalized and canonicalized source paths and throws `source ... is outside allowed roots ... Use a dangerous override...` when the source is outside the allowlist.
  - file: [src/agents/sandbox/validate-sandbox-security.ts:281](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/agents/sandbox/validate-sandbox-security.ts#L281)
  - command: `nl -ba src/agents/sandbox/validate-sandbox-security.ts | sed -n '254,356p'`
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **Tests explicitly preserve current rejection behavior:** Tests assert that bind sources outside configured allowed roots throw `outside allowed roots`, and only pass when `allowSourcesOutsideAllowedRoots: true` is set.
  - file: [src/agents/sandbox/validate-sandbox-security.test.ts:259](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/agents/sandbox/validate-sandbox-security.test.ts#L259)
  - command: `nl -ba src/agents/sandbox/validate-sandbox-security.test.ts | sed -n '259,287p'`
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **Dangerous override exists in config, but it is an explicit opt-out:** The Docker sandbox config includes `dangerouslyAllowExternalBindSources?: boolean`, confirming that external bind sources are not accepted by default.
  - file: [src/config/types.sandbox.ts:53](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/config/types.sandbox.ts#L53)

  - file: [docs/gateway/sandboxing.md:304](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/docs/gateway/sandboxing.md#L304)
