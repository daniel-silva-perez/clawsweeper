## Summary

Current main still does not implement the requested configurable memory-search exclusion. The issue remains valid: `memorySearch` has no `ignorePaths` or equivalent exclude contract, builtin indexing still recursively includes `memory/dreaming/**`, and builtin/QMD watchers only use hardcoded dependency/cache ignores. Related dreaming fixes address promotion and self-ingestion, not `memory_search` index or watcher scope.

## Best Possible Solution

Keep this issue open and implement a unified memory-search exclusion contract, likely `memorySearch.ignorePaths` or a clearly named equivalent. It should be added to schema, types, generated config help/docs, config resolution, builtin `listMemoryFiles`, builtin chokidar watching, QMD collection/update/watch scope, and scope hashing, with focused tests for `memory/dreaming/**`, dependency/cache defaults, `extraPaths`, and builtin/QMD parity.
