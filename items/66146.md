## Summary

Keep this PR open. Current main still has the OpenAI-compatible transport gap the PR is trying to fix: Responses and Completions only send max-token fields when `options.maxTokens` is explicitly present, even though model configs expose `model.maxTokens` and OpenRouter capability parsing populates it. The PR is useful, but it should not be landed as-is because it bundles an unrelated TUI refactor with review-reported regressions and lacks focused regression coverage for the model fallback.

## Best Possible Solution

Keep the PR open, but ask the contributor to rebase and narrow it to a focused transport fix. The best patch should resolve an effective positive max-token value from explicit options first and `model.maxTokens` second, send it as `max_output_tokens`, `max_tokens`, or `max_completion_tokens` according to the existing compatibility logic, and add regression tests for both Responses and Completions when `options.maxTokens` is omitted. The TUI refactor should be dropped from this PR or handled separately after its review comments are fixed.
