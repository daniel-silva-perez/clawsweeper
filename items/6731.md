## Summary

Keep #6731 open. Current main still has configurable hard controls, sandboxing, audit warnings, and advisory prompt safety text, but it also intentionally defaults host exec on gateway/node to full/off in trusted operator mode and documents that there is no separate dangerous-command heuristic gate in YOLO mode. The requested native hard/default dangerous-operation gate remains a core safety/product decision, not a cleanup-closeable issue.

## Best Possible Solution

Keep #6731 open until maintainers either implement a native hard/default dangerous-operation gate in core exec/tool policy, or explicitly decide that the current opt-in model of exec approvals, sandboxing, tool policy, audit warnings, and docs is the final intended behavior. A concrete implementation path should cover default safe mode, dangerous-command classification, approval UX, sandbox interaction, Windows PowerShell cases such as Remove-Item, and regression tests/docs.
