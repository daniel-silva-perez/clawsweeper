## Summary

Keep #72273 open. Current main still hardcodes Discord gateway startup READY timeout to 15s and runtime READY timeout to 30s inside the bundled Discord plugin lifecycle, with no env/config override in the Discord config schema, generated metadata, public docs, or monitor options. The report is fresh, user-authored, unlabeled, and actionable in the bundled Discord plugin rather than a stale or ClawHub-only request.

## Best Possible Solution

Keep the issue open and implement this in the bundled Discord plugin: add validated per-account/root Discord config fields for startup/runtime gateway READY timeouts, thread them through `monitorDiscordProvider` into `runDiscordGatewayLifecycle`, update generated config metadata and public Discord docs, and add lifecycle/config tests covering defaults plus overrides. A dynamic default based on the configured multi-account startup stagger would also directly address the reported setup.
