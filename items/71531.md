## Summary

Keep #71531 open. Current main does not implement `channels.slack.logLevel`: the Slack config schema is strict and has no `logLevel` field, the Slack monitor does not read such a value, and `createSlackBoltApp` constructs Bolt `App` without passing `logLevel` or a custom logger. The linked PR #71550 appears to be the right implementation path, but it is still open/unmerged in the provided context, so the issue should not be closed as implemented yet.

## Best Possible Solution

Keep #71531 open and review/merge PR #71550 or an equivalent patch. The likely best fix is to add a validated Slack log-level enum to the Slack config surface, thread the resolved account-level value into `createSlackBoltApp`, map it to Bolt's `LogLevel`, and cover both config parsing and App constructor forwarding in Slack plugin tests plus user-facing Slack docs.
