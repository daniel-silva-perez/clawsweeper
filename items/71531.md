## Summary

Keep #71531 open. Current main still does not accept or forward `channels.slack.logLevel`, and the linked PR #71550 is an open, unmerged implementation path rather than shipped behavior. The request is concrete, recent, dependency-backed, and actionable in the Slack plugin/config surface.

## Best Possible Solution

Keep #71531 open and let PR #71550, or an equivalent maintainer-approved patch, close it after landing. The fix should add a lowercase `channels.slack.logLevel` config contract matching `@slack/logger` values, thread it through Slack schema/types/docs/generated config surfaces, forward it to Bolt's `App` constructor, and keep focused tests for both configured and omitted behavior.
