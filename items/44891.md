## Summary

Keep open. Current `main` still routes Gemini memory embedding requests through `withRemoteHttpResponse`, and that helper still calls `fetchWithSsrFGuard` without `mode: "trusted_env_proxy"`. In `fetch-guard`, missing `mode` resolves to strict mode, so `HTTPS_PROXY`/`HTTP_PROXY` are not honored for this path.

## Evidence
