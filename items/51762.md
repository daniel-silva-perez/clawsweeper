## Summary

Keep open. Current main still does not implement the PR’s core request: there is no `agents.defaultAgentId` config surface, no `OPENCLAW_DEFAULT_AGENT_ID` handling, and the early/default agent-dir and session-key paths still fall back to the hardcoded `main` agent. The linked issue #17477 being closed as stale does not supersede the PR, and the provided PR discussion shows the branch addressed several review findings but still likely needs rebase/rework rather than cleanup closure.

## Best Possible Solution

Keep the PR open and ask for a rebase/rework against current main. The best fix is a single coherent default-agent contract covering `agents.defaultAgentId`, `OPENCLAW_DEFAULT_AGENT_ID`, `resolveDefaultAgentId()`/`listAgentIds()`, early bootstrap agent-dir resolution, config-aware default-agent dir callers, main-session/session-key derivation, session store path selection, schema/generated docs, and focused regression tests for config, env, bootstrap dir, session key, and store path behavior.
