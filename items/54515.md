## Summary

Keep open. Current main has fixed the misleading same-base correction-release warning, but the packaging/version-skew issue in the title is still present: the root package version is ahead of bundled extension package versions on `main`, the latest tagged release `v2026.4.22` shows the same skew, and `scripts/openclaw-npm-release-check.ts` does not currently enforce root/extension version alignment.

## Evidence

- **Current main still has root vs bundled extension version skew:** `package.json:3` is `2026.4.24`, while `extensions/deepgram/package.json:3`, `extensions/deepseek/package.json:3`, and `extensions/groq/package.json:3` are all `2026.4.20`. That is the same class of mismatch the issue reports.
  - file: [package.json:3](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/package.json#L3)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Latest release tag still shows the mismatch:** Read-only inspection of the tagged release tree shows `v2026.4.22:package.json` at `2026.4.22`, while `v2026.4.22:extensions/deepgram/package.json`, `.../deepseek/package.json`, and `.../groq/package.json` are all `2026.4.20`. So this was not eliminated by the latest shipped release.
  - command: `git show v2026.4.22:package.json | sed -n '1,8p' ; git show v2026.4.22:extensions/deepgram/package.json | sed -n '1,20p' ; git show v2026.4.22:extensions/deepseek/package.json | sed -n '1,20p' ; git show v2026.4.22:extensions/groq/package.json | sed -n '1,20p'`
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Release check does not enforce extension/root version alignment:** `scripts/openclaw-npm-release-check.ts:590-606` only collects root package metadata, tag, and packed tarball errors before succeeding. `package.json:1409` defines a separate `plugins:sync` script, but `package.json:1432` wires `release:openclaw:npm:check` only to `scripts/openclaw-npm-release-check.ts` with no plugin-version alignment step.
  - file: [scripts/openclaw-npm-release-check.ts:590](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/scripts/openclaw-npm-release-check.ts#L590)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **The confusing warning symptom was fixed separately:** `src/config/version.ts:103-123` now suppresses warnings for same-base stable/correction families, and `src/config/version.test.ts:70-88` covers `2026.3.23` vs `2026.3.23-1/-2`. The changelog entry at `CHANGELOG.md:2169` documents that fix under `## 2026.3.23`. This addresses part of the report, but not the remaining packaged version skew.
  - file: [src/config/version.ts:103](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/config/version.ts#L103)

That suggests some mismatch may be tolerated by design, but it does not negate the issue's remaining concern about bundled package metadata skew and release-pipeline enforcement.
