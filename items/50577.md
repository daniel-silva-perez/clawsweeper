## Summary

Keep open. Current `main` still does not resolve slash-containing bare model IDs in `sessions.patch`, so this is not `implemented_on_main` or `cannot_reproduce`. The PR also bundles unrelated model-compat behavior, so it still needs maintainer review/splitting rather than closure.

## Evidence

- **`sessions.patch` still has no slash-ID fallback on main:** Current `main` calls `resolveAllowedModelRef(...)` once and immediately returns `invalid(resolved.error)` on failure; there is no retry or unique-provider inference path in this call site.
  - file: [src/gateway/sessions-patch.ts:412](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/gateway/sessions-patch.ts#L412)
  - command: `nl -ba src/gateway/sessions-patch.ts | sed -n '390,455p'`

  - file: [src/agents/model-selection-shared.ts:610](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/agents/model-selection-shared.ts#L610)
