## Summary

Current main already supports opt-in cross-agent memory access through per-agent memory scope configuration: `memorySearch.extraPaths` for selected external memory files/directories and `memorySearch.qmd.extraCollections` for selected cross-agent QMD transcript collections. The shipped design is scope-based rather than adding `allowReadFrom` and per-call `agent` selectors, but it covers the requested capability well enough.

## Close Comment

Codex review: current `main` already supports opt-in cross-agent memory access through per-agent memory scope config. For general memory files, `memory_get` allows reads from configured `memorySearch.extraPaths` (`src/memory-host-sdk/host/read-file.ts:14` and `src/memory-host-sdk/host/read-file.ts:86`). For transcript memory, `memorySearch.qmd.extraCollections` is documented and implemented as a per-agent directional cross-agent search scope (`docs/concepts/multi-agent.md:145`, `docs/reference/memory-config.md:285`, `src/memory-host-sdk/host/backend-config.ts:379`). This was shipped in the `v2026.3.31` changelog (`CHANGELOG.md:1624`, `CHANGELOG.md:1655`). The final design is scope-based rather than adding an `allowReadFrom` plus per-call `agent`/`agents` params, but it covers the requested opt-in cross-agent memory-read capability on current main, so I’m closing this as implemented.
