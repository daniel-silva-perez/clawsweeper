## Summary

Keep open. Current `main` still contains the reported fallback that bypasses a parent agent's effective model for subagent spawns, so the underlying bug is not implemented on `main`. The PR's one-line deletion matches the happy-path report, but against current code it also removes the last valid fallback to `agents.defaults.model.primary` when a per-agent override is malformed, so the proposed change is not strong enough to close conservatively without a more complete fix and test coverage.

## Evidence

  - file: [src/agents/model-selection.ts:247](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/src/agents/model-selection.ts#L247)

- **`runtimeDefault` already resolves the parent agent's effective model:** `resolveDefaultModelForAgent()` injects the effective per-agent primary model into `agents.defaults.model.primary` before calling `resolveConfiguredModelRef()`, which is exactly the path the PR is trying to rely on.
  - file: [src/agents/model-selection.ts:195](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/src/agents/model-selection.ts#L195)
