## Summary

Keep this issue open. Current main has the normal systemd restart path and health checks, but I did not find source, tests, changelog, or merged-history evidence that the reported direct `openclaw gateway restart` `exit=null` behavior on systemd was intentionally fixed. The only directly linked fix PR, #68302, is closed unmerged, and its proposed `process.exitCode = 0` helper-side patch is not present on main.

## Best Possible Solution

Keep open and fix narrowly in the gateway restart command path. First reproduce whether the direct CLI is being killed because it runs inside the service cgroup; if so, use a command-level detached handoff similar to the update restart helper, while preserving existing restart health checks and non-zero failure behavior. Avoid a helper-level `process.exitCode = 0` side effect because it can mask legitimate earlier failures and was already flagged in the closed unmerged PR.
