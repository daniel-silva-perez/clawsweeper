## Summary

Keep #7707 open. Current main has adjacent provenance/source concepts, session visibility filtering, temporal decay, active-memory untrusted prompt framing, and Memory Wiki provenance, but it still does not implement persistent source-based memory trust tiers, review/expiration policy by trust level, or trust-weighted retrieval/downranking. The existing April 26 steipete Codex review and HMAKT99 AKF comment are useful prior discussion, and the current source still supports the same keep-open conclusion.

## Best Possible Solution

Keep #7707 open for maintainer/product/security design. A complete solution should add structured origin trust metadata at memory ingestion, persist it in memory schemas/backends, expose documented config policy, enforce retrieval-time filtering/downweighting/review gates, support expiration by trust tier, preserve provenance through compaction and Memory Wiki flows, surface trust state in UI/docs where relevant, and add regression tests across builtin/QMD/active-memory paths. HMAKT99's AKF comment and existing provenance features are useful prior art.
