## Summary

Current main still contains the reported resource-lifecycle gap. The guarded provider transport wraps streamed responses and releases the guarded fetch dispatcher only on no body, natural EOF/read error, or explicit stream cancel; there is no FinalizationRegistry, idle watchdog, or abandoned-stream cleanup path. The latest release tag has the same wrapper, and the existing tests do not cover abandoned streamed response cleanup.

## Best Possible Solution

Keep this issue open and fix the shared guarded provider transport lifecycle. The likely patch belongs in `src/agents/provider-transport-fetch.ts`, with regression coverage that proves `result.release()` runs when a streamed `Response` is abandoned before EOF and without explicit `cancel()`. The fix should also audit Anthropic Messages and Anthropic Vertex streaming consumers so the SDK path either drains/cancels explicitly or is protected by a robust wrapper-level safety net.
