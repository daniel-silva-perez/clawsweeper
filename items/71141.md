## Summary

Keep #71141 open. Current main still lacks the specific Telegram reaction-directive path the PR targets: reply normalization does not parse `[[react...]]`, Telegram delivery still rejects textless/media-less payloads before any reaction handling, and the Telegram outbound adapter does not execute `channelData.telegram.reaction`. The related issue #71140 remains a concrete, source-backed bug report, and current main already has a native Telegram reaction primitive that this PR can wire into.

## Best Possible Solution

Keep the PR open as a plausible focused fix for #71140. The best path is to review or adapt its changes into current main with tests covering reply directive parsing, `channelData.telegram.reaction` preservation, reaction-only Telegram replies, and outbound adapter execution before optional text/media sends.
