## Summary

Keep this PR open. Current main still has the strict SDK `client.listTools()` path, so the linked MCP compatibility bug is not implemented on main. The PR is a focused implementation candidate and v3 addresses the prior explicit non-object schema concern, but it still replaces SDK `Client.listTools()` with raw `client.request()`, which bypasses metadata caching in the pinned MCP SDK v1.29.0.

## Best Possible Solution

Keep this PR open and request a small rework: retain the lenient `tools/list` parsing and catalog-time rejection of explicit non-object root schemas, but preserve the SDK `listTools()` metadata side effect before `callTool()` can run. Add a harness regression with a real or mock MCP server that omits `inputSchema.type`, plus coverage that output-schema validation or task-required metadata still works after discovery. After that, this is a good candidate to land for #63602.
