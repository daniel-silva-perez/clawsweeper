## Summary

Keep open. Current `main` still crashes on malformed `content` entries in the same UI helper paths this PR targets, so there is no strong basis to close it as already implemented or obsolete.

## Evidence

- **Current normalizer still dereferences malformed entries:** `normalizeMessage` reads `x.type` in the tool-content prepass and later `item.type` inside `flatMap` without first checking that each `content` entry is a non-null object, so `null`/`undefined` array members still throw.
  - file: [ui/src/ui/chat/message-normalizer.ts:251](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/ui/src/ui/chat/message-normalizer.ts#L251)
  - command: `nl -ba ui/src/ui/chat/message-normalizer.ts | sed -n '236,335p'`
  - sha: [4630ce3d9e2f](https://github.com/openclaw/openclaw/commit/4630ce3d9e2f089062ccd386b0ae241325961d1a)
- **Current extractors still dereference malformed entries:** `extractThinking` and `extractRawText` cast each array member and immediately read `item.type`, so malformed `content` blocks still raise `TypeError` instead of being skipped.
  - file: [ui/src/ui/chat/message-extract.ts:49](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/ui/src/ui/chat/message-extract.ts#L49)
  - command: `nl -ba ui/src/ui/chat/message-extract.ts | sed -n '40,120p'`
  - sha: [4630ce3d9e2f](https://github.com/openclaw/openclaw/commit/4630ce3d9e2f089062ccd386b0ae241325961d1a)
- **Reasonable repro still fails on current main logic:** A direct reproduction of the current `normalizeMessage` prepass with `content=[null, undefined, ['nested'], {type:'text',text:'Recovered'}]` throws `TypeError: Cannot read properties of null (reading 'type')`. Matching repros for the current `extractRawText` and `extractThinking` loops also throw the same error.
  - command: `node -e "const content=[null,undefined,['nested'],{type:'text',text:'Recovered'}]; ..."; node -e "const content=[null,undefined,['nested'],{type:'text',text:'Hello there'}]; ..."; node -e "const content=[null,['nested'],{type:'thinking',thinking:'Plan A'},undefined]; ..."`
  - sha: [4630ce3d9e2f](https://github.com/openclaw/openclaw/commit/4630ce3d9e2f089062ccd386b0ae241325961d1a)
- **Current tests do not cover malformed-block regression:** The checked-in tests exercise normal string/array content and thinking extraction, but there is no regression case for `null`/`undefined`/array-valued blocks in the current `main` test files.
  - file: [ui/src/ui/chat/message-normalizer.test.ts:19](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/ui/src/ui/chat/message-normalizer.test.ts#L19)
