## Summary

Keep open. Current main still has the reported lifecycle gap: supervisor orphan reconciliation is a no-op, supervisor/process registries are in-memory, and generic background PTY exec can run without an explicit timeout or watchdog. #66001 is an open same-author implementation candidate tied to this issue with GitHub closing syntax, so this issue should remain open until that PR is reviewed, merged, or explicitly closed.

## Best Possible Solution

Review #66001 as the implementation candidate. Maintainers should decide whether OpenClaw wants durable cold-start orphan reaping for PTY exec runs or the smaller in-process reconciliation contract, then land the chosen fix with focused regression tests around lost session ownership, PTY background runs, and process-list visibility.
