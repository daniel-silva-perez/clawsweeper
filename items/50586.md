## Summary

Keep open. Current main does not include the proposed `chcp 65001` behavior, so this is not `implemented_on_main`, and there is not enough evidence to call the underlying Windows encoding report obsolete. The PR also carries an unresolved correctness risk because OpenClaw routes arbitrary plugin-supplied commands through the same Windows batch wrapper path, so forcing UTF-8 before every `.cmd`/`.bat` launch could break legacy batch files encoded in the system code page.

## Evidence

- **Current main still uses plain cmd.exe wrapper:** `resolveChildProcessInvocation()` wraps Windows batch commands as `cmd.exe /d /s /c <command line>` with no `chcp 65001` prefix, and both `runExec()` and `runCommandWithTimeout()` call that helper.
