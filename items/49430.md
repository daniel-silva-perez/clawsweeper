## Summary

Keep open. Current main still has the sub-agent documentation drift this PR is trying to fix, and the PR discussion shows useful active review work rather than an obsolete change. The patch also needs rebase/current-source updates before merge, especially for the current announce timeout default, ACP-only `streamTo`, and complete sub-agent tool-deny wording.

## Best Possible Solution

Keep a canonical #47558 docs fix open and rework/rebase this PR or a successor. The final patch should document `agents.defaults.subagents.announceTimeoutMs` with the current 120000ms default, add `streamTo` only as ACP-only, separate global and per-channel thread-binding keys for current supported channels, generalize depth/announce/cascade/auto-archive language through `maxSpawnDepth` 1-5, and accurately describe always-denied versus leaf-only sub-agent tools before closing the issue.
