## Summary

Keep #7456 open. Current main still does not implement wizard-wide Back navigation for `openclaw onboard`: the interactive CLI path remains a linear call into `runSetupWizard`, the shared prompt contract and Clack adapter have no back/navigation result, and the Gateway wizard RPC only supports start/next/cancel/status. The prior #7762 work is useful context, but it was closed unmerged; #5880 covers interrupted-session resume, not in-session back navigation.

## Best Possible Solution

Keep #7456 open. The best path is a core wizard navigation contract: add an explicit Back/previous-step result to `WizardPrompter` and the Gateway wizard schemas/session, refactor `runSetupWizard` into a typed state machine that preserves answers across backward and forward movement, align existing local submenu Back behavior with that general model, and add CLI/RPC regression coverage. PR #7762 can be used as prior art, but should not be treated as shipped or canonical because it was closed unmerged.
