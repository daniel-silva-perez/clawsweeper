## Summary

Keep #69512 open. Current main has partial Claude CLI permission-mode handling, but it does not forward or enforce `exec-approvals.json` allowlists for Claude Code internal shell execution. OpenClaw-owned gateway/node exec still enforces allowlist misses, while the CLI backend path launches Claude with backend args and relies on Claude's permission surface.

## Best Possible Solution

Keep the issue open. The likely path is an Anthropic-owned `claude-cli` permission bridge, exposed through a generic CLI-backend/native-permission seam if needed, that maps Claude Bash permission requests to OpenClaw exec approval evaluation and denies allowlist misses. Add regression coverage for allowed and denied commands under per-agent `tools.exec.security: "allowlist"`, plus docs that define Claude CLI parity and known limitations.

## Risks / Open Questions

- A correct fix is security-sensitive: naive `--allowedTools` wiring may not match OpenClaw shell parsing, safe-bin, inline-eval, and ask-fallback semantics.
