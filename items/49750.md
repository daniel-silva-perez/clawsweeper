## Summary

Keep open. Current main still lacks the agent-prefixed group/session normalization that #49750 targets, and restart-sentinel routing only partially overlaps the PR. The PR has useful focused tests and implementation work, but its review comments identify real edge cases that should be fixed before merge rather than closing it as obsolete.

## Best Possible Solution

Keep the PR open and revise it against current main: use the existing explicit-kind session conversation parser or equivalent logic for agent-prefixed group/channel keys, add focused resolveGroupSessionKey regression coverage, and update restart-sentinel routing so parsed thread targets fill missing route fields ahead of stale base-session fallback while preserving partial sentinel/account hints.
