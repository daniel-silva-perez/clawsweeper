## Summary

Keep open. Current `main` still does not provide the richer device-identifying details requested for `openclaw devices list/remove`, and the paired-device CLI output can still hide the exact `deviceId` that `devices remove` requires.

## Evidence

- **Paired CLI output still lacks requested metadata:** `openclaw devices list` renders paired devices with only `Device`, `Roles`, `Scopes`, `Tokens`, and `IP`. There is no browser, OS version, platform/device family, or last-active column in the current CLI output.
  - file: [src/cli/devices-cli.ts:407](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/cli/devices-cli.ts#L407)
  - command: `nl -ba src/cli/devices-cli.ts | sed -n '398,450p'`
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **`devices remove` still depends on a raw device ID:** The remove command still requires `<deviceId>`, but the paired-device table prints `displayName || deviceId`, so the exact ID can be hidden whenever a display name is present.
  - file: [src/cli/devices-cli.ts:414](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/cli/devices-cli.ts#L414)
  - command: `nl -ba src/cli/devices-cli.ts | sed -n '398,450p'`
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **Data model has richer fields that are not surfaced here:** The pairing model already stores `platform`, `deviceFamily`, `clientId`, `clientMode`, `createdAtMs`, `approvedAtMs`, and token `lastUsedAtMs`, which supports the report that the current device list is not exposing enough distinguishing detail.
  - file: [src/infra/device-pairing.ts:23](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/infra/device-pairing.ts#L23)
  - command: `nl -ba src/infra/device-pairing.ts | sed -n '20,90p'`
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **Docs only describe requested-vs-approved access context:** The current CLI docs for `openclaw devices list` mention requested-versus-approved access, but do not document added device-identification fields for disambiguating same-type devices.
  - file: [docs/cli/devices.md:15](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/docs/cli/devices.md#L15)
