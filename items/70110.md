## Summary

Current main still has the reported Nextcloud Talk reaction wiring gap: the plugin advertises reactions and has a reaction sender, but it does not register a channel message action adapter, so shared `message` tool `react` dispatch cannot reach `sendReactionNextcloudTalk`. PR #72348 is an open, unmerged implementation candidate that references this issue with closing syntax, so this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land PR #72348, or an equivalent fix that adds a Nextcloud Talk `ChannelMessageActionAdapter`, registers it from `extensions/nextcloud-talk/src/channel.ts`, dispatches `react` to `sendReactionNextcloudTalk`, and adds regression coverage for discovery, dispatch, message-id fallback, and reaction request shape.
