## Summary

Keep #70110 open. Current main still appears to advertise Nextcloud Talk reactions and contains a low-level `sendReactionNextcloudTalk` helper, but the plugin does not wire a `react` message action to that helper, so the reported user-visible bug is still actionable.

## Best Possible Solution

Keep the issue open and fix it in the bundled Nextcloud Talk plugin: add a channel `actions` adapter or narrow message-tool API that exposes `react`, resolves the current inbound `messageId` fallback, calls `sendReactionNextcloudTalk`, defines any remove/list semantics explicitly, and adds regression tests covering `message` tool dispatch for Nextcloud Talk reactions plus docs alignment.
