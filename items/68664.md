## Summary

Keep open. Current main has the broad runtime migration-before-validation flow, but it does not implement the specific cleanup for already-nested legacy `agents.defaults.memorySearch` keys. Related PR #68685 is open, unmerged, and explicitly targets this remaining gap, so this issue should not be closed before that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open until #68685 or an equivalent maintainer-reviewed core fix lands. The fix should add a runtime/doctor migration for already-nested `agents.defaults.memorySearch` legacy keys before validation, decide whether to remap `chunkSize`, `chunkOverlap`, and flat `maxResults` to the current nested schema or strip them intentionally, and add focused config-load plus migration regression coverage.
