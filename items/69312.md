## Summary

Keep this PR open. Current main still only protects fenced code blocks before parsing `MEDIA:` directives, so 4-space or tab-indented code examples can still be treated as real media directives. The PR is a focused implementation candidate tied to the concrete reproduction in #69311, with active review iteration rather than cleanup material.

## Best Possible Solution

Keep the PR open and have a maintainer review or land a corrected version that adds indented-code protection in `src/media/parse.ts`, preserves code indentation during cleanup, adds the media and fence regression tests, and deliberately aligns `MEDIA:` case semantics across parsing, streaming buffering, and reply delivery tests.
