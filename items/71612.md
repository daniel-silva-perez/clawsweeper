## Summary

Keep PR #71612 open. Current main still records tool-loop history against the stable cron agent session key used during tool execution, and `recordToolCall` still only appends/trims by history size with no idle-gap or run-boundary reset. The PR is recent, focused, and includes concrete reproduction plus proposed regression coverage; the related session/reset items do not supersede this specific cron tool-loop false-positive.

## Best Possible Solution

Keep the PR open for maintainer review. The product fix should scope tool-loop history to a real isolated cron run boundary for jobs sharing `agent:<id>:cron:<jobId>`, either by accepting the PR's idle-gap reset with regression tests or by passing an explicit per-run diagnostic key/reset hook into tool diagnostics.
