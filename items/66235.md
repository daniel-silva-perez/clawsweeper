## Summary

Keep open. Current main still does not expose `stopReason` or `hasToolCalls` to `message_sending` hook handlers. The hook event type, shared outbound delivery metadata builder, inbound-dispatch hook shim, docs, and tests all show the hook remains delivery/routing focused. Related PR #39761 overlaps but is still open feature work, not a shipped implementation or a clear reason to close this issue now.

## Best Possible Solution

Keep the issue open and implement a backward-compatible plugin hook contract change: thread assistant completion metadata from agent-originated delivery into `DeliverOutboundPayloadsParams`, expose optional `stopReason` and `hasToolCalls` on the `message_sending` event or typed metadata, update docs, and add focused tests for shared outbound delivery plus inbound/agent reply paths. If #39761 becomes the chosen broader solution, link this issue there and close only after the merged code demonstrably exposes the requested fields.
