## Summary

Keep this issue open. Current main still does not expose `stopReason`, `hasToolCalls`, or an equivalent agent completion context to `message_sending` hook handlers. The requested signal exists upstream in agent run metadata, but the outbound delivery API and hook event builder only carry delivery/routing metadata. Related PR #39761 is still open candidate work, not shipped behavior and not a reason to close this issue.

## Best Possible Solution

Keep the issue open and implement a backward-compatible hook contract change. The likely fix is to thread agent completion metadata from agent run results into outbound delivery params, then expose optional `stopReason` and `hasToolCalls` or a broader typed `agentContext` on `PluginHookMessageSendingEvent`, with docs and focused tests for shared outbound delivery plus inbound/agent reply paths. If #39761 becomes the chosen broader solution, close this issue only after that PR merges and current main demonstrably exposes the requested signal.
