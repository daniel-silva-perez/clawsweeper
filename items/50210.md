## Summary

Keep open. Current main still has the gateway/session-default mismatch that #50210 is trying to fix: session defaults are resolved from global configured defaults rather than the default/requested agent’s effective model. The PR is still useful, but it should not land as-is because the latest PR state includes an unresolved context-token mismatch and an unrelated hooks change.

## Best Possible Solution

Keep the PR open, but ask for a narrowed gateway-only update: make `getSessionDefaults` accept/resolve the selected agent id, derive model, context window, and thinking defaults from that same effective agent model/config, pass `opts.agentId` through `listSessionsFromStore`, add focused regression tests for default-agent and filtered-agent cases, and split the unrelated hooks canonicalization into a separate PR.
