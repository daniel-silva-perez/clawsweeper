## Summary

Keep PR #50692 open. Current main still does not implement Anthropic native web_search: the bundled Anthropic plugin registers its model provider and media-understanding provider only, its manifest declares no webSearchProviders contract, there are no Anthropic web-search provider files on main, and the public web-search docs do not list Anthropic. The PR is therefore not obsolete or implemented-on-main, and it is not superseded by the linked feature issue #49949. The provided PR review context also shows substantive unresolved PR-head concerns around gating native injection behind configured Anthropic web search and passing search config into the wrapper, so the best maintainer action is changes-requested/follow-up work rather than auto-close.

## Best Possible Solution

Keep #50692 open for maintainer review and changes. The implementation should stay in the bundled Anthropic plugin, declare the webSearchProviders manifest/public-artifact surface, register a WebSearchProviderPlugin, compose a provider stream wrapper that only activates when Anthropic native search is explicitly selected and web_search is enabled, pass the resolved Anthropic search config into payload construction, suppress or coordinate the managed web_search tool when native search is active, and add docs plus contract/runtime tests for those behaviors.
