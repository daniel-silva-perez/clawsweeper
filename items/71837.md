## Summary

Keep open. Current main has partial memory-core denoise and REM phase-signal work, but it does not implement the PR’s central behavior: normal REM candidate truths are not fed back into grounded short-term candidates, deep gating still does not count phase reinforcement for signal/diversity, and the broader scaffold/meta filtering from the PR is still missing in key paths.

## Best Possible Solution

Keep #71837 open. Rebase it onto current main, preserve the already-shipped phase-signal and session-denoise safeguards, add the missing REM-to-grounded short-term candidate path with stable query identity and confidence-aware weighting, broaden the meta/scaffold filters, and validate the focused memory-core/session extraction tests before maintainer merge consideration.
