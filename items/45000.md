## Summary

Current main already strips raw `<tool_call>...</tool_call>` / Qwen-style XML tool payloads from assistant-visible output, and the changelog records that fix as shipped in v2026.4.10.

## Risks / Open Questions

- This proves the reported visible XML leak is fixed. If a current Ollama/Qwen stack still fails to emit native `tool_calls` and tools do not execute, that would be a separate runtime/provider behavior issue rather than the specific leak reported here.

## Close Comment

Closing from a Codex review because the reported chat leak is already implemented on current `main` and was shipped in `v2026.4.10`. The fix is now in the shared assistant-visible sanitizer (`src/shared/text/assistant-visible-text.ts:199` and pipeline use at `src/shared/text/assistant-visible-text.ts:588`), and the visible reply path goes through that sanitizer (`src/agents/pi-embedded-utils.ts:33`). There is also direct regression coverage for Qwen-style `<tool_call><function=...>` payloads in `src/shared/text/assistant-visible-text.test.ts:155`. The changelog entry at `CHANGELOG.md:963` documents this exact fix for visible replies. If you still see tool execution failures with a current OpenClaw build, current Ollama, and the same model family, please open a fresh issue with a new repro focused on the remaining runtime behavior rather than the already-fixed XML leak.
