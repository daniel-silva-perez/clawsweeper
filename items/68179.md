Keep this PR open. Current main still strips Discord explicit targets down to `target.id` in `parseDiscordExplicitTarget`, so `user:<id>` becomes a bare snowflake before downstream routing. The PR is focused, has relevant regression coverage in its diff, and is not already implemented or superseded on main.
