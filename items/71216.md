## Summary

Keep this issue open. Current main still lacks the requested first-class config/runtime surfaces: there is no root `sandbox` section, no `routing.rules` tag-based provider/model router, no `instances.*` multi-gateway declaration surface, and no `gateway.nodes.denyPaths` schema or runtime path enforcement.

## Best Possible Solution

Keep this issue open and split the work deliberately: decide whether a root `sandbox` key should alias or wrap `agents.defaults.sandbox`; design tag-aware provider/model routing in the agent/model-selection path with explicit metadata propagation; decide whether multi-instance declarations belong in config or the existing profile/gateway-management model; and implement `gateway.nodes.denyPaths` only with schema/types/docs, runtime path enforcement, doctor coverage, and security tests.

## Risks / Open Questions

- The issue bundles four separable product/API surfaces; a single schema-only PR would be hard to review and would not prove runtime behavior.
- `gateway.nodes.denyPaths` needs enforcement at node/tool/file chokepoints plus doctor/security tests; adding only schema/types/docs would create false assurance.
- `routing.rules` and `instances.*` affect model-selection metadata, profile/gateway management, and trust boundaries, so they need maintainer design before implementation.
