## Summary

Keep #71216 open. Current main has adjacent pieces, especially `agents.defaults.sandbox.mode`, model primary/fallback config, isolated gateway profiles/ports, and node command allow/deny policy, but it does not implement the requested `sandbox`, `routing.rules`, `instances`, or `gateway.nodes.denyPaths` schema surfaces as a complete core feature.

## Best Possible Solution

Keep this open for maintainer design. Split it into scoped follow-ups: decide whether to alias or document the existing `agents.defaults.sandbox.mode` path versus adding top-level `sandbox`; design a real gateway-wide path-deny contract before adding `gateway.nodes.denyPaths`; decide whether tag-based provider routing belongs in agent/model config, runtime tags, or a plugin API; and decide whether multi-instance validation should extend profile/service management or become a new `instances.*` config surface.
