## Summary

Keep PR #69954 open. Current main still deletes `sessions cleanup --fix-missing` entries based on a single resolved stored `sessionFile` path and does not check the canonical `<sessionId>.jsonl` fallback before pruning. The PR targets a real remaining cleanup-safety bug tied to #50248, though it should address the review-noted repair-only JSON accounting gap before merge.

## Best Possible Solution

Keep the PR open and revise it into a complete cleanup fix: when a persisted `sessionFile` is missing, check the sessionId-derived canonical transcript before pruning, repair stored metadata when the canonical file exists, report repair-only mutations accurately in dry-run and applied JSON, and keep focused regression tests for stale-path repair, true missing transcripts, and valid stored-path behavior. Consider aligning doctor’s missing-transcript check with the same fallback or tracking that as a follow-up.
