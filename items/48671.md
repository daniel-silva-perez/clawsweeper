## Summary

Current `main` does not implement this PR’s requested behavior. The webchat session dropdown is still built only from persisted `sessions.list` rows plus the active `sessionKey`, so known agents without stored sessions can still disappear. The PR remains useful, but it is stale against current `main` because the logic now lives in `ui/src/ui/chat/session-controls.ts`, not `ui/src/ui/app-render.helpers.ts`.

## Evidence
