## Summary

Keep this PR open. Current main has the malformed tool-call repair wrapper and the later #70294 transport enablement, but it still only parses valid ASCII-quoted JSON or balanced ASCII-quoted prefixes. I found no current implementation or regression test for Unicode smart quotes used as JSON string delimiters, which is the central behavior this PR targets for #67488.

## Best Possible Solution

Keep this PR or an equivalent follow-up open until maintainers either land a corrected smart-quote delimiter repair or explicitly decide #67488 should be handled another way. The best path is to rebase onto current main, keep the #70294 transport gate intact, add tests for Unicode structural quote delimiters, add tests preserving content-level smart quotes inside ASCII strings with trailing junk, and then review the parser changes against `extractBalancedJsonPrefix` contracts.
