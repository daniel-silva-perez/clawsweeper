## Summary

Keep this issue open. Current main has partial adjacent improvements, including default canvas-node selection, a separate `canvas snapshot` image path, browser screenshots, and Control UI `[embed ...]` rendering, but `canvas(action="present")` and `canvas(action="navigate")` are still node-backed controls that return JSON success rather than a visible chat preview. The reported `node required` class also remains reachable when no default canvas-capable node can be selected.

## Best Possible Solution

Keep this issue open for maintainer product triage. If `canvas present` or `canvas navigate` should confirm visual state in chat, add a preview-producing result, optional automatic snapshot, or Control UI status artifact with regression coverage. If they are only node controls, tighten tool descriptions/docs/error guidance so no-node cases direct users to select/pair a canvas node, call `canvas snapshot`, use browser screenshot/snapshot for webpages, or use `[embed ...]` for Control UI inline rendering.
