## Summary

Keep #49511 open. Current main still renders fenced `mermaid` blocks as ordinary code blocks in the Control UI chat markdown path, and there is no Mermaid dependency, renderer module, style import, or chat lifecycle scheduler on main. The PR is not obsolete; it targets a real missing core UI behavior, though it needs maintainer pickup/review because the discussion shows unresolved rendering lifecycle, accessibility, dependency, and CI concerns.

## Best Possible Solution

Keep this PR open as a valid Control UI feature request/implementation attempt. The best maintainer path is to revive or replace it with a scoped core UI change that renders `mermaid` fences in chat, lazy-loads Mermaid, sanitizes SVG output, schedules rendering anywhere the transformed markdown can appear, and includes regression tests for case-insensitive fences, streamed/incomplete diagrams, load/render retry, empty definitions, accessibility fallback, and non-chat markdown consumers.
