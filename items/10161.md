## Close Comment

Codex review: closing this as a ClawHub-scope request rather than an OpenClaw core issue. Current project direction explicitly says new skills should be published to ClawHub first and that new core skills that can live on ClawHub are out of scope (`VISION.md:69`, `VISION.md:101`). Current main already documents and implements the external distribution path for this: ClawHub is the public registry for OpenClaw skills/plugins (`docs/tools/clawhub.md:10`), native skill installs are handled through `openclaw skills install <skill-slug>` (`docs/tools/clawhub.md:23`), and the codebase already has ClawHub-backed skill install/update flows (`src/agents/skills-clawhub.ts:400`). There is also already an experimental Skill Workshop path for turning reusable procedures into workspace `SKILL.md` files (`docs/plugins/skill-workshop.md:16`). For `service-ability-creator`, the expected path is to publish/maintain it as a ClawHub skill or plugin and let OpenClaw consume it through the existing external skill/plugin system. If you hit a concrete limitation in those host/plugin seams, please open a focused issue for that specific gap.
