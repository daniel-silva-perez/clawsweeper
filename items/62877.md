## Summary

Current main still returns the raw `Usage: /btw <side question>` text when `/btw` has no side question. The existing unit test also asserts that exact unescaped string, so the reported markdown-channel problem is not implemented or superseded on main.

## Best Possible Solution

Keep the issue open. Fix the producer string in `src/auto-reply/reply/commands-btw.ts` to render the whole command as code, for example `Usage: `/btw <side question>`` or an equivalent escaped form, update `commands-btw.test.ts`, and then audit nearby auto-reply command usage strings with angle-bracket placeholders for the same markdown-channel behavior.
