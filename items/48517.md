## Summary

Keep open. Current `main` has subagent and sandbox primitives, but the plugin hook/runtime contracts still do not let a plugin request quarantined execution of an arbitrary operation or receive standardized trust/provenance metadata back. This is an unimplemented core architecture feature, not a ClawHub-only idea, and the issue is only 39 days old as of 2026-04-24 so `stale_insufficient_info` does not apply.

## Evidence

- **`before_tool_call` contract is still narrow:** `PluginHookBeforeToolCallResult` only exposes `params`, `block`, `blockReason`, and `requireApproval`. There is no isolation directive, sandbox request, trust level, or provenance payload in the public hook type.
  - file: [src/plugins/hook-types.ts:336](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/plugins/hook-types.ts#L336)
  - command: `nl -ba src/plugins/hook-types.ts | sed -n '332,470p'`
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Hook runner only merges block/params/approval:** `runBeforeToolCall()` merges modified params, sticky `block`, and `requireApproval`, then stops on `block=true`. The runtime path does not branch into any isolated execution flow from this hook.
  - file: [src/plugins/hooks.ts:781](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/plugins/hooks.ts#L781)
  - command: `nl -ba src/plugins/hooks.ts | sed -n '777,816p'`
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Subagent hook is about delivery/thread binding, not trust boundaries:** `PluginHookSubagentSpawningResult` only allows `{ status: "ok", threadBindingReady?, deliveryOrigin? }` or `{ status: "error" }`. It does not carry sandbox constraints, trust metadata, or provenance tagging.
  - file: [src/plugins/hook-types.ts:447](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/plugins/hook-types.ts#L447)

  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Plugin runtime subagent API lacks requested controls/results:** `api.runtime.subagent.run()` accepts `sessionKey`, `message`, optional model/provider overrides, `lightContext`, `deliver`, and `idempotencyKey`; results are just `runId` / wait status / session messages. The SDK docs show the same limited surface. There is no plugin-facing way to request constrained isolated execution with trust-tagged output.
  - file: [src/plugins/runtime/types.ts:8](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/plugins/runtime/types.ts#L8)

That supports the issue's premise that primitives exist, but this is still not the same as a plugin hook being able to quarantine an arbitrary operation and receive structured trust metadata back.
