## Summary

Keep #70783 open. Current main supports selecting and managing stored sessions, but it does not implement the requested tab strip for multiple simultaneous chat sessions with per-tab state, close/reorder controls, or persisted open tabs.

## Best Possible Solution

Keep the issue open and implement this in Control UI if maintainers accept the UX direction: add a browser-local, per-gateway open-chat-tab collection keyed by sessionKey; render a tab strip above the chat view; preserve per-tab draft, attachments, scroll/sidebar/search state, and active model/thinking controls; support create, switch, close, and optionally drag reorder; keep the existing Sessions view as the administrative/history table; add UI tests around tab persistence, stale history isolation, and `/new` tab creation semantics.
