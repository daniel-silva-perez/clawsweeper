## Summary

Keep PR #72085 open. The underlying docs gap still exists on current main: runtime and schema constrain `commands.bashForegroundMs` to 0..30000 ms, but the user-facing slash-command docs and config-channel docs still only mention default 2000 and `0` backgrounding. The PR is not closeable as implemented or superseded, though the submitted patch shape is too broad for the stated one-bullet docs fix.

## Best Possible Solution

Keep the PR open and ask for a narrow docs cleanup: document the 0..30000 ms range in `docs/tools/slash-commands.md` and likely `docs/gateway/config-channels.md`, and update `src/config/schema.help.ts` plus generated schema docs if `config.schema.lookup` should expose the bound in help text. Avoid adding a large duplicated command section to `docs/gateway/configuration-reference.md`.
