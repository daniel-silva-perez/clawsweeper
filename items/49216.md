## Summary

Keep PR #49216 open. Current main partially covers the common/default path, but the PR still addresses real remaining behavior: deleted archive cleanup is still gated by `resetArchiveRetentionMs` when no live session was pruned, and legacy archive timestamp formats are still not parsed or tested.

## Best Possible Solution

Keep this PR open and ask for a refresh against current main. The best path is a narrow sessions-store fix that always sweeps expired deleted archives in the store directory during enforced maintenance, keeps reset cleanup controlled by `resetArchiveRetention`, and preserves the PR's legacy timestamp parser/tests if those filename variants need compatibility. Do not close as implemented unless a separate main commit adds those behaviors and regression coverage.
