## Summary

This issue should stay open. Current main still renders `openclaw memory status` from the short-term/deep dreaming resolver, so light-only dreaming can still be summarized as `Dreaming: off`. There is also an open implementation candidate, #67910, whose body uses closing syntax for this issue, so the right maintainer path is to review and land or close that PR rather than close the issue now.

## Best Possible Solution

Review #67910 against the current `memory-core` CLI path. If the patch is acceptable, land it with a focused regression test for global dreaming enabled, light enabled, and deep disabled, letting the PR close this issue. If that PR is not suitable, implement the same phase-aware summary in `extensions/memory-core/src/cli.runtime.ts` using the full dreaming config or phase resolvers, with coverage in `extensions/memory-core/src/cli.test.ts`.
