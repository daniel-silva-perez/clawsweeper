## Summary

Keep open. Current main has not implemented the requested hardening: the Nextcloud Talk verifier still returns immediately on signature length mismatch and then uses a manual charCode XOR loop. The PR is focused on a live bundled plugin webhook-auth path and remains useful, though its patch should address the existing review comments before merge.

## Best Possible Solution

Keep PR #50516 open. The best path is to update the patch to use the repo’s established timing-safe comparison approach, or Node `crypto.timingSafeEqual` with a length-mismatch dummy comparison that uses distinct fixed-size buffers and avoids extra attacker-controlled allocation, then keep the focused Nextcloud Talk signature tests.
