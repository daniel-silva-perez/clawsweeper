## Summary

Keep #71569 open. Current main partially improves Mattermost visual spam by using a single draft preview post and in-place finalization when possible, but the central report remains valid: `channels.mattermost.streaming` is documented while Mattermost schema/types/generated metadata still omit it, and the inbound Mattermost runtime starts draft preview streaming unconditionally instead of honoring `off | partial | block | progress`. The related #71565 is superseded by this broader issue, and the provided #70253 ClawSweeper context independently points at the same unresolved Mattermost streaming config path.

## Best Possible Solution

Keep the issue open and fix it in the Mattermost plugin. Add `streaming` to the Mattermost config schema, generated metadata, and TypeScript types; make the inbound monitor resolve and honor `off`, `partial`, `block`, `progress`, and `streaming.preview.toolProgress`; preserve the current in-place finalization behavior; and add regression tests proving `channels.mattermost.streaming` is accepted and changes delivery behavior. Then decide explicitly how Mattermost should notify on final answers when previews are edited in place.
