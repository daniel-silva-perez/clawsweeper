## Summary

Keep #71569 open. Current main partially improved Mattermost draft previews and finalization, but the documented `channels.mattermost.streaming` surface is still not wired as a Mattermost config contract or mode gate: the strict Mattermost schema/types omit `streaming`, and the runtime still creates the draft stream unconditionally while disabling block streaming for the run.

## Best Possible Solution

Keep the issue open. Implement Mattermost's documented streaming config in the plugin schema and TypeScript config types, preserve scalar compatibility if docs keep showing `streaming: "partial"`, resolve the effective Mattermost preview mode before creating a draft stream, and add tests for schema acceptance, account inheritance, `off` disabling previews, and finalization/fallback behavior. Then update docs if the actual supported shape is `streaming.mode` instead of scalar `streaming`.
