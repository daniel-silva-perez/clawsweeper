## Summary

Current `main` does not implement this PR's feature. `openclaw system event` still only accepts `--text`/`--mode`, the gateway wake contract still only consumes `{ mode, text }`, and docs still describe main-session-only behavior. This PR should stay open rather than be closed as already implemented or obsolete.

## Evidence

- **CLI surface still lacks targeting flags:** `registerSystemCli()` defines `system event` with only `--text`, `--mode`, and `--json`, and forwards only `{ mode, text }` to `callGatewayFromCli("wake", ...)`. There is no `--agent-id` or `--session-key` handling on current `main`.
