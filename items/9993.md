## Summary

Current `main` has stronger built-in config-write validation and agent-side mutation guards, but it still does not implement the requested `config:pre-apply` hook or any equivalent extensibility point that can inspect `currentConfig` plus `pendingConfig` and veto the write before persistence.

## Evidence

- **Config writes validate, then write directly:** `config.patch` parses, merges, restores redacted values, runs `validateConfigObjectWithPlugins`, preflights SecretRefs, and then calls `writeConfigFile`; `config.apply` follows the same direct validate-then-write path. There is no hook dispatch in either flow.
  - file: [src/gateway/server-methods/config.ts:471](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/src/gateway/server-methods/config.ts#L471)
  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
- **Plugin hook surface has no config pre-write event:** The plugin hook union lists model/prompt/tool/message/session/gateway lifecycle hooks, but no config pre-apply or config pre-write hook.
  - file: [src/plugins/hook-types.ts:56](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/src/plugins/hook-types.ts#L56)
  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
- **Internal hook docs also omit config interception:** Documented internal hook events cover command, session, bootstrap, gateway startup, and message events only; there is no config write event documented for hooks.
  - file: [docs/automation/hooks.md:34](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/docs/automation/hooks.md#L34)
  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
- **Main already has partial mitigations, but not the requested extension point:** Docs state OpenClaw-owned config writers validate the full post-change config before commit, and the agent-facing gateway tool now hard-limits writes to a narrow allowlist that still includes model changes. That reduces the original incident surface, but it is not a customizable pre-apply hook.
  - file: [docs/cli/config.md:357](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/docs/cli/config.md#L357)
  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)

## Risks / Open Questions

- The motivating failure mode around syntactically valid but unavailable model IDs is not clearly eliminated by write-time validation alone.
- Closing as implemented would discard an unmet extensibility request: custom validators still cannot observe and block config writes through a first-class hook API.
