## Summary

Keep open. Current `main` still has the same data-source split the report describes: `openclaw nodes status` is driven by `node.list`, but `openclaw nodes list` still builds its paired rows from `node.pair.list` and only consults `node.list` for `--connected` / `--last-connected` filtering. That means the mismatch is still plausible on current `main` and in the latest shipped release.

## Evidence

- **`nodes status` reads from `node.list`:** The `status` command calls `callGatewayCli("node.list", opts, {})` as its primary source, then parses that into the displayed node rows.
