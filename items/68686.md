## Summary

Keep this PR open. Current main fixes the token-refresh part by making OAuth auth epochs stable across access/refresh token rotation, but it still resets CLI reuse on auth-profile mismatch and same-version auth-epoch mismatch. The PR is a focused implementation candidate, and its follow-up commit addresses the bot-reviewed config-priority edge case.

## Best Possible Solution

Keep the PR open for maintainer review, but ask for a rebased/narrowed implementation: preserve the CLI session only for auth-only changes that are proven same conversation/same identity, keep system-prompt and MCP mismatches as fresh-session resets, and add regression coverage for auth-only resume plus simultaneous auth+config changes. Avoid merging broad auth-profile resume until the provider-owned CLI session/account-boundary behavior is explicitly validated.
