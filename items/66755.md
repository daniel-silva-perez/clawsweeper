## Summary

Keep this PR open. Current main still only treats exact `NO_REPLY` payloads as silent, and mixed-content normalization still strips a trailing token while delivering the remaining preamble. This PR remains an active implementation candidate for open bug #66701, while related PR #69394 is also open and proposes a broader competing policy rather than a merged or maintainer-selected superseder. The PR also still has an unresolved P1 review concern around bare `think` matching, so it needs revision or explicit maintainer judgment rather than cleanup closure.

## Best Possible Solution

Keep this PR open as an active implementation candidate for #66701. The best path is to fix the bare-marker regex false-positive and land the narrow `src/auto-reply/tokens.ts` behavior with regression tests, or have maintainers explicitly choose a broader canonical implementation such as #69394 and close this PR only after that decision is resolved.
