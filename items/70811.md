## Summary

Keep this PR open. Current main still validates omitted plugin config as `{}` in both non-activating manifest-only snapshots and CLI metadata collection, which matches the linked false-positive config-validation bug. The PR remains open, unmerged, recently updated, and contains focused tests for the missing omitted-config paths; #64946 is related plugin validation noise but tracks a different wrapped/unwrapped schema disagreement rather than this omitted preflight config case.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or an equivalent loader change that distinguishes omitted config in non-activating metadata/snapshot paths from real activation, preserves normalized/defaulted `api.pluginConfig`, keeps activation and explicit-config validation strict, and retains the focused regressions in `src/plugins/loader.test.ts` and `src/plugins/loader.cli-metadata.test.ts`.
