## Summary

Keep PR #70811 open. Current main still has the reported loader behavior: both non-activating snapshot loads and CLI metadata loads validate `entry?.config` unconditionally, and `validatePluginConfig` still converts `undefined` to `{}` before AJV validation. The linked #70658 report has a concrete reproduction with a strict external plugin schema, and the PR discussion shows the author addressed earlier bot feedback around scoping, normalized `pluginConfig`, and defaults. This is not obsolete cleanup or a ClawHub-only item; it is a plausible focused core loader fix that still needs maintainer review.

## Best Possible Solution

Keep the PR open. The best path is a focused loader change in `src/plugins/loader.ts` that skips or defers validation only for non-activating loads with truly undefined plugin config, preserves strict validation for real activation and explicit config, keeps `api.pluginConfig` normalized with safe schema defaults, and lands the omitted-config regression tests in `src/plugins/loader.test.ts` and `src/plugins/loader.cli-metadata.test.ts`.
