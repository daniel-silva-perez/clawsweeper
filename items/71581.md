## Summary

Keep open. Current main still preserves forced exec-event and cron-event heartbeat session keys, canonicalizing only main aliases and same-agent keys; it does not consult session.dmScope or the single-owner allowlist before running the heartbeat. The provided related items are distinct and do not supersede this request.

## Best Possible Solution

Keep the issue open. The best fix is a focused core/session-routing change for internal system-event wakes: when session.dmScope is main and the same single-owner allowlist condition applies, resolve same-agent direct/per-channel forced session keys to resolveAgentMainSessionKey, and make enqueue/drain semantics consistent so exec and cron events are visible in the main session. Add regression coverage for exec completion and cron wake paths with a per-channel direct originating key, plus a guard that multi-owner or non-main dmScope deployments remain unchanged.
