## Summary

Keep open. The report is recent, author is not protected but labels are not protected, and it describes a concrete Feishu WebSocket failure with logs. Current main still routes Feishu websocket mode through the Feishu monitor transport, and I did not find strong source-backed evidence that main now caps SDK reconnect attempts, resets the session after repeated failures, or changes status probing to detect this degraded state. This is a plausible channel reliability bug, not a stale or ClawHub-only cleanup item.

## Evidence

- **Issue has concrete reproduction data:** The issue body provides environment, OpenClaw version 2026.3.13, Feishu WebSocket mode, observed repeated SDK log lines `unable to connect to the server after trying N times`, and a manual `openclaw gateway restart` recovery path. There are no comments or timeline updates resolving it.
- **Current main still defaults Feishu to websocket mode:** `monitorSingleAccount` resolves `connectionMode` from account config and defaults to `websocket`.
