## Summary

Current main still has the reported Lobster fallback behavior, and an open unmerged PR specifically targets this issue with closing syntax. This should remain open until #68106 or an equivalent fix lands.

## Best Possible Solution

Review and land #68106, or an equivalent Lobster plugin change in `extensions/lobster/src/lobster-runner.ts`, that preserves path/stat errors for workflow-file-like pipeline inputs and adds regression coverage for missing relative workflow files while keeping normal inline pipelines unchanged.
