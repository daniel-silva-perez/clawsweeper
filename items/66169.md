## Summary

Keep open. Current main still has the iMessage echo-cache gap this PR targets: echo text normalization only converts CRLF and trims whitespace, while the inbound self-chat and echo-drop paths pass reflected body text into that cache. A leading attributed-text prefix would still prevent text fallback matching on main. The PR is a focused iMessage-owner change with targeted tests, not a stale or superseded item, but it needs the existing bot review comments cleaned up before merge.

## Best Possible Solution

Keep this PR open as a plausible focused iMessage bug fix. The best path is to update the echo-cache implementation to resolve the bot review comments, keep the targeted echo-cache and self-chat regression tests, then run the iMessage monitor tests and changed gate before maintainer review/merge.
