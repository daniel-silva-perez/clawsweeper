## Summary

Keep open. Current main has a safer local-prefix install path that can use an OpenClaw-managed Node under `~/.openclaw`, but the requested dedicated/bundled macOS TCC-scoped runtime is not fully shipped or enforced for the reported npm/global/system-Node case. The macOS docs still explicitly say the app does not bundle Node or the Gateway runtime, and runtime resolution still supports shared Homebrew/system/version-manager Node paths.

## Best Possible Solution

Keep the item open or explicitly fold it into the protected #7227 security architecture thread with maintainer judgment. The concrete path should either make the macOS app/local install launch exclusively through a signed app-bundled helper or OpenClaw-managed Node, update TCC-facing docs and migration guidance for global npm users, and verify LaunchAgent/runtime resolution cannot silently fall back to a shared Node for permissioned workflows.
