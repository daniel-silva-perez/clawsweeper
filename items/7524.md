## Summary

Keep #7524 open. Current main still only collapses direct-message peers through `dmScope`; group/channel peers continue to resolve to isolated channel-scoped session keys, strict config schemas expose no `groupScope` or equivalent route-session override, and the relevant bundled group paths pass `peer.kind: "group"` into the same core routing. The broader #19929 is open but has not been made canonical, and the latest maintainer-facing comment explicitly kept #7524 open.

## Best Possible Solution

Keep #7524 open as the focused tracker unless a maintainer explicitly supersedes it with #19929. The best implementation is a core routing/config contract for selective group/channel-to-main or named shared sessions, wired through `buildAgentPeerSessionKey()`, `buildAgentSessionKey()`, `resolveAgentRoute()`, binding overrides, strict schemas/types/help/generated metadata, public docs, and regression coverage for at least BlueBubbles, Signal, and Telegram group routing.
