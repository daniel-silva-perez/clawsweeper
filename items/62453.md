## Summary

Keep open. Current `main` still emits the local-agent `--json` envelope through `runtime.log(...)`, and OpenClaw routes logs to stderr in JSON mode, so the reported stdout/stderr bug is still present. The proposed PR should also stay open because its `(runtime as OutputRuntimeEnv).writeJson(...)` change would narrow a declared `RuntimeEnv` contract instead of using the existing safe `writeRuntimeJson(...)` helper already used elsewhere.
