## Summary

Keep #7057 open. Current main still has the two reported WSL-sensitive test surfaces without WSL-specific guards or temp-dir cleanup hardening, and the related mitigation PRs were closed unmerged. The latest issue discussion already kept this open, and current source at 4ad8b613 confirms the same unresolved paths remain.

## Best Possible Solution

Keep #7057 open until a focused test-only fix lands for `src/agents/pi-tools.workspace-paths.test.ts` and `src/agents/pi-tools.safe-bins.test.ts`. The maintainer path should use the existing WSL detector only where justified, harden temp-dir creation/cleanup around cwd-sensitive tests, and include WSL proof or diagnostics without bundling unrelated changes.
