## Summary

Keep #39983 open. Current main still sends Ollama memory embeddings to `/api/embeddings` with a `prompt` payload and expects `embedding[]`, while the current Ollama API documents `/api/embed` with `input` and an `embeddings[][]` response. The related PRs in the issue timeline are useful, but none has landed in current main.

## Best Possible Solution

Keep the issue open and land one focused fix in `extensions/ollama/src/embedding-provider.ts`: call `/api/embed`, send `input` instead of `prompt`, parse the `embeddings[][]` response, preserve existing base URL/auth/header/SSRF behavior, and update `extensions/ollama/src/embedding-provider.test.ts` to assert the new contract. After one PR lands, close or supersede the duplicate open PRs against the merged fix.
