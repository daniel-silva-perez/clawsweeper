## Summary

Keep #71792 open. Current main still rolls stale daily/idle sessions into a new session and emits plugin session lifecycle hooks, but it does not emit the internal `command:reset` event that bundled `session-memory` listens for. The PR is active, focused on the remaining bug, and the related context does not show a merged or shipped replacement.

## Best Possible Solution

Keep #71792 open and review it as the focused core fix for auto daily/idle rollover memory persistence. The likely best path is to emit the existing internal `command:reset` event for non-command stale rollovers with `commandSource` values such as `auto:daily` and `auto:idle`, while preserving the current bundled `session-memory` command-event contract unless maintainers decide on a broader lifecycle-hook redesign.
