## Summary

Keep this PR open. Current main still creates new sessions for daily/idle stale rollover and emits plugin lifecycle hooks, but it does not emit the internal `command:reset` hook that bundled `session-memory` listens for. The PR is an active, focused implementation candidate for open #11960; related #37955 is closed and #43533 was closed unmerged, so there is no implemented, duplicate, or superseded close reason. Security review found no unexplained workflow, dependency, package, secret, or third-party execution changes in the provided PR file list/diff.

## Best Possible Solution

Keep this PR open and review it as the focused fix for auto daily/idle rollover memory persistence. The likely best path is to land this PR, or an equivalent change in `src/auto-reply/reply/session.ts`, that emits the existing internal `command:reset` hook for non-command stale rollovers with `commandSource` values like `auto:daily` and `auto:idle`, passes the resolved agent workspace, avoids double-firing for explicit resets and fresh sessions, and keeps the added daily/idle/no-op regression tests.
