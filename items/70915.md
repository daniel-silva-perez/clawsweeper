## Summary

Keep #70915 open. Current main still registers the ZAI media-understanding provider as a plain passthrough to the generic image runtime, while ZAI onboarding/config supports coding endpoints and the built-in catalog includes image-capable GLM vision models. The linked issue reports a real remaining product path, and the PR is a recent, focused fix attempt with tests rather than obsolete or duplicate cleanup material.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a ZAI-owned fix that routes image-capable GLM vision requests away from coding endpoints to the standard ZAI chat completions endpoint, with regression tests covering both global/CN coding URLs and avoiding a shared-config/models.json race.
