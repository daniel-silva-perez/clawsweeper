## Summary

Keep open. Current main has a partial related mitigation for read-only sandbox workspaces by copying skills into the sandbox workspace, but the reported plain `tools.fs.workspaceOnly: true` case still appears valid: managed/global skill paths are still prompt-facing paths outside the agent workspace, and the host read tool still rejects paths outside the workspace root when `workspaceOnly` is enabled.

The issue is less than 60 days old and author association is NONE, so stale/maintainer-author auto-close rules do not apply.
