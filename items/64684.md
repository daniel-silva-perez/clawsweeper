## Summary

Keep this PR open. Current main still does not implement the PR’s central behavior: failed update.run paths write only a doctor hint, restart sentinel payloads/messages have no doctorSummary field, and config-reload restart-check failures only log the failure instead of emitting doctor-backed diagnostics. The related closed #19992 and open #55347 do not supersede this focused contribution. Security review found no workflow, dependency, lockfile, package metadata, or third-party supply-chain changes in the provided PR file list; the main execution-sensitive piece is the new local doctor subprocess and should remain fixed-argv, no-shell, timeout-bounded, and output-bounded.

## Best Possible Solution

Keep the PR open for maintainer review or author refresh. Rebase it onto current main, preserve the update.run failure doctor-summary and restart-sentinel formatter work if maintainers want this UX, verify or restore the config-reload failure diagnostic path safely, and keep the subprocess hardening. If maintainers prefer the broader self-healing design, fold this focused bounded-summary behavior into #55347 rather than closing it as already implemented.
