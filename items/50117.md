## Summary

Keep open. Current `main` does not implement per-agent LLM request header injection. The provider request config only supports static `headers`/`auth`/`proxy`/`tls` overrides, the request-header resolver has no `agentId` input, and the provider/plugin stream-wrapper contexts also omit `agentId`, so there is no shipped path that could emit `X-OpenClaw-Agent` on outbound LLM requests.

## Evidence
