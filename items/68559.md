## Summary

Keep this PR open. Current main has not implemented the requested QMD/memory indexing behavior because `listSessionFilesForAgent` still indexes usage-counted transcripts, including reset and deleted archives. The PR is a focused implementation candidate, but it needs maintainer review/revision before merge because the proposed broad `.checkpoint.` substring filter can drop valid primary session IDs, and the test change does not explicitly assert reset/deleted exclusion.

## Best Possible Solution

Keep the PR open and revise it: change the memory/QMD discovery path to use the existing exact primary-transcript classification, avoid broad substring checkpoint filtering, add explicit reset/deleted/checkpoint exclusion assertions, and preserve separate usage-accounting behavior unless maintainers intentionally change that broader contract.

## Risks / Open Questions

- The proposed `.includes(".checkpoint.")` filter can silently skip legitimate primary transcript files whose allowed session ID contains `.checkpoint.` but is not an exact compaction checkpoint artifact.
- The PR test should put reset/deleted transcript variants in the excluded set so the central behavioral change is directly covered.
- If the intended policy is memory-indexing-only, the fix should stay on the memory/session discovery path; usage accounting and doctor/disk-budget helpers appear to have separate contracts.
