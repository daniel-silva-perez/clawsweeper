## Summary

Keep open. Current main still has no `LLM_MAX_CONCURRENT` or provider limiter, and the OpenAI, Azure OpenAI, and OpenAI completions transports still call the SDK directly. The PR is focused on a real core transport gap, but the provided review context shows it is incomplete and unsafe to merge as-is rather than obsolete or already implemented.

## Best Possible Solution

Keep the PR open for maintainer/author follow-up rather than closing it. The right implementation should add a tested core transport limiter that validates the env cap, wraps OpenAI responses, Azure OpenAI responses, and OpenAI chat completions, holds the slot until streaming completes, handles queued aborts, and documents the supported configuration if it becomes public.
