## Summary

Keep open. I did not find evidence that current `main` fixes this web chat image-overlay regression. The current UI still renders chat images as plain browser-draggable `<img>` elements with no `draggable="false"` or `-webkit-user-drag: none` protection, and the same pattern is present in the latest shipped release `v2026.4.22`. That matches the reported symptom of a transient full-window drag ghost that disappears on click, so the item is not strong-closeable as implemented or obsolete.
