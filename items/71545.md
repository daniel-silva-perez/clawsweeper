## Summary

Keep open. Current main still calls the cached MCP Client directly for session-scoped tool execution and has no streamable-http lost-session detection, reconnect, metadata refresh, or one-shot retry. The PR is a focused core-runtime fix, and the latest PR patch includes maintainer hardening plus regression tests for the reported streamable-http session-loss behavior.

## Best Possible Solution

Keep #71545 open and continue review toward landing or porting the hardened reconnect implementation: streamable-http callTool should detect SDK session-loss errors, establish a fresh transport/client with the existing custom schema validator, refresh tool metadata, share concurrent reconnects, retry once, and preserve retryability after failed reconnect attempts. The PR's latest patch already follows that shape and includes targeted regression tests.
