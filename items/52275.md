## Summary

Keep PR #52275 open. Current main preserves delivery metadata for newly queued outbound sends, but it still does not implement the PR's legacy queue-entry canonicalization before recovery or retry bookkeeping. The PR branch should not be merged as-is because the provided review context shows unrelated gateway edits with serious correctness/build issues; the useful path is to extract a narrow delivery-queue fix with focused tests.

## Best Possible Solution

Keep the item open but do not merge the current branch as-is. Extract a narrow outbound queue-storage change that canonicalizes legacy records before load and failure retry bookkeeping, preserving `forceDocument`, mirror idempotency/group context, `session`, and `gatewayClientScopes`, and add focused regression tests for both recovery load and `failDelivery` paths. Drop the unrelated gateway readiness/server implementation changes from this PR.
