## Summary

Current `main` no longer matches this report. The macOS companion app now starts a native mac node session on launch, connects it through `GatewayEndpointStore`, and `GatewayEndpointStore` resolves direct remote mode from `gateway.remote.url` instead of hardcoding `127.0.0.1:18789`. The same implementation is present in release `v2026.4.23`.

## Evidence

- **App launches native mac node mode:** The companion app starts `MacNodeModeCoordinator.shared.start()` during app launch, so node capabilities are managed by the native macOS app path rather than the old subprocess-only path.
  - file: [apps/macos/Sources/OpenClaw/MenuBar.swift:273](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/apps/macos/Sources/OpenClaw/MenuBar.swift#L273)

- **Native node connects with resolved gateway config:** `MacNodeModeCoordinator` calls `GatewayEndpointStore.shared.requireConfig()` and then connects the node session with that resolved URL/token/password, advertising the companion app node capabilities directly.
  - file: [apps/macos/Sources/OpenClaw/NodeMode/MacNodeModeCoordinator.swift:62](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/apps/macos/Sources/OpenClaw/NodeMode/MacNodeModeCoordinator.swift#L62)

## Close Comment

Codex review: current `main` already implements this well enough.

- `apps/macos/Sources/OpenClaw/MenuBar.swift:273` starts `MacNodeModeCoordinator` at app launch, so the companion app now owns its node session directly instead of depending on the old `openclaw-node` subprocess behavior described here.
- `apps/macos/Sources/OpenClaw/NodeMode/MacNodeModeCoordinator.swift:62` resolves the gateway config through `GatewayEndpointStore` and connects the mac node session with that URL, token, and password.
- `apps/macos/Sources/OpenClaw/GatewayEndpointStore.swift:309` uses `gateway.remote.url` for remote `direct` transport and only falls back to `127.0.0.1` for the SSH-tunnel transport.
- The same implementation is already in release `v2026.4.23`; the relevant macOS node-mode files are unchanged between fixing commit `d38ed0831d7372c8505dc944aea69b5b83da1435` and release commit `9b18ae877018a34933e27bb7b2986cb97e9602aa`.
