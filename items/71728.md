## Summary

Keep #71728 open. The RFC is fresh, coherent, and asks for a core trust contract: claim-level confidence/source metadata plus autonomous completion reporting for unverified claims. Current main has useful adjacent truthfulness mechanisms, but not the requested structured output or protocol surface.

## Best Possible Solution

Keep the RFC open for maintainer product/design judgment. A credible path is an additive core/protocol/SDK-visible trust signal for agent output, with optional claim-level confidence/evidence metadata where available, wording guidance for known vs inferred claims, and autonomous completion-report fields for unverified claims. Reuse memory citations and memory-wiki claim metadata as inputs, but do not treat them as the whole solution.
