## Summary

Keep open. Current main still does not implement the PR’s central behavior: the Control UI chat model picker preserves catalog order for concrete options, has no browser-local recent-model/MRU storage, and does not rank custom-provider models ahead of built-ins. The PR is an open, unmerged contributor feature that remains plausibly useful, though it now needs to be rebased onto the refactored `chat-model-select-state.ts` / `chat/session-controls.ts` code and should address the prior review concerns around safe storage and bundled-provider classification.

## Best Possible Solution

Keep the PR open, ask for a rebase onto the current Control UI picker architecture, and implement the feature in `ui/src/ui/chat-model-select-state.ts` plus `ui/src/ui/chat/session-controls.ts`: preserve the default option at the top, rank available concrete models by safe browser-local MRU, then true custom providers, then existing catalog order, persist only successful explicit concrete selections, ignore stale recents, and add focused tests for MRU ordering, custom-provider classification, stale entries, default clearing, and storage failures.
