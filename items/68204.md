## Summary

Keep open. Current main has several observability pieces, but they are still separate: agent events, trajectory bundles, cache trace, ACP relay logs, and task records each use their own shape. I did not find a canonical run trace schema adopted across agent, ACP, subagent, and task-flow paths. Related #43145 is useful prior art, but it is open/unmerged and narrower than this issue's ACP/task-flow acceptance criteria.

## Best Possible Solution

Keep this issue open as an architecture/observability tracking item. The best path is to define a small versioned run-trace contract, explicitly decide how it relates to existing trajectory and agent-event surfaces, then adapt agent events, ACP parent-child relay, subagent lifecycle, task records, and TaskFlow state changes into that schema with focused tests. Treat #43145 as related prior art, not as shipped coverage.
