## Close Comment

Closing from a Codex review. The logged failure here comes from the plugin loader only when it cannot resolve the plugin runtime module (`src/plugins/loader.ts:1912-1916`), and production resolution specifically expects `dist/plugins/runtime/index.js` (`src/plugins/sdk-alias.ts:647-669`). Current main explicitly builds that artifact (`tsdown.config.ts:229`) and has a guard test that locks it into the unified dist graph (`src/infra/tsdown-config.test.ts:48-70`). The same packaging entry is already present in the shipped `v2026.4.22` release commit (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). On that basis this specific `Unable to resolve plugin runtime module` report is already implemented on main/release. If you can still reproduce it on `v2026.4.22` or newer, please open a fresh issue with the exact install path and whether the packaged `dist/plugins/runtime/index.js` file is present.
