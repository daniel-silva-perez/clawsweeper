## Summary

Keep #69494 open. Current main still resolves the macOS browser-proxy local gateway auth path as plain strings only, so SecretRef dictionaries for `gateway.auth.token` or `gateway.auth.password` still drop to nil before `MacNodeBrowserProxy` can send auth headers. The PR is an active, focused fix for a documented SecretRef surface, and maintainer discussion confirms the bug while requesting merge-shape/test tightening rather than closure.

## Best Possible Solution

Keep #69494 open and review/land it, or an equivalent patch, after final macOS validation. The best implementation path is to add an app-side SecretRef resolver seam for the local `gateway.auth.token` and `gateway.auth.password` path used by `GatewayEndpointStore.localConfig()`, keep remote auth behavior unchanged unless explicitly product-approved, avoid blocking actor/main endpoint refresh on keychain lookup, and retain regression coverage for the actual browser-proxy `localConfig()` path.
