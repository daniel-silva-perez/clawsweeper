## Summary

Keep this issue open. The report is recent and concrete, and current main still has only generic WhatsApp reconnect/watchdog handling; I did not find a group-specific recovery path, Baileys group reliability hooks, or regression coverage for repeated 408 reconnect cycles where DMs continue but group upserts stop.

## Best Possible Solution

Keep the issue open as a WhatsApp plugin reliability bug. Add a focused regression harness for repeated close/reopen cycles with both DM and group traffic, then implement group-subscription or Baileys group reliability recovery in `extensions/whatsapp`: likely socket-level `cachedGroupMetadata`/`getMessage` support, maintained group metadata event hooks, and/or a group-specific liveness watchdog that can recover when DMs continue but group upserts stop.
