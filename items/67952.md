## Summary

Keep this PR open. Current main still implements `ltm list` as a count-only command that duplicates `ltm stats`, so the PR addresses a real remaining memory-lancedb CLI behavior gap rather than obsolete or already-shipped work. The review comments are also useful: the supplied latest patch context appears to address the DB-level limit concern, but the PR still needs normal maintainer review and focused regression tests before merge.

## Best Possible Solution

Keep the PR open or replace it with an equivalent maintainer patch: add a bounded `MemoryDB.list` path that returns non-vector memory records, make `ltm list` print entries as JSON with validated `--limit` and optional newest-first ordering, keep `ltm stats` as the count command, and cover the CLI behavior with focused memory-lancedb tests before merge.
