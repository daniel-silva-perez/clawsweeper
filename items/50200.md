## Summary

Keep #50200 open. Current main still lacks the inbound Slack media fallback this PR proposes: `resolveSlackMedia` only uses `url_private_download ?? url_private` from the event payload and returns `null` when both are missing, while the Slack message pipeline does not pass a `WebClient` that could call `files.info`. The existing `download-file` action already uses `files.info`, so the PR is a plausible focused fix for the linked open bug #50129 rather than obsolete cleanup.

## Best Possible Solution

Keep #50200 open and review it as the active implementation for #50129. The best path is to finish the inbound Slack media `files.info` fallback in the Slack plugin, pass the Slack `WebClient` through the direct-message and thread-starter media paths, add regression coverage for missing private URLs, and address the existing review comments before merge.
