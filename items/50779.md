## Summary

Keep open. I did not find strong evidence that current `main` fixes this specific Control UI bug. The current UI image extractor still ignores direct tool-result image blocks shaped like `{ type: "image", data, mimeType }`, while the current `read` tool path and tests still use that shape alongside the visible `Read image file [image/png]` text that the report describes.

## Evidence

- **UI extractor only handles `image.source` or `image.url`:** `extractImages()` renders `type: "image"` blocks only when they carry `source.data` or `url`; it does not read direct `data`/`mimeType` fields from tool-result image blocks.
  - file: [ui/src/ui/chat/grouped-render.ts:143](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/ui/src/ui/chat/grouped-render.ts#L143)
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Current read-tool image result shape is still direct base64 image data:** `normalizeReadImageResult()` looks for image blocks with `type === "image"`, `data`, and `mimeType`, which matches the shape the UI extractor above does not consume.
  - file: [src/agents/pi-tools.read.ts:292](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/agents/pi-tools.read.ts#L292)
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Current tests still model the reported read-tool output text:** The media extraction tests still use a read-tool-like result containing `Read image file [image/png]` plus `{ type: "image", data, mimeType }`, which matches the issue's observed sidebar text.
  - file: [src/agents/pi-embedded-subscribe.tools.media.test.ts:98](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/agents/pi-embedded-subscribe.tools.media.test.ts#L98)
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **UI tests cover other image shapes, not the direct tool-result image block shape:** The Control UI render tests exercise `input_image` and `image.url` cases, but I did not find a UI test covering a tool-result `{ type: "image", data, mimeType }` block rendering inline.
  - file: [ui/src/ui/chat/grouped-render.test.ts:668](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/ui/src/ui/chat/grouped-render.test.ts#L668)
  - command: `rg -n "type: \"image\"|input_image|image_url" ui/src/ui/chat/grouped-render.test.ts`
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
