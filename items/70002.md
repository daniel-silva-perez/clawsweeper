## Summary

Keep #70002 open. Current main still has both docs workflows running in forks without a repository guard, and both jobs depend on `OPENCLAW_DOCS_SYNC_TOKEN` for cross-repo docs publishing/dispatch work. The PR is a narrow 2-line CI fix that directly addresses the remaining fork-failure path rather than an obsolete or duplicate change.

## Best Possible Solution

Keep the PR open for maintainer review and, if accepted, land the job-level `github.repository == 'openclaw/openclaw'` guard in both docs workflows so fork pushes skip upstream-only docs publishing while upstream `openclaw/openclaw` behavior remains unchanged.
