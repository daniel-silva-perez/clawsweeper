## Summary

Keep #69608 open. Current main still lacks the PR's shared daily-memory/session-summary model and several of the reported behaviors remain visible in code: reset prompts still ask the model to reread startup files, session-memory summaries are written as ordinary UTC-dated notes without a sentinel/provenance index, startup context loads dated slug files without session-summary filtering, and dreaming/doctor/REM paths still use canonical-only daily-file matching. The PR is broad and needs maintainer review, but it is a plausible, recent bug-fix PR with concrete reproduction details and tests rather than cleanup-ready noise.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to narrow or split it into mergeable pieces: first fix runtime reset/startup prompt and missing-bootstrap placeholder behavior, then introduce the shared daily-memory/session-summary helpers with focused tests, then migrate memory-core dreaming/doctor/REM/short-term callers onto that shared model. If maintainers decide the XL refactor is too broad, ask for a smaller PR preserving the same regression tests and reproduction evidence.
