## Summary

Current main already fixes the reported send-vs-poll misclassification by treating zero-valued poll duration defaults as unset, and it includes a regression test for that exact `message send` case.

## Evidence

- **Zero-valued poll durations no longer count as poll intent:** `hasPollCreationParams()` only treats numeric poll params as present when they are finite and non-zero, so `pollDurationHours: 0` and `pollDurationSeconds: 0` no longer trigger poll classification.
