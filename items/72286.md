## Summary

Keep this PR open. Current main still only supports Control UI Talk through OpenAI browser WebRTC sessions, while Google Live is limited to backend realtime bridges. The PR is an active draft implementation candidate for real missing behavior across the Google provider, gateway protocol, and browser UI, not stale cleanup or a ClawHub-only request.

## Best Possible Solution

Keep this PR open as the implementation candidate. Maintainers should review whether Google Live browser Talk belongs in the bundled Google provider and Control UI, then either land a polished provider/gateway/UI/docs/test change or redirect it into a narrower generic browser realtime transport API design.
