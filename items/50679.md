## Summary

Current main no longer exposes a send composer outside the Chat tab, so the reported flow of sending from Overview/Sessions and remaining on that tab is obsolete. The only `onSend` wiring lives inside the `state.tab === "chat"` render branch, and other session navigation entry points already switch back to Chat.

## Close Comment

Closing from a Codex review because this does not reproduce on current `main`. The current Control UI only renders the chat composer when `state.tab === "chat"` and wires `onSend` only inside that branch (`ui/src/ui/app-render.ts:2209`, `ui/src/ui/app-render.ts:2262`), so the reported flow of staying on Overview/Sessions while sending from a bottom chat input no longer exists. Session-navigation entry points that open a conversation also already switch back to Chat (`ui/src/ui/app-render.ts:1641`, `ui/src/ui/app-render.ts:1763`). If you still see a send path outside the Chat tab on a newer build, please open a new issue with the exact surface and a screenshot or repro video.
