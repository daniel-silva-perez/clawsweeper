## Summary

Keep open. Current main still does not implement the proposed instruction-file write guard: write/edit tools are created without a protected-basename wrapper, sandbox write/edit paths are likewise unguarded, and apply_patch mutates parsed add/update/delete/move hunks without checking SOUL.md, MEMORY.md, IDENTITY.md, CLAUDE.md, TOOLS.md, BOOT.md, or TASKS.md. The PR is a plausible security-hardening candidate, the provided review discussion shows useful follow-up commits for apply_patch, path aliases, and symlinks, and the related #58684 was closed unmerged and covered a narrower non-main memory-write policy.

## Best Possible Solution

Keep this PR open for maintainer review or replacement with an equivalent core implementation. The best path is to decide the approval/product boundary for agent edits of bootstrap/instruction files, then implement a central mutation guard covering write, edit, and apply_patch in host and sandbox modes, with regression tests for path normalization, canonical targets, aliases, symlinks, and allowed non-protected memory files.
