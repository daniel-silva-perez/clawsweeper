## Summary

Keep this issue open. Current main still has a credible Control UI blank-screen path matching the discussion: cron jobs loaded from persisted state can lack a valid payload, and the global Control UI render path plus cron render/edit paths still dereference `job.payload` without guards.

## Best Possible Solution

Keep this issue open. The maintainer path is to reproduce on current main with the reporter's snapshot or a synthetic stale/malformed cron row, then add a null-safe guard or normalization/filtering layer before Control UI render/edit paths, plus a focused jsdom or controller regression test. If persisted cron shape is the root cause, repair or filter malformed rows in cron service or doctor code while keeping the Control UI resilient to bad snapshots.
