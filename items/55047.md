## Summary

Keep open. Current main does not provide enough evidence for any conservative close reason: the report is a recent core Control UI regression, the related PR #54552 was closed unmerged, and current main still has an unguarded cron payload dereference in the shared render path that could plausibly blank tab content before the lazy tab view renders.

## Best Possible Solution

Keep the issue open. A maintainer should reproduce with browser console errors after an update, then either revive/land the #54552-style defensive render fix with a regression test or identify the actual lazy-load/stale-asset failure path. Ask the reporter for console logs, gateway logs, screenshot, current OpenClaw version, and whether `openclaw doctor --fix` or clearing Control UI browser cache changes the behavior.
