## Summary

Keep open. Current `main` has adjacent improvements for long-running turns, but it still does not implement a user-visible paused/partial-progress fallback for chat surfaces after tool activity stops. The runtime tracks liveness internally, yet that state is not wired into Telegram-style reply delivery.

## Evidence

- **Reply options expose tool/progress hooks, not lifecycle-to-chat delivery:** `GetReplyOptions` defines `onToolResult`, `onToolStart`, `onPlanUpdate`, `onApprovalEvent`, and `onPatchSummary`, but there is no callback for lifecycle/liveness state that a channel dispatcher could use to emit `paused` or `still working` messages automatically.
  - file: [src/auto-reply/get-reply-options.types.ts:58](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/auto-reply/get-reply-options.types.ts#L58)
  - command: `nl -ba src/auto-reply/get-reply-options.types.ts | sed -n '58,90p'`
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Dispatcher only forwards explicit tool/plan/approval/patch payloads:** `dispatch-from-config` can send tool summaries and a limited `Working:` status, but only when an explicit tool/plan/approval/patch event is emitted. There is no silence timer, inactivity threshold, or lifecycle-driven fallback that would automatically tell the user the run is paused or still assembling results after tool activity stops.
  - file: [src/auto-reply/reply/dispatch-from-config.ts:784](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/auto-reply/reply/dispatch-from-config.ts#L784)

  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Paused/abandoned liveness is still internal:** The embedded subscribe lifecycle handler emits `livenessState` such as `paused`/`abandoned` on lifecycle end, but the visible non-test consumer in this review is the web UI tool stream. I did not find corresponding channel reply plumbing that would turn those lifecycle states into Telegram messages.
