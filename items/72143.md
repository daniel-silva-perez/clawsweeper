## Summary

Keep #72143 open. Current main still contains the user-facing fallback prefix, applies it when the session follow-up path throws, and has tests asserting that behavior. The same code path calls `callGatewayTool` once and immediately falls back on failure, so the requested retry/backoff and internal-only resume-failure logging are not implemented on main. Related PR #72148 is open and explicitly only covers the prefix removal, leaving the retry/backoff follow-up out of scope.

## Best Possible Solution

Keep the issue open. The best fix is to update `src/agents/bash-tools.exec-approval-followup.ts` so the session resume/follow-up call is retried with bounded short backoff, direct fallback preserves only the command result/status without the internal prefix, and resume failures are logged internally with approval/session/target/error metadata. Update the existing follow-up tests to assert prefix-free fallback content, retry attempts, and internal logging. If PR #72148 lands first, either extend it or open a focused follow-up for retry/backoff before closing #72143.
