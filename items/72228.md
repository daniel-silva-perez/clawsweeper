## Summary

Keep #72228 open. Current main still has a plausible core runtime dependency gap: core logging imports `tslog`, bundled plugin external runtime roots mirror core `dist` files into `plugin-runtime-deps`, and the automatic installer/verifier only derives missing specs from bundled plugin `package.json` dependencies, not root/core runtime dependencies such as `tslog`. The issue is recent, unlabeled, non-maintainer-authored, and actionable in repo code/tests.

## Best Possible Solution

Keep the issue open and fix the packaged external runtime root contract. The likely durable fix is to make mirrored core dist files resolve core runtime dependencies from the OpenClaw package root or include and verify required root/core runtime deps such as `tslog` in the external runtime install root before any mirrored core module can be imported. Add a focused regression test that simulates an external `plugin-runtime-deps` root with missing `tslog` and verifies startup/loader behavior surfaces a repairable error or reinstalls the dependency instead of crashing.
