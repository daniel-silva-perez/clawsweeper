## Summary

Keep this issue open. Current main still lacks client-provided tool passthrough for `/v1/chat/completions`, and PR #66278 is an open, unmerged implementation candidate that explicitly links this issue as a closing target.

## Best Possible Solution

Keep this issue open and treat PR #66278 as the active implementation candidate. The maintainer path is to review, revise, and land that PR or an equivalent gateway fix that parses Chat Completions `tools` and `tool_choice`, forwards client tools into agent runs, returns OpenAI-compatible `tool_calls` in non-stream and streaming responses, accepts `role: "tool"` follow-up messages with `tool_call_id`, and adds focused contract tests.
