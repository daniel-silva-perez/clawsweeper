## Summary

Keep open. Current `main` does not implement client-tool passthrough on `/v1/chat/completions`. The Chat Completions handler only accepts `model`/`stream`/`messages`/`user`, builds an agent run without `clientTools`, and always serializes a plain assistant message with `finish_reason: "stop"`. By contrast, `/v1/responses` already has explicit client-tool extraction, `tool_choice` handling, runtime passthrough, and `function_call` output support.
