## Summary

Keep open. Current main has moved part of this area to generic conversation-binding plumbing and Feishu already has some command-context resolution, but the PR is not fully obsolete: Feishu still does not advertise full current-conversation binding/lifecycle support through the plugin contract, Feishu config does not expose `threadBindings`, and current Feishu reply dispatch still suppresses non-streaming `block` replies that this PR was trying to deliver for ACP.

## Best Possible Solution

Keep the PR open for maintainer/product review, but require a rebase onto current main and narrow it to the remaining gaps: Feishu should either fully advertise and implement current-conversation binding through `conversationBindings` (`supportsCurrentConversationBinding`, manager/lifecycle hooks, config/docs/tests) or explicitly stay outside that generic contract; ACP block delivery should be fixed with Feishu-specific regression tests that prove ACP-visible block output reaches the user without exposing internal blocks. Drop obsolete hard-coded command-gating changes already covered by the generic resolver.
