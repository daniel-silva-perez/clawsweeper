## Summary

Current main still matches the reported mismatch. The macOS app advertises the screen capability and declares/implements `screen.record`, but the gateway classifies `screen.record` as a dangerous node command and excludes it from the macOS default allowlist unless `gateway.nodes.allowCommands` explicitly opts in. The linked PR #57192 is open and unmerged, so it is related active work rather than a shipped fix.

## Best Possible Solution

Keep the issue open for maintainer/product resolution. If macOS `screen.record` should be callable by default as the issue requests, update `src/gateway/node-command-policy.ts`, wizard denylist seeding, audit/tests, and docs so macOS is allowed end-to-end without enabling other dangerous commands unintentionally. If maintainers want `screen.record` to remain opt-in, update the macOS/nodes docs and CLI/status/error guidance to make the required `gateway.nodes.allowCommands: ["screen.record"]` opt-in explicit.
