## Summary

Current `main` has Feishu card transport and callback plumbing, but it does not implement generic Feishu inline/interactive buttons for outbound message presentation the way the issue requests. The feature request is still materially open.

## Evidence

- **Feishu presentation renderer degrades everything to markdown text:** `buildFeishuPresentationCard(...)` creates a card body with a single `markdown` element from `renderMessagePresentationFallbackText(...)`; it does not emit Feishu `action` blocks or button elements.

There is no branch that converts shared `buttons` blocks into native Feishu interactive controls.

## Risks / Open Questions

- Docs currently describe Feishu as rendering to interactive cards, which may overread as generic button support even though `buttons`/`select` presentation blocks still degrade to text fallback.
- There is partial Feishu card-action infrastructure for launcher/approval flows, so future cleanup should distinguish between existing specialized card UX and the still-missing generic inline button feature.
