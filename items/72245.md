## Summary

Keep PR #72245 open. Current main still lacks the PR's resolved provider-hook plugin cache and provider-ref cache-key normalization, the related pricing hot path remains present, and the PR is a focused core runtime performance hardening change rather than cleanup noise, a ClawHub item, or a duplicate of shipped work.

## Best Possible Solution

Keep #72245 open for maintainer review. The best path is either to land this narrow provider-hook runtime cache hardening after targeted validation, or supersede it with an equivalent maintainer patch that normalizes provider refs in hook cache keys, caches resolved hook plugin/null results, and preserves the existing cache reset behavior.
