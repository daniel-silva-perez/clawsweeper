## Summary

Keep PR #66168 open. Current main does not implement the requested MCP Apps gateway surface, and the PR is not cleanly closable as ClawHub/plugin scope because the requested first-class `mcp.*` WS methods, resource registry, and tool `_meta.ui` contract touch core gateway/protocol/tool-resolution behavior. The PR also has design/security review questions that need maintainer judgment rather than automated cleanup.

## Best Possible Solution

Keep the PR open for maintainer product/API review. The best path is either to revise it into a narrowly documented, additive core MCP Apps gateway contract with tests for scoping and policy, or to first add a supported plugin/API seam that lets a ClawHub plugin implement the feature without reaching into core internals.
