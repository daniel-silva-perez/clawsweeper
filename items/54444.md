## Summary

Requesty is not implemented on current main, but the repo's current scope and docs route third-party/OpenAI-compatible providers to the external plugin path rather than bundling them into core. This request fits the ClawHub/community provider-plugin lane, not OpenClaw core cleanup.

## Evidence

- **Vision keeps optional capability out of core by default:** `VISION.md` says optional capability should usually ship as plugins, plugin authors should host and maintain plugins in their own repos, and the bar for adding optional plugins to core is intentionally high.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/VISION.md#L54)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Community plugins explicitly include providers and use ClawHub/npm distribution:** `docs/plugins/community.md` defines community plugins as packages that can add providers, says ClawHub is the canonical discovery surface, and points users to `openclaw plugins install` instead of adding them to the main repo.
  - file: [docs/plugins/community.md:9](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/docs/plugins/community.md#L9)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Plugin guide says you do not need to add a provider plugin to this repo:** `docs/plugins/building-plugins.md` says plugins can add providers and explicitly says you do not need to add your plugin to the OpenClaw repository; publish to ClawHub or npm instead.
  - file: [docs/plugins/building-plugins.md:11](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/docs/plugins/building-plugins.md#L11)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Provider-plugin guide is the documented path for OpenAI-compatible proxies:** `docs/plugins/sdk-provider-plugins.md` is the guide for adding an OpenAI-compatible proxy or custom LLM to OpenClaw, and it documents ClawHub publishing requirements for external provider plugins.
  - file: [docs/plugins/sdk-provider-plugins.md:5](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/docs/plugins/sdk-provider-plugins.md#L5)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **No Requesty implementation exists on current main:** Searches on the checked-out repo found no `Requesty`/`requesty` references and no `extensions/requesty` package, so this is not already bundled on `main`.
  - command: `rg -n "Requesty|requesty" docs src extensions .github && rg --files extensions | rg 'requesty'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)

## Risks / Open Questions

- Maintainer policy could still choose to bundle Requesty later if it becomes a high-demand/core provider, but current repo guidance points the other way.
- The linked draft PR was not reviewed here; GitHub CLI access was unavailable without `GH_TOKEN`, so this decision is based on current main scope/policy evidence rather than PR implementation details.

## Close Comment

Closing after Codex review. Current `main` points this to the community plugin path rather than a new bundled core provider: `VISION.md:54` says optional capability should usually ship as plugins and that the bar for adding optional plugins to core is intentionally high; `docs/plugins/community.md:9` says community plugins can add providers and should be published on ClawHub/npm; `docs/plugins/building-plugins.md:16` explicitly says you do not need to add your plugin to the OpenClaw repo; and `docs/plugins/sdk-provider-plugins.md:5` is the documented path for adding an OpenAI-compatible proxy/custom LLM as a provider plugin. I also checked current `main` and there is no existing bundled Requesty implementation (`rg -n "Requesty|requesty" docs src extensions .github`, `rg --files extensions | rg requesty`). Given that scope, this is better shipped as a ClawHub/community provider plugin instead of a bundled OpenClaw provider.
