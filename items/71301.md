## Summary

Keep this issue open. Current main has useful partial work: authored docs are included in the npm package, agents are pointed at the local docs directory when it exists, and `config.schema.lookup` provides field-level config docs. The central request remains missing: there is no native local-first docs retrieval surface such as `docs.search`, `docs.get`, docs API lookup, or changed-since tooling for agents, and the CLI docs command still queries the live docs MCP endpoint.

## Best Possible Solution

Keep this issue open and narrow it to the missing core capability: ship a release/package docs manifest plus local search/get index over bundled docs, expose a local-first docs retrieval surface to agents, keep live Mintlify search as an explicit fallback, and add tests for package-installed docs discovery plus agent/tool behavior. A ClawHub or bundled skill can be part of the implementation only if it still preserves version-coupled packaged docs and a reliable agent-facing retrieval path.
