## Summary

Keep #71301 open. Current main has partial groundwork: docs are packaged, agents are pointed at a local docs path when available, and config schema lookup exists. The central request is broader, though: first-class, version-coupled docs retrieval for agents. Current main still only has a live `openclaw docs` CLI search path and no native local `docs.search`/`docs.get` style agent tool or bundled search/index layer.

## Best Possible Solution

Keep the issue open and scope an incremental core implementation: generate a release-time docs manifest/search index from the authored docs, ship it with the package, add an agent-accessible local-first retrieval surface such as `docs.search` and `docs.get`, keep `config.schema.lookup` as the schema-specific path, and use live docs only as an explicit latest/fallback source.
