## Summary

Keep open. Current main still has trusted session and agent context at the exec tool boundary, but spawned exec environments only receive the existing `OPENCLAW_SHELL=exec` marker and the shared `OPENCLAW_CLI=1` marker. The provided related PR #68859 is still open and, by its own description, only covers `OPENCLAW_SESSION_KEY`, while this issue also requests `OPENCLAW_AGENT_ID` and a documented/tested child-process env contract.

## Best Possible Solution

Keep this issue open for a focused core exec-runtime change. Add a shared helper that injects trusted `OPENCLAW_SESSION_KEY` and `OPENCLAW_AGENT_ID` after env sanitization/sandbox env construction, applies it consistently to gateway, PTY, sandbox/Docker/backend, and node-host exec paths, documents the runtime markers, and adds regression tests for precedence and stale inherited values. Review #68859 as a possible starting point, but broaden it before considering the issue resolved.

- The related open PR #68859 appears to address only `OPENCLAW_SESSION_KEY`; the issue also asks for `OPENCLAW_AGENT_ID` and docs/tests for the new contract.
