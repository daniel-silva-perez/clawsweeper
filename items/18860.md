## Summary

The current main checkout still does not expose the requested plugin-facing resolved-tools hook or an equivalent pull API. The newer `tools.effective` operator/UI surface is not a substitute because it omits JSON Schema parameters and is not available through `OpenClawPluginApi`.

## Best Possible Solution

Keep PR #18860 open as the current concrete implementation/design context for a plugin-facing resolved-tools/schema discovery seam. The best path is a narrow maintainer-approved contract change in `src/plugins/hook-types.ts`, `src/plugins/hooks.ts`, `src/plugins/types.ts`, and `src/agents/pi-embedded-runner/run/attempt.ts`, plus focused contract tests and docs in `docs/plugins/hooks.md`; if maintainers prefer the pull-based `getResolvedTools()` design from #21330, designate a canonical follow-up before superseding this PR.
