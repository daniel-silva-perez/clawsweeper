## Summary

Keep open. Current `main` still auto-pins an auth profile on new sessions, persists that auto override back into session state, and explicitly does not prioritize `lastGood` when ordering profiles. That means the reporter's requested behavior ('new sessions should start with no `authProfileOverride` / use the default profile') is not implemented on current `main`, so there is not strong evidence for a conservative close.
