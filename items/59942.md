## Summary

Keep open. The issue is recent, concrete, and externally corroborated; current main evidence still shows the built-in memory-lancedb plugin registering tools, CLI, hooks, and a service but not a memory runtime/capability, and a related open issue/PR indicates this detection gap is still unresolved rather than already shipped.

## Evidence

- **Reporter provided a concrete reproduction and related plugin-side report:** Issue #59942 includes OpenClaw 2026.4.2, macOS arm64, Node 25.6.0, runtime logs showing memory-lancedb-pro registered, and contradictory `openclaw status` / `openclaw doctor --non-interactive` output. The linked plugin-side issue CortexReach/memory-lancedb-pro#472 contains the same report and is closed on the plugin side, so the OpenClaw-side mismatch remains relevant to triage.
- **Current main plugin registration does not show memory runtime/capability registration:** The current main view of `extensions/memory-lancedb/index.ts` shows `api.registerTool`, `api.registerCli`, lifecycle hook registration, and `api.registerService`, while searches of that file found no `registerMemoryRuntime` or `registerMemoryCapability`. That supports the reported hypothesis that doctor/status may be checking a different memory-runtime registration path than the plugin uses.
  - file: [extensions/memory-lancedb/index.ts:2253](https://github.com/openclaw/openclaw/blob/d6ef1fcf24eea3a5985d653a3baf400e32aa1b75/extensions/memory-lancedb/index.ts#L2253)
  - sha: [d6ef1fcf24ee](https://github.com/openclaw/openclaw/commit/d6ef1fcf24eea3a5985d653a3baf400e32aa1b75)
- **Related OpenClaw issue remains open with an unmerged fix PR:** Related issue #60177 describes the same symptom, labels it bug/regression, identifies the missing `api.registerMemoryRuntime(...)` path, and references PR #65066, reported as open and not merged, to register a runtime for doctor/status. This is evidence that maintainers have not yet shipped the cleanup on main.
- **SDK docs make runtime/capability registration the memory-plugin detection contract:** The public Plugin SDK overview lists `api.registerMemoryCapability(capability)` as the preferred exclusive memory-plugin API and `api.registerMemoryRuntime(runtime)` as the legacy-compatible memory runtime adapter, matching the suspected path used by doctor/status.
