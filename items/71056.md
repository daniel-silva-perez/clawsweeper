## Summary

Keep #71056 open. Current main has the lower-level guarded-fetch machinery for trusted env-proxy mode, but it does not expose or wire an operator-facing `tools.web.fetch.useEnvProxy` flag for `web_fetch`. The PR is a focused implementation for the linked open issue #71053, and the provided review thread shows the earlier SSRF gating concern was addressed in the PR head.

## Best Possible Solution

Keep the PR open and review it as the current implementation candidate for #71053. The ideal patch should add `tools.web.fetch.useEnvProxy` to config types, zod schema, generated schema/help/labels, include proxy mode in the web_fetch cache key, pass the flag into `fetchWithWebToolsNetworkGuard`, and rely on the existing `fetch-guard`/`shouldUseEnvHttpProxyForUrl` safety gate so ALL_PROXY-only and NO_PROXY-bypass paths stay DNS-pinned. If #58034 is still active, maintainers should choose one canonical PR and close the other as superseded after confirming which branch has the stronger tests and generated config alignment.
