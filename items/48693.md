## Summary

Current main already separates Feishu read-only inspection from runtime SecretRef resolution, so unresolved `channels.feishu.appSecret` no longer throws in doctor/status/plugin-read paths; the same implementation is present in the latest tagged release `v2026.4.22`.

## Close Comment

Closing as implemented on main after Codex review. The current Feishu channel code now keeps read-only/config inspection paths separate from runtime credential resolution: unresolved SecretRefs are treated as unavailable in inspect mode (`extensions/feishu/src/accounts.ts:70`, `extensions/feishu/src/accounts.ts:290`), while runtime startup still stays strict (`extensions/feishu/src/accounts.ts:305`, `extensions/feishu/src/channel.ts:1205`). Feishu status/security/message-tool discovery paths use the read-only resolver instead of eagerly materializing runtime credentials (`extensions/feishu/src/channel.ts:164`, `extensions/feishu/src/channel.ts:188`, `extensions/feishu/src/channel.ts:1182`), and regression coverage was added in `extensions/feishu/src/accounts.test.ts:366`. The latest tagged release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`) already contains the same Feishu code as current `main` for these paths. If someone still sees this on a newer build, please open a fresh issue with the exact config shape and command path that is still resolving during registration.
