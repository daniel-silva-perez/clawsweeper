## Summary

Keep open. Current main still applies the nested skills-root redirect before checking whether the configured root already contains loadable skills, so the user-visible bug described by this PR remains present. The PR is a focused implementation candidate with a targeted regression test and should receive normal maintainer review rather than automated closure.

## Best Possible Solution

Keep this PR open and review or land a version of the fix that verifies the original configured directory is a loadable skills root before redirecting to its nested `skills` directory. Preserve the regression test for the root-plus-nested layout and align the early-exit scan with the loader’s candidate limits and security checks.

## Evidence

An empty decoy `SKILL.md` under the nested directory is enough to trigger the redirect.
