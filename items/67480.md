## Summary

Keep open. Current main still applies the nested `dir/skills` redirect before checking whether the original directory is already a valid skills root, so the user-visible bug described by this PR still exists in the checked-out main code. The PR is a plausible focused implementation candidate with a targeted regression test, but it still needs normal maintainer review rather than automated closure.

## Best Possible Solution

Keep this PR open and review or land a version of the fix that verifies the original directory is a loadable skills root before redirecting to `dir/skills`, with the provided regression test preserved and any candidate-limit edge case resolved during review.

## Evidence

- **Current main still redirects unconditionally to nested skills root:** At current main, `resolveNestedSkillsRoot()` checks for `dir/skills/*/SKILL.md` and returns `baseDir: nested` without first checking whether `dir/*/SKILL.md` already contains valid root-level skills. In the reported layout, that selects `~/.openclaw/skills/skills` and bypasses sibling skills under `~/.openclaw/skills/<name>`.
