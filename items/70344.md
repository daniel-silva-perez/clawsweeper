## Summary

Keep #70344 open. Current main only suppresses assistant-visible output when the assistant phase resolves to `commentary`; it does not cover unphased `stopReason: "toolUse"` or tool-call content blocks. `sessions_send` also still starts the A2A announce flow and current inter-session route resolution preserves existing external routes for every inter-session turn, with no `sourceTool: "sessions_send"` current-turn suppression. The PR is relevant, but review feedback flags correctness issues, so it should be revised rather than closed as obsolete.

## Best Possible Solution

Keep the PR open but require a narrower revision: suppress unphased tool-use/tool-call prelude text at the delivery boundary, make `sessions_send` current-turn delivery session-local without mutating the target session's persisted external route, remove the A2A announce behavior only where product intent confirms it, and add focused regressions for each boundary. Treat #70372 as related follow-up for Anthropic/Bedrock thinking narration, not a full superseder because it does not cover `sessions_send`.
