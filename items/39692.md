## Summary

Keep PR #39692 open. Current main has the Control UI i18n framework, zh-CN locale loading, and some localized labels, but substantial hardcoded English remains in the exact view surfaces this PR targets: Agents, agent status/files, Logs, channel config/form helpers, channel cards, Debug, and exec approvals. The PR is stale against the generated-locale workflow, but the underlying work is still real and not fully superseded by the related shell-label PR or the already-closed base i18n issue.

## Best Possible Solution

Keep the PR open or ask for a focused rebase. The best implementation is to move the remaining hardcoded Control UI view strings into `ui/src/i18n/locales/en.ts`, regenerate `zh-CN` with `pnpm ui:i18n:sync`, preserve the current UX details called out in review (`openclaw security audit --deep` monospace, `Cron Jobs`, and `never` semantics), and add targeted i18n coverage for the affected views or locale-key parity.
