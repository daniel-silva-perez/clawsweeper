## Summary

Keep open. Current main still sends Signal inbound message text to the reply pipeline as raw `BodyForAgent`/`RawBody`, so the requested hardening is not implemented. This PR is also not safe to merge as-is because it wraps only the legacy envelope `Body` path while the model-facing prompt path prefers `BodyForAgent`; the existing review comments correctly identify that bypass and additional history-rendering concerns.

## Best Possible Solution

Keep the PR open but require a rework before merge. The fix should protect the actual Signal model-facing path, either by deriving `BodyForAgent` from wrapped untrusted user text or by using structured `UntrustedContext` consistently, while leaving command/raw parsing semantics deliberate. Group-history handling should wrap user text once or use stable marker ids, keep application-generated message ids outside the untrusted block, and add focused Signal tests for direct messages, group messages, debounced bodies, quote-only context, and pending group history.
