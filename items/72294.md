## Risks / Open Questions

- The PR may drop distinct late final text while still delivering media, which can separate attachments from captions, warnings, or context unless the reply pipeline guarantees the streamed card already contains the full safe text.
- The added suppression state is mutable per dispatcher and interacts with async `onError` / `onIdle` callbacks that the shared dispatcher does not await; a reply-token or synchronized close path may be needed to avoid cross-reply state leakage.
- Validation was not executed because this was a read-only cleanup review; CI or Testbox should verify the targeted Feishu test and changed gate before merge.
