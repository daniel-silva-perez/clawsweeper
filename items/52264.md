## Summary

Keep open. Current main still does not implement the PR's startup hardening: relative OPENCLAW_STATE_DIR overrides are resolved on demand through path.resolve(), and neither gateway startup entrypoint normalizes explicit state-dir overrides before config/plugin startup work. The related #52250 root cause appears to have separate gateway-bindable registry handling on main, but this PR's explicit state-dir pinning remains unshipped work rather than cleanup-close material.

## Best Possible Solution

Keep the PR open, but ask for a focused rebase: add a narrow startup normalization helper for explicit state-dir overrides, consider whether OPENCLAW_CONFIG_PATH should be normalized in the same startup pass, call it before any config/path reads in both gateway CLI and direct server startup, and keep focused cwd-pinning tests without unrelated generated baseline churn.
