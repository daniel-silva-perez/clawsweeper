## Summary

Keep open. Current main still has the reported recovery notice behavior and does not expose `gateway.systemEvents.recoveryNotice` or any equivalent suppression mode. The related items in the provided context do not supersede it: #71289 narrowed whole-file recovery triggers, #49579 is an open WhatsApp-specific banner filter, and #39307 asks for a different/broader user-visible reset signal.

## Best Possible Solution

Keep #72043 open as core Gateway/config work. The likely implementation belongs in `src/config/types.gateway.ts`, `src/config/zod-schema.ts`, schema help/generated docs, and the recovery/restart-sentinel paths: add a typed `gateway.systemEvents.recoveryNotice` mode, gate prompt enqueue and outbound delivery for recovery notices while preserving restart continuations, and cover all modes in focused gateway tests. Coordinate defaults with #39307 so internal/operator agents still get safe recovery context unless configured otherwise.
