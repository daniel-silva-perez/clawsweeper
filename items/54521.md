## Summary

Keep open. Current `main` fixes the Linux/systemd half of the report and the `entry.js` vs `index.js` drift, but the issue is not fully resolved because the macOS LaunchAgent path still writes service environment values inline and the doctor audit still treats non-file-backed `OPENCLAW_GATEWAY_TOKEN` as embedded.

## Evidence

- **Systemd install now moves matching `.env` secrets behind `EnvironmentFile=`:** `writeSystemdUnit` reads state-dir `.env` vars, writes them to a separate gateway env file, removes matching inline entries, and passes `environmentFiles` into `buildSystemdUnit` instead of inlining those values in the main unit.

  - file: [src/daemon/systemd.test.ts:682](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/daemon/systemd.test.ts#L682)
