## Summary

Keep open. Current main still implements the reported broad sibling-home scan: Doctor enumerates `/home` on Linux and `/Users` on macOS, skips only dot-prefixed entries, probes each sibling `.openclaw` directory, and prints any matches in the multiple-state-dir warning. I found no current-user scoping, backup-name filtering, redaction of non-current-home paths, or env/config opt-out in the current source or focused tests. The related closed report #68752 covers the same noisy multi-user-host behavior, but it is closed and does not supersede this still-reproducible issue.

## Best Possible Solution

Keep this issue open and patch the Doctor state integrity check in `src/commands/doctor-state-integrity.ts`. A conservative fix should constrain discovery to the current effective home/service user or remove broad sibling-home enumeration, add an explicit config/env suppression if maintainers want the diagnostic to remain optional, avoid printing non-current-user state paths verbatim, and add focused tests for `/home/alice`/`/home/bob` plus backup-sibling directory cases. Update Doctor docs if the scan semantics change.
