## Summary

Keep PR #52012 open. Current main does not implement `agents.defaults.compaction.modelFallbacks` or an equivalent compaction-specific model fallback path. The general agent model fallback machinery exists, but the compaction runtime still resolves one compaction target, fails immediately on model/auth resolution errors, and only retries the same model with a fallback thinking level for reasoning-mode rejections. The PR is also not merge-ready as-is based on the provided review discussion, but that is a request-changes path rather than an auto-close reason.

## Best Possible Solution

Keep #52012 open, but require a tighter implementation before merge: add `agents.defaults.compaction.modelFallbacks` to the public type, strict schema, generated/help metadata, and docs; wire the direct and queued compaction paths to try configured fallback refs across resolution, auth, and runtime compaction failures without regressing existing prompt/runtime/checkpoint behavior; and add focused tests proving no behavior change when unset plus fallback success/failure cases. If maintainers prefer a smaller patch, supersede this PR with a narrower core implementation and close this PR only after that lands.
