## Summary

Keep open. Current main has not implemented the PR's requested diagnostic split: bundled channel entry validation still collapses malformed entries to a generic bundled contract warning, and both plugin loader paths still report generic missing register/activate failures. The PR remains potentially useful, but its current head also needs the existing review regression fixed because it drops documented debug-mode module-shape details for ordinary missing-register errors.

## Best Possible Solution

Keep the PR open and ask for a small revision: add the bundled-channel-entry-specific diagnostic without regressing the documented debug module-shape output, use the normalized `record.kind`/`hasKind` path where appropriate, and add focused regression coverage for both the bundled wrong-loader message and the non-bundled debug-mode missing-register contract.
