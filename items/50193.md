## Summary

Keep PR #50193 open. Current main has not implemented provider-grouped chat model options: the Control UI chat model select still renders one flat option list from `resolveChatModelSelectState`. The PR is still aimed at a real UI gap, but it needs rebase/rework because current main moved this surface into `ui/src/ui/chat/session-controls.ts` and `ui/src/ui/chat-model-select-state.ts`, and existing review comments identify label-regression risks that current tests intentionally guard against.

## Best Possible Solution

Keep the PR open, but require a rebase and redesign against current main. The best implementation is to add a grouped chat-model select state shape in `ui/src/ui/chat-model-select-state.ts`, render provider optgroups in `ui/src/ui/chat/session-controls.ts`, preserve existing labels from `buildChatModelOptionFromLookup`/`buildCatalogDisplayLookup`, and add focused tests for duplicate names across providers, duplicate names within one provider, unknown providers, current/default fallback options, and provider-group ordering. If friendly provider names are required, wire them from an existing provider registry/shared metadata source or add a small documented gateway/UI contract instead of hard-coding a stale map in the component.
