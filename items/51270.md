## Summary

Keep open. Current main still intercepts TUI `/model list` and `/model status` in the local `model` switch and sends any non-empty argument to `client.patchSession` as a model id, so the PR is not obsolete. The PR is also not obviously merge-ready as-is because the shared `/model` directive path already defines richer, agent-aware list/status behavior that the narrow TUI patch should preserve.

## Best Possible Solution

Keep the PR open or replace it with a refreshed TUI fix that reserves `/model list` and `/model status` before `sessions.patch`, but routes them through the shared model directive/command behavior or an equivalent agent-aware adapter. The fix should preserve the documented detailed `/model status` output, avoid treating reserved subcommands as model ids, and add TUI regression tests asserting that `patchSession` is not called for `/model list` or `/model status`.
