## Summary

Keep open. Current main still contains the Gateway pricing refresh path implicated by the report, and the related fix PR #72306 is open and not present in this checkout. The issue is a same-day regression with another user confirmation, so it should stay open until the pricing refresh shutdown/startup behavior is fixed and verified.

## Best Possible Solution

Keep the issue open. Land PR #72306 or an equivalent fix that unrefs the scheduled pricing refresh timer and aborts in-flight LiteLLM/OpenRouter pricing fetches during Gateway shutdown, then verify Windows 11 npm-global startup, dashboard reachability, and Ctrl+C exit under forced pricing-request timeouts. If dashboard readiness still fails, keep or open a narrower follow-up for startup decoupling.
