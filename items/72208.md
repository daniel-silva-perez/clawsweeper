## Summary

Keep open. This is a fresh regression report against the latest release, and current main still contains the gateway-startup pricing refresh path for OpenRouter and LiteLLM with timeout handling but no clear post-report fix, duplicate, or full regression coverage for the reported Windows npm-global hang and unresponsive Ctrl+C behavior.

## Best Possible Solution

Keep the issue open. The best next step is a focused reproduction on Windows 11 with npm-global 2026.4.24 and current main while forcing OpenRouter and LiteLLM pricing requests to timeout, then verify whether the gateway reaches HTTP listen/readiness and whether SIGINT closes promptly. If reproduced, fix the coupling between startup/shutdown and model-pricing refresh, and add a regression test that proves timed-out pricing refreshes cannot block dashboard availability or Ctrl+C shutdown.
