## Summary

Keep #49028 open. Current main still does not support Vietnamese in the Control UI: `vi` is absent from the Locale union, lazy locale registry, navigator locale resolver, language picker labels, generated locale files, and i18n registry tests. The PR discussion also shows the only concrete review concern, the Vietnamese `instances`/`version` translation collision, was addressed by the author in follow-up commit `32855cc`, so the remaining issue is refresh/integration rather than closure.

## Best Possible Solution

Keep the PR open, but ask for a refresh against current main: add `vi` to `scripts/control-ui-i18n.ts` `LOCALE_ENTRIES`, `ui/src/i18n/lib/types.ts`, `ui/src/i18n/lib/registry.ts`, navigator resolution, and `en.languages`; regenerate `ui/src/i18n/locales/vi.ts` plus `.i18n` metadata with `pnpm ui:i18n:sync`; update `src/i18n/registry.test.ts`; then validate with `pnpm ui:i18n:check` and the targeted registry test.
