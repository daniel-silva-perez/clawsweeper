## Summary

Keep open. Current main still has the web chat `New session` action wired to send `/new` into the active session, and the UI has no `sessions.create` create-and-switch path. The PR is a concrete draft for a real remaining Control UI workflow gap, though it likely needs rebasing onto the current split `chat/session-controls.ts` code and should use the current `sessions.create` API rather than relying on `sessions.patch` as the creation primitive.

## Best Possible Solution

Keep the PR open as a focused implementation attempt, but ask for a rebase onto current main. The best path is to implement web chat create-and-switch via `sessions.create`, wire `onNewSession` away from `/new`, keep session selector state synchronized in the current `chat/session-controls.ts` surface, add explicit recent-first ordering tests, and cross-link the work to the broader open #50404 session-sidebar/history request.
