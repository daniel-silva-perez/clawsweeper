## Summary

Current main still has the warning-classification gap this PR targets: built-in group allowlist entries such as `group:memory` are valid and documented, but the warning path only checks known core tool IDs, not core group IDs. The PR is a plausible focused fix, though its branch still needs review-comment cleanup before merge.

## Best Possible Solution

Keep the PR open and land a narrow version of the fix: classify own keys of `CORE_TOOL_GROUPS` as gated core entries in `src/agents/tool-policy-pipeline.ts`, add focused coverage for `group:memory`, preserve the existing dedupe-cache regression test, and keep the changelog attribution compliant with the repo style.
