## Summary

Keep open. Current `main` does not implement the proposed `before_identity_resolve` hook or the related `resolvedPeerId` routing path. DM identity collapse on `main` still depends on static `session.identityLinks`, and the Slack, MS Teams, and WhatsApp inbound routes still pass raw sender ids into `resolveAgentRoute`. This PR is not obsolete or incoherent, so there is no strong basis to close it.
