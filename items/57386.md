## Summary

Keep open. Current `main` still has the Google Chat webhook registered as a plugin-managed route, but the Google Chat plugin does not publish any gateway-auth bypass path/artifact. The gateway HTTP server still enforces gateway auth on protected plugin paths unless such a bypass exists, so the reported pre-handler `401 Unauthorized` remains plausible on `main` and is not strongly disproven.

## Evidence

- **Google Chat webhook route is still plugin-auth managed:** The Google Chat monitor registers its webhook route with `auth: "plugin"`, so requests are expected to reach plugin-side verification rather than gateway token auth.
