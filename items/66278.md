## Summary

Keep this PR open. Current main still lacks the Chat Completions tool protocol work proposed here, while the adjacent Responses endpoint already has client-tool support. The PR is not obsolete or implemented on main; it remains the focused implementation candidate for #54405 and the closed stream-output report #54174.

## Best Possible Solution

Keep this PR open. The best path is to review or rework it on current main, resolve the stream finalization concern, and land a focused gateway adapter fix that parses Chat Completions `tools` and `tool_choice`, forwards `clientTools`, emits OpenAI-compatible non-stream and streaming `tool_calls`, handles `role: "tool"` plus `tool_call_id` follow-up messages, and keeps regression coverage in `src/gateway/openai-http.test.ts`.
