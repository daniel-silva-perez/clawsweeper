## Summary

Keep this PR open. Current main still lacks the Chat Completions tool protocol work proposed here: the `/v1/chat/completions` adapter does not parse `tools`/`tool_choice`, does not forward `clientTools` into the agent command, and cannot emit OpenAI-compatible `message.tool_calls` or streamed `delta.tool_calls`. The related issue #54405 is still open, and this PR remains the focused implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open. The best path is to review or rework it on current main, resolve the stream finalization concern, and land a focused gateway adapter fix that parses Chat Completions `tools` and `tool_choice`, forwards `clientTools`, emits OpenAI-compatible non-stream and streaming tool-call responses, handles `role: "tool"` follow-up messages, and keeps regression coverage in `src/gateway/openai-http.test.ts`.
