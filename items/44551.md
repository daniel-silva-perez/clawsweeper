## Summary

Keep open. Current main already has substantial Control UI Chinese localization, browser-locale detection, a saved language picker, docs, and tests, which aligns with the existing comment that much of the Web UI can display Chinese. However, the issue’s central concrete request is not fully implemented: audit/event-log operation names and tool labels such as `exec`, `read`, `cron run`, `subagents list`, and `browser` are still rendered from raw event/tool identifiers or English tool-display metadata in current main.

## Best Possible Solution

Keep the issue open and treat it as a focused Control UI i18n gap: add a translation-aware operation/tool-display layer for event-log and tool-card labels, move remaining hard-coded debug/agent/tool strings through the existing English source and i18n generator, regenerate `zh-CN`, and add focused render/registry tests that prove `exec`, `read`, `cron run`, `subagents list`, and `browser` display as Chinese when the UI locale is `zh-CN`.
