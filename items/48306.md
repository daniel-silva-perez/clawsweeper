## Summary

Keep open. Current `main` shows partial mitigation, not a full fix for #48306. The latest stable path is healthy again (`appcast.xml` now leads with `2026.4.22`, and `v2026.4.22` has the expected macOS assets), but the repo still does not enforce the missing-asset failure mode away: the public mac release workflow is explicitly validation-only and still requires a private publish plus a manual `appcast.xml` commit, and the mac updater failure callback still does not surface a clearer recovery path.

## Risks / Open Questions

- The latest stable release is healthy again, so the bug is not continuously reproducing today; however, closing would assume the manual private mac publish/appcast handoff is now reliable enough, and current `main` does not prove that.
- Current `main` still lacks a repo-visible guard that checks the appcast's referenced GitHub asset is present/reachable before a stable release is considered complete.
- Current mac updater code still does not implement the clearer failure mode requested by the issue when a download target is missing.
