## Summary

Current main still routes model provider requests through strict SSRF-guarded fetch without the env-proxy auto-upgrade that already exists for related media-understanding and memory remote HTTP paths. The PR is a focused implementation candidate, not obsolete or superseded by the related proxy work.

## Best Possible Solution

Keep this PR open as a plausible focused bug fix. Maintainers should review and, if CI is green, land an implementation that applies trusted env-proxy mode to model provider guarded fetch only when no explicit provider dispatcher policy exists and shouldUseEnvHttpProxyForUrl(url) says the target will actually use HTTP_PROXY/HTTPS_PROXY; keep the regression tests for both the env-proxy path and explicit dispatcher preservation.
