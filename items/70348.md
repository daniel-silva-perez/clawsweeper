## Summary

Keep open. Current main still routes `chat.history` through the shared display projection without filtering the compaction marker, pre-compaction memory-flush prompt, or post-compaction refresh text requested by this PR. The submitted PR also needs rework because its text-prefix filter can hide user-authored messages, and current UI code intentionally renders `__openclaw.kind === "compaction"` as a visible divider, so this should not be auto-closed as implemented or superseded.

## Best Possible Solution

Keep #70348 open but request a safer implementation in the shared gateway chat display projection. The fix should use metadata/provenance for internal memory-flush and post-compaction artifacts, explicitly decide whether synthetic compaction markers remain visible dividers or are hidden from `chat.history`, and add regression tests that hide internal artifacts while preserving user-authored messages with similar text.
