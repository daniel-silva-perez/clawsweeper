## Summary

Keep open. Current main has useful partial hardening for remote CDP, but it does not implement the requested auto-reconnect behavior for a WebSocket that drops after opening or while a command is in flight. The related PR was closed unmerged, and current tests still assert hard failure for that core case.

## Best Possible Solution

Keep this issue open and scope the fix in the browser plugin. Preserve the existing handshake retry and safe-read reconnect behavior, then add bounded recovery for recoverable post-open transport closes: evict stale cached CDP/Playwright state for later actions, retry clearly idempotent read/snapshot/list operations, and avoid replaying ambiguous mutating commands unless they have explicit idempotency or dedupe. Add focused tests for close-after-open, mid-command close, next-action recovery, retry exhaustion, and non-replay of unsafe mutations. Treat closed PR #67748 as background, not as shipped code.
