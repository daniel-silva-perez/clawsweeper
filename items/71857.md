## Summary

Keep this PR open. Current main still rejects `owner/slug` ClawHub skill inputs before registry lookup, while this PR is a focused implementation candidate for the linked user bug and adds the missing regression coverage. The PR diff is limited to skills slug parsing, tests, and changelog; I did not find an unrelated security or supply-chain-sensitive change in the provided/current diff.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or request changes on a narrow shared ClawHub skill slug parser in `src/agents/skills-clawhub.ts` that accepts optional `<owner>/<slug>` for requested install/update inputs, validates both segments, canonicalizes to the bare slug before registry/download/install-dir/lockfile use, preserves legacy tracked-slug update compatibility, and ships with the focused regression tests and changelog entry shown here.
