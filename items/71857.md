The current PR file list is limited to skills slug parsing, its tests, and changelog; I did not find security-sensitive infrastructure, dependency, workflow, lockfile, or secret-handling churn in the current net diff.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or request changes on the narrow `src/agents/skills-clawhub.ts` parser that accepts optional `<owner>/<slug>` for requested install/update inputs, validates both segments before any registry or filesystem use, canonicalizes to the bare slug, preserves legacy tracked-slug update compatibility, and ships with the focused regression tests and changelog entry already present in the PR.
