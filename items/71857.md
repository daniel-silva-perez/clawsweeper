## Summary

Keep this PR open. Current main still rejects owner-prefixed ClawHub skill slugs before any registry lookup, while the PR is a focused implementation candidate for the already-confirmed user bug. The linked bug report was closed as superseded by this PR, not because the fix shipped on main. The PR diff is limited to skills slug parsing, regression tests, and a changelog entry; I did not find unrelated security- or supply-chain-sensitive changes.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or request changes on a narrow parser in src/agents/skills-clawhub.ts that accepts optional <owner>/<slug> for requested ClawHub install/update inputs, validates both segments against the existing ASCII/hyphen/traversal guards, canonicalizes to the bare slug before registry/download/install-dir/origin/lockfile use, preserves legacy tracked-slug update compatibility, and ships with the focused regression tests and changelog entry shown in the PR.
