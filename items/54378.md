## Summary

Keep open. Current `main` still uses the same WebChat attachment flow the report describes: images are read as data URLs, stored in reactive chat state, previewed from that state, and converted back to base64 on send. I did not find a clear fix on `main`, and I did not have enough evidence to conservatively close this as implemented or obsolete.
