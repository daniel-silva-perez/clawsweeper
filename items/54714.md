## Summary

Keep open. Current main still has the original identity-precedence bug on the `agent.identity.get` path used by the Control UI: `ui.assistant` is resolved before `agents.list[].identity` for every agent, and the Agent Context renderer prefers that runtime identity result over the agent-list payload. The linked fix PR #54730 was closed unmerged, and current tests do not cover the requested subagent per-agent identity precedence.

## Best Possible Solution

Keep #54714 open and land a focused fix similar to the unmerged #54730 direction: for non-default agents, `resolveAssistantIdentity()` should prefer `agents.list[].identity` over global `ui.assistant` while preserving the existing default-agent behavior. Add regression coverage for default global priority, subagent per-agent name, subagent emoji/avatar, and partial fallback. Then make the Agent Context display the configured emoji/avatar semantics clearly instead of only `custom`/`—`, and decide separately whether isolated-workspace avatar-path docs need a clearer user-facing note.
