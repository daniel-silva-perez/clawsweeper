## Summary

Keep open. Current `main` has compaction checkpoint support, but it does not implement the first-class checkpoint engine requested in #17211. The shipped surface is limited to compaction-specific `sessions.compaction.list/get/branch/restore`, with compaction-only checkpoint metadata and no native checkpoint diff API or task-boundary `checkpoint.create/resume/diff` contract.

## Evidence

- **Gateway method surface is narrower than the issue request:** `src/gateway/server-methods-list.ts` lists `sessions.compaction.list`, `sessions.compaction.get`, `sessions.compaction.branch`, and `sessions.compaction.restore`, but there is no `checkpoint.create`, `checkpoint.resume`, `checkpoint.diff`, or equivalent diff method.
  - file: [src/gateway/server-methods-list.ts:68](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/src/gateway/server-methods-list.ts#L68)
  - command: `sed -n '1,220p' src/gateway/server-methods-list.ts`
  - sha: [a59d1bd46d9f](https://github.com/openclaw/openclaw/commit/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3)
- **Checkpoint data model is compaction-specific, not a general task-boundary checkpoint schema:** `SessionCompactionCheckpoint` stores compaction metadata such as `reason`, `tokensBefore`, `tokensAfter`, `summary`, `firstKeptEntryId`, and `preCompaction`/`postCompaction` transcript references. It does not provide the requested structured `done/saved/dropped/next` contract.
  - file: [src/config/sessions/types.ts:81](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/src/config/sessions/types.ts#L81)

That is useful resume plumbing, but it is not the first-class native checkpoint engine proposed by the issue.
