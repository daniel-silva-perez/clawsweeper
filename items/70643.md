## Summary

Keep #70643 open. Current main has atomic temp-file/rename writes, schema validation, destructive-write rejection, and last-known-good recovery, but it does not linearize concurrent `openclaw.json` read-modify-write cycles with an advisory lock. The directly linked PR #70655 proposed that exact lock but was closed unmerged, and no current regression test covers concurrent config writers.

## Best Possible Solution

Keep #70643 open and implement linearized config writes in core: hold a shared advisory lock around the full `openclaw.json` read/merge/validate/write cycle, re-read inside the lock for mutation helpers, cover include-write behavior explicitly, and add a regression test with concurrent config writers so the gateway/CLI race cannot reappear.
