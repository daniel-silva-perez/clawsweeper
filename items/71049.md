## Summary

Keep #71049 open. Current main still creates the MS Teams stream controller unconditionally for personal/DM conversations, sends the informative stream status on reply start, and exposes no schema/type/docs option that disables that DM stream path while falling back to native typing plus one final message.

## Best Possible Solution

Add an MS Teams plugin-owned config option, likely under the channel streaming surface, that disables the personal/DM `TeamsHttpStream` path. The implementation should update the strict schema, exported types, config UI metadata/docs, and tests so DMs can use native typing keepalive plus normal final-message delivery without the informative stream status.
