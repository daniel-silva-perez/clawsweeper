## Summary

Keep PR #68262 open. Current main has partial visible-history filtering for default heartbeat prompts and internal runtime-context envelopes, but it has not implemented this PR’s provenance-based source tagging: heartbeat/cron/exec-event turns are not tagged as `internal_system`, gateway history sanitization does not drop that provenance, and the Control UI defense-in-depth filter does not check it. The PR discussion shows a concrete pivot away from brittle text-prefix filtering after review concerns and a fresh subagent-completion leak repro, so this remains active useful work rather than an obsolete PR.

## Best Possible Solution

Keep the PR open and review a provenance-based version of the fix. The best path is to tag heartbeat, cron, and exec-event prompts at origin with `InputProvenance.kind = "internal_system"`, preserve that provenance through transcript persistence, drop or classify those rows in gateway/session-history projections, and keep the Control UI check as defense-in-depth. Existing text/envelope filters can remain as legacy compatibility, with regressions for default/custom heartbeat prompts, cron/exec events, SSE/session-history snapshots, and user messages that merely quote internal prompt text.
