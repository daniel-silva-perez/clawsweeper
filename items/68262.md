## Summary

Keep PR #68262 open. Current main has partial text/envelope-based hiding for heartbeat and internal runtime-context leaks, but it does not implement the PR’s provenance-based fix: heartbeat/cron/exec-event turns are not source-tagged as `internal_system`, gateway sanitization does not drop that provenance, and the Control UI does not check it. The PR is recent, focused, and responds to concrete review concerns about brittle prefix filtering rather than being obsolete or safely closable.

## Best Possible Solution

Keep the PR open and review a provenance-based version of the fix: set `InputProvenance.kind = "internal_system"` at the heartbeat/cron/exec-event origin, drop those user messages in gateway/session-history projections and the Control UI, keep existing text/envelope stripping only as legacy compatibility, and add regressions for default/custom heartbeat prompts, cron/exec events, SSE/session-history snapshots, and user messages that merely quote internal prompt text.
