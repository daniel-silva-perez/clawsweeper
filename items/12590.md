## Summary

Keep open. On current `main` (`3361593442e295e9bb1f55cbfb38861ea9413e45`), the reported bookkeeping bug still appears present in source: `runMemoryFlushIfNeeded` persists `memoryFlushCompactionCount` as the post-increment compaction count, while the dedup gate still treats `memoryFlushCompactionCount === compactionCount` as already flushed for the current cycle. The current tests also still encode that synchronized-counter state, so there is not strong evidence to close this as fixed or obsolete.

## Evidence
