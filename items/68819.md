## Summary

Keep this PR open. Current main still routes the bundled `claude-cli` backend through the supervisor child adapter as bare `claude`, the adapter still only shims npm-family commands, and the existing Windows `.cmd` workaround in `exec.ts` is not used on that supervisor path. The linked bug was already closed as superseded by this PR, while the narrower duplicate PRs are closed and unmerged.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate for the Windows `claude-cli` supervisor spawn bug. The best path is to rebase or port the patch onto current main, verify the Windows command-resolution semantics against Node/libuv behavior, keep the focused regression tests for PATH/env/fallthrough cases, and land this or an equivalent core fix rather than closing it as obsolete.
