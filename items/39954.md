## Summary

Keep PR #39954 open. Current main still documents the raw `tools.profile` contents but does not include this PR’s practical explanation that `messaging` is intentionally narrow or that users who want broader command/control should choose `full`. The underlying behavior is real in current code and tests, so the docs change remains useful rather than obsolete.

## Best Possible Solution

Keep the PR open and port the useful wording into the current `docs/tools/index.md` profile section: add a compact note that `tools.profile: "messaging"` is intentionally narrow, point broader command/control users to `tools.profile: "full"`, and align wording with the current profile table and related docs/help PRs rather than closing the contribution as stale.
