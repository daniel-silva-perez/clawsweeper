## Summary

Keep #71487 open. Current main still has TTS backend, CLI, docs, and a per-message browser-native "Read aloud" button, but there is no first-class Control UI Settings surface for a persisted TTS enable toggle, provider picker, default voice picker, or sample playback. The existing issue discussion already reached the same keep-open conclusion, and current HEAD has not added the requested UI.

## Best Possible Solution

Keep the issue open. Implement a first-class Text-to-Speech section in Control UI Settings or Quick Settings that uses existing tts.status, tts.providers, tts.enable, tts.disable, tts.setProvider, and tts.convert where possible, clearly distinguishes configured and unavailable providers, filters voices by selected provider, and adds a deliberate persistence/API path for default voice. Cover the new UI/RPC behavior with focused tests.
