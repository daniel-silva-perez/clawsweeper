## Summary

Keep this issue open. Current main has part of the requested lifecycle cleanup, but it still does not keep the run-side typing TTL alive on each keepalive tick, so configured presence can still expire during a healthy long run. Open PR #71828 is a current targeted candidate for that remaining gap, while the earlier broader PR #66365 is closed unmerged.

## Best Possible Solution

Keep the issue open and review/land PR #71828 or an equivalent broader active-run presence lease. The implementation should refresh the run-side presence lease while the run is active, preserve stale-presence safety cleanup, and stop only after run completion plus dispatch idle.
