## Summary

Keep open. Current main has useful run-complete plus dispatch-idle cleanup, but it still does not keep the run-side typing TTL alive on keepalive ticks, so configured presence can still disappear during a healthy long active run. Open PR #71828 is the current targeted candidate for the missing refresh and regression test; the broader PR #66365 is closed unmerged.

## Best Possible Solution

Keep this issue open and review/land PR #71828 or an equivalent active-run presence lease. The implementation should refresh the run-side presence lease while the run is active, preserve stale-presence safety cleanup, avoid increasing provider-facing keepalive cadence, and stop only after run completion plus dispatch idle.

## Risks / Open Questions

- Provider rate limits are a real design constraint from the discussion; the fix should refresh the internal lease without increasing channel send cadence or overriding configured surfaces.
- PR #71828 appears to address the typing TTL gap specifically. Maintainers may still want a broader active-run presence lease if reactions or channel-local progress surfaces have the same lifecycle problem.
