## Summary

Keep this PR open. Current main still forwards wizard select and multiselect labels directly into Clack with no blank-label fallback, and the existing wizard tests only cover the search filter. The related onboarding crash PRs in the discussion address channel metadata fallback, higher-level trim guards, or text prompt results, not this select/multiselect adapter case.

## Best Possible Solution

Review this PR as a plausible focused bug-hardening change rather than closing it. The best path is to land a narrow adapter-level normalization in `src/wizard/clack-prompter.ts` with unit coverage for select, multiselect, and searchable paths, while tightening the PR wording or tests around the exact crash condition if needed.
