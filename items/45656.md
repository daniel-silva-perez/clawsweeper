## Summary

Keep open. Current `main` still uses the v2 split composer layout the report describes: the visible chat bar is one rounded container, but the editable area is only the upper `textarea` and the lower section is a separate toolbar. The controls are still 30x30 icon-only buttons with meaning exposed via `title`/`aria-label` rather than visible labels. Diffing the latest release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`) to current `main` (`f03252aaf9cc946dd6a4f6189f30d5c5dcd64022`) shows additions for Talk/status and queue behavior, not a usability rework for this composer regression.
