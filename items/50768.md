## Summary

Keep open. Current `main` does not implement reusable sub-agent templates, and the issue is a coherent feature request rather than a stale or incoherent report. OpenClaw currently supports only direct `sessions_spawn` overrides plus agent-level subagent defaults, with dedicated agents as the closest existing workaround.

## Evidence

- **`sessions_spawn` has no `template` parameter:** The tool schema accepts `task`, `label`, `runtime`, `agentId`, `model`, `thinking`, `cwd`, `runTimeoutSeconds`, and related fields, but no `template` field or template-resolution path.
