## Summary

Current `main` has partial interactive-card text extraction for some fetched Feishu messages, but it does not implement the full request. The code only pulls markdown/div body text in the fetch/thread path, does not extract card headers/templates/actions, and does not expose raw card JSON or structured card metadata on the Feishu message types. Live inbound event parsing also still treats interactive payloads as generic JSON strings rather than a parsed card structure.

## Evidence

- **Fetch parser is text-only:** `parseInteractiveCardContent` only reads `div.text.content` and `markdown.content` from `elements` or `body.elements`; it does not parse headers, templates, buttons, form fields, or other card structure.

## Risks / Open Questions

- The original blocker about seeing only `[Interactive Card]` appears partially improved in fetch/thread code, so future triage should distinguish between 'plain-text extraction exists' and the broader feature request for structured interactive-card support.
- There is existing card-action infrastructure for OpenClaw-authored Feishu cards, but that is not the same as generic parsing of arbitrary user-sent interactive card actions/forms.
