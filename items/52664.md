## Summary

Keep open. Current main still does not expose `rawBody` on `before_prompt_build`, `before_agent_start`, or `agent_end` hook events. The repo has the clean inbound `CommandBody`/`RawBody` concept, but it is not threaded through reply run params, `FollowupRun`, queue drain, embedded runner params, or hook payloads. PR #52664 remains a plausible focused API addition, with the existing discussion usefully narrowing followup/collect handling and leaving steer semantics as a maintainer decision.

## Best Possible Solution

Keep #52664 open. Rebase it onto current main, add the optional `rawBody` field to the current hook event type files, thread `CommandBody ?? RawBody ?? Body` through immediate, deferred, and collect queue runs alongside `transcriptPrompt`, and add focused tests for immediate, followup, collect, and the documented steer behavior.
