## Summary

Keep open. Current `main` still does not include the requested fallback clarification in `docs/date-time.md`, even though the runtime and other docs already document that `userTimezone` falls back to the host timezone. This PR is therefore not implemented on `main`, and the request remains valid.

## Evidence
