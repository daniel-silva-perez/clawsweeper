## Summary

Keep open. Current main still has the exact false-success shape this PR targets: multiple Discord send success paths return `messageId: "unknown"` when Discord returns a 2xx response without an ID, so the requested behavior is not implemented on main. The PR is also not mergeable as-is because its own review context shows unrelated bundled changes and an incomplete webhook path, so the best outcome is a narrow Discord-only follow-up rather than auto-closing as shipped.

## Best Possible Solution

Keep the item open, but replace or revise it as a narrow Discord-only fix: require a concrete Discord message ID on all successful delivery result paths that are supposed to create a message, keep non-2xx Discord errors throwing, add focused regression tests for direct and webhook 2xx-without-id responses, and drop the unrelated Matrix/model-selection/Ollama/schtasks edits from this PR.
