## Summary

Keep open. Current `main` does not implement user-facing timestamps for TUI chat messages: message components only render text, history replay ignores message timestamps when rebuilding the chat log, and there is no `/timestamps` command or settings toggle in the TUI surface.

## Evidence

- **Chat log API has no timestamp parameter for rendered user/assistant entries:** `addUser`, `startAssistant`, and `finalizeAssistant` accept only message text. The only timestamp-aware path here is `reconcilePendingUsers`, which uses timestamps internally to match pending messages, not to display them.
  - file: [src/tui/components/chat-log.ts:105](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/tui/components/chat-log.ts#L105)
  - command: `nl -ba src/tui/components/chat-log.ts | sed -n '85,230p'`
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Rendered message components have no timestamp label UI:** `UserMessageComponent` and `AssistantMessageComponent` constructors take only `text` and render styled markdown/body content with no timestamp field or prefix.
  - file: [src/tui/components/user-message.ts:4](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/tui/components/user-message.ts#L4)
  - command: `nl -ba src/tui/components/user-message.ts | sed -n '1,120p'; nl -ba src/tui/components/assistant-message.ts | sed -n '1,160p'`
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **History replay drops message timestamps when rebuilding the TUI log:** `loadHistory()` iterates `record.messages` and, for `user`/`assistant` roles, extracts text and calls `chatLog.addUser(text)` / `chatLog.finalizeAssistant(text)` without reading or forwarding `message.timestamp`.
  - file: [src/tui/tui-session-actions.ts:288](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/tui/tui-session-actions.ts#L288)
  - command: `nl -ba src/tui/tui-session-actions.ts | sed -n '286,360p'`
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **No `/timestamps` slash command exists on current main:** The TUI slash command registry includes commands like `/help`, `/gateway-status`, `/model`, `/verbose`, `/settings`, and `/exit`, but nothing for timestamps.
  - file: [src/tui/commands.ts:57](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/tui/commands.ts#L57)
