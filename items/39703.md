## Summary

Keep #39703 open. Current main still has FIFO-style cron cache eviction: cache hits return without reordering the Map entry, while capacity eviction deletes the first insertion-order key. The PR remains useful and appears to be the canonical focused tracker for this remaining cron LRU fix; related local review context already points duplicate PR #39706 back to #39703.

## Best Possible Solution

Keep #39703 open as the canonical remaining fix, or land an equivalent maintainer-owned patch that adds LRU promotion on cron cache hits and regression coverage proving the promoted key survives eviction while the next-oldest key is evicted.
