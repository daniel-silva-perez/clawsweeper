## Summary

Keep open. Current `main` contains relevant CLI/gateway mitigations, but the evidence is not strong enough to say issue #45222 is fully fixed on `main`. The new code waits for one-shot gateway clients to finish teardown and yields before starting the WebSocket client, which plausibly reduces this failure family, but there is no direct reproduction proof or regression test covering the reported `openclaw cron list` loopback-handshake failure on Linux/VPS environments.
