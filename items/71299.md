## Summary

Keep #71299 open. Current main has related Telegram hardening for restart replay, send-failure logging/retry, and generic delivery diagnostics, but the reported distinguishing symptom is still a possible no-attempt outbound path: Web UI shows the reply, Telegram misses it, and no send log appears. The maintainer comment already left this open pending a retest, and the linked logging PR #71377 was closed unmerged.

## Best Possible Solution

Keep #71299 open and ask for a retest on a build containing the current Telegram offset persistence, send-failure logging, and delivery diagnostics. If a missing Telegram reply still has no `telegram ... failed` log or `message.delivery.error` diagnostic, the next fix should instrument the outbound dispatch boundary and Telegram adapter enough to distinguish hook cancellation, empty payload normalization, lost session route, queue claim suppression, and actual Telegram API failure.
