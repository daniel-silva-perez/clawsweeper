## Summary

Keep open. Current `main` does not contain the cursor-based pagination described in the issue comments: `chat.history` still accepts only `sessionKey`, `limit`, and `maxChars`; the server still reads the full transcript file and slices the last `limit` messages; the UI still requests `limit: 200` and renders only the last 200 messages with a hidden-history notice. That is not strong evidence for `implemented_on_main` or `cannot_reproduce`.
