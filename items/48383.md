## Summary

Keep open. Current main added plugin-level `session_start`/`session_end` hooks, but it still does not implement the issue's requested internal `session:start` lifecycle hook with first-turn context injection semantics.

## Evidence

- **Internal hooks docs do not list `session:start`:** The current internal hooks event table lists `command:*`, `session:compact:*`, `session:patch`, `agent:bootstrap`, `gateway:startup`, and message events, but no `session:start` entry.
  - file: [docs/automation/hooks.md:36](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/docs/automation/hooks.md#L36)
  - command: `nl -ba docs/automation/hooks.md | sed -n '34,55p'`
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **No internal session-start emitter on main:** The only current internal `session` hook emit sites are compaction hooks (`compact:before` and `compact:after`). Repo search found no `createInternalHookEvent("session", "start", ...)` or equivalent trigger.
  - file: [src/agents/pi-embedded-runner/compaction-hooks.ts:185](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/src/agents/pi-embedded-runner/compaction-hooks.ts#L185)
  - command: `rg -n 'createInternalHookEvent\("session", ' src -g '*.ts'`
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **Main does have a plugin `session_start` hook, but it is a separate seam:** New sessions call `hookRunner.runSessionStart(...)`, and the public plugin hook event only carries `sessionId`, optional `sessionKey`, and optional `resumedFrom`.
  - file: [src/auto-reply/reply/session.ts:849](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/src/auto-reply/reply/session.ts#L849)
  - command: `nl -ba src/auto-reply/reply/session.ts | sed -n '840,860p'`
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **Plugin `session_start` is fire-and-forget and cannot mutate initial context itself:** The hook runner defines `runSessionStart(...)` as a void hook path, not a modifying hook. The event type also lacks the richer channel/conversation/thread metadata proposed in the issue.
  - file: [src/plugins/hooks.ts:958](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/src/plugins/hooks.ts#L958)
  - command: `nl -ba src/plugins/hooks.ts | sed -n '956,968p'`
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **Prompt injection exists, but without new-session lifecycle metadata:** `before_prompt_build` can inject `prependContext`/system context, but its event only exposes `prompt` and `messages`. It does not provide `isResume`, channel/thread IDs, or a dedicated session-start contract.
  - file: [src/plugins/hook-before-agent-start.types.ts:21](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/src/plugins/hook-before-agent-start.types.ts#L21)

## Risks / Open Questions

- There is partial overlap from the newer plugin hook system, so future triage should avoid treating `session_start` as equivalent to the requested internal `session:start` hook without checking the missing context-mutation semantics.
