## Summary

Keep open. Current main still models Discord `actions.messages` as one boolean gate, discovery has no per-operation read/edit/delete/send controls, and the runtime still checks that same `messages` gate for reads, sends, edits, and deletes. The existing maintainer-side Codex comment keeping #7234 open remains accurate against current main `4ad8b613c9b60466224d9b3737fff7d63ada652a`.

## Best Possible Solution

Keep #7234 open. The likely maintainer path is a backward-compatible Discord gate design, such as `channels.discord.actions.messages: boolean | { send?: boolean; read?: boolean; edit?: boolean; delete?: boolean }` or equivalent top-level Discord gates, wired through config types, zod/generated schema, account gate resolution, discovery, runtime checks, docs, and focused Discord tests. This belongs in the bundled Discord plugin/config surface rather than ClawHub because the current official Discord integration exposes and enforces the coarse gate.
