## Summary

Keep open. Current `main` does not implement this PR's compaction observability changes: compaction start still emits only `{ phase: "start" }`, the dispatcher does not pass the start event through so a `reason` cannot be surfaced, and compaction end still emits only `{ phase, willRetry, completed }` without `hasResult` or sanitized `errorMessage`. The existing tests on `main` also still assert the older payload shape.
