## Summary

Keep open. Current main has several exact-duplicate and delivery idempotency guards, but the attached report describes WebChat/Omlx/Qwen-style repeated assistant preambles and intent statements across separate tool/approval steps, not just a repeated final event or duplicated stream chunk. I did not find code, docs, tests, or related-item evidence that current main fixes that broader behavior.

## Best Possible Solution

Keep this issue open and request a current-main WebChat reproduction with the raw transcript, gateway `chat` and `agent` events, provider/model config, and whether tools or approvals were involved. If the duplicate is UI/history rendering, add a regression around `ui/src/ui/controllers/chat.ts` or `ui/src/ui/app-gateway.ts`; if it is duplicate event delivery, fix the gateway/subscriber idempotency path; if it is model-generated repetition for Qwen/Omlx, tune the WebChat/system prompt or provider guidance and cover it with a focused regression.
