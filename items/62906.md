## Summary

Keep open. Current main has several duplicate-delivery and commentary-suppression fixes, but I did not find high-confidence evidence that the reporter's webchat + local/custom Qwen route is fully fixed. The attached report is about repeated visible assistant preambles and repeated explanations around tool/approval turns, and current suppression depends on phase metadata that generic OpenAI-compatible/local completion streams may not provide.

## Best Possible Solution

Keep #62906 open and narrow it to a reproducible webchat/local-Qwen route. Maintainers should confirm whether `omlx` uses OpenAI-compatible chat completions or Responses; if it is unphased completions, the likely fix is either adding reliable commentary/final phase tagging for that transport or landing a conservative last-mile sanitizer like the open #60551 direction, with regression coverage for repeated visible preambles around tool/approval turns.
