## Summary

Keep open. Current main has partial avatar plumbing for Control UI, but the agent-list UI still does not render configured agent avatars. Backend code resolves and serves per-agent avatars, and chat/welcome surfaces can use them, yet the Agents screen still renders a plain text-only `<select>` and other agent metadata panels reduce avatar state to `custom`/`—` instead of showing the image.
