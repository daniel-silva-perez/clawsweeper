## Summary

Keep open. Current main still has the request-scoped fallback path and still runs cleanup through the same subagent in a finally block after fallback, so the reported second warning remains possible. The related PR #70554 is an active implementation candidate, but it is not merged into the checked-out main behavior.

## Best Possible Solution

Keep this issue open until PR #70554 or an equivalent memory-core fix lands. The fix should either run dreaming narrative generation with a gateway-valid background subagent runtime or skip/suppress request-scoped cleanup after fallback, and it should update regression coverage so a request-scoped fallback writes the diary entry without emitting the cleanup warning.
