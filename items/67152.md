## Summary

Keep open. Current main still contains the reported request-scoped fallback plus unconditional cleanup path, and the existing regression test explicitly expects deleteSession after that fallback. PR #70554 is an open implementation candidate for the same surface, but it is not merged into current main.

## Best Possible Solution

Keep this issue open until PR #70554 or an equivalent memory-core fix lands. The fix should make background dreaming use a gateway-valid subagent runtime or skip/suppress request-scoped cleanup after fallback, and add a regression test that the fallback path does not emit the cleanup warning.
