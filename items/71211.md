## Summary

Keep open. Current main still returns exec stdout/stderr-derived output through foreground results, runtime outcomes, background process polling/logging, command-output events, and node-host formatting without applying the existing secret redaction helper at those exec-owned boundaries.

## Best Possible Solution

Keep this issue open and patch the exec-owned output boundary. Apply configured redactSensitiveText or an equivalent resolved redaction pass before stdout/stderr-derived text or details leave exec: foreground buildExecForegroundResult, runtime outcome construction, process poll/log/list details, command-output events, notify/follow-up summaries, and node-host formatNodeRunToolResult. Add focused regression tests for env-style assignments, bearer tokens, and common API-key formats while preserving existing output caps and transcript persistence safeguards.
