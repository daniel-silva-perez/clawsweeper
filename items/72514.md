## Summary

Keep open. Current main already has several supporting primitives for this report, but not enough to close it as implemented: parent-visible session history/status and subagent controls exist, while explicit subagent retry/resume and seamless QClaw/webchat-WeChat handoff remain unresolved product behavior.

## Best Possible Solution

Keep this open and narrow it into an owner-reviewed core session/subagent UX task: reproduce the QClaw/webchat-WeChat flow on current main, decide whether parent-to-child send/history needs more tests or docs, add an explicit retry/resume/status surface for aborted subagent runs if product-approved, and document the cross-channel continuity path using dmScope, identityLinks, and memory/plugin options.

## Risks / Open Questions

- The report names QClaw and WeChat, so maintainers should verify the affected OpenClaw/QClaw/plugin versions before deciding whether this is core behavior, plugin behavior, or configuration/documentation friction.
- Cross-channel history sharing has privacy and multi-user safety tradeoffs; any automatic continuity should remain opt-in through identity links, memory, or an explicit UX flow.
- Subagent retry/resume could duplicate side effects unless it is tied to idempotency, task ledger state, and clear parent notification semantics.
