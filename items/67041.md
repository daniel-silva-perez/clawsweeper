## Summary

Keep this PR open. Current main still does not add default redaction for Basic auth headers, standalone JWTs, or Pomerium JWT assertion headers in the shared logs.tail redaction pipeline, while the linked bug remains an actionable security hardening report. The PR is a plausible implementation candidate, but it should be amended before merge to fix the Basic-auth padding boundary and strengthen test placement/isolation.

## Best Possible Solution

Keep the PR open as the implementation candidate for the linked logs.tail redaction bug, but request a small amendment: remove or replace the Basic-auth trailing word boundary, add focused tests in the redactSensitiveText/redactSensitiveLines coverage for Basic auth including padded base64, standalone JWTs without Bearer, and x-pomerium-jwt-assertion, then run the targeted logging tests before merge.
