## Summary

Keep open. Current `main` still does not implement the requested auto-trigger behavior for bundled `session-memory`: the hook is documented and wired only for `command:new` and `command:reset`, with no idle, shutdown, or periodic checkpoint trigger support, and it still writes recent transcript lines rather than an optional LLM summary.

## Evidence

- **Bundled hook metadata still limits events to reset/new:** The bundled hook frontmatter declares only `command:new` and `command:reset` in `metadata.openclaw.events`, and the docs say it saves context when you issue `/new` or `/reset`. No idle, shutdown, or periodic trigger config is documented there.
  - file: [src/hooks/bundled/session-memory/HOOK.md:3](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/src/hooks/bundled/session-memory/HOOK.md#L3)
  - command: `nl -ba src/hooks/bundled/session-memory/HOOK.md | sed -n '1,140p'`
  - sha: [a59d1bd46d9f](https://github.com/openclaw/openclaw/commit/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3)
- **Handler hard-gates to command new/reset only:** The implementation returns unless `event.type === "command"` and `event.action` is `"new"` or `"reset"`, so the bundled hook cannot currently fire on idle, shutdown, or periodic checkpoints.
  - file: [src/hooks/bundled/session-memory/handler.ts:53](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/src/hooks/bundled/session-memory/handler.ts#L53)

It documents `session-memory` as `command:new` and `command:reset` only; there is no idle or shutdown event in the published event table for internal hooks.
