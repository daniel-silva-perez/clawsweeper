## Summary

Keep open. Current main has in-memory tool_use/tool_result pairing repair, and it runs before model calls and compaction context, but the on-load session-file repair only drops syntactically malformed JSONL lines. The requested durable rewrite of valid JSONL transcripts with orphaned/displaced tool_result blocks is not implemented on main. The issue discussion also identifies the same persistence gap and mentions session rotation as a workaround, so the report is coherent and still actionable.
