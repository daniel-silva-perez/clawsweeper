## Summary

Keep open. Current main still emits `gateway.trusted_proxies_missing` as `warn` for `gateway.bind="loopback"`, and the current test/docs still encode that behavior. The PR is meaningful but not merge-ready as-is because the latest PR discussion flags security-relevant false-negative risks in the proposed effective-bind classifier for `auto`/`custom` fallback cases.

## Best Possible Solution

Keep the PR open, but require a revised implementation before merge. The fix should derive audit severity from a fail-closed effective-bind classification aligned with `resolveGatewayBindHost`, avoid environment-only assumptions for `auto`, account for `canBindToHost` fallback behavior, and add regression tests for loopback, non-loopback, custom loopback, custom fallback, and auto container/non-container paths. Keep #70357 as the user-facing tracker until a safe implementation lands.
