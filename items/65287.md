## Summary

Keep #65287 open. Current main still lacks Anthropic native compaction request parameters, context_management.edits injection, compaction-block round-tripping, and the requested mixed Anthropic prompt-cache TTL policy. The related open #10213 overlaps the server-side compaction half, but does not explicitly track the mixed TTL/cache-policy acceptance criteria, and PR #65288 was closed unmerged.

## Best Possible Solution

Keep #65287 open, or have a maintainer deliberately consolidate it into #10213 after copying over the mixed TTL requirements. The implementation likely belongs in the Anthropic transport/payload-policy path: add opt-in Anthropic compaction params, inject the required beta/header and context_management.edits only when enabled, preserve Anthropic compaction blocks through history and streaming, and change long-retention cache shaping so stable system/tool/workspace prefix blocks use 1h while volatile conversation/compaction/user tail blocks stay at 5m.
