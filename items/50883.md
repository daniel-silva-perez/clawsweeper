## Summary

Keep PR #50883 open. Current main still does not implement `openclaw sessions --kind`; the registered command, route-first parser, command implementation, docs, and tests only cover `--active` plus scope/output flags. The PR is focused and still useful, but it needs to address the existing review feedback: route-first `sessions` invocations must parse and forward repeated/comma-separated `--kind` values too.

## Best Possible Solution

Keep the PR open and ask for a small revision: add a shared session-kind parser/type, wire `--kind` through both Commander registration and `parseSessionsRouteArgs`, apply the filter in `sessionsCommand` alongside `--active`, expose filter metadata consistently in JSON if desired, and add focused tests for the direct command parser plus route-first path.
