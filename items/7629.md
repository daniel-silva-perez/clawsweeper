## Summary

Keep open. Current main still leaves the bundled `memory-lancedb` plugin on vector-only LanceDB retrieval: it creates/opens the table without creating an FTS index, `MemoryDB.search` accepts only a vector and calls `vectorSearch`, query text is embedded and then dropped before DB search, and both runtime config plus manifest schema reject the proposed `hybrid` settings. The closed related PR #7636 was not merged, and the existing maintainer/Codex issue comment already correctly kept this item open.

## Best Possible Solution

Keep #7629 open as a focused bundled plugin parity enhancement. The best implementation likely belongs in `extensions/memory-lancedb`: add typed `hybrid` config plus manifest/docs entries, create or maintain an FTS index on `text`, pass original query text through recall/forget/CLI/hooks, use LanceDB hybrid search or a verified fallback/reranker when enabled, preserve vector-only fallback, and add targeted tests for config validation, FTS setup, exact-keyword recall, fallback behavior, and query text propagation.
