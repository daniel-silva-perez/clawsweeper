## Summary

Current `main` only partially covers this request. The web cron UI now renders the job prompt/payload on each job card, but it does not implement the requested click-to-reveal details flow, and the job `description`/summary field is still not shown in the job list/details view.

## Evidence

- **Cron row click does not reveal extra details:** `renderJob()` wires the card click to `props.onLoadRuns(job.id)` and renders the same fixed fields inline; there is no expanded detail panel or toggle for showing job metadata on click.
  - file: [ui/src/ui/views/cron.ts:1481](https://github.com/openclaw/openclaw/blob/68e2d6f0889e5365ecdc8f01ab5e3bf6065cbeeb/ui/src/ui/views/cron.ts#L1481)
  - command: `nl -ba ui/src/ui/views/cron.ts | sed -n '1480,1610p'`
- **Prompt is shown, but only the prompt/delivery fields:** `renderJobPayload()` displays `job.payload.message` under the `Prompt` label for agent-turn jobs (or system text for system events), plus optional delivery info. That covers part of the request, but not the requested summary/description.
  - file: [ui/src/ui/views/cron.ts:1578](https://github.com/openclaw/openclaw/blob/68e2d6f0889e5365ecdc8f01ab5e3bf6065cbeeb/ui/src/ui/views/cron.ts#L1578)
  - command: `nl -ba ui/src/ui/views/cron.ts | sed -n '1480,1610p'`
- **Description exists in the cron model and edit form, but is not surfaced in the job view:** The shared cron job type still carries optional `description`, and the controller hydrates it into form state for editing, but the job rendering code does not display it in the cron dashboard.
  - file: [src/cron/types-shared.ts:6](https://github.com/openclaw/openclaw/blob/68e2d6f0889e5365ecdc8f01ab5e3bf6065cbeeb/src/cron/types-shared.ts#L6)
  - command: `nl -ba src/cron/types-shared.ts | sed -n '1,20p'; nl -ba ui/src/ui/controllers/cron.ts | sed -n '442,470p'`

## Risks / Open Questions

- The issue may look partly addressed because prompts are now visible in the cron UI, but closing it would lose the remaining request for a human-readable summary/description surface and the requested click-to-reveal interaction.
