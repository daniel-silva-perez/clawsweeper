## Summary

Keep open. Current `main` has related UI work, but it does not implement the requested always-visible context usage indicator. The chat view only shows a warning pill once usage reaches 85%+, and docs still point users to `/status` or `/usage` for routine token/context visibility.

## Evidence

- **High-usage warning is conditional, not persistent:** `getContextNoticeViewModel` returns `null` unless `totalTokens/contextTokens` is at least `0.85`, so nothing is shown for normal usage levels. That does not match an always-visible indicator.
  - file: [ui/src/ui/chat/context-notice.ts:61](https://github.com/openclaw/openclaw/blob/40be5ad58187fd20b8defe5fc30e649190c131a8/ui/src/ui/chat/context-notice.ts#L61)

  - sha: [40be5ad58187](https://github.com/openclaw/openclaw/commit/40be5ad58187fd20b8defe5fc30e649190c131a8)
- **Test locks in thresholded behavior:** The UI test is named and asserted as "renders only for fresh high current usage" and expects no notice below the high-usage threshold.
  - file: [ui/src/ui/chat/context-notice.test.ts:24](https://github.com/openclaw/openclaw/blob/40be5ad58187fd20b8defe5fc30e649190c131a8/ui/src/ui/chat/context-notice.test.ts#L24)
