## Summary

Current `main` already implements the requested behavior: the shared `message` tool send schema does not require a `card`, Feishu accepts media-only sends, and the same behavior is present in the shipped `v2026.4.22` release.

## Close Comment

Closing after Codex review because current OpenClaw already implements the requested behavior, and the same implementation is present in shipped release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). The shared message-tool send schema has optional `message`, `media`, `path`, `filePath`, and `presentation` fields and does not require a `card` payload (`src/agents/tools/message-tool.ts:90`). On the Feishu side, `action=send` only builds a card when `presentation` is provided; otherwise media-only sends are accepted, and only the unsupported `card + media` combination is rejected (`extensions/feishu/src/channel.ts:695`). There is also an explicit regression test for a media-only Feishu send with no card (`extensions/feishu/src/channel.test.ts:803`). If you still see the old schema requirement, please open a fresh issue with the exact schema source and version that produced it, because the repository code and current release no longer match the reported failure.
