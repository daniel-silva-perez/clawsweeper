## Summary

Current `main` still lacks retry/backoff for the Telegram media download step under proxy/network failure. The code only retries `ctx.getFile()`, and the checked-in test suite still asserts that `fetchRemoteMedia` download failures are not retried.

## Evidence

- **`getFile()` is retried, but only that step:** `extensions/telegram/src/bot/delivery.resolve-media.ts:112` wraps `ctx.getFile()` in `retryAsync(...)` with 3 attempts and returns `null` after exhaustion.
  - file: [extensions/telegram/src/bot/delivery.resolve-media.ts:112](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/extensions/telegram/src/bot/delivery.resolve-media.ts#L112)
  - command: `nl -ba extensions/telegram/src/bot/delivery.resolve-media.ts | sed -n '112,140p'`
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Media download still happens as a single fetch:** `downloadAndSaveTelegramFile()` calls `fetchRemoteMedia(...)` directly at `extensions/telegram/src/bot/delivery.resolve-media.ts:235`, and both sticker/media paths call `downloadAndSaveTelegramFile()` without any surrounding retry wrapper at `:295` and `:404`.
  - file: [extensions/telegram/src/bot/delivery.resolve-media.ts:235](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/extensions/telegram/src/bot/delivery.resolve-media.ts#L235)
  - command: `nl -ba extensions/telegram/src/bot/delivery.resolve-media.ts | sed -n '188,245p' && nl -ba extensions/telegram/src/bot/delivery.resolve-media.ts | sed -n '289,304p' && nl -ba extensions/telegram/src/bot/delivery.resolve-media.ts | sed -n '397,414p'`
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Current test still encodes the bug:** `extensions/telegram/src/bot/delivery.resolve-media-retry.test.ts:232` is still named `does not catch errors from fetchRemoteMedia (only getFile is retried)` and expects the download failure to throw after a single `getFile()` success.
  - file: [extensions/telegram/src/bot/delivery.resolve-media-retry.test.ts:232](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/extensions/telegram/src/bot/delivery.resolve-media-retry.test.ts#L232)

  - file: [src/media/fetch.ts:127](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/media/fetch.ts#L127)
