## Summary

Current main already has a native post-compaction continuity path: OpenClaw injects a post-compaction system refresh for the next turn, re-injects configured AGENTS.md sections, and runs an automatic pre-compaction memory flush to preserve durable context.

## Close Comment

Closing from a Codex review: current OpenClaw already ships native post-compaction continuity recovery. After compaction, core enqueues a `[Post-compaction context refresh]` system event for the next turn (`src/auto-reply/reply/agent-runner.ts:1572`, `src/auto-reply/reply/post-compaction-context.ts:71`). That refresh re-injects configured `AGENTS.md` sections via `agents.defaults.compaction.postCompactionSections` (`docs/gateway/configuration-reference.md:1416`), and the injected guidance can point the agent back to the correct dated memory files (`src/auto-reply/reply/post-compaction-context.test.ts:246`). OpenClaw also runs an automatic pre-compaction memory flush to save important context before summarization (`docs/concepts/memory.md:105`). This behavior is present in the shipped `v2026.4.22` release. The exact `injectFile` JSON shape from this report did not land, but the requested post-compaction continuity mechanism is already implemented on main, so this issue is no longer actionable as an open core request.
