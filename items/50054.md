## Summary

Keep open. Current main does not implement the PR's distributed ACP session locking or documented Redis fail-closed configuration. The existing ACP manager serializes same-session turns with an in-process SessionActorQueue, and both normal ACP dispatch and `/acp steer` still call `acpManager.runTurn()` without a distributed/session-lock layer. The PR remains potentially useful, but its own review context includes unresolved correctness concerns that need maintainer review before merge.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a revised core ACP locking design that preserves queued turn semantics, covers both normal dispatch and `/acp steer`, handles Redis renewal/timeout failures deterministically, documents the explicit env vars, and adds regression tests for same-session concurrency across the affected ACP entrypoints.
