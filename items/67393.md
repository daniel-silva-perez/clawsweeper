## Summary

Keep open. The report is recent and describes an unresolved Gateway-startup VIRT/VSZ symptom on OpenClaw 2026.4.14 with normal RSS and a systemd MemoryMax workaround. Current main has adjacent memory improvements, but I found no source, test, changelog, release, or related-thread evidence that the 22GB startup virtual-memory mapping behavior is fixed or intentionally documented as benign.

## Best Possible Solution

Keep this issue open and ask for a focused current-main or latest-release reproduction: exact OpenClaw and Node versions, `ps` RSS/VSZ over time, `/proc/<pid>/smaps` or `pmap -x` summaries showing the anonymous mappings, enabled plugins/providers, whether `gateway --safe-mode` or minimal config changes the mapping count, and whether startup-only VIRT differs after the current Gateway/memory deferral changes. If the behavior is confirmed benign Linux/Node virtual address reservation, document that in the Linux/Gateway troubleshooting docs; if RSS/commit or OOM pressure grows, fix the concrete Gateway startup path or upstream dependency that owns the mappings.
