## Summary

Keep open. Current `main` still mis-resolves Windows-authored `sessionFile` values on POSIX, so this is neither implemented on main nor obsolete. The proposed PR also appears incomplete as written: its basename-only rewrite would conflict with the current cross-agent session-path behavior that `main` explicitly tests for.

## Evidence

- **Current POSIX path logic still treats Windows paths as relative:** `resolvePathWithinSessionsDir()` only gives absolute-path handling to candidates where `path.isAbsolute(trimmed)` is true, then returns `path.resolve(realBase, normalized)`. On POSIX, Windows drive paths do not satisfy `path.isAbsolute`, so they fall through as relative paths.
  - file: [src/config/sessions/paths.ts:186](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/src/config/sessions/paths.ts#L186)
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **Reproduction on current main matches the reported broken path shape:** Using Node's POSIX path rules with `sessionsDir=/home/node/.openclaw/agents/main/sessions` and candidate `C:\\Users\\foo\\sessions\\abc.jsonl` yields `/home/node/.openclaw/agents/main/sessions/C:\\Users\\foo\\sessions\\abc.jsonl`; `C:/Users/foo/sessions/abc.jsonl` similarly resolves under the POSIX sessions dir.
  - command: `node - <<'NODE' ... path.resolve(base, normalized) ... NODE`
- **Main currently enforces preserving real cross-agent absolute session paths:** Tests on `main` assert that a bot1 lookup can resolve and preserve a bot2 absolute session path, including cross-root paths. A basename-only rewrite into the current `sessionsDir` would violate that behavior for Windows-authored absolute paths.
  - file: [src/config/sessions.test.ts:607](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/src/config/sessions.test.ts#L607)
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **No current test coverage for Windows-authored session paths on POSIX:** The session path safety tests cover unsafe IDs, explicit dirs, symlink aliases/escapes, and POSIX absolute cross-agent paths, but there is no case for `C:\\...` or `C:/...` `sessionFile` inputs.
  - file: [src/config/sessions/sessions.test.ts:20](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/src/config/sessions/sessions.test.ts#L20)
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
