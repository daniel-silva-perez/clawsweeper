## Summary

Keep this issue open. Current main still leaves the model-facing `message.target` schema ambiguous for Discord DMs, and two open PRs with closing syntax are active implementation candidates rather than reasons to close before merge.

## Best Possible Solution

Keep this issue open and use the linked PRs as implementation candidates. The core fix should expose Discord/Slack/Mattermost `<channelId|user:ID|channel:ID>` syntax in the model-facing `message.target` schema, while the Discord-local error-message improvement can be reviewed as complementary retry guidance. After the accepted PR merges and CI/relevant tests are green, GitHub can close this issue as implemented.
