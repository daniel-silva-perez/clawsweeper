## Summary

Keep open. Current `main` fixes some sub-points from the report (bundled Discord voice dependencies and account-scoped voice config merging), but there is not strong evidence that the main bug report is resolved. The code still treats Discord voice receive breakage as an upstream `@discordjs/voice` problem, and the recovery logic only runs after explicit receive/decrypt errors, not for a fully silent pipeline with no speaking events.

## Evidence

- **Docs still describe the upstream receive bug as an active caveat:** `docs/channels/discord.md:1003` still says repeated `DecryptionFailed(UnencryptedWhenPassthroughDisabled)` logs may be the upstream Discord.js receive bug `discord.js#11419`, which is inconsistent with calling the issue fully fixed on current `main`.
