## Summary

Keep open. Current `main` has some of the requested building blocks already shipped, including daemon install during onboarding, `openclaw dashboard` auth URL handling, and a Telegram channel resolver. But there is still no single supported one-line/bootstrap path that non-interactively configures Telegram owner access end to end. The non-interactive `openclaw onboard` surface has no Telegram owner / DM policy flags, `openclaw channels add` is still credential-oriented rather than owner/bootstrap-oriented, and the Telegram docs still describe manual pairing approval or manual numeric `allowFrom` setup.
