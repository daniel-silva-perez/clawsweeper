## Summary

Keep open. Current main has follow-up build/runtime changes that make parts of the March report look stale, but I did not find strong evidence that the documented stock self-build path is fully fixed. The root Docker flow still defaults `OPENCLAW_EXTENSIONS` to empty, `scripts/docker/setup.sh` still forwards that empty value into `docker build`, and the Dockerfile still has no explicit post-`pnpm prune --prod` self-reference repair. With the review restricted to read-only inspection, that is not enough to close conservatively.

## Evidence

- **Dockerfile still defaults bundled-extension build arg to empty:** The root Dockerfile keeps `ARG OPENCLAW_EXTENSIONS=""`, and both the ext-deps stage and the runtime-assets prune stage iterate only over the provided values. There is no `__all__` fallback or empty-string normalization in the Dockerfile itself.
