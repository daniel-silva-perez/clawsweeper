## Summary

Current main does not implement declarative SKILL.md cron/event trigger metadata, last-triggered skill status, or a skills audit command. Existing plugin hooks expose partial cron access, but the requested first-class skills loader/CLI contract still needs maintainer product/API design rather than automated cleanup closure.

## Best Possible Solution

Keep the issue open for maintainer design. The best path is either a core, versioned skill trigger contract such as metadata.openclaw.triggers with validation, cron reconciliation, last-triggered status, CLI audit output, docs, and tests, or an explicit plugin-first design that first exposes enough public skill-loader/status API for a ClawHub plugin to implement this without reaching into core internals.
