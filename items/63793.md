## Summary

Keep PR #63793 open. Current main has the related /acp command-routing fix, but it does not implement this PR's separate ACP lifecycle behavior: fail-fast spawn health checks or stale gateway-disconnect recovery for close, cancel, active-turn cancel, and early-turn retry. The PR is recent, focused, and not superseded by #63656; it still needs maintainer review and should address the remaining status-content review concern before landing.

## Best Possible Solution

Keep this PR open for ACP/Gateway lifecycle maintainer review. The best path is to preserve the focused fix, ensure gateway-disconnect recovery is constrained to explicit stale close codes, apply the same `dead`/`no-session` status-content semantics used by cached-handle reuse to the spawn health probe, and land with the proposed manager and `/acp spawn` regression tests if review passes.
