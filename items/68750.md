## Summary

Keep open. Current main does not contain this PR's `prompt_cache_hit_tokens` normalization, and the linked MiniMax compaction bug remains open. The PR is narrowly scoped and does not introduce a security or supply-chain concern, but its functional rationale needs maintainer review because current `normalizeUsage()` would ignore an unknown `prompt_cache_hit_tokens` field rather than double-count it, and the PR has no regression test.

## Best Possible Solution

Keep this PR open for maintainer review rather than cleanup closure. The best path is to require a focused regression test in `src/agents/usage.test.ts` (or the relevant transport test) using a captured provider usage shape, confirm whether `prompt_cache_hit_tokens` is actually emitted by MiniMax or by another OpenAI-compatible provider, and then either land a generic alias normalization fix or redirect the linked issue to the real remaining compaction root cause.
