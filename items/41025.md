## Summary

Keep open. Current main still flattens remote markdown images to alt text, and the Control UI CSP/docs still intentionally block remote http(s) image fetches. The related PR #48993 is closed and unmerged, so it does not supersede this PR. The remaining work needs maintainer review because enabling remote inline images also changes Control UI privacy/network behavior.

## Best Possible Solution

Keep this PR open for maintainer review. If OpenClaw wants remote markdown images in core, the fix should cover the renderer, regression tests, CSP behavior, docs, and privacy mitigations such as `referrerpolicy`, lazy loading, or a managed/proxied image path. If maintainers decide remote image fetching should remain blocked, close after that explicit product/security decision and point users to the documented CSP boundary.
