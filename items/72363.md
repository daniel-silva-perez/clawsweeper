## Summary

Keep open. Current main still routes Telegram non-streaming turns with default tool-progress messages enabled, and the current test suite explicitly asserts that behavior. There is an open, unmerged PR #72482 with closing syntax for this issue, so it should remain open until that PR is reviewed and merged or rejected. The reasoning/thinking half also deserves verification alongside the tool-progress fix rather than being closed from existing partial mitigations.

## Best Possible Solution

Keep this issue open while maintainers review PR #72482 or an equivalent Telegram dispatch change. The best fix is to suppress default tool-progress delivery for non-streaming Telegram turns, add or update the regression test currently asserting the opposite behavior, and verify the GLM/MiniMax reasoning_content path with focused tests or live evidence so only the final assembled answer is delivered when preview streaming is off.
