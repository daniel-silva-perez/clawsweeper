## Summary

Keep open. Current main still leaves Telegram non-streaming turns with default tool-progress delivery enabled, the Telegram test suite asserts that behavior, and open PR #72482 is an unmerged implementation candidate with closing syntax for this issue. Existing reasoning safeguards are relevant but not enough to close the combined GLM/MiniMax report yet.

## Best Possible Solution

Keep this issue open while maintainers review #72482 or an equivalent Telegram dispatch change. The best fix is to suppress default tool-progress delivery for non-streaming Telegram turns, update the regression test currently asserting the opposite behavior, and add focused coverage or live proof for the GLM/MiniMax reasoning_content path so non-streaming Telegram sends only the final assembled answer.
