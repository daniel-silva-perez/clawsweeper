## Summary

Keep open. Current main still uses the OpenAI Realtime API-key fallback directly from process.env.OPENAI_API_KEY on both the browser client-secret path and server-side voice bridge path, and the current SecretRef helpers do not resolve a literal keychain:openclaw:OPENAI_API_KEY string. PR #72126 is a focused, active bug-fix PR for linked issue #72120 rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open. The best path is to land a narrow OpenAI Realtime fix, or an equivalent shared helper, that resolves supported keychain/API-key refs before both createBrowserSession and createBridge use the key, with regression coverage for OPENAI_API_KEY=keychain:openclaw:OPENAI_API_KEY and any final review concerns addressed.
