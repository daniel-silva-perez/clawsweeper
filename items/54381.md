## Summary

Keep open. Current `main` at `8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf` still caches each subsystem file logger on first use and never refreshes it after the root logger rotates to a new dated file, so the reported post-midnight rollover bug remains plausible in the current implementation.

## Evidence

- **Subsystem logger still caches child logger forever:** `createSubsystemLogger()` initializes `let fileLogger = null` and only calls `getChildLogger({ subsystem: resolvedSubsystem })` when that variable is empty. After that, both normal log emits and `raw()` reuse the same cached child without any date or settings revalidation.
  - file: [src/logging/subsystem.ts:327](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/logging/subsystem.ts#L327)
  - command: `nl -ba src/logging/subsystem.ts | sed -n '302,442p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Root logger rotates based on resolved file path:** `getLogger()` recomputes settings each call and rebuilds the root logger when `settingsChanged()` sees a different `file`, `level`, or `maxFileBytes`. The default file path is date-derived via `defaultRollingPathForToday()`, so root rotation is expected across local-date boundaries.
  - file: [src/logging/logger.ts:132](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/logging/logger.ts#L132)
  - command: `nl -ba src/logging/logger.ts | sed -n '132,245p'; nl -ba src/logging/logger.ts | sed -n '330,340p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Child loggers are bound to the current root instance:** `getChildLogger()` calls `getLogger()` once and returns `base.getSubLogger(...)`. Because `createSubsystemLogger()` caches that returned child, later root rebuilds do not automatically replace the subsystem child logger reference.
  - file: [src/logging/logger.ts:248](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/logging/logger.ts#L248)
  - command: `nl -ba src/logging/logger.ts | sed -n '248,259p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Docs still promise one file per day in local timezone:** Gateway logging docs say the default rolling file is `openclaw-YYYY-MM-DD.log`, one file per day, using the gateway host local timezone. That documented behavior conflicts with the current subsystem logger cache design above.
  - file: [docs/gateway/logging.md:20](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/docs/gateway/logging.md#L20)

The latest release tag also shows the same one-time subsystem child cache in `src/logging/subsystem.ts`.
