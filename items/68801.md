## Summary

Keep PR #68801 open. Current main still has the leak shape the PR describes: `agentRunStarts` is populated on lifecycle start and deleted only on lifecycle end/error, while `pruneAgentRunCache()` only sweeps `agentRunCache`. However, the submitted one-line TTL sweep is not clearly safe because `agentRunStarts` is also the fallback source for terminal lifecycle events that omit `startedAt`, and documented/default agent runs can last much longer than the 10-minute cache TTL.

## Best Possible Solution

Keep the PR open but request a safer revision: prune only starts that are known inactive or use a lifecycle-aware cleanup path, and add regression coverage for both orphan cleanup and an active run lasting beyond `AGENT_RUN_CACHE_TTL_MS` whose terminal event omits `startedAt`. The fix belongs in `src/gateway/server-methods/agent-job.ts` or a nearby gateway run-lifecycle seam, not as a blind TTL mirror of the terminal snapshot cache.
