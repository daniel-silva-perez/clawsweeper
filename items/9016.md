## Summary

Keep open. Current `main` still does not expose exact per-message OpenRouter `usage.cost` to agents. The runtime computes local estimated cost from token counts, `session_status` only returns a rendered status card with estimated cost text, `sessions_history` strips `usage`/`cost`, and the existing `usage.cost` RPC is gateway-only rather than an agent tool.

There is no code here reading provider-returned `usage.cost`, `cost_details`, or `upstream_inference_cost`.
