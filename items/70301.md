## Summary

Keep open. Current main already has the core `/bot-approve` auth-gating behavior, but PR #70301 still contains useful unmerged follow-up fixes from review: preserving `/bot-approve ?` behavior after moving through the framework path and preventing the QQBot-only command from being callable through other plugin command channels.

## Best Possible Solution

Keep the PR open or replace it with a narrower follow-up that preserves the already-shipped `requireAuth: true` behavior while adding the missing QQ-only framework guard, restoring `/bot-approve ?` help on the framework path, and extending QQBot command-auth tests to cover those cases. Once that follow-up lands, this PR can be closed as superseded or merged after rebase.
