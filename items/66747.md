## Summary

Keep this issue open. Current main still matches the reported macOS/package and registry-restore failure surfaces: the npm CLI bin remains a shebang-bearing `openclaw.mjs`, the source entrypoint also has a shebang and packaged `dist/entry.js` remains part of the gateway argv surface, there is no provenance/xattr diagnostic or alternate shebang-free entrypoint, and both task registries still call `chmodSync` directly during restore/open paths.

## Best Possible Solution

Keep open. The best maintainer path is to fix or explicitly diagnose macOS 26/npm provenance failures in the packaged CLI and LaunchAgent gateway path, make task and task-flow registry permission tightening idempotent or tolerant of benign `chmod` EPERM when ownership/mode are already acceptable, and add macOS/package acceptance coverage for an npm-installed `openclaw gateway probe`, `openclaw status`, and registry restore startup.
