## Summary

Keep this issue open. Current main still matches the reported failure surfaces: the published CLI bin is still the shebang-bearing `openclaw.mjs`, there is no `com.apple.provenance`/macOS 26 detection or alternate shebang-free entrypoint, and both task registries still run unconditional `chmodSync` during restore/open. The provided related closed reports cover different gateway handshake/LaunchAgent issues and do not supersede this macOS provenance/chmod report.

## Best Possible Solution

Keep open. The best path is to fix or explicitly diagnose macOS 26/npm provenance startup failures in the packaged CLI or LaunchAgent path, make task and task-flow registry permission tightening tolerant of benign `chmod` EPERM when ownership/mode are already correct, and add a macOS/package acceptance regression covering `openclaw gateway probe`, `openclaw status`, and registry restore on an npm-installed build.
