## Summary

Keep open. Current main has partial restart recovery, but it does not implement #64585's requested generic startup scan/replay for stuck gateway chat sessions. Existing recovery is limited to queued deliveries/restart continuations and `abortedLastRun` main sessions from stale transcript-lock evidence, and it still fails assistant-tail cases instead of replaying through `sessions.send`.

## Best Possible Solution

Keep #64585 open. The best fix is a dedicated, capped startup recovery pass that enumerates gateway session stores newest-first, uses existing visible-text/tool-call transcript helpers to identify unanswered user tails and empty assistant tails, skips internal/non-chat channels, preserves delivery context, and replays safely through the `sessions.send`/gateway handler pipeline with focused detection and selection tests. Treat the current restart-sentinel, cron, subagent, and stale-lock main-session recovery paths as useful but narrower partial fixes.
