## Summary

Keep open. Current `main` only partially overlaps this umbrella feature request: OpenClaw already has telemetry/export, onboarding/config wizards, partial UI localization, and extensive CI workflows, but key requested items like auto-tuning, a YAML policy engine, sandboxed native plugins with atomic upgrades, and serverless/FaaS extensibility are not implemented on `main`. This also does not fit a pure `clawhub` close, because the request is mostly about core product/runtime/platform behavior rather than a standalone skill/plugin.
