## Summary

Keep open. Current `main` does not implement a cron `payload.kind` for direct shell/exec execution; the cron contract, tool docs, validation, and runtime still only support `systemEvent` and `agentTurn` payloads.

## Evidence

- **Cron payload type is still a two-variant union:** `CronPayload` only allows `{ kind: "systemEvent" }` or `{ kind: "agentTurn" }`; there is no `exec` or `shell` variant.
  - file: [src/cron/types.ts:112](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/cron/types.ts#L112)

- **User-facing cron tool docs list only two payload kinds:** The built-in cron tool help documents only `systemEvent` and `agentTurn` under `PAYLOAD TYPES (payload.kind)` and ties `main` to `systemEvent` and isolated/current/session targets to `agentTurn`.
  - file: [src/agents/tools/cron-tool.ts:466](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/agents/tools/cron-tool.ts#L466)
