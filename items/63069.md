## Summary

Keep #63069 open. Current main still contains the reported completion profile path behavior: zsh and fish completion install/check paths are hardcoded under HOME, `ZDOTDIR` and `XDG_CONFIG_HOME` are not consulted, and bash install can write `.bash_profile` while the installed check only reads `.bashrc`. The linked PR #63082 is still the canonical open fix candidate in the provided GitHub context, not merged or shipped on current main.

## Best Possible Solution

Keep #63069 open and use #63082 or an equivalent maintainer-reviewed patch as the implementation path. The fix should centralize shell profile resolution, respect `${ZDOTDIR:-$HOME}/.zshrc` for zsh and `${XDG_CONFIG_HOME:-$HOME/.config}/fish/config.fish` for fish, make bash check/install use the same `.bashrc`/`.bash_profile` fallback, update doctor/setup hints, and add focused tests for those path-resolution cases.
