## Summary

Keep this issue open. Current main still hardcodes QQ Bot media SSRF hosts and does not expose or wire a `channels.qqbot.mediaAllowHosts` setting. The existing Codex review comment remains accurate against `e962381dbf1ecdb1cd4784fdeb6c2e07ddd3435b`: the reporter's example host is now covered by the fixed `*.qq.com.cn` default, but the requested user-configurable allowlist is still missing.

## Best Possible Solution

Implement this in the bundled QQ Bot plugin: add `mediaAllowHosts` to the QQ Bot account/channel config type, Zod schema, plugin manifest schema, docs, and generated metadata; normalize the configured hosts with the existing hostname allowlist helpers; merge them with the current conservative QQ/Tencent defaults before inbound and fallback media downloads; and add focused tests for default hosts, custom host merging, HTTPS-only behavior, and continued rejection of unsafe/private-network fetches.
