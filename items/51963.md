## Summary

Keep PR #51963 open. Current main still does not ship Telegram inline-query support, so the PR is not obsolete or implemented. However, the PR also should not be auto-closed as duplicate/superseded because it contains a substantial implementation attempt for the same missing feature tracked by #54794, and the provided review context shows concrete unresolved correctness/security concerns that need maintainer or author follow-up before any merge path.

## Best Possible Solution

Keep #51963 open as a candidate implementation for the still-missing Telegram inline-query feature, or replace it with a narrower maintainer-owned PR. The right implementation belongs in the Telegram plugin: add inline_query and chosen_inline_result handlers, preserve access/DM policy and dedupe semantics, avoid prompt/file-path injection, cover auto-edit/concurrency cleanup with tests, update ChatType/adapter contracts only if still needed, and document the user-facing @botname inline-query flow. Track the remaining product request through #54794 until a safe implementation lands.
