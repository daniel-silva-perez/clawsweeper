## Summary

Keep this issue open. Current main supports explicit `models.providers.<provider>.request.proxy.mode = "env-proxy"`, but it does not implement the requested startup warning when proxy env vars are present and remote providers have no request proxy configured. The exact implementation PR #68023 was closed unmerged, so there is no merged/superseding fix to close against.

## Best Possible Solution

Implement the non-breaking warning in or near `logGatewayStartup`: detect configured proxy env vars with the existing proxy-env helpers, inspect configured model providers for remote HTTP(S) base URLs, skip local/loopback/.local/private-only cases as intended, and warn once when a remote provider lacks explicit `request.proxy`. Add focused startup-log tests for warn, no env, all configured, and local-provider cases, using PR #68023 as useful prior design context but reviewing the closed patch rather than assuming it is current.
