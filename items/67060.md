## Summary

Keep this issue open. Current main still supports the explicit provider `request.proxy.mode = "env-proxy"` workaround, but it does not implement the requested startup warning when proxy environment variables are present and remote model providers have no request proxy configured. The linked implementation candidate, PR #68023, is closed and unmerged in the provided GitHub context.

## Best Possible Solution

Implement a non-breaking startup warning in or near `logGatewayStartup`: reuse the existing proxy-env helpers to detect configured proxy variables, inspect configured model providers for remote HTTP(S) targets, skip local/private/non-remote or explicitly configured `request.proxy` cases, and warn once with the `models.providers.<provider>.request.proxy.mode = "env-proxy"` remediation. Add focused startup-log tests for warn, no-env, all-configured, and local-provider cases, using the closed PR #68023 as prior design context rather than as a merged fix.

## Risks / Open Questions
