## Summary

Keep open. Current main at 8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf does not contain this PR's fix: OpenRouter onboarding still forces `openrouter/auto`, `openrouter/free` is still missing from the bundled OpenRouter catalog, and the shared OpenRouter reasoning wrapper still only special-cases `auto`.

## Evidence

- **Onboarding still overwrites primary model to auto:** `applyOpenrouterConfig` unconditionally calls `applyAgentDefaultModelPrimary(..., OPENROUTER_DEFAULT_MODEL_REF)`, and `OPENROUTER_DEFAULT_MODEL_REF` is `openrouter/auto`. There is no logic to preserve an existing `openrouter/*` primary such as `openrouter/free`.
