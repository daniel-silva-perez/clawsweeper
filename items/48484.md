## Summary

Current `main` exposes only normalized provider usage summaries in `/status` and `session_status`, not the raw Anthropic rate-limit header fields requested in #48484. I found no code or docs on `main` that reference `anthropic-ratelimit-*`, `requests-remaining`, `tokens-remaining`, or the corresponding reset headers.

## Evidence

- **No header support on main:** Repository search found no references to the requested Anthropic rate-limit header names anywhere under `src`, `extensions`, or `docs`. That is strong evidence the raw headers are not currently surfaced.
  - command: `rg -n "anthropic-ratelimit|requests-remaining|tokens-remaining|requests-reset|tokens-reset" src extensions docs`
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)

  - file: [src/status/status-text.ts:192](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/status/status-text.ts#L192)
