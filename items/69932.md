## Summary

Keep PR #69932 open. Current main still has unguarded BlueBubbles monitor typing start, restart, and stop calls, and current monitor tests do not cover the Private-API-disabled path. The related #65447 work covers outbound send/attachment Private API cache refresh, not these inbound reply typing callbacks. The PR is recent, focused, and the latest commit addresses the bot review’s requested disabled-path test and stop-typing guard.

## Best Possible Solution

Keep the PR open for maintainer review. The likely path is to land a focused monitor-level typing guard plus the disabled-path regression test if maintainers accept suppressing typing while Private API status is unknown; otherwise adjust the patch to use a tri-state or lazy-probe approach that skips explicit `false` without regressing typing on transient cache misses.
