## Summary

Keep this PR open. Current main still does not contain the proposed post-publish macOS asset verifier or the appcast CODEOWNERS protection, and the missing-asset failure mode is real. The patch is a plausible implementation candidate, but it needs maintainer release-owner review and should address the remaining polling/API-error review comments before merge.

## Best Possible Solution

Keep the PR open for maintainer review. If maintainers want this guard, revise the workflow to fail fast on non-retryable GitHub API errors, recheck before timeout, and decide whether appcast absence should warn or fail; then align the CODEOWNERS entry with the intended release-owner policy and link the work to the open macOS missing-asset tracker #48306 if appropriate.
