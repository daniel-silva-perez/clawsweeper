## Summary

Keep #70953 open. Current main still implements and documents exec approval as an async `approval-pending` result: the tool returns after request registration, while gateway/node execution waits for the approval in a detached follow-up path. That is the behavior the issue asks to change, not an already-shipped fix or duplicate cleanup candidate.

## Best Possible Solution

Keep the issue open and have maintainers decide the desired exec approval contract. A concrete fix would likely move foreground gateway/node exec approval handling from detached follow-up execution to an awaited approval decision path with timeout behavior, while preserving or explicitly exempting background exec approvals and updating docs/tests that currently promise immediate `approval-pending` returns.
