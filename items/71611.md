## Summary

Keep #71611 open. Current main still lacks the requested Windows EBUSY retry/copy fallback in memory atomic reindex, while the PR is a relevant but unmergeable attempt because its patch breaks template literals and sidecar/backup path handling.

## Best Possible Solution

Keep the PR open for revision or replace it with a clean maintainer patch: port the cron `renameWithRetry` behavior into `extensions/memory-core/src/memory/manager-atomic-reindex.ts`, preserve suffix handling and the `randomUUID()` backup path, add focused regression tests for EBUSY retry and EPERM/EEXIST fallback, and keep #64187 as the canonical bug until the corrected implementation lands.
