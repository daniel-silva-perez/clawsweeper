## Summary

Keep open. Current main has overlapping work, especially plugin-owned conversation resolution and Telegram current binding coverage, but this PR is not safely obsolete: native ACP `sessions_spawn(runtime:"acp", thread:true, mode:"session")` still uses a channel-level default placement instead of the same context-sensitive placement path as `/acp spawn`, and the native current-bind path lacks the slash command's sender ownership guard. The PR also has unresolved P1 review findings in the provided discussion, so it should be reworked or superseded by narrower maintainer-reviewed fixes rather than auto-closed.

## Best Possible Solution

Keep the PR open for maintainer review, but ask for a rebase or narrower replacement. The best fix path is to preserve current main's plugin-owned conversation-resolution approach, make native `sessions_spawn` share the `/acp spawn` context-sensitive placement and sender-ownership rules, add focused regression coverage for Discord DM, existing Discord thread, Telegram topic/DM, Feishu topic/session-key announce routing, and avoid reintroducing the Feishu action-handler regression from the stale PR diff.
