## Summary

Keep this PR open. Current main does not include the Memory v2 sidecar, ingest wiring, rerank wrapper, sidecar CLI, or `memoryV2.*` plugin config proposed here. Existing QMD/MMR/temporal-decay work overlaps in the broad ranking/search area, but it does not implement this PR's per-location SQLite metadata, pin/salience/supersession signals, or default-off `memory-core` sidecar foundation. The PR is also large and has substantive review concerns, so the best maintainer path is review/split/fix rather than auto-close.

## Best Possible Solution

Keep this PR open for explicit maintainer product review. The strongest path is to split the branch into smaller reviewable slices, resolve the outstanding review comments, and decide whether Memory v2 belongs inside bundled `memory-core` or should be expressed as a separate memory plugin/API extension. Do not close it as implemented because current main lacks the proposed Memory v2 sidecar, config, ingest, rerank, and CLI surfaces.
