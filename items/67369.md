## Summary

Keep open. Current main partially satisfies this request with `openclaw sessions --agent`, `--all-agents`, `--active <minutes>`, and `--json`, but the requested age-query and size-query surface is still missing: no `--older-than`, no duration-style `--newer-than`, no `--larger-than`, and no session row byte-size metadata to filter on.

## Best Possible Solution

Keep this issue open and implement the remaining core CLI/protocol work deliberately: add duration parser-backed `--older-than` and `--newer-than` filters, define whether `--larger-than` means transcript bytes, session-store bytes, or token count, expose the chosen size metadata consistently in CLI JSON, gateway `sessions.list`, and `sessions_list` rows, then update `docs/cli/sessions.md` plus focused parser/command/protocol tests.

- Session cleanup and disk-budget maintenance paths address retention/pruning, not queryable session metadata, so they should not be treated as an equivalent shipped solution.
