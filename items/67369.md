## Summary

Keep open. Current main partially covers the request with `openclaw sessions --agent`, `--all-agents`, `--active <minutes>`, and `--json`, but the requested stale-session and size-query surface is still missing: there is no `--older-than`, no duration-style `--newer-than`, no `--larger-than`, and no session row size/byte metadata to filter on.

## Best Possible Solution

Keep this issue open and implement the remaining core CLI/API work deliberately: add duration parser-backed `--older-than`/`--newer-than` filters, decide whether `--larger-than` means transcript bytes, session-store bytes, or token count, expose the needed size metadata in JSON/gateway rows, and update `docs/cli/sessions.md` plus focused command/parser tests.
