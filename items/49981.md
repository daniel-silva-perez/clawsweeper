## Summary

Keep open. Current main still does not implement this fix: the relocated memory host HTTP helpers do not accept or forward `timeoutMs`, provider batch submit/status/download paths still call those helpers without per-request budgets, and batch timeout retry still only matches timeout text instead of `AbortError`. The PR’s idea remains valid, but it needs a rebase onto the current file layout.

## Evidence

  - file: [src/memory-host-sdk/host/remote-http.ts:6](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/src/memory-host-sdk/host/remote-http.ts#L6)
