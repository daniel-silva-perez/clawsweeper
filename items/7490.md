## Summary

Keep #7490 open. Current main still does not implement the requested optional per-agent `description` field: `AgentConfig` has no `description`, the strict runtime and generated schemas omit and reject it, `agents_list` only returns id/name/configured/model/runtime metadata, and the linked PR #7530 was closed unmerged rather than shipped. A maintainer review comment on the issue also explicitly kept it open with the same current-behavior rationale.

## Best Possible Solution

Keep #7490 open until OpenClaw either implements the additive core config/tool behavior or a maintainer explicitly rejects the product surface. A complete fix should add `description?: string` to `AgentConfig`, accept and document it in `AgentEntrySchema`, generated schema metadata/help/docs, preserve strict schema behavior for other unknown keys, include a trimmed optional `description` in `agents_list` responses for configured agents, and add focused tests for inclusion and omission.
