## Summary

Keep #71537 open. Current main partially handles reset archives for an empty live transcript, but it still does not cover the PR's remaining archive-only lookup path for the session-memory hook, and the session-logs skill still documents active `*.jsonl` searches that skip reset/deleted archives. The provided #60409 cross-reference is complementary rather than superseding because it targets gateway history/RPC behavior, not the bundled hook or skill docs.

## Best Possible Solution

Keep the PR open. The best path is to rebase it on current main, keep the scoped session-memory archive-only fallback and session-logs archived transcript guidance, resolve or split the unrelated lockfile hunk, and leave gateway history fallback work to the separate #60409 track.
