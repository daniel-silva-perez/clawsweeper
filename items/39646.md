## Summary

Keep open. Current main still contains the raw default-model status behavior that PR #39646 targets, and the review-noted gaps are still present in the status command paths.

## Best Possible Solution

Keep this PR open or replace it with a narrower maintainer patch. The best fix is to centralize provider-validating default-model resolution for `models status` JSON/rich output, probe candidates, and `openclaw status` session defaults, then add regression tests for a removed provider with a resolved fallback model. Split or drop the unrelated bundled changes before merge.
