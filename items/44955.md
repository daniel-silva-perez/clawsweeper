## Summary

Keep open. Current `main` does not implement the PR's provider-level `contextWindow` precedence, and the remaining inline-provider path still drops provider-level metadata entirely. The current docs/schema also steer users toward per-model `contextTokens`, so this likely needs a product/contract decision rather than conservative closure.

## Evidence

- **Registry merge path still ignores provider-level contextWindow:** In `applyConfiguredProviderOverrides`, resolved models still use `metadataOverrideModel?.contextWindow ?? discoveredModel.contextWindow`; there is no `providerConfig.contextWindow` fallback on current `main`. That means the PR's core behavior is not already implemented.
