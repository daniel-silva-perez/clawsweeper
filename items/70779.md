## Summary

Keep open. The issue is recent, has concrete logs, and current main still unconditionally sends `stream_options.include_usage` on the OpenAI-compatible completions stream path, including generic/custom providers. That matches the reporter's suspected regression area rather than proving the issue fixed or obsolete.

## Best Possible Solution

Keep the issue open and treat it as a focused regression candidate. The likely maintainer path is to make `buildOpenAICompletionsParams` honor `compat.supportsUsageInStreaming === false` before sending `stream_options.include_usage`, add transport tests for generic custom/OpenAI-compatible Gemini-style providers and explicit opt-out, and document/configure a workaround if some custom backends need include-usage disabled.
