## Summary

Keep this PR open. Current main has related session maintenance, missing-transcript cleanup, doctor orphan archival, and pressure-gated disk-budget cleanup, but it still lacks the proposed reusable orphan-transcript pruning primitive and an ordinary on-demand cleanup path that scans already-unindexed JSONL transcripts and unlinks them. The PR is recent, scoped to session maintenance plus tests, and the discussion shows active author follow-up on review feedback.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is either to merge the tested primitive as a staged building block and follow with explicit openclaw sessions cleanup wiring, or ask for a narrower follow-up that implements the full cleanup command directly while collecting preserved paths from all relevant stores, resolveSessionTranscriptCandidates, topic transcript derivations, and compaction checkpoint metadata.
