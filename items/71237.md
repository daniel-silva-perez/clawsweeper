## Summary

Keep PR #71237 open. Current main has related cleanup paths, but it does not contain the proposed `pruneOrphanedTranscripts` utility or an equivalent unconditional orphan-transcript sweep in `sessions cleanup`. Existing behavior archives transcripts removed by store maintenance, can archive detected orphans through `doctor`, and can remove orphan-like artifacts only when the optional disk budget is configured and exceeded. The PR remains a focused, recent core maintenance improvement with useful field data and addressed review feedback.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a safe, tested core maintenance primitive only if its preservation contract is strong, then wire a follow-up `openclaw sessions cleanup` path that builds the union of live transcript paths across store entries, custom `sessionFile` values, topic transcripts, and compaction checkpoints.
