## Summary

Current main does not implement the reported fix. The sandbox backend registry is still a module-scoped Map, while plugin registration can happen through the public plugin SDK and sandbox creation later calls requireSandboxBackendFactory on the core path. The linked PR #67623 is still open and #67628 was closed unmerged, so the bug should remain tracked here.

## Best Possible Solution

Keep #67610 open until main changes the sandbox backend registry to a process-wide shared registry or otherwise guarantees plugin SDK and core reads use the same registry instance, with a regression test that simulates plugin/Jiti cross-module registration. #67623 or a successor PR should close it once merged.
