## Summary

Keep open. Current main still sends the planned backup source paths directly to `tar.c(...)` without an ENOENT skip/retry path for files that disappear during archiving, and the linked implementation PR #67472 is still open and explicitly intended to fix this issue. Per the cleanup rules, the issue should remain open until that PR or a superseding fix lands or is closed.

## Best Possible Solution

Keep this issue open and review #67472, or fold the same deleted-file ENOENT handling into a broader backup archiver fix such as #72251. The implementation should live in `src/infra/backup-create.ts` and include a regression test proving `openclaw backup create` succeeds when a planned session transcript disappears during archive creation.
