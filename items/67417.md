## Summary

Keep open. Current main still routes `openclaw backup create` through a direct `tar.c(...)` archive call with no ENOENT skip/retry path for files deleted during archival, and an open implementation PR (#67472) explicitly links itself as the closing fix for this issue. Per maintainer cleanup rules, the issue should stay open until that PR or a superseding fix is merged or closed.

## Best Possible Solution

Keep this issue open and review #67472, or fold its ENOENT deleted-file handling into a broader backup archiver fix such as the related volatile-file/retry work in #72251. The code path that likely needs the fix is `src/infra/backup-create.ts`, with a regression test proving `openclaw backup create` succeeds when an already-planned session file disappears during archival.
