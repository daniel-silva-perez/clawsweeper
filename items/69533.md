## Summary

Keep #69533 open. Current main still hardcodes the Dream Diary narrative system prompt and prompt builder inside memory-core, the memory-core dreaming config/schema/docs expose scheduling and phase tuning but no narrative prompt/template field, and the Control UI only toggles/reads/repairs/backfills dreaming rather than editing prompt text. The provided related title-search items are broader system-prompt or prompt-cache reports and do not supersede this dream-specific customization request.

## Best Possible Solution

Implement this in the memory-core plugin, not as a generic core system-prompt override: add a typed and schema-validated dreaming narrative prompt/template config under plugins.entries.memory-core.config.dreaming, define supported placeholders and fallback behavior, thread it into buildNarrativePrompt/startNarrativeRunOrFallback, add focused tests for default compatibility and placeholder rendering, and expose it in the Dreaming settings UI plus docs.
