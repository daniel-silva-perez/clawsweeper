## Summary

Keep PR #70851 open. Current main still has the reported chat-picker gap: the picker filters global, unknown, and cron rows, but not subagent, ACP, node-backed, or spawn-depth internal runtime sessions. Gateway rows already carry some internal-session signals such as `spawnDepth`, but there is no explicit `visibility=internal` row/API contract on main, and the UI-side row type and session event updater are not wired for that signal. Related #69552 is a narrower closed/unmerged attempt, not a shipped canonical fix; #64121 is adjacent picker UX work, not a replacement for this internal-session classification bug.

## Best Possible Solution

Keep the PR open, but steer it toward the product boundary described in its body: add an explicit gateway/session-row internal visibility classification for subagent, ACP, and runtime/node-backed sessions; expose a list/query contract that lets normal chat pickers hide `visibility=internal` by default; keep the currently active internal session visible so the UI is not stranded; preserve admin/debug Sessions visibility through an explicit opt-in; and add focused tests for subagent, ACP, node-backed, `spawnDepth`-only, active-internal, and false-positive flexible-key cases.
