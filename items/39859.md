## Summary

Current main still does not implement PR #39859's central fix. `resolveContextTokensForModel()` returns a positive override before looking up any known model context window, so status and session metadata paths can still over-report effective context when an override is larger than the selected model's known limit. The provider-alias lookup concern from the PR discussion appears addressed on main, but the override-capping behavior remains missing.

## Best Possible Solution

Keep this PR or its linked issue #39857 open until main implements the narrow resolver behavior: resolve the known model context window first, then return `Math.min(contextTokensOverride, modelTokens)` when both values are known, while preserving existing override behavior for unknown model windows. Add focused regression coverage in `src/agents/context.test.ts` or `src/agents/context.lookup.test.ts` for larger-override capping, lower-override preservation, and configured provider alias lookup.
