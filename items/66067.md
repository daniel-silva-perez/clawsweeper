## Summary

Keep this PR open. Current main still has the TUI thinking-spacing behavior described by the linked bug #66008: thinking fragments are joined with single newlines and then rendered through the TUI Markdown path, while the current tests still expect newline-joined thinking. The PR remains a focused implementation candidate, with an actionable CRLF review follow-up rather than evidence that the work is obsolete.

## Best Possible Solution

Keep this PR open as the implementation candidate for the TUI show-thinking whitespace bug. The maintainer path is to update normalizeThinkingWhitespace to normalize CRLF/bare CR safely, add the requested regression test, keep the formatter and stream-assembler spacing regressions, run the targeted TUI tests, and then review or merge if CI is clean.
