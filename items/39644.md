## Summary

Current main still has the two Windows behaviors this PR targets. PowerShell is listed as a supported completion shell and documented, but completion installation still falls through to the unsupported branch and the profile source helper still emits bash-style source syntax. Windows time-format detection also still invokes literal powershell instead of the existing resolvePowerShellPath() helper that prefers PowerShell 7 / pwsh. The PR discussion and follow-up commits look useful; the patch needs a rebase/manual port from the older completion-cli.ts layout to current completion-runtime.ts, not closure.

## Best Possible Solution

Keep this PR open and port it to current main: add the PowerShell install branch in src/cli/completion-runtime.ts, emit PowerShell dot-source syntax and a correct reload hint, switch src/agents/date-time.ts to resolvePowerShellPath() with non-interactive flags, and add regression tests for PowerShell install/idempotence plus the time-format command path.
