## Summary

Keep open. Current main has version-aware bundled runtime dependency repair for the Gateway/doctor path, but the postinstall eager-install path targeted by this PR still treats an existing package sentinel as sufficient unless an optional native child is missing. The PR is focused and linked to a concrete regression, but it should be revised before landing to reuse the existing semver-backed behavior and address the provided review comments.

## Best Possible Solution

Keep the PR open and revise it narrowly: add postinstall stale-version detection using the same semver-backed satisfaction semantics as src/plugins/bundled-runtime-deps.ts, add the stale-version regression test, update the @snazzah/davey fixture so optional-native-child coverage still exercises that path, and drop/rebase any already-landed lockfile-only drift before running the changed script/test gate.
