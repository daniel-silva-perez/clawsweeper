## Summary

Keep #49796 open. Current main still canonicalizes symlinked session paths to the real filesystem target in the production resolver, and the PR's logical-path preservation plus legacy real-path remapping tests are not present on main. The linked bug #49708 remains a real product issue at least for stored path portability, even if doctor has since added canonical comparison logic for some orphan checks.

## Best Possible Solution

Keep the PR or linked issue active. The best implementation is to preserve logical caller-visible session paths while retaining realpath-based containment checks, normalize legacy real stored paths back under the logical sessions directory, and add focused regressions for symlinked session dirs, sibling-agent fallback, and doctor orphan detection.
