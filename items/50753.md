## Summary

Issue #50753 is a useful idea, but current OpenClaw project scope points this kind of operational wrapper toward an external plugin/skill package rather than new core CLI surfaces.

## Evidence

- **Vision keeps optional capability out of core:** `VISION.md:52-60` says OpenClaw has an extensive plugin API, that core should stay lean, optional capability should usually ship as plugins, and the bar for adding optional plugins to core is intentionally high.

## Close Comment

Closing this as a `clawhub`-type request after Codex review. The idea is useful, but current project scope points it away from OpenClaw core: `VISION.md:52-60` says optional capability should usually ship as plugins, `VISION.md:66-70` / `VISION.md:99-107` push new skills toward ClawHub rather than core, and `docs/tools/index.md:19-45` documents plugins as the supported path for adding extra tools/skills. Current main already has the lower-level primitives this would build on, including gateway health snapshots/monitoring in `docs/gateway/health.md:15-39` and cron failure destinations in `docs/automation/cron-jobs.md:145-150`. That makes AlertPipe / HealthPipe a better fit as a ClawHub skill or external plugin package layered on those surfaces, rather than a new core CLI feature.
