## Summary

Current `main` still lacks built-in security scanning for ClawHub skill folder installs and updates. The existing built-in dangerous-code scan applies to the separate gateway installer-metadata path (`skills.install` for install specs), while `openclaw skills install` / ClawHub-source installs download and copy skill folders without invoking that scan. This is a real, still-open feature gap rather than an already-implemented or out-of-scope item.

## Evidence

- **Docs explicitly say native ClawHub skill installs are not on the scanned path:** `docs/tools/skills.md:136-137` says gateway-backed `skills.install` runs the built-in dangerous-code scanner, and then immediately states that `openclaw skills install <slug>` is different and does not use that installer-metadata path.
