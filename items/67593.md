## Summary

Keep this issue open. Current main still does not surface Kimi/Moonshot provider usage: the default usage provider list and `UsageProviderId` union exclude `kimi`/`kimi-coding`, the bundled Kimi plugin registers auth/catalog/runtime behavior but no usage auth or usage snapshot hooks, and user-facing usage docs list the supported providers without Kimi. The linked prior PR was closed unmerged, so it does not supersede or implement the request.

## Best Possible Solution

Keep this issue open and implement the feature in the bundled Kimi provider boundary: add Kimi/Moonshot usage auth and snapshot fetching via the existing `resolveUsageAuth` and `fetchUsageSnapshot` provider hooks, extend or generalize provider-usage discovery so `kimi`/`kimi-coding` are queried by `/status`, Control UI, and `openclaw status --usage`, add regression tests and docs, and treat the old `/quota` wording as a product decision because current main documents provider quota under `/status`/usage surfaces rather than a separate `/quota` command.
