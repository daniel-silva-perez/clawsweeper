## Summary

Keep #71592 open. Current main still advertises `/status` and `/compact` in the TUI command surface, but the TUI command switch has no `status` or `compact` handler and its default path sends the raw slash text as a chat message. In `openclaw tui --local`, that goes through `EmbeddedTuiBackend` and `agentCommandFromIngress`, not the Gateway `chat.send` command-dispatch path. The existing related items are adjacent but do not supersede this local embedded TUI fallthrough bug.

## Best Possible Solution

Keep the issue open and fix the local embedded TUI command boundary. Recognized shared commands such as `/status` and `/compact` should either execute through the same shared command dispatcher/adapter in local mode or return an explicit local unsupported/system response, but they should not fall through to `sendMessage(raw)`. Add focused regression tests proving local `/status` and `/compact` do not reach `sendChat` / `agentCommandFromIngress`, while preserving Gateway-mode forwarding for commands intentionally handled by `chat.send`.

## Risks / Open Questions

- A broad fix that blocks all unknown slash commands in the TUI could regress intended Gateway-mode forwarding for shared commands such as `/context`; the fix should distinguish local embedded mode from Gateway-connected mode.
- `/compact` may need a shared local command adapter rather than a trivial TUI-only case, because the existing shared handler updates session state and emits system events.
