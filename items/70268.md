## Summary

Keep open. Current main still drops explicit wake-hook session targeting and routes wake events to the configured main session, while PR #70268 is an active focused fix for open bug #64556. The related #35231 is closed unmerged and does not supersede this narrower PR.

## Best Possible Solution

Keep PR #70268 open. The best path is to finish maintainer review, ensure direct and mapped wake hooks carry `sessionKey`/`agentId` through normalization, prefix policy, canonicalization, enqueue, and heartbeat targeting, then add or keep regression coverage for both direct wake and mapped wake routes, including the global-scope edge cases raised in review.
