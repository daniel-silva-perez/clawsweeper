## Summary

Keep this issue open. Current main still sends any truthy OpenAI-completions `choice.delta.content` through the text append path without a runtime string/array guard, so the Mistral typed-block array shape described in the issue can still be coerced into `[object Object]`. There is also an open unmerged PR, #68418, with closing syntax for this issue, so the conservative maintainer path is to review/land or supersede that PR rather than close the issue now.

## Best Possible Solution

Keep this issue open until #68418 is reviewed and either landed or deliberately superseded. The likely fix belongs in `src/agents/openai-transport-stream.ts`: add a runtime unpacker for OpenAI-compatible typed-block `delta.content` arrays, route `{type:"text"}` blocks through the text path and `{type:"thinking"}` blocks through the thinking path, preserve the existing string fast path, and add a regression test in `src/agents/openai-transport-stream.test.ts` that proves no `[object Object]` appears in assembled text or streamed deltas.
