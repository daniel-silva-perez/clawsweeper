## Summary

Current main still leaves local-managed browser profiles on the raw CDP tab-open path, while only remote CDP profiles use the persistent Playwright path that performs redirect-hop inspection. The related sandbox SSRF propagation reports are addressed separately, but this issue's requested local-managed Playwright interception remains unimplemented.

## Best Possible Solution

Keep this issue open and implement a focused browser-plugin change in `extensions/browser`: make local-managed OpenClaw profiles use the existing Playwright-backed `createPageViaPlaywright`/`gotoPageWithNavigationGuard` path when strict SSRF mode requires redirect-hop inspection, while preserving Chrome MCP existing-session behavior and safe raw-CDP fallbacks where Playwright is unavailable. Add local-managed strict-mode regression tests for hostname and IP-literal network navigation, and update browser docs if the profile capability matrix changes.
