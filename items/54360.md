## Summary

Keep open. Current `main` and the latest release still contain the `user-invocable: true` line in `skills/gh-issues/SKILL.md`, so this PR is not implemented on `main`. The PR description is partly inaccurate because OpenClaw runtime still treats `user-invocable` as a supported frontmatter key, but there is still a real mismatch: the stricter skill-validation path in `skills/skill-creator/scripts/quick_validate.py` rejects that key today.
