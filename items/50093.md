## Summary

Keep open. Current `main` does not implement WhatsApp reconnect backfill: the socket is created with `syncFullHistory: false`, reconnect opens a fresh socket/listener without any history-fetch step, and inbound `append` messages older than 60 seconds before connect are explicitly dropped.

## Evidence

- **WhatsApp socket disables full history sync:** `createWaSocket()` passes `syncFullHistory: false` to Baileys, which is direct evidence that full reconnect/history replay is not enabled on current main.
  - file: [extensions/whatsapp/src/session.ts:158](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/extensions/whatsapp/src/session.ts#L158)
  - command: `nl -ba extensions/whatsapp/src/session.ts | sed -n '120,170p'`
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **Reconnect path only reopens socket and listener:** `openConnection()` closes the old connection, creates a new socket, waits for connection, and attaches a listener. There is no cursor restore, history fetch, or backfill call in the reconnect path.
  - file: [extensions/whatsapp/src/connection-controller.ts:339](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/extensions/whatsapp/src/connection-controller.ts#L339)
  - command: `nl -ba extensions/whatsapp/src/connection-controller.ts | sed -n '339,382p'`
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **Stale append/history messages are intentionally skipped:** Inbound handling accepts `messages.upsert` type `append`, but explicitly drops append messages older than 60 seconds before connect. The paired test asserts that a 5-minute-old history sync message must not be delivered to `onMessage`.
  - file: [extensions/whatsapp/src/inbound/monitor.ts:656](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/extensions/whatsapp/src/inbound/monitor.ts#L656)
  - command: `nl -ba extensions/whatsapp/src/inbound/monitor.ts | sed -n '638,670p'`
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **Regression test codifies no old-history replay:** The unit test named `skips stale append messages (older than 60s before connect)` verifies that a stale append/history-sync message is ignored rather than replayed after reconnect.
  - file: [extensions/whatsapp/src/monitor-inbox.append-upsert.test-support.ts:37](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/extensions/whatsapp/src/monitor-inbox.append-upsert.test-support.ts#L37)
