## Summary

Keep this issue open. PR #67916 is still open, authored by the reporter, and uses closing syntax for this issue; current main also still hard-denies local assistant attachments before the server metadata check when `localMediaPreviewRoots` is empty.

## Best Possible Solution

Keep this issue open while PR #67916 remains open. Maintainers should review and verify that PR or land an equivalent fix that avoids the client-side allowed-folder hard deny while bootstrap roots are still unloaded, relies on the existing server metadata endpoint for the authoritative decision, and adds a focused regression test for `localMediaPreviewRoots: []`.
