## Summary

This issue should stay open. Current main still uses exact-only silent-token detection, and final reply normalization strips a trailing `NO_REPLY` but returns the remaining preamble as deliverable text. Two open, unmerged PRs, #66755 and #69394, explicitly reference this issue with closing syntax, so they are implementation candidates rather than grounds to close the issue now.

## Best Possible Solution

Keep this issue open and review the active implementation candidates. Maintainers should choose or adapt the preferred policy, add regression coverage around `src/auto-reply/tokens.ts` and `src/auto-reply/reply/normalize-reply.ts`, merge one PR, then let the issue close from the merged fix.
