## Summary

Keep open. Current main still gates WhatsApp outbound direct-message targets with `allowFrom` only, has no `allowTo`/outbound-only config field in the WhatsApp config contract, and the open PR remains the focused implementation attempt for the user problem described in #25039/#25513. It needs rebasing and completion against the current plugin-local WhatsApp layout, especially the directory lookup concern raised in review, but it is not obsolete or implemented on main.

## Best Possible Solution

Keep the PR open as the focused implementation path. Rebase it onto current main and implement `allowTo` end to end in the WhatsApp plugin/config contract: types and Zod schema, account resolution, outbound resolver/adapters, action authorization, cron/heartbeat implicit target handling, directory lookup/autocomplete, docs, and regression tests. If maintainers prefer a different field name, settle that in review rather than closing the contribution.
