## Summary

Keep open. Current main mitigates related Bonjour/ciao watchdog cancellation paths and documents the disable workaround, but it does not fix the exact reported malformed mDNS packet crash: OpenClaw still stages @homebridge/ciao 1.3.6, the plugin only classifies unhandled promise rejections for cancellation and one interface-churn assertion, and the reported `getNetAddress` assertion is an uncaught exception path that still exits the gateway.

## Best Possible Solution

Keep #67578 open and fix the Bonjour plugin/dependency boundary: patch or upgrade `@homebridge/ciao` so `MDNSServer.handleMessage` treats `getNetAddress` address/netmask version mismatches as malformed packet/interface noise, logs at warn/debug, and continues. Add focused regression coverage for the exact `IP address version must match` assertion and preserve the existing `OPENCLAW_DISABLE_BONJOUR=1` / `openclaw plugins disable bonjour` workaround documentation.

## Risks / Open Questions

- Closing this as implemented would conflate the fixed ciao watchdog cancellation/restart-loop family with the still-unfixed malformed inbound packet assertion path.
- A local-network peer can plausibly still trigger a gateway exit if ciao throws this assertion through `uncaughtException`.
- A broad process-level suppression would be risky; the fix should be scoped to the Bonjour/ciao packet handling path or an upstream/local ciao patch.
