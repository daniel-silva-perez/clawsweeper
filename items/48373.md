## Summary

Keep open. Current `main` still silently ignores `content` on `feishu_doc` `action: "create"`: the create branch only calls `createDoc(...)` and returns its result, and `createDoc` has no content-writing path. The parameter-schema pipeline still flattens union tool schemas into one object schema, so `content` remains exposed as an allowed top-level property even though the `create` variant does not consume it.

## Evidence

- **Create action still ignores content:** `registerFeishuDocTools` handles `case "create"` by calling only `createDoc(client, p.title, p.folder_token, ...)` and immediately returning that result; unlike `write`/`append`/`insert`, it never reads `p.content` or calls `writeDoc`.
  - file: [extensions/feishu/src/docx.ts:1472](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/extensions/feishu/src/docx.ts#L1472)
  - command: `nl -ba extensions/feishu/src/docx.ts | sed -n '1430,1495p'`
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **createDoc has no content parameter/path:** `createDoc` accepts `(client, title, folderToken, options)` and only creates the doc plus requester permissions. There is no markdown/content argument and no follow-up write step inside the helper.
  - file: [extensions/feishu/src/docx.ts:907](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/extensions/feishu/src/docx.ts#L907)
