## Summary

Current `main` already supports configurable inbound debouncing for Telegram through the shared `messages.inbound` config, and Telegram batches same-sender bursts into one synthetic message before dispatch. The long-text split case is also covered by Telegram-specific fragment coalescing.

## Close Comment

Closing from a Codex review because this is already implemented on `main` and shipped in `v2026.4.22`. Telegram reads the shared inbound debounce settings through `messages.inbound.debounceMs` / `messages.inbound.byChannel.telegram` (`src/auto-reply/inbound-debounce.ts:21`, `extensions/telegram/src/bot-handlers.runtime.ts:159`) and batches same-sender bursts into one synthetic message before dispatch (`extensions/telegram/src/bot-handlers.runtime.ts:212`). The long split-text case is also covered by Telegram-specific fragment coalescing and tests (`extensions/telegram/src/bot.media.stickers-and-fragments.e2e.test.ts:207`). The requested behavior exists; it landed under the shared `messages.inbound` config surface rather than a Telegram-only `channels.telegram.inboundDebounceMs` key.
