## Summary

Keep PR #71863 open. Current main still has the Signal daemon shutdown race that the PR targets: `stop()` is synchronous, monitor teardown does not await the child process exit, and Signal config reload still stops then immediately starts the channel. The PR is a focused candidate fix for open issue #22676, is recent, mergeable, and the author addressed the bot review follow-ups in c8bf546a.

## Best Possible Solution

Keep PR #71863 open for maintainer review. The best product/code path is to land this PR or an equivalent focused fix that makes the Signal daemon stop contract async, sends SIGTERM, waits for signal-cli exit, escalates to SIGKILL on timeout, and awaits that shutdown from monitor abort/reload teardown before starting a replacement Signal channel. Preserve the added regression coverage for stop wait/escalation and aborted monitor shutdown, then close #22676 when the fix reaches main.
