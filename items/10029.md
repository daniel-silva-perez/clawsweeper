## Summary

Keep open. Current `main` at `754acc4478fe361db1e5303fd60f589d52beea7a` does not implement syntax highlighting in the WebChat/Control UI markdown renderer. Code fences are rendered as escaped plain text with an optional `language-*` class and copy-button chrome, but there is no highlighting pass in the UI render path.

## Evidence

- **WebChat markdown renderer does not highlight code:** `md.renderer.rules.fence` and `code_block` escape raw code text and emit `<pre><code...>` directly. There is no `highlight.js`, `shiki`, or similar call in this UI renderer.
  - file: [ui/src/ui/markdown.ts:422](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/ui/src/ui/markdown.ts#L422)

  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
- **Control UI package has no direct syntax-highlighting dependency:** The UI package depends on `markdown-it`, `marked`, and related markdown packages, but does not list `highlight.js`, `prism`, or `shiki` as a direct dependency for the WebChat renderer.
  - file: [ui/package.json:11](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/ui/package.json#L11)
