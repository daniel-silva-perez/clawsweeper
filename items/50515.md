## Summary

Keep PR #50515 open. Current main still interpolates channel logging fields without control-character sanitization, and these helpers are public plugin SDK surfaces used by bundled channels. The PR remains useful, but it needs rework before merge because review comments identified real gaps: reuse the existing log sanitizer, preserve JS caller coercion, and sanitize `channel` too.

## Best Possible Solution

Keep the PR open and rework it rather than closing it. The best fix is to update `src/channels/logging.ts` to use the existing `sanitizeForLog` from `src/terminal/ansi.ts` through a small coercing wrapper, apply it consistently to `channel`, `reason`, `target`, and `error`, and add focused `src/channels/logging.test.ts` coverage for newline/CR/NUL/DEL/ANSI injection, malicious `toString()`, non-string JS-style inputs, and a crafted `channel` value.
