## Summary

Current `main` has a general compaction safeguard, but it does not implement the issue's requested loop-aware guard end to end. The code adds structured compaction summaries, tool-failure/file-op context, preserved recent turns, and an optional quality-audit retry, yet there is no dedicated `agents.defaults.compaction.guard` config, no transcript-tail risk classifier for stale reminder/system recurrence or duplicate-assistant clusters, and no `recommend-reset` escalation path. This looks like partial adjacent work, not a full implementation of #48238.
