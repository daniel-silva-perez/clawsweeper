## Summary

Keep open. Current main still has the reported behavior gap: `sessions.usage` discovers sessions across configured agents, but unscoped `usage.cost` still calls the transcript-backed cost summary once with no `agentId`, so it scans only the default agent transcript directory. Related PR #42063 was closed unmerged as superseded by this PR, and #13219 is a broader usage-log feature request, not a shipped fix for this bug.

## Best Possible Solution

Keep #70205 open, but require cleanup before merge: isolate the gateway `usage.cost` aggregation and UI date-only label fix from unrelated branch drift, preserve explicit `agentId` scoping and agent-scoped cache keys, keep the multi-agent gateway regression tests plus timezone label test, and leave #13219 open for the broader dedicated usage-log proposal.
