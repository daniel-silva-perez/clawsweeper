## Summary

Keep open. Current main still lacks ACP backend fallback/failover: ACP config exposes a single backend, runTurn only retries a fresh handle on the same resolved backend, and runtime registry selection is initial-backend selection rather than post-failure failover. The PR is not merge-ready as-is because the provided review discussion identifies serious control-flow and safety issues, and the branch includes unrelated changes, but the core requested capability remains unresolved and no canonical superseding item is evident.

## Best Possible Solution

Keep open, but request a focused redesign before merge. A good path is a narrow ACP manager change that derives backend candidates from resolved session metadata plus global config, retries only before streamed output or tool side effects, handles ACP_SESSION_INIT_FAILED and ACP_TURN_FAILED intentionally, closes old handles before switching, preserves failure bookkeeping, and adds schema/docs/tests. Split the unrelated install-smoke and runtime-deps staging changes into separate PRs.
