## Summary

Current main still does not suppress a whitespace-doubled silent-token payload. The shared matcher only accepts one exact token or a JSON action envelope, and normalization strips just the trailing token, leaving a visible `NO_REPLY` remainder for `NO_REPLY\n\nNO_REPLY`. The related plugin-side Quiubo guard is a valid workaround, and #69472 may be relevant broader work, but this core/shared behavior is not implemented on main.

## Best Possible Solution

Keep this issue open and fix the shared silent-token predicate or normalization path in `src/auto-reply/tokens.ts` and `src/auto-reply/reply/normalize-reply.ts`. The narrow product behavior should treat token-only sequences such as `NO_REPLY\n\nNO_REPLY` and custom-token equivalents such as `HEARTBEAT_OK\n\nHEARTBEAT_OK` as silent, flow through `isSilentReplyPayloadText` and plugin runtime delivery, add regression tests, and preserve #19537 mixed-content semantics.
