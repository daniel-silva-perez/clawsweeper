## Summary

Keep open. Current main still classifies silent replies with an exact single-token regex, and the reported `NO_REPLY\n\nNO_REPLY` case still leaves a visible `NO_REPLY` after current normalization logic.

## Best Possible Solution

Keep the issue open and implement a narrow core fix in `src/auto-reply/tokens.ts` so all-whitespace-separated repetitions of the silent token are treated as silent, with regression coverage in `src/auto-reply/tokens.test.ts` and the outbound/normalization path. Do not suppress substantive mixed-content replies.
