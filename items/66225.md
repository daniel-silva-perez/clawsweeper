## Summary

Keep this PR open. Current main still has the narrow final-tag regexes in the streaming subscriber and exported user-facing sanitizer, so the self-closing `<final/>` leak this PR targets is not implemented on main. The linked bug #65867 remains open, and the companion persistence PR #69685 was closed rather than merged, so this remains a plausible implementation candidate.

## Best Possible Solution

Keep this PR open and have a maintainer review the branch. The best path is to land this fix or an equivalent shared final-tag parser with regression coverage for `sanitizeUserFacingText`, streaming with enforcement off, streaming with enforcement on, and the malformed-attribute parser-hardening case; after that lands, #65867 can be closed as implemented.
