## Summary

Keep this PR open. Current main still has the narrow final-tag regexes in both the streaming subscriber and user-facing sanitizer, so the self-closing `<final/>` leak described by the PR is not implemented on main. The related bug remains open, and the companion persistence PR was closed rather than merged, so this PR is still a plausible implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open and have a maintainer review the current branch. The best path is to land this fix or an equivalent shared-helper implementation with regression coverage for sanitizer, streaming without enforcement, and enforcement mode, while resolving the remaining parser-hardening review comment before merge. After that lands, the related Gemini final-tag leak bug can be closed as implemented.
