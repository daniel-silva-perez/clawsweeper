## Summary

Current `main` still does not implement WhatsApp audio preflight transcription before group mention gating, and the related media-carryover request is also not implemented. The issue should stay open.

## Evidence

- **WhatsApp group gating still checks mentions from `msg.body` and skips before any audio transcription path:** `applyGroupGating()` builds mention state from `debugMention(params.msg, ...)`, then immediately skips unmentioned group messages. There is no preflight transcription call or transcript injection in this path.

  - sha: [40be5ad58187](https://github.com/openclaw/openclaw/commit/40be5ad58187fd20b8defe5fc30e649190c131a8)
- **WhatsApp inbound extraction still reduces audio-only messages to a placeholder body:** Audio-only inbound messages are normalized to `"<media:audio>"`, which means mention detection sees placeholder text unless another step transcribes first.
  - file: [extensions/whatsapp/src/inbound/extract.ts:284](https://github.com/openclaw/openclaw/blob/40be5ad58187fd20b8defe5fc30e649190c131a8/extensions/whatsapp/src/inbound/extract.ts#L284)
