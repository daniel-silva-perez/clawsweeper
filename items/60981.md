## Summary

Keep open. Current main still only has a workspace-only filesystem guard and lacks PathGuard, `allowedPaths`, `denyPaths`, structured policy denials, and spawned-subagent filesystem ceilings. This PR is a plausible core security implementation candidate, but the provided review discussion shows unresolved correctness/security concerns, so it should be reviewed or replaced rather than closed as obsolete.

## Best Possible Solution

Keep this PR open for maintainer handling. The best path is to request a rebase and changes addressing the review-raised PathGuard wiring gaps, or replace it with a narrower core implementation under `tools.fs` that adds canonicalized deny-wins `allowedPaths`/`denyPaths`, structured denials, media/apply_patch/prompt-image coverage, schema/docs updates, and monotonic spawned-subagent ceilings. Coordinate remaining scope with #7722 and the broader security profile in #8719.
