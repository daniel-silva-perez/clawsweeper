## Summary

Keep this PR open. Current main still has only the channel-level guard in `mergeDeliveryContext`, so the same-channel different-`to` fallback-thread case proposed by this PR is not implemented on main. The latest PR scope is narrow, active, and aligned with a real shared helper behavior rather than the older, superseded spawn-site diff.

## Best Possible Solution

Keep the PR open for maintainer review. If the latest diff remains green, the best path is to land the narrow `mergeDeliveryContext` guard plus regression test, because current main does not yet protect same-channel different-`to` merges from inheriting a stale fallback `threadId`.
