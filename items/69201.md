## Summary

Keep this PR open. Current main does not implement the same-channel `to` mismatch or orphan fallback `threadId` guards in `mergeDeliveryContext`, and the rewritten PR is now a focused helper/test change rather than obsolete spawn-site work. The primary Slack subagent-threading issue discussed in the PR was already addressed elsewhere, but this PR targets a remaining adjacent helper-level leakage path that is not covered on main.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to review and, if CI is green, land the narrow helper-level guard with its regression tests, while checking the direct `to` comparison semantics across current merge callers before merge.
