## Summary

Current `main` does not add a core `humanDelay.mode = "proportional"`; core still only supports `off`, `natural`, and `custom`. But this request now fits the plugin surface: OpenClaw’s scope guidance says optional capability should usually ship as plugins, and current outbound delivery exposes an async `message_sending` hook that can inspect outgoing content and delay before send. That makes proportional pacing a better ClawHub plugin than a new core config mode.

## Risks / Open Questions

- There is no in-tree ClawHub/plugin implementation to point the reporter at yet; this decision is about scope, not availability of a ready-made package.
- The current hook surface exposes content and conversation context but not an explicit block-vs-final reply kind, so a plugin would need its own pacing heuristics or per-conversation state if it wants to mimic core `humanDelay` semantics exactly.

## Close Comment

Thanks for the suggestion. This was a Codex maintainer review against current `main`, and I’m closing this as a better fit for a ClawHub plugin rather than OpenClaw core.

The reason is scope plus available seams: `VISION.md:54-59` explicitly keeps optional capability in plugins where possible, and current OpenClaw now has an outbound `message_sending` plugin hook (`docs/plugins/sdk-overview.md:212-215`). That hook is async/sequential in `src/plugins/hooks.ts:733-741`, and outbound delivery awaits it before sending in `src/infra/outbound/deliver.ts:567-607`, so a plugin can inspect `event.content.length` and apply proportional pacing without expanding core config.

For reference, core `humanDelay` on `main` is still intentionally limited to `off | natural | custom` in `src/config/types.base.ts:94-100` and `src/config/zod-schema.core.ts:527-533`, so this request is not already implemented in core. If someone wants to package proportional outbound pacing, ClawHub/plugin space is the right place for it now.
