## Summary

Current `main` does not implement this PR’s google-vertex ADC fallback. `resolveApiKeyForProvider()` still throws immediately for an unresolved explicit `profileId` before it ever reaches the existing env/ADC fallback path, and both embedded-run and BTW session flows can still supply auto-selected profile ids through that explicit-profile branch.

## Evidence

- **Explicit profile still hard-fails before env/ADC fallback:** In `resolveApiKeyForProvider()`, an unresolved explicit `profileId` throws `No credentials found for profile ...` at lines 426-435. The env fallback path is only reached later at lines 540-550, so it is unreachable for stale explicit profiles.
