## Summary

Keep open. Current `main` already tracks per-session lifecycle state in gateway data, but the Control UI sessions list still does not render any active/completed/timed-out indicator or dedicated last-run state, so the enhancement request remains unmet.

## Evidence

- **Sessions view has no status column or badge:** The sessions table renders columns for Key, Label, Kind, Updated, Tokens, Compaction, Thinking, Fast, Verbose, and Reasoning. There is no status column or lifecycle badge in the table header or row rendering.

- **Session rows only show kind and updated time:** `renderRows` formats `updatedAt` and renders a kind badge, but it does not read or display `row.status`, `row.startedAt`, `row.endedAt`, or `row.abortedLastRun` anywhere in the visible row UI.
  - file: [ui/src/ui/views/sessions.ts:442](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/ui/src/ui/views/sessions.ts#L442)
