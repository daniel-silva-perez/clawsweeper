## Summary

Keep open. The issue is a current, concrete 2026.4.24 regression report and the directly linked fix PR #72290 is still open. Current main does not yet show the proposed protections: inbound dedupe is keyed on provider/peer/session/MessageSid, and webchat `chat.send` user transcript updates still emit role/content/timestamp/media without a user-turn idempotency key.

## Best Possible Solution

Keep #72176 open as the canonical regression until #72290 or an equivalent maintainer-reviewed fix lands. The fix should stabilize inbound dedupe when channel retry ids change and add transcript-level idempotency for user turns, with regression coverage for webchat/WeChat-style same body and timestamp but different ids/seq numbers.
