## Summary

Keep open. Current `main` at `ebd7f19a3b20001960474ec9ba9cc424e45f7b23` still accepts `gateway.auth.mode: "none"` with `gateway.tailscale.mode: "serve"`, explicitly tests that startup path as valid, and `auth.mode === "none"` still authorizes HTTP/WS requests without credentials. The Tailscale integration still publishes the whole gateway port via `tailscale serve --bg --yes <port>`, so this is not implemented on main and not obsolete.
