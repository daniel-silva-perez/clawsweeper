## Summary

Keep this PR open. Current main still renders the status Context line from persisted totalTokens without honoring totalTokensFresh: false, while the PR is a focused active implementation candidate for the still-open compaction status bug #72140.

## Best Possible Solution

Keep this PR open and review it as the active implementation candidate for #72140. The best path is to update the status formatter to use the existing fresh-total contract before trusting persisted totals, fall back to the newest valid compactionCheckpoints[].tokensAfter only when totals are stale or unavailable, simplify the unreachable guard noted in review, and keep the focused regression test in src/auto-reply/status.test.ts.
