## Summary

Keep this PR open. Current main still renders `/status` context usage from `entry.totalTokens` without honoring `totalTokensFresh: false`, while compaction checkpoint data with `tokensAfter` already exists and is a reasonable fallback source. The PR is a focused active implementation candidate for open issue #72140, with a regression test in its proposed diff and only a minor cleanup note in review.

## Best Possible Solution

Keep this PR open and review it as the active implementation candidate for #72140. The best path is to adapt the status formatter on current main to use `resolveFreshSessionTotalTokens(entry)` before trusting persisted totals, fall back to the newest valid `compactionCheckpoints[].tokensAfter` only when totals are stale or unavailable, simplify the unreachable guard noted in review, and keep the focused regression test in `src/auto-reply/status.test.ts`.
