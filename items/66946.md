## Summary

Current main still carries the Control UI bootstrap path and basePath composition that this issue reports. The frontend fetch URL is built by prepending the inferred/configured basePath to a constant that already includes an `__openclaw` segment, the gateway handler mirrors that computed path, docs and tests still assert the old endpoint shape, and the two related fix PRs were closed unmerged.

## Best Possible Solution

Keep this issue open and land a focused core fix. The likely path is to make the bootstrap config endpoint basePath-relative, update `src/gateway/control-ui-contract.ts`, `ui/vite.config.ts`, `ui/src/ui/controllers/control-ui-bootstrap.test.ts`, `src/gateway/control-ui.http.test.ts`, and `docs/web/control-ui.md`, and add explicit coverage for `gateway.controlUi.basePath: "/__openclaw__"`.
