## Summary

Keep open. Current main still defines the Control UI bootstrap config path as `/__openclaw/control-ui-config.json`, and the browser loader still prepends `gateway.controlUi.basePath` to that constant. With a base path such as `/__openclaw__`, the constructed URL remains the duplicated-looking `/__openclaw__/__openclaw/control-ui-config.json`. The two related PRs in the provided GitHub context attempted the same fix but were closed unmerged, so there is no landed implementation to close this issue against.

## Best Possible Solution

Keep this issue open and land a focused core fix. The likely fix belongs in `src/gateway/control-ui-contract.ts` by changing the bootstrap config endpoint to `/control-ui-config.json`, updating `ui/vite.config.ts`, adjusting `ui/src/ui/controllers/control-ui-bootstrap.test.ts` and `src/gateway/control-ui.http.test.ts` to cover `gateway.controlUi.basePath: "/__openclaw__"`, and updating `docs/web/control-ui.md`. Consider a temporary compatibility alias for the old computed endpoint so cached Control UI bundles do not break during upgrades.
