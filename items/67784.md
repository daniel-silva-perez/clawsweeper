## Summary

Keep open. Current main still lets the Mattermost plugin handle the message tool `send` action before the generic core send fallback, and that Mattermost handler only reads `params.media` plus `replyToId`/`replyTo`. It does not read `threadId`, `filePath`, or `path`, and it does not destructure or forward `mediaLocalRoots` into `sendMessageMattermost`. The linked open PR #67796 is the right implementation candidate, but it is unmerged and appears partial, so this issue should stay open until that PR is reviewed, expanded, and landed or replaced.

## Best Possible Solution

Keep this issue open. Review and expand #67796, or land a replacement PR, so `extensions/mattermost/src/channel.ts` reads `threadId`, falls back from `media` to `filePath`/`path`, forwards `mediaLocalRoots` and `mediaReadFile` into `sendMessageMattermost`, and adds Mattermost action tests for local file upload into a thread.
