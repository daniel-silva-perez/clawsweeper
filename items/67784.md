## Summary

Keep this issue open. Current main still routes Mattermost `message action=send` through the Mattermost plugin action handler, and that handler still does not consume `threadId` or forward `mediaLocalRoots`/`mediaReadFile` to `sendMessageMattermost`. The `filePath`/`path` portion is partly mitigated upstream because the generic message-action runner normalizes those aliases into `params.media` before plugin dispatch, but the Mattermost handler itself remains incomplete for local file upload into a thread. The linked PR #67796 is open, unmerged, and only addresses the media-alias fallback, so it is an implementation candidate rather than a reason to close this issue.

## Best Possible Solution

Keep this issue open. Review and expand #67796, or land a replacement that updates `extensions/mattermost/src/channel.ts` so `handleAction` reads `threadId`, preserves the existing `replyToId` behavior, resolves media aliases consistently across `media`/`mediaUrl`/`path`/`filePath`/`fileUrl`, forwards `mediaLocalRoots` and `mediaReadFile` into `sendMessageMattermost`, and adds focused Mattermost action tests for local file upload into a thread.
