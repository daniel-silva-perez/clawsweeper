## Summary

Keep #64080 open. Current main still has the Slack room/channel routing asymmetry the PR targets: top-level room messages under replyToMode="all" compute a message-thread reply target but still resolve the inbound session key through the base channel session. The PR is a focused fix for linked issue #64078 and is not implemented or superseded on main.

## Best Possible Solution

Keep the PR open and adapt it to current main by changing `canonicalThreadId` in `prepare-routing.ts` to use the auto-thread message timestamp for top-level Slack channel/group messages when `replyToMode="all"`, remove the now-unused `isRoomish` dependency from that resolver, and add/update regression tests so `SessionKey` and `MessageThreadId` align for channel/group top-level messages while `off`/`first` and true thread replies stay unchanged.
