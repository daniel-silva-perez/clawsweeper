## Summary

Keep open. Current main still implements the opposite routing behavior from this PR: top-level Slack channel/group messages under replyToMode="all" keep the shared room session key while separately setting MessageThreadId for threaded reply delivery. The linked canonical bug #64078 remains relevant, and the PR may need a refresh, but the underlying behavior is not implemented on main.

## Best Possible Solution

Keep this PR or its canonical issue #64078 open until a maintainer decides the intended Slack behavior. If the report is accepted, refresh the fix against extensions/slack/src/monitor/message-handler/prepare-routing.ts, update the thread-session-key tests so replyToMode="all" channel top-level messages get per-message :thread:<ts> sessions while off/first behavior stays shared, and address the unused isRoomish review note. If maintainers decide shared channel sessions are intentional, close the PR with that product rationale and update the contradictory changelog/test expectations around non-off modes.
