## Summary

Keep open. Current `main` has a first-party backup archive feature, but only for `openclaw backup create` and `openclaw backup verify`. That partially covers the issue, including state/config/sessions and likely default cron data via the state dir, but there is still no general `backup restore`/`backup import`/`backup list` flow or selective restore/export path for cron jobs and session history.

## Evidence

- **Backup CLI only exposes create and verify:** The registered `openclaw backup` subcommands on `main` are `create` and `verify` only. There is no general restore/import/list subcommand in the shipped CLI surface.
  - file: [src/cli/program/register.backup.ts:10](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/src/cli/program/register.backup.ts#L10)
  - sha: [3361593442e2](https://github.com/openclaw/openclaw/commit/3361593442e295e9bb1f55cbfb38861ea9413e45)
- **Docs describe archive creation, not restore:** The user docs for `openclaw backup` describe creating a local archive for state, config, auth profiles, credentials, sessions, and optional workspaces, with examples only for `backup create` and `backup verify`. They do not document a general restore/import workflow.
  - file: [docs/cli/backup.md:9](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/docs/cli/backup.md#L9)
  - sha: [3361593442e2](https://github.com/openclaw/openclaw/commit/3361593442e295e9bb1f55cbfb38861ea9413e45)
- **Changelog shows the landed feature was partial:** The changelog entry for `2026.3.8` says the shipped work added `openclaw backup create` and `openclaw backup verify` for local state archives. It does not claim restore/import/list support.
  - file: [CHANGELOG.md:2793](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/CHANGELOG.md#L2793)
  - sha: [3361593442e2](https://github.com/openclaw/openclaw/commit/3361593442e295e9bb1f55cbfb38861ea9413e45)
- **Cron data is only indirectly covered via state/config storage:** Default cron storage resolves to `~/.openclaw/cron/jobs.json`, and the backup docs say the backup includes the OpenClaw state directory. That means cron jobs are probably included in the archive today, but only as raw state, not via the selective export/import utility requested in the issue.
  - file: [src/cron/store.ts:26](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/src/cron/store.ts#L26)
  - sha: [3361593442e2](https://github.com/openclaw/openclaw/commit/3361593442e295e9bb1f55cbfb38861ea9413e45)
- **No general restore/import command found on main:** A repo-wide search for `backup restore|backup import|backup list|migrate export|migrate import` only found Matrix-specific room-key backup restore docs, not a general OpenClaw backup restore/import/list command.
  - command: `rg -n "backup restore|backup import|backup list|migrate export|migrate import" src docs CHANGELOG.md`
