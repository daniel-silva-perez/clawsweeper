## Summary

Keep open. Current main still lacks the Feishu reply-path mediaLocalRoots plumbing: sendMediaFeishu/loadWebMedia require explicit roots for local media, but createFeishuReplyDispatcher neither accepts nor forwards them, and both Feishu bot dispatcher call sites still omit agent-scoped roots. The PR head described in the provided context also addresses the earlier review concern by adding bot.ts call-site wiring, so this remains useful implementation work rather than cleanup material.

## Best Possible Solution

Keep the item open. The best maintainer path is to land one canonical Feishu reply-media fix that derives getAgentScopedMediaLocalRoots at both Feishu bot reply-dispatcher call sites, threads mediaLocalRoots through CreateFeishuReplyDispatcherParams into sendMediaFeishu, and adds focused reply-dispatcher coverage proving local media replies carry the allowed roots.
