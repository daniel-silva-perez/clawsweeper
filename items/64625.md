## Summary

Current main still has the reported Slack custom-invoke gap: bundled Slack registers `createSlackActions` with `options.invoke`, and that branch forwards the raw `toolContext` without adding `ctx.mediaLocalRoots` or `ctx.mediaReadFile`. The fallback runtime branch adds those fields, but the bundled plugin does not use that fallback path. The related PR #64632 targeted this exact bug but is closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep this issue open. Apply the narrow fix in `extensions/slack/src/channel-actions.ts` by constructing an augmented Slack action context from `toolContext` plus `ctx.mediaLocalRoots` and `ctx.mediaReadFile`, and pass that same augmented context to both `options.invoke` and the fallback runtime branch. Add a focused regression test for `createSlackActions` with `options.invoke` and `upload-file` proving both media fields reach the invoke context.
