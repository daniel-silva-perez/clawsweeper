## Summary

Keep this issue open. Current main still lacks the requested `plugins.entries.amazon-bedrock-mantle.config.discovery.enabled` contract: the Mantle manifest has an empty `configSchema`, the registration path does not read or pass `api.pluginConfig`, and `resolveImplicitMantleProvider` still attempts IAM bearer-token generation when no explicit token is present. The provided GitHub context also shows open PR #67341 with closing syntax for this issue, so the right maintainer path is to review/land or close that implementation candidate rather than auto-close the issue first.

## Best Possible Solution

Review PR #67341 against current main or replace it with an equivalent fix in `extensions/amazon-bedrock-mantle`: add the manifest schema/UI hint for `discovery.enabled`, thread plugin config through `register.sync.runtime.ts`, short-circuit `resolveImplicitMantleProvider` before IAM token generation when disabled, and add regression coverage proving disabled discovery does not call IAM token generation or `/v1/models`. Close this issue only after that lands.
