## Summary

Current `main` already persists cron execution history to JSONL and exposes basic run-history filtering/search, but it does not fully implement the issue’s requested surface. In particular, `openclaw cron runs` only exposes `--id` and `--limit`, and I found no current support for time-range filters, success-rate summaries in `cron list`, or explicit history prune/vacuum commands.

## Evidence

- **Persistent per-job run logs are written on every finished cron run:** Gateway cron handling resolves `runs/<jobId>.jsonl` and appends a `finished` record with status, error, summary, delivery info, timing, model/provider, and usage.
  - file: [src/gateway/server-cron.ts:549](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/src/gateway/server-cron.ts#L549)
  - sha: [3361593442e2](https://github.com/openclaw/openclaw/commit/3361593442e295e9bb1f55cbfb38861ea9413e45)
- **Run-log storage is durable JSONL with built-in status/query filtering:** Run-log code stores entries under `runs/<jobId>.jsonl`, supports `status/statuses`, `deliveryStatus/deliveryStatuses`, `query`, `sortDir`, paging, and all-jobs aggregation.
  - file: [src/cron/run-log.ts:36](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/src/cron/run-log.ts#L36)
  - sha: [3361593442e2](https://github.com/openclaw/openclaw/commit/3361593442e295e9bb1f55cbfb38861ea9413e45)
- **Docs describe persistent run history and run-log retention:** Cron docs tell users to inspect run history with `openclaw cron runs --id <jobId>` and document `cron.runLog.maxBytes` / `cron.runLog.keepLines` pruning for run-log files.
  - file: [docs/automation/cron-jobs.md:30](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/docs/automation/cron-jobs.md#L30)
  - sha: [3361593442e2](https://github.com/openclaw/openclaw/commit/3361593442e295e9bb1f55cbfb38861ea9413e45)
- **CLI surface is narrower than the issue request:** `openclaw cron runs` currently accepts only `--id` and `--limit`; there are no CLI options here for status filters, time ranges, follow mode, export mode flags, or pruning/history maintenance commands.
  - file: [src/cli/cron-cli/register.cron-simple.ts:137](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/src/cli/cron-cli/register.cron-simple.ts#L137)
  - sha: [3361593442e2](https://github.com/openclaw/openclaw/commit/3361593442e295e9bb1f55cbfb38861ea9413e45)
- **Filtering exists in RPC/UI, but not the requested time-range/success-rate/history commands:** The protocol allows status, delivery-status, query, sort, limit, and offset filters, and the Control UI exposes scope/search/status/delivery/sort controls. A targeted search over cron CLI/docs/UI/protocol found no `--since`, `--range`, `older-than`, `historyRetentionDays`, `success rate`, or `cron history` surface.
  - file: [src/gateway/protocol/schema/cron.ts:327](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/src/gateway/protocol/schema/cron.ts#L327)
