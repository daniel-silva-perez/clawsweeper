## Summary

Current main still has the exact remaining heartbeat outbound-session routing gap: isolated heartbeat runs switch the agent turn to `runSessionKey`, but outbound delivery is still built with the base `sessionKey`. The PR is a focused implementation candidate, not obsolete cleanup.

## Best Possible Solution

Keep this PR open for maintainer review. The likely code direction is to pass `runSessionKey` into `buildOutboundSessionContext` for heartbeat outbound delivery, keep the base `sessionKey` for heartbeat task state and dedupe bookkeeping, and land the focused regression test proving `deliverOutboundPayloads(...).session.key` is the isolated `:heartbeat` key for fresh-base and wake re-entry paths.
