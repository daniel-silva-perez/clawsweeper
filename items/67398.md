## Summary

Keep this PR open. Current main still builds the heartbeat outbound session context with the base `sessionKey` even after isolated heartbeats switch the agent run itself to `runSessionKey`, so the PR is not obsolete and targets a real remaining code path.

## Best Possible Solution

Keep this PR open for maintainer review. The likely product/code direction is to pass `runSessionKey` into `buildOutboundSessionContext` for heartbeat outbound delivery when `isolatedSession` is active, keep the base `sessionKey` for task timestamp/dedupe state where current code intentionally uses it, and land a focused regression test that proves `deliverOutboundPayloads` receives the isolated `:heartbeat` session key for both fresh-base and wake re-entry paths.
