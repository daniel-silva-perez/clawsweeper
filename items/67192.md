## Summary

Keep this issue open. Current main still routes OpenAI-compatible stream chunks through a truthy `delta.content` branch without a runtime string guard, and fallback reasoning extraction still only handles string `reasoning_content`/`reasoning`/`reasoning_text`, not the block array/object shapes described in the report. The bundled Mistral plugin and docs make `mistral/mistral-small-latest` adjustable reasoning a supported current path, and #67203 plus #67292 are still identified in the provided GitHub context as open implementation candidates with closing syntax, so this issue should remain the canonical bug until a focused fix merges.

## Best Possible Solution

Keep the issue open and have maintainers review #67203 and #67292. The best fix likely belongs in `src/agents/openai-transport-stream.ts`: only treat `delta.content` as text when it is a string, normalize Mistral `reasoning_content` block arrays/objects into standard thinking/text deltas, and add regression coverage for Mistral object/array stream chunks so channel output cannot degrade into `[object Object]`.
