## Summary

Keep this issue open. Current main still has the risky OpenAI-compatible streaming path the report describes, and two open PRs explicitly reference this issue as implementation candidates; maintainers should review/land one of those fixes or keep the issue as the canonical bug until a focused fix reaches main.

## Best Possible Solution

Keep the issue open and have maintainers review #67203 and #67292. The best fix likely belongs in `src/agents/openai-transport-stream.ts`: treat `delta.content` as text only when it is a string, normalize Mistral `reasoning_content` block arrays/objects into standard thinking/text deltas, and add regression tests covering Mistral object/array stream chunks so the Telegram `[object Object]` failure cannot recur.
