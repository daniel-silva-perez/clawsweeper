## Summary

Keep open. The issue is fresh and detailed, with a strong differential report that isolates the failing path to the shipped CLI entry bootstrap. Current main still has a distinct entry-main prelude before `runCli()`, and I did not find source, test, or history evidence strong enough to prove the Linux Codex OAuth image fetch failure is fixed.

## Best Possible Solution

Keep #72087 open. Reproduce current main's shipped `openclaw`/`openclaw.mjs`/`dist/entry.js` path against an awaited direct `runCli(argv)` path on a Linux Codex OAuth host with the reported IPv6 condition. If confirmed, isolate the entry prelude side effect, make entry and direct `runCli()` network behavior equivalent, and add regression coverage for entry-main bootstrap parity plus a guarded live Codex image CLI lane if the failure cannot be made deterministic.
