## Summary

Keep open. Current `main` has policy gating and provenance tagging for `sessions_send`, but it does not implement HMAC/token/signature-based verification of inter-session sender identity. The request is still live core security work, not a ClawHub/plugin-scope item.

## Evidence

- **`sessions_send` only attaches unsigned provenance metadata:** The tool builds `inputProvenance` with `kind`, `sourceSessionKey`, `sourceChannel`, and `sourceTool` derived from the current session context; there is no signature, token, or MAC field in the send path.
  - file: [src/agents/tools/sessions-send-tool.ts:283](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/agents/tools/sessions-send-tool.ts#L283)

  - file: [src/agents/tools/agent-step.ts:42](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/agents/tools/agent-step.ts#L42)
