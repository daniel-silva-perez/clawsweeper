## Summary

Keep open. The report is recent, has multiple independent confirmations, and current main contains only adjacent Telegram polling/gateway liveness fixes rather than a source-backed resolution of the Debian 13 >140% CPU/stall symptom from #63002.

## Best Possible Solution

Keep #63002 open and ask affected reporters to retry the latest release/current main, then attach sanitized gateway logs around the stall, CPU profile or top/ps snapshots, channel/plugin details, Node/runtime version, and any `Polling stall detected`, `getUpdates conflict`, heartbeat, or channel-health messages. If the logs match the Telegram polling path, confirm whether the current watchdog/transport fixes resolve it; otherwise isolate the hot loop and add a targeted regression test for that gateway/channel path.
