## Summary

Keep open. Current `main` still passes `cron.list` rows straight into Control UI state and still dereferences `job.payload.kind` in `ui/src/ui/app-render.ts`, so malformed persisted cron rows can still plausibly break chat navigation instead of being safely ignored. I did not find a current regression test covering malformed cron rows in this render path.

## Evidence

- **UI still dereferences cron payload kind directly:** `cronModelSuggestions` maps `state.cronJobs` and reads `job.payload.kind` without an outer payload guard.
  - file: [ui/src/ui/app-render.ts:802](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/ui/src/ui/app-render.ts#L802)
