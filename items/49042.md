## Summary

Keep #49042 open. Current main still does not expose `finalLlmOutcome` on `agent_end`; the hook contract and emitters only provide `runId`, `messages`, `success`, `error`, and `durationMs`. The newer `model_call_ended` telemetry is useful but not equivalent because it is per model-call telemetry and lacks the requested final agent outcome shape, HTTP status fields, prompt/runner/provider source distinction, and terminal assistant failure mapping. The PR discussion is substantive and the author addressed earlier stale-assistant/compaction edge cases, but the remaining provider-timeout classification review and current hook-type file layout need maintainer/API follow-up rather than cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer/API review. The best path is a rebased implementation that adds an optional `finalLlmOutcome` to the current public hook contract in `src/plugins/hook-types.ts`, populates it consistently from the embedded runner and any other `agent_end` harness emitters or explicitly scopes the contract, documents the `success` versus `finalLlmOutcome.ok` semantics, includes regression coverage for stale history, compaction, terminal assistant error turns, structured HTTP status, and provider timeouts, and preserves sanitized plugin-visible error text.
