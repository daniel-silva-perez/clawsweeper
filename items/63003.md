## Summary

Keep PR #63003 open. Current main still does not expose effective mounts from `openclaw sandbox explain`, and the related user problem from #62988 remains unsolved in the checked-out code. The PR is recent and focused, but its remaining review concerns should be handled before landing, especially runtime alias canonicalization and backend-specific mount roots.

## Best Possible Solution

Keep the PR or its linked issue active. The best implementation is to add `sandbox.mounts` and the text `Effective mounts:` section in `src/commands/sandbox-explain.ts`, but derive sandboxed status through `resolveSandboxRuntimeStatus`/main-session alias canonicalization and expose backend-aware effective roots for Docker, SSH, and OpenShell. Add focused tests for JSON mounts, text output, custom Docker binds, direct-session empty mounts, alias handling, and non-Docker backend paths.
