## Summary

Keep open. Current main still does not implement the requested `sandbox.mounts` JSON field or `Effective mounts:` text output, and this PR remains the canonical implementation candidate after #62988 was closed as superseded. It should be revised before landing because the current diff can misreport effective boundaries for main-session aliases and non-Docker sandbox backends.

## Best Possible Solution

Keep this PR open as the active implementation candidate. Revise it to use the canonical sandbox runtime status path, derive mounts from the selected sandbox backend, preserve the requested JSON/text output, keep focused tests for JSON, text, Docker binds, direct runtime, aliases, agent workspace overrides, and remote roots, and drop unrelated test-file churn before maintainer review.
