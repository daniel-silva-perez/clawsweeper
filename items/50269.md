## Summary

Keep open. Current main still does not capture or print the daemon/gateway status probe duration for `openclaw gateway status --deep`; the PR addresses a real remaining gap, though it needs rebasing onto the current probe labels and `DaemonStatus.rpc` shape.

## Best Possible Solution

Keep the PR open or ask for a small rebase: add elapsed-time capture to `src/cli/daemon-cli/probe.ts`, thread `durationMs` through `DaemonStatus.rpc`, print a human-only duration line in `src/cli/daemon-cli/status.print.ts` for both success and failure, and preserve the current JSON contract with focused gather/print tests.
