## Summary

Keep #63030 open. Current main has some cache-stability work, but the reported cross-path prompt divergence is not fully resolved: normal chat still builds inbound/group context in the auto-reply path, heartbeat builds a different system-event context, gateway inter-session task-completion wakes still forward only caller-provided extraSystemPrompt, and the image-pruning follow-up still mutates older history messages in place. PR #63096 is relevant and directly linked, but it is open/unmerged in the provided context and only covers the announce/completion-wake slice.

## Best Possible Solution

Keep the issue open until main has a shared, canonical volatile prompt-context assembly path for normal chat, heartbeat, and inter-session task_completion/announce turns, with regression tests comparing section order or digests for the same persisted session. The image-pruning cache churn should be split into its own canonical follow-up or fixed by making image pruning append-only, compaction-time, or otherwise prefix-preserving.
