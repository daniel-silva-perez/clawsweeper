## Summary

Keep open. Current `main` has partial `argPattern` plumbing for exec allowlist entries, but the matcher explicitly applies argument-pattern enforcement only on Windows; on non-Windows hosts it returns on the first binary-path match and ignores `argPattern`. The public docs and CLI still describe/path expose only binary-path allowlist entries, and there is no `denyArgs`/`denyExec` surface. That does not satisfy this issue's requested path-based filtering use case, especially for the macOS multi-agent scenario described.
