## Summary

Keep PR #50933 open. Current main still handles daily/idle stale-session rollover and archival, but it does not emit the PR's requested internal `session:idle_reset` / `session:daily_reset` events, and bundled `session-memory` remains wired only to manual `/new` and `/reset` command hooks. The linked #50891 user problem is therefore still real on main. The PR discussion shows focused work and several addressed review concerns, but the branch still needs maintainer review/rebase or a clearer superseding implementation before merge.

## Best Possible Solution

Keep the issue/implementation path open until one maintained PR lands. The best fix is to emit a well-defined internal automatic reset hook before transcript archival for idle/daily rollovers, register and update bundled `session-memory` for that event, add regression coverage for legacy transcript lookup and duplicate manual-reset suppression, and update hooks docs/CLI examples.
