## Summary

Keep this PR open. Current main still hardcodes plugin command results to `shouldContinue: false`, the SDK type remains `ReplyPayload` with no `shouldContinue` opt-in, and plugin docs still describe `registerCommand` as bypassing the LLM. The PR targets a real core/plugin API gap rather than ClawHub-only work, and the provided discussion shows the author addressed the main review concerns in the latest diff.

## Best Possible Solution

Keep the PR open or recreate its minimal core change: extend `PluginCommandResult` with `shouldContinue?: boolean`, have `handlePluginCommand` strictly forward `shouldContinue === true` into `CommandHandlerResult`, suppress the reply on continuation, preserve the default terminal behavior, and add focused tests plus plugin docs/JSDoc updates.
