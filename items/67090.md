## Summary

Keep this PR open. Current main still implements and documents post-compaction AGENTS.md extraction as an H2/H3 contract, while this PR proposes broadening that contract to H1-H4. The change is plausible core work, but maintainer discussion already asked for a real-world H1 case and a reason to include H4; that product decision is still unresolved.

## Best Possible Solution

Keep this PR open for maintainer/product decision. The author should answer the maintainer's concrete-case question. If H1 support is accepted, narrow the parser change to the intended heading levels and update implementation, regression tests, docs/gateway/config-agents.md, config type comments, schema help/generated config help, and public docs text so the contract is explicit. If no real H1 workspace case exists, maintainers can close manually as not planned after that clarification.
