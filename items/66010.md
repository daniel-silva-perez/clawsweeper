## Summary

Current main still lacks the requested native subagent cascade circuit breaker. Existing code has adjacent guardrails for spawn depth, active-child limits, subagent run timeouts, announce delivery retry backoff, tool-loop blocking, and auth-profile cooldowns, but none track consecutive failed subagent spawns by target/task, trip a cooldown, support override/reset, or expose breaker state in config/CLI/UI.

## Best Possible Solution

Keep this issue open. Implement a native subagent breaker in the core spawn and registry path: persist breaker state keyed by requester plus target agent and/or normalized task signature, update it on subagent completion outcomes, reset on success, check it before gateway spawn, support explicit override/manual reset, add strict config/schema/docs, and expose breaker state through subagent status, tasks/agents CLI, and Control UI. Treat provider billing/rate-limit breakers as related follow-up scope unless maintainers intentionally combine them.

## Risks / Open Questions

- Closing now would lose a still-missing core reliability feature; provider cooldowns, tool-loop detection, and announce retry backoff do not prevent repeated failed subagent spawns.
- Breaker keying and defaults need product design: target-agent-only may block unrelated work, while task-signature-only may miss broken agent configuration. Provider-level cost/rate-limit circuit breakers should remain coordinated but separate scope.
