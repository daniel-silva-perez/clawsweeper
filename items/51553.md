## Summary

Keep open. Current main still has the exact gap this PR targets: `updateLastRoute` runs under `withSessionStoreLock` but calls `loadSessionStore(storePath)` without `{ skipCache: true }`. Adjacent lock-holding writers already use `skipCache: true`, and `loadSessionStore` can return a cached store when cache is enabled. This remains a plausible focused bug fix, not a stale or implemented PR.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land this one-line `skipCache: true` consistency fix, ideally with a focused regression test that proves `updateLastRoute` re-reads the store inside the lock instead of using a stale cached snapshot.
