## Summary

Keep open. Current `main` does not contain the core mitigation from PR #48424: chat history is still requested at 200 messages, rendered with a 200-message cap, and chat refresh still requests an unbounded sessions list. The PR’s idea still addresses a live gap, but it is stale against today’s UI structure and would need rework rather than closure.
