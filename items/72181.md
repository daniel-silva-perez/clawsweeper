## Summary

Keep open. Current main does not implement BlueBubbles trainer/supervised reply mode: the config schema and account type have no trainerMode/trainerNotifyNumber equivalent, and allowed inbound DMs still flow through the normal reply pipeline that sends back to the original BlueBubbles target. This is a fresh, concrete feature request for the bundled BlueBubbles plugin, not stale, duplicate, implemented, or clearly outside the repository.

## Best Possible Solution

Keep #72181 open for BlueBubbles owner/product review. A good implementation would stay under extensions/bluebubbles, add typed/schema-backed trainerMode and trainerNotifyNumber config, intercept allowed inbound DMs before createChannelReplyPipeline dispatch, route drafts and owner commands through the existing BlueBubbles send path, enforce pending-message TTLs, and add focused monitor/config tests plus user-facing docs.
