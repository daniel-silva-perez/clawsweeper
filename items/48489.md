## Summary

Keep open. Current `main` does not appear to implement internal tool-call/result surfacing on the OpenResponses SSE path. The HTTP stream handler still only emits assistant-text lifecycle events plus pending client-tool `function_call` turns, while the output schema still excludes `function_call_output` as an output item.

## Evidence

- **OpenResponses output schema still omits tool-result output items:** `src/gateway/open-responses.schema.ts` includes `function_call_output` only in input `ItemParamSchema` (`type: "function_call_output"` at line 127), but `OutputItemSchema` only allows `message`, `function_call`, and `reasoning` starting at line 236. That means the SSE/response output contract on `main` still cannot emit `function_call_output` items.
  - file: [src/gateway/open-responses.schema.ts:236](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/gateway/open-responses.schema.ts#L236)
  - command: `nl -ba src/gateway/open-responses.schema.ts | sed -n '115,155p' && nl -ba src/gateway/open-responses.schema.ts | sed -n '236,265p'`
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **HTTP SSE handler does not consume internal `tool` events:** In `src/gateway/openresponses-http.ts`, the SSE listener handles `evt.stream === "assistant"` at line 931 and `evt.stream === "lifecycle"` at line 955, but there is no `evt.stream === "tool"` branch in that handler. By contrast, WebSocket chat handling explicitly treats `evt.stream === "tool"` as a first-class event in `src/gateway/server-chat.ts` line 901.
  - file: [src/gateway/openresponses-http.ts:923](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/gateway/openresponses-http.ts#L923)
  - command: `nl -ba src/gateway/openresponses-http.ts | sed -n '923,963p' && nl -ba src/gateway/server-chat.ts | sed -n '901,965p'`
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Current output-item helper only builds `function_call`, not tool-result items:** `src/gateway/openresponses-shape.ts` has `createFunctionCallOutputItem(...)`, but the helper returns an output item with `type: "function_call"` at line 27. There is no companion helper for emitting `function_call_output` results into response output/SSE.
  - file: [src/gateway/openresponses-shape.ts:19](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/gateway/openresponses-shape.ts#L19)

I did not find any current test asserting streamed internal tool start/result events or `function_call_output` output items on the HTTP SSE path.
