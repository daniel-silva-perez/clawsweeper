## Summary

Keep #71849 open. Current main still routes Voice Call realtime remembered-context questions through the full `openclaw_agent_consult` embedded-agent path, has no `realtime.fastContext` config or direct bounded memory/session lookup, ignores non-final transcript chunks for call-state transcript handling, and the docs only describe the shared consult tool. The related open PR #72237 is not a superseding fix because it changes consult thinking/fast-mode overrides for the full consult path, not the requested direct fast context resolver.

## Best Possible Solution

Keep the issue open and implement it in the bundled Voice Call plugin: add a default-off `realtime.fastContext` config/schema/manifest/docs surface, accumulate and normalize partial transcripts, perform a hard-deadline memory/session lookup before full consult, inject snippets or an explicit miss into the realtime session, and retain `openclaw_agent_consult` as fallback. Coordinate with #72237 if it lands, but do not treat it as closing this issue unless it adds the actual direct fast-context resolver.
