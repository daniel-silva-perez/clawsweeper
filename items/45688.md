## Summary

Keep open. Current `main` still matches the reported regression: the main advanced Config view is forced into accordion groups, hides the global search bar in that mode, excludes many settings sections from the main Config page, and spreads them across separate top-level tabs with per-tab search state instead of restoring the old flat searchable/filterable list.

## Evidence

- **Advanced Config is still forced into accordion mode:** `renderConfigTab` for the main Config tab hard-codes `settingsLayout: "accordion"` and routes back to Quick Settings instead of using the flat tabbed/searchable layout.
  - file: [ui/src/ui/app-render.ts:1032](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/ui/src/ui/app-render.ts#L1032)
  - command: `nl -ba ui/src/ui/app-render.ts | sed -n '1010,1065p'`
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Search UI is absent in accordion mode:** `renderConfig` renders `renderAccordionNav()` when `settingsLayout === "accordion"`; the search input is only rendered in the non-accordion branch immediately below.
  - file: [ui/src/ui/views/config.ts:978](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/ui/src/ui/views/config.ts#L978)
  - command: `nl -ba ui/src/ui/views/config.ts | sed -n '970,1038p'`
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Main Config still hides many sections and spreads them across separate tabs:** The main Config tab excludes communication, automation, infrastructure, AI/Agents, `ui`, and `wizard`, while separate `communications`, `appearance`, `automation`, `infrastructure`, and `aiAgents` tabs each render their own scoped config view.
  - file: [ui/src/ui/app-render.ts:1051](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/ui/src/ui/app-render.ts#L1051)
  - command: `nl -ba ui/src/ui/app-render.ts | sed -n '1010,1140p'`
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Search is scoped to the current section/tab, not global:** `renderConfigForm` filters entries by `activeSection` first, then applies search only within those filtered entries, which matches the complaint that search is local to the selected bundle rather than a global config search.
  - file: [ui/src/ui/views/config-form.render.ts:388](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/ui/src/ui/views/config-form.render.ts#L388)
