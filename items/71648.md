## Summary

Keep #71648 open. Current main still has the reported `OpenClawChannelBridge` leak shape: the two pending Maps are unbounded, are not cleared on `close()`, and are only deleted on matching replies/resolution events. The PR is a focused fix with regression tests and an addressed review comment; related #71646 remains the open bug report, while #56420 is orthogonal session-binding work rather than a replacement.

## Best Possible Solution

Keep the PR open and either land it after maintainer validation or apply an equivalent core fix in `src/mcp/channel-bridge.ts`: timestamp pending Claude permissions and approvals, start/stop a lazy TTL sweeper, clear both Maps on `close()`, guard post-close writes, and keep the regression tests. Track the separate cap/FIFO concern as a follow-up if maintainers want adversarial-burst protection. Treat #56420 as orthogonal session-binding work, not a superseding fix.
