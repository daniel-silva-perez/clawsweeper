## Summary

Keep this issue open. Current main still routes Google/Gemini providers into the tagged reasoning contract and still injects the exact <think>/<final> wording that the report says interferes with tool invocation. The Google transport has native function-declaration and function-call parsing plumbing, and current tests cover that plumbing, but there is no current fix that removes Google from tagged mode or amends the tagged prompt to explicitly preserve native tool calls. The related ClawSweeper items are about display sanitization or broader tool-observer behavior, not this Google reasoning/tool-call failure.

## Best Possible Solution

Keep this issue open and fix it in the Google provider/reasoning prompt path. The most direct maintainer path is to either make Google Gemini native for reasoning output when using native tool/function calling, or amend the tagged reasoning prompt so native tool calls are explicitly outside the `<think>/<final>` text contract and must be emitted through the provider's function-call mechanism. Add focused regression coverage around `resolveReasoningOutputMode`/system prompt composition for `google-generative-ai` with tools, plus a Google transport test that keeps native `functionDeclarations` and `functionCall` handling intact.
