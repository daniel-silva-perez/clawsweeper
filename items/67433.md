## Summary

Keep open. Current main still implements `POST /hooks/agent` as an asynchronous fire-and-forget hook that returns only `{ ok, runId }`; it does not parse `blocking`, wait for agent completion, return `text`, or surface blocking failure semantics. The provided PR discussion shows this PR remains an active implementation candidate for the still-open result-return request tracked by #49019, with meaningful review feedback and follow-up fixes rather than an obsolete cleanup candidate.

## Best Possible Solution

Keep this PR open as the concrete implementation path. Maintainers should review the latest branch against current main and either land it after validating the blocking response contract, error/idempotency/trust semantics, docs, and tests, or request changes and keep #49019 as the canonical product/API tracker if the API shape needs refinement.
