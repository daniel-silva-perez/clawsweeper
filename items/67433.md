## Summary

Keep open. Current main still implements `POST /hooks/agent` as a fire-and-forget trigger returning only `{ ok, runId }`; it does not parse `blocking`, await completion, return agent text, or expose blocking error semantics. The discussion shows this PR is an active implementation candidate for the still-open result-return request tracked by #49019, with review feedback already addressed in follow-up commits but not yet merged.

## Best Possible Solution

Keep this PR open as the concrete implementation path. Maintainers should review the latest branch against current main and land it only if the blocking response contract, failure semantics, idempotency behavior, docs, and tests are correct; if the API shape changes, keep #49019 or a narrower follow-up as the canonical product tracker.
