## Summary

Keep open. Current `main` still routes `openclaw directory peers/groups` through Slack's config-backed directory adapter, not the live Slack API readers, and Slack does not implement a `self` resolver, so the reported `[]` / `null` behavior remains plausible on `main` for a workspace that has valid tokens but no `channels.slack.dms`/`allowFrom`/`channels` directory config.

## Evidence

- **CLI examples explicitly promise Slack directory/self behavior:** The CLI help still advertises `openclaw directory self --channel slack` and Slack peer search by name, so the issue matches an advertised surface rather than an unsupported hidden path.
  - file: [src/cli/directory-cli.ts:79](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/cli/directory-cli.ts#L79)
  - command: `nl -ba src/cli/directory-cli.ts | sed -n '70,230p'`
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Directory CLI does not call live directory methods:** `runDirectoryList` resolves only `plugin.directory.listPeers` / `listGroups`; it never prefers `listPeersLive` / `listGroupsLive`, so Slack live directory readers are bypassed by the CLI.
  - file: [src/cli/directory-cli.ts:169](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/cli/directory-cli.ts#L169)
  - command: `nl -ba src/cli/directory-cli.ts | sed -n '152,186p'`
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Slack plugin wires config-backed lists and separate live readers:** Slack registers `listPeers` and `listGroups` from `directory-config.ts`, while the live API readers are only exposed as `listPeersLive` / `listGroupsLive` via `createRuntimeDirectoryLiveAdapter`.
  - file: [extensions/slack/src/channel.ts:426](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/extensions/slack/src/channel.ts#L426)
  - command: `nl -ba extensions/slack/src/channel.ts | sed -n '426,435p'`
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Slack config-backed directory only reads configured allowlists/maps:** The config adapter builds peers from `allowFrom`, `dm.allowFrom`, `dms`, and per-channel `users`, and builds groups only from configured `channels` keys. A valid Slack token alone would not populate these results.
  - file: [extensions/slack/src/directory-config.ts:29](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/extensions/slack/src/directory-config.ts#L29)
  - command: `nl -ba extensions/slack/src/directory-config.ts | sed -n '23,60p'`
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Slack self is null by default:** `createChannelDirectoryAdapter` defaults `self` to `nullChannelDirectorySelf`, which always returns `null`, and the Slack plugin does not override `self` when constructing its directory adapter.
  - file: [src/channels/plugins/directory-adapters.ts:3](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/channels/plugins/directory-adapters.ts#L3)

- **Live Slack API directory readers exist on main:** `listSlackDirectoryPeersLive` uses `users.list` and `listSlackDirectoryGroupsLive` uses `conversations.list`, confirming that current main has live Slack directory code but the CLI path above does not use it.
  - file: [extensions/slack/src/directory-live.ts:68](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/extensions/slack/src/directory-live.ts#L68)
