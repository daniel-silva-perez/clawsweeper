## Summary

Keep open. Current main does not implement the PR's core optimization: `listSessionsFromStore()` still builds every candidate `GatewaySessionRow` before search, activeMinutes, and limit are applied, and each row still resolves child sessions by scanning the full store. The PR discussion also contains useful independent validation and a resolved review nit, so this remains salvageable maintainer-review work rather than conservative cleanup.

## Evidence
