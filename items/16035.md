## Summary

Keep open. Current `main` does not implement default GitHub-profile or public-disclosure behavior for OpenClaw agents, and the current automation guidance still explicitly tells operators not to identify as AI in public-facing content. The only clear built-in disclosure I found is channel-specific support for Microsoft Teams' AI-generated badge, which does not satisfy this GitHub-focused request.

## Evidence

- **Current docs still instruct the opposite behavior:** `docs/automation/standing-orders.md` says `Never identify as AI in public-facing content`, so current maintainer guidance on `main` is contrary to the requested default disclosure behavior.
  - file: [docs/automation/standing-orders.md:118](https://github.com/openclaw/openclaw/blob/b22bf36b/docs/automation/standing-orders.md#L118)
  - command: `nl -ba docs/automation/standing-orders.md | sed -n '112,130p'`
  - sha: [b22bf36b](https://github.com/openclaw/openclaw/commit/b22bf36b)
- **Disclosure exists only for a specific channel, not GitHub/profile-wide:** The Teams adapter explicitly marks outbound messages as AI-generated so Teams renders a badge, showing disclosure is implemented narrowly for one channel rather than as a general GitHub/profile default.
  - file: [extensions/msteams/src/messenger.ts:301](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/extensions/msteams/src/messenger.ts#L301)
