## Summary

Keep open. Current `main` has Discord-side channel/account gating, but the gateway still derives `turnSourceChannel` from the delivery override (`request.channel`) and then forwards that as the run's origin channel. That means a CLI/webchat/API run delivered to Discord can still be tagged as Discord-originated for exec-approval routing, so this cleanup is not implemented on `main`. The PR's exact file-level patch is stale because Discord approval routing moved into the newer native/shared approval helpers, but the underlying issue remains plausible and unresolved in the current codepath.
