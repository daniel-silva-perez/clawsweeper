## Summary

Keep open. Current main does not implement the Feishu iOS video-as-file 502 fix: the Feishu message-resource downloader still makes a single `messageResource.get` call with the provided `image|file` type, and inbound non-image media still routes to `type: "file"` with no retry/fallback or video-file heuristic. The linked issue remains a concrete open bug report, and this PR is a focused bug-fix attempt with useful discussion, even though its latest approach still needs maintainer review against the bot-review concerns and the related #50164 attempt.

## Best Possible Solution

Keep the PR open or explicitly consolidate it with a canonical Feishu 502 fix such as #49855/#50164. The best code path is an extension-owned fix in `extensions/feishu/src/media.ts` and `extensions/feishu/src/bot-content.ts` that robustly handles retryable Feishu HTTP 5xx failures for iOS video-as-file downloads, respects the Feishu SDK/messageResource type contract unless a documented endpoint is used, and adds regression tests for the failing video-file scenario and common SDK error shapes.
