## Summary

Keep open. Current `main` still has no WhatsApp inbound-reaction handling: the WhatsApp monitor only consumes `messages.upsert` into normal inbound messages, drops payloads that do not yield text/media/contact placeholders, and exposes only outbound/ack reaction config in docs and types.

## Evidence

- **WhatsApp inbound monitor has no reaction-specific path:** `attachWebInboxToSocket()` only registers `messages.upsert` and routes each upsert through `enrichInboundMessage()` before enqueueing a normal inbound message; there is no separate reaction event handler or system-event enqueue path.
  - file: [extensions/whatsapp/src/inbound/monitor.ts:642](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/extensions/whatsapp/src/inbound/monitor.ts#L642)
  - command: `nl -ba extensions/whatsapp/src/inbound/monitor.ts | sed -n '640,760p'`
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **Non-text/non-media inbound payloads are dropped:** `enrichInboundMessage()` requires extracted text or a media placeholder and returns `null` otherwise. `extractText()`/`extractMediaPlaceholder()` only cover conversation text, captions, contacts, and media types, with no WhatsApp reaction payload support.
  - file: [extensions/whatsapp/src/inbound/monitor.ts:469](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/extensions/whatsapp/src/inbound/monitor.ts#L469)
  - command: `nl -ba extensions/whatsapp/src/inbound/monitor.ts | sed -n '460,515p'; nl -ba extensions/whatsapp/src/inbound/extract.ts | sed -n '234,294p'`
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **WhatsApp config surface only documents outbound/ack reactions:** The WhatsApp docs describe `reactionLevel`, `ackReaction`, and the outbound `react` action, but do not document any inbound reaction notification setting or behavior.
  - file: [docs/channels/whatsapp.md:361](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/docs/channels/whatsapp.md#L361)
  - command: `nl -ba docs/channels/whatsapp.md | sed -n '360,441p'`
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **WhatsApp config types lack inbound reaction notification support:** `WhatsAppConfig` includes `actions.reactions`, `ackReaction`, and `reactionLevel`, but no `reactionNotifications`-style inbound setting that other channels use for reaction events.
  - file: [src/config/types.whatsapp.ts:17](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/config/types.whatsapp.ts#L17)

  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
