## Summary

Keep open. The specific prompt example in the report looks outdated on current `main` because `buildRuntimeLine()` no longer injects agent counts or cron details, but the requested per-agent isolation is still not implemented: there is no agent-level `runtimeContext`/`isolated` config, and cron listing remains gateway-global rather than agent-scoped.

## Evidence

- **Current runtime line has no agent-count or cron fields:** `buildRuntimeLine()` only emits `agent`, `host`, `repo`, `os`, `node`, `model`, `default_model`, `shell`, `channel`, `capabilities`, and `thinking`. There is no agent-list count, list, or cron metadata in the runtime line on current `main`.
