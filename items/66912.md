## Summary

Keep open. Current main still lacks the Telegram reply-target self-authored media guard, while this PR is a focused implementation candidate. The current PR head appears to wire the guard, but it still needs a focused regression test and cleanup or justification for unrelated generated Plugin SDK baseline hash churn.

## Best Possible Solution

Keep this PR open and ask for a clean Telegram plugin patch: add the self-authored reply-target guard in resolveReplyMediaForMessage, add regression coverage in extensions/telegram/src/bot.test.ts for reply_to_message.from.id === ctx.me.id with no media fetch, preserve normal reply-media ingestion, and remove or justify the docs/.generated/plugin-sdk-api-baseline.sha256 change.
