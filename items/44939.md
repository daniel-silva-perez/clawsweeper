## Summary

Keep open. Current `main` still forces TLS for non-loopback manual iOS gateway hosts, so the requested behavior is not implemented, and the existing iOS test suite still codifies the old behavior. One PR hunk is already stale (`bootstrapToken: nil` is on `main`), and the proposed `resolveManualUseTLS` simplification needs follow-up because `maybeAutoConnect()` reads `gateway.manual.tls` with `UserDefaults.bool(forKey:)`, so an unset key would be treated as `false` and could silently downgrade old/manual profiles to `ws://`.

## Risks / Open Questions

- Closing this PR would drop a still-unimplemented behavior change: `main` still overrides explicit `tls=false` for non-loopback manual iOS connections.
- If merged as written, the PR likely needs an additional migration/default guard for `gateway.manual.tls` during auto-connect to avoid unintended plaintext downgrades when the key is absent.
- The PR is partially stale because the Share Extension `bootstrapToken: nil` fix is already present on `main`, so the branch needs rebasing or trimming rather than cleanup closure.
