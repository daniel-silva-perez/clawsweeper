## Summary

Keep this issue open. Current main has partial local estimation for compaction/preflight paths and transcript-usage fallback when prior assistant usage metadata exists, but it still does not implement the requested local estimated context snapshot for missing or all-zero provider usage in /status or /usage.

## Best Possible Solution

Keep this open and implement a shared estimated context snapshot path for sessions with missing or all-zero provider usage. Reuse readSessionMessages plus estimateMessagesTokens or a shared chars-per-token helper, store/render an explicit estimated source, show it in /status and /usage, and keep compaction gates consuming the same estimate without replacing fresh provider snapshots. Add regression coverage for MiniMax-style all-zero usage and transcripts with messages but no usage metadata.
