## Summary

Keep open. Current main has partial mitigation: stale or missing totals can drive preflight compaction from a local transcript-message token estimate, and status/session listing can recover totals from transcript usage snapshots. The specific no-usage-provider request is still incomplete because /status and session summaries do not estimate context from raw JSONL messages when provider usage is missing or all zero; those paths still report unknown/unavailable rather than an explicitly estimated context count.

## Best Possible Solution

Keep the issue open and implement a shared read-only transcript estimate helper, likely reusing readSessionMessages() plus estimateMessagesTokens(), for sessions whose provider usage is missing, zero, or totalTokensFresh is false. Feed that estimate into /status, session rows, and context notices with an explicit estimated/stale marker, while continuing to use exact transcript/provider usage when available. Add regression tests for a MiniMax-style zero-usage transcript and for a transcript with messages but no usage metadata.
