## Summary

Keep open. Current `main` still implements inbound audio as transcription-only preprocessing, not raw-audio passthrough to the reply model. The shared media-understanding contract only exposes `transcribeAudio`, the audio execution path always returns `audio.transcription`, transcribed audio is stripped from the prompt media note, ACP forwarding only sends image attachments, and the audio config surface has no `auto|passthrough|transcript-only` mode described in the issue.
