## Summary

Keep open. Current main already gets control-character repair inside `parseStreamingJson` from `@mariozechner/pi-ai`, so this PR's wrapper is redundant at the two edited call sites. But the Anthropic transport still does a raw `JSON.parse(data)` on each SSE frame before those call sites, so a malformed `partial_json` payload with raw C0 bytes would still fail earlier. That means there is not strong evidence to close this PR as implemented, and the proposed patch does not clearly address the current failure point on `main`.
