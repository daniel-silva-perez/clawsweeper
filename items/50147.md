## Summary

Keep #50147 open. Current main still has the core conditions described by the PR: the embedded runner passes `attempt.lastAssistant` into payload construction unguarded, and `buildEmbeddedRunPayloads` still falls back to that assistant message when no new assistant text exists. The PR is not ready to land as-is because the posted review comments identify real timeout and compaction regressions, but it remains useful unresolved work rather than something implemented or obsolete.

## Best Possible Solution

Keep the PR open, but require a narrower revision: suppress stale `lastAssistant` fallback only for externally interrupted runs with no current-attempt assistant text, preserve explicit timeout payload delivery, keep usage and compaction bookkeeping, and add focused regression coverage for interrupt abort no-replay plus timeout/compaction guard cases.
