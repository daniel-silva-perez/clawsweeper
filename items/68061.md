## Summary

Keep open. Current main still does not implement Mermaid rendering in WebChat/Control UI: Mermaid fences are rendered as ordinary escaped code blocks, and the UI package has no Mermaid dependency or render hook. The linked feature issue #48996 was closed because this PR became the active implementation tracker, not because the feature shipped. The PR should not merge as-is because the provided review context identifies build, security, behavior-regression, English-copy, dependency, and unrelated-CSS-churn blockers.

## Best Possible Solution

Keep this PR open as the active implementation candidate, but request a narrower Control UI/WebChat Mermaid patch. The best path is lazy/browser-safe Mermaid rendering in the existing markdown surface, preserved sanitizer and task-list behavior, English UI strings, removal of unrelated CSS churn, cleaned dependency changes, and tests for Mermaid fences, non-browser imports, and sanitizer behavior. If maintainers decide Mermaid rendering should not be in core WebChat, that should be an explicit product decision rather than automated cleanup.
