## Summary

Keep open. Current `main` still does not wire session abort/`/sessions/:key/kill` to terminate exec child processes spawned under that session. The code has session-scoped exec/process bookkeeping, but the kill paths only abort agent runs and clear queues; bulk exec termination remains exposed only through the `process` tool per exec `sessionId`, not through session-level kill/abort.

## Evidence

- **Session abort only aborts agent run and clears queues:** `handleStopCommand` resolves the target, clears queued work, calls `replyRunRegistry.abort(...)` and `abortEmbeddedPiRun(...)`, then stops subagents. It never touches exec/process state or the process supervisor.
  - file: [src/auto-reply/reply/commands-session-abort.ts:85](https://github.com/openclaw/openclaw/blob/68e2d6f0889e5365ecdc8f01ab5e3bf6065cbeeb/src/auto-reply/reply/commands-session-abort.ts#L85)
  - sha: [68e2d6f0889e](https://github.com/openclaw/openclaw/commit/68e2d6f0889e5365ecdc8f01ab5e3bf6065cbeeb)
- **Subagent kill path still does not terminate exec children:** `killSubagentRun` marks the session aborted via `abortEmbeddedPiRun(sessionId)` and `clearSessionQueues([childSessionKey, sessionId])`. No exec-session lookup, no process kill, and no supervisor scope cancel are invoked here.
  - file: [src/agents/subagent-control.ts:172](https://github.com/openclaw/openclaw/blob/68e2d6f0889e5365ecdc8f01ab5e3bf6065cbeeb/src/agents/subagent-control.ts#L172)
  - sha: [68e2d6f0889e](https://github.com/openclaw/openclaw/commit/68e2d6f0889e5365ecdc8f01ab5e3bf6065cbeeb)
- **HTTP session kill delegates only to subagent kill helper:** `POST /sessions/:sessionKey/kill` loads the session, then calls `killControlledSubagentRun(...)` or `killSubagentRunAdmin(...)`. That route does not enumerate or terminate exec processes for the session.
  - file: [src/gateway/session-kill-http.ts:114](https://github.com/openclaw/openclaw/blob/68e2d6f0889e5365ecdc8f01ab5e3bf6065cbeeb/src/gateway/session-kill-http.ts#L114)
  - sha: [68e2d6f0889e](https://github.com/openclaw/openclaw/commit/68e2d6f0889e5365ecdc8f01ab5e3bf6065cbeeb)
- **Exec runs are session-scoped, so the missing link is orchestration not bookkeeping:** Tool setup now prefers `sessionKey` as the exec/process `scopeKey`, and exec runtime stores both `scopeKey` and `sessionKey` on each process session and supervisor run. That creates the association the issue asks for, but nothing in session abort/kill consumes it.
  - file: [src/agents/pi-tools.ts:397](https://github.com/openclaw/openclaw/blob/68e2d6f0889e5365ecdc8f01ab5e3bf6065cbeeb/src/agents/pi-tools.ts#L397)
  - sha: [68e2d6f0889e](https://github.com/openclaw/openclaw/commit/68e2d6f0889e5365ecdc8f01ab5e3bf6065cbeeb)
- **Supervisor supports scope-wide cancellation, but current session kill paths do not use it:** `createProcessSupervisor()` exposes `cancelScope(scopeKey, ...)`, but the user-facing `process` tool only lists/kills per exec `sessionId`, and session abort/kill paths never call the scope-wide cancel primitive.
  - file: [src/process/supervisor/supervisor.ts:56](https://github.com/openclaw/openclaw/blob/68e2d6f0889e5365ecdc8f01ab5e3bf6065cbeeb/src/process/supervisor/supervisor.ts#L56)
  - sha: [68e2d6f0889e](https://github.com/openclaw/openclaw/commit/68e2d6f0889e5365ecdc8f01ab5e3bf6065cbeeb)
- **No session-level CLI surface for exec-list/kill children is documented:** `docs/cli/sessions.md` documents `openclaw sessions` listing and `sessions cleanup` only. Background exec management is separately documented under the `process` tool (`list`, `poll`, `log`, `write`, `kill`, etc.), which operates on exec `sessionId`s rather than OpenClaw conversation session ids.
  - file: [docs/cli/sessions.md:8](https://github.com/openclaw/openclaw/blob/68e2d6f0889e5365ecdc8f01ab5e3bf6065cbeeb/docs/cli/sessions.md#L8)
  - sha: [68e2d6f0889e](https://github.com/openclaw/openclaw/commit/68e2d6f0889e5365ecdc8f01ab5e3bf6065cbeeb)
