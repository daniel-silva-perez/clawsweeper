## Summary

Keep open. Current main still maps `minimal`/`low` to `output_config.effort: "low"` for adaptive Claude Opus 4.7 in the Anthropic Messages transport, and the GitHub Copilot transport path can still use that request builder when a reasoning-capable Copilot Claude model reaches it. The default Copilot catalog now reduces exposure by setting `reasoning: false`, but that is a mitigation, not the requested provider/model clamp. The auth-profile cooldown concern is also still consistent with current non-timeout failure handling for `format` errors.

## Best Possible Solution

Keep #70322 open. The best fix is a narrow provider/model compatibility guard in the Anthropic Messages adaptive-effort path: for `github-copilot/claude-opus-4.7`, clamp `minimal` and `low` to `medium` while preserving direct Anthropic Opus behavior and out-of-scope Copilot Claude models. Add regression tests for Copilot Opus 4.7 low/minimal, a direct Anthropic preservation case, and a non-Opus guard. Separately, consider whether `invalid_reasoning_effort`/schema-format rejections should avoid shared auth-profile cooldown or at least be model-scoped.
