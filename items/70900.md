## Summary

Do not close #70900. Current main still has the exact `surface_error` throw guard that the PR changes, and the underlying signal split described in the PR remains present: `failoverReason` can be classified from raw error text while `failoverFailure` is gated on `stopReason === "error"`. The PR is a focused bug fix with a plausible regression test, not obsolete cleanup.

## Best Possible Solution

Keep #70900 open for maintainer review and likely land the minimal `&& params.failoverFailure` guard plus its regression test, or land an equivalent fix that preserves #70848/#70124 real provider-error surfacing while keeping successful turns with stale classified error text on the normal path.
