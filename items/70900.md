## Summary

Keep this PR open. Current main still has the unguarded assistant surface_error throw path, while the PR is a focused source/test change for a real edge case where failoverReason can be classified from stale error text even though failoverFailure is false.

## Best Possible Solution

Keep this PR open and review or land it, or land an equivalent minimal fix in src/agents/pi-embedded-runner/run/assistant-failover.ts that requires params.failoverFailure before throwing from the assistant surface_error branch. Keep the #70848/#70124 behavior for genuine provider failures, and include the proposed regression test for failoverFailure=false with a classified failoverReason.
