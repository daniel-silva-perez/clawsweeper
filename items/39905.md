## Summary

Keep PR #39905 open. Current main still lacks the PR's central Telegram per-conversation `replyToMode` and preview `streaming` config surface and resolution path. One related sub-problem, quoted DM preview streaming duplicating/finalizing as a second message, is already fixed on main and covered by tests, but that does not supersede the conversation-scoped override feature.

## Best Possible Solution

Keep this open until a maintainer either ports the scoped override feature to the current Telegram plugin layout or explicitly rejects the product direction. A current-main implementation should add typed and schema-validated scoped fields for Telegram direct/group/topic config, resolve effective `replyToMode` and preview streaming with topic > direct/group > account precedence before normal dispatch, apply the same reply threading resolution to native command replies, update user-facing Telegram/config docs, and preserve the already-shipped DM preview single-message behavior.
