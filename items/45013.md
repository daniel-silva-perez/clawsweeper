## Summary

Keep open. Current `main` does not implement Ollama query-specific instruct-prefix embedding, so the PR’s requested behavior is still missing rather than already shipped or obsolete.

## Evidence

- **Current Ollama embedding path still sends raw text unchanged:** `createOllamaEmbeddingProvider()` builds one `embedOne()` helper that POSTs `JSON.stringify({ model: client.model, prompt: text })`. The provider then wires `embedQuery: embedOne` and `embedBatch` to `texts.map(embedOne)`, so query and document embeddings are still handled identically on `main`.
  - file: [extensions/ollama/src/embedding-provider.ts:148](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/extensions/ollama/src/embedding-provider.ts#L148)
  - command: `nl -ba extensions/ollama/src/embedding-provider.ts | sed -n '142,176p'`
  - sha: [4630ce3d9e2f](https://github.com/openclaw/openclaw/commit/4630ce3d9e2f089062ccd386b0ae241325961d1a)
- **No main-branch tests cover instruct-prefix behavior:** The current Ollama embedding tests only cover normalization, base URL/header resolution, and API-key fallback. There are no assertions that inspect the request prompt for model-specific query prefixes or that distinguish `embedQuery()` from `embedBatch()`.
