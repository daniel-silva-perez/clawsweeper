## Summary

Keep open. Current main still only implements the existing Signal JSON-RPC/SSE daemon path; the PR's central request, bbernhard/signal-cli-rest-api container REST/WebSocket support plus an apiMode selector, is not implemented on main and is still tied to open issue #10240. The Signal channel is already a bundled plugin, so this is not a clear ClawHub-only close.

## Best Possible Solution

Keep the PR open and review it as the active implementation for #10240. The best path is a focused Signal bundled-plugin change that adds a typed `channels.signal.apiMode` contract, adapter/container client coverage, docs for the public Signal page, and validation against the bbernhard REST/WebSocket endpoints while preserving native signal-cli behavior.
