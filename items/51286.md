## Summary

Keep open. Current main still has the behavior PR #51286 is intended to fix: `sandbox explain` returns any colon-containing session key unchanged, while channel inference only works for canonical `agent:<agentId>:...` keys. The linked bug #51245 remains applicable, and the PR’s latest commit also addresses the bot review’s request to assert `elevated.channel` and `allowedByConfig`.

## Best Possible Solution

Keep PR #51286 open and have a maintainer review or land an equivalent patch on `src/commands/sandbox-explain.ts`: guard colon-containing keys with `parseAgentSessionKey`, wrap non-agent-scoped keys with `buildAgentMainSessionKey`, and keep the regression test that asserts `sessionKey`, `elevated.channel`, and `elevated.allowedByConfig` for `telegram:slash:<id>`.
