## Summary

Keep open. Current `main` does not implement optional Google Chat user OAuth. The Google Chat config/schema only expose service-account inputs, the auth path hard-codes the `chat.bot` scope, all outbound API helpers use that single token path for reactions/uploads/DM lookup, and the docs/runtime still explicitly say reaction-based typing requires user OAuth and falls back because service-account auth does not support it. The latest release tag `v2026.4.22` still contains the same user-OAuth-not-supported note.
