## Summary

Keep this issue open. Current main still matches the reported QQBot dispatcher behavior: `kind: "tool"` text is buffered, a 60s tool-only fallback is scheduled, and a later block/final response clears the fallback before the buffered text is delivered. Telegram has separate tool-progress preview support, but the QQBot path remains missing.

## Best Possible Solution

Keep open for a QQBot-specific streaming fix. The likely path is to make QQBot deliver visible progress when `streaming.mode: "partial"` is active, either by immediate progress send or a QQBot preview/edit path, while preserving existing media fallback and tool-summary suppression behavior. Add a focused regression test in `extensions/qqbot/src/engine/gateway/outbound-dispatch.test.ts` for tool-kind text followed by a block/final reply.
