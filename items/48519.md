## Summary

Keep open. Current `main` still limits provenance to inbound user-message metadata and transcript-local tool-result transforms; it does not implement persistent-state provenance across session boundaries, and the hook seams the issue depends on are still missing.

## Evidence

- **Input provenance is only defined for message-level user metadata:** `InputProvenance` only models `external_user`, `inter_session`, and `internal_system`, and `applyInputProvenanceToUserMessage(...)` only attaches it when `message.role === "user"`. There is no corresponding persistent-state provenance type or propagation path here.
  - file: [src/sessions/input-provenance.ts:12](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/sessions/input-provenance.ts#L12)
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Session persistence path does not carry provenance chains into tool results:** The guarded session manager applies `inputProvenance` only via `transformMessageForPersistence`, while `tool_result_persist` receives just the message plus tool ids and returns only a rewritten message. No provenance chain is passed through this persistence seam.
  - file: [src/agents/session-tool-result-guard-wrapper.ts:72](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/agents/session-tool-result-guard-wrapper.ts#L72)
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Hook contracts still lack provenance/persistence fields:** `PluginHookBeforeToolCallEvent` exposes only `toolName`, `params`, `runId`, and `toolCallId`. `PluginHookToolResultPersistEvent`/context expose only tool ids, the `AgentMessage`, and `isSynthetic`. `before_message_write` likewise only exposes the message. That does not cover provenance-aware tool results or persistent-state writes.
  - file: [src/plugins/hook-types.ts:318](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/plugins/hook-types.ts#L318)
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Requested `before_persistence_write` seam is absent on main:** A repo search for `before_persistence_write` returned no matches in `src`, `docs`, `extensions`, or `test`, so the persistence hook explicitly referenced by the issue is not present.
  - command: `rg -n "before_persistence_write" src docs extensions test`
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Docs still describe provenance as transcript/user-turn scoped:** The transcript hygiene docs only document inter-session provenance on persisted user turns, and the agent-loop docs describe `tool_result_persist` as rewriting tool results before they are written to an OpenClaw-owned session transcript. That is narrower than cross-session persistent-state provenance.
  - file: [docs/reference/transcript-hygiene.md:80](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/docs/reference/transcript-hygiene.md#L80)
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
