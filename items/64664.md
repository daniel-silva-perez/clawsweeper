## Summary

Current main still has the reported approval-restart UX gap. Gateway approval state is process-local, unknown/stale approvals still produce the raw `unknown or expired approval id` sentinel, `/approve` still formats that raw error for users, and Telegram inline approval callbacks still rethrow stale approval failures as retryable callback errors. The related narrow fix PR #67982 was closed unmerged, so there is no shipped implementation to close against.

## Best Possible Solution

Keep this issue open. The near-term maintainer path is to map `isApprovalNotFoundError` in both `src/auto-reply/reply/commands-approve.ts` and the Telegram inline approval callback path to clear guidance that the approval is no longer available, may have expired or been invalidated by a gateway restart, and should be re-requested. Add focused regression coverage in `src/auto-reply/reply/commands-approve.test.ts` and Telegram callback tests. Treat disk-backed approval persistence as a separate product/storage follow-up.
