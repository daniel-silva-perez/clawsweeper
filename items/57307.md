## Summary

Keep open. Current `main` already has recency-based temporal decay for builtin search and recency/diversity-aware dreaming promotion, but it does not implement the issue's requested per-memory importance field, write-time scoring, access-based reinforcement, or importance-weighted retrieval across builtin and QMD.

## Evidence

- **Builtin retrieval score has no importance factor:** Hybrid ranking is computed only from vector and BM25 scores, then optional temporal decay/MMR are applied. There is no stored importance multiplier or access-boost term in the retrieval formula.
  - file: [extensions/memory-core/src/memory/hybrid.ts:127](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/extensions/memory-core/src/memory/hybrid.ts#L127)
  - command: `nl -ba extensions/memory-core/src/memory/hybrid.ts | sed -n '40,170p'`
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **Existing decay is recency-only exponential decay:** Current decay uses `exp(-lambda * age)` from file timestamps with a configurable half-life. It does not use write-time importance scores, Weibull decay, or last-access boosts.
  - file: [extensions/memory-core/src/memory/temporal-decay.ts:24](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/extensions/memory-core/src/memory/temporal-decay.ts#L24)

- **Memory index schema has no importance/access metadata:** The builtin memory schema stores file/chunk hashes, paths, timestamps, text, embeddings, and `updated_at`, but there is no column for importance, pinned state, access count, or last-access time.
  - file: [packages/memory-host-sdk/src/host/memory-schema.ts:18](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/packages/memory-host-sdk/src/host/memory-schema.ts#L18)
