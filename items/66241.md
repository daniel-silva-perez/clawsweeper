## Summary

Keep open. Current main still keeps the Discord component/modal registry only in process-global maps, so the PR's central restart-survival behavior is not implemented. The related maintainer PR #70626 is a plausible storage primitive direction, but it explicitly does not migrate any consumers, so it does not yet supersede the Discord fix.

## Best Possible Solution

Keep the item open, but do not land the current implementation as-is. The best path is to rework Discord component/modal registry persistence around the shared state-dir contract and the maintainer persistent-store direction from #70626 if it lands, preserving route overrides and allowlists, using bounded/validated atomic storage, and adding focused restart, expiry, malformed-file, and authorization regression tests.
