## Summary

Keep open. Current main still does not expose the PR's five active-run helpers through `openclaw/plugin-sdk/agent-runtime`, and the newer `agent-harness` SDK surface is only a partial, experimental trusted-plugin alternative. The PR also has an unresolved trust-boundary review point around exposing the global abort overload to external plugins.

## Best Possible Solution

Keep the PR open, but do not merge it as-is without maintainer API-shape review. The best product direction is a documented, additive run-control surface for trusted plugins, likely coordinated with the open `api.runtime.agent` abort-family request: expose queue/abort/active/streaming/wait helpers through a stable SDK/runtime contract, constrain or explicitly document the global abort overload, and update package exports or runtime types, SDK docs, API inventory, and focused tests.
