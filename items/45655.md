## Summary

Keep open. Current main has moved image/video generation onto dedicated tool surfaces and docs, but the generic chat-model config/runtime still does not represent output modalities or explicitly reject media-output-only models configured under `models.providers.*.models`. That means the core confusion in #45655 still appears unresolved for the reported Poe custom-provider path.

## Evidence

- **Generic model config still only models input modalities:** `ModelDefinitionConfig` still exposes only `input: Array<"text" | "image">` and has no output-modality field, so a media-output-only chat model still cannot be described or validated correctly in `models.providers.*.models`.
  - file: [src/config/types.models.ts:76](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/config/types.models.ts#L76)
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Runtime catalog still tracks only input capability:** `ModelCatalogEntry` only carries `input?: ModelInputType[]`, and `ModelInputType` is limited to `text | image | document`; there is still no catalog/runtime notion of image/video output capability for selected chat models.
  - file: [src/agents/model-catalog.types.ts:1](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/agents/model-catalog.types.ts#L1)
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Docs now describe media generation as tool-driven, not selected-chat-model output:** The media overview says all media capabilities are tool-driven, and the model-providers overview says that page covers LLM/model providers. This clarifies the intended architecture, but it does not add a guard for unsupported media-output models placed in `models.providers`.
  - file: [docs/tools/media-overview.md:12](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/docs/tools/media-overview.md#L12)
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Dedicated image/video config paths exist:** Current docs route image generation through `agents.defaults.imageGenerationModel` and video generation through `agents.defaults.videoGenerationModel`, confirming the supported path is separate from the selected reply model.
  - file: [docs/tools/image-generation.md:10](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/docs/tools/image-generation.md#L10)
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Dedicated video-generation tool shipped, but not a chat-model rejection fix:** Changelog shows the built-in `video_generate` tool was added in 2026.4.5, and image generation was standardized earlier, but I did not find a corresponding fix that rejects or clearly errors on media-output-only models configured as normal chat models.
  - file: [CHANGELOG.md:1188](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/CHANGELOG.md#L1188)
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **No explicit unsupported-output guard found in current tree:** Targeted search for output-modality / unsupported-media-output checks in current main did not find a chat-runtime validation path for configured media-output models under `models.providers`.
  - command: `rg -n "selected model outputs|unsupported.*output|outputModalit|media-output|chat/session runtime" src docs -g '!dist'`
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
