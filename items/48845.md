## Summary

Keep #48845 open. Current main still does not implement the PR's central Feishu fixes: fetched Feishu messages with missing/invalid `chat_type` still parse to `undefined`, reaction synthetic IDs still use the old UUID-stamped format, and outbound replies still target the synthetic reaction message id when no root/reply target is present. The PR also contains useful follow-up from review comments, including preserving unique synthetic IDs and distinguishing created/deleted reactions, but it needs maintainer review because later comments still flagged repeat-event dedupe and topic-session metadata risks.

## Best Possible Solution

Do not close this PR automatically. A maintainer should review or revise it into a focused Feishu fix: add safe chat-type recovery for fetched reaction messages, preserve a unique per-event synthetic dedupe key while separately anchoring replies to the original message id, and cover created/deleted/repeated reaction plus p2p/topic-session cases in Feishu tests.
