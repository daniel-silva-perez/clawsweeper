## Summary

Keep this PR open. Current main still has the Web UI/gateway mismatch the PR is trying to address: runtime/directive resolution can apply effective per-agent reasoning defaults, but `chat.history` only returns effective thinking, and the chat view still decides reasoning visibility from the persisted session row. The PR is a plausible implementation candidate, but its own review context flags stale-cache issues that should be addressed before landing, and the related feature issue remains open for the broader/default-config work.

## Best Possible Solution

Keep the PR open for maintainer review/rework. The best path is to make `chat.history` expose effective thinking and reasoning state through a documented, backward-compatible gateway response, update the Web UI so effective values stay in sync when model/session settings change, and add regression coverage for fresh-session defaults, per-agent precedence, and reasoning visibility. Leave the broader related issue open or split it until the separate `config.patch` gap is verified/fixed.
