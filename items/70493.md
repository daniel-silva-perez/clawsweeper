## Summary

Keep this issue open. Current main still has the Playwright-unavailable browser route path, sandboxed/sub-agent browser calls still route through a distinct sandbox bridge path, and I did not find the isolated sub-agent navigate/snapshot/act regression coverage that the maintainer comment explicitly required before closure.

## Best Possible Solution

Keep this issue open. The best fix is to make isolated sub-agent browser actions consistently use a Playwright-capable runtime, either by packaging/loading the browser plugin runtime in the isolated gateway path or by explicitly delegating browser actions to the host/node browser owner when that is the intended architecture. Add a regression test that exercises navigate or snapshot through the real isolated sub-agent path and asserts it does not return the Playwright-unavailable error when the main gateway or browser node supports Playwright.
