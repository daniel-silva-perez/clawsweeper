## Summary

Keep #70493 open. Current main has browser diagnostics and some existing-session route support, but the isolated-session browser path is not proven fixed: the Playwright-unavailable error still exists, sandboxed sessions still default to the sandbox bridge rather than automatically delegating to a host/node browser runtime, and I found no isolated `sessions_spawn` browser regression coverage. The maintainer comment on the issue explicitly says not to close until isolated-session coverage exists.

## Best Possible Solution

Keep the issue open and implement the maintainer-suggested fix path: verify which runtime owns browser control for isolated sessions, make Playwright availability checks happen on that runtime or package the needed dependency there, and add an isolated `sessions_spawn` regression test covering `browser` snapshot or navigate without returning the Playwright-unavailable error.
