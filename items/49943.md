## Summary

Keep open. Current `main` does not show a strong fix for the reported PTY/session bug, and one core behavior from the report is still present: PTY SIGKILL still goes through process-tree termination on Unix, which can kill the whole process group rather than only the PTY session.

## Evidence

- **PTY adapter still uses process-tree kill for SIGKILL:** `createPtyAdapter().kill()` still calls `killProcessTree(pty.pid)` for the default SIGKILL path instead of targeting only the PTY handle. That matches the issue's reported cascade-kill mechanism rather than disproving it.
  - file: [src/process/supervisor/adapters/pty.ts:166](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/src/process/supervisor/adapters/pty.ts#L166)
  - command: `nl -ba src/process/supervisor/adapters/pty.ts | sed -n '150,190p'`
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **Process-tree helper kills Unix process groups:** `killProcessTreeUnix()` sends signals to `-pid`, i.e. the process group, before falling back to the direct pid. If unrelated routed work shares that group, the current implementation can still produce the blast radius described in the issue.
  - file: [src/process/kill-tree.ts:46](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/src/process/kill-tree.ts#L46)
  - command: `nl -ba src/process/kill-tree.ts | sed -n '1,140p'`
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **Tests still lock in the reported behavior:** The PTY adapter test explicitly expects SIGKILL to use `killProcessTree`, so current `main` treats process-group killing as intended behavior rather than a fixed regression.
  - file: [src/process/supervisor/adapters/pty.test.ts:95](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/src/process/supervisor/adapters/pty.test.ts#L95)
