## Summary

Keep PR #51930 open. Current main already installs manifest-backed plugins under the canonical `openclaw.plugin.json` id, but it does not implement the PR's `@openclawbrain/openclaw` alias or suppress the manifest/package mismatch info message for that package. The PR may need architectural adjustment because a hardcoded third-party alias is not clearly the best core shape, but the remaining behavior is not implemented or superseded on main.

## Best Possible Solution

Keep the PR open for maintainer judgment, but prefer a generic package/manifest identity equivalence contract or external package-name alignment over duplicating a one-off OpenClawBrain alias in core. If maintainers accept host-side handling, use one shared alias source and keep focused install/discovery regression tests.
