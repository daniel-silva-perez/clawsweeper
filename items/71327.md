## Summary

Keep this PR open. Current main still lacks the LINE schema guard, while the LINE runtime treats `dmPolicy: "open"` as unconditional DM acceptance. The PR is a focused LINE-owned schema hardening change with regression tests, and the current head appears to have addressed the bot review gaps by adding omitted-`allowFrom` and account happy-path coverage plus an account-specific message.

## Best Possible Solution

Keep this PR open for maintainer review and likely landing after confirming the named-account inheritance policy and migration wording. The product-correct path is a LINE-owned schema hardening change that reuses `requireOpenAllowFrom`, keeps the focused top-level and account-level regression tests, and documents the explicit migration for open versus allowlisted DMs.

## Risks / Open Questions

- The PR intentionally tightens validation and can reject existing hand-written LINE configs that used `dmPolicy: "open"` without `allowFrom: ["*"]`; release or migration wording should make the two intended fixes clear: add `"*"` for truly open DMs, or switch to `dmPolicy: "allowlist"` for restricted senders.
- Account-level validation should be reviewed against `resolveLineAccount` inheritance semantics. Runtime merges top-level LINE config into named accounts, while the PR requires the account-level object itself to include `"*"` when that account sets `dmPolicy: "open"`; that may be a deliberate explicitness choice, but it is the main maintainer judgment point before landing.
