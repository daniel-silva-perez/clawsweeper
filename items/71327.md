## Summary

Keep PR #71327 open. Current main still has the LINE schema gap: `LineConfigSchema` and account schemas define `dmPolicy`/`allowFrom` but do not call the shared `requireOpenAllowFrom` guard, while the LINE runtime treats `dmPolicy: "open"` as accepting all DM senders. The PR is a focused bundled LINE plugin fix with regression tests, and the supplied patch appears to address the bot review's coverage/message concerns.

## Best Possible Solution

Keep the PR open for maintainer review and likely land after any requested polish: reuse `requireOpenAllowFrom` in the LINE top-level and account schemas, keep the focused `extensions/line/src/config-schema.test.ts` coverage, and add any maintainer-required docs/changelog migration note for explicit `allowFrom: ["*"]` open-DM configs.
