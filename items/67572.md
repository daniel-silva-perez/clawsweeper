## Summary

Keep open. Current main has overlapping, partial non-image attachment support, but it does not implement this PR's main staging contract for normal `chat.send`: non-image offloads are saved and represented as `media://inbound/*` markers, while the dispatch context still lacks `MediaPaths`/`MediaTypes` for the agent workspace/media-understanding path. The PR is also paired with the open same-author issue #48123, so closing only this half would be premature.

## Best Possible Solution

Keep this PR open as the implementation candidate for the remaining `chat.send` non-image staging/workspace parity work. Maintainers should either land a rebased, smaller version that integrates with current main's `media://inbound/*` support and adds/keeps the `ctx.MediaPaths`/media-understanding tests, or close this PR and #48123 together only after deciding that the shipped `media://`/PDF-tool path fully replaces workspace staging.
