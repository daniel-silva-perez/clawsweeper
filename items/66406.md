## Summary

Keep #66406 open. Current main still sends the raw `message(action="react", emoji="👍")` string directly as Feishu `emoji_type`, so the reported `reaction type is invalid` failure remains plausible. The related PR #66478 is a useful exact fix proposal, but it is still open/unmerged in the provided context and the current checkout does not contain its normalization helper or tests.

## Best Possible Solution

Keep the issue open until #66478 or an equivalent fix lands. The best code path is to normalize common unicode emoji inputs to Feishu reaction type strings in the Feishu plugin boundary before add/list/remove API calls, while preserving explicit Feishu type-string pass-through, and add regression tests for inputs like `👍`, `👀`, lowercase type names, trimming, and unknown fallback behavior.
