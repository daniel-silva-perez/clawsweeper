## Summary

The reported packaging gap is addressed on current `main` and in the shipped `v2026.4.22` release: the npm package now explicitly includes `docs/reference/templates/`, and release validation blocks publishes where the packed tarball or an installed package cannot bootstrap the workspace templates.

## Close Comment

Closing after Codex review. Current `main` and the shipped `v2026.4.22` release already cover this: `package.json:45` explicitly publishes `docs/reference/templates/`, `scripts/openclaw-npm-release-check.ts:62` now fails release validation if those template paths are missing from the tarball, and `scripts/lib/workspace-bootstrap-smoke.mjs:51` checks that an installed package can actually bootstrap the workspace without the `Missing workspace template` failure. That means the package-content issue described here is implemented/fixed in the current release line. If you can still reproduce this on a fresh install of `v2026.4.22` or newer, please open a new issue with the exact install command, package version, and resulting on-disk layout, since that would point to an install/upgrade corruption path rather than the published tarball contents.
