## Summary

Keep #50755 open. Current main still only supports global `session.maintenance`; it does not accept `agents.list[].maintenance`, does not expose an agent-aware maintenance resolver, and normal session writes still use a global or already-resolved global maintenance config. The PR is not obsolete, but the review discussion shows it needs rework before merge rather than cleanup closure.

## Best Possible Solution

Keep #50755 open for maintainer review or rework. The best implementation is a core config change: accept `agents.list[].maintenance`, share the maintenance schema, resolve global plus normalized per-agent policy from a single loaded config, pass the resolved policy through cleanup/load/save paths without applying one agent's policy to shared or explicit stores, and add regression tests for normal writes plus the review-comment edge cases.
