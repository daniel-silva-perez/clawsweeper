## Summary

Current main still skips the token-threshold check for fresh persisted session totals when the transcript-byte trigger is inactive, which is the runtime path this PR targets. The PR overlaps with #64384 and #65600, but it is not safely obsolete or clearly superseded because it also addresses the `totalTokensFresh: undefined` legacy-session edge raised in review.

## Best Possible Solution

Keep this PR open as a valid implementation candidate. The best maintainer path is to consolidate it with #64384 and #65600, rebase onto current main, and land one tested fix that evaluates explicitly fresh persisted totals against the preflight threshold, falls back to transcript estimation for stale or unknown freshness, and preserves the newer transcript-byte trigger.

A session with `totalTokensFresh: true`, a high cached-inclusive `totalTokens`, and no transcript-byte trigger still exits here.
