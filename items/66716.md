## Summary

Keep open. Current main still short-circuits `runPreflightCompactionIfNeeded` when persisted `totalTokens` is fresh, so the threshold check is skipped for the exact fresh cached-token path this PR targets. The related issue was closed after a review that verified cached tokens are persisted, but current source still does not use that fresh cached-inclusive total for preflight compaction. This PR overlaps with #64384/#65600, but it is not clearly superseded because it also handles the `totalTokensFresh: undefined` legacy-session edge noted in review.

## Best Possible Solution

Keep this PR open as a real implementation candidate. The best maintainer path is to consolidate it with #64384/#65600, rebase onto current main, and land one tested fix that evaluates explicitly fresh persisted totals against the preflight threshold while preserving transcript fallback for stale or unknown freshness and the newer transcript-byte trigger.
