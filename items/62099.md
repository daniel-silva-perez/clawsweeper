## Summary

Keep open. Current main still has unguarded auth-profile bookkeeping saves that can throw from the reported path, and #67077 is an open, unmerged implementation candidate that explicitly closes this issue. ## Best Possible Solution

Keep this issue open and review #67077 on its merits. The best fix is to make only the post-success auth-profile bookkeeping saves non-fatal, keep security-sensitive token persistence errors fatal, and land regression coverage for EPERM from both the lock-guarded and direct save paths.
