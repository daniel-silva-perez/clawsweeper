## Summary

Keep this PR open. Current main at 7c0fdae9b95bfcc667af3ad341b63ecc194464e4 still lacks the explicit active-memory chat-type support described by the PR and by #65775: the manifest enum, runtime normalization, and chat-type resolver only recognize direct/group/channel. The PR is a focused implementation candidate, but the existing review comment identifies a real edge case in the proposed parser ordering that should be addressed before landing.

## Best Possible Solution

Keep the PR open as the active implementation candidate, ask for the review-comment fix to parse explicit session keys before broad group/channel/direct marker checks, and land either this PR or an equivalent active-memory patch with schema, runtime normalization, resolver, docs, and regression coverage for explicit sessions.
