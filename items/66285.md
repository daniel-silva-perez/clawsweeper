## Summary

Keep this PR open. Current main still lacks explicit active-memory chat-type support for the linked #65775 behavior: the manifest schema, runtime config normalization, chat-type union, resolver, docs, and tests still only cover direct/group/channel. The PR is a focused candidate, but its proposed resolver ordering should be fixed before landing because explicit session keys can contain raw session-id text with other chat-type markers.

## Best Possible Solution

Keep this PR open as the active implementation candidate. Update resolveChatType to recognize the structured agent:<agent>:explicit:<sessionId> prefix before broad marker checks, or parse the session key segments directly, then land an equivalent active-memory patch with schema, normalization, resolver, docs, and regression coverage for marker-containing explicit session IDs.
