## Summary

Keep this PR open. Current main still lacks the proposed post-rename chmod in the main config write path, while adjacent code already treats 0o600 config permissions as a security boundary. The PR is narrow and consistent with existing hardened atomic-write patterns; it is not obsolete, duplicated, or implemented on main.

## Best Possible Solution

Review and land the focused permission hardening after adding a small POSIX regression test in src/config/io.write-config.test.ts, or at minimum manually verify the hot-save path tightens a pre-existing 0664 config file to 0600. The change belongs in src/config/io.ts because gateway config.set/config.patch/config.apply all use that shared write path.
