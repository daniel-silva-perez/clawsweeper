## Summary

Keep #7717 open. Current main still lacks a Discord role-mention wake trigger: role mentions are preserved as Discord metadata and used for other-mention filtering, but there is no `triggerOnRoles` config/schema surface and no wake-path comparison between mentioned role IDs and configured trigger role IDs.

## Best Possible Solution

Keep #7717 open and implement this in the bundled Discord plugin as an additive guild/channel config surface such as `triggerOnRoles?: string[]`. Preflight should compare Discord `mentionedRoles`/`mention_roles` against the effective configured role IDs and treat a match as a wake mention while preserving `requireMention`, `ignoreOtherMentions`, @everyone/@here, explicit bot mentions, reply-to-bot behavior, sender allowlists, and role-based routing. Add focused config schema/type metadata tests, preflight tests for matched and unmatched role mentions, and update https://docs.openclaw.ai/channels/discord.
