## Summary

Keep open. Current `main` still allows `memorySearch.experimental.sessionMemory` + `sources: ["memory", "sessions"]` while the bundled `session-memory` hook writes recent transcript content into `workspace/memory/*.md`, so the same conversation can be indexed once from `sessions/*.jsonl` and again from the generated memory markdown. The current docs describe both features independently and do not warn about the overlap.
