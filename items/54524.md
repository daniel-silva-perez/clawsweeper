## Summary

Current main still allows the reported overlap. Memory search can index both `memory` and `sessions`, the bundled `session-memory` hook still writes session-derived Markdown under `<workspace>/memory/`, and search deduplication is keyed by chunk identity rather than cross-source provenance. The related PR #40618 is an open partial mitigation, not a shipped fix.

## Best Possible Solution

Keep this issue open. The lowest-risk maintainer path is to add public docs warnings in memory search/config and hook docs: use `sources: ["memory"]` when relying on the bundled `session-memory` hook, or disable/redirect the hook when indexing transcripts directly. A stronger code fix would add explicit provenance or a cross-source dedupe key so duplicate embedding work is avoided without suppressing real memory notes. Review #40618 as a partial mitigation if maintainers want configurable hook output, but it does not close this issue until merged and documented as the recommended path.
