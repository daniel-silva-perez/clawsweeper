## Summary

Keep open. Current main still has user-bus-only gateway runtime status and PID-only stale-listener detection, so the reported headless Linux systemd false positive is not implemented on main. This PR is a plausible fix candidate, but it should not be closed or merged as-is because the provided review context identifies unresolved functional issues in the PR diff.

## Best Possible Solution

Keep this PR open as a real gateway CLI bugfix candidate. The best path is to revise the PR so system-bus probing explicitly targets --system, primary runtime selection only considers gateway units while node units remain auxiliary context, cgroup-based dedup is limited to known safe OpenClaw sibling services, and regression tests cover those edge cases before the normal changed gate runs.

## Risks / Open Questions

- Do not merge as-is: primary gateway selection can choose openclaw-node or openclaw-node-host when no gateway unit is present, causing gateway status to report a node service as the gateway.
- Do not merge as-is: treating any different *.service cgroup as distinct can hide stale or conflicting OpenClaw listeners running under unrelated services.
- The system-bus probe should pass --system explicitly; relying on systemctl defaults leaves an avoidable edge case around session bus environment variables.
