## Summary

Keep open. Current main still does not provide a web UI path to restore or reopen a pre-`/new` chat session. The `+` button starts `/new`, the gateway rotates that chat to a new `sessionId` under the same session key, and the old transcript is only archived on disk as `*.jsonl.reset.*`. Control UI/WebChat exposes live `chat.history` plus `sessions.list`/`sessions.patch`, but no restore flow for reset archives.
