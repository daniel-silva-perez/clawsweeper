## Summary

Keep this PR open. Current main still has the unsafe `model.id.trim().toLowerCase()` and `model.name.trim().toLowerCase()` calls in `canonicalizeLegacyResolvedModel`, so the proposed defensive guard is not obsolete. The related closed PR #69997 targeted a different stored-override path and was not merged, so it does not supersede this work. The remaining maintainer ask is a regression test for undefined or non-string `id`/`name` on the OpenAI Codex legacy alias path.

## Best Possible Solution

Keep the PR open as a valid bug-fix candidate. Ask for or apply a focused regression test that feeds an OpenAI Codex resolved model with undefined or non-string `id`/`name` through the legacy `gpt-5.4-codex` canonicalization path, then land this guard or an equivalent maintainer patch.
