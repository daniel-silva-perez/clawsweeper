## Summary

Keep open. Current `main` still appears to match the reported interrupt-abort replay bug: the embedded runner derives `lastAssistant` from the full session snapshot, `run.ts` still feeds that value into payload fallback with no abort guard, and `runReplyAgent` still has no `meta.aborted` early-exit before normal payload handling.

## Evidence

- **Attempt state still uses the full snapshot's last assistant:** `lastAssistant` is still set from the reversed full `messagesSnapshot`, while `currentAttemptAssistant` is separately derived from only the current attempt. That means an abort with no new assistant message can still carry a prior-turn assistant message in `attempt.lastAssistant`.
