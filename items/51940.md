## Summary

Keep #51940 open. Current main still resolves and patches the channel config form at the channel root (`channels.<id>`) and does not pass the default/configured account into the form, while current config contracts support account-scoped channel config. The PR is not ready to merge as-is because review comments identify real schema-fallback and patch-target regressions, but the underlying fix is still relevant rather than obsolete.

## Best Possible Solution

Keep the PR open and ask for a revision that resolves the scoped account path only when the account schema/value actually exists, falls back consistently to the channel root schema and patch path, trims account IDs at the boundary, and adds a regression test covering account-scoped values plus root fallback behavior.
