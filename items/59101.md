## Summary

Current main has useful watcher-scope and QMD collection-scope mitigations, but the reported one-shot CLI failure is not fully implemented away: `openclaw memory index` still constructs the normal memory manager, and non-status managers still start Chokidar watchers before indexing. Open PR #49851 tracks the more direct fix, but it is still open and its discussion notes remaining gaps, so this issue should stay open.

## Best Possible Solution

Keep this issue open until a PR lands that adds a watcher-free one-shot CLI manager mode for memory indexing. The fix should route `memory index` and `memory status --index` through that mode, consider `memory search` if it shares the same one-shot CLI risk, make QMD manager caching respect the purpose, and add regression tests proving Chokidar is not started for those CLI paths while gateway/runtime managers still watch normally.
