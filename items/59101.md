## Summary

Current main still leaves the reported watcher-startup path in `openclaw memory index --force`. The CLI index command builds a normal memory manager, the builtin manager starts Chokidar watchers for non-status managers, and the QMD path creates a full QMD manager that also starts a watcher. Current changelog entries show watcher-scope mitigations, not a watcher-free one-shot CLI mode. Open PR #49851 remains open and unmerged with the direct proposed fix, so this issue should stay open until that work or equivalent lands.

## Best Possible Solution

Keep this issue open and review, rebase, or replace PR #49851 with an equivalent watcher-free one-shot memory manager mode. The fix should route `memory index` and `memory status --index` through that mode, decide whether `memory search` needs the same treatment, account for QMD manager caching/modes, and add regression tests proving those CLI paths do not start Chokidar while gateway/runtime managers still watch normally.
