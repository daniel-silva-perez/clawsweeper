## Summary

Keep this issue open. Current main still does not classify legacy Word/OLE MIME types (`application/x-cfb` or `application/msword`) as binary in the media-understanding auto-embed guard, and the provided GitHub context shows open PR #54380 is the unmerged implementation candidate with closing syntax for this issue.

## Best Possible Solution

Review and land #54380, or an equivalent scoped patch, that adds `application/x-cfb` and `application/msword` to the media-understanding binary MIME guard in `src/media-understanding/apply.ts` and adds focused regression tests in `src/media-understanding/apply.test.ts`. After that merges, this issue can close as implemented.
