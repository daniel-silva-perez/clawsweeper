## Summary

Keep open. Current main still hardcodes non-Azure custom-provider onboarding to `input: ["text"]`, so the bug that #72171 addresses is not implemented or obsolete. The PR is a focused fix for linked issue #51869 and its remaining review comments are scope/coverage refinements, not evidence that the work should be closed.

## Best Possible Solution

Keep #72171 or a canonical follow-up open. The best product fix belongs in `applyCustomApiConfig()`: infer or ask for image-capable custom models, set `input: ["text", "image"]` for matching new non-Azure models, upgrade existing text-only entries on re-onboard without clobbering richer user metadata, and cover both paths in `src/commands/onboard-custom-config.test.ts`. Maintainers should decide whether Azure custom deployment vision detection is in scope for this PR or a separate follow-up.
