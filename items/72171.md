## Summary

Keep open. Current main still creates non-Azure custom-provider models with `input: ["text"]`, so the linked vision-support bug remains present. The PR is a focused implementation candidate with source and test changes for that exact path; review comments describe refinements to the PR rather than evidence that the work is obsolete or already shipped.

## Best Possible Solution

Keep this PR open or choose a single canonical follow-up among the open custom-provider vision PRs. The best fix belongs in `applyCustomApiConfig()`: mark known image-capable custom models as `input: ["text", "image"]`, upgrade prior text-only entries on re-onboard without clobbering richer user metadata, and keep focused tests for non-Azure, Azure, text-only, and existing-config paths.
