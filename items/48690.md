## Summary

Keep open. Current `main` still has no task-level timeout in the command lane queue, so the underlying stuck-lane problem is not implemented away, and the PR still appears to need follow-up to honor existing longer caller timeout budgets before conservative maintainer cleanup would close it.

## Evidence

- **Lane worker still waits forever on a hung task:** `drainLane()` still does a direct `await entry.task()` and only unblocks the lane after that promise settles. A never-settling task still prevents `completeTask()` and `pump()` from running for that lane.
  - file: [src/process/command-queue.ts:189](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/process/command-queue.ts#L189)
  - command: `nl -ba src/process/command-queue.ts | sed -n '155,220p'`
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **No timeout option exists on the queue API in main:** `QueueEntry` and `enqueueCommandInLane()` only carry `warnAfterMs` and `onWait`; there is no `taskTimeoutMs` plumbing or timeout-specific error type on current `main`. That means the PR's requested behavior is not already present.
  - file: [src/process/command-queue.ts:36](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/process/command-queue.ts#L36)
  - command: `nl -ba src/process/command-queue.ts | sed -n '36,58p'`
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Embedded runner callers currently rely on much longer agent budgets:** `runEmbeddedPiAgent()` enqueues session/global lane work without setting any lane task timeout, while the agent timeout resolver defaults to 48 hours and supports explicit no-timeout semantics. A fixed queue fallback would need to preserve those caller budgets.
  - file: [src/agents/pi-embedded-runner/run.ts:251](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/agents/pi-embedded-runner/run.ts#L251)

  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Cron callers also enqueue without queue-level timeout overrides:** Manual cron runs enqueue onto `CommandLane.Cron` with wait logging only, while cron timeout policy separately allows 10-minute generic jobs and 60-minute agent turns, with `undefined` meaning disabled for explicit nonpositive settings. The PR still needs caller-budget integration rather than a blanket fallback.
  - file: [src/cron/service/ops.ts:704](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/cron/service/ops.ts#L704)
