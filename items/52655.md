## Summary

Keep #52655 open. Current main still has only process-wide `main`, `cron`, and `subagent` lane concurrency wiring; it does not expose or resolve per-agent `lane` / `laneConcurrency` config, and the normal inbound/followup/memory embedded run call sites still do not pass an agent-specific lane. The PR remains a plausible focused implementation of a missing capability, with review comments addressed in the PR discussion and follow-up scope for #55566 incorporated in its head commit.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase it onto current main, preserve the typed config/schema/help/docs alignment, ensure generated config artifacts follow the repo's current generated-file policy, and validate the inbound, followup, memory-flush, startup, and hot-reload lane paths with focused tests before merge.
