## Summary

Keep PR #51319 open. Current main still has no macOS zombie detection in the shared `isPidAlive()` path, and session write locks still depend on that helper when deciding whether a lock owner PID is alive. The gateway lock path is partly mitigated by a port probe before PID liveness, but the PR's session-lock problem is not fully implemented on main. The PR is also not mergeable as-is because its branch includes unrelated runtime log artifacts noted in review, so the best path is a clean, narrow follow-up patch rather than cleanup-closing the item.

## Best Possible Solution

Keep the item open and ask for a clean patch that adds macOS zombie detection to `src/shared/pid-alive.ts` using `spawnSync("ps", ["-o", "state=", "-p", String(pid)])`, treats any state beginning with `Z` as zombie, adds mock-based cross-platform tests for the Darwin path, and removes the committed runtime log artifacts from the PR branch.

## Evidence

- **Current shared PID liveness only detects Linux zombies:** `isZombieProcess()` returns false on every non-Linux platform, so on macOS `isPidAlive()` still falls back to `process.kill(pid, 0)` and will treat a zombie PID as alive.
