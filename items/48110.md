## Summary

Keep open. Current `main` still has the unfixed code path: gateway session defaults ignore per-agent model overrides, and the TUI still consumes those defaults when a session has no persisted runtime model entry.

## Evidence

- **Gateway defaults still use global model only:** `getSessionDefaults` on current `main` has no `agentId` parameter and always calls `resolveConfiguredModelRef(...)`, so it resolves `agents.defaults.model.primary`, not the per-agent override path.

- **Per-agent-aware resolver exists but is not used for session defaults:** `resolveSessionModelRef` already switches to `resolveDefaultModelForAgent({ cfg, agentId })`, showing the intended per-agent resolution exists elsewhere, but `getSessionDefaults` does not use it.
  - file: [src/gateway/session-utils.ts:1082](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/src/gateway/session-utils.ts#L1082)
