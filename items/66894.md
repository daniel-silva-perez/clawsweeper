## Summary

Keep this PR open. Current main still has the `/compact` cancellation gap described by the linked bug: the manual compact handler calls `compactEmbeddedPiSession` without passing an abort signal and without registering the compaction in the active-run registry. The PR remains a relevant implementation candidate, though it needs maintainer review for the unrelated profile/launchd-label change and focused regression coverage.

## Best Possible Solution

Keep this PR open as the active implementation candidate for the linked `/compact` cancellation bug. The best path is to revise/review it so manual compaction registers an abortable active-run handle when the queued task starts, forwards an abort signal into compaction, cascades abort to any active reply operation, cleans up in `finally`, adds focused regression coverage, and removes or splits the unrelated profile/launchd-label change.
