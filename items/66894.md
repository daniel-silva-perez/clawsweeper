## Summary

Keep open. Current main still appears to have the manual `/compact` cancellation gap described by #66535: the command handler calls `compactEmbeddedPiSession` without an `abortSignal`, and the manual compaction path is not registered in `ACTIVE_EMBEDDED_RUNS`. This PR is still a relevant implementation candidate, though it should be reviewed for the unrelated profile-label change before landing.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #66535. The best path is to review or revise it so the manual `/compact` lane registers a compacting active-run handle when the queued task actually starts, wires an abort signal through to `compactEmbeddedPiSession`, cascades abort to any active reply operation, cleans up in `finally`, and removes or splits the unrelated profile-label change.
