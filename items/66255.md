## Summary

Current main does not provide a supported OpenClaw clipboard disable switch or native-crash isolation for the reported gateway crash path. It still ships `@mariozechner/pi-coding-agent@0.70.2`, whose upstream clipboard guard only skips native loading when Linux has no display or `TERMUX_VERSION` is set; the issue discussion includes a wider Fedora Wayland reproduction where `WAYLAND_DISPLAY` is present, so that guard would still load `@mariozechner/clipboard-linux-x64-gnu`. No open PR or superseding issue was found for the explicit opt-out requested in the discussion.

## Best Possible Solution

Keep this issue open. The best fix is either an upstream Pi/clipboard change that makes native clipboard loading explicitly optional and lazy, or an OpenClaw-owned gateway/runtime guard such as a documented `OPENCLAW_DISABLE_CLIPBOARD=1` path that prevents the native addon from loading before it can crash the process. Add focused regression coverage around the environment guard and document the supported workaround once the behavior is defined.
