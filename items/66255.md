## Summary

Current main does not yet provide a supported OpenClaw opt-out or isolation layer for the Pi native clipboard crash path. The issue discussion includes concrete coredump evidence and a useful `TERMUX_VERSION=1` workaround, but main still pins `@mariozechner/pi-coding-agent@0.70.2`, which can load `@mariozechner/clipboard` whenever Linux has `DISPLAY` or `WAYLAND_DISPLAY`, and that package still resolves to `@mariozechner/clipboard-linux-x64-gnu` on Linux x64 glibc.

## Best Possible Solution

Keep this issue open. The best path is either an upstream Pi/clipboard fix that makes native clipboard loading explicit and lazy, or an OpenClaw-owned gateway/runtime guard such as a documented `OPENCLAW_DISABLE_CLIPBOARD=1` path that prevents the native addon from loading before it can crash the process. Add focused regression coverage for the environment guard and document the supported workaround once the behavior is defined.
