## Summary

Current main does not fully resolve #71069. OpenClaw does support Gemma 4 only through canonical provider/model refs, and it intentionally resets stored session overrides that fall outside the configured agent model allowlist, but the reported reset path still emits only the generic fallback message and does not explain which Gemma ref was rejected or how to select a supported one.

## Best Possible Solution

Keep the issue open for a targeted UX/config fix: either accept an intentional alias/normalization for the reported Gemma variant if it maps to a real configured provider model, or improve the stored-override reset message to include the rejected provider/model key and point users to /models or canonical provider/model refs such as google/gemma-4-26b-a4b-it, openrouter/google/gemma-4-26b-a4b-it, ollama/gemma4, or an explicitly configured local provider model.
