## Summary

Keep #71784 open. Current main still routes live builtin memory search through a single query-embedding call with timeout only, while the retry classifier and existing retry loop cover batch/rate-limit style failures rather than the reported live `fetch failed` / socket-close transport failures.

## Best Possible Solution

Keep #71784 open. A conservative fix should add a narrow transient transport-error classifier and bounded backoff retry for live memory query embeddings, either in `embedQueryWithTimeout()` or a shared single-embed remote HTTP helper, while preserving immediate auth/config failures. Add focused tests for `fetch failed`, `other side closed`, `ECONNRESET`, `EPIPE`, `UND_ERR_SOCKET`, and TLS socket disconnects on the live query path, then decide from live validation whether dispatcher keep-alive tuning is also necessary.
