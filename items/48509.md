## Summary

Keep open. Current `main` does not implement a dedicated `before_persistence_write` hook, nor an equivalent durable-write context on existing plugin hooks. Plugins still only get generic tool interception plus transcript/message persistence hooks.

## Evidence

- **Hook registry has no persistence-write hook:** The public hook name union and exported hook list include `before_tool_call`, `tool_result_persist`, and `before_message_write`, but no `before_persistence_write` or similar durable-state hook.
  - file: [src/plugins/hook-types.ts:72](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/plugins/hook-types.ts#L72)
  - command: `nl -ba src/plugins/hook-types.ts | sed -n '60,115p'`
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Existing hook signatures are generic or transcript-scoped:** `before_tool_call` only exposes `{ toolName, params, runId, toolCallId }`. `tool_result_persist` and `before_message_write` are separate hooks for transcript/message persistence, not durable state categories like memory/file/cron/config/preference writes.
  - file: [src/plugins/hook-types.ts:318](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/plugins/hook-types.ts#L318)
  - command: `nl -ba src/plugins/hook-types.ts | sed -n '318,388p'`
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Runtime only passes generic tool-call context:** The agent runtime invokes `runBeforeToolCall` with `toolName`, normalized params, `runId`, and `toolCallId`; it does not attach write-target metadata, session-boundary metadata, or any persistence classification.
  - file: [src/agents/pi-tools.before-tool-call.ts:248](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/agents/pi-tools.before-tool-call.ts#L248)
  - command: `nl -ba src/agents/pi-tools.before-tool-call.ts | sed -n '248,285p'`
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Current persistence hooks are wired only around session transcript writes:** The session guard wrapper wires `before_message_write` and `tool_result_persist` around `SessionManager` persistence of agent messages/tool results. This matches transcript persistence only, not durable writes performed by other tools.
  - file: [src/agents/session-tool-result-guard-wrapper.ts:39](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/agents/session-tool-result-guard-wrapper.ts#L39)

Git history grep for `48509`, `before_persistence_write`, and `durable state` also returned no matching commits.

## Risks / Open Questions

- The contract gap is real, but the final design is still open: maintainers may prefer extending `before_tool_call` with persistence metadata instead of adding a brand-new hook.
- The request touches public plugin SDK surface, so closing it now would discard a concrete feature gap without evidence that current main already covers it.
