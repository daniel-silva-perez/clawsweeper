## Summary

Keep #48509 open. Current main has broad `before_tool_call`, transcript persistence hooks, plugin approval routing, and mutating-tool heuristics, but it does not expose a dedicated durable-state write hook or equivalent `writeTarget` / `crossesSessionBoundary` context. The related provenance and classification proposals remain open, and the related comments add external implementation input rather than superseding this request.

## Best Possible Solution

Keep the issue open as a plugin SDK/API design item. Maintainers should decide whether to add a dedicated `before_persistence_write` hook or enrich `before_tool_call` with a persistence context, then instrument durable write paths such as file/write/edit/apply_patch, memory promotion/flush, cron add/update/remove, config set/patch/apply, and saved preferences with docs and contract tests. Coordinate with #48503 and #48519 rather than closing this as implemented or ClawHub-only.
