## Summary

Keep #71964 open. Current main still unconditionally emits hard-coded version-manager/package-manager fallback dirs into gateway/node service PATH, while the audit still flags those same version-manager paths as non-minimal. The PR directly targets the still-present contradiction from #71944; it is not implemented on main and the only related later PR (#72109) is closed unmerged, so this is not safe to close as implemented or superseded.

## Best Possible Solution

Keep the PR open and either merge a polished version of its approach or ask for a small follow-up update: add an injectable existence predicate in `src/daemon/service-env.ts`, gate only hard-coded version-manager fallbacks on actual directory existence, keep explicit env-configured roots and stable user-bin dirs unconditional, and update daemon service-env/audit tests so the generated service PATH and doctor audit agree.
