## Summary

Keep this PR open. Current main still lacks the requested fail-fast behavior for missing local probe auth, and the PR remains a plausible implementation candidate after maintainer review of the auth-mode, override, redaction, and terminal-rendering concerns raised in discussion.

## Best Possible Solution

Keep this PR open as the implementation candidate. Revise/review it so fail-fast applies only to local loopback probes where auth is explicitly required but unresolved, preserves unset/none/trusted-proxy and explicit override behavior, avoids sending auto-resolved credentials to user-supplied targets without explicit intent, redacts/sanitizes failure reasons on every output surface, and lands with focused regression coverage.

## Risks / Open Questions

- The PR is not obsolete, but it should not land without resolving the override credential-forwarding question for user-supplied RPC/probe URLs.
- Failure reasons may expose SecretRef labels or config paths if propagated directly to normal user-facing output; redaction policy should be explicit.
- Terminal-safe rendering must cover every probe-error output path, not only daemon status.
- Targeted regression tests for probe auth, gateway status, shared status, and daemon status should run before merge.
