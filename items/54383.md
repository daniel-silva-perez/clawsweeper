## Summary

Keep open. Current main still budgets compaction against the 1M context window and forwards `reserveTokens` straight into the summarization path without clamping to the model's output cap, so the reported Anthropic `max_tokens > 128000` failure remains plausible.

## Evidence

- **Anthropic 1M context is still enabled from config:** `resolveContextTokensForModel()` returns `ANTHROPIC_CONTEXT_1M_TOKENS` when `params.context1m === true` for supported Anthropic models.
  - file: [src/agents/context.ts:437](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/context.ts#L437)

  - file: [src/agents/context-window-guard.ts:23](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/context-window-guard.ts#L23)
