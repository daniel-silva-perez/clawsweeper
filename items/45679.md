## Summary

Current `main` does not implement a Telegram config option that automatically sends verbose/streaming messages silently. The codebase supports explicit per-message `silent` delivery and a separate `silentErrorReplies` config for error replies, but there is no `silentVerbose`/`verboseSilent` config key or verbose-based auto-silent branch.
