## Summary

Keep open. Current `main` still has the reported Slack Socket Mode routing regression for name-keyed channel allowlists: inbound room events are matched ID-first, channel-name matching is disabled unless `dangerouslyAllowNameMatching` is enabled, and the async startup resolver only updates `ctx.channelsConfig` while leaving the cached `channelsConfigKeys` snapshot stale. That means a config keyed as `#claw`/`claw` can continue to miss routing even after startup resolution logs `channels resolved`, which matches the issue report and maintainer comment rather than disproving it.
