## Summary

Keep #69849 open. Current main still ships @agentclientprotocol/sdk 0.20.0, and the published 0.20.0 SDK schema still contains multiple optional/nullish array Zod fields without default([]), including the ACP session/setup and tool-call surfaces named by the report. OpenClaw’s ACP bridge consumes that SDK directly, and I found no OpenClaw-side normalization, dependency bump, or duplicate canonical issue that closes the reported Claude validation problem.

## Best Possible Solution

Keep the issue open under the core:mcp-acp surface. The best fix is to get @agentclientprotocol/sdk to generate default([]) or otherwise Claude-safe JSON Schema for optional/nullish array fields, then bump OpenClaw’s root dependency and the bundled acpx resolution/locks. If upstream cannot ship quickly, add a narrowly scoped OpenClaw schema-normalization workaround for the ACP SDK schemas used during Claude/API Pro tool registration, with a regression test covering the affected optional array fields.
