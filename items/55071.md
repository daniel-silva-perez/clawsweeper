## Summary

Current main still does not implement skill-driven plugin tool visibility. There is no `metadata.openclaw.activates.plugins` contract in skill metadata, the parser and snapshots do not preserve activation dependencies, and plugin tool construction is still driven by static tool policy allowlists plus enabled plugin state. Also, open PR #72533 references this issue with closing syntax and is unmerged, so this issue should stay open until that PR is reviewed, landed, or closed.

## Best Possible Solution

Keep this issue open for maintainer design and PR handling. The likely path is to add a typed `metadata.openclaw.activates.plugins` contract, parse and preserve it through skill entries/snapshots, define the exact pre-model activation trigger, and feed activated plugin ids into request-time plugin tool planning while preserving existing plugin enablement and tool deny policy. Maintainers should review #72533 separately; if it is still README-only as the provided context indicates, it should not be treated as an implementation of this feature.
