## Summary

Keep this PR open. Current main at 7c0fdae9b95bfcc667af3ad341b63ecc194464e4 still does not expose the requested trust identity fields on `PluginHookAgentContext`, and the main agent plus compaction hook contexts still omit `sourceProvider`, sender identity, group, and parent-session fields. The branch is stale and large, but it remains a real implementation candidate, is paired with the open same-author security-hooks umbrella #36671, and has plugin-author comments showing practical demand.

## Best Possible Solution

Keep this PR open for maintainer review or a narrowed rebase. The best path is an additive plugin SDK/context update that exposes `sourceProvider`, `senderId`, `senderName`, `senderIsOwner`, `groupId`, and `spawnedBy` on `PluginHookAgentContext`; derives normalized origin identity consistently; preserves the intended null semantics; threads the fields through early run, attempt, queued/memory, compaction, and reset hook contexts; and either completes or splits the outbound `sessionKey` message-hook correlation work.
