## Summary

Current `main` still treats `openclaw agent` as an agent-turn command, not a direct message-injection/send command. The docs say `--deliver` sends the agent's reply, and the implementation still runs the agent turn first and only then forwards resulting payloads to outbound delivery. If the turn yields no payloads, it logs `No reply from agent.` and returns success, so the reported gap is not implemented away on `main`.
