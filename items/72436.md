## Summary

Keep this draft PR open. Current main still lacks the proposed append-only `recovery-events.jsonl` sidecar and the PR is a focused implementation candidate for that missing behavior, not cleanup-close material.

## Best Possible Solution

Keep this draft PR open for maintainer review or narrowing. The best path is to evaluate the proposed core session recovery event log against the existing session store, transcript, compaction checkpoint, and outbound queue contracts, then either land a privacy-reviewed version or coordinate it with the adjacent passive recovery checkpoint work in #72209.
