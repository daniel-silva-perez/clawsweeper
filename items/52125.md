## Summary

Keep #52125 open. Current main still sends cron isolated announce text payloads directly through outbound delivery without the normal `maybeApplyTtsToPayload` transform, while direct reply dispatch does apply that transform. The existing cron delivery tests cover direct announce delivery and media/silent-token behavior, but not `[[tts]]` tag synthesis or stripping. The maintainer deep-review comment from 2026-04-25 also explicitly says this issue remains open on current main.

## Best Possible Solution

Implement a shared pre-delivery TTS transform for cron isolated announce delivery before `deliverOutboundPayloads`, using the same `maybeApplyTtsToPayload` contract as direct replies with the resolved delivery channel and agent id. Add a regression test where a cron `delivery.mode: "announce"` final reply containing `[[tts]]` or `[[tts:text]]` is converted to a synthesized media/voice payload and the visible text no longer contains raw TTS tags.
