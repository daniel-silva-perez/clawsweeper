## Summary

Current `main` already handles the reported `?token=` Control UI bootstrap path: the UI imports a query-string token as a one-time legacy fallback, strips it from the URL, and passes the resulting token into the gateway WebSocket client. Tests and docs on `main` cover that exact flow, so the reported behavior is implemented/fixed on `main` even though the issue was filed against `2026.03.11`.

## Evidence

The inline comment explicitly says query-param tokens remain a one-time legacy fallback for compatibility.
