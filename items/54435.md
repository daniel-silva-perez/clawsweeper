## Summary

Keep open. Current `main` still special-cases literal `session.store` paths by loading only one store file and assigning every entry to the default agent, which matches the reported failure mode. The multi-agent discovery path exists elsewhere, but this gateway listing function does not use it for literal store paths. The latest release `v2026.4.22` still contains the same code.

## Evidence

- **Current main still collapses literal store paths to the default agent:** `src/gateway/session-utils.ts:1015-1035` checks `!isStorePathTemplate(storeConfig)` and then loads exactly one `storePath`, resolves `defaultAgentId`, and merges every session entry with that agent id before returning. `isStorePathTemplate` is only `store.includes("{agentId}")` at `src/gateway/session-utils.ts:604-605`, so any literal per-agent path takes this single-store branch.
  - file: [src/gateway/session-utils.ts:1015](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/gateway/session-utils.ts#L1015)
