## Summary

Current main already implements Ollama image-tool/media-understanding routing. The Ollama plugin now registers an image-capable media-understanding provider, there is a regression test covering `ollama/*` routing for the image tool, and the docs/changelog now describe the supported config as `agents.defaults.imageModel.primary: "ollama/<model>"` with `models.providers.ollama.apiKey: "ollama-local"` as an auth marker rather than a provider id.

## Close Comment

Closing as implemented on current `main` after a Codex review. OpenClaw now registers Ollama as an image-capable media-understanding provider (`extensions/ollama/src/media-understanding-provider.ts:13`, registered from `extensions/ollama/index.ts`), and there is a regression test covering image-tool routing for `ollama/*` (`extensions/ollama/index.test.ts:544`). The changelog records this fix in `v2026.4.21` (`CHANGELOG.md:346`). Current docs also show the supported config shape as `agents.defaults.imageModel.primary: "ollama/<model>"`, while `ollama-local` is used as the API-key marker under `models.providers.ollama.apiKey`, not as the provider id (`docs/providers/ollama.md:213`, `docs/providers/ollama.md:288`). If this still reproduces on a current build with the canonical `ollama/...` provider/model config, please open a fresh issue with the exact updated config and logs.
