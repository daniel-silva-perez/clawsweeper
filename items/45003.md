## Summary

Keep open. Current `main` still archives sessions on daily reset, but there is no built-in restore/reindex script and the checked recall/listing surfaces do not expose `.reset.*` archives as recovered history.

## Evidence

- **Daily reset still treats any pre-boundary session as stale:** `evaluateSessionFreshness()` still marks a session stale when `updatedAt < dailyResetAt`, which matches the issue's described first-post-upgrade trigger.
  - file: [src/config/sessions/reset-policy.ts:72](https://github.com/openclaw/openclaw/blob/40be5ad58187fd20b8defe5fc30e649190c131a8/src/config/sessions/reset-policy.ts#L72)

  - sha: [40be5ad58187](https://github.com/openclaw/openclaw/commit/40be5ad58187fd20b8defe5fc30e649190c131a8)
- **Gateway chat history does not fall back to reset archives:** `chat.history` loads the session entry and reads messages through `readSessionMessages()`. That reader resolves only normal transcript candidates and picks the first existing file; the candidate resolver does not scan `.reset.*` archives.
  - file: [src/gateway/server-methods/chat.ts:1874](https://github.com/openclaw/openclaw/blob/40be5ad58187fd20b8defe5fc30e649190c131a8/src/gateway/server-methods/chat.ts#L1874)
