## Summary

Keep #52457 open. Current main still has only explicit `streamTo:"parent"` plus a narrow implicit path for subagent requester sessions; it does not include `acp.progressForward`, the default-on behavior, or the added thread-bound guards from the PR. This is a core ACP behavior/config change, not a ClawHub item or an already-shipped/superseded fix.

## Best Possible Solution

Keep #52457 open for maintainer review. If the default-on UX is accepted, adapt the PR onto current main with `acp.progressForward` in config schema/help/docs, regenerated config baselines, tests for opt-out/default-on behavior, and hardened thread-bound detection. If maintainers do not want default-on progress chatter, close manually with that product rationale rather than through ClawSweeper cleanup.
