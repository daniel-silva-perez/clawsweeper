## Summary

Keep #39317 open. Current main still treats an exit-0 `codex-cli` JSONL run as successful after parsing stdout text, without any semantic failure channel for sandbox-denied command/tool records. The proposed focused fix in #39460 was closed unmerged, and the related #66081 artifact-finalization PR is explicitly scoped away from failure classification.

## Best Possible Solution

Keep the issue open. Revive the substance of #39460 on current main: parse structured Codex JSONL records for sandbox-denied or failed command/tool/action/error/status signals after exit 0, throw a `FailoverError` before `buildCliRunResult`, add an explicit local `sandbox_denied` failover/status/auth-profile policy if maintainers want that public contract, avoid assistant-message keyword matching, and add parser plus runner regression tests for real denials and false-positive discussion cases. Treat #66081 as related final-answer reliability work, not a replacement for semantic failure classification.
