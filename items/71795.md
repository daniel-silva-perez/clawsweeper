## Summary

Keep #71795 open. Current main still queues every accepted Mattermost inbound user post as a trusted system event whose text includes the user message preview, and the shared reply path prepends queued system events to the next prompt as `System:` lines. Discord does not do this for normal user messages; it only enqueues Discord-specific system events and otherwise sends the clean user body through `BodyForAgent`.

## Best Possible Solution

Keep the issue open and fix the Mattermost monitor path so ordinary inbound user posts are not enqueued as trusted system events. Preserve `BodyForAgent: bodyText`, keep real event surfaces such as reactions/interactions as system events, and add a Mattermost regression test proving a normal post dispatches clean user content without adding a prompt-leading `System: Mattermost message ...` event. Compare against the existing Discord pattern when shaping the test.
