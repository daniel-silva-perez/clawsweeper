## Summary

Keep open. Current main at e962381dbf1ecdb1cd4784fdeb6c2e07ddd3435b still computes both memoryFlush and preflight compaction token thresholds from `reserveTokensFloor`, while the Pi settings path treats `reserveTokensFloor` as a minimum floor under configured `reserveTokens`. The provided GitHub context also shows an open same-author PR, #69379, with closing syntax for this issue, so the issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review or land #69379, or an equivalent core fix that resolves an effective compaction reserve as `max(configured reserveTokens, reserveTokensFloor)` for both preflight compaction and memoryFlush thresholds, with regression coverage for `reserveTokens > reserveTokensFloor`. After that merges, this issue can close as implemented.
