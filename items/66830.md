## Summary

Keep open. Current main still computes the preflight and memoryFlush thresholds from `reserveTokensFloor`, while the existing Pi settings path treats `reserveTokensFloor` as a floor under configured `reserveTokens`. There is also an open same-author implementation PR, #69379, whose body explicitly uses closing syntax for this issue, so the issue should stay open until that PR or an equivalent fix merges.

## Best Possible Solution

Keep this issue open and review/land #69379 or an equivalent core fix that resolves an effective compaction reserve as `max(configured reserveTokens, reserveTokensFloor)` for both preflight compaction and memoryFlush thresholds, with regression tests covering `reserveTokens > reserveTokensFloor`. After the fix merges, the issue can close as implemented.
