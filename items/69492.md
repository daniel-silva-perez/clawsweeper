## Summary

Keep #69492 open. Current main still has no `SystemEvent` audience/classification field, no completion-audience propagation from exec defaults, and no system-event peer predicate at transcript/history boundaries. The code has useful tactical hygiene, but existing tests still assert user-relay behavior for exec completion events when delivery is available, and the issue discussion says PR #72201 only addresses one awareness-event slice rather than the full delivery-routing/consumer-path problem.

## Best Possible Solution

Keep #69492 open as the canonical design/work item until a maintainer-approved system-event audience contract lands, or until maintainers explicitly choose narrower tactical fixes as the canonical path. The next implementation should define whether `SystemEvent.audience` is hidden runtime-context only or also delivery-routing, wire it through the relevant producers/consumers, and add regression tests for exec, cron awareness, heartbeat relay, `chat.history`/session history display, and channel delivery boundaries.
