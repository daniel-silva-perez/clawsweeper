## Summary

Keep open. Current `main` has partial Android SMS node support, but it does not implement the requested `sms.read` command or an SMS messaging channel. The shipped command surface is `sms.send` plus `sms.search`, and the SMS query parser accepts filters like `phoneNumber`, `keyword`, `includeMms`, `limit`, and `offset` rather than the requested `id` / `threadId` lookup contract. The supported channel docs also list many messaging channels but no SMS channel.
