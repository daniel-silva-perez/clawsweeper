## Summary

Keep #38781 open. Current main still has the reported macOS companion-node contract gap: the agent node exec path invokes `system.run.prepare`, the gateway rejects undeclared commands, and the Swift macOS node still neither defines, advertises, nor dispatches `system.run.prepare`. Related PR #69725 is a useful alternate fallback path, but it is only related context from the provided data, not a merged superseding fix on current main.

## Best Possible Solution

Keep this PR open until maintainers either merge a direct macOS `system.run.prepare` implementation or intentionally supersede it with a merged fallback such as #69725. The strongest implementation path is to align the Swift macOS node with the TypeScript node-host contract: add the shared command/prepare payload types, advertise `system.run.prepare`, dispatch it in `MacNodeRuntime`, return the same canonical `systemRunPlan` shape expected by `executeNodeHostCommand`, and add focused macOS tests for declared commands, prepare payload shape, wrapper/rawCommand consistency, and path hardening.
