## Summary

Current `main` has partial session-management UX already: the Control UI has a Sessions page with search, inline label editing, delete, and navigation into chat; the chat view has a grouped session switcher; and the TUI has a searchable session picker. But the issue asks for a broader Topic/Session Management UI similar to Cherry Studio, and that is not fully implemented on `main`: the web chat does not have a sidebar topic list, the exposed session row data does not include message count / created-at / tags, and the available actions do not cover explicit topic create/archive/restore/import/export flows.
