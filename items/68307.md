## Summary

Keep this PR open. Current main still calls `resolveBootstrapContextForRun` from the commands system-prompt path without a `warn` callback, while truncation warnings are only emitted when a caller supplies `warn`. The compact, attempt-runner, and CLI prepare paths already wire `makeBootstrapWarn` to WARN logging, so the PR addresses a real consistency gap rather than obsolete work.

## Best Possible Solution

Review and land this focused PR, or an equivalent main-branch patch, by wiring `makeBootstrapWarn` and a subsystem logger into `resolveCommandsSystemPromptBundle`; add a narrow test so future changes cannot silently drop bootstrap truncation warnings in the commands/system-prompt path.
