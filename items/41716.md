## Summary

Keep #41716 open. Current main does not implement the requested bidirectional node file transfer: the `nodes` agent tool has no `file_pull`/`file_push` actions, the node command constants and gateway policy have no `file.read`/`file.write`, and current CLI/docs/tests only cover existing node invoke/media/status surfaces. The provided discussion identifies PR #67495 as the active implementation, but that PR is still open and not merged into current main.

## Best Possible Solution

Keep #41716 open as the feature tracking issue and use PR #67495, or a maintainer-approved successor, as the implementation path. The best fix belongs in the core gateway/nodes surface: add the node command contract and policy entries, implement `nodes` tool file_pull/file_push behavior with size/security checks, add targeted tests/docs, and only close the issue once that work is merged and shipped.
