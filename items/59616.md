## Summary

Current main still appears to match the reported bug: the gateway config reloader watches only the primary config path and does not add or remove watches for `$include` files. The existing include scanner is present, but `startGatewayConfigReloader` does not use it. Related PR #59632 was closed, not merged, per the provided timeline, so the issue remains actionable.

## Evidence

- **Reloader watches only the root config path:** `startGatewayConfigReloader` creates the chokidar watcher with `chokidar.watch(opts.watchPath, ...)` and registers `add`, `change`, and `unlink` handlers, but there is no `watcher.add()` or include-path watch set update in the shown current-main implementation.
  - file: [src/gateway/config-reload.ts:1358](https://github.com/openclaw/openclaw/blob/d6ef1fcf24eea3a5985d653a3baf400e32aa1b75/src/gateway/config-reload.ts#L1358)
  - command: `web open github main file at provided current main SHA`
  - sha: [d6ef1fcf24ee](https://github.com/openclaw/openclaw/commit/d6ef1fcf24eea3a5985d653a3baf400e32aa1b75)
- **Include scanner exists separately:** `collectIncludePathsRecursive` exists and resolves nested `$include` paths, returning discovered include files, matching the issue's suggested infrastructure, but this function is not imported or called by `src/gateway/config-reload.ts` in current main.
  - file: [src/config/includes-scan.ts:494](https://github.com/openclaw/openclaw/blob/d6ef1fcf24eea3a5985d653a3baf400e32aa1b75/src/config/includes-scan.ts#L494)
