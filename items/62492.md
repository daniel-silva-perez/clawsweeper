## Summary

Keep open. Current `main` still ignores `update.run` responses where `result.status === "skipped"` because the UI only shows feedback when `ok === false`, while the backend still maps skipped outcomes to `ok: true`. The same behavior is present in the latest release tag `v2026.4.23` (`a9797214338ba31c52c796adbb75afb16e0684a9`).

## Evidence

- **UI only surfaces update failures when `ok === false`:** `runUpdate()` clears `state.lastError`, calls `update.run`, and only sets a message inside `if (res && res.ok === false)`. A skipped response therefore leaves the banner/button with no visible feedback.
  - file: [ui/src/ui/controllers/config.ts:178](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/ui/src/ui/controllers/config.ts#L178)
  - command: `nl -ba ui/src/ui/controllers/config.ts | sed -n '160,230p'`
  - sha: [ecac696643c6](https://github.com/openclaw/openclaw/commit/ecac696643c604c6d854341bfe6424bfa31423e9)
- **Backend explicitly returns `ok: true` for skipped outcomes:** `update.run` responds with `ok: result.status !== "error"`, so any `result.status === "skipped"` payload is treated as successful by the frontend gate above.
  - file: [src/gateway/server-methods/update.ts:129](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/src/gateway/server-methods/update.ts#L129)
