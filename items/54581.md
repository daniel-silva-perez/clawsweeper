## Close Comment

Closing from a Codex review: current `main` already has this fixed. Manual model selection now filters out the internal `openrouter/auto` route and keeps concrete free OpenRouter models selectable (`src/flows/model-picker.ts:29`, `src/flows/model-picker.ts:145`, covered by `src/commands/model-picker.test.ts:89` and `src/commands/model-picker.test.ts:405`). When a model is selected, OpenClaw persists that exact model string instead of rewriting it to `openrouter/auto` (`src/flows/model-picker.ts:538`, `src/plugins/provider-model-primary.ts:47`). Current main also resolves the legacy OpenRouter free alias to a concrete configured `openrouter/...:free` model rather than drifting to the auto/default route (`src/agents/model-selection-shared.ts:121`, `src/agents/model-selection.test.ts:1146`). The checked-in changelog records this alias fix in the `2026.4.15` notes (`CHANGELOG.md:552`, `CHANGELOG.md:660`). If you can still reproduce this on a current build, open a fresh issue with the exact selected model id and the resulting config snippet.
