## Summary

Keep #6615 open. Current main still lacks the requested exec-approvals denylist mode, `ask: "on-match"`, and denylist argument matching. The discussion links useful external policy-engine prior art, but current OpenClaw core surfaces remain allowlist/full/deny only, and the related implementation PRs were closed unmerged.

## Best Possible Solution

Keep #6615 open until OpenClaw either implements denylist/on-match/argument-aware deny matching across the exec approvals model, config/schema, runtime policy, gateway protocol, CLI/session surfaces, Control UI/macOS surfaces, docs, and tests, or maintainers explicitly redirect the capability to ClawHub with any missing plugin authorization API tracked separately.
