## Summary

Keep open. The issue is a concrete, recent feature request from a non-maintainer with no protected labels, and I did not find high-confidence evidence that current main already supports a `tools.exec.shell` default shell executable. Current exec docs describe policy/routing knobs such as `tools.exec.host`, `security`, and `ask`, but not a global shell override, so none of the allowed close reasons apply.

## Evidence

- **Issue request is concrete and recent:** The provided GitHub context for #59998 asks for an optional `tools.exec.shell` string in `openclaw.json` to globally select bash/Git Bash/Cygwin/etc. for exec commands. It was opened 2026-04-03, less than 60 days before this review, has no comments or timeline, and has no protected labels.
- **Current docs cover exec policy, not shell selection:** The exec approvals docs describe `tools.exec.*` as policy input and list behavior around `tools.exec.host=auto`, `tools.exec.security`, and `tools.exec.ask`; this does not satisfy the requested default shell executable setting.
