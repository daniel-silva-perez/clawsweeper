## Summary

Current main still does not implement the requested value-first first-run onboarding. QuickStart, Control UI welcome suggestions, and macOS onboarding chat reduce some friction, but the primary onboarding path still completes setup and then starts bootstrap/persona work before offering a runnable example task/template experience.

## Best Possible Solution

Keep this open for maintainer UX/product triage. A strong implementation would add a first-run path that performs only the minimum required setup for a working model/session, presents a few bootstrap templates or example tasks, lets the user run one quick useful task, and then offers optional agent name, reply style, symbols/avatar, persona, channel, search, skill, and other customization steps. Likely touch points are the CLI wizard, wizard finalization/bootstrap handoff, macOS onboarding page order and kickoff prompt, Control UI first-chat affordances, public onboarding docs, and focused wizard/onboarding tests.
