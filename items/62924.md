## Summary

Keep open. Current main captures media-understanding provider/model in structured runtime objects and exposes successful media decisions in /status, but the agent-visible inbound media body still only renders User text plus Transcript/Description. The issue specifically asks to expose the actual chosen media model in the current-turn inbound body, and that path is not implemented on main.

## Best Possible Solution

Keep the issue open and implement this in the media-understanding body/prompt path: render a concise current-turn Analysis model line from MediaUnderstandingOutput provider/model or MediaUnderstandingDecisions chosen provider/model, with tests for image/audio/video and absent-telemetry behavior. The /status media line can remain as separate diagnostic telemetry, but it does not replace the requested inbound-body signal.
