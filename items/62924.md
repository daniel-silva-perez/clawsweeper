## Summary

Current main still has the reported gap. Media-understanding execution records provider/model telemetry, and status output can display chosen provider/model, but the inbound body formatter used for BodyForAgent still renders only media headers, user text, and transcript/description text. The reporter's validated local workaround remains aligned with a missing upstream formatter/apply-path change.

## Best Possible Solution

Keep this issue open. Implement a narrow change in src/media-understanding/format.ts to render an Analysis model line from output.provider/output.model for current-turn successful media-understanding outputs when known. Add focused coverage in src/media-understanding/format.test.ts and an apply-path regression in src/media-understanding/apply.test.ts proving BodyForAgent receives the telemetry without guessing when unavailable.

## Risks / Open Questions

- A fix should only expose current-turn media-understanding telemetry that actually exists; it should not infer or reuse a conversation model when provider/model is unavailable.
- Synthetic placeholder outputs such as tiny-audio fallbacks should either omit the analysis-model line or label it clearly, so users do not mistake OpenClaw-generated placeholders for real provider analysis.
