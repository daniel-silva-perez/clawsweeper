## Summary

Keep PR #62991 open. Current main still does not directly merge arbitrary `models.providers.<id>.models[]` into `loadModelCatalog()`, and the gateway image-support path still fails closed when a configured-only multimodal provider is absent from that catalog. The PR addresses a real remaining gap, but it should be revised for canonical provider-id dedupe and covered by a regression test before maintainer cleanup closes it.

## Best Possible Solution

Keep the PR open. The best fix is to add a shared, canonical configured-provider merge into `loadModelCatalog()`, preferably reusing or aligning with `buildConfiguredModelCatalog()`, applying `shouldSuppressBuiltInModel`, canonical provider-id dedupe, valid input filtering, and focused tests for custom multimodal providers plus alias collisions.
