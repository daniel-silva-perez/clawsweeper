## Summary

Keep open. Current main has plugin-owned doctor compatibility hooks for legacy config migrations and channel-specific warnings, but it does not expose a general plugin API for adding arbitrary `openclaw doctor` checks/repairs. Because the request is specifically for upgrade-stable doctor extensibility with `--fix` behavior, this is a plausible core/plugin-SDK capability gap rather than a ClawHub-only plugin request.

## Best Possible Solution

Keep the issue open as a plugin-SDK/core design item. The best path is an additive doctor contribution API, likely a typed `registerDoctorCheck`/doctor contribution surface with read-only diagnostics plus explicit repair hooks that respect `--fix`, `--non-interactive`, config persistence, plugin trust/loading policy, and docs/tests. Existing `doctor-contract-api` should stay scoped to compatibility migrations unless intentionally expanded and documented.
