## Summary

Keep this issue open. Current main exposes partial Dreams visibility: enabled/idle state, promoted-today and signal counts, phase next-run times, and a diary reader. It still does not expose the requested last run timestamp, run result or partial-failure state, per-run processed trace count, or diary updated/workspace status in the Dreaming UI. The existing issue discussion reached the same conclusion and remains consistent with current main.

## Best Possible Solution

Keep this issue open and implement a memory-core-owned Dreaming run summary. Extend doctor.memory.status or add a sibling Dreaming status method to expose last sweep time, outcome such as ok/error/skipped/partial, processed/promoted/failed counts, diary path and updatedAt/workspace status, and next run. Then render those fields in the Dreams tab and add focused gateway status, UI controller, and view tests.
