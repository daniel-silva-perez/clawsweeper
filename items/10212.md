## Summary

Current main already ships the requested built-in startup hook path. OpenClaw documents and implements `boot-md`, which runs workspace `BOOT.md` on gateway startup, and release notes show the BOOT startup wiring shipped in `2026.2.19`.

## Risks / Open Questions

- `boot-md` provides the supported startup automation path, but the issue body also asks OpenClaw to identify user-specific hot-edited installed `dist/` files; that local patch inventory is not derivable from the repository itself.

## Close Comment

Closing as implemented on main after a Codex review. OpenClaw already ships the built-in startup hook path this request asked for: `boot-md` runs workspace `BOOT.md` on gateway startup (`docs/automation/hooks.md:202`, `src/gateway/boot.ts:36`, `src/hooks/bundled/boot-md/handler.ts:22`). The BOOT startup wiring was also shipped in release `2026.2.19` (`CHANGELOG.md:4533`). What remains in this issue body is local operator work around a one-off edited installed `dist/` tree, which the repository cannot infer or manage from source control. If there is still a docs gap around using `BOOT.md` for this workflow, please open a narrower follow-up with the exact desired operator flow.
