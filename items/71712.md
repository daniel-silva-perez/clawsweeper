## Summary

Keep open. Current main has a broad owner/admin cron surface and an owner-only agent cron tool, but it does not implement the RFC’s narrower `schedule_task` contract: playbook allowlists, gateway-injected non-forgeable creator provenance, recurring `activeUntil`, per-agent quota, or schedule-create audit entries. The issue is recent, has a confirming usage comment, and describes a real core scheduler/security boundary rather than ClawHub-only work.

## Best Possible Solution

Keep the issue open for maintainer design. The likely core path is a separate gateway-mediated scheduling tool/RPC that accepts a playbook identifier plus validated params and schedule, injects creator/session provenance and created time from the authenticated gateway context, enforces recurring expiration and per-agent quotas, writes schedule-create/update audit entries, and documents the contract alongside Scheduled tasks and Background tasks. Coordinate `activeUntil` with #71711 and provenance semantics with the broader provenance work rather than treating this as a ClawHub plugin item.
