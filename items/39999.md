## Summary

Keep open. Current main still fails and reports packaged global update verification/doctor problems, but it does not automatically reinstall the previous package version after a post-update doctor failure. The PR remains useful, though its proposed guard has an unresolved logic concern and would need revision before merge.

## Best Possible Solution

Keep the PR open and revise it into the active package update path: after a successful package-manager update, if post-update verification or `openclaw doctor --non-interactive` fails and `beforeVersion` is known, reinstall `${packageName}@${beforeVersion}`, rerun doctor when the rollback entrypoint exists, preserve the overall error result, and update docs/tests if maintainers accept automatic rollback as product behavior.

## Risks / Open Questions

- The PR patch likely needs rebasing onto the current package update flow, which now includes global install verification, entrypoint resolution, and restart verification changes.
- Rollback should be gated so failed package-manager updates do not trigger a redundant reinstall of the already-installed prior version and obscure the original failure.
- A shipped fix should include focused tests for doctor failure after successful update, verification failure after successful update, and failed update with no rollback.
