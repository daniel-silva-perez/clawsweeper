## Summary

Current `main` already uses fully qualified catalog model refs in the Control UI picker, preserves already-qualified refs instead of prepending the active provider, and has regression tests covering the reported failure mode.

## Close Comment

Closing this as implemented on `main` after a Codex review. The Control UI model picker now builds option values from the configured catalog’s full `provider/model` ref (`ui/src/ui/chat-model-ref.ts:266`), preserves already-qualified refs instead of prepending a stale current-session provider (`ui/src/ui/chat-model-ref.ts:149`), and sends the selected value unchanged in `sessions.patch` (`ui/src/ui/chat/session-controls.ts:251`). There is also regression coverage for the exact payload shape (`ui/src/ui/chat/session-controls.test.ts:160`) and for stale-provider qualified refs (`ui/src/ui/chat-model-ref.test.ts:225`). The fix is documented in the `v2026.4.10` changelog under Sessions/model selection (`CHANGELOG.md:863`).
