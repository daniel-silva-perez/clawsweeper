## Summary

Keep open. Current main does not implement the PR's cross-account Discord thread-binding isolation. Discord preflight still resolves runtime bindings only for the current bot account and then falls back to configured/general routing when no local binding is found, so the reported non-owning-bot path remains plausible. The PR discussion shows useful fixes to review feedback, but the equivalent behavior is not present on main.

## Best Possible Solution

Keep the PR open or replace it with a narrow equivalent fix: add a Discord cross-account active-thread-binding lookup after local binding miss, suppress generic routing for non-owning accounts, ignore stopped/removed/disabled/stale foreign owners using the effective config, and add focused regressions for foreign-owner suppression plus stale/disabled-account cases.
