## Summary

Keep this PR open. Current main still mirrors any trimmed non-empty delivery-mirror text into session transcripts, and I did not find an equivalent operational-alert ingress filter or regression coverage on main. The PR remains a focused implementation candidate, with maintainer judgment still needed on whether the filter belongs in the generic transcript mirror layer or closer to the alert producer.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to either land a narrowly scoped, regression-tested ingress filter for known non-user-facing operational alerts, or ask the author to move the suppression closer to the alert producer if owner-specific backup/git text should not live in generic session transcript code. Broader delivery replay and duplicate recovery behavior can remain tracked separately by #51628.
