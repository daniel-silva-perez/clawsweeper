## Summary

Keep this PR open. Current main still mirrors any non-empty text into `provider: "openclaw", model: "delivery-mirror"` transcript entries, and I did not find an equivalent operational-alert ingress filter or regression coverage on main. The related delivery-mirror fixes reduce other duplicate/replay paths, but they do not implement this PR's narrow behavior of dropping stale operational alert text before it is persisted as transcript mirror content.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to either merge a narrowly scoped, tested ingress filter in `src/config/sessions/transcript-mirror.ts`, or ask the author to move the filtering closer to the alert-producing owner if maintainers do not want owner-specific backup/git alert text in generic session transcript code. Broader delivery replay work can remain tracked separately by #51628.
