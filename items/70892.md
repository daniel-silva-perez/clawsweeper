## Summary

Keep open. Current main still lacks user-facing per-section prompt gates for the CLI quick reference, assistant/reply tag guidance, or messaging component guidance. The available_skills part is partially covered by the existing agents.defaults.skills / agents.list[].skills allowlist, but the broader request is not implemented and belongs in core prompt/config code rather than ClawHub.

## Best Possible Solution

Keep the issue open for a focused core config/docs change: add typed, schema-backed prompt section gates if maintainers want this operator control; document how they interact with promptMode, prompt cache boundaries, and systemPromptOverride; and reuse or extend the existing agents.defaults.skills allowlist instead of duplicating available_skills filtering.
