## Summary

Keep open. Current `main` has a partial Overview `Model Auth` card that can show `% left` usage windows for monitored OAuth/token providers, but it does not implement the broader Control UI usage-limits surface requested here. API-key-only providers are explicitly excluded, and the Usage tab only loads session/cost history, not provider quota summaries.

## Evidence

- **Overview card is limited to model-auth monitoring:** The dashboard surface is a `Model Auth` card. Its own comments say it is hidden for `api-key-only` setups after load, and its hint text only renders usage windows attached to monitored auth providers.
  - file: [ui/src/ui/views/overview-cards.ts:142](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/ui/src/ui/views/overview-cards.ts#L142)
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **UI helper excludes API-key-only providers:** `isMonitoredAuthProvider()` explicitly excludes API-key-only providers and only includes missing providers or profiles of type `oauth`/`token`.
  - file: [ui/src/ui/model-auth-helpers.ts:12](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/ui/src/ui/model-auth-helpers.ts#L12)
