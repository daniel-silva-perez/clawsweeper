## Summary

Keep this issue open. Current main still routes HTTP MCP tools/call results through a text-only normalizer that preserves only type and text, so the reported image/audio/resource field loss is still present. The supplied GitHub context also shows open PR #68046 as an unmerged implementation candidate that directly targets this issue, so the issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Review and land PR #68046 if its pass-through behavior and tests meet the gateway contract. Otherwise apply the same minimal fix in src/gateway/mcp-http.handlers.ts: preserve typed MCP content blocks and only synthesize text blocks for strings or malformed entries, with regression coverage in src/gateway/mcp-http.test.ts for image, audio, resource, and resource_link blocks.
