## Summary

Current main still leaves the hooks CLI on the full plugin diagnostics path. The broader startup preload mitigation that addressed #64306 does not cover `openclaw hooks list/info/check` or the enable/disable validation path, which still load plugin diagnostics and therefore can still hit the Windows/Node plugin module-loading hang described in the report.

## Best Possible Solution

Keep this open and fix the hooks CLI path directly. `openclaw hooks list/info/check/enable/disable` should derive plugin-managed hook visibility from manifest/config/workspace snapshot data, or from a bounded hook-specific metadata API, without importing full plugin runtime modules. Add regression coverage proving these commands avoid the `loadModules: true` diagnostics path while still showing plugin-managed hooks as documented.
