## Summary

Keep open. The broad `status`/`health` bootstrap shape from the linked #64306 has been narrowed on current `main`, but the hooks-specific commands reported here still run through full plugin diagnostics/module loading during `hooks list` and `hooks enable`, so there is not enough hook-specific Windows-safe evidence to close this as implemented. ([github.com](https://github.com/openclaw/openclaw/issues/70451))

## Best Possible Solution

Keep #70451 open and address the hooks CLI path directly: make `hooks list`, `info`, `check`, and `enable` use a cold metadata/config path where possible, or add an explicit Windows-safe/config-only mode that avoids full plugin runtime diagnostics before enabling a named hook. Add a Windows/Node 24 regression or smoke for `openclaw hooks --help`, `hooks list`, and `hooks enable <bundled-hook>`, and document the supported config-only workaround plus whether a gateway API exists.
