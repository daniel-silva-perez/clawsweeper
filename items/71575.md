## Summary

Keep #71575 open. Current main still makes Telegram preview tool progress visible by default, including `Working…`, tool names, command/progress text, and exec output surfaced through item progress events. The related #71663 discussion already treats #71575 as the canonical remaining issue rather than a duplicate to close.

## Best Possible Solution

Keep #71575 open as the canonical Telegram bug. The likely product/code fix is to make Telegram partial preview hide or strictly sanitize tool metadata and raw output by default, probably making `streaming.preview.toolProgress` opt-in or generic-only, then update Telegram dispatch tests, docs, and the related #71663 duplicate state to match.
