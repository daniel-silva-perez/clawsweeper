## Summary

Keep #71575 open. Current main still intentionally exposes Telegram tool-progress preview lines by default, including `Working…`, tool names, and command/progress text. The related #71663 was closed as a duplicate of this issue, so #71575 remains the canonical open tracker rather than something current main has fixed.

## Best Possible Solution

Keep #71575 open as the canonical Telegram tool/progress visibility bug. The likely fix belongs in the Telegram preview dispatch path and/or shared preview-progress contract: either default Telegram `streaming.preview.toolProgress` to false for user chats, or strictly sanitize progress previews so they contain only safe high-level status and never raw command output, tool internals, or duplicates. Update `extensions/telegram/src/bot-message-dispatch.test.ts`, relevant Pi tool-event coverage, and Telegram/streaming docs to match the chosen product behavior.
