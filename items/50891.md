## Summary

Keep open. Current `main` still limits the bundled `session-memory` internal hook to manual `/new` and `/reset` command events. Automatic idle/daily session rollovers now emit plugin `session_end` lifecycle hooks, but the bundled internal hook and its docs were not updated to consume those paths, so the reported gap remains.

## Evidence

- **Bundled hook metadata still targets manual commands only:** `src/hooks/bundled/session-memory/HOOK.md:3` says the hook saves memory when `/new` or `/reset` is issued, and `src/hooks/bundled/session-memory/HOOK.md:10` registers only `command:new` and `command:reset`.
  - file: [src/hooks/bundled/session-memory/HOOK.md:3](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/hooks/bundled/session-memory/HOOK.md#L3)
  - command: `nl -ba src/hooks/bundled/session-memory/HOOK.md | sed -n '1,30p'`
