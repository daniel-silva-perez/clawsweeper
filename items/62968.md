## Summary

Keep open. Current main still matches the reported diagnostic gap: completion-mode direct-send failure is preserved only in dispatch phases, queue fallback can return a delivered top-level result, and the caller only logs direct-send errors when overall delivery failed.

## Best Possible Solution

Keep the issue open and fix it in the core agent announce path. The narrow implementation should warn when a failed `direct-primary` phase is recovered by `queue-fallback`, either inside `runSubagentAnnounceDispatch` before fallback or in the caller by inspecting `delivery.phases`. Add a focused regression test that forces direct failure plus successful queue fallback and asserts the diagnostic is emitted once without changing successful delivery semantics.

## Evidence

- **Dispatch masks recovered direct failure at top level:** In completion mode, `runSubagentAnnounceDispatch` appends the `direct-primary` result, runs `queue-fallback` when direct delivery fails, and returns the queue result when fallback delivers. That leaves the direct error only in `phases`, not in the top-level result inspected by the caller.
