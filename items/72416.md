## Summary

Keep this PR open. Current main still only accepts stable `openclaw.install.minHostVersion` floors in `>=x.y.z` form, while the PR is an open, focused implementation candidate for prerelease-aware plugin host floors with matching docs and tests.

## Best Possible Solution

Keep the PR open and review or land it as the implementation candidate. The change belongs in the core plugin install/manifest compatibility contract, not ClawHub, because external plugins cannot express beta host compatibility until `src/plugins/min-host-version.ts` accepts prerelease floors and the public plugin docs describe that contract.
