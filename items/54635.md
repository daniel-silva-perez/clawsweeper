## Summary

Keep this issue open. Current main still emits warnings for mutating tool errors before applying the recoverable-error suppression used for non-mutating tools, and current regression tests explicitly expect recoverable mutating tool errors to produce user-facing failed notifications. The existing edit recovery wrapper is only a partial mitigation for proven post-write throws, and the related open PR/context is not a shipped replacement for this broader warning-policy problem.

## Best Possible Solution

Keep this issue open. The likely maintainer path is a focused change around `src/agents/pi-embedded-runner/run/payloads.ts`, with possible `lastToolError` clearing changes in `src/agents/pi-embedded-subscribe.handlers.tools.ts`, plus regression tests proving recoverable or proven-applied mutating failures do not emit false `failed` notifications while real mutating failures remain visible. Coordinate with #59868 and PR #60664, but do not treat either as shipped replacement work.
