## Summary

Keep open. Current `main` still intentionally surfaces mutating-tool warnings unconditionally, and the tests codify that behavior, so the reported false-positive warning path is not implemented away on `main`.

## Evidence

- **Mutating warnings are still unconditional:** `resolveToolErrorWarningPolicy` returns `showWarning: true` for mutating tool errors, without checking `isRecoverableToolError` or `hasUserFacingReply`.
  - file: [src/agents/pi-embedded-runner/run/payloads.ts:107](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/pi-embedded-runner/run/payloads.ts#L107)

  - file: [src/agents/pi-embedded-runner/run/payloads.errors.test.ts:300](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/pi-embedded-runner/run/payloads.errors.test.ts#L300)
