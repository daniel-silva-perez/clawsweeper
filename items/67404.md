## Summary

Keep this PR open. Current main still does not refresh QMD session exports directly from transcript-update events, and this PR is a plausible focused implementation path, but it needs review/revision for the pending-update race noted in the PR discussion before it should be merged or considered obsolete.

## Best Possible Solution

Keep the PR open and ask for a revision or maintainer review. The best implementation is to wire QMD session exports to transcript-update events when QMD session export is enabled, debounce/filter by agent, clean up timers and unsubscribe on close, and use the forced/queued QMD update path so transcript changes that arrive during `pendingUpdate` always get a follow-up export/index pass. Add focused `qmd-manager` coverage for the debounce, agent filter, cleanup, and in-flight update race.
