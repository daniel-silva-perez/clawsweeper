## Summary

Keep PR #39244 open. Current main still lacks `tools.exec.nsSandbox`, an `exec-bwrap-sandbox` implementation, and any bwrap wrapping path for safeBins-approved gateway exec commands. Existing safeBins validation and sandbox backends are related defenses, but they do not provide the requested safeBins-only Linux namespace filesystem isolation.

## Best Possible Solution

Keep PR #39244 open for explicit maintainer/security judgment. The best path is a narrowed, rebased implementation of optional Linux bwrap isolation for safeBins-approved gateway exec, wired through config types, strict schema, schema help/generated docs, gateway allowlist evaluation, approval-required execution, runtime spawning, and focused tests. If maintainers decide existing safeBins hardening plus full sandbox backends are sufficient, that should be an explicit product/security decision rather than automated cleanup.
