## Summary

Keep open. Current main already covers the interrupt/refine user problem through Control UI `/steer`, `/redirect`, and the `sessions.steer` gateway path, so that half of PR #51163 is largely superseded. But current main does not implement the PR's automatic `ThreadContextStore` for cron/isolated-agent replies in the same IM thread, and issue #50556 was closed as superseded by this open PR rather than by shipped code. Because this PR remains the canonical tracker for that missing cross-session thread-context behavior and may still contain useful work, no conservative close reason applies.

## Best Possible Solution

Keep PR #51163 open but ask the contributor or maintainer to rebase and narrow it. The best path is to drop or redesign the `/interrupt` command in favor of the shipped `/steer` and `/redirect`/`sessions.steer` surfaces, then salvage the cross-session thread-context-store work as a focused implementation with serialized writes, provider-safe keys, Telegram normalization, tests, and docs. If maintainers prefer not to pursue that automatic store, open a narrower replacement issue before closing this PR as superseded.
