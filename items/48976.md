Keep open. Current main still does not normalize Feishu `@_user_N` placeholders when quoted/replied messages are fetched, and the linked canonical issue #48786 remains open. The PR contains a useful direction but is not safe to close as implemented or merge as-is because it bundles unrelated changes and has substantive review concerns.

## Best Possible Solution

Keep #48976 open for maintainer/contributor follow-up, or replace it with a fresh narrow PR that only updates the Feishu quoted-message fetch path: add `mentions` to the fetched message item shape, reuse or safely adapt `normalizeMentions()` after parsing fetched content, escape generated tag attributes correctly, move trimming outside the replacement loop if trimming is desired, and add `getMessageFeishu()` regression coverage for quoted text with Feishu mention metadata. Track the remaining product bug through #48786.
