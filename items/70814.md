## Summary

Keep open. Current main does not contain the PR's proposed recovery for numeric Telegram chat IDs stored in `delivery.channel`, and the supplied related-item context does not identify a canonical merged/superseding fix for that remaining runtime shape. The PR is recent and includes focused regression intent, but it still needs maintainer review because the current patch may change implicit-vs-explicit delivery authorization semantics and also includes unrelated macOS formatting churn.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to preserve the focused regression coverage for numeric Telegram cron announce targets, but adjust the implementation so inferred targets either fail closed or continue to apply the same allowFrom/authorization rules as implicit delivery, then remove unrelated macOS formatting changes from the branch.
