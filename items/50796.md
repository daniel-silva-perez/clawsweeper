## Summary

Keep open. Current `main` does not implement this PR’s fix, and the normal streamed Control UI chat path still sanitizes only inline reply/audio directives, not assistant internal scaffolding such as `<relevant-memories>...</relevant-memories>`. The PR also appears incomplete because it only targets the fallback/injected message path, not the primary streaming path.
