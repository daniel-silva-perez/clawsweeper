## Summary

Keep #7359 open. Current main still records outbound Slack sends into the destination channel/thread session, while Slack DMs and Slack channel sessions remain separate by default and the session tools default to tree visibility. That means a normal DM/main session still cannot safely recall an unrelated Slack channel transcript containing the agent's own outbound message.

## Best Possible Solution

Keep #7359 open and implement a scoped Slack self-activity recall path. The likely maintainer-safe shape is an outbound activity ledger or session-history bridge keyed by agent, Slack account, destination channel/thread, and sent Slack message id, exposed from DM context only for the agent's own outbound activity. Add regression coverage for sending from a Slack DM to a Slack channel and then asking from the DM whether the agent sent that message.
