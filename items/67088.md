## Summary

Keep open. Current main still short-circuits browser opening whenever SSH_* is present and DISPLAY/WAYLAND_DISPLAY is absent on every non-Windows platform, so macOS can still hit the reported `ssh-no-display` path before reaching the native `open` handler. There is also an open implementation PR with closing syntax for this issue, so this issue should stay open until that PR is reviewed and merged or closed.

## Best Possible Solution

Keep this issue open and review PR #67115, or replace it with an equivalent fix that narrows the SSH/no-display short-circuit to Linux only, or otherwise excludes darwin before returning `ssh-no-display`. Add focused regression coverage for darwin with SSH_* and no DISPLAY resolving to `open`, while preserving Linux SSH/no-display behavior.
