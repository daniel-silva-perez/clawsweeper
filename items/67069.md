## Summary

Keep open. Current main still only forwards Feishu `root_id` as `MessageThreadId` for topic-session group scopes, while this PR asks for unconditional `root_id` metadata. The PR patch is also not mergeable as written because it adds duplicate inbound-context keys and leaves the existing guarded `MessageThreadId` assignment in place.

## Best Possible Solution

Keep this PR open for contributor or maintainer revision: update the existing `ReplyToId` and `MessageThreadId` assignments instead of adding duplicate keys, add focused Feishu tests for the exact intended non-topic and topic-session behavior, and reconcile the result with #60848 so there is one coherent Feishu threading fix.
