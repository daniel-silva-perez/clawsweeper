## Summary

Keep open. Current main does not implement the PR's stated unconditional Feishu `root_id` -> `MessageThreadId` pass-through, and the PR patch is not mergeable as written because it adds duplicate inbound-context keys. Related PR #60848 is a plausible overlapping implementation candidate for guarded Feishu topic-thread behavior, but it does not clearly supersede the unconditional metadata request.

## Best Possible Solution

Keep this PR open for maintainer or contributor action: revise the patch to update the existing `ReplyToId`/`MessageThreadId` fields rather than adding duplicates, decide whether Feishu should expose unconditional `root_id` metadata despite the current normal-group guard, and reconcile the result with #60848's guarded topic-session fallback and outbound `replyInThread` work before landing one tested Feishu threading fix.
