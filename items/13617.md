## Summary

Keep open. Current `main` does not implement the requested dry-run/testing framework for cron jobs and skills. Cron has route preview and manual run/history commands, and skills has search/install/update/list/info/check, but there is no user-facing `cron test`, `cron validate`, or `skills test` command or equivalent full no-side-effects execution path.

## Evidence

- **Cron docs only expose preview/show/run flows:** The current cron CLI docs describe `openclaw cron list` and `openclaw cron show` as delivery-route previews, and the admin command list includes only `list`, `show`, `run`, and `runs`; there is no documented `test` or `validate` subcommand.
  - file: [docs/cli/cron.md:19](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/docs/cli/cron.md#L19)
