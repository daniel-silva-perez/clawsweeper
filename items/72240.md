## Summary

Keep #72240 open. Current main has some internal supervisor fields and diagnostic events, but the foreground exec result and process polling surfaces still do not expose the structured SIGKILL cause metadata requested by the issue, and the related open issues only cover overlapping pieces rather than superseding this macOS diagnostic gap.

## Best Possible Solution

Keep this issue open and implement the diagnostic surface in core exec/process handling. The likely path is to propagate RunExit terminationReason, exitSignal, noOutputTimedOut, durationMs, and captured stdout/stderr tail into foreground ExecToolDetails, process poll/list/log output, completion notifications, and diagnostic events; add a stable runId/log pointer; distinguish supervisor timeout/manual cancel from externally observed SIGKILL when known; and add regression tests for overall timeout, no-output timeout, manual cancel, and plain OS signal cases. Coordinate but do not collapse it into #66399 or #69242 unless maintainers choose one canonical umbrella.
