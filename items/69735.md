## Summary

Keep open. #69735 describes a still-present Ollama inbound tool-call argument parsing bug, and the active fix is tracked by open PR #69910; the other linked PR attempts were closed rather than merged. ([github.com](https://github.com/openclaw/openclaw/pull/69910))

## Best Possible Solution

Keep #69735 open until a canonical fix lands. The maintainer path should be to land or update PR #69910, applying `ensureArgsObject` or the same unsafe-integer-preserving parser in `buildAssistantMessage`, widening the Ollama tool-call argument type to `Record<string, unknown> | string` where appropriate, and adding regression tests for inbound JSON-string arguments, object pass-through, malformed strings, unsafe integers, and the streaming final-message path.
