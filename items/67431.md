## Summary

Keep this PR open for scoped contributor follow-up. Current main already implements the chat-control and markdown-sidebar aria labels with coverage, but the PR still contains unresolved work and review risk: the grouped tool disclosure aria-label would remove contextual accessible text, and the streaming aria-busy change is absent from main and needs explicit screen-reader validation before merge. The same author also has paired open accessibility work in #67432, so closing only this PR would be premature.

## Best Possible Solution

Keep the PR open and ask for a scoped rebase: drop the chat-control and sidebar label changes already on main, preserve contextual accessible names for grouped tool disclosures or append state without overriding context, and split or explicitly validate the streaming aria-busy behavior with a targeted screen-reader test plan. Coordinate the remaining accessibility sweep with the paired same-author #67432 instead of closing only this PR.
