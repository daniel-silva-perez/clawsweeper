## Summary

Keep open for scoped maintainer/contributor follow-up. Current main already has the chat-control and sidebar aria labels with browser test coverage, but the PR is not fully implemented or cleanly obsolete: current main lacks the streaming aria-busy change, and the grouped tool disclosure label proposed here conflicts with review feedback because current markup preserves contextual visible text in the button's accessible name.

## Best Possible Solution

Keep the PR open and ask for a scoped rebase: drop label changes already present on current main, preserve contextual accessible names for grouped tool disclosures, and split or explicitly validate the streaming aria-busy behavior with a targeted screen-reader test plan before merging.
