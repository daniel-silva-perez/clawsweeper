## Summary

Keep open. Current main still has the reported UX surface: activated plugin loading emits the empty `plugins.allow` warning for workspace/global discoveries, and current regression tests intentionally preserve both the open-allowlist and untracked-provenance warning behavior. The maintainer discussion already explicitly kept this open as a broader wording/remediation issue, and the remaining fix is not shipped on main.

## Best Possible Solution

Keep this open and make a narrow loader/docs UX change: preserve the security warning, but distinguish first-run/no explicit plugin config from a configured allowlist mismatch, include the exact discovered plugin ids to add under `plugins.allow`, and point users to `openclaw plugins list --enabled --verbose` and `openclaw plugins inspect <id>` for ID discovery. Do not close it until that user-facing remediation is implemented or a maintainer explicitly scopes it differently.
