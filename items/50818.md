## Summary

Keep open. Current main has part of the async exec routing work: `requestHeartbeatNow` accepts `sessionKey`, and internal exec/node/ACP/CLI call sites use `scopedHeartbeatWakeOptions` for ordinary agent session keys. But the PR is not fully obsolete or implemented: current main still lacks the PR's cron-run event remapping helper, lacks `mainKey` threading through that remap, and still intentionally treats exec completions as internal-only when `heartbeat.target` is `none`. The discussion also shows useful review-driven work in the PR around cron isolation, custom `session.mainKey`, and false-positive exec event matching, so this is not a safe conservative cleanup close.

## Best Possible Solution

Keep the PR open for maintainer judgment rather than auto-closing it. The best path is to rebase or split it: preserve any still-needed cron-run `resolveEventSessionKey`/`mainKey` alignment fixes, drop or explicitly decide the `target: none` delivery-contract change, and cross-link remaining external callback/session-targeting work to #52305 or a narrower follow-up.
