## Summary

Keep #52448 open. Current main still only seeds the standard bootstrap files into sandbox workspaces and does not implement the requested README.md/CLAUDE.md/docs authority-link materialization. The PR is still a plausible core sandbox feature, but the provided review thread shows unresolved security-boundary concerns around approved symlink names resolving outside the seed root and hardlinked content being copied into read-only/none sandboxes.

## Best Possible Solution

Keep the PR open for maintainer/security review or ask for a revised implementation. A safe version should define an explicit authority-source contract, validate copied targets against that contract, reject hardlinks and nested symlinks using existing boundary-file/path guard patterns, define refresh semantics for reused agent-scoped sandboxes, and add regression coverage for safe in-scope links plus outside-link, nested-symlink, hardlink, stale-copy, and Windows symlink cases.
