## Summary

Keep open. Current main still does not implement configurable model API request pacing, and the PR remains a plausible implementation path for open issue #49077, though it needs review fixes/rebase rather than conservative cleanup closure.

## Best Possible Solution

Keep PR #49221 open as the concrete implementation attempt for #49077. The best path is to rebase it on current main, add the config fields to the current schema/help/docs surfaces, wire pacing into both embedded runs and compaction, preserve per-provider limiter history across config interval changes, normalize provider lookup consistently with existing provider config resolution, and add focused tests including per-provider `requestDelayMs: 0` overriding a global delay.
