## Summary

Keep open. Current `main` has partial OpenAI Responses compatibility work, but it does not fully implement #10137’s requested normalization layer. I found support for `store`/prompt-cache/context-management/reasoning shaping, but not a stable config surface for default non-empty `instructions`, omitting unsupported fields like `text`, or broader proxy-specific request normalization.

## Evidence

- **Partial payload policy only:** `resolveOpenAIResponsesPayloadPolicy` and `applyOpenAIResponsesPayloadPolicy` currently normalize `store`, prompt-cache fields, `context_management`, and disabled reasoning payloads. They do not add fallback `instructions`, coerce `input`, or strip arbitrary unsupported fields such as `text`.
