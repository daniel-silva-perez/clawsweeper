## Summary

Keep #70253 open. Current main still leaves the reported Mattermost streaming/config problem unresolved, and the channel-level bot-to-bot/A2A routing concern is still a plausible missing product/API behavior rather than something implemented or clearly superseded. The reporter’s follow-up analysis matches current source: the main Mattermost inbound reply path still forces block streaming off for the generic reply pipeline while running draft preview streaming, Mattermost still lacks an allowBots-style config surface, and tools.agentToAgent remains scoped to session tools rather than channel fan-out.

## Best Possible Solution

Keep the issue open, likely split or scope it into two maintainer-trackable fixes: first, make the Mattermost inbound reply path honor the documented/configured streaming and blockStreaming settings with regression coverage; second, decide and implement an explicit channel-level bot-to-bot/A2A policy for Mattermost and any shared routing APIs needed for other channels, rather than assuming tools.agentToAgent covers channel event fan-out.
