## Summary

Keep #71452 open. Current main still has the exact generic 25-row human-readable formatter cap, and affected CLI paths still use that formatter even when provider code can return more rows or expose pagination/limit behavior. Teams has partial pagination work for pins and channel lists, but the central formatter/requested-limit problem remains unsolved.

## Best Possible Solution

Keep the issue open. Implement a narrow message formatting/action contract follow-up: pass the requested limit and pagination metadata into the human-readable renderer, remove or make configurable the hardcoded 25-row cap, ensure list-pins/channel-list/read providers respect limits or expose next-page metadata consistently, and add regression tests covering more than 25 read, list-pins, and search rows.
