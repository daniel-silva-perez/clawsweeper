## Summary

Keep this PR open. Current main still only exposes `ltm list`, `ltm search`, and `ltm stats` for the bundled memory-lancedb plugin, while this PR is the focused open implementation candidate for `ltm reindex --timeout` and `ltm drop-index`. The work is not obsolete or superseded, but it needs the existing timeout-validation review feedback and focused tests before it is merge-ready.

## Best Possible Solution

Keep this PR open. The best maintainer path is to request/update the PR with positive-integer timeout validation, add focused memory-lancedb CLI/unit tests for reindex and drop-index behavior, then review and land if these bundled plugin maintenance commands are desired.
