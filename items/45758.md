## Summary

Keep #45758 open. Current main still treats the core config as JSON/JSON5 at `openclaw.json`: discovery only considers `openclaw.json` plus legacy `clawdbot.json`, parsing goes through JSON5, and writes serialize JSON. The newer `config.patch` and `openclaw config set` paths address the issue discussion’s comment-preserving mutation concern, but they do not implement YAML config files.

## Best Possible Solution

Keep this open for maintainer product judgment. The best code path, if accepted, would be a core config IO change with explicit `.json`/`.json5`/`.yaml`/`.yml` precedence, safe YAML parser settings, validation parity, tests, docs, and a clear write-back policy. A lower-risk interim docs improvement could recommend `openclaw config set/get` and `config.patch` for comment-aware scripted edits, but that does not close the YAML support request.
