## Summary

Keep open. Current main still does not implement the requested Anthropic context-management beta support: raw `anthropicBeta` headers and `context1m` exist, but there is no first-class Anthropic `context_management` payload shaping, no compaction/context-editing stream and replay handling, and no `memory_20250818` provider-defined tool path. The issue discussion is concrete and recent, including a production +1 asking for server-side compaction, `pause_after_compaction`, and post-compaction reinjection behavior.

## Best Possible Solution

Keep this open for maintainer/product design in the bundled Anthropic provider. A maintainer-quality fix should add documented Anthropic model params for beta/header handling, validated `context_management` body shaping for compaction and context editing, stream and replay support for compaction blocks and `pause_after_compaction`, request tests for direct Anthropic transport and provider wrappers, and a clear design decision on whether `memory_20250818` is implemented as provider-owned Anthropic behavior or requires a generic provider-defined-tool SDK seam.

- Anthropic beta APIs are moving targets; any implementation should re-check current Anthropic docs and SDK types at coding time.
