## Summary

Current `main` does not implement this PR's fix. The embedded reply path still forwards only `params.replyOptions?.images` and never converts inbound `MediaPath`/`MediaPaths` files into agent `images`, so inbound channel images can still be reduced to placeholder text rather than actual image content.

## Evidence

- **Embedded reply dispatch still omits inbound media-file conversion:** `replyResolver(...)` spreads `params.replyOptions` and sets typing callbacks, but there is no helper that reads `ctx.MediaPath`/`ctx.MediaPaths` and merges them into `images` before the agent run.
  - file: [src/auto-reply/reply/dispatch-from-config.ts:910](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/auto-reply/reply/dispatch-from-config.ts#L910)
