## Summary

Keep open. Current main still builds a HealthSummary with durationMs and preserves it in JSON output, but the non-JSON openclaw health text path does not print the gateway probe duration. The PR remains a small, relevant fix for linked issue #50245, and current docs already describe the text command as reporting a probe duration.

## Best Possible Solution

Keep the PR or linked issue open. The best fix is still a narrow change in src/commands/health.ts to log summary.durationMs in the non-JSON openclaw health path, with a regression assertion in src/commands/health.test.ts or src/commands/health.command.coverage.test.ts. If maintainers prefer a different label or placement, adjust the PR rather than closing it as obsolete.
