## Summary

Keep open. Current main preserves explicit empty plugin scope semantics, but it does not implement this PR's requested O(1) loader fast-path: `loadOpenClawPlugins({ onlyPluginIds: [] })` still reaches cache lookup, reentry setup, plugin discovery, and manifest registry loading before filtering to an empty registry. Related #65298 appears to have fixed correctness around `[]` vs `undefined`, not this performance shortcut.

## Best Possible Solution

Keep the PR open. The best path is to land a narrow loader change that short-circuits explicit empty scopes after resolving the normalized scope/cache key and preserving the reentry guard and activation clear/activate side effects, plus a focused unit test that mocks or spies on discovery/manifest/cache work to prove the fast-path behavior. Related #65298 should remain treated as the correctness foundation, not a superseding implementation of this optimization.
