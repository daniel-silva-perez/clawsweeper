## Summary

Current `main` does not include the TUI gateway continuity work described in #48589. The code still uses per-connect random gateway client IDs, resolves empty TUI sessions to the generic main key, has no TUI affinity/orphan-short-reply modules, and does not synthesize the PR's TUI-specific origin routing, so there is not strong evidence to close this PR.
