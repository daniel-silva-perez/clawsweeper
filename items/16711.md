## Summary

Keep open. Current `main` has an auto-growing chat textarea, but it does not implement the requested composer expand button/modal or the requested modal keyboard/actions flow from #16711.

## Evidence

- **Plain Enter still sends from the main composer:** `ui/src/ui/views/chat.ts:766` shows the placeholder `Message ... (Enter to send)`, and `ui/src/ui/views/chat.ts:1035-1049` handles plain `Enter` as send when `Shift` is not held. That does not match the requested expanded-editor behavior of `Enter = newline` and `Ctrl/Cmd+Enter = Send`.
  - file: [ui/src/ui/views/chat.ts:766](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/ui/src/ui/views/chat.ts#L766)

- **Toolbar still lacks an expand button:** `ui/src/ui/chat/run-controls.ts:18-84` renders only `New session`, `Export`, and `Send/Queue/Stop` controls on the right side of the composer toolbar. There is no composer expand/maximize action there.
  - file: [ui/src/ui/chat/run-controls.ts:18](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/ui/src/ui/chat/run-controls.ts#L18)
