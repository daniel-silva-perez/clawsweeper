## Summary

Keep open. Current main does not contain the PR's one-shot hooks CLI exit behavior: `hooks list`, `info`, `check`, `enable`, `disable`, and bare `hooks` still run through `runHooksCliAction` without a success-path exit. The reported root cause is plausible in current code because hooks reporting builds plugin diagnostics, which loads plugin modules in non-activating discovery mode but still calls plugin `register()`. The PR is recent, contributor-authored, has no protected labels, and its review comments were addressed in later PR commits according to the provided discussion.

## Best Possible Solution

Rebase and land a focused version of this PR: add a clearly named one-shot hooks CLI runner that waits for stdout to drain, calls `defaultRuntime.exit(0)` outside test runtimes using `isVitestRuntimeEnv`, covers `list`, `info`, `check`, `enable`, `disable`, and bare `hooks`, and keeps deprecated `hooks install/update` and hidden relay behavior unchanged. Keep the guard test for Vitest env variants. A larger redesign to avoid loading plugin modules during diagnostics could be considered separately, but it is not required to fix this one-shot CLI invariant.
