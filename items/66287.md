## Summary

Current main still implements thresholded memory-core deep promotion, and the reported single-occurrence important-event gap remains. The existing ClawSweeper comment on this issue is consistent with current source, tests, and docs: there are manual/global workarounds, but no per-event priority, critical-feedback classifier, or explicit durable remember path that safely bypasses the default signal and diversity gates.

## Best Possible Solution

Keep this open for maintainer/product judgment in memory-core. The strongest repo path is a tested high-priority durable lane for single-event candidates, such as explicit remember-directive metadata, a critical-feedback classifier, or manual priority flag, that can bypass or separately satisfy signal/diversity gates without weakening default promotion for ordinary singleton notes. Update memory-core tests and the public memory/dreaming docs once the chosen behavior is defined.
