## Summary

Current main still has the reported memory-core behavior: automatic deep promotion is thresholded by signal count and query/day diversity, so a single-turn important event can remain staged instead of being promoted to MEMORY.md. Manual CLI overrides, explicit memory-writing, and grounded backfill are useful workarounds, but they do not implement the requested automatic single-occurrence importance path.

## Best Possible Solution

Keep this open for maintainer/product judgment. A repo fix likely belongs in memory-core deep promotion: add a tested explicit durable/high-priority path for single-event candidates, such as a remember directive, critical-feedback classifier, or manual priority flag, that can bypass or separately satisfy signal/diversity gates without making ordinary singletons pollute MEMORY.md. Update docs and tests with the chosen behavior. If maintainers prefer this as custom policy, split it to a plugin/ClawHub direction only after confirming the current memory plugin API is sufficient.
