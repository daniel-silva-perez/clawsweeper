## Summary

Issue #50686 requests a new Zoom Team Chat core/bundled channel integration, but current OpenClaw direction is to keep optional commercial integrations out of core and ship them as external plugins. Current `main` has channel-plugin APIs and docs for exactly this path, and no Zoom implementation is present.

## Evidence

- **Core scope guardrail:** Project direction says optional capability should usually ship as plugins, be hosted in their own repo, and that the bar for adding optional plugins to core is intentionally high.
  - file: [VISION.md:55](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/VISION.md#L55)
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Commercial integration guardrail:** The current vision explicitly lists commercial service integrations that do not clearly fit the model-provider category under 'What We Will Not Merge (For Now)'. Zoom Team Chat fits that bucket more than core runtime work.

## Risks / Open Questions

- If maintainers now want to officially own a Zoom integration despite the current plugin-first scope, this close would be premature.
- A separate external Zoom plugin may already exist or may be feasible quickly, but that does not change the core-repo scope decision.

## Close Comment

Closing after a Codex maintainer review. This request is for a new Zoom Team Chat core integration, but current OpenClaw scope points this kind of work to plugins rather than the core repo: `VISION.md:55` says optional capability should usually ship as plugins and be maintained in their own repo, and `VISION.md:103` lists commercial service integrations outside the model-provider category as out of scope for core for now. The supported path is already documented in `docs/plugins/building-plugins.md:16` and `docs/plugins/sdk-channel-plugins.md:11`, which describe publishing new messaging channels as OpenClaw plugins via ClawHub/npm. Current `main` also has no Zoom channel implementation or docs. If Zoom support is still desired, the right next step is an external channel plugin published through ClawHub or npm, with maintainer re-evaluation only if there is later explicit buy-in for official core ownership.
