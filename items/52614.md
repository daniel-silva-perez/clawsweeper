## Summary

Keep PR #52614 open. Current main still lacks the PR’s replay/preexisting-message guard, and the existing shipped suppression only covers transcript-only OpenClaw `delivery-mirror`/`gateway-injected` assistant turns. Related PR #50176 is closed and unmerged, so it is not a shipped canonical fix for the remaining behavior.

## Best Possible Solution

Keep this PR or an equivalent maintainer-owned follow-up open and land a scoped embedded-subscribe fix: snapshot `session.messages` at subscription time, suppress synchronous initial replay and preexisting assistant lifecycle events from outward delivery, keep transcript state intact, and add the regression test for old assistant finals plus a new assistant reply.
