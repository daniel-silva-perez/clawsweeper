## Summary

Keep open. Current main still lacks the requested public TUI footer status contribution API and render-path support, and the provided GitHub context shows open PR #72557 by the same contributor with resolving syntax for this issue. ContextClaw is a concrete plugin-author use case, not a speculative request, and the current workaround is still monkeypatching the bundled TUI.

## Best Possible Solution

Keep this issue open. The best path is to review #72557 as the implementation candidate for the public PluginStatusProvider/registerStatusProvider seam, decide whether it should also wire the TUI footer render path, and coordinate broader footer templating or cost/balance display choices with #53774. Close this issue only after merged main gives ContextClaw and other plugins a supported way to contribute TUI footer status without monkeypatching.
