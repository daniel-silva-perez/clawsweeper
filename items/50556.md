## Summary

Keep open. Current `main` has thread-bound routing and explicit cross-session recall tools, but it does not implement the requested generic cross-session context understanding for cron/triggered replies in the same IM thread. The codebase still treats cron runs as lightweight fresh turns, and current cron delivery logic strips generic `:thread:` session suffixes back to the parent peer instead of preserving a shared thread context store.

## Evidence

- **Cross-session recall is still explicit/manual:** Docs describe `sessions_history` as the cross-session recall path, which implies recall is fetched explicitly rather than automatically injected into new triggered sessions.
  - file: [docs/concepts/multi-agent.md:26](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/docs/concepts/multi-agent.md#L26)
