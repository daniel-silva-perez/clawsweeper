## Summary

Keep this PR open. Current main at d3fd275aa5fc8b8eff8ca9b1148ad86da8abc2ed still lacks the proposed volatile-file filter, tar EOF retry loop, enriched final EOF errors, log callback plumbing, and skippedVolatileCount result/summary/JSON field. The PR is recent, focused, and the author addressed the bot review findings in follow-up commits; the closed linked bug report and adjacent backup issues do not supersede the implementation candidate.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase or review the focused backup-archiver change, decide whether the built-in volatile skip semantics are acceptable, update user-facing backup docs if accepted, and keep #67417/#67990 separately tracked unless maintainers intentionally consolidate the backup consistency work.
