## Summary

Keep open. Current main still lacks the BR ninth-digit equivalence that PR #50895 is trying to add, and the underlying WhatsApp outbound/inbound/heartbeat paths still use exact normalized E.164 comparisons. The PR is not ready to close as implemented or obsolete; it needs to be rebased onto the moved WhatsApp plugin files and should also address the post-5247 review comments about reactions and session routing before merge.

## Best Possible Solution

Keep PR #50895 open and adapt it to the current WhatsApp plugin surfaces. Add a local equivalence helper in the WhatsApp plugin API/SDK surface, use it for allowlist authorization in outbound resolution, inbound policy, command auth, same-phone gating, main last-route pinning, and heartbeat recipient authorization, while preserving the original explicit chat/JID for reactions and outbound session routing. Add regression tests for both BR directions across those paths.
