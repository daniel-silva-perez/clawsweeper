## Summary

Keep open. Current `main` still does exact-match WhatsApp phone comparisons in the outbound allowlist, inbound allowlist/self-chat checks, main-session last-route refresh, and heartbeat recipient authorization, so the BR ninth-digit equivalence this PR targets is not implemented on `main`. The PR also still has unresolved routing risks from its own review thread around reactions and per-peer session keys if outbound targets are rewritten to the allowlist’s canonical form.

## Risks / Open Questions

- If the PR is merged as described in the provided GitHub context, rewriting `resolution.to` to the allowlist’s canonical BR form still risks sending reactions to a different chat JID than the message’s actual `chatJid` path uses.
- That same canonical-target rewrite also risks splitting per-peer/per-channel-peer session routing, because outbound session keys are derived from the resolved target identity.
