## Summary

Keep this PR open. Current main still does not implement the root `--log-file` flag or an `OPENCLAW_LOG_FILE` file-sink override, so the PR is not obsolete. It remains a useful implementation candidate, but the provided review context and current route/status code show functional gaps that should be revised before merge.

## Best Possible Solution

Keep this PR open as the implementation candidate, but require revisions before merge: add root `--log-file` to parser/help, promote it before both Commander and route-first execution paths, implement an explicit `OPENCLAW_LOG_FILE` resolver with precedence over `logging.file`, thread an env parameter through log-path resolution so daemon status can use the inspected service environment, and keep focused tests/docs for these boundaries.
