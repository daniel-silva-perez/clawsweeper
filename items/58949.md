## Summary

Keep PR #58949 open. Current main still has the early return in `safeEqualSecret` for null/undefined inputs, so the proposed change is not implemented on main and is not obsolete. The PR also has unresolved review feedback about additional edge-case tests and a minor hardening suggestion, so it needs maintainer review or contributor follow-up rather than conservative cleanup closure.
