## Summary

Keep PR #71953 open. Current main still has a real build-profile gap: bundled hook handlers are built into `dist/bundled/...`, but the `gatewayWatch` path runs only `tsdown`, `runtime-postbuild`, and `build-stamp`, while `runtime-postbuild` does not copy bundled hook `HOOK.md` manifests and the standalone `copy-hook-metadata` step is skipped.

## Best Possible Solution

Keep the PR open for maintainer review. The best fix is a narrow runtime-postbuild change that copies bundled hook `HOOK.md` manifests whenever gateway watch/runtime postbuild artifacts are produced, with focused tests covering copy, missing manifest, and missing source directory. An equivalent alternative would be to make the gatewayWatch build path run the existing copy step, but folding it into `runRuntimePostBuild` better matches the current run-node path.
