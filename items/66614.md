## Summary

Keep this issue open. Current main still has fence-aware block chunking only, with no markdown-table span guard in `EmbeddedBlockChunker`, and the chunker tests still cover paragraph flushing and code fences rather than table-safe streaming. The provided discussion and PRtags context point to #66344 as an open PR with closing syntax for this issue; per maintainer policy, that is an implementation candidate, not a reason to close before merge.

## Best Possible Solution

Keep the issue open and review/land #66344 or a smaller replacement. The fix should add markdown table-span detection in the markdown/chunker layer, thread table-safe break checks through `EmbeddedBlockChunker` including `flushOnParagraph`, and add regression tests for text + table + text, partial streamed tables, tables inside fences, and oversized-table fallback behavior.
