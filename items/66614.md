## Summary

Keep this issue open. Current main still has fence-only protection in `EmbeddedBlockChunker`; there is no table-span parser or table-safe break guard, and the active related PR #66344 remains the implementation candidate rather than merged proof.

## Best Possible Solution

Keep this issue open and review/land #66344 or a smaller equivalent fix. The fix should add markdown table-span detection in the shared markdown/chunker layer, gate all `EmbeddedBlockChunker` break paths on table-safe boundaries as well as fence-safe boundaries, and add focused regression tests for streaming text + table + text, partial table buffers, fenced tables, and oversized table fallback.
