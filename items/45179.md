## Summary

Keep open. Current main has safer temp-root handling, but it does not implement the requested temp-artifact provenance model, durable-path classification, janitor awareness, or broad runtime-critical `/tmp` guardrails.

## Evidence

- **Temp handling is centralized, not classified:** `resolvePreferredOpenClawTmpDir()` only picks/creates a secure temp directory such as `/tmp/openclaw` or an `os.tmpdir()` fallback. It does not record provenance, deletion contracts, or durable-path routing.
  - file: [src/infra/tmp-openclaw-dir.ts:34](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/infra/tmp-openclaw-dir.ts#L34)
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Agent code still writes temp artifacts without provenance metadata:** CLI runner writes a system prompt into a fresh temp directory and returns only a best-effort cleanup callback; there is no sidecar, registry entry, or artifact class.
  - file: [src/agents/cli-runner/helpers.ts:289](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/agents/cli-runner/helpers.ts#L289)
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Some temp writes still use raw os.tmpdir():** Bundle MCP config paths are created under `os.tmpdir()` for Gemini and Claude/Codex adapter flows, again with cleanup callbacks only and no provenance/durable-path enforcement.
  - file: [src/agents/cli-runner/bundle-mcp.ts:235](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/agents/cli-runner/bundle-mcp.ts#L235)
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Doctor does not cover the requested temp-path guardrails:** `noteStateIntegrity()` checks cloud-synced state dirs, missing/writable dirs, permissions, duplicate state dirs, and orphan agent dirs, but not whether runtime-critical paths resolve into `/tmp`/`/private/tmp` or whether temp artifacts are misrouted durable outputs.
  - file: [src/commands/doctor-state-integrity.ts:586](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/commands/doctor-state-integrity.ts#L586)
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **No temp provenance/registry surface exists in current tree:** A repo search for `artifact provenance|safe to delete|canonical durable path|created by: session|created by: tool|janitor integration|temp registry|temp artifact` returned no matches.
  - command: `rg -n "artifact provenance|safe to delete|canonical durable path|created by: session|created by: tool|janitor integration|temp registry|temp artifact" src docs extensions packages scripts`
