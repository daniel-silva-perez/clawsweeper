## Summary

Keep open. Current main still contains the unguarded `provider?.models?.some(...)` call that matches the reported TypeError, the existing unit coverage does not cover non-array `models`, and the provided GitHub context shows open unmerged PR #66761 uses closing syntax for this issue. That PR is an implementation candidate, so this issue should remain open until the fix lands or the PR is closed.

## Best Possible Solution

Review #66761 as the canonical implementation candidate. The likely core fix belongs in `src/agents/models-config.providers.policy.lookup.ts`: guard `provider.models` with `Array.isArray(...)` before calling `.some(...)`, and add regression tests in `src/agents/models-config.providers.policy.lookup.test.ts` for non-array runtime provider shapes. After that PR merges and ships, this issue can close as implemented.
