## Summary

Current `main` already has the session-selection persistence and navigation wiring needed for this report: chat session changes update shared app state and persisted settings, chat-tab navigation keeps the selected session in the URL, popstate restores the `session` query param, and a browser test covers navigating from `/sessions?session=...` to chat while preserving the same session key.

## Close Comment

Closing from a Codex review because current `main` already implements the session persistence needed for this bug. The v2 UI now keeps the selected chat session in shared app state and persisted settings (`ui/src/ui/app-render.helpers.ts:68`), syncs that session into the URL when switching sessions (`ui/src/ui/app-render.helpers.ts:509`), restores it from browser navigation state (`ui/src/ui/app-settings.ts:500`), and has a browser test for navigating from `/sessions?session=...` back to chat while preserving the same session (`ui/src/ui/navigation.browser.test.ts:171`). This is present on `main` and is best evidenced by commit `37c37eecfbe4bef413c33b8606db62cf51c2d1d4`. The latest tagged release `v2026.4.22` predates that commit, so I’m leaving `fixedRelease` unset.
