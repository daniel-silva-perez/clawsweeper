## Summary

Keep open. Current main only mitigates stale TUI streaming with a 30s delta-silence watchdog; it still does not implement the requested prompt transition after the last visible content token. The TUI remains driven by chat delta/final and lifecycle events, and gateway lifecycle finalization can still wait on async post-response work. The discussion also contains useful named-worker reproduction context, so this is still an actionable TUI UX tracker.

## Best Possible Solution

Keep this issue open as the TUI UX tracker. The best fix is to decouple visible content completion from full run lifecycle completion, either by adding a gateway/TUI content-complete or finalizing phase, or by having the TUI transition from `streaming` to a distinct non-streaming finalizing/idle state after visible deltas stop while preserving active-run ownership. Add regression coverage for delayed lifecycle final events and the named worker-agent path from the discussion.
