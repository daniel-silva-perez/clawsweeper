## Summary

Keep open. Current main has shipped partial fixes for the active-send/session.message reload path, but the remaining same-run final-chat reload amplifier described here is still present: final chat events still route through a helper that returns true for every final payload, and the gateway still calls chat.history after those finals even when the assistant message was already appended locally. The open maintainer PR #72325 explicitly tracks the remaining WebChat final/reconciliation race, so this issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and handle it through #72325 or an equivalent narrow Control UI/WebChat fix: change final-event reload policy so well-formed same-run assistant finals already appended locally do not trigger redundant chat.history reloads, preserve legitimate hydration/deferred session.message behavior, and land the accompanying gateway/helper/controller regression tests. Close this issue only after that PR merges and current main proves the final/stream reconciliation path is covered.
