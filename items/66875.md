## Summary

Keep this issue open. Current main still contains the same redundant final-event chat.history reload path, while only the active session.message/send flicker path is partially fixed. The open maintainer PR #72325 is the right implementation candidate for the remaining WebChat final reconciliation race.

## Best Possible Solution

Keep this issue open and resolve it through #72325 or an equivalent narrow Control UI/WebChat fix. The fix should avoid redundant chat.history reloads for well-formed same-run final assistant payloads already incorporated locally, preserve legitimate hydration for tool/no-message finals and deferred session.message reloads, and update app-gateway/helper/controller tests before closing this issue.
