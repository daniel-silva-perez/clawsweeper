## Summary

Keep this issue open. Current main still returns the rotated device token as plaintext in the `device.token.rotate` RPC success payload, and current tests, Control UI code, and CLI docs still treat that plaintext token as the expected contract. The related redaction PR is closed and unmerged in the provided context, so it does not resolve the report.

## Best Possible Solution

Keep this issue open and handle it as a gateway protocol/security design fix. The likely path is to change `src/gateway/server-methods/devices.ts` so normal `device.token.rotate` responses do not expose `entry.token`, add regression tests asserting the token is absent or explicitly redacted, and update Control UI, CLI behavior/docs, and public device-token guidance to use a safer bootstrap or out-of-band retrieval path if plaintext delivery remains necessary in a narrow provisioning flow.
