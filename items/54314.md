## Summary

Keep open. Current `main` does not implement the PR’s dedicated cron-owned nested lane: cron-owned embedded runs still resolve onto the shared `nested` lane, and gateway startup/hot reload only apply `cron.maxConcurrentRuns` to `CommandLane.Cron`, not a separate `cron-nested` lane.
