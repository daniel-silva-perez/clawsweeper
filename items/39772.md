## Summary

Keep PR #39772 open. Current main still constructs one Telegram draft-stream reply parameter object and reuses it for every message-preview send, so the proposed reply_to gating is not implemented. The PR is a focused Telegram bug fix with tests, but the discussion still contains substantive review concerns around first-send failure recovery and replyToMode="all" semantics, so conservative cleanup should not close it.

## Best Possible Solution

Keep the PR open for maintainer review or author revision. A robust fix should add draft-stream state that prevents deleted-message preview artifacts while preserving replyToMode="all" where intended and only consuming the first reply target after a send is known to have landed, with regression coverage for forceNewMessage, thread params, and failed-first-send/materialize recovery.
