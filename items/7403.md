## Summary

Keep open. Current main still does not provide an enforceable Private/Secret/Incognito session mode; the issue remains the canonical tracker for suppressing workspace context and preventing transcript/session-memory persistence. #12242 was already closed as a duplicate, and #52478 was closed unmerged.

## Best Possible Solution

Keep #7403 open as the canonical product request. A complete fix should add an explicit private/ephemeral mode across Gateway protocol schemas, session config/store entries, transcript creation and appending, workspace bootstrap and memory read/write paths, Mission Control and channel controls, with regression tests proving private sessions neither read workspace memory/bootstrap files nor write session history or memory files.
