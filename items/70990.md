## Summary

Keep this PR open. Current main still lacks the proposed `model_failover` and `model_failure_terminal` typed plugin hooks, and the PR remains a plausible narrow core plugin API change rather than ClawHub-only work because external plugins cannot observe these failover seams without core hook names, runner methods, and emission points.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to decide the exact `model_failover` and `model_failure_terminal` event contract, review privacy/gating for terminal failure payloads, add public hook documentation, keep the focused hook-runner and failover-emission tests, and then land if maintainers accept this narrow core plugin API surface. Broader fallback notices, workspace-context re-injection, and circuit-breaker policy should remain under #65824 or separate follow-ups.
