## Summary

Keep open. Current main still lacks the requested `model_failover` and `model_failure_terminal` plugin hooks, and the PR is a focused core plugin API extension for a real gap called out by #70976/#65824 rather than optional ClawHub plugin content.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a focused core plugin API change: add typed failover and terminal-failure hook names/events, emit them fire-and-forget from the existing failover decision and before-reply terminal failure seams, keep user-facing fallback messaging unchanged, and add tests plus docs for plugin authors.
