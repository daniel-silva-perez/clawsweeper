## Summary

Keep PR #42810 open. Current main still does not contain the proposed ACP control-plane audit logger, audit logger dependency, JSONL audit sink, or session lifecycle audit events. There is related gateway/config audit logging on main, but it covers config/update control-plane writes rather than ACP session init/close/turn auditing. The PR is not ready to merge as-is because review comments identify real correctness and product-boundary issues, but the central capability is neither implemented nor superseded.

## Best Possible Solution

Keep the PR open, but ask for a focused rebase/split that limits the diff to ACP audit work and resolves the review findings. The best product path is a maintainer-reviewed core audit contract and durable sink wired into the real ACP session manager, with tests for session init/close/turn events, best-effort audit failure isolation, retention behavior, buffer bounds, query correctness, and docs/config surface if the feature is user-configurable.
