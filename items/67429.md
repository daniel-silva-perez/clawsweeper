## Summary

Keep this PR open. Current main still does not implement null-clear behavior for cron payload model, fallbacks, or thinking overrides; the affected normalization, protocol schema, type contract, and service merge paths still only support null clearing for toolsAllow. The PR is a focused fix candidate with useful author follow-up, but it needs revision for the unresolved review concern that preserving null in the shared payload normalizer can regress cron.add by letting null reach the non-null create schema.

## Best Possible Solution

Keep this PR open and request a small revision: make null-clear a patch-only contract in CronPayloadPatch types/schema, preserve null through normalizeCronJobPatch without changing normalizeCronJobCreate behavior, delete model/fallbacks/thinking in mergeCronPayload when patch values are null, coerce null to undefined only when building a replacement payload, and add regression coverage for both cron.update null-clear and cron.add optional-null compatibility.
