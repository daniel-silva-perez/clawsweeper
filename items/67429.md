## Summary

Keep this PR open. Current main still does not allow cron.update patches to clear payload.model, payload.fallbacks, or payload.thinking with null; only toolsAllow has that null-clear path today. The PR is a focused candidate, but it still needs the review concern addressed so null preservation is patch-only and does not regress cron.add create normalization.

## Best Possible Solution

Keep this PR open and request a small revision: make null-clear a patch-only contract in CronPayloadPatch types/schema, preserve null through normalizeCronJobPatch without changing normalizeCronJobCreate behavior, delete model/fallbacks/thinking in mergeCronPayload when patch values are null, coerce null to undefined only when building a replacement payload, and add regression coverage for cron.update null-clear plus cron.add optional-null compatibility.
