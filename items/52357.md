## Summary

Keep open. Current main has already moved past the PR's 504-key parity target and now has generated zh-TW key parity for 734 keys, but the requested Taiwan-standard terminology is still not fully present on main. The PR is stale/manual generated output and should be reworked through the current i18n generator/glossary path, but the underlying requested capability remains real and no maintainer-selected canonical follow-up has landed.

## Best Possible Solution

Keep a zh-TW terminology fix open, choose one canonical PR branch, update `ui/src/i18n/.i18n/glossary.zh-TW.json` with the Taiwan-standard mappings, regenerate `ui/src/i18n/locales/zh-TW.ts` plus metadata/translation memory via the current `ui:i18n:sync` workflow, and preserve the existing structural parity guard.
