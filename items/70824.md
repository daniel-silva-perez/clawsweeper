## Summary

Keep PR #70824 open. Current `main` does not implement the explicit `input` preservation the PR targets, and the related closed PRs/issues do not provide a shipped replacement. The PR still needs maintainer review because its gateway cross-provider fallback was challenged as potentially too broad, but that is a review/change-request reason, not a cleanup-close reason.

## Best Possible Solution

Keep the PR open for maintainer review. The likely best path is to land or recreate the explicit `input` preservation with regression coverage in `src/agents/models-config.merge.ts`, then handle the gateway cross-provider capability question separately with provider/routing-aware tests so image support is not treated as globally interchangeable across providers.
