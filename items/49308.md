## Summary

Keep #49308 open. Current main still does not handle Discord `ChannelNameChange`, `ChannelIconChange`, or `ChannelFollowAdd` in the Discord system-event resolver, while the preflight path only drops/enqueues messages when that resolver returns non-null. The PR is a narrow, still-relevant fix for linked issue #24108 rather than obsolete or already implemented work.

## Best Possible Solution

Keep the PR open and refresh it against current main. The best path is to add the three missing Discord `MessageType` cases in `extensions/discord/src/monitor/system-events.ts`, include a focused regression test proving those message types are enqueued/dropped as system events, and keep the changelog entry with the required attribution style before maintainer merge.
