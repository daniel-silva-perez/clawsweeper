## Summary

Keep open. Current `main` does not implement Slack Canvas content reading: the Slack manifest/docs still request `files:read` but not `canvases:read`, the Slack message tool only exposes generic message/file actions, and the read/download paths use `conversations.history`/`conversations.replies` plus `files.info` without any Canvas API calls or Canvas-content parsing.

## Evidence

- **Slack manifest lacks Canvas scope:** The bundled Slack manifest on current `main` requests `files:read`/`files:write` and other standard scopes, but not `canvases:read`, so the requested Canvas read permission is not wired into the shipped integration.
