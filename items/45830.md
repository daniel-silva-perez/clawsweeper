## Summary

Keep open. Current `main` does not contain this PR’s origin-snapshot contract or fallback logic. The existing isolated-cron `delivery.channel: "last"` path still clears last-route metadata on fresh sessions, skips snapshotting anything at cron creation time, and falls back from session state straight to default channel selection.

## Evidence

- **Isolated sessions still clear last-route metadata:** Fresh isolated cron sessions still wipe `lastChannel`, `lastTo`, and `lastAccountId`, which is the condition this PR was written to compensate for.
  - file: [src/cron/isolated-agent/session.ts:75](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/cron/isolated-agent/session.ts#L75)
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Cron tool does not snapshot origin delivery context:** On current `main`, `cron-tool` only infers delivery when no target is present. An explicit `delivery.channel: "last"` makes `hasTarget` truthy, so no extra fallback metadata is captured.
  - file: [src/agents/tools/cron-tool.ts:576](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/agents/tools/cron-tool.ts#L576)

- **Delivery resolver still falls back directly to default channel selection:** If no channel is resolved from session state, `resolveDeliveryTarget()` goes from `preliminary.lastChannel` straight to `resolveMessageChannelSelection({ cfg })`; there is no origin-snapshot fallback on `main`.
  - file: [src/cron/isolated-agent/delivery-target.ts:104](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/cron/isolated-agent/delivery-target.ts#L104)
