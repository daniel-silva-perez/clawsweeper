## Summary

Keep open. Current main still does not contain the PR's scope-retention behavior for the reported non-local token/password auth path: missing-device operator bypass still depends on sharedAuthOk, sharedAuthOk is still computed separately from primary auth, and tests still encode scope clearing for the Tailscale/shared-token device-less case. The PR patch itself is stale against current main and should be adapted, but the underlying bug remains tracked by #51396 and related open PR #57889.

## Best Possible Solution

Keep the item open for maintainer review. The best path is to consolidate #51413, #51396, and the related open #57889 discussion, then land a current-main-compatible gateway fix that preserves non-local operator scopes only when server-derived primary token/password auth succeeds, with regression coverage for token and password plus the Tailscale/sharedAuthOk mismatch path. If maintainers choose #57889 as the canonical implementation PR, close this older PR as superseded at that point.
