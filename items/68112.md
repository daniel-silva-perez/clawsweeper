## Summary

Keep this PR open. Current main still awaits startup catch-up before arming the cron timer, so the central failure mode described by this PR is not implemented on main. The PR is a focused implementation candidate rather than an obsolete or duplicate cleanup item, even though the linked bug report is now closed.

## Best Possible Solution

Keep this PR open for maintainer review. Either land this PR after verifying the latest head against the startup catch-up failure, one-shot, timeout-disabled, and persistence-error cases, or replace it with an equivalent main-branch fix that guarantees start() arms the timer even when startup catch-up or repair persistence fails while preserving cron state semantics.
