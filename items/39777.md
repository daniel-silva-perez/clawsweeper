## Summary

Current main still matches the reported Control UI behavior: Debug snapshot refresh returns silently while disconnected, and the Snapshots card has no inline loading or disconnected-state UI beyond the button label.

## Best Possible Solution

Keep the issue open. The maintainable fix belongs in the Control UI Debug surface: pass connection state or a derived refresh-disabled reason into `renderDebug()`, show an inline loading/progress state inside the Snapshots card while `debugLoading` is true, show a disconnected callout or disabled reason when refresh cannot run, and add focused tests for `loadDebug()`/`renderDebug()` behavior. PR #39806 can be used as a starting point if it is refreshed and validated.
