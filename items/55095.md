## Summary

Do not auto-close #55095. Current main still intentionally returns `INVALID_REQUEST: missing scope: operator.write` for operator WebSocket callers that invoke write-scoped methods without an approved/write-capable scope. The issue is recent, has a second affected user comment, and the provided context does not show a canonical duplicate or shipped fix for remote Python/custom WebSocket clients.

## Best Possible Solution

Keep the issue open and ask for a redacted custom WebSocket reproduction: protocol version, first `connect` frame, whether `scopes` includes `operator.write`, whether the client signs `connect.challenge` with a device identity, the returned `hello-ok.auth` or pairing error, the exact write RPC method, gateway auth mode/bind/remote URL, and `openclaw devices list --json` output with secrets redacted. If the repro shows the Python client is following the documented protocol, fix the gateway auth/pairing path; if it is a docs/client gap, add a custom WebSocket client example or helper that performs scope/device-token pairing correctly.
