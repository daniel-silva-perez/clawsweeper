## Summary

Keep open. Current main still has the active-gateway cleanup-hint bug: doctor and status render cleanup commands from the current OpenClaw profile whenever any extra gateway-like service is detected, instead of rendering commands for the detected extra services. The SecretRef half has adjacent read-only handling on main, but the cleanup issue remains and the PR review comments identify revisions needed rather than an obsolete change.

## Best Possible Solution

Keep the PR open or ask the contributor to revise it: pass concrete detected `ExtraGatewayService` records into cleanup-hint rendering from both doctor and status, preserve actionable hints when legacy cleanup is skipped, generate scope-correct launchd/systemd commands, and add focused regression tests. Treat the SecretRef portion separately against the current read-only command-secret resolution path before adding any broad doctor-only escape hatch.
