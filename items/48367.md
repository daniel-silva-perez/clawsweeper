## Summary

Keep open. Current `main` does not implement the requested first-class custom Tailscale control-server support. Gateway config/schema still only expose `tailscale.mode` and `resetOnExit`, the CLI only accepts `--tailscale` and `--tailscale-reset-on-exit`, runtime validation only covers existing serve/funnel constraints, and docs still present direct `bind: "tailnet"` as the manual workaround rather than an explicit Headscale/custom-control-server flow.
