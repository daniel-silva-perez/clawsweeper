## Summary

Keep open. Current `main` still loads `~/.openclaw/.env` into the gateway process only, but sandbox exec env is built from sandbox config/request overrides rather than inherited host env, and the OpenShell backend still creates sandboxes with provider flags only and no credential-value resolution path.

## Evidence

  - file: [src/infra/dotenv.ts:281](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/infra/dotenv.ts#L281)

- **Config load path uses dotenv for process env only:** `maybeLoadDotEnvForConfig()` returns early unless `env === process.env`, then calls `loadDotEnv({ quiet: true })`.
  - file: [src/config/io.ts:915](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/config/io.ts#L915)
