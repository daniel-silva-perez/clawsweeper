## Summary

Keep open. Current main still loads builtin sqlite-vec through node:sqlite loadable-extension APIs and only degrades to stored JSON embeddings plus in-process cosine search when that load fails. The useful /opt/homebrew/bin workaround in the issue discussion helps QMD/PATH-based setups, but current code and docs do not prove that builtin OpenClaw memory now loads sqlite-vec on macOS when node:sqlite cannot enable extension loading.

## Best Possible Solution

Keep this issue open until maintainers either validate and document that current supported macOS Node builds load sqlite-vec successfully, or change the builtin memory vector-load path to use a SQLite build/binding that supports loadable extensions on macOS. If native acceleration remains unavailable by design, document the macOS limitation and the Homebrew/QMD workaround while keeping the in-process cosine path clearly labeled as degraded fallback behavior.
