## Summary

Keep PR #71898 open. Current main at 6b6dcafcee9ca86385b7c1ee2442752314d50257 still has the reported `memory_search` bug: it accepts and filters `corpus=sessions` hits, but the final result shaping hard-codes every memory-manager hit as `corpus: "memory"`. The PR is a narrow relevant fix with a focused regression test, and the provided review conversation shows the only bot readability concern was addressed in the second PR commit.

## Best Possible Solution

Keep the PR open and land this or an equivalent narrow memory-core fix after normal CI: type surfaced result `corpus` as `MemorySource`, set it from `result.source`, and keep the dedicated regression test proving `corpus=sessions` returns session transcript hits with `corpus: "sessions"`.
