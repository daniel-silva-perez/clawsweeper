## Summary

Keep open. Current `main` does not contain the proxy circuit-breaker implementation this PR proposes, and the affected proxy call paths still use direct proxy dispatch without any shared open/half-open/backoff state.

## Evidence

- **Core proxy helper still lacks circuit breaker logic:** `src/infra/net/proxy-fetch.ts` still sends every proxied request straight through `undiciFetch(..., { dispatcher })` and `resolveProxyFetchFromEnv()` still returns a raw `EnvHttpProxyAgent` fetch. There is no `proxy-probe` state, no fallback-to-direct path, and no backoff logic on current `main`.
