## Summary

Keep open. Current main still does not implement PR #51318's central request: a per-invocation `--skip-bootstrap` flag for `openclaw agents add`. Existing support is limited to `openclaw onboard --skip-bootstrap` and the persistent `agents.defaults.skipBootstrap` config default, which does not solve the PR's automation/CI use case for one-off agent creation. The PR remains potentially useful, but should address the review feedback about not letting `--skip-bootstrap` force non-interactive mode and should drop or justify the unrelated image-generation test change before landing.

## Best Possible Solution

Keep the PR open and ask for a narrow revision: add `--skip-bootstrap` to `openclaw agents add`, pass `skipBootstrap` through `AgentsAddOptions`, OR it with `agents.defaults.skipBootstrap` at both `ensureWorkspaceAndSessions` call sites, exclude it from `hasExplicitOptions`, add tests for non-interactive and interactive CLI paths, and update `docs/cli/agents.md`.
