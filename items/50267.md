## Summary

Keep #50267 open. Current main still does not visibly pass OpenRouter attribution headers through the generic image-understanding `complete()` path, and the later related PR #57561 is reported in the provided context as closed and unmerged rather than shipped. The PR needs an update because its original broad `resolveProviderAttributionHeaders(model.provider)` approach would also hit the OpenAI hidden-attribution concern raised in review; the current policy-gated request attribution helper is the safer direction.

## Best Possible Solution

Keep the PR open but ask for a current-main update: apply OpenRouter attribution on the image direct-completion path using `resolveProviderRequestPolicyConfig` or `resolveProviderRequestAttributionHeaders` with `provider`, `api`, `baseUrl`, `capability: "image"`, and `transport: "media-understanding"`, then add regression tests proving native/default OpenRouter image calls receive the three attribution headers while custom proxy/OpenAI hidden-attribution cases do not.
