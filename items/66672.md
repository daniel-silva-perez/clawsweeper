## Summary

Keep open. Current main has related safety primitives, but it does not implement the requested first-class Ask, Plan, and Modify workflow modes. Existing controls are tool-policy profiles, exec-specific approvals, advisory planning, optional workflow/diff plugins, and sandbox knobs; none provides a global mode switch where Ask is enforced read-only, Plan previews impacted changes and blocks mutation until validation, and Modify enables execution after confirmation.

## Best Possible Solution

Keep open for maintainer product/design work. The likely implementation belongs in core runtime/config/UI: add an enforced read-only Ask mode, a Plan mode that captures structured intended actions and blocks mutating tools until explicit confirmation, and a Modify mode that enables existing write/exec-capable paths after validation. Regression tests should prove enforcement at the tool-policy boundary, not only prompt wording.
