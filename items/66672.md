## Summary

Keep open. Current main has related safety controls, but not the requested first-class Ask, Plan, and Modify workflow modes. Tool profiles and allow/deny lists are granular policy knobs, `update_plan` is advisory progress tracking, and exec approvals gate shell execution rather than all modifications behind a validated plan.

## Best Possible Solution

Keep open for maintainer product/design work. A strong implementation would add first-class workflow modes in core: Ask as an enforced read-only tool policy, Plan as structured preview plus explicit confirmation before any mutating tool is available, and Modify as the existing write/exec-capable path. The likely scope is config/schema, effective tool policy, channel/UI mode controls, system-prompt guidance, docs, and regression tests proving the mode is enforced rather than prompt-only.
