## Summary

Current `main` clearly changed Slack Socket Mode auth-failure handling so `account_inactive` no longer retry-loops indefinitely, but the repo does not prove the underlying reported regression is fixed: the code still classifies `account_inactive` as a permanent auth failure and skips startup, and there is no local evidence that valid multi-bot Slack Socket Mode configs now connect successfully. Conservative cleanup says keep this open.
