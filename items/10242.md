## Summary

Keep open. Current `main` advertises Feishu threads, but the implementation still only supports replying into an existing message/thread or binding a DM to its single existing conversation. I did not find evidence of restored Feishu DM child-thread creation or a way to start a fresh DM thread/session from OpenClaw.

## Evidence

- **Feishu thread-reply still requires an existing message id:** The Feishu action handler exposes `thread-reply`, but it requires `messageId` and passes `replyInThread` only when replying to that existing message. There is no path here to create a new DM thread without a parent message.
  - file: [extensions/feishu/src/channel.ts:685](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/extensions/feishu/src/channel.ts#L685)
  - command: `nl -ba extensions/feishu/src/channel.ts | sed -n '570,760p'`
  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
- **Outbound send falls back to plain direct send when no reply target is provided:** `sendReplyOrFallbackDirect()` sends a top-level direct message whenever `replyToMessageId` is absent. The only threaded behavior is `message.reply(..., reply_in_thread: true)` against an existing message id.
  - file: [extensions/feishu/src/send.ts:126](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/extensions/feishu/src/send.ts#L126)
  - command: `nl -ba extensions/feishu/src/send.ts | sed -n '120,185p'`
  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
- **Feishu session bindings do not support child placement:** The Feishu session-binding adapter advertises only `placements: ["current"]` and explicitly returns `null` for `input.placement === "child"`. That blocks OpenClaw from creating a new child conversation/thread for Feishu DMs.
  - file: [extensions/feishu/src/thread-bindings.ts:253](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/extensions/feishu/src/thread-bindings.ts#L253)

  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
- **ACP test locks Feishu DM `--thread here` to the current DM conversation:** The current test expectation for `/acp spawn codex --thread here` in a Feishu DM binds `placement: "current"` with conversation id `user:ou_sender_1`, not a new per-thread child conversation.
  - file: [src/auto-reply/reply/commands-acp.test.ts:1371](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/src/auto-reply/reply/commands-acp.test.ts#L1371)

  - file: [docs/channels/feishu.md:271](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/docs/channels/feishu.md#L271)
