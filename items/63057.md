## Summary

Keep this PR open. Current main still contains the legacy Anthropic `options.thinkingBudgetTokens || 1024` fallback, the zero-budget path is still reachable, and the PR is a narrow implementation candidate rather than obsolete. It should get focused regression coverage before merge.

## Best Possible Solution

Keep this PR open as the implementation candidate. Add focused tests in `src/agents/anthropic-transport-stream.test.ts` proving that an explicit `thinkingBudgetTokens: 0` omits the legacy thinking block and that an undefined budget still uses the existing `1024` default, then let maintainers decide whether #63016's broader default-thinking behavior needs separate product follow-up.
