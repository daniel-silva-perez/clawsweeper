## Summary

Keep open. Current `main` still has the reported Feishu behavior: `checkBotMentioned()` treats any message containing `@_all` as a bot mention, `parseFeishuMessageEvent()` consumes that result directly, and the Feishu config schema has no `respondToAtAll` knob. The PR is still useful, but the review history also shows an unresolved regression in the proposed fix for messages that contain both `@_all` and a direct bot mention, so this is not strong-close material.
