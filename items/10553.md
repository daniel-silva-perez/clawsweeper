## Summary

## Close Comment

Closing from a Codex review. Current `main` already has the intended extension point for this: memory is an exclusive plugin slot, not a core-only subsystem (`docs/tools/plugin.md:109`, `docs/cli/memory.md:12`), and the Plugin SDK exposes `api.registerMemoryCapability(...)` plus `api.registerMemoryEmbeddingProvider(...)` specifically so third-party memory backends can integrate without core changes (`docs/plugins/sdk-overview.md:168`). The project vision also says optional capability should usually ship as plugins, new skills should go to ClawHub first, and core skill/service additions have a high bar (`VISION.md:54`, `VISION.md:68`, `VISION.md:101`). Given that scope, a MemOS integration belongs as an external OpenClaw plugin plus optional ClawHub skill/docs, rather than an OpenClaw core issue.
