## Summary

Keep open. Current main at 54f8e4145e43b35920d9b876bf1415eba21c2fd1 still routes `pnpm check:docs` through the Unix-only `git ls-files ... | xargs oxfmt` pipeline, and no replacement docs-format helper has landed. This PR targets a real unresolved Windows/PowerShell tooling bug, but the submitted implementation still has review blockers around Windows npm `.cmd` shim handling and incomplete coverage of the sibling `format:docs` write command.

## Best Possible Solution

Keep this PR open for maintainer review or ask the author to amend it. The best fix is a single docs-format wrapper that removes `xargs` from `format:docs:check` and preferably `format:docs`, preserves the current tracked docs file set, chunks arguments safely, handles Windows package-manager shims using an existing repo pattern such as `pnpm exec` or Windows-only `shell: true`, and reports child-process launch errors clearly. Once one corrected PR lands, close the duplicate PRs against that shipped fix and #44293.
