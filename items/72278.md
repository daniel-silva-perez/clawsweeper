## Summary

Keep open. Current main still has only low-level SecretRef batching by exact ref and source/provider group; it does not add the requested first-class external-vault provider pattern, backend identity model, or pass-scoped backend/entity memoization. The public docs still show 1Password and Vault examples in the one-exec-provider-per-secret shape this issue is asking OpenClaw to replace or steer away from.

## Best Possible Solution

Keep this issue open as a core SecretRef product/API design item. The likely path is to align it with #17311: define a supported shared external-backend provider pattern, update secrets.providers/configure/docs so backend paths live in SecretRef ids rather than provider commands, add pass-scoped backend memoization hooks or adapter behavior, and cover repeated refs through that supported path with regression tests.
