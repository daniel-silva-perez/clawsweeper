## Summary

Current main still has the reported stale optional `hello-ok.auth` schema/client contract, while the gateway handler constructs `auth` unconditionally. The issue also has open implementation PRs (#68161 and #68227) with closing intent, so it should stay open until maintainers review and land or close that PR path.

## Best Possible Solution

Keep this issue open and review the open implementation PRs. The best fix is a coordinated gateway protocol update that makes `HelloOkSchema.auth` required, updates protocol docs/examples, adds or adjusts a focused validation/regression test, and regenerates/updates affected client models such as the macOS protocol model and web UI type. After that PR lands, GitHub can close this issue as implemented.
