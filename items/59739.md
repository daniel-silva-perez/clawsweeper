## Summary

Keep open. The request is a concrete, recent Telegram feature request, not stale, not maintainer-authored, and has no protected labels. Current public docs still describe group chats as non-main sessions, with Telegram config supporting group/topic routing and mention/access controls but not a per-group option to collapse a group into the main DM session. Related open issues reinforce that shared or blurred group context is still an active design area, not conservative cleanup material.

## Evidence

- **current docs: groups are non-main sessions:** The Groups docs say group sessions use `agent:<agentId>:<channel>:group:<id>`, Telegram topics append `:topic:<threadId>`, and direct chats use the main session or per-sender sessions. They also explicitly say groups always use non-main session keys in the personal DMs + public groups pattern. ([docs.openclaw.ai](https://docs.openclaw.ai/groups))
  - file: [docs/channels/groups.md:135](https://github.com/openclaw/openclaw/blob/d6ef1fcf24eea3a5985d653a3baf400e32aa1b75/docs/channels/groups.md#L135)
