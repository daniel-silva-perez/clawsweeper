## Summary

Keep this issue open. Current main still normalizes `agent:<id>:subagent:<uuid>` QMD session keys to `undefined`, so the QMD scope parser has no channel or chatType to match against and the default-deny scope can reject subagent memory search. The related PR #70297 is about narrowing the default QMD scope policy and does not supersede this subagent normalization bug.

## Best Possible Solution

Keep the issue open and fix the QMD scope contract for spawned subagent session keys. The narrow product/code path is to make `agent:<id>:subagent:<uuid>` either inherit the parent session's QMD scope classification or receive an explicit, documented subagent classification, then add regression coverage in `src/memory-host-sdk/host/qmd-scope.test.ts` and the mirrored `packages/memory-host-sdk/src/host/qmd-scope.test.ts` if that package surface remains active. Treat #70297 as a separate default-scope policy decision, not a replacement for this bug.
