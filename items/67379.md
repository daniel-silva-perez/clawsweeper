## Summary

Keep this issue open. Current main still drops agent-prefixed subagent session keys during QMD scope normalization, so the default deny scope has no channel/chatType to match and the QMD search path skips those sessions. The related items in the provided context do not supersede this specific QMD subagent scope bug.

## Best Possible Solution

Keep the issue open and fix the QMD scope contract for spawned subagent session keys. The narrowest path is to make agent:<id>:subagent:<uuid> classify or inherit scope deliberately, then add regression coverage in src/memory-host-sdk/host/qmd-scope.test.ts and the mirrored package test if that package surface remains active. Treat #70297's default-scope policy change as a separate maintainer decision, not as a replacement for this normalization bug.
