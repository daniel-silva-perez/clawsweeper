## Summary

Keep open. Current `main` does not implement the PR’s snake_case transcript-repair change: `session-transcript-repair` still recognizes only camelCase tool-call block types, while the replay path still runs `sanitizeToolCallInputs()` and the user-facing error text still handles the exact missing-input failure. There is not strong evidence to close this as implemented or obsolete.

## Evidence

That means snake_case Anthropic/OpenAI-style blocks are still invisible to this repair path.
