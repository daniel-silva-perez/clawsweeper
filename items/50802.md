## Summary

Keep open. Current main still does not implement the PR's central fix: session transcript repair only recognizes camelCase tool-call block types, while the linked bug and PR are about Anthropic/native snake_case `tool_use`/`tool_call`/`function_call` blocks. The PR is a focused fix for an open linked bug (#48915), and the latest provided PR context shows the author also addressed the earlier review concern by extending the inner sanitization guard, but that behavior is not present on current main.

## Best Possible Solution

Keep PR #50802 open and finish it as the canonical fix for #48915. The best implementation is to recognize `tool_use`, `tool_call`, and `function_call` consistently across session transcript repair and shared tool-call extraction, then add focused regression tests for missing-input dropping, valid snake_case preservation in mixed turns, and `sessions_spawn` redaction for snake_case blocks.
