## Summary

Keep open. Current main still exposes Discord scheduled-event support only for list/create, and the shared message-tool action contract has no `event-edit` or `event-delete` action. This is a concrete bundled Discord plugin/message-tool enhancement, not a duplicate and not a clean ClawHub close, because the requested shared action names and schema are core-owned.

## Best Possible Solution

Keep this issue open and implement the missing lifecycle actions in the shared message-tool contract plus the bundled Discord plugin: add `event-edit`/`event-delete` to `src/channels/plugins/message-action-names.ts`, schema/target metadata, Discord action discovery and parsing, guild runtime handlers, REST helpers for Discord scheduled-event update/delete, focused Discord tests, and Discord docs. The existing issue comment's proposed merge-before-PATCH shape is a good implementation note to preserve required scheduled-event fields safely.
