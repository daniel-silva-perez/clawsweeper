## Summary

Keep this issue open. Current main still exposes Discord scheduled-event support only for list/create, and the shared message-tool action contract has no `event-edit`, `event-delete`, or `eventId` schema support. The reporter's follow-up accurately traces the existing Discord implementation path and describes a concrete extension to bundled Discord behavior rather than an already-shipped feature or external-only ClawHub item.

## Best Possible Solution

Keep this issue open and implement the missing lifecycle actions as a coordinated shared-contract plus bundled Discord plugin change: add `event-edit` and `event-delete` to the shared message-action list and schema, add Discord discovery/API mapping/runtime actions, add REST helpers for scheduled-event fetch/update/delete using Discord's PATCH/DELETE endpoints, preserve event fields safely before PATCH, add focused Discord tests, and update the public Discord docs at https://docs.openclaw.ai/channels/discord.
