## Summary

Keep #7281 open. Current main still lacks configurable `fallbackChannel` or `fallbackChannels` support for hook mappings: the config/schema, mapping pipeline, hook dispatch, cron delivery model, and direct delivery path all carry a single primary delivery target. The related closed Slack availability report #54029 does not supersede this fallback-chain feature request, and the issue discussion already contains a same-day Codex/maintainer review keeping it open.

## Best Possible Solution

Keep #7281 open until maintainers either implement configurable hook delivery fallback chains or explicitly reject the product behavior. A complete fix should add typed `fallbackChannel` or `fallbackChannels` config and validation, propagate ordered fallback targets through hook mapping resolution and `dispatchAgentHook` into cron/direct delivery, retry alternate targets only after primary delivery failure, keep this distinct from `channel: "last"` and `failureDestination`, document it on the public configuration reference, and add focused regression tests.
