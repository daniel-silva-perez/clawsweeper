## Summary

Keep open. Current main still drops Slack no-mention channel messages before any hook emission, while comparable Telegram and Signal silent-ingest paths already emit internal message:received hooks. The PR is not obsolete, but the submitted implementation needs rework because Slack uses channels.slack.channels, not the generic groups config surface used by resolveChannelGroupPolicy.

## Best Possible Solution

Keep this PR open and ask for a revision: add or document the Slack channel-level ingest contract on channels.slack.channels, read that Slack-specific policy in the mention-skip path, emit internal message:received via hook-runtime when enabled, simplify the unreachable slackFrom branch, and keep focused regression tests for wildcard ingest and channel-specific ingest:false override.
