## Summary

Current `main` already provides graceful config and cron reload without a full gateway restart. The implementation is automatic rather than a separate `openclaw gateway reload` command: gateway config changes are watched and hot-applied when safe, `cron` config changes trigger an in-process cron service swap, and the cron scheduler force-reloads `cron/jobs.json` from disk during timer processing.

## Evidence

- **Gateway config hot reload is documented and includes cron as no-restart automation:** `docs/gateway/configuration.md:498-546` says the Gateway watches `~/.openclaw/openclaw.json`, applies changes automatically, uses `hybrid` hot reload by default, and lists `cron` under automation fields that do not require restart.
  - file: [docs/gateway/configuration.md:498](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/docs/gateway/configuration.md#L498)

  - file: [src/gateway/config-reload-plan.ts:40](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/gateway/config-reload-plan.ts#L40)
