## Summary

Keep PR #63268 open. Current main does not implement the requested `doctor --fix` warning for a root-resolved config diverging from a service-user/running-gateway config, and the linked bug #63265 remains a plausible focused core doctor issue rather than cleanup-close material.

## Best Possible Solution

Keep the PR/linked bug open and ask for a robust core doctor preflight: resolve the exact path doctor will write, compare it against the managed service or running gateway config source when available, warn without destructive advice, and cover root/service-user, explicit `OPENCLAW_CONFIG_PATH`, custom state dir, and multi-gateway cases in tests.
