## Summary

Keep open. Current main has the later strict-agentic plan-only retry/blocked-state path, but it does not implement this PR's generic auto-reply control-plane stall guard, and the memory session-delta retry behavior is still absent. The PR should not be closed as superseded by #64241 because that merged PR explicitly refs #62989 as non-superseded, and current memory sync code still drops/decrements pending delta work before logging failed `session-delta` sync errors.

## Best Possible Solution

Keep the PR open, but ask for a focused rebase/split. The best maintainable path is to keep the already-shipped strict-agentic plan-only behavior on main, then land separate narrow patches for the still-missing pieces: a generic `dispatchReplyFromConfig` control-plane stall guard that refreshes on repeated plan/control progress and always cleans abort listeners, plus a memory-core session-delta retry/backoff path that restores pending thresholds after transient fetch failures. Unrelated subagent, diagnostics, command-queue, search fallback, and compaction changes should be split into separate PRs or dropped unless they have their own reproduced bug and tests.
