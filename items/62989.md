## Summary

Keep this PR open. Current main has related strict-agentic/update_plan and timeout work, but the PR’s central dispatch-level plan-stall fail-close and memory session-delta retry/backoff behavior are still absent. Several additional hunks remain unimplemented or differ intentionally, and existing review comments identify regressions that need author or maintainer handling before landing.

## Best Possible Solution

Keep this PR open, but request a rebase and split into smaller reviewed fixes. The likely useful pieces are dispatch-level fail-closed handling for plan/control progress stalls, memory session-delta retry/backoff that preserves pending work, and any memory search fallback that preserves current hybrid/vector semantics. Treat shipped strict-agentic behavior on main as the baseline and resolve the existing review findings before any landing decision.
