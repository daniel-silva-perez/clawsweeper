## Summary

Keep open. Current main still does not implement PR #49135's Telegram block-streaming coalesce defaults, and I did not find a shipped/superseding implementation. The PR remains a focused fix for the linked Telegram newline/message-splitting regression, though it now needs a small rebase/port to the evolved Telegram shared plugin base plus regression coverage.

## Best Possible Solution

Keep #49135 open and ask for/perform a rebase onto current main. Port the same behavior into `extensions/telegram/src/shared.ts` by adding `streaming: { blockStreamingCoalesceDefaults: { minChars: 1500, idleMs: 1000 } }` to the `createChannelPluginBase` call and including `"streaming"` in both current `Pick` type surfaces. Add a focused regression assertion in `extensions/telegram/src/shared.test.ts` for the Telegram default, then run the Telegram extension/changed lanes before landing. Because this is user-facing, add a changelog fix entry with contributor attribution when it lands.
