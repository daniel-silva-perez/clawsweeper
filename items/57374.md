## Summary

Keep open. Current `main` still performs gateway restart health probes with env-only auth, so the PR's behavior is not implemented on `main` and the reported mismatch remains plausible.

## Evidence

- **Restart health probe still uses env-only credentials:** `confirmGatewayReachable()` reads only `process.env.OPENCLAW_GATEWAY_TOKEN` / `OPENCLAW_GATEWAY_PASSWORD` before probing `ws://127.0.0.1:${port}`; there is no config-resolved probe-auth input on current `main`.
