## Summary

Keep open. Current `main` already avoids injecting full `SKILL.md` bodies into the base system prompt by default, but the stronger boundary requested here is not clearly implemented: selected skills are still read into the same agent context on demand, and the repo’s own security docs still describe skill sandboxing/credential isolation as recommended future work rather than shipped behavior.
