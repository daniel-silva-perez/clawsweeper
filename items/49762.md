## Summary

Keep PR #49762 open. Current main still does not add response duration to the per-response `/usage` footer; the only reply-run duration on this path is emitted to diagnostics, while the footer formatter and caller still pass only usage/cost data. The PR is not merge-ready as-is because review comments identify real formatting, mode, and measurement semantics concerns, but the requested capability remains unimplemented and not superseded by the related usage-footer items.

## Best Possible Solution

Keep the PR open, but revise it before merge. The best implementation is to rebase onto the current `agent-runner-usage-line.ts` split, add focused formatter/runReplyAgent tests, preserve existing token/cache/cost formatting when duration is absent, decide whether duration belongs only in `/usage full` or a new mode, and label/sample the timing consistently as generation time or end-to-end turn time.
