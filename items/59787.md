## Summary

Keep open. The PR discussion identifies an unresolved correctness bug on the PR head: `sendTarget.hasText` is set whenever a target exists, so media-only sends can still suppress the subsequent text reply. That means the PR does not yet satisfy its own stated fix, and there is not enough verified current-main evidence to close it as implemented elsewhere.

## Evidence

- **PR head sets hasText unconditionally:** The provided PR patch for head `336659226e08b8e0fee731d4b03e924da43266d0` assigns `sendTarget.hasText = true` inside `if (sendTarget)`, independent of whether `content` or `message` text exists.
  - file: [src/agents/pi-embedded-subscribe.handlers.tools.ts:427](https://github.com/openclaw/openclaw/blob/336659226e08b8e0fee731d4b03e924da43266d0/src/agents/pi-embedded-subscribe.handlers.tools.ts#L427)
  - sha: [336659226e08](https://github.com/openclaw/openclaw/commit/336659226e08b8e0fee731d4b03e924da43266d0)
- **Suppression depends on hasText:** The PR changes suppression to skip targets where `!target.hasText`, so setting `hasText` true for media-only sends defeats the intended media-only behavior.
  - file: [src/auto-reply/reply/reply-payloads-dedupe.ts:154](https://github.com/openclaw/openclaw/blob/336659226e08b8e0fee731d4b03e924da43266d0/src/auto-reply/reply/reply-payloads-dedupe.ts#L154)
  - sha: [336659226e08](https://github.com/openclaw/openclaw/commit/336659226e08b8e0fee731d4b03e924da43266d0)
- **Review discussion confirms unresolved issue:** Reviewer `yfge` explicitly noted on 2026-04-03 that `sendTarget.hasText` is currently set to true whenever `sendTarget` exists and suggested deriving it from the same text check so media-only sends do not suppress the following text reply.
