## Summary

Keep open. Current `main` still limits `session.reset` to `daily | idle`; there is no user-configurable `mode: "never"` or equivalent documented manual-only mode.

## Evidence

- **Type contract still excludes `never`:** `SessionResetMode` is only `"daily" | "idle"`, and `SessionResetConfig` uses that type for base/type/channel reset config.
  - file: [src/config/types.base.ts:124](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/config/types.base.ts#L124)
  - command: `nl -ba src/config/types.base.ts | sed -n '118,175p'`
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **Runtime schema rejects `never` and `idleMinutes: 0`:** `SessionResetConfigSchema` only allows `daily` or `idle`, and `idleMinutes` is `.positive()`, so a config-level manual-only sentinel like `0` is not accepted.
  - file: [src/config/zod-schema.session.ts:17](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/config/zod-schema.session.ts#L17)
  - command: `nl -ba src/config/zod-schema.session.ts | sed -n '1,60p'`
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **Reset policy resolver has no `never` branch:** `resolveSessionResetPolicy` resolves only `daily` or `idle`, with default mode `daily`. `evaluateSessionFreshness` disables idle expiry when `idleMinutes <= 0`, but there is no public reset mode for manual-only behavior.
  - file: [src/config/sessions/reset-policy.ts:4](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/config/sessions/reset-policy.ts#L4)
  - command: `nl -ba src/config/sessions/reset-policy.ts | sed -n '1,110p'`
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **Docs still document only two reset modes:** Gateway config docs show `mode: "daily" // daily | idle` and describe reset policy as only `daily` or `idle`.
  - file: [docs/gateway/config-agents.md:1109](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/docs/gateway/config-agents.md#L1109)
  - command: `nl -ba docs/gateway/config-agents.md | sed -n '1100,1165p'`
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **Internal test confirms partial but not user-exposed behavior:** There is a unit test that treats `idleMinutes=0` as never expiring by inactivity, which suggests a possible internal workaround, but the config schema still blocks users from setting it.
  - file: [src/config/sessions/sessions.test.ts:128](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/config/sessions/sessions.test.ts#L128)
  - command: `nl -ba src/config/sessions/sessions.test.ts | sed -n '120,150p'`

## Risks / Open Questions

- There is partial runtime support for a non-expiring idle policy via `idleMinutes=0`, but because validation and docs exclude it, closing this as implemented would be misleading.
