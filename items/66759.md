## Summary

Keep open. Current main still hard-codes Feishu card footers to agent/model/provider metadata, exposes no `cardFooter`/`showContext`/directive-style config, and tests still assert the static note behavior. Related Feishu footer work exists (#60748 and PR #64805), but it has not shipped on main and does not fully supersede this specific custom context/compaction footer request.

## Best Possible Solution

Keep this open as concrete Feishu plugin work. Implement a Feishu-owned footer config surface, likely top-level plus per-account under `channels.feishu`, and wire selected safe fields through `resolveCardNote` for streaming and non-streaming cards. Coordinate with #60748 and PR #64805, update schema/manifest/docs, and add dispatcher tests for configured context usage, compaction count, missing/stale usage data, and default backward-compatible behavior.
