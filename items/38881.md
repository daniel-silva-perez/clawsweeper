## Summary

Keep open. Current main still does not implement this PR's core behavior: inbound image messages do not auto-switch the main agent run to `agents.defaults.imageModel` when the selected reply model lacks vision. `imageModel` is currently used by the separate media-understanding path, and the PR remains materially relevant, though it still needs maintainer review for the bot-raised design/correctness concerns.

## Best Possible Solution

Keep this PR open until a maintainer either lands a refined implementation or explicitly chooses the existing media-understanding behavior as the intended product boundary. A good fix would live around `src/auto-reply/reply/get-reply.ts` with focused regression tests, thread the real alias index, respect heartbeat/session/channel overrides, handle `MediaPath(s)` and `MediaUrl(s)` safely, honor `imageModel` fallbacks, and avoid duplicate image-understanding work when the main run switches to a vision model.
