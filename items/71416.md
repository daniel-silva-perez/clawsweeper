## Summary

Current main still implements the reported behavior: `openclaw agent` catches any Gateway failure, logs a stderr diagnostic, and continues through the embedded agent path. Tests and docs preserve that behavior; there is still no opt-in fallback flag, nonzero exit behavior, stdout banner, or stable JSON transport/fallback marker.

## Best Possible Solution

Keep #71416 open for a focused CLI behavior change. The best path is to make fallback explicit and script-detectable: fail closed unless an `--allow-embedded-fallback` flag is passed, or add a stable JSON marker such as `meta.transport`/`meta.fallback` plus a clear warning path, then update `src/commands/agent-via-gateway.ts`, CLI registration, tests, and `docs/cli/agent.md` together.
