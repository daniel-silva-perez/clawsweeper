## Summary

Current main already has both sides of the oversized-session defense: the Control UI only requests and renders the last 200 history messages, and the gateway truncates/replaces oversized `chat.history` payloads before they reach the browser. Those guardrails are also present in the shipped `v2026.4.22` release.

## Close Comment

Closing this as implemented on main after a Codex review. Current OpenClaw now defends this failure mode in two places: the Control UI only requests and renders the last 200 chat-history messages (`ui/src/ui/controllers/chat.ts:152`, `ui/src/ui/chat/build-chat-items.ts:7` and `:201`), and the gateway truncates or replaces oversized `chat.history` entries with `[chat.history omitted: message too large]` before sending them to the browser (`src/gateway/server-methods/chat.ts:1914`). The regression path is covered by `src/gateway/server.chat.gateway-server-chat-b.test.ts:300`, which writes a transcript with a 120 KB nested tool-result payload and verifies the placeholder is returned while recent smaller messages still survive. Those same UI windowing guards and oversized-history tests are already present in the shipped `v2026.4.22` release tag (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`), so this no longer matches current released behavior.
