## Summary

Keep PR #71154 open. Current main still rejects shell line continuations before allowlist analysis, tests assert that fail-closed behavior, and both gateway/node-host paths still report analysis failures as allowlist misses. The PR is not implemented or superseded on main; the related #71225 was closed unmerged, while #71153 still tracks the same user-visible bug.

## Best Possible Solution

Keep this PR open for maintainer review. If OpenClaw wants POSIX line-continuation support in core exec allowlist mode, the fix belongs in `src/infra/exec-approvals-allowlist.ts`/analysis plus the gateway and node-host denial paths, with tests that prove allowlisted multiline commands work while prior command-substitution bypass cases and unparseable syntax still fail closed.
