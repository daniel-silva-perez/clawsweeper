## Summary

Current `main` already has optional FTS5-backed session-transcript indexing and `memory_search`/CLI support, but the issue asks for more than that. I found no strong evidence that OpenClaw already ships the requested LLM-powered cross-session summarization flow or a dedicated `/search` command in CLI/messaging, so this is not safe to close as implemented.
