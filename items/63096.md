## Summary

Current main does not contain the PR's gateway completion-wake prompt reconstruction. The agent gateway path still forwards only caller-provided `extraSystemPrompt`, while the normal chat path separately builds inbound/group prompt context; the PR's new regression tests are also absent. The linked broader bug remains open, so this PR is not safely obsolete or implemented on main.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase/adapt the gateway completion-wake context reconstruction to current main, preserve explicit caller `extraSystemPrompt`, and add focused tests in `src/gateway/server-methods/agent.test.ts` proving inter-session `task_completion` wakes match normal session context shaping without reintroducing prompt-cache drift.
