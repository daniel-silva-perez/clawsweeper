## Summary

Keep this PR open. Current main still has the reported gap: gateway `agent` forwards `request.extraSystemPrompt` unchanged, the subagent completion announce path sends `inter_session` `task_completion` wakes without `extraSystemPrompt`, and the normal chat path is still where `Inbound Context` and group context are assembled. The PR remains a focused implementation candidate for #63030, and the diff is limited to gateway source/tests with no observed security-sensitive workflow, dependency, script, secret, or publishing changes.

## Best Possible Solution

Keep this PR open as the implementation candidate for the completion-wake portion of #63030. The best path is to refresh it onto current main, retain the synthesized persisted context helper with explicit `extraSystemPrompt` pass-through, keep the focused regression tests, and have gateway maintainers verify prompt parity with normal chat before merging. If heartbeat-specific suffix drift remains after this lands, track that separately under #63030 rather than closing the broader cache-stability work prematurely.
