## Summary

Keep PR #71203 open. Current main still only refreshes `models.json` during `prewarmConfiguredPrimaryModel`, which returns early when no explicit default primary model is configured and only targets `resolveOpenClawAgentDir()` when it does run. The PR addresses a real remaining gap from #71192 by adding startup refresh for configured agent directories, though its latest diff still deserves review for the synthetic `main` fallback/override-dir concern raised in review comments.

## Best Possible Solution

Keep the PR open and finish it rather than closing it. The best path is to land a startup helper that refreshes `models.json` for the active default agent dir plus explicitly configured agent dirs, with a regression test covering configured non-default agents and the no-default-primary case. Before merge, adjust or justify the `main` fallback behavior so startup cache refresh does not write outside the selected override agent dir unintentionally.
