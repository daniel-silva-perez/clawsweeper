## Summary

Keep this PR open. Current main still refreshes startup model cache only through primary-model prewarm: it returns before any cache write when no explicit default primary model is configured, and otherwise writes only the active default agent dir. The PR remains a focused implementation candidate for the stale non-default agent `models.json` cache bug tracked by #71192, though the current diff still needs maintainer review around synthetic `main` and override agent-dir semantics.

## Best Possible Solution

Keep this PR open for review or revision. The best implementation is a startup refresh helper that refreshes unique concrete configured agent dirs independently of primary-model warmup, preserves default and override agent-dir semantics, avoids synthetic `main` duplicate writes, catches per-dir refresh failures without aborting gateway startup, and adds focused regression coverage for non-default agents and the no-default-primary case.
