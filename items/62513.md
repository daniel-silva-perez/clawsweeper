## Summary

Keep open. Current `main` still performs the reported `hasIndexedContent()` preflight on every memory search, and that helper still hits SQLite directly. The issue timeline only shows a cross-reference to an open PR, not a landed fix on `main`.

## Evidence

- **Per-search preflight still calls hasIndexedContent:** `MemoryManager.search()` initializes `let hasIndexedContent = this.hasIndexedContent();` before running the preflight, so each search still evaluates the indexed-content check. If the first check is false it calls the helper again after a forced sync.
