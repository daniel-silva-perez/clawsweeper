## Summary

Keep PR #51282 open. Current main still does not provide `gateway.handshakeTimeoutMs`; it only ships the 10s default plus `OPENCLAW_HANDSHAKE_TIMEOUT_MS`. The linked env/default fix covers part of the original timeout problem, but it does not make the timeout configurable from the config file as this PR requests. The PR remains potentially useful, though its branch needs the already-raised schema/label/test fixes and should avoid loading config on every pre-auth socket.

## Best Possible Solution

Keep the PR open as a still-relevant feature request, but require a revised implementation: add `gateway.handshakeTimeoutMs` across `GatewayConfig`, Zod schema, labels/help, generated config docs/baselines as applicable, and tests; preserve env > config > default priority; and wire the resolved value through gateway startup/reload state rather than calling `loadConfig()` from each pre-auth WebSocket connection. If maintainers decide the shipped `OPENCLAW_HANDSHAKE_TIMEOUT_MS` is intentionally sufficient, close with that explicit product decision rather than an automated cleanup close.
