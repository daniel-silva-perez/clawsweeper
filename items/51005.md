## Summary

Keep #51005 open. Current main still rejects `freshness`, `date_after`, and `date_before` in Brave `llm-context` mode, while the PR addresses a real provider behavior gap and the related Tavily PR #49200 is not a substitute for this Brave-specific fix.

## Best Possible Solution

Keep #51005 open. The remaining product/code path is to port the fix onto current main's Brave plugin runtime: remove the over-restrictive llm-context guards, pass supported freshness/date ranges to `/res/v1/llm/context`, keep `ui_lang` rejected, update cache keys, update `docs/tools/web.md` and `docs/tools/brave-search.md`, and add focused regression tests covering freshness, explicit date ranges, date_after-only, and date_before-only semantics.
