## Summary

Keep open. Current main still does not verify or repair the global `openclaw` bin link after npm global updates. The related PR #63794 attempts the right class of fix, but it is still open, unmerged, and has review concerns plus an unrelated file in the diff, so it is premature to close the issue as implemented or superseded.

## Best Possible Solution

Keep #63719 open until main ships a focused updater fix. The best path is to add a post-npm-global-install verification/repair step that reads normalized package `bin` metadata, derives the owning global bin directory from the npm prefix/root, removes only confirmed stale npm temp bin entries, recreates or clearly errors on a missing/broken `openclaw` link, and covers the behavior with regression tests. PR #63794 can be used as a starting point after removing unrelated changes and addressing the review comments.

## Evidence

It explicitly skips non-directories, so it does not address a hidden temp file/symlink left under the global bin directory such as `~/.npm-global/bin/.openclaw-OKPnnPWD`.
