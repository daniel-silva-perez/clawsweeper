## Summary

Current `main` still drops `RuntimeLogger` metadata in `src/plugins/runtime/runtime-logging.ts`, while the public runtime contract and existing plugin call sites use `logger(message, meta)`. This is not already implemented on `main`, and the PR's proposed `logger.<level>(message, meta)` shape does not match the repo's structured logging convention, so conservative maintainer cleanup is to keep the PR open.
