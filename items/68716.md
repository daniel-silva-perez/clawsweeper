## Summary

Keep this PR open. Current main still lacks Discord guild-admin requester authorization, and the linked bug #68703 remains open, so the PR is a plausible implementation candidate. It should not be closed as obsolete, but it needs maintainer review/revision for the CLI/manual owner regression called out in review.

## Best Possible Solution

Keep this PR open and ask for a revision rather than closing it. The fix should remain in the Discord plugin: mark privileged guild-admin message actions as trusted-requester-only for tool-driven contexts, pass the trusted ctx.requesterSenderId into the guild-admin runtime, verify the requester has the matching Discord permission before bot-credentialed mutations, and add regression tests. The implementation also needs an explicit treatment for trusted owner/CLI/manual execution so local admin commands are not broken by a missing Discord sender id.
