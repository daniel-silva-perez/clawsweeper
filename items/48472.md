## Summary

Keep open. Current `main` at `4013c658537ee2051c44186f197b0710e84c9049` still lacks the PR's core behavior: daemon status probes still route `requireRpc` checks through `callGateway(... method: "status")`, restart health still uses a fixed 3s local probe timeout with no remaining-budget clamp, and `probeGateway` still only supports `none|presence|full` detail levels without the proposed health-only/loopback-device-identity path.
