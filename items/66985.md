## Summary

Keep PR #66985 open. Current main still compares `boundNode` and `requestedNode` as raw strings before resolving node names/prefixes to canonical IDs, while config/docs explicitly allow node bindings by id or name. The linked bug #61229 matches this failure mode, and the PR remains a focused, relevant fix rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open and either merge it after maintainer review or port the same fix directly: in `executeNodeHostCommand`, resolve both `boundNode` and `requestedNode` through the current node list before comparison, compare canonical IDs, keep stale/unresolvable bindings fail-closed, and add regression tests for display-name match, partial-ID match, wrong-node rejection, and not-found errors.
