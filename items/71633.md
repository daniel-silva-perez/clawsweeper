## Summary

Keep open. Current main still ships the default-enabled Bonjour plugin on plain @homebridge/ciao@1.3.6 with no pnpm patch, and the upstream 1.3.6 NetworkManager Windows arp polling path still calls child_process.exec without an options object. This PR remains a narrow current-main fix for linked bug #71492, not conservative cleanup to close.

## Best Possible Solution

Keep #71633 open for maintainer review. The best path is to land this narrow @homebridge/ciao@1.3.6 pnpm patch, or an equivalent upstream dependency update if one becomes available, with lockfile consistency and either Windows manual verification or a focused regression check proving ciao's Windows arp polling exec receives windowsHide: true.
