## Summary

Keep open. #72203 is a fresh, concrete regression report against v2026.4.24, and current main does not show a relevant token-mismatch fix after that release. The code has the intended QuickStart/dashboard token plumbing, but the reported setup health-check failure remains plausible and needs maintainer reproduction rather than cleanup closure.

## Best Possible Solution

Keep #72203 open. Reproduce on a clean Ubuntu 25.04 curl install with QuickStart defaults, then compare `gateway.auth.token`, `OPENCLAW_GATEWAY_TOKEN`, state-dir `.env`, the generated systemd user unit environment, `openclaw dashboard --no-open`, and the token sent by the setup health probe. If reproduced, fix setup/finalize/daemon install so first-run service startup, health checks, and dashboard bootstrap all use one resolved token source, and add a regression test for the clean QuickStart token-auth install path plus any stale-env/state-dir token drift case discovered.
