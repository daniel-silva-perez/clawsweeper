## Summary

Keep this PR open. Current main at e962381dbf1ecdb1cd4784fdeb6c2e07ddd3435b still routes the CLI default picker, CLI allowlist picker, and gateway model catalog through `loadModelCatalog` without a `models.mode === "replace"` branch, while the same-author linked bug #64950 remains open. The PR is still a plausible fix candidate, but it needs maintainer resolution on implicit/plugin-discovered model semantics and focused regression coverage.

## Best Possible Solution

Keep this PR and #64950 open until a maintainer-chosen fix lands. The best implementation is a shared mode-aware catalog path for the CLI picker and gateway `models.list` that honors `models.mode: "replace"`, explicitly defines whether implicit/plugin-discovered models are included, preserves configured metadata and aliases, and adds focused regression tests for the default picker, allowlist picker, and gateway catalog output. The adjacent open PR #42797 may also need maintainer comparison because it touches overlapping gateway model-picker scoping.
