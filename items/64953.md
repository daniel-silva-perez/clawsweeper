## Summary

Keep open. Current main still ignores `models.mode: "replace"` in both the CLI model picker and gateway model catalog, while the config help/docs define replace mode as using only configured providers. The related fixing PRs in the supplied context are closed/unmerged and the linked bug #64950 remains open, so there is no landed superseding fix to close this against.

## Best Possible Solution

Keep #64953 or the canonical #64950 path open and land a maintainer-reviewed fix that makes the picker and gateway catalog honor `models.mode: "replace"` without accidentally dropping intended implicit/plugin-discovered models. The cleanest direction is likely to define replace-mode semantics in the catalog layer or a shared helper, then cover CLI picker and gateway `models.list` behavior with focused tests.
