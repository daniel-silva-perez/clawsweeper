## Summary

Current main still has the Discord chunking bug this PR targets: long non-whitespace lines are split only by whitespace fallback, then hard-split at the limit. The PR is recent, focused, and materially relevant to open issue #38597; the prior Ext-B review concern was addressed in the PR discussion/head commit, so this should proceed through normal PR review/CI rather than ClawSweeper closure.

## Best Possible Solution

Keep #71384 open. The best maintainer path is to review and land this PR, or an equivalent codepoint-aware CJK breakpoint implementation plus regression tests in `extensions/discord/src/chunk.ts` and `extensions/discord/src/chunk.test.ts`; then close #38597 as fixed and leave the older unmerged #39399 superseded by this PR.
