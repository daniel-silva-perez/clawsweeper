## Summary

Keep open. Current main has external-content wrapping and existing exec/tool approval machinery, but the requested end-to-end model is not implemented: there is no persistent external-content taint carried into exec approvals, no taint-aware block on content-derived execution/secret/outbound actions, and no exec risk-class system matching the issue.

## Evidence

- **External content is wrapped with warnings, not persisted as execution-taint metadata:** The external-content layer adds `EXTERNAL_UNTRUSTED_CONTENT` markers and a warning banner telling the model not to treat the content as instructions or execute commands from it.

The documented global rule is specifically `inter_session` provenance, and gateway task tracking only special-cases `inter_session`.

## Risks / Open Questions

- This is an umbrella security issue spanning provenance, exec approvals, secret access, and outbound messaging; it likely needs decomposition, but that is not a reason to close it.
- Closing as implemented would overstate current protections: main has strong wrapping and approval primitives, but not the persistent taint enforcement the report requests.
