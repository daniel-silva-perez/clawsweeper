## Summary

Keep open. Current `main` still sends memory remote embedding requests through `fetchWithSsrFGuard` without opting into trusted env-proxy mode, while the guard defaults to strict pinned-DNS behavior unless `mode` is explicitly set. That matches the report’s proxy-only `fetch failed` theory rather than showing the bug is already fixed on `main`.

## Evidence

- **Memory remote embeddings still use bare guarded fetch:** `withRemoteHttpResponse()` calls `fetchWithSsrFGuard` with only `url`, `fetchImpl`, `init`, `policy`, and `auditContext`. It does not pass `mode: GUARDED_FETCH_MODE.TRUSTED_ENV_PROXY`, `proxy: "env"`, or any equivalent env-proxy helper.
  - file: [src/memory-host-sdk/host/remote-http.ts:14](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/memory-host-sdk/host/remote-http.ts#L14)
  - command: `nl -ba src/memory-host-sdk/host/remote-http.ts | sed -n '1,120p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Guarded fetch defaults to strict mode:** `resolveGuardedFetchMode()` returns strict unless the caller explicitly sets `mode`, or uses the deprecated `proxy: "env"` plus `dangerouslyAllowEnvProxyWithoutPinnedDns`. Memory remote embeddings do neither.
  - file: [src/infra/net/fetch-guard.ts:113](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/infra/net/fetch-guard.ts#L113)
  - command: `nl -ba src/infra/net/fetch-guard.ts | sed -n '90,390p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Strict path still does local hostname resolution; env proxy path only activates when mode is set:** Inside `fetchWithSsrFGuard()`, trusted env proxy creates `EnvHttpProxyAgent` only when `mode === TRUSTED_ENV_PROXY`. Otherwise the code falls through to `resolvePinnedHostnameWithPolicy(...)` and pinned dispatcher setup, which is the local-DNS path the report says fails in proxy-only environments.
  - file: [src/infra/net/fetch-guard.ts:357](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/infra/net/fetch-guard.ts#L357)
