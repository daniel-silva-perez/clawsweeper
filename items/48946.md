## Summary

Keep #48946 open. Current main still does not implement Feishu missed-message recovery after gateway sleep/reconnect, and the linked user problem is not otherwise tracked by an open canonical item. The PR is not mergeable as-is because the provided review discussion identifies serious API, lifecycle, and routing defects, but the underlying Feishu plugin gap remains concrete enough for maintainer/product review rather than automatic closure.

## Best Possible Solution

Keep the PR open for maintainer judgment, but require a replacement implementation before merge: wire recovery into the Feishu monitor lifecycle, use `createFeishuClient` plus the correct Lark chat/message list APIs, reconstruct events from top-level message fields, reuse shared `chatHistories`, sequential routing, and durable dedupe, add config/permission docs, and cover startup/reconnect backfill with Feishu plugin tests. If maintainers decide this should not live in the bundled Feishu plugin, redirect it explicitly to the published `@openclaw/feishu` ClawHub/NPM plugin path or a plugin API discussion.
