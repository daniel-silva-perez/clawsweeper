## Summary

Keep #72037 open. Current main still generates the Codex ACP wrapper with a default `npm exec`/`npx` path for `@zed-industries/codex-acp@^0.11.1`, while the bundled `@openclaw/acpx` package only depends on `acpx@0.6.1`. The related #60693 discussion points at the same bundling concern, and #71549 was an unmerged ACPX bump, but neither provides a shipped fix or a better canonical tracker for this focused regression report.

## Best Possible Solution

Keep this focused issue open and fix the ACPX Codex packaging path in `extensions/acpx`: resolve and use a plugin-local bundled `@zed-industries/codex-acp` binary before any npm/npx fallback, add a regression test that the default Codex ACP command no longer invokes `npm exec`/`npx`, and update ACP agent docs to distinguish bundled adapter availability from explicit command overrides.
