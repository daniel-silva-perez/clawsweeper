## Summary

Keep open. Current `main` does not implement the requested chat-side `/sessions` list or `/session <key>` switch flow. The built-in `/session` command is still limited to `idle` and `max-age` thread-binding controls, and the docs still point users to CLI/web surfaces for session inspection. `/focus <target>` provides a partial Discord/Telegram-only binding mechanism, but it is not the requested cross-channel session list/switch feature.

## Evidence

- **Built-in `/session` only exposes binding-expiry actions:** The command registry defines `session` with only `idle` and `max-age` choices; there is no built-in `/sessions` command and no `/session <key>` variant.
  - file: [src/auto-reply/commands-registry.shared.ts:324](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/src/auto-reply/commands-registry.shared.ts#L324)

  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)

The implementation also says these subcommands are only for channels with focused conversation bindings.
