## Summary

Keep open. Current `main` now ships official backup archive creation and verification, but the request in #62461 is broader: there is still no general `openclaw backup restore` or `migrate import/export` workflow, and the migration docs still rely on manual archive extraction plus `openclaw doctor`. The linked standalone `JIRBOY/openclaw-backup` tool is useful prior art from the discussion, but it does not make the core item implemented or out of scope.

## Evidence

- **Backup CLI exists, but only as create/verify:** `registerBackupCommand` defines `openclaw backup create` and `openclaw backup verify <archive>` only. There is no general restore/import subcommand in the shipped backup CLI surface.
  - file: [src/cli/program/register.backup.ts:10](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/src/cli/program/register.backup.ts#L10)

They do not document a general restore/import command or cross-platform path adaptation flow.
  - file: [docs/cli/backup.md:9](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/docs/cli/backup.md#L9)
