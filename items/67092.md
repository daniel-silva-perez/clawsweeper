## Summary

Current main still preserves assistant text before an orphan closing reasoning tag, including shared delivery/history sanitization and stateful streaming paths. The issue is not implemented or superseded because the two linked fix PRs were closed unmerged.

## Best Possible Solution

Keep this issue open. Land a focused core fix that treats an orphan closing reasoning tag outside an active reasoning block as a privacy boundary when answer text follows it, and apply the same rule to the stateful streaming sanitizer or route streaming through the shared implementation. Add regression coverage in `src/shared/text/reasoning-tags.test.ts`, `src/shared/text/assistant-visible-text.test.ts`, `src/agents/pi-embedded-subscribe*` tests, and UI/channel preview tests for the malformed `reasoning </think> final` shape while preserving code/literal tag examples.

## Evidence

A malformed `reasoning </think> final` shape therefore keeps the reasoning prefix with the final answer.

## Risks / Open Questions

- A fix that blindly drops everything before any literal `</think>` could remove legitimate user-visible prose or documentation examples; keep the existing fenced/inline code protections and add literal-tag regression cases.
- There are at least two relevant stripping implementations: the shared text helper and the stateful streaming `stripBlockTags` path. Updating only one would leave previews, block replies, or stored assistant text with inconsistent behavior.
