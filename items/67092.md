## Summary

Keep open. Current main still preserves text before an orphan closing reasoning tag, and the shared delivery/history sanitizers and several channel preview paths depend on that same behavior. The two linked fix attempts were closed unmerged, so they do not implement or supersede this issue.

## Best Possible Solution

Keep this issue open and land a focused core fix in `src/shared/text/reasoning-tags.ts`: treat a closing reasoning tag encountered outside an active reasoning block as a privacy boundary for malformed model output, discarding the accumulated prefix when there is answer text after the orphan close. Add regression tests in `src/shared/text/reasoning-tags.test.ts`, `src/shared/text/assistant-visible-text.test.ts`, and at least one assistant extraction/history or channel-preview path that currently uses the shared sanitizer.

## Evidence

- **Shared sanitizer keeps prefix before orphan close:** When a reasoning tag is found while not inside a thinking block, the implementation appends the text before the tag to `result`; if the tag is a closing tag, it does not reset that accumulated prefix and later appends the post-tag tail. That means `reasoning </think> final` is still reduced to `reasoning  final`, not just `final`.
