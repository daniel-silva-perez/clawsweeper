## Summary

Keep PR #71582 open. Current main does not contain the PR’s substantive exec/system-event heartbeat routing fix: wake calls still only scope by session key, heartbeat wake overrides still only model `target`, and current tests still assert the pre-fix wake payloads. The PR is recent, focused, has concrete reproduction/verification in its body, and the only review finding in the provided discussion is a maintainability nit about duplicated helper logic, not evidence that the work is obsolete.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to resolve or consciously accept the duplicated `scopedExecEventWakeOptions` helper concern, then validate the touched heartbeat wake, runner, cron, node exec-event, and bash notifyOnExit surfaces before merge. If maintainers prefer a smaller patch, extract the shared exec-event wake helper into a routing/session utility and keep the PR’s regression coverage for pinned `to`, `accountId`, `isolatedSession`, raw cron/wake overrides, synthetic node fallback, and canonical `global` routing.
