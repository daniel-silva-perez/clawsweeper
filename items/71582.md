## Summary

Keep PR #71582 open. Current main at 6b6dcafcee9ca86385b7c1ee2442752314d50257 still uses the old exec-event heartbeat wake routing, does not model the PR's pinned-route clears, and still has tests expecting the old wake shape. The provided PR discussion shows useful active work: Greptile identified duplicated helper logic, and the latest PR context says the author moved the shared exec-event wake helper into `src/routing/session-key.ts`. That implementation is not present on main, and no canonical superseding PR in the provided related context replaces it.

## Best Possible Solution

Keep PR #71582 open and continue maintainer review of the latest head. The best path is to preserve the shared `scopedExecEventWakeOptions` direction, verify the pinned-route regression coverage across agent exec, gateway node events, heartbeat wake coalescing, heartbeat runner, cron wake, and outbound target resolution, then run the appropriate changed gates before merge.
