## Summary

Standalone macOS microphone recording is a useful capability, but current OpenClaw core docs/scope emphasize keeping optional device capabilities out of core when they can live as skills/plugins. An existing ClawHub skill already covers microphone recording on macOS, so this issue can close as ClawHub-scoped rather than a core node command request.

## Evidence

- **Core scope favors plugins for optional capability:** VISION.md states that core stays lean and optional capability should usually ship as plugins; new skills should go through ClawHub first, and core additions require a strong product/security/maintainer reason.
  - file: [VISION.md:45](https://github.com/openclaw/openclaw/blob/d6ef1fcf24eea3a5985d653a3baf400e32aa1b75/VISION.md#L45)

- **Current public node surface omits standalone audio recording:** The current README lists node commands as camera snap/clip, screen record, location, notifications, plus macOS-only system.run/system.notify; it does not document a standalone audio or microphone record node command.
  - file: [README.md:132](https://github.com/openclaw/openclaw/blob/d6ef1fcf24eea3a5985d653a3baf400e32aa1b75/README.md#L132)
  - sha: [d6ef1fcf24ee](https://github.com/openclaw/openclaw/commit/d6ef1fcf24eea3a5985d653a3baf400e32aa1b75)
