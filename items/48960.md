## Summary

Keep PR #48960 open. Current main still silently drops malformed or incomplete `SKILL.md` files: the local loader returns `null` on frontmatter parse failure or missing description, and the workspace loader does not log when a candidate `SKILL.md` produces zero skills. Current docs and `openclaw skills list/check` help users verify loaded skills, but they do not provide the requested load-time diagnostics. The PR discussion shows useful intent, but the branch still needs production hardening around path redaction/log volume and logger-test assertions before merge.

## Best Possible Solution

Keep the PR open or replace it with a narrower maintainer patch that adds safe skill-load diagnostics: aggregate/cap parse-failure reporting, redact or compact path details, keep per-source chatter at debug, compute counts from raw source totals, and test through the repository's logging capture path that receives formatted strings.
