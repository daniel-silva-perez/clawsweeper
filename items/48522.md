## Close Comment

Closing from a Codex review because current OpenClaw already addresses the reported Telegram UX problem. Since `v2026.4.21`, preview streaming sends tool-progress updates into the live preview for Telegram/Discord/Slack (`CHANGELOG.md:321`), and the docs now spell out that `channels.telegram.streaming.preview.toolProgress` defaults to `true` and reuses the same preview message for tool/progress activity (`docs/channels/telegram.md:274`, `docs/concepts/streaming.md:172`). The Telegram dispatcher on `main` also wires tool starts, plan updates, approvals, command output, and patch summaries into that preview while suppressing duplicate standalone tool-progress messages (`extensions/telegram/src/bot-message-dispatch.ts:390`, `extensions/telegram/src/bot-message-dispatch.ts:969`, `src/auto-reply/reply/dispatch-from-config.ts:901`). That resolves the original “preview freezes / no feedback during tool calls” report without needing a blanket recommendation to force block streaming on.
