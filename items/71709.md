## Summary

Keep open. Current main still leaves the session-memory slug generator on the embedded runner's shared auth-profile failure policy, so a billing-classified helper failure can still write profile-wide billing disable state. The existing helper-local policy exists and is used elsewhere, but `generateSlugViaLLM` has not adopted it.

## Best Possible Solution

Keep the issue open. Fix `src/hooks/llm-slug-generator.ts` by passing `authProfileFailurePolicy: "local"` to the `runEmbeddedPiAgent` call, then add focused regression coverage in `src/hooks/llm-slug-generator.test.ts` asserting the slug helper uses local failure policy and falls back without writing shared auth-profile failure state. Treat #33962 as related slug-generator performance/model-selection work, not a replacement for this auth-state poisoning fix.
