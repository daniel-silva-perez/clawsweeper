## Summary

Current main already derives the system-prompt tool list from the runtime-effective filtered tool set, so tools removed by `tools.profile`/`tools.allow` are not advertised. The same implementation is present in shipped release `v2026.4.22`.

## Close Comment

Codex review: closing as implemented on `main`. Current OpenClaw resolves the effective tool policy first in `src/agents/pi-tools.ts:363` and applies the policy pipeline before building the tool set shown to the model (`src/agents/pi-tools.ts:674`). The embedded runner then passes that filtered `effectiveTools` array into system-prompt construction in `src/agents/pi-embedded-runner/run/attempt.ts:792` and `src/agents/pi-embedded-runner/run/attempt.ts:954`. The prompt renderer itself only lists the provided tool names and now labels the section `Tool availability (filtered by policy):` in `src/agents/system-prompt.ts:541` and `src/agents/system-prompt.ts:679`. There is also regression coverage for the filtered tool set and prompt behavior in `src/agents/pi-tools-agent-config.test.ts:133`, `src/agents/system-prompt.test.ts:291`, and `src/agents/pi-embedded-runner/run/attempt.test.ts:65`.
