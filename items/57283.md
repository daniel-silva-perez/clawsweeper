## Summary

Keep open. Current `main` at `5dfc1b90e176a9e4d13b9e91ad6affec6b92f213` still awaits auth-profile failure marking before attempting profile rotation in both the prompt-side and assistant-side failover paths, and the marking code still writes through the auth-store file lock. This PR is not implemented on `main`, and the report remains plausible.
