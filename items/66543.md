## Summary

Keep open. Current main does not implement the cron model-ID whitespace repair, but the submitted PR is not safe to close as implemented or merge as-is: it strips all internal whitespace from model and fallback strings, while OpenClaw explicitly supports model aliases and current alias resolution preserves internal spaces.

## Best Possible Solution

Keep the PR open for revision or supersede it with a narrower maintainer fix. The best implementation should repair accidental whitespace only for clear provider/model refs or after alias lookup fails, preserve bare alias strings, cover payload and top-level create/patch paths for both `model` and `fallbacks`, and add regression tests for the OpenRouter corruption case plus spaced primary/fallback aliases.

## Risks / Open Questions

- The linked user bug may still affect current main because no shipped path repairs `custom-1/ minimax/ minimax- m2.5- free` into a valid provider/model ref.
- The PR's blanket whitespace stripping can corrupt valid alias strings such as `My Fast Model`, `GPT Test Z Alias`, or `Kimi K2.5 (NVIDIA)`, including during persisted-job hydration.
- Fallback aliases need the same care as primary model aliases because cron forwards `payload.fallbacks` into the shared fallback resolver.
