## Summary

Keep this PR open. Current main still only trims cron model/fallback strings, so the OpenRouter whitespace corruption reported in the linked bug is not implemented on main. However, the PR is not safe to close as implemented or merge as-is because its blanket internal-whitespace removal can corrupt supported model aliases with spaces, including primary model aliases, fallback aliases, top-level compatibility fields, and persisted cron jobs hydrated through the same normalizer.

## Best Possible Solution

Keep the PR open for revision or supersede it with a narrower maintainer fix. The best fix should repair accidental whitespace for clear provider/model refs, or after alias lookup fails, while preserving bare alias strings. It should cover payload and top-level create/patch paths for both `model` and `fallbacks`, and add regression tests for the OpenRouter corruption case plus spaced primary and fallback aliases.

## Risks / Open Questions

- The linked OpenRouter cron model-ID corruption still appears unresolved on current main because no shipped path strips accidental spaces from provider/model refs before resolution.
- The PR's blanket `.replace(/\s+/g, '')` can break valid spaced aliases such as `MiniMax M2.5`, `DeepSeek R1`, `Qwen3 8B`, or user aliases like `My Fast Model`.
- Because cron store hydration re-runs normalization, the PR could break existing persisted jobs that already use spaced primary or fallback aliases.
