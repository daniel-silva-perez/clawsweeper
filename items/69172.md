## Summary

Current main does not implement the PR’s central request: Google/Gemini web search, image generation, media understanding, and embeddings still use native Google endpoints/config paths and do not expose the proposed Gemini base URL or API-type override behavior. The PR is also not merge-ready because the provided review context shows unrelated changes and multiple P1/P2 regressions, but the underlying product bug/request is real enough that it should not be auto-closed.

## Best Possible Solution

Keep the PR open for maintainer judgment, but ask for a narrower rework or replacement PR: isolate Google override support from unrelated changes, add a shared resolver only where the product wants custom proxy support, preserve existing native-Google behavior and safety tests, and cover web search, embeddings, image generation, and media understanding with focused regression tests.
