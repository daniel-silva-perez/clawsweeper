## Summary

The issue’s core claim no longer holds: the reporter later verified that GitHub Copilot’s Chat Completions endpoint silently ignores `web_search_options`, so there is no reproducible Copilot-hosted Bing search capability for OpenClaw to wire up on current main. Code inspection also shows native `web_search` injection exists only for OpenAI/OpenAI Codex paths, while the GitHub Copilot wrapper only adjusts headers/payload rewriting and does not expose a hosted-search hook.

## Evidence

- **Copilot provider wrapper has no hosted-search injection path:** The GitHub Copilot stream wrapper only adds Copilot headers for Anthropic transport and rewrites payload connection-bound IDs for Responses transport; it does not add `web_search_options` or a native `web_search` tool.

## Close Comment

Closing after a Codex review. The original request depended on GitHub Copilot’s Chat Completions API accepting `web_search_options`, but the follow-up investigation on this issue showed that Copilot currently ignores that parameter and returns no hosted-search annotations/tool output. Current `main` matches that reality: OpenClaw’s native search injection is limited to the OpenAI/OpenAI Codex paths (`extensions/openai/native-web-search.ts:19`, `src/agents/codex-native-web-search.ts:34`), while the Copilot wrapper only handles headers/payload rewriting (`extensions/github-copilot/stream.ts:52`). With no supported Copilot API surface to target, there is not a reproducible OpenClaw change to make here. If GitHub exposes supported Copilot hosted-search request fields later, please open a new issue with the exact request/response contract and a minimal repro.
