## Summary

Issue requests a core Anthropic Claude Agent SDK migration, but current main documents this class of runtime as a plugin/provider-harness concern rather than OpenClaw core. The repo has explicit plugin and agent-harness extension points for native agent daemons, and VISION.md says optional capability should usually ship as plugins while heavy orchestration layers duplicating existing agent/tool infrastructure are out of scope for core.

## Close Comment

Closing as `clawhub` after a Codex review of current `main`. The repo’s current architecture points this kind of integration away from OpenClaw core and toward a plugin/harness implementation instead: `VISION.md:54` says optional capability should usually ship as plugins, `VISION.md:99` says heavy orchestration layers that duplicate existing agent/tool infrastructure are out of scope for core, `docs/plugins/sdk-provider-plugins.md:22` routes native agent daemons to a provider + agent harness instead of core protocol work, and `docs/plugins/sdk-agent-harness.md:20` documents the harness surface for local CLI/native-agent runtimes. Current Anthropic support on `main` is still the API-key and Claude CLI paths in `docs/providers/anthropic.md:10`, and a repo-wide search found no `@anthropic-ai/claude-agent-sdk` integration. If someone wants to pursue this, the maintainable path here is a separate OpenClaw plugin/harness published through the plugin ecosystem / ClawHub, not a core migration.
