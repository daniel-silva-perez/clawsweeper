## Summary

Keep open. Current main has partial Feishu/WebChat/TUI session-sync plumbing, but it still does not implement the issue's requested two-way message mirroring between TUI/WebChat and Feishu.

## Evidence

- **WebChat still routes replies to itself:** docs/web/webchat.md:12-15 describes WebChat as using shared sessions, but explicitly says "Deterministic routing: replies always go back to WebChat," which does not match automatic Feishu mirroring.
  - file: [docs/web/webchat.md:12](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/docs/web/webchat.md#L12)
  - command: `nl -ba docs/web/webchat.md | sed -n '1,35p'`
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **TUI delivery is opt-in and only for assistant replies:** docs/web/tui.md:213-230 documents `--deliver` as "Deliver assistant replies to the provider (default off)" and says to enable it only if you expect messages in a chat channel, so TUI does not auto-sync operator-typed messages outward by default.
  - file: [docs/web/tui.md:213](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/docs/web/tui.md#L213)
  - command: `nl -ba docs/web/tui.md | sed -n '190,230p'`
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **Shared chat UI hard-codes non-delivery on send:** ui/src/ui/controllers/chat.ts:236-242 always calls `chat.send` with `deliver: false`, so WebChat/TUI-originated messages are injected into session history without automatic external-channel delivery.
  - file: [ui/src/ui/controllers/chat.ts:236](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/ui/src/ui/controllers/chat.ts#L236)
  - command: `nl -ba ui/src/ui/controllers/chat.ts | sed -n '232,243p'`
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **Current main only shows partial support:** ui/src/ui/app-gateway.ts:539-541 reloads active chat history on `session.message`, and src/auto-reply/reply/dispatch-from-config.test.ts:996-1017 covers Feishu reply routing when WebChat carries Feishu origin metadata. That supports transcript visibility/reply continuation improvements, but not the full two-way sync requested here.
