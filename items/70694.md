## Summary

Keep open. Current main’s happy path forwards the sessions_spawn task into both the child system prompt and the child agent message, and the reported `in 8 / prompt/cache 40.6k` stats line is not by itself proof that the prompt was absent. But the issue reports recent repeated bad subagent behavior for `runtime:"subagent"`, not ACP, and current main does not contain regression coverage or diagnostics for long task delivery / suspiciously low child input-token accounting. There is not enough concrete implemented-on-main or duplicate evidence to close conservatively.

## Best Possible Solution

Keep #70694 open and narrow it to a maintainer-owned regression/diagnostics task: add a test or harness that spawns a long `runtime:"subagent"` task and captures the child `agent` payload/model input, then add observable diagnostics if the recorded child usage suggests the task was not delivered. If investigation shows only stats-label confusion, fix the stats wording/docs and close with that evidence.
