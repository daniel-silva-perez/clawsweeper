## Summary

Keep open. Current `main` documents and tests skill discovery paths and precedence, but the issue is not fully implemented or disproven: the watcher still uses chokidar glob targets, the security docs still describe ClawHub skills as a critical unresolved risk, and the publish-pipeline complaints appear to involve the external `clawhub` CLI/service rather than a clearly fixed OpenClaw code path.

## Evidence

- **Skill discovery paths are intentionally supported on main:** The docs explicitly list `skills.load.extraDirs`, `~/.openclaw/skills`, `~/.agents/skills`, `<workspace>/.agents/skills`, and `<workspace>/skills` with defined precedence, and the loader code merges those roots in that same order.
  - file: [docs/tools/skills.md:13](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/docs/tools/skills.md#L13)
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **Loader still reads extra dirs and agent/workspace skill roots:** `loadWorkspaceSkillEntries` resolves `skills.load.extraDirs`, plugin skill dirs, managed skills, personal `.agents/skills`, project `.agents/skills`, and workspace `skills/` before merging by precedence.
  - file: [src/agents/skills/workspace.ts:533](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/src/agents/skills/workspace.ts#L533)
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
