## Summary

Keep open. Current main only enforces the read-tool workspace boundary when `tools.fs.workspaceOnly` is explicitly true, while the linked privacy-isolation bug is about default cross-agent host reads. PR #70682 is not merged, is not implemented on main, and also appears to preserve `workspaceOnly=false` host reads while bundling many unrelated changes, so it needs maintainer security review and revision rather than conservative cleanup closure.

## Best Possible Solution

Keep the PR open but require revision before merge: split unrelated fixes out, make the read-tool change narrowly address #70573, decide whether default reads should become workspace/agent-contained or whether a new agent-level path policy is required, and add regression coverage for the reported cross-agent `MEMORY.md` read path plus the intended `tools.fs.workspaceOnly` behavior.
