## Summary

Keep open. Current main still has the groundedCount-based waiting badge and Daily Log Review grouping, while the underlying memory code keeps dailyCount and groundedCount as separate signal sources. The PR is a plausible partial fix for the linked bug, and the newer #72552 may be the better comprehensive path, but it is still open and not merged.

## Best Possible Solution

Keep this PR open for maintainer review, but prefer a single comprehensive fix that aligns all Daily Log Review surfaces with the chosen semantics and adds a focused badge/section regression test. The most concrete path is to review/land #72552 if it fully includes this PR's waiting-badge fix plus the staged section and summary cleanup, or ask this PR author to fold those changes in.
