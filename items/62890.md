## Summary

Do not auto-close #62890 yet. The latest release and current lockfile both contain `@img/colour` through `sharp@0.34.5`, but current `main` has moved image processing out of the root package and into the manifestless `media-understanding-core` public artifact. I could not prove that packaged Windows gateway installs now stage that plugin-owned `sharp` dependency before inline screenshot/image rendering needs it.

## Best Possible Solution

Keep the issue open until maintainers either restore/prove a packaged root/runtime installation path for `sharp` and `@img/colour`, or add a targeted packaged-install regression check that loads `src/media/image-ops`/browser screenshot normalization from a Windows-like packaged gateway without root `sharp` preinstalled. If the manifestless public artifact path is intended, `loadBundledPluginPublicArtifactModuleSync` or the media image-ops loader should explicitly prepare `media-understanding-core` runtime dependencies before importing `image-ops.js`.
