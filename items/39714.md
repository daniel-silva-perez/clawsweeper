## Summary

Keep open. Current main still lacks the PR's central behavior: the sandbox browser bridge and CDP readiness path remain loopback-only, the active plugin SDK/browser runtime bridge contract has no advertised/container-reachable URL, embedded sandbox info still passes only the local bridge URL, and attach-only remote tab creation still follows the persistent Playwright path for remote CDP profiles. The draft is not mergeable as-is because it targets old `src/browser/**` paths and includes unrelated churn, but the problem it is trying to fix remains real.

## Best Possible Solution

Keep this open or replace it with a current-path PR. The best fix is to add a deliberate advertised/container-reachable bridge URL seam through `src/plugin-sdk/browser-bridge.ts` and the bundled browser runtime, preserve the host-local `bridgeUrl` for gateway/browser-tool calls, pass an advertised URL into embedded sandbox info, resolve Dockerized sandbox CDP host/port separately from bridge URL handling, add the attach-only remote tab-open bypass in `extensions/browser/src/browser/server-context.tab-ops.ts`, and cover Dockerized gateway plus non-Docker host cases with focused tests and docs.
