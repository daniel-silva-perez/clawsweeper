## Summary

Keep PR #51964 open. Current main still lacks the PR's central APNG-to-PNG normalization in the MIME sniffing path, and the gateway attachment path still prefers sniffed MIME values for inline images. The linked bug #51881 remains a plausible current behavior on main rather than an already-shipped fix.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land or rebase an equivalent fix in the central `src/media/mime.ts` sniff path with a focused regression test; maintainers may also decide whether header-sourced `image/apng` should be normalized in a follow-up, but that is separate from this PR's buffer-sniffed bug fix.
