## Summary

Keep PR #50206 open. Current main still does not implement `injectAgentHeader` or any equivalent opt-in `X-OpenClaw-Agent` header on outgoing LLM provider requests. The related source issue #50117 was closed as superseded by this PR, and the later User-Agent attribution work is native-provider attribution, not per-agent proxy tracking. The PR still needs maintainer review and likely a fix for the normalized-provider lookup issue called out in review.

## Best Possible Solution

Keep #50206 open or replace it with an explicit maintainer implementation. The likely maintainer path is to add a documented opt-in `models.providers.*.injectAgentHeader` behavior or a broader dynamic-header facility, resolve provider config through the normalized provider lookup path, define header precedence, and add tests for enabled/disabled configs, missing `agentId`, normalized provider aliases, caller-supplied header precedence, and wrapper propagation.
