## Summary

Keep open. Current `main` does not already implement this PR’s Signal refactor, and the remaining differences are behavior-sensitive rather than dead code. Signal outbound now lives in `extensions/signal/src/channel.ts`, where it still hand-wires `sendText`/`sendMedia` and separate formatted-send paths; meanwhile `createDirectTextMediaOutbound` on `main` still hardcodes plain-text chunking and exposes `sendPayload`, so the PR’s proposed migration is not present and would still need maintainer review against current outbound semantics.
