## Summary

Keep open. The provided PR context shows the follow-up commit addressed the bot review concerns by adding `off`/`none`, documenting scope resolution, and clarifying the Discord-wide override wording, so none of the allowed close reasons fit.

## Best Possible Solution

Keep the PR open for maintainer docs review. The best path is to rebase or apply the focused docs update to `docs/channels/discord.md`, preserving the corrected `off`/`none` scope values, the `ackReactionScope` account-to-Discord-to-global resolution order, `removeAckAfterReply`, and clear global/Discord-wide/per-account examples. A small follow-up could also update `docs/gateway/config-agents.md` so its generic scope list includes `off` and `none`.
