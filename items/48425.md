## Summary

Keep open. Current `main` still serializes the full session store with plain `JSON.stringify`, so the reported `sessions.json` bloat is not already fixed. The PR also should not be closed as-is because its global replacer would strip any nested `skillsSnapshot` key, including valid ACP `backendExtras` data.
