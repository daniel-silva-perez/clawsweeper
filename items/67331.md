## Summary

Keep this PR open. Current main has not implemented the standardized `permission_denied` error helper or mutation payload guards, and the PR is still a plausible implementation candidate, but it needs review fixes before landing. The provided PR discussion also shows unresolved review comments about mount-boundary classification and preserving ENOENT semantics, and the same author has an open dependent PR (#67332), so conservative cleanup should not close it as obsolete.

## Best Possible Solution

Keep this PR open and request changes/review. The best path is to fix the review comments by preserving ENOENT/not-found behavior for ordinary missing sandbox reads, mapping `Sandbox path escapes allowed mounts` to `workspace_boundary`, adding regression coverage for those cases, and then re-evaluating this PR together with the dependent #67332.
