## Close Comment

Closing this as `cannot_reproduce` based on a Codex review of current `main`.

- The exact PR patch is **not** on `main`: `extensions/discord/src/channel.ts:429` still has `messaging.targetResolver` without a Discord-specific `resolveTarget` hook.
- Current core target resolution already handles this case: `src/infra/outbound/target-normalization.ts:120` makes the plugin hook optional, and `src/infra/outbound/target-resolver.ts:357` falls back to the normalized explicit target when the hook is absent.
- For explicit Discord DM targets, that means `user:<id>` is still carried through isolated cron delivery: `src/cron/isolated-agent/delivery-target.ts:249` returns `idLikeTarget?.to ?? docked.to`.
- The actual Discord send path does not require cached gateway state for DMs: `extensions/discord/src/send.shared.ts:219` creates a DM channel through `Routes.userChannels()` for `user` recipients before posting the message.
- There is already direct coverage for that send behavior in `extensions/discord/src/send.sends-basic-channel-messages.test.ts:277`, which verifies `sendMessageDiscord("user:123", ...)` opens the DM channel and then sends the message.
