## Summary

Current main already enforces stable marker-based persistence for env-backed provider secrets in models generation, and the PR's proposed switch to resolved plaintext values would regress shipped secret-persistence hardening.

## Close Comment

Closing after Codex review. Current `main` already implements the shipped behavior here: env-backed provider secrets are intentionally normalized and persisted as markers like `OPENAI_API_KEY`, not as resolved plaintext values (`src/agents/models-config.providers.secret-helpers.ts:245`, called from `src/agents/models-config.providers.normalize.ts:77`). The newer source-snapshot path also re-applies marker values from the source config during `models.json` planning (`src/agents/models-config.providers.source-managed.ts:73`), and the regression coverage on `main` explicitly expects that stable marker output across repeated writes (`src/agents/models-config.runtime-source-snapshot.test.ts:258`). This matches the changelog hardening already shipped for models secret persistence (`CHANGELOG.md:3204`, `CHANGELOG.md:2653`). Because this PR would revert that non-plaintext persistence behavior rather than add missing functionality, I’m closing it as already implemented on `main`.
