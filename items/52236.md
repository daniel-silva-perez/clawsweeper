## Summary

Keep #52236 open. Current main still has the exact Mattermost monitor guard this PR tries to remove, and the PR discussion itself says the patch is only a prerequisite, not a complete fix for heartbeat or cron current-thread scheduling. The author abandoned the draft, but the remaining behavior is still repo-owned Mattermost plugin/core routing work and is not clearly implemented or fully superseded on main.

## Best Possible Solution

Do not auto-close #52236. A maintainer should either turn this into a complete Mattermost thread-routing fix or supersede it with a new PR/issue that covers all three pieces together: persist `deliveryContext.threadId` for non-direct Mattermost thread sessions, explicitly handle heartbeat thread preservation when targeting a thread-scoped session, and resolve or document the cron current-thread scheduling constraint. Coordinate with #45082 and #52120 rather than treating this draft patch as shipped.
