## Summary

Keep #71803 open. Current main still arms a CLI no-output watchdog for claude-cli runs and Claude live-session turns, resets it only on stdout/stderr, and has no Monitor/pending-tool expected-idle exemption. The related broader watchdog-cap work in #40982/#41022 remains open, but this issue adds a narrower Monitor-state failure mode that is not implemented on main.

## Best Possible Solution

Keep the issue open and fix the CLI runtime path: either make the no-output watchdog aware of pending Monitor/tool-wait state, or emit a non-semantic watchdog heartbeat while such a tool is pending. A regression test should cover a claude-cli live-session turn that is silent past the watchdog window because it is legitimately waiting on a pending Monitor/background task, without killing the session. Coordinate with #40982/#41022 if the broader cap policy changes, but do not treat those as already shipped.
