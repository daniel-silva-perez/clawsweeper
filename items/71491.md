## Summary

Keep open. Current main contains the prior #70030 Moonshot tool-call-id fix and an optional `thinking.keep` forwarding path, but I did not find a Moonshot-side replay sanitization or payload patch that preserves or synthesizes `reasoning_content` for assistant tool-call turns after LCM compaction or cross-model fallback. The supplied discussion also links Martian-Engineering/lossless-claw#488 as an open LCM-side root cause, so this is still a real fresh bug rather than an obsolete report.

## Best Possible Solution

Keep #71491 open. The best path is a provider-owned Moonshot/Kimi replay fix, likely a `sanitizeReplayHistory` or stream payload patch that preserves existing `reasoning_content` and synthesizes an acceptable placeholder for thinking-enabled assistant tool-call turns when history has been compacted or inherited from another model. Add focused tests mirroring the DeepSeek foreign-history placeholder coverage, and coordinate with Martian-Engineering/lossless-claw#488 so LCM preserves OpenAI-style `reasoning_content` during compaction.
