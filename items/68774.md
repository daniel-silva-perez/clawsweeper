## Summary

Keep this PR open. Current main still has the exact prefix-only dreaming contamination detector the PR describes, and `rehydratePromotionCandidate` still re-reads live daily-note ranges without checking whether the resolved range overlaps an `openclaw:dreaming:*` fence. Later main changes such as default `dreaming.storage.mode: "separate"` and daily-ingestion stripping reduce the common path, but they do not make the PR obsolete because inline/both storage remains supported and old or relocated candidates can still reach the rehydration/apply path.

## Best Possible Solution

Review and land this PR, or land an equivalent patch on current main that broadens the dreaming narrative-lead detector for inline metadata-before-marker shapes, adds a rehydration-time `openclaw:dreaming:*` fence overlap guard, and keeps the proposed regression tests. Consider a separate performance follow-up if repeated per-file rehydration/fence parsing becomes measurable.
