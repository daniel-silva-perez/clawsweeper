## Summary

Keep open. Current main still fixes only the exec-approvals atomic-write half; the transcript half remains actionable in the same core surfaces the report called out, including idempotency scans and manual max-lines compaction.

## Best Possible Solution

Keep this issue open. Treat exec-approvals persistence as already fixed, then land a focused core follow-up that introduces streaming or bounded transcript-scan helpers for idempotency, latest-assistant lookup, duplicate delivery-mirror checks, sessions.get history, usage extraction where applicable, and max-lines compaction, with regression coverage using large transcript fixtures.
