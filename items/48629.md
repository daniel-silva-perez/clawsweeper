## Summary

Keep open. Current `main` has related approval APIs and hook surfaces, but it does not implement the specific unified delegation-invariants layer proposed in #48629: there is no documented Confirm/Stop/Receipts spec, no dedicated `on_takeover`/`on_receipt`-style hooks, and no receipt contract matching the issue’s requested fields. This remains an architecture/docs proposal, not an already-shipped or clearly out-of-scope item.

## Evidence

- **Repo vision does not make this a ClawHub-only item:** The vision doc says core should stay lean and optional capability should usually ship as plugins, but the explicit ClawHub-first rule is for new skills. It also separately warns against heavy orchestration layers as default architecture, which is a design tradeoff to evaluate rather than proof the issue belongs outside OpenClaw.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/VISION.md#L54)
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Current internal hook surface is different from the requested invariant hooks:** Documented internal hooks are command/session/message lifecycle events such as `command:stop`, `agent:bootstrap`, and `message:sent`. They do not define the proposed `on_authorization_*`, `before_action`, `after_action`, `on_takeover`, or `on_receipt` hooks.
  - file: [docs/automation/hooks.md:9](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/docs/automation/hooks.md#L9)
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Current plugin approval contract is narrower than the proposed Confirm primitive:** `before_tool_call` can request approval with `title`, `description`, `severity`, `timeoutMs`, `timeoutBehavior`, and `onResolution`, but it does not expose the issue’s proposed authority fields like `scope`, `limits`, `ttl`, `revocable`, `risk_level`, or a receipt schema.
  - file: [src/plugins/hook-types.ts:336](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/plugins/hook-types.ts#L336)
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Approvals exist today as separate protocol families, not a unified delegation-invariants layer:** The gateway protocol documents `exec.approval.*` and `plugin.approval.*` request/resolve methods and matching lifecycle events. That shows related primitives exist, but they are still separate approval mechanisms rather than the consolidated Confirm/Stop/Receipts contract requested by the issue.
  - file: [docs/gateway/protocol.md:373](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/docs/gateway/protocol.md#L373)

- **Requested hook/spec names are absent on main:** A repo-wide search for `delegation invariants`, `before_action`, `after_action`, `on_takeover`, and `on_receipt` returned no matches, which supports that the issue is not already implemented on current `main`.
