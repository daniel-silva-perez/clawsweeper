## Summary

Keep PR #51303 open. The linked bug report #50887 is plausible, the PR is focused on the implicated child-process streaming path, and current main still decodes stdout/stderr Buffer chunks independently with `chunk.toString()`, which is exactly the UTF-8 boundary hazard described in the PR. No current-main implementation, release note, or narrower superseding item was found in the inspected source/context.

## Best Possible Solution

Keep #51303 open and adapt it rather than close it. The likely product fix belongs in `src/process/supervisor/adapters/child.ts`: use one UTF-8 streaming decoder per stdout/stderr listener, flush on stream end before wait/dispose can drop trailing bytes, and add a regression test in `src/process/supervisor/adapters/child.test.ts` covering split multi-byte Buffer chunks for both streams. Then merge after the changed process-supervisor checks pass.
