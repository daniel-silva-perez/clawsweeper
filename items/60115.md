## Summary

Keep PR #60115 open. Current main still has the Mattermost DM-threading gap the PR describes: resolveMattermostReplyRootId does not accept chat kind, it still falls back to replyToId for every chat type, and the existing tests do not cover direct-message suppression. The PR remains a focused, plausible fix with regression tests and positive review evidence.

## Evidence

- **Current main helper still lacks direct-message guard:** On current main, resolveMattermostReplyRootId accepts only threadRootId and replyToId, then returns threadRootId or falls back to replyToId. There is no kind parameter and no kind === "direct" short-circuit, so a downstream payload replyToId can still become the outbound Mattermost root_id.
