## Summary

Keep PR #49286 open. Current main does not implement the requested Discord system-message coverage: `ChannelNameChange`, `ChannelIconChange`, `Call`, `GuildInviteReminder`, and several other Discord-native message types still are not mapped by `resolveDiscordSystemEvent`, so the linked user-visible bug can still fall through preflight as ordinary message text. The PR remains a focused, relevant fix path with regression coverage, even though linked issue #24108 was later closed stale.

## Best Possible Solution

Keep the PR open or ask for a refreshed equivalent fix: add the missing Discord-native `MessageType` cases in `extensions/discord/src/monitor/system-events.ts`, preserve pass-through for user-authored message types, and land focused mapper plus preflight regression tests for `ChannelNameChange`.
