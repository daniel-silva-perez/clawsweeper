## Summary

Keep this PR open. Current main still does not fall back to reset archives in the gateway transcript read path: `readSessionMessages` and title-field extraction only read existing primary transcript candidates and return empty/null data when the primary file is gone. Reset still archives transcripts as `.reset.<timestamp>`, and the affected `chat.history` / sessions-history surfaces still depend on this helper. The PR remains a focused implementation candidate, with related reset/archive work complementary rather than superseding it.

## Best Possible Solution

Keep this PR open as the focused implementation candidate for reading the latest reset archive when the primary transcript is missing. Maintainers should review/rebase it against current main, run the gateway changed checks, and either land this patch or fold equivalent reset-archive fallback behavior into a broader reset-history recovery change while leaving archive listing and logical-chain metadata to the related follow-ups.
