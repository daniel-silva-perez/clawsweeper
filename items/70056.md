## Summary

Keep this PR open. Current main still has the stop-timeout stale-store behavior that the PR targets, and the PR is paired with open same-author bug #70024, but the current diff needs revision before merge because a timed-out old task can later clobber replacement lifecycle state.

## Best Possible Solution

Keep the PR open and ask for a revision in `src/gateway/server-channels.ts` that makes stop-timeout runtime state honest while guarding stale timed-out lifecycles from mutating runtime, restart, or cleanup state for a replacement task. Add a regression test where a timed-out task eventually settles after a successful restart, then land this PR or an equivalent maintainer patch to resolve #70024.
