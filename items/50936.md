## Summary

Keep #50936 open. Current main still has the bundled iOS app emitting `location.update`, but the gateway does not handle that event, so the PR is not obsolete or implemented on main. The PR discussion also identified real prompt-delivery, privacy, and sanitization concerns; the PR head appears to have follow-up commits for those, so this should remain available for maintainer review rather than be auto-closed.

## Best Possible Solution

Keep the PR open or replace it with an equivalent maintainer-reviewed core patch. The right implementation belongs in core gateway/node handling: add a `location.update` case in `src/gateway/server-node-events.ts`, validate finite coordinates, sanitize node-provided strings, enqueue an untrusted `location:<nodeId>` system event for the canonical session, wake heartbeat with a classified location reason, add explicit heartbeat prompt/provider handling, update node protocol docs, and cover the path with gateway and heartbeat tests.
