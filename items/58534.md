## Summary

Keep open. Current main has partial mitigation for the Control UI polling symptom, but the core sessions.list scale issue is still present: the gateway still walks the full session store, performs per-session subagent work, builds every row, sorts the full result, and only applies limit afterward. The protocol also exposes limit but not cursor/pagination, and related cleanup PRs are still open in the provided timeline.
