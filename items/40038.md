## Summary

Keep open. Current main still uses Ollama's old `/api/embeddings` + `prompt` + `embedding[]` contract in the active bundled Ollama memory embedding provider, and tests still assert that behavior. The linked bug #39983 remains open, and maintainer discussion already closed duplicate PR #40042 in favor of keeping review on #40038. Related PR #63313 is broader and only adds `/api/embed` for batch embedding fallback, so it does not supersede this PR's query-path migration.

## Best Possible Solution

Keep #40038 open as the canonical focused fix for #39983, but require a rebase/port onto the current bundled Ollama plugin path. The implementation should update `extensions/ollama/src/embedding-provider.ts` and its tests from `/api/embeddings` + `prompt` + `embedding` to the current `/api/embed` + `input` + `embeddings[0]` contract, with maintainer judgment on whether to include a fallback for older Ollama behavior.
