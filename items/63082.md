## Summary

Keep PR #63082 open. Current main still has the shell-completion profile path bug this PR is trying to fix: zsh and fish profile paths are hardcoded, bash check/install behavior still differs, doctor hints still print hardcoded source paths, and there is no resolver test coverage for ZDOTDIR or XDG_CONFIG_HOME.

## Best Possible Solution

Keep the PR or linked issue active. The best fix is to add a single shared, tested shell profile resolver in `src/cli/completion-runtime.ts`, use it consistently from `isCompletionInstalled`, `usesSlowDynamicCompletion`, `installCompletion`, update/doctor messaging, and cover `ZDOTDIR`, `XDG_CONFIG_HOME`, PowerShell non-Windows config, and bash `.bashrc`/`.bash_profile` fallback behavior.

## Evidence

- **Current main hardcodes profile resolution:** `getShellProfilePath` reads only `HOME` and returns `~/.zshrc`, `~/.bashrc`, `~/.config/fish/config.fish`, or the default PowerShell path. It does not consult `ZDOTDIR` or `XDG_CONFIG_HOME`.
