## Summary

Current main does not appear to implement the requested Discord daemon-install bridge. Discord setup still supports `DISCORD_BOT_TOKEN` as the preferred env-backed credential, but `buildGatewayInstallPlan()` only merges durable `.env`/config env vars, auth-profile env refs, and generated service env; it does not inspect `channels.discord.token` or channel credential SecretRefs. The related PR #67881 directly targeted this bug but is closed unmerged in the provided context, so this issue should stay open for a real fix or maintainer decision.

## Best Possible Solution

Keep this issue open. The best fix is to revive or reimplement the #67881 approach in current main: during gateway daemon install, detect the default Discord `channels.discord.token` env-backed SecretRef, copy the resolved install-time env value into the managed service environment when present, keep existing dangerous-env filtering, and add a focused regression test in `src/commands/daemon-install-helpers.test.ts`. If maintainers prefer not to persist channel SecretRefs, update onboarding/configure docs and wizard copy to warn that daemon services need the env var supplied via durable `.env`, config env, or service metadata.
