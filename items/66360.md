## Summary

Keep open. Current main still has no per-transcript `.jsonl` size cap or rotation setting: `rotateBytes` is resolved and applied only to `sessions.json`, disk-budget cleanup evicts old/orphaned artifacts or whole session entries rather than rotating an active transcript, and the config schema/docs expose no `transcriptRotateBytes` or `transcriptMaxLines`. The linked PR #66546 is relevant and appears to target this exact bug, but it is still unmerged in the provided context, so the issue should remain open until a fix lands and ships.

## Best Possible Solution

Keep #66360 open and use PR #66546, or a smaller equivalent maintainer patch, as the implementation path: add transcript-specific maintenance config, schema/help/docs, safe hot-path or explicit-cleanup rotation for active transcripts, and regression tests for oversized `.jsonl` rotation/retention. Once merged and released, close #66360 as implemented with the landing SHA and release tag.
