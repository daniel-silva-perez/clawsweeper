## Summary

Keep open. Current main still lacks a bare OpenAI-style `tool_calls` JSON sanitizer on the shared visible-text path and on the Telegram/direct partial preview path implicated by the issue. The linked fix PRs #71954 and #72011 were closed unmerged, and the current source/tests do not show their advertised bare-JSON streaming guard.

## Best Possible Solution

Keep #70797 open and implement a narrow sanitizer for bare OpenAI-style `{"tool_calls":[...]}` assistant text on both final visible-text cleanup and Telegram/direct partial preview delivery. Add regressions for visible prose plus complete and chunked/incomplete `tool_calls` JSON, while preserving ordinary JSON and code examples. The previous #72011 direction is close to the right product boundary but was closed unmerged, so there is no canonical shipped fix yet.

## Risks / Open Questions

- A fix needs to be streaming-aware so incomplete standalone JSON fragments are withheld until they can be classified, without stripping ordinary JSON or fenced/inline code examples.
- Preventing visible leakage does not necessarily recover execution when a provider has already downgraded a structured tool call into assistant text; that can remain a separate provider/runtime boundary follow-up.
