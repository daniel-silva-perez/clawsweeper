## Summary

Keep open. This is a recent bundled gateway/WhatsApp crash or hang report with concrete logs plus follow-up comments on v2026.4.22, so it is not stale or ClawHub-scoped. Current main has partial health-monitor and WhatsApp watchdog hardening, but I did not find source/test evidence that the reported 9-minute mid-session gateway/webchat wedge or the config-reading shutdown symptom is fully fixed.

## Best Possible Solution

Keep this issue open and investigate it as a bundled gateway/WhatsApp reliability bug. The next maintainer step should be to reproduce on current main or the next release with `openclaw health --verbose`, gateway stability bundles, and logs around `config.get`, then either wire WhatsApp transport liveness into the channel snapshot/adjust watchdog thresholds or fix the config-read shutdown path if that proves to be the trigger.
