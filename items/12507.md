## Summary

Keep open. Current `main` still installs ClawHub skills into workspace `skills/`, loads `SKILL.md` files directly, and does not implement signature or publisher-verification checks for skills. The repo’s own threat model still documents malicious skill installation/update poisoning as an open risk and explicitly recommends update signing.
