## Summary

Keep open. Current `main` still has a plausible path to the reported state: successful cron payload delivery can coexist with an overall `error` run status, and that error status is what gets persisted and shown in the cron UI.

## Evidence

- **Fatal status is computed independently of delivery success:** `resolveCronPayloadOutcome` marks a run fatal whenever any error payload exists without a later successful text payload (`hasFatalErrorPayload = hasErrorPayload && !hasSuccessfulPayloadAfterLastError`). In the same helper, successful payloads are still kept for delivery via `pickDeliverablePayloads`, so a run can still have deliverable content before/alongside a fatal error outcome.
  - file: [src/cron/isolated-agent/helpers.ts:113](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/src/cron/isolated-agent/helpers.ts#L113)
  - command: `nl -ba src/cron/isolated-agent/helpers.ts | sed -n '113,203p'`
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **Runner returns `status: error` even when delivery metadata is attached:** `runCronIsolatedAgentTurn` builds the final result with `status: hasFatalErrorPayload ? "error" : "ok"`, then separately attaches `delivered` and `deliveryAttempted`. There is no normalization that flips the run back to `ok` when delivery succeeded.
  - file: [src/cron/isolated-agent/run.ts:822](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/src/cron/isolated-agent/run.ts#L822)
  - command: `nl -ba src/cron/isolated-agent/run.ts | sed -n '822,838p'`
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **Cron state/UI persist and display the raw run status separately from delivery:** `applyJobResult` copies `result.status` directly into `job.state.lastRunStatus` and `job.state.lastStatus`, while delivery is tracked in separate `lastDelivered` / `lastDeliveryStatus` fields. The Control UI renders `lastStatus` as the run status pill and `deliveryStatus` as a separate chip, so `error` + `delivered` remains representable in current main.
  - file: [src/cron/service/timer.ts:417](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/src/cron/service/timer.ts#L417)
