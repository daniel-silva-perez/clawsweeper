## Summary

Keep #70512 open. Current main supports Feishu group-topic session routing and adjacent DM thread/ACP behavior, but ordinary Feishu DM routing still resolves direct conversations to the sender Open ID only, with no Feishu DM `sessionScope`/per-topic config. The related #10242 report is adjacent thread capability work, not a canonical replacement for per-topic DM session isolation.

## Best Possible Solution

Keep this open and implement it in the Feishu bundled plugin, not core: add an opt-in Feishu DM session-scope config with a sender-only default, derive direct peer IDs from sender plus DM topic/root metadata when enabled, update docs/config schema, and add p2p route tests covering root_id, thread_id, fallback behavior, ACP bindings, and unchanged default routing.
