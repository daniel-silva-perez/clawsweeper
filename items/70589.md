## Summary

Keep #70589 open. Current main and latest v2026.4.24 still route the local `openclaw agent --local --json` envelope through `runtime.log()`, while JSON mode routes logs to stderr; the PR remains a focused fix for a real stdout/stderr bug tracked by #62440.

## Best Possible Solution

Keep the PR open and review or land an equivalent patch: import/use `writeRuntimeJson` in `src/agents/command/delivery.ts`, replace the local JSON `runtime.log(JSON.stringify(...))` call, and keep the regression test asserting `runtime.writeJson()` is used for local JSON delivery. After that ships, close #62440 and any obsolete duplicate PRs.
