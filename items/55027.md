## Summary

Keep open. Current main still does not implement true Feishu progressive block delivery: Feishu reply options still force-disable core block streaming, and the dispatcher still drops block payloads unless they can be consumed by the streaming-card path. The open related #51117 tracks the hardcoded disableBlockStreaming part, but it does not fully cover this issue's card-only/drop behavior and requested non-card-animation block delivery.

## Best Possible Solution

Keep #55027 open unless maintainers intentionally consolidate it into a broader Feishu block-streaming issue. The likely fix belongs in the Feishu plugin dispatcher: add a real block-delivery path that honors a documented channels.feishu.blockStreaming/per-account setting, sends each block promptly without relying on card playback when requested, preserves final de-duplication, and adds regression tests for streaming=true/card, streaming=false/card, and plain text render modes. Also align docs/schema/manifest so the advertised Feishu blockStreaming option matches implementation.
