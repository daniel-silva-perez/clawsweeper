## Summary

Keep this issue open. Current main has partial fixes for the original Dreaming noise report: common role/stopword examples are filtered, isolated cron transcripts can be skipped when session-store classification identifies them, and deleted/checkpoint transcript artifacts are skipped. The remaining discussion is still actionable, though: direct cron prompt fallback still preserves paired assistant narration, Dreaming has no operator-facing session/cron exclusion config, REM read-path filtering only blacklists a very small set of legacy tags, and light-dream narrative themes are still built from raw conceptTags. A newer open related issue (#72611) and an open implementation PR (#68876) also show the corpus-ingestion half is still active rather than fully settled on main.

## Best Possible Solution

Keep this issue open until maintainers either land a complete memory-core Dreaming fix or explicitly split and close it in favor of a canonical follow-up such as #72611. The concrete path is to preserve the current stopword and transcript-classification improvements, then add reliable configurable session/cron exclusions, handle unclassified cron narration without reintroducing spoofing risks, normalize/filter legacy conceptTags on REM and light narrative read paths, and add focused regression tests for decimal/template tags and isolated/rotated cron transcript ingestion.
