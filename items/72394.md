## Summary

Current main does not transfer pending base-session system event entries into the heartbeat-created isolated session. The runner peeks base-session events during preflight and can inline/consume those inspected entries, but after creating the `:heartbeat` session it only peeks the active isolated-session queue separately. Existing tests cover adjacent behavior and currently assert that base-session hook/wake events are not treated as active isolated-session events, so this issue remains a valid narrow follow-up rather than something already implemented or obsolete.

## Best Possible Solution

Keep this issue open and implement a narrow heartbeat/system-events seam that transfers or aliases the inspected base-session system event entries across the isolated heartbeat boundary before the model run. Add focused tests for cron, exec, trusted/untrusted flags, delivery context, and exactly-once consumption so future `peekSystemEventEntries(runSessionKey)` consumers see the expected active-session events without replaying base-session entries.
