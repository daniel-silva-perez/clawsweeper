## Summary

Keep #67956 open. Current main has partial CLI JSONL delta parsing and emits assistant agent events for Claude-style stream-json output, but those events feed Gateway/WebSocket chat buffering, not the external channel preview callback path. CLI backend config also has no `streamingMode` or capability metadata like the issue requests, so the central channel-preview problem is not implemented on main.

## Best Possible Solution

Keep #67956 open and implement the missing bridge deliberately: add a documented CLI backend streaming policy/capability surface, decide compatibility defaults for JSONL/text/resume modes, forward eligible CLI assistant deltas into the existing `replyOptions.onPartialReply` channel preview callbacks with debounce/coalescing, preserve final delivery semantics, and add diagnostics/tests that distinguish backend-no-delta from channel-preview-drop.
