## Summary

Keep open. Current `main` still has the exact `@_all` early-return that this PR removes, so the fix is not implemented on `main` and the reported Feishu `requireMention` bug remains plausible.

## Evidence

  - file: [extensions/feishu/src/bot-content.ts:233](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/extensions/feishu/src/bot-content.ts#L233)

- **Mention gating still flows through the buggy helper:** `parseFeishuMessageEvent()` sets `mentionedBot` from `checkBotMentioned(event, botOpenId)`, so any `requireMention` behavior depending on `mentionedBot` still inherits the `@_all` behavior on current `main`.
  - file: [extensions/feishu/src/bot.ts:106](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/extensions/feishu/src/bot.ts#L106)
