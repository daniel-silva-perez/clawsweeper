## Summary

Keep #68079 open. Current main still does not inject `X-Session-Id` for the bundled Z.AI provider; it only composes existing `tool_stream` and Z.AI thinking payload behavior. The related cache-TTL work is shipped and complementary, but it does not solve the sticky-routing header request. The PR remains a plausible provider-local fix, though it should address the review comments about header-safe session IDs and case-insensitive replacement before merge.

## Best Possible Solution

Continue the PR as a provider-local Z.AI fix: compose `X-Session-Id` injection with the current `wrapZaiStreamFn` behavior, validate header-safe session IDs, replace existing `x-session-id` headers case-insensitively, and keep focused ZAI tests plus docs/changelog. No core close action is warranted because the requested behavior is not implemented on main.
