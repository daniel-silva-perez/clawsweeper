## Summary

Keep open. Current `main` still permits `reserveTokensFloor` values equal to the model context window, and the memory-flush/preflight-compaction gate treats the resulting threshold as zero and returns `false`, so the reported failure mode remains plausible on `ebd7f19a3b20001960474ec9ba9cc424e45f7b23`.

## Evidence

- **No config guard against `reserveTokensFloor >= contextWindow`:** The config schema only requires `reserveTokensFloor` to be a nonnegative integer; there is no validation or clamp against the resolved context window.
  - file: [src/config/zod-schema.agent-defaults.ts:175](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/config/zod-schema.agent-defaults.ts#L175)
  - command: `nl -ba src/config/zod-schema.agent-defaults.ts | sed -n '160,182p'`
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Gate returns null when threshold is zero or negative:** `resolveMemoryFlushGateState()` computes `threshold = Math.max(0, contextWindow - reserveTokens - softThreshold)` and immediately returns `null` when `threshold <= 0`. With `contextWindow=200000`, `reserveTokensFloor=200000`, and any positive soft threshold, both `shouldRunMemoryFlush()` and `shouldRunPreflightCompaction()` are disabled.
  - file: [src/auto-reply/reply/memory-flush.ts:52](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/auto-reply/reply/memory-flush.ts#L52)
  - command: `nl -ba src/auto-reply/reply/memory-flush.ts | sed -n '30,60p'`
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Preflight compaction uses the raw unclamped floor:** The preflight path computes `threshold = contextWindowTokens - reserveTokensFloor - softThresholdTokens` from config-derived values and then calls `shouldRunPreflightCompaction(...)`; there is no prior normalization against the context window in this path.
  - file: [src/auto-reply/reply/agent-runner-memory.ts:429](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/auto-reply/reply/agent-runner-memory.ts#L429)

- **Memory-flush path uses the same unclamped threshold input:** The memory-flush path computes `flushThreshold = contextWindowTokens - memoryFlushPlan.reserveTokensFloor - memoryFlushPlan.softThresholdTokens` and passes the same values into `shouldRunMemoryFlush(...)`, so the zero-threshold early return also suppresses flushes there.
  - file: [src/auto-reply/reply/agent-runner-memory.ts:580](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/auto-reply/reply/agent-runner-memory.ts#L580)
