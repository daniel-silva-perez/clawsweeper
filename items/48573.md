## Summary

Keep open. Current `main` has new parent-linking and orphan-reconciliation work, but I did not find strong evidence that it actually reaps ACP/embedded child session rows when a parent agent dies abruptly. The code still treats an existing ACP child session entry as authoritative, so the reported zombie-session-store failure mode is not clearly fixed.

## Evidence

- **ACP spawn creates a child session row but does not register parent-death cleanup:** `spawnAcpDirect()` writes the child session with `spawnedBy: requesterInternalKey` via `sessions.patch`, then starts the child run. In this path I did not find a corresponding parent-exit/orphan-reaper hook.

  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **ACP session deletion is only wired in the spawn-failure cleanup path:** `cleanupFailedAcpSpawn()` calls `sessions.delete`, but that function is for failed spawn cleanup. I did not find an equivalent delete/reap path for the normal case where the parent dies after the child session was created.
  - file: [src/acp/control-plane/spawn.ts:17](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/acp/control-plane/spawn.ts#L17)
