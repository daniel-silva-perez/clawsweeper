## Summary

Keep open. Current `main` still enforces one active run per session and does not contain this PR's fast-lane override, transient immediate-run path, or separate lane-key support.

## Evidence

- **Queue design still guarantees one active run per session:** The current queue docs explicitly say inbound auto-reply runs are serialized and that `runEmbeddedPiAgent` enqueues by session key to guarantee only one active run per session, which is the behavior this PR is trying to relax for short follow-ups.
