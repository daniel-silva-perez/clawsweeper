## Summary

Current `main` does not contain the proposed fix. The refresh state still starts at `0`, and both live snapshot refresh call sites only rebuild when the next snapshot version is newer than the cached one. For an existing session with `skillsSnapshot.version: 0` after a gateway restart, the comparison still resolves false, so the stale-skills report remains plausible. This PR should stay open, though it likely needs a rebase onto the current `refresh-state.ts` layout.

## Evidence

- **Global skills snapshot version still initializes to zero:** `src/agents/skills/refresh-state.ts` still sets `let globalVersion = 0;`, so a cold restart begins with snapshot version `0` rather than a boot-unique value.
  - file: [src/agents/skills/refresh-state.ts:9](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/agents/skills/refresh-state.ts#L9)
  - command: `nl -ba src/agents/skills/refresh-state.ts | sed -n '1,120p'`
- **Refresh predicate does not force rebuild when next version is 0:** `shouldRefreshSnapshotForVersion()` returns `next === 0 ? cached > 0 : cached < next`. If both cached and next are `0`, refresh is false.
  - file: [src/agents/skills/refresh-state.ts:71](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/agents/skills/refresh-state.ts#L71)
  - command: `nl -ba src/agents/skills/refresh-state.ts | sed -n '65,72p'`
- **Primary agent path still depends on that zero version:** `src/agents/agent-command.ts` reads `getSkillsSnapshotVersion(workspaceDir)` and decides `needsSkillsSnapshot` from `shouldRefreshSnapshotForVersion(currentSkillsSnapshot.version, skillsSnapshotVersion)`. With persisted `version: 0` and cold-start `0`, existing sessions do not refresh.
  - file: [src/agents/agent-command.ts:576](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/agents/agent-command.ts#L576)
  - command: `nl -ba src/agents/agent-command.ts | sed -n '574,583p'`
- **Legacy auto-reply path has the same behavior:** `src/auto-reply/reply/session-updates.ts` also computes `snapshotVersion = getSkillsSnapshotVersion(workspaceDir)` and refreshes only when `shouldRefreshSnapshotForVersion(existingSnapshot?.version, snapshotVersion)` is true, so the same stale-after-restart condition remains there too.
  - file: [src/auto-reply/reply/session-updates.ts:145](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/auto-reply/reply/session-updates.ts#L145)
  - command: `nl -ba src/auto-reply/reply/session-updates.ts | sed -n '145,150p'`
- **No regression test for initial snapshot version on current main:** `src/agents/skills/refresh.test.ts` only covers watcher ignore behavior; there is no test asserting that initial snapshot version is `> 0` after module load. A repo-wide search found only mocked uses of `getSkillsSnapshotVersion` / `shouldRefreshSnapshotForVersion` in tests.
  - file: [src/agents/skills/refresh.test.ts:20](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/agents/skills/refresh.test.ts#L20)
