## Summary

Keep open. Current main still only ships the macOS gateway service as a per-user LaunchAgent, and the CLI has no LaunchDaemon/--daemon install mode. Docs explicitly say headless/custom LaunchDaemon is not shipped; existing LaunchDaemon awareness is limited to deep status discovery, not installation or lifecycle management. The related local title matches are adjacent macOS/local-gateway issues, but none supersede this LaunchDaemon request.

## Best Possible Solution

Keep the issue open as a core macOS gateway-service feature request. The likely path is a maintainer-designed LaunchDaemon install mode for `openclaw gateway install` or a clearly documented custom unsupported path, with lifecycle/status/doctor/uninstall tests and macOS app behavior updated together.
