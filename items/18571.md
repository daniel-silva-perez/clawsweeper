## Summary

Current `main` has a pre-delivery outbound plugin hook, but it still does not provide the `sessionKey`, `runId`, or tool-call/session-history context requested by #18571. The request is only partially covered today, so there is not strong evidence to close it.

## Evidence

- **Pre-delivery outbound hook exists and can cancel delivery:** `message_sending` is documented as a terminal pre-delivery hook that can return `{ cancel: true }`, so OpenClaw already has the outbound interception point in principle.
  - file: [docs/plugins/sdk-overview.md:212](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/docs/plugins/sdk-overview.md#L212)
  - command: `nl -ba docs/plugins/sdk-overview.md | sed -n '204,218p'`
  - sha: [a59d1bd46d9f](https://github.com/openclaw/openclaw/commit/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3)
- **Current outbound hook payload lacks session/run identifiers:** `PluginHookMessageSendingEvent` only includes `to`, `content`, `replyToId`, `threadId`, and `metadata`; `PluginHookMessageContext` only includes `channelId`, `accountId`, and `conversationId`. There is no `sessionKey`, `runId`, or tool-history field on the hook surface.
  - file: [src/plugins/hook-message.types.ts:1](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/src/plugins/hook-message.types.ts#L1)
  - command: `nl -ba src/plugins/hook-message.types.ts | sed -n '1,120p'`
  - sha: [a59d1bd46d9f](https://github.com/openclaw/openclaw/commit/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3)
- **Delivery wiring does not pass session/run identifiers into message_sending:** Outbound delivery calls `runMessageSending(...)` with only destination/content/routing metadata and a context of `{ channelId, accountId, conversationId }`. The current implementation does not thread `sessionKey` or `runId` into that hook call.
  - file: [src/infra/outbound/deliver.ts:590](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/src/infra/outbound/deliver.ts#L590)
  - command: `nl -ba src/infra/outbound/deliver.ts | sed -n '584,640p'`
  - sha: [a59d1bd46d9f](https://github.com/openclaw/openclaw/commit/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3)
- **Cancellation happens before send, but without the requested verification context:** `deliverOutboundPayloadsWithQueueCleanup(...)` runs `applyMessageSendingHook(...)` before sending each payload and skips delivery when it returns `cancelled`, which matches the requested timing but not the requested context surface.
  - file: [src/infra/outbound/deliver.ts:845](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/src/infra/outbound/deliver.ts#L845)
  - command: `nl -ba src/infra/outbound/deliver.ts | sed -n '844,888p'`
  - sha: [a59d1bd46d9f](https://github.com/openclaw/openclaw/commit/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3)
- **Other hook surfaces already carry run/session context, highlighting the gap:** `PluginHookAgentContext` includes optional `runId` and `sessionKey`, and `PluginHookLlmInputEvent` includes `historyMessages`, but those fields are not present on `message_sending`. This indicates the requested deterministic verification context is still missing specifically on the outbound hook.
  - file: [src/plugins/hook-types.ts:154](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/src/plugins/hook-types.ts#L154)
  - command: `nl -ba src/plugins/hook-types.ts | sed -n '140,210p'`
  - sha: [a59d1bd46d9f](https://github.com/openclaw/openclaw/commit/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3)
- **Internal hooks docs still expose only post-delivery outbound event:** The internal hooks docs list `message:sent` as the outbound event and describe it as firing after outbound delivery, not as a pre-delivery hook with richer session/run context.
  - file: [docs/automation/hooks.md:47](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/docs/automation/hooks.md#L47)
  - command: `nl -ba docs/automation/hooks.md | sed -n '40,125p'`
  - sha: [a59d1bd46d9f](https://github.com/openclaw/openclaw/commit/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3)

## Risks / Open Questions

- There is a partial workaround surface in `reply_dispatch` that carries `runId` and `sessionKey`, but it does not provide the final outbound payload on the same hook surface, so it is not an equivalent implementation of this issue.
