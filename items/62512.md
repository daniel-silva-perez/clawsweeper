## Summary

Keep open. The request is coherent and recent, and current `main` does not already implement an in-app DOM-grounded guide. The repo is plugin-first, but the documented plugin surface currently covers tools/channels/providers/hooks/gateway routes/methods and only light Control UI metadata, not the kind of first-class UI-driving seam this issue proposes, so there is not strong evidence for a `clawhub` close either.

## Evidence

- **Vision is plugin-first only when the capability fits the plugin API:** `VISION.md` says optional capability should usually ship as plugins, but explicitly says that if a useful feature cannot be built as a plugin yet, maintainers welcome plugin-API extension work instead of one-off core behavior.
