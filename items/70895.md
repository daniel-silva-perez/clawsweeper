## Summary

Keep this PR open. Current main still forces every isolated agent turn into a fresh cron session, so webhook-sourced `/hooks/agent` runs with stable hook session keys cannot reuse a fresh existing session. This PR is also paired with open bug #70894 by the same author and the PR body links it with closing syntax, so the best maintainer path is review/land this PR or a narrow equivalent, not conservative cleanup closure.

## Best Possible Solution

Review and land this PR or a narrow equivalent on main: compute forceNew from true scheduled isolated cron semantics rather than every isolated agent turn, allow hook/webhook-sourced isolated turns with explicit hook provenance or stable hook session keys to reuse fresh existing sessions, and add regression coverage while preserving fresh-per-run behavior for ordinary isolated cron jobs.
