## Summary

Keep PR #70895 open. Current main still contains the behavior the PR targets: hook-dispatched agent runs are executed as isolated cron jobs, and the isolated cron runner still forces a new session whenever `sessionTarget === "isolated"`, which bypasses stable webhook `sessionKey` reuse. The PR is focused, has a direct linked bug report (#70894), and the provided diff adds the missing regression coverage rather than being obsolete or superseded.

## Best Possible Solution

Keep the PR open for maintainer review or merge after normal CI. The best code path is a narrow change in `src/cron/isolated-agent/run.ts` that keeps ordinary isolated cron jobs fresh-per-run while letting hook/webhook-sourced isolated runs call `resolveCronSession` with `forceNew: false`, with a regression test covering stable webhook `sessionKey` reuse.
