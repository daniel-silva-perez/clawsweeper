## Summary

Keep #2597 open. Current main has explicit context-visibility surfaces such as `/status`, `session_status`, UI metadata, and context docs, but the requested passive agent-visible `context=X%` field in the system-prompt Runtime line is still not implemented. The latest provided maintainer comment also explicitly kept the issue open for that reason.

## Best Possible Solution

Keep the issue open until OpenClaw either implements bounded context-usage plumbing from fresh session token accounting into embedded-runner runtime params, `RuntimeInfoInput`, and `buildRuntimeLine()` with focused tests, or a maintainer explicitly decides that `/status`, `session_status`, `/context`, UI warnings, memory flush, context-engine plugins, and community backup/sync workflows are the intended product boundary.
