## Summary

Keep #70518 open. Current main does not implement heartbeat-specific skill allowlisting: the heartbeat config schema/type/generated metadata/docs omit `allowSkills`, and `runHeartbeatOnce` never passes a heartbeat `skillFilter` into `getReplyFromConfig`. The reply and skill-loading layers already have the needed `skillFilter` semantics, including `[]` meaning no skills, so the PR addresses a real current gap rather than obsolete work.

## Best Possible Solution

Keep the PR open and review it as a focused config/runtime fix. The best path is to add `heartbeat.allowSkills` to the schema, exported config types, generated schema/help/labels, docs, and config-honor inventory; thread the resolved heartbeat allowlist into `getReplyFromConfig` as `skillFilter`; and keep regression coverage for non-empty, empty, and unset behavior.
