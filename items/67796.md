## Summary

Keep this PR open. Current main still has the Mattermost message-tool send path resolving media only from `params.media`, so the `filePath`/`path` fallback gap remains. The PR is a focused implementation candidate, but it should address the existing review note about blank `media` short-circuiting fallback and add regression coverage before merge. The broader linked Mattermost issue is not resolved on main because the related thread-context and mediaLocalRoots PRs are also still open/unmerged in the provided context.

## Best Possible Solution

Keep the PR open as a plausible Mattermost message-tool bug fix. The best path is to revise the fallback to normalize each candidate independently so blank `media` falls through to `filePath`/`path`, add handleAction regression tests for both aliases, and coordinate with the still-open linked work for `threadId` and `mediaLocalRoots` rather than treating those parts as already shipped.
