## Summary

Keep open. Current main already ships the original `session_status` runtime-model display behavior, but this PR also carries still-relevant fallback-status discussion. Current main can still collapse selected and active model refs for fallback sessions, and the PR branch's latest review comment says its attempted fallback preservation is still wrong for channel/group-selected models.

## Best Possible Solution

Keep this PR open or replace it with a focused follow-up. The right fix should preserve current main's runtime-model display for non-fallback sessions, avoid synthesizing providers for unknown runtime-only models, and derive fallback selected-model semantics from `fallbackNoticeSelectedModel` or the actual routed selection rather than the global default. Add tests for runtime-only display, active fallback visibility, unknown-provider preservation, and channel/group-selected fallback sessions.
