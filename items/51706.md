## Summary

Keep open. Current main already ships the original runtime-model display fix via #55425, but this PR now also carries unresolved fallback-status work. Current main still feeds the runtime model into the selected-model status path for session_status, which can collapse selected and active fallback refs; the PR branch’s attempted preservation has a review-noted channel/group selected-model bug and should not be merged as-is.

## Best Possible Solution

Keep this PR open or replace it with a focused follow-up. Preserve the shipped #55425 runtime-model display behavior for non-fallback sessions, add a display-only runtime model path that does not overwrite selected-model semantics during active fallback, derive fallback selected refs from fallbackNoticeSelectedModel or the actual routed selection rather than the global default, and add tests for runtime-only display, unknown-provider preservation, active fallback visibility, and channel/group-selected fallback sessions.
