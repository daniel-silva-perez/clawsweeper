## Summary

Current `main` still skips the `sessions_send` A2A announce flow on the synchronous timeout path. The issue is coherent and not implemented: `src/agents/tools/sessions-send-tool.ts` returns `status: "timeout"` before calling `startA2AFlow`, while the existing async A2A helper already supports waiting on a `waitRunId` in the background.
