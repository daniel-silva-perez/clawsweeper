## Summary

Current main and shipped releases already cover the reported Codex OAuth breakage: the OpenAI Codex route was corrected away from the removed/Cloudflare-prone legacy path in v2026.4.20, and the onboarding/auth-store cleanup plus headless device-code path shipped in v2026.4.22. The report’s `api.responses.write` check targets the public OpenAI Responses API, but current OpenClaw documents and tests `openai-codex/*` as the ChatGPT/Codex OAuth PI route and preserves upstream OAuth scopes without local rewriting.
