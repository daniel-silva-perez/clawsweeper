## Summary

PR #48672 is a docs-only request to add third-party plugin discoverability entries, and current `main` explicitly directs that community plugin discovery belongs on ClawHub rather than via docs-only PRs to `docs/plugins/community.md`. This falls outside current OpenClaw core/docs scope.

## Close Comment

Closing after Codex review. This PR only adds third-party plugin discoverability entries to `docs/plugins/community.md`, and current `main` explicitly routes that use case to ClawHub instead of docs-only PRs. The community plugins page now says `docs/plugins/community.md:14-16` “Do not open docs-only PRs just to add your plugin here for discoverability; publish it on ClawHub instead,” and `docs/plugins/community.md:147-151` repeats that docs PRs are only for actual source-doc changes. That matches the project direction in `VISION.md:54-61` and the ClawHub docs in `docs/tools/clawhub.md:10-15`. If these plugins should be discoverable, the right path is publishing and maintaining them through ClawHub rather than extending the source-doc list here.
