## Summary

Keep this issue open. Current main still resolves the Docker sandbox workspace inside the containerized gateway namespace and passes that path directly as the Docker bind source. The docs now warn about Docker-out-of-Docker host-path parity, but the bundled Docker setup/runtime does not translate `/home/node/.openclaw/sandboxes/...` back to the host `OPENCLAW_CONFIG_DIR` path or set a host-visible default sandbox `workspaceRoot`.

## Best Possible Solution

Keep the issue open and fix the Docker gateway sandbox path contract. The likely maintainer path is to translate container-visible sandbox/workspace paths back to the host `OPENCLAW_CONFIG_DIR`/`OPENCLAW_WORKSPACE_DIR` before `docker create`, or update Docker setup to configure a host-visible `agents.defaults.sandbox.workspaceRoot` with an identical gateway-visible parity mount. Add regression coverage so `/home/node/.openclaw/sandboxes/...` is not emitted as a host Docker bind source for Docker Desktop macOS.
