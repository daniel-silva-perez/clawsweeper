## Summary

Keep PR #52522 open. Current main still uses the resolved bootstrap workspace path itself as the boundary for standard and extra bootstrap file reads, so the proposed behavior has not shipped. However, the open review comment identifies a real isolation concern in this PR's DEFAULT_AGENT_WORKSPACE_DIR heuristic, and there is no canonical merged replacement in the provided related context. This needs maintainer review or a revised patch, not conservative auto-close.

## Best Possible Solution

Keep the PR open, but request a revision before merge. The durable fix should make the bootstrap loader boundary explicit: pass or derive the actual configured workspace boundary for the current agent/run, preserve strict rejection for symlinks escaping that boundary, and add regression tests for symlink-to-shared-file within the intended boundary, symlink escape rejection, nested configured workspaces, extra bootstrap files, and external/custom workspace directories.
