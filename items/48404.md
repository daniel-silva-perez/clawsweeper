## Summary

Keep open. I did not find strong evidence that current `main` definitively fixes or obsoletes this ImmortalWrt/OpenWrt startup segfault report. The current CLI bootstrap still executes the same startup-side-effect areas that could plausibly matter on this platform, while the startup changes I did find are framed as latency/RSS optimizations rather than an explicit segfault fix. I also could not run a target-host reproduction from this checkout-only, read-only environment.

## Evidence

- **CLI wrapper still runs compile-cache/bootstrap logic on startup:** `openclaw.mjs` still enables Node compile cache before handing off to the CLI, and only special-cases bare root help after that bootstrap. This remains part of the current startup path that the report says crashes.
