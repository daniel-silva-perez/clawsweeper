## Summary

Keep open. Current main still uses whitespace-only Discord long-line splitting and hard-splits CJK text at the limit when no whitespace exists. The issue has a focused active fix path in PR #71384, but that behavior is not present in main yet.

## Best Possible Solution

Keep the issue open and review or land the focused Discord fix in PR #71384, or implement the same behavior directly in extensions/discord/src/chunk.ts: preserve whitespace priority, add CJK punctuation and safe CJK character-boundary fallbacks, and add Discord chunking regression tests for Chinese, Japanese, Korean, punctuation, mixed-script, and long-token edge cases.
