## Summary

Keep open. Current main still does not implement TUI transcript paging or mouse-wheel scrollback, and PR #51128 contains focused work toward that behavior rather than obsolete code. The PR also still needs maintainer review because the discussion includes post-final review concerns around scroll anchoring during in-place updates and terminal mouse-mode cleanup on non-normal exit paths.

## Best Possible Solution

Keep #51128 open, or supersede it with a maintainer-owned TUI scrollback PR. The best implementation belongs in core TUI: add `ChatLog` viewport/scroll state and render clipping, wire PageUp/PageDown and SGR wheel events only for the transcript area when overlays are inactive, integrate mouse-mode cleanup with existing terminal restore paths, update the TUI docs, and cover the behavior with focused tests.
