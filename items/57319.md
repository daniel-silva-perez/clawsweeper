## Summary

Issue #57319 requests a new Lossless Claw (LCM) capability, but current OpenClaw main treats LCM as a community plugin concern rather than core-maintained behavior. The repo already exposes generic context-engine seams for compaction/assembly, and docs explicitly list Lossless Claw as a third-party plugin to install from npm/ClawHub.

## Close Comment

This looks like Lossless Claw plugin work rather than an OpenClaw core issue.

- Codex review of current `main` found that OpenClaw's project vision keeps optional capability in plugins maintained in their own repos, not in core (`VISION.md:52`).
- The repo docs list **Lossless Claw (LCM)** as a **community plugin** with its own npm package and external repository (`docs/plugins/community.md:64`).
- Current Codex integration already uses generic context-engine hooks for compaction and lifecycle (`extensions/codex/src/app-server/compact.ts:35`, `src/agents/harness/context-engine-lifecycle.ts:62`) instead of shipping LCM-specific indexing or search logic in core.
- The requested Gemini-embedding semantic search over the LCM summary DAG is therefore best tracked in the Lossless Claw plugin/ClawHub surface, not `openclaw/openclaw`.

If you want to pursue this, please open it against the Lossless Claw plugin repository or ship it as a plugin update.
