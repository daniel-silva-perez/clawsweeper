## Summary

Keep open. Current main still has age-only stuck-session diagnostics: there is no diagnostic progress marker, no repeated-warning backoff state, and reply/ACP progress callbacks do not update diagnostic session activity. #72010 is an active replacement for #71969, which was closed unmerged by the too-many-PRs guard, so it is not superseded by a canonical landed/open item.

## Best Possible Solution

Keep #72010 open for maintainer review. The best path is to review or land the remaining diagnostics progress/backoff implementation, with targeted tests for logging, generic dispatch, ACP dispatch, and docs/config baseline updates; if rebase-only cleanup has already landed separately, ask the author to rebase and keep the PR focused on the missing diagnostic behavior.
