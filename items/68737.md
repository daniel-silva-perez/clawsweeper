## Summary

Keep open. Current main still has the reported Mattermost health-check failure behavior: REST health-check errors are logged and polling continues, but the WebSocket is not terminated to force the existing reconnect loop. The PR is a focused implementation candidate, not cleanup-close material.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to add focused Mattermost WebSocket monitor tests for consecutive health-check failures, remove or justify the unsafe default parameter, then either land this narrow recovery fix or intentionally fold it into the broader Mattermost liveness work tracked by #57621.
