## Summary

Keep #64712 open. Current main still emits fallback channel-summary status lines unconditionally, and the PR is not obsolete or implemented. However, the submitted patch needs revision: current Feishu account listing can already return the helper fallback `default` account, so the PR's `accountIds.length > 0` guard is not a reliable "real account" test, and review comments correctly flag status-diagnostic regressions for configured-but-unresolved channels such as Nostr SecretRef configs.

## Best Possible Solution

Keep the PR open and request a revised fix. The best path is to distinguish authored/configured accounts from helper fallback defaults at the shared account-list/status-summary boundary, suppress only truly absent channel configs, preserve diagnostics for configured-but-unresolved accounts, and add targeted tests for Feishu no-account fallback, Feishu named accounts without an authored default, and Nostr SecretRef or partial configs.
