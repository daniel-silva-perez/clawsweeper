## Summary

Keep open. Current `main` still blocks sandbox Docker `network: "host"` unconditionally, exposes no `dangerouslyAllowHostNetwork` config key, and documents/tests only the existing `dangerouslyAllowContainerNamespaceJoin` escape hatch.

## Evidence

- **Host mode is still unconditionally blocked in core validation:** `getBlockedNetworkModeReason()` on current `main` only accepts `allowContainerNamespaceJoin`; it returns `"host"` for any normalized `host` network with no override path.
  - file: [src/agents/sandbox/network-mode.ts:10](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/sandbox/network-mode.ts#L10)

  - file: [src/agents/sandbox/validate-sandbox-security.ts:359](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/sandbox/validate-sandbox-security.ts#L359)
