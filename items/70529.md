## Summary

Keep #70529 open. The issue is recent, has an explicit maintainer/Codex triage comment saying to keep it open as the Desktop-specific variant, and current main still makes `profile="user"` / `existing-session` Chrome MCP attach depend on spawning ambient `npx`. Related #62288 and #63926 cover broader attach brittleness and login handoff UX, but they do not replace this packaged-Desktop failure mode.

## Best Possible Solution

Keep #70529 open as the Desktop-specific browser attach bug. Fix the existing-session Chrome MCP launch path so packaged Desktop does not require ambient `npx` (for example by bundling/resolving the driver explicitly or invoking a bundled/pinned package path), and add a structured diagnostic/fallback that points users to the managed `openclaw` profile when existing-session attach is unavailable. Leave #62288 for broader attach hardening and #63926 for browser login handoff UI.
