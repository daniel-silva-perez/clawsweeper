## Summary

Keep this PR open. Current main has shipped partial fixes for sticky auto-failover and reset cleanup, plus bounded overflow recovery, but it still does not implement the PR's dedicated failoverState model or the /new-vs-/reset context-engine conversation identity split. The PR is still potentially useful, but it is not merge-ready because the existing review comments identify real consistency and legacy-reset problems.

## Best Possible Solution

Keep this PR open, require a rebase onto current main, and narrow it before maintainer review. The failover section should either prove that a dedicated failoverState model is needed beyond the shipped override-source cleanup or be dropped. The LCM section should implement memoryConversationKey consistently across all context-engine lifecycle calls while preserving legacy /reset continuity. The overflow section should keep only behavior that improves current bounded overflow recovery, with logging reviewed for data exposure. Address the P1/P2 review comments before considering merge.
