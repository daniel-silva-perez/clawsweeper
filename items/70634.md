## Summary

Keep #70634 open. Current main still serializes per-session agent work with FIFO queues and has no origin/priority field that lets human-originated channel/webchat messages jump ahead of inter-session `sessions_send` or agent-generated follow-up work. The KinthAI comment adds useful production confirmation and a concrete two-tier design, but that design is not present in current code.

## Best Possible Solution

Implement an explicit queue priority/origin contract: mark human/channel/webchat ingress at the boundary, mark `sessions_send`/agent-generated work as background, carry that through command and follow-up queue entries, drain human-priority items before background while preserving FIFO within each tier, and add regression tests covering session-lane and follow-up-queue starvation with inter-session loops.
