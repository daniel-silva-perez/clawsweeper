## Summary

Keep open. Current `main` still drops the queued inbound message id on the followup delivery path, so `replyToMode: "all"` cannot apply implicit reply threading for followup replies.

## Evidence

- **Queued followup retains the inbound message id:** `followupRun` is still enqueued with `messageId: sessionCtx.MessageSidFull ?? sessionCtx.MessageSid`, so the needed id exists before followup delivery.
  - file: [src/auto-reply/reply/get-reply-run.ts:684](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/auto-reply/reply/get-reply-run.ts#L684)
  - command: `nl -ba src/auto-reply/reply/get-reply-run.ts | sed -n '660,710p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Followup runner does not forward that id:** `resolveFollowupDeliveryPayloads(...)` is called from the followup runner without any `currentMessageId` or `queued.messageId` field.
  - file: [src/auto-reply/reply/followup-runner.ts:428](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/auto-reply/reply/followup-runner.ts#L428)
  - command: `nl -ba src/auto-reply/reply/followup-runner.ts | sed -n '420,520p'`

- **Followup delivery cannot apply implicit threading:** `resolveFollowupDeliveryPayloads` has no `currentMessageId` parameter and calls `applyReplyThreading` with only `payloads`, `replyToMode`, and `replyToChannel`, so implicit `replyToId` injection for `replyToMode: "all"` is impossible on this path.
  - file: [src/auto-reply/reply/followup-delivery.ts:26](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/auto-reply/reply/followup-delivery.ts#L26)

- **Core threading helper still requires currentMessageId:** `applyReplyThreading` only derives the implicit reply target from `currentMessageId`; without it, the helper cannot set `replyToId` for automatic threading.
  - file: [src/auto-reply/reply/reply-payloads-base.ts:86](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/auto-reply/reply/reply-payloads-base.ts#L86)
