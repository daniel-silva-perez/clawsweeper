## Summary

Keep #71656 open. Current main has partial fixes for session-corpus noise and some scan fairness, and deep promotion ranking is weighted/recency-aware, but daily and session corpus ingestion still enter the short-term store with fixed source scores. The remaining request is concrete, recent, and not fully covered by #57307 or #68876.

## Best Possible Solution

Keep the issue open and narrow it into implementation work: add content-type-aware source scoring before daily/session dreaming calls recordShortTermRecalls, extend corpus pre-filtering/deduplication for repeated blocks and low-signal status/test chatter, and add regression coverage proving session-corpus scans continue across files/days under budget pressure. Link the scoring portion to #57307 if maintainers want a canonical scoring-policy thread, while keeping #71656 for the combined corpus filtering and scan-stall specifics.
