## Summary

Current `main` already covers the reported Ollama image-capability problem and the later explicit-provider lookup failure. Ollama discovery now reads `/api/show` capabilities and marks vision models as `text+image`, and the media-tool lookup path now normalizes configured provider/model refs before registry lookup.

## Close Comment

Closing as implemented on `main` after Codex review. Current OpenClaw already detects Ollama vision capability from `/api/show` and marks those models as `text+image` (`extensions/ollama/src/provider-models.ts:98` and `:214`, covered by `extensions/ollama/src/provider-models.test.ts:45` and `:201`). The later explicit-provider `Unknown model` path is also handled now because media-tool lookup normalizes configured provider/model refs before registry lookup (`src/agents/tools/media-tool-shared.ts:418`, covered by `src/agents/tools/media-tool-shared.model-resolution.test.ts:5`). The changelog shows these fixes shipped in `2026.4.7` and `2026.4.14` (`CHANGELOG.md:1089` and `:674`), so this report is already addressed in released OpenClaw versions. If you still see it on a current build, please open a fresh issue with the current release, the exact `models.providers.ollama` config shape, and the relevant Ollama `/api/show` response for the model.
