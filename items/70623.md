## Summary

Keep this issue open. Current main implements and shipped the Telegram phantom fallback fix, but the issue also reports a separate Discord inbound blackout. Maintainer comments explicitly left the issue open for that Discord half, and I did not find source, test, changelog, or history evidence proving the Discord Docker inbound/no-session report is fixed.

## Best Possible Solution

Keep this issue open for the Discord half. Treat Telegram as fixed in v2026.4.24, then collect sanitized Docker Discord startup logs, effective channels.discord config, channels status/probe output, and proof of whether MESSAGE_CREATE reaches DiscordMessageListener before preflight/session creation.
