## Summary

Close as `clawhub`. The request is a useful integration idea, but current main explicitly keeps optional capability in plugins/ClawHub and excludes heavy orchestration layers from core. Main already ships built-in cron/webhook primitives, plus plugin and hook surfaces that are a better fit for a Temporal-backed integration outside core.

## Close Comment

Codex review: this looks like a good integration idea, but it is better scoped as a ClawHub/plugin project rather than OpenClaw core. Current `main` already provides the built-in scheduler and webhook mapping surfaces in `docs/automation/cron-jobs.md:12` and `docs/automation/cron-jobs.md:243`, and the plugin architecture already supports hook-only or non-capability plugins with hooks, services, and routes in `docs/plugins/architecture.md:53` and `docs/plugins/architecture.md:85`, plus the public hook runtime in `src/plugin-sdk/hook-runtime.ts:1`. The repo vision is explicit that optional capability should usually ship as plugins and that core is avoiding heavy orchestration layers that duplicate existing agent/tool infrastructure (`VISION.md:54`, `VISION.md:107`). On that basis, I’m closing this as a better fit for a ClawHub/plugin integration instead of a native core feature.
