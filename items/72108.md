## Summary

Current main still has the plugin-sdk declaration boundary gap that #72108 fixes. The dts config includes all `src/plugin-sdk/**/*.ts` files and only excludes `*.test.ts`, so the dot-named Vitest fixture `src/plugin-sdk/qa-runtime.test-helpers.ts` remains in the declaration input set. The PR is narrow, not superseded, and still actionable.

## Best Possible Solution

Keep #72108 open and merge it, or independently apply the same one-line `src/**/*.test-helpers.ts` exclusion in `tsconfig.plugin-sdk.dts.json`. Then validate with `pnpm build:plugin-sdk:dts` and the normal changed/build gate before landing.
