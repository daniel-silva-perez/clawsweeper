## Summary

Current main still does not provide the proposed native auth-gated `/health/deep` no-inference model-contract endpoint. The gateway probe handler only recognizes `/health`, `/healthz`, `/ready`, and `/readyz`; readiness exposes only readiness/channel state; `/v1/models` lists OpenClaw agent targets rather than backend provider/model/fallbacks; and the existing docs/tests cover the current shallow probes, not this API. The issue is recent, contributor-authored, concrete, and asks for maintainer API/security direction before a PR, so it should remain open.

## Best Possible Solution

Keep this open for maintainer product/API/security direction. The likely path is an additive Gateway probe or authenticated readiness-detail extension backed by the existing configured model/fallback resolver plus bounded routing/error telemetry hooks, with tests for local versus remote auth, unauthorized no-body/no-leak behavior, no-inference operation, redaction, status semantics, and public Gateway health/model-contract docs. If maintainers prefer a plugin route, first document or add the missing SDK/runtime helper and auth contract needed to implement it outside core.
