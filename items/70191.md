## Summary

Keep #70191 open. Current main does not ship a native `/health/deep` model-contract probe: HTTP probes are limited to shallow liveness/readiness, gateway health is channel/session/plugin oriented, and model status is a separate CLI surface rather than a cheap authenticated Gateway JSON endpoint. Plugin routes and hooks can approximate parts of the idea, but the requested native route, `/ready`-style detail auth, and active model/provider/fallback contract still need maintainer product/API judgment.

## Best Possible Solution

Leave the issue open for maintainer design review. If accepted, implement an additive Gateway probe or observability endpoint/RPC that reuses model-resolution and fallback helpers, performs no inference, gates detailed output with the chosen auth policy, redacts sensitive errors, and adds tests for auth, payload shape, status semantics, and route precedence. If maintainers prefer ClawHub, first document or expose any missing SDK helpers for model-contract resolution and readiness-style route auth.
