## Summary

Keep this PR open. Current main and the latest release still show the old `reserveTokensFloor`-only context overflow advice, while the paired bug report #67314 remains open and this PR is its explicit implementation candidate. The related #22133 discussion is useful supporting evidence for the heartbeat model-bleed root cause, but the PR should be revised before landing because review comments identified provider/model attribution gaps and an over-specific heartbeat diagnosis for generic fallback mismatches.

## Best Possible Solution

Keep this PR open and ask for a revision: preserve the original primary model before any live-switch/fallback mutation, pass the effective provider with the effective model when resolving context windows, make the mismatch wording heartbeat-specific only when there is heartbeat evidence or otherwise neutral, and add focused regression tests for the two overflow paths before maintainers consider landing it for #67314.
