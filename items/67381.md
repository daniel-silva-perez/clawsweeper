## Summary

Keep this PR open. Current main and the latest release still show the old `reserveTokensFloor`-only context overflow guidance, while #67314 is an open same-author bug report and this PR is its explicit implementation candidate. The PR is not obsolete or superseded, but it should be revised before merge because the current hint heuristic can still over-diagnose generic small-model fallback/live-switch mismatches as heartbeat bleed and it lacks focused regression tests for the two overflow paths.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #67314. The best path is to revise the hint so heartbeat-specific wording is used only with heartbeat evidence or is phrased neutrally for generic model mismatches, then add focused tests for both overflow return paths, including small-window model mismatch and cross-provider context-window lookup, before maintainers consider landing it.
