## Summary

Keep open. Current main has some adjacent Control UI primitives, but not the requested split chat + dedicated live exec/log panel. The chat already supports a generic right-side sidebar and inline live tool cards, yet the sidebar only renders markdown/canvas tool details and there is no exec-specific panel with streamed stdout/stderr, run controls, or historical run views.

## Evidence

- **Split UI exists, but only as a generic sidebar:** The chat view uses `chat-split-container` and opens a `chat-sidebar`, but the sidebar content is routed through `renderMarkdownSidebar(...)` rather than any exec/log-specific component.
