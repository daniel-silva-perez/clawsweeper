## Summary

Keep #72130 open. Current main bounds OpenClaw's own compaction checkpoint transcript snapshots and has optional session-directory disk-budget cleanup, but I could not verify that current main owns or bounds the reported Hermes/git-backed checkpoint store with SQLite/JSON blobs. The report is fresh, concrete, and high-impact enough for maintainer triage rather than auto-close.

## Best Possible Solution

Keep the issue open and ask maintainers/reporter to identify the exact checkpoint directory and owning component. If it is OpenClaw-owned, add bounded retention/size enforcement or disable server-default checkpointing in that owner path; if it is a third-party Hermes/OpenProse-style plugin store, move tracking to that plugin once the boundary is confirmed.
