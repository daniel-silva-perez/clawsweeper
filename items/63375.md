## Summary

Do not auto-close PR #63375. Current main already exposes and documents `api.logger`, and `openclaw config schema` now builds from manifest metadata rather than activating plugin runtime registration, but the specific developer-facing best-practice note proposed by the PR is not present on the main plugin-building guide. The PR likely needs maintainer review or revision rather than automated closure because its direction is plausible but the submitted wording appears partly stale/inaccurate.

## Best Possible Solution

Keep the PR open for maintainer review or request changes. A good version would add a short, accurate note to `docs/plugins/building-plugins.md` or the SDK docs that plugin code should prefer `api.logger` for diagnostic logs and avoid writing directly to stdout during registration, CLI metadata, discovery, and other machine-readable CLI paths; it should remove the optional-logger claim and avoid presenting `openclaw config schema` as a current full-register corruption path unless a fresh reproduction proves that behavior still exists.
