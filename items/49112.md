## Summary

Keep open. Current main still lacks `openclaw doctor` coverage for gateway lock files, so the PR is not obsolete or implemented. The branch should be reworked before merge because current main now has richer gateway-lock owner detection than the PR patch described in the review context, and the PR’s age-only stale logic would be unsafe for live locks.

## Best Possible Solution

Keep the PR open, but rebase and rework it around current main’s gateway-lock semantics. Add a shared inspection helper that reuses or mirrors `resolveGatewayOwnerStatus`, reports gateway lock state through a new doctor health contribution, removes only locks that the acquisition path would safely reclaim, and adds regression tests for live aged locks, dead/stale locks with and without `--fix`, PID reuse, and port-free stale recovery. Then update the gateway/doctor docs to match the actual lock model.
