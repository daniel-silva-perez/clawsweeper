## Summary

Keep #72206 open. The issue is fresh, has no protected maintainer author/label, and current main still contains the claude-cli live-session path that can surface the reported generic `FailoverError`/`reason=unknown` on a fresh live process exit while reuse can succeed. The supplied related context does not supersede it: #71793 was the large-stdout-buffer issue and #71803 is an open watchdog/idle-timeout issue, while #72206 reports immediate fresh-live-session startup failure.

## Best Possible Solution

Keep the issue open and investigate the fresh Claude stdio startup path specifically. A good fix would add a regression test that simulates a newly spawned live process exiting before/just after the first turn with empty or early stderr, preserve startup stderr/exit details through `handleClaudeExit`, and verify `openclaw system event --mode now --expect-final` can start a fresh claude-cli live session without exhausting fallback models. The most likely code surfaces are `src/agents/cli-runner/claude-live-session.ts`, `src/agents/cli-runner/execute.ts`, and the bundled `extensions/anthropic` CLI backend defaults.
