## Close Comment

Closing this as already implemented on current OpenClaw.

- Codex review found the shared public `message` agent tool is registered in `src/agents/openclaw-tools.ts:198` and exposed as tool `message` in `src/agents/tools/message-tool.ts:695`.

- User-facing docs already document outbound attachment delivery with `MEDIA:<path-or-url>` in `docs/start/openclaw.md:187`.
- The same implementation/docs are present in release `v2026.4.23` (`9b18ae877018a34933e27bb7b2986cb97e9602aa`).
