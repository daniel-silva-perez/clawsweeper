## Summary

Keep PR #72212 open. Current main does not implement the PR’s central continuity diagnostics and restore guardrail work, and the related title-search items cover different backup/restore or queue-diagnostics scopes rather than superseding this PR.

## Best Possible Solution

Keep the PR open for maintainer judgment. The best path is to review or split the work by boundary: choose whether continuity diagnostics should use the existing diagnostic-events system or a new helper, then separately evaluate checkpoint boundary metadata/protocol/docs, restore next-turn freshening, live outbound target re-resolution, approval mismatch diagnostics, subagent terminal-vs-announce state, heartbeat orphan diagnostics, and ACP idle-handle eviction diagnostics.
