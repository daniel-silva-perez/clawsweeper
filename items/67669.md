## Summary

Keep this PR open. Current main does not implement `openclaw models custom-test`, and the existing `models status --probe` path only covers provider/auth probing, not the PR's proposed all-configured-custom-model verification surface. The PR is recent, focused, and potentially useful, but it needs maintainer review and likely a redesign around the existing runtime probe path before landing.

## Best Possible Solution

Keep this PR open for maintainer review. If the feature is accepted, reshape it around the existing `models status --probe` runtime probing infrastructure so custom provider verification uses normal model transport semantics, covers per-model APIs and all configured custom models intentionally, preserves machine-readable JSON and exit codes, and avoids leaking provider URLs or credentials.
