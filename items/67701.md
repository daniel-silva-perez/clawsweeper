## Summary

Keep this PR open. Current main has partial mitigations through `promptTokens`/`lastCallUsage` and preserves an existing `totalTokens` when usage is absent, but it still does not persist `compactResult.result.tokensAfter` from overflow auto-compaction when provider usage is unavailable and the session has no prior token total. The branch needs cleanup before merge, but the core bug fix is not implemented on main.

## Best Possible Solution

Keep the PR open, but ask for a focused rebase or maintainer cherry-pick: thread overflow/timeout auto-compaction `tokensAfter` into current session accounting, use it when usage and prompt snapshots are unavailable, preserve the existing `promptTokens`/`lastCallUsage` behavior, and add regression coverage for a session with no prior `totalTokens`. Split unrelated ACP, cron delivery, and Telegram changes into separate PRs.
