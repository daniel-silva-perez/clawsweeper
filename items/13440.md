## Summary

Current `main` does not implement pluggable session-store backends. The session config still treats `session.store` as a filesystem path string, and the runtime still loads, locks, and writes `sessions.json` directly on disk. This issue is still a real enhancement request, not something safely closable as implemented, unreproducible, incoherent, stale, or ClawHub-scoped.

## Evidence

- **Config surface is still path-only:** `SessionSchema` defines `session.store` as `z.string().optional()`. There is no object/driver selector such as `{ driver, path }`.
