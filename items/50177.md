## Summary

Keep PR #50177 open. Current main appears to have already solved the original overflow-clipping symptom by removing `overflow: hidden` from `.agent-chat__input`, but the PR also fixes keyboard navigation not scrolling the active slash-menu item into view. That keyboard-scroll behavior is still absent on main, and the menu can legitimately become scrollable with large runtime command catalogs or arg option lists.

## Best Possible Solution

Keep the PR open, but ask for a current-main rebase and likely narrow the patch to the remaining behavior: after slash-menu ArrowUp/ArrowDown changes, scroll the active item into view with a query scoped to the current chat/menu container. Preserve the existing unclipped composer behavior, avoid unnecessary wrapper churn unless a current clipping reproduction still exists, and add a focused UI test or explicit browser verification for command mode and argument-picker mode.
