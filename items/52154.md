## Summary

Keep open. Current main at 76cf013df50742a2430aae3764f91c7a87ee5a30 still does not implement `commands.shellProfile`: the config schema/types do not expose the setting, exec still calls `getShellConfig()` with no profile parameter, and docs only describe default shell selection plus PATH handling. The PR is a real core exec feature request, not a ClawHub/plugin item or an implemented/superseded cleanup candidate.

## Best Possible Solution

Keep #52154 or a canonical follow-up open until maintainers decide the product contract and implement shell profile loading consistently across direct exec, PTY supervisor launches, chat `/bash`, gateway approval follow-up runs, and documented config/schema/types/tests. The safest implementation should explicitly exclude or constrain profile loading where it would bypass allowlist-enforced commands.
