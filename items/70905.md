## Summary

Keep #70905 open. Current main still has `agentId` available in outbound/session plumbing, but the public channel outbound adapter context does not expose it and `deliverOutboundPayloads` does not copy it into the adapter context used by `sendText`, `sendMedia`, `sendPayload`, or formatted/presentation sends. This is a concrete plugin SDK/channel seam gap, not a ClawHub-only plugin request.

## Best Possible Solution

Keep the issue open and implement an additive core SDK change: add optional `agentId?: string` to `ChannelOutboundContext`, propagate the effective delivery agent id from `params.session?.agentId ?? params.mirror?.agentId` through `ChannelHandlerParams`, `createChannelOutboundContextBase`, and every adapter context path, then cover `sendText`, `sendMedia`, `sendPayload`/formatted sends with direct-delivery regression tests and plugin docs.
