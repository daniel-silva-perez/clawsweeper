## Summary

Current main does not implement the requested strict or loud handling for auth-store credentials whose `auth.profiles` declaration was removed. The Pi path can still resolve and inject credentials from the per-agent auth profile store, and related issues do not fully supersede this narrower removed-declaration/Pi-orphan failure mode.

## Best Possible Solution

Keep the issue open and fix the auth boundary explicitly: make Pi/runtime auth resolution ignore stored profile ids that are not declared in `openclaw.json` for the selected provider, or emit a high-visibility warning and add `openclaw doctor` detection/remediation for undeclared stored credentials. Cover the Pi candidate-selection path and any `auth.order` or per-agent store order drift with a focused regression test, while coordinating with #43945 and the closed #40363 history rather than treating them as replacements.
