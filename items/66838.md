## Summary

Keep open. Current main still uses the old default cron heartbeat wrapper that tells the model to relay the reminder to the user, and the colocated tests still assert that behavior. The PR is a focused implementation candidate for a real main-session cron/heartbeat surface, with prior review feedback reflected in the supplied PR context rather than superseded by current main.

## Best Possible Solution

Keep this PR open for maintainer review as the current focused implementation candidate. The likely path is to update the default branch of `src/infra/heartbeat-events-filter.ts` to a neutral execution wrapper, keep the internal-only delivery path intact, preserve heartbeat ack normalization behavior, and land with the focused heartbeat prompt and ghost-reminder regression tests once CI is green.
