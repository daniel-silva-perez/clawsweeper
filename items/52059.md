## Summary

Keep open. Current main does not implement `gateway.auth.scopes`, and the existing gateway behavior still clears scopes for device-less shared token/password operator sessions. The PR discussion is useful and shows the author addressed review feedback on the branch, but the requested core gateway capability remains absent on main and is not superseded by a canonical item in the provided context.

## Best Possible Solution

Keep the PR open for maintainer judgment. If the product direction still wants device-less token/password auto-pair approval, rebase the change onto current main, add schema/types/help/docs, and cover token/password positive cases plus trusted-proxy/non-configured negative cases. If maintainers prefer the current device-token/bootstrap model, close manually with that security rationale rather than as an automated cleanup.
