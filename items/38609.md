## Summary

Current main implements the central Feishu card-action fixes from PR #38609: Schema 2.0 `open_chat_id` callbacks are normalized, `open_message_id` is used as the reply target without reusing it as the synthetic inbound message id, and chat type is resolved through the Feishu chat API with a `p2p` fallback. The remaining form-field pass-through work is separately tracked by narrower Feishu card-action items such as #42908/#43202, so this PR is now obsolete as a mainline cleanup item.

## Best Possible Solution

Close PR #38609 as implemented/superseded by current main's safer Feishu card-action implementation. Keep the shipped token-based synthetic message id design, use the real `open_message_id` only as the Feishu reply target, and let the remaining `form_value`/input pass-through work continue in the narrower Feishu card-action follow-ups (#42908/#43202).
