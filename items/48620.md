## Summary

Current `main` does not implement a way to view or manage approved paired chat contacts and their DM/group permissions from the CLI. The existing pairing surface only lists pending requests, and the docs still point approved state to the on-disk `*-allowFrom.json` files.

## Evidence

- **Pairing CLI only exposes pending requests and approve:** `pairing list` is described and implemented as pending-request inspection only; there is no command for listing approved contacts or editing their permissions.

- **Pairing docs still say list= pending:** The CLI reference says `openclaw pairing list` lists pending pairing requests for one channel, with no approved-contact management surface documented.
  - file: [docs/cli/pairing.md:28](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/docs/cli/pairing.md#L28)
