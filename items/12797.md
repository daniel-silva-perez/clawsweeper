## Summary

Current `main` already coalesces forwarded Telegram text+media bursts into one inbound turn, with shipped changelog coverage and a regression test for the exact reported flow.

- This fix is targeted to forwarded Telegram bursts. Broader batching requests such as arbitrary same-user burst concatenation or long non-forward split messages would need separate evidence/issues if still desired.

## Close Comment

Closing as implemented on `main` after Codex review. OpenClaw now has Telegram forward-burst coalescing in `extensions/telegram/src/bot-handlers.runtime.ts:213`, where forwarded entries get a dedicated debounce lane, media-only forwarded updates are allowed into the debounce window, and flushed entries are combined into a single text+media turn before dispatch. There is also a regression test for the reported forwarded text + attachment flow in `extensions/telegram/src/bot.media.downloads-media-file-path-no-file-download.e2e.test.ts:421`, which verifies only one processing turn is emitted and that both the text and media are present. This shipped in `v2026.4.22`; the release changelog records it at `CHANGELOG.md:4178` in commit `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`. If you are still seeing a different Telegram split-message case beyond forwarded bursts, please open a new issue with exact reproduction steps and channel config.
