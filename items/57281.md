## Summary

Keep open. Current `main` still performs auth-profile cooldown marking before profile rotation on the rate-limit failover paths, and that cooldown write still waits on the auth-store file lock. The reported contention window is therefore not clearly fixed on `main`.

## Evidence

- **Assistant failover still blocks on cooldown write before rotation:** `handleAssistantFailover` awaits `maybeMarkAuthProfileFailure(...)` and only then awaits `advanceAuthProfile()`. For non-timeout failures, including `rate_limit`, this preserves the ordering described in the issue.
