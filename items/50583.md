## Summary

Current `main` does not show an implemented Composer 2 integration. OpenClaw has broad model-provider support and some dynamic-model hooks, but there is no bundled Cursor/Composer provider, no Composer 2 model entry, and no model-add path for it. This remains a live feature request, not conservative cleanup.

## Evidence

- **Supported provider docs omit Cursor/Composer:** The current provider quickstart lists bundled/supported model providers such as Anthropic, Moonshot, OpenAI, OpenRouter, xAI, and Z.AI, but no Cursor or Composer provider/model surface.
  - file: [docs/providers/models.md:25](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/docs/providers/models.md#L25)
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Dynamic model support exists, but only for OpenRouter refs here:** The OpenRouter plugin resolves arbitrary OpenRouter model ids through `resolveDynamicModel`, which shows OpenClaw's generic forward-compat path; it is not a Composer-specific implementation.
  - file: [extensions/openrouter/index.ts:45](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/extensions/openrouter/index.ts#L45)
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Chat model-add flow has no Composer/Cursor adapter:** `/models add` currently has built-in adapters only for `openai-codex`, `ollama`, and `lmstudio`. There is no adapter for a Cursor or Composer provider.
  - file: [src/auto-reply/reply/models-add.ts:244](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/auto-reply/reply/models-add.ts#L244)
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Repo search found no Composer 2 implementation:** `rg -n "Composer 2|composer 2|Composer2|composer2|Composer" .` found only unrelated UI `Composer` strings and no model/provider implementation for Composer 2.
  - command: `rg -n "Composer 2|composer 2|Composer2|composer2|Composer" .`
