## Summary

Keep open. Current `main` at `4013c658537ee2051c44186f197b0710e84c9049` does not implement `tools.exec.allowedHosts` or per-host `exec-approvals.json` allowlists, so this PR is not obsolete or already shipped.

## Evidence

- **Exec config schema still has no `allowedHosts`:** `ExecToolConfig` includes `host`, `security`, `ask`, and `node`, but no `allowedHosts` field.
  - file: [src/config/types.tools.ts:242](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/config/types.tools.ts#L242)
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Runtime config validation has no `allowedHosts` support:** `ToolExecBaseShape` validates `host`, `security`, `ask`, `node`, etc., with no `allowedHosts` property or refinement for cross-host rules.
  - file: [src/config/zod-schema.agent-runtime.ts:456](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/config/zod-schema.agent-runtime.ts#L456)

  - file: [src/agents/bash-tools.exec-runtime.ts:196](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/agents/bash-tools.exec-runtime.ts#L196)
