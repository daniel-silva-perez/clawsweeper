## Summary

The reported hardcoded Google OAuth secret issue is no longer present on current `main`, and the latest shipped tag already uses an externalized Google OAuth flow instead of embedding Google OAuth client secrets in OpenClaw source. The specific bundled plugins named in the report are also no longer shipped.

## Risks / Open Questions

- The current `google-gemini-cli` integration still reuses Gemini CLI OAuth credentials by reading the locally installed Gemini CLI package. If maintainers want to revisit that integration design, it should be tracked separately from this specific hardcoded-secret report.

## Close Comment

Closing after Codex review. The specific hardcoded-secret paths from this report are not present in current OpenClaw, and the shipped Google OAuth flow now resolves credentials from environment variables or from the locally installed Gemini CLI package instead of embedding Google OAuth client secrets in OpenClaw source (`extensions/google/oauth.credentials.ts`, `docs/providers/google.md`). The old bundled plugins named here are also retired legacy IDs (`src/config/validation.ts:43`). The latest shipped tag, `v2026.4.22`, already contains this externalized implementation at commit `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`, so this specific issue is no longer actionable on current OpenClaw. If you still see an actual hardcoded Google OAuth secret in a released OpenClaw artifact, please open a new issue with the exact shipped tag, file path, and artifact contents.
