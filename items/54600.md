## Summary

Keep open. Current `main` does not include this fix: isolated cron execution still forwards `job.payload.message` directly, while cron normalization only trims string messages and leaves malformed object-valued `payload.message` fields intact. The PR addresses a real defensive gap for persisted bad data and is not implemented on `main`.
