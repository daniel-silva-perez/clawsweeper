## Summary

Keep open. Current main has friendlier restart copy and some interrupted-session recovery, but it still rejects new inbound command enqueues during the gateway drain window, so the requested automatic retry/replay of unprocessed messages is not implemented.

## Best Possible Solution

Keep this issue open and implement the remaining core gateway behavior: allow drain-window inbound tasks to enter a preserved FIFO lane without starting until `resetAllLanes()`, or add a durable per-channel pending inbound queue if full stop/start replay is in scope. Keep the friendlier restart message as fallback and add regression tests for multiple messages arriving during drain, order preservation, SIGUSR1 replay, and durable restart behavior if implemented.
