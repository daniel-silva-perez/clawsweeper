## Summary

Keep #71889 open. Current main at 6b6dcaf still seeds the interactive `agents add` workspace prompt from `resolveAgentWorkspaceDir()`, and that resolver still appends non-default agent ids under `agents.defaults.workspace`. Since setup/onboarding store the main workspace in `agents.defaults.workspace`, the reported default `~/.openclaw/workspace/<id>` is still the observable current-main behavior, while docs still describe peer-level `~/.openclaw/workspace-<agentId>` defaults. The two linked fix PRs (#71899 and #71941) are open and useful, but neither is shipped on current main, so the issue should remain open as the canonical bug report.

## Best Possible Solution

Keep #71889 open and land a narrow fix that makes the interactive `agents add` default peer-level when `agents.defaults.workspace` is the setup/onboarding main workspace, while preserving explicit per-agent workspaces and intentionally custom shared workspace bases. Add regression coverage for the resolver/helper and the wizard prompt default; docs already describe the desired peer-level behavior.
