## Summary

Keep open. Current main still lacks Feishu normal-reply `message_sent` and internal `message:sent` hook emission, while this PR remains an open, focused implementation candidate for the reported Feishu regression.

## Best Possible Solution

Keep this PR open and review/land it, or land an equivalent Feishu-owned patch. The right fix belongs in `extensions/feishu/src/reply-dispatcher.ts` and `extensions/feishu/src/bot.ts`: emit canonical plugin `message_sent` and internal `message:sent` for normal replies, pass routed session and group mirror metadata, preserve streaming-card `messageId`, avoid false failure hooks when only companion media fails, and keep the added Feishu regression tests.
