## Summary

Keep PR #72138 open. Current main still sends Feishu normal conversation replies through the custom Feishu reply dispatcher without canonical `message_sent` or internal `message:sent` emission, while the generic outbound delivery path already emits those hooks. The PR remains a focused fix for open bug #72125, with review comments to resolve before merge rather than a conservative-cleanup close reason.

## Best Possible Solution

Keep the PR open or land an equivalent Feishu-owned fix. The maintainable path is to emit canonical plugin `message_sent` and internal `message:sent` from `extensions/feishu/src/reply-dispatcher.ts`, pass routed session key and group mirror metadata from `extensions/feishu/src/bot.ts`, include streaming-card message identity where available, avoid false failure hooks when streaming text was delivered but companion media failed, and cover plain text, card, media, streaming finalization, and failure cases in Feishu tests.
