## Summary

Keep #69527 open. Current main has improved dynamic OpenRouter model resolution, but the issue is not cleanly obsolete: `models status` can still render the configured `openrouter/auto` default as `openrouter/openrouter/auto`, and `infer model run` still returns `attempts: []` unconditionally, so a fallback to OpenAI would remain invisible in the exact surface the reporter used. The related PR #69580 was closed unmerged and does not supersede the issue.

## Best Possible Solution

Keep the issue open and fix the observable workflow: make `openclaw infer model run` surface model fallback attempts when the primary fails, add a regression test with `agents.defaults.model.primary = "openrouter/auto"` and an OpenAI fallback proving the primary is attempted first, and clean up the OpenRouter status/model-id display so `openrouter/auto` is not shown as `openrouter/openrouter/auto`. Coordinate with the related OpenRouter id mismatch tracked in #62655 rather than relying on the closed, unmerged #69580.
