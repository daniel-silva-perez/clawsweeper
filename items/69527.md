## Summary

Keep open. Current main still drops model-fallback details from `openclaw infer model run --json`: both local and gateway envelopes hard-code `attempts: []` even though `runWithModelFallback` collects failed candidate attempts. The OpenRouter `openrouter/auto` path remains documented and provider-owned, and related PR #69580 was closed unmerged, so there is not enough shipped/current evidence to close this as implemented or cannot-reproduce.

## Best Possible Solution

Keep this issue open. The best maintainer path is a provider-owned regression fix: verify `openrouter/auto` and `openrouter/@preset/...` against current OpenRouter behavior, add targeted OpenRouter model-resolution tests, and plumb `runWithModelFallback` attempt details through `agentCommand`, gateway responses, and `openclaw infer model run --json` for both local and gateway transports.
