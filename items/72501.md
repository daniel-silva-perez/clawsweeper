## Summary

Keep this PR open. Current main still has Telegram poll sending and `poll_answer` in fallback update types, but it does not route `poll_answer` updates into sessions or persist a poll-id registry. The PR is a relevant implementation candidate, not obsolete, although it needs maintainer review for authorization parity and a rebase/update before merge.

## Best Possible Solution

Keep this PR open as the active implementation candidate for Telegram public-poll vote routing. The best path is to rebase it onto current main, preserve the restored CODEOWNERS state, add or verify authorization checks for `poll_answer` events, then run the Telegram changed gate/relevant tests before maintainer review. The older merged poll action PR shipped outbound poll creation but did not complete this persistent answer-routing behavior.
