## Summary

Current `main` has `openclaw models status --probe`, but that is an auth probe, not a fallback-chain test. There is still no `/models test-fallback` or `openclaw models test-fallback` entry point, and the probe path only selects the first candidate per provider instead of intentionally skipping the primary and walking the configured fallback chain.

## Evidence

- **Models CLI has no fallback-test subcommand:** `registerModelsCli` wires `list`, `status`, `set`, `set-image`, `aliases`, `fallbacks`, `image-fallbacks`, and `scan`. There is no `test-fallback` command or equivalent entry point under `models`.
  - file: `src/cli/models-cli.ts:12`
  - command: `nl -ba src/cli/models-cli.ts | sed -n '1,230p'`
  - sha: `d16b879334eeec07e610b1ea0063fbd134b61048`
- **Documented probe is auth-focused:** The docs describe `openclaw models status --probe` as a live probe of configured auth profiles/providers. They do not describe a command that skips the primary model or verifies the fallback chain end-to-end.
  - file: `docs/cli/models.md:37`
  - command: `nl -ba docs/cli/models.md | sed -n '20,90p'`
  - sha: `d16b879334eeec07e610b1ea0063fbd134b61048`
- **Probe implementation only uses the first candidate per provider:** `buildCandidateMap` preserves multiple candidate models per provider, but `selectProbeModel` returns `direct[0]`. A chain like `openai/A -> openai/B` therefore probes only `openai/A`, not the fallback entry, so it does not verify the configured fallback chain itself.
  - file: `src/commands/models/list.probe.ts:133`
  - command: `nl -ba src/commands/models/list.probe.ts | sed -n '133,164p'`
  - sha: `d16b879334eeec07e610b1ea0063fbd134b61048`
- **Status probe is fed from config candidates, not fallback execution:** `modelsStatusCommand` builds `modelCandidates` from the default model, fallbacks, image model/fallbacks, and allowed models, then hands them to `runAuthProbes`. That path is an auth-health check, not the runtime fallback mechanism described in the issue.
  - file: `src/commands/models/list.status-command.ts:267`
  - command: `nl -ba src/commands/models/list.status-command.ts | sed -n '267,310p'`
  - sha: `d16b879334eeec07e610b1ea0063fbd134b61048`
