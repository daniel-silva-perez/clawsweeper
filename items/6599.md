## Summary

Keep open. Current main does not implement a `/models test-fallback` command or CLI equivalent that intentionally skips the primary model and exercises the configured fallback chain. The closest existing feature is `openclaw models status --probe`, but that only runs direct per-provider auth/model probes and does not invoke the runtime fallback-chain path.

## Evidence

- **No CLI fallback-test command:** `registerModelsCli()` defines `models list`, `status`, `set`, `set-image`, `aliases`, `fallbacks`, `image-fallbacks`, and `scan`. The only live-check surface is `models status --probe`; there is no `test-fallback` subcommand.
  - file: `src/cli/models-cli.ts:12`
  - sha: `178a314a4c1cfa98ac0e8545707162427e38f9c5`
- **No `/models test-fallback` parser path:** The `/models` command parser recognizes only `providers`, `list`, and `add`. Any other first token falls back to list parsing, so `/models test-fallback` would not dispatch a dedicated fallback-test action.
  - file: `src/auto-reply/reply/commands-models.ts:204`
  - sha: `178a314a4c1cfa98ac0e8545707162427e38f9c5`
- **Existing probe is direct auth probing, not fallback-chain execution:** `models status --probe` builds candidate model refs from the configured default/fallback lists and calls `runAuthProbes()`. Inside probe execution, `selectProbeModel()` picks one model per provider and `probeTarget()` calls `runEmbeddedPiAgent()` directly for that model/profile. It does not call `runWithModelFallback()` or intentionally skip the primary model to verify chain behavior.
  - file: `src/commands/models/list.status-command.ts:267`
  - sha: `178a314a4c1cfa98ac0e8545707162427e38f9c5`
- **Fallback runtime exists separately:** The actual fallback-chain runner is `runWithModelFallback()`, which walks resolved candidates at runtime. That path exists, but the probe/test surfaces above do not use it for an operator-facing fallback verification command.
  - file: `src/agents/model-fallback.ts:652`
  - sha: `178a314a4c1cfa98ac0e8545707162427e38f9c5`
- **Docs match the current limited surface:** The docs list `openclaw models status` and describe `--probe` as a live auth check. They do not document any `/models test-fallback` or `openclaw models test-fallback` command.
  - file: `docs/concepts/models.md:165`
  - sha: `178a314a4c1cfa98ac0e8545707162427e38f9c5`
