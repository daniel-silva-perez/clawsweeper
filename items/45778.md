## Summary

Keep open. Current `main` does not appear to implement either requested path: cron still only accepts `systemEvent` and `agentTurn` payloads, isolated/chat delivery is still coupled to agent-turn execution, and the child-process env injection that exists today exposes session/channel metadata but not a reply target env for later `openclaw message send` use.

## Evidence

- **Cron payload contract still has only two kinds:** `CronPayloadSchema` is a union of `systemEvent` and `agentTurn` only. There is no `directExec` or similar non-LLM execution payload in the gateway protocol schema.
  - file: [src/gateway/protocol/schema/cron.ts:130](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/gateway/protocol/schema/cron.ts#L130)
  - command: `nl -ba src/gateway/protocol/schema/cron.ts | sed -n '126,176p'`
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Cron validation still requires agent turns for isolated jobs:** `assertSupportedJobSpec` rejects non-`agentTurn` payloads for `isolated`, `current`, and `session:*` jobs, and `assertDeliverySupport` keeps channel delivery on the isolated/session path. That means there is still no built-in isolated chat-delivery path for a direct command payload.
  - file: [src/cron/service/jobs.ts:156](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/cron/service/jobs.ts#L156)
  - command: `nl -ba src/cron/service/jobs.ts | sed -n '156,214p'`
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Docs still describe announce delivery as post-agent-turn behavior:** The scheduled-tasks docs say isolated jobs share chat delivery through the agent `message` tool and otherwise `announce`/`webhook`/`none` control what the runner does with the final reply after the agent turn. There is no documented direct command execution mode here.
  - file: [docs/automation/cron-jobs.md:129](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/docs/automation/cron-jobs.md#L129)
  - command: `nl -ba docs/automation/cron-jobs.md | sed -n '128,150p'`
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Existing child-process env injection lacks reply target vars:** The CLI runner injects `OPENCLAW_MCP_AGENT_ID`, `OPENCLAW_MCP_ACCOUNT_ID`, `OPENCLAW_MCP_SESSION_KEY`, and `OPENCLAW_MCP_MESSAGE_CHANNEL`, but not a reply-target env such as the issue's proposed `OPENCLAW_REPLY_CHANNEL` / `OPENCLAW_REPLY_TARGET`.
  - file: [src/agents/cli-runner/prepare.ts:182](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/agents/cli-runner/prepare.ts#L182)

- **Repo search found no proposed reply env vars:** A repo-wide search for `OPENCLAW_REPLY_CHANNEL|OPENCLAW_REPLY_TARGET` returned no matches on current `main`, which supports the conclusion that the env-injection alternative is not implemented.
