## Summary

Keep this PR open. Current main still lacks a `compaction.memoryFlush.model` config key and still runs memory flush with the session provider/model, so the PR is a relevant implementation candidate for the linked feature request rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open and review it as the implementation path for #66802. The likely best product/code path is to add `agents.defaults.compaction.memoryFlush.model` consistently across config types, zod schema, generated schema/help/labels/docs, and runtime resolution, with tests proving provider/model and model-only overrides, fallback to the session model, threshold calculation against the resolved flush model, and CLI-provider gating against the resolved flush provider.
