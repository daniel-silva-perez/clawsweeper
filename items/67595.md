## Summary

Keep open. Current main has a partial fix for the LaunchAgent embedded-token/service-metadata portion, but the separate `channels list` provider-usage path still fetches usage by default and still surfaces raw provider/network fetch failures instead of treating usage as optional.

## Best Possible Solution

Keep this open, or split only after creating/linking a focused follow-up for the `channels list` failure. The remaining fix should make provider usage optional and robust in both human and JSON output: either skip usage when appropriate, or convert provider fetch exceptions into structured non-fatal usage warnings/snapshots, with regression coverage for no chat channels plus auth profiles and a rejected provider fetch.
