## Summary

Keep open. Current main appears to have addressed a narrower stale-service fast path via #70752, but the broader macOS plaintext LaunchAgent projection and the `channels list` usage-error behavior are not clearly resolved. The launchd renderer still writes arbitrary service environment entries inline to `EnvironmentVariables`, service audit still treats inline `OPENCLAW_GATEWAY_TOKEN` as a repairable issue, and `channels list` still runs provider usage collection by default after printing auth profiles, with raw usage fetch failures surfaced in the human path and no equivalent catch in JSON mode.

## Best Possible Solution

Keep this issue open or split it into two focused follow-ups. For the gateway side, make macOS `gateway install --force` and doctor repair converge by ensuring launchd service metadata never receives inline `OPENCLAW_GATEWAY_TOKEN`, while preserving the systemd `EnvironmentFile` behavior and updating status/audit hints. For `channels list`, make empty chat-channel configurations succeed cleanly by either skipping usage when only auth profiles exist or isolating provider-usage fetch failures into structured, non-fatal per-provider warnings with consistent text and JSON behavior.
