## Summary

Keep #70853 open. Current main still rotates silent auto-reply sessions to a fresh sessionId/sessionFile without preserving recent transcript context, and the exact linked fix PR #70898 is open/unmerged rather than shipped on main.

## Best Possible Solution

Keep the issue open and review/land PR #70898 or an equivalent patch that seeds or logically injects a small replay-safe tail of prior user/assistant transcript records when `resetReplyRunSession` silently rotates a live session. Add focused regression coverage for the auto-reply reset helper and leave explicit `/new`/`/reset` behavior fresh.
