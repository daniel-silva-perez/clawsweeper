## Summary

Keep PR #49409 open. Current main still has the session-memory hook attempt LLM slug generation by default on `/new` and `/reset`, then writes a per-session `YYYY-MM-DD-slug.md` file. Main has since added reader compatibility for date-prefixed slug artifacts, so part of the original reader-mismatch concern is mitigated, but the non-blocking writer/canonical daily-file request is not implemented. The PR is relevant but not merge-ready as-is because its review discussion identifies a real concurrent header-write race and timeout parsing bug in the proposed diff.

## Best Possible Solution

Revise this PR or replace it with a narrower one that makes `/new` and `/reset` session-memory writes non-blocking by default, appends safely to canonical `memory/YYYY-MM-DD.md` if that remains the desired storage contract, updates the hook docs/tests, and preserves the existing workspaceDir/session-key regression coverage. The daily-file header logic should be concurrency-safe in a single open/append path, or the format should avoid first-writer-only headers entirely.
