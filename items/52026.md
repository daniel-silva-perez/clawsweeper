## Summary

Keep open. Current main still does not implement the PR's core dispatch-idle barrier before queued followups, and the related superseding PR in the provided context was closed unmerged rather than shipped. The PR branch also had review-noted races, so it should be reworked rather than closed as already handled.

## Best Possible Solution

Keep the item open, but do not merge the stale branch as-is. The better fix is a current-turn-scoped dispatch barrier passed from the active dispatcher into followup draining, with tests for same-turn final/block delivery ordering, isolation from newer turns, chained followup sends, and `/stop` while a followup is waiting.
