## Summary

Keep open. Current `main` does not implement the PR's requested npm engine-aware version selection or the `--ignore-engine` CLI escape hatch. OpenClaw currently enforces only `openclaw.install.minHostVersion` after a package is resolved/downloaded and during manifest-registry loading, which is a different, narrower mechanism.

## Risks / Open Questions

- Closing this PR as implemented would be incorrect: the requested install/update behavior is not present on current `main`.
- `minHostVersion` only blocks incompatible installs/loads after resolving the requested package; it does not preserve user tag semantics while selecting a compatible published version, which is the core behavior this PR adds.
