## Summary

Keep open. Current main still has the embedded PI abort/wait/isActive/queue helpers internally, but the injected plugin runtime surface `api.runtime.agent` only exposes embedded-run start helpers plus identity/session helpers. The related gateway abort work helps external Gateway clients via `chat.abort`; it does not supersede this plugin-facing runtime API request.

## Best Possible Solution

Keep the issue open and implement the additive plugin runtime contract: re-export the abort/wait/isActive/queue helpers from `src/plugins/runtime/runtime-embedded-pi.runtime.ts`, add matching lazy properties in `createRuntimeAgent()`, declare them on `PluginRuntimeCore["agent"]`, and update the runtime shape test plus plugin runtime docs/API inventory. Treat the Gateway abort fix as related shipped work, not as the canonical resolution for this plugin-facing request.
