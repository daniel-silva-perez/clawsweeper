## Summary

Keep open. Current main still has the embedded PI abort/wait/isActive/queue helpers internally, but the plugin runtime object injected as `api.runtime.agent` only exposes the embedded run start helpers plus agent/session utilities. The related Gateway `chat.abort` work is adjacent and shipped, but it does not satisfy this plugin-facing API request from the CoClaw plugin context.

## Best Possible Solution

Keep this issue open and implement the additive plugin runtime contract: re-export `abortEmbeddedPiRun`, `waitForEmbeddedPiRunEnd`, `isEmbeddedPiRunActive`, and `queueEmbeddedPiMessage` from `src/plugins/runtime/runtime-embedded-pi.runtime.ts`; lazy-bind them in `createRuntimeAgent()`; declare them on `PluginRuntimeCore["agent"]`; and update the runtime shape test plus plugin SDK runtime docs/API inventory. Treat the Gateway abort fix as related shipped work, not as the canonical resolution for this plugin-facing request.
