## Summary

Keep open. Current main enforces the reported Control UI device-identity behavior and has docs/UI guidance, but it does not emit the requested startup/runtime log warning for a configuration that will reject non-localhost insecure HTTP Control UI connections. The related #32473 remains open as the broader user-facing problem, so this narrow developer/operator UX follow-up is still actionable rather than implemented or safely duplicate-closed.

## Best Possible Solution

Keep this issue open as a narrow operator-UX improvement. The likely implementation belongs in gateway startup logging after runtime config and TLS/bind/origin state are known, warning when Control UI is enabled, device auth is not disabled, and a configured/direct non-localhost HTTP browser origin would be unable to create device identity. Add focused coverage in src/gateway/server-startup-log.test.ts and link users to https://docs.openclaw.ai/web/control-ui#insecure-http or https://docs.openclaw.ai/gateway/security.
