## Summary

Keep this PR open. Current main still silently resolves mixed `command` + non-empty `url` MCP server configs as stdio because `resolveMcpTransportConfig` checks stdio before HTTP and returns immediately when `command` is present. The PR is a focused implementation candidate for #72111, aligns code with existing docs that already say mixed configs are rejected, and its diff is limited to the transport resolver plus targeted tests.

## Best Possible Solution

Review and land this focused PR after normal CI/changed-gate validation, or apply the same guard and tests on main. The fix belongs in `src/agents/mcp-transport-config.ts`, before stdio resolution, so mixed transport configs fail early with a clear diagnostic while pure stdio, pure SSE, and pure streamable-http configs keep their current behavior.
