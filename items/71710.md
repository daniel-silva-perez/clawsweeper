## Summary

Keep open. The report is recent, concrete, and still maps to current main: `openclaw agent` waits on the Gateway request without a CLI-level SIGTERM cancellation path, the Gateway request API only has an internal timeout timer rather than an external AbortSignal, and the public agent CLI docs do not document `timeout -k` or equivalent cron supervisor escalation. Related #66399/#70026 cover adjacent supervisor and SIGKILL behavior, but they do not fully supersede this cron-spawned `openclaw agent` SIGTERM surface.

## Best Possible Solution

Keep the issue open and fix it in core. Add a top-level `openclaw agent` termination path that installs SIGTERM/SIGINT handlers, aborts the pending Gateway/local agent run through an AbortController or best-effort Gateway abort for the known runId/session, flushes output, and exits with the signal-derived code after a bounded grace period. Also add CLI/cron docs recommending `timeout -k <delay>` or equivalent supervisor escalation until the runtime fix is shipped, with regression coverage around CLI signal handling and Gateway request cancellation.
