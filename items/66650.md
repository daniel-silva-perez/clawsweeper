## Summary

Keep this PR open. Current main still has the reported mismatch: `sessions_history.limit` has no 1000 maximum, the tool forwards oversized finite limits to `chat.history`, and the gateway rejects `chat.history.limit > 1000` before its handler clamp runs. The PR is a narrow candidate fix for open bug #66573; related PR #66615 is closed unmerged and does not supersede it.

## Best Possible Solution

Keep this PR open and review it as the canonical narrow fix for #66573. The maintainer path should clamp `sessions_history` outgoing limits to the gateway's 1000-message contract, add `maximum: 1000` to the tool schema, and include a runtime-clamp regression test; any decision to raise the `chat.history` protocol cap should remain a separate gateway/API decision.
