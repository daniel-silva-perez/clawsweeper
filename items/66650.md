## Summary

Keep this PR open. Current main still has the reported bug boundary: `sessions_history.limit` has no `maximum: 1000`, the tool forwards finite caller-supplied limits without clamping, and the gateway schema rejects `chat.history.limit > 1000` before the handler's hard cap can run. The related gateway-limit PR #66615 is closed and does not supersede this narrower tool-side fix.

## Best Possible Solution

Keep the PR open and review it as the canonical narrow fix for #66573. The maintainer path should clamp `sessions_history` outgoing limits to the gateway's 1000-message contract, add `maximum: 1000` to the tool schema, and include a runtime-clamp regression test; only raise the gateway `chat.history` cap through a separate explicit protocol decision.
