## Summary

Keep #59933 open. The report is recent, has concrete Telegram reproduction steps plus a Discord corroboration comment, and current main still appears to have tool-layer visibility filtering that can hide same-agent non-current group/channel sessions unless `tools.sessions.visibility` is explicitly widened. Existing `kinds` filtering does not by itself solve that visibility behavior.

## Evidence

- **Issue context:** The issue reports same-agent Telegram group sessions existing on disk but missing from `sessions_list`; the April 14 comment independently narrows a Discord case where the session store and CLI enumerate the session but the tool-facing list hides it.
  - command: `provided GitHub context for issue #59933`
  - sha: [d6ef1fcf24ee](https://github.com/openclaw/openclaw/commit/d6ef1fcf24eea3a5985d653a3baf400e32aa1b75)
- **Tool applies post-gateway visibility filtering:** `sessions_list` calls gateway `sessions.list`, then checks each returned key with `visibilityGuard.check(key)` and skips entries that are not allowed before applying kind filtering and returning rows.
