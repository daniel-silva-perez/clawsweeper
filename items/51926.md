## Summary

Keep open. Current main still has the exact agents `nodes` tool mismatch from PR #51926 / issue #51903: the schema exposes `node` as optional for the flat tool input, but the `describe` branch still hard-requires it before calling the existing default-node resolver. The PR remains a focused useful fix, though it should add a regression test for `{ "action": "describe" }` without `node` before landing.

## Best Possible Solution

Keep PR #51926 open or have a maintainer take it over. The best fix is the PR's narrow behavior change in `src/agents/tools/nodes-tool.ts` plus a focused `src/agents/tools/nodes-tool.test.ts` case asserting that `{ action: "describe" }` calls `resolveNodeId({}, undefined, true)` and forwards to `node.describe`; all other node actions should continue requiring explicit `node` where they do today.
