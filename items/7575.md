## Summary

Keep #7575 open. Current main still does not ship Sysbox/sysbox-runc gateway isolation, socketless Docker-in-Docker sandboxing, or a supported `sandbox.docker.runtime` selector; the latest issue discussion also explicitly keeps this as the canonical Sysbox/runtime-architecture tracker rather than treating the existing Docker-socket sandbox work as completion.

## Best Possible Solution

Keep #7575 open for maintainer product judgment as the canonical Sysbox/runtime-architecture tracker. A good resolution would either add a conservative `sandbox.docker.runtime` config path with schema, docs, tests, and host setup guidance, or ship/endorse a Sysbox sandbox backend plugin using the existing sandbox backend API if maintainers decide this belongs outside core.
