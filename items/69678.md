## Summary

Keep #69678 open. Current main still hard-caps isolated cron summaries at 2000 characters before they reach the run-log entry, and `cron.runLog` still exposes only `maxBytes` and `keepLines`. The related ClawSweeper matches cover broader cron UI/routing/status issues and do not supersede this narrow run-log summary-length request.

## Best Possible Solution

Implement this in core cron rather than closing it: add a `cron.runLog.maxSummaryChars` config field with schema/types/help/docs/generated metadata, resolve it with a conservative default, thread it into isolated-agent summary selection or the run-log append boundary, and add focused tests proving outputs over 2000 characters are retained up to the configured cap while `maxBytes` and `keepLines` still bound total run-log storage.
