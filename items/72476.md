## Summary

This PR should stay open. Current main still resolves configured `thinkingBudgets` into generic extra params but drops them before calling the pi-ai stream function, while both the pinned pi-ai contract and OpenClaw’s Anthropic/Google-style transport paths can consume `SimpleStreamOptions.thinkingBudgets`. The PR is a focused implementation candidate, not obsolete or superseded, but it should address the review comment about rejecting zero/negative token budgets before merge.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to update the PR to validate `thinkingBudgets` as positive token counts, keep the regression test proving configured budgets reach pi-ai stream options, then run the targeted extra-params test and changed gate before merge.
