## Summary

Current `main` at `5dfc1b90e176a9e4d13b9e91ad6affec6b92f213` still implements the old fallback behavior for disallowed cron `payload.model` values, and both tests and docs on `main` explicitly encode that behavior. This PR is not implemented on `main`, so there is no strong basis to close it conservatively.
