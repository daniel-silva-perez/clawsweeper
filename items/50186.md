## Summary

Current `main` only partially matches the request. OpenClaw now generates/docs Linux systemd units with `KillMode=control-group`, but the doctor/status code still does not inspect or report `KillMode`, `TasksCurrent`, `MemoryCurrent`, cgroup hygiene, or related warnings/remediation. That leaves the core observability part of issue #50186 unimplemented, so this should stay open.

## Evidence
