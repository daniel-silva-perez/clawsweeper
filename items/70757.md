## Summary

Keep this PR open. Current main still authenticates BlueBubbles inbound webhooks with the BlueBubbles API password, while this PR is the active same-author implementation candidate for the open linked security bug. The provided PR diff is security-sensitive but aligned with the stated fix and does not show an unexplained supply-chain or CI execution expansion.

## Best Possible Solution

Keep this PR open, or land an equivalent maintainer-approved fix. The best product path is to add a dedicated BlueBubbles `webhookSecret`, authenticate inbound BlueBubbles webhooks only against that secret, enforce that it is present and distinct from the API password across inherited account configs, expose it through setup/CLI metadata and SecretRef surfaces, and update docs and regression tests consistently. After merge, #70755 can close as implemented.
