## Summary

Keep #70757 open. Current main still authenticates BlueBubbles webhooks with `channels.bluebubbles.password`, documents putting that same password in the public webhook URL, and has no BlueBubbles `webhookSecret` config surface. The PR addresses a real remaining auth-boundary bug linked from #70755, and the review discussion shows later commits addressed the main bot-review gaps.

## Best Possible Solution

Keep the PR open and continue review toward landing a focused fix that adds a distinct BlueBubbles `webhookSecret`, authenticates inbound webhook requests only against that field, updates setup/manifest/config schemas/docs/secret registration, and keeps regression coverage for password-only and matching-secret rejection.
