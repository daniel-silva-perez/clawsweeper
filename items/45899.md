## Summary

Current `main` does not contain the PR’s Feishu-specific runtime behavior. The dispatcher still only logs the raw send error and closes streaming, with no blocked-local-`MEDIA:` classification, no Feishu hint text, no one-time plain-text fallback, and no matching tests. Newer docs and prompt guidance on `main` cover some of the local-path guidance generically, so the PR may need a rebase/update, but the request is not implemented well enough to close as already done.
