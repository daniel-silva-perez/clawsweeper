## Summary

Keep open. Current main still returns retryable `UNAVAILABLE` immediately for startup-gated `chat.history`/`models.list`; the server-side barrier proposed by this PR is not implemented. Later work added client retries, including TUI retry in #69164, but that does not fully cover this PR's stated stale/cached Control UI bundle case or the server-side hold behavior.

## Best Possible Solution

Keep this PR open for maintainer review or a rebased successor. The best implementation path is still a gateway-owned startup barrier or equivalent server-side queue for startup-gated methods, with regression tests covering dispatch-after-sidecars, timeout fallback, non-gated bypass, and legacy no-barrier behavior. Client retries from #65365/#69164 should remain defense-in-depth, not the only fix for cached pre-retry clients.
