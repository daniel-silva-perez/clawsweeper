## Summary

Keep this PR open. Current main has related Anthropic Opus 4.7 forward-compat behavior and no longer simply drops text-only image attachments, but it has not implemented this PR's specific gateway image-support fallback: `resolveGatewayModelSupportsImages` still only applies the Claude-name fallback to `claude-cli`, not direct `anthropic`, and the current tests do not cover the direct Anthropic stale-catalog or catalog-miss cases proposed by the PR.

## Best Possible Solution

Review or update this focused PR against current main. The likely best code path is to land a small gateway capability fix with the two proposed regression tests, or replace it with a more generic provider-owned capability seam if maintainers do not want additional provider-name checks in `src/gateway/session-utils.ts`. Do not close it as implemented until current main either includes the direct `anthropic` fallback or has an explicit maintainer decision that the offload path is the intended behavior.
