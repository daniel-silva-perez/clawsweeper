## Summary

Keep open. Current `main` does not support closing this conservatively: the bundled `gh-issues` skill still ships with `user-invocable: true`, the repo's skill-creator validator still rejects that key, and the skill remains well above the skill-creator size guidance. The report's claim that `user-invocable` is unsupported by OpenClaw itself is outdated, but there is still a real repo inconsistency plus the size concern is still present.
