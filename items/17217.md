## Summary

Current `main` already documents what a session is and how to reset one, but it does not implement the issue’s requested immediate TTL pruning semantics or a dedicated manual context-prune API/control. The issue is therefore only partially addressed and should remain open.

## Evidence

- **Session boundaries are now documented:** `docs/concepts/session.md` defines routing and lifecycle, including daily reset, optional idle reset, and manual `/new` or `/reset` boundaries.
  - file: [docs/concepts/session.md:57](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/docs/concepts/session.md#L57)

  - file: [docs/concepts/session-pruning.md:28](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/docs/concepts/session-pruning.md#L28)
