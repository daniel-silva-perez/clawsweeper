## Summary

Keep PR #70629 open. Current main still hardcodes `minimal: 128` for the Gemini 2.5 direct Google `thinkingBudget` path and lacks the PR's regression test, so this is not implemented or superseded. The earlier related shipped fix only handled `thinkingBudget=0` for Gemini 2.5 Pro, not the minimal-budget floor.

## Best Possible Solution

Keep the PR open and adapt or merge it after maintainer review: update `extensions/google/transport-stream.ts` so affected Gemini 2.5 minimal reasoning requests do not emit an invalid `thinkingBudget`, add regression coverage in `extensions/google/transport-stream.test.ts` for the affected model IDs, and verify whether the 512 floor should apply to all three PR-listed families or only Flash Lite/current live-rejecting models.
