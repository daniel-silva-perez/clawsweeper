## Summary

Keep open. The PR addresses a real docs confusion, but current main has not added the proposed tool-name-vs-shell-command clarification, and the PR as written still needs correction because it documents `openclaw sessions list`, while current CLI/docs use `openclaw sessions` for listing sessions.

## Best Possible Solution

Keep the PR open and ask for a narrow revision: remove the duplicated sentence, replace `openclaw sessions list` with the current `openclaw sessions` listing form, keep `openclaw agents list`, and point users to the public CLI docs/help for command discovery.
