## Summary

Current main already blocks the reported Windows `:\...` / `:/...` malformed exec-path case, and the same guard plus regression tests are present in released `v2026.4.23`.

## Evidence

- **Drive-less Windows rooted paths are explicitly rejected during executable resolution:** `resolveExecutablePathCandidate` returns `undefined` for `^:[\\/]` on Windows, preventing the reported `C:\...` drive letter loss from being re-resolved against the workspace.
  - file: [src/infra/executable-path.ts:6](https://github.com/openclaw/openclaw/blob/9b18ae877018a34933e27bb7b2986cb97e9602aa/src/infra/executable-path.ts#L6)

  - sha: [9b18ae877018](https://github.com/openclaw/openclaw/commit/9b18ae877018a34933e27bb7b2986cb97e9602aa)
- **Regression test covers malformed drive-less executable tokens:** Windows-only test asserts `resolveExecutablePath` leaves `:\Users\...\git.exe` and `:/Users/.../git.exe` unresolved instead of treating them as cwd-relative executables.
  - file: [src/infra/executable-path.test.ts:81](https://github.com/openclaw/openclaw/blob/9b18ae877018a34933e27bb7b2986cb97e9602aa/src/infra/executable-path.test.ts#L81)

## Close Comment

Closing this because current `main` already handles the reported malformed Windows exec-path case, and the fix is already shipped in `v2026.4.23`.

- Codex review checked `src/infra/executable-path.ts:6`; drive-less Windows rooted paths like `:\Users\...` are now rejected during executable resolution instead of being resolved against `cwd`.
- Codex review checked `src/infra/executable-path.test.ts:81`; there is a Windows regression test proving `:\Users\...\git.exe` and `:/Users/.../git.exe` are not treated as cwd-relative executables.
- Codex review checked `src/infra/exec-command-resolution.test.ts:434`; there is also regression coverage that these malformed tokens are not expanded into corrupted allowlist/audit paths like the ones described here.
- The same code is present in release `v2026.4.23` at `9b18ae877018a34933e27bb7b2986cb97e9602aa`, so this is not just an unreleased `main` change.
