## Summary

Keep open. Current main still does not implement Telegram inline-query support: the Telegram plugin has update-type/probe awareness, but no `inline_query` handler, no `answerInlineQuery` path, no `chosen_inline_result` handling, and no config/docs surface for enabling inline mode. Related PR #51963 appears to target the same feature but is not shipped and prior review context reported unresolved correctness/security concerns, so it should not supersede this issue for conservative cleanup.

## Best Possible Solution

Keep the issue open. The best path is a focused Telegram plugin implementation: add an explicit inline-query config/schema/docs surface, register a safe `bot.on("inline_query")` handler, authorize the querying user, route the query through the agent with an inline-safe session/result lifecycle, return `InlineQueryResultArticle` results via `answerInlineQuery`, handle chosen-result/callback correlation and expiry safely, and add targeted Telegram runtime tests. PR #51963 can inform that work, but it should not close this issue until the feature lands or maintainers choose a canonical replacement.
