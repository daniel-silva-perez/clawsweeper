## Summary

Keep PR #39761 open. Current main still does not expose `agentContext`, `MessageSendingAgentContext`, or equivalent agent reasoning metadata on the `message_sending` plugin hook, and the outbound delivery path still invokes the hook with only delivery fields. This is a real core plugin API gap, not ClawHub-only work, because external plugins cannot add missing event fields or thread embedded-runner metadata into core delivery hooks themselves.

## Best Possible Solution

Keep the PR open for maintainer review or adaptation. The best product path is to add a documented, backward-compatible core plugin API field for agent-originated `message_sending` events, wire it from embedded-runner metadata through the shared outbound delivery path, and cover it with focused hook contract tests. If maintainers prefer a narrower scope, split provider/model identity or retry-queue persistence into follow-up issues rather than closing this implementation as obsolete.
