## Summary

Keep open. Current `main` still downloads reply-target media in Telegram without checking whether the replied-to message was authored by the bot, so the substance of this PR is not implemented on `main`.

## Evidence

- **Runtime reply-media path still lacks a self-authored guard:** `resolveReplyMediaForMessage` on current `main` returns early only for missing reply targets or missing media, then immediately resolves and downloads the replied-to file. There is no check that `msg.reply_to_message.from.id` differs from `ctx.me.id` before `bot.api.getFile(replyFileId)` and `resolveMedia(...)`.
