## Summary

Current `main` already contains the shipped gateway-startup fix for this regression: the configured primary-model warmup now stays on the static model path instead of loading provider runtime graphs during boot, matching the reported `2026.3.28` regression window and the `2026.3.31` recovery report.

## Evidence

- **Shipped fix in changelog:** The `2026.3.31` release notes explicitly say gateway startup now keeps configured primary-model warmup on the static model path so container boots do not snapshot-load provider runtime graphs just to validate a configured model.
  - file: [CHANGELOG.md:1898](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/CHANGELOG.md#L1898)
  - command: `nl -ba CHANGELOG.md | sed -n '1892,1902p'`
- **Regression-introducing change window matches report:** The `2026.3.28` changelog added configured primary-model prewarm before channel startup, matching the user's reported version boundary for the regression.
  - file: [CHANGELOG.md:2324](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/CHANGELOG.md#L2324)
