## Summary

Keep open. Current main at e962381dbf1ecdb1cd4784fdeb6c2e07ddd3435b still treats ordinary `*.trajectory.jsonl` sidecars as primary session transcripts, so the reported `openclaw doctor` orphan false-positive remains possible. This PR is the active implementation candidate for open #70680, while related #72047 was closed unmerged; it should be reviewed rather than closed as obsolete.

## Best Possible Solution

Keep this PR open as the active fix candidate for #70680. Maintainers should review the focused session-artifact classifier approach, confirm the dotted `.trajectory` session-id behavior, address or document the disk-budget cleanup semantics, split unrelated agent workdir changes if needed, and then land the regression-tested doctor/session fix once CI is acceptable.
