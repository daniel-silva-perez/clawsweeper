## Summary

Keep open. Current main still treats trajectory sidecars as primary transcript candidates, and doctor still uses that broad predicate for orphan transcript scans. The PR is a focused, active fix for open issue #70680; the related #72047 is closed unmerged, so it does not supersede this PR.

## Best Possible Solution

Keep #70812 open as the active fix path for #70680. The best product/code outcome is to land a narrow helper-level classification fix plus doctor regression coverage, preserving valid session IDs that end in `.trajectory` and making any disk-budget sidecar policy explicit.
