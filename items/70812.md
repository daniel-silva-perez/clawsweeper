## Summary

Keep open. Current main still treats any non-archived `.jsonl` file, including `<session>.trajectory.jsonl`, as a primary session transcript; doctor then uses that predicate directly for orphan transcript candidates. Trajectory sidecars are documented and generated beside session files, and current tests do not cover excluding them from orphan scans. PR #70812 is a focused, recent bug-fix attempt for open issue #70680, and the related #72047 PR is closed unmerged, so there is no implemented or superseding mainline fix to close this PR against.

## Best Possible Solution

Keep #70812 open as the active fix candidate for #70680. The best path is to land a reviewed version that excludes real trajectory sidecars from doctor orphan and usage-counted transcript scans while preserving valid primary transcripts whose session ID ends in `.trajectory`; if disk-budget sidecar retention needs separate treatment, track that explicitly rather than closing this focused bug fix.
