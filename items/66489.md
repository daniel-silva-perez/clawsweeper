## Summary

Current main has the permanent delivery-error mechanism, but it still does not classify the reported local-media allowlist rejection or the proposed Telegram "message is too long" error. The broader local-media staging consolidation is related but does not supersede this narrow delivery-queue classifier request.

## Best Possible Solution

Keep this issue open. The narrow fix belongs in `src/infra/outbound/delivery-queue-recovery.ts`: add classifier coverage for the local-media allowlist rejection, add policy/recovery tests proving it moves straight to `failed/`, and evaluate whether Telegram `message is too long` is still observable on current text/caption chunking before adding that pattern. Enqueue-time media validation could be a follow-up, but the classifier should remain as a safety net for already-queued entries.
