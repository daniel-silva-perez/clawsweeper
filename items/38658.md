## Summary

Keep PR #38658 open. Current main at 4a3030df9efa72c44bab567a4390ac0c4876a73f still has Slack's pending-only channel-history behavior: accepted requireMention=false channel messages are not retained for the next turn, and dispatch clears room history after processed turns. The PR remains a focused plausible fix, but it changes Slack prompt-context semantics and still needs maintainer judgment on chronological ordering and docs/tests updates.

## Best Possible Solution

Keep the PR open for maintainer review. If always-on Slack channel context is desired, carry the resolved requireMention/history-retention policy into dispatch, retain processed room messages for requireMention=false in both delivered-reply and no-reply paths, preserve chronological order by recording at ingress or sorting by timestamp, update the Slack/group-history docs, and land regression coverage that replaces the current opposite assertion. If maintainers want pending-only history to remain the cross-channel contract, close the PR with that explicit product rationale.
