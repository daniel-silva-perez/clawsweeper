## Summary

Keep open. This is a plausible feature request, not stale, with a concrete proposed config shape, supporting discussion, and an open related PR (#58683). Current main has fixed `heartbeat.every` plus a single `activeHours` gate, but not daypart/variable interval scheduling.

## Evidence

- **Current config contract has fixed interval plus one active-hours window:** `heartbeat` exposes `every` and optional `activeHours`; there is no `schedule[]` or profile/daypart config in the typed public config shape.
  - file: [src/config/types.agent-defaults.ts:333](https://github.com/openclaw/openclaw/blob/a7de722f4f5979d37f8809c9aaf6d9d8907d5e3f/src/config/types.agent-defaults.ts#L333)

- **Runtime schema is strict and would not accept schedule blocks:** `HeartbeatSchema` is a strict object with `every`, `activeHours`, and other existing fields, but no `schedule` or `profiles`. A proposed `heartbeat.schedule[]` config would be rejected as an unknown key.
  - file: [src/config/zod-schema.agent-runtime.ts:19](https://github.com/openclaw/openclaw/blob/a7de722f4f5979d37f8809c9aaf6d9d8907d5e3f/src/config/zod-schema.agent-runtime.ts#L19)
