## Summary

Current `main` already supports interrupting active runs and steering/refining instructions mid-execution in the Gateway + Control UI, so issue #50554 is implemented on `main`.

## Evidence

- **Gateway interrupt-and-steer implementation:** `interruptSessionRunIfActive()` aborts tracked/embedded runs, clears queued work, waits for the active embedded run to end, and `handleSessionSend()` reuses the same session to send the replacement message when `sessions.steer` is called.

  - file: [ui/src/ui/chat/slash-command-executor.ts:808](https://github.com/openclaw/openclaw/blob/81666e586aa6d4305e6472d7075e62f63f701831/ui/src/ui/chat/slash-command-executor.ts#L808)
