## Summary

Current `main` does not appear to implement the requested OpenRouter attribution fix. The OpenRouter attribution policy still emits `X-OpenRouter-Title` and `X-OpenRouter-Categories` instead of the documented `X-Title`, and repo tests explicitly lock in those header names. The request path on `main` does forward wrapper headers into the OpenAI SDK client, so this is not a strong candidate for `implemented_on_main` or `cannot_reproduce`.
