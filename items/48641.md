## Summary

Keep open. Current `main` still resolves Discord account config with a shallow root-vs-account merge, and the Discord tests explicitly codify `allowFrom` override behavior instead of merging global and account-scoped lists. The monitor path then feeds that single resolved `allowFrom` list into DM authorization, and unauthorized DM preflight exits early with only verbose logging, which matches the issue's reported silent-drop failure mode. I did not find evidence in the checked-out `main` or the `v2026.4.22` diff that this issue has already been fixed.

## Evidence

- **Discord account config still uses generic shallow merge:** `mergeDiscordAccountConfig()` delegates to `resolveMergedAccountConfig()`, which in turn uses `mergeAccountConfig()` with object spread (`...base, ...accountConfig`). That merges objects shallowly and does not combine array fields like `allowFrom`.
