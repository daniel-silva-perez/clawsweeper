## Summary

Keep this PR open. Current main still lacks the turn-start stale `pluginDebugEntries` cleanup, the preserve-activity path for metadata-only session updates, and the verbose-refresh suppression needed after a cleanup failure. The PR remains a focused bug-fix candidate with relevant regression coverage and no obvious security-sensitive drift in the supplied diff.

## Best Possible Solution

Keep this PR open and review it as a focused runtime/session-store bug fix. The best path is to land or revise the generic turn-start cleanup in `src/auto-reply/reply/agent-runner.ts`, add preserve-activity support to `updateSessionStoreEntry()` in `src/config/sessions/store.ts`, suppress disk rehydration after a cleanup failure for that turn, and keep Active Memory's writer behavior extension-owned. The proposed regression tests in `src/config/sessions.test.ts` and `src/auto-reply/reply/agent-runner.misc.runreplyagent.test.ts` are the right guardrails, with normal CI validation before merge.
