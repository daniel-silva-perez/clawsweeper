## Summary

Keep open. Current main still lacks the debug Event Log sticky-at-bottom behavior proposed by PR #70105: the debug view renders a plain list with no scroll handler, lifecycle only schedules chat/log scrolling, `app-scroll.ts` has no debug event-log scheduler/scroll state, and the event buffer still stores newest-first entries. The related #69520 is scoped to chat scroll interrupts, not the debug Event Log surface.

## Best Possible Solution

Keep #70105 open for maintainer review or land an equivalent focused Control UI fix: add debug Event Log scroll containment, track `debugEventLogAtBottom`, schedule debug event-log auto-follow only when switching tabs or already at bottom, preserve overview's recent-first presentation if shared buffer ordering changes, and keep the focused `app-scroll.test.ts` coverage.
