## Summary

Keep open. Current main fixed the related non-atomic write corruption class, but it still has no separate `exec-runtime.json` store. Runtime metadata remains part of exec allowlist entries and ordinary gateway/node exec usage still writes those updates back to `exec-approvals.json`.

## Best Possible Solution

Keep this issue open as a focused architecture/security hardening feature. A maintainer-quality fix should add a backward-compatible `exec-runtime.json` store for usage/audit metadata, stop ordinary gateway/node exec usage from writing runtime fields to `exec-approvals.json`, preserve existing allowlist reads and legacy metadata display/migration, update Control UI and CLI views to merge runtime metadata correctly, and add docs/tests proving an allowlist-only approvals file can be read-only while runtime tracking remains writable.
