## Summary

Keep open. The linked corruption bug #63707 appears fixed on current main by atomic temp-file replacement, but this issue asks for a separate `exec-runtime.json` store. Current code still keeps `lastUsedAt`, `lastUsedCommand`, and `lastResolvedPath` on exec allowlist entries, still writes those updates back to `exec-approvals.json` during gateway and node exec, and docs/tests still describe that one-file behavior.

## Best Possible Solution

Keep this issue open as a focused feature/architecture request. A maintainer-quality fix would add a backward-compatible `exec-runtime.json` store for usage/audit metadata, keep existing `exec-approvals.json` reads working, stop ordinary exec usage from writing the allowlist file, update gateway/node/CLI views to merge or display runtime metadata correctly, and update docs/tests to prove allowlist-only files can be read-only while runtime tracking remains writable.
