## Summary

Keep PR #71379 open. Current main still has the reported CLI gap: `image.edit` capability metadata and the `openclaw infer image edit` command do not expose or forward `--size`, `--aspect-ratio`, or `--resolution`, while the shared image runtime already accepts and forwards those fields. The PR is a focused fix, though the existing review comments about resolution validation and temp-file cleanup should be resolved before merge.

## Best Possible Solution

Keep the PR open and have the author or maintainer finish it on the PR branch: add the three geometry options to `image edit`, forward them into `runImageGenerate()`, update `image.edit` capability metadata and CLI docs/changelog, add focused coverage, and resolve the review comments about resolution validation and temporary file cleanup before landing.
