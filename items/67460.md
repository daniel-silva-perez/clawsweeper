## Summary

Keep this PR open. Current main still lacks the mention-gating behavior requested by the linked feature issue, and the PR is a plausible implementation candidate, but it should not be closed as implemented or obsolete. The latest PR head still needs maintainer review because it carries an unresolved Discord role-context regression and unrelated postinstall/pdf-test changes alongside the mention-gating feature.

## Best Possible Solution

Keep this PR open and have a maintainer review it as an implementation candidate for explicit-agent-mention precedence. Before landing, restore Discord `memberRoleIds`, split the unrelated postinstall/pdf-test changes into separate PRs or remove them, verify that each channel’s `hasAnyMention` signal means an agent mention rather than any user/role mention, and update the SDK/docs/tests around the new mention policy.
