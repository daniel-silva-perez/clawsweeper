## Summary

Keep open. Current `main` still routes Google Chat conversations by space, not by thread, and the requested config surface is not present. The issue remains a valid feature request; the referenced PR in the issue context appears to be separate open work rather than something already shipped on `main`.

## Evidence

- **Google Chat docs still describe space-scoped sessions:** The user-facing docs explicitly say Google Chat messages are routed by space: DMs use `agent:<agentId>:googlechat:direct:<spaceId>` and spaces use `agent:<agentId>:googlechat:group:<spaceId>`. There is no documented per-thread mode here.
  - file: [docs/channels/googlechat.md:145](https://github.com/openclaw/openclaw/blob/68e2d6f0889e5365ecdc8f01ab5e3bf6065cbeeb/docs/channels/googlechat.md#L145)
  - command: `nl -ba docs/channels/googlechat.md | sed -n '140,155p'`
  - sha: [68e2d6f0889e](https://github.com/openclaw/openclaw/commit/68e2d6f0889e5365ecdc8f01ab5e3bf6065cbeeb)
- **Config type has no per-thread session option:** `GoogleChatAccountConfig` includes routing and threading-related options like `groupPolicy`, `historyLimit`, `replyToMode`, and `typingIndicator`, but there is no `threadSessionMode` or `sessionThread` field in the current config contract.
  - file: [src/config/types.googlechat.ts:40](https://github.com/openclaw/openclaw/blob/68e2d6f0889e5365ecdc8f01ab5e3bf6065cbeeb/src/config/types.googlechat.ts#L40)
  - command: `nl -ba src/config/types.googlechat.ts | sed -n '40,110p'`
  - sha: [68e2d6f0889e](https://github.com/openclaw/openclaw/commit/68e2d6f0889e5365ecdc8f01ab5e3bf6065cbeeb)
- **Inbound Google Chat routing keys off the space id only:** The Google Chat monitor resolves the inbound route with `peer.id = spaceId` and then writes `SessionKey: route.sessionKey`. The message thread resource is only copied into `ReplyToId`/`ReplyToIdFull`, so inbound session selection stays space-scoped on current `main`.
  - file: [extensions/googlechat/src/monitor.ts:173](https://github.com/openclaw/openclaw/blob/68e2d6f0889e5365ecdc8f01ab5e3bf6065cbeeb/extensions/googlechat/src/monitor.ts#L173)

- **Thread-suffixed session helper exists but is not wired into Google Chat inbound routing:** OpenClaw has a generic `resolveThreadSessionKeys` helper that would produce `:thread:<threadId>` session keys when used. Current Google Chat inbound routing does not call this helper, which aligns with the missing feature on `main`.
  - file: [src/routing/session-key.ts:237](https://github.com/openclaw/openclaw/blob/68e2d6f0889e5365ecdc8f01ab5e3bf6065cbeeb/src/routing/session-key.ts#L237)
