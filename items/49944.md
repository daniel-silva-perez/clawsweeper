## Summary

Keep open. Current `main` still hardcodes Control UI chat tool auto-expansion off, so `verboseDefault=full` is not wired through to default tool-card expansion. The existing tool-expansion plumbing is present, but the Control UI caller never enables it.

## Evidence

- **Control UI caller forces tool auto-expand off:** `renderChat(...)` is invoked with `autoExpandToolCalls: false`, so the Control UI never derives expansion from session verbosity.
  - file: [ui/src/ui/app-render.ts:2241](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/ui/src/ui/app-render.ts#L2241)
  - command: `nl -ba ui/src/ui/app-render.ts | sed -n '2200,2265p'`
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **Chat view only follows the passed prop:** `renderChat` syncs expansion state from `Boolean(props.autoExpandToolCalls)` and passes that same boolean into `renderMessageGroup`; it does not read `activeSession.verboseLevel` for tool expansion.
  - file: [ui/src/ui/views/chat.ts:810](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/ui/src/ui/views/chat.ts#L810)

- **Expansion state helper would support the requested behavior if enabled:** `syncToolCardExpansionState()` initializes both inline tool cards and standalone tool-message disclosures from the `autoExpandToolCalls` flag, including flipping already-visible cards open when that flag turns on.
  - file: [ui/src/ui/chat/tool-expansion-state.ts:24](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/ui/src/ui/chat/tool-expansion-state.ts#L24)
