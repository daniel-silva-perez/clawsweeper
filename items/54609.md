## Summary

Current main already supports per-account Feishu routing through top-level `bindings[].match.accountId`; the reported `channels.feishu.accounts.*.agentId` approach is not part of the supported Feishu account schema.

## Evidence

## Close Comment

Closing as implemented on main. Codex review of current main found that Feishu inbound routing is already account-aware (`extensions/feishu/src/bot.ts:590`) and the router has explicit account-scoped binding support (`src/routing/resolve-route.ts:654`, with coverage in `src/routing/resolve-route.test.ts:486`). The supported way to map one Feishu account to one agent is top-level `bindings[].match.accountId`, not `channels.feishu.accounts.<id>.agentId`; the Feishu account schema is strict and does not define an `agentId` field (`extensions/feishu/src/config-schema.ts:191`). The docs now state this directly in `docs/gateway/config-channels.md:736`. If this is still happening on a current build with explicit bindings like `{ agentId: "dragon-writer", match: { channel: "feishu", accountId: "dragon-writer" } }`, please open a new issue with the exact `bindings` block and a fresh log excerpt.
