## Summary

Keep PR #58426 open. Current main still persists unversioned CLI ClawHub installs as resolved versioned specs, and `plugins update` reuses that recorded spec, so the user-visible pinning bug described by the PR remains. This is core CLI install/update metadata behavior, not a ClawHub/plugin-discovery item.

## Best Possible Solution

Keep PR #58426 open and merge or adapt the focused CLI fix on main: persist `raw` for direct `clawhub:` installs, persist `preferredClawHubSpec` for bare-name ClawHub fallback, keep explicit `clawhub:pkg@version` installs pinned, update the CLI install tests accordingly, and remove or split the unrelated doctor-test change before final maintainer validation.
