## Summary

Keep open. Current `main` still has the reported Feishu HTTP server cleanup race: `stopFeishuMonitorState()` calls `server.close()` and immediately deletes/clears `httpServers` without waiting for the close callback, and the webhook transport cleanup path has the same pattern. I did not find a merged fix or test coverage for the HTTP server cleanup path, and the latest release tag `v2026.4.22` still contains the same code.

- The lack of HTTP server cleanup tests increases the chance of regressions even if a fix lands later.
