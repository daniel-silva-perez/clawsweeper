## Summary

Keep #49707 open. Current main still treats all-zero provider usage as missing data, and the submitted PR remains incomplete because later usage aggregation, persistence, and status fallback paths still require positive token counts.

## Best Possible Solution

Keep the item open, but require a refreshed fix that distinguishes "usage was observed" from "usage has positive counters" through embedded subscribe, usage accumulation, session persistence, and status fallback. Add focused regression coverage for a provider response shaped like `{ input: 0, output: 0, total: 0 }` proving `/status`/session metadata can show a fresh zero total instead of `?`.
