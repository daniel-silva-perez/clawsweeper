## Summary

Keep open. Current main ships the Slack opt-in workaround `channels.slack.thread.requireExplicitMention`, but it does not make `requireMention: true` suppress implicit Slack thread mentions by itself. The issue’s exact reported behavior still matches current main unless the extra thread flag is enabled.

## Best Possible Solution

Keep the issue open for explicit maintainer product judgment. The concrete implementation path is to decide whether Slack `requireMention: true` should imply `allowedImplicitMentionKinds: []` for thread replies, then update the Slack gating tests, docs, and changelog. If maintainers intentionally want the current additive option instead, this issue should be manually closed with that rationale and a pointer to `channels.slack.thread.requireExplicitMention`.
