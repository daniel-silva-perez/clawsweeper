## Summary

Keep open. Current `main` clearly added a Telegram/media fix for trusted explicit local proxies, but the broader issue is not fully resolved in core: `web_fetch` still intentionally keeps strict DNS pinning for untrusted URLs even when `HTTP_PROXY` is set, with only a narrow RFC2544 fake-IP opt-in, and the WeChat path lives in the external `openclaw-weixin` plugin rather than this repo.

## Evidence

- **Telegram SSRF guard now has a trusted explicit-proxy path:** `fetchWithSsrFGuard` now special-cases trusted explicit proxies: it validates the operator-configured proxy host separately, clears the target-scoped hostname allowlist for the proxy host when `allowPrivateProxy` is true, and skips target DNS pinning only after hostname-policy checks.
  - file: [src/infra/net/fetch-guard.ts:174](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/src/infra/net/fetch-guard.ts#L174)
  - command: `nl -ba src/infra/net/fetch-guard.ts | sed -n '174,206p'; nl -ba src/infra/net/fetch-guard.ts | sed -n '348,366p'`
  - sha: [ecac696643c6](https://github.com/openclaw/openclaw/commit/ecac696643c604c6d854341bfe6424bfa31423e9)
- **Telegram/media callers opt into that proxy-side DNS mode:** Core media fetching switches to `trusted_explicit_proxy` when `trustExplicitProxyDns` is set, and Telegram builds explicit-proxy dispatcher policies with `allowPrivateProxy: true`. Telegram media SSRF policy also now allows RFC2544 benchmark-range answers by default, with an additional dangerous opt-in for broader private-network cases.
  - file: [src/media/fetch.ts:132](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/src/media/fetch.ts#L132)

  - sha: [ecac696643c6](https://github.com/openclaw/openclaw/commit/ecac696643c604c6d854341bfe6424bfa31423e9)
- **`web_fetch` is still intentionally strict for untrusted URLs:** `web_fetch` calls the guarded fetch without `useEnvProxy: true`; it only passes an optional `allowRfc2544BenchmarkRange` SSRF override. The test suite explicitly asserts that untrusted `web_fetch` URLs keep DNS pinning and do not use `EnvHttpProxyAgent` even when `HTTP_PROXY` is configured.
  - file: [src/agents/tools/web-fetch.ts:414](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/src/agents/tools/web-fetch.ts#L414)
