## Summary

Keep open. Current main has the surrounding auth-profile rotation, cooldown, lastGood, and ingress-authorization work, but it still does not implement this PR's specific final-profile persistence contract: embedded results do not return a final auth profile id, and session-store persistence after a successful run does not update authProfileOverride to the winning rotated profile.

## Best Possible Solution

Keep the item open as a useful but not merge-ready fix. The best path is a narrow rebase against current main that propagates the final winning auth profile from the embedded run or attempt layer into post-run session persistence, updates authProfileOverride while preserving a user-pinned source only if that behavior is still intended, and adds regression tests for successful fallback winner persistence without regressing the current allowModelOverride guard.
