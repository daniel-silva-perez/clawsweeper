## Summary

Keep PR #52571 open. Current main still has the session-key-only reminder guard, the linked bug #52528 was closed as a duplicate of this PR, and the PR remains the canonical open fix attempt. It is not ready to merge as-is because the raw PR diff contains the U+0008 regex bug and review feedback about `sessionTarget` filtering/negative coverage remains actionable.

## Best Possible Solution

Keep #52571 open. The best path is to rebase/update the PR, fix the literal U+0008 regex to use real word-boundary escapes or a safer matcher, require `job.sessionTarget === "isolated"` for sweep-like suppression, add negative coverage for unrelated isolated cron jobs, then run the changed auto-reply/cron test lane before merging. If maintainers prefer the broader isolated-cron solution in #43404, explicitly supersede #52571 there rather than silently closing this canonical #52528 fix attempt.
