## Summary

Keep open. Current `main` still has a plausible loss window for gateway-managed session transcripts during `SIGTERM` shutdown: the run loop only drains active work for `SIGUSR1` restart, while `SIGTERM` goes straight through the stop path, and assistant transcript mirroring still happens after outbound delivery succeeds. The reporter's specific `writeTextAtomic(...)` explanation does not match current transcript-write code, but the underlying bug report remains credible on current `main`.

## Evidence

- **SIGTERM uses stop path; drain logic is restart-only:** `SIGTERM` calls `request("stop", "SIGTERM")`, while the active-task/active-run drain block is guarded by `if (isRestart)`. That means the extra wait-for-drain behavior is not used on `SIGTERM` shutdowns.
  - file: [src/cli/gateway-cli/run-loop.ts:145](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/cli/gateway-cli/run-loop.ts#L145)

  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Delivered outbound replies are mirrored into transcripts only after send succeeds:** `deliverOutboundPayloads` appends the assistant mirror only after `results.length > 0`, so there is a post-delivery/pre-persist window if shutdown interrupts the process.
  - file: [src/infra/outbound/deliver.ts:1004](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/infra/outbound/deliver.ts#L1004)
