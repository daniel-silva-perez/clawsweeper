## Summary

Keep PR #70040 open. Current main still has only English/Czech memory-lancedb auto-capture triggers and category detection, so the requested Chinese trigger support is not implemented on main. The PR is also not cleanly superseded: the related Chinese/CJK PRs called out by PRtags are closed and unmerged in the provided context. The best maintainer path is to keep this as useful but incomplete work and request a fuller patch that handles short Chinese inputs plus tests.

## Best Possible Solution

Keep the PR open and ask for changes: add Chinese/CJK trigger coverage in `extensions/memory-lancedb/index.ts`, adjust the length gate or trigger ordering for short high-signal Chinese inputs, extend `detectCategory`, and add focused tests in `extensions/memory-lancedb/index.test.ts`. The prior closed PRs #16392, #16672, and #20241 are useful references but are not canonical shipped replacements.
