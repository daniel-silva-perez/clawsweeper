## Summary

Keep open. Current main still treats Feishu `card.action.trigger` callbacks as side-effect-only: the registered handler does not return a callback response payload, `handleFeishuCardAction` is typed and implemented as `Promise<void>`, and structured quick/confirm actions await synthetic command dispatch before completing. The issue is recent, coherent, externally reported, and describes a plausible UX feature gap rather than something already implemented or stale.
