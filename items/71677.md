## Summary

Current main still does not implement the requested Feishu instant card update path. The bundled Feishu card.action.trigger handler returns no callback payload, handleFeishuCardAction is still Promise<void> and awaits synthetic command dispatch, and the ocf1 interaction envelope has no disable-state field for transitional disabled buttons.

## Best Possible Solution

Keep the issue open and implement it in the bundled Feishu plugin: add a typed opt-in ocf1 disable-state or processing-state field, have card.action.trigger return an immediate Feishu card callback response while dispatching the synthetic command asynchronously, preserve the existing duplicate-token guard, and add focused tests for returned WebSocket/webhook callback payloads plus current quick-action and approval card regressions.
