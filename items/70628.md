## Summary

Keep open. Current main still matches the reported Telegram DM bug: the no-visible-response/no-error path synthesizes `NO_REPLY`, runs it through the outbound silent-reply rewrite policy, and delivers it as a visible Telegram reply. The existing regression test still asserts that buggy DM rewrite behavior, while the proposed fix is only an open PR (#70630), not shipped on main.

## Best Possible Solution

Keep #70628 open as the canonical bug until #70630 or an equivalent fix lands. The fix should make successful no-visible Telegram direct turns a handled no-op, avoid sending a synthetic transport-visible `NO_REPLY`, preserve the real error fallback path, mark status reactions as done for the silent success case, and flip the current Telegram regression test to assert no chat message is sent.
