## Summary

This issue should stay open. Current main still synthesizes and delivers a Telegram DM `NO_REPLY` fallback for successful no-visible-response turns, and open PR #70630 from the same author uses closing syntax to fix exactly that behavior.

## Best Possible Solution

Keep this issue open and review or land PR #70630, or apply an equivalent narrow Telegram adapter fix. The fix should treat successful no-visible-response Telegram direct turns as handled silent no-ops, avoid synthesizing transport-visible `NO_REPLY`, preserve the real dispatch-error and non-silent-delivery-failure fallback paths, and flip the Telegram dispatch regression test to assert no DM delivery for this successful silent case.
