## Summary

Current main still does not expose a documented native `sessions_spawn` policy to suppress or redirect sub-agent completion announcements. The only caller-visible parent routing is ACP-only `streamTo: "parent"`; native subagent runs still default to completion announce delivery for channel-bound requester sessions.

## Best Possible Solution

Keep this open and implement a documented native `sessions_spawn` completion-delivery policy, likely `completionDelivery: "channel" | "parent" | "none"` or equivalent. Thread it through the tool schema, native spawn params, registry/task delivery state, announce dispatch, docs at https://docs.openclaw.ai/tools/subagents, and focused regression tests for channel-bound requester sessions such as BlueBubbles/iMessage, WhatsApp, Telegram, and Discord.

## Risks / Open Questions

- A hidden `expectsCompletionMessage: false` path exists, but it is not advertised in the tool schema or docs and only offers a binary internal switch, not explicit `channel | parent | none` semantics.
- The implementation needs to preserve current channel delivery as the default and avoid regressing ACP `streamTo: "parent"`, thread-bound session delivery, and channel-bound completion behavior covered by existing tests.
