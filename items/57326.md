## Summary

Keep #57326 open. Current main still contains the exact helper dispatch gaps named in the issue: queued followups, models auth probes, and LLM slug generation still call the embedded runner directly, while cron still checks `isCliProvider()` before resolving canonical providers to configured CLI runtimes.

## Best Possible Solution

Keep #57326 open. Apply the existing `resolveCliRuntimeExecutionProvider(...) ?? provider` plus `isCliProvider(...)` dispatch pattern to queued followups, `models status --probe`, LLM slug generation, and cron canonical-provider runtime selection. Route CLI-backed helper runs through `runCliAgent()`, preserve active runtime config/auth/session context, and add focused regression tests for those helper paths. Also review startup prewarm so configured CLI runtimes do not emit spurious harness warnings or failures.
