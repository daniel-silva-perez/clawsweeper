## Summary

Issue #57326 still matches current `main`: the three helper surfaces named in the report still dispatch through `runEmbeddedPiAgent()` instead of the CLI runner, so there is not enough evidence to close this as fixed or obsolete.

## Evidence

- **Queued followups still use embedded runner:** `createFollowupRunner()` imports `runEmbeddedPiAgent` and its fallback execution path calls `runEmbeddedPiAgent(...)` directly for followup turns. The file only uses `isCliProvider()` later for usage accounting, not for dispatch.

## Risks / Open Questions

- CLI-backed default models can still be misrouted for queued followups, `openclaw models status --probe`, and LLM slug generation on current `main`.
