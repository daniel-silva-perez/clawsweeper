## Summary

Keep this issue open. Current main still resolves exec approvals file and socket paths through home-based `~/.openclaw` defaults instead of `OPENCLAW_STATE_DIR`, and open PR #62938 remains an unmerged implementation candidate with closing syntax for the same bug.

## Best Possible Solution

Keep this issue open and review, update, land, or replace PR #62938. The code fix should live in `src/infra/exec-approvals.ts`, resolving approvals file and socket paths under `OPENCLAW_STATE_DIR` / `resolveStateDir()` first, then falling back to the existing `OPENCLAW_HOME` and default-home behavior. Add focused regression coverage in `src/infra/exec-approvals-store.test.ts` for state-dir precedence while preserving the current symlink-safety tests.
