## Summary

Keep open. Current main still resolves exec approvals file/socket paths through the home-based `~/.openclaw` defaults and does not check `OPENCLAW_STATE_DIR`, while the linked exact-fix PR #62938 remains open and unmerged.

## Best Possible Solution

Keep #62917 open until #62938 or an equivalent patch lands. The likely fix belongs in `src/infra/exec-approvals.ts`: resolve `OPENCLAW_STATE_DIR` before `OPENCLAW_HOME` for `exec-approvals.json` and `exec-approvals.sock`, preserve the existing home fallback, and add regression coverage for state-dir precedence plus the doctor split-state scenario.
