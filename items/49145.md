## Summary

Keep #49145 open. Current main has only partial Signal quote-context handling: it can expose visible quote body/sender, but it does not carry `ReplyToId`, does not wake the heartbeat after Signal reactions, and does not send Signal `quote-timestamp`/`quote-author` for outbound quoted replies. The PR remains a live implementation candidate, though it needs maintainer review against the many prior review comments before merge.

## Best Possible Solution

Keep #49145 open. The best path is to rebase and review the focused Signal implementation against current main, preserving the lazy Signal send-runtime boundary and adding/retaining regression tests for inbound `ReplyToId`, `authorNumber`/UUID quote sender handling, reaction wake via a supported SDK export, outbound `quote-timestamp`/`quote-author`, group replies, retry/payload author propagation, and first-chunk-only quoted sends.
