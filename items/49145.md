## Summary

Keep #49145 open. Current main has some Signal quote body/sender handling, but it does not implement the PR’s central remaining capabilities: inbound `ReplyToId` from `dataMessage.quote.id`, reaction-triggered heartbeat wake, outbound Signal quote parameters, or first-chunk-only quoted send behavior. The provided PR discussion also shows #49145 supersedes earlier closed/unmerged attempts (#47368, #40292, #47633), so those related items do not supersede the open work.

## Best Possible Solution

Keep #49145 open as the canonical Signal reply-context PR unless a maintainer chooses to split it. The best path is to land a focused implementation with tests for `dataMessage.quote.id`/`authorNumber`/`authorUuid`, visibility-gated `ReplyToId`, reaction `requestHeartbeatNow` wiring through an appropriate SDK seam, Signal RPC `quote-timestamp`/`quote-author` send params, and first-chunk-only quote behavior across text, media, payload, queue/retry, and group-author paths.
