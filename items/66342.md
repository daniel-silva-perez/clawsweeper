## Summary

Keep open. Current main already isolates channel start failures from taking down later channels, but it does not implement this PR's Slack-specific graceful-stop behavior: Slack socket auth failures and retry exhaustion still throw out of `monitorSlackProvider`. The PR is a focused, active bug fix, but the return-based patch still needs maintainer review because it can bypass the gateway catch path that records `lastError`.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to either revise the Slack provider to publish an explicit terminal failure status before resolving on non-recoverable auth errors and retry exhaustion, with regression coverage for startup auth, disconnect auth, and max-attempt paths, or have a maintainer explicitly close it as obsolete if the current generic gateway isolation is considered sufficient.
