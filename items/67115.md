## Summary

Keep this PR open. Current main still has the macOS regression described by the linked bug: `resolveBrowserOpenCommand()` returns `ssh-no-display` before it reaches the Darwin `open` path whenever SSH environment variables are present and no display variables are set. The PR is a narrow implementation candidate for that exact condition, but it should get a regression test before landing.

## Best Possible Solution

Review and land this PR or an equivalent narrow patch, ideally with a focused `resolveBrowserOpenCommand()` regression test proving that Darwin with SSH variables and no display variables still resolves to `open`, while Linux without a display continues to return `ssh-no-display` or `no-display` as intended.
