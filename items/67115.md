## Summary

Keep this PR open. Current main still returns `ssh-no-display` before the Darwin `open` branch when SSH environment variables are present and no display variables are set, so the linked macOS dashboard regression is not implemented on main. The PR is a narrow implementation candidate, with no obvious supply-chain or secret-handling risk, but it should get a focused regression test before landing.

## Best Possible Solution

Review and land this PR or an equivalent narrow patch, ideally with a focused `resolveBrowserOpenCommand()` regression test proving that Darwin with SSH variables and no display variables resolves to `open`, while Linux without a display continues to return `ssh-no-display` or `no-display` as intended.
