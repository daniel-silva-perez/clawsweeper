## Summary

Keep open. Current `main` only added recognition for Edge LaunchServices bundle IDs, but it still returns `null` when `osascript` or `defaults` cannot resolve the app path/executable, so the known-path fallback proposed by PR #48606 is not implemented on `main`.

## Evidence

- **Current macOS detection still hard-fails on lookup failure:** `detectDefaultChromiumExecutableMac()` accepts `com.microsoft.edgemac*`, then immediately returns `null` if `/usr/bin/osascript` returns no app path or if `/usr/bin/defaults` returns no executable name. There is no fallback map or alternate known-path resolution in current `main`.
