## Summary

Keep PR #48593 open. Current main still has the inconsistent narrow `<final>` stripping paths that the PR targets, so the PR is not obsolete or implemented-on-main. The PR also incorporates useful follow-up discussion about self-closing tags and attributed `<think>` tags, but it should be revised before merge because the widened `\b[^<>]*` pattern has a reviewed false-positive risk for hyphenated custom tags.

## Best Possible Solution

Keep this PR open or replace it with an equivalent maintainer patch. The best fix is to update all final/thinking tag scanners used on delivered-message paths to a shared, exact tag-name matcher that supports attributes and self-closing tags without matching hyphenated custom tags, then add focused regression tests for streaming `enforceFinalTag`, non-enforced streaming cleanup, `sanitizeUserFacingText`, and the live helper regex. Keep #41494 as the umbrella for undelimited/plain-text Gemini reasoning leaks.
