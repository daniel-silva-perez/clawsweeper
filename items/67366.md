## Summary

Keep open. Current main still has the same unsafe prompt-result trimming on the Telegram token replacement path, and the related fix PRs in the provided context were closed unmerged rather than shipped.

## Best Possible Solution

Keep this issue open. Patch the shared wizard prompt handling so every `prompter.text` result is normalized before trimming, especially `promptSingleChannelToken` and the generic `rawValue` text-input path. Add a regression test that mocks undefined from the Telegram token replacement prompt and verifies onboarding or `openclaw channels add` does not throw, then close after the fix merges and ships.
