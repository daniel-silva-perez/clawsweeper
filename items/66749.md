## Summary

Keep this PR open. Current main still has the behavior the PR targets: both exec completion paths enqueue the event and then unconditionally request an immediate `exec-event` heartbeat wake, while heartbeat resolution maps forced subagent session keys back to the main session. The PR is also paired with an open same-author issue via closing syntax, so it should be reviewed or rejected on its merits rather than cleanup-closed.

## Best Possible Solution

Keep this PR open for maintainer review. The likely best code path is a narrow guard that skips `requestHeartbeatNow` for `isSubagentSessionKey(sessionKey)` in both exec-event call sites while preserving `enqueueSystemEvent`, plus regression coverage for both explicit exec system events and background notify-on-exit. If accepted, landing this PR or an equivalent patch should close the linked issue.
