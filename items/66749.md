## Summary

Keep this PR open. Current main still has the exact behavior the PR targets: both exec completion paths enqueue the system event and then unconditionally call `requestHeartbeatNow` with an `exec-event` wake, while heartbeat session resolution deliberately maps subagent session keys back to the main session. The PR is also paired with an open same-author issue via closing syntax, so it should be reviewed or rejected on its merits rather than cleanup-closed.

## Best Possible Solution

Keep the PR open for maintainer review. The likely product/code path is to land a narrow guard that skips `exec-event` heartbeat wakes for `isSubagentSessionKey(sessionKey)` while preserving `enqueueSystemEvent`, with regression coverage for `emitExecSystemEvent` and preferably `maybeNotifyOnExit` as well. If that approach is accepted and passes the targeted agent tests, landing this PR or an equivalent patch should close the linked issue.
