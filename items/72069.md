## Summary

Keep this PR open. Current main still lacks cross-provider fallback prompt seeding from Claude CLI session history, and this PR is a focused implementation candidate for the concrete continuity bug reported in #69973.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or an equivalent implementation that seeds a bounded Claude CLI summary/recent-turn prelude only when a fallback chain started on claude-cli and retries on another runtime, with regression coverage in `src/agents/command` and `src/gateway`.

## Evidence

- **Linked report and discussion describe a real fallback continuity bug:** The linked bug report #69973 includes a concrete claude-cli -> openai-codex fallback path after a 402 billing error, notes that Claude's local JSONL still had the fuller history, and the PR body records steipete's triage recommending that fallback prompts be seeded from prior transcript/compacted context.
- **Current retry prompt keeps the cold fallback behavior when OpenClaw has no session history:** `resolveFallbackRetryPrompt` only accepts `body`, `isFallbackRetry`, and `sessionHasHistory`; if `sessionHasHistory` is false it returns the original body with no prior context prelude. That is the claude-cli cold fallback case described in #69973.
