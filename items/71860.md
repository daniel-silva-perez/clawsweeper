## Summary

Keep PR #71860 open. Current main and the latest release still treat successful structured exec completions as relayable heartbeat events, and tests still encode visible delivery for `Exec completed (..., code 0)`. Related #72253 overlaps and may become the better consolidation path, but it is still open and there is no maintainer/canonical signal strong enough to close this focused, recently updated PR as superseded.

## Best Possible Solution

Keep #71860 open for maintainer review. The best path is to decide the heartbeat exec-completion contract, then either land/refine this narrow successful-code-0 suppression with its regression tests or explicitly consolidate it into #72253 if maintainers choose that broader PR as canonical.
