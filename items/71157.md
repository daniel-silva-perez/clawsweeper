## Summary

Current main does not implement WhatsApp outbound disappearing-message expiration. Baileys supports `ephemeralExpiration` on `sendMessage` options, but OpenClaw's WhatsApp config schema has no `disappearingMessagesSeconds` field, the typed active-send options cannot carry an expiration, and the text/media/poll send paths call Baileys without that option. The provided related items cover WhatsApp quoting, mentions, and other outbound delivery fixes, not this expiration request.

## Best Possible Solution

Keep #71157 open. The right implementation belongs in the bundled WhatsApp plugin/config surface: add channel-level and account-level expiration config, resolve account override before sending, pass positive values as Baileys `ephemeralExpiration` through text, media, quote, poll, fallback, and message-tool paths, and add schema/docs/tests covering unset, zero/null, account override, media, quoted replies, and polls.
