## Summary

Keep this issue open. Current main still does not expose a WhatsApp channel/account disappearing-message expiration setting, does not resolve an effective per-account value, and does not thread Baileys `ephemeralExpiration` through the WhatsApp outbound send paths. The existing ClawSweeper comment and related reports point to quote/mention/delivery work, not this expiration feature.

## Best Possible Solution

Implement this in the bundled WhatsApp plugin: add channel-level and account-level `disappearingMessagesSeconds` to config types, strict Zod schema, generated metadata/help, and docs; resolve a positive effective value with account override over channel default and `0`/`null`/omitted meaning no OpenClaw override; thread that value through active send and poll options; merge it into Baileys message generation options for normal text, quoted replies, media, voice-note companion text, payload/chunked/fallback sends, and polls; add focused tests for configured, account override, unset, quoted, media/voice-note, payload/chunked, and poll paths.
