## Summary

Current main still contains the reported Discord explicit-target bug. `parseDiscordExplicitTarget` parses `user:<id>` correctly but returns the bare `target.id`, and the later Discord outbound normalizer treats bare numeric IDs as `channel:<id>`. The provided GitHub context also shows open PR #72580 explicitly closing this issue, so the issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land PR #72580, or apply the same narrow Discord plugin fix directly: return `target.normalized` from `parseDiscordExplicitTarget` while preserving bare numeric IDs as channel targets. Add regression coverage through `discordPlugin.messaging.parseExplicitTarget` for `user:<id>` and a cron or heartbeat delivery-target path that proves the prefixed DM target survives to outbound delivery.
