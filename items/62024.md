## Summary

Keep open. Current main still contains the reported Discord explicit-target bug: the plugin parses `user:<id>` correctly but then returns the bare id, allowing later outbound normalization to treat it as a channel. An open implementation candidate, PR #72580, explicitly tracks this issue and is not merged, so the issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Review and land PR #72580, or apply the equivalent narrow fix in the Discord plugin: return `target.normalized` from `parseDiscordExplicitTarget` while preserving the existing bare-numeric-as-channel behavior. Add focused regression coverage through `discordPlugin.messaging.parseExplicitTarget` and at least one cron or heartbeat delivery-target path for `user:<id>`. After the fix merges, close this issue as implemented by the merged commit.
