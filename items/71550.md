## Summary

Keep this PR open. Current main still does not expose or forward `channels.slack.logLevel`, and the PR is a focused implementation candidate for #71531. It is not ready to treat as obsolete: the latest patch fixes the earlier lowercase/type-contract concern, but dependency source review shows OpenClaw passes a preconstructed `SocketModeReceiver`, while Bolt only applies `App.logLevel` to an internally-created receiver. The PR should be adjusted to pass the level into the receiver/socket-mode path that emits the pong-timeout WARN.

## Best Possible Solution

Keep this PR open as the active implementation path for #71531. Update the runtime wiring so `channels.slack.logLevel` is a lowercase config contract and is passed to the explicitly constructed `SocketModeReceiver` or shared receiver logger that owns `SocketModeClient` and `SlackWebSocket` output; forwarding to `App` can remain if useful but should not be the only path. Add tests that assert configured `error` reaches the socket receiver and that omitted config preserves defaults, then update Slack/config docs or generated config metadata as required. Keep #72808 separate because it is about stale/silent Slack disconnect behavior, not log-noise suppression.
