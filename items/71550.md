## Summary

Keep #71550 open. Current main still does not expose or forward `channels.slack.logLevel`, and the linked PR/issue describe a real missing Slack config surface rather than obsolete cleanup. The PR likely needs one more review pass because OpenClaw constructs a custom `SocketModeReceiver`, so forwarding only to Bolt `App` may not silence socket-mode warnings.

## Best Possible Solution

Keep #71550 open as the active implementation path for #71531. The best maintainer path is to revise the PR so `channels.slack.logLevel` uses Bolt-compatible lower-case values, is represented in schema/types/docs, is forwarded to the custom `SocketModeReceiver` as well as Bolt/App or WebClient paths as needed, and has tests proving configured and omitted behavior before merge.
