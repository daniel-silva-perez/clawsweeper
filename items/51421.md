## Summary

Keep #51421 open. Current main still persists `memoryFlushCompactionCount` with the post-increment compaction count, while the current gate skips when that value equals `compactionCount`. The related #69138 PR is closed unmerged and #12590 remains open, so there is no shipped or canonical replacement for this fix.

## Best Possible Solution

Keep the PR open or land an equivalent focused fix: preserve the pre-increment `memoryFlushCompactionCount`, update the current `agent-runner-memory.test.ts` expectation to the pre-increment value, keep/add the consecutive compaction-cycle regression coverage, then close #12590 when the fix ships.
