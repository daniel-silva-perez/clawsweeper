## Summary

Keep this PR open. Current main does not contain the proposed early Signal typing indicator, and the group-history pre-dispatch work described in the PR still exists. The PR is still a plausible implementation candidate, but it needs maintainer review/rework because the proposed unconditional pre-dispatch send would bypass existing typing and outbound-delivery suppression contracts.

## Best Possible Solution

Keep the PR open and rework it into a policy-aware implementation. The cleanest path is to start Signal typing earlier through the dispatch/reply pipeline after route/session `sendPolicy` and `typingMode` have been resolved, or add an equivalent Signal-local guard that respects `sendPolicy: "deny"`, `typingMode: "never"`, and the intended group/DM scope. Add regression coverage proving early group feedback happens while denied or disabled sessions do not emit typing.
