## Summary

Keep PR #51125 open. Current main still lacks Cyrillic/Greek confusable-letter normalization for external-content boundary markers, and the PR discussion shows the proposed patch still needs at least one coverage fix before it would be merge-ready.

## Best Possible Solution

Keep the PR open, rebase it onto current `src/security/external-content.ts`, add a complete confusable-letter map used by `foldMarkerChar`, derive any regex coverage from that map if needed, and add regression tests for uppercase/lowercase Cyrillic and Greek marker spoofs including Cyrillic lowercase `т`.
