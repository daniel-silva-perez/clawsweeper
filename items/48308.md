## Summary

Keep open. Current `main` still does not automatically pass Telegram thread/topic information into plugin tool params, and the public plugin tool context still has no dedicated `messageThreadId`/`topicId` field. The thread id is only preserved inside nested `deliveryContext`, while plugin commands already get a first-class `messageThreadId` field.

## Evidence

## Risks / Open Questions

- A plugin could potentially work around this by reading `ctx.deliveryContext?.threadId`, but that is not the issue's requested OpenClaw behavior and does not satisfy automatic param injection for tools that declare `topicId`/`messageThreadId`.
