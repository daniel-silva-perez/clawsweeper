## Summary

Keep #70578 open. Current main has partial guardrails for the silent-loss path, including preserving source-only keys during write projection, rejecting destructive size drops, and adding newly installed plugins to restrictive allowlists. But meaningful requested behavior remains unresolved: the root config schema is still strict and tests still expect unknown top-level fields to be invalid, and bundled plugin CLI command surfaces remain gated by `plugins.allow`.

## Best Possible Solution

Keep the issue open for maintainer judgment. The best path is to split the remaining work into explicit decisions: either implement and test root-level unknown-field preservation across config round-trips, or document strict root validation as intended fail-closed behavior; separately decide whether bundled plugin CLI surfaces should stay governed by `plugins.allow` or get a narrow exemption. Add regression coverage around plugin-install/config-write no-clobber behavior either way.
