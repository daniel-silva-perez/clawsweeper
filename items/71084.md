## Summary

Keep #71084 open. Current main still skips `memoryFlush` and `preflightCompaction` for CLI-backed providers in the shared memory runner, and the colocated test/docs explicitly lock in that behavior. PR #71081 proposed removing those gates but was closed unmerged in favor of this discussion. Current main also has a separate CLI transcript compaction lifecycle, but that is a post-turn CLI-specific compaction path, not the preflight memory flush behavior requested here.

## Best Possible Solution

Leave #71084 open for explicit maintainer product/architecture direction. If maintainers want parity, implement a CLI-aware memory flush/preflight compaction path or carefully remove the CLI gates in `src/auto-reply/reply/agent-runner-memory.ts`, update `agent-runner-memory.test.ts`, and revise `docs/reference/session-management-compaction.md` / config docs. If maintainers want CLI backends to keep using only the separate transcript compaction lifecycle, answer the issue with that rationale and keep the docs/tests as the policy proof.
