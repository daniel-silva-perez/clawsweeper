## Summary

Current main already supports unified DM/session continuity across channels: direct messages share the main session by default, and `session.identityLinks` explicitly links provider-specific identities for cross-channel session sharing. This was also documented as shipped in the `2026.1.16-1` changelog.

## Risks / Open Questions

- The existing implementation covers shared cross-channel session history, but automatic identity discovery/linking is not implied; multi-user setups still rely on `session.dmScope` and optional `session.identityLinks` configuration.

## Close Comment

Closing after Codex review because this is already implemented on current `main`. OpenClaw already shares DM history by default (`docs/concepts/session.md:24-25`, `src/config/sessions.test.ts:166-184`), and for the same person across multiple channels it supports explicit cross-platform linking via `session.identityLinks` (`docs/concepts/session.md:50-52`, `src/routing/session-key.ts:145-170`, `src/routing/resolve-route.test.ts:195-210`). The changelog shows this shipped in `v2026.1.16-1` (`CHANGELOG.md:6147-6155`). If the remaining request is for automatic identity matching/linking UX rather than the underlying capability, please open a new issue scoped to that gap.
