## Summary

Current `main` does not contain the PR‚Äôs date-rollover fix. `createSubsystemLogger` still caches a subsystem file child logger forever, so long-running processes can keep writing through a stale child after the root logger‚Äôs daily rolling path changes.

## Evidence

- **Subsystem child logger is still one-time cached:** `createSubsystemLogger` keeps `let fileLogger = null` and both `emitLog` and `raw` only rebuild it on `!fileLogger`. There is no midnight or settings-change check before reusing that child.
  - file: [src/logging/subsystem.ts:325](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/logging/subsystem.ts#L325)
