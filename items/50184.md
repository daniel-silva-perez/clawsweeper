## Summary

Keep open. Current `main` still forces Telegram DM reply previews onto message transport in the upper reply layer, which overrides the lower-level DM-prefers-draft behavior. The latest release snapshot shows the same logic, and repo tests currently lock that behavior in, so this issue is neither implemented nor obsolete.
