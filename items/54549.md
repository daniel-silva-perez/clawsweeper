## Summary

Keep open. Current `main` already supports background exec completion wakes in the general case, but it does not implement the PR's proposed per-call `onComplete="notify"` contract, and the docs still describe quiet-success / no-wake cases as requiring `process` follow-up. That means the request is not already implemented on `main` well enough to close conservatively.

## Evidence

- **No per-call `onComplete` parameter on main:** The current exec tool schema exposes `background`, `yieldMs`, `timeout`, `pty`, `host`, `security`, `ask`, `node`, and `elevated`, but no `onComplete` field.
  - file: [src/agents/bash-tools.schemas.ts:3](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/bash-tools.schemas.ts#L3)

- **Immediate backgrounding already exists via `background=true`:** `createExecTool` only checks `params.background === true` for immediate backgrounding, then resolves the running-session result when `yieldWindow === 0`. This is existing behavior, not the proposed new parameter.
  - file: [src/agents/bash-tools.exec.ts:1398](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/bash-tools.exec.ts#L1398)
