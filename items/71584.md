## Summary

Keep #71584 open. Current main still has the reported local validation gap for `ollama/deepseek-v4-flash:cloud`: `max` is a recognized canonical thinking value, but the fallback/base thinking profile for providers without a custom profile only exposes `off`, `minimal`, `low`, `medium`, and `high`. The bundled Ollama provider does not publish a `resolveThinkingProfile` advertising `max`, so UI/session validation rejects `/think max` before any Ollama request is sent.

## Best Possible Solution

Keep the issue open and fix it in the bundled Ollama provider path. A conservative maintainer fix would add an Ollama-owned thinking profile for models that support `max` (or a generic bridge from configured supported reasoning efforts if that is the intended contract), then add focused tests that `ollama/deepseek-v4-flash:cloud` lists and accepts `/think max` and sends the provider-supported highest reasoning effort without regressing existing native `think: true/false` behavior.
