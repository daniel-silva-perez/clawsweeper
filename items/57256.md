## Summary

Keep open. Current `main` still routes status memory readiness through embedding/local probe paths that can report `enabled (plugin …) · unavailable` even when a live gateway memory plugin is functioning.

## Evidence

- **Status overview still uses local memory snapshot, not live gateway memory health:** `scanStatus()` resolves the Memory row via `resolveStatusMemoryStatusSnapshot()`, which calls `getMemorySearchManager(... purpose: "status")` locally rather than asking the gateway for live runtime state.
  - file: [src/commands/status.scan.ts:69](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/commands/status.scan.ts#L69)

- **Unavailable row still rendered when local manager cannot materialize:** If no memory snapshot is returned, the UI still prints `enabled (plugin <slot>) · unavailable`. That matches the reported false-negative symptom.
  - file: [src/commands/status.command-sections.ts:153](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/commands/status.command-sections.ts#L153)
