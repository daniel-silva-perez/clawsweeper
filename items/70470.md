## Summary

Keep #70470 open. Current main still awaits WhatsApp `sock.readMessages()` for every accepted non-self inbound message before the message is handed to the agent path, and read receipts default to enabled. The related prior PR #38304 was closed unmerged and addressed a different self-chat/allowFrom read-receipt suppression case, not the reported dependency on a sender account's read-receipt setting.

## Best Possible Solution

Keep the issue open and fix the WhatsApp plugin so inbound message processing is decoupled from read-receipt delivery, for example by making `readMessages` non-blocking or time-bounded and adding a regression test where `sock.readMessages` rejects or never resolves while an accepted message still reaches the agent/calendar flow. Also revisit the unused `shouldMarkRead` field or remove it if it is not the right contract.
