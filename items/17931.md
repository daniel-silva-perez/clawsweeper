## Summary

Keep open. Current `main` does not implement the requested per-skill read-only bind mounts, so the feature request itself is still unresolved. The issue body overstates current behavior for `workspaceAccess="ro"` and `"none"`: those modes now mount `/workspace` read-only and the fs bridge blocks writes unless `workspaceAccess === "rw"`. But in `workspaceAccess="rw"`, skills still live under the writable workspace mount with no dedicated read-only overlay for `skills/*`.
