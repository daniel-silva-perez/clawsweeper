## Summary

Keep open. Current main mitigates the provider replay failure by filtering `openclaw`/`delivery-mirror` assistant rows out of in-memory replay before Anthropic turn validation, but the reported raw transcript write still exists: outbound delivery still calls `appendAssistantMessageToSessionTranscript`, which persists a `role: "assistant"`, `provider: "openclaw"`, `model: "delivery-mirror"` row. I found no current config toggle or alternation guard that disables/skips the mirror write itself.

## Best Possible Solution

Keep this issue open. The best maintainer fix is to preserve the current replay filter, then add an explicit delivery-mirror policy for raw transcript writes: either a config-backed disable path or a guard that skips/retypes transcript-only mirror entries when appending them would create an assistant-to-assistant raw transcript sequence. Add regression coverage for message-tool outbound delivery from subagent/cron-like sessions and for Anthropic replay staying clean.
