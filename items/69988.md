## Summary

Keep #69988 open. Current main still has compaction only under `agents.defaults.compaction`; it does not accept `agents.list[].compaction`, does not resolve per-agent compaction, and the Pi runtime paths still consume defaults-only settings. The PR is recent, focused, and the discussion shows active fixes for review feedback, so it is not a conservative cleanup-close candidate.

## Best Possible Solution

Keep #69988 open as the focused implementation path for per-agent compaction overrides. The best product/code direction is to land an agent-aware schema/type/resolution change that accepts `agents.list[].compaction`, deep-merges supported nested compaction objects over `agents.defaults.compaction`, threads the effective config through Pi settings, direct/manual compaction, safeguard/runtime setup, timeout and post-compaction side effects, and adds targeted schema/runtime/docs coverage. After that lands, close duplicate feature issues and older PRs against the merged PR.
