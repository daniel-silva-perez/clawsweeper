## Summary

Keep PR #71828 open. Current main still has the controller-level typing TTL expiry the PR targets: keepalive ticks call the channel typing callback, but `triggerTyping()` does not refresh `createTypingController`'s own `typingTtlMs` timer, so the controller can clean up and stop Discord typing during a long still-active run. The PR's one-line production change plus regression test remain useful rather than obsolete or superseded.

## Best Possible Solution

Keep PR #71828 open for maintainer review. The best path is to land this targeted controller TTL refresh and regression test, or deliberately fold the same behavior into a broader #65919 active-run presence implementation while preserving cleanup after run completion plus dispatch idle.
