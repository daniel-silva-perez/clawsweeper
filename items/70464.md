## Summary

Keep open. Current main has partial related fixes (#67023 timestamp normalization/fallback cleanup and #70737 isolated managed cron), but it has not implemented the central #70464 behavior: `dreaming-narrative-*` keys still include `nowMs`, `memory-core` still has no pre-run cleanup, and plugin `deleteSession()` still routes through admin-scoped `sessions.delete`. The PR also has an unresolved correctness concern from review: its proposed synthetic scope would not cover request-scoped clients unless the dispatch path is changed too.

## Best Possible Solution

Keep the PR open for maintainer review, but require a corrected, narrow patch: preserve least privilege, make memory-core dreaming cleanup work for request-scoped clients or move cleanup to an internal lifecycle-independent session cleanup API, keep idempotency distinct from the stable session key, add regression coverage for request-scoped and fallback cleanup paths, and drop unrelated runtime-deps changes. The remaining user-facing bug should continue to be tracked with #68252 until a corrected fix lands.
