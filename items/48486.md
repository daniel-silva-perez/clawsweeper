## Summary

Keep open. Current `main` still does not implement the reported near-invisible CSS checks in `sanitizeHtml()`: the sanitizer only matches exact hidden styles such as `font-size:0`, `color:transparent`, and zero-alpha colors, and there is no same-foreground/background comparison. The extraction path still feeds `sanitizeHtml()` output into `web_fetch` text extraction, so the reported gap is still present. Separately, `SECURITY.md` suggests this is better framed as defense-in-depth hardening than a boundary-bypass vulnerability, but that is not an allowed auto-close reason in this review task.

## Evidence

- **Hidden-style matcher still misses the reported cases:** `HIDDEN_STYLE_PATTERNS` only covers `display:none`, `visibility:hidden`, `opacity:0`, exact zero font sizes, `color:transparent`, and zero-alpha `rgba/hsla`. There is no threshold for tiny non-zero fonts, no near-white color handling, and no background-color parsing.
