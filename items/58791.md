## Summary

Keep PR #58791 open. Current main does not implement `registerStreamFnWrapper`, and the proposal is a plausible core Plugin SDK/runtime seam rather than an item that can be conservatively closed. The PR discussion also contains unresolved design/security concerns about wrapper placement, provider selection, invalid wrapper returns, and wrapper bypass behavior, so it needs maintainer judgment rather than cleanup closure.
