## Summary

Keep open. Current `main` contains partial mitigations for memory reliability, but not strong evidence that issue #48711 is fully fixed or obsolete. Fresh-session transcript recall is still opt-in, proactive recall is still an optional plugin, and the docs still tell users to persist durable state into workspace files instead of trusting chat history alone. One sub-issue appears partially addressed on `main`: OpenAI transport session ids are normalized and truncated before being sent as headers.
