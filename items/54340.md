## Summary

Current main already lets Feishu streaming cards use a configured agent identity name instead of the raw agent/session id. The header title is built from `identity.name`/`identity.emoji` and only falls back to `agentId` when no identity name is configured, so the reported `main` title behavior is obsolete on current main.

## Close Comment

Closing as implemented on current main. Codex review checked `extensions/feishu/src/reply-dispatcher.ts:46` and found the streaming card header now comes from the routed agent outbound identity (`identity.name` / `identity.emoji`) and only falls back to the raw `agentId` when no identity name is configured. The Feishu bot wires that identity in from `resolveAgentOutboundIdentity` in `extensions/feishu/src/bot.ts:1087`, and that identity is sourced from agent config in `src/infra/outbound/identity.ts:25`. User-facing config for this already exists via `agents.list[].identity.name` as shown in `docs/cli/agents.md:204`. This behavior is also present in the shipped `v2026.4.22` release (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). If you still need a Feishu-specific per-channel header override separate from agent identity, that would be a new feature request rather than the original bug.
