## Summary

Keep open. Current main partially covers Teams channel thread routing for revoked live-thread sends, but it does not implement the full PR request: proactive message-tool sends still force top-level routing, raw proactive activity/file sends still clear activityId without adding a thread suffix, and replyStyle=thread without a TurnContext still throws. The PR discussion also contains a recent production reproduction on v2026.4.21 and a follow-up commit addressing review feedback for file-send paths, so this is not conservative cleanup material.
