## Summary

Keep open. Current `main` still implements the reported behavior: `authorizeGatewayConnectCore` returns immediately on `auth.mode === "trusted-proxy"`, so failed trusted-proxy checks never reach shared-secret or Tailscale auth. The current tests and docs also explicitly enforce that `trusted-proxy` rejects loopback/local-direct requests and that mixed token + trusted-proxy configs are invalid.
