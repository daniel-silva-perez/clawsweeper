## Summary

Keep open. Current main still saves the default agent's primary model and fallback edits through the per-agent `agents.list[index].model` path, while the config/docs contract treats `agents.defaults.model` as the default model and `agents.list[].model` as a per-agent override. The PR is still relevant, although its review discussion correctly identifies a remaining stale-override cleanup gap before it should land as-is.

## Best Possible Solution

Keep this item open or replace it with a maintainer-owned patch that routes default-agent primary and fallback edits to `agents.defaults.model`, preserves non-default agent writes to `agents.list[].model`, removes or migrates stale default-agent `agents.list[].model` overrides when saving defaults, and adds focused regression coverage for default, non-default, and `defaultId === null` path resolution.
