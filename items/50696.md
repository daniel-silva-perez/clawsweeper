## Summary

Current `main` does not contain this fix, and the PR as proposed is not strong enough to close conservatively. The overview card still renders `displayName` before `label`, which matches the reported bug, while existing shared session-name logic elsewhere in the UI already uses the stricter `label`/`displayName` precedence and special-cases `label === key`. That means the PR is still useful, but it likely needs revision rather than closure.
