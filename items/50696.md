## Summary

PR #50696 should stay open. Current main still renders recent overview sessions with `displayName` before `label`, while adjacent UI session-selection code and tests already establish the safer label-first behavior from the related merged PR #45130. The PR is focused on a real remaining web UI surface, though it should ideally add a small regression test or reuse the existing display-name helper.

## Best Possible Solution

Keep PR #50696 open and ask for a maintainer-ready revision: use the same label-first, key-duplicate-aware logic as `resolveSessionDisplayName` for `ui/src/ui/views/overview-cards.ts`, preferably by reusing/extracting a shared helper, and add a focused overview-card regression test for a row with `label`, raw-id-like `displayName`, and `key`.
