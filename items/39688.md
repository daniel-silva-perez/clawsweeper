## Summary

Current main still does not deliver `event.messages` produced by `message:received` or `message:sent` internal hooks. The report remains a real core behavior gap, and the linked fix work is still open, including maintainer-labeled PR #41375.

## Best Possible Solution

Keep this issue open until maintainers either merge a fix such as #41375 or explicitly update the hook contract/docs to define which message hook surfaces can deliver `event.messages`. The implementation should route hook-produced replies only where a safe reply target exists and add regression coverage for delivery, not just hook emission.
