## Summary

Keep open. Current `main` does not implement a live browser voice conversation loop in `/chat`. The chat UI still exposes only browser speech-to-text for draft entry plus manual read-aloud, while realtime voice support is scoped to separate Talk/Voice Call surfaces. The issue’s xAI premise is also not implemented on `main`: xAI streaming STT is exposed for Voice Call, but xAI realtime voice is explicitly documented as not exposed yet.

## Evidence

- **Chat UI still has a mic/STT button, not a Live voice loop:** `renderChat` shows a mic button only when browser STT is supported. Clicking it starts/stops `startStt(...)`, appends final transcripts into the text draft, and labels the control `Voice input` / `Stop recording`. There is no live-mode toggle, no automatic send-on-speech loop, and no assistant audio playback path here.
