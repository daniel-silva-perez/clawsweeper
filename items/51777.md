## Summary

Keep #51777 open. Current main still has the same Feishu onboarding persistence gap, and the PR patch is now stale because the implementation moved out of src/commands/onboard-channels.ts into src/flows/channel-setup.prompts.ts.

## Best Possible Solution

Keep the PR open, but request a rebase onto current main and a regression test for fresh Feishu onboarding where the user explicitly selects pairing. The implementation should update src/flows/channel-setup.prompts.ts or the DM policy contract so missing persisted policy keys can be written without unconditionally invoking non-idempotent channel setters such as Twitch.

## Evidence
