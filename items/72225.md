## Summary

Current main still has the exact Control UI chat-session refresh behavior this PR targets: three chat refresh paths pass `activeMinutes: 0`, and `loadSessions` drops non-positive values before sending `sessions.list`. The PR is a narrow, relevant UI bug-fix candidate rather than obsolete maintainer-cleanup material.

## Best Possible Solution

Keep this PR open for maintainer review, or land an equivalent scoped fix that replaces the three remaining Control UI `activeMinutes: 0` chat-session refresh callsites with `CHAT_SESSIONS_ACTIVE_MINUTES` and updates regression coverage for initial chat refresh and session-switch refresh.
