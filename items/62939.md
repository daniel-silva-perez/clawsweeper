## Summary

Keep open. Current main has meaningful partial hardening for wrapped external content, but it does not implement the requested boundary-level defense across non-owner message submission, central tool-result transport transformation, and stable system-prompt trust guidance. The linked PR #62973 is closed unmerged, and current main still leaves the core requested behavior incomplete.

## Best Possible Solution

Keep this open as a focused core security hardening request. The best path is to inventory the existing `wrapExternalContent` coverage, add the missing central boundary behavior for untrusted tool/MCP results and non-owner or external-channel message bodies, add stable system-prompt guidance before the cache boundary, and cover the runner, transport transform, and representative channel ingress with tests. If maintainers prefer the broader CaMeL/provenance work as the canonical track, this issue should be explicitly linked there only after these concrete requirements are represented.
