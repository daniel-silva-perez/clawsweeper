## Summary

Keep open. Current main has partial observer-mode plumbing: Gateway always streams tool events to WebSocket clients even when session `verbose` is `off`, Control UI documents live tool-call cards, and the UI has a local `chatShowToolCalls` toggle. But the requested feature is not implemented well enough to close: tool activity is still rendered inline in the main chat thread rather than in a separate activity panel/observer surface, and the issue's per-connection observer-mode ask is not satisfied by the existing session-level `/verbose` control.
