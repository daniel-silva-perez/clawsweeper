## Summary

Keep open. Current main still generates and rewrites the macOS gateway LaunchAgent without `ProcessType=Interactive`; existing tests and service-audit checks cover other launchd policy keys but not this one. Related #62308 and #58061 remain useful open tracking/fix candidates, but nothing equivalent is merged or shipped on main.

## Best Possible Solution

Keep this issue open and consolidate it with the existing launchd/App Nap work. The narrow code path is to add `ProcessType=Interactive` in `src/daemon/launchd-plist.ts`, cover both fresh install and restart-rewrite behavior in `src/daemon/launchd.test.ts`, and decide whether service-audit/doctor should detect or repair already-installed plists missing the key before closing.
