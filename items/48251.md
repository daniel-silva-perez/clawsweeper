## Summary

Current main already resolves these renamed bundled packages to their canonical plugin ids by preferring `openclaw.plugin.json:id` during discovery, so the PR's added `*-plugin` fallback mapping is redundant.

## Close Comment

Closing after Codex review: current `main` already implements the behavior this PR is trying to add, and it was already shipped in `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). Discovery prefers `openclaw.plugin.json:id` before falling back to the npm package name in `src/plugins/discovery.ts:429-451`, with both package discovery paths wiring that manifest id through in `src/plugins/discovery.ts:887-917` and `src/plugins/discovery.ts:1023-1053`. The affected bundled plugins already declare canonical ids in `extensions/brave/openclaw.plugin.json:2`, `extensions/google/openclaw.plugin.json:2`, `extensions/perplexity/openclaw.plugin.json:2`, and `extensions/xai/openclaw.plugin.json:2`, even though their package names use the `*-plugin` suffix. That means discovery on current `main` already resolves these packages as `brave`, `google`, `perplexity`, and `xai`, so the extra normalization table in this PR is redundant.
