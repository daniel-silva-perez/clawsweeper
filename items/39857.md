## Summary

Keep #39857 open. Current main still lets `agents.defaults.contextTokens`, persisted `contextTokens`, or direct `contextTokensOverride` win before known model-window lookup in the shared resolver, status-summary, and session persistence paths. The linked fix PR #39859 remains open and directly tracks the missing behavior.

## Best Possible Solution

Keep #39857 open until PR #39859 or an equivalent maintainer-approved change lands. The fix should make `resolveContextTokensForModel()`, reply/status setup, status summaries, and session persistence treat overrides or stored values as ceilings when a smaller known model window is available, while preserving existing behavior when the model window is unknown. Add focused regression coverage for resolver, status display, and session metadata persistence.
