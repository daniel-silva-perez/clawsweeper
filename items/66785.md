## Summary

Keep open. Current main includes OpenRouter onboarding/auth support and model-picker improvements for large catalogs, but it does not fully resolve the reported setup failure. The installer still appears to fall through into `openclaw onboard` after detecting an existing config, which matches the issue's “forces wizard every time even with config file present” claim.

## Best Possible Solution

Fix the installer or Umbrel setup entrypoint so an existing `$OPENCLAW_CONFIG_PATH` or legacy config exits the onboarding path after doctor/dashboard handling instead of falling through to `openclaw onboard`. Add a focused regression check for the existing-config install path without relying on `OPENCLAW_NO_ONBOARD`, then verify the packaged Umbrel wizard can search/filter large model lists and select OpenRouter.
