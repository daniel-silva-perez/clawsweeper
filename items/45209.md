## Summary

Keep open. Current `main` does not implement the requested Feishu Bitable toggle: the Feishu tools config type/schema/default merge path still has no `bitable` field, and `registerFeishuBitableTools` still registers Bitable tools without checking any per-account tools toggle.

## Evidence

- **Config type lacks Bitable toggle:** `FeishuToolsConfig` on current `main` only includes `doc`, `chat`, `wiki`, `drive`, `perm`, and `scopes`; there is no `bitable?: boolean` field.
  - file: [extensions/feishu/src/types.ts:91](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/extensions/feishu/src/types.ts#L91)

- **Config schema lacks `tools.bitable`:** The Feishu config Zod schema exposes `doc`, `chat`, `wiki`, `drive`, `perm`, and `scopes` only. Users on current `main` cannot set `tools.bitable: false` through the validated config surface.
  - file: [extensions/feishu/src/config-schema.ts:94](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/extensions/feishu/src/config-schema.ts#L94)
