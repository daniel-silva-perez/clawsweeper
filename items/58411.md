## Summary

Keep #58411 open. Current main still has the reported slash-command/tool asymmetry: `/acp spawn --bind here` has a current-conversation binding path, but `sessions_spawn(runtime="acp")` only exposes `thread: true` for persistent ACP binding and does not provide a `bindTo: "current"` equivalent. For Discord, that tool path can still select child-thread placement instead of current-thread/current-conversation placement.
