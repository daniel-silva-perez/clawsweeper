## Summary

Current `main` does not appear to implement this request. OpenClaw has first-party cloud docs for generic VPS, Docker, Kubernetes, GCP, and Azure, plus Bedrock AWS credential setup, but there is no comprehensive AWS deployment guide covering EC2, ECS, and Lambda, and no first-party AWS install page to close this issue against.

## Evidence

- **Install docs do not include an AWS guide:** The install hub lists cloud/deployment pages for VPS, Docker VM, Kubernetes, Fly.io, Hetzner, GCP, Azure, Railway, Render, and Northflank, but no AWS page or EC2/ECS/Lambda guide.
  - file: [docs/install/index.md:163](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/docs/install/index.md#L163)
  - sha: [3361593442e2](https://github.com/openclaw/openclaw/commit/3361593442e295e9bb1f55cbfb38861ea9413e45)
- **Only first-party AWS deployment mention is a brief generic note:** The Linux server page says AWS (EC2 / Lightsail / free tier) works well and links to a community video, but it does not provide OpenClaw-maintained AWS deployment steps.
  - file: [docs/vps.md:31](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/docs/vps.md#L31)
  - sha: [3361593442e2](https://github.com/openclaw/openclaw/commit/3361593442e295e9bb1f55cbfb38861ea9413e45)
- **AWS-specific docs currently cover Bedrock auth on EC2, not OpenClaw deployment:** The Bedrock provider doc includes EC2 instance-role / IAM setup for Bedrock model discovery, which addresses AWS credentials for the provider, not EC2 service setup, ECS task definitions, Lambda applicability, networking, persistence, or production deployment guidance.
  - file: [docs/providers/bedrock.md:87](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/docs/providers/bedrock.md#L87)
  - sha: [3361593442e2](https://github.com/openclaw/openclaw/commit/3361593442e295e9bb1f55cbfb38861ea9413e45)
- **No AWS install page exists in the docs tree:** A docs filename search found no `docs/install/aws*` page or other AWS deployment doc under `docs/install/`.
  - command: `rg --files docs/install | rg '^docs/install/aws|^docs/install/.*/aws|aws'`
  - sha: [3361593442e2](https://github.com/openclaw/openclaw/commit/3361593442e295e9bb1f55cbfb38861ea9413e45)

## Risks / Open Questions

- Generic Docker and Kubernetes docs may help AWS users indirectly, but they do not satisfy the issue's explicit request for AWS-specific EC2/ECS/Lambda guidance.
- There may be external/community AWS examples, but they are not implemented in this repository's maintained docs surface.
