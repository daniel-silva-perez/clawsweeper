## Summary

Keep open. Current main still has the reported mismatch: `openclaw memory status` scans session totals by counting every `.jsonl` file, while the session indexing/usage-counted path explicitly excludes compaction checkpoint transcripts. The status renderer then compares indexed DB/source counts against that broader scan total, so a primary transcript plus checkpoint can still render as `sessions 1/2` or `Indexed x/(x+1)`. The provided related items cover other memory/status issues and do not supersede this specific checkpoint count bug.

## Best Possible Solution

Keep #69759 open. The best fix is to make `scanSessionFiles()` use the same shared session artifact contract as indexing, likely `isUsageCountedSessionTranscriptFileName()` or a shared session file listing helper, so memory status scan totals and indexed counts agree. Add a focused regression test for a sessions directory containing one primary `.jsonl` transcript and one `.checkpoint.<uuid>.jsonl` snapshot, asserting the status denominator excludes the checkpoint.
