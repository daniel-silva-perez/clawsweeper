## Summary

Keep open. Current main does not expose the requested WebChat setting, and the native WebChat flow still refreshes chat history after terminal chat events. The optimistic user message with inline attachment blocks is not persisted in the same shape as the transcript entry, so there is not strong evidence that the reported post-attachment user-message replay/rerender is fully addressed on `main`.
