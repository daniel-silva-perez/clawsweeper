## Summary

Keep open. Current main has absorbed parts of this scope-hardening area, especially explicit internal admin gates for directive persistence and several built-in mutating commands, but it has not made the PR obsolete. The PR still covers unresolved plugin runtime config-write hardening, and its proposed `chat.send` hardening has an unresolved review concern because current main intentionally keeps `CommandAuthorized: true` while forwarding gateway scopes for downstream, command-specific checks.

## Best Possible Solution

Keep the PR open but require a rebase/rework against current main: preserve read-only `operator.write` command behavior, build on the existing `GatewayClientScopes` propagation and `requireGatewayClientScopeForInternalChannel` pattern or a command capability metadata model, and add focused tests for plugin runtime config writes through `api.runtime.config.writeConfigFile` from internal `operator.write` versus `operator.admin` callers.
