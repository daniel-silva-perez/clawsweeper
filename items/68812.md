## Summary

Keep #68812 open. Current main still hard-codes the memory embedding retry/backoff and non-batch indexing concurrency values, and the typed/strict config contract only exposes adjacent controls such as remote batch concurrency and inline batch timeout. The related PR #68822 is the right active implementation thread, but it is open/unmerged and does not make the issue obsolete or implemented on main.

## Best Possible Solution

Keep #68812 open until OpenClaw lands a typed, validated memory embedding tuning surface. The likely path is to repair or replace #68822 by adding agent-scoped memorySearch embedding/indexing settings to MemorySearchConfig, resolved config, strict Zod schema, generated help/labels/docs, and tests, then wire those values into retry loop parameters, retry delay capping, and non-batch getIndexConcurrency without changing the existing provider Batch API meaning of remote.batch.concurrency.
