## Summary

Keep open. Current main still derives lastMessagePreview from user/assistant-shaped transcript tail entries without checking the transcript entry type, so a diagnostic.heartbeat line with an assistant message can still become the session preview. The related #66544 PR addresses displayName/origin-label leakage, not this transcript preview scanner.

## Best Possible Solution

Keep this PR open for maintainer review. The likely best path is to land a focused fix in readLastMessagePreviewFromOpenTranscript with regression coverage in src/gateway/session-utils.fs.test.ts, choosing explicitly whether the filter should cover only diagnostic.heartbeat or all diagnostic.* entries, while coordinating separately with #66544 for displayName/origin-label leakage.
