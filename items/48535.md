## Summary

Keep open. Current `main` still does not implement the PR’s gateway-side `models.mode: "replace"` filtering. `models.list` passes the full gateway catalog into `buildAllowedModelSet()`, and when there is no `agents.defaults.models` allowlist that helper explicitly returns the full catalog unchanged, so the dashboard can still receive all built-in models instead of only configured providers.

## Evidence

- **`models.list` still uses the full catalog with no replace-mode branch:** Current `main` loads `catalog`, passes it directly to `buildAllowedModelSet`, and falls back to `catalog` when `allowedCatalog` is empty. There is no `cfg.models?.mode === "replace"` handling in this handler.

  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
