## Summary

Keep open. Current `main` still does not implement Telegram `message read`: the Telegram action map and discovery surface exclude `read`, unsupported actions still throw, and the docs still list `read` only for Discord/Slack/Matrix rather than Telegram.

## Evidence

- **Telegram action map has no read alias:** The Telegram message action map includes `send`, `poll`, `react`, `delete`, `edit`, `sticker`, `sticker-search`, `topic-create`, and `topic-edit`, but no `read` entry.
  - file: [extensions/telegram/src/channel-actions.ts:40](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/extensions/telegram/src/channel-actions.ts#L40)
  - command: `nl -ba extensions/telegram/src/channel-actions.ts | sed -n '36,60p'`
  - sha: [3361593442e2](https://github.com/openclaw/openclaw/commit/3361593442e295e9bb1f55cbfb38861ea9413e45)
- **Telegram message-tool discovery still omits read:** `describeTelegramMessageTool()` builds the advertised Telegram action set and never adds `read`; `handleAction()` then throws `Unsupported Telegram action` when the action cannot be resolved.
  - file: [extensions/telegram/src/channel-actions.ts:124](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/extensions/telegram/src/channel-actions.ts#L124)
  - command: `nl -ba extensions/telegram/src/channel-actions.ts | sed -n '120,200p'`
  - sha: [3361593442e2](https://github.com/openclaw/openclaw/commit/3361593442e295e9bb1f55cbfb38861ea9413e45)
- **CLI docs still do not advertise Telegram read support:** The CLI message docs say `read` is supported for `Discord/Slack/Matrix` and do not include Telegram.
  - file: [docs/cli/message.md:100](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/docs/cli/message.md#L100)

  - sha: [3361593442e2](https://github.com/openclaw/openclaw/commit/3361593442e295e9bb1f55cbfb38861ea9413e45)
- **Telegram channel docs list action support without read:** The Telegram channel docs enumerate `sendMessage`, `react`, `deleteMessage`, `editMessage`, and `createForumTopic`, with aliases `send`, `react`, `delete`, `edit`, `sticker`, `sticker-search`, and `topic-create`; `read` is absent.
  - file: [docs/channels/telegram.md:435](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/docs/channels/telegram.md#L435)
