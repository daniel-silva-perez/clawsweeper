## Summary

Keep #71536 open. Current main shows Telegram `/restart` is intentionally owner-gated and can emit the reported “You are not authorized to use this command.” from more than one path, but the issue is new, labeled regression, and specifically says allowlist and owner config changes did not unblock it. There is not enough source/test/history evidence to close it as fixed or stale.

## Best Possible Solution

Keep the issue open and ask for a redacted `channels.telegram` account config, `commands.allowFrom`, `commands.ownerAllowFrom`, the sender id from `/whoami`, and gateway logs around the `/restart` attempt. If reproduced, add a focused Telegram native-command regression proving `/restart` succeeds for the intended owner configuration, or improve the denial path to distinguish command allowlist failure from owner-identity failure.
