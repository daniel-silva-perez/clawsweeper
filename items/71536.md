## Summary

Keep open. This is a recent regression report against 2026.4.23, and current main still has two gates that can return exactly the reported Telegram native error for /restart: Telegram native command authorization and the owner-only restart gate. I did not find source, tests, or release evidence proving Telegram /restart owner authorization is fixed on main.

## Best Possible Solution

Keep the issue open and reproduce the full Telegram /restart path on current main. Add a focused regression test that exercises Telegram native /restart with an authorized owner sender, including the intended combinations of channels.telegram.allowFrom or pairing store, commands.allowFrom, and commands.ownerAllowFrom. If behavior is correct but config is confusing, improve diagnostics so Telegram distinguishes command allowlist failure from non-owner failure and update the slash-command/Telegram docs with a concrete Telegram ownerAllowFrom example.
