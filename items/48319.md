## Summary

Keep open. Current `main` still contains the exact outbound Signal placeholder behavior from the report, and the latest release `v2026.4.22` ships the same code path.

## Evidence

- **Current main still injects visible media placeholders for attachment-only Signal sends:** `sendMessageSignal` sets `message = kind === "image" ? "<media:image>" : \`<media:${kind}>\`` when `opts.mediaUrl` is present and `text` is empty, then skips markdown formatting because `messageFromPlaceholder = true`. That matches the reported leak for audio/voice messages.
