## Summary

Current main still leaves the persistent-session `sessions_send` duplicate-message report unresolved. The merged #69817 work fixed only parent-owned one-shot ACP children; normal peer sessions and persistent ACP/session targets still enter the A2A ping-pong plus announce path, and the tool schema still has no per-call ping-pong disable or max-turn option.

## Best Possible Solution

Keep this issue open as the canonical tracker. The likely product/code path is to make `sessions_send` A2A context-aware for persistent peer sessions or add an explicit per-call ping-pong max/disable option in `src/agents/tools/sessions-send-tool.ts` and `src/agents/tools/sessions-send-tool.a2a.ts`, while preserving the shipped parent-owned one-shot ACP child skip and adding regression coverage for persistent peer-session sends.
