## Summary

Keep open. Current `main` contains partial follow-up work that could address related symptoms, but the evidence is not strong enough to say issue #45698 is fixed or obsolete. `channels.status` is now fanned out concurrently on the gateway side, and the changelog notes nearby dashboard freeze/pending-request fixes in 2026.3.13, but the Control UI still opens the Channels tab with `probe: true`, still waits up to 8s for `channels.status`, and the browser `GatewayBrowserClient.request()` path still has no client-side timeout/abort. That leaves the reported slow-tab / progressive-stall behavior plausible on current `main`.
