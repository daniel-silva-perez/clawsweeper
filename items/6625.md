## Summary

Keep #6625 open. Current main still implements hard sub-agent run timeouts and post-timeout partial-progress handling, but I found no configurable pre-timeout grace window or injected checkpoint warning for native or ACP sub-agents. The existing issue discussion already includes a maintainer-posted Codex review keeping this open, and current main at 4ad8b613 still matches that assessment.

## Best Possible Solution

Keep #6625 open for core sub-agent lifecycle work. A good fix would add a documented `agents.defaults.subagents` warning/grace setting, schedule a warning before `runTimeoutSeconds`, deliver it through the existing child-session control/message path for native and ACP sub-agents where feasible, and add focused tests/docs while preserving current post-timeout partial-progress behavior.
