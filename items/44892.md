## Summary

Keep open. Current `main` does not implement the proposed `link-understanding` URL metacharacter filter, and the reported defense-in-depth gap is still present: `LinkUrl` is interpolated into template-enabled CLI args without escaping, while explicit shell-wrapper argv remains an allowed execution path that callers must validate themselves.

## Evidence

- **Link URL is injected into CLI argv via raw template substitution:** `runLinkUnderstanding` builds `templCtx.LinkUrl` from the extracted URL, then maps every non-command argv entry through `applyTemplate(...)` before calling `runExec(...)`. There is no escaping or validation step between URL extraction and process invocation.
  - file: [src/link-understanding/runner.ts:56](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/src/link-understanding/runner.ts#L56)
  - command: `nl -ba src/link-understanding/runner.ts | sed -n '41,72p'`
  - sha: [4630ce3d9e2f](https://github.com/openclaw/openclaw/commit/4630ce3d9e2f089062ccd386b0ae241325961d1a)
- **Template engine performs plain string interpolation:** `applyTemplate` replaces `{{...}}` placeholders with formatted values directly. It does not shell-escape or otherwise sanitize substituted text.
  - file: [src/auto-reply/templating.ts:276](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/src/auto-reply/templating.ts#L276)
  - command: `nl -ba src/auto-reply/templating.ts | sed -n '275,283p'`
  - sha: [4630ce3d9e2f](https://github.com/openclaw/openclaw/commit/4630ce3d9e2f089062ccd386b0ae241325961d1a)
- **Explicit shell-wrapper execution is still an intended caller path:** `shouldSpawnWithShell` is hardcoded false, but the adjacent security comment says callers that need a shell should use an explicit shell-wrapper argv and validate/escape at the call site. That means configs like `sh -c ...` remain possible, so link-understanding is the relevant call site for validation.
  - file: [src/process/exec.ts:121](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/src/process/exec.ts#L121)
  - command: `nl -ba src/process/exec.ts | sed -n '117,127p'`
  - sha: [4630ce3d9e2f](https://github.com/openclaw/openclaw/commit/4630ce3d9e2f089062ccd386b0ae241325961d1a)
- **Link extraction currently only enforces protocol and SSRF-style hostname checks:** `isAllowedUrl` accepts any syntactically valid `http:`/`https:` URL whose hostname is not blocked. There is no current check for shell metacharacters or shell-wrapper-sensitive URL content.
  - file: [src/link-understanding/detect.ts:19](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/src/link-understanding/detect.ts#L19)
