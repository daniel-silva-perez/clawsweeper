## Summary

Keep #71899 open. Current main still has the reported `agents add` default-workspace bug: `agents add` uses `resolveAgentWorkspaceDir()` for the wizard initial value, and the current resolver appends non-default agent IDs to any `agents.defaults.workspace`, including the setup-created main workspace. The PR is a focused implementation for the open linked bug #71889, with reporter confirmation and regression coverage in the PR context, so it is not obsolete or closeable as cleanup.

## Best Possible Solution

Keep #71899 open as the active fix path for #71889. The best product direction is to preserve custom shared workspace bases, but treat setup-managed main workspaces as main-agent workspaces so new agents default to peer-level `workspace-<agentId>` paths. Before landing, resolve or explicitly dismiss the remaining Windows canonicalization concern and rerun the changed agent/docs checks in CI or a maintainer environment.
