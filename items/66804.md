## Summary

Keep open. Current main has useful Active Memory mitigations, including thinking off by default and a hard timeout deadline, but it still does not make MiniMax-M2.7 complete Active Memory recall or retry/fall back after the selected model times out. The issue discussion has multiple recent confirmations, and #65309 remains a broader latency tracker rather than a conclusive duplicate for this MiniMax-specific failure.

## Best Possible Solution

Keep this issue open unless maintainers intentionally consolidate it into #65309. A real fix should either make MiniMax-M2.7 viable for the Active Memory subagent through provider-specific safer defaults/tuning, or add a tested fallback-on-timeout path, such as retrying config.modelFallback or using a bounded local memory_search summary path without extending the foreground reply wait beyond the configured budget.
