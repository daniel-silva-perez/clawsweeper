## Summary

Keep open. Current main has useful Active Memory mitigations, including MiniMax thinking suppression, configurable timeouts, hard deadline handling, and explicit model selection controls, but it still treats a selected recall model timeout as `status: "timeout"` with `summary: null` and no retry/local fallback. The issue discussion has multiple recent MiniMax-M2.7 confirmations, including OpenClaw 2026.4.22, so this is not safely closeable as implemented or stale.

## Best Possible Solution

Keep this issue open unless maintainers intentionally consolidate it into #65309. The likely fix belongs first in `extensions/active-memory/index.ts`: either make MiniMax-M2.7 viable for Active Memory through tested provider-specific safer defaults such as a faster MiniMax tier for recall, or add a bounded fallback-on-timeout path that retries `config.modelFallback` or produces a local `memory_search` summary without extending the foreground wait beyond the configured budget. Add regression coverage for selected-model timeout fallback and, when feasible, live validation for MiniMax API-key and OAuth provider ids.
