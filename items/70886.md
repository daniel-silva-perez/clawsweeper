## Summary

Current main does not implement the reported ZAI vision routing fix. ZAI onboarding still stores one provider-level coding base URL for the whole catalog, the vision catalog entries do not carry a non-coding override, and the ZAI media-understanding provider delegates directly to the generic image helper. The related fix PR #70915 is still open/unmerged, while #70908 was closed unmerged.

## Best Possible Solution

Keep #70886 open until a focused fix lands. The likely maintainer path is to route ZAI vision models from coding-plan base URLs to the matching non-coding ZAI endpoint inside the ZAI plugin/onboarding/media-understanding boundary, with regression tests for zai-coding-global and zai-coding-cn image requests. PR #70915 is the active candidate to review or supersede; #70908 was closed unmerged.
