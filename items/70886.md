## Summary

Keep open. Current main still applies ZAI Coding Plan base URLs provider-wide for the bundled ZAI catalog, the ZAI media-understanding provider does not rewrite vision requests to a non-coding endpoint, and open PR #70915 remains an unmerged implementation candidate with closing syntax for this issue.

## Best Possible Solution

Keep this issue open while maintainers review and either land or close #70915. The likely fix belongs in the bundled ZAI plugin boundary: route only ZAI vision models such as glm-5v-turbo, glm-4.6v, and glm-4.5v from Coding Plan endpoints to the same-region non-coding multimodal endpoint, with regression coverage proving text/coding models stay on the Coding Plan URL and vision models use the standard URL.
