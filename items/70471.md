## Summary

Keep PR #70471 open. Current main does not implement the PR's central Feishu hardening: comment recipient gating, tool-delivery final-reply suppression, angle-bracket escaping for comment writes, account-scoped typing reaction state, or force-fresh credential-aware bot identity probing. The related PR #66427 is closed but unmerged in the provided context, so it is not a shipped canonical replacement.

## Best Possible Solution

Keep #70471 open or replace it with an updated Feishu PR that lands the missing behavior on current main: strict `to_user_id.open_id` recipient gating, a scoped delivery guard for tool-visible comment replies, Feishu-safe comment text escaping, account-scoped typing reaction cleanup, and credential-aware force-fresh bot identity probing with fallback tests. The related #66427 can remain historical context, but it should not be treated as the shipped fix because it closed unmerged.
