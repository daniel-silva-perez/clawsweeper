## Summary

Keep #39392 open. Current main at 4a3030df9efa72c44bab567a4390ac0c4876a73f still lacks the focused Agents Overview fix for stale primary-model UI state after switching agents, and the related fix PRs are not shipped on main.

## Best Possible Solution

Keep #39392 open until a focused Control UI fix lands. The best path is to key/remount the Agents Overview model controls by selectedAgent.id or otherwise synchronize the select value and options with one explicit selected value, then add a regression test for switching main -> another agent -> main and verifying the displayed Primary model matches the selected agent's resolved config. PR #39401 remains useful context as the closest open candidate; #70633's selectedPrimary approach is also relevant, but neither implementation is present on current main.
