## Summary

Keep #72076 open. Current main does not ship the proposed cua-driver computer plugin, and this should not be auto-closed as ClawHub because the PR discussion contains explicit maintainer signal that desktop computer-use needs a dedicated SDK/security/permissions model beyond a simple generic tool plugin. The PR is also active, has useful prior art from #1946, and has unresolved review defects, so the best cleanup path is maintainer triage or superseding it with a canonical computer-use SDK/design item, not automated closure.

## Best Possible Solution

Keep #72076 open for maintainer judgment, but do not merge it as-is. The best path is to define the computer-use extension SDK and permission/security contract first, then either adapt the cua-driver work to that approved API or close it in favor of a canonical maintainer-owned computer-use SDK/design item.
