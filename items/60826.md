## Summary

Keep open. Current main still persists raw `process.argv.slice(0, 8)` into `config-audit.jsonl` records on write and observe paths, and the linked fix PRs are closed/unmerged in the provided GitHub context.

## Best Possible Solution

Keep the issue open and land a focused config-audit fix that centralizes process-info capture, redacts or omits argv before every config audit write/observe/recovery record, reuses `src/logging/redact.ts` where appropriate, and adds regression tests for `--flag value` and `--flag=value` secret-bearing flags across the current audit call sites.
