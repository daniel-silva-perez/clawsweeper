## Summary

Keep open. Current `main` still lets `sessions_history` accept and forward caller-supplied limits above the Gateway `chat.history` maximum, and the provided GitHub context shows #66650 is still open as the implementation candidate for this issue.

## Best Possible Solution

Review #66650 against current `main`, then land it or an equivalent focused fix that adds the Gateway maximum to the `sessions_history` tool schema, clamps outgoing `chat.history` requests to 1000, and adds regression coverage for oversized requested limits. If maintainers reject that approach, close or supersede #66650 first and keep this issue tracking the remaining bug.
