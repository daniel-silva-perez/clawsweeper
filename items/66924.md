## Summary

Keep this PR open. Current main still has the Google provider Gemma forward-compat path clone the Gemini 3 flash template without any Gemma-specific context-window override, so a template with a 1,048,576-token window can still be surfaced for `gemma-*` fallback models. The PR is a focused implementation candidate for the linked Google provider display bug, though maintainers may want to review whether a hardcoded 131,072-token fallback fully matches the original report’s configured `models.json` context-window expectation.

## Best Possible Solution

Do not auto-close this PR. Review it as a Google provider bug-fix candidate, and either land a provider-owned fix with the added regression coverage or revise it to preserve the discovered/configured Gemma context window when available while still preventing inherited Gemini flash 1M context display.
