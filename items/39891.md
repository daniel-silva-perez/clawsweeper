## Summary

Keep PR #39891 open. Current main still matches the reported Google Chat bug: the channel creates a real typing message before dispatch, clears it only from the delivery path, and exact allowed NO_REPLY finals can be skipped before delivery. The PR is a focused fix for linked issue #39843, though it should be refreshed with a regression test before landing.

## Best Possible Solution

Keep this PR or its linked issue active. The best maintainer path is to refresh the Google Chat fix against current main, delete the orphaned `typingMessageName` when dispatch returns without delivery, and add a focused monitor-level regression test for the reaction-plus-NO_REPLY path before closing #39843.
