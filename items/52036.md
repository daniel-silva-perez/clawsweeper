## Summary

Keep #52036 open. Current main has adjacent hardening for idle-time cleanup, but it has not implemented this PR's appendMessage serialization or the concurrent regression tests. The PR also has unresolved P1 review concerns about changing appendMessage to async and tracking pending tool calls before write acceptance, so it is not safe to close as implemented or merge as-is.

## Best Possible Solution

Keep the PR open for maintainer judgment. The best path is a revised fix that preserves the current appendMessage contract, serializes the guard state/write critical section only if the race is reproduced, handles beforeMessageWriteHook/originalAppend failure without leaking pending ids, and adds the concurrent regression coverage described in the PR. If maintainers decide the current wait-for-idle cleanup fully supersedes the user reports, close manually with a canonical issue or PR link.
