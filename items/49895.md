## Summary

Current `main` already covers the issue’s three asks: `sessions_spawn` accepts arbitrary user labels, subagents have built-in cleanup/auto-archive lifecycle handling, and `thread: true` is no longer limited to Discord because Matrix and Feishu register subagent thread-binding hooks.

## Evidence

- **Arbitrary subagent names are accepted:** `sessions_spawn` exposes `label` as an unconstrained string, and spawn logic preserves the trimmed label instead of restricting it to an identifier format.
  - file: [src/agents/tools/sessions-spawn-tool.ts:100](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/src/agents/tools/sessions-spawn-tool.ts#L100)
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **Spawn path keeps user-provided labels:** `spawnSubagentDirect` reads `params.label?.trim() || ""`, so meaningful names such as non-English labels are allowed.
  - file: [src/agents/subagent-spawn.ts:551](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/src/agents/subagent-spawn.ts#L551)
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
