## Summary

Keep open. Current main still lacks outbound message lifecycle hook coverage for the webchat/openclaw-tui `chat.send` path: the hook-aware delivery implementation exists in `deliverOutboundPayloads` and some inbound dispatcher wrappers, but webchat builds its own reply dispatcher and TUI sends through `chat.send`. The related umbrella #50126 is open, but it does not explicitly cover this webchat/TUI path, and PR #12584 is closed unmerged in the provided context.

## Best Possible Solution

Keep this issue open unless a maintainer explicitly folds it into #50126 with webchat/openclaw-tui called out. The best fix is to route `chat.send` and embedded TUI final/block reply delivery through the same outbound observer/hook seam used by channel delivery, or add an equivalent hook-aware wrapper that fires `message_sending` and `message_sent` with session/run context. Add targeted regression tests for webchat `chat.send` and openclaw-tui, and document which hooks are guaranteed per surface, especially where `before_message_write` and `agent_end` are agent-harness hooks rather than delivery hooks.
