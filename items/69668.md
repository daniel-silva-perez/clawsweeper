## Summary

Keep #69668 open. Current main supports multi-agent isolation and a chat-header session selector grouped by individual agent, but it does not implement the requested team/project parent nodes, team metadata, workspace-derived grouping, or hierarchical Control UI Sessions view. The Sessions tab remains a flat searchable/sortable table, and the agent configuration schema exposes per-agent fields such as id/name/workspace/identity/subagents rather than a team/group ownership field. The related #45086 work is adjacent but only covers agent/session switching, not team/project hierarchy.

## Best Possible Solution

Keep the issue open for explicit Control UI/product design. A likely implementation would add a first-class team/project grouping contract, probably under agent configuration or a documented session metadata layer, then wire it through sessions.list/agents.list and render a hierarchical Control UI Sessions or chat-adjacent session navigator. Existing agent IDs, labels, and the chat dropdown's per-agent optgroups can be reused as foundations, but they are not enough to close this request.
