## Summary

Current main does not implement this fix. The gateway still emits full accumulated assistant text across tool boundaries, while the web UI still snapshots pre-tool text into a separate segment when a tool starts. That combination still plausibly produces the duplicated pre-tool text described in PR #54374, so there is not strong evidence to close it.

## Evidence

- **Gateway delta payload is still full accumulated text:** `emitChatDelta` merges assistant text into `mergedText` and broadcasts that full snapshot on every `chat` delta. There is no segment-boundary field or slicing logic on current main.
  - file: [src/gateway/server-chat.ts:683](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/gateway/server-chat.ts#L683)

  - file: [src/gateway/server-chat.ts:930](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/gateway/server-chat.ts#L930)
