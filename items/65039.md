## Summary

Keep open. Current main still matches the reported DAVE-disabled pre-Ready Discord voice join abort: `/vc join` waits for `Ready`, destroys the voice connection on readiness failure, returns the same failure shape, and only records a session after `Ready`. The existing DAVE receive recovery is post-join/post-Ready and does not cover a voice WebSocket close after Hello before UDP/Ready. Related open issues are adjacent, but there is no maintainer-designated canonical replacement, and this issue adds useful Hello-before-Ready trace detail.

## Best Possible Solution

Keep this issue open until maintainers decide the intended non-DAVE Discord voice behavior. The likely fix belongs in `extensions/discord/src/voice/manager.ts`: add explicit pre-Ready close/abort diagnostics and regression coverage for `entersState` failure before session activation, then either implement bounded retry/fallback to DAVE-enabled join or reject/deprecate `channels.discord.voice.daveEncryption=false` with clear operator guidance if Discord now requires DAVE. If maintainers consolidate with #39825 or #63098, preserve this issue's Hello-before-Ready trace in the canonical thread before closing it as duplicate.
