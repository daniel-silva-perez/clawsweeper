## Summary

Keep open. Current main still forwards the Feishu `react` emoji parameter unchanged into the Feishu reactions API, and there is no equivalent Unicode-to-Feishu-type normalizer or regression test on main. The PR is a focused implementation candidate for the active Feishu bug #66406, but its mapping table should be checked against the current Feishu emoji_type vocabulary before landing.

## Best Possible Solution

Keep this PR open for maintainer review or land an equivalent Feishu-owned fix: add a pure emoji normalizer backed by Feishu's current emoji_type vocabulary, apply it before add/remove reaction calls and any emoji filter path, and keep focused tests for Feishu type passthrough, Unicode conversion, trimming, unknown inputs, and the reported thumbs-up regression from #66406.
