## Summary

Keep open. Current main still passes the Feishu reaction emoji parameter through unchanged, while the Feishu reaction helper sends that value directly as `reaction_type.emoji_type`. There is no equivalent normalization helper or unit test on main, the linked bug #66406 remains open, and the related alternative PR #71970 was closed unmerged rather than superseding this work.

## Best Possible Solution

Review and land this PR or an equivalent Feishu-owned patch in `extensions/feishu`: add a pure Unicode-to-Feishu-type normalizer, apply it before add/remove/list reaction calls that accept an emoji filter, and keep focused tests for enum passthrough, case normalization, Unicode conversion, whitespace trimming, and unknown-value behavior.
