## Summary

Keep open. Current main still does not add the requested `web_fetch` / `browser` fallback guidance to runtime `web_search` missing-key messages, while the PR's original edit target has been replaced by provider-owned web search runtimes. The work is still useful, but the patch needs a refresh rather than maintainer cleanup closure.

## Best Possible Solution

Keep this PR open and request a refresh against the provider-owned web search runtimes: update missing-key messages in the relevant `extensions/*` provider files, preserve literal env var names, and add focused message coverage where practical. If maintainers want this guidance to apply beyond the original five providers, include current API-key-backed providers such as Exa, Firecrawl, MiniMax, and Tavily too.
