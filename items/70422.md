## Summary

Keep #70422 open. Current main still only recognizes `MiniMax-VL-01` as a MiniMax VLM model, and the active docs/tests continue to route MiniMax chat primaries to the plugin-owned `MiniMax-VL-01` image-understanding path. The related #69691 is closed/unmerged and narrower, so it does not supersede this PR.

## Best Possible Solution

Keep the PR open and review it as the active implementation candidate. The best path is to land a focused allowlist/docs/test update for MiniMax models that are actually supported by the VLM runtime, while explicitly deciding whether selected M2.7/highspeed refs should be forwarded to the MiniMax VLM endpoint or documented as routing through the shared VLM endpoint behavior.
