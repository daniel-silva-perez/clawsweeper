## Summary

Current `main` already disables Slack SDK retries for outbound write operations, which addresses the duplicate-send path described in #50597. Slack sends now use a dedicated write client with `retryConfig: { retries: 0 }`, and the changelog records this as shipped in the `2026.1.22` release notes.
