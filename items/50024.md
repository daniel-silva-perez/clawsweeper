## Summary

Keep open. Current `main` still hardcodes `webchat` for the initial `sessions_send` agent dispatch and for follow-up `runAgentStep` calls, while gateway session state persists that request channel into `lastChannel`/delivery context and reply routing later prefers that stored route. The open PR's fix is not implemented on `main` yet.
