## Summary

Current main still matches the reported bug. The pi-embedded `handleAgentEnd` lifecycle terminal emitter sends `phase: "end"`, optional liveness/replay fields, and `endedAt`, but does not include `aborted` or `stopReason`. The richer fallback emitter in `agent-command.ts` remains guarded by `!lifecycleEnded`, and that flag is set when the pi path emits its terminal lifecycle event. The related PR that claimed to fix this is closed unmerged in the provided GitHub context, so this issue should remain open.

## Best Possible Solution

Keep this issue open. The fix should land in `src/agents/pi-embedded-subscribe.handlers.lifecycle.ts` by adding `aborted` and `stopReason` to the pi-embedded `lifecycle:end` payloads emitted through both `emitAgentEvent` and `onAgentEvent`, with regression coverage for natural completion and abort/`stopReason: "aborted"` behavior. The closed unmerged PR #66574 can be used as prior art, but the implementation should be reviewed against the current `main` lifecycle and gateway session-state contracts.
