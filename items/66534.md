## Summary

Keep this issue open. Current main still emits the pi-embedded `lifecycle:end` payload without `aborted` or `stopReason`, while the richer fallback in `agent-command.ts` is skipped once that terminal lifecycle event has already been observed. The linked fix PR #66574 is closed unmerged, so there is no implemented-on-main or duplicate/superseded close path yet.

## Best Possible Solution

Keep this issue open. The fix should update `src/agents/pi-embedded-subscribe.handlers.lifecycle.ts` so both the global `emitAgentEvent` and direct `onAgentEvent` `lifecycle:end` payloads include `aborted` and `stopReason`, deriving them from the attempt result or terminal assistant metadata consistently with `agent-command.ts`. Add regression coverage for natural completion and aborted runs, and verify the gateway session lifecycle state maps the enriched terminal event correctly.
