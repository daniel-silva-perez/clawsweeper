## Summary

Keep #6757 open. Current main has manual `/compact`, Gateway `sessions.compact`, UI/control compaction, auto-compaction, and context-engine/plugin compaction hooks, but it still lacks the requested in-session agent-triggered self-compaction surface via `session_status(action: "compact")`, the agent `gateway` tool, or a dedicated CLI/session tool. The closest PR, #43401, was closed unmerged, and the latest maintainer/Codex comment on the issue also kept this open.

## Best Possible Solution

Keep #6757 open until maintainers either implement a first-class in-session self-compaction API, likely `session_status(action: "compact")` with mutation classification, active-run/deadlock handling, permission semantics, compaction counter/token persistence, and focused tests, or explicitly decide agents must not be able to compact their own active session. Existing manual `/compact`, admin `sessions.compact`, UI compaction, automatic compaction, and context-engine/plugin hooks should not be treated as satisfying this request.
