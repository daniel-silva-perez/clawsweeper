## Summary

Keep open. Current main still uses a fixed 1,000 ms native approval handler bootstrap retry and does not implement the PR's bounded exponential backoff, `PAIRING_REQUIRED` 300 s backoff, or existing-context guarded retry behavior. The related #70641/#70649 context is complementary and closed, not a landed replacement for this retry-after-failure fix.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is to review/rebase and land the bounded retry-backoff behavior in `src/infra/approval-handler-bootstrap.ts`, including `PAIRING_REQUIRED`-specific delay and existing-context guarded retry, with focused regression tests in `src/infra/approval-handler-bootstrap.test.ts`. Coordinate separately with the broader multi-account startup storm work discussed in #70641/#70649.
