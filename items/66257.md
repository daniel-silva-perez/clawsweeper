## Summary

Keep this PR open. Current main has shipped related loopback pairing fixes, including the broader #45504 area, but the narrower CLI behavior in this PR is not present on main: `devices approve` still calls the gateway first, and the local fallback guard still only blocks explicit `--url`, not explicit token/password auth.

## Best Possible Solution

Review this PR as a focused implementation candidate rather than closing it. The likely best path is to rebase it, keep or adapt the explicit token/password no-fallback guard, and add regression coverage that proves `devices approve <requestId>` either uses the local pending request directly on eligible loopback or intentionally relies on the newer gateway-side local pairing behavior.
