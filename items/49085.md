## Summary

Keep open. Current main still does not route `openclaw acp` diagnostics away from stdout before the ACP stdio/NDJSON transport, and the PR addresses a real core CLI/protocol correctness issue. The related issue #49060 and earlier PR #49072 are still open rather than merged, so this PR is not implemented or clearly superseded on main.

## Best Possible Solution

Keep the PR open, but ask for a current-main rebase that routes ACP bridge diagnostics to stderr before CLI command registration, preserves `openclaw acp client` stdout behavior, and covers the direct `serveAcpGateway()`/standalone server path by enabling console capture where needed. Link/consolidate with #49060 and decide whether #49072 should be closed in favor of this more complete fix after maintainer review.
