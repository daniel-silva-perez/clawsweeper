## Summary

Keep open. Current docs explain that `openclaw agent` requires a session selector and that Lobster is an optional plugin tool rather than an `openclaw lobster` subcommand, but the reporter's follow-up says the failure still occurs even with `--agent` and includes a gateway-handshake/no-TCP symptom plus a headless Linux discovery delay. I could not get a local read-only shell to run in this sandbox (`bwrap: loopback: Failed RTM_NEWADDR`), so there is not enough current-main reproduction proof to close conservatively.

## Evidence

- **Reporter follow-up says selector fix was not sufficient:** The issue comment states `openclaw agent` requires `--agent`, but even with `--agent` the process made zero TCP connections and sat idle until killed; that is a distinct plausible CLI/gateway bug beyond the documented missing-selector error.
- **Current public docs document required session selector:** `openclaw agent` docs say to pass at least one of `--to`, `--session-id`, or `--agent`; this explains the no-selector command but not the reporter's with-`--agent` hang.
  - file: [docs/cli/agent.md:113](https://github.com/openclaw/openclaw/blob/d6ef1fcf24eea3a5985d653a3baf400e32aa1b75/docs/cli/agent.md#L113)
  - sha: [d6ef1fcf24ee](https://github.com/openclaw/openclaw/commit/d6ef1fcf24eea3a5985d653a3baf400e32aa1b75)
- **Current public docs clarify Lobster CLI expectations:** Lobster docs say bundled workflows run in-process, no separate `lobster` binary is required, and Lobster is enabled as an optional plugin tool. This suggests `openclaw lobster` being unknown may be expected/design rather than a core CLI bug.
  - file: [docs/tools/lobster.md:241](https://github.com/openclaw/openclaw/blob/d6ef1fcf24eea3a5985d653a3baf400e32aa1b75/docs/tools/lobster.md#L241)
  - sha: [d6ef1fcf24ee](https://github.com/openclaw/openclaw/commit/d6ef1fcf24eea3a5985d653a3baf400e32aa1b75)
- **Local verification blocked:** Read-only shell inspection could not run any command in this environment; even `pwd`, `sed`, and `pnpm docs:list` failed before execution with `bwrap: loopback: Failed RTM_NEWADDR: Operation not permitted`.
  - command: `pwd; sed -n '1,220p' .agents/skills/openclaw-pr-maintainer/SKILL.md; pnpm docs:list`

## Risks / Open Questions

- A genuine current-main regression may remain in the `--agent` local/gateway path or in headless gateway discovery timing.
