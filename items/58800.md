## Summary

Keep PR #58800 open. Current main does not implement the Slack edit identity preservation: send paths already accept and fallback custom identity, but edit/update paths still omit username/avatar fields. The PR targets a real gap and linked bug #58737, but the provided review discussion identifies remaining issues in the PR itself, including missing missing-scope fallback parity and a later wiring bug that passes a thread timestamp instead of the resolved Slack identity. This is not conservative cleanup-close material.
