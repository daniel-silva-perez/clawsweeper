## Summary

Keep open. Current main has separate history-image pruning, but it does not strip memory/hook-injected `<relevant-memories>` blocks before prompt image detection, so the PR’s central bug is still plausible and not obsolete.

## Best Possible Solution

Keep the PR open for maintainer review or revision. The best fix is a targeted boundary change that prevents untrusted hook/memory context from being scanned as current-turn image attachments, with regression tests for `<relevant-memories>` and `<relevant_memories>` while preserving real current prompt and attachment image loading.
