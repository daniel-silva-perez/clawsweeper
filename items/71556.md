## Summary

Keep open. Current main has Z.AI-specific payload patching, so the issue is not correct that no Z.AI thinking payload is ever sent: thinking-off and preserved-thinking paths do send `thinking`. But the central reported bug remains for the normal `/think on` path: without `params.preserveThinking`, the bundled Z.AI plugin intentionally sends no `thinking` object, and the generic OpenAI-compatible stream builder still has no `thinkingFormat === "zai"` enabled-thinking path.

## Best Possible Solution

Keep the issue open. The likely fix belongs in the bundled `zai` provider stream wrapper: map non-off OpenClaw thinking levels to an explicit Z.AI `thinking: { type: "enabled" }` payload, keep `clear_thinking: false` only for `params.preserveThinking` with replayed `reasoning_content`, preserve `extra_body.thinking` override precedence, and add tests for default-on, off, preserved-thinking, and configured CN/Coding base URL cases.
