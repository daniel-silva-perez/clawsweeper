## Summary

Keep open. Current `main` still does not retry Discord outbound sends on transport-level failures, so the PR's core fix is not implemented on `main` and the report remains actionable even though the file layout has since moved into `extensions/discord/`.

## Evidence

- **Reply delivery retry predicate is still status-only:** `isRetryableDiscordError()` only checks `status/statusCode` for `429` or `>=500`, and `sendWithRetry()` uses that predicate directly. A raw `TypeError("fetch failed")` or `ECONNRESET` error still falls through without retry.
  - file: [extensions/discord/src/monitor/reply-delivery.ts:51](https://github.com/openclaw/openclaw/blob/40be5ad58187fd20b8defe5fc30e649190c131a8/extensions/discord/src/monitor/reply-delivery.ts#L51)
  - command: `nl -ba extensions/discord/src/monitor/reply-delivery.ts | sed -n '44,90p'`
  - sha: [40be5ad58187](https://github.com/openclaw/openclaw/commit/40be5ad58187fd20b8defe5fc30e649190c131a8)
- **Shared Discord request runner still retries only rate limits:** `createDiscordRetryRunner()` delegates to `createRateLimitRetryRunner()` with `shouldRetry: (err) => err instanceof RateLimitError`, so Discord transport errors are still excluded from the shared request retry path.
  - file: [extensions/discord/src/retry.ts:15](https://github.com/openclaw/openclaw/blob/40be5ad58187fd20b8defe5fc30e649190c131a8/extensions/discord/src/retry.ts#L15)
  - command: `nl -ba extensions/discord/src/retry.ts | sed -n '1,80p'`
  - sha: [40be5ad58187](https://github.com/openclaw/openclaw/commit/40be5ad58187fd20b8defe5fc30e649190c131a8)
- **Network fetch failures escape before HTTP error wrapping:** The proxy REST client awaits `fetch()` directly. Only once a `Response` exists does it convert `429` into `RateLimitError` and other non-OK responses into `DiscordError`. Transport failures during `fetch()` therefore escape as raw exceptions.
  - file: [extensions/discord/src/proxy-request-client.ts:316](https://github.com/openclaw/openclaw/blob/40be5ad58187fd20b8defe5fc30e649190c131a8/extensions/discord/src/proxy-request-client.ts#L316)
  - command: `nl -ba extensions/discord/src/proxy-request-client.ts | sed -n '292,364p'`
  - sha: [40be5ad58187](https://github.com/openclaw/openclaw/commit/40be5ad58187fd20b8defe5fc30e649190c131a8)
- **Current tests still miss the reported transport-error case:** The Discord reply-delivery tests cover retries for `429` and `500`, a non-retry `400`, and exhausted `429` retries, but there is no current coverage for `TypeError("fetch failed")` or `ECONNRESET` in this file.
