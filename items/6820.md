## Summary

Keep #6820 open. Current main still does not expose `xhigh` for base `openai-codex/gpt-5.2`: the OpenAI Codex provider xhigh allowlist includes nearby Codex models but omits base `gpt-5.2`; the thinking profile adds `xhigh` only from that allowlist; and the exact/prefix matcher does not make `gpt-5.2` match `gpt-5.2-codex`. The related fix PRs in the provided context, #6013 and #43054, are closed unmerged, and current tests still do not cover the requested provider/model pair.

## Best Possible Solution

Keep #6820 open until the bundled OpenAI Codex provider either adds base `gpt-5.2` to `OPENAI_CODEX_XHIGH_MODEL_IDS` with focused regression coverage for `listThinkingLevels("openai-codex", "gpt-5.2")` and directive/hint behavior, or maintainers explicitly document that base Codex GPT-5.2 remains high-only while `gpt-5.2-codex` is xhigh-capable.
