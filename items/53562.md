## Summary

Keep open. Current main still does not expose `voice.autoJoin[].sessionChannelId` or route Discord voice transcripts/replies to a paired text channel. The recent maintainer comment and related PR review both match the current code: auto-join only passes the voice channel, the session route is derived from that voice channel, and `processSegment()` still runs with `deliver: false`.

## Best Possible Solution

Keep #53562 open and implement this inside the bundled Discord plugin: add optional `sessionChannelId` to the Discord voice auto-join type, strict zod schema, UI/config metadata, and docs; pass it through auto-join and session state; route the voice session to that text channel when configured; provide an explicit Discord delivery target when enabling text delivery; preserve the setting through decrypt-failure rejoin recovery; and cover the config, routing, delivery, and recovery behavior with focused Discord voice tests.
