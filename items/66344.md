## Summary

This PR should stay open. Current main still lacks Markdown-table protection in `EmbeddedBlockChunker`: the chunker only parses code fence spans and all safe-break checks still call `isSafeFenceBreak`. The related duplicate PR #66568 was closed in favor of this PR, while #66614 and #66605 remain open and are directly tied to the same table-splitting behavior and integration coverage.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a table-span implementation wired into `src/agents/pi-embedded-block-chunker.ts`, with unit and integration coverage for the linked table-splitting behavior, after addressing the remaining review comments about streaming-tail buffering and one-pipe markdown table rows.
