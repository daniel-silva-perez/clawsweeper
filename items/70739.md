## Summary

Keep PR #70739 open. Current main still lacks a downstream SSE heartbeat for the public OpenAI-compatible streaming endpoints: `/v1/chat/completions` and `/v1/responses` only set SSE headers and then write real model/lifecycle events, while the only existing SSE keepalive pattern is limited to `sessions-history-http.ts`. The PR is therefore not obsolete or implemented on main; it remains a focused gateway transport fix, though existing review comments around interval validation, timer cleanup, and test timer restoration should be addressed before merge.

## Best Possible Solution

Keep the PR open and ask for the remaining review fixups, then validate and merge this PR or an equivalent main implementation that adds a hardened SSE comment heartbeat to both `/v1/chat/completions` and `/v1/responses` streaming paths without changing their event schema.
