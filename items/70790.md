## Summary

Keep this PR open. Current main still lacks an embedded-runner egress guard that removes `tools: []` and orphaned `tool_choice` before strict OpenAI-compatible providers receive the payload. The related `toolsAllow` forwarding path is already present on main, so the remaining work is the payload compatibility guard plus focused regression coverage.

## Best Possible Solution

Keep this PR open for maintainer review or a successor patch. The best fix is a narrow embedded-runner egress sanitizer that strips empty `tools` arrays and removes `tool_choice` when no non-empty tools remain, composes with existing `onPayload` wrappers, preserves non-empty tools, covers the intended embedded stream paths, and adds focused regression tests. Do not redo the already-present `toolsAllow` forwarding work.
