## Summary

Current main still matches the reported runtime-context leak path. The helper emits the exact human-readable preface, queues it as a hidden `openclaw.runtime-context` next-turn custom message before the model prompt, and the pinned Pi dependency converts custom messages to user-role LLM content. Existing sanitizer coverage still targets delimited and legacy internal runtime-context blocks, not this new header.

## Best Possible Solution

Keep this issue open for a core embedded-runtime fix. Stop placing the human-readable runtime-context preface into ordinary conversational custom-message content, or teach the Pi conversion path to treat `openclaw.runtime-context` as structured non-user LLM context instead of user-role content. Add regression coverage for the current header across runtime-context prompt assembly, channel-envelope deltas, the Pi custom-message conversion path, and user-facing outbound sanitization.
