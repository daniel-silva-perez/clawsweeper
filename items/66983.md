## Summary

Keep #66983 open. Current main has native/mobile canvas and A2UI support, but it still does not implement the requested first-class web canvas node path: `web` is not a recognized node platform in the command policy, and node-role WebSocket clients cannot bypass device identity/pairing with only shared Gateway token or the Control UI `dangerouslyDisableDeviceAuth` break-glass setting. Existing bootstrap-token and trusted-CIDR auto-approval paths help some trusted node deployments, but they do not satisfy the issue's browser-tab-as-canvas-node request.

## Best Possible Solution

Keep the issue open for maintainer design. The most direct path is an additive gateway/protocol change: add an explicit `web` node platform with a canvas-only default command surface, define how browser node clients create/sign device identity in secure contexts, and add a narrowly scoped, documented node-role auth or trusted-proxy/auto-approval mechanism if maintainers accept the security tradeoff. The implementation should include node-command-policy tests, WebSocket auth/pairing tests for browser/proxy cases, and public docs under the gateway/platforms web-node surface.
