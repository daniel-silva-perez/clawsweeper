## Summary

Keep #51556 open. Current main still has the sync allowFrom silent-swallow behavior the PR is trying to address, but the PR’s original two-line change targets code that has since moved and the existing review correctly points out that a sync-only `readFile` throw would leave async behavior inconsistent.

## Best Possible Solution

Keep the PR open but request an update against current main. The best fix is to define one contract for non-ENOENT allowFrom store failures, implement it in the shared async and sync helper paths, account for existing sync runtime callers, and add focused EACCES/EIO regression tests.
