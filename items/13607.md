## Summary

Current `main` has a per-agent `reasoningDefault` and the runtime/test coverage shows new runs inherit it unless a session override is set, but the exact global-default request is not fully implemented: there is no typed/documented `agents.defaults.reasoningDefault` config surface, and agent resolution does not read such a global default today.

## Evidence

- **Per-agent reasoning default exists:** `reasoningDefault` is defined on agent entries as `"on" | "off" | "stream"` and documented as a per-agent default.
  - file: [src/config/types.agents.ts:85](https://github.com/openclaw/openclaw/blob/68e2d6f0889e5365ecdc8f01ab5e3bf6065cbeeb/src/config/types.agents.ts#L85)

  - file: [src/auto-reply/reply/get-reply-directives.ts:426](https://github.com/openclaw/openclaw/blob/68e2d6f0889e5365ecdc8f01ab5e3bf6065cbeeb/src/auto-reply/reply/get-reply-directives.ts#L426)

## Risks / Open Questions

- For single-agent deployments, `agents.list[].reasoningDefault` may already satisfy the reporter's practical need, so maintainers could decide this is close-enough despite not matching the requested global config shape.
- The checkout appears shallow/snapshot-like, so exact landing commit provenance for the feature could not be established from local history.
- `src/agents/tools/gateway-tool.ts` allowlists `agents.defaults.reasoningDefault`, which suggests an incomplete or planned surface; that inconsistency makes an `implemented_on_main` close too risky.
