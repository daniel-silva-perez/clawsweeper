## Summary

Keep open. Current `main` still routes Feishu inbound messages through a stable per-chat sequential queue before `handleFeishuMessage`, and the queued task still awaits the full reply dispatch path. The gateway only switches to `enqueue-followup` when `isActive` is already true, so there is no strong evidence on `main` that Feishu `messages.queue.mode = "collect"` now bypasses or cooperates with that serial lane well enough to batch followups.
