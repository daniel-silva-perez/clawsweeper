## Summary

Keep open. Current main still has only partial cwd/workspace breadcrumbs, not the requested per-session working-context capsule, status output, compaction/new-session wiring, or docs terminology cleanup. The live related PR #71135 is open, draft, unmerged, and explicitly limited to a storage utility with no runtime or `/status` integration.

## Best Possible Solution

Keep this issue open as the product/API tracker. Review #71135 as a possible first storage primitive, then land follow-up work that updates and persists the capsule from run/exec/subagent/ACP paths, exposes cwd/repo/branch/sandbox/temp-clone/push-target state in `/status`, and updates the workspace and compaction docs to distinguish semantic continuity from working-state continuity.
