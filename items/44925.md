## Summary

Keep open. Current `main` has partial hardening for subagent announce retries and restart-aborted orphan recovery, but it does not fully implement the requested guarantees. The give-up path still ends in internal failure bookkeeping plus logs rather than a user-facing notification, generic subagent timeouts still stop the run without automatic restart, and completion text is still frozen from assistant/tool output without verifying claimed file creation on disk.
