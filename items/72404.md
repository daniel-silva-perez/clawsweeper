## Summary

Keep open. Current main still has the behavior this PR targets: explicit-only model entries without `input` survive provider merging unchanged, and image support treats missing `input` as not image-capable. The PR is also paired with the same-author open bug report #71921 and uses closing syntax, so the maintainer path is review/land/rework the PR rather than cleanup-close it.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate for #71921. The likely product path is to land or refine the merge-layer fix with regression coverage for explicit-only vision-capable models, then separately decide whether to address the Bedrock first-boot staging race and stale catalog refresh.
