## Summary

Keep open. Current main still exposes the affected Alibaba/Qwen-hosted models through the generic OpenAI-compatible stream path, and that path still only promotes structured `delta.tool_calls` into tool-call blocks while preserving `reasoning_content`/reasoning fields as thinking. The Kimi tagged-tool parser exists, but it is scoped to the separate Kimi provider and only rewrites text blocks, not thinking blocks from the Qwen/ModelStudio OpenAI-compatible path.

## Best Possible Solution

Keep this issue open and fix it in provider-owned stream normalization. The likely path is a Qwen/ModelStudio OpenAI-compatible wrapper or shared OpenAI-compatible seam that detects validated tagged Kimi-style tool-call sections in reasoning/thinking content, converts them to structured tool-call blocks, and only changes `stopReason` after successful extraction. Add focused regression coverage for `qwen/kimi-k2.5`, `qwen/MiniMax-M2.5`, and the existing Kimi wrapper so text-block and thinking-block behavior are explicit.
