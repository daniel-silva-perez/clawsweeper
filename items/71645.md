## Summary

PR #71645 should stay open. Current main still sends Mattermost tool-start status as a plain draft-stream update on the active preview, so streamed assistant text can still be replaced before it is flushed. The PR is a focused Mattermost bug fix using existing draft-stream lifecycle primitives and is not implemented or superseded on main.

## Best Possible Solution

Keep PR #71645 open for maintainer review. The best path is to preserve Mattermost streamed assistant text across tool-start events using the existing draft-stream lifecycle, add a focused regression test for partial text followed by `onToolStart`, and update the Mattermost preview-streaming docs if the chosen behavior intentionally creates a separate status post at tool boundaries.
