## Summary

Keep #64980 open. Current main has Matrix outbound live-preview machinery, including `m.replace` edits and MSC4357 live markers, but the inbound Matrix agent context still receives the initial live preview as normal message text and does not expose `is_streaming`, `is_final`, `m.relates_to`, or equivalent metadata to agents. The existing replacement-event filter only drops edit events, not the original live preview event that can contain a partial mention.

## Best Possible Solution

Keep the issue open and solve it in the bundled Matrix channel/core inbound context path. A conservative implementation would detect Matrix live preview/root events before agent dispatch and either defer/suppress them until the final edit or add a typed, channel-agnostic inbound metadata field so agents and hooks can ignore non-final preview content without parsing raw Matrix event shapes.
