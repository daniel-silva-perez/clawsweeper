## Summary

Current main still has the unguarded `provider?.models?.some(...)` lookup, so the crash described by linked issue #66744 remains reachable. This PR is a focused implementation candidate and should stay open for maintainer review or replacement by an equivalent main-branch fix.

## Best Possible Solution

Keep this PR open for maintainer review and either land the narrow `Array.isArray` guard with regression coverage, or apply an equivalent main-branch fix that prevents non-array runtime provider `models` values from crashing provider policy lookup. Treat upstream provider normalization as separate follow-up work, not a reason to close this focused crash fix.
