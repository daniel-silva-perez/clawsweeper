## Summary

Keep open. Current `main` has partial improvements for this area, but the issue is not implemented end-to-end: ACP spawns are now tracked as detached tasks, and isolated hook runs avoid duplicate fallback delivery when the message tool already sent to the resolved target, but hook-triggered ACP still cannot request `mode:"session"` without `thread:true`, the ACP spawn result still does not return a harness/session resume identifier, and hook mappings still lack explicit `suppressAutoDelivery` / `deliveryMode` / topic-thread targeting fields.
