## Summary

Keep this PR open. Current main still has only global memory-core dreaming configuration: it does not type, validate, document, or honor `agents.list[].dreaming.enabled`, and `resolveMemoryDreamingWorkspaces()` still includes every configured agent. The PR is paired with open issue #67413 by the same author and is a plausible implementation candidate, but it still needs maintainer review and likely a fix for the all-agents-disabled fallback behavior before merge.

## Best Possible Solution

Continue reviewing this PR or a successor against #67413. The implementation should add `agents.list[].dreaming` to the exported config type, strict Zod schema, generated schema/help/docs surfaces, and memory dreaming workspace resolution; it should preserve default-agent fallback only when no agents are configured, skip disabled agents when they are explicitly configured, and add tests for mixed enabled/disabled agents, all-disabled agents, and config validation. Maintainers should also decide whether per-agent schedule support from #67413 belongs in this PR or a narrower follow-up.
