## Summary

Keep this PR open. Current main still has the MCP HTTP gateway data-loss bug: `tools/call` sends tool results through `normalizeToolCallContent`, and that helper still rebuilds every array content block as `{ type, text }`, discarding fields needed by image, audio, resource, and resource_link blocks. The PR is a focused implementation candidate for the still-open related issue #68037, but its review comments identify a real follow-up requirement: validate known content block shapes before passthrough instead of accepting any object with a string `type`.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate for the linked MCP HTTP gateway bug. The best path is to update the patch so `normalizeToolCallContent` preserves valid MCP text/image/audio/resource/resource_link blocks with shape checks, falls back only for malformed entries, and adds focused regression tests on the `tools/call` HTTP path.
