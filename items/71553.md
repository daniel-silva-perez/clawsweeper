## Summary

Keep #71553 open. Current main still builds WhatsApp ack-reaction options with only `sender.jid`, so group messages whose normalized sender identity is LID-only do not pass a reaction participant. The PR is a focused fix with a matching regression test, and current code already supports LID participants once they are supplied to the WhatsApp send API.

## Best Possible Solution

Keep the PR open and adapt the patch to current main's `createAckReactionHandle` implementation: set the ack reaction participant to `sender.jid ?? sender.lid` while preserving the current direct-message behavior, then add the LID-only group ack regression test and run the WhatsApp extension changed lanes before merge.
