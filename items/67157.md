Keep open. Current main still does not include this PR's launcher-backed npm bin, stable local tarball upgrade script, live memory-pro host smoke, or launcher e2e coverage. Main has adjacent release/postpublish package validation, but it is not equivalent, and the PR still has unresolved functional and install-surface review risks that need maintainer handling.

The best path is a revised implementation that either keeps `openclaw.mjs` as the canonical launcher and adds local tarball/memory-pro regression coverage around it, or changes the bin target while updating the release guard, entry bootstrap parity, gateway argv detection, package/release tests, and the existing review findings together.

## Risks / Open Questions

- Merging the current PR shape would change the npm-published executable path while the release metadata guard still rejects any bin target other than `openclaw.mjs`.
- The direct run-main launcher can bypass existing `src/entry.ts` bootstrap behavior, including respawn/env/warning/profile/container setup, unless the implementation is reworked for parity.
- Gateway liveness detection may misclassify npm-shim command lines that expose the new launcher path because current argv detection does not recognize that script.
- Security/supply-chain review found no new dependencies, workflow permissions, lockfile changes, or secret access, but the PR does alter the package bin and adds a global install/live execution script, so it remains an install-surface change needing maintainer validation.
- The added live/local scripts have unresolved reliability issues in async error handling, output parsing, tarball path selection, and verifying the intended installed binary rather than another `openclaw` earlier on PATH.
