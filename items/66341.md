## Summary

Keep this PR open. Current main still requires operator.admin for cron.add/update/remove/run and has no persisted creatorScopes or cron execution scope ceiling. The PR is plausible core authorization work, but it is not obsolete or implemented on main, and its branch still needs maintainer/security review plus cleanup of unrelated browser and QA-lab changes.

## Best Possible Solution

Keep this PR open for maintainer review. Ask the author to split unrelated browser WSL2 and QA-lab formatting changes, then review a focused cron-only branch that lowers the intended cron methods, persists creatorScopes at creation time, enforces the ceiling in embedded and CLI/MCP execution paths, and includes regression coverage for cross-scope update/run attempts.
