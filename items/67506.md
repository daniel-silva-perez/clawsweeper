## Summary

Keep this PR open. Current main still has the root project-shard Vitest config forcing `runner: nonIsolatedRunnerPath`, which is the behavior this PR says caused the boundary+tooling shard to hang. The PR’s regression test is also not present on main, and the linked issue has been closed while this PR remains the only provided implementation candidate.

## Best Possible Solution

Review and land this PR, or recreate the same narrow fix on main: remove the root-level `runner: nonIsolatedRunnerPath` override from `test/vitest/vitest.project-shard-config.ts`, let child project configs own runner selection, and keep a regression test asserting project-shard configs leave `test.runner` undefined.
