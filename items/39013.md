## Summary

Keep #39013 open. Current main still has the deterministic `/context list` display bug for path-only virtual bootstrap files: hook-mutated bootstrap files are accepted with a normalized `path`, `buildBootstrapInjectionStats` still copies `file.name`, and `/context list` renders that field directly. The memory/accumulation part should be rechecked separately, but the still-present display bug is enough to keep this issue open.

## Best Possible Solution

Keep the issue open. First fix `buildBootstrapInjectionStats` to derive a stable display label from `file.name`, then `path.basename(file.path)`, then `file.path`, then an explicit placeholder, and add a regression test for a runtime hook-injected `{ path, content, virtual: true }` bootstrap file with no `name`. Then recheck the session-memory/accumulation behavior on current main and either exclude injected bootstrap/system content from session indexing or split that remaining behavior into a narrower tracked issue.
