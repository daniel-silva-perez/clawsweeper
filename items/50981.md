## Summary

Keep #50981 open. Current main still has the timeout propagation gaps the PR targets, but the PR as proposed has unresolved review-blocking semantics and is not superseded by a canonical merged fix in the provided context.

## Best Possible Solution

Keep the PR open and request changes or a replacement patch: forward numeric agent timeout only where the caller is actually waiting, preserve fire-and-forget default run budget unless maintainers choose otherwise, align lock wait with the effective run budget without doubling it, and add regression tests for gateway schema validation, timeoutSeconds=0 behavior, A2A helper timeouts, and contended session locks.
