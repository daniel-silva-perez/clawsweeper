## Summary

Current main still contains the reported formatter bug for punctuation-wrapped Home Assistant entity IDs. The sanitizer strips edge punctuation into a candidate, but the file-like underscore preservation branch still checks the raw token, so backtick-wrapped dotted IDs miss copy-sensitive preservation and are chunked with literal spaces. The issue should remain open until PR #39565 or an equivalent fix lands.

## Best Possible Solution

Keep #39505 open until PR #39565 or an equivalent focused fix lands. The fix should classify the punctuation-stripped core token for file-like and dotted entity IDs, preserve backtick- and quote-wrapped Home Assistant IDs exactly, and add regression tests for bare and markdown-wrapped long dotted IDs without weakening the narrow-terminal long-token wrapping guard.
