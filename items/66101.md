## Summary

Keep this PR open. Current main still lacks the proposed main-specific OpenClaw self-inspection boundary, and the live PR diff changes a security-sensitive exec path with unresolved bypass concerns. The work is in-repo and plausible, but it should be revised before merge rather than closed as obsolete or implemented.

## Best Possible Solution

Keep the PR open for maintainer review or author revision. The best path is to decide whether main should get this narrow self-inspection exception, then revise the implementation to use parser/segment-based detection instead of a command regex, make path handling cross-platform, reject all mutating doctor aliases and flag forms, block mixed shell/pipeline forms, and add focused tests for those bypass cases before merge.
