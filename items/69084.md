## Summary

Keep this PR open. Current main still has the narrow webchat heartbeat visibility gap: live webchat filtering is keyed to `showOk`, while the documented contract treats `showOk` as OK-ack visibility and `showAlerts` as alert-content visibility. The PR is relevant, and the author followed up with manual validation and test updates, but the proposed one-flag swap is not clearly merge-ready because review comments correctly identify the `showOk=true, showAlerts=false` OK-only case as a possible regression.

## Best Possible Solution

Keep the PR open and ask for a precise webchat heartbeat filter: use `showOk` for `HEARTBEAT_OK`/ACK suppression and `showAlerts` for non-OK alert content, with gateway tests for default behavior, `showOk=false/showAlerts=true`, `showOk=true/showAlerts=false`, and both message flags disabled. Broader system-event audience work can remain in #69492, but it should not replace this narrow gateway visibility fix unless that broader fix lands and proves the same webchat cases.
