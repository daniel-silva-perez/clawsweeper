## Summary

Keep open. Current main already clears inherited sessionFile metadata for fresh cron session rollovers, but it does not fully implement this PR's store-path-aware transcript rebinding: cron still derives missing transcript paths with resolveSessionTranscriptPath(runSessionId, agentId), and the executor fallback does the same. The PR discussion and follow-up comments describe that store-aware path resolution is part of the intended fix, so the PR still contains potentially useful work rather than being safely obsolete.
