## Summary

Current `main` does not implement the requested high-priority/bypass-queue system-event path. The CLI and gateway only support normal queued system events plus `now|next-heartbeat` wake modes, and heartbeat handling still defers delivery when the target session lane is busy.

## Evidence

- **CLI surface has no priority/bypass flag:** `system event` only accepts `--text`, `--mode`, and shared RPC flags. `--mode` is restricted to `now|next-heartbeat`, and the CLI sends a normal `wake` request with `{ mode, text }`.
  - file: [src/cli/system-cli.ts:57](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/cli/system-cli.ts#L57)
  - command: `nl -ba src/cli/system-cli.ts | sed -n '1,180p'`
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Gateway handler only enqueues the event:** `systemHandlers["system-event"]` resolves the main session key and calls `enqueueSystemEvent(text, { sessionKey })`; there is no direct message-stream injection or bypass path.
  - file: [src/gateway/server-methods/system.ts:54](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/gateway/server-methods/system.ts#L54)
  - command: `nl -ba src/gateway/server-methods/system.ts | sed -n '1,220p'`
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **System events are still consumed only on a later turn:** Queued system events are drained by `drainSystemEventEntries(sessionKey)` and formatted into `System:` lines during reply/heartbeat prompt construction, which means they surface on the next agent turn rather than immediately in-session.
  - file: [src/auto-reply/reply/session-system-events.ts:85](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/auto-reply/reply/session-system-events.ts#L85)
  - command: `nl -ba src/auto-reply/reply/session-system-events.ts | sed -n '1,220p'`
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Busy session lanes are deferred, not bypassed:** Heartbeat explicitly skips when the resolved session lane has queued work and returns `requests-in-flight`; the wake layer then retries later. The regression test covers that skip behavior, and troubleshooting docs describe it as deferred delivery.
  - file: [src/infra/heartbeat-runner.ts:782](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/infra/heartbeat-runner.ts#L782)
  - command: `nl -ba src/infra/heartbeat-runner.ts | sed -n '760,940p'`
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Docs still describe queued/deferred behavior:** CLI docs say `system event` enqueues an event and the next heartbeat injects it; troubleshooting docs say `requests-in-flight` means the heartbeat wake was deferred. That matches the current code, not the requested bypass behavior.
  - file: [docs/cli/system.md:34](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/docs/cli/system.md#L34)
