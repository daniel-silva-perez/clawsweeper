## Summary

Keep open. Current main still writes session transcript headers and SessionManager-created entries with UTC `toISOString()` timestamps, and the local-offset helper/wrapper from PR #50300 is not present. The PR still targets a real remaining behavior gap, though it likely needs rebasing and maintainer review around the private SessionManager wrapping concerns raised in the PR discussion.

## Best Possible Solution

Keep the PR open, but ask for a rebase onto current main and a maintainer-quality implementation that centralizes local-offset transcript timestamp formatting across manual header creation, raw `SessionManager.open(...).appendMessage(...)` paths, branching/forking, recovery rewrites, gateway injection, and embedded runner/command transcript writes. Prefer a robust adapter or upstream-supported hook over silent private monkey-patching, and add regression tests for header creation plus appended entries preserving the same instant with an explicit local offset.
