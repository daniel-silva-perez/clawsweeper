## Summary

Current main still does not make `backup create --no-include-workspace` materially exclude workspaces that live under the state directory. The default workspace is under `~/.openclaw`, the planner always includes the `state` asset outside `--only-config`, and archive creation passes included asset source paths directly to `tar.c(...)` without a nested filter. CLI and docs copy still says workspace directories/files are excluded or archive size is reduced. Related exclude-pattern work in #40786/#44288 remains open and unmerged, so this issue should stay open.

## Best Possible Solution

Keep this issue open until backup either makes `--no-include-workspace` filter nested workspace directories out of the archived state tree, or intentionally changes CLI/docs copy to say it only skips separate workspace discovery and does not filter the state payload. A strong fix should touch the backup planning/archive path, update `src/cli/program/register.backup.ts` and `docs/cli/backup.md`, and add regression coverage proving a file under `.openclaw/workspace` is absent from the actual archive when workspace exclusion or an explicit exclude is requested. If maintainers prefer generic excludes, review and land a focused version of #44288 or equivalent core work.
