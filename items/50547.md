## Summary

Current main does not support the issue's stated root cause. Discord DMs require explicit pairing first, guild channels require explicit allowlisting and are mention-gated by default, inbound Discord messages are processed by the inbound worker rather than cron jobs, and current tests cover the mentioned guild-message path. This makes the report obsolete/mismatched against current main rather than an actionable maintainer bug on today's code.

## Evidence

- **Discord setup docs require DM pairing before Discord chat works:** The current Discord setup guide says to DM the bot, receive a pairing code, approve it, and only then chat via Discord DM.
  - file: [docs/channels/discord.md:150](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/docs/channels/discord.md#L150)
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)

  - file: [docs/channels/discord.md:181](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/docs/channels/discord.md#L181)

## Close Comment

Closing after a Codex review because I can’t reproduce this as a current `main` bug. On current `main` (`ebd7f19`), Discord setup explicitly requires first DM pairing before Discord DMs are processed (`docs/channels/discord.md:150`, `docs/channels/pairing.md:20`), and guild/server replies are a separate setup step with explicit allowlisting (`docs/channels/discord.md:181`). Guild traffic also defaults to allowlist + mention-gated behavior (`extensions/discord/src/config-schema.test.ts:67`, `extensions/discord/src/monitor/message-handler.preflight.ts:995`), so the empty cron-jobs file is not evidence of a missing Discord handler. Discord inbound work is routed through the inbound worker and the session is created/persisted automatically during message processing (`extensions/discord/src/monitor/message-handler.ts:79`, `extensions/discord/src/monitor/message-handler.process.ts:536`). There is also a current test covering the explicit-mention guild path (`extensions/discord/src/monitor/message-handler.preflight.test.ts:1163`). If this still reproduces on current `main` or a current release after completing DM pairing and configuring the guild allowlist, please open a new issue with current logs and the exact `channels.discord` config shape.
