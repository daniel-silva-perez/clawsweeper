## Summary

Keep open. Current `main` still implements session storage as on-disk `sessions.json` files, and there is no PostgreSQL/MySQL-backed session-store configuration or implementation in core.

## Evidence

- **Session config only exposes a file path:** `SessionConfig` has `store?: string` plus maintenance knobs, but no backend/driver setting for PostgreSQL or MySQL session storage.
  - file: [src/config/types.base.ts:158](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/config/types.base.ts#L158)
  - command: `nl -ba src/config/types.base.ts | sed -n '150,230p'`
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **Default session store path is still `sessions.json`:** The resolver hardcodes the default store to `agents/<agentId>/sessions/sessions.json` and treats the configured `session.store` as a filesystem path template.
  - file: [src/config/sessions/paths.ts:34](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/config/sessions/paths.ts#L34)

- **Session loading is JSON-file based:** `loadSessionStore()` reads the store with `fs.readFileSync(..., "utf-8")` and `JSON.parse(raw)`, which confirms the live implementation is file-backed rather than database-backed.
  - file: [src/config/sessions/store-load.ts:76](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/config/sessions/store-load.ts#L76)

  - file: [src/config/sessions/store.ts:411](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/config/sessions/store.ts#L411)
