## Summary

Keep open. Current `main` does not implement the PR’s `agents.defaults.systemPrompt.identityLine` / `disabledSections` config surface, and the runtime prompt builders still have no way to accept or thread those values. The only documented/default config surfaces today are full `systemPromptOverride` and `promptOverlays`, which are different features.
