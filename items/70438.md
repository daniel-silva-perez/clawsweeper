## Summary

Keep #70438 open. Current main has shipped the related #72299/#70947 non-video WebChat attachment support, so PDFs and other non-video files are no longer silently swallowed. However, the specific requested public configuration surface and text-inline behavior are still absent: WebChat config only exposes chat history truncation, and Gateway currently offloads non-image files as media refs with a hardcoded 5 MB parse limit rather than decoding text/json/csv/code inline behind `text_inline` and `max_inline_text_chars` knobs.

## Best Possible Solution

Keep #70438 open as the remaining product/config follow-up. The best path is to build an explicit `gateway.webchat.attachments` or equivalent Control UI/Gateway policy surface in `src/config/types.gateway.ts`, `src/config/zod-schema.ts`, generated config docs/help, and the Gateway attachment parser, with tests for `pdf: drop|offload`, text/json/csv/code UTF-8 inline fencing plus truncation, max inline chars/bytes, current image behavior, and unsupported binary drop/offload policy. Cross-link #72299/#70947 as the shipped baseline so the issue no longer tracks basic non-image upload support.
