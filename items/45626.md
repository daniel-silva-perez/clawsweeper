## Summary

Keep open. Current `main` at `f03252aaf9cc946dd6a4f6189f30d5c5dcd64022` still does not expose `fileId` in the shared message-tool schema, while the Slack `download-file` and Telegram `sendSticker` paths still consume `fileId` at runtime. I did not find any current channel schema contribution that adds `fileId` back in.

## Evidence

- **Shared message-tool schema still omits fileId:** `buildStickerSchema()` currently defines `emojiName`, `stickerId`, `stickerName`, `stickerDesc`, and `stickerTags`, but no `fileId`. `buildMessageToolSchemaProps()` still spreads that schema into the top-level message tool parameters.
  - file: [src/agents/tools/message-tool.ts:282](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/agents/tools/message-tool.ts#L282)
