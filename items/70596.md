## Summary

Keep PR #70596 open. The reported builtin local embedding cold-start gap still exists on current main, but the PR as described is not safely closeable or merge-obsolete: current gateway startup only schedules the memory startup helper for QMD, the helper itself still skips builtin memory, and the PR discussion has a concrete unresolved risk about poisoning the cached builtin manager after a failed prewarm.

## Best Possible Solution

Keep the PR open and request changes rather than closing it. The best implementation is a narrow explicit-local builtin prewarm that is actually scheduled from post-attach startup, avoids remote/auto provider side effects, does not poison the cached default manager on transient local initialization failure, and adds regression tests for the scheduler path, builtin explicit-local success, non-local skip, and failure/retry behavior. The unrelated provider-runtime test-support change should be split out or justified separately.
