## Summary

Keep open. Current main still sends BlueBubbles inbound message text to the model through raw `BodyForAgent`, while this PR only wraps the legacy `formatInboundEnvelope`/`Body` value. The PR is a plausible hardening direction, but it is not safe to close or merge as-is because the active reply path prefers `BodyForAgent` and derives prompt text from it. The existing review comments correctly identify needed revisions: at minimum the empty-body guard, and more importantly wrapping or structurally isolating the actual model-facing BlueBubbles body.

## Best Possible Solution

Keep this PR open but request a targeted revision before merge: wrap or otherwise isolate the actual BlueBubbles model-facing body (`BodyForAgent` or a structured `UntrustedContext`/user-context path), keep `RawBody`/`CommandBody`/`BodyForCommands` stable for command parsing, add the Discord-style non-empty guard, and update focused BlueBubbles tests plus a prompt-path assertion proving inbound text reaches the agent with the intended untrusted-content boundary.
