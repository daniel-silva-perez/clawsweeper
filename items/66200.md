## Summary

Keep open. Current main still routes BlueBubbles inbound text into raw `BodyForAgent`, and this PR only wraps the legacy `formatInboundEnvelope` `Body` value. The existing review comments are valid: the patch needs to protect the actual model-facing path and should keep the Discord-style non-empty guard.

## Best Possible Solution

Keep this PR open and request a targeted revision. The safest path is to wrap or structurally isolate the actual BlueBubbles model-facing body, likely via `BodyForAgent` or a structured `UntrustedContext` style path, while preserving raw command and dedupe fields. Add the Discord-style non-empty guard and focused BlueBubbles tests that prove inbound text reaches the agent with the intended untrusted-content boundary.

## Risks / Open Questions

- Merging this PR as-is would create a false sense of hardening because the model can still see raw BlueBubbles text through `BodyForAgent`.
- A complete fix must preserve `RawBody`, `CommandBody`, and `BodyForCommands` for command parsing, reset handling, dedupe, and reply metadata while isolating only the model-facing text.
- The existing Greptile and maintainer feedback about the non-empty guard still needs to be addressed so attachment-only or placeholder-only flows do not gain marker-only prompt text.
