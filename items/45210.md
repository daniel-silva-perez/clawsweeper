## Summary

Keep open. Current `main` does not implement persistent per-session/topic cleanup pinning. Session maintenance is still a global config surface, and the only targeted exemption in the codebase is an `activeSessionKey`/`--active-key` escape hatch rather than a stored pinned-session list or per-channel maintenance override.

## Evidence

- **Session maintenance schema is global-only:** `SessionSchema` defines `session.maintenance` with `mode`, `pruneAfter`, `pruneDays`, `maxEntries`, `rotateBytes`, `resetArchiveRetention`, `maxDiskBytes`, and `highWaterBytes`. The object is `.strict()`, so there is no `pinnedKeys` or comparable persistent pin/protect field.
  - file: [src/config/zod-schema.session.ts:73](https://github.com/openclaw/openclaw/blob/40be5ad58187fd20b8defe5fc30e649190c131a8/src/config/zod-schema.session.ts#L73)
  - command: `nl -ba src/config/zod-schema.session.ts | sed -n '20,95p'`
  - sha: [40be5ad58187](https://github.com/openclaw/openclaw/commit/40be5ad58187fd20b8defe5fc30e649190c131a8)
- **Existing per-channel overrides apply to reset, not maintenance:** The same schema has `resetByType` and `resetByChannel`, but those are for `session.reset` behavior only. There is no matching `maintenanceByType` or `maintenanceByChannel` surface implementing the issue's alternative proposal.
  - file: [src/config/zod-schema.session.ts:42](https://github.com/openclaw/openclaw/blob/40be5ad58187fd20b8defe5fc30e649190c131a8/src/config/zod-schema.session.ts#L42)
