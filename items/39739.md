## Summary

Keep open. Current main still only accepts the global "*" wildcard or an exact parsed URL origin for `gateway.controlUi.allowedOrigins`; there is no scheme-level wildcard path for `chrome-extension://*`, and related PR #38290 explicitly excludes wildcard/prefix expansion while PR #39867, which targeted this issue, is closed unmerged.

## Best Possible Solution

Implement a conservative, tested `gateway.controlUi.allowedOrigins` matcher for `chrome-extension://*` if maintainers accept the product/security tradeoff. The change likely belongs in `src/gateway/origin-check.ts` with focused tests in `src/gateway/origin-check.test.ts` and docs in the public Gateway/Control UI configuration pages. A stricter scheme-specific implementation is preferable to generic arbitrary prefix wildcards unless maintainers explicitly want broader pattern support.
