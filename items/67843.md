## Summary

Keep this PR open. Current main still omits `agentDir` and `workspaceDir` from the provider stream wrapper context, while the OpenAI/Codex wrapper path reads `ctx.agentDir` and native Codex auth lookup requires `agentDir` to inspect the auth-profile store. The PR is a focused implementation candidate for a real remaining gap linked to #65909, not obsolete cleanup.

## Best Possible Solution

Review and land this PR, or cherry-pick the same narrow fix: pass `agentDir` and `workspaceDir` into `wrapProviderStreamFn` from `applyExtraParamsToAgent` and keep the regression test that captures provider wrapper context. After applying it on current main, run the targeted extra-params test and the normal changed TypeScript gate.
