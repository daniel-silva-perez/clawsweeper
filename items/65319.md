## Summary

Keep open. Current main improves command-alias diagnostics when a plugin manifest declares ownership, but the ownerless fallback still recommends adding the typed command token to `plugins.allow`, while config validation still treats unknown command tokens as stale plugin ids. The maintainer discussion explicitly left this open and called #64732 adjacent rather than canonical.

## Best Possible Solution

Keep this issue open until the lossless path is verified end to end. The best fix is either for lossless-claw to declare `lcm` and `lossless` as manifest `commandAliases` with `runtime-slash` metadata, or for OpenClaw core to make the unknown-command fallback consult available plugin/runtime command metadata before emitting `plugins.allow` advice. Coordinate broader fallback cleanup with #64732, but do not treat it as a duplicate unless maintainers designate it as canonical.
