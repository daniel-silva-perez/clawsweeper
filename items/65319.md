## Summary

Keep open. Current main has generic manifest-backed command-alias diagnostics, but the affected external lossless-claw package still does not declare the needed alias metadata, and current core still falls back to telling users to add an unknown command token to plugins.allow. The maintainer thread also explicitly left this open and treated #64732 as adjacent rather than a canonical duplicate.

## Best Possible Solution

Keep this issue open until the lossless case is verified end to end. The best fix is either to update the lossless-claw plugin manifest with commandAliases for lcm and lossless, or to extend OpenClaw core diagnostics to consult available runtime/native command metadata before emitting plugins.allow advice. Coordinate any broader fallback cleanup with #64732, then close only after a shipped package or core change proves openclaw lcm, openclaw lossless, and command-token entries in plugins.allow produce non-contradictory guidance.
