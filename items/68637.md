## Summary

Keep this PR open. Current main still rejects empty or whitespace-only `write.content`: `REQUIRED_PARAM_GROUPS.write` requires `content` without `allowEmpty`, while `assertRequiredParams` rejects blank strings unless that flag is set. The linked issue #68636 and earlier PR #40797 are both closed, but the current checked-out main SHA still lacks the requested behavior, so this PR remains a plausible focused fix rather than obsolete.

## Best Possible Solution

Review and land this PR, or an equivalent one-flag change plus focused regression tests, so `write` can create/truncate empty or whitespace-only files while still rejecting missing, null, undefined, or non-string `content`. If maintainers intentionally want empty writes rejected, close with an explicit product rationale and adjust tests/docs to make that contract clear.
