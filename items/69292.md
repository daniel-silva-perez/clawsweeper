## Summary

Keep open. Current main has not implemented the PR's proposed doctor auto-enable skip, and no provided related PR is merged or canonically supersedes it. The PR is also not cleanly closeable because current activation policy makes the workspace-plugin part risky: workspace plugins are disabled by default unless allowlisted or explicitly enabled, which matches the unresolved review concern that simply skipping `plugins.entries` for workspace channel plugins could leave them inactive.

## Best Possible Solution

Keep the PR open, but ask for a maintainer/author revision: preserve the intended global installed-plugin skip if that is still the desired doctor behavior, do not skip required workspace-plugin trust materialization, and add regression coverage for restrictive `plugins.allow` plus workspace/global/bundled channel cases. If the remaining user-visible warning is actually the bundled-vs-installed duplicate diagnostic, track or fix that at the manifest-registry layer instead of closing this PR as obsolete.
