## Summary

Keep this PR open. Current main still lacks the additive realtime consult thinking-level and fast-mode override surfaces for both Voice Call and Control UI Talk. Voice Call still derives consult thinking from the selected model default, the shared realtime consult runtime cannot receive fast mode, and Talk realtime config/protocol/UI handoff still omits the proposed one-shot overrides. Related #71849 is broader fast context retrieval work, not a duplicate or superseding implementation.

## Best Possible Solution

Keep this PR open for maintainer review. If accepted, land or adapt the additive implementation so Voice Call and Control UI Talk realtime consults can pass one-shot thinking-level and fast-mode overrides without mutating persistent session state, while keeping #71849 as the separate canonical follow-up for realtime voice fast context retrieval.
