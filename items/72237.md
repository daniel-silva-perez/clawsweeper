## Summary

Keep this PR open. Current main still lacks the additive realtime consult thinking-level and fast-mode override surfaces for both Voice Call and Control UI Talk. The PR targets a concrete behavior gap: configuring the full OpenClaw agent run behind realtime `openclaw_agent_consult` calls without mutating normal session state. Related #71849 remains broader fast context retrieval work, not a superseding implementation.

## Best Possible Solution

Keep this PR open for maintainer review. If accepted, land or adapt the additive implementation so Voice Call and Control UI Talk realtime consults can pass one-shot thinking-level and fast-mode overrides without mutating persistent session state, while leaving #71849 as the separate follow-up for realtime voice fast context retrieval.
