## Summary

Current `main` does not implement the requested `openclaw approvals pending` CLI surface. The CLI only exposes `get`, `set`, and `allowlist`; pending approval listing exists only in lower-level gateway/MCP plumbing, so the issue is still actionable.

## Evidence

- **CLI subcommands stop at get/set/allowlist:** `registerExecApprovalsCli` defines `approvals get`, `approvals set`, and `approvals allowlist`, with no `pending` subcommand or equivalent watcher/json listing entrypoint.
  - file: [src/cli/exec-approvals-cli.ts:474](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/src/cli/exec-approvals-cli.ts#L474)
  - command: `nl -ba src/cli/exec-approvals-cli.ts | sed -n '474,619p'`
  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
- **CLI docs do not document pending listing:** The approvals CLI reference documents `get`, `set`, and `allowlist add|remove`, and its common-options section only describes those commands. There is no documented `pending` subcommand.
  - file: [docs/cli/approvals.md:60](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/docs/cli/approvals.md#L60)
  - command: `nl -ba docs/cli/approvals.md | sed -n '60,185p'`
  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
- **Gateway already has pending-approval list plumbing:** Gateway request handlers expose `exec.approval.list`, which returns all pending approval records, and the manager has `listPendingRecords()` for unresolved approvals. This shows the backend primitive exists, but it is not wired into the OpenClaw CLI requested by the issue.
  - file: [src/gateway/server-methods/exec-approval.ts:97](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/src/gateway/server-methods/exec-approval.ts#L97)

  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
- **Pending approval listing exists in MCP, not CLI:** The MCP bridge exposes `permissions_list_open` to list open exec/plugin approvals it has observed. That is adjacent functionality on a different surface, not the requested `openclaw approvals pending` CLI command.
  - file: [src/mcp/channel-tools.ts:159](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/src/mcp/channel-tools.ts#L159)
