## Summary

Keep open. Current main still has the background exec notify-on-exit path that formats failed exec output as an untrusted system event and then treats exec completions as user-relay heartbeat prompts. I did not find a Discord-specific suppression, sanitization, or card-only routing path for these events.

## Best Possible Solution

Keep this issue open and fix it in the exec notification or heartbeat-to-channel delivery boundary. A conservative fix would preserve exec-card updates while preventing background exec failure system events from being sent as raw `System (untrusted)` Discord chat messages: suppress Discord user relay for exec notify events that already have a visible exec card, sanitize the failure text into a short user-facing summary, or add a scoped channel/config policy. The existing global `tools.exec.notifyOnExit=false` behavior is only a broad workaround, not a Discord-specific default fix.
