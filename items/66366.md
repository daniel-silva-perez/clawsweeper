## Summary

Keep open. The prior ClawSweeper comment remains accurate against current main: background exec failures still become untrusted system events, exec-event heartbeats still build a user-relay prompt from those event details when Discord delivery is available, and the Discord channel plugin does not add a Discord-specific suppression, sanitization, or card-only path.

## Best Possible Solution

Keep this issue open and fix it at the exec notification or heartbeat-to-channel delivery boundary. A conservative maintainer path is to preserve exec card updates and legitimate async completion notifications, but suppress Discord user relay for exec notify events already represented by an exec card, sanitize failures into a short user-facing summary, or add a scoped channel/config policy instead of relying on the global `tools.exec.notifyOnExit=false` workaround.
