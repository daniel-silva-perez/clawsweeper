## Summary

Keep #72019 open. Current main still sends the same subagent task through both the subagent system prompt and the first child user message; the linked PRs are unmerged fix attempts, not shipped behavior.

## Best Possible Solution

Keep #72019 open until a narrow core fix lands. The fix should ensure the long subagent task appears exactly once in the first child request, preserve multiline task text, and add regression coverage around the assembled subagent system prompt plus initial child user message. Among the linked attempts, #72053 appears closest because it also addresses multiline preservation, but maintainers should choose and land one canonical implementation.
