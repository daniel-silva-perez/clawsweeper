## Summary

Keep this open. Current main still does not discover, diagnose, or repair the reported layout where the real config is `~/.openclaw/moltbot.json` and the canonical `openclaw.json` is missing or skeletal. The related #54201 symptom work improved gateway messaging/recovery, but it does not close this migration gap.

## Best Possible Solution

Keep this open for a narrow config-recovery fix. Avoid restoring broad runtime legacy auto-detection, but add an explicit doctor/startup diagnostic for `~/.openclaw/moltbot.json` when canonical `openclaw.json` is missing, empty, or skeletal; offer a safe `doctor --fix` copy/merge path when feasible; and update the public migration/changelog wording so users can recover without guessing.
