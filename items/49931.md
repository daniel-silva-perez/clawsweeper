## Summary

Keep open. Current `main` still lacks a configurable `tools.exec.shell` setting, and Windows gateway exec remains hardwired to PowerShell selection rather than a user override.

## Evidence

- **Exec config type has no shell field:** `ExecToolConfig` exposes `host`, `security`, `ask`, `node`, `pathPrepend`, `safeBins`, and related settings, but no `shell` override.
  - file: [src/config/types.tools.ts:242](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/src/config/types.tools.ts#L242)
  - command: `nl -ba src/config/types.tools.ts | sed -n '240,285p'`
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **Runtime schema rejects unknown exec keys:** The runtime zod schema for `tools.exec` includes no `shell` property and is `.strict()`, so `tools.exec.shell` is not currently accepted config.
  - file: [src/config/zod-schema.agent-runtime.ts:456](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/src/config/zod-schema.agent-runtime.ts#L456)

  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **Docs still describe fixed Windows shell choice:** The exec docs state that non-Windows uses `SHELL`, while Windows prefers PowerShell 7 and falls back to Windows PowerShell 5.1; the config section lists no `tools.exec.shell`.
  - file: [docs/tools/exec.md:73](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/docs/tools/exec.md#L73)

- **Gateway exec hardcodes PowerShell on Windows:** `getShellConfig()` returns `resolvePowerShellPath()` with `-NoProfile -NonInteractive -Command` on `win32`.
  - file: [src/agents/shell-utils.ts:54](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/src/agents/shell-utils.ts#L54)

  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **Exec runtime always uses the platform-selected shell helper:** Host exec builds `childArgv` directly from `getShellConfig()` and the raw command string, so there is no per-config shell override path in the gateway runtime.
  - file: [src/agents/bash-tools.exec-runtime.ts:669](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/src/agents/bash-tools.exec-runtime.ts#L669)

  - file: [src/infra/node-shell.ts:3](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/src/infra/node-shell.ts#L3)
