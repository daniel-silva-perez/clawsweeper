## Summary

Keep open. Current `main` still places the `New session` and `Send` actions adjacent in the composer toolbar with 30x30 hit targets and a 4px gap, and there is no mobile-specific override that separates them. The current `/new` path also does not preserve the in-progress draft despite passing `restoreDraft: true`, so an accidental tap can still disrupt composition.

## Evidence

- **New session and Send are still adjacent in the same toolbar group:** `renderChatRunControls` renders the `New session` button and the `Send` button inside the same `.agent-chat__toolbar-right` container with no separator or confirmation step.
  - file: [ui/src/ui/chat/run-controls.ts:20](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/ui/src/ui/chat/run-controls.ts#L20)
