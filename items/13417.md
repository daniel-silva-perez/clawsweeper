## Summary

Keep open. Current `main` still rejects WhatsApp newsletter/channel JIDs: outbound target resolution only supports normalized E.164 user targets or `@g.us` group JIDs, and any other `@...` target is normalized to `null` and fails with the existing `<E.164|group JID>` error.

## Evidence

- **Outbound message path still uses WhatsApp target resolver:** The WhatsApp outbound adapter resolves every target through `resolveWhatsAppOutboundTarget(...)` before sending.
  - file: [extensions/whatsapp/src/outbound-adapter.ts:36](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/extensions/whatsapp/src/outbound-adapter.ts#L36)
  - sha: [3361593442e2](https://github.com/openclaw/openclaw/commit/3361593442e295e9bb1f55cbfb38861ea9413e45)
- **Resolver error contract still only allows E.164 or group JID:** `resolveWhatsAppOutboundTarget` returns `missingTargetError("WhatsApp", "<E.164|group JID>")` when normalization fails, and only special-cases `isWhatsAppGroupJid(...)` for accepted JIDs.
  - file: [extensions/whatsapp/src/resolve-outbound-target.ts:32](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/extensions/whatsapp/src/resolve-outbound-target.ts#L32)
  - sha: [3361593442e2](https://github.com/openclaw/openclaw/commit/3361593442e295e9bb1f55cbfb38861ea9413e45)
- **Normalizer rejects non-`@g.us` WhatsApp JIDs:** `normalizeWhatsAppTarget` accepts `@g.us` group JIDs and known user JID forms, but if the candidate still contains `@` after those checks it returns `null`. That rejects `xxxxx@newsletter`.
  - file: [extensions/whatsapp/src/normalize-target.ts:74](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/extensions/whatsapp/src/normalize-target.ts#L74)
  - sha: [3361593442e2](https://github.com/openclaw/openclaw/commit/3361593442e295e9bb1f55cbfb38861ea9413e45)
- **Current config/docs still describe WhatsApp targets as E.164 or group JID only:** The type comment for `defaultTo` says `(E.164 or group JID)`, and the WhatsApp docs say `allowFrom` accepts E.164-style numbers.
  - file: [src/config/types.whatsapp.ts:62](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/src/config/types.whatsapp.ts#L62)
  - sha: [3361593442e2](https://github.com/openclaw/openclaw/commit/3361593442e295e9bb1f55cbfb38861ea9413e45)
- **No newsletter support surfaced anywhere on current main:** Repo-wide search found no `newsletter`, `@newsletter`, or newsletter-specific WhatsApp actions in source/docs/tests.
  - command: `rg -n "newsletterMetadata|newsletterFollow|newsletterUnfollow|newsletterReactMessage|sendMessage\(.*newsletter|@newsletter" . -g '!node_modules'`
  - sha: [3361593442e2](https://github.com/openclaw/openclaw/commit/3361593442e295e9bb1f55cbfb38861ea9413e45)
