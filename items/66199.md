## Summary

Keep open. Current main has not shipped the iMessage hardening, and the submitted patch is not safe to merge as-is because it wraps only the legacy `Body` envelope while the active prompt path prefers raw `BodyForAgent`/`BodyStripped`. The existing review comments correctly identify the same bypass, so the useful outcome is to revise this PR or replace it with a patch that protects the actual model-bound iMessage text path.

## Best Possible Solution

Keep this PR open for revision, but do not merge the submitted patch as-is. The fix should protect the actual iMessage model-bound body path, likely by separating command/raw parsing from the prompt body and ensuring the text consumed through `BodyForAgent`/`BodyStripped` or an equivalent prompt assembly path carries the untrusted-content boundary. Add focused iMessage tests proving the model prompt path is protected while command parsing, reply context, history, and echo behavior remain deterministic.
