## Summary

Keep #70014 open. Current main still appears capable of skipping the primary/default workspace when `agents.list` is present: dreaming resolves workspaces from the configured agent list and only falls back to the default agent when that list is empty. The related #65653 session cleanup/narrative-session work is separate and does not close this enumeration bug.

## Best Possible Solution

Keep the issue open and fix the memory-core workspace resolver or its caller so managed dreaming includes the primary/default workspace in addition to configured agent workspaces, with path deduplication and correct `agentIds`. Add a regression test using `agents.defaults.workspace` or a `main` primary workspace plus the four AGI agents to assert all five workspaces are processed; if maintainers intend configured-only behavior, document that explicitly and answer the reporter with the required config shape.
