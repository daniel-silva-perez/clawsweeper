## Summary

Keep this issue open. Current main still builds memory-core dreaming workspaces from configured agents first, and only uses the runtime primary workspace as a fallback when no configured workspace resolves. A setup with configured AGI sub-agent workspaces can therefore still skip the primary workspace reported here.

## Best Possible Solution

Keep this issue open and fix memory-core dreaming workspace enumeration so scheduled dreaming processes the runtime/primary workspace plus configured agent workspaces, deduped by normalized path. The likely implementation belongs in the runtime dreaming path that already receives ctx.workspaceDir, with matching doctor/status handling and regression coverage for a non-empty agents.list of sub-agents while the primary workspace is only available from runtime/status context.
