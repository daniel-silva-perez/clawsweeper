## Summary

Keep open. Current `main` still appears to implement the duplicate path described in #44922: a main-session cron job enqueues the raw `systemEvent` into the session queue, then the heartbeat runner separately detects the same tagged cron event and builds the `"A scheduled reminder has been triggered..."` wrapper prompt, including for internal-only delivery.

## Evidence

- **Main-session cron still enqueues raw system event and immediately triggers cron heartbeat:** `executeMainSessionCronJob` enqueues the job text into the target main session with `contextKey: \`cron:${job.id}\`` and then calls `runHeartbeatOnce` with `reason: \`cron:${job.id}\``. That preserves both halves of the duplicate path reported in the issue.
  - file: [src/cron/service/timer.ts:1206](https://github.com/openclaw/openclaw/blob/40be5ad58187fd20b8defe5fc30e649190c131a8/src/cron/service/timer.ts#L1206)

  - sha: [40be5ad58187](https://github.com/openclaw/openclaw/commit/40be5ad58187fd20b8defe5fc30e649190c131a8)
- **Heartbeat preflight still opts into tagged cron events:** `resolveHeartbeatPreflight` peeks pending system events, treats any `contextKey` starting with `cron:` as tagged cron work, and enables pending-event inspection/bypass based on that tag.
  - file: [src/infra/heartbeat-runner.ts:564](https://github.com/openclaw/openclaw/blob/40be5ad58187fd20b8defe5fc30e649190c131a8/src/infra/heartbeat-runner.ts#L564)

- **Heartbeat run prompt still wraps tagged cron event text in reminder prompt:** `resolveHeartbeatRunPrompt` filters pending events where the run reason is cron or the event has a `cron:` context key, then calls `buildCronEventPrompt(cronEvents, ...)`. There is no branch here that suppresses the wrapper for `sessionTarget: "main"` or for `delivery.mode: "none"`.
  - file: [src/infra/heartbeat-runner.ts:673](https://github.com/openclaw/openclaw/blob/40be5ad58187fd20b8defe5fc30e649190c131a8/src/infra/heartbeat-runner.ts#L673)

That means the raw cron `systemEvent` is still present in prompt/session history separately from the heartbeat wrapper.
