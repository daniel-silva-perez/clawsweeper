## Summary

Keep open. Current `main` still re-applies Tailscale Serve/Funnel on gateway startup without any preservation path for existing Funnel state, and it still rejects `gateway.tailscale.mode="funnel"` unless gateway auth mode is `password`.

## Evidence

- **Startup always reapplies Tailscale exposure:** Gateway post-attach startup still calls `startGatewayTailscaleExposure(...)` whenever `tailscaleMode` is not `off`, so every restart re-runs the Tailscale setup path.
  - file: [src/gateway/server-startup-post-attach.ts:457](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/gateway/server-startup-post-attach.ts#L457)
  - command: `nl -ba src/gateway/server-startup-post-attach.ts | sed -n '455,470p'`
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **No preservation logic before enabling Serve:** `startGatewayTailscaleExposure` unconditionally calls `enableTailscaleServe(params.port)` for `serve` mode and has no status/probe branch to preserve an existing Funnel setup.
  - file: [src/gateway/server-tailscale.ts:21](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/gateway/server-tailscale.ts#L21)

  - file: [src/infra/tailscale.ts:409](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/infra/tailscale.ts#L409)
