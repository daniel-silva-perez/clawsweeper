## Summary

Current main still has partial Codex OAuth/usage support, but it does not implement the requested product contract: explicit quota_wait versus reauth_required diagnosis, Control UI quota-wait guidance/actions, or a brokered host-side reauth execution API. The existing ClawSweeper keep-open comment remains consistent with the newer main SHA.

## Best Possible Solution

Keep this open as a core/provider plus Control UI product issue. Extend the existing models.authStatus/provider-usage path with an explicit diagnostic object such as ok, expiring_soon, quota_wait, and reauth_required; preserve the existing WHAM-aware cooldown behavior; surface Codex 5-hour and weekly renewal details in local time; and add a narrow authorized broker/adapter execution API that is only available when diagnosis is reauth_required. Broader reason-aware cron guardrails can remain tracked separately in #14376.
