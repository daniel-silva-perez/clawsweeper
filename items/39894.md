## Summary

Current main still does not implement the PR's central outbound-normalization fix. The shared outbound payload plan parses raw reply text before delivery/JSON projection, so copied inbound metadata can remain visible when mixed with normal text and MEDIA directives. The PR is not obsolete, although it is mixed with unrelated CI/macOS/test-runner work and needs a narrow rebase or split.

## Best Possible Solution

Keep PR #39894 or its canonical bug #39847 open until current main gets a narrow core fix. The likely fix is to strip copied inbound metadata before `parseReplyDirectives()` in the shared outbound payload plan, with a regression test covering `Conversation info (untrusted metadata)` followed by visible text plus `MEDIA:`. Split or drop the unrelated CI/macOS/test-runner changes because at least some of that work has already landed separately.
