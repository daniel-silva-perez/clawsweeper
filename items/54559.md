## Summary

Current `main` still matches the report. `genericRepeat` remains warn-only in `src/agents/tool-loop-detection.ts:541`-`560`, with no `criticalThreshold` branch, while blocking is reserved for `known_poll_no_progress`, `ping_pong`, `unknown_tool_repeat`, and the global breaker in `src/agents/tool-loop-detection.ts:446`-`539`. `process` is only treated as a known poll tool for `action === "poll" || action === "log"` in `src/agents/tool-loop-detection.ts:155`-`164`, so `process({ action: "list" })` falls into the warn-only generic path. The current tests explicitly lock this behavior in `src/agents/tool-loop-detection.test.ts:320`-`332` and `src/agents/pi-tools.before-tool-call.e2e.test.ts:226`-`244`, and the docs still describe `criticalThreshold` as blocking repetitive loops in `docs/tools/loop-detection.md:69`-`74` and `docs/gateway/config-tools.md:138`-`145`.
