## Summary

Keep this PR open. Current main has the related per-session bootstrap refresh from #72406/#64871, but the lower-level guarded workspace file cache still keys identity only by canonical path, dev, inode, size, and mtime. The ctime hardening proposed here remains a focused, unsuperseded follow-up.

## Best Possible Solution

Review and land this PR, or an equivalent maintainer patch, that adds `stat.ctimeMs` to `workspaceFileIdentity`, updates the cache identity comment, and keeps focused regression coverage for same-inode, same-size edits with restored mtime. After merge, close this PR as implemented by that landed commit.
