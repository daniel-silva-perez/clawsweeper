## Summary

Current main still has the reported TUI /sessions title path: transcript title extraction returns the gateway-injected `Sender (untrusted metadata)` prefix as user text, the gateway derives titles from the raw session entry instead of the computed row `displayName`, and the TUI prefers `derivedTitle` over `displayName`. The linked fix PR #39790 is still open/unmerged in the provided context, so this issue should remain open until that or an equivalent fix lands.

## Best Possible Solution

Keep #39722 open until PR #39790 or an equivalent maintainer-approved patch lands. The fix should strip inbound metadata before title/preview extraction, ensure the computed gateway row displayName is preferred or passed into title derivation, preserve the TUI fallback behavior intentionally, and add regression tests for both the metadata envelope and external displayName precedence.
