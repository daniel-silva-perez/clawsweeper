## Summary

Keep #39608 open. Current main still creates the WhatsApp Baileys socket with `syncFullHistory: false` and no explicit `shouldSyncHistoryMessage`, while bundled Baileys rc.9 defaults an omitted predicate to `!!syncFullHistory`. That keeps the reported linked-device group-sync root cause plausible. Recent issue comments also report the same no-upsert/no-log symptom on 2026.3.28 and 2026.4.20, and the related #14069 was closed stale rather than merged or left as a canonical live tracker.

## Best Possible Solution

Keep this issue open until the WhatsApp plugin deliberately handles linked-device group sync. The likely code path is `extensions/whatsapp/src/session.ts`: pass an explicit Baileys `shouldSyncHistoryMessage` predicate, or add a documented config/product choice if maintainers do not want that default. Add a focused regression test in `extensions/whatsapp/src/session.test.ts` asserting the `makeWASocket` options, then live-smoke a re-link/reconnect group mention path if credentials are available. If maintainers decide linked-device group inbound after relink is unsupported, update `docs/channels/whatsapp.md` and `docs/channels/group-messages.md` with that limitation instead of closing as fixed.
