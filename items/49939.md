## Summary

Current `main` does not implement the requested default 1M context window for Anthropic Claude Opus 4.6 / Sonnet 4.6. The only built-in 1M normalization in the Anthropic plugin is for Opus 4.7, so the issue should remain open.

## Evidence

- **Anthropic plugin only upgrades Opus 4.7 to 1M:** `applyAnthropicOpus47ContextWindow` returns a 1,048,576-token context only when `isAnthropicOpus47Model(modelId)` is true, and both `resolveDynamicModel` and `normalizeResolvedModel` use that function. There is no equivalent 4.6/Sonnet 4.6 upgrade path.
  - file: [extensions/anthropic/register.runtime.ts:323](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/extensions/anthropic/register.runtime.ts#L323)
  - command: `nl -ba extensions/anthropic/register.runtime.ts | sed -n '320,420p' && nl -ba extensions/anthropic/register.runtime.ts | sed -n '548,580p'`
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **Shared context cache special-cases only Opus 4.7:** `src/agents/context.ts` defines `ANTHROPIC_CONTEXT_1M_TOKENS = 1_048_576`, but the discovery override path applies it only via `shouldUseDiscoveredAnthropicOpus47ContextWindow(model.id)`. No 4.6/Sonnet 4.6 equivalent appears in this runtime path.
  - file: [src/agents/context.ts:27](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/src/agents/context.ts#L27)
  - command: `nl -ba src/agents/context.ts | sed -n '1,120p'`
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **Current tests still model Opus 4.6 as 200k and only verify 4.7 normalization to 1M:** Anthropic plugin tests create the `claude-opus-4-6` template with `contextWindow: 200_000`, then assert that `claude-opus-4-7` resolves/normalizes to `1_048_576`. There is no corresponding assertion that 4.6 or Sonnet 4.6 default to 1M.
  - file: [extensions/anthropic/index.test.ts:201](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/extensions/anthropic/index.test.ts#L201)
  - command: `nl -ba extensions/anthropic/index.test.ts | sed -n '200,320p'`
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **User override exists, but only as explicit config metadata:** The config schema accepts `contextWindow` on provider model definitions, and catalog metadata overlay uses a configured `contextWindow` when present. That means a manual override path exists, but it does not satisfy the issue's request to update the built-in Anthropic 4.6 defaults.
  - file: [src/config/zod-schema.core.ts:337](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/src/config/zod-schema.core.ts#L337)
