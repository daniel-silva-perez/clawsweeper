## Summary

Keep open. Current `main` still computes Discord channel allowlist matches with `channelConfig?.allowed !== false` in the active inbound preflight path, so there is no basis to close this PR as already implemented or obsolete. The same unsafe default also remains in other Discord policy helpers, which means the underlying allowlist gap is still present on `main` and a conservative cleanup should not close the item.
