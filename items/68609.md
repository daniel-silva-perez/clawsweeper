## Summary

Keep open. Current main still has the observability gap described in the report: the preemptive compaction helper returns `route: "fits"` with the requested diagnostic fields, but the embedded run call site only logs `[context-overflow-precheck]` for truncation or compaction routes. I did not find an implementation, test, or related shipped change that logs every precheck evaluation or otherwise makes this issue obsolete.

## Best Possible Solution

Keep the issue open. The likely maintainer path is to add a debug-level precheck decision log or structured trace immediately after `shouldPreemptivelyCompactBeforePrompt()` in `src/agents/pi-embedded-runner/run/attempt.ts`, including `route`, `estimatedPromptTokens`, `promptBudgetBeforeReserve`, `overflowTokens`, reserve/effective reserve, provider/model, and session identifier, then add focused regression coverage for the `fits` route. Separately, verify whether cache-ttl pruning or context assembly can hide pressure from the estimate for large-context Anthropic sessions before changing compaction thresholds.
