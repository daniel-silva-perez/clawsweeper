## Summary

Keep #72031 open. Current main at 6b6dcaf still has the reported bug: media-understanding image and audio/video auth paths call requireApiKey unconditionally, while Bedrock resolves to aws-sdk auth without a static apiKey. The related fix PR #72092 is open/unmerged and the earlier PR #72066 was closed unmerged, so the fix has not shipped on main.

## Best Possible Solution

Keep #72031 open until PR #72092 or an equivalent patch lands on main. The best implementation is the narrow media-understanding fix described there: allow empty apiKey only when ResolvedProviderAuth.mode is aws-sdk, avoid writing empty runtime API keys, preserve existing errors for all other auth modes, and add regression coverage for amazon-bedrock image plus the audio/video auth resolver path.
