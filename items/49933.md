## Summary

Keep open. Current `main` still resolves `agents.defaults.memorySearch.remote.batch.timeoutMinutes` into batch state, but the non-batch builtin indexing path ignores that config and uses hardcoded remote/local embedding timeouts instead. The direct remote provider fetch stack also does not accept or forward a configurable timeout, so there is no strong evidence that #49933 is fixed on `main` or in the latest release.

## Evidence
