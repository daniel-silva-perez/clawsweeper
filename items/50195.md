## Summary

Keep open. Current `main` does not implement either requested surface: the WhatsApp outbound send path has no `linkPreview` passthrough or related config, and gateway RPC still exposes only single-message `send` rather than a batch send method.

## Evidence

- **WhatsApp send options do not include link preview:** `ActiveWebSendOptions` only defines `quotedMessageKey`, `gifPlayback`, `accountId`, and `fileName`; there is no `linkPreview` field to pass through from callers.
  - file: [extensions/whatsapp/src/inbound/types.ts:12](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/extensions/whatsapp/src/inbound/types.ts#L12)
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **Plain-text WhatsApp sends still build `{ text }` only:** The outbound WhatsApp send path falls back to `payload = { text }` for non-media messages, then sends that payload directly to Baileys. There is no branch that forwards prebuilt preview metadata.
  - file: [extensions/whatsapp/src/inbound/send-api.ts:75](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/extensions/whatsapp/src/inbound/send-api.ts#L75)
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **WhatsApp config schema lacks preview controls:** The WhatsApp config type/schema includes delivery controls like `textChunkLimit`, `chunkMode`, `mediaMaxMb`, `replyToMode`, and `reactionLevel`, but no `linkPreview` or `generateHighQualityLinkPreview` setting.
  - file: [src/config/types.whatsapp.ts:81](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/src/config/types.whatsapp.ts#L81)
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **Gateway `send` RPC is single-message only:** `SendParamsSchema` accepts one `to` plus one optional `message`/`mediaUrl`/`mediaUrls` payload. There is no `messages` array or batch shape in the protocol schema.
  - file: [src/gateway/protocol/schema/agent.ts:87](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/src/gateway/protocol/schema/agent.ts#L87)
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **No batch send RPC found on current main:** Repository search for `send.batch`, `batch_send`, or `sendMessages` under gateway/docs/agents returned no matches, which is consistent with the server-method surface only exposing `send`.
  - command: `rg -n 'send\.batch|batch_send|sendMessages|batch send' src/gateway docs src/agents -S`
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **User-facing WhatsApp docs do not document preview or batch-send support:** The WhatsApp docs list delivery knobs such as `textChunkLimit`, `chunkMode`, `mediaMaxMb`, `sendReadReceipts`, `ackReaction`, and `reactionLevel`; there is no documented link-preview or batch-send feature.
  - file: [docs/channels/whatsapp.md:570](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/docs/channels/whatsapp.md#L570)
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
