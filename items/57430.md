## Summary

Keep open. Current `main` has a generic fix that raises the implicit default thinking level to `medium` for reasoning-capable models, but `openrouter/auto` is still modeled as non-reasoning and the OpenRouter docs still say the `auto` route skips reasoning injection. That does not strongly resolve issue #57430.

## Evidence

- **Generic fallback was added, but only for reasoning-capable models:** `CHANGELOG.md:163` and the release appcast both describe the shipped fix as raising the implicit default thinking level for reasoning-capable models. The matching regression test expects `medium` only when the selected model has reasoning metadata and no explicit thinking default is configured.
  - file: [CHANGELOG.md:163](https://github.com/openclaw/openclaw/blob/9b18ae877018a34933e27bb7b2986cb97e9602aa/CHANGELOG.md#L163)

- **`openrouter/auto` still advertises `reasoning: false` on main:** The bundled OpenRouter catalog still defines the default `auto` model with `reasoning: false`, so the new implicit-`medium` path does not naturally apply to the exact route reported in the issue.
