## Summary

Current main still does not satisfy #29387. Bootstrap injection is resolved from `workspaceDir` only, the standard bootstrap loader enumerates files under that workspace directory, and the embedded/CLI/compaction callsites do not pass or inspect `agentDir` for either merge/override behavior or a warning. Docs now steer multi-agent users toward per-agent workspaces, but that is guidance, not the requested runtime diagnostic or agentDir bootstrap support. Related PRs #29450 and #29545 were closed/unmerged in the provided context and their head commits are not present in this checkout.

## Best Possible Solution

Keep #29387 open as the canonical tracker. The best product path is either a focused diagnostic like #29450 that warns when recognized bootstrap filenames are present under an explicitly configured `agentDir` but not injected, or a deliberate agentDir merge/override contract like #29545 with caller coverage, allowlist behavior, hook precedence, regression tests across run paths, and docs explaining workspace versus agentDir semantics.
