## Summary

Keep open. Current main still does not expose an ACP token-only auth flag or pass `deviceIdentity: null` from `openclaw acp`; the linked PR remains a focused implementation path for the closed feature request #50671, with small cleanup still needed around docs, tests, and final flag naming.

## Best Possible Solution

Keep the PR open and either rebase/finish it or land an equivalent patch: add an ACP option with an agreed flag name, carry it through `AcpServerOptions`, standalone parser/help, Commander registration, docs, and startup tests, then pass `deviceIdentity: null` to `GatewayClient` only when the flag is set. The existing GatewayClient and gateway role policy already provide the needed token-only operator-auth contract.
