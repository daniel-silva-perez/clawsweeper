## Summary

Current `main` only partially covers the provider-context fix. The new chat `/model` path resolves the default provider from config/agent state, but the PR's claimed companion fixes are not present on `main`: `src/gateway/server-methods/models.ts`, `src/flows/model-picker.ts`, and `src/status/status-message.ts` still hardcode `DEFAULT_PROVIDER`. That is not strong enough evidence to close this PR as already implemented.
