## Summary

Keep this issue open. Current main addresses important parts of the Control UI live-rendering regression, including stale history reloads, optimistic user/assistant tail preservation, reading-indicator visibility, and active-memory timeout containment. But the original report is broader, and current main still lacks the reporter’s concrete textarea/focused-IME draft preservation path for swallowed or delayed input text.

## Best Possible Solution

Keep this issue open or split it deliberately: keep the shipped Control UI stale-reload and active-memory fixes, link #70702/#71878 as the solved assistant-reply-until-refresh subset, and add a focused Control UI textarea/IME fix in `ui/src/ui/views/chat.ts` with coverage for focused/composing draft preservation. If maintainers decide the explicit accepted/processing UX model should be separate, create that narrower follow-up before closing the umbrella.
