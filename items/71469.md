## Summary

Keep PR #71469 open. Current main still has the reported Gemini CLI OAuth local-browser deadlock surface: the auth URL is only printed from the `openUrl(authUrl)` catch block, while the actual OpenClaw `openUrl` helper resolves `false` instead of throwing for unsupported/failed opens and the auth call sites discard that boolean. The PR’s proposed log-before-open change and regression test are not present on main, and a maintainer comment already reviewed the PR as valid and worth landing.

## Best Possible Solution

Land PR #71469 or an equivalent Google plugin fix that logs the Gemini OAuth authorization URL before attempting best-effort browser launch, keeps the local callback flow unchanged, and adds a regression test asserting log-before-open ordering for the local login path.
