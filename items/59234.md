## Summary

Keep open. The report is recent, has several independent confirmations across ARM Linux/Raspberry Pi/WSL, and the provided discussion points to an upstream QMD fix that was merged but not released at the time of the comments. I did not find strong enough evidence from current documentation/release context to safely close this as implemented on main.

## Evidence

- **Multiple independent reproductions:** Issue comments report the same QMD/searchMode=search behavior on Raspberry Pi 4B, Raspberry Pi 5 aarch64, Oracle ARM Linux, and WSL through April 21, 2026.
- **Reporter follow-up says not fixed in later OpenClaw releases:** The reporter stated that as of v2026.4.14 they still needed a local qmd-wrapper workaround and saw no direct release-note fix; they also noted upstream tobi/qmd#530 was merged but not yet released at that time.
- **Current public QMD docs still describe QMD boot maintenance separately from searchMode:** The public QMD docs describe OpenClaw creating collections and running qmd update + qmd embed on boot/periodically, while searchMode is described as controlling searches. That is not strong evidence that the requested vector-probe/searchMode mitigation exists.
