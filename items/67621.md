## Summary

Keep open. Current main has a partial mitigation for the exact `security=full` plus `ask=off` YOLO configuration, where script preflight is skipped, but the underlying scanner still applies the `$[A-Z_]...` regex to raw Python/Node source text without string-literal context whenever script preflight runs. There is also an open implementation PR, #67924, with closing syntax for this issue, so the maintainer-safe path is to review/land or explicitly close that PR rather than closing this issue first.

## Best Possible Solution

Keep this issue open while #67924 is reviewed, updated against current main if needed, and either landed or deliberately closed. The likely maintainer fix belongs in `src/agents/bash-tools.exec.ts`: make source-file preflight ignore `$VAR` matches inside Python string/comment contexts while preserving detection for actual shell-variable misuse, and add focused regression coverage in `src/agents/bash-tools.exec.script-preflight.test.ts` for `$SIM` in docstrings/help text plus a true injection outside strings.
