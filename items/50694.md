## Summary

Keep open. Current `main` still has the negative/zero threshold behavior this PR is trying to fix, so there is not strong evidence to close it as already implemented or obsolete.

## Evidence

- **Flush gate still rejects non-positive thresholds:** `resolveMemoryFlushGateState()` still computes `threshold = Math.max(0, contextWindow - reserveTokens - softThreshold)` and immediately returns `null` when `threshold <= 0`. With `reserveTokensFloor >= contextWindowTokens`, that makes `shouldRunMemoryFlush()` return `false` instead of triggering compaction.
