## Summary

Keep open. Current main still computes memory-flush and preflight thresholds from the raw reserveTokensFloor, so reserveTokensFloor >= contextWindowTokens still suppresses the token-based flush/compaction gate. The related issue #50611 remains valid, and this PR’s latest headroom-based clamp is still useful rather than obsolete.

## Best Possible Solution

Keep #50694 open or consolidate it deliberately with the parallel #51316/#50611 work. The product fix should clamp reserveTokensFloor at the memory-flush/preflight gate against available headroom, keep diagnostics consistent, and add regression tests for reserveTokensFloor == contextWindowTokens, reserveTokensFloor > contextWindowTokens, and small context windows.
