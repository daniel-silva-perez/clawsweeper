## Summary

Keep this PR open. Current main still lacks the MiniMax `providerAuthAliases` manifest entries, while the auth resolver depends on manifest `providerAuthAliases` or deprecated auth-choice ids before it rewrites provider ids for auth lookup. The PR is a narrow implementation candidate for the still-open same-author issue, and the proposed manifest assertion matches existing BytePlus/Volcengine coverage patterns.

## Best Possible Solution

Keep this PR open and review/land it, or apply an equivalent fix that adds `providerAuthAliases` for `minimax-cn` and `minimax-portal-cn` in the MiniMax manifest plus a focused manifest regression test. That should close the paired MiniMax auth issue after merge.
