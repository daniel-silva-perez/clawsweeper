## Summary

Keep open. Current main still handles overloaded provider errors through auth-profile rotation, optional overloaded backoff before rotation/fallback, and then fallback/surface-error paths; it does not implement PR #49800's same-profile retry loop. The PR discussion also shows the author responded to review by extracting the retry helper, resetting retry budget before fallback, and adding behavioral coverage, so this remains useful work rather than conservative cleanup fodder.

## Best Possible Solution

Keep #49800 open as the focused tracker/implementation for same-auth-profile retry on transient overloaded errors. Best maintainer path is to refresh the PR against current `main`, preserve existing profile-rotation and model-fallback behavior, add/keep focused regression coverage for prompt and assistant overloaded paths, and address the abort-normalization review comment before merge.
