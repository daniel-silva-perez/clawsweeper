## Summary

Keep #70986 open. Current main still has the reported memory-wiki lint behavior: wikilinks are extracted from raw markdown without skipping frontmatter, fenced code, or inline code, and lint validates targets only by relative path and basename, not frontmatter title. The related #64696 remains open and broader/narrower rather than a shipped superseding fix.

## Best Possible Solution

Keep the issue open and fix it in `extensions/memory-wiki`: make `extractWikiLinks` markdown-aware enough to skip frontmatter, fenced code, inline code, and comments before applying wikilink extraction; change lint target resolution to use a shared normalized lookup that includes path, basename, page title, slugified title, ids where appropriate, and relative Markdown targets; add focused tests for fenced/inline false positives, frontmatter-title links, Unicode/special-character titles, and relative paths if that scope is included.
