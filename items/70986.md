## Summary

Keep this issue open. Current main still matches both reported memory-wiki lint defects: wikilink extraction scans raw markdown with regexes, including fenced imported source content, and lint’s broken-link resolver only accepts path-derived targets rather than frontmatter titles. The related #64696 remains open and narrower, so it does not supersede this broader extractor/resolver bug.

## Best Possible Solution

Keep this issue open and fix it in extensions/memory-wiki. Make wikilink extraction markdown-aware by excluding frontmatter, fenced code, inline code, and comments before regex extraction. Replace lint’s path/basename-only target set with a normalized resolver that includes relative path, basename, title, slugified title, id, and future aliases. Add regression tests for fenced and inline [[...]] literals, imported source fences, natural Unicode or special-character frontmatter-title links, and relative Markdown links if included in scope. Coordinate with #64696, but do not treat that narrower open issue as a substitute for this broader fix.
