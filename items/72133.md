## Summary

Keep #72133 open. Current main has strong Control UI support and a partial channel-path workaround via `/usage`, but the native mobile request is not implemented: Android drops usage/cost metadata while decoding chat history, Android renders only message content, and the shared iOS chat bubble does not render decoded usage metadata. Channel replies can append a token/cache/cost line when `responseUsage` is enabled, but that footer does not include the requested context percentage/model parity from the Control UI.

## Best Possible Solution

Keep the issue open. Implement native mobile parity by carrying usage, top-level cost, model, and context-window data through Android and iOS chat models, then render a subtle assistant-message footer using the Control UI thresholds. For channels, extend the existing `responseUsage`/`/usage` footer path if maintainers want true parity, adding context percentage and model while keeping it opt-in.
