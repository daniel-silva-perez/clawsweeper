## Summary

Keep this PR open. Current main has not applied the proposed one-line fix: the bundled Google Gemini CLI backend still maps the `flash` CLI alias to `gemini-3.1-flash-preview`, while its default live-test model is already `google-gemini-cli/gemini-3-flash-preview`. The CLI runner applies `modelAliases` directly before passing `--model`, so the stale alias remains a real backend path rather than being fully covered by the broader Google provider normalization. The existing review discussion about verifying `pro` and `flash-lite` is relevant, but it is a merge-review concern, not a reason to close the PR.

## Best Possible Solution

Keep the PR open, review it as a focused bug fix, add a small regression test around Google Gemini CLI alias resolution, and verify the remaining Gemini preview aliases before merging or requesting a tiny follow-up change.
