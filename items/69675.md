## Summary

Keep #69675 open. Current main has not implemented or documented the PR's parent-interpreted ACP follow-up semantics; shipped docs and bound-session tests still describe normal follow-ups as routed directly to the ACP harness. The Greptile review feedback appears addressed in the current PR diff, so the blocker is product/runtime alignment rather than stale docs hygiene.

## Best Possible Solution

Keep the PR open for maintainer judgment and product alignment. The best path is either to implement parent-orchestrated rewriting in session continuation/bound ACP routing and then document it, or narrow the PR to document current direct-routing behavior. If the proposed semantics are intended as future design guidance, move them to an explicit design/follow-up issue instead of presenting them as shipped behavior.
