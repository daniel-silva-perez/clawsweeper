## Summary

Keep PR #49958 open. Current main still has the no-configured-authorizer behavior that the PR is trying to change, and the linked Feishu bug #49915 is still represented by current tests/code. However, the PR as proposed is not ready to close or land because review discussion identified real scope and SDK-parity issues that still need a narrower implementation.

## Best Possible Solution

Keep the item open and ask for a revised patch or replacement PR that scopes the no-allowFrom fallback to the intended group command-gating path, preserves the existing DM command-authorization contracts, updates any mirrored SDK/root-alias implementation if shared semantics change, and adds focused regression coverage for Feishu group slash commands with no allowFrom plus DM non-regression coverage.
