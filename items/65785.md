## Summary

Keep open. Current main still does not let MiniMax web_search resolve credentials from MINIMAX_OAUTH_TOKEN, while the linked issue and PR target that exact remaining behavior gap.

## Best Possible Solution

Keep the PR open and adapt it to current main: add MINIMAX_OAUTH_TOKEN to the MiniMax web-search runtime lookup and missing-credential message, mirror it into provider/contract envVars and setup/autodetection expectations, add a regression test for OAuth-token fallback, and update the MiniMax Search docs. Consider including MINIMAX_API_KEY in envVars at the same time because runtime already accepts it.
