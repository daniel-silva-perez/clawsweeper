## Summary

Keep #71178 open. Current main still lacks the narrow `update.run` continuation path called out by the maintainer: the update sentinel is written without `continuation`, the protocol schema does not accept `continuationMessage`, the agent gateway tool does not forward it for `update.run`, and tests still assert no continuation is present. The provided discussion also includes explicit maintainer guidance that this specific report should stay open, with #71573 as the targeted but unmerged fix.

## Best Possible Solution

Keep #71178 open. Land or otherwise implement the #71573-style fix: add `continuationMessage` to `UpdateRunParamsSchema` and restart request parsing, forward it from the agent gateway tool's `update.run` branch, build a `buildRestartSuccessContinuation(...)` payload in `src/gateway/server-methods/update.ts` only for successful session-scoped updates, and update tests that currently assert `continuation` is undefined. Then decide whether graceful-drain/fallback notices and system-prompt reset bridge behavior stay here or move to narrower follow-ups.
