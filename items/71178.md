## Summary

Keep open. Current main still has the `update.run`-specific continuation gap identified in the maintainer discussion: restart/config continuation delivery exists, but `update.run` does not build or persist a continuation, the protocol schema lacks `continuationMessage`, the agent gateway tool does not forward it for `update.run`, and tests still assert the continuation is absent. PR #71573 is related but is open in the provided context, and the later discussion notes remaining graceful-drain and post-reset bridge questions.

## Best Possible Solution

Keep #71178 open. The best path is to land or review PR #71573, or an equivalent patch, so `update.run` uses the same restart-continuation contract as `restart` and config writes. After that, a maintainer should explicitly decide whether the remaining graceful-drain/fallback and post-reset bridge gaps stay in this issue or move to narrower follow-ups.
