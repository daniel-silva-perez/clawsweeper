## Summary

Keep open. Current main does not implement the PR's smartExtraction path for memory-lancedb, and the PR is still a plausible bundled-plugin feature rather than obsolete cleanup. It is not merge-ready as-is because review comments identify unresolved config-schema and capture-safety problems that need author or maintainer follow-up.

## Best Possible Solution

Keep the PR open for focused rework. The best path is to update the memory-lancedb manifest and UI hints for smartExtraction, preserve shouldCapture/captureMaxChars before any LLM call, treat existing memories as untrusted prompt data, make extraction failures observable or fall back safely, and add targeted config/parser/hook tests plus docs for the new opt-in behavior.
