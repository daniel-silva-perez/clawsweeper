## Summary

Keep open. Current main still does not implement smartExtraction for memory-lancedb, and the PR remains a plausible bundled-plugin feature rather than obsolete cleanup. It is not merge-ready as provided because review discussion identifies unresolved manifest/config, capture-safety, prompt-injection, and observability issues that need author or maintainer follow-up.

## Best Possible Solution

Keep this PR open for focused rework. The best path is to update the memory-lancedb manifest and UI hints for smartExtraction, preserve shouldCapture/captureMaxChars before any LLM call, treat existing memories as untrusted prompt data, make extraction failures observable or fall back safely, and add targeted config/parser/hook tests plus docs for the new opt-in behavior.

## Risks / Open Questions

- Closing now would discard an active, scoped memory-lancedb feature that is not implemented on current main and has no canonical replacement item in the provided related context.
- Merging the PR as-is would risk invalid user config because the manifest schema is not updated for smartExtraction.
