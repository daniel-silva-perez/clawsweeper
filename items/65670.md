## Summary

Keep this PR open. Current main still preserves only the selected Node executable's own bin directory in daemon service PATH planning, while QMD defaults to resolving `qmd` from the gateway PATH and the documented service workaround confirms PATH visibility is the relevant failure mode. The PR discussion contains a concrete macOS launchd reproduction and `spawn qmd ENOENT` log, and the bot review described the patch as a narrow implementation candidate rather than obsolete work.

## Best Possible Solution

Review, rebase if needed, and land this PR or an equivalent narrow fix in `src/commands/daemon-install-plan.shared.ts`, with regression coverage in `src/commands/daemon-install-plan.shared.test.ts`. The implementation should derive only the matching `.vite-plus/bin` from a selected `.vite-plus/js_runtime/node/.../bin/node` path and should not broaden daemon PATH from the user's shell environment.
