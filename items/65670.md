## Summary

Keep this PR open. Current main still preserves only the selected Node executable's own bin directory in daemon service PATH planning, while QMD defaults to resolving `qmd` from the gateway process PATH. The PR remains a focused implementation candidate for adding the matching `.vite-plus/bin` companion directory with regression coverage.

## Best Possible Solution

Review, rebase if needed, and land this PR or an equivalent narrow fix in `src/commands/daemon-install-plan.shared.ts`, with regression coverage in `src/commands/daemon-install-plan.shared.test.ts`. The implementation should derive only the matching `.vite-plus/bin` from a selected `.vite-plus/js_runtime/node/.../bin/node` runtime and should not import arbitrary shell PATH entries.
