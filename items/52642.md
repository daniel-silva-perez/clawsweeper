## Summary

Keep open. Current main still has both gateway reasoning gaps described by PR #52642: `sessions.patch` can persist `reasoningLevel`, but the gateway `agent` run path does not forward it into `runEmbeddedPiAgent`, and live thinking emission is still gated on a channel `onReasoningStream` callback that WebSocket chat dispatch does not provide. The PR is not merge-ready as-is because its discussion includes unresolved regression findings and unrelated session-memory/reset changes, but it contains a real bug-fix direction that is not fully covered by the narrower related PRs.

## Best Possible Solution

Keep this open or supersede it only after creating/chosen a canonical item that covers both parts: forward the stored session `reasoningLevel` into gateway `agent` embedded runs, decouple WebSocket thinking event emission from the optional channel callback, preserve existing Telegram/Feishu callback behavior, and add focused regression tests for `agent.run`/`chat.send` with `reasoningLevel: "stream"` and no `onReasoningStream`. The submitted PR should be narrowed before merge.
