## Summary

Keep open. Current main fixes the `agents list` unresolved-SecretRef crash and several plugin-preload hang surfaces, but the whole report is not fully implemented: `agents add` still inherits broad `agents` plugin preloading, non-JSON `agents list` still loads plugins for provider enrichment, and CLI finalization only has memory-runtime cleanup rather than a generic external-plugin handle teardown path. The related reports cover important sub-parts, but they do not fully obsolete the reported native-tools-bridge-style process hang across the named `agents` subcommands.

## Best Possible Solution

Keep the issue open but narrow it to the remaining CLI exit-hang surface. The best fix is either to keep remaining config-only `agents` paths, especially non-interactive `agents add` and non-JSON `agents list` where possible, off broad runtime plugin loading, or to add a real short-lived CLI plugin-runtime teardown contract that closes external plugin handles. Treat the SecretRef crash and the already-covered bind/delete/list-json pieces as resolved sub-parts, not as a complete close reason.
