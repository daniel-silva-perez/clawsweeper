## Summary

Keep this PR open. Current main still does not implement an equivalent root/effective-root guard in either CLI entrypoint, and the linked #67478 failure mode remains plausible through state-dir resolution and Linux user-systemd repair paths. The PR is the active implementation candidate and has incorporated review feedback for static imports, container forwarding, legacy entry coverage, narrowed bypass semantics, and effective UID handling.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or an equivalent core fix that guards both src/entry.ts and src/index.ts before state/config side effects, checks both real and effective UID, preserves intentional container forwarding, includes focused regression tests, and updates the DigitalOcean/install guidance or wrapper flow so root SSH users get a clear supported command path.
