## Summary

Current `main` still shows the default Feishu card header as `main`, so this PR is not implemented on `main` and should remain open.

## Evidence

- **Current header fallback still uses raw agentId:** `resolveCardHeader` still does `const name = identity?.name?.trim() || agentId`, so the default agent with no identity resolves to `main`. See `extensions/feishu/src/reply-dispatcher.ts:50-54`.
  - file: [extensions/feishu/src/reply-dispatcher.ts:50](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/extensions/feishu/src/reply-dispatcher.ts#L50)

  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **Streaming cards always receive that header:** The streaming path still passes `header: cardHeader` unconditionally into `streaming.start(...)`, so the `main` title is still sent for default-agent streaming cards. See `extensions/feishu/src/reply-dispatcher.ts:285-292`.
  - file: [extensions/feishu/src/reply-dispatcher.ts:285](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/extensions/feishu/src/reply-dispatcher.ts#L285)

- **Static card replies also still receive that header:** The non-streaming structured-card path also passes `header: cardHeader` unconditionally, so static cards still show `main`. See `extensions/feishu/src/reply-dispatcher.ts:428-445`.
  - file: [extensions/feishu/src/reply-dispatcher.ts:428](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/extensions/feishu/src/reply-dispatcher.ts#L428)
