## Summary

This PR should stay open for maintainer judgment, not auto-closed by conservative cleanup. The ClawHub/community-plugin path is strongly supported by current policy and plugin docs, but the PR was reopened twice after prior ClawHub closures by Sliverp, who is credited in-repo for Chinese channels. As-is, the PR also has unresolved catalog/security issues: it promotes a floating external npm package without integrity metadata and uses the package name as the channel id instead of the user-facing `yuanbao` id described in the PR.

## Best Possible Solution

Have maintainers make an explicit product decision. If Yuanbao remains community-owned, keep it as `openclaw-plugin-yuanbao` on ClawHub/npm and close manually as plugin/community work. If OpenClaw wants official catalog promotion, replace this with a focused curated-catalog PR using channel id `yuanbao`, docsPath `/channels/yuanbao` only if the dedicated page is accepted, an exact pinned `npmSpec`, `expectedIntegrity`, provenance/security review, and explicit owner approval. Add core SDK work only if the external plugin proves a missing public channel API.
