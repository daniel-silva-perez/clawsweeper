## Summary

Keep PR #51389 open. Current main partially addresses the original HTTP-400 pain by continuing fallback on unrecognized errors when more candidates remain, but it does not implement the requested configurable `fallbackOnErrors` policy. The public model config type, strict zod schema, config resolver, runtime fallback API, docs, and tests still have no `fallbackOnErrors` surface, and the PR discussion identifies remaining end-to-end gaps that would need maintainer review rather than cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer/product review or replace it with a narrower implementation. If OpenClaw wants this public contract, implement it end to end: add the type and strict schema/generated metadata/docs, resolve defaults and per-agent overrides, thread the policy through all text fallback callers, decide whether media/PDF/image configs support it or deliberately exclude it, preserve it in model-scan rewrites, and add tests for `default`, `all`, custom allowlists, and fail-fast disallowed statuses.
