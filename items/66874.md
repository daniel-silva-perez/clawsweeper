## Summary

Keep this open. Current main still supports Claude CLI as a bundled Anthropic CLI backend, but the official Docker image does not pre-install `@anthropic-ai/claude-code`, the release workflow does not add a Claude CLI build path, and the user-facing Docker docs still lack a Docker-specific persistent `claude` binary plus `~/.claude` credential persistence recipe.

## Best Possible Solution

Keep this open until maintainers choose the Docker product path. A conservative fix would add a Docker-specific Claude CLI section under `docs/install/docker.md` and cross-link it from `docs/providers/anthropic.md` or `docs/gateway/cli-backends.md`, with a persistent npm prefix or `/home/node` volume, `~/.claude` persistence, an explicit `agents.defaults.cliBackends.claude-cli.command` example, and API-key fallback guidance. If maintainers prefer the happy path, add a pinned `@anthropic-ai/claude-code` install to the official Docker image and update Docker release/test/docs coverage accordingly.
