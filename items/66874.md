## Summary

Keep this issue open. Current main still supports Claude CLI as a first-class backend, but the official Docker image path has not caught up: the Dockerfile and Docker release workflow do not install `@anthropic-ai/claude-code`, and the user-facing Docker docs do not give a Claude-specific persistent install plus `~/.claude` persistence recipe.

## Best Possible Solution

Keep this open until maintainers choose and implement the Docker product path. A conservative fix would add a Docker-specific Claude CLI section under `docs/install/docker.md` and cross-link it from `docs/providers/anthropic.md` or `docs/gateway/cli-backends.md`, showing a persistent npm prefix or `/home/node` volume, `~/.claude` persistence, `agents.defaults.cliBackends.claude-cli.command`, and API-key fallback guidance. If maintainers prefer the happy path, add a pinned `@anthropic-ai/claude-code` install to the official Docker image and update the Docker release/test/docs surface accordingly.
