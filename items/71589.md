## Summary

Keep #71589 open. Current main still suppresses embedded assistant-visible output only when the assistant phase resolves to `commentary`; it does not suppress unphased assistant messages ending in `stopReason: "toolUse"` or containing tool-call blocks. The PR is an active, focused fix for linked issue #71588, and the provided discussion shows the author responded to the bot review gaps for `text_end` coverage and snake_case tool-call variants.

## Best Possible Solution

Keep #71589 open for maintainer review. The best path is to land or refine the narrow embedded-agent suppression so unphased tool-use assistant preludes do not become user-visible replies, already-buffered suppressed text is cleared, explicit `final_answer` output still streams/delivers, and regression coverage covers both `message_end` and `text_end` block-reply modes. Any unrelated qa-lab/status/lockfile cleanup should be split or clearly justified before merge.
