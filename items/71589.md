## Summary

Keep #71589 open. Current main still suppresses only assistant messages resolved as `commentary`; it does not suppress unphased assistant messages that end in `stopReason: "toolUse"` or contain tool-call blocks, which is the behavior this active PR is trying to fix for #71588.

## Best Possible Solution

Keep #71589 open for maintainer review. The best path is to land or refine the narrow embedded-reply suppression for unphased tool-use assistant continuations, preserve explicit `final_answer` streaming, clear already-buffered suppressed commentary/tool-use text, and keep regression coverage for both `message_end` and `text_end` block reply modes. Review the extra qa/status/lockfile changes for scope before merge.
