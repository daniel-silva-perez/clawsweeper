## Summary

Keep this issue open. Current main still returns the updated `skills.entries.<skillKey>` object directly from `skills.update`, so raw `apiKey` and `env` values can still be echoed in the successful gateway response. There is also an open unmerged PR, #69998, whose body uses closing syntax for this issue; per maintainer policy, the issue should remain open until that PR is reviewed and merged or closed.

## Best Possible Solution

Review #69998 as the implementation candidate, or apply the same narrow fix directly: import/use `redactConfigObject()` in `src/gateway/server-methods/skills.ts` for the config-mode `skills.update` response, and add a regression test that verifies `apiKey` and secret-like `env` values are redacted in the response while full values are still written to config.
