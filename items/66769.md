## Summary

Keep this issue open. Current main still returns the updated `skills.entries.<skillKey>` object directly from `skills.update`, so plaintext `apiKey` and secret-like `env` values can still be echoed in the successful gateway response. There is also an open unmerged implementation candidate, #69998, whose body uses closing syntax for this issue; per maintainer policy, the issue should remain open until that PR is reviewed and merged or closed.

## Best Possible Solution

Review #69998 as the implementation candidate, or apply the same narrow fix directly in `src/gateway/server-methods/skills.ts`: import/use `redactConfigObject()` for the config-mode `skills.update` response and add a regression test proving `apiKey` and secret-like `env` values are redacted in the response while full values are still written to config.
