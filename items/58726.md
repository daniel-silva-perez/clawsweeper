## Summary

Keep open. The issue is reporter-authored, has no protected labels, and is a plausible runtime safety request. Current public docs/source evidence show configurable loop detection exists, but it is disabled by default, uses 10/20/30 thresholds rather than the requested default 5/max 10, and the documented same-tool/same-args detector is warning-oriented rather than a hard agent abort after repeated identical calls.
