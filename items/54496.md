## Summary

Current main still lacks a silent reasoning persistence mode. OpenClaw can enable model thinking and can hide or stream reasoning in user-facing delivery, but durable transcript storage and replay surfaces do not expose a separate policy to discard thinking blocks while preserving reasoning capability.

## Best Possible Solution

Keep this issue open and implement a core transcript-persistence policy separate from thinking production and reasoning visibility. A maintainer-quality path would keep `thinkingLevel` for model reasoning effort, keep `reasoningLevel` for user/channel visibility, add a documented discard/persist mode for thinking blocks, strip `type: "thinking"` and `redacted_thinking` before durable transcript reads/writes or move replay-required opaque provider state outside user/history/memory transcript APIs, and add coverage for `sessions.get`, ACP `loadSession`, compaction/replay, memory/session indexing, plugin hooks, and provider replay compatibility.

## Risks / Open Questions

- A fix must preserve provider replay compatibility for signed or opaque thinking blocks, especially Anthropic, Bedrock, Google, OpenAI Responses, and Codex-style continuations.
- A plugin `before_message_write` workaround can mutate transcript writes, but treating this as only plugin-owned would leave core config, UI/session controls, provider replay policy, docs, and transcript API semantics unresolved.
- Changing durable transcript persistence affects `sessions.get`, ACP `loadSession`, compaction/replay, session-history tools, memory indexing, transcript events, and plugin hook expectations.
