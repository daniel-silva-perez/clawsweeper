## Summary

Keep open. Current `main` has reasoning visibility only as `on|off|stream`, and that setting controls reply display, not transcript persistence. The codebase and docs still preserve/store thinking blocks in session history, so the requested "silent reasoning" mode is not implemented on `main`.
