## Summary

Keep open. Current `main` no longer looks like a pure silent-exit path, but it still does not surface a structured session-lane busy/timeout error for `openclaw agent`. The queue only emits an internal warning, and the CLI still treats gateway failures as generic fallback-to-embedded behavior rather than exposing lane diagnostics to the caller.

## Evidence
