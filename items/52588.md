## Summary

Keep PR #52588 open. Current main still does not implement `OPENCLAW_TUI_USER_BG` or `OPENCLAW_TUI_USER_TEXT`, and the shipped docs/tests only cover light/dark palette selection via `OPENCLAW_THEME` and `COLORFGBG`. The PR remains a plausible focused TUI feature, but its current design needs review/fix for contrast-safe partial overrides before merge.

## Best Possible Solution

Keep the PR open and ask for a focused rebase/fix. The best path is to implement the env override in `src/tui/theme/theme.ts` with strict `#RRGGBB` validation, make background-only overrides choose or require a readable `userText`, add regression tests for valid, invalid, and low-contrast/partial override cases, and document the vars in `docs/help/environment.md` and `docs/web/tui.md`.
