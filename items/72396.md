## Summary

Current main still treats OPENCLAW_PLUGIN_STAGE_DIR as one writable plugin runtime-dependency stage root. The requested layered read-only baseline roots plus final writable install root is not implemented, and the issue is a concrete core plugin runtime-deps enhancement rather than a ClawHub-only item or a duplicate of the closed EACCES/doctor-repair work.

## Best Possible Solution

Keep this issue open and design it as a core bundled plugin runtime-dependency staging enhancement: parse a documented stage-root list, resolve dependency sentinels left-to-right across immutable baseline roots plus the final writable root, install and repair only into the final writable root, register aliases for the root that satisfies each dependency, keep cache keys deterministic across OpenClaw version and runtime-deps manifests, and add Docker/systemd/offline docs plus focused tests. Treat closed #71730 and #72263 as related single-root repair history, not canonical trackers for layered stage directories.

## Risks / Open Questions

- Path-list semantics need careful POSIX and Windows handling because path.delimiter differs and colon is currently accepted as part of the unparsed explicit stage value on POSIX.
- A correct implementation needs coordinated changes to sentinel lookup, install locks, retained manifests, loader alias registration, runtime mirroring, doctor repair, and health reporting; changing only env parsing would create partial behavior.
