## Summary

Current main still treats OPENCLAW_PLUGIN_STAGE_DIR as a single writable bundled plugin runtime-dependency stage root. It does not implement layered read-only baseline roots plus a final writable install root, and the comments describe a concrete container/proxy-restricted deployment need that belongs in core runtime-deps handling rather than ClawHub.

## Best Possible Solution

Keep this issue open and design it as a core bundled plugin runtime-dependency staging enhancement: parse a documented stage-root list, resolve dependency sentinels left-to-right across immutable baseline roots plus the final writable root, install and repair only into the writable root, register aliases for the root that satisfies each dependency, preserve deterministic cache keys across OpenClaw version and plugin runtime-deps manifests, and add Docker/systemd/offline deployment docs and focused tests. Treat closed #71730 and #72263 as related single-root repair history, not canonical trackers for layered stage directories.

## Risks / Open Questions

- A partial env-parser-only change would be unsafe; sentinel lookup, install locks, retained manifests, loader alias registration, runtime mirroring, doctor repair, startup health, and docs need to move together.
