## Summary

Keep #70229 open. Current main still unconditionally adds Docker `--security-opt no-new-privileges` and does not expose `sandbox.docker.noNewPrivileges` in config types, schema, resolution, docs, or dangerous-config reporting. The linked bug #43996 has a concrete Docker repro and the PR is a focused attempt to add the requested compatibility escape hatch while preserving the secure default.

## Best Possible Solution

Keep #70229 open for maintainer review or landing after normal validation. The best product path is a narrow core config escape hatch with default `true`, schema/types/docs coverage, agent/shared-scope precedence tests, Docker args coverage, and dangerous-config reporting for `noNewPrivileges=false`, which matches the PR's stated direction and #43996's compatibility problem.
