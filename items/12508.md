## Summary

Keep open. Current `main` has some hook-safety controls, but it does not implement the requested hook-chain integrity protection: no per-hop cryptographic checksum/verification, no tamper-detection path between hook handlers, and no audit log of message modifications.

## Evidence

- **Hook runner still trusts sequential mutation results:** `createHookRunner()` executes modifying hooks sequentially, merges each handler result into the accumulated value, and only emits debug/error logs about execution/failures. There is no checksum, signature, or tamper-verification step around the data passed between hooks.
  - file: [src/plugins/hooks.ts:320](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/src/plugins/hooks.ts#L320)
  - command: `nl -ba src/plugins/hooks.ts | sed -n '276,390p'`
  - sha: [3361593442e2](https://github.com/openclaw/openclaw/commit/3361593442e295e9bb1f55cbfb38861ea9413e45)
- **Current enforcement is limited to two coarse hook policy gates:** `registerTypedHook()` only enforces `allowPromptInjection` and `allowConversationAccess` policy checks before registering typed hooks. It does not enforce a manifest-declared per-hook allowlist or any hook-chain integrity contract.
  - file: [src/plugins/registry.ts:1176](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/src/plugins/registry.ts#L1176)
