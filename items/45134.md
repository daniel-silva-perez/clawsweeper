## Summary

Keep open. Current `main` still has the two unfixed paths from the report: `text_end` block replies are emitted immediately before `message_end` reveals `stopReason`, and the same streamed chunks are appended into cumulative `assistantTexts` that `buildEmbeddedRunPayloads()` later turns into the final payload.

## Evidence

- **Immediate text_end block delivery still happens:** `emitBlockChunk()` still calls `pushAssistantText(chunk)` and then immediately emits `onBlockReply` payloads. There is no buffering keyed on `stopReason`, so pre-tool text can be sent before `message_end` determines whether the turn ended with `toolUse`.
  - file: [src/agents/pi-embedded-subscribe.ts:616](https://github.com/openclaw/openclaw/blob/40be5ad58187fd20b8defe5fc30e649190c131a8/src/agents/pi-embedded-subscribe.ts#L616)
  - command: `nl -ba src/agents/pi-embedded-subscribe.ts | sed -n '616,706p'`
  - sha: [40be5ad58187](https://github.com/openclaw/openclaw/commit/40be5ad58187fd20b8defe5fc30e649190c131a8)
- **No toolUse discard/rollback at message_end:** `handleMessageEnd()` finalizes `assistantTexts` and may emit block replies, but there is no branch that discards buffered block replies or splices `assistantTexts` back when `assistantMessage.stopReason === "toolUse"`.
  - file: [src/agents/pi-embedded-subscribe.handlers.messages.ts:708](https://github.com/openclaw/openclaw/blob/40be5ad58187fd20b8defe5fc30e649190c131a8/src/agents/pi-embedded-subscribe.handlers.messages.ts#L708)

- **State still lacks pending block-reply suppression fields:** `EmbeddedPiSubscribeState` on `main` has `assistantTextBaseline`, but it does not define any `pendingBlockReplies`/`suppressPreToolText`-style state that would be needed to hold and later discard pre-tool text for block-streaming channels.
