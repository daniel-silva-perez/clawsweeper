## Summary

Keep this issue open. Current main supports DREAMS.md and the Control UI Dream Diary, but a successful REM phase report still does not guarantee a DREAMS.md entry: REM phase report writing is separate from the narrative diary path, the diary path is conditional and best-effort, and the UI only reads DREAMS.md/dreams.md for the Diary surface.

## Best Possible Solution

Keep this issue open for maintainer triage. Decide whether a successful REM phase with usable material must always create or update DREAMS.md. If yes, make the scheduled REM path write a deterministic fallback diary entry when the subagent is unavailable, returns no text, or times out, and add a regression test for separate-storage REM report plus Dream Diary creation. If the diary is intentionally best-effort only, update the Dreaming docs and Control UI empty state to explain that memory/dreaming/rem/YYYY-MM-DD.md can exist without a diary entry.
