## Summary

Current main still does not guarantee `DREAMS.md` after a successful REM phase report. The REM report path writes `memory/dreaming/rem/YYYY-MM-DD.md` independently, while the diary path is conditional and best-effort; the Control UI diary surface only reads `DREAMS.md`/`dreams.md`, so the reporter's empty-tab outcome remains plausible. Related PRs in the provided context do not supersede this issue.

## Best Possible Solution

Keep this issue open for maintainer triage. Decide whether a successful REM report must always create or update `DREAMS.md`; if yes, make the scheduled REM path write a deterministic fallback diary entry when subagent narrative generation is unavailable, empty, or times out, and add a regression test for separate-storage REM report plus diary creation. If the current best-effort contract is intentional, update the Dreaming docs and Control UI empty state to explain that phase reports can exist without a Dream Diary entry.
