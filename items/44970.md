## Summary

Keep open. Current `main` has partially superseded the assistant-side part of PR #44970, but it does not fully implement the PR’s request: prompt-side fallback escalation still depends on the stale `fallbackConfigured` snapshot, and auth-profile exhaustion still throws raw errors when that flag is false. That means the PR’s main cleanup is not already on `main` well enough to close as implemented.
