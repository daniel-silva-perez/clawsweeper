## Summary

Keep PR #49793 open. Current main already covers Xiaomi image understanding through the `mimo-v2-omni` catalog entry and generic image-capable provider auto-registration, but it still has no Xiaomi media-understanding provider registration and no Xiaomi video understanding path. The PR is stale against current main and needs rework rather than closure.

## Best Possible Solution

Keep the PR open, but request a revised Xiaomi media-understanding implementation for current main: register a Xiaomi media provider, add manifest `contracts.mediaUnderstandingProviders` plus metadata/defaultModels for `mimo-v2-omni`, use the current OpenAI-compatible video and provider HTTP helper patterns, ensure video auto-selection does not reuse text-only `mimo-v2-flash`, and add focused tests for registration, request construction, response parsing, and model selection.
