## Summary

Keep #65418 open. Current main still applies `sanitizeTransportPayloadText()` to signed Anthropic `thinking` blocks immediately before sending Anthropic Messages payloads, while that sanitizer is explicitly defined and tested to remove unpaired UTF-16 surrogates. The earlier replay/history policy preserves signatures, but the final Anthropic transport conversion can still mutate the signed text, so the reported central bug is not implemented on main.

## Best Possible Solution

Keep the issue open and fix the Anthropic transport converter so signed `thinking` content is passed through unchanged when `thinkingSignature` is present, while retaining `sanitizeTransportPayloadText()` for unsigned text fallback, user text, and normal text blocks. Add a focused regression test around `convertAnthropicMessages` or the Anthropic stream request payload that includes a signed thinking block with an unpaired surrogate and asserts the final payload preserves `thinking` byte-for-byte with its signature.
