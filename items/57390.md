## Summary

Keep open. Current `main` still ships `node-llama-cpp` as an optional peer dependency, the global npm updater still uses plain `npm i -g` without any logic to restore dropped optional peers after a successful update, and release checks explicitly enforce that packaging shape.

## Evidence

- **`node-llama-cpp` still packaged as optional peer:** `package.json` on current `main` declares `node-llama-cpp` under `peerDependencies` and marks it optional via `peerDependenciesMeta`, not `optionalDependencies`. That matches the issue's reported failure mode for global npm updates.

- **Release guardrails require the unfixed packaging:** Release validation still fails unless `package.json` keeps `node-llama-cpp` as an optional peer dependency. That is strong evidence no packaging fix for this report has landed on `main`.
