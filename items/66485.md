## Summary

Keep PR #66485 open. Current main already implements the `/usage` context-percentage half more conservatively than the PR, but the chat-history refresh after compaction is not fully implemented on the current websocket/UI path, and the related merged PR #71297 explicitly left #66485 out of scope.

## Best Possible Solution

Keep the PR open for a narrowed, rebased fix. Preserve current main’s `/usage` implementation, then add a tested active-session-scoped chat-history refresh for successful compaction, preferably through the existing transcript/session event architecture or a `sessions.changed reason=compact` UI reload path rather than an unscoped sessionless agent-event reload. Add app-gateway coverage for successful compaction, unrelated sessions, and active-run safety.
