## Summary

Keep open. The report is recent and concrete, and current main still routes the macOS companion chat through the native SwiftUI WebChat surface with a ScrollView/LazyVStack plus scrollPosition-driven auto-scroll. I found no source, test, docs, changelog, or post-v2026.4.24 diff evidence that scrollbar-thumb dragging has been fixed or covered. The related #42527 closure addressed an older/general scrolling crash, but this issue is newer and targets the rewritten SwiftUI surface itself.

## Best Possible Solution

Keep the issue open for macOS maintainer reproduction on current main/latest app. If it reproduces, capture a hang sample and harden the SwiftUI message scroll container, likely by guarding scrollPosition-driven auto-scroll during explicit scrollbar dragging or replacing the message list with a more controllable AppKit-backed scroll view. Add focused macOS regression coverage or documented manual verification using a long chat history and scrollbar-thumb dragging.
