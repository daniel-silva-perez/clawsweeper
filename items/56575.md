## Summary

Keep PR #56575 open. Current main has adjacent session reset, cache-TTL pruning, heartbeat keep-warm documentation, and context-engine maintenance, but it does not implement this PR’s explicit per-model `sessionCacheTtl` / `timeBasedContextCompact` compact-or-reset modes or a gateway session-cache maintenance sweep. Related #62475 remains open and treats #56575 as a build-on candidate, not a full superseding implementation. 

## Best Possible Solution

Keep this open for maintainer product judgment or replace it with a narrower canonical follow-up. The best path is to split the remaining work into current-main-shaped pieces: clarify whether proactive prompt-cache preservation belongs under prompt caching/context pruning or session maintenance; reuse existing `session.reset` idle semantics for stale-context reset where possible; add any missing persisted activity/cache timestamps only if the chosen scheduler needs them; and implement a tested gateway/background maintenance path that addresses the existing review comments before merge.
