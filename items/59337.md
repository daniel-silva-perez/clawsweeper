## Summary

Keep open. This is a fresh enhancement request for direct CLI shell execution on paired nodes; current docs still explicitly say node shell execution must use the exec tool with host=node and that openclaw nodes run is not the supported path. Related PR #60788 fixed agent exec host=node routing, but did not restore a direct nodes run-style CLI workflow.

## Evidence

- **Current nodes CLI docs block shell execution via nodes invoke/run:** The public nodes CLI reference says system.run and system.run.prepare are blocked for nodes invoke, and says to use the exec tool with host=node instead of openclaw nodes run. This directly matches the reporter's requested missing CLI workflow.
  - file: [docs/cli/nodes.md:149](https://github.com/openclaw/openclaw/blob/d6ef1fcf24eea3a5985d653a3baf400e32aa1b75/docs/cli/nodes.md#L149)
