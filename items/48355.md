## Summary

Keep open. Current `main` has partial overlap through `api.runtime.events.onAgentEvent`, but it does not implement this PR's proposed plugin API/hook contract: `OpenClawPluginApi` still has no top-level `onAgentEvent`, plugin hooks still do not include `thinking_start`/`thinking_end`, and `agent_end` still does not carry `tokenUsage` or `toolCallCount`.

## Risks / Open Questions

- There is existing overlap with `api.runtime.events.onAgentEvent`, so any future maintainer decision should reconcile that surface with the PR's proposed top-level `api.onAgentEvent` instead of treating the feature as entirely new.
- `AgentEventStream` already includes `"thinking"` in `src/infra/agent-events.ts`, so the remaining gap is specifically plugin-facing contract wiring and hook payload shape, not raw event vocabulary.
