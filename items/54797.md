## Summary

Keep #54797 open. Current main still renders the Control UI chat session dropdown as a native select with grouped options only; it has no clear-completed action, per-entry dismiss control, or completed-subagent separator. The Sessions tab now has bulk deletion via `sessions.delete`, and the gateway exposes subagent lifecycle state, but that is only a workaround and does not implement the dropdown-specific request.

## Best Possible Solution

Keep the issue open. Implement a status-aware chat session selector cleanup path: identify completed subagent rows using the session key plus `status`/`endedAt`/`subagentRunState`, expose either an adjacent `Clear completed` action or a custom menu with per-row dismiss and separators, call the existing `sessions.delete` RPC for eligible rows, refresh `sessions.list`, and add focused Control UI tests for completed subagent rows in the selector.
