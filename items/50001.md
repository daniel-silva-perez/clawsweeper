## Summary

Keep open. Current `main` still has the exact placeholder mismatch this PR fixes, so the change is not implemented on `main` and the reported export regression remains credible.

## Evidence

- **Current template still contains broken script placeholders:** `template.html` still has multi-line JS blocks (`{ { MARKED_JS; } }`, `{ { HIGHLIGHT_JS; } }`, `{ { JS; } }`) rather than literal `{{...}}` tokens, so the template text does not match the replacement keys used by the exporter.
  - file: [src/auto-reply/reply/export-html/template.html:62](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/src/auto-reply/reply/export-html/template.html#L62)
  - command: `nl -ba src/auto-reply/reply/export-html/template.html | sed -n '56,84p'`
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **Exporter still expects literal `{{...}}` tokens:** `generateHtml()` still calls `.replace("{{JS}}", ...)`, `.replace("{{MARKED_JS}}", ...)`, and `.replace("{{HIGHLIGHT_JS}}", ...)`, which will not match the current template contents.
  - file: [src/auto-reply/reply/commands-export-session.ts:91](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/src/auto-reply/reply/commands-export-session.ts#L91)
  - command: `nl -ba src/auto-reply/reply/commands-export-session.ts | sed -n '28,100p'`
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **Read-only reproduction on current main shows replacements fail:** A Node read-only check loaded the current `template.html`, applied the same replacement chain as production, and reported `remaining: ["MARKED_JS","HIGHLIGHT_JS","JS"]` with `containsAppJs: false`. That means the app JS placeholder is not injected on current `main`.

  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **Recurrence guard from the PR is not present on main:** `.prettierignore` currently contains only `docs/.generated/`; it does not ignore `src/auto-reply/reply/export-html/template.html`, so the preventive part of the PR is also absent.

The security test performs the same `.replace()` calls on the real template but does not assert that placeholder removal succeeded.

## Risks / Open Questions

- Closing this PR would leave a still-reproducible export-template mismatch on `main`, with strong evidence that exported HTML omits the intended JS payload.
- There is a coverage gap on `main`: current tests do not verify the real `template.html` placeholder text matches `generateHtml()` replacement keys, so the regression can persist unnoticed.
