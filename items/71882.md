## Summary

Keep #71882 open for maintainer review rather than auto-closing it. The underlying status false-negative is still present on current main and is tracked by canonical issue #56968, but this PR is a flawed implementation attempt rather than obsolete work: it adds a memory-lancedb-pro-specific core gate and a console-subsystem filter that does not silence raw console.log/info output. The linked plugin-side PR is useful context, but the OpenClaw-side fix should be generic and extension-agnostic.

## Best Possible Solution

Keep the PR open but require changes: replace the memory-lancedb-pro hard-code with a generic memory-plugin status capability/runtime path, query runtime-provided status before built-in sqlite/qmd store gates when the active plugin owns memory, and suppress or route all probe-time plugin stdout safely for JSON mode. Regression tests should cover a third-party memory slot with no built-in store/config plus raw console.log/info during status probing.
