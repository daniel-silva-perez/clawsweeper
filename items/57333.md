## Summary

Current `main` no longer shows a code path where fresh WebChat messages get rerouted to an active subagent automatically. WebChat defaults to the selected agent's main session, `chat.send` uses the UI's explicit `sessionKey`, and subagent/chat events update only `lastActiveSessionKey`, not the live session being used for new sends.

## Close Comment

I reviewed current `main` with Codex and this no longer matches today's WebChat session-routing code.

- WebChat now documents that it defaults to the selected agent's main session, and thread-bound subagent routing is Discord-only (`docs/channels/channel-routing.md:132`, `docs/tools/subagents.md:115`, `docs/platforms/mac/webchat.md:8`).
- Fresh WebChat sends go through `chat.send` with the UI's explicit `sessionKey`; there is no fallback that automatically targets an active subagent (`ui/src/ui/controllers/chat.ts:232`).
- Current session changes are explicit UI actions only: agent change switches to that agent's main session, and session selection switches to the chosen key (`ui/src/ui/app-render.ts:2290`, `ui/src/ui/app-render.ts:2296`).
- Incoming chat/subagent activity updates only `lastActiveSessionKey`; it does not overwrite the live `sessionKey` used for fresh messages (`ui/src/ui/app-gateway.ts:202`, `ui/src/ui/app-gateway.ts:440`).

If you still hit this on a current build, please open a new issue with the exact WebChat session shown in the switcher/URL and a fresh reproduction trace.
