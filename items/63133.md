## Summary

Keep open. Current main still skips memory collection on the lean `status --json` path but does not carry an explicit skipped/checked state, so downstream output cannot distinguish an intentionally skipped memory probe from an attempted-but-unavailable probe. The PR's approach remains directionally useful, especially after its later commit addressed the bot review by preserving `unavailable` for checked null results.

## Best Possible Solution

Keep the PR open for maintainer review or rebase it into a narrow patch: add an explicit checked/skipped memory collection state to `StatusScanResult`, propagate it through JSON/text status rendering, render `not checked` only for the fast-path skip, keep `unavailable` for attempted probe failures, and retain focused regression coverage for both cases.
