## Summary

Keep this PR open. Current main still skips memory inspection on the lean `status --json` fast path by returning `memory: null`, but the scan result, JSON payload, and human formatter have no `checked`/`skipped` state. The PR head remains available and its latest patch directly addresses the prior review concern by reserving `not checked` for the explicit skipped fast path while preserving `unavailable` for attempted probe failures.

## Best Possible Solution

Keep this PR open for maintainer review, rebase if needed, and CI. The best fix is to land this focused checked/skipped memory-collection state, or reimplement the same behavior on main: lean `status --json` without `--all` should report skipped memory collection as `not checked`, while attempted probe failures and unavailable memory runtimes should continue to render as `unavailable`.
