## Summary

Current main still accepts and resolves `memorySearch.outputDimensionality`, but the bundled OpenAI memory embedding path does not pass it as an OpenAI `dimensions` request field. #55507 is an open, unmerged implementation candidate with closing syntax for this issue, so this issue should remain open until that PR is updated/reviewed and either landed or closed.

## Best Possible Solution

Keep this issue open. Review and update #55507, or replace it with a current-main patch that threads `memorySearch.outputDimensionality` through `extensions/openai/embedding-provider.ts`, `packages/memory-host-sdk/src/host/embeddings-remote-provider.ts`, and `extensions/openai/memory-embedding-adapter.ts`, sends OpenAI `dimensions` for inline and batch requests where supported, includes dimensions in cache-key data, and adds regression coverage for request bodies and cache-key/reindex behavior.
