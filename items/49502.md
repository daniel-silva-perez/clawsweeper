## Summary

Keep open. Current main still does not include usage/cost metadata on `agent.wait` terminal responses: the terminal snapshot type, dedupe parsing, cached wait response, and race-resolved wait response all omit `meta`. The PR targets a real gateway protocol gap from #49494/Paperclip, but the review thread raised correctness issues that should be fixed before merge rather than closing the PR as obsolete.

## Best Possible Solution

Keep the PR open. The best path is a refreshed implementation that adds optional `meta` to `AgentWaitTerminalSnapshot` and both `agent.wait` response paths, computes usage/cost from existing `agentMeta` and `estimateUsageCost` best-effort, preserves metadata when lifecycle wins before dedupe is written, documents the protocol shape, and adds focused tests for cached, live, cache-only, no-usage, and cost-config-failure paths.
