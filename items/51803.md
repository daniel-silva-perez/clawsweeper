## Summary

Keep open. Current main still does not persist or replay followup queues or drain-window inbound messages across gateway restarts, and PR #51803 remains useful unfinished work with substantive unresolved review concerns rather than a safe auto-close candidate.

## Best Possible Solution

Keep the PR open for maintainer judgment or ask the contributor to rework it around a durable normal-dispatch replay path: persist enough followup/inbound context to preserve routing, queue mode, user role semantics, and wake behavior; handle drain-window arrivals without system-event caps; remove the debug log artifact and split the unrelated `/model` fix. The canonical product bug remains #51620.
