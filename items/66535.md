## Summary

Current main still lacks the cancellation wiring described by the report, and an open closing PR (#66894) is already the implementation candidate. The issue should stay open until that PR is reviewed and merged, or until an equivalent fix lands on main.

## Best Possible Solution

Review and either land or close #66894. If maintainers choose not to use that PR, the equivalent fix should live in src/auto-reply/reply/commands-compact.ts and its runtime barrel: create an AbortController for manual compaction, register a minimal EmbeddedPiQueueHandle with setActiveEmbeddedRun after the previous run has drained, pass abortSignal into compactEmbeddedPiSession, clear the handle in finally, and add regression coverage proving /stop or abortEmbeddedPiRun can cancel in-progress /compact.
