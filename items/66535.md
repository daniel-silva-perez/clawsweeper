## Summary

Current main still lacks the cancellation wiring described in the report. The manual `/compact` handler calls `compactEmbeddedPiSession` without passing an `abortSignal` and without registering a compaction handle in `ACTIVE_EMBEDDED_RUNS`, while the existing abort paths depend on that registry. The provided GitHub context also shows #66894 is an open, unmerged closing PR for this same issue, so the issue should stay open until that PR is reviewed and merged or replaced.

## Best Possible Solution

Review and either land or close #66894. If maintainers do not use that PR, the equivalent fix should wire manual compaction cancellation in `src/auto-reply/reply/commands-compact.ts`: create an `AbortController`, register a minimal compacting `EmbeddedPiQueueHandle` only after any previous run has drained, pass `abortSignal` into `compactEmbeddedPiSession`, clear the handle in `finally`, and add regression coverage proving `/stop` or `abortEmbeddedPiRun(sessionId)` cancels an in-progress `/compact`. Consider whether `src/gateway/server-methods/sessions.ts` needs the same contract for `sessions.compact`.
