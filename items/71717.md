Keep #71717 open. It is a fresh Windows 11 core exec report where every Feishu/Web UI exec command fails with `spawn EPERM`, and current `main` does not show a shipped or source-backed fix for that symptom. The current host-exec path still launches a Windows PowerShell shell through the supervisor, while existing related fixes cover different Windows exec failures.

## Best Possible Solution

Keep the issue open.
