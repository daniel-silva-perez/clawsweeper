## Summary

Keep this PR open. Current main still lacks the PR's diagnostic behavior: internal hook handler errors are logged only as a formatted message, with no stack trace or structured event metadata. The linked bug remains open, and the related root-cause PR #69997 was closed unmerged, so it does not supersede this PR.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to either land this diagnostic change with a focused regression test that asserts `triggerInternalHook` logs `stack` plus `{type, action, sessionKey}` metadata, or replace it with an equivalent maintainer-owned patch. If the original `f.toLowerCase is not a function` reproduction is confirmed, a separate root-cause fix in the model override normalization path should be reviewed as well.
