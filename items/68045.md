## Summary

Keep this PR open. Current main still lacks the requested Control UI grouped-message stripping, group-level empty suppression, and per-line system-event/internal-instruction filtering. The PR is a focused implementation candidate with tests, but the provided review context shows remaining P1 concerns on the latest PR commit that should be resolved before landing rather than closing the PR as obsolete.

## Best Possible Solution

Keep this PR open and ask for a focused revision or maintainer follow-up that preserves `normalizeMessage` assistant directive/media cleanup while applying the shared assistant/user text stripping contract, adds group-level suppression for all-empty rendered groups, extends both `stripInboundMetadata` and `stripLeadingInboundMetadata` with narrowly-scoped system-event/internal-instruction filtering, and keeps regression tests for Control UI grouped bubbles, TUI leading-only history formatting, and multiline system-event lines.
