## Summary

Keep this PR open for maintainer review. Current main still has the exact direct `.trim()` assumptions this PR targets and does not contain the proposed regression coverage. The current `@clack/prompts` / `@clack/core` dependency appears to normalize empty text prompts to `""`, so the original user crash may already be obsolete, but that is not strong enough evidence to auto-close a focused bug-fix PR that hardens a public prompter boundary.

## Best Possible Solution

Do not auto-close this PR. A maintainer should either land a small version of the prompter-boundary normalization plus focused tests, or explicitly close it as unnecessary after deciding that `WizardPrompter.text` implementations must strictly return strings and that the current Clack dependency guarantee is sufficient.
