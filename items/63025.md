## Summary

Keep this PR open. Current main still has a dispatch-level zero-output path: an undefined resolver result becomes an empty replies array, counts are collected, completion is recorded, and the function returns without queuing a final reply. The PR is still a plausible implementation candidate, though it should be reviewed carefully against direct/chat silence policy and queued-followup paths.

## Best Possible Solution

Keep this PR open and review/update it against current main. The best fix is a narrow dispatch-level fallback after final queued counts are known, only for direct-style user delivery with no routed origin, no suppressed delivery, no queued tool/block/final output, no block-stream output, and no intentional handled/no-text or queued-followup branch, plus regression tests for direct empty output and group/channel silence.
