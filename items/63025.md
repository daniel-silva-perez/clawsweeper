## Summary

Keep open. Current main still has the silent no-output path: when `replyResolver` returns no payload, `dispatchReplyFromConfig` turns that into `replies = []`, queues no final reply, reads zero queued counts, records completion, and returns `queuedFinal: false`. The PR's direct/private empty-run fallback and regression tests are not present on main. Existing silent-reply handling covers explicit empty/NO_REPLY payloads, not a missing payload from dispatch.

## Best Possible Solution

Keep the PR open and either land an updated version of its narrow direct/private empty-run fallback or ask the contributor to rebase and validate it against current main. The implementation should remain scoped to true zero-output dispatches, preserve intentional silent branches such as handled hooks and channel/group acknowledgements, and keep regression coverage for both fallback and no-fallback contexts.
