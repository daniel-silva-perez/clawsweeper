## Summary

Keep this issue open. Current main still does not implement the reported `volcengine-plan/doubao-seed-2.0-pro` imageModel path: explicit `agents.defaults.imageModel` is still selected, but the bundled Volcengine Plan catalog remains text-only and lacks the reported model, the Volcengine plugin has no dedicated media-understanding provider contract, and the generic image runtime rejects unknown or non-image-capable model metadata before sending image requests. Related PR #55688 remains only a partial open candidate and is not represented on current main.

## Best Possible Solution

Keep this issue open. Maintainers should live-verify the Volcengine Ark coding image endpoint and exact model id, then either add a Volcengine-owned image-capable catalog/media-understanding path with regression coverage for the reported imageModel flow, or document that `volcengine-plan` imageModel is unsupported and point users to a supported `volcengine/*` image-capable model or custom provider config. Review #55688 as a partial catalog-capability candidate, not as a complete fix until this reported model/endpoint is covered or explicitly ruled unsupported.
