## Summary

Keep open. Current main still does not show source, docs, or test evidence that `agents.defaults.imageModel.primary = "volcengine-plan/doubao-seed-2.0-pro"` works for image understanding. The Volcengine coding catalog is still text-only, the reported model is absent from the bundled catalog, and the plugin does not register a Volcengine-owned media-understanding provider or regression test for this Ark coding image path.

## Best Possible Solution

Keep the issue open. Maintainers should verify the Volcengine Ark coding image endpoint with the reported model, then either add a Volcengine-owned image-capable catalog/media-understanding path with regression coverage, or explicitly document that `volcengine-plan` imageModel is unsupported and point users to a supported `volcengine/*` image-capable model or custom provider config with `input: ["text", "image"]`.
