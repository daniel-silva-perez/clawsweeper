## Summary

Keep open. Current `main` still appears to carry the reported Synology Chat reply-routing bug: inbound Synology DMs store `To`/`OriginatingTo` as prefixed values like `synology-chat:<id>`, and the outbound send path still forwards that raw `to` into `parseNumericUserId()`, which only does `Number.parseInt(userId, 10)` without stripping the prefix. I did not find a current-main fix or a regression test covering prefixed peer IDs.
