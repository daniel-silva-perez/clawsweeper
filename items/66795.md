## Summary

Keep this PR open. Current main still has the Telegram model-picker fallback bug: `resolveTelegramSessionState()` resolves the agent-aware default provider for override parsing, but its final no-override fallback still reads `agents.defaults.model`, so the picker can mark the global default instead of the routed agent's `agents.list[].model`. The PR is a narrow implementation candidate with a focused regression test and no observed security-sensitive file changes.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this fix or an equivalent one that computes the routed agent default once in `resolveTelegramSessionState()`, uses it for both stored override parsing and the final no-override fallback, keeps the regression test for an agent default that differs from the global default, and reruns the Telegram extension lane before merge.
