## Summary

Keep open. Current main added recovery around edit-tool failures, but it explicitly preserves exact-match edit semantics and still rethrows mismatch cases instead of implementing fuzzy/hunk-based replacement or a force-overwrite path. That means the core behavior reported in #62465 is only partially mitigated, not resolved.
