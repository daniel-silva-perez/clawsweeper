## Summary

Keep open. Current main fixes part of the old behavior by restarting channels when `connected === false` after the startup grace, but Mattermost still does not publish transport-liveness timestamps or any ping/pong keepalive. The Mattermost watchdog on main only checks `users/me.update_at` to recover bot disable/enable stale sessions, which is narrower than the reported silent network/WebSocket stall. The reported gap is therefore still plausible on current `main`.

## Evidence

- **Gateway policy now restarts explicit disconnects, but stale-socket detection requires transport activity:** `evaluateChannelHealth()` returns `disconnected` immediately when `snapshot.connected === false`, and only returns `stale-socket` when `snapshot.connected === true` and `lastTransportActivityAt` is present.
  - file: [src/gateway/channel-health-policy.ts:106](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/src/gateway/channel-health-policy.ts#L106)
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **Health monitor restarts any unhealthy channel/account:** The monitor evaluates each account, logs a restart reason, stops the running channel if needed, resets restart attempts, and starts it again.
  - file: [src/gateway/channel-health-monitor.ts:127](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/src/gateway/channel-health-monitor.ts#L127)

  - file: [extensions/mattermost/src/mattermost/monitor-websocket.ts:98](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/extensions/mattermost/src/mattermost/monitor-websocket.ts#L98)

  - file: [extensions/mattermost/src/mattermost/monitor.ts:1965](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/extensions/mattermost/src/mattermost/monitor.ts#L1965)

## Risks / Open Questions

- A silent Mattermost websocket stall that never emits `close` can still evade the gateway stale-socket path because Mattermost does not publish `lastTransportActivityAt`.
- Closing as implemented-on-main would likely discard a still-real gap around network-level liveness detection and recovery.
