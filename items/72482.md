## Summary

Keep this PR open. Current main still has the Telegram non-streaming gap the PR targets: with `streamMode: "off"`, Telegram creates no answer draft stream, but still passes `suppressDefaultToolProgressMessages: false`, so generic tool/progress messages can be delivered. The PR is a narrow candidate fix with a matching regression-test update, but maintainers should confirm whether the linked #72363 reasoning-token portion is already covered or should remain tracked separately.

## Best Possible Solution

Review and, if maintainers agree with final-only Telegram non-streaming semantics, land this narrow PR with its regression test. Before merge, either confirm #72363's reasoning-token bleed is already covered by current reasoning-lane handling or adjust the PR/linked issue relationship so the broader reasoning part remains tracked.
