## Summary

Keep this PR open. Current main still emits `gateway.trusted_proxies_missing` for plain `gateway.bind: "loopback"`, and the current regression test still expects that warning. The PR addresses a real noisy audit finding, but the submitted implementation is stale against the current split audit code and the raw `bind !== "loopback"` flip is too coarse for effective-loopback configurations.

## Best Possible Solution

Keep this PR open and request a revision against `src/security/audit-gateway-config.ts`. The fix should base `gateway.trusted_proxies_missing` on effective gateway exposure, stop warning for definitely local-only loopback-resolving binds, still warn for genuinely exposed reverse-proxy cases, and add regression coverage for plain loopback, LAN, custom loopback, and exposed custom/auto behavior.
