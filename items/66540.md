## Summary

Current main still contains the reported false-default veto: `resolveReplyThreadingForPayload` refuses to apply the implicit `replyToId` whenever `payload.replyToCurrent === false`, while the directive parser still emits `replyToCurrent: false` as the normal no-tag default. The linked fix attempts were closed unmerged, and the nearby tests cover implicit threading only when the field is absent, not when a normalized payload carries `replyToCurrent: false`.

## Best Possible Solution

Keep the issue open. The narrow fix should remove `params.payload.replyToCurrent === false` from the implicit-threading skip condition in `src/auto-reply/reply/reply-payloads-base.ts`, then add a regression test in `src/auto-reply/reply/reply-plumbing.test.ts` for a payload like `{ text: "Hello", replyToCurrent: false }` with `replyToMode: "all"` or `"first"` and `currentMessageId: "42"` resolving to `replyToId: "42"`. Maintainers should also confirm whether followup delivery has the current message id available where this issue observed the loss.
