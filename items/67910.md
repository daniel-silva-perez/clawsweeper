## Summary

Keep open. Current main still has the reported `memory status` behavior: the CLI dreaming summary resolves the deep-promotion config and reports `off` when the deep phase is disabled, even though the broader dreaming config has separate light/deep/REM phase state. The related #67923 is closed and unmerged, so it does not supersede this PR. This PR should not be auto-closed, but it needs maintainer review because the current patch has accuracy concerns around deep-disabled and `limit=0` configs and includes some unrelated surface changes.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a narrower status fix that reports the effective scheduled dreaming sweep accurately and, if phase-level details are shown, makes clear which phases are configured versus which managed execution path will actually run. Address the existing review comments for deep-disabled and `limit=0` configs, and split unrelated `--force` or timezone cleanup unless maintainers want them bundled.
