## Summary

Keep #71867 open. Current main still reports doctor model-auth health from one implicit auth store and labels the note generically as `Model auth`; it does not enumerate agent stores, include an agent id, or detect divergent per-agent auth state. The related #71923 PR is the active focused fix, but it is not on main yet.

## Best Possible Solution

Keep #71867 open until #71923 or an equivalent fix lands. The implementation should make doctor auth health enumerate the relevant agent auth stores or at least label the checked store with its agent id, detect divergent per-agent auth state when multiple stores exist, and add focused regression coverage for multi-agent doctor output.
