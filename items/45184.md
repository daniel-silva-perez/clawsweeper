## Summary

Current `main` does not implement an opt-in Matrix dispatch skip for messages that mention a different configured bot account. The handler only checks whether the current bot was mentioned, and when `requireMention` is disabled it still dispatches room messages even if another bot is the one explicitly mentioned.

## Evidence

- **Dispatch gating only checks the current bot mention:** `resolveMentions(...)` is called with `userId: selfUserId`, and the room gate only skips when `shouldRequireMention` is true and `wasMentioned` is false. There is no branch that inspects mentions of other configured bot accounts before dispatch.
  - file: [extensions/matrix/src/matrix/monitor/handler.ts:900](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/extensions/matrix/src/matrix/monitor/handler.ts#L900)
  - command: `nl -ba extensions/matrix/src/matrix/monitor/handler.ts | sed -n '890,990p'`
  - sha: [4630ce3d9e2f](https://github.com/openclaw/openclaw/commit/4630ce3d9e2f089062ccd386b0ae241325961d1a)
- **Mention parser is single-user only:** `resolveMentions` builds `mentionedUsers` from `m.mentions.user_ids`, but only treats a mention as actionable when `mentionedUsers.has(params.userId)`. It does not expose or evaluate whether some other configured bot account was mentioned.
  - file: [extensions/matrix/src/matrix/monitor/mentions.ts:218](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/extensions/matrix/src/matrix/monitor/mentions.ts#L218)

- **No config surface for exclusive other-bot mention skipping:** The Matrix config schema exposes `requireMention`, `allowBots`, and `autoReply` for rooms/accounts, but there is no `exclusive` mention mode or `skipIfOtherBotMentioned`-style option.
  - file: [extensions/matrix/src/config-schema.ts:44](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/extensions/matrix/src/config-schema.ts#L44)

- **Docs only describe sender-side bot gating:** Current Matrix docs say `allowBots: "mentions"` accepts messages from other configured Matrix bot accounts only when they mention this bot. That is a sender gate for bot-authored traffic, not a skip for human messages that mention a different bot while this room has `requireMention: false`.
  - file: [docs/channels/matrix.md:264](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/docs/channels/matrix.md#L264)

There is no test covering suppression when a different configured bot is mentioned.
