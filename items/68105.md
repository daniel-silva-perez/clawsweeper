## Summary

Keep open. Current main still keeps RTL bidi isolation inside the TUI formatter only. Gateway live chat projection and outbound delivery paths pass assistant text through without applying RLI/PDI isolation, and the non-TUI outbound tests do not cover RTL wrapping. The issue’s distinction from the closed TUI and Telegram-preview reports remains valid.

## Best Possible Solution

Keep this issue open and implement a shared bidi-isolation helper, likely under `src/shared/text/`, then apply it on the canonical user-visible outbound path before channel chunking/send and reuse it from the TUI formatter. Add focused regression tests for shared text, outbound delivery/plain-text surfaces, and the no-double-wrap behavior already covered in TUI.
