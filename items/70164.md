## Summary

Keep open. The issue is recent, has concrete core gateway logs, and current main still has a plausible source-level path where lifecycle end can emit a chat final without any message if no assistant text event reached the gateway. I found buffering/flush coverage for normal assistant text, but no source, test, changelog, or release evidence that the reported MiniMax no-assistant-output-after-tool case was fixed.

## Best Possible Solution

Keep the issue open and add a focused regression around MiniMax or Anthropic-compatible streaming after a tool result where the model produces no visible assistant text before lifecycle end. The fix should belong in the embedded runner/subscription/gateway path: either recover/retry the empty assistant turn or surface an explicit error chat final, but not silently emit state:"final" with no message.
