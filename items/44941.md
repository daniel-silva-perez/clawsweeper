## Summary

Keep open. Current `main` added fallback lifecycle events and Control UI session-scoped fallback toasts, but it does not implement the issue's requested default per-session/per-topic chat notification behavior. The only in-chat fallback/recovery notices are still gated behind verbose mode, and the default verbose setting is `off`.

## Evidence

- **Fallback/recovery events exist, but chat notices are still verbose-gated:** `agent-runner.ts` now emits `phase: "fallback"` and `phase: "fallback_cleared"` lifecycle events for the session, but the actual reply-text notices are only added inside `if (verboseEnabled)` blocks.
  - file: [src/auto-reply/reply/agent-runner.ts:1497](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/src/auto-reply/reply/agent-runner.ts#L1497)
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **Tests prove no default in-chat notice when verbose is off:** The e2e test explicitly runs with `resolvedVerboseLevel: "off"` and asserts the reply text does not contain `Model Fallback:` or `Model Fallback cleared:` even though both lifecycle phases were emitted.
  - file: [src/auto-reply/reply/agent-runner.runreplyagent.e2e.test.ts:890](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/src/auto-reply/reply/agent-runner.runreplyagent.e2e.test.ts#L890)
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **Default verbose mode is off:** The agent config docs say `verboseDefault` defaults to `"off"`, so the existing reply-text fallback notices are opt-in rather than built-in default behavior.
  - file: [docs/gateway/config-agents.md:363](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/docs/gateway/config-agents.md#L363)
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **Main does have session-scoped UI fallback toasts:** Control UI consumes the lifecycle events, scopes them by `sessionKey`, and renders `Fallback active` / `Fallback cleared` indicators for the active session.
  - file: [ui/src/ui/app-tool-stream.ts:388](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/ui/src/ui/app-tool-stream.ts#L388)
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **No `fallbackNotify` config surface found:** A repo search for `fallbackNotify` / `fallback notify` returned no matches, so the issue's proposed default-on notification toggle does not exist on current `main`.
  - command: `rg -n "fallbackNotify|fallback notify" src/config docs src --hidden`

## Risks / Open Questions

- This issue is partially addressed for Control UI users: they now get default session-scoped UI indicators on fallback and recovery.
- The requested delivery target is stricter than what `main` ships today: there is still no evidence of default notification messages being posted into the affected chat/topic when verbose mode is off.
- The relevant implementation landed after the latest listed release `v2026.4.22`, so even the partial UI behavior is not yet tied to a shipped release in the provided context.
