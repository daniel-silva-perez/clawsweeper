## Summary

Current `main` does not implement a Dashboard/WebChat message-reply action. The UI can display existing reply metadata, but the composer/controller has no reply-target state or right-click/context-menu hook, and the `chat.send` WebSocket schema still lacks any `replyToId`/`replyToCurrent` field. This remains an open feature request, not a cleanup close.

## Evidence

- **WebChat uses `chat.send` for browser chat:** The WebChat docs say the UI talks to the gateway through `chat.history`, `chat.send`, and `chat.inject`, so a reply feature would need to flow through the existing WebSocket chat surface.
