## Summary

Keep this PR open. Current main still targets the Feishu typing reaction at `replyToMessageId`, which is the topic/thread root in `group_topic` and `replyInThread` flows, so the user-visible bug described by the PR is still present in source. The PR is a focused implementation candidate with matching regression-test intent; it is not obsolete, not implemented on main, and not superseded by the linked closed bug report.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land or reimplement the narrow fix: thread `ctx.messageId` into `createFeishuReplyDispatcher` as an optional inbound-message target, use it for `addTypingIndicator` with `replyToMessageId` fallback, and keep/add the focused dispatcher and bot tests covering topic/replyInThread behavior.
