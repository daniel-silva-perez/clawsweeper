## Summary

Keep #48883 open. Current main still emits relay-style cron reminder prompt prose, accepts rich main-session systemEvent text, defaults systemEvent jobs to the main session, and even tells the agent to write reminder prose into systemEvent payloads. The PR is not implemented or superseded; it is a plausible narrow remediation, but the posted review comments identify issues that should be addressed before maintainers consider landing it.

## Best Possible Solution

Keep #48883 open and ask for a rebase/revision on current main: replace cron relay prose with a minimal main-session wake token path, add a main-session systemEvent text validator/coercer, preserve rich reminders through isolated agentTurn jobs, update agent/tool/CLI/UI guidance and tests, and explicitly address the posted review comments about deliverToUser semantics and legacy job updates. Coordinate the result with the broader #66496 system-event classification work rather than closing this PR as obsolete.
