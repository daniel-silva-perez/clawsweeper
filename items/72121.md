## Summary

Keep #72121 open. Current main at 6b6dcafcee9ca86385b7c1ee2442752314d50257 still rejects `sk-ant-api03-...` on the reported `models auth paste-token --provider anthropic` path, because that path calls a validator that only accepts `sk-ant-oat01-`. A separate Anthropic API-key auth method exists, and open PR #72124 appears to target this exact bug, but the fix is not on current main.

## Best Possible Solution

Keep #72121 open until a main-merged fix lands. Best path is to land or adapt #72124: widen the Anthropic pasted-credential validator to accept `sk-ant-api03-` where `paste-token` should store a usable credential, or update the CLI/help/error text to direct API-key users to `models auth login --provider anthropic --method api-key`; add regression coverage for both `sk-ant-oat01-` and `sk-ant-api03-` on the helper and CLI path.
