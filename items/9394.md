## Summary

Current `main` does not implement rich-text formatting or iMessage effects for the legacy `imessage` channel. The legacy outbound path still sanitizes to plain text and sends only `text`/`file` through `imsg rpc`, while project docs explicitly describe iMessage as plain-text/legacy and point users to BlueBubbles for effects. This remains a valid unimplemented feature request rather than cleanup-close material.

## Evidence

- **Legacy iMessage outbound is plain-text only:** The channel outbound config uses `sanitizeForPlainText(...)` with text chunking, not a rich-text renderer or effect-aware path.
  - file: [extensions/imessage/src/channel.ts:241](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/extensions/imessage/src/channel.ts#L241)
  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
- **`imsg` send request exposes no formatting/effect fields:** `sendMessageIMessage(...)` converts markdown tables, strips inline directives, then sends RPC method `send` with params limited to `text`, `service`, `region`, optional `reply_to`, optional `file`, and target selectors (`to`/`chat_id`/`chat_guid`/`chat_identifier`). There is no rich-text span payload, formatting flag, or effect id.
  - file: [extensions/imessage/src/send.ts:97](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/extensions/imessage/src/send.ts#L97)
  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
- **Docs still classify iMessage as plain-text/legacy:** The markdown-formatting docs say Slack/Telegram/Signal render from the shared formatting IR, while iMessage still uses plain text or its own rules. The iMessage channel doc calls the `imsg` integration legacy and recommends BlueBubbles for new setups.
  - file: [docs/concepts/markdown-formatting.md:55](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/docs/concepts/markdown-formatting.md#L55)
  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
- **Effects support exists on BlueBubbles, not legacy `imessage`:** Channel overview says BlueBubbles is the recommended iMessage path with full feature support including effects, while `iMessage (legacy)` remains a separate deprecated path. BlueBubbles docs/config include `sendWithEffect: true` for message effects.
  - file: [docs/channels/index.md:14](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/docs/channels/index.md#L14)
  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
