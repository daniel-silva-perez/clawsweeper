## Summary

Keep PR #72213 open. Current main still has the reported Discord preview tool-progress duplicate path: Discord block preview progress only suppresses an exact repeat of the immediately previous line, while the active callbacks can emit the same tool as both `tool: session_status` and `session_status`. The PR is a narrow display-layer fix with a targeted regression test for linked issue #72205, and the supplied discussion/reviews support the same root cause rather than showing that it has already shipped elsewhere.

## Best Possible Solution

Keep the PR open for normal maintainer review and CI. The best path is to land this narrow Discord plugin fix, or an equivalent change, with the regression test in `extensions/discord/src/monitor/message-handler.process.test.ts` so preview tool-progress lines deduplicate by logical tool name instead of exact adjacent text.
