## Summary

Keep PR #72213 open. Current main still has the reported Discord block-preview duplicate tool-progress path: the handler only deduplicates the immediately previous exact line, while two active callbacks can emit the same tool with and without the `tool:` prefix. The PR is a focused, recent bug fix with a targeted regression test for the linked open issue #72205, and I did not find evidence that main already implements or supersedes it.

## Best Possible Solution

Keep the PR open for normal maintainer review and CI. The best product/code path is to land this narrow Discord plugin fix, or an equivalent change, with the regression test in `extensions/discord/src/monitor/message-handler.process.test.ts` so block preview tool-progress lines deduplicate by logical tool name rather than exact adjacent text.
