## Summary

Keep this PR open. Current main still eagerly resolves the compatibility provider env-var export at module import time, while OpenRouter's auth env var is manifest-owned. The PR remains a relevant implementation candidate, but it needs review fixes and careful handling of the security-sensitive provider env-var hardening it added.

## Best Possible Solution

Keep this PR open as the implementation candidate for the unresolved main-branch lazy-loading bug. Narrow or approve the production change that makes `PROVIDER_ENV_API_KEY_CANDIDATES` delegate to `PROVIDER_AUTH_ENV_VAR_CANDIDATES`, fix the regression test assertion order, and handle the broader provider env-var security hardening as a reviewed follow-up or a clearly scoped part of this PR.
