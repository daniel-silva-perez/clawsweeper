## Summary

Keep #50968 open. Current main still hardcodes LiteLLM and Copilot Proxy model limits during onboarding/catalog setup and has no `/v1/model/info` capability probe. The PR is stale against moved LiteLLM files, but it still contains useful product work and review feedback; #51055 remains the linked open bug rather than a shipped or superseding fix.

## Best Possible Solution

Keep the PR or its linked issue active and port the behavior to current plugin-owned code: add a provider-owned async `/v1/model/info` probe for `extensions/litellm` auth/onboarding, add the analogous Copilot Proxy probe in `extensions/copilot-proxy/index.ts`, preserve the existing 128k/8192 fallback, send available API-key auth, avoid duplicate `/v1`, patch stale saved limits only after successful discovery, and cover both success and fallback paths in extension tests.
