## Summary

Keep this PR open. Current main still has the reported product gap: host-read local HTML/XML/CSS/XML documents are MIME-detected by extension, but `assertHostReadMediaAllowed()` does not allow those fallback document MIME types. The PR is a plausible implementation candidate, but the current head should be revised before merge because the proposed fallback allow path is extension-based and needs the same content/text validation used for CSV and Markdown.

## Best Possible Solution

Keep the PR open and ask for a revision that extends the existing validated text fallback policy to HTML/XML/CSS/XML. The fix should require no disallowed binary sniff plus text-content validation, and should add regression tests for real `.html`/`.xml`/`.css` host-read sends and binary or ZIP payloads renamed to those extensions.
