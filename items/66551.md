## Summary

Keep this PR open. Current main still rejects host-local HTML/XML/CSS/XML fallback text documents in the host-read guard; #51562 only shipped MIME mapping/outbound fallback, and #67047 only shipped the CSV/Markdown validated-text pattern. The latest PR head addresses the earlier security blocker by routing HTML/XML/CSS/XML/text through the same sniff/text validation path and adding binary/ZIP rejection coverage, so it remains a plausible implementation candidate rather than obsolete.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate. The best path is to land a small `src/media/web-media.ts` change that extends the existing validated text fallback policy to HTML/XML/CSS/XML, keeps binary and ZIP renamed-file rejection coverage in `src/media/web-media.test.ts`, fixes the changelog attribution, rebases on current main, and explicitly decides whether validated `text/plain` belongs in the same change.
