## Summary

Keep open. The RFC is coherent, recent, and still not implemented on current main. OpenClaw has a few ad hoc freshness indicators, but shared agent tool results, `session_status`, and `memory_search` do not expose the proposed standardized `freshness` envelope, and external plugins cannot globally add a pre-model tool-result wrapper today.

## Best Possible Solution

Keep the issue open for maintainer design. The best path is a backwards-compatible core/SDK contract for optional tool-result freshness metadata, documented for plugin authors, with targeted adoption in `session_status`, `memory_search`, cron status/list/runs, and cached web tools. Tests should prove existing `content`/`details` consumers and provider replay still work.
