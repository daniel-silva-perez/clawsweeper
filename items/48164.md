## Summary

Current main already implements cross-channel Control UI session refresh for shared sessions. The gateway broadcasts transcript updates as `session.message` for any appended session message, the Control UI subscribes to session events on connect, and the UI reloads chat history when a `session.message` arrives for the active session. This behavior is also present in the released `v2026.4.22` tag, so the issue's reported stale-webchat behavior on `2026.3.2` is obsolete on current releases.

## Close Comment

Closing from a Codex review. Current `main` already implements this behavior, and the released `v2026.4.22` tag contains it as well. The gateway now broadcasts transcript updates as `session.message` for subscribed session viewers (`src/gateway/server-session-events.ts:83`), the Control UI subscribes to session events on connect (`ui/src/ui/app-gateway.ts:330`, `ui/src/ui/controllers/sessions.ts:128`), and the UI reloads the active session’s chat history when a matching `session.message` arrives (`ui/src/ui/app-gateway.ts:481`). There is also regression coverage for that refresh path in `ui/src/ui/app-gateway.sessions.node.test.ts:129`. Given the report was against `2026.3.2`, this looks like behavior that has since been implemented on current releases. If you still hit stale shared-session updates on `v2026.4.22` or newer, please open a new issue with exact channel pair, session key shape, and gateway logs.
