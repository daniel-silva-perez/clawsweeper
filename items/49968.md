## Summary

Keep PR #49968 open. Current main still has the mixed daily-boundary behavior the PR targets: daily reset uses host-local Date#setHours, session-memory filenames/headers use UTC slices from the event timestamp, cost usage buckets use the gateway host timezone, and /usage cost computes Today independently. The PR is broad and needs current maintainer review/rebase, but it is a concrete, actively maintained core bugfix rather than something already implemented, duplicate, or ClawHub-only.

## Best Possible Solution

Keep #49968 open. The best maintainer path is to rebase or split it against current main, introduce a shared human-day/day-bucket helper or equivalent, update daily reset, session-memory, and /usage cost consistently, preserve explicit usage request date modes, and keep plugin memory flush paths authoritative with focused tests.
