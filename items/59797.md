## Summary

Keep open. The issue is a recent, concrete regression report for `openclaw channels login --channel openclaw-weixin` on Windows/OpenClaw 2026.4.1, and there is no strong evidence from current main that it has been fixed or is unreproducible. Later related open reports describe the same Weixin QR/login surface on 2026.4.5, so this does not meet the conservative close bar.

## Evidence

- **Concrete original report:** Issue #59797 reports OpenClaw 2026.4.1 on Windows 10 with `openclaw-weixin` 2.1.3 failing before QR display: `Failed to start login: AbortError: This operation was aborted`; expected behavior is a QR code.
- **Related later regression remains open:** Issue #62120, cross-referenced from #59797, reports `openclaw channels login --channel openclaw-weixin --verbose` hanging before QR display on OpenClaw 2026.4.5 with `@tencent-weixin/openclaw-weixin` 2.1.6, suggesting the same login/QR surface was still suspect after 2026.4.1.
- **External plugin exists but does not justify clawhub close:** Tencent/openclaw-weixin is an already-posted external WeChat plugin with documented install and QR login flow, so this is not a request to add a new core integration; the report concerns OpenClaw host/plugin login behavior with an existing plugin.
- **Local inspection blocked:** Read-only shell commands in the checked-out repository failed before execution with `bwrap: loopback: Failed RTM_NEWADDR: Operation not permitted`, so I could not verify current main source/tests locally or attempt a reproduction without mutating the checkout.
