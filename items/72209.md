## Summary

Keep PR #72209 open. Current main does not implement passive session recovery checkpoints, recovery-bundle storage, or system-prompt recovery discipline for automatically recreated sessions. The PR is recent, open, contributor-authored, and not protected, but the requested behavior is still meaningful and not superseded by existing restart recovery, task ledger, session reset, or compaction checkpoint surfaces. The Greptile review identifies P2 concerns that should be addressed during normal review rather than used as a cleanup close reason.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to refine the implementation on the PR: address the bot review concerns, keep recovery checkpoints passive and informational, ensure approvals/auth/permissions are never inherited, add focused tests around update_plan/sessions_yield/sessions_spawn and prompt composition, and decide whether docs are needed for the new recovery-state surface.
