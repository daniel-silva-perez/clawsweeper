## Summary

Keep PR #72209 open. Current main still does not implement passive session recovery checkpoints, recovery-bundle storage, or system-prompt recovery discipline. Existing restart/orphan recovery paths actively resume interrupted sessions via synthetic gateway messages, which is not equivalent to the PR's passive, informational-only recovery model. The PR is recent and meaningful, with review concerns to address during normal maintainer review rather than a cleanup close reason.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to refine PR #72209 rather than close it: address the redaction and sync-I/O review comments, keep recovery data passive and non-instructional, preserve the no approval/auth/permission inheritance invariant, add focused tests around prompt composition and update_plan/sessions_yield/sessions_spawn checkpoint behavior, and decide whether user-facing docs are needed for the new recovery-state surface.
