## Summary

Keep #70185 open. Current main still has the exact CLI-metadata gap the PR targets: `memory-core/cli-metadata.ts` only registers the CLI command, and `loadOpenClawPluginCliRegistry()` only wires `registerCli`, so `api.registerMemoryCapability()` would still be swallowed by the API builder no-op in CLI-metadata mode. The full memory plugin does register `publicArtifacts`, and wiki bridge import consumes `listActiveMemoryPublicArtifacts()`, so the PR remains a plausible focused fix rather than an obsolete or already-implemented change.

## Best Possible Solution

Keep the PR open and land this fix or an equivalent current-main patch after maintainer review. The implementation should register memory-core publicArtifacts from the CLI metadata entry, add a `registerMemoryCapability` handler in `loadOpenClawPluginCliRegistry()` with the same memory-kind and dual-kind slot guards as full registration, restore memory state on CLI-metadata registration failure, and add focused regression coverage for the CLI wiki bridge/publicArtifacts path.
