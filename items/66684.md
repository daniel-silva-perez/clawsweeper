## Summary

Keep open. Current main still contains the concrete remote-script execution pattern reported in the Docker build, and public Docker rebuild docs still show the same Bun `curl | bash` install pattern. The cited credential-pattern examples look stale or false-positive, but the remaining Docker/supply-chain surface is real enough for maintainer hardening rather than conservative closure.

## Best Possible Solution

Keep the issue open and narrow it to Docker/supply-chain documentation hardening: either replace the Bun/Homebrew remote installer pipes with pinned or verified installation steps where practical, or document why those build-time installer risks are accepted. The stale credential-pattern hits can be acknowledged as false positives once the actionable Docker/script surfaces are resolved.
