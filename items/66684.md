## Summary

Keep open. Current main still contains the concrete remote-script execution pattern in Docker build surfaces and user-facing Docker rebuild docs. Several cited credential-pattern hits appear stale or false-positive, but the remaining Docker/supply-chain hardening concern is real enough for maintainer action rather than conservative closure.

## Best Possible Solution

Keep this issue open and narrow it to Docker/supply-chain hardening. Replace the Bun and Homebrew remote installer pipes in `Dockerfile`, `Dockerfile.sandbox-common`, and `docs/install/docker.md` with pinned or verified installation steps where practical, or document why the build-time risk is accepted. Separately acknowledge the stale credential-pattern examples as false positives under `SECURITY.md` once the actionable Docker surfaces are resolved.
