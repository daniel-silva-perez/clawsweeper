## Summary

Keep open. Current main has related bundle MCP/LSP infrastructure, but it does not include this PR's unified claw-code parity aliases, compat utility tools, task/team tools, Hermes bridge, MCP resource/auth bridge, or parity harness. The PR also has unresolved security and correctness review findings, so it is not a conservative cleanup close.

## Best Possible Solution

Keep the PR open for maintainer review, but request a split or restack. A practical path is separate focused PRs for core runner/tool-call normalization gaps, MCP/LSP bridge extensions, any task/team or Hermes surfaces, and optional compatibility utilities as plugin/ClawHub work where current APIs are sufficient. Config, network, and control-plane tools need explicit owner authorization and security review before they can be considered for core.
