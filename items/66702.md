## Summary

Keep open. Current main has the narrower #71868 fix for old pruned replay media markers, but the central requested behavior is still missing: image hydration is not scoped to current-turn media, and injected memory/untrusted context can still be scanned for image refs before the model call.

## Best Possible Solution

Implement a focused embedded-runner/media fix: make image hydration current-turn aware, load only explicit current-turn attachments when present unless the user asks to compare prior images, exclude injected `<relevant-memories>` and `Untrusted context` blocks from prompt-ref detection, and add regression tests for stale refs in memory/untrusted context plus recent image-heavy session history.

## Evidence
