## Summary

Current `main` has shipped cron run-history logging, but it does not fully implement the issue’s warning/health surface. `cron.runs` records finished executions for triage, yet there is no dedicated cron health endpoint and `openclaw cron list` still does not flag overdue or repeatedly failing jobs.

## Evidence

- **Cron execution history is implemented:** The Gateway appends a JSONL run-log entry on each finished cron event, including status, error, duration, next run, and delivery details, and exposes that history through the `cron.runs` RPC.
  - file: [src/gateway/server-cron.ts:417](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/src/gateway/server-cron.ts#L417)
  - command: `nl -ba src/gateway/server-cron.ts | sed -n '417,578p'`
  - sha: [3361593442e2](https://github.com/openclaw/openclaw/commit/3361593442e295e9bb1f55cbfb38861ea9413e45)
- **Run history is part of the documented/admin flow:** Docs and troubleshooting now direct users to `openclaw cron runs --id <jobId>` to inspect recent `ok` / `skipped` / `error` executions, which covers the logging/triage part of the request.
  - file: [docs/gateway/troubleshooting.md:383](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/docs/gateway/troubleshooting.md#L383)
