## Summary

Keep PR #49058 open. Current main covers the single-session side for Mattermost DMs, but it does not implement the PR's reply-reference context path (`ReplyToBody` / `ReplyToSender`) or the later Mattermost typing/liveness fixes. The PR discussion also shows concrete review follow-up rather than a low-signal or superseded change.

## Best Possible Solution

Keep the PR open for maintainer review or ask for a current-main rebase/split. The best narrow path is to preserve Mattermost DM reply context by fetching the referenced post only for direct replies, populate `ReplyToBody`/`ReplyToSender` without mutating raw inbound body, preserve explicit outbound `replyToId`, avoid attachment downloads for quoted context, and add focused Mattermost monitor/client tests. The websocket liveness and typing changes should either be reviewed as deliberate companion fixes or separated into follow-up PRs.
