## Summary

Keep open. Current `main` supports `reasoningDefault` only as a per-agent override (`agents.list[].reasoningDefault`), not as the requested global `agents.defaults.reasoningDefault`. The issue's requested fallback chain is still not implemented in the config schema or runtime resolution path.

## Evidence

- **Global agent defaults schema still lacks `reasoningDefault`:** `AgentDefaultsConfig` lists `thinkingDefault`, `verboseDefault`, and `elevatedDefault`, but no `reasoningDefault`. The matching Zod schema also omits it.
  - file: [src/config/types.agent-defaults.ts:297](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/config/types.agent-defaults.ts#L297)
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Top-level config docs still omit the requested option:** The `agents.defaults` example shows `thinkingDefault` and `verboseDefault`, and the bullet list documents `verboseDefault`/`elevatedDefault`, but there is no `agents.defaults.reasoningDefault` entry.
  - file: [docs/gateway/config-agents.md:324](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/docs/gateway/config-agents.md#L324)
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Runtime reasoning resolution still skips any global default:** Reply directive resolution is `directive -> sessionEntry.reasoningLevel -> agentEntry.reasoningDefault -> "off"`; there is no `cfg.agents?.defaults?.reasoningDefault` step.
  - file: [src/auto-reply/reply/get-reply-directives.ts:426](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/auto-reply/reply/get-reply-directives.ts#L426)

  - file: [src/config/types.agents.ts:81](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/config/types.agents.ts#L81)

## Risks / Open Questions

- There is partial support for per-agent reasoning defaults, so maintainers could mistakenly assume the global default also exists.
- `src/agents/tools/gateway-tool.ts` mentions `agents.defaults.reasoningDefault`, which may create false confidence even though the config schema and runtime fallback do not honor it.
