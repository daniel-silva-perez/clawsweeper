## Summary

PR #49980 should not be auto-closed. Current main still resolves Telegram `requireMention` with `activationOverride` before `topicConfig?.requireMention`, so the exact topic-vs-activation precedence change proposed by the PR is not implemented. Broader topic-vs-group behavior has improved, but this narrower semantic remains unresolved and the PR discussion identifies only a missing inverse regression test, not obsolescence.

## Best Possible Solution

Keep the PR open. The best path is either to accept the semantic change with focused tests for both topic-false-vs-activation-true and topic-true-vs-activation-false, or explicitly reject it and document that topic-scoped `/activation` is intended to override topic config.
