## Summary

Keep open. Current `main` still evaluates Telegram `activationOverride` before `topicConfig?.requireMention`, so a topic-level `requireMention` setting can still be overridden by session activation state. That matches the PR's reported bug, and the current docs/config helpers indicate topic-level config is supposed to be the more specific source.
