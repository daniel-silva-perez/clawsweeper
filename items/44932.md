## Summary

Keep open. Current `main` still treats any `doctor.memory.status` probe exception as `checked: true, ready: false`, so a timeout is still converted into a warning-producing failure state rather than an inconclusive probe.

## Evidence

- **Timeout path still marks probe as failed:** `probeGatewayMemoryStatus()` catches all gateway errors and returns `checked: true`, `ready: false`, with `gateway memory probe unavailable: ...`, which matches the false-warning behavior described in the issue.
  - file: [src/commands/doctor-gateway-health.ts:85](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/src/commands/doctor-gateway-health.ts#L85)

  - sha: [4630ce3d9e2f](https://github.com/openclaw/openclaw/commit/4630ce3d9e2f089062ccd386b0ae241325961d1a)
- **Doctor still uses a 3s non-interactive memory probe timeout:** `runGatewayHealthChecks()` passes `3000` ms for non-interactive doctor runs to both gateway health and memory probe calls, so the reported timeout condition remains plausible on current `main`.
  - file: [src/flows/doctor-health-contributions.ts:410](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/src/flows/doctor-health-contributions.ts#L410)
