## Summary

Keep open. Current main still has abort primitives, but it does not implement the unified per-session run-generation invalidation or broad stale-output fencing requested by #70319. The linked PR #70363 is open/draft and, per its own body/comment, only partially implements the requested work while deferring pre-tool gating, new-message takeover, and additional emission-site fences.

## Best Possible Solution

Keep #70319 open as the canonical product/architecture issue until a maintainer-approved implementation lands. The best path is to either land a focused version of #70363 plus follow-ups, or split the remaining work into concrete issues for run-generation/current-run APIs, pre-tool gating in the embedded runner, stale fences for typing/reply/followup/tool-result emissions, subprocess scope cancellation, and new-message takeover semantics.
