## Summary

Keep open. Current main still swallows all `resolveWorkflowFile` failures in the Lobster runner and then falls back to `runtime.runToolRequest({ pipeline })`, so the linked missing-relative-workflow-path bug is not implemented on main. This PR is a plausible implementation candidate for open bug #68101, but the existing review comment identifies a real merge blocker: the PR's extension-only heuristic can misclassify valid inline pipelines ending in file-like tokens such as `config.json`.

## Best Possible Solution

Keep this PR open as an implementation candidate, but revise it before merge: only preserve path errors for inputs that are clearly workflow file references, keep ordinary inline pipelines on the runtime path, and add paired regression tests for both the missing relative workflow file and a valid inline pipeline containing a file-like `.json` argument. If maintainers prefer a different patch, close this PR only after #68101 is fixed on main or explicitly superseded by a merged PR.
