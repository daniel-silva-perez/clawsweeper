## Summary

Keep this PR open. Current main still lacks the generic reply-run quiet-timer watchdog and still has core auto-reply paths that finalize empty reply results without a deterministic user-visible fallback. Existing Telegram preview/tool-progress behavior and the separate Google/Vertex empty-final PR are adjacent, but neither supersedes this PR's core watchdog behavior.

## Best Possible Solution

Keep this PR open and review it as a focused implementation candidate. The likely best path is to land or adapt a narrow core auto-reply fix that adds the generic reply-run progress watchdog and deterministic empty-final fallback, while reconciling its empty-final behavior with #67781 if that PR remains active.
