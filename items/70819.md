## Summary

Keep #70819 open. Current main still fails closed on any install-root-escaping symlink under a node_modules segment, and the current OpenClaw peer-dependency symlink is only created after the dependency-tree scan. The PR addresses a real remaining plugin install scanner gap, and the related symlink-discovery PRs are not the same issue.

## Best Possible Solution

Keep #70819 open or land an equivalent focused fix in the install security scanner: allow only exact node_modules/openclaw and node_modules/.bin/openclaw symlinks whose real target resolves inside the trusted host OpenClaw package root, while preserving the current block for arbitrary, spoofed, nested, or vendored escaping node_modules symlinks. Keep the regression tests for both allowed and denied shapes; consider splitting unrelated CI-stability changes if maintainers want the security-scanner fix isolated.
