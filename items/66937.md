## Summary

Keep open. Current main has some LM Studio keyless/synthetic-auth support, especially for configured models and non-interactive setup, but the reported interactive onboarding path still requires a non-empty LM Studio API key prompt and the docs still describe a required token/placeholder workflow.

## Best Possible Solution

Keep this issue open and fix LM Studio's plugin-owned interactive setup. The likely change belongs in `extensions/lmstudio/src/setup.ts`: allow blank API-key input for local unauthenticated LM Studio, use the existing `lmstudio-local`/synthetic-auth semantics, avoid writing a real API-key profile for that case, and keep real-key/env/ref/header flows intact. Add a focused interactive regression test for blank API-key input and update the public LM Studio docs to describe keyless local setup accurately.
