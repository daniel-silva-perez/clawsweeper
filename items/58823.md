## Summary

Keep open. Current main still implements the opposite precedence from this PR: per-agent subagents.model, then the agent's own model, then global agents.defaults.subagents.model. The cron isolated-agent path mirrors that same order, and current tests explicitly assert that the agent model beats the global subagent default. PR #58823 is therefore not implemented or obsolete; it remains a focused behavior change/fix for open issue #58822.

## Best Possible Solution

Do not auto-close. Maintainers should make the product decision on precedence. If the intended contract is explicit per-agent subagents.model, then global agents.defaults.subagents.model, then agent model fallback, merge or adapt this PR with its regression tests. If #58003's named-agent behavior is preferred instead, close #58822/#58823 manually and update the subagents docs to state that an agent's own model outranks the global subagent default.
