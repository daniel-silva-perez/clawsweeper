## Summary

Keep #69931 open. Current main supports sending WhatsApp reactions and opt-in `message_received` hooks for ordinary WhatsApp inbound messages, but I did not find a WhatsApp inbound reaction/deletion event path or plugin hook payload that exposes emoji plus target-message metadata. Because the missing underlying data/API is the core of the report, this should not be closed as ClawHub yet.

## Best Possible Solution

Keep the issue open and narrow it into a WhatsApp inbound reaction/event API task: listen to the relevant Baileys reaction/deletion event path, normalize authorized events into a documented plugin-visible contract with emoji, add/remove state, actor, chat, target message id, and account metadata, add privacy/config gating similar to `reactionNotifications`, and cover it with WhatsApp tests plus docs. Once that core/plugin API exists, ClawHub skills can build higher-level automations on top of it.
