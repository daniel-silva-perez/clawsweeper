## Summary

Keep this issue open. Current main still ships the same over-broad memory-recall prompt and `memory_search` tool description described in the report, and the bundled memory plugin registers those strings into the agent prompt/tool surface. There is adjacent GPT-5-only tool discipline guidance to prefer tools for mutable state, but it does not remove the mandatory memory rule and does not cover the reporter's DeepSeek environment.

## Best Possible Solution

Update the bundled memory-core prompt section and `memory_search` tool description to distinguish historical recall from current state/configuration verification, then add a focused prompt/tool-description regression test. The fix likely belongs in `extensions/memory-core/src/prompt-section.ts`, `extensions/memory-core/src/tools.ts`, and `extensions/memory-core/index.test.ts`; if agent prompt composition needs stronger precedence, add a targeted test around `buildAgentSystemPrompt` as well.
