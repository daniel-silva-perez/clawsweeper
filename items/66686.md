## Summary

Keep this issue open. Current main still ships the broad mandatory `memory_search` prompt and tool description reported by the user, and the bundled memory plugin actively registers that prompt/tool surface. Adjacent guidance now says mutable facts need live checks, but it does not carve out current system/configuration questions from the memory-specific mandatory rule, and the existing tests still preserve the current `run memory_search` guidance rather than the requested exception.

## Best Possible Solution

Update `extensions/memory-core/src/prompt-section.ts` and `extensions/memory-core/src/tools.ts` so memory recall is framed for historical prior work, decisions, people, preferences, and todos, while current system state/configuration/time/service questions are directed to live verification tools first. Add focused regression coverage in `extensions/memory-core/index.test.ts`; if prompt precedence remains ambiguous, add a `buildAgentSystemPrompt` test around the combined Execution Bias and Memory Recall sections.
