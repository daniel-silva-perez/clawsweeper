## Summary

Keep this issue open. Current main already exposes cron identifiers and running state through the core model, gateway API, and CLI, but the Control UI cron job list still renders only last-run status, next run, and last run. It does not visibly show each job's identifier or treat `state.runningAtMs` as an active running state, so the recent feature request is only partially satisfied.

## Best Possible Solution

Keep open and scope it as a cron visibility enhancement. The likely implementation is to render each cron job's `id` and a computed active state from `job.state?.runningAtMs` in `ui/src/ui/views/cron.ts`, add concise English i18n labels, preserve existing last-run status as separate history state, and add focused `ui/src/ui/views/cron.test.ts` coverage for running, idle, disabled, and identifier display.
