## Summary

Keep this open. Current main has useful partial fixes for auth-profile timeout poisoning and deferred context-engine maintenance, but the central requested live gateway contract is still missing: `agent.wait` and the gateway lifecycle/dedupe snapshots continue to expose flat `ok|error|timeout` outcomes without queue, preflight, provider, post-turn, or gateway-draining attribution.

## Best Possible Solution

Keep this issue open as the OpenClaw tracking item for an additive timeout-attribution contract. The best implementation path is to rework the useful OpenClaw-side pieces from #65511 into focused PRs that add phase-aware timing/timeout metadata to lifecycle events, gateway dedupe snapshots, `agent.wait` responses, protocol schema/docs, and tests, while leaving owner-specific lossless-claw scheduling work to #54488 or the external lossless-claw PR.
