## Summary

Keep open. Current main has stronger restart deferral for queued replies, but the specific early-commentary behavior is not fully implemented: ACP output still defaults to final-only buffering, and Discord preview streaming is opt-in/off by default. The related #55236 covered late main-result reflection, but this issue's early standalone commentary expectation remains a real product gap.

## Best Possible Solution

Add a Discord/ACP boundary that materializes user-visible assistant text before restart-sensitive tool/config actions, or make live ACP delivery the Discord default with a conservative per-channel opt-out. Cover it with a regression where an `agent_message_chunk` precedes a `tool_call` that triggers gateway restart, and assert the early Discord block is delivered before restart/final completion.
