## Summary

Keep this issue open. Current main still suppresses assistant output classified as commentary before Discord partial, block, or final delivery, so there is still no durable standalone Discord message for early commentary before restart-sensitive work.

## Best Possible Solution

Keep this issue open for maintainer product and channel-delivery work. If early assistant commentary is intended to be user-visible, add a channel-safe materialization and flush path for approved commentary at assistant-message, tool, or restart boundaries, route it through Discord's reply dispatcher and restart-deferral machinery, and add regression tests. If commentary suppression is intentional policy, document that explicitly before closing.
