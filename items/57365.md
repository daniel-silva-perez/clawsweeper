## Summary

Keep open. Current `main` does not implement this PR’s behavior: external plugins still only contribute `channelConfigs` from the manifest registry record, while source-based channel config discovery remains bundled-only.

## Evidence

- **Non-bundled registry records still use manifest-only channel configs:** `buildRecord()` still sets `channelConfigs` from `params.manifest.channelConfigs` passed through `mergePackageChannelMetaIntoChannelConfigs(...)`. There is no external-plugin source discovery path here.
