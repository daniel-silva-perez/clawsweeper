## Summary

Keep open. Current main still does not expose structured non-self Feishu @mentions in inbound conversation metadata, so the PR addresses a real remaining product gap. The proposed patch also needs maintainer work before merge because current Feishu mention payload types and tests show valid mention IDs can arrive without `open_id`.

## Best Possible Solution

Keep #39853 open and implement the behavior with a robust Feishu mention extraction helper that excludes the receiving bot, falls back across `open_id`, `user_id`, and `union_id` where appropriate, adds a typed `Mentions`/metadata contract, and includes focused tests for inbound metadata serialization and Feishu ID fallback. Maintainers should also reconcile this with related Feishu sibling mention-routing PR #38939 rather than auto-closing either from stale cleanup.
