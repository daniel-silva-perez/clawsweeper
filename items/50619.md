## Summary

Keep open. Current `main` still applies `/model` by immediately patching the session model override, with no preflight comparison of current session size against the target model's `contextWindow`/`contextTokens`, and no warning or auto-compaction path. There is a passive high-usage context banner in the UI, but it is independent of `/model`. One secondary point in the issue is partly outdated: built-in providers already support per-model `models.providers.*.models.*.contextWindow` and `contextTokens` overrides.

## Evidence
