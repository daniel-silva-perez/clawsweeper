## Summary

Keep PR #51970 open. Current main still has the original tail-exposing `maskApiKey()` behavior and tests, and the helper is used in `/models list` and auth labels. The PR addresses the right surface, but its current diff still needs revision because the proposed `> 4` branch reveals 4 of 5-6 characters and a review comment already called that out.

## Best Possible Solution

Keep the PR open and ask for a small revision: choose a conservative masking contract that never exposes the suffix and does not reveal most of very short secrets, then update the unit tests and the existing `/models list` and auth-label expectations around `maskApiKey()`. A constant configured/missing display or a length-aware first-character-only branch for short values would address the remaining review concern.
