## Summary

Keep #62910 open. Current main still has the exact `scripts/ui.js` pattern the PR targets: Windows `.cmd`/`.bat` runners enable `shell: true`, then `run()` and `runSync()` still pass a separate args array to `spawn()`/`spawnSync()`. The linked gateway issue #62881 was closed based on a Codex app-server fix in a different path, not because the UI script surface was fixed. The PR is plausible but should be revised and tested rather than auto-closed.

## Best Possible Solution

Keep the PR open but ask for a current-main revision: either route `scripts/ui.js` through the existing pnpm runner/windows cmd helper or add an equivalent tested resolver that materializes Windows `.cmd` calls without `shell:true` plus a separate args array. Add regression coverage for the resulting spawn spec, including `.cmd` runners, safe forwarded args, and paths or args containing spaces.
