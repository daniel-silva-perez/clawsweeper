## Summary

Keep this PR open. Current main still contains the Node v24 DEP0190 spawn shape in `scripts/ui.js`, so the PR is not obsolete, but the submitted fix folds Windows shell args with a plain `args.join(" ")` and does not add regression coverage. That remains a plausible direction only after revising it to preserve Windows argument boundaries, preferably by reusing the existing pnpm runner / cmd.exe quoting helper.

## Best Possible Solution

Keep the PR open and ask for a current-main revision that routes `scripts/ui.js` through `scripts/pnpm-runner.mjs` or the same `buildCmdExeCommandLine()`-based Windows resolver. The revised fix should avoid `shell: true` plus a separate args array, preserve args containing spaces, keep unsafe metacharacter rejection, and add regression coverage for `.cmd` launchers, paths under `Program Files`, forwarded args with spaces, and the final spawn/spawnSync invocation shape.

## Risks / Open Questions

- Merging the PR as-is could fix the DEP0190 warning while regressing forwarded UI args that contain spaces on Windows, because `args.join(" ")` does not preserve argument boundaries.
- The current tests would not catch a future reintroduction of `spawn(file, args, { shell: true })` for `.cmd` launchers.
- Because this is a shell command construction path, the fix should reuse the existing tested Windows quoting helper or an equivalent resolver rather than adding an untested ad hoc command string.
