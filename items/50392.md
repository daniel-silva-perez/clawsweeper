## Summary

Keep open. Current main still has the /bash exec-policy inheritance gap this PR targets: the direct chat command builds a fresh exec tool without passing configured security, ask, PATH prepend, safe-bin, or approval-running-notice policy. The PR is not obsolete or duplicated by the related reports in the provided context, but it should be updated before merge to address the review comment about approvalRunningNoticeMs and ideally keep the policy forwarding consistent with the normal agent exec path.

## Best Possible Solution

Keep #50392 open and update the PR or land an equivalent scoped fix in src/auto-reply/reply/bash-command.ts. The fix should pass the relevant configured tools.exec policy into createExecTool, including security, ask, pathPrepend, safeBins, safeBinTrustedDirs, safeBinProfiles, strictInlineEval if intended, timeout/notify fields, and approvalRunningNoticeMs, with a colocated regression test that mocks createExecTool and proves /bash forwards the configured policy. If maintainers want /bash to honor per-agent exec defaults like normal agent runs, derive the policy through the same agent-aware config path rather than only cfg.tools.exec.
