## Summary

Keep PR #71801 open. Current main still forwards Codex `request_user_input` header/question/option strings directly into the chat prompt with no display sanitization, per-field bounds, or prompt-length cap. The PR is a focused, recent hardening patch for a real user-visible Codex app-server bridge path, but it should not be auto-closed or considered implemented because main lacks the fix and the PR still has unresolved review/security concerns to address before merge.
