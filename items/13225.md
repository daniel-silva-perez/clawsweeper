## Summary

Keep open. Current `main` has `before_tool_call` hooks that can mutate params, block, or require approval, and the hook context already includes `agentId`/session metadata. But the requested capabilities are still not implemented: the `before_tool_call` event payload has no preceding model output, and the result type has no model/session redirect mechanism. Separate hooks exist for model override before a run starts (`before_model_resolve`) and for observing LLM input/output, but they do not provide a pre-tool-call reroute seam.

## Risks / Open Questions

- The comment use case about missing `agentId` is partly obsolete because `PluginHookToolContext` already includes `agentId`; closing the whole issue on that basis would be incorrect because the core redirect/pre-tool-output request is still unmet.
- There may be alternative architectural paths for delegation, but none found in current `main` expose the requested capability directly enough to justify closure.
