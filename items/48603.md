## Summary

Keep open. Current `main` still normalizes many rate-limit failures to a generic `⚠️ API rate limit reached. Please try again later.` message, and the tests explicitly lock that behavior. Model/provider labels are only surfaced in fallback notices after a successful model switch, not in the direct error path described in #48603.

- **Generic rate-limit copy is hard-coded:** `RATE_LIMIT_ERROR_USER_MESSAGE` is a fixed generic string, and `formatRateLimitOrOverloadedErrorCopy()` falls back to it for rate-limit errors when no special retry/quota hint is present. There is no model interpolation in this path.
