## Summary

Keep this PR open. Current main still emits and documents the misleading Telegram-only reasoning stream wording, while Feishu reasoning preview support is already wired and tested. The PR is a valid implementation candidate, but it is incomplete as-is because the companion docs still need the same wording fix. Related #68317 overlaps and may be the better canonical patch because it also tracks docs, but it is still open and unmerged, so it is not enough to auto-close this PR without maintainer choice.

## Best Possible Solution

Review this PR together with #68317, choose one canonical patch, and land a single change that updates both the directive acknowledgement and the reasoning docs. A neutral acknowledgement such as `Reasoning stream enabled.` plus docs that mention supported preview-capable channels would avoid hard-coding a channel list that can drift again.
