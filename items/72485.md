## Summary

Keep this PR open. Current main still has the serial per-collection QMD CLI subprocess fanout, and this PR is a fresh, focused implementation candidate tied to the open same-author tracker #72484. It needs normal maintainer review rather than conservative cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to decide between source-grouped invocations and the overlapping parallelization approach in #69583, preserve mixed memory/session diversification, and either add a regression test for missing collection fields in grouped QMD output or document the QMD JSON contract before landing. Once an implementation merges, the paired tracker #72484 can close normally.
