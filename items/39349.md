## Summary

Keep #39349 open. Current main at 4a3030df still suppresses an explicit `pty:true` whenever exec runs with a sandbox runtime, and I did not find the requested warning or an equivalent fix. The linked bug #38601 remains the canonical user problem, but this PR is a focused implementation attempt rather than something obsolete or already shipped.

## Best Possible Solution

Keep #39349 open or land an equivalent narrow fix. The best maintainer path is to add the warning on the existing `warnings` path when `params.pty === true && sandbox`, add a regression test around `createExecTool` proving the warning appears only when PTY is suppressed by sandbox routing, and optionally update https://docs.openclaw.ai/tools/exec to document the limitation.
