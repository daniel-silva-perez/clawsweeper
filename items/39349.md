## Summary

Keep #39349 open. Current main still converts an explicit `pty:true` request to `false` when sandbox routing is active, and there is no current warning branch, regression test, or docs change that makes the PR obsolete. The linked #38601 remains the same unresolved core exec UX bug.

## Best Possible Solution

Keep #39349 open or replace it with an equivalent narrow core exec fix: push a user-facing warning on the existing exec `warnings` path when `params.pty === true && sandbox`, and add a regression test proving the warning appears only when sandbox suppresses PTY.
