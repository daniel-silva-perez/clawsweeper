## Summary

Keep open. Current `main` still renders large non-user messages in the macOS chat UI without any size cap or collapse path, so the reported freeze is not clearly fixed. Only tool-result payloads get a truncated preview; generic/system messages still flow straight into full markdown rendering, and that code is unchanged from release `v2026.4.22` to current `main`.
