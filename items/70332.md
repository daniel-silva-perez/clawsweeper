## Summary

Keep open. Current main still lacks the central behaviors in PR #70332: heading-level-aware stripping for nested/legacy managed daily dreaming blocks, diary-style request-scoped fallback prose, and body-only duplicate prevention for Dream Diary entries. Related #70523 overlaps only the raw fallback pollution path for daily memory files and remains open; it does not cover this PR’s dedupe and managed-block stripping work.

## Best Possible Solution

Keep PR #70332 open, but require a focused revision: preserve italic diary body text while deduping by body, add heading-level-aware stripping for marker-based and legacy heading-only managed dreaming blocks, convert request-scoped fallback output into human-readable diary prose, and reconcile the fallback overlap with #70523 rather than closing this PR as implemented or duplicate.
