## Summary

Keep open. Current main still resolves exec-approvals file and socket paths through hardcoded `~/.openclaw` defaults instead of `resolveStateDir()`, so the PR’s central bug is not implemented or obsolete. The linked issue #62917 and PR discussion describe a coherent current bug, and the newer related PR #66404 is not a clear superseding canonical fix because its public review flags security-regression concerns.

## Best Possible Solution

Keep #62938 open. The best maintainer path is to land or request an updated focused fix that routes exec-approvals file/socket defaults through `resolveStateDir()`, keeps the existing write-safety invariants, adds `OPENCLAW_STATE_DIR` regression tests, and either fixes or deliberately splits the remaining hardcoded host-label/plugin-binding paths noted in review.
