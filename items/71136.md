## Summary

Keep this issue open. Current main still uses strict-priority token resolution in live context-token paths, and maxTokens is still handled inconsistently by provider transports rather than through a shared minimum resolver.

## Best Possible Solution

Implement a shared token-limit resolver that computes the minimum of valid agent defaults, per-agent overrides, provider model metadata, discovered/cache metadata, and safe defaults for contextTokens and maxTokens. Wire it through agent run setup, auto-reply, gateway session defaults/status/session persistence, and provider request construction, then add focused tests for high configured caps with lower model/discovery limits.
