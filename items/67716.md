## Summary

Keep open. Current main still defers background context-engine maintenance only for turn maintenance, while bootstrap maintenance is invoked through the default foreground path. Background maintenance is also the only path that opts into consuming deferred compaction debt, so the reported bootstrap/reconcile gap remains plausible on current code. The hot-cache details appear plugin-owned, but the host scheduling/API behavior is still not resolved in core.

## Best Possible Solution

Keep this issue open. A narrow core fix should add a regression path where bootstrap/reconcile maintenance records deferred debt and then ensure OpenClaw schedules or runs an allowed background maintenance pass to consume it. The hot-cache behavior should be coordinated with lossless-claw so deferred debt is either paid down, explicitly superseded by policy, or surfaced through a maintain result/reason that OpenClaw can schedule around.
