## Summary

Keep open. Current `main` contains partial follow-up for this cluster: gateway status now separates reachability from RPC (`--require-rpc`), unresolved gateway auth SecretRefs degrade to probe `authWarning` instead of a blanket hard failure, doctor/session maintenance can archive orphan transcripts, and cron has failure-destination plus retry/backoff support. But the issue is not fully implemented end to end: at least one reported ask still remains explicitly open on `main`, because `tools.profile` unknown-entry warnings for `apply_patch` are still emitted by code and locked in by tests.

## Evidence

- **Gateway status no longer treats RPC as the default health contract:** Docs say `gateway status` defaults to service/WebSocket/auth-capability reachability, while `--require-rpc` is the opt-in read probe; default timeout is now 10000ms.
  - file: [docs/cli/gateway.md:182](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/docs/cli/gateway.md#L182)
  - command: `nl -ba docs/cli/gateway.md | sed -n '167,205p'`

  - file: [src/cli/daemon-cli/status.gather.ts:397](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/src/cli/daemon-cli/status.gather.ts#L397)
