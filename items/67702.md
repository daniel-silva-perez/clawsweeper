## Summary

Keep this PR open. Current main still starts the git update path before any gateway service preflight, stop, or restart preparation, so a same-install gateway can keep running while `git rebase`/checkout and rebuild work replace `dist` in place. The PR targets a real remaining git-mode update race, but its current auto-stop approach still needs the review feedback addressed before it is safe to merge.

## Best Possible Solution

Keep this PR open and request a revised implementation in `src/cli/update-cli/update-command.ts`: detect a same-install running gateway before in-place git checkout/build work, handle direct dist and wrapper service entrypoints, stop or block only when restart recovery is available, and guarantee restart/restoration on success, skipped updates, and errors. Add focused `src/cli/update-cli.test.ts` coverage for absolute, relative, and wrapper commands, different installs, `--no-restart`, dirty/skipped updates, and failed update paths.
