## Summary

Keep open. Current main has partial safeguards for deleted agents and stale cron isolated sessions, but it does not implement the central multi-location cleanup/reset requested here. Agent deletion still prunes core config/session-store/files only; it does not remove or disable cron jobs for that agent, there is no first-class agent reset command, and the context-engine/plugin contract has no per-agent or per-session deletion hook for Lossless Claw/LCM-owned state. The linked session-corruption reports are narrower and do not supersede this cleanup gap.

## Best Possible Solution

Keep this issue open. The best fix is a scoped core cleanup/repair path: extend `agents delete`/`agents.delete` or add `agents reset --clean-state` to purge core session-store/transcript state, remove or disable cron jobs whose `agentId` matches the deleted agent or clearly report them, add doctor/session cleanup for orphaned agent state, and introduce a versioned context-engine/plugin lifecycle hook so Lossless Claw and other engines can clean session/agent-owned state themselves. Add regression coverage for CLI and Gateway deletion plus cron jobs targeting the deleted agent.
