## Summary

Keep this issue open. Current main still lacks the specific diagnostic for a populated `plugins.allow` whose entries are channel-style IDs rather than discovered plugin IDs, and an open contributor PR (#68389) explicitly targets this issue. Per maintainer cleanup rules, the issue should stay open while that implementation candidate is unmerged.

## Best Possible Solution

Keep this issue open and handle it through #68389 or a maintainer-authored replacement: add a diagnostic-only path that distinguishes an empty plugin allowlist from a populated allowlist whose entries are channel IDs, npm package names, or otherwise unknown plugin IDs, ideally using discovered plugin/channel metadata to suggest the actual plugin ID without changing allow/deny semantics.
