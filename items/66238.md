## Summary

Current main still only partially addresses the report. `status --json` has a lean routed path that avoids plugin registry preload, but it explicitly disables `channelSummary`. The `tasks list --json` and `tasks audit --json` paths still fall through to the shared status/health/sessions/tasks registrar, whose module top-level imports unrelated command implementations and task audit/maintenance modules. The merged agents-list work in #71746 fixes the related #71739 surface, but it does not cover the tasks/status surfaces here.

## Best Possible Solution

Keep this issue open. The best fix is to add or split a tasks-only lightweight JSON path for `openclaw tasks list --json` and `openclaw tasks audit --json`, remove the shared status/health/sessions/tasks import coupling for JSON callers, and implement a tested `status --json` channelSummary strategy that preserves complete JSON without preloading full plugin/runtime machinery. Add import/load regression coverage for these CLI JSON paths.
