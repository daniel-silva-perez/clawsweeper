## Summary

Current main only partially addresses the report. `status --json` has a fast routed path that avoids plugin registry preload, but `tasks list --json` and `tasks audit --json` still enter the shared status/health/sessions/tasks registration module, which eagerly imports unrelated command implementations. The status fast path also explicitly disables `channelSummary`, which conflicts with the issue’s expected complete JSON output. The related #71739/#71746 context is similar CLI JSON plugin-load work for `agents list --json`, not a canonical replacement for this tasks/status report.

## Best Possible Solution

Keep this issue open. The best fix is to split or route `openclaw tasks list --json` and `openclaw tasks audit --json` through a tasks-only lightweight path, remove the shared status/health/sessions/tasks import coupling for JSON callers, and decide a tested lightweight `status --json` channelSummary strategy that preserves complete JSON without loading full plugin/runtime machinery.
