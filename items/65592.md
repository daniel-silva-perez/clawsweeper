## Summary

Keep this PR open. Current main documents the SearXNG JSON API prerequisite, but it still does not show that provider-specific note during `openclaw onboard` or `openclaw configure --section web`. The PR remains a focused implementation candidate, especially after the review discussion moved the implementation away from the late `runSetup` hook and toward a pre-credential `credentialNote` path.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate. The likely best path is a generic, plugin-owned pre-credential note field if maintainers want this as a supported web-search provider contract, plus setup-flow tests for normal credentials, SecretRef mode, and quickstart behavior. If maintainers do not want a new public field, the narrower alternative is a SearXNG-specific setup path that still displays the guidance before collecting the base URL.
