## Summary

Keep open. Current main does not implement PR #50164's Feishu 502 fallback: the downloader still performs one `messageResource.get` call with the supplied `type`, inbound non-image Feishu resources still map to `type: "file"`, and existing tests only assert the current single-call `image|file` contract. The linked bug #49855 remains open in the provided context, and the PR discussion shows the author addressed the concrete review note about preserving the original 502 error when the fallback retry also fails.

## Best Possible Solution

Keep the PR open for maintainer review. The remaining product fix belongs in the Feishu plugin media downloader, with regression coverage for `type=file` HTTP 502 on video-like payloads and clear behavior when the fallback also fails. Maintainers should choose the canonical path across #49855, #49967, and #50164 rather than closing this as obsolete while main still lacks any recovery path.
