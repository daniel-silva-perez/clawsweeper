## Summary

Keep open. Current main still treats structured exec completions, including successful `code 0` and `signal SIGTERM` events, as relayable heartbeat exec events when delivery is enabled. There is no trusted `local-exec`/`exec:` source gate or internal-only classifier on main, and current tests still assert a `code 0` exec completion is sent to Telegram. The PR remains a focused implementation candidate for the linked restart-cleanup chat-noise bug, with useful Greptile review context and related broader tracking in #69492/#67273.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a narrow core heartbeat fix that marks trusted/local structured `code 0` and cleanup `signal SIGTERM` exec completions internal, preserves user relay for actionable failures such as `code 1`, and either lands this focused patch or deliberately folds it into the broader #69492/#67273 event-audience work.
