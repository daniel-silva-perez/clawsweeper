## Summary

Keep this PR open. Current main still treats structured exec completions, including `code 0` and `signal SIGTERM`, as relayable exec-completion events and has no trusted local-exec/internal-only classifier. The PR remains a focused implementation candidate for the linked restart-cleanup chat-noise bug, with useful bot review context and tests in the PR diff.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a narrow core heartbeat fix that marks trusted/local structured `code 0` and cleanup `signal SIGTERM` exec completions internal, preserves user relay for actionable failures such as `code 1`, and either lands this focused patch or deliberately consolidates it with the broader #69492/#67273 event-audience work.
