## Summary

Keep this PR open. Current main still lacks both claimed fixes, and the PR remains useful bug-fix work, but the provided review comments identify real correctness issues that should be revised before merge.

## Best Possible Solution

Keep this PR open and ask for a focused revision: add provider-qualified catalog id matching plus regression coverage; change Control UI attachment reads to report deltas or append operations against live app state; block only sends that would depend on in-flight attachments; preserve `/new`, `/compact`, `/btw`, and local slash-command behavior; and cover concurrent multi-file paste/select/drop races in Control UI tests before merge.
