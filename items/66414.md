## Summary

Current main does not contain the two fixes proposed by this PR: provider-qualified model catalog ids still are not matched, and Control UI chat still sends while FileReader attachment reads can be in flight. The PR should stay open as useful bug-fix work, but its current patch is not merge-ready because the review discussion identifies real stale-state risks in the pending counter and attachment append flow.

## Best Possible Solution

Keep this PR open and ask for a small revision: implement provider-qualified catalog id matching with regression coverage, move attachment-read pending updates to a live state/delta contract, append completed FileReader attachments against the latest state, preserve `/new` and other override/local command behavior, and cover multi-file/concurrent read races in Control UI tests.
