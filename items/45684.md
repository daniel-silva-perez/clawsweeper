## Summary

Current `main` still matches the reported Slack DM threading bug. DM tool context stores the native `D…` channel id, while Slack auto-threading only accepts `channel` targets and rejects `user:U…` DM sends. The fix in PR #45684 is not implemented on `main`, and the current tests still encode the broken DM behavior.

## Evidence

- **DM threading context still stores native DM channel id:** `buildSlackThreadingToolContext` only strips `channel:` targets; otherwise it falls back to `NativeChannelId`, so a DM `To` value like `user:U…` does not become the comparable target address used by the message tool path.
