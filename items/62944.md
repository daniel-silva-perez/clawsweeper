## Summary

Keep open. Current main still hardcodes `timeoutMs: 30_000` at all three explicit `image` tool description call sites, while the config/schema surface exposes configurable image-understanding timeouts. The linked PRs in the provided context are open or closed-unmerged, so they do not prove the issue is shipped or superseded.

## Best Possible Solution

Keep the issue open and fix the explicit `image` tool path by resolving a timeout from the current config contract, likely `tools.media.image.timeoutSeconds` plus any intended per-entry model timeout behavior, then forwarding that value to `describeImage` and `describeImages`. Add focused tests in `src/agents/tools/image-tool.test.ts` proving configured and default timeouts are passed for single-image, multi-image, and fallback paths.
