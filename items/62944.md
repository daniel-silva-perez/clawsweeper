## Summary

Keep open. Current main still hardcodes a 30,000 ms timeout in the agent image tool for batched, single-image, and serial fallback paths, even though the media-understanding config contract exposes timeoutSeconds and the separate media runner already resolves it. The provided GitHub context also shows open, unmerged PR #63483 as the implementation candidate, so the best maintainer path is to review that PR rather than close this issue now.

## Best Possible Solution

Keep this issue open and review PR #63483 as the current implementation candidate. The likely fix belongs in src/agents/tools/image-tool.ts: resolve the configured image timeout once from the existing media-understanding config contract, pass it to every describeImage and describeImages call, preserve the default fallback, and add regression coverage for batched, single-image, and serial fallback paths.
