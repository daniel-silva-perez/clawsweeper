## Summary

Current main still does not accept `agentId` inside `channels.telegram.accounts.<id>`. The documented workaround uses top-level `bindings`, and #63380 remains the open, unmerged implementation candidate for restoring account-level compatibility, so this issue should stay open.

## Best Possible Solution

Review #63380 as the implementation candidate. If maintainers want backward compatibility for configs that worked in 2026.4.5, land a focused schema/type/runtime-compatible fix with regression coverage for `channels.telegram.accounts.<id>.agentId`. If maintainers want `bindings` to remain the only supported routing path, close #63380 with that rationale and improve docs or doctor output so upgraded users can find the migration path.
