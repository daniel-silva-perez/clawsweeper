## Summary

Keep #59390 open. Current main still exposes `streamTo` in the unified `sessions_spawn` schema and still rejects `streamTo` whenever `runtime` resolves to `subagent`, which is exactly the reported failure mode. The issue is also corroborated by duplicate open reports and a linked open PR, so it needs maintainer handling rather than cleanup closure.

## Evidence

- **Unified schema still exposes ACP-only field:** `SessionsSpawnToolSchema` includes `streamTo: optionalStringEnum(SESSIONS_SPAWN_ACP_STREAM_TARGETS)` in the single `sessions_spawn` tool schema, so the field is visible outside the ACP-only execution path.
  - file: [src/agents/tools/sessions-spawn-tool.ts:222](https://github.com/openclaw/openclaw/blob/d6ef1fcf24eea3a5985d653a3baf400e32aa1b75/src/agents/tools/sessions-spawn-tool.ts#L222)
