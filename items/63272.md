## Summary

Keep open. Current main still does not implement `openclaw skills uninstall <slug>` or any lockfile-aware ClawHub skill removal helper, and this issue is the focused canonical follow-up after the broader earlier request was closed as superseded.

## Best Possible Solution

Keep #63272 open for maintainer review and a focused implementation in `src/agents/skills-clawhub.ts` plus `src/cli/skills-cli.ts`: add `openclaw skills uninstall <slug>` that plans and executes removal of the workspace skill directory and `.clawhub/lock.json` entry, supports dry-run/confirmation/JSON behavior consistent with existing CLI patterns, reuses safe install-path validation, and updates docs/tests. Dashboard or gateway RPC support can remain a separate follow-up if maintainers want it.
