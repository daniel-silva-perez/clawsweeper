## Summary

Keep open. Current main still only normalizes outbound reply media from mediaUrl/mediaUrls, so filePath/path/media aliases are not preserved on the auto-reply delivery path. The PR is not obsolete, though it should be rebased and updated for current main plus the precedence concern raised in review.

## Best Possible Solution

Keep the PR open and update it against current main. The best fix is to add alias normalization for filePath/path/media in the outbound reply path, preserve current fields such as sensitiveMedia and replyToId, add focused fallback/precedence tests, and check whether media-only alias payloads need earlier normalization before delivery filtering.
