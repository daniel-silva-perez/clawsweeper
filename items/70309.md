## Summary

Keep #70309 open. Current main still only exposes `mcp.servers` as editable configuration and session-scoped runtime tool discovery; there is no generic Control UI panel or Gateway method for per-MCP-server connectivity, tool names, redacted error details, or one-click refresh/reconnect. The related context supplied for this review contains no comments and the local title-search matches are unrelated or plugin-specific, so there is no canonical duplicate/superseding item to close against.

## Best Possible Solution

Keep the issue open and implement this as a core Gateway plus Control UI diagnostics feature: add a scoped Gateway method that probes configured `mcp.servers` across stdio/SSE/streamable-http, returns per-server status, redacted error details, tool counts/names, timestamps, and supports explicit refresh/reconnect by disposing/recreating the relevant cached runtime. Render it in Control UI near Infrastructure > MCP or Debug, with docs updated under the public Control UI and MCP pages. This should remain core because it diagnoses OpenClaw-managed `mcp.servers`, not a service-specific ClawHub plugin.
