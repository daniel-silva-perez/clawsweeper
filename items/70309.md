## Summary

Keep this issue open. Current main still has MCP configuration and session-time MCP tool materialization, but it does not implement the requested Control UI connectivity panel with per-server status, tool names, manual refresh/reconnect, or surfaced redacted error details.

## Best Possible Solution

Keep this issue open. The best path is an additive Gateway operator diagnostics API backed by the MCP config/runtime loader that returns each configured server name, transport summary, connected/error state, tool names/count, and redacted last error, plus a scoped refresh/reconnect action. Then render it in Control UI under Infrastructure > MCP and document the operator behavior.
