## Summary

Current main still lacks both behaviors targeted by the PR: title extraction does not strip inbound metadata, and gateway-derived titles ignore the externally computed row displayName. The PR remains actionable, with the existing review discussion only calling out missing regression coverage.

## Best Possible Solution

Keep the PR or the linked issue #39722 open until main implements the narrow fix: strip inbound metadata before deriving first-user-message titles, pass the computed row displayName into deriveSessionTitle or equivalent logic, and add regression tests for metadata stripping plus external displayName precedence/fallback.
