## Summary

Keep #71335 open. Current main still resolves `memorySearch.sync.watch` to `true` by default with no gateway-mode branch, and both builtin and QMD memory managers create chokidar watchers whenever that resolved flag is true. The related #64549 is a separate open auto-indexing bug in the same subsystem, not a canonical duplicate for this gateway default/FD-consumption policy change.

## Best Possible Solution

Implement a core memory-sync defaulting path that makes omitted `agents.defaults.memorySearch.sync.watch` resolve to false only for the intended headless gateway runtime, while preserving explicit defaults/per-agent overrides. Add focused resolver tests plus builtin and QMD watcher tests proving omitted gateway defaults do not create chokidar watchers and explicit `sync.watch: true` still does, then update schema help and public memory docs.

## Risks / Open Questions

- `gateway.mode=local` is also the normal local Gateway startup guard, so the implementation needs a precise product definition of headless gateway versus interactive desktop/local gateway before flipping the default.
- Builtin and QMD memory backends both consume the same resolved sync flag, so a fix should preserve parity and keep explicit `sync.watch: true` overrides working.
