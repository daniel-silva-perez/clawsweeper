## Summary

Keep this issue open. Current main still defaults unset `agents.*.memorySearch.sync.watch` to `true` without gateway/headless context, and both builtin and QMD memory managers create chokidar watchers when that resolved value is true. The related #64549 was a separate auto-indexing bug in the same subsystem and does not supersede this default-policy request.

## Best Possible Solution

Keep this issue open and implement a narrow runtime-aware memory sync default: thread an explicit gateway/headless execution context into memory sync resolution or manager creation, default `sync.watch` to false only there, keep explicit user overrides authoritative, and add resolver plus builtin/QMD manager coverage for gateway and non-gateway behavior. Update schema help and memory docs if the default changes.
