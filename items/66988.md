## Summary

Keep this issue open. Current main still follows the reported path: the gateway returns a redacted config snapshot, the redaction fallback sets `raw` to `null` when the redacted raw text cannot be safely restored, and the Control UI treats `raw: null` as Raw mode unavailable, forcing Form mode and showing the reported disabled message. The related direct fix PRs #67010 and #67212 were closed unmerged in the provided GitHub context, so the comment saying it was fixed is not reflected in current main.

## Best Possible Solution

Keep this open unless maintainers explicitly decide that unsafe raw round-trips must hide Raw mode entirely. A durable fix should separate raw display availability from raw editability in the config snapshot/UI contract, preserve a safely redacted raw string for read-only viewing when possible, expose an explicit unsafe-round-trip/editability reason, keep save/apply blocked while unsafe, and update gateway plus Control UI regression coverage and docs.

## Risks / Open Questions

- The current force-Form behavior is documented and test-backed as a safety policy, so changing it should be an explicit maintainer product/security decision rather than a drive-by one-line reversal.
- Showing redacted raw text while blocking edits needs a clear API/UI contract; otherwise the UI may imply raw edits are safe when the gateway cannot round-trip the snapshot.
