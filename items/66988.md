## Summary

Keep this issue open. Current main still has the reported behavior: config.get returns the redacted snapshot, the redaction fallback still sets raw to null when round-trip restore is unsafe, and the Control UI treats raw: null as Raw mode unavailable, forces Form mode, disables the Raw tab, and shows the exact disabled message. The related direct fix PRs #67010 and #67212 were closed unmerged, so the comment saying it was fixed is not reflected in current main.

## Best Possible Solution

Keep this issue open unless maintainers explicitly decide that unsafe raw round-trips must hide Raw mode entirely. A product-quality fix should separate raw display availability from raw editability: preserve a safely redacted raw string for read-only viewing when possible, expose an explicit editability or unsafe-round-trip reason in the snapshot/UI contract, keep save/apply blocked while unsafe, and add regression coverage for the gateway redaction path plus Control UI controller/view behavior.
