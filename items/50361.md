## Summary

Keep open. Current main still dedupes Feishu final replies by raw string equality only, while this draft PR adds a narrower normalization step for visually identical finals that differ by CRLF, zero-width characters, or trailing whitespace. The existing tests cover exact duplicate suppression but not the formatting-equivalent duplicate case from the PR body, and the related reports do not prove this specific work is already merged or cleanly superseded.

## Best Possible Solution

Keep the PR open or fold its focused `fingerprintFinalText` behavior into the canonical Feishu duplicate-final fix. The best path is a small Feishu-owned change that normalizes only presentation-equivalent final text before recording/checking `deliveredFinalTexts`, with the proposed regression test rebased onto current main.
