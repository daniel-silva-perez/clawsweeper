## Summary

Keep open. Current `main` still contains the reported hot paths: the Control UI polls `node.list` every 5 seconds, `node.list` still rebuilds node state on each request from pairing data plus live registry state, and there is no strong evidence in the checked-out `main` or in the diff since `v2026.4.23` of a targeted fix for the reported long-uptime heap growth/OOM. The only post-release change in the directly relevant paths is a device-pairing write optimization, not a read-path or retention fix.
