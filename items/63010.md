## Summary

Keep open. Current main still persists ClawHub plugin installs with a resolved version in `spec`, and the update path reuses that stored `spec` for ClawHub records, so an install-created unpinned ClawHub record can remain pinned on update. The related #64025 is broader ClawHub plugin/skill discoverability work, not a canonical fix for this install/update metadata bug.

## Best Possible Solution

Update ClawHub install persistence to preserve the user's selector: versionless `clawhub:<pkg>` and bare ClawHub-first installs should store a versionless ClawHub spec, while explicit `clawhub:<pkg>@<version>` installs stay pinned. Keep resolved version, integrity, package, family, channel, and resolvedAt metadata in the install record. Add regression coverage for install-then-update behavior proving versionless records resolve latest and explicit version records remain pinned.
