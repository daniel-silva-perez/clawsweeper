## Summary

Keep #38349 open. Current main still does not implement contextual group activation: the shared activation contract, session/config/protocol schemas, Gateway validation, and WhatsApp/Telegram pre-dispatch gating remain limited to mention/always behavior. The related implementation PR #38350 was closed unmerged, and #1721 is an older closed feature request that this issue supersedes rather than a shipped fix.

## Best Possible Solution

Keep #38349 open as the canonical product/API issue. The best path is either to implement a first-class contextual activation contract across config schemas, session persistence, Gateway protocol, WhatsApp and Telegram pre-dispatch gating, observability/CLI/docs/tests, or first add a typed plugin/channel activation seam that lets a ClawHub plugin make the peeking/engaged decision before bundled channel runtimes drop unmentioned group messages. PR #38350 and #1721 remain useful design context, but neither is shipped on current main.
