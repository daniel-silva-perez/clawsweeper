## Summary

Keep PR #51737 open. Current main already covers much of the chat-header provider-qualified picker behavior, but it still has the agent overview picker gap this PR targets: configured `agents.defaults.model.primary` / `fallbacks` refs are collected by a broader helper but are not included in the overview model-select options, so uncataloged configured refs can still appear only as synthetic `Current (...)` options.

## Best Possible Solution

Keep the PR open for a maintainer-quality refresh: port the overview-picker part by making `resolveConfiguredModels()` reuse the broader configured-model collector while preserving alias labels and dedupe behavior, then add the PR's agents overview regression test. Drop or rework the chat-header portion around the provider-qualified model handling already present on current main.
