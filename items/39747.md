## Summary

Keep open. Current main still lacks the bootstrap-before-normalization behavior this PR is trying to add for plugin-only heartbeat targets. Telegram is now covered as a bundled channel id, but the documented/config-validated contract also allows plugin ids, and those can still collapse to `none` when the active channel registry is not already populated.

## Best Possible Solution

Keep the PR or a successor open until main bootstraps channel plugin metadata before heartbeat target normalization and includes a regression test for a plugin-only `agents.defaults.heartbeat.target` when the active channel registry is not already populated. The maintainer should also decide whether `heartbeat.target` should count as startup channel intent in `resolveGatewayStartupPluginIds`.
