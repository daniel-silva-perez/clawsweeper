## Summary

Keep open. Current `main` has partial session infrastructure, but the Control UI does not yet implement the requested ChatGPT-like multi-session flow. The chat toolbar's "New session" action still resets the active chat instead of creating a separate session, and the chat header only switches among existing sessions.

## Evidence

- **Chat toolbar New Session still routes to `/new` on the current chat:** `ui/src/ui/app-render.ts:2271` wires `onNewSession` to `state.handleSendChat("/new", { restoreDraft: true })`, so the primary UI action does not create a distinct session record.
  - file: [ui/src/ui/app-render.ts:2271](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/ui/src/ui/app-render.ts#L2271)
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **`/new` is handled as a reset-style command on the active session:** `ui/src/ui/app-chat.ts:417-423` dispatches `/new` by sending the literal `/new` message through the current session flow with `refreshSessions: true`; it does not call a session-creation RPC or switch to a new session key.
  - file: [ui/src/ui/app-chat.ts:417](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/ui/src/ui/app-chat.ts#L417)
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **Chat UI session control only selects from existing sessions:** `ui/src/ui/chat/session-controls.ts:23-46` renders a `<select>` populated from `resolveSessionOptionGroups(..., state.sessionsResult)` and only calls the switch handler when an existing option is chosen.
  - file: [ui/src/ui/chat/session-controls.ts:23](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/ui/src/ui/chat/session-controls.ts#L23)
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **Control UI docs describe listing and patching sessions, not creating new chats:** `docs/web/control-ui.md:116` documents current Control UI session support as `sessions.list` plus per-session override editing via `sessions.patch`.
  - file: [docs/web/control-ui.md:116](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/docs/web/control-ui.md#L116)
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
