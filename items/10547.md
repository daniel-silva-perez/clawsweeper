## Summary

Current `main` already provides the core platform this proposal asked for: pluggable context engines for custom assembly/compaction/recall, safe transcript rewrite support, persistent compaction with full history kept on disk, and semantic memory recall tools.

## Evidence

- **Context engine plugin slot covers custom recall and compaction:** The docs describe context engines as the component that controls context assembly, summarization, and cross-session recall, and note plugin engines can implement their own compaction strategy and background post-turn work.
  - file: [docs/concepts/context-engine.md:10](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/docs/concepts/context-engine.md#L10)
  - command: `nl -ba docs/concepts/context-engine.md | sed -n '1,280p'`
- **Background maintenance and post-turn indexing are first-class:** The context-engine contract includes `afterTurn` for post-run persistence/background compaction and `turnMaintenanceMode: "background"` for deferred maintenance.
  - file: [src/context-engine/types.ts:47](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/src/context-engine/types.ts#L47)
  - command: `nl -ba src/context-engine/types.ts | sed -n '1,240p'`
- **Safe session transcript mutation API exists:** `ContextEngineRuntimeContext` exposes `rewriteTranscriptEntries`, and the runtime implementation rewrites transcript entries by branching and re-appending safely under a session write lock.
  - file: [src/context-engine/types.ts:137](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/src/context-engine/types.ts#L137)

## Risks / Open Questions

- OpenClaw does not appear to ship an exact built-in feature named AMP; this closure relies on the current context-engine and memory architecture satisfying the proposal's requested platform capabilities rather than matching the proposal's terminology one-for-one.

## Close Comment

Closing after a Codex review against current `main` (`754acc4478fe361db1e5303fd60f589d52beea7a`). The core platform requested here is already present on `main`: OpenClaw now has pluggable context engines for custom assembly/compaction/cross-session recall (`docs/concepts/context-engine.md:10`, `docs/concepts/context-engine.md:67`, `docs/concepts/context-engine.md:190`, `docs/concepts/context-engine.md:251`), a runtime-supported safe transcript rewrite path for session refactoring (`src/context-engine/types.ts:137`, `src/agents/pi-embedded-runner/transcript-rewrite.ts:87`), deferred background maintenance after turns (`src/context-engine/types.ts:54`, `src/agents/pi-embedded-runner/context-engine-maintenance.ts:345`), and shipped semantic memory recall tools plus automatic pre-compaction memory flush (`docs/concepts/memory.md:31`, `docs/concepts/memory.md:61`, `docs/concepts/memory.md:103`). Compaction is also persistent while keeping the full conversation history on disk (`docs/concepts/compaction.md:15`, `docs/concepts/compaction.md:24`). Because the requested architecture is already implemented well enough in current OpenClaw, I’m closing this proposal.
