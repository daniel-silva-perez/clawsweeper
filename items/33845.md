## Summary

Current main still keeps Slack thread participation in a process-local dedupe cache, so the restart-survival behavior this PR targets is not implemented. The PR addresses a real Slack channel bug, but the branch should not land as-is until the open security and storage-correctness concerns are resolved and maintainers choose the canonical persistence primitive.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to preserve the Slack restart-survival fix, then rework it onto the existing `openclaw/plugin-sdk/persistent-dedupe` helper or the accepted persistent keyed-store contract if #70626 lands; remove production-exported test hooks, constrain persistence to the state-dir contract, use bounded/atomic/locked JSON storage, preserve recency/LRU behavior, and keep regression coverage for restart round-trip, TTL expiry, missing/corrupt state, clear semantics, max-entry behavior, and split module instances.
