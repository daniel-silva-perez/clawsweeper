## Summary

Keep open. Current main still does not implement the recursive skill scanning or gateway startup provider-auth warning this PR proposes, and the submitted PR has unresolved review blockers against current auth and skill-loading contracts.

## Best Possible Solution

Keep this PR open or ask for a rework. The best path is likely two focused changes: implement recursive skill discovery with an explicit `maxCandidatesPerRoot` scan budget, watcher coverage, and tests for nested bundle layouts; separately implement any startup auth warning through the existing auth availability contract so it is read-only, best-effort, and cannot block gateway startup or warn for valid auth sources.
