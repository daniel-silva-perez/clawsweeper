## Summary

Keep PR #49841 open. Current main does not implement a Nacos config source, and the remaining request is not currently satisfiable as a ClawHub-only plugin because gateway startup/config loading and hot reload are core file-backed paths with no config-source plugin hook. The PR is a plausible core design discussion, but it still needs maintainer review and fixes for the review-thread concerns before it could be merged or superseded.

## Best Possible Solution

Keep the PR open for maintainer judgment. The best path is either a provider-neutral core config-source abstraction at the config/gateway startup boundary, with Nacos implemented behind it and fully tested across gateway startup, hot reload, CLI reads/writes, env/shell-env handling, SecretRef/source snapshots, and docs, or an explicit new plugin API for config sources followed by a Nacos plugin. Current main is file-backed and lacks the external extension point needed to close this as ClawHub work.
