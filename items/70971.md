## Summary

Keep #70971 open. Current main still appends bundled MCP/LSP tools to the final effective tool list without reapplying `toolsAllow`; main has only a partial MCP materialization gate, which does not make the final allowlist authoritative for bundled tools.

## Best Possible Solution

Keep the PR open and either land or rework the focused fix so runtime `toolsAllow` is applied after bundled MCP/LSP tools are materialized and policy-filtered. Add a regression test proving scheduled `agentTurn` allowlists are authoritative across base tools, bundled MCP, and bundled LSP, while preserving the no-op behavior when `toolsAllow` is unset.
