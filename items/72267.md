## Summary

Keep open. Current main still emits zsh completion output with a leading blank line before `#compdef`, and the PR removes exactly that leading newline. The existing zsh test covers sourced/deferred registration, not the `fpath` autoload first-line requirement described by the PR and zsh documentation.

## Best Possible Solution

Keep PR #72267 open and merge or adapt its one-line zsh generator fix. Add a focused regression assertion such as `getCompletionScript("zsh", program).startsWith("#compdef openclaw\n")`; optionally add a zsh `fpath`/`compinit` integration check when zsh is available.
