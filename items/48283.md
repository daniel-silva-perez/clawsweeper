## Summary

Current `main` does not implement this PR’s generic abort cleanup. It only strips aborted assistant messages for the special `sessions_yield` path, while the normal prompt-abort path still records `promptError` and then snapshots `activeSession.messages` without removing aborted assistant entries. On conservative maintainer cleanup, this PR should stay open.
