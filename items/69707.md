## Summary

Keep PR #69707 open. Current main still forwards configured `embedding.dimensions` to the OpenAI SDK request in `memory-lancedb`, and the current test suite still asserts that outbound behavior. The PR remains a plausible focused bug fix for OpenAI-compatible providers such as vLLM that reject `dimensions`, with regression coverage and a follow-up commit addressing invalid dimension validation.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land or adapt its scoped change in `extensions/memory-lancedb/index.ts`: omit `dimensions` from the upstream OpenAI-compatible embedding request, validate configured dimensions as a finite positive integer, truncate returned vectors locally, fail clearly when vectors are too short, and replace the old forwarding test with focused regression coverage.
