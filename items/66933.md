## Summary

Keep this PR open. Current main still has only group/channel-scoped `toolsBySender`; it does not implement the PR’s global `tools.toolsBySender`, per-agent `agents.<id>.tools.toolsBySender`, or channel-scoped sender key support in the main and bundled tool-policy pipelines. The PR is an active implementation candidate, not obsolete cleanup.

## Best Possible Solution

Keep this PR open for maintainer review. The product decision is whether OpenClaw core should add global/per-agent per-sender tool capability tiers; if accepted, the implementation should be rebased onto current main, include schema/docs/tests for `tools.toolsBySender` and `agents.<id>.tools.toolsBySender`, and settle the channel-key and no-sender wildcard semantics raised in review.
