## Summary

Keep open. Current `main` does not implement per-group `debounceMs` for WhatsApp, and the same pattern is absent from comparable Telegram/Discord group config surfaces. The request is coherent and still represents missing functionality rather than an obsolete or already-fixed report.

## Evidence

- **WhatsApp schema lacks group-level debounce:** `WhatsAppGroupEntrySchema` only accepts `requireMention`, `tools`, `toolsBySender`, and `systemPrompt`. `debounceMs` exists separately in the shared channel/account shape, not inside `groups.*`.
  - file: [src/config/zod-schema.providers-whatsapp.ts:21](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/config/zod-schema.providers-whatsapp.ts#L21)
  - command: `nl -ba src/config/zod-schema.providers-whatsapp.ts | sed -n '1,120p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **WhatsApp types expose debounce only at channel/account scope:** `WhatsAppGroupConfig` has no `debounceMs`, while `WhatsAppSharedConfig` defines top-level/account `debounceMs` for the channel config.
  - file: [src/config/types.whatsapp.ts:25](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/config/types.whatsapp.ts#L25)
  - command: `nl -ba src/config/types.whatsapp.ts | sed -n '20,120p'`

- **WhatsApp runtime only resolves channel/account debounce overrides:** `resolveExplicitWhatsAppDebounceOverride()` checks `channels.whatsapp.accounts.<id>.debounceMs`, `accounts.default.debounceMs`, then `channels.whatsapp.debounceMs`. It never consults `groups.<jid>`.
  - file: [extensions/whatsapp/src/auto-reply/monitor.ts:64](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/whatsapp/src/auto-reply/monitor.ts#L64)

  - file: [docs/channels/groups.md:226](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/docs/channels/groups.md#L226)
