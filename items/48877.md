## Summary

Keep PR #48877 open. Current main still does not implement declarative multi-level `menus`/`routes` for Telegram `customCommands`: the public type and Zod schema only accept `command` and `description`, runtime custom command handling only registers Telegram BotFather menu entries through `setMyCommands`, and docs explicitly say custom commands are menu entries only and do not auto-implement behavior. The PR is a focused Telegram bundled-channel enhancement rather than an obsolete duplicate; it still needs maintainer product review and cleanup rather than automated closure.

## Best Possible Solution

Keep the PR open for maintainer judgment. If bundled declarative Telegram menus are desired, update the PR against current main with strict schema validation, docs/config metadata, callback payload limits, and targeted Telegram runtime tests. If maintainers prefer the plugin boundary, close later with an explicit ClawHub/plugin direction and a documented example using `api.registerCommand` plus Telegram interactive handlers.
