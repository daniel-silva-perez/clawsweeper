## Summary

Keep open. Current main still has the old TUI Ctrl+C status strings with no `activeChatRunId`-based hint, while `/abort` remains the explicit active-run stop path. The linked issue #38501 is still the same observable UX problem, and the PR is a small, focused text-only change rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open or land an equivalent focused TUI change on main. The best path is a narrow update to `src/tui/tui.ts` that appends an active-run hint to the Ctrl+C clear/warn statuses, with wording that does not overpromise Esc, plus a small regression test if the TUI status surface is testable.
