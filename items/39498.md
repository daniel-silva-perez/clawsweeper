## Summary

Keep #39498 open. The ByteString crash class is still real on current main for Comfy Cloud header-bound API keys, but the PR's global SecretInput normalization change is too broad and would silently mutate valid non-header credentials such as Matrix passwords. This needs a scoped header-bound fix rather than closing or merging as-is.

## Best Possible Solution

Keep the PR open and request a scoped patch. Preserve trim-only semantics for normalizeSecretInputString, hasConfiguredSecretInput, and non-header password/body credentials; apply normalizeSecretInput or normalizeOptionalSecretInput only at header-bound API-key/token call sites such as Comfy Cloud X-API-Key and similar provider HTTP auth/header paths; add regression coverage for the Comfy header crash and for Matrix-style password/body credentials preserving non-Latin1 characters.
