## Summary

Do not close this PR as conservative cleanup. Current main does not include the proposed memory-slot scope change or its regression tests, and the memory-wiki CLI still depends on active memory public-artifact registration. The PR is relevant to a real, reproduced bug, but it likely needs rebase/rework against current loader behavior and coordination with overlapping open PRs (#68041 and #69458) before maintainers decide what to land.

## Best Possible Solution

Keep this PR open for maintainer review, but require a current-main rebase and proof that the selected fix actually registers or preserves the active memory public-artifact capability for `openclaw wiki status`, `openclaw wiki doctor`, and `openclaw wiki bridge import`. Maintainers should dedupe it against #68041/#69458 and land the narrowest approach that preserves plugin activation boundaries and has regression coverage.
