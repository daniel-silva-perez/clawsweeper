## Summary

Keep PR #51448 open. Current main still does not forward configured `tool_choice`/`toolChoice` from agent extra params into `SimpleStreamOptions`, while the OpenAI HTTP and WebSocket transports already honor `options.toolChoice` when it reaches them. The PR targets a real remaining gap, though it should be updated for current main and should address the review comment about alias precedence before merge.

## Best Possible Solution

Keep the PR open and update it against current main. The best fix is to canonicalize `tool_choice`/`toolChoice` with source-order precedence before stream option construction, forward the resolved value as `streamParams.toolChoice`, and add focused regression tests in the current extra-params test suite for both snake_case config forwarding and higher-precedence camelCase override behavior.
