## Summary

Keep open. Current main still contains the reported warning path and the restart command can still turn a post-restart health/reporting failure into a false restart failure. A same-author PR remains open as only a partial cleanup, and its own body says it does not necessarily solve the false non-zero restart.

## Best Possible Solution

Keep this issue open while maintainers review PR #66685 as a partial cleanup. The best path is to land or revise that PR if the expired-approval suppression is correct, then add a targeted lifecycle/restart-health fix or regression proof for Linux/systemd restarts so a healthy post-restart gateway does not produce a false non-zero result.
