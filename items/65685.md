## Summary

Keep this issue open. Current main still has the reported behavior: the `enqueue-followup` path drains stale active snapshots, but for a real queued-behind-active-run case it still unconditionally calls `typing.cleanup()` before returning, which clears/seals the queued followup typing controller. The prior ClawSweeper comment correctly identified the same bug, and the current checkout at `e962381dbf1ecdb1cd4784fdeb6c2e07ddd3435b` has not fixed it.

## Best Possible Solution

Keep this issue open and fix `src/auto-reply/reply/agent-runner.ts` by computing whether the queued followup is still behind an active run after enqueue. If the active snapshot is stale, drain immediately as today; if the original run is still active, keep or refresh typing, for example through the existing typing signaler, instead of calling `typing.cleanup()`. Add focused tests proving stale snapshots drain and still-active queued followups do not clean up typing immediately.
