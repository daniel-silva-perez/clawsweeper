## Summary

Keep open. Current main still has the two reported source-level gaps: ACP runtime availability/lookup depends on an already-registered process-local backend, while the bundled acpx service registration happens only when the plugin service starts; and embedded run metadata still has no completed liveness state, with normal terminal success resolving to "working".

## Best Possible Solution

Patch the source paths instead of closing: add a narrow ACP runtime ensure/registration seam used before explicit embedded ACP availability checks and manager backend lookup, then add regression coverage for `openclaw agent --json`/local fallback ACP `sessions_spawn` visibility. Separately, add final-state reconciliation for `sessions_yield` descendant completion in the explicit agent RPC result path, including tests that final JSON has usable finalAssistantVisibleText, clears stale replayInvalid, and uses the intended terminal liveness contract.
