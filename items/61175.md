## Summary

Keep this issue open. Current main still has the misleading dual-error path: a failed npm plugin install without a bundled fallback still attempts hook-pack installation and formats the plugin failure with `Also not a valid hook pack: ...`. Recent maintainer discussion also explicitly kept this as the broader tracker and closed #69181 into it.

## Best Possible Solution

Keep this issue open and fix the CLI install routing. The best implementation is to make hook-pack fallback conditional on a clear hook-pack/non-plugin classification, while reporting plugin security-policy failures and valid-plugin validation failures as plugin errors only. Add a regression in `src/cli/plugins-cli.install.test.ts` for an npm plugin security-block failure without `--dangerously-force-unsafe-install` so `installHooksFromNpmSpec()` is not called and `Also not a valid hook pack` is absent.
