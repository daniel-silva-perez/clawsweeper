## Summary

Keep open. Current main has useful partial fixes, including immutable active disable windows and direct `models auth login` cleanup, but the central reported recovery path remains: billing failures still persist as long `disabledUntil` windows, same-provider fallback chains still prevent the single-provider billing probe path until near expiry, and `openclaw configure --section model` still re-authenticates through the plugin auth-choice path without clearing stale cooldown/disabled usage state.

## Best Possible Solution

Keep this issue open and finish the shared provider auth recovery work. Treat same-provider-only fallback chains that share disabled profiles as lacking a real fallback for throttled billing recovery probes, and make successful provider re-auth through `openclaw configure --section model` clear stale cooldown/disabled usage state for the affected provider/profile. Add regression coverage for Anthropic same-profile Opus fallback billing recovery and GitHub Copilot configure re-auth clearing a `format` cooldown.
