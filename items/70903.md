## Summary

Keep open. Current main contains partial mitigations, but it still does not solve the reported recovery path for a billing-disabled provider with same-provider fallback candidates. Billing disables are still persisted with a default 5h base and 24h cap, same-provider billing fallbacks are skipped until near expiry, and `openclaw configure --section model` still uses an auth-choice path that upserts credentials without clearing stale profile state.

## Best Possible Solution

Keep this issue open and implement a focused recovery path for persisted provider lockouts: either clear affected auth-state entries after successful provider re-auth in the configure/auth-choice path, add a safe health/probe path for billing-disabled same-provider fallback chains, or provide a documented first-class reset command. Add regression tests for the exact Anthropic same-profile fallback case and the `openclaw configure --section model` GitHub Copilot re-auth case from the issue comment.
