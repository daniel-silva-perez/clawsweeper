## Summary

Current `main` does not implement outbound secret-value sanitization before channel delivery. The only delivery-time sanitization I found is HTML/plain-text cleanup, and there is no `messages.sanitizeSecrets` config surface or matching implementation on the checked-out `main` SHA.

## Evidence

- **Outbound delivery only runs plain-text formatting sanitization:** `normalizePayloadsForChannelDelivery` applies `handler.sanitizeText(...)` when present, but that hook is for channel text cleanup and there is no secret-handle replacement step in this path.
  - file: [src/infra/outbound/deliver.ts:382](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/infra/outbound/deliver.ts#L382)
