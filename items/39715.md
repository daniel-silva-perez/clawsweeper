## Summary

Keep open. Current main still does not implement PR #39715's central behavior: there is no post-wait stdout/stderr drain in the supervisor or child adapter, and exec still snapshots aggregated output immediately when the managed run resolves. The PR discussion remains relevant because the author addressed multiple review-found ordering/cancel races on the branch, but those changes are not present on main.

## Best Possible Solution

Keep this PR open. The best path is to land or adapt a minimal supervisor/child wait drain with the settled/cancel/runId-reuse guards already discussed on the PR, plus regression coverage for backgrounded stdout/stderr flushed at process exit. Close only if a maintainer verifies current main no longer loses block-buffered output in the reported sandbox/background exec path.
