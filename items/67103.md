## Summary

Keep this PR open. Current main still does not implement Control UI LaTeX/KaTeX/MathJax rendering, the canonical feature request #42840 remains open, and this PR is a plausible implementation candidate rather than obsolete cleanup. It is not mergeable as-is because the PR head removes existing markdown helper definitions while leaving their call sites, weakens link-hardening behavior, adds broad style allowlisting, and has delimiter false-positive gaps called out in review.

## Best Possible Solution

Keep this PR open as a rework candidate for #42840. The best path is to update it against current main, preserve the existing markdown cache and DOMPurify link-hardening hooks, add robust delimiter parsing that avoids currency and code-span/code-fence false positives, narrow any KaTeX-required style allowance, update the lockfile for the new dependency, and add focused tests for inline math, display math, invalid LaTeX, mixed markdown, currency text, and backtick-delimited code. If the author does not continue, a successor PR can take over the same shared Control UI markdown-renderer approach.

## Risks / Open Questions

- The PR should not merge until the missing markdown helpers are restored or the call sites are rewritten; current PR head would break rendering and remove existing DOMPurify link hardening.
- The PR broadens sanitized style attributes globally and adds a new third-party dependency; maintainers should require narrow KaTeX styling and a proper pnpm-lock.yaml update/dependency review before merge.
- The delimiter parser needs stronger coverage for normal currency text, single- and multi-backtick code spans, code fences, invalid LaTeX, display math, and mixed markdown before this becomes a safe Control UI default.
