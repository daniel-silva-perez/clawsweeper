## Summary

Keep open. Current main still suppresses text-only tool summaries in non-forum group chats, and there is no channel/default config key that lets a user opt back in. The existing tests explicitly codify suppression for normal groups and delivery only for DMs, Telegram forum topics, media-only tool payloads, and exec-approval payloads.

## Best Possible Solution

Keep the issue open and implement a narrowly scoped opt-in delivery policy: add a typed/schema-backed config field for channel defaults and provider group/topic overrides, default it to false, feed it into dispatch-from-config's shouldSendToolSummaries/verbose-progress gate, and add tests proving non-forum groups remain silent by default but deliver text summaries when explicitly enabled. Update public docs to warn that this should only be enabled in trusted groups.

## Risks / Open Questions

- Any implementation should stay opt-in and clearly documented because /verbose can expose tool output, internal diagnostics, paths, commands, and potentially sensitive data in shared chats.
- The option needs schema/type/docs/test coverage across channel defaults and per-group/per-topic or per-channel scopes so unknown strict provider config is not silently rejected or inconsistently honored.
