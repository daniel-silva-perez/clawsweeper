## Summary

Keep this issue open. Current main still suppresses text-only tool summaries in ordinary group chats and does not expose a typed config opt-in under channel defaults, Telegram groups/topics, or WhatsApp groups. The related #44027 is still only an open implementation candidate and covers `/verbose` behavior rather than the requested config policy.

## Best Possible Solution

Keep this issue open. The best path is a narrow, typed, schema-backed opt-in such as `channels.defaults.allowVerboseToolResults` plus provider/group/topic overrides, defaulting to false. Feed that policy into `dispatch-from-config` and ACP projection gates, add regression tests proving ordinary groups stay silent by default but deliver text summaries when explicitly enabled, preserve forum/media/exec-approval behavior, update public docs with a trusted-group warning, and coordinate the behavior with #44027.

## Risks / Open Questions

- Verbose tool output can expose commands, file paths, URLs, diagnostics, and tool-returned data in shared chats; any implementation should stay explicit opt-in and documented for trusted groups only.
- The config-based policy requested here should be reconciled with #44027 so OpenClaw does not gain two inconsistent group tool-result delivery mechanisms.
