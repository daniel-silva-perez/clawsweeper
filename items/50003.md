## Summary

Keep open. Current main still does not provide `openclaw onboard --non-interactive --github-copilot-token` or an equivalent non-interactive GitHub Copilot onboarding hook. The original silent no-op appears to have become an explicit interactive-mode error, but the central requested capability remains useful and unshipped. The PR is stale/conflicted against the current file layout and needs rework, not cleanup closure.

## Best Possible Solution

Keep the PR open, but ask for a rebase and a smaller current-main implementation: add GitHub Copilot token flag metadata to `extensions/github-copilot/openclaw.plugin.json`, implement `runNonInteractive` on the bundled GitHub Copilot auth method using the current plugin auth contracts, preserve existing profile/default-model behavior, and add focused tests under the current `src/commands/onboard-non-interactive/local/*` and GitHub Copilot plugin test layout. Do not carry forward the obsolete generated metadata file change.
