## Summary

Current main still contains the reported broad sandbox env-var value heuristic. `validateEnvVarValue` warns on any 80+ character string made only of base64 characters, `sanitizeEnvVars` still calls it without key-name context, and the existing tests still assert the warning-positive path for a non-credential key rather than covering the false-positive cases. Related fix PRs in the provided context were closed unmerged, so they do not supersede the remaining work.

## Best Possible Solution

Keep this issue open until the sandbox env-var heuristic is tightened in `src/agents/sandbox/sanitize-env-vars.ts` and covered by regression tests. A conservative fix should either require stronger base64 structure or combine value heuristics with credential-like key names while preserving the existing explicit blocked-key protections and documenting any user-visible behavior change if needed.
