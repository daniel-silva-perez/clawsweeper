## Summary

Current main still matches #71646. `OpenClawChannelBridge` keeps `pendingClaudePermissions` and `pendingApprovals` as unbounded instance Maps with no TTL/sweeper, no close cleanup, and no cap; cleanup still happens only on matching permission replies or approval-resolved events. The linked fix PR #71648 remains the active path rather than evidence that main is fixed.

## Best Possible Solution

Keep #71646 open until #71648 or an equivalent maintainer-approved fix lands on main. The fix should clear both pending Maps on bridge close, expire stale Claude permission and approval entries using the existing approval expiry metadata where available, preserve the MCP protocol surface, and add focused regression tests. If cap/FIFO burst protection is not included in the same patch, track it explicitly as a follow-up rather than closing the issue as fully resolved.
