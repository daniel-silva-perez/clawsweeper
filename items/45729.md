## Summary

Keep open. Current `main` does not implement a Brave web-search `baseUrl` override: the Brave plugin schema/docs only expose `apiKey` and `mode`, and the runtime still hardcodes Brave’s default endpoints. This differs from Perplexity and Kimi, which already expose `webSearch.baseUrl` overrides.

## Evidence

- **Brave runtime still hardcodes Brave endpoints:** `BRAVE_SEARCH_ENDPOINT` and `BRAVE_LLM_CONTEXT_ENDPOINT` are fixed constants, and both request paths are built directly from those constants via `new URL(...)` with no config override path.
  - file: [extensions/brave/src/brave-web-search-provider.runtime.ts:30](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/extensions/brave/src/brave-web-search-provider.runtime.ts#L30)

  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)

  - file: [extensions/brave/src/brave-web-search-provider.shared.ts:7](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/extensions/brave/src/brave-web-search-provider.shared.ts#L7)
