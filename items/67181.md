## Summary

Keep open. Current main still constructs and sends the exact user-visible resume-fallback banner on exec approval followup session-resume failure, including for successful `code 0` command output. The issue discussion’s Telegram confirmation matches the same channel-generic fallback path, so this is not Discord-only and is not implemented away.

## Best Possible Solution

Keep this issue open and fix `src/agents/bash-tools.exec-approval-followup.ts`: for successful command results, direct fallback should send only the sanitized result or neutral user copy, while the session-resume failure reason stays in logs. Update `src/agents/bash-tools.exec-approval-followup.test.ts` so the successful `code 0` fallback no longer expects the internal prefix, and coordinate with the related #72143/#72148 work if that PR is still active.
