## Summary

Current main still enforces the requested 16k context-window minimum as a hard block. The runtime guard exports a 16,000-token hard minimum, `resolveEffectiveRuntimeModel()` throws when a model is below it, custom-provider onboarding clamps smaller windows up to that minimum, tests assert the blocking behavior, and local-model docs still describe below-16k as blocked.

## Best Possible Solution

Keep this issue open for maintainer product judgment. A complete implementation should change the context-window guard from a hard sub-16k block to warning-only behavior or an explicit override, update the embedded runtime path, decide how custom-provider onboarding should preserve smaller configured windows, and revise the guard/onboarding tests plus local-model docs so the behavior is consistent.
