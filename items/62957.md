## Summary

Keep #62957 open. Current main still enforces the requested 16k context-window hard minimum: the guard defines 16,000 as the hard floor, runtime setup throws a FailoverError when a resolved model is below it, tests assert the block behavior, onboarding clamps custom model metadata up to the hard minimum, and docs still tell users OpenClaw blocks below 16k.

## Best Possible Solution

Keep the issue open for maintainer product judgment. A complete fix would replace the runtime hard block with warning-only behavior or an explicit opt-in override, then update `src/agents/context-window-guard.ts`, `src/agents/pi-embedded-runner/run/setup.ts`, custom-provider onboarding normalization, the guard/onboarding tests, and local-model docs so the contract consistently permits sub-16k experimentation while warning about limitations.
