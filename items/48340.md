## Summary

Keep open. Current `main` has substantial Feishu channel support, but I did not find evidence that OpenClaw now does automatic two-way Feishu <-> TUI mirroring. The TUI is still session-scoped and provider delivery is still opt-in, so this remains a plausible product gap rather than an issue already implemented on `main`.

## Evidence

- **TUI defaults to one session, not a cross-surface inbox:** `resolveTuiSessionKey()` defaults the TUI to the current agent's main session when no session is specified, rather than subscribing to all channel conversations.
  - file: [src/tui/tui.ts:138](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/src/tui/tui.ts#L138)

  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **TUI history loading is per-session only:** Gateway-mode TUI history requests call `chat.history` with a single `sessionKey`, which matches the session-scoped model in the docs and code.
  - file: [src/tui/gateway-chat.ts:192](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/src/tui/gateway-chat.ts#L192)
