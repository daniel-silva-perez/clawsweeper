## Summary

Current `main` does not show the requested TUI multiline-compose feature as implemented. The docs still say `Enter` sends, the runtime still wires Enter submit directly, and there is no configured `Shift+Enter`/`Ctrl+Enter` newline path or documented/configurable enter-send toggle.

## Evidence

- **Docs still list only Enter-to-send:** The TUI keyboard shortcuts section documents `Enter: send message` and does not mention any multiline compose shortcut, config toggle, or help/footer hint for newline insertion.
  - file: [docs/web/tui.md:86](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/docs/web/tui.md#L86)
- **Runtime still binds editor submit directly:** `runTui` creates the submit handler and assigns it to `editor.onSubmit`; nearby keybindings wire Esc/Ctrl shortcuts only. There is no assignment for `onAltEnter`, `Shift+Enter`, or a configurable send/newline mode here.
  - file: [src/tui/tui.ts:953](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/src/tui/tui.ts#L953)
  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
- **Custom editor has no Shift+Enter handling:** `CustomEditor.handleInput` special-cases Alt+Enter, Alt+Up, Ctrl+L/O/P/G/T, Shift+Tab, Esc, Ctrl+C, and Ctrl+D, then falls through to the base editor. There is no `Key.shift("enter")` handling or enter-send configuration surface.
  - file: [src/tui/components/custom-editor.ts:16](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/src/tui/components/custom-editor.ts#L16)
  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
- **Existing tests cover pasted/submitted multiline text, not interactive newline keybinds:** The submit-handler tests verify preserving internal newlines once multiline text already exists and verify burst-coalescing for pasted rapid submits, but they do not cover Shift+Enter/Ctrl+Enter composition behavior.
  - file: [src/tui/tui.submit-handler.test.ts:49](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/src/tui/tui.submit-handler.test.ts#L49)
  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
- **Unused Alt+Enter hook:** A repository search found no `onAltEnter =` assignment under `src/tui`, so even the existing Alt+Enter hook exposed by `CustomEditor` is not currently connected to newline insertion or any other behavior.
  - command: `rg -n "onAltEnter\s*=" src/tui`
