## Summary

Keep open. Current `main` does not implement the requested generic per-action, per-sender authorization for the `message` tool. The repo documents shared multi-user messaging as outside OpenClaw’s hostile multi-tenant trust model, but the actual code only hard-blocks a small owner-only tool set (`cron`, `gateway`, `nodes`) and otherwise offers coarse whole-tool allow/deny controls or plugin-specific checks. The provided issue timeline also links an open PR (#64868), which is further evidence that this has not landed on `main` yet.

## Evidence

- **Security docs explicitly describe this as an existing shared-agent risk:** The security guide says OpenClaw is not a hostile multi-tenant boundary, that multiple untrusted users messaging one agent share that agent’s delegated tool authority, and specifically calls out shared Slack workspaces as allowing any allowed sender to induce tool calls within the agent’s policy.
