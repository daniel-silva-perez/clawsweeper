## Summary

Keep this PR open. Current main still does not add `--daemon-env` to `openclaw onboard --install-daemon`, and the PR is a focused implementation candidate for the closed linked feature request. It should not be cleanup-closed, but it does need maintainer/security review before merge because the proposed approach merges arbitrary env keys after the existing daemon environment filtering.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to either implement a narrower, reviewed opt-in for the known `OPENCLAW_ALLOW_INSECURE_PRIVATE_WS=1` bootstrap case, or thread `--daemon-env` through a shared daemon-install helper that normalizes portable env keys, warns or fails on malformed input, rejects dangerous host-env keys using `host-env-security`, and has regression coverage for CLI parsing plus both interactive and non-interactive install paths. If accepted, update `docs/cli/onboard.md` and `docs/reference/wizard.md` to document the supported flag and security limits.
