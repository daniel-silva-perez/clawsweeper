## Summary

Keep open. Current `main` added a generic `agents.defaults.memorySearch.outputDimensionality` knob and `/v1/embeddings` request-time `dimensions` handling, but the OpenAI memory-search provider still does not pass any dimension override into its actual embedding requests. Issue #17424 asks for configurable dimensions on memory search for OpenAI-compatible embeddings; that is not implemented end-to-end on current `main`.

## Evidence

- **Memory-search config now resolves an output dimension field:** `resolveMemorySearchConfig` reads `overrides?.outputDimensionality ?? defaults?.outputDimensionality` and returns it in the resolved memory settings, so there is a config surface, but under `outputDimensionality`, not the requested `dimensions` key.
  - file: [src/agents/memory-search.ts:195](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/src/agents/memory-search.ts#L195)

- **Public config/docs scope the knob to Gemini/Bedrock-style providers:** The type comment says `outputDimensionality` is for Gemini embedding-2 models, and the docs/help text describe Gemini and Bedrock families rather than OpenAI `text-embedding-3-*` memory-search usage.
  - file: [src/config/types.tools.ts:385](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/src/config/types.tools.ts#L385)
