## Summary

Current main still does not implement the MiniMax usage-host behavior this PR is trying to address. The original GET-method bug from the linked report is fixed, but MiniMax usage polling still hardcodes the CN host and ignores configured global/CN baseUrl context. The overlapping #65059/#65054 work is still open and unmerged, so it is useful consolidation context rather than a shipped or clearly canonical superseder.

## Best Possible Solution

Keep this PR open for maintainer review or consolidation with #65054/#65059. The best final fix is a MiniMax-owned, region-aware usage host resolver that derives the endpoint from the same credential/config source as the usage token, preserves `GET /v1/api/openplatform/coding_plan/remains`, supports global and CN baseUrls, handles invalid/custom baseUrl safely under the current URL trust policy, and adds regression tests for URL, method, global, CN, invalid baseUrl, and mixed `minimax`/`minimax-portal` configurations.
