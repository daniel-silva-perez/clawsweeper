## Summary

Keep #53663 open. Current main still does not contain the PR's autofill-resistant attributes on the two skills filter inputs, and the proposed focused browser coverage is not present. The linked issue #53648 was closed after confirming the filters are controlled by empty skills state, but this PR's narrower browser-autofill hardening remains unlanded and still plausibly useful.

## Best Possible Solution

Keep the PR open or ask for a refreshed patch: rebase onto current main, apply the autofill-resistant search attributes to the installed Skills and per-agent Skills filters, consider whether the current ClawHub search should receive the same attributes, and update/add focused browser tests against the current props shape.
