## Summary

Keep open. Current `main` has durable background work surfaces for cron, heartbeat, and subagents, but provider batch APIs are only wired for memory embedding jobs, not for background agent turns. The request in #9865 is therefore not implemented on `main` and is still a valid enhancement.

## Evidence

- **OpenAI capability table does not claim batch/background agent-turn support:** The provider docs list supported OpenAI surfaces as chat/responses, images, videos, TTS, batch speech-to-text, realtime voice, and embeddings. There is no batch API support listed for cron, heartbeat, subagents, or general background agent execution.
