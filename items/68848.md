## Summary

Keep this PR open. Current main still has the reported no-registration wake-state leak path, and this PR is the focused implementation candidate paired with open issue #68847 by the same contributor.

## Best Possible Solution

Keep this PR open as the active fix candidate for #68847. The likely product fix belongs in `src/gateway/server-methods/nodes.ts`: clear the speculative `nodeWakeById` entry on the `!registration` early return while preserving concurrent `state.inFlight` coalescing, with a narrow regression test proving unregistered node ids do not accumulate wake-state entries.
