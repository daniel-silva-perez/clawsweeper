## Summary

Current `main` is not safe to close this issue. Anthropic's main streaming path now distinguishes OAuth tokens and uses `Authorization: Bearer`, but the live native-PDF/media-understanding path still routes Anthropic auth through `src/agents/tools/pdf-native-providers.ts`, which unconditionally sends `x-api-key`. That matches the issue follow-up about direct HTTP Anthropic callsites, so this is not fully implemented on `main` and not obsolete enough for `cannot_reproduce`.
