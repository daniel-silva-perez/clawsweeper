## Summary

Keep open. Current main still routes MiniMax through the generic hybrid Anthropic/OpenAI replay hooks, which produce a strict Anthropic replay policy with `sanitizeToolCallIds: true`; the MiniMax-specific opt-out proposed by this PR is not present, and current tests still assert the old behavior. The PR discussion is concrete and relevant: it reopens an earlier API-drifted attempt and ties the change to MiniMax error 2013 while preserving OpenAI-compatible sanitization.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this provider-local wrapper, or an equivalent MiniMax-owned replay hook, so `minimax` and `minimax-portal` disable tool-call ID sanitization only on `anthropic-messages` while preserving strict sanitization for OpenAI-compatible paths. A broader SDK option for hybrid replay families can be a follow-up only if more providers need the same knob.
