## Summary

Keep this PR open. Current main still has the MiniMax web-search metadata/onboarding mismatch the PR targets: runtime can use MINIMAX_API_KEY, but web-search provider metadata, setup visibility, docs, and tests still encode the Coding Plan env vars only. The PR is focused and useful, but the existing review concern about false-positive auto-detection for generic MiniMax API keys remains a real product tradeoff to resolve before merge.

## Best Possible Solution

Keep this PR open for maintainer review or author revision. The best path is to preserve the useful MiniMax metadata/docs/onboarding alignment, add focused coverage for MiniMax Search setup/auto-detection, and explicitly resolve the Token Plan tradeoff before merge, for example with maintainer approval for this auto-detection behavior, lower precedence, explicit configured-key support without blind env auto-detection, or a search-capability check if MiniMax exposes one.
