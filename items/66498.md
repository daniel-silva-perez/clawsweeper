## Summary

Current main still does not implement Gemini `web_search` freshness. The bundled Google plugin marks `freshness`, `date_after`, and `date_before` as unsupported, the runtime returns an unsupported-filter response before building the Gemini request, and the outgoing Gemini request still contains only `tools: [{ google_search: {} }]` with no time-range payload. The existing discussion correctly treats #17925 as related but broader, not a canonical replacement for this narrower provider-argument gap.

## Best Possible Solution

Keep this issue open and implement the feature in the bundled Google plugin, not core. Verify the Google API contract, normalize OpenClaw freshness values in `extensions/google/src/gemini-web-search-provider.runtime.ts`, pass the resulting time range into the Gemini Google Search request, include it in the cache key, update `extensions/google/src/gemini-web-search-provider.ts` and `docs/tools/gemini-search.md`, and add focused Google web-search tests for supported freshness values and unsupported filter combinations.
