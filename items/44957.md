## Summary

Keep open. In the checked-out `main` at `4630ce3d9e2f089062ccd386b0ae241325961d1a`, isolated cron runs still start fresh each time, and the codebase has no `outputHistory`/`recentOutputs` support in cron payloads, state, prompt assembly, or gateway schema. This PR is not implemented on the reviewed checkout.

## Evidence

- **Docs still describe isolated cron runs as fresh and direct history use cases to custom sessions:** The current docs say isolated jobs run with a fresh session, while `session:custom-id` is the execution style for workflows that build on history.
  - file: [docs/automation/cron-jobs.md:87](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/docs/automation/cron-jobs.md#L87)

- **Cron types have no output-history fields:** `CronAgentTurnPayloadFields` ends at `toolsAllow`, and `CronJobState` ends at delivery state fields; there is no `outputHistory` flag or `recentOutputs` state on current main.
  - file: [src/cron/types.ts:116](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/src/cron/types.ts#L116)
