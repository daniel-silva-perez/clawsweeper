## Summary

Keep #71482 open. Current main still has the data-loss-shaped path the PR describes: `loadSessionStore()` collapses missing/invalid stores to `{}`, while `scrubDreamingNarrativeArtifacts()` treats an empty reference set as authoritative before archiving stale dreaming transcripts. The PR is not cleanup-closeable, but its exact guard also needs revision because review comments correctly flag that it would disable legitimate orphan cleanup for truly empty stores.

## Best Possible Solution

Keep the PR open, but ask for a tighter fix: make this cleanup path distinguish session-store read/parse failure from a valid empty store, either by exposing/throwing load status for this caller or by validating the store file before destructive archival. Add focused tests for malformed or unreadable `sessions.json` with old dreaming transcripts, plus a genuinely empty store where stale dreaming orphans should still archive.
