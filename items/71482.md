## Summary

Keep #71482 open. Current main still has the reported risk path: `loadSessionStore()` suppresses read/parse failures and returns `{}`, while `scrubDreamingNarrativeArtifacts()` treats an empty store as an empty reference set and may archive old dreaming `.jsonl` transcripts that are not in that set. The PR is focused on the right code path, but the proposed empty-store guard is over-broad and the existing review comments correctly note that it would also disable legitimate orphan cleanup for truly empty stores. This should be revised rather than closed or merged as-is.

## Best Possible Solution

Keep the PR open for revision. The maintainer-quality fix should distinguish failed/untrusted session-store reads from a real empty `sessions.json`, skip destructive dreaming orphan archival only when the store could not be read reliably, preserve cleanup for legitimate empty stores, and add focused memory-core tests for both cases.

## Risks / Open Questions

- Merging the PR as written would trade the data-loss path for a cleanup regression on agents whose store is legitimately empty but whose sessions directory still contains stale dreaming transcripts.
- The correct fix likely needs a read-status/error distinction around session-store loading, plus regression tests for both corrupt/unreadable stores and legitimate empty stores.
