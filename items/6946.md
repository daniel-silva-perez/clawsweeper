## Summary

Current main still does not implement the requested Telegram processing-indicator message lifecycle. It has adjacent feedback mechanisms, but no configurable standalone message that is sent immediately on inbound receipt, deleted on final reply, or edited on failure.

## Best Possible Solution

Keep the issue open. The best path is an opt-in implementation in the bundled Telegram plugin: add schema/types/docs/generated metadata, send a standalone placeholder immediately after an eligible inbound Telegram message is accepted, carry the sent message id through dispatch, delete it after successful final reply delivery, and edit or replace it on dispatch failure. Use the native-command/preview cleanup code and closed PRs #43021/#43720 as prior art, but implement fresh against current main with Telegram dispatch tests for success, error, preview-streaming, and group/topic behavior.
