## Summary

Keep open. Current main still has the unguarded `WebSocketTaskBox.sendPing()` continuation that PR #51311 targets, the keepalive loop still exercises that path, and the latest release snapshot checked from the provided context still lacks the guard. Related PRs #41653 and #51371 appear to cover the same bug, but no canonical fix has landed or been selected, so this focused crash fix should not be auto-closed as implemented or obsolete.

## Best Possible Solution

Keep this PR open until a maintainer selects or lands a canonical fix for the `WebSocketTaskBox.sendPing()` double-resume crash. The best path is a small `GatewayChannel.swift` guard that imports the correct `os` module, guarantees at-most-once continuation resume, and adds a duplicate pong-handler regression test; then close #51311, #41653, and #51371 duplicates against the landed canonical PR as appropriate.
