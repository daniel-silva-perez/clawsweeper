## Summary

Keep open. Current `main` does not implement a configurable Brave Search base URL: the bundled Brave provider still hardcodes Brave endpoints, its strict plugin config schema only allows `apiKey` and `mode`, and the docs expose only those settings. The original PR also targets pre-plugin-architecture files, so it would need a port rather than a straight merge.

## Evidence

- **Brave runtime still hardcodes Brave endpoints:** `extensions/brave/src/brave-web-search-provider.runtime.ts` defines fixed `BRAVE_SEARCH_ENDPOINT` and `BRAVE_LLM_CONTEXT_ENDPOINT` constants pointing at `https://api.search.brave.com/...`, so there is no configurable base URL on current `main`.

## Risks / Open Questions

- Any eventual fix now needs to be implemented in the bundled Brave plugin (`extensions/brave/**`) and its manifest/docs, not the older `src/agents/tools/web-search.ts` path used by the PR.
- If a Brave `baseUrl` override is added later, the cache key should include the effective base URL to avoid stale cross-endpoint cache hits.
