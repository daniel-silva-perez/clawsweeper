## Summary

Keep this PR open. Current main still uses a traversal-wide WeakSet in diagnostic payload redaction, so acyclic shared sibling objects are still treated as `[Circular]`. The PR targets the implicated code path and adds regression coverage, but it needs maintainer review/rebase because the provided patch references a non-exported sanitizer helper and its inline writer fixture is missing the current `flush` method. Security review found no unrelated CI, dependency, lockfile, package, or script changes; the only security-sensitive surface is the intended diagnostic redaction logic itself.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land the active-recursion-path plus `WeakMap` memo fix in `src/agents/payload-redaction.ts`, add focused regression coverage for shared sibling references and real cycles, and reconcile the test patch with current main’s exported sanitizer name and `QueuedFileWriter.flush` contract. Separately decide whether cache-trace digest `stableStringify()` should receive equivalent active-path semantics.
