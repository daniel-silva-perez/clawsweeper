## Summary

Keep this PR open. Current main still uses a traversal-wide `WeakSet` in diagnostic payload redaction, so a shared sibling object is still classified as `[Circular]` after the first branch. The PR addresses a real remaining bug and the discussion improved it by adding memoization after review feedback, but it likely needs maintainer review/rebase against current main before landing.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land the active-recursion-path plus `WeakMap` memo fix in `src/agents/payload-redaction.ts`, add focused regression coverage for shared sibling references and real cycles, then reconcile the test patch with current main’s exported sanitizer name and `QueuedFileWriter.flush` requirement. During review, also decide whether cache-trace digest `stableStringify()` should receive the same active-path treatment or remain a separate follow-up.
