## Summary

Keep PR #71671 open. Current main still has the reported no-progress streak bug: the helper skips over intervening non-matching or missing-result records instead of treating them as a tail break, and the resulting count feeds both known polling and global circuit-breaker blocking. The PR is a focused bug fix with regression tests and is not clearly superseded by the broader stale-history PR #40217.

## Best Possible Solution

Keep #71671 open for maintainer review. The best path is to rebase it on current main, preserve the small `getNoProgressStreak` contiguous-tail fix, keep both regression tests, and run the changed checks. If maintainers continue the broader #40217 stale-history reset, this narrower fix should either land first or be explicitly incorporated there rather than closed as cleanup.
