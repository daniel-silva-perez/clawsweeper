## Summary

Current `main` already contains multiple shipped fixes and regression tests covering Telegram voice-note transcription paths that previously produced raw `<media:audio>` placeholders or failed to load OpenAI STT providers after runtime/plugin reloads.

## Evidence

- **Telegram now substitutes a preflight transcript for voice-note placeholders:** `resolveTelegramBody()` runs `transcribeFirstAudio(...)` for audio-only Telegram messages and replaces `<media:audio>` with the transcript when one is available.
  - file: [extensions/telegram/src/bot-message-context.body.ts:200](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/extensions/telegram/src/bot-message-context.body.ts#L200)
- **Regression test covers DM voice-note placeholder symptom:** Current test asserts Telegram DM voice notes are preflight-transcribed and no longer arrive as raw `<media:audio>` placeholders.
  - file: [extensions/telegram/src/bot-message-context.body.test.ts:149](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/extensions/telegram/src/bot-message-context.body.test.ts#L149)
- **Bundled media providers are re-added when `plugins.allow` is set:** Capability-provider fallback now merges bundled capability plugin ids into the effective config before loading provider registries, which prevents bundled media-understanding providers like OpenAI STT from disappearing after registry rebuilds.
  - file: [src/plugins/capability-provider-runtime.ts:45](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/plugins/capability-provider-runtime.ts#L45)
- **Allowlist fallback has a dedicated media-understanding regression test:** The allowlist regression test exercises the media-understanding provider registry fallback path for bundled providers.
  - file: [src/media-understanding/provider-registry.allowlist.test.ts:20](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/media-understanding/provider-registry.allowlist.test.ts#L20)
- **Shipped release notes match the reported failure mode:** `CHANGELOG.md` records: Telegram DM voice-note preflight transcription restored (`2026.4.5`), bundled media providers merged into `plugins.allow` for voice transcription (`2026.4.7`), and OpenAI audio multipart/model fields plus provider-failure surfacing fixed (`2026.4.12`).
  - file: [CHANGELOG.md:1242](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/CHANGELOG.md#L1242)
