## Summary

Keep open. Current main still lacks structured Matrix approval event metadata: Matrix pending approvals are built as text plus allowed reaction decisions and delivered through the chunking `sendMessageMatrix` path. This PR is an active contributor implementation candidate, not obsolete or implemented on main, and the provided discussion shows review feedback plus a follow-up commit aimed at the delivery fallback concerns.

## Best Possible Solution

Keep this PR open and review it as a Matrix plugin change. The best path is to add focused `approval-handler.runtime` tests for the metadata shape, single-message event content, fallback to `sendMessageMatrix` when the Matrix single-event limit is exceeded, and dependency-injected senders; then land or request revisions based on that validation and the intended native Matrix client contract.
