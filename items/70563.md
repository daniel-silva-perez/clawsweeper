## Summary

Keep this issue open. Current main still has the Kimi Coding failure shape: native `sessions_spawn` can forward a thinking override into the child run, but the Kimi provider wrapper only toggles the top-level `thinking` payload and does not preserve or synthesize `reasoning_content` for replayed assistant tool-call turns. Related generic Anthropic/subagent thinking fixes and DeepSeek/Venice replay fixes do not cover `extensions/kimi-coding`.

## Best Possible Solution

Keep this issue open and fix the provider-owned replay path in `extensions/kimi-coding`. When Kimi thinking is enabled, preserve existing provider reasoning metadata or add a Kimi-accepted `reasoning_content` placeholder for replayed assistant tool-call messages, then add focused regression coverage for thinking-enabled `sessions_spawn` plus Kimi tool-call continuation. Cross-check #50178 for any reusable provider seam, but do not close this Kimi-specific report as implemented or duplicated yet.
