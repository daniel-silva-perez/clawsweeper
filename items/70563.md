## Summary

Keep #70563 open. Current main has fixes for adjacent thinking replay cases, including latest assistant thinking-block preservation and DeepSeek/Venice `reasoning_content` placeholders, but I did not find an equivalent Kimi Coding fix. The bundled Kimi provider still just toggles `thinking`, deletes generic reasoning fields, and has no provider replay/payload patch that adds missing `reasoning_content` to assistant tool-call replay messages. The related closed #35023 covers Anthropic/Copilot-style thinking-block replay, not this Kimi `reasoning_content` failure.

## Best Possible Solution

Keep this issue open and implement a Kimi-owned fix after verifying the current Kimi Coding API contract. The likely code path is a provider replay/stream wrapper for `extensions/kimi-coding` that either preserves or synthesizes `reasoning_content` on assistant tool-call replay when thinking is enabled, or explicitly disables unsupported thinking/tool-call combinations. Add focused regression coverage for Kimi thinking plus `sessions_spawn`/tool-call continuation and update the Kimi docs if users need a specific `thinking` setting.
