## Summary

Keep issue #63471 open. Current main still requires either a pre-existing `gateway.mode=local` config or `--allow-unconfigured`; the bundled Docker Compose command does not pass that escape hatch, so the reported Dockge/raw-Compose first-run failure remains a valid install UX gap.

## Best Possible Solution

Keep this open for a focused Docker install UX fix. The best path is to make the supported Dockge/raw-Compose first-run surface self-contained, either by adding a safe bootstrap command/flag to the bundled Compose path or by seeding the minimal `gateway.mode=local` config before gateway startup, then updating `docker-compose.yml`, `.env.example`, Docker docs, and targeted startup/Compose tests. Do not move this to ClawHub; it is core repository Docker/install behavior, not an optional plugin capability.
