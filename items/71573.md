## Summary

Keep PR #71573 open. Current main still lacks the update.run continuation path the PR is trying to add: the protocol schema rejects continuationMessage, update.run writes a restart sentinel without continuation, the agent gateway tool does not forward continuationMessage on update.run, and current tests explicitly assert that update.run sentinel continuations are undefined. The existing restart sentinel consumer/helper already supports agentTurn continuations, so the PR is a plausible targeted fix for the linked #71178 rather than obsolete or superseded.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to finish/land a focused fix that adds continuationMessage to the update.run protocol contract, parser, server sentinel payload, agent gateway tool forwarding, generated Swift models, and update.run regression tests, while explicitly resolving or accepting the bot-reported continuation size/session authorization concerns against the existing restart continuation contract.
