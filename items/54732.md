## Summary

Keep open. Current main still scopes the message tool action enum and plugin-contributed schema properties, but the shared/core message schema sections are still added unconditionally. The requested action-aware trimming of moderation, presence, thread, event, sticker, gateway, channel-management, and shared poll fields remains unimplemented.

## Best Possible Solution

Implement the requested action-aware core schema trimming in `src/agents/tools/message-tool.ts`: pass the resolved action set into `buildMessageToolSchemaProps()`, map action families to their shared sub-schema sections, keep always-needed routing/send basics, and add focused schema tests proving basic-channel configs do not expose moderation/presence/thread/event/sticker/channel-management fields while channels with those actions still do.
