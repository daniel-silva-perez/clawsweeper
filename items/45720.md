## Summary

Keep open. Current `main` already implements much of the Telegram Local Bot API work (`channels.telegram.apiRoot`, custom Bot API roots for bot clients/probes/audit/downloads, and guarded absolute `file_path` reads via `trustedLocalFileRoots`), but it does not carry the feature through all username/chat lookup paths. In particular, the runtime target resolver and Telegram doctor username repair still call `lookupTelegramChatId(...)` without passing the configured account `apiRoot`, so the PR still contains material behavior not present on `main`.
