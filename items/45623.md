## Summary

Current `main` already addresses the reported gateway-startup failure mode: it binds the HTTP/WS listener before channel/plugin sidecars finish, keeps liveness separate from readiness, narrows Discord startup loading, and adds a client-side handshake-timeout mitigation.

## Evidence

- **Listener now binds before sidecars:** `startGatewayServer()` attaches WS handlers, then calls `startListening()` before post-attach sidecars start; the adjacent comment states HTTP is live before sidecars finish.
  - file: [src/gateway/server.impl.ts:808](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/gateway/server.impl.ts#L808)
  - command: `nl -ba src/gateway/server.impl.ts | sed -n '780,860p'`
- **Readiness stays red while listener stays live:** Readiness is driven by `getStartupPending: () => !startupSidecarsReady`, returning `startup-sidecars` until sidecars settle; `/readyz` returns 503 from that readiness result while `/healthz` remains a live probe.
  - file: [src/gateway/server/readiness.ts:48](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/gateway/server/readiness.ts#L48)
  - command: `nl -ba src/gateway/server/readiness.ts | sed -n '1,220p'`
- **Sidecars start asynchronously after bind:** Post-attach startup defers channel/plugin sidecars with `setImmediate(...)`, logs `starting channels and sidecars...`, and only clears startup-gated methods after sidecars are ready.
  - file: [src/gateway/server-startup-post-attach.ts:471](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/gateway/server-startup-post-attach.ts#L471)
  - command: `nl -ba src/gateway/server-startup-post-attach.ts | sed -n '384,520p'`
- **CLI handshake timeout mitigation exists on current main:** Gateway calls now yield one event-loop turn before opening the WebSocket so heavy synchronous module loading does not starve the `connect.challenge` handshake path and trigger timeout errors.
  - file: [src/gateway/call.ts:479](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/gateway/call.ts#L479)
  - command: `nl -ba src/gateway/call.ts | sed -n '430,540p'`

## Close Comment

Closing this as implemented on current `main` after a Codex review. The gateway now binds/listens before channel and plugin sidecars finish (`src/gateway/server.impl.ts:808`), keeps liveness separate from readiness so `/readyz` can stay red while the listener is already up (`src/gateway/server/readiness.ts:48`, `src/gateway/server-http.ts:325`), and the CLI connection path now yields once before opening the WebSocket to avoid handshake timeouts during heavy startup loading (`src/gateway/call.ts:479`). The changelog also records this startup fix in the `2026.4.10` release (`CHANGELOG.md:890`). If you still see a long pre-listen window on a newer build, please open a fresh issue with the current version, config, and startup trace logs.
