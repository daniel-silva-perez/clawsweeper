## Summary

Keep PR #39957 open. Current main still does not contain the PR's phase-independent suppression for `message_end` block replies on intermediate assistant turns with `stopReason: "toolUse"` and tool-call blocks. A newer commentary-phase suppression exists, but it only applies when the assistant message is explicitly marked `phase: "commentary"` or has matching text-signature metadata; it does not cover the unphased repro shape described in the PR and linked issue.

## Best Possible Solution

Keep this PR open and either merge/rebase the narrow `message_end && stopReason === "toolUse" && has tool-call block` suppression with regression coverage, or implement the same product behavior through the current display-phase projection. Preserve the PR's non-goals: normal `stop` turns, `toolUse` turns without tool-call blocks, and `text_end` block-reply flow should continue to emit as they do today.
