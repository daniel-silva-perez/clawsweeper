## Summary

Keep open. Current main does not implement the PR's requested `session_status` machine-readable timestamp. The shared status card only renders a time line when one is supplied, but the current `session_status` path delegates to `buildStatusText` without supplying one, and the current time helper work on main is limited to cron/session-reset/compaction prompt contexts rather than the status card surface. The PR likely needs a rebase onto the newer shared `src/status` status-text layout, not conservative closure.

## Best Possible Solution

Rebase/rework the PR against the current shared `src/status` status-text architecture. The likely clean fix is a shared status time-line helper, wired through `BuildStatusTextParams` or computed inside `buildStatusText`, with focused regression coverage proving `session_status` includes human-readable local time plus machine-readable UTC and, if desired, local ISO-with-offset. If `/status` shares the same card, cover that behavior deliberately too.
