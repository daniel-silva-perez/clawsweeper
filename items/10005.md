## Summary

Keep open. Current `main` still defines `session_status` as a `/status`-style card/text surface with only `sessionKey` and `model` parameters, and it always returns rendered status text in tool content. I found no config path like `session.statusCard` or `agents.defaults.statusCard`. There is a Control UI-only `chatShowToolCalls` toggle, but that just hides tool cards in that UI and does not add the requested config-driven JSON-only behavior for `/status` or `session_status`.
