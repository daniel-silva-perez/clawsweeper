## Summary

Keep PR #52643 open. Current main still does not expose `openclaw_version` or `bot_name` on plugin hook events, and there is no bot-name registry SDK entrypoint. The PR is stale and its approach needs rework, but the requested plugin API capability remains a real, unimplemented core/plugin-contract gap.

## Best Possible Solution

Keep the item open and ask for an updated PR that treats this as a plugin hook contract change: add typed version/bot identity metadata in a stable event or context shape, carry `accountId` through agent/session/tool/subagent hook contexts where available, use an account-aware Feishu identity source with cleanup and recovery coverage, and update SDK exports, tests, and `docs/plugins/hooks.md`. Split the Feishu probe response correction only if it still applies to the current `pingBotInfo` endpoint.
