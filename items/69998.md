## Summary

Keep #69998 open. Current main still returns the mutable `skills.update` config entry directly, so the linked credential-echo bug remains on main. The PR is a focused active fix, the duplicate PRs referenced in the discussion are closed unmerged, and the remaining maintainer concern is the redaction-sentinel round-trip behavior rather than obsolescence.

## Best Possible Solution

Keep #69998 open. The best path is to finish the PR by either restoring/keeping existing secret values when `skills.update` receives the redaction sentinel, or changing the success payload so it does not return writable secret-bearing config. Keep the exact-sentinel regression test, add round-trip coverage if sentinel restoration is chosen, then merge and close #66769.
