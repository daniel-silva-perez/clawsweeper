## Summary

Keep #69998 open. Current main still returns the config-mode `skills.update` entry directly after writing it, so the reported plaintext `apiKey`/secret `env` response leak is not implemented on main. The PR is a focused candidate fix for open issue #66769, and the related duplicate PRs #66928 and #66986 are closed unmerged rather than superseding it.

## Best Possible Solution

Keep #69998 open as the canonical PR for #66769. The best implementation path is a narrow gateway fix that redacts only the `skills.update` response with `redactConfigObject()`, preserves full secret values in the written config, keeps the exact-sentinel regression coverage for `apiKey` and secret-like `env` keys, and either handles `__OPENCLAW_REDACTED__` as keep-existing on `skills.update` writes or records a maintainer decision that this response is not a writable round-trip surface.
