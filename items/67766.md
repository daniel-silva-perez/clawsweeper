## Summary

Keep this PR open. Current main still fails closed when the selected provider/model is missing from the gateway model catalog, the linked #67759 bug remains open, and #67768 was already closed as superseded by this PR rather than replacing it.

## Best Possible Solution

Keep this PR open as the canonical candidate for #67759. The best path is maintainer review of the explicit-config fallback semantics, then either land a focused gateway fix with the existing alias-aware tests or request changes that preserve provider ownership while still respecting deliberate user model input metadata on catalog miss.
