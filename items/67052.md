## Summary

Keep open. Current main includes a 30s TUI streaming watchdog mitigation from #67401, but it does not implement the requested prompt transition when visible response content finishes. The TUI still marks chat deltas as `streaming` and clears that status only on chat final/lifecycle terminal handling or after the watchdog timeout, while the gateway still derives chat final from terminal lifecycle events that can wait for post-content flush work.

## Best Possible Solution

Keep this issue open until streaming UI state is decoupled from full lifecycle/chat-final completion. A good maintainer fix would add a lightweight content-complete or finalizing signal emitted when model-visible content is done, or update the TUI to move from `streaming` to a distinct non-streaming finalizing/idle state after visible deltas end while still tracking the active run. Add regression tests for delayed final/lifecycle events, concurrent active runs, tool-only or hidden deltas, and the existing watchdog fallback.

## Risks / Open Questions

- The existing watchdog reduces indefinite stuck-status impact, so a real fix should preserve its recovery role while avoiding a misleading immediate `idle` state if tools, delivery, or persistence are still legitimately active.
- A TUI-only silence heuristic could regress concurrent-run or hidden/tool-only output behavior; a backend content-complete/finalizing signal would be cleaner if it can be added compatibly.
