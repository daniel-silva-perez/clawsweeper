## Summary

Keep open. Current main has a 30s TUI streaming watchdog mitigation, but it still does not implement the requested prompt transition after the last visible content token. The footer remains driven by chat delta/final and lifecycle terminal events, so delayed lifecycle or chat-final delivery can still show `streaming` until the terminal event arrives or the watchdog fires.

## Best Possible Solution

Keep this issue open until the TUI streaming state is decoupled from full lifecycle/chat-final completion. A good maintainer fix would add a lightweight content-complete or finalizing signal from the run pipeline, or have the TUI transition from `streaming` to a distinct non-streaming finalizing/idle state after visible deltas end, with regression tests for delayed final events and concurrent active runs.
