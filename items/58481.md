## Summary

Keep this issue open. Current main still does not set OS-level TCP keepalive on the underlying WhatsApp WebSocket socket; it only configures Baileys, proxy agents, application/watchdog reconnect behavior, and docs for recovery after disconnects. The issue is recent, has a same-problem comment, and no merged or open PR was found that supersedes it.

## Best Possible Solution

Keep this issue open and implement the mitigation in the WhatsApp plugin socket setup, likely near `extensions/whatsapp/src/session.ts` or a small WhatsApp-local helper. The fix should enable TCP keepalive on the underlying Node socket with a conservative interval such as 15000 ms, avoid adding WhatsApp-specific behavior to core, preserve proxy-agent support, and add a focused unit test with a mocked Baileys/ws socket proving the keepalive call is applied.
