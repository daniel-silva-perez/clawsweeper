## Summary

Keep open. Current main partially addresses the requested error-specific behavior, but it does not implement the core requested 401 dead-key behavior: generic/ambiguous HTTP 401 still maps to `auth`, and `auth` goes through the ordinary cooldown lane, while only high-confidence `auth_permanent` and billing failures use the disabled lane. The issue is recent, concrete, reproducible, non-maintainer-authored, and not suitable for stale/clawhub/incoherent closure.
