## Summary

Keep open. Current main still lacks the wake-agent behavior: Slack reply button block actions are still treated as plain interaction events that enqueue a system event and update the Slack UI, while queued system events are drained during an agent run. This PR is a concrete implementation candidate, but it overlaps with open #61502 and #61374 and currently needs maintainer review/rebase rather than auto-close.

## Best Possible Solution

Keep this PR open for maintainer review. Reconcile it with #61502 and #61374, choose the safest Slack interaction wake design, rebase onto current main, and then validate the Slack extension behavior with targeted tests plus the changed gate before merge or superseding closure.
