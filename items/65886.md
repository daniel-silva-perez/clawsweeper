## Summary

Keep open. Current main still does not implement configurable TUI status-line command rendering, and the PR remains a plausible TUI/config feature or plugin-API design candidate. It is not cleanly closeable as ClawHub because the current plugin API does not expose a TUI render/status-area seam; the author’s linked petsonality patch is useful evidence of active demand, not a substitute for an OpenClaw-supported surface.

## Best Possible Solution

If accepted, land it as a core TUI/config change with updated types, strict Zod schema, generated schema/docs, focused lifecycle and validation tests, and explicit trusted-local-command documentation. If maintainers prefer externalization, first design a public TUI status/render plugin seam, then move command rendering into a plugin.
