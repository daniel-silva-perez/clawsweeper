## Summary

Keep open. Current `main` still defines the message tool's `channel` parameter as an unannotated optional string, while routing still only accepts registered provider ids/aliases. The system prompt documents valid provider names, but the tool schema itself still does not describe or constrain `channel`, so the reported mismatch remains plausible and the requested enhancement is not implemented on `main`.
