## Summary

Keep PR #71354 open. Current main still has the reported `tailscaleMode === "off"` short-circuit, so non-cold-start `openclaw status` cannot resolve a runtime Tailscale DNS name when config mode is off. The display layer already supports `off · <dns>` when a DNS name is available, and the PR's follow-up commit addresses the cold-start regression noted by reviewers. This is a focused core status bug fix, not implemented or superseded on main.

## Best Possible Solution

Keep the PR open and continue review. The best path is to land a focused version that removes the `tailscaleMode === "off"` scan-time short-circuit, preserves `skipColdStartNetworkChecks`, and adds regression tests for both normal and cold-start status scans. Maintainers should also make an explicit call on whether showing the local Tailscale hostname while config mode is off is intended status behavior, or require a safer/less revealing runtime-state signal.

## Risks / Open Questions

- The PR context shows no regression test was added; maintainers should ask for focused coverage that `tailscaleMode: "off"` still probes in normal status scans and skips during cold-start scans.
- Aisle's security comment raises a real product/privacy tradeoff: non-cold-start status would execute a PATH-resolved `tailscale` binary and may print the tailnet hostname even when config mode is off. That should be accepted explicitly or mitigated before landing.
