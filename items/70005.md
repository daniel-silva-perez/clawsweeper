## Summary

Keep #70005 open. Current main has useful mitigations for the reported noise class, including filtering heartbeat/cron scaffolding and blacklisting `assistant` from REM theme tags, but it does not implement the requested semantic-importance material selection. REM reflections are still selected from recurring concept-tag counts, session transcript ingestion still records sanitized transcript lines without a semantic-density gate, and the documented deep ranking still includes frequency/recurrence as a first-class signal. The issue is recent, concrete, and no canonical related item in the provided context tracks the remaining semantic-selection work.

## Best Possible Solution

Keep the issue open and implement the remaining fix in `memory-core`'s material-selection layer: add a semantic-density/importance scorer before session-corpus/daily signals feed light/REM, downrank boilerplate and high-frequency role/scaffolding artifacts, upweight user decisions/preferences/conclusions/unusual events, and add regression fixtures with assistant-heavy/heartbeat-heavy transcripts plus one meaningful decision. Only add a public `dreaming.promptStyle` or selection-strategy knob if maintainers want that product surface documented in the memory config reference.
