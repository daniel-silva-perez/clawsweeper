## Summary

Keep this issue open. Current main still runs `/v1/chat/completions` through the normal agent ingress path with full prompt assembly, has no HTTP light-context/voice-mode contract, and has not landed the proposed stable `x-openclaw-session` header. The linked PR #69060 is still only a partial implementation candidate for session/QMD warm-cache behavior, not the larger 15-20K-token context problem.

## Best Possible Solution

Keep this issue open. Review and either land or close #69060 for the stable-session/QMD warm-cache subproblem. For the larger fix, add an explicit HTTP prompt policy in `src/gateway/openai-http.ts` and `src/gateway/http-utils.ts`, document it at `https://docs.openclaw.ai/gateway/openai-http-api`, and thread it through `buildAgentCommandInput`/`agentCommandFromIngress` so `/v1/chat/completions` can intentionally reduce bootstrap context, prompt sections, tools, skills, and memory behavior with regression coverage in `src/gateway/openai-http.test.ts`.
