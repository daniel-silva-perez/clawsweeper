## Summary

Keep open. This is a recent, coherent core exec-approvals feature request for argv/subcommand-aware allowlisting. Current public docs describe manual allowlists as resolved executable path glob matches, with basename-only entries ignored, which does not satisfy allowlisting `openclaw cron list` while prompting for other `openclaw` subcommands.

## Evidence

- **Issue is recent and actionable:** Issue #59786 was created on 2026-04-02 and asks for subcommand-level exec allowlist patterns such as `openclaw:cron:list` or `openclaw cron *`; it is not older than 60 days and has a clear security use case.
  - sha: [d6ef1fcf24ee](https://github.com/openclaw/openclaw/commit/d6ef1fcf24eea3a5985d653a3baf400e32aa1b75)
- **Docs describe binary-path allowlisting, not subcommand allowlisting:** The Exec approvals docs say allowlist patterns are case-insensitive glob matches that should resolve to binary paths, with basename-only entries ignored. The examples are path patterns such as `~/Projects/**/bin/peekaboo`, `~/.local/bin/*`, and `/opt/homebrew/bin/rg`. ([docs.openclaw.ai](https://docs.openclaw.ai/tools/exec-approvals?utm_source=openai))
