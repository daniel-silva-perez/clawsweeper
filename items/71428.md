## Summary

Keep open. Current main still implements only the host-config opt-in, not the requested manifest-owned `permissions.conversationAccess` contract. The Smarter-Claw workaround described in the issue is a concrete signal, but the missing piece is in OpenClaw's core plugin manifest/loader/security policy path, so this is not closeable as ClawHub or already implemented.

## Best Possible Solution

Add an explicit additive manifest permission contract, for example `permissions.conversationAccess`, normalize it in `src/plugins/manifest.ts`, carry it through `src/plugins/manifest-registry.ts`, and merge it into the loader/registry hook policy path while preserving the existing `plugins.entries.<id>.hooks.allowConversationAccess` config override. Add regression tests for manifest-only allow, no permission rejection, and config-vs-manifest precedence, then update the public plugin manifest and hooks docs.
