## Summary

Keep open. Current OpenClaw main still bundles `@clawdbot/lobster` 2026.4.6 and loads that embedded runtime in-process; the upstream Lobster runtime still contains the repeated Ajv `.compile()` call sites described in the issue, including `ask`, `llm.invoke`, resume validation, and workflow input validation. OpenClaw's own plugin schema validator has a cache, but that does not cover the embedded Lobster runtime path reported here. The independent reproduction comment strengthens the issue rather than making it stale or insufficient.

## Best Possible Solution

Keep #71148 open. The best path is to fix the embedded Lobster runtime by introducing content-stable validator caching or compile-once workflow/input validators for the reported schema paths, add focused regression or memory/perf coverage upstream where feasible, then bump `@clawdbot/lobster` in `extensions/lobster` and verify the bundled OpenClaw plugin still loads the published `@clawdbot/lobster/core` runtime correctly.
