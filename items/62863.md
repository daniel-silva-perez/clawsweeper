## Summary

Current main has partial edit-tool hardening, but it does not fully implement the issue’s requested four-part stability work. The remaining gaps are concrete enough to keep tracking: broader whitespace-tolerant matching, read keyword/line aliases, and user-facing guidance are not fully covered.

## Best Possible Solution

Keep the issue open, but narrow it before implementation: add a safe fallback matcher only after exact/current fuzzy matching fails, require a unique normalized match, add regression tests for spaces/tabs/blank-line variants, decide whether `read line=` and `read keyword=` belong in the core read tool or a separate follow-up, and document the preferred small-edit/apply_patch workflow on the public tools docs.
