## Summary

Keep this PR open. Current main has an adjacent partial fix that hides ACP-only `sessions_spawn` fields when ACP is unavailable, but it still does not implement the PR's strict-gateway simplification: there is no `simplifiedSchemaForGateway` option or equivalent property-stripping path, and the nested `attachments` / `attachAs` subagent schema remains advertised. The linked issue #66719 was closed as superseded by this PR, so this PR is the remaining canonical place to resolve or reject the schema-level design.

## Best Possible Solution

Keep this PR open for maintainer review or rework. The best implementation is a provider/deployment-aware `sessions_spawn` schema filter, likely connected to model compat or another explicit runtime tool-assembly option, that preserves current ACP availability hiding while stripping only the intended strict-provider-unfriendly properties and adding regression coverage for the activation path and final schema shape.
