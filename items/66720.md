## Summary

Keep this PR open. Current main has ACP-availability schema hiding, but it does not implement the PR's strict-gateway simplification or any equivalent activation path that strips `lightContext`, `attachments`, and `attachAs` from `sessions_spawn`. The linked #66719 is closed as superseded by this PR, so this remains the canonical place to review, rework, or reject the schema-level fix.

## Best Possible Solution

Keep this PR open for maintainer review or rework. The best path is a provider/deployment-aware `sessions_spawn` schema filter, wired from the model compatibility or runtime tool-assembly path that actually serves affected OpenAI-compatible gateways, while preserving current ACP availability hiding and adding regression coverage for the final schema shape and activation path.
