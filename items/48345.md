## Summary

Keep open. Current `main` still intentionally runs overdue recurring cron jobs during gateway startup catch-up, and the repo does not expose a `skipMissedRuns`-style config or per-job option. Existing restart controls only throttle catch-up volume (`maxMissedJobsPerRestart`, `missedJobStaggerMs`); they do not skip missed windows.

## Evidence

- **Startup path still executes missed jobs after restart:** `planStartupCatchup()` collects runnable overdue jobs, logs `cron: running missed jobs after restart`, and marks them for immediate execution. There is no branch for skipping overdue recurring jobs.
  - file: [src/cron/service/timer.ts:975](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/src/cron/service/timer.ts#L975)

  - file: [src/cron/service.restart-catchup.test.ts:79](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/src/cron/service.restart-catchup.test.ts#L79)
