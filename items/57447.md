## Summary

Current `main` still enforces `tools.sessions.visibility` for `sessions_send` before the A2A allowlist check, and sandboxed label resolution is still restricted to same-agent/spawned sessions. The reported cross-agent send behavior is therefore still present, not already fixed on `main`.

## Evidence

- **Visibility guard still blocks cross-agent send before A2A policy:** `createSessionVisibilityGuard()` returns `forbidden` for any cross-agent target when `visibility !== "all"`; the A2A enabled/allow checks run only after that early return.
  - file: [src/agents/tools/sessions-access.ts:209](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/agents/tools/sessions-access.ts#L209)
  - command: `nl -ba src/agents/tools/sessions-access.ts | sed -n '192,258p'`
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **`sessions_send` still uses that guard for send actions:** `createSessionsSendTool()` computes session visibility and calls `createSessionVisibilityGuard({ action: "send", ... })`, then returns the guard error directly.
  - file: [src/agents/tools/sessions-send-tool.ts:241](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/agents/tools/sessions-send-tool.ts#L241)
  - command: `nl -ba src/agents/tools/sessions-send-tool.ts | sed -n '233,255p'`
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **Sandboxed label lookup still rejects cross-agent targets up front:** When `restrictToSpawned` is active, `sessions_send` rejects a label lookup for a different `agentId` and also passes `spawnedBy` into `sessions.resolve`, which prevents cross-agent target resolution.
  - file: [src/agents/tools/sessions-send-tool.ts:123](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/agents/tools/sessions-send-tool.ts#L123)
  - command: `nl -ba src/agents/tools/sessions-send-tool.ts | sed -n '116,169p'`

- **Current docs still describe `sessions_send` as governed by `tools.sessions.visibility`:** The configuration docs say `tools.sessions` controls `sessions_list`, `sessions_history`, and `sessions_send`, and only `visibility=all` allows any session, with cross-agent access additionally requiring `tools.agentToAgent`.
