## Summary

Keep open. Current `main` partially covers Telegram reactions with inbound reaction notifications and outbound reaction prompt guidance, but it does not implement the issue as written: there is no separate `reactionGuidance` config, and the injected `## Reactions` prompt does not include the requested guidance for interpreting received user reactions.

## Evidence

- **Prompt section is send-focused only:** `src/agents/system-prompt.ts:881` adds `## Reactions`, but the text only covers when/how the agent should send reactions. It does not include the requested 'treat user reactions as soft CTAs' guidance or emoji interpretation examples.
  - file: [src/agents/system-prompt.ts:881](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/src/agents/system-prompt.ts#L881)
  - command: `nl -ba src/agents/system-prompt.ts | sed -n '881,902p'`
  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
- **Telegram hook uses `reactionLevel`, not a `reactionGuidance` option:** `extensions/telegram/src/channel.ts:675` resolves prompt guidance from `resolveTelegramReactionLevel(...).agentReactionGuidance`, and `src/config/types.telegram.ts:182` defines `reactionLevel?: "off" | "ack" | "minimal" | "extensive"`. There is no Telegram `reactionGuidance` config matching the issue request.
  - file: [extensions/telegram/src/channel.ts:675](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/extensions/telegram/src/channel.ts#L675)

- **Inbound reactions exist, but only as raw system events:** `extensions/telegram/src/bot-handlers.runtime.ts:978` enqueues `Telegram reaction added: ...` system events, and `docs/channels/telegram.md:629` documents only `reactionNotifications` and `reactionLevel`. That covers notification plumbing, but not the requested bidirectional prompt guidance behavior.

## Risks / Open Questions

- Main now partially overlaps the request, so the issue may need re-scoping rather than a straight implementation from the original text.
- If product intends `reactionLevel` to supersede a separate `reactionGuidance` setting, that decision is not explicit in current code or docs, so closing as implemented would be premature.
