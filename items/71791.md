## Summary

Keep PR #71791 open. Current main still stores WhatsApp group history with only senderJid and filters allowlisted supplemental group history using only that cached JID, while the inbound message path already preserves senderE164. The PR remains a narrow plugin bug fix with a targeted regression test for LID-backed sender history matching an E.164 allowlist.

## Best Possible Solution

Keep PR #71791 open for maintainer review and normal validation. If CI passes, the best path is to land the focused WhatsApp plugin fix: preserve optional senderE164 in pending group history, compare it during supplemental-context allowlist filtering, and keep the new unit test as the regression guard.
