## Summary

Current `main` already replaced the old hardcoded overload retry behavior with configurable global knobs under `auth.cooldowns`, including `overloadedBackoffMs` and `overloadedProfileRotations`. That makes the original dist-file patch workaround obsolete, but it does not fully implement this issue's requested shape: there is still no per-agent override surface in `AgentConfig`, and the exposed setting is a single fixed delay rather than a full per-agent backoff-policy object. Conservative maintainer cleanup should leave this open as only partially addressed.
