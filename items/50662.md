## Summary

Current `main` does not contain either requested behavior. `openclaw gateway restart` still probes with only `OPENCLAW_GATEWAY_TOKEN` / `OPENCLAW_GATEWAY_PASSWORD`, and embedded failover policy still rotates auth profiles before model fallback on `rate_limit` when a fallback is configured. This PR is not implemented on `main`, and the report remains plausible and well-scoped.
