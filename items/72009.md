## Summary

Keep open. Current main still hardcodes the reply typing indicator TTL at 2 minutes and only exposes typing mode/interval config. PR #72009 is a focused, relevant implementation that adds `typingTtlSeconds`, raises the default to 5 minutes, updates schemas/docs/tests, and addresses the Codex review note about session-over-agent precedence in `c9cd643d67c7d0c40a245f36484b2163eee53be6`.

## Best Possible Solution

Keep #72009 open as useful PR work. The best path is to finish maintainer review and merge or refine this focused implementation, then close/retire older duplicate typing-TTL threads such as #42105 if #72009 becomes the canonical shipped fix.
