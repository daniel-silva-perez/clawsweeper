## Summary

Current `main` does not implement this PR's Feishu status-line or deferred-close behavior. The dispatcher still closes the streaming card immediately on each `final`, does not wire the proposed tool/assistant/compaction callbacks, and the Feishu tests still assert the old close-on-final behavior.

## Evidence

- **Immediate close-on-final still on main:** `deliver()` still merges the final text and calls `closeStreaming()` immediately for active Feishu streaming sessions, which is the pre-PR behavior that can create one closed card per final payload.
