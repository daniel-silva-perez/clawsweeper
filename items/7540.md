## Summary

Keep #7540 open. Current main still does not subscribe the bundled WhatsApp/Baileys plugin to `call` events, and the config, typed config, plugin manifest, docs, and hook path only expose message/action behavior. Recent maintainer comments also explicitly mark the request as still valid and WhatsApp-plugin-owned.

## Best Possible Solution

Keep the issue open and implement the feature in the bundled WhatsApp plugin: add an opt-in channel/account setting if desired, subscribe to Baileys `call`, normalize and sanitize call payloads, apply the existing account and inbound access-control boundaries, expose contact activity through a documented hook/session-event/tool surface, and add focused config, type, docs, and tests.
