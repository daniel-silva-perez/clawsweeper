## Summary

Current main still has the reported restart/resume gap. Exec approvals are held only in a fresh in-memory manager per gateway process, missing/stale IDs still resolve to INVALID_REQUEST, session replay strips runtime details but does not turn approval-pending transcript content into an expired/error result, and restart recovery resumes generic tool-result tails without approval-specific repair. The prior ClawSweeper comment is consistent with the current main state; no canonical merged fix or duplicate was found in the provided related context.

## Best Possible Solution

Keep this issue open. The fix should live in the session replay/restart-recovery path or in an explicit approval-manager persistence contract: when a resumed transcript contains an exec tool result with status "approval-pending" whose ID is absent from the live manager, rewrite it to an expired/error tool result so the agent re-issues exec, or securely persist and re-register the pending approval with the original command binding metadata. Add a regression test that simulates gateway restart with an approval-pending transcript tail and proves the resumed turn does not keep resolving the stale ID.
