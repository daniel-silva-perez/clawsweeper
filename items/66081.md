## Summary

Keep this PR open. Current main still finalizes codex-cli runs from parsed stdout and does not request or read Codex's `--output-last-message` artifact, so the PR is not obsolete or implemented on main. It remains a plausible implementation candidate for #65074, but the review comments need to be addressed before landing.

## Best Possible Solution

Keep this PR open and either revise it or replace it with an equivalent patch that is scoped to the `codex-cli` backend: inject `--output-last-message` before the prompt argument, read a non-empty artifact as the final reply while preserving stdout for telemetry/fallback, keep cleanup inside the existing guarded flow, mask the artifact path in CLI argv logs, and add focused tests in `src/agents/cli-runner.spawn.test.ts` and `src/agents/cli-output.test.ts`.
