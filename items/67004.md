## Summary

Keep PR #67004 open. Current main still reads Feishu documents only through the docx SDK path, while the PR targets legacy `/docs/:token` Feishu documents that need doc v2/meta handling and direct REST calls. The PR is a focused Feishu plugin change with author follow-up addressing several review comments, so it is not obsolete, duplicate, or an external-only ClawHub item.

## Best Possible Solution

Keep the PR open and review it as a Feishu plugin feature. The likely best path is to land a scoped implementation in `extensions/feishu` that detects old Feishu `doc` tokens via the doc v2 meta API, reads legacy documents through authenticated REST calls, transparently redirects upgraded documents to docx tokens, adds regression tests, and updates the Feishu skill/docs and scope guidance if old-doc reads require additional permissions.
