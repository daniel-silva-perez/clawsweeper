## Summary

Keep PR #67652 open. Current main still does not preserve loose `media` alias attachments in the plugin-sdk reply-payload helpers; the requested behavior is not implemented or superseded on main.

## Best Possible Solution

Keep this PR open, or replace it with an equivalent maintainer patch, until main supports `media?: string | string[]` in reply-payload normalization and the shared outbound media helpers. The best implementation should normalize and trim alias values, preserve the first valid alias as `mediaUrl` for legacy single-media paths, treat blank media values as missing, update the helper parameter types/cast consistently, and add focused regression tests in `src/plugin-sdk/reply-payload.test.ts`.
