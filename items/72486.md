## Summary

Keep this issue open. Current main does not include Vietnamese in the Control UI locale type, lazy locale registry, generated locale bundle set, language picker source list, docs, or i18n tests. The request is a valid core Control UI localization task rather than a ClawHub/plugin cleanup candidate.

## Best Possible Solution

Add Vietnamese as a first-class Control UI locale in core: extend the Locale type and registry, add vi/vi-VN browser fallback behavior, generate the Vietnamese locale bundle and .i18n metadata through scripts/control-ui-i18n.ts, add the language label, update Control UI docs, and update i18n registry/translation structure tests.
