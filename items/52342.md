## Summary

Keep PR #52342 open. Current main still classifies inbound Feishu messages using only `chat_type`, so the reported `chat_type: "p2p"` plus `oc_` OpenChat/topic-group path is not implemented on main. The PR is not safe to auto-close as obsolete, but it does need maintainer review because current tests and outbound routing still preserve some `oc_`/p2p direct-chat assumptions.

## Best Possible Solution

Keep the PR open for maintainer review or ask the author to rebase and reconcile it with current `topic_group` support, the remaining direct-chat test assumptions, and the open outbound follow-up in #58537. The best product fix is a Feishu-owned chat classification helper used consistently by inbound handling, mention-forward detection, and outbound session routing, with regression tests for `p2p` + `oc_` OpenChat/topic groups and any verified real DM ID shapes.
