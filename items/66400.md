## Summary

Keep this PR open. Current main still has the TUI-side race for gateway/remote tool-first runs: `handleAgentEvent` drops agent events for run IDs that have not yet been registered by a chat event, and the current `AgentEvent` type used by the TUI does not expose `sessionKey`. The PR remains a useful implementation candidate, and its second commit addresses the bot reviewâ€™s `isLocalBtwRunId` and optimistic-pending-state concerns.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate. The best path is to land this approach or an equivalent TUI fix that accepts same-session early agent events via `evt.sessionKey`, preserves the `isLocalBtwRunId` and optimistic-pending bookkeeping from `handleChatEvent`, updates the TUI `AgentEvent` type, and adds focused regression coverage for a tool-first gateway event before the first chat delta.
