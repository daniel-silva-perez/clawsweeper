## Summary

Current main still does not implement the central request. Supervisor cancellation and timeout paths still send SIGKILL directly, and tests still assert immediate hard-kill behavior. Some adjacent drain/process-tree safeguards exist, but they do not replace the requested supervisor-level SIGTERM-first grace period.

## Best Possible Solution

Keep this issue open and implement a focused supervisor change: send SIGTERM first for supervised exec/CLI cancellation and timeouts, use a documented bounded grace period before SIGKILL, preserve child and PTY drain fallback behavior, and update supervisor/adapter tests to assert SIGTERM-first plus forced escalation. Coordinate with #71710 for agent-side signal handling, but do not close this supervisor-side bug as implemented yet.
