## Summary

Keep PR #71086 open. Current `main` still has the old mutating-tool warning behavior, so the PR is not implemented or obsolete. The PR is also not ready to close as cleanup: its current patch suppresses warnings based only on `hasUserFacingReply`, and an unresolved P1 review comment correctly identifies that this can hide real mutating-tool failures after partial or failed assistant output.

## Best Possible Solution

Keep the PR open and ask for a small revision rather than closing it. The best fix is to suppress stale mutating-tool warnings only when the turn both produced a user-facing reply and reached a successful terminal state. Ideally pass an explicit successful-completion flag from the run layer, where aborted/timed-out/prompt-error state is known, then add regression coverage for both cases: successful recovery suppresses the stale warning, while partial/error/interrupted turns still surface the mutating-tool failure. Also update the stale block comment above the warning policy.
