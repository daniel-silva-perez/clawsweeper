## Summary

Current main already implements this via the optional `active-memory` plugin: recall runs from a `before_prompt_build` hook on eligible user turns, builds its query from the latest prompt plus recent turns, and injects the recalled context before the main reply. The feature shipped in `v2026.4.12`.

## Risks / Open Questions

- This is implemented through the optional `active-memory` plugin, so the operator must enable that plugin for the target agent.
- Default `allowedChatTypes` is `direct`; Feishu group chats require explicit opt-in if that was the intended surface.

## Close Comment

Closing as implemented on main after Codex review. OpenClaw already ships the optional `active-memory` plugin, added in `v2026.4.12` (`CHANGELOG.md:748`), and its current implementation runs on the `before_prompt_build` hook for eligible user chat turns, building recall from the latest message plus recent turns before the main reply (`extensions/active-memory/index.ts:1964`). Feishu DM sessions route through normal direct session keys such as `agent:main:feishu:direct:...` (`extensions/feishu/src/bot.test.ts:360`), which the Active Memory chat-type logic treats as eligible direct chats (`extensions/active-memory/index.ts:854`). If the intended target is Feishu group chat, that is also supported via `allowedChatTypes`, but groups are opt-in (`docs/concepts/active-memory.md:36`).
