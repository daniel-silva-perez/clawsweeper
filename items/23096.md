## Summary

Keep open. Current main has the generic SecretRef env/file/exec architecture, but it does not include the Bitwarden/Vaultwarden docs, bw resolver script, tests, package inclusion, or a public secret-provider plugin seam that would make this safe to close as ClawHub-only.

## Best Possible Solution

Keep this PR open as the focused Bitwarden/Vaultwarden implementation candidate. The best path is to port it once #72548 or an equivalent manifest-first secret-provider plugin API lands, likely as an extensions/secrets-bitwarden plugin declaring secret provider ownership, exposing a createBitwardenSecretProvider factory, and carrying docs, tests, and package metadata. If maintainers choose the generic exec-wrapper route instead, the PR should be rebased and reviewed for docs/script/test/package alignment rather than closed as implemented or ClawHub-only.
