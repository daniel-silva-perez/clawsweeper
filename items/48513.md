## Summary

Keep open. Current `main` has partial pieces of this request, but not the requested dedicated novice-friendly model-management UI. Control UI exposes model configuration through the generic config editor and offers a dropdown-only session model picker, yet the code still lacks a specialized Settings → Models workflow with provider templates, grouped provider/model presentation, metadata display, and save-time validation of provider/model combinations.

## Evidence

- **Control UI docs still describe generic config editing, not a dedicated models manager:** The current Control UI docs list session-level model overrides and generic `config.get`/`config.set` schema-form editing of `~/.openclaw/openclaw.json`. There is no documented Settings → Models management surface with provider templates or model validation.

- **Models is just a generic config-form section:** The UI labels `models` as one section inside the generic config-form renderer (`AI model configurations and providers`), which supports the existing config editor but does not implement a bespoke provider/model management page.
