## Summary

Current main still does not implement the requested `memorySearch.local.maxThreads` control for local node-llama-cpp embeddings. The prior ClawSweeper comment correctly kept this open, and the same gap remains at `e962381dbf1ecdb1cd4784fdeb6c2e07ddd3435b`.

## Best Possible Solution

Keep this issue open. The best fix is to add a validated positive integer `agents.defaults.memorySearch.local.maxThreads` with per-agent override support, wire it through public config types, strict Zod schema, schema labels/help/generated metadata, resolved memory-search config, memory embedding provider create options, node-llama shim types, and `createLocalEmbeddingProvider`, then pass it to `getLlama` only when configured. Add focused tests for default omission, configured forwarding, invalid-value rejection, and public memory config docs.
