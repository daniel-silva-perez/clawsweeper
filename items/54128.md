## Summary

Current main at 7c0fdae9b95bfcc667af3ad341b63ecc194464e4 still does not implement `memorySearch.local.maxThreads` for local node-llama-cpp embeddings. The existing config, strict schema, resolved memory-search config, provider options, runtime shim, tests, and public memory docs still only carry `modelPath`, `modelCacheDir`, and `contextSize`; `getLlama` is still called with only `logLevel`. The prior Codex review comment in the discussion correctly kept this open, and the same gap remains on current main.

## Best Possible Solution

Keep this issue open. The best implementation is a validated positive integer `agents.defaults.memorySearch.local.maxThreads` plus per-agent override, wired through public config types, strict Zod schema, schema help/labels/generated schema, resolved memory-search config, embedding provider create options, the node-llama wrapper type, and `createLocalEmbeddingProvider`, then passed to `getLlama` only when configured. Add focused tests for default omission, configured forwarding, validation rejection for invalid values, and update the public memory config docs.
