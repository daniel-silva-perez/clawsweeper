## Summary

Keep this issue open. Current main has meaningful partial coverage, including persisted fallback notices, status-card fallback display, and cron delivery previews for list/show, but it still does not implement the central production operations asks: startup primary-model reachability remains warning-only, there is no compact agent-readable diagnostic surface for current/active model plus last provider/fallback error, and cron add still lacks dry-run/preflight plus fail-closed announce-target validation before persistence.

## Best Possible Solution

Keep this issue open, but narrow it into a production execution-discipline tracker: add a configurable startup/provider preflight that can live-probe the configured primary model and either block or loudly mark degraded startup; add a safe agent-visible diagnostic RPC/tool with selected model, active/responding model, fallback state, and last provider/fallback error category/code; and add cron add dry-run/preflight plus clearer announce delivery target validation or fail-closed preview semantics. Treat closed related fallback and cron issues as completed subwork, not proof that this whole tracker is done.
