## Summary

Keep #70605 open. Current main still forwards context-file strikethrough text into the system prompt; the PR targets a real unimplemented surface, but it needs maintainer judgment and safer parsing before it should land because context files are freeform Markdown.

## Best Possible Solution

Keep the PR open for maintainer review. If OpenClaw wants this behavior, implement it in the context-file prompt sanitizer with Markdown-aware or carefully scoped parsing, preserving code spans/fences and malformed delimiter cases, and add focused `src/agents/system-prompt.test.ts` coverage for markdown strikethrough, HTML tags with attributes, multiline blocks, tilde fences, and non-strikethrough `~~` text.

## Risks / Open Questions

- The PR’s regex-based approach is behaviorally sensitive: bot review already identified over-stripping risks around `~~~` fences and malformed or non-strikethrough `~~` delimiters, so this should not be landed as a blind cleanup.
- Removing struck-through text from user-authored bootstrap files is a product-policy choice as well as a token-saving change; maintainers should decide whether strikethrough means deleted instruction content in OpenClaw context files.
