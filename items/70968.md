## Summary

Keep PR #70968 open. Current main still builds the gateway install environment by merging durable/config/auth-profile managed values into the serialized service environment, then only adds `OPENCLAW_SERVICE_MANAGED_ENV_KEYS`; tests still assert those literal values are present. The PR addresses a real remaining bug, though its broad purge should be adjusted for service-owned keys such as PATH before landing.

## Best Possible Solution

Keep the PR open and revise it into a targeted gateway install-plan fix: keep `OPENCLAW_SERVICE_MANAGED_ENV_KEYS`, omit literal persisted values for managed `.env`/config/auth-profile secret keys from the serialized service environment, preserve generated service fields like PATH and gateway service metadata, and update the daemon install helper tests to cover both the stale-secret fix and the service-owned-key guard.
