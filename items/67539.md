## Summary

Keep this issue open. Current main has the Gemini TTS runtime and human-facing docs from the motivating merged PR, but it still lacks the requested provider-owned prompt-hint hook and active-provider prompt injection path.

## Best Possible Solution

Add an additive, backwards-compatible provider-owned prompt hint hook to `SpeechProviderPlugin`; have `extensions/speech-core/src/tts.ts` resolve only the active provider/config/model/target and append that provider's short hint when TTS is enabled; cover active-provider-only injection and `modelOverrides` gating in tests. Gemini can then own its square-bracket audio-tag guidance in `extensions/google/speech-provider.ts`.

## Risks / Open Questions

- The hook would expand a public plugin contract, so implementation should update SDK exports, plugin docs, and contract tests in the same change.
- Prompt hints must be gated by the resolved active provider and `modelOverrides` policy so disabled or unsupported controls are not advertised.
