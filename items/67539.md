## Summary

Keep this issue open. Current main has the Gemini TTS runtime behavior from the motivating merged PR and documents that Gemini square-bracket audio tags pass through, but the requested provider-owned prompt-hint hook is still absent. The active TTS system prompt remains generic and the `SpeechProviderPlugin` contract has no `buildPromptHint`-style seam for speech providers to add model/provider-aware runtime guidance.

## Best Possible Solution

Add an additive, backwards-compatible speech-provider prompt hint hook to `SpeechProviderPlugin`, have `extensions/speech-core/src/tts.ts` resolve only the active TTS provider/config/model/target and append that provider's short hint when TTS is enabled, and cover it with tests for active-provider-only injection plus `modelOverrides` policy gating. Gemini can then own the square-bracket audio-tag hint in `extensions/google/speech-provider.ts` while other providers omit or tailor the hook.
