## Summary

Keep open. Current main still does not implement MS Teams outbound `sendPayload`: the Teams runtime adapter and channel config expose text/media/poll only, while the shared outbound delivery path only routes interactive or channelData payloads to native handling when `handler.sendPayload` exists. Approval payloads still generate `/approve` interactive button descriptors, and Teams already has Adaptive Card send/poll primitives, so this PR remains a focused implementation candidate rather than obsolete cleanup. The linked #64690 is closed, but that closure is not supported by the checked-out main behavior at 7c0fdae9b95bfcc667af3ad341b63ecc194464e4.

## Best Possible Solution

Keep this PR open and review or land an equivalent Teams-plugin-owned `sendPayload` implementation in `extensions/msteams/src/outbound.ts` plus channel wiring in `extensions/msteams/src/channel.ts`. The implementation should preserve media handling, dep-aware sends, Adaptive Card `messageBack` behavior for approval buttons, and focused Teams outbound tests; after merge, #64690 can remain closed as implemented.
