## Summary

Keep this PR open. Current main still uses `schtasks` for the Windows gateway service lifecycle and still documents/tests the Startup-folder fallback, so the PowerShell `Register-ScheduledTask` behavior is not already implemented or superseded. The PR remains a plausible core daemon change, but it needs maintainer review and follow-up on the remaining review risks before merge.

## Best Possible Solution

Keep this PR open for maintainer review. The best implementation path is a focused Windows daemon update in `src/daemon/schtasks.ts` that makes PowerShell ScheduledTasks the primary path, sets the long-running and battery settings, preserves reliable uninstall cleanup of both Scheduled Task and Startup-folder artifacts, keeps a valid `schtasks` fallback for older environments, and adds targeted tests for PowerShell success, PowerShell-unavailable fallback, denied/non-admin behavior, restart, runtime parsing, and uninstall cleanup.
