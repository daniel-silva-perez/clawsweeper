## Summary

Keep #69648 open. Current main still does not recognize `MiniMax-M2.7` as a MiniMax VLM model: the VLM endpoint gate remains hardcoded to `MiniMax-VL-01`, and current tests assert that only `MiniMax-VL-01` on `minimax`/`minimax-portal` uses `/v1/coding_plan/vlm`. The default product path now routes MiniMax image understanding through the plugin-owned `MiniMax-VL-01` model, but that does not implement the issue’s requested behavior for `MiniMax-M2.7`; the directly related fix PR #69691 is still open/unmerged in the supplied GitHub context.

## Best Possible Solution

Keep the issue open for maintainer/product decision. Either land a focused fix that adds `MiniMax-M2.7` to the MiniMax VLM routing contract with regression tests, or explicitly close the product question by documenting and enforcing that MiniMax image understanding must use `MiniMax-VL-01` even when users configure M2.7 as image-capable.
