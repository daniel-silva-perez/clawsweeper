## Summary

Keep open. Current `main` still constructs Feishu post replies by embedding `<at ...>` HTML into a single `md` post element, so the PR's reported mention-rendering bug is not implemented on `main` and does not look obsolete.

## Evidence

- **Outbound send path still inlines mention HTML:** `sendMessageFeishu()` still prepends mentions via `buildMentionedMessage()` before building the post payload.
  - file: [extensions/feishu/src/send.ts:464](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/extensions/feishu/src/send.ts#L464)
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **Post payload is still md-only:** `buildFeishuPostMessagePayload()` on `main` only accepts `messageText` and emits one `{ tag: "md", text: messageText }` element; there is no native `at` element emission here.
  - file: [extensions/feishu/src/send.ts:432](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/extensions/feishu/src/send.ts#L432)
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **Mention helper still produces `<at user_id>` HTML strings:** `buildMentionedMessage()` still formats mentions as inline `<at user_id="...">name</at>` text, which matches the broken payload shape described in the PR.
  - file: [extensions/feishu/src/mention.ts:106](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/extensions/feishu/src/mention.ts#L106)

  - file: [extensions/feishu/src/reply-dispatcher.ts:455](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/extensions/feishu/src/reply-dispatcher.ts#L455)
