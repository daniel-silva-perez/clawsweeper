## Summary

Keep open. Current main still builds Feishu post replies by prepending inline `<at user_id="...">name</at>` text and then wrapping the whole result in a single `md` element; it does not emit native `tag: "at"` elements for outbound post messages. The linked issue #49833 was closed after a Codex review, but that close rationale treats the existing inline string markup as sufficient, which does not resolve this PR's stated native-post-element problem. Related PRs #49837 and #49850 are open alternatives, not shipped canonical fixes.

## Best Possible Solution

Keep this work open until OpenClaw ships a structured Feishu post payload path. The likely best implementation is to carry `MentionTarget[]` into `buildFeishuPostMessagePayload`, emit native `at` elements with `user_id` and `user_name`, keep the model-generated body literal, preserve any helper exports that are public, and add regression tests for no mention, single/multiple mentions, literal `<at>` text in the body, unusual display names, and existing card paths.
