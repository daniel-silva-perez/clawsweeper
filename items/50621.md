## Summary

Keep open. Current `main` still applies the cron outer timeout to `sessionTarget: "main"` `systemEvent` jobs, and the `wakeMode: "now"` path still awaits `runHeartbeatOnce` instead of treating the job as unconditional fire-and-forget.

## Evidence

- **Non-agent cron jobs still get a fixed outer timeout:** `resolveCronJobTimeoutMs()` returns `DEFAULT_JOB_TIMEOUT_MS` (10 minutes) for any non-`agentTurn` payload. Only `agentTurn` can use `timeoutSeconds` or disable the timeout with `<= 0`. That means `systemEvent` main-session jobs still have a hard cron timeout on current `main`.
  - file: [src/cron/service/timeout-policy.ts:8](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/cron/service/timeout-policy.ts#L8)
  - command: `nl -ba src/cron/service/timeout-policy.ts | sed -n '1,120p'`
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **`systemEvent` payloads still cannot configure timeout behavior:** `CronPayload` defines `systemEvent` as only `{ kind: "systemEvent", text: string }`, while `timeoutSeconds` exists only on `CronAgentTurnPayload`. The requested workaround or fix is therefore not present in the current public cron payload contract.
  - file: [src/cron/types.ts:112](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/cron/types.ts#L112)
  - command: `nl -ba src/cron/types.ts | sed -n '108,130p'`
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Main-session wake-now jobs still wait on heartbeat execution:** `executeMainSessionCronJob()` enqueues the system event, then for `wakeMode === "now"` it calls and awaits `runHeartbeatOnce()`. It only returns early in the specific busy-lane fallback (`requests-in-flight`) or after a short busy wait ceiling, not as unconditional fire-and-forget once the main-session turn starts running.
  - file: [src/cron/service/timer.ts:1206](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/cron/service/timer.ts#L1206)
  - command: `nl -ba src/cron/service/timer.ts | sed -n '1190,1285p'`
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Docs still describe main-session cron as system-event enqueue, not a long-running configurable agent turn:** The docs say main-session jobs enqueue a system event and optionally wake heartbeat, while `timeoutSeconds` is documented on the isolated agent hook surface. There is no documented no-timeout/configurable timeout path for main `systemEvent` cron jobs.
  - file: [docs/automation/cron-jobs.md:84](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/docs/automation/cron-jobs.md#L84)
