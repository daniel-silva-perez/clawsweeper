## Summary

Keep PR #49083 open. Current main still uses ordinary string inequality for both TLS fingerprint checks in `src/gateway/client.ts`, while the shared `safeEqualSecret` helper already exists and is tested for unequal and different-length inputs. The PR remains a narrow, relevant gateway security hardening change rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open and either rebase/merge the narrow change or apply the same implementation on main: import `safeEqualSecret` in `src/gateway/client.ts` and use it for both normalized TLS fingerprint comparisons, then run the gateway changed lanes before landing.
