## Summary

Current main does not implement the requested upgrade-safe Control UI chat message width customization. The chat group width is still hard-coded in bundled CSS, the renderer does not pass a width/style override, and the typed/validated config surfaces only expose Control UI hosting/security settings plus `gateway.webchat.chatHistoryMaxChars`, not a layout max-width or custom CSS hook. The prior mobile-width changelog entry addresses narrow-screen overflow, not this wide-monitor configurable max-width request.

## Best Possible Solution

Keep this issue open and implement a narrow, documented Control UI layout setting. The likely path is to add a typed config or browser-local Appearance field for chat message max width, validate/sanitize the CSS value, pass it through the Control UI bootstrap or UI settings layer, set a root/shell CSS custom property, update `ui/src/styles/chat/grouped.css` to use a `var(..., min(900px, 68%))` fallback, and cover the schema/runtime/CSS behavior with focused tests and docs.
