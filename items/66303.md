## Summary

Keep this PR open. Current main has not fully implemented the template cleanup: the HEARTBEAT workspace template still contains a Markdown fence and a docs-only Related footer, and the workspace loader still writes the post-front-matter template body directly into new workspaces. The prior runtime mitigation skips fence markers, but it does not make the current docs footer empty/actionless, and the related duplicate PR #70267 is closed unmerged.

## Best Possible Solution

Keep this PR open for maintainer review, but refresh it before landing. The best fix is to make docs/reference/templates/HEARTBEAT.md represent only the literal workspace scaffold, or to split docs rendering metadata from runtime template loading so docs-only fences and Related links are never written into user workspaces. Update src/agents/workspace.test.ts and heartbeat gating coverage so a newly generated default HEARTBEAT.md is clean and skipped as empty.
