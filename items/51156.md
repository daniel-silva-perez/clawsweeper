## Summary

Keep PR #51156 open. Current main still has the lower-level <final> filtering mechanism and run-level `enforceFinalTag` plumbing, but it does not implement the requested user-facing `agents.defaults.enforceFinalTag` config/schema/docs path, and interactive/cron entry points still do not enable the tag prompt/filter for non-tag providers like Anthropic/OpenAI based on config. The PR is a focused core bug fix for related open issue #25592, with useful discussion evidence including an independent local backport report that the approach fixed Anthropic plain-text reasoning leakage.

## Best Possible Solution

Keep the PR open and ask the contributor or maintainer to rebase it onto current main, preserving the core approach: add `agents.defaults.enforceFinalTag` to the strict config schema and TypeScript config type, document it on the public agent defaults docs page, enable both the system-prompt `reasoningTagHint` and subscriber `enforceFinalTag` for interactive and cron runs when configured, and add focused regression coverage for config-driven non-tag providers plus the cron path.
