## Summary

Keep open. Current main still has the Kimi Coding stream wrapper that rewrites parsed tagged tool-call output to `stopReason: "toolUse"`, PR #71274 was closed unmerged, and the related Moonshot fix covers the separate OpenAI-compatible Moonshot provider rather than `kimi-coding` on `anthropic-messages`. The issue’s proposed root cause is partly contradicted by existing tests, but the fresh concrete loop report is not proven fixed on main.

## Best Possible Solution

Keep #71273 open. The next maintainer step should be a current-main Kimi Coding reproduction using `kimi/kimi-code`; if it reproduces, add a regression around repeated identical tagged tool calls after tool results and fix the provider wrapper, replay policy, or default loop guard as appropriate. If it does not reproduce, document the exact current-main transcript and close as cannot-reproduce later.
