## Summary

Keep open. Current main has a persisted `agent_id`/`agentId` field, and `sessions_spawn` resolves the target agent id, but the native subagent task registration path still does not pass that id into `createRunningTaskRun`, so `task_runs.agent_id` can remain null while `child_session_key` carries the agent id.

## Best Possible Solution

Keep this issue open and fix the core task attribution path. Thread the resolved `targetAgentId` from `spawnSubagentDirect` into `registerSubagentRun`, extend the registration params as needed, pass it to `createRunningTaskRun`, and add a regression test that a cross-agent `sessions_spawn` produces a task with `agentId` populated. Also review ACP spawn task creation so `agent_id` semantics stay consistent across `subagent` and `acp` runtimes.
