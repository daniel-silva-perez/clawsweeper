## Summary

Keep this PR open. Current main still has the session-store behavior the PR targets: unresolved model context lookup falls back to DEFAULT_CONTEXT_TOKENS and that value is written into the persisted session entry. The PR is still a plausible implementation candidate, though it needs maintainer review for the stale-context review comments and unrelated branch changes before landing.

## Best Possible Solution

Keep this PR open as the implementation candidate. The best path is to narrow or split unrelated changes, address the existing review comments about stale contextTokens across model/provider changes and authoritative store merges, then land the session-store fix with regression coverage showing unresolved lookups do not persist DEFAULT_CONTEXT_TOKENS while resolved runtime/config values still persist correctly.
