## Summary

Keep #38945 open. Current main still has the same missing behavior at the moved memory-core paths: MMR tokenization remains ASCII plus CJK-only, and FTS query planning/building still omits combining marks and NFC normalization. Later work added CJK-specific MMR/FTS support and QMD chunking fixes, but it does not supersede this PR’s non-CJK Unicode and combining-mark tokenizer fix.

## Best Possible Solution

Keep the PR open and have the memory-core implementation port the narrow tokenizer changes to `extensions/memory-core/src/memory/mmr.ts`, `extensions/memory-core/src/memory/hybrid.ts`, and the shared keyword-search token regex if needed, with regression tests for combining-mark scripts and NFC/NFD normalization. This should be treated as a focused memory-core bug fix, not a ClawHub or duplicate cleanup item.

## Evidence
