## Summary

Keep #50979 open. Current main has an adjacent Slack formatting change in inbound metadata, but it does not add the PR's current `messageToolHints` guidance. The PR discussion also uncovered a real remaining product/code question: plain-text Slack sends are Markdown-converted, while Slack block payloads are forwarded as mrkdwn, so any prompt hint needs to be scoped more carefully before this can be closed or merged.

## Best Possible Solution

Keep the PR open for maintainer/product handling. The best path is to reconcile #50979 with the shipped inbound-formatting work from #59100, then add narrowly scoped Slack prompt guidance: standard Markdown for normal text replies/tool sends that go through `markdownToSlackMrkdwnChunks`, explicit mrkdwn expectations for Slack Block Kit `blocks`, and regression tests for both disabled/enabled interactive replies plus a block payload case.
