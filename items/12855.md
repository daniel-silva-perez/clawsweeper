## Summary

Keep open. Current `main` has a built-in `update.auto.*` auto-updater for package installs, but it does not implement the full request around configurable confirmation modes, explicit time-of-day/cron scheduling, timezone selection, or post-update channel notifications.

## Evidence

- **Config surface is narrower than the issue request:** `update.auto` on current `main` only exposes `enabled`, `stableDelayHours`, `stableJitterHours`, and `betaCheckIntervalHours`. There are no `mode`, `schedule`, `timezone`, `notifyAfterUpdate`, or `notifyChannel` fields.
  - file: [src/config/types.openclaw.ts:77](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/src/config/types.openclaw.ts#L77)

  - file: [docs/gateway/configuration-reference.md:3624](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/docs/gateway/configuration-reference.md#L3624)
