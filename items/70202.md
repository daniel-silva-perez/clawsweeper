## Summary

Keep #70202 open. Current main supports a primary model, configured fallbacks, an allowlist-backed `/model` picker, and post-onboarding configuration, but the onboarding wizard still prompts for one default model and does not expose the existing multi-select allowlist/fallback flow during onboarding.

## Best Possible Solution

Keep this open and implement the concrete onboarding gap by reusing the existing `promptModelAllowlist`, `applyModelAllowlist`, and `applyModelFallbacksFromSelection` flow after default-model selection in onboarding, with focused wizard tests and docs updates. Treat automatic task-complexity routing as separate provider/plugin work rather than part of this core onboarding issue.
