## Summary

Keep open. Current main still has the reported empty-catalog cache invalidation behavior: `loadModelCatalog()` clears `modelCatalogPromise` whenever `models.length === 0`, so the PR is not obsolete or implemented elsewhere. The PR is a focused implementation candidate for linked bug #66094, with regression tests and follow-up responses to the bot review comments in the provided discussion.

## Best Possible Solution

Keep this PR open and have a maintainer review or land it, or port an equivalent fix to current main: cache empty catalog results briefly, retry after a bounded window, keep import/filesystem failures retryable, make `useCache: false` bypass/clear the negative cache, guard stale concurrent loads, and retain focused tests in `src/agents/model-catalog.test.ts`.
