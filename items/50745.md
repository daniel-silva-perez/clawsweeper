## Summary

Keep open. Current main does not ship `lookupPnLidEntry`; it only has internal WhatsApp LID-to-E.164 resolution used by inbound normalization. The PR is still a plausible WhatsApp plugin/API change, but it needs maintainer review and likely a refresh against the newer central LID-resolution path plus the review comments about authDir fallback and changelog handling.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to refresh the PR against current main, add the listener method in the current `extensions/whatsapp/src/inbound/types.ts` and monitor return surface, reuse or extend the existing `resolveJidToE164`/authDir fallback logic for `@lid` and `@hosted.lid`, add focused listener tests, and fix the changelog to append rather than replace unrelated release notes.
