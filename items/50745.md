## Summary

Current `main` does not implement the PR's requested public WhatsApp `lookupPnLidEntry` API. The repo already has internal LID-to-E.164 resolution logic, including `@hosted.lid` and auth-dir fallback, but the exported `ActiveWebListener` surface still only exposes send methods, so this is not `implemented_on_main` and should remain open.
