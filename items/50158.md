## Summary

Keep open. Current `main` still does not implement the PR's requested schema-level validation for `gateway.nodes.denyCommands`. The config schema accepts `denyCommands` as a plain `string[]`, and runtime enforcement still does exact-name matching after trimming. `main` has added audit/docs warnings for ineffective pattern-like deny entries, but that is advisory and not the same as rejecting bad config at parse time, so there is not strong evidence for any close reason.
