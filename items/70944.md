## Summary

Keep #70944 open. Current main does not implement the PR's Discord identity-links-to-sender_principal path: Discord sender identity has no trusted principal field, preflight does not pass identityLinks into sender resolution, dispatch context does not include TrustedSenderPrincipal, inbound metadata does not render sender_principal, and the shared identity-link resolver is still private to session-key routing. The PR discussion is useful and active: bot review identified PluralKit attribution semantics, and the author updated the PR head to use pkInfo.sender for trust resolution.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase/update #70944 against current main, preserve the PluralKit pkInfo.sender fix from the review discussion, add the Discord sender-principal flow through preflight/process/native commands, expose a documented identity-link resolver through the plugin SDK if maintainers accept that public surface, and keep regression tests around sender identity and inbound-meta rendering.
