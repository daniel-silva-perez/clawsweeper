## Summary

Keep open. Current main has substantial adjacent usage infrastructure and shipped user-facing surfaces, but it does not implement this PR's requested root `openclaw usage` CLI command or equivalent CLI session/model breakdowns. The PR also has unresolved review findings, so the right maintainer path is rework or product-scope decision rather than automated close.

## Best Possible Solution

Keep the PR open for maintainer/product review. If OpenClaw wants this CLI surface, rework it as a thin command over the existing `usage.cost` and `sessions.usage` contracts, use the shared usage formatting/date parsing behavior, aggregate all in-range sessions before applying display limits, and address the existing review comments. If maintainers do not want a root command, close it explicitly as a product decision and point users to `openclaw gateway usage-cost`, `/usage cost`, Control UI Usage, and the remaining dedicated logging discussion in #13219.
