## Summary

This PR should stay open. Current main has not implemented the requested Discord behavior: a direct non-default accountId with no account token still falls back to channels.discord.token. The PR is a plausible implementation candidate, but the latest review context also identifies a remaining account-scan/default-account edge case that should be resolved before landing.

## Best Possible Solution

Keep this PR open and request a focused revision: make direct explicit non-default Discord account sends strict when the account token is missing, while preserving channel-token fallback for implicit default-account and account-scan/discovery flows. The final patch should include targeted coverage in the Discord token and account/action discovery tests.
