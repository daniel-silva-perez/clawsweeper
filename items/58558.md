## Summary

Keep open. Current main still defines `tool_result_persist` and `before_message_write` as synchronous-only hooks, and the runner explicitly warns and ignores Promise-returning handlers. The issue is recent, coherent, and supported by concrete plugin/governance use cases from the reporter’s Axonflow plugin and a commenter’s MemOS Cloud deployment, so it does not match any conservative cleanup close reason.

## Evidence

- **Hook type contract is sync-only:** `PluginHookHandlers.tool_result_persist` returns `PluginHookToolResultPersistResult | void`, while nearby hooks such as `message_sending`, `before_tool_call`, `after_tool_call`, and lifecycle hooks include Promise return types. `before_message_write` is also sync-only.
  - file: [src/plugins/hook-types.ts:729](https://github.com/openclaw/openclaw/blob/a7de722f4f5979d37f8809c9aaf6d9d8907d5e3f/src/plugins/hook-types.ts#L729)
