## Summary

Keep open. Current main still has the older generic `tools.profile` and `agents.list[].tools.profile` help text in both the source help map and generated config schema, while current docs already document the practical `messaging` vs `full` distinction this PR is trying to expose in config help. The related PRs in the provided context remain open and are adjacent docs/status surfaces, not a landed replacement for this config-help change.

## Best Possible Solution

Keep #39963 open until either a maintainer lands this config-help clarification or a canonical PR explicitly supersedes it. The practical implementation path is to update `FIELD_HELP` for `tools.profile` and `agents.list[].tools.profile`, regenerate the base config schema artifact, and coordinate wording with the related docs/status/troubleshooting PRs (#39954, #39996, #40039).
