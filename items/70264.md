## Summary

Keep open. Current main still matches the reported heartbeat-enabled gap for `runtime:"acp", streamTo:"parent"`: ACP terminal events are relayed into the parent session as system events and request a heartbeat wake, but the heartbeat runner still only builds actionable prompts for exec completions and cron events, while ACP wake reasons are not action-priority. The task terminal direct-delivery path also mirrors to the parent transcript without requesting a parent wake. A maintainer already triaged the issue as valid, and the current code still supports that triage.

## Best Possible Solution

Keep #70264 open as the canonical heartbeat-enabled ACP `streamTo:"parent"` parent-wake bug unless maintainers explicitly make #56442 or another issue the canonical tracker. The core fix should split lightweight progress relay from terminal completion delivery, likely by adding an opt-in ACP parent completion mode such as `parentUpdates:"notify"`/`wakeOwnerOnDone`, routing terminal ACP completion through the existing subagent announce/direct parent-turn path or by teaching heartbeat to build an ACP completion prompt. Add regression coverage for heartbeat-enabled `streamTo:"parent"` completion waking with an actionable ACP event, heartbeat-disabled behavior if the chosen design supports it, and default no-wake compatibility; then update the ACP `streamTo` docs/tool description.
