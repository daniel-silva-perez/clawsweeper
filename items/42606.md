## Summary

Keep this PR open. The proposed hardening is still valid, but current main has not implemented it and the PR patches the old `src/browser` location rather than the live browser plugin files under `extensions/browser`.

## Best Possible Solution

Port this narrow hardening to `extensions/browser/src/browser/bridge-server.ts` and `extensions/browser/src/browser/bridge-server.auth.test.ts`: generate a per-response script nonce, set a restrictive document CSP plus `X-Content-Type-Options: nosniff` and `X-Frame-Options: DENY`, and preserve the existing one-time token flow with the noVNC password only in the URL fragment.
