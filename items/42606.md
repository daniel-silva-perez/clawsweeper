## Summary

Keep this PR open. The hardening request is still valid on current main, but the PR targets stale `src/browser` paths while the live noVNC bridge now runs through the bundled browser plugin under `extensions/browser`.

## Best Possible Solution

Port the narrow hardening to `extensions/browser/src/browser/bridge-server.ts` and `extensions/browser/src/browser/bridge-server.auth.test.ts`: generate a per-response script nonce, return CSP plus HTML together, set `Content-Security-Policy`, `X-Content-Type-Options: nosniff`, and `X-Frame-Options: DENY`, and keep the existing one-time token flow with the noVNC password only in the URL fragment.
